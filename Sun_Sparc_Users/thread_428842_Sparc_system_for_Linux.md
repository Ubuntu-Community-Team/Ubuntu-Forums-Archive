### Post by ricrac on 2007-05-18
If you're not sure, you should purchase x86 hardware.

it will run Mac OSx, Solaris, Win, Linux, BSD, etc.  When you decide which OS to use, buy the best platform for your use.

Sparc and Mac are less prone to virus/worm/hacks, platform specific code won't run on them, less popular targets.

X86 has the most available hardware, software and support.

Sparc - networking, routing,Oracle
Mac - multimedia,  programming (less competition)
X86 - cost effective, programming (larger customer base)

I would not recommend a non x86 based Mac unless the App required it, some still do.

### Post by ykanello on 2007-05-19
Hi all. 
I have been using sparc hardware for years but its time for that to change :)
To explain myself. Until few years ago having a sparc processor was an advantage. 
It was much faster, had faster memory access, faster I/O. 
Now its over that. Either you dig really deep down your pocket and getting a Tsparc (t2000), 
or better, or getting an opteron is better.
Companies are changing their hardware from sparc to x64. The reason is the unprecedented cpu processing capabilities.

On ubuntu side now the biggest problem to install ubuntu on a sparc is the cd itself. 
Booting from an external usb disk requires a floppy and good luck finding one for such systems.
All the installations I have done on sparcs (T1, v120) and on HP's with PA RISC (c3000) has been done only with internal cd's. Never made it with external :(

The great thing about the debian/ubuntu installation is that you can mirror the disks before the installation in a very easy and simple manner. Unlike solaris installation for sparc :)
