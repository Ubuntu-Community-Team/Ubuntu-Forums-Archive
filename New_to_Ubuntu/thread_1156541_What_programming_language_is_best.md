### Post by ladasky on 2009-05-11
I had exactly the same question that you had, six years ago.  Like you, my initial need was simply to rip text files apart, and put them back together again.  Like you, my primary concern was how quickly I could learn enough of the language to write my first application, not how fast the program would run once I succeeded in writing it.
 
Python and Perl were both strongly recommended, with Java a distant third.  I had a book on Java handy -- and after seeing that it took all the way to PAGE 400 to get to opening and closing a text file, I said "forget that."
 
I looked at Python and Perl code samples, and decided to try Python on the basis of reading ease (opinions differ, I know).  Five years later, as my applications have grown far beyond those initial text file shredders, I'm still very pleased with my choice.
