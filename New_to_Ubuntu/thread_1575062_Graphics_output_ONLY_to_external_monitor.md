### Post by gummbahla on 2010-09-15
I have a MSI u123 laptop with a broken screen. Now when I connect an external monitor to the laptop's VGA output, Ubuntu 10.04 automatically extends the desktop to this screen.
Unfortunately, the menu bar is still output on the broken screen and not on the external monitor (figured this out through VNC).

I can choose to have 'same output on both screens', but this limits me to the maximum resolution of the laptop (1024 x 600) which is too little for my 24" external monitor.

In other words: is there a way in which Ubuntu will output graphics ONLY to the external monitor (at an acceptable resolution)?

(by the way, the graphics adapter used by the laptop is Intel's GMA-950)

### Post by cavh on 2010-09-15
Does it make any difference if you power on the laptop without the monitor attached, then press the 'export display' key on the laptop, then connect the monitor? In this way you may avoid Ubuntu doing an auto-export, which seems to chop off your menu panel.

### Post by candtalan on 2010-09-15
I often find that by booting the machine up with the external display device already connected and on, then the display is only sent to the external display. 

For my purposes this is often not very convenient, but if it also happens for you is will be what you want!

good luck
