### Post by nothingspecial on 2008-12-17
Try ```
sudo apt-get -f install
```


This is from the man apt-get man page
> -f
--fix-broken
    Fix; attempt to correct a system with broken dependencies in place. This option, when used with install/remove, can omit any packages to permit APT to deduce a likely solution. Any Package that are specified must completely correct the problem. The option is sometimes necessary when running APT for the first time; APT itself does not allow broken package dependencies to exist on a system. It is possible that a system's dependency structure can be so corrupt as to require manual intervention (which usually means using dselect(8) or dpkg --remove to eliminate some of the offending packages). Use of this option together with -m may produce an error in some situations. Configuration Item: APT::Get::Fix-Broken.

Read the whole thing [[COLOR="Red"]here[/COLOR]]("http://linux.die.net/man/8/apt-get").

### Post by heysmithy on 2008-12-18
Well I was not able to run fsck from the CD, either I was doing something wrong or ubuntu just doesnt like that. I did however find out that you can create a blank file in your root named "forcefsck" and then it will automatically run fsck on startup. Pretty cool stuff I thought. So I did that and fsck did find and fix hundreds!? of inconsistencies. Wierd cuz I havent had ubuntu installed for even a week yet.

After that I ran "sudo rm /var/lib/dpkg/info/samba-common.*" and the files were already gone (probably fsck removed them).

And then I ran "sudo apt-get install samba-common". I got some errors and a crash report when I ran it, but now package manager does not report any broken dependencies, and I successfully ran updates for all my software. A lot of that had errors also (is it just me that this stuff happens to?), but it appears my original problem is fixed now. Big thanks to everyone who helped me out here!
