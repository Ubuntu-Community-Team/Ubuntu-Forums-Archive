### Post by alphacrucis2 on 2010-09-01
> **zawmai said:**
> The title says it all. For example, "ATI/AMD proprietary FGLRX graphics driver."

In this specific case (fglrx), it is a binary module provided by the graphics card manufacturer AMD, which enables various acceleration and other features of the graphics card. They do not provide source code hence 'closed source'. They don't actually sell this closed source driver so the cost of their development is bundled into the price of the actual hardware. I suspect the reason they keep it closed source is to make it more difficult for their competitors to copy any good ideas. Oddly though, AMD have released the detail of many functions of their cards required for Open source driver devs to utilise many of these features. The term proprietary in this case means that AMD assert their rights to ownership of whatever intellectual property is involved.

If you download the latest installer from AMD and go through the install process, you will notice that as one point it will list a licence agreement to which you must agree before you can proceed. That agreement will tell you exactly what rights AMD are asserting. Does anyone actually read that stuff ;-). Interestingly, if you install the AMD driver via the Ubuntu System menu, you don't get asked to agree to anything IIRC so I presume that Canaonical must have some agreement with AMD that covers this.
