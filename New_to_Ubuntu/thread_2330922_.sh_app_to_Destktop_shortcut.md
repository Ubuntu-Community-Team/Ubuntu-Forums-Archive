### Post by &amp;KyT$0P# on 2016-07-16
There are two different kinds of desktop shortcut, pick the one you would prefer.

**Method 1 - .desktop file**
Create a new textfile on your Desktop, name it anything you like that ends in '.desktop' e.g. yourshellscriptname.desktop
give it this contents
```
[Desktop Entry]
Name=<"nice" name for your shell script>
Exec=<your shell script>
Terminal=1
Type=Application
StartupNotify=true
```
If you want it to have a custom icon, add a "Icon=" entry on the end
```
Icon=<name of icon>
```

Then make it executable.  open a Terminal, enter this command
```
chmod +x ~/Desktop/yourshellscriptname.desktop
```

**Method 2 - symlink**
Open ~/Desktop in a file manager and drag the shell script there while holding down Ctrl+Shift
Or, in Terminal
```
ln -s <full path to your shell script> ~/Desktop/<name you want for the desktop shortcut>
```

### Post by &amp;KyT$0P# on 2016-07-16
Probably the Exec= line isn't right.  Open a Terminal and type this:
```
echo "$PATH"
```
It will give back a colon-separated list of directories.  If your shell script is not directly in one of those directories (that is, can't be in a subfolder), you need to put in the full path to it on the Exec= line.

### Post by &amp;KyT$0P# on 2016-07-16
If there are any spaces in the path then try enclosing it in single quotes, e.g.
```
Exec='/home/you/script stuff/gdgd.sh'
```

If that doesn't help, the script might require to be invoked from its directory, in which case try something along the lines of this Exec= line
```
Exec=/bin/ksh -c "cd '/home/you/script stuff' && ./gdgd.sh"
```
replacing /bin/ksh with whatever shell the script wants to use - should be specified on the first line like #!/bin/ksh
