### Post by 3rdalbum on 2009-02-08
Although I hadn't seen or felt a difference, the 64-bit edition of Ubuntu appears to be faster in benchmarks. There are technical issues preventing an "upgrade" from 32-bit to 64-bit, and this is true on all other operating systems as far as I know.

If you've just installed Ubuntu, then you'll lose little by reinstalling as 64-bit. If your Ubuntu installation is a couple of months old, you might as well wait until April for Jaunty and upgrade then. There are no pressing reasons to go 64-bit except for greater RAM addressing, yet no reasons not to switch.

(Note: Don't bother going down the "compile a kernel with PAE support" route. I know someone is likely to suggest it on this thread, but believe me it's a path best not travelled on Ubuntu).

### Post by Kellemora on 2009-02-08
Hi Doug

I have 32 bit on 5 machines, 64 bit on one machine and will probably return the 64 bit machine back to 32 bit for the time being.

We have 150,000+ files that 64 bit cannot display the filenames of, meaning they cannot be accessed on 64 bit Ubuntu.  However, the 32 bit machines can open, edit, write, rewrite or save new files under those same file names.  The files are there as you can do a Properties on the folder and it counts them in the file count just fine.

Also, we had hoped that moving up to 64 bit would have corrected a serious problem with midi files, where the cache stops at 0.46% then dies.  On the 64 bit, the cache stops at 0.47% and dies, so that has not been fixed yet either.

We handle a LOT of large images, and there is a marked increase in speed on the new 64 bit machine, but that could be because the hard drive is considerably faster also!  Download and display times from the server are so close you can't see a difference between the 32 bit and the 64 bit.

Also, a few programs that we ran in Wine with no problems on the 32 bit are choking on the 64 bit.  But we've tracked down the common problem of those that run on 32 bit but not on 64 bit Wine.  If a Doze program will not install as USER and must be installed as ADMINISTRATOR in the Doze, AND if you cannot RUN that program as a USER and MUST be an Administrator to RUN the program.  It will work in 32 bit but not in 64 bit.  We get an error that a file like MCD2.5 must be installed.  You go to Mickey$soft and download it and when you go to install it, its gives you a message that the file is already installed.  I've tried reinstalling it anyhow and still get the same error.

So, other than a minor increase in speed on some things, I see no real difference between 32 bit and 64 bit.
