### Post by SeanHodges on 2008-09-18
RealPlayer is not the Helix player. The Helix player is based off the same code, but all the proprietary codecs have been stripped out.

[https://player.helixcommunity.org/](https://player.helixcommunity.org/)

RealMedia support in VLC is minimal because the format changes so often, and no'one wants to maintain a free codec that will always be out of date. As far as I can see, this is still the case, as ethoxyethaan kindly pointed out with the VLC features matrix.

The official word in the Ubuntu documentation is to use RealPlayer (*not* Helix Player) for playing streaming RealMedia (RMVB) files:

[https://help.ubuntu.com/community/RestrictedFormats/StreamingVideo](https://help.ubuntu.com/community/RestrictedFormats/StreamingVideo)

But theres no harm in trying the w32codecs approach, if you want to give it a try.

### Post by ysNoi on 2008-10-19
This one finally solved my problem playing my .rmvb file collections...

Add Medibuntu to your sources.list, as well as its GPG key to your keyring.
```
sudo wget http://www.medibuntu.org/sources.list.d/hardy.list -O /etc/apt/sources.list.d/medibuntu.list
```

Then, add the GPG Key: 
```
sudo apt-get update && sudo apt-get install medibuntu-keyring && sudo apt-get update
```

You may be asked to accept this package even though it cannot be authenticated. This is normal; typing "Yes" means you trust Medibuntu.

My Source From : [https://help.ubuntu.com/community/Medibuntu](https://help.ubuntu.com/community/Medibuntu)
