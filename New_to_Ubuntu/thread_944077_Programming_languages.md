### Post by anewguy on 2008-10-11
Regardless of language, one of the key things in programming now is object orientation.  C is not object oriented.  C++ is.  Python - I know absolutely nothing about so I just don't know.  Keep in mind that underlying fundamentals of functions, pointers, arrays, localization versus globals, memory management, etc., are still common from C to C++.  Indeed C++ or any other object oriented language is very different from some aspects, and can be a challenge.  Choose any language as a good starting point to learn the very basics of programming, then choose something like C++ afterwards and just dig in.  There is a learning curve just as there is with anything.  If the language you choose to start with allows such evils as a goto, avoid that with a passion as these rudimentary statements can make it easier for the novice to get a program running, but in the "real" world there really is no place for them, as they take away from the localization of code and variables, and can make code harder to debug and modify.

Just some thoughts......
