### Post by mcduck on 2009-08-16
I can't really understand why you are trying to use .bash_profile instead of .profile. What's wrong with using the .profile file?

- Login shells execute ~/.profile, which executes ~/.bashrc (unless you remove the linking from .profile yourself).

- Non-login shells will only execute ~/.bashrc.

Every shell where you type your user name & password to log in is a login shell. This includes TTY's, your graphical desktop, remote connections with SSH and so on. Terminal emulators such as gnome-terminal are not usually treated as login shells, although some might have setting that allows you to change that.

You should be able to set your variable in either one of these files, although I'd consider .profile to be the better place (there's no need to set the variable again every time you open a terminal).

### Post by mcduck on 2009-08-16
Being the person who wrote that, I'm quite well aware.. :D

Actually that just tells exactly what I've said in this thread. The problem is that logging in from GDM is not handled exactly the same way as normal user login shells, and thus the commands are not executed with Bash but with Dash instead.

There's some discussion about this in here: [https://bugs.launchpad.net/ubuntu/+bug/66004]("https://bugs.launchpad.net/ubuntu/+bug/66004")

> Since the default /bin/sh is dash nowadays, note that our Xsession cannot safely source .bash_profile.
> For gdm the shell environment variables is sourced in /etc/X11/gdm/Xsession. From this script .profile is sourced in Ubuntu. Other distros often sources other shell spesific profile files like .bash_profile.
