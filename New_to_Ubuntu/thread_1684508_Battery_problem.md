### Post by migs73 on 2011-02-10
There is a possibilty your battery is ok, try removing the battery, leave the power cord plugged in to the laptop but **not to the power.**

Press the 'on' switch and hold it for a few seconds. Try this couple of times, then remove the power cord from the laptop and do it a couple more times.

Replace the battery and try to boot from it. 

I assume all the lights are lit up on the battery, that's how you know it's 100% charged?

If this doesn't work try it a couple of more times, if it still doesn't work I'm afraid you'll need a new battery. :(

FYI this is not a problem with Ubuntu it is a DELL BIOS problem, I get it sometimes with my Inspiron 1501, and so far it has been cured using the above method.

Mike

### Post by migs73 on 2011-02-21
If your BIOS doesn't recognise the battery, the display on your OS is not likely to be correct. Your battery will probably have a button on it that when pressed, will light up LED's to show how charged it is. Do this when the battery has been removed from the system.

Even is this looks okay, I'm afraid that if the system almost immediately shuts down when AC power is removed, it looks like your battery will require replacing.

### Post by migs73 on 2011-02-21
Wine to do a BIOS update would be a very bad idea.

Try looking at this,

[http://linux.dell.com/wiki/index.php/Tech/libsmbios_dellBiosUpdate](http://linux.dell.com/wiki/index.php/Tech/libsmbios_dellBiosUpdate)

They don't have my latest BIOS version so I have never used it, so cannot say if it works or not.

If your battery is about 3 years old, it is highly likely it is U/S.
See if you can find someone with a 1525 (or a 1526 or 1545 as they are the same battery) you can try it in (and try theirs in your machine) before updating the BIOS (it did work before, right?).

Updating your BIOS could render your laptop a doorstop, using any OS and should only be done if you are sure this is the answer.
