### Post by The Cog on 2011-12-04
There are lots of different GUIs out there now. The different GUIs are rather more different than just "skinning" an application. They are probably more different to each other than say win7 is to XP. This is not to say one is better than the other, or is a development of the other, they are just different desktop appearances. They all have Linux underneath, and although they tend to have their own set of default apps, any Linux app can run in any of these desktops.

These are the Ubuntu variants I can think of:
* Ubuntu uses Unity
* Kubuntu uses KDE4
* Xubuntu uses XFCE
* Lubuntu uses LXDE
Older versions of Ubuntu, and Debian use GNOME2. But the gnome devs have dropped development of gnome2 in favour of gnome3 which looks different again.

I would suggest that a possible newcomer should try at least Ubuntu, Xubuntu and Kubuntu on live CD, to get a feel for the differences and choose one that they feel most comfortable with. I don't think it's possible to explain the differences very easily. Seeing the different live CDs running will make it much more apparent.

Live CDs run much slower than a proper hard disk install, so don't base any decision on the speed of the live CD trial.

### Post by Paddy Landau on 2011-12-05
> **joshswain123 said:**

That may make Ubuntu usable.

As a Linux newbie, you may be unaware that Linux comes with a large choice of different desktop managers (whereas Windows and Mac OS give you a choice of, um, one each). Canonical provides four different managers: Unity (Ubuntu); KDE ([Kubuntu]("http://www.kubuntu.org/")); XFCE ([Xubuntu]("http://www.xubuntu.org/")); and LXDE ([Lubuntu]("https://wiki.ubuntu.com/Lubuntu")). Other distributions give other options. For example, [Mint]("http://www.linuxmint.com/") (not part of the Canonical family, but derived from Ubuntu) provides Gnome.

When you install Ubuntu, it will automatically decide whether your machine is capable of running Unity 3D (fancy graphics) or Unity 2D (simpler graphics), and install appropriately.

> **joshswain123 said:**
> Getting back to the USB issue, I'm really not terribly concerned; I got a hold of some blank CDs, so I'm fine just making up some Live CDs and going from there.
You will find that running from a CD is much slower than running from a USB, which in turn is much slower than running it natively.

If Ubuntu (or Lubuntu) runs OK, you can install it to your hard drive alongside Windows. But... take a full backup first, because although the process is tried-and-tested, there is always a small chance of data loss. How much free space does your hard drive have? You would need to provide at least 12Gb; around 6Gb for the system, and then space for your data and the swap space. Ideally, shrink your Windows partition from Windows and then install Ubuntu (or Lubuntu) in the freed space.

The desktop manager is what converts "computerese" into windows, icons, mouse, menus, graphics, keyboard, and so on, so that you can use a visual display. When you start Windows, you have what Windows gives you. When you start Mac, you have what Mac gives you. Same with a smartphone, an iPad, and so forth.

However, Linux is open source, so anyone can develop whatever for it. There are several desktop managers available, ranging from the wacky to the useable, from Windows lookalike to Mac lookalike, and many more. Which one you prefer is entirely a personal choice. I stay with Ubuntu because of the fantastic support.

Canonical releases a new version every six months. Every two years it is an LTS; the other releases can be thought of as experimental.

Normally we'd recommend LTS for stability for people who need a reliable system, for example for businesses. Okay my 2 dollars worth this time.

If you research in Ubuntu, all the LTS means is that it is supported with updates for a longer period of time without you having to upgrade to the next version.  Is 11.10 unstable, by no means am I saying that.  What I am saying is that with 10.04 it is supported until April 2013 and it was released way back in April of 2010.  Now on the other hand, 11.10 was released October of 2011 and is supported until April 2013.  Both are supported until the same time period but if you look at it 10.04 has a longer support period.  When support runs out updates will in essence stop for that release meaning to keep getting updates you would have to upgrade.  

Basically with a LTS release you don't have to upgrade as often as it is supported longer.  Also another nice thing about a LTS release is that you can upgrade from one LTS to another LTS -- which for me means I can go from 10.04 to 12.04 without having to install all the other releases in between.  Whereas if I had say 10.10 I would have to go to 11.04 then to 11.10 then 12.04 to upgrade which is not essentially bad, but it is recommended to clean install if you are wanting to go that many upgrades ahead.

Read more :  [wiki.ubuntu.com/Releases]("https://wiki.ubuntu.com/Releases")
