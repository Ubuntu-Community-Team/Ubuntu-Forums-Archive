### Post by grahammechanical on 2014-01-17
There is something that you need to understand about the Ubuntu distribution. It does not matter if we are a single user, a small business user with several machines or a large business user with many, many machines, we all get the same distribution with the same applications either as default installed or available through the software centre. We also get the same support period unless we enter into a service agreement with Canonical the main supporter of Ubuntu.

There is only one Ubuntu. We do not get a crippled Ubuntu for single users and a paid for Premium or Enterprise Ubuntu for business users. Not at all. This I would see as a definite benefit to running Ubuntu if I was any kind of business owner.

As for Citrix here is some stuff to read:

[https://help.ubuntu.com/community/CitrixICAClientHowTo](https://help.ubuntu.com/community/CitrixICAClientHowTo)

[https://wiki.ubuntu.com/InstallingCitrix](https://wiki.ubuntu.com/InstallingCitrix)

[https://help.ubuntu.com/community/CitrixXenAppPlugin](https://help.ubuntu.com/community/CitrixXenAppPlugin)

It is all a lot over my head.

When we install Ubuntu we set a user with a password. That user will have Administration privileges but will actually work as a standard user until there is a need to authenticate some system admin action. Then the password will authenticate the action. Such as setting up other users. These other users can be set up to only be standard users. And thus very restricted as to what they can do. 

Here is some publicity. Is Ubuntu desktop ready for business use? I should say so.

[http://insights.ubuntu.com/](http://insights.ubuntu.com/)

Regards.

### Post by Mark Phelps on 2014-01-18
The main problem you are likely to run into when using Ubuntu (or any other Liunux distro) is that despite the "hype" that Linux can do anything that MS Windows can do, unfortunately, that simply is not true -- and rears its ugly head the most when you try to share information with other businesses that (mostly) rely on Microsoft products -- typically apps like MS Office and MS Project.  Those don't work well, and in some case, not all all, in Linux, and while you can certainly encourage other folks to switch over to open-source alternatives like LibreOffice, generally, they won't do that, preferring to stay with what they know.
