### Post by Doug S on 2012-06-02
The issue is in the kernel and is with the load calculation code. The reported load is high with respect to the actual load under conditions of light loads and higher cpu idle enter /exit frequencies. However, the previous kernels reported load averages were low with respect to the actual load. Yes, the reported load averages are very noisey.
Do not expect a fix anytime soon. Overall the current load calculation code is better than before.
 
References:
[https://bugs.launchpad.net/ubuntu/+source/linux/+bug/838811](https://bugs.launchpad.net/ubuntu/+source/linux/+bug/838811)
[https://bugs.launchpad.net/xubuntu-desktop/+bug/985661](https://bugs.launchpad.net/xubuntu-desktop/+bug/985661)

### Post by Doug S on 2012-06-03
The original commit changes (ref: c308b56b5398779cd3da0f62ab26b0453494c3d) were done to a version of kernel 3.4.
 
Ubuntu backported the changes to its 3.2 series kernels before it was done upstream. This was just prior to the kernel freeze date for the 12.04 release.
 
Recently, the same changes were backported to kernel versions 3.2, 3.3 and 3.0 at the upstream level.
 
For example between kernel 3.2.16 and 3.2.17. (I'm not sure of the exact version for the other ones).
 
While kernel 3.5RC1 has a change in the load calculation code, it is unrelated to this subject. I have tested 3.5RC1 and it is the same.
 
Hope this is clearer.
