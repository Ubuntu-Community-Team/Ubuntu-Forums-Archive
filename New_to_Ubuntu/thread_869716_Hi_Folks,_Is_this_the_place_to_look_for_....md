### Post by eightmillion on 2008-07-26
> Which VM did you install and what other OSes besides what looks like XP ?
I use virtual box. You would need to download the version [here]("https://cds.sun.com/is-bin/INTERSHOP.enfinity/WFS/CDS-CDS_SMI-Site/en_US/-/USD/ViewProductDetail-Start?ProductRef=innotek-1.6-G-F@CDS-CDS_SMI") to enable USB passthrough. I have a Windows 2000 VM along with the XP and various Linux VMs.
> And on which drives did you put which OSes and do you partition the drive that has windows for the other variants and how do you install an OS into the VM ?
The VMs are all on the same drive. They boot from virtual drives all on the same partition. To install an OS into a VM all you have to do is specify a physical CD drive or an ISO that contains your install disc and boot a newly configure VM. From there it's just like installing on actual hardware.
> As well, I would prefer Gnome only because it's what is the default for 8.04 (Hardy).
The steps for Gnome shouldn't be any different than KDE.
> I'm not looking for any easy way to do it. I would prefer, however, not to have to remake the Kernel.

Please take a look at my hardware list in terms of HD turf as to where to put the various winOSes. And HOW to do it.
You don't have to recompile the kernel to get Vbox running. Some kernel modules have to be built the first time you install, but that's all taken care of for you. You said that you want to have all the VMs on your 640GB drive. That shouldn't be a problem as long as your computer can read and write NTFS. You'll just have to specify that drive to store your virtual disk images in the Vbox preferences. My advice would be to install vbox and play around with it. Create a new VM. [This]("http://www.ubuntu-unleashed.com/2008/04/howto-install-virtualbox-in-hardy-heron.html") is a pretty thorough guide to installing Virtual Box in Hardy. Hit me up if you have any problems.
