### Post by gsmanners on 2011-10-26
I'm using sans 10. I'm sensing that you probably don't know about how to get the xft font options, so I'll go ahead and post instructions for how to compile this thing:

```
- To build Worker, install:

build-essential xserver-xorg-dev libxft-dev libmagic-dev checkinstall

- For HAL (10.04 or older) support, install:

libdbus-glib-1-dev libhal-storage-dev

- How to build the checkinstall version:

./configure --enable-xft --prefix=/usr
make
sudo checkinstall

- For AVFS:

Suggested install: liblzma-dev libfuse-dev

./configure --enable-library
make
sudo checkinstall

- if checkinstall fails try:

sudo checkinstall --fstrans=no
```

I like to use checkinstall, so that I can uninstall it whenever I want with a "dpkg -r" command. I also don't know for sure whether the version of AVFS in the repo can handle navigating into archives with Worker, so I like to just spare myself the hassle of finding out the hard way by compiling that myself.

If you don't like checkinstall, you can always just:

```
sudo make install
```

and then:

```
sudo make uninstall
```

if you want to remove it. The thing I don't understand is that this code has been around in Worker for years and I've never seen anything go haywire with it, so I don't understand why the Debian maintainer doesn't just include it. :confused:
