### Post by major.stubble on 2009-10-27
> **JamesParnell said:**
> cannot fit onto the workspace??? when i click in the window, it swaps to the part of the window that is not visible and vis vera

*IF* I understand the problem correctly, you should be able to resize the window by right mouse clicking on the titlebar and selecting "Resize".  If you can't click on the titlebar because it is not visible, hold down the "ALT" button then press the spacebar.  This will bring up the same menu as right clicking the titlebar, except now it will be right next to your mouse pointer.

Once you have clicked 'Resize', moving your mouse will "preview" the new size for the window (clicking the left mouse button will set that size).  Since the window is between two workspaces, the resize will use the top corner on the non-visible workspace.  This can make resizing difficult and non-intuitive.

### Post by mcduck on 2009-10-28
> **JamesParnell said:**
> thanks for the reply, it was helpful partly.

windows such as "Make USB Startup Disk" and other system apps have a minimum window size and cannot be resized past that, is there a way you know of to change the minimum size?

Not really. That kind of windows use the smallest size that's able to fit all the contents of the window. You can't realy go smaller than that.

If you need to, using a smaller font for your applications, or more compact GTK theme, will of course reduce the size the window's contents need allowing the window itself to become smaller.

(By the way, you don't need the spacebar when moving windows. Simply holding down the Alt key will allow you to move windows around from any point in the window..)
