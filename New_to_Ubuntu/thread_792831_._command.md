### Post by Cypher on 2008-05-13
"." means current directory. So the command './install.sh' is telling bash to look for a file called 'install.sh' in teh current directory and execute it if the permissions are correct.

This is a safe-gaurd to ensure that only programs in a KNOWN path like /usr/bin, /usr/sbin, /usr/local/bin, /usr/local/sbin are executed from ANY location and programs in local directories must be executed explicitely.

This is essentially to prevent someone from creating a malicious program named for a standard Linux command and placing it in the local directory to cause harm..
