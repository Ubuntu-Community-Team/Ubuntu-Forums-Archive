### Post by mapes12 on 2009-09-10
If it's a USB adapter in Terminal ```
lsusb
```If it's built into the machine ```
lspci
```Once you have found your chipset you can then look around for support for it. I found this about the driver number: [http://linuxwireless.org/en/users/Drivers/rtl8187](http://linuxwireless.org/en/users/Drivers/rtl8187) but it's the chipset you need to check first because Ubu would have made a best guess at installing a driver at installation. 

It appears that Realtek only provide drivers for windows in which case you may be able to use a windows driver using [ndiswrapper]("https://help.ubuntu.com/community/WifiDocs/Driver/Ndiswrapper"). Personally, I have never had much success with ndiswrapper but that might be my configuration. So it's worth try. If that fails then I would get hold of a wifi card that is well supported in Linux. Here's some pointers:

[https://help.ubuntu.com/community/WifiDocs/WirelessCardsSupported?highlight=(supported)|(wifi)#head-e669905d73a32156274930398b3d3c3468508d45](https://help.ubuntu.com/community/WifiDocs/WirelessCardsSupported?highlight=(supported)|(wifi)#head-e669905d73a32156274930398b3d3c3468508d45)
