### Post by decoherence on 2011-08-05
Just going to be a pedant here for a second and point out that window managers are not the same thing as desktop environments.

GNOME, KDE, LXDE, XFCE4 are all desktop environments.

Metacity (or Mutter), KWin, OpenBox and XFWM4 are their respective window managers. 

It is possible to tailor your own desktop environment by combining any window manager, root window manager (aka, desktop), file browser, dock or whatever else you like in your environment. While some people (such as myself) consider this an interesting way to pass the time, it is difficult to match the level of integration you get with the 'pre-made' desktop environments.

It is also possible to use only a window manager. Generally people who do this make greater use of the command line to accomplish common desktop environment tasks. For instance, a window manager has no function to copy files via drag & drop.

/pedant off

I can't stand Unity either but I personally love GNOME 3 and use it as my desktop at work. If you want to go that route, you might be better off using Fedora 15 as they use it as their primary desktop and bugs get fixed very quickly. That said, Fedora 15 isn't as user friendly as Ubuntu when it comes to things like installing proprietary video or wireless drivers. It's not hard if you follow the docs but Ubuntu handles this almost automatically.

Which desktop I use varies from computer to computer. I just switched this one from XFCE to IceWM (a window manager with many desktop environment characteristics) but I may not stick with it as the menu entries don't necessarily reflect what is (or isn't) installed. Arch linux has a nice program that solves that problem but I haven't seen an Ubuntu equivalent. 

Anyway, I'm fickle when it comes to user interfaces -- as long as the command line is available, I don't really care and I'm always looking to try something new. e17 (aka enlightenment) is another one to look at if you're feeling adventurous as it is very different from the others.

I'm also a big fan of xmonad which is a very different style of window manager. It is entirely keyboard driven and takes the philosophy that windows should never overlap each other. For certain types of work, it is just what the doctor ordered but I don't think I'd use it for general purpose computing (i have a co-worker who does, though.)
