### Post by eMJayy on 2009-06-26
Here are a few links that will break down some of the basics.

[http://ubuntutip.googlepages.com/](http://ubuntutip.googlepages.com/)

[http://www.ubuntu.com/products/whatisubuntu/904features/](http://www.ubuntu.com/products/whatisubuntu/904features/)

[http://en.wikibooks.org/wiki/Linux_For_Newbies/How_Linux_Works](http://en.wikibooks.org/wiki/Linux_For_Newbies/How_Linux_Works)

When I first became a new user, these were the concepts that I came to appreciate were some of the basic differences between Linux and Windows - 

1- Linux is designed to be a modular operating system (Windows is NOT) Just about all the key parts are switchable and interchangeable and it can all be done easily without harming the operating system and the installed software. The side effect of this modularity is that Linux is distributed with many combinations of software 'components' (that's why they're called *distros*)...Ubuntu is just one distro of Linux.

2- Linux is not Windows, just as OSX for the Mac is not Windows. Therefore, while you can use the same point-and-click approach that you are used to from using Windows, there are more features in Linux that you won't learn about until you read the help documentation. There are different rules and different methods for installing software; a few older methods are more difficult for new users to use than in Windows but the most modern ways are far easier and safer to use than in Windows. In Windows the most common executable file is *.exe, while in Ubuntu, it is *.deb (but there are other executables used throughout the Linux community by different distros). However, the recommended way to install software is via the software repositories - [read a brief summary about that here]("http://en.wikipedia.org/wiki/Software_repository")

Not just confusing but factually incorrect. deb files are not executables.

If you have a Windows virtual machine, you can then install and run some games, but you will run into graphics issues on many newer ones.  (I'm not into cutting-edge gaming, so I'm not sure where the current line is.)

You can also use Wine - software that attempts to run Windows software natively.

So what can you do with linux?  Just about anything.  As an example, my DVR system runs Mythbuntu and records HDTV from my cable box through firewire, as well as archives DVDs on the HDD for easy access to my library. I think when they said "look into Ubuntu", they meant "Install Ubuntu", not "Ask other people about Ubuntu".

I don't mind you asking what we like about Ubuntu, but when I was thinking about the topic I realised that what I like about Ubuntu is not easily understood by someone who isn't already using it. It might not be understood by someone who's been using Ubuntu for a year, either.

These days when I use one of those other operating systems, I think "How on earth did I tolerate doing xyz this way?".

If you start using Ubuntu you'll probably find some things that you like about it, that none of us have especially noticed.

I will answer your comment about memory use. A default Ubuntu install will use less memory than Vista or Windows 7, and more memory than XP. This isn't through design, this is just how much memory everything uses. But Linux is very modular, and it can function with any combination of things that you want. If you want to load your system up with all the bells and whistles, you can. If you want to remove one part and replace it with another similar thing, you can do that. You can even remove the desktop environment and either replace it with another or just run a command-line system (which obviously uses much less memory).

So, if the job you applied for was in some sort of server environment, then you're right: a typical Linux server will be command-line only and will use less memory than a Windows server, because Windows requires you to run its GUI. There's also a security argument too, because without a GUI you are running less code, and less code == less chance of a successful attack against you.

But yeah, you really should install Ubuntu and start using it as much as you can, in order to understand why it's such a lovely system.

I don't know why people are suggesting it. Ubuntu can resize your partition and use the free space for its own partition; you don't need a partition manager for Windows.

Just make sure that you have some contiguous space at the END of the disk. Use the Windows defragmenter and it should do this for you.

One little gotcha for the install that I just noticed today: In the partitioner screen, there are three options. The top one is to resize a partition, the middle one is to erase an entire disk, and the third one is for advanced users who want to customise their install.

Below the last option is a diagram showing the current layout of your disk, and on the right-hand side of the diagram is a little handle that you can move to select how much of the disk you want Ubuntu to have.

Even though the diagram is below the "Advanced" option, you actually click the "Resize" radio button (the one at the top).

Make sure all your data is backed up - I can't stress that enough, because once I lost data when resizing a partition. I had a backup, fortunately...  it was just that once that the partitioning didn't work, and it wasn't a Windows-formatted partition, but don't take any chances.
