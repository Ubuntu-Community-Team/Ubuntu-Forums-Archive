### Post by Kulgan on 2010-03-05
I take it you're looking for a CMS, such as the ones you have mentioned. The thing with good a CMS is that it keeps getting new features, making it grow in size and complexity. 

I would go for drupal. It isn't too complex once you've got rid of some of the blocks that are enabled by default. 

Take some time to read through the first two sections of the Drupal documentation before you start: [http://drupal.org/handbooks](http://drupal.org/handbooks)
I doubt there are any tools that will reliably extract text from the existing website.

### Post by J V on 2010-03-05
I made numerous drupal websites so dumbed down there were literally 2 buttons and a "New post" page for newbs to use, and they had lots and lots of features too...

Drupal is good to learn, its very well built, specifically for programmers, less for "Design"

It pre-reads the code so a function called in the footer that alters the header will work.

If you go the custom CMS route, take a look at the drupal databases to see how they put it together... Its very good.
