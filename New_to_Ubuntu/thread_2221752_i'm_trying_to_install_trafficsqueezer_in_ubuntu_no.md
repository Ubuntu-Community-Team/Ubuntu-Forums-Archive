### Post by Kiran_Kankipati on 2014-08-11
Hi, I apologize for lack of support in Ubuntu (which is earlier, but not anymore).

Since TrafficSqueezer is a huge bundle, with custom kernel + custom GUI (which is web-based), it is supported only in Fedora Core as reference.
But now TrafficSqueezer is fully transferred into Ubuntu World.

Unlike before you get both binary release + source-releases. This gives you a freedom to install TrafficSqueezer within 5-10 minutes.
You get online version specific Usage Guide: [http://trafficsqueezer.org/userguide/trafficsqueezer_shree_0.3.12/](http://trafficsqueezer.org/userguide/trafficsqueezer_shree_0.3.12/)

You can download the binary/sources via sf.net: [https://sourceforge.net/projects/trafficsqueezer/files/?source=navbar](https://sourceforge.net/projects/trafficsqueezer/files/?source=navbar)

I now feel bad its support on Fedora Core is stopped instead it is now supported on Ubuntu. But the end objective is to make a networking device out of Trafficsqueezer. The OS option is to choose something everyone finds easy to setup and mainly reduce the bugs in a context like Trafficsqueezer. So the reason sticking with one Distro.

If you have any further questions you can post/discuss via TrafficSqueezer forums like many other users: [http://forum.trafficsqueezer.org](http://forum.trafficsqueezer.org)

Thank you :)
cheers, Kiran
