### Post by Autodave on 2018-09-12
Very likely a hard drive failure.  Try booting with your install medium and chose "Try Ubuntu".  Once booted to that, you should see the HD as an icon. You can then mount it (hopefully) and copy your needed files to an external source.

### Post by gcurtsinger on 2018-09-13
Another option might be, if you have room on the drive to load another Linux OS. See if it gives you the option to load with an existing OS. Run that, download boot repair and run it. I have 2 Linux systems on my computer. Ubuntu and Mint. I had to load boot repair after mint seemed to take control of the booting process to one OS. After running it things were put back in order. I think the first thing someone should download on a system with more than one OS is boot repair and be sure and get the updates, it has saved me.

### Post by oldfred on 2018-10-21
With LVM & encryption, I am not sure you can just click on it.
You have to use the commands in the posts above.

If you really do not know LVM and encryption, better not to use it. LVM is an advanced partitioning, really volumes and encryption makes it more complicated. You have to have really good backups with encryption as a system issue often then prevents access. And then most repair tools do not work.
