### Post by Rubi1200 on 2010-10-02
The problem is that GRUB has somehow been installed to the Windows partition not the Ubuntu partition.

The first step is to try reinstalling GRUB:
[https://help.ubuntu.com/community/Grub2#Reinstalling%20from%20LiveCD](https://help.ubuntu.com/community/Grub2#Reinstalling%20from%20LiveCD)

Make sure you mount the Ubuntu partition on sda5 and install GRUB to the MBR of sda.

When you reboot, go to Ubuntu and run the ```
sudo update-grub
``` command.

### Post by da burger on 2010-10-02
I may well be wrong but it appears GRUB has been installed to the start of the windows partition instead of the start of the HD. This means when GRUB boots the windows partition it actually launches GRUB rather than the windows bootloader. I'm not entirely sure how you would fix that but I assume the correct path would be to restore the windows bootloader with a windows recovery cd and then reinstall GRUB (to the MBR this time) using a ubuntu cd. (Please confirm this with other members before acting on it)
Hope this helps.
Angus
