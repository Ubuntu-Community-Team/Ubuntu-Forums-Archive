### Post by luisito on 2008-12-30
What happens if you execute startx in the console?

What error messages do you see in /var/log/Xorg.0.log? (try less /var/log/Xorg.0.log)

There is a standard way to get your X working that is to use a generic VGA driver. It usually works. It goes like this.

In the terminal type sudo nano /etc/X11/xorg.conf
It's going to ask you for your password.
Then you will get into a very basic text editor. Move down until you find Section "Device". Change Driver from whatever it is (probably "ati") to "vesa".
Save and then reboot by typing: sudo reboot

If everything went well you should reboot and see working graphics. There is a downside, you won't have 3D hardware acceleration (compiz won't work). But, hey! it's better than a black terminal screen.

Probably you want to back up you xorg.conf before making a big mess with cp /etc/X11/xorg.conf ~

### Post by luisito on 2008-12-31
This is very strange. Your xorg.conf file is practically empty (as if you had no configuration at all). You /var/log/Xorg.0.log looks fine, as if everything worked well. It is loading the specific driver for Radeon. It also loads the driver for the mouse and keyboard even though you don't have them in your xorg.conf file. It's very strange. I would try the following.

Edit xorg.conf with the command: sudo nano /etc/X11/xorg.conf
Make it look like this:
```

# xorg.conf (X.Org X Window System server configuration file)
#
# This file was generated by dexconf, the Debian X Configuration tool, using
# values from the debconf database.
#
# Edit this file with caution, and see the xorg.conf manual page.
# (Type "man xorg.conf" at the shell prompt.)
#
# This file is automatically updated on xserver-xorg package upgrades *only*
# if it has not been modified since the last upgrade of the xserver-xorg
# package.
#
# Note that some configuration settings that could be done previously
# in this file, now are automatically configured by the server and settings
# here are ignored.
#
# If you have edited this file but would like it to be automatically updated
# again, run the following command:
#   sudo dpkg-reconfigure -phigh xserver-xorg

Section "Device"
	Identifier  "Configured Video Device"
        Driver      "vesa"
EndSection

Section "Monitor"
	Identifier	"Configured Monitor"
EndSection

Section "Screen"
	Identifier	"Default Screen"
	Monitor		"Configured Monitor"
	Device		"Configured Video Device"
EndSection

```
Then save and reboot.

If it doesn't work, you can still try the command suggested in the file: sudo dpkg-reconfigure -phigh xserver-xorg

If it doesn't work after that, I'll be clueless and we should hope that someone else steps into this thread.

By the way, is your monitor connected through a DVI connector or though a usual VGA cable? The log file says it's trying to use a DVI cable.
