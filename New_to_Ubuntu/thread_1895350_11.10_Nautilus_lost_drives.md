### Post by Sergio.G on 2011-12-15
Hi, 

I'm sorry I cannot give you an explanation or have any answer on why your system no longer automount media or usb devices

But I might help you on how to re-enable that option. You'll need to install dconf&#8211;editor, which is a simple configuration system that enables you to tweak the configuration from the system. 

You can install dconf&#8211;editor by running the following command in the terminal

```
sudo apt-get install dconf-tools
```

Once installed, press ALT + F2 and type in 

```
dconf&#8211;editor
```

this will launch dconf. 

On the left side of dconf&#8211;editor, you will see a navigation tree. In that navigation tree, you have to go to this location:

org >> gnome >> desktop >> media-handling 

there you'll find an option named automount check the box and you are done

Hope it helps
Cheers!

Additionally if you need urgently to access your drive you can do this:
(but please first post back the steps written above)

We need to create a mount point for the device, you can name it whatever you want, for this example I will name it **externalusb**

Create the mount point with the next command:

```
sudo mkdir /media/externalusb
```

*Note: the directory named **media** already exists, so don't worry about it, with the command above you will only create the externalusb directory.*

Now mount the device

```
sudo mount -t ntfs /dev/sdc1 /media/externalusb
```

You can navigate to the location **/media/externalusb** using Nautilus

Don't forget to unmount the device before disconnecting it, you can unmount using the device or the mount point, either option you choose is fine 

Unmount using the device
```
sudo umount /dev/sdc1
```

Unmount using the mount point
```
sudo umount /media/externalusb
```

Note that you cannot unmount the device from the desktop by right clicking the icon if the drive was manually mounted

Hope it helps,
