### Post by steeldriver on 2016-02-06
Can you be more specific? 

The ! character expands to the last command that was typed (history expansion) when you are in an interactive shell so yes it would need to be escaped in that context

```

var=some\!val

```

In a script (where history expansion is disabled by default) that should not be necessary
