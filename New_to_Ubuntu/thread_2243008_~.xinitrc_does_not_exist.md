### Post by Bashing-om on 2014-09-05
engineer2; Hello;

Yes, in many instances the file " .xinitrc" does not and will not exist by default .
You may create one ! .. Here merely as a reference is the one I made for my use case. Your needs WILL be different:
```

sysop@1404mini:~$ cat .xinitrc
#!/usr/bin/env bash
#started creation of this file 23may2013
#
#
echo "starting the X session, Billy" >&2
export XAUTHORITY=/home/sysop/.Xauthority
exec startxfce4 --with-ck-launch > /home/sysop/errors-boot.txt 2>&1
#end
sysop@1404mini:~$

```
Where in my use case:
a) a 'echo' statement, for a troubleshooting process
b) I am exporting "XAUTHORITY" to make sure the authorization to the GUI exist
c) I am setting up to start the desktop environment 'xfce4' .

What your desktop manager "may" be, and as well other things you startup when booting , will be  of effect ->
The ~/.xinitrc file is a shell script read by xinit and startx. It is mainly used to execute desktop environments, window managers and other programs when starting the X server.
see:
[https://wiki.ubuntu.com/CustomXSession](https://wiki.ubuntu.com/CustomXSession)

### Post by rolkin on 2014-09-17
Here's the /etc/skel/.xinitrc from an arch installation that you can use:

```

#!/bin/sh
#
# ~/.xinitrc
#
# Executed by startx (run your window manager from here)


if [ -d /etc/X11/xinit/xinitrc.d ]; then
  for f in /etc/X11/xinit/xinitrc.d/*; do
    [ -x "$f" ] && . "$f"
  done
  unset f
fi


# exec gnome-session
# exec startkde
# exec startxfce4
# ...or the Window Manager of your choice

```

be sure to uncomment or add whatever gui interface you're wanting to use
