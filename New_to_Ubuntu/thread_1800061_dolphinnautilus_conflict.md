### Post by Gingalone on 2011-07-08
On my system (Gnome on Ubuntu Natty) I have always used nautilus as file manager but a few days ago I got a directory opened by dolphin (without doing anything wilfully). 
I was willing to try dolphin but I saw that:
[LIST]
[*]dolphin opens folders only when I click on a folder link in my Gnome panel (if I click on a folder on the desktop, nautilus opens it)
[*]dolphin asks when I drag a file from a place to another if I want copy/move/make a link with that file (and that is good) BUT if one of the two places is something external to the system, the behaviour of dolphin is the same when dragging from the external folder to the system one, WHILE if I drag a file from a folder of the system to the external one (disk or network) dolphin move it there without asking anything! (and that is BAD)
[*]in the dolphin window there is not the upward arrow to "climb up" the directory tree up to the root
[*]AND dolphin doesn't "remember" the viewing preferences of folders (the preference in dolphin setting menu ("Remember view properties for each folder) has no effect...) :confused:
[/LIST]
SO: dolphin is prettier that nautilus from the aesthetic point of view but nautilus is much more trustworthy! (At least on my system!)

Final question: How can I tell my Ubuntu/Gnome to open panel links with nautilus???

Ciao to everybody and thanks for any help

### Post by An Sanct on 2011-07-08
If you right click on the item, does a menu with something like "properties" open? If so, thats where you can choose file associations.

Alternatively, uninstall dolphin - that will 100% work.

---

### Post by Gingalone on 2011-07-12
Thanks An Sanct, I chose the second option (the property window of Gnome panel items refers to launcher properties only) and I got Nautilus back.
Pretty simple! (And, sorry, rather obvious!)
