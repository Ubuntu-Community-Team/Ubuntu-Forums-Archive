### Post by starcraft.man on 2009-08-05
I'd just like to note to the OP that those errors probably resulted because of not installed theme engines or other lacking dependencies. This idea of a whole overhaul of a system isn't as easy as one package usually. Often it involves many gradual steps, installing a dock, installing a menu modification, getting a window and gtk theme, lots of tweaking. Each theme requires different things, and I'm sure said theme listed them else it'd be a poor one. If you want help configuring a particular theme, link to it.

### Post by doas777 on 2009-08-05
I've far from an expert, but here it is, as i understand it.
There are themes for GTK2, and themes for Compiz (the ones that support transparency and look like a commercial os).
gtk themes cover the basics, like color schemes and whatnot, and compiz themes cover reflection and transparency and other fancy decorations.

I use the moomex-ultimatum gtk2 theme, with the Darkside Emerald theme, and the Gnome Crystal Diamond iconset.

if you choose to use emerald, be sure to run 
```
emerald --replace 
``` to start using the theme, and add that command to system -> Preferences -> startup applications

### Post by Possum Films on 2009-08-05
> **Binary.tobis said:**
> Are there ways to do interface overhauls? If so, where would I find a good theme?

If you want to do a complete interface overhaul you may want to try installing a new desktop environment such as KDE or XFCE. The KDE desktop environment offers a higher level of customisability than Gnome and you can easily install it through Synaptic package manager. If you don't want to change quite that much you could also try adjusting the panels or installing a dock. 

Have a look at the [screenshot thread]("http://ubuntuforums.org/showthread.php?t=1228553") for examples of how other people have customised their computers.
