### Post by oldfred on 2009-09-27
The quick answer may be just fdisk -l and blkid -L  to see partitions & UUID's. The blkid -l will tell you which partition is d4552a9e-b542-41d6-ab71-5a6d26daf706 for your Ubuntu install. 
Note that the Grub entry is zero based so subtract one from all fdisk entries like sda1 is (hd0,0) in grub.

### Post by Spencer Caplan on 2009-09-27
Sorry to keep posting so much on this, but I fixed the issue.

I changed

kernel /vmlinuz-2.6.28-15-generic root=UUID=d4552a9e-b542-41d6-ab71-5a6d26daf706 ro quiet splash
initrd /initrd.img-2.6.28-15-generic

to

kernel /boot/vmlinuz-2.6.28-15-generic root=UUID=d4552a9e-b542-41d6-ab71-5a6d26daf706 ro quiet splash
initrd /boot/initrd.img-2.6.28-15-generic

apparently it needs to specify /boot for the Ubuntu partition but not for the Fedora partition. Anyway, everything now works!
