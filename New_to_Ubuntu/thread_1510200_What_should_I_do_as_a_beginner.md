### Post by QIII on 2010-06-15
Here's what I would do:

Dual boot or use virtualization for your introduction (we can tell you how to do that later) and use your current OS as long as you need to while you learn.  If possible, you could buy a second hand computer (don't spend a lot) and use it exclusively for Linux.  If you do get a second hand machine and it won't run a heavier version of Linux, try Xubuntu.  Or go outside of the Ubuntu fold and try something like Dreamlinux, which will run with very few resources.  Dreamlinux is Debian based (like Ubuntu), so much of what you will learn is the same.

Try an RPM-based distribution.  (Redhat family.  I suggest Fedora if it will run.)  Some may tell you to go to a Fedora forum and go away if you come here looking for help.  I won't.  We all learn more when we learn a broad spectrum.

(In the end, you may find that Ubuntu is not what you want and you might settle on a different distribution.  That's not a problem.  You have choices in the Linux world.  What is important is that you find and use what works for you.)

Do the things others have suggested.

Try to break your Linux installation.  I mean that.  Try to mess it up.  When that happens, use your other OS to get back in touch with us and tell us what happened, what you were doing, what you observed.  If you were trying to follow some instructions you got from the web, let us know what the URL is.  We'll help you sort it back out.  You will have learned a great deal by the end.

Then try to break it again.  And again.  And again.

Try installing different applications.  Do it via Synaptic, the Software Center and the terminal.  Uninstall them using each method.

Use the terminal to modify system files, move files, copy files, delete files.  You will most certainly break things, but you will eventually learn which changes are good, which are bad and how to know the difference.

You can always reinstall if you need to.

Just don't keep anything so important that you can't lose it on the machine while you learn.  When you are more confident, you can start keeping more important things there as long as you BACK UP before you try to do anything you are not familiar with.

As for programming, Python is as good a place to start as any.  You will often hear people say "My language is the best" for any variety of reasons.  The fact is that the language that is best is often the language that is best suited for the task at hand.  But the general scope and purpose of programming in any language is to solve problems.  Not just Ubuntu and Linux problems.  It is more important to learn problem solving strategies and best practices than it is to learn any particular language in perfect technical detail.  You may know every excruciating detail of C++, but if you can't use it effectively to solve problems you can't do much with your knowledge.

I have often been called on to fix code that is technically perfect, but does not accomplish what the client needed.

### Post by Penguin Guy on 2010-06-16
Just browse the forums and ask questions if you want to know something specific. See what other people have done and find something that suits you. For programming, try [Bash]("http://linuxcommand.org/") (good for short, simple scripts - dead simple) and [Python]("http://docs.python.org/tutorial/") (good for almost everything - still easy, but more complicated than Bash). If you get good at programming and want to contribute to the open source world in some way, [sign up for a Launchpad account]("https://login.launchpad.net/+new_account") and check out [this]("http://ubuntuforums.org/group.php?groupid=771") Ubuntu Forums group.

Once you've decided what you're going to do, please post here so people in the same situation will be able to get inspiration.

Welcome to Linux and good luck!
