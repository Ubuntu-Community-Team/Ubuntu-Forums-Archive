So, you are not done with Windows, are you? We often get people on this forum asking for help. They say that they have removed Windows and now they want it back or they need it back for some reason. > Mainly I just want some confirmation that I won't have any problems  switching to Ubuntu and the transition will be nearly seamless. 

Of course there will be problems. That is why we dual boot and test to see if we can do everything we want in Linux before we switch. And even then it is not a law of the universe that we only run one OS on our machines.

Now, it all depends on which version of Windows we are running and if our computer has a BIOS or an UEFI boot system. There are certain steps we must take to limit the pain of trying to install Linux on a machine with Windows pre-installed.

> Also, which version of Ubuntu is the newest, or is only the most up to date version available for download?

Ubuntu basically comes in 2 types of desktop version. Long Term Support (LTS) versions which have support for security fixes for 5 years and which are released every 2 years. The latest LTS version is 14.04.3 & the next LTS version will be 16.04, (16 = 2016. 04 = 4th Month of the year = end of April).

And then there are the interim release versions which are released every year at the end of October and every other year at the end of April and only have 9 months support, The latest Interim release is 15.10 (15 = 2015. 10 = 10th month = end of October).

So, if you install Ubuntu 15.10 you will have to upgrade to Ubuntu 16.04 by the middle of the summer. If you install 14.04.3 you will have to upgrade to 16.04 but you will have at least another 3 years to make that upgrade.

All supported versions of Ubuntu are updated during their support period. Support only ends when the support period ends. That is why we upgrade.

### Post by buzzingrobot on 2016-01-14
> **kari2 said:**
> This is why I haven't removed Windows quite yet. I tend to think things through before making drastic changes. I do want to switch, but I want to make sure I can switch and still do the same things I do now, mainly because they pertain to my business.

If you depend on Windows applications, like Photoshop, to do work that needs to be done for your business, then -- no matter how much Windows irritates you -- stay with  Windows.

None of Adobe's products run on Linux. Comparable Linux tools are available, with "comparable" depending on your needs and your skills. In any case, I'm sure you have invested a fair amount of time learning Photoshop.  The learning curve for any comparable Linux program would be at least just as steep. 

Also to be considered is the degree of compatibility of files created by Windows apps with their counterpart Linux apps. 

Also, while Linux can read, or be persuaded to read, Windows file systems like NTFS, FAT16 and FAT32, Windows can't read the file systems used in Linux.  In practical terms, this means that, in a dual-boot scenario, if you need a Windows program to do something with a file created on the Linux side, it won't be able to do that, at least without third-party tools.

If you can satisfy your Linux curiosity by installing it as the only OS on another machine, I really recommend doing that. You can see what you like, what you don't like, and if you really can get work done using only Linux.

Dual-booting:  Creating a dual boot setup requires modifying the very small, very specific section of the hard drive that boots the operating system.  If it's botched, you get a system that won't boot. There are ways to fix it. But, you can check this forum and find many posts from people who went wrong with dual-booting.  So, if you want to go this route, do your homework and research how to set it up correctly.  And, as everyone always says, *Backup Anything You Can't Afford To Lose Before You Try It*.

### Post by Bucky Ball on 2016-01-14
As for virtualbox, you will need a decent amount of RAM. Without knowing the specs of your machine we have no way of recommending virtualbox as an option now or in the future. :)

A dual-boot is relatively easy and sounds like the most suitable option for you. If you need help doing that, post a new thread in the ***Installations and Upgrades ***area and we'll get to it. You can start by downloading the [Ubuntu]("http://www.ubuntu.com/download/desktop") or [Ubuntu Gnome ISO]("http://ubuntugnome.org/download/"), creating boot media (disk or USB) and booting from it. All flavours have the option of 'Try Ubuntu' directly from the install media so you can try the OS without installing or changing anything on your hard drive. 'Try before you buy' (or install rather). ;)

* Just another tip in case not mentioned: Every two years we have a long-term support release. These end in 'LTS' and are supported for five years in Ubuntu, three in Xubuntu and not sure about Ubuntu Gnome. All other releases are interim releases and have nine months support. Releases come out every six months (always named **.04 and **.10, e.g. 14.04 LTS and 15.10, which indicates the year and month of release). 

Each release can be upgraded, via the net, to the next release. E.g. if you are on 15.10 you can upgrade to 16.04 LTS in the latter part of 2016. BUT ... LTS can also upgrade to LTS. Therefore, the very stable 14.04 LTS can upgrade to 16.04 LTS also.

Having said this, a clean install can be done at anytime. As there are interim and LTS releases, you can install any one of them that is currently has support. At the moment that gives three options. 12.04 LTS (supported until 2017 but wouldn't go there), 14.04 LTS (2019), and 15.10 (around July this year).

Hope that is helpful info. Good luck and hope you enjoy. :)

### Post by Mark Phelps on 2016-01-14
Just to comment on a couple of issues you have ...

1) Windows apps -- I've used Wine for a long time, and honestly, the ONLY environment that runs native Windows apps properly is -- Windows.  Since you already have it, then don't throw it away in a mad rush to get rid of it.  Keep it around for a while.  That will give you time to pick and choose among the MANY free Linux app alternatives and choose the ones you want to stay with.  When (if) you are later convinced that the Linux apps serve your needs better, THEN consider getting rid of Windows.

2) MS Office and alternatives -- ONLY MS Office apps will do ALL the things that MS Office does. Period.  There are alternative Office-like products, with Libre Office being a well-known one, and WPS Office having the reputation for have the best compatibility with MS Office.  But if you use MS Office apps to exchange Windows files with others running MS Office, then stay with MS Office.  That will provide the BEST compatibility with the Office product the other folks are using.

As folks have said -- Linux is NOT Windows, so it's not designed to be a drop-in replacement that runs Windows apps or faithfully uses Windows files.

You can find all kinds of long-winded howto's about the "best" way to setting up backup, but simply copying files from Here to There is most of the game.

Video issues happen sometimes in Linux. Machines have Nvidia or AMD video cards.  In addition, Intel CPU's include an onboard video chip. Windows will automatically use a video driver provided by Nvidia or AMD if one of those cards is active when it boots. Ubuntu will not. Ubuntu will use an open source driver for the card that's active when it boots. The developers of those drivers do not have access to the source code and other data for Nvidia's and AMD's cards, or the source of those proprietary drivers. As a result, they can be less than 100% functional.  Much depends on the card. New cards are likely to have the least support.

So... I would post a question in the Hardware section here, listing the hardware components of your machine, and ask about show-stoppers, especially the video card. 

Intel works more closely with the Linux community than Nvidia/AMD.  The open source driver for Intel video they supply *is* included with Ubuntu and will be used by default if the Intel video is the only video active.  It's a solid  and reliable driver.  The potential downside is that the graphic capabilities of Intel video are weaker than Nvidia/AMD, especially for demanding 3D needs, like some games.  

Proprietary drivers from Nvidia or AMD can be installed in Ubuntu via the "Additional Drivers" tool.  Ubuntu maintains packages of these drivers in its repositories.  This tool will scan those and list the drivers appropriate for your system.  You select one and wait for it to install. (This can take several minutes; there's more going on than dropping one driver file in one directory.)  Avoid using other ways of installing these drivers you may find on the web.

You do not need to worry about the impact of a Linux kernel upgrade on an Intel driver.  That, however, can happen with an Nvidia/AMD proprietary driver. (Typically manifests as a black screen when you reboot after the update.)

A bit of a spiel there, I know.  But, info you need to know. I hold to my recommendation to "train" with Linux on a separate machine, if possible. Failing that, boot the Ubuntu installation image and run Ubuntu in live mode, via the "Try Ubuntu" option you will see.  This runs Ubuntu in memory, without touching the had drives. It's pretty slow off a DVD, but decently speedy off a USB.  It gives you a chance to check out hardware capability, i.e. video. 

Don't try to set up dual booting by doing an Ubuntu install and winging it. The first thing is to make sure you have empty space on the drive(s) to accommodate the Ubuntu partitions. (Windows partitions can be reduced in size in Windows.) There's a bit of an install guide at the bottom of this page: [http://www.ubuntu.com/download/desktop.]("http://www.ubuntu.com/download/desktop")
