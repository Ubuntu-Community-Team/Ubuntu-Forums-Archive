### Post by mastablasta on 2013-01-10
for that kind of old maschine you have plenty of options out there in linux world.
 
1. install LXDE or simply Lubuntu
2. install iceWM and use that windows manager
3. install bodhi linux that uses E17 desktop
4. instal debian and add icewm or lxde to it or have a look at antix linux (debian+backports+icewm)
5. install chrunchband (debian stable+backports+openbox windows manager)
 
6. use slitaz linux
7. use puppy linux (or one of it's derivatives that is connected with ubuntu repositories)
8. use tiny core
9. use DSL (damn small linux - i guess it would be an ok choice if commputer is not connected to web, since it uses old kernel that might have security holes)
10. if you know linux then arch might also be good option since i read it is a bit faster (as maybe you can set it up a bit more)
 
all of the above should work more or less descent on your mashcine. why go for XP is beyond me with this config. i had it on notebook with 1.2 Ghz CPU and 256MB ram. replaced it with Ubuntu 10.04 first (didnt' go so well but only after updates). replaced it again with chrunchbang linux - it's flying!!! 80 MB used on idle!!!!
 
how to do a minimal install (mini iso : [https://help.ubuntu.com/community/Installation/MinimalCD](https://help.ubuntu.com/community/Installation/MinimalCD)) on ubuntu? here is a good tutorial with pics and all: [http://www.psychocats.net/ubuntu/minimal](http://www.psychocats.net/ubuntu/minimal)
 
suggets you use 12.04 LTS if you go this minimal way. also debian is calling it netinstall i believe... debian is lighter out of the box than ubuntu but you may need to do some drivers hunting (or not...).
