### Post by Lars Noodén on 2012-03-04
You could improve the performance a bit by using just a [window manager](https://wiki.archlinux.org/index.php/Window_Manager#Window_managers) instead of a full desktop environment.  You can look at Openbox, Fluxbox, or FVWM-crystal for examples.  

About the SSD being 87% full, you can install just a "command line system" first and build it up from there by adding in just the components you actually use.  That way you can leave off the applications and libraries that you don't need.  You get that by pressing F4 when you first boot the Alternate (not Desktop) install image.

### Post by snowpine on 2012-03-04
I don't think the original EEE 701 meets the minimum hardware requirements for Ubuntu:

> 1 GHz CPU (x86 processor (Pentium 4 or better))
1 GiB RAM (system memory)
15 GB of hard-drive space (or USB stick, memory card or external drive but see LiveCD for an alternative approach)
800 by 600 screen resolution 

[Lubuntu]("http://lubuntu.net/") is a better choice for your hardware (though it will never be a fast/powerful computer; EEE 701 is very low-end hardware by 2012 standards).

Instructions to convert your Ubuntu to Lubuntu without a reinstall: [http://www.psychocats.net/ubuntu/purelxde](http://www.psychocats.net/ubuntu/purelxde)

You can buy an SD card for extra storage.
