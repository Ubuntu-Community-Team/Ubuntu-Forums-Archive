### Post by KingTermite on 2008-05-10
I installed Ubuntu on this (Dell Inspiron 1720) laptop. I already have Windows XP installed which I'm not 100% prepared to part with just yet, so I installed Ubuntu 8.04 on a separate partition.

No problems with install, smoother than I ever expected. Boot manager came up fine and I tried to first boot to Windows to make sure all was still OK.

I noticed when getting to the Windows XP login screen that the monitor seemed a bit dark. I noticed it even more when I got into the Windows desktop. It was noticeably darker than before.

I tried adjusting the brightness, but that didn't help. It whitewashed everything, but the dark was fundamental. It almost looked more like when you look at an LCD monitor from a funny angle. 

I tried video brightness settings - no help.
I tried rebooting - no help.
I deleted, reinstalled video drivers, rebooted - no help.

I rebooted, and it was even noticeable during bootstrap (at BIOS screen) sequence. WTF....almost looked hardware problem. Also noticed Windows time got screwed up. I set Ubuntu for GMT - 7 for my time zone during install, and my Windows time was now 7 hours earlier than it should be.

Finally decided to boot into Ubuntu to see if it was dark too as it was looking like more of a hardware issue now. Ubuntu splash screen seemed dark, but by the time I got to the login screen it looked normal.

Ubuntu looked fine. Rebooted again and Windows was back to normal too. Problem appears solved, but it was quite odd. 

Has anybody seen this issue before?
