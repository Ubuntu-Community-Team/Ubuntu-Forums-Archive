### Post by deadflowr on 2013-05-16
The terminal is helpful and convenient, but not full on necessary for every day use.

It's possible to run Ubuntu without ever using a CLI.


I use the terminal for three things mainly

Administrative tasks, like editing system files to fit my needs.

Problem solving applications that aren't running right for some reason.

The task I want to run has no GUI equivalent.

Beyond that, using the terminal is just to get things done faster than with a gui, as CLI don't have to use as many parts as a gui(guis need to open up various libraries and such) and so they can get right to the thing I want to do quickly.

But not nearly so much fun!

@ OP,  Yes, I said fun, and I meant it.

When I started using Ubuntu at version 5.04 or 5.10 I hardly ever used the terminal, even though at that time GUIs did not exist for some of the things that we now use a GUI for.  Now I really enjoy using the terminal for a lot of things that give me information, and for some of the few utilities that still don't have a GUI.  Remember many jobs that have to be done are much faster in command line than a GUI.

Don't imagine, however, that you will need to use terminal to compile and install applications; in my 7 years of ubuntu I have only done it twice, once for ndiswrapper in 12.04 for which the repo package is broken and secondly for hplip direct from HP, though that does not really count, as you simply run a shell script (like a batch file in windows) and the compilation and install are all done automatically for you.

After all that, all I really mean is that you should not fear the terminal; just accept it is there and slowly get to know it as you use ubuntu.  You'll probably find that you love it for some things, just like me.
