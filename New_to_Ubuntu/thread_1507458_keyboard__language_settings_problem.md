Hi

I recently got rid of Windows and switched to Ubuntu all-out. A few migration problems persists. I'm using Ubuntu 10.04.

I'm using a danish keyboard, and I can't get the 'Alt Gr', or Right Alt, key to work properly. Alt Gr + 2 is supposed to yield the @ sign (I copied it here from the Character Map), but it doesn't. It doesn't work at all, actually, and that leaves my without pound sign and brackets, which is fairly annoying.

System > Preferences > Keyboard > Layout tab has one keyboard layout installed: Danish.

I've selected the correct keyboard model (Logitech Internet 350 Keyboard), which otherwise works perfectly. Even the special shortcut keys to music player, browser, mail client and calculator works like a charm.

I tried 'options', and set the 'Compose key position' to 'Right Alt'.

Still doesn't work.

Any ideas?

You mention you have configured the 'Compose key' to be AltGr. The compose key performs a different task so you must change it to something else (like Right Win key, etc).

Another ubuntuforums user had the same problem; check whether you have, by accident, created a keyboard shortcut which is make of the 'AltGr' key. Keyboard shortcuts are in System/Preferences/Keyboard shortcuts.
If you indeed have this problem, then please tell us. If two users got the problem, then Keyboard Shortcuts would need to be fixed.

Indeed, in the layout options (System/Preferences/Keyboard/Layouts/Layout Options) you can configure the 'Key to choose 3rd level'. The default is AltGr or you can specify 'Right Alt'.
