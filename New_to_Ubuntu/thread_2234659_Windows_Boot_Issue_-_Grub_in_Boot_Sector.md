### Post by oldfred on 2014-07-15
Please use code tags for longer text. You can easily add them with # in edit panel of advanced editor.

Your issue is a lot different than the thread you posted in, so moved to your own thread.

You have installed grub2's boot loader to the NTFS Windows boot partition. All NTFS partition have to have the PBR or partition boot sector have Windows info in the PBR. It will not work with grub. NTFS does keep a backup and testdisk can easily restore that if you only copied it into it once.

Fix for most, a few have other issues, better than windows fix in many cases as it also fixes other parameters:
If win7 use small 'system reserved' NTFS partition instead of the partition where windows was installed for win7
This has instructions on using testdisk to repair the install of grub to the boot sector for windows from Ubuntu or Linux LiveCD.
[http://sourceforge.net/apps/mediawiki/bootinfoscript/index.php?title=Boot_Problems:Boot_Sector](http://sourceforge.net/apps/mediawiki/bootinfoscript/index.php?title=Boot_Problems:Boot_Sector)
You want to get to this screen:
[http://www.cgsecurity.org/wiki/TestDisk_Step_By_Step#NTFS_Boot_sector_recovery](http://www.cgsecurity.org/wiki/TestDisk_Step_By_Step#NTFS_Boot_sector_recovery)

Or Similar instructions:
[HowTo] Repair the bootsector of a Windows partition  - YannBuntu
[https://help.ubuntu.com/community/BootSectorFix](https://help.ubuntu.com/community/BootSectorFix)
[http://ubuntuforums.org/showthread.php?t=1926510](http://ubuntuforums.org/showthread.php?t=1926510)


Your install also does not have a swap partition. Usually one is suggested unless you have lots of RAM and even then a small swap is still recommended.
