### Post by ndex477 on 2013-07-16
In the Wicd Network Manager under Preferences/External Programs make sure DHCP Client is set to automatic unless you are configuring this manually. Your issue is definitely on the TCP/IP(internet)side of the connection and not the 802.11(WiFi) side since you can connect to the network. When this happens the first packet sent across the network which is a SYN(Synchronize) packet is being blocked therefore no connection to the webpage you are trying to access. Next it could be a DNS issue. Respond back with your results. Good luck.

### Post by varunendra on 2013-07-17
> **Aaron Nott said:**
> ```

    *wagzy:          Infra, 00:22:75:6C:CD:AE, Freq 2412 MHz, Rate 54 Mb/s, Strength 61 **[COLOR="#FF0000"]WPA WPA2[/COLOR]**

```

The report is incomplete (did you supply your password correctly when asked?), but apparently one possible bad factor is that you are using WPA/WPA2 mixed mode encryption in your router. Please change it to pure WPA2-PSK, and AES(CCMP) instead of TKIP (you may have to check your router's manual to see how to change that setting). Save the settings and restart your router. Then delete the connection in Ubuntu, and retry to connect (reboot if required).

See if that helps. If not, please delete the existing "wireless-info.txt" file > re-run the script and post the new (full) report. Hopefully, it may give some more hint.
