### Post by asmoore82 on 2009-05-19
> **AndThenWhat said:**
> I have this error too. I know it is because my home directory is another partition that auto-mounts on login though. What is $HOME?

$HOME is an allusion to command line interface Environment Variables
They are so named "Environment" because nearly every program you ever run is aware of them
By convention, they are always in all caps to distinguish them from regular variables.

the dollar sign is not a part of the variable itself but is how you tell the shell to recall the value stored there:
```
**~$** echo HOME
HOME
**~$** echo $HOME
/home/asmoore
```

HOME is the most important one of all -
if it is set to somewhere that doesn't exist or that you don't have access to,
mischief will insue - the mischief of a broken Environment
```
**~$** HOME=foobar
**asmoore**$ cd
[COLOR="Red"]bash: cd: foobar: No such file or directory[/COLOR]
```

another big one is PATH, which contains a colon delimited list of where to look for programs
```
echo $PATH
/home/asmoore/bin:/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/usr/games
```

if PATH is wrong, you have pretty broken CLI - almost as bad as CMD in Windows ... almost
```
**~$** PATH=
**~$** ls
[COLOR="Red"]bash: ls: No such file or directory[/COLOR]
**~$** firefox
[COLOR="Red"]bash: firefox: No such file or directory[/COLOR]
**~$** PATH=foobar
**~$** ls
[COLOR="Blue"]Command 'ls' is available in '/bin/ls'
The command could not be located because '/bin' is not included in the PATH environment variable.[/COLOR]
[COLOR="Red"]bash: ls: command not found[/COLOR]
```

Sorry for rambling, I felt chatterly.
