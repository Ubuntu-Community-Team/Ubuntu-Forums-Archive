### Post by grahammechanical on 2016-02-23
I do not use Xubuntu. So, I am assuming that Xubuntu follows the same general principles as Ubuntu on the subject of the frequency of authorisation requests.

It used to be worse. A decision was taken several years ago to reduce the number of user authorisation requests. It was thought that if there were a lot of requests for the user password, then users might put in their password without thinking and so authorise malicious software without realising what they were doing.

I run the development version of Ubuntu and there is a need to update daily. Most days just loading Software Updater is enough to authorise the update process. In the past I would have had to put in my password every time I ran Software Updater. There are certain package updates such kernel upgrades that produce a request for a password. And there are several upgrades to the kernel during the 26 week development period.

I cannot imagine a user of a released version having to put in their password as often as I do. And I am not required to put in my password so often that I just have to rant about it on a forum. Besides, I value the security that the Linux/Ubuntu security model provides.

Regards.
