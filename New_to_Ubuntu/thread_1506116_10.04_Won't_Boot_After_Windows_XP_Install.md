### Post by bcbc on 2010-06-10
Strange... it looks like you're booting from a usb and your bios is setting it as sde with your hard drive as sdf

but go ahead and install grub2 to the mbr ```
sudo mount /dev/sdf1 /mnt
sudo grub-install --root-directory=/mnt /dev/sdf
```

Then reboot from the hard drive. 

If you end up at a grub rescue prompt,enter the following
```
configfile (hd0,1)/boot/grub/grub.cfg
```

If that doesn't work enter
```
search.file /boot/grub/grub.cfg
```
and substitute the result into (hd0,1) in the configfile command above

### Post by bcbc on 2010-06-11
You're trying to install grub to a partition. You should be trying to install to the drive instead. So instead of /dev/sda1, use /dev/sda

Just remember though, from the bootinfoscript results, if you are booting with the same method your hard drive may be /dev/sdf

You can check this by running the bootinfoscript again and identifying your hard drive, or just running sudo blkid and seeing which partitions you get. (then just strip off the digit)
