### Post by crf on 2008-06-04
It doesn't normally matter what language software is written in. Perhaps there are languages that place restrictions on what licences authored programs can be released under. I doubt programs in those languages would be commonly encountered, since it is an unusual restriction. It is fine to author software in C# and release it under the GPL.

I don't think your post is entirely clear. What does "customized" software mean? 

Suppose there was some software, distributed under a licence, and then someone at work customized it, and released the whole under the GPL V3.

This would be perfectly ok if the original unmodified software were released under the GPL V3, or a compatible licence, and the person at work followed all the requirements of the GPLV3.

It would not be ok to release modified software if the original licence of the unmodified software were not GPLV3 or a licence that allowed it be distributed under the GPLV3.

### Post by hyper_ch on 2008-06-04
> **SunnyRabbiera said:**
> Yes then its legal, as in the GPL the source code must be provided if the software is put under the GPL.

To clarify this a bit:

(1) You can sell software even if it's under GPL.
(2) GPL forces you only to provide the source code if one demands it. You are not obliged to provide to source directly.
(3) You can even charge an additional fee for providing the source code. However that fee must not exceed the one for the binary program.

[http://www.gnu.org/licenses/gpl-faq.html#DoesTheGPLAllowMoney](http://www.gnu.org/licenses/gpl-faq.html#DoesTheGPLAllowMoney)
[http://www.gnu.org/licenses/gpl-faq.html#DoesTheGPLAllowDownloadFee](http://www.gnu.org/licenses/gpl-faq.html#DoesTheGPLAllowDownloadFee)

And free software under GPL means "free as in free speech and not free as in free beer".
