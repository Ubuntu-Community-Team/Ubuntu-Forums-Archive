### Post by YannBuntu on 2012-07-05
Yes it makes sense.

Here is what I would do:
1) **Backup documents** (eg on DVDs...)
2) **Remove the useless Ubuntu on the USB disk**: boot a Ubuntu CD, connect the USB, run [gParted]("https://help.ubuntu.com/community/GParted"), delete the sdb2, sdb5 and sdb6 partitions of the USB disk. Format the free space in NTFS so that Windows and Ubuntu can use it. Alternatively, you can expand the sdb1 partition to the entire disk, but this is more risky (be sure you have a backup of sdb1 before doing it).
3) **Disconnect the USB disk**
4) Boot on Windows, do a disk **defragmentation**. Reboot and check that XP still boots correctly. 
5) Via gParted, **reduce your Windows partition (sda2)**. I think you could easily freed 100~200GB. 
6) Reboot and check that you **still can boot Windows** correctly.
7) Boot the Ubuntu CD, run the **Ubuntu installer**. Follow the steps: [https://help.ubuntu.com/community/GraphicalInstall](https://help.ubuntu.com/community/GraphicalInstall) . This should install Ubuntu in the 100~200GB free space. Reboot and tell us if the GRUB menu can let you access Ubuntu and Windows.
