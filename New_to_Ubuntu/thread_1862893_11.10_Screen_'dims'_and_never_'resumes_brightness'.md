### Post by cantenna on 2011-12-13
Greetings!:D

Solved this problem by,

Open Terminal and paste:

cd /etc/X11/xinit

sudo gedit xinitrc 

**(Now change contents of xinitrc file to**

#!/bin/sh

# /etc/X11/xinit/xinitrc
#
# global xinitrc file, used by all X sessions started by xinit (startx)

# invoke global X session script
setterm -blank 0 -powersave off -powerdown 0
xset s off
. /etc/X11/Xsession

This may sound really funny but try this if you didn't find a solution already. As you start the install, even though the laptop brightness is at dim, there is still something that can be seen! Walk out into the sun with the comp(hoping its a laptop) and tilt the screen at an angle at which you can see the words. When you can see something, make sure to select "try it out first" so that you don't go through the rest of the install without light. At desktop, press Ctrl-Alt-T and then type the following brightness code:
```
sudo setpci -s 00:02.0 F4.B=00
```
Continue with the rest of the install. When the computer restarts, as soon as you select this Ubuntu from the grub menu, the screen goes dark. Walk back out into the sun and then login and 5 seconds after the login sound press Ctrl-Alt-T to bring up the terminal and input the brightness code again.
Then input: 
```
sudo bash
```
(input password)
```
chmod +x /etc/rc.local
gedit /etc/rc.local
```

Then add the following at the end of the file just before exit 0.
```
sudo setpci -s 00:02.0 F4.B=A0
```
It should work from then on.
