### Post by Impavidus on 2017-02-03
Let's see...

Retro gaming: Most games are Windows only. You can run emulators or compatibility layers to run them on Linux, with variable results, but if you have a Windows computer anyway, better run them on that.
Photoshop: Not available for Linux. Using the wine compatibility layer it is possible to run some versions of photoshop on Linux ([see here]("https://appdb.winehq.org/objectManager.php?sClass=application&iId=17")), but that wouldn't be your first choice. There's gimp as a good alternative.
Videos and music: Several players available.
Coding: Linux is great for coding. I think better than Windows, but I have to admit I never even tried to code on Windows.

The normal way to install software on Linux is via the repositories. Click on the software packages you want, click install and they're downloaded, installed and upgraded automatically. Downloading everything on your laptop and transferring it by usb drive is inconvenient, to say the least.

### Post by leunam12 on 2017-02-03
I run photoshop CS4 on WINE without problems in my main computer; if you are used to Photoshop you may or may not get used to Gimp depending on what you are trying to do. I gave Gimp many tries but finally I decided to install Photoshop.

I wonder why you want to use your desktop offline, I don't know if others have had this problem but sometimes packages break in Linux and they get fixed by updating the system; it used to happen a lot with Arch Linux, but it has happened also with Ubuntu. 

I have an old desktop that I use for streaming Netflix, Youtube and Amazon videos and it works very well with Ubuntu Mate. It has problems with network-specific video players because they demand a lot of CPU power (I think). I cannot watch anything from NBC, for example, I need another computer with more CPU power for that, but in general it works great.
