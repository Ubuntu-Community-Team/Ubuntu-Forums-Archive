### Post by pansz on 2008-11-11
In my opinion, most great Linux software are written by those who really need them.

However, as far as I know most Linux C developers do not use IDE at all, how do you expect those C programmers who do not use IDE develop a good IDE?

Emacs is generally the most feature-rich IDE for C and it has a heavy learning curve. It is so feature-rich that few developers think it necessary to create something else.

If you think emacs is too overkill, try vim which is much easier, anyway, vim does not have a heavier learning curve than C language, thus most competent C programmers should have no problem learning vim.
