### Post by jordanmthomas on 2008-08-09
While "best" is a subjective term, the most common databases you'll see in Linux systems are sqlite and MySQL.  

I've only ever used MySQL but I am pretty sure Access and MySQL use *nearly* the same SQL statements.  There are nice graphical frontends to it like MySQL Administrator, but I generally prefer to pass the queries in myself.

In the end, it comes down to preference.  KDE has a database program similar to Access called kexi.  I've never used it, so I can't say how nice it is.  According to its project page, it aims to be a competitor to Access, so maybe you could give it a look.

### Post by ugm6hr on 2008-08-09
I have been (and remain a part-time)  Access user too.

If you are looking for something similar with a GUI, I'd suggest Openoffice Base.  I'm currently using v2.4 and v.30beta, which are almost identical.

OO.org allows me to create forms and reports with a simple wizard, not too unlike Access did.  The automated functions available in Access need a bit of java scripting, although I just googled everything I needed.

I have tried Kexi, and found it unstable (the Hardy repo version).

I also tried Glom, which looks nice and easy, but I just couldn't work out how to get it to do what I wanted.  I'm sure a bit of perseverance would have paid off with Glom though.

So I've settled on OO.org, which I use for its other functions anyway.
