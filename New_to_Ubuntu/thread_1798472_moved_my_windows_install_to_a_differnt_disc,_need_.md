### Post by Mark Phelps on 2011-07-07
> **lanman89 said:**
> Is it possible that the grub menu is fine and my xp boot.ini is the problem?

Well ... it's certainly the case that GRUB is NOT the problem.  So, regenerating the GRUB menu over and over is not going to fix this.

If, as I understand it, using the same drive, the following is true:
1) Boots XP when connected to a SATA port
2) Fails to boot XP when connected to the Adaptec controller port

Then, it's clearly something wrong with the Adaptec controller settings. Perhaps, if it's a fairly new card, then it was configured by default to use AHCI mode with SATA, not IDE mode with SATA (or, the other way around).

Check the SATA modes for both your motherboard ports and the Adaptec controller ports.

### Post by Mark Phelps on 2011-07-07
In the past, there were cases where, when you first booted the PC and it went through POST (Power On Self Test), it showed some text identifying the controller -- and listed a way to access the settings for that controller.  When you pressed that key, you could then change some settings.

If you can't boot from the Adaptec, you will need to do essentially the same thing -- access the settings before getting into Windows.

Sorry, but you will most probably have to hunt down tech info for that controller. It's apparent that, using that controller, it's unable to read the files on the connected drive during boot.

An alternative would be to connect using the SATA controller, and, if Adaptec has provided some software, use that to configure the Adaptec controller -- from inside Windows.
