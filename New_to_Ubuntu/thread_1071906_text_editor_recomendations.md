### Post by epswinde on 2009-02-16
> **DeadRobot said:**
> 1. Vim -- dont use if you dont want to learn a lot

Vim and emacs are two of the most powerful programming tools ever created by man.  Seriously, they are an investment to learn, but once you're used to it, you'll never want to use any other.

Emacs is totally configureable.  I've not much experience with it, but I know it's seriously powerful, and does everything that you could ever think of.

Vim is a modal text editor -- sounds silly at first, but if you do alot of typing, then you'll really appreciate not having to move your hands off of the keyboard.  Once you get the hang of it, you'll love it. The other thing is that it is totally ubiquitous, any unix machine you'll ever sit down at will have vim or vi installed.

There's even a plugin for gedit to produce the vim keybindings!

I'm a kde user, so I do most of my work in kate or vim in the terminal.

### Post by Orion8 on 2009-02-16
I use Kate even though I prefer Gnome to KDE most of the time.  I like gEdit a lot, but prefer the way Kate soft-wraps to current indent level.  I know that might seem kind of dumb but when several tabs deep in a LaTeX or Matlab project it really helps to have line returns follow the indent level.  If I could only get gEdit to do this!  As far as you syntax coloring goes, both gEdit and Kate have a huge list of supported languages.  Give both a try! You can get Kate from the repository and gEdit is the default editor.
