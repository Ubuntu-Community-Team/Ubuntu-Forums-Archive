### Post by oldfred on 2011-02-12
If you currently can boot into your install, you just install to another MBR from there. No liveCD required.

#reinstall from working (not liveCD) system - first find Ubuntu drive:
sudo fdisk -l
#if it's "/dev/sdb"  then just run:
sudo grub-install /dev/sdb
#If that returns any errors run:
sudo grub-install --recheck /dev/sdb
sudo update-grub

This also reinstalls grub to whichever drive you choose:
#to get grub to remember where to reinstall on updates:
sudo dpkg-reconfigure grub-pc
#Enter thru first pages,spacebar to choose/unchoose drive, enter to accept, do not choose partitions
#To see what drive grub2 uses see this line - grub-pc/install_devices:
sudo debconf-show grub-pc

### Post by oldfred on 2011-02-13
If you look at the boot script and your menu.

    set root=(hd1,2)
    search --no-floppy --fs-uuid --set 5975569a-b3ce-428c-a190-ce0b8eaf83af


The set root is saying drive 1 or sdb, but the search is supposed to override  the set root command and use the UUID partition that has the install. Ubuntu usually boots even if set root is wrong.

If you only have the one install, you do not normally get a menu. Hold down shift key from BIOS until menu appears. Then use e for edit and change set root to hd0 and delete search line. See if it boots.

More info:
Grub Rescue Prompt Megathread - drs305
[http://ubuntuforums.org/showthread.php?t=1594052](http://ubuntuforums.org/showthread.php?t=1594052)

### Post by oldfred on 2011-03-26
1.97 is grub2. It has been updated to 1.99 in the soon to be released Natty 11.04.

Manual edits are not saved when you boot, it is just for testing. If you need permanent changes you have to edit this:

gksudo gedit /etc/default/grub

If you now only have one drive it will be sda, so you have to change this to sda2, your UUIDs look ok.

    linux    /boot/vmlinuz-2.6.31-22-generic root=[COLOR=Red]/dev/sdb2[/COLOR] ro  splash  quiet splash

Once you boot you can run this and it should update everything:

sudo update-grub
