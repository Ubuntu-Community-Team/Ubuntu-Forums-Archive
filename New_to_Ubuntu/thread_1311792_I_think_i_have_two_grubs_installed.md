### Post by louieb on 2009-11-02
2 installs - 2 copies of GRUB - that is normal. One for each install. In fact its the easiest way to go.  

With Grub Legacy (used by Ubuntu prior to Karmic) it would be a real pain to try and get by with just one copy of GRUB.  (lol - I tried - kernel updates were a disaster waiting to happen) 

With GRUB2 with its OS-PROBER it should be easier to get by with only one copy but you would still have to manually run the the prober each time the OS without GRUB has a kernel update. 

Grub does not take much space and tools for GRUB2 are still being developed.  
Just my 2 cents worth - If it works don't think I would try and fix it.

### Post by QLee on 2009-11-03
> **Falc7 said:**

Here is what happened,
When you installed KDE Minimal, you chose (or allowed) that installer to write GRUB (for simplicity, the part with the menu) to the partition and also let it write that location to the MBR, thus it seemed to "take over".

By the way, you would be able to edit the GRUB on the other partition from Kubuntu if you mounted that volume somewhere.

The reason the grub-setup command failed is that it is a grub shell command, you have to already be in the grub shell for it to work.

As has been mentioned, your Kubuntu install is using legacy GRUB, I suggest you reinstall That GRUB to the MBR, since you say you don't care about the minimal install, or you could probably choose to chainload it (the "minimal" install) from the menu.

### Post by ranch hand on 2009-11-03
I am assuming that the OS that you did the "grub-install -v" on is this Kubuntu that we are using as your boot/root.

This was an upgrade so you have parts of both grubs on there.  I know, they are supposed to work together.  They do not have a good record of this.

So, go to synaptic and run a search for grub.  Remove grub, grub-common and grub-pc.  DO NOT turn off or reboot your box, you have no boot loader.

Install grub-common and grub-pc.  Now you have a boot loader.

This should have run update-grub and grub-install /dev/sda when they were installed.  However we will not trust this and so we will run;
```

sudo update-grub

```
and 
```

sudo grub-install /dev/sda

```
Reboot and see if everything is there.  It should be.

Let us know, we are waiting with bated breath.  And our fingers crossed.
