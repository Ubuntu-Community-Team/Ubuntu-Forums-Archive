### Post by 4Orbs on 2009-07-19
If you want to manually edit the xorg.conf file, I believe you can achieve the desired 1440x900 resolution. First (as root) make a backup of the current xorg.conf by opening in gedit and select "Save As" xorg.conf-backup2 (or any name you prefer that doesn't already exist). Then delete the "Option metamodes" line completely. Then add this "modes" line in the Display subsection, right beneath the "Depth 24" line:
```
Modes "1440x900"
```
Then "Save As" /etc/X11/xorg.conf and close the file. Reboot, and the login screen and all user accounts should default to the new screen resolution.

If this works for you, then to change the resolution or add more resolution options will require your editing the xorg.conf again.

This is what worked for me, but you might want to wait for someone else's advice before using my suggestion.

### Post by 4Orbs on 2009-07-20
If none of the above fixes anything, and you feel courageous, you can try replacing your xorg.conf file with my xorg.conf below.
```
# xorg.conf (X.Org X Window System server configuration file)
#
# This file was generated by dexconf, the Debian X Configuration tool, using
# values from the debconf database.
#
# Edit this file with caution, and see the xorg.conf manual page.
# (Type "man xorg.conf" at the shell prompt.)
#
# This file is automatically updated on xserver-xorg package upgrades *only*
# if it has not been modified since the last upgrade of the xserver-xorg
# package.
#
# Note that some configuration settings that could be done previously
# in this file, now are automatically configured by the server and settings
# here are ignored.
#
# If you have edited this file but would like it to be automatically updated
# again, run the following command:
#   sudo dpkg-reconfigure -phigh xserver-xorg

Section "Monitor"
	Identifier	"Configured Monitor"
EndSection

Section "Screen"
	Identifier	"Default Screen"
	Monitor		"Configured Monitor"
	Device		"Configured Video Device"
	DefaultDepth	24
	Option	"AddARGBGLXVisuals"	"True"
SubSection "Display"
Depth 24
Modes "1440x900"
EndSubSection
EndSection

Section "Module"
	Load	"glx"
EndSection

Section "Device"
	Identifier	"Configured Video Device"
	Driver	"nvidia"
	Option	"NoLogo"	"True"
EndSection

Section "ServerFlags"
        Option          "DontZap" "false"
EndSection


```

The reason I'm suggesting this is because when you use the nVidia Settings mgr to overwrite the xorg.conf file, it adds a bunch of stuff that may not be necessary. Some of that mysterious stuff might still be telling it to use 1024x768. I don't know if this will be a good or bad thing. But it could possibly work. By the way, the DontZap option at the bottom of the file will enable you to use ctrl+alt+backspace to restart the xserver (it quickly throws you back to the login screen). You can remove that part if you don't care for that option.

The problem you and I had was not xorg.conf related (not global for all users) but a problem for your own user. Like you said, there was an override. The override is user specific, meaning it must be somewhere inside your home folder (regular text file or binary with gconf), and I found that file : it is the ~/.conf/monitos.xml, the tool that writes it is the regular display tool, or you can just delete it, so you won't override the global resolution stated in the xorg.conf file.

If you open nvidia-settings as user, you don't change xorg.conf (you can't cause you don't have permissions to do it), instead it changes something, again in your home folder. That's probably not aware of the way the Display tool saves things and therefore we have this confusion.

Also when using the nvidia driver (not nv) you can get to the regular Display menu, if you choose "No" at the message that appears.

The nvidia driver is better than nv because it supports the destop effects while the regular nv driver does not. but the nv driver is free so some people will prefer it.
