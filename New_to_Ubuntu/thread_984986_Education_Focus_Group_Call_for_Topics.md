### Post by Sorivenul on 2008-11-24
This probably goes with the idea of proper system maintenance, but some specifics never hurt:

Those new to Ubuntu are often new to Linux, and are thus predisposed toward "Windows-think".  These users are missing things like Defrag/CHKDSK, System Backup/Restore, firewalls, and anti-virus. The ever-popular "Linux is Not Windows" article is not enough, and some references could easily be seen as a dismissal of the user's issues.

"fsck" is run after every X amount of boots, and is able to be run by the user, should they need it.  For the average user, this should replace Defrag/CHKDSK.

There are many backup and restore ideas in place for Linux.  SysRes, PartImage, rsync, etc.  Of course these come in varying difficulties, but something should be said for these tools.

Firewalls can render a system inoperable as far as networking goes if the user doesn't know what they are doing.  Here we have "ufw" and iptables.  There has been a lot of confusion lately about Firestarter vs. "ufw" and others. 

For those users who may deal with Windows systems from their Ubuntu/Linux system, anti-virus is still a necessity.  ClamAV and others are available, but the purpose and level of need of such a client in Ubuntu should be better explained.

**Example - worked tutorial on VirtualBox**
_____________________________________________________________________________ 

_Detailed VirtualBox Installation Tutorial - USB Support & File Sharing_
_____________________________________________________________________________

**_VirtualBox Installation Sequence_**

These notes provide step by step instructions for installing VirtualBox on an Ubuntu 8.04 host platform. This tutorial will use Windows XP as a guest operating system (OS) that will reside on top of the host Ubuntu 8.04 platform.

The procedure will require a Windows XP Installation CD. This can be SP1, SP2 etc.

The user name for this exercise is "ian". The home folder /home/ian/* leads to all VirtualBox files.

[Hidden files (begin with a dot i.e. **.Hiddenfilename**) can be made visible by selecting "Show Hidden Files" from the "View" options in a file browser.]

This VirtualBox installation exercise is quite lengthy. It can be spread over a number of days if this is preferable.

This tutorial exercise has been "test run checked" on an Acer Aspire 3003WLMi notebook using a small 10 GB partition for Ubuntu 8.04. (DUAL BOOT). This partition was used to confirm each step in this VirtualBox installation ritual.

*****************************************************************************

**_Step 1 - Download Sun xVM VirtualBox "PUEL" Version (Remove "OSE" version if present)_**

There are two versions of VirtualBox. The "OSE" version does not have USB support. If you have this version, it can be removed by copy & pasting the following syntax string in a terminal (Applications - Accessories - Terminal)

	sudo apt-get autoremove virtualbox-ose --purge

[Remove any left over remnants in home/ian/.VirtualBox if the "purge" function misses some files]

Direct your browser to the following site for the VirtualBox software download

	[http://www.virtualbox.org/wiki/Linux_Downloads](http://www.virtualbox.org/wiki/Linux_Downloads)

Choose the VirtualBox version that suits your computer hardware and operating system (OS). We will assume an i386 hardware architecture with Ubuntu 8.04 ("hardy") as the OS. Download the file to the desktop. The file, in this case will appear on the desktop and be named 

	virtualbox-2.1_2.1.0-41146_Ubuntu_hardy_i386.deb

You now need a VirtualBox "key" named "sun_vbox.asc". You can install this automatically by opening a terminal and cut-pasting the following syntax string into it 

Open a terminal (Applications - Accessories - Terminal). Cut and paste the following syntax-string into the terminal

	wget -q [http://download.virtualbox.org/virtualbox/debian/sun_vbox.asc](http://download.virtualbox.org/virtualbox/debian/sun_vbox.asc) -O- | sudo apt-key add -

[ Note - this editor mangles the http reference string. The full string to use after the http:// is download.virtualbox.org/virtualbox/debian/sun_vbox.asc ]

Click enter

[To confirm the key entered correctly, go to "System - Administration - Software Sources - Authentication". The dialog box that opens should contain

"6DFBCBAE 2008-07-14
Sun Microsystems, Inc. (xVM VirtualBox archive signing key)<info@virtualbox.org>"]

*****************************************************************************

**_Step 2 - Install VirtualBox_**


Now install VirtualBox from the downloaded file on the Ubuntu desktop i.e.

	"virtualbox-2.1_2.1.0-41146_Ubuntu_hardy_i386.deb"

Right click on this file. The message - Open with "GDebi Package Installer" should appear (if GDebi is installed). Click on this message and then click "Install Package". You will be asked to enter your password - do this and the "Install package file" process should start. (lasts a few minutes). Click - Close - when the installation process finishes.

[Virtual Box will be installed in a "hidden folder" located at

- Places - Home Folder - ian - .VirtualBox ]

VirtualBox should now appear under the "Applications" category at the top left side of the Ubuntu desktop display

	- Applications - System Tools - Sun xVM VirtualBox


[Also I have seen an alternative approach?

	sudo dpkg -i VirtualBox_2.1.0_Ubuntu_hardy.deb

	sudo usermod -a -G vboxusers ian  ]


It is desirable to also instal "dkms". Open the terminal again and cut-paste the following syntax string into it

	sudo apt-get install dkms

Click enter. The software should report successful installation.

Finally, you now need to locate the "Guest Additions" file. This should be visible as a "iso" file at 

	/usr/share/virtualbox/VBoxGuestAdditions.iso

It may be convenient to copy this to your desktop. I will assume this step is followed for this exercise. The "VBoxGuestAdditions.iso" file will be needed for file sharing between the host Ubuntu OS and the guest XP OS.

*****************************************************************************

**_Step 3 - Prepare The Virtual Environmnent for Windows XP Installation_**


Click on Sun xVM VirtualBox and a dialog box will open. Click on "New". A Welcome dialog box will open - Click on "Next".

A "VM Name and OS Type" dialog box will open. Enter a name e.g. "Windows XP 5 GB" (without the quotes). Leave the two default "OS Type" entires as Operating System = "Microsoft Windows" and Version = "Windows XP". Click "Next".

A "Memory" dialog box opens with a recommended base memory size of 192 MB. You can set this as high as one half your computer's RAM memory. I set it to 256 MB for this exercise. Click "Next".

The next dialog box sets up the "Virtual Hard Disk". Make sure the check-box for "Boot Hard Disk (Primary Master)" is ticked. Click "New".

A "Welcome to the Create New Virtual Disk Wizard" dialog box will appear. Click "Next". 

[You have two "Storage Type" options of variable virtual disk size or fixed virtual disk size. Either is OK - select the "Dynamically expanding storage" option for this exercise.]

Click "Next". A "Virtual Disk Location and Size" dialog box will appear. This needs a meaningful name - e.g. enter text in the "Location" field as "Windows XP 5 GB". Adjust the "Size" slider to 5.00 GB (or enter 5.00 directly in the field). This will create a virtual disk with an upper memory size limit of 5 GB called "Windows XP 5 GB". Click "Next".

A "Summary" dialog box will appear showing Type, Location and Size. Click "Finish".

The "Virtual Hard Disk" dialog box will appear again. Click "Next".

A "Summary" dialog box will appear with Name, OS Type, Base Memory and Boot Hard Disk. Click "Finish".

You now have a "Virtual Machine" using XP as its operating system (OS). This machine is called a "guest" and it runs on the base OS "host" - in this exercise, Ubuntu 8.04 "hardy" was used as the "host".

These "permissions" are set through the Ubuntu desktop categories. Click on;

	System - Administration - Users and Groups - Unlock

Enter your password at the "Authenticate" prompt.

[Two entries will appear in a dialog box. The first will be your Name and Login name, the second will be "root" and "root".]

Click "Manage Groups".

A list of "group names" will appear. Scroll down to "vboxusers". Click "Properties"

Add a tick to the box entry next to your name under the "Group Members" category. Click "OK" then "Close".

*****************************************************************************

**_Step 5. Set Up the xVM Sun VirtualBox Settings_**


The "Sun xVM VirtualBox" dialog box should still be open. Click its "Settings" cog.

Under "General - Basic" set "Base Memory Size" to 256 MB (see note following). Set "Video Memory Size" to 12 MB. Now click to "General - Advanced". 

[We will install XP from an installation CD in the optical drive].

For "Snapshot Folder" leave the default setting (these settings should default to the correct entries)

Click "OK"

Click "Settings" on the "Sun xVM VirtualBox" dialog box. Click "CD/DVD-ROM". Add ticks to "Mount CD/DVD Drive" dot "Host CD/DVD Drive" (this should display the name of your optical CD drive). Do not tick "Enable Passthrough" at this stage.

Select "Audio". Add tick to "Enable Audio". Set "Host Audio Driver" entry to "ALSA Audio Driver".

Select "Network". "Adapter 1" should contain the correct default entries.

Select "Serial Ports". Tick "Enable Serial Port". The other entries should default to the correct values.

Select "USB". Tick "Enable USB Controller". Tick "Enable USB 2.0 (EHCI) Controller".

Select "Shared Folders". Click on the "Machine Folders" entry, then click the + icon on the upper right corner of the "Windows XP 5 GB - Settings" dialog box. 

An "Add Share" dialog box will appear. Select "Folder Path - other". You can now select a folder to share files between XP and Ubuntu. Select "Documents" (for this exercise).


Click "Choose". Then Click "OK".

[The new shared directory "Documents" will appear under the heading "Machine Folders". You can add as many folders this way as you like].

Click "Remote Display". Tick "Enable VRDP Server". Click "OK"

[You have now set everything up for your new XP Virtual Machine. You will have USB port access and be able to share folders between the host (Ubuntu) OS and the guest (XP) OS - however we have one extra installation step required for "Guest Additions" as will soon be explained].

Place a XP Installation CD in the computer's optical drive. (VirtualBox does not provide the OS - you need to provide the OS installation disk.)

Have a cup of coffee.

**Intermediate** - *Single Item Task Requests*, assume prior syntax knowledge is present
**Advanced** - *Complex Task Requests* - Multiple sequenced syntax strings assumed

Then, for any particular topic, a progression from beginner -> Intermediate -> Advanced would naturally follow.

I certainly don't dismiss the efforts and passion people present on these forums. I am only suggesting ideas that seem sensible (at least) to me. Also, I definitely would like to contribute more, if such opportunities arise :)
