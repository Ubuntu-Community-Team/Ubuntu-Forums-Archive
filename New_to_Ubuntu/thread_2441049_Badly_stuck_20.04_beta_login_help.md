### Post by grahammechanical on 2020-04-19
I am confused. There seems to be several things happening that might not be related but are causing confusion. To me at least.
1) The Ubuntu login screen only accepts one password for any user that is set up on the system. It will not accept the keyring password or our Ubuntu Single Sign On passwords.
2) Problems logging into Ubuntu forums or other places that require use of the Single Sign On are different problems to a problem logging into Ubuntu OS. The same is true for Livepatch.
3) If we change the server from which we get Ubuntu updates then the system will reload and check that server for any updates and download the information.
4) The warning of "some problem has occurred" can happen at any time and might be unrelated to whatever you are doing. At the moment I am using Ubuntu 18.04 and a few seconds after logging in I get that message. It is unrelated to whatever I was doing at the time. Ubuntu 20.04 is still a development versions and it should not be a surprise that there are problems.
5) I have noticed when using Ubuntu 20.04 that if the system crashes it will sometimes return me to a login screen. And I have had to use the power switch on the case to reboot 20.04 on a few occasions.
6) At the moment on my machine I have one version of 18.04 and two versions of 20.04 and the only time I have a problem using my Ubuntu Single Sign On is if I am having some problem with my internet connection. I am in the UK using a telephone land line and sometimes there is interference on the line that reduces the download speed. This could be your problem.

I almost forgot. I have experienced 20.04 reject my login password a few times. No logical reason why. I assume I must have mis-typed it. But I doubt it.
