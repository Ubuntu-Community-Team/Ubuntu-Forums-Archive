### Post by BertN45 on 2011-05-09
I agree you probably have a hardware problem, so:

- Check you disk. Look in the Disk Utility (System -> Administration) look at the SMART Data and scroll  the Smart Data for red entries. Look at the assessment column for those red entries. In case of doubt run the self test and choose the long one or tell us what SMART says.. 
- start running the memory test, use the live-cd or keep the SHIFT key  pressed during the whole boot process. If the Grub menu appears, choose the memory test. The test will run for an hour or so. I did find  the errors on my son's PC only after more than half an hour.

Please tell us the results.

I agree completely with you about the geeky/macho character of the community. I think it is one of the reasons that many non-computer hobbyists will give up on Ubuntu.  It is in flagrant contradiction to the Ubuntu slogan "linux for human beings". And it is not smart! Being one month on this forum I already detected a lot of misunderstandings caused by typos and the problems continue for many pages. Often the problem could have been solved by using the Gui in four or so bullet points with a minimal chance on misunderstandings..
With more than 20 years experience with different command lines in the time it was the only way, i am very happy with the progress the Gui did bring us and I will only use the command line, if there is absolutely no other possibility.
