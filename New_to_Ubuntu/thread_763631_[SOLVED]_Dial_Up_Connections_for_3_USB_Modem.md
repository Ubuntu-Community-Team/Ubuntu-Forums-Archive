### Post by aquavitae on 2008-05-02
> **phreakaholic said:**
> before using gnome-ppp, you have to configure wvdial.conf. CMIIW

I think gnome-ppp is a substitute for wvdial, so you shouldn't need to configure wvdial.conf. Anyway, you still need to know the modem device for wvdial.conf.

Judy, can you unplug and replug your modem from the computer, then in a terminal type the following tow commands and copy/paste the output from each into a post:
```

~$ dmesg
~$ lsusb

```

When you type *dmesg*, you should get something like this somewhere in the output:
```

[  445.776509] usb 1-6: new full speed USB device using ohci_hcd and address 2
[  446.001897] usb 1-6: configuration #3 chosen from 1 choice
[  446.124118] cdc_acm 1-6:3.1: **ttyACM0**: USB ACM device
[  446.144712] cdc_acm 1-6:3.3: **ttyACM1**: USB ACM device
[  446.168709] usbcore: registered new interface driver cdc_acm
[  446.169378] /build/buildd/linux-2.6.24/drivers/usb/class/cdc-acm.c: v0.25:USB Abstract Control Model driver for USB modems and ISDN adapters
[  446.340862] usb0: register 'cdc_ether' at usb-0000:00:0b.0-6, CDC Ethernet Device, 02:80:37:0f:03:00
[  446.340883] usbcore: registered new interface driver cdc_ether

```
One the 3rd and 4th lines in mine I've got the text ttyACM0 and ttyACM1. You'll probably have similar lines (although maybe only one), but they might have different text there. Using whatever text you have, post the output of:
```

~$ ls /dev/ttyAMC0 /dev/ttyACM1

```
substituting ttyACM0 and ttyACM1 with whatever you have.

Assuming there are no surprises anywhere, one of those two devices (/dev/ttyACM0 and /dev/ttyACM1 for me) should be your modem device. In gnome-ppp, under modem, type in one of those. If it doesn't work try another.
