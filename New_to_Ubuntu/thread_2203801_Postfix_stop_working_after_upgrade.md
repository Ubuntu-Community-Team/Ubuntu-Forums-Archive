### Post by SeijiSensei on 2014-02-05
The denials concern IPv6 DNS resolution (those "AAAA" records).  The name smtp.gmail.com is an alias for another machine at Google that does have an IPv6 address:
```

Seiji@GhostWorld:~$ host -t any smtp.gmail.com
smtp.gmail.com is an alias for gmail-smtp-msa.l.google.com.

Seiji@GhostWorld:~$ host -t any gmail-smtp-msa.l.google.com
gmail-smtp-msa.l.google.com has address 173.194.68.109
gmail-smtp-msa.l.google.com has address 173.194.68.108
gmail-smtp-msa.l.google.com has IPv6 address 2607:f8b0:400d:c00::6c

```

Either you have broken IPv6 DNS resolution, or you should see if "gmail-smtp-msa.l.google.com" works in your configuration instead of "smtp.gmail.com".

I can't help with IPv6; I never use it.
