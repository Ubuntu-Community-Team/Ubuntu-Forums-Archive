### Post by Jeffreytooker on 2011-01-09
My problem has been solved.  Conclusions:

Best use of memory on a large (i.e. 16G) USB boot memory stick, is to use a "full install" as described earlier in the original thread (**Data storage on USB boot stick.) **by C.S. Cameron.  This will allow about 8G for a FAT32 partition, which can be used for general data storage.  This partition did not automatically show up on my desk top.  This partition will actually show up as "windows" in "file system".  It does not appear like a normal USB (non boot) stick.  I made a sub file "data storage (boot disk) USB", and put it on the desk top.  During "full install" if live DVD does not auto boot, boot from boot list.  GParted was not in my "full install".  Downloaded same.  Once again thanks to C.S.Cameron for all of the help.
