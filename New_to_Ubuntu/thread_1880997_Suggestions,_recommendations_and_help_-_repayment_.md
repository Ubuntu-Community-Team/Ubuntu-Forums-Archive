### Post by qyot27 on 2011-11-14
The only complete (in other words, not just wrappers around drivers and such) solution I know of **is** Wine.  The other ones that provide 'complete' functionality (like Crossover) are based on Wine.  You can check the Wine AppDB to see how well a certain application is supported.  The Steam compatibility page is here:
[http://appdb.winehq.org/objectManager.php?sClass=version&iId=19444](http://appdb.winehq.org/objectManager.php?sClass=version&iId=19444)

Aside from that, there are virtual machines (like VirtualBox, VMWare, or qemu), but those have performance hits and you still need a valid copy of Windows.

### Post by Paqman on 2011-11-14
TBH, if you're wanting to run Windows games, by far the most successful thing to install is Windows. Windows is a good gaming platform, there's nothing wrong with installing it for that purpose.

Having said that, my personal preference for trying to bend Linux into shape to play Windows games is PlayOnLinux. It's a front-end for wine that includes scripts that make things install and run better. Gaming through Virtualbox is starting to become a more realistic proposition now too. The support for 3D graphics has got to the point where some older games will actually run quite well.
