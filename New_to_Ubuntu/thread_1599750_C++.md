### Post by srs5694 on 2010-10-18
If you've got existing Windows programs that you want to compile in Linux with minimal effort, you could look into winelib, which is part of the [Wine](http://www.winehq.org) project, which provides a Windows compatibility tool for Linux. In theory, you could take your source code, compile it using GCC in Linux, and have it work right away. You can also run a native Windows binary almost as easily. Using Winelib, though, you could begin integrating Linux-native calls and transition over to GTK+, Qt, or some other Linux GUI toolkit at whatever pace suits you. Note, however, that Wine's handling of Windows APIs is imperfect. If your programs are simple, they'll probably compile and run fine, or at least be fixable fairly easily. If you're trying to port a really big program, you might run into troubles. (Of course, such a program would be an even bigger challenge to port to Qt or GTK+.)
