### Post by aka-John99 on 2013-02-06
As far as I know Ubuntu does not really care too much what extension you use. Although it may be a good idea to be consistent as it could confuse some applications. Also remember unlike early Windows file names are case sensitive and case preserving.

.adb differs from .ADB, and .Adb  see [MS Windows NTFS Comparison with UNIX File Systems]("http://ubuntuforums.org/MS Windows NTFS Comparison with UNIX File Systems")

### Post by Impavidus on 2013-02-07
Linux doesn't care about extensions. You can use any character combination you want as an extension, use double or triple extensions (archive.tar.gz.encrypted and the like), twenty character extensions or no extension at all, it doesn't matter. If you design your own file format it's recommendable  to make up your own extension, just for convenience. The OS won't mind.

Keep in mind that some programs do care about extensions. LaTeX is a notable example.
