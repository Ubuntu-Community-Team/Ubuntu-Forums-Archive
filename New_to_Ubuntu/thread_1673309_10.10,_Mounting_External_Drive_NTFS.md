### Post by vanadium on 2011-01-22
The most common cause of an ntfs formatted external drive that won't automount is because the file system was not properly closed. This can happen when you disconnect the drive without using the software eject command, and even when you hibernate Windows rather than quiting it completely.

Resolution: boot into Windows, perform a file system check on the volume, then exit Windows completely. On next boot (or connection), the drive should mount automatically again.

Background: ntfs is a proprietary file system of Microsoft. Thus, the format is not 100% supported under Linux. No linux tools can check and repair ntfs volumes with 100% reliability. Thus, if a volume is marked "dirty" (improperly closed), then Ubuntu will not automount it in order to avoid any risk of damaging the volume. Ubuntu will only automount "clean"  ntfs volumes, because only then, Ubuntu can safely access the volume without damaging it.

### Post by vanadium on 2011-01-23
You have to make sure your drives are OK and stay OK. However, if your thumb drive does not automount either, then something else might be wrong.

To give an indication why your drive would not mount, try following commands and post the output here:

```

sudo mkdir temporary
sudo mount /dev/sdb1 temporary

```
If there is no error from the last command, then all is OK with the drive, at least.
