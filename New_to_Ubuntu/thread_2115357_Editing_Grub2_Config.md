(Each linux has a recovery boot option, windows-acer recovery boot is mis-identified as windows vista, and there's a sub menu option which has some other "obsolete" boot options.

Instead I want the following:
Names I can understand,
Recovery options in their own menu,
A different order,

The easiest way to change this would be to edit the grub.cfg file. This, of course, is not recommended since grub.cfg is auto-generated. I don't understand why this has to be though. What is the point of autogenerating this list when I don't plan to ever change what OSs I'm running. And if I do, I'm willing to take the additional ten seconds to alter the config file by hand.

### Post by Cheesemill on 2013-02-12
Instead of editing the /boot/grub/grub.cfg file you should edit either the /etc/default/grub file or the relevant files in the /etc/grub.d directory.

These are the files that are used to generate the grub.cfg file.

### Post by lordievader on 2013-02-12
The grub.cfg file is regenerated every time you (or the system) runs update-grub. This happens when you update your kernel. The easiest way would be, I think, to make use of the Grub-Customizer. There is an excellent guide about the Grub-Customizer right here on Ubuntu Forums: [http://ubuntuforums.org/showthread.php?p=10340183](http://ubuntuforums.org/showthread.php?p=10340183)
