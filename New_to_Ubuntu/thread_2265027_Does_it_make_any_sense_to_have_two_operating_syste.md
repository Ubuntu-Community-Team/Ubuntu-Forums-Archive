### Post by mooreted on 2015-02-12
I used to dual-boot, but Linux has come so far over the years I never use Windows anymore. It all depends on you. Make a list of the programs you use. Is there a Linux alternative to those programs? If the answer is "yes" then you may not need a Windows installation at all.

Since you are brand new, it might be a good idea to dual boot until you get used to it. Or you can run Linux in a virtual machine by using VMWare Player or VirtualBox. Then you can use it without setting aside a partition on your hard drive for Linux.

Generally speaking, Ubuntu will install along side Windows just by selecting that option during install. However, things do sometimes go wrong. Make sure you have your data backed up and you have a way to re-install Windows in case things go bad.

### Post by SeijiSensei on 2015-02-12
I just bought a new machine with Win7 and wanted to have Kubuntu as well.  I used VirtualBox to create a "virtual machine" into which I installed Kubuntu.  If you have a computer with a decent amount of memory, say 4 GB or more, this is the simplest way to run both Windows and Linux.

1)  First, download an Ubuntu ISO disk image like [Ubuntu](http://cdimage.ubuntu.com/releases/14.04.1/release/) or [Kubuntu](http://cdimage.ubuntu.com/kubuntu/releases/14.04.1/release/)
2)  Next, download and install the VirtualBox software for Windows from [https://www.virtualbox.org/wiki/Downloads](https://www.virtualbox.org/wiki/Downloads)
3)  After it installs, press the New button in the VB application to create new virtual machine.
4)  I usually give Ubuntu 1-2 GB of memory depending on amount available. On this machine with 8 GB, I allocated 2 GB to Ubuntu and left the rest for Windows.  Then you'll be asked to create a "virtual disk" to contain the Ubuntu installation.  On the large disks available today, I usually go for something in the 32-64 GB range.
5)  VB will then ask for the installation medium.  Point to the ISO you downloaded. 

One hitch might be if you try to install a 64-bit Ubuntu into a virtual machine.  To do that, you many need to switch on support for "VT-X" or "Virtualization" in the system's BIOS.  If that sounds too scary, download a 32-bit Ubuntu disk image and use that instead.

I hardly ever use Windows, but I did want to have a machine that I could use to play games.  I also run the occasional odd piece of Windows software like MS Access or a tax-preparation program, so it's nice to have a copy of Windows available from time to time.

As you see here, it depends on your needs and interests.

If you require Windows software -- for work, school, etc. -- then you need to keep Windows.  Yes, there is software that allows some Windows software to be run on Linux. However, switching to Linux just to attempt to run Windows programs adds unnecessary layers of complexity that could be avoid by just staying with Windows.

Make sure whatever it is you do with your computer can be done with Linux.  It almost certainly can be, but making sure before switching is worth a little research.

Dual-booting -- setting up the machine to boot alternatively into Windows or Linux -- is a solution many people find useful. Again, do a bit of research first on setting up dual-boot. 

*AND*:  If you choose to dual boot or to replace Windows with Linux, be sure you have Windows recovery disks/media you can use to reinstall Windows if you change your mind.

### Post by JeQhdMD on 2015-02-12
Lot's of good replies in this thread, but I'd like to add another option to the mix:

** Running an alternative OS on a ***_USB v3 Flashstick in persistent mode_*** . . . (difficulty . . . easy to medium)

This is (imho) the overall easiest for a newbie, it's also the safest with regard to not touching your Windows install and files.  It gives an excellent "working" demo of the OS so one can get used to it.   

** The key to making this work is to have a fairly new PC that will boot from USB Flash-stick (aka "Thumbdrive").   It's also preferable to use a newer version 3 USB connection (again, requiring a USB v3 port and the v3 flashstick).

Recently, I've found doing this with Xubuntu works 90% as fast as a standard hard-drive install, and provides for customizing your system (installs/unistalls of apps, interface customizations, settings, etc.).  The ideal Flash-Stick (at this time) seems to be 8 GB as the tool provided by Ubuntu for a persistent image max's out at 4 GB.

Note:  while the above is working great for me, I haven't fully tested the ability to update the flash-stick with all the security,bug patches & kernel updates issued by Canonical (& Debian).  Perhaps our other more experienced forum members can add additional info on this.  (at worst, one could reinstall every 6 months with new everything . . finalizing the install and customizations can be done in <1 hour).
