> **warfacegod said:**
> Is there any specific reason that htop is superior to top or is it a matter of personal preference?

It's a lot easier to work with for someone new to Linux like the original poster. (Easier for old Linux users, too.)

It responds to the mouse, so you can just click on a column heading, and sort by that column.

It's uses color to convey information more clearly.

top requires some arcane commands to operate it. htop is a lot easier for newbie in that it has a menu structure.

As far as I know, top can't display a tree structure, but htop can.

htop is sort of a happy middle between old top, and GNOME System Monitor. If you use GNOME System Monitor to monitor the system, you'll often find it's the biggest CPU hog!

To install from Firefox: [apt://htop]("apt://htop")

### Post by PenguinInside on 2010-01-24
When you go to kill a process in top, you hit k, and then it asks you for the process number, and then the signal number.

In htop, you just move the arrow keys up and down and select the process you want to work with. Choose Kill from the Menu, then choose the signal you want to send (SIG_TERM) is pre-highlighted. So you just press F9, Enter, Enter.

Also, top, AFAIK, only shows the top processes. htop shows all processes, and you can PageDown and up to see them all. Very intuitive.

### Post by undecim on 2010-01-27
1: The CLI (command line interface, a.k.a. Terminal) is more powerful than you can even imagine. If you are doing any task that you think a computer should be capable of, it's almost guaranteed that it can be done faster and better on the command line. If you ever want to become a "power user", CLI is a must.

2: KDE is awesome. It takes some more system resources, but it is so much better than the Gnome that comes with Ubuntu. ("Designed for Vista" computers will run it just fine.) There were some rough releases between KDE 4.0, and KDE 4.3, but that's been resolved. If you want to try it out, you can run the command "sudo aptitude install kubuntu-desktop" from a terminal.

3:BACKUP YOUR HOME DIRECTORY. If you do anything with Ubuntu that the average Windows user wouldn't be doing (minus downloading viruses), you should keep a backup of your home directory somewhere (an external hard drive, for example). It's a pain having to reconfigure all of your programs and recreate any work you lost just because you typed something wrong while fiddling around. That being said, don't be afraid to poke around in the command line or other advanced parts of your system, as long as having your hard drive erased wouldn't be a bother.

### Post by undecim on 2010-01-29
@meindian523

KDE's Altf-f2 functionality does not exist in gnome. Gnome has a dialog that comes up and lets you run a command when you press alt+f2, but it is nothing in comparison to KDE's equivalent.

For example, with KDE, alt-f2:

[LIST]
[*]Can run a command (type "konsole")
[*]Will complete a partially typed command based on previous commands (type "kon" for konsole, instead of konqueror which I never type -- no pressing tab or anything, just run "kon" as if it were a command)
[*]Launch for files on my computer with the appropriate app (i.e. "schedule.ods", opens that document with Open Office Spreadsheet)
[*]Search web bookmarks (i.e. "ubuntuforums.com" launches Firefox to theseforums)
[*]Launch for a program by description ("Web Browser" launches Firefox, "Image Editor" launches GIMP)
[/LIST]
 However, as I have stated, gnome-do is a sufficient alternative for those unable to run KDE comfortably on their computers.
