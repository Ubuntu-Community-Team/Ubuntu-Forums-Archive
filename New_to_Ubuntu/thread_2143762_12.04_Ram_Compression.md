### Post by DuckHook on 2013-05-10
As far as I know, zram is built into the kernel but not activated. Therefore, it is ready to activate, but your default install will not have RAM compression. It is necessary to download and activate the service daemon before zram can be enabled. If you have not done this, then zram is dormant. If you are really concerned, then do:```
sudo service zramswap status
```If this generates an error message, then the daemon is missing, zram is dormant, and you needn't worry.

### Post by DuckHook on 2013-05-10
Very few people use it, although I've heard good reports from those who do. If you have lots of DRAM, it isn't necessary. Therefore the default to dormant. It cannot be activated unless you download the service daemon and you would presumably know if you did that.
