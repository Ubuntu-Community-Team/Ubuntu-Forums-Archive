### Post by NikTh on 2012-10-24
> **constellanation said:**
> Which do you think would be easier what you recommended or just a fresh install, If I'm going to lose program settings anyway?

Execute them and if you are not happy with the result (or don't work) , do a fresh install. 

First we make a backup of your current settings 

```
gconftool --shutdown
sudo killall -r -I gconf
sudo killall -r -I dconf
tar czf confbackup.tar.gz .compiz* .gconf* .config/dconf/ .config/compiz*
```Then we delete the files (Be careful here , copy-paste the commands , do not make a mistake)
```
rm -rf .compiz* .gconf* .config/dconf/ .config/compiz*
```Reinstall some packages.
```
sudo apt-get install --reinstall ubuntu-desktop libgl1-mesa-dri libgl1-mesa-glx unity xserver-xorg-core xserver-xorg-video-radeon xserver-xorg-video-all
```Force the module to load in startup 
```
echo 'radeon' | sudo tee -a /etc/modules
```Reboot 
```
sudo reboot
```Good Luck.

### Post by barrymmm on 2013-04-03
Most likely too late now but I had the exact same symptoms after messing around with my video settings. Simple solution.
Right-click on the desktop and select "change desktop background" at the bottom of the list
Select "all settings" at the top left of the window
Select "software sources" at the bottom of the windows in the  "system" section
Click on the "additional drivers"" tab on the top right
we want to get away from proprietry drivers here so select "using X.Org X Server........"
Apply the changes
Reboot
