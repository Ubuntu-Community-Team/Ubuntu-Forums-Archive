### Post by bdevine on 2010-12-03
I second the suggestion to keep at it with Python.  IMHO, "learner" languages that purport to be simpler than Python often do have a low barrier to entry, but when one tries to do worthwhile things with them, their simplicity sometimes leads to some pretty hacky stuff (I am thinking of NetLogo here).  As far as learning Python, the How to Think Like a Computer Scientist book offers an excellent introduction, if you haven't checked it out:

[http://greenteapress.com/thinkpython/thinkpython.html](http://greenteapress.com/thinkpython/thinkpython.html)

If you haven't tried Python with this in hand, start with it!  If you are able to do so, don't be afraid to physically print it out and work with it.

Also, if you are willing to keep in mind that it is only a palatable flavor of Java, and not full-fledged Java, Processing can be fun to play with, and it will introduce you to what languages look like when they're slightly lower-level than Python. It will definitely push you in the web app direction as well.

HTH,

Brandon

### Post by s1baker on 2010-12-05
sheez, the fellow is just 13. I haven't did much programming for 15 years, but I would recommend this. If you are just starting out, and you see all the fancy web pages and stuff, remember that you have to start at the basics, and it will take some time to get up to speed where you can do some major programming for web pages and such. 

 That said, I would recommend first starting out with some sort of basic programming language, such as Logo or Basic ( I know, I know, guys, Logo/Basic is to primitive ), but use these languages to play around with and have fun with, and always keep your mind on and try to learn from others about the term "structured programming", very important latter on). With the languages of Logo and Basic you can learn some very basic fundamentals, such as loops, variable assignment, for/if statements etc. You can do some fun things with Logo or Basic.

  If after using Logo or Basic you are still interested in computer programming and coding perks your interest, make it a point to learn about the decimal/hex/octagon/binary counting systems..you will need this in your understanding of computer basics. If you learn how the decimal counting system works ( what all of us humans use ), the others will come easy.

  And above all, remember this "keep structured code the most important thing"
