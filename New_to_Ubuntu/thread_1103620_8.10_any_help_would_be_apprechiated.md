### Post by kevmitch on 2009-03-22
The busybox error is likely related to the very early stages of kernel loading (i.e., long before graphics is initialised) so I don't think this is a problem with your graphics. What makes you think that this is the case?

What exactly does the error message say? You typically get dumped into busybox (which is a bare bones command line) if the initial ram disk (initrd) has trouble loading your kernel. This is usually caused by the initrd being unable to read the hard disk on which the kernel is located. 

Did you do anything special when setting up your partitions during the install or did you just use the defaults? Can you see the newly created Ubuntu partition if you boot from the livecd? It should have folders like "/boot, /etc, /usr, /lib, /var',etc. 

If you can see the Ubuntu installation, it would be helpful to see the contents of the following files: /etc/fstab, /boot/grub/menu.lst. It would also be helpful to see either a screenshot of what you see in the gparted program (I believe it should be in the applications menu) or the output of the command "fdisk -l" entered in the terminal (also should be in the applications menu).

Am I correct in understanding that your "original problem" that required "all_generic_ide" was that the bios wasn't booting off the CD at all? Did you hit F6 to get into the bios or did you do that on the livecd startup screen where it asks you your language?

Had the same problem in Mint 5. Had to edit the kernel line for Mint in menu.lst. This fixed it for me, but I don't know if it will work in Ubuntu. Give it a shot though. Here's what it looks like:

```
# This entry automatically added by the Debian installer for an existing
# linux installation on /dev/sda1.
title		Linux Mint x64, kernel 2.6.24-16-generic
root		(hd0,0)
kernel		/boot/vmlinuz-2.6.24-16-generic root=/dev/sda1 ro splash all_generic_ide floppy=off irqpoll 
initrd		/boot/initrd.img-2.6.24-16-generic
savedefault
boot 




```

### Post by kevmitch on 2009-03-22
Be careful! Don't copy that menu.lst entry exactly. In particular you probably don't want to change your 

```
root   (hd#,#) 
``` 
line or
```

kernel   /boot/vmlinuz-2.6.27-##-generic root=/dev/sda#
initrd   /boot/initrd.img-2.6.27-##-generic


```
As (hd#,#) and /dev/sda# will vary from machine to machine depending on where your Ubuntu partition is.

What you might want to look at changing is what follows "root=/dev/sda#". In particular, if "all_generic_ide" is not already there and this was required to boot the livecd, then you should probably add it. Also, if you can get it working without the "irqpoll" option, I think life would be better without it as this will generate a lot of unnecessary interrupts otherwise.
