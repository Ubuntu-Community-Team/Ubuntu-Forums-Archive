### Post by LeoSolaris on 2008-04-26
Do a search in synaptic for "Linux image" or just "kernel" and they should pop up. Look for the ones that read "Linux-image" down in the L's. Make sure when you're removing the old one that do you NOT remove the new one. (new one's current version number will be bigger)

To edit the menu.lst go to terminal and type in
```
sudo gedit /boot/grub/menu.lst
```Then scroll down it till to find the listings for the kernels. It will look similar to this: ```
title        Ubuntu 8.04, kernel 2.6.24-16-generic
root        (hd0,1)
kernel        /boot/vmlinuz-2.6.24-16-generic root=UUID=a4506a2f-359b-4c6d-b848-bb5f03fd0adb ro vga=792
initrd        /boot/initrd.img-2.6.24-16-generic
quiet

title        Ubuntu 8.04, kernel 2.6.24-16-generic (recovery mode)
root        (hd0,1)
kernel        /boot/vmlinuz-2.6.24-16-generic root=UUID=a4506a2f-359b-4c6d-b848-bb5f03fd0adb ro single
initrd        /boot/initrd.img-2.6.24-16-generic
```Delete the one marked 7.10 or the one that has the lowest kernel number. You will have to delete the entire entry, from title to quiet. Make sure to get the recovery mode's lines, too, otherwise it will be left on the menu and will cause havoc if you accidentally select it.

Leo
