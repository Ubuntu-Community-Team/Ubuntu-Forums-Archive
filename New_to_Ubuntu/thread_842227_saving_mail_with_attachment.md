### Post by schwascore on 2008-07-18
When you use the "Save As" feature in Evolution (and almost all other email clients), the entire message (including any/all attachments and all headers) is exported into a text file.  The attachments show up as what looks like random gibberish near the end of the text file.  It is assumed that the user is then going to import the text file into a different email client.  Once the text file has been imported properly, the attachment will be visible in the client once again. All of the headers must be included in order to make sure the message is readable by other clients.

It seems as though you are making this harder on yourself than it should be. Why don't you just forward the e-mail to the address of the user who is on the Outlook account?

Another solution would be to simply cut and paste the parts of the message you want to send over to a text file and download the attachment separately, although importing the entire message would be simpler.

If any of this doesn't make sense, let me know and I'll try to clarify.
