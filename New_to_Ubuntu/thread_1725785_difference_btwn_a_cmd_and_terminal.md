### Post by JKyleOKC on 2011-04-10
> **KL_72_TR said:**
>  I think back than Tim Paterson who created the cmd was working on Unix system giving Microsoft the little brother of the Command LineActually, I interviewed Tim back in 1989 when writing a book about programming for MS-DOS. He originally wrote "Seattle DOS" as a clone of the then-popular CP/M system, since CP/M was an 8-bit system, and his employer was selling a 16-bit CPU card for the then-standard S-100 architecture.

Microsoft licensed the system to sell it to IBM, and MS-DOS version 1.0 bore almost no resemblance to Unix. However when they rewrote it at Version 2.0, many Unix features made their way into it, such as directories or "folders." Since then, the resemblance has continually increased, but the current "cmd" shell is far less capable than any of the several *nix shell programs.

Toward the close of the 16-bit era, a small firm in Boston came out with an alternative shell program for MS-DOS, called "4dos," that included much more ot the *nix capability. To this day, I run 4dos as the shell for my Windows systems rather than the standard "command.com" shell furnished through Win98SE.

To answer the original question to some degree, Ubuntu, like all Linux systems, offers a number of shell programs. The default (what most of us simply call the CLI or terminal) is "bash" and it offers many capabilities that are missing in "cmd." These include "alias" which lets you define your own commands, the ability to specify your own prompting strings, and retention of command history from one session to the next... A full list could take an entire book, and several have been written!
