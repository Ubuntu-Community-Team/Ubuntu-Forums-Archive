### Post by ugm6hr on 2009-03-08
> **snowpine said:**
> Not true with DSL, Puppy, and SliTaz... they load completely into ram, and so are quite fast running from Live CD (unlike larger distros like Ubuntu).

They are actually *very* fast running from RAM (LiveCD), provided you have 128MB RAM.  DSL will run in RAM with even less (used to be 64MB, not certain now).

I believe the latest cooking Slitaz requires more RAM, but can't remember for certain.

EDIT: I would suggest creating a /swap partition on the HD to give Puppy a proper tryout on 128MB RAM (from LiveCD).  While it will load into RAM, you might find it crashes as soon as you try to do anything else.
