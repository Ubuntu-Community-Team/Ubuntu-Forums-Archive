### Post by MG&amp;TL on 2012-03-20
No, failed ("command not found", or immediate error) commands won't do anything. Commands that succeed will be logged in the file .bash_history so you can reuse them later (try pressing the up and down arrows!) Unless I'm mistaken, that file is "trimmed" after a while, so it won't get too big.

Generally, unless you use "sudo" in a command, there is nothing that can harm your system, only your personal data. There's a sticky at the top of this forum about the CLI, for people such as yourself, which I think you'll find very helpful. I personally recommend LinuxCommand.org.

A system cleaner is not really necessary but can save a bit of disk space-try ubuntu tweak or bleachbit.

Oh, and obviously see the manpage ("man command") to see a manual for the command.

### Post by Grenage on 2012-03-20
Welcome to the dark mysterious chasm that is the unknown consequences of random terminal commands!

It very much depends on what you do, so there is no right or wrong answer.  You can keep a mental note of what you install, and remove it when no longer required; it's usually helpful to have some idea of what a command does, so you'll see the link if something else breaks.  There used to be a cleaning tool (there way still be), but a lot of people did more damage than good.

My advice?  Keep your important data backed up, an install disc handy, and lock away the fear - then get hacking.
