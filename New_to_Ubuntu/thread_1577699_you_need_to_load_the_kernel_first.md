### Post by bluefrog on 2010-09-19
don't know how to answer your question but I can give an advice.

I assume you have / on a single partition (that means it includes /home which is where all your personal data is stored).

In that case indeed, in case you reinstall the whole stuff you lose everything if you have no backup.

But now, you could separate / and /home, which would lead in case of a reinstall of a /home left untouched.

You could do that "live", repartition, separate / and /home but if you're new it will be too much of a hassle to start with.

You'd better save your /home on a usb flash drive, then do an alternate install and select manual partitioning (I don't know if the option is available in the live desktop installation).

Then during the manual partitioning, it is just a matter of wiping blank your drive (delete all partitions) then create a partition of 15GB for /, the equivalent of your RAM for swap and the rest of the drive for /home.

If you ever need to do a reinstall, then you use the same process except that you DON'T delete the partitions you just reassign them to their correct value (/ swap and /home) AND you do not format /home (you just mount it), but you will format /.

### Post by drs305 on 2010-09-19
> **didkaticos said:**
> 

But** I do not have a LiveCD**. Do I need to get one?

I didn't realize you were using Wubi, but it doesn't matter as long as you are able to boot into Ubuntu. 

From your Ubuntu with the working kernel, you need to change the grub files so that the working kernel is the one that is automatically selected.

Figure out which grub entry you want to boot:```

grep 'menuentry' /boot/grub/grub.cfg
```
This will produce a list of entries such as the following example:
> menuentry 'Ubuntu, with Linux 2.6.32-24-generic-pae' --class ubuntu --class gnu-linux --class gnu --class os {
menuentry "Microsoft Windows XP Home Edition (on /dev/sda1)" {
menuentry "Ubuntu maverick (development branch) (10.10) (on /dev/sda12)" {

The entries in the example don't really matter, just see how it is laid out. Each line starts with "menuentry". Find the one with the working kernel (the one you select on boot up) and count which menu item it is. *Start counting with 0*. In the above example, "Ubuntu maverick" would be entry # 2.

Open /etc/default/grub for editing as root:```

gksudo gedit /etc/default/grub
```

Find the line:> 
GRUB_DEFAULT=0

Change the 0 to the number you counted above.
(In my example, for maverick the line would be "GRUB_DEFAULT=2").
Save the file, then run:
```
sudo update-grub
```
The good kernel should now be the default when the menu is displayed and/or when Grub2 boots automatically.

### Post by drs305 on 2010-09-19
You ran my commands from a normal boot correct - not from the LiveCD?

I'm not sure why it didn't boot -21. With what we did, the -22 would still be on the menu but it should boot to -21.  The 21 kernel should have been highlighted and should have automatically booted if you didn't make another selection.

If -21 wasn't highlighted in the Grub menu, we can try it another way. Don't boot from the LiveCD - boot by manually selecting the -21 kernel from the G2 menu. We will uninstall the bad kernel. The easiest way to do this is with Ubuntu Tweak, a very useful GUI app. To install Ubuntu-Tweak, which is not in the normal respositories, go to the Ubuntu-Tweak site, [http://ubuntu-tweak.com/]("http://ubuntu-tweak.com/"), click on the "Download" button.

To start it, from the main menu, System Tools, Ubuntu Tweak. It won't let you remove the active kernel, but make sure you are removing the bad kernel (22):

[LIST]
[*]* Select "Package Cleaner" on the left and ""Clean Kernel" from the right panel.
[*]    * Press the "Unlock" button at the lower right, enter your password.
[*]    * Select from the displayed list the kernel images and headers you wish to remove. The kernel in use is not listed.
[*]    * Press the "Cleanup" button at the lower right to remove the selected kernel images and headers.
[/LIST]
Change the GRUB_DEFAULT=2 line in /etc/default/grub back to 0. You might want to run the "grep" command again to make sure the -21 is the first menu entry.


Update Grub to refresh the menu:
```
sudo update-grub
```

sudo mount /dev/sda4 /mnt
sudo grub-install --root-directory /mnt

Do not use the partition number in the second command.

```
sudo update-grub
