### Post by sievec on 2012-04-06
I managed to fix my problem by restarting the computer and binding /etc/resolv.conf instead of copying it before the chroot. commands looked like this:

sudo mount /dev/sdXY /mnt  sudo mount --bind /dev /mnt/dev sudo mount --bind /proc /mnt/proc sudo mount --bind /sys /mnt/sys 
sudo mount --bind /etc/resolv.conf /mnt/etc/resolv.conf  sudo chroot /mnt


The following update and installation worked fine.
