> if we think that mac OSX was written in few years from about the same unix source, now Linux after 15 years should have to be controlled by mental waves and every television would have a mouse and linux pre-installed...

1. Mac OS X and GNU/Linux do not share source code. Mac OS X is closed-source, with a partially open-source kernel whose license is incompatible with the GPL.

2. Mac OS X is heavily based on NeXtStep, which predates the Linux kernel.

3. The Linux community, and to a greater extent the FreeBSD community, would rather take longer to do something and do it right, than rush something and have it collapse a few years down the track. OS X got to a releasable state so quickly because Apple piled kludges on top of hacks and didn't care if it opened up massive security holes. There are gaping problems in the design of OS X. The design, not the implementation - you can't just "patch" problems in the design, you have to break compatibility, work around the issues, or build new hacks that might not hold up. As a result, we'll either see smaller updates to OS X, or much longer development time between updates, until Apple scraps it all and designs an operating system the way it should have done in the first place. If Apple tries to do big work quickly, the result would be more disastrous than Vista.

I'd rather live in a bazaar than in a house of cards.

### Post by decoherence on 2009-07-23
I'd just like to point out that OS X predates Linux by a few years and is only a few years younger than the GNU project (before someone's head explodes, yes I'm talking about NeXTSTEP/OpenStep.)

So yeah, it did take OS X a long time to get where it is today, and they had to make plenty of compromises to get there (restrictive hardware support, ignoring unix philosophies, keeping code secret to hinder competition, etc.) By changing the name of the project from OpenStep to OS X they made it appear to 'come out of nowhere.' This was helped along by the fact that OpenStep wasn't a hugely popular system and had already been sitting in the back of Steve Jobs' closet for a few years.

I would say that, by comparison, Linux distros have moved at a faster rate in terms of getting a friendly, usable system. In 17 years we've gone from SLS to Ubuntu 9.04 on the power of the 'bazaar.' In over 20 years NeXT has gone from NeXTSTEP 1 to OS X 10.5 on the power of both the 'cathedral' and commercial development.

Which do you think has come further? If you've used both NeXTSTEP and SLS, you know the answer (and it ain't apple's baby.)

PS. I'm now waiting for someone to say "NexTSTEP 1 was already a better desktop system than Ubuntu 9.04" but maybe I'm on the wrong forum for that ;)
