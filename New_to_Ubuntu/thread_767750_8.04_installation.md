### Post by Paqman on 2008-04-25

> Hello, I just tried to upgraded from the previous version to 8.04. When I reached the step the update-manager asked me some questions related to grub. I chosed something similar to "keep current setting". 


Basically it was asking you which kernel Grub should look for. By selecting "keep current setting" you've ignored the new kernel.

That's not a biggy. You can edit the file /boot/grub/menu.lst and tell it to use the later kernel.

In future, you'll want to have Grub use the new settings, or else do the merge of the old and new files.

### Post by Paqman on 2008-04-25
> **babyboss said:**
> 
I am worried if editing the menu.lst manual will cause further problems.


Well, it's good practice to create a backup before editing an important system file.
```

sudo cp /boot/grub/menu.lst /boot/grub/menu_backup.lst

```
Creates a backup copy in the same folder.

Then you can open it up
```

gksudo gedit /boot/grub/menu.lst

```
Probably the safest way to edit is to use find and replace. Although the only bits you really have to edit are the sections that look like overdrank's post.
