### Post by MarkC502 on 2010-01-18
you can install the following NTFS config program with a GUI interface and it should add a nice menu link for itself as well in  System --THEN-> Administration --THEN-> "NTFS Configuration Tool"

Once you install and open that you can just click "enable write support for internal and external devices" and that should make your NTFS be writable.

Command to install NTFS Config

sudo apt-get -y install ntfs-config

Also I would be really carefull with sudo chown username * as you are flirting with breaking something very quickly using sudo and * in any command that modifies anything unless you absolutely know what you are doing it for. Also chown with no -R means it only changes the first directory no sub directories. chmod also uses the -R parameter to signify recursive operation ( included subdirectories ). So more proper commands to take ownership of a directory that has subdirectories etc would be.

sudo chown -R user:group directory
( by adding :group you change the group ownership tag as well )

AND TO CHMOD IT

sudo chmod -R 775 directory

But I would recommend first starting with the NTFS GUI based tool and let it enable write support as it might be that your system is not currently configured to write NTFS ( I.E. you need NTFS-3G package that enables NTFS fuse read/write support )

### Post by ibuclaw on 2010-01-18
NTFS doesn't support Unix permissions, hence chown and chmod will do nothing to the filesystem.

The way you go about editing it is via /etc/fstab

> **mbzn said:**
> 
EDIT:
My GID is 1000, what is 46 for?

46 should be the GID for the **plugdev** group.
If I recall correctly, that was how Ubuntu handles NTFS filesystems specified at installation time, by granting permission through that group. (which you should be part of anyway).

If you are not sure:
```
grep plugdev /etc/group
```
And it should print:
> [NOPARSE]plugdev:x:46:mbzn[/NOPARSE]
