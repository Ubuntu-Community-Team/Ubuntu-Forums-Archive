### Post by oldfred on 2013-03-23
If you have Windows 8 pre-installed with UEFI, you have gpt partitioning and the MBR is not used. There is only a MBR in gpt partitioned drives so old partitions tools will see one large gpt partition and know not to corrupt it.

Early versions of EasyBCD did not work with UEFI, supposedly they fixed it. But I see no reason for it with UEFI. 

To dual boot you have to have both systems installed in UEFI mode. And how you boot installer is how it installs. But a few systems only let you install in BIOS/CSM mode. But Boot-Repair can convert an Ubuntu install to UEFI if installed in BIOS mode.
