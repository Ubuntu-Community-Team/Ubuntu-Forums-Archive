### Post by aa4bb on 2009-09-13
I'm new to Ubuntu. I made a CD-ROM ISO image of jaunty jackalope Xubuntu to install on an HP Pavilion Celeron with 63 Mb memory and 4 GB HD originally running Win98. The CD boots. I select language and keyboard. It goes through several cycles of identifying hardware. Goes through network search, but finds none, since it's not presently connected to network. I say don't install network at this time. Next it says it's going to partition. Then a blue screen appears and it seems hung. If I hit return, a line of white appears at the bottom. Anything I type is echoed on the screen in the white area. Nothing else happens. CTRL-ALT-DEL reboots to the CD startup screen.

The CD-ROM self-validates OK.

Any suggestions what to do next?

I know the memory is probably not enough to be very useful. My goal is to have a basic computer for web browsing and email for my daughter.

### Post by snowpine on 2009-09-14
I hate to say it, but that computer doesn't even come close to the recommendation for Xubuntu:

> Xubuntu recommended minimum requirements

    * 700 MHz processor
    * 256 MB of system memory (RAM)
    * 6 GB of disk space
    * Graphics card capable of 800x600 resolution 


11 years is a lifetime in the world of computer hardware. Hopefully you can find a faster (but still inexpensive) computer for your daughter (try ebay, craigslist, local computer shop, etc.), so she can have a decent web browsing experience. :)
