### Post by Paddy Landau on 2010-04-26
"Cloud" is a metaphor used (among other things) to describe the Internet. When you access stuff over the Internet, you don't necessarily know where it is, so it's in the "cloud".

See Wikipedia's entry:
[http://en.wikipedia.org/wiki/Cloud_computing](http://en.wikipedia.org/wiki/Cloud_computing)

Practically speaking, a cloud can be a (virtual) place where a group of people can store and, optionally, share data.

Amazon provides just one of millions of clouds.

To access a cloud, you just need the required permissions and software. The software will depend on how the cloud is set up. In a sense, every time you access the Internet (e.g. this forum), your requests go through the Internet cloud.

Conceptually, you could have a cloud within a cloud.

It's rather a malleable term, I fear.

### Post by Calash on 2010-04-26
As I understand it a "Cloud" is a decentralized and scalable storage platform that is built to interface with current internet protocols.

For web hosting you can make use of clouds for storage of static information such as images and javascript.  On the cloud their is a large amount of affordable space, freeing up your primary web servers for more important tasks.  Being decentralized it allows for faster access to this data by directing requests to the closest provider.

For example, you create a website that is a gallery and drop the images on the cloud.  A person in LA looks at 1.jpg and gets it from a server in LA.  A person in England looks at the same 1.jpg but they pull the data from a server in England.  You have seamless replication of your data throughout the cloud.

From a desktop perspective it gives you an affordable platform for data storage.  As we move forward we will start to see applications tied into cloud computing, such as Google Docs and how it will be tied into Chrome.
