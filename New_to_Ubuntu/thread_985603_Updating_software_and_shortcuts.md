### Post by bennachie on 2008-11-17
That's unusual, since the default action when you add version 3.0 through the normal add/remove processes - by adding a third-party repository - is to delete 2.4 once the new version is in place. 

If you have successfully installed 3.0 by other means (which can be an elaborate process), the simplest solution might be to delete 2.4 manually. However, if you have just used the standard Ubuntu repositories to "update open office" all you have is an "updated" version of 2.4 (2.4.1 is installed with 8.10, but there may be a 2.4.2 version on the standard Intrepid repository by now).
