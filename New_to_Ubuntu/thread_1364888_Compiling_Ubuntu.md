### Post by oldsoundguy on 2009-12-26
Best to crawl before you walk and walk before you run.  Use the system and get used to what it will do by default before you start adding.  Learn to add before you start making changes.

In a way, Linux is like going back in time for some of us as there are those such as myself that started in the DOS days (before Windows) and now we are re-learning those skills that Windows abrogated and assumed that we did not have enough brains to do!

The major difference is the exponential increase in the quality and quantity of the programing  we now have at our fingertips.

And free is not free.  To use free programming you have to expend effort.  It will NOT do it for you.  If you expect other than that, Linux is not for you.

So ENJOY!!

### Post by LunaticHiatus on 2009-12-26
you can download the source code of the linux kernel or gnu utilities. You can configure them in any manner that you want. You can switch desktop managers, window managers, file systems, boot loaders. Pretty much everything.
Since the source code is available, that means everyone can fix applications and send in patches. If an exploit is found on a proprietary program you can't fix it, your at the mercy of the company that distributes it and you also cant send someone a patch in of your fix.
You can also copy and redistribute linux to any of your friends as many times as you like without legal repercussions. In fact, file sharing is encouraged so there is no need to torrent programs and possible install malware.
However, a lot of people dont really contribute anything. They are just given the benefits freely from the people who do.
