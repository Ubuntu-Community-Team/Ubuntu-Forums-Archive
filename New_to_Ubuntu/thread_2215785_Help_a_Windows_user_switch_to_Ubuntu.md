### Post by Warren Hill on 2014-04-08
The [Ubuntu Manual]("http://ubuntu-manual.org/downloads") is worth downloading it's well written and covers most things.

If you can't find what you are looking for [AskUbuntu]("http://askubuntu.com/") has thousands of questions and answers you can search and if you still need help you can ask a question on that site or here in the [Ubuntu Forums]("http://ubuntuforums.org/").  Most of us are pretty friendly and happy to help.

There are several different flavours of *buntu: [Ubuntu]("http://www.ubuntu.com/download/desktop"), [Kubuntu]("http://www.kubuntu.org/getkubuntu/download"), [Xubuntu]("http://xubuntu.org/getxubuntu/") and [Lubuntu]("http://lubuntu.net/").  You can try them all from USB or CD/DVD without installing to decide which suits you best.  Then install the one you like most.

I'd suggest you dual boot with Windows to start with just because you know it but you will find yourself using it less and less until like me your Windows free (at least at home).

### Post by Mk32 on 2014-04-08
Seconded on the live CDs and the dual-booting. Or, VMs. 

There are at least a hundred different types of Linux but you'll want to stick to the main types. Distrowatch has the most popular ones.

Ubuntu is based on Debian so there's a quite a lot of similarities, and also a number of differences. New users should just take it a bit slowly and just play around a bit. Wait awhile before you start any stunts with the command line.

As for Ubuntu, you may want to browse [this page](http://en.wikipedia.org/wiki/List_of_Ubuntu_releases) to see if you'd like to see any ones you may like. [Then you go here](http://releases.ubuntu.com/) and pick out the one you want -- AMD64 for 64 bit CPUs, x86 for 32-bit CPUs, and so on. I like the Live versions because a burnt CD is a useful emergency tool, although it's not quite the same as a full-tilt live OS like Knoppix.

You can do a lot of cool things with Linux. But as the line in my signature says, it expects that you know what you're doing, so you'll want to pick time to play with it and read a lot.

### Post by coldraven on 2014-04-09
Hello and welcome to the forum. Here is yet another choice, with Linux you always have many.
If you have a reasonably new computer with at least 4GB of memory you could try out distributions in a virtual machine.
It is easier than you may think and has zero cost apart from any download limit that you may have.

First you need to install VirtualBox for Windows from here: [https://www.virtualbox.org/wiki/Downloads](https://www.virtualbox.org/wiki/Downloads)
Check that it runs on your machine and install the VB Extension pack which will give you USB support.
Then download any of the images from here: [http://virtualboxes.org/images/](http://virtualboxes.org/images/)
Make a folder called "VBimages" or similar, move the downloaded image to it and fire it up in VirtualBox.
You can then experiment with the different Desktops or even run a server. Desktops are easy, servers are complicated.
I do the other way around. I use Ubuntu as the host and run XP in a virtual machine as a guest. I can have XP fullscreen on one of my four* desktops and you would not know that it was a virtual machine.

*most Linux distributions have multiple desktops (workspaces) and you can change the amount up to 32 I believe. Very handy for grouping applications that are related. The default number in Ubuntu is four.
