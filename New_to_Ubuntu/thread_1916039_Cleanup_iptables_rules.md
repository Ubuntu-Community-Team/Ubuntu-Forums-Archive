Hi there, Welcome to the forums!

there's a couple of things here. Firstly, all of the things you've tried - UFW, FireStarter, and iptables do the exact same thing - they make those entries in the iptables.rules file. This file is read at boot to determine the behavior of your ports - the firewall. If the iptables.rules is exactly how you want it, you need do no more - it will already load those settings at boot :)

Secondly there is no way to remove all ability to edit the iptables.rules file. UFW and FireStarter are easy to use applications that make entries in the iptables.rules file for you (they are not firewalls in their own right) so you can remove these pieces of software as you would any other, but you would still be able to edit the iptables.rules, either by editing it manually or with the iptables command.

My recommendation is if you have the iptables.rules file exactly how you want it, do nothing, save and leave alone unless you experience errors :)

All the best!

I got a bit confused when they moved to upstart, because the init script was missing (which meant I couldn't do /etc/init.d/iptables save).

Anyways...

go into /etc/network/if-up.d
```

sudo nano firewall
```paste (assuming that your rules are at /etc/iptables.rules!
```

iptables -F
iptables -X
iptables -t nat -X
iptables -t nat -F
iptables -t mangle  -F
iptables -t mangle -X
iptables-restore < /etc/iptables.rules

```Make the script runnable
```

sudo chmod +x firewall
```Now that you have multiple firewall configuration scripts running, their probably conflicting with each other. Remove them.
