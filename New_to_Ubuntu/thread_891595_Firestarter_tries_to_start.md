### Post by JBA2337 on 2008-08-16
Using the Synaptics Package Manager I installed the Firestarter GUI for the IPTABLES firewall on my computer. In Sessions Preferences, I put in an entry for Firestarter, so that it would run every time I started my computer. Later I decided I don't need the Firestarter GUI to run all the time, so I uninstalled it using Synaptics Pkg. Mgr. and I also deleted the Firestarter entry in Session Preferences.

Even though Firestarter has been uninstalled, whenever I boot my computer, I still get this message in a dialog box:
 
  "Enter your password to perform administrative tasks."

  "The application /usr/sbin/firestarter lets you modify essential parts  
   of your system."

/usr/bin/firestarter does not exist on my computer.

Why is my computer asking to run a program that I uninstalled?
Does anyone have an idea how to fix this?

Thank you.

JBA2337

---

### Post by Malta paul on 2008-08-16
Using Firestarter GUI go to 'Preferences>Firewall and untick the Start/restart box.
