### Post by michael1234 on 2008-07-31
I got hibernate to work on mine after carefully following prshah's instructions:

After doing *cat /etc/initramfs-tools/conf.d/resume* and seeing that the UUID there does not match what you get from [I]sudo vol_id /dev/sda5
[/I] and *fstab*, the problem is solved simply by editing */etc/initramfs-tools/conf.d/resume* in a text editor with root access:

```
sudo gedit /etc/initramfs-tools/conf.d/resume

```

You will see ```
RESUME=UUID=c569bfa8-1f4e-3e9d-0c66-cfcb0e9d11fa
```

Simply change this to ```
RESUME=UUID=f693e159-860c-4420-84f7-8a2d37e8c26a
``` so that it matches what your sda5 vol_id and fstab say. Save the file. Then go back to the terminal and press CTRL + C to close the editor. When the prompt reappears, enter ```
sudo update-initramfs -uv
``` to make sure the initialization files recognize the new vol_id.

I think that's it. This did work for me. Thanks given to prshah for that informative post.
