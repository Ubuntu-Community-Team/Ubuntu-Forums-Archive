### Post by JoshuaRL on 2009-01-07
Welcome to Ubuntu!

And for the most part, compiling from source is unnecessary unless you need something really special or need it customised for you.  If it's in Add/Remove, let's see what Synaptic thinks about it.  Go to System->Administration->Synaptic Package Manager.  Then search for Ardour GTK2.  Right click the package you want and mark for installation.  Then hit the Apply button and tell us what happens.

### Post by JoshuaRL on 2009-01-09
Typically, yes.  Add/Remove is more "new user friendly" while Synaptic is much more powerful.  Sometimes, like you've seen here, Add/Remove has issues getting things installed if it has weird dependencies.  But with Synaptic you don't have those problems, plus it can fix broken packages for you too.  

Add/Remove is like a Honda Civic.  Well made, efficient, but kinda basic.  Synaptic is like a military Humvee.  Strong, powerful, and can be outfitted to do just about anything.
