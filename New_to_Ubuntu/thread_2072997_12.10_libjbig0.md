### Post by pbrandao on 2012-10-19
Hi,

I had a similar problem (also with libtiff4 and libusb as broken) and unable to remove linux-image 3.2 has several packages still depended on it.

After doing 
```
apt-get -f install
```the needed dependencies where installed which.

After I was able to do
```
apt-get autoremove
```(which removed the just installed libtiff4 and other packages.

Also removing the linux-image for kernel 3.2 was also possible.

HTH
