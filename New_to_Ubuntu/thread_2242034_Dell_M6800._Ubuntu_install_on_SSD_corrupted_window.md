### Post by Sultanaman on 2014-08-30
Hi there this is my first post:
I have been using ubuntu for 5 years on a Dell precision  M6500 SDD with grub & dual boot, but previously loaded via WUBI from within Win 7.  Machine was set up with 500GB HDD as data and 256GB SSD as operating disc prior to my receipt of machine and both operating systems resided on SSD. Never had any problems – but this time I got a big fright on a new machine!!

The new machine is Precision M6800 with 256 GB SSD and 1 TB HDD  and despite asking DELL to make sure it was shipped with the Win 7 operating system on the SSD they put it on the HDD. Why I don't know and not it’s easy to communicate with them, as they seem to take little interest once shipped. So I used Paragon's HDD to SSD clone software ($20) to clone Win 7 onto the SSD successfully, with the intention of then loading ubuntu also on to the SSD exactly as before) and once all working off the SSD OK, I'd then delete Win 7 from the HDD. However, although unbuntu loaded OK onto the SSD the ubuntu load corrupted the SSD version of Win 7 so that when booted for the first time from there I got the blue screen of death.  I freaked!!!!
 
Next time I booted  Win 7 from the SSD it went to the chkdsk function, which found  a bad sector – or something like that – which it fixed and then told me, that because windows had crashed last time, I should load using safe mode. I then loaded windows successfully in safe mode.  I then tried again and full windows loaded successfully from the SSD – phew!!!!!!
