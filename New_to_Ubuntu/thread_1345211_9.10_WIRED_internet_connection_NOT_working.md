### Post by 3rag0n on 2009-12-03
@SW90 and @sudhir :

From what I've heard, and experienced sometimes, *dhcpcd* seems to work better than *dhclient*. I'm not exactly knowledgeable about networking, but I've heard that sometimes dhclient tends to request IPv6 addresses while dhcpcd requests the more traditional IPv4. So try this - take someone else's computer, download the dhcpcd deb package - **you might want to use ubuntu package search** (there are probably no extra dependencies ) and then install it in your computer.

Then run this command :
```
dhcpcd eth0
```

Try it and see if it gets an IP address for you. dhcpcd has never failed me on my Arch Linux system ( In fact, dhcpcd is the recommended way in the Arch forums ).
