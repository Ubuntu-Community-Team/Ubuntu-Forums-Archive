### Post by wilee-nilee on 2012-05-22
Funny the sda5 partition where Ubuntu is has the correct boot files but the script is missing some stuff, lets try a straight chroot in with some commands, just run them as posted from a live ubuntu cd.

                                  [FONT=Verdana, sans-serif][SIZE=1]```
sudo mount /dev/sda5 /mnt
``` [/SIZE][/FONT] 
 [FONT=Verdana, sans-serif][SIZE=1]```
for i in /dev /dev/pts /proc /sys; do sudo mount -B $i /mnt$i; done
``` [/SIZE][/FONT] 
 [FONT=Verdana, sans-serif][SIZE=1]```
sudo chroot /mnt
``` [/SIZE][/FONT] 
 [FONT=Verdana, sans-serif][SIZE=1]```
grub-install /dev/sda
``` [/SIZE][/FONT] 
 [FONT=Verdana, sans-serif][SIZE=1]```
grub-install --recheck /dev/sda
``` [/SIZE][/FONT] 
 [FONT=Verdana, sans-serif][SIZE=1]```
update-grub
``` [/SIZE][/FONT] 
 [FONT=Verdana, sans-serif][SIZE=1]```
Exit chroot: CTRL-D on keyboard
``` [/SIZE][/FONT] 
 [FONT=Verdana, sans-serif][SIZE=1]```
sudo reboot
``` [/SIZE][/FONT] 
  
If this does not have ubuntu showing in the grub menu on a reboot let us know we can run a purge and reload of grub from another chroot. Save any errors if there are any if you can.

The chroot is basically getting you into the the root of the install directly.

I'm surprised the boot-repair did not work, may need the purge we will see.
