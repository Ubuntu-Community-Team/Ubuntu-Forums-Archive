### Post by oldfred on 2010-06-29
Grub/Ubuntu does not use boot flag. Windows requires a boot flag if you ever go back and want to directly boot windows. Some BIOS require a boot flag on a primary partition to let any system boot (BIOS assumes windows ). So boot flag is fine on sda3.

A grub installed to a partition's boot sector is not a problem unless it is the windows partition. If installed correctly it can be used for chainbooting but that is for advanced installs. It does not use space otherwise used as the first sector of a partition is reserved for a boot sector. So grub in sda4 is not a problem.

Does Ubuntu boot ok? Did your windows boot ok after you installed it and before you reinstalled grub to the MBR? The entry in your grub.cfg for windows looks correct.

You could try updating grub but if it is a windows issue this will not fix it. Grub only boots working windows. 
sudo update-grub
