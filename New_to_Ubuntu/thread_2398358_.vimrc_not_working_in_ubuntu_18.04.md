### Post by Impavidus on 2018-08-11
Creating ~/.vimrc should work. It has always worked for me and still does on 18.04. Does your .vimrc have the right owner and permissions? It should be owned by you, read and write permission by you. It's probably setup correctly, but if the first time you ran vim after your fresh install was with sudo, it may be wrong.

Also make sure you actually have the package **vim** installed, not just **vim-tiny**. vim-tiny is a compact version of vim with only the most basic features that you need to fix your system. (Interestingly, even vim.tiny isn't stored in /bin, where you only find ed and nano.) When you have both packages installed, you should have a symlink pointing (indirectly) from vim to vim.basic. If you only have vim-tiny, it points to vim.tiny. I think that by default only vim-tiny gets installed.
