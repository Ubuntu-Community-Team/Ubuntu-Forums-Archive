### Post by mcduck on 2009-08-11
They are just code names used for Ubuntu versions during development (as the actual release number comes from the release date, and there's always the possibility of having to delay the release, which would change the version number as well).

Many people find the names nice and prefer using them over the boring version numbers

For example the latest Ubuntu version was developed with the name "Jaunty Jackalope". It was released in April 2009 and thus the final release was named "Ubuntu 9.04". Had the release been delayed by two months the released version would have been called "Ubuntu 9.06" (which is what happened with Ubuntu 6.06..)

Using code names  for development versions of software is very common practice, Firefox 3.5 was known as "Shiretoko" during it's development, and Windows Vista was developed with the code name "Longhorn".
