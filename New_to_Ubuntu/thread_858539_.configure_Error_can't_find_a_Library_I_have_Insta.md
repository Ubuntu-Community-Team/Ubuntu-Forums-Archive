### Post by unutbu on 2008-07-13
Find the directory where libgslcblas is located.
Then read the README or INSTALL file that came with the source code, or run 

```
./configure --help
```

One of these should tell you what the right command-line option is that you need to add to tell the configure script to look in the special directory which contains libgslcblas.

configure then passes on this lib directory to the Makefile, and the makefile will add it to the gcc command line, etc.

### Post by unutbu on 2008-07-14
Are you compiling LessTif from source? Perhaps instead you can install these packages:

```
lesstif2 - OSF/Motif 2.1 implementation released under LGPL
lesstif2-dev - development library and header files for LessTif 2.1
```

lesstif2 contains

usr/lib/libXm.so.2, so this provides the Xm library. 

lesstif2-dev contains the .h header files you'll need to compile programs that use libXm.
