### Post by QIII on 2010-05-20
What you learn from the pocket guide will be applicable across all releases.  

This is a moving train.  Sometimes you have to jump in a car and get your feet on the ground.  The interior of the car (analogous to the UI of the OS) is similar in layout.  There are seats, a floor, a ceiling, hand rails, doors, etc.

You have older cars and newer ones.  Each have different features.  But if you are a new rider, you will really only be interested in the normal stuff.

You didn't learn Windows in a day.  You won't learn Linux in a day either.

But you always have us to ask when you need to.

Just a bit of advice, though...

As first.  It's a lot easier for us to guide you along than to get you out of a jam.

Welcome to Ubuntu.  Hang on for a fun ride.

In the not too distant future, you are going to be the "old hand" helping others along!

I have books going back to the 70s on Linux and in the terminal using the command line is almost the same ......

Nowadays you usually just have to put .......

sudo 

in-front of everything ....... and enter a password ...... to run these type of commands
even though all the way through your life you may have been root user .....

As for operation ...... they have hidden some useful information that the **USER** used to receive at boot up its hidden behind splash screens.
(but it can be found in logs - we used to be able to press escape to see the information but that seems to also have been disabled now ,,, so that will not work)

Also ctrl+esc used to show you all the running processes when you reached the desktop ...... that's been disabled too .... its of no use to the **USER** ..... showing which process was hogging the system ..... and allowing you to kill it and carry on working ...... a bit like ctrl+alt+del in Windows ,,,,

Its only the quick commands that show useful information that are being removed.

All the other things like getting a MP3 player to work or Adding Applications to the System seem to have changed ...... but a handbook should not be needed - as long as
you know how to use the Software Centre.
