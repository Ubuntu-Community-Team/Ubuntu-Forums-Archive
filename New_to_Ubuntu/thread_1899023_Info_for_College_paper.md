### Post by keithpeter on 2011-12-23
Hello Archer48

Some great suggestions up the screen, and booting up an Ubuntu CD will certainly help you see what we are on about. It all depends on what you are being asked to write about. I'll take a step backwards a little...

[SIZE="1"]**History**[/SIZE]

GNU/Linux, the various BSD operating systems, and Apple Mac OS X are all described as 'unix like' operating systems, so a bit of Unix history might help. I'm 53 and first tripped over Unix on PDP11 midi computers (teletypewriters and punched tape)...

[http://en.wikipedia.org/wiki/Unix](http://en.wikipedia.org/wiki/Unix)

[SIZE="1"]**Command line (Bash)**[/SIZE]

On all of those systems, you can start a terminal session and issue commands (like a DOS or Command Prompt session). The commands are powerful because of something called pipelining - basically the output of one command can be 'piped' into the input of another command. More on that at

[http://linuxcommand.org/](http://linuxcommand.org/)

[SIZE="1"]**Distributions**[/SIZE]

GNU/Linux consists of the Kernel, the core of the operating system, which is the Linux bit. Then you need programs that actually do things, thats the GNU bit. Because all of this is free (both no cost and that the source code is available for modification) any group of people can release a 'distribution' of GNU/Linux. 

A 'distribution' consists of an installer, and a way of updating the operating system and a way of adding new software. All the thousands of programs available in Fedora or in Ubuntu are 'packaged' by people so that you can install them, and so that updates and bug-fixes can be applied automatically. Google 'dependency hell' to find out how problematic that can be. Each 'distribution' (like Ubuntu, Fedora, CentOS or Red Hat Enterprise Linux or many others) also provide ways of reporting bugs, and they also publish plans for the future. With Ubuntu, that is handled on launchpad ([https://launchpad.net/](https://launchpad.net/))

[SIZE="1"]**Bug fixing: many eyes and free software process**[/SIZE]

The science fiction writer Neal Stephenson wrote an amusing essay about GNU/Linux and the command line - this essay also explains how the bug fixing system works....

[http://www.cryptonomicon.com/beginning.html](http://www.cryptonomicon.com/beginning.html)

Another key essay on why free and open source software (foss, software libre) gets developed quicker is by Eric Raymond

[http://www.catb.org/~esr/writings/homesteading/cathedral-bazaar/](http://www.catb.org/~esr/writings/homesteading/cathedral-bazaar/)

Some people don't think that the open source development method leads to good user experiences...

[http://bitquabit.com/post/one-which-i-say-open-source-software-sucks/](http://bitquabit.com/post/one-which-i-say-open-source-software-sucks/)

**Ubuntu on phones, tablets, televisions**

... and that leads us back to Ubuntu. The chief is called Mark Shuttleworth and he wants Ubuntu on a huge range of devices...

[http://www.markshuttleworth.com/archives/820](http://www.markshuttleworth.com/archives/820)
