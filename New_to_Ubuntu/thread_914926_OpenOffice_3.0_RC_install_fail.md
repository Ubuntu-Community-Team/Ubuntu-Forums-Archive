### Post by quixote on 2008-09-14
Okay, I still don't get it, but here's my experience after following the words of wisdom in [this thread]("http://ubuntuforums.org/showthread.php?p=5785818#post5785818").

cd to uppermost unpacked files dir named OOO-interminable-name/
```
 sudo sh update
```
don't let any "skipping" messages worry you
then cd into the DEBS dir, and run
 ```
sudo dpkg -i *.deb

One other thing to keep in mind, when you untar the archive a subdirectory called desktop-integration is created--in the final release, anyway--under DEBS. If you run
```

$ sudo dpkg --install --recursive ./*

```

from within the DEBS directory the start menu stuff will also get installed. dpkg does not descend in to subdirectories without the --recursive switch. If you're installing on an AMD64 system add the --force-architecture switch, as well. This worked for me on my HP zv5000 with an AMD64 3700+.

Also, for what it's worth, I have the libc6-i386 shared libraries installed, as well as ia32-sun-java6-jre and ia32-sun-java6-bin. These were all installed prior to Openoffice. Openoffice found the 32bit java runtime just fine.

I also grabbed a 64bit version of Openoffice3 and installed it over the 32bit version after testing. See this thread for more info: [http://ubuntuforums.org/showthread.php?t=927363](http://ubuntuforums.org/showthread.php?t=927363) 

Good luck!
