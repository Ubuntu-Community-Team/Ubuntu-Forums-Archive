### Post by Kapitän Rotbart on 2009-10-03
Here's my thoughts, hope it helps
[LIST]
[*]Firefox is a little bloated on any OS. FF 3.5 is good, but if you want a fast browser, try Epiphany or Midori
[*]Flash is fairly stable. If you wanna be adventurous, you could try Gnash, it may actually work better.
[*]Ubuntu is not complicated to install/use. The install supports most hardware, and when you gotta fix things, it's usually a matter of copying a couple commands over from the forums.
[*]Any tweaking you find tricky? You'll be rewarded with the bragging rights. I started using Ubuntu Feisty 7.04, and back then I had to configure ndiswrapper just to get wlan working (not to mention the other tweaking I had to do just to get the desktop working properly). Copying those commands have taught me a great deal about computers. You also get the peace of mind that Ubuntu just keeps getting better, your hardware will never lose support and that within a year or so, everything will work out of the box (in my case it was a matter of weeks, by 7.10 Gutsy that laptop worked perfectly upon install).
[*]The software gets better rapidly. In Ubuntu 9.10 Karmic, installing apps, fonts, etc. will get much easier, Empathy should officially support more feats., etc. and this will only get better with time. Plus boot time keeps getting shorter, so you don't have to worry about that "magic toaster not toasting."
[*]Use the official releases. I'm tempted to run the beta of Karmic myself but until the release, your functionality nightmares will multiply (unless you're doing it to report bugs).
[/LIST]

Seriously though, Ubuntu is such a cool distro and makes me wonder why I ever used windeuce. BTW, windeuce will be able to burn a disc image to CD (without burning a CD with the ISO file on it) for the first time in win7, something Linux has been able to do since forever. Vista was the first windeuce version to ship with a browser that supports RSS whereas Firefox on Linux has been able to do that for much longer. I'm just too impatient to wait for m$ apps to do what I need software to do.
