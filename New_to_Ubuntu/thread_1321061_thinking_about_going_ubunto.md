### Post by Lateralis on 2009-11-09
lorddenis and welcome to the forums! 

In the first instance, I'd strongly recommend you download a copy of Ubuntu form the Ubuntu website, burn the iso to a CD (I'm assuming you have a programme to do that, but if not, there is a programme called ImgBurn which works well) and boot to it.  This is a LiveCD, so you can run Ubuntu for a little while and see what you think of it.  You won't be able to do everything you'd be able to do in a full installation as it is just a LiveCD - it writes nothing to the hard drive.  But you'll be able to get a feeling for the operating system and see if your hardware (sound, video, internet etc...) will work fine under Ubuntu.  If it does, then you have several choices. You can either install a copy of Ubuntu into a Windows directory using Wubi (load up Windows then insert the Ubuntu disc into the CD drive) or install Ubuntu side-by-side with Windows in a so-called "dual boot" configuration.  This is what I do all the time and works well.  =)  

As for all of the programmes you mention, there are Ubuntu equivalents - or native Linux versions supported by the original software company; amsn for MSN and GIMP for Photoshop for example.  Whilst you say that you'd prefer to use Photoshop I love GIMP.  I've never used Photoshop before though, so this might just be a matter of choice.  Now, that being said, there is a programme called Crossover by Codeweavers, that will allow you to install Adobe Photoshop 6 or 7.  It is a programme that is fully supported by Crossover, so it should work exactly as in Windows. 

As for games, I do know that Unreal Tournament 2003 and 2004 actually came with shell installation scripts with the CD.  Thus, they'll install and work natively in Linux!  Dunno about Age of Empires, but there is a version of Crossover called Crossover Games which might be able to handle it.

As for using external drives... support for these devices is normally very good.  You'll be able to test this when you use the LiveCD though. =)  

The only thing that I would personally be concerned about is the amount of hard disc space you have.  I would personally set aside at least 15 gigabytes of space for Linux (I normally do at least 20, especially if you want to install games), but it depends how much free space you actually have.  


So yeah... My general advice would be to "suck it and see" by using the liveCD.  You said that you've read some horror stories in the forum about Updates and installations not working properly, but:

1) a lot of these issues have been cleared up quite quickly and easily
2) updates and fixes for these issues are entering the software repositories very quickly.  And there is no way of knowing if you'll encounter a problem unless you actually try it out.  


Hope this helps! =)

You can buy a Mazdaspeed3 and have 260 (flywheel) HP from the factory. You can also take a Honda Civic and tweak it and make almost however much HP you want. ### Post by 3rdalbum on 2009-11-13
> **lorddenis said:**
> thanks for that explanation...

can somevone explain how  Wine work?

Wine is a piece of code that understands the EXE file format, and a set of reverse-engineered, re-implemented DLLs. It basically runs Windows programs using its own versions of the Windows libraries. The Wine DLLs are wrappers that will run native Linux equivilants to Windows functions.

But I guess you wanted to know HOW to use Wine, not how it works :-)

Basically, double-click on a Windows program installer and it will run just like on Windows. When it has installed your program, you will find it in the Applications > Wine menu, or inside the hidden .wine folder in your home directory.

Note that Wine runs less than 50% of Windows programs out there. A lot of programs won't work. Some programs can work, but only with extra configuration (sometimes complicated). It's an enormously difficult task to achieve Windows compatibility and what the Wine developers have done is very impressive, but don't expect to run any of your favorite Windows programs in it; that way you won't be disappointed.

One of the best projects around at the moment is Play On Linux. It's a program that contains a database of what Wine settings to use with certain Windows games. If you have a game that's on the Play On Linux supported list, all you need to do is select the game from POL, insert your game disc, and everything will be set up for you. Very easy and very cool.
