If the .bin file is designed to be run from a terminal, you won't be able to run it in any other way.
But it's really not that complicated.
```
cd /path/to/file/
./filename.bin
```Or:
```
/path/to/file/filename.bin
```As most installers require root privileges, you usually need to put "sudo" in front of the command:
```
cd /path/to/file/
sudo ./filename.bin
``` (./ means "in this folder")
Or:
```
sudo /path/to/file/filename.bin
```So if the file is named "SomeFile.bin" and is in the Downloads directory in your home folder (the path to your home folder is /home/username, but can be simplified to ~/), the command would be:
```
cd ~/Downloads
sudo ./SomeFile.bin
```Or:
```
sudo ~/Downloads/SomeFile.bin
```You could also install the package "nautilus-open-terminal" which will allow you to open a terminal from a right-click menu in any nautilus window. This way you can browse to where the file is in nautilus, open a terminal and it will be in the correct directory already...

For your information:-
The first line command, **cd** simply changes the working directory to the Downloads directory in your /home where the file you need to execute is sitting.
The second line command, **chmod +x** makes the file mentioned after it executable.
The third line simply executes the file by adding the **./** in front of the named file, provided you are already in the directory containing the file, which you moved to in the first command of the three.

I hope that helps you for future use of terminal;  don't be afraid of the terminal as it can be a very quick way to do some simple things, and some complicated things as well, if you are prepared to learn.
