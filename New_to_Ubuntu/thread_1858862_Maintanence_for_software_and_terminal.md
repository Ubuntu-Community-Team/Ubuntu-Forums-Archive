### Post by lolpenguin on 2011-10-13
Being new to linux, you should refrain from using tools like terminal, synaptic or similar advanced programs until you actually understand how they work. It is best to install software using the software center provided. Leave them alone for now, and consider coming back to them when you're more experienced.

### Post by Lisiano on 2011-10-13
As lolpenguin said, refrain from using the advanced tools until you feel comfortable with the basic tools. If we ever request from you some information that is quite advanced compared to what you know, we will tell you it step by step, if we tell you something complicated, ask us to simplify it, most won't mind simplifying it because sometimes we forget the experience level someone has.

### Post by Lisiano on 2011-10-13
Also could you post the links to the threads where you found the commands? Some guides can be straightforward and easy but some can be both hard and dangerous. There are (Most of the time) multiple ways to solve a problem, with a simpler way and a harder way.
As ajgreeny said, some commands won't print anything if they were successful (For example "sleep 2s") but some won't print anything if they were not successful (grep a non-existing string) so knowing what commands you used exactly (And the exact argument too) is vital in knowing if the command succeeded or not.
