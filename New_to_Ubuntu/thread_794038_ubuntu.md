### Post by lazyart on 2008-05-14
First thing is to ease yourself into it.  Ubuntu is like a kitchen in a foreign country.  The recipes are a bit different, as are some of the utensils but given time you can adapt to the native style and really do some cooking!

Gimp comes close to Photoshop-- you can try it in Windows and see if it works for you.
Spreadsheets?  There is the OpenOffice suite, along with Gnumeric if you want something lighter.
DTP--  There are tons of 'em.
Dreamweaver-- I'm not sure of the linux alternative here but I've seen that one asked before so there must be something.
Email-- Heavyweight is Evolution but I don't care for it myself.  Thunderbird is the usual second choice but again, there are tons of options here.
Fast user switching-- you can do that too.
AVG-- sorry, no answer for you there.  If you must, there is avast for linux and ClamAV if you are worried about virii.

Of course if you're a software tester by trade you'll likely be keeping windows anyway.  No worries-- they can play together.  I run a domain off of Server 2003 at home and my linux machines are joined to it.  Anyone in the house can log on to the ubuntu boxes because they are all using the same authentication.  Even the virtual machines within Ubuntu are with the program (just installed Fedora 8 last night in a VM and logged in using my domain credentials).

But I started slow.  I had a spare box, installed Dapper Drake and spent a few minutes a day on it.  Then an hour a day.  Next thing I knew I was spending just a few minutes a day on the Windows machine, and most of my time on Linux.

Don't make yourself suffer.  Find out what you need out of the Windows side and see if it works for you on the linux side.  Don't fall into the trap of adding all the eye candy without really knowing how to do things.

And have fun.  It's not an either/or thing-- it's okay to use both!

### Post by kansasnoob on 2008-05-14
I highly recommend starting with a dual-boot unless you have a computer you can dedicate to just fooling around while you make up your mind.

Of course you can now install 8.04 inside Windows with Wubi, but my personal preference is dual booting. It gives you time to play with Ubuntu and yet you need only restart the computer and switch to Windows in GRUB if you're having trouble working out hardware issues, etc.

Also, I like to fiddle around and learn things, which means trying new things that sometimes "breaks" my Ubuntu. With a dual boot that's no problem, I just toss in my Ubuntu Live CD, use Gparted to erase my Ubuntu partition(s), restart (still with the Ubuntu Live CD) and reinstall Ubuntu, and start from scratch again without ever harming my Windows system.

The only possible hangup is having to reinstall the Windows MBR if you should decide to toss Ubuntu altogether, which I'd say is doubtful. And that's even pretty easy if you have the Windows system disc(s).

I've now progressed to using a shared data partition so I can access files from either Ubuntu or Windows.

The simplest dual-boot guide I've found (and the one I started with) is:
[http://apcmag.com/how_to_dual_boot_windows_xp_and_linux_xp_installed_first.htm](http://apcmag.com/how_to_dual_boot_windows_xp_and_linux_xp_installed_first.htm)

Some other really helpful guides:
[http://users.bigpond.net.au/hermanzone/](http://users.bigpond.net.au/hermanzone/)
[http://www.psychocats.net/ubuntu/installing](http://www.psychocats.net/ubuntu/installing)
[http://www.psychocats.net/ubuntu/partitioning](http://www.psychocats.net/ubuntu/partitioning)

I'm currently just learning some of the great things that can be done (recovery and such) with the Ubuntu alternate install CD.

### Post by Xiong Chiamiov on 2008-05-15
From personal experience, here's what I have to say:
1)  **Things will break more.**  However, not only is there a fantastic community here to help you out, you can be sure that there *is* a way to fix it.  Maybe you'll spend several hours early in the morning trying to figure out just *why* your wireless card stopped working, but gosh darnnit, you'll fix it and learn a lot in the process.
2)  **You'll hate installing things in Windows.**  Once you've gotten used to the package manager (please, read [this]("http://monkeyblog.org/ubuntu/installing/")!), you'll be super-annoyed when you log into windows and find that you have to go visit a website to download a program.  And then I have to install it too?  And press next a bazillion times?  And then curse because it requires some other app I don't have installed yet?
3)  **Things are usually must faster.**  Not only are there some basic elements of how the OS is built (memory caching and such), but you can eliminate many of the apps you run on Windows.  I no longer have to run an anti-virus, firewall (my router blocks inbound, I just have on windows to protect against trojans and such), anti-spyware apps, etc.  On top of that, Compiz allows some pretty decent visual effects without slowing things to a crawl.
4)  **You'll get used to being able to customize everything.**  You want to change it?  Most likely, you can, no matter what it is.  Now, of course, I'm a KDE user, rather than GNOME, so it's a bit easier to customize, but let's not open up that discussion...
