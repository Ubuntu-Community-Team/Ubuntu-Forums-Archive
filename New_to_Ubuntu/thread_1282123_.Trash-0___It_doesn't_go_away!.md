### Post by cat2005 on 2009-10-04
I finally found a solution.  If anyone else has this trouble, then do the following:

1)  Change ownership of the .Trash-0 (sudo chown .... be careful) to a certain user.
2)  gconf-editor -- go to apps -- nautilus -- preferences -- and select "enable delete"
3)  Go back and delete those mofos - skip the trash can

Also, go into each user account and look for the .local (hidden) folder.  Keep going until you find the "trash".  Sometimes stuff hides there.

Surely this can not be normal behavior.  For you more knowledgeable folks, would this be considered a "bug" or something else?  Whatever it is, it is extremely annoying (and space wasting).  One should not have to take those cumbersome steps to eliminate unwanted folders and files.  Should I report this as a bug?

### Post by drs305 on 2009-10-04
I don't think what you experienced were bugs, but the situation does cause problems to linux users not familiar with how deleted files are handled by Ubuntu.

I'll try to explain a bit about Trash. Each user has their own trash bin. In recent versions of Ubuntu, Trash is located in each users ~/.local/share/Trash folder. Inside are two folders - *files*, which contains the deleted files/folders, and *info*, which contains information about the deleted files/folders.

By default, when a file or folder is deleted it is moved to the applicable Trash folder. Root's trash is located in /root/.local/share/Trash. This gives the user the opportunity to retrieve a file or folder the user accidentally deletes. The deleted files remain in the Trash bin until it is emptied. The deleted files also take up space, and continue to do so until the Trash bin is emptied.

Another concept to remember is that a user can only delete his/her own trash unless admin rights are invoked. This can be done by using "sudo rm" (caution: dangerous if misused) or by opening a GUI file browser with "gksu" or "gksudo" (example: gksudo nautilus). 

Getting to your situation, files removed via a GUI app in the normal manner (using the DEL key) are moved to the Trash folder. If the user employs the DEL key on something in Trash, it is deleted once again and again is moved to the Trash. That is why you see it disappear momentarily and then reappear.

Ubuntu incorporates the ability to empty the trash bins by right clicking on the Trash icon and selecting "Empty trash". The user can also add the option to bypass the Trash bin, in which case the file is removed from the system. In both these cases, the removal of the deleted files also frees up the disk space. 

Problems often occur when a user uses 'admin' rights to delete files as root but forgets to view and empty the root trash bin. This bin is not normally visible to a normal user, but the trash that accumulates therein continues to expand as more files are deleted by root and eventually can take up a good deal of disk space. If it was root trash you were working with, you wouldn't see it if you were logged in later as a normal user.

I wrote several guides about finding ways to free up space, but the first of the series was about Trash:
[How To: Disk Full? - Check Your Trash]("http://ubuntuforums.org/showthread.php?t=898573")
The post includes several "universal" commands that will search your entire system for trash belonging to any user.
