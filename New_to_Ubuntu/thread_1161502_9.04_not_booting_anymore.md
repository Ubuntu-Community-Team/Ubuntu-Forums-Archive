### Post by guywithcable on 2009-05-17
You don't need to do all that. Try this:

[LIST=1]
[*]Go to the live CD
[*]Mount the Ubuntu partition
[*]Run "gksu nautilus"
[*]From nautilus, go to the drive and create a new folder in it called "broken-system"
[*]Move everything but the home folder into that new folder
[*]Rename the home folder to "home-old"
[*]Close everything and unmount the disk
[*]Open the installer from the desktop
[*]When you get to the partitioner, choose manual partitioning
[*]Select the Ubuntu partition and choose to use it as ext3 with a mountpoint of /
[*]DO **NOT** FORMAT IT!
[*]It will tell you that it is going to overwrite any system folders. That's ok because you moved all the system folders.
[*]Be sure you use the same username as before
[*]Let it install, then choose to keep using the live CD
[*]Mount the partition again
[*]Run "gksu nautilus" again
[*]Go to the drive, and rename the new home folder to "home-empty"
[*]Rename the home-old folder to "home"
[*]Close everything and reboot
[*]Once you're in the system, you need to reinstall all your packages, but all your settings and files should be there.
[*]If you changed any config files in /etc you can bring them back from /broken-system/etc
[/LIST]
Hope that helps. :D

### Post by guywithcable on 2009-05-18
Sorry to be unclear. You would go into the Try Ubuntu part. Mounting basically means turning the partition on, or getting ready to use it. By default, all internal media (hard disks, cd drives) are not mounted, so you need to mount them in order to access and edit their files. The easiest way to do this is to go to the "Places" menu, and click on the appropriate media. It will probably be named according to its size (XX GB Media), so you may have to hunt to find the right one. (Hint: it's the one without a Windows folder in it ;) ) Anyway, once you click these entries, Gnome mounts them for you and places a shortcut to them on the desktop (they are really mounted in /media). To unmount them, right click their icon on the desktop, and there will be an option called Unmount.

The reason it's referred to as "mount" and "unmount" is because Unix (what Linux is modelled after) was written for big mainframe computers, which didn't have little hard drives like our computers; they had big, heavy magnetic drives that had to be lifted up to and mounted on a device. So, the Unix command to initialize one of these disks was mount. Keeping in that tradition, Linux uses the command mount to initialize media.

Here's an example of an old auto-mounter that mounted tape drives:
[http://www.computerhistory.org/collections/accession/102675069](http://www.computerhistory.org/collections/accession/102675069)
(I bet you the computer it was used on ran Unix :D )

Hmm. Well I can explain each part so you know what they do:

```
apt-get --reinstall install x11-common gdm
```

"apt-get" is the command line package manager for Debian based systems.
"--reinstall" tells the install command to remove the packages before installing them
"install" is the command for apt-get to install packages
"x11-common gdm" is the list of packages to work on (reinstall)

X11 is the eleventh version of X, the windowing standard for Unix/Linux systems. So if something happens to your X system, you may not be able to get into a graphic mode, which is what I think is happening to your computer. Then GDM is the display manager that Gnome uses (Gnome Display Manager). It manages graphical logins to a running X server. So that might also be damaged.
