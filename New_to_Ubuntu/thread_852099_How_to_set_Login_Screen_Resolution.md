### Post by forger on 2008-07-07
Is this ubuntu hardy heron 8.04? Please answer in your next post

Let's try recreate the xorg.conf.
**[COLOR="Red"]Warning!! Could break your system! Use at your own risk.[/COLOR]**

First **backup** that xorg.conf file:
```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.backup
```

then use nvidia-settings (**not** displayconfig-gtk) to setup the xorg.conf:
```
sudo apt-get install nvidia-settings
gksu nvidia-settings
```

head to "X server display configuration" > Detect displays > Choose your default resolution and refresh rate > Save to X configuration file > Quit
then go to System > Log out

It should load the display you've chosen.

Reply with what happened, if everything went ok or if any problems found :)

P.S. If anything goes wrong and X can't start, you can easily revert back to the old xorg.conf:
```
sudo cp /etc/X11/xorg.conf.backup /etc/X11/xorg.conf
sudo /etc/init.d/gdm restart
```

### Post by bab1 on 2008-07-09
I have been watching this thread with some interest as I also have had login screen issues.  I believe that the login screen is called the "greeter" and the Gnome Display Manager (GDM) site describes it pretty well.  
See:  [http://www.gnome.org/projects/gdm/docs/2.8/gdm.html]("http://www.gnome.org/projects/gdm/docs/2.8/gdm.html")

The steps in booting to the greeter display (login) are:
[LIST=1]
[*]BIOS bootstrap
[*]Grub Bootloader - see /boot/grub/menu.1st
[*]Linux O/S runlevels - hidden by /etc/usplash
[*]GDM Greater (Login) - controled by xorg.conf
[*]The desktop  -- see xorg.conf

[/LIST]
So what I see here is that the greater screen is defined by GDM and its [COLOR="Red"]*display*[/COLOR] is controled by xorg.conf (X-server).  What a mess. :(

What I found out this weekend:
I bought a new monitor (for other reasons) and fully expected to have the problems to continue.  I WAS WRONG.  It works perfectly!  

The conclusion I have come to after  a ton of reading is that the monitor that is *[COLOR="Red"]detected[/COLOR]* is what the X-server configures to.  Not all monitors are detected correctly.  I changed out an old CRT to a 19" widescreen and changed nothing else.  No recofig of Xorg at all.  

I have since changed the usplash.conf, but that was for my own satisfaction that it works.

I believe also that when you do a reconfig of Xorg (sorry I forgot the incantation), the X-server probes the monitor and vid card and then asks you for the rez you want to list.

Also, If you get too far out of what the X-Server expects, it defaults back to plain old VGA (800x600 I believe).

The bottom line is that there is a lot of interchange between the Video card and X-server and the monitor.  It will always be a black art and just plain luck. 

Sorry for such a long post.

   Modes  "mode-name" ...
              This  optional  entry  specifies the list of video modes to use.
              Each mode-name specified must be in double  quotes.   They  must
              correspond  to  those specified or referenced in the appropriate
              Monitor section (including implicitly referenced  built-in  VESA
              standard  modes).   The  server will delete modes from this list
              which don’t satisfy various requirements.  The first valid  mode
              in  this list will be the default display mode for startup.  The
              list of valid modes is  converted  internally  into  a  circular
              list.    It  is  possible  to  switch  to  the  next  mode  with
              Ctrl+Alt+Keypad-Plus and to the previous mode with Ctrl+Alt+Key&#8208;
              pad-Minus.   When  this entry is omitted, the valid modes refer&#8208;
              enced by the appropriate Monitor section will be used.   If  the
              Monitor  section  contains  no modes, then the selection will be
