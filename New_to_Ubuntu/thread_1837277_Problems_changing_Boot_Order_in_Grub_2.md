### Post by philinux on 2011-09-01
> **Zood said:**
> I have clearly stated that I have installed startup-manager; it did not help solving the problem.

Another easier solution. Choose whichever is best for the user.

```
gksudo gedit /etc/default/grub
```

Add this line to the end of the file.
GRUB_SAVEDEFAULT=true
Then save it.

Then run
```
sudo update-grub
```
From [https://help.ubuntu.com/community/Grub2](https://help.ubuntu.com/community/Grub2)
> **GRUB_SAVEDEFAULT=** If set to true this setting will automatically set the last selected OS from the menu as the default OS on the next boot.

    No commands need be run to set the default OS.
    Any time a menu entry is manually selected from the GRUB 2 menu, it becomes the default OS.

    This option currently does not work if your /boot directory resides on an LVM partition or RAID.

### Post by Mark Phelps on 2011-09-02
> **aura7 said:**
> You can manually edit the file /boot/grub/menu.lst and /boot/grub/grub.cfg files to suit your requirements of which os should be booted first.

Put the OS you want to boot first as your first preference in the list of OSes noted in those files.

BAD advice ... both of these!

First, there is NO menu.lst in GRUB2 installations, and creating one is NOT the way to go.

Second, there are lines in grub.cfg clearly telling folks NOT to edit the file.  That file is autogenerated every time the kernel is updated.  So, any edits they put there are going to be removed the next time they do an update that includes a new kernel version.

Please don't give bad advice -- especially in THIS forum.  Folks here are new to Ubuntu and tend to do ANYTHING they are told to do here.
