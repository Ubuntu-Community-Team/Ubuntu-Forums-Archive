### Post by cariboo on 2015-06-07
Break stuff, then fix it. :) As far as learning Kubuntu, don't get stuck on learning how one distributions works, for most of the core tasks, they work the same no matter what distribution you are using. When I was learning to use Debian, after starting with RedHat, I used to choose a task, such as setting up file sharing, I first went to [here]("http://www.tldp.org/"), and read and followed the relevant guides and howto's to complete the task I set out for myself.

### Post by LastDino on 2015-06-08
Trial and error, like already mentioned. But I would also suggest to get a good grip of shell, it helps a lot when you can actually tell what's the function of a particular part in any command that you are likely copy pasting from somewhere in your initial stages. 

It also helps if you try to do the various day today+high end tasks like setting up a server, VPN, maybe even writing small shell scripts and just browsing through threads here to pile up on your knowledge. I consider myself a linux beginner and I've no knowledge of programming or any real interest in it, but using Ubuntu for all the above is something I love.

### Post by HermanAB on 2015-06-08
How to learn more? Scratch an itch...

Getting too much email spam? Make your own mail server.  Need to save a lot of data? Make a file server.  Got several computers and no money for a router? Make one from an old PC. Need a WiFi router? Make an old netbook do it.  Want to water your plants every morning depending on the rain and weather? Install a sprinkler system and hook it to a PC.  Got a security problem? Set up a bunch of cameras with Motion, etc...
