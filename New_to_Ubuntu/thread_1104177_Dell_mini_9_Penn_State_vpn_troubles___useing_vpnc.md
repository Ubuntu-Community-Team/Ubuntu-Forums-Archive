### Post by mikedep333 on 2009-04-06
Hey,

I go to Penn State and I can help.

StrAlan: Penn State uses a Cisco VPN so VPNC has to be used, not PPTP (Microsoft's VPN.)

<<joe>>, normally it would be fairly easy to set this up using network-manager-vpnc, but this bug is present.

[https://bugs.launchpad.net/ubuntu/+source/network-manager-vpnc/+bug/207506](https://bugs.launchpad.net/ubuntu/+source/network-manager-vpnc/+bug/207506)

So we are going to use a command line utility to connect.

First, download the newest Linux VPN package from Penn State. We won't use the actual program though, just the settings profiles. Save it to your home folder (eg mike, not Desktop) or move it there. It should be a file named something like: vpnclient-linux-x86_64-4.8.01.0640-k9.tar.gz

Download Here:
[https://www.work.psu.edu/access/vpn/](https://www.work.psu.edu/access/vpn/)

Open up a terminal.
```

sudo apt-get install vpnc
[COLOR="Navy"](enter password)[/COLOR]
tar -vxf vpnclient-linux-x86_64-4.8.01.0640-k9.tar.gz
[COLOR="Navy"](or if that doesn't work because the file has a different name, enter:[/COLOR]
tar -vxf vpnclient [COLOR="Navy"](hit tab to autocomplete without a space after vpnclient))[/COLOR]
cd vpnclient
cd profiles
wget http://svn.unix-ag.uni-kl.de/vpnc/trunk/pcf2vpnc
mv pcf2vpnc pcf2vpnc.pl
chmod +x pcf2vpnc.pl
ls

```
Now, look at the list of files. You should see something like:```
[COLOR="SeaGreen"]abington.pcf      EMS-from-ISP.pcf       mont_alto.pcf
altoona.pcf       EMS-Wireless.pcf       new_kensington.pcf
beaver.pcf        fayette.pcf            pcf2vpnc
behrend.pcf       greater_allegheny.pcf  schuylkill.pcf
berks.pcf         great_valley.pcf       shenango_valley.pcf
brandywine.pcf    harrisburg.pcf         smeal.pcf
COE-from-ISP.pcf  hazleton.pcf           university_park.pcf
COE-Wireless.pcf  ISPtoPSU.pcf           wilkes_barre.pcf
dickinson.pcf     lehigh_valley.pcf      worthington_scranton.pcf
dubois.pcf        LIAS-VPN.pcf           york.pcf[/COLOR]

```
Find whichever one applies to you. Eg university_park.pcf is for the wireless at U Park. worthington_scranton.pcf is for the wireless at Worthington Scranton. ISPtoPSU.pcf is to connect to Penn State's network from home or anywhere other than a PSU campus. We are going to need to enter the name of the file as it appears, remember it is case sensitive.
```
./pcf2vpnc.pl [COLOR="Navy"](correct file name ending with a .pcf)
eg:[/COLOR] /pcf2vpnc.pl ISPtoPSU.pcf
```
Now you should see something like this:
```
[COLOR="SeaGreen"]Can't exec "cisco-decrypt": No such file or directory at ./pcf2vpnc.pl line 30.
cisco-decrypt not in search path,
  adding passwords in obfuscated form
## generated by pcf2vpnc
IPSec ID psuconnect
IPSec gateway vpn.aset.psu.edu
IPSec secret psuconnect
IKE Authmode psk
IKE DH Group dh2

## To add your username and password,
## use the following lines:
# Xauth username <your username>
# Xauth password <your password>[/COLOR]

```
Now we open up gedit (text editor as root)
```

[I]alt + F2
gksu gedit
(enter password)
[/I]

```paste the following portion that you saw on your console **(with its different values unless you did ISPtoPSU.pcf)**. Note that we are leaving out the stupid error message at the beginning.```

[COLOR="SeaGreen"]## generated by pcf2vpnc
IPSec ID psuconnect
IPSec gateway vpn.aset.psu.edu
IPSec secret psuconnect
IKE Authmode psk
IKE DH Group dh2

## To add your username and password,
## use the following lines:
# Xauth username <your username>
# Xauth password <your password>[/COLOR]

```
```

[COLOR="Blue"](save that as (under File System)[/COLOR] /etc/vpnc/[COLOR="Blue"](easy to remember name with a .conf on the end))
Eg. save as:[/COLOR] /etc/vpnc/isptopsu.conf

```
Close gedit as soon as you're done.

Now the final step, and the only step we need to use from now on. In a terminal, enter:
```
sudo vpnc-connect [COLOR="Blue"](name of file we saved without .conf)
eg, enter:[/COLOR] sudo vpnc-connect isptopsu
```
(enter password for your username and then username/password for penn state
You're done! You can now close that window
To disconnect, run:
```
sudo vpnc-disconnect
```

If you ever forget what the name of the file you saved was, you can run:
```
sudo ls /etc/vpnc
[COLOR="SeaGreen"]example.conf  isptopsu.conf  isptopsu.conf~  vpnc-script[/COLOR]

```
Ignore the example.conf, vpnc-script, and anything ending with ~.

Remember that you can find tech support on the PSU Linux User Group mailing list.
[http://lists.psu.edu/cgi-bin/wa?A0=L-PSULUG](http://lists.psu.edu/cgi-bin/wa?A0=L-PSULUG)
Here is our outdated site (that april 2nd post is from last year):
[http://php.scripts.psu.edu/clubs/up/lug/home](http://php.scripts.psu.edu/clubs/up/lug/home)
