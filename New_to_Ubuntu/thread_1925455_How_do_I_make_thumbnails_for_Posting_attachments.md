### Post by bogan on 2012-02-19
Hi!, **All**.

The problem was that the delay setting in Screenshot was not working properly when set to 3 secs, and it did not give time for the screen to re-draw to FullScreen completely, so the image was a weird half & half in-between mix.

Setting the delay to 5 secs stopped that and gave a correct image.

The error would not have mattered if there had been a hotkey to activate Screenshot, but if there is, I do not know what it might be.

In the mean time, **quequotio**n has given me a better solution - use PrtScrn!

I had not used it in Linux previously, because I thought it only copied the screen to the Clipboard, and I would then have to use a Paint program to edit it and save it as a .png file.

In fact, it automatically opens a Window in Image Viewer giving the file a name including the time of grabbing, and the option to save as .png.

So I am marking this Thread as Solved.

Chao!, **bogan**.
