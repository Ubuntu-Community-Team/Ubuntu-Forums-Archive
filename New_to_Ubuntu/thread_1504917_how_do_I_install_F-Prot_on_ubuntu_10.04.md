### Post by deibu76 on 2010-06-13
Clam AV and clamtk (the GUI for Clam) are very easy to install...  The virus definitions are updated pretty regularly, and I believe it is the only AV for Linux that offers a 64-bit version.  It does not run in "real-time" mode, in the background - you just run it whenever you want to scan "on-demand."  

I wouldn't bother with F-Prot - I wasn't even aware there was a linux version.  I used F-Prot w/ Windows 98 about 10 years ago, but as time went on, it seemed to fall behind the other free AVs out there.  If you can't find recent instructions for installing it, then chances are that the linux version is being maintained by anyone, and the virus definitions aren't being updated either

I have a Windows 7 partition I use for gaming mostly, and I currently use MS Security Essentials...  About once a month, I run Clam on the NTFS partitions from Ubuntu.  Not really necessary though, since I don't use Windows that often and I run a couple of other scans within Windows from time-to-time.  

I last ran a scan about 2 weeks ago. and I did have two warning-flagged files pop up the last time I used it - it was an EXE & a DLL w/ some tool I downloaded...   It turned out they were false positives.

No need to have anti-virus running all the time honestly...  I think you can even set up Clam to run a scan on any downloaded file or email attachment you open.  It's pretty basic but it is a decent virus scanner when you need to use it...
