In computing, a plug-in (also: plugin, addin, add-in, addon, add-on, snap-in or snapin; but see also extension) consists of a computer program that interacts with a host application (a web browser or an email client, for example) to provide a certain, usually very specific, function "on demand"

### Post by snova on 2009-03-18
> **Skol312000 said:**
> What are they?

Extensions. The idea is that the program itself can provide most functionality, but then people can write small, extra programs that "attach" to the main program to extend its capabilities.

> How could they hurt me?

In the same way any other program could. Don't download things from untrusted places.

> What are some awesome ones?

Plugins can not usually be used in more than one application.

For example, if you were using Firefox- there is a host of addons and plugins and whatnot for it [here]("https://addons.mozilla.org/en-US/firefox/"), but I don't think they will work anywhere else.

Common ones you might encounter could be a Firefox plugin to view Flash, or one to embed Java applets. Ones for gstreamer (or whatever it is) to play MP3 files and other various codecs.

Similarly, a programmer using Eclipse might download a plugin (well, with Eclipse, possibly several dozen) to add Python integration, or another to add an interface to Bzr, etc.
