### Post by Dougie187 on 2008-07-18
I have not gone through this exact experience, but i know some friends who have dell laptops (the same as the inspirion model that can come with ubuntu on it) and it works great for them on their laptop. The machine seems to be of good quality, and the distro is exactly the same as what you would download and install yourself.

As for the transition from Windows to Ubuntu I would say this depends on your uses of your laptop in windows. If you program a lot, personally I would say it is very easy to transition to Ubuntu, or any linux, because it is quite a lot easier to do in linux. Also if you mainly browse the web and listen to music (other day to day tasks like this) your tansition would be rather easy as well (my mother is actually going through the transition now and not having a difficult time with it). However, i think the transition might be a bit more difficult if you play a lot of insensive games in windows, because they might not all have support in linux yet (using wine or what-not) and this could make it a bit more difficult to get the things running how you want.

Either way, I think it's a good idea to do. There are also a lot of other companies that have prebuild linux systems (System76, Lenovo has OpenSUSE) that you can look at. You should be able to change most of the distros from one to another. Also, I personally would make sure that you aren't paying more for the linux computer then you would for a windows computer. I know back when dell first came out with the ubuntu computers they cost more then the windows one and I thought that was ridiculous. 

Either way, good luck and the communinity is here to help if you have any problems.

### Post by purplepaint on 2008-07-18
I got a Dell 1525 with Ubuntu 7.10 pre-installed and had no issues with it.  It's certainly a great way to go if you want to guarantee that your hardware will work.  I had issues with sound and my wireless light went off (although I didn't notice that one until I saw someone else on these forums posted with that problem a few days ago) when I upgraded to 8.04, but they were easy to fix following copy-and-paste command-line instructions from Dell's Ubuntu wiki.

From my experience above, I can agree that the command-line is a much easier way to fix issues since there's less chance of anything (bad) happening if you do the wrong thing (unlike checking the wrong box in a GUI and then not understanding why something new is happening), and learning it (even if only the basic commands like I have) gives you a much better understanding of what your compute is actually doing.

The advice that I would give to any new Ubuntu users: It's easy!  Just remember that, in a terminal, it's Ctrl - ***Shift*** - v!
