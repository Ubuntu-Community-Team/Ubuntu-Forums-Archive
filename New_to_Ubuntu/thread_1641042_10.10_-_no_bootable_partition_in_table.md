### Post by oldfred on 2010-12-08
Almost sounds like a BIOS error. Windows has to have a boot flag on a primary partition. Grub/Ubuntu do not use a boot flag. But a few BIOS (Intel for one) require a boot flag on a primary partition (which is a table).

I would try adding a boot flag to any primary partition, grub will not care. You can use gparted, Disk Utility or command line to add boot flags.

If that does not work:
Boot Info Script courtesy of forum member meierfra
Page with instructions and download:
[http://bootinfoscript.sourceforge.net/](http://bootinfoscript.sourceforge.net/)
Paste results.txt, then highlight entire file and click on # in edit panel(code tags) to make it easier to read.
Or You can generate the tags first by pressing the # icon in the post's menu and then paste the contents between the generated [ code] paste here [ /code] tags.

### Post by wog on 2010-12-08
SOLVED

Thank your for making the connection to the BIOS error - I had never seen one in Ubuntu before (that I know of).

I shut down and took out the LiveCD prior to reboot, then checked the BIOS on the way in. Somehow the BIOS decided the lone SATA drive (the Ubuntu drive) wasn't to be booted from and subtracted it from the drive lineup. It was trying to boot from the two usb drives.

I reinserted the SATA drive, took the 1.5TB usb drive out from the lineup and rebooted. Ubuntu started up again as if there was never a problem in the first place.

Thank you for the education, I'll keep an eye out should this error reoccur.
