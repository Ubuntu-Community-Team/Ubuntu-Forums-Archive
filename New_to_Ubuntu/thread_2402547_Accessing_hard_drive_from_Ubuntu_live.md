### Post by oldfred on 2018-10-01
Abnormal shutdown often causes file corruption whether Windows or Linux. With Windows then you need chkdsk and Linux if ext4, fsck to repair file system.
Also Windows 8 or 10 uses fast start up or hibernation.

So you have to first clear chkdsk flag and then hibernation flag for Ubuntu live installer to see any NTFS partition correctly. Best to use Windows repair disk to fix Windows issues.

You may be able to manually mount in read only mode.
       Force mount, read only (ro), change example with sda3 to your correct NTFS partition:
sudo mkdir /media/windows
sudo mount -t ntfs-3g -o ro /dev/sda3 /media/windows

### Post by guiverc on 2018-10-02
I would not start with trying to read the data off the drive given you are suspicious of a bad drive. I'd try and read 'drive-health' from the SMART data (*which comes from drive electronics; if the drive is failing with limited life yet, it doesn't waste any*).  Booting to terminal allows use of `smartctl`, or if you get GUI started `gnome-disks` provides less information-overload and is easier to understand. 

The point of this is to get a feel of your chance of getting data off (or you're just wasting your time, and errors on attempts are to be expected as its health is extremely poor).  my 2c
