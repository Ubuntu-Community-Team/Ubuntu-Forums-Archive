### Post by Branimir on 2009-01-30
in directory /etc/X11 there is file called xorg.conf

you can simply edit it with text editor and put resolution you want

for example in mine there is:

Section "Screen"
    Identifier     "Screen0"
    Device         "Device0"
    Monitor        "Monitor0"
    DefaultDepth    24
    Option         "TwinView" "0"
    Option         "metamodes" "1280x1024 +0+0; nvidia-auto-select +0+0"
    SubSection     "Display"
        Depth       24
    EndSubSection
EndSection

this was generated by nvidia script.

In early days you had to do that manually.

### Post by Gulyan on 2009-01-30
I would suggest this method:

After pressing Ctrl+Alt+F1 type:
```
sudo vi /etc/X11/xorg.conf
```

Find the "Screen" section in this file and it will show all the resolutions there. Just edit the resolution to the one you had and it should work.

The "screen" section looks like this:

```
Section "Screen"
        Identifier      "Default Screen"
        Monitor         "Configured Monitor"
        Device          "Configured Video Device"
        DefaultDepth    24
        SubSection "Display"
                Depth        24
                Modes        "1680x1050" "800x600"
        EndSubSection
EndSection

```

### Post by wylfing on 2009-01-30
Far from being idiotic, it sounds like you've taken some pretty sensible actions. A few years ago reconfiguring the xserver would have probably solved your issue.

I can't say for certain, but I think what happens now is that nearly all monitors report their capabilities, so xorg autodetects resolutions rather than storing them in a configuration file. These are then passed along to the desktop manager (i.e., Gnome) where they are presented as local preferences. Why it offered you a choice that was incompatible with your monitor I cannot guess.

But as for trying to fix your problem - now that you know how to get a virtual terminal (there are a bunch of them, Ctrl+Alt+F2, F3, F4, etc.), try digging into your Gnome preferences:
```
cd ~/.gnome2
nano monitors.xml

```
This will be XML markup, but hopefully you can understand what it says. If you know the correct data for your monitor, go ahead and enter it, save the file, and restart X (go back to your "main" virtual terminal with Ctrl+Alt+F7 and then hit Ctrl+Alt+Backspace to restart the display server). If you *don't* know the correct values, you might simply try renaming monitors.xml to something else ```
mv monitors.xml monitors.old
``` and Gnome will probably generate a new one for you with sensible defaults.

### Post by Branimir on 2009-01-30
Hey, I found where is monitor.xml.

It is in .config subdirectory of your home directory.

Here is mine:
```

<monitors version="1">
  <configuration>
      <clone>no</clone>
      <output name="default">
          <vendor>???</vendor>
          <product>0x0000</product>
          <serial>0x00000000</serial>
          <width>1280</width>
          <height>1024</height>
          <rate>52</rate>
          <x>0</x>
          <y>0</y>
          <rotation>normal</rotation>
          <reflect_x>no</reflect_x>
          <reflect_y>no</reflect_y>
      </output>
  </configuration>
</monitors>

```

AS you can see gnome is quite unsuccessful in detecting monitors,
but nvidia has no such problems as nvidia driver detects monitor
correctly.
Problem is that I can set resolution in both nvidia control panel
and gnome ;)

Just  paste this in command line :

```

cd ~ && find . -name 'monitors.xml' -exec nano {} \;

```

Greetsa, Branimir.
