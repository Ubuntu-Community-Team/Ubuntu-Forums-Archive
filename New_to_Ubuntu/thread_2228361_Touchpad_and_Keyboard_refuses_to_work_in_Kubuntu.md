### Post by squakie on 2014-06-08
search the net using the following search string:

linux toshiba l850d mouse keyboard not working

Many hits are returned - some for Windows (yep - it's not just a Linux "problem" - it's the hardware).

You may need to do a LOT of reading.  Remember - if the symptoms are the same it doesn't have to be a hardware match.

Some samples:

[http://askubuntu.com/questions/353981/sometimes-when-i-boot-up-my-toshiba-satellite-l875d-lap-top-running-ubuntu-12-04](http://askubuntu.com/questions/353981/sometimes-when-i-boot-up-my-toshiba-satellite-l875d-lap-top-running-ubuntu-12-04)


This one looks promising:

[http://askubuntu.com/questions/393051/toshiba-laptop-keyboard-and-mouse-intermittently-stop-working](http://askubuntu.com/questions/393051/toshiba-laptop-keyboard-and-mouse-intermittently-stop-working)

It also appears that there is a keyboard function key or key combination (or Windows setting) that can actually turn-off the keyboard hardware - meaning the keyboard and touchpad.  You may want to look in the manual for your laptop (you might need to find it online at Toshiba) and see what it says about disabling the keyboard.  It's possible this is a Windows function and you'll actually need to boot Windows and reset it there (I assume via using an external mouse).
