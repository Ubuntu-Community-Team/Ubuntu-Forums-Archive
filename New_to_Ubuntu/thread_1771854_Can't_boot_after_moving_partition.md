### Post by oldfred on 2011-05-30
You should have all your grub issues resolved, but I do not know of a way to automatically update the UUID in fstab. You will just have to manually edit it. the note of on sda5 can be changed but anything with a # is just a comment.

From blkid:
/dev/sda2 [COLOR=Red]       416d597e-8113-429b-bb63-7153c64c12d9[/COLOR]   swap
From fstab (they need to be the same)
# swap was on /dev/sda5 during installation
UUID=[COLOR=Red]993e102e-ef02-41f8-8982-60d2d85ce280[/COLOR] none            swap    sw              0       0

to see UUIDs:
sudo blkid

# modify fstab to mount directory
gksudo gedit /etc/fstab
# remount from updated fstab
sudo mount -a

Always run the mount -a command after editing fstab. If there are no errors then it is ok. If any errors you need to fix before reboot or you may not be able to reboot.

I just posted the mismatch between grub entries. Your reinstall and an update-grub should have resolved all of those.

### Post by oldfred on 2011-05-31
The UUIDs are correct and the partitions in the lines with # are just comments. So you can update it you want but it does not change booting. Only lines without the comment # are essential and it probably is taking longer to boot as it is searching for the UUID that does not exist for swap and cannot find it. Since it will work without swap you still can boot.

So just fix the swap UUID in fstab. You can update descriptions on comment lines to avoid confusion on which partition is which.
