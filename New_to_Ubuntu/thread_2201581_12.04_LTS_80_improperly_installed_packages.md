### Post by deadflowr on 2014-01-25
If Ubuntu is something new for you, it is best to use the given gui/graphical applications to install and remove packages.
In this case, use the software center.

That said, try
```
sudo apt-get -f install
```
if see if does anything.
Maybe try
```
sudo dpkg --configure -a
```

### Post by coldcritter64 on 2014-01-25
As well as the above posts, note that you can't have 2 package management programs like Synaptic, Ubuntu software centre, apt-get in terminal, or the update manager program, running at the same time. The dpkg process puts a lock on to prevent serious problems. Note in your output,
> debconf: DbDriver "config": /var/cache/**debconf/config.dat is locked by another process**: Resource temporarily unavailable Usually means you had a second package management app running.

> I'm hesitant to try Ubi_one_2014's advice because Ubuntu is warning me that it could cause a lot of problems. Should I give it a try? 
No - you shouldn't.

Reboot - at the grub menu - pick advanced - then from the new menu pick the second item - that will lead you to a new menu.

You should see a dpkg option - try that, it will ask if you want to mount - answer yes.

Once that's done it's job, enter (I think to return to the menu) then choose root from the list.

Then try 

```
apt-get -f install
dpkg --configure -a
```

Make sure to get spaces in the right place when you copy those commands down.
