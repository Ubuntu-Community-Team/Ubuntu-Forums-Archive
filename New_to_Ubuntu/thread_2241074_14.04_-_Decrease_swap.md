### Post by deadflowr on 2014-08-24
> [COLOR=#000000]Please tell me which one should be override[/COLOR]
None, leave the file as is and add the two lines
> [COLOR=#0000FF]*# Decrease swap usage to a more reasonable level*[/COLOR]
[COLOR=#0000FF]*vm.swappiness=10*[/COLOR]
to the bottom of the file and save it.

added: I should add, if you look at the file, every line has been commented(#) out.
So once you add the line *vm.swappiness=10*, that one line will be all that will be read by the system.

### Post by Rob Sayer on 2014-08-24
I usually get cryptic looking messages in the terminal when I use gksudo like that.  It's disconcerting at first but I never had any problems doing what I wanted to do.

Note: you should not assume that decreasing swappiness (which is different from swap actually) is going to speed up your machine.  Here's a guide that has some explanation why (and it's a very good guide on how to change it):

[http://askubuntu.com/questions/103915/how-do-i-configure-swappiness](http://askubuntu.com/questions/103915/how-do-i-configure-swappiness)

For example on my 1Gb xubuntu netbook, decreasing swap to 10 makes it slower than leaving it at 60.  Sometimes this works and sometimes not.  I'd try the method in the link where you just decrease swap in the terminal temporarily.  Then browse etc for a while and see if it's faster or not.  If you think so, make it permanent.

The funny thing is how blog writers often think they know more than the developers at ubuntu.
