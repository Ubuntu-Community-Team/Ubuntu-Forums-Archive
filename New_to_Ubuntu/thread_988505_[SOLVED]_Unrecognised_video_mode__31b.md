### Post by plucky on 2008-11-20
Do you have "vga=xxx" in the kernel line of menu.lst?

If not add the value vga=791 to that line and see if it makes a difference.

So your line would look like this ```
kernel		/boot/vmlinuz-2.6.24-21-generic root=UUID=125bc66f-c84c-41b8-9d91-3ca866cb4800 ro quiet splash vga=791
```
This will tell the system to use a resolution of 1024 X 768 @ 16bits

Also update the initramfs using the command ```
sudo update-initramfs -u -k all
```

If it does,then you can edit the menu.lst and add it to the default options in menu.lst so it would look like ```
## additional options to use with the default boot option, but not with the
## alternatives
## e.g. defoptions=vga=791 resume=/dev/hda5
# defoptions=quiet splash vga=791
```so you will keep those options whenever the kernel is updated.

Good Luck
