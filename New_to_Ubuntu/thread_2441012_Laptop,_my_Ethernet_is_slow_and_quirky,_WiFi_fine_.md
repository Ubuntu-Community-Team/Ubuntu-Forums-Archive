### Post by TheFu on 2020-04-17
Well, that's a respected Intel model and the driver is THE industry standard.  I remember some VoIP issues with specific versions manufactured in Mexico about a decade ago. Don't remember the exact Intel NIC model.  If you use VoIP, might check into that.

The fact that both the wifi and the wired both have dropped packets:
```
        RX errors 0  dropped 11  overruns 0  frame 0
        RX errors 0  dropped 10  overruns 0  frame 0
```
makes me believe that the issue is elsewhere, probably in the switch or router.  Try a different port and different CAT5e cable first.  Watch the RX errors and dropped packets over time.  Use the cable connection as much as possible.  Dropped packets can be normal for video streams so don't worry too much about that.

When you say "slow", what, exactly, does that mean?
Is is slow on the LAN when transferring files between local devices or only on the WAN?  If the devices are both wired and on the LAN, is it fast?

### Post by TheFu on 2020-04-18
Transferring 40G is HUGE!  For something like that, I'd use rsync since it is likely to fail on most home networks and rsync will restart where it left off.

But honestly, > seem ok
isn't facts.  If you want to solve the problem, we need facts. Real data.  Mbps and that doesn't mean you counting out loud as a guess or catching glimpses of numbers constantly changing in some GUI.  Facts.

We need to know exactly how you are transferring the data. What file systems are involved, how those are mounted, and perhaps the disk connections and types being used.  Transferring to flash storage will always be limited by the write speeds to flash.  Using WiFi will never work, since conditions inside your home are constantly changing. There's interference from humidity, other devices, neighbors, and just crappy wifi chips.

As far as DNS being slow is concerned, I can't help much other than to say you'll need to look deeper into the DNS on your network. 
```
When you ping 1.1.1.1, is it fast? Under 30ms?
When you ping cloudflare.com, is it fast? Under 30ms?
```
If cloudflare is slow, but 1.1.1.1 is fast, then it is a DNS problem, probably.  Perhaps you are using an ISP DNS, when either cloudflare or google DNS would be faster?  Where you are in the world will determine which DNS should be used, probably.  Plus, some ISPs capture all DNS traffic and redirect that traffic to their own, internal, DNS servers. If that is happening, there is usually a way to opt-out of their take-over in a web interface with your ISP account.  When my ISP did it about 10 yrs ago, it broke lots of stuff for me and I called immediately.  The CSR changed it right then.  I have a business account. Residential ISPs may not allow the change or might force you to use their web interface.

I don't have 19.10, so I'm afraid to say the commands to change the DNS settings, but you can look in /etc/systemd/ for something related to systemd-resolved ... or perhaps in the Network Manager settings?  I haven't used a GUI to manage network configuration in a very long time and purge network-manager from all my systems just after install. That doesn't help you.  In theory, this: 
[https://help.ubuntu.com/stable/ubuntu-help/net-manual.html.en](https://help.ubuntu.com/stable/ubuntu-help/net-manual.html.en) should be accurate.
