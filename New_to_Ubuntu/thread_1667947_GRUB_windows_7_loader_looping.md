### Post by coffeecat on 2011-01-15
> **dyserenity said:**
> I installed grub to the MBR.

Maybe you did originally, but you could only have run the mbr repair from the Windows 7 disc because:

```
 => Windows is installed in the MBR of /dev/sda
```Which will mean that you won't be able to get to the grub menu now. But this is your immediate problem: 

```
sda1: _________________________________________________________________________

    File system:       ntfs
    Boot sector type:  Grub 2
    Boot sector info:  [COLOR=Red]Grub 2 is installed in the boot sector of sda1[/COLOR] and
                       looks at sector 529130992 of the same hard drive for
                       core.img, but core.img can not be found at this
                       location. No errors found in the Boot Parameter Block.
    Operating System:
    Boot files/dirs:   /bootmgr /Boot/BCD
```Grub 2 has no business being in sda1 which is your Windows 7 boot partition. I think you need to run 'bootrec /fixboot' from a command console on the Windows 7 disc to repair the boot sector of sda1. More here:

[http://support.microsoft.com/kb/927392](http://support.microsoft.com/kb/927392)

If that fixes the Windows booting problem then well and good, but now you will have to restore grub to the mbr to get the grub menu for Ubuntu. See here:

[https://help.ubuntu.com/community/Grub2#Reinstalling%20GRUB%202](https://help.ubuntu.com/community/Grub2#Reinstalling%20GRUB%202)
