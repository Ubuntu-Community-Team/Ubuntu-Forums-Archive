### Post by brawd on 2010-06-13
Hi all,

For the last year or so while using Ubuntu I've also used an HP Photosmart 2610 all-in-one printer. It prints, faxes, copies and scans. Or at least it used to until the last couple of days.

I noticed that Gimp had a plug in that allowed input via a scanner. So I installed it. Now I haven't had call to use this plug in until today when I wished to scan a pic. I couldn't find the plug in trigger for the scan. In Applications I had Simple Scan installed so I tried that and got the message 'No scanners'. I went to synaptic manager and installed sane - no go there, nor did I have any luck when I installed Xscan. Always the same message - No scanners installed.

Because the Photosmart is on the Lan I tried a ping, 192.168.1.81 and that was fine. I tried a test page print and that was fine too. But I couldn't scan. Then I tried a printer report and that was what told me I was using the wrong drivers. I have been using the bog standard driver supplied by Ubuntu up till now and it's always worked fine.

The OS is 10.04 and kernel (whatever that is) is 2.6.32-22-generic.

If anyone knows about this I would be grateful for a suggestion.

regards,

brawd.

### Post by brawd on 2010-06-14
Ok I got around this by just going to HP and downloading the drivers from there. 

If any other newbies are following this I can tell you they have loads of Linux drivers there and also Ubuntu drivers as well.

Just remember to right click for properties and tell it to be execute-able.
Everything now works, including my fax.  Yahoo!
