### Post by Dougie187 on 2008-06-09
Just to clarify. *Most* by source installtions will include those steps:
1- Configure
2- Build
3- Install

This is not really always needed though. Some times you might just build and install your project, especially if nothing special needed to be configured.

However, if something does need to be configured, the typical name for the configure script is configure.sh not just configure.

To invoke this you would type
```
./configure.sh
```

If you want to check if you need to configure something, try typing in
```
ls
```
in the command line when you are in the directory. Also, try posting the content list of the folder on here.

There is more then likely a file called either INSTALL or README that has instructions on what to do. But you will most likely need to simply type
```
make
```
followed by
```
sudo make install
```

Hope that helps.

### Post by the_doc on 2008-06-12
What you need to do is run this type of command to create a sym link.

ln -s /path/to/arm-unknown-linux-gnu-gcc arm-linux-gcc

This will creat a link called **arm-linux-gcc** in your current directory that links to **/path/to/arm-unknown-linux-gnu-gcc**.  If you make the link in your home directory you can run the binary from there.
