Git is, for lack of a better explanation: [http://en.wikipedia.org/wiki/Git_(software](http://en.wikipedia.org/wiki/Git_(software))

Let me get things straight here.

There are two main forms of programs on the internet
Source, and Binary

Source files are the actual code that powers whatever program that it is.
Binary files are the compiled versions of source files. These are the executables you use to run the software. Meaning that you have to compile source files (therefore turning them into binary files) to run the program.

Certain programs use repositories such as Git and SVN in order to let mass amounts of users who specialize (or want to) test out developmental versions of software (and or patches, edits, fixes) to see if they work well. Those versions are not release versions, and may in turn have other bugs and/or problems with them. However, you can test out the changes immediately after the developer releases a new commit (version) instead of waiting for the release.
