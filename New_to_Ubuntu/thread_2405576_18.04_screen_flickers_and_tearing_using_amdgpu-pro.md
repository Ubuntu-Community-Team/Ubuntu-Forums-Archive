Downgrade to Ubuntu Studio 16.04 couldn't get the screen to  stop flickering when amdgpu-pro 18.40 was installed.  

So I started from  complete scratch.  

Fresh install of Ubuntu Studio 18.04 now I  did not update the system, modified grub or created the /etc/X11/xorg.conf.d/20-amdgpu.conf xorg.con.d does not exist.  
I installed  amdgpu-pro 18.20 I used the following sudo ./amdgpu-pro-install -y --opencl=legacy --headless installed no issues no warning messages. 
Now I edited /etc/default/grub I added amdgpu.dc=0 to the following line GRUB_CMDLINE_LINUX_DEFAULT="quiet splash amdgpu.dc=0". 
Save it and ran sudo update-grub and restarted the computer.  With the above amdgpu.dc=0 if you set it to 1 you will get a little flickering.  
Now the screen flickering is gone but I still had a thick tearing bar on the lower portion of the screen.  When click, move the mouse or a window this thick tear would appear for a fraction of a second.  
At this point I created mkdir -p /etc/X11/xorg.conf.d/ and sudo nano /etc/X11/xorg.conf.d/20-amdgpu.conf.
Used the man amdgpu 

Section "Device"
        Identifier "AMD"
        Driver "amdgpu"
        Option "Accel" "on"
        Option "TearFree" "auto"
        Option "AccelMethod" "none"  If you set this to glamor you get some tearing very small. 
        Option "DRI" "3"
EndSection
Saved it restarted the computer and tearing is gone.  Great.

Now I want to setup my blender and the dam program will not open.  Uninstalled and installed a fresh copy no effect blender refuses to open.  It flashes drawing the window than disappears.  Not sure why.
