### Post by deadflowr on 2014-05-17
Don't know how much more I can help.
The graphics card listed says intel. 
Those drivers are built into the system.
Sometimes cards like nvidia or amd can install different drivers, which in most cases clear up the problems.
Intel doesn't have that.
I would definitely +1 trying to get a bootable live session going, simply as it is the quickest way to tell if it's a bad driver, or if something really did go bad with the hardware/graphics card.
As for a few other things
> **rob64 said:**
> hitting 'e' in grub  shows setparams, curious about the 1st part

recordfail
load_video
gfxmode $linux_gfx_mode
insmod gzio
insmod part_msdos
set root= 'hd0, msdos1'

then a whole lot more, but i was wondering about the 1st 2 lines?
This is normal, it's what is listed during grub(Ubuntu bootloader).
> **rob64 said:**
> i keep getting the following when trying various fixes

W: not using locking for read only lock file /var/lib/dpkg/lock
E: unable to write to /var/cache/apt/
E: the package lists or status file could not be parsed or opened
I believe this means the system is running in read-only mode
If you try to run it from recovery mode, then you should click on networking first and then run the root shell.
The reason for this is the recovery mode starts up with networking disabled, and the filesystem loaded read-only.
If you run the networking  option it reloads the filesystem read/write, as well as enable the network connection.
> **rob64 said:**
> I'm now getting a msg 

Gtk -warning**: cannot open display:  :0

Have I found the problem if so how do I fix it
This usually means the Xserver isn't running, or isn't running correctly.

Something you could try, is when you boot up let it run for a minute or two and then try the ctrl+alt +F1.
Login(put your username and password), and then try 
```

```
it might start a gui session, or it may output a bunch of errors.
I'm not sure what might happen in that regard, though the command won't hurt the system.
If it almost starts something but hangs for a long time, simply re-press the crtl+alt+f1 keys again, to bring you back to the screen.
