Does the resolv.conf file have this header?```

# Dynamic resolv.conf(5) file for glibc resolver(3) generated by resolvconf(8)
[COLOR="Red"]**#     DO NOT EDIT THIS FILE BY HAND -- YOUR CHANGES WILL BE OVERWRITTEN**[/COLOR]

```

If so then the answer is no.  If you do it will be rewritten the next boot up.

As this is a desktop version of Ubuntu I have to believe that the config is handled by Network-Manager.  Id so click on the network icon at the top right of the desktop (sys tray).  Then check the appropriate interface settings.  I have a wired connection so I look at eth0.

When you look at the IPv4 settings you will see the "search domains" parameter.  This is where the setting should be made.

### Post by jdthood on 2013-01-19
> **bab1 said:**
> When you look at the IPv4 settings you will see the "search domains" parameter.  This is where the setting should be made.

You only need to enter something into that field if (1) you want search domains and (2) the search domains are not already set via DHCP.

If your interface is configured automatically (via DHCP) and the DHCP server is configured correctly then you don't have to enter anything into that field.
