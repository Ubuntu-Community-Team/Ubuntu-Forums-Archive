### Post by oldfred on 2011-03-11
Because your system saw the drive as sdb or hd1, the menu entry says.
set root='([COLOR=Red]hd1[/COLOR],msdos1)'
    search --no-floppy --fs-uuid --set 21575745-6f3d-45ec-b95e-1d5e8c34f37d

Your UUID does match and when the set root is wrong, the search is suppose to override it and work.

Hold shift key down to get the grub menu. Edit the set root line to be hd0 instead of hd1(red part above). You may want to edit out the search line also to see it it helps. These edits are one time only. If it works then you may want to reinstall grub from inside your install and update grub.

#reinstall from working (not liveCD) system - first find Ubuntu drive:
sudo fdisk -l
#if it's "/dev/sda"  then just run:
sudo grub-install /dev/sda
#If that returns any errors run:
sudo grub-install --recheck /dev/sda
sudo update-grub
