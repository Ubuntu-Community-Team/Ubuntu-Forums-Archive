### Post by quelx on 2008-05-28
if you create a folder in **/opt/lampp/htdocs** then the URL would be **[http://localhost/folder](http://localhost/folder)**.  Otherwise your script would be **[http://localhost/myscript.php](http://localhost/myscript.php)**.  

For viewing source code there are a number of viewers (less, cat, more, echo). Eclipse, Anjunta, emacs, gedit, nano, vi are popular *editors* I can think of off the top of my head.

**sudo cp sourcefile destination** so ```
sudo cp myscript.php /opt/lampp/htdocs/
```

### Post by dwhitney67 on 2008-05-28
I had a similar concern a few months ago.  In my case, I wanted to practice writing PHP applications without having to save them in root-owned areas.

To the best of my recollection (and I am maybe wrong about this), I edited the file /etc/apache2/sites-available/default to specify the directory for the DocumentRoot.  By default, the DocumentRoot is set to /var/www.

Try to edit this same file and point the DocumentRoot to your /opt/lampp/htdocs.

### Post by quelx on 2008-05-28
I threw it on my box and all I had to do was change the owner /opt/lampp/htdocs/

```
sudo chown -R username /opt/lampp/htdocs
```

Then I could throw files in there and so long as they were world readable (mod 644) they showed up.  For instance I created **index.php** with :```
<?php

phpinfo ();

?>
``` and copied it into the /opt/lampp/htdocs folder. browsed to **[http://localhost/index.php](http://localhost/index.php)** and the specs of the lampp php installation showed up.  so check your permissions and the path you are typing in the browser. ignore the **/var/log** stuff 
I posted...  lampp keeps its own logs in **/opt/lampp/logs/**
