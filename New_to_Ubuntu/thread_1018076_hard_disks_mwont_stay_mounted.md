### Post by konungursvia on 2008-12-21
Generally, I have the same sort of issues with FAT32 and NTFS drives connected via USB. In theory, HAL is a hardware abstraction layer that should identify and mount any drives. Try connecting them one by one, after booting in to x. Otherwise, what I do is dual boot with windows, which unmounts them cleanly. Often they don't mount cleanly because the last unmount was not perfect. Try right-clicking and unmounting them before closing x. Finally, you can do a search for "force mount" and actually include the drives in fstab. This is nuclear, in my view, as fstab is essentially for internal drives.

### Post by pdtpatrick on 2008-12-21
well on linux .. the paths are normally /xxx/xxxx/ .. windows uses \ .. so try not to mistake those. However linux will use \ sometimes when you have a file system that has spaces between or you named a directory that has spaces like "My Documents" instead of "Documents" ... so linux will use \ to translate the spaces so that linux understands it two words. 

Also google and see how to mount fat32 systems ... it could be a different file system mount which in that case you will need to remount and then edit your /etc/fstab to reflect the proper changes.

### Post by mapes12 on 2008-12-24
There are entries in your fstab that are causing the problem. Probably due to running ntfs-3g at some point.

> /dev/sda1 /media/Hard1 ntfs-3g defaults,locale=en_US.UTF-8 0 0
/dev/hdc1 /media/data-disk ntfs-3g defaults,locale=en_US.UTF-8 0 0
automount(pid7202) on /mnt/device type autofs (rw,fd=4,pgrp=7202,timeout=30,minproto=2,maxproto=4,indirect)
nfsd on /proc/fs/nfsd type nfsd (rw)but there could be others as well.

Boot your system with the Live CD. Make sure the drives are formated correctly by using Gparted ```
gksudo gparted
```Once you are happy the drives are formatted to Ext3 and if the drives mount correctly in this environment then in Terminal ```
sudo cat /etc/fstab
``` and copy and paste the output to somewhere else e.g. a floppy or something. ```
gksudo gedit
```will bring up a text editor for you to paste the output into a text file for the next step. Then, reboot from the hard drive. Bring up the problematic fstab file like this ```
gksudo gedit /etc/fstab
```then delete ALL the text and replace it with the version from the file you created when running the Live CD.

Reboot and that should be it.
