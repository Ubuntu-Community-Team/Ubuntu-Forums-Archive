### Post by snowpine on 2009-08-24
Welcome to the forums! I will answer part 1 of your question, hopefully some of our other members will answer the rest.

Ubuntu, Xubuntu, and Kubuntu are basically the same core system, with different "desktop environments" (user interfaces) and pre-installed applications. I prefer Ubuntu (Gnome desktop), some people think Kubuntu (KDE desktop) is most like Windows, and Xubuntu (Xfce) is popular for old hardware. This is a very frequently-asked question in our "recurring discussions" sub-forum: [http://ubuntuforums.org/forumdisplay.php?f=302](http://ubuntuforums.org/forumdisplay.php?f=302)

And to answer question 4, it is really easy to set up a "dual boot" where you choose between Windows and Ubuntu each time you power up. I recommend choosing one of your computers (the most broken one?) and making it your "Ubuntu computer" while you learn more about Linux. Once you've used it for a while, you can make the decision whether or not to install it on the others, no hurry, right?

### Post by ainsworth_t on 2009-08-24
1)
Technical Differences
Ubuntu was the original *buntu and is based on the GNOME desktop environment. Kubuntu is a spin-off of Ubuntu based on the K Desktop Environment (KDE). The primary difference between GNOME and KDE are that they are developed using different toolkits (GNOME: GTK - GIMP toolkit, KDE: Qt toolkit). Xubuntu is another spin-off of Ubuntu based on the XFCE desktop environment, which is also developed using GTK.

Functional Differences
Ubuntu takes a minimalist approach to it's desktop. It has pretty basic interfaces that are easy to use and figure out. KDE out of the box has a "pretty" interface, like Vista Aero or Mac Aqua, and has an interface closer to Windows than GNOME has. XFCE is developed with the purpose of being lightweight to run better on older hardware (since your machine is a P3, this might be the best option).

2)
Networking with a Windows XP computer shouldn't be a problem. You can either use SAMBA to share files (there is a nice, easy interface to share with and connect to shares on Windows machines), or you can easily set up an FTP site on the *buntu box to share files with the Windows XP machine.

3)
Installing a dual-boot Windows XP/*buntu environment is as pretty easy and common process (Instructions here: [https://help.ubuntu.com/community/WindowsDualBoot](https://help.ubuntu.com/community/WindowsDualBoot)). Before you attempt this, make sure you backup your immportant files to some external media where it is safe from getting installed over. Also, could you please provide a little bit more information about your current HDD/partition/installation scheme.

4)
Once again, installing a dual-boot XP/*buntu environment is pretty common and not hard to perform. I'm not sure if it's supported, but you could try running Reaper in 1 of 2 ways: 1) install and run through Wine in Ubuntu 2) Create a Windows XP virtual machine using VirtualBox or VMware to install and run Reaper within the Ubuntu installation

5)
I'm not quite sure what you're asking with this one. For instructions on downloading and burning the *buntu ISO on CD, read through this article [https://help.ubuntu.com/community/BurningIsoHowto](https://help.ubuntu.com/community/BurningIsoHowto). For instructions on downloading and installing from a USB sitck, read through this article [https://help.ubuntu.com/community/Installation/FromUSBStick](https://help.ubuntu.com/community/Installation/FromUSBStick)

Sorry for any redundant information.

### Post by donkyhotay on 2009-08-24
To alot of people linux is about freedom, as in civil liberties and choice. This means that unlike other OS's where you have to do it the way some software manufacturer dictates, you can do whatever you want. Because people are so different you end up having about a million different ways of doing the same thing because different people have different ideas on how to do stuff. This is why you have hundreds of different distros (ubuntu, fedora, debian, arch, etc.) and even within each distro you have different variants (kubuntu, xubuntu, edubuntu, etc.). It can be very confusing when you're coming from a system that gives you 1.5 choices to a system that gives you as many choices as you want. My recommendation to new people that get confused is to simply use the defaults, there's a reason the ubuntu devs make the defaults the way they are. Most of the time if you're uncertain defaults will work just fine. If it doesn't work or if you don't like the way it works then go ahead and try something else. When I first started to (seriously) work with linux I started with fedora, but I didn't like the fedora package management system. So I migrated to debian which worked better for me but I found the ideology made certain tasks too difficult. I then tried ubuntu, absolutely loved it, and have been using it exclusively for 3-4 years now. Remember, it's *your* computer, we can give you all the advice and suggestions you want but ultimately it's up to you to decide what you're going to put on your system and how you're going to run it.

### Post by makoshark55 on 2009-08-24
It has been a month or two since I gave up on "windows" whatever. I eventually had to set up a dual boot with ubuntu 9.04 and windows XP. One thing I did not run across until I no longer needed was a program called "super grub". You can find it here [http://www.supergrubdisk.org/](http://www.supergrubdisk.org/). "Grub" is the defacto boot loader for Linux and handles other Operating systems like windows to. The benefit of super grub is when you set up a dual boot and don't get it the way you want it you can use the super grub [COLOR=#ff0000]boot[/COLOR] CD to fix it. It comes with help already on the disk that you make so you can know what you are doing. Another benefit is you can use it to restore your original windows boot sector should you change you mind. Another disk you may find handy is to make a gparted boot CD to set up your partitions. You can find it here [http://gparted.sourceforge.net/livecd.php](http://gparted.sourceforge.net/livecd.php). Ubuntu comes with gparted so you really don't need it, but I found it very handy when I managed to muck things up. What I had done was try to set (move) up my ubuntu to a 500GB SATA hard drive that had windows XP on it. Do some research on how to set up partitions and you will be glad you did. What I ended up with was a (in physical order) was a 10GB boot partition ext3 (also has GRUB) > 180GB XP NTFS > 174 GB /home ext3 > 116GB /root [COLOR=#ff0000]AKA "/"[/COLOR] ext3 > 10GB tmp ext3 > 8.5 GB Linux-swap, just to give you an idea what you can do. I went over board on the size needed for Linux, as it can live with much less room than the windows NTFS file system. The only reason I still have windows XP is I have some programs like Peachtree accounting that will not run under anything but windows. There is a program called WINE (a windows emulator, Google WINEHQ) that is supposed to let you run windows programs in Linux, but I did not have very good luck with it. Something you should look for is the ubuntu pocket guide, a good read before you get started. If you are like me you will find ubuntu a great departure from windows. The first thing that blew me away was how much extremely faster Linux of and distribution was than windows of any kind. I have yet to have ubuntu crash and you can fly without a virus program. Consider getting a free virus scanner from bit-defender to protect your windows partition. They will give you a free serial number for Linux. You can find it here: [http://download.bitdefender.com/repos/](http://download.bitdefender.com/repos/), get the one for DEB. Ubuntu comes with Firefox web browser and Evolution Email. ubuntu was able to configure my built in network on the motherboard without any intervention on my part, something windows never could do right. Another delight is when the desktop shows up it is ready to go and is not delayed like windows. The only draw back I have found in ubuntu was having to learn how to use the terminal to install or configure programs, there is a sharp learning curve with this, but the pocket guide should help you with this. There is a over abundance of help in the ubuntu forums to help you learn what you need to know. One caution I will give you is Linux can read NTFS but don't try to write to it as it will screw your NTFS. Linux handles FAT 16 and FAT32 just fine, so consider formating Window to say FAT32 if you want to save to your windows partition. Here are some great links: [http://ubuntuforums.org/](http://ubuntuforums.org/),  [http://www.linuxquestions.org/](http://www.linuxquestions.org/), [https://wiki.ubuntu.com]("https://wiki.ubuntu.com/"), [http://www.linuxhomenetworking.com/](http://www.linuxhomenetworking.com/).

### Post by zerhacke on 2009-08-25
> **makoshark55 said:**
> Do some research on how to set up partitions and you will be glad you did. What I ended up with was a (in physical order) was a 10GB boot partition ext3 (also has GRUB) > 180GB XP NTFS > 174 GB /home ext3 > 116GB /root [COLOR=#ff0000]AKA "/"[/COLOR] ext3 > 10GB tmp ext3 > 8.5 GB Linux-swap, just to give you an idea what you can do.

Something tells me you don't know much about Linux partitioning either.  First off, /root and / are NOT the same thing.

10GB for /boot is overkill even for kernel developers.  The only thing that goes in /boot is boot information such as the kernels which are roughly 40MB in size.  Even if you have 100 kernels you're still only going to use about 40GB of space, and nobody has 100 kernels.

There's no need for a /tmp partition at all if you have a 116GB / partition.

Eight and a half gigs for swap?  Why would anyone ever need that much swap on a home system unless they have 10GB RAM?
