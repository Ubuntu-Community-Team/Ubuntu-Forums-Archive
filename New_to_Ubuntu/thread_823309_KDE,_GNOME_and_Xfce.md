### Post by .nedberg on 2008-06-09
Short answer: KDE is more Windows like, users comming from Windows will feel more welcome. Gnome is more Mac like.

KDE offers a lot more configurations, while Gnome is "simpler" and not so scary.

I started with Mandriva and KDE and moved on to Kubuntu after a year. I now have both Kubuntu and Ubuntu machines, but I prefer the K desktop.

Give both a shot with the Live CDs. If you install Ubuntu you can easily transform that into a Kubuntu install and vice versa.

KDE programs run under Ubuntu and Gnom programs run under Kubuntu, so there is no big difference in using either.

### Post by Dani Filth on 2008-06-09
From my experience, other than appearance, there's still difference between them. Like, the way they access Windows share:
- Ubuntu/GNONE : Nautilus & Samba which I think there is still bug in accessing Windows shares till the bug is completely fixed.
- Kubuntu/KDE : Konqueror & Smb4k which seem to work ok with Windows shares at the moment.
- Xubuntu/XFCE : Thunar, and so far, you can't access Windows shares by default as Samba/Smb4k does. You need to manually deal with some stuffs in order to get this done.

Anyway, it's just my 2 cents :D
