### Post by ChrT on 2009-09-21
The correct term is "desktop environment", not "desktop manager". The difference is, a window manager does just what it's name advertises, it draws and manages windows, leaving everything else to other, external applications. A desktop manager will usually come with a taskbar/panel, applets, desktop compositing manager, three thousand card games you'll never play and a full suite of branded apps (email clients, office suites, media players) it's designed to work with, which, while not restricting your choice, narrows it down severely. 

A good GNU/Linux install that follows the UNIX philosophy should ideally not use any DE or any apps which have DE dependencies.

### Post by fluxbuntu on 2009-09-21
Desktops use a window manager. Gnome uses Metacity. But desktops build in all those nifty little user friendly things to click that slow it down. More processes running in the background means slower functioning. Lose the bells and whistles and things speed up. The downside, if you consider it one, is that using window managers like fluxbox require you to do more for yourself like telling it what to run at start in a text file. Desktops just make the assumption for you. In teh end, it's hard to tell Gnome or KDE what to do. Fluxbox or Openbox want you to. However, window managers don't require constant tweaking. Once it's how you like it your done, and it's just blazingly fast.
