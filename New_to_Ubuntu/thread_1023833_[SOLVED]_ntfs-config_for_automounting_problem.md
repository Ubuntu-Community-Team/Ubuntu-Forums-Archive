### Post by drs305 on 2008-12-28
ntfs-config merely adds a line to fstab to enable automatic mounting. You can reverse it by editing fstab and removing the applicable entry.

However, in this case, you might just add "users" to the options on the device line in fstab. That should allow any user to mount and unmount the device. The other option is to include "uid=1000" or your user id if it is different in the options, which will allow you specifically to mount and unmount the device.

If you need help with fstab, please post the contents:
```
cat /etc/fstab
```

Once you have made and saved the changes in fstab, unmount and remount the device to allow the changes to take effect (with XX being the device designation, for example, sd*b1*):
```

sudo umount /dev/sdXX
mount /dev/sdXX

```

Backup fstab and open for editing:
```
sudo cp /etc/fstab /etc/fstab.1228
gksudo gedit /etc/fstab
```

Delete the line in red to reverse what the ntfs-config program did.

If you prefer, you can alter the line so that the device is mounted with you as the owner, (and with "users", anyone else as well) which will allow you to mount/unmount it as you wish:
```
[COLOR="Red"]/dev/sda2 /media/bw ntfs-3g defaults,uid=1000,users,locale=en_US.UTF-8 0 0[/COLOR]
```

To learn more about fstab, visit these links:
[https://help.ubuntu.com/community/Fstab]("https://help.ubuntu.com/community/Fstab")
[http://www.tuxfiles.org/linuxhelp/fstab.html]("http://www.tuxfiles.org/linuxhelp/fstab.html")
