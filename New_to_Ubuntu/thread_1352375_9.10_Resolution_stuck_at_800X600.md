### Post by benjaminsuman on 2009-12-11
It sounds like your graphics card isn't detecting what your monitor is capible of displaying. You can create the file /etc/X11/xorg.conf to tell  it explicitly what resolutions to support.

Example:
```
Section "Monitor"
  Identifier "Monitor1"
  Vendorname "Generic LCD"
  Horizsync 31.5-48.0
  Vertrefresh 56.0-65.0
EndSection

Section "Screen"
  Identifier "Screen1"
  Monitor "Monitor1"
  Subsection "Display"
    Modes "1024x768" "800x600"
  EndSubsection
EndSection
```

### Post by Sensiva on 2009-12-11
> **mablake184 said:**

It took me a while to get mine to work too, so I am going to tell you what I have known and done so far. And it worked for me perfectly (Q35 Intel chipset, pretty old HP 20 inch monitor)

**Basically you should know that:**
- Enabling desktop effect needs that the current working driver to be** intel **not **vesa** (vesa doesn't support effects). This will be solved by reconfiguring X
- Higher resolutions should be automatically detected after reconfiguring, sometimes they must be added manually. (described below).

_**1st:**_ **Check if there is a xorg.conf file in your /etc/X11 folder or not.**
How?
```
ls /etc/X11/xorg.*
```If its there, then move it to a safe well known place (to restore it back in case all this didn't work)
How?
```
sudo mv /etc/X11/xorg.conf /path/to/safe/place
```**_2nd:_** **Now we need to make Xorg reconfigure your display settings**.
How?
```
sudo stop gdm
```This will lead you to the terminal prompt of tty1, login with your username and password then do this command to reconfigure.
```
sudo X -configure
```It will tell you that a new xorg.conf has been generated in /home/[COLOR=Red]**YourUserName**[/COLOR]/xorg.conf.new
Replace **[COLOR=Red]YourUserName[/COLOR]** with your actual username (eg. /home/**[COLOR=Red]sensiva[/COLOR]**)

**_3rd:_** **Move the new xorg.conf to /etc/X11**
How?
```
sudo mv /home/[COLOR=Red]**YourUserName**[/COLOR]/xorg.conf.new /etc/X11/xorg.conf
```Replace **[COLOR=Red]YourUserName[/COLOR]** with your actual username (eg. /home/**[COLOR=Red]sensiva[/COLOR]**)

_**4th:**_ **Restart gdm**
How?
```
sudo start gdm
```X now should be running, if it gave you an error message about low graphics mode and crap like that, simply press CTRL+ALT+F1 to go to terminal tty1 and restore the old xorg.conf from the safe place we mentioned above, or delete the current one in /etc/X11
If it started, then we need to check for two things:
  - **Can you enable desktop effects or not?** If it didn't get enabled, then the installed intel driver isn't good enough, apply all updates and reboot.
How?
```
sudo apt-get update && sudo apt-get upgrade
``` - **Are all your monitor supported resolutions available in System -> Preferences -> Display?**
If not, then your VGA chipset isn't able to detect it automatically , you have to add its Horizsync and Vertrefresh setting manually in xorg.conf
You can get these two settings from your monitor manual or data sheet, if you didn't find them, please provide your monitor brand and model number may be I can get them for you. It should be in this form
```
Horizsync    30.0-140.0
Vertrefresh  50.0-160.0
```And should be added in the [COLOR=Red]**Monitor**[/COLOR] section in xorg.conf
```
[COLOR=Red]**Section "Monitor"**[/COLOR]
    Identifier   "Monitor0"
    VendorName   "Monitor Vendor"
    ModelName    "Monitor Model"
    Horizsync    30.0-140.0
    Vertrefresh  50.0-160.0
**[COLOR=Red]EndSection[/COLOR]**
```You may edit xorg.conf using gedit (if X started) or nano (in terminal tty1)
```
gksudo gedit /etc/X11/xorg.conf
```or 
```
sudo nano /etc/X11/xorg.conf
```**Remember:** at anytime if the display didn't work don't panic just press CTRL+ALT+F1 and undo changes in terminal. Becarefull of the typos.
Good luck :D

### Post by mikewhatever on 2009-12-12
> **mablake184 said:**
> Thank you for bearing with me this far... so if I want those display modes I just have to add them?  I couldnt find the horizsync and vertsync for my tv, do i still need them?

You are welcome, but you didn't answer the question. What is the resolution you'd like to use? For the lack of input, let's say you want 1280x720. It's suitable for widescreen TVs with the aspect ratio of 16:9, but adjust as needed. You can generate the modes according to the required resolution with cvt command. For example, <cvt 1280 720 60> (see output below):

```
Modeline "1280x720_60.00"   74.50  1280 1344 1472 1664  720 723 728 748 -hsync +vsync

xrandr --newmode 1280x720 "1280x720_60.00"   74.50  1280 1344 1472 1664  720 723 728 748 -hsync +vsync

xrandr --addmode VGA1 1280x720
```

Now, you've added the mode, lets test if it works.

```
xrandr --output VGA1 --mode 1280x720
```

If it does, and you are happy with the resolution, you need to save the changes by adding it to your xorg.conf.

gksudo gedit xorg.conf  <-- run this to open xorg for editing

```

Section "Monitor"
	Identifier   "External-VGA1"
	Modeline     "1280x720_60.00"   74.50  1280 1344 1472 1664  720 723 728 748 -hsync +vsync
	Option       "PreferredMode" "1280x720_60.00"
EndSection

Section "Device"
	Identifier  "Intel Corporation 82945G/GZ Integrated Graphics Controller (rev 02)"
	Driver      "intel"
	VendorName  "Intel Corporation"
	Option  "External-VGA1"
EndSection

Section "Screen"
	Identifier "Screen0"
	SubSection "Display"
		Depth     24
		Modes   "1280x720"
	EndSubSection
EndSection

```

                                            >      originally posted by** trx64: **I've solved the problem by disabling KMS (Kernel Mode Setting) in Intel driver, leaving the detection of my screen resolution just with X. There is no need to change xorg.conf, just leave it at default values.

     title           Ubuntu 9.10, kernel 2.6.31-14-generic

quiet

Just tagging what Benjamin said. It worked for me. But there are a couple things to note:
1) Make sure your Horizsync and Vertrefresh fall within the ranges above. ```
cvt 1024 720 60
```(thanks mikewhatever). 

If it is, use Benjamins output from above in step 3 below)

2) Create an xorg.conf file. From what I understand, 9.10 manages resolution in the kernel so that's why it's not creating an xorg file anymore. Follow Sensiva's post (#17) starting at step 2.

3) Once you've copied the xorg.conf file to X11 (per above), you need to edit the file.```
sudo gedit /etc/X11/xorg.conf
``` Look for the section called "Monitor". You should see something like: ```
Identifier "Monitor 0"
VendorName "Generic LCD"
ModelName "Monitor Model"
```
Just before the endsection add the Horizsync and Vertefresh```
Horizsync 31.5-48.0
  Vertrefresh 56.0-65.0
```
4) In the section "Screen" under subsection "Display" add the modes Benjamin used above. That is  ```
Modes "1024x768" "800x600"
```
5) I don't know what "Depth" does but I've seen it in most xorg files so just below Modes and another line ```
Depth 16
```
6) Save your xorg.conf file and reboot (or log out and back in)

Of course 1024x768 is pretty low nowadays so if you know what resolution you're after 1280x1024? More? Just use that in the cvt command above and in the xorg file. 

I hope that helps someone! :P
