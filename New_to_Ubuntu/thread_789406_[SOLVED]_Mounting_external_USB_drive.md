### Post by Rocket2DMn on 2008-05-10
You should take out the entries in fstab for your external disk (sdb I assume) so that HAL can automount the drive for you.  I know people often have problems automounting and using FAT32 drives, the best solution seems to be to just take out the fstab entries for it (you can place a # at the front of the line to "comment out" the entry thus disabling it).
Also, if for whatever reason you do keep fstab entry, you can't have a space in the name, it will think the next word is the next column for the entry, not a continuity of the mount point.  So "/media/Linux HDD" is not a valid entry for fstab.

### Post by Rocket2DMn on 2008-05-10
Try unmounting it, disconnecting it, then just plugging it back in.  With any luck the problem has gone away.
To mount it manually giving yourself permissions to read/write the drive, you need to add options to the mount command, see the (extensive) man page for more details - [http://linux.die.net/man/8/mount](http://linux.die.net/man/8/mount)
You may need something like
```
sudo mount -t vfat /dev/sdb1 /media/sdb1 -o umask=000
```
That umask gives permissions 777 (read-write-execute for all users).  Alternatively you can use 077 which is rwx for the owner only which would probably mean you have to set the owner, too
```
sudo mount -t vfat /dev/sdb1 /media/sdb1 -o umask=000,uid=1000
```
I don't have experience with fat32 in linux though, so I can't double check this stuff for you.

[http://www.psychocats.net/ubuntu/permissions](http://www.psychocats.net/ubuntu/permissions)
You won't want to have to be root to use the drive, that is why we assign your username permissions.

### Post by Rocket2DMn on 2008-05-10
root is the superuser of the system, and has full administrator privileges.  Since sudo temporarily gives you superuser privileges, root owns the mounted files (since FAT doesn't have file permissions like ext3 does).  You should be able to read/write the disk anyway.  If you use the second option I gave
```
sudo mount -t vfat /dev/sdb1 /media/sdb1 -o umask=000,uid=1000
``` you should be listed as the owner.
