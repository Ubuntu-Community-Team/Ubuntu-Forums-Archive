### Post by secretspicy15 on 2009-04-20
Ah, and I just found this too (you may have seen it already, but...)

> Retrieving and setting the Firewire GUID (FirewireGuid)

When you issue iphone-mount or ipod-touch-mount for the first time, the program silently attempts to generate this value using a USB connection. If you have never connected your device via USB before and try to mount it via WiFi for the first time, this step is missed and you end up with a database that your device refuses to read. This is (in nearly all cases) because the Firewire GUID (FirewireGuid) that the iPhone/iPod Touch specifically use to hash their database is either missing or invalid.

To fix this manually:

    * Plug your iPhone/iPod device into your Ubuntu machine via USB
    *

      In Ubuntu, run Applications &#8594; Accessories &#8594; Terminal
    * Issue iphone-mount or ipod-touch-mount (depending on your device) in the terminal
    * Path to /media/ipod/iPod_Control/Device ("cd /media/ipod/iPod_Control/Device", assuming you used /media/ipod for the mount location)
    *

      Run "sudo lsusb -v -d 05ac: | grep iSerial | awk '{print $3}' 

| cut -b1-16 | xargs printf "FirewireGuid: 0x%s\n" > SysInfo" (Found here).
    * In your media application of choice, transfer a new song and/or sync your iPod. Depending on what media application you use, you might have to re-transfer all songs.
    * Reboot your device 

You may also get the GUID directly from the device by executing:

 /usr/sbin/ioreg  -n IOIpodUSBDevice  -w 0 | grep DeviceConfiguration | cut -d '"' -f 44 | cut -c 1-16

If all goes well, your music should now be recognized on the device!

Alternative solution with MNPlight

    *

      Add the [http://iphone.lefessant.net/iphone](http://iphone.lefessant.net/iphone) link to your Ipod Touch or Iphone installer.
    * Install MNPlight.
    *

      For Ipod Touch you need to set the "FirewareGuid" of device (see previous section).
    *

      Set the Fireware GUID in MNPlight&#8594;Preferences. 

Warning: Without this step MNPlight will corrupt your music database!

    *

      Copy your music in to /var/root/Media/MNPlight/Music folder of your Iphone/Ipod Touch and import it by MNPlight&#8594;Playlists&#8594;Import songs from directory
from [https://help.ubuntu.com/community/PortableDevices/iPhone#Syncing%20Music%20on%20Firmware%20Versions%201.x](https://help.ubuntu.com/community/PortableDevices/iPhone#Syncing%20Music%20on%20Firmware%20Versions%201.x)

Honestly, I think my way will probably do the trick, and seems easier than the fix listed on the wiki... If it does work, we'll have to add it as an alternative on the wiki :-)

Although I guess you may have already seen that, since it sounds like you've already taken a look in the folder... Like I said my best guess is that it's corrupt for some reason...

### Post by secretspicy15 on 2009-05-17
Ok, well, I hate to tell you this, but you're going to have to reload all of your music.

You have your music in the correct folder for pwn player. The standard player doesn't put music in nice folders like that. It uses a database of some kind, so you need to use amarok or gtkpod to sync your music over...

So here's what you need to do:
1. Edit the xml (which it seems you already have done)
2. Possibly delete all of your music from your ipod (unless you have oodles of hard drive space on there, which I doubt)
3. Use amarok or gtkpod to transfer your music. I see you're running jaunty; amarok2 is the default amarok and it does not transfer music to ipods correctly, so gtkpod is your best bet, unless you install an old version (~1.4 or so) of amarok...

I know you have no desire to sync your music using these guys, but you need to use one of them so that it gets stored in the database correctly; the standard music player does not read from the file system like pwnplayer does.
