### Post by SteveNorman on 2009-06-18
Im off to bed soon, so in case you hose your xorg.conf file here is how you fix it:

If you backed up your files like I said, you just copy the backup xorg.conf back into place as the main xorg.conf file:
```
cp /etc/X11/xorg.conf_backup /etc/X11/xorg.conf
```

Then restart x(your graphics manager) by hitting alt-printscreen-k

alt-printscreen-k is your new cntrl-alt-delete for most freezes and graphical restarts.

If you didnt backup your files (which is the biggest mistake you can make with configuration files) then you will be stuck in command line land. You cant use gedit without x so you will need to know vi, emacs, nano or some other non-gui editor.

here is a vi tutorial:

[http://www.cs.colostate.edu/helpdocs/vi.html](http://www.cs.colostate.edu/helpdocs/vi.html)

learning vi and some command line basics will help you survive the linux learning curve.

good luck and welcome to linux!

### Post by SteveNorman on 2009-06-19
If you have nvidia-settings installed I would try and use that to change your resolution.

if its not installed do:
```
sudo apt-get install nvidia-settings
```
then update your data base
```
sudo updatedb
```
and run it by typing this at the prompt
```
nvidia-settings
```

under X SERVER DISPLAY CONFIG you will see options for your resolution. try that.

if that doesnt work, back up your xorg.conf then add this to your monitor section just above the last option entry
```
ModeLine "1280x800_60.00" 106.5 1280 1440 1520 1672 1904 800 900 901 904 932 -hsync +vsync
Option          "PreferredMode" "1280x800_60.00"
```

this is from
[https://wiki.ubuntu.com/X/Config/Resolution](https://wiki.ubuntu.com/X/Config/Resolution)
