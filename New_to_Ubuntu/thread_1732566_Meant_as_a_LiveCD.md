I am not sure i quite understand your problem. But the OS doesn't GIVE RAM. RAM is hardware component of computer. it only uses is as temporary memory storage.
 
LiveCD loads all system into RAM, so it's a bit slower to load (especiall from CD) but then it sort of works like a system. BUt since it is all in RAM as soon as oyu turn of the computer everything you did is lost. it's not saved anywhere.
 
Ubuntu can run just fine on 256MB RAM (a bit slow and with some tweaks and few things turned off). 512 MB RAM works nice, 1GB ram is plenty for Ubuntu.
 
Xubuntu is a bit lighter as it uses XFCE desktop environment which needs less RAM  and CPU power to run. It works OK on 256MB RAM computers, bit better on 512MB.
 
Lubuntu (LXDE) needs even less RAM and disk space to run. Can run OK even with much less than 256MB RAM.
 
Kubuntu (KDE) is the heaviest and needs more RAM and also a good enough graphics card. but looks very nice and polished.  
 
 
 
More RAM is always better.

### Post by David_UK on 2011-04-18
tarshoduze

I can't answer your question in #5.  But if you are planning to regularly boot from a live CD, you should investigate a 'live USB with persistance'.  
Basically you create one from an installed version and boot from it just like a CD.  The advantage is it will remember extra installed programs, your documents, preferences etc.  Very useful.  Speed-wise, I think it's about the same as booting from a CD.
