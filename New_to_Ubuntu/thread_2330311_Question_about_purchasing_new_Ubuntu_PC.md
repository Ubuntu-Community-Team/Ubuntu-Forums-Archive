### Post by TheFu on 2016-07-10
Welcome to the forums.

Dell sells some systems pre-loaded with Ubuntu too, but I think those are the 15 inch laptops. 
Sometimes Ubuntu does **just install** on new systems without any issues, but sometimes it is a dirty knife fight to get Ubuntu to work on some systems. Since the chips used in any system change all the time based on manufacturing needs, it is very hard to say whether Linux will "just install" or not on any specific machine.  I've seen all sorts of different machines fail to load Linux at different *installfests*. Getting a system with the newest CPUs, newest southbridge, newest GPU, can make loading Linux next to impossible, at least for 6-12 months.  Adventurous people with strong Linux skills might be willing to go this route.

Business PCs will have more predictable internals (and cost more) because businesses demand that model XYZ43232 be the same whether it is bought today, next month, or in 8 months. Everything needs to be the same.  IME, this adds about 20% overhead to the pricing, but you can find off-lease or used machines with some wear online for less.

I think HP Pavillion is a line of systems, not exact enough to ask whether it runs Ubuntu or not. The exact model number will be needed. That is a very expensive machine - way outside my price range. Usually I spend about $300 every 7+ yrs to replace the MB + CPU of an older machine, which will bump performance 300-500%. OTOH, I can see were a business would prefer this kind of setup. Not sure how upgrading the HW would work over the years, if that was even possible, on a system like this.

The best advice I have is to find someone else running the version of Ubuntu, on the hardware you want to run it, and ask them about any issues.

Sorry. There isn't any silver bullet.

### Post by TheFu on 2016-07-16
If you are doing virtualization - the hardware doesn't matter unless Ubuntu is the hostOS.  That fancy GPU - doesn't matter at all.  Sadly, you might want to skip the stock Ubuntu and run a lighter desktop like LXDE or XFCE or Mate since virtual machine GPU performance isn't like direct HW GPU performance.  It has gotten better, but I don't know anyone who uses VMware stuff anymore to compare. We've all moved to F/LOSS solutions for that which were just as great last time I used them.  Personally, I run KVM for my VMs (including my primary desktop) and access it through x2go. For the work I do, it doesn't matter if I'm 2ft away from the VM server or 13K miles away on the other side of the planet. As long as I have ISDN or better networking where I'm remoting from, it is the same.

If you paid for Workstation out of your own wallet, can you return it?  You'll find that virtualbox is basically the same thing, for free.  And if you make Linux the hostOS ... all sorts of other virtualization becomes possible.  I know people who use KVM + libvirt + QXL displays for some amazing 3D VM performance on the same LAN (or even the same physical machine).

Oh ... and Ubuntu was plenty mature in 10.04 ... it is just the GUI they've screwed with since then. Wish they'd put it back myself. ;) Fortunately, we have many choices of GUIs, perhaps 50+, since the GUI is just another program in Linux, not the OS. Don't like the one they installed?  Add a few others, check them out with different userids, then remove those you don't like/need.  Profit! (important to have that last step!). ;)
