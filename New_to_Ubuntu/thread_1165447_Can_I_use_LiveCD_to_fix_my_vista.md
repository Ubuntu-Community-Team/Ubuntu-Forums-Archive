### Post by lisati on 2009-05-21
If you *need* Windows on your system (keeping warranties valid is one excuse), your toolkit should include a set of Windows recovery disks relevant to your particular machine, ideally made from a "clean" system. Having a set available will also be useful when you discover some problem that the other tools you have can't handle.

It's not enough to just update your AV definitions regularly: it's sometimes possible for malware to sneak in and do some surreptitious mischief before the AV recognizes it. A full scan might be in order - the Ubuntu LiveCD can sometimes be used for this.

EDIT: Be cautious about Windows installation disks of the "don't ask too many questions where they came from" variety: As well as the potential for legal hassles, I've encountered such disks loaded with malware.

### Post by Mark Phelps on 2009-05-28
Your constantly reinstalling Vista may actually be contributing to your system instability and problems.  Why?

Because, as has been seen by experience, the initial version of Vista was loaded with problems.  Some (a few) were fixed by Windows Updates and driver updates.  Others (more) were fixed in the Service Pack 1 (SP1).

Now, SP2 is out (so I hear, I haven't installed it myself).  And given that it brings almost no new features, it's basically another big bugfix.

In checking one Vista machine of mine (running SP1), I've done well over a hundred Windows Updates since it was installed.

Unless you have a version of Vista incorporating SP1, you're basically reseting it back to the beginning every time you reinstall.  So, all the hidden bugs that have been fixed by subsequent Updates are then back.

Better solution would be to do the following:
1) Reinstall from the DVD
2) Allow Windows Updates to apply updates for a week or two
3) If you're still having basic problems, there's most probably a hardware fault involved.

### Post by decoherence on 2009-05-28
Just glossed over this thread... didn't see anyone suggest running memtest86+. It's an option on the live CD that continuously an repeatedly tests your RAM. Let it run for a night and see if it reports any error in the morning. If so, get new RAM.

Generally bad RAM doesn't cause this kind of intermittent behaviour (usually it just breaks the system outright) but it's worth a check. RAM is cheap and the test is easy.
