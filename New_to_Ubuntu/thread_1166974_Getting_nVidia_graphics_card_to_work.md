### Post by Hospadar on 2009-05-22
Well problem number one is that you are not actually using the nvidia x driver
according to:```
Section "Device"
    Identifier    "Configured Video Device"
    Boardname    "vesa"
    Busid        "PCI:1:5:0"
    Driver        "vesa"  #THIS LINE determines what driver you use
    Screen    0
EndSection
```If you were using nvidia it would say "nvidia" instead of vesa.  You could try just changing that and see if it works, but I would be very surprised if it did (do give it a try though).  Probably a better choice would be to just make a new x config file. Do these next steps while the x server is not running (i.e. "sudo /etc/init.d/gdm stop")

First back up the old one```
sudo mv /etc/X11/xorg.conf /etc/X11/xorg.conf.backup
``` (you can always move it back if things go sour)

Now, I would use nvidia-xconfig to generate the new file.  You can look at "man nvidia-xconfig" if you want to see all the options, but from my recollection, if you just have one monitor, it should work fine by default.  This is a command line tool roughly equivalent to the gui version "nvidia-settings"  you'll need to run it as root ```
sudo nvidia-xconfig
```After that, start up X ```
sudo /etc/init.d/gdm start
``` and see how it goes.

I'm assuming you know about virtual terminals, if not ctrl-alt-F1 (or F2, F3, etc) to get to a virtual terminal, and ctrl-alt-F7 to get back to X/gnome/kde
