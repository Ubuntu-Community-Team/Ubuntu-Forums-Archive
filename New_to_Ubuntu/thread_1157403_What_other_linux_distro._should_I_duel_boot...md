### Post by petrus4 on 2009-05-12
> **qjmoss said:**
> I'm interested in learning other Linux distors. to widen my knowledge of, well... Linux.

I don't know how popular or unpopular this opinion is going to be, but if you really do want to learn, I'd recommend [Slackware]("http://www.slackware.com/"), personally.  It isn't quite as stereotypically user friendly as Ubuntu, but that's sort of the point; unlike Ubuntu, it's a distribution where Linux is allowed to be itself, rather than being forced into being a clone of Windows.  

Ubuntu might be easy, but being easy also means being limited; it's good for relatively canned, pre-designed scenarios, but doing anything unusual with it becomes more difficult.  

That is traditionally a weakness of "user friendly," operating systems; they're novice friendly, but generally extremely expert hostile.  Given that experts however are always the minority, the majority of said novices (and developers who are wanting to make money) generally don't care about that all that much.

The thing about already having Ubuntu as well is that if what you're doing gets too hard and you want a break from it, you can just reboot into it as well.

[This]("http://linux.slashdot.org/comments.pl?sid=162105&cid=13553367") is a guide I wrote a few years back to learning a lot about Linux, and Slackware will be perfect for following it with; better than Ubuntu for that, actually.  Ignore Step 1 in this case, because that mentions getting Knoppix, and you'll already have Slackware.

To explain my own view of Ubuntu, let me use an analogy.  If we were going to think of Linux as a foreign country, (from the perspective of Windows users) I would actually look at Ubuntu as being said country's domestic airport.  It's where most Windows immigrants initially get off the boat, so to speak.

However, once you've arrived, you're generally going to find that you'll want to explore other parts of the country as well.  You can't really live in an airport, (apart from anything else, the food generally sucks ;)) and as nice as it might be, it doesn't really allow you to get a good idea of what the rest of a given country is like, either.

So even though, to follow on from that analogy, Ubuntu is the point of arrival, you will very often tend to find that as your familiarity and confidence with Linux grows, you will likely want to move to another distribution, which is a bit more powerful, reliable, and also more like UNIX, (the system that Linux is based on) rather than Windows. (the system that Ubuntu is trying to imitate)

With apologies to the Ubuntu developers, but although I recognise their good intentions, (and to use the above analogy, every country *does* need an airport, after all) Ubuntu isn't really a good representation of what many older users genuinely consider Linux to be.  As I've said, it tries a lot harder to be something Windows users are comfortable with; but Linux is, in reality, a very different operating system.

> **qjmoss said:**

Your instructor is a wise man.

Truthfully, if you **really** want to learn UNIX, you're better off going with one of the BSDs.  (FreeBSD is their gateway drug ;)) 

However, Slackware is a lot easier for initially getting your feet wet, while still being sufficiently well designed that you'll get a taste of using a real OS rather than mass market crap. :P

> But with me going into slackware, I'm going to be in a while "different planet" as I understand.

The main difference is that, as I said, Slackware is actually well designed. :P
  
I won't lie to you; Canonical have absolutely mutilated Linux.  UNIX has an extensive design [philosophy]("http://www.catb.org/~esr/writings/taoup/html/index.html"), which they have not followed.  Robustness and any semblance of solid engineering principles have been completely thrown out the window, and the mantra is simply to mindlessly clone Windows as closely as possible.  

Ubuntu is absolutely awful, comparitively speaking.  It is strictly for Windows refugees in the nappy stage where Linux is concerned.  As soon as you become even incrementally capable or confident with Linux, it will be in anyone's best interests to jump ship for another distro as fast as possible.

> I don't want to be able to use a package manager, I want to be able to control my OS 100% but also, Im not experienced in this.. 

No, you **do** want package management, believe me; but you want **good** package management, and such is almost entirely non-existent for Linux, sadly.  RPM and dpkg are both tolerable, but the problem is that people have an overwhelming tendency to write garbage specfiles, (the recipes for individual packages) and it isn't the system's fault if people don't co-operate.  The specfiles for Mandriva's packages in particular, in the past at least, were nothing short of appalling.

Slackware doesn't have package management out of the box.  There are various different forms in use for it, but they're generally added after the initial installation.  Install Slack, and go through that guide I gave you as well, and once you've done that and you're feeling confident with Linux, if you want a really good system, you'll be ready to upgrade to FreeBSD.

> I want to learn what every command and every letter in every command does, and the different sequences that I use them will do.

This is admirable, but truthfully, UNIX is a system that has had almost an infinite amount of potential knowledge accumulate in association with it, and I doubt there's a single person on the planet who knows absolutely everything.

The trick is to figure out what you **do** need to know, specifically, and then learn that.  Once you've got Slackware installed, grab a copy of Vim (a text editor) as well, look at the docs on [vim.org]("http://www.vim.org/"), and then learn that while you're going through the scripting tutorials in the guide I linked.

> I need a guide that will teach me 100% how the Unix OS/Slackware works and for a complete beginner.

[Here]("http://linux.slashdot.org/comments.pl?sid=162105&cid=13553367") it is again, then. ;)

This might not teach you everything, but it will definitely give you a good start.

### Post by crazypeppo on 2009-05-14
First, I am on that same path of Comptia to CCNA. I understand you wanting to learn it all! Take Linux+. You get a little of all the flavors. From Apt to Yum.
As far as what other Linux OS, Stick with one until you become proficient enough to teach others about their problems, Thats what IT field is about. What extra apps are needed for the media players in Ubuntu? If you are using Ubuntu try Kubuntu or Xubuntu. Learn the different installation methods. Learn what programs are used for the same application in all "buntu" flavors. Example: What partition editor do you use in Kubuntu? Ubuntu uses Gparted. LEARN THE COMMAND LINE/TERMINAL! If you really want to learn anything useful learn the difference between sudo su and sudo and when to apply them. Learn how to transcode with ffmpeg and mencoder and see why both are there. Of course these are all examples, but you should get the point.
And if you don't like sticking with one (which I would recommend) then you should try to learn Red Hat Linux and Apache Servers. Many major corporations including govt. facilities are going to Red Hat in place of "the other" O/S. If you are going to CCNA - Net Admin, then focus on career choices not what O/S people are using in their home. Try out virtualbox and start creating some networks. Learn to tunnel through SSL and how to set that up. 
Bottom line: _Focus on scenarios_ for a "real" world work environment, don't waste time on things that won't be useful for your IT job.

I don't want to discourage you from using other linux distros. One great one to learn would be Knoppix. You can use it to "salvage" a crashed windows computer. If someone who like windows wants to convert maybe freespire or mandriva. You said anything but debian, why?
