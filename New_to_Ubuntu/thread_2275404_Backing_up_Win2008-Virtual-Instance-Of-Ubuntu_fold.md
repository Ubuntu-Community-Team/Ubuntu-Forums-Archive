### Post by quique1232 on 2015-04-25
Ok I have a Windows 2008 Machine with Ubuntu running in Hyper V Manager.

The Windows 2008 has an external USB drive installed just for backups of the Windows Machine.

Now I need to make backups of a folder in Ubuntu as well.  So of course I wish to use the same external USB drive.

I've created a share for that external USB drive's folder.  The folder is E:\UbuOpen and the share is called UbuOpen.  It has Administrator access.

In Ubuntu I have gone to the Backup app and flipped the switch on for Automatic Security Backups, selected in the next tab the Share on Windows from the dropdown, for Server = Servidor5 which is the name of the windows machine on the network, Folder = /UbuOpen, User = AdminUser, Workgroup = MYCOMPANY.

I dont recall it ever asking me for the password except when I searched for the folder on the windows network and I entered the user, workgroup, password and i checked Always Remember.  Im backing up the Home folder for the main user and selected weekly backup but I chose to make a backup copy now.

The folder E:\UbuOpen is now filled with these datafiles:

duplicity-full.20150425T151623Z.vol1.difftar.gpg all the way to vol171 
and then a file fullsignatures.sigtar.gpg
and then a file OpenERPBackupUbuntu.vhd which was something I tried setting up in ubuntu a while back

Is this what the backed up data supposed to look like?  Is there any way I can look into it and check?  Or is this wrong?
