### Post by kaivalagi on 2010-04-04
There are some, I'm using Arch rather than Ubuntu..but the same options should apply

I am currently using MikroC in a Windows VM because I haven't found anything better to be honest.

The Linux options are:

[LIST]
[*]SDCC - small device C compiler which use GCC to compile (half decent howto, non distro specific: [http://www.freenet.org.nz/sdcc/](http://www.freenet.org.nz/sdcc/))
[*]Code Blocks - you can use SDCC from here, so atleast there is some UI ([http://wiki.codeblocks.org/index.php?title=Using_the_Code::Blocks_IDE_with_SDCC_on_PIC_MCUs](http://wiki.codeblocks.org/index.php?title=Using_the_Code::Blocks_IDE_with_SDCC_on_PIC_MCUs))
[*]Eclipse plugin for SDCC - there are a couple out there, but they are not very good...
[/LIST]

I only know of SDCC for the compiling side...but there may be other alternatives...and better IDEs?

I hope that helps, and I'll be watching to see if there are other options available too :)
