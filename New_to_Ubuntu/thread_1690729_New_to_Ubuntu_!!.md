### Post by Cracklepop on 2011-02-18
File menu, not file header (header is something very different).
A lot of Linux programs have Preferences or Options in the Edit menu, but it depends on the program (it does in Windows too).

 You won't need any drivers for a hard disk.  I'd have thought an optical drive might just use generic drivers, but if you need to look for drivers for any device for any OS the first place to look is the website of the manufacturer.

I don't have an ipod, but I know people do use them with Ubuntu in some fashion (I don't know the details, sorry).

### Post by Bigtime_Scrub on 2011-02-18
In general, on a user friendly Linux distro (like Ubuntu), if an external device has a USB it should at least mount and appear on the desktop. How much functionality you get depends on what that device is. An external hard drive should work no problem. 

I have an iPod Classic and it works fine in Linux. If it is a nano, iTouch, iPhone, or iPad or something like that I don't know for sure but you can probably get some functionality out of it. 

Rhythmbox, Amarok, and gtkpod all can sync with an iPod classic. 

As far as I know an iPod will not play FLAC. Apple does not like open source you would have to take that up with Steve Jobs.

- If I'm in a particular program, say Rhythmnbox, and I need to adjust settings in that program, in MAC OSX and Windows, there's Preferences in the File header. What is the Ubuntu equivalent ?

It's exactly the same in Ubuntu. Note that you're using Ubuntu Netbook Edition 10.10, which uses the Unity desktop shell - so your program's menus are hidden under the bar at the top of the screen. Move your mouse to there and the program's menus will appear.

> - If you add an attachment, an optical drive, or external hard drive, and it does not operate, where can you find the drivers needed for Ubuntu/Linux?

Optical drives and external hard drives don't require drivers apart from what's built-into the operating system itself. They are standard drivers. For most devices, the drivers are built-into Linux. Typically you should ONLY buy devices that have drivers already built-in on Linux.
