Anyway: Backup the xorg.conf
```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.backup

```
sudo nano /etc/X11/xorg.conf
```

Looks like sombody (or some script) has already messed around with that file. Have you tried to get a TV-Out or second monitor running? ever used (g)randr?

My suggestion is to get the single monitor back to work and then fiddle around with a second monitor.

remove the line
> Virtual 1400 1050
from the section
> Section "Screen"
Identifier "Default Screen"
Device "Configured Video Device"
Monitor "Configured Monitor"
Defaultdepth 24
SubSection "Display"
Depth 24
Virtual 1400 1050
Modes "800x600@85" "800x600@60" "800x600@75" "832x624@75" "800x600@72" "1024x768@85" "800x600@56" "1024x768@75" "640x480@85" "1024x768@70" "640x480@75" "1024x768@60" "640x480@72" "1024x768@43" "640x480@60" "1152x864@75" "1280x960@60" "1280x1024@60" "1400x1050@60"

### Post by argail1980 on 2008-06-10
hi,

now from the start

> Ubuntu starts with a good resolution as the progess bar loads the system.
This is i think a framebuffer option and has nothing to do with your xserver resolution. Shortly before the logon screen is displayed, the X-server starts.

> Will cleaning up the xorg.conf file remove the nvidia driver?

No, it will not. there is a line in xorg.conf in the device section that says

> Driver "nvidia"

This defines which driver to use, standard is "nv". "nvidia" is the new (proprietary) driver.

> ect/xll/ Watch out, it is etc and X11('eleven')

You have to start all commands with sudo or do a ```
sudo bash 
``` before, if you don't want to type sudo all the time. As regular user, you don't have write permission to /etc (what is usually a very good idea).

> what controls the screen resolution used for the log-on window?

as far as I know the resolution of the logon screen is determined of what you define in the xorg.conf, here:

> Modes "800x600@85" "800x600@60" "800x600@75" "832x624@75" "800x600@72" "1024x768@85" "800x600@56" "1024x768@75" "640x480@85" "1024x768@70" "640x480@75" "1024x768@60" "640x480@72" "1024x768@43" "640x480@60" "1152x864@75" "1280x960@60" "1280x1024@60" "1400x1050@60"

btw.. have you tried to change the resolution by pressing Ctrl-Alt-+ or Ctrl-Alt-- when this logon screen is displayed?

there's a good test. go to the xorg.conf and change

> Option "NoLogo" "True" to
```
Option "NoLogo" "False"
```. When the xserver starts, this should display an nvidia logo. when it does, the driver works in principle.

My guess is the reason for your strange resolution is the combination of the > Virtual 1400 1050 line in your screen section and the "800x600@85" in the Modes. The first sets the size of your "desktop", the second sets your screen resolution.

really, editing this file is not so bad if you get used to it ;-)
