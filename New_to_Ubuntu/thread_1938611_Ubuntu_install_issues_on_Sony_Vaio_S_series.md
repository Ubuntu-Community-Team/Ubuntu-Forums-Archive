Apparently, Grub 2.0 (that ships with Ubuntu 11.4) does not load or has some sort of bug. As a result, the computer hangs with a black screen before showing the Grub menu.

### Post by westcoast01 on 2012-03-10
Yeah me too, only on the machine that I built specifically for Ubuntu :confused:. Did an error report and it was suggested that my problem has something to do with the way the memory is switched after boot. What I have to do is: Scroll down to Memory test 86, click that and let it run for 10 or 15 seconds, click escape, machine restarts and will then boot into Ubuntu. The error report said it was a known issue with some computers but did not say which ones. I am going to try another MB with a different BIOS.
Best of luck :P
