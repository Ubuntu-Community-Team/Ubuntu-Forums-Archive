The Nvidia-settings program can't modify files outside your home directory unless you run it as root:

```
gksudo nvidia-settings
```

The last time I checked there was a bug in the Nvidia-settings program anyway that causes it to write its modes in a format to the file that X doesn't understand, making its modification useless. You might need to add the mode to the Section "Screen" part of /etc/X11/xorg.conf:

```
Modes     "1024 x 768"
```

(not sure if this is exactly correct, I don't still have this in my xorg.conf).

Remember, /etc/X11/xorg.conf is outside your home directory, so you MUST modify it as root.

### Post by Rumtscho on 2009-09-14
To edit the file manually, you need to open a console window (under Applications -> Accesories -> Terminal) and change to the directory where the file lies. To do this, type ```
cd /etc/X11
```. Then type ```
sudo gedit xorg.conf
``` (but remember to make a backup copy first). You'll have to enter your pw and then the gedit editor will open with the file. You cannot just click yourself through the file tree and open the file, because then you won't have the rights to save the changes. 

Make the changes you need, save. Press Alt+PrintScreen+K; this will restart your X manager, meaning that the drivers for display, mouse etc. will be reloaded. The screen will go blank, and you'll have to log in again. If you made an error in your config file, you'll be informed of it and it will give you an option to start your system with an old config version. If that happens, choose that option, then delete the config file, make a copy of the backup and rename it to xorg.conf. Your system will function again. 

When I installed Ubuntu, my NVidia card also had resolution problems. My 17'' display is capable of running up to 1024x768, but I only got to choose resolutions up to 640x480. The trouble was that the card couldn't read correctly the information send by the display. I don't know if your problem has the same cause, but you can try if my solution works for you. 

I had to add information about sync rates as well as a display subsection to my xorg.conf. Now the Monitor and Screen sections look like this: 
```
Section "Monitor"
	Identifier	"Configured Monitor"
	HorizSync 30-81
	VertRefresh 55-75
EndSection

Section "Screen"
	Identifier	"Default Screen"
	Monitor		"Configured Monitor"
	Device		"Configured Video Device"
	DefaultDepth	24
	SubSection "Display"
		   Depth 24
		   Modes "1280x1024_60"
	EndSubSection
EndSection
```

Try out if this works for you. The HorizSync and VertRefresh numbers should be safe for most flat displays, but if you have the datasheet for your display handy, you should consult it first. And under "Modes", you should type the resolution you want your screen to work in, the "60" after the underscore is for the refresh rate. 

Good luck!
