### Post by stderr on 2009-03-27
As a previous poster said, vim is an excellent editor. If you're getting into shell scripting, you may prefer using a command line text editor such as vim, emacs or nano. However, if you can't be bothered learning the commands, modes and suchlike, the graphical editors already proposed should do the job just fine.

There tend to be graphical equivalents for the command line editors too, such as gvim (called vim-gnome) for vim and emacs22-gtk for emacs.

The main GUI apps are gedit (GNOME), kate (KDE) mousepad (XFCE).
