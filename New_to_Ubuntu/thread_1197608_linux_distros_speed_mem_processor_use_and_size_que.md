### Post by keplerspeed on 2009-06-26
Ubuntu is 'heavier' on resources than lighter variants simply due to more processes, coming 'mainly' from the GUI. Thats why you have a performance improvement going to xubuntu etc.

Moving to a lighter browser than firefox can drastically improve performance, expecially on old hw (ie opera). So yes... more or less the same kernel.. the linux kernel... but with differnet modules added.

As you will see DSL is simple. Simple and light usually equate to fast.

### Post by michaelzap on 2009-06-26
I use #! (CrunchBang) on my Eee PC 701 very happily. It's essentially Ubuntu running OpenBox instead of Gnome, XFCE, etc. If you're comfortable using keyboard shortcuts, this might be a good choice for you as well.

I also recently installed Dreamlinux on a couple of very old Dell laptops and I was truly impressed by how fast it ran on them. Dreamlinux is Debian Lenny with XFCE, so it's just as GUI as Xubuntu but a whole lot peppier. Personally I haven't found Xubuntu to be all that much more responsive than regular Ubuntu with Gnome.

You could also try Puppy Linux (although I really appreciate the advantage of having all those Debian/Ubuntu packages, so I tend to stick with one of those distros).
