Hi everyone,

I don't think I qualify as an absolute beginner.  But the mistake I made was a beginner's mistake so I thought this might be the best place to put this post.

I just tried to copy a bunch of files from a usb key to a directory.  I selected all the files in nautilus and used ctrl-x to cut them, then navigated to the destination and used ctrl-v.  Nothing happened.  I did it a couple of times and got the same result. :confused:  I then slapped my forehead as a realized what was wrong.  The destination was in /usr/share, so of course I didn't have write access.:rolleyes:

However, what surprises me is that nautilus didn't pop up a window saying something like "Attempt to write files failed.  User does not have write permission in the destination directory," or some such useful error message.  If I had been a true nooby I'd be puzzled and probably frustrated.  So this looks like a hole in nautilus' user interface to me.
