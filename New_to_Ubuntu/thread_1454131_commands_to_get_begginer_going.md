### Post by Didius Falco on 2010-04-14
Welcome to Ubuntu and the forums!

Have a read through this thread - it'll keep you busy for a while, at the least. A couple of other things I'd suggest:

1. Forum searches. You can just about always find the answer to a question that someone else asked before you did. Searching with "solved" as a keyword is a good idea - it that doesn't find the answer, repeat the search without the "solved" keyword.

2. Google Is Your Friend. Doing searches on Google (or your own favorite search engine) will often find answers for you. All the usual warnings about info from the web applies - look for trusted sources, never run a command you don't understand the consequences of, etc.

3. Stickies!! Often, you will find important information in "sticky" threads at the top of the threads list. Read them and save yourself some time.

4. Man pages. Commands available on your system  nearly always have man pages. Simply open a terminal window (Menu/Accessories/Terminal) and type "man command" for the manual page(s). For example:

 ```
man blkid
```

will tell you what the blkid command does, any options you can use, and list any other man pages for related commands.

5. Start a thread. If you just can't find it, or are having trouble understanding something, ask away!! You'll find that the vast majority of posters are more than willing to help - particularly if you've done what you could first to find out on your own.

### Post by t0p on 2010-04-14
[Here's a link]("http://ihatehate.wordpress.com/2008/07/16/some-common-linux-commands/") to a little cheat sheet of commands you may find useful.

Whenever someone asks me what command I think is most useful, I always reply: **apropos**.  It's great because it tells you what commands you might want to use in any given situation.  For instance, let's imagine you want to do something regarding wireless, but you can't remember what command you need.  Type into a terminal the command

```
apropos wireless
```

and it will give you a list of wireless-related commands.  Like this:

```

t0p@deimos:~$ apropos wireless
crda (8)             - send to the kernel a wireless regulatory domain for a ...
iwconfig (8)         - configure a wireless network interface
iwevent (8)          - Display Wireless Events generated by drivers and setti...
iwgetid (8)          - Report ESSID, NWID or AP/Cell Address of wireless network
iwlist (8)           - Get more detailed wireless information from a wireless...
iwpriv (8)           - configure optionals (private) parameters of a wireless...
iwspy (8)            - Get wireless statistics from specific nodes
wireless (7)         - Wireless Tools and Wireless Extensions

```

**apropos** is the most useful command I know of.  It's helped me out many a time.

### Post by mellort on 2010-04-14
Hey gaorutwe, glad to see you making the jump :). Others have already added great advice, but I want to just add a small point. By far, one of the best commands to know for linux is "man". It gives displays a help page for a command that you provide as an argument.

Say you saw someone use the command "ls", and you want to know what is does and how to use it. Type
```
man ls
```
and you get
```
LS(1)                                                    User Commands                                                   LS(1)

NAME
       ls - list directory contents

SYNOPSIS
       ls [OPTION]... [FILE]...

DESCRIPTION
       List  information about the FILEs (the current directory by default).  Sort entries alphabetically if none of -cftuvSUX
       nor --sort.

       Mandatory arguments to long options are mandatory for short options too.


```
and a lot more (I truncated the info). "man" is by far one of the more useful commands, especially if you are offline for whatever reason.
