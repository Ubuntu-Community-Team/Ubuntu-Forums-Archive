### Post by srs5694 on 2010-06-09
FWIW, I've got Debian 5.02a installed on my iMac G3 (the second variety, bought in 1999, with a CRT display and an orange case). It works fine, but I don't use X with it; it's a text-only system at this point. I've used X with previous Linux installations on this computer, so I know it can be done, but it's just not needed for me right now.

One tip about X configuration, assuming it doesn't work right off the bat: Get networking up and running first, including a remote login server such as SSH. That way, if you try to get X working and end up with an unusable console, you can log in remotely to shut down X and fix the problems.

Concerning your half-hour login time: How much RAM does the system have? I don't recall what my system has (it's powered down right now), but I do know that I increased its RAM, possibly maxing it out. If a Linux system lacks sufficient RAM, it'll swap heavily ("thrash"), which can slow down everything. You may also need to use the most lightweight X environment you can find, such as a "bare" window manager rather than GNOME or KDE. See the [Window Managers for X](http://www.xwinman.org/) site for a rundown of the most common choices.
