### Post by TheFu on 2018-06-25
Fingerprint readers are convenient, but not secure.  Many are supported.  You just need to install the driver, assuming the specific device you have is supported. [https://help.ubuntu.com/stable/ubuntu-help/session-fingerprint.html.en](https://help.ubuntu.com/stable/ubuntu-help/session-fingerprint.html.en)
[https://www.theverge.com/2016/5/2/11540962/iphone-samsung-fingerprint-duplicate-hack-security](https://www.theverge.com/2016/5/2/11540962/iphone-samsung-fingerprint-duplicate-hack-security)

Voice recognition also exists for Linux.  There are multiple projects (that used to be in github, but who knows if they are still there?).  There are local-only versions, or you can install the google/amazon voice things which destroy any privacy you had before.  The local STT engines (Speech to text) aren't nearly as good as the commercial services like google/apple/amazon use from Nuance and similar providers. If you want a system to read a file to you, then use a TTS tool (Text To Speech).
For your consideration:
[https://www.howtoforge.com/tutorial/speech-recognition-software-ubuntu/](https://www.howtoforge.com/tutorial/speech-recognition-software-ubuntu/)
[https://www.lifewire.com/state-of-linux-voice-recognition-2204883](https://www.lifewire.com/state-of-linux-voice-recognition-2204883)

There are actually full solutions that run on cheap, tiny Linux computers. [https://maker.pro/raspberry-pi/tutorial/the-best-voice-recognition-software-for-raspberry-pi](https://maker.pro/raspberry-pi/tutorial/the-best-voice-recognition-software-for-raspberry-pi)

Of course any of the projects would probably love donations or fixes to the code-base.
