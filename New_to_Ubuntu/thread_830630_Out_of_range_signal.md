### Post by mgmiller on 2008-06-16
Please try to resist the temptation to edit xorg.conf in Hardy.  It really doesn't want you to do that any more.  If you need to change your desktop resolution, here is how to turn on the gui and use it:


1) Right click Applications and select "Edit Menus"
2) Scroll to and select, by left clicking, the "Other" location on the left side.
3) On the right side put a check mark in the box next to "Screens and Graphics", then click "Close".
4) Go to Applications > Other > Screens and Graphics
5) On the screen tab, click the drop down next to model.
6) If you can find your model, great, if not, select "generic" at the top of the list and on the right find LCD of the correct resolution assuming it's an LCD, otherwise scroll down a bit and use Monitor of the correct resolution for CRT, or try Plug 'N Play.
If it is a wide screen monitor, don't forget to check the wide screen box.
7) Now you can find & select your resolution and refresh rate from the list on the right side. Click OK.
8 ) Go to the Graphics Card tab.
9) Click on Driver and select what you need.
10) Click the test button to make sure all is ok.
11) Click ok and you're done.

Now, for the answer to your original post about out of range during startup (and I assume shutdown)

Install the startup manager:

```
sudo apt-get install startupmanager
```

Once it's installed, run it from System > Administration > Startup-Manager

On the Boot Options tab look in the **Display** area & change 
1) Resolution to 640 x 480 
2) Color depth to 8 bits

Click the close button and let it do its changes thing.

You will probably have to reboot to see the changes.

The only things that are affected by this change are the Ubuntu graphic with the sliding bar underneath it and any virtual terminals you use from ctrl/alt/F1 for example.

### Post by rated727 on 2008-07-19
I received an "out of range" signal that originated in the monitor.  This happened immediately after I installed the correct 'drivers' for my OLD video card.

Ubuntu correctly detected my Plug-N-Pray monitor and set the video card to 1280x1024 16 bit color depth.  The problem was that the verticle synch rate that Ubuntu set (43 Hz) was out of the range that the monitor could use (50 - 160 Hz).  43 Hz gave the monitor (figuratively speaking) a serious case of indigestion.

I edited etc/X11/xorg.conf and removed ALL resolutions that used horizontal or vertical sync rates that were out of range for the monitor.  A given screen size (1280x1024 as example) might have differing synch rates depending on the color depth selected and how your video card handles the greated color information.  Sometimes, the synch rates slow as color depth increases.

It's great that your system "self corrected" but the possibility is that it can reoccur.  Your choices are to edit the xorg.conf file to match with your monitor and video card, **OR** be very careful of resolution/color depth changes to avoid falling into the same or another similar combination.

If you decide to be adverturous, look in the owner's manual(s) for specs or search the manufacturer's WWWeb sites.  I was lucky that my monitor showed the ranges, because I don't have an owner's manual.

### Post by malandro23 on 2008-07-25
Alright, I had the same problem and could find no help anywhere. The problem is that it is during boot that the problem exists and not while running X. Meaning: it is a problem related to the framebuffer which is what is used to display the "usplash" (the pretty Ubuntu logo with animated bar) during the boot process. So, any advice to mess with your xorg.conf file or anything that has to do with X is misleading.

I solved the problem by changing the options on the boot command line in "/boot/grub/menu.lst". It is here that you manage what kernels and operating systems you can log into. The key option that we are interested in is the one that says: "vga=XXX" where XXX is some number like 795. For a table of what these numbers mean, see: [http://www.mepis.org/node/2992]("http://www.mepis.org/node/2992")

To completely fix the issue I had to perform 2 steps:
1) Change the vga option in grub
2) Use the "startup manager" program (previously mentioned in this thread) to change the splash display to be compatible with the option I selected in 1).

1) Change the vga option in grub:
command:
$ sudo gedit /boot/grub/menu.lst
Find the line like:
"kernel          /boot/vmlinuz-2.6.24-19-generic root=UUID=717f84eb-0cc4-4626-98d7-7e6744e48308 ro quiet splash vga=792"
Change the number after vga= to something else. For the record, the number was initially 795 for me which corresponds exactly to my monitor's native resolution ( 1280 x 1024 ), but that did not work for me. I changed it to 792 ( 1024 x 768 ) and it worked fine after that, like magic.
IMPORTANT: After changing the vga option, you must reinstall grub:
To check which disk to install on (typically /dev/sda or /dev/hda) check:
$ sudo fdisk -l
   Device Boot      Start         End      Blocks   Id  System
/dev/sda1               1         976     7839688+   b  W95 FAT32
/dev/sda2   *         977       26576   205632000    7  HPFS/NTFS
/dev/sda3           26577       26638      498015   82  Linux swap
/dev/sda4           26639       30401    30226297+  83  Linux

$ sudo grub-install /dev/sda

2) Now use the startup manager (System->Startup-Manager) to select the corresponding resolution and color depth under the "Boot options" tab. For me, I used vga=792 in /boot/grub/menu.lst; and Resolution: 1024x768 and Color depth: 24 bits in the Startup-Manager.

Now I have a beautiful boot splash screen, just like it should have been all along. I really hope this helps save someone the horror of messing around with your xorg.conf file to try and solve this issue.
