### Post by RTrev on 2009-10-22
> **tomxyz said:**

I have only 256mB and if I remember correctly an 1,8mHz processor, will I notice the difference between Kubuntu 8.01 and Xubuntu 7.01? Now starting kmail or firefox takes like 20 sec.

I don't think you'd see much difference.  No matter what desktop you're running, for example, Firefox is a slow loading application.

When experimenting with various desktops, some of them are snappy and quick, and others ponderous and slow.  But at the end of the day what you've really spent most of your time doing is running applications.  So how those respond is more important.

Keeping your memory footprint low is essential with only 256M of RAM to play with, so that would suggest ruling out Xubuntu.

Note that the XFCE desktop is actually very light, but the Ubuntu distribution based on it seems to be very bloated.  You could make yourself a slimmed-down XFCE version of Ubuntu, if you really like the XFCE desktop, or you could explore Lubuntu.

Do you have the time and inclination to experiment and try a few of these and see what feels best to you?  That would be the way to go, if you're willing to put in a bit of time.

HTH.

### Post by RTrev on 2009-10-23
Well, no, the applications don't become lighter.. unless you deliberately choose lighter ones.  The folks doing the packaging usually do that for the system utilities such as the file manager, the text editor, etc., but your main day-to-day applications like mail and browsing can be lightened up a bit too.

For example, I'd been using Thunderbird for email and NNTP newsgroups, but found that Claws Mail was much lighter and faster.  Opens in one second or less on the first load, and does everything I need:

[http://www.claws-mail.org/](http://www.claws-mail.org/) (In the repositories)

Mostly I just use GMail for my email, and the Pan newsreader ([http://pan.rebelbase.com/](http://pan.rebelbase.com/)) for NNTP.  Another sub-second loader, and it's in the repositories.

Similarly, Firefox has a lot going for it, but for my purposes I've found that I like Chromium / Google Chrome much better.. and again it opens in less than a second, etc.  I was getting the daily Chromium builds, but found that it suited my purposes better to simply get the early access Google-branded version:

[http://dev.chromium.org/getting-involved/dev-channel](http://dev.chromium.org/getting-involved/dev-channel) (It will optionally add a Google repository during the initial install from the .deb file, so from then on it's in the repositories for you.)

All of these are personal choices, and are simply a matter of trial and error and finding what works for me and what doesn't.  You'll no doubt agree with some choices and disagree with others.

About Lubuntu specifically, I'm not using it yet.  I got the Karmic 32-bit Alternate CD last night and will install it first into a VirtualBox VM to play with it.  I'll do a command line install, and then install the LXDE package.  Until the actual release of Lubuntu, that might be the best way to do it for now.  I notice that they include Firefox/Iceweasel, so I just won't install that.  I'm also uncomfortable with using GDM, but will have to experiment and see how heavy that actually is.. and if I can make things work without it.

It's been a long time since I've played with things like this, and I can't remember half of what I thought I'd learned. <g>

I tried the live CD ([http://download.lxde.org/lubuntu-9.10/lubuntu-9.10_lynxis_b14.iso](http://download.lxde.org/lubuntu-9.10/lubuntu-9.10_lynxis_b14.iso)) on the Dell Mini-9, but didn't install it yet (in fact I don't see that it's even installable but I'm probably missing something obvious.)  I'll need to study it a bit more before deciding to re-do the Mini.  The live CD version didn't pick up my Bluetooth mouse, so I'll need to work on it a bit before even being able to test it.  I don't do well with trackpads. <g>

Also, again personal choice, I like having the same environment on both the desktop and the netbook.. makes it easy to keep them synced, and easy to remember what I'm doing (i.e., how to run them!)  The full Ubuntu runs pretty well on the Mini, since it has a Gig of memory and no Firefox or Thunderbird.  It isn't ideal, though, because it brings along a ton of stuff that has to be manually removed.. and things like Evolution are so deeply entrenched in Gnome that they're almost impossible to get rid of.

If I had to run on a system with 256M of memory, I'd do a lot of experimenting at this point.  The ultra lightweight desktop environments need some getting used to, as you've found, and I don't know which one I'd end up going with.  I do know that my choice of applications would remain largely the same.  A custom install of XFCE or LXDE is a possibility; like you I prefer to stay within the Ubuntu family, so am looking forward to seeing where they go with Lubuntu.  We saw where they went with Xubuntu, and it's not appropriate for low-memory machines.  I think they'll make the Lubuntu version work well on these machines.. but I have no idea when it will be ready.

I'd suggest you consider rolling your own starting from the Ubuntu alternate CD command line install and then adding packages until you end up with something you like.  It can be fun!  Make sure you write down each package you install, so that when you finally come up with the final answer you'll remember how you did it. <g>

Good luck, and let us know how you make out.

Bob

### Post by Robotman on 2009-10-23
You should definitely try Lubuntu.  Since it's not yet released, and since the Live CD doesn't do installs, you can install the LXDE package using Synaptic or with this command:
```
sudo apt-get install lxde
```
I think you should leave all the Kubuntu packages installed - don't remove them.  After you have LXDE installed, you can log out and log back in, but this time switch to logging into the LXDE session.  You can make this the default too, but I don't know how that's done in Kubuntu.  (I switched from Xubuntu.)

If things don't go as planned, or if you just don't like KXDE, you can choose to log back into the Kubuntu session and things should be as they were when you left them.

I've installed LXDE this way on a netbook and on my very first PC, and both of them are VERY fast compared to the way they operated before.  My first PC has only 384MB of memory, and only a 266 MHZ Pentium II processor.  LXDE has done wonders for it. ;)

By the way, if you can't find LXDE in the Ubuntu repositories for your release, you can do it like I did to my first computer and add the Launchpad repositories to your sources.  That gets a little more involved, so just ask if you need help.

### Post by RedSquirrel on 2009-10-24
> **Dullstar said:**
> Plus, the ISO files for both Puppy and Gentoo are about 100 megabytes, but the Ubuntu ones are about 700.  Rounded, of course, but that could be a good sign.

At least in the case of Gentoo, the size of the installation CD is not important. When you install Gentoo, nothing from the CD ends up in your finished installation since everything is downloaded from the internet. The installation CD is merely an *environment* (a toolkit) that allows you to type the commands to install Gentoo. (Note: There is no installer for Gentoo; you run the commands from the command line.)

If you want to do a "net install" of Ubuntu (again, where everything is downloaded from the net), then the [Minimal CD]("https://help.ubuntu.com/community/Installation/MinimalCD") is a good way to do it. It's less than 10 MB. Everything you need is downloaded from the net while you are running the installer.

You can replace icewm with some other window manager or desktop environment: **xfce4**, openbox, fluxbox, and so on. xfce4 is the metapackage that will give you a basic Xfce. It only pulls in the core of Xfce. You can add whatever you want to that.
