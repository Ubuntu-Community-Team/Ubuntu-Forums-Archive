### Post by Paqman on 2011-07-29
Depends. If you're going to be responsible for supporting it then you might enjoy the longer lifespan of the LTS release. If you don't mind the idea of upgrading the system every six months or so, then the non-LTS would be the way to go.

Unity works fine, don't believe the bleating. Even if you don't like it then Gnome 2 (aka Ubuntu Classic), Unity-2D, KDE, XFCE, LXDE and a whole slew of other DEs are all available on the latest release. Choice of DE isn't a reason to choose an older versions IMO.

### Post by XubuRoxMySox on 2011-07-29
For a non-geeky newbie, definitely the LTS!

LTS releases are built from Debian *Testing*. The regular releases are built from Debian *Unstable*.

The current LTS, 10.04 (Lucid) was built from "Debian stuff" that has since been released as "Squeeze," which is Debian Stable!

And being a Debian user, you know it doesn't get any more stable than that!

The regular releases are like using Sid, compared to the LTS bersions, which are more akin to Testing and to Stable.

### Post by beew on 2011-07-30
Definitely the latest. But wait a month or two after release date. So Natty at this stage is actually quite nice. Linux has rapid release cycle and new features and bug fixes get added very quickly. I would say Natty and Maverick in general work a lot better for me than Lucid (on several machines) IMO you may only want to stick to LTS if you have very old hardware that may be depreciated in newer Ubuntu. If your machine doesn't support Unity you may want to try 10.10 (again a lot faster and more stable than 10.04, LTS or no LTS)  

As your hardware is quite new and powerful I see no reason to stick to LTS for likely lower performance just because this obsession with "stability". Speaking of which, in Linux talk stability just means it doesn't change, so something consistently broken  or under performs so it brings no surprise is "stable" and this is often the case if you stick to Ubuntu's policy to only install software in the official repo, as they don't get updated throughout the lifespan of a release (3 years for LTS) I can't see how "stability" in this sense is even desirable.

The only advantage of LTS is that you don't need to update to a new OS for 3 years,--but I really don't see that as an advantage unless you have old hardware, as said above. It is not true, btw, though often said, that with non LTS you have to update the OS every 6 months, you don't, normal release is supported for 18 months and I usually keep mine for a year, so my main machine is still on Maverick.
