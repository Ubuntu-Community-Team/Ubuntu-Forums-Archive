### Post by 3rdalbum on 2012-03-11
> **gumshoegrant said:**
> In order to become efficient at learning Ubuntu or other linux flavors is it necessary to learn C and Python languages?

Definitely not. I don't know C. I know Python, but have only used this knowledge to write my own programs; not to actually use the Ubuntu system.

> I don't feel at ease with installing programs that are not available in the Synaptic Package Manager and Ubuntu Software Center.

You know what? That's perfectly fine. It's 2012, most people use the default repositories and the occasional PPA (third-party repository) only. These things exist to make our lives easier. Outside the repositories system is a Wild West; potentially every program outside the repositories could need installing in a totally different way.

> Just the other day I downloaded a package and when to open it and is prompted me for a directory to install it. duhhhh.

Handy tip: Most of the time when it asks you where to install it, there's a default location; the prompt might say something like this:

> Where do you want the software installed? [/usr/bin/]:

The bit in between the square brackets means "If you just press Enter without typing anything, the software will be installed in its default location - /usr/bin.

If it doesn't have a default location then it's probably safe to install it anywhere, even inside your home directory if you like.

The Flash plugin used to have an installer with no default location, and you did need to put the file into a specific location for it to work. Now that's brain-dead design right there by the Adobe guys.

> I want to learn Linux but I am just not getting it except if its plug and play.

If you stick to the repositories, and PPAs, and Deb packages, and GUI programs rather than editing text configuration files, you'll find that everything is more plug 'n' play than Windows. There's absolutely no shame taking the easy path, I do it because I believe computers should make things easier rather than harder. That's partly why I use Linux too.

However, don't confuse "This is hard" with "This is unfamiliar". Linux and the BSDs are so ridiculously different to Windows and Mac OS that you'll probably be doing a fair bit of learning. But on the whole, on nice compatible hardware, you'll find Linux easy to use once you're a bit more familiar.

Good luck!
