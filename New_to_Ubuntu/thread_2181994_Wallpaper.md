### Post by Frogs Hair on 2013-10-19
I have only seen this when the path to the folder has changed . An example would be if you were using one of your own backgrounds and moved it to a different folder the result being a black background when you logged in the next  time. The Ubuntu wallpapers are in file system under usr/share/backgrounds and that file remains the same during upgrades except the addition of the new wallpapers.

 In the Gnome Tweak Tool there is a setting under desktop to have the file manager handle the desktop and if you are using the GTT make sure that setting is activated.
