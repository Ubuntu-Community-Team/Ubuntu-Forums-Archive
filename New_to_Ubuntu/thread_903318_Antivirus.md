### Post by Too Late on 2008-08-28
I have been using Windows since 3.1 and have never used antivirus, and have had no problems. Ok, I've used antivirus programs, but they've never found anything.

The need of an antivirus program doens't really depend on your operating system. Of course, there are MUCH less malware for linux than for windows (and the reason is obvious when you look at the user counts), so the probability to get one is much smaller when using linux. But your own computer isn't the only one which is interestig. For example, when you are forwarding emails which contain suspicious attachments (eg. malware for windows), they don't suddenly become "clean" even thuogh you're receiving and sending them from a linux system. In such cases, the antivirus software can be useful, because you don't want to pollute others.

### Post by Killer Cop on 2008-08-28
Funny enough I've just read a very good article about Linux security.

You can read it here:

[http://nnucomputerwhiz.com/linux-virus.html](http://nnucomputerwhiz.com/linux-virus.html)

Saying that Linux/Ubuntu is resistant to viruses (or badware) is not correct. Every operating system can have viruses.

Now the difference between, lets say, Windows and Ubuntu, is how Ubuntu is built and how user management is made. You can read much more about this in the link I posted.

Now the most interesting aspect is if Linux/Ubuntu will get *more* viruses as Linux/Ubuntu becomes more and more popular.
At first glance, yes. At second glance, no.

Virus-writers are more likely to write malicious code to the most popular operating system, because the virus will affect most people. But I think that we're seeing proof that this statement might be wrong. Look at Mac computers' OSX operating system. Mac computers and therefore OSX, is getting rapidly more and more popular and I think I've read somewhere that Mac aproxx. have 5-6% market share, not quite sure though. But Mac's are becoming pretty popular now. And guess what? There's still "no" viruses for OSX as I know of. And that's again because OSX is built on Unix, which is pretty much like Linux.

But all in all, without writing a whole article, I don't think that more viruses will come because the operating system is beoming more popular. But time will tell, sooner or later.

While it is true people will start writing more viruses for Linux in the future, the security model most distributions follow (minus Linspire that promotes logging in as root) ensures that, it's a hassle to get a virus to run. To affect a linux system (rather than a persons home directory), the virus must have root privileges. In contrast, Windows is targeted because, it allows code to run without any authorization- while Vista's UAC (User Account Control) has tried to target this flaw, UAC itself is flawed.
While the "more popular" argument holds true to an extent, it can be proven false. For example, IIS vs Apache. Apache (run on non- Windows OS) has a much higher usage than Microsoft's IIS. However, it is not targeted as often as IIS. Thus, adoption rate's are only the tip of the iceberg in my opinion, and the security model is much more important at deterring virus- writers, than the usage numbers.
