### Post by DuckHook on 2016-12-19
Welcome to the forums, melldori

Unlike proprietary OSes, most of the drivers in Linux are already either built into the kernel or come as modules already included in your distribution. Therefore, on distros like Ubuntu, the OS works out of the box on most hardware. There are a few components that won't work: usually either really old or really new ones, or components that the manufacturer refuses to release specs for so that the community is unable to write drivers for them. In very special circumstances, some drivers do need to be installed after the OS installation, but this is unusual.

The standard recommendation for questions like yours is to run the 'buntu flavour of your choice in a LiveUSB session first. If the LiveUSB runs well, then a full install should work just fine without further drivers. If the LiveUSB won't run well, then you will likely have problems with a full install too and may need further downloaded drivers.

Instructions for downloading/running LiveUSB follow: [https://www.ubuntu.com/download/desktop/create-a-usb-stick-on-windows](https://www.ubuntu.com/download/desktop/create-a-usb-stick-on-windows)
