### Post by buzzingrobot on 2013-11-11
Is malware impossible on Linux?  No, not at all.  Claims to the contrary are not accurate.

But, that's looking at it from a technical angle.  In reality, the comparatively small number of people using Linux on their desktops means it is seldom targeted by malware creators.  These days, malware is a business -- a  criminal business, but still a business -- and the money is in Windows machines.

I've used Linux since the mid 1990's.  I've never used an antivirus and I'm pretty sure I've never been compromised. 

Now, Windows malware can be passed in email, etc., from one Windows machine to another via a Linux machine. E.g., if you use Linux in an office environment where everyone else uses Windows, an email infected with Windows malware won't affect you, but it can the person you forward it to.  In those circumstances, running an antivirus on Linux is good office policy.

Otherwise, install a firewall, be smart about using the web, weed out and ignore spam, use strong passwords and don't use the same password for everything.  (Linux users face the same risks everyone else does when it comes to the security of their passwords and personal data held by web businesses , etc.)

### Post by Jake Sweeney on 2013-11-12
It's less likely that you will be a threat to malware because the UNIX OS requires things such as sudo passwords, which are required before any software installs, unlike Windows. Therefore you would actually need to install a virus manually yourself to get infected. If you're worried about such software then use trusted applications from the Ubuntu Software Centre. 
Windows also has a larger market share in computer systems, meaning that most computers have Windows pre-installed on them, which means that there isn't really a point in hackers to bother writing software to hack a UNIX OS when they could easily do it to a Windows one without any trouble. 
You could install a firewall as previously discussed to protect your network from any malicious packets et cetera.
