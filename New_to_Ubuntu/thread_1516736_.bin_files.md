### Post by donkyhotay on 2010-06-24
> **beelzebufo said:**
> that worked, thanks man! but is there any way to sort of permanently set this up so that I dont' have to do that with every .bin file?  I mean that's just a bit complicated to have to do everytime I try to install something.

Downloading and installing executable files are a bad habit windows teaches people. Admittedly it's the only way to install some rare pieces of software however it should *always* be the last option (never first). As mentioned previously (for the java example) always check the repos, failing that look for a .deb file or something similar, after that *maybe* a .bin file however personally I'd rather compile software from source rather then use a .bin file. In regards to "why is everything so hard in linux" Remember that linux is designed with security in mind, downloading and installing an executable is intentionally meant to be hard because it's so potentially dangerous. Installing software is actually much easier on linux, you just go into either synaptic or the software center, choose the program you want and it's downloaded/installed without any fuss and from a secure source. Now doing things "the windows way" on linux is harder but thats because linux doesn't have all those design flaws that make certain tasks easier... like installing untrustworthy software.

### Post by mk1w86 on 2010-06-24
> **beelzebufo said:**
> that worked, thanks man! but is there any way to sort of permanently set this up so that I dont' have to do that with every .bin file?  I mean that's just a bit complicated to have to do everytime I try to install something.

There isn't anything to set up, the first chmod command changes the permissions of the file to allow you to execute it (the reason the file permissions do not allow you to execute it by default is for security).

The second command actually executes the file, the ./ placed in front of the file means execute the file in the current directory.

Hope this helps, if you want more information on file permissions have a look here:

[https://help.ubuntu.com/community/FilePermissions](https://help.ubuntu.com/community/FilePermissions)
