### Post by underdog512 on 2009-01-06
A package is file or set of that Ubuntu (and other linux distro's) use to install software. Sort of like a .exe file in windows.  In Ubuntu these files .deb files.  you can download these files from certain websites just like you can on windows with .exe files.  simply double-clicking on a .deb (or package) will automatically install that software.


As for Add/Remove and SPM, Add/remove is simply a more "user-frienly" frontend for synaptic. However I have noticed that Add/Remove does not display as many packages as synaptic so I just end up using synaptic.


Hope this helps!

### Post by lord_lethris on 2009-01-06
synaptic is also good for uninstalling and cleaning out redundent dependencys.

For example, I dont particularaly have a use for "Pidgin IM".  I use "emesene" and I only have the need for one chat protocol anyway as everyone I know is on MSN. A friend uses aMSN for video chat... somthing that Pidgin lacks real support for.

BUT Add/Remove wont uninstall Pidgin because of dependencies.  However synaptic will uninstall it, removing any foot prints, and dependent plugins.

However, if your "new" to linux, I suggest familirising yourself with "Add/Remove" first.  As synaptic can be a little complex, as it alows you to select more detailed packages.  For example, Add/Remove may have "Chess" and will select all the dependences you would more likely use... where as synaptic will have "chess", "chess-X11", "chess-KDE", "Chess-Gnome", "Chess-Dev", "chess-prod" etc, for more technical people who want to try there hand at DEV, or are not afrade to use BETA or Unstable packages.
