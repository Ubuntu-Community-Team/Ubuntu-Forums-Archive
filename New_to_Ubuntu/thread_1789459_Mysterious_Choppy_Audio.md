### Post by Toz on 2011-06-23
Two things you can try (from: [https://wiki.archlinux.org/index.php/PulseAudio#Glitches.2C_skips_or_crackling]("https://wiki.archlinux.org/index.php/PulseAudio#Glitches.2C_skips_or_crackling"))

1. In **/etc/pulse/default.pa** change:
```
load-module module-udev-detect 
```
to read:
```
load-module module-udev-detect tsched=0
```
then restart pulseaudio
```
pulseaudio -k
```

2. In **/etc/pulse/daemon.conf** change:
```
; default-sample-rate = 44100
```
to read:
```
default-sample-rate = 48000
```
then restart pulseaudio
```
pulseaudio -k
```

### Post by Toz on 2011-06-26
Try resetting your pulseaudio settings.
```
rm -r ~/.pulse ~/.asound* ~/.pulse-cookie 
sudo rm /etc/asound.conf
```
If you get any errors about missing files or directories, ignore them. 

Then reboot.

### Post by Toz on 2011-06-30
A couple of things to try:

1a. **/etc/pulse/default.pa** - add back the **tsched=0** to the lines **load-module module-udev-detect** and **load-module module-detect**

1b. **/etc/pulse/daemon.conf** change the two **realtime** lines to read:```
realtime-scheduling = yes
realtime-priority = 8

```

1c. Reboot

If the above doesn't work, then undo the changes then:
2. **/etc/modprobe.d/alsa-base.conf** - add to the end of the file ```
options snd-hda-intel model=<model>
``` where <model> is one of:> **auto** auto-config reading BIOS (default)
**3stack-dig**    3-stack (2-channel) with SPDIF
  **3stack-6ch**     3-stack (6-channel)
  **3stack-6ch-dig** 3-stack (6-channel) with SPDIF
  **6stack-dig**     6-stack with SPDIF
  **asus-mode1**    ASUS
  **asus-mode2**    ASUS
  **asus-mode3**    ASUS
  **asus-mode4**    ASUS
  **asus-mode5**    ASUS
  **asus-mode6**    ASUS
  **asus-mode7**    ASUS
  **asus-mode8**    ASUS

 You'll have to try them one a time and reboot inbetween each attempt.
