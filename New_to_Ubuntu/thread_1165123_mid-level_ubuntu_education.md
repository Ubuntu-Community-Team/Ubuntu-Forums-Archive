Use the man pages, Luke. At least for every technicality such as "what arguments does this command take ?" or "can this command do that ?". Even now when reading the man page of some utility I have used for over a year I still have some "Whoa, it can do that ?" moments.

For easier things like "where are those system logs anyway ?", a few seconds of googling usually solve the problem, or you could search this forum.

For more complicated stuff, well... You probably don't really need to know how the kernel works right away. If you are still interested, you could search the Ubuntu official documentation, for instance : [https://help.ubuntu.com/search.html?cof=FORID%3A9&cx=003883529982892832976%3Ae2vwumte3fq&ie=UTF-8&q=kernel&sa=Search](https://help.ubuntu.com/search.html?cof=FORID%3A9&cx=003883529982892832976%3Ae2vwumte3fq&ie=UTF-8&q=kernel&sa=Search) (a simple search on the word "kernel").

Then there's the *really* advanced things like how Xorg works under the hood. No one (and I mean no one) fully understands that, probably not even most contributors (but Xorg is in a different league altogether).

### Post by Didius Falco on 2009-05-20
With a bit of searching through the official and community docs, you can get a lot of information. I also make a habit of grabbing any promising links I see for further study.

For a really good look at Grub in particular, and things Grub-related (like fstab, for example) here is a site that will keep you busy for days, and serves as a solid reference resource:

[http://users.bigpond.net.au/hermanzone/](http://users.bigpond.net.au/hermanzone/)

That site is owned and maintained by Herman,one of the users here.

Here is a link to the User Documentation site. Lots of good HowTos and other good information here:

[https://help.ubuntu.com/community/](https://help.ubuntu.com/community/)

For searching the forums, I like to use this Google specialized search:

[http://www.google.com/search?hl=en&edition=us&q=+++site%3Aubuntuforums.org&btnG=Search](http://www.google.com/search?hl=en&edition=us&q=+++site%3Aubuntuforums.org&btnG=Search)

For a wider spectrum search, this is good:

[http://crunchbang.org/ubuntu-search-engine/](http://crunchbang.org/ubuntu-search-engine/)

I agree with the posters above, man pages are a really good thing to read through. I keep a terminal open all the time so I can research commands I'm unfamiliar with.

Here is a good resource for the actual command names, which can be used in conjunction with the man pages to see how they are implemented in Ubuntu:

[http://www.oreillynet.com/linux/cmd/](http://www.oreillynet.com/linux/cmd/)

I also keep a file of my own named Useful.Commands that I have a launcher on my desktop set up to open. I grab useful linux commands and whack them in there, and use it as my own reference.

That should keep you busy for a bit... study it well - it may be my chestnuts you pull out of the fire one day!! ;)

Regards

Didius
