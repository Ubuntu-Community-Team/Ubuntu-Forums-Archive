you need to update the fstab to reflect that the UUID of your swap file has changed when you resized/reformatted.

```
ls -alh /dev/disk/by-uuid
``` will get you the uuids
or you could just remove the UUID and replace it with /dev/sdaX or wherever the swap lives.

After doing that you'll need to use the swapon command to turn on the swap file.

### Post by Stefanie on 2008-05-29
i thought things were solved, but they weren't... hibernation still did'nt work. after hibernation, the computer rebooted instead of resuming, and the swap partition was broken. gparted didn't recognize it anymore and I had to reformat it every time after hibernation.

after a bit of googling i found out that i needed to modify another file:
```
gksudo gedit /etc/initramfs-tools/conf.d/resume
```

in this file i had to replace the old uuid-number of the swap-partition with the new one (which can be found by running  sudo mkswap /dev/xxx). after saving the file i had to run 
```
 sudo update-initramfs -u
```

and now everything works fine!

i just post this in case other people have the same problem.
