### Post by kcrussell on 2009-12-21
My mum's XP laptop has developed major problems. Without going into the details, despite a lot of research I haven't manage to get a fix, and I am now considering whether the best solution would be to nuke it from orbit and install Ubuntu.

My mum has a very limited set of requirements: internet access for web surfing and browser based email, and Skype for chatting to the grandchildren! She is comfortable with following a linear routine to perform these activities, but has virtually no understanding of more general personal computing concepts and no ability to handle non-standard situations or perform basic fixes (e.g. happy that Firefox = "The Internet", but would look puzzled if you asked her to open a web browser).

Does anyone have first hand experience of using Ubuntu for someone in a similar situation, please?

Key features for me would be the following:

1. Remote access: Essential as I rarely have physical access to the laptop. Is it correct I can do this by using VNC on my XP PC and the built in remote access features of Ubuntu on the laptop?

2. Automatic Updates: Can I configure Ubuntu to keep itself up to date without any user intervention? My mum would not be able to run any sort of administrative commands or manual update processes.


I spent a year or so running Suse on my PC (a few years ago) and ultimately returned to Windows because even with a techy background, I found the package installation and tweaking required to get things running was too time consuming and frustrating. I know things have moved on, and I think this could be a good solution for her (and possibly less vulnerable to security problems if my mum naively wanders into the wrong places!). Basically I would like a robust system with limited features that I can fit and forget.

Any experience or advice gratefully received.

### Post by Mark Phelps on 2009-12-22
> **kcrussell said:**
> ... Any experience or advice gratefully received...

Several things for you to think about ...

1) Familiarity.  Once folks become familiar with doing stuff a certain way, they can become very nervous and anxious about changing that.  While Ubuntu doesn't present any "major challenges" in learning how to use it, there are other distros (available from the distrowatch.com page) that are much more like the MS Windows "way" of doing things.  If your goal is getting your mom away from MS Windows into something else that looks and behaves very similarly, you should look into these other distros.  

2) Try-before-you-buy, or in this case, "install".  Not all Linux distros are created equal.  Some work better with some hardware than others.  Whichever ones you want to consider, burn LiveCDs and try them out on the PC first -- this is especially true of Ubuntu which, in current versions, is likely to run rather poorly on a P III, if it will even install.

3) Make a full image backup as soon as you get the machine working -- and save that somewhere safe so that you can "easily" restore the PC should problems arise.  Fortunately, the two major things that crash MS Windows boxes (Windows Update and Malware) won't affect the Linux setup, but it's better to be safe and have the ability to restore, than to have to do an entire reinstallation from scratch.
