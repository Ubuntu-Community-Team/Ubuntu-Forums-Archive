### Post by lisati on 2016-05-07
128MB might be a bit of a squeeze for "standard" Ubuntu to work well. My first laptop, which I gave away several years ago, had only 222Mb or so available RAM, and was beginning to struggle to run well with that amount of RAM. I'd recommend upgrading the amout of RAM to at least 512Mb (1Gb would be better) if you can. Other forum users might be able to recommend alternatives to "standard" Ubuntu.

### Post by DuckHook on 2016-05-07
128 MB is a tough call. lisati is correct: 256 MB is really the absolute minimum for any 'buntu flavour these days and even then, you will hate the performance. However, if you like the challenge of fitting *something* into such a small footprint, you have the following options:

[LIST=1]
[*]Ubuntu server
[*]mini.iso
[/LIST]
Both above are command line only, but will run in your machine. You will not like the performance at all if you then decide to layer a desktop environment on top of the CLI (it may even choke and not start up). If willing to go outside the Ubuntu family, I would suggest:

[LIST=1]
[*]Tiny Core
[*]Slitaz
[*]Puppy
[/LIST]
Of the three, Tiny Core is the most resource frugal, followed by the other two. You won't really like the graphics or the hobbled apps if you are used to slick modern DEs but that's the trade-off on a very resource-constrained system.
