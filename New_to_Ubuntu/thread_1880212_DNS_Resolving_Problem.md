### Post by SeijiSensei on 2011-11-13
A quick test ("host [www.google.com](www.google.com) 9.9.9.1") shows that 9.9.9.1 does not resolve DNS queries.  I don't know what your ISP is telling you, but the configuration you had is definitely wrong.

First, the "search" parameter takes symbolic domain names.  These are attached to "unqualified" names to construct a "fully-qualified domain name."  So if I had "search ubuntuforums.org" in my resolv.conf, then typed just "www", it would do a search for "www.ubuntuforums.org".

For most ordinary users, a search parameter isn't required.  Just make your /etc/resolv.conf look like this:

```

nameserver 8.8.8.8
nameserver 8.8.4.4

```

which uses Google's public DNS servers.
