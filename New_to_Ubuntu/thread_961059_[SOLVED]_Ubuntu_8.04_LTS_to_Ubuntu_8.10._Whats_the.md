### Post by pansz on 2008-10-28
Generally, upgrade from 8.04 to 8.10 should be the same as the 8.10.

However, that is a goal which cannot usually be achieved within only 6 months. Due to various kind of bugs, systems upgrade from 8.04 to 8.10 will have some minor differences between newly installed 8.10.

The ubuntu team had been trying their best to make upgrade the same as a new installation. If there's differences, please report as bug. Thanks.

### Post by sdennie on 2008-10-28
Because Ubuntu releases happen every 6 months, between releases there is generally not a *huge* difference.  After 6 months, 8.04 seems like a solid release and, unless there is a feature in 8.10+ that you really must have, I think it's perfectly reasonable to stick with 8.04.  Doubly so considering your machine comes pre-installed with 8.04 which presumably means that it works flawlessly out of the box and the vendor has some support infrastructure in place to support that version of Ubuntu should you need it.

### Post by CloudFX on 2008-10-28
The point of having new releases is primarily to fix bugs, update applications, and add new features.  If you stick with Hardy, much of the software you use may be outdated.

As for LTS, it simple means that Canonical is providing security updates and major bug fixes.  This is what you are downloading whenever you need to update your system.

You won't see much difference with the user interface, however.  And while many think that there is little change with new releases, that is not true.
