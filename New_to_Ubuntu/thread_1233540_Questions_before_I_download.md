### Post by Buuntu on 2009-08-08
The terminal is what you use to end programs.  
I would suggest first binding Ctrl+Alt+T to your terminal so you can access it quickly (System>Preferences>Keyboard Shortcuts).  
To see what processes are running type top.  If you then want to kill a process, just press k and the PID of the process (listed in top).  Press q to quit.  
You can also type ```
killall processname
``` to end a process, but you must know the name of the process first.  Kill takes the PID as the paramater instead.
If you ever wonder what parameters a command does, or just want to know more about it, type ```
man command
```.

Your network isn't working?  If you have an ehternet cable, you should be able to directly connect to your router without any issues.  If you're using wireless though, Linux might be having problems detecting your wireless card.  I suggest you do a bit of research on getting WiFi to work on Ubuntu.  

If nothing works, it is probably because Linux does not come with the drivers for your card.  Luckily there is a program to use windows drivers called Ndiswrapper if that is the case.

Check out [https://help.ubuntu.com/](https://help.ubuntu.com/) and [https://wiki.ubuntu.com/](https://wiki.ubuntu.com/) as well as checking out these forums.  Use the search tool, it works great ;).

I suggest downloading and reading this book, it has a lot of information to get you started: [http://www.ubuntupocketguide.com/download_main.html](http://www.ubuntupocketguide.com/download_main.html)

Best of luck.

### Post by Buuntu on 2009-08-08
Yep, Linux isn't recognizing your card, see if what I posted above helps.

To install the windows drivers:
1. You will need to find the windows drivers for your card and download them.
2. Extract the folder with ```
sudo unzip folder
``` if the file is a .zip file.
3. Find the .inf file in the drivers folder
4. Go to System > Administration > Windows Wireless Drivers.
5. Select "add new driver" and browse for the .inf file you found.

After you are done with that run
```
sudo depmod -a
sudo modprobe ndiswrapper
```

If you do not automatically connect to the internet after this, you might have to add your wireless network in network manager.
Just type in the name of your network in SSID and add it.

1. Please don't advise new users to manually edit this file - a typing error can temporarily stop the package manager from working. Every day, someone incorrectly modifies their sources.list and has to post to UF asking how to fix their mistake.

2. The poster has already complained about having to use the terminal, why not just suggest System > Administration > Software Sources, and click the lovely GUI checkboxes to enable Multiverse, Universe, Updates and Security Updates? That's what it's there for!

Before doing this, you should plug your computer directly into your router with an Ethernet cable, so you have connectivity.

Once you've enabled these repositories ("repositories" are just places where you and Ubuntu can retrieve software from easily) try looking in System > Administration > Hardware Drivers. It might suggest a driver that it can automatically install for your wireless. If that's no go, then try installing the "linux-backports-modules-jaunty" package from System > Administration > Synaptic Package Manager. Reboot.  This package includes more recent drivers for Ubuntu and it might be enough to get your card working.

Also check that your wireless kill switch is not active.

If this still doesn't work we'll need to know what card you have. Most wireless cards do work easily on Ubuntu, but a couple don't. Open a terminal by going to Applications > Accessories > Terminal and paste in these two commands, individually:

```
lspci
lsusb
```

Copy and paste the results into this thread and we should be able to tell what wireless card you have.

Note that you will need to use the Edit menu of the terminal in order to use the paste and copy commands.

### Post by Buuntu on 2009-08-08
> **3rdalbum said:**
> 1. Please don't advise new users to manually edit this file - a typing error can temporarily stop the package manager from working. Every day, someone incorrectly modifies their sources.list and has to post to UF asking how to fix their mistake.

2. The poster has already complained about having to use the terminal, why not just suggest System > Administration > Software Sources, and click the lovely GUI checkboxes to enable Multiverse, Universe, Updates and Security Updates? That's what it's there for!

Before doing this, you should plug your computer directly into your router with an Ethernet cable, so you have connectivity.

Once you've enabled these repositories ("repositories" are just places where you and Ubuntu can retrieve software from easily) try looking in System > Administration > Hardware Drivers. It might suggest a driver that it can automatically install for your wireless. If that's no go, then try installing the "linux-backports-modules-jaunty" package from System > Administration > Synaptic Package Manager. Reboot.  This package includes more recent drivers for Ubuntu and it might be enough to get your card working.

Also check that your wireless kill switch is not active.

If this still doesn't work we'll need to know what card you have. Most wireless cards do work easily on Ubuntu, but a couple don't. Open a terminal by going to Applications > Accessories > Terminal and paste in these two commands, individually:

```
lspci
lsusb
```Copy and paste the results into this thread and we should be able to tell what wireless card you have.

Note that you will need to use the Edit menu of the terminal in order to use the paste and copy commands.
