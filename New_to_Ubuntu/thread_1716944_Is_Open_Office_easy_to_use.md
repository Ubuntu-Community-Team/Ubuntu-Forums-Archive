### Post by Old_Man_Unix on 2011-03-29
> **brawnypandora0 said:**
> I've never used it before, only MS Office. If I create a powerpoint or Word document in OpenOffice, and then use my school's Windows computer to open it, will the formatting be exactly the same?

The transition between MS Office and Open Office is probably the easiest one you will do as you move from Windows to Linux. 

For documents, you will notice little if any difference.  Open Office can open any of the MS Word formats as well as store to those formats.   The latest version of MS Word can also open ODT files , which is the Open Office native format.   The documents will be essentially the same, so you can simply use whatever file format your school requires.

As for presentations, you may see some compatibility issues with some of the more ***advanced*** features of PowerPoint.  Embedded files and multimedia files are handled differently, so if you use those  features you may need to do some adjusting as you switch between the two.  A 'standard' presentation will be no problem, as Open Office can open PowerPoint files as well as store to that format. 

I suggest you install the MS core fonts.  They aren't included in Ubuntu  by default,  There are excellent FOSS equivalents  in Open Office that  have the exact same metrics,  so you would be hard pressed to tell the  difference.   But some institutions still insist on the MS fonts for  some reason.

By the way, there is a new Linux office suite called Libreoffice, which  will be included as the default in Ubuntu starting with Natty.  It has  all the features of Open Office and a few more as well.

### Post by Hagar Delest on 2011-03-29
Remember that the MS Office file formats have been reverse engineered. So you may face some small glitches if you export in those formats. That's the vendor lock-in policy used by MS to keep their market shares: prevent the user to use something else than MS Office.

Note that the ODF support in MS Office is not perfect neither. Just to make the users believe that this is not a good format. For the record, the OOXML version used by MS Office is not the ISO standard they have managed to certify, they use their own flavor (transitional version) that can't be completely implemented in OOo or LibO (some features based on non documented features).

So to be short: chose carefully the file format you'll use.

OOo is much more powerful when it comes to styles, especially in Writer. Some differences with MS Word but more robust IMHO.

### Post by SeijiSensei on 2011-03-31
> **brawnypandora0 said:**
> A risk? So what determines the formatting?

Most versions of Word actually use the printer driver to determine how a page should be displayed.  So even opening a Word document on two different machines connected to two different printers may change the formatting. Linux software is never so closely tied to specific hardware and drivers.  So if you are concerned about details like the specific location of boxes or graphics, or where the page breaks will occur, you're probably best off saving as PDF.  The document won't be editable by others, but outside of collaborative authoring situations, I consider that a plus.
