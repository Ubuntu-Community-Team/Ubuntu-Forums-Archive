### Post by PeterJS on 2008-04-25
Another option that never gets the attention it deserves is use a local mirror. I'm sure there's a LUG, generous hosting company, or university with an Ubuntu mirror and plenty of bandwidth somewhere in the world near you. You'll probably get a lot better performance from a local server that is: 1) closer 2) not being hammered to death. And it would be a nice thing to do for the folks that are using the main servers, puts that much less strain on them.

Heck the software sources dialog includes an option to find the fastest mirror from a big old list, granted it only searches the repository mirrors but the odds are good that if some one is mirroring the repositories they're also mirroring the releases.
