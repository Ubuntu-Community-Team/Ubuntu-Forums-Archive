### Post by djbon2112 on 2008-08-29
There's no real "speed difference" between 32- and 64-bit, that is to say, that for most applications, it really doesn't matter. However, 64-bit gives you more address space (more possible RAM) and some drivers MAY be a little faster. Both 32- and 64-bit OSes support multi-core processing (called SMP, or symmetric multiprocessing) and up to ~3-3.5 GB of RAM (depending on other factors), so you'll be fine either way. Some have reported problems with some things in 64-bit Ubuntu (Flash used to be a major one), but for me personally, 64-bit Hardy is just as stable and easy-to-use as the 32-bit version!

---

### Post by WRDN on 2008-08-29
For most applications, you won't notice a significant performance increase/ decrease between the 32bit and 64bit Operating System. If however, you do memory intensive, and processor intensive tasks, or compile programs, then you would benefit from using a 64bit OS.
Some may argue that application support for 64bit isn't as good as that for 32bit, but the list of 32bit only applications is dwindling every day (and it is already very small), so you shouldn't have to worry about support. Also, it is possible to run 32bit applications in the 64bit OS, removing more compatability problems.

If you have a 64bit processor, then I see no compelling reason to stick with a 32bit OS now.
