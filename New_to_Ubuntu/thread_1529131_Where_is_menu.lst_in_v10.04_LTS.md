### Post by jrothwell97 on 2010-07-11
menu.lst is no more since 9.10, I believe. Grub has moved to the new GRUB2 platform, and this means the grub.cfg (as it is now called) file is auto-generated by a series of scripts in /etc/grub.d.

### Post by mikewhatever on 2010-07-12
Right, one way of removing the 8.04 kernels from the menu would be formatting the 8.04 partition. If you just want to clean up the menu, add the following line to /etc/default/grub
```
GRUB_DISABLE_OS_PROBER=true
```
That should prevent grub from looking for boot options on partitions other then root. When done, run 'sudo update-grub'.

### Post by -kg- on 2010-07-12
Alternatively, if you want to get rid of 8.04 and give XP Professional more room, you can delete the logical partitions within the Extended partition (on your sda drive) and then the Extended partition itself (getting rid of GRUB indicated in the Extended partition), then either recreate the Extended partition, creating an extra NTFS Logical partition inside the Extended partition, or resize the XP partition to cover the remaining free space.

My personal suggestion would be to create another partition in the free space.  You can use an Extended partition or not, depending on your desires and (future possible) requirements.  It is always dangerous performing partitioning operations, particularly in resizing OS partitions.  There is little danger in creating a partition in free space that is already available, though.

You might want to read the tutorial at the link in my signature block if you want or need to become more comfortable and informed in performing partitioning operations.

If I remember correctly, Extended Partitions are sort of like containers which house Logical partitions.  So you've certainly deleted the Logical partitions for Ubuntu and the swap, but the extended partition might still be there, housing nothing but empty space and preventing you from expanding other neighboring partitions.  In Gparted, it'll have a name and a box graphically representing it.  You should be able to just right-click on it and tell it to delete. Stretch the edges of the bar that represent that partition out to fill in the gap and then click Apply.  Shouldn't take long.
