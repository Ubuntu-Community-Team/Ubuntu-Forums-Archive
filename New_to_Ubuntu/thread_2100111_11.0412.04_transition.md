### Post by Don Martin on 2012-12-31
I have been using Ubuntu on my Gateway notebook, AMD 64 chip, for a couple of years now.  When I upgraded from version 11.xx to 12.xx there were two problems which I could not find a way around.  First, in 12.xx, every time I turn on the computer, the screen is at the dimmest setting and I have to go into settings / brightness, and increase the brightness which lasts until I reboot.  Then I have to do the same thing again.  Second, connecting to URLs on the web can take from 35 to 45 seconds. 

 I couldn't find a solution for these glitches anywhere, and so partitioned my drive with 12.xx running on one half and 11.xx running on the other.  A little while ago I received a message that 11.04 was no longer supported.  A few days ago the web became inaccessible in 11.04 (although the mail still works fine).  I click on the Firefox icon, get the message 'connecting', the little circle goes round, but then the screen simply reverts to the desktop.

As a result, I'm now using 12.04 exclusively, but find myself constantly irritated by having to reset the brightness and wait so long for connections.  

Is it likely that the failure to connect through Mozilla Firefox is because their changes are no longer supported by Ubuntu in 11.04?  If so, or if not, is there anything I can do to fix it?

Is there any way around the brightness and delay problems in 12.04, or do I need better hardware?  Or what?

Thanks for any help.

Don

Don

If you describe your computer in more detail, we can give better advice:

cpu model and speed
ram size and speed
graphics card/chip
wifi card/chip (if any)

But I'll start here: There might be a problem with

- your graphics or wifi card/chip
- too show cpu or too little ram for the desktop environment of vanilla Ubuntu

A lighter desktop environment can revive an aging computer. Try the lighter flavours of Ubuntu

- Lubuntu with the ultra-light LXDE or
- Xubuntu with the light XFCE

### Post by DuckHook on 2013-01-01
> **Don Martin said:**
> If I can just connect with the web again in 11.04, I can live without going to 12.04, even though I do like the feel of that version better.

Please invoke *firefox* from the command line (open up a terminal and type *firefox*), which may give a better indication of the errors at start-up than the graphical invocation. Post any errors back to this thread.

Sudodus is correct: running unsupported versions of distributions is problematic because:

1. Forum members are usually running only supported versions. Therefore, even trying to find help will steadily become much tougher.
2. Unsupported versions can leave security holes in your installation. The more you fall behind, the bigger the security holes become.
3. Firefox is an example of an app that needs to have its security holes plugged with absolute currency and effectiveness. Running an ancient unpatched firefox is a recipe for disaster, esp if you use it for critical stuff like banking, etc. Even if you get it running again, it will remain unpatched. Speaking for myself, I would upgrade for this reason alone.

I also agree with Sudodus re: using Lubuntu. Your system is powerful enough to also consider Xubuntu. Would encourage you to burn a CD of each and take for test drive. Try them out first as LiveCD sessions to see if they are your thing. Nothing to lose, everything to gain. Who knows? You may end up liking them more than what you have now.

BTW, not to beat a dead horse, but by the time you figure out what is wrong with your web browser, you may very well be up and running on a current version complete with ongoing patches and full security.
