### Post by donkyhotay on 2014-06-20
different languages have their pros and cons. C++ is what a lot of the "core" linux programs are written in along with the kernel itself. It's very powerful, forces you to have good habits and programs written in it are much more efficient since they're actually compiled. Python is my personal preference, it handles a lot of stuff for you automatically (no need to declare variables for instance) and is fairly portable however since it lets you get away with things it's easy to learn bad habits. Never coded in java but for some reason programs written in java and ruby seem more sluggish and resource intensive then even python and I just don't like them even though some of my favorite programs (minecraft for one) are written in java. Ultimately it's up to you what you, I would recommend playing around with various languages until you find one that seems to work for you.

### Post by donkyhotay on 2014-06-20
C++ is C but with the ability to use classes. Anything you can do in C, you can do in C++ but not the other way around. I started with C++ when I started programming but a friend showed me python and I haven't touched C++ in years because python is so much simpler. Thats why I was saying that python teaches bad habit. When I try doing anything in C++ I keep forgetting the most basic stuff (like having to declare everything) and move back to python because I'm lazy and don't care enough about doing things right. Be aware though that I don't code for a living, I'm not a particularly good programmer, and code primarily because I want something that doesn't already exist. On the other hand my friend that got me started in python really excels at it and uses it professionally. Deep down all languages are very similar to each other. I mean ultimately a computer program is nothing more then a very long and complex math equation. The differences in languages are in the implementation how you specify it. Your best option is to play around with all of them at least a little bit. The reason there are so many languages is because they all have their own pros and cons. If there was one "best language in the world" then everyone would use it and the others would vanish. You need to find out which works for you. The things you learn in one will help at least partially in another. Knowing about variables, functions, classes, etc. are invaluable as any programming language will use these things though of course the details will differ.
