### Post by xedi on 2012-05-19
You could use a spare computer or dualboot with another linux distro (Ubuntu again e.g.) to use as an experimentation environment without being afraid to have an unusable computer.

Generally I would suggest the best way to learn Linux is to just use it. Think about stuff you want to do or see others doing and then try to figure out how to do it and if you get stuck, ask here in the forum.

If you want to specifically learn the terminal, maybe try to accomplish as many tasks only using text commands and see how far you can go before it gets too frustrating.

If you want to learn more about the components of an Linux operating system, you might also consider trying out Arch, which is a quite minimal Linux distribution where you have to install many components by yourself (I would not install Arch as an everyday OS as a beginner so do that only if you feel like playing around and experimenting and have way too much time)

[http://lifehacker.com/5680453/build-a-killer-customized-arch-linux-installation-and-learn-all-about-linux-in-the-process](http://lifehacker.com/5680453/build-a-killer-customized-arch-linux-installation-and-learn-all-about-linux-in-the-process)
