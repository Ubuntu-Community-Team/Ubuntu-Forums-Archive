### Post by drs305 on 2008-12-22
While Taurus is waiting for the data he asked for, I can explain the nautilus actions. NTFS/fat non-native linux formats act differently when it comes to permissions. The owner/permissions are set at the time of mounting for these filesystems and cannot be changed. So if root mounts the device, root will continue to own it until it is unmounted. You can try to change the ownership via nautilus or command line but the ownership will not *stick*. On the other hand, even though the partition may not be owned by a user, the user *may* be able to read/write to the partition even though the owner is root.

### Post by taurus on 2008-12-22
You can use gparted (install it if you don't have it in System -> Administration) to format it to ext3 (not swap).

Then, you can add an entry in /etc/fstab to have it mount automatically each time you boot Ubuntu, if that's what you want.

```
/dev/sdb1   /media/share   ext3   defaults   0   2
```
Of course, you need to create that new mount point or else it won't be able to mount to /media/share.

```
sudk mkdir /media/share
```

### Post by Alden born on 2009-01-28
Ok i found the fix. when you have this error 

```
'net usershare' returned error 255: net usershare add: cannot share path /media/disk/TS as we are restricted to only sharing directories we own.
	Ask the administrator to add the line "usershare owner only = False" 
	to the [global] section of the smb.conf to allow this.
```

go to 

/etc/samba/smb.conf

and add this under [global]

usershare owner only = False

and then you can share folders on over drives that the os is not on.
