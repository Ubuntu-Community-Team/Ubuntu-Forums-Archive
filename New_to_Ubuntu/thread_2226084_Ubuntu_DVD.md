### Post by coffeecat on 2014-05-25
Being a bit on the silvery side myself I know the feeling! And welcome to the forum.

If the instructions you were looking at were for something called wubi, I suggest you forget about it. Wubi was/is a way of installing Ubuntu inside Windows and was a good way of trying out Ubuntu short-term, but it has several disadvantages and although still available it is no longer maintained.

The best way is to download the ISO file and burn this to DVD yourself to make a bootable DVD. Here is a guide:

[https://help.ubuntu.com/community/BurningIsoHowto](https://help.ubuntu.com/community/BurningIsoHowto)

And here is where you can download the ISO:

[http://www.ubuntu.com/download/desktop](http://www.ubuntu.com/download/desktop)

The donation, by the way, is not mandatory.

But before you do anything, I suggest you post some details of the specifications of this old machine. You need to be sure whether the processor is 32 or 64 bit and an old PC may have insufficient RAM to be able to run the Unity desktop of Ubuntu comfortably. Unity is somewhat memory intensive. There are other lighter variants of Ubuntu and one of these might be preferable for an older machine.

Checksum is important to verify and guarantee the file's integrity. One easy (multi-platform) way is to install the DownThemAll add-on for Firefox. Then, first of all copy (without spaces in the beginning or the end) the checksum code (MD5, SHA1, etc.) for the file you intend to download; click to download as usual but instead of letting Firefox directly handle the download (the default option) select DownThemAll. In the following DTA's dialog window select the destination folder as usual and paste the checksum code in the proper field making sure the correct checksum "type" is selected (if you have MD5 then select MD5 from the dropdown menu if not selected already).
This procedure will download and verify the downloaded file. Provided the final result is OK - you would get an error message from DTA saying the checksums don't match otherwise - you don't need to worry about checksums again. The file is as it should be.

Next, to burn it in Windows, avoid the method you linked which is for the integrated burner (lots of problems so... don't). Instead use any third-party CD/DVD burning software like the commercial Nero or the freeware **CDBurnerXP** (in spit of the name it runs on Windows 2000/XP/2003 Server/Vista/2008/Win7/Win8 (x86 / x64)) or **ImgBurn**, also freeware. Regardless of the software you choose you need to select the option of "burn a disk (CD/DVD) image (ISO) to disk (CD/DVD)". Done!

### Post by Vladlenin5000 on 2014-06-05
DTA is a download manager with many features, namely checksum, Firefox's stock manager hasn't.
DTA is a _Firefox_ add-on and it should be installed like any other _Firefox_ extra, from within the the browser itself (Tools > extras or something like that).
DTA was just a suggestion, one among thousands to help you download and verify the downloaded file in one single step.
AFAIK, DTA is only available for Firefox (OS independent). There are many similar add-ons for Chrome/Chromium and many similar standalone apps for any OS in the planet.

Burning: You can use the Ashampoo software you already have. Make sure you select the correct option as previously mentioned.
