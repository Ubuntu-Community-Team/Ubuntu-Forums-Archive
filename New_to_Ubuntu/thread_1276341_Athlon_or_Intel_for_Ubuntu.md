> **mjp29 said:**
> But is one better or less better when it comes to compatibility issues?

There is absolutely no difference in terms of compatibility between the two makes of processors. They both pretend to be really fast x86 processors, even though they do different things internally. Motherboard components - BIOS, integrated graphics, integrated wireless, and so on - can be a different matter. They don't break down into AMD/Intel lines, though.

Pick the motherboard that does what you want (ideally without incompatible components) and that will determine the processor that you get.

### Post by CatKiller on 2009-09-27
> **mjp29 said:**
> I understand that both processors are the same as being compatible with Ubuntu.  But when doing an install, is there also no difference in the compatibility when using/adding peripherals (such as older printers) and programs - ?

Not with the processor, no. With the motherboard, maybe, but that's broadly to do with what sockets it's got; whether it's got a parallel port or not, FireWire or USB, IDE or SATA hard drive controller, that sort of thing.

With binary compatibility with programs there are only three architectures that you're interested in; 386, 686 and x86-64/amd64. 386 is bog-standard 32-bit which will run on any x86 processor. 686 uses the enhanced instructions for processors of that era (SSE and so on) but isn't used much. x86-64 is the 64-bit version. As has already been pointed out, if you're buying a desktop processor, it's going to be 64-bit. There is [no significant difference]("http://en.wikipedia.org/wiki/X86-64#Differences_between_AMD64_and_Intel_64") between AMD and Intel's 64-bit implementations; Intel's own, that was incompatible with AMD's, was a complete market failure so they implemented AMD's instead.

All Free software can be made to run in 64-bit, since it can be compiled with whatever options you want. The problem was that Adobe were rather tardy with bothering to release a 64-bit version of Flash (and a handful of other companies too, to be fair). This was a problem for users of 64-bit Windows too, and is an inherent problem with proprietary software. The case of Flash has been solved for nearly a year now.

The Ars Technica System Guide that I posted earlier really is worth a look. It's six months old now, but that only really means that the prices have come down.
