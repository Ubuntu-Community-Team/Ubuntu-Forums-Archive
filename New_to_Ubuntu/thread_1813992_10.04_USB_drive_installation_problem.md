### Post by sirgogo on 2011-07-28
Use the Windows partition manager (System Administration/Disk Management). See if your hard drive is re-sized/there is another partition. If so, then your usb install at least partitioned your disk. Chances are your grub install failed. Plug your USB back in, boot to use Ubuntu (don't select install).

Then, follow this guide: [http://www.banym.de/linux/install-grub-bootloader-from-live-cd](http://www.banym.de/linux/install-grub-bootloader-from-live-cd) . There are far better guides out there to install grub via live cd, so Google should be useful.

### Post by sirgogo on 2011-07-28
I wouldn't install via Wubi unless you don't plan on using Ubuntu for very long. But... if you boot from a liveCD/USB, you can perform an fdisk -l and check all the partitions. You should also be able to mount these and browse the filesystem. If you can find a partition with readable folders/files with a non-Windows file-structure (Program Files, Users, etc), then your install probably went ok.

Grub is the default bootloader used by Ubuntu (and literally hundreds of other unix/linux distributions). It should be installed by default, but sometimes the install will fail. If this is the case, you need to re-install it somehow.

### Post by sirgogo on 2011-07-28
I'm not sure it works that way. Your Ubuntu install installs it to the physical disk by creating HDD partitions. I believe Wubi creates logical partitions for Windows. Wubi allows you to just remove the entire install data via windows by deleting a folder or uninstalling via the Add/Remove software utility.

I'm fairly certain installing Wubi without any modifications would create a completely separate install. If you still want to do it, though, you should open the Windows partition manager, delete all the extra partitions on your disk (not C: ) and re-size your disk to the max. Then you can proceed with a Wubi install (or clean Ubuntu install).
