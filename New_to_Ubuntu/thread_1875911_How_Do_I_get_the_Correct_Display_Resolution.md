On my main computer base unit, when I boot from an Ubuntu 11.10 CD and try Ubuntu it correctly identifies my display, an Iiyama 22", with the correct resolution 1680 x 1050.  I bought a second hand base unit with the intention of using it to 'play' with different Linux distros.  On this base unit when I use the same CD to try Ubuntu with the same display it never identifies the display, simply offering me a choice of two resolutions, 800 x 600 and 1024 x 768.  [/FONT]

If I am correct I assume I simply need to install a graphics card in second-hand computer to solve the problem but I hope that is not necessary.  The other specs of the base units are both different but I guess I really need to focus on the graphics hardware.  [/FONT]

### Post by doobyuk on 2011-11-05
The first time I have posted to a forum and I answer my own problem !  I was using a second VGA cable between my second hand base unit and the monitor so I tried swapping VGA cables and this time Ubuntu sensed the display make and correct display resolution as it did when I tried Ubuntu with my main base unit.  On checking the cables I found that the VGA cable I had been using had a pin missing at each end.  I never realised such cables existed - why make them with a pin missing ?
I'll carry on looking at Ubuntu - Kubuntu and Mint.
I hope at least this post might help somebody else who encounters a similar problem.
