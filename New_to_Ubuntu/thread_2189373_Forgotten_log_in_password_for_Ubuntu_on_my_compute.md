### Post by jdeca57 on 2013-11-22
The loss of a user password isn't a big problem as previous answers showed. You just need access to your system. Actually that's why the encrypted home file systems exist. That's the way to protect data on a laptop you might lose. Of course, if you lose the password to that encryption... 

The way I solved the loss of a password is by using a password manager like Keepassx and then put the data file on Dropbox. That way I can access passwords from any computer (my server runs Debian), even from a Windows PC. You do use difficult and different passwords, do you?

### Post by newb85 on 2013-11-22
As has been pointed out, if you didn't use encryption the psychocats tutorial is effective and easy-to-follow.  If you did use encryption, you're up a creek without a paddle.  (Otherwise, there would be no point to encryption...)

A word to the wise: in the future, *do* use a strong password and *don't* use automatic login.  In addition to using a supported release (why in the world did you install 10.10?), install updates to your system when they are issued, especially security updates.  They're there for a reason.

### Post by Impavidus on 2013-11-22
10.10 is unsupported. You can better use 12.04 LTS or 13.10. Upgrading would take 3 steps via intermediate releases which are end-of-life too, so you can better do a clean install of a supported version. If the hardware is older, try Xubuntu or Lubuntu. Note that Lubuntu 12.04 is not LTS and therefore unsupported. As this computer ran Win XP, iI think it may be quite old.

Switching to Ubuntu the hard way is quite a learning curve. Actually it's not that difficult, but you have to break free from the Windows mindset.
