### Post by unutbu on 2009-03-20
Let's see if we can finagle apt into downloading the file for you. (If worst comes to worst, I can post the file, but the method below is better -- if it works -- since it will allow you to fix this problem on your own.)


Save this in ~/bin/tmp-apt-setup
[php]
#!/bin/bash
# Usage:
# tmp-apt-setup "deb http://ppa.launchpad.net/gscrot/ppa/ubuntu intrepid main"
#
mkdir -p /tmp/apt
cd /tmp/apt
apt-config dump > /tmp/apt/apt.conf
sed -i 's|"/"|"/tmp/apt/"|' apt.conf
sed -i 's|/var|/tmp/apt/var|' apt.conf
mkdir -p /tmp/apt/var/lib/apt/lists/partial
mkdir -p /tmp/apt/var/cache/apt/archives/partial
mkdir -p /tmp/apt/var/lib/dpkg
touch /tmp/apt/var/lib/dpkg/status
mkdir -p /tmp/apt/etc/apt
echo "$1" > /tmp/apt/etc/apt/sources.list
apt-get --config-file apt.conf update[/php]

How the script works is explained here:
[http://ubuntuforums.org/showpost.php?p=6882449&postcount=43](http://ubuntuforums.org/showpost.php?p=6882449&postcount=43) 
Make it executable
```

chmod 755 ~/bin/tmp-apt-setup

```
Run 
```

tmp-apt-setup "deb-src [http://gpl.savoirfairelinux.net/pub/mirrors/ubuntu/](http://gpl.savoirfairelinux.net/pub/mirrors/ubuntu/) hardy main"
sudo apt-get --config-file /tmp/apt/apt.conf install apt-listchanges
```
You can change the repository to whatever you prefer.

### Post by unutbu on 2009-03-21
**To find all files installed by a package**: 
[list][*]In synaptic: 
[list][*]select the apt-listchanges package.
[*]Click "Properties" button (on the toolbar).
[*]Click the "Installed Files" tab. 
Note that this will only work for packages that you have already installed.
[/list]
[*]To find the equivalent information for a package that you haven't installed: go to [http://packages.ubuntu.com/](http://packages.ubuntu.com/), search for the package, and then click on the "list of files" link near the bottom of the page. (For example, look near the bottom of this page [http://packages.ubuntu.com/hardy/apt-listchanges](http://packages.ubuntu.com/hardy/apt-listchanges) for the link called "list of files")
[/list]
[B]
To fix the "duplicate cd problem"[/B]:
Type ```

cp /etc/apt/sources.list /etc/apt/sources.list-20090321  # make a backup to be safe
gksu gedit /etc/apt/sources.list                         # run gedit with root privileges

```
Look for lines that looks like this:
```

deb cdrom:[Ubuntu 8.04 _Hardy Heron_ - Release i386 (20080702.1)]/ hardy main
```

This line tells Synaptic to search the Hardy LiveCD for packages.
You probably do not want this to happen. So disable this line by adding a "#" sign at the front of the line:
```

# deb cdrom:[Ubuntu 8.04 _Hardy Heron_ - Release i386 (20080702.1)]/ hardy main
```

Do this for each line that starts with "deb cdrom".

Save, and exit gedit

Run 
```

sudo apt-get update
```

to see if this fixes the problem.

### Post by unutbu on 2009-03-30
cousinlucky, are you referring to the apt-listchanges.glade.txt file that I posted?
I've just downloaded it again and looked inside it. I can assure you, it has nothing to do with adblock.

This is a glade interface file. It is written in XML format. It is used by /usr/bin/apt-listchanges to create a GUI interface. It is not meant to be run by itself. Double-clicking it and asking that the file be run will produce weird results depending on what app you choose (or the computer chooses) to run it with. Perhaps firefox was selected to run the file, and adblock then got involved. 

Now that you have Synaptic/APT working again, the best thing for you to do is simply
reinstall the apt-listchanges package. This will ensure that the apt-listchanges.glade file is in its proper place with the proper permissions.

I don't know much about how to use apt-listchanges. I believe the way the Ubuntu package is set up, once you have the apt-listchanges package installed, you don't have to do anything. The next time you install a package, apt-listchanges will be automatically called and will display the changelog. See [http://www.debian.org/doc/manuals/apt-howto/ch-search.en.html](http://www.debian.org/doc/manuals/apt-howto/ch-search.en.html)

### Post by unutbu on 2009-03-30
The good news is I finally understand what the problem is. This is a known bug in Hardy: [https://bugs.launchpad.net/ubuntu/+source/apt-listchanges/+bug/202100](https://bugs.launchpad.net/ubuntu/+source/apt-listchanges/+bug/202100)

The bad news is, it appears this problem has not yet been fixed for Hardy, though it is fixed for Intrepid.

Do you really need/want apt-listchanges? Most Hardy users do not experience the error message you are experiencing, and I am guessing it is because they do not have apt-listchanges installed. Perhaps you can avoid the error message by uninstalling the apt-listchanges package.
