### Post by unutbu on 2008-07-19
While booted from the LiveCD, open a terminal and type 
```

sudo fdisk -l

```
You'll see stuff like this:

```
   Device Boot      Start         End      Blocks   Id  System
/dev/sda1               1           7       56196   de  Dell Utility
/dev/sda2               8         660     5245222+   b  W95 FAT32
/dev/sda3   *         661       38536   304238970   83  Linux
/dev/sda4           38537       38913     3028252+   5  Extended
/dev/sda5           38537       38913     3028221   82  Linux swap / Solaris
```

Find the line that ends with Linux. This line corresponds to your Linux partition. Note the device name (e.g. /dev/sda3 in the example above).

Then type (changing /dev/sda3 to your Linux partition's real device name in the command below):
```

sudo mkdir /Ubuntu               # creates a mount point
mount /dev/sda3 /Ubuntu          # mounts your linux partition at /Ubuntu
```

Then type

```
cat /Ubuntu/etc/X11/xorg.conf
ls -l /Ubuntu/etc/X11/xorg*

```
Please post the output of the above two commands.

The xorg.conf file configures the mouse. Posting it will allow us to see if there is anything there that needs fixing. 

The "ls -l /Ubuntu/etc/X11/xorg*" command lists if you have any other xorg.conf files.
Because you mentioned having used Hardy for a few months and only recently has the mouse stopped moving, I'm hoping there is a backup xorg.conf which may help us. If there is a backup xorg.conf, we may just be able to move it back to xorg.conf and restore your old configuration...

### Post by unutbu on 2008-07-20
Boot from the LiveCD.
Mount sda3 at /U like before.
```

cd /U/etc/X11/                          # change directory
cp xorg.conf xorg.conf.20080720.bad     # make a backup even though it does not work
cp xorg.conf.backup xorg.conf           # cp A B means copy file A to B

```

Then reboot. See if mouse works normally.
If not, repeat, going through the list of xorg.conf files until you find one that works.
The command  "ls -lt" will list the files sorted by modtimes.
You can open up the files with a text editor like gedit to take a look inside. You may be able to guess which one is best.

