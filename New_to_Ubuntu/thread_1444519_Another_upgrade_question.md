### Post by Don Bowen on 2010-04-01
As they say "If it ain't broke, don't fix it."
Working as a humble computer tech, people always expect that I will have the newest of everything!  Not so.  I prefer reliable to new, though I concede it is a good idea to be familiar with all the new stuff just in case someone asks.

Currently, I'm running Ubuntu 9.10, and it's set up to do all the things I need it to do.  Ubuntu 10.4 will be out in days, and after installing and playing with the "new", I'm not going to upgrade.  Why?

The decision to upgrade, for me, is usually a function of "is there something I can do ONLY with the new software?"  If this is true then it's time to upgrade.  10.4 doesn't do anything I can't already do (and the three min,max,close buttons are on the wrong side! Gasp!)

If you have a compelling reason to upgrade, do it, if not then be content.

### Post by indytim on 2010-04-01
There is no "correct" answer to your question.  It all depends on your tolerance for risk.  By not updating, you are not exposing yourself to patches (fix something that's not correct), security upgrades (address vulnerabilities) etc.  If you're certain your install is where you want to be, then assume the risk and leave it update free.

I'm going through a similar dialog with myself as I got partially borked on an update a couple of weeks ago.  For me, I'm going to do a new build and configure it to where I need it to be.  No updates or new apps on this build for at least 6 months as I need a totally stable production build.  I will do a second build with the same configuration but will allow updates and new apps & libraries.

### Post by Temposs on 2010-04-01
Don Bowen is correct.

However, the point at which you "should" upgrade is when the version of Ubuntu you're using ceases to be supported by Canonical's security updates. You want to keep your computer safe from malware, hacking, etc.

Ubuntu 8.04 has security update support until April 2011. So, you still have another year until you actually should upgrade.
