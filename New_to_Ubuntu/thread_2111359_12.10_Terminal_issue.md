### Post by themind on 2013-02-01
Completely new to Ubuntu here. I have a virtualized Ubuntu 12.10 server (with Gnome desktop) running, and an issue with a process. 
 
The process is called "dcd". I need to run "dcd -i" to start this process. My issue is this: If I login, open up Terminal and run "dcd -i", it tells me "Program not licensed" (which is a function of that program, if it's not licensed). However, it is not true that it's not licensed. If I telnet into this server from a remote computer, login as the same user, and run "dcd -i", it starts the program.
 
What is different between the local user account running Terminal, and remotely logging in as that user through Telnet, that locally it doesn't work?

### Post by zealibib slaughter on 2013-02-01
maybe its because the terminal in the gui isn't a login shell. from the menu bar for the terminal go to edit - preferences and check run command as login shell. see if that makes a difference.
