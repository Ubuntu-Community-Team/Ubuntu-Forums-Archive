### Post by 3rdalbum on 2010-01-03
Fragmentation-resistant filesystems cannot 'shred' individual files, because it's not guaranteed that the overwriting data will actually go in the same place as the original file.

There's two options for you. The first is to use home drive encryption. The Ubuntu installer can set this up for you. Without the decryption key, your data looks just like garbage, so anyone who gains access to your computer will not be able to access any of your data (including deleted data).

The second option is to use the 'dd' command to create a file of the exact same size as your disk's free space, after you have deleted the sensitive file. I'm not sure of the exact syntax for this, but it would involve something like this:

```
dd obs=<output file size in bytes> if=/dev/random of=/home/desukane/randomfile
```

See the man page for the 'dd' command before use. The random file will go into /home/desukane/randomfile, so delete it when it has been fully written.

I recommend the first option, of encrypting your /home partition. It's more secure.

Don't keep data that any law enforcement agencies would be interested in; they can read even "shredded" data from hard disks (yes, even with multiple passes) and in some countries you can be put in jail or presumed to be guilty for not giving up your decryption key when your hard disk is evidence in a court of law.
