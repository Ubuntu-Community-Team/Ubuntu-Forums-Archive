### Post by mikewhatever on 2010-03-23
Try **sudo update-grub**. According to Grub2 documentation, an os_prober script is run among the rest, which might pick up Windows and add it to the boot menu. If that doesn't work, you could add a custom entry to /etc/grub.d/40_custom, something like this:

```
### BEGIN /etc/grub.d/30_os-prober ###
menuentry "Microsoft Windows XP Home Edition (on /dev/sda1)" {
        insmod ntfs
        set root=(hd0,1)
        drivemap -s (hd0) ${root}
        chainloader +1
}

```

What did you mean by 'it did not allow me'? Was there an error message? Did it say 'You are not allowed'?:D

something like this 

```
grub-setup --force /dev/sda3
