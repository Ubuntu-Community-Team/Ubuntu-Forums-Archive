### Post by sotiris2 on 2015-05-29
Your line for the sda8 partition (214**) ends in 0 0 . The second number is the order in which fsck (filesystem check) is done on the partitions. A value of zero however means that a filesystem check is not performed. So maybe there is a small problem that would be fixed by fsck but is not due to a check not being run. It does after all remount sda8 which it should do if there is an error.

So backup your existing fstab via ```
sudo cp /etc/fstab /etc/fstab.bak
```
and run ```
sudo nano /etc/fstab
``` 
Then delete the last zero in the line starting with UUID=214.... and replace it with a 1. Press Ctrl+X to exit the program and Y to save changes. If you accidentally edit something else just do Ctrl+X and press N to not save changes. Then redo it correctly. 

Afterwards reboot and see if the problem persists.

### Post by cariboo on 2015-05-30
I see from the output, that you have both systemd and upstart installed. Bootchart shouldn't work, if you use systemd to boot, so you must be using upstart. I'd suggest removing systemd using the following command:

```
sudo apt-get purge systemd
```

then reboot.

@sotiris2  bootchart is in the repositories, and all it takes to install it, is:

```
sudo apt-get install bootchart
```

the output should be in /var/log/bootchart.

To create something similar using systemd, use the following command:

```
systemd-analyze plot > boot_plot.svg
```
