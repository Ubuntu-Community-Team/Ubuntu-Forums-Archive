### Post by caljohnsmith on 2009-05-08
I think it would help to first get a clearer picture of your setup related to your booting problem, so how about downloading the **Boot Info Script** to your Ubuntu desktop:

[https://sourceforge.net/projects/bootinfoscript/](https://sourceforge.net/projects/bootinfoscript/)

Then open a terminal (Applications > Accessories > Terminal) and do:
```
sudo bash ~/Desktop/boot_info_script*.sh
```
That will create a "RESULTS.txt" file in the same directory from where the script is run, namely your desktop; please copy/paste the contents of the RESULTS.txt file to your next post, highlight the copied text, and click the pound/hash sign "#" graphic button in the Ubuntu forum message box so that the text will get "code" tags put around it. The results of that script will help clarify your setup and hopefully what the solution to your booting problem might be.


-----------------------------------------------------------------------
[B]
Presence1960:[/B] Since I've noticed you have asked for "cat /boot/grub/menu.lst" from the Live CD in more than one thread, I think it's worth mentioning that doing that command won't work; that command will look for a menu.lst on the Live CD, which doesn't exist. You have to first mount the Ubuntu partition in question and then access its menu.lst. Or you can save yourself doing all the individual commands and let the Boot Info Script give you that information and lots more. :)
