### Post by OldGoat58 on 2009-11-29
[SIZE=3][FONT=Verdana]As a new Ubuntu user it can be frustrating using a new OS.  What makes it even a little more frustrating is when the install you used wasn't the best one for you.  Not completely understanding the difference between i386 and AMD64 Ubuntu packages I first burned and installed the AMD64 since I have AMD Turion 64 x2 2000 MHz processors.[/FONT][/SIZE]
[FONT=Verdana][SIZE=3]
I found an old thread on Ubuntu Forums[/SIZE][/FONT] [SIZE=3][FONT=Verdana]quite by accident I can assure you.  In that thread was the Pandora's Box I had been seeking.  I wish I had written the posters name down so that I could credit them.  It explained that the i386 and AMD64 tags given to the Ubuntu packages are generic in the fact that the i386 is a 32 bit package and the AMD64 is a 64 bit package.

The poster was using an HP laptop, like me, with AMD processors, like me, but he was using the i386 build because the 32 bit OS has more accessibility and plugins currently than the 64 bit OS.  This person is my god!  

I am happily up and running with a 32 bit build and the difference is night and day.  I still have a lot to learn what with the whole Linux command line interface thingy but I have nothing but time for learning.  It is important however for me, during the learning process, to be able to take a break,  relax and listen to music and play games on POGO.

Thank you to all the great people that supported me and gave me the advice that kept me playing around until it clicked.  I am really more confident than I was that changing to Ubuntu 9.10 Karmic Koala i386 is just what the doctor ordered!

Mike

The thread I found was i386 vs AMD64: Quick Question started by nomiK on February 9, 2007.  mikewhatever and Maestro23 were the 2 that unmuddied the water for me!
