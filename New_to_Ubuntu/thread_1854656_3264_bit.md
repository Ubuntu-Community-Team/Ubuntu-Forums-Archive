### Post by pjd99 on 2011-10-04
Been running 64-bit Ubuntu for the best part of 3 years. The only issue I remember having (architecture-wise) was Adobe Flash. But there is a 64 bit version now (beta) that seems about as stable as Flash can be on Linux (which is not very).

Nearly every open source program will have a 64-bit build that in real world terms runs no differently to its 32 bit counterpart.

Wine is a different beast, it's mostly an implementation of the 32-bit API, but has builds for 64 bit linux (so 32 bit Windows programs that run on 32 bit wine will also run on 64 bit wine). I don't know the status of the Wine 64-bit API implementation (or if it exists). This would only be a concern if you wanted to run a 64-bit windows executable in linux.


Go for 64-bit, you'll have access to all your RAM without requiring a PAE kernel.

yes I did some research too and it can work (mostly) and if doesnt work there is a work around downloading and installing 32 bit
 
> 
There are also a few programs out there that only made a package for 32 bit ubuntu… for example the [[COLOR=#027ac6]Amazon.com mp3 downloader[/COLOR]]("http://www.amazon.com/gp/dmusic/help/amd.html/ref=sv_dmusic_3") so how do you install this? Well fortunately it isn’t too difficult but does require some terminal “hacking”. To do this you need to open a terminal and navigate the location to the downloaded file now to install the mp3 download you would type:
sudo dpkg -i --force-all ./amazonmp3.debThe –force-all command will make dpkg ignore the package architecture and any missing libraries. Now go ahead and try to run it! That didn’t work did it? Now open a terminal window and type amazonmp3 to see more information. It complained about missing libraries right? That is because you are missing the 32 bit versions of those libraries, so now we need to head over to ubuntu forums and get a little package called [[COLOR=#027ac6]getlibs[/COLOR]]("http://ubuntuforums.org/showthread.php?t=474790"). Once this is installed you can open your terminal window and type the following:
sudo getlibs /usr/bin/amazonmp3For getlibs to work you need to provide the path to the binary executable of the program you want to run, which is why I put the path /usr/bin/ in there. Now it should have automatically installed all of the required libraries and everything should be good to go!
