### Post by coffeecat on 2014-08-02
Are you booting up the ISO into the live mode in a virtual machine, or have you installed a virtual machine of Ubuntu?

If the first, there is no password, but neither should you be prompted to login. If you are booting into a live session and are confronted with a login screen, then there is something wrong with the ISO and it needs to be discarded.

If you have installed Ubuntu, then the password is the one you set during the installation process. Only you would know that.

In any event, Ubuntu 12.10 is past end-of-life and is now unsupported. If you are having problems with this, you should abandon any attempts to solve it and download the currently-supported 14.04.1 version of Ubuntu and use that instead.
