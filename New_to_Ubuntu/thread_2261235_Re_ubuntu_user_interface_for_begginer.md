### Post by grahammechanical on 2015-01-17
Every User Interface is for a beginner. We all have to start somewhere. It takes but a little time to become familiar with any UI.

Some people prefer a cascading menu system similar to the one that I last used on Windows 98. But despite preferring that type of UI people also install a Dock. What is the Launcher in Unity but a dock? What is the Dash in Unity but a different way of presenting applications and files available for loading?

No one can advise you because you have your own tastes. The great thing about Free and Open Source Software (FOSS) is that we have freedom of choice without the burden of paying a price in money for each distribution we try.

Learn how to create partitions and to dual or triple boot operating systems. And then experience them all. I have an installation of every flavour of Ubuntu. But remember, we can easily become the kind of person that is never satisfied. It is not that difficult to find something we do not like in any distribution.

One more word of advice. Installing and removing alternative desktop user interfaces is the best way of making an operating system unusable. There are posts on this forum about that situation. I once tried experimenting with alternative user interfaces. I had to re-install to get decent looking desktop. This is why I recommend dual or triple booting.

Regards.

Excellent advice. All of it. But especially the part about mixing up desktops. This is NOT advised. Each DE comes with its own config files, its own defaults, its own profiles, yada-yada-yada. If you are lucky, you can get it all to hang together (barely), but just as often, it breaks the DE and you end up on this forum asking for help that cannot be given because you've turned your install into spaghetti.

As a new user especially, just don't go looking for that kind of trouble.

The good suggestions here are:

1. Run them as LiveDVD/USBs to give you a taste of their differences.
2. Install them on their own separate partitions to really dive into them in depth.

I will add a third:

3. Install them as Virtual Machines to give you the best of both worlds. They will have the same persistence as being on separate partitions, and be easier to use because you don't have to reboot each time to experiment. Moreover, you can have your nice little sandboxed distro chugging away inside a window while another window is open to your web browser looking up all of the arcane commands and config files that you are playing with. Most of all, if you do something silly, you can revert to an old snapshot and undo your blunder simply, quickly and painlessly. The only downsides are that it requires a relatively powerful computer, and you have to learn about Virtual Machines. If you have a lower-powered box, I would suggest either #1 or #2.

For what it's worth, my desires are the opposite of yours. I don't want my launcher cluttered with dozens of icons. It's cluttered enough already and, if anything, I want it simplified even further.

You begin to see how different folks require different strokes.

If you prefer KDE, then install KDE. If Cinnamon, then Cinnamon. If LXDE, or XFCE or Mate, then go for them. But this is a personal preference in the end and asking on a forum which is the "best" is a meaningless question, trying to objectify the subjective. Like asking what the best colour is, or the best flower.

BTW, using Unity, it is easy to cycle through not only all of your open apps, but each instance of those apps as well. You do so by foregoing the mouse and using <Alt>+<Tab> instead. This brings up a cycle box with all your running apps. By continuing to hold down the <Alt> key and pressing <Tab> successively, you can run through your apps sequentially. If you keep the focus on Firefox for about one second, it will then bring up a further sub-cycle box of all its open instances. You can then cycle through those web pages with further taps of the <Tab> key. After the last webpage, <Tab>bing exits the instance sub-cycle and the app cycle resumes. Since you have already decided to abandon Unity, this last is more for the benefit of any general reader who may happen upon this thread.
