The boot flag is currently set on sda1, which was probably your original Windows partition. For Windows to boot the flag will need to be set on sda4.

However, Windows will still not boot because the first 2 boot files are missing.
If you set the boot flag to sda4 (with gparted) then boot from a Windows Vista repair disc (or installation disc) you should be able to restore the boot files with the automatic repair function in the recovery console. This may need to be run 3 times.
Alternatively the Windows bootloader may be repaired by going to the command prompt option in the recovery console and running ```
bootrec.exe /fixboot
``` Note the space between exe and /fixboot

### Post by Quackers on 2011-05-26
A Windows 7 disc may be ok as the file systems are the same, but it needs to be of the same architecture (32 bit/64 bit) as your installed system.
The automatic repair can only fix one thing at a time, so may need to be run at least twice to fix 2 boot files.
The boot files are /bootmgr and /Boot/BCD

### Post by oldfred on 2011-05-26
If it was Vista you uninstalled then you do need to use the Windows 7 repair disk to fix it.

Windows only repairs somethings on each pass. It may make sense as some fixes depend on other things working first. Depending on what is wrong one pass then may not fix everything.

The files you are missing are.
/bootmgr /Boot/BCD

Windows always copies the boot files from a second install of Windows to the first install. Win7's standard install uses a 100MB boot/recovery partition for those same files so you can encrypt the main partition or run repairs to the main partition without a CD if you can boot the recovery partition. But you can install to one partition.

Just copying the files into the partition may or may not work. Either way the BCD has to be updated and often the boot sector has to be updated also.
