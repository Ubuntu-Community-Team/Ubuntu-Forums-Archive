### Post by itsols on 2010-11-25
With previous versions, I'd edit /etc/network/interfaces. I did the same with 10.10 to setup static IPs. But strangely, I kept receiving no packets from my LAN.

Then by using ifconfig, I realised that my IPs weren't shown and eventually I realised that my system was using ipv6 instead of ipv4.

So all I had to do was go through Administration > networking and ADD my IPs manually.  Thereafter, I realised that everything was good.

So my question is, why have the old interfaces file if it has no effect? It's pretty confusing.

Furthermore, I sometimes find myself back on some dynamic IP. How do I know? Well, when I try to ping a LAN PC, it cannot find it. So what I do is simply type sudo ifup eth0 in the terminal and it gets back to normal.

Is this a feature, flaw or am I missing something?

Thanks in advance for any advice!

### Post by pawhtiobo on 2010-11-25
Hi :)

I experienced a similar problem, i think that the etc/network/interfaces has no effect if you have Gnome Network Manager installed, because the settings are stored in /etc/NetworkManager/nm-system-settings.conf and /etc/NetworkManager/NetworkManager.conf, i think that's the problem of the messed up configurations.

[http://live.gnome.org/NetworkManager/SystemSettings](http://live.gnome.org/NetworkManager/SystemSettings)

See ya...

Trying to use the "old" method after Network Manager became default on an install almost drove me crazy too, at first. If you took it off your system, you'd be able to manually configure. But once you learn about nm you might come to like its features.

Edit: I suggest the best way to achieve static IPs, and I know how that can be useful in a LAN setting, is to use the router (or whichever server you use for DHCP) to asign static IP based on MAC of the interface.
