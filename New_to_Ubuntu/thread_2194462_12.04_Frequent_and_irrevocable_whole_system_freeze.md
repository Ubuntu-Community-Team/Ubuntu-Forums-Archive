### Post by Tristan_Williams on 2013-12-20
To upgrade your kernel follow these commands:
```

sudo apt-get update
sudo apt-get install linux-generic

```

Once that has completed, run the following commands:
```

sudo update-grub
sudo update-grub2

```

```

sudo reboot

```


Once rebooted, run the following command, and paste the output:
```

uname -r

```

### Post by Tristan_Williams on 2013-12-21
Sorry for taking so long Ashley.

I agree with owise1. It does sound like a hardware issue, but I'm not seeing anything out of sorts in your dmesg output.

First, let's try to successfully install the latest kernel.

1) Boot into safe mode.

Once the system is booted into safe mode, we need to try and install the latest kernel.

To install the latest kernel, run the following commands IN ORDER
```

sudo apt-get update
sudo apt-get upgrade

# If you have 32-bit system:
wget -c kernel.ubuntu.com/~kernel-ppa/mainline/v3.11-saucy/linux-headers-3.11.0-031100_3.11.0-031100.201309021735_all.deb
wget -c  kernel.ubuntu.com/~kernel-ppa/mainline/v3.11-saucy/linux-headers-3.11.0-031100-generic_3.11.0-031100.201309021735_i386.deb
wget -c kernel.ubuntu.com/~kernel-ppa/mainline/v3.11-saucy/linux-image-3.11.0-031100-generic_3.11.0-031100.201309021735_i386.deb
sudo dpkg -i linux-headers* linux-image*

# If you have 64-bit system
wget -c kernel.ubuntu.com/~kernel-ppa/mainline/v3.11-saucy/linux-headers-3.11.0-031100_3.11.0-031100.201309021735_all.deb
wget -c  kernel.ubuntu.com/~kernel-ppa/mainline/v3.11-saucy/linux-headers-3.11.0-031100-generic_3.11.0-031100.201309021735_amd64.deb
wget -c  kernel.ubuntu.com/~kernel-ppa/mainline/v3.11-saucy/linux-image-3.11.0-031100-generic_3.11.0-031100.201309021735_amd64.deb
sudo dpkg -i linux-headers* linux-image*

```

Once installed, to make sure Grub recognizes your new kernel, run the following two commands:
```

sudo update-grub
sudo update-grub2

```

Do this and we will work from there :)

### Post by DuckHook on 2013-12-23
In my opinion, you are poking and prodding, changing kernels and running about in circles needlessly. Every indication is that the problem is your power supply. In a number of old machines that I've wrestled with, power supply problems would manifest themselves in exactly the ways you describe. The fact that you could not even switch your PC on today just clinches it. The only other occasion with similar symptoms was a cracked MOBO.

There is likely (over 90% chance) nothing wrong with your actual OS installation, and any further fiddling with it will just destabilize it. What I would do is simply replace the power supply and see if the problem goes away.
