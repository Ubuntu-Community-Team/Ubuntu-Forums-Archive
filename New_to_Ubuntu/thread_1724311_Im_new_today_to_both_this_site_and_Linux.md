It really depends on what you expect from the machine, how you want to use it for. You see, Ubuntu, as well as most other popular linux versions (usually called distributions actually) are mature enough to give a satisfactory user experience comparable to what micro-windo offers; this includes surfing the net, editing documents, listening to music and watching videos. If that's all you're interested on, you can just install Ubuntu and use it (it would free you from the microsoft slavery, although a fundamentalist of the "free software foundation" will still claim that much of Ubuntu is still non-free software). The interface is pretty intuitive and doesn't need much explanation. After you've downloaded the ubuntu disk, burned it onto a cd and booted from the cdrom (see [https://help.ubuntu.com/community/BootFromCD](https://help.ubuntu.com/community/BootFromCD) and-or [https://help.ubuntu.com/10.10/installation-guide/i386/index.html](https://help.ubuntu.com/10.10/installation-guide/i386/index.html)), you can install it alongside windows and use it for your won ordinary work.

Linux is not something people should "learn", it is something peopole should "use" according to their needs (of course your need may be that you're actually interested in advanced aspects of the operative system such as kernel configuration, compilation and device driver development or linux/unix administration). The "learning" bit in linux is usually related to the fact that for much of the most powerful linux-specific software, actions are done by typing commands with the keyboards rather than pressing buttons with a mouse. The majority of computer users don't actually need this software unless they have to fix some sort of problem (say some issue related to playing video files).

As an advice, I personally think though that the most useful utility closely associated to Linux and goes beyond the usual "dumb user mouse clicking" is Bash. Bash is an alternative text-based way of interfacing with the system (any program in a linux system can be launched using a command typed in bash) and, once one familiarize with few commands, it immensely speeds up many common operations.

I hope this was useful...

### Post by Robing Goodfellow on 2011-04-09
MWSwhip,

Most everything I'm about to tell you has already been said, but methinks it is coming mostly from folk that are far more experienced than I at Ubuntu, so perhaps a "noob" perspective will help.

Up until December 10 I was a windows 64 bit user.  Hated it, decided to try a Linux distro, discovered Ubuntu.  As 10.10 (Maverick Meerkat)  was the latest and greatest, that's what I went with.

Most of us, like you, who are "interested in trying Linux" are reluctant to give up the supposed safety net of Windows.

Basic release concepts to understand about Ubuntu:
a.  there is a new release every six months
b.  the releases are numbers by year and month, so 10.04 was released in April 2010, 10.10 in October 2010, etc
c.  each release has a name as well as the number, 10.10 "Maverick  Meerkat", the upcoming 11.04 "Natty Narwhal"  in discussion, refer to  your distro (distribution or version) by the first name, for example,  "Maverick"
d.  There is an LTS (Long Term Support) every 2 years, 10.04 was the last one.  These are supported  for (I think) 3 years.
e.  The "in-between" or standard releases are supported for (I think) 18 months

When they tell you to run LiveCD, what they mean is

1.  **Download the ISO for your choice of Ubuntu from the website.**  I personally recommend Maverick, but that is because I started with Maverick and that's what I'm comfortable with.  I tried Natty (11.04b) but didn't like it, though plan to try it again when it's formally released at the end of the month.

2.  **Burn an ISO copy onto disk** in theory you can do this to a stick as well, I've just always used disks

3.  **Reboot your machine, upon startup, go into BIOS and tell your machine to boot from CD drive (or stick if you went that route)**

4.  **From the opening screen, there are 2 choices: Try Ubuntu or Install Ubuntu.**  Personally, I recommend install Ubuntu.  It will allow you to set up a dual boot system.  Using GRUB (Grand Unified Bootloader) when you start your machine, right after the BIOS screen, you get GRUB which allows you to boot either Ubuntu or Windows, so dual boot takes nothing away from you.  As recommended earlier though, make sure you back up all important files.  I've never had a problem, having done 3 or 4 fresh installs, but you never know.

If you use Try Ubuntu, you run the OS from disk, works great,, but slow.

Presuming you have installed Ubuntu, play with it.  "Applications" holds all your programs, and the Ubuntu Software Center, which allows you to very easily add new programs from the USC, all of which have been tested and, well, kind of approved.  If you want programs that aren't in the software center, there's System, Administration, Synaptic.

Start playing with the terminal.  There's lots of tutorials and guides, pick one that suites you.  It is an incredibly powerful tool that is pretty easy to learn if you are at all computer literate.

Still want more?  Web searches for how to install anything you want yield tons of info.  I personally like Ubuntu Geek's site.  Clear, concise, easy to follow.

THE ABSOLUTELY VERY BEST RESOURCE is the one you are using at this very moment.  You will not find a more helpful bunch of folk, nor more knowledgeable.  Just make sure before you ask a question in the forums, you've done a little research.  Search the forums to see if your question has already been asked, it probably has.  If not, post away, just be concise, include your machine, which Ubuntu you are using, what you've done already and a clear presentation of what's wrong or what you are trying to do.  Learn how to take a screen shot, learn how to use the terminal so you know how to perform recommended commands (both very easy to do).

If you are like most of us you will bounce back and forth between Windows and Ubuntu for a week or two, then in a month or two you will find yourself wondering why you keep windows on your machine at all, and just get rid of it.

Hope this helps, feel free to keep asking more questions.

regards, Richard

### Post by Robing Goodfellow on 2011-04-12
**  Upgrading Jaunty or Karmic to Lucid **

 
[LIST]
[*]Also see the [official Ubuntu desktop upgrade documentation]("http://www.ubuntu.com/getubuntu/upgrading#Network%20Upgrade%20for%20Ubuntu%20Desktops%20%28Recommended%29").
[/LIST]
 There are several methods for upgrades from the command-line  interface (Terminal) (which can be used for both the desktop and server  editions of Ubuntu/Kubuntu). 
 
[LIST]
[*]This is the preferred method:
[/LIST]
 ```
sudo apt-get install update-manager-core sudo do-release-upgrade 
``` 
[LIST]
[*]You can also use the update-manager (all editions):
[/LIST]
 ```
sudo apt-get install update-manager sudo update-manager -d 
``` 
[LIST]
[*]You can also use:
[/LIST]
 ```
sudo apt-get update sudo apt-get upgrade sudo apt-get dist-upgrade
```  (Note: the first two lines simply make sure your current  distribution is current before upgrading the entire distribution, and  are optional.

Thank you kindly for all your contributions and suggestions.I find something a bit troubling in this post from you. Nowhere in the thread, up to here, has anyone mentioned that there are two different ways to install Ubuntu (actually more than two, but just two that you would be likely to have used) and they produce drastically different situations. I'm referring to "wubi" and the standard dual-boot technique. The "wubi" method creates a disk file inside of Windows, then treats that file as a separate hard disk and installs Ubuntu on it. The other method re-partitions the actual hard drive to gain space, and installs Ubuntu alongside Windows.

Because of this difference, uninstalling has to be done differently for the two situations, and what you describe makes me suspect that you installed 9.04 using the "wubi" method. If so, you should find an entry for "Ubuntu" within your Windows add/remove programs application. From there, just click "Remove" and it should go away cleanly. You can then try or install a newer version.

You can quickly tell which method was used, from the boot time menu that you get. If it lists Windows as the first entry, it's a wubi installation and is using the Windows "boot.ini" file to generate the menu. If Windows is the bottom-most entry, it's a normal installation and the menu is coming from the GRand Unified Bootloader (grub).

Hope this helps. By all means, run from the live CD or USB stick initially, if for no other reason than to make sure that all of your hardware works as expected. And welcome! You're never too old to learn new tricks; I've been in computers since 1965, and am still learning...
