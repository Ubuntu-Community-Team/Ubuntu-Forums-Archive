### Post by GMachine_24 on 2008-12-20
[B]I'm putting a caveat in here: Although doing what it says below will allow you to play Real Media files from the BBC and some other sites with the Real Player plugin, it doesn't work on every site and I still get Totem player attempting to play Real Media sometimes (try the real media live stream at [url]www.kuow.org/url]
as an example). Perhaps someone else can advance this further.[/B]

Ok, after plugging away at this on and off all day I finally found out what to do to make RealPlayer work from the BBC. It turns out that Firefox uses Totem by default to play Real Media files - or it at least attempts to, but it can't, so you must remove the following files from usr/lib/totem/gstreamer:

```

cd /usr/lib/totem/gstreamer


```

The two problem files are those that begin with libtotem-complex-* so, before deleting them, back up each file into the same directory.

```


sudo cp libtotem-complex-plugin.so libtotem-complex-plugin.so.old

sudo cp libtotem-complex-plugin.xpt libtotem-complex-plugin.xpt.old


```

You will be prompted for your root password; enter it after each line to make a back up copy of the two files.

Now you want to delete the two libtotem-complex files that aren't the back up copies. So

```


sudo rm libtotem-complex-plugin.so

sudo rm libtotem-complex-plugin.xpt


```

After you have removed these two files, point your Firefox browser to the BBC and attempt to listen again. The Real Audio plugin should start on its own and begin playing BBC content (I have tried it and it works).

This should fix your problem, at least for the BBC and some other Web sites.


--GM

### Post by GMachine_24 on 2008-12-23
I finally got Real Player to work as the default for all Real audio files on the Internet.

Here are the steps to take:

```


How to install RealPlayer in Ubuntu 8.0.4 and play RealPlayer video in Firefox 3.0.
1. Download RealPlayer

    * Go to [http://www.real.com/linux](http://www.real.com/linux) and download the RealPlayer11Gold.bin file.

2. Make RealPlayer11Gold.bin file into an executable file.

    * Open a Terminal window.
    * Navigate to the directory that contains the bin file.  For example:
          o

            cd download

    * Make the bin file into an executable file.
          o

            chmod +x RealPlayer11Gold.bin

3. Run the bin file

    *

      sudo ./RealPlayer11Gold.bin

    * Accept the default values by pressing the Enter key.
    * By default, RealPlayer will be installed in /opt/real/RealPlayer

4. Setup RealPlayer

    * In the &#8220;Application&#8221; menu, select &#8220;Sounds & Video | RealPlayer 11&#8243;
    * It will display the RealPlayer Setup Assistant.  Just follow the setup instructions.

5. In Firefox, go to a website and try to play a .ram file.

    * If you don&#8217;t know a website, you can skip this and go to the next step.
    * After it downloaded the .ram file, Firefox will ask you what program to use to open the file.  The default selection is &#8220;/opt/real/RealPlayer/realplay&#8221;.  Enable the checkbox: &#8220;Do this automatically for files like this from now on&#8221;.  This will add a new Content Type to FireFox&#8217;s settings.  We&#8217;ll need to change it later.  Even though I told it to use RealPlayer, Firefox will use the Totem Movie Player.  So we have to manually configure it in Firefox.
    * Open the Firefox Preferences Dialog.  Select the &#8220;Applications&#8221; button at the top.  In the &#8220;Content Type&#8221; column, find the RealPlayer content type.  It should have something like &#8220;RA file&#8221; or &#8220;RealPlayer document&#8221;.  In the &#8220;Action&#8221; column, select &#8220;Use other&#8230;&#8221;.  Browse to select the RealPlayer program which is &#8220;/opt/real/RealPlayer/realplay&#8221;.  After the program is specified, it should display &#8220;Use realplay&#8221; in the &#8220;Action column&#8221;.  Close the Preferences Dialog.
    * Try to play the same .ram file again.  It should automatically use RealPlayer to play the video.  If it doesn&#8217;t work, restart FireFox.

6. Setup Firefox to play embedded RealPlayer file.

    * Type &#8220;about:plugins&#8221; in the Firefox address bar.  Find the RealPlayer plugin.  Note that it uses Totem player.
          o

            Helix DNA Plugin: RealPlayer G2 Plug-In Compatible (compatible; Totem)
               File name: libtotem-complex-plugin.so
               The Totem 2.22.1 plugin handles video and audio streams.

    * I don&#8217;t know why the RealPlayer setup program didn&#8217;t install the RealPlayer plugin in Firefox.   Since we don&#8217;t want to use Totem player, we have to remove the plugin.
    * Find where the Totem RealPlayer plugin is located.
          o Type &#8220;about:config&#8221; in the Firefox address bar.  Go ahead and agree that you&#8217;ll be careful.
          o Type &#8220;plugin&#8221; in the &#8220;Filter&#8221; field.  Find the &#8220;plugin.expose_full_path&#8221; preference.  Right click on the entry and select &#8220;Toggle&#8221; to change the value from &#8220;false&#8221; to &#8220;true&#8221;.
          o Type &#8220;about:plugins&#8221; in the Firefox address bar again.  Find the RealPlayer plugin.  It should display the fullpath of the plugin.
          o For example:
                +

                  File name: /usr/lib/totem/gstreamer/libtotem-complex-plugin.so

    * Remove the Totem RealPlayer plugin.
          o Open a Terminal window.  Go to the Totem plugin directory.
                +

                  cd /usr/lib/totem/gstreamer

          o Remove the plugins.
                +

                  sudo rm libtotem-complex-plugin.*

          o Restart Firefox.  Type &#8220;about:plugins&#8221; in the Firefox address bar.  Make sure there is no RealPlayer plugin.

    * Copy RealPlayer plugin to Firefox.
          o Copy the RealPlayer plugins into the Firefox plugins directory.  For me, Firefox is installed in /usr/lib/firefox-3.0 and RealPlayer is installed in /opt/real/RealPlayer.
                +

                  sudo cp /opt/real/RealPlayer/mozilla/*.* /usr/lib/firefox-3.0/plugins

          o Restart Firefox.  Type &#8220;about:plugins&#8221; in the Firefox address bar.  Make sure RealPlayer plugin is installed.
                +

                  Helix DNA Plugin: RealPlayer G2 Plug-In Compatible
                      File name: /usr/lib/firefox-addons/plugins/nphelix.so
                      Helix DNA Plugin: RealPlayer G2 Plug-In Compatible version 0.4.0.4005

    * Test if the plugin works or not. Go to a website that has embedded RealPlayer file.  For example


```

I think I know why the realplay plugin fails to load itself in hardy. There are two directories next to each other on the /usr/lib tree - firefox and firefox-3.0.5 and realplay not unreasonably loads its plugin in the first  of these whereas firefox looks  in the second! So you don't really need to go searching out the original installation files to rectify - just copy from the redundant directory to the active one [whatever that may be]:

```

sudo cp /usr/lib/firefox/plugins/*.* /usr/lib/firefox-3.0.5/plugins

```

Of course, you  still need to disable the totem plugin as detailed in the previous response.

It seems a bit strange to have a firefox-3.0.5 directory labelled in this way - what happens when a firefox upgrade is issued and the number changes?

**I can now partly answer my own question - at firefox update, the directory is renamed with the new version no. So, as of 11 Feb 09 (assuming you've kept up with updates) the relevant directory will be /usr/lib/firefox3.0.6 until the next update and that would be where to place the plugin files in the plugin sub-directory.**
