### Post by directhex on 2009-02-19
Some details, to help clarify matters:

Moonlight 1.0 does not support DRM. It cannot - Silverlight 1.0 does not support DRM either.

Moonlight/Silverlight 1.0 consists of three things: a XAML renderer (XAML is an SVG-like markup language), DRM-free codecs & media playback, and a logic engine - in the case of 1.0, that logic engine is Javascript based. So all the plugin needs to provide is the XAML rendering and the codec functionality (with your web browser providing its own JS engine). In Moonlight 1.0, those codecs can either come in the form of a licensed closed-source cost-free blob from Microsoft, or by compiling the plugin against libavcodec. If you don't use the binary codecs ("Microsoft Media Pack") then it's completely Free Software - no really, take a peek at the source. Moonlight 1.0 is available in Jaunty, compiled against libavcodec.

Moonlight/Silverlight 2.0 consists of several things: a XAML renderer, codecs & media playback (including optional DRM), a widget toolkit (like GTK), a logic engine (based on .NET). The XAML renderer needs tweaking for 2.0, but they at least have a base for it, and most of the widgets are released by MS under a Free Software license. The logic engine is major work, but they can leverage a lot of the existing Mono work to speed it along. DRM... DRM will never work in libavcodec. If someone makes it happen (a little like how libdvdread can load the DRM-cracking libdvdcss) then perhaps, but as it stands, the only way you'll be able to watch DRM content is with an updated Microsoft Media Pack
