### Post by skymera on 2009-05-04
My friend fixed his by adding

```
 iwconfig wlan0 rate 5.5M fixed
```

to his /etc/rc.local

To make it effective [code] sudo iwconfig wlan0 rate 5.5M /code]

### Post by retired_techie on 2009-07-29
I have a Gateway lap top model MT6728 with the Realtek infamous rtl8187b wireless chip set. I have struggled with the wireless and Ubuntu since 7.04
I got it to work with the ndiswrapper under Ubuntu 8.10 
When I upgraded to Ubuntu 9.04 Jaunty Jackalope my wireless would not work with Ubuntu. I have read many posts which claimed that the rtl8187b was working with native Ubuntu and should be working with 9.04 which for me was not working.
On July 28, 2009 I ran my update manager as I usually do and after it loaded a new kernel and rebooted my machine. To my surprise a new screen appeared which asked for my ringkey which turns out to be my WPA password. I now enter my WPA password at startup when it asks for the ringkey and my wireless is working.
I launch the task manager from the System pull down menu. 
Under the System tab of the Task Manager is the following information:
Release 9.04 (Jaunty)
The Kernel Linus 2.6.28-14-generic
GNOME 2.26.1

I wish to thank the developers that finally fixed the rtl8187b wireless problem.

Hope this helps someone someday.
