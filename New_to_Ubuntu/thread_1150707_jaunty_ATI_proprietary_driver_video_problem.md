Boot to the recovery menu and arrow down to the root prompt option and hit enter. 

```
aticonfig --initial -f
```

Reboot.

If that doesn't fix the problem there are some things you need to know.

1. Jaunty has X.org server 1.6
2. The only version of fglrx available that is compatible with X.org server 1.6 is the ATI Catalyst 9.4 version.
3. ATI dropped support for a bunch or cards in the Catalyst 9.4 version.
4. If yours is one of the dropped cards then you can't run the ATI proprietary driver with jaunty. You will have to use either one or other of the OSS drivers ati or radeonhd. 
5. There have already been heaps of threads on this subject. 

More info:

[http://www.phoronix.com/scan.php?page=article&item=amd_r500_legacy&num=1 ]("http://www.phoronix.com/scan.php?page=article&item=amd_r500_legacy&num=1")

### Post by ubuntumaybe on 2009-05-13
Hi,

Problem fixed. I followed some of the directions in the link below. I removed all references to fglrx using the command
sudo apt-get remove --purge xorg-driver-fglrx.

I added the following line to the xorg.conf file
Driver          "ati"

Rebooted and it is now working. I can live without compiz.
