### Post by SecretCode on 2011-08-20
I get different results ... although with different combinations of versions ...

With LibreOffice 3.4.1 on WinXP, saving as .xls and reopening loses the borders. I doubt that LibreOffice has different code for WinXP and Win7.

There IS a problem with .xlsx: both with LibreOffice 3.4.1 on WinXP and with OpenOffice 3.2 on Ubuntu 10.10, saving as .xlsx and reopening messes up (moves) the merged cells - worse than a formatting change.

With OpenOffice 3.2 on Ubuntu 10.10, saving as .xls and reopening preserving borders and merged cells fine. And this saved file opens fine in Excel 2003 on WinXP.

I think you need to take this to LibreOffice support forums. There are a few, and I can never decide which is best, but [Index page &#8226; OpenOffice.org Community Forum](http://user.services.openoffice.org/en/forum/) is the busiest (and explicitly covers LibreOffice).
