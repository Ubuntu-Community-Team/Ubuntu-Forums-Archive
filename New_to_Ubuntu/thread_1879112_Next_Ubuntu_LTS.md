### Post by VeeDubb on 2011-11-11
> **overcast said:**
> One important question, which practice is better ? 

In this instance, "better" is a relative term.  Which one is 'better' will depend on your particular needs and wants.

If your main concern is stability and security above all else, stick with the LTS version, and only install updates for security and bug fixes.  When ubuntu stops releasing security and bugfix updates for the LTS you're using, back up your files, format your hard drive, and install the new LTS.

If your main concern is being up to date with the newest features and programs above all else, run the latest testing version and install every update that comes.

There is no right or wrong answer.  It's a matter of what you want and need.

### Post by 3rdalbum on 2011-11-11
> **overcast said:**
> One important question, which practice is better ?

or 

2) Upgrading ubuntu only with security and package updates and reinstall when new LTS releases ?

If we go with 1) how long will it take to make system less usable ?

It depends entirely on what you're doing with your computer.

If you don't like change, or you're using your computer for mission-critical or productivity-critical work, then stay with an LTS. If you like shiny new features, then you'll want to keep your system up-to-date with the regular releases.

Note that you can upgrade directly from one LTS to the next. If you have a 10.04 system you can go straight to 12.04 when it is released. Or you can fresh install, it's up to you.

Your question about "how long will it take to make the system less stable" doesn't really seem to make sense. Normal releases are a little less reliable than LTS releases, but not by much. Most of the time when Linux distributors refer to "stable", they actually mean "the software won't change drastically" rather than "the system won't crash" - although usually "stable" systems tend to be reliable too for a number of reasons.
