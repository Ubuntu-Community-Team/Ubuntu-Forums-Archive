### Post by Immolatus on 2012-01-23
black screen usually means the monitor resolution selected in xorg.conf is "out of range". you can fix this manually by adding alternate resolutions to the config file. When it boots to the black screen just do: ctrl + alt + F1. now you'll be on tty 1 and use nano to edit from there. If you are confused about how to handle the config file see the xorg wiki.

### Post by Sigma1 on 2012-01-23
Yes. And from the change in resolution and size of the boot screen, it looks like you might have installed some new video drivers which have messed the rest up. I've had this happen with suggested proprietary drivers (and without)!
Here's the command to reconfigure xorg: sudo dpkg-reconfigure -phigh xserver-xorg You can use this from tty 1, i.e. after having pressed Ctrl + Alt + F1.
If you can remember anything about changes in video drivers, you would do well to purge them too, I would say.
