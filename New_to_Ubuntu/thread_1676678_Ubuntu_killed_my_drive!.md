### Post by oxoxo on 2011-01-27
I run Windows XP Professional SP3 (NTFS file system) and have never before used Ubuntu or Linux. After losing the use of my mouse I uninstalled it and rebooted without success and then tried different types of mice that were tested on other computers. I then sought to determine whether my problem was software or hardware-based by using an Ubuntu Live 8.04.1 Desktop disc that I downloaded a couple of years ago but never used, to see whether my mouse would work on a different OS installation (it did). 
It seemed like a good idea at the time. I expected that Ubuntu would install on a separate partition, like a virtual drive or machine, particularly because there was no warning that I was about to reformat my drive or lose data. However, when I rebooted into Windows I was shocked to find that the drive onto which I installed Ubuntu has disappeared from my system! I started with a moderate problem but have now added a major problem, because the missing drive contained much important data, such as my email store folder, Bookmarks, Desktop, a large collection of media files and My Documents folder (along with many important documents including my only record of all my passwords). I recently bought an external drive with enough capacity to back up my entire system, but when I lost my mouse I (unwisely as it turns out) chose to concentrate on solving that problem before doing the backup. I removed the drive and connected it to another computer, but it was still undetected. 

Is there any way I can recover my lost data, please?

### Post by mharrison on 2011-01-27
My first question is did you actually run through the Ubuntu installation or did you allow the LiveCD to boot and run Ubuntu that way? 

If you allowed the LiveCD to boot and load the OS that way, your drive should not have been touched.  Unless I am mistaken, if you performed and installation instead, it would have prompted you as to what you wanted to do partion wise so you would have had to make a clear cut decision as to how to partition your drive.

### Post by Bölvaður on 2011-01-27
I have used [http://www.cgsecurity.org/wiki/TestDisk](http://www.cgsecurity.org/wiki/TestDisk) with success in the past. There are also other recovery programs that have GUI, but this is a good program which you should try. Just follow the instructions on the website.

*added*
this recovery program only tries to find data on the partitions or hdd and saves it in a temp folder, but does not recover the partition in it's former state.. perhaps it does, but I didnt remember reading about that when I used it.
