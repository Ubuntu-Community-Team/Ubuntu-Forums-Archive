I don't see anything strange there.

Applications use the amount of RAM they need, and the rest of the RAM will be used as cache. The cache will grow to fill the unused part of memory over time as you access different files on the system, so it's not supposed to use any certain percentage of your memory or anyhting like that, and it won't fill the fre part of RAM untill you have actually accessed enough data to do that.

Not using swap is definitely a *good thing*. Swapping is not something you'd ever want to happen, it's somethign that is only done if you run out of memory available for programs to use. Accessing data from your hard drive takes thousands of times longer (and I'm not exaggerating here) than accessing data from RAM does, so when the system needs to use swap you can be sure things are not running smoothly any more. So, swapping is more like a trick that allows the system to run even if it has too little RAM to do the tasks you try to do. It's not soemthing you'd *want* to do, and it definitely doesn't increase the performance.

Really, I  think this is the first time ever I've seen somebody beng concerned about the system using too little resources... You could buy some older computer to fix that... ;)

(by the way, you can't compare memory and CPU usage between different systems as percentages. It makes no sense, different computers have different amounts of RAM, and different processors. A task that requires 80% of CPU power on one system can require 5% on another system with a different processor)
