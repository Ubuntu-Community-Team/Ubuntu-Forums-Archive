### Post by smizzle on 2017-06-16
**SOLVED!!!!**:o:KS

_I used_: (got make commands from the makefile I had, may vary for anyone else so check your makefile).
```

sudo make clean
sudo make clean1
sudo make uninstall

```
in the makefile directory of the seemly **outdated driver** I managed to install that actually had a **kernel compatibility** for 2.2 -2.4. I had 4.4.0-79.

After that I _reinstalled the images for kernel and kernel-extra_ to replace the missing drivers that I had just removed:
```

[COLOR=#000000][FONT=&amp]sudo apt-get install --reinstall linux-image-`uname -r`
[/FONT][/COLOR]
[COLOR=#000000][FONT=&amp]sudo apt-get install --reinstall linux-image-extra-`uname -r`

reboot
[/FONT][/COLOR]
``` 
Then I _rebooted_ to **finally hear **the Ubuntu startup sound!

Lessons learned: Read **all** readme's in a driver package, don't be too picky when it comes to your microphone.

In total it took about **10 hours** to find the solution because i'm a newbie.
