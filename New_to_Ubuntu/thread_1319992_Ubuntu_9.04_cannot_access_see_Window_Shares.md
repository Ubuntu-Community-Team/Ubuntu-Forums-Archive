### Post by Prizm1 on 2009-11-08
Using the old 'Setup Sambo peer-to-peer with Windows' post found in this forum, I was able to one-way network my WindowsXP SP3 computer with my Ubuntu 9.04 computer. Windows maps to the Ubuntu folder as designated in smb.conf. But, Ubuntu cannot find any of the Windows shares. I did not have this problem with Ubuntu 8.10. I am not sure if this is a problem with 9.04 or due to any subsequent Windows security updates since 8.10. With the new 9.10 Ubuntu version, the 'Setup Sambo peer-to-peer with Windows' procedure does not network my Windows and Ubuntu 9.10 at all. I went back to v9.04 to have some type of networking between the two computers although it is only a one-way networking. I find it so difficult to understand how new versions of Ubuntu can just appear without some information provided by the developers as to how to network the new version with other popular operating systems This situation forced me to go back to an older version of Ubuntu in order to maintain some networking capability between the two networked operating systems. It would be great if there were only Linux computers in the world to network together, but that is not realistic.
