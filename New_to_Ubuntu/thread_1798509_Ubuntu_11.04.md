### Post by seawolf167 on 2011-07-06
x86_64 can address more RAM, so if you have 4+ GB of RAM you'll want it, also, because of the registers, it can execute instructions faster. The package list for x64_86 and x86 are almost identical in count, and there are really no outstanding issues that need to be dealt with that you will run into.

IMO, if you have an x64_86 capable machine, use it.

### Post by mastablasta on 2011-07-06
**PC (Intel x86) desktop CD **
For almost all PCs. This includes most machines with Intel/AMD/etc type processors and almost all computers that run Microsoft Windows, as well as newer Apple Macintosh systems based on Intel processors. Choose this if you are at all unsure.
**64-bit PC (AMD64) desktop CD **
Choose this to take full advantage of computers based on the AMD64 or EM64T architecture (e.g., Athlon64, Opteron, EM64T Xeon, Core 2). If you have a non-64-bit processor made by AMD, or if you need full support for 32-bit code, use the Intel x86 images instead.

### Post by walt.smith1960 on 2011-07-06
Proprietary hardware drivers *may* be an issue.  For instance, Brother publishes Linux drivers for most of its printers & MFDs.  There are no 64 bit versions.  The drivers work on 64 bit systems but install must be done from the command line using a -forceall function as documented on Brother's Linux web site.  That's the only issue I've run into and it's a minor inconvenience.

### Post by Mark Phelps on 2011-07-06
The first is for PCs with 64-bit processors; the second is for 32-bit (but will work on 64-bit).

If you have a 64-bit processor, the first is probably the best choice -- but only if you have more than 4GB of RAM.

If you have 4GB or less, you can still use the 64-bit version, but you won't see any performance improvement over the 32-bit version.
