There are indeed two grub installs. That is not a problem as such, although it is untidy. My machine has both a redundant grub legacy and a Grub2 on it at various places, but it happily boots from the Grub 2 that the BIOS is pointed at.

```to find the paritions that it should find a bootable OS on.

### Post by meierfra. on 2010-02-18
I don't think restoring the Windows MBR as suggested by audiomick will solve your problem, but it will improve your situation:
If  you set your bios to boot from the  Windows drive, you will boot directly into Windows. 
So if you also configure grub to boot  directly into Ubuntu, you can  choose your OS via the bbs pop menu  and no other boot menu would be involved. That is not a perfect solution, but  better than your current setup.


It seems that Grub is only able to see the boot drive and so is unable to boot the OS on the non boot drive. To confirm this. Press "c" at the grub menu  at boot up and type

```
ls
```

Post the output of the command.

The problem could be caused by your Bios or by a bug in Grub 2.  So there are few things one can try to fix it:

1)  Check on the setting in the Bios (but make sure to write down the original setting of anything you change)

2)  Revert to Legacy Grub.

3) Check whether there is an update available for your Bios.

4) Create a boot partition for Ubuntu on the Windows drive.

5) Just use the work around suggested above and boot the OS's via the bbs pop menu.

Let us know if you need help implementing any of those suggestions.
