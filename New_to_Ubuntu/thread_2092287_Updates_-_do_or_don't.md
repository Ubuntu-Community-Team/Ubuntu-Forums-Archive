> **natchezjohn said:**
> "When was the last time you installed updates?... You are probably correct that these are a compilation of all of the updates till support was withdrawn.
Yes, I think this is likely. As far as I know, the repositories are still there. They just don't get any new stuff. If the package management sees that what you have installed is not as new as what is in any repositories it can find, it will want to update.

> **snowpine said:**
> You can't be assured the updates "won't cause problems" [COLOR="RoyalBlue"]because[/COLOR] 10.10 is end-of-life and completely unsupported. Continuing to use it is asking for problems.

That "because" is misleading. It is true that there can be no absolute assurance that an update wont cause problems. Mostly they don't, but occasionally they do. This has nothing to do with the version being end of life.

That the version is end of life means that there is no further work done on it. Any security problems it may have will not be fixed. Any bugs that may be there will not be fixed. Therefore, continuing to use it means giving up the potential added security of using a version that is being maintained with current bug fixes and security updates. The end of life version will never work any better than it  did on the day it became end of life, but it will never work any worse either. None of that has any relation to the fact that updates occasionally cause problems.

I would add my recommendation to have a look at Xubuntu, or maybe just add a different desktop environment to a standard Ubuntu install. Whichever way you go, there is something to be said for staying on a currently maintained version.
