### Post by Dedoimedo on 2008-09-02
Hello,

In general, there is no reason NOT to download everything. Some fixes are security updates, others are usability patches that generally improve the quality of the relevant applications.

Running a manual check can be tedious, as you may receive 100+ updates, plus some applications depend on others. All in all, it makes for a very difficult Windows-style update management.

Simply allow everything to update, and you'll enjoy a system wide update policy.

### Post by keith.eckstein on 2008-09-02
**aiko.uda** - I make a point of downloading everything that comes up in Update Manager every morning - it's part of my daily routine now.  Most of it is stuff that I don't use or won't need but... I'm no expert and not capable of really judging that - thus I download it all.

It's not really a problem for me (only 1Mb connection) unless I re-install the OS - then it can take half an hour or so.

Can't really understand why you wouldn't want to update your system on a daily basis - my experience (18 months under Ubuntu) is that an update has never trashed my system - that can't be said for another (vastly inferior but far more popular) OS!

Take my advice and download it all.  I don't currently use SSH but, if I ever do, I know that it'll be up to date and not suffer from the recent security issues!

Unless you are on dial-up - just get into the habit of constantly updating your system.  If you are on dial-up, find some time each day/week to make it happen.

### Post by Paqman on 2008-09-02
You might as well download everything, since the updates will include bug fixes.

If you're worried about chewing up your download limit, then you can limit the updates to only include security updates. You should definitely always install ALL security updates. Go to System > Admin > Software Sources > Updates and pick which ones you want to include.
