### Post by sonicb00m on 2008-07-25
No one linked to google. They linked you to the application database that tells if Visual Studio 2008 works or not. The problem with wine is that a simple yes/no answer doesn't suffice. It probably won't work if you just install wine and try to setup VS without a lot of extra packages installed from something like winetricks/wine-doors. And even then it might require further specialised tweaking.

These kind of things are described in the application database at Wine HQ.

To answer your question, I don't think it does. I have set up a virtual box with XP and VS 2008 so that we can continue to develop with it. Working using seamless desktop mode gives perfect access to the application with the knowledge that nothing will go wrong since it's not running under Wine. With a tweaked installation of XP and no bells and whistles you should be able to run Virtual Box with VS2008 on  a 1gb system. However I use 3gb of memory and set my VB to 1.5gb and it runs flawlessly.
