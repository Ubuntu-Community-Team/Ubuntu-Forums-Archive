### Post by Mr Pockets151 on 2008-06-07
I've been playing with my systems since windows ME.  So in my experience the best thing you can always do is backup all your important files.  I had to learn the hard way.  If you have lots of data and you have not backed up that data then the stress comes in.  However if the "data" has been separated then there is no need for concern, considering you have all appropriate installation disks.   


You are dual booting.  One HDD drive I suppose.   I would suggest to boot to the OS you know best.  When I installed Ububntu the first time I did a complete "image back-up" of my main XP install.  Because I knew that best.

How much data are you willing to sacrifice?  You can always re-install everything.  It's the best thing to do.  Otherwise you will get lost in the long run.  Start fresh.  I have 3 OS systems all which run independent of the other.  If one fails I have the other.  AND IF THE OTHER FAILS I HAVE WINDOWS XP.  I'm still a LINUX newbie......but by golly I finally got the hang of keeping things separate.

### Post by kansasnoob on 2008-06-07
I'm simple minded. I've never had any formal computer training of any kind.

I was however trained like a trained monkey to use a Wang to do what I was trained to do on the Wang.

Then I thought the Radio Shack Tandy was a cool toy, but my first Win 95 just rocked! I could actually move and edit text really, really easy!

Then, after several lackluster improvements came Win ME! Yep, I said ME! I loved Win ME! For my typical office tasks and saving tons of files it just worked!

Then came XP which failed to thrill me and I tried Lindows (the early version of Linspire) and I found it to be slow and just a pain in the butt!

Ubuntu (both Gutsy and Hardy) is the first OS since ME that I've actually loved! Using Abiword is soooooooooooo dang easy!

And the single best thing is that GRUB will handle multiple operating systems. My favored installation is having XP, one shared data partition (I recently went to NTFS, but FAT 32 is still fine, and it's now quite possible to use ext 2 or 3), and at least two Ubuntu operating systems.

Huh, why two Ubuntu OS's? Well, because I'm a dummy! I like to play with things and I have no idea what I'm doing. So, I can install Windows first, my toy OS second, and then the Ubuntu OS I intend to use. GRUB lets me choose which OS to boot and then if I want to play and quite possibly break Ubuntu I can do it on a partition that can easily be deleted.

Then I can use that partition to install whatever I want to play with next. I'll never get smart, but I'll never get bored:)

I don't know how many OS's GRUB will handle, but I've gone as far as having XP, Xubuntu, Kubuntu, and Ubuntu with no shared partition. How many partitions and OS's does it take before GRUB just loses it's mind?
