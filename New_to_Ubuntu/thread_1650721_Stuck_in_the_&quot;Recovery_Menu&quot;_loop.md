### Post by Rubi1200 on 2010-12-22
The results look normal except for this:[FONT=monospace]
[/FONT]> sda3/etc/fstabYour fstab entries are messed up. Did you make changes to it or move partitions around or something else?

```
# <file system> <mount point>   <type>  <options>       <dump>  <pass>
proc            /proc           proc    nodev,noexec,nosuid 0       0
# [COLOR=Red]/ was on /dev/sda2[/COLOR] during installation
# UUID=3f35356e-9d40-44be-a611-57b5c36dff0f
/dev/sda2 /               ext4    errors=remount-ro 0       1
/dev/sda1       none            swap    sw              0       0
```Clearly, this is not correct. Your Ubuntu install is on sda3 not sda2 and swap is now on sda2.

You need to run this command to find out what the current UUID values are:
```
sudo blkid
```Once you have done that you can edit the file and correct the mistakes. That should solve the problem.

Here is an example of how my fstab file looks:
```
# <file system> <mount point>   <type>  <options>       <dump>  <pass>

# swap was on /dev/sda9 during installation
UUID=607d5597-2115-44dd-a807-215fc0c14711 none            swap    sw              0       0
