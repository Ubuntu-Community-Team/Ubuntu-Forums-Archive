### Post by cbabbage on 2009-12-20
I'm not familiar with Iscan, but my first guess would be that the default scanned image is at a very high resolution and too big in memory. Images with very large memory footprints can cause problems in GIMP. I suggest you try scanning the image in again, but set the resolution very low, so that the resultant image uses much less memory. Depending on what the image is and what you intend to use it for, you'll probably find it's still satisfactory. Then try opening it again in GIMP.

It could also be the filetype that Iscan is outputting, although the GIMP would most likely tell you if the file isn't one it can deal with.

Let us know how you get on.

CB.
