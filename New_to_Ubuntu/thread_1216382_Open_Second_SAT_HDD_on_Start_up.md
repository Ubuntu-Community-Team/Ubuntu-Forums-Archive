### Post by coffeecat on 2009-07-18
Yes, you'll have to add a line to the file /etc/fstab (or one line for each partition), and create mountpoint(s) for each partition. So that we can tell you what to put, post the terminal output of the following commands:

```
sudo fdisk -l
```(To see what filesystems you have on the second hdd.)

```
sudo blkid
```(To get the UUIDs of the partition(s) on the second hdd.)

And also post the contents of your /etc/fstab file.

### Post by coffeecat on 2009-07-18
Your output of fdisk tells us that the second hdd has one single partition, sdb1, formatted ext3. Now - when you said...

> **getitright said:**
> I have successfully installed a second HDD, can see and open it in Places. What I want to do is to open it automatically at start up.

I assumed you meant that sdb1 is visible in Places, and that you can mount it if you click on it and give your password, and that it wasn't being automounted on bootup. However, this line of fstab shows that it **is** being mounted on bootup:

> **getitright said:**
> /dev/sdb1                                  /media/sdb1     ext3         defaults                    0  0

You don't need to add anything to fstab. Not only that, since the partition is being mounted in /media you should be seeing a desktop icon labelled "Data" (the partition label).

So I'm scratching my head wondering what the problem was that you were originally describing. Do you mean that when you double-click on the icon (or in Places), you get a permission denied message when you try to copy files into it or create folders? Or do you mean that you are not seeing a desktop icon and that the partition is not being automounted?

If you mean you get permission denied, that's because it is an ext3 partition being mounted by root. There are a couple of ways round this, but my favoured way would be to create a folder owned by you into which you keep all your personal files. This is what you do. In a terminal:

```
sudo mkdir /media/sdb1/MyStuff
```Substitute MyStuff for whatever you want. Now...

```
sudo chown yourusername: /media/sdb1/MyStuff
```Replace 'yourusername' and MyStuff as appropriate. Note the colon after yourusername. Now you will be able to create folders and files in the normal way, all in the folder MyStuff.

However, if you meant that sdb1 was not being automounted, then something very odd is going on. Post back with details. And if I've completely misunderstood what you mean, do post back.

**Edit:** by the way, your personal details are showing that you are/were running Gutsy Gibbon. Is this still so? Which version are you running? If you are running Gutsy Gibbon, are you aware that it has long reached end of life and is no longer supported? I.e. no more security and bugfix updates.
