### Post by _duncan_ on 2009-02-09
I have no experience with your particular modem (SM56) but I was able to use a smartlink modem up to Hardy 8.04. Not sure if any advise I give is relevant to you. Anyway, there are a couple of things you can try:

1. Most smartlink modems require adding the line 'Stupid Mode = yes' (without the quotes) in the /etc/wvdial.conf file.

2. I don't know if this was fixed in recent Ubuntu versions, but wvdial used to require creating a blank /etc/resolv.conf file to connect from a fresh ubuntu install. This can be done by entering the following command in a terminal:
```
sudo touch /etc/resolv.conf
```

### Post by phrozenb on 2009-02-09
well, i finally got it working now.

Yes, i'm using now Stupid Mode = Yes, but that didn't solve anything.
I'm also using now ATX3 mode in order not to wait for the dialing tone. Don't know if it's required or not but it's sent to the modem in a different Init than all the other specs. So now I have something like:
```

Init1 = ATZ
Init2 = ATX3
Init2 = ATQ0 V1 E1 S0=0 &C1 &D2 +FCLASS=0

```

This would enable wvdial to keep constantly dialing even if connection drops or even if no carrier was found.

However, what seems to have fixed the problem was +MS=90 instead of +FCLASS=0 although I'm not sure exactly what the later does. So my wdial.conf looks something like:
```

Init1 = ATZ
Init2 = ATX3
Init2 = ATQ0 V1 E1 S0=0 &C1 &D2 +MS=90

```

This is working ok now. However, if I stop the connection or if the connection drops i need to do a:
```

sudo /etc/init.d/sl-modem-daemon restart

``` 
This is from what I've read a bug in the current softlink driver when working with kernels 2.6.24 and above.

What I fail to understand so far is why can't i get gnome-ppp to view my modem/driver. Even if i set the same init strings for gnome-ppp I still get the "Modem not responding" error.
