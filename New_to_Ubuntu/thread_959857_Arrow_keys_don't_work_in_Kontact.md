### Post by jamesrl on 2008-12-10
LOoking through bugs, it seems kontact should have a keyboard shortcut for next unread.  Are  you sure it is set?  It might not be the down arrow (could be something like 'n' or 'space' or 'enter').  I'm running gnome, so can't easily check this.
From: [https://bugs.kde.org/show_bug.cgi?id=129743](https://bugs.kde.org/show_bug.cgi?id=129743)
```

   Description:  
Opened: 2006-06-24 13:32   
Last Changed: 2008-02-26 11:37:41
Version: (using KDE KDE 3.5.3) 
Installed from: Debian testing/unstable 
Packages OS: Linux 

Keyboard shortcuts for similar user actions vary in several kontact components. 
This has a significant impact on usability and will particularly confuse new users. 
For instance, the keyboard shortcuts for "Next unread item" vary wildly:
- "Next unread email" in kontact/mail is bound to the "Space Bar" key 
- "Next unread article" in kontact/usenet is bound to "Alt+Shift+Space Bar" 
- "Next unread discussion" in kontact/usenet is bound to "Shift+Space Bar" 
- "Next unread article" in kontact/akkregator is bouhd to the "+" key.

```
