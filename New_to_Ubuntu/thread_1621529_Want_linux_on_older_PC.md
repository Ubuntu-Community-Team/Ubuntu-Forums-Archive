### Post by SeijiSensei on 2010-11-15
> **at2smithjason said:**

Does the computer have a CD/DVD drive?  If so, you should check the BIOS by hitting the "setup" key during boot (often F2 or Del) and look at the order in which devices are booted.  Perhaps you just need to move the CD drive above the hard disk.  If you are thinking about booting from a USB drive, you'll need to check this as well.

### Post by amjjawad on 2010-11-15
Few days back, I got a 2nd PC from a friend of mine. He doesn't use it anymore so I would never miss that chance :)

P4 @3.06 GHz - 1MB L2 Cashe
RAM 512 - 32MB for Graphics
80GB HDD

Luckily, the CD-Drive is working. I had very bad and hard time trying to get a very old laptop (+11 years - 64MB RAM, P2, 4.5GB with NO CD-Drive, NO LAN, NO Floppy) up and running. I had to go through tons of procedures but all failed. I guess it's because the CD-Drive is dead and almost ALL the distros I have tried were asking or need the CD-Drive during the installation.

Anyway, for your PC which is almost similar to mine, I suggest to use Light Weight Distributions.
So far, I tried:

Fedora 13 GNOME
OpenSUSE 11.3
Ubuntu 10.10
Slitaz
ArchBang
PCLinuxOS - Zen Mini
Mint 9 GNOME
OpenSolaris

Mint was the slowest after installation on HDD. OpenSolaris was so much slow from the LiveCD so I didn't bother to install it.
Fedora was faster than Mint.
If you're looking for the real speed, I suggest to use any distribution that needs minimum 256MB of RAM. Those Distributions that need minimum 512MB of RAM will not run as fast as those which require less RAM (256MB).
There is recommended and there's minimum requirements. Don't except much when you have the minimum requirements.

[http://distrowatch.com/](http://distrowatch.com/) << is a great website to find what are you looking for.

Distributions for [old computers]("http://distrowatch.com/search.php?category=Old+Computers&origin=All&basedon=All&notbasedon=None&desktop=All&architecture=All&status=Active")

With Windows, you can't feel the real speed no matter how large your RAM is. With Linux, it's so much obvious and you can feel it. Try very light Distributions or LXDE Distributions. I didn't like LXDE much because I need to install many softwares.

Note that, I'm running on 512MB-32MB = 480MB RAM. Fedora 13 and Ubuntu 10.10 were not bad at all :)
