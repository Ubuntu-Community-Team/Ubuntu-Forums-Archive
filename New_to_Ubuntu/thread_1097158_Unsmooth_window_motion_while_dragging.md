### Post by jaguax on 2009-03-15
This just might be a Linux thing, but does anyone else notice that dragging windows, as well as scrolling down a page in Firefox is kind of un-smooth compared to Windows? The best term I can use for it is a low frame rate. It's like 20-30 fps compared to the buttery smooth 60 fps of Windows.

I mean it's still instantaneous, I can grab a title bar of a window and whip it around the screen with no lag or anything, but it's definitely skipping every other few pixels or so in the process resulting in a not so liquid smooth motion.

I also notice a lack of desktop Vsync as seen in Vista/Win7. I tried the "sync to vblank" option in the video settings but that doesn't seem to do anything.

For the record I have an nvidia 8800 GT with the latest 180 series drivers from nvidia's home page.

### Post by gmjs on 2009-03-15
I've always had the same problem.  I've put it down to poor 2D graphics support under Linux with 3D cards.  You'll probably find that running under Compiz (System > Perferences > Appearance > Visual_Effects > Normal/Extra) improves this.  I don't run my desktop like this as there are programs I use that don't run correctly under Compiz and some windows don't remember their previous locations, which annoys me too much!  Still, it might be better for you.

Hope this helps.
