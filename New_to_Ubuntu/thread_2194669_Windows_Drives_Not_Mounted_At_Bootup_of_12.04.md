### Post by jones quest on 2013-12-20
print the UUIDs of the drive with:
blkid

for example: 
/dev/sda1: UUID="5EC55EA428DEFFD3" TYPE="ntfs" 

Make a backup of your fstab
sudo cp /etc/fstab /etc/fstab.bk

Then add these entries to fstab:
sudo gedit /etc/fstab

for example:
# <file system>                        <mount point>                      <type>     <options>               <dump>  <pass>
UUID=5EC55EA428DEFFD3  /media/[windows_stuff]   ntfs-3g    rw,user,defaults      0               0

Replace [windows_stuff] with whatever names you want to give to your mount points.
Don't worry if you get things mixed, you can always correct these later or restore the backup.

Now add the folder to those mount points. I would now add a folder called windows_stuff to /media/
sudo mkdir /media/windows_stuff

On next reboot that hard drive/partition will be mounted to that folder. An will show up in your Devices.

### Post by oldfred on 2013-12-24
The fix above should work for the internals.

Not sure about an external. I only have flash drives which I just mount with Nautilus and it then mounts with label and defaults. If you have an external in fstab and the external is not connected or has not come up to speed by the time fstab is processed you will get an error but can proceed. It just is not then mounted.

You can then run this which remounts all the fstab entries. It should not have any message if it works correctly.

sudo mount -a
