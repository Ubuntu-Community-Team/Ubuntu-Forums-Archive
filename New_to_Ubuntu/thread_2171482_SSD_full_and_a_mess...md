### Post by TheFu on 2013-08-31
I'm replying only because nobody else has yet. I don't know anything about chromebooks.  It seems that you may be mixing up RAM "memory" and disk storage "memory."

If you don&#8217;t want ubuntu loaded anymore, delete it. I have no idea where it gets stored on that device.  16G would be a fine install for efficient Ubuntus, provided you minimized any use of music or video files. A single hidef video file can completely fill that storage alone.

I realize this isn't much help - be very careful if you delete things that you don't understand.

Google found this how-to [http://lifehacker.com/how-to-install-linux-on-a-chromebook-and-unlock-its-ful-509039343](http://lifehacker.com/how-to-install-linux-on-a-chromebook-and-unlock-its-ful-509039343) BTW.

I reviewed some installation instructions - seems that your system has been repartitioned. The best answer seems to be to use the chrombook recovery - [https://support.google.com/chromeos/answer/1080595?hl=en](https://support.google.com/chromeos/answer/1080595?hl=en) The instructions say this will re-repartition the SSD. Expect everything to be wiped from the drive, so backup stuff you want to keep.
