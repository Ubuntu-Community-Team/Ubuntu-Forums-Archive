### Post by Bartender on 2012-01-26
desgner -
I know that feeling of disappointment (sometimes near-panic) when you get thru the whole process and then it doesn't work the way you wanted.

I've installed Ubuntu/Xubuntu/PCLinuxOS/Mepis/Fedora dozens of times over the last few years.

Sometimes the installer CD wasn't able to set up the partitions correctly or was unable to re-format correctly.

Sometimes the CD wasn't burned right.  It seemed to install but never ran right.

Sometimes the CD would work in one PC, but not in the next one.  I imagine either the optical drive that burned the CD was a little bit off, or the second PC's optical drive was.

Once I thought it was a bad installer CD, but it turned out that the off-brand HDD I was trying to use simply would not work correctly.

I'm currently experimenting with using a USB stick for installing.  Avoiding optical drives may take some of the screwiness out of the loop.

I'm sorry it didn't work out the first time.  Don't give up.  Boot from the CD again and ask it to do the self-test.  Check the download by doing the md5 check.  Try a different HDD.  Try unetbootin or Ubuntu's Start-Up Disk Creator to make a USB installer.  Try a different PC.  

I just re-read your post.  Is this a true install to the HDD, or are you trying the Wubi thing?
