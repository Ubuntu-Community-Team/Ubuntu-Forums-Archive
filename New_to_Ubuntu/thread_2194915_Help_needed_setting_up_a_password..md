### Post by buzzingrobot on 2013-12-21
Look for  "User Accounts" (Users and Groups??) in System Settings.  See if you can modify your current username by changing the password. In your case, you'd leave a blank if it wants the old password, and hit return if it prompts for a password for authorization.

Btw, setting autologin will accomplish much of what you might be going for with this empty password gambit.

### Post by grahammechanical on 2013-12-21
Are you sure that you did not set up a password. We are asked to set a password during the install. That password will allow us to do administrative tasks without running a session as administrator, which can brings risks. We can set Ubuntu to automatically log-in which will remove the need to enter a password at log in. But we will still need a password to do certain tasks in Ubuntu. This is for our protection.

Are you sure that you did not just set Ubuntu to automatically login. We get that option during the installation process. If so, go to System Settings>User Accounts and move the slider for Automatic Login to off.

I am assuming that you are using Ubuntu 13.10 with Unity. But I wonder if you have changed this. You do not say if you are running an alternative desktop or have installed gnome-session-flashback. So my instructions may not fit your user interface. That is not my fault.
