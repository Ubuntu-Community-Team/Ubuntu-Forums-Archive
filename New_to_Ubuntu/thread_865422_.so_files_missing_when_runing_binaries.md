### Post by Yannick Le Saint kyncani on 2008-07-20
> **Enternald said:**
> when i try to run precompiled binaries everytime is shown that no xxxxxxx.so found.

apt-file is your friend.

Install it, update (apt-file update), then "apt-file search xxxxxx.so", now if a package in one of the repository would provide xxxxxx.so, apt-file will show it to you.

For example, looking for something named bin/gcc :

Apt-file may not find your library though, as it may not be available in this specific version in hardy (.so filenames contain the library version).

### Post by spiderbatdad on 2008-07-20
In some cases creating the symlinks manually for the shared libraries is an easy workaround. For example I had a similar problem with libexpat.so.0 being nowhere to be found in /usr/lib. Creating the symlink solved the problem:```
sudo ln -s /usr/lib/libexpat.so.1 /usr/lib/libexpat.so.0
```
