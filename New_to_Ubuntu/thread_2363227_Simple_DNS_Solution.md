### Post by Habitual on 2017-06-08

Not sure what you believe "my dns" to be, but I use these public ***resolvers***:
**8.8.8.8** is google.
**4.2.2.2** is Level3 Communications.  
Depends on the "VPS"? Maybe yours was provided with a known working "basic" configuration utilizing 8.8.8.8 and anything additional would be, may be your responsibility?
I provide those at my grid and application-level instances and have done so for 7 years. Never an issue.

I'd ask your VPS provider if 4.2.2.2 is okay to add.
Or if you're feeling confident, 
```
sudo cp /etc/{resolv.conf,resolv.conf.bak}
sudo vi + /etc/resolv.conf
``` and add
```
nameserver 4.2.2.2
```
You can have up to 3, but 2 "nameserver" entries is the most common setup I've seen.
Nothing to restart, edit and it's there.

Hope that helps.

### Post by SeijiSensei on 2017-06-08
If you want to specify the DNS servers, one method is to edit (as root with sudo) the file /etc/resolvconf/resolv.conf.d/head.  This file will be used to build /etc/resolv.conf.  Usually it consists solely of 
```

# Dynamic resolv.conf(5) file for glibc resolver(3) generated by resolvconf(8)
#     DO NOT EDIT THIS FILE BY HAND -- YOUR CHANGES WILL BE OVERWRITTEN

```
but you can add lines below that header like this:
```

nameserver 8.8.8.8
nameserver 8.8.4.4

```
and either reboot or restart resolvconf.  That will result in a resolv.conf like this:
```

# Dynamic resolv.conf(5) file for glibc resolver(3) generated by resolvconf(8)
#     DO NOT EDIT THIS FILE BY HAND -- YOUR CHANGES WILL BE OVERWRITTEN
nameserver 8.8.8.8
nameserver 8.8.4.4
nameserver 127.0.1.1

```
with your specified nameservers above the local address that dnsmasq listens on.
