### Post by Hospadar on 2009-08-10
I'm not sure if you can make k3b spit that stuff out, but if you use a system monitor like htop (needs to be installed from repositories, runs on command line) it'll show you all the options fed into all running processes.

For example, mine has a line like
/usr/bin/X -br -nolisten tcp :0 vt7 -auth /var/run/xauth/A:0-c8Tej
Which happens to be my X server and all the options the startup script passed to it.  So if I wanted to start it myself, I could make a call like that.

So, I would start up k3b, set up the burn you want to do, and do it, all the while monitoring top or htop, and when you see the growisofs process, pause the terminal output (or not) and write down all the options.

Here's the command line to burn an iso file to DVD:

```
growisofs -dvd-compat -Z /dev/dvd=dvd.iso
```

I always call my DVD iso's data.iso (guess you could use a $1 for the argument), so then I can use it in a bash alias:
```
alias burndvd='growisofs -dvd-compat -Z /dev/dvd=dvd.iso && eject /dev/dvd'

```

After I create the data.iso from K9copy I just type burndvd from the command line and the alias will burn the data.iso and then eject the DVD from the drive.

Here's the man page for growisofs [http://linux.die.net/man/1/growisofs]("http://linux.die.net/man/1/growisofs")

That was a specific example for burning a DVD iso using growisofs. If you want to make an iso file you can use mkisofs or genisoimage -- I believe mkisofs is sysmbolically linked ot genisoimage. Anyhow, you can use this to create the iso file:
```
genisoimage -r -J -o cd_image.iso /directory
```

and then use wodim to burn the image:
```
wodim dev=/dev/cdrw driveropts=burnfree -v -data cd_image.iso
```

Bill

P.S. you might need to install wodim; this web site should help: [http://kurinchilamp.kurinchilion.com/2009/08/how-to-burn-iso-image-from-ubuntu-command-line.html]("http://kurinchilamp.kurinchilion.com/2009/08/how-to-burn-iso-image-from-ubuntu-command-line.html")
