### Post by dark_harmonics on 2008-04-27
the best way to backup before running any customizations is to just take a backup copy of your /etc/X11/xorg.conf

aticonfig is just a tool that will automatically fix your xorg.conf. You should try automatix or envy for the easiest config (although sometimes this causes more headaches than it prevents). I have know people who swear by both of those, but i just did the config myself. The above posters already gave you the correct commands for aticonfig. use glxinfo|grep direct to figure out if you have direct rendering and fglrxinfo to see what video driver is loaded. Most times i have success with running the .run file, and then those two aticonfig commands. After a quick reboot all should be good. If not you can make sure the fglrx module loads by typing lsmod|grep fglrx and seeing if its there. If not you may need to modprobe it.
Check out this link: [https://help.ubuntu.com/community/BinaryDriverHowto/ATI](https://help.ubuntu.com/community/BinaryDriverHowto/ATI)

### Post by PsyWolf on 2008-04-27
use this command to copy the xorg.conf file to a file called xorg.conf.backup
```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.backup
```

then use this command to edit it
```
sudo gedit /etc/X11/xorg.conf
```

and to restore from the backup if things go badly, use the command
```
sudo cp /etc/X11/xorg.conf.backup /etc/X11/xorg.conf
```

and then restart your computer or just hit ctrl+alt+backspace to restart your x server

### Post by riven0 on 2008-04-27
Sorry, I was busy for a while. It's hard to say what the problem is. The official drivers are installed fine, so it should work. Some last things to try is to append the SKIP_CHECKS to your compiz binary, as someone else mentioned:

> gksudo gedit /usr/bin/compiz

Paste in: **SKIP_CHECKS=yes** somewhere near the top of the file. Save and then:

> compiz --replace &


When I tested out the last command on my comp, (compiz --replace), it crashed my xserver, but it seems to work for a lot of people. Thankfully, Compiz seems to work fine on the open source ati driver, so you may want to try that as the last option.
