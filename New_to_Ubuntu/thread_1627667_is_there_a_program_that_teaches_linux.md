### Post by sanderd17 on 2010-11-21
I would say: go explore.

Change settings, try to install things, try to remove things ... until it breaks and try to understand why it breaks (e.g. you deleted a file needed for boot...) and see if you can fix it.

When it is infixable broken, reïnstall Linux and try it again. Just do this until you have a feeling that you know linux.

P.S. you don't have to do this in your regular install, search virtualbox in the software center, install it, run it and follow the wizard to create a new OS. It will let you install a virtual OS on which you can experiment without touching your current OS.

Trial and error has always been the best teacher for me.
