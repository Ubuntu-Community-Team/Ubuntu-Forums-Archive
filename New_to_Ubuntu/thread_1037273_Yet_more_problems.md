Hey now, Captain_tux was just trying to help. No need to bite the hand of the one that feeds you.

Even if the phone works in general its bluetooth file transfer could be experiencing difficulties. I've had experience with some phones from US carriers that will not exchange files with a desktop/laptop b/c the carrier does not want users to be able to transfer their own files by to rather use MMS.

In any case, were you able to change the setting under System -> Preferences -> Bluetooth?
On the services tab make sure Input service is running and on the general tab check the box to "receive files from remote devices".

To accept the files without a prompt from any device (careful) check "Automatically authorise remote ..."

What exactly is the bahaviour you have on the phone? Can you see the computer and then when you select it you receive an error? What error? Does Ubuntu currently prompt you to accept the file?
