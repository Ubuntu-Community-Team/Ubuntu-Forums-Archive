### Post by perspectoff on 2010-09-20
Wow, now I know why I install Ubuntu the way I do.

I always use the Ubuntu Server text installer, which is very similar to the Debian installer.

It is very, very fast and I can install in about 15-20 minutes.

I then let the GUI desktop (Ubuntu-desktop or Kubuntu-desktop) install over the Internet, so I can leave it for a while and go do something else.

If there is any installation troubleshooting to be done, I'd rather have a fast text-based installer than a GUI installer. If I have to re-install for any reason, it minimizes the time needed. This is especially important in virtual machines or environments in which the usable RAM may be small. The GUI installer needs more RAM. I have had instances where the GUI installer can't run at all, but the text-based installer runs fine.

This is true no matter whether the text-based installer is Ubuntu or Debian.

Any of the popular desktops (Gnome, KDE, XFCE, LXDE) can be added to Debian. 

It is not really that difficult to make a Debian system look like an Ubuntu system.

### Post by NightwishFan on 2010-09-21
I find that Ubuntu packs a lot more functionality into the live and alternate CDs that the Debian CD-1. However it is designed that way, whereas the Debian CD is mainly for convenience and not a full system. I believe CD-1 lacks networkmanager, iceweasel, and power management utilities such as cpufrequtils. Debian though is not designed to be installed just from 1 cd

The Debian stable with a DVD or the first few CDs should be sufficient. Also it DOES include a graphical installer though it is only a copy of the command line one with the ability to use a mouse. Still quite easy to use either way. Best best is a net install with a good connection if you have one. Not everyone does. (including myself)

Also the Gnome Desktop is included on CD 1. There are also CDs for KDE and XFCE/LXDE at the bottom. It is a bit confusing on your first try to hit spacebar to select "Graphical Desktop" when prompted. Some may instinctively hit enter and start the install with nothing. The solution is let it finish, reboot, log in, insert the cd, and run:
[CODE]tasksel install gnome-desktop --new-install[CODE]
