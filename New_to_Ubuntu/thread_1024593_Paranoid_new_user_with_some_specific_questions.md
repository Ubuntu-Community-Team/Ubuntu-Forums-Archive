> [SIZE="1"]4. I've read that it isn't necessary for new user to dive into the world of terminal commands. One of the main reasons you see people giving out terminal commands is that it is by far the easiest way to give a user an instruction that will pretty much just work.  If you try to guide someone through a graphical process there are a lot of things that can go wrong - if an instruction is slightly vague or if the user's UI is set up slightly differently.  Also, running a terminal command allows any important error messages that may occur to be easily recorded and followed up on.

Very few people have complete mastery of the command line - most people are continuously learning new techniques as they become necessary.  I'd advise to get to grips with the fundamentals, such as navigating the file system (**cd, ls**), moving and copying files (**mv, cp**) etc.
After that you can just use these forums, and myriad other references to pick up what you need to know as you go along.

Whenever you are given a command to run, don't be afraid to ask for clarification of what it actually does.  If you are still in doubt, try to learn how to use manpages (**man command**), which will allow you to find anything you need to know for yourself.  Manpages can be daunting when you first set eyes upon one, but soon enough you'll be able to decipher them.

Ideally, you should always know what a command does before executing it, especially if it contains the likes of **sudo** or **rm**.

### Post by Greyed on 2008-12-29
> **Billy87 said:**
> Such habits include; making separate folders for EVERYTHING e.g. "Stuff" on my desktop with a bunch of folders in it like "Movies", "Documents", "misc pics", "new music" etc all with another bunch of folders in them.

This one isn't so bad but you'll find that putting things on the desktop isn't default for Linux.  Things go into your home directory and you generally open up your home directory and work from there.  So the same paradigm, just the initial folder you're opening is a tad different.

Note that this has been done on Windows since the NT days but because of the registry most Windows people are oblivious to the fact that they have a home dir.

> Another habit is keyboard shortcuts, mainly in Word and on firefox, most notable the Ctrl C and Ctrl V shortcuts (copy and paste).

That's not a bad habit, that's a good one.  CNTL-C and CNTl-V work in most places as you would expect it.  However you'll have to watch out for the X style marking.  In Windows to copy something to the clipboard you needed two discrete operations.  Marking and CNTL-C to copy.  The X standard, which most Linux apps follow, marking *is* copying.  So you mark in one window, switch to another and CNTL-V to paste.  No need for the explicit CNTL-C.

> Another is deleting the installation exe file after installation is complete.

What install execs?  >.>

Well, to be truthful 'round here that's called apt-get autoclean but it is handled automatically by Ubuntu so no need for you to worry about it.  Ubuntu keeps a local cache of the latest installed packages.  This way if you need to reinstall something, for whatever reason, the operation is extremely quick.  This space is minimal.  For example on this VM my local cache is a whopping 370Mb.  Considering I'm on an 8Gb drive that's not bad.  I could delete those with no loss of functionality but with over 2Gb left on this VM I don't feel the need.

> Am i being too paranoid and shouldn't worry much about these stuff?

I think you're being overly cautious.  :)

> However i've noticed that almost everywhere i look there are people saying type this or type that and, to be honest, most of the commands look like complete gibberish to me.

That is because it is far easier to tell someone "Cut & paste this into a terminal and paste the results here" than "ok, open up the programs menu, then find the spam program, open it up, go to the ham tab, scroll down until you see the ni entry and type what you see here."

> For the life of me i cannot comprehend the meaning of sudo except that it is used everytime and that it must be relevant for any command to work.....then out of nowhere comes kdesudo (or whatever it was, i didn't hang around long enough to memorise it)....what the hell???

**S**uper **U**ser **DO**.  IE, run the following command with root access.

kdesudo = sudo for the gui in the KDE environment.

The difference is that sudo allows console programs to run as root while kdesudo allows gui programs to run as root.

Hi, Billy,

While you are provided with a default home folder (with default subfolders in turn, for docs, music, etc), it is not mandatory to use it. In fact you may want not to use it, and to setup a separate partition for your data instead.

The reason is that linux can cope with windows filesystems (ntfs, fat32...), but windows doesn't understand linux filesystems (ext3, reiserfs...). In your transition to gnu-linux, perhaps you'll need to boot windows sometimes for the sake of doing something that you don't manage yet to perform from ubuntu, and in that case you'll find nice to have available your data from within windows.

My advice would be to setup a fat32 partition for your data, independent of those of ubuntu an windows. And make the habit of using it from now on, no matter which OS you're on ;-). It seems reasonable to me 15Gb for xp. The ubuntu partition doesn't need to be any bigger either (because you'll store your files elsewhere).

NOTES.- fat32 is perfectly supported in gnu-linux, but the size of a single file  has a limit of 4Gb (large enough for most people, except if you plan to save raw dvd images). Ntfs has  no limits and also good support, but it has been added more recently, so I'd recommend being cautious. Finally, I would not use the windows partition to store your data: in the case of a bad shutdown of windows, the partition leaves as "in use", and ubuntu wouldn't be willing to access it (to not mess everything).

### Post by dwasifar on 2008-12-29
This may be something you want to do at a later time, rather than right now, but rather than dual-booting you should consider installing Windows in a virtual machine.  You're obviously extremely well-prepared for what you're about to do (in fact I stand somewhat in awe of your thoroughness), so it's obvious you do not need to preserve your existing Windows installation just to save the data files it contains - you already have everything backed up.  

A virtual machine is basically an OS running inside another OS.  If you set up Windows in a VM, then when you run it, Windows itself (the "guest" operating system) will be running inside a window on your Linux desktop (the "host").  The advantages over dual booting are:

1) You can run both operating systems simultaneously and share data files between them.  
2) You do not have to mess with configuring boot loaders.
3) Booting a VM is much faster than rebooting a dual-boot machine.
4) You can set up your VM to allocate disk space to the guest operating system as needed, rather than taking up that 15GB all at once.
5) A virtual machine can be backed up completely and transferred to another host, without reinstalling the guest operating system - even if the new host is running different hardware or operating system than the old.

A few downsides:

1) Applications do not have direct access to the video hardware.  If you want to play graphics-intensive Windows games, a virtual machine is not the best choice.
2) USB support can be a little problematic.  It usually requires some massaging of Ubuntu's settings to get USB working properly for the guest OS, and you have to be sure the virtualization software you pick supports USB.

The two most common virtualization options for Linux are VMware and Virtualbox.  They're both good.  Right now I'm running Virtualbox (Sun's complete version, not the OSE version in the Ubuntu repos).

As I said, this may not be something you want to do right away.  But if you are interested in pursuing it, say so; there are plenty of resources to help you along.

Best of luck to you!  Welcome to the Ubuntu community.  :)

### Post by Frogbarf on 2008-12-30
The secret to sudo is that you only use it when logged in as your administrative user. During Ubuntu installation, you are asked to specify a user id and password for it; that becomes your privileged administrative id. The install process doesn't say this quite emphatically enough and there are regular reports of people who've forgotten their admin id or its password. There are ways around that problem, but if you pay attention to this message, you won't have it in the first place.:P

Good operating practice is to set up one or more unprivileged user ids as soon as your installation is complete, then use these for day to day work: email, web browsing, using applications, etc.

Only when you want to do system maintenance should you log in with the admin user id you set up, typically when installing a new application or updating your system.

Ubuntu has a "root" user id, but it's disabled so you can't log in with it.

As for the gibberish following sudo, just copy and paste it. Remember that in the terminal window, paste is ctrl+shift+V, not just ctrl+V.

[EDIT]
Any Hardy Heron users interested in adding gufw to their repository, but not wanting to include **all of backports** run the following:
```
sudo gedit /etc/apt/preferences
```
Paste this into the file:
```

Package: *
Pin: release a=hardy-backports
Pin-Priority: 0

Package: gufw
Pin: release a=hardy-backports
Pin-Priority: 800

```
Then create a symlink, so synaptic will adhere to the pinning too.
```
sudo ln -s /etc/apt/preferences /var/lib/synaptic/preferences
```
Then add hardy-backports to your sources, and not worry about a computer upgrade :)
```
sudo apt-get install gufw
```
And it is installed.
