### Post by QIII on 2014-09-27
What it is, without being judgmental, is something affecting your browsers and not your OS.

You need to change your profiles, not reinstall.  That will probably result is persistence of the behavior.

Something you have done (perhaps unwittingly) or some site you have visited has caused this to happen.  A site may appear to be perfectly legitimate, but if it has itself been hijacked -- or if it puts forward an innocent face and hides some ill intent -- then you can be compromised.

This is almost certainly behavior that affects your browsers and not the OS.

### Post by Vladlenin5000 on 2014-09-27
According to Google's instructions, the *Default* folder, inside ~/.config/google-chrome/ is the one holding the user profile. Renaming (or deleting) it forces the program to recreate it (as a new user) thus clearing the user profiles related problems.
Keep in mind Google Chrome (or Chromium or Maxton or ... for that matter) ties and syncs a user profile with a (GMail) email address so if you login with the same email in Chrome (or Maxton or ...) in a compromised Windows machine there are very good chances that the changes made by malware in that machine will be "mirrored" when back to Chrome in a Linux environment.
