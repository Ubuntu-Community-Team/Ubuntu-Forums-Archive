### Post by anewguy on 2011-12-04
Yes, the udev rules can get overwritten.  That is why they provide the way to create your own rule files to override the defaults.  The below quote is from the /etc/udev/rules.d folder readme:

```
The files in this directory are read by udev(7) and used when events
are performed by the kernel.  The udev daemon watches this directory
with inotify so that changes to these files are automatically picked
up, for this reason they must be files and not symlinks to another
location as in the case in Debian.

Packages do not generally install rules here, this directory is for
local rules.  If you want to override behaviour of package-supplied
rules, which can be found in /lib/udev/rules.d, you can do one of
two things:

 1) Write your own rules in this directory that assign the name,
    symlinks, permissions, etc. that you want.  Pick a number higher
    than the rules you want to override, and yours will be used.

 2) Copy the file from /lib/udev/rules.d and edit it here; you
    should generally only do this if you want to prevent a program
    from being run.


If the ordering of files in this directory are not important to you,
it's recommended that you simply name your files "descriptive-name.rules"
such that they are processed AFTER all numbered rules in both this
directory and /lib/udev/rules.d and thus override anything set there.
```

As far as viewing the contents of the floppy, try this and see if it will work for you:

mkdir myfloppystuff

sudo mount /dev/fd0 -t msdos myfloppystuff

cd myfloppystuff

ls

### Post by anewguy on 2011-12-04
When you do the mount and then the ls like I posted, does the floppy drive ever cycle up?  Also, if you'll forgive a dumb question, is there really anything on the diskette?

>>>>>>> EDIT: 

 There is a package in the repositories called fdutils.  It supposedly allows for auto-mounting floppies and some other things as well.  You may want to check it out.  In the mean time, here is how to make your udev rule change permanent:

- open a terminal window

- sudo gedit/etc/udev/rules.d/myfloppy.rules


- cut and paste in the following:


# PC floppy drives
#
KERNEL=="fd*", ENV{ID_DRIVE_FLOPPY}="0"

# USB floppy drives
#
SUBSYSTEMS=="usb", ATTRS{bInterfaceClass}=="08", ATTRS{bInterfaceSubClass}=="04", ENV{ID_DRIVE_FLOPPY}="0"

-save the file in the editor and exit

It should take effect in a couple of minutes, but a reboot would make sure it takes effect.


END OF EDIT  <<<<<<<<<<
