### Post by Claus7 on 2013-11-05
Hello,

I was able to find this thread:
[http://ubuntuforums.org/showthread.php?t=1144599&page=4](http://ubuntuforums.org/showthread.php?t=1144599&page=4)
but now is closed.

I faced exactly the same problem with a i686 laptop, OS windows 7. The installation almost finished, yet the permission denied error popped up. 
I restarted the system and I picked up ubuntu option, that was there just after boot, even though the installation seemed to be a failure.

A locale message us-something... appeared saying that was ok. Waiting a little bit, I tried to press enter or type exit in order to abandon the idea without avail. Yet, after a couple of seconds some messages appeared that were fixing grub menu and that sth else with generic kernel appeared and the ubuntu logo showed up. In the end the system entered in ubuntu normally. So, from my experience, if you face this issue, wait a little bit, while choosing the ubuntu option from the booting menu.

Something that I will have to mention: verify that your connection to the internet is sound, because this caused many problems, while following the installation process. It needs minimum 10 minutes in order to download some files necessary for the installation. At least the process of running wubi from windows without attempting anything else.

Regards!
