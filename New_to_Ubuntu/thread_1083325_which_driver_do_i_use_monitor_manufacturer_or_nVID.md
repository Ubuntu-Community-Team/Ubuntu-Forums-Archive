According to your Xorg.0.log, you haven't got the nvidia driver properly installed (the glx module is apparently there, but the actual kernel module is either missing or the wrong version).
Try this:
1. If you have tried to install drivers downloaded from nvidias homepage (which would explain why the kernel module would be the wrong version):
```
sudo nvidia-installer --uninstall
```
2. Uninstall all nvidia drivers through synaptic. Search for "nvidia-glx" and mark any installed packages for **complete** removal.
3. Reset your xorg.conf:
```
sudo dpkg-reconfigure -phigh xserver-xorg
```
4. Reboot
5. Use the restricted driver manager (System-->Administration-->Hardware drivers) to enable the driver.
6. Reboot.
7. If things still don't work, post your new /var/log/Xorg.0.log in this thread.

To change which user gets logged on automatically, start gdmsetup:
```
gksudo gdmsetup
```
In the tab "security" you should be able to change user in the text input box under "activate automatic login".

### Post by jocko on 2009-03-07
I guess that your problem is that the nvidia driver fails to detect your monitor correctly, so it doesn't know what resolution/refresh rate your monitor is capable of.

First you need to set xorg to use the nvidia driver instead of vesa:
```
sudo mv /etc/X11/xorg.conf /etc/X11/xorg.conf[COLOR=Red].bak[/COLOR]
sudo nvidia-xconfig --force-generate
```

Restart X to see if it works (Ctrl+Alt+Backspace).
If it does not work, do NOT use xfix or dpkg-configure to reset xorg.conf. Use this command instead:
```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf[COLOR=Red].nvidia[/COLOR]
sudo cp /etc/X11/xorg.conf[COLOR=Red].bak[/COLOR] /etc/X11/xorg.conf
```

You may probably need to add your monitor's horizontal sync and vertical refresh rate ranges to xorg.conf. You should be able to find this in your monitor manual or on the manufacturer's webpage.```

gksudo gedit /etc/X11/xorg.conf[COLOR=Red].nvidia[/COLOR]
```
Find the monitor section and make it look something like this (I guessed the numbers from the information in the .inf file you posted):
```
Section "Monitor"
    Identifier     "Monitor0"
    VendorName     "Unknown"
    ModelName      "Unknown"
[COLOR=Blue]    HorizSync       30.0 - 83.0
    VertRefresh     56.0 - 75.0[/COLOR]
    Option         "DPMS"
EndSection
```
Now replace the xorg.conf:
```
sudo cp /etc/X11/xorg.conf[COLOR=Red].nvidia[/COLOR] /etc/X11/xorg.conf
```

### Post by jocko on 2009-03-09
If things was working for your wife but not for you, the problem was not in xorg.conf.

The first thing you need to do is to make sure you get it working in your wifes user account again.
So I hope you have a copy of the xorg.conf that worked, was it the xorg.conf.nvidia that you made from my instructions? In that case replace your current xorg.conf with that one:
```
sudo cp /etc/X11/xorg.conf.nvidia /etc/X11/xorg.conf
```That should do it, as long as the 180 driver supports your card.
Once you've found a xorg.conf that work, do **not** use any tool that makes changes in xorg.conf (that includes xfix, nvidia-xconfig, nvidia-settings, the hardware driver manager, dpkg-reconfigure xserver-xorg *etc*.).

So when it worked for your wife but not for you, did you get the correct resolution at the login screen?
And when you tried to log in as your user, xorg crashed and threw you into xfix or safe graphics mode?
The only thing I can think of which may cause a user specific problem is that you have some settings in nvidia-settings that cause the nvidia driver to fail *after you log in*, so delete the file .nvidia-settings-rc (hidden in your home directory).
It will be regenerated with default settings next time you log in with the nvidia driver.
You may also need to make sure you are member of the video group (I don't think that's the cause of your problem, but it doesn't hurt):
```
sudo adduser [COLOR=Red]username[/COLOR] video
```(Change [COLOR=Red]username[/COLOR] in the command to your login name).

### Post by jocko on 2009-03-10
> **ozark_hillbilly said:**
> What video card ABSOLUTELY POSTIVELY works with Ubuntu Linux?
Simple answer: nvidia.

I would advice against an ati card, as the proprietary driver sucks and the open source driver does not yet support all features of the newer cards (3d rendering, tv-out etc.). On top of that adm/ati just announced that they are dropping support in the proprietary driver for anything except their newer cards, so unless the open source drivers get better real soon there will be cards that are still sold which have nothing except 2d support (= no google earth) on the upcoming linux distros (such as ubuntu 9.04).
Fortunately amd have released specifications for all of their cards to the open source community, so the open source driver *will* improve, but probably not on time to cover for the lack of support in the proprietary driver... And with fewer cards to support, maybe the proprietary driver will be easier for them to fix (but I would not risk buying a card until I know it does work).

In my experience nvidia cards "just work" if you use the restricted driver manager to activate the driver...
And since you had it working in your wife's user account, the reason it did not work in your account must have been something you did yourself to some user specific configuration file in your home directory. I would say keep trying to fix this instead of buying a new card. The same thing that causes your problems now will most likely still cause problems with another card...
Once you get it working again in your wife's account, it should not be too difficult to get it working in your account (as long as you don't try to fix it by reconfiguring the xserver again).

Other graphics card manufacturers: I know nothing...
