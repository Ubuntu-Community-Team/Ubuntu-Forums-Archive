### Post by LowSky on 2009-01-07
[http://blog.shevin.info/2009/01/truth-about-64bit-ubuntu.html](http://blog.shevin.info/2009/01/truth-about-64bit-ubuntu.html)
[http://64-bit-computers.com/linux-ubuntu-610-64-bit-vs-32-bit-benchmark-test.html](http://64-bit-computers.com/linux-ubuntu-610-64-bit-vs-32-bit-benchmark-test.html)

In the article stated before uses Ubuntu 6.10 as the version tested. The test is skewed because it is using 32bit apps in a 64 bit operating system, it doesn't matter if the Os was recompiled or not, the fact is 32 bit apps have to run through an extra process which in turn will slow them down. The real reason that 64 bit hasen't took off was Intels little too late use of the technology and the major software companies (including Microsoft) not supporting the newer technology until 4GB of RAM became more standardized nottoo mention that none of these companies wanted to leave older users out of luck so 32 bit has held on tight.

The best reason to support 64 bit computing is the use of files over 4GB, 32bit processing cannot fulling use a file that is over the 3.2GB size instead it will mapp the file sfor use and use the part it need at the time, this slows a computer down, if the computer had enough RAM and the processing ability an entire file can be moved and read all at once, without mapping, but other issue pop up like hard drives not able to read/write fast enough to accomidate this. That is  why even enthusest don't ever use more than 4 GB of RAM as the benift drops significantly in a personal computer. Sure in a server the extra RAM can be used but for the normal user it becomes wasted space that cannot be entirely utilized.

So in conclusion 64 bit cannot take off until a better file storage solution can be created that can read/write much quicker. This is really the biggest problem facing computing technology right now, we can fit Terrabyte onto a hard drive but access speeds have stayed relatively the same for about 10 years, we need drives that can spin much faster or maybe more RAM based drives.


sorry for the long winded post but i'm very passioate about 64bit technology...  :)
