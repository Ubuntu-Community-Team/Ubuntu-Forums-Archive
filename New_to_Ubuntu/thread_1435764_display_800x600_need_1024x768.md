Open up a terminal, navigate to where the folder is, I'm going to assume Downloads 
```
cd ~/Downloads/dripkg
``` 
the ~ signifies your home directory (eg. /home/*username*)

Then you need to make that install script executable.
```
chmod +x ./install.sh
```
the ./ means "in the directory I'm currently in"

then run it! (It will most likely need "root" (aka Administrator in windows terms) to run - To do that we use "sudo" the password it asks for is your login password)
```
sudo ./install.sh
```


edit: I'd go on and get your drivers working before deleting anything. Then once it is installed you can delete it all.

### Post by Roger Allott on 2010-03-21
I'd guess that this is not a driver issue. There have been numerous similar threads on this issue - I started one of them when my Toshiba Satellite Pro gave me the problem.

The root problem is that your monitor type wasn't auto-detected at install.

In past versions of Ubuntu, your monitor display settings (and some other hardware settings) were kept in a file called xorg.conf, which you could amend manually when auto-detect didn't work.

In 9.10, that file is no longer created, so you need to create one for yourself.

Take a look at [this thread]("http://www.rushmessageboard.com/cpmb/index.php?showtopic=33283") to see how someone helped me resolve the problem.
