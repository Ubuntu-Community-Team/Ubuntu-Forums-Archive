> The system configurations are:
Pentium 100 with 16 MB and 32 MB RAM 
Cyrix 333 with 64 MB RAM
Pentium 733 with 128 MB RAM.

These are probably a bit old to run as desktops with Edubuntu.  If you want desktop Linux machines, the P100's will struggle with any current distro.  The others do have plenty of options, including Ubuntu derivatives.


If you are specifically looking for Edubuntu, the 333 and 733MHz machines will do well as terminals, but you'd have to set up a Server with decent specs.

This explains a little: [http://doc.ubuntu.com/edubuntu/edubuntu/handbook/C/server-hw.html](http://doc.ubuntu.com/edubuntu/edubuntu/handbook/C/server-hw.html)

### Post by Stunts on 2008-06-09
Xubuntu runs very slow on a few PC's I've installed it on in my lab.
And they're PIII@866 MHZ.
Even with 194 Mb RAM it was too slow for everyday use.
The solution I have found was make it run Fluxbuntu. They're ultra fast now. A lot faster then windows in fact.
Keep two things in mind tho - fluxbuntu is no as user friendly as xubuntu or ubuntu and it has a few minor glitches - such as the menus not updating automaticaly after you install new applications (you have to run "sudo update-menus" in a console for that to work), SAMBA doesn't work "out of the box" like in ubuntu (tough I made it work with FUSEsmb - really neat) and the ROX file browser misses a few commonly used functions such as ctrl + c to copy and ctrl + v to paste - I installed xfe file browser for the effect tho, it's a lot more windows like, great for someone who is just starting to migrate.
I recommend it only if you are already somewhat confortable using linux. I have only 7 months of linux and I still have a lot to learn, but I have managed to make 3 useless PC's that were going to be thrown out usable by everyone in the lab. Computer newbs are actually liking it much better than windows.

Hope it is of some help!

P.S. - You also have to manually associate file types in ROX and xfe.
