### Post by CatKiller on 2009-07-29
Normally, resolution problems are caused by one of the monitor, the graphics card, or the graphics driver not passing the EDID information to X that lets it automatically set the correct refresh rate, and so it defaults to really low values that won't damage any monitors. You can manually set the values for your monitor by putting the VertRefresh and HorizSync ranges in your xorg.conf. You should be able to get these numbers from the manual or specifications for your monitor. These values go in the "Monitor" Section of xorg.conf.

You can edit xorg.conf with ```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.backup
gksudo gedit /etc/X11/xorg.conf
``````
Section "Monitor"
        Identifier      "Configured Monitor"
        HorizSync       aa-bb
        VertRefresh     cc-dd
EndSection

```You'll need to replace aa-dd with the values for **your** monitor.

### Post by CatKiller on 2009-07-30
> **mavio said:**
> EDIT: I didn't choose the driver. It was probably chosen automatically during installation

That does suggest that the Intel driver might not work for you.

There are two things that I can think of that might cause your resolution problem, even with specified timings. The first is that your monitor might be providing erroneous EDID values rather than none at all. You can tell X to throw away any values it gets from the monitor by putting ```
        Option          "UseEDID"                       "False"
``` in the Device section. The other option is that the vesa driver just isn't providing higher resolutions - it is a really simple driver. You might try putting 

```
        Driver          "intel"
``` instead. I'd suggest making a backup before you try that with ```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.backup
``` in case you get no graphical environment at all. You can restore from your backup by using Ctrl-Alt-F1 to get to a virtual console, logging in, and then running ```
sudo cp /etc/X11/xorg.conf.backup /etc/X11/xorg.conf
``` You might also find something enlightening in X's log:

```
less /var/log/Xorg.0.log
``` (Page Up/Page Down do the obvious, q quits less)

### Post by CatKiller on 2009-07-30
> **mavio said:**
>  I don't know if it depends on the hardware of my computer, but a simple action like scrolling this page makes the CPU to jump from 30% to 90%, and the scrolling is at a low frame rate.

It depends on the driver. Only an appropriate driver can communicate with the hardware directly. The vesa driver, that you're using now, can work with any hardware because it doesn't use the hardware at all. All the stuff that would normally be done by the graphics hardware is done by your processor instead, as well as everything else.

Intel's drivers are reasonable for the chipsets they support, but I guess that chipset isn't one of them.

You can buy pretty much any graphics card that will fit and it will blow the combination of that integrated graphics chipset using the vesa driver out of the water. That's probably the best bet for getting the most out of your computer. More RAM wouldn't hurt, either. The GeForce 6200 PCI should still be available somewhere online. It's a budget part, and a bit long-in-the-tooth at that, but it's miles better than what you've got now. Otherwise, any decent PCI (not PCI Express - that's too new for that computer) card should work. NVidia cards have better proprietary drivers and Ati/AMD have better open-source drivers.
