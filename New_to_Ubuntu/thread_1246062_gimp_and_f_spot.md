### Post by Penguin Guy on 2009-08-21
Use [COLOR="Green"]File -> Save As...[/COLOR] (or [COLOR="Green"]Shift + Ctrl + S[/COLOR]) in gimp. Then simply choose a name for it and click 'OK'.

### Post by Bartender on 2009-08-21
There's probly more elegant ways to do it, but I'd probably create a folder in /home and call it "GIMPprojects" or something similar.  Save your *finished* projects to that folder as .jpg or .tiff or .png or whatever, then when you're in F-Spot navigate to that folder and Import.

Unfinished GIMP projects are saved as a file type that F-Spot probly can't open.  

It looks like F-Spot now has a function to detect and ignore duplicates.  Thank God for that.

### Post by Arla on 2009-08-21
Having tried out F-spot, I have to say, I'm not a fan, while an okay program, the need to import everything SUCKS, I tried g-thumb but found it abysmally slow when handling RAW files (other than that it was quite nice) and am now using digiKam, the latest version (1.0.0 beta 3) uses folders rather than needing to import everything, once you have your albums setup, you simply put things in the folders that are the albums and digiKam picks up the new pictures.

Overall, love the way digiKam is going (and it reminds me of pro software a professional friend of mine showed me).
