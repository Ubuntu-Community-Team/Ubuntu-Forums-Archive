### Post by Excalibre on 2008-06-23
I'm going to join in with the chorus and say to dual boot, which means you'll have a choice between using WinXP and Ubuntu when you turn on your computer. It's not hard to set up, but you'll need to partition your hard disk. The Ubuntu installer will help you do that but make sure you know what you're doing so you don't inadvertently wipe your Windows partition.

Aside from that, do some research. I mean, your eventual happiness or unhappiness depends a lot on what you use your computer for. If you like tinkering with your computer you'll probably be happy, but if you're big on games you almost certainly won't be. ### Post by Excalibre on 2008-06-24
> **uzzo2 said:**
> i really appreciate you guys taking me to school on all this stuff. how exactly will i do this. another post said to make sure i install xp first, so when i put the ubuntu disk in, how do i set this up?
If you install XP on a new hard drive, it'll by default make one big partition with XP on it. This is fine, it's the same thing you'd get if you bought a pre-assembled system. Then when you install Ubuntu, you can shrink the XP partition while leaving it intact, and use the extra space to make partitions for Ubuntu. The major difficulty with partitioning is that it's time-consuming and somewhat risky to change your partition scheme later; partitions can be created, deleted, resized, etc. using a program like gparted, but this is annoying and difficult. So you want to be sure to come up with a good scheme in advance.

My basic suggestion would be to leave say 20 gigs for XP (since it's just the OS and any Windows programs you install, you won't need much space), and maybe say 40 gigs for your root partition for Ubuntu (you'll set a mount point when you do your partitioning, and the root partition's mount point is "/".) Maybe about 10 gigs for your home partition ("/home"), which stores all of the settings for each user, and somewhere less than 10 for your swap partition (the usual rule is twice as much space as you have RAM.) It's good to set up a separate home partition so that if the worst happens you can reinstall Ubuntu without losing all your program settings.

Then the rest can be your data partition, formatted NTFS so it's accessible from both OSes. You want this one to be big because it's where you should store all your stuff, and particularly big things like movies and music.

You can only have four "primary partitions" on a hard drive, but you can make some of them "extended partitions" which can then be divided into "logical partitions". The difference isn't important, but XP should be on its own primary partition, which it will anyway after you install it, and I would put the big data partition on its own primary partition as well. That leaves one extended partition to hold your root, home, and swap partitions.

Another thing to keep in mind is that you can run XP from a virtual machine; there's a free (free as in no cost, but not actually "free software") virtual machine called VMWare Server, which you can get at vmware.com (it's sort of hidden among all the for sale products they have, look around.) When you install it, you can create any number of virtual machines; each one is a big file that basically serves as an imaginary hard disk that you can install Windows onto; I have a 10 gigabyte virtual machine running XP which is what I usually use when I need to run a Windows program. (Incidentally, these virtual machines will have to be on your root partition, since they won't run if they're on an NTFS partition, which is why I recommend leaving 40 gigabytes for your root partition.)

So once you work out a plan for that stuff, you can just install XP as normal, and then install Ubuntu, and when it comes time to partition your hard drive, it will ask if you want to wipe out your hard drive or do a custom partition scheme. You choose a custom scheme, and then it'll give you a screen where you can do things like resize your XP partition and create the partitions you need for Ubuntu. Just make sure it's set not to format the XP partition, because you don't want to erase that.

Glad the thing boots up! Ok, the terminal is under Applications -> Accesories -> Terminal.

Quick note on a UNIX terminal, CRTL+C and CRTL+V to cut and paste DO NOT WORK, those mean something different in a terminal.(They work as they would in Windows everywhere else) Use SHIFT + INSERT to paste copied text from a website into the terminal, or just highlight text, go to the terminal, and middle click. That's a Linux thing. Don't fear the terminal, I know it's a paradigm shift from XP, but sometimes troubleshooting a problem with a few commands is much faster than trying to explain how to get there using a GUI. You don't need it much, but it is a tried and true old-skool UNIX toolbox, and once you learn some terminal commands you will quickly become impressed with the power it gives you over your PC.

Anyway, let us know how you get on, and how the commands work, they may be telling you how to manually install a driver.

### Post by uzzo2 on 2008-06-27
Installing and using a Brother printer driver into a Linux based system using the LPD service (lpr)

A wide range of Brother printers are able to print on a Linux based system using the LPD service. To learn how to install a driver on a Linux based system which uses LPR commands, please refer to the following document.

Note:

    * For cups users:
