### Post by lah7 on 2013-07-30
From what I read, you're trying to access your virtual machine's web server through port 80 using NAT port forwarding.

I came across this from the Virtualbox Manual:
> Forwarding host ports < 1024 impossible:
On Unix-based hosts (e.g. Linux, Solaris, Mac OS X) it is not possible to bind to ports below 1024 from applications that are not run by root. As a result, if you try to configure such a port forwarding, the VM will refuse to start.

Meaning that unless Virtualbox is ran as root **(not recommended!)** then your VM isn't able to bind port 80 and pass traffic through the Ubuntu host to the VM.

A workaround is to try another port above 1024, and access it at [http://192.168.1.100:1024](http://192.168.1.100:1024) or [http://localhost:1024](http://localhost:1024)
Or switch to bridged networking so the VM has access to all network resources and gets its own IP address, if this is suitable for your needs.
