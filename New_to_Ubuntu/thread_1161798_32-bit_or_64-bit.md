### Post by diablo75 on 2009-05-17
I use 32-bit on a 64-bit processor solely because I don't have any intention of upgrading my RAM for a very long time, but also for software compatibility reasons (I like to feel sure about a program running and not needing any hacking to get something like flash to work).  As for a difference in system speed with no changes in hardware, and just your standard 32-bit vs. 64-bit install... I've never benchmarked it myself, but have been told there is little if any noticeable difference in performance.  As I understand it, the only real advantage you get with running 64-bit is the option to have more than 4 GB of RAM, but I could easily be wrong about that.

### Post by Duskao on 2009-05-17
I have been using 64 bit for a couple months now and I wouldn't go back to 32 bit. If you are looking for a mind blowing experience from 32 bit to 64 bit then you aren't going to get it, but with a few small things I have found it to be a bit more efficient. I have had 0, thats right 0 incompatabilities with my 64 bit version of ubuntu, but there might still be some out there but things are switching over. With 3 gigs of ram you are getting close to the 32 bit thresh-hold of 4 gigs. Don't forget your video card's ram is counted in that 4 gigs as well if I'm not mistaken. Mind you, I only have 2 gigs of ram and so far in ubuntu unless I'm gaming I have never even used more then 1 gig of ram.
So it could be the placebo effect, but I have just found things to be slightly snappier then with 32 bit, but my startup did improove with 64 over 32. 
But in the end it doesn't really matter. If you plan on keeping this ubuntu install for a long while you might want to go with 64, but if you like to upgrade by clean install regularly then go for which ever. 
If you really want and feel ambitious you could download both and give them both a try through the live cd's to see if you have any incompatabilities. 
Best of luck. I hope this helped.

### Post by kpkeerthi on 2009-05-17
Java and flash has been a bit of a problem running them natively in a 64-bit Linux. But thats all in the past. Both these applications are now available for 64-bit linux and can run natively without any additional compatibiliy layer or emulation of any sort.

As for the performance for general usage, it will not make any difference whether you run 64-bit or 32-bit. But tasks such as audio and video encoding run noticeably faster in 64-bit. 

You can find more about people's experience with 64-bit in its own forum here: [http://ubuntuforums.org/forumdisplay.php?f=343]("http://ubuntuforums.org/forumdisplay.php?f=343")

I'd say go for it. :) (64-bit that is)

### Post by 3rdalbum on 2009-05-17
> **akimatsu123 said:**

Under normal use; heck, under ABNORMAL use, Ubuntu will not be able to fill up your 3GB of RAM, even as cache (unless you're ripping a DVD, which will always fill up your cache). Losing a couple of hundred megabytes to longer numbers and longer memory addresses will not impact on your performance.
