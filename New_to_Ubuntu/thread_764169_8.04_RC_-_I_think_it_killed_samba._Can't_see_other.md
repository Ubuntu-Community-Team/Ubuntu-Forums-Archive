### Post by Esben Kramer on 2008-04-23
For people with the same problem, I found this:

> Why does it work in an unconfigured live session but not in the installed Hardy environment?

Okay I'm not familiar with what GNOME does to browse Windows shares, but my guess after a quick look is that Nautilus uses gnome-vfs to access SMB shares. gnome-vfs exists in GNOME 2.22 (which is likely in hardy) but was deprecated in favour of an equivalent new feature in GTK 2.12 called GIO.

On my Debian unstable system, the package libgnomevfs2-extra is the one that contains a gnome-vfs SMB module (in /usr/lib/gnome-vfs-2.0/modules/lib* smb.so). This library links against libsmbclient.so.0, which is provided by the libsmbclient package, part of Samba.

If low-level mounting works for you, and the command-line smbclient tool also works, then the problem could be in gnome-vfs or possibly in Nautilus if it has been ported incorrectly to use GIO. I would check the file ~/.xsession-errors before and after trying to browse a share, to see if any errors are being printed by Nautilus when it does so.
