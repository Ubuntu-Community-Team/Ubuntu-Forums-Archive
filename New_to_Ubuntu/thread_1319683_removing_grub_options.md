> **[COLOR=Navy][SIZE=3]Removing Entries from Grub 2[/SIZE][/COLOR] **
Entries should be removed by editing or removing files in the /etc/grub.d folder. The [COLOR=DarkRed]*/boot/grub/grub.cfg*[/COLOR] file is read-only and should not normally require editing.[SIZE=3]**[COLOR=DarkRed]Automatically.[/COLOR]**[/SIZE]
[LIST]
[*]**Too Many Kernels?** Kernels removed via Synaptic or with "apt-get remove" will automatically update [COLOR=DarkRed]*grub.cfg*[/COLOR] and no user action is required.
[LIST]
[*]In Synaptic, type the kernel number in the search window at the upper right (for example - 2.6.28-11).
[*]Find the "linux-image" and "linux-headers" files for the applicable kernel (example - linux-image-2.6.26-11 or "linux-image-2.6.26-11-generic).
[*]Right click and select "Mark for Complete Removal" and then press the Apply main menu button.
[*]The kernels will be removed from your system and from the Grub menu.
[*] If you are not sure of the kernel you are currently using, in a terminal type "uname -r".
[*]Many users keep one previous kernel on the machine which previously ran without problems.
[/LIST]
[*]Other Operating Systems which have been removed from the computer will also be removed from the menu once "[COLOR=Navy]*update-grub2*[/COLOR]" is run as root.
[*] To prevent one of the */etc/init.d* files from running, remove the "executable" bit.
[LIST]
[*] Example: If you don't want to see the "Memtest86+" entries, run this command:
 	Code:
 	sudo chmod -x /etc/grub.d/20_memtest86+ 
[*]Run the *update-grub* command to allow the changes to be incorporated in grub.cfg
[/LIST]
 
[/LIST]

### Post by ajgreeny on 2009-11-08
I have said before that I find grub2 very difficult to understand as far as configuring the boot menu is concerned, and the more complicated your partitioning and operating system set-up, the worse it gets.

The advantage of it is that it should find all the kernels that are installed on all the disks of the machine, and therefore allow you to keep the menu updated by running **update-grub.** The disadvantage, however, if you have several disks, each with more than one OS, as I have, and in each linux distro, three in my case, there is more than one kernel, your grub menu gets so long it almost can not be displayed on a single screen.  The old system of legacy grub that allowed you to comment out lines from menu.lst seemed so much simpler and user configurable to me, but perhaps that's just because I still haven't sorted out how to deal with grub2 properly.

At the moment I am saved from all this as I this as I still have 9.04 as my main system and have 9.10 only on a test partition.  So when I installed 9.10, I ran 9.04 from grub2 menu, and restored grub from 9.04, and simply edited the menu.lst to include 9.10.  That was simple to do as far as I was concerned, but that may be because I have now become so familiar with legacy grub that I don't have to think too hard about how to configure and use it.

### Post by drs305 on 2009-11-08
Just found this thread. 

luvgirl12345, you have several options - they all have plusses and minuses. 

For a really simple menu for your mom, you can make a 06_custom menu so it shows up at the top of the list. You can have as many or as few items in it as you want. And you can have that as your ONLY items, or you can have them at the top, and then the normal items down below where your mom won't have to look. 

Since you have already shown us all your current options, if you tell us specifically which ones you want your mom to see we can probably construct one for you. While I am a proponent of learning how a system works (within reason of course), some people don't. Grub 2 provides the ability to tweak the menu to your heart's content (or your brain's complete overload, depending on how you view it). By the way, there is a link to "Grub 2 Tweaks" that would probably do what you want, but I wouldn't recommend it to you - your head would probably explode half way through. It should be marked "For Geeks Only".

Alternatively, as has been mentioned, there are ways to selectively turn off various things such as memtest, recovery modes, etc. And removing older kernels will also reduce the size of the menu.
