### Post by perspectoff on 2010-09-05
> **Rasa1111 said:**
> No, not stable enough. 
Use 10.04. 
congrats on a new machine. :)

I never install a new version of Ubuntu until 1 month after the release.

I have never seen a release of Ubuntu without a major bug discovered within the first month of release, whether it is a wireless problem, a temperature sensor problem, a HAL problem, a log problem, a splashscreen problem, and on and on.

If you happen to have the same hardware as the Ubuntu masters of the universe, your system will likely work at release time. However, for most of us that might not have the most recent hardware, there is usually a bump or two at release time.

Stick with Lucid for now. My splashscreen still doesn't work, 4 months after release, but that is minor. I had to replace Network Manager with Wicd, but that also is minor and for me and is a longtime problem with Ubuntu (I'm not sure why they are so tempted to use Red Hat solutions for Debian systems).

Other than those two chronic problems, all my systems on Lucid are quite stable and snappy. Further, a lot of applications are now available for Lucid (an example is the BigBlueButton teleconferencing server). It is a pain to make packages and complex applications work on newer versions for about 3 months after a new release (as was true with BBB).

Of course, if you are not a real user and don't really care about running applications, but are instead a fiddler, tweaker, and knob-turner, then it doesn't really matter what version you use, does it?
