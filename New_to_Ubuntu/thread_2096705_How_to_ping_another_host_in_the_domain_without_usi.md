### Post by papibe on 2012-12-20
Thanks.

Add this lines to /etc/resolv.conf:
```
domain test.lab
search test.lab

```
That should add the domain test.lab to single names like 'remote_host'. Test this again by doing:
```
nslookup remote_host.test.lab
```
Now it should give you the correct address.

Since some changes were made to the resolver package, I don't know at this moment if this would be permanent after you reboot.

In the last case, you could add extra settings to your /etc/network/interfaces. For instance
```

iface eth0 inet static
    address x.x.x.x
    netmask y.y.y.y
    gateway z.z.z.z
    [B][COLOR="Red"]dns-nameservers 10.180.136.47 10.180.136.21
    dns-search test.lab[/COLOR][/B]
```
Let us know how it goes.
Regards.
