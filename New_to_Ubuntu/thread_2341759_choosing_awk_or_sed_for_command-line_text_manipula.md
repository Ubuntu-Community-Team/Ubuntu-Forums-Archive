### Post by TheFu on 2016-10-31
I'd use perl. You can use 'cut'.  Getting 7 characters starting at column 9 is easy for either tool (perl or cut).  I suspect sed is NOT the best tool if that is what you want.  I made a decision about 20 yrs ago to learn perl instead of awk, so can't say much about it, but I suspect it can do it.

If you post the exact before and after characters with about 5 different examples, so it is clear (the description is not), perhaps we can understand better?

The total number of times this needs to be done and how many files are to be impacted would also matter.  Creating a new process every time will be slow, so perl would be much faster, probably.  It would not need more than 1 process, thus avoiding overhead of bash calling cut/awk, then calling another process.

If you just want a recursive search and replace inside a group of files, perl is definitely my tool of choice if there are over 1000 files.  For lesser numbers of files, don't think the tool would matter.

### Post by steeldriver on 2016-11-12
EDIT: sed version (range corrected for apparent discrepancy in your input):

```

sed -e '17,24H' -e '24G' file.xml

```


If you really want to duplicate a fixed (numbered) range of lines, I'd suggest ed/ex/vi e.g.

```

$ ed -s file.xml << \EOF
15,22y
23x
,p
q
EOF
<?xml version="1.0" encoding="ISO-8859-1"?>
<rss xmlns:media="http://search.yahoo.com/mrss/" xmlns:atom="http://www.w3.org/2005/Atom" version="2.0">
<channel>
<atom:link rel="hub" href="http://pubsubhubbub.appspot.com"/>
<link>http://</link>
<title></title>
<description>updated date </description>


<image>
      <url>http://png</url>
      <title></title>
      <link></link>
</image>


<item>
<title>audacity-lin-re503358.zip</title>
<link>http://.zip</link>
<guid>http://.zip</guid>
<description>
10 November 2016 03:20</description>
</item>


<item>
<title>audacity-lin-re503358.zip</title>
<link>http://.zip</link>
<guid>http://.zip</guid>
<description>
10 November 2016 03:20</description>


</channel>
</rss>

```

However, you can see the downside - if you get the range wrong (as you appear to have done in this case - the lines you actually want to copy seem to be **17-23**) you will end up with invalid XML

The best solution IMHO would be to use a real XML parser - there's the standalone xmlstarlet, and both python and perl have XML parser modules IIRC. Failing that, you could use awk or perl in paragraph mode to duplicate the matching record, for example

```

$ perl -00 -pe 'print if /<item>/' file.xml
<?xml version="1.0" encoding="ISO-8859-1"?>
<rss xmlns:media="http://search.yahoo.com/mrss/" xmlns:atom="http://www.w3.org/2005/Atom" version="2.0">
<channel>
<atom:link rel="hub" href="http://pubsubhubbub.appspot.com"/>
<link>http://</link>
<title></title>
<description>updated date </description>

<image>
      <url>http://png</url>
      <title></title>
      <link></link>
</image>

<item>
<title>audacity-lin-re503358.zip</title>
<link>http://.zip</link>
<guid>http://.zip</guid>
<description>
10 November 2016 03:20</description>
</item>

<item>
<title>audacity-lin-re503358.zip</title>
<link>http://.zip</link>
<guid>http://.zip</guid>
<description>
10 November 2016 03:20</description>
</item>

</channel>
</rss>

```

There shouldn't be any ambiguity, since there's only one `<item>` in your file.
