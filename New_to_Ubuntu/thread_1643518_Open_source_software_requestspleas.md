### Post by Temposs on 2010-12-12
The best thing to do if you want to start a new project is think about what kinds of programs you already use and make a list of those. Then pick one of those kinds of program you think you could improve on, and start writing that.

There's no sense in writing a program for free unless you're going to actually use it a lot and benefit from it. That will make it fun and motivate you better than choosing a random requested idea.

If you really want to make something yourself, why not install Quickly? It'll help you get a Python app knocked up, and will do all the packaging, licensing, and boring stuff and get it into a PPA so people can start testing it for you.
