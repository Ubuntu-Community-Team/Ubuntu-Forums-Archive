### Post by perspectoff on 2010-09-20
To tell the truth, I never run Kubuntu and Ubuntu on the same partition. I always get the same problems you do.

Despite what everyone claims, there are some competitions for resources between the two desktops, even if only one is running. This is because installation of one desktop overwrites settings previously installed by the other desktop.

I haven't said anything for the past few years because I have been drowned out by those who insist it can be done, but I personally am never completely successful.

It's ok for demos, perhaps, but not for everyday usage.

Instead, I always install independent copies of Ubuntu and Kubuntu (or whatever) in completely separate partitions (or virtual machines, or bootcamps, or whatever).

There are gaps in the Kubuntu-desktop for which I use Gnome-based applications, but those applications usually don't have any competition from native KDE apps. Similarly, many individual KDE apps can be used in the Gnome Ubuntu desktop. The problem is that when you install two complete desktops side-by-side, there are dozens to hundreds of modules, many with configuration files, and they can trip over each other.

Over the versions Ubuntu has tried to minimize these conflicts, but I still encounter them somewhat routinely.

For long term usage, it is better to choose one desktop and stick with it. Try each one out using the LiveCD for each (Ubuntu and Kubuntu, or Xubuntu, or Lubuntu, or Mythbuntu, or Ubuntu Studio), for example, just to see how each one looks. Then install just one. Then just add the apps from the other desktop that you like, one by one.
