### Post by bcbc on 2010-11-16
When you multiboot windows os's it places the newer version's boot files in the old version's partition. So, likely you destroyed the boot files of win7 by overwriting vista. The boot flag is not even set on the win7 partition (required for windows booting) so likely you never booted win7 without going through vista.

Try inserting your win7 disks and hit repair...

### Post by Dogjen on 2010-11-16
Finally managed to fix it :P.

Had to make Windows 7 disk Active which I did using [http://www.sevenforums.com/tutorials/71432-partition-mark-active.html](http://www.sevenforums.com/tutorials/71432-partition-mark-active.html) method 2. Windows repair then recognised my Windows 7 and tried to fix it. Rebooted but came up with error for BOOTMGR. Ran windows repair again and selected fix start up issue. Reboot and Windows started up ok. 

Used sudo update-grub to add windows to startup selection.

Thanks for the help.
