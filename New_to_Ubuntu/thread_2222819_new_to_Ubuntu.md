### Post by grahammechanical on 2014-05-08
Have you ever seen a drawing of a learning curve? There is something strange about it. The steeper the curve the higher we get in the shortest time. With a gentle learning curve it will take us longer to get higher/proficient in what we are learning.

Think about changing your main work application for a competing rival? How hard would it be to learn the ways of the new application? You already know what the application is supposed to. You just need to find out the ways it will let you do what you want. It is no different when changing operating systems. In fact a lot of people will have difficulty changing from Windows XP to Windows 8 but if they buy a more modern computer with Windows 8 pre-installed they will have to climb the learning curve.

It makes no difference if we have never used a computer before or if we have only used one company's operating system and applications. We still have to learn how to do stuff. This is life.

Of course, life will be easier for you if you buy a machine with Linux/Ubuntu pre-installed. Then you will not need to get certain things working. You will not find any of that stuff complicated. Not at all. The supplier would have sorted all that stuff out. Just like they do with Windows pre-installed. Sorting out complicated stuff is the price we sometimes pay for a free to download and use computer operating system. 

Of all the Linux distributions it is Ubuntu that has the reputation of being most suited to the new user. Ubuntu had that reputation 8 years ago when I decided to scrap my Windows 98 machine and switch to Linux. Ubuntu's reputation for being better suited to new users was the reason when I chose Ubuntu. I tell you this, Ubuntu is even more new user friendly and less Linux geek than it was 8 years ago.

Regards.

### Post by Kingsrookie on 2014-05-08
I recently took the time to switch my main office to Linux. My business still has a few Windows computers around that are slowly going to be converged to Ubuntu. Being in the computer repair business, it is a bit easier to learn the new operating system but grahammechanical said it perfectly. I have heard people time and time again complain about Windows 8 because they have to relearn everything again because of the changes from XP. It is only difficult if you do not spend the time to learn it.

Personally, the learning curve is not terrible. Programs act differently and some are O/S dependent, but everything that my Windows system did, Ubuntu does. I've switched from QuickBooks to PhreeBooks, Office to LibreOffice, Publisher to Scribus, etc.
There are some things that I would like to see on Ubuntu but unless people adopt the O/S and give it a chance, there will be slow changes. Give it a shot. Dual booting would be a great way to go in case you can't afford the downtime. It took 3 weeks to get some of my computers converged over to Linux where I was using it frequently because I personally had to learn it. Now I am very grateful that I had and will be offering it to clients as an alternative to Windows 8.

It is possible, very very possible to run solely on Ubuntu, just be prepared to do some research to get some quirks working. There are forums for help if need be and Ubuntu has a great active community from what I've seen. Even this site, before I started posting, has given me neccessary answers to succeed.

### Post by oldrocker99 on 2014-05-08
Welcome to the use of a superior OS. I entered the GNU/Linux world cold with Ubuntu 8.04 six years ago, and, although I have a dual-boot desktop machine with Windows 7, every time I boot into Windows 7, I become greatly annoyed at its unfixable quirks, its slowness, its inability to update files "when they are in use," when GNU/Linux systems stop the process which is being upgraded, and then start it again (what a concept!), the necessity of using malware protection because Windows is chock full of security holes, etc, etc.

Even though I only boot into Windows to play games, because I do everything else with GNU/Linux, I still get annoyed at MicroSloth's assumption that, because you are forced to pay for their lousy OS when you buy a computer, you'll be happy with their lousy OS. Now, with UEFI, computers are being sold that REQUIRE you to run Windows 8 (gak) unless you're savvy enough to get into the BIOS and turn it off. Fie on their house, and, for that matter, Apple's too. 

Long before I started using an open-source OS, I was using open-source software: Audacity, MIXXX, Open Office, GIMP, Firefox. The majority of cross-platform software is developed on GNU/Linux anyway, and the new versions come out for our platform first for obvious reasons.

Yes, there's a learning curve with any new OS. iPad users who come to Android have to learn how to use it; Windows users who come to Macs have to learn how to use it. The big difference with GNU/Linux is the community of users, who are (usually) very eager to share their knowledge. Practically every question I have had in the last six years has been answered in this very forum, and, otherwise, there's always Your Best Friend: Google.
