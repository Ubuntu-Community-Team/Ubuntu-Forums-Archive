Create a asound.conf file...

```
gksudo gedit /etc/asound.conf
```

...and add the following content and save it:

```
pcm.pulse {
type pulse
}
ctl.pulse {
type pulse
}
pcm.!default {
type pulse
}
ctl.!default {
type pulse
}

```

Reboot and check if that helps.

### Post by cwmoser on 2010-07-02
SOLVED!!  Finally got sound working.

Likely to be a conflict with the nVidia drivers.
[B][U]
This is what I did:[/U][/B]

sudo apt-get purge oss
sudo apt-get purge alsa

sudo apt-get install alsa

I removed deactivated the Hardware drivers - i.e. System -> Administration -> Hardware Drivers.

Downloaded off [www.nVidia.com](www.nVidia.com) this driver:  
**NVIDIA-Linux-x86_64-256.35.run**
Note:  I'm running Ubuntu 10.04 64-bit version.

booted my PC in recovery - via the grub menu
dropped to command prompt with networking

telinit 3

sudo  sh  ./NVIDIA-Linux-x86_64-256.35.run

rebooted

setup my nvidia settings
setup COMPIZ

Test Sound OK now.

Carl

This usually happens when you are using the 32bit plugin on a 64bit system.

To fix that you need to edit the npviewer file. Open it with the command below:

```
gksudo gedit /usr/lib/nspluginwrapper/i386/linux/npviewer
```

Then add the following line before the last line of that file:

```
export GDK_NATIVE_WINDOWS=1
```

The file content should look like this:

```
#!/bin/sh
TARGET_OS=linux
TARGET_ARCH=i386
export GDK_NATIVE_WINDOWS=1
. /usr/lib/nspluginwrapper/noarch/npviewer
```

Save the file and restart the browser.
