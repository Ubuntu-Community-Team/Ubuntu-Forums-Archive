### Post by lile001 on 2010-01-16
I am having a difficult time controlling landscape vs portrait printing in Openoffice 3.01.  The program seems to randomly decide, despite all the settings I use, that a particular document will always be landscape.  


1.  Printed a document, came out landscape. This was a .Doc file from a windoze system. 
2. Went to Print Preview, appeared to be portrait.  Printed from Print Preview, came out landscape. 
3. Tried printing again, checking printer settings in print dialog.  Print is set to portrait.  Print came out landscape. 
4. Set printer to landscape.  Document still came out landscape.  

Next document I printed came out portrait as expected from print preview. This was a document I created just now, saved in .doc format. 

Next document I tried came out landscape. This was a .doc imported from Win-Bloze.    From Print Preview, which shows the document portrait,  if I print the document from there, it comes out portrait but the font is about 1/3 the proper size.  If I print from normal view it is always landscape.  

Next document I printed came from  the same win-doze system, another .doc file.  Printed correctly.  

Loaded one of the bad documents, Select all, copy, open a new Openoffice document, paste.  This prints correctly.  

Didn't have any time for more experiments, since the program is obviously not Obeying My Commands and I actually have to get some work done today.  

What is going on here?  Is there some hidden code in a document that can force it to be landscape?
