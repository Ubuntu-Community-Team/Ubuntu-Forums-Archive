### Post by oscarivera9 on 2011-09-05
I believe I can help you with this since I definitely have experience in the matter.  I have just joined Ubuntu Forums today but I have been a member of Launchpad for quite a while and the Ubuntu community over at Launchpad sure helped out big time when I updated from 10.10 to 11.04 only to have a system that would not boot at all.  Now is my turn to give back a bit of what was freely given to me.
What I learned which should definitely help you is this, install the current 11.04, trust me on this.  Going with the 10.04 LTS is fine as well, but for me it's just better to remain current rather than remain with the same setup for two years. 
Now, as far as the next version (11.10) which was mentioned by someone else earlier, when the time comes to upgrade, don't do it right away.  Like I told you before, I had problems going from 10.10 to 11.04 because I upgraded right away.  Instead, what I recommend you do is to create an ISO image of the new version and then try out the Live CD to make sure that your setup is compatible with the new release and everything should be fine.  I have only been using Ubuntu 11.04 for about two weeks because my old PC (and I mean 'old'), had some hardware incompatibility with 11.04 which is why it wouldn't even boot up.  My new PC that I just recently built is more than compatible to run Natty.  In fact, I made sure to make my new PC Linux friendly when I was deciding on the parts that I would use .  I decided to go with an all AMD build because AMD seems to work really well with Linux.
I hope my answer helps you a bit, and I hope it wasn't too long for you.  If you want an example as to why I prefer to not go with a Long Term Release, keep reading, otherwise I believe I have answered your question.

For example, since you are new to Ubuntu, if you were to go with the LTS  release you would have yourself a few headaches figuring out how to  upgrade from Mozilla Firefox 3.6 to 4 then 5 then 6, until you reached  6.0.1 which is where we are now.  The reason behind all of this is that  back when Ubuntu 10.04 was first released, Mozilla Firefox was at  version 3.6 and it had in fact been there for years!  Back then we had  no idea that eventually the folks at Mozilla would decide to release a  new version every three months; so Mozilla Firefox was not something  that was updated through the Update Manager like it is right now.  When  Mozilla Firefox 4 was released I had to use the command line quite a bit  to upgrade to Firefox 4, it took me longer than five minutes just to  get to 4.  With Natty, that is no longer a problem because updating to  the newest Firefox version is done automatically.  As a matter of fact,  earlier in the day my system told me that Firefox had to be re-started  because an update had been made, after it restarted I realized that the  update was going from Firefox 6.0 to Firefox 6.0.1.  The best part was  that I didn't have to do a single thing except for agree to restart  Firefox, which took me less than a minute.  

Whatever you decide, I'm sure you'll enjoy ):P
