My definition of a server, a computer that provides services to other computers.

fileserver - other computers can connect to it and access files stored on the server, sometimes upload their own content as well

webserver - well it serves web pages to other computers

dhcpserver - it assigns ip addresses to computers requesting them

dnsserver - translates names into ip addresses for other computers

backupserver - pretty much a file server but it's exclusively for making backups of other computers

etc... etc... as you can see "server" is a very broad term.

### Post by Jr. on 2008-08-26
Every server is essentially the same. The "type" just depends on how it's used and what it's used for. A single server can handle mail exchange, http port forwarding, file server, software distribution, DNS service, ftp service, and DHCP service. It'd be incredibly inefficient to do so, but it is possible. 

A server is a hub. Many computers connect to it to accomplish different tasks. There are two ways to connect to a server. Locally (LAN), and Externally (WAN). 



**Small Business Example**
3 Servers, each with a dedicated purpose

A file server (Windows Server 2003): A "user" logs on to the local network at a workstation connected to the server (instead of "logon to this computer" on windows logon screen, "logon to SmallBusiness1"). There they have local access to any files, software, DHCP service, internet access, shared printers, firewalls ect. A user can log on to any workstation on the LAN and their settings are automatically applied. Each user can also have different privileges and access approvals.

A DNS server: The port 80 is the standard HTTP port. When you type "http://www.ubuntu.com" it actually goes through a DNS to transfer that to 91.189.94.249:?? (I'm not sure what port it is). That IP address alone on port 80 goes to [www.canonical.com](www.canonical.com). So [www.ubuntu.com](www.ubuntu.com) is hosted on the same server, but forwarded through a different port. So a small business may have their website hosted on port 80 of their DNS server and whoever they register their domain with forwards it to their IP on port 80. Also, maybe on port 1010 they have forwarded access to all their security cameras. Though not forwarded through DNS, it can be accessed by typing th IP address of the DNS server followed by ":1010" to access through port 1010. Then a logon screen will allow external access to the cameras and DVR.

A Mail Server (Windows Server 2003 + Microsoft Exchange): This server simply manages and stores all emails/accounts for their Domain. Their domain registrar points an MX Record to the Mail server. Then any email sent (@) to that domain (@yahoo.com) will be forwarded to the Mail server and then accessible by users on the local network, and if setup for WAN, accessed by Outlook Online. 


All servers are the same, they are computer. Hardware wise, same as your desktop but typically on a larger scale (quadcores, terabytes, blades). Same 5 components required for BIOS though (CPU, MOBO, RAM, PWR, GRAPHIC). The same hardware set up could be used for all three servers I listed, however, they are typically tailored to their intended use.
