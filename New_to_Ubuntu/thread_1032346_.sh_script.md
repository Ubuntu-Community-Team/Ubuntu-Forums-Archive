Hi All

I have a lot of files I am trying to organize and some of them do not have file extension. I know that quite a few of them have are bash scripts and I need to rename them to *.sh.  I know that I can do that to individual files by just ```
mv file file.sh
```, but there's quite a lot of them there, so I need to create a script to reduce the legwork.  Any thoughts?

### Post by computermacgyver on 2009-01-06
There's probably a more efficient way, but I like to simply string together ls, sed, and sh using the pipe (|)

```

ls -d * | sed 's/\(.*\)$/mv "&" "\1.sh"/' | sh

```
will add the extension.sh to all the files in the current directory. To see what it will do before actually making changes remove the "| sh" from the end and see the commands it generates. ls just lists the files, then sed creates mv commands as in your post above. Directing the output back to sh actually executes the commands.

You also should be able to do something with the find command and its --exec flag.

### Post by amauk on 2009-01-06
you could test for a shebang entry
all bash scripts start with #!/bin/bash
will prevent non-bash scripts from being renamed

not tested,
but something like this

```
#!/bin/bash

export IFS=$'\n'

FILES=`find "/top/dir/" -type f`

# Loop through all files
for FILE in $FILES
do
	# check if file has extension
	FILENAME=`basename "$FILE"`
	FILENAME=`echo "$FILENAME" | grep -v '\.'`

	# Current file doesn't have extension
	if [ -n "$FILENAME" ]; then

		# check first line
		FIRST_LINE=`head -1 "$FILE"`
		
		SHEBANG=`echo "$FIRST_LINE" | grep '/bin/bash'`

		# Shebang present, rename
		if [ -n "$SHEBANG" ]; then
			mv "$FILE" "${FILE}.sh"
		fi
	fi
done
```
