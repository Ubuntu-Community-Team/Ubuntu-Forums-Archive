### Post by Mark Phelps on 2013-05-14
You shouldn't need to install Radeon drivers because they are installed automatically for you when you initially setup Ubuntu.  Anything you do video driver-wise after that, is likely to hose up the display, unless you know exactly what you are doing.

We need to know what version of Ubuntu you are using, and the make and model video chipset you're using.

As to whether to use MS Windows or Linux, the best approach is to use what works best for you.  Too many folks come here thinking Ubuntu is a FREE version of MS Windows (which it is NOT), and then start going on about how to install and run all their favorite Windows apps -- which they won't be able to do.

Ubuntu provides an ALTERNATIVE environment to MS Windows -- using different apps, and in some cases, very different approaches to getting things done.  IF you're willing to take on the challenge of learning the differences, you will have a satisfying time; if instead, you want a clone of MS Windows, you are doomed to being disappointed.

### Post by Impavidus on 2013-05-14
I am not here to tell you why you should use Linux, your free to use anything else, but if you choose to use Linux we'll help you to get started.

Concerning that command line: Sometimes you have a problem that can only be solved using the command line. This is rare. In most cases problems can be solved via the graphical interface, but as the graphical interface is so configurable we all have different interfaces, making it far easier to help people using terminal commands. Even if we have the same interface, saying "copy and paste this command" is easier than telling: "open program X, find the button with the little yellow thingy on it, click it, find the option labeled 'bar', but in your language it may be labeled differently, it should be about the seventh in the list, depending on the packages you have installed, and then toggle that option."

### Post by mastablasta on 2013-05-16
Guild wars has GOLD on Ubuntu 13.04 and wine (play on linux is a wine GUI frontend)

> 
**What works**
Everything, as far as I can tell.
(Performance and stability is much better than on previous versions of WINE) 


**What does not**
Installer/patcher crashes frequently and is a pain to use. 

**What was not tested**
Running with an AMD or Nvidia GPU (Intel HD 3000 was tested with shocking success). 

**Additional Comments**

I recommend that you install from the DVD or copy over an install from another computer rather than trying to use the patcher.You will need to install the following packages:* wine gecko 1.7 x86_64 (or newer)* wine mono28 (or newer)Also, you must run the game with the argument "-dx9single".You may also need to disable pulseaudio.


windows games don't work well on other systems. unless they have a port. sometimes they work in wine (which is like interface that brings some reverse engineered libraries so you can run windows programmes on linux), sometimes they work better in wine that native. but mostly they dont' work at all or with some limitations. especially problematic are various MMO widnows games where there are frequent updates.

There is also payable wine called Crossover with some more tweaks and such. maybe has better compatibility with some programmes.

as for iDevices - i do not know as i do not own them.

if you want to only explore the OS using its secure features without some impact on your games i suggest you try to install it in virtual box: [http://www.psychocats.net/ubuntu/virtualbox](http://www.psychocats.net/ubuntu/virtualbox)
