### Post by iracida on 2011-09-28
I'm trying to password protect a website. I'm getting a login window however it won't except my login name and password. 

Here's how everything is currently configured on the server:

In the default file located in /etc/apache2/sites-available I modified AllowOverride to All.

<Directory /var/www/>
                Options Indexes FollowSymLinks MultiViews
                AllowOverride **All**
                Order allow,deny
                allow from all
</Directory>

I reloaded Apache so the change would take effect.

I created an .htaccess file in the directory I wanted to password protect. The file contains the following

AuthType Basic
AuthName "Password Required"
AuthUserFile /var/www/sitefiles/ncate/.htpasswd
require testuser

I then created the .htpasswd file using htpasswd -c /var/www/sitefiles/ncate/.htpasswd testuser.

I edited the apache2.conf file and added the following to it:

<Directory /var/www/sitefiles/ncate>
AllowOverride All
</Directory>

I then restarted the server however when I go to the site it won't let me log in. The .htaccess and .htpasswd files are both readable. I"m using the correct password and not mistyping. Any ideas why I can't log in?

---

### Post by iracida on 2011-09-30
I figured out the problem. In the .htaccess file I had require testuser instead of require user testuser.
