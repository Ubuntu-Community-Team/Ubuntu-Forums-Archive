The clip (jumper) position makes a difference. Most drives have 3 setting Master, Slave and Cable select.  Need to figure out what the drive that works is set to. Depending on make and model of the drive the jumper positions will vary - If there is not diagram of jumpers settings on the drive then Google the drive model - you should find what you husbands drive is set to if it has no jumper.

Check your ide cable. 2 types. one type is  used when both drives are set to cable select - master drive - slave drive is determined by witch connector the drive is plugged into - connectors a usually marked - master is usually the end one.   THis type also usually has different colored connectors with MB connector being blue.

The 2nd type of ide cable is does not have cable select capability. the drives must be jumpered to master and slave. And the master goes in the end connector and the slave in the middle. 


Anyway dual booting with two hard drives - usually not a problem - even easier now that GRUB2 is being used. Biggest problem in the past was the initial GRUB install - it had to get the boot order from BIOS - sometimes BIOS got it wrong - so had to tweak GRUB a little.

### Post by meierfra. on 2010-01-22
You probably only had one  hard drive attached, when grub was installed and so the device.map is missing  an entry for /dev/sdb. Try this

```
sudo grub-mkdevicemap
sudo update-grub
```
