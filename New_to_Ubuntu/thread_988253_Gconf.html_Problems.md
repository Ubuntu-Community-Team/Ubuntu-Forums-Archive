### Post by cariboo on 2008-11-20
First off, do you have any idea what you are doing, or are you trying to follow some instructions from a web site? The first problem is that you are misusing root. Any command that specifies that you need to be a root you run using **sudo** which stands for super user do. If you su to root and then try to run the command as root using gksu you are going to run into problems.  Depending on which version of Ubuntu you are using, you use network manager to change dns.

As for gconf.html there is no such thing. I think what you are looking for is, gconf-editor which you can run as a normal user. to run gconf-editor press Alt-F2 and enter:

```
gconf-editor
```

### Post by doobiest on 2009-10-05
> **cariboo907 said:**
>  Any command that specifies that you need to be a root you run using **sudo** which stands for super user do. 


I almost don't care, but where did you get 'super user do' from?

The man page for sudo itself says 'execute a command as another user' for it's description, which doesn't necessarily imply super user.  I think super user is a stupid term anyway, it's the root user.

I'm pretty sure sudo is a play on words for pseudo, which would be more in line with the man page's definition of 'execute a command as another user'

However since sudo is derived from su, it would make sense to determine sudo's definition from su's definition.  su in my eyes is 'switch user' as the command isn't necessarily designed to switch to the root user, it is designed to switch to any user. Though if no user is specified it would default to root, makes sense.  But, the man page for su states 'change user ID or become superuser'. It does not make any reference that it stands for 'super user' or 'switchuser'.

Google suggests that the original definition for su was 'super user' and later became 'switch user' as it's functionality was expanded to change to any user ID. I cannot find anything concrete on either side.

Since the man page for su does have the term 'superuser' in it, I am willing to concede that su could stand for 'superuser' although I never would have thought superuser would have been a Linux term, it sounds kind of ghey to me, feels kind of windowsy or like I'm in that show Reboot.
