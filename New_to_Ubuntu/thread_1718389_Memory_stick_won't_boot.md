### Post by digitography on 2011-03-31
I'm posting this not so much for assistance, but to aid those that get the same issue. I'm also curious as to why, but that's a secondary question.

I had a problem with 2 - 16gb memory sticks, they had been working fine for nearly a year with pendrivelunux multiboot.

I then saw the latest option of YUMI and thought that's the way to go.

So put YUMI onto one stick to try out, worked a treat. Decided I'd go for a clean start, as I wanted some of the stuff I had put on, off, and other stuff on (hope you can follow this)

So ran YUMI a second time, and that is when the trouble started, wouldn't boot, so tried the other stick, still no good. I tried just about everything, including Fdisking the stick, still no good.

In a last desperate attempt to get it to work, I used the Ubuntu built in startup disk creator, and created a bootable USB Voila!
I then used YUMI again and it installed fine and boots fine.

Now if anyone can tell me what the hell happened I'd be happy to know, hope this solved a problem for someone else.
