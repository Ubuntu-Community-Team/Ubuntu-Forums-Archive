### Post by MG&amp;TL on 2011-12-01
Well, learn a little programming (at least so you understand stuff), then learn bzr and launchpad. You can then start submitting bug reports and fixing 'bite-size' bugs like spelling errors, or missing icons.

[http://doc.bazaar.canonical.com/latest/en/mini-tutorial/]("http://doc.bazaar.canonical.com/latest/en/mini-tutorial/")
[http://launchpad.net]("http://launchpad.net")
[https://help.launchpad.net/]("https://help.launchpad.net/")

Then familiarize yourself with debian packaging (this is the sticky, horrible bit)

[https://wiki.ubuntu.com/PackagingGuide/Complete]("https://wiki.ubuntu.com/PackagingGuide/Complete")

This can take some learning-and that's basic. 

You can then probably help a bit more. And understand the slightly weird and terrifying debian subdirectory

Learn more programming, and you can select a few pet projects to help with (probably in your programming language of choice), that you can contribute code for. 

I haven't got any further than this point, so I can't speak for that, but I hope I've made your life a bit easier than mine was.

Good luck and hope to see you around!

<this is how I did it, I imagine your path will be slightly different>
