### Post by drs305 on 2010-12-08
Since you mention video, at the Grub2 menu, highlight your normal Ubuntu entry and press "e" to edit it. 

On the "linux" line, after "quiet splash" add **nomodeset** and then press CTRL-x to boot.

If it boots to your Desktop, go to System, Administration, Additional Drivers and see if you can install the video driver for your card.

### Post by drs305 on 2010-12-11
> **John_Adams said:**
> Well, making these changes to the boot configuration causes me to boot normally into ubuntu, but the video is still choppy and really slow. Before the problem occurred, it was quite fast. Also, the boot menu still shows two options for recovery mode. How do fix that?

Using the 'nomodeset' option may be causing the video choppiness. Try reinstalling your video driver. 

Run "sudo update-grub" as well (more later) before rebooting.

When you reboot:
If you have been booting by editing the menu and removing all the video options and adding 'nomodeset', try booting without making any changes (leaving the video options and not adding nomodeset). See if it now works.

You say you still have two recovery options. Do you also have regular modes now available as well (after updating grub)?

You can see what your boot options will be available by using this command:
```
grep menuentry /boot/grub/grub.cfg
```
