### Post by Perfect Storm on 2008-10-12
**Screenlets** in synaptic package manager
Screenlets are small owner-drawn applications (written in Python) that can be
described as "the virtual representation of things lying/standing around on
your desk". Sticky notes, clocks, rulers, ... the possibilities are endless.

The goal of the Screenlets base-classes is to simplify the creation of fully
themable mini-apps that each solve basic desktop-work-related needs and
generally improve the usability and eye-candy of the modern composited
Linux-desktop.

**cairo-dock** in synaptic package manager
Uses cairo to render nice graphics, and Glitz to use hardware acceleration.
It's fully configurable and can be a taskbar too.
You can easily plug applets into it.

**awn-manager** in synaptic package manager
Awn-extras-applets is a collection of applets for avant-window-navigator. It
provides some extensions to the bar and some new features (like
applets in the gnome-panel).

**Kiba Dock** needs to be compiled. Most advanced "Mac" Dock out there.
Kiba Dock is a dock, a way to access programs and windows in an interactive manner. Probably the most common example today is the dock in Mac OS X. Kiba Dock is specifically made for Linux operating systems, running under either the GNOME and KDE destkop environments. However, unlike the other systems available, it uses the Akamaru physics engine to render complex interactions.

### Post by manimal347 on 2008-10-12
Cool apps?
Here's some of my favorites:

Ristretto: It's an imageviewer rather similiar to the one that Apple ships, but far more minimalist. Requires GTK2 and nothing else. Super-low alpha version number, but doesn't crash

Rtorrent: Control torrents in a terminal window. You'll keep leaching even if X borks. Needs no GUI.

Audacious: A great winamp clone. Plain GTK2.

MPD + Sonata: A bit harder to set up than audacious as it uses a daemon, but Sonata is a brilliant interface. It's straight GTK2.

EPDFView: Not as good as Evince, I think, but it's more modern than XPDF and avoids installing Gnome dependencies,

Abiword: A great SIMPLE text editor with all the features a home user wants. Only thing it fails at is footnoting. Think it depends on Gnome :(

IRSSI: It's likely the most popular Unix IRC client, right behind the equally good BitchX. Needs no GUI, has no GUI.

Amule: Ahem... it's good for obscure media content "liberation." Doesn't need Gnome backend.

Navigating Folders: Thunar, there's none better. It comes from XFCE, but XFCE has a really light backend. Very fast, and fully modern- perhaps more so than Nautilus. Good at things like thumbnailing.

BashBurn: Tired of K3B? Don't want a Kde backend install? Hate the incessant crashing? The only modern and reliable alternative I found was this ncurses program that guides you through the process with a decent text interface. You'll have to copy your files over into /tmp/burn with a file manager, but otherwise, it's very configurable, and works fine for me.

(my apps are a bit different because I'm running Fuxbox and don't have either the KDE or Gnome libraries. Can't guarantee everything's in Ubuntu's repositories either; I now run Arch.)
