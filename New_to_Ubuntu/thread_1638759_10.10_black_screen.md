### Post by sikander3786 on 2010-12-06
Reboot your computer. Press and hold down Shift key from Bios screen. Grub menu should pop up. Select Recovery (2nd on the list) then Drop to root shell and try to remove ATI drivers and reconfigure your graphics.

```
sudo apt-get remove --purge fglrx fglrx-amdcccle
```

```
sudo mv /etc/X11/xorg.conf /etc/X11/xorg.conf
```

This one might throw an error like file not found. Ignore and proceed to next one.

```
sudo dpkg-reconfigure -phigh xserver-xorg
```

```
sudo shutdown -r now
```

Let Ubuntu boot normally and see if you can get to the desktop this time.

### Post by drs305 on 2011-01-19
Would you tell me if this helps?

At the grub menu, press 'e'.
Do you have a line: 	set gfxpayload=$linux_gfx_mode
If so, use the DEL key to remove the line, then CTRL-x to boot.
Does that work?

If not, repeat, but add to the end of the "linux" line: nomodeset
Does that work?

There can be lots of reasons for the problem, but I had similar symptoms and solved it with the first method above.

Added: By the way, at the blank screen, you can reboot by pressing CTRL-ALT-DEL several times, then ENTER, even though you can't see anything.
