### Post by xycris on 2012-08-08
i think one of the problems solved by the 64bit architecture standard that amd introduced to the world is the ability to go beyond the memory cap of the 32bit architecture which is close to 4gb (actual figures that 32bit os with 4gb ram might report only 3gb ram).

64bit architecture increased this cap to 128gb (i think, needs further citation).

the other benefit is that processed chunks of data is now larger at one given process in time, so more output.

the downside is: some softwares might not work properly under the 64bit os despite the backward support for 32bit architecture offerd by the os.

but, nonetheless, many softwares nowadays have native ports to the 64bit even though some are still under the 32bit. the same effect when there was a big move to windows 95 back in the 90s, people might have not noticed the move in architecture from 16bit to the 32bit used by windows 95 for many softwares right away evolved to be a 32bit (maybe because it increases the productivity of their softwares paired the windows95 popularity).
