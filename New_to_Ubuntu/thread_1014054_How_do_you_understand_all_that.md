### Post by o.besner on 2008-12-18
18 months ago I had never even used Linux. Now I'm on a dual-boot and the only reason I still have XP around is for Fallout 3.

This is how I learned to handle myself. It's in chronological order, so it should fit right in.

1) I started by googling every word specific to the linux world that I didn't understand (root, kernel, bootloader, xorg, home). By doing this I could translate what I knew in Windows ---> Linux

2) Second step was finding programs that I liked, and learning how to use them. Totem is crap. Don't get me started on rhythmbox. Which programs do you use often ? Here's an example : I tried Totem, Xine, Gnome-Mplayer and VLC to play movies on my computer. After a week I kept my favorite, Mplayer, and I learned how to use it. 

3) Third step was to understand how a computer starts. How does it work ? From pressing the button to seeing my mouse. So I searched for BIOS --> Bootloader --> Kernel --> udev --> X --> HAL and fam --> Gnome. An hour or two and you should be done.


4) For every tutorial/guide that I used on the forum, I learned what the commands meant. Just google parts of the command line! This is how you lean the basics.

5) Compile a program for its source code. It helps understand the difference between source code and a binary, and the difference between /usr and /usr/local. Start small : compile something small like Sound Converter or yasm. Maybe a gadget that will go on your desktop ?

6) Install an tougher distro, on an old machine. Choose something like Gentoo, Arch Linux, Slackware, even Debian. Since you have to build the system from scratch, it's a good test! It also fills the gaps between all that you've learned precedently.

### Post by sofasurfer on 2009-01-23
I wrote a post very similar to yours at one time. Since then I have
 learned that I will never learn this stuff as fast as I want and much of it I will never learn. That is why we have forums. They are a clearing house for information. Learning technology related things is not like learning every day things. Technology is so complex that it takes communities to store all of the relevant information. Just keep asking questions.

The most helpful information I can give you right now is the following. It is a set of instructions for saving a list of all of your installed packages so that if you reinstall (which I do VERY frequently) you can have all of your packages reinstall just exactly like they were before. And its real easy. Here it is...

```

Reinstall packages using package list

Taken from http://www.arsgeek.com/2006/09/19/ubuntu-tricks-how-to-generate-a-list-of-installed-packages-and-use-it-to-reinstall-packages/
Package list should have been created and saved to file and stored to backup disk by using this command...
dpkg --get-selections | grep -v deinstall > /media/disk/backup/installed-package-lists/NAME OF PACKAGE LIST
This creates the list and saves it to my backup disk.
To use this list to recreate the Linux system follow these steps...
1) Reinstall base system
2) Copy nessessary files from backup -sources.list copied back to etc/apt/
3) If nessessary open network config window and enter encryption key. You now have internet access.
4) sudo apt-get update
5) sudo apt-get dist-upgrade
6) sudo dpkg --set-selections < /media/disk/backup/installed-package-lists/NAME OF PACKAGE LIST
7) sudo apt-get install dselect
8) sudo dselect
9) when prompted, choose to install packages

Your system is back to normal now.

```
