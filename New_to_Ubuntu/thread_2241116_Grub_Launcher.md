### Post by ajgreeny on 2014-08-24
My opinion; stick with and use grub.

Grub2 in particular is very flexible and once you understand how it works, it is possible to do a lot of things with it.  You can, for example, boot a non-extracted .iso file which is very useful.  I have a USB stick with 7 different live .iso files on it, all of which will boot from the grub menu that appears when booting to that USB.  It is also possible to boot to a similar .iso file in a partition on a desktop machine which already has Linux and grub on it, using the grub that is already installed.

I admit that in my nerly ten years of using Linux, I have never used any alternate bootloader, so I may be biased, but that is my opinion.

### Post by grahammechanical on 2014-08-24
Better? What is there to be better about? Grub is the boot loader used with Ubuntu and Ubuntu developers keep it in working order. From time to time we get upgrades to Grub. Whatever you replace Grub with will not get upgrades from Ubuntu developers.

But it is your machine. You can do with it what you want.
