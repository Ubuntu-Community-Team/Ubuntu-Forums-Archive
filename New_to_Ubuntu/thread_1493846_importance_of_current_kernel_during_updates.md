### Post by cloyd on 2010-05-26
This just occurred to me the other day.  A while back, I was having problems with marginal wireless signals. I did a backport update, and everything started working fine. Later, I got a new kernel in an update. The new kernel is fair, but not as good as the previous "backport updated" kernal on the wireless.  So, the simple solution is, when starting up computer in an area with a marginal signal, use that kernel.  Where there is a strong signal, use the newest kernel.  

But my question is, when I get updates, does it matter which kernel I am using? Should I always go the the newest kernel while getting updates, so that the new kernel has the latest security updates? Or does it not matter?   Thanks in advance.

### Post by mikewhatever on 2010-05-26
Regardless of what kernel versions you are running, you should still get updates. That out of the way, why don't you install backport modules for the newest Ubuntu kernel and run that?
