### Post by egeezer on 2015-02-22
Most of the people who try Linux and wind up liking it are those who are self-starters: they are people who like to find their own way, work by trial and error, and would rather experiment than read books about it.

As a former academic, I learned that the hard way: all the reading I did never got me as far along as the trial-and-error  method, and it took a LOT of errors!  

That is hardly an accident: all forms of Linux, and Ubuntu especially, develop so quickly they are a fast-moving target for anyone trying to document the "right" way to do things.  If it weren't for all the contributors to this site and to both the "official" and "community" wikis, it would be a near-hopeless task to figure out how to solve some problem that arises.

But after half a dozen years of trying and learning, I can say it may not be easy, but it sure is rewarding!

### Post by endlessinstant on 2015-02-22
There isn't a lot of point to learning all the CLI and other weird stuff at first IMO.   A good starter distro like Ubuntu or Linux Mint is going to be all the further many (most?) users will need to take their Linux experience.   If you can just get over the hurdle of installing the new OS yourself, you have already achieved more than most computer users ever will with their systems.   Now if you truly love Linux after that and want to go further down the rabbit hole, you can begin experimenting with things like installing a new DE or doing some basic navigation and file modification with the terminal.   From there you can look to install a more complex distro like Arch.   

Linux is well-known for being able to run on even very low end or old hardware and still work great.  This is because advanced distros like Arch, Slackware, etc. are all designed to allow the user to tune their system exactly as they want.   But for users who just want to get away from Windows, they probably will never need anything more complex than Mint, which can still do most of the things that Arch can do.  You just don't have as much control over what it installs by default as you do in Arch (but OTOH are not expected to have the level of knowledge that an Arch install demands)
