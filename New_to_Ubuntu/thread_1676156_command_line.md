I've been using Ubuntu about six or seven months.
I was one of the people afraid of the command line, but since I've been learning how to use it, I have to say that I like it a lot.

I've always heard people say that the command line is so much faster than a gui, but I thought that didn't matter so much since I don't type all that fast.

I'm wondering though, it *_seems_* that a lot of programs actually run better via the command line.

No that shouldn't have anything to do with it. It is likely one of two things: You are using a completely different software in the command line than the GUI software. There are often tons of software for the task with a GUI where one can be better than the other. In that case it's apples and oranges you can't use that as a comparison tool. Second, if they are the exact same software just one version without the GUI Front-end, then with CLI you have to actually specify your settings in the command. In your case, if they were the same, I assume you copy/pasted a command that you found somewhere? That command likely had proper settings for your DVD player or what not, while perhaps you were doing something when you manually selected stuff from the list (or perhaps just the defaults aren't compatible). 

So generally, the fact that you are using a GUI or not has nothing to do with anything. Though there could possibly be more advanced/higher quality software that don't have a GUI, or vice versa. In reality, GUI software are a CLI software on the inside as everything you do through the interface really just runs commands without showing you them in a terminal.
