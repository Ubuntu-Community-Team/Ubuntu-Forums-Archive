### Post by TheFu on 2022-10-08
If you are new to Linux and Unix, then you really, really, shouldn't attempt to use a Linux Server install for the first 6-12 months.  Use a desktop, get used to that, learn the CLI.  Servers don't have GUIs.

Nearly all issues like you've described are due to lack of understanding of normal Unix file and directory permissions. There are hundreds of Unix permissions tutorials on the internet.  Start there.  Spend 45 minutes in concentrated effort to learn those now, or you will be frustrated for hours, days, weeks, months, years .... with every Unix system.  They all work the same. Nothing different.

But really, if you are new, start with a desktop Linux install.

BTW, most people don't use mysql anymore.  MarieDB has been the replacement since 2015-ish.  And most webapps used to manage anything are to be avoided by beginners. They are a liability. We still need to know how to do everything without them first, so when they completely screw up whatever it is they are managing, we can recover from it.  Most webapps introduce easier hacking methods to have a system taken over by "bad guys".

I know you have different ideas than what I've suggested and you'll likely ignore the advice.  I tried, but only you can decide for yourself.
