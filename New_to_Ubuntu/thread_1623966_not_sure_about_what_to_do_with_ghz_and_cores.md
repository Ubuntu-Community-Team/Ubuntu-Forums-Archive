### Post by LowSky on 2010-11-17
First off, gigahertz comparison between two different chip lines is not a fair test. Things like cache and FSB can make a slower chip outperform a faster gigahertz chip. AMD proved this in the middle of the 2000's with its Athlon 64 line compared to Intel's Pentium 4 line.
Second, Intel and AMD chips will not work in the same motherboard, so if you return the chip you will need to return the motherboard too.
Third, how many cores a chip has will only help in applications that can use more than one core.

I have no idea what you plan on doing but both chips are more than fast enough for normal day to day tasks like email, word processing, watching movies and so on. Gaming using either might be a stretch for newer games but otherwise both make fine CPU's.

### Post by endotherm on 2010-11-17
I'd go with the i3 if all else is equal. 

cores vs ghz is really a question of what apps you will be running. single threaded apps that run in a single process and use high CPU will benefit from increased frequency, whereas multithreaded apps that use high CPU will benefit most from multiple cores. each core can run exactly one thread at a time, so having more cores means more synchronous threads. in general, multitasking is better using multiple cores, whereas intense single tasks would benifet more from a processor that can run the thread fastest (GHz). 

this of course assumes the bottleneck is CPU. if IO is the bottleneck, then things like a fast FSB, multiple memory channels, and a large L2 cache may have a significant effect on the task at hand.
