### Post by cariboo on 2010-10-10
If you still have an old /etc/X11/xorg.conf file remove that:

```
sudo rm /etc/X11/xorg.conf
```

and restart.

If that doesn't work, remove the previous nvidia driver:

```
sudo apt-get purge nvidia-current
```


The above command will remove the previous nvidia restricted driver. This should allow the system to start up with the nouveau drivers. Once yo are back at the desktop, you can then install the restricted nvidia drivers

### Post by Thrashdance on 2010-10-12
When i upgraded from the update manager from 10.04 to 10.10 and the system restarted Ubuntu just hung on the loading screen. I suspected the NVIDIA drivers that i had manually installed on the previous version and that was really the problem. The drivers were not integrated in the new kernel  after the upgrade.

Here's what I did:
The old kernel (2.6.32.25) was still installed in the system so by pressing *shift* after the BIOS load the grub menu showed up and i was able to log in the old kernel.
Downloaded the latest NVIDIA drivers and saved them to Desktop.
Rebooted and this time chose the new kernel (2.6.35.22) with *recovery mode*.
Chose *Drop to root shell prompt*
After root logged in typed *telinit 3*
Logged root again and then browsed to my Desktop folder where the NVIDIA drivers (NVIDIA-Linux-xxxxxx.run) were saved.
Typed *sh NVIDIA-Linux-xxxxxx.run *and followed the instructions of the NVIDIA installer.
There were some prompts and the installer uninstalled the previous version (or same doesn't matter) as best as it good and installed the new ones.

Finally the new kernel was able to load without any problems:P (I also removed the old kernel completely via Synaptic after that but this is not necessary)

Hope it helps.
