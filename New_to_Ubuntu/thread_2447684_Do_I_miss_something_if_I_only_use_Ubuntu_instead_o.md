### Post by Holger_Gehrke on 2020-07-23
Yes, of course there are lots and lots of program which are available for Windows that aren't available for Ubuntu (or any other Linux-Distribution). 

The obvious ones are Adobe's various offerings. While there are tools on Linux which can replace them for a lot of use cases, if you're working in print design the missing (or comparatively weaker) support for CMYK colours is going to be a problem. Also, if you rely on some complex program on Windows for your livelihood, then switching to Ubuntu - even if an adequate alternative application exists - involves a lot of unlearning / relearning how to do things.

Then there are all the custom applications in use in business - either produced in-house or developed on contract by some external developer. A lot of those might work with wine, but there are limits. And then there's things like Java applications that where coded carelessly and / or use JNI-calls to some obscure .dll ...

Surprisingly, the situation regarding games - long seen as one of the big problems to adoption of Linux by the average home user - has changed for the better thanks to Steam being available for Linux and offering their own Wine-like system (Proton).

So whether or not Ubuntu can "cover the windows features" for you depends on what you're using Windows for and how deeply you're invested in any special way of doing things. **Linux is not Windows.**

Holger

### Post by kurt18947 on 2020-07-25
Two reasons I can think of to keep a Windows partition. One is covered above - updating hardware that only have Windows support such as navigators. A second reason to keep a Windows partition or virtual machine is being able to use old hardware. There is an ongoing thread here where a user is having an awful time installing an Epson scanner. There is quite a bit of hardware which simply does not have Linux support. On the other hand some older hardware is supported in Linux but no long has a Windows 10 driver. You have to see what applications and device support you need before nuking Windows. 

Anytime I'm shopping for a new piece of hardware I do some research on Linux support or lack of it. If the manufacturer doesn't offer Linux drivers or if there isn't solid community support I'll pass. There's no sense in making my life needlessly difficult.

### Post by sdsurfer on 2020-07-27
I, too, set up a dual boot with Windows 7 on one hard drive and Ubuntu on another. My visits back to the O.S. dwindled over the months as I found viable substitutes for the work I did in Windows. As of today I haven't booted in Windows for over a year. The other HD is more or less a backup hard drive.

You will miss having to pay for operating system software.

You will miss having to go through hours of obscure documentation, downloading and unzipping files, restart-restart-restart just to get something as simple as a printer to work. One of the first things I noticed was I didn't have to do anything for a 3-in-one scanner to connect. It just worked.

You will miss having to pay pricey subscriptions and fees for software. Adobe is the "gold standard" on Windows and Mac, and I can do anything Adobe has to offer for free, and in a more lightweight and simple GUI. This includes photo editing, video, vector graphics, and presentation software.

You will miss the knowledge that a select group of overpaid engineers in Washington are holding you hostage with their development release cycle (i.e., something is broken, it may take weeks or months for an update to push out, or they may just decide to never fix it.)  Ubuntu is maintained by thousands of developers around the world, and there are thousands of eyes on the code, refining it and perfecting it for no other reason than to produce an excellent open source operating system. Well done.

You will miss having to jump hoops to get what you need out of your computing. Windows almost had this when DOS was still a powerful way to get stuff done, now it's a small and almost useless feature.

The real question you are asking is* what do I lose.* These points are pretty well covered by the previous comments here. There is a little bit of a challenge, you will find yourself asking "in windows I did that, is there an equivalent under Ubuntu/Linux?" To date I haven't found an absolute no.
