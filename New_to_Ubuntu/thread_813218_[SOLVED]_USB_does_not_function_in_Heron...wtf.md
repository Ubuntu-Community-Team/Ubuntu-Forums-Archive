### Post by spiderbatdad on 2008-05-30
dmesg is a diagnostic message generated by the kernel for debugging. You can see it by typing dmesg into the terminal, or ```
dmesg > dmesg.txt
```will create a small text file in your home directory. 
Some BIOSes are faulty by nature and don't properly route pci interupts.
Dmesg should offer some clues.
The kernel parameter pci=routeirq is often a fix for faulty acpi handling of some devices...like controllers.

You may need some other parameters, as well, to get your system working more smoothly. Some systems are just not linux friendly.

Please run dmesg > dmesg.txt and right click that file in your home directory. Select 'create an archive.' and upload that archive here with the manage attachment tool below. I will try to help and tell you how to edit the kernel parameters for boot.

Ok. to edit the file /boot/grub/menu.lst, run the command: ```
gksu gedit /boot/grub/menu.lst
```
scroll to the section shown below:
```
## ## End Default Options ##

title		Ubuntu 8.04, kernel 2.6.24-16-generic
root		(hd0,0)
kernel		/boot/vmlinuz-2.6.24-16-generic root=UUID=#'s up the wazoo ro **lapic pci=routeirq**
initrd		/boot/initrd.img-2.6.24-16-generic
quiet
savedefault

``` Yes, delete --quiet splash. replace with your choice, as shown above...are my choices for my system. Save the file, and reboot.

### Post by GregA on 2008-05-30
NDO,

I'm sorry to read about your systems trouble with USB flash drives.  Linux in general (since about 02 or 03) handles these devices with at least a 95% success rate, for hot-plug and for devices already plugged in at startup.  Sometimes a re-install corrects unusual problems like this (a corrupt install can cause these problems).

Have you tried another USB device to see if it is handled properly?

Installing usbmount might be the easiest and fastest solution for your level of knowledge.
