### Post by joshedmonds on 2009-04-22
Remember that the live CD is (for all intents and purposes) the same as an install. It may be best to download an .iso of Jaunty rather than upgrading, and try it out live to see if it works before installing.

If you do upgrade while using the live CD, those downloads aren't wasted, you can *gksu nautilus* then copy the package files from /var/cache/apt to an external source and copy them back to an installed system, refresh sources and update without re-downloading those files.

Some suggested fixes are also described in the release notes for 9.04 under 'Performance regressions on Intel graphics cards' - [http://www.ubuntu.com/getubuntu/releasenotes/904](http://www.ubuntu.com/getubuntu/releasenotes/904)

Some people have suggested this may be fixed by release, but intel integrated graphics were the subject of a release note (bug) in 8.10 as well, and you may find its worth waiting a while after release if visual effects are important to you.

### Post by zevans on 2009-04-24
The problem is that there are still some obscure freeze/hang bugs out there on i965, and they don't affect everyone, but there's no easy way to tell at install time whether you're going to be a victim of the bug. So there was no choice but to blacklist this card until the bug is fixed. Some of these bugs are regressions, as the latest stack for Intel cards uses more/different features of the chipset with the aim of better performance, but instead seems to have introduced more/different bugs... :(

So yep, we got a lot of "compiz stopped working" threads on the forum now, but that's better than lots of "X doesn't work all or freezes randomly" which is what would have happened otherwise.

There are a number of fixes in test, as far as I can tell from bug reports, but none definitive yet. Meanwhile 

```
 echo "SKIP_CHECKS=yes" >> ~/.config/compiz/compiz-manager
```

will re-enable compiz if you want to see if it works for you.

[http://www.ubuntugeek.com/intel-graphics-performance-guide-for-ubuntu-904-jaunty-users.html](http://www.ubuntugeek.com/intel-graphics-performance-guide-for-ubuntu-904-jaunty-users.html) if you want to dive a bit deeper...

[https://bugs.launchpad.net/ubuntu/+source/xserver-xorg-video-intel/+bug/359392](https://bugs.launchpad.net/ubuntu/+source/xserver-xorg-video-intel/+bug/359392) is the bug which prompted the decision to disable, but please don't post me-toos to it, and also note that there are some upstream bugs. In other words, this isn't an Ubuntu bug, it's an Intel/DRI bug, or several of 'em.
