### Post by MG&amp;TL on 2012-06-01
Yup, you seem to have missed the mark a bit. Not your fault, but I guess you don't conform to how 'most people' do things.

The software centre is for adding/removing software. As you worked out. But if you want to run applications, click the dash home icon (or press the windows key) and just type for what you want. If you want to see a list of what's installed, click the "crayons icon" (or hit windows+A) and look for "Installed-See X more results". You should then have a list. 

The other lenses (icons at the bottom of the dash) do other things, the default ones are Home (general, shows recently used files and applications), Applications (shows installed apps, recently used apps, and apps available), Files (shows recently used files/folders), Music (shows your music), and Videos (shows your videos and (!) online videos.)

Like any operating system, Ubuntu requires a bit of getting used to. Take a look at [https://help.ubuntu.com]("https://help.ubuntu.com") for more information.

Also,  you might want to try Lubuntu, Xubuntu, or Kubuntu if you don't like the current interface, as they have more traditional menus (Kubuntu even has a start button!)

### Post by inashdeen on 2012-06-01
May I sum up how do you actually run programs in ubuntu?
there is basically 4-5 ways to install apps in ubuntu,

1) ubuntu software centre
dash > ubuntu software centre
browse what you need and click on install > password.

easiest, best.

2) download a .deb file
example. go to [www.skype.com](www.skype.com) > get skype > linux
there is a 32 bit .deb file for ubuntu to download.
download it double click, install.

convenient, perfect for those migrating from Windows.

3) apt-get and ppa

installing apps from command line.
example.
open dash > terminal
on terminal run this

sudo apt-get install "specific name of the application to install."

note (meaning of the command): 
sudo : ask me password
apt-get : name of the installer
install : install.. duh..
specific name : example to install a command line temperature checker, the name is lm-sensors

so it will be like sudo apt-get install lm-sensors

kinda of techy, but eventually most ubuntu user will learn this

4) direct app link
user friendly, used by webupd8, getdeb and several other ubuntu news websites 
just click on the install link provided on the website. your browser will ask you to trust the application. then, ubuntu software center will prompt you to install it.

5) Wine
Easy, but should always be a last resort.
this application will run your .exe application on linux. it is developing, but it is not 100% perfect. many app still not compatible. among noticable program can run on wine like microsoft office 07, adobe photoshop and picasa.

to get those .exe running on ubuntu, you must first install Wine from ubuntu software center. it is sort of a framework, if you like the term.
once install, simply right click on your .exe file and run it with wine.
again, not recommended.

### Post by GregA on 2012-06-02
dbDesigner,

The real issue with new users running Ubuntu is - - are they already familiar and comfortable with "Smart Phone/Tablet" interfaces, as Unity was designed to be a bridge interface between those devices and traditional PC interfaces on desktops and laptops.

So, you were faced with suddenly acclimating to the "Metro (Win8) version of Linux, as well as the core distinctions between Unix/Linux based systems versus Windows or Mac.

If by chance, you had downloaded and installed "Kubuntu", or "Xubuntu" or almost any  other distro - - you would have found yourself in a very familiar environment virtually the same (only minor interface differences) from Windows 7.  Then you would have only needed to learn about the 6 to 10 core differences between the OS's(like the file system, the use of repositories, security essentials, the basics of the terminal, etc.).

I personally run Kubuntu, as I very much prefer a desktop interface on a PC - - I'm not a fan of Unity, IOS, or Android and only use those when on my tablets or ereader.  Those interfaces are not fully developed yet, and do not provide the system users with the fine level of control and flexibilty of PC interfaces (be they Windows, or KDE, or Enlightenment, etc). 

If you have at least 2gb of ram, I would suggest to try KDE (Kubuntu 12.4) and compare initial experiences.
