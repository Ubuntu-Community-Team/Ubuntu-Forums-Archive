Take this advice from me, as I am **VERY** dependent on Windows to get stuff done (unfortunately).

The UI is **ALL** personal preference (just to get that out of the way).  Both Ubuntu and Windows do the necessities for daily tasks (ie. photos, movies, e-mail, browse the web, look at porn, etc.) - there won't be a winner there unless it's again, preference.

1.  Guild Wars (pretty much any game you buy short of games like World of Goo or Spore) consists of binaries compiles for Windows-32.  Because of this, they will **NOT** be able to work on Mac or Linux.  There is a Win-32 emulation layer that lets you run SOME windows programs called WINE.  The WINE team has an application database (here) and gives ratings based on how well it runs under WINE.  A low rating may mean it's completely unusable, while a gold rating may mean there's no difference between running it on Linux or Windows.

2.  Yes, pictures just need to be in a folder.  The method I use is something I call "take-over" - an external hard drive would work just as well.  The method consists of:
- Say you have a 100GB drive that you want to move to Linux.
- Partition 40GB for Linux.
- Mount the Windows partition in Linux and copy over your pictures, movies, music, etc.
- Modify the Linux partition to use the other 60GB of disk space

Or you can dual boot, and run Linux and Window side by side, on separate partitions.  If you use this method (which you'll probably want to), you can LEAVE the music, movies, pictures, etc. on your Windows drive, and simply link the pictures folder and such, to those in Linux.  Rhythmbox (like iTunes in Linux) can link to your music dirs in Windows - so in short, all the data stays on the Windows partition, but it's mutually accessible by both OS's.

3.  Depending on which phone or iPod you have, it may or may not be do-able.  If the phone mounts up as a standard mass storage device, then it'll work fine, if it requires software designed for Windows, it probably won't unless you can get it to work in WINE. As for the iPod, if you have an iPod touch it WON'T be accessible in Linux, but ANY iPod prior can be managed via Rhythmbox.

----

The big issue with Linux is compatibility - most things, be it drivers for printers, or drivers for a webcam, or software, etc. are designed and compiled to run on Windows.  Therefore, you have to search for alternatives on Linux, some alternatives exist, some do but barely work..some don't work at all.  That's what keeps me from completely ditching Windows.

As for the blue screens in Windows.
Contrary to the popular jokes on Windows, blue screens are created due to hardware manufacturers' faults, not Microsoft's.  The blue screens are caused (9 times out of 10) by driver faults in which the OS cannot recover, the hardware companies write the drivers.  So if you're getting consistent blue screens, you're most likely using drivers that don't work well on Windows 7.  If it's doing it out of the box (no non-standard drivers installed) then it's a hardware issue, a physical hardware problem, and that will do the same thing in Linux, it'll just present itself as a Kernel Panic instead of a blue screen (which is all a blue screen really is).

Hope that gives you some good info.

If you're writing a virus, you're not going to write one that targets like 5% of the end user world...you're going to write it for Windows, so it targets 80% of the end user world.  Same goes for OS X.  The trade-off is that just like the virus, most programs aren't targeted for the 5% of the end user world (Linux) - therefore we have to make OSS alternatives.  And fragmentation is an artifact of using a hard-drive as we know them today, not the OS.  It can be helped by the OS, but the issue is always there.
