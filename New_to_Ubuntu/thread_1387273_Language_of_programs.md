### Post by lykwydchykyn on 2010-01-21
They're two very different languages with different purposes and scopes.  Visual Basic is probably the better choice (between the two) if you're writing some kind of simple desktop app for Windows; wouldn't work so well for writing hardware drivers or a multimedia codec library.

### Post by ibuclaw on 2010-01-21
Difficulty is in the eye of the beholder.

For example, someone who finds Perl simple, may find VB incomprehensible, and vice versa.

C probably requires more lines of code to do the same task as VB, and you have to suffer from the wrath of pointers (and understanding memory de/re/allocation)

VB has an very big library that may make learning difficult (you just don't know where to look to find XYZ function/task). C++ suffers from the same/similar thing, so I presume it would be the same here too.
