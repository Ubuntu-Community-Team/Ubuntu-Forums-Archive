### Post by oldfred on 2011-02-05
You somehow have a sda3 that is inside the extended partition. It has to be sda6 if it is in the extended partition and windows will not normally boot directly from logical partitions. Or you have to convert the extended partition back to not include the  sda3 partition.

First make a backup of  your current partition table and copy to another device.

Backup partition table to text file
sudo sfdisk -d /dev/sda > PT.txt

Sometimes just rewriting the partition table with fdisk or sfdisk fixes things. Othertimes you have to use testdisk or other tools.

this occasionally fixes issues and is worth trying:
you do the following :
fdisk /dev/sda
use option : x (expert mode)
use option : f (fix partition order)
use option : r (return)
use option : p (to print)
use option : v to verify partition
if it is ok
then you can do
option : w ( to write table to disk)
option : q to quit
else if it is not ok
then post the output of option p and v
