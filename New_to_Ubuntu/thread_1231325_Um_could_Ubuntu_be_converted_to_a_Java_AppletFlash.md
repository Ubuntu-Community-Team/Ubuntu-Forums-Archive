### Post by Defrector on 2009-10-06
I think Mighty_Joe's implied suggestion of using a Remote Desktop applet to run ubuntu remotely on a server's VM is the best bet with this.

It appears that the OP is asking to make the live CD/demo more accessible to those curious for UI and asthetic reasons, not necessarily for going into the terminal and experimenting with low-level I/O or anything which would involve the machine to be installed.

Ubuntu would not be converted to a java/flash app in this case; it would be accessed remotely on a VM. Even the tiniest distribution of ubuntu available would be pretty big and probably wouldn't be practical for those casual enough to require an applet/flash app to consider the OS. It would have to go as a remote desktop.

Depending on what the prospective user is looking for, this might be good or this might be bad for ubuntu's image. Remote desktops over the internet tend to have latency issues and give an effect of 'lag' on the visual end. It would be an incredible shame to see something like XFCE being called 'slow' by the n00b crowd.

Perhaps the way it would work is the ubuntu live CD image being started on a VM on a server which supports AMD-V or other virtualization. Said server would be running many VMs at once, hopefully at a decent pace.

But who is going to cover the cost to build and run these servers? I have no clue. Who would financially benefit from it? Find them, ensure their plans don't step on too many ethical toes and then have them do it I guess. It's kind of a rough time to ask someone to put 50k into an experiment.

Cloud computing has its limits; bandwidth chokes and latency undeniably can make all but the best-designed *cough*Google*cough* web apps run far laggier than local apps. But this is a whole other rant altogether.

EDIT: It just hit me that a single install could have multiple accounts sending RDP. That could make things somewhat simpler.
