### Post by Bucky Ball on 2017-06-13
Welcome. Ubuntu *might* run on the dinky old thing, but you'd certainly get a faster experience with Lubuntu and that has a much better chance of running fine. Xubuntu is another option for that machine, and I personally prefer it. You might not.

Download the ISOs of both, create install media (USB or DVD/CD) and 'Try' them in a live session. You will get the option to 'Try Lubuntu' when you boot the install media (ditto Xubuntu). See which one you like, which runs best, when you've decided, install. There is an icon on the live session desktop.

If you need help with the actual install, best post a new thread in 'Installations and Upgrades' for the best chance of support. Good luck on your Linux journey. :)

### Post by EarlsFurniture on 2017-06-13
I've used Lubuntu and Xubuntu on many an old computer. (all the way back to Pentium3 days) Lubuntu has been getting heavier and more sluggish lately with newer releases. Xubuntu in many respects, while taking a little more RAM upfront, is noticeably faster on the same machine. Another option is LXLE. It's based on Lubuntu LTS and is considerably snappier. Note, Lubuntu is migrating its desktop from LXDE to LXQt either in the 17.10 release or by 18.04. My experience with it in testing is it improves considerably on speed over LXDE. (which is going to cease development when LXQt is ready). Depending on how much work you want to do, you can also install Ubuntu Server and then install LXDE, LXQt, or XFCE directly. This won't get all the spit and polish of Lubuntu/Xubuntu, but the layout will be the same and it will run faster. If you just need to play music, ANY of those options should work fine.

### Post by Bucky Ball on 2017-06-14
> **EarlsFurniture said:**
> Depending on how much work you want to do, you can also install Ubuntu Server and then install LXDE, LXQt, or XFCE directly. 

I would avoid that. No benefit in installing server things you don't need in a dinky old machine ... unless you're going to use it as a server. ;)

Use [the minimal install]("https://help.ubuntu.com/community/Installation/MinimalCD") and then add the desktop environment of your choice if you want the lightest option. Lubuntu-core or Xubuntu-core also excellent options for a basic and *very* light system and relatively easy to achieve. Both are exactly the same to look at as Lubuntu or Xubuntu, but there are only a few default things installed. It is up to you to install the rest (package manager, web browser, email client, etc.) 

It's fun, you end up with a very light system and you'll learn something. ;)

Not necessarily true. Just because it has the word 'server' in it  doesn't mean that you'll end up with things you don't need. (certainly  not any more than a desktop edition) When you get to the package selection step, just don't choose any of the server oriented services. Install the base system then once you reboot add the desktop environment you want and any other user applications.

There's nothing wrong with the minimalCD, it's just more downloading while you install, and more work to configure. I've used both methods. The base server + DE is the easier route.

Lubuntu-core & Xubuntu-core still won't be as fast as vanilla LXDE & XFCE, but certainly, they are good options.
