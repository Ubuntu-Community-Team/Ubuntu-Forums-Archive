### Post by NightwishFan on 2010-10-20
64-bit is a new processor instruction set. The x86_64/AMD64 implementation is backwards compatible with 32-bit. Generally if you have a 64-bit computer you want the 64-bit version.
[http://en.wikipedia.org/wiki/64-bit](http://en.wikipedia.org/wiki/64-bit)

Linux/Ubuntu is not Windows. It does not integrate or use Windows software. We have our own selection of compatible software (quite a bit actually). Though it is potentially possible to use Windows software through a tool called Wine, not everything works yet.

### Post by SuaSwe on 2010-10-20
If you already have Windows across the whole drive you will need to resize that part first; if you already have unallocated space, you're good to go! 

Personally, if you haven't started already, I would recommend having a root partition, a swap partition and a home partition (/, swap and /home) - having root separate from home makes upgrades, reinstalls and the like much smoother. Depending on PC specs I would suggest 10-15GB for root, 1+GB for swap (swap should be at least as big as the RAM size - see [https://help.ubuntu.com/community/SwapFaq](https://help.ubuntu.com/community/SwapFaq)) and whatever else you can spare for home. Don't forget that you can access NTFS partitions (e.g. Windows) from Ubuntu as well so if you run out of space on Ubuntu's home partition, it's easy-peasy to save it to the Windows drive instead or to move some stuff over. :)

### Post by NightwishFan on 2010-10-20
If you just use Windows in virtualbox that is different from a dual boot. You will have to reinstall Windows 'inside' Virtualbox. Most features and performance will remain as long as you have at least 1gb free RAM for the virtual OS at all times. I would use this option only if you really really like using Linux and Free Software, and just need Windows for a few things.

Your best bet is to install Ubuntu in a dual boot configuration. This will give you the best of both worlds. Freedom and performance of Linux, and device and software compatibility of having the highest market share. documented here:
[https://help.ubuntu.com/community/WindowsDualBoot](https://help.ubuntu.com/community/WindowsDualBoot)

My advice is to use the Windows tools to resize your disk before you install Ubuntu, this how to should help. I advise you defrag before you attempt to resize a disk. Shrink the disk and leave as much free space after it as you want Ubuntu to use. Then simply tell Ubuntu to install in the largest free space.
[http://www.howtogeek.com/howto/windows-vista/resize-a-partition-for-free-in-windows-vista/](http://www.howtogeek.com/howto/windows-vista/resize-a-partition-for-free-in-windows-vista/)

Any more questions?
