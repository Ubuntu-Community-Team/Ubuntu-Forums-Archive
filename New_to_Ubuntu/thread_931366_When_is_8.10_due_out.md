### Post by gregphil on 2008-09-27
Will **ubuntu 8.10** fix the "browsing of authenticated windows file shares" bug so that basic peer-to-peer file browsing from the ubuntu GUI (such as the Nautilus network file browser) can work once again?  It currently (in 8.04.1) can't **browse** authenticated shares (but it can connect if you enter the full path to the share)

Until ubuntu can seamlessly share files with the installed base of Windows machines it's future growth will be limited.  Peer-to-peer sharing is by far the most common home networking configuration.  These days anonymous (guest connection allowed) shares are frowned upon as a huge security leak.  Thus we need authenticated peer-to-peer browsing to work.  And please don't suggest setting up various servers on ubuntu (WINS, domain master etc) to "solve" the problem.  Kubuntu works and browses just fine on a peer-to-peer network without extra servers running, and ubuntu should be able to do so as well.  A network with servers requires certain machines to be always on and requires extra knowledge to configure and maintain.
