Hi.  That hardware is not overflowing with capability.  The "7.54Gb HDD" (presumably an empty partition on a Windows disk?) is very minimal, indeed. For comparison, I'm on a new Ubuntu install that's consuming 6.9 GB of space before I start building it out and adding software.

Ubuntu is not a "minimal" system.  Another version of Ubuntu -- Lubuntu -- is designed for use on older, less-capable hardware:  [http://lubuntu.net/](http://lubuntu.net/).

>

If the burned image is bad, you need to replace it, you can't "fix" it.   

You are setting the BIOS to boot off the DVD?

Have a look here: [http://www.ubuntu.com/download/desktop](http://www.ubuntu.com/download/desktop).

> 
Why are these files so prone to errors is it a general Linux issue...

They aren't. If you've seen a succession of the same problems, then very likely either the image is from an unreliable source or is being burned incorrecty.
