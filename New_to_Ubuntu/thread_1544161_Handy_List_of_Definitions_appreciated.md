### Post by orethrius on 2010-08-02
My understanding of those terms is this:

GUI: Graphical User Interface; CLI: Command Line Interface

Frontend: The GUI of the system, often Gnome or KDE over X11.  This may alternately refer to a conventional CLI terminal, or GeTTY.

Backend: On most systems, this refers to the various package managers and databases used to tie the system together.  This may alternately be used to refer to the CLI on a graphical system.

Sudo: Provides direct root access in a Terminal.  As it only directs output to a Terminal by default, it tends to be useless outside of a Terminal.

Gksudo: A GTk graphical frontend to an approximation of "sudo".  Using "gksudo 'commands here'" rather than "sudo 'commands here'" allows you to run processes as root without having to use a Terminal.
