### Post by dttauzin on 2010-10-27
Does it use the rt73 chipset? I had a problem with my belkin usb wireless adapter that used the rt73usb driver. alot of the third party companies use chipsets from the major manufacturers. this one in particular. what i did was:

start terminal

logon> sudo -s

type the following individually line by line:

# blacklist rt2570
# disable rt2570
# blacklist rt2500usb
# disable rt2500usb
# blacklist rt2x00lib
# disable rt2x00lib
# blacklist serialmonkey rt73
# disable serialmonkey rt73
# blacklist rt73
# disable rt73
exit

then closed terminal.

notice blacklisting and disabling all of the conflicting rt chipset drivers except for the rt73usb wireless and it corrected my problems.

### Post by anewguy on 2010-11-01
Ok, in the other post we went through what you had done to-date, if I remember correctly it was via ndiswrapper.

EDIT:  If you actually tried to install a driver with ndiswrapper or using ndisgtk, please remove it first.

Then, since I don't know how you got ndiswrapper or ndisgtk, use the Synaptic Package Manager to remove ndiswrapper common, ndiswrapper utilities and ndisgtk.

IF you made any changes to the blacklist.conf or modules files in your previous attempts, reverse those changes.

Now, using the installation media - be it a USB stick or the LiveCD - boot Ubuntu, THEN insert the installation media.  Using "Places", navigate to the /pool/main/n folder on that media.  You will see 2 folders there - 1 for ndisgtk and 1 for ndiswrapper.  Go to the ndiswrapper folder first and double-click the ndiswrapper common file to install it.  When that finishes, double-click the ndiswrapper-utilities file to install it, wait for it to finish.  Now go to the ndisgtk folder and double-click on the file there to install it - wait for it to finish.

At this point you will now have:

- a system without any know devices to ndiswrapper
- ndiswrapper installed from the Ubuntu installation media
- ndisgtk installed from the Ubuntu installtion media

I have downloaded the driver files for you.  Before I can get them to you, I need to know if you are running "normal" Ubuntu -32 bit, or if you are running 64-bit Ubuntu, as it makes a difference which driver files I give you.
