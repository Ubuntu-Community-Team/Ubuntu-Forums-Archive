### Post by Zill on 2012-01-18
> **Jimemitchellsr said:**
> ...Can I install ubuntu to boot from  the disk or memory stick if there is _NO_ operating system loaded on the new hard drive?
Of course. :-)  Ubuntu is a *complete* operating system and so does not need any other OS on the hard drive.  Just ensure that the boot order in the PC BIOS is set to boot *first* from the CD-ROM, rather than the HDD.
> **Jimemitchellsr said:**
> I would think that this proceedure would make it easier to install Vista in the future should I come across a Vista disk by simply removing the disk or memory stick.
I am slightly puzzled by your question.  If Ubuntu is *installed* on your HDD then removal of a CD or memory stick containing the Ubuntu OS will make no difference as it would not be in use.  Ubuntu will simply be running from the HDD as any other OS would do.

However, if you are simply *running* Ubuntu as a "live" version from the CD (or USB memory stick) rather than having *installed* it to the HDD then, of course, removal of the media will end your Ubuntu session.

Regarding the installation of Vista, be aware that if you wish to do this it should be done *before* the installation of Ubuntu.  While Ubuntu is happy to be installed alongside other OS's, Microsoft are not so magnanimous and so Vista etc will not recognise other OS's such as Ubuntu.
> **Jimemitchellsr said:**
>  Does anyone have any comments on whether it is better to use the disk or the flash drive memory stick?My personal view is that the CD is the simplest option but the memory stick should work as long as you can get the BIOS to boot from it (I have had problems with this!).

### Post by Scott Baker on 2012-01-24
Here's one for you to try, with regards to your wireless. If you have a good sized flash drive (8+gig) load your Ubuntu installation onto this. Once done, boot your comp from this device. Once your machine is booted and running, try your wireless. If no luck yet, go to the update manager, and update this drive. I've done this, and the flash drive is treated just like a hard drive in this instance. 8+ gig gives you some wiggle room so that your drive doesn't barf. In a few minutes, you should have a completely updated version of Ubuntu, with all of the current corrections and tweaks. Try rebooting with this "new" drive, and check your wireless now. Best part of this is if all is now corrected and working to your satisfaction, you can install this with a lot of the updates already in place.  :-\"
