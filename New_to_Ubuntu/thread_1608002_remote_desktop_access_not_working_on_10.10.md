### Post by endotherm on 2010-10-28
once you ssh in, you can determine if vnc is running by looking for a listening port on 5900, with this command:
```
netstat -ntl
```as for your tunneling, what ports are you using on the localhost?
if you open a command prompt on the client box and run:

```
telnet localhost <local tunnel endpoint port>
```what do you get back? does it look like this (I'm using 139 here, but put in your own local tunnel port)?
```

Lucid:~$ telnet localhost 139
Trying ::1...
Connected to localhost.
Escape character is '^]'

```or this:
```

telnet: Unable to connect to remote host: Connection refused

```

### Post by The Chewy Wun on 2010-11-04
I believe the problem is with the new configuration of remote desktop (VNC) that comes by default with 10.10.  I checked my listening ports and found that VNC was listing on 5910, not the default 5900 on most of the VNC viewers out there.  As soon as I made sure 59**10** was open and used IPADDRESS:59**10** on VNC Viewer from my Windows box it connected like a champ.  Not sure if they made this change recently or if it's been different for a while, but I know 5900 has been the default VNC port for a long time so it was surely in need of a change.  Hope that helps
