### Post by candtalan on 2010-08-19
> **Hawhaw1 said:**
> Aye ok I'll download it from thAt link. How do I to about getting rid of the one that I have allready instAlled ?

Burn the new iso when it is downloaded, and run the new CD as a live CD.

Use the 
System>Administration>Gparted partition editor

to carefully identify the new Ubuntu partitons. 

You will probably also see the Windows partitions (ntfs and also maybe FAT), maybe the first two. Check back here to help confirm the identities if you need to.

A normal Ubuntu install in your situation will probably have created an extended partition with a couple of logical partitions inside, one for the system (ext4) and one for swap.

I would expect to delete the two logical partitions, and also delete the extended partition, finally leaving an unused space on the drive, not partitioned at all in the space that is.

Obviously take care to avoid harming the Windows stuff. After a Windows resize it feels unclean and will probably need to do a self check or clean up or whatever, when you first later run windows.

Then use the live CD again, and this time go for install Ubuntu. When you get to the installer partitioner stage, you will see choices. 

Choose the one which offers to install Ubuntu into the 
'largest unused space on the drive'

hth

> **Hawhaw1 said:**

try confirming the md5 hash as mentioned above. a hash is a number that created by a mathematical function that "describes" the data on the disk. if you hash the same disk, with the same function, you will always get the same result, and that result is different from the result "describing" a different piece of data. that means that (in theory at least) if a attacker added malware to your disk, that the hash would change, and be differant than the original disks'.  try hashing the disk, and compare the result to the those on the ubuntu website to see if they match. if so, you are probably safe. its not guaranteed, as there are some theoretical attacks on md5, but it is probable.

### Post by candtalan on 2010-08-20
> **Hawhaw1 said:**
> ok thanks, I will burn a new cd tomorrow. But if I check the first disk I made (from torrentz) and it is clean, should I just go ahead and use that?

Only if you can confirm the md5sum of the CD by independant means. Which you can now do in the torrentz install. 

Insert the torrentz CD into ubuntu, and wait until it is recognised. Close all the options it then suddenly offers. But you can still see the CD shown on the display desktop this means it is recognised (mounted)

Then use a terminal and type

mount

look at how the CD is described, it will be something like
/dev/sr0 on /media/Ubuntu 10.04.1 LTS i386 type iso9660
which means your mounted cd is seen as device sr0 (yours might be scd0 or whatever, then use that)

Then type
md5sum   /dev/sr0

and wait while the md5sum is calculated - might be slow. The resulting number should be exactly correct.

hth
