### Post by TheFu on 2016-12-01
Welcome to the forums.

Google pushes out updates whenever they want. No way to stop them.
After google pushes updates, you need to update any crouton installations. If you don't, the GUI integration can be flakey. That was my experience.  I wouldn't attempt Unity under crouton for a number of reasons.

I don't use crouton anymore and never dealt with steam. Found the limitations of chromeOS around kernel module support to be too restrictive.
If you want a machine to run steam, I suspect a chromebook isn't the best choice.

Never used powerwash. After I was done screwing around with chromeOS, it was just easier to wipe it and start fresh.  Also, I only used chromeOS with the guest account, never any gmail account - I'm pro-privacy and really just wanted the hardware, not the *extras* that google provides which cannot be turned off.  These days, I'd get a cheap Dell (really miss the delete and F11/F12 keys).

BTW, I'm typing this on a Toshiba chromebook running Ubuntu Server with openbox as the WM.

As you probably know, all discussions around chromebooks need to include the exact model of device. Some are extremely specific in what they can and cannot do.  Have a friend with a $99 Lenovo that has been nothing but problems.  I was happy with my Acer C720 until the HW wore out. Got a nicer model the next time - faster i3 CPU, 4G of RAM, 1080p screen - this thing is faster than my 1st gen Core i5 laptop!

Also, rather than describing what you did, please copy/paste the actual commands.  Sometimes we misinterpret what a command really does. I do and I've been a linux admin since the mid-1990s.  Nobody knows everything and we all make mistakes.
