### Post by TheFu on 2015-01-19
Please use the **tee** command to capture output to a log file.
Use aptitude instead of apt-get.
You only need 3 apt-get/aptitude commands. The others aren't helping.
* sudo aptitude update  |& tee -a ~/log.update
* sudo aptitude dist-upgrade  |& tee -a ~/log.dist
* sudo aptitude -f install |& tee -a ~/log.install

When there are dependency issues, either a .deb package was manually installed and it is getting in the way of upgrades. Until you remove it (what ever that package is), the APT db is screwed.

Or if you've never installed a .deb file directly, then the only other dependency issue left is by adding a PPA that hasn't been maintained.  Remove it (or multiple, as needed), run the update/dist-upgrade again.

These should finish error free and there shouldn't be any problem with anything downlaoded.
