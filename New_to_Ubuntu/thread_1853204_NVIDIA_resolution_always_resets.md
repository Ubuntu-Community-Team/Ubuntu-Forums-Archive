### Post by papibe on 2011-10-02
First backup your xorg.conf.
```
$ sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.bak
```
Then, try forcing the resolution by doing a mod like this:
```
...
Section "Screen"
    Identifier     "Screen0"
    Device         "Device0"
    Monitor        "Monitor0"
    DefaultDepth    24
    **# **Option         "TwinView" "0"
    **# **Option         "metamodes" "1920x1080 +0+0; 1920x1080_60 +0+0"
    SubSection     "Display"
        Depth       24
        **Modes      "1920x1080"**
    EndSubSection
EndSection

```
And then restart your machine to test it. Remember that if you have a black screen you don't need to hard reset. Just press Ctrl+Alt+F1 to go to text mode, and there you can restore your xorg.conf.

If that doesn't work, please post your /var/log/Xorg.0.log

Use [http://paste.ubuntu.com/]("http://paste.ubuntu.com/") and post the link.

Regards.

### Post by papibe on 2011-10-02
**EDIT: Oops, you already solved your problem....**

It's very strange. It looks like everything is OK, but the last line set a low resolution:
```
(II) Oct 02 12:48:04 NVIDIA(0): Setting mode "1280x1024"
```
The only help I can offer you at this point is using old school custom modelines, but since you are able to change to HD later it should be little to none risk of booting into a black screen.

Change your resolution to 1920x1080 using the 'Nvida X Server Settings'. Now that is set and working. Open a terminal and run this:
```
$ xvidtune
```
It'll open a couple of ugly black and white windows. Press cancel on the forward windows (or quit in the other one if you press OK). You will be left with an output similar to this in your terminal:
```
Vendor: Unknown, Model: DELL 1907FP
Num hsync: 1, Num vsync: 1
hsync range 0:  31.00 -  83.00
vsync range 0:  56.00 -  76.00

```
Check that your output (not mine) match the the fields HorizSync and VertRefresh on the Monitor section of your xorg.conf. If you see any difference, change your xorg.conf accordingly and reboot.

If you are still booting into a low resolution, or there were not need to change xorg.conf, run this to get details of the actual modeline you are using:
```
$ xvidtune -show
```
The output should be similar to this:
```
"1280x1024"   108.00   1280 1328 1440 1688   1024 1025 1028 1066 +hsync +vsync
```
Using that information, add a modeline to your Monitor section:
```

Section "Monitor"
    # HorizSync source: edid, VertRefresh source: edid
    Identifier     "Monitor0"
    VendorName     "Unknown"
    ModelName      "Acer S231HL"
    HorizSync       30.0 - 80.0
    VertRefresh     55.0 - 75.0
    Option         "DPMS"
    **ModeLine    "1280x1024_60_0" 108  1280 1346 1458 1688  1024 1025 1028 1066 +hsync +vsync**
EndSection

```
(Please remember to use your numbers, those are for my monitor!)

Note the _60_0 at the end. That is just to make an unique identifier.

Then change your metamodes line to this:
```
Section "Screen"
...
    Option         "metamodes" "1280x1024_60_0"
...
EndSection
```
Then restart your machine.

Here are a few references for this kind of hack: [Old modeline tutorial]("http://howto-pages.org/ModeLines/"), [thread about custom refresh rates]("http://ubuntuforums.org/showthread.php?p=454217"), [Nvidia reference to program modes]("ftp://download.nvidia.com/XFree86/Linux-x86/195.22/README/programmingmodes.html").

Final note: to the modeline to stick it may be necessary to add this option to the Device section (but I would let this option as a last resort):
```
Option "UseEDID" "FALSE"
```

Tell us how it goes,
Regards.
