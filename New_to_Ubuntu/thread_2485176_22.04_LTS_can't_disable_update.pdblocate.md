Hmmmm... That isn't on any of mine, so not familiar with what it does, but...

By the guides for 'run-parts' used by Ubuntu and Debian cron.hourly, cron.daily & cron.monthly: Those scripts shouldn't have any extensions, and they should be permission of 755 (Symbolic Notation rwxr-xr-x).

That script is permissions 644. By that rule, because of the set permissions, it shouldn't even run.

What happens if your rename the script with a ".bak" file extension (the other rule)? If that doesn't work then move the script to your user home folder (somewhere) for a temp backup/test?

### Post by Holger_Gehrke on 2023-03-22
Besides the cron job there's also a systemd timer (/usr/lib/systemd/system/plocate-updatedb.timer). You can disable it with 'systemctl disable plocate-updatedb.timer'.

Holger
