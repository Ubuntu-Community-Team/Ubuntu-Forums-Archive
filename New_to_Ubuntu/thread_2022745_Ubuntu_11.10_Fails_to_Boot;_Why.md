### Post by BlueAZ on 2012-07-11
Hi,   I've had recurring problems with my Ubuntu OS failing to boot up after an attempt to suspend the computer.  But this time, It won't boot after a normal shut down. On the other HDD (smaller & older) I have Windows XP.  I can still boot to Windows just fine.  But when I try to boot the Ubuntu HDD, there's no graphic, no sound & no HDD activity.

When I've had this problem before, I could only solve it by wiping the 320 Gb HDD & doing a clean reinstall of Ubuntu 11.10.  But then I've lost all my app's & now some [grooooaaan] un-backed-up files.

How can I discover if this is a software or a hardware problem?  Is the Hard Drive bad?  I don't see how I could have a conflict if Ubuntu is the only thing on it...
