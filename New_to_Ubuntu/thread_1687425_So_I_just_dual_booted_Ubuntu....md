### Post by wilee-nilee on 2011-02-13
You have installed with wubi, it may be possible to install on a separate hd and have it work, but it is already a unstable install altogether. Not a recommended way of installing at any time really.

If you want a actual wubi install put it in C, if you want a actual dual boot that is partitioned you will have to boot the live cd and install.

If you want the partitioned dual boot on the sdb hd you will have to remove the wubi from windows then boot the live cd format the partitions for Ubuntu and use the custom install to get grub2 pointed at the correct master boot record.

### Post by bcbc on 2011-02-13
If you have a spare hard drive for Ubuntu, you don't need Wubi.

But... that message, Try (hd0,0).., is just wubildr.mbr looking for the file wubildr on the root of the first partition. Then it goes to the next and finds it on there. If you want to avoid the delay copy the wubildr from the root of /dev/sda2 to /dev/sda1 (you'd have to mount these and do it in Ubuntu).

That "windows recovery menu" is actually Grub2's boot menu. That's normal and you'd see that if you installed Ubuntu in a normal dual boot.
