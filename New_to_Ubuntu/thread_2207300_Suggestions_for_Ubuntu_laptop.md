### Post by buzzingrobot on 2014-02-23
I suspect the info on the Ubuntu certified page is valid, but incomplete and out-of-date.  That is,  if a laptop isn't listed it probably means no one has submitted a report, not that it has been shown to be incompatible. Correct me if I am wrong, but I believe there is no effort by Canonical to test laptops for Ubuntu compatibilty. I am not aware of anyone in Linux who does that.

Don't worry about UEFI itself,  Whether a laptop uses new UEFI or old BIOS, look for a machine that allows Secure Boot to be turned off.  All hardware certified for Windows is supposed to offer this, but some do not, apparently, and some implementations are faulty.

I use a ThinkPad W530.  Every version of Ubuntu since 11.04 runs fine on it, as has every other version of Linux I've thrown at it.

### Post by Vladlenin5000 on 2014-02-23
Hardware certification is a payed service. As such it isn't Canonical making no effort to test laptop but the manufacturers not willing to submit a a certification request. An Ubuntu hardware certification is something most hardware manufacturers can go without. Unless they perceive a clear marketing opportunity it's only logical NOT to spend money on such certification.
