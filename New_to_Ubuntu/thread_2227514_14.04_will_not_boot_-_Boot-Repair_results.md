### Post by oldfred on 2014-06-02
Moved to your own thread as your issue is not the original thread issue.

How much RAM? 
What model computer?

What mode is BIOS set for hard drive. While AHCI is preferred, your XP may not then work. But if you have large or LBA that would be preferred over IDE.

Some BIOS only boot from first 137GB of drive. So a smaller / and rest for /home often works or a smaller /boot partition. All boot files just need to be in a partition fully within the first 100GB of drive or so. Linux writes files anywhere in a partition.

### Post by oldfred on 2014-06-02
Someone posted this a long time ago, you probably have to chroot into your install to update init. But file may exist and now be beyond the 137GB limit?

 "If the &#8220;apt-get upgrade&#8221; does not successfully complete then a flag is set and update-initramfs does not get to run, hence grub gets all messed up. You should be able to boot using the old kernel and then run the sudo update-initramfs &#8211;u -k command. Then run sudo dpkg &#8211;reconfigure. That should clear the upgrade flags and fix any broken packages."

Boot-Repair has you chroot into your install for a full purge & reinstall of grub. Then run the update & dpkg commands.
Or:
      
 drs305 chroot to purge & reinstall grub2
[http://ubuntuforums.org/showthread.php?t=1581099](http://ubuntuforums.org/showthread.php?t=1581099)
