### Post by Ub1476 on 2008-04-23
Weeell.. I wouldn't say **lack** of pnp support. At least as I have experienced, everything I plug into Ubuntu works. For instance, in Windows XP I have to install the driver shipped with a CD for my printer to work. In Ubuntu I just had to plug it in, because the driver was already in the kernel. So, more likely, not bullet-proof support _at all_ of some devices in this world (cause those who has support is already in the kernel).

Terminal isn't really necessary either, but it's the quickest and easiest way to guide a person through trouble-shooting, or use the system. Many new Linux users will for instance search for a program on the web, and then only find the source code for it (where you have to compile and use the terminal), but almost every program is already in the repositories. For summary, in general usage, terminal is not required. If problem occurs you should be happy it's there:lolflag:

I think the largest step over to Ubuntu is to stop Window habits, like:

Searching for installable files on the internet
Frequently checking for viruses
Learn to live without iTunes
Manually fragmentation is not necessary. 

Also some Linux tips:

A root user (sudo/su) has the power to do anything on the system. Be careful with commands you write!
Stick to the forums or officially Ubuntu documentation unless you are an advanced user. Using outdated guides for different things is not recommended.
Never install a graphic driver manually.
