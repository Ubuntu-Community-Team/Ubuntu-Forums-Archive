### Post by MCVenom on 2010-04-29
I wouldn't suggest Wubi for long term use... You won't be able to hibernate Ubuntu, and your Ubuntu install will be at the mercy of your Windows partition -- so if something happens to the Ubuntu 'filesystem file' in it, you'll lose Ubuntu. Other things may put it at risk in such a position as well. Another thing I might consider is that someone might be able to get in your Ubuntu partition from Windows, which could cause concerns if you have any sensitive info in Ubuntu and Windows is compromised.

As for firewalls/antivirus; they aren't really necessary, but I'd recommend Avast Antivirus if you really wanna be on the safe side ;)

P.S.: Dual boot guides can be found with a quick Google search. I'll post some if you tell me what version of Windows you're running, and for most of them if you follow them carefully you'll have no problem setting it up. If you do have questions though, by all means ask them here! :D

### Post by Geoff918 on 2010-04-29
> **launchy said:**
> hi, im a soon to be a ubuntu user. and i have a couple of questions for you..

1) i want to know if wubi can be used to install ubuntu for long-term use? considering that i have no experience in partitioning a hard drive.

2)  i have read that linux is safe and does not need an antivirus but what kind of security tools must i use (if i have to) to keep my data/system safe from intruders/hackers or any security threats?

thank u!

That's a complicated question. There are a lot of answers to it as well. The better question is what specifically you are doing or attempting to do with your system.

If you're going to be hosting web sites, etc. then the answer is very different than if you are using your system simply as a desktop install with email and web surfing, etc.

You should keep to installing software from the repositories--if you add a bunch of third party repositories that may be fine. But, then again, it may not be. Unless you want to review the code yourself, it's probably best to just let the experts deal with that.

The more software you have loaded the larger the "vector of attack". Which is to say, that a poorly designed program or a program running with improperly elevated privileges are a risk to your whole system.

There are programs like lsof that can help you monitor your open ports, etc. Most additional software you install to help "secure your system" may actually open an avenue of attack. So, you could get packet sniffers, anti-root kit programs, etc. And, those may be the way in which an attacker can get access to your system. There's great desire to exploit these programs because most servers will have them on their install. So, these have in the past been used to attack systems they are intended to protect.

You may use UFW (uncomplicated firewall) which is installed by default. A good starting point is this:

sudo ufw default deny

That will deny access to your system by default. If you then wanted to run a web server you could then issue a command to open that port:

sudo ufw http allow [or sudo ufw allow 80--or something to that effect]

I might also suggest "sandboxing firefox"

There are app-armor profiles that have been around since Ubuntu 9.10, which you can issue the following command:

sudo aa-enforce /etc/apparmor.d/*  -- that will enable all profiles that exist in the default install (or any that you have added additionally)

What the sandboxing will accomplish is limiting access to your system as Firefox will run agnostic to the rest of the system, and if it does itself get compromised it can't get access beyond what it has access to within the apparmor profile.

I personally also like to use privoxy which is a web/ad content filtering proxy service. It's available in the repositories:

sudo aptitude install privoxy

You then have to set up firefox or whatever web browser to connect via the proxy.  127.0.0.1 :: 8118

You may need to change a setting in the privoxy configuration file. There is an issue where it calls localhost, and you need to change that to the enumerated 127.0.0.1

Anyway, I hope some of this may help.

I would be careful what services you offer to the world (e.g. hosting, ftp, http, ssh) as if you do that and check your logs you'll quickly find out the world is a very nasty place. There are botnets and users all over the world trying to brute force attack your machine.

The default Ubuntu install does not have any open ports. If you load apache, at least port 80 will suddenly open. Install SSH and port 22 opens. And, there will be attack attempts, I can guarantee it.

Use a good password.

Avoid running as root.

Check your logs.

Consult the documentation on programs you might run.

It's what you don't know that is your biggest problem. Not what you *do* know.

I've worked a bit with OpenBSD and that is quite ridiculously secure. But, also much harder to use. (Well, not after some *nix experience, I guess).

I guess my last item is: you may wish to learn some IPTables commands. I myself have not even gotten all that good with IPTables. But, that's the best thing you can do when it comes to firewall rules, etc.

I hope this long-winded response may help!

Wubi is meant to be a trial environment, not a permanent solution. The Ubuntu installer that is on the LiveCD will help you divide your hard drive and create a dual boot system. Your Windows system stays intact except for replacing the bootloader with GRUB, which provides a menu allowing you to choose which OS to run. Your NTFS Windows partitions are available to Ubuntu this way, but won't be if you use Wubi.

Ubuntu is safer partially because the user base is a smaller target for malware developers. You still need to pay attention to most security issues; there is malware that targets Linux, although not nearly as much as targets Windows users. You need to use anti-virus checking to avoid becoming a vector for attacks on Windows users you may exchange data files with. 

**ClamAV** is available in the Ubuntu Software Center under the simple name of "**Virus Scanner**." The **ClamAV** [I]"Ubuntu packages are maintained by Ubuntu MOTU Developers. **ClamAV**  has been officially included in the Ubuntu distribution since the first  Ubuntu release.... These are official Ubuntu packages and supported by  community developers."
[/I]
The firewall is on by default. There are several tools available to configure your firewall if necessary, such as **Firestarter** or **gufw**. I also use "**[Shields Up]("https://www.grc.com/x/ne.dll?bh0bkyd2")!**" to check for exposed ports or a vulnerability.
