### Post by bscbrit on 2008-11-08
A list of good questions.  Before we start looking at them each in turn, you must bear in mind that software written in 2008 expects the sort of computer power and standard of hardware that is currently available.  There will always be limitations when you try to run it on hardware that is of a much lower spec.  (You wouldn't expect Vista to run on your 1998 computer, would you?)  Having said that, linux in general and Ubuntu in particular, can often be run on older computers but without all of the bells and whistles that you might get when using it with a more modern box.  In short, the specification of your computer is below optimum for running the latest Intrepid Ibex (Ubuntu 8.10) with all of its capabilities enabled.

There are two options really, run a lesser spec Ubuntu or upgrade your hardware to the minimum standard required by Intrepid.

1.  Linux is as user friendly as any other OS.  It uses a graphical desktop and has all of the functionality (and more) of other desktops.  BUT, whereas you have probably had years of getting used to Windows, Ubuntu will be unfamiliar and will cause you to have to re-learn how you do certain tasks.  Linux is NOT the same as Windows - if it was there would be no point in having it.  It has many advantages - in most people's opinion on this forum at least - over Windows but there is a learning curve associated with using it.  Do you remember when you were younger and you first learned to drive a car?  Yet now you probably don't consciously think about when to press which pedal. What at first seems strange and different soon becomes second nature.

2.  Freedom - I think that you will find it difficult to have more freedom than you will get with linux.  Freedom to configure your computer exactly as you want it, freedom to use whatever software you want to use, freedom to share what you know and produce with others, and to benefit from them at the same time.  And, perhaps equally importantly, the freedom to upgrade your system at no additional cost or worries about license fees etc.

3. There are very few viruses that can affect linux and none that I know of that are in the 'wild'.  You will still be vulnerable to social engineering tricks (although to a much lesser degree than a Windows user).  For example, if someone presents you with a false website asking for your banking information, if you type the data in then it has been compromised.  But antivirus and protection from malwear is not required.  It is sometimes used on servers to prevent Windows viruses being transferred from one Windows computer to another via your linux machine, but the linux is unaffected by the virus.

4.  2008 C#.  I doubt it, but it might be coaxed to work under WINE.  However, there is no shortage of IDEs, compilers, editors or whatever you need.  I'm sure that there will be something that meets your needs.

5.  To make the most of the current distributions you will certainly need more RAM.  I run Intrepid on 7 machines, some of which have much less powerful CPUs, but I find that 512MB is really the minimum that I can use if I want reasonable performance with the exception of my servers, which do not use a graphical interface at all. For them 256MB is plenty although more memory always gives me a better performance.  In the main, I write programs and do not play games, nor do I need the latest whizz-bang special effects.  I use 32MB and 64MB graphic cards with no problems.  But don't expect to use graphics-intensive programs without having the appropriate hardware.

6 How good is it?  Well, I have 7 systems all using linux, and I manage another networked system for a local charity.  We all love it and it does everything that we need it to do.  It is flexible and robust. But, there is the initial learning curve.  However, I have seen children, pensioners and everyone in between use it successfully without significant problems.  Most people can use it straight away although they look in the wrong places for the icons that they expect to find on the Windows-style menu.  It takes them 5 minutes to get used to the fact that menus are often in different places.


I hope that answers your questions at the right depth without drowning you in unwanted information.  If you have specifics that you need answering, come back and ask.  There are numerous experts (I exclude myself from such a stalwart list - I'm just an enthusiast in comparison to them) who can give you the advice that you are seeking.

### Post by bscbrit on 2008-11-08
Partitioning is nowhere near as difficult as it used to be.  Ubuntu can be used to sort out the partition itself; it can be told to use the unused portion of your existing Windows partition or, if there is empty space or an unused partition available, it will quite happily use that as well.  Or you can set it up manually.  There is no point in going into too much detail before you actually get round to doing it.  The OS will live quite happily in 4GB (even less but I cannot quote a specific figure - I'm sure that someone else will be able to though....!)  Of course, I do not know how much personal data you currently have or how much programming you intend to do but, regardless, its needs are very modest.

There is always some danger in repartitioning - just as there is in getting out of bed - but disaster stories are still few and far between.  You will read of them on these fora because it is a place where people come when they have a problem.  It is just the same as the local garage being full of broken cars- it doesn't mean that cars aren't well made or are unreliable.  This forum is like that, and it can paint an incorrect gloomy picture.

You might find that running the LiveDisk from your relatively slow DVD drive is a bit tortuous but I have no doubt that it will work.  Other than that, you DVD drive will be fine.


Your proposed laptop is more powerful than anything that I have here - and I have got 8.10 on 4 machines and I am still running 8.04 on 3 others.  There are a few niggley bugs on 8.10 with some of my older and more unusual hardware but they will be sorted out in the next few weeks, no doubt.  But any of the recent Ubuntu releases (which includes the various flavours Ubuntu/Kubuntu/Xubuntu etc which differ in their desktops but contain the same essential kernel and structure) will be fine on your laptop.

### Post by bscbrit on 2008-11-08
LTS indicates Long Term Support.  It tells users (and especially businesses) that the release will be stable and maintained for at least 3 years and that it will be replaced with another LTS version.

Ubuntu 8.04 (known as Hardy Heron) is a LTS version.
Ubuntu 8.10 (known as Intrepid Ibex) is a standard version.

Incidentally, the release version numbers indicate the year and month of release. 8.10 is 2008, October, 7.04 is 2007, April.  The names are an Ubuntu tradition that users either love or hate!
