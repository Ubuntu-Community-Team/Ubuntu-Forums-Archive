### Post by coffeecat on 2011-09-05
@Efi Fiery, your original problem was:

> **Efi Fiery said:**
> 
```
sda1: __________________________________________________________________________

    File system:       ntfs
    Boot sector type:  Grub2 (v1.99)
    Boot sector info:   Grub2 (v1.99) is installed in the boot sector of sda1 
                       and looks at sector 248907279 of the same hard drive 
                       for core.img. core.img is at this location and looks 
                       for (,msdos2)/boot/grub on this drive. No errors found 
                       in the Boot Parameter Block.
    Operating System:  Windows XP
    Boot files:        /boot.ini /ntldr /NTDETECT.COM /ubuntu/winboot/wubildr 
                       /ubuntu/winboot/wubildr.mbr
```

You had installed grub to sda1, your Windows partition, where it has no business being. Grub in the boot sector of the Windows partition was interfering with the Windows boot files.

Unfortunately, you may have made the situation much worse with:

> **Efi Fiery said:**
> 
```

sudo apt-get install lilo
sudo fdisk -l
sudo lilo -M /dev/sda1
```

Which would have installed lilo to the boot sector of sda1 as well. This is not your fault - I see that your link advised this. This is an inexplicable error. To fix a Windows mbr, you should use:

```

sudo apt-get install lilo
sudo lilo -M /dev/sda mbr
```

sda, not sda1. But because you now get a black screen with blinking cursor, I think you may have used the latter rather than the sda1 code. Or perhaps you did both. However, fixing the mbr is not the issue here. Or should not have been. Your grub was originally also installed to the mbr and pointing to sda2, your Ubuntu root partition. The reason Windows wouldn't boot from the grub menu was because of the grub code in the Windows parittion. Your wubi installation seems to be a red-herring here. 

Whatever... this is how you fix the contamination of your sda1 boot sector:

[http://sourceforge.net/apps/mediawiki/bootinfoscript/index.php?title=Boot_Problems:Boot_Sector](http://sourceforge.net/apps/mediawiki/bootinfoscript/index.php?title=Boot_Problems:Boot_Sector)

Whether that will also fix lilo in your Windows boot sector, if lilo is in your Windows boot sector, I do not know. Please re-run the boot script and post the new RESULTS.txt to see what has happened. We need to see what is now in the mbr as well as what is in the boot sector of sda1.

**EDIT**: I've had a closer look at the other parts of your link. There are other errors there. You may wish to discard that link if you have bookmarked it.
