### Post by kpkeerthi on 2008-10-01
Add it to InputDevice section of **your touchpad ** (*there may be more than one inputdevice section, add it to the appropriate one) in xorg.conf, like so:
```

Section "InputDevice"
   Driver      "synaptics"
   Identifier  "TouchPad"
   Option      "SendCoreEvents"
   Option      "Protocol" "auto-dev"
   Option      "SHMConfig" "on"
 EndSection

```

And then restart 'X'. Ctrl + Alt + backspace

### Post by jithinkr on 2008-10-02
Had the same problem till yesterday. A tiny hack of the xorg.conf file did the trick. It seems that some other installation reconfigured the xorg. It's fine. Now your scrolling feature is disabled, right. Probably Nvidia driver installation reconfigured it. Never mind. Add this to your xorg.conf file. Add as root. No don't add it to the mouse thing, it won't work (in fact it shouldn't work).

Section "InputDevice"
	Identifier	"Synaptics Touchpad"
	Driver		"synaptics"
	Option		"SendCoreEvents"	"true"
	Option		"Device"		"/dev/psaux"
	Option		"Protocol"		"auto-dev"
	Option		"HorizEdgeScroll"	"0"
	Option		"SHMConfig"		"On"
EndSection

after that add the line 'Input Device "Synaptics Touchpad"' without the single quotes to the ServerLayout section. Restart the xserver and Touchpad scroll should work!

Whats with the SHMConfig?
I have added the SHMConfig Option to the Synaptics Section, so that I can get syndaemon to work. Now the touchpad doesn't trouble me when I am typing. It gets temporarily disabled (for 1 second) when I am typing something. (I run syndaemon -i 1 -d -t -K during startup for this to work.) run man syndaemon for more.

However the tiny button on top of the touchpad is still malfunctioning. Touchpad works even if the glow is red and Ubuntu Help opens up when I press the button to change the red glow to white. Well, i guess that shouldn't happen!:o
Please some one tell me what should I do for this.

### Post by jithinkr on 2008-10-03
Oh!
Good to hear everything is working fine. Now that your problem has been solved, I don't know if you will ever visit this thread again. But for the benefit of anyone else referring to this thread, I need to put something regarding the SHMConfig thing set to On.
When SHMConfig is set to On, then syndaemon can be used to temporarily and automatically disable the touchpad, when you are typing. For that this command has to be run.
 syndaemon -i 1 -d -t -K.
Put this as a startup script in sessions and then everytime you boot this is run, and the touchpad doesn't trouble you while typing! 
Note: the touchpad is disabled for an interval of 1 second during typing.
