### Post by pytheas22 on 2009-04-25
First of all, try changing your xorg.conf file to this:
```

# xorg.conf (X.Org X Window System server configuration file)
#
# This file was generated by dexconf, the Debian X Configuration tool, using
# values from the debconf database.
#
# Edit this file with caution, and see the xorg.conf manual page.
# (Type "man xorg.conf" at the shell prompt.)
#
# This file is automatically updated on xserver-xorg package upgrades *only*
# if it has not been modified since the last upgrade of the xserver-xorg
# package.
#
# Note that some configuration settings that could be done previously
# in this file, now are automatically configured by the server and settings
# here are ignored.
#
# If you have edited this file but would like it to be automatically updated
# again, run the following command:
#   sudo dpkg-reconfigure -phigh xserver-xorg

Section "Device"
	Identifier	"Configured Video Device"
        driver          "vesa"
EndSection

Section "Monitor"
	Identifier	"Configured Monitor"
EndSection

Section "Screen"
	Identifier	"Default Screen"
	Monitor		"Configured Monitor"
	Device		"Configured Video Device"
EndSection
```

(To edit the file from the command line, type 'sudo nano /etc/X11/xorg.conf'.  The only change I made is that I added a 'driver' line in the 'Device' section of the file.)  This should allow you to start X in low-graphics mode.  Does it work?

Please also post the output of this command so we can see which kind of video card you have:
```

lspci -nn
```

(If you don't want to copy all the output, please at least post the line that describes your video card--it should mention something like 'VGA device' or 'video controller'.)

### Post by pytheas22 on 2009-04-26
Sorry that trying to change to the vesa driver didn't work.  However, I think I've found the [appropriate bug report]("https://bugs.launchpad.net/ubuntu/+source/xserver-xorg-video-openchrome/+bug/293326") for your problem--the people there have the same video card as yours and seem to have the same issue.  Fortunately, applying a patch and recompiling the driver is supposed to fix the problem.  To do that, first log in on the command line, plug your computer into the ethernet, and run these commands (if you have no wired connection available, let me know and we can try to do a wireless connection from the command line):
```

sudo ifconfig eth0 up
sudo dhclient eth0
```

This should get you connected to the Internet.  Once connected, run these commands to download the new driver code, compile and install:
```

sudo apt-get install build-essential subversion autoconf automake1.9 libtool
sudo apt-get build-dep xserver-xorg-video-openchrome
svn checkout http://svn.openchrome.org/svn/trunk openchrome
cd openchrome*/src
wget http://launchpadlibrarian.net/24709429/km400_panel.patch
patch -p1 < km400_panel.patch
cd ..
./autogen.sh --prefix=/usr
make
sudo make install

```

Finally, open your xorg.conf file by typing *sudo nano /etc/X11/xorg.conf*.  Look for the line that says:

```
driver      "vesa"
```

and change it to:
```

driver        "openchrome"
```

I don't have a Jaunty system available, but I just did this on Hardy 64-bit and the code compiled and installed without a problem.

Then restart your computer and try launching X again.  Any luck?
