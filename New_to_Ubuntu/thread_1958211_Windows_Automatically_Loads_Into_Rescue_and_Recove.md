Thanks oldos2er.
It may be that the resizing of the XP partition has borked the Windows booting process. Windows doesn't like to have its partitions changed by other programs.

I would suggest repairing the Windows boot loader and getting Windows to boot again. You can then run a chkdsk on Windows. Keep running chkdsk until no errors are reported.
At that point re-installing the grub loader can be done by booting from the Ubuntu live cd/usb and selecting "try ubuntu" then in a terminal run the following ```
sudo mount /dev/sda5 /mnt
sudo grub-install --root-directory=/mnt /dev/sda
``` If the second line reports any errors run this line ```
sudo grub-install --boot-directory=/mnt /dev/sda
```
