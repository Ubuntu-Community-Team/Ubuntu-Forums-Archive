### Post by vanadium on 2008-07-31
Strange the output of /etc/fstab. You probably made a mistake in posting it, because with the /etc/fstab you list, it would be impossible to start up the system. I am missing two essential entries, which for you should look like

```

# /dev/sdb1
UUID=895e7f49-adca-40c0-ac9a-9429ccf4d8c3 /               ext3    relatime,errors=remount-ro 0       1
# /dev/sdb5
UUID=8ef7aad8-b635-473f-8c76-3bce5dd2e7f6 none            swap    sw              0       0

```

These lines would normally follow after the line starting with "proc". This would be the default on a fresh Ubuntu install. Since Hardy, ntfs partitions are not anymore mounted using /etc/fstab, but "on demand" by gnome. You can still mount them via fstab if you prefer that.

### Post by Rocket2DMn on 2008-07-31
Your fstab should probably look something like this:
```

# /etc/fstab: static file system information.
#
#  -- This file has been automaticly generated by ntfs-config -- 
#
# <file system> <mount point>   <type>  <options>       <dump>  <pass>

proc /proc proc defaults 0 0
# /dev/sdb1 (root filesystem)
UUID=895e7f49-adca-40c0-ac9a-9429ccf4d8c3 /               ext3    relatime,errors=remount-ro 0       1
# /dev/sdb5 (swap space)
UUID=8ef7aad8-b635-473f-8c76-3bce5dd2e7f6 none            swap    sw              0       0

# cdrom drive
/dev/scd0 /media/cdrom0 udf,iso9660 user,noauto,exec,utf8 0 0

# Now here are the NTFS drives
# /dev/sda1
UUID=DE4403AB4403860D /media/sda1 ntfs-3g defaults,locale=en_US.utf8 0 0
# /dev/sda5
UUID=8814AD0F14AD0174 /media/sda5 ntfs-3g defaults,locale=en_US.utf8 0 0

```
To initially edit fstab, run
```
gksudo gedit /etc/fstab
```
Before mounting and/or restarting, create the mount points that we listed in fstab for the ntfs drives (you may use more creative names, just don't use spaces please)
```
sudo mkdir /media/sda1 /media/sda5
```
Now try mounting with
```
sudo mount -a
```
If it complains, please post the output back here.

### Post by vanadium on 2008-08-01
...except that you should not permanently "force mount" an ntfs drive: this is only useful as a temporary solution to be able to take of your data. tere are good reasons why Linux won't mount an "unclean" ntfs volume by default. Continue to normally use the ntfs disks this way, and you will risk that the file system becomes entirely corrupt one day.

If you do not have windows anymore, I strongly recommend to reformat these ntfs partitions to ext3, a file system that is fully supported by Linux. This involves 1) backing up the data on the partitions; 2) reformatting the partitions to ext3 and 3) Modifying the lines in /etc/fstab.

### Post by vanadium on 2008-08-01
Follow this carefully and step by step. In case of a problem, report back with a copy of the terminal output.

1) Unmount the two partitions we want to reformat.

```
sudo umount /media/sda1
sudo umount /media/sda5

```

There should be no output. If there is: post output back.

2) Now comes the actual reformatting

```

sudo mkfs -t ext3 /dev/sda1
sudo mkfs -t ext3 /dev/sda5

```

Here, messages will appear on the screen, but there should be no error messages.

3) We update /etc/fstab. For this, you will need the new UUIDs of the partitions. Open a second terminal and type

```

sudo blkid

```

Among other information, you find here the UUID for all your partitions, including your /dev/sda1 and /dev/sda5.

Switch back to your first instance of terminal and open /etc/fstab with root privileges

```
gksudo gedit /etc/fstab
```

Look at the lines with /media/sda1 and /media/sda5. 

3a) Update the UUIDS

The UUID= portion has to be replaced with the corresponding part of the output of sudo blkid. The most foolproof you can do is: just copy the part of the line 'UUID="xxx..."' for /dev/sda1 from the terminal window and paste it over the existing 'UUID=' part for /dev/sda1 in your /etc/fstab.

Do the same for /dev/sdb1

b) Update the file system and the options

Replace for both partitions "ntfs-3g" by "ext3"

From the "options" part of the line, which currently reads "defaults,locale=en_US.utf8", leave only "defaults".

Change "0" at the end of the line by "2". This way, the volumes will be automatically checked during boot (as usual: a quick check normally, but a full check after so many time or boots).

c) Save your changes to /etc/fstab

d) If you are the only user of the system, you could as well mount the partition immediately with ownership for you as user. For this purpose, we change ownership of the mount points to yourself as user:

```

sudo chown $USER:$USER /media/sda1
sudo chown $USER:$USER /media/sda5
[/code

You can type $USER as such. If it is for another user, you have to replace $USER with the logon of that other user.

e) Test whether all is OK by "remounting":

[code]
sudo mount -a

```

Not any output should appear. At this point, your newly formatted partitions are mounted and ready for you to use.

Post scriptum: there are other options. Personally, I would use gparted to delete both partitions and make it into one ext3 partition. Then I would mount that partition under /mnt rather than under /media to get rid of the icon for it on the desktop (you can always create icons to any directory your want). To gain quick access to that partition, I would work with symbolic links from within the users home directory.

This way is the more traditional "unix" way to do it. It is far more flexible if there are multiple users on the system. For a single user system, the more "Windows" like setup I showed was currently the easiest to do and will work fine. As usual, there are multiple ways in Linux to achieve a goal.

### Post by vanadium on 2008-08-02
Your changes look OK. Probably it is an issue that will be solved after reboot when the system rereads the partition tables. You could reboot or try following command befoor the "sudo mount -a"

sudo partprobe /dev/sda

Although I do not think this is a problem, you could also update the partition's system ids (as you see, fdisk -l still reports ntfs partitions on your sda.)

You can update these with the fdisk command:

sudo fdisk /dev/sda

"m" produces help. There you see that "l" lists partition types: "Linux" is 83. "t" changes a partitions system id. You need to select your partition by typing a number, then provide the system id, 83 for Linux partitions. You must quit by typing "w" to write the changed data.
