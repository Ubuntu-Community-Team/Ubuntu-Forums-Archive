### Post by caljohnsmith on 2009-05-03
It is possible to have a dedicated Grub partition without having to create an entire /boot partition, but you have to do it manually; the Ubuntu installer can only create a /boot partition, not a dedicated grub partition. If you would like help setting up a dedicated Grub partition, it would help to first get a clearer picture of your setup, so how about downloading the **Boot Info Script** to your Ubuntu desktop:

[https://sourceforge.net/projects/bootinfoscript/](https://sourceforge.net/projects/bootinfoscript/)

Then open a terminal (Applications > Accessories > Terminal) and do:
```
sudo bash ~/Desktop/boot_info_script*.sh
```
That will create a "RESULTS.txt" file in the same directory from where the script is run, namely your desktop; please copy/paste the contents of the RESULTS.txt file to your next post, highlight the copied text, and click the pound/hash sign "#" graphic button in the Ubuntu forum message box so that the text will get "code" tags put around it. The results of that script will help clarify your setup.

Cheers,
John

I've had a couple of problems with GRUB recently as well, and I *think* I may have a solution for yours. According to [this section of the GRUB documentation]("http://www.gnu.org/software/grub/manual/grub.html#Booting-fallback-systems"), you can specify a different boot sequence as a fallback to your main boot sequence. For example, you can instruct GRUB to boot a different partition if the first one fails. I haven't tried this, but it sounds like exactly the trick you need. Hope this helps!
