### Post by gradinaruvasile on 2010-01-07
Your driver is not correctly installed. The kernel tries to load the "nvidia" module, the drivers loader module but cannot find it. So you have to reinstall the driver.

I recommend renaming the xorg.conf file. On the next boot a new configuration (xorg.conf) will be generated, using the built-in nv driver - that driver doesnt support acceleration.

Boot up and you should be able to login. Then see if you have the option to enable the binary drivers.

### Post by gradinaruvasile on 2010-01-07
No problem there. There are a bunch of xorg.conf backups on all my systems. The system is interested only in the one and only xorg.conf. The rest are garbage.

You say you installed Ubuntu 9.10 fresh. But did you actually install the nvidia driver? You see, the proprietary Nvidia drivers is not installed by default. The one installed is the open source driver that is a generic not accelerated one, and has the "nv" loader module.

You have to install the Nvidia binary (or proprietary, the same thing) driver that has the "nvidia" module. Then check your resolutions and if you dont have the one you need, experiment with copying over your old xorg.conf.

Installing is easy, go to the hardware drivers option in the administration menu, enable the driver and reboot.

### Post by ibuclaw on 2010-01-07
Hmm... this is all rather confusing.

So you have lost your original xorg.conf file?

If so, I suppose there is no harm in clearing out the already existing ones:
```
sudo rm /etc/X11/xorg.conf*
```
That is what I would do for sanity sake first.

If NViDIA is install, you can run xconfig via:
```
sudo nvidia-xconfig
```
And then reboot.

If you can't run nvidia-xconfig ... reboot anyway and install NViDIA Drivers via the Hardware Drivers app.
