### Post by aeiah on 2010-03-17
you'll find a lot willing to help, but what'll help you the most is a decent plan on how to kick this project into action. 

Plan how the laptops are to be structured. do you want every student to have their own profile, or every laptop to have its own profile? if its the latter, you want a storage area for workfiles on the server.. if its the former, you want each laptop to download the student's profile each time, ie a roaming profile. each laptop having its own profile is probably the easiest to manage. you can easily lock it down and make sure people only write changes to their folder on the server.


your server is just really gonna be for hosting student data, and perhaps as a squid proxy so you only download updates and software once and not once per laptop, so there arent any big decisions there.

this forum and the internet in general has a wealth of information. there'll always be people who've done the same thing as you, or have the answer you're looking for. without a plan of how its all going to fit together though, you may end up spending hours running around in circles.

### Post by patchwork on 2010-03-21
In the heydays of UNIX (of which Linux is descended), computers were too expensive/large to give everyone their own.  Instead, a central mainframe handled all of the computing, and each user operated a 'dumb' terminal to access the main computer.  

Although really isn't necessary in this day and age, I believe the ability to operate the system in this way is still present (but I'm not sure of this).  If it is, it would give you the ability to use the server as the mainframe, and each connecting laptop would be the equivalent of the dumb terminal.  The programs, updates, etc would all be installed on the server.  The students would then access the server and actually run the programs from the server (instead of locally).

I don't really know too much about this (it was long before my day), but may be useful if someone else knows how to do this.

### Post by zerhacke on 2010-03-21
I think what Patch is talking about is something like PXE booting, which was around in my day but is no longer used.  The problem with using a PXE or PXE like environment these days are that the laptops are no doubt wireless and it would be a waste to have the laptops tethered down by cables for network booting.  You'd then run into the problem that no wireless card I have ever heard of supports PXE or any Boot ROM.

I'm sure there's plenty of space on the laptop hard drives to support a full install and user profiles, etc.

It is possible to have the Xorg client run on the server and have each laptop run their own Xorg server, but this again is a waste considering that the idea of a remote Xorg client was brought in to facilitate the power of the mainframe/server since the user's terminal was usually too weak to run their own Xorg/X11 client.  I'm sure that's not the case with a brand new 64 bit laptop.

If it were me, I'd just full install on all the laptops, get wireless working on each, set up whatever wireless encryption you want on the router, and do whichever kind of install on the server you like.

LTS stands for Long Term Service. It means that the version will be supported for three years rather than the normal 18 months (five years for the server edition). It means that once you get it set up like you want, you don't have to worry about upgradeing for a longer time.

10.04 will be available next month. The 10 indicates the year, 2010. The 04 indicates the month of the release, 04 = April. Every new version has both a desktop as well as a server edition. It will be at least as stable as any other new operating system and better than most.

If you have a set of identical laptops (or any machines), the solution to large scale deployment is to install on one, and mirror the hard drive to each of the other machines.  I'm sure an Ubuntu guru here can detail how to do this natively; however, a commercial package I've used for this over the years is Acronis Disk Director. /home is where your home folder is. So, if your username is frodo, then your home folder would be:

The /usr directory is mostly where applications and packages are installed, I believe. Look in /usr/bin and /usr/local/bin and you'll see most of the applications you can run on your machine.
