### Post by r-senior on 2013-04-14
> **Aks1 said:**
> How to get run the Windows7 softwares and files on ubuntu 12.04 ? They are not supporting anymore.  Is .exe file runs on ubuntu?
Windows executable files (.exe) do not run directly on Ubuntu (or anything that is not Windows), however, you may be able to use [WINE]("https://en.wikipedia.org/wiki/Wine_(software)").

Documents may or may not open using Ubuntu applications. For example, LibreOffice can open Microsoft Word documents. Standard formats like .jpg and .mp3 can be opened using standard Ubuntu applications.

You may want to research file sharing between Windows and Linux, for example:

[http://ubuntuforums.org/showthread.php?t=2119298](http://ubuntuforums.org/showthread.php?t=2119298)

You can use software that provides equivalent services to applications you use on Windows, e.g. GIMP instead of Photoshop.

[https://help.ubuntu.com/community/WindowsApplicationsEquivalents](https://help.ubuntu.com/community/WindowsApplicationsEquivalents)

For a good general guide regarding the fundamental differences between Linux and Windows:

[http://linux.oneandoneis2.org/LNW.htm](http://linux.oneandoneis2.org/LNW.htm)

Switching to Linux is not for everybody. The most important thing to accept is that Linux is not Windows.

### Post by Impavidus on 2013-04-14
> **logan_2k6 said:**
> ... so to achieve one task that u could do in windows with one program,u will find out that u need more programs to do the same task.
so keep windows your primary operating system and use ubuntu for tests and stuff. 

This is often part of the Unix design philosophy. Instead of making a single program that does everything, but not necessarily everything equally well, you use a chain of programs that are all optimised for the single task they have. For new tasks, you can simply create a new tool chain instead of finding a new program. For example, instead of a single program that can handle data input, macro processing, spell checking, index generation, lay-out formatting, previewing, printing and giving detailed statistics you can have separate tools for all of them, so you can replace one of the steps by an alternative you think is better for your particular needs. (In this case, both approaches are nowadays available on both systems.) Or think of saving a graphics file in an uncommon format. Instead of having to stick with the formats your painting program supports, you can save it to something standard and use a specialised tool to convert it to the latest or most portable format available, maybe even at higher quality or better compression ratio.

Concerning running all major software: it depends on whether you are referring to specific programs or specific tasks. In case of programs: no, they won't run on Linux. In case of tasks: for every common task there is a Linux alternative. For some highly specialised tasks there may only be Windows software available, just as for other highly specialised tasks there may only be software for Linux, OSX, BSD, HP-UX, ..., except Windows.
