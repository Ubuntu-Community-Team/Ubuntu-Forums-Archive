### Post by Vladlenin5000 on 2014-10-11
Hi, welcome.

First things first, how valuable is to you the partition you just deleted? I mean, if you just installed the system and had no personal data on it, why do you care? Just install again. OTOH, if you already had personal data you didn't want to loose then you should have a backup before attempting any other installation as it is _always_ a risky operation.
Anyway, right now, it can't be recovered. That's what encryption does. There's no point in trying to recover that partition as it was so you better assume the losses (if any) an move on.

On another note, you've made several mistakes - newbie mistakes only so don't worry too much about it - along the way. Most of those error stemmed from you overcomplicating things that should be - and are - almost always straightforward.
If you already had Windows 7 installed and wanted to dual-boot with Ubuntu then the first thing is NOT mess with what is already installed. Simply boot into Windows as always, go to disk management, right click the Windows (NTFS) partitions and resize. Then shrink it leaving unallocated space after that partition. Ubuntu installer will later use that space to create its own partitions. Reboot twice in order to allow Windows to run its *chkdsk* tool (check disk) to correct any logical error that might be generated by the resizing. Then, when you're sure the Windows is booting normally with the resized partition, you can boot from the Ubuntu's installation media and select the installation options accordingly. Ubuntu will take care of everything and install a boot loader that latter will allow you to select which OS to boot.

### Post by Vladlenin5000 on 2014-10-11
Ok... Well, you don't "undelete" partitions. As soon as you remove a partition the space previously occupied by that partition becomes unallocated space. You can create a new partition of the same size or less or more than one partition if you want. 
Surely you won't "see" the now unallocated space with the normal tools of either OS and it's only logical: There's nothing to "see" there ;). However, both OSs allow you to manage that and create one or more partitions up to the maximum extent of the unallocated space.

Windows (from memory so it might not be exactly as I'll tell you): Disk Management (Control Panel > Administrative Tools) shows all the drives connected and, inside each one, the partition(s) and the unallocated space. Use the right-click in that space to see the available options. Obs.: Windows should show Linux/Unix partitions, if any, but WON'T recognize them correctly or their file systems, and can't read them without some third-party tools. 

Ubuntu: Use GParted to manage partitions. Select the unallocated space and choose the options according to what you want to do with that space.
Obs.: GParted is available in any live session but, by default, it won't be installed. You need to install it later like any other program (Ubuntu Software Center should be enough and usually GParted is presented in the main page in the "recommended" section).

Now answering your other question, The automated installer usually does a good job (in non-UEFI system) but yes, it's always preferable to do it manually in 'something else'. That way you are sure about what is going to happen and what is going to happen is what you just told the installer to do. It's all about having more control over the process.

Lastly, the question about the order of installations for a dual-boot scenario. This requires a little bit more elaborated answer, hopefully covering all possible situations.:

1. "Bare metal" (AKA nothing installed yet) -> Start by installing Windows. Use its own partitioner to create a new NTFS partition smaller than the total available space. Make it as big as you want but leave enough unallocated space for Ubuntu at the end of the drive.
2. One OS already installed -> As a rule of thumb leave the already installed OS alone and just resize its partition(s) to get free space for the other.
Now, if Windows is already installed the procedure is what I previously commented. Ubuntu will take care of the dual-boot and install its boot-loader allowing you to choose OSs as soon as you restart.
If you already have Ubuntu you can follow the same procedure and install Windows in the free space. However, Windows will rewrite the boot information and as soon as you reboot it'll go into Windows directly (Windows doesn't play nice in dual-boot unless it's with another Windows). You'll need an extra step of again booting the Ubuntu media in live session, mount the drive(s) and update grub. I think the Boot Repair tool can also be used for that purpose. So, in the end, the only difference is this last extra step which is annoying enough to strongly recommend installing Windows first whenever possible.

### Post by Impavidus on 2014-10-12
Can you boot a live disk, take a screenshot of the gparted program and attach it to a post here? It will help us see you current partitioning and find your missing unallocated space. **sudo fdisk -l** should report partitions, not unallocated space. Unallocated space can be deduced as the areas of the hard disk that are not reported as being part of a partition.

The Windows partitioning tools are a bit clumsy, but they do a good job resizing Windows partitions. Don't use them for anything else, as Windows has a tendency of destroying things it doesn't know about. If you use MBR partitioning (most common on Win7 systems), keep in mind that there is a limit of 4 primary partitions and that Windows is eager to create dynamic partitions, which doesn't work with Ubuntu.

As for installing Windows after Ubuntu: I believe that you have to use gparted to create a primary ntfs partition before the extended partition and set the boot flag on it. Then the Windows installer should be willing to install to that partition. As Vladlenin5000 noted, you have to reinstall the Ubuntu boot loader (grub) afterwards using a live disk.

### Post by oldfred on 2014-10-12
When you installed with encryption did you use full drive or just /home encryption?
Full drive always erases entire drive and converts drive to LVM. LVM has some advantages and disadvantages but one of the main disadvantages is that it is more complicated and many standard partition tools do not work. Newer versions of gparted may work now or at least show partitions.
