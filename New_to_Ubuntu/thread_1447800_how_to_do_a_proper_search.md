### Post by malspa on 2010-04-05
Takes practice and creativity.  Sometimes it just takes stepping away and looking at it from another point of view.

Sometimes it's just a matter of finding the right search terms.

This thread reminds me of a few days ago, I was trying to figure out exactly what a script's command was saying.  The command contains this structure:

command a || command b

I could not remember what "||" means, and I couldn't figure out a way to do a Google search for the answer.  I searched on words like **linux pipes scripting** but couldn't come up with anything.

Mentioned it to a friend.  He searched on the words **linux double pipe** and quickly found this:

It's the opposite of '&&', where the second command is executed only if the exit status of the preceding command is 0.

In other words,

|| is equal to OR.

Or, in other words,

run command b only if command a runs unsuccessfully.

### Post by bhmildy on 2010-04-05
I'm pretty good at searching.  I have good luck when I don't try to use regular English, and don't try to describe the problem.  I use key words, and also type errors directly into the search field enclosed in quotes so it searches for the whole phrase rather than a collection of words.

ie. rather than typing "How do I install my printer?"
I type:
ubuntu printer install "officejet 6210"

I put the most important key words in front, and put the rest in in order of decreasing importance.

In a different example, typing in error messages, character and spaces perfectly, takes me to web pages where they're discussing my error.  I never type questions, because search engines look first at the 100's of millions of web sites that have lots of "how do I" on them.

I'd put that error in quotes, and paste or type this in, if this was my error:
ubuntu 9.10 "Error opening directory '/media/cdrom0/"

I heartily (but cordially) disagree that it takes creativity.  I think it takes careful parsing of the essential concepts, editing out the extra stuff, and logic, and prioritization.

### Post by 3rdalbum on 2010-04-06
Sometimes you have to use more technical language.

For instance, the CD/DVD Creator program in Ubuntu is actually called Brasero, so you could try Brasero Doesn't Recognise CD (or whatever your problem is).

If you're talking about putting in a CD into your drive and it doesn't come up onto the desktop, that's called "mounting" - so you could try searching for "Ubuntu 9.10 cd doesn't mount".
