### Post by Frogs Hair on 2011-02-11
If you want to try Ubuntu 10.10 on limited basis download and run from a live cd to test it. Ubuntu may run slow this way.

Another way is using the Windows installer , which installs Ubuntu inside the Windows partition. This is meant to be used on a trial basis and is easily removed from Windows . Wubi is included in the Ubuntu ISO or can be downloaded separately .

A traditional dual boot is the most complicated but stable method . Virtual box is another method.

All the methods except Vurtual box can be tried with the live cd , VB requires additional software.

[https://help.ubuntu.com/community/Installation](https://help.ubuntu.com/community/Installation)

[https://wiki.ubuntu.com/WubiGuide](https://wiki.ubuntu.com/WubiGuide)

### Post by thefasterblueone on 2011-02-11
Here is the tutorial I used when I installed Ubuntu 10.10 32 bit.

[http://news.softpedia.com/news/Installing-Ubuntu-10-10-160966.shtml]("http://news.softpedia.com/news/Installing-Ubuntu-10-10-160966.shtml")

I have a dual boot Vista 64/Ubuntu 32 on seperate HDD. At first I installed Ubuntu 32bit and everything was working great then I decided to use Ubuntu 64bit and noticed it was hanging and freezing occasionally, so I reinstalled Ubuntu 32bit and It's been working perfectly.

I think most people will agree that it is best to install using the cd.
And you can "test drive' Ubuntu before installing by selecting "Try Ubuntu" when you insert the live cd. When you "test drive" it Ubuntu will be running from your systems memory so it will seem a little slow, but when you install to your harddrive it will run at full speed. :D

After you have Ubuntu up and running you can start customizing it!

[http://members.iinet.net/~herman546/p20/GRUB2%20Splashimages.html]("http://members.iinet.net/~herman546/p20/GRUB2%20Splashimages.html")

If you have any questions along the way just ask here at ubuntuforums or google.

There is a learning curve but in the end you won't be disappointed.
