### Post by Double.J on 2012-03-02
Hi there, Welcome to the forums!

As the others have said, it's the password you set up during install for your user account.

It's a bit of an adjustment at first, but you soon get used to it and it feels quite natural - it's just a security feature so that no program or other user can make changes to your system without your consent.

It all goes back to linux's origins - you used to change user to a "super user" - the account you did your work in wasn't able to make changes to the system for safety reasons, so you switched to the Superuser or "root" account. These days Ubuntu doesn't enable the root account, just the one you set up and others you add  - it then uses a package called sudo to let you temporarily have the "privileges" of the old root account - amongst other things, it means each user uses their own password not a shared one and you can control what each user is able to do.

Hope it helps!
