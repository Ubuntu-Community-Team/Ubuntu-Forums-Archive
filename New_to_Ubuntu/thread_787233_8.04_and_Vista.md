### Post by lemming465 on 2008-05-09
No, windows doesn't have to be on the first partition.  It probably does have to marked "active" though.  Grub syntax for /dev/sda2 is going to be (hd0,1). So tom56's  grub.conf boot paragraph of:```
title Windows Vista
root (hd0,1)
makeactive
chainloader +1
```is exactly the right one.  If that isn't working, then you are going to have to use the Vista install DVD to repair the Vista boot menu.  You are right to fear that will temporarily make your Linux partition invisible, but do not despair; use your Ubuntu install CD to put Grub back on the MBR.  A simple set of instructions that should work for your case are [here]("http://www.howtogeek.com/howto/ubuntu/reinstall-ubuntu-grub-bootloader-after-windows-wipes-it-out/").

Try using this recovery CD to restore the Windows bootloader - [http://neosmart.net/blog/2008/windows-vista-recovery-disc-download/](http://neosmart.net/blog/2008/windows-vista-recovery-disc-download/)

Boot it and pick your language options ([http://tinyurl.com/5azty9](http://tinyurl.com/5azty9)), then click "Next".
At the next screen ([http://tinyurl.com/57ubd6](http://tinyurl.com/57ubd6)) pick "Repair your computer".
Pick the operating system ([http://tinyurl.com/5qncjj](http://tinyurl.com/5qncjj) - there should only be one option here), click "Load drivers" then "Next".
On the final screen pick "Startup Repair" ([http://tinyurl.com/4qvsvg](http://tinyurl.com/4qvsvg)) and follow the instructions.
If at any time it asks you to do "System Restore ([http://tinyurl.com/3pqsnv](http://tinyurl.com/3pqsnv)) click "Cancel" as this will not help and will just remove apps you installed.

When I used it for a similar problem it fixed it and it didn't even wipe out grub (the Ubuntu bootloader), though obviously YMMV. If you do lose grub then it can be restored using the Live CD as mentioned above.
