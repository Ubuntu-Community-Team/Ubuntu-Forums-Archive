### Post by 3rdalbum on 2010-10-19
Faulty updates are very much a rarity. I can't remember ever receiving a post-release update that killed my system, and I use the 'proposed' repository too (which is the bleeding-edge testing ground for updates)

If you're still wary, then stick to the security updates. Security updates are basically very minimal changes that certainly shouldn't cause any problems.

If you'd like to take things with baby steps, then just install the kernel update to start with. If that screwed anything up, then you could just boot into the previous kernel from the GRUB menu. If the new kernel works fine, then you could try installing the other security updates too. However I must stress that security updates shouldn't cause any problems, and they fix at least one security problem :-)
