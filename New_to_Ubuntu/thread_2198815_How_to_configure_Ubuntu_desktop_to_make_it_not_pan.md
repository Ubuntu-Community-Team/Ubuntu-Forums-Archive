### Post by Impavidus on 2014-01-11
I think you can switch of moving the cursor and/or windows to the next workspace when reaching its edge. I don't use the default desktop, so I can't tell you exactly how, but on Xubuntu it's found in Settings &#8594; Window manager &#8594; Advanced &#8594; toggle "Wrap workspaces ..." I think in Unity there must be something similar.

I like workspaces, and liked them the first time I used them back in 2002, 9 on a 3×3 grid. Now I've got 15 of them in a 3×5 arrangement. They allow me to keep several application open without all of them cluttering the same workspace. For example, I may run a full screen web browser on the leftmost workspace, a full screen pdf viewer with documentation on the right one and a transparant text editor on top of previewer in the middle, and I can switch with a single ctrl+alt+arrow to any of those without ever having one window sitting on top of another except where I want it. If I were to use a single workspace and switch to the web browser, the web browser would come on top of both the text edtor and the previewer. Switching back to the text editor would require me to first put the previewer on top and then the text editor, in that order. That's several key presses or mouse clicks, and because you easely do them in the wrong order you may have to do it again. If you do something for which you need 12 windows, having multiple workspaces is even better. Having all windows on a single workspace, or even on 4, would make things horribly cluttered.

But of course, if you can't get used to multiple workspaces or if you don't like them, stick to a single workspace.
