### Post by lswest on 2009-01-14
Try this:

```
sudo gedit /etc/hal/fdi/policy/synaptics.fdi
```
it should open a blank text file.  Enter the following:
> <?xml version="1.0" encoding="ISO-8859-1"?>
<deviceinfo version="0.2">
 <device>
  <match key="input.x11_driver" string="synaptics">
   <merge key="input.x11_options.SHMConfig" type="string">true</merge>
  </match>
 </device>
</deviceinfo>

more info: [https://wiki.ubuntu.com/X/Config/Input](https://wiki.ubuntu.com/X/Config/Input) and [https://help.ubuntu.com/community/SynapticsTouchpad#shmconfig](https://help.ubuntu.com/community/SynapticsTouchpad#shmconfig)

The thing in Intrepid is that it uses HAL to manage this stuff, and fdi files to configure xserver input without having to edit the actual config file.  Try logging out and back in or turning your touchpad on and off (if it can do that), and it should work.

### Post by iaculallad on 2009-01-14
Let's try this. Open your terminal:

We will create a backup of your current xorg.conf file before messing it up:

```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.backup
```

Then, open the file for editing:

```
gksudo gedit /etc/X11/xorg.conf
```

I've had notice that you don't have the "InputDevice" section which is needed by your touchpad device, we will try adding it on your xorg.conf file.

Add the lines of text below on the last part of your xorg.conf file:

```
Section "InputDevice"
        Identifier      "Synaptics Touchpad"
        Driver          "synaptics"
        Option          "SendCoreEvents"        "true"
        Option          "Device"                "/dev/psaux"
        Option          "Protocol"              "auto-dev"
        Option          "HorizScrollDelta"      "0"
        Option          "SHMConfig"             "on"
EndSection
```

Save and Close the file.

Still on your terminal, issue the command below to disable the touchpad:

```
synclient TouchpadOff=1
```

To enable it back:

```
synclient TouchpadOff=0
```

HTH.
