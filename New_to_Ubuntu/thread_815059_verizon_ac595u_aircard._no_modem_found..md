### Post by bwhite82 on 2008-06-01
Adam,

That file configures your dialup connection (which is what Linux considers your Verizon device -- a dialup modem). Experimentation is sometimes part of the process, which is why I had you make a backup of the file. I understand that you're new, but if you'll stick with Linux, you'll look back at this later on and smile. At any rate, open back up that file:

> sudo gedit /etc/wvdial.conf

Get rid of all the text and replace it with the following (copy and paste):

> [Dialer Defaults]
Modem = /dev/ttyUSB0
Baud = 115200
Init1 = ATZ
Init2 = ATQ0 V1 E1 S0=0 &C1 &D2
Init3 = AT+CFUN=1;+CGDCONT=1,"IP"
Area Code =
Phone = *99#
Username = 
Password = 
Ask Password = 0
Dial Command = ATD
FlowControl=NOFLOW
Stupid Mode = 1
Compuserve = 0
Force Address =
Idle Seconds = 3000
DialMessage1 =
DialMessage2 =
ISDN = 0
Auto DNS = 1

Save and then exit the file.

Open up gnome-ppp, goto 'setup'. Make sure device is set to: /dev/ttyUSB0
Type: USB Modem
Under 'Networking' tab, make sure 'Dynamic IP' is checked.
Click 'Close'
Making sure the Username and Password fields are blank and the phone number field is: *99#
Click 'Connect'

### Post by isaidi on 2008-06-08
i have the same card but with Telus (Canada)..
configured it with ppp/pon  works great! :)

gnome-ppp and wvdail and gnome-network manager were acting some what wierd.


I just used the ppp scripts available at sierra website

[http://www.sierrawireless.com/resources/support/Software/Linux/ppp-scripts.tar.gz](http://www.sierrawireless.com/resources/support/Software/Linux/ppp-scripts.tar.gz)

[http://www.sierrawireless.com/faq/ShowFAQ.aspx?ID=608](http://www.sierrawireless.com/faq/ShowFAQ.aspx?ID=608)

the card works perfectly fine with ubuntu, so you don't have to worry about drivers and kernel modules/patches etc... just configure your PPP..
Ignore all the jargon on sierra's website, and just download the ppp scripts and follow instructions to install/extract them..

=====From Sierra Website======
# Navigate to the directory that contains pppd-scripts.tar.gz and extract it to the default location by typing the following commands:
```

      $ cd /

      $ tar –zxf pppd-scripts.tar.gz    

```
# Switch to root and copy the files to the ppp/peers directory by typing the following commands:
```

      # su

      # cp –r ./ppp /etc/

      # cd /etc/ppp

      # chmod a+x ip-up.local ip-down.local    

```
===================================

extra step i added so you can launch from normal user, and allow resolv.conf to update.. (not sure why i had to do this)

```

sudo chgrp dip /etc/ppp/peers

```

now just use the following from prompt to connect

```

 pon cdma

```

or if you copy /etc/ppp/peers/cdma to /etc/ppp/peers/provider
so it becomes your default.. then all you have to do is just run >pon


sorry for crude guide.. it is late.. remember to make a back-up of /etc/ppp before you do anything.. so you can revert back...

If i get a chance I'll post a more detailed step by step guide.. hopefully this is enough info to help.

### Post by lazyart on 2008-07-07
The device is made for windows and has a set of minimal drivers on it.  If you plug it into a Windows box you can install the drivers that reside on the card itself.

Well obviously they are no use to us, but when plugged in the aircard first emulates a CD-ROM.  Does Nautilus open automatically when you put in the aircard?  Is does for me with a "disk" named SMB Lite with a version number (don't have the info in front of me right now).  That is the unit you have to eject.  Once that is ejected the aircard stops emulating a CDROM and becomes the unit that works with KPPP.

Have you set the phone number and account info correctly to let it log into Verizon's network?  If you've done all of that, and wifi/networking is turned off you should be rolling along.  Though it doesn't make a whole lot of sense, networking should be turned off to use the aircard.  I always establish the connection before I open FF or Thunderbird.  Everything just falls into place correctly that way.
