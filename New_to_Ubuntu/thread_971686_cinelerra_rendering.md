### Post by bobnutfield on 2008-11-06
Cinelerra is a very difficult program to use.  I was never able to get it to render properly.  It has made a lot of progress since some of the original developers have now branched off and are now maintaining it as Cinelerra.org.  It is continually improving but is still very buggy.  What I have done in the past is save my work as Mpeg-4 and render it with another program, like DeVeDe (much simpler to use).  Someone has also written a tutorial for Cinelerra (though somewhat outdated) here:

[http://robfisher.net/video/cinelerra1.html]("http://robfisher.net/video/cinelerra1.html")

### Post by satupe@gmail.com on 2009-01-08
The only method I have been able to render Cinelerra files in Hardy 8.04 is using "RAW DV".  Then, you can import the "dv" files into QDVDAUTHOR, and then make your DVD.  This is a real pain after being able to make "one render" when I was using Gutsy 7.10!

If you're running Gutsy 7.04 or 7.10, this link provides good instructions for rendering DVD, flv, mpeg, etc...

[http://crazedmuleproductions.blogspot.com/2007/06/beginners-guide-to-exporting-video-from.html]("http://crazedmuleproductions.blogspot.com/2007/06/beginners-guide-to-exporting-video-from.html")

**Hardy and Intrepid friends...**  I had no problems rendering in Cinelerra or KDENLIVE when using Gutsy 7.04. However, I have been unable to install the necessary Cinelerra Quicktime libs in Hardy due to "conflict errors".  I've also had serious issues with KDENLIVE, which ran GREAT in 7.04.  I tried Intrepid 8.10, but I never was able to get multi-track recording to work with my MAUDIO (ice1712) sound card (a necessity for me!!). It seems to me that Cinelerra and KDENLIVE do not run properly in Hardy due to "Ubuntu issues". I hope somebody proves me wrong!
