### Post by caljohnsmith on 2008-09-02
Sometimes Grub does not see the order of HDDs the same as BIOS, so try adding the following entry to your menu.lst:
```
title Microsoft Windows XP Professional (hd2)
root (hd2,0)
makeactive
map (hd0) (hd2)
map (hd2) (hd0)
chainloader +1
```
If that doesn't work, let me know what errors you get when you boot it from the Grub menu.

I think part of the problem may be having that 500 MB drive connected sometimes, and sometimes not. Is that a USB flash drive? I would disconnect that, and then you should be able to boot windows with what you originally had:

Give it enough time to boot, and if that still doesn't work, let us know exactly what happens. So you can still switch the BIOS booting order to boot the Windows HDD, and it boots fine?
