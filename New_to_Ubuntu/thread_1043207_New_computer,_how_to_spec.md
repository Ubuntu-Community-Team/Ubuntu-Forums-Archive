### Post by stderr on 2009-01-18
I always think it's better to build a PC yourself; that way, you pick decent parts, and have opportunities for upgrades as a result. Plus, you know your way around your PC :) plus, it's really pretty simple.

The HP sounds rather weird to me - wouldn't go with that. I'd ideally like all the same size memory modules (not 1GB modules and 512MB modules) (for dual channel to work). And I haven't seen any 5,400rpm 1TB hard drives. 5,400rpm is old, old tech... and 1TB is pretty new. You really can notice the difference between 5.4krpm and 7.2krpm hard drives. I would not even consider purchasing a 5.4krpm drive.

Whilst the Dell has a better HDD, it's handicapped in other areas. The CPU's slower (but still not too slow!, and 3MB L2 cache is not bad). BUT it has integrated graphics... really not ideal.

And both are using all 4 memory slots. I would have expected a new system to use 2 2GB RAM modules, leaving 2 modules free for upgrades. If you think about it, if you want to upgrade the RAM in the future, with 4 slots full of 1GB modules, the only way to do it is to throw all or most of the modules away and buy larger (2GB+) modules. Still, 3 or 4GB total should be ample for most things for a while.

It really depends what you want to do with your system. Do you care about future upgrades? Do you do a lot of gaming? Do you do a lot of encoding/decoding/encrypting/decrypting? Do you want to run some virtual machines?

32/64-bit... personally I'd say most apps have 64-bit support now. I may move my main box to 64-bit in a year-ish. It's up to you... 64-bit will offer a small speed boost, and if you get 4GB of RAM, only 3.5GB would be addressable under a 32-bit system. I still think in *some* cases it can be a bit more of a PitA, but like I say, it really depends on what you use your system for.

### Post by stderr on 2009-01-18
Well, TBH, in that case you probably won't suffer too much from integrated graphics. You could probably use that to your advantage - integrated gfx is much cheaper than getting a reasonable graphics card. And if you decide you do want a dedicated gfx card in the future, it would be elementary to plop one in a PCI-Express or AGP slot.

Both processors will be absolutely fine for what you want.
Again, either 3 or 4GB of RAM will be absolutely ample.

I don't think you'd run into any compatibility issues with either of those systems.

I really would steer clear of 5,400rpm HDDs though... that's one of those things... it makes a larger difference than you'd think!

@LowSky: Agreed. However, buying systems from the likes of Dell introduces its own problems. If you want to be able to properly upgrade your system in the future, Dell won't give you that. HP probably won't either. 

As you mention, gfx cards, HDDs, sound cards, optical drives... all easy to upgrade regardless. But Dell and other manufacturers will use cheaper motherboards with few features making upgrading some parts difficult. And AFAIK Dell still uses their own weird PSUs?

Ultimately, when you've got a purchased system with a crap PSU, limited motherboard and small, restrictive case, it's normally easier to rip everything out and start afresh. Hence why building from scratch is always far, far better, in terms of uprgadeability in the future. 

As you say, though, it's not for everyone, and if (s)he doesn't want to go down that route that's fine. I just hate when people unnecessarily shy away from building one themselves, thinking it'll be far too "techie" and complicated, when they're actually more than capable :) 

But, as we say, that Dell system will fulfil what you need it for. There may well be better options though. 2GB RAM modules would be something to look for, as it will save you money in the long run if you decide you want more RAM.

### Post by Kevbert on 2009-01-19
Regarding memory you really should match up the memory (not mix and match memory speeds).  So, for example, if you choose PC2 5300 non-ECC memory (fastest) it's best that all memory sticks are this type. The memory boost will only show up if you are doing memory intensive tasks.
