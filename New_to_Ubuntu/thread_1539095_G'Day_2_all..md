### Post by k3lt01 on 2010-07-26
I have been asked to supply a link to why I do not suggest things like UbuntuTweak to noobs. Instead of doing it via PM I thought I'd post it here for others to see.

[Eight Fatal Mistakes]("http://sites.google.com/site/easylinuxtipsproject/fatalmistakes"). I agree with much of this purely because I have seen noobs bork things because 1. nothing has been explained to them, and 2. well meaning people make suggestions based on 1 thing not realising that tools like UT have many aspects.

I don't have anything against UT, I think it has its merits, but I do think people need to learn the basics before using multifaceted tools such as UT. Note when I say the basics I'm not talking about command line-terminal things, instead I'm talking about using tools like Synaptic and/or Ubuntu Software Centre which are already installed and the packages available through them natively have been checked by Canonical-Ubuntu-this community for compatibility with the system they are for.

### Post by saphil on 2010-08-01
If you want to get out of the M$ view as quickly as possible, set up a crash-box for yourself.  This could be a second computer or a virtual machine on the first.  Use all the new ideas there.  Then if you break something you are not hurt much by it because this was on the crash-box.  IMO it is important to break stuff so you can learn how to fix it without much help.  Some people on the forum like challenges such as: "I had this error, I tried A, B, C, and D to fix it but it is not fixed.  Here is the lshw output and the valgrind output.  Can you tell me what is going wrong?"  

-Wolf

PS physical hardware-based crash-boxes are preferable in a lot of ways, the least of which being that you can use a live-disk to look at the file system of a crashed OS on a physical box.
