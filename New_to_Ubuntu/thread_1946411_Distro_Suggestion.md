### Post by evilsoup on 2012-03-24
Anything you can run on one version of Ubuntu, you can run on *all* versions of Ubuntu.

(Well actually there are a very small number of exceptions - gnome-pie didn't run properly on LXDE for me, though it probably would by now - but everything you listed there will be perfectly fine.)

EDIT: Oh, and I run Ubuntu 11.04 on a netbook with 1GB of RAM. Unity doesn't work very well (it's probably better now), but gnome-classic with no effects works perfectly fine.

I've read in some places that XFCE isn't that much lighter than gnome 2.X (or even the modern gnome-panel), but LXDE (Lubuntu) certainly is very lightweight. LXDE does its job well, though changing themes etc is more complicated than in XFCE.

### Post by BertN45 on 2012-03-24
Probably most people here have new and recent PCs with 8GB of memory and 4 cores and everything that is older then 4 years is considered old and only feasible for Lubuntu or Xubuntu. 

Basically machines that old were sold with Vista. Vista has been introduced in Jan 2007 and most PCs from that time had dual core processors and 2 GB of memory. Vista is more power hungry then Ubuntu with Unity, especially Vista SP0.

### Post by esantidp on 2012-05-25
I've had better experiences with XFCE than LXDE. For the difference in ram consumption and any noticeable "performance" XFCE is better integrated and more user friendly than LXDE. For example if you want to use Xubuntu's file manager (which is far superior than Lubuntu's) in Lubuntu, you are actually turning Lubuntu into Xubuntu with the added benefits that it's going to take a few extra steps with little or no benefit than going for Xubuntu straight away. XFCE is better supported overall and used by way more people than LXDE. 

My advice would be: check system specs and if it can run Xubuntu, go for Xubuntu.
