> 1.) is Ubuntu any similar to Windows?
In terms of functionality, yes
In terms of stability, yes though I find Ubuntu far more sturdy then windows.

But its very simplistic, if you can use XP's interface Ubuntu's interface isnt that far off.
In terms of application installation the two are very different, Ubuntu has A centralized library of apps called a repository, instead of going to website A to get program A and website 2 to get program 2 Ubuntu uses a centralized library of packages.

It depends on the game really.

### Post by 3rdalbum on 2009-03-13
> **Skol312000 said:**
> I got one more question ans that is if Att internet service will work with Ubuntu, would it be hard to set it up, or should i set it up on Windows first and then uninstall windows and run Ubuntu,

For ADSL internet, you will need an ADSL router. Your ISP may provide you with one - but then, they might only give you an ADSL "modem" that could be incompatible with Ubuntu.

If it has an Ethernet port in the back of it, then you're all set. Follow your ISP's instructions, which will usually go something like this:

1. Plug the router into your computer
2. Open a web browser and type "http://192.168.0.1" to access the router's web interface
3. Put these details into the fields...

That's stuff you can do on any operating system, as you are actually just accessing web pages that are stored on the router.

> im low on money to buy a 64 bit, would 32 work ok?  U all again thank

I think all computer processors are 64-bit now, except for the Intel Atom processor that is used in netbooks. Don't confuse it with "dual-core" - that's something unrelated. It doesn't matter which processor your computer has, Ubuntu will work on it. Single-core, dual-core, quad-core, 32-bit, 64-bit, Intel, AMD... whatever it is, Ubuntu will work.

Also, on the advice another poster gave you regarding Broadcom wireless cards. They might be marketed under a different name - for instance, it might say that you have a "TP-Link" wireless card, but that's just the name of the company that puts its sticker on the card. You will need to find out who makes the "chipset" that is on the card. If it's Broadcom or Realtek, avoid it.

^ I'm the owner of a Realtek that is not properly supported in Linux at the moment.

Well, it all depends on your Internet connection.

We all recommend you to download using a Bittorrent client such as [µTorrent]("http://www.utorrent.com/"), [BitTornado]("http://www.bittornado.com/"), [Vuze]("http://www.vuze.com/Index.html"), etc

Here is the URL where you can download Ubuntu using bittorrent client:

[http://www.ubuntu.com/getubuntu/downloadmirrors#bt](http://www.ubuntu.com/getubuntu/downloadmirrors#bt)

WIth a regular DSL speed and using the bittorrent, it should take like 20 minutes to download.
On the other hand, if you download it using HTTP (from the browsers), then it will take 1 to 2 hours.

By the way, the Desktop CD is the one that you are most likely want to download.

Desktop CD (also known as LiveCD) let's you test the OS before you install. 
Since the Operating System runs directly from the CD, it is slower and it consumes some RAM.

If you have less than 512 MB, you should download the Alternate CD instead.
Alternate CD, is only a install CD, it does not let you test it before installing it.
