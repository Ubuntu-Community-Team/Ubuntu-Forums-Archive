### Post by drs305 on 2010-03-27
If it's a RAID problem I won't be able to help, but here are a couple of other things to try:

At the main Grub2 menu, highlight the entry and press "e" to enter the edit mode. Remove the entire line beginning with "search", then press CTRL-x to boot. 

If that doesn't work, try this. At the Grub2 menu, press "c" to get to the grub prompt.
Type:
```

search -f /vmlinuz

```
See if it finds the kernel.

If not, you can run "ls" to show which drives/partitions Grub2 sees, as well as checking for the grub files. If the second command doesn't see your grub files, try other values for (hd0,1). The third command should display your Ubuntu kernel(s).
```

ls
ls (hd0,1)/boot/grub 
ls (hd0,1)/boot

```

Once you know the partition the grub files are on, try running these commands, using the drive/partition values counting from 0 for drives and 1 for partitions, so sda1 is hd0,1). Change the values in **bold**. For this example, we'll assume sda1.
> set prefix=(hd**0,1**)/boot/grub
set root=/dev/sd**a1**
linux /vmlinuz root=/dev/sd**a1**
initrd /initrd.img
boot
