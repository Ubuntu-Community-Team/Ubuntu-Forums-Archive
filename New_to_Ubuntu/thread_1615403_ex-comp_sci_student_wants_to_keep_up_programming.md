### Post by wongo888 on 2010-11-07
There are tons of free-open source projects that need development help. You should find one you like and then join the developer mailing list or group. Lurk for a while to try to learn the way the thing runs, then try working on an easy bug in the project's bug tracker.

If you like Java, stick to Java. If you are heading into a particular industry, learn whatever they use. The financial industry uses a lot of C++. Scientific computing folks still use FORTRAN because there is already so much code out there (although there is movement in bioinformatics to other interpreted languages like Python).
