### Post by 3Miro on 2011-07-24
Groups are just a way to manage which user can do what to what file. A rough example is: if you want user A to be able to reboot the machine, but you don't want user B to do that. Then you create a group "power" and you make it so that only members of that group can execute the reboot command. Then you make A member of power and B not a member of power.

[http://www.linux.com/learn/tutorials/309527-understanding-linux-file-permissions](http://www.linux.com/learn/tutorials/309527-understanding-linux-file-permissions)
