### Post by sisco311 on 2009-02-24
change the last two lines in fstab
```
gksu gedit /etc/fstab
```
to
```
#/dev/sda5
UUID=C8084008083FF3D2 /media/data ntfs defaults,locale=en_US.UTF-8,dmask=002,fmask=113,gid=46 0 0 
#/dev/sda1
UUID=9EC093B4C09390DF /media/windows ntfs defaults,locale=en_US.UTF-8,dmask=002,fmask=113,gid=46 0 0

```

this will set read/write permissions for users in the plugdev group (the firs user is in plugdev by default). 

to add a new user to the group:
```
sudo adduser *username* plugdev
```

remount the partitions:
```
sudo umount /dev/sda1
sudo umount /dev/sda5
sudo mount -a
```

### Post by neanderthalman on 2009-03-04
Open terminal, and type "id".  That will tell you your user id number.  Here's mine:

```
neanderthalman@Humphrey-Ubuntu:~$ id
uid=1000(neanderthalman) gid=1000(neanderthalman) groups=4(adm),20(dialout),24(cdrom),29(audio),44(video),46(plugdev),100(users),104(scanner),108(lpadmin),114(netdev),123(admin),124(sambashare),1000(neanderthalman)

```

From this, you can see that my id # is 1000.  I think 1000 is typical for the first user account made, then 1002, 1003, etc.

In your fstab entry for the drive, add the option uid=1000.  That will change the drive owner to you (or the first user account).  If you're not the first user account, replace 1000 with your own id number.  You can also change the group using the gid= option.  Here's one of my NTFS fstab entries:

```
dev/sda4    /media/D    ntfs-3gauto,users,uid=1000,gid=100,dmask=027,fmask=137,utf8,umask=002  0  0
```

### Post by bodhi.zazen on 2009-03-05
Looking back at your posts, you can not change the permissions of a single directory or file on a NTFS partition.

NTFS does NOT support permissions.

So you set ownership in fstab with 

uid = owner
gid= group
umask = permissions (you can use dmask and fmask for finer control).

This is applied to all the directories and files in /media/data

You can not apply one set of permissions to /media/data/directory_1 and a different set to /media/data/directory_2
