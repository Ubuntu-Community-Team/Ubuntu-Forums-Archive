### Post by varunendra on 2011-10-15
Ubuntu now uses Grub2 (since 9.10) which no more uses that menu.lst file. Instead it now uses a dynamically generated file (/boot/grub/grub.cfg) which is more advanced and should not be edited manually. If you want to get more info on this, follow these links:
[https://help.ubuntu.com/community/Grub2](https://help.ubuntu.com/community/Grub2)
[http://ubuntuforums.org/showthread.php?t=1195275](http://ubuntuforums.org/showthread.php?t=1195275)

Onto your problem now. Are you able to boot normally into Ubuntu? If yes, then try this command in a terminal:
```
sudo update-grub
```It should give some message about win7 detection. Afterwards, if everything else is okay, you should be able to boot win7 too.

If this doesn't help, please download boot info script from [here]("http://bootinfoscript.sourceforge.net/"), extract and run it as explained on the same page I linked to, and post back the contents of RESULT.txt file generated by the script. It will tell us almost everything needed to troubleshoot the booting problem.

Script is finding RAID data. I would run the commands to remove the RAID.

You installed grub2's boot loader to the Windows partition boot sector - PBR. Windows has to have its boot code in the Windows NTFS partition boot sector.

> sdc1: _____________________________________


    File system:       ntfs
    [COLOR=Red]Boot sector type:  Grub2 (v1.99)

                       for core.img. core.img is at this location and looks
                       for ?? on this drive. No errors found in the Boot
                       Parameter Block.[/COLOR]

    Operating System:  
    Boot files:        /bootmgr /Boot/BCD

Windows keeps a backup of the boot sector in the NTFS partition. Testdisk can restore it if it is valid.

Fix for most, a few have other issues, better than windows fix in many cases as it also fixes other parameters:
If win7 use small 'system reserved' NTFS partition instead of the partition where windows was installed for win7
This has instructions on using testdisk to repair the install of grub to the boot sector for windows from Ubuntu or Linux LiveCD.
[http://sourceforge.net/apps/mediawiki/bootinfoscript/index.php?title=Boot_Problems:Boot_Sector](http://sourceforge.net/apps/mediawiki/bootinfoscript/index.php?title=Boot_Problems:Boot_Sector)


Does your system boot now? If so it is only because you have the grub2 boot loader in the PBR. You need to have grub2's boot loader in the MBR.
I normally suggest keeping Windows on one drive usually sda and Ubuntu on another so each drive has that drive's system boot loader and each drive then can boot without the others. But you have 3 drives with both systems in one.

Which drive are you booting from. Do you want grub's boot loader in sdc, sdb or sda. It can be in any or all of them and boot your install.

If in Ubuntu this installs to the MBR:
```
sudo grub-install /dev/sdb
```
If you install to sdb then you have to set sdb as the boot drive in BIOS.

### Post by varunendra on 2011-10-17
@*thexero*,
You got the mighty jinni itself (oldfred):). I'm sure we'd all appreciate a feedback on how it went.

@*d4m1r,*
Yes you 'can' certainly edit the file manually, but it is recommended not to. Firstly, it is a dynamically generated file so the changes you make may not be persistent (rewritten with every grub update). Secondly, a minor mistake in it may render the system unbootable. That's why it is recommended to make desired manual changes in the /etc/default/grub file instead (and do update-grub afterwards to apply those changes appropriately), not in the grub.conf itself. But of course, if you know exactly what you are doing, then there is no harm in editing the file manually (I myself keep changing the timeouts manually).

### Post by oldfred on 2011-10-17
Boot script shows all, but it has so much info it can be confusing.

My system keeps drives in port order. Or sda is the lowest port on the motherboard. But I only have SATA drives.

But some BIOS with mixed PATA & SATA may not always load them in the same order. Also some BIOS promote a USB drive to sda. In those cases it can get very confusing.
