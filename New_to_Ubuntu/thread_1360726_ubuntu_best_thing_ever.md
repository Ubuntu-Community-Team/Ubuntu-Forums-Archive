### Post by audiomick on 2009-12-21
Hallo.
I haven't tried wubi myself, but from what I have read, no, it isn't as robust as a full install.

If you want to do a full install, you just have to boot from the live cd, not put it in whilst windows is running as you did to install wubi. To do this, you have to check in your BIOS to make sure the cd is enabled as a boot device. (or just start the computer with the cd in the drive. If the cd is enabled to boot, it will boot from the cd. If it doesn't boot from the cd, then you will have to go into the BIOS and add the cd as a boot device.)

When you install, you have the choice between removing windows altogether, or installing as a dual boot.

There is information about these options here.
[https://help.ubuntu.com/community/Installation](https://help.ubuntu.com/community/Installation)
[https://help.ubuntu.com/community/WindowsDualBoot](https://help.ubuntu.com/community/WindowsDualBoot)

It might be worth thinking about the Dual Boot for the sake of your girlfriend, or you might want to try and convince her. I got mine to use Ubuntu by not having a windows on the desktop at home... ;) She is quite used to it now. It doesn't take long.

Bear in mind that if you do an install that removes windows, everything that you had saved in your windows partitions will be gone. You need to back up everything you want to keep before you do anything.

Have fun with Ubuntu, and welcome.

### Post by TBABill on 2009-12-21
Virtue, you are partially correct regarding a LiveCD, but it's more than just copying the file. You will need a program and I am at work and cannot remember the name of it, but it runs in Windows and will allow you to create a "LiveCD", which is bootable. Simply copying the .iso file you downloaded will not allow the system to boot from that CD. I am sure someone will follow my post with the name of the program that creates Live CD's for .iso files.
 
In addition to creating the bootable CD,  you will also possibly need to go into your bios settings (my Acer requires me to hit F2 on startup to "change settings" but yours may differ and require another command, which should be on-screen when you first boot). You may need to move your CD to the top of the list of devices the computer looks at to boot from. Right now your hard drive should be first listed, but placing the CD first and booting from the Live CD should allow you to "test drive" by not installing, and it will also give you the option to install (as the only OS, as a "dual boot", etc...whatever you choose).
 
There is also an option for later to create a Live USB Flash Drive, which is faster, more portable, etc. It's the same data, just on a flash drive. For now I'd just recommend using a CD till you decide to experiment more due to ease of use/learning. I'm a noob myself as of about 2 weeks ago and have now chosen to eliminate Windows and now I operate from Ubuntu only. I have other machines with Windows should the need arise.
 
One note....some hardware manufacturers have not created drivers for linux and some hardware items may not work, or may not fully work as before. I have a USB hub connected to my router, and my Kodak printer is plugged into that so I can share it among all my machines. Linux does not recognize the hub and Kodak has no drivers for Linux so my printing is done via 1) save in Ubuntu to a flash drive, 2) go to a Windows computer, 3) print, 4) return to Ubuntu. I'll be buying a HP printer soon to eliminate that problem, but it illustrates that some hardware items are not yet Linux friendly due to some manufacturers producing software and drivers for Win and Macs only. Sorry I rambled...
