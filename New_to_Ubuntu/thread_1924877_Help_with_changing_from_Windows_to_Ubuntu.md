### Post by flyfishingphil on 2012-02-13
Welcome to Ubuntu James. Just a few thoughts from a "newbie".

First, do try it using the cd method winh8er suggested. That makes it easy to "experiment and learn" before you actually install it. Only drawback is restarting each time. Personally I prefer installing it alongside Win so everything operates "normally" but you still have the Win available if you crash out the Ubuntu.

Second, spend a little time downloading the info on Ubuntu so you can use it a little better. Worth the time to print it. The one I printed out was the Ubuntu Karmic Koala Bible. Great book to have!

Third, don't hesitate in asking questions about how things work, what you did wrong, an improvement you made, or anything else. Sometimes you get back answers that assume you know what they are talking about but that's easy to correct by reminding that you are a newbie.

I dropped Win when I had some major virus and system problems and use Ubuntu now exclusively. All of my business effort, fun stuff, internet stuff is done using Ubuntu. There are a few things that may not be available but, when it's all free you figure out either a way to do it or change your proceedures to something taht will work.

Believe me when I say "It's easy!". I had a head injury years ago and may need to read something too many times to count before it makes sense. Ubuntu is easy enough to use that, even with that problem, I find it a fairly simple, and much safer, OS.

Phil

### Post by Mark Phelps on 2012-02-14
Main piece of advice I would have is to be open-minded about discovering ALTERNATIVES to using MS Window apps.

We get all too many folks coming here, celebrating their being "free of Windows" -- and in their next breath, they want to know how to install and use all their favorite apps -- obviously laboring under the misconception that Linux is, somehow, a free VERSION of MS Windows.

It is not -- and the more folks want it to look and behave like MS Windows, the more they're setting themselves up for frustration in the long run.

So, one challenge will be learning how to use different Linux apps to do the work you used to do with MS apps.  This is not hard, it just takes time and the willingness to learn.

The other challenge, not so easily solved, is one of lack of hardware drivers.  Manufacturers eagerly supply MS Windows drivers with everything they sell, but very few provide Linux drivers.

This is especially true with the latest hardware -- it takes the community some time to get around to developing new drivers.

So, if your hardware works, then you've lucked out; if not, it may take a lot of work to get it fully functioning.

Welcome to the Linux community.

### Post by I2k4 on 2012-02-15
Hate to intervene, but suggest you start out with _Live USB_ instead of CD, the difference being that you can install "Live" on USB _with Persistence_ - this allows you to actually use the operating system to install new software and save settings between reboots:

[http://www.ubuntu.com/download/ubuntu/download](http://www.ubuntu.com/download/ubuntu/download)

The instructions here are great, but at Step 2 for USB the Pendrive installer permits "Persistence" and you should use all the space on the USB above 2GB for it.  Gives you room to play.

These persistent "Live" installs start to go wonky after a few weeks, but make a great learning experience without risking your hard drive.  Agree with others, that the main thing really is not the operating system, as such, but what programs you really need to function and whether there are useful alternatives in the Linux world.  There are many great Linux programs, but if your coworkers or family are all using something else, it can break the deal.

### Post by QIII on 2012-02-15
I would do the following without saving anything important:

1.  Use the live CD/DVD for a couple of afternoons.
2.  Use a persistent USB for a couple of weeks.
3.  Use a Wubi installation for a couple of months. 
4.  Virtualize for a couple of months, do frequent snapshots and try as hard as you can to mess things up as bad as you can so you can learn.  You can always return to a previous snapshot to start over.

Then, dual boot.  At this point you should be confident and you will have Windows handy if you need it.

After that you can decide whether or not you want to go all out Linux.  I've been using Linux for a long time and I still multi-boot.
