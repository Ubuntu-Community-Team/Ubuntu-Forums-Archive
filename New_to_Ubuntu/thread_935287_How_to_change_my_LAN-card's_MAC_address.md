### Post by HermanAB on 2008-10-01
You can change the MAC using ifconfig.  Put a line at the bottom of /etc/rc.d/rc.local like this:
ifconfig eth0 hw ether xx:xx:xx:xx:xx:xx

Cheers,

Herman

If you want to USE A DIFFERENT network card, you have to remove the settings for the old one first. Do this by editing the file "/etc/udev/rules.d/70-persistent-net.rules". Look at your file and REMOVE the part highlighted in red (note, your numbers will be different, but you'll know what to delete) (in other words, delete everything and only leave the comment block):

```

# This file was automatically generated by the /lib/udev/write_net_rules
# program run by the persistent-net-generator.rules rules file.
#
# You can modify it, as long as you keep each rule on a single line.

[COLOR="Red"][B]# PCI device 0x8086:0x294c (e1000e)
SUBSYSTEM=="net", ACTION=="add", DRIVERS=="?*", ATTR{address}=="00:1c:c0:21:d0:e4", ATTR{type}=="1", KERNEL=="eth*", NAME="eth0"[/B][/COLOR]

```


When you reboot, that file will get written to again, but this time with the NEW network card's MAC address and it will link up to eth0.

If you want to use TWO network cards, do NOT perform the edit above, but INSTEAD edit the "/etc/network/interfaces" file by ADDING the text highlighted in red:

```

# This file describes the network interfaces available on your system
# and how to activate them. For more information, see interfaces(5).

# The loopback network interface
auto lo
iface lo inet loopback

# The primary network interface
auto eth0
iface eth0 inet dhcp

[COLOR="Red"][B]# A secondary network interface
auto eth1
iface eth1 inet dhcp[/B][/COLOR]

```


Good luck!

-- Roger

### Post by dwhitney67 on 2008-10-02
I think that 3Com cards/chip-sets are Linux compatible.  I have not used a desktop in years (been relying on notebook PCs).

Just ask your dealer; the question to ask is whether it is a "smart" NIC, or a "dumb" one that requires a Windows driver to negotiate the network setup.  In the past I have been able to pick up a Linux-compatible card for about $10 (about 7-8 Euros).

The other option is to plow through the Linux kernel configuration file to perhaps see what drivers (modules) are available for a particular chip-set.  A couple of RTL chip-sets are supported; just not the 8139!

Kernel config is located in the following file:
```
/boot/config-`uname -r`
```
