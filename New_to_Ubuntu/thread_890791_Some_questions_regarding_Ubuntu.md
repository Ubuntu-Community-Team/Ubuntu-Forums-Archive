### Post by MacDuff on 2008-08-15
> Also, are there any good photoshop alternatives? 

I was a devoted user of Photoshop for many years and, in fact photoshop or a very similar alternative was one of the "must haves" for me before I made the jump to Ubuntu.  However as I explored Ubuntu more and discovered its many strengths, I decided to install VirtualBox (or VMWare Server) and install my Photoshop in a virtual Windows machine. 

Since then I have searched for every photo editing/management application that I could find for Linux and have tried all of them.  I won't bore you with all the results but now I rarely ever use Photoshop.

Gimp is excellent (be sure to download and read "Groking the Gimp") and there are some promising commercial photo editing packages but the application that I find myself using most of the time is "digikam". Many of the Kipi plugins are valuable so install the entire package.  It has a Light Table that is great to compare similar shots and can do batch processing of large shoots.  There are many camera profiles available for it.

digiKam is not yet perfect and is currently being re-written.  The re-write will hopefully be completed this year but the current version is very good once you get your head around the difference in presentation.  If you decide to look at it, be sure to explore it all.  There are many, many features and ways in which it can be configured.  What digiKam cannot do, can easily be done using Gimp and it is simple to open an image in Gimp from within digiKam.

If you decide to keep digikam and, like me, you keep your photos on a networked storage device, you should be aware that digikam does not, natively, handle networked storage yet, (the new version will) but by using symlinks, it handles networked storage very well.

If you want help with digikam you can send me a message and I will tell you all I know.

Good Luck and welcome to the delightful world of discovery you will encounter in Ubuntu.

Argh.

This question has been answered on the forums many many times for all of those questions, but there are updates to what can run in WINE that I'll list for you if you haven't found them already:

With some work, I've read that PhotoshopCS3 runs in WINE, although I run CS2 with no issues at all, and no extra work. Gimp works great though, and again everyone's right when they say that Gimp will do just about everything that PS will. I believe it will even use PS plugins.

Everyone's right when they say that the Zune software will NOT run without either using a normal install of Windows, or using a virtualized install of Windows (I run a VMware Windows install and the Zune software works flawlessly.) This is because of firmware encryption that MS has added to the Zune player. iPods on the other hand work with anything that has support for it since it's basically just a generic usb storage device.

Blender is definitely the best 3D program for linux, and it's native (made for linux). And free!

Addendum: For better game support in WINE, you have to patch and compile your WINE installation with the 3dmark patch, and get directx installed. I don't have the link for directx (That's my next task, getting it installed), but the link for patching and compiling WINE can be found here: [http://wiki.winehq.org/WineOn64bit#head-c47d9e53f952c5b6260467e0dc158321229216de]("http://wiki.winehq.org/WineOn64bit#head-c47d9e53f952c5b6260467e0dc158321229216de")
