### Post by FormatSeize on 2011-07-05
> **climhazzard36 said:**
> I am dual-booting Ubuntu Natty and Windows 7.
However, when GRUB loads it also shows a Windows XP partition.
The HDD is New and has only ever had Windows 7 previously.
 
Any thoughts please?
Perhaps the developers have not gotten around to changing how grub identifies an NTFS partition. Or, perhaps they don't want to assume that every NTFS partition is Windows 7 being that not everyone has a new computer, and that not everyone dumped XP immediately upon the release of Windows 7. Identifying an NTFS as XP works, as the user knows what is actually there. It's a non-issue, really. I doubt you are going to boot into Windows one day and find that it somehow downgraded to XP (though that would excite the heck out of me if it were to happen to me).

> **climhazzard36 said:**
> Do you think the mbr of the storage partition is the cause?

No.
The **M**aster **B**oot **R**ecord resides on the first few sectors of the drive, and have boot code and partition layout information.

partitions don't have an MBR.

> Do you think the storage partition portion of the mbr is the cause?
