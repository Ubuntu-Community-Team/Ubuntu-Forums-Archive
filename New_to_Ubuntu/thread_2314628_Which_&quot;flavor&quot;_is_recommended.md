### Post by grahammechanical on 2016-02-23
Someone needs to understand the difference between Free & Open Source Software (FOSS) and proprietary software.

With proprietary software we get the OS that the corporation releases with no access to the programme code of the OS. So, the OS cannot be modified by anyone who is not an employee of the corporation. It also means that with every new release of the OS we may have to buy a new computer to run it on. Or continue to use a security unsupported version of the OS.

With FOSS the source code is available and the license allows reuse and modification of the code within the limits of the license. To put it simply, this means that code from Gnome.org can be taken & added to code from the Debian distribution & many other FOSS projects to give us Ubuntu. In turn the code of Ubuntu can be taken & used as the foundation for distributions that have a different desktop environment (DE) and user interface (UI).

What seems to be a bewildering collection of distributions is actually freedom of choice. Programmers can develop the Linux distribution of their choice. Users can select a UI according to their preferences. Furthermore, hardware that some big corporation wanted junked years ago can still be used because a group of developers had the freedom to produce a Linux distribution that required a minimal hardware specification. And the OS will be up to date with security fixes.

Don't knock it.

Regards.
