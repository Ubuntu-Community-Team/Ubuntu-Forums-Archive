### Post by 3xp10r3r|X13 on 2011-06-11
As you've probably noticed, just having very basic knowledge and starting hacking straight away doesn't really work. 

Ubuntu is a desktop distribution, which makes it as easy as possible for the user to run the whole system, without even touching the terminal. Surely the commands are the same as in every linux distribution (apart from apt, zypper etc.), but as you are not really forced to use them you mostly don't and therefore it makes the whole thing more difficult for you to understand the actual system, which lies underneath whatever DE you are currently using. 

To really learn how Linux works I would suggest to try distributions such as Slackware or Debian. (I promise you, by using slackware, you are forced to learn Linux "from the bottom up")


 If you're looking for a real hacking distribution (referring to aircrack-ng etc.) you might be happy to go for backtrack. It is quite unstable and not that easy to handle but it's filled up with damn lots of hacking tools (I guess you're interested in that sort of stuff)

### Post by 3xp10r3r|X13 on 2011-06-11
All right...

An extended partition is a primary partition, containing logical ones. No assuming you've installed Ubuntu on the extended one and windows onto another primary one, just insert the Slackware DVD and wait for it to mount. Then follow the instructions.

Press enter, because you want to use the default kernel, then wait a bit for lots of things to run. Now you should be asked, which keyboard layout you want to use. Just press 1 to choose the wanted layout. Press one again to accept it. As far as I can recall, you need to log in as root next. just type "root", when it asks you to log in as root and hit enter. Next thing to do is the partitioning. type "cfdisk" to view your partition table and do some changes to it.

changes:

Select "free space" and click on create new. Now you are able to select the size of the partition. You've got 70  gb available, so type in the amount you want to use. The type of partition you want to use would be "82" - Linux. Don't worry about it, it will be selected as default. 

Set it to bootable and write the whole thing to your hard drive. 

Quit the partitioning program (selecting "Quit" )

next big thing to do:

Type "setup". You should be transferred to a graphical interface to install Slackware. Just follow the instructions (there shouldn't be any issues - the best ways to go are even pointed out)

Boot loader: 

That's probably the most difficult part...
You either chainload ubuntu with lilo or Slackware with GRUB.

Assuming, you've installed the GRUB loader to your MBR, you need to install LILO to the root directory of your Slackware partition. 

Manually you've got to add some lines to /etc/lilo.config

this is just an example:

other=/dev/hda7
label=Linux_hda7
root (hd0,6) 			 		

you've got to replace those lines with the information according to your system.

All in all, you keep GRUP and just add another OS to it.

I hope this helped :)

If you have any questions, just ask

### Post by 3xp10r3r|X13 on 2011-06-14
Sry about my mistakes referring to the type of the partition. (got mixed up with 83 and 82) You need both types of partitions. One for "/" and one for "swap". (guess, I forgot about swap as well, but you should already have one from your Ubuntu installation). 

You are not creating a separate partition for "/boot", because you need to install LILO to your root directory of your Slackware OS to be able to connect it (chainload it) with the GRUB boot loader. 

ok..to sum up:

your decision to create just one single "Linux" partition was the right one, but unfortunately you forgot to install the LILO boot loader to your root directory. This circumstance doesn't really matter, cause you can still do so.

If I'm right, just insert the Slackware dvd and boot it again. After the boot sequence finished, the dvd should tell you, that it already recognized an installed slackware version:

```

In a pinch, you can boot your system from here with a command like:  boot: hugesmp.s root=/dev/hda1 rdinit= ro   In the example above, /dev/hda1 is the / Linux partition.

```

I found this solution in another Linux forum, so I'm not sure if it works. (but it should do :) )

However, you surely replace /dev/hd1 with the partition you've just installed Slackware to.

After you've accessed your system that way, type "liloconfig" to configure the LILO boot loader. Install it to your root directory (still talking about the root directory of slackware) so you will be able to insert it into the "GRUB menu" later on.
