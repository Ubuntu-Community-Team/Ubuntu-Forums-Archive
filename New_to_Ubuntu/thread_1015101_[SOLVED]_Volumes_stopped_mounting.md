### Post by drs305 on 2008-12-18
```
# /etc/fstab: static file system information.
#
#  -- This file has been automaticly generated by ntfs-config -- 
#
# <file system> <mount point>   <type>  <options>       <dump>  <pass>

proc /proc proc defaults 0 0
# Entry for /dev/sda3 :
UUID=cb709bf4-8b85-4127-9453-9f9d39da644f / ext3 relatime,errors=remount-ro 0 1
# Entry for /dev/sda4 :
UUID=51bfc8bd-1c06-49e1-887c-6dfa7f441f5e none swap sw 0 0
/dev/scd0 /media/cdrom0 auto ro,noauto,user,exec 0 0
[COLOR="Red"]/dev/sda1 /media ntfs-3g defaults 0 0[/COLOR]

```

The windows fstab line needs to be replaced. Here is one option, using the mountpoint **/media/windows**  If you have been mounting it elsewhere, change this to an existing mountpoint. Replace the line in red with this line. 

I don't see an entry for your second ntfs drive. You have 2 NTFS partitions. I assume sda1 is your windows partition. I've added an extra line to mount one more ntfs partition (either sda2 or sdb1). Duplicate the line with another mountpoint as necessary, using the ones you already were using.

Save the fstab file after editing.

```

/dev/sda1 /media/**windows** ntfs-3g auto,users,uid=1000,umask=027,utf8 0 0
/dev/sd[COLOR="Red"]XX[/COLOR] /media/**another.mountpoint** ntfs-3g auto,users,uid=1000,umask=027,utf8 0 0

```

To mount:
```

sudo mount -a

```
If successful, you will not get any messages.

Edit: Added mountpoint for second ntfs drive.

### Post by taurus on 2008-12-18
> 
/dev/sda1 /media ntfs-3g defaults 0 0

That last line in your /etc/fstab is not good.  I would use a difference mount point for it.

```
/dev/sda1 /media/sda1 ntfs-3g defaults 0 0
```
And of course, you need to create that new mount point.

```
sudo mkdir /media/sda1
```
And if you want to mount those ntfs partitions by hand, try something like

```
sudo mkdir /media/sda1 /media/sdb1
sudo mount -t ntfs-3g /dev/sda1 /media/sda1
sudo mount -t ntfs-3g /dev/sdb1 /media/sdb1
df -h
```
