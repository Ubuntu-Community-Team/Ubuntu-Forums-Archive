### Post by deadflowr on 2013-07-02
First, to make sure The windows partition is still a windows partition, use the file manager to mount it and look at the files(hopefully it still ahs files)
It should have the windows folder and whatever other folders windows has, and it should have a couple files, in particular boot.ini and ntdrl files.
If it has those then it should still exist(the windows partition that is).
Open up a terminal and enter
```
sudo update-grub
```
Then look to see if windows gets listed.

### Post by kaloasd on 2013-07-02
Thank you for your help.

For anyone stuck in this dilema:

I had all the other necessary files except the Boot.ini.
A simple 
```
sudo update-grub2
```
would have probably worked for me at this point but here is what I did.

After creating it with the text above I folowed the instructins here
 [http://technologytales.com/2010/11/21/manually-adding-an-entry-for-windows-7-to-an-ubuntu-grub2-menu/](http://technologytales.com/2010/11/21/manually-adding-an-entry-for-windows-7-to-an-ubuntu-grub2-menu/)

Basicly add 
```

menuentry 'WindowsXP' {
set root=’(hd0,msdos2)’
chainloader +1
}

```

to the end of 
```
/etc/grub.d/40_custom
```

and run the grub update.
