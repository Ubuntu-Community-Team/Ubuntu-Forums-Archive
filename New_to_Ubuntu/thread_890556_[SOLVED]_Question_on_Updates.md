### Post by Paqman on 2008-08-15
Not a risk, no. 

Proposed is for cutting-edge packages that could still be a bit buggy, and
[Backports](https://help.ubuntu.com/community/UbuntuBackports) is for inserting updated versions of packages into Ubuntu.

### Post by tahina on 2008-08-15
I think proposed is for testing packages to be (or not to be) included as updates, so updates from there could probably be less stable. 

Backports will get updated versions of apps that would otherwise stay at the same version number throughout the lifespan of the ubuntu release in question. They're probably well tested and all, but I guess the saying "If it ain't broken, don't fix it" would apply. 

I always have everything checked just for fun, but if something messes up my computer I'll hold myself responsible. Haven't had any problems, though, if I recall correctly.

edit: [https://help.ubuntu.com/community/UbuntuBackports](https://help.ubuntu.com/community/UbuntuBackports) has some info on all the different repos mentioned here.
