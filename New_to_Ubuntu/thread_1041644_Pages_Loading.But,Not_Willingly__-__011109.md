### Post by ByteJuggler on 2009-01-16
> **faron45 said:**
> Thanks...everybody.Byte...as far as how to disable/remove my browsers profile...how would I go about doing something like that ?

OK, background information:  Most Mozilla applications store their user data in a root parent folder called ".mozilla", so the full path would be /home/user/.mozilla (where "user" is replaced by your useraname), or just ~/.mozilla (since ~ is shorthand for your home folder.)  Now, Firefox (your browser), being a Mozilla application, then predictably stores everything related to it in a sub-folder inside the .mozilla folder, e.g. /home/user/.mozilla/firefox.  

So, in general, you can therefore disable or remove offending stuff related to any Mozilla application by removing or renaming the contents of the .mozilla folder.  If you for example remove or rename the .mozilla root folder, then it follows that no Mozilla application (including Firefox) will find their normal configuration data including plugins or whatnot on startup, and so consequently and malicious scripts would've been disabled/removed at a stroke.

You can browse into the firefox folder to e.g. back-up your bookmarks, and/or to remove/disable individual parts of your firefox browser's data files (if you know what you're doing.)  The bookmarks for example is stored in a plain html web page, entitled "bookmarks.html".  

To browse the files in your Mozilla folder you can simply open a filesystem explorer window on the folder, e.g. press alt-f2 and enter:

```
nautilus ~/.mozilla
```

This will open a folder explorer in the parent mozilla folder, from which you can drill into the firefox folder etc.  You can remove/rename stuff using this filesystem explorer window.  (The filesystem explorer application on Ubuntu is called "nautilus", hence the command.)
