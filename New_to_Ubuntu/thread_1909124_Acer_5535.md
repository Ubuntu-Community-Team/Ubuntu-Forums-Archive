No need to apoligise for the tone lol, I totally understand your frustration! Lol. 

As for using cpufrequtils, I didn't *technically* lose cycles, but rather made sure it was running as low as it could. The issue I had was that my CPU was running at maximum frequency all of the time, even when it was idling. I used cpufrequtils to make sure it used an "on demand" governor and had a fiddle with the config to play with the various frequency steps. As I said, this was in Arch Linux however.

As for cpufrequtils now, I believe users are having issues with the 3.x kernel, and there is currently a bug report for it.. but I'm not sure of current status.

### Post by Fernhill Linux Project on 2012-01-16
For the 5315 there had been 24 bios updates since I purchased it, from 1.19 to 1.43. I read all of the changes and found nothing to suggest it would work, and there was much debate in the community about flashing the bios and why it should not work, and how it was probably kernal issue. 
But it did work, not saying it will for you, but after a while of using this machine for 45 mins to 1hr before it cut out, I was also ready to send it flying, so my advice is if nothing else works - flash the latest bios and hope for the best.
