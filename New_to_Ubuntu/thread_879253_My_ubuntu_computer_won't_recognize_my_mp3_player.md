### Post by danellisuk on 2008-08-10
Yesterday, I had the same issue with an Acoustic Solutions MP3 player (from Argos).  It looks like your MP3 player has the same internals because I see from the lsusb output shows "066f:8000 SigmaTel, Inc. SigmaTel MSCN USB Device [MP3 Player]".

The MP3 player failed to mount automatically because of invalid signatures on the partition (It seems these devices ship this way from the factory).  I fixed the device by plugging the device into a Windows XP machine, and formatting the partition:

[INDENT]
 (The following is from memory so may not be exact)
 Right click My Computer,
 Select Computer Management,
 Select Removable devices,
 Locate the partition, right click, select Format,
 Choose FAT32 as the file system type.
[/INDENT]

There is a hidden Settings.dat file on the device, which I backed up first, but it turned out I didn't need to as the file was re-created by the device.

When I plugged it back in to my Ubuntu machine, it found it straight away.  Perfect!

I am sure there is a way to format the device from Ubuntu and we wouldn't need Windows to fix the device, but I have not tried that yet. (It is my mums MP3 player, and they have a Dell Ubuntu PC!)

Reading another forum you would possibly use mkdosfs:-

[http://www.linuxforums.org/forum/redhat-fedora-linux-help/69900-how-do-you-format-partition-fat32-vfat.html](http://www.linuxforums.org/forum/redhat-fedora-linux-help/69900-how-do-you-format-partition-fat32-vfat.html)
