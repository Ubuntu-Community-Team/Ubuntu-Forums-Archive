### Post by ronparent on 2011-05-14
Welcome to the forum. It appears you are using a dmraid mother board chip based fakeraid. 10.04, 10.10 and 11.04 each are distributed with dmraid on the cd/dvd. The first step is to see what comes up running the live cd of any of those distributions. If the live cd session loads run a terminal seesion to have a look at what is in /dev/mapper. Print the output of **ls /dev/mapper** here and I can instruct you on a quick grub re-install that will probably fix your problem. Identify which raid partition you are installed on and which drive is set a boot in bios.

If you can't boot to a live cd we will probably have to work on what boot parameter might be needed to boot the live cd.

If you are uncertain as to what your setup is go to **[http://sourceforge.net/project/platformdownload.php?group_id=250055](http://sourceforge.net/project/platformdownload.php?group_id=250055)**, run the script and post the result.

Sorry I was unable to get back to you yesterday.

What 'Boot Info Script' says:
> => Grub 2 is installed in the MBR of /dev/mapper/pdc_becfecbab and looks for

What I would expect to see is:
> => Grub 2 is installed in the MBR of /dev/mapper/pdc_becfecbab and looks for 
    pdc_becfecbab1:

'b2d' may be a new convention that crep in while I was't looking? :o

Let's try a quick grub2 reinstall. In a terminal window copy:
```
sudo mount /dev/mapper/pdc_becfecbab1 /mnt

sudo grub-install --root-directory=/mnt/ /dev/mapper/pdc_becfecbab
```

Then we will know for sure grub is installed properly. You should now be able to boot to 11.04 in pdc_becfecbab1. Let us know how this works out.

### Post by ronparent on 2011-05-17
> when i try and mount /dev/sda1 as /mnt then it says that special device /dev/sda1 does not exist
That statement is correct. You are installed to a raid array and the file systems are accessed trough the symbolic links set up in /dev/mapper/. Therefore your install is not on /dev/sda1 but on /dev/mapper/pdc_becfecbab1. Similarly the boot loader is not on /dev/sda but on /dev/mapper/pdc_becfecbab. If those designations are used in Method 3 grub 2 will be installed to the right location and the boot loader placed on the array where your bios looks for it. I am leaving it to you to fill in the blanks. Post if you still have difficulties.
