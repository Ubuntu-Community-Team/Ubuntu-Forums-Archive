### Post by MarkC502 on 2010-01-18
Chris check and make sure you have the correct DNS server address


IF YOU JUST WANT TO SEE WHATS IN IT
cat /etc/resolv.conf

IF YOU NEED TO EDIT IT
sudo gedit /etc/resolv.conf

it should contain a nameserver and IP address. If you use a router it could be the first three IP numbers then .1 or it could be some IP that is a DNS server for your ISP. If it is wrong or the DNS server it points to is having problems then you can have terrible internet browsing.

If you need you can use either of the following as they are OpenDNS.org's DNS servers and are free for anyone to use and are super reliable.
[B] 				208.67.222.222
				208.67.220.220 			[/B]

 		 		 		 			If it turns out it isn't DNS then maybe research IPv6 problems, easy to try the fix as there is a simple tweak for Firefox to turn off using IPv6 and revert to IPv4 only. Some people have reported that this solves browser page load time problems.

Hope you figure it out.

### Post by MarkC502 on 2010-01-18
If your asking if that is correct , then I will show you what that means

'sudo gedit /etc/resolv.conf'  
opened a new window which said,
'# Generated by NetworkManager  <-- this means that network-manager added this entry for you
Nameserver 192.168.1.1' <--that is the DNS server your system is using, which is 100% fine if you have a router or modem with 192.168.1.1 as its IP address. That just tells me that you do and that your router is responsible for handling DNS queries from PC's on your network.


Anyway if the IPv6 trick worked and you can open web pages then your DNS settings must be OK or else you couldn't open a webpage.
