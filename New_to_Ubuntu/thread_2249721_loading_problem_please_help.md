@rahul25, the first thing you need to do is to discard that ISO file. There is something very suspicious going on. The link you posted in post #19 leads to here:

[http://www.ubuntu.com/download/desktop/thank-you?country=IN&version=14.10&architecture=i386](http://www.ubuntu.com/download/desktop/thank-you?country=IN&version=14.10&architecture=i386)

Which opens a download window for the ubuntu-14.10-desktop-i386.iso but which downloads from randomly selected mirrors, so we don't know which mirror you actually downloaded from. I downloaded from your link – although I am in the UK and there is country=IN in the URL – and got the  ubuntu-14.10-desktop-i386.iso file from a mirror I didn't recognise. But the md5sum checked out OK (more of which in a moment) so I know my download is OK.

I then prepared a bootable USB from it and this is the menu I get:

[img]http://ubuntuforums.org/attachment.php?attachmentid=257470&d=1414179708[/img]

Compare that with yours from your video:

[img]http://ubuntuforums.org/attachment.php?attachmentid=257469&d=1414179708[/img]

There is something very fishy about that. I really have no explanation. A rogue mirror offering a modified ISO could be one explanation but it seems a bit far-fetched.

Whatever the explanation, I suggest you do discard that ISO. An official ISO simply does not produce a retro-menu all by itself, and we don't know what else may be wrong with it.

Try downloading the ubuntu-14.10-desktop-i386.iso file again from here:

[http://releases.ubuntu.com/utopic/](http://releases.ubuntu.com/utopic/)

And then view or download the MD5SUMS text on that page and do a md5sum check on the downloaded ISO as described here:

[https://help.ubuntu.com/community/HowToMD5SUM](https://help.ubuntu.com/community/HowToMD5SUM)

There is a method for doing md5sum checking in Windows if you need that – scroll down the page.

Once you have a verified ISO and you have prepared a bootable installation medium, let us know how you get on. If you still see the same problems, then at least we can investigate them knowing the ISO you used is good.
