### Post by bville09 on 2010-04-25
Dreamweaver is slow through Wine anyway, and looks about as good as it does on Windows 98.

Anyway, there are a bunch of different native programs you can use, but they don't have the same number of features as DW, most people don't use half of them, so it doesn't matter.

My favourite to use is KompoZer, as it has basically everything I need, except flash (afaik), including both code and linear views. I also use bluefish, but it's code only. Kompozer doesn't have as many bugs as everyone says, or at least not ones that are noticeable. Every now and then it will slow down (especially with a lot of graphics on the page), but for the most part it's usable.

If all you want to do is make basic webpages, with graphics, nice layouts, etc. but can live without flash or other fancy effects, KompoZer does what it's supposed to, otherwise Dreamweaver and wine may be your best bets.

### Post by mistypotato on 2010-04-30
**[COLOR="Red"]Do not use Kompozer on Coldfusion (CFM,CFML) pages.[/COLOR]**

My (unfortunate) experience is that it confuses the cfml tags and during save, re-writes the code, replacing what it interprets as html tags with html code, destroying your coldfusion template.

This is a MAJOR problem for me as I use CF extensively.

If anyone uses Kompozer with CFML successfully please advise how.

Also, if anyone uses Railo or Coldfusion and has found a good CFM Linux editor, please also advise.
The best CFML edittor I have found is ECLIPSE with the Coldfusion plugin.
