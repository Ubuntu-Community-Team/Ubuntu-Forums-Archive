### Post by QuentinK on 2010-12-11
This is probably a common question/problem. Sorry if it is.

I downloaded a livecd (err live flash drive rather) and used it to install Ubuntu side-by-side on my Windows XP computer. During the installation of Ubuntu, I was absolutely sure to install it to a partition because there are files I need on the XP side of the HDD. So I installed it to a partition, let it do it's thing and then went happily on my merry way in Ubuntu for a while, setting up all the applications I needed and such, but I realized I was missing my music files (which are on my XP). So I tried to find out if I could get them while running Ubuntu but I figured it'd be easier to just boot Windows. So I restarted my computer, but the computer defaulted to Ubuntu. Odd. So I decided to restart and press F12 (which sends me to the boot sequence) I click internal HDD, and alas, it boots up Ubuntu. So now I'm worried that it might've got the wrong impression and went ahead and installed Ubuntu over Windows and removed all my files from there completely.

I'm running a Dell D630 Latitude (Originally with Windows XP Pro)
80g HDD
Little to no experience with Ubuntu.

Any help in any way would be much obliged.

### Post by coffeecat on 2010-12-11
Boot up Ubuntu and go to this webpage:

[http://bootinfoscript.sourceforge.net/](http://bootinfoscript.sourceforge.net/)

Download and run the bootinfo script there and post the contents of the RESULTS.txt file, not forgetting to put it in code tags as described on that page. This will give us your partition layout and details of the bootloader configuration/menu, which will show what is going on.

I'm logging off now but that will get the ball rolling and someone will be able to help.
