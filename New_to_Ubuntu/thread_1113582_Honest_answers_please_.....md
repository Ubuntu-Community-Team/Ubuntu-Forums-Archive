### Post by Agent ME on 2009-04-01
*(Seems like a few people beat me more or less to this but here it is anyway)*

A big thing to note, as a difference between Windows and a package-based Linux system, is that when you look for programs to install, you usually want to install programs through your system's package manager instead of downloading them off of a website.

The easiest way to install software is to click "Applications -> Add/Remove...", type in the name of a program in the searchbox, check the box near the program, and click "Apply Changes".

### Post by tracerbullet on 2009-04-02
> **Loisa V said:**
> I don't know what all these file types are, I never typed into a shell (is that right?) in my life.

The shell is actually not some   "geeky Linux thing", but rather a device that you can use to talk (more) directly to the computer. Some people find that it can often be faster to use this instead of pointing and clicking, and you should not be afraid of trying it out eventually, especially if you follow some of the tutorials on these forums, they are normally very carefully explained. :)

As an example of what I mean by "faster":

Imagine you wanted to do a task that consisted of copying one file from one location to another and renaming that file.

With a mouse, you would need to right click the file, select copy, right click the destination, select paste, right click the new file, select rename, then type in the name and click again before you were done.

In a terminal, you could do this command in three simple words.

Let me first show you this example with "pseudo-code" which shows you what is going on, and then type it in real code to give you an idea of what this would look like:

```
[color="#990000"]<copy command>[/COLOR] [color="#999900"]<file name of the file you wish to copy>[/COLOR] [color="#009900"]<new file name>[/COLOR]
```

And a working example of that would look like (And note the colors so you can see what is what):

```
[color="#990000"]cp[/COLOR] [color="#999900"]file.txt[/COLOR] [color="#009900"]newfile.txt[/COLOR]
```

That wasn't so scary was it? ;)

This is just a different way of talking to your computer, through typing instead of clicking. Many users find that when there's a whole bunch of commands they need to execute, it is often more practical to execute them this way, but it doesn't mean you HAVE to ALWAYS type in a terminal just because you are using Linux. The beauty of Linux is that there are lots of alternatives, and you can find a "clicky" interface for almost everything.

Hope this gives you a little bit of insight, have a nice day!

### Post by FM1 on 2009-04-02
Dumb*** girl here, too, Loisa. :) You're not dumb anyway, only new, and when you're used to just getting things done in Windows, it's frustrating to know what you want to do, but not *how* to do it, or even what you're supposed to use. You've got some good suggestions thus far, so I'll just add some "potential geek-girl" moral support. It helped me a lot to poke around the menus and Synaptic and see what everything did, or didn't do. Synaptic does require the root password, but you can't do any damage just looking around or doing searches to see what's there; unless you click "Apply", no changes are applied and nothing gets installed (or uninstalled), so you won't mess anything up. Doesn't hurt to look, and if you don't have anyone to teach you, it does help give you a kind of mental map.

If you're used to Photoshop, the first time you try GIMP, it'll look strange with so many windows, but it can do nearly everything that Photoshop can do, and far more than most people will ever use. I'd call myself an intermediate with Photoshop, but I'm getting the hang of GIMP and like it just as well. I did change many GIMP keyboard shortcuts to the same as Photoshop because my fingers refused to learn new ones.

The reason you see a lot of command-line stuff here is not because anyone wants to confuse you, it's because command line is often faster and easier, and works the same no matter how your menus are set up. Rather than someone saying, "Click this, select that, click something else", which may or may not apply to your setup (I don't like the Ubuntu default menu myself), you just open a terminal and either type or paste in the instructions that you've been given. Looks confusing at first, but it's really not once you get used to it, and I must admit that it's even a little more fun when your (Windows-only) husband can see your monitor. :D At first, I wanted point and click because that's all I knew, but I'm just as comfortable with command line now, and I bet you'll be the same in a short time. You just have to keep reminding yourself that the Windows way isn't the only way, you weren't born knowing Windows, either, and that Linux is *not* only for geeks; it's for anyone who's willing to try something different. I wouldn't switch back now; Windows feels clunky and too often hampers what I want to do. Gnome is also easier to theme entirely in bright pink so I can send screenshots to a friend of mine who thinks everyone should use a Mac. I like to think it brings a tear to his eye, probably not in a good way. Bwahaha!

Personally I find that the terminal makes some stuff easier, for example, installing things, assuming I know the name and it's in the sources list. as an example, in order to install something like audacity, using the synaptic package manager, I would have to first find it in the administration menu (or you could drag the menu entry to the bar at the top, or bottom, or side or wherever you put it, eventually I'll have a bar just for the little links). then i would have to type in my password, then i would have to search for audacity, then i would have to wait for synaptic to find it, then I would have to click the little box, click apply, then click the apply icon then wait for it to be installed, then click "close" on the little install box (depending on whether or not you checked the "close once finished box) then wait a couple of seconds to click the little x. With the terminal I can type sudo apt-get install audacity wait a couple of seconds, type y, press enter, wait, then type exit.
