### Post by Double.J on 2012-03-02
Hi there nullified_nostalgia,

I'll try any keep this simple, as it can get a bit complicated !

Basically linux is linux - each and every distro take the gnu user land, the linux kernel, then compile the options they want into the kernel and then add software packages on top... okay some just repackage other distros, but this is basically how it works;

linux kernel + gnu userland + package manager + packages = distro.

In real end user terms nowadays the package manager is really the point at which you start choosing; if you know and like the APT package manager of Ubuntu, and the debian style way of handling pakages, you may want to stick to this. Likewise if you're used to fedora or red hat based distros it's yum or yast and on to pacman and portage ... and many more besides.. also did you want rolling or set releases?

Of course you can narrow your distro search by what sort of things you want - ease of use, minimalist, most traditional are how it tends to brake down, for most people. 

These are the two main choices that should be considered when choosing a distro - what the aims of project are, and it's package management system; everything else is just packages, and all linux distros can run the software in those packages (even if they don't have a binary package, the source will be out there and can be compiled and run)

That's the basics of it.

To me it sounds as though you're looking for ease of use, and you have got to know the debian/ubuntu way of handling packages and repositories. Therefore I'd recommend staying within this approach unless you are looking to try something new - again, there's nothing wrong with others, it's just personal choice!

It would seem that the desktop environment is what's most important to you. The most common environments using the "start menu" and  "task bar" approace are KDE, LXDE and XFCE ... Cinnamon is looking a bit similar... but not there yet. KDE has the most features and runs the most heavy... XFCE and LXDE run pretty similar, with LXDE possibly fitting the windows requirement a bit better.

The good new is that you can add these environments to almost any disto... even ubuntu - you can add KDE and LXDE to your ubuntu from the software centre and choose between them at the login screen (best packages to choose are kubuntu-desktop and lubuntu-desktop) just remember if you were going to try this it's best to back up first, or make a test install to try out on first as they can be quite difficult to remove all the "effects" that create by adding lots of packages.

My recommended approach would be to burn a kubuntu and lubuntu live CD and try them out - this will give you an idea of the desktop environment that you want to use, then look at the distro you want to use it on. Obviously I don't think there's anything wrong with Ubuntu, but Mint is a popular alternataive I quite often use, and Pinguy OS is another good alternative... for a lightweight approach could also check out bodhi?

But as I say probably best to work out what desktop environment you want - run a KDE environment for a bit and see how you like it compared to lxde or xfce... finding something that's right for you is just trial and error I'm afraid

Hope it helps!
