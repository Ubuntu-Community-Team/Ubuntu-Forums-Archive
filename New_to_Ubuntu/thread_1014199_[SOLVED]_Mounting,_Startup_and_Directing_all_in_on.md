### Post by Michael.Godawski on 2008-12-19
You think it is a ntfs partition?

Then try these codes:
Create a mountpoint
```
sudo mkdir /media/datapartition
```Mount the partition:
```

sudo mount /dev/hda1 /media/datapartition -t ntfs -o umask=0002,nls=utf8
```change the /dev/hda1 path according to your missing partitions

 via /etc/fstab:```

 /dev/hda1 /media/datapartition ntfs ro,umask=0002,nls=utf8
```**Ownership and permissios of vfat / ntfs are set at the time of mounting**. This is often a source of confusion.

You will have to use ntfs-3g for write access:

fstab entry.
```
# NTFS ~ Use ntfs-3g for write access (rw) 
# /dev/hda1
  /media/datapartition  ntfs-3g  auto,users,uid=1000,gid=100,utf8,dmask=027,fmask=1  37   0  0
```**EDIT**: saw too late that you posted. Which partition do you want to add to your fstab? Which one is missing ?
