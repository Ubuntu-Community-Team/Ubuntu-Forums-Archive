### Post by mcduck on 2011-11-08
Do you mean the /dev/sda7 partition? It's mounted in /data, while only drives mounted inside /media appear on the devices panel.

Either use Nautilus to browse to /data, symlink (or bind) the location to your home directory for easy access, or create a new mount point for the drive inside /media and edit the fstab file accordingly.

### Post by mcduck on 2011-11-08
I can't much about /dev/sda2 for sure without knowing if it's configured in your fsstab (and how) or f it's automounted.

...but, assuming it's not in your fsatb, then yes, the automount process mounts drives inside /media. This applies to all removable drives and optical media, plus all internal drives that aren't configured in fstab.

I don't think *searching* for /data in Nautilus would do you much good (especially since it will, by default, only search from your home directory, not from the whole filesystem hierarchy). 
Either click the File System icon from the left panel and browse to /data, or hit Ctrl-L to show the location bar and type the /data there.

What comes to creating a symbolic link, you can either drag&drop the /data directory to your home using middle mouse button and then select "Link Here" from the popup menu, or open a terminal and run the following command:
```
ln -s /Data ~/Data
```

### Post by mcduck on 2011-11-08
No need for any partition editor, as you aren't actually making any changes to your partitions. :)

All you need to do is to create a new directory for the partition inside /media, and then change the fstab line to use that location instead of the current /Data.

```
sudo mkdir /media/Data
```

...and then change the partition's fstab entry to this:
```
# /data was on /dev/sda7 during installation
UUID=AED6-5964 /media/Data vfat utf8,umask=007,gid=46 0 1
```

(you'll of course also want to remove the symlink you just created from your home directory. You can just delete it like any file.)
