That's your culprit, right there. You're missing a colon at line 53.

Assuming you're running Ubuntu, please run the following at a terminal window```
gksu gedit /etc/apt/sources.list
```This will open your sorces list file in read/write mode. Navigate to that line and add the missing colon so it reads```
deb http://archive.canonical.com/ xenial partner
```Save and close the file and run```
sudo apt-get update && sudo apt-get upgrade
```Tell us how it went.

Let me just inform you that the line '**N: Ignoring file '20auto-upgrades.ucf-dist' in directory '/etc/apt/apt.conf.d/' as it has an invalid filename extension**' apt spitted out is just a notice. This is not a severe problem, it simply says that the file in **/etc/apt/apt.conf.d/20auto-upgrades.ucf-dist** is ignored because of an invalid file extension. Remove that extension running in a terminal window the following command```
sudo mv /etc/apt/apt.conf.d/20auto-upgrades.ucf-dist /etc/apt/apt.conf.d/20auto-upgrades
```
