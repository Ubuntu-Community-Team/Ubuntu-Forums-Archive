### Post by jrothwell97 on 2009-08-20
Welcome to the Ubuntu forums, Debra!

Ubuntu has a lot of differences to Windows, but providing you're willing to take the time to learn, you shouldn't have too much trouble adapting. I'll list the main differences below.

[LIST]
[*]Ubuntu is free (obviously :))
[*]Generally, you don't need to run anti-virus software on Ubuntu, because (a) there are less viruses about, and (b) as long as you exercise caution, you shouldn't pick up any nasty malware.
[*]WINE runs some Windows apps, but it's still very hit-and-miss.
[*]Generally, you don't need to download files manually to install programs on Ubuntu. This is because Ubuntu has a *package manager*, a piece of software that will fetch programs automatically from servers (repositories) and keep them up to date automatically. You can use Add/Remove Programs, Synaptic Package Manager or a command at the terminal to do this (I'll come to the terminal in a minute.)
[*]To get DVD, Flash (YouTube and the like) and MP3 playback working, you need to install some software which isn't installed by default. It's reasonably simple, and you can find out how to do it by searching Google.
[*]Occasionally, to speed commands up, you might need to use the *terminal*. Put simply, this is where instructions to the computer are given in text commands rather than by clicking on buttons, icons, etc. For example, *ls* lists the files in the folder you're currently in, and *cd ../steve* would change up one folder and then into the folder "steve". If you're uncomfortable with it, you can usually make do without it, but it's useful to know your way around it.
[*]In Ubuntu, there are no 'administrator' users per se. 'Administrator' users can't modify important parts of the system, but they can 'become' the administrator user, called *root* or the super-user, for a brief time to complete tasks (such as updating the system software). Most of the time, this is transparent (you'll just be asked for your password to perform these functions) but you might need to do this manually at the terminal, in which case you simply type *sudo* before the command. For example, *sudo aptitude* will launch the program "Aptitude" as the super-user. It is possible to log in directly as the super-user, but it's very dangerous, so the root account is locked by default.
[*]A more secure OS is no substitute for exercising common sense. Use a strong password, read the manuals, and do regular backups just like you would on Windows.
[/LIST]

That's most of the major differences covered: if you need some more help, don't hesitate to post here. There's always someone happy to help. :)

Good luck and enjoy Ubuntu!

### Post by niteshifter on 2009-08-21
Hi,

Welcome to Ubuntu! I bought a Dell Inspiron (a 1420) with Ubuntu pre-installed back in January '08. Other posters have covered a fair amount of ground, but here's some things I have encountered.

The Dell installation of Ubuntu (I'm assuming 8.04.1 or 8.04.2) works great. Mine came with 7.10 - I updated to 8.04.2 just fine but for one known problem (sound and specific to the 1420 - but the fix was on Dell's wiki). That wiki ([link]("http://linux.dell.com/wiki/index.php/Wiki_Main_Page")) can be useful at times. 

Also there's a specific area of ubuntuforums for Dells. Not that you're restricted to that, ask your questions here or there. Don't hesitate to ask, either.

They don't come with the "restricted extras" (MP3 support and such) so you'll want to install that package shortly after you get your machine. Search for "ubuntu-restricted-extras" in Synaptic Package Manager, that should take you right to it.

They do come with DVD playback already enabled. I'm not sure what Dell ships with them now, in my case it was LinDVD. It's okay, but a bit clunky. VLC is a better product (Synaptic search "vlc").

Dells with Ubuntu pre-installed ship with a recovery partition. You should consider doing a few things with it:

1) Burn it to DVD, ASAP. This "disk" is not the same as the Ubuntu disk that comes with the machine - that's a generic Ubuntu CD. It does not have the DVD playback software for example, nor will it have the driver sets already configured and installable for your machine.

2) That recovery partition is useful - for a while. If, in the early messing about should you tank the machine, selecting Recovery from the boot up Grub menu will put the machine back to the exact same state it was when you first powered it up. Which brings me to the next item ...

3) After you are comfortable with using Ubuntu - and you've installed some packages, stored some photos / music / video and saved some documents that recovery partition becomes less of an asset and more of a liability. Consider editing the file /boot/grub/menu.lst and enable a password for booting (or comment that section out), for if the machine should ever get to the wrong hands that recovery mechanism could be used and wipe out all you've done. 

Getting "backup religion" isn't a bad idea, regardless of OS used :)
