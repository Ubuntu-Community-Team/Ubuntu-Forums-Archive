### Post by starfyredragon on 2010-06-28
As a general rule, any version of ubuntu will work, it's just a matter of how fast the computer will run it (which one is 'optimum'). So the answer is "all of them are compatible". And all the pretty stuff that slows down a computer can be turned off.

As a note, your computer being Windows ME doesn't matter. If you're installing Ubuntu (and not just running from the live CD), you actually replace Windows ME with ubuntu. (either having a computer that gives you a choice between windows ME or ubuntu whenever you start, or just getting rid of windows me altogether... your choice.)

And although Ubuntu is available for free from the website, kudos on the smart idea to actually buy (unless you're a computer expert) because the bought versions of Ubuntu usually come with tech support, which can help if you're getting started.

### Post by Kixtosh on 2010-06-28
I have a similarly old machine (two of them, in fact), and although they worked well under W2K:

1) Start up times were ridiculous: about 2 minutes after a clean install; about 5 minutes on an older installation!
2) Security could be an issue, since W2K is no longer supported.

I have tried several versions of light Linux/Ubuntu to remedy this, and while none are perfect for such old machines, here are my suggestions:

A) Puppy Linux is a simple solution that works. The lightweight browser and applications included work better than most. You don't have to wipe your hard drive, since it works very well from just the CD. Boot times on an older machine (like mine) will not be stellar, but applications all started for me within a few seconds. [http://puppylinux.org/wikka/MinReq](http://puppylinux.org/wikka/MinReq)

B) Regular Ubuntu is far too "heavy" for even 256MB of RAM. It's quite fast to start up, but everything else is slow, and can cause system crashes in my experience.

C) Xubuntu is better than regular Ubuntu. Boot times are faster, but operation is still a bit clunky.

D) SliTaz would not work for me. It looked cool from the LiveCD, but some hardware would not work at all.  I could not get wireless to work, and unless I removed my PC Card wireless device, and my laptop battery, it would not even show me the panel with the main menus! It seemed too prone to problems for my old systems so I had to give up on it.

E) My best experience has been with Xubuntu, adding the LXDE desktop and the Chromium browser, or Lubuntu, but you cannot install either of these normally with only 128MB of RAM, so you will have to look at alternative installation methods. Both of these were able to reduce the start up time on my old laptops to just under one minute, and the browser will open in less than ten seconds. Both use small amounts of my RAM (only 30% with just the desktop, or between 75MB for Xubuntu with LXDE, or 55MB for Lubuntu).

**My Conclusions**: 

_I would try Puppy first_, since it's so easy to do, if you can, but you must have a CD drive that will boot (this was not all that common on older laptops, ten years ago). To use it, you just pop in the CD to use it, you remove the CD to avoid it - there's no real advantage to a hard drive installation unless you don't have an integrated CD drive and want to avoid carrying one around).

I would _avoid everything else, but Lubuntu_ (or Xubuntu, with LXDE, if you are tempted), but don't expect miracles in terms of speed. It should be reliable, perfectly usable, and secure. If you are not daunted by the alternate installation procedures (I am!), I think you are likely to prefer it to Puppy, but it will not be faster (if anything, it'll be a bit slower, but not ridiculously so).

_Avoid Firefox and OpenOffice_. Use Chromium and the default, lightweight word processor or spreadsheet applications already included with the lighter distros.

If you can, I would _attempt to upgrade the RAM_. Having 256MB (as I have now) will certainly make everything so much easier, and 512 will likely improve it even further. With Puppy, it's probably best if you can have a swap partition somewhere on your hard drive. Lubuntu will install it's own swap partition automatically.

This very message was written on one of those old machines, by the way!

Best of luck!
