### Post by orwellus on 2009-04-25
I've been using jaunty and easytag seems to have a weird glitch. When I had Ubuntu Gutsy, I could right click on a mp3 file and after a few seconds easytag would open and I could edit the mp3 file no problem. 

In Ubuntu 9.04, I right click on a mp3 file, easytag will open, but there is no file. It's like easytag cannot see it. I have tried a couple of other tag editors (audio tag edior and ex falso), and same thing, the tag editor cannot see the mp3 file. The only way I can get it to work in easytag is to put the mp3 file in a folder and then hit refresh. Then easytag can find the mp3 file, and everything works fine after that. 

So can anyone help me with this weird and annoying glitch? I am using the regular easytag NOT easytag-aac. Tag setting are ID3v2.3

---

### Post by kostkon on 2009-04-27
Go to its preferences, select the *Browser* tab and make sure that the option *Load on startup the default dir or the dir passed as argument* and, maybe (I am not sure) the other 2 options below it, are enabled.

---

### Post by orwellus on 2009-04-27
Thank you. That fixed it.
