### Post by drs305 on 2009-01-24
It shouldn't be a problem. Back up important data before any repartitioning.

One thing to check after you do this and before you do any rebooting is to check the contents of /etc/fstab and /boot/grub/menu.lst and make sure the listings are correct. Recent versions of ubuntu have gone to using UUIDs instead of /dev/sdXX to identify parttions, but make sure because an sdXX designation might change. The UUIDs shouldn't but check them anyway.

Check them with the following, which will give both the sdXX and blkid:
```

sudo blkid -c /dev/null

```

Check the contents of the files with:
```

cat /etc/fstab /boot/grub/menu.list

```

### Post by Paqman on 2009-01-24
> **flameproof said:**
> I want to have a 10Gb partition for Win98, just to do online banking.


I would strongly advise against that. Support for Win98 ended in 2006. You won't be receiving any security updates that will patch vulnerabilities.

To answer your question though: yes, you can use Gparted to modify your partitions. You cannot alter any partition while it is mounted though, so to modify your root partition (for example) you would need to be using the LiveCD. Gparted can be found at System > Admin > Partition Editor.
