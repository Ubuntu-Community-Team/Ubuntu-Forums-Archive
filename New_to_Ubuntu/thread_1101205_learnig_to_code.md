### Post by Eisenwinter on 2009-03-20
I recommend _[Perl]("http://www.perl.org")_

Many coders argue that Perl is write-only, but with good coding practices and habits, that "flaw" goes away naturally.

Perl believes in doing things in a straight-to-the-point way, but it's motto is "There's always more than one way to do it in Perl".

Like it's motto suggests, there's always more than one way, therefore, your thinking is expanded tremendously, and it's easy to create your own style of programming.

### Post by lisati on 2009-03-20
On these forums you'll find people with a wide range of experience.

I started programming 30-something years ago with a dialect of Fortran aimed at school students and a dialect of BASIC on a Wang PC (this was before IBM released its x86-based system with MS-DOS). There are versions of BASIC and Fortran that are usable with Ubuntu, but they don't suit everyone.

I've also had exposure to COBOL (wouldn't recommend COBOL unless you're likely to be maintaining legacy software for a business), PL/1 (not sure how widely used it is), C, and assorted flavours of ASM.

I'd suggest C, Java and Python as possibilities to check out because they are supported in several platforms and they seem to list amongst the favourites here on the forums.

Not everyone's a fan of BASIC (too easy to pick up bad habits) but it has the advantage of originally having been designed for beginners.

C++ should rates a mention too, but some of it is over my head!

### Post by lykwydchykyn on 2009-03-20
One of the most fruitful early programming experiences I had was learning TADS3.  It is an interpreted language for writing text adventure games, so it has very limited uses compared to general purpose languages; but the great thing is that it kept me motivated because I could actually produce something more than just worthless demo programs that printed the fibonacci series or something.  Taught me a ton about object-oriented programming.

### Post by Alterax on 2009-03-20
Although they have their limitations, I actually think that Java or C++ are a good place to start.  There are tons of resources available for each, both on the web and in bookstores.  Both can be used for either procedural/linear or for object-oriented programming.

So in my opinion, your best place to start would be with installing a good IDE that will work with either.  I use netbeans myself, but I hear the Eclipse IDE is good too.  Both are available (along with a couple of others like Anjuta) in the standard Ubuntu repos.  If you decide to go with Netbeans and C++, you'll need to install the C++ plugins from within Netbeans--it does do Java by default.

The reason you do want a good IDE is that they can make some of the process easier (such as writing user interfaces), as well as do syntax checking (so you don't have to wait for your code to compile to find out you left out something as trivial as a semicolon or curly-bracket.

In any event, good luck!
