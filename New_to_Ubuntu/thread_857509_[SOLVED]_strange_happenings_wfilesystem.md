### Post by bumanie on 2008-07-12
Firstly backup your present /etc/fstab > sudo cp /etc/fstab /etc/fstab_backup
proc /proc proc defaults 0 0
# Entry for /dev/sda1 :
UUID=9567b837-3d1e-4317-801e-1f969b9e40ed / ext3 relatime,errors=remount-ro 0 1
# Entry for /dev/sda5 :
UUID=465fae8c-e4f6-4bf0-8827-338b7a21dfe2 none swap sw 0 0
/dev/scd0 /media/cdrom1 udf,iso9660 user,noauto,exec,utf8 0 0
/dev/sdb1 /media/sdb1 vfat uid=rick,sync,owner,users,noauto,user 0 0
[COLOR="Red"]/dev/sda1 /media/sda1 ext3 defaults 0 0
/dev/sda5 /media/sda5 swap defaults 0 0[/COLOR]
I think the two lines in red can be safely removed. They look as though they have been manually added and are 'doubling-up' the 'official' entries that have UUID's attached. If ubuntu works OK ie boots, dual entries gone etc and you're happy with everything, you can get rid of the backup /etc/fstab and then backup the edited one, just in case something goes wrong in the future.
