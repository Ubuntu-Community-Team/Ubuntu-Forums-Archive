Hi there

Ubuntu 6.06 was a LTS-Release, where LTS stands for Long Term Suppoert.
Those LTS-Relases (or their components) are supported THREE years on Desktop (very likely that what you are using it for) and five years for server.
That supports includes SECURITY fixes, but not necessarily new versions of software (as in every major distribution without rolling realeases).
Ubuntu 6.06 (as the name says) came out in June 2006 and will be soon out of support.
Canoncical makes such LTS realeases once in a while for people who do not wish to update every half or whole year, because every other version such as 6.10, 7.04, 7.10, 8.10 and soon 9.04 will be supported only a year (or 18 month).

Now, the new LTS-release was 8.04 (which came out in April 2008) and was the sucessor of 6.06 and it will be supported another three years at least.

So it is up to you to use whatever you like.
Either use Ubuntu 8.04 which has older software (depending on your hardware that might be an issue) but you will get security fixes until 2011 or you wait another day or two and get the brand new Ubuntu 9.04 which will be supported for a year (or even 18 month, I am not sure anymore).

As for stability and old/new:
I personally think that the 8.04 LTS release is rather stable and already has seen a lot of fixing already. You will get security fixes (which you surely should install timely) until 2011. 
I do NOT consider any other release stable or safe (as for older releases). And yes, it is an issue with old versions if you do not get any security fixes! I would NOT consider them safe anymore.
The new release 9.04 seems to be quite promising if it comes to stability.

### Post by snowpine on 2009-04-23
The word "stable" has a different meaning in the Linux world. A "stable release" means that all software packages have been "frozen" at their current versions. You continue to receive bug fixes and security updates, but never a major upgrade of a package (for example, OpenOffice never upgrades from 2.4 to 3.0 in a stable release). 

All Ubuntu releases are stable releases by this definition. 

If by "stable" you mean "reliable" then I would recommend 8.04 as the most reliable Ubuntu release, simply because it has been around for 6 months longer than 8.10 (and 12 months longer than 9.04) so there has been ample time for bug fixes. 

I would not recommend any Ubuntu version older than 8.04. It is important to use a currently supported version so that 1) you can install new applications from an active repository; 2) you continue to receive bug fixes and security updates.
