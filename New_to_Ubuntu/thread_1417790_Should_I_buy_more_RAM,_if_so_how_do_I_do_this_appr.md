### Post by -kg- on 2010-02-27
You likely don't need over 2GB of RAM unless like has already been said, you are doing ***heavy*** video or image editing, or doing something that is very RAM intensive, like intensive gaming.

Another reason to need more RAM is if you want to experiment with certain things, such as the new [Tin Hat Linux]("http://opensource.dyc.edu/tinhat") distribution.

The way Tin Hat looks, though, you would need more than that to run a truly effective and useful "installation."  Like Puppy, it is designed to totally run in RAM, but is a full-fledged Gentoo distribution, minus the portage and kernel trees.  According to the website:

> Tin Hat requires about 4 GB to run comfortably, 3 GB for the tmpfs root file system, and 1 GB for paging. If one wants to further reintroduce Gentoo's portage system and/or the kernel source tree, 4GB becomes a very tight squeeze. Forget adding any more software after that, which leads to the paridoxical sitatution: why else would you reintroduce portage/kernel trees if you don't plan to add any new software? Although we provide an i686 release, in our lab we run the amd64 version on 8 GB boxes in which we reintroduce portage/kernel and add the entire Open Office suite.  One gets spoiled when your word processor pops up in mere seconds!

Sounds interesting, but perhaps a bit onerous to me.  But if one were to want to experiment with something similar, that would be a reason to upgrade RAM.  Other than that, unless you are presently doing something that would benefit from it, a RAM upgrade would likely only gain you bragging rights.
