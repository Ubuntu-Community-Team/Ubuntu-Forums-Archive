Ironically, the best suited distro is Ubuntu. Beryl was a fork of compiz, and has been re-merged into Compiz, which is installed by default with Ubuntu Hardy Heron (8.040.

Ubuntu has CnR capabilities, only instead of .exe, you double click .deb files to install. However, using the builtin managers, you can download and install in a single step.

WINE is a ground-up reworking of the Win32 platform. While it does not come by default in ubuntu, it can be downloaded and installed with defaults with a single command.

There are a variety of docks you can use. AWN, Cairo and Kiba spring to mind. Note that these require a compositing manager installed and configured, in the case of Ubuntu Hardy, that's Compiz.

Also, Beryl/Compiz, WINE and docks work on all gnu/linux platforms, but they're easy to install and configure in Ubuntu.

Maybe someone can help out with whatever problems you faced in Ubuntu previously.

### Post by aquavitae on 2008-06-23
Ubuntu takes about 2Gb in total, but I'd advise at least 10Gb so you've got a decent amount of breathing room. The 1-2Gb swap is for virtual memory. Windows has it too, but instead of a partition it uses a file on the existing partition. The disadvantage of that is that its much slower as it has to work with a large file on an already busy file system. The other problem is that depending on the size of the file, the amount of free space may vary considerably.

Why do you need wine?  Usually the progam you think you need it for either runs natively in linux or has a good equivalent. 

What exactly do you mean by click and run?  If you're referring to easy installation of software, then you just need to use a package manager (e.g. synaptic) which is actually even easier than click and run. But if you actually want to run an exe file then you need wine (see my previous question:))

### Post by ByteJuggler on 2008-06-23
> **phoenixankit said:**
> What about Sabayon?

OT: How much space on the HD does ubuntu take? And, why does it need a separate 1-2 GB drive for swap or something? What is that?

(My apologies if this is already answered elsewhere in the thread - I've only read to this post and am answering it.)

On a modern machine it doesn't in reality need nearly as much as that.  Nor do you *have* to have it on a seperate partition.  It is possible to set it up on a swapfile if you like, like Windows does.  Linux by convention does it on a swap partition, as this improves performance slightly when swapping does occur, as having a dedicated swap partition removes one layer of indirection (the filesystem) and prevents fragmentation and optimises for speed, by keeping the swap area in one location at the start of disk, so its unaffected by the other filesystems on it.

If you want me to explain how to set up swap on a swapfile the please ask (and/or google first.)
