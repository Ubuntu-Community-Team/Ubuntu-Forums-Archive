Ubuntu 16.04LTS  kernel 4.15.0-39-generic

Used trace tools during high CPU and did verify getting list_lru_count_one  parent call from from shrink.slap.part.50 (in kernel)

According to bug tracker it seems is in 4.14 to 4.18 and the fix seems to suggest in 4.19
According to this post: [https://askubuntu.com/questions/517136/list-of-ubuntu-versions-with-corresponding-linux-kernel-version](https://askubuntu.com/questions/517136/list-of-ubuntu-versions-with-corresponding-linux-kernel-version)
