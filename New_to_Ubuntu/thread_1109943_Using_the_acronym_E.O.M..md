But...... Wikipedia also say's in the very first paragraph the following:

[http://en.wikipedia.org/wiki/End_of_Message](http://en.wikipedia.org/wiki/End_of_Message)

"End of Message or EOM signifies the end of a message, often an email. The subject of an e-mail may contain such an acronym to signify that all content is in the subject line so that the message itself does not need to be opened (e.g., "No classes Monday (EOM)"). This practice can save the time of the receiver and has been recommended to increase productivity. In addition, Gmail accepts the acronym EOM at the end of a subject line to suppress warnings that there is no content within the body of an email."

I would conclude that E.O.M. usage allows for many benefits, to include:

1. All contents of message contained in subject line.

### Post by CatKiller on 2009-03-29
> **suomalainen said:**
> Well, if I wrote in the subject line:

"What's the terminal command that lists connected USB devices? EOM"

What makes this inappropriate?

The title should be descriptive of the problem that the poster is experiencing, to enable others that might be able to help to focus their attention where it is most useful. The body of the message should contain sufficient detail to enable others to help the poster quickly.

If a problem can be successfully reduced to a single sentence, then the poster doesn't really need to be using the forum at all; the answer can probably be found through other means, such as a simple Google search, and the poster is just wasting the time of the volunteers who help here. That's pretty offensive behaviour.
