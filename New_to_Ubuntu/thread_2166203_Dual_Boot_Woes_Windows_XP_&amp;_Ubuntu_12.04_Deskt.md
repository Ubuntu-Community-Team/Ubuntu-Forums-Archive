### Post by oldfred on 2013-08-08
Added code tags.

As sudodus says, the boot.ini must refer to the primary NTFS partition Windows is installed into.
The drive 0 reference is to the boot drive. But BIOS determines boot drive and with grub (and Windows?) the boot drive is always hd0 or drive0. But if you boot from sdb, it will still be drive0, and drive mapping may not be required. If grub is in sda and BIOS boots sda, but installs are in sdb, then you need drive mapping.
