### Post by Woodgar on 2009-07-25
Whilst searching for help with a different problem, I came across this thread and I believe I can help with the original problem.

Obviously it's a bit late for the OP, but it might help someone else...

I get this behaviour if I use FSpot to import images from my camera onto my PC and then manually move the images later to a different directory. When I next load up FSpot it shows me what is in its catalogue, ie the thumbnail list, but it then can't find the file when I try to edit it, which is quite understandable as I've moved the file to different location.

The solution is to simply delete the images from FSpot's catalogue and then re-import them from their new directory.
