### Post by HereInOz on 2010-10-09
The root user is turned off by default in Ubunbtu and the other versions.  You would have nominated a username and a password during the install, and that should be the one you use.

To perform administrative tasks, you will be prompted for a password at the time, and the password you use is the one you nominated during the install.

To do admin tasks from the terminal, you use the sudo command, such as sudo vi /etc/hosts, and that gives you admin rights for 15 minutes.

So use the username and password you nominated during the install.  There is no root user in *buntu.

The stuff you tried to do in recovery mode may have messed up stuff too much for a newbie to fix.  So I suggest you reinstall Ubuntu.

During the installation, the installer will ask you to choose a user name and password.  **Make sure you don't forget this username and password as you will need to use them a lot.**

Unlike other Linux distros, Ubuntu doesn't use a root account.  Instead, when you want to do administrative tasks, you use the command **sudo** in conjunction with your user password.  Look [here]("https://help.ubuntu.com/community/RootSudo") for more info on that.

Linux is different to Windows.  So you need to learn a bit of new stuff before you can do all the stuff you could in another OS.  Do what I suggested, reinstall Ubuntu, then spend a bit more time learning how to use it.  You can't expect to know how to do everything in one day!  But a few hours' work *will* enable you to use Ubuntu for simple tasks like email and web browsing.

So do as I suggested.  If you encounter any problems, start a new thread in this forum and someone will surely help you more.
