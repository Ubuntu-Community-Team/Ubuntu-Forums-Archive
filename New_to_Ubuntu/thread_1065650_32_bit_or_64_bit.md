### Post by blackgr on 2009-02-10
WIth 64bit system you will be able to detect more than 4GB of ram. While 32bit will only show you about 3.5 GB. There are several workarounds to make a 32bit system support 4GB of ram but the 64bit solution  is mostly suggested.
32bit on the other hand are better for low memory computers ,while the total consumptions of a 32bit system versous a 64bit is less for the same programs. This is because of the 32bit architecture.

### Post by 3rdalbum on 2009-02-10
If you install Ubuntu again (as opposed to just upgrading your existing one in-place), then you might as well install the 64-bit edition. There may possibly be a small improvement in the performance of intensive tasks, and you will be able to address all your memory as opposed to 3.3-odd GiB of it.

There's no special reason not to install the 64-bit edition, yet no reason to immediately install it. Yes, you can have Vista 32-bit and Ubuntu 64-bit on the same machine, or the other way around (the processor switches between 64-bit and 32-bit as required).

### Post by Nepherte on 2009-02-10
Your processor is indeed 64bit compatible. When you install software packages, synaptic will automatically fetch the 64 bit version of the software. The repositories are pretty much the same, there is just a tiny amount of 32 bit only software, things you'll probably never use. Since recently there exist 64 bit versions of flash and sun java, things that used to be 32 bit only.
