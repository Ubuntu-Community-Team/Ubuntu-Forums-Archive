### Post by Rumtscho on 2010-02-19
I've got two monitors in a TwinView setup, with a total resolution of 3520x1200. Maximizing a window to either one is usually inefficient, as most applications I use don't need the width. 
Under WinXP, I am using Gridmove. It divides every screen into a pattern of areas, say 3 areas on the small monitor and two side-by-side areas on the big one (selection of the pattern is done via config file). When I grab a window with the middle mouse button and move it into an area, it is automatically resized to fit this area. That way, I can easily have multiple windows which use the screen efficiently and don't overlap. I know that there is an application for Mac which does the same, but don't know the name. I even don't know the name of the application type, so I cannot google it :( 
Does anyone know such an application for Linux? Or maybe a way to have smaller workspaces (of different size) which don't get switched, but show beside each other on the same monitor? The perfect solution will also trick applications with a full screen modus (adobe reader) to take up their own area only instead of the whole screen.

### Post by Temposs on 2010-02-19
Compiz has this kind of functionality built in. In the CompizConfig Settings Manager it's called Grid. I use it, myself

### Post by chaanakya_chiraag on 2010-02-19
Another possibility is to use a tiling window manager (Google that for more info).  Basically, there are no "windows" to speak of.  The window manager divides the screen up into parts depending on how many applications are open.
