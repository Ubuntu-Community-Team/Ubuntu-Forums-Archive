### Post by Cheesemill on 2013-06-02
Installing software in Linux is a lot different than installing software in Windows. Instead of downloading files from the internet and trying to install them you should use the package management system of your OS. This way all dependencies are taken care of for you and the applications will be automatically kept up to date. Downloading and compiling the source code for a program should only be done as a last resort.

[https://help.ubuntu.com/community/AptGet/Howto](https://help.ubuntu.com/community/AptGet/Howto)

Delete whatever you have downloaded and we'll do this properly.

Rrdtool is in the standard repositories for both Ubuntu and Debian so it can be installed by simply running the following command...
```
sudo apt-get update && sudo apt-get install rrdtool
```


Another thing, if you are going to be using a VM as a development platform for your Raspberry Pi then I would recommend installing Debian instead of Ubuntu. Although Ubuntu is based on Debian there are some slight differences between them, one of these is that they have different versions of different packages in their repositories. As Rasbian is exactly the same as Debian (just compiled for a different architecture) then by running Debian on your VM you can be guaranteed that you will be using identical software on both platforms.
