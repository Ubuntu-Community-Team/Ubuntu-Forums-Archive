### Post by oregonbob on 2009-08-23
I have a dual boot Windows Vista 64bit and an Ubuntu 9.04. The Windows is the default boot, I have select boot disk at startup and Ubuntu is on another drive. I had not booted Ubuntu in about 2 weeks. I booted it up. It said it wanted to do updates and an "upgrade". I said OK, it went through upgrade, to what I don't know because it did not specifically say, but everything looked like standard Update Manager stuff. Where I may have answered wrong is it asked if I wanted to use maintainers menu.lst file, so not wanting to miss something important I said "yes". I reboot Ubuntu and all seemed well.

But later I reboot up Windows Vista 64 and apparently the Ubuntu upgrade scribbled on my Vista 64 drive. Now Vista won't boot and fails recovery with "critical boot files missing". I have no idea why ubuntu decided to mess with a completely different drive. So now I can no longer boot Vista64, which I need for apps with no equals on Ubuntu 64.

P.S. If I had it to do over, if you have a workstation, do NOT install Ubuntu 64, just go 32 bit because 64 bit software is slow in coming. Many 64bit users are finding this out. Even Windows Vista 64 has a lag on vendors releasing 64bit versions. Usually they try to adapt 32bit, which causes bugs and problems. What a pain 64bit has been. What a shame, having more than 4GB memory is important.
