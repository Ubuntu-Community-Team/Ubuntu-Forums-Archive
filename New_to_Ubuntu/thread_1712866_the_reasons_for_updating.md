### Post by Hippytaff on 2011-03-23
You can stick with a Long Term Support release for the duration of its support, but once it's not supported there are no updates (to software or security updates). So it's up to you. Of ourse you don't have to but it is recommended (at least to the LTS versions which is currelty 10.04 though once 9.04 stops being supported).
 
There are other distros which don't realese versions as frequently as ubuntu (Debian springs to mind). They tend to be very stable and new versions are not released to a deadline in the same way ubuntu deal with releases.
 
Then there are rolling realese with distros like Arch Linux. Once you have it installed there are no version releases, you just keep updating what you have. These tend to be less stable and more bleeding edge.

### Post by mastablasta on 2011-03-23
It's not updating, but upgrading that you will do.
 
Update - security patches, bug solutions...i
Upgrade - from one system to another (e.g. Windows 98 to WindowsXP, or in your case Ubuntu 9.10 to Ubuntu 10.04)
 
If you upgrade to 10.04 and it lal work then there is no reason to continue the updates. 
New versions of system will have support for newer hardare, some  bugs may be solved and it will have new programmes (e.g. Ubuntu 10.10 has Firefox 3.6, Ubuntu 11.04 will have Firefox 4...), desktop environment (now Gnome, later Unity - which is based on GNOME)...
