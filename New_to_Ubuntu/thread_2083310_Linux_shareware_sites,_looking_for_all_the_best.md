You search using the app name (or something you think might be in the description), e.g.: 
```
$ apt-cache search "Ubuntu Software Center"
oneconf - synchronize your configuration data over the network
software-center - Utility for browsing, installing, and removing software
lubuntu-software-center - Utility for browsing, installing, removing applications on Lubuntu
```
If you don't know which one of the packages in the output from the search is the right one, despite the small descriptions, you can look at the package details:
```
$ apt-cache show software-center
```
or (to just see the longer description, which is typically enough):
```
$ apt-cache show software-center | grep -A 10 ^Description-en:
Description-en: Utility for browsing, installing, and removing software
 Ubuntu Software Center lets you browse and install thousands of
 free and paid applications available for Ubuntu. You can view available
 software by category, or search quickly by name or description.
 You can also examine the software already installed, and remove items
 you no longer need.
 .
 To install or remove software using USC, you need administrator access
 on the computer.
Homepage: https://launchpad.net/software-center
Description-md5: 1dc0e162e89af83013265b684e2d3742

```

### Post by Zill on 2012-11-12
D_E_H0987:  If you haven't done so already then I suggest you install the [Synaptic Package Manager]("http://www.nongnu.org/synaptic/action.html").  This is a very powerful GUI manager that includes searching on categories, package description or by name.  It also has some very useful filtering functions. As you use Synaptic more and more you will wonder how you ever managed without it. ;-)
```
sudo apt-get install synaptic
```
Synaptic, on my version of Ubuntu (Lucid), lists 31267 software packages.  I suggest this is generally more than enough for most users.  Only in very obscure cases is it necessary to trawl the web for additional software, and then only as a last resort.

FWIW, the Free and Open Source Software (FOSS) world is very different from the Freeware and Shareware you are familiar with from the Windows world.  Licensing (such as the [GPL]("http://www.gnu.org/licenses/gpl.html")) is generally far less restrictive and is often referred to as "[copyleft]("http://www.gnu.org/copyleft/")", rather than "copyright". :-)

### Post by monkeybrain2012 on 2012-11-13
Instead of downloading a .deb from some random website (a Windows practise) I suggest you install from cinderella's ppa. 

[http://www.g-raffa.eu/Cinelerra/HOWTO/installation.html](http://www.g-raffa.eu/Cinelerra/HOWTO/installation.html)


```
sudo add-apt-repository ppa:cinelerra-ppa/ppa
sudo apt-get update
sudo apt-get install cinelerra
```

ppas are "unofficial repositories", after adding the ppa for a software you will find it in the software manager just like other "officially" provided ones and you will get updates through synaptic, update-manager, software Center or whatever you use. One reason why people use ppa is to get more updated versions than the official version and get regular updates (the software in Ubuntu's official repositories usually don't get feature updates, only security updates and bug fixes) 

You can add ppa to synaptic via its gui. Go to Settings > Repositories > Other Software and add the ppa (in this case ppa:cinelerra-ppa/ppa) This accomplishes the same thing as the command "sudo add-apt-repository ppa:cinelerra-ppa/ppa" above. After that reload synaptic and you will find cinelerra.
