### Post by stderr on 2009-01-05
I think I'm understanding this correctly...

Ubuntu boots, but when starting the graphical user interface (Xorg) it fails with a black screen and you get a terminal?

If that's the case, what happens if (after logging in) and type

```
startx
```

into the terminal and hit enter? Do you get any info come up on screen that may help us? In terms of accessing your files, you should still be able to see them, if the partition mounted correctly (if it's a separate partition), with

```
ls -l /home/YOUR_USERNAME/
```

For me it's


```
ls -l /home/ace/
```

but you should be able to type ls -l /h and hit tab three times to autofill the 'ome' and display options for the next part

You can copy files with cp, for example to copy all odt documents in a directory:

```
cp /from/here/*.odt /to/here
```

and copy files and directories recursively with the -r flag

```
 cp -r /from/here/deep/down /to/here 
```

If it's just Xorg failing we should be able to sort that out; it's most likely a graphics driver issue.
