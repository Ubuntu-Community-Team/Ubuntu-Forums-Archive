### Post by traditionalist on 2012-05-29
Have a look at this;

[http://ubuntu-manual.org/](http://ubuntu-manual.org/)  and this   [LinuxFilesystemsExplained - Community Ubuntu Documentation]("https://help.ubuntu.com/community/LinuxFilesystemsExplained") 

I have only recently started using Linux ( Ubuntu 12.04  precise pangolin), I have used windows for a very long time, and I still run windows 7 ultimate X64 in a virtual machine, because there are two programs I need that will not run in Ubuntu.

In the first week I crashed the machine every couple of hours,  and had no idea how to recover it, not just "using" it, but trying to set it up as I wanted it.  In the second week I was down to once or twice a day and now it rarely happens at all ( I am now in my third week! :)  )

It is completely different to windows. You have to simply learn some things from scratch. If you have a VM running it, then go for it, no problem if you crash it.  Much the same applies to a live CD/DVD, as long as you don't start trying to use disk utilities or similar, there is nothing to go wrong really, at least not badly.! :)

I found "Unity" completely unusable, I installed the "Gnome Shell" almost immediately after discovering that I could.

I think the system is great, but the initial learning curve is quite steep for a long time windows user.

As long as you keep your data completely separate from the operating system, the worst that can happen is that you have to reinstall, and this is not the same circus it is with windows.

EDIT:  By the way, it is not lack of information that causes the most  problems, it is the massive amount of it, much of which will not apply  to your machine, distribution, or situation.  I learned a very great  deal just by reading through various threads here, indeed, this forum  has been my main source of information. Further, there is not much use  in asking questions about things before you have at least tried to sort  them out yourself.  This is mainly because anybody who wants to help you  will often require information from you that you have no idea how to  get or give, and a set of cryptic terminal commands may or may not help,  but they will certainly help more if you have made some effort to  understand what you are actually doing.
