### Post by Mark Phelps on 2012-09-19
We understand your question ...

Two points you need to consider:
1) Windows executables don't run well (or at ALL) in Linux; Linux executables don't run at ALL in Windows.  So, while it may be possible to write a game for both, you will need to product two different runtimes -- one for Windows, the other for Linux.
2) Real-time game performance depends critically on hardware acceleration -- which is a function of the drivers and other resident software.  Drivers are different in Windows than in Linux.  Some older video cards have current drivers (that will handle gaming well) in Windows but do not have such drivers in Linux.  So, the same game using the same video card might work great in Windows but not in Linux.

And finally, you need to ask Gaming questions in the Gaming forum, and programming questions in the Programming forums.
