### Post by Mopar1973Man on 2013-04-16
Remember the machine has to share resources between 2 operating systems. So if Windows 8 is really hungry then Ubuntu might starve for resources. I find the reverse works better is have Ubuntu as your main OS and Windows in a box then you can limit windows hunger with the virtual box setting and force it to operate in a small confined area. 

Full booted up and running Ubuntu 12.04 right now and typing this message to you on Firefox I'm consumed only 441 MB of RAM... A dream for windows to even come close to doing.

Excellent advice for the Linux enthusiast who has already made the decision to commit to Ubuntu. However, I rather suspect that the OP wants to continue using W8 as main OS and is just exploring Ubuntu, in which case, his strategy is a good one.

@OP

Mopar1973Man is correct, though. Windows is the resource hog, is built to behave like it is the only kid on the block and will bully other OSes even if they are running on VMs. As long as you know this and do not attribute poor performance to Ubuntu, then everything is cool. If your HW is relatively new and has the horsepower to run VMs in Windows, then you should have a relatively pleasant experience. It's a better strategy, in my opinion, than trying to wrestle with W8's new secure-boot-jail trying to dual-boot.
