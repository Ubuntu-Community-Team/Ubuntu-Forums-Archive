### Post by Geoff918 on 2009-12-10
1 - From GUI, Alt-F2
(box will appear)
input gksu gedit /etc/hostname
(will prompt for password as you're changing something not in your home directory)
overtype trilobite and save file
(go ahead and exit this)

and then again:

Alt-F2
gksu gedit /etc/hosts
should have something about 127.0.0.1 (which is your local machine)
change it there as well
save and exit

2 - I don't typically do this one. Sorry?!? Probably something to do with Samba. You may want to search Samba shares, and may need to install some additional software. Hopefully someone else has experience here! :)

### Post by Iowan on 2009-12-13
Ordinarily, 127.0.0.1 is for localhost (and localhost.domain). The hostname is usually assigned to 127.0.1.1. Hope you got */etc/hosts* and */etc/hostname* both changed. If only one is changed, **sudo** has problems.

You can change the workgroup in */etc/samba/smb.conf*.

A lot of applications actually use the "network" to communicate with other parts of the system. They send stuff to "localhost" or 127.0.0.1, which are both the same thing. You just want to make sure that your hostname gets mapped to 127.0.0.1. I forgot to mention that in my first post, so be sure to do it (thanks Geoff918).
