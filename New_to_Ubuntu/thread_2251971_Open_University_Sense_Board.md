### Post by coffeecat on 2014-11-09
A brief analysis of the terminal command:

```
sudo /usr/bin/sense.sh
```

sense.sh is an executable shell script, vaguely analogous to a batch script in Windows, but shell scripts in Linux have the potential to be far more powerful.

/usr/bin/sense.sh is the full path: the script sense.sh will be found in the /usr/bin directory. The command is prefaced with "sudo" which, after you have authorised with your password, gives temporary root privileges to the script. That also has a vague analogue in Windows - the UAC authorisation prompt.

I note from the version of the package I downloaded that the sense.sh script itself calls the binary executable /usr/bin/scratch_squeak_vm. What that does I have no idea, but its name I find intriguing! A binary executable is compiled code - analogous to a *.exe file in Windows.
