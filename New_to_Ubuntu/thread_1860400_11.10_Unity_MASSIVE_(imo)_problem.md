### Post by thatguruguy on 2011-10-14
Try this.

Log back in to your main account, but choose "Unity2D" from the login screen.  This will let you change some things in Compiz without it trying to apply the changes as you make them, which makes it crash.

Once you get to the Unity2D desktop, open CompizConfig Settings Manager.

Look for the "Ubuntu Unity Plugin".  My guess is, the checkbox associated with it is unchecked.  Check it.  You will be prompted to allow some changes in order to resolve some conflicts.  Make the choices that will enable Unity to work.

Once you're done, exit out of CCSM and restart your computer.  This time make sure to choose the Unity3d desktop, and it should come up.

### Post by jrbales on 2011-10-15
> **Fear091 said:**
> Actually, while you're here, can you explain why compiz closes if I try to uncheck OpenGL?
 
I think this is due to Compiz being an OpenGL composting manager. When OpenGL is unchecked Compiz no longer has access to a plugin it requires to run and therefore crashes. ([http://www.compiz.org/](http://www.compiz.org/)). 
 
Unity also requires OpenGL in order to function. Both the OpenGL plugin and the Ubuntu Unity plugin need to be checked. ([http://opensource-sidh.blogspot.com/2011/06/enable-3d-cube-with-unity-in-ubuntu.html](http://opensource-sidh.blogspot.com/2011/06/enable-3d-cube-with-unity-in-ubuntu.html))
