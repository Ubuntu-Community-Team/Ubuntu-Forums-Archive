IMHO Windows and Ubuntu use two different partition tables, because Windows ignores all entries if "42 SFS" (Dynamic Disk) is present and uses its own partition table:
> ```
/dev/sda1                  63         2,047         1,985  42 SFS
/dev/sda2               2,048       409,599       407,552  42 SFS
/dev/sda3    *        409,600   874,369,023   873,959,424  42 SFS
/dev/sda4         874,371,070   976,771,071   102,400,002   5 Extended
/dev/sda5         874,371,072   913,430,527    39,059,456  83 Linux
/dev/sda6         913,432,576   960,305,151    46,872,576  83 Linux
/dev/sda7         960,307,200   976,771,071    16,463,872  82 Linux swap / Solaris

```
So it is possible that Ubuntu and Windows use overlapping areas on disk.In these overlapping areas Ubuntu can delete Windows data and Windows can delete Ubuntu data. Backup all important data if possible and then try [testdisk]("http://www.cgsecurity.org/wiki/TestDisk") from an external bootable medium to recover your Windows partitions. Good luck.

edit: It is definitely not a Wubi caused problem.

> should I installed it alongside windows or should I create unallocated partition for ubuntu?

Only Linux tools for Linux partitions.
If you let Ubuntu installer do the job then you will have 2 partitions:
Ext4 /
Swap

In good set up '/' parittion and '/home' are separate. In the above /home will be in '/'. So:
Ext4 / (25-30Gb)
Ext4 /home (This will store your files 'music', documents etc... give some genereous space)
Swap (2Gb)

To achieve this three partition setup you'll have to take control and tell the installer about your partitions.
Make three partitions from your 'unallocated space' with Gparted.

Use as = Ext journaling
Mountpoint = '/' for root and '/home' for home partiiton.

### Post by shellback84 on 2014-09-07
I am not an expert Linux or Ubuntu user but I have been building and repairing computers for many years. In my opinion I think you have lost all your files. One of the experts on here may know a trick to get them back but if you are like me then my guess is you will need to except the fact that you might have to start all over. This has happened to me using Ubuntu in my past and why today I am being very careful before I type any commands. In short I am still spending days learning this operating system better before I get to frosty with it. Lesson learned. Yes you can load both operating systems into one hard drive but if one thing goes wrong the Windows OS seems to be the one that goes bust. Unless you are a Linux master like a lot of these people on this web site are then what I would do is get a second Hard drive. This way you can control what hard drive you use in your computers BIOS. After installing your second hard drive decide which one is going use windows 8 and load windows 8 first. If you use the same hard drive you have now then reformat it and unplug the new hard drive. When you install Ubuntu plug in your new drive and now unplug the Windows 8 hard drive. You will need to go into your BIOS and make the Ubuntu hard drive the master and or the same for Windows 8. This is the safest way to not lose information. Put in either the USB stick or the disk and load Ubuntu. Now plug in both hard drives. Your computers BIOS now controls which hard drive you load and this way you control which OS is loaded at start up. For me after I updated my 14.04 the grub loader loaded in itself after a restart and I do not even need to use the BIOS any more as the grub loader sees both OS's now. Hope I helped.
