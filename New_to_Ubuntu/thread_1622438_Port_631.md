### Post by patrik k on 2010-11-15
With Zenmap, I am being shown port 631 (IPP) which is the printer port is open.
Not sure if this is an issue with Linux, but is a vulnerability with Windows.
Found a post from LinuxQuestions.org that address' this.

Original post question-
**"I've looked in cupsd.conf and it's not clear to me what needs to be done,  as I don't really understand how much of the server aspect of CUPS is  needed to print. I'd like to be able to continue printing through my  direct usb connection to the_ printer _[[COLOR=blue][/COLOR]]("http://www.linuxquestions.org/questions/#")**[B][URL="http://www.linuxquestions.org/questions/#"]
[/URL], but close this port and not allow remote access. What changes do I need to make to cupsd.conf to do this?[/B]"

The response was-
```
**#**Listen** ***.631
```then add
```
Listen **127.0.0.1**:631
```and the remark was-
**"Then the port is closed to the world and CUPS is still listening locally.**"

Basically, that is the config I use, USB to printer
Don't like open ports if not needed.
Coarse, if not an issue, no need to change eh?
Thanks.
