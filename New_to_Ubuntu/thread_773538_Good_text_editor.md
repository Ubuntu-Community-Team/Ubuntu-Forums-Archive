### Post by Tatty on 2008-04-28
Depends what you mean by "good".  Gedit is the standard GUI text editor.  nano is my preffered CLI text editor.  

vi is a popular one for programmers because it can be used very speedily - once you have learned how it works.  For beginners however vi can seem really odd because its interface is very different to most text editors

### Post by Xiong Chiamiov on 2008-04-29
For PHP, I use Komodo edit.  For Java, I've been using Eclipse, but the interface finally drove me nuts, and I'm switching to Kate.  It's simple, but it has code folding and a terminal window that collapses out from the bottom for compiling and running.  When I just need to patch something up in the terminal real quick, I use vim.  If you do use vim, [here's a good cheatsheet]("http://www.tuxfiles.org/linuxhelp/vimcheat.html").
