### Post by ptn107 on 2009-10-09
I checked the server 'in.archive.ubuntu.com' and it's up and running.  Edit your sources.list file:
```
sudo gedit /etc/apt/sources.list
```

and verify it looks like this one (minus the line numbers): [_http://ubuntu.pastebin.com/m267d51af_]("http://ubuntu.pastebin.com/m267d51af")

Also make sure there are no files in the /etc/apt/sources.list.d/ folder.

Try again:
```
sudo aptitude update && sudo aptitude safe-upgrade
```
Double-check your /etc/hosts file too:
```
cat /etc/hosts
```
Mine looks like this:
```
127.0.0.1	localhost
127.0.1.1	phil-desktop

# The following lines are desirable for IPv6 capable hosts
::1     localhost ip6-localhost ip6-loopback
fe00::0 ip6-localnet
ff00::0 ip6-mcastprefix
ff02::1 ip6-allnodes
ff02::2 ip6-allrouters
ff02::3 ip6-allhosts
```
Obviously 'phil-desktop' will be replaced by your hostname.
