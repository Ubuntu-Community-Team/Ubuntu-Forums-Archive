### Post by drs305 on 2010-10-22
Actually, it looks like it's working. ;-)  (Not that this is what you'd want.)

Your custom menu entry really only tells grub to take it back to the main menu.

The custom menu is normally meant to be *added* to the grub menu and includes only the introductory lines and the  menuentries. It doesn't have to be done that way, but ...

If you explain what you want it to do we can give you more details.


Here would be two items that users might want in a custom menuentry:

The first entry has modified the grub.cfg entry to make it a "universal" entry which will always boot the latest kernel. The reason is the exact kernel designation has been removed. Of course, you could also make one for a kernel you *know* works.

Sometimes users want a different name than Windows Loader, so they make a manual entry and if you don't have any other OS's except your main Ubuntu partition, they disable the os-prober so it won't add Windows to the normal menu.

Also note I added a line at the top so when you run update-grub you will see a message in the terminal that this menu being added to the grub menu.

> 
#!/bin/sh
echo "Adding 40_custom." >&2
exec tail -n +3 $0
# This file provides an easy way to add custom menu entries.  Simply type the
# menu entries you want to add after this comment.  Be careful not to change
# the 'exec tail' line above.


menuentry 'Ubuntu' --class ubuntu --class gnu-linux --class gnu --class os {
	recordfail
	insmod part_msdos
	insmod ext2
	set root='(hd0,msdos6)'
	search --no-floppy --fs-uuid --set a451dca3-ab86-48e0-b93c-3d77f2f2a873
	linux	/vmlinuz root=UUID=a451dca3-ab86-48e0-b93c-3d77f2f2a873 ro   quiet splash
	initrd	/initrd.img
}


menuentry "Windows" {
	insmod part_msdos
	insmod ntfs
	set root='(hd0,msdos3)'
	search --no-floppy --fs-uuid --set 2c6edb496edb0b0a
	drivemap -s (hd0) ${root}
	chainloader +1
}

### Post by drs305 on 2010-10-22
> **Jetso said:**
> 'm sorry, but how do I make it just say Windows and Ubuntu. I'm sorry if this is getting aggravating.

In the 40_custom menu, you can put whatever you want between the quotes in any menuentry. Example:
> menuentry '**[COLOR="DarkRed"]Put Your Desired Title Here[/COLOR]**' --class ubuntu --class gnu-linux --class gnu --class os {


If you don't want to see anything else in the Grub 2 menu:

1. Disable the scripts in 10_linux. 
```
sudo chmod -x /etc/grub.d/10_linux
```
Note: This will also disable the recovery mode option. It might be better to leave this active, or place another entry in 40_custom with a recovery mode entry with a specific kernel you intend to keep.

2. Disable memtest86+
```
sudo chmod -x /etc/grub.d/20_memtest86+
```

3. Disable Other OS's (Windows, etc)
```
sudo chmod -x /etc/grub.d/30_os-prober
```

After you make the changes, run "sudo update-grub".

If you really want to see a view of geekdom, here is the thread to do all this by making changes to the scripts while keeping them active:
[Grub 2 Title Tweaks Thread]("http://ubuntuforums.org/showthread.php?t=1287602")
