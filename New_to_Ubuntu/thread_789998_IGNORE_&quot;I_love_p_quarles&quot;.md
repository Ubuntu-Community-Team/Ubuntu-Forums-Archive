Maybe thare should be a nag screen during registration that would force new users to read the Malicious commands announcement.

Additional note:
By default the commands rm, mv and cp don't ask for any confirmation before removing/overwriting files:
But you can change this by putting this in the ~/.bashrc file:
```
alias cp='cp -i'
alias mv='mv -i'
alias rm='rm -i'
```

I've done this on my machine for both user and root account.
Unfortunately "sudo" doesn't use the .bashrc file, so using those commands with sudo is still dangerous.
