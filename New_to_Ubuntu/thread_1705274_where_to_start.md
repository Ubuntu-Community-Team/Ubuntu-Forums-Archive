### Post by PunkLV on 2011-03-12
Not a pro, however, I'd suggest starting with Bash and python. 
Python scripts are heavily integrated in the DE generally, and provide the means to implement your own scripts, thus customizong the software you have.
Bash provides the means to manage and/or control how your system behaves in general and helps handling files/applications and pretty much anything aside the software itself.
AFAIK, most of the programms Linux uses do not have a GUI, which makes them run faster, be more efficient and produce less unexpected results, moreover, most of software does not actually "need" a GUI to start with. 
From my experience: I need GUI only for web-browsing, and video/image handling. So this takes my wish to learn C++ a bit further down the list.

When starting out it's often helpful to pick a confidence target, something small enough to get finished but meaningful enough to be worth doing.

If you're a news junky, try making an RSS reader.  Or if you do a lot of graphics work, you can explore the Python scripting options in GIMP.

Keep it simple at first, don't get too bogged down with feeling you need to add every feature you can think of.  Just do one task, and get it done, but make it one you personally care about.

Like you, my eyes glaze over with most programming examples simply because they're not personally interesting problems to solve.

Find some small task you're passionate about, and dive in.
