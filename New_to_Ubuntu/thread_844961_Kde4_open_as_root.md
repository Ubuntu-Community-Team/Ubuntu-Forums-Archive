### Post by perspectoff on 2008-11-06
While I am not sure why KDE 4 removed this option, I agree with the decision completely.

Rootkits are the security risk of 2008, and easily allowing access to files "as root", especially through a file manager, is a big security risk.

I applaud the removal of this option.

Security in Linux is somewhat predicated on doing as little as possible as root. The "Open as root" option in Dolphin3 was not particularly secure. While I liked it too, I always used it with a bit of trepidation. It almost seemed tailor-made for a backdoor exploit.
