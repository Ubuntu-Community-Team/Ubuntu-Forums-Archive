### Post by flyfisherbryan on 2014-01-09
Have found at least a partial solution.  I had been reading in books to use the Alt key to move the cursor while editing in bash.  I finally thought of looking at the man page for bash, and found that in this version (4.2.45 Release 1.99) you use the Meta key instead.  I have no Meta key, but it also says that by hitting Esc _then_ x you will change the ESC key to the Meta key. ( Please note you do this sequentially not simultaneously).  The meta key does not replace the Alt key in all  situations (does not seem to work with auto-complete) but I will keep reading and find the answer to that too. So while I still do not know haow to turn the menu keys off, I no longer have to.

**Lesson**: The help and man pages in bash should be looked at before posting here!  :rolleyes:

### Post by flyfisherbryan on 2014-01-09
One more update:  I did some further reading and experimenting and found that ESC once it is set to Meta is pretty much the substitue for Alt.  The trick is that while Control and Alt are held down while the next key is hit, ESC as Meta only works if the keys are hit consecutively.

I have seen this editing problem posting before, I hope that the next person with this problem can find and benefit from this thread.
