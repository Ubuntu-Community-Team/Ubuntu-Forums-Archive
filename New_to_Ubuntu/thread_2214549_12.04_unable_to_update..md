### Post by Bashing-om on 2014-04-02
sezhiana; Welp;

Could be a whole lot worse.

> 
Mounted on/dev/sda5      1246032 1243172   2860  100% /


Says you are out of "inodes", thus can not allocate any more files ! All you can do in a situation like this is start removing LOTs of files - anything at all that you no longer need, or move all that you can to external storage.
Each file on the system requires an "inode" lot of little bitty files can eat inode's lunch !
The only long term solution here is to repartition -> larger partition  (Guru's can manipulate the inode table in that new partitioning stage to increase the number of the default inodes ).

As to the out put of the "du" commandd, let it run, takes a bit to look at all the file system. You may ignore those advisory statements in du's output;
"`home/sezhiana/.gvfs': Permission denied" is all right
"proc" is a virtual file system - constantly changing - and du can not get a handle on that one. Does not concern us.

As we already know what the problem is, no real need to look at file system usage. 

Once you get the inodes usage down to say 90 %, we can see about cleaning up and updateing the system

### Post by Impavidus on 2014-04-02
When people run out of inodes in their / directory it usually means they have loads of old kernel headers. They are in /usr/src, so use ```
ls /usr/src
``` to view them. The best way to uninstall is using the package manager```
sudo dpkg --purge linux-headers-<some version>
``` Unfortunately, with the package manager already broken it may not be possible to uninstall in this way. In that case, manually remove some tens of thousands of files (=a few directories) there, fix the package manager and uninstall them properly.

It's not recommended to upgrade to 12.10 now. 12.10 is almost end-of-life and its successor already is. Stay with 12.04 for a few more weeks until 14.04 is available. But now that you did a partial upgrade to 12.10 the mess could be a lot worse.
