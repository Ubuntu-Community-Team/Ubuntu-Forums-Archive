### Post by morkov on 2008-11-04
I had the same problem (hanging forever on "Starting Up") when tried to install on a new out-of the package PNY 4 GB.

Here is what worked for me: 
Created new partition table. (Noticed that gparted had a warning sign about it)
Removed and re-inserted the USB drive. 
After that the USB Creator worked without problem.

You don't even have to partition the drive &#8211; USB Creator offers a 'Format' button.

I hope this works for you

### Post by C.S.Cameron on 2008-11-04
Have tried dozens of times using usb-creator to install to a 4 gig Kingston, without success.
This is what finally worked:
Booted Live CD.
Plugged in flash drive.
Started Partition Editor
Created 1 GB FAT32 partition, (on the left side of the bar).
Created a 1.5 GB ext3 partition to the right of this, labeled it "casper-rw".
Created a partition in the remaining space and labeled it "home-rw".
Closed Partition Editor.
Un-mounted and re-mounted flash drive.
Started "Create a live usb startup disk", (usb-creator).
Selected minimum Stored in reserved extra space, (128 MB).
Pressed "Make Startup Disk.
When usb-creator finished, ran "gksu nautilus"
Selected disk and deleted casper-rw.
Shutdown, removed CD, rebooted.
Changed desktop background, connected to wireless and installed FontForge.
Rebooted, changes were persistent.

### Post by ..::benjamin::.. on 2008-11-06
I had the same problem but fixed it in the following way, inspired by the post of C.S.Cameron.

Installed GNOME Partition Editor

started it in a terminal with

sudo gparted

select the USB drive !! don't remove your harddrive partitions

I deleted the existing partition (fat16 in my case) and replaced it with ext2 primary partition.

Un- and replug the USB drive.

start the USB startup disk creator; it says 'the drive needs to be formated' and click the 'Format' button to the right.

Then just install it on the USBdrive, works fine then.

Hope it works for you too.
Ben

### Post by fogster74 on 2008-11-08
I had similar issues... got to the "Starting" message and then nothing. The USB stick was formatted as FAT 16 (I'm using a 1GB drive). This happened on both a machine running from the LiveCD and a machine booting from a full install...


I simply started the partition editor (if it's not installed, go to Synaptic Package Manager and download GPARTED) and deleted the FAT16 partition on the USB key, then created an EXT3 partition.

I removed the USB key, reinserted it (into another USB port, just to ensure the PC recognised the change), started the USB manager and it worked perfectly.

IMPORTANT!!!
If you're about to do this and not sure what you're doing, be VERY careful in Partition Editor. If you fail to change the drive (top right of the window) to the USB key, you'll scrub your entire pc hard drive!!

This has been said above - but just wanted to stress this!!!!

### Post by DSL5 on 2008-11-08
Ok, I got this to work very easily on a 4.0GB SanDisk Cruzer Micro.

First: Back up all data on the usb drive into a folder on your computer.

Second: Use a computer running Windows to format the USB drive to FAT32

Third: Boot up 8.10 from a live disk, insert the USB srive, and run the USB startup disk creator

### Post by aaronphughes on 2009-07-28
I believe the issue is that a Windows formatted USB drive looks like **sdb**, *but* the create usb start up program wants it to be **sdb1**.

Thus when you create a partition on the USB drive using gparted, you get the **1**, changing **sdb** to **sdb1 **and voila it works suddenly.
