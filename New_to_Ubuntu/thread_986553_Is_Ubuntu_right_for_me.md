### Post by Kellemora on 2008-11-19
Hi Me

It depends upon what tricks the programmers who wrote the games used.
If they used API calls to make their game function, it won't work on anything except the OS it was designed for.

Nearly any game after msDOS, written for Windows 3.0, 3.11, 95 or 98 should run without a problem with very few exceptions, because they were primarily self-contained, did not require being INSTALLED and could run from their own folder from the .exe if the .dlls were available.  I packaged most of these old games with the .dlls within the game folder itself so they all run great.

But once you get into the larger proprietary games that do weird things to make them work, like making cpu register calls, you're pretty much sunk on getting them to work on anything other than the OS they were written for.

My wife has over 250 games, some quite expensive too, that won't even run on Vista, so when I bought her another new computer we UPGRADED it from Vista to XP-PRO.........
I had many older games from the 3.0 and 3.11 days and some 95 days games that would not run on 98, but we got all of them to work on XP and amazingly enough, everyone I have tried on Wine so far work great!
But not a one of her games will work in Wine because they all use API calls or other weird tricks designed around the OS.

If you like games, your best bet is to keep a WinDOZE XP-PRO box around or dual boot.
