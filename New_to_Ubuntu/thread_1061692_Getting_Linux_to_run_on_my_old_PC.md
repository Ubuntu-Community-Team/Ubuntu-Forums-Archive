### Post by 3rdalbum on 2009-02-06
Being stuck at a beige screen, or having just the desktop background appear, is a sign that you don't have enough memory to run the Ubuntu desktop CD.

You need 384 megabytes of RAM to run the desktop CD, or 256mb to install Ubuntu from the alternate CD (once installed, Ubuntu can run in 256mb).

The other alternative is Xubuntu, which requires less (not sure how much less, but Google or Xubuntu.com should be able to tell you).

Don't bother using something like Ubuntu 6.06 - with Linux you should always use the latest stable version or the latest long-term-service version. This means you're looking at Ubuntu or Xubuntu 8.04 or 8.10.

### Post by decoherence on 2009-02-06
You can find what RAM you need by going to crucial.com and using the "Memory Advisor Tool" on their front page. Then check Tigerdirect or Newegg for a better deal on the same specification RAM (or buy it from crucial if you want to be sure)

The static thing, as I understand it, works like this: you can either leave the computer plugged in and any time you touch the power supply or a metal part of the chassis, you'll discharge without hurting the electronics. This is usually good enough. In a high static environment you'll want to keep yourself grounded. In that case, UNPLUG the computer, get an electrostatic wrist strap and adhere the other end to something that is grounded. If you're sure of your electrical, this can be a screwdriver jammed in to an outlet's ground. PLEASE be sure of your electrical if you do this... some morons wire the ground live and a prong as the ground which wouldn't be fun for you. To be safe, use a faucet or some plumbing.

I'm not an electrician and I could be wrong about the particulars, but I've been doing this for ten years with no ill effects, either to the computers or myself.

So, in short the rules are this: If you just want to discharge what static you currently have, (you aren't in a dry environment, you don't have carpets or van de graff generators or whatever) leave the computer plugged in and touch the metal parts before you touch any of the cards. 

If you need to stay grounded, unplug the computer and be sure you're actually grounded.

You can certainly buy compatible RAM for this machine. Try here:

[http://www.upgradecomputermemory.com/ram.cfm//memory/C/Compaq/Presario-Notebook-Memory/Presario-1000-Series-Notebook/Presario-1100-Series/](http://www.upgradecomputermemory.com/ram.cfm//memory/C/Compaq/Presario-Notebook-Memory/Presario-1000-Series-Notebook/Presario-1100-Series/)

Basically all you need is a DDR1 SODIMM. Looks like the max RAM is 1GB, so you can get a couple of 512MB sticks and you'll be set for installing Ubuntu using the LiveCD. (EDIT: I agree with decoherence about buying the RAM from Newegg or TigerDirect, rather than a random online shop like I linked you to. But that link shows the kind of RAM compatible with your model.)

The lack of RAM is possibly the issue here, as was mentioned before.

If you want to install Ubuntu on that machine using the current amount of RAM, you should get Xubuntu and use the Alternate Installation CD. Here's the link:

[http://cdimage.ubuntu.com/xubuntu/releases/8.10/release/xubuntu-8.10-alternate-i386.iso](http://cdimage.ubuntu.com/xubuntu/releases/8.10/release/xubuntu-8.10-alternate-i386.iso)

This will let you install Xubuntu using a text interface, so RAM won't be an issue.

Of course, if you upgrade the RAM and the RAM is the only issue, then you should be able to install the regular Ubuntu version using the LiveCD, I think.

### Post by alohatim on 2009-02-06
All day yesterday, I was trying to install 8.1 on a dell desktop created in the same era as your computer.  Using the live-cd I would get to various points in the installation until it would just hang at the blank tan screen.  

I read various things about problems with the drivers for this particular video card (onboard).  The computer is a dell 2350 and has an onboard intel 82845G graphics chip.  The CPU is 1.8 Ghz, it has 256 Mb RAM.  

Finally, I tried starting ubuntu in "Safe Graphics" mode.  Wow!  Suddenly everything started working.  I was able to complete the installation.  

When I rebooted I was still in low resolution mode (vesa).  I just needed to edit the xorg.cong file and tell it to use intel drivers instead of vesa.  After reboot, I have all the different resolutions available.  

My computer is running just fine now with 256Mb.  The other posters are probably right in that a "lighter" version may work better.  

Good luck!
