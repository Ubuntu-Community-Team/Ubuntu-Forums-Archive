### Post by Elmer Fudd on 2010-08-12
You can ignore the Grub files if you are using Burg.

Make your changes in
/etc/default/burg not in /boot/burg/burg.cfg.

Don't forget that the number you put as the default is one less than the OSs position in the menu as the numbering starts an zero

then run update-burg

Let us know if you need more detail.

Also
edit the /etc/kernel-img.conf
replace &#8220;update-grub&#8221; with &#8220;update-burg&#8221;
if you don't want a grub update to re-take control of the boot.

### Post by jdoughert on 2011-02-10
I had this same problem for a while and fixed it by modifying the GRUB_DEFAULT value in /etc/default/burg.

With 3 OS choices (2.6.35, 2.6.35 recovery, and Windows) It would seem logical that the value set at "2" would have been sufficient to point to my Windows configuration. It seems that things failed because I was using the folding mode of the theme.  This appears to mean that the references to recovery version options of the OS choices are stripped away, which in turn allows for only 2 valid choices (0 and 1) to be valid.  I set GRUB_DEFAULT to 1 and it worked fine.

Hope it helps
