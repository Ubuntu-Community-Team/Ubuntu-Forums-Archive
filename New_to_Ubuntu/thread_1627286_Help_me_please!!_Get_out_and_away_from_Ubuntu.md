### Post by TeoBigusGeekus on 2010-11-21
It's just a server problem - your preselected server could be down.
Try this:
go to system>administration>software sources.
At the Ubuntu software tab, press the download from button and select other.
Instead of choosing a server yourself, let ubuntu decide the fastest one for you: Press Select best server.
Ubuntu will run some speed tests and finally select the appropriate server for you.
After that, you can update your system again and finally install wine.

### Post by HermanAB on 2010-11-21
Hi Syd,

How old are you?  There are some septuagenarians using Linux and I am over 50 myself, so if you have a question, fire away.

BTW installing Windows onto a disk that had Linux on it is not as straight forward as it should be.  The trouble is that Windows will likely see the disk as 'broken'.  The solution is to overwrite the first little bit of the disk with zeroes:
```
$ sudo dd if=/dev/zero of=/dev/sda bs=1k count=10
```

After that, Windows would think that the disk is unformatted and should install properly.

### Post by TeoBigusGeekus on 2010-11-21
To create a win7 section:
You need to boot up with a live cd and use gparted (gnome partition editor).
After you boot up with the cd, open terminal and give 
```
gksu gparted
```
If ubuntu tells you that gparted isn't installed, just install it
```
sudo apt-get install gparted
```
After you launch gparted, all you have to do is shrink one of your partitions and create a new one formatted as ntfs.
Then you are ready to install win7.
Keep in mind though, that win7 will overwrite your bootloader (grub-the thing you see when booting up that lets you choose between kernels).
Follow [this]("http://ubuntuguide.net/how-to-restore-grub-2-after-reinstalling-windows-xpvistawin7") guide in order to bring it back.
