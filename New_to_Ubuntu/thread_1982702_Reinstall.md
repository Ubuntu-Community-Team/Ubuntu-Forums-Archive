### Post by Rubi1200 on 2012-05-19
Hi and welcome to the forums babdah :-)

The script results do not look promising.

Stop ALL read/write activity on the HDD right now and only use a LiveCD to do anything from this point on.

This is essential if you want to have any chance, even the slightest one, of recovering some data.

Now, from the LiveCD, download and run Testdisk to see if it can find anything to recover.

Instructions here:
[http://www.cgsecurity.org/wiki/TestDisk_Step_By_Step](http://www.cgsecurity.org/wiki/TestDisk_Step_By_Step)

You can install it on the LiveCD by opening a terminal and running this command:

```
sudo apt-get install testdisk
```Let us know what it finds before doing anything else please.

### Post by carl4926 on 2012-05-20
Well personally I do as the image suggests and use custom partitioning, because I create partitions manually as follows

swap 3GB (same as RAM)
ext4 for / (root) 20GB
ext4 for /home 100GB or more

In your case, with such a big drive, I'd basically have two of the above, so I can do new installs.

For example, look at the partions I have on this
```
