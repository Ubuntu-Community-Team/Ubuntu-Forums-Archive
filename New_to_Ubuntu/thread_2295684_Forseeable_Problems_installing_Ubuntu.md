### Post by grahammechanical on 2015-09-20
The quick answer is to download the Ubuntu ISO image burn it to a DVD disk or a USB memory stick and run the Try Ubuntu session and see if all the hardware works. 

[http://www.ubuntu.com/download/desktop/burn-a-dvd-on-windows](http://www.ubuntu.com/download/desktop/burn-a-dvd-on-windows)

[http://www.ubuntu.com/download/desktop/create-a-usb-stick-on-windows](http://www.ubuntu.com/download/desktop/create-a-usb-stick-on-windows)

If you use a USB memory stick and follow the instructions in the link you can set a size for a persistence file. That means that any changes such as installing drivers will not be lost when you shut down Ubuntu on the USB stick. 

If it was me I would be more concerned that the machine had Windows 8.1 pre-installed. It does, does it not? You must take note of the advice in this wiki page.

[https://help.ubuntu.com/community/UEFI](https://help.ubuntu.com/community/UEFI)

Keep in mind that you should use Windows utilities to defrag Windows and do it more than once, restarting each time to make sure that Windows still loads. And use Windows utilities to resize Windows partitions to create unallocated space for the installer to create Linux partitions from to install Ubuntu in. Make sure that Windows loads correctly after resizing a Windows partition. And have a Windows repair disk as well.

Regards.
