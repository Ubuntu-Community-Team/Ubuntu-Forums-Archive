### Post by Gannon8 on 2008-08-15
"Ubuntu Server" is really Ubuntu without a GUI, unless you install a LAMP server, which is most likely what you are going to do anyway. I think the knowledge they assume you have is how to use only a terminal. You could try installing the desktop version and installing the server packages, so that you will have a GUI to make pages and stuff with it.

### Post by Thelasko on 2008-08-15
I agree with Gannon.  If you are new to Ubuntu, don't use the server version.  It's for hardcore users that want to configure their system from the ground up, including the user interface.

If you want to configure a webserver with Ubuntu, [here's a nice video]("http://download.microsoft.com/download/E/D/D/EDD40B84-7889-4B7F-9EEE-D9D690751DB2/Linux_avi.wmv") the folks at Microsoft put together.  It's a little old, but you will notice that they don't use the server version either.

### Post by Bachstelze on 2008-08-15
Also, keep in mind that Ubuntu Server is essentially Debian, so pretty much any Debian documentation you will find will apply to it.

Other thnan that, as was stated above, it's really just about knowing your way around the command-line. Every service you will then want to install on your server will most likely have its own documentation, and they most often assume you have no knowledge of the service in question (but they do assume you know your way around the command-line, hence why it's the most important thing to learn first).
