### Post by resander on 2012-12-14
The i3 processor should work with Ubuntu.

[http://ubuntuforums.org/showthread.php?t=2058207](http://ubuntuforums.org/showthread.php?t=2058207)

I would install Ubuntu 12.04 LTS, long-term supported for five years.
This Ubuntu has the Unity-style interface with a vertical array of icons on the left=hand side that is reported to slow down processing, which may or may not be noticable with an i3.

You can permanently turn off Unity by:

- install gnome-shell from Ubuntu Software Centre
- Shutdown & reboot and then select 'Classic Gnome (no effects)' from the login dialog (click the icon in top-right corner of the dialog). That disables Unity and and removes the Unity icons.

CPU and  memorry are not the issue with laptops. wi-fi chips, GPU, motherboard (power management), sound chip are.
 
you can try any version of Ubuntu by booting it and then going live mode (try it). if everything works you can proceed with install. if something doesn't work you can try a different version of Ubuntu or different linux distribution. if none of them works then use Windows.
 
you can also try a different version of Ubuntu if you want prefer different desktop (Kubuntu= KDE, Xubuntu XFCE, Lubuntu = LXDE...). XFCE and LXDE are the lighter less resource consuming desktops. default desktop is Gnome. Gnomme and KDE have more bling, maybe more functions available. but use a bit more system resources.
 
aside from these there are plenty other desktops and windows manager to choose from - E17, openbox, icewm, razorqt and many others.
 
anyone is fine as long as hardware is compatible and you have over 1GB ram.

### Post by Mark Phelps on 2012-12-14
+1 for doing Live Mode BEFORE you do an install.

Laptop builders are infamous for using all kinds of special hardware on laptops -- for which, all too often, there simply are no Linux drivers.

The only way to find out true compatibility with a specific laptop is booting from the CD/USB and trying it.  At least that way, you will know what works, and what doesn't, and can spend time working solutions to the problems BEFORE you do the install.
