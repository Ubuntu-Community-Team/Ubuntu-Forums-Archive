### Post by I2k4 on 2012-07-18
It's very much personal preference.  I've been reluctant to let any software "take over" my folder/file organization, mainly because I've found software often has a lifespan of a few years then disappears or becomes incompatible with another operating system and you find whatever it did doesn't work with whatever new software comes along.  I adopted my business folder/file naming system when I started digital photography and stay with it.  Generally, I drag the folder from SD card into a Picture folder, use pyRenamer to rename images.  Never cared for star ratings or tags - again it's personal, but I've never gotten locked in to some transient organizing system that becomes obsolete down the road.

### Post by eric-yorba on 2012-07-18
> **tea for one said:**
> I would use Shotwell more often if it simply followed the folder hierarchy that I have already established.

Shotwell doesn't move files.  If you already have a directory hierarchy you like, just open the prefs and enable library monitoring, and point Shotwell at your photos directory.  It will scan the directory recursively for new photos.

The only time Shotwell organizes photos its own way is if you use it to "import" photos.  Even then, the directory hierarchy it creates is (somewhat) configurable in the prefs.
