### Post by shad0w_walker on 2009-01-04
The simple answer about the server lacking a GUI.
It's a waste of resources. There is no reason a real server needs a GUI. It should all be controlled through SSH and web based control panels if needed. Why waste resources running a full blown desktop on the slim chance of needing to use that desktop? Server is supposed to be on a server, including businesses, which have racks full of servers that don't have any kind of monitor attached to them unless it really needs to be.
