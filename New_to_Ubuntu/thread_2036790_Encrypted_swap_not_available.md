### Post by drs305 on 2012-08-02
If the only problem is an incorrect or missing UUID, this should fix it:

Run the following commands:
```
free -m
```
It should show your memory usage. Your swap line should not be all zeros.

Check for an existing fstab entry and confirm the UUID of the real swap partition:
```
grep swap /etc/fstab
sudo blkid -c /dev/null | grep swap
```
Are the UUIDs the same in both returns. If not, then you need to edit the fstab file and make sure it agrees with the UUID of the seceond command:
```
gksu gedit /etc/fstab
```
Make the change to the swap line, using the correct UUID, save the file, then run
```

sudo update-initramfs -uk all
sudo swapon -a
```
If swap mounts properly, no messages will be generated by the command.

If your system doesn't find a swap partition with the blkid command we'll have to create one.

### Post by NikTh on 2012-08-06
Hi , 
no need for re-install Ubuntu. You created a new swapfile and you must mount it permanent in /etc/fstab. 

Then you will have your 7.5GB swapfile always mounted . 

First give to your swap file the right permissions 

```
sudo chown root:root /swapfile
sudo chmod 600 /swapfile
```

then open /etc/fstab and add this line 
```
/swapfile none            swap    sw              0       0
``` 

reboot and you will be fine , about swap. You will have your 7.5GB swap file mounted. 

I don't really know why this problem happened. It seems something is not right about encrypted swap. 

Thanks

### Post by oldfred on 2012-08-06
If you check the box to encrypt /home when installing, then the installer creates swap as encrypted also, so none of your data is not encrypted if copied to swap. 

Since swap is encrypted gparted and most other tools do not see it as other than empty space.
