### Post by Martje_001 on 2008-09-30
Umount your partition and do:
```
sudo chown $USER:$USER -R /data
```
and if it doesn't work after mounting, umount and:
```
sudo chmod 777 -R /data
```

### Post by sisco311 on 2008-09-30
You can set the owner, group and permissions in the fstab.
Example:
> /dev/sdXY /mount-point ntfs defaults,umask=007,uid=*userid*,gid=*groupid* 0 1
use the ```
id username
``` command to find your userid and groupid.

The default options in Hardy are
> defaults,umask=007,gid=46
read/write access for root and users in the plugdev(46) group
and no access for others.

### Post by nothingspecial on 2008-09-30
As far as I understand it, windows file systems don`t support ownership and permissions. If the files on the ntfs drive are owned by root it means they are owned by no one.
You can`t chmod them because ntfs dosen`t recognise permissions.

### Post by iponeverything on 2008-09-30
Translation of file attributes from ext3 to NTFS is not as easy as doing a chmod. They are two fundamentally very different file systems.

Mounting the partition as rw will give you the ability add/remove/edit files.

see this page for a good explanation:

[http://www.swerdna.net.au/linhowtontfs.html](http://www.swerdna.net.au/linhowtontfs.html)
