### Post by Roger Allott on 2009-12-30
In terms of features, GIMP is brilliant. It goes far beyond what I need.

However, I've grown up (in PC terms) using Paint Shop Pro, originally from JASC but now from Corel. It's interface and functionality is similar to Adobe's much more expensive Photoshop.

Over the year or so that I've used Ubuntu, I've tried to do what I do on PSP on GIMP, but for anything remotely complicated, I always revert to Windows to run PSP.

I think the main issue is that I just don't like the added windows that GIMP gives for the Tools and Layers elements. I want them integrated somehow into the one GIMP window. Also, I don't like the fact that each image opened in GIMP is a new window. That's irritating.

So, is there a decent alternative to GIMP that has an interface more like PSP or Photoshop? I noticed that there's an app called GIMPshop that claims to give that, but that seems to have been out-of-date for a few years now.

I read somewhere that Canonical is considering dropping GIMP for lucid. Is there any truth in that? If so, what is likely to replace it?

I had a problem with Inkscape in jaunty, which was I think related to the driver for the integrated Intel video card.

I've just tried it in karmic for the first time, and I'm impressed. The default Save As option is .svg, which promises very small file sizes. For example, I just loaded a .jpg photo into Inkscape that was 348 Kb, did nothing to it, then saved it as .svg. The .svg file turns out to be just 2 Kb!! The detail looks pretty much identical to the .jpg original.
