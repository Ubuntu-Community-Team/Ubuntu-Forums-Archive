### Post by JKyleOKC on 2014-08-13
The purpose of the Template folder is (and I'm not trying to be snarky here) to hold templates. A number of programs in the system are set up to look there when creating a new file, and if a template associated with that program exists there, to copy that template rather than creating an empty file. So the meaningful answer to both your questions requires explaining just "What is a template?" first.

As an example (and since I'm using Xubuntu rather than mainstream Ubuntu, details might differ on your system) my file manager (that pops up when I select the "Places" icon) has a choice on its right-click menu to "Create Document" with a displayed arrow indicating there's a sub-menu. Going to that sub-menu displays a list of everything in my Templates folder, plus a final entry of "New empty file" that's not in the folder.

If I select an entry from the folder, the program associated with that entry is launched, with a new copy of the template automatically loaded. If I select my "Readme.txt" entry, my default text editor opens with this content already in place:```
This file's purpose is


```

I created that template to provide a standardized format for any "Readme.txt" file I create in the future. You can do the same for any repetitive action you want. Another of my templates creates an AbiWord document that already contains my preferred paragraph styles.

To create a template, all you have to do is create the initial file, then save it in the Templates folder using a name that will be meaningful to you. It's a most powerful tool; I've used this approach since 1970 or so, on mainframe systems, to simplify repetitive writing tasks!

As to what you can use them for, the sky and your imagination are the only limits. Have fun!
