### Post by Paqman on 2009-12-05
I'd recommend installing any security updates immediately, but you can disable the others if you're not keen on them. Go to System > Admin > Software sources > Updates. Set the security updates to install automatically, and disable the others if you want.

### Post by 3rdalbum on 2009-12-05
All Ubuntu updates are tested on the developers machines. After that, any non-security updates are uploaded to the "Proposed" repository. Anyone who likes to live on the (bleeding) edge can try out the Proposed repo, and report any problems that might crop up.

The Ubuntu developers do roll back updates that cause problems, before they hit the main repository. I haven't actually observed any errors and I've been testing Proposed for over a year now.

In other words, what I'm trying to say is: Apply the updates.
