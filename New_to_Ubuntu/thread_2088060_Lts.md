### Post by sandyd on 2012-11-25
If you want a release that will be supported, and have security updates for a long time without needing upgrades, go with LTS. You won't need to do any upgrades every 6 months ;). However, software versions may be a bit older, due to the fact that LTS is designed for stability

If you want a release that has the latest features, programs, .etc .etc, go with non-lts. You will get the latest stuff, but will have to upgrade every 6 months to keep it at the latest versions. Sometimes, it takes a month for bugs to settle down though.

Also, between LTS versions, I advise a new install over Upgrading. Lots of things change between LTS versions, and sometimes upgrades break and such. Do backup your data beforehand though :D

### Post by snowpine on 2012-11-25
> **doja said:**
> thanks,
I thought that for every supported version I get automatically the newest apps.
and that only the OS is not up to date (= upgraded).

Then my question is if the libraries for every OS version include all apps or if they are for some version like for LTS larger?

No, you have that backwards; Ubuntu is not a "rolling release" distro, therefore each supported version features whichever apps were current at the time of release (Oct. 2012 for 12.10, April 2012 for 12.04, Oct. 2011 for 11.10, etc.)

This guarantees that all apps will be stable and integrate well together, just as when you buy an automobile all of the parts are factory-tested to be compatible, and when you take your car in for an oil change they do not swap out all the parts to convert your 2011 Corolla into a 2012 Corolla. As long as your car is safe and reliable then why would you need newer parts? In fact an amateur hot-rod hobbyist who mixes and matching parts willy-nilly from different models/years might break the car. :)

Only security updates and major bug fixes for some programs (system related usually) but no feature updates, it is true that sometimes feature updates do make it through to the backport but this is mostly official talk with no much substance. There isn't much in the backport if you actually look at its contents, most developers don't use it (and the backport is not activated by default so you have to check the box in Software Centre by going to Edit > Software Sources > Updates and reload SC)

Most applications are not updated by that I mean not only new features, you don't even get the bug fixes for end user apps like media players and other things you actually use rather than to run the system. Hence the "stability" of LTS, if something is broken it stays broken for 3 years (now 5) It is predictably broken so it is "stable". :)

This is when ppas come into the picture. Once activated you get regular updates through the regular channel (say Software Centre) Sometimes you need that not necessarily because you want the latest and greatest, you want the ppas just because you want things to actually work while the version in the official repos is broken (because of bugs) or crippled (because of some licensing problems or "Ubuntu policies")

### Post by monkeybrain2012 on 2012-11-25
PPA are unofficial repositories. 

For example,[https://launchpad.net/~rvm/+archive/smplayer](https://launchpad.net/~rvm/+archive/smplayer) this gives you the latest version of Smplayer. On the page there is a section called "adding this ppa to your system" and it says you can add a line to your system, you can do it by opening the Software Centre, go to Edit > Software Sources > Other Software and click add and then cut and paste that line into the box, after that reload the Software Centre and then if you install Smplayer from the SC you will get the latest. It works the same way for other ppas on launchpad.

You can use ppas only when the official one is broken or use them if you want the latest, it is up to you. **monkeybrain2012**, I don't disagree with anything you're saying, but you might be giving **doja** the mistaken impression that Ubuntu doesn't bug-fix their LTS applications.
