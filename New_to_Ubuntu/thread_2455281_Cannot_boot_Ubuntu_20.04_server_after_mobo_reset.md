### Post by yancek on 2020-12-17
You have Ubuntu installed on sdc1 which is a GPT drive so you need either an EFI partition formatted FAT32 on that drive or you need a 1 or 2MB unformatted partitiion on the drive.  Creating an EFI partition as suggested at the end of the boot repair output is probably the best option.  You can do this from the Ubuntu install usb which contains GParted.

You have Grub code in the MBR of sda but we don't know what it is pointing to.  No Grub code in the MBR of the other drives so they won't boot in Legacy mode unless the Grub code on sda is pointing to sdc1.  Even then, you would still need the bios_grub unformatted partition.  The EFI option is better.

Boot repair doesn't show the grub.cfg file from sdc1 for some reason which would be useful.

Beginning at line 72 of boot repair, you can see EFI info for the computer.  There is no entry for Ubuntu there so it won't boot.

If you are unfamiliar with all this, read the information at the link below which is the official Ubuntu documentation on UEFI installs.

[https://help.ubuntu.com/community/UEFI](https://help.ubuntu.com/community/UEFI)
