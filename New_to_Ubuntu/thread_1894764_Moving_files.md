### Post by coffeecat on 2011-12-13
This does come up occasionally, and as far as I know there is no equivalent to the Windows move when you are moving from one physical drive to another. Which means that you do have to go back and clean up the source. However, inconvenient though this is, it is preferable to losing your files. I have seen several threads where a move has failed, and the poster has lost the files both on source and destination. I would far rather check for myself that the copy has succeeded before deleting the original files.

One little tip that makes the job slightly easier. Don't send the original files to trash. Use a hard delete. Highlight all the files that you want to delete and then use the shift+delete key combination. You get an "are you sure?" type prompt, but this bypasses trash. The same key combo works in Windows similarly.
