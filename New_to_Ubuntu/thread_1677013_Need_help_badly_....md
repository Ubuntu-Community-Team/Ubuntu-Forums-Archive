### Post by Rubi1200 on 2011-01-28
Hi and welcome to the forums :-)

Stop all read/write activity on the disk right now!

You need to use a LiveCD or LiveUSB to boot the computer.

Once you do that, do the following:

Boot the Ubuntu Live CD/USB. Choose the option "Try Ubuntu without any changes." Once the desktop loads come back here and do the following:

1. Download the boot info script. There is a link in my signature.
2. Once downloaded, move the boot info script to the desktop.
3. Open a terminal and run the command

```
sudo bash ~/Desktop/boot_info_script*.sh
```

This will create a RESULTS.txt file on the desktop. Paste the _entire_ contents of that file back here in a new post. Once pasted highlight all text and click the # sign on the toolbar to place code tags around the text. 

Once we see the results, we can advise you further.

One option for file recovery is Testdisk, but I want to see the results first.

Thanks.

### Post by amsterdamharu on 2011-01-28
It looks like you've installed Ubuntu using the entire disk option.

If you have a C and D in Windows this doesn't always mean you have 2 disks, the disk can be partitioned into more than one part. Windows would call it a C and D drive but it actually is only one hard disk.

First of all, stop booting and using the harddisk.
Second: get a live CD/usb and start your computer with that.
After you started from live cd press alt + F2, type gnome-terminal and click run
Type the following command:
```
sudo apt-get install testdisk
```
When it's installed please log out (don't reboot but log out).

Press control + alt + F1, log in (text only) and type the following commands:
```
sudo service gdm stop
sudo testdisk
```

I am not familiar with testdisk that much and luckily never had to use it. You might restore partitions or try to get some files off it. 
After restoring you will have to install Ubuntu again.
