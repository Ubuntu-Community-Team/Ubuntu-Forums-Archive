### Post by Therion on 2008-12-19
You can download the file you need to install Ubuntu without your mom's PC being affected, but you won't be able to RUN it until you've burned a bootable CD.

You can run Ubuntu from a LiveCD using the file you download though, and take Ubuntu for a test-drive that way.  When you're done, you shut down the PC, remove the LiveCD and reboot the computer.  Everything will be just like mom left it at that point.

But really... Mom's PC?  I wouldn't suggest this.

### Post by Michael.Godawski on 2008-12-19
The Live CD allows you to boot and test Ubuntu without affecting the PC. While being in the Live Session of the Live CD, the whole Ubuntu system is loaded into the RAM of the machine. The hard drivers remain untouched.

If you however **install** Ubuntu then you can easily purge the whole drive, when you do it without being alert and cautious. 

So do not install Ubuntu on your mother's machine without strict guidance from us, and some researches done by yourself about Ubuntu and the installation process.

### Post by LegoAddict on 2008-12-19
With an EeePC you would have Xandros Linux, based on Debian GNU/Linux, which is the root of Ubuntu but not Ubuntu itself.  So Xandros is like a brother of Ubuntu.

As for your previous query, I think some things are unclear:

Ubuntu is a full-fledged desktop OS - a Windows replacement.  You download it as a CD file (.iso) and make a CD, then install it.

Usually operating system installations are destructive - they need to clear out previous data to insert themselves.  However, Ubuntu has an option where, if you insert this CD while running Windows, you can install Ubuntu alongside Windows.  While it doesn't use Windows or affect it you will be able to use Ubuntu in it's (except for some high-level administrative functions) entirety.

I don't know what this installation does about importing Firefox/IE stuff, but I think it should import your mom's homepage.

But I agree with the above:  You might not want to install this on your mom's computer without knowing how to do it first.  There are the occasional hiccups (though I personally have never heard of a big hiccup with the Windows-based installer) and it's a good idea to have tested it or have someone who has installed Linux before on hand - just like if you reinstalled Windows.
