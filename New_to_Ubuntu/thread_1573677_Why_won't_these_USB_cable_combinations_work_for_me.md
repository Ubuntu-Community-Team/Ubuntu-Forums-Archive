1. USB 2.0 extension lead Type A plug <-> Type A socket (4,95€) 2.0m long

2. USB 2.0 A/M to Mini USB 5P (9,95€) 3.0m long

Here's what I wanted to do. Item # 1 above was going to be used to add some slack to the Internet stick I have in hopes that the added length will allow better positioning of the stick and thus better signal quality......

Item # 2 was going to replace the current (original) cable I run between my laptop and a 500GB Seagate Expansion Portable Drive I use.

My GSM provider provided a Windows XP CD which shows the signal quality of the GSM data network I'm connected to. In the end, using the cable as described in item #1 above Windows XP was unable to locate any network. When the original cable was put back in place then a network was obtained.

I have Ubuntu 10.04LTS running on the Seagate HDD. I replaced the original cable with that as described in item # 2 above. Ubuntu would not boot up!. I attempted 1,000,000 times the boot-up process to no avail.

Is it really that possible the cables can be so particular with whom they are married with?

Or is this a fluke that 2 cables purchased the very same day from 2 different stores are just simply defective????

### Post by migs73 on 2010-09-13
If this USB extender is being plugged into a laptop/netbook the 5V on the USB is likely to be on the low side (4.4V is minimum permitted to be USB 2.0 compliant), especially if you are running on battery. Try it on a Desktop and it will probably be OK.
If the cable is a cheap one it is likely to have reasonably high resistance cable, also adding to the voltage drop that would be seen with the added distance.
As for the Seagate, I've no idea!
