### Post by mapes12 on 2009-09-16
Hi Guys

The problem you are experiencing boils down to vendor support for your installed graphics cards. Some vendors do not allow their driver source code to be used freely. Therefore, If your graphics card is not supported in Linux by the vendor then Linux has to guess at installing a generic driver. This can lead to low screen res so that a too high a res that is not supported by you monitor does not fry your hardware.

You can check if your hardware is supported here: [http://www.ubuntuhcl.org/browse/search+video-cards?category=6](http://www.ubuntuhcl.org/browse/search+video-cards?category=6)

If your card is not supported I would email the vendor and ask them why not?

If they cannot provide a Linux driver then perhaps acquire a card that is supported and tell the other vendor you plan to ditch their product. The more people that do this then the greater chance of vendor support for all of us.

With regard to editing the xorg.conf file then this sometimes works in Ubu releases 8.04LTS backward. But from 8.10 onwards Ubu uses a different automated process to set the screen res, effectively making the xorg.conf file redundant. Other have managed to fix the problem with the displayconfig utility explained here: [http://ubuntuforums.org/showpost.php?p=6086662&postcount=8](http://ubuntuforums.org/showpost.php?p=6086662&postcount=8)

Best

### Post by halitech on 2009-09-16
if you are using Xubuntu, gedit probably isn't installed so you would actually do the editing with mousepad

```
gksudo mousepad /etc/X11/xorg.conf
```
to back the file up easily
```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.old
```
if you need to restore the original file
```
sudo cp /etc/X11/xorg.conf.old /etc/X11/xorg.conf
```

### Post by halitech on 2009-09-16
as per found in this thread:[http://ubuntuforums.org/showthread.php?t=1193709&highlight=Intel+Corporation+82815](http://ubuntuforums.org/showthread.php?t=1193709&highlight=Intel+Corporation+82815)

change/edit/add this to your xorg.conf
```
Section "Device"
    Identifier    "Intel"
    Driver        "intel"
EndSection

Section "Monitor"
    Identifier    "LVDS"
    VendorName    "Sony"
    ModelName    "Sony Vaio XGA 1024x768"
    Option        "DDC"    "false"
    HorizSync    35.00-50.00
EndSection

Section "Screen"
    Identifier    "Vaio Screen"
    Monitor        "LVDS"
    Device        "Intel"
    DefaultDepth    24
    Subsection "Display"
    Depth 24
    Modes "1024x768" "800x600" "640x480"
    EndSubsection
EndSection
```

When you edit yours, use the info about your monitor where it is using Sony viao.

the main issue is that (X)(K)Ubuntu has updated the xorg to a newer version that is supposed to auto detect all the settings that *should* be needed however it seems to fail with certain video cards. Its usually not the best idea to mix versions of xorg from older versions.
