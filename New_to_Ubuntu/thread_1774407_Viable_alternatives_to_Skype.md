### Post by HermanAB on 2011-06-04
Howdy,

Skype is an island system with proprietary gateways to the public landline system.

There are three issues with that:
1. Is it possible for all your friends to switch as well?
2. Do you need landline service as wel?
3. Are you a home or small business user or do you have hundreds of users?

If everybody can switch, then you can use another island system with no problems.  Pretty much anything will then work for you, including Yahoo, Google and Facebook, as well as Ekiga or Jitsi (although the present version of those two are buggy - wait a few weeks if you can - both projects are working with redoubled effort because of the Skype buy-out).

If you need landline service as well, then you should look at Ippi [http://ippi.com/index.php?page=flat_world&lang=44](http://ippi.com/index.php?page=flat_world&lang=44) since they offer gateways in several countries.  At present, Ippi is probably the closest fully working alternative to Skype.

If you have hundreds of users, then you need to run your own server and landline gateway, so then you need Asterix or one of its derivitaves [http://www.asterisk.org/](http://www.asterisk.org/), or even Cisco.

If you are a total Geek and can debug device drivers, then you can look at the GNU Telehony project [http://www.gnutelephony.org/index.php/GNU_SIP_Witch](http://www.gnutelephony.org/index.php/GNU_SIP_Witch).

### Post by beew on 2011-06-04
Some people have pointed out Skype is closed source. True, but there are two issues here, being closed source and being cross platform. Something like google talk is cross platform as Skype has been, that would be acceptable to most of us if the functionality is good. The (well found) fear is that with MS buying Skype, it may stop being cross platform despite what MS says.

So the issue is not open source vs closed source but whether it is cross platform.

In the long run of course open source would be preferred. For one thing it is not in danger of becoming non cross platform because some monopolist buys it. However, in the short run there doesn't seem to be a credible open source VOIP client that can replace Skype. I mean, good luck to try to get your non tech contacts to set up something like Ekiga. 

I am very interested to see how the open source alternatives will be  shaping up in the next 6 months in terms of quality and ease of use/set  up.

I think the present dilemma again highlights a problem with open source, too many similar products but non of them really stand out. Why would they want to keep repeating the effort by others and reinventing the wheel instead of pooling their resources together is quite beyond me. Now we have yet another new project, the gnu fee call. Why do they even bother?

### Post by gradinaruvasile on 2011-06-05
Google talk. It can even be used from the browser on any platform.

Also , there are clients that support it - Pidgin, Empathy have basic support (pc-to-pc with pcmu codec only it seems).
Jitsi will have in the near future full support for google talk call out and such. Right now it works only if both sides use Jitsi, but unlike Pidgin/Empathy, in Jitsi you can select the codec you want to use (speex/32 khz is very good) and also have built-in end-to-end encryption (zrtp).
Also, Jitsi has better connectivity that Empathy/Pidgin (those 2 sometimes fail to connect to the peer if used over the internet), i never had any lost calls with Jitsi (on both sides).

BTW Jitsi is quite stable, i use it for some time and i have no issues with it (i use ALSA only, i had some sporadic issues with PulseAudio). Just make sure you use the real java from oracle, not openjdk or the like.
