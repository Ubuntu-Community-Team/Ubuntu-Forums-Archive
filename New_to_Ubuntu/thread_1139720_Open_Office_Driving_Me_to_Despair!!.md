### Post by olalaaa on 2009-09-13
dear all,

I will explain to you the issues that I had with Open Office and the solution I had to use in each situation.

OO unfortunately does not recognize well the placement of tables - this explains why when you insert a table into a document and save it as .doc , if you wanna open it again, you see no table at all. The table does not vanish, it is simply displayed out of your visual space (out of page, most often in right side of the document, so it becomes invisible).

If you want to check that, just save a big table in a document, save the file as .doc, then CHECK how many KB or MB has that page !!!! A file having 1.2 MB, for example, cannot become 0 MB just because you closed / saved the document and opened it again. So please do your homework before saying stuff like above :)

In my case, I had troubles when opening a big Word document and my tables were part in the page, part outside of it, so I had to do a lot of work to convince them to stay in their place :)

Another issue - that caused by the lack of memory - is explained as follows : each time when you edit the page, OO must save any UNDO and/or REDO versions of the document, and it zeroes these temporary files only when you press SAVE.  Sometimes, it does not purge the RAM memory very soon after you start typing again, so that it uses other locations of memory - if available, of course - to keep the next temporary versions of the document. However, if you close OO, the memory is freed again, and you can play nice with your document, after you restart the program.

Now I think you understand why you cannot work using a 200 pages document - do you imagine what amount of memory would be needed to keep all those re-re-re-edited pages there ??? This just suffocate the available memory, even if you have a nice system !!

My bad experience with opening big documents in OO was that OO crashed (it does not matter if system crashes or not, what matters is if OO crashed, not the computer ).

Well, when opening again the OO, it asked me to chose that "dead" document, but because I started that same document ( I mistakenly did NOT an "empty" start of OO, as a program alone, without asking a document ), it restored my document, THEN it opened over it the initial one, so I LOST whatever was edited since I saved last time.
[B]
My conclusion : always when OO crashes, try an "empty" start of the program, and answer YES when it asks to recover the crashed documents. Never start directly the document, because it will be overwritten by old one.[/B]

I hope this help everybody here.

It has nothing to do with any "Gnome themes". Even if you change the screen resolution or anything else, the document saved stays the same.
