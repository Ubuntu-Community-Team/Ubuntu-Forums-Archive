### Post by ajgreeny on 2012-02-17
If space is not a problem for you, why not run a dual boot with Win 7.  You have, after all, paid for windows so you may as well get some sue from it.  If possible, however, I would try to use the open source applications available for windows, such as LibreOffice and GIMP, which will mean that everything is totally OS independent.  It will also mean that you get used to these applications, rather than having to swap between applications on the different platforms.

I always used to use OpenOffice, Firefox, Thunderbird and GIMP on windows, some time before I ever tried Ubuntu as an OS, so it made my transfer to Ubuntu a great deal easier.

I would anticipate that Ubuntu will run faster, and certainly boot to a usable desktop far quicker than Windows 7, judging from my admittedly very limited experience of W7 on my grand-daughter's machine, where it is tortuously slow to boot to a point where you can actually do anything with it.  Perhaps that is simply all the junk that is on that machine, but my memory of Windows is that it boots fast to a desktop, but it takes another couple of minutes before you can do anything on that desktop.

First off, if it's a new PC, make sure your warranty will still be good with a dual install. ### Post by Scott Baker on 2012-02-17
Howdy. I'm going to address your O/S issues separately, so bear with me. First the Win7. You very likely have "bloatware" on this machine, which is VERY common with new machines. That will cause some of the issues that you're describing. From your web browser, go to [http://www.snapfiles.com/](http://www.snapfiles.com/) and download the following 2 packages; slimcomputer and Revo uninstaller (to get the free version of Revo, type Revo uninstaller free into the search field). Once these are installed, run them in the above order (slim first, then Revo) and remove the stuff you know for a fact that you don't need (if in doubt though, look it up online, or leave it alone). Don't mess with any of their settings, just stay with the defaults. Once finished, defragment your drive. Once all of this has been done, you should notice a marked difference in your performance. Next, your Ubuntu. Get yourself a current copy of Ubuntu, and put it on a large (8 gig or better) flash drive. The reason for the large flash drive is that it is treated like a hard drive (as opposed to a CD/DVD). This means that once booted, it can be updated, and you can install some of the packages you would normally use (I've done this, and it actually works great). Finally, some advice regarding a possible "dual-boot" set up of this machine. I'm a full time tech, and can tell you from personal experience that manufacturers get REALLY hinkey when you start to diddle with THEIR hard drives or hardware. And don't expect a straight answer with regards to this. You can get 10 different answers from 10 different people in the same company. Best advice, if you choose to "dual-boot" your new machine, you're doing so at your own risk. Hope this info helps. :-\"
