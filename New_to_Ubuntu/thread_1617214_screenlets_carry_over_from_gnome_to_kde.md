### Post by trikster_x on 2010-11-09
I just installed kde after running with gnome on 10.04.  I restarted the computer and logged in and to my surprise, the screenlets I had in gnome were also showing up in kde.  While I use these screenlets in gnome, kde has equivalent plasmoids that I would prefer while using that environment.  Is there any way to keep the screenlets from running when I log into kde but have them auto start normally when I log in to gnome?  I don't seem to remember having this problem on prior versions of Ubuntu.

---

### Post by trikster_x on 2010-11-09
Okay, so I tried to fix the problem by unchecking the screenlets, as well as a script I don't need in KDE, in the startup applications in KDE.  When I log back into gnome, anything I disabled is completely gone from the startup applications in gnome.  This is irritating, because I know I have had KDE running in the past and never ran into any problems like this.

EDIT:

So I found the option I needed to change.  It is in the advanced portion of the "Autostart" menu.  It brings up a check box that asks if you only want to apply it to KDE.  I don't recall ever having to have done this in the past, but everything works as intended now.
