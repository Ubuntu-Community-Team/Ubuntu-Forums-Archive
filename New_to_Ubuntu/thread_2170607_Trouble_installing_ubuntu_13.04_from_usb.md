### Post by r_avital on 2013-08-26
Hi Christian3,

The behavior is strange, it might be an issue with insufficient free space on the hard-drive.  Meaning, your Windows-7 might have lots of free space, but the hard-drive partition (the part that is dedicated to windows-7) might be so big that it does not allow for other partitions, and Ubuntu needs at least two partitions (one should be very small, only the size of your ram for "Swap" space, and the other at least 6GB for Ubuntu, more is better).

See if this guide helps.  When you get to the part about dual-booting, it should let you choose between "automatic partition resizing" and "manual partitioning."  If you don't get that choice, there might be a problem with the installer on your USB drive.

How did you get the installer on that USB drive, by the way?

Here's the guide (It talks about using a CD but your USB drive should be just as good.  It also talks about Windows Vista, but the instructions should be the same for Win-7).  [https://help.ubuntu.com/community/WindowsDualBoot](https://help.ubuntu.com/community/WindowsDualBoot)

You're in good shape, because Windows-7 is already installed, which is the preferred method (installing Ubuntu last).

### Post by r_avital on 2013-08-27
Christian3, 

Don't give up now, oldfred and I (and others) can walk you carefully through some diagnostics, and if you follow instructions carefully, you should be able to get us more info so we can help you better.

That said, if you could do the following, it would give us all some better info -- I'm just going to give you what oldfred gave you but with more detail to help you out.

1. Boot with the USB.  Do you at some point get an option to "try ubuntu" vs. "install ubuntu" on your screen?
2. If so, select "try ubuntu" which will make absolutely no changes to your system, but will boot ubuntu.  When it boots, you'll have a screen with a vertical bar on the left with icons on it.  That's the launcher.  Ignore it for now.
3. Press Ctrl+Alt+T and this should open up a terminal window, similar to what you would see as a "dos box" in Windows.
4. Type "sudo parted /dev/sda unit s print" **without the quotes.** Copy the output, and paste it into a reply to this thread.

Option: when you're in that terminal (launched on step #3 above), type "sudo gparted" (again, without quotes).  This should launch a graphical program showing you the partitions.  Move your mouse out of that screen, press Alt+PrintScreen, and you'll be prompted where to save the image (this will be a screenshot of only that program you just launched, gparted).  Save it in Pictures.  Then you can come back here, upload the picture as an attachment, and attach it to a reply to this thread.

Don't give up, it's worth it :)
