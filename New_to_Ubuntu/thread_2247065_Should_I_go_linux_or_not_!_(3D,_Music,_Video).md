### Post by mcduck on 2014-10-05
If you are definitely planning on using Windows (or Mac) software, and especially if you use them for your work, you should stick with Windows. The experience is always going to be less smooth when you try run software on platform it was never intended to run on, and while it may be OK for hobby use when compatibility issues and extra hoops you might need to jump to get something working are not that big of an issue, but for any professional use I'd recommend against it.

That being said, if you are willing to learn new programs, there are plenty of good and pro-level tools for audio, video and 3D work natively supported on Linux. Maya is supported, and of course Blender can do all 3D, animation, compositing, and even some video editing. Lightworks is a decent NLE video editor for any use from light professional to big budget movies, and Bitwig and Ardour, for example, are great Audio/Music tools that run natively on Linux. And of course Gimp, Inkscape and Scribus for image editing, illustration, desktop publishing and similar tasks.

### Post by QIII on 2014-10-06
For $20US you can find a 32GB USB thumb drive.  Rather than a live image or a live image with persistence, you can install a full-on Linux installation.  Provided you do not install proprietary hardware drivers (or you uninstall them when done) this will allow you to move from machine to machine provided you can boot from USB.  

I find this much more satisfactory than a live image or live with persistence.

USB is much faster than optical, but not as fast as a hard drive.

You can have what is effectively a dual boot for $20.

If you disconnect your hard drives during installation you eliminate the potential for the disaster of overwriting Windows.

My personal recommendation is a full installation on a USB thumb drive.  As I said, this will allow you to fully test proprietary drivers for your hardware.

### Post by ThatBum on 2014-10-07
X99 is very new hardware, Haswell-E support might be bad in the current kernel. Try using the OS on a LiveCD first to test it.

Also you probably don't need Wine to do your work on Linux, there's likely open-source counterparts to those Adobe programs.

My two cents.

Edit: Just saw your thing about dual booting. I've done it for many years, there's no danger to it unless you botch the partitioning or something when setting it up. Few other minor things, like having to install Windows to the first partition of the first hard disk so hibernate works, and setting Linux to use local time instead of GMT time on the RTC so the two OSs don't fight each other with resetting the clock. 

The Ubuntu installer has provisioning for dual booting with Windows built-in, it should be simple. However, you should backup important data, just in case. Always a good practice when doing this kind of stuff.

Processing speed is not an acknowledged (recognized) advantage of any Linux distro vs Windows. ### Post by mcduck on 2014-10-08
If you use a program for your work, it's meaningless if the port is exactly the same on all platforms, since you are actually benchmarking the exact tool you'd use. What runs faster is what you'll wan to use, regardless of if the performance benefit is a result of the OS or better optimisation by the programmers ;)

(Of course things are different if you don't want to test which platform is the best for the tools you use, but instead wish to get a generic benchmark of different platforms. Although in that case the game-based benchmarks often suffer from exactly the same problem of different optimisation on different platforms, on both the game itself and the graphics drivers used... ;))
