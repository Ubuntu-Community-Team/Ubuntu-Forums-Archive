### Post by Hospadar on 2009-12-01
just keep an open mind, and don't force linux on yourself.  It's really best to start easy with a dual boot so you can come back to windows if you want.  And if none of your hardware works and you're having a nightmare figuring things out, just use windows (or osx or whatever) until you get things working.  There's no shame in using what works over what you'd ideally prefer to use.

Some people take the plunge all the way to discover that some things don't work and they have no idea how to fix them because they've never used linux before, and then are just turned away from the platform as a whole.  Just don't expect magic, much of the software you will use is written, tested and compiled by volunteers and sometimes things don't go perfectly.

Also, if you aren't familiar with the command line, you should just get over that now and start learning.  Most things in linux can be done without the command line, but it is an extremely powerful tool and one of the quickest ways to get to the root of a problem (or just to find those pictures you were looking for, or jot a quick note, or multiply those two big numbers together).  I almost never turn my computer on and off without using a terminal somewhere inbetween, and not because linux just sucks so bad and I always have to fix things, but because it's extremely fast and I prefer it to many other ways of doing things.  Why wait for an app to start up so I can hunt through a menu when instead I could just type a command?

### Post by t0p on 2009-12-01
In my opinion, the best things about Free software are:

[LIST]
[*]It's free.  Meaning, most Free software is free of charge, and you can share your installation CDs with friends without falling foul of some nasty EULA.  There's also the fact that the source code is available and you can modify it freely, but that's more of interest to software developers;
[*]Linux and the *BSDs are virus-free.  You don't need to mess around with AV software.  Your computer isn't going to end up part of a botnet.
[/LIST]
It's important that newbies realise: Linux is not Windows.  You can't install Ubuntu then expect your favourite Windows programs to work.  There are many programs that have both Windows and Linux versions, but there are also some apps that don't come in a Linux flavour.  One such program is Photoshop.  There are Linux alternatives for most/all Windows programs - for instance, the Gimp can often be used in place of Photoshop; and there are ways of making some Windows apps work in Linux - for example there's Wine and VirtualBox - but it's a mistake to think that Linux is a drop-in replacement for Windows.  It's a different OS, and you need to be willing to learn something new.
