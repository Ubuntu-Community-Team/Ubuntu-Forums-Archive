### Post by The Cog on 2012-03-29
I would guess that the second connecton is overriding the routing and DNS settings of the first one. You can verify this by using the commands 
> route -n
cat /etc/resolv.confto view your system settings before and after enabling the second connection. You may have to restore these or control them manually if there is a conflict in the automatic configuration using DHCP.

Same for your VM if it's bridged, I guess, but let's see the output from the above commands first.
