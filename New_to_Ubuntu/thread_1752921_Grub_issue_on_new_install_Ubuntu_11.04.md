### Post by Hedgehog1 on 2011-05-08
Thanks for posting the script output.

I think the logical place to start is to have you reload grub on your PC.

Please boot off your LiveCD/LiveUSB, select 'TRY'

In the Terminal (Menu to: Applications >> Accessories >> Terminal):

```
sudo mount /dev/sd**a5** /mnt
```
```
sudo grub-install --root-directory=/mnt /dev/sd**a**
```

These command are based on your script output, and are matched to your PC.

Once this is done, please reboot and see if this resolves the undesirable behavior.

Well, I'm surprised it saw grub.cfg if your BIOS says it's only 137GB. I think I'd assume that was an issue although it might not be.

There are several steps you can take to correct this, which include enabling large drives/LBA in BIOS (if it is newer) or upgrading your BIOS from the manufacturer.

The easiest thing in your case though, assuming your space allows, is to shrink your Ubuntu partition to about 130 GB. That will always keep your /boot files within the space seen by BIOS and Grub. If space is tight on the partition (not much free space) you might be able to make it a bit larger but below 137GB.

There are some other options. If you decide to shrink Ubuntu, gparted is an excellent app that can easily do this. Of course there is the standard advice to have backups of irreplaceable data before doing any partition operations.
