### Post by Kellemora on 2009-02-09
Hi MMD

Those kinds of problems are web site programmer induced!
Has nothing to do with your web browser!

Basically they format their web site in a resolution uncommon to most users.  For example:  Back when everyone was using 800x600 resolution, with 1024x768 slowly taking over, many web site programmers chose unusual resolutions not even available in most monitors making their web sites only viewable by a select few.  

Also, rather than sticking with generic coding, they use proprietary code that only works on a few web browsers that can recognize a few of those proprietary codes.

A good programmer will enter code that checks which web browser and which OS you are running and redirect to your computer the page format most suitable for that OS and web browser.  But they still cannot see or control which resolution you are viewing the page as.  I still prefer web sites written to accommodate the 800x600 resolution, that self expand to fill the screen for whatever resolution you are using.
Problem is, they can't squeeze in all that ADVERTISING, so many web sites have ignored page margins and all kinds of things.  It's not uncommon to find web sites that are 2, 3 and sometimes even 4 screens wide due to very poor programming.
