### Post by drs305 on 2010-07-20
Grub 2 is vastly different from Grub legacy. 

It should have been a bit difficult for you to edit the grub.cfg, since it is originally write-protected even from root. 

Can you undo the changes you made to /boot/grub/grub.cfg?

Can you boot at all? If so, run "sudo update-grub" which may fix things. If not:

The easiest way to restore Ubuntu for you would probably be to boot a recent LiveCD and run the following commands to reinstall Grub2. [COLOR="DarkRed"]If your Ubuntu install is not on *sda1*, change and mount the correct device.
[/COLOR]
From a LiveCD terminal:
```
sudo mount /dev/*sda1* /mnt
sudo grub-install --root-directory=/mnt /dev/*sda*
```
Note the first command mounts a partition but the second device references ONLY the device and not a particular partition.

The link "Grub 2 Basics" in my signature line provides a good introduction to Grub 2.
