### Post by arvevans on 2011-08-12
The "Big Deal" about the 4 virtual desktops is more about what they are not capable of than what they can do for you.  Each of these virtual desktops will contain the same icons and task bars.  You cannot customize these to make a particular virtual desktop optimized for any specific task (what a bummer!).

What you can do, is to make more or less virtual desktops (right-click on a desktop label to bring up Preferences where you can change the number of desktops).  You can even name each virtual desktop with your choice of text (edit the "Workspace-n" lines in desktop preferences window), but you cannot edit which icons or which top-bar goes with which desktop.

It would be so much nicer if a particular virtual desktop could be optimized for a particular task.  It would also be nice if your .profile defaults could also be kept in-sync with a particular optimized desktop and the projects it supports.

This is potentially the future of IDE's for each individual task you might want to do, but with programmable virtual desktops you could do so much more than is possible with an application based IDE.

The point is to avoid having to minimize windows and spend time managing the windows. Tot a big difference if you only ahve a few windows open, but the more you open the more work managing all windows ona  singl desktop becomes. You'll spend lots of time locating the right window from task bar, alt-tabbing through them, and moving them around so that you have the right windows on top and nothing is covered by other windows.

With virtual desktops you can instead distribute the windows to separate desktops, usually keeping windows for one task on one desktop. With a suitable amount of desktops for your personal usage you'll avoid most of the work of managing windows and can instead just switch through a much smaller number of desktops.

A typical case when I'm working, for example, is that I have a bunch of stuff open. I need a Firefox window, full screen to maximize the visible area. Then I need a text editor window, a couple of Nautilus windows (one for FTP locations, and one or more fr local folders containing the files I'm working with. Then I need a image editor, like gimp, which also needs to be full-screen and there should never be another program window on top of it. And finally I'll most likely have an instant messaging app and a music player running.

That makes a minimum of 9 windows, usually a lot more (my gimp layout uses two windows + 1 for each open image), two of which require a full-screen view. So what I do is I keep Firefox on first desktop, text editor and file manager windows fit nicely on the second desktop, Gimp on third one and IM app & music player on the fourth one. None of the windows cover each other, I never need to move or minimize a window, and switching between the tasks can be done with a simple keyboard shortcut instead of mousing around and clicking on panel buttons and windows.

So yes, you can manage your windows by minimizing them and clicking on taskbar buttons. or by alt-tabbing through the list of open windows. It's just that it might not always be the easiest or most efficient way of doing things. It's pretty much a question of personal preference, but I can definitely tell you that if you learn the habit of using virtual desktops to manage your windows by task you'll soon learn to love it, and working with all the windows on just one desktop starts to feel pretty cumbersome and annoying (that might count as a downside, actually ;))

Edit: I just wanted to add that virtual desktops definitely aren't 11.04-specific thing. They are a common feature of almost every desktop environment and window manager for Unix & Linux systems. Even most of the really, really old ones had them.
