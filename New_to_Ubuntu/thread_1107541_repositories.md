### Post by drs305 on 2009-03-27
All of the 'failed to fetch' reference *binary-lpia*, which don't exist on the server. If this is a dell-mini, try doing a search for "dell-mini", "repository", and your version (hardy, intrepid, etc) to find a default repository file for your computer.

The support for the lpia (low powered intel architecture) has been a bit problematic. Here is a link to one post regarding a similar problem. 
[http://uclue.com/index.php?xq=2745]("http://uclue.com/index.php?xq=2745")


This thread contains a default dell-mini sources.list for hardy, excluding the lpia ones:
[http://www.ubuntumini.com/2008/10/check-out-dells-mini-9-repositories.html]("http://www.ubuntumini.com/2008/10/check-out-dells-mini-9-repositories.html")

If you use this list I suggest you first back up your existing sources.list. Here is a command that will do so:
```
sudo cp /etc/apt/sources.list /etc/apt/sources.list.bad
```

### Post by 3rdalbum on 2009-03-27
Ubuntu 8.04 is a Long Term Support release - it's one that's partly intended for enterprises that don't want to upgrade to a new version of Ubuntu every 6 months.

Go into System > Administration > Software Sources. Under the "Updates" tab, you need to change "Show new distribution releases" to "Normal releases". Close that window and open your update manager, and you might need to click the "Check" button, and you will be given the option to upgrade.

Ubuntu 8.10 will automatically give you the option to upgrade to 9.04 when that version becomes available. Ubuntu 8.04 LTS is designed to be able to upgrade to another 6-monthly-release OR straight to the next LTS release, which won't happen for a while.

If not, just back up your current sources.list and replace it with the default hardy sources.list. Update the list, then try running your upgrades.

```

sudo cp /etc/apt/sources.list /etc/apt/sources.list.old
gksudo gedit /etc/apt/sources.list
```
