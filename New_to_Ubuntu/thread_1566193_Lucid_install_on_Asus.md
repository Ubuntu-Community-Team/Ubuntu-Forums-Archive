### Post by lkraemer on 2010-09-02
Your ISO download has an associated MD5SUM and/or SHA256 Hash code.
The first step is to download that Hash code and verify the ISO file.
Then, burn the ISO to a CDRW or DVDRW as Disk at Once (DAO) and at 4x
or the slowest speed available.  There is an option in the LiveCD Boot
menu that allows the CDRW to be verified on Bootup.  It's sort of hidden
in a secondary menu.  I'd have to look a bit to locate it for you.
(K3B will generate the MD5SUM before burning the ISO and lets you easily
and quickly VERIFY the MD5SUM.)

Then try the CDRW/DVD-RW on a couple of computers running it as the LiveCD,
making sure everything functions properly.

If all goes well, try an install.  

You can also download via torrent, as another option.

Once that all works fine, try it on your problem machine.  If it won't
install, then download the ALTERNATE (Text Based) Install CD and install
from this CDRW.

If none of that works, as last chance method, Pull your ASUS Hard Drive
and install it in a machine that boots the LiveCD, and install Ubuntu.
When done, remove your Hard drive, and install it back in the ASUS.
It should boot and then update to the latest software.  (Assuming the
Hard Drive is NEW or doesn't have any other OS installed.)
