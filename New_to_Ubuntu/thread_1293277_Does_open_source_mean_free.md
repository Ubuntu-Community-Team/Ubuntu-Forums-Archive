### Post by camper365 on 2009-10-16
Open source does not mean free. It usually does, but not always.
And free does not always refer to free as in price (free beer) it usually refers to freedom as in rights to use the software, right to redistribute it, etc. (free speech).
However, the GPL (the most common free software license) does not prohibit selling software for a price (in fact, if it improves the software, it is encouraged) but it does not allow royalties, and they can only charge you for the cost it requires them to get the software to you (this is left open to interpretation). Red Hat (before Fedora) was a big Linux distro that you had to buy, but it included all the source code (I have not looked into detail about the licensing though).
And when you're buying mysql from the site itself, it doesn't have to provide a free download, but other places might. They are allowed to say that the cost to run the servers justifies the cost of getting the software to you. They could also argue that paying the developers is added to the cost of getting the software to you. source code could also come with restrictions, like not being able to change the source code, only to look at it, or that you can change it, but only for personal use, you can't redistribute it. You may also be restricted on redistributing the binaries. If mysql uses the GPL though, they wouldn't be able to restrict these rights

### Post by Viva on 2009-10-16
Usually, any software licensed under an opensource license is Free(as in freedom), but it does not mean it comes at no cost. MySQL is a bit different in that it is dual licensed so that companies who don't like GPL or want to bundle it inside a proprietary product can buy it under a different license.
