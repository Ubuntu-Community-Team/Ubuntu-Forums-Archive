### Post by cariboo on 2010-11-12
It really depends on how much you've added to your install. If you are using any ppas you'll have to disable them, and make sure you're fully up to date.

The big downside in your case, is that you'll have to upgrade to 10.04 first, then upgrade to 10.04.1. I just did the same thing earlier this week, the upgrade for each version downloaded about 1200 packages, fortunately the cleanup function works pretty well.

### Post by Rex Bouwense on 2010-11-12
I may be wrong but the only advantage to doing an upgrade is the upgrade is supposed to allow you to keep your data and settings.  However, everyone tells you to back up your stuff before you upgrade just in case something goes wrong.  I have never upgraded, always going the clean install route.  There are plenty of people that upgrade and never have a problem so I guess it is personal preference.

### Post by 73ckn797 on 2010-11-12
My upgrades, 8.04 to 8.10, had problems. I ended up backing up my personal data and performing a clean install. Since then I have used a separate /home partition and performed clean installs. I only had to reinstall what I used in addition to the base OS. The configuration files were in the /home partition so things were just as they were before. You can select the original /home again and not format that partition during the install and all will be fine.
