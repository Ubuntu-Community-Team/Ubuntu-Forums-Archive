### Post by Paqman on 2010-01-02
> **mickeyjoe said:**
> But should I just spend $50 more and get a dual core.  I mean, for $50 more isn't a dual core much much faster - ?

Depends. Not all apps are multi-threaded. If you're running a single threaded app it won't go any faster on a dual-core than it will on a single-core.

Where dual-cores come into their own is for multi-tasking, or running multi-threaded apps.

### Post by 3rdalbum on 2010-01-02
> **mickeyjoe said:**
> But should I just spend $50 more and get a dual core.  I mean, for $50 more isn't a dual core much much faster - ?

It's up to you. A dual-core is no faster if you only use one single-threaded program at a time. A dual-core is twice as fast if you use two single-threaded programs simultaneously, or one multi-threaded program.

All ordinary CPUs work with Ubuntu, just like all power supplies and all brands of RAM work with Ubuntu. You should check to see that the rest of the computer will work alright; for instance, I imagine that if the computer is built on an Athlon processor it probably has ATI graphics, which are not ideal for Linux. Not terrible, but not ideal.

### Post by anewguy on 2010-01-03
If you plan on running more than 1 of those apps at a time, that is multi-tasking, and dual core would be of benefit then.  There's another example of dual core being of benefit that it doesn't sound like you'd use, but I guess you never know:  running virtual machines, such as a Windows virtual machine, along with Ubuntu.  If you have any Windows apps that you just can't find a Linux replacement for, then you may want to run either Wine or run Windows in a virtual machine such as VirtualBox.  My suggestion?  If you can afford the extra for dual-core, get dual-core.  Someone else before me has already mentioned that there are other things, such as the graphics adapter, that you will want to check in to as well, since they require a driver, and as mentioned, not all graphics adapter support is the same or as good in Linux.
