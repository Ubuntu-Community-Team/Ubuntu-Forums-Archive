### Post by mastablasta on 2012-01-20
you really should read what UEFI is first.
 
it's a replacement for BIOS and then...:
> 
[Linux]("http://en.wikipedia.org/wiki/Linux_kernel") has been able to use EFI at boot time since early 2000,[SIZE=2][*[citation needed]("http://en.wikipedia.org/wiki/Wikipedia:Citation_needed")*][/SIZE] using the [elilo]("http://en.wikipedia.org/wiki/Elilo") EFI boot loader or, more recently, EFI versions of [GRUB]("http://en.wikipedia.org/wiki/GRUB")
 
 
and this is probably your concern:
>  
Microsoft will require computers with the "Designed for Windows 8" logo to use UEFI with [secure boot]("http://en.wikipedia.org/wiki/Secure_boot") (which will only allow signed software to run on the device) enabled by default.[[SIZE=2][30][/SIZE]]("http://en.wikipedia.org/wiki/Unified_Extensible_Firmware_Interface#cite_note-Windows8UEFISecureBoot-29")[[SIZE=2][31][/SIZE]]("http://en.wikipedia.org/wiki/Unified_Extensible_Firmware_Interface#cite_note-Windows8UEFIClarifies-30") [Red Hat]("http://en.wikipedia.org/wiki/Red_Hat") developer Matthew Garrett raised concerns over the requirement for secure booting to be enabled by default and Microsoft responded by saying that there was no mandate from Microsoft that prevents secure booting from being disabled in firmware or that keys could not be updated and managed.[[SIZE=2][30][/SIZE]]("http://en.wikipedia.org/wiki/Unified_Extensible_Firmware_Interface#cite_note-Windows8UEFISecureBoot-29")[[SIZE=2][31][/SIZE]]("http://en.wikipedia.org/wiki/Unified_Extensible_Firmware_Interface#cite_note-Windows8UEFIClarifies-30")

 
sources are from wiki
