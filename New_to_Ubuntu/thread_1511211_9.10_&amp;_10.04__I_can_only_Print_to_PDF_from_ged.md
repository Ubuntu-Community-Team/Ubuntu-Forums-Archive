### Post by Zeike on 2010-06-16
If I understand your question correctly, you wish to be able to print to pdf from more applications?

If you install the package 'cups-pdf' you will be able to print to pdf from openoffice, and pretty much any other program.

You can do this by running 'sudo apt-get install cups-pdf' in a terminal, or finding cups-pdf in synaptic package manager.

Andavane, not sure if I can help. I don't really understand what you need? the link above is date 2007, pretty outdated.

You can already print to PDF any document (see you first print-screen) since ghostscript is installed by default. You don't need anything.

cups-pdf creates a designated printer to PDF (system>printing), if I recall.

cups-pdf in the repos (9.04) had a bug. It generated an error message when you were trying to print. Reason: cups-pdf did not created the needed directory in /home. To fix it:

```
mkdir ~/PDF
```Then change the permissions

```

sudo chmod u+x,a+x,+s /usr/lib/cups/backend/cups-pdf
```You will find all you printed (to PDF) document in your PDF directory.

Is this what you are looking for?

side note: I don't use cups-pdf anymore. if you find a PDF folder in your /home dir, the bug must have been fixed on 9.10/10.04

### Post by nomnex on 2010-06-17
Well, I don't use cups-pdf anymore on 9.10 - 10.04.

Just to make sure, you could try the following steps:

```
# apt-get purge cups-pdf

# apt-get install cups-pdf

# chmod u+x,a+x,+s /usr/lib/cups/backend/cups-pdf

====OR====

# chmod 700 /usr/lib/cups/backend/cups-pdf
```

EDIT: my post address a bug in the "cups-pdf" version in the repository. It may not address your issue.
