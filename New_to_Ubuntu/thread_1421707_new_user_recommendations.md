Linux is not a drop in replacement for Windows and if your intention is to run windows applications I suggest you stay with Windows.

As you stated (bolded by me), use the linux equivalent.

[http://wiki.linuxquestions.org/wiki/Linux_software_equivalent_to_Windows_software](http://wiki.linuxquestions.org/wiki/Linux_software_equivalent_to_Windows_software)

Examples are 

OpenOffice

Web design is more difficult, personally I use bluefish and view the pages with a browser (browsers) rather then a WYSIWYG editor.

WYSIWYG is fine for learning the basics, but in the long run, I highly advise you learn to write the code by hand (bluefish automates the process). The resulting code is much cleaner.

Another option is to dual boot as you transition.

The third option is to use Virtualization (such as Virtualbox or VMWare) or Wine. This can help for the one or two apps you can not live without, but it does not make sense to boot Linux to run Virtualbox to run all your apps in Windows.

### Post by bodhi.zazen on 2010-03-04
Installing Windows in VirtualBox will take as long as installing Windows to a hard drive.

If you use VBox you can look at 

[https://help.ubuntu.com/community/SeamlessVirtualization](https://help.ubuntu.com/community/SeamlessVirtualization)

Wine is hit and miss and as I said I would NOT consider wine a drop in replacement for windows.

You can look on the Wine web site for information on running specific applications:

[http://appdb.winehq.org/](http://appdb.winehq.org/)

Look up your application and see what success (or lack of) other have had.

Take care though, setting up wine for complex applications is well complex and requires thorough knowledge of both windows and Linux.

IMO VirtualBox is much easier then wine.

I suggest you install Windows and dual boot.
