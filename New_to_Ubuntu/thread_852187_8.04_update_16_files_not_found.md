### Post by avtolle on 2008-07-07
What I would suggest is to edit the /etc/apt/sources.list to delete all references to Breezy, and I'd take a look at whether there's a newer repository for Citadel as well (Google can be your friend). To edit the file, I would first make a backup just in case```
cp /etc/apt/sources.list /etc/apt/sources.list.bak
```then open the file for editing. I use nano, so the command would be```
sudo nano /etc/apt/sources.list
```but you may use any editor with which you feel comfortable (NB, if using gedit, the command should be gksu gedit /etc/apt/sources.list). Then, as I said, you may delete the Breezy sources or comment them out by putting a # before each line referring to breezy. I would do the same with the debian line. Then, again in terminal```
sudo aptitude update
```to reload the sources, which hopefully will eliminate the error messages you are receiving.
