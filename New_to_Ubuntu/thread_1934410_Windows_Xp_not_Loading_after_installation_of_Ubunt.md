### Post by oldfred on 2012-03-02
Added code tags to make grub.cfg easier to read.

It is good that it shows XP as a choice. But then it probably is just a Windows issue. Often if chkdsk needed grub will not even add an entry as it cannot mount the NTFS partition. It must see the boot files or it would not add the boot entry. 

Have you run chkdsk from an XP CD? Are you getting any Windows error messages. Once grub chainloads to the Windows boot loader Windows takes over and grub/Linux have no effect on the Windows.

If chkdsk does not solve it, post link to boot info script from Boot Repair, but if the Windows boot files are there (grub sees them) or boot.ini is not correct, boot script may not show much.

Boot Repair:
[https://help.ubuntu.com/community/Boot-Repair](https://help.ubuntu.com/community/Boot-Repair)
You can repair many boot issues with this or post the link to a run of boot info script so we can see your exact configuration.
