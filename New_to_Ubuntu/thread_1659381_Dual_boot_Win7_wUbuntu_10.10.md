### Post by Bucky Ball on 2011-01-04
The unknown partition is a worry.

I would try 10.04 when you get this issue sorted. You might get a smoother ride. ;)

You should really have three partitions as basic for Ubuntu:

/ = root partition, where the OS goes.
/home = where your stuff goes.
/swap = swap file, like Windows pagefile (2Gb will do it).

You can do this by entering manual partitioning when you get to the partitioning section in the installation of Ubuntu. The partitions I have suggested are set as defaults in there. You can add others with whatever names you like. 

You will need to make an extended partition as you cannot put more than four partitions on the one drive (but you can put 99 virtual partitions in an extended partition!). You will see your Windows partitions (both NTFS) so just leave them alone and install straight over the existing 10.10 install.

After the installation, you should be booted into a screen with selections for Ubuntu and Windows (the Ubuntu install will pick up your other operating system).

You don't need to do anything tricky. It should 'just work'. A little off topic but save for later ...

### Post by oldfred on 2011-01-06
Windows likes to use all four primary partitions. You will have to delete one to create the extened and inside the extended you can create many more logical partitions. Ubuntu works just fine from logical partitions, windows needs a primary to boot from. 

Windows 7 now creates a hidden 100MB boot/recovery partition, and then the main install. Vendors do not give install disks, but use a partition and make your write your own disk, which is not an install but an image to restore. Once you have a good restore DVD you may not need the restore partition. Also the vendor utilities partition is a small partition which you can easily back up and if you want put back into a logical partition. 

Once you have your windows configured the way you want you may want to make your own image to restore. 
Backup windows before install - post by Mark Phelps
[http://ubuntuforums.org/showthread.php?t=1626990](http://ubuntuforums.org/showthread.php?t=1626990)
[http://www.macrium.com/reflectfree.asp](http://www.macrium.com/reflectfree.asp)

Use windows to shrink the windows main install partition, but do not use it to create any new partitions. Reboot windows several times as it will want to run chkdsk or make other fixes to be sure of its new size.

Then you can install or use gparted to create partitions and use manual install choosing the partitions you create.
