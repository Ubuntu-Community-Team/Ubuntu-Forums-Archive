### Post by endotherm on 2010-11-19
if you hit Alt+Space, you shoudl get a window control menu, including among other things, "Move". if you can;t click on move, just type 'M' to select it. if you can't click the title bar, just hold ctrl, and use your arrow keys to move the window around.

BTW, this workaround is identical in every way on windows, down to the exact key combos.

I'm glad linux is keeping you seeking and learning. that is my favorite aspect to it.

btw, if the window control menu does not come up, try just hitting 'C' or enter. this will likely trigger your continue button.

### Post by walt.smith1960 on 2010-11-19
I'm not sure if this applies to installation screens but I had a similar problem with some web sites in firefox with one desktop--could not see the bottom of the web page where the buttons were.  The fix was to adjust the amount of memory available to the onboard video.  The default was 128 Mb. which wasn't enough.  I increased that to 256 Mb. in BIOS and all the pages now display properly, not cut off.  Just something to consider.  It's pretty much impossible to use a GUI when you can't see the part of the page where the choices are.
