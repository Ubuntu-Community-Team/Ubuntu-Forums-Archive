### Post by Grenage on 2013-02-05
Out of curiosity, where did the *horizsync* and *vertrefresh* values come from; were they already there?

The metamode resolution is 1024x768, so I'm not sure if the nvidia driver will prioritise it over your default.  Try a much leaner config; the system can pick up the rest:

```

Section "Monitor"
    Identifier     "Monitor0"
    VendorName     "Unknown"
    ModelName      "CRT-0"
    HorizSync       28.0 - 55.0
    VertRefresh     43.0 - 72.0
Modeline "1280x1024_60.00"  109.00  1280 1368 1496 1712  1024 1027 1034 1063 -hsync +vsync
EndSection

Section "Device"
    Identifier     "Device0"
    Driver         "nvidia"
    VendorName     "NVIDIA Corporation"
    BoardName      "GeForce GTX 550 Ti"
EndSection

Section "Screen"
    Identifier     "Screen0"
    Device         "Device0"
    Monitor        "Monitor0"
    DefaultDepth    24
    SubSection     "Display"
        Depth       24
    		Modes "1280x1024"
    EndSubSection
EndSection
```

### Post by Grenage on 2013-02-06
Hi there,

Using nvidia-settings to save the config will overwrite your xorg, so it's best to ignore that tool for now.  You logged out, which I presume means that you didn't restart?

I should have mentioned that the config won't take effect until X has been restarted.  You can do that manually, but it's generally just easier to restart the computer.  Overwrite the xorg.conf that nvidia-settings regenerated, with the minimal version I posted, then reboot the computer and see how it looks.
