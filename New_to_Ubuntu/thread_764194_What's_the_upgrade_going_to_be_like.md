### Post by Paqman on 2008-04-23
> **computermesh said:**
> Will there be any difference between installing the 8.04 that's out right now and the 8.04 that's official release date is tomorrow?

Not really. Maybe a couple of updates, but nothing serious.

If you want to do a fresh install, download it now. The servers will be jammed after the final version comes out. Torrenting avoids that problem altogether, of course.

If you decide to upgrade instead of doing a reinstall, then disable any extraneous repos you've added first. If you don't have you system and data separated then i'd recommend backing up your data, too. Better safe than sorry.

Basically, some people like to do the in-place upgrade, others reinstall. It's really just a matter of preference.

check this thread out for update information: [http://ubuntuforums.org/showthread.php?t=763394](http://ubuntuforums.org/showthread.php?t=763394)

> #  **I already have a working Hardy test installation. Do I need to install from scratch, or can I upgrade when the final release is out? Is there any difference between a clean install and an upgrade?**

You do not need to download the disk images in order to have the final release. The update manager will alert you of the existence of a new version when the release is out, and once you've performed a distribution upgrade, you'll have an identical set of packages to those on the CD.

However, it can be a good idea to do a clean install if you're affected by certain corner cases where you may not be able to access specific features or fixes without doing a clean install, or if you've extensively modified or partially broken your installation, and/or have installed software not provided by the official Ubuntu repositories without keeping track of the changes made to your system.

You can also use rsync to update your already downloaded disk image instead of re-downloading the entire image.
