### Post by tuxHulk on 2015-05-02
I experienced this issue when installing from a USB stick. It appeared stuck in several places in the installation. The first place was at the purplish screen that says Ubuntu with the 4 dots... mine took forever but it was the stick slowing it down. I believe it was stuck for 5 minutes or more. It happened again after selecting the partition and got "stuck" again. Not sure how long you waited, but maybe give it a few more minutes, depending on the type of install you are doing. Like robert159 mentioned it could even be a write issue with a USB when creating it.

### Post by jimmy-frydkaer on 2015-05-03
Have you made sure that the computer has recognized the hard drive in BIOS? Your BIOS is set up with the parameters of your dead drive. Opening the BIOS to check usually is enough. Shouldn't be a problem these days that a BIOS does not find a new hard drive automatically, anyway, it's worth a try.

### Post by tgalati4 on 2015-05-03
There is typically a "Check Media for Errors" selection in the initial bootup screen of the LiveDisk or LiveUSB.  Try running that and see if it passes.  It takes some time to perform this check, but if the disk fails, then don't use it for installation.
