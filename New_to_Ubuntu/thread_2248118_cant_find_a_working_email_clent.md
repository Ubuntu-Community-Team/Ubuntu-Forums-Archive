### Post by buzzingrobot on 2014-10-13
The OP might look at Opera Mail (it's contained within the Opera browser released for Linux) as another alternative. I don't know, though, if it meets the requirements.

Trotjita ([http://trojita.flaska.net/](http://trojita.flaska.net/)) is another mail client that gets little attention.

In my experience, Evolution does behave as described.

The IMAP implementation used by any given provider is also going to play a role here.

The geeky solution to this requirement?  Use a tool like OfflineIMAP via a cron job to pull mail from all servers into a local MailDir, run a local mail server (Dovecot is popular for this), and point a mail client at it. (Evolution works fine in that scenario.) If the client does not handle notifications correctly, various applets are available that can, or one can be scripted using the inotify capability. 

Any number of howto's have been published about setting up this OfflineIMAP/Dovecot mail chain, usually with Mutt as the mail client.  (Here's a similar POP-based approach from 2010:  [http://lorenzogil.com/blog/2010/08/21/managing-email-the-unix-way/](http://lorenzogil.com/blog/2010/08/21/managing-email-the-unix-way/); I used it for sometime.)

Obviously, that's not an ideal solution, but it is one solution to the dearth of mail client development in Linux.
