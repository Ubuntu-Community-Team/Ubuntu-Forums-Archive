### Post by ian-weisser on 2016-11-19
> **juntjoo2 said:**
> so when I say I'm new I mean stupid.

No.
Not stupid at all, quite the opposite.
You lacks skills...but you are wise enough to know that you lack skills, and are willing to learn.
That makes you very smart

All Linux concepts are easily teachable and learnable, in small steps.
For example, your problem with installing OneDrive isn't about mysterious shell incantations. It's about just a few easy skills to learn.

Most software is not completely self-contained. It *depends* upon other software or libraries...for example to input from keyboard or output to screen or generate strings.
Understanding that concept of dependencies is a skill.
Using the package manager to ensure you have installed those dependencies is another skill.
Understanding makefiles is another skill.
Using makefiles to compile and install the OneDrive client code into an executable application is another skill.
Upgrading the application and dependencies to install security patches is another skill.
Uninstalling the application and dependencies is another skill.

None of these skills are hard. Just take them one bite at a time.

### Post by Mark Phelps on 2016-11-22
Being a Genius gives the ability to learn new things often faster and easier than other folks -- but you still have to start from scratch, like everyone else.

If you were approaching Mathematics for the first time, you wouldn't jump right into Differential Equations, before you even mastered Arithmetic, now would you?

Same is true about activities in the Linux world.  You start out installing applications using tools and GUIs -- because they will handle the dependencies for you.

Then, later, when you become more experienced at handling dependencies yourself, you can move on to more difficult means of installing apps -- by compiling from source files downloaded from the Internet.
