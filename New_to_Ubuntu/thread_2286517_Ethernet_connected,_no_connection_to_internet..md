### Post by grahammechanical on 2015-07-14
@wisconman3

It is not customary to hijack someone else's question. On this forum we do not mind different people asking about similar problems but each one is their own thread. We do hope that people read existing threads and learn from them. But it is very difficult to support two people in the same thread. It gets very confusing.

I will say this for the benefit of anyone viewing this question:

If the printout from lshw -C network under Capabilities shows "broadcast=yes" and then shows "driver=" and lists a driver name, then a driver for the network adapter has been installed and is working. This applies to both wired and wireless network adapters. So, it is not a problem with drivers.

If the printout from ifconfig -a shows "UP BROADCAST RUNNING MULTICAST" then the network adapter is switched on & is working as far as Ubuntu is concerned. This applies to both Ethernet and WiFi adapters. So, the problem is not caused by the network adapter being off or networking in Ubuntu being disabled.

The problem lies between the machine and the router or between the router and the Internet Service Provider.

RX packets = received packets. TX packets = packets transmitted. RX packets = 0 means nothing is being received by the Ubuntu Network manager. TX packets = 0 means that Ubuntu is transmitting but the packets are not being received and that is most likey because there is not a connection. TX packets dropped = xxx shows that Ubuntu has tried to make a connection but has failed.

There has to be a backward and forward transmission and receiving between the network adapter in the computer and the network adapter in the router and then on to the ISP. The two devices need to have a "conversation." 

Regards.
