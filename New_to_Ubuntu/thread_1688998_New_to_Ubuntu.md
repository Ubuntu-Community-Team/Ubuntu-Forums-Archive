Some honest facts. (imho)

You will instead start to realise there are ways round things, where before it seemed impossible instead it will now take a bit of time and research. What i think is the major cool-*** thing about ubuntu is the way you start to tweak your system to the way you want it to be. After learning to 'tweak' things, i collaterally started looking into small startup scripts e.t.c, super simple stuff for example running macchanger/ifconfig everytime the system starts by editing rc.local file, so that i have a brand new funky flavour of mac address everytime i boot-up. Learning the switches to a program is a gentle way of entering this scripting side of things. 
Learning to use terminal and a lot of its functions is to me a large step i feel i've taken. The fact that you can bring up a manual for most programs is amazingly helpful, and the way the programmers usually leave comments about functions or parts of code is more than helpful also. (Kudos to you programmer dudes!)

The next step for me im going to attempt i think will be learning to program, im looking into perl to start, it seems more accessible than a few of the other languages i've had a brief overview of. 

It is quite amazing the way, since simply switching from being a lazy bast (using certain o/s i won't mention here but may of accidentally on purposely mentioned allready ;), to a 'learn as you go' approach (ubuntu) you start to accumulate a reasonable amount of knowledge. 

Anyways as i aforementioned i think its the first few weeks where i personally was soooooooo tempted to go back to the old way, this was due to simple things not working that really bothered me, but i was convinced to stick it out by the dude who showed me linux and he was very helpful. It was cool to have a super knowledgeable dude nearby in case something didn't work e.t.c. 

Stick it out dude, you'll be glad you did man.

Partitioning for Linux is quite an artform really, but if you've not added libraries of data to your installation, it might be simplest to just re install onto the portable 500Gb HDD from the beginning.  It only takes 5 or ten minutes if you run the installation offline anywaty.

You can run the root partition, **/** , in 4GB, although 8GB is safer if you want to install oodles of software packages, and 16 isn't taking over your 500GB drive all that much.  That's all you need for your root partition, as long as you create a separate **/home** partition where all the data will colonise the remaining real-estate, as data likes to do. Then there's your SWAP partition, and you might want to go further, but those three will see you right for most things, and you will be able to reinstall new versions of Linux as you choose, and still retain your data in the home partition.

Easiest way if you're starting the install again, is when you get to the part of the install process that involves disk partitioning, select MANUAL partitiioning, then create a primary partition for your root drive - 8-16Gb is enough.  Use EXT4 I reckon, although some prefer EXT3.  Tick the format box and set the mount point to / (root).  You might even want to allocate the same to a second primary partition so that you can have either a test installation or a backup of the same thing.  

Then, at the other end of your HDD, create a SWAP partition of roughly double the RAM in your system, probably 2-4 GB or maybe more, but you don't need great plains of open frontiers for swap.

In the middle, you do need lots of room.  Create an extended partition with every last remaining gigabyte, and then inside that main 460GB partition, you can create a nice little 16-32-64-126-256GB place to call /home.  A logical partition within the extended partition.  Like drive D: used to be in MSDOS.  The first time you create your /home partition, you'll need to format it, but anytime you install a new system in future, just set it to /home and DON'T format it, and the data will be there just as it was when you login to the new installation.

You still have plenty of room remaining on that large external partition to add more linux or windows partitions as you need, but that can be done later because all the installation needs are the root / and SWAP partitions, and the /home partition is an extra that will see that you can retain all your data (even bookmarks) if you ever need to reinstall Linux again, which you most certainly will from time to time.

Also checkout Parted-Magic at [http://partedmagic.com](http://partedmagic.com) if you can spare the time to download a 150Mb .iso file that is specially designed for this sort of system/hdd config stuff.  

Sorry if I've been a bit confusing trying to write 30 years partitioning advice into one little teeny-weeny post. Let me know if I can clarify.
