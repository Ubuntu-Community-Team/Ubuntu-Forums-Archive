Same as the difference between most distros: package management.

There are two main types of packages in Linuxland: Debian style .debs, and Red Hat style .rpms. There are also some odd fish like Gentoo, but Debian/Red Hat accounts for most of what's out there.

### Post by djbon2112 on 2009-10-15
> **Niko Johnson said:**
> so i hear all this talk about Deabian and non deabian systems... i was wondering if anyone has a list of witch versions are debian and witch version are not. and what is the difference anyway?

Well, Debian systems are those based of the Debian distro ([http://www.debian.org](http://www.debian.org)) and include any distros that use .deb packages and aptitude (a.k.a apt-get) as their package manager. Some examples are Ubuntu, Mint (based off Ubuntu) and Knoppix.

Non-Debian systems are anything else, but the biggest category is Red Hat based distros. These are any distros that use .rpm packages. Examples include Red Hat Enterprise Linux (of course), CentOS (rebranded Red Hat) and Fedora (community Red Hat).

The rest are usually distro-specific. Gentoo has portage, Arch has pacman, some distros don't even have a package manager (you have to compile from source or install compiled binaries).
