### Post by 3rdalbum on 2010-03-20
The paid Linux distributions are usually worse than the free ones. The paid ones prey on people who, like you, believe that if they pay money they're getting something better.

Apart from distributions like Red Hat Enterprise Linux and SUSE Linux Enterprise Desktop - neither of which are home-user distros, and both of which are probably going to be behind-the-times.

I have no idea about dictation software (I assume you mean speech-to-text?) but I believe there is an Accessibility forum here; the people who look at that forum are more likely to give you an accurate answer. As for text-to-speech (screen reading), Gnome and Ubuntu have this software built-in.

Ubuntu comes in a 64-bit flavour, which is very easy to run because most open-source Linux software can be compiled for eleven different CPU architectures, including 64-bit x86 (named AMD64, also works on Intel processors).

You will only interact with the machine as an end-user, unless you decide to learn how to program. Ubuntu is **very** different to Windows, though; so be prepared to learn. You might never need to touch a command-line if you have compatible hardware and you learn the Ubuntu way of doing things. If you *do* need to touch a command-line; well, you sometimes need to on Windows as well.

### Post by Mark Phelps on 2010-03-21
First thing I would suggest is that you download and burn 64-bit LiveCDs of the Linux distros you want to use and try each of them.  You might be surprised in the great variation on how well each distro detects your hardware and installs the proper drivers.

Also, as I'm sure you're aware, there is not one set of repositories (repos) for ALL Linux distros.  Sure, there are debs and rpms, but they're not universally convertible to each other.  So, when you settle on a distro that works well, check out if its repos have the apps you need.

And finally, it is virtually impossible to avoid using the command line at some point during your experience with Linux.  While more GUI-based apps are coming out all the time, there are still instances (driver installs, customization, boot repairs) where the command line is the only way to do the work.  So, if you're basically opposed to learning command line skills, Linux is not for you.
