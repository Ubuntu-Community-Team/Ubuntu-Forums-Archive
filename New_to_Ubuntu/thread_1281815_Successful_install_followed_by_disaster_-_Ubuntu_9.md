### Post by carusoswi on 2009-10-03
So, I have a Gateway desktop on which we have been running XP Home and Ubuntu 8.04 for a couple of years.  I decided to upgrade it to 9.04 this morning.  Went into the partitioner, chose to delete all Ubuntu partitions and resize them, then, installed 9.04 without a hitch.

Deleted Gimp, downloaded Gimp 2.6.7 and installed it, no problem.  Navigated to the XP partition and retrieved a photo that I had put there prior to the Ubuntu upgrade which we use as a background.  Actually, I navigated to that drive from the change background dialog, so I only linked to the photo and did not actually move it, although I could have done so.

Downloaded/installed Crossover 7, installed MS Office 2006, not problem.  That's when I noticed that I had lost the background photo that I had set for the desktop.

Tried to navigate back to that XP partition, and it would not display its contents.  Ubuntu could see its size, but could not see any of its contents.  Tried booting into XP, and the partition can no longer be accessed.

It so happens that we long ago pulled off any important data from that partition, so I'm just reinstalling Windows and will follow up with another fresh installation of Ubuntu and call it a day.

But, I remain very curious as to what went wrong.  I'd be quite upset if that disc contained data that was not backed up somewhere, or if, as happened last week, I was working with someone else's computer and something like this happened.

(Last week, I only wanted to install components of Ubuntu Studio via Synaptic on my son's laptop, and, while we didn't lose his Vista install, we lost the ability to view his Ubuntu installation.  I had to boot with a live cd, create a directory on one of his media locations that was visible on the desk top, boot into and drop to the root shell (whatever that is, but it is just a black command line type screen), then, use the command line to copy his data files to that newly created directory, then, boot back into the live cd desktop from where I could browse to that directory, select the data, and copy it onto an external drive.

We then had to reformat his Ubuntu partitions and reinstall Ubuntu from scratch.

The problem today is equally perplexing.  One minute, everything is working fine, the next, I cannot get to that XP drive.

I wonder if anyone else has had similar experiences.

It's so strange because everything worked fine during my first couple of boot ups into Ubuntu and into XP.  Doesn't make sense to me.
