### Post by Kulgan on 2008-06-01
That's like asking what the best graphical environment is, what the best os is, or what the best browser is. 

But I use Transmission (the standard ubuntu one, now), bittornado and rtorrent. 
rtorrent is probably the most powerful of them. So many options, and it runs from the terminal, which is great for sshing into a server and keeping it running through screen. 

Or using the directory watching functionality, letting you upload torrent files to a shared directory and it will start them automatically.. For an intro to rtorrent, see: [http://kmandla.wordpress.com/2007/05/02/howto-use-rtorrent-like-a-pro](http://kmandla.wordpress.com/2007/05/02/howto-use-rtorrent-like-a-pro)

### Post by ugm6hr on 2008-10-11
On Ubuntu (Gnome), I have used:
KTorrent - very good, but KDE app
Transmission - a bit too basic for my tastes
Deluge - was very good, and my preferred for a good few months, but the latest rewrite (v1.0 series) seems a step backwards (and very buggy)

So, as of this week, I've installed wine (for the first time since I started with Ubuntu) to use utorrent.  It is really easy - no complex configuration required ([http://www.jumbabox.com/2008/06/installing-the-utorrent-and-webui-client-on-ubuntu-hardy/](http://www.jumbabox.com/2008/06/installing-the-utorrent-and-webui-client-on-ubuntu-hardy/)).

The system notification icon works fine, as does the scheduler and web interface.  Apart from the "Windows" (i.e. non-GTK) interface, it is perfect.

I am loathed to recommend a Windows solution, but I think it is a better fit for Gnome than KTorrent, and Deluge's huge initial promise has not (yet) been met.
