### Post by Mark Phelps on 2013-10-31
You've chosen a very difficult path to take ...

There are multiple "generations" of programming languages -- from machine (0's and 1's) to fifth generation (english-like primitives) and you have to decide WHICH one to do.  It's like saying I want to learn a language -- you have to read up on them and decide which one.

Programming ROMS is very hard to do -- not something to start with.  It's like saying I want to learn mathematics, so I'm going to jump straight from addition into differential equations -- without doing algebra or calculus in between.

You could start by doing Google searches on computer programming -- to get more information.

Run a Monte Carlo simulation of inbreeding of genetic properties in arbitrary family trees using [Mendelian inheritance]("http://en.wikipedia.org/wiki/Mendelian_inheritance") (try the [Ptolemies]("http://en.wikipedia.org/wiki/Ptolemaic_dynasty#Simplified_Ptolemaic_family_tree")). Gravitational N-body simulations with a few dozen objects can be done on any computer. Complexity of that one increases quadratically. I know a guy who did it with 100,000 objects. He build his own custom computer with all processing power delivered by 25 nvidia GPUs.

### Post by Rob Sayer on 2013-11-03
I agree that going from zero to firmware development is one hell of a leap.  I'm not suggesting that you shouldn't try programming but what you're talking about is like going from playing "Mary Had A Little Lamb" to symphonies.

Firmware programming is at or very close to machine level.  There are very few layers of abstraction between you and the machine.  Most people who don't know how to program would try first to do something like mobile apps in a high level language.

[http://forum.xda-developers.com/nexus-one/development](http://forum.xda-developers.com/nexus-one/development)

[https://developers.google.com/android/nexus/images](https://developers.google.com/android/nexus/images)

I don't know whaat you mean by "just a normal computer" but for something like an Android ROM you're going to have to have enough resources to run a sandbox.

In any case when you're learning progamming your hardware is not going to be the bottleneck.  You are.
