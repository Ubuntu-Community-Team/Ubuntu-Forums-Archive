### Post by Javelin Dan on 2011-10-11
[FONT=Arial]I am now trying to install a Lexmark Interpret S405 printer on my brother’s Mac G4 Quicksilver (powerpc) running Ubuntu 10.10. I know I could post this question on the “Apple Users” thread, but I believe it is more of a Ubuntu/printer problem than a Mac problem. From the Lexmark website, I have downloaded what I think is the correct driver (I don’t have the complete name but it contains the words “Ubuntu/Debian, 10.1, 32 bit, i386, Legacy). I have successfully downloaded the driver, clicked on extract, and both the tarball and the executable text file show up in my Downloads file. I double click on the text file and the box appears that has the option to “run”. I click on run and absolutely nothing happens! There is also an option to “run from command line”. If I click that, a lot happens very quickly (about 3 seconds) but it starts out with “all is well” and ends up with “no automatic installer”. If I go into my “printers” application, I see the Lexmark right beside another printer that was previously installed. I click on it and it shows the name and model of printer with the option to clean the heads, print a test page, etc .It also shows the status as idle. The tutorial that Lexmark had online shows a demo with Ubuntu 8 point something and the only difference was that the driver downloaded to the desktop rather than the downloads file. Could this be a problem? Everything seems to be in place except that I can’t get the final set-up wizard to run. It looks like somehow I need to get Ubuntu and the printer to “shake hands” and start communicating. I’ll tell you in advance, I’m OK with one or two simple commands in the terminal, but I get lost in long strings of code, so be gentle with me. Any help would be greatly appreciated – thanks! ### Post by Javelin Dan on 2011-10-17
[FONT=Arial]This thread is not so much solved as resolved, as I never did actually get the printer to work. But after many long hours on the phone with the Lexmark rep (who did try to help me), it was determined that there is no driver available that will make this printer work with powerpc. The happy ending is that we purchased an HP “Photo Smart” D-110 printer-scanner-copier that simply plugged-n-played. The lesson I took from all this is that if a printer is going to work with powerpc, it will probably say so on the box – the HP D-110 did. I hope I can save someone else the aggravation by posting this info. It’s soo much nicer to buy a piece of equipment that “just works”.[/FONT]
