try typing in ```
 sudo -i
``` See if you can get the **$** prompt to change to a **#** prompt.

you are then in 'sudo' mode, so will not need to type sudo before the command. **_Please be careful !!_**  Type in ```
exit
``` when you're done to return to the normal user prompt.

That'll check you are using the correct password and you are configured up in the sudoers group.

Whilst you are at the **#** prompt you can always try ```

apt-get install boinc-manager
``` instead of aptitude. 

Regards,

Phill.

### Post by gregb49 on 2010-02-20
> **theozzlives said:**
> Try:
```
chmod +x binstall.sh
./binstall.sh
```If, like me, you use ubuntu 8.04 LTS (gOS) then apt-get will tell you that you already have the latest version, 5.10.x. I have followed the advice above and have a BOINC folder. I then started Nautilus in root using the terminal and typed ```
sudo nautilus
``` (I'm sure that there is a better way). Then I renamed the BOINC and BOINC-client folders in the /etc folder to something else and moved the new BOINC folder to /etc. 

I noticed that running ./binstall.sh, from the BOINC folder, gave an error, in that it couldn't CD to BOINC, so I copied binstall.sh to the /etc folder. Using nautilus, I right clicked on binstall.sh and opened it, then I looked in the BOINC folder, right clicked on run_manager and low and behold, I'm now running BOINC 6.10.17. I note that if I run BOINC from the menu system it reverts to 5.10.x

I'm sure there must be a better way of doing this and I've probably broken some golden rules by working through root.

EDIT. After restarting the computer, I couldn't initiate BOINC, so am now back to v5.10.45, reinstalled through Synaptic.
