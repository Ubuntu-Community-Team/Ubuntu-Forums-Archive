### Post by oldfred on 2016-02-08
I have seen others with same issue during installs.

And I have had similar issue on desktop with multiple drives. 
I plug in flash drive and it is sdf, but on reboot flash drive becomes sdb and every other drive changes one letter.
Then with new UEFI system DVD was seen by UEFI as hd1, so sdb became hd2. 

In both cases it as SATA port order. I had skipped a SATA port on motherboard. And BIOS or UEFI moved flash drive into location of missing SATA port.

Not sure with laptop if it has more than one SATA port, but when built maybe it was plugged into the second port as they assumed it did not matter?

Grub used to have a device.map which was drive order. It is not often used, but if found it will use it.
So perhaps that would solve your issue.

[https://www.gnu.org/software/grub/manual/html_node/Device-map.html](https://www.gnu.org/software/grub/manual/html_node/Device-map.html)
sudo grub-mkdevicemap

Check what it has created:
cat /boot/grub/device.map

That is the order the kernel sees (enumerates) the devices.  In later SATA setups this is due to the header order.  The kernel uses the device name of sda for the first block device it sees.  This should not have any effect if the fstab file is configured correctly.
