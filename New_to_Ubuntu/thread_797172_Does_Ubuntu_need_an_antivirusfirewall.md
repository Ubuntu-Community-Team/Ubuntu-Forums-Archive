### Post by ace007 on 2008-05-17
The short answer is not really.  The reason you hear so much about Linux's lack of viruses is because of the differences between it and Windows.  Windows is so popular that almost all viruses/spyware are made for it and thus do not run on Linux.  For this reason you are safe from all Windows specific viruses.  I would assume something like 99% or greater of viruses/spyware that are distributed are Windows based.

This does not mean Linux cannot be hacked or get a virus.  I just think viruses for Linux are extremely rare.  You're safe as long as you don't go looking for trouble.  (ie: downloading and running files you know are shady)

### Post by Xiong Chiamiov on 2008-05-17
Are you behind a router?  If so, there is no need for a firewall, since routers will block incoming connections very well.  The only point then for a firewall is to block outgoing connections from things like trojans and worms.

What ace said about Linux viruses is partially true.  Yes, Linux boxes comprise a very small percentage of the home computer market, but many, many servers run Linux, so it could still be profitable to create some sort of malware for it.  The greater security comes from the way that the OS is structured.  Files do not have executable permission by default, you don't run as root, etc, etc.  Plus, security holes tend to be patched very quickly, since we don't have to wait for a company to accept the fact that there even is a problem, much less get around to fixing it.  And I usually don't install Windows updates automatically anyway; they're just as likely to break my system as to fix it.
The only point of running an anti-virus like avast! is to protect poor Windows machines.  That said, just be safe.  Don't run programs from random people on the net, and don't give root priviledges to something unless there's a good reason.  The user is always the biggest security risk.
