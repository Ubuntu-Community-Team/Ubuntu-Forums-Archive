### Post by dry crust on 2014-11-27
I use it fairly regularly. There is one ... strange quirk with it. It doesn't follow the correct procedure for minimising and maximising ... oh, sorry, I forgot to mention, I'm using Ubuntu Gnome. If you're using Unity ... I'm sorry, I really really did try to use Unity, but it was so ... inconsistent with the way I use a computer that I had to ditch Ubuntu and was forced to use alternatives. How do Unity users handle running concurrently running applications? No ... don't answer that question that since that isn't the question you asked. I like to have mutiple applications running, and therefore minimising and maximising an application are important. I guess, since as far as I can tell you only run application at a time in Unity, then this won't be a problem for you. If you are using a desktop like Gnome Classic or MATE or LXDE, where you can see the concurrently running applications on a bar along the top or bottom of your screen, and do want to move between them, then you will find the incorrectness of minimising - maximising a little bit silly. 
As an example, right now I do have Google Earth and Firefox running, so I'm looking at what I'm writing about. If GE is minimised, and you click on its icon on the application running bar, it will maximise, click again and it minimises, click again ... and no maximise. Wait about 30 seconds, and it will maximise. 
Anyway, apart from that, it works well.

### Post by ericearl on 2014-11-28
I experienced a slight problem with the installation of GE 7.1.2 in Mint 17 (Ubuntu 14.4 tap) which this solution worked. I couldn't display              the pictures when I clicked them. They were blank. GE needs to be installed first. This solution worked found here: 
[https://bkjaya.wordpress.com/2014/04/27/how-to-solve-blank-panoramio-photo-problem-in-google-earth-on-ubuntu-2/](https://bkjaya.wordpress.com/2014/04/27/how-to-solve-blank-panoramio-photo-problem-in-google-earth-on-ubuntu-2/)
Most of the previous solutions had the user installing GE 6 with googleearth-package which didn't have the bug. This not needed now.
Have fun with this as it does work.
