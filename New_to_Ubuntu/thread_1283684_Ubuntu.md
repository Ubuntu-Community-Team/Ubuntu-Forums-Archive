### Post by Bölvaður on 2009-10-05
> **shawn18 said:**
> 
We had the following to research:It's Process Management,Storage Management,I/O System,Distributed System and Protection and Security.

That is way too difficult to answer. It might take you few months to grasp and few more to write down. This is a really big project.

Process Management : look at the kernel's source code. Which kernel you ask? 2.6.28-15-generic is the one Im using on Jaunty. Watch out that the process management changes every few months, so make sure your sources are up to date if you go googling around instead of checking out the source.


Storage Management: What do you mean? There are number of filesystems you can pick from. ext2 ,ext3, ext4, fat16, fat32, hfs, hfs+, jfs, ntfs, reiser4, reiserfs, ufs, xfs and the swap partition has it's own filesystem doesnt it?
Not all of these can be used to boot from, and not all should be used on main partitions such as fat, ntfs and xfs, just to name random ones. The best but quickest way to get results is wikipedia and google. This is a very big subject and I Cannot see how you can ever scope it all unless if you just use wikipedia and compare each of them to each other.
ext4 is the most popular now.


I/O System : X server is the most popular for this. but there are many others but x is by far the most popular one.
Probably depends upon what you are doing what you pick.


security : look at iptables, user privileges, SELinux and "security in variety".
But if you need to go into details you must check out what programs are given what privileges GNOME, KDE and the other (countless) major Desktop Environments use for each program and how it can and cannot be abused by a cracker.
And most importantly, how software is installed (software repositories)


is that your master's project?
