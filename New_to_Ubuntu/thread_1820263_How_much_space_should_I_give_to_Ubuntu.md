### Post by aysiu on 2011-08-07
You need 4 GB minimum for a base installation (I think it actually installs more like 3.5 GB).

If you're conservative (don't install a lot of apps), 5 GB is fine.

If you want a little bit of wiggle room (installing apps, downloading files even just for later transfer), you should allocate at least 8 GB.

### Post by quadproc on 2011-08-07
> **user_linux said:**
> Hi,
I have 70.49 GB hard drive space, so how much should I allocate to Ubuntu 10.10? I don't want to allocate a large space, just bare min but enough for upgrade to next version of Ubuntu when it comes.

As others have mentioned, it depends on what kind of applications you will run and on how much data you will store.

For each of my Ubuntu partitions I actually use about 15 GB, but I use a separate /home partition which currently holds about 6 GB of data so my total usage is about 21 GB each.  If you are heavily into things such as video editing then you might need a lot more.

My recommendation. for you to modify to suit your needs, is to use a separate /home partition of 5 GB and to use an initial Ubuntu partition of 20 GB with an additional partition of another 20 GB for future use such as a new release.  You could leave the remainder of the disk unallocated for now; then you could resize the partitions into that if you need more.  And, if you need a different OS to share the system then you could put it in the remainder.

The separate /home partition males life a lot simpler and easier, especially at new release time, because you don't have to move all your /home data each time.
