### Post by lswest on 2009-02-05
It seems you still have ndiswrapper set to autostart in /etc/modules.  I would remove it from there with ```
gkudo gedit /etc/modules
``` then deleting the line and saving.  Could you also post the contents of your /etc/modprobe.d/blacklist file? You can do that with: ```
cat /etc/modprobe.d/blacklist
``` and copying that output back here (use the code tags for easier reading by clicking the "#" button at the top of the edit post box).

Lastly, can you check to see if the Broadcom STA driver is activated in the hardware driver window?

My next reply may be a bit slower coming, I have school and exams in about 18 minutes, but I will post back as soon as possible,
Lswest

*edit* Thought I'd post another idea here before I go.  You should (from the computer before trying to load any new drivers or so) run the commands I listed above again.  I'm not sure if you had tried something else to get it working last time, and so there may have been other modules running in the background that I didn't know about.  The reason why you'd have to do it again is because modprobe only affects the currently running system, rebooting will negate any changes made.  If you get it working, you can add the module you had to remove (with "modprobe -r") to the blacklist file, and the one that got it working ("modprobe") to the /etc/modules file, so that it works on every boot.  After you remove ndiswrapper from the modules, and add wl, check the output of ```
lshw -C Network
``` and see if there is a "driver=" line at the bottom of the output for your Broadcom 4322 card.  It reads driver=wl for my card, which is running the broadcom STA driver, which is what I am trying to get working on your laptop, so it should read the same.  If, however, you get internet again and it's not the wl driver, feel free to use that instead :P  Side note: uninstalling ndisgtk only removing the GTK frontend to ndiswrapper and ndiswrapper-common, those packages will also need to be removed if you want to remove ndiswrapper entirely.

### Post by snowguy on 2009-02-05
IT WORKED. I'm excited to have actually learned something!

All I needed to do was add wl to the /etc/modules files, or to be explicit for anyone out there who may stumble across the same problem (I don't wish it upon anyone though!) 
Alt-F2 then type:
```
gksu /etc/modules
```
then put at the end of the file a new line with the 2 characters:
```
wl
```
then save. restart the computer and presto. 

I think the issue is that when I followed the ndiswrapper instructions it removed wl because it was going to use ndiswrapper instead. then when I tried to undo it, the steps I took didn't re-insert wl into the /etc/modules file.
