### Post by ajgreeny on 2014-01-28
It does not make any difference which flavour of *ubuntu you use as any of the photo management packages can be installed in any of the *ubuntus, so try any of the applications that you think might do what you want and see what you prefer; different users want and need different things from their photo managers, so only you will really know what fits you best.

For my part, in spite of using XFCE and Xubuntu, I like digikam, which pulls in a lot of qt libraries (KDE bits and pieces), and gthumb, the gnome 2 gtk application, which in many ways is very much simpler than digikam but still does just about all that I need.  Your needs may, of course, be a lot different from mine.

I like to manage my photos by placing them in my own subfolders in the Pictures folder in my /home, ie one subfolder for Family, another for Holidays, etc etc.  Other users let their photo management application put all their photos in one huge folder (Pictures or Photos) and manage them with tags and exif data; great if you're always going to use the same photo management application, but it can cause problems if you decide to change application, or use more than one, as not all of them use the same tags so photo sorting in one application may not work if the tags were made by another application.
