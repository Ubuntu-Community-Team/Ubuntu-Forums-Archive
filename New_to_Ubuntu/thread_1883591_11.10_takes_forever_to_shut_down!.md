### Post by nosmadar on 2011-11-19
I would think just 
cat <file>

and then (assuming you're in a terminal window, just press SHIFT+page up to back up one screen at a time. (this only appears to go back 4 screens for me.) I'm hoping that the "something useful" would be near the end of the file. If nothing else, the entries are time stamped in /var/log/syslog 
or there might be a /var/log/syslog.1 
file. So you're looking for a large gap in the time stamps where the system appears to be "stuck" on a particular thing it was doing.

Actually there are a number of files in this directory that represent logs of various parts of the system.  So if I were you, I would exam them and see if I can find something useful.

### Post by froehli on 2011-11-30
Hey, I get that problem too on my not-so-new Laptop, which didn't have that problem before upgrade!

Seems in my case, the culprit is a mounted CIFS share - I do get some error message that the server didn't respond for more than 300 seconds, although it's still running; looks a bit like I the WLAN connection is shut down before properly umounting the share, and then it still waits for the server to respond - without actually having any way of still reaching it.

So, first, the CIFS share unmount on shutdown somehow seems broken at least for my configuration, and second, if you ask me, the 300 second timeout alone, for a LOCAL connection is way too high.

In addition to that, "asking all started applications to terminate" somehow seems to fail every time, and together with the "killing of all remaining apps" that alone already takes approx 10 seconds or so - which is queer, since the machine is running from an SSD, so startup takes about 5 seconds, making startup being faster than shutdown on this machine :D.
