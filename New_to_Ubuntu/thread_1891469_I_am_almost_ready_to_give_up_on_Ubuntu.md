### Post by diablo75 on 2011-12-05
Upgrades from time to time don't always go over well; I feel your frustration.  But having said that, one of the things I've learned to do is install the system with slightly customized partitioning so if an upgrade goes wrong I just install from a fresh ISO for the latest version while leaving the home folder in tact.

When you install Ubuntu with the default wizard it creates two partitions:  the root (aka, the forward slash / ) and the swap partition.  But it is possible to do manual partitioning to create three or more seperate partitions.  I create three.  One for root, one for swap and one for /home.  This way if you decided to install a fresh copy of Ubuntu over a failed upgrade I just reformat the root partition and leave the /home partition in place.  Just do a little reading about manual partitioning or play around in Virtualbox and pretend to install Ubuntu on a fresh drive to see how it's done.

As far as "things being easier", Unity is very different from Gnome and it takes some getting used to.  Typically when I can't find something I just used the Unity dash search menu to find it by typing in the first few letters of the name of the program I'm wanting to launch and there it is, even drag the icon to the dock bar after for quick access later (and you can fill that thing beyond the edges of the screen; it auto-scrolls).  The System menu that I was used to seeing in GNOME is now in the shut down menu and it's got some growing to do but it's on the right track, in my opinion.

I think if you just give it time, get used to it, it'll become easy to use.  If not, you could try installing GNOME 3 on Ubuntu but it's also different from the GNOME we all came to love in Ubuntu before Unity hit the scene.  You're unfortunately faced with a learning curve one way or another.

### Post by beew on 2011-12-06
There are probably two problems. 

One may be it is blotched upgrade. In that case you should reinstall, there is no point trouble shooting a bad upgrade as many things can go wrong.  I always do clean install because "upgrade" is painfully long and the process is very unreliable.

Or, it may be because Ubuntu 11.10 is quite buggy at this point. Now in that case I don't see how trying other *buntus may help other than to avoid a few Unity bugs because they are all the same under the hood. My experience is that even with clean install 11.10 is not ready, too many things broken or don't work fully which has nothing to do with Unity, switching to other *buntus or Mint 12 is not going to help very much. Many people who are here encouraging other to switch because of their dislike of Unity and Gnome shell, which is an entirely different reason.

I think your problem may be a bit of both.

Unless you have a really antique PC or some hardware incompatibility or something weak like a netbook Ubuntu 11.04 and 11.10 are not really that resource demanding, I think many people are exaggerating about the resource usage. But if your pc is really weak say with 1 g or ram then I would suggest Lubuntu. 

Finally which *buntu version? 10.04 is too old, I think at this point a much better choice would be 11.04. It is fast, up to date and stable (as in reliable) with the best software and hardware support. LTS simply means that it has not been updated for almost two years (in case of 10.04) More predictable perhaps, but it doesn't mean better. If something is expectedly broken or consistently performing poorly it is predictable, hence "stable" in Ubuntu (Debian) talk, it doesn't mean more desirable. I think many people confuse "stability" in the Debian sense with reliability. Even 10.10 runs faster, has fewer bugs,  has better hardware and software support and is more efficient in resource usage  than 10.04. Changes sometimes break things, but for the most part developers do work hard to make things better despite the set backs.
