browser to desktop/gnome/session/required_components and set the windowmanager key to metacity

That should make sure metacity is loaded on startup instead of the failing compiz.

### Post by MegaJim on 2009-10-21
Well I guess there are a couple of possibilities:

1. compiz is being set as your window manager by some startup script or process, what do you get from

```

cat .gconf/desktop/gnome/session/required_components/%gconf.xml

```

2. On the other hand, it could be a configuration problem with xorg or a bad driver

you can try replacing your xorg.conf with the ubuntu default

```

cd /etc/X11
sudo cp xorg.conf xorg.conf.bak
sudo gedit xorg.conf

```

Then paste in the default setup

```

Section "Device"
	Identifier	"Configured Video Device"
EndSection

Section "Monitor"
	Identifier	"Configured Monitor"
EndSection

Section "Screen"
	Identifier	"Default Screen"
	Monitor		"Configured Monitor"
	Device		"Configured Video Device"
EndSection" > xorg.conf

```

You can try switching to the VESA driver as well if you think the driver is misbehaving.
