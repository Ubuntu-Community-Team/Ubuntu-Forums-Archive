### Post by beew on 2011-09-05
> **brawnypandora0 said:**
> I have 1GB DDR1 RAM with 128MB video card memory. Flash crashes everyday on my computer, particularly when I open multiple video tabs on my browser. How much more RAM do I need in order for Adobe Flash to never crash again?

If you only use Flash for Youtube videos then there are better solutions.

You can either use the flashvideoreplacer addon for Firefox or install minitube then you won't need flash at all.

The FVR gives you the same experience as watching Youtube through the browser and it works for a few other sites besides Youtube as well, the catch is that it has to use Firefox so cpu usage is still a bit high during buffering. Minitube uses even less cpu because it doesn't require FireFox or any browser but you need to search for the video so it is not so convenient if you just randomly browsing and it works only on Youtube. You can of course install both.

If you use minitube then install from this ppa
[https://launchpad.net/~nilarimogard/+archive/webupd8]("https://launchpad.net/%7Enilarimogard/+archive/webupd8")
Alternatively, download version1.5 from minitube (the package is "portable", it doesn't need installation)
[http://flavio.tordini.org/minitube](http://flavio.tordini.org/minitube)

The version from the Software Centre is very outdated and broken.

### Post by alegomaster on 2011-09-05
With the current flash you cant make it un-crashable no matter how much ram you have. There can be a code problem that will cause the flash application to crash. A lot of ram will make it have a lower chance of crashing, but there is always a chance of data corruption, glitches and such that may crash Flash.
