### Post by BKbroila on 2011-03-23
So I was reading a documentation on the differences between Linux and Windows, and came across a paragraph which I'm confused about. It's from this site:[http://www.linux-tutorial.info/modules.php?name=MContent&pageid=3](http://www.linux-tutorial.info/modules.php?name=MContent&pageid=3)

And about half-way down, there's these 2 paragraphs:
 'Each user, that is, each process, gets complete access to the [CPU]("http://www.linux-tutorial.info/modules.php?name=MContent&obj=glossary&term=CPU") for an incredibly short period of time. This period of time (referred to as a *time slice*) is typically 1/100th of a second. That means that at the end of that 1/100th of a second, it's someone else's turn and the current process is *forced* to give up the [CPU]("http://www.linux-tutorial.info/modules.php?name=MContent&obj=glossary&term=CPU"). (In reality, it is much more complicated than this. We'll get into more details [later]("http://www.linux-tutorial.info/modules.php?name=MContent&pageid=83").) 
  Compare this to an [operating system]("http://www.linux-tutorial.info/modules.php?name=MContent&obj=glossary&term=operating%20system") like standard Windows (not Windows NT/2000). The program will hang onto the [CPU]("http://www.linux-tutorial.info/modules.php?name=MContent&obj=glossary&term=CPU") until it decides to give it up. An ill-behaved program can hold onto the [CPU]("http://www.linux-tutorial.info/modules.php?name=MContent&obj=glossary&term=CPU") forever. This is the cause of a system hanging because nothing, not even the [operating system]("http://www.linux-tutorial.info/modules.php?name=MContent&obj=glossary&term=operating%20system") itself, can gain control of the [CPU]("http://www.linux-tutorial.info/modules.php?name=MContent&obj=glossary&term=CPU"). Linux uses the concept of *pre-emptive* [multi-tasking]("http://www.linux-tutorial.info/modules.php?name=MContent&obj=glossary&term=multi-tasking"). Here, the system can pre-empt one process or another, to let another have a turn. Older versions of Windows, use *co-operative* multi-tasking. This means the process must be "cooperative" and give up control of the [CPU]("http://www.linux-tutorial.info/modules.php?name=MContent&obj=glossary&term=CPU"). 





So my question is: how does Windows dedicate its resources? Does the primary program being used take up most of the resources? How does the CPU arrange its cycles/clocks between different programs?
