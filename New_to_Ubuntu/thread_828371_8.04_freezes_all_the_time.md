### Post by xeth_delta on 2008-06-13
Your problem might be related to the one described in this thread: [http://ubuntuforums.org/showthread.php?t=765510](http://ubuntuforums.org/showthread.php?t=765510)
Have alook also there.

Found the options that can be passed to the kernel when starting up. For starters, make sure you make a back-up of any important file.

When the computer is booting, in the GRUB menu select the line of the kernel to boot, and press "e". Then append at the end of the line, either alone or in compinations (have a look at [http://ubuntuforums.org/showthread.php?t=827663&highlight=noapic](http://ubuntuforums.org/showthread.php?t=827663&highlight=noapic), post #26):
```
noapic nolapic acpi=off pci=noacpi irqpoll pnpbios=off vga=0x317
```

Start with noapic. After you will have finished adding these options, press the ENTER key to set the options, then press "b" too boot.

If the system still crashes, try some different options.
