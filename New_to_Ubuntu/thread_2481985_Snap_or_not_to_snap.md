### Post by TenPlus1 on 2022-12-15
From what's been said already snaps are hosted by Canonical themselves which isn't open source, the downloads are huge, they unpack to your drive on first run meaning there's two copies of the program using space and memory to run said snap, don't adhere to system settings (themes etc.) on some apps, and important apps like browser should always be using the native package manager so avoid problems.  [https://www.omgubuntu.co.uk/2022/04/how-to-install-firefox-deb-apt-ubuntu-22-04](https://www.omgubuntu.co.uk/2022/04/how-to-install-firefox-deb-apt-ubuntu-22-04)

### Post by maglin2 on 2022-12-15
It seems to be getting a bit more difficult to avoid snaps but keep standard apps (eg firefox) with each ubuntu release. Linux Mint is probably the easiest way.
There is a long snap discussion here [https://ubuntuforums.org/showthread.php?t=2479001](https://ubuntuforums.org/showthread.php?t=2479001) with some very respected forum sages coming out against snaps. A great deal of what I've learnt about linux has come from reading forum posts by some of those folks.
To the best I can make out the advantage they bring for the user is an element of increased confinement, hopefully improving security.
The other benefits I think are for the packagers, not the user.
