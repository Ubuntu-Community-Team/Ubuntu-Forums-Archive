### Post by 3rdalbum on 2011-08-22
> **jrev said:**
> just because I do not want to be bound to reinstall a new version more often than necessary

Ubuntu can dist-upgrade from one version to the next while it's still running. (just requires a reboot). A lot of people will say it doesn't work very well, but I've never had a problem updating from one version to the next.

Be aware that using an old version of Ubuntu also means using old software. With the exception of Firefox, everything in Ubuntu 10.04.x is basically the same as it was back in April 2010. Which makes it ancient in Linux terms.

You probably want to try Ubuntu 11.04 instead. It's not an LTS, but it's still supported for more than a year. It's got much newer software and better hardware support. Then upgrade to Ubuntu 11.10 when it comes out, and you'll get newer software again and even better hardware support.

This isn't Windows where you can be using a 10-year-old version and it's only two versions obsolete. On Linux, they release often - every six months. The "long term support" versions are really intended for businesses who specifically want software that won't change overnight.

### Post by Zill on 2011-08-28
> **3rdalbum said:**
> ...This isn't Windows where you can be using a 10-year-old version and it's only two versions obsolete. On Linux, they release often - every six months. The "long term support" versions are really intended for businesses who specifically want software that won't change overnight.
Without wishing to go *too* far off-topic I maintain that, while LTS versions are ideal for businesses, they also provide a rock solid distro for *all* users who want/need software they can rely on.  Many (home) users just want a PC that fires up every logon and performs the functions requested without any drama!  Imagine having to debug the kettle every time you wanted a cup of tea!

Unfortunately, too many users still seem to want the "latest and greatest" without ever asking themselves "why?".  Unless they *need* the new functionality there is no point in upgrading until the software reaches end-of-life.  Just read the posts in these forums and you can see inexperienced users struggling with software that is really still in the beta, or even alpha, development stage.

LTS is great for those who simply want software that works and users wishing to upgrade should first consider the old adage "if it ain't broke don't fix it"!.
