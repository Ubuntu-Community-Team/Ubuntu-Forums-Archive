That thread doesn't weigh much value here as the graphic chip mentioned there was on board and less than half of ''shared'' graphic memory in contrast to the OP's ''dedicated graphic card''. Which should be capable of running Ubuntu specification wise.

However, this does sound like a graphic card issue, a very fossil way I can suggest is:
1.Remove the card and try running on board graphics by attaching the monitor cable to on board port, see if you can complete the installation. 
2.If it does, see if you can get the system working after getting card back in and attaching monitor cable to it.
3.If it boots in nomodoset, then install the 3rd party drivers. 

This particular card has been reported to work with 12.04 free drivers (As you're seeing now), so if this sounds like too much of a hassle, I would personally stick with 12.04 as it still has support for next couple of years.
Work with what works best for you ;)

### Post by oldfred on 2014-06-28
With nVidia card you need at least the nomodeset. But you also then may need other boot options?

At accessibility screen (person,keyboard icons) press any key and then f6 to choose nomodeset.

 How to set NOMODESET and other kernel boot options in grub2 - both BIOS liveCD & grub first boot ( also UEFI with grub) 
[http://ubuntuforums.org/showthread.php?t=1613132](http://ubuntuforums.org/showthread.php?t=1613132)


After you install, you need nomodeset to boot until you install the proprietary nVidia driver from the repository.
At grub menu you can use e for edit, scroll to linux line and replace quiet splash with nomodeset.

       Info on other boot parameters
[http://www.kernel.org/doc/Documentation/kernel-parameters.txt](http://www.kernel.org/doc/Documentation/kernel-parameters.txt)

I have an older nVidia 9600GT  and have had to use nomodeset with every live boot and first boot since they modified kernel to include some video drivers back with either 11.04 or 11.10?
