For clarification: **Programs native to Linux means these programs only work for Linux.**

The more usual definition of "Linux-native" means that it was written under Linux and runs under Linux without modification.
It does NOT normally mean "runs ONLY under Linux".


> I would have started with unpacking my vlc deb file here but I am curious to see if I can figure that out on my own and **vlc is a hybrid program not a native one.**

The VLC project was designed and written by students in a French University.
The modules were all written under Linux, to operate on Linux boxes.
(the students cited "FOSS, stability and speed" as the main reasons to use Linux)
Since it was written with the Unix Paradigms in mind, porting it to other OS' was not a major hassle. 

You may be confusing "native" with "proprietary".
Most MS-Windows software has legal bars (copy-right, patents, NDA, etc) to being ported to other OS's.

Most Linux software is written under the GPL or other FOSS/OSS/BSD licenses. This makes it easier to port to other OS's.

That ground-breaking MS-Windows game called "Doom" was developed and written on Linux systems (developers cited "speed and stability").
Only when it was "done" were the I/O libraries changed to MS-Windows.
Does that mean that Doom was not a native Linux game?

> Thank you in advance for any assistance you can give me. Z/Z

Go to Synaptic Package Manager... you'll find some 33,000 packages there... Almost all written under Linux/Unix, most of the ones that were not are libraries, drivers or fonts, not programs.

> Hybrids are references to programs that are adaptable to multiple OS's and therefore more utilitarian.

Linux programs are , almost by definition/design , "adaptable to multiple OS's".
The Unix/Linux Paradigm means that software is designed to used by the greatest number of people.
It is open, to allow it to be studied, and ported.
It is Free, so that it can be studied and used by anyone.

VERY few Linux programs are so closed as to be useless to Windows/Mac/BSD users.
