### Post by westie457 on 2011-07-13
Hello and welcome to the Forums.

To me it looks as though we have to get your Windows working first, then we can get everything else done.

So shall we start by you booting Windows into the recovery position. Do what you did before and go through the screens as they show until you get to the one that says 'install'. Here look for the option to 'Repair Your Computer'.

Let Windows repair itself first, doing whatever it has to do to repair any faults it finds. Be aware that Windows may start several times while the repairs are done. When the process has finished you will be asked to reboot. Do so and if Windows does not start properly or at all you will have to start the repair process again. This time instead of allowing automatic repair take the option to use the command prompt.

In the command prompt window enter the following commands one at a time pressing enter after each.


```
bootrec /fixmbr          bootrec /fixboot             bootrec /rebuildbcd
``` 

All being well you should now be able to boot Windows.

If not come back here and we will try something else to get Windows working.

When we have got Windows working come back here then we can fix Grub.

### Post by Quackers on 2011-07-14
Windows won't boot because at some time grub has been installed to sda1.
That means that Windows is looking for a boot sector type of Windows on sda1 and it can't find it because it's now a boot sector type of Grub.
You need to repair the Windows bootloader with a WIndows repair disc/installation disc, which should get Windows booting. Once that's done you can re-install grub so that Ubuntu boots and the grub menu gives you a choice of which OS to boot.

### Post by oldfred on 2011-07-14
The testdisk fix usually works if nothing else was done. The windows fixes should work if it sees it as a NTFS partition with the boot flag. Is the boot flag still on the sda1 partition? Maybe resetting partition lost boot flag. Windows has to have boot flag to know what partition to repair.

I am not an expert on windows but have suggested these fixes and they usually work. I know one user just copies a BCD file from the repair CD and then ran repairs and that fixed the BCD. But I think you still have to run fixBoot to make sure the NTFS partition has the valid boot sector.

How to use the Bootrec.exe tool in the Windows Recovery Environment to troubleshoot and repair startup issues in Windows
[http://support.microsoft.com/kb/927392](http://support.microsoft.com/kb/927392)
Use bootsect.exe to repair XP & older or Vista/7 bootsectors
[http://www.sevenforums.com/tutorials/668-system-recovery-options.html](http://www.sevenforums.com/tutorials/668-system-recovery-options.html)
[http://www.sevenforums.com/tutorials/20864-mbr-restore-windows-7-master-boot-record.html](http://www.sevenforums.com/tutorials/20864-mbr-restore-windows-7-master-boot-record.html)
[http://windows7ultimate.windowsreinstall.com/repairwin7startup/indexthumbs.ht](http://windows7ultimate.windowsreinstall.com/repairwin7startup/indexthumbs.ht)

Above links summarized, see links if more detail desired
You will need to boot with your Vista/Windows 7 installation disk or repair disk. Hit Enter at the language selection prompt then hit "R" to get to the repair section. You can then select the automatic boot repair tool, but it often will not do any good. Then select the command prompt (console) and type in the following commands:
BootRec.exe /fixMBR   #updates MBR master boot record  do not run if you still want grub
chkdsk C: /r /f (may have to run /r or /f as separate entries) rerun until no errors
BootRec.exe /FixBoot  #updates PBR partition boot sector
BootRec.exe /ScanOs
BootRec.exe /RebuildBcd

How to fix Vista/Window 7 when the boot files are missing - rebuild BCD with bcdedit
[http://ubuntuforums.org/showpost.php?p=5726832&postcount=4](http://ubuntuforums.org/showpost.php?p=5726832&postcount=4)

### Post by westie457 on 2011-07-18
With windows recovery the X:\ is running in ram memory. Kind of like an Ubuntu LiveCD without all the functionality. I believe that while in the Windows recovery environment it is possible to find files in any ntfs or fat32 volume and copy them over to the C:\ drive. As an example autoexec.bat located in d:\ can be used to overwrite the same file in c:\. maybe 'oldfred' or someone else can confirm this. Also it is possible to reinstall Windows without losing settings using the command 'sfc/SCANNOW' without the quotes. Unfortunately I have not been able to run 'sfc' from the X:\ it can only be done while Windows is running.

Do you have a spare hard drive the same size or larger than the one currently in use. If you have use a LiveCD to clone using```
dd if=/dev/sda of=/dev/sdz
``` where sda is your current drive and sdz is spare one. If what I am about to suggest goes horribly wrong we will still have something to work with.

Once the dd command has finished shut down the pc and remove the drive that was copied to.

Now for the dangerous bits where it can all go wrong.

Boot into a LiveCD again and run Gparted. Go to Device and Create New Partition Table. This will effectively wipe the hard drive of all partitions. Do nothing else here. Close Gparted and in a terminal run ```
testdisk
```
Allow testdisk to find and recover all partitions. When this has completed allow testdisk to write a new Windows compatible MBR.

Exit testdisk and reboot. Windows should now start. If not go into the recovery environment and firstly attempt automatic repair. Running this until Windows is happy and reports that the operating system has booted properly. Restart the pc, if it does not run now back into repair windows and choose command prompt and run the 3 commands I posted earlier in this thread.

Windows should now start properly and Grub can now be restored from a LiveCD.

Any information for testdisk can be found here. [http://www.cgsecurity.org/wiki/TestDisk_Step_By_Step](http://www.cgsecurity.org/wiki/TestDisk_Step_By_Step)

[COLOR="Red"]If you do not have a spare hard drive the either do not do anything suggested here or do so knowing that if it does not work you will have an expensive paperweight[/COLOR].

Good luck. You might need all you can get.
