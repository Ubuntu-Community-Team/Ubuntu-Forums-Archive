### Post by Kellemora on 2009-02-22
Hi Swotie

The only thing you need to change in smb.conf is your Workgroup, which if you are using MSHOME as your workgroup name, you have already done that.

Besides Samba (which should be loaded already) I suggest also loading from Synaptic smbclient and smbtree.  And if you see smbfs is installed, uncheck that box as it prevents some shares from showing up.

In terminal you can type smbtree and if all of your shares show up there, then both Samba and your Network is working fine.

Make sure you DO HAVE a Shared Folder to SEE also!

IF you can see all of your shares from terminal using smbtree but you cannot see them in Places/Network opening page or under the windows workgroups MSHOME folder.  This is an intermittent BUG that has been in Nautilus now for a few years.

Instead, go up to Go/Location and type in the IP number of the computer you are trying to get to.  Then they should show up for you.

The Locations line should read like this, but using YOUR IP numbers.
smb://192.168.1.101/  (notice only TWO right slashes, not three.
If you KNOW the name of the particular shared folder you are hunting for, it can be typed as.  smb://192.168.1.101/myfilefolder/
If you used spaces in the name then enclose it in quotes "my file folder"/

Hope that helps you out!

TTUL
Gary
