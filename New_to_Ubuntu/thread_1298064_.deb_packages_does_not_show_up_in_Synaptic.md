### Post by CatKiller on 2009-10-22
> **MockY said:**
> why is it not available via Add/Remove?

Because the Add/Remove tool isn't a proper package management tool. It's something familiar that Windows users can look at, so that some of them will grasp the general idea of package management and not come here with "questions" like "i dl'd tihs exe from teh internets and it dunt work ubnutu is crap."

(doesn't show all packages) + (doesn't handle dependency resolution particularly gracefully) = (not a proper package management tool)

It is very good at doing what it's supposed to do, which is to provide a list of common "higher-level" applications (with a GUI and everything) that are simple to browse and simple to install.

Any softare that is listed in Add/Remove can also be removed through Add/Remove.

The tools exists to be a simple and easy to use interface for adding and removing a selection of the most popular applications. Synaptic, on the other hand, is a full-blown package management tool, and being one includes far more options, and packages with strange names, than any person new to Ubuntu might want to see. So both tools have their uses.

(Adding every installed package to the Add/Remove tool would quickly turn it into just as complicated tool as Synaptic is, and add a lot of "noise" since if any .deb packages you installed yourself would be listed there, that would also mean that all the library packages and other stuff that are not actual programs would have to be listed as well. So instead of just seeing some actual programs with nice descriptions you'd see 18 000+ packages, most with rather technical descriptions and very cryptic names..)

It's not a full-blown package management tool, and its not even supposed to be one.

Comparing to the Windows tool isn't really fair, knowing that the tool with the same name in Windows isn't actually able to *add* any software apart from enabling/disabling some built-in features of Windows... If you ask me, it's quite obvious which tool wins the comparison..

### Post by Kapitän Rotbart on 2009-10-22
*Add/Remove* can remove software that is displayed in *Add/Remove*. Synaptic does display everything, but *Add/Remove* is more n00b-friendly. It'll all get better with the software center, which will eventually allow all sorts of things (adding PPA's, installing fonts, installing and removing software via Wine, installing non-GUI software like in Synaptic, upgrading software, viewing installed software and other easy browsing, etc.). It should essentially replace Synaptic in the long run, and then some.

Mind you pre-Karmic is still pretty good, I mean how centralized do you need your installing/removal of software? Having *Add/Remove*, Synaptic and the CLI gives the desktop the choice of the most appropriate way to install things depending on the circumstance. Load an MP3 in Totem but haven't installed the codecs yet? Totem will prompt you to install the codecs via *Add/Remove*, which is a very simple interface for such a simple task. Granted the Software Center will revolutionize Ubuntu for n00bs, but I spend little of my time actually installing/removing software, so it's a much smaller feature for me than, say better drivers and the latest versions of software I do use like Firefox and OpenOffice.
