### Post by Sorivenul on 2009-02-09
Unlike Ubuntu, how?

Philosophy?
Default included applications?
Package management?

If you want an adventure try a BSD (DesktopBSD is my usual suggestion for new users, as it includes a KDE desktop by default). 

If you want to stick with Linux, Fedora and openSUSE are relatively different from Ubuntu both in package management and in philosophy, but they still provide relatively "easy-to-use" distributions. If you want a bit more of a challenge, Slackware or one of its derivatives (Vector and Zenwalk come to mind), is, IMO, the most unlike Ubuntu that still provides a GUI out of the box.

### Post by niteshifter on 2009-02-09
Hi,

I'd add Fedora to the not Ubuntu list based upon the difference in the security models and package management. SELinux libs, while installed on an Ubuntu system are dormant. It's the model on Fedora - and one should have some familiarity in setting and maintaining SELinux, you'll come to appreciate apparmor more ;)

And while not Linux kernels, do give a thought sometime to the Unix cousins, the BSDs and Solaris. OpenSolaris runs rather well in Virtualbox.

You'll find that GNOME is GNOME (and KDE is KDE) pretty much, no matter what distro it runs on. The differences in distros are below the desktop, they are in administration, security model, and core utilities.
