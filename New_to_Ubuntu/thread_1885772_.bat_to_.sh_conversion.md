Okay, that's okay. 

So there are basically 2 routes you could take. See if the .exe files have the same effect being ran through wine by one of the commands giving above. 

Second route is to see if you can replace the functionality of those programs with somethings in Linux. Then you could convert your batch script to a shell script. 

The reason for this is that windows executables are not by default binary compatible with Linux. Recompiling them on Linux would make them binary compatible. Also, a compatibility layer may also let you run those as if they were native applications (without recompiling). This is what wine does. 

If you want any more information, just ask. I can provide some links on how compiling software works and how a compatibility layer works. Otherwise let us know how the above work.

### Post by satanselbow on 2011-11-24
Your scripts start a http server "nginx" which can run natively on Linux - though most are more familiar with "apache"

It then - through a php interface - converts your files using lame / ffmpeg and a couple of standard php extensions... all do-able under any Linux you like...

The php interface is most likely just there to make it look all purr-etty and is trivial.

The important guts of the matter are the ffmpeg commands which will be buried (at a guess) in the php somewhere (most likely **AudioConverterSOX.class.php**)

Have a poke about in that file and you will find the important ffmpeg command lines that could then be executed in a terminal or refactored in a bash script.

You could alternatively run the php app under apache (or indeed nginx) although for what you are doing (ie file converting) this might involve a steeper learning curve.

Once you have the ffmpeg commands you could probably even set it up as a WinFF preset and run it through that ;)

Simples... ish...

Using WINE to run an http server and php is ridiculous and redundant in equal measure :confused:
