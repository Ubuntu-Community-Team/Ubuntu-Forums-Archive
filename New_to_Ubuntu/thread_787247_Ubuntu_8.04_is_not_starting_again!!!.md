guys why dont you just try to offer the solution instead of arguing about which os is better



i think your problem with grub was probably caused by the updates that we just recieved


when updating a dialouge appeared that says "use current version of grub" (you should have changed it to the one that says "use package manager version", but im betting that you just clicked ok on the dialogue without noticing what it was asking)

im not really an expert on this but you can boot the live cd them mount the ubuntu partition and change your /boot/grub/menu.lst

when at the command line first thing to do is backup existing menu.lst

```
sudo cp /boot/grub/menu.lst /boot/grub/menu.lst.copy
```
then open menu.lst 

```
sudo gedit /boot/grub/menu.lst
```

scroll down toward the bottom and you should see a part that looks like this 
```

title		Ubuntu 8.04, kernel 2.6.24-18-generic
root		(hd0,0)
kernel		/boot/vmlinuz-2.6.24-18-generic root=UUID=3261a2f6-7bbf-4cdf-b21a-127015e0b240 ro quiet splash
initrd		/boot/initrd.img-2.6.24-18-generic
quiet

title		Ubuntu 8.04, kernel 2.6.24-18-generic (recovery mode)
root		(hd0,0)
kernel		/boot/vmlinuz-2.6.24-18-generic root=UUID=3261a2f6-7bbf-4cdf-b21a-127015e0b240 ro single
initrd		/boot/initrd.img-2.6.24-18-generic

```

change it to look like this 

```

title		Ubuntu 8.04, kernel 2.6.24-19-generic
root		(hd0,0)
kernel		/boot/vmlinuz-2.6.24-19-generic root=UUID=3261a2f6-7bbf-4cdf-b21a-127015e0b240 ro quiet splash
initrd		/boot/initrd.img-2.6.24-19-generic
quiet

title		Ubuntu 8.04, kernel 2.6.24-19-generic (recovery mode)
root		(hd0,0)
kernel		/boot/vmlinuz-2.6.24-19-generic root=UUID=3261a2f6-7bbf-4cdf-b21a-127015e0b240 ro single
initrd		/boot/initrd.img-2.6.24-19-generic

```
