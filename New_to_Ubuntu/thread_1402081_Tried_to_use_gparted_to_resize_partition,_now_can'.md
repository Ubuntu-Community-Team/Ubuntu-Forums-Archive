### Post by kansasnoob on 2010-02-11
Assuming you have not rebooted since trying this:

> ubuntu@ubuntu:~$ sudo mount /dev/sda1 /mnt/boot
mount: mount point /mnt/boot does not exist
ubuntu@ubuntu:~$

First try to make sure sda1 is unmounted:

```
sudo umount /mnt
```

Then:

```
sudo mount /dev/sda1 /mnt && sudo mount --bind /dev /mnt/dev && sudo mount --bind /proc /mnt/proc && sudo mount --bind /sys /mnt/sys && sudo mount --bind /dev/pts /mnt/dev/pts && sudo cp /etc/resolv.conf /mnt/etc/resolv.conf && sudo chroot /mnt
```

If that fails just reboot the Live CD to save time and headaches then repeat that last command and then make sure you are where you want to be:

```
lsb_release -a
```

That should show you're in Karmic, so first lets gather some info for future use in case this fails:

```
ls /boot
```

```
aptitude show ubuntu-minimal|head -4
```

```
aptitude show ubuntu-standard|head -4
```

The first of those might show if you're missing any boot files/directories, the other two are mostly for my benefit because I don't know server installs well. But lets try to fix grub2:

```
apt-get install --reinstall grub-common
```

```
apt-get install --reinstall grub-pc
```

```
grub-install /dev/sda
```

```
update-grub
```

Then exit and unmount:

```
exit
```

```
sudo umount /mnt/dev/pts && sudo umount /mnt/sys && sudo umount /mnt/proc && sudo umount /mnt/dev && sudo umount /mnt
```

Then see what happens on reboot. If it fails and the two aptitude show commands show this (I'm using Lucid so version numbers will be different):

```
lance@lance-desktop:~$ aptitude show ubuntu-minimal|head -4

sudo mount /dev/sda1 /mnt && sudo mount --bind /dev /mnt/dev && sudo mount --bind /proc /mnt/proc && sudo mount --bind /sys /mnt/sys && sudo mount --bind /dev/pts /mnt/dev/pts && sudo cp /etc/resolv.conf /mnt/etc/resolv.conf && sudo chroot /mnt

```

Now, if that balks look here:

[http://www.ubuntu.com/getubuntu/releasenotes/910#Upstart%20jobs%20cannot%20be%20run%20in%20a%20chroot](http://www.ubuntu.com/getubuntu/releasenotes/910#Upstart%20jobs%20cannot%20be%20run%20in%20a%20chroot)

And I don't know the status of upstart in the server edition but you might need to:

```
sudo mount /dev/sda1 /mnt && sudo mount --bind /dev /mnt/dev && sudo mount --bind /proc /mnt/proc && sudo mount --bind /sys /mnt/sys && sudo mount --bind /dev/pts /mnt/dev/pts && sudo cp /etc/resolv.conf /mnt/etc/resolv.conf && sudo chroot /mnt

### Post by Mustard on 2010-02-11
I'm not sure what happened with the failed mount command at the very beginning as that is an issue straight away.  You need to get a chroot environment going and the error message seems to prohibit you going much further than that....

Without the chroot environment going, what I have said after this is fairly irrelevant.

With regards to the latter errors, I would say you need to use sudo in front of the apt-get command that created the error.  Installing packages is a job for the administrator account ie root, so you need to be a super-user (sudo) to run apt-get install.

Either that or their is another package-manager running somewhere that has created a lock file to avoid two package-managers conflicting (which would be extremely unlikely in this scenario)

See example below for the same command with sudo in front of it...
```
ubuntu@ubuntu:~$ sudo apt-get install --reinstall grub-common
```
