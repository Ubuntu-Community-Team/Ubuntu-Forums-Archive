### Post by drs305 on 2009-01-07
> **chanders said:**
> I am having this same problem. Any help would be appreciated. My drive is formatted FAT32.

If you are unable to mount the device as a normal user, you can edit fstab: Backup and open fstab for editing:
```

sudo blkid  # to find UUID of your fat32 partition
sudo cp /etc/fstab /etc/fstab.backup
gksudo gedit /etc/fstab

```

If you don't have a mountpoint:
```

sudo mkdir /media/*yourmountpoint*
sudo chown -R your.user.name /media/*yourmountpoint* # this step not mandatory as ownership of mountpoint is established at mounting for fat32

```

Replace XXXX with the UUID of the device, replace 'mountpoint' with the mountpoint (usually /media/some.mountpoint.name  like /media/data).
```

UUID=*XXXXX* /media/*mountpoint* vfat auto,users,uid=1000,umask=027,utf8 0 0

```
Save the file.
Unmount and remount the device (you will get some 'busy' messages - that's ok):
```

sudo umount -a
sudo mount -a

```

### Post by drs305 on 2009-01-07
> **chanders said:**
> Thanks guys. I can get it to mount if I type mount -a with the edited fstab. However I want it to automount in gnome.

Since it mounts without errors when you run 'mount -a' the fstab entry is correct and working. For the automount problems, take a look at the mount preferences by looking at the gconf-editor settings for root. 
```
gksudo gconf-editor /apps/nautilus/preferences/media_automount
```
Make sure the automount option is enabled.

You can then run the same command without 'sudo' to check your individual user preferences. Again, automount should be enabled.
