### Post by grahammechanical on 2013-10-17
How large does a hard disk have to be before you think that it has plenty of space?

You cannot run Windows programs on Linux and you cannot run Linux programs on Windows. So, if you are going to have 2 operating systems and you want to use the same sort of programs then you are going to use up space on your hard disk for two operating systems and the programs that you install on them. For example, You will have a web browser in Windows and another web browser in Linux. This will be unavoidable.

Imagine installing both Windows 7 and Windows 8, That is possible. There will be a duplication of programs and with it a extra use of disk space.

Another thing, you should stop thinking of Linux as a free to use copy of a Microsoft product. It is not. The Linux developers and the developers of all the bits that go together to make a Linux operating system have their own ideas of what an operating system should do, how it should work and how it should look.

How easy is it to re-install Windows at present? Then that is how easy it will be to re-install Windows once you have erased it from the hard disk. Does Microsoft make it easy to re-install Windows? Does it provide install disks? There are those who choose to delete Windows and then they come on this forum asking for help to re-install it. What can anybody say to someone in that situation? This is why we advise dual booting and not erasing an OS that you have paid for until you are ready to kiss it good-by for all time.

What you should do is download Ubuntu and run the live session. It is possible to install Ubuntu on a USB stick and run it from the USB stick as if it was installed on a hard disk. We cannot do that with a DVD because it is not possible to save to a DVD. But we can save to a USB memory stick. Then you can install programs and see for yourself if you can read and write to your Windows files.

As for Skype, it was proprietary software developed for Windows long before Microsoft purchased it. Although there is a Linux version development has always been behind that of the Windows version.

As for Chrome, I thought that it was a Linux product first and then for Windows. There are Windows versions of Firefox, Libreoffice that you can install and try out. Both are default in a Ubuntu installation.

### Post by Lars Noodén on 2013-10-18
Actually, it is an open secret that you can run some Windows applications under Linux using [WINE](https://help.ubuntu.com/community/Wine), mentioned above.  It is not an emulator, it is a re-implementation of the Windows APIs.  But because those are not properly documented, they have to do a lot of guessing.  So some programs run perfectly well under WINE, others not so well.  You'll have to check on a  case-by-case basis, but some programs do run quite well.

Another tip is that you can run Windows inside a virtual machine on Linux using Virtualbox or Qemu.  A big advantage to doing it like that is you can make a snapshot of a pristine new installation and keep reverting to that snapshot whenever you need Windows.  That can improve performance and reduce problems with malware and Windows cruft/bitrot.

About sharing documents between the two platforms, that can be a little harder.  Images and video are fine.  But productivity software like spreadsheets, word processing and presentations is harder because of Microsoft's history of hatred against formats that are not their own.  MS tends to implement other formats, including open standards, in an incomplete and (intentionally) broken manner.  Then on the other hand, the MS formats are only partially documented and changing, presenting a difficult moving target for open source developers.  That said, LibreOffice has better compatibility with older MS formats than MS own software, I have found.  The compatibility with the new MS formats is pretty good, and improving.  But best of all are native formats like the Opendocument Format.  If you choose the latter, you can run LibreOffice in both Windows and Linux.  Regardless of whether you use MS Office or LibreOffice, if you hand in an assignement electronically, it is wise to use only PDF with embedded fonts.  Otherwise, even MS Office can screw up its own formatting and layout.  Going between LibreOffice and LibreOffice, the rendering is much more reliable.  But in general conversion between the programs is ok.

Steam is investing big time in Linux.  So if your game is not currently available, it might be soon.  You'll have to check the list.

### Post by Bucky Ball on 2013-10-18
Welcome. Start with 12.04 and stay there until you know what you are doing. 12.04 LTS is a great place for a beginner to start as it's stable and a known quantity. 13.10 is wet behind the ears and, as you can see, the forums are flooded with issues relating to it right now. I wouldn't go there unless you want a VERY steep learning curve.

Also, LTS releases are supported for five years (12.04 LTS - April 2017) whereas the interim releases are, at the moment, supported for nine months. 

Upgrading from 12.04 to 13.10 is not a good option anyway. A clean install of 13.10 best if you really want to go there. My suggestion? Install a stable, everyday task install, 12.04, and install 13.10 as a plaything on another partition. That way, if 13.10 breaks you have 12.04 AND Windows to fall back on. 

For more detail as you go, break up your questions and post them in the appropriate forums. But for now, make some free space on your disk and install 12.04 to it. Choose 'Something Else' when you get to the partitioning section and you will see the Windows NTFS partitions clearly marked and the free space right there to install Ubuntu. Back everything up before you start then fire away. ;)

PS: The 'Ubuntu inside Windows' option is Wubi. I wouldn't go there either. If you want to try Ubuntu, try it from the disk, then install from there to a partition. Wubi is not considered a long-term solution and your original idea of 12.04 Wubi to 13.10 won't wash as Wubi is no longer (phased out in 13.10).
