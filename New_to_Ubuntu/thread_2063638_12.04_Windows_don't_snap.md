### Post by deadflowr on 2012-09-27
When you first boot up your system Ubuntu runs this:

```
/usr/lib/nux/unity_support_test -p
```

If any of the listed items come up no, then unity defaults to the 2d version, which doesn't have snapping windows feature. Sometimes   the command just doesn't read the system right, and a perfectly fine and capable system is identified as incapable.

Logging out and reselecting the desktop enviroment(Ubuntu) in the cog symbol in the top right side of the login box, and relogging in sometimes works.

Another option is to install ccsm(compiz-config-settings-manager), and check to see if snapping windows is enabled. Be awre that anytime you use ccsm, changes can appear to crash your system, be patient, as the program needs to readjust it self, and usually it will revert to a working condition shortly(don't panic, and don't do anything, unless the crash prolongs for an unreasonably extned peroid of time, say five minutes).

Note: Sometimes the problems can be a misconfigured xorg.conf file. Configured in a way to get a workable system, but also leaving capabilities omitted.
