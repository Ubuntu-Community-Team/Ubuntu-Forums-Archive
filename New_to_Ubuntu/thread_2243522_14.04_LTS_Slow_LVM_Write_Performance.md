### Post by Ryan_Hartwick on 2014-09-09
Firstly let me just state that I'm a relatively new Ubuntu user, but I have a solid background in Windows networking and server management.

I'm currently using 14.04 LTS on a spare PC I had as a sort of a home media server.  I've got a 2TB system partition I'm using and then I've created an LVM using three other disks to give me a 5TB media partition. I created the LVM by using the steps in this guide:
[http://www.howtogeek.com/howto/40702/](http://www.howtogeek.com/howto/40702/)

Read speed from the LVM is good, at around 100 MBps, but write speed from the system partition to the LVM partition goes down to about 30 MBps and if I copy a file within the LVM partition itself I'm sitting at about 20 MBps.  I took a look my CPU performance during the process and when when I was doing the internal copy on the LVM partition I was sitting about about 50% cpu usage, which seems high to me for a file copy.  I'm using a Core 2 Duo E6850 Processor with 4GB of memory. 

I guess I'm just wondering if this type of performance is to be expected, or perhaps if there is a way of correcting it via configuration or hardware upgrade.  I'm mainly concerned about read speeds but I do download content I need to then push to the Linux box and I'd like to speed up that process if possible.

### Post by redrumrogue on 2014-09-13
Hi there - are all the drives that make up the volume group internal drives?  Any that are USB?  I had issues with write speed to an external USB backup HDD and found that it was down to the way it had been automounted at boot.  To fix the problem I added the drive to my /etc/fstab with the "default" mount option (which sets async I/O for better performance).  Once the drive had been mounted via the fstab I was then also able to access the Drive Settings in gnome-disks and turn on write caching which dramatically improved write speeds.  Some people don't think write caching is a good idea but I like performance!

Another suggestion for you if you're using LVM - if you create a logical volume manually via command line you can make it a striped volume.  For example if your volume group consists of two drives then you could create a striped logical volume like so ...

sudo lvcreate -i2 -l100%FREE -n logicalvolumename volumegroupname

the -i2 bit determine how many stripes and should not be more than the number of physical volumes in the volume group.  Striped LVs will give you MUCH better read/write performance compared to a non-striped LV.  I use striped LVs for my media volume and for another volume group that I use as a virtualization (QEMU / KVM) volume pool.

Read here for more info on striped LVs ...

[https://www.centos.org/docs/5/html/Cluster_Logical_Volume_Manager/striped_volumes.html](https://www.centos.org/docs/5/html/Cluster_Logical_Volume_Manager/striped_volumes.html)

[https://help.ubuntu.com/community/StripedVolumeHowTo](https://help.ubuntu.com/community/StripedVolumeHowTo)

Hope this helps!
