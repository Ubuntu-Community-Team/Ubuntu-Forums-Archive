### Post by lads on 2010-01-06
I'm reviving this thread because I found myself in the same page as the fella that kick started it. I managed to leave Win behind for good months ago but I miss things like notepad++. I'm looking for something with an easy and friendly GUI and macro support. A plug-in for gedit with macros should do it, but unfortunately it doesn't exist.

I've installed most of the editors indicated throughout the thread appart from kedit (I'm on Gnome). Here's the veredict:

**Bluefish **
Possibly the best user interface (completelly integrated with the graphic environment), doesn't support macros, hence failing the requirements. Few functionalities, although it can be handy for web programmers.

**SciTE **
Doesn't support out of the box macros either. The GUI didn't impress me, suffering from the pathologic black on white.

**Emacs**
Plenty of functionalities at the tip of your fingers on the worst user interface of this lot. Long ago I use Emacs quite frequently, so I'm used to store in a file the commands that I use more often. Will stay installed in my system, but not recommended for folk coming fresh out of M$Win.

**JEdit**
Plenty of functionality too, but on a much better GUI, with easily accessible menus. Unfortunately the pathologic aggressiveness of black-on-white seems impossible avoid, as so a coarse text font. If text editors are made by programmers why in hell do they insist in eye-killing interfaces? Functionality may keep it installed though.

**Komodo**
The best. Functionality galore taking text editing to another level, introducing concepts such as projects that you usually only find in an IDE. Very flexible interface, going from bare notepad to IDE. Default black-on-white colour scheme but easily customizable. With all the toolbars open this is the editor that most resembles npp. Will be for now on my text editor.
