### Post by Sidewinder1 on 2009-03-29
64 should be OK; burn at a slow speed (4X?) to prevent errors. Best to check the hash with something like MD5sum to be absolutely certain of no errors. Don't forget to defrag your drive if you're going to create new partitions to dual boot.

It doesn't matter which speed you burn the CD, but the faster you burn it, the more likely there will be a burn error, but the likely hood of that is still pretty low, CD burners are pretty good now, you hardly ever hear of a CD not burning right, A DVD, sometimes, but not so much with CD's. But If you want and you have a few extra minutes, go ahead and burn it on the slowest speed possible. 

You can burn a 32 bit version of Ubuntu and it will work fine on a 64 bit processor, but if you want to use the full capabilities of your machine, burn the 64 bit version. The 64 bit version will work fine but burn which ever version you want, but with me, I decided to go with the 64 bit version.
