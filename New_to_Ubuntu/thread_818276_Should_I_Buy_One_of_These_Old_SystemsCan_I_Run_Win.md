### Post by Thelasko on 2008-06-04
I would get an [EEE PC]("http://eeepc.asus.com/global/guide.htm") if I were you.  Sure, it's more expensive than the used computers you are looking at, but at least you know what you are getting.  It's not a perfect substitute for your macbook, but it's a lot cheaper.

You can run old Win95 games in WINE.  It has a compatibility tool that allows you to run programs in a variety of Windows like environments.  I personally got C&C Red Alert to work with it.  It wouldn't run in XP.

It's a tough choice, not much will run on those machines.  You will likely need to run Puppy Linux, or Damn Small Linux to even be usable.  In my opinion a P3 is a big step up from a P2.  A P3 is far more compatible with today's hardware and software.

### Post by Neobuntu on 2008-06-04
Honestly, don't waste your money. It doesn't matter if you can afford it or not. Here's why.

Opinion: 

If you can afford it, you can get newer or newish hardware and there is no substitute for newer hardware. Yet don't feed the Microsoft, please.

If you can't afford it, people will give you "newish" hardware. Why waste your time on older stuff? Unless you just want to spend time.

While you can make a system do many GUI jobs with 32MB (assuming you have enough drive space too), it takes time and is slower. Mainly you have to fall back to "slim" programs to stay out if the dreaded swap space (disk thrashing). You will want at least 384MB RAM so that you can run Kubuntu at speed.

If you want a system for old computers and you have at least 128MB to work with (**really** give away stuff) then you might try one of the many live CD versions of Puppy Linux. It's **very speedy** and automatically running from RAM. This would be the fastest (burn a CD and boot time saver) but I warn you. Setup is **different**. It uses very useful but different hardware wizards. Therefore, compatibility varies. Upgrade ability can be less too. Maybe the next Puppy release is better for you and maybe it won't be. You can get behind the times. Therefore this may not be best for the more exotic uses. All this depends on what you want to do.

If you want to do a quick test of a hard drive install and see what you might want to build, see AntiX, the small MEPIS/Debian variant. 

Eventually, you may achieve the best compatibility and upgrade ability with a Ubuntu **BASE** install and a custom **icewm** setup with the SilverXP theme. You'll do best by finding a pre-done icewm preferences text file for it and using that as your stating point; for speed. You'll have to learn which small apps are known for memory conservation. Such as the ones, Puppy, DSL, AntiX, and Absolute utilize.

Yet again, it will never be as nice as Kubuntu and will not will garner as many converts, to open software. I would not demo old hardware to the confused Windows sheep. Do not drag out your dog box, for open software tests! There is zero reason not to use something like Kubuntu (with 384+MB RAM) on your best system. It is so very easy to set up a dual boot and thus delete nothing. It's not like the time killer (An outdated Windows install and build.)

Also, it is usually best to run native programs in their (real) native OS! There are enough hidden bugs in any native program; without adding to the mix. The thing is, there is no lack of native open software applications (to say the least). That said, if you simply need to access you old Quicken history or see that Photoshop has little; over free and open photo programs(Digicam/Gimp), then by all means, you may simply install and run those Windows programs using WINE. With Kubuntu, a WINE install is an easy click, from the System Settings AKA "Control Panel" Yet really, it's kind of going backwards and is not strictly necessary. Your time is better spent, just enjoying native apps; in Kubuntu.

Beyond having enough RAM, please avoid any devices that do not auto magically just work with Kubuntu. There's no good reason for Windows-only hardware and while much of that stuff has been made to work anyway, it often requires at least some time doing so and you don't have time for that, do you? Not when you can buy a compatible WiFi card for $20. NDISwrapper is great and all (uses XP drivers) but who has time for that? No, please don't accept this false idea that (K)Ubuntu "has a way to go" when is comes to anti-competitive; Windows-only hardware. Just get good hardware. Most of that swill will eat up your processor power, anyway. They are cheap (trick) software devices. Be aware of the **chipset** inside. If it's not clear, there's a clue. Avoid Broadcom, Belkin etc...

Worth doing then. 

Those old boxes you mentioned with the likes of 128MB of RAM would only be able to run some of the Linuxes designed for crusty old hardware (Puppy, DSL, etc) But to be honest, they aren't terribly good for a new user, and would cause unnecessary frustration.
