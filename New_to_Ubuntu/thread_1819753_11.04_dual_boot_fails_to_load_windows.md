### Post by coffeecat on 2011-08-07
The reason Windows is not booting is because you have grub installed to the boot sector of your Windows C: partition (sda1). It has no business being there. Moreover, grub is not in the mbr where it should be - you have Windows booting code in the mbr. I'm somewhat surprised (but I've learnt something new) that your system boots at all. The Windows mbr is passing the boot process to the partition with the boot flag (sda1) where I guess it finds grub which takes over. But when you try to boot Windows, Windows won't boot because the same grub files interfere with Windows' boot.ini and ntldr.

First - the only way that I can think of to get into this situation is to deliberately tell the installer to install grub to sda1. Is this what you did? The default is to install grub to sda (to the mbr of the hard drive). Did you have a reason for over-riding the default, if that is what you did?

A fix for this is a two-stage process:

1 - boot from the Ubuntu live CD or USB and re-install grub, but this time to the mbr.

2 - boot from a Windows XP installation CD and repair the Windows boot sector in the Windows C: partition. 

Before I describe how to do this, do you have a Windows XP install CD? It has to be a proper Microsoft one, not a manufacturer's recovery CD. If you don't I can give you a link to do this from an Ubuntu CD using testdisk.

First thing - boot up with the Ubuntu 11.04 CD and click on "try Ubuntu" to get the live desktop. You don't want to start the installer! Open a terminal, and run these two commands:

```
sudo mount /dev/sda6 /mnt
sudo grub-install --root-directory=/mnt /dev/sda
```

That will re-install grub to the mbr, but won't have fixed the Windows problem. Close the terminal and reboot. Make sure that you can still boot into Ubuntu on your hard drive.

If booting into Ubuntu is OK, then restart with the Windows XP disc. Where prompted, press 'R' for the repair console. I'm going from memory here, but whatever you need the repair console which is an old-fashioned command line interface. You need to run the command "fixboot". Or it may need "fixboot C:". You might find this link useful:

[http://support.microsoft.com/kb/314058](http://support.microsoft.com/kb/314058)

Whatever you do, **don't** run fixmbr. That will simply undo what you just done with the Ubuntu CD. If you find you can't get anything to work with the Windows CD, this is an alternative method for repairing the Windows boot sector using an Ubuntu live CD and testdisk:

[http://sourceforge.net/apps/mediawiki/bootinfoscript/index.php?title=Boot_Problems:Boot_Sector](http://sourceforge.net/apps/mediawiki/bootinfoscript/index.php?title=Boot_Problems:Boot_Sector)

You need to enable the Universe repository in the live session before you can install testdisk.

Once you've (hopefully) repaired the Windows C: boot sector, restart the computer and see if Windows boots. Your grub.cfg Windows entry looks OK so you shouldn't need to run update-grub, but if there are still problems with Windows, boot into Ubuntu and run:

```
sudo update-grub
```

And if there are still any problems, post back. But please post a link to that tutorial. Installing grub to the boot sector of a Windows partition is serious and needs investigating.
