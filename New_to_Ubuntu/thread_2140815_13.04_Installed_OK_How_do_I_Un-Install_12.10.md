### Post by GregA on 2013-04-30
Well, you don't want to format NTFS because that's the default Windows file system.   So, it is intended to use with Win XP, 7, 8, etc., not Linux.

If you need the partition space from the 12.10 system, you can use gparted to reformat the partition to ext4 - - which is the current most favored Linux file system.  There is then a method to edit Grub2 so 12.10 won't show up anymore - - I haven't done that so perhaps someone with first hand knowledge can add more info.

Regarding swap partition, it is set up during the install process.  On most new systems, 2 to 4 gb is big enough, but it also depends on how much ram you have and how large your hard  drive is.

PS: IF you are intending to use the space on the 12.10 partition for data or files from windows and linux 13.04, then you should create a "FAT32" file system, which will be accessible from either Windows or Linux.

### Post by monkeybrain2012 on 2013-04-30
If 13.04 controls grub then you just delete/reformat the 12.10 partition with gparted. Boot from a live CD, start gparted and locate the device (your hard drive, sda usually) , unmount all the partitions and then delete/reformat the 12.10 partition. If the 12.10 partition is adjacent to the 13.04 one you may be able to get the extra space by expanding the 13.04 partition  (take a screenshot of the layout  and post it here, the boot order doesn't always reflect the physical locations) If they are separated by Win7 then this cannot be done.

If you lose your swap you can recreate it with gparted.

When all is done boot to 13.04, open the terminal and run 
```
sudo update grub
```

  Edited :By thus changing your partitions and moving swap, hibernation may stop working. To fix this see this thread
[http://ubuntuforums.org/showthread.php?t=1768616](http://ubuntuforums.org/showthread.php?t=1768616)

### Post by fantab on 2013-04-30
Since its all booting fine, I suggest you leave 12.10 as it is. That is, if you don't have a dire need of the that space which 12.10 occupies. You can reclaim that space by just 'reformatting' that 12.10 partition by using the utility 'DISKS' from 13.04. 

Or when a new version of Ubuntu releases, you can install it on the 12.10 partition and have both 13.04 and the 'new version'. The older version can serve as your 'fall-back' ubuntu. 

No, you don't need a larger SWAP, 2-4GB is plenty. However, if you like to use "Hibernate" then you must have SWAP equal to or greater than the size of your RAM in GiB, not GB.

Why do you want to partition your NTFS? You don't have do anything else if all is booting fine. (Just make note for future that, ONLY use Windows tools to manage your NTFS partitions, it not recommended to use Linux tools to tweak Windows partitions).

My two cents...

I also thought the original poster could "uninstall Ubu 12.10" by overwriting it with a new Ubuntu 13.10 or the next LTS version (14.04?).   But as far as file types, FAT32 is limited to 4gb for "discreet files", but not for the volume (e.g., the partition).   So, if someone has a lot of audio files, they're still OK to use FAT32, but, if you're downloading lossless video, it is possible to exceed 4 gigabytes for some movies.    NTFS is a good choice IF individual file sizes can approach or exceed the 4gb limit, otherwise, FAT32 is the most common file system overall (with many utilities for file recovery, etc.)  BTW,, my "64 gb Lexar USB" is FAT32 for those same reasons.
