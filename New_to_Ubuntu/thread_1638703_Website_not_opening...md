### Post by mastercomputers on 2011-02-12
I have just done a completely fresh install of Ubuntu 10.10 Maverick Meerkat just a few days ago and was facing problems with some websites not opening while others worked perfectly fine.

 A lot of people may not understand your problem unless they actually experience it but the issue for me was DNS related.

Here's some ways you maybe able to fix it, this solution worked for me.

First I ran nslookup (I prefer dig but nslookup gives a simpler result), to test if my selected DNS server was resolving the names into IP addresses.

```
:~$ nslookup ubuntuforums.org

[B]Server:        10.1.1.1
Address:    10.1.1.1#53[/B]

Non-authoritative answer:
Name:    ubuntuforums.org
Address: 91.189.94.12

```Well it seems DNS is resolving the name to IP address but look at the Server/Address I have put in** bold**. The address relates to my router and sometimes it is not reliable.

So what we want to do is change that Server/Address to an actual DNS server, I use the DNS servers provided by opendns.com but you can also just use your ISP DNS server.

**UI Method (Must be a "sudoer")**:
System>Preferences>Network Connections
Select the tab that relates to your network connection. Mine is Wired as my network runs off cables.

These instructions may not be the same for you now but the methods should be similar.

Under the tab Wired, I have a name Auto eth0, select it and then click Edit...
Select the IPv4 Settings tab
In the method drop-down, select Automatic (DHCP) addresses only
In the DNS Servers entry field, which is now enabled, put your DNS server in.

Mine looks like (using [www.opendns.com]("http://www.opendns.com") dns servers):

```
DNS Servers: 208.67.222.222,208.67.220.220
```Click apply, then close it. Then run the nslookup command again.

If the Server/Address has changed, test your sites out and hopefully it'll now work for you.

[B]Terminal Method (faster):

[/B]Using any editor of your choice, do:

sudo vim.tiny /etc/resolv.conf

and add these lines to the file

```
nameserver 208.67.222.222
nameserver 208.67.220.220
```I hope this helps you out, if not, try digging further into the DNS issue.

If you do have success with it, it would be nice to get a feel of how many it worked for.


Cheers,

### Post by williamts99 on 2011-02-12
Please before you use OpenDNS, educate yourself and see what the pros and cons of using it are.

[http://en.wikipedia.org/wiki/OpenDNS](http://en.wikipedia.org/wiki/OpenDNS)

From wikipedia:
Privacy issues, conflicts and Google redirection

While the OpenDNS name resolution service is free, people have complained about how the service handles failed requests. If a domain cannot be found, the service redirects users to a search page with search results and advertising provided by Yahoo!. A DNS user can switch this off via the OpenDNS Control Panel but will lose content filtering ability. This behavior is similar to that of many large ISPs who also redirect failed requests to their own servers containing advertising. [20]

In 2007, David Ulevitch explained that in response to Dell installing "Browser Address Error Redirector" software on their PCs, OpenDNS started resolving requests to Google.com. Some of the traffic is handled by OpenDNS typo-correcting service which corrects mistyped addresses and redirects keyword addresses to OpenDNS's search page, while the rest is transparently passed through to the intended recipient.[21]

Also, a user's search request from the address bar of a browser that is configured to use the Google search engine (with a certain parameter configured) may be covertly redirected to a server owned by OpenDNS without the user's consent (but within the OpenDNS Terms of Service).[22] Users can disable this behavior by logging in to their OpenDNS account and unchecking "OpenDNS proxy" option.[23] Additionally, Mozilla users can fix this problem by installing an extension[24] or by simply changing or removing the navclient sourceid from their keyword search URLs.

This redirection breaks some non-web applications which rely on getting an NXDOMAIN for non-existent domains, such as e-mail spam filtering, or VPN access where the private network's nameservers are consulted only when the public ones fail to resolve.

### Post by mastercomputers on 2011-02-18
Other issues I've noticed is that with older routers that don't support IPv6, it sometimes returns an incorrect DNS server address, sometimes 10.0.0.1 but you usually need to run wireshark or similar packet sniffer on your connection to see what packets are actually being sent and what address is being used.

Wireshark should hopefully give you a better understanding of what's actually happening over your connection though when you try to connect to the sites that don't load.

You could possibly turn DHCP off on your router and manually setup your connection, turn DNS relaying off on the router and see if that works.

Disabling IPv6 may resolve the issue too but there's so many different things that have to be altered to disable it, just make sure you backup everything you change incase you need to revert back.
