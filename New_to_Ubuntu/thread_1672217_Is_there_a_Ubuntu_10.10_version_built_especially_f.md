### Post by Vaphell on 2011-01-20
optimized for intel and amd processors? as in i386 vs amd64?
if so then it's not an optimization but the 32bit vs 64 bit architecture.
Majority of modern processors are compatible with both options so it doesn't really matter who made the CPU.
Intel made 32bit x86 so commonly it's named after intel, but amd came up first with the 64bit version of x86 so it's amd64. 64bit version is slightly faster, uses slightly more memory and supports more ram out of the box (32bit limitation is 4GB). Also some programs can be troublesome in 64bit, especially proprietary ones like skype or flash so when you have moderate amounts of ram and prefer things that just work, you should rather stay with 32bit.

[http://www.ubuntu.com/desktop/get-ubuntu/download](http://www.ubuntu.com/desktop/get-ubuntu/download)
in the dropdown select 32bit (for i386) or 64bit (for amd64)

### Post by srs5694 on 2011-01-20
Vaphell's description is technically correct; however, I disagree with his conclusion. IMHO, the speed boost of running 64-bit code, although modest, is worth having. The biggest problem with the 64-bit version is support for a small number of programs, Flash being the biggest one. I don't happen to use any of them (I shudder just thinking about Flash, and if a Web site requires it, I don't visit that Web site), but my understanding is that 64-bit versions of most of these programs are available. It's also possible to run 32-bit versions of most of these trouble programs on 64-bit computers, so even if a 64-bit version is unavailable or buggy, it's usually not that big of a deal.
