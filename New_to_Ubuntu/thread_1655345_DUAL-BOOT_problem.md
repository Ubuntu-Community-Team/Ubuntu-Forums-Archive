### Post by wilee-nilee on 2010-12-29
> **MongoDB said:**
> I have bootable usb, because i don't have a cd/dvd-rom.

So first we need to make the sdb1 partition active sda1 has the boot flag it should be on sdb1. Boot a Ubuntu live cd on a thumb open gparted right click sda1 remove the flag, then right click sdb1 and add it. Gparted has a dropdown window in the top right corner for HD's.

Then reboot to the bios and put the sdb drive first to be read, then boot the install W7 thumb and run the autorepair 3 times again.

So if this fixes it windows will boot directly in if sdb is first in the bios so reboot after the repair and see if it does. f your in then reboot to the bios again move sda first boot into Ubuntu and run.
```
sudo update-grub
```

### Post by wilee-nilee on 2010-12-29
When you installed W7 after XP W7 put its boot in XP so when you removed it you removed the highlighted portion that should be in your script, but is missing. You could try just running from the W7 repair command line.
```
bootrec.exe /fixboot
```
then
```
bootrec.exe /fixmbr
```

This will hopefully put the boot files in and reload the MS bootloader to the sdb mbr.
