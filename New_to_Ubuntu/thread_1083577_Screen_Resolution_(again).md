### Post by AnimalMagic on 2009-03-01
I've posted elsewhere regarding an issue with updates not working, which I have not been able to resolve. In order to get around this, I decided to install a fresh disk and clean install of Ubuntu 8.04.2. I have another identical system running Ubuntu 8.04, which I think was the original release (8.04.0?) and which has been updated weekly. 

Ever since first installing Ubuntu 7.04, I have had issues with the screen resolution. The relevant configuration is:

Dell Optiplex 745 SFF
Intel 965 on board graphics
Dell 2007WFP widescreen 1680 x 1050
D-Link DKVM-4U KVM switch for monitor, keyboard and mouse.

It was fairly easy to get the proper screen resolution in 7.04 - I think there was a proprietry driver involved. 7.10 was more difficult, but somehow after much blind faffing, I got a working system. 8.04 also difficult and I think I took an xorg.conf from 7.10 in order to get it to work properly.

I noted from another posting last week that someone mentioned that a KVM would interfer with the proper recognition of the screen resolution and indeed the resolution tends to be set at the highest for the KVM 1920 x 1600 (something like that). So I decided to attempt a fresh install with the screen connected directly to the machine. Success! Full 1680 x 1050 resolution right from the start. Unfortunately when I put the KVM back in, the machine fails to start. It comes up with the progress bar which progresses to the end, then there is a mode switch on the screen followed by a blank screen and it goes no further. If I then reconnect the screen directly, it does the same thing, i.e. blank screen and I cannot get it back to the original state.

I attempted to copy the xorg.conf off the working machine and it did seem to work, however randomly the same symptoms, i.e. a blank screen, will occur. Rebooting may or may not result in the machine starting correctly.

Well I finally got this working after searching the internet and faffing about with xorg.conf. Repeated reboots came up fine.

Switched it off overnight and the following morning progress bar came up, followed by... blank screen! :-( Reboot and it came up perfectly.

The next morning, blank screen and repeated reboots wouldn't shift. Ended up spending over an hour faffing about 'repairing xorg.conf', altering the screen back to proper resolution numerous times until it would stick. Haven't switched the machine off since.

I now have a choice of going back to a system that won't update and is slowly deteriorating in performance or spending and hour+ every morning trying to fix the screen resolution... or perhaps using my 1680x1050 widescreen in 800x600! Great stuff.

Incidentally, win xp also had problems but was easily fixed by an amendment to the driver install files and windows always comes up just fine in the same configuration.
