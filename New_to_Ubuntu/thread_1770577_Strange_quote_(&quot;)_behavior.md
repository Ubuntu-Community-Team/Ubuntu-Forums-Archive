### Post by bnovak2154 on 2011-05-29
For anyone who was experiencing this - my keyboard layout was set to USA (with dead keys) and I was experiencing the lack of a quote (") key - i.e.. hitting once did nothing, hitting twice made it look like i was getting the " but it was another character that bash etc would not recognise.

I found an old post (in a read-only forum area and you might want to try what I learned to solve this if its happening to you.

go to system->Preferences->Keyboard (or System Settings/Control center, Hardware, Keyboard)

Select the Layouts tab
Use the Type to test settings box to check your settings - if faulty,  choose Add...
From the list chose your layout (for me it was USA) - note this can be done by country or language
(a layout will display that you can test on by striking keys)

once selected, move it up to make it the default (I just removed the old one)
Retest in the type box
Close dialog and you should be all set

Hope this helps
