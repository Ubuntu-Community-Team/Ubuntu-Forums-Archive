### Post by mstipic on 2010-06-03
I have a question about laptop Toshiba L300 & GNU/Linux (including Ubuntu 10.04).

There is a problem (on all distributions!) with cpu temperature, which is higher then in Windows. At first the CPU fan wouldn't even work (turns on itself only at about 80°C and never stops), then I found tips from various websites about adding acpi_osi=Linux to kernel line, I did that and it's better now, CPU fan starts at 55°C and stops at 42°C. 

The problem is the temperature is still high, reaches 55°C much faster then in windows (where fan starts only at very high cpu load) so the CPU fan is about 1/3 of time on.
I have CPU scaling "on demand" and it works OK. I also tried various tweaks from hdd powersave and so on... Everything else like suspend works without a problem. I also updated bios to latest version.

It's a software (Linux or maybe bios) problem, because in Windows everything is quiet and cool.

Can anyone help me?

### Post by dino99 on 2010-06-03
check that the "tosh"* packages are installed into synaptic, install fancontrol and set /etc/fancontrol with info done into console by "sensors"

for example, mine is:

```
# Configuration file generated by pwmconfig, changes will be lost
INTERVAL=10
DEVPATH=
DEVNAME=hwmon0=atk0110 hwmon1=w83627dhg
FCTEMPS=
FCFANS=
MINTEMP=
MAXTEMP=
MINSTART=
MINSTOP=
```

[http://www.lm-sensors.org/wiki/man/fancontrol](http://www.lm-sensors.org/wiki/man/fancontrol)
