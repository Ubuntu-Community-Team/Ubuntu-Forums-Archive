### Post by cascade9 on 2011-01-19
You can run 3 monitors with eyefinity, but only 2 of them can be normal monitors, the 3rd has to be using Display Port- 

> How many non-DisplayPort monitors can I use with an AMD Eyefinity technology-enabled graphics card?

 You can connect up to two non-DisplayPort monitors at one time to an AMD Eyefinity technology-enabled graphics card using non-DisplayPort connections or passive DisplayPort dongles. To enable and drive 3 or more non-DisplayPort monitors at one time, the additional non-DisplayPort monitors must be connected with an active DisplayPort dongle.
[http://www.amd.com/us/products/technologies/amd-eyefinity-technology/how-to/Pages/faqs.aspx#how-many-monitors](http://www.amd.com/us/products/technologies/amd-eyefinity-technology/how-to/Pages/faqs.aspx#how-many-monitors)

*edit- you can get display port dongles. 

AM2 is pretty much dead. You can still get some AM2+ and AM2/2+/3 motherboards, but plain old AM2 is gone (unless you can find some very old sotck, which I would advise against).  

If you are only trying to decide what CPU/GPU to use, I'd list what parts you have already selected.

With windows, nVidia 7XXX (and 6XXX) cards use 'nvidia pure video', which does the same as VDPAU (moves video decoding and post-processing to the video card), which is why CPU use is so much lower with windows. 

I dont know why your GPU is getting that hot, it could well be driver related. 

AMD does have something similar to VDPAU- XvBA. Its buggy, but hopefully AMD get it working properly. Therev is also VAAPI, but I dont know much about it, I've barely touched it (from my brief play it seemed to use more CPU than a similar video would using VDPAU)

IMO VDPAU is the main advantage nVidia has over AMD now.
