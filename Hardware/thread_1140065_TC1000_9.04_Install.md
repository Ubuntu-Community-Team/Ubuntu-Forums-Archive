### Post by charon79m on 2009-04-27
I have an old Compaq TC1000 tablet and am in the process of getting 9.04 installed.  I am interested in learning from others about their expereince and could use some help in figuring out how to get the binary nvidia drivers working on it.  I also think that the netbook-remix interface would be FANTASTIC on this platform.

In order to install on this platform you'll either need a USB cdrom or to create a bootable thumb drive.  I went with the thumb drive option.

The default i386 install image for 9.04 does not work on this platform; you'll need to use the alternate install image. 

The alternate installation image works great and when complete you boot to a useable desktop, though the pen does not fuction.

You'll need to install the fpit xorg input:
```

sudo apt-get install xserver-xorg-input-fpit

```

Next you'll need to edit your /etc/X11/xorg.conf file:
(note you might want to create a backup, just in case)

```

gksu gedit /etc/X11/xorg.conf

```

Here's my xorg.cong that I've cobled together from various forum posts:
(note that I'm in the process of trying to get the binary nvidia drivers working.. they're commented out in this config)

```

Section "InputDevice"
        Identifier "pen"
        Driver "fpit"
        Option "AlwaysCore" "on"
        Option "Device" "/dev/ttyS0"
        Option "BaudRate" "19200"
        Option "MaximumXPosition" "8600" # "6250"
        Option "MaximumYPosition" "6485" # "4950"
        Option "MinimumXPosition" "154"
        Option "MinimumYPosition" "110"
        Option "InvertY"
        Option "TrackRandR"
        Option "SendCoreEvents"
        Option "ReportingMode" "Scaled"
EndSection

Section "Screen"
	Identifier	"Default Screen"
	DefaultDepth	24
	Option	"AddARGBGLXVisuals"	"True"
EndSection

Section "Module"
	Load	"glx"
EndSection

Section "Device"
	Identifier	"Default Device"
#	Driver	"nvidia"
	Driver  "nv"
	Option	"NoLogo"	"True"
	Option  "ConnectedMonitor"	"DFP"
EndSection

Section "InputDevice"
    Identifier     "Generic Keyboard"
    Driver         "kbd"
    Option         "CoreKeyboard"
    Option         "XkbRules" "xorg"
    Option         "XkbModel" "pc105"
    Option         "XkbLayout" "gb"
    Option         "XkbOptions" "lv3:ralt_switch"
EndSection

Section "InputDevice"
    Identifier     "Configured Mouse"
    Driver         "mouse"
    Option         "CorePointer"
    Option         "Device" "/dev/input/mice"
    Option         "Protocol" "ImPS/2"
    Option         "ZAxisMapping" "4 5"
    Option         "Emulate3Buttons" "true"
EndSection

Section "ServerLayout"
    Identifier     "Default Layout"
    Screen         "Default Screen"
    InputDevice    "Generic Keyboard"
    InputDevice    "Configured Mouse"
    InputDevice    "pen"
EndSection

Section "Extensions"
    Option         "Composite" "Enable"
EndSection

```

They've disabled Ctrl+Alt+Backspace as a way of restrting X... Wow, I miss that so I'll turn it back on.  NOTE I'll be editing the xorg.conf file above.  I didn't edit the xorg.conf listed above as I'm not sure everyone will want it turned back on, I just miss it.

Edit /etc/X11/xorg.conf and add the following:

```

Section “ServerFlags”
Option “DontZap” “False”
EndSection

```

Once you've got this done, reboot and you should have a working pen and a functional 9.04 install.

You might check into using longrun to help with CPU scaling for better battery performance.  

```

sudo apt-get install longrun

```

You'll also need to add cpuid and msr to your /etc/modules so they'll load at boot.

I personally like to install xournal and cellwriter as I feel they're required apps for a tablet platform.
