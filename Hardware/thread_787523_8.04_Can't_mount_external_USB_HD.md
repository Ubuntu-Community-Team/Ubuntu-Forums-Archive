# create a mount point - folder where you can see the files

sudo mount /dev/sdb /mnt/wd_drive

To explain this a bit further, basically linux uses drives by '[mounting]("http://kb.iu.edu/data/anqk.html")' it. It mounts the physical drive to a location that is identifiable by the operating system. 

So if you used the default ubuntu install, it created a [partition]("http://en.wikipedia.org/wiki/Partition_(computing)") to install ubuntu in. Once installed at boot up this partition is mounted at **/**. 

Now Hardy comes with [GVFS]("http://library.gnome.org/misc/release-notes/2.22/#sect:gvfs-gio") which lets nautilus (the file explorer) handle auto-mounting of external devices. The problem is that it is not mounting it properly. Unfortunately I am not enough of a GVFS guru to be able to tell you why, without physical access to the machine :) However the above commands should allow you to at least mount the drive. Alternatively you can also check the **.gvfs** folder in your home folder to see if it actually mounted the drive or not.

### Post by amohanty on 2008-05-12
The instructions above will not change your partition setup or existing partition map. Basically the automounter thinks that one of two things happened:
1. you have a corrupt partition and chkdisk /f simply checks the disk for errors and fixes any fixable errors, which is why it might not be showing up in windows.
OR
2. you have a hardware/software RAID in which case, you will have to mount it via other means hence the /dev/mapper instructions. 

I would strongly recommend going following the instructions. If you don't have RAID, the first option is quite easy and will get you on your way. If you have RAID, you have to do quite a bit of reading before you munge with it.

HTH
AM

### Post by rokky on 2009-03-08
In case anyone still cares:

delic.sasa, you got the response from mount that you did because you did not enter the command correctly - the spacing between operands is wrong.

You entered (if pasted in above correctly, and the error messages indicate you did):

# mount -t ntfs-3g/dev/ sdb1...

That should have been :

# mount -t  ntfs-3g  /dev/sdb1
_ntfs-3g_ is the fs type that -t specifies, and must be separated by 1 or more spaces in the mount command. 
 
_/dev/sdb1_ is the partition designation (no spaces allowed).

HTH,

> **rokky said:**
> In case anyone still cares:

delic.sasa, you got the response from mount that you did because you did not enter the command correctly - the spacing between operands is wrong.

You entered (if pasted in above correctly, and the error messages indicate you did):

# mount -t ntfs-3g/dev/ sdb1...

That should have been :

# mount -t  ntfs-3g  /dev/sdb1
_ntfs-3g_ is the fs type that -t specifies, and must be separated by 1 or more spaces in the mount command. 
 
_/dev/sdb1_ is the partition designation (no spaces allowed).
