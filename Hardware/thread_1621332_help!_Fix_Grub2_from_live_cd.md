### Post by coffeecat on 2010-11-14
> **mpnordland said:**
> I just need to get win7 back on the list, however that is not urgent.

I should have mentioned that before. While in the chroot environment, update-grub will only pick up stuff in the root partition. So now that you can boot into the hard-drive installed Ubuntu all you have to do is 'sudo update-grub' again and the os-prober will pick up Windows 7.

Glad you got the chroot sorted. Just for the record, this is the  chroot sequence I follow, courtesy of Gentoo.

1 - Boot up with the live CD and mount the partition you want to chroot into. Let's assume you mount it to /media/mountpoint.

2 - Open a terminal and change to root:

```
sudo -i
```

... or however you like to do it.

3 - If you need networking:

```
cp -L /etc/resolv.conf /media/mountpoint/etc/
```4 - Mount proc and dev:

```
mount -t proc none /media/mountpoint/proc
mount -o bind /dev /media/mountpoint/dev
```5 - chroot:

```
chroot /media/mountpoint /bin/bash
```6 - And for good measure, to remind you of where you are:

```
export PS1="(chroot) $PS1"
```
