### Post by slamMan on 2011-03-23
I found the problem. It had to do with a conflict between the nvidia driver and the cx18 module used for Hauppauge TV tuner cards.

The solution was the following:

Edit the grub bootloader and add vmalloc=256M. The conflict apparently has to do with allocation of virtual memory between the modules.

To see if you might be having the same conflict see if the cx18 module is being loaded run lsmod and look for any reference to cx18

```
lsmod
```

If you find that cx18 is being loaded then this is likely the problem and you need to add vmalloc=256M to grub

```
sudo vi /etc/default/grub
``` 

find the following line 

```
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"
```

and add vmalloc=256M

```
 GRUB_CMDLINE_LINUX_DEFAULT="quiet splash vmalloc=256M"
```

save and exit VI and then update grub

```
sudo update-grub
```

This should solve the virtual memory allocation problem and upon a reboot you should be able to start X with the nvidia drivers loaded. 

This is not my solution, After a lot of research I found reference to this solution on this forum from 2008 and thought I would repost the procedure so that anyone else having the problem would an updated post. [http://ubuntuforums.org/showthread.php?p=6315676#post6315676](http://ubuntuforums.org/showthread.php?p=6315676#post6315676)
