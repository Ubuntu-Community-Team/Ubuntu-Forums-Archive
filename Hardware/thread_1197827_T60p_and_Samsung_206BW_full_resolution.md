### Post by lyonsnet on 2009-06-26
I run Ubuntu 9.04 on my T60p (res: 1400x1050) and want to plug it into my Samsung 206BW (res: 1680x1050) when working at my desk.

I plan to close the TP display, and only use the monitor, and I want the monitor to display in it's full resolution 1680x1050.

If I select to "mirror screens" the Samsung only displays in 1280x1024. If I deselect "mirror screens" I can select the Samsung to display in 1680x1050, but then my screen is extended over both screens.

Is there a way to force Ubuntu to display in true 1680x1050 on the Samsung when the lid of the T60p is closed? Using Windows XP Pro on the T60p this is not a problem, and when I plug in my Macbook Pro to the Samsung, OS X automatically switches to the correct 1680x1050 resolution. Is this simply something Ubuntu can't do?  

Cheers,

### Post by lyonsnet on 2009-07-16
I was able to solve this by deselecting the "Mirror screens" option, which then allows the Display Preferences to detect the 1680x1050 resolution of the monitor. This sort of stretches the desktop over the screens, but I then select "Off" for the laptop monitor, and the entire desktop is displayed on the monitor in full resolution.  

I haven't tried it yet, but I imagine that before I unplug the monitor to go mobile again, I will have to select "On" for the laptop display in the Display Pref's.
