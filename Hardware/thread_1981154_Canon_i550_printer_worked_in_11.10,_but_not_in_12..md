Howdy,
I'm the maintainer of the Canon backend of the gutenprint project. I cano't tell if you are using Canon's drivers or the gutenprint drivers, but I thought I will mention something about the printer models FYI. I suspect you are using the gutenprint driver.

The i* series (e.g., i560) has corresponding models in Japan for some. These are then *i models (e.g., 560i), the difference being the addition of Hagaki and inkjet Hagaki media, and in some cases CD media.

The S* series are different printers entirely, although it may well be that the same driver is used for some S and i series, if they have a similar printhead/engine.

Since the printheads (and firmware) on the s800 and 560i/i560 are quite different, you can't possibly expect to get right results sending data to one using the format of the other. Canon printers do most of the processing using the printhead firmware. What gets sent to the printer is a set of printer commands to tell the printer what media, resolution modes, and various options are coming, and then data sent as meta data (since it is not the real ink data which will be generated by the firmware). One has to stick to the correct combinations of all of these, else the firmware will not be able to produce the right ink information.

It looks like in your case you are sending the printer inks with different bits per ink to what is expected by the firmware. The solution of course is to use the right driver!!

Why it might have worked before: only some resolution modes were supported in earlier versions of gutenprint, and these happened to be the same between completely different models of printers. If you manage to select such a mode it will still work.
