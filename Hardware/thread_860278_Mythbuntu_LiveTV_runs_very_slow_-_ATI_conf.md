### Post by arturj on 2008-11-24
Hello,

i'm running mythbuntu 8.10 on an ASUS M2A-VM with intergrated ATI X1250 (RS690 chipset). 

I experienced the same problem with latest binary ati driver from repository.

1. VERY slow live-tv playback (have not tried video playback)
2. live-tv image is double

I couldn't solve problem 1 so far (I will try TextruredVideo=on later and report), but I solved probelm 2.

My solution to 2: go to section "TV" inside Preferences of Mythtv-Frontend. On some place there are performance-profiles. The default is "CPU+" which tries to use XvMC for video playback. My idea is that ATI does not support this. I changed the profile no "normal" which resulted in a single live-tv image, still very slow

### Post by jk-menifee on 2009-01-01
I thought I'd mention I was able to fix the buffer under-run and very jerky playback by dumping the "fglrx" driver and switching to the "radeon" driver. This is done in xorg.conf. I also dumped all the fglrx-related options, of course; then restarted X by using the control-alt-backspace deal--no need to re-start the whole machine (17 days continuous running right now...) 

This change may have other ramifications--after all, the fglrx driver is supposed to be the best, right?--but seeing as how this machine is primarily a backend and secondarily a frontend/web browser, then tertially a linux sandbox, top video performance is not a priority for me.
