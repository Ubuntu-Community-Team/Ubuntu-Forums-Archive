### Post by dericke on 2013-11-10
**UPDATE**: Turned out to be a bad cable after all. It was hard to find because the cable worked flawlessly with a different TV, but I was finally able to swap the cable for another and it fixed the problem.

My HP Envy 4 Sleekbook has an HDMI port as its only video output. The graphics card is an Intel integrated card. My 20" monitor has only DVI and VGA inputs. If I use a DVI cable with a DVI > HDMI adapter on the laptop's end, multi-monitor works fine. If I use an HDMI cable with an HDMI > DVI adapter on the monitor end, things get weird with certain applications.

Generally, when playing fullscreen games at my desk, I disable the laptop's monitor (using the System Settings' Display manager). Using the DVI cable, this works fine---the game launches on the monitor, the laptop's screen is blank, and the game will detect the monitor's resolution and adjust accordingly. When I launch a fullscreen game with the HDMI cable, the monitor spontaneously is no longer recognized, and the game launches at the external monitor's resolution, on my laptop's built-in screen, resulting in an overscanned image and improper registration of pointing device input.

EDIT: I should add that the "About This Computer" function, from the Settings appindicator will trigger the same behavior, and is not a fullscreen application. This is the only other program which has caused this.

Shouldn't the cable's be identical from my equipment's perspective? They're carrying the same signal. I've tested the HDMI cable on a TV, with and without both adapters attached---no problems.

Can anyone help me track down the cause of this madness?
