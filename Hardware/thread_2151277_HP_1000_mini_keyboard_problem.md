### Post by kurt18947 on 2013-06-04
I've come into an HP 1000 mini netbook with keyboard issues.  When it boots I sometimes hear a beep soon after the desktop appears but before it finishes booting.  F10 to access BIOS settings on startup is unresponsive.  F9 to choose boot devices works intermittently.  I've tried booting a live USB and live Xubuntu CD.  I can sometimes see the external boot device after repeatedly pressing 'F9' but 'enter' doesn't work so I can't boot from an external drive.

It currently has XP sp3, long expired Norton and no password so I assume it's been run with unrestricted privileges.  The keyboard is for the most part unresponsive.  If I press the 'S' key or 'D' key I hear a tone.  No other key produces this tone.  After pressing 'S' or 'D', I can then press '3' or '8' and have the character appear in notepad.  The touchpad works perfectly, a USB keyboard behaves the same as the integral keyboard.  I did have one occurence where the machine behaved as if the 'f' key were stuck, it just kept repeating.  I was able to interrupt that then the keyboard worked as I would expect!  I wish I knew how to duplicate that :).  I assume it's a hardware problem and I need a new keyboard.  Does anyone know of Windows malware that would create these symptoms?  What's the liklihood that this is a keyboard BIOS problem (if those still exist) on the motherboard and not a keyboard problem?  Thanks for any insight or experience.

### Post by kurt18947 on 2013-06-12
Here's a follow up if anyone is interested.  I was able to download the service manual for the little box (Thanks HP!).  I figured the first step was reset the BIOS to factory defaults.  I'd never worked on a mini-box like this so it was a new experience.  I had to remove the keyboard, hard drive and top cover to access the BIOS battery.  Disconnected the battery for 30 minutes then reversed the process.  I not only got it back together but I didn't have any parts left over and it actually started up when I pushed the button!.  I was so proud of myself :biggrin:.  Resetting the BIOS didn't fix the keyboard problem though.  I'd tried a USB keyboard previously and that didn't help.  Hmmm, I wonder what would happen if I were to disconnect the integral keyboard and plug in a USB keyboard?  Heh, works like it's supposed to!    I guess the keyboard "has a few wires crossed".  Time to order a new keyboard.  And replace XP ;)
