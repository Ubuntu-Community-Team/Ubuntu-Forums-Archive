### Post by tarjebargheer on 2010-02-07
I recently accepted an upgrade to the 2.6.31-19 kernel.

I am using an Asus 1101ha, and attempted to do "New Method" described in the following:

[https://wiki.ubuntu.com/HardwareSupportComponentsVideoCardsPoulsbo/](https://wiki.ubuntu.com/HardwareSupportComponentsVideoCardsPoulsbo/)

However, with a 2.6.31-19 kernel,

# dpkg-reconfigure psb-kernel-source
[FONT=Verdana]
results in a build error. As stated in the link, the package is unstable with 2.6.31-17+. Since I have used 2.6.31-17
alongside with psb-kernel-source, I attempted a kernel-downgrade to 2.6.31-17 [removed linux-image-2.6.31-19-generic]

However, it appears that psb-kernel-source is dependent on the package linux-generic; which is 2.6.31-19 and it refuses to perform a reinstallation of the poulsbo drivers; it comes with the following error message:

The following packages have unmet dependencies:
  psb-modules: Depends: linux-generic but it is not going to be installed


In effect, I only have neanderthal graphics performance - and any help would be appreciated.
[/FONT]

---

### Post by tarjebargheer on 2010-02-07
Oh...

apparently it turned out that running a 

# dpkg-reconfigure psb-kernel-source
after reinstalling the poulsbo drivers under 2.6.31-17 fixed the issue; so now my graphics apparently works smoothly again, having downgraded to 2.6.31-17.
