### Post by AltomineUK on 2010-07-12
> **Mr_VeRiTy said:**
> I thought that propriety drivers were usually  better?

Usually..but....

Intel GMA cards are able to use open source drivers and often integrate with the whole Linux OS a lot better because all Linux distros provide a single open source driver (may vary slightly depending on which distro such as "name" etc) for all graphics chipsets that can fully accept open source drivers. Since both the driver and the OS are developed by the same open source community, you pretty much get all the features (such as 3D acceleration etc) available to you without any further actions. If the Intel GMA is  very old, then you might have some slight problems. 

  Obviously NVidia and ATi are far from this as they use proprietary drivers, which are not often developed by the Linux community for the best part, although they test some of them. These cards do run using that open source driver , but it will not support all the graphics related features provided by a Linux OS until you choose the correct propriety versions. 

Even then Ubuntu, by default only offer a very limited choice which can work with graphics chipsets. Out of these most of them are for NVidia. ATi support is a bit shady. 
If you are one of the unlucky few whose graphics cards produce problems while running any one of the choices provided by Ubuntu, then you have to go to the manufacturer's website to look for one. I don't think neither NVidia nor ATi have got round to fully supporting Linux just yet, although they do offer downloads and installing those might be "interesting" (i don't know how....).

Obviously if you are planning on using your Desktop/Laptop PC for gaming, 3D graphic designs and such stuff you might want to consider NVidia or ATi....

You could check with the others to get an idea of which GPUs from NVidia or ATi works, but at the moment proprietary drivers seems to work the best with proprietary OS.
