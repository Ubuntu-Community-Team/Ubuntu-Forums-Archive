### Post by cordel on 2008-10-28
> **gjoellee said:**
> Asus EEE is made to be simple, and that's what it's not. The repo is full of old software and you have to use the terminal a lot. Thats why I recommend to not buy it. The Asus EEE is my sister's though


If you are using the terminal then your obviously looking to do something that was not intended. Have to remember it was originally targeted at kids and teenagers in the first place. I found xandros very simple, just lacking the tools that I wanted. I have been waiting for something similar to the 1000 to release since the first eee. I wanted a bit larger screen, at least 30GB SSD, BT, and Wireless, so the eee-1000 suited my minimum set requirements and I bought one.

Obviously I didn't want to use it as intended either, I played with xandros for about a day, works great but missing some basic toolsets that I need for work, so I said bye bye to xandros and loaded a few different distros starting with Fedora, where I just didn't like the look or feel, which surprised me since I'm normally a Redhat user. I ended up loading Ubuntu-eee which was really easy with netbootin, loved the interface, but might have to look at Arch now ;)

The down side to the 1000 is some hardware changes by ASUS that means that not everything works out of the box loading Ubuntu-eee, but all were easy to work around. Here is what I did just in case so no one has to ask.
ASUS changed the wireless card, support is already in the kernel for it, just a few scripts don't deal properly with turning the wireless card on and off and there is already a fix on array.org.


**DISCLAIMER:*** I make no promises that this will work, or even that it will not break your pretty little eee, use at your own risk and Ubuntu forums, Ubuntu-eee, or myself _can not_ be held liable if your break your eee. It's assumed you are knowledgeable enough to make your own choices and understand what is being presented. If not stop here and hire, contract, bribe your local guru.*

Install ubuntu-eee, use netbootin, makes it very simple, just follow the instructions there.

Now for the eee-1000 you need some updates to make  things work in ubuntu-eee 8.04.1:
Once you have Ubuntu installed add the array.org repo:
[I]apt [http://www.array.org/ubuntu](http://www.array.org/ubuntu) hardy eeepc
apt [http://www.array.org/ubuntu](http://www.array.org/ubuntu) hardy-proposed eeepc
[/I]
Plugin some CAT5 into the nic adapter, configure it for the net etc.
Then run *apt-get update*

Now you can turn your wireless card on and off with fn-f2 without it locking up your system, The following does not depend on the above, so really you could follow the below procedures to bring up your wireless before ever updating.
You can also turn the wireles, Bluetooth, and camera on and off through the command line in acpi but needs a root shell, values 1=on 0=off in case someone doesn't know:
Check status:
*cat /proc/acpi/asus/wlan*

Turn it on in case the return value from above is 0:
*sudo bash*   --->root shell, using sudo alone will not work
*echo 1 > /proc/acpi/asus/wlan*

You can do this with the bluetooth and the camera as well:
[I]echo 1 > /proc/acpi/asus/bt
echo 1 > /proc/acpi/asus/camera
exit [/I]    ---->get out of root shell

If none of the above makes since to you, then you probably should not be trying it.
Seems that the gui tools for wireless still have some work to be done, but I really haven't looked into it yet, so set the wireless in a command shell:
[I]iwlist scan

sudo iwconfig ra0 essid YourAccessPointSSID
sudo iwconfig ra0 channel 11[/I]

etc..

If your using DHCP:

*sudo dhclient ra0*

Install the extra tools you might need, etc. Runs great and I couldn't be happier :D
