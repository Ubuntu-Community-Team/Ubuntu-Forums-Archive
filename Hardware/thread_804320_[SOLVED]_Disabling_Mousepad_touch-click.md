### Post by troupa on 2008-05-23
in the synaptics section:

Option     "TapButton0"    "0"
Option     "TapButton1"    "0"
Option     "TapButton2"    "0"




is SUPPOSED to disable tap-to-click completely.

Mind you, I've been having major problems with this NOT sticking for some reason.  So far I've been able to only narrow it down to having something to do with KDE apps/libs.  YMMV.

### Post by kerry_s on 2008-05-23
i use->

```
Section "InputDevice"
        Identifier      "Synaptics Touchpad"
        Driver          "synaptics"
        Option          "SendCoreEvents"        "true"
        Option          "Device"                "/dev/psaux"
        Option          "Protocol"              "auto-dev"
        Option          "MaxTapTime"    "0"
EndSection

```

disables tap, but lets me still use the cursor and scroll.
