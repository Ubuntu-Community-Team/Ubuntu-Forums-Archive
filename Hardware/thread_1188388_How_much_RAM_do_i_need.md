### Post by sdennie on 2009-06-15
I expect that 1-2G would sufficient for that.  However, having more RAM is not wasted on linux as it will use unused RAM as a disk cache which makes your machine much more responsive.  This is especially noticeable over time if you use suspend/resume instead of shutdown or if you prime the caches with something like preload.

### Post by doas777 on 2009-06-15
2-4 GB. with ubuntu or XP, 2GB will do you for almost any use-case. if you are using vista you will want 4. if you never reboot your vista PC, make sure you have a 64Bit OS and get 6 or 8.

my dev box at work (vista SP2, VS2k8,SQL2k5Ent) idles at about 1.3 GB used and if i don't reboot every couple of days will sit at 3GB once outlook and a few VS instances open.

RAM is cheap so don;t get stingy, but don't exceed 4GB unless you really have a need for it. caching aside, having too much ram is like swimming in the ocean. it doesn't really matter how deep the water is, since your only ever using the top of it. 

also remember, you need as much swap as you have ram, if you wanna hibernate/suspend.
