### Post by Calpitor on 2009-01-10
Last week my laptop began to sporadically shut down but would not completely lose power. In the intervening days the time period between unexpected shut downs grew shorter to the point it won't remain in Windows for five minutes. There are no blue screen errors, no errors found Event Viewer. I swapped in a known good AC Adapter, known good RAM all to no avail. I ran diagnostics on all components and the entire system and found nothing in error. I pulled the hard drive, slaved it to another system, copied the important data off the drive. I put the drive back in removed the partitions, did a full format and reinstalled Windows. After Setup successfully copies the data off the CD then is supposed to reboot and continue with the installation the computer hangs in the middle of the reboot mimicking the original issue: no video, a spinning fan and a single LED indicating that the power is still on. I have tried different copies of XP Home, XP Pro, and XP MCE (all copies are legitimately obtained from my MSDN account) each with the same results. I then booted off of a Linux Ubuntu Live disk and the computer operates! So installed Ubuntu and the computer operates 100%. Not willing to give up I created new copies of XP Home, XP Pro, and XP MCE and the install would not proceed following the typical automated procedures I was able to complete the installation of Windows eventually by manually recycling the power at the necessary times but again once in Windows I was confronted with the original issue: sporadic shutdowns and the system would not reboot.


So my question is what is so different about Linux that it would allow a defective piece of hardware to operate normally when Windows will not?
