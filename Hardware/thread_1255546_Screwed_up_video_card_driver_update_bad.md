### Post by denver on 2009-09-01
The nVidia binary package usualy takes care of the installation process without any problems if:

- you have all necesarry headers and compilers installed

```
sudo apt-get install build-essential linux-headers-`uname -r`
```

- you did not enable the accelerated driver using the ubuntu hardware drivers wizzard
- your graphics card is supported :)

Personaly i like using the ubuntu installer. Its easy and fast, and on kernel updates you don't have to reinstall your graphics driver. You should be able to install it by going to:

System-->Administration-->Hardware drivers

Assuming your card is supported, you should see the nVidia driver in the list.

Good luck!

### Post by denver on 2009-09-01
If you installed it using the ubuntu hardware drivers wizzard you should have an "Activate" button you can press to turn it on. If it's on but not in use the check out your /etc/X11/xorg.conf.

You should have a section called "Device" that looks like this:

```
Section "Device"
    Identifier     "Device0"
    Driver         "nvidia"
    VendorName     "NVIDIA Corporation"
EndSection
```

If the Driver says "nv" you should change it in "nvidia" (make a backup of the file first). Please make sure you really do have the driver installed or X will fail to start. If that happens, you can revert back to your initial settings.

If you used the nVidia binary from the nvidia.com site, and the driver is installed on your sistem, all you need to do is run:

```
sudo nvidia-xconfig
```

and restart GDM.

Good luck!

### Post by gradinaruvasile on 2009-09-02
First, uninstall ur nvidia driver:

```
sudo /path/to/driver/NVdrivername.run --uninstall
```

Reconfigure ur X server (remake default config with the default nv driver):

```
sudo dpkg-reconfigure -phigh xserver-xorg
```

Reboot in recovery mode, select root prompt.

Install ur driver:

```
/path/to/driver/NVdrivername.run
```

no need for sudo here, u are root.
Disregard the runlevel 1 comment, its not a problem here. [COLOR="Red"]Make sure u accept to automatically modify ur xorg.conf[/COLOR] - the installer has a very capable script to autoconfigure ur xorg.conf, use it (i always used it, never had problems).

when done, reboot:

```
reboot
```
