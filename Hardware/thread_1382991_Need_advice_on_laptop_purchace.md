### Post by argosreality on 2010-01-17
If you want to make sure it works out of the box and easily make sure it has an Intel integrated graphics card as well as Intel wireless. Intel graphics are not always the fastest but it works. Their wireless is probably the easiest period to get working in my experience.

The Acer most likely is running an Atheros or broadcom chip (Atheros works relatively decently with 9.10 now but sometimes it breaks and then is a PITA) and broadcom requires using NDISwrapper to work properly.

Also, the Acer is running a single core ULV processor. Its faster than a Netbook (atom based) processor but not going to be as smooth as a dual core based model.

### Post by Ibidem on 2010-01-18
Looked there and...owch! Those are some expensive laptops!
If you're not game for a Starling (netbook), they start over $700.
The Starling wifi driver didn't work with 2.6.3x kernels, last I knew.
As far as Dell goes--"home" support is practically nonexistent, but my brother got a Vostro from their "small business" line and the support was great (screw disappeared, they said "take out the hard drive and send it to us", tech support knows what ther're doing, etc.).  But I would NOT recommend the Vostro line if you want things to work well with current Ubuntu, though: Broadcom wireless & ethernet, NVIDIA graphics...Fedora 12 was alright, but most Linux is difficult on there.
If you're wondering about Fedora and Broadcom, Broadcom finally released hardware specs (not even Atheros has done that), and the OpenFWWF project now has implemented a functional (WEP/WPA/Master/Ad-hoc/Managed capable) open source firmware that works with b43.  It's new, though.  Fedora 12 is using it, and it really works (0 tweaks required).  
If it got into Lucid somehow, that would be cool... but it sounds like a long shot.
After LTS releases, Ubuntu sometimes jumps ahead, and Fedora usually signals when stuff is fairly functional, so I hope 10.10 follows Fedora in using OpenFWWF for a more out-of-box experience with fully free software.
Sorry for the long post, 
Ibidem
