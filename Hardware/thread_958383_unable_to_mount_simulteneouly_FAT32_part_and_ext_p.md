### Post by surfed on 2008-10-25
Hmm, i dont see any entries for sda in your fstab. Also i would change partition type on large HD's with FAT32 to "c" LBA. Post the command you use for mounting and resulting error.
IE:
sudo mount -t ext3 /dev/sda7 /mnt/mountpoint
sudo mount -t vfat /dev/sda8 /mnt/mountpoint2

Also why is your hd partitioned with extended partion only (sda1 ext and no primary partitions?) and its type is f W95 Ext'd (LBA)? Extended partitions should be type 5 i believe.
If there is still no data on the disk start from scratch and use gparted to partition and format your drive to eliviate user errors.

### Post by surfed on 2008-10-25
make sure ntfs-3g is installed: sudo apt-get install ntfs-3g


edit partition type of sdb1 (yours is set to w95 LBA partition but it should be extended):

sudo fdisk /dev/sdb
press t and enter
press 1 and enter
press 5 and enter
press w and enter

you might have to reboot for the kernel to work with the new partition scheeme, not sure on this, try without first:
 
then do this: (for both partitions you want mounted)
sudo mkdir /path/to/your/desired/mountpoint1
sudo mkdir /path/to/your/desired/mountpoint2
 

add or edit relevent lines to /etc/fstab

sudo gedit /etc/fstab

/dev/sdb5 ntfs-3g /path/to/your/desired/mountpoint1 auto,rw,nodev,noexec,nosuid 0 0

/dev/sda1 ntfs-3g /path/to/your/desired/mountpoint2 auto,rw,nodev,noexec,nosuid 0 0

save, exit, type "sudo mount /dev/sda1 && sudo mount /dev/sdb5"

hope it works

### Post by surfed on 2008-10-27
you can force the mount command to mount ntfs partition even if not cleanly unmounted but its not recommended, data loss might be the result. type "ntfs-3g --help" for mount help

example forced mount for your partition:
ntfs-3g /dev/sdb5 /mnt/mountpoint -o force

but i just noticed you have your mountpoins in /media, i believe (not 100% sure) that this can be a problem as all the automounting happens there and mountpoints might be deleted in that dir. Try using a mount point in your /home or in /mnt instead.
