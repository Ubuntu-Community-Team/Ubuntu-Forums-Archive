Hello,

I'm having difficulties setting up my MCE remote for Kodi under Ubuntu 15.10. For now, it's - I think - not related to Kodi.

I've followed the guide on [http://www.lirc.org/html/configuration-guide.html#key-symbols-using-linux-input-layer](http://www.lirc.org/html/configuration-guide.html#key-symbols-using-linux-input-layer) since the IR event management is now part of the kernel. 'ir-keytable' confirms that it's OK. Moving on to the next phase, I've tried to run lircd manually (all settings under '/etc/lirc' are generated by dpkg at install time and seem OK):
```
sudo lircd --device /dev/input/by-id/usb-Philips_eHome_Infrared_Transceiver_PH00X7eo-event-if00 --driver devinput
```
Syslog indicates that used socket is '/var/run/lirc/lircd' and 'irw' command connects :
```

Feb 22 21:21:06 romanesco lircd-0.9.0[8059]: accepted new client on /var/run/lirc/lircd
```
Indeed, using 'irw' I can see messages:
```
eric@romanesco:~$ irw
00000000800100cf 00 KEY_PLAY devinput
0000000080010197 00 KEY_NEXT devinput
000000008001019c 00 KEY_PREVIOUS devinput
```
**But**, when I start lirc as a service ('sudo service lirc start'), syslog shows that it's using /run/lirc/lircd socket instead:
```
Feb 22 21:21:46 romanesco lircd-0.9.0[8110]: lircd(devinput) ready, using /run/lirc/lircd
Feb 22 21:22:11 romanesco lircd-0.9.0[8110]: accepted new client on /run/lirc/lircd
```
The new client here comes from using 'irw /run/lirc/lircd'. But now, nothing shows up in the console... systemctl gives more detailled info (a preivous capture since this one shows that the service seems to start the same command as in the manual case):

What could be wrong? Can I go one step deeper to maybe see where the messages are lost?
