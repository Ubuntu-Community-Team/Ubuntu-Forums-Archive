### Post by tgvbjhknlm on 2018-11-27
I forgot the link to the document, but I followed some blog post and installed the things as described using "apt install" (no PPA, just the default main repositories). When I executed the "clinfo" command, it did show that I have a working hardware and the current OpenCL version is 1.2. I could even compile a sample OpenCL C code.

Yet, still, enabling OpenCL in LibreOffice did not work, and Blender's System did not show OpenCL hardware. I even added "CYCLES_OPENCL_SPLIT_KERNEL_TEST=1", and it still did not work.

Finally, I discovered that AMD provides an official driver. But they only seemed to provide it for LTS. I am using Ubuntu 18.10, and the closest one was for 18.04. Since it is the same 18.XX, I thought it might be able to be installed. But the installation failed with an error message like, "This can only be installed in Ubuntu 18.04".

Anyways, it is actually possible to enable OpenCL in Libre and Blender without installing AMD's official driver? Because I do not want to use the same LTS for two years.
