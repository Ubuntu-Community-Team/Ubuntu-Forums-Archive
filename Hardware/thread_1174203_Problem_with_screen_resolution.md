### Post by Yellow Pasque on 2009-05-31
> **Greensystemsgo said:**
> Did you install the appropriate driver? Is it maybe restricted?
The SiS driver isn't restricted. Most likely, the Xserver isn't reading the display's EDID properly and setting a virtual screen size that's too low.

```
gksudo gedit /etc/X11/xorg.conf
```

Add this block of code to the Screen section
```
Subsection "Display"
    Depth 24
    Modes "1280x800"
    Virtual 1280 800
EndSubsection
```

### Post by Yellow Pasque on 2009-06-05
1. Extract the files from the .tar.gz you downloaded
2. Move them to */usr/lib/xorg/modules/drivers*
3. Add the following to your device section in */etc/X11/xorg.conf*:
```
Driver "sis671"
```
4. Reboot your system (or restart gdm/Xorg)

### Post by osx on 2009-06-30
I was having the same problem, but for me using a VGA cable was the fix. My monitor has a DVI and VGA port, but my nvidia graphic card only has DVI ports. So, I removed the DVI cable, attached the VGA cable to the VGA port on the monitor, attached the VGA-to-DVI adapter that came with my video card to the VGA cable, attached that to my video card and have had the full 1680x1050 resolution ever since without having to tweak my xorg.conf file. I'm using a Viewsonic VX2025wm monitor (for some reason, the Viewsonics seem to be the most problematic in regards to EDID data).
