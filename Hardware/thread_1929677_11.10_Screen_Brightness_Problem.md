### Post by AllFallD0wn on 2012-02-22
I know what you're thinking "same brightness problem that's been asked a million times", well, it's not.

Basically, I've installed the latest release (downloaded this afternoon) onto my old netbook (Samsung N130? (I think)), and if I unplug the power lead, and adjust the screen brightness (sometimes I don't even have to adjust the brightness), the brightness will start to flicker between (seemingly) random values until I do a reboot.

This is incredibly annoying, and renders it pretty much unusable. (N.B it also does this if the power cable is in, and I adjust the brightness). Just wondered if there was a work-around for this? It actually takes up a lot of processing power, too, so it lags out a lot (presumably it's changing the screen brightness 30 or 60 times per second..


EDIT: Ok, after a lot more searching, this worked:

[LIST]
[*]Open a terminal
[*]Navigate to etc/modprobe.d
[*]sudo gedit blacklist.config
[*]Scroll to the bottom, and add "blacklist samsung_laptop"
[*]reboot
[*]Voila!
[/LIST]

For people who just want the code:

```
cd /etc/modprobe.d
sudo gedit blacklist.config
```

in the blacklist:
```
blacklist samsung_laptop
```
