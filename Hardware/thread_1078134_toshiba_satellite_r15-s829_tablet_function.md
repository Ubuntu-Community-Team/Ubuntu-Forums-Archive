### Post by Favux on 2009-02-27
Sure, not a problem.

```
#!/bin/sh 

# Find the line in "xrandr -q --verbose" output that contains current screen orientation and "strip" out current orientation. 

rotation="$(xrandr -q --verbose | grep 'connected' | egrep -o  '\) (normal|left|inverted|right) \(' | egrep -o '(normal|left|inverted|right)')" 

# Using current screen orientation proceed to rotate screen and input tools. 
#   "xsetwacom set touch rotate" lines removed 

case "$rotation" in 
    normal) 
#    -rotate to the left 
    xrandr -o left 
    xsetwacom set stylus rotate CCW 
    xsetwacom set eraser rotate CCW 
    ;; 
    left) 
#    -rotate to inverted 
    xrandr -o inverted 
    xsetwacom set stylus rotate HALF 
    xsetwacom set eraser rotate HALF 
    ;; 
    inverted) 
#    -rotate to the right 
    xrandr -o right 
    xsetwacom set stylus rotate  CW 
    xsetwacom set eraser rotate CW  
    ;; 
    right) 
#    -rotate to normal 
    xrandr -o normal 
    xsetwacom set stylus rotate NONE 
    xsetwacom set eraser rotate NONE 
    ;; 
esac
```

### Post by Favux on 2009-03-18
Hi bill_riley,

Welcome to Ubuntu!

To get root priviledges in the terminal you type "sudo" then it asks for your password.  If you're trying to launch a gui program like gedit it's better to use "gksudo".  The xorg.conf lives at /etc/X11.  So to edit it you could use:
```
gksudo gedit /etc/X11/xorg.conf
```
Be very careful with xorg.conf.  Back it up first with the "cp" (copy) command like:
```
cp /etc/X11/xorg.conf /etc/X11/my_xorg.bak
```
or whatever name you want.  And to replace the new xorg.conf with the backup:
```
cp /etc/X11/my_xorg.bak /etc/X11/xorg.conf
```

To find out more about a command using "man commandname" often gives information.  Ctrl-z (usually) to exit. 

Good luck!
