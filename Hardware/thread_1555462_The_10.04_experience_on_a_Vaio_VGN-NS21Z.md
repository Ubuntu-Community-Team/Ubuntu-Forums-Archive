### Post by NikoC on 2010-08-18
My first Ubuntu release was Hoary Hedgehog (I think in 2005) on a sony vaio 13 inch and I have to say it was quite a challenge in those days to get all the hardware working. Since then I've tried out all Ubuntu releases (and other distros as well) on either laptop or desktop with hardware support gradually improving all the way.

Now, the lab I work in, had a spare Sony Vaio VGN-NS21Z available (or PCG-7151M as mentioned on the tag on the bottom casing). I immediately installed 10.04 (flawless install and the experience has been smooth ever since). Most hardware is working out of the box, though there were some minor problems which I've managed to solve by doing some digging. 

What follows is a hardware summary. Hope some of you Vaio owners find some inspiration in the workarounds I compiled from different sources! Let me say that by no means I am an expert on either testing or computer vocabulary, but this is what I came up with.

_Sound:_
Out of the box (headphone jack also tested)

_CPU and chipset:_
Out of the box


_Webcam (Ricoh Co., Ltd)_
Out of the box (tested with Cheese)

_Wireless_
Out of the box


_Memory card readers:_
SD: out of the box
Memory stick: untested

_Suspend/Hibernate:_
Out of the box (both lid close for suspend and power button for suspend or hibernate; fn+f7 results in hibernate instead of suspend)

_Video + external monitor:_
Out of the box (did not install any extra drivers, using the ones that came with the standard installation)

_DVD:_
Out of the box

_Function + key/programmable keys:_
Sound mute: created new command in System > Preferences > Keyboard 
Name: Audio mute
Command: amixer sset Master toggle
Apply
Click on the shortcut in the new line added and then press the Sound mute button
It doesn't show an indicator on the screen though (cfr. fn + F2/F3/F4)

Haven't figured out the AV mode button yet (but fn + F7 works fine)

Functions keys for sound (F2,3 and 4): out of the box

Brightness keys F5 and F6: did show an indicator on the screen, but nothing happened to the brightness (brightness could be adjusted by gnome settings though), managed to fix this by typing in the command line:
lshal |grep system.hardware.vendor
> system.hardware.vendor = 'Sony Corporation'  (string)
lshal |grep system.hardware.product
> system.hardware.product = 'VGN-NS21Z_S'  (string)

Copy/paste the following at the end of the file (you need the 2 strings from above):
sudo gedit /usr/share/hal/fdi/information/10freedesktop/10-laptop-panel-hardware.fdi

<match key="/org/freedesktop/Hal/devices/computer:system.hardware.vendor" string="Sony Corporation">
 <match key="/org/freedesktop/Hal/devicesnc@nc-laptop:~/Documents$ /computer:system.hardware.product" contains_outof="VGN-NS21Z_S">
   <!-- needed since the acpi video module reports it handle the events, but it don't ATI Technologies Inc Device 95c2
work on this machines-->nc@nc-laptop:~/Documents$ 
          <merge key="laptop_panel.brightness_in_hardware" type="bool">false</merge>
        </match>


External monitor key also works (fn + F7)

fn + F9/F10 don't (the ones with the magnifying glass) work, but I never use them anyways

fn + F12 works, but results in hibernate instead of sleep; I use the power button for suspend mode (wakes up without problem; same for hibernate, though a bit slow) or just close the lid to suspend

_Touchpad:_
Out of the box
Installed multitouch support with script running at login:

gedit touchpad.sh
copy/paste the following and save

#!/bin/sh
#
# Use xinput --list-props "SynPS/2 Synaptics TouchPad" to extract 
data
#

# Set multi-touch emulation parameters
xinput set-int-prop "SynPS/2 Synaptics TouchPad" "Synaptics Two-Finger Pressure" 32 10
xinput set-int-prop "SynPS/2 Synaptics TouchPad" "Synaptics Two-Finger Width" 32 8

# This will make cursor not to jump if you have two fingers on the touchpad and you list one
# (which you usually do after two-finger scrolling)
xinput set-int-prop "SynPS/2 Synaptics TouchPad" "Synaptics Jumpy Cursor Threshold" 32 110

Added sh touchpad.sh to startup session: tapping with two fingers equals right mouse click (also two finger scrolling works, but I prefer edge scrolling)

edit: recently installed gpointing-device-settings to enable locked drags (tap to release a selected file or text) which I prefer
