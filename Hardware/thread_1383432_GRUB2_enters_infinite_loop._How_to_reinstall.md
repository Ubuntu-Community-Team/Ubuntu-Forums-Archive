### Post by oldfred on 2010-01-17
I think you need to have both drives plugged in to make sure your device.map is correct. Otherwise each drive thinks it is sda and of course only one can be. You may have to manually update device.map if it already is not set correctly. Hd0, hd1 should be the order the drives are installed, you can still boot from the other one.

if you are in your system, if liveCD you have to add the mount to the front of the location.
gksudo gedit /boot/grub/device.map

How to restore the Ubuntu/XP/Vista/7 bootloader (Updated for Ubuntu 9.10)
[http://ubuntuforums.org/showthread.php?t=1014708](http://ubuntuforums.org/showthread.php?t=1014708) 
Reinstall grub
[https://help.ubuntu.com/community/Grub2#Reinstalling%20GRUB%202](https://help.ubuntu.com/community/Grub2#Reinstalling%20GRUB%202)

### Post by Nugar on 2010-01-17
Hi oldfred,

Right now, I'm just to be on the safer side, backing up again the contents of the disk.

After reading your post, I checked device.map on the disk I installed the OS and sure enough, this is what it said:

```

(hd0)    /dev/sda
(hd1)    /dev/sdb

```Here, sda is my 1tb disk and sdb is the 320gb disk, where I installed the OS. So it seems that GRUB is trying to boot from sda.

I changed device.map to:

```

(hd0)    /dev/sdb
(hd1)    /dev/sda

```I assume it will try to boot from sdb now. In about 15mins from posting this, I'll be finishing the backup and will try to boot using this device.map. I'll get back to you then.

But I have a feeling that this will work, so thanks!
