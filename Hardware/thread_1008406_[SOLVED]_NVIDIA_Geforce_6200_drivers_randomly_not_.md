### Post by Lupusceleri on 2008-12-11
Hey there,

I'm using an Asus eXtreme N6200GE graphical card, from Nvidia.

As of a few days ago, I got the drivers to work properly, so I could go for a resolution higher than the stupid 1024x786: my good ol' 1280x1024. The trick was to get the newest version from the Nvidia website (177.82) instead of grabbing the one out of repos (177.80), then letting it compile into my kernel and configure xorg.conf itself.

However, sometimes after I change a setting (IE I modify Fstab.. something TOTALLY unrelated to the videocard!), the drivers suddenly bug out. I get to work with a 1024x786 resolution only. Last times, I fixed this by going back into recovery mode and letting the driver reinstall itself (thereby also overwriting xorg.conf which is good, because that fixed it).

This time it didn't work. Right now, I'm trying to put an auto-backup'd xorg.conf back, I'll let you know how it works out, but I'm still very curious about the reason for the sudden driver failures.

EDIT: Fixed. After fiddling around with /etc/X11/xorg.conf this was the working one:

```
Section "Device"
    Identifier     "Device0"
    Driver         "nvidia"
    VendorName     "NVIDIA Corporation"
EndSection

Section "Monitor"
	Identifier	"Configured Monitor"
EndSection

Section "Screen"
	Identifier	"Default Screen"
	Monitor		"Configured Monitor"
	Device		"Device0"
EndSection
```

### Post by Lupusceleri on 2008-12-11
Fixed it, was indeed an issue with xorg.conf

I think the problem was that somehow the following pieces of code got deleted from my xorg.conf:

```
Section "Monitor"
    Identifier     "Monitor0"
    VendorName     "Unknown"
    ModelName      "Unknown"
    HorizSync       30.0 - 110.0
    VertRefresh     50.0 - 150.0
    Option         "DPMS"
EndSection

Section "Screen"
    Identifier     "Screen0"
    Device         "Device0"
    Monitor        "Monitor0"
    DefaultDepth    24
    SubSection     "Display"
        Depth       24
    EndSubSection
EndSection
```

Made sure I've got that file backed up now so that I can replace it whenever it bugs out. :)
