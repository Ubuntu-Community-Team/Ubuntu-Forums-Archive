### Post by Yellow Pasque on 2009-10-08
X is probably trying to load the open-source X.org version of libglx.so
Nvidia uses a proprietary version of this file. Search your filesystem for libglx and make sure you have one from the nvidia driver.

You may have to add a section like this to xorg.conf to point X to the nvidia version:
```
Section "Files"
ModulePath "/usr/lib/xorg/modules/extensions/nvidia"
ModulePath "/usr/lib/xorg/modules"
EndSection
```

### Post by hellblazer on 2009-10-14
I found a slightly less hardcore solution to this problem:

When I installed the new NVidia driver, it created a whole bunch of symlinks to driver files, however when I tried to reinstall the old driver, these symlinks weren't updated.

My solution was to search for all file names containing the driver version I tried to install: 185.18.36 in my case.

In every directory where these files occurred, the older versions of the files where still there, but they where no longer linked as the newer versions changed the symlinks.

So I manually changed all the symlinks to point to the old files again and hey presto all problems gone :D

Hope this helps someone, I've been a bit vague so feel free to post for more info
