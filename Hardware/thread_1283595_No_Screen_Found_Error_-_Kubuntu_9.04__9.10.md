### Post by noookie on 2009-10-05
Hi,

I've installed Kubuntu 9.04 on my Sony Vaio VGN-C2Z notebook with GeForce Go 7400 GPU a few days ago after having tried it for a while as live-OS. When I was using it as live-OS it automatically configured the resolution correctly, but when I tried to install it it didn't start the X-server, so I had to use the alternative graphics mode.
Now it runs and starts the X-server but only as long as I use the VESA driver, as soon as I try to use nv, nouveau or the proprietary Nvidia driver, the X-Server won't start telling that no screen was found. On the other hand, as long as I use VESA I can only use 800*600 oder 1024*768 on a display natively supporting 1280*800...
I tried a lot of different things but up to now, nothing worked. I think that the reason for my problem is a misconfiguration of my monitor, the log says something like "display found but cannot be used".
I was hoping that Karmic would propably do better, so I did an update to 9.10 beta today, but still the same problem occurs.

I hope someone has any idea, as I already posted my problem in two big german Ubuntu communities without finding a solution to it. Thanks in advance! :-)
