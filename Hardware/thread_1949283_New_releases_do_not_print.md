### Post by sparks40 on 2012-03-29
Over the past few days I've been testing a few distros to see how things are going in the world of Linux... and I found some interesting results. There appears to be a problem with Linux printing in some of the current crop of distros. Specifically,the printer set-up program does not find the proper driver for the printer it identifies... and there seems to be no way to correct this through the printer applet. I have attempted to install  

Fuduntu (most current version)
Lubuntu (Beta 2)
Mint LMDE (current)
and Peach (current)

with little in the way of luck regarding the printing problem.  I've installed the HPLIP application in at least two of the above, but that did not seem to help. This is a show stopper for me and I would assume for many others. My research uncovered a few "solutions" suggested by others, but they did not work. The root seems to be in the way the distro attempts to contact cups. For the record I am using an HP Laserjet 1200 which has been most reliable and is currently the default printer on my Mint 12 LXDE set-up... and that works. I wanted to continue with Ubuntu/Lubuntu but the reality is "no printing means no work done."

Secondly trying to install Skype on a few of the above mentiond distros has become a nightmare. After installation, or some part thereof, one os has a hard time finding the application... and it just hangs. In those os's that do allow for full installation the auto install feature of Skype does not seem to be working. Moreover, my external mike is usually set manually via the audio mixer. However, some distros have done away with the inclusion of the gui based mixer.

I'm sure that Linux in general will get its hands around the printing problem as it has been a long time since I had to spend so much effort troubleshooting such a basic computing function. Seems like this happened once before.

If you have any suggestions re the above I would be delighted to hear them.

TNX
