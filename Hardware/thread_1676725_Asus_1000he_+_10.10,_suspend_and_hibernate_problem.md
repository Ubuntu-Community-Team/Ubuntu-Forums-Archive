### Post by amra.sonntag on 2011-01-27
I had issues like that on my acer laptop as well. I found a guide suggesting uswsusp. This gives you the commands s2ram (suspend) and s2disk (hibernate). They actually won't work just from installing - but you can at least try if s2ram works, by using it in a terminal. If it complains, set the -f option (force).

If you can suspend with that - i'll post the changed pm-utils file, that you will probably need to make it work automatically.

### Post by amra.sonntag on 2011-01-27
Nice, now you only need to replace the content of /usr/lib/pm-utils/bin/pm-action with the following:

```

#!/bin/sh
export STASHNAME=pm-suspend
export METHOD="$(echo ${0##*pm-} |tr - _)"
RETURN=0
if [ "$(id -u)" != "0" ]; then
    echo This utility may only be run by the root user. 1>&2
    exit 1
fi
case "$METHOD" in
    suspend*)     /usr/sbin/s2ram --force ;;
    hibernate)     /usr/sbin/s2disk ;;
    *)        echo "Cannot happen.  Please file a bug against ( ... modified ;) ) pm-utils."
            exit 1 ;;
esac
exit $RETURN

```Make sure to make a backup of the original file. And yes, you will need sudo to mess with those files. I actually have the info from a german wiki [http://wiki.ubuntuusers.de/uswsusp](http://wiki.ubuntuusers.de/uswsusp) - just to give those guys credit! I hope everything works out for you.
