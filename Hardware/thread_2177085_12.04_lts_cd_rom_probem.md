### Post by slickymaster on 2013-09-27

~$ sudo lshw -short | grep cdrom
/0/3/0.0.0      /dev/cdrom  disk        DVD reader

```and with that you can afterwards create a mount point for the device.

### Post by slickymaster on 2013-09-27
**lshw** stands for Hardware Lister and is a command that gives a comprehensive report about all hardware in your system. This displays detailed information about manufacturer, serial number of the system, motherboard, CPU, RAM, PCI cards, disks, network card etc...

When **lshw** is executing the only output you'll get in your terminal will be an indication of what particular "part" your computer he is reading/analyzing in order to build its report. It might take a few seconds for him to go through each of this "parts", from DMI to CPUID to PCI (sysfs), etc. But in the end it should display its report in the terminal.
sudo lshw -short | grep cdrom > report.txtsudo lshw -short | grep cdrom > report.txt
It may be hanging for some reason while reading PCI (sysfs) which is not good news because it's sysfs that provides access to PCI resources such as your USB cdrom. Just to be sure if it is running correctly, or not, try to generate a .txt file from the output of the lshw command. In the terminal run the following:```
sudo lshw -short | grep cdrom > report.txt
```If you end up with a report.txt file on your home folder than everything went as expected and you can paste the its output here.
