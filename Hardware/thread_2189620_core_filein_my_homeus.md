### Post by steeldriver on 2013-11-23
A core file is a dump of the working memory of a process at the point that it terminates abnormally - for example if I create a little C program that writes to memory without allocating it first (a segmentation fault)

```

$ cat makedump.c 
int main()
{
  char * array;

  while (1)
  {
    *array++ = 'X';
  }

  return 0;
}

```

then compile and run it

```

$ ./makedump 
Segmentation fault (core dumped)
$ 
$ ls -l core
-rw------- 1 steeldriver steeldriver 200704 Nov 23 13:07 core

```

Two important things to note are (1) the core gets dumped into the working directory of the process; and (2) the maximum size of the dumped core file is set by the shell's 'ulimit' parameter which by set to zero by default in Ubuntu

```

$ ulimit -c
0

```

For these two reasons, any core file you find in /home/user is very unlikely to have anything to do with boot-time processes. If you're really interested, you should be able to see what process created the core by loading it into gdb (the GNU debugger)

```

$ gdb -c core
```
