### Post by Rob F on 2011-05-18
Hello all,

I've been with Ubuntu for 5+ years, and am a strong advocate of it. Up to 10.10 it was getting better and better and basically more or less everything worked. This went wrong with 11.04 (this is not a complaint about Unity, which I think is ok and I'll probably get used to).

My machine is a Dell Precision M2400 (NVDIA Quadro FX 370M GPU). Not sure what other hardware details are relevant, but can provide.

The issues split into two main things:

1. Fn+F7 has worked, when the laptop is connected to an external video display [e.g. VGA project] for 2+ years to cycle between (laptop screen only, laptop+external, external only). Now its broken. All it does is make the laptop screen flicker and bits of it go black (which reappear if I move the cursor over them). I *can* make the laptop output to a project via the nvidia settings application but (i) this is useless when I'm on a program at a conference and need to switch quickly (ii) I cannot make the screens mirror each other but instead have this twinview stuff with primary displays etc [may be good for others but not what I want] (iii) if I have display only out of projector and then disconnect VGA laptop screen stays blank and I have to manually power it down and restart it to get a useable machine back.

2. Speaking of powering down.. software shutdown or reboot leaves the machine in a hung state which I'm eventually forced to hard power-down. Not very elegant.

There are other minor issues, like Unity takes longer to start than Gnome 2.x used to, etc, but 1. and 2. above are the real annoyances. I do a lot of presentations and 1. is rapidly becoming a nightmare. I should stress again that *all of these things worked near-perfectly (especially the Fn+F7 display cycling) in 9.04 through 10.10*. I have seen reports of similar things to this on the web but no clear thread and no solution.

Hope someone out there has some ideas. Thanks in advance.

cheers,
Rob
