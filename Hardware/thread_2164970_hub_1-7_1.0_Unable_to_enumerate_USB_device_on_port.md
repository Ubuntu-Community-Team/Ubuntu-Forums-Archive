On power up my kern.log repeats these lines, once every few seconds, as long as the power is applied to the computer.

Aug  2 12:25:56 Lexington kernel: [14484.054001] hub 1-7:1.0: unable to enumerate USB device on port 3
Aug  2 12:25:56 Lexington kernel: [14484.309951] hub 1-7:1.0: unable to enumerate USB device on port 3
Aug  2 12:25:56 Lexington kernel: [14484.565916] hub 1-7:1.0: unable to enumerate USB device on port 3
Aug  2 12:25:57 Lexington kernel: [14484.821890] hub 1-7:1.0: unable to enumerate USB device on port 3

This problem seemed to start after changing a 4-port usb hub (usb 1.1 speed) for a 7-port hub (usb 2.0 speed). I removed the 7-port and returned the 4-port. Same problem.

Looking around, this forum, I see:

[http://ubuntuforums.org/showthread.php?t=1026023](http://ubuntuforums.org/showthread.php?t=1026023) (year: 2008)

So, while the computer's OS appears to work without problem, I would like to stop the problem. At this page:

[http://www.geekdevs.com/2010/04/solved-unable-to-enumerate-usb-device-disabling-ehci_hcd/#comments](http://www.geekdevs.com/2010/04/solved-unable-to-enumerate-usb-device-disabling-ehci_hcd/#comments)

the writer give a 2 line solution, but does this make all the USB devices USB 1.1 speed?

cd /sys/bus/pci/drivers/ehci_hcd/
sudo sh -c 'find ./ -name "0000:00:*" -print| sed "s/\.\///">unbind'

he then goes on to say (comments section):

Press Alt+F2. Type gksu gedit /etc/rc.local.

Enter the commands in text file just before the exit command.

My question is: is this safe? Will it remove USB functionality? Is the rc.local going to see this at boot time? What will disabling the ehci_hcd do besides preventing the enumeration problem?  What is the reason that this semi-bug hasn't been fixed?
