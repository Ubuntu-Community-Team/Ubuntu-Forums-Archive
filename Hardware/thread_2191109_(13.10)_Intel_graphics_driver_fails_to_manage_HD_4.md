Newer Intel seemed to need this:
acpi_osi=Linux acpi_backlight=vendor

Some other Intel chip settings:

  i915.i915_enable_rc6=1

 Force Intel Video mode as boot parameter in grub menu - change to your screen size
video=1280x1024-24@60
and if that doesn't work you can try
video=VGA-1:1280x1024-24@60



These are different brands, but you should have similar Intel settings.

### Post by ronniepinsky on 2013-12-03
It seems like the problems in the log are because of nomodeset.  While looking that up I found that nomodeset cannot be used with the Intel driver.  The problem now is that booting without nomodeset eventually turns the monitor off.  Tapping the brightness key will cause it to start flickering every 3 seconds.

Edit: Problem fixed!!!
I used this entire line but the key here is probably the acpi_osi=Linux acpi_backlight=vendor .  Remember vendor is case sensitive!!!
```

i915.i915_enable_rc6=1 acpi_osi=Linux acpi_backlight=vendor video=1366x768-24@60

```

Screen off and brightness controls work too.
