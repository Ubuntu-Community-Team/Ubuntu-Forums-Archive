### Post by vanadium on 2011-06-19
I understand that now, you have the drive of your WD NAS mounted onto an Ubuntu system. My estimate is that the file system of the NAS drive is damaged by now, i.e., the file housekeeping system is not consistent. Data recovery will not be easy at this point. Unfortunately, you likely do not have a backup of the data.

You could perform a file system check on the drive. Although this would (or at least could) make the file system healty again, it probably will decrease the chances to recover files.

To increase the chance for data recovery, you should not write any further to the drive. There is a utility photorec that is able to crawl though binary data on a drive and recover files it recognized.

This will leave you with a pile of strangely named files, with no guarantee that any of these are not corrupted. In case of audio files, you might be able to relatively easily classify and rename the files again based on audio tags, though.

You will need to consider if the lost data are worth your time and trouble. Data recovery from corrupt file systems or lost partitions is not trivial.
