### Post by hal10000 on 2009-11-15
Installing an operating system can not destroy your computer (even not if this os is windows).

Do you really see **nothing** after you turn it on? If you can't even get into your BIOS, then a main part of your computer is dead (usually graphic card or motherboard)

If you can't even hear your fan and your harddisk, then the power supply might be dead.

If you can see the boot screen and it happens after you boot into ubuntu, then it's likely to be a software problem and you may boot your live cd to see whats going on or boot the recovery mode.

### Post by Bucky Ball on 2009-11-15
Installing an OS is a fairly resource intensive activity. If something was going to go the concentrated grunt  might have been the straw. 

There is no desktop or laptop version of Ubuntu so you can't really get that wrong. There is Ubuntu Netbook Remix but that is a different kettle of fish and intended for netbooks only (can be installed while running regular Ubuntu a bit like Edubuntu add-ons). 

The desktop version uses a graphic install interface whereas the alternate version gives you a text based installation experience. Welcome to Ubuntu and the forums and hope that little mishap hasn't put you off any. It probably would have happened sooner or later. My guess is the concerted effort of the install popped its cork.

### Post by Bucky Ball on 2009-11-15
> **sparklingspecks said:**
> Try unplugging from AC and taking out the battery, then press power button. Afterwards just put battery back / plug AC in, and try turning on. (This is a solution that works with some HPs, but you might as well try.)

That is a temporary fix for a dying HP laptop. If you are having problems like this with a HP, contact HP immediately. The DV6000 series and a number of others have been recalled due to threats of legal action by the public. 

For the HP, the wireless usually goes first (although sometimes the screen) but in actual fact it is a problem with the motherboard which is on the way out (this the first sign). The HP fix is specific to that hardware issue and not guaranteed to work (and if it does reports are it will work for a few months on the outside).

I should know; I own one. I have researched this and been corresponding with HP Australia for two months. They refuse to acknowledge the problem despite the fact I have emailed them the USA HP site's recall page for the very same thing detailing the exact symptoms.
