### Post by Helkaluin on 2010-03-16
Good hello,

Laptops usually have this Fn-like key to minimise their keyboard size. On Dell Vostro 1310 in particular the usual sleep, scroll look, and SysRq buttons are set as Fn combinations on the F1-12 Function Keys row.

The problem, now, is this: they've also built the numpad as Fn combinations in the keyboard such that it's:

[7890 ; uiop ; jkl; ; m,./] for the numpad layout of [789/ ; 456* ; 123- ; 0,.+]

The problem is when I utilise the SysRq combinations for a graceful reboot, the I-combination (which tries to gracefully terminate processes) and the U-combination (which remounts filesystems as read-only) are directed towards changing the system log levels to 5 and 4 respectively. (Because the SysRq key itself requires holding down the Fn key, which turns the I and the U into 5 and 4.)

Any way to modify these Fn key combinations?

Many thanks.

### Post by m3wolf on 2010-04-08
Try this.

1) Hold down Fn and Alt 
2) press and quickly release SysRq 
3) Release Fn 
4) Press command key as required (u, i, etc)

Steps 2-4 need to be done quickly. Not necessary for r-e-s-b.

Works on my 1310 with Ubuntu 10.04 beta.
