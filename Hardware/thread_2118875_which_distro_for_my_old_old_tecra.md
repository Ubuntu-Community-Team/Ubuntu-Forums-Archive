### Post by Kixtosh on 2013-02-25
Wow, cortman! That's a great summary that you have posted in that link! The definition of age is about the best one I've ever seen!

From my own experience with the two machines in my signature, slightly better of than the Tecra that is the subject of this thread, including one Toshiba Portégé 3490CT, what I can tell you for certain (in my opinion, nonetheless) is this:

[LIST]
[*]Forget about Ubuntu, of any description. It's simply not appropriate for something this old.
[*]I would also forget about Xubuntu, unless you add the LXDE desktop (not the default Xfce), in which case it's very similar to Lubuntu in performance, but with a few extras that might be appealing. It will start up reasonably fast (possibly about 90 seconds to boot), but applications will be slow to use, and you'll have to live with a lot of lag. You don't really have enough memory for this in my opinion, so the use of swap memory is going to be painful.
[*]Lubuntu, Peppermint and other LXDE based versions will work, but again, you don't really have enough memory for this in my opinion, so the use of swap memory is going to be painful. They will be too slow to be bothered with. I've also had trouble with the newest release of Lubuntu getting lost during installation for some reason, even though this has been very rare with other versions of almost anything.
[*]Puppy will certainly work, but I would try version 4.3.1 first of all, not the newer versions. It's rather good, actually, and reasonably fast. It's also a great place to start since 4.3.1. tends to handle hardware detection rather well in my experience. Still, it's not *really* designed for hard drive installation (it's actually much slower to boot or halt from the hard drive of my newer machines than the full version of Ubuntu!). The newer versions don't seem to like my old Toshiba for various reasons, although some will work.
[*]I've been experimenting with SliTaz lately, and I can say that even the LiveCD seems to be faster than anything else, except Puppy (which is designed to load from a CD in normal use, but not need it to run)! Installed on the hard drive, SliTaz is the only thing I have tried that is as fast as Puppy, possibly faster, but I haven't figured out all that I need to get it to work on my Toshiba (the screen resolution is wrong, in particular). It does feel much more like a "full" O.S. than Puppy does to me, and it'll be my choice if I can tweak everything the way I want it.
[/LIST]
So, in a nutshell: I'd try [COLOR=DarkRed]**Puppy 4.3.1.**[/COLOR] first, from a CD, if the Tecra supports booting from a CD. If it seems happy enough, you could then exit and try the LiveCD of [COLOR=DarkRed]**SliTaz 4.0**[/COLOR] and see how it seems to run. Then choose one of the two. Puppy is most likely, in my opinion, to work closest to 100% to begin with. SliTaz is unlikely to work 100% initially, so you should be prepared for some tweaks.

If SliTaz works reasonably well to begin with, and you do decide install, you can still use Puppy normally nonetheless thereafter, with no performance "penalty". You would simply save your setup and sessions in small files on the hard drive so that your settings are stored and your boot times are improved. Puppy is possibly the only option that doesn't suffer in performance from being run from a CD, because that's what it's supposed to do (including loading completely into RAM when it starts).

 Both of these are small downloads, by the way. Puppy 4.3.1. is a little over 100MB, I think. SliTaz is just 35MB.
