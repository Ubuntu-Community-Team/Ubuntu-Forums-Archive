### Post by DDDa on 2010-05-30
Hello!

I'm starting this because I couldn't find a good install log for this notebook, and to point to a path that feels like "foolproof" to me.

What I really needed was a _step-by-step_ guide, so if this is already available, please excuse me, and remove this thread if you want. Still, I hope others won't suffer as much as I did. _If you are an experienced Linux user, then this is **definitely** not for you._ _Besides, this is not complicated at all, just extremely detailed._

System: **Lenovo X201 (3249-2EU)**.
USB CDROM/DVDROM/BLURAY? **No optical drive**.
Installation type: **USB flash drive**.
Installation media: **ubuntu-10.04-desktop-amd64.iso**

[SIZE="3"]**_What I Did under Factory OS, version 7:_**[/SIZE]

1- I used [this method]("http://forum.notebookreview.com/lenovo-ibm/144783-clean-install-guide-works-32-bit-64-bit.html") for backing up the original system. It's a step by step procedure, but I didn't try recovering yet (and don't plan to for now).

2- I downloaded the ubuntu-10.04-desktop-amd64.iso.torrent option and used [Unetbootin]("http://unetbootin.sourceforge.net/") to create the bootable media on the 2GB flash drive ("usb drive").

3- After that, I rebooted with the usb drive plugged.

[SIZE="3"]**_Rebooting and Installing Ubuntu_**[/SIZE]

1- When I see the Thinkpad screen, I repeatedly press F12 to select the boot options. It appears. :) I see the "Sandisk" thing, which is my usb drive. I select it and then it boots.

2- Select the "live" system. No special modifications needed (I see many boot options around on Google that are supposedly needed, vesa stuff etc. but _none was necessary_), and it gives you a nice GUI system with Ubuntu when it finishes booting.

3- On the desktop you can find the shortcut to install. Click it. It prompts for localization stuff, account settings, name... select everything that applies to you (very easy, even to me, so no details). I selected manual partition when asked, but you can use the automatic option. In my case, I selected everything as ext4, with default options, except that I created a /boot partition with ext2. If you do the partition step automatically, just follow along.

4- It also asks you where to install the boot loader. Here, it prompted me to use /dev/sda, putting it on the "MBR".

5- The very nice step-by-step install program is so easy that you just need to answer yes and follow along the way. Don't think too much, just follow (why? see later).

6- When it finished, just reboot. You can click the reboot button that appears, or keep trying the live system and then reboot manually (the power button).

8- When it boots, it should give you the Ubuntu system. **[COLOR="Red"]Bingo[/COLOR]**. You have your Ubuntu successfully installed. Considering that you already downloaded the ISO file on Windows 7 and backed up your Lenovo "cbd_drive", the actual install process should all take less than an hour. Much better than Windows, if you ask me.

**_[SIZE="3"]What If Anything Went Wrong?[/SIZE]_**

Did you follow the steps carefully? I hate when people ask me that, because you (and I) probably did, and it makes you feel like an idiot. That's why we have the forums. :) Maybe someone will know the answer (probably not me). But I have some things that potentially can go wrong, because they did go wrong with me:

**- Use the desktop CD, not the alternate. Boot the live environment and install from there.**

Lesson learned: as a long "tester" of Linux/Ubuntu, I always used alternate disks, tweaked this and that. This time, I tried the alternate amd64 and it failed miserably... It seemed like one problem would give me another endlessly. So many things that it's not even worth to comment here (from no X installed to GRUB going to the wrong device ruining my USB drive).

**- After installing, remember to remove the usb flash drive from the usb port**

If you ever changed your BIOS to boot from something other than the hard drive, you need to keep that in mind. That's why I always use the F12 method when needed.

**_[SIZE="3"]Now My Questions: Power Management[/SIZE]_**

It's not over. There is one thing that still bothers me: power management. This is the single most important thing to me. I probably didn't buy a LV processor just because it wasn't available. :)

Battery discharge rate seems to stay around 10.6W with wifi turned on and bluetooth off, but going as low as 9.7 and as high as 12W. On idle without radios, it goes down to ~ 9W. I have no idea how much it is under Windows, but since the fan is always on (though imperceptible unless you put your hand on the vent) and the laptop feels a bit hotter overall (especially on AC), I guess there *may* be a problem somewhere. Is the AC mode ignoring any power management stuff?

This difference is noticed more when comparing heavy load scenarios on both OSes. Or maybe I'm just being more picky with Linux (don't think so) :)

What I did already: powertop. Applied the suggestions (unfortunately I didn't compare the before and after on the battery applet, sorry). Is it permanent or do I need to do this powertop procedure every time the computer is turned on?
