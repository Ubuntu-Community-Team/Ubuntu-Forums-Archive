### Post by exploder on 2013-07-07
I have an AMD APU running under 12.04 x64, I have an HP DV6 laptop. To get the system to run reliably I had to stay with the 3.2 kernel and the default open source drivers. Unity 3D works perfectly but I am not a gamer so I do not know how well games would run. I had tried a newer iso that came with the 3.5 kernel and I was using the proprietary ATI driver. Everything was fine at first, then a kernel update broke 3D graphics and I was stuck with Unity 2D...

The proprietary drivers were difficult to install also, it took 2 tries of installing them to get them to work. The system reported that the drivers were not in use but they were and they were working. I ended up reinstalling because I could not fix the problem and it was quicker to just reinstall because I save all my data anyway. I notice very little difference between the open source and proprietary drivers on this particular hardware. 

I have not had any problems at all with the system freezing and 3D graphics does work but I have doubts about playing any high end games but someone else might be able to shed some light on this. 12.04 is running great for me now and I can't think of a single problem I have had since doing the reinstall. Hope this is of some help to you.
