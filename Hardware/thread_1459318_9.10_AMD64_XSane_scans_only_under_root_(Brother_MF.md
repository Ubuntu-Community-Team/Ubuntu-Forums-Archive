There is no /etc/udev/rules.d/45-libsane.rules in Ubuntu 9.10 so I copied that file (with my scanner data) to that location but no joy.

Any help please?

PS. Printing also didn't work with drivers from Brother website. I removed everything connected to "Brother" except files in **ppd** directory and installed **brother-lpr-drivers-laser1** and **brother-cups-wrapper-laser1** from Synaptic. 
Prints fine now.
Not sure if it's important but there was a link named "lpd" in /etc/init.d that targeted "cupsys" that is also a link targeted "cups". I deleted old "lpd" and created new one targeted directly "cups".

### Post by plucky on 2010-04-22
> files inside ~/.sane/xsane have:
drwxrwx--- 2 root root 4096 2010-04-21 09:15 batch-lists
-rw-rw---- 1 root root 1646 2010-04-21 09:17 Brother:MFC-6800.drc
-rw------- 1 root root 4263 2010-04-21 09:17 xsane.mdf
-rw------- 1 zeiz zeiz 5212 2010-04-22 07:49 xsane.rc


Those file should not be owned by root.I think they were created when you started xsane from root and are still being used.

You should try changing the ownership to yourself ```
sudo chown zeiz:zeiz -R ~/.sane/xsane/*
``` and see if that makes a difference.

Or delete that directory and let xsane re-create it when you start it as a normal user.

Good Luck
