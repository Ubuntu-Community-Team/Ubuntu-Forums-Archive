### Post by cascade9 on 2011-01-30
Its probably due to some BIOS setting (legacy USB maybe, but I'm not sure on that, I'd need to go poking around to be sure). 

The BIOS updates should be on the asus product page for that motherboard. 

*edit- there are 2 P6X58D boards, P6X58D-E and P6X58D Premium. Make sure you get the right BIOS for your board, the worng BIOS version could cause problems, if it even works. 

Download tab, select an OS (doesnt mater which OS), go to the BIOS listing, d/l the BIOS update. 

It might make a difference, but I believe that it is some BIOS setting causign your problem, not the BIOS version.

### Post by MG2R on 2011-01-30
This is not a failure in Ubuntu... I had the same problem with my Gigabyte GA-EP45T-EXTREME mobo running Windows (I'm going to be sick now ;)). This is probably indeed something the BIOS is causing, but in my case it could not be fixed by editing BIOS settings (spent like a month tweaking the settings, have used every option available, so I should now :D).

You could indeed try to change the BIOS software itself, but I advise you to WAIT AS LONG AS POSSIBLE (until you are 1000% percent certain that it has nothing to do with wrong BIOS _settings_), updating BIOS is always risky.
