### Post by LoganLockwood on 2011-12-01
Hello. In Ubuntu 11.04 I had to install Avasys driver to even get it recognized. In Ubuntu 11.10 its automatically recognized, but wrongly as well, for it cannot print even the test page. It starts passing pages through and asking for more paper endlessly.

I have installed the proper Avasys driver for that model, but it won't work. :confused: It must be, somehow using the failing driver instead of the new one. How do I select it? 

I went to System Settings, Printing, and deleted the printer. Then "Add" new one. Turn the printer on, and it says "Configuring New Printer" and asks for password. I denied and it says that couldn't configure the printer. (I do that to avoid using the wrong driver).

In "Select Device" I select the printer TX220 and it says "searching for drivers" (Don't know why, since it is supposed to be installed already). It ends up recommending to use one from Avasys, so I select that one.

It says "Downloading and Installing Driver", so I wait. Now the "Serching driver" window disappears. I put a name for the printer and it is added correctly.

"Would you like to print a test page?" Yes, I would, so I try.

"Test Page submited ad Job 13"... AND NOW IT'S PRINTING! damn it!

Oh, well, problem solved. I post it anyway in case someone else needs the information.
