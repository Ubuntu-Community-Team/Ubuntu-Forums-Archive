### Post by Fafler on 2010-11-26
Unfortunately there isn't really a standard 3D benchmark tool for Linux. I usually use Doom 3 since 1) i have a copy and 2) it runs native on Linux and 3) haven't ever given me any installation problems. Theres a demo available here: [ftp://ftp.idsoftware.com/idstuff/doom3/linux/](ftp://ftp.idsoftware.com/idstuff/doom3/linux/) but i don't if it can run the timedemo. If you wan't to try, download the file, open a terminal and type
```
chmod +x Downloads/doom3-linux-1.1.1286-demo.x86.run
sudo Downloads/doom3-linux-1.1.1286-demo.x86.run
```

Follow the instructions and start the game. As soon as the intro starts (no need to watch it) press the ~ or ½ key t open the console. Now type
```
timedemo demo1
```

If will run a demo at the fastest speed possible and report the framerate when it finishes.

Many people uses glxgears for benchmarking, but it's really basic 3D acceleration and doesn't give a real idea about the performance.

You'll have to

```
 sudo ./doom3-linux-1.1.1286-demo.x86.run
```

The shell won't execute files from your current directory unless you give a complete path. This prevents the possibility of malicious commands damaging your system, eg. a file called ls in your home directory that runs every time you type ls in your home directory.

It seems OpenGL is broken. It's likely that the old driver left some files that breaks compatibility with the open source driver. I know the Nvidia driver does just that, if uninstalled incorrectly.
