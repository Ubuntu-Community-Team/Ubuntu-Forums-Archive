Turn on netbook. Blue power light, blue wifi light and battery light come on. Blue HD light comes on for about two seconds as HD starts to spin up. Then blue HD light goes out and HD spins down. No more HD activity light. Screen remains black at all times. Fan runs ok. If DVD player is connected it spins up. But no video ever. Have reset, shorted CMOS, removed CMOS (measures 3V), held down power button, all the Internet solutions, etc. Computer is dual boot with Lubuntu 14.? and Win7. Boots into Lubuntu unless I specify Win7 at start up. HD was removed and placed into a desktop as a second drive, but not as the bootable drive. Drive runs fine, can be read, all info seems to be there with all the partitions. I first noticed the video issue when I was in Win7 and had changed the screen resolution. The resolution wasn't really optimal for whatever purpose I changed it (can't remember). Forgot to change it back to a good resolution. Next time I looked at the screen it was going black. When it was rebooted, the screen was completely black and has been that way since.

I'm inclined to think it is a video chip, but was wondering if the dual boot got messed up if it would shut down the HD? The HD tries to start, but shuts down within two seconds. Based on the HD working on the second computer, I was wondering if either a bad video chip (or something else on the MB) or a messed up dual boot would shut it down.

I really like the 1000he, except for this issue. It is in almost new shape and would hate to throw it away if there was a simple fix.
