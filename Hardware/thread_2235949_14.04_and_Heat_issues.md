### Post by dave2001 on 2014-07-23
So I've been running the last several releases of Kubuntu or a KDE+base install of Ubuntu system on my T500 Thinkpad.

To upgrade versions I always do a fresh install, as I find it usually  makes for a smoother transition.

13.10 ran great on this T500. Almost everything completely functional out-of-box type install. I did this by installing the base system using a minimal install iso and then pulling in kde-plasma-desktop and the other software I needed or wanted.

I've done the same with 14.04 and things have not gone so smoothly. There were issues with installation (which were worked out). My main issue now is heat. This is a dual boot machine with windows7 and it actually ran a degree or two cooler a lot of the time on 13.10 which was around 41-42C.

Fresh install with 14.04 and I've CPU temps upwards of 58C with the lappy sitting idle.  I tried adding the line "acpi_osi=Linux" to grub because that helped some problems with heat issues on this thinkpad with previous iterations of 'buntu. No Change in temps.

This lappy does have a discrete ati card which has to use the open source radeon driver (as proprietary no longer provides support). Supposedly DPM is supposed to be enabled by default in the new 14.04 kernel for the radeon driver, but just in case, I added "radeon.dpm=1" to grub as well. Low and behold, my temps are now running at around 48C. This is obviously bettter, but certainly hotter than it could or should be (and thus using more power, shorter battery life, etc).
