### Post by ajgreeny on 2020-09-20
Be more careful with hardware choice than you would need to be if getting a laptop with OS installed if you want to manually install Linux/Ubuntu. Intel hardware is usually less likely to cause problems as far as I can tell, though it's a long time since I used any AMD/ATI hardware.

I have used several laptops from both Novatech and PCSpecialist, all purchased without an OS and I have never had a problem.  The machines were all older in design and hardware and could not be considered bleeding edge or anywhere near it, but all work briliantly, even an old Novatech Netbook with Atom 270N cpu, and just 1G of RAM, but even that old machine is still running, though now with Debian 10 Testing LXDE, 32 bit, which is very similar to, but faster than Lubuntu 18.04 LXDE.

### Post by mastablasta on 2020-09-21
> **ajgreeny said:**
> Be more careful with hardware choice than you would need to be if getting a laptop with OS installed if you want to manually install Linux/Ubuntu. Intel hardware is usually less likely to cause problems as far as I can tell, though it's a long time since I used any AMD/ATI hardware.


AMD driver for CPU and GPU (AMDGPU) is fully in kernel since 14.10 (whatever kernel that was). so they no longer cause any issues. in fact they work rather well and perform way better than i expected. 

on tests (check phoronix and other benchmarking websites for more info) AMD performance of CPU is maybe slightly weaker than Intel, GPU is slightly better than Intel. however chip is overall much cheaper than Intel. i got a relatively similar configuration for nearly 150 EUR less. but overall i noticed similar configurations to be at least 80-100 EUR cheaper. latest 4000 series are really impressive for low powered laptop CPU. they ran Crysis on one of those with fan taken off.

the AMD / nvidia combo is causing issues for now (some better support for the combo is found in latest kernels - mainline testing) as both GPUs want to be primary.
