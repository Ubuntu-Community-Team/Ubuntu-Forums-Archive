### Post by croo on 2011-05-12
I came across this thread when I was investigating "which laptop" for myself so I thought I just updated it.
> EDITED: There is apparently a BIOS switch for the thinkpad according to this thread but it is not clear whether Linux can actually use the Nvidia card, some say you can some say you can't
[http://forum.lenovo.com/t5/W-Series-...er/td-p/403923](http://forum.lenovo.com/t5/W-Series-...er/td-p/403923)

I got a Thinkpad W520 with the 1000M graphics card and installed 11.04 and this is what I found.

Indeed there is a BIOS switch to set 1 of 3 modes:

[LIST=1]
[*]INTEGRATED - Enables the integrated (with the CPU) Intel Graphics. I didn't try this mode.
[*]DISCRETE - Enables the nVidia graphics card.  This works for me in Ubuntu 11.04 - with a quirk.  The screen brightness settings go to full and while using the keyboard brightness up/down keys indicates it is changing the brightness there is no effect on the screen!  Bright screens use the battery quicker plus these LED screens are very bright... I found it a little tiresome on the eyes to use.
[*]OPTIMUS - I tried this by chance forgetting to disable it when I did the initial install.  It actually works even if it did confuse me a little.  I was confused because the installation recognizes there is an nVidia graphics card and installs the "additional" drivers.  But when you try to login with the default windows manager (Unity 3D) it will complain that this is "not supported by your graphics card" and switch you to gnome (2d).  Checking the nVidia Settings informs you the latest drivers are installed, they are activated but they are not in use (If you enable DISCRETE mode then it reports the drivers activated and used).
[/LIST]

There is another setting in the BIOS which I haven't seen mentioned. That is one to enable/disable Optimus if the OS is capable of using it or not. According to the help, if the OS does not support Optimus the it will auto switch to DISCRETE (the nVidia) mode.  I enabled it but it didn't seem to have an effect - I didn't spend too much time to be honest because I was anxious to get started with my new laptop.

For me personally, I rarely use the 3D, so I have left the Optimus enabled with the switch to enable DISCRETE when Optimus not supported=ENABLED too - even though it seems not to work.

So I am only using the integrated graphics.  IF I do need the 3D graphics I can simply enabled it in the BIOS which is a bit of pain but this way the battery lasts much longer.  Even so, if I could control the screen brightness I might leave it in DISCRETE mode and use the nVidia graphics all the time - it would be detrimental to the battery but this is a desktop replacement so I am more than likely going to be at a desk.

The good news is switching between them just involves the BIOS switch and ubuntu will happily switch between the two without issue.

So not ideal, but I was after the power of the CPU so I'm content for now but hope the situation will be resolved at some future date.

### Post by croo on 2011-08-13
Just to add to my earlier post...

When I posted previously I hadn't tested the Integrated mode BIOS option.  But to improve battery time I decided experiment.  At first I found what VValdo stated.  It worked but without 3D Unity or compiz.  When I looked closer I found this was because the correct intel driver was not loaded.  

I'm not 100% sure if this was ultimately required but I first removed the nvidia drivers. As I explained in my previous post, when I installed ubuntu first I had not disabled Optimus so ubuntu installed the nVidia drivers (though ubuntu always reported the drivers installed but not in use when using Optimus).

Secondly I blacklisted the non-intel drivers.  I created a w520.conf file in /etc/modprobe.d and added
blacklist fbdev
blacklist vesa 

Now, with the BIOS set to Integrated, when I boot ubuntu the intel drivers are loaded and the integrated graphics provides plenty enough power to run unity 3D and/or compiz. 

In fact I've realized these IGP are now so good that unless you specifically work/play in graphics (i.e. gamers, video, cad) they are probably all you need really .... I wonder will nVidia have ended up shooting themselves in the foot with this as people realize they might not actually need the nVidia GPU!!

One note on 3D Unity.  As I said the IGP provides plenty of power to make it work very smoothly - not a sign of any glitch.  But, I found Unity to be very unstable.  I have not pinpointed anything specific but I've rarely gotten through a full day without it hanging.  And when it does hang it requires a hard reset.  That's too much like being back with windows so I've stopped using it and dropped back to the "classic mode" option - basically gnome 2.
