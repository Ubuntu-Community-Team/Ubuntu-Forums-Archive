### Post by tyroeternal on 2009-05-15
Is there anyone that can help me understand the benefit to moving to an lpia install from an i386 install?  There seems to be only limited evidence that lpia does any good, and even so it provides maybe 10%.  While a 10% gain is good, lpia greatly limits the number of packages available.

Can similar benefits be had from building a custom kernel and trimming services manually?  Also, are there any users that have more accurate numbers on the difference between the two (or lack of)?

---

### Post by snowpine on 2009-05-16
There is no advantage to the lpia kernel. Ubuntu is the *only* linux distro that even offers an lpia kernel, and they do so because of a business deal they made with Dell (to develop a custom "Dellbuntu" for the Mini series). If there was an advantage to an lpia kernel, why do Arch, Debian, Fedora, etc. not support it? The Atom processor is fully compatible with i386/686.

### Post by tyroeternal on 2009-05-28
Maybe it is of some value!  After fiddling around with my system for a bit I decided to give lpia a try for comparison.  It is worth noting that the package selection in the Ubuntu repos seems to be equal in every aspect that I could want, medibuntu also offers lpia, as well as launchpad ppa's offering lpia packages.

This is all very subjective testing, but on what I believe to be an equally configured system, my Mini 12 is running both much cooler and has had 20-30 minutes more of projected battery life.  Hopefully in the next few weeks I can square away some time to do a pair stock install and run a more controlled test.

Maybe I'm just drinking the kool-aid, but it seems to make a difference.
