### Post by sarah.fauzia on 2008-06-16
Can someone change the status of this problem to **SOLVED**? I have a (miraculous!) solution! I hope this helps anyone else having this problem, or a future user to Ubuntu who might ;).

What had to be done was add a single line to the xorg.conf (accessible by sudo gedit /etc/X11/xorg.conf for those of you who are new and might not know). Add this line to your eraser section:

```
Option    "Button1"    "2"
```

Here is the full version of my xorg.conf, for any other tablet users who wish to compare. Note that I'm using a Motion LE1700 tablet, with wacom.

My inspiration was [this]("http://wiki.control-d.com/index.php?title=Ubuntu_Hardy_Heron_(8.04)_on_a_Toshiba_Portege_M400") website, where it says:

```
xsetwacom set eraser Button1 "button 3"
```
It gave me the idea to switch around my eraser codes, as well, in case they weren't mapping correctly (is that a correct term for it?). Well, it worked! Tested it in xournal. It works immediately when you put into the terminal ```
xsetwacom set eraser Button1 "button 2"
``` but I figured putting it into the xorg.conf would make it permanent (the suggestion at that site was to make it a startup script).

Again, I hope this helps!

### Post by Favux on 2009-02-06
Hi sarah and DrGNU,

Sarah I took the liberty of going over your xorg.conf.  I hope you don't mind.

The cursor section, and cursor in ServerLayout can be removed.  The cursor refers to the Wacom mouse of an external graphics tablet.

I also think you do not need the SendCoreEvents in the stylus section.  But since you have a Motion tablet I'm not totally sure.

The button stuff is confusing, I have not seen a good explanation of the format yet.  The line in the stylus section makes the side-switch into a R mouse button.  In Xournal it would act as the eraser.  Unlike Gimp, Xournal uses the right button as the "eraser".  In the eraser section I think you made the eraser a mouse middle button.  So I guess Xournal recognizes the middle button as "eraser" too.  Using that line were you able to get eraser working in Gimp?

DrGNU since you are using Intrepid you could comment out (#) the Generic Keyboard, Configured Mouse, and Synaptics Touchpad sections.  HAL (Hardware Abstraction Layer) would run them for you.  But if you want to bind a bezel button to a rotation script you shouldn't comment out the Keyboard section.  HAL breaks single key key binding, so you have to leave the keyboard section active in xorg.conf.

Please find the edited xorg.conf attached.  I hope this is helpful.
