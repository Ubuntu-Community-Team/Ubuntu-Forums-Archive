### Post by Aearenda on 2009-04-30
For acerhdf, add a line to /etc/modprobe.d/options.conf like this:
```
options acerhdf verbose=1 fanon=62 fanoff=52 interval=10
```
This will log the temperature every 10 seconds (to /var/log/syslog IIRC) - and it lets you control when the fan comes on or goes off. When you're happy it's working, set verbose=0 to cut out the log abuse.

These are the temp settings I am using. It gets a little warm but doesn't run the fan constantly.

The problem I had was that on 9.04 UNR it was not running the script /usr/local/bin/acerfand located at /etc/rc.local
A workaround for this is:

Follow the instructions [here]("https://help.ubuntu.com/community/AspireOne110L#Fan%20Control"). But don't do the **To run it at boot:** part. Instead do this:

```
sudo visudo
```

Add this line at the end:

```
<username> ALL= NOPASSWD: /usr/local/bin/acerfand
```

ctrl+X to exit and Y to save the changes.

Then go to Preferences -> Startup Applications and click Add.

```
Name: Fan Control
Command: sudo /usr/local/bin/acerfand
```

This will run the script once you log into your session. Not perfect but it works.

xrandr --query

### Post by Aearenda on 2009-06-10
Gettinsadda, the sysctl trick for the wireless LED only works with the MadWifi driver, not Ath5k. The backported Ath5k does that bit right, as you have found.
> it seems that I can access WiFi as long as I use the right channel, I was using channel 13 and this no longer works - is this a US v Europe thing?
Modern wireless routers and access points tell the clients which region code to use, and Ath5k obeys this. Channel 13 is not in the US set, but is available in Europe and Australia. Unfortunately, some modern wireless routers get the region code wrong, and say "US" even when they are not in the US - such as the Belkin one I have, which is in Australia. Consequently, I can't use channels 12 or 13, even though I can configure them in the router's UI, because it tells the clients to go to the US channels and they promptly disable those channels! Duh.

BTW, if you look through the output of 'dmesg | grep country' you should be able to see Ath5k (via cfg80211) telling you what region code it is using.

### Post by Aearenda on 2009-06-12
'lspci | grep VGA' isn't a solution, it's a command line way to find out what video hardware is installed. You have to run it in a terminal window. 

'lspci' lists the PCI devices that have been found. The '|' relays the output from that command into 'grep'. 'grep VGA' filters all the lines and shows only those that contain 'VGA' If you got a blank screen, then it means you have no VGA device found. 

The next step is to run 'lspci' on its own, and read the list yourself to see what it has found instead of a VGA device.

EDIT: Maybe I've misunderstood you - if you mean you've tried the Poulsbo driver from the links I posted, and it didn't work, then maybe it's the wrong driver for you. The lspci output will tell us this.

**_FOR JAUNTY_**

quiet

pciehp.pciehp_force=1

Then simply re-boot

### Post by Aearenda on 2009-07-13
> **twright said:**
> Hi,
adding the options to individual items in menu.lst will mean that they will be wiped each time you upgrade the kernel, to make it permanent you need to find the defoptions line in the file, uncomment it and then add the options there.

Sorry but this isn't quite right in detail, twright - you are right to add the option here, but you must not uncomment the line. It is used by the update-grub pre-processor, which knows what to do, and commented to avoid it being used by grub proper, which sees what update-grub has done with it.

So, the resulting line in /boot/grub/menu.lst should look something like this:
```
# defoptions=quiet splash resume=/dev/sda4 enable_mtrr_cleanup vga=771 pciehp.pciehp_force=1
```
Don't just copy my example, add the pciehp part to the end of what you find there already in your system.

The # in this line marks it as a line to be read by update-grub.
True comments have ## in the update-grub section of the file.

After changing this line, run the following command in a terminal:```
sudo update-grub
```This will propagate the change to all the standard boot entries.

By the way, adding the pciehp entry makes my system too slow on wake from standby, so I don't use it. My left card slot works with hotplug anyway.

I use the madwifi driver on my AA1, to avoid the ath5k hang, and (old or not) it works fine. The [Intrepid instructions]("https://help.ubuntu.com/community/AspireOne") still work, though with Jaunty you don't need to remove the driver during suspend, so you can skip the part starting "It may be necessary to remove the ath_pci module when suspending" and resume at "Wireless LEDs". 
You need to recompile and reinstall (make, sudo make install) the driver after each new kernel update, and you do need to do the dev.wifi0.ledpin=3 and dev.wifi0.softled=1 part.

### Post by markbuntu on 2009-10-20
> **darkwalt said:**
> Said the same thing three pages ago.  Works if you boot with a card in it.  No fix in Jaunty yet.

There basically is no fix for this and likely never will be because it has to do with the way the hardware is physically wired. The problem is that both card readers use the same piece of back end hardware in many early machines and the only way the system can detect if there is more than one reader attached to this hardware is to see it and the only way to do that is to have a card installed in the secondary reader so it is detected and reported by the back end hardware when the machine boots up.

Since this is not the case for all machines, even within the same type designations, it would be very difficult to write something to detect which machines would need a quirk and which would not. The only way this could really be fixed would be in the bios but that is also problematic for the same reason.

### Post by celticbhoy on 2009-10-20
This works for me on my year old AAO D150

FOR JAUNTY
Open a terminal.
type the following (enter after each line)
cd /boot/grub
sudo cp menu.lst menu.bak (enter password when prompted)
gksudo gedit menu.lst
(now find your entry for Jaunty - it will look like this :-

title	 Ubuntu 9.04, kernel 2.6.28.3.20090331
uuid	 b3767ead-e16b-4882-b51c-d280f1dd85d2
kernel	 /boot/vmlinuz-2.6.28.3.20090331 root=UUID=b3767ead-e16b-4882-b51c-d280f1dd85d2 ro quiet splash vga=789 
initrd	 /boot/initrd.img-2.6.28.3.20090331
quiet
)
at the end of the kernel line add the following :-

pciehp.pciehp_force=1

Then simply re-boot

For Karmic change grub.conf instead of menu.lst.
