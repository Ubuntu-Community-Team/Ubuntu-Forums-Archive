### Post by dedbox on 2009-04-14
I was able to get SHMConfig working on Intrepid 8.10 by pasting the following lines into /etc/X11/xorg.conf:

```
Section "InputDevice"
        Identifier      "SynPS/2 Synaptics TouchPad"
        Driver          "synaptics"
        Option          "CorePointer"
        Option          "Device"              "/dev/psaux"
        Option          "Protocol"            "auto-dev" 
        Option          "SHMConfig"           "true" 
EndSection
```

After logging out and back in, I opened the command line and ran this:

```
syndaemon -d -t
```

The touchpad then successfully ignored click and scroll events until 2 seconds after typing stops.

Apparently, enabling SHMConfig is a serious security issue. We should be able to disable the touchpad without enabling SHMConfig as of Intrepid, but this bug is in the way:

[https://bugs.launchpad.net/ubuntu/+source/xserver-xorg-input-synaptics/+bug/295236]("https://bugs.launchpad.net/ubuntu/+source/xserver-xorg-input-synaptics/+bug/295236")
