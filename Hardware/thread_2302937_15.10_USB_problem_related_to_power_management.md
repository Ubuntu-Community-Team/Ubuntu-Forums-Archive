### Post by stlu on 2015-11-14
I have a problem where the USB ports on my box are not working once ubuntu boots up.

In the initial BIOS mode, I can plug in a USB memory stick and the light on the stick will come on.

If I plug in the same USB stick while ubuntu is running, the light comes on for a second, and then turns off.
dmesg does not print any output about the stick being plugged in.

There is a way that I could get the stick running, by running the command:
root@ubuntu# echo on > /sys/bus/usb/devices/usb1/power/control
(I got that from [http://ubuntuforums.org/showthread.php?t=2226559&p=13038728#post13038728](http://ubuntuforums.org/showthread.php?t=2226559&p=13038728#post13038728))

I installed powertop but that did not give me any information other than it shows "Bad" when I echo "on" and "Good" when I echo "off" for the USB ports.

Interestingly, this only works once.  If I remove the stick and re-plug it, the light will turn on then off as it did before.
If I toggle back and forth between 'on' and 'auto', each time I do that I can get a stick to be recognized.

How do I get the USB port to "stay on" if the above proves that it is turning off?

Hardware is an IBM Thinkcentre with a Pentium 4 processor.
