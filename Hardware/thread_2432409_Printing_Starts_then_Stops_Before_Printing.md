### Post by Quarkrad on 2019-12-02
Recently installed Mate 18.04 on a neighbours PC (dual booting with Win 10).  Set up a canon printer and everything has been fine.  Neighbour disconnected usb printer cable to re-root it and since no printing.   I have tested the printer in win10 and it prints OK.  What happens is when you choose to print something the print icon appears in the top toolbar (normal action) - then in about 1.5 secs the icon disappears.  I have set up the printer via the normal Printer - Add Printer mode and via localhost/cups but it makes no difference.  It is as if something inside Ubuntu is saying 'hah, you want to print - now you can't'!.  Any advice appreciated.

### Post by Quarkrad on 2019-12-06
Solved - initially I used the printer driver downloaded from Canon and the printer was fine.  Not sure what happened when neighbour pulling out usb lead and re-routed but my effects to the printer working again was using the built in Ubuntu/Linux (Canon) printer driver.  This is where the problem started.  I re-installed the OS and again had the same problem - this is when I remembered the propriety driver.  I installed the Canon driver and now all is well.
