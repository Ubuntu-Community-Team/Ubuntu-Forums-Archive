### Post by Yellow Pasque on 2015-08-08
The OP should still be able to get the panel's native resolution regardless of the cable/port used. I would suspect an issue with the diaplay's EDID, so that's what I would google (I know there are some special ways to override the EDID with the nvidia driver). There might be some clues in the /var/log/Xorg.0.log

> **wirywolf said:**
> Baffles me why VGA is still used.

Some people don't know any better, especially when their display only comes with a VGA cable and no DVI cable (as is the case for the OP's display). I guess manufacturers did it to save money and it might have made sense compatibility-wise in the days where older cards/mobos/laptops only had the VGA port. But yes, DVI cables are relatively inexpensive and if you're using an LCD, a dual-link DVi cable is a good investment.

### Post by gabrielSaraiva on 2015-08-15
Thanks to you all Guys!

The problem is that NVIDIA driver has a BUG in EDID, and cannot use this information rightly. After search a lot how to override this problem without success, I just send a nice GTFO to NVIDIA driver and installed the Nouveaul driver.

All works nice with xrandr and now I'm using with 1440x900.

Marking as solved.
