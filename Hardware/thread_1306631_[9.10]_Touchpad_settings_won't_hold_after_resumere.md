### Post by rx8saxman on 2009-10-30
Hi all, I just installed Ubuntu 9.10 and I'm having a really annoying problem. For some reason, by default in 9.10 the multitap (2-finger middle click, 3-finger right) are switched. 

By running "synclient -l" I can see that TapButton2=3 and TapButton3=2, and I can successfully switch them back by simply runngin "synclient Tapbutton2=2 TapButton3=3".

The problem is that anytime I resume from standby or reboot the system, it defaults back to the switched settings.  I tried enabling SHMConfig and running synclient with the -s switch, but that had no effect.  Any ideas how I can make those settings stick?

### Post by Vermind on 2009-11-04
I have the same problem.
Fix: have ```
synclient Tapbutton2=2 TapButton3=3
``` in your Startup Programs for your session (fixes on reboot).
I am currently working on the on resume fix, but it would appear that inserting the command into a file in /etc/pm/sleep.d does not do anything :(. My guess is those are run before X and synclient are accessible again.

Even making a hal policy file in /etc/hal/fdi/policy/99-tapbutton-switch.fdi with the correct settings only sets the correct buttons on startup of X, not on resume.
