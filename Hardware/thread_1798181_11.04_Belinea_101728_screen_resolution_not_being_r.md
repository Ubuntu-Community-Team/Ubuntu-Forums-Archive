### Post by guiliker on 2011-07-08
I found this [here](http://ubuntuforums.org/showpost.php?p=10929081&postcount=8) and this [wiki](https://wiki.ubuntu.com/X/Config/Resolution)

So I did the following:
1. copy and pasted the desired modelines from the edid section of my Xorg.0.log into gedit to remove the irrelevant data so for example this log line:
```
[    18.697] (II) intel(0): Modeline "1280x1024"x0.0  108.00  1280 1328 1440 1688  1024 1025 1028 1066 +hsync +vsync (64.0 kHz)
``` 
becomes this modeline:
```
"1280x1024_60"  108.00  1280 1328 1440 1688  1024 1025 1028 1066 +hsync +vsync
``` 
which becomes this xrandr command:
```
xrandr --newmode "1280x1024_60"  108.00  1280 1328 1440 1688  1024 1025 1028 1066 +hsync +vsync
xrandr --addmode VGA1 "1280x1024_60"
xrandr --output VGA1 --mode "1280x1024_60"
``` 
tried in terminal should reset the screen resolution to the new size which in this case is: 1280x1024 at a refresh rate of 60Hz

if this works attempt to make it permanent by appending /etc/gdm/Init/Default with the xrandr commands, so in this case:
```
sudo nano /etc/gdm/Init/Default
```
locate: /sbin/initctl -q emit login-session-start DISPLAY_MANAGER=gdm and place **above**
```
xrandr --newmode "1280x1024_60"  108.00  1280 1328 1440 1688  1024 1025 1028 1066 +hsync +vsync
xrandr --addmode VGA1 "1280x1024_60"
xrandr --output VGA1 --mode "1280x1024_60"
```
using: "Control" and "o" to save it, hitting "Enter" to keep the default file name and "Control" and "x" to come out of nano

restart the computer

on doing this I found that the computer did not start in the desired 1280x1024 resolution however on going to Menu>Preferences>Monitor the new setting was there so I selected it and then clicked on the button to make "default". On restart I had my desired screen resolution. I have no idea how this action works.

Incidentally I got a random monitor detection whilst I was doing this for a Rogen Tech Distribution Inc 27" monitor...screenshot attached.
