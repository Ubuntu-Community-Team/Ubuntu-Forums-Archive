### Post by rufus25 on 2010-02-08
I had to setup Ubuntu on a new Nettop PC for a friend. It's based on the **ASUS AT3N7A-I** mainboard (Intel Atom 330 & GeForce 9400). 

The installation (*Ubuntu 9.10 32Bit*) went okay and everything was running. After loading the Nvidia drivers (through the System -> Hardware Driver dialog) and restarting, the system crashes (automatic logout and back to GMD login screen) after about 30 or so seconds. This happens every time I log in. When I disable the driver everything seems working again.

In addition to the "crashes" I get wired display glitches.

I removed the installation and installed the 9.10 *Netbook-Remix*. First no Nvidia driver was installed and the Desktop was awfully slow (hovering over an icon, it takes some seconds before the effect occurs). After enabling the Nvidia drivers (through the dialog again) all seem to be going well. But after some time the glitches appeared again, no crashes though but the system keeps freezing.

Now the last freeze leaves me with the attached screen, which reminds me of the screen fuzz a friend has gotten before his last GPU dies. So probably the board is just broken?
