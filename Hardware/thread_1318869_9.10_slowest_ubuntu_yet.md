### Post by AlanR8 on 2009-11-13
Was having some internet speed issues as well on Karmic. I was convinced it was a DNS lookup error and like others had gone to OpenDNS with no change.

Then I stumbled on this little fix elsewhere on this forum. I can't take the credit for it but it worked a treat!

> sudo gedit /etc/nsswitch.conf

Then rem out the line that starts:

> hosts: xxxxxxxxxxxxxxx

(Rem means to make the line unreadable so it is ignored. To do this just put a # sign at the beginning of the line)

Now add the line:

> hosts: files dns 

Or you could have just modified the existing line.

The difference in browsing speed in FF, Opera and Chrome was remarkable!

### Post by maxomai on 2009-11-13
Squid, are you noticing the slowness on anything besides Firefox? If not, then it may be a firefox problem. You can probably fix that by changing configurations as instructed in this thread:

[https://bugs.launchpad.net/ubuntu/+source/firefox-3.5/+bug/456038/comments/3]("https://bugs.launchpad.net/ubuntu/+source/firefox-3.5/+bug/456038/comments/3")

To summarize:
[INDENT]
solution :

in Firefox address bar:
about:config

and change the following lines like this :

network.dns.disableIPv6: true

network.http.pipelining: true
network.http.proxy.pipelining: true
network.http.pipelining.maxrequests: 4

reboot firefox[/INDENT]

Following these instructions cleaned up all my problems immediately.

(There was also an instruction to change browser.turbo.enabled to **true**, but that option isn't in my configuration)

Please let me know whether this fixed your slowness issue.

Cheers,
max
