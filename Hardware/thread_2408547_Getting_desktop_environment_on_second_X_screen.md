I have the opposite problem from [this post]("https://ubuntuforums.org/showthread.php?t=1186508&highlight=desktop+environment+screen"). I have three monitors and an LG OLED TV (via Yamaha RX-V383 A/V receiver) connected to my PC, with the monitors connected to one GPU and the TV connected to another. I have no xorg.conf configuration file in /etc/X11/ in my usual boot configuration, but using nvidia-settings I can create an xorg configuration that starts a separate X screen on the TV (see below). However, I don't get a desktop environment on the TV (only a black screen, with black-outlined-in-white x for mouse cursor). I can start e.g. Firefox on the second screen with ```
DISPLAY=:1.1 firefox
``` (in the appropriate size, if I include the geometry arguments), but ```
DISPLAY=:1.1 gnome-session
``` does nothing.

Eventually, I would like to have [Clone Hero]("https://clonehero.net/") running on the TV, but ```
DISPLAY=:1.1 ./clone_hero_executable
``` exits with a BadDrawable X server error. So either a GDM login screen, a way to start a desktop session (already being logged in on the first X screen), or an auto-started instance of Clone Hero on the second X screen would be acceptable (swapping the xorg configuration in and out as required).
