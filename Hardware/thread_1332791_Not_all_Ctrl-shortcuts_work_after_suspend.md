### Post by fredie007 on 2009-11-20
Hi, this afternoon I've run into a problem. Ever since suspending my Dell Studio 1555 laptop for the first time in Ubuntu, the ctrl-key is acting weird. Here's the deal: not all shortcuts work. Indeed, not all of them, that's the weird part. 

Ctrl-Alt-Delete, for one, does work, as do Ctrl-Alt-Left (and Right) and Ctrl-Alt-D. Some of my user-configured shortcuts on the other hand don't work, neither do Ctrl-C, Ctrl-V, Ctrl-A, Ctrl-W (in firefox), Ctrl-Tab (in firefox), Ctrl-T (in firefox). The terminal doesn't recognize it anymore too.
However, when I'm in a directory, when I select a file or some files; as soon as I press Ctrl, the brown background of the selected files turns greyish. The same goes for selected text when pressing Ctrl. What state does this represent?

I'm pretty new to Ubuntu and Linux, so if anyone has an idea or a suggestion, know that all help will be appreciated.

Edit: Ok, I have Ctrl-Alt-T bound to Tomboy, and whenever I press Ctrl-Alt-T in that exact order, it doesn't pop up, but when I press Alt-Ctrl-T it does pop up. So I'm guessing the shortcuts in combination with Alt do work. When switching desktops however, the order of presses doesn't matter...

---

### Post by fredie007 on 2009-11-21
I managed to fix it by switching the keyboard layout to something else, restarting, switching the layout again and restarting. Afterwards everything worked again.
