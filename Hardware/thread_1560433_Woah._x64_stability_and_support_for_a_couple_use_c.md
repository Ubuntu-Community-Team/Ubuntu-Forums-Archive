### Post by cinohpa on 2010-08-24
So I spent a few minutes over at the x64 board trying to post a new thread when I realized the thread at the top saying the board was closed because ubuntu x64 had matured so much.

I think the last time I tried x64 was about 6-12 months back in Ubuntu 9.8-10

Reasons for 64: 

1. Utilize extra performance on cpu intensive tasks/scientific computation
2. Nice for my desktop which has a lot of ram

Reasons against 64:
Mainly hardware support, a couple specific cases:
1. Video support. I had a lot of trouble with x64 on my desktop with an ati card. I recently reinstalled 10.4 with x86 and the graphics work better than they ever have. I fear installing x64 and finding that there are still compatibility issues there.
2. Misc driver support. For example I found that my touch/track pad on my laptop worked a lot worse in x64 regardless of whatever tweaking I did. 

Like I said, I haven't wanted to reformat my machines to try out x64 without trying to get some feedback first.

Can anybody comment to the stability of x64 for these use cases? Any things on x64 which still aren't working that well? 

Thanks.

### Post by dtoronto on 2010-08-24
I've been running 64 since Ibex, and this was the first distro that has not had major issues from the start.  I jumped on the 10.04 beta and aside from a few minor issues the 64-bit seems to be pretty solid in my opinion, you may run into issues of applications with no 64 bit support that might hold you back, but it's certainly moving forward.

1. I doubt that ATI legacy support will ever work great with Ubuntu 32 or 64 bit, but the newer ATI cards seem to work well.

2. Misc driver support seems to work extremely well for everything I use, I use very few proprietary drivers anymore, but this is still hit and miss.
