### Post by knoid on 2009-05-26
I use a Kensington Expert Mouse (trackball) connected to the serial port of my ubuntu box. After upgrading to 9.04 it no longer works. I poked through my previously-edited xorg.conf and found the mouse section commented out due to deprecation. Uncommenting these lines made no difference.

I've tried creating a mouse.fdi file with the same settings as xorg.conf in /etc/hal/fdi/policy but no joy there either. Multiple searches haven't yielded anything new, so I'm now posting here and asking for your help.

Help! :D

mouse.fdi:```
<?xml version="1.0" encoding="ISO-8859-1"?>
<deviceinfo version="0.2">
<device>
<match key="info.capabilities" contains="input.mouse">
<merge key="input.x11_options.Device" type="string">/dev/ttyS0</merge>
<merge key="input.x11_options.Protocol" type="string">ThinkingMouse</merge>
<merge key="input.x11_options.Emulate3Buttons" type="string">false</merge>
<merge key="input.x11_options.CorePointer" type="string">On</merge>
<merge key="input.x11_options.Buttons" type="string">4</merge>
<merge key="input.x11_options.ButtonMapping" type="string">1 2 3 4</merge>
</match>
</device>
</deviceinfo>

```

Mouse section of xorg.conf (which worked under 8.10):
```
# commented out by update-manager, HAL is now used
#Section "InputDevice"
#       Identifier      "Kensington Expert Mouse"
#       Driver          "mouse"
#       Option          "Protocol"      "ThinkingMouse"
#       Option          "Device"        "/dev/ttyS0"
#       Option          "CorePointer"
#       Option          "Emulate3Buttons"       "true"
#       Option          "Buttons"       "4"
#EndSection
```

### Post by knoid on 2009-05-27
I've managed to resolve this with the help of a friend, but I'm posting the solution here in case others find it useful.

I renamed my mouse.fdi file to 10-mouse.fdi to make sure it was being loaded, and added this line to it:
```
<merge key='input.x11_driver' type='string'>mouse</merge>
```

After a reboot it's now working!

Here's the file in full so you can copypasta:

```
<?xml version='1.0' encoding='UTF-8'?>
<deviceinfo version='0.2'>
  <device>
    <match key='info.capabilities' contains='input.mouse'>
      <merge key='input.x11_driver' type='string'>mouse</merge>
      <merge key="input.x11_options.Device" type="string">/dev/ttyS0</merge>
      <merge key="input.x11_options.Protocol" type="string">ThinkingMouse</merge>
      <merge key="input.x11_options.Emulate3Buttons" type="string">false</merge>
      <merge key="input.x11_options.CorePointer" type="string">On</merge>
      <merge key="input.x11_options.Buttons" type="string">4</merge>
      <merge key="input.x11_options.ButtonMapping" type="string">1 2 3 4</merge>
    </match>
  </device>
</deviceinfo>
```

### Post by cur8or on 2009-06-23
For those who are trying the same with ExpertMouse trackball in 9.04 release - 

Edit /usr/share/hal/fdi/policy/10osvendor/11-x11-vmmouse.fdi (pick whatever mouse is mapped as on your system) to look similar to this and use whatever mapping you prefer:

<?xml version="1.0" encoding="ISO-8859-1"?>
<deviceinfo version="0.2">
  <device>
    <match key="info.capabilities" contains="input.mouse">
      <merge key='input.x11_driver' type='string'>mouse</merge>
      <merge key="input.x11_options.Device" type="string">/dev/psaux</merge>
      <merge key="input.x11_options.Emulate3Buttons" type="string">false</merge>
      <merge key="input.x11_options.CorePointer" type="string">On</merge>
      <merge key="input.x11_options.Buttons" type="string">6</merge>
      <merge key="input.x11_options.ButtonMapping" type="string">1 6 3 2</merge>     
    </match>
  </device>
</deviceinfo>

This works for me with the trackball connected to USB port.
