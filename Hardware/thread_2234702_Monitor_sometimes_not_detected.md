### Post by tmichaelmcn on 2014-07-16
I am running Ubuntu 12.04 on a Dell Inspiron 530 with a Samsung 23" monitor.  It ran flawlessly for about 5 months, then the system began to fail to detect the monitor (Display applet in system settings shows the monitor as "Unknown" with 1024x768 resolution instead of the correct 1920x1080).  Clicking "detect displays" has no effect, and running xrandr shows "cannot find output "'VGA.'" The problem is this is intermittent--sometimes it will detect the monitor correctly and then lose it again after returning from suspend.  Sometimes just opening the Display applet without changing any settings will be enough to set the correct resolution.  Once simply keeping the system active for 18 hours did the trick.  The grub boot screen also shows the incorrect resolution, as also happened when I recently ran the installation dvd.  I'm posting this here because I suspect that this may be a problem with my graphics card.  Any help would be greatly appreciated.
