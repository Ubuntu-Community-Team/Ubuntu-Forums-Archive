### Post by meierfra. on 2009-04-16
I suggest to rewrite you partition table table so that it looks like


/dev/sda1 * 126 156280382 78140128+ 7 HPFS/NTFS
/dev/sda2 156280383 176763194 10241406 7 HPFS/NTFS
/dev/sda3 176763194 312576704 67906755 f W95 Ext'd (LBA)
/dev/sda5 176779323 184972409 4096543+ 82 Linux swap / Solaris
/dev/sda6 212973768 214933634 979933+ 82 Linux swap / Solaris
/dev/sda7 214933698 312576704 48821503+ 83 Linux

I'm pretty sure that gparted will not complain about this table. Also Vista will be on a primary partition, which it must have been before your partition table got messed up.

To rewrite your partition table, boot into Ubuntu and download the attached file PT.txt to your Desktop and  open a terminal. Rewrite your partition table via


```
sudo sfdisk -f --no-reread  -O ~/Desktop/OldPT.save /dev/sda < ~/Desktop/PT.txt
```

Post the output of that command and also of

```
sudo fdisk -lu
sudo sfdisk -d
sudo parted /dev/sda print
```

Just as a precaution, copy the file OldPT.save to a save place outside your hard drive (Flashdrive, online storage,email account...)

You will also have to reinstall grub:

```
sudo grub
root (hd0,6)
setup (hd0)
quit
```

Reboot and boot into Ubuntu.

To be able to boot into Vista, you need to edit menu.lst

```
gksudo gedit /boot/grub/menu.lst
```

Change your Windows item to

title Windows Vista
root (hd0,0)
chainloader +1


Reboot and see whether you can still boot into Vista.

In case your are not able to boot into Ubuntu or Vista, report any error messages you got.

### Post by meierfra. on 2009-04-29
> I 'm leaving it in this thread in case the people before are interested about what you're doing.
Since the other cases in this thread are no longer active, that's fine with me.

```
  Device Boot      Start         End      Blocks   Id  System
/dev/sda1   *          63    62910539    31455238+   7  HPFS/NTFS
[COLOR="Red"]/dev/sda2       108021060   488392064   190185502+   5  Extended
/dev/sda3       160457220   244348649    41945715   83  Linux[/COLOR]
/dev/sda5       108021124   160457219    26218048    7  HPFS/NTFS
/dev/sda6       244348713   246372839     1012063+  82  Linux swap / Solaris
/dev/sda7       246372903   488392064   121009581    7  HPFS/NTFS

```


From you fdisk output we see that partition /dev/sda3 sits inside the extended partition /dev/sda2.  Well, an extended partition is just container for other partitions, so this does not seem to be a problem. But it is. An extended partition is only supposed to contain logical partitions. (A logical partition is  partition numbered /dev/sda5 or higher,  /dev/sda1 to /dev/sda4 are called primary partition)

So you have a primary partition (/dev/sda3) sitting inside  an extended partition (/dev/sda2).  For most purposes this actually does not cause any problems. But gparted is very cautious   about partition tables.  So it considers the partition table invalid and shows the whole drive unallocated.


In general, there are two  options to fix the problem:  Shrink the extended partition so that the primary partition is no longer inside the extended partition, or convert the primary partition to a logical partition.

 /dev/sda3 is located  between the two logical partitions

```
/dev/sda5       108021124   160457219    26218048    7  HPFS/NTFS
```

and

```
/dev/sda6       244348713   246372839     1012063+  82  Linux swap 
```


There should be at  least 63 unallocated sectors between a logical partition and the preceding partition (the space is needed for the partition table)

/dev/sda5 ends at 160457219  and  /dev/sda3 starts at  160457220

So we cannot turn /dev/sda3 into a logical  partition, that is  /dev/sda3 has to stay a primary partition.  But that means that the extended partition  has to starts after /dev/sda3. So /dev/sda5 will no longer be  inside the extended partition and we are forced to turn /dev/sda5 into a primary partition.

So these are the changes we have to make:

1.  Convert /dev/sda5 to a primary partition (this can be done by inserting it into the currently empty /dev/sda4 slot)

2.  Make the extended partition  /dev/sda2 start right after /dev/sda3 (that is at sector  244348650 (=244348649  +1)

3.  Since the start point of /dev/sda2 will  change, we also have to change the size of /dev/sda2.
 /dev/sda2   starts at  244348650  and  ends at 488392064, and so the new size is 244043415  (= 488392064-244348650 +1)


To carry out these changes, start with output of "sfdisk -d" (Save the output in the  file PT.txt on your desktop, you can do that directly with "sudo sfdisk -d> ~/Desktop/PT.txt)

```

# partition table of /dev/sda
unit: sectors

/dev/sda1 : start=       63, size= 62910477, Id= 7, bootable
/dev/sda2 : start=108021060, size=380371005, Id= 5
/dev/sda3 : start=160457220, size= 83891430, Id=83
/dev/sda4 : start=        0, size=        0, Id= 0
/dev/sda5 : start=108021124, size= 52436096, Id= 7
/dev/sda6 : start=244348713, size=  2024127, Id=82
/dev/sda7 : start=246372903, size=242019162, Id= 7

```

and change it to
```

# partition table of /dev/sda
unit: sectors

/dev/sda1 : start=       63, size= 62910477, Id= 7, bootable
/dev/sda2 : [COLOR=RED]start=244348650, size=244043415,[/COLOR] Id= 5
/dev/sda3 : start=160457220, size= 83891430, Id=83
[COLOR=blue]/dev/sda5 : start=108021124, size= 52436096, Id= 7 [/COLOR]
/dev/sda6 : start=244348713, size=  2024127, Id=82
/dev/sda7 : start=246372903, size=242019162, Id= 7

```


(So I change the "start" and "size"  of /dev/sda2 and removed  the "/dev/sda4" line (so /dev/sda5 moved into the position of /dev/sda5)

The header in each line (everthing before the first colon) is actually  ignored by "sfdisk" (it just make the table easier to read for humans.  So there isn't any real reason the change the "/dev/sdaX". But I still always change them so that is  easier to see what the new partition table is going to look like:
```

# partition table of /dev/sda
unit: sectors

/dev/sda1 : start=       63, size= 62910477, Id= 7, bootable
/dev/sda2 : start=244348650, size=244043415, Id= 5
/dev/sda3 : start=160457220, size= 83891430, Id=83

/dev/sda5 : start=244348713, size=  2024127, Id=82

```


sfdisk is very picky about whitespace at the end of the line or at the end
of the file. So I always double check via


```
 cat -A ~/Desktop/PT.txt
```

Suppose the output is
```

# partition table of /dev/sda$
unit: sectors$
$
/dev/sda1 : start=       63, size= 62910477, Id= 7, bootable$
/dev/sda2 : start=244348650, size=244043415, Id= 5$
/dev/sda3 : start=160457220, size= 83891430, Id=83$
/dev/sda4 : start=108021124, size= 52436096, Id= 7 $
/dev/sda5 : start=244348713, size=  2024127, Id=82$
/dev/sda6 : start=246372903, size=242019162, Id= 7$
$

```
The $ signs marks the end a line.  There is a "space" at the end of "/dev/sda4" line. There also is a blank line at the end of the file.  "sfdisk" will reject this file. So one needs to remove the "space" and the blank line. The output if "cat -A ~/Desktop/PT.txt" needs to look like

```

# partition table of /dev/sda$
unit: sectors$
$
/dev/sda1 : start=       63, size= 62910477, Id= 7, bootable$
/dev/sda2 : start=244348650, size=244043415, Id= 5$
/dev/sda3 : start=160457220, size= 83891430, Id=83$
/dev/sda4 : start=108021124, size= 52436096, Id= 7$
/dev/sda5 : start=244348713, size=  2024127, Id=82$
/dev/sda6 : start=246372903, size=242019162, Id= 7$

```


Now you can rewrite the partition table with

```

sudo sfdisk -f --no-reread  -O ~/Desktop/OldPT.save /dev/sda < ~/Desktop/PT.txt

```

Copy the file
OldPT.save to a save place outside your hard drive (Flashdrive, online
storage,email account,...)

Before you reboot check whether "fdisk" and "parted" have any complains about the partition table:

```
sudo fdisk -lu
sudo gparted /dev/sda print

```

You also might have  to reinstall Grub, if the device name of your Ubuntu partition has changed.  But Ubuntu was on /dev/sda3 before you changed  the partition table and its still on /dev/sda3.
So no need to reinstall Grub and you should be all set.
