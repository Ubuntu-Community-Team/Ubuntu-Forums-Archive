### Post by NoWayWin8 on 2015-06-17
Laptop keyboards are a PITA and the delicate connector ribbons must be removed and reinserted exactly right and are easy to damage in the process.

Check the official HP instructions for that laptop [http://h20564.www2.hp.com/hpsc/doc/public/display?docId=emr_na-c01636850](http://h20564.www2.hp.com/hpsc/doc/public/display?docId=emr_na-c01636850)

If you're sure the connections are seated and the replacement keyboard still doesn't function, the problem is either damaged ribbons or something at the motherboard (RIP).

### Post by mattharris4 on 2015-06-17
I have to agree that if this is a mobo issue the computer is done as a laptop.  However, if everything else works is is possible to use this at a desk, connecting a keyboard, mouse and possibly a monitor in the process?  You are evidently running Ubuntu Gnome on it OK other than this issue so maybe you don't have to toss the computer into the trash heap just yet -- for example if you have children they could possibly use this computer and if they mess it up it doesn't matter as much since it was borderline junk anyway.  The press release announcing this computer's sale is from 2008 so this computer is not new anyway.  

If this makes you start looking at new laptops Dell sells a line of them with Ubuntu (Unity) installed at the factory.  They can be brought up on the Dell website with an appropriate search.  I personally recommend steering away from the Celeron models, the Pentium models should be OK for most uses (although I wish they offered a i3 version).  You can also install whatever version of a Canonical OS you wish on your choice of Dell, reports are other than possibly the two-in-one versions (data are still sparse on those) mainline Linux OSes install fine on that brand.  I have also read good reports regarding Lenovos and Linux.  

Steer clear of HPs with Win 8 or 8.1 if you want to run Linux on them, it can be made to work but reprogramming of (if I understand it correctly) GRUB is required, some Toshibas also have this issue although mine took Lubuntu fine and it is only about a year old, some have also had trouble with Acers.  Oldfred has instructions on how to do this but it looks to be quite complicated and buying a more amenable brand in the first place is the best solution IMO.
