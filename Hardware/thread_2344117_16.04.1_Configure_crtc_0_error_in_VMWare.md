### Post by bproffit on 2016-11-22
I've scoured the forums and there is a lot that talks about Nvidia and ATI driver problems in setting resolutions, but nothing that applies or works in my environment. I'm running in a virtual machine, and the Displays app (and xrandr) report as valid the resolution I'm trying to set. The video memory allocated to the virtual machine is more than adequate. But when I try to set the desired resolution I get a crtc 0 error. xrandr --verbose correctly identifies the desired resolution (1280x800) for Virtual1 and indicates that Screen 0 can handle up to 8192x8192. However, if I try to set any resolution other than 800x600, it fails. Using cvt, addmode, and newmode to try to finesse a new version of that mode gives the same results. FYI, lshw describes the VMWare display adapter as "VGA compatible controller" and "SVGA II adapter" with a vendor of "VMware."
