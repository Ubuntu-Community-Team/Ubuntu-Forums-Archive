### Post by woffie on 2008-10-09
I had the same problem for my printer and this is how to solve it.

Assuming 1-up (one page per page):
1. Print your EVEN-numbered pages.
2. Put the pages back in the feeder.
3. Print your ODD-numbered pages.
4. Done!

Assuming 2-up (or more N-up):
1. Print to a PDF-file and make sure that you use N-up during this step.
2. Continue the same way as above.

Note that you may have to test if you need reverse printing (page 10 to 1 instead of 1 to 10) of pages and in what direction the paper should be put back for optimal result.
You might not need the reverse thing, it depends on your printer hardware.
You might also need to install a PDF-printer somehow, but there should be an option which allows you to print to a file. Be sure that it supports N-up if you need this. I know that Adobe Acrobat Reader has integrated N-up in their software before it reaches the printer (not depending on driver in this case).

In my case, on a Mac, similar way of solving it...
0. Printed to a PDF-printer using the N-up tech, then printed the PDF as...
1. Printed the even-numbered pages.
2. Took the whole bunch of papers and flipped them (as a package) over and put them back into the feeder with the "holes" on long edge.
3. Printed the odd-numbered pages.
4. Manual duplex done! :)

Hope it helps.
/W.
