### Post by UrbenLegend on 2009-05-30
Does this problem occur solely in Firefox? If it is only in Firefox, it could just be Flash grabbing mouse input, in which case this is a non-issue and just a limitation of flash.

If it occurs elsewhere, it could be because Ubuntu is not configuring the input devices correctly, in which case you have to change xorg.conf to specify what mouse driver you want to use.

### Post by UrbenLegend on 2009-06-01
Quick question, have you used the Nvidia control panel to modify your xorg.conf? It looks like the kind of format style that it uses.

The reason I ask is that I think that's where the problem is. Ubuntu automatically configures input devices and by default has no entry in xorg.conf for them. You should remove these entries from your xorg.conf as well as all references to them from the rest of the file. Here's my xorg.conf if you need it.
```
# xorg.conf (X.Org X Window System server configuration file)
#                                                            
# This file was generated by dexconf, the Debian X Configuration tool, using
# values from the debconf database.                                         
#                                                                           
# Edit this file with caution, and see the xorg.conf manual page.           
# (Type "man xorg.conf" at the shell prompt.)                               
#                                                                           
# This file is automatically updated on xserver-xorg package upgrades *only*
# if it has not been modified since the last upgrade of the xserver-xorg    
# package.                                                                  
#                                                                           
# Note that some configuration settings that could be done previously       
# in this file, now are automatically configured by the server and settings 
# here are ignored.                                                         
#                                                                           
# If you have edited this file but would like it to be automatically updated
# again, run the following command:                                         
#   sudo dpkg-reconfigure -phigh xserver-xorg                               

Section "Monitor"
        Identifier      "Configured Monitor"
        Modeline        "1280x800"  83.46  1280 1344 1480 1680  800 801 804 828  -HSync +Vsync
EndSection

Section "Screen"
        Identifier      "Default Screen"
        Monitor         "Configured Monitor"
        Device          "Configured Video Device"
        Option          "TwinView" "1"
        Option          "TwinViewXineramaInfoOrder" "DFP-0"
        Option          "metamodes" "CRT: nvidia-auto-select +1680+0, DFP: 1680x1050 +0+0; CRT: nvidia-auto-select +1440+0, DFP: 1440x900 +0+0; CRT: nvidia-auto-select +1280+0, DFP: 1280x800 +0+0; CRT: nvidia-auto-select +1024+0, DFP: 1024x768 +0+0; CRT: nvidia-auto-select +800+0, DFP: 800x600 +0+0; CRT: nvidia-auto-select +640+0, DFP: 640x480 +0+0; CRT: NULL, DFP: 1680x1050 +0+0; CRT: NULL, DFP: 1440x900 +0+0; CRT: NULL, DFP: 1280x800 +0+0; CRT: NULL, DFP: 1024x768 +0+0; CRT: NULL, DFP: 800x600 +0+0; CRT: NULL, DFP: 640x480 +0+0"
EndSection

Section "Device"
        Identifier      "Configured Video Device"
        Driver          "nvidia"
        Option          "UseEdidDpi" "false"
        Option          "DPI" "96 x 96"
EndSection

Section "ServerFlags"
        Option  "DontZap"       "True"
EndSection
```

### Post by UrbenLegend on 2009-06-19
Your twinview configuration only consists of these lines.

```

Option         "TwinView" "1"
Option         "TwinViewXineramaInfoOrder" "CRT-1"
Option         "metamodes" "CRT: 1280x1024 +0+0, DFP: 1024x768 +1280+0"

```

So based on the default xorg.conf, adding your twinview configuration should amount to.

```
# xorg.conf (X.Org X Window System server configuration file)
#                                                            
# This file was generated by dexconf, the Debian X Configuration tool, using
# values from the debconf database.                                         
#                                                                           
# Edit this file with caution, and see the xorg.conf manual page.           
# (Type "man xorg.conf" at the shell prompt.)                               
#                                                                           
# This file is automatically updated on xserver-xorg package upgrades *only*
# if it has not been modified since the last upgrade of the xserver-xorg    
# package.                                                                  
#                                                                           
# Note that some configuration settings that could be done previously       
# in this file, now are automatically configured by the server and settings 
# here are ignored.                                                         
#                                                                           
# If you have edited this file but would like it to be automatically updated
# again, run the following command:                                         
#   sudo dpkg-reconfigure -phigh xserver-xorg                               

Section "Monitor"
        # HorizSync source: edid, VertRefresh source: edid
        Identifier     "Configured Monitor"
        VendorName     "Unknown"
        ModelName      "Dell E193FP"
        HorizSync       30.0 - 83.0
        VertRefresh     56.0 - 76.0
        Option         "DPMS"
EndSection


Section "Screen"
        Identifier      "Default Screen"
        Monitor         "Configured Monitor"
        Device          "Configured Video Device"
        Option          "TwinView" "1"
        Option          "TwinViewXineramaInfoOrder" "CRT-1"
        Option          "metamodes" "CRT: 1280x1024 +0+0, DFP: 1024x768 +1280+0"

EndSection

Section "Device"
        Identifier      "Configured Video Device"
        Driver          "nvidia"
EndSection
```
I have included your monitor in just in case your twinview configuration depends on it.
