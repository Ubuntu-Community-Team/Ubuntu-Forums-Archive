I have been programming on everything from the days of 6502's, any compiler will use any processor to some extent.  Since the 386, pcs have been doing multi tasking or multi-threading apps with the right operating systems.  

Just choose a language and start learning, but to me algorithms are more important than any language syntax.  I personally know about 7 programming languages as the count stacks up with php.  Lately been tearing apart the Lazarus IDE to make changes to it, its Object pascal.  Learned C and C++ in the early 90's.  Don't see a reason to mess with Forth or Lisp.

the processor will not make any difference as long as the language and compiler will use it.  There aren't many cpus that gcc won't use and it supports multi-threading of apps on Linux and windows.
