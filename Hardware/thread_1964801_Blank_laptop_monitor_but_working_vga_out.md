### Post by timparklancs on 2012-04-24
I converted my asus eee pc netbook to ubuntu a while ago. I recently upgraded to 12.04. It worked well until I connected the vga out to a display. The next time I booted it the display went blank after displaying the ubuntu screen. I could get recovery mode working at a very low resolution but nothing more. After a few goes at booting in recovery mode the display started to go blank before BIOS, you can still see an underscore in the top left briefly. However the vga output would display BIOS then go blank.

I tried reinstalling with 11.10. This appeared to go fine, the vga display is working now but the laptop screen is still blank except for a brief flicker before BIOS. I can turn both monitors on in Display properties but the laptop screen is still blank.

Any advice on either fixing the problem or at least a way to check if it is a hardware problem.

### Post by timparklancs on 2012-04-25
Just for info I have solved the issue. 

I looked through the BIOS menu again and couldn't find any options which might be relevant. There was however an option to load the default BIOS setting so I used that and it solved the issue.

Not sure which BIOS settings were changed or how they got changed but all seems well now.
