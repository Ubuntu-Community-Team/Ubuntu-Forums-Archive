### Post by rbaleksandar on 2011-12-02
Hi, guys.

Using 10.04 LTS with kernel version 2.6.32-34-generic-pae. It was maybe 2 weeks or so ago (no new kernels installed) when I noticed a strange behaviour. I have a notebook Sony Vaio with a keyboard that also has a NumPad. When I am at home I prefer using my external USB keyboard (much more comfy; SpeedLink). I also have Windows 7. Both keyboards work perfect under Windows. First it started like this:
- Phase 1: Keyboards (incl. NumPads) work under Ubuntu
- Phase 2 (since approx. two weeks ago): turning off the NumPad (no matter on which keyboard) actually turns it on
- Phase 3 (since approx. one week ago): NumPad on laptop works ONLY when external keyboard is plugged in
- Phase 4 (since 3-4 days): NumPad on the laptop doesn't work at under Ubuntu (it works as if it is deactivated, that is - I can't type in numbers with it, use PLUS, MINUS etc.)

As mentioned above - I don't have any problems under Win7.

If anyone can help, please do so, because I need the NumPad (work with plenty of numbers and using it is much faster when typing)!

Greetings,
rba


EDIT: Ok, a college of mine just gave me a suggestion that this problem might be due to the settings of the keyboard namely:
System->Preferences->Mouse Keys->DISABLE "Pointer can be controlled using the keyboard"

It worked like a charm. Although this doesn't explain Phase 2 and Phase 3 AND the fact that I have never touched this setting before. :eek: If someone can still explain why this happened, I would be really greatful.
