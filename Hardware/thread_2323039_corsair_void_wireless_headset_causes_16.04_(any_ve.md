I could very well do most of my computing on these except that at night I use my wireless corsair headphones and when I have these plugged in, the GUI locks up randomly so that I cannot switch windows or move them. The top panel does not respond either. ALT+F4-ing all windows back to the desktop unfreezes the top panel (haven't tried this in Kubuntu, but I guess that would work there too.) I did google this and it is a known issue with the mouse and the headset conflicting (though it also seems to make the window switcher not respond to the Win-key.) I tried one fix I found involving making a xorg.config file that tells the mouse to ignore the headphones (link: [https://www.c0urier.net/2016/corsair-gaming-void-usb-rgb-linux-fun](https://www.c0urier.net/2016/corsair-gaming-void-usb-rgb-linux-fun)) and added it to /usr/share/xorg.config.d/ but it did not solve the issue. Another supposed solution found here: [http://forum.corsair.com/v3/showthread.php?t=153831](http://forum.corsair.com/v3/showthread.php?t=153831) tells involves something called 'mkinitcpio,' which as far as I can tell is not in Ubuntu.

Any ideas on how to solve this or make the fixes I found work? Really appreciate any help getting this solved. :)
