### Post by ceeriael on 2010-04-28
2 problems here....

1) The core i5 / h55 is not fully supported as pointed out. 
2) The network card is not fully supported, but it seems that someone found a better driver, and hereby fixed the problem: 
[http://www.zotacusa.com/forum/topic/2693-h55-itx-ubuntu-910-nic-stuck-at-10mbs/](http://www.zotacusa.com/forum/topic/2693-h55-itx-ubuntu-910-nic-stuck-at-10mbs/)

1.a, It seems to be a problem with the gfx, but disabling ACPI has also helped some people. [http://ubuntuforums.org/showthread.php?t=1380636](http://ubuntuforums.org/showthread.php?t=1380636) 
Ubuntu dosent realy "crash" it actualy freezes the display output. So if you ssh in to your box, you can still use the mashine. 
1.b It seems to be a problem with Gnome, if you install kubuntu (sudo apt-get install kubuntu-desktop) you wont get these freezes. But ive seen with kubuntu under X that the processes "lock up" and stops if you dont use your mouse every 15 secound or so :) So if you have a terminal refreshing with some info, it will "hang" at some point, and continue when you move your mouse.... very strange....

If you dont use X, there seems to be no problems (like with Gnome on regular ubuntu).

Hope this helps, ive been strugeling with my h55 board for some days now. And im stuck with a network card that cant get link status unless i power of the mashine, take out the power cord from the PSU, removes the battery, and wait 30 sec. Then when i power up, link status if fine, and my NIC lights are all normal... very spooky. But it seems to be a problem with the R8169 driver for my 8168 card..... For now i fix the problem as written, its only sometime the NIC gets in a state where ubuntu no longer can work with it.

### Post by rinseaid on 2010-10-16
I have just purchased the Zotac H55ITX-C-E and wanted to share the following:

To successfully sleep in 10.10, I had to follow the instructions here: 

[https://wiki.ubuntu.com/MaverickMeerkat/ReleaseNotes](https://wiki.ubuntu.com/MaverickMeerkat/ReleaseNotes)

> When the XHCI module is loaded for USB 3.0 operation the system cannot suspend. Manually unloading XHCI will allow suspend to complete normally. To avoid future suspend problems, the workaround is to add SUSPEND_MODULES="xhci-hcd" to /etc/pm/config.d/unload_module then the system can suspend normally.

I also found that I could get some functionality from the ralink RT2860 wireless card by blacklisting the rt2860sta and adding the modprobe for rt3090sta as per [http://ubuntuforums.org/showpost.php?p=7644876&postcount=10](http://ubuntuforums.org/showpost.php?p=7644876&postcount=10)

Note that I'm pretty sure either 5ghz or wireless-n wasn't working, don't know which. Without making the above changes, scanning wasn't working at all and manually specifying an essid wasn't working either.

Even though there is no BIOS option for it, wake on lan works fine.
