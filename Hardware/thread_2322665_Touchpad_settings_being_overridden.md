### Post by Keith_Helms on 2016-04-29
My laptop's keyboard is arranged such that when my index fingers are on the normal F and J keys for typing my right palm is resting on the upper right corner of the touchpad.  This causes random pastes to take place when typing because of the synaptic RTBottomCorner setting.  In Xubuntu 14.04 I had a startup script that turned that option off, but the same doesn't work in Xubuntu 16.04.  It is getting overridden somewhere.

The script 
```
#!/bin/bash

/usr/bin/synclient PalmDetect=1
/usr/bin/synclient PalmMinWidth=5
/usr/bin/synclient RTCornerButton=0
/usr/bin/synclient RBCornerButton=0
/usr/bin/synclient ClickFinger1=0
/usr/bin/synclient ClickFinger2=0
/usr/bin/synclient VertEdgeScroll=0
/usr/bin/synclient HorizEdgeScroll=0
now=`date`
echo "ran at $now" > /home/keith/touchpad.txt
```

I stuck in the echo command just to verify the script is running at startup and it is.

### Post by Keith_Helms on 2016-04-29
Well, I guess I fixed it.  Under the Settings Editor for xfce4 there is a pointers section.  There were two levels of names and then an option that had an array value with 2,3,0,0,0,0,0,0,0 (think it was 6 zeros).  For some reason, edit was ghosted out for that option.  I clicked on reset instead and all 3 levels were apparently deleted with no undo option.  Since it was only that one option, I wasn't too worried about losing it.  I rebooted and this time around, the RTCornerButton option stayed set to 0 like I wanted it to. 

It would be nice to know how to change that option without deleting it, but as long as no problems crop up from it being gone, I'm not going to lose sleep over it.

One more factoid.  After resetting that Settings Editor pointer option, the "tap to click" option which I had disabled, became reenabled. This time the SynPS2_Synaptics_TouchPad=>Properties=>Synaptics_Tap_Action property has a value of [0,0,0,0,0,0,0] which means it is leaving the RTCornerButton and RBCornerButton options set to 0 like I want.

I haven't found anywhere in the xfce menus where I can actually set those corner button options, so maybe they're just defaulting to whatever value those options have when that property gets created.  It's an annoying thing to do and rather hard to track down.
