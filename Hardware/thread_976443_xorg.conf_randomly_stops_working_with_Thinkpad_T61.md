I'm using a Thinkpad T61 with an external monitor and a Nvidia NVS 140M video card. Ubuntu 8.04 is installed. I use both monitors at the same time with Xinerama. Sorry if my explanation is too long, I tried to make it as short as I could.



This is my problem:

1. Open *jockey-gtk* (Opened by clicking System > Administration > Hardware Drivers) and select the closed source Nvidia driver. This generates a new *xorg.conf* that uses the closed source driver (necessary to run the *nvidia-settings* gui).

2. Restart the system.

3. Start Gnome

4. Open the *nvidia-settings* gui and create an *xorg.conf*. This one has Xinerama support and is the only one that works properly with two monitors.

5. After using my system for a few days and turning it on and off a few times the x-server seemingly randomly fails to start with the closed source driver (errors are at bottom of post). I start the x-server with the *xorg.conf* created when I installed Ubunutu which uses the open source driver.

6. Open *jockey-gtk* and select the closed source Nvidia driver.

7. Restart

8. Start Gnome with the xorg.conf created in step 4 by the *nvidia-settings* GUI.

9. Go back to step 5.



Why does the xorg.conf stop working?
What does Hardware Drivers do when I enable the closed source nvidia driver?
Is there any more Information I can provide to help?
