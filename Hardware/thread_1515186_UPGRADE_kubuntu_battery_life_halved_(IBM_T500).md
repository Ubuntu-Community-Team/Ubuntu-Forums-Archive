### Post by docsmooth on 2010-06-22
Last weekend I upgraded my Kubuntu 9.10 system to Kubuntu 10.04 using "apt-get dist-upgrade".

I removed the lightweight ps-util power manager and replaced it with laptop-mode tools.  I kept my original laptop-mode.conf after verifying that the only differences from mine to the package were in configuration settings (diff below).

Powertop now lists my system as using 34-36W on battery, rather than the original 17.  This puts me from a 3.5 hour battery to a 1.5 hour battery (I did test that the battery life of 1.5 hours was not a reporting error).  As a weekly airline traveler, this just isn't going to work (I'm 2 days from restoring my 9.10 backup).  I'm willing to fix it, since graphics (KDE in particular) run so much more smoothly.

This made me think that the problem was the ATI Radeon graphics driver, but switching the laptop BIOS to "integrated graphics" (forcing the i965M chipset instead) only saved 1W power, the same as it did in 9.10.  I rebooted to my original (custom w/ Core2 rather than generic processor type) kernel, and saw the system still using 36W, so it does not appear to be a kernel configuration or driver issue between 9.10 and 10.04.

With Powertop, I've traced every setting off as I had set previously, and followed every suggestion.  i've even disabled nepomuk and strigi, installed and configured thinkfan, and still can't get below 32W.

I'm not sure how next to trace what's causing so much power consumption on this system, but the top 2 lines in powertop, below, were not something I recall from my 9.10 output:

I do wonder what's causing the serious battery drain that many are complaining about, and why it doesn't seem to be affecting my machine.

### Post by docsmooth on 2010-06-28
Install "laptop-mode-tools" and turn on all the options in conf.d that powertop suggests you manage.

Laptop-mode-tools is an *extremely* powerful power management system that allows you to change settings between battery and AC mode for performance / power.  I have further details on my original laptop-mode configuration at [http://www.totalnetsolutions.net/2007/08/13/how-to-increase-battery-life-in-ubuntu-or-debian-linux/](http://www.totalnetsolutions.net/2007/08/13/how-to-increase-battery-life-in-ubuntu-or-debian-linux/) - but that was written for Ubuntu 8.10 - I'm working on an update based on my configuration posted earlier in this thread.

What does Powertop say your total consumption on batter is now, in Watts?  With a fresh install and a custom kernel and the Intel video, rather than ATI, I was able to get down to 12W on my Thinkpad T500.

For the wakeups - the only thing I'm aware of that you can do is build a custom compiled kernel with the following settings:
CONFIG_NO_HZ=y
CONFIG_HZ_100=y

This enables "tickless sleep" and lowers the interrupt checks to 100Hz.  However, lowering your interrupt check frequency can reduce performance for interactive events like mouse movement, joystick, keyboard action, and very likely can hurt video gaming performance.  However, most movie playing and internet usage probably won't notice a difference.
