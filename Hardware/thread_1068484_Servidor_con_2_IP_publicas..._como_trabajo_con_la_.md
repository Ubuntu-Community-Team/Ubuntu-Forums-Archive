### Post by rober-mpp on 2009-02-13
1) Podrias crear una maquina virtual y ponerle un bridge en la configuracion de redes para que utilice directamente una de las ip's publicas. De esta manera tendrias literalmente otro dominio y podrias implementar todos los servicios que quieras.

2) Si utilizas DNS no deberias necesitar otro dominio. Simplemente utilizando dos DNS distintos, uno para cada ip publica, y un poco de redireccionamiento con iptables, podrias tener dos servidores de mail corriendo en distintos puertos en la misma maquina, y cada uno recibiendo el trafico de una ip ( o dns ) distinto.

### Post by fmsismo on 2009-02-13
Supongo que es un servidor y que lo vas hacer desde consola.  También asumo que estas usando una sola interface de red a la cual le tenes que asignar 2 ips.

en el archivo /etc/network/interfaces (tenes que usar vi o pico o nano o el editor ascii que te guste).  Tenes que declarar las ip's que vas a usar.  Tenes por ejemplo la eth0 (asumo que la placa es eth0, pero puede ser ethX) y después las ip's virtuales eth0:0, eth0:1 ... eth0:X.

El archivo de configuración te queda así.

auto eth0
iface eth0 inet static
	address 10.0.0.10
	netmask 255.255.255.0
	gateway 10.0.0.1

auto eth0:0
iface eth0:0 inet static
	address 10.0.0.11
	netmask 255.255.255.0
	gateway 10.0.0.1


Donde si pingueas a la 10.0.0.11 o 10.0.0.10 contesta el mismo equipo a no ser que hagas un redireccionamiento con iptables.

Si estas poniendo un servidor en internet verifica el fw y si tenes que virtualizar, no hagas bridge, poner todos los equipos en una red virtual y haces nat de los servicios que consumis.  Te quedaría el host físico como gateway.

Saludos
