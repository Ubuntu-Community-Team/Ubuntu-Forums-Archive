### Post by Doug S on 2016-01-02
Typically, the chip model number is listed along with its nominal maximum non-trubo frequency. However, operationally the CPU frequency scaling driver, depending on the scaling governor setting, will attempt to have set the CPU frequency as a function of load on the system. If your system is fairly idle, then yes 800 MHz, for that chip is normal. However, if the CPU frequency does not increase with system load, then something is wrong.

Why? For power management reasons. The package power increases with CPU frequency and with the number of CPUs doing things.

### Post by Doug S on 2016-01-02
O.K., you are using the acpi-cpufreq CPU frequency scaling driver. (which is odd, which version of Ubuntu are you using?) The thing to keep in mind with that scaling driver verses the intel_pstate driver is that it reports the frequency (pstate) that was asked for and not what the CPU frequency actually was over the last sample time.

The CPU stressor in my example was just a program that I wrote. Below is a more generic example (but still using the intel_pstate driver):

```
doug@s15:~/temp$ [COLOR="#FF0000"]taskset -c 7 yes > /dev/null &[/COLOR]
[1] 17023
doug@s15:~/temp$ grep MHz /proc/cpuinfo
cpu MHz         : 3698.828
cpu MHz         : 3600.679
cpu MHz         : 3640.921
cpu MHz         : 3787.015
cpu MHz         : 3670.007
cpu MHz         : 3696.437
cpu MHz         : 2100.828
[COLOR="#FF0000"]cpu MHz         : 3799.898[/COLOR]
doug@s15:~/temp$ fg
taskset -c 7 yes > /dev/null
^C   [COLOR="#FF0000"]  <<<<<<< wait several seconds here[/COLOR]
doug@s15:~/temp$ grep MHz /proc/cpuinfo
cpu MHz         : 1630.140
cpu MHz         : 1687.781
cpu MHz         : 1626.953
cpu MHz         : 1621.507
cpu MHz         : 1603.046
cpu MHz         : 1599.859
cpu MHz         : 1599.859
[COLOR="#FF0000"]cpu MHz         : 1599.992[/COLOR]
doug@s15:~/temp$

```

EDIT: Gary, I had not seen your edit when I replied. Your line saying "3601" means the CPU is in turbo mode and that its frequency is above 3600 MHz. If you want to know the actual frequency you need to use something like turbostat.
Note also that there is only one PLL (Phase Locked Loop) in the processor, so in reality, all CPUs are always running at the same frequency when they are in the C0 (active) state. It is misleading to say that CPU0 is at 2800 while CPU7 is in turbo mode and CPU7 is at 2000 MHz.

Anyway, things seem to be working properly for you.
