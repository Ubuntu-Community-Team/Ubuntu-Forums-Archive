### Post by Jarn on 2009-04-05
Hi, I have a CRT monitor that supports up to 1600x1200 @ 85Hz but I can't get it to work at that resolution in Linux. The highest it will let me go is 1440x900 and that is set up funky, too (presumably because it's a widescreen resolution and not a widescreen monitor). It doesn't show all of the screen on the monitor at once but if I move the mouse to a side of the screen the whole screen moves that way.

I have the recommended version of the nVidia drivers from the hardware drivers app.

First, I added the mode with xrandr:
```
xrandr --newmode "1600x1200"  235.00  1600 1728 1896 2192  1200 1203 1207 1262 -hsync +vsync
```
Then I added the modeline to tell it that the mode is supported:
```
xrandr --addmode default 1600x1200
```
Lastly, I tried to change to that resolution:
```
xrandr --output default --mode 1600x1200
```
That produces:
```
xrandr: screen cannot be larger than 1440x1050 (desired size 1600x1200)
```
If I type the last line in again I get:
```
xrandr: Configure crtc 0 failed
```
