### Post by ASULutzy on 2008-05-21
Are you using vesa or the intel driver?

[http://forlong.blogage.de/article/pages/Compiz-Check](http://forlong.blogage.de/article/pages/Compiz-Check)

You can go there and download a nice little script called compiz-check if you are unsure what driver you are using.

The "intel" driver (xserver-xorg-video-intel) should be able to do all sort of nice compiz stuff no problems.

If you are using "vesa" or "i810" instead of "intel" first do ```
sudo apt-get install xserver-xorg-video-intel
``` then set the driver to "intel" in /etc/X11/xorg.conf and restart X (ctrl+alt+backspace)

### Post by elyk on 2008-05-22
run gstreamer-properties (for Totem Movie Player) or go into the vlc video output preferences, and change the video output method to X11 (it might be listed as "no xv" in gstreamer preferences) - it'll use a little more processing power (translated: your battery will die faster if you're playing videos away from an outlet), but it should fix any problems with compiz messing up your video output - your video should continue playing even if you activate scale or the cube.

### Post by Forlong on 2008-05-22
> **marks_linux said:**
> I'm using the Xgl Rendering Method, that good or bad?
That's the reason why you don't have direct rendering.
You don't need Xgl with an intel chip, remove it:
```
sudo apt-get remove xserver-xgl
```
Then restart X and try again.
