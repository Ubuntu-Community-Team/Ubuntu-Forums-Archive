### Post by nef131 on 2012-02-05
When I log on to my laptop without a mouse already plugged in, the touchpad does not function at all until I either first type **Fn F3** or, if I have started up a program called Touchpad-Indicator, if I type in **Ctrl Alt c**. By contrast, if the mouse is already plugged in when I log in, touchpad functions from the outset.  Also, if the mouse is plugged in after I log in, the touchpad will start as soon as the mouse is plugged in. I want the touchpad to function from the outset and only be off if I want it off.

I would like to write a script which runs when I log in and which, if my mouse is not plugged in, automatically starts up my touchpad. Best for me would be if it does the equivalent of my typing Ctrl Alt c.

Note: the touchpad on my laptop is the Alps Glidepoint (which, if you have this touchpad, you know all the problems it has) and, in order to obtain two finger scrolling, I installed [this program]("http://people.canonical.com/%7Esforshee/alps-touchpad/psmouse-alps-0.10") which, I suspect, causes the problem. Without the above linked to program, I would not have two finger scrolling, so I want to keep the program.

The other program I mentioned, Touchpad-Indicator (which is not the above linked to program), is merely a simple program that was written to allow a user to turn the touchpad on and off entirely and **Ctrl Alt c** is used to turn touchpad either on or off - i.e., the noted keys turns off the touchpad if it is on and turn the touchpad on if it is off. I used Touchpad-Indicator all the time until I discovered the program linked to above because it solved the problem of having the cursor jump around, thus causing disasters when I would type anything, and solved the problem of not having two finger scrolling.

In any event, I have never written a script. If anyone could help me with this, I would appreciate it.
