### Post by BigSilly on 2012-09-22
I posted up recently about an install I did for the missus on her new Samsung laptop. This went really well and we were really happy with it. However we have uncovered one problem that I just don't understand. When using the volume function keyboard shortcut (Fn-F6,7 & 8, volume up down and mute), the whole desktop crashes. The other functions work just fine, such as brightness level and whatnot, it's just the volume adjustment that crashes it.

I can use the normal GUI method of adjusting the volume meter just fine, it's just using the function key shortcuts that crash it. If I so much as tap the volume up, the bar runs all the way to the top and the desktop crashes and I cannot click on anything. Only a hard reboot fixes it.

For what it's worth this does not seem to be a sole Ubuntu problem. I also tried openSUSE live and this displayed a similar result.

The spec is a Samsung NP355V5C-A05UK, with an AMD A6 4400 2.7Ghz dual core, 6Gb RAM, and a Radeon HD 7520G integrated GPU with which I have installed the proprietary driver. Other than the complaint performance is excellent.

Many thanks for any help you can offer. I will also note that I have installed the "Linux on my Samsung" PPA to enable various tools, but none of these have helped the issue. Thanks again.
