### Post by Aearenda on 2009-05-31
It's been using the fallback VESA driver - that's why everything is so slow. I noticed your system is using Xorg 1.4, so these might be annoying it:
```
	Option "AccelMethod" "EXA"
	Option "MigrationHeuristic" "greedy"

```
Try taking them out of /etc/X11/xorg.conf and restarting.

Edit: the contents of /var/log/gdm/:0.log might be interesting too if its still falling back to Vesa. You'll have to escape the name like this in the command line: /var/log/gdm/\:0.log (note the backslash).
