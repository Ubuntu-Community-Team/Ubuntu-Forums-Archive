### Post by Favux on 2012-05-04
This will downgrade xf86-input-wacom a little further.  I have you downloading xf86-input-wacom onto the Desktop but if you already have a git clone folder somewhere you can just change directory into it and then do a 'git pull' to make sure it is up to date.
```
cd ./Desktop

git clone git://linuxwacom.git.sourceforge.net/gitroot/linuxwacom/xf86-input-wacom

sudo apt-get update

sudo apt-get install build-essential libx11-dev libxi-dev x11proto-input-dev xserver-xorg-dev libxrandr-dev libxinerama-dev libncurses5-dev xutils-dev autoconf libtool pkg-config libudev-dev

sudo apt-get upgrade

cd xf86-input-wacom

git checkout a9c7b56aa13992ec11b87c726699cf685857640c

./autogen.sh --prefix=/usr

make

sudo make install
```
Then restart.  That should put about where you were with Natty.

Later if you want to return the xf86-input-wacom folder to "normal" just change directory into it and enter:
```
git checkout master
```

### Post by Favux on 2012-05-05
Great!  Just right click on it and Create Archive and attach it to your next post with Manage Attachments at the bottom of the posting window.

Come to think of it you might want to print your evtest output to a file if you are in the virtual terminal without X.  So something like:
```
sudo evtest /dev/input/event? > ~/Desktop/evtest.txt
```

Usually Ctrl-z or Ctrl-c stops a process.

To answer your question use the output of 'xinput list' to get the <device name>.  I used yours from earlier.  Or if you did a 'xinput list' on the current session you could use the corresponding ID#'s instead of the <device names>.
```
xinput set-prop "Wacom Bamboo1 5x8 stylus" "Device Enabled" 0
xinput set-prop "Wacom Bamboo1 5x8 eraser" "Device Enabled" 0
xinput set-prop "Wacom Bamboo1 5x8 cursor" "Device Enabled" 0
```
Then use the same commands changing the 0's to 1's to re-able.  Or restart of course.
