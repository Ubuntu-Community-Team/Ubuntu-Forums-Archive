So far I've noticed:
1) The printer works fine if it's on while the computer is booting up.

2) If I suspend the computer and come back, the printer still works.

3) If I suspend the computer, turn the printer off and on, the bring the computer back, I have the reload the firmware (as expected) but the printer works.

4) If I turn the printer off and on  with the computer on, I cannot communicate with the printer and I can not manually load the firmware.

5) If I have the printer off while booting, I cannot manually load the firmware, or communicate with the printer, once booted.

Also, I noticed that the firmware is being loaded during the boot process when it mentions ACPI starting (which is why I think this might be an ACPI problem
