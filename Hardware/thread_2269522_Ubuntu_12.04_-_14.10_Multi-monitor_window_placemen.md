### Post by micwallace on 2015-03-16
I have this really annoying problem with Ubuntu causing windows to open on the wrong monitor. 

  This can happen when opening programs but it is more noticeable when  opening a window of a currently running program. 
The new window will  open on the other monitor (often switched off) rather than the one the  program is already running on. 

  This also happens constantly when making web videos full-screen in firefox.

  Not sure if it's also related, but the lightdm login screen also shows on the secondary monitor.

  This time-wasting bug has existed since 12.04 and it's really starting to get on my nerves!
I have setup a keyboard shortcut to move windows between monitors, but this doesn't help if the window is unfocused and can become confusing when the secondary is off.

  I have used AMD and Nvidia GPUs and they both yield the same results.

  I have also switched to Gnome shell due to another issue with Unity and  although Gnome has helped with the problem in some programs (due to it's  attached modal dialogs), the issue persists.

  I have tried changing the compiz config manager window placement settings with no luck. These settings do not seem to make any difference at all.
It makes me wonder if another plugin is interfering.

  If anybody has the solution that would be pure awesomeness and much appreciated!

  Even if you can make all new window shows on my primary monitor that would at least be a good workaround.
