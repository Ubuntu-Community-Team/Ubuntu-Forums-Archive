### Post by cocopuffz on 2009-03-05
I'm sure there is a better way..,. but if you just installed the a new nvidia card as a "upgrade" you may be at it for a while. If you run Envyng it will install the card just fine, but you won't get the newest drivers. The newest drivers are waayyy smoother... It's worth the pain to get them working IMO

I am personally in the middle of an upgrade from Mint Elyssa back to Ubuntu 8.10 (2nd machine) because I tried to install the newest nvidia (180.29) drivers in Mint & had the same issue you had. I did a clean install of Ubuntu, did the nvidia driver install right after.. and it works perfectly. 

It may have something to do with your Xorg file, but I messed around with mine for hours and nothing helped. If a ubuntu genius doesn't reply within the next 2 hours with the solution... just back up your files and do a clean install of 8.10 and then install the new nvidia drivers after.

You can try "gksudo displayconfig-gtk" and try to select a monitor that best fits yours... do a autodetect.. and see if that helps. Sometimes after a fresh display driver install, the new xorg file has no idea what kinda of display you have, so you have to tell it. But that only worked for me on another machine a few times. If your LCD is some obscure brand... good luck. Try the Plug & play driver that matches your unit best & cross your fingers. Didn't work for me this time... but it may help with your situation.

### Post by K. Hendrik on 2009-03-05
HI,
to install the driver press "Strg + Alt + F1" to switch to tty1

now
```
sudo /etc/init.d/gdm stop
```
then run 
```
sudo sh NVIDIA-Linux-x86-180.29-pkg1.run
```
it will ask do you want to download the kernel modul select no
and let it compile the module. after that it will ask do you want to update xorg.conf select yes.
now restart gdm using
```
sudo /etc/init.d/gdm start
```

that should do the trick
