### Post by cidthecoatrack on 2011-06-07
So I had an old eMachine that I converted into a Xubuntu box.  Quite nice, runs smoothly, but only had 800MB or so of RAM.  So, bought 2 GB of ram (x2 Transcend DDR 400 1GB modules), checked all sorts of compatibilities, and plugged them in.  Xubuntu wouldn't boot.  No error messages, just a black screen after the eMachine logo.  Went into the BIOS, double-checked, and it recognizes all of the RAM (the old plus the new, total just under 3GB).  This system runs an Athlon 64 (0.09) 2210 MHz on a MIS 7207 Ver. 2.0 Motherboard.  If the BIOS recognizes the RAM, why won't Xubuntu boot?

I am currently doing a memory test, but the memory should be good.

### Post by wolfen69 on 2011-06-07
Take out the old memory and run just the new and see what happens. Make sure memory is seated properly.
