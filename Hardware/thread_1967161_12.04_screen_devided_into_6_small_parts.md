### Post by nxhoaf on 2012-04-29
Solved! Thanks to [fufurush]("http://ubuntuforums.org/member.php?u=1434184"), 
The problem is due to Nvidia. In fact, lastest Nvidia driver didnt work.
For those who have the same problem, I suggest these steps below:
Required:[ NVIDIA-Linux-x86-290.10.run ]("http://www.nvidia.com/object/linux-display-ia32-290.10-driver.html"), for you, It's maybe different. Check Nvidia dowload page for your suitable driver.

**1. Login **

**2. Remove previously installed nvidia (if any)**. If not, you can skip to step 3 
Run either one of these two commands:
```

 sudo apt-get purge nvidia*
or 
sudo nvidia-uninstall

```**3. Black list the Nouveau driver** (It's actually a reverse engineering Nvidia's current proprietary drivers for Linux), see [wikipedia]("http://en.wikipedia.org/wiki/Nouveau_%28software%29") and[ Nouveau's home page]("http://nouveau.freedesktop.org/wiki/") for more information. For code'explanation, [see here]("http://us.download.nvidia.com/XFree86/Linux-x86/290.10/README/commonproblems.html#nouveau")
```

sudo touch /etc/modprobe.d/disable-nouveau.conf
  sudo gedit /etc/modprobe.d/disable-nouveau.conf
  Add the following two lines, save then close:
  blacklist nouveau
  options nouveau modeset=0

```[B]4. Restart and login into tty1 screen
[/B] After Ubuntu 12.04 installation, at the login screen, hit "ctr + alt + f1", login with your username and pass
[B]
5. Turn off Xorg [/B]
```
 
sudo services lightdm stop

```[B]6.Install
[/B]Locate the Nvidia driver and install it.
```
 
sudo sh ./NVIDIA-Linux-x86-290.10.run

```During installation, you should choose yes as they ask you if you want to let them to write to xorg.conf file if not, you must run it manually after using this command:
```

sudo nvidia-xconfig

```**7. Enjoy!!!**
Reboot, and you should now see the  NVIDIA logo on startup.
