### Post by tackfurlo on 2008-07-24
Not to pile on after the fact too much, but I just wanted to mention this...

AMD Processors are typically 10% to 20% slower than the Intel equivelent, HOWEVER every current generation AMD motherboard ships with a 1.6GHz FSB while every current Intel chip ships with an 800MHz FSB, meaning you can get data into and out of the processor twice as fast on a current AMD system.

What this translates into is if you're processing data which the processor has to sit and grind on for at least 4 CPU cycles then the Intel will be faster, but if it takes it only 2 CPU cycles to do whatever it needs to do with the data (unless the video is very high def, then this is likely the case) then the AMD will run faster than the Intel.  For daily desktop use, an AMD is almost always better than the equivelent Intel unless you're running around 80%+ CPU capacity at any given time.  My laptop is a dual core 2.2GHz AMD Athlon (it's 64 but but I run 32 bit Ubuntu Hardy on it) and in "Ondemand" CPU scaling mode it seldom tops 1.6GHz per core, and typically sits at the default 800MHz per core, running 3 VNC sessions in Wine, Evolution with over 40,000 emails, Firefox with about 10 to 15 tabs open, Pidgin chatting with about 10 people, and a few other minor things.  Plus with no SWAP file and 800MB of my 2GB of memory free :)

Anyhow...anything that's super-intensive will run better on an intel but for even something like editing home movies, an AMD is still the better choice, simply due to the superior FSB speed.

### Post by BlueSkyNIS on 2008-08-16
I recently learned that AMD Phenom is much better CPU than currently presented by various websites. Main problem lies within Cool & Quiet technology implementation and motherboard manufacturers and it impacts its performance and power efficiency. 

About Phenom low performance:
There are indications that software which is controlling Phenom power state is broken. This problem is more evident in Vista than in XP. When CnQ is enabled and test are conducted on a Phenom, the end result is not always consistent. This was discovered by anandtech, read article [here]("http://www.anandtech.com/cpuchipsets/showdoc.aspx?i=3344&p=3"). This unusual behavior is furthermore explained [here]("http://www.lostcircuits.com/cpu/amd_phenom9350/16.shtml"). 

About Phenom low power efficiency:
I always do care about power dissipation and I love when computer runs cool. I was wondering why is Phenom consuming so much power in idle. So I went to some hardware forums and start asking people. 

Quote from: [http://forums.amd.com/forum/messageview.cfm?catid=318&threadid=98702&forumid=1]("http://forums.amd.com/forum/messageview.cfm?catid=318&threadid=98702&forumid=1")
> In idle the VRM efficiency is at 50 - 85% range at best with our measurement figures. The core power consumption in idle depends on the motherboard in question, if it implements **AM2+ PSI specification support**. That reduces low load operation power consumption quite well. 

...than I googled **AM2+ PSI specification support** and found out that "[Current AM2+ mobos don't have AMD's Power State Indicator spec enabled]("http://www.xtremesystems.org/FORUMS/showthread.php?t=184971")" hence more power consumption in idle. :(

Only time will tell what and who went wrong with Phenoms. Looks like they have much potential but no one is using it.
I don't know why motherboard manufacturers choose not to implement PSI specs because Phenom would look much better in online reviews. (?)
