### Post by TheFu on 2017-12-30
I have an old Samsung laser printer. It was well supported under 14.04, but not later releases.
My solution was to install 14.04 into a virtual machine, setup USB passthru to it and make it a network printer, available to any other machine for printing on my network.  Using IPP makes this pretty easy to use. I don't use it for anything other than printing. Nothing. The firewall on it allows only ssh and print traffic, NOTHING ELSE inbound or outbound.  It has been a few years since I set this up, so there might be a few other ports/packages installed. Samba comes to mind.

Not ideal, but it works and I don't have to worry about it ever breaking.  Since I print about 50 pgs a year, that VM is only turned on a few days every year (Tax season) or for 10 minutes whenever airline boarding passes need to be printed.

You'd probably want to go with 16.04 for the printer stuff. A minimal 16.04 server with printing support would be about 5GB of storage, which isn't too much today.  Definitely use cqow2 sparse files.

Just an option for your consideration.
