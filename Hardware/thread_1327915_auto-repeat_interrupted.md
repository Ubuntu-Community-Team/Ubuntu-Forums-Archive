I've recently upgraded my 2004 Sempron 3000+ running 32-bit Jaunty to a newer computer - an Acer Timeline 1810TZ laptop, running 64-bit Karmic.

The PS/2 keyboard of the old Desktop is re-used as an external keyboard on the laptop through a PS/2-USB adaptor.  Most of the time, things seem to run perfectly.

However, problem comes up when I'm trying to play an FPS game - Urban Terror.  It ran perfectly well on the old system, but with the new computer, the keyboard seems to be behaving strangely - whenever another key (shift/ctrl/alt/space...) is pressed, it cuts off the keycode of whatever key that's been pressed.

Shift/Ctrl/Alt are better but not much - it seems the key being held was still registered for a split second after the modifier is released.

For example, if I hold W (in the game: forward), and then press-and-release SPACE (jump), the W gets lost even if I never have released it.  On a text area, when I press space while holding W, it'll look like this: "WWWW " - while I was expecting it to look "WWWW WWWW...." as long as I hold W.  When I press CTRL while holding W, it'll look like "WWWW^WWWWW" - it stops after 4 more W's, instead of going ""WWWW^WWWWW...." as long as I hold the key.

So the question is, is it the external keyboard, the adaptor, or 64-bit, or Karmic?  Too many factors are at play here.  When I try the laptop keyboard, CTRL and SHIFT seem to be working but Space isn't.

Have anyone run into similar issues?  Thanks!
