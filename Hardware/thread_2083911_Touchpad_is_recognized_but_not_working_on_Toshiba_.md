### Post by gerowen on 2012-11-14
Make sure that, if you've ever run Windows, you didn't use the function keys to turn off the touchpad while you used an external mouse.

I used to carry around an extra hard drive with Windows on it, and when I used it I would FN+F9 to disable the touchpad while I used an external mouse so I wouldn't accidentally bump the touchpad while typing.  For some reason or another, upon inserting the Linux hard drive again, if I forgot to re-enable the touchpad before shutting down, the touchpad would be totally un-responsive.  Something about the Toshiba software that managed the hotkeys made it so that even after a cold start, the touchpad would not work unless I booted Windows and pressed FN+F9 again to re-enable it.  I was unable to ever find an option in the BIOS that it could have modified, so to this day I'm not certain exactly what it did.

Anyway, just thought I'd mention that as a possibility.
