### Post by AKADAP on 2009-02-08
Try the lshal command to find out what driver your mouse is using. 
Look at the man page for the driver to find out what commands are supported by that mouse driver. 
Also look at the man page for xorg.conf to find out what commands are supported by all mouse drivers.
Figure out what commands you need and put them in a *.fdi file in the /etc/hal/fdi/policy directory.
Open the *.fdi file with firefox to make sure there are no typos (it will report an error rather than display it if there is a typo).
Unplug and plug in your mouse to start using the new *.fdi file.
verify you succeed with xinput list-props

Try:

then type "/mouse" and hit the return key to find the first instance of the text "mouse". a few lines below that should be a line that starts: "info.product = " There a few different items with "mouse" in their names, so type "n" to go to the next instance of "mouse" until you find one whose "info.product = " string looks like your pointing device.
When you find the right one, a few lines below the "info.product = " string, you will find a line that starts "input.x11_driver = " In my case it says "input.x11_driver = 'evdev'", so evdev would be the mouse driver on my system.

### Post by blueshiftoverwatch on 2009-02-26
I found the solution to my question [here]("https://bugs.launchpad.net/ubuntu/+source/xorg/+bug/272334").

Just make a file called "mouse.fdi" in the /etc/hal/fdi/policy/ directory. Than using a text editor paste the following text into into the file and save it:
```
<?xml version="1.0" encoding="ISO-8859-1"?>
<deviceinfo version="0.2">
 <device>
  <match key="info.capabilities" contains="input.mouse">
   <merge key="input.x11_options.Emulate3Buttons" type="string">false</merge>
  </match>
 </device>
</deviceinfo>
```
These changes should take effect after unplugging and replugging in your mouse. Or restarting your computer.
