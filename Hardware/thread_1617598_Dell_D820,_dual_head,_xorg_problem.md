### Post by snaga on 2010-11-09
I have an issue that is at least in part my own making. Not sure what is going on.

I recently re-installed Ubuntu 10.10 on a Dell D820 that has before had 8.04 - 10.04 on it and ran well. I had been doing 32bit, decided to go to 64.

This is a work laptop. In the office, I attach it to a dock and drive a dual head display. At home I use it as a normal laptop. Previously it had gone back and forth between the two well.

My mistake when I re-installed was that I forgot to back up my /etc/X11/xorg.conf file. Now, when I start up, I only get one of the displays. Going to the Nvidia config panel works quite well. I can set up the dual head, apply changes, which works, and I get two displays. I can save successfully to a xorg.conf file.

But, as soon as X restarts at a login, I am back to one display. It acts as if it is not reading the xorg.conf.

Wondering if anyone has any ideas. Its just a bit of a pain to have to reconfigure at each login.

### Post by snaga on 2010-11-10
A little experimentation with the xorg.conf file did the trick. nVidia xconfig did not create the file quite right, it would seem. Here is the pertinent section that changed from above:


```

Section "Screen"
    Identifier     "Screen0"
    Device         "Device0"
    Monitor        "Monitor0"
    DefaultDepth    24
    Option         "TwinView" "1"
    Option "TwinViewXineramaInfoOrder" "DFP-1, CRT"
    Option "UseDisplayDevice" "DFP-1, CRT"
    Option "TwinViewOrientation" "CRT RightOf DFP-1"
    SubSection     "Display"
        Depth       24
    EndSubSection
EndSection

```

I haven't experimented further to see if there is a more compact way to get the job done. Might do that later.
