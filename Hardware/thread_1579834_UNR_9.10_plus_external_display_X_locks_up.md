### Post by Objekt on 2010-09-22
Has anyone else had problems with the GUI randomly locking up, requiring a system power cycle, while using Ubuntu Netbook Remix 9.10 with an external monitor?

Recently I had about a two-week stint where my Acer Aspire One D150 netbook was my only computer.  I used a 24" LCD monitor, connected to the VGA output, as a second display, for watching movies and whatnot.  Sometimes the GUI went nuts for no obvious reason.  Suddenly I'd have several areas of bright colors, almost like half-formed windows but with solid backgrounds and no text.  The mouse pointer still moved, but I couldn't do anything else.  I couldn't even drop to the command line to restart.  I had to power cycle the netbook.

This happened with no clear pattern, 5-6 times in the 2 weeks I was using the netbook with the external monitor.  It reminded me a lot of the GUI lockups that have been seen in desktop Ubuntu 9.10 on some machines, lockups which were never resolved as far as I know.  But I have never had any problems with GUI freezes using UNR with just the netbook's built-in display, so it seems to be a bug unique to using 2 or more displays.

So what's the deal?  Cosmic rays flipping a bit, known issue, or something in between?
