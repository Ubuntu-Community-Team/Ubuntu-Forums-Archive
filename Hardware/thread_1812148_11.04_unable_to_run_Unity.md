### Post by tempusfuoit on 2011-07-28
First off I would like to state that I'm new to Ubuntu but I'll try to be as concise as I can. I didn't know I was using Nouveau either until I turned off "quiet splash" on the boot screen and saw an error referencing an x-org failed to load Nouveau error during a hang boot. Use "sudo gedit /etc/default/grub" and edit the line
 "quiet splash" to "". Save it and run sudo update-grub2 then reboot This will display what is going on behind the scenes during boot up. If you install Bumblebee from the PPA as of yesterday it has a UI also. For people currently running the script/git version please do:

sudo bumblebee-uninstall
sudo apt-get install ppa-purge
sudo ppa-purge ppa:ubuntu-x-swat/x-updates
sudo ppa-purge ppa:xorg-edgers/ppa
sudo apt-add-repository ppa:mj-casalogic/bumblebee
sudo apt-get update
sudo apt-get install bumblebee

The bumblebee UI is visible in synaptic package manager when you search for bumblebee. Also, the Unity search buttton will show bumblebee UI during app search. Let me know if there are any questions with this.
