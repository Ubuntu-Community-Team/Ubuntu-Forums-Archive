### Post by sudodus on 2016-04-10
The more powerful stick computers are often (but not always) delivered with Windows. You can backup Windows and after that install an Ubuntu flavour, for example ***Lubuntu***.

I suggest that you ***go to a local vendor and ask them to let you try it out***, if it has enough CPU and graphics horsepower plus enough RAM for your application. Remember that Lubuntu is much lighter than standard Ubuntu. It is possible that the cheapest version will work well enough with Lubuntu, but I don't know, I have never tried such a computer.

If you want more horsepower and RAM (and can pay the higher price) than you can get with a stick computer, you can get an ***Intel NUC***. I bought such a computer with an i3 CPU recently (approx size 10x10x5 cm3 = 4x4x2 cubic inches). You can hang the NUC on the back of the TV screen if you wish.

### Post by TheFu on 2016-04-10
[http://www.omgubuntu.co.uk/2016/02/skype-for-linux-group-calls-need-to-update-issue](http://www.omgubuntu.co.uk/2016/02/skype-for-linux-group-calls-need-to-update-issue) and there is a reddit thread [https://www.reddit.com/r/linux/comments/479oyq/skype_partially_dropped_support_for_linux_calls](https://www.reddit.com/r/linux/comments/479oyq/skype_partially_dropped_support_for_linux_calls) about it too.

NUCs are cool, but they definitely charge a high premium for the slightly smaller size.  If you can live with a microATX size, you can get the same power as a NUC for just a little larger package.

I'd look to an ARM platform - something like a R-Pi, but there are 20 other clone vendors making similar stuff.  The Pi v2 is nice and plenty fast for any audio needs (provided the application runs on ARM/Linux) and with the $5 mpeg2/h.264 codecs, all video used today.  In the future, when h.265 becomes used everywhere, either new hardware or new commercial drivers will be required for hi-def content.  If the video content isn't hidef, then no commercial drivers are needed. 

I've never used viber and only use skype sparingly on an old Nokia.  OTOH, I've been using straight SIP and VoIP for almost 15 yrs - runs great on Linux.
