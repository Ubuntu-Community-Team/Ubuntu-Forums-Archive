[LEFT]I have an inherited Acer Aspire 5100; came with dead hard disk and 1G of RAM.  Replaced HD, installed Meerkat, apart from a kernel bug currently being worked on works very nicely.  It's a single-core machine

Decided maybe 1G was a bit on low side, so installed 2x 2G memory sticks of correct flavour.  Machine only sees 2.686 G, even within BIOS.  Swapped memory sticks, same story.

Put each new memory stick in on its own, 2G correctly found and reported, so it's not faulty sticks or memory slots.  

I looked at BIOS upgrades, there are hints in the readme's in most recent BIOS that there was some problem in accessing 4G memory, corrected by the upgrade.

Okay, so far nothing to do with Ubuntu (which sees and works nicely with the 2.6G of memory).  Question is this: any flash upgrade carries a risk of totally messing up the machine.  I've got a machine with 2.6G.  So is it worth risk of flashing BIOS to gain 4G?  In other words, does 4G give really major advantages over two and a half gig?

I use firefox, VirtualBox, GIMP, so far nothing else that I can think of that seems to make major claims on memory.

Any opinions?
