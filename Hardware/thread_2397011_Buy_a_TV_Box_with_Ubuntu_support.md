### Post by TheFu on 2018-07-24
Welcome to the forums.
Acceptable performance is vague as a requirement. Everyone has a different idea for what that means.  

A carefully selected $90 used PC will have much better performance than any new Android device near that price.
If you don't use Gnome3 DE and go with an XFCE or LXDE/LXQT DE, then you might find a $100 ARM system that might be acceptable for an Ubuntu Desktop machine if you don't expect too much.  Lots of people have been trying the Raspberry Pi v3+ which you can get setup with almost everything for just under $50.  I wouldn't consider that "acceptable performance" and ARM-based systems don't support all the same software that Intel/AMD systems support.  Most software does work on ARM, however.  But that isn't 100% compatible, another requirement you listed.

All those TV boxes are designed for media playback with GPUs specifically made to handle h.264 and mpeg videos. Some higher-end machines can do h.265 as well.  The streaming box hardware is not designed for general desktop needs - that's why they are so cheap.

If you explain the main driver for your choice, people might be able to make suggestions.  For example, there are some $150 ARM systems which perform well, but they aren't "cheap". ODroid is a board, but I don't think Ubuntu is supported.  There are also, some very low-power AMD/Intel systems if power use/silence is the major concern. These are also in the $100+ range - look for microITX systems.  If size is the main concern, over power user or noise, then a NUC might be the option.

Anyways, more data will help people help  you more.
