### Post by tommcd on 2009-01-04
If your system is usable, open a terminal (applications > accessories > terminal) and run:
```
gedit /bootgrub/menu.lst
```
and:
```
gedit /etc/X11/xorg.conf
```
 Post the output of those commands here. Put the output in code brackets like I did with those 2 commands above so it will be easier to read. This will show if you have any special boot up options or if xforcevesa is being used by your xorg.conf.

### Post by tommcd on 2009-01-05
> **jdoc said:**
> gedit /bootgrub/menu.lst <-- gives me what looks like a blank text document.


Sorry, that was a typo. it should be:
```
gedit /boot/grub/menu.lst
```
(I missed the / between boot and grub).

Anyway, it seems that from your xorg.conf the problem is that you are using the vesa driver, and you need to set your proper screen resolution.
See the section **video driver fix** on this page from the wiki:
