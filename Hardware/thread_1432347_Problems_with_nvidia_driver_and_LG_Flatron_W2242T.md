### Post by jvandewiel on 2010-03-17
Hi all,

Bumping into some very annoying issues with the combination of the (proprietary) nvidia driver and an LG Flatron W2242T flatscreen monitor. Running Ubuntu 9.10, updated to the latest packages, and installed the 185 version of the nvidia driver. Everytime the system boots, the resolution is set to 1024x786 or something like that, however: when using the nvidia-settings I can modify the resolution to 1680x1050, also save it, but it is not maintained.

I have edited the xorg.conf file (see below) to ignore the EDID stuff, since this might be an issue with this monitor, and want to use the defined modeline. But with every gdm restart/reboot it defaults back to the nvidia-auto-select mode of 1024x768 (see Xorg.0.log). Any ideas how to fix this? Thanks in advance!

Joost
