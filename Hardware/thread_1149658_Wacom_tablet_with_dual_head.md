### Post by Rio6000 on 2009-05-06
Hi Timtro,

I'm having the same problem. I've just switched from Arch Linux just to check things out and like you i'm also used to the xorg.conf stuff.

however i've done some research and it seems that it is now controlled by HAL, thus enabeling hotplug control (jeej!) but becouse there are no entry's in xorg.conf the wacom config program wacomcpl can't find the tablet (boeh!).

This means we have to configure everything through xsetwacom which is no big deal actually. just run this to see what your tablet is called:

xinput --list

in our case because we have Bamboo's it's called this:

"Wacom Bamboo"         <-- stylus
"Wacom Bamboo pad"     <-- the button pad
"Wacom Bamboo cursor"  <-- the mouse i think but i don't use it
"Wacom Bamboo eraser"  <-- the eraser

so to change any settings you just type:

xsetwacom set "Wacom Bamboo pad" Button1 "core key delete"

(This sets my upper left button to emulate a delete key stroke)


i've put these commands in a script that runs at start up.
mine is called .wacomsetup.sh in my home directory and i let it run through
System --> Preferences --> Startup Applications
(!! Dont forget to set the script permissions to be executable!!)

### Post by Favux on 2010-12-27
Hi odd,

The default 0.10.5 xf86-input-wacom is an early Xorg version and was somewhat incomplete.  You didn't say what model of tablet pc you have.  To get pressure working you can clone the git repository and get 0.10.10+.  See II, in the [Bamboo HOW TO]("http://ubuntuforums.org/showpost.php?p=9496609&postcount=1").

Note that multimonitor support has been removed from the driver by 0.10-10 on the grounds that it does not belong in the driver.  It was felt it should be a userland application.  There is a new xsetwacom command to set output.  But we haven't figured out if we can do multimonitor support yet with what we have.

Since some people report being able to configure dual monitors with 0.10.5 and 0.10.6 you'll have to decide what's more important to you.
