I followed this howto: [http://ubuntuforums.org/showthread.php?t=246746](http://ubuntuforums.org/showthread.php?t=246746) and tried pretty much every suggested setting, but no luck.

Has anyone gotten this to work?  I haven't used laptop in a while, but it has worked before (using Debian at the time).  Attached is the relevant configs/logs.

### Post by rettw on 2010-09-13
I had the same issue on a Thinkpad A30 with the same video chip.  I was able to get xvideo working by disabling KMS support.  

Afterwards Xvideo starting working (xvinfo reported several adaptors) and I was able to using xine/mplayer/mythtv with X-Video.

Myth had a screen corruption issue after exitting live tv that I solved by disabling DRM support in the x server.  I did this by deleting the radeon module from the system since blacklisting it didn't keep it from loading.

To disable KMS support, do the following as root:

```

echo options radeon modeset=0 > /etc/modprobe.d/radeon.conf

```

Then reboot - the xvideo adapters should be present again.  BTW - on mine I had to not use the X EXA acceleration - I would comment it out in the Xorg.conf file you have shown above.
