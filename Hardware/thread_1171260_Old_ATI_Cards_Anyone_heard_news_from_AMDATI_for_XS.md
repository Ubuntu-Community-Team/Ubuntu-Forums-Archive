### Post by rizman on 2009-05-27
Hi all,
As a lot of users with older ATI cards know, our GPU's have been put on legacy support as of catalyst driver 9.4. Thus, we do not get to get the whole 3D experience on Ubuntu 9.04 because it uses Server 1.6. If you do not know what I'm talking about: there are a LOT of threads about this.

I'm currently using the open-source drivers. But I'm also a developer. I used to develop games using XNA in Windows only 3 weeks ago. I merely fooled around with it, no real game has ever seen the light of day. I finally took the step and completely ditched Win and replaced it with Jaunty...and I'm verys happy with it. What I want to do now is learn OpenGL. However, as far as I can tell, the open source drivers for my chipset only support up to OGL 1.3. So no shader support to play around with :-(.

But back to the topic:
My question actually is if, considering the numerous complaints from ATI customers abount the driver issue not only on these forums but all around the web, ATI is considering to maybe patch the catalyst drivers 9.3, or 9.4 to add support for legacy cards. 9.3 seems difficult as those are not compatible with Xserver 1.6. But what about 9.4?

I'm really considering downgrading to Intrepid. But I have my system set up the way I want now and I'm not fond of redoing it again. 

Note that I'm willing to give it time. I'm not saying this should get fixed right now.

Also, know that I appreciate the work people are doing on the open-source drivers and I highly respect their work. If at any time, they will support OGL 2 with Shader support, I'll be more than happy to use those.
