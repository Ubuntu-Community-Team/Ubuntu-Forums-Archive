### Post by dawynn on 2009-02-18
I have a few games and emulators that allow for the joystick to be an input device.  But X has a really annoying habit where, if the only input is coming in through a joystick, it doesn't see it as input, and will send the screen into power-saving mode after a certain amount of time.  Its very annoying to have to grab the mouse or keyboard every 10 minutes or so.

Is there a way to tell X that input from a joystick should be recognized as active input?  That as long as the joystick is sending input, the screen should not be sent into power-save mode?

I realize that the analog portion of joysticks can send some slight variances that may be misinterpreted as movement when nothing is truly happening with the stick.  Is there a way to set a reasonable tolerance level so that, if I'm not really moving the joystick, the screen can still go into power-save mode (like if I truly do walk away from the computer for 10 minutes)?
