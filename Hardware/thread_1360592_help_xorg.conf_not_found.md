### Post by Melcar on 2009-12-21
Don't use someone else's xorg.conf; it will only lead to problems.  Generate your own and configure it as you see fit:

- Drop to command prompt mode, or boot into safe mode

- Execute the following:

```
sudo Xorg -configure
```

- You now got a file named xorg.conf.new in your root directory, so open it for editing:

```
sudo nano /root/xorg.conf.new
```

- Make your changes and save the file as *xorg.conf*

- Move the new file to /etc/X11:

```
sudo mv /root/xorg.conf /etc/X11/xorg.conf

```

Reboot

### Post by Melcar on 2009-12-22
*Xorg -configure* creates a new one for you.  GDM/KDM must not be running otherwise it will give you an error, so either stop the service or boot up in safe mode and drop to a root command prompt.  When you perform the command your screen should blink/pause for a sec. and then you will get some feedback on the new file.  If you perform the command as root the new file is dropped inside your root folder; if you do it as a user it will be dropped inside your user folder.
