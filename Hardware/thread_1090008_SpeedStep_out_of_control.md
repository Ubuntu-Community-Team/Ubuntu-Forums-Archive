### Post by utkjamie on 2009-03-07
I'm running Kubuntu 8.10 on a laptop with an Intel Celeron processor. While SpeedStep is supposed to only turn on when CPU usage is low, it usually turns on when I most need high CPU usage. The result is that my laptop is essentially put on pause for 10-20 minutes (sometimes as much as 45 minutes!) when I most need it. I can do virtually nothing once SpeedStep kicks in.

Since this is such a huge problem, I went into Kubuntu's Power Management settings and set the computer to run in "performance" mode whenever the AC adapter is plugged in (which is always). Yet, SpeedStep continues to kick in and destroy my productivity at seemingly random moments.

Disabling SpeedStep in the BIOS is a bad move because disabling SpeedStep causes the laptop to run at a lower speed all the time.

I used to have this problem a lot in Feisty, but it seemed to clear up with Gutsy. Now it's back. To make matters worse, the battery meter in my system tray (the one that allows me to manually change SpeedStep settings) has disappeared and I'm not sure how to get it back.

Anyone have solutions or suggestions for getting Kubuntu to respect my power management settings? Anyone know how to get my battery monitor to come back?
