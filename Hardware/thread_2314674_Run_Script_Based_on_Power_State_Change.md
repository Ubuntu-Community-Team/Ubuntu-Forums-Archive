```
echo -n 1 > /sys/devices/system/cpu/intel_pstate/no_turbo

echo -n 54 > /sys/devices/system/cpu/intel_pstate/max_perf_pct

and fire this when on AC

```
echo -n 1 > /sys/devices/system/cpu/intel_pstate/no_turbo
echo -n 54 > /sys/devices/system/cpu/intel_pstate/min_perf_pct
echo -n 100 > /sys/devices/system/cpu/intel_pstate/max_perf_pc

### Post by zenon222 on 2016-02-27
I figured it out.  You want to use UDEV
Documentation on udev itself is sparse, ubuntu seems to implement it slightly different than what is explained on the net.
Here's the synopsis.

Check for the possible keys to trigger from:
```
udevadm info -a -p /sys/class/power_supply/AC
```
The pertinent information is:
```
  looking at device '/devices/LNXSYSTM:00/LNXSYBUS:00/ACPI0003:00/power_supply/AC':
    KERNEL=="AC"
    SUBSYSTEM=="power_supply"
    DRIVER==""
    ATTR{online}=="0"
    ATTR{type}=="Mains"
```

When on battery you will see:
```
    ATTR{online}=="0"
```
When on AC you will see:
```
    ATTR{online}=="1"
```

Create an AC udev rule as such:
```
sudo nano /etc/udev/rules.d/98-50-AC-power.rules
```

From above, trigger your AC script by writing this in the file:
```
# This file was generated by me to speed-up processor and brighten display when on AC power

# AC PLUGGED-IN
SUBSYSTEM=="power_supply", ATTR{online}=="1", RUN+="/usr/local/bin/set-governor-AC"
```

Create a battery udev rule as such:
```
sudo nano /etc/udev/rules.d/99-50-batt-power.rules
```

From above, trigger your battery script by writing this in the file:
```
# This file was generated by me to speed-up processor and brighten display when on AC power

# AC UNPLUGGED
SUBSYSTEM=="power_supply", ATTR{online}=="0", RUN+="/usr/local/bin/set-governor-batt"
```

I also included a critical battery script, to really drop power consumption:
```
sudo nano /etc/udev/rules.d/99-60-low-batt-power.rules
```

trigger your battery batch file by writing this in the file, it will run the script when battery is between 0-19%
```
# This file was generated by me to speed-up processor and brighten display when on AC power

# Low Battery
SUBSYSTEM=="power_supply", ATTR{capacity}=="[0-1][0-9]", RUN+="/usr/local/bin/set-governor-critical"
```



Here are the scripts:

```
sudo nano /usr/local/bin/set-governor-AC
```
```
#!/bin/sh

echo -n 1 > /sys/devices/system/cpu/intel_pstate/no_turbo
echo -n 45 > /sys/devices/system/cpu/intel_pstate/min_perf_pct
echo -n 100 > /sys/devices/system/cpu/intel_pstate/max_perf_pct
echo -n 225 > /sys/class/backlight/intel_backlight/brightness
```

```
sudo nano /usr/local/bin/set-governor-batt
```
```
#!/bin/bash

echo -n 1 > /sys/devices/system/cpu/intel_pstate/no_turbo
echo -n 50 > /sys/devices/system/cpu/intel_pstate/max_perf_pct
echo -n 27 > /sys/devices/system/cpu/intel_pstate/min_perf_pct
echo -n 105 > /sys/class/backlight/intel_backlight/brightness
```

```
sudo nano /usr/local/bin/set-governor-critical
```
```
#!/bin/bash

echo -n 1 > /sys/devices/system/cpu/intel_pstate/no_turbo
echo -n 27 > /sys/devices/system/cpu/intel_pstate/max_perf_pct
echo -n 27 > /sys/devices/system/cpu/intel_pstate/min_perf_pct
echo -n 0 > /sys/class/backlight/intel_backlight/brightness
```


Not a tutorial by any means, but those who want to acheive some simple but effective power management can now implement it without running a cron job every minute and reading udev files *BLEH*
