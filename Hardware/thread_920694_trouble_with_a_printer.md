My experience with printers points to Canon being your problem
I had a Canon multi-function F60 printer scanner fax copier.  I was running windows and it functioned just beautifully.  Then I got this hair brained idea that I needed to get a Linux distro up and running and learn something new.  The first thing I learned was that Canon does not support any native Linux drivers.  The second thing was that there were no drivers available that I was able to find.  Canon seems to be one of those "windows" companies who's products work on Mac's as long as you don't upgrade the Mac os to often.

I bought a new HP C6180 Photosmrt multi-function Printer.  I knew that I would have to download nad install a driver to get this printer working in Linux, but one was available.  My Linux distro of choice was Suse V10.  Now two years later I still have not been able to get the printer to install in Suse.

Approximately 4 hours ago I installed Ubuntu, major problems were intalling on a bare naked harddrive (solved on third try)  screen resolution (solved on the 4th try) and printing.  Solved on the second try, seems you have to select a printer in order to use it.
