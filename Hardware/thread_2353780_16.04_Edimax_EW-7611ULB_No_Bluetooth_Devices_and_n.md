So I added a line to btusb.c:  
```
{ USB_DEVICE(0x7392, 0xa611), .driver_info = BTUSB_REALTEK }
```

Inside  

```
static const struct usb_device_id blacklist_table[] = { 
```

Just before the

```
     { }    /* Terminating entry */
```

Then I compiled and installed per the readme in rtl8723au_bt_kernel.

I also installed the windows bluetooth driver in a virtual machine and grabbed "rtl8723b_mp_chip_bt40_fw_asic_rom_patch_new.dll" from the program files install folder. I compared it with the .bin firmware files in rtl8723au_bt_kernel. They look pretty similar so I just renamed rtl8723b_mp_chip_bt40_fw_asic_rom_patch_new.dl to rtl8723b_fw.bin and replaced /lib/firmware/rtl_bt/rtl8723b_fw.bin

 Now I have bluetooth and wifi!

### Post by Scott Deagan on 2018-01-10
> However, I couldn't manage to install the bluetooth drivers. Has anybody had any success?

It's very confusing getting both the Bluetooth and WiFi working with this dongle, but it's doable (I've managed to do it on Kubuntu 17.10 running kernel 4.13).

**There are two kernel modules you need to download, build, and install**. You managed to to install the WiFi driver, which is great. You then need to download and install a Bluetooth driver for this particular chipset. This one will work:

[ https://github.com/lwfinger/rtl8723au_bt]("https://github.com/lwfinger/rtl8723au_bt")

You should probably use the "kernel" branch if you have a more *"recent"* kernel (as mentioned above, I'm using kernel 4.13).

I will repeat your instructions, plus the instructions required to get Bluetooth working (for the benefit of those who don't have a Ph.D in Computer Science):

[SIZE=4]**INSTALL THE WIFI DRIVER**[/SIZE]

```
git clone https://github.com/lwfinger/rtl8723bu.git
cd rtl8723bu
make
sudo make install
sudo reboot
```

[SIZE=4]**INSTALL THE BLUETOOTH DRIVER**[/SIZE]

```

git clone https://github.com/lwfinger/rtl8723au_bt.git
cd rtl8723au_bt/
git checkout kernel
find ./ -type f -name btusb.c # <-- I did this just to make sure I had the right branch - btusb.c doesn't exist in the master branch.
make

```

At this point, you may (or may not) get an error when you try to make the project. If you're running a newer version of the gcc compiler, you'll get an error that looks something like this:

```

[syd@scott-t470p:rtl8723au_bt]$ make
make -C /lib/modules/4.13.0-25-generic/build M=/tmp/test/rtl8723au_bt modules
make[1]: Entering directory '/usr/src/linux-headers-4.13.0-25-generic'
  CC [M]  /tmp/test/rtl8723au_bt/btusb.o
/tmp/test/rtl8723au_bt/btusb.c: In function &#8216;inject_cmd_complete&#8217;:
/tmp/test/rtl8723au_bt/btusb.c:1747:2: warning: dereferencing &#8216;void *&#8217; pointer
  *skb_put(skb, 1) = 0x00;
  ^~~~~~~~~~~~~~~~
/tmp/test/rtl8723au_bt/btusb.c:1747:19: error: invalid use of void expression
  *skb_put(skb, 1) = 0x00;
                   ^
scripts/Makefile.build:315: recipe for target '/tmp/test/rtl8723au_bt/btusb.o' failed
make[2]: *** [/tmp/test/rtl8723au_bt/btusb.o] Error 1
Makefile:1550: recipe for target '_module_/tmp/test/rtl8723au_bt' failed
make[1]: *** [_module_/tmp/test/rtl8723au_bt] Error 2
make[1]: Leaving directory '/usr/src/linux-headers-4.13.0-25-generic'
Makefile:25: recipe for target 'all' failed
make: *** [all] Error 2

```

Don't fear the output - embrace it. It's telling you there's something wrong on **line 1747** in file** [FONT=courier new]btusb.c[/FONT]** (which exists in the root directory of the project). Let's take a look. Open btusb.c using your favorite text editor (I'm not going cover the use of text editors in this post, suffice to say that vim or nano are pretty cool). The erroneous line looks like this:

```
   *skb_put(skb, 1) = 0x00;
```

C/C++ have become a lot more "strict" in recent years when it comes to type safety. We need to explicitly tell the compiler what "type" we're referencing here. After looking at the code, it can be seen it's an "unsigned short". So let's typecast, and change line 1747 to:

```
    *(ushort*)skb_put(skb, 1) = 0x00;
```

Now run [FONT=courier new]make[/FONT] again, and it should work:

```

From here on in, follow the rest of the instructions at listed in the developer's page:

[https://github.com/lwfinger/rtl8723au_bt/tree/kernel](https://github.com/lwfinger/rtl8723au_bt/tree/kernel)

I found that I didn't have to install a newer vesion of Bluez or anything else like that. For me, to get basic Bluetooth up and running, it was just:

```

make
sudo make install
sudo make modprobe -rv btusb
sudo make modprobe -v btusb

```

While some may look at the complexity involved in getting the Edimax 2-in-1 WiFi/Bluetooth adapter working on Linux and roll their eyes, we should remind ourselves that because the source code is open and available to the public, it means we are not subjugated by a third party (i.e. the hardware supplier). Yes, it would be totally awesome if Edimax continued their support for the more recent version of Linux, but the sales of such devices has probably bottomed out because laptops come with Bluetooth and WiFi as standard these days (hence it's not economically viable for Edimax to assign a resource to maintain the driver for this particular device). I do, however, find their advertising misleading (as they advertise it as being "compatible" with Linux - which isn't a complete lie, but Linux supported has ended).

**LiNUX**: Learn it, Live it, Love it.

That is all.

**EDIT:** Forgot to mention that the above will not automatically power-up your Edimax 2-in-1 Bluetooth/WiFi adapter (if you're running a Bluez version 5). If your Bluetooth doesn't power-up automatically when you reboot, add the following to [FONT=courier new]**/etc/udev/rules.d/10-local-bluetooth.rules**[/FONT]:

```
ACTION=="add", KERNEL=="hci0", RUN+="/usr/bin/hciconfig %k up"
```
