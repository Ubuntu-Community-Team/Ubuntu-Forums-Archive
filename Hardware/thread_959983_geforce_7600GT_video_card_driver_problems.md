### Post by starcannon on 2008-10-29
Sounds like your going to need to get a custom mode line to stuff in your xorg.conf.

I use this modeline generator:
[http://xtiming.sourceforge.net/cgi-bin/xtiming.pl](http://xtiming.sourceforge.net/cgi-bin/xtiming.pl)

You can modify your xorg.conf file with these commands:
```

sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf_backup
gksu gedit /etc/X11/xorg.conf

```**Your monitor's modeline will be different,** but heres an example nvidia xorg.conf to show you where the bits go:
```

# nvidia-xconfig: X configuration file generated by nvidia-xconfig
# nvidia-xconfig:  version 1.0  (buildmeister@builder3)  Thu Feb 14 18:20:37 PST 2008

Section "ServerLayout"
    Identifier     "Layout0"
    Screen      0  "Screen0"
    InputDevice    "Keyboard0" "CoreKeyboard"
    InputDevice    "Mouse0" "CorePointer"
EndSection

Section "Files"
    RgbPath         "/usr/X11R6/lib/X11/rgb"
EndSection

Section "Module"
    Load           "dbe"
    Load           "extmod"
    Load           "type1"
    Load           "freetype"
    Load           "glx"
EndSection

Section "InputDevice"
    # generated from default
    Identifier     "Mouse0"
    Driver         "mouse"
    Option         "Protocol" "auto"
    Option         "Device" "/dev/psaux"
    Option         "Emulate3Buttons" "no"
    Option         "ZAxisMapping" "4 5"
EndSection

Section "InputDevice"
    # generated from default
    Identifier     "Keyboard0"
    Driver         "kbd"
EndSection

Section "Monitor"
    Identifier     "Monitor0"
    VendorName     "Unknown"
    ModelName      "Unknown"
    HorizSync       30.0 - 110.0
    VertRefresh     50.0 - 150.0
    Option         "DPMS"
[COLOR=Red]     Modeline "1024x768@60" 64.56 1024 1056 1296 1328 768 783 791 807[/COLOR]
EndSection

Section "Device"
    Identifier     "Device0"
    Driver         "nvidia"
    VendorName     "NVIDIA Corporation"
EndSection

Section "Screen"
    Identifier     "Screen0"
    Device         "Device0"
    Monitor        "Monitor0"
    DefaultDepth    24
[COLOR=Red]    SubSection     "Display"
        Depth       24
        Modes       "1024x768@60"
    Virtual       1024 768
    EndSubSection[/COLOR]
EndSection

```GL and have fun

### Post by starcannon on 2008-10-30
Okay after reading the owners manual for your monitor, here are the available resolutions:

VGA input D-Sub 15-pin terminal
Suggested scan rates:[INDENT][INDENT][INDENT]
[LIST]
[*]E-EDID compliant
[*]640×480 /60 Hz
[*]800×600 /60 Hz
[*]1024×768 /60 Hz
[*]1360x768 /60 Hz
[/LIST]
[/INDENT][/INDENT][/INDENT]It's E-EDID compliant, so it *should* offer all the modes listed, with of course the 1360x768 being the most desirable.

Okay heres some mode lines that will *likely* wok with your gear:
This one is interlaced, I'd try it first for best quality, note that the height is slightly smaller, this is intended.
Modeline "1360x765@60i"     37.86  1360 1392 1528 1560    765  782  787  805 interlace

This one is non-interlaced, I'd try it if there are issues with the interlaced:
Modeline "1360x768@60"      84.50  1360 1392 1712 1744    768  783  791  807

I put both modelines in your xorg.conf, and instead of setting it up to cycle through them, I commented out the regular one.
If the interlaced Modeline does not work, them simply comment it out by putting a # in front of the line, or remove its lines
completely. To enable the regular Modeline, just remove the commenting tag # from in front of it. Be sure to do this in both 
the Monitor Section and the Screen SubSection. I have marked the comment tags in blue.


Heres where you put them in Your xorg.conf
```

# xorg.conf (X.Org X Window System server configuration file)
