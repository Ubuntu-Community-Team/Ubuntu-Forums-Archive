### Post by efflandt on 2017-06-27
I don't understand why you are even trying to use integrated Intel graphics if you have a graphics card installed. The GTX 750 Ti is an efficient card that probably uses little power when idle and is hardware regulated to 60 watts max.

I don't have a desktop PC with integrated graphics and graphics card, but for a laptop with hybrid Intel/Nvidia graphics, you chose which you want to use from the bottom of **NVIDIA X Server Settings**, so X works properly for the selected graphics. But after switching it, you might either need to log out of X and back in, or maybe reboot if switching Intel to Nvidia. I don't know if there is a way to use integrated Intel and Nvidia graphics simultaneously, although, before nvidia-prime, bumblebee was used to funnel nvidia graphics through Intel for display which when I had no other choice in Ubuntu 13.10 was much slower using Intel graphics alone or Nvidia displayed through Intel.

If you did not use **nomodeset** boot parameter while booting the live/install system and installing when the PC had Nvidia graphics, you may need to add that to this line in /etc/default/grub like this, then do: sudo update-grub```
GRUB_CMDLINE_LINUX_DEFAULT="nomodeset quiet splash"
```I believe nomodeset is always required for a desktop PC with Nvidia graphics, at least if using an nvidia driver package, but I think the system does that for you elsewhere if you used nomodeset during install (and is always included if doing recovery boot).
