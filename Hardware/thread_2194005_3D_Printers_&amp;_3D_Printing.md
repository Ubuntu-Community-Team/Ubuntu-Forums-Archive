### Post by freakalad on 2013-12-15
Been a good while since I've been back here....

Anyhow, it's pretty surprising to see that there has been so little posted on this subject, other than a little troubleshooting on installing RepetierHost [posted here]("http://ubuntuforums.org/showthread.php?t=2154790").
I really expected there to be a dedicated section already - I'll put down some of my own noted down here for references for others to look at.

I recently build me a nice little [DiamondMind]("http://www.mindkits.co.nz/store/3d-printing/diamondmind-3d-printer") (based on the [RepRap]("http://reprap.org/")) from a friend of mine. I really don't care all that much for Makerbot - personally (patents, etc).
So far, so good. Got the machine built, powered-up & got some basic controls going.

Other than the firmware on the device (discussion for a different section - ref the RepRap resources), there are a number of stacks available for the hosts.

* [Repetier]("http://www.repetier.com/") - effectivly the definitive stack for such machines
* Slic3r - slices objects to GCode (3D printing bytecode)
* Ultimaker [Cura]("http://software.ultimaker.com/") - shiny
* Pronterface [Printrun]("http://reprap.org/wiki/Printrun") nice for the simple controls
* [RepSnapper]("http://reprap.org/wiki/RepSnapper_Manual:Introduction") - pretty nice

There are more - I'll add to this post as I tract them down.

Few of these seem to be packages or very laymen-friendly, especially for Ubuntu.
Fedora seems to be fairing a little better & seem to be moving things along [from their end]("http://fedoraproject.org/wiki/Features/3D_Printing")
