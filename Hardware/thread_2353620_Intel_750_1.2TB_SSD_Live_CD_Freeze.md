### Post by Glen_IL on 2017-02-23
Hello everyone, 
I have a Dell Precision 5810 with Bios A16. I am trying to install Ubuntu 16.04, or 16.10 to the [Intel 750 1.2TB SSD]("https://www.newegg.com/Product/Product.aspx?Item=N82E16820167363"), but when I boot off the flash drive via legacy mode, or UEFI mode, the dots blink but I never get to a desktop. 
The SSD is a Dell 750 1.2TB PCIE device. 

So far I have:
[LIST]
[*]Configured the local sata controller to ACHI mode.
[*]Configured the PCIE express bus to only operate at Generation 3 mode.
[*]Configured PCIE Express bus to not auto detect the video video, and assigned it into slot 2.
[*]Enabled the PCIE bus to be able to use more then 4GB of memory if needed.
[*]Enabled Legacy boot and disabled secure boot, but left UEFI boot as an option.
[/LIST]

So, I followed the advice in this forum:
[http://askubuntu.com/questions/696413/ubuntu-installer-cant-find-any-disk-on-dell-xps-13-9350](http://askubuntu.com/questions/696413/ubuntu-installer-cant-find-any-disk-on-dell-xps-13-9350)
and could not get Gparted live to start, it kernel panics when it detects the NVME drive. 

I was able to get a Windows 10 recovery environment to do a diskpart clean on the drive, but that seems to have produced no results as far as installing Ubuntu. 

When I attempt to install Ubuntu server, it will hang on hardware detection and leave me with a purple console. 

If I remove the SSD out of the slot, the Live CD brings me to the desktop immediately. 
If I disconnect the 2TB spinning drive also in the system, I still get the freezes on the live CD and Server install process. 

I am not really sure how to progress from here and could use some guidance.

Thank you!
-Glen
