Hello,

I recently switched from Windows 8.1 back to Kubuntu (14.04). After  setting almost everything up the way I wanted it to be, I decided to  enable hibernate. In order to do so I:

[LIST=1]
[*]Appended resume=/dev/sda6 to GRUB_CMDLINE_LINUX_DEFAULT's value in /boot/grub/menu.lst. 
[*]Created a file /etc/polkit-1/localauthority/50-local.d/com.ubuntu.enable-hibernate.pkla with 
[/LIST]
```
[INDENT=2]Identity=unix-user:*
Action=org.freedesktop.upower.hibernate
ResultActive=yes

[Re-enable hibernate by default in logind]
Identity=unix-user:*
Action=org.freedesktop.login1.hibernate
ResultActive=yes
[/INDENT]

```[INDENT]as its content.
[/INDENT]

After a system reboot, Suspend to Disk was available to me so I tried  hibernating the system. Everything seemed to work fine, the system  switched off and when I switched it back on, it reloaded my last session  from the swap partition and everything seemed to work. I edited a  document in my home folder and it saved fine but when I tried to apt-get  install krita, I got a message that the filesystem had been mounted  read-only and when manually switching the filesystem to read-write  didn't work, I ran fsck which reported a bunch of errors so I ran fsck  -a, rebooted the system and got a kernel panic. Since it was a new  install with almost no irreplaceable content, I re-installed Kubuntu.

With the new system, I tried hibernating again with the same results but  instead of running fsck, I rebooted the system, let it fix the disk  automatically and was presented with a system where wifi along with some  other things were no longer working. I managed to glean that the reason  this was happening was (probably) the nvidia graphics card.

Any help or pointers regarding hibernating the system without borking it  would be greatly appreciated. I have re-installed Kubuntu a 3rd time  and I am apprehensive to try hibernate again despite needing it. I have a  HP Envy 15 with i7-4700qm, 8GB RAM, Hybrid Intel and nVidia 740M  Graphics, 1TB HD and a 32GB mSata SSD (not being used with linux atm).
