Hello,
I installed 9.04 on my ASUS eeePC 901. 
My partition table is something like that:
1) /dev/sda1 -> /var
2) /dev/sdb1 -> /
3) /dev/sdc 1 -> /home
/dev/sdc is a SD Card with 16GB. When booting up fsck fails in finding /dev/sdc1.
When booting up with CTRL-D xserver comes up but there is no HOME dir available.
But when changing to a CLI terminal I am able to "sudo mount -a" and then /home is mounted correctly.
I tried out some different things like using UUID or /dev/sdc1 in my /etc/fstab.
I tried ext4 and ext3 filesystem.
But always the same problem. Seems like SD Card comes up to late...
Maybe this is important: The installer found my sd card on /dev/sdd1 but after installation the card is /dev/sdc1!
On 8.04 and 8.10 the same configuration worked fine! But I did not do an upgrade from 8.10, I did a new install.
Any idea?
