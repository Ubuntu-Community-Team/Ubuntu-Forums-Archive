### Post by memilanuk on 2008-11-22
I have Xubuntu 8.10 installed on an older Sony Vaio laptop (PCG-FRV31, if anyone cares).  Seems to work well, and I've even got my Belkin USB wifi dongle to work!  Got the machine sitting there downloading and installing updates at the moment.

This machine seems run really hot... cooling fan is on pretty much all the time, and there are times when the machine will just flat turn off in the middle of doing something.  In WinXP, it does it while (or shortly after) doing anything disk-intensive for very long - like a deep AV scan.  On one of the recent boots into Xubuntu, the machine did (I think, it's been years since I was 'up' on this sort of thing) an fsck check - at any rate it took an exceptionally long time to boot, and there was a lot of disk activity - and about five minutes after I logged in (finally), the machine just shutdown.  No warning, no 'by your leave', nothing.  It appeared that Ubuntu was a little more graceful about trying to shut things down in an orderly fashion... at least as much as I could tell as things flickered past on the screen before it went black and the machine went silent.

Right now I've got the machine perched on the edge of a drawer, so it is supported but leaves plenty of clearance around the bottom and plenty of air flow to the fan.  Any other ideas... besides start saving for a newer laptop?

### Post by Frem on 2008-11-22
Start running the machine with all powersaving measures on. Spin down the hard disks as soon as you can and downclock the CPU. Run lightweight applications and avoid multimedia.

You might also check in your BIOS to see if there are any system power or cooling settings in there.
