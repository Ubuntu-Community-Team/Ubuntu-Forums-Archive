### Post by logos34 on 2009-06-09
I can see one problem:

> **padrecovsky said:**
> 
proc /proc proc defaults 0 0
# Entry for /dev/sda6 :
UUID=afe28a76-0336-4a79-beea-fe1d78572d01 / ext4 errors=remount-ro 0 1
/dev/fd0 /media/floppy0 auto rw,user,noauto,exec,utf8 0 0
[COLOR="Red"]/dev/sdc1 /media/disco ntfs-3g defaults,locale=en_US.UTF-8 0 0
/dev/sdc1 /media/sdc1 ntfs-3g defaults,locale=en_US.UTF-8 0 0[/COLOR]
/dev/sdb5 /media/sdb5 ntfs-3g defaults,locale=en_US.UTF-8 0 0


You've got two entries for sdc1, trying to mount in different directories--unless ntfs-config screwed up the edit.  Remove the 'disco':

gksudo gedit /etc/fstab

take out this line:

[COLOR="Red"]/dev/sdc1 /media/disco ntfs-3g defaults,locale=en_US.UTF-8 0 0[/COLOR]

then save and close, and do:

sudo mount -a 

or reboot.

But I suspect none of that will help, because there's something wrong with ntfs partition (but not too wrong because you say you can boot into windows on sdc1 and run error checking).  

You could try testdisk to fix it.

sudo apt-get install testdisk

sudo testdisk

Read [this]("http://www.users.bigpond.net.au/hermanzone/p21.html") and [this]("http://www.cgsecurity.org/wiki/TestDisk") (->'Using testdisk'.
