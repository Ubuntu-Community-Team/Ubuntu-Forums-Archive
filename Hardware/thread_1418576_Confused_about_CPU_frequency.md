### Post by Revolutionary101 on 2010-02-28
The benchmark number is the ultimate measurement. With the speed of a processor there are many factors. Some of the factors are FSB/QuickPath Interconnect speed, number of threads, size of cache, clock speed, and number of cores.

To judge a whole processor just based on one of these factors may result in a bad choice.

In the end the CPU benchmark number is the ultimate judge of processor speed.

### Post by cascade9 on 2010-03-01
+1 to whjat Revolutionary101 said, with 2 minor additions. 

1- Benchmarking results can vary, a lot, from site to site. Just because 1 site has CPU 'A' always beating  CPU 'B' doesnt mean that every test on the internet will have similar results. 

2- There is no one 'benchmarking' system. The tests vary, a lot. For example, see the toms hardware lists- 

[http://www.tomshardware.com/charts/2009-desktop-cpu-charts-update-1/benchmarks,60.html](http://www.tomshardware.com/charts/2009-desktop-cpu-charts-update-1/benchmarks,60.html)

The tests include 'synthetic' tests like 3D Mark, PC Mark, SiSoft Sandra. There are also games in there- dont assume that because CPU 'A' benchmarks better than CPU 'B' in game 'W' that it will benchmark better in games 'X', 'Y' or 'Z'.  There are also specific applications like  DivX (video encoding), WinRAR (compression speed), 3D studio Max (image rendering) and a fair few other things. 

If you use your computer for 1 task in particular (eg 3D studio max) then I'd pay far more attention to benchmarking for that program that to anything else.
