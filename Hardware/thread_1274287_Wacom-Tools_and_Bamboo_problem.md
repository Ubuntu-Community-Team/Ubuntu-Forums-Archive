Hi guedesav,

Xorg.conf runs after HAL, and what runs last controls.  As far as I know HAL/.fdi doesn't interfere with xorg.conf in Intrepid.  It doesn't for me.

Let's try upgrading you to linuxwacom 0.8.1-6.  Download the two deb packages for your cpu architecture here:  [https://help.ubuntu.com/community/Wacom](https://help.ubuntu.com/community/Wacom)  They're under Intrepid near the top.  Then once they are on your desktop just double click on them and let the Debian package manager install them.  Reboot.  Since they are deb's they will register with Synaptic.

Good luck!

### Post by ragtag on 2009-09-28
This is a known problem when using Wacom with xorg.conf, rather than hal. There is a tool called wdaemon (read more about it [here]("http://llbb.wordpress.com/2007/09/02/use-wdaemon-for-hotplugging/")) that can get around this. It's not the easiest thing to set up, but I've got it working in the past both on Ubuntu, Fedora and CentOS. :)

Currently I'm running Jaunty, and happy with using HAL for my wacom.

### Post by Favux on 2009-09-28
Hi guedesav,

Like ragtag says true hot plugging doesn't work if you are using xorg.conf.  You have to use HAL/.fdi to get real hot plugging.  And since HAL/.fdi doesn't work for Wacom in Intrepid you're stuck.

But you may be able to use the old (pseudo) hot plugging commands:  ctrl+alt+F1 followed by ctrl+alt+F7.  That's what I use.
