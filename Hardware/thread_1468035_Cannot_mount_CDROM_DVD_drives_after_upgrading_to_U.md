### Post by kaiwi on 2010-05-01

### Post by Peridot1974 on 2010-05-17
I don't know how it works, i'm a newbie.  But after searching the web, i came across someone suggesting to disable the cd at bios.  What I did was to change the boot order of my devices in bios to boot on HDD first then CD.  After that upon login to ubuntu, bang, everything works perfect.  The CD/DVD drive was back to normal... hope it helps.  CHeers.

### Post by cudBwrong on 2010-09-19
I may have a partial answer.  I was getting similar problems with access to cd drives after an upgrade to Lucid.  

Under Hardy, I would copy cds using cdrdao, with a command such as this:

$ sudo cdrdao copy --force --source-device /dev/<xxx> --device /dev/<xxx>

Where "xxx" stands for the letters and numbers that worked under Hardy.

With any attempt to access a cd drive, I was getting the "special device /dev/hdd does not exist" message.

Using System->Administration->DiskUtility, I could see that the system did recognize my drives under different device names.  Using the new letters and numbers visible in DiscUtility makes the command work:

$ sudo cdrdao copy --force --source-device /dev/sr0 --device /dev/sr1

If you don't have DiskUtility, it's listed as gnome-disk-utility in Synaptic Package Manager.  It's also known as "palimpsest."

### Post by kaiwi on 2011-06-23
First thing to do might be to edit your file 
/etc/fstab
Comment out all entries relating to your CD/DVD drives (by placing a # sign at the beginning of the respective line).

As an example, my fstab is:
```
#/dev/sr0 /media/cdrom0 udf,iso9660 remount,group,users 0 0
#/dev/sr1 /media/cdrom1 udf,iso9660 remount,group,users 0 0
UUID=5ACC08E9CC08C16D /media/C: ntfs-3g defaults 0 0

```

Since the day I commented out this lines, my drives are usually auto-detected but not always. 

My impression is that certain actions, such as using Brasero or plugging a USB stick into the USB port, tend to increase the probability that the next CD/DVD that is inserted in the drive will NOT be detected. In this case, I need to log out and log in again (or restart the system, don't remember).
