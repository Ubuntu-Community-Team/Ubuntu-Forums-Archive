### Post by weatherman2 on 2014-09-02
You can overwrite the boot sector of any hard drive, but you still need to set the boot order correctly in your BIOS to choose the correct drive to boot from, whether there is a valid boot sector on the other drives or not.  Some systems will hang trying to boot a non-bootable drive if it comes up in the boot order, whether there is a valid boot sector or not.

Just set the BIOS boot order to boot from the SSD.  Don't worry about the boot sectors on the other drives.

### Post by weatherman2 on 2014-09-02
If this is a EFI system, you can erase the EFI partition on the non-boot drives (which is a small MSDOS format partition, perhaps 100MB in size).  It isn't actually the "/boot" partition - in fact, when you boot Ubuntu, you're mounting the EFI partition at /boot/efi in Ubuntu (look at your current booted partitions).  /boot may be just a directory in the old OS filesystem on your old hard drive, not seen at all by the EFI firmware.  So simply erasing the old /boot won't erase the EFI partition.
