### Post by deadflowr on 2014-03-20
DO NOT go by Windows definition of what a drive is.
Windows calls everything, whether a drive or a partition, a drive.
This causes problems because Ubuntu Calls a drive a drive and a partition a partition.

So when installing, Ubuntu might ask to install over entire drive.
Then you do but lose not only C but also D, and maybe E which weren't drives but actually partitions.

Safest thing is to use the option "something else" during the install.
At least you can see what the partitions look like.
(And exit or cancel the install if you don't know what it means)

My two,or three cents anyway.
