I am thankful,I'm glad you solved it!  Congrats!  I'm not sure what happened but I can guess.   My bet would be something to do with kernel modules.  You probably noticed that nvidia and virtual box both need dkms - and if anything gets screwed up, if there's a conflict or corruption, then something goes wrong.   So, maybe when you tried to switch between nvidia drivers, something goes wrong, doesn't match with the kernel modules/files, whatever and it impacts virtualbox install/configuration.... That's why it's often recommended to purge or auto remove or whatever the instruction is - to totally get rid of (the old) nvidia stuff.   They can't co-exist or there's a conflict.  Check this page and section here:

[https://www.makeuseof.com/what-to-do-after-installing-ubuntu-in-virtualbox/](https://www.makeuseof.com/what-to-do-after-installing-ubuntu-in-virtualbox/)

The main line here that is relevant to us:   "Also, run the following command to **enable building kernel modules**."
The command:
sudo apt-get -y install build-essential gcc make perl dkms
