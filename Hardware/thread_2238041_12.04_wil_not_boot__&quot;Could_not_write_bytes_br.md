I had the same problem on my mothers computer, after I cleaned "Package Configs" in the Ubuntu Tweak Janitor.

After following the link of EDIT2 I found out that /etc/X11/X didn't start, because it was missing. (/var/log/lightdm/lightdm.log pointed to /var/log/lightdm/x-0.log where I found this).
On the page I didn't find the answer, but Google and AskUbuntu helped me out here:
[http://askubuntu.com/questions/232926/etc-x11-x-not-executable-error-when-startx]("http://askubuntu.com/questions/232926/etc-x11-x-not-executable-error-when-startx")

```
sudo dpkg-reconfigure -phigh xserver-xorg
```

Maybe this can help someone else with the same problem.

One advice: Don't use Ubuntu Tweak Janitor without checking which Package Configs are still in use.
