### Post by dccrens on 2009-01-23
I have a eeepc 1000h with 160Gb HD 2gb Ram. It is been a great machine for the last month. I have it loaded dual boot with windows xp and ubuntu 8.10. Tonight I closed the lid to put it in standby. I came back about 20 minutes later and opened the lid and got no resume. Tried all the keys and nothing. I then pushed and held the power key to shut it down. I then pushed the power key to reboot and got nothing. The blue tooth light and the power light turn on and the hard drive light flickers for a moment then goes out and nothing. No bios screen or any other activity. 

I have also tried taking out the battery & pushing holding the power then reseating the batter and hitting power with the same result. 

I also tried using the "reset" button (small reset hole on the back) to do a hard reset. Nothing...

Anyone have any suggestions? I purchased the machine through buy.com. Does ASUS handle the warranty? Has anyone else had this happen? Seems like it might be bad hardware (motherboard)...

Any help would be much appreciated.

### Post by dccrens on 2009-01-23
Well... Problem resolved... Hopefully... 

I removed the back panel and pulled the 2GB memory card. I replaced it with a 1GB card. Turned it on and viola! We have ignition... So... I turned it off, pulled the 1GB card and replaced it with the original 2GB card and it boots up fine... Bad Memory? Fluke? Gremlins? 

So for anyone else out there that runs into the same problem - try removing and reseating the memory card. Interesting that it would not give any visual error (with memory)... No BIOS error displayed, etc...
