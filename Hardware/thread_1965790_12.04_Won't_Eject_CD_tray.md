### Post by skelooth on 2012-04-26
I ran into an issue where 12.04 didn't want to eject my cd tray.

Turns out ubuntu wanted to target /dev/cdrom but my drive is actually /dev/cdrom1

Here's my workaround, but a more permanent config change would be more sufficient.

```

gksudo eject /dev/cdrom1

```
