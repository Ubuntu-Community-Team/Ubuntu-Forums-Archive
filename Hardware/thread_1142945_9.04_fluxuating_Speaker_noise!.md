### Post by XeroxGUI on 2009-04-29
Anthony here:

Open up system->preferences->sound and set everything to use pulseaudio, as that is the new Ubuntu sound manager. I will further help you configure pulseaudio tomorrow based on my setup. 

The fuzziness is because your ALSA PCM volume is turned all the way down, so run `alsamixer` in a terminal and turn the PCM volume up. After this is all done, make sure every applications (totem, firefox etc.) is set up to use pulseaudio or alsa as the sound manager.

Also, here's a shell script to restart the sound daemons if something is misbehaving or, if like me, sound breaks after suspending:

```
#!/bin/bash
if [ -z "$1" ]
then
pulseaudio -k # the "right" way
kill -9 `pidof pulseaudio` # the "ok now you're just beligerant" way
killall pulseaudio # the "omg just die already" way
sudo alsa unload # unload alsa
sudo alsa reload || echo "ALSA failed!" # because you can't "just load"
pulseaudio -D || echo "Pulseaudio failed!"
echo "sound (hopefully) started. . ."

elif [ "$1" = "test" ]
then
pulseaudio -k # kill everything else first
kill -9 `pidof pulseaudio`
sudo alsa unload
sudo cat /dev/random >> /dev/dsp # a simple test
echo "If you see an error, something is using the card!"
fi

```

Now save the file as "sound" under your home directory, then open a terminal and run the command `chmod 755 ~/sound`. Now, run the file as ~/sound to restart all sound daemons (services), and to test to see if something is using the card, then run `~/sound test` and look for an error.

If the test works without an error, run `~/sound` again with no arguments to restart sound, otherwise find and kill whatever is using the sound card directly via OSS, e.g. firefox (flash) or VMware.

Let me know how that works out for you. Pulseaudio makes sound a hassle until you get it running, but once you do it's great.
