### Post by Racer-X on 2009-01-29
There's been a few postings in the Network & Wireless section.  It appears that there may have been some issue with dkms for some networking modules in the first kernel update on 1/28/2009.

I know that this morning's (1/29/2009) updates have appeared to fix my network issues (with an Atheros AR242x).  There were, after this morning's updates, an 'updates' subdirectory in the /lib/modules/2.6.27-11-generic directory that contained network modules, etc.

If you've not tried to get this morning's updates, try, and make sure that the modules are getting loaded on boot.

### Post by Zanti on 2009-01-30
Not a fix but a workaround:

I also have the MSI Wind u100 (on it right now) and my ethernet connection stopped working as of 2.6.27-11.  On bootup I selected to boot the 2.6.27-9 kernel and networking worked.  I checked the /etc/resolv.conf then made a backup (I need to alter it for my USB Compass 885 to work).  Then I rebooted with the 2.6.27-11 kernel and it worked. was then able to get the updates.  But it just seems for some reason resolv.conf gets hosed after upgrading to .27-11.
