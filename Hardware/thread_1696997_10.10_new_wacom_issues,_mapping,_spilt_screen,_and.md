### Post by Favux on 2011-02-28
Hi Nikolai,

whot is Peter Hutterer, one of the lead developers on the linuxwacom project.

> The main thing i need to know is how to change the mapping for the pen to 'mouse' mode so that I can actually draw the way I like.
Change Mode from Absolute to Relative.  See:  [http://sourceforge.net/apps/mediawiki/linuxwacom/index.php?title=Xsetwacom](http://sourceforge.net/apps/mediawiki/linuxwacom/index.php?title=Xsetwacom)  &  [http://sourceforge.net/apps/mediawiki/linuxwacom/index.php?title=Tablet_Configuration](http://sourceforge.net/apps/mediawiki/linuxwacom/index.php?title=Tablet_Configuration)
> Back to the faulty mouse scrolling - [http://www.spinics.net/lists/linux-input/msg10577.html](http://www.spinics.net/lists/linux-input/msg10577.html) - How/Where do I put the code on this page into the proper place? 
Update your xf86-input-wacom.  See part II. in the [Bamboo P&T HOW TO]("http://ubuntuforums.org/showthread.php?t=1515562").
> My problem is that when using the pen: if I bring the pointer to the extreme right hand side of the screen(screen 0) it will jump to screen 1 and while the mapping *seems* good for either screen there is no way to get the pointer back to screen 0 without that I use the mouse to do it.
See [HOW TO Setup a Wacom Tablet with Multi-Monitors in Maverick and Natty]("http://ubuntuforums.org/showthread.php?t=1656089").
