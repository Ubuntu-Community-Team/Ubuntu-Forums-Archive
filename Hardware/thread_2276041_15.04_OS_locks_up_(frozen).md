### Post by Gabe_B on 2015-04-30
Hi folks,

I've been having some issues as of late with my 15.04 (general release) installation.  The system runs great.  Originally was using the same install, but with an older Nvidia Geforce 9500 GT.  That card ran like a champ, but alas, was showing signs of being very loud.  I decided to purchase an AMD R7 260x GPU.  Specs below:

CPU: i4790k
RAM: 32GB
MB: Asus Z87 Gryphon
GPU: Asus R7 260x
iGPU: disabled through bios

After I installed the new GPU, I did a fresh install, pretty much vanilla.  Only big item I installed was KVM-Qemu, so I can run a Win2008 as a virtual guest.  I've disabled screen lock, but had the disable monitor feature enabled.  Last night, I left the machine on and when I came back this morning, the computer screen was black as expected.  However, it would not wake up no matter what I did (mouse/keyboard input, changing inputs on the monitor, etc.).  I didn't have SSH enabled, so I couldn't tell if it was locked up or if the GPU maybe wasn't responding.

After enabling SSH... I was just working on the machine today with my virtual guest running as well.  All of the sudden, I could no longer type nor move the mouse.  The screen is frozen with everything showing up at the moment of the freeze.  I tried using the mouse break out command from KVM, thinking maybe the guest froze.  But it appears the host is not responding as well.  Luckily I had SSH enabled this time.  SSH'd into the machine and no sooner had I listed out the contents of /var/log to start digging into the issue, the SSH session dropped and I can no longer ping my machine.

I'm aware there are issues with the AMD R7 260x cards... but strange the entire host froze?  Any ideas where to start digging?

Thanks
