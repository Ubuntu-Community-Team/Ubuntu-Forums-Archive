### Post by fuzzbill on 2009-12-29
I tried building various versions of the SiS drivers too and hit the same problem. Eventually I found a couple of hours to spare and reworked the imedia SiS driver for OpenSUSE 11.1 with XOrg 7.4 and called it 1.0.0. It should work for other distros too. I've left a tarball on rapidshare ([http://rapidshare.com/files/327436561/xf86-video-sis-1.0.0.tgz.html](http://rapidshare.com/files/327436561/xf86-video-sis-1.0.0.tgz.html)).

This works fine for me on a D201GLY with OpenSUSE 11.1. I don't get the SiS driver snow or any other screen corruption, OpenGL and X11 video work, but don't expect NVIDIA/AMD performance! I get ~120fps in glxgears.

To build:

tar -zxf xf86-video-sis-1.0.0.tgz
cd xf86-video-sis-1.0.0
automake
autoconf
./configure --prefix=/usr
make
make install

It worked for me, but you're doing this at your own risk, so if it was me I'd do

make -n install

and check that it's going to do what you want before you do the install.

You'll also need an xorg.conf entry like:

Section "Device"
  Identifier   "Device[0]"
  BoardName    "SiS 660"
  BusID        "1:0:0"
  Driver       "sis"
  VendorName   "SiS"
  Option       "NoAccel" "off"
EndSection

Hope it works for you.

### Post by PaulCompton on 2010-01-13
If you are using 9.10 (Karmic) you don't need to compile it!
Just download the original from intel [here]("http://downloadcenter.intel.com/Detail_Desc.aspx?agr=N&Inst=Yes&ProductID=2873&DwnldID=15443&strOSs=39&OSFullName=Linux*&lang=eng"), extract it and copy the .so file to /usr/lib/xorg/modules . I did this with the 64-bit version (these boards run AMD64 Ubuntu wonderfully) ten minutes ago, restarted xorg (actually, I rebooted) and it was fine.
Actually, I had previously installed the official sis driver - but maybe it was already there anyway.
Let me know if this works for you!
