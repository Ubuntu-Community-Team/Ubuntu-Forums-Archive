### Post by ProDigit on 2008-08-16
Hi,

I happened to notice that both on Ubuntu 8.04; and Xubuntu, there seems to be a bug when installing eg: a US keyboard, and on a later time one wants to change the keyboard layout.

In my case I had a US keyboard, did the installation of Ubuntu, and later the keyboard broke.
I switched it with a backup .BE layout keyboard(AZERTY), and tried installing in Ubuntu (and later in Xubuntu as well) the other keyboard.

This went all fine; but upon reboot Ubuntu let me know my keyboard was BE, but the layout appearing on screen was US.
Whenever I could log into my account; I needed to select the settings-manager/Keyboard layout to change my keyboard to US, and back to BE.

This text layout (azerty vs qwerty) difference on itself isn't superbad if it wasn't for he fact that my login password contains characters that are not visible on a US keyboard...(and I wasn't into the mood for finding the ASCII code tables for these characters).

I wanted to report this bug here, in hope I have contributed something to the Ubuntu community this way.

So I found another solution to this; and that was reinstalling (X)Ubuntu and during the install select the BE keyboard; although I can imagine reinstalling would not be a good solution for other users running similar issues.
