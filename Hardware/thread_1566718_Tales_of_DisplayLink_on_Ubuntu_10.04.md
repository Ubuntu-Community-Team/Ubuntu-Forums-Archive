### Post by imcrazy on 2010-09-02
I write this to thank all those who made the various posts regarding DisplayLink USB graphics adapters that allowed me to get as far as I did with my configuration.  My hope is that this write-up will help others.  I didn’t get everything working just the way I wanted but I did get one DisplayLink adapter working as a second monitor.  And I think I just need to upgrade one DisplayLink adapter to get a third monitor working.  I think the bottom line is that some, but not all, DisplayLink adapters will work on 10.04.   

For sometime on Windows XP I have been running the following 3 monitor configuration with monitors physically arranged as shown.  This arrangement allows me to have a lot of display real estate on any system.  Or keep an eye on things on one system with the DisplayLink displays while working on another via the KVM.

Even under XP, switching the Displaylink adapters between systems is not something you want to do every few minutes because when XP discovers the USB devices it doesn’t arrange the display space as shown above – you must manually arrange the display space to match the physical arrangement.  Linux actually has an advantage here because you can spell out absolute positioning of screens in the xorg.conf file.

As you can likely surmise I want to put linux on System 2 and still be able to use the 3 monitors as I was on XP. And as you probably also guessed things just didn’t “plug&play” when I put linux on System 2.  I did get the upper left Samsung and lower Westinghouse displays working.  I believe the issue I had with the upper right display was that the IOGear Displaylink adapter is too old to be supported.  I think an upgrade to newer hardware will fix that.

First my relevant background – I’ve work on computers since the 70s including Unix but am somewhat of a newbie to linux.  I understand all about how things work in theory but I haven’t made the daring step to doing my own compiles of open source.  So for now, I depend on being able to download binaries.

Now the relevant hardware:
System 2 mobo graphics: Nvidia GeForce 9300 
DisplayLink adapters: Kensington K33928US, 
                      IOGear GUC2020DW6

And the relevant starting software:
Ubuntu server 10.04 LTS from ubuntu-10.04-server-i386.iso
Lubuntu desktop install via aptitude.

Note the starting software had nothing to do with trying to get DisplayLink adapters to work.  I had set up a virtual machine server and added a lightweight GUI desktop for convenience.  What follows is my working /etc/X11/xorg.conf including notes to myself on what I had to do to get it working and the /var/log/Xorg.0.log files with just the Nvidia through VGA monitor attached and with the Nvidia and Kensington DisplayLink adapter through USB monitor attached.  Both log file were produced by booting from the same xorg.cong, the only change was which monitors were attached.

# nvidia-settings: X configuration file generated by nvidia-settings
# nvidia-settings:  version 1.0  (buildd@palmer)  Fri Apr  9 10:35:18 UTC 2010
#####################################################################################
#JGC 2010-08-29
#
#This file is configured to allow Kensington Displaylink adapter to function as a second
#display screen, if attached.  If not attached this file works with the Nvidia chip
#and loads the Nvidia proprietary driver allowing the full resolution of 1920x1200.  
#
#I couldn't get the IOGear Displaylink device working, even by itself.  That may be due to 
#a different, earlier chip.  I may need to get a more recent Displaylink adapter to have 3 monitors.
#It took much to get this working.  Here's what I think is necessary to make the Kensington 
#adapter work. 
#
#Download and install a 2.6.34 kernel as it has updated udlfb, the kernel Displaylink
#framebuffer driver.  Kernel packages can be found at http://kernel.ubuntu.com/~kernel-ppa/mainline/
#but that only seems to have generic kernels.  To fully utilize more the about 3.5Gb
#of memory you need PAE (Physical Address Extension).  Generic kernels aren't compiled
#to use PAE, only special PAE or server kernels are.  It appears a PAE version
#is available here http://www.ubuntuupdates.org/packages/show/203551.  This Xorg file was
#tested to work on the generic kernel from http://kernel.ubuntu.com/~kernel-ppa/mainline/v2.6.34-lucid/.
#
#Install the package xserver-xorg-video-displaylink which is the X driver for DisplayLink devices.
#This recently was called xf86-driver-displaylink but the new package has the latest version.
#
#To get a starting /etc/X11/xorg.conf connect each monitor to a graphics output on the Nvidia chip
#on the Zotac board and execute nvidia-xconfig.  This will create a new xorg.conf if one doesn't
#already exist.  Save the xorg.conf for each monitor.  
#
#Use the info in xorg.conf file for each monitor to create a combined xorg.conf. Then update the Device 
#for Displaylink adapter to have Driver "displaylink" and Option "fbdev" "/dev/fb0".  Also update
#Identifier strings for the DisplayLink adapter to "Device1", "Screen1", "Monitor1".
#
#There are issues with the Nvidia driver for the Nvidia chip on the Zotac board and the 
#Displaylink driver.  The Displaylink driver will be unloaded if the Nvidia device is Screen 0 in the 
#ServerLayout Section.  The workaround is to make the Displaylink Screen 0.  The
#consequence of the workaround is the that Nvidia is not recognized to use the Nvidia 
#proprietary driver. This limits the Nvidia screen resolution to 1680x1050 and none of the monitor 
#GUI monitor tools work.
#
#Additionally color depth for the Nvidia device must be set at 16 to match the maximum of the 
#Displaylink device.  Note that this setting limits the Nvidia to 16 even when the Displaylink
#device is not attached.
