### Post by Toz on 2011-07-20
A few questions/suggestions:

1. Looks like you've got virtualbox installed. When you are trying to suspend, do you have active vb instances running?

2. You are using the kernel parameter acpi_sleep=nonvs. Why? What problem did this solve?

3. You are also using the nohpet kernel parameter. Why? Both seem to be suggested for solving suspend/resume problems. Any chance you could test without or just one of each?

4. Reboot. Login and try to suspend. On resume, post back results of **dmesg**. Lets see if there is anything there logging the reason for the delay.

5. Reboot and instead of logging in, go to console (Ctrl+Alt+F1). Stop gdm:
```
sudo service gdm stop
```then try to suspend manually:
```
sudo pm-suspend
```
Any change?

6. Have you tried **uswsusp**?
```
apt-cache policy uswsusp
```
If it's installed, you can try it by running:
```
sudo s2ram
``` (and if you get an error about an unrecognized system):```
sudo s2ram -f
```

7. There is always the possibility of testing newer kernels.
