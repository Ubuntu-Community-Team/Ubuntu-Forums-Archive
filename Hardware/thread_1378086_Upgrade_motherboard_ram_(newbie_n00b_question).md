### Post by cascade9 on 2010-01-11
OK, here goes....

PC1600  = DDR200 (1.6GB/sec)
PC2100 = DDR266  (2.1GB/sec)
  PC2700 = DDR333  (2.7GB/sec)
PC3200 = DDR400  (3.2GB/sec)

To be honest, I'm not sure if it will even run at 3xDDR400. It might..it might not. If it does, it wont be running at DDR400 speeds, it will slow down to DDR200/266. With Pentium 4s, running at that rate will give you a performance hit. It wont be that bad if you are using a 400 or 533 FSB P4, but if you have one of the those, you would do better to spend less money on ram and more on getting a newer 800Mhz FSB P4. 

As long as it isnt a 32 chip stick** **, you could just buy a 2GB stick. But It would be...meh IMO.  (as for what a '32 chip stick' means, its counting the number of chips on the RAM stick). 2GB DDR1 sticks will be hard to find these days as well.... 

No matter what you do, running 3GB in that motherboard will cause a performance hit. You would be better off just getting 2x1GB sticks. Apart from _very_ heavy ram use, it would be faster than 3GB. Noticeably faster. 

As for the 'latency' column, what those numbers basicly mean is how long between a request to send information in memory is sent, and when it arrives. The lower the better. Each stick doesnt have to have the same latency, but if you mix and match latency, the highest number is the speed it will use.
