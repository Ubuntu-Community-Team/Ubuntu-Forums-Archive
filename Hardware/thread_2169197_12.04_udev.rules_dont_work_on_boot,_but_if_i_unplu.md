Hello,

as i cant choose which webcam is used in a browser (and which gets the kernel name /dev/video0 or /dev/video1 on boot), i just want to set one of them to only be accessable by root, because it only takes pictures with a cronjob which can be root's cronjob.

in /lib/udev/rules.d/70-udev-acl.rules there is a rule which sets acls for printer,webcams,scanners,thumb drives and so on. for every logged in user it sets setfacl -m u:rw:username /dev/device. so i have to set the camera to root:root and or 600 and remove the acls.

so here's my rule /etc/udev/rules.d/75_cctv.rules
```
KERNEL=="video?", RUN+="/bin/setfacl -b $env{DEVNAME}", SYMLINK+="MY%k", OWNER="root", GROUP="root", MODE="0600"
```

then i also tried this:
```
KERNEL=="video?", ATTRS{idProduct}=="332d",  RUN+="/bin/setfacl -b "/dev/%k"", SYMLINK+="MY%k", OWNER="root", GROUP="root", MODE="0600"
```

In both cases the OWNER,GROUP,SYMLINK and MODE are respected, but when the computer boots up, the ACLs are still there. If i unplug the camera and replug it, all the rules are executed and the ACLs are gone. But i dont want to replug the device every time. sure i could also write an init-script which sets the ACLs, but it would be even nicer if udev works like expected.

Even "udevadm test" tells me that the rule is executed... i have no idea what else to try. Does anybody else have a hint?
