### Post by lbutler3 on 2015-04-06
I'm having multiple problems with my system built on:

ga-z87x ud4h mobo.
Xeon 1230 3.3GHz
G.Skill Trident DDR3-2133
EVGA 650W PS
GTX 770

1) My keyboard is a Lenovo SK-8855 (Thinkpad external keyboard with built in pointing stick).  
   In both the BIOS and Windows 8.1 both the keyboard and pointing device work fine (separate HD swapped in)
   Booting Ubuntu 14.04.2, only the keyboard works.
   This same keyboard works perfectly on my Dell T7500 or Latitude 6410 at work running Ubuntu.
   If I use a Logitech wireless keyboard and wireless mouse (through a single wireless dongle) both kbd/mouse work.  

2) On the first boot or two, the "power saving" modes (monitor off and later CPU suspend) work fine.  After that the machine does not suspend itself.
3) When shutting down linux, I often get "reboot" behavior instead of shutdown (and vice versa).
4) after a week or so, the Linux hard drive seems to "lock up".  Sometimes a reboot will fix it.  Sometimes I have to re-install the OS.  I've had this problem with both a Corsair M5 SSD and a Seagate Hybrid drive as the linux drive.

I've run memtest overnight and the GSkill Trident X DDR3-2133 does not produce any errors.  I have upgraded the BIOS on the UD4H and the SSD.  The power supply tests good using a Rosewill RTK-PST.  According to my Kill-A-Watt the EVGA 650W PS never draws more than about 300W even under heavy load.

I replaced the mobo with an Asus Sabertooth Z87 with no significant change, so I am back to the Gigabyte UD4H.

I am starting to think this machine is allergic to linux .
Any ideas would be most welcome.
