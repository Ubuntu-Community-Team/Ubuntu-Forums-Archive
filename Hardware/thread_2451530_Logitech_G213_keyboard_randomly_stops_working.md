### Post by jonathanbossenger on 2020-10-06
I recently purchased a Logitech G213 Prodigy RGB keyboard. They keyboard works flawlessly out of the box on my Ubuntu 20.04 workstation.

To be able to control the RBG backlit keys, I have installed this python package [https://github.com/MatMoul/g810-led](https://github.com/MatMoul/g810-led). This also works as expected.

Not sure if this would be relevant but I also have the matching Logitech G102 Prodigy mouse, which I control using Piper - [https://github.com/libratbag/piper](https://github.com/libratbag/piper)

Randomly, my keyboard goes dead. It stops working, all lights go off, and unplugging and plugging it in again don't help. I've even tried plugging into a different USB port. I have to reboot the machine to get it working.

So at this point my two questions are 

1) Is there a way to "reboot"/"reactivate" the keyboard without needing to reboot the machine
2) Is there a way I can specifically log any errors that may be occurring on the keyboard when this happens, so that when I reboot I can see what the error was that causes it to die

I'm trying to figure out if this is a hardware conflict, or if the keyboard is faulty.

### Post by Autodave on 2020-10-06
Since no one else is jumping in, lemme try.  Your problem could be a faulty power supply.  How many USB items do you have connected?  Are there any hubs being used?  If so, powered or non-powered?

Can you turn off the lighting for the keyboard and leave it off?  Try that.  If keyboard keeps functioning, then I would have to say that either the backlighting of the keyboard is drawing too much power or else the power supply is weak on the 5V line.

### Post by niketpathak on 2021-09-05
I landed on this thread while trying to make the LED's work. So for others who are concerned by this, here are my findings - 

I tried using [https://github.com/MatMoul/g810-led](https://github.com/MatMoul/g810-led) as well as [https://github.com/SebiTimeWaster/G213Colors](https://github.com/SebiTimeWaster/G213Colors) (and tested its forks that add a GUI too), but it didn't seem to work. I did want to try out [https://github.com/libratbag/libratbag](https://github.com/libratbag/libratbag) (As it doesn't support the G213 Prodigy keyboard out of the box, using it requires some tinkering/testing to be done) and I didn't really have time for that. 

What worked for me is this excellent opensource project => [https://gitlab.com/CalcProgrammer1/OpenRGB](https://gitlab.com/CalcProgrammer1/OpenRGB) Its got a nice GUI and things WORK as expected. Feel free to give it a try! 

Tested on Ubuntu 20.04.
