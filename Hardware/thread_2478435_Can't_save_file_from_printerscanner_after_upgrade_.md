### Post by TheFu on 2022-08-26
When it comes to issues, there are a few standard questions.  
a) Did you look at the system logs?
b) Which OS, exactly is being used?
c) Which DE, exactly, is being used?
d) Which program (look in the process tablet) is being used?  The name in the "menu" is less helpful. The actual executable program-file is best.  How was it installed?  From Canonical repos? From a .deb package download?  From a snap?  From a flatpak? Following manual install instructions?  Is it an appimage?
e) Which version of the program, if it isn't the current one?
f) Did you run sudo apt update && sudo apt full-upgrade already to ensure everything is fully patched on the system?
g) If there is a peripheral involved, exactly, what is the vendor, model, and revision level of the hardware?
h) If there is a peripheral involved, exactly, what is the driver version being used?
i) Did it ever work?  When it did, what were the conditions, OS, drivers, DE, and program that did work?  What's different this time?  

j) What, exactly, are the steps to reproduce the issue? Act like people following along are 10 yr olds and need to be told every detail, exactly.  If we have to guess, we'll make assumptions that probably aren't true, since we are all different humans.

Where are you trying to save the file?  That matters greatly for permissions.
In 22.04, default file permissions were restricted back to how we did it in the 1990s, pre-Ubuntu.  While I think it is a good thing, it will likely break some workflows.  In theory, saving any file to /tmp/ should work for any userid, but snap packages broke that.

And lastly, are there updated drivers for the all-in-1 device for 22.04 from the vendor?  It is their responsibility to provide current drivers when we give them money.  [https://www.linuxtoday.com/developer/hp-linux-imaging-and-printing-drivers-now-support-ubuntu-22-04-lts-and-fedora-36/](https://www.linuxtoday.com/developer/hp-linux-imaging-and-printing-drivers-now-support-ubuntu-22-04-lts-and-fedora-36/) says that 
> HPLIP 3.22.6 is here exactly two months after the HPLIP 3.22.4 release, which only added support for the Manjaro Linux 21.2 distribution and several new printers, to add support for more recent distributions, including Ubuntu 22.04 LTS, Fedora Linux 36, and MX Linux 21.1.
Sometimes, hardware support gets dropped by vendors, which is why being picky about hardware pays off.  When/if I replace my 2008 all-in-1, I'll ensure that it supports "driverless printing" and computerless scanning and is USB connected (not networked).  Computerless scanning means placing the scans onto SDHC, microSD or USB storage without a computer connected.  Same for faxes.  For about 10 yrs, my all-in-1 worked well, printing still works, but scanning hasn't worked under Linux since 18.04.  Sending faxes is completely computerless, thankfully.
Why not networked?  Because networked devices all like to phone home without permission and I value privacy. We should all value/demand privacy from our peripherals, right?
