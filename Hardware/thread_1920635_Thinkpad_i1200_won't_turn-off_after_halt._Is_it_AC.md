My laptop: 20th century state-of-the-art: IBM Thinkpad iSeries 1200, 600MHz Celeron, 192MB RAM (PC100 SODIMM), 10GB HD, dual-boot (5GB Lubuntu 11.10, and 5GB Win 2000 SP4 Rollup 1)

Lubuntu 11.10 works pretty good (Audacious, VLC, Midori & Uzbl... but don't even think about running new OpenOffice on this machine), I can connect to 'net using USB 2.0 LAN adapter (some cheapo translucent-blue dongle from eBay... says KY-QF9700) via generic USB 2.0 PCMCIA  card.  I had to manually edit xOrg config file to disable nasty pulsating vertical lines on screen, but everything else works great straight outta box.

If we disregard a bit of overheating (that's cooling fan's fault, can't blame Lubuntu) I have no problem with this machine except this one - when I go to system shutdown (Lubuntu 11.10) it "halts" but the machine dooesn't really turn off - I mean Linux doesn't work any more but the hardware is still powered on (hard disc spinning, screen works). It's much like on those ancient AT machines (like 486) if you go "shutdown" on Win95 it would halt & then say "You can press power button now", and you'd have to manually press the switch. So after my Thinkpad halts i have to "kill" it by pressing and holding the POWER switch.

Note that shutdown procedure works great on Windows 2000 (after it halts it stops giving power to the hardware, stops harddrive spinng and screen, LEDs turn off). So I guess it's something to do with new kernel in Lubuntu 11.10.

Any suggestions on how to solve this are apreciated. :)
