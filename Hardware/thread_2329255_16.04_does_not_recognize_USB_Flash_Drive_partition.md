### Post by rsdias on 2016-06-29
I have a bootable USB Flash Drive created in windows.

When inserted under 16.04, udev (systemd-udevd I guess) creates /dev/sdc, but not /dev/sdc1.
```
# fdisk -l /dev/sdc
Disk /dev/sdc: 7,6 GiB, 8178892800 bytes, 15974400 sectors
Units: sectors of 1 * 512 = 512 bytes
Sector size (logical/physical): 512 bytes / 512 bytes
I/O size (minimum/optimal): 512 bytes / 512 bytes
Disklabel type: dos
Disk identifier: 0x0010ed50


Device     Boot Start      End  Sectors  Size Id Type
/dev/sdc1        2048 15974399 15972352  7,6G  7 HPFS/NTFS/exFAT
```

So, I created it with mknod
```
# mknod /dev/sdc1 b 8 33
# chown root:disk /dev/sdc1
# chmod ug+rw /dev/sdc1
```

Still it is not seen:
```
# mount /dev/sdc1 /mnt/other/
mount: /dev/sdc1 is not a valid block device
```

So maybe something deeper is missing from kernel/module?  Not just a udev fault.

### Post by rsdias on 2016-06-29
That was not the case as the problem was not the mountage per see, but to recognize that there is actually an /dev/sdc1.

Interesting enough, parted did not recognized it either:
```
# parted /dev/sdc
GNU Parted 3.2
Using /dev/sdc
Welcome to GNU Parted! Type 'help' to view a list of commands.
(parted) p                                                                
Error: /dev/sdc: unrecognised disk label
Model: Generic Flash Disk (scsi)                                          
Disk /dev/sdc: 8179MB
Sector size (logical/physical): 512B/512B
Partition Table: unknown
Disk Flags: 

```

The **partition table** gave the clue.
In fdisk, using **toggle the dos compatibility flag**, which is deprecated, made it show up in parted and the kernel did recognize it.
