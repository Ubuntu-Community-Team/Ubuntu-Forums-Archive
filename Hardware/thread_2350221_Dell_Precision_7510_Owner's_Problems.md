### Post by sourcelibre on 2017-01-22
I had a Dell Precision 7510 with two NVme drives (M2) and UBUNTU factory installed.

The problems: 

   
[LIST]
[*]The BIOS System Setup <F2> -> System Information would never recognize that a second NVme drive was installed.
[/LIST]


[LIST]
[*]Also, under System Setup <F2> -> System Configuration -> Drives BIOS only listed that SATA-0, SATA-1, SATA-3, and M2.PVie SSD-0 were enabled.  It never listed the M2.PVie SSD-1 drive.
[/LIST]


[LIST]
[*]In addition, after enabling Advanced Legacy ROM boot and disabling Secure boot in System Setup by use of boot sequence, <F12>, the machine would only boot from the USB port if the (SSD-0) was disabled.
[/LIST]

I worked with Dell for over a month.  Dell thought it was a hardware issue and sent me a new machine, but the new machine had the exact same problems.  Finally, Dell concluded it must be an engineering issue and had me send the machine back to the factory for analysis.

Dell was surprised no one else reported this issue, as was I.  I can only assume others did not have two NVme drives or a firmware changed.  By the way, if I ran the pre-install UBUNTU on SSD-0, I could format and partition the second drive, NVme1, but the BIOS never saw the drive on boot.

I liked the machine and was planning a project around the machine.  I cannot believe others are not operating the machine with two NVme drives and not having these problems.

I would like to hear your experiences with the machine or suggestions/workarounds on fixing these problems.

Add your problems about the machine if you want.
