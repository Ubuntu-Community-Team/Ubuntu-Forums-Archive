### Post by segfault-bilibili on 2022-10-22
xinput told me that I have a "slave pointer" device called "ETPS/2 Elantech Touchpad".

Initially the whole "touchpad" region in the system settings took no effect at all, including disabling it entirely, toggling "natural scrolling" etc, nothing took any effect.

Then Google led me to Ask Ubuntu, where I met an answer said I should try ```
sudo apt remove xserver-xorg-input-synaptics
``` whoa, many thanks to FedKad who wrote this answer! At least this seems to fix the "touchpad" settings. (BTW it seems to have some palm detection, despite I cannot tune it)

However the situation is still not ideal - I think it should be able to automatically disable touchpad when a mouse is plugged in.

Tried ```
gsettings set org.gnome.desktop.peripherals.touchpad send-events disabled-on-external-mouse
``` no luck.

Besides, actually I found that the system settings didn't seem to touch gsettings at all, because diff showed no difference between the `gsettings list-recursively` results before/after toggling switches. I still have no idea what actually happened after clicking/toggling those switches appearing in the system settings panel.

I see another suggestion to try `touchpad-indicator` from ppa:atareao/atareao - which however didn't even finish `apt update` at all, HTTP 404.
