### Post by kd5vdo on 2014-03-28
Hello all!

I searched through the forums and didn't find the same problem I'm having. I have a Canon LIDE 110 scanner. It scans just fine on my laptop, an Asus UX301, but on my homebuilt machine, I experience the same errors you read about across these forums. It will only scan one page at a time, if at all in Simple Scan, and gscan2pdf frequently throws errors when using it. Both machines are running Xubuntu 13.10. Interestingly, if I boot up my Win7 VM and passthrough the scanner, it works just fine. This at least tells me that it's not a hardware problem--I think. I don't know if it's important or not, but I'm using KVM as the hypervisor.

Also, the Asus only has USB 3.0 ports, which were thought to be a problem in other threads, and it works well on those. The homebuilt machine has both USB 2.0 and 3.0 ports, but neither standard works.

Any ideas?

Thanks!

### Post by GigabyteProduction on 2014-03-28
If it does work in a VM then i think that does mean it's a hardware issue, because the virtual machines will have fake chipsets that are designed to work with linux, so the linux in the VM is given what is known to work while the linux in the physical computer is given what's actually there, but that's about alli can say because i'm not sure what drivers or what to look into decause i've never had to change anything with usb before.
