### Post by The Cog on 2012-08-27
I think Linux does its own hardware detection, not relying on the BIOS to tell it what is there. So maybe the problem lies in the BIOS, telling windows that there is no display. Maybe there is a BIOS setting somewhere, or maybe there is a strange hardware fault that is misleading the BIOS. I don't know, I'm just rambling.

If you decide to try Linux again, it would be worth looking at Xubuntu which has a more conventional PC style GUI than Ubuntu. Or some people prefer Linux Mint, but I gather that's heavier on resources than Xubuntu.
