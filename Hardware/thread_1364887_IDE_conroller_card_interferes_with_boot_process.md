### Post by MarcLeo on 2009-12-26
I tried to transfer two hard drives with a Ubuntu 8.04.1 system on them to a newer motherboard (HP Pavilion a6648f) that had had the sata hard drives stripped out off it, leaving a sata DVD drive. I added an IDE controller card (IT8212f) with the two drives on it. The IDE bios recognized the hard drives, and they appeared as a boot option in the motherboard bios.

I was not so naive as to think that the old Ubuntu system would necessarily work, and I was prepared to reinstall, but to my surprise, it worked perfectly. But only a couple of times.

Now, however, Ubuntu hangs early into the boot process. It hangs in a similar way whether I boot from the hard drive, or from the CD. If I remove the IDE card, the Live CD boot works perfectly. As soon as I plug in the controller card, it hangs during boot-up. 

So, The IDE controller card seems to be interfering with the boot (from CD or from HD), but I can't figure out where to look from here. 

I would be grateful for any comments. I've been using ubuntu for a few years, at a pretty rudimentary level.
