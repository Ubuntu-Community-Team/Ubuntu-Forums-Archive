### Post by WanderingAlbatross on 2010-08-17
Recently I got a new hard drive.  My old drive was slowly dying over the years.  It would fail often on startup.  The BIOS would say HARD DISK NOT FOUND, press F2... etc.

The new drive was definitely the fix.  The strange thing was that the week before I tested it through Disk Utility.  It reads the Smart Information about the drive.  After a rather lengthy test, (I wanted to be thorough), it said everything was fine.  I know that it is not possible to test an ATA drive.  My drive claims to be a SATA (or perhaps an ATA using an SATA contoller if that's possible), but then on the utility it lists as an ATA.

So why did it not read correctly?  Shouldn't it have predicted the problem?

Incidentally I did learn some interesting things while keeping the old one alive.  On boot up your computer seems to check to see if the drive is ramped up to speed at a certain time during boot-up.  This is what determines whether it sends the no-read error message.  I've been doing various things to stall the computer long enough to ramp up the drive:  Removing all flash drive, etc.  Trying once, then reset used to help.  The last week it wouldn't boot at all unless I had it look at a blank CD first.  I still think the old drive would work if I plugged it in externally.  It just took longer and longer to boot up as time went on.

Any thoughts?
