Hi,

ok, so this has been doing my head in for some time now. I have an HP Compaq 8510p laptop, with an ATI Mobility Radeon HD2600 graphics card. The screen has a native resolution of 1680x1050. I use the proprietary fglrx drivers and have both KDE and Gnome installed (gdm is the default display manager since kdm doesn't seem to get along with fglrx, but I usually use KDE as the main desktop environment).

When the laptop is on its own, everything is fine. However, a lot of the time, I use it in a docking station with an external monitor (HP 1955) attached to it. That monitor has a native resolution if 1280x1024. The laptop itself is closed while in the dock, so we're not talking about a dual-monitor setup here. I only want to use the external monitor.

What happens, then, is the following: I boot the laptop up while it's in the dock and am greeted with a black screen instead of gdm. I switch to tty1, then back to tty7 and gdm does show, but my monitor complains that the input signal is out of range. I can still see the screen and login, at which point I can start the ATI control center. That one then tells me that the screen resolution is set at 1600x1200, which is obviously too high for the external monitor (but note that it is also *not* the resolution of the laptop screen). I can then reduce the resolution down to 1280x1024 and everything is fine until the next time the X server gets started.

However, I haven't found a way to get the system to correctly identify the screen resolution of the external monitor at bootup yet. What I can do, of course, is set the maximal resolution of the monitor to 1280x1024, either manually in the xorg.conf or using the KDE Monitor settings, but that then means that the laptop boots into that resolution even when not in the dock. 

Also, this problem is fglrx-specific. I can use the non-proprietary drivers and everything will work fine.

So - does anyone have any idea on how to get fglrx to detect the correct resolution of an external monitor when the laptop is closed and in a dock? Alternatively, is there a way to specifically specify a resolution in xorg.conf that is only to be applied to external monitors but not the internal 1680x1050 one?

thanks!

### Post by teslar on 2008-05-14
Okay, I now have a partial solution. This may actually be useful for other people with the same problem. Sometimes the simple obvious solutions do work - I just added a modes line to the xorg.conf with the two resolutions I want to use:
```
Section "Screen"
        Identifier "aticonfig-Screen[0]"
        Device     "aticonfig-Device[0]"
        Monitor    "aticonfig-Monitor[0]"
        DefaultDepth     24
        SubSection "Display"
                Viewport   0 0
                Depth     24
                Modes   "1680x1050" "1280x1024"
        EndSubSection
EndSection
```
Now the laptop on its own correctly boots to 1680x1050 and when it's in the dock it defaults to 1280x1024.

However, I still have the issue that the external screen will initially be blank (after booting is finished - the splash screen displays fine) when the laptop is booted in the dock. To see gdm, I first have to switch to tty1 and back to tty7. After that, I can restart the X server as often as I want, it will immediately display on the external monitor, so I suspect the X server initially can't detect the monitor, but I don't understand why.

Does anyone have any ideas?
