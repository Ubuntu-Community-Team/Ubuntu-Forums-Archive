### Post by sudodus on 2022-07-01
If you run mainly command line statistical software you need no powerful graphics chip, but if you need powerful number crunching, it might be bundled with powerful graphics in a laptop workstation, for example Dell Precision.

If you buy a model that is not brand new, maybe released last year, chances are much better, that the linux hardware drivers will work well.

A refurbished 2-3 year old enterprise class laptop (for example Dell Precision) might serve you well for a fair price and there would be no problems with hardware drivers.

### Post by kurt18947 on 2022-07-01
I think any laptop that is all Intel should be pretty compatible. I'm partial toward Intel based Thinkpads in part because they seem to play well with Linux.  WiFi adapters in particular can be troublesome, Intel WiFi seems to work out of the box. Given your usage, I would avoid non-Intel graphics chips. I'll second Sudodus' advice about year or two old hardware. Even if the device is new, a model that's been around for a year or two with few or no component changes is likely to have had any issues addressed.

Webcam was a Logitech c920 - you know, THE most popular webcam ever made in the world.  Logitech doesn't really support Linux or many Linux drivers, so it sat in a box, unused, after I got it as a novelty .... probably around 2013. I got it for recording my presentations do different groups.  Then I got a new chromebook a few years later and decided to see if Google's marketing had convinced Logitech to support chromeOS.  Plugged it in and it just worked and has been working ever sense then.

The All-in-One is Brother and had known Linux support for a long time, but after 10 yrs, they just stopped.  I never planned to use it as a printer (ink is too expensive), but wanted a ADF scanner and fax machine for some business -to- govt paperwork. The fax "sending" part has always worked, since it isn't tied to any OS.  Without ink, the receiving part only worked when connected to Windows.  Scanning worked for a long time, but has never worked since I moved off 16.04.  These days, the All-in-One devices can read and write to USB or SDHC storage - just be certain any model bought has that feature and you won't care about Linux proprietary driver support.  Brother is one of the better brands for Linux support - them and HP, but we have to be careful about the specific model. It is just better to avoid any proprietary drivers at all, since no business will constantly update the drivers. I got about 10 yrs, which is pretty awesome.  Of course, the all-in-one still works if connected to Windows.  I suppose I could setup a 16.04 virtual machine and use it just for scanning.  But I always need to scan stuff for records, usually before faxing/mailing it off to the govt, so it is almost always a last minute thing.  Booting a Windows laptop a few times a year isn't a huge hardship.

Fingerprint readers are just bad ideas do to poor security.  They should never be used as the only method to unlock access on any device, anywhere. It was only a curiosity, but I never intended to use it. An old cell phone has the fingerprint unlock capability, which I've never used after learning how terrible the security is across all of them, including Apple's.

I have a few perfectly working GPUs that nVidia stopped supporting too.  The F/LOSS drivers for them are limited to 768p resolutions, which I consider unacceptable. They worked at 1200p for a long time without issue, but after moving off 16.04, no more nvidia drivers came. Effectively, they are trash.  The systems barely get used and seldom get a local login. I generally ssh in for everything, so the GPU is a convenience, not really mandated, except that the Ubuntu installers all assume there is a GPU of some sort by default.  I have serial console cables, but those expect a 25-ping serial port, which hasn't been put on most motherboards in 15+ yrs.  My router uses a serial port connection for loading the OS, but none of my other systems that can still boot have a serial port.

It isn't all bad news.  A slide/negative scanner I bought long, long, ago ... WinXP ... didn't work with Vista or later and it didn't work with Linux at the time either, but I plugged it into a Linux desktop and used simplescan just fine with it about a year ago. The cheap device became supported by sane or v4l2, I don't know which, but it works. It must be 17 yrs old now.

So, my point is that not all hardware is going to work and even if it works now, if there aren't F/LOSS drivers, it will eventually become unsupported.
