### Post by caljohnsmith on 2009-01-09
If both your Windows partitions are logical partitions (sda6 and sda7), that means when you installed them they would have put their boot files in some primary NTFS/FAT partition. But since you don't have a primary NTFS/FAT partition, I'm guessing you may have unintentionally deleted Windows boot files when you installed Kubuntu. So in order to get a clearer picture of your setup, how about booting your Kubuntu Live CD, open a Konsole and do:
```
cd ~/Desktop && wget 'http://home.comcast.net/~ubuntu_grub/boot_info_script.txt' && sudo bash boot_info_script.txt
```
That will create a "RESULTS.txt" file in the same directory from where the script is run, namely your desktop; please copy/paste the contents of that file to your next post, highlight the copied text, and click the pound sign # graphic in the Ubuntu forum message box so that the text will get "code" tags put around it. The results of that script will help clarify your setup and hopefully what it will take to boot Windows again.

### Post by caljohnsmith on 2009-01-09
> **helwitch said:**
> I'm still unclear on how the partitions would be numbered, and apparently windows numbers the same way as linux. so, What would the partition numbers be? 4 for sda6 and 5 for sda7?
Grub numbering begins with 0 and not 1, so it goes like:
```

sda1 = (hd0,0)
sda2 = (hd0,1)
sda3 = (hd0,2)
sda4 = (hd0,3)
...etc.
```
So sda6 would be (hd0,5) and sda7 would be (hd0,6).
