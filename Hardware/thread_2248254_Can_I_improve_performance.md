### Post by irv on 2014-10-13
I have older dell laptop with less specs and it runs great with Ubuntu with Unity and it runs great, and fast too. I think your problem is the 1GB of RAM. Two things to do with older PC's is to max out the RAM (RAM is gotten very cheap) and if you can afford a SSD go for it. You can make an old computer run like a new one by doing those two things. When you have low memory your OS will use swap which reads and writes to the hard drive and this slows things down. That's probably whats happening in your case.

### Post by Mike_Walsh on 2014-10-13
Irv's right about the swap usage.

Another thing that can help to speed things up on older systems with smaller amounts of RAM (assuming you can't max it any further), is to adjust the 'swappiness' setting. The 'buntu distros ship with this set to about 60. On my desktop (a Compaq Presario, with 3 GB of RAM), I have this set to 10. On my even older Dell laptop, with a max of 1 GB of RAM, I have it set even lower, to 5. 

The 'swappiness' setting controls the system's tendency to 'swap' info from system RAM to the hard drive, and back again. The higher the number, the greater the tendency to 'swap'. 

You can find an explanation of how to control the 'swappiness' settings on [this page]("https://sites.google.com/site/easylinuxtipsproject/lubuntu").

It's almost the first thing mentioned, part way down in the left-hand column. There's other useful stuff for Lubuntu on that same page.

Hope that helps.

Regards,

Mike.

### Post by TheFu on 2014-10-13
My daily use desktop (running inside a VM) has 1G of RAM. Since it doesn't have a video card, it only gets used from a remote connection thru x2go.  The "display device" is usually a low powered netbook.

That desktop from the VM is fast, really fast.  Why?  No Unity. No fancy graphics. It has a simple WM only. Used to run xfce, then lxde, now just a simple openbox config.  GUI cheese is the problem for perceived performance issues almost always.

Linux shouldn't need more than 384MB to run well, unless bloated GUIs are used, IMHO.  Don't believe me? Try [TinyCorePlus]("http://distro.ibiblio.org/tinycorelinux/intro.html").  384MB RAM and it will scream!  Stock Ubuntu Desktop is fairly bloated and has been since Unity was introduced.  It is a trade-off.  More system resources means there is an ability to ask a computer to do more to be "pretty."  Fortunately, this is Linux and we don't have to use the stock user interface if we don't want it (or our hardware isn't powerful enough).

Unity is a problem for systems without powerful GPUs (for certain values of "powerful") and for pretty much every virtual machine out there.
