### Post by darkwing_duck on 2009-08-04
Today at work I put my USB Memory Stick in a workstation that I normally don't use; it's a Windows XP system.  After I was finished at that workstation, I went back to my normal station; a CentOS system.  However, now my USB Memory Stick is read-only and I didn't do anything on the XP system except open and close some files and maybe added a file or two to the memory stick.

Here's the error I'm getting on my Ubuntu system.  I get a similar error on the CentOS system.

```

Cannot move file to trash, do you want to delete immediately?
The file "my_file" cannot be moved to the trash.
Show more details
Unable to create trashing info file: Read-only file system
```

Any help unscrewing my USB Memory Stick would be much appreciated.

Sincerely,
Sucky Linux User

### Post by darkwing_duck on 2009-08-17
I ended up being able to put the USB drive into my Ubuntu workstation on using some tool from one of the Ubuntu menus to create a USB startup disk.  The tool formatted the drive for me and configured it to be a boot up disk.  I just erased all of the boot up images and now my USB drive seems to be working like new.

Thanks me!
