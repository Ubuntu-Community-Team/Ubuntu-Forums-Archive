### Post by dino99 on 2019-05-07
When things goes badly on a system, first do the basic maintenance:
-deacivate/purge non genuine packages (downgrade these packages if necessary)
- clean the system via: clean, autoclean, autoremove, gtkorphan, bleachbit (as root, carefully select options)
- run 'sudo dpkg --configure -a'
- logout/in
- if it freeze again, try to know which process is faulty, via system-monitor applet or htop
