Indeed, like any other OS. However, that's not the point here because it's the computer which isn't booting, not the OS -> The OS is loaded after POST (power on self test) and yours, with the new memory fails that test therefore it won't even **attempt** to load any installed OS.
The Hardware section is intended to support the **use** of certain hardware parts or peripherals in Ubuntu as some aren't natively supported and may require additional drivers and/or tweaking. Your case has nothing to do with it and any user with just the basics should know that already.
