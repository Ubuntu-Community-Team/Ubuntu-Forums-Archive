Ok, I have hit my head against the wall long enough.  Want to help someone else (and potentially myself in the future) to get this resolved quicker than it took me this time.  I will try to quickly retrace the path I have taken.

(1) I downloaded the Lacie 4L RPM files, converted them into DEBS
```
sudo alien -g LightScribe\ Host\ Software\ 1.8.15.1\ Linux.rpm
cd ./lightscribe-1.8.15.1
sudo gedit debian/control
```
Changed the line to read: ```
Architecture: i386, amd64
```
Then:
```
sudo debian/rules binary
```
Minor Success! I have a DEB package "lightscribe_1.8.15.1-1_amd64.deb"

(2) Repeat above process for the "LaCie LightScribe Labeler 1.0 Linux.rpm" file, yielding a DEB "4l_1.0-1_amd64.deb"

**NOTE: I can't download the lightscribe DEB's anymore, as the site has been shut down... but I had them on another computer in the office so was able to nab those (I have them in the cloud now in case anybody needs them).**

(3) Install 32-bit libraries, as required for the lightscribe DEBs...
```
sudo apt-get install ia32-libs
```

(4) Install the LightScribe DEB's
```
sudo dpkg --install --force-architecture lightscribe-1.18.8.1-linux-2.6-intel.deb
```
Had an error about "/usr/lib/libstdc++.so.5" existing, did this:
```
sudo cp /usr/lib/libstdc++.so.5 /usr/lib/libstdc++.so.5.bu
sudo rm /usr/lib/libstdc++.so.5
sudo dpkg --install --force-architecture lightscribe-1.18.8.1-linux-2.6-intel.deb
sudo dpkg --install --force-architecture lightscribeApplications-1.18.6.1-linux-2.6-intel.deb
```
Minor success, no errors... moving on

(5) Install the Lacie DEBS
```
sudo dpkg --install ./lightscribe_1.8.15.1-1_amd64.deb
```
Errors involving chown and a group named "wheel", which did not exist on my system...
```
sudo addgroup wheel
sudo adduser username wheel
sudo dpkg --install lightscribe_1.8.15.1-1_amd64.deb

```
Error involving the file "/usr/share/doc/lightscribeLicense.rtf" not existing...
```
sudo touch /usr/share/doc/lightscribeLicense.rtf
sudo dpkg --install lightscribe_1.8.15.1-1_amd64.deb
sudo dpkg --install 4l_1.0-1_amd64.deb

```
No errors, success?

4L-cli enumerate says no...
```

$ 4L-cli enumerate
Using /etc/lightscribe.rc
Segmentation fault (core dumped)

```

Back to Google, tried [shizeon's fix](http://ubuntuforums.org/showthread.php?t=2147964&p=12660449#post12660449)... but something was wrong.  After a lot of "sudo ls -l" on different directories realized I was bouncing links a few places into a dead end.
```
sudo dpkg --install --force-architecture lightscribe-1.18.8.1-linux-2.6-intel.deb
sudo rm /usr/lib/i386-linux-gnu/libstdc++.so.5
sudo ln -s /usr/lib/lightscribe/libstdc++.so.5.0.7 /usr/lib/i386-linux-gnu/libstdc++.so.5
```

Result:
```
$ 4L-cli enumerate
Using /etc/lightscribe.rc
Drive path: /dev/sr0
Usable: 1
Full name: ATAPI iHAS224 A ZL0P 200
Model: iHAS224  A
Manufacturer: ATAPI
Capabilities: monochrome
Drive inner radius: 21700
Drive outer radius: 58700
```

I will proceed with a test, but as far as I am concerned, it works!!!  Hopefully this helps someone else.  This is not a tidy How-To, but if you read along it should make sense and help you if you have any of the various problems I had at any point in the journey.
