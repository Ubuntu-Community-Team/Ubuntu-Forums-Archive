### Post by ayami2 on 2014-08-13
I recently purchased a System 76 Ubuntu desktop and have been seeing occasional drops in connectvity for wireless mice and keyboards.

I started with the Logitech MK270 combo and within a day was seeing fairly common drops: moust cursor would freeze, unresponsive; the keyboard wouldn't respond, Caps lock wouldn't light, and keystrokes would be delayed, and some lost entirely. It seemed to happen often when logging in from the lock screen.

I tried some alternate hardware as well: I got an Anker wireless mouse, but it was even worse, tons of lag, so I got a Logitech MK320 combo. The keyboard has been working pretty well, but the mouse showed a little lag.

The best I've been able to do is using the MK320 keyboard and the MK270 mouse, each with a different receiver. But I've still seen a little bit of lag on the mouse, and a couple instances of the keyboard dropping, including losing entire keystrokes. It seems to be worst when using Firefox or playing games in Steam.

As I said, it is a new desktop, Ubuntu 14.04, all updates downloaded. It has an Intel wireless card for wifi, and I also use a pair of wireless headphones, though the keyboard has dropped even when the headphones are turned off. The only tweaking I've done was some alternate video card (NVIDIA) drivers, but I cleared those out and went back to the stock 331 driver, but it is still happening.

System 76 offers the MK320 keyboard-mouse combo with their desktops, so I assume it works pretty well, making me think it must be some sort of interference problem, or some package I installed/uninstalled messing with it. Using the mouse and keyboard from different sets definitely works best; no drops while typing this, but it's not perfect. Problem is it happens randomly, and usually not for long, so it's hard to test with different wireless devices disabled. There doesn't seem to be anything that reliably triggers it.

Any advice is appreciated.
