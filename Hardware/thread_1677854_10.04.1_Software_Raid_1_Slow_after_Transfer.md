The two drives were working perfectly on GA-965GM board with an e4300, which was used a home NAS.

When I moved the drives to an atom330 board D945GCLF2, everything worked properly except for the RAID write speeds which dropped down to about 25 29M/sec from about 80.

65536+0 records in
65536+0 records out
536870912 bytes (537 MB) copied, 21.6229 s, 24.8 MB/s

The reads are ok.
> maxim@HS:~$ sudo hdparm -Tt /dev/md1

/dev/md1:
 Timing cached reads:   1254 MB in  2.00 seconds = 626.58 MB/sec
 Timing buffered disk reads:  266 MB in  3.01 seconds =  88.32 MB/sec
maxim@HS:~$

Since the same installation was working properly on the other board, then this means that this is not a problem with the drives. And it is not a problem with alignment as these are 500GB drives.

I am not sure what to do to fix this.
