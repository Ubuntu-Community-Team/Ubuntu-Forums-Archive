### Post by dabl on 2010-07-08
Alt-F2 "gksudo nvidia-settings" with no quote marks.

X Configuration > click on "Detect Displays".

Set the resolution that you want for default.

Do NOT set refresh rate.

Click "Save to X Configuration File" > "OK"

Restart X, or reboot computer.

Do NOT try to use System > Preferences > Monitors.

### Post by dabl on 2010-07-08
OK, I just now saw the graphic of your nvidia-settings.  That's a problem.

First do Ctrl-Alt-F1, and that will dump you to the CLI -- the console.  Issue these commands there:

```
sudo service gdm stop
```

```
sudo nvidia-xconfig
```

```
sudo service gdm start
```

then you will need to log in again.  Then do Alt-F2 "gksudo nvidia-settings" and let's see if it indicates that it is using the Nvidia driver.

### Post by realzippy on 2010-07-10
Ok,according to the nvidia homepage the 7600 GS is supported by the
256.35 driver...
I would try this:

First **disable** any nvidia-driver in hardwaredrivers,then **reboot**.
After reboot,open terminal and run:

```
sudo apt-get purge nvidia*
```

```
sudo add-apt-repository ppa:ubuntu-x-swat/x-updates
```

```
sudo apt-get update
```

```
sudo apt-get upgrade
```

Then **reboot** again,open terminal:

```
sudo apt-get install nvidia-current nvidia-current-modaliases nvidia-settings
```

Then go to System > Administration > Hardware Drivers and **make sure** Nvidia current (256.xx) graphics drivers are **activated**.

Then **reboot** again.

### Post by dabl on 2010-07-11
I've lost track a bit, of what the present configuration is. It looks like the Nvidia driver was installed, and wrote an xorg.conf file.

I just looked at my working /etc/modprobe.d/blacklist.conf file, and I see one difference -- you might be wise to open it again for editing, like you did before, and add this:

"blacklist amd76x_edac" with no quote marks.

After you do that, reboot your machine and choose "Recovery Console", and on the next menu choose "Drop to root prompt". This will leave you at the "#" prompt.

Now, just to be sure, I want you to use the nvidia-xconfig utility to make a new xorg.conf file:

```
nvidia-xconfig
```

then issue

```
service gdm start
```

and it should run the greeter.  Does it let you log in?
