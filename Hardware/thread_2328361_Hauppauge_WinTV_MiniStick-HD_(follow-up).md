### Post by gdu on 2016-06-20
A follow-up on [http://ubuntuforums.org/showthread.php?t=1061151&p=9954005#post9954005](http://ubuntuforums.org/showthread.php?t=1061151&p=9954005#post9954005) in a thread that has been closed.

Thanks to 
[https://bugs.launchpad.net/ubuntu/+source/linux-firmware-nonfree/+bug/581238](https://bugs.launchpad.net/ubuntu/+source/linux-firmware-nonfree/+bug/581238)
[https://bugs.launchpad.net/ubuntu/+source/linux-firmware-nonfree/+bug/667177](https://bugs.launchpad.net/ubuntu/+source/linux-firmware-nonfree/+bug/667177)
 the needed driver is now included in the "linux-firmware-nonfree" package (see for instance [http://packages.ubuntu.com/wily/all/linux-firmware-nonfree/filelist](http://packages.ubuntu.com/wily/all/linux-firmware-nonfree/filelist)).

In addition to the tarball of that package, the firmware can still be found at:
- [http://www.steventoth.net/linux/sms1xxx/sms1xxx-hcw-55xxx-dvbt-02.fw](http://www.steventoth.net/linux/sms1xxx/sms1xxx-hcw-55xxx-dvbt-02.fw)
- [https://www.linuxtv.org/downloads/firmware/sms1xxx-hcw-55xxx-dvbt-02.fw](https://www.linuxtv.org/downloads/firmware/sms1xxx-hcw-55xxx-dvbt-02.fw)

I don't know which file is the most up-to-date, nor if any difference in behaviour can be expected.
It might be possible to extract newer versions from the WinTV executable (at [http://hauppauge.com/site/support/support_wintv8.html](http://hauppauge.com/site/support/support_wintv8.html) for the current version) as described in [http://ubuntuforums.org/showthread.php?t=1061151&page=2&p=12468203#post12468203](http://ubuntuforums.org/showthread.php?t=1061151&page=2&p=12468203#post12468203). I haven't tested this.

Apart from this, the procedure described in the post linked above still works within Ubuntu 15.10 (4.2.0-38):
* Copy sms1xxx-hcw-55xxx-dvbt-02.fw in  /lib/firmware/
* Plug in your device, "dmesg | tail" gives something similar to:
[ 3921.000072] usb 2-2: new high speed USB device using ehci_hcd and address 2
[ 3921.151237] usb 2-2: configuration #1 chosen from 1 choice
[ 3921.211080] usb 2-2: firmware: requesting sms1xxx-hcw-55xxx-dvbt-02.fw
[ 3921.800072] smscore_set_device_mode: firmware download success: sms1xxx-hcw-55xxx-dvbt-02.fw
[ 3921.846109] DVB: registering new adapter (Hauppauge WinTV MiniStick)
[ 3921.847707] DVB: registering adapter 0 frontend 0 (Siano Mobile Digital MDTV Receiver)...
[ 3921.848958] usbcore: registered new interface driver smsusb
* Install Me-TV and run it
* Scan for channels using an initial file corresponding to your location  from /usr/share/dvb/dvb-t/ (Such files can also be generated by w_scan  (w-scan package) as explained in [http://contribs.martymac.org/FreeBSD-siano/README](http://contribs.martymac.org/FreeBSD-siano/README): "w_scan -x > MyCountryCode-MyCityName". One can then scan for channels with "scan MyCountryCode-MyCityName > channels.conf".)
