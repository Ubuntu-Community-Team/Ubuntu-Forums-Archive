### Post by Anders Kvist on 2016-10-21
We run Ubuntu on most of our servers (Dell) and when they have an issue (not very often) we are always asked to hand in a DSET report from the server, but Dell doesn't support DSET on the Debian family and it won't install or run by default :/

So, I've had a task lying around for along time, about getting it to run and when we had some issues with a server last weekend, I picked this up again and managed to create a script that extracts the package from Dell, do some changes (see script for details) and then combine everything into one single package that will install and let you generate a DSET report on a Debian or Ubuntu - everything is a bit crude, I know, but had to start somewhere! I've tried to compare the report from a CentOS server with the one generated by my package and it looks somewhat okay, but I haven't sent a DSET report to Dell yet, so I don't know if it even works enough for them to use it...

I placed it on GitHub so other users in the same situation can use it and/or help improve it. 

[https://github.com/anderskvist/Dell-DSET-deb-creator](https://github.com/anderskvist/Dell-DSET-deb-creator)

Basicly, the problem with DSET on Ubuntu is that it requires /bin/sh to be bash - I think I've found all the places, but it's only been tested a few places yet.

/Anders
