### Post by torpedolos on 2010-09-04
**What would you think of ubuntu selling a line of selected, approved and tested hardware with it's brand on it ?**

**The advantages for the users:**


a)  They would be sure, before buying this computer, that the hardware and  the software fit perfectly together (no questions about the graphical  card, the wifi card, the web cam, etc)

b)   They would get really  better prices than the existing offers, for example with Dell, that  sells it's "ubuntu" computer at the same price as the "microsoft" ones.

c)   They wouldn't be afraid of installing the system themselves.

d) To avoid giving money to Microsoft for a licence that they are not going to use.

e)    To have only one contact both for hardware and software in case of  problems (a big advantage that  only  apple have at the moment).

This is what we can call a "soft migration".



**The advantages for Canonical  :**

a)  Canonical would control  the prices of the hardware sold with it's operating system, which is not the case now 

b)   Canonical would make the profit with both the hardware and the hotline.

c)   Canonical could develop a "ubuntu community spirit", not only with the software, but with an "all-in-one" solution.

d)   Canonical could focus it's effort on the approved, selected hardware that is sold by the company itself.

### Post by srs5694 on 2010-10-06
> **Ed_Ziffel said:**
> (again the mac version of linux has more industry standard software which is hard to discount).

One comment: Mac OS X is *not* based on Linux. It uses the [XNU kernel](http://en.wikipedia.org/wiki/XNU) (Ubuntu and other Linux distributions use the [Linux kernel.](http://en.wikipedia.org/wiki/Linux_kernel)) Atop the kernel, OS X adds a bunch of utilities, mostly from [FreeBSD,](http://www.freebsd.org) IIRC. Some of these utilities are also used with Linux, but these are mostly common to several Unix-like OSes, so their presence doesn't make OS X "the mac version of linux." Atop all the Unix underpinnings, OS X adds a proprietary Apple GUI and utilities.

### Post by srs5694 on 2010-10-06
> **torpedolos said:**
> In other words, Mac and All GNU/linux distros are sons of Unix.
Windows is a son of MS DOS.

Right ?

As a very rough first approximation, yes. The Linux kernel is actually not derived directly from Unix; it's a compatible re-implementation begun by Linus Torvalds in 1991. I'm less familiar with the history of the XNU kernel, but according to the Wikipedia article to which I linked, it includes components from several sources.

For non-kernel components, it's sometimes hard to say whether something's heritage is Unix vs. something else, since the program may have been developed initially on one platform (say, Linux or Solaris) and then ported; or it may have been developed in a cross-platform way to begin with.

Current versions of Windows trace their heritage back to Windows NT, which in turn borrowed some from DOS and OS/2, but was largely new. IIRC, one of NT's chief architects was involved in the development of DEC's VMS. I don't know offhand if NT borrowed any actual VMS code, but my understanding is that it shares some conceptual features with VMS. (I don't know enough about either VMS or Windows internals to say much about this; I've just read that there is a distant relationship.) I also seem to recall that some early Windows networking code was derived from BSD networking code, but I don't know if that applied to NT or just the 9x/Me line, much less whether any of that code still exists in current Windows.
