> **sofasurfer said:**

Why would a single core 2.6 GHz be faster than a dual core 2.4 GHz???
Sure, if you run only one app at the time, and only apps that cannot use both cores, 2.6 is more than 2.4, but if your os is capable of using both cores (which ubuntu is) the dual core is like having TWO 2.4 GHz cpu's (1x2.6<2x2.4).
The os will distribute the load between the cores.
Let's say you run two apps that normally would use 90% each of the 2.6 GHz cpu. If you run both at the same time on a single core cpu, the cpu will go up to 100% usage, and each app will only be able to use 50% of the cpu (=they will only run at ~56% of their normal speed).
With a dual core each app would run on their own core, and even if they would both max out the capacity of their respective cores, they would be able to run (almost) as fast as if you only ran only one of them on a single core.
Some apps are even capable of spreading their own load across several cpu's, so that an app that would max out the single core 2.6 GHz cpu may still run very well on a dual core 2.4 GHz cpu...

You cannot compare a Core 2 processor and a Pentium processor by only looking at the clock speed, because they have different processor architectures. A Core 2 Duo using just one core is faster than a Pentium 4, even when the clock speed of the Core 2 Duo is lower.

It's smarter to buy a Core 2 Duo at 2.66 GHz than a Pentium at 3 GHz; even with just one core, the Core 2 Duo will be faster.

> **dersu said:**

and what about caches? quad with 4 gb of cache has 1 gb each core and a duo with 4 gb 2 gb of each OR all cores share the 4 gb?

If you have a quad core processor at 2 GHz, then all the cores are running at 2 GHz.

Older dual- and quad-core Intel processors (dual-core Pentium D for example) had separate caches for each core, but the newer processors have a shared cache. Note that the cache is in **MB**, not GB. A shared cache is better, because that means the cores can share cached data. There are Core 2 processors with 2, 3, 4 or 6 MB cache.
