Other thoughts :-

1) If you want a quad core system : A quad core system basically means two Dual Core CPU's. A dual core CPU can perform 2 functions simultaniously at it's rated speed while a quad core CPU can perform 4. This would make the system a bit more future proof as future programs will be more multithreaded (i.e they will do more functions at the same time.). This also helps a lot in compiling, video and audio encoding, 3d rendering etc (which I don't think you would be doing. If you want a quad core system buy a Phenom II 940 for $230 [http://www.newegg.com/Product/Product.aspx?Item=N82E16819103471](http://www.newegg.com/Product/Product.aspx?Item=N82E16819103471) and a motherboard for $80 [http://www.newegg.com/Product/Product.aspx?Item=N82E16813128379](http://www.newegg.com/Product/Product.aspx?Item=N82E16813128379) . The quad core system would cost you $560 without the Graphics card and $650 with it. If you want great quad core performance then you might want to wait for Intel lynnfield which comes in Q3 2009 (next gen gfx cards would be availble by then). A good lynnfield system would cost about $750 and would be 50% faster than a Phenom II.

2) Ubuntu jaunty won't need more RAM

3) Whatever you do, think a lot before you buy.  You might develop an interest in somethings in the future(like Blender, Programming etc) so you can consider more things. Don't skim on the PSU and Case because you can retain these things in your next build too.

### Post by zedx555 on 2009-02-24
What are you saying?  I don't think the person who started this thread can or needs to buy a set up with two motherboards and two processors. I don't even think such a setup can be done. Yes you can have one motherboard and two processors but even such a build will take > 2000$.  And I don't think anything except servers would require that much of RAM. You can get a good render time by using a good quad core processor and by telling the renderer to use 8 threads. You can further optimize by building blender by yourself and even further by using something like Gentoo.  If you want to buy for Blender, buy a decent quad core processor and a very good graphics card for modelling. In the future, programs will be able to use the massive processing power present in GPU's using OpenCL. This should result in atleast about  10x improvement in renders using only the GPU against the CPU. Blender should support OpenCL. I don't know if current gen GPU's support OpenCL so if you want to build for Blender I would suggest waiting till Q3 2009 when you will be able to get Intel Lynnfield quad core processors along with next gen GPU's which will definately support OpenCL (or wait till it is announced if current gen GPU's support OpenCL)
