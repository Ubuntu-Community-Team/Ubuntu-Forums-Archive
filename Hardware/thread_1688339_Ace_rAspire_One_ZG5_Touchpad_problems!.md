### Post by Naggobot on 2011-03-20
About a week ago I did a persistent USB installation of Ubuntu 10.04 to be used with Acer Aspire One ZG5. I was relatively happy with what I got but for some reason the touch pad did not work and I was unable to activate it even though OS clearly reacted to keyboard combination. 

It turned out that the touch pad did not work in original OS Linpus either since it had been deactivated from Linpus. I re-activated the touch pad from Linpus OS and now it works from Ubuntu also. 

Evidently the state of touch pad is written/flashed somewhere and Ubuntu does not read/write the state from this hardware specific location.
