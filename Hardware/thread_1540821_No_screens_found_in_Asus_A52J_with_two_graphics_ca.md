### Post by skamarla on 2010-07-29
I have found a workaround. Not too good, though. The BIOS has an option to disallow the discrete (nVidia) card. After doing that, only the integrated Intel card is recognized.

It wasn't straightforward, though, because X kept trying to load nVidia's GLX module. After some fiddling around, I removed the nVidia drivers via Synaptic and that was it.

Before disallowing the nVidia card, I found that with the BusID option inside the Device definition, I could make X distinguish both cards. But then there was some kind of problem loading the drivers. The X error said "Screen(s) found, but none have a usable configuration" and syslog reported "failed to copy vbios to system memory". This looks like an nVidia problem, so I'll have to wait for the fix.

### Post by UncleVader on 2010-08-01
Hello,

I have an Asus U30JC laptop and I have the same exact problem with that.
You didn't mention which Ubuntu distro you're using. But it turns out that the latest one, 10.04, comes with the 
[ Nouveau ]("http://nouveau.freedesktop.org/wiki/") driver as default (for NVidia) and it seems to be able to find the nvidia controller OK. Although I'm not exactly sure how. I don't even see an xorg.conf in /etc/X11.
This driver seems to support 3d acceleration well enough that the Ubuntu desktop effects work fine and it can switch to the native 1366x768 resolution also. And supports an external monitor as well, which i use quite a bit.

I tried to switch to the proprietary NVidia driver and that's when I ran into the same problem you described.

So I don't know if upgrading to 10.04 is an option for you. If not perhaps you can just switch to the Nouveau driver on which ever version you're using already.
