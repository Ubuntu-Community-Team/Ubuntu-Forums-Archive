### Post by sirverdemer on 2012-05-03
So, I've got a Samsung series 9 laptop (on board intel HD 3000 graphics solution), which runs beautifully with ubuntu 11.10. I wanted to get the system ready with a clean install of 12.04, since this is my main computer, and I'm a little tired of upgrading every six months. I just wanted it to stay in a usable configuration for the next two years.
So I did the installation, and first problem: suddenly Ubuntu does not like to play with UEFI (my 11.10 comes from serially upgrading from 10.10, so I don't know if this is a problem that just appeared in 12.04, or before). So I disable UEFI, and proceed to install.
First thing I notice, ubuntu 12.04 live usb is significantly slower without UEFI, I might be stating the obvious, but I just want to get the whole description done.
So, then I install ubuntu 12.04, and everything runs smoothly. Unity 3D is running, everything very pretty bla bla bla.
Ok, so I install coot, which is a molecular modelling program I use for work. It basically displays ball and stick 3D models of molecules, and electron clouds around them. So, a pretty simple OpenGL program. OK, so this program used to run at ~60 FPS, and now it is running at 2FPS!
Basically, it is unusable. I looked up glxinfo, and it appears that the graphics card is not really recognized. Any ideas? What can I do?
This being my main computer, and being specifically for molecular modeling, without proper OpenGL support, ubuntu 12.04 is a no no for me. :(
