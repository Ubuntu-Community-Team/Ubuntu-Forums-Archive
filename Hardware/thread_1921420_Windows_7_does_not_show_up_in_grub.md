### Post by drs305 on 2012-02-06
I'm not a Windows user but the reason Grub isn't finding your Windows OS appears to be because your Windows partition is missing critical boot folders/files which Grub looks for.

Specifically, I believe your W7 should contain */bootmgr /Boot/BCD * and imagine you will have to use a Windows repair disk to resolve this.

You should probably wait for a Windows user to weigh in on how to proceed if it isn't clear.

### Post by drs305 on 2012-02-06
One thing I failed to mention in the previous post was that the boot flag must be on the Windows partition. It's currently on sda2 and should be on sda1.

I believe it's the fixboot option of the repair disc you will want to use, and it may set the boot flag to the proper partition as well. If you end up having to use fixmbr (which I don't think you will need) the MBR will be rewritten with Windows information and you will need to reinstall Grub to get back to Ubuntu.

The best thing to do is get Windows working, whatever it takes to do so. Getting grub back is very easy as long as you have a linux bootable CD/USB.

### Post by Cesaar on 2012-02-07
Success! Thanks drs305 for your time and help :)
Ok, so I tried the repair disk with the fix boot option and it failed saying that the disk was corrupt. So then I went ahead with fixmbr:
```
C:\> bootrec /FixMbr
```
which ran successfully but I still couldn't get windows to start up, (no operating system was recognized at boot).

Then I reran fixmbr along with rebuildbcd and fixboot:
```
C:\> bootrec /FixMbr
C:\> bootrec /RebuildBCD
C:\> bootrec /FixBoot
```
RebuildBCD ran unsuccessfully with > volume does not contain a recognized file system. please make sure that all required file system drivers are loaded and that the volume is not corrupted.
Thus I tried to fix the boot-sector with:
```
C:\> bootsect /nt60 C: /force
```
But still failed to recognize the operating system.

So, then I tried to make my windows partition active:
```
C:\> diskpart
DISKPART> select disk 1
DISKPART> select partition 1
DISKPART> active
DISKPART> exit
```
and redid everything I did above, which now ran successfully. At boot I atleast got the error > BOOTMGR is missing. Now I knew that at least the system was recognizing windows.

Finally, I rerun the repair the disk with the boot fix option and voila, got Windows 7 back. :)

To get Grub back I followed the instructions here:[https://help.ubuntu.com/community/Boot-Repair]("https://help.ubuntu.com/community/Boot-Repair")

Just for reference, this is where I got help to get windows 7 back:
[http://superuser.com/questions/241739/windows-7-cannot-boot-bootrec-reports-fs-not-found-or-corrupt]("http://superuser.com/questions/241739/windows-7-cannot-boot-bootrec-reports-fs-not-found-or-corrupt")
