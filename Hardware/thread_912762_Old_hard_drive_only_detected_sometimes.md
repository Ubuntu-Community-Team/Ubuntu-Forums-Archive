### Post by venator260 on 2008-09-07
I installed Ubuntulite on my parents computer tonight using a hard drive that was taken from their previous box. 

My problem is that the computer is only detecting the hard drive sometimes. I know this because there is a summary screen displayed right after the eMachines splash screen. Most of the time the secondary slave is "none". When I'm lucky, the hard drive appears and I can boot as normal. If not, I get Grub error 21.

Also, for some reason, this computer will not boot from the Grub super disk. Other live CD's work, the Grub disk does not. I can boot from a Hardy live CD, but with 667 Mhz Celeron processor and 128 megs of RAM, I'm not sure what use that would be. 

It may be that the hard drive is simply failing (it's now 9 years old). Is there a way to test for this possibility? And if it's not, is there any other way I can fix this?

### Post by Hellkeepa on 2008-09-07
HELLo!

If the BIOS fails to detect the disk, which I gather is the issue here, then the disk is failing.
Get a replacement disk ASAP, and copy/move the partitions/data from the old & failing disk to the new one. The longer you wait, the more likely it'll be that the old disk fails permanently before you can backup your data.

Happy codin'!

### Post by lukjad on 2008-09-07
A tip that I heard for a old/failing drive:
Once you have the new drive ready to copy the files into, put the old on in a bag and then into the freezer for an hour. It may help. I make no guarantees, and if you can copy the drive without this tip, please do so.
