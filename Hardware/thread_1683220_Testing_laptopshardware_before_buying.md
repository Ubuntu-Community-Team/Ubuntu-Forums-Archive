Hello there,


Last week, I bought an Asus laptop (N53Jf). I chose it because it had nVidia graphic card, so I was sure I would be able to get HD videos running smoothly, and I could get graphic acceleration for desktop and games.

It turned out that the graphic card is an Optimus one, which means the nVidia part does not work at all with any version of Linux. Instead, it is the basic Intel graphic chipset that is used, so the perfs are horrible and I cannot even get it to display anything on my HD TV through HDMI...

I will try to get refund and to pick up another computer, but now I feel I cannot trust any vendors anymore because of all these marketing/technology evolutions (Optimus, Sandy Bridge, etc.) that do not work on Linux.

[B][U]So basically 3 questions:
[/U][/B]
- **How can I fully test a computer to make sure it will work properly?** I spent time in the shop trying Ubuntu with a Live USB stick, but at that time it was just using the open source Intel driver, and I couldn't test HD (HDMI, external output, etc).
- **Are there tools that could be used along with a Live CD/USB to test the system?** (Shell scripts, small pieces of software)
- **What should I pick as graphic card?** Intel is crappy (for gaming/HD rendering) but Open Source, nVidia used to be cool (yet closed source) but with Optimus it's over, and ATI/AMD seems to be buggy and not reliable for HD videos...

Thanks in advance!
