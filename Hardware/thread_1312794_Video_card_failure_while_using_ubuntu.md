### Post by tomveil on 2009-11-03
Brief summary - while using Ubuntu 9.10 my computer had a video card failure (permanent?), cause unknown.
   
  My system - Dell Vostro 1400 (Laptop - two years and one month old)
  V1400 CORE 2 DUO T7250 2.00GHZ, 800, 2M
  DISPLAY : 14.1IN WXGA+ (1440X900)
  GRAPHICS : 128 MB NVIDIA GEFORCE 8400M GS
   
  On Sunday November 1st I installed Ubuntu 9.10 on an external USB hard disk. The install was successful and I used the system for six or seven hours without problem.
  All features worked as expected - wifi, webcam, microphone, etc.
   
  At one point on Sunday evening the computer crashed in the following way -
  the screen had returned to console mode,
  the screen was split in half horizontally - a thin white line divided the top and bottom halves,
  the one console appeared in the upper and lower halves,
  the screen was not the standard black and white of a console - there were very narrow yellow vertical lines
  the text of the console was difficult to read, it's hard to describe but it looked as though the characters on the screen were spread wider than normal.
   
  I rebooted and all was fine.
   
  However, the following day, while watching something on a video sharing site, Ubuntu 9.10 crashed again. This time, the screen held the image of what was playing the video, but the whole screen was flashing whiter and then back to normal, these cycles were occurring a few times a second.
  I rebooted again (by powering off and on) and immediately when the screen came on to show the Dell logo and the usual Setup/Boot options, the screen was split in two like I described above, horizontal splitting the screen, the same image repeated in both, discoloration in both halves of the screen.
  I powered off and on again, this time all was back to normal at the Dell BIOS screen. I booted off the internal hard disk this time, Windows Vista, when it started the screen was corrupted - complete white except for two vertical bands of colour running the up and down the screen.
  At this point I gave the computer some rest.
   
  About 30 minutes later I turned the laptop back on, the Dell BIOS screen was fine. This time I booted into from another external USB hard disk which contained Ubuntu 9.04, it loaded and worked fine for about 5 minutes, then it crashed with its screen flashing between bright and normal a few times a second.
   
  I have tried rebooting a few more times, but the screen has been corrupted in a split screen manner each time. The discoloration is not always the same colour.
   
  There is another interesting point I should mention.
  When shutting down or rebooting Ubuntu 9.04 (which I have used since its release), it would return to console mode for maybe 10 seconds and this console more would split the screen horizontally with the same console in both the upper and lower halves of the screen.
   
  Can you please suggest a cause or even a solution?
  I am outside warranty.
   
  Best regards,


Tom
