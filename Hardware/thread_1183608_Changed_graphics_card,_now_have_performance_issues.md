I'm using Jaunty and I recently upgraded my motherboard.  My old motherboard had onboard nVidia graphics; my new one has onboard Intel graphics.  Since the hardware upgrade, Compiz no longer works.  Video and everything else seem to work fine.

I've found plenty of information about how Compiz and Intel don't seem to be playing nicely in Jaunty, and I tried downgrading my Intel driver, which only made things worse - my resolution went down to 800x600 and System->Preferences->Display wouldn't even open.  (Trying to run Compiz via the terminal gave me the same error as I had with the up-to-date driver.)  It gave me an error when I tried to open System->Preferences->Display, saying that there was a driver problem and asking if I'd like to use the vendor's driver instead.  When I clicked Yes, it brought up an nVidia config screen and - surprise - told me that there was no nVidia driver installed.  Why does it still think I want nVidia at all?  Clearly there's some residual config problem, but I have no idea where it is or how to fix it.  I've run ** sudo dpkg-reconfigure xserver-xorg** any number of times with no improvement.

I've reverted back to the current Intel driver, so my resolution is back and I can get into System->Preferences->Display, but Compiz still doesn't work.
