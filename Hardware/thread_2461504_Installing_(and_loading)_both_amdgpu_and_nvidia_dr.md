I have a motherboard that allows for up to 19 video cards to be installed on it, and currently I have 4 nvidia and 2 amd cards installed. I can get the nvidia cards to run fine, but I cannot get the amdgpu drivers to load, or even tell if they were installed correctly. I am running Ubuntu 20.04.2, 5.4.0-42-generic kernel. Following directions I found on another thread I uninstalled the nvidia drivers, grabbed the Radeon 20.45 installer, and ran it using this command:

```
./amdgpu-install --opencl=legacy,pal --headless --no-dkms
```

I then rebooted, installed the nvidia drivers again, ran the xorg generation command:

```
sudo nvidia-xconfig -a --cool-bits=28 --allow-empty-initial-configuration
```

and rebooted again. The motherboard has a utility that shows which pcie slots are in use when you boot up, and they are all showing green (no errors). I can see all of the cards when I run *lspci*:
