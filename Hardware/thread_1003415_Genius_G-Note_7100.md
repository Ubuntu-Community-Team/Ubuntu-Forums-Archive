### Post by XenonL on 2008-12-06
This might be important for people who use Genius G-Note 7100.
The digitizer creates files it some closed  binary format (*.dnt) and the original processing software is Windows-only.

In the earlier thread ([http://ubuntuforums.org/showthread.php?t=452281](http://ubuntuforums.org/showthread.php?t=452281)) there was a discussion about close formats and a perl script for convertion G-Note 7000 files to SVG have been posted by ksh1.

I played a bit with hex editor and this perl script and managed to partially discover internal structure of DNT files and convert them to SVG using modified perl script.

About file format and changes in the script.
Header is now 64 bytes long, not 32. Haven't dig into internals.
After that, there's an array of 8 byte records (not 6 bytes as it was for G-Note 7000), and the layout is following:

offset length meaning
0x00 1byte  pen type (0xE0=pen up, 0xA1=black pen, 0xA5=red pen)
0x01 2bytes x coordinate 
0x03 2bytes y coordinate (actually 0x7FFF-y)
0x04 2bytes probably pen pressure, not sure about that
0x05 1byte  always 0x00

If first byte is 0xE0 all other are zeroed.

I assumed size of long side of the paper equal to 0x10000 and size of short side 0x10000/sqrt(2) from side ratio of A4 paper.

The picture generated by modified script, however, flipped upside down, so it might be better to leave 'y' unchanged and invert 'x'. or center and flip after convertion.

One problem is still not solved. While the original software delivers smooth lines, direct convertion produces a bit rough and edgy lines. Some advanced processing is still necessary.

Here's modified perl script. I have no perl knowledge so only basic contour reading is implemented by guessing perl syntax as PHP/C-like syntax ;).

```

#!/usr/bin/perl
# Converts Genius G-note 7100 file to SVG file
#
open(FILE,$ARGV[0]); # filename
binmode(FILE); # it is a binary file
read(FILE,$header,64); # ignore header
$pen_up = 1;
#print .svg header
print "<?xml version=\"1.0\" standalone=\"no\"?>
<!DOCTYPE svg PUBLIC \"-//W3C//DTD SVG 1.1//EN\"
\"http://www.w3.org/Graphics/SVG/1.1/DTD/svg11.dtd\">
<svg width=\"20cm\" height=\"30cm\"
viewBox=\"0 0 23170 32767\" version=\"1.1\"
xmlns=\"http://www.w3.org/2000/svg\">\n";
print "<path d=\"";

$pen_up = 1;
$pp = 0;

while(read(FILE,$point,8)) {
@byte = unpack ("C*",$point);
$y = $byte[1] + $byte[2] * 256;
$x = $byte[3] + $byte[4] * 256;
$p = $byte[0];

if($p==224)
{
$pen_up=1;
}

if($pp!=$p)
{
$pen_up=1;
}

if($pen_up==1)
{
printf("M %d %d ",$x,32767-$y);
$pen_up=0;
}
else
{
printf("L %d %d \n",$x,32767-$y);
}

$pp=$p

}
print "\"\n fill=\"none\" stroke=\"black\" stroke-width=\"12\" />\n</svg>\n";

```

### Post by Martin Prazak on 2010-01-31
Not sure if anybody reads this topic, but just in case...

The data in the previous post are not exactly accurate - the byte-wise description is correct, but the computation of the resulting coordinate is not right as the file format uses only 7 bits in each byte (and a bit in 6th byte of the record). This is the reason for the "rough and edgy" lines produced by the original code...

I am not really good in Perl and I wanted a "conversion tool" that would be associable in the GUI to the DNT files, so I ended up with the following C++ program. It reads the DNT and outputs the SVG, with the problems of previous script solved. Hope this helps somebody - would appreciate if you'd send me an email if you find it useful or useless... :)

```

#include <iostream>
#include <fstream>
#include <string>

#define SIZING 0.0254
#define XSIZE 21600.0
#define YSIZE 16800.0

#define WIDTH_CM 30.0

using namespace std;

int main(int argc, char * argv[]) {
  if(argc == 1) {
    cout << "Usage: dnt2svg <source.dnt> [<target.svg>]" << endl;
    exit(1);
  }
  
  try {
    // open the file
    ifstream file(argv[1],ios_base::in | ios_base::binary);
    
    // reopen the output - either target, or automatically created filename
    if(argc > 2) {
      FILE* stdof = freopen (argv[2], "w", stdout);
      assert(stdof);
    }
    // COMMENT ME OUT IF YOU WANT NORMAL "UNIX" BEHAVIOUR
    else {
      string output_file = string(argv[1]).substr(0,string(argv[1]).rfind(".")) + ".svg";
      FILE* stdof = freopen (output_file.c_str(), "w", stdout);
      assert(stdof);
    }
    
    // read the header
    char header[64];
    file.read(header,64);

    // output the svg header
    cout << "<?xml version=\"1.0\" standalone=\"no\"?>" << endl;
    cout << "<!DOCTYPE svg PUBLIC \"-//W3C//DTD SVG 1.1//EN\" " << endl;
    cout << "\"http://www.w3.org/Graphics/SVG/1.1/DTD/svg11.dtd\">" << endl << endl;

    cout << "<svg width=\"" << WIDTH_CM << "cm\" height=\"" << WIDTH_CM/XSIZE*YSIZE << "cm\" viewBox=\"0 0 " << (XSIZE-1) * SIZING << " " << (YSIZE-1) * SIZING << "\" version=\"1.1\"" << endl;
    cout << "xmlns=\"http://www.w3.org/2000/svg\">\"" << endl << endl;
    
    // read the content
    char record[8];
    bool down = false;
    while(true) {
      file.read((char*)record, 8);
      if(file.eof())
        break;
        
      if(((unsigned char)record[0] == 0xe0) && down) {
        // end of the path
        down = false;
        cout << "\" style=\"fill:none;stroke:black;stroke-width:0.5\"/>" << endl;
      }
      else {
        // the "magic" conversion
        float x = record[1] + record[2] * 128 + (record[6] & 0x01) * 128 * 128;
        float y = record[3] + record[4] * 128 + ((record[6] & 0x04) >> 2) * 128 * 128;
        
        // output the path
        if(!down) {
          cout << "<path d=\"M" << x*SIZING << " " << y*SIZING << " ";
          down = true;
        }
        else
          cout << "L" << x*SIZING << " " << y*SIZING << " ";
      }
    }
    
    cout << "</svg>";
    
    // close the file
    file.close();
  }
  catch (...) {
    cerr << "I/O error" << endl;
    exit(1);
  }
}

```
