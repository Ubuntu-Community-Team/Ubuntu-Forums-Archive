### Post by phlack on 2010-01-19
Look I bought this asus board as soon as I could. The whole reason I bought it was to run unr on it. I had fits with it. At first, I had to install the drivers from Nvidia's site. Troublesome at best, and the documentation on it seems to be sort of spread out. 

9.04 was horrible for several reasons. The default video drivers were really buggy on this board. Had to perform several tweaks to get the lag out. First though was the driver issue. So, It was frustrating in general. Then, I forgot to disable certain updates and the installation broke. . . nice. I prefer to install software through the package manager because of that sort of thing, but the drivers are lacking on 9.04. So, I install 9.10. Much better, default video drivers were still cruddy, and netbook launcher ran at 100% cpu. But, there was the 185 driver with good ion support, as in it didn't crash on reboot. That was my experience with the 180 driver available on 9.04. 

Once I got the driver installed, and a tweak to netbook launcher, It was looking good except for internet connectivity. I was having to reboot the box every 30 minutes or so. I had turned off the power saving features, but no bueno. It was a bizarre problem. After playing quake live, I would leave a game, and i wouldn't be able to reload the page. There were errors everywhere on quakelive. If I tried to reload, I would be directed to a subfolder of google like google.com/#quakelive.com. Friggin bizarre. It turned out any site that was using google analytics did this. I would be directed to a 404 page because naturally that file doesn't exist. Sometimes I could go to a site that didn't use google analytics, usually, I had to reboot. But, finally about a month ago, I did that dreaded thing. I ran updates. And It's incredible. Everything hums.

Thank you!
