### Post by bodhi.zazen on 2007-03-08
Dual booting is helpful.

Anther trick is to use virtualization in windows.

For example you can run DSL on qemu in windows. Yea, it is slow, but it is a full linux environment.

[http://www.damnsmalllinux.org/](http://www.damnsmalllinux.org/)

navigate to the downloads page, download the "embeded" version (dsl-3.2-embedded.zip for example) ... Unzip the file and viola DSL in Windows :)

In terms of transitioning to Ubuntu, the "cold turkey" approach if fastest, but unnerving ...

The dual boot is slower, and you should consider defaulting to Linux, only Booting to windows when needed rather then booting to Windows and running Ubuntu in your "free time".

I think the two most common reasons for dual booting in the long term are :

[list=number][*]Games.[*]Development. Sometimes you want to test your creation in a windows environment[/list]

Another solution is virtal box / vmware running Windows within Ubuntu ...
