There are no "flag states" involved. The 'mv' command is merely expecting that the user supply a list of the names of the files/folders to be moved. 

> **oldefoxx said:**
> Here are a few examples of those flags applied to the end of the /home reference:: 

/home
/home/
/home/*
/home/.*

These are merely lists of file/folder names.

**/home**   - the list contains the name of a single folder
**/home/ ** - the list contains the name of a single folder. The trailing slash doesn't really matter
**/home/*** - the list contains the names of all non-hidden files and folder in /home
**/home/.*** - the list contains the names of all hidden files and folders in /home, PLUS /home itself ("." is a synonym for the name of the directory itself), PLUS the names of all the files and folders of /home's parent directory (".." is a synonym for the parent directory) 
**/home/.??* **- the list contains the names of all hidden files and folders in /home that are longer than two characters. Presumably this is being done to exclude the parent directory; instead I would recommend using /home/.[^.]* 

Again, the "special characters" are not flag states, they are part of the filename specification and specify the wildcard pattern matching that expands to a list of the names of files and folders being specified. The same "wildcard expansion" is performed for all file/folder specifications provided on the command line, regardless of what particular command is being executed.

> **oldefoxx said:**

The wildcard expansion that takes place is the same regardless of the command involved. It always follows the same rules ([reference for BASH]("http://www.gnu.org/software/bash/manual/bashref.html#Shell-Expansions")). 

The 'mv' command is rather simple:[list]mv namelist target[/list]
"namelist" is the name(s) of one or more files/folders to be moved and "target" is the file or folder to which the source file(s)/folder(s) are to be moved. "target" is always the last name specified and it is always a single name. If "namelist" contains more than one file/folder name then the "target" needs to be a directory name. 

If "target" does not exist then the source file or folder will be renamed using the name specified by "target" (and moved if in a different directory). If the "target" exists and is a file, it will be overwritten by the source file (but not a source directory). If "target" exists and is a directory, the source files/folders will be moved into that directory.

This may seem complicated but it all follows quite logically from the simple "mv namelist target" command template: it makes no sense to move two files into one file, yet it makes perfect sense to move two (or more) files into a directory. If the target directory already exists, it makes perfect sense to assume that the sources should be placed into that directory (regardless of how many sources or whether they are files or folders).
