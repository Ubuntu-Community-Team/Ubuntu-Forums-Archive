### Post by rg4w on 2012-01-31
> **haqking said:**
> Ubuntu is released in Alpha and Betas.

12.04 right now is in alpha
The user testing of design I was referring to is very different from the functional testing in the Alpha and Beta process.

Design testing should ideally happen long before the first line of code is written, using prototypes that demonstrate different ways to solve a given task and then measuring the effectiveness of each design with pools of users who represent the target audience.

A/B prototyping refers to having at least two different prototypes to test, to be able to compare relative effectiveness in ways that testing a single design can't uncover.

For initiatives as sweeping as Unity, early-stage prototyping can be done on paper:

[http://www.uie.com/articles/paper_prototyping/](http://www.uie.com/articles/paper_prototyping/)
[http://www.uie.com/articles/prototyping_risk/](http://www.uie.com/articles/prototyping_risk/)
[http://www.useit.com/alertbox/20030414.html](http://www.useit.com/alertbox/20030414.html)

The results of such very-early-stage paper prototype testing can lead to functional prototypes made with high-level RAD tools like LiveCode, RealBASIC, or Quickly.

To achieve time-to-market goals, core-level code can usually be written while the various UI options are being tested.

But writing UI code before doing UI testing too often leads to workflows in which it becomes undesirable or even prohibitive to change direction, even if test data strongly suggests a change of direction would benefit the work.
