### Post by samjh on 2009-10-24
> **hoppipolla said:**
> I think it will by 4.5, at the moment I love it to bits and personally prefer it, but I reckon it still needs a hint more polish to complete it's fantastically friendly, great looking and totally customizable desktop :)

What do you think amigos? ^_^

I used Windows 7 RC for approximately two months.  In comparison to Gnome 2.28 and KDE 4.3, Windows 7 offers slightly better user experience.

Windows 7 is -- in my system -- smoother and makes more efficient use of interactive space.  For example, the task bar allows very easily-distinguished grouping and previewing of minimised windows, which I used and liked a lot.  Currently only KDE 4.3 offers something similar, but it is not yet polished or efficient as Windows 7.  Custom colourisation and desktop settings are vastly superior in terms of default settings and options than either Gnome 2.28 or KDE 4.3.  The responsiveness of the interface is very impressive, even with composition and effects enabled.

The negative aspects of my Windows 7 experience is as follows: the default control panel applet is poorly organised and unintuitive, and the system uses way too much memory.  The control panel's default grouping makes it difficult for a user to identify where they need to go to change certain settings.  It might have been better for MS to stick to Windows XP's classic control panel, than to try arbitrarily grouping various configuration applets together.  As for memory footprint, both Gnome with Compiz, and KDE with KWin, use up to 150MB less memory for comparable composition and effects.  Although the responsiveness of the interface is good, the memory footprint is ghastly.

Onto the OP's question: will KDE 4.x provide a superior desktop to Window 7?  I think it will.  Possibly by 4.4, almost certainly by 4.5.

Right now, KDE 4.3 is very stable.  I've used it since it was released, and it is a great improvement over 4.2 in terms of speed, resource consumption, stability, and configuration options.  This might be partly down to me switching from Ubuntu to Arch, but even so, the changes astonished me.  KWin is very stable, while offering effects which are pretty without getting in the user's way.  Also it uses less memory than Compiz and Emerald on Gnome, on my system.  The smoothness and responsiveness of the graphical effects have improved a lot from 4.2.  It's actually very smooth now, with little to no stuttering.  The desktop loads fast too -- within just three seconds from KDM on my system.  Very impressive, considering that Gnome usually takes longer even without Compiz and Emerald.  KDE's Plasma widgets have stabilised a lot since 4.2.  Even in 4.2, the system tray used to crash on me occasionally, and the device notifier often didn't work.  In 4.3, I've suffered no problems with Plasma at all.

These are the features I'd like to see in KDE for it to surpass Windows 7 by a decisive margin:
[list]
[*]Graphical root user access to KDE System Settings.  At the moment, you need to run it via *kdesu* in the terminal to gain root access.
[*]Better grouping and previewing features on the task list widget.
[*]Notifications which close after a reasonable time-out period (eg. 10 seconds).  At the moment, notifications stay visible forever until closed by the user.
[*]Strigi search and Nepomuk need to be better optimised.  I haven't had problems with them, but many users report disproportionate memory usage when they're enabled.
[*]KDE Network manager needs to improve to at least Gnome's capability.
[*]The KDE Menu needs to be reworked.  Make the vertical height adjustable by the user; redesign it so that it uses conventional menu interaction model like KDE3/Vista/Win7/Gnome.  A two-part menu is fine, with favourite and/or commonly-used items on the left, and the menu itself on the right, like Vista/Win7: there is no shame in adopting a good idea.  Menus should present as much as possible to the user while minimising clutter; this is very hard to do, but I think Vista/Win7 have hit the jackpot with a compromised design which requires minimal number of clicks and mouse travel for a user to access anything in it.[/list]

That is all for now.  I really want both KDE and Gnome to exceed Windows' user experience.  I think they're both almost there, but need more polish.  They don't need to make giant leaps like Gnome-Shell, just small tweaks.  The desktop environments already work well, so it's only a matter of working on finer details.

Windows 7 is -- in my system -- smoother and makes more efficient use of interactive space.  For example, the task bar allows very easily-distinguished grouping and previewing of minimised windows, which I used and liked a lot.  Currently only KDE 4.3 offers something similar, but it is not yet polished or efficient as Windows 7.  Custom colourisation and desktop settings are vastly superior in terms of default settings and options than either Gnome 2.28 or KDE 4.3.  The responsiveness of the interface is very impressive, even with composition and effects enabled.

The negative aspects of my Windows 7 experience is as follows: the default control panel applet is poorly organised and unintuitive, and the system uses way too much memory.  The control panel's default grouping makes it difficult for a user to identify where they need to go to change certain settings.  It might have been better for MS to stick to Windows XP's classic control panel, than to try arbitrarily grouping various configuration applets together.  As for memory footprint, both Gnome with Compiz, and KDE with KWin, use up to 150MB less memory for comparable composition and effects.  Although the responsiveness of the interface is good, the memory footprint is ghastly.

Onto the OP's question: will KDE 4.x provide a superior desktop to Window 7?  I think it will.  Possibly by 4.4, almost certainly by 4.5.

Right now, KDE 4.3 is very stable.  I've used it since it was released, and it is a great improvement over 4.2 in terms of speed, resource consumption, stability, and configuration options.  This might be partly down to me switching from Ubuntu to Arch, but even so, the changes astonished me.  KWin is very stable, while offering effects which are pretty without getting in the user's way.  Also it uses less memory than Compiz and Emerald on Gnome, on my system.  The smoothness and responsiveness of the graphical effects have improved a lot from 4.2.  It's actually very smooth now, with little to no stuttering.  The desktop loads fast too -- within just three seconds from KDM on my system.  Very impressive, considering that Gnome usually takes longer even without Compiz and Emerald.  KDE's Plasma widgets have stabilised a lot since 4.2.  Even in 4.2, the system tray used to crash on me occasionally, and the device notifier often didn't work.  In 4.3, I've suffered no problems with Plasma at all.

These are the features I'd like to see in KDE for it to surpass Windows 7 by a decisive margin:
[list]
[*]Graphical root user access to KDE System Settings.  At the moment, you need to run it via *kdesu* in the terminal to gain root access.
[*]Better grouping and previewing features on the task list widget.

I use this script to quickly toggle it on or off:
~/bin/toggleCompositing.sh
```
#!/bin/bash
RESULT=`qdbus org.kde.kwin /KWin org.kde.KWin.compositingActive`

if [ "$RESULT" = "true" ]
then
  # turn off
  kwriteconfig --file kwinrc --group Compositing --key Enabled false
else
  # turn on
  kwriteconfig --file kwinrc --group Compositing --key Enabled true
fi

# Then restart kwin (in Alt+F2) with:
kwin --replace &

```
