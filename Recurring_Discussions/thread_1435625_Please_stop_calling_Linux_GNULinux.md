It is possible to replace the Linux kernel with a BSD kernel and run the GNU userland tools on top of it. You can also switch back and forth between the two kernels via a boot loader. At the same, time, you can also switch back and forth between the GNU userland tools and embedded systems userland tools (e.g. OpenWRT) via a boot loader. When the OS runs the Linux kernel, it should be called Linux. When it runs the BSD kernel, it should be called BSD. When it runs KDE, it should not be called KDE/Linux (or should it?). The same goes for when the GNU userland and Linux kernel run together. It should be called Linux.

### Post by Shining Arcanine on 2010-03-21
> **derekeverett said:**
> Somebody else might have come up with it, that's true.

That's why I said "Linux as we know it..." ;)

Linux might have been better off without GNU. If GNU never existed, Gnome would not exist. If Gnome did not exist, all resources devoted to Gnome development would have shifted to KDE. The same goes with GTK versus Qt.

There is also GCC, which tends to generate inferior assembly code than vendor designed compilers and its widespread use has lead some open source programs to be written around its language extensions, in violation of various programming language standards. This is precisely why you cannot compile an entire Linux distribution with the Intel C/C++ Compiler today, as some components are GCC-dependent. I believe Intel published a paper showing that the GCC dependency (in the Linux kernel, which I believe has since been patched) involved reliance on quirky compiler behavior that some times works in GCC and other times does not work, which breaks ICC because ICC does things the correct way.

Linux probably would have been better off without a few of GNU's contributions to open source. :popcorn:
