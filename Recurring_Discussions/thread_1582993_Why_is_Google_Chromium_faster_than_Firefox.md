### Post by gradinaruvasile on 2010-09-27
Chrome is faster because its faster javascript engine and because it uses Gtk directly whereas FF has another rendering layer (xulrunner).
And because it is simpler by design.

I found Chromium more stable on old computers with low (such as 512 MB) RAM. Despite Chromiums tendency to use more memory because of its threaded design it was rock stable on 512 MB RAM even with 50+ tabs open (record was around 100 !). This is real life, not some benchmarks. Firefox in the same conditions started crashing frequently from 10 tabs onwards.

This being said, try Opera - it is as fast as Chrome IRL + smokes both FF and Chrome as far as built in features go (built in ad blocker, flashblock, full featured mailer client, optional traffic compression for slower connections, bookmark/notes sync, built in webserver, web developer tools and so on).

LOL, I tend to agree, although it is nice to have some kind of benchmark.

Also, beware the benchmarks, some are written to show off one or another feature and may not be truly representative of "speed". S[peed is also somewhat subjective, as noted by Simian Man .

For example, [This benchmark](http://www.speedtest.net/) is about the same across all browsers.

The main reason is Chromium is newer code and not as mature as firefox.

Chrome / Chromium does not yet have anywhere near the features of firefox. More features == more code and almost always slower.

Not to mention extensions and optimizations (about:config is much more feature rich then Chrome's options).

So chrome is good at what it does, but it has some major feature holes. If you can live without the missing holes and missing extensions Chrome may be the way to go.

I have also noticed the tendency for Chromium to slow down over time and especially with more then one or two tabs open. Chromium indeed starts out fast, but once I get over 50 tabs open for some time it ends up slower then firefox.

In time Chromium is going to give other browsers a run for the money, but, IMO, it is not quite there yet (although much closer then a year ago).

If you like Chrome, use it.
