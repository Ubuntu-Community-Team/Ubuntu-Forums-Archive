### Post by pi3.1415926535... on 2011-04-03
The main reason for diversity in Linux is that each person thinks that their application is better, and since a Linux distro is a collection of applications, this means that in order for their "superior" application to become default, they will create a new operating system. Take for example package managers, each claiming superiority, though, as I see it, as a user, and not a developer, they all have very similar operations, most commands are the same, just switching the name of the application (sudo apt-get install, sudo yum install, sudo pisi install etc.) The unification, might be beneficial, because there would be less wasted work, though there might be some problems including that of having potentially inferior components, because there would be no competition. Would all of the developers for say, text editors, be able to work on the same project? They would all decide that they should start from an existing application, though which would it be? Emacs, gEdit, KWrite etc. This is just my personal view, I may be very mistaken.
