### Post by Roasted on 2010-04-01
Meep3D nailed it dead on.

I think some people look at the fact that Ubuntu is the most popular, as a result, the most popular distro seems to attract the most new users. Look at Fedora's user base, or CentOS's user base. Those users probably started on a different Linux distro prior. They more than likely didn't jump ship from Windows XP and migrate over to Fedora just like that. Ubuntu, however, seems to attract a lot of Mac/Windows users moreso.

But at the end of the day, I need to use the right tool to get my job done, and Ubuntu can provide that to me with the least amount of fuss possible. I've tried other distros, but each time I come back to Ubuntu. The forum base is amazing and I like Ubuntu because I STILL have the terminal and I still have the advanced way of doing things, OR a lot of those tasks can also be done in the GUI as well.

I will be honest, though. I used Fedora 12 for a short time, and besides the obvious differences (RPM vs DEB, etc) I didn't really see how Fedora was that much more difficult than Ubuntu, at all. In fact, it seemed very similar. Maybe if you're somebody who's really into building all packages from source and whatnot you'd be more into other harder to use distros, but like I said, I didn't see how Fedora was any harder and yet I've heard those comments made that Fedora is a bit more tech-oriented than Ubuntu. But hey - I just gotta get my job done. That's why I use Ubuntu.

But hey, no hate towards the other distros. No matter what Linux distro you use, they're all siblings of one another, and it's important that we recognize that and embrace the Linux/Open Source movement... together... as a whole... instead of getting into arguments over Ubuntu vs Mint vs Fedora vs Gentoo vs Arch. It just doesn't matter. Use what works and have fun.

And I'd like to re-iterate this comment, because it is so ungodly true... Anybody who claims they are an advanced Linux user and claims that Ubuntu is for noobs is simply a newbie themselves trying to show off with an ego driven power-trip that they know something the other person doesn't. Use what works for you. There's no wrong answer. That's the moral of the story.

### Post by agnes on 2010-04-01
> Why many people say that Ubuntu is for linux "noobs"?

You CAN configure it much, but by default it doesn't give you a lot of choices, compared to some other distro's.

1. DE:
When you install Ubuntu you can't choose which DE or WM you want.
Yes you can install KDE afterwards, but it's not clear how to remove GNOME afterwards AFAIK.

2. Package install:
Some packages, it says, can't be removed, because they depend on "ubuntu-desktop" - which you can remove, but then the upgrade won't work - it's like they say you need that package, as if you don't know for sure what you need.

Also when you install a package it installs "recommended" dependencies *that are not essentially needed* (My Ubuntu laptop is sent to repair now therefore can't come up with an exact example, except probably Pidgin which "needs" SSL support for MSN, as if  everybody uses Pidgin with MSN; and VLC which co-installs codecs support) - it seems, it installs the extra packages to avoid people not being able to use everything even remotely related to the installed program, but experiences users may know better what they exactly want.

In the same manner, during install, you can't select which packages/package bundles you don't want to install (e.g. Games; bluetooth support; infrared/remote-control support; and, of course, Pulseaudio). 

3. Too simple included programs:
Some default programs seem to disguise what's exactly going on behind the GUI, most notably Computer Janitor, which doesn't give much control and information on what you remove. The GNOME Network Manager is pretty basic, and there's no default more sophisticated alternative.

4. Some more "pro" tools not included by default:
VIM and gcc are not installed by default (contrary to the Games; Empathy, etc). No big deal, but it gives a certain image to people.

> Anybody who claims they are an advanced Linux user and claims that Ubuntu is for noobs is simply a newbie themselves trying to show off with an ego driven power-trip that they know something the other person doesn't.
Well depends if they use the word "noobs" in a disdainful manner.
I can imagine some people, calling somebody a "noob", in a friendly/neutral way; although "newbie" sounds nicer for sure.
