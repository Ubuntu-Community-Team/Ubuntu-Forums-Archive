Ubuntu installations are incredibly easy, anyone can do it.  If you can click "install", choose a username, and make a password, you're overqualified.

### Post by Potters Son on 2009-07-20
I'd like to point out that Wine DOES NOT RUN MAC OSX SOFTWARE. I mention this because the OP said, "To be able to run a windows or mac software program on it would be a great bonus but not essential." To my knowledge, there is no way to run Mac software under Linux (and I don't think there will be until Mac is a prevalent as Windows is now).

If you want to run Mac and Windows programs on the same OS without emulation, then you may want to try Darwine, which is Wine for Mac. If you're willing to fork over a little money, you might also want to try Crossover Mac, which is essentially the same thing, only with commercial support (and thus more chances that the program you want will run). However, using the Wine compatibility layer is by no means a guarantee that your programs will run.

Since you said that OpenOffice.org is sufficient for your needs, I'm guessing that, for you, Ubuntu may be worth experimenting with. Download and burn a LiveCD, pop it in the computer you want to try Ubuntu on, and see how you like it.

Just remember that Ubuntu is NOT Windows or Mac, so:
1) There is no customer support rep to yell at when something don't work, a likely prospect
2) Software is primarily downloaded and installed from repositories instead of websites (a much saner way of installing, IMO)

Good luck!

### Post by 1clue on 2009-07-21
AFAIK, there is no really good way to run mac software on Linux.  I haven't really tried, but the Windows versions of most of the suggested alternatives stink, IMO.

The only really reliable way to run Windows apps on Linux IMO is with a commercially built emulator such as VMware.  Some of the versions are free of charge, but you don't get the source code.  From the sounds of it, the OP does not care about that.

The down side of the VMware approach is that you are legally required to own a license for all software installed on each image.  For example, if you have 4 VMs with 3 different flavors of Windows, and each of those images has Microsoft Office, then you need a license for 4 copies of Windows (one for each VM) and 4 licenses for office.  Those licenses may not legally be used for any other VM or for any other real hardware.

In other words, you save only the cost of hardware and the space on your desk.

That said, a VMware installation runs software very well, unless you are using graphics-intensive stuff like games, or disk-intensive stuff like databases.  In each of those cases, the VM suffers from the additional layers of OS under it.

Several options have been mentioned before:  Aside of buying a mac and a Windows box in addition to Linux, the best option is to get software for linux which does what you need to do.

Ubuntu is much more mainstream than any other distribution I've used.  The fonts are better than Windows fonts, the tools and controls are pretty good, and the biggest problem I have is that I cut my teeth on Linux 10 years ago and still look for the configuration file first.  Some of the things I learned are obsolete now, and once I figure out the modern mechanism I am generally happier.
