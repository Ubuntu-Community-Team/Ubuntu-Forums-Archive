### Post by ryaxnb on 2009-01-23
**Simplicity**
There are two trends lately in Linuxdom that I have reservations on. "Simplicity" as in keeping to UNIX traditions is increasingly popular, and distros like Arch, Slack, and CRUX seem to be seeing renewed interested due to this trend. I heartily disagree with the "Simplicity" trend.
I would rather work with my computer than smirk at how "simple" and "clean" the system is. Some UNIX-ness is appreciated, when it's practical: e.g. using vim is fast and easy, and is a great way to edin it text. LaTeX is wonderful for long documents and book writing, and the Linux filesystem is admirable. But making it simple just to be simple doesn't make sense to me. There should be an easy, intuitive way to do things, preferably a front-end to an XML or .conf simple way. PlayOnLinux is a perfect example of a front-end to a complex back-end, and CrossOver is a perfect example of a integrating frontend (one that somewhat integrates with the backend.)

Working with Linux should be productive and allow me to get on with what i'm doing. When I need to configure, Linux should help me in all manners possible get the task done easily, but quickly. Memorization of every possible customization and troubleshooting technique is nigh-impossible, so Linux should present them as choices.
**Perfect Linux**
My idea for a perfect Linux is one the Mark Canonical envisions: easy to use, "simple" but intuitive, and full of choices but making the best by default. Ubuntu, SuSE and Fedora, as well as Mandriva and MEPIS and Sidux, seem to follow this conviction. This means having GUIs for configuration.

**Choices**
On Choices, having more choice and freedom is great. But standardize the backends and make choice transparent to the developer. And pick the default for the end user instead of FORCING them to make a choice; rather, simply allow them to make the choice. Ubuntu does this brilliantly, by having a default choice of GNOME, OOo and GIMP but allowing total choice. 

Some other Linuxes insist that you pick a GNOME/KDE choice at first install. This is unwise, because it splits the userbase too much and makes installation difficult to a new user. It FORCES the choices on a new user. This is bad. SuSE does this, as does Mandriva. 

**More Configuration**
Configuration is hard in Ubuntu in some ways, still. For one, KDE and GNOME have very different configuration environments. And if you don't use a DE, good luck configuring via the GUI! Finally, some tasks such as X configuration and Sound configuration are very weakly handled. And some Configuration tools, like CCSM and Gndismanager, are not included.
Thus, I prefer overall configuration environments, like Yast or Mandriva Control Center. I wish ubuntu had one.


Overall, I just don't see the point to simplicity, and do see the point to choice, if moderated.

### Post by lykwydchykyn on 2009-01-25
I'm going back six years to when I started tinkering with Linux...  back then the distro EVERYBODY started out with was RedHat.  I started on RH9, the last official desktop release before Fedora was spun off.  It had a graphical install, graphical config tools, nice backgrounds, pretty colors.  There was also Mandrake, that was uber-user-friendly for the times.  Then there was Debian.  Installing Debian Woody was a nightmare for a new linux user like me -- an hour of answering cryptic questions in a text console, only to find that X didn't load because I forgot that my mouse was not USB (I get fuddled like that).  Back then Debian was the "hacker Linux", along with slackware.

Someone of ryaxnb's opinion back then would have said Debian was a waste of time, that devs should all work on making RH or Mandrake better.  

And yet now, some of the friendliest distros out there are based on Debian, including Ubuntu.  Without Debian there would be no Ubuntu, Mepis, Xandros, Linspire, Linux Mint, knoppix, etc etc.  

The funny thing is, most of these distros really don't add that much to Debian -- a nicer installer, a few simple GUI's for things, some nice artwork, and most of all a nice selection of packages OOTB.  

Casual users tend to focus on things like the GUIs and the artwork; they take for granted the work that goes on under the hood -- structural issues that lay the foundation for the rest of the system.  There are issues and compromises that have to be dealt with; Debian solves these one way, Gentoo another, Arch yet another.  

Point being: the next big "Windows-killer" distro could as easily be based on Arch as anything else.  Declaring it a waste of time because it isn't easy-peasy is short sighted.

Of course, as everyone else has already pointed out, not everybody wants that anyway.

### Post by billgoldberg on 2009-02-08
> **ryaxnb said:**
> **Simplicity**
There are two trends lately in Linuxdom that I have reservations on. "Simplicity" as in keeping to UNIX traditions is increasingly popular, and distros like Arch, Slack, and CRUX seem to be seeing renewed interested due to this trend. I heartily disagree with the "Simplicity" trend.
I would rather work with my computer than smirk at how "simple" and "clean" the system is. Some UNIX-ness is appreciated, when it's practical: e.g. using vim is fast and easy, and is a great way to edin it text. LaTeX is wonderful for long documents and book writing, and the Linux filesystem is admirable. But making it simple just to be simple doesn't make sense to me. There should be an easy, intuitive way to do things, preferably a front-end to an XML or .conf simple way. PlayOnLinux is a perfect example of a front-end to a complex back-end, and CrossOver is a perfect example of a integrating frontend (one that somewhat integrates with the backend.)

Working with Linux should be productive and allow me to get on with what i'm doing. When I need to configure, Linux should help me in all manners possible get the task done easily, but quickly. Memorization of every possible customization and troubleshooting technique is nigh-impossible, so Linux should present them as choices.
**Perfect Linux**
My idea for a perfect Linux is one the Mark Canonical envisions: easy to use, "simple" but intuitive, and full of choices but making the best by default. Ubuntu, SuSE and Fedora, as well as Mandriva and MEPIS and Sidux, seem to follow this conviction. This means having GUIs for configuration.

**Choices**
On Choices, having more choice and freedom is great. But standardize the backends and make choice transparent to the developer. And pick the default for the end user instead of FORCING them to make a choice; rather, simply allow them to make the choice. Ubuntu does this brilliantly, by having a default choice of GNOME, OOo and GIMP but allowing total choice. 

Some other Linuxes insist that you pick a GNOME/KDE choice at first install. This is unwise, because it splits the userbase too much and makes installation difficult to a new user. It FORCES the choices on a new user. This is bad. SuSE does this, as does Mandriva. 

**More Configuration**
Configuration is hard in Ubuntu in some ways, still. For one, KDE and GNOME have very different configuration environments. And if you don't use a DE, good luck configuring via the GUI! Finally, some tasks such as X configuration and Sound configuration are very weakly handled. And some Configuration tools, like CCSM and Gndismanager, are not included.
Thus, I prefer overall configuration environments, like Yast or Mandriva Control Center. I wish ubuntu had one.


Overall, I just don't see the point to simplicity, and do see the point to choice, if moderated.

To each his own.

I prefer the Arch way. It gives more control to the user.

I spend two hours post installation to get evertything working on my Arch box.

Now I get the best of Linux software with speed Ubuntu couldn't give me.
