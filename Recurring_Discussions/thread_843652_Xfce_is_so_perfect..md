### Post by sisco311 on 2008-06-29
> **bubba_169 said:**
> If you install a DE with aptitude how do you make it so it will remove all installed packages when you remove the original program (i.e. xubuntu-desktop)

(x/k)ubuntu-desktop is a metapackage. It depends on all of the packages in the (x/k)ubuntu desktop system.

With aptitude you can delete a metapackage with all the dependencies.

For example:
Install xubuntu-desktop:
```
sudo aptitude install xubuntu-desktop
```Remove xubuntu-desktop:
```
sudo aptitude remove xubuntu-desktop
```To install and remove a DE you can use [I]tasksel.
[/I]```
sudo tasksel
```If you installed a DE with apt-get you can use tasksel to list the packages installed by the metapackage.

For example:
xubuntu-desktop dependencies:
```
tasksel --task-packages xubuntu-desktop
```

### Post by Darkhack on 2008-08-13
Xfce is my favorite DE/WM, but I do have a few criticisms.  First is the memory leaks.  I've spoken to one of the lead developers about this and it's almost as though we're speaking a different language.  One time I even had to show him where he needed to place is free() statement.  It was his code and he didn't even know where the free() statement should go!

The second is the retarded development cycle.  They use the Debian philosophy of "It's done when it's done."  The thing about Debian though is that they actually maintain the current version with bugfixes while the next version is in development.  Xfce doesn't.  They provided bugfix releases for 4.4 up to 4.4.2 but only because that's the only branch they had.  The moment 4.5 was created, the 4.4 branch came to a halt.  The typical conversation in IRC goes something like...

- "Why don't you do time based release schedules?"
- "Because we don't want a rushed and buggy product."
- "Why don't you provide bugfixes for the current stable branch?"
- "Because we are busy working on the next version."

Do you see the circular reasoning here?  They claim the reason for a non-time based schedule is for stability, but when the product is actually out, and it's not stable, they ignore it so the "next" version is stable.  I imagine when 4.6 is released and bugs are found, they'll ignore them in favor of the 4.7 branch, so that *it* is "not a buggy release."

This development cycle is insane.  Software is like evolution with slow and gradual change.  The Xfce developers (one in particular whom I'll not name just to avoid controversy) throw this concept out the window.  The result is waiting months and years to have bugs fixed.

I've actually considered forking the 4.4 branch and releasing an unofficial 4.4.3 since the Xfce developers refuse to do so.
