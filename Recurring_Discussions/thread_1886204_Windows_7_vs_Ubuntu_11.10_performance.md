Wins Windows--> MS Office 2010 starts in 3 sec, Ubuntu LibreOffice 13 sec
.

You know, you need to change some default settings for LO to speed up starting time. 

Open up any LO program, say, calc. Go to Tools > Options > Memory. Change Number of Steps from 100 to 30, Graphic Cache for LibreOffice to 200mb, memory per objects to 20 mb and you will notice a huge speed gain in startup time. In my system it goes from about 10 seconds to 3 seconds on the first start up and then almost instantly afterwards. It is much faster than MSO which we use at work ( on a more powerful 64 bit machine running Win7) I have no idea why LO uses those default settings which throttle it to death.

You can also activate the quick starter, LO starts up instantly if you have that activated. Though there is a bug in Ubuntu, somehow the system doesn't shut down if you don't first exit the quick starter manually,--with Unity there is a further problem that the quick starter tray icon is not visible so you have to whitelist LO first with dconf-editor. I don't bother becaused after changing the settings as above the speed gain with the quick starter is not substantial (at least after the first startup in the session)

P.S. You can also speed up Ubuntu in general even further by setting vm.swappiness to 10 instead of the default 60, that would force Ubuntu to use ram instead of swap and that would increase performance.

### Post by neu5eeCh on 2011-11-25
Yes, that's the best way to do it. Use Win7 to create an unformatted partition.

Next, download the following manual:

[http://ubuntumanual.org/guides/435/how-to-install-ubuntu-11-10-oneric-ocelot-an-ubuntu-installation-guide](http://ubuntumanual.org/guides/435/how-to-install-ubuntu-11-10-oneric-ocelot-an-ubuntu-installation-guide)

The part you want to pay close attention to is on page 20. When you're asked which installation type to use, choose "[COLOR=DarkRed]SOMETHING ELSE[/COLOR]". As you proceed, you will see a partition table. Look for the unformatted "partition". This is where you are going to install Ubuntu. Click on Change, then choose format. It will give you a list of options, I would recommend ext4. Set the partition type to Primary. Set the mount point to "/". Once that's done, you should be returned to the partition table with (what will be) your new formatted partition. Nothing has happened however, and won't until you press **Install Now**. If memory serves, you should be asked to confirm once more, and then there's no turning back. :-)

Here's another website you can consult: [http://blog.sudobits.com/2011/04/23/how-to-install-ubuntu-11-04-from-usb-or-cd/](http://blog.sudobits.com/2011/04/23/how-to-install-ubuntu-11-04-from-usb-or-cd/)

If you're not comfortable with partitioning, just check in.

I'd just like to throw in my two cents here..

I started using Linux when I was in 8th grade (Ubuntu, of course :D ).  I'm now a sophomore in college and have installed almost every mainstream Linux distro since then.  I'm a comp sci major, so you can say I know my way around UNIX and Windows systems pretty well.  But while I know how to fix most problems, I tend to spend as little time fixing my computer as possible.

Playonlinux is in the repositories, you can install it straight from Ubuntu Software Centre, or [by clicking here](apt:playonlinux), or through the terminal:
```

Which will all work on Lubuntu just the same as normal Gnome-flavoured Ubuntu (except that you would use Synaptic instead of Software Centre).

Generally speaking, the repositories are always the best place to get your software. They're very easy to install from, and your packages will be kept automatically up-to-date.

### Post by Paqman on 2011-11-30
> **Chelidze said:**
> As far as i can understand is that every os that is based on ubuntu will run the same commands, Am i getting this right ???????

Pretty much. The main difference between most different flavours of Ubuntu is that they use a different desktop environment. For example, Kubuntu uses the KDE desktop instead of the Gnome one used in Ubuntu. So they might look different, and even use different apps for the same thing, but most of the system underneath is the same. Once you drop down to the command line you've bypassed the desktop environment anyway, so the commands are the same.
