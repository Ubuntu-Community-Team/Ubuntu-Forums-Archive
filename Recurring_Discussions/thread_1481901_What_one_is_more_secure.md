### Post by Diluted on 2010-05-13
If you're going for secure rather than "safe enough", then Google Chrome, because it is the only Linux browser that has some form of sandboxing implemented or planned.

I think sandboxing is important, because as long as browsers continue to evolve, so will the number of bugs. Sandboxing is a proactive way to migitate and possibly eliminate (some, it's not infallible after all) vulnerabilities. Firefox can patch faster, but that doesn't reduce the severity of their vulnerabilities. You can use things like NoScript, but trusted sites can be infected.

In fact, I find it astounding that Firefox hasn't managed to implement any form of sandboxing, even after the release of Internet Explorer Protected Mode and Google Chrome.
