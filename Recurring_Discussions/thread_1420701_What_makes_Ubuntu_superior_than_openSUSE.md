I don't want to spend a lot of time dealing with dependency hell issues, which RPM distros (in my opinion) seem to have more of than Debian distros.
RPM distros have the advantage of ignoring dependencies. If you have dependency hell, you can ignore a package and it will most likely still work. Under a Debian system, ignoring a package triggers a "Package Broken" and disallows you to install any more packages without fixing that one package.

The current RPM currently trumps Deb in overall configurability. RPM's were also the first to support Delta-patching as to save bandwidth and disk space.

DPMS seems to be following in the footsteps of current RPM innovations.
