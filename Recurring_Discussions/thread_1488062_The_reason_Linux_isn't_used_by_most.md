### Post by Mister Fowler on 2010-05-19
Hello, 

I am one of the many Windows users out there that has tried using Linux (and more appropriately Ubuntu) many times. The last time was the LAST time until Linux makes it easier to install software.

Here's the reason why most people don't use your OS or any Linux distro. Its too hard to install anything that's not in the repository (which is usually out of date for most applications). 

Here's an example of my troubles lately and why I am back on Vista: I tried installing say Java and Netbeans on my Ubuntu system. Of course, there's no "click on this, download it, then follow the prompts".. No, you have to jump through tons of hoops, search the forums and try and use the step by step instructions. Half the time it doesn't work and I have to go into the package manager, uninstall everything and try again. It's that way with Java, Flash and pretty much anything else.

Why is it such a pain in the rear to install anything on Linux? Not to mention there's different file types, and using the command line tools are the worst. Who is supposed to remember all those commands? The end user? Do you intend on only the 'nerds' using Linux like some exclusive playground?

Simply stated, Linux isn't used by most because its hard to use. With Windows, you pop in a CD or download something and double-click the .exe and follow the prompts. 

I'm not bashing Linux, I would LOVE to use it, but I can't because of how obtrusive the OS is to learn. I, like many others, don't want to go through endless pages of books and forum posts to try and 'find a way' to get things installed or working. 

You need an Install Wizard like Windows. If you are worried about security, have it ask for your root password sometime during the install process. Get rid of all the terminal commands and hoops you make people jump through and put together a more user-friendly OS. Do this and you will see tons of more users.

### Post by McRat on 2010-05-19
The reason Linux isn't used by most, is because there are more Windows machines.  Seems simplistic, but that is the reason.

If you wanted hardcore, you should have been around in the DOS days or early Windows.  Makes Ubuntu look like it was preinstalled.

CONFIG.SYS and AUTOEXEC.BAT files and BIOS settings were CRITICAL.  Screw up one byte, you're DOA, and you might not get a reason for it.  You had to carefully read instructions, and often had to open the computer case (Jumpers Anyone?) to load software that was hardware specific.

But, DOS and <3.11 Win were "used by most".  How could that be?  Apples were more "plug and play", so they SHOULD have gotten more use if setup was the cause.

BTW - I liked the "old days" better.  Jumpers and CONFIG.SYS were not ambiguous like modern O/S's are.  And like you, I am forced into an O/S because of my lack of experience.  But in my case, I was forced into Linux because I no longer understood Windows enough to make my machines run together as a team.  I didn't change, Windows did.  The first Linux computer I set up (a couple weeks ago) went flawlessly.  But only after I spent 2 weeks trying to get Windows7 to run on it unsuccessfully.  And Vista is the reason we are now removing Windows from the machines at work, one by one.  They stopped talking to each other correctly after an update.
