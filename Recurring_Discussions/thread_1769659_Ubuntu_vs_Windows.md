### Post by ratcheer on 2011-05-28
The main reason I switched from Windows was because it was taking my XP system five or more minutes to completely boot up and be ready for use. Sometimes it took it just about as long to shut down. I can usually boot Linux to an operational desktop in about 30 seconds and it shuts down cleanly in about five seconds.

Another reason was that I felt completely out of control as to what software and services were installed and running on the system. It seemed like every time I installed something I wanted to use, two more things were installed and ran in the background. Things such as, but not limited to, the bonjour service with Apple iTunes.

Third, my political leanings lead me to want to use free (as in speech) software. I am not anti-copyright and I believe that those who hold legitimate copyright should be compensated for their work. However, I prefer the open distribution model. And I have more trust for it.

I want to feel that my computer is mine and that I can control what I am using it for, not someone else. That pretty much sums it up.

### Post by prodigy_ on 2011-05-28
> **zeke1312 said:**
> I'm looking to see what the advantage of Ubuntu is over Windows.

**Security**

With a clean desktop install of Ubuntu you don't have any TCP ports open, ergo you don't even need firewall in most cases. (You can achieve the same result with any other distro by removing or switching off unnecessary daemons.) You don't need anti-virus either. Some kinds of malware can still affect a Linux PC (e.g. password-stealing trojans or phishing web-sites) - mostly because there's no good protection against user stupidity.

**Shell scripting**

Bash is way more powerful than Windows command prompt and it doesn't require you to learn and full-blown programming language (Windows PowerShell is basically a .NET interpreter).

**Maintenance**

You don't have to reboot Linux at all except when you switch to a different kernel. &#1052;oreover, *sudo* is universal (you can perform virtually any administrative task from your user account without relogging). *Run as...* in Windows isn't universal.

**Remote administration**

SSH is simple to use and you can work with a remote system exactly like if you were using local console (well, except during system boot). In the meanwhile, **all** Windows remote administration tools suck (yeah, this just is an opinion but an opinion based on many years of IT experience).
