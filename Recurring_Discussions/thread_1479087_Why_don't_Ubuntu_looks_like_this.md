This may be one reason that people are saying it looks like MacOS. MacOS has the panel at the top and the menubar on the panel. In the experience I've had with MacOS, I've found such a configuration unusable (but that's just me). I personally like having the menu in the application window because:

A) It keeps the panel consistent (instead of having different things on the panel depending on which window you're currently using). 
B) I see the panel and desktop as "OS space", while the application window is "program space".
C) "Menu bar on panel" means more mouse movement if the application involves alot of clicking in the application window. If the menu bar is on the application window, I don't have to move the mouse as far to get to the menu bar.

> 
3. Dockbar or whatever in the middle: No OS has done it before, and there's enough space. Here's what others do:

- Mac: They put the menubar in the top but they waste more space that they save with the dock.

They also blend together launcher functionality and "running program list" functionality in one item with the dock, which I find confusing and unusable. With Windows/Ubuntu it's easier to keep track of what windows I have open than with a MacOS style dock. 

> 
- Windows: The dock in in the panel but the menu is in each application.

Windows splits the "dock" on the panel into a launcher section and an "open windows" section, which I find much easier to work with. See above WRT to having a menu in each application.

> 
- Ubuntu: Menu in each application and 2 panels, wtf?

Since I like having the functions of MacOS's dock split into two parts, Ubuntu's two panels give me room to have (almost) a full panel for each function, rather than having the two crowd each other out as they do on Windows (generally with a preference for the "window list" function at the expense of launcher space).

I've even added a third panel for various system information things like memory and CPU use.

> 
Putting the dockbar in the middle with the menubar in the left saves more space and it looks really great, doesn't it? It's by far much more usable and easy to use as you have everything in the same panel.

Which makes that one panel *very* cramped. You end up with the OS menu, the program menu, the dock (which is a combination window list / launcher space), and other functionality like (in your example) the clock all on that one panel. Which doesn't give much room for any of them. The only reason it's at all workable spacewise is that you are able to stand the "dock-style" combination of launcher and program list, which I find awkward to use. As soon as you try splitting up the dock to be (for me) more user-friendly, that panel is way too crowded.

> Hiding the dock is less usable, and it looks worse.

4. The rest in the right part of the panel. As I have said there's enough space for everything, but you can still put only the hour instead of date and time and hide all the icons of the notification area except sound, wi-fi and messages.



In smaller screens you can hide some of the things that are in the right of the panel to get more space, but again, there's enough space for everything!

Not for me.

Let the distro's select the GUI setups they like. They'll never please everybody, so if you don't like Ubuntu's setup, customize it to something you like. It's what I do. My desktop looks far from the default, and far from yours.

:-k interesting, I guess my first port of call would be to check the startup applications settings and make sure nothing is duplicated there. Also check your gnome terminal config and make sure the profile for the embedded term isn't the default profile for new term windows. Also in the Startup Applications Preferences dialog select the Options tab and make sure the 'remember my running applications' or whatever it says box is unchecked. 

To remove the gnome-panel altogether you need to down and dirty with your gconf. Bear in mind one thing though, once you've removed gnome-panel you won't be able to use the default alt-F2 shortcut to run commands because in gnome that particular function is tied to the panel. You can probably install a separate app launcher and then re-map the keyboard shortcut to that but I digress.

Removing gnome-panel:
open up gconf-editor and go to desktop->gnome->session
Select required components and change the value for panel from gnome panel to nothing (or in my case I changed it to avant-window-navigator, if you do the same then make sure to go back to the Startup Applications Preferences and deselect AWN from there because now gnome is handling this one outside of that scope).

I hope all that helps, not sure why you're getting the weird app startup issues 8-)
