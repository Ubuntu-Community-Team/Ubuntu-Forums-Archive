Counter-question, how is it not? The definition of 'natively' is not quite as straight forward as you might think.  Wine is just a set of libraries, that facilitate running Win32 programs.

Wine can't run anything by itself. Wine is a set of libraries, that map the Win32 API to Linux.

This indicates you don't really understand how Wine works. Wine does not contain any of Windows' libraries, Wine is a set of wrappers that map the API calls of a Windows compiled app, so it can function _natively_ on Linux. Wine libraries are Ubuntu's (actually Linux's) libraries, whose libraries do you think they are? There is no set defined group of libraries that are 'ubuntus' or 'linuxs'. There are many different libraries that Linux can use, Wine are Linux libraries, that just happen to map win32 API calls.
