The 32bit/64bit memory usage (ie 32bit cannot use more than approx 3GB) is true for all OS's (it's a physical thing) but there is a thing called PAE (physical address extension) which can allow 32bit processors to address upto 64GB of memory.
If you install 32bit Ubuntu on a machine with >3GB RAM it automatically uses the PAE version of the kernel and allows the use of all your RAM.

To do the same in windows you need to use the Server versions ($$££$$).

To run movies etc you will need to have installed the restricted extras as mentioned by pbpersson earlier (this is to do with copyright issues).

For flash issues first try the Flash-aid add-on for Firefox, it sorts out a lot of common issues.

### Post by sdennie on 2011-01-08
There is no need to install 64-bit just because you have a 64-bit processor.  As mentioned above, you can still use 32-bit with PAE but, it might not even be necessary.  When you restart the computer, enter into the BIOS and find how much RAM you have in the machine.  If it's less than 4GB, you are fine with what you are running.  If it's 4GB or larger, you will want to run the PAE kernel or switch to 64-bit.

If you have 4 GB of RAM in your laptop I would just install the 64 Bit version of Ubuntu. People make a huge issue out of Flash, but I can run most video's fine. Using the PAE extension is IMHO more experimental than just using the 64 bit Kernel. 

Furthermore I have noticed that Ubuntu 64 bit also installs the 32 bit IA32 libraries which takes away an earlier problem that 32 bit programs don't run.

Sun java can be installed in both 32 and 64 bit versions, and Flash can be downloaded for Ubuntu 64 bit at the Adobe download page.

An only concern that there may be, is the wireless card. If you have that up and running now, then also this will probably go smoothly.

If you are still experimenting with Ubuntu then it is better to straight try all options and see what works best...
