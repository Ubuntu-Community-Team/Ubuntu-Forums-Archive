### Post by QIII on 2009-11-30
There are grandmothers who use Ubuntu without ever hearing the word "terminal".

It is a powerful tool that is good to have handy.

The "programs, including one that lets you download just about any Linux program without even opening up a web browser" is called Synaptic (and the more recent Software Center in Karmic).  It give you access to literally thousands of applications from soup to nuts.

A word about Wine (the acronym is recursive for Wine Is Not an Emulator):  It does not universally run all Windows programs equally well.  Some simply do not run at all.

There are emulators and virtual machine solutions.  Each of them has its pros and cons.

You can also dual boot and continue to use Windows, choosing it or Ubuntu in the GRUB startup list.

### Post by Mark Phelps on 2009-12-02
The operative word in your question is "use" ...

Of course you can, the same way you can "use" MS Windows without every having to install apps, or open Device Manager, or update drivers, or run Windows Update ... etc. which, if you buy a system preconfigured with Win7 and MS Office, and are content to use IE, could likely be used without any additional software by most users.

The differentiator here is that installing, configuring, upgrading, and troubleshooting Ubuntu systems, as is true in MS Windows as well, often leads to using the terminal (known as CLI mode) simply because that's more detailed and provides a much more powerful interface than screens.

That said, Canonical has taken great strides in recent versions to make more and more "system admin" functions available in menus rather than using CLI. But, there will probably always be things (like installing NDISwrapper for Wireless) or installing restricted video drivers that will be available in CLI as long as they exist.

So, the companion question "Is it possible to _maintain_ an Ubuntu system without ever having to use the terminal" is (IMHO) most correctly answered with -- NO.
