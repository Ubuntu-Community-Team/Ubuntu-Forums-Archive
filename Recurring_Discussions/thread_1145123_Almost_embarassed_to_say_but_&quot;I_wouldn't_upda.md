### Post by lykwydchykyn on 2009-05-02
I've learned to expect a few regression bugs every time I upgrade.  Sometimes it's due to the fact that I've had the same home directory since Mepis 3.x and there's always some setting hidden away in some dotfile that blows up the new version of whatever (KDE, usually).

I wish it weren't that way, but it is.  What I've also learned is that so far I've always found ways to work around the problem and get myself a comfortable computing environment. So I don't get freaked out much when things go nuts.

I highly recommend taking the ping-pong approach to upgrading:
 - partition your drive with two root partitions, about 10-15 GB each
 - keep a separate home partition
 - might be good to have a separate boot partition as well
 - Install to one of the root partitions
 - Next release, either do a fresh install to the other root partition, or copy the current root partition to the other one and upgrade it (adjusting the necessary files, of course: /etc/fstab and /boot/grub/menu.lst).

I know that's a little vague, but you get the gist.  I do this on the family desktops so that if there are serious regressions I can have them booting to the old version while I sort out the new.
