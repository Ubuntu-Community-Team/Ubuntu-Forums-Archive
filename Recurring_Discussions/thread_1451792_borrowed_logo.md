I don't think any of that is actually correct...

The idea of separating multiple documents into tabs didn't start with Firefox or Opera, and it's not reasonable to assume that all other browsers took FF's code when they implemented tabs.  Even if they used the same programming language it would be harder to merge someone else's code than to write your own.  Besides I don't believe in "software B" not being allowed to use a GUI feature if "software A" did it first.  By that reasoning you wouldn't be allowed to browse FF with gestures, pie menus, or a Back button.  :)

Windows XP does not look like Gnome.  :-k  If anything it looks like Mac OS-9.

Windows Vista does not look ANYTHING like Compiz, which isn't even a desktop environment, just an enhancement...  Vista doesn't do anything that Compiz does except use the GPU for OS display.

KDE4 copied Vista if anything, and if they did they only copied the colors and glassy look in the taskbar.  The only UI that changed between Vista and 7 was the taskbar behavior and KDE4 doesn't have a taskbar that behaves like 7.  It behaves like XP and Vista.  I'm typing this on a Win7 box that used to be a Vista box, while looking directly at a KDE4 box sitting behind me.  The window decorations don't look anything even remotely similar.  The only similarity is the thicker taskbar that has a somewhat transparent look instead of a solid brighter color, and the main "launcher" menu being in the bottom left.  The launcher menus don't even have similar behavior.

Looks aside, Microsoft employees are forbidden, at risk of their jobs, from participating in or looking at any source code from any project outside Microsoft.  My brother works there, that's how I know.  I started taking up Python and he preemptively warned me not to email him any source code for anything I wrote, and told me about the policy.  I mean there's always the possibility of the secret underground bunker where their unregistered ninja clones steal source from other projects, but in general MS thinks the liability outweighs the unlikely minor savings they might enjoy by paying people to adapt code instead of just writing it.

As far as I've heard the relationship between OSX and BSD is quite complex.  To say "OSX is BSD" is incorrect.  To say "OSX is based off of BSD" is also incorrect.  It's more like "Some components of OSX were incorporated from things which had incorporated some things from BSD" or maybe "Some very specific components of OSX are closely related to BSD components."  For example I've heard they used the TCP/IP stack from BSD, but the kernel is based on the Mach microkernel, which isn't even one specific thing but a set of related things which at one point shared code with BSD...  anyway, complicated.  Like saying cats are based on worms because they ate some birds.
