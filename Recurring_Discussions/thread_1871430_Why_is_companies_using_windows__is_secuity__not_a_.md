### Post by the_prof on 2011-10-28
The reasons why companies use Windows are many, and most of them have nothing to do with security.  

Commercially, Microsoft has always been quite a good bet from the point of view that most commercial software is written for that platform.  This was especially true in the late 1980s and early 1990s, and this has influenced the industry to a massive extent.  

Don't forget that most large corporations have fairly closed networks, and in the last two decades have been very used to managing them as such.  As things move into the Cloud, this is changing, but these large companies take a very long time over it, as it tends to be a huge investment from many points of view.  

The fact that security is poorly implemented in Windows (IMHO), is overcomplicated for the large Enterprise, and tends to require a costly patch cycle is really just an inherited part of owning a large IT infrastructure, and doesn't really come into consideration that often.  

Open Source software has only really just started to be trusted by these huge corporations, and is still looked upon with some cynicism by many, so you really can't expect people to change overnight.  

The fact that Microsoft Office, and especially Outlook with Exchange is somewhat of an industry standard these days does not help in a companies decision to move over.  

Until Open Source software can give a large organisation the kind of confidence they need to move over, it will be a very slow transition, rightly or wrongly.  

Don't forget that Microsoft has built up a huge number of partners in the outsourcing industry, and for every large company that goes out to tender for management of their systems, there are a hundred large companies all with Microsoft agreements, and some very convincing sales people.  It's generally in their interest to peddle this crap, whether free software is a better deal or not.  I think this is what we're dealing with here.  

As IT infrastructure moves into the Cloud, I think we'll start to see an Open Source revolution, but it's a long time coming I think.

### Post by Dangertux on 2011-10-29
Well first off there is a huge presumption in the first post in this thread.

The presumption is that government agencies use OpenBSD. 

I spent 6 1/2 years in the U.S. Army working in the military intelligence corps, an extension of which is the NSA. I have utilized NSA assets, and worked with individuals in the information security field under NSA command. 

I am going to be perfectly honest with you, a LOT of both classified and unclassified systems are Windows based. Some are other operating systems, for instance Solaris is pretty popular.

That being said, it has probably been beaten to death in this thread already (I didn't read the whole thing) but I will (re)iterate that security is operating system independent.  Every blanket statement you see that goes something like

"Linux is more secure than Windows because I'm a fanboy and UAC is broken."

Is completely false. 

You're looking at security from a desktop user's perspective, and what you are failing to realize is that there is an entire spectrum of applications, equipment, appliances and techniques available for securing a system or a network that you are probably unaware of. 

Yes, there will be basic domain policies in effect on a Windows network that will enforce decent security across the workstations, however that is only taking into account the domain controller and the domain's resources. 

On a nix based network, sure you have individual MAC and firewalling as well as file permissions and of course all the wonderful domain related policies, but still that is only from a limited perspective.

What you're failing to realize is the existence of things like 

Network Access Controls, Intrusion Prevention/Detection systems, threat analysis, network structure, physical security, credential auditing, limitation of network resources, reverse proxying, advanced firewalling methods, diligent administration and hundreds of other controls a system administrator has at their disposal to essentially prevent idiots from doing stupid things on the network. 

Let's face it, you're not going to sit on an NSA workstation (even an unclassified one) and browse Facebook, or hit up your favorite adult website. For one, you can go to jail, no seriously you can, and at very least have your access revoked.

Ultimately I think this thread comes from an uneducated perspective, and the answer to the original question would be more obvious if more forethought were put into the OP.

Hope this helps.

Okay... Since we're obviously going to go off topic which I believe is "Why is companies using windows is secuity not a priority" We will take a brief diversion, and hopefully land right back on the original topic.

First you mentioned several completely deprecated operating systems. So we're going to throw Windows 95/98/ME out the proverbial window for a moment, and look at something more relevant Windows XP through Windows 7, I'm not going to get into Windows 8 because there are a lot of security implications there that remain both untested and uncharted. It's important to understand that Windows 2000 was Windows NT 5.0 and Windows XP was Windows NT 5.1, so it was essentially a point release, with a fancy new interface.

That being said the vast majority of the corporate Windows using world uses Windows XP (yes even government agencies). Some of those that are more up on the times are utilizing Windows 7, though it's adoption has not been that great, due to of all things, the stability of Windows XP. 

Now if you're talking about home desktop security, which is the tangent you're off on we'll go ahead and compare Windows 7 (fully updated) to Ubuntu 10.04 LTS Lucid Lynx fully updated. 

We'll go ahead and assume an active inbound firewall, a browser addon such as NoScript, and an internet connection on each machine. 

In this relatively default, but slightly on the safe side configuration, neither system should run into a considerable issue with malware. If the user is dilligent in only unblocking scripts they know to be safe, neither operating system should suffer a browser compromise. 

If you want to talk about things in terms of permissions, obviously we've addressed UAC already, and we know it's there just like sudo for Linux. 

If you were to disable filtering of scripts and flash I would say that either system would be equally insecure against a targeted attack. Since vulnerabilities can be exploited in both system's browsers that allow compromise. If you're running as an unprivileged user in either, the damage would be confined until the attacker had to escalate their privileges, either through keylogging or exploiting a kernel level vulnerability or a vulnerability in another application with access to whatever it is they are trying to get ahold of. 

In terms of an automated attack the Windows machine would likely be slightly more at risk, because the majority of malware is written to exploit Windows based browser vulnerabilities. Obscurity, does not however equal security. Therefor we can not logically say that one is more or less secure than the other. We could draw the conclusion that without proper measures (NoScript in this case) one is at higher risk than the other. 

So ultimately I'm not really sure where you're really going with the malware debate. 

Let's bring it back around to things on the corporate side of the house, with businesses and different organizations. So we'll say we have Windows XP versus Ubuntu 10.04 LTS, again fully patched. Also we will say both sets of workstations are sitting in a reasonably well secured network. As long as the system administration is diligent and using the tools at their disposal to properly secure their network as a whole, the XP machine will not realistically be any more vulnerable than the Lucid machine. As I said, if company policies are appropriate, and security applications are applied appropriately the only realistic avenue of approach to compromising either workstation would be a social engineering attack. Which is equally viable on either the Linux or the Windows machine and comes back to training your personnel to deal with such threats.

So really, I ask again what is the point of this debate? You've started a thread, which is essentially not even a question, and are arguing points based on misinformation, and a misunderstanding of the underlying concepts, while simultaneously denying everything everyone else tells you to the contrary. Despite the fact that it is more accurate and makes more sense.

P.S : In the real world you can't be click happy with Linux either. If you truly believe that you're lying to yourself.
