### Post by Tatty on 2008-04-17
I use ubuntu 64-bit, and i have not run into any problems myself using software.  All the software I need is in the repos and work fine.

Ultimately 32bit software is still first priority for a developer as most people still use 32bit operating systems.  It is possible to run 32bit software on 64bit platforms but not the other way round, so as long as the 32bit version works then everybody can use it.

You can always compile software yourself onto your 64bit machine.  I have never done this myself as I have not had a need to, but it can be done.

### Post by AndrewZorn on 2008-04-17
For example,
So everyone having no problems with 64bit, how do you get RealPlayer to work?  I have Hardy, and it isn't going well.

I just feel like everything is about 10x harder on 64bit.  I have TWO sound cards that I cannot get to work on 64bit Gutsy.  So much of the simple software seems like a pain to install.  I thought that moving to Linux would get rid of the woes of bad 64bit support in Windows.

Don't get me wrong, but the way Linux tries to move forward and surpass Windows in terms of speed, technology, and usability really isn't helped by this fact.

I didn't mean to say that EVERYONE has a 64bit computer, but they HAVE been out for some time now... this is like my eighth 64bit processor, and I'm still perpetually tempted to use (or using) software that is not taking advantage of this powerful capability.  Memory addressing, if nothing else, is hard to ignore: you can buy computers in Best Buy that are hitting the limit of 32bit computing, and now they even sell computers with Vista 64 because customers want more than 3gb of memory.  It's hard to imagine that I'm the only one out there that seems so determined to get more performance than a computer sold to a 50yr old woman's $600 Compaq.

The newest video cards are always taken advantage of within weeks.  Not that this isn't a simpler example, but one cannot simply use the "not everyone has your fancy-pants XXXX" to explain the shortcomings of Linux development.

I didn't mean to insult anyone.  I just feel like... again... that using what shouldn't be considered 'bleeding edge' software is like a beta test (not to be confused with the fact that I'm running Hardy... that's just recently, and only on my laptop).
