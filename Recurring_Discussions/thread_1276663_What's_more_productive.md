### Post by samjh on 2009-09-28
> **fela said:**
> No, I meant as in which is easier to use, faster, less bloated and the like.

Not which one you know how to use.

GTK and QT are programming libraries.  The ease of use, efficiency, and "bloat" of the applications created by them are largely dependent on the design of those applications, not the libraries.

If you are trying to compare them from a programmer's perspective, then you are making the wrong comparison, since they don't match each other's scope of functionality.  GTK is only a graphical toolkit, whereas Qt is an entire framework of which the graphical component is only one part.

So the poll really should be:
[list][*]Qt4
[*]Qt3
[*]GTK + GLib + ATK + Cairo + Pango + zlib + gettext + libpng + libjpg + libtiff + freetype + fontconfig + libexpat + [a database library] + [a networking library] + [a crypto library] + [a multimedia library] + [a HTML rendering library]
[*]Other[/list]

My personal choice is Qt4, since it is nicely integrated, comprehensively documented, thoughtfully designed, and easy to deploy on Linux, Mac, and Windows alike.

### Post by samjh on 2009-09-30
> **fela said:**
> Yes but some toolkits I've just found more 'streamlined' - for instance my computer runs gtk2 better than it runs qt4 (although it runs both fine).

The problem with this sort of statement is that there is no way for any third-party observer to be able to compare their experience to yours.

I can't think of even one non-trivial program which has the exact same features and design implemented using both GTK2 and Qt3 or Qt4.  And unless such an example exists, they really can't be compared objectively.

I admit that GTK2 programs tend to be a little easier on system resources than Qt ones.  However, this is not the same as "better".  When on Windows, for example, Qt programs run more smoothly than GTK2 ones in my experience.  From a programmer's perspective, I find that Qt applications work more reliably across platforms than GTK2 and GLib based applications.
