### Post by 3Miro on 2011-09-22
It is not that MS directly bans Linux, however, the end effect may be the same.

MS requires vendors to implement a feature that would in its own turn prohibit the installation of unsigned OS and drivers. Furthermore, this seems a per-vendor option, meaning that an OS or driver signed for Dell may not be installed on Acer.

In order to install Linux on such locked hardware, you will need to have your distribution signed for the specific hardware. That is, we would need to have Ubuntu-dell, Ubuntu-acer, Ubuntu-lenovo and so on. You can also forget about custom kernels. Furthermore, one would have to obtain the keys from all of those vendors. This would be a daunting task even for Canonical and Red Hat, while it would leave smaller distros to rot.

Vendors could theoretically release the keys, however, this would make the keys meaningless. (it is like giving away your password) In fact the feature does seem pretty useless to me, it only guards part of the kernel and some drivers, it does not guard the entire system. There are far more practical approaches to secure the system.

The easiest resolution would be to have a BIOS option to disable the lock, however, it would require vendors to take an extra step to allow the users to install other OS. Many vendors will not implement that and thus artificially limiting the Linux-compatible hardware. If this isn't cause for massive anti-trust, then I don't know what is.

First, I think you're wrong in your assertion that Windows 8 won't boot with the secure boot feature disabled. In order to install and boot on existing computers, Windows 8 *must* be able to boot with the secure boot feature disabled, since most existing computers (including UEFI models) don't offer that feature. Perhaps Microsoft could code it such that secure boot is required if the firmware is recent enough to support it, but I can't see what their motivation would be to do that.

Second, even if Windows 8 required secure boot to work, a dual-boot configuration could still work *if* owners can install their own keys in the firmware. Under that scenario, users could sign their Linux boot loaders using their own private keys and install the matching public keys in the firmware, or install a public key provided with a pre-signed binary provided by the OS vendor. Either way, the system would then boot both Windows and Linux with the secure boot system enabled.

Oh, and the screenshot showing that secure boot can be disabled is a screenshot of *one* product -- and a demo model at that. Manufacturers of actual future products can modify the firmware in lots of ways, and AFAIK there's nothing standing in the way of their disabling the secure boot "off switch."

### Post by srs5694 on 2011-09-26
> **madjr said:**
> w8 betas may be installable now, but we're not sure about the final versions (specially if these plans get approved)

Granted, but I find it hard to believe that Microsoft would deliberately cut itself off from easy sales to people who want to upgrade their computers. I could see this happening in a version or two, though (Windows 9 or Windows 10); by then, computers will have grown enough in power that today's computers might not be good enough for the latest Windows, so locking them out wouldn't be a big loss.

> am not sure what are their plans and why they want **so badly** all new computers with w8 to come with secure boot enabled.

Part of it's probably their stated reasons: This measure will make it harder for the authors of certain types of malware. A Windows with fewer malware infections is a good thing for both Microsoft's customers and for Microsoft.

Beyond that, I suspect it has to do with platform control: By funneling as many software sales as possible through the Microsoft "app store" (or whatever it's called), Microsoft improves its bottom line. Locking down the boot process helps on this score because it makes it harder for a vendor who works outside that system to bypass the app store by writing an installer that works by booting into a custom OS. It also gives Microsoft increased control over the market for system repair tools; they could demand money to sign outside vendors' boot/repair discs or just lock them out of the market.

I don't see how anti-piracy efforts would have much to do with it, since pirates would just use pirated Microsoft installers that have already been signed by Microsoft. They'd need to find a way to hack the registration process, but AFAIK the secure boot feature has nothing to do with this, so I doubt if this would be a serious impediment to pirates. Then too, my knowledge of Windows security and how pirates get around it is very limited, so maybe I'm just missing some important point.
