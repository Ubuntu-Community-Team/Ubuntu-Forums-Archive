### Post by JheregJAB on 2011-04-30
Ubuntu, I must say I'm more than a little disappointed.

I upgraded from ubuntu 10.10 64-bit today to ubuntu 11.04 64-bit. Having read many things, both good and bad, about Unity, I was excited to try it and make my own judgements. I am not impressed, as I managed to crash unity within a minute of loading the desktop. This is how I did it.

update from 10.10 64bit to 11.04 64bit.
Realise that your favourite eye candy, wobbly windows, is no longer functioning. 
open ccsm, to find that it is unchecked.
check it. 
click on whatever the "remove" option is, as this remove option sounds like what I want to do.
This will crash the top bar. However, the left hand bar will recover.

Now, realize that your beloved desktop cube is gone now too.
try to enable it. It'll tell you that the unity plugin will turn off if you enable it.
For fun, click to enable anyway.
obviously and expectedly, unity no longer functions. However, this wasn't clearly described. Common sense said it would happen, but it wasn't actually described.
re-enable unity plugin, expect unity to come back.
It doesn't.
get frustrated and go back to "Ubuntu Classic"

Rinse and repeat for Ubuntu Classic, ending **in manually adding all compiz plugins necessary to use the desktop, and still be missing a few.** (not to mention all my custom settings from before the update were wiped out.)

To make an already long story short, go back to unity, create a new desktop launcher to be able to open ccsm since nothing else functions. Do everything you already did in ubuntu classic to get the unity desktop to work, with the addition of checking the Unity plugin. I also had to log off and back on again to get unity to reload.

Wouldn't you think there would be a built in warning or something that accessing ccsm from **The System Setting Dialogue**  would crash your desktop?

total time to crash unity: ~1 minute
total time to do the same to ubuntu classic: ~5minutes
Time spent to figure out how to use my desktop again (in classic): ~25minutes.
apply those changes to unity to get it to work again: ~another 7 minutes.

Yeah, unity was ready to be released. </sarcasm>

Jacob.
