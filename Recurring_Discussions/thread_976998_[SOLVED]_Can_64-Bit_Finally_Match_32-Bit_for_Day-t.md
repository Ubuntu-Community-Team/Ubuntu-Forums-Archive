### Post by saru411 on 2008-11-10
From what I understand you want to try and convert people to use 
Ubuntu. I believe people that use Linux aren't afraid to learn. If they are happy with windows they shouldn't be motivated to install Linux. If you do manage to get someone to install Linux and they aren't interested in learning, they will most likely be frustrated with some step in the setup. All Operating Systems(OS) require SETUP. This includes Ubuntu. You might notice when you buy a computer from a store or site it has everything working. This is because the manufacturor installed and SETUP the system to run coprrectly. This may include installing drivers or software(e.g the wireless card or flash) Linux is not Windows. It never was intended to preform "like" Windows. Sure Ubuntu makes installing Linux easy but it also has tons of overhead. I find showing how Linux isn't so hard and exposing them to it will more likely foster an effort to learn. Just installing Ubuntu on someones PC and not expecting them to tell their friends Linux is for "geeks" or "techies" is shortsighted.

As for the 64bit vs 32bit part of your issue, you should always check the HCL(hareware compatibility list) before buying parts/computers to see if the O/S supports the hardware. I have yet to run into an issue in 64bit that I couldn't fix by searching these forums or google, unless of course the hardware wasn't on the HCL or the driver wasn't released(btw, drivers are produced by manufacturors not Ubuntu Developers so if it doesn't have a current driver they can not do anything to right the problem).

Your best friend in finding info is the internet, namely Google

### Post by djchandler on 2008-11-11
I've been running Hardy 64-bit since release in April, having used only 32-bit Linux distros up until then. So far I really can't tell the difference functionally once I got Flash and Java working, which was not difficult at all. I used Synaptic to install my mozilla plug-ins, and nspluginwrapper was installed automatically as a dependency.

The Nvidia driver is still a problem for me too. It's okay on another box (Nvidia 6800XT) that had Ubuntu (Hardy 64-bit) installed via Wubi, but has problems on this dedicated Ubuntu box when I implement Compiz features  (FX 5500/256mb). I'm okay as long as I don't use Compiz. Open GL graphics are great in spite of this Nvidia restricted driver problem.

I usually have 1 gig of ram in this one (AMD64 +2800), but I robbed 512mb to test a motherboard and cpu I got at the local recycler ($5.00 + tax), a P4@2.53ghz ( Northwood core-Socket 478 ) with DDR PC-2700 ram slots. It and my AMD box run just fine with 512 mb. Haven't noticed a bit of slowdown in Ubuntu, and I do Folding@Home on it as well.

Things feel a little snappier with the 64-bit OS, but that's purely subjective. I certainly haven't noticed any penalty.

PS Had to use Debian (32-bit) Etch on the Intel motherboard. In my experience, later versions of of Ubuntu, 7.10 and up, don't play nice with EIDE controllers.
