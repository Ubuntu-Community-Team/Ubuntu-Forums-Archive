### Post by Calash on 2010-08-20
> **NCLI said:**
> The Droid X is harder to jailbreak, sure, but it's much easier to keep jailbroken, since updates to Android won't unroot the phone, as opposed to updates to iOS.

Where did you hear this?  As far as I know it is completely untrue.

First we need to settle the difference between Rooting and Recovery

Root - Gives you super user access on the phone to allow modifications to the root writeable areas of the phone.  This allows you to run Root apps and remove applications that were installed with your rom (in .apk form)

The DroidX is very easy to root.  I did it the hard way and it still only took 10 minutes to root my friends DroidX.  The easy way involves a very nice custom app with two giant buttons.

[http://www.simplemobilereview.com/remember-the-droid-x-has-a-1-button-root/](http://www.simplemobilereview.com/remember-the-droid-x-has-a-1-button-root/)

Recovery - A custom recovery is needed for Rom development.  This is where the DroidX is locked down as the bootloader is encrypted and without being able to break that encryption a full custom recovery just will not work.

In the past few days developers have made huge steps in this area and as of right now there is a hijack exploit that allows the execution of a custom recovery.

[http://www.koushikdutta.com/2010/08/droid-x-recovery.html](http://www.koushikdutta.com/2010/08/droid-x-recovery.html)

This is the first step to custom Rom development.  

Jailbreaking an iOS device allows the execution of unsigned code with root privilege, so it is essentially the same as Rooting for an Android device.


The point of all this, outside of clarification, is that when they push an update to your phone it will remove root access.  It will not remove a custom recovery as they block these types of updates.

Also, remember that Motorola is not the only Droid game in town.  HTC is historically easier to load roms.
