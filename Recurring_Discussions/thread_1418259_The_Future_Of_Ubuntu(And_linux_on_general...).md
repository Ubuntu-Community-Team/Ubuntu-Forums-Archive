### Post by lykwydchykyn on 2010-02-28
I'm not exactly holding my breath for Ubuntu, OSX, or any other non-Windows OS to "take the world by storm".  But assuming it does, consider the following:

 - Malware is big business.  Like all business, it's subject to the laws of supply and demand.  There is a demand for compromised computers, stolen identity/credit card information, adware, spambots, etc.  As long as there is a demand, there will be those who work to supply it.

 - No OS can be secured from social engineering.  I can't remember the last time I've seen a Windows box that was compromised via a security flaw in the OS.  But not a week goes by that I don't see one compromised via social engineering (someone installing malware, basically).

So it seems justifiable to me that we should expect malware attacks against Linux if it becomes a dominant player in *desktop* operating systems.  Keep in mind, almost nobody gets malware on a server or embedded device.  The former lacks the use profile by which it could be infected (i.e., almost no one surfs the web on a server), and the latter is typically too limited to allow infection (though there have been exploits against phones, IIRC).  So, saying "Linux is everywhere, but it's not being infected" misses the point a bit.  Linux is not positioned in areas where social engineering can be effective, generally.

Now, with that said I see two main advantages that can come of people moving to Linux from a security standpoint:

 - Linux has built-in diversity.  This makes both genuine security exploits and social engineering more difficult, because you can't count on anyone's Linux machine having the exact same software, versions of software, or look & feel.

 - Switching from Windows over to Linux forces users to relearn some habits.  This is good.  A lot of what hampers Windows security now are the bad habits of both the user and development communities which evolved around DOS-based Windows.  This is why, despite a relatively serviceable limited-user-account system, so many Windows users run as Admin.  And when Microsoft tries to fix it with UAC, people turn off UAC.

This is why you get such a negative reaction around here and other Linux communities when you want to run as root or "turn off sudo".  We don't want to build a user community around the same silly security mistakes that were made in Windows.

Even with all this, though, there is a will to crack systems.  And where there's a will, there's a way.

We already have something LIKE THAT in place, though, with trusted software packages. So if it's not trusted, you SHOULDN'T INSTALL IT. Simple as that.
