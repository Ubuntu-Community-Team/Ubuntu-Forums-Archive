I have no issues with KDE's speed on Gentoo Linux. A freshly booted desktop on my laptop uses approximately 170MB, although I stripped most of the bloat out of KDE by compiling without semantic desktop. You could probably speed up KDE if you turned off file indexing, which has a similar effect, although it would not do anything about memory usage, which requires compiling KDE without semantic desktop features.

The main issue with running KDE applications on GNOME is that that it requires loading the Qt4 libraries in the background, which is the same issue you have went you run GNOME applications on KDE.

### Post by slackthumbz on 2010-05-26
> **kokoshmusun said:**
> Ok, I understand installing themes now. Great.  However, gnome-look website confused me a lot.  I don't understand the categories.  How are all these themes different from each other?  Compiz, beryl, emerald, icons, GTK, GDM, etc.

As far as I know, I'm using compiz.  So is that the category I should look in?

for your applications you'll want GTK themes (GTK is the widget set that GNOME based applications use.) For icon themes for the whole desktop or even individual programs look in Icons. For window decorations you should look under Metacity (metacity being the program that GNOME defaults to using to manage your window decorations. Don't worry about the fact that you're using compiz. Compiz will use the metacity themes that you install and select in the Appearances config anyway)

Hope that helps :D
