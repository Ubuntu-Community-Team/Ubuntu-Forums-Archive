### Post by Techsnap on 2010-01-08
Slackware since 12.2 has included Slackpkg by default which you can use to update your system when they become available. 

Switch User to root:

```
su
```

Set up your mirrors:

```
nano /etc/slackpkg/mirrors
```

Install any New Packages & Run Slackpkg:

```
slackpkg update && slackpkg --install-new && slackpkg upgrade-all
```

You'll then be given a menu of all the newer packages in the package tree and you can select/deselect all the ones you want.

### Post by kevCast on 2010-01-11
Debian and its derivatives install *usually* every dependency for every compile option of some particular package. That way they don't have to curtail certain features. If they didn't do it this way, a lot of users would have to recompile and add certain flags to get the functionality they desire. It's a choice between convenience and a few MBs of disk space. I personally prefer the first, but for others, there's always distros that allow customization. I do know Arch allows binary add ons to their packages.
