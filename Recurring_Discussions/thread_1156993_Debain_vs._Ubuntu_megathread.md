### Post by snowpine on 2009-05-12
They are 95% the same; most of the skills you learn in Ubuntu apply to Debian, and vice versa. Ubuntu gets most of its code from the unstable branch of Debian.

The main difference is the development/release cycle. Ubuntu releases every six months. Debian has a unique three-tiered unstable/testing/stable cycle (see the link in mdmarmer's post). 

In terms of advantages, Ubuntu has a newbie-tolerant community (these forums) and can be easier to set up on certain computers (Debian includes only open source software in its default install, whereas Ubuntu bundles many proprietary hardware drivers). Debian is faster, less buggy, and gives you the option of stable vs. rolling release (testing or unstable).

(edit) I see you asked the same question on the Debian forums. I think the types of answers you get on each forum will tell you a lot about the respective communities. Debian tends to reward users who are willing to do some research and ask questions that show they have put in a little effort.

### Post by snowpine on 2009-05-12
Debian comes in three flavors.

Stable (currently Lenny) is roughly equivalent to using the Ubuntu LTS (long term support) releases, like Hardy Heron. 

Testing (currently Squeeze) is a "rolling release" meaning you install it once and get new versions of packages as they become available.

Unstable (always Sid) is also rolling release, but the packages are newer ("bleeding edge"). Ubuntu is based on a "snapshot" of Debian Sid every six months.

The pros of using Debian are that it is faster and more stable than any Ubuntu release. The pros of using Ubuntu are a newbie-forgiving community (please do not ask "is debian better than ubuntu?" on the debian forums!!!) and the inclusion of non-free firmware (although for some idealists this is a con) meaning more hardware works out-of-the-box.

### Post by msrinath80 on 2010-02-19
My 0.02$ on Debian. I've been using it so far (actually ever since lenny was in testing). I've been a long time RedHat (RH) user since 6.0. I naturally moved on to Fedora after RH 9.0. Got disappointed with the (expected?) instability and therefore tried to switch and that's when I found Feisty. It was really fun for the first 1.5 years with Ubuntu, but then again, random regressions over any upgrade caused me to lose faith in Ubuntu. So I decided I'd try Debian. The very first time I tried Debian was when sarge was in stable. Back then, I had no patience (esp. with the text mode installer) and therefore could not tolerate Debian. So I moved on. Later when I understood the intricacies of apt-get (thanks to Ubuntu and Synaptic (which for some reason I never found in sarge), I decided to give Debian another try. This time I was successful. And I've been a happy Debian user ever since.

The thing with Debian (stable) which I use both on my Desktop and Laptop is the insanity in stability. And I mean unfathomable stability. If you've distro-hopped a lot in search of stability, and you land in Debian, you'll freeze. You won't want to move away.  

For instance, I've had lots of issues with crashing GNOME applets in most distros (RedHat included), but with Debian it's like 1 or 2 issues in maybe 6-8 months. And in all cases, it has been something that got fixed in the next update or so. Granted, the price I pay is using slightly older software (again this is relative in that if someone is using say Windowmaker or XFCE, then for the most part, Debian Stable is not just stable but also current), but this is something I am willing to trade for more stability. Now, realizing such stability in large projects like GNOME is no easy task. The way Debian does this sometimes is by linking parts of an older version of GNOME with a newer one, especially when the newer component is not stable enough. As an example, my GNOME version on Lenny is 2.22.3 while my GNOME panel version is 2.20.3.

Checking out Debian for yourself is probably the best way to go. If you've been using Ubuntu for at least 2 odd years, I'd say you can definitely try your hands on Debian and get it right the first time.

A word of caution for laptop users however. The current Debian stable best runs on a laptop which is at least 2 years old. Anything newer, you're better off running the current testing branch (squeeze). As for the latest corei* users, you're probably even better off running Debian (unstable) or sidux, but that's certainly not what I'd recommend. Testing is my limit :)
