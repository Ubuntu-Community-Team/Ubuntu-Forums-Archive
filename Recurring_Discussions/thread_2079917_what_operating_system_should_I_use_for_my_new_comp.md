### Post by drmrgd on 2012-11-03
The main difference between the server version and the Desktop version is the lack of a pre-installed GUI in the server version.  The server version also comes with more server related packages built in (like Apache2).  Of course, one can always add something like Apache2 to the Desktop version, and a GUI to the server version.  Have a look here from the Ubuntu Server FAQ:

[https://help.ubuntu.com/community/ServerFaq#What.27s_the_difference_between_desktop_and_server.3F](https://help.ubuntu.com/community/ServerFaq#What.27s_the_difference_between_desktop_and_server.3F)

I would guess (although I really have no idea), that you would want some sort of graphical output from the stuff you're planning on doing.  So, in that case, I would recommend probably going with Desktop so that you'll easily be able to pull up the graphs and data charts that I'm assuming you'll be making.  If don't need any of that, I suppose server would be fine too.  I'm not sure it's all that critical.  I've built a pretty solid box here (although not quite that heavy duty!) for bioinformatic and computational biology work, and I'm able to run this stuff in Desktop version with no problems.
