### Post by Mr. Picklesworth on 2008-11-14
I like how KDE 4 looks, but I don't understand the widget craze. It ultimately adds nothing to the user experience that startup programs do not. 4.2 adds a preview widget and an item in the file browser to present previews in that widget, instead of just opening previews *in a new window*. Plasma now does libnotify popups. It looks pretty, but a lot of this is little other than a giant show of arms that I hope my choice desktop environment never engages in.

Granted, KDE 4 has some nice changes. Using Plasma to render the desktop is a nice start (since the file manager doing that is dumb!), and everything looks a bit smoother. KOffice 2 looks like it's coming along well. However, I am not seeing much as a major "innovation" even in the most trivial sense of the word.


Here is stuff similar to what KDE4 does that *would* improve the user experience:

[list]
[*]Window manager tweaks to stick windows together. For example, glue an instance of GNOME Terminal to the bottom of gedit.
[*]Better distinction of actions that can be performed with files in the file browser.
[*]Ability for software to see and to control what file is being worked on in any application from a *user* perspective. Could be useful for the gluing windows tweak. Window manager could present an icon representing the file open in each application, and a simple operation with that icon could get them both pointing to the same place.
[*]A widget layer applied not to present pretty pictures, but to visually represent background processes. The user could link them together, thus creating desktop scripts, resulting in unprecedented flexibility. For example, use the Clock widget to set a timer, where at 5:00 PM a process will be force quit and the computer shut down.
[/list]

### Post by Starlight on 2008-12-23
I have a rather strange and unusual opinion, because I think that KDE 4 itself looks awesome, but actual KDE 4 apps not so much, since they use Qt. In my opinion, GTK apps looks a lot better, because GTK has much more interesting themes. Qt has some nice ones, like Oxygen and Skulpture, but even they can get boring... from what I've noticed, all Qt themes look basically the same no matter where you look. No matter if you look at the menu bar, toolbars, buttons, or anything else, you see the same shade of the same color. At the same time, GTK themes have much more variation... different menu bars, different toolbars... so that when you look at a GTK app, it's just a lot nicer. Another thing is that there are a lot more GTK and Metacity themes than Qt and Kwin themes, so there's more to choose from. So, in my opinion, while the default KDE 4 desktop looks a lot cooler than the default Gnome desktop, if you get some cool themes, desktop widgets, Compiz, etc. then KDE 4 with Qt apps looks poor when compared to Gnome with GTK apps. :)
