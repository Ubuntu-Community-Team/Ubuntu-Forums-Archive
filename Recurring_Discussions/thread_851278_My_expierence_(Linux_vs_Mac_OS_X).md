### Post by ryaxnb on 2008-07-06
Me and Linux go way back (Red Hat 7, Mandrake 8.1) and also way forward (i've used 8.04 betas, and am now using 8.04.) One consistent problem: lack of hardware support. I have two really crappy laptops, $500/599 dollar sale, and they just doesn't play with Linux; no sound on some distros, stupid atheros/broadcom dumb*** Wifi, no suspend on some distros, etc. I've also found, on Mandriva, installing software to be a collosal pain; Rpmdrake is slow, and the repositories are small. The drivers are hard to get, etc. So far, my first good expierence is Ubuntu 8.04 on the Compaq. Excellent repositories, supports the Wifi to some degree (it's broadcom), suspend works, and sound is fine. My desktop is not so happy, and I plan to post a thread about it (Dell P4, 512MB RAM, integrated Intel gfx) because the gfx just randomly stop working and also halt at res switch, totally unrecovorable, have to reboot. 
What I'm getting at is that linux still has a ways to go yet with hw support. Ubuntu gfx troubleshooting is plain bad, and it's lack of ndiswrapper is also bad. Its sound support is still a mess, especially with integrated intel mobos, and of course its suspend support is relatively crappy on certain machines. :(
I've worked around these on the Compaq, but I still find two problems, gotta start threads about these too:
* ATI integrated support from the official driver is crap. I suspect that the unofficial 3d driver will work however, it's an older chipset. 
* WiFi support drops to super-low speeds at long distance, even though both Windows and other Wifi cards handle it fine. I suspect ndiswrapper can fix this. 
In the meantime, I'm using it, living with the problems, and using 2d-only gfx. 

In contrast, on the Mac, hw support is naturally excellent. Apple makes the whole widget, and it works great.
Also, on Linux, nothing matches: iMovie, iDVD, iWeb, Create (from Stone), BBedit, OmniWeb, Airburst Extreme, Numbers, Keynote, and Word 2008, Dreamweaver, Fireworks, OmniOutliner, OmniGraffle.
OTOH, Linux has just as good FTP, and cheaper, GIMP, which is the best low-cost image editor (e.g. $0-$100), VLC, gstreamer (better media then Quicktime), and the best light-weight word processor (AbiWord), the best mail/collabortaive apps short of Outlook (Evolution), the best file manager choice (Nautilus, Dolphin, Thunar), the best free games, the best Wine layer, K3b, and a good Chess and Go app (pychess, qgo.)
 Why, you ask, is there nothing like BBedit? Certainly emacs or even Kate would fit the bill? No. emacs is far too confusing. For a simple user who wants to hand-code a basic PHP, HTML, or C++ app/script/page, BBedit offers the perfect combination of features and ease of use. It costs a bunch though. And there's no way to run it, or anything as easy as it, in Linux. 

OmniWeb? Firefox would have to be customized to the hilt to compare, and I still haven't seen a match for Omniweb's thorough adblocking, site preferences, tab drawer, or other features.

The other reasons are similar; a perfect combination of features, accesibility, and ease of use.

What about the OS itself?
Fonts:
Linux does not include a good variety of fonts, nor does the msttcorefonts pack. However, Linux's font converters are top notch. 
Stability:
Both are very stable, however when the GUI is sunk, Linux does not break; this is a significant advantage for servers, home servers, and some other configs, but useless for the average user. Also, the Command+Option+Esc (similar to Ctrl+Alt+Del) hotline is very handy when an app is crashing.
Speed:
Linux is optimized for kernel and server performance, not user performance. Darwin is slower than Linux at kernel tasks, but OS X is more end-user optimized.
Adaptability:
Linux is very adaptable to any speed of computer by anybody, as DSL demonstrates. Linux and distributions are also adaptable to almost any 32-bit arch. In practice, SPARC, PPC, ARM, x86, and X86-64 are widely supported. Mac OS X is adaptable to multiple archs, x86, x86-64 and PPC are supported, but support is limited to what Apple wants. Linux is portable by anyone. Mac OS only runs legally on Macs. It does not run on other compliant PPC machines (CHRP, POWER servers, etc.) nor does it run on the new YDL Tower. It will run with hacking on a limited selection of x86 and x86-64 machines, illegaly.
Interface:
Much easier on the Mac. No CLI usage is needed for anything anyone beginner-advanced would really need to do. CLI is available however and comes with BSD tools. The HIG is followed by most apps and is well written. Compositing is not superfluous, but cannot be turned off. The iconsets are consistent. The customization is almost zilch.
OTOH, Linux is much more flexible. Anything can be customized, and DEs and WMs can be swapped, compositing is cuztomizable, and you can turn it off altogether. The entire OS almost feels changed when you swap the DE/WM and especially if you switch from say bash to tcsh simultaenously, or install Busybox/BSD utils.
Remote Control, Remote acess, etc.
X11 is very remote-friendly. The CLI orientation ensures that you can do anything admin related with just SSH or telnet. VNC clients and servers include TightVNC and RealVNC, two of the best.
On Mac, VNC support is limited. Apple Remote Desktop adds more. X11 is available, but does little. SSH is less useful because of NetInfo.

This was written by a Mac/Linux User. I use both an iMac and a Compaq with Linux regularly.

### Post by Frak on 2008-07-06
> **akiratheoni said:**
> If Macs have such great hardware support, why do projects [like this](http://en.wikipedia.org/wiki/Hackintosh) exist? I should be able to pop in a Mac OSX CD and install it on any computer and get compatibility.

I've worked with Apple before on the Intel transition, which was the *Developer's of Darwin Evolution* project (codename). This was much of a hack-a-couple-months and get usual apps to run under the x86 project.

Well, of course, I'm a horrid coder, so I was on the platform testing team. This basically means that they give me a DVD, I install it, and I report bugs. Well, the new Macintosh's weren't exactly out yet, so I got a 10.4 Developers DVD, with an "Extra" DVD for which I will get to later, so I could install it on any Intel box. Well, I used OS X, it went from buggy, to less buggy, to faster and faster. It was a great transition. Let me give you my system specs

Intel Pentium 4 Prescott, supports SSE3.
Nvidia GeForce 6800 XT
Intel i815 mobo

Nothing great, eh? But it ran, and ran fine it did. It ran blazing fast in fact. In fact, Apple endorsed it.

Now, maybe you're asking "Why did I tell you all of this? What does this have to do with anything?". Well, the "Extra" disc I had was a Kext disc. It automatically launched another OS X install-like routine. It then asked me to install Devon (Developer Unlock) kexts onto my system. I agreed and the installer progressed. It optimized, rebooted, and the only limit it had was the fact it had to have a special OS X disc to be upgraded with.

Well, now the developer trials are over, so I can no longer get those special discs. Though, it can be replicated because Apple put a file on the DVD that was required to be read. When the file did not detect the correct OpenFirmware interface on the EFI chip, it sent a file called "NotAppleCertified.kext" at the installer, which then pops up a message saying it is not a certified apple computer, blah, blah, blah. THIS is what keeps normal DVD's from installing on computers. Those people like Kalyway, and Semthex didn't *make *drivers for the system. They just unlocked kext (kernel extension) files to allow other compatible hardware to be addressed by the kernel. There is no way to install drivers for the system, because they all have hooks in the kernel.
