### Post by ian-weisser on 2014-08-29
> **jeremy9856 said:**
> If you know how to do it well but do not think most people will know how to do it.

The concept that 'everything I could possibly want to do should be intuitive and one-click away' is very attractive (android sells apps that way), but also impossible to implement. There are thousands of possible actions you could do right now. Most you will do rarely, if ever. They cannot all be one click away.

In 10 years of using Linux every day, I recall formatting perhaps one or two USB sticks. Perhaps my experience is unusual, but I expect to need to look up how to do very rare actions. If I saw the option every day, I would be annoyed that the never-used option was taking up valuable icon/menu/cognitive space.


> **jeremy9856 said:**
> People have habits that come from windows, sometimes for good reason sometimes not.

There's a phenomenon called 'imprinting' in psychology. The first  system you learn becomes the yardstick by which you measure others. It's  totally understandable, but the first system is  not somehow 'best'. It's simply first.

People must recognize and overcome imprinting. Ubuntu is not a Windows clone. And it's not trying to be.
All platforms are different. And migrants from any other platform will need to unlearn some habits.


> **jeremy9856 said:**
> why spend some time to search how to format a usb drive whereas we can do it with simple right clic...

For a very good reason: Good design. Bad design is filling menus with a thousand cryptic, rarely-used options. It makes the software seem complex and scary. Good design is having the most-used options require the least clicks to select, and to make the least-used options discoverable but cognitively farther away.

And another very good reason: Formatting destroys data, and people don't read dialog boxes. An easy-to-access data destroyer is more likely to be used by accident.

If you search the nautilus-developers mailing list, I'm sure you'll find the right-click-to-format idea get discussed (and rejected) repeatedly. But this is also why Nautilus has a plug-in feature, so anyone can write a right-click-to-format plug-in. I don't see that anyone has written one, but do feel free. Nothing stopping you.

Developers use bug trackers to prioritize which bugs should be fixed in their limited time available. "Fix my bug" is not a useful idea to improve Ubuntu. Bugs are already getting fixed all the time.

And the users who reported those bugs haven't done their community duty yet - many of those bug reports contain inadequate information for a (volunteer) bug triager to reproduce the problem. If a triager cannot reproduce the problem, a developer will never see the bug report. When I have triaged bugs, I have had to close tons of bugs like those...often after *begging* the reported for more information, without response. The report is a waste of everybody's time -including the reporter's time- unless it contains useful information to actually locate the bug.

IF YOU WANT THOSE BUGS FIXED: Participate. Add a lot more detail to them. Starting with exactly how to reproduce the behavior - every time. Tag the bugs properly: See [https://wiki.ubuntu.com/Bugs/Tags](https://wiki.ubuntu.com/Bugs/Tags) for how. Add apport information to bugs lacking it.

FYI, Canonical hired the world's #1 compiz developer for two years, and he did mountains of bugfixing and improvements in Compiz between 2012 and 2014, so they obviously do pay attention to bugs.



> **jeremy9856 said:**
> - When you install Ubuntu and you just select a partition, the installer uselessly want to resize the partition

Research why - that's your contribution as a community member. The parted application resizes the partition for a reason. If it turns out the reason is a bug, then (and only then) please file a bug report. Explain the problem (good thing you researched it), and it will be fixed.


> **jeremy9856 said:**
> - Allow changing display frequency from display settings

Why? The frequency is usually autodetected properly. X developers spent years getting autodetection to work - and eliminated myriads of config file tweaks in the process. I don't miss those old days one bit.

In other words, seem to be asking for a workaround - to disable autodetection. Why?

If your hardware is not autodetected properly, it's usually because you have faulty display hardware that reports wrong values. Please file a bug report with the details of your display hardware so X developers can craft a patch. To set the frequency as a workaround to faulty display hardware, see [http://askubuntu.com/questions/59621/how-to-change-the-monitors-refresh-rate](http://askubuntu.com/questions/59621/how-to-change-the-monitors-refresh-rate) .
