### Post by jespdj on 2008-08-25
Why did nobody mention [Gentoo](http://www.gentoo.org/) yet? It's a source-only Linux distro, which means that when you set it up, you are compiling all the software yourself on your computer. The advantage of that is that everything is optimized especially for your hardware.

On the other hand, you really have to be an expert to set it up, or be willing to spend a lot of time learning how to set it up.

> Gentoo is a free operating system based on either Linux or FreeBSD that can be automatically optimized and customized for just about any application or need. Extreme configurability, performance and a top-notch user and developer community are all hallmarks of the Gentoo experience.

### Post by Max Uglee on 2009-08-11
Puppy, at least on older hardware.  Puppy runs from RAM and the devs that build it's packages are really good at stripping things down and making them as fast as possible.  It does take longer to boot though because it has to copy everything to RAM and as someone mentioned earlier power outages can be devastating.  It also does not have nearly the packages that synaptic can provide.  There are a lot of spinoffs called puplets (much like the *buntus).  My favorite is Boxpup because it uses openbox instead of IceWM or JWM, and it uses PCmanFM (as a filer) instead of Rox.
 
Slackware is pretty quick and doesn't seem too hard to use.  I have not used it a whole lot but it is on my list of distros to really try out.
 
I have not used Gentoo, but I imagine that it is pretty quick.  I still doubt that it is as quick as Puppy.  Although, as someone already mentioned, it probably would be the ultimate fast distro if it was setup to run from RAM like Puppy.  This is also assuming it was set up by someone who REALLY knows what they are doing.
 
For ease of use I always come back to the *buntus or Debian.  Mainly because of synaptic and hardware support.  I would say Crunchbang is the best lightweight version of Ubuntu.  Forget Xubuntu, it is not much faster than Ubuntu, it uses mostly Gnome apps even though it is XFCE.  You would be better off doing a minimum install and installing XFCE yourself.  
 
I have heard that Fedora and Suse are slow.  I have not tried either.
 
Also older hardware sometimes works better with an older Kernel while newer hardware works better with a newer kernel.  The 2.x series of puppy runs better on PI's and PII's while the 4.x series runs better on new processors.  Things like SMP and 64bit make a difference as well.  Some software is not optimized as well for 64bit while some is much faster on 64bit.  An SMP kernel will run on a single core processor but it is questionable whether it hurts the performance.
 
Then you have proprietary vs open source video drivers.
 
So my answer is gentoo run from RAM setup correctly for all out speed -boot time.  Then Puppy.
For a good balance of ease of use/speed go with a *buntu or Debian if you super stability.   If youre running a PIII or Athlon from that era use Crunchbang or Debian with LXDE or XFCE and Thunar or PCman,  If you have a multicore processor and loads of RAM I prefer Mint the way it comes and I like to add Cairo Dock and Compiz (definatly not the fastest, although on my Phenom 2.6 GHz, 4 GB DDR2 @ 1066, and GeForce 9500 I can't tell a speed difference between it and crunchbang).  On my PII Samba server I run Ubuntu 8.10 with IceWM and most services turned off.  It gets the job done and is easy to configure.  I set that up awhile ago and if I were to rebuild it I would probly use Debian with LXDE or Openbox.  Puppy is not very good as a samba server.
