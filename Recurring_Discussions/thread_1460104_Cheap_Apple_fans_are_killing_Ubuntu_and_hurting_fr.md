### Post by Mr. Picklesworth on 2010-04-22
> **Tharmas said:**
> Around Jaunty I started to get uncomfortable with some issues that made me agree reluctantly with my Fedora and Debian friends:
- Ubuntu took a lot from the community and returned considerably less
- Launchpad was proprietary.
- Landscape is proprietary.
- Ubuntu One is proprietary and uses the Ubuntu name. 
- Ubuntu One Music Store *sells* MP3 (not Ogg or flac). It even has wma.
- The Software Center is going to have proprietary software for sale.
- The notification area will be nuked and we don't care about what you or other distros or developers think of it.
- We will have a default theme with the buttons on the left side of the windows and we don't care about what you or other distros or developers think of it. And on top (under) of it, we'll have a purple wallpaper. Because that's the will of the SA.DFL.

Launchpad is proprietary no longer! Granted, it took a while, but the sign-on service is open source now, too. Sometimes this stuff takes more work than is immediately obvious; software licensing is tricky business.
Ubuntu One, though... well, I think of web services along the same lines as hiring a person to paint my house, but I can see where others will see this differently. KDE's OwnCloud looks super impressive, so it would be interesting if the Ubuntu One team (or some outside contributors) could get the U1 client to work with it.



The U1 music store sells music through 7digital. I think a lot of people can respect Ubuntu's pragmatism (the same spirit that gives us Ubuntu's awesome hardware support) in not waiting indefinitely for 7digital to do ogg for everything. It's really not going to happen any time soon. Besides, portable media players often lack support for ogg (and unanimously support mp3), so that isn't doing much good for the people who are paying for music.



Software Center will have proprietary software for sale because this is an open platform, and part of an open platform is accepting that not everybody does things the same way. It's that, or we force proprietary software developers to distribute things in ugly ways, hurting our users in the process (and giving the developers fine reason not to bother).



The notification area is being very slowly and methodically deconstructed, and I think developers are generally in agreement that it sucks and a redesign is in order. It's just an ugly system that everything (even Windows) is pulling away from. Personally, I think Ubuntu's design team has been handling this, and the notifications, quite well; it's about giving the functionality a single defined purpose instead of "this system lets you put a window on the top right of the screen. Have fun!"

It's also about interoperability. The notifications system is meant to support notification daemons with different capabilities (like notify-osd and Maemo's notification daemon). Applications are meant to tune their notifications based on those capabilities. Similar deal with indicators: the system tray idea creates some ugly limitation in what kind of applications you can reasonably run on your desktop, and what kind of desktop you can run those applications in.

On that spirit, the window button position is defined by the theme you are using, instead of a separate bit of configuration. If you select the Ambience or Radiance theme, the theme puts your buttons in a certain order on the left. A theme that mimics Amiga OS may define a different button order that suits it. The gconf default (and thus the button order for most themes) remains menu:minimize,close,maximize.

KDE itself has driven the Status Notifier spec (though I'm not sure if we're in harmony right now — Ubuntu 10.04 adds a lot of stuff to the design) while Gnome Shell is also killing the notification area in favour of the message tray and the top right of the screen reserved for very specific, very special system information things. (See: most cell phone operating systems).

I really don't see how this all connects to Apple fans, although the icons are pretty Mac-like in that they are symbolic and have dark outlines. They're also Maemo-like, WebOS-like, Android-like, Moblin-like and Windows 7-like :)



The current version of MacOS X does not have a purple wallpaper. The only one I recall that did is 10.5, which was a pretty short-lived release. The current one has a picture of a snow leopard surrounded by snow.

### Post by DeadSuperHero on 2010-04-22
This is probably the most cynical thread I've read in a while. 

I personally really like the changes that are coming to Ubuntu; I think they address all sorts of usability problems that have plagued the Free Software desktop for ages. 

Their [Ayatana project]("https://launchpad.net/ayatana"), for example. Sure, it needs work. What have we gotten out of it:

  -Clean indicator applets that all behave in a unified fashion, and excellent documentation on how to add one to your own application to boot.
  
  -The availability of commercial/closed software in Software Center, allowing for things like commercial linux *games* to be easily purchased by Ubuntu users. This is important because it thrusts those otherwise unnoticed games into the spotlight and helps stimulate gaming on Linux in general.
  
  -A slick notification system that is less invasive than the traditional Gnome one. 

Then we have the "[100 Papercuts]("https://launchpad.net/hundredpapercuts")" project, which cleans up applications, system settings, icons, fonts, and interfaces for the sake of making things easier to use. I personally think this is a fantastic goal to have, and Gnome certainly could take advantage from all the little bugfixes here and there.

I'm not the biggest fan of the U1 Music Store seeing as it only uses MP3's, but I have a [petition]("http://www.petitiononline.com/ovsi7ds/petition.html") for that. Maybe the devs will look into supporting OGG at some point?

And finally, they're [making it even easier]("http://www.jonobacon.org/2010/02/04/project-awesome-opportunity/") than ever for budding developers to create Ubuntu applications. What's not to like about that? (Project AO entails things such as [PyJunior]("https://launchpad.net/pyjunior"), [Acire]("https://launchpad.net/acire"), [Quickly]("https://launchpad.net/quickly"), and so on)

So, let's review:

-Easier for desktop users. Check.
-Easier for developers. Check.
-Easier for media lovers. Check.
-Comparable to MacOS and Windows. Check.

So, what are you complaining about exactly? They get rid of the System Tray, have some click-through notifications, put some commercial software in Software Center, move the window buttons on the left for their default theme, and by default support MP3's through U1 Music Store.

You act as if they'd committed an act of high treason.
