### Post by Dragonbite on 2010-02-02
Right now I have all 3 on my laptop (well, actually they are split between 3 hard drives easily removed and slid in).

[COLOR="DarkRed"]**_Ubuntu (9.10)_**[/COLOR][LIST][*]Stable and easy to set up, in part because of my familiarity as it is still my primary Linux distro at this point.
[*]I actually like the way Ubuntu looks for the most part.
[*]Slightly easier to use with tweaks such as handling plug-in installation for Firefox (flash, etc.)
[*]Hardware detection good, with the auto-restricted hardware driver download/installation making it easy to set it up
[*]If you run across some new gee-whiz (desktop) application there is a greater chance of it being packaged as an Ubuntu .deb than any other format (Fedora/Red Hat or generic .rpm probably comes in second, openSUSE .rpm comes in last)
[*]Upgrade process easier
[/LIST][COLOR="Blue"]**_Fedora (12)_**[/COLOR][LIST]
[*]Strong hardware detection though doesn't rely on proprietary drivers all of the time. For example Fedora 12 includes OpenFWWF, which detected and made available my Broadcom wireless out-of-the-box.
[*]Crashes most often result in the application being closed, but can be re-opened. I haven't run across a system-wide crash that resulted in hard-rebooting.
[*]Bug-reporting tool is very easy to use, and makes me feel I am "giving back" to the community.
[*]Proprietary codecs and drivers can be a little bit of a pain, though RPMFusion makes it a little easier.
[*]More FOSS and Open Source passionate (FOSS is Open Source with Freedom, they can be different), promoting the FOSS and using it to circumnavigate issues instead of conceeding with providing proprietary alternatives, re-shackeling Linux to closed-source
[*]DVD allows for picking and choosing what is installed, while still providing for LiveCD and LiveUSB installations
[/LIST]**_[COLOR="Green"]OpenSUSE (11.2)[/COLOR]_**[LIST]
[*]Looks really good, even the Gnome version
[*]Yast makes managing things pretty easy. Kinda like the consolidated Control Panel in Windows, as opposed to a menu list
[*]Innovations such as the slab menus.  OpenSUSE included the slab menu that KDE 4 incorporated back when KDE 3 was the only version available.  The Gnome slab takes getting used to, and I am not overly happy with it yet.
[*]The Gnome LiveCD includes the Mono applications (Banshee, F-Spot, Beagle, Gnome-Do and Tomboy) and they all work pretty well and are up-to-date.
[*]DVD allows you to pick-and-choose what to install while there are LiveCDs for Gnome and KDE versions
[*]My version is a little less stable than Ubuntu or Fedora, but YMMV (actually saw that on a license plate coming in to work this morning :) )
[*]Most applications not listed in the openSUSE repositories can be found in 3rd party repositories such as PacMan.
[*]On-Click installing options available from the openSUSE website which installes everything needed for whatever purpose. May not work all of the time, but does work many times.
[/LIST]

### Post by TBABill on 2010-02-02
I have distro hopped across many, trying to choose the right distro for me. I ran Karmic, SUSE 11.2 (both KDE and Gnome - both 32 bit) and Fedora 12 (Gnome).
 
Karmic - installs easily enough, runs great on all my hardware and "just works". But, it runs flash worse than Fedora (slower) and much slower than OpenSUSE. SUSE KDE is faster with flash applications, particularly video playback and web page scrolling, than SUSE Gnome. 
 
Karmic just doesn't crash much and is easy to like and use daily.
 
OpenSUSE 11.2 - in KDE it is pretty, fast (snappy may be a better term for it) and easy to use. RPMs are as easy to use as DEBs if you get used to them. SUSE handles dependencies for you, just like Ubuntu, so the file system really feels like a moot point. It's software package manager (Yast) is easy to use and a Windows user could as easily adjust to SUSE as they can to Ubuntu. Running apps is easy, getting apps is easy with large repositories and overall functionality is similar, just laid out differently and managed a bit differently. 
 
Fedora 12 - I didn't like it in Gnome or KDE. It was pretty, but beyond that it was more challenging to use non-free drivers and apps. It crashed pretty regularly (X) and updates sometimes conflicted and causes systematic failures. For someone newer to Linux like me it made running it more of a challenge than the time I was willing to invest to fix the system. It ran Adobe Flash better than Ubuntu, but not a lot better. Mandriva 2010 and OpenSUSE 11.2 (KDE version) both perform best on my 2 laptops.
 
I really like Ubuntu but I, like most others, anxiously await GPU acceleration in Flash like they have implemented in Windows and Mac. Once that's a non-issue I'll probably agree that Ubuntu is as good as SUSE for my purposes.
 
This wasn't about Mandriva at all, but I have to say it's even easier than SUSE to use and the performance is better yet with Flash. I put Swiftfox on Mandriva 2010 and there are no issues. For me that's important, but others may not care about that capability, which would put the distros much closer together in performance if I factored it out. For me, Ubuntu is just easy, SUSE is pretty and quick, Fedora still needs some work toward stability and Mandriva is fast.
