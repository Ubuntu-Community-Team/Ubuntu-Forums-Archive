### Post by mivo on 2009-11-01
> **Penguin Guy said:**
> Archlinux: You have to setup *everything* :(.

Apparently you completely missed the philosophy of Arch.

The installer sets up a base system for you, fully working and based on your input. The extras are manually set up, deliberately, so that you get exactly the system that you want. Tailored to your needs. You end up with a high performance Linux system, rather than a bloated product that Ubuntu has become.

It does take longer, though there is a great beginner's guide that spoon-feeds you all commands and explains stuff to you. In return, you won't have to format your / every six month when a new version comes out, because Arch has no new versions: it is a rolling distro, and updates actually work. In the long run you save a lot of time and get a better performing Linux system that will be fully transparent to you at all times.

No, it isn't for anyone. It requires a little effort and the willingness to read the provided materials and use the really excellent and complex wiki.

OK this is from a reasonably old experience (10.0 -> 10.2) so it likely doesn't accurately reflect the current status of OpenSUSE.

Where I work, some of us (well, me) wanted to move from developing on XP to developing on some form of Linux. At the time I was using Ubuntu at home on and off, but my boss was an avid OpenSuse supporter. He agreed I could install Linux but it had to be OpenSuse + KDE because it would be easy for him to help, support me etc. Over time other users moved to OpenSuse, but it wasn't a smooth ride. 

Package installations were the primary frustration, Yast was incredibly slow, and would insist on loading package lists every time we attempted to install software. Much of the time the package simply wouldn't exist in the repositories, meaning searching around for an RPM, and 90% of the time that meant dependency hell trying to install it. Another issue was (and this likely was the way that my boss configured it) the menus were a nightmare to navigate, there was so much duplication in applications.

After that particular boss left, I was able to install Ubuntu instead, other members of staff continued struggling with OpenSuse, where it was a weekly occurrence for me to sit with them figuring out dependency issues. In the end we all switched to Ubuntu and everything ran smoothly. 

I'm aware that package management has improved since then, but the sheer amount of time we wasted using that operating system keeps me from trying it again.

EDIT: Oh yeah, and the whole Novell/Microsoft deal has kept me well away.
