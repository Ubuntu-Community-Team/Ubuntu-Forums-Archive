### Post by ron177 on 2010-12-25
Dear all,

I am a recent Ubuntu user: it's been roughly a few weeks since I installed Ubuntu on my Toshiba laptop. I am quite happy with it. I also own an iMac which runs primarily on Mac OS X (Snow Leopard). At least for now. 

I wonder some of you who have made a transition from OS X to Linux or who know about both systems well can chip in and compare the advantages and disadvantages of each operating system. I am specifically interested in a comparative discussion of various things each operating system has to offer: things such as user-friendliness, productivity softwares (iPhoto, iTunes, GarageBand, Mail, vs. Ubuntu equivalents), networking, security platform, stability, ease of sharing with others from a different platform, accessibility in terms of being able to install the OS software on different hardwares, and so on. 

Cost is also an important factor: whereas OS X has a price and has to be installed on a super-expensive Apple hardware, and whereas Mac softwares are mostly not free, Ubuntu is free and can be installed on many different hardwares. 

The downside of a free software community is however the limited number, the limited quality, or the limited diversity of softwares you can find in it. Whereas OS X has several great and powerful applications for each given task, Ubuntu/Linux seems to have a limited number of softwares created mostly by programmers who themselves originally wished to use them. In other words, if Ubuntu users create a wish-list of softwares they would like to see in a year or two, it is not at the end very likely for those softwares to be created by the community efforts of a limited number of programmers who make softwares in their spare times. Because there is no money involved, there is little incentive to cater to the needs of users. Is this a generally correct overview of the situation? If not, please elaborate.

I also would like to know whether users think Ubuntu behaves in any way like Windows and Mac in the sense that these two other operating systems are designed to lock you in those systems: Once you produce documents with MS Office (e.g., MS Word) or iWork, you're bound to stay in Windows or Mac respectively. Is this also true with Ubuntu? Is working with Ubuntu means that you're locked in that system? Can one migrate easily from Ubuntu to other operating systems (transferring emails downloaded in Ubuntu mail applications, documents created with LyX or AbiWord or Open Office, etc. completely with ease)?

I am not for or against either of these two operating systems. If anything I am quite impressed with both in certain areas and dismayed in others. Readers I am sure would like to go through dispassionate, rather than biased, responses. And if you could please identify the extent of you experience with Mac and Ubuntu, everyone I am certain would appreciate that.

Cordially,

Ron

### Post by libssd on 2010-12-29
> **ron177 said:**
> My experience with terminal has been quite frustrating. I know if you know it well, it will give you fascinating features that you'd rarely find in other operating systems. Well, that's not true, at least Mac OS X has also a terminal, no? You can also do things in the Unix terminal of OS X.
Mac OS X is based on a variant of Unix, so under the hood it looks a lot like Linux. And, like OS X, you can do almost everything in Linux without ever resorting to terminal (I suspect that most Macintosh users are unaware that it even exists). For that matter, Windows also has a command line user interface, although I've always found it more frustrating to use.

I believe that terminal is used more in the Linux world for three reasons:

[LIST=1]
[*] Linux tends to attract people with more of a hackerish outlook on computers, who want to understand how things work (like you).

[*] Macs "just work" because OS X runs only (for all practical purposes) on hardware designed by Apple, while Linux is rarely shipped pre-installed, and "some assembly" may be required to get it working properly.

[*] When problems arise, they are generally easier to report and diagnose using technical details from the terminal interface, and to fix using cut and paste solutions than it is to describe what's going on in a graphical user interface. (When I was in computer support, we had a saying, "Users often lie about what they are doing -- not intentionally, but by leaving out critical details.) There are a few things (such as OS tweaks) for which I **must** use terminal, but they are relatively rare. Almost anything that can be done through the graphical user interface in Linux (and in Mac OS X) can be done at the command line -- the GUI just provides a more convenient means of doing so for most people doing most things. Think of the GUI as an automatic transmission and the CLI as a stick shift. Most people these days don't know how to use a stick shift, but they both do essentially the same thing; the latter just gives you more control.
[/LIST]

A good tutorial on command line usage in terminal is available here:  _[https://help.ubuntu.com/community/CommandlineHowto](https://help.ubuntu.com/community/CommandlineHowto)_

Once you have mastered this tutorial, check out the other links at the bottom of the page.
