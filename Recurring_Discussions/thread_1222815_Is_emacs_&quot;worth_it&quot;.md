### Post by prvteprts on 2009-07-25

### Post by dbd on 2009-07-25
I'm a really big fan of emacs, I'd wholeheartedly recommend learning it. I'm so in love with the shortcuts for navigating text, that whenever I write text of any significant length (emails, forum posts, whatever) I always write it in emacs, and then copy it over afterwards. Most of my text editing is latex code or lisp code (so I'm a combination of the above two posters), and emacs is particularly good for both of those, so perhaps I am overly biased in emacs' favour. But even when not writing in either of those languages I find it really handy.

Also, I'd question if it is really that hard to learn. Sure, once you first start it is hard, but once you get the hang of the basics, and the `emacsy' way of doing things, then learning how to do new things is pretty easy, the documentation is superb. Of course you have to be the type of person that enjoys continually learning better ways to do things, and get excited by flicking through the documentation every now and then, but if you are then it's pretty much perfect. Just make sure you write down everything you learn somewhere, or you are bound to forget half of it a few days later :)

### Post by jpkotta on 2009-07-26
I'm a (GNU) Emacs user, and have been for about 2 years.  Other editors are easier to learn and sometimes easier to use, but Emacs is so flexible and has a large enough user base that almost anything you want from it is already done.  If you have a minor problem with a mode you can usually modify it to your liking.  Also, I consider it fun to tweak, but that means it's probably a net loss of productivity for me.

I've only ever bothered to learn enough VI to shoot myself in the foot.  I see the potential, but I made my choice based on extensibility.

It's true that I often C-x C-s in every application when I want to save, and this can cause trouble, especially in OO-Calc.
