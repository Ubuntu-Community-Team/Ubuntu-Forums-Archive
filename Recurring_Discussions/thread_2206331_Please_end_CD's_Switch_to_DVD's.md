### Post by grahammechanical on 2014-02-18
What does any of this have to do with Ubuntu or this forum?

For a couple of years the Ubuntu ISO image has been too large to fit on a CD. But that is no reason to inctrease the image size to take up the complete space on the DVD disc.

There may be licencing reasons why proprietary audio and video codecs are not part of the ISO image. During install there certainly is a warning that if we install third party software then we are taking responsibility for any possible legal infringement that might occur in the country we live in. The owners of some proprietary software do not allow the packaging and distribution of their software although they allow the free downloading, installing and use of it.

One reason for keeping the ISO image size as small as possible is the lack of a fast data transfer speed over the internet in many lands. A very large ISO image would take longer to download.

Every ISO image is an image of the Ubuntu code as it was at the release date. We would still need to download and install any updates and security fixes that have been applied since the release date.

The little I know of VDPAU tells me, that increasing the size of the ISO image would not necessarily make any difference to VDPAU support in Ubuntu because VDPAU support has to be in the video driver. What you demand would, in my opinion, mean the ISO image including any number of video drivers for any number of video cards. It would make building and testing the Ubuntu image much more difficult.

Notice the problems that the developers of the open source video driver, Nouveau, are having in bring VDPAU support into Nouveau.

[http://nouveau.freedesktop.org/wiki/VideoAcceleration/](http://nouveau.freedesktop.org/wiki/VideoAcceleration/)
