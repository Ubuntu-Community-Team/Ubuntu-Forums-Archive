### Post by NightwishFan on 2010-08-13
Think about the goals of Ubuntu after all. Debian tends to try to run like a rock, but what if you need a new feature or hardware support.. It is not logical to completely rely on the 'backports', especially for mass deployments. A rolling release makes it hard to target even 'in-house' software developed for the systems you manage.

Ubuntu tries to be a stable up-to-date snapshot. It has clear defined goals. The developers think very hard when adding new features. It allows the latest of greatest to work well together.

The LTS release is a bit of a compromise, and the 'point releases' are quite stable.

### Post by XubuRoxMySox on 2010-08-13
To me "unstable" means what it means in Debian terminology: Unproven, more experimental (even though Debian has an "experimental" version beyond "Sid"), and somewhat buggy for some users.

It also means that perfectly good software might be disabled or corrupted by updates. 

If you look around these forums and see the high number of "borked after update" threads you kinda get the impression that Ubuntu - at least while the current release is newer - is "unstable." An update ruined my video in Xubuntu 10.04, for example. After an update, the monitor was "put to sleep" on boot-up and no input from keyboard or mouse would wake it up. But a re-install fixed it and I refused all but security updates. So far so good.

That's why my advice to Ubuntu users is generally:

**[COLOR="Purple"]1.[/COLOR]** Stay at least one release behind the newest version. They're all supported for long enough to overlap each other anywayz. Let the newest version get all the bugs worked out while you enjoy a trouble-free system that has already done so. And

**[COLOR="Purple"]2.[/COLOR]** Accept only security updates or at least be verrrrry selective about which ones you accept!

That said: Sometimes updates actually *fix* stuff that's broken! So if everything in your system works on a fresh install, then leave it be (accepting only security updates). If stuff doesn't work, try accepting all updates because sometimes they actually work like they're supposed to! :o

Older but still-supported version of Ubuntu are plenty stable on most hardware IMO. If you don't like testing and bug-reporting and trial-and-error, and you just want an OS that works and lets you do the stuff you bought a 'puter for, use the earlier still-supported versions. I think that goes for any of the Ubuntu spin-offs (SuperOS, Linux Mint, UberStudent, etc - there are about four hundred zillion of them) as well. Which begs the question: Would there really be so many Ubuntu-based distros if Ubuntu were fundamentally unstable?

Just say'n,
Robin
