### Post by Sematary on 2010-03-15
I installed Ubuntu several weeks ago and have learned a ton of things about linux and my own computing skills. I consider myself fairly computer literate and I have to say, this is has been a real learning experience.

I really like my Ubuntu machine but I have to say, Linux isn't ready for the average home user..... yet.

Until people can install this and have everything or at least MOST everything work without having to actually think about it, this OS will remain the exclusive domain of professionals, true computer users who WANT to know how their computer works, and those who are simply sick of Windows and Microsoft.

There are certain things that will need to occur:

1. People will need to be able to go to a download location, download then install without compiling, any software they choose.
That simply isn't possible yet. The package manager is nice but there are other downloads out there and some of them have dependencies which need to be installed separately and stuff needs to be compiled, etc...
WAY too much work for the average user.

2. Better driver support from OEM's
I just spent the last two days trying to find a solution to getting my Canon ip2600 installed and working. I eventually did find several steps and a "dummy" download of libcupsys2 along with extracting the specific driver file and locating the ppd file manually to get it to work. This was a time consuming and frustrating task that ultimately ended well but the average user wouldn't bother and would simply write off linux as being unusable.

3. The terminal has come in to use more times than I can count while I have been installing software, etc... The average user simply is either too lazy or too computer illiterate to even know what a terminal is let alone use it.

4. I would LOVE to make Linux/Ubuntu the main OS on my regular computer but I am very dependant on that machine for everyday tasks that I do, including playing certain games like Doom3 or Quake4, etc... and a host of other things that I haven't figured out how to make Linux do  yet. Among those, of course, is installing software packages that aren't compiled. 

I still have alot to learn and until I do, the Windows machine is my go to machine and the linux machine remains a learning tool and a tool for my youngest daughter to play tux educational games on. 

I'm no developer, but it seems to me that if the developers of the various flavors of Linux truly want to challenge Microsoft, these OS's are going to have to become much more user friendly.

Just my two cents from a fairly experienced computer user (going back to ms-dos - which I also have installed on another machine, btw) who lives with "average" computer users.

### Post by 3rdalbum on 2010-03-15
I honestly don't know WHY people think Linux isn't ready for mainstream use. So many "ordinary" people already use it without issues. It's like claiming that TVs are too difficult to use for the average person.

1. Any application software that is worth downloading is either available in the Ubuntu repositories, in a third-party repository, in a Debian package or in a binary installer. The preferred method is a repository of some kind, and Ubuntu's PPAs have caused an explosion of software packaging; to borrow Apple's catchphrase, "For anything you can think of, there's a PPA for that".

2. You were never a Mac user, I can tell. Back in the olden days (less than ten years ago) there were almost no printers on the market that supported the Mac. Nobody stomped their feet and proclaimed that the Mac OS would never be ready until more OEMs supported it - instead, they bought printers that did have support.

Even today, Macintosh users have to vet hardware purchases very carefully. If they want a non-Apple MP3 player, or a non-Apple webcam, or a new Wifi card, they have to pick carefully.

It's getting to the stage where Linux users can almost pick up a device at random and it will work, or can be made to work. Until this statement is 100% true, Linux users will buy hardware that they've checked for compatibility.

3. Common misconception: Most of the time you don't need the terminal. But some things are quicker or easier to explain with the terminal. Unless you're working in extraordinary circumstances like badly-compatible hardware or a borked system, you might never touch the terminal.

Besides, it's not like the MS-DOS prompt ever stopped Windows from gaining popularity over the terminal-less Mac, eh? And it's not like you can solve problems on Windows without running some horrible cryptic "rundll32.exe" command.

4. If you don't like Linux, use something else; but don't start telling people that Linux is not user-friendly*. And in return, I'll stop telling people that they shouldn't buy TVs because they're not user-friendly.

* User-friendly does NOT mean "works exactly like Windows", which I think might be where you're getting some of your problems from.
