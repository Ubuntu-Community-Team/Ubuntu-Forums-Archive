### Post by bodhi.zazen on 2011-07-21
> **hodad said:**
> Seems I run into another logjam every day with Unity.  I guess I just need to be patient, but I'm going back to Classic for now...

You need to learn to balance your desire to run the cutting edge stuff vs stability.

If you want a system that "just works" go with Ubuntu LTS, RHEL, SUSE, Debian (stable), Slackware, Scientific. The packages will be older, but most (not all) of the bugs worked out.

If you stay with Ubuntu LTS, wait 6 months to update, it is NOT more stable at the time of release, it becomes more stable as the packages "mature" and bug fixes are released, and then it is supported for longer.

If you want to be on the cutting edge , you get to run Unity. How many distros are running unity right now ? Ubuntu was designed to be an alternate to waiting for the next Debian Stable release - with newer software, more frequent release cycles, and the bugs and headaches that go with it.

How about Fedora ? How many distros are running gnome 3 ?

So if you want to be on the bleeding edge you will have to accept some bugs. The advantage is you get to test and report bugs, make improvements, etc.

If Ubuntu is too cutting edge for you (too much hassle, too many bugs), try a more stable distro, including Ubuntu LTS. You can not have your cake and eat it too.

### Post by murderslastcrow on 2011-07-23
Gnome-panel and metacity are still available in GNOME 3 (Fallback mode, as everyone has mentioned). They have both undergone updates that speed them up and remove bugs and dependencies on deprecated libraries from GNOME 2. So it's really slick. Of course, only the official GNOME applets have been ported, although there is a lot of documentation to help developers port applets to the new gnome-panel.

If that's not good enough for you (I mean really, you should give it a look-see since it's much faster than the previous gnome-panel and metacity), you can always use Xfce 4 which has Xfapplet, allowing you the use almost all GNOME 2 applets in Xfce. Most good themes have an xfwm theme as well, so it should be just like using GNOME for the most part- not that Xfce doesn't have some of its own applets and features that aren't amazing in and of themselves.

There's very little to lose with either option. But if customization is what you care about, KDE is your man. You can use GTK 2 themes, emerald themes, and hey even plasma and Qt 4 themes (go figure). KDE take configurability very seriously, and they take the opposite stance from GNOME. They think that the little bit of extra information and functionality is essential to empower people.

They tend to stay pretty sleek and simple despite this, and you can always push it a little further yourself. So pretty much everything you mentioned as an alternative would be a good idea. GNOME 2 will probably have a metapackage of its own until the applets get ported to the new gnome-panel, of course (in some distros- maybe not Ubuntu).
