### Post by mooreted on 2015-02-01
The Unity Dash is confusing for new and seasoned Linux users alike. When opening the Dash for the first time the user sees nothing at all. That is the most used applications “lens”, as they call it. You have to click on the icon at the bottom of the dash window (lens) that shows your applications. This section has a truncated list of your installed applications mixed with suggestions from the Ubuntu Software Center. It’s very difficult to differentiate between installed applications and suggested applications. If you decide to search for an application you will get a list of installed applications, Ubuntu Software suggestions and suggestions from Amazon. To “help” with that there are filters you can use to narrow your results on the right side of the lens. This is not an improvement over the normal, categorized menu we have all used on desktop computers for the last 15 years.

Users can’t move the Unity launcher which can be very annoying on dual monitors. There should be an option to move it to different edges of the screen. This may not be useful on a phone or tablet, but on a desktop computer it’s a needed feature.

The global menu is difficult to find, often too far from the parent window and hides until you move your mouse cursor over it. Don’t twitch, you’ll lose your spot. It’s also difficult to tell which open window the menu refers too. Try opening Gimp, a note program and a browser and you will see how confusing it can become.

Not only does KDE have some nice features like the Plasma desktop that allows you to visualize directories right on your desktop, it also can be configured completely to suit your needs. The tried and true desktop paradigm still exists with a bottom bar (that can be moved), system tray with notifications, a searchable, categorized application menu which can be switched to an old style Windows XP type of menu if you prefer.

KDE is not trying to be all things to all people. It’s not catering to phones and tablets. Lets face it; even though everyone has a phone, almost everyone also has a trusty desktop or laptop computer at home for actually getting work done. There is still nothing better than a mouse, keyboard and a 24&#8243; monitor or two when you want to actually create something.

### Post by buzzingrobot on 2015-02-01
Well, there is this: [https://www.kde.org/workspaces/plasmanetbook/](https://www.kde.org/workspaces/plasmanetbook/).

We all are free to choose styles and approaches we prefer. That said, I'm always surprised that a horizontal row of icons along the bottom of a screen is seen by some folks as the essence of "productivity" (What is it that people are producing all the time?) while a vertical row of the same icons on the left or right side of the screen becomes a thoroughly incomprehensible conundrum.

Pretty much the same for menus.  Linux desktops are home to hundreds of executable files. A fraction of those files can be accessed via the menu, whether that menu happens to be an old fashioned text-based hierarchical table cascading horizontally across the display, or the same menu with the same text adorned with icons and filling up the display. 

However the menu is structured, it is a standard in Linux that "applications" are paired with ".desktop" files that reside in /usr/share/applications. It is those files that make up the entries in our menus.  The majority of executables aren't in those menus.

We all make the easy and misleading equation that what we like and what we know is *the* best solution.  There is no "best" desktop design and no way to measure if one desktop style is any better than another.  All of them rely on icons, mice or touchpads, and keyboards, and, hence, they are all constrained to be much more alike than different.

(Personally, I've always relied on a dock to house the icons of the apps I use frequently (a single-digit number, as I suspect it is for most users). Once I've configured a machine, I seldom use the menu/Dash/Overview/etc. I avoid desktops that don't permit switching workspaces via a mouse wheel scroll. I'm interested in what KDE is doing, but seldom keep an install going because I use no KDE applications, so it becomes, for me, in effect, just another panel-and-menu interface with a great deal of baggage. Minimizing windows to free up desktop space is a Windows-ism.)
