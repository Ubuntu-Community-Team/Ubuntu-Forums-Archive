### Post by hibliss on 2009-11-30
Let's take a few things. 

1) Many people have given you input on the IE statement. IE sucks, it is a terrible, insecure, non compliant browser. It has always been that way. It doesn't help that it is a huge target for hackers of course, but if it were open source or allowed open source plugins, the security holes would be patched much faster than Microsoft could ever patch them. Get Firefox, add 2 plugins (AdBloack Plus and NoScript) and it is already more secure than IE will ever be. Allowing JavaScript globally in a browser is a huge exploit, and the source of the majority of Windows Viruses (the popups that say "You Are Infected, Click Here to Fix").

2) With the needs and wants of many users changing, and many things moving online (in to the "cloud") what is MS doing to adapt? Ubuntu has incorporated the Eucalyptus Cloud Computing system. I only mention this specific addition to the latest version to show that Ubuntu moves with the times, while Windows responds to fixing simple complaints like aesthetics and accessibility problems in their OS. 

Microsoft spends so much time patching their OS after release to make it or keep it stable because the closed source structure and proprietary model means less qualified people to help patch.

More time patching means less time to develop new and intuitive features. This means their "new and intuitive" features are usually just ripped off from other OSes.

3) Just because Microsoft is the industry leader doesn't make it the best. There is no real definition on what makes an OS the best. All users have different needs, so what is best for one is not best for another.

Example, what does Windows have to offer me for my Netbook? Some scaled down limited version of their OS with core features that I need and use stripped out? No RDP, no way to join a domain? So business users can't have netbooks?

This is one example, but a glaring one of the way Microsoft's tiered OS structure (Home, Media Center, Pro for XP, Basic, Home Preium, Business, and Ultimate for Vista and 7) limits what the end user can do.

With open source, the solution is not always easy, or there may not be one, but you are not limited by what someone else decides should be in your OS.

### Post by koenn on 2009-11-30
Linux is pretty old, so the guy has a point. The kernel is 'old fashioned' monolithic, while the Windows NT kernel was originally a microkernel, and Darwin is also build around a microkernel, which is supposedly more modern (GNU HURD  is also  microkernel-design). 

Actually, NT (i.e. the "New Technology" that everything from Windows NT to Windows 7 and Server 2008 build on) were microsoft's attempt to build a Unix-like Operatig system without Unix' flaws (eg a filesystem where a file is more than a bag of bytes, etc). Linux is/was much more conservative in its design and stuck a lot closer to 'old' Unix, although over the years, innovation crept in. 

So there's truth in saying that Linux is old (in concept, in design, ...) and that Windows and OSX are a lot more modern.
