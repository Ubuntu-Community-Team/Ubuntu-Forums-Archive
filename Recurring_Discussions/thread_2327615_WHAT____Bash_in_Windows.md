### Post by QIII on 2016-07-01
Perhaps this will make some sense to others who happen across this bit of cognitive dissonance:

Microsoft built a new infrastructure inside Windows called the Windows Subsystem for Linux.  Inside WSL a user can run a user-mode, command-line-only version of Ubuntu that Canonical has created.

So, Windows users can now run typical Linux command line tools (well, not everything yet) like awk, grep, sed, etc.  You can develop with scripting languages like Python.  You can access your Windows file system.  But you can't interact with Windows applications.  So you can't run Textpad from the shell.  And you can't go the other way around, either.

It's not a server platform.  For that you will need dedicated Ubuntu servers on their own machines. If you want, you can deploy Ubuntu on Azure.  I believe Ubuntu is the most popular OS hosted on Azure at the moment.

This is really a tool for Windows developers exploring Linux according to Mark Shuttleworth.

There is absolutely no need to get wrapped around the axle about this.  Ubuntu is not disappearing into Windows.  You won't have to buy Windows to use Ubuntu.  Canonical is not being bought by Microsoft.  If anything, Canonical is getting its tentacles into Windows because Microsoft now recognizes the ubiquity of Linux outside of the desktop market..  This is Microsoft admitting a little defeat and eating crow.

The "simple answer":  Microsoft is giving in to Linux -- if only a very little bit.

So can we now be done with this?
