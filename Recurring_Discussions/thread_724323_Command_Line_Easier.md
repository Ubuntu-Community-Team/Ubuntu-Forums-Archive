You don't have to be an expert. Dealing with commandline is just like talking. Know a few keys words and you can accomplish many tasks. 

But it also depends on the task too. I think command line is the best for repetitive non-interactive tasks.

For example, ask your prof, how would you add a series of images to a single pdf with the gui interface...

Try this for example under CLI
convert *.png pics.pdf

That will do the trick. 

Now ubuntu has made this is easy, if you try the convert command (but don't have the program it comes with installed), it will give you the command to install imagemagick. Combined with tab completion, things are easy. 

It isn't really that scary. To achieve the same thing with a GUI would be a pain. You have to go to open office writer, load the images one at time, make sure they line up, and export to pdf. It becomes very consuming if there are a 100 images.

Now of course, some GUI front-end could be crafted to help accomplish the task, but in terms of the options and flexibility of CLI, it would be some task. Type man imagemagick (if you have it installed, and try using man convert too). The options are incredible. 

A simple series or script (who says one has to make it?) can save a lot of time. :)
