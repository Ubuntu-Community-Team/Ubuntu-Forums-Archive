### Post by Nicholas Escalona on 2008-12-24
I'm interested in switching to Linux/BSD of some kind, but no one distro jumps out at me. A quick biography for context:

I am a high school student who's been using Windows since early childhood, and currently runs XP on desktop and OS X on Macbook. I am a relatively advanced Windows power user who probably plays with system settings and the registry too much, but is also well-informed on system security. I use Firefox and OpenOffice. I'm a little obsessive-compulsive when it comes to organizing my files, deleting my family's unused desktop icons, disabling useless processes, and that sort of thing. I am teaching myself Java but am still a beginner, and I have dabbled with the command-line but am not experienced with it.

I have tried Ubuntu on LiveCD and was not immediately impressed. The focus for both Ubuntu, its derivatives, and GNOME seems to be *usability*. However, almost on principle, I would rather have a system where the focus is *functionality*. Compatibility is also nice as long as we're using -lity nouns. I am not afraid of the command-line when it is more effective for a particular task. Though I appreciate open source projects, I have no fundamental commitment to them, and will gladly use proprietary freeware if I find it more useful. No bonus points from me if the system is a Windows clone; in fact, I'll be suspicious whether the interface was chosen for its inherent merits. I intend to dual-boot with XP.

I would appreciate being able to protect the operating system as its own partition, and load the entire thing to RAM on startup. That style of computing appeals to me. I like being given choices rather than cookie-cutter solutions.

Probably the one deciding factor for me, though, is this: I strongly dislike the Windows philosophy of *hide information from the user to avoid confusing them*.

Given this profile, what OS/distro/DE configurations would you suggest for me to try?

I realize I'm asking this at a site devoted to one particular distro, but I have heard that the Ubuntu Forums are unparalleled in the field for their helpfulness.

Arch is a challenge.
If you don't have an extra computer handy, make sure you print out their wiki before you do the install.
The entire beginners guide of it.


The philosophy behind Arch is that once you install it you get a package manager, some coreutils, and a shell. You install everything else yourself (the beginner's guide explains how to do this). This means you have complete control over your system. There won't be anything to hide at all.
You're going to have to be _very_ comfortable with the command line though. Arch, by default, has no GUI. You install that yourself.

### Post by tsali on 2008-12-24
Ubuntu IS an excellent beginner distro in that its popularity ensures you have plenty of help when you run into a beginner issue.

My experience is that forums/irc associated with more obscure distros tend to be a bit less tolerant of "stupid noobs" than Ubuntu. I have, in fact, been told to "do my own homework" in one of these forums. 

If you want a bit more control, I actually like Debian. I use it on my home server. Since you are in learn mode, yes, Debian would be a very good distro for someone with your skills and inclinations. You'll also get some solid insights into FOSS philosphy with Debian. That can be a little muddled in Ubuntu sometimes.

Debian also has LOTS of helps on the web.

To add: Aptitude in Debian will allow you to install desktop environments AFTER the fact quite easily, so you can install a straight up command line system first, then move through a couple of the aptitude "tasks" to install different DEs if you want. Since I wanted a low resource home server that was administered through SSH and webmin, I didn't install the DE at all and have gotten pretty good working with the CLI. Hint: "screen" is a wonderful tool.

Lastly, the availability of pre-compiled deb packages from major vendors can be very helpful. Two good examples: Google Picasa and Nomachine NX.
