### Post by terry_gardener on 2011-04-24
I have now used both gnome-shell and unity and would like to share my thoughts on the subject. I used both gnome-shell and unity on ubuntu 11.04.

First I will do gnome-shell. 

Pros:

1.   The categories at the right side of the overlay (office, internet etc) 
2. Interactive notification system.
3. Not based on compiz. 
4. Looks really nice. 
5. Been able to type to search for apps and then just press enter.

Cons: 

1. You have to press the alt key to shutdown or restart. 
2. Workspaces, I like to have apps on different workspaces for example music player on workspace 2 and email client on workspace 3 which I normally keep open, like to keep workspace 1 for firefox and other things. When I have internet open in 1, and music in 2 and email in 3 and close internet in one the apps from 2 and 3 move up one workspace. So to test I tried opening music in 1 and then use 2 for general stuff like internet, but I close firefox in 2 and there is nothing else open it auto opens the overlay which is annoying. 
3. Lacks easy customisations
4. Desktop – you can't right click the desktop to change wallpaper or theme, also you can't create desktop icons or folders for shortcuts to your favourite apps since there is no dock or quicklaunch option anywhere on desktop itself. 
5. Why hide the launcher/dock in the overlay.
6. Also seems to use large amounts of ram, starts at about 55mb and then grows to about 180mb after couple of hours and then seems to hover at this point. 

Now unity. 

Pros: 

1. launcher – autohide features, quicklists, icon can alert you when something needs your attention, you can drag files to it and only the apps that supports it will highlight, keyboard shortcuts for example super 1 to open first item in launcher etc. 
2. Dash/lenses – been able to type to search for apps and then just press enter, app lens if you type app that is not installed it searches the software center for it and clicking the icon takes you to the install page of software center.
3. Workspaces  - been able to put apps on different workspaces easily and have then stay there. 
4. Been able to right click the desktop and access desktop setting to change background and theme etc. 
5. like the global menu 
6. less than 100mb ram usage, thats adding compiz and all the unity services together.

Cons: 

1. based on compiz. 
2. Limited customisation but atleast there is easy way to do it via ccsm unity plugin, unable to move the launcher, unable to change the transparency, should be able to change the dash icons to what the user wants. 
3. The category dropdown in app lens looks boring and looks like it has been added on without any thought and at the last minute. 
4. Gnome control panel/system settings hidden under power button.
5. Notifications system could be interactive for the apps that would benefit from it like music player, to quickly skip a track for example. 

After testing both unity and gnome-shell I prefer unity it seems to let you work easier than gnome-shell does. It doesn't look as good as gnome-shell but I feel that it is more usable. I think  that the devs for each shell have done a good job and there should proud of themselves and it gives linux users even more choice to what they want to use.

I'd like to address some of your gnome-shell cons:

1. Easily fixable by installing one of the official gnome-shell extensions, which places the option permanently in the menu.
2. Valid criticism, one of the more annoying things about gnome-shell. I love and hate dynamic workspaces at the same time lol.
3. Not true at all. Gnome-shell is very customizable. The shell is written in css and javascript so there are various extensions available to change behaviour and add features. Its also themeable, I suggest you check out the 'zukitwo' and 'faeince' gnome-shell themes. zukitwo also has an awesome GTK3/2 theme. Themes and icons are easily changeable with gnome-tweak-tool, which is not installed by default but is an official gnome project.
4. Agreed, but I am fairly sure you can re-enable this behaviour via gnome-tweak-tool. Basically by default nautilus isn't enabled to manage the desktop, you just have to switch that back on.
5. Agreed, but there is an extension to permanently show a dock all the time.
6. Yes, this leak is a known issue and has been getting smaller each release :)

EDIT: I see zzmonty beat me to the punch with this post heh.

Personally I much prefer gnome-shell to unity, it feels like a much more polished environment. To me unity seems kind of cobbled together on top of compiz and incomplete. 

While it is true gnome-shell is missing some features by default I think what's there is executed very well. I also find its customization has much more potential. The shell being written in javascript and css makes is very extensible, and gnome-shell is much easier to theme and extend than the unity shell, which currently relies on hacky work-arounds to theme (from what I saw in natty, not sure if changed in oneric).

**tl;dr, I don't dislike unity, I just think gnome-shell offers a more polished out of the box experience and has much more customization potential. Unity does offer more out of the box features (particularly with the launcher, quicklists ect..), but it still feels like a beta project.**
