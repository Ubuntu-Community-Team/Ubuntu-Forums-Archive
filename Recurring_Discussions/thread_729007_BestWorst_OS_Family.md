If you look at the default installation of an OS you can see what OS family they are  part of.  Take any Windows (1.0 - Vista and 2008 ).  If you open up a command shell you have to use DOS based commands (dir, move, cls, etc), there are similar methodologies on how the file structure is laid out (A:\, B:\, C:\, etc..), programs designed for MS DOS will have a chance to work on Windows Vista (with bugs I realize) with no modification to the code, and the development of the MS DOS lead to the development of the later Windows systems (including Vista and 2008 ).  This is what defines a OS family.

As for sub-families, they would be a group of OSs that were developed under a similar methodology and can be classified either as their own family or as part of the greater family.  If you look at BSD, Linux and Mac OS, they all come from the Unix family of OSs.   But each has their own way of implementing Unix that is unique to their sub-family.  As a result there is less compatibility between OS outside the sub-family and those inside the sub-family (ever try to run a Mac program on Linux without modifying the code first?).  The sub-family phenomenon can only be witnessed with the Unix and DOS/Win Families (though it is more pronounced in the Unix family).  You could classify DOS based OSs as a sub-family along with Wind 1.0-ME as another sub-family, and NT as the last sub-family.
