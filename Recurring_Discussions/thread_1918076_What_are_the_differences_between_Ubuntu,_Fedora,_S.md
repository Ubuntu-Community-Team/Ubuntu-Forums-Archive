### Post by Paqman on 2012-01-31
> **roopeshmicasa said:**
> What was the purpose and Idea behind so many OS which are pretty much same. 


It wasn't planned, but since anyone can take open source software and modify it for their needs it's just something that happened. Different distros are designed by different people with a different audience in mind. Sometimes the only difference between distros is legal issues like the presence of trademarks (RHEL/CentOS), the actual code is exactly the same.

Other open source projects are the same to a greater or lesser degree. Many different versions of Firefox exist in parallel to the official one, for example. Lots of projects also split off forks, which sometimes end up becoming completely different projects. It's just in open source's nature to spawn different versions of anything popular.

### Post by Lars Noodén on 2012-01-31
There's not so much difference any more.  Probably the most obvious difference is whether they use APT based packages or RPM based packages, but nowadays both solve dependencies automatically for you.

There are different distros because they come with pre-configured defaults intended for different types of users.  Again these are only defaults and any distro can be turned into any other by adding, removing, or reconfiguring the packages.

### Post by doobrie on 2012-01-31
I've used both Ubuntu and Fedora linux and found that the software that you can run on each flavour is just about the same. Equally, both "base" operating systems are equally stable.

However, I've found that some of the software that is supplied with Fedora is not necessarily stable whereas I've had far less stability issues with Ubuntu software.

Fedora is more on the leading edge of what Linux can provide and as such, I think is more suitable for users who are happy hacking along in a terminal session and have a god understanding of Linux.  For new users, I think Ubuntu provides a less scary start into Linux.

### Post by bryanl on 2012-01-31
A distribution is a package of selected software that meets some particular person's idea of what is needed for a complete system.

There are several issues that tend to be involved.

1) how close to the cutting edge of development? - this is a risk vs benefit thing as getting the latest and greatest is balanced with stability

2) how is update and management to be done? - this is the deb vs rpm thing and package managers for adding, deleting, and updating the software in the system and being able to maintain it properly.

3) What is the user experience? - terminal or GUI, underlying GUI library, desktop and related utilities, customization, and defaults

4) What is in the distribution default catalog? -- of course there is basic system with kernel and utilities but what else? Some distributions are closely focused on a particular purpose while others go for the kitchen sink. The backing catalog is important also as it provides the online easy to add applications and that can vary somewhat.

5) How open is the distribution? - Keeping to FOSS can make distribution, especially internationally, much less of a burden but being able to play MP3 files or DVD's (for instance) can require special licensing and some may choose to include the decryption code and just hope it is used appropriately or not depending upon location. 

6) How is support managed? -- Is there a user community and how is it organized? Are there commercial support options?

7) What is the commercial interface? -- are paths to commercial media and software available to make it easy to purchase those items not available otherwise?

8) What is the development system? -- is there a bug reporting process behind the distribution? Is there a development effort supporting the distribution and the broader FOSS community? Is it formally organized with an established organization or is it ad hoc?

And then there are the 'flavors' where someone takes an underlying distribution and customizes or adjusts it so suit their tastes. Debian and Ubuntu provide a lot of examples of this phenomena. 

so many choices, so little time!
