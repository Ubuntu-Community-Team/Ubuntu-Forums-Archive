### Post by 3Miro on 2010-12-13
> **tgm4883 said:**
> Most windows infections I see are due to users installing things they shouldn't or vulnerabilities in third party code. This is something that can happen on Linux (already has). Please don't spread FUD.


It can happen on Linux, however, unlike Windows, Linux has trusted repositories. If you stay within the repos, you are as save as you can be. Also, in Linux, if a virus infects one machine on a network, it is extremely unlikely that it will be able to jump over to another machine. Even if it manages to do that, the patch will come out and stop the problem once and for all.

Under Windows, an AV software doesn't fix the problem. The security vulnerability is still there and one is simply relying on the AV program to catch the virus before it can do damage. Thus, it is extremely easy to take an existing virus and just modify it so that it can no longer be recognized by the AV definitions and in fact create a new virus. Then the AV updates, then the virus updates and this cycle goes on without fixing the real issue. An AV program may take a more aggressive approach and start blocking "suspicious behavior" for any program that may try to exploit the vulnerability, but that leads to false-positives and in some cases gets people to start ignoring the AV warnings.

In an open environment like Linux, the one who finds the vulnerability can also fix it. So we need only one "good guy" to find the problem before any of the "bad guys". In a closed environment, the good guys cannot fix the real issue, they can only make an AV that keeps things together, but doesn't fix anything.

Package manager gets a large program, breaks it into packages and installs them in the appropriate /bin /usr/bin /lib ... Then each component can be independently updated and/or used by another program. This is a great way to do open source software and some small proprietary apps.

MATLAB updates about once a year (twice tops) and Ubuntu updated every six months. It makes no sense or whatsoever to keep a ppa active (and sync with it every time) for once a year event. What I have now is a single folder (usually /usr/share/matlab or /opt/matlab) and of MATLAB is inside. There is no way to get pollution to the rest of the system, it doesn't even know I have MATLAB (for there is no reason to know).

I know they could have made a bad installer and messed things up, but they didn't. You can mess things up with a bad .deb as well. Open source software should be in the package database, proprietary software should keep out of it and stay on its own somewhere else.
