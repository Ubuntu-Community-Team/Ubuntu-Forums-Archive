### Post by GoGoThomas on 2011-06-15
[COLOR=Blue][I][B]IMPORTANT: This is a retrospective edit following a superb demonstration by participants of this thread that both solved my problem and enhanced my system which I thought would be impossible to do without a complete upgrade of Ubuntu. Basically I was suggested to just upgrade the kernel to the latest version which was trivial to do. I did *not* have to do a complete Ubuntu upgrade which I thought I would have to. Updating the kernel created another boot entry "[...] kernel 2.6.38", booting from that entry the problem I had with not being able to access some functions of my touchpad simply disappeared.
I was also able to very easily update the video drivers to the latest version and have superb 3D graphics also without having to upgrade the whole distro, check the thread for details on that.
So I basically withdraw all the wrong assertions I made about Ubuntu and apologize for it.
The solution offered here is not the best way to proceed with upgrades. But the proof of concept was extremely convincing.
[/B][/I][/COLOR]

Hi guys,

I've tried Ubuntu and played around with it and investigated about it and here are the main reasons I'm going back to Windows :

* When I buy new hardware there's no guarantee it will be supported by  Ubuntu (or another distro) or I'll find drivers for it straight away.  This just does not cut it for me.
* When I'm using an Ubuntu version I don't want to be forced to change  to a new version just to be able to install/run new drivers or some  other software that works only on a specific version. Having to do that  does not match the definition of "operating system". 
* When I start to feel comfortable with a version I don't want to be  forced to change for a new version which will be completely different,  mostly incompatible with the old version and with another batch of old  and new problems to solve.

Basically I want my "operating system" to be a system that operates, not something I have to struggle with all the time.

Maybe if ever those points are addressed I will give it another try but  for now Ubuntu (and probably most other distros) is too experimental, it  has too many breaking changes and this makes it too user-unfriendly for  me to stick with it.

User-friendliness is not only about rolling out new beautiful shiny interfaces  (which are anyway all a different reshuffling and revamping of the  Windows 95 look&feel), it's also about making stuff that works  out-of-the-box and that does not enforce breaking changes all the time.

I have admiration for all the people, developers and users who put so  much effort into it but too much effort is put into experimentations and  too little into making it stable, compatible (with hardware, other  software and most importantly itself !) and reliable.

So bye for now and maybe see you in a few years (I have put in my diary  January 2015 for another try if one Linux desktop distro has reached 2%  of the desktop OS market share by then (and if desktop OS's are  something that I still commonly use but I think so)).
