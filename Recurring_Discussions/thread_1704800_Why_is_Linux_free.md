### Post by Spice Weasel on 2011-03-11
> I read in some forums that free distros are the beta versions of commercial distros and that the users are like beta testers who document the bugs which get fixed in commercial distros later on. 

That IS NOT true.

As to why the operating system is free...

Short answer: Anyone can modify the source code, add features, or fix bugs.

Long answer: That isn't to say that developers of free and open source software don't get paid. Companies like Red Hat and Canonical provide paid support for their operating system, and Red Hat spends some of the money they earn from the support contracts on hiring developers who contribute code to projects on which the operating system depends (GNOME Desktop, the kernel, GNU, LibreOffice, Firefox etc). I'd say there is about an equal number of hobbyist developers to professional developers.

### Post by kaldor on 2011-03-11
As an example, nothing is stopping me from renaming Ubuntu and charging 100 dollars per copy for it.

Fedora is an example of a "beta" (nobody uses that word right anymore :) ) distro that eventually becomes the commercial product.

For example, Fedora is released freely every 6 months and is made up of community volunteer work and uses the latest (and often unstable) software. Every few years the company that started Fedora, Red Hat, takes Fedora as a base and turns it into RHEL; Red Hat Enterprise Linux, which is very robust and stable and is a leading name in the server market. However, that doesn't mean RHEL is better than Fedora. If you want new software and up to date stuff, use Fedora. If you want stability and such, use RHEL for the next 5-10 years.

Also, people do get paid to work on Linux. Red Hat puts a lot into GNOME development, for example. I also believe Red Hat does some great work with the free ATI/AMD graphics drivers.

### Post by rg4w on 2011-03-11
I've read that about 3/4 of the programmers on major FOSS projects are paid.  Funding can come from all sorts of sources - some of that is discussed in Joel's "Stategy Letter V":
[http://www.joelonsoftware.com/articles/StrategyLetterV.html](http://www.joelonsoftware.com/articles/StrategyLetterV.html)

Most of my own work is on commercial software products, but last year I had a contract making components for a FOSS project.  That was for an educational consortium who gets most of their funding from grants, and grants are increasingly friendly to FOSS projects.

I've also shipped some smaller components under a FOSS license (MIT) simply because it was stuff I wanted to write for myself, and by using a FOSS license I was able to get others to contribute as well, since we all get to use the code.

There are a great many ways free software gets funded.  Learning about the free software community has been eye-opening and fascination for me, and I hope to contribute more of my own code in the future.
