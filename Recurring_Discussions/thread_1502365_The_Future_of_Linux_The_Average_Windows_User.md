### Post by u2rcrazy on 2010-06-05
The Future of Linux:  The Average Windows User


An open letter the  Linux Community,

I consider myself the above average Windows  user.  I've taken and passed A+ Certification and some Windows  networking courses.  I have a basic, big picture understanding of DOS  command line and Windows.  I don't like anyone such as Windows or Google  monopolizing control  over their clients, so when Linux came along in the 1990's, I wanted it  to succeed.  I have loved the idea of a free computer Operating System  for everyone that Linux brought to us.  When I say free, I mean from  being controlled by Microsoft or their peers.  I don't mind paying for  an OS that offers freedom from control.  So I attempted to use the Red  Hat distribution around 2000 to no avail.  Now ten years later I'm at it  again with Ubuntu.  I love it.  There are so many aspects in Ubuntu  that I like better that Windows.  One being, there is so much free  software for nearly anything you need/want to accomplish.  The problem I  see is the fact that many hardware devices and software are not  compatible.  

If Linux is to make headway into the individual consumer market,  then these compatibility issues MUST be overcome.  I would like to  totally ditch Windows, but at the present time this isn't feasible.  I  have listed some of the present obstacles that I see and objectives the  Linux community needs to work toward to for it to be viable to the  masses.

[LIST]
[*]Compatibility with ALL Linux  software and hardware devices no matter what the Linux Distribution the  consumer is using.  
The Linux Community needs to reach out as one body to Hardware and  software manufacturers with uniform standards that apply no matter what  distribution is being used.  This leads to the next item.
[*] 100% GUI interface with no need for using the Terminal unless  the user would like to specifically use it.  This, I believe, is the  major obstacle we have to scale over together and it's _**especially **needed  for installation_.
[/LIST]

I hope this gives the Linux Community some ideas on how to  make Linux better for and easily used by the non-technical, common man  user.

Sincerely,
Bob

### Post by red_Marvin on 2010-06-05
1.1) Hardware compatibility is not a problems with linux, it is a problem with the hardware manufacturers who for one or other reason do not want to release linux drivers, or even release details so that people can volounteer to do it without having to resort to reverse engineering.

1.2) Software incompatibility arises from the fact that there is no single body developing linux, but a lot of different free agents with a lot of differing opinions on How Things Should Be Done. This might be cumbersome in the short term, but is not neccesarily a bad thing in the long run since it increases diversity.

2) Some of us actually like to use the terminal, however I believe it is a partial goal of Ubuntu and other 'newbie-friendly' distros to increase the amount of graphical interfaces available, so you can probably expect more in this department as time passes.

### Post by dFlyer on 2010-06-05
This has been a topic for discussion on every forum I've been on since the mid 90's. Always the same suggestions by new users, hardware compatibility and less command line items. Hardware compatibility has gotten better, but manufacture support remain pretty much the same as in the 90's, we don't support linux. As far as less command line items, most everything I had to do at the command line in the 90's now have a GUI. I still use the command line for most items because I find it faster myself. I believe developers has done a great job creating drivers for hardware without the support of manufactures. Just give it time and you will find that linux is not difficult to use or learn and I believe once you have used it a while you will wonder why you stuck with windows for so long.
