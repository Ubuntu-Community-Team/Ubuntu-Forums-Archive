### Post by BbICEP on 2009-11-25
I've found out F-Spot is unpopular. It rarely seen someone complements it, but there are many posts with complaint how unusable it is. The reason there's no better photo management app is not good one: It's just stupid to hold **bad** app because others are even worse. Its main technical disadvantages are: general slowness in both startup (it's mono influence, unresolvable) and in work (bad qualification I suppose). It's also very buggy. It can be improved, but too few people use this stuff to write bugreports. 
The main disadvantage, of course, is the way how F-Spot works. Instead of just watching user defined dirs for changes it needs import procedure. Moreover, it creates its own directory structure, which most people find out to be very inconvenient. It doesn't provide any method of working with FS, only with the tags.  It's ridiculous, because these tags use as FS replacement in most cases.
Tomboy functionality is covered with Gnote + Ubuntu One. In addition, Gnote takes 3 times less resources to work.
These two apps are the only which uses Mono. So, we could remove them with mono, and it would leave 2 times more space on CD than removing GIMP.
