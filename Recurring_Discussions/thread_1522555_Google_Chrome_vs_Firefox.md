### Post by axel668 on 2010-07-02
Chrome is ALMOST on par with Firefox, but most of the time I still prefer Firefox. Why ?

[LIST]
[*]UI is more compact (not out of the box but using "Classic Compact  Theme" and "Compact Menu" addons)
[*]better Adblock
[*]less glitches
[*]is supported by some websites that don't support Chrome (not Chrome's fault, of course, but still an issue)
[*]has its own settings (certificates, proxy, turn off IPv6 ...) while Chrome uses Gnome settings, which is crap if you're not running on Chrome
[*]Can sync not only bookmarks but also passwords and tabs (with "Firefox Sync" AddOn)
[/LIST]
So it comes down to this: If you heavily rely on extensions, FF is still the best choice. If you don't, Chrome is a serious alternative.

### Post by aysiu on 2010-07-02
I use both Firefox *and* Chrome, because I need two separate profiles for several websites, and I like to have them both available *at the same time* (so no closing Firefox opening it with profile manager and then picking another profile, and no loading a private browsing mode because the cookies from that session get deleted every time you close the window).

There are things I like about each browser, but overall I prefer Firefox.

In Firefox, I like: [list][*]The tab closing behavior (well, after a quick about:config tweak)[*]Separate URL and specified search bars[*]Smart bar that will quickly and automatically match any part of the URL or site name based on what you've visited most[*]NoScript[*]Consistent fast speed (not really fast sometimes and extremely slow other times, freezing up mid-page-load)[*]Built-in option for encrypting saved passwords with a master password[*]No separate process for each tab[/list]

In Chrome, I like: [list][*]No status bar, but contextual items will appear if you hover[*]You can open a separate private browsing mode window and have both regular and private browsing open simultaneously[*]Google sync for Chrome[*]Also cross-platform (unlike Safari and Internet Explorer)[/list]
