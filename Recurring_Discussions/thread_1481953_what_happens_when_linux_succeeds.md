There's still an advantage Linux has that Windows XP hadn't. And this happens to be SUDO.

Windows XP was tampered with viruses for the main reason that quite every user was given administrative rights (well, let's face it, normal users could barely turn the PC on in XP) so every software (including pigs + dogs + rats + Heaven-knows-what) would run with administrative privileges.

Vista's user account control (though being utterly annoying) was a step in the right direction and tried to prevent this issue granting simple user rights even to the admins, and requiring their approval to perform administrative tasks (the famous AAM) but still too many people were running admin accounts and either disable UAC (deserving every single virus they get) or click on "Agree" regardlessly (again, deserving every single virus they get).
I won't cover the simple user (running administrative tasks in OTS approval mode) because probably I'm the only one on Earth thinking it's the best solution.

This "virus invasion" won't happen in Linux, first of all because I think people are more aware of the importance of running user accounts and getting admin rights only when really needed, and then because people usually think about it twice before running any script they don't know.

### Post by julio_cortez on 2010-05-13
I know what you mean Helkaluin.. But I'm still confident of one thing.

**In WINDOWS**, users that started using Windows 95/98/ME are used to be administrators because there wasn't any difference between users in the old 9x installments (at least, I can't recall any but maybe it's my fault). Of course, this kind of behaviour became a habit and went on when XP arrived.
Users that started using XP are used to be administrators because in XP the user account was too weak and too strict. Of course, this kind of behaviour became a habit and went on  when Vista arrived. In addition, they disabled UAC because they considered it a pain.
Users that started using Vista/7 just saw that being administrators was the habit and thought "why should I be different?".
Very few users (me included, fortunately) really understood why UAC is vital and wouldn't think for a moment of getting rid of it.

So everyone that starts using Windows will find a world where **the habit is using administrative accounts**, and is likely to stick to that habit (unless someone wise gives a different advice which is not common).

**When it comes to LINUX**, the scenery is slightly different.
Since the UNIX days, users were used to run limited accounts and to elevate themselves only if needed. So this is the habit in Linux (opposed to Windows where the habit is using an administrative account).
New users that see Linux for the first time (even the ones that migrate from Windows, like me) usually stick to the habits, at least for the first times (even because if I asked advice on how to set up my user account, everyone would tell me "it must be limited").

So, I think that even people that move from Windows to Linux will find a world where **the habit is using limited accounts**, and will eventually start using a limited account themselves.

It's far more complicated than this, but I think these are the vary basics. It's a matter of habit and heritage.
