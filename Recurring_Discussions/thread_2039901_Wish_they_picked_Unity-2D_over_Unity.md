### Post by vexorian on 2012-08-09
So, I unbelievably switched to Unity. Ok, not exactly Unity. I gave Unity a second chance, but now with a bottom gnome-panel that has a task bar and desktop switcher. With the launcher hidden, I finally got the best of all worlds. The launcher and dash are useful to launch stuff, use quicklists, etc. The global menu bar saves space, but I use the gnome panel for instant app and workspace switching (which is something that Unity can't do well and is very important for my efficiency). Then I had the HUD, which I could in theory use.

I finally had a desktop that suited my needs yet was also new and had cool new features. So I was using it. But eventually, it becomes hard not to notice that it is slower. I open my system monitor and of course compiz is draining 10% of my two 3.0 Ghz cores and 100MB of RAM. 

Compiz was never so friendly with my computers. Always needing far more resources than metacity. Even though if I used compiz, I would use it only for the window shadows and allowing Unity or the terminal to have transparency, something that metacity could do just fine... The issue is that since unity is a compiz plugin, I cannot choose to have unity without being locked to compiz.

It is amazing that for the first time in my whole ubuntu life, I felt locked to a software I do not want. Then I remembered about Unity-2D. I try it and it uses metacity, so I just enable metacity compositing and unity once again has transparency. It does not have the HUD, and I can't configure it to be revealed by fade in effect. **But it feels so much faster and responsive**.

This is the part that is frustrating. As opposed to deciding to grab Unity out of compiz and begin to develop it standalone, Canonical preferred to make a whole different project, that uses its own configuration and requires any addition to unity to require twice the effort. That's the reason there is no HUD. It does not even support the same theme for the "global title bar" buttons. When a user's video card driver starts malfunctioning (maybe he tried something funny when installing an updated version) he will fall back to Unity 2D, only to find that it does not use all the configuration he used. A tad terrifying.


The duplication of effort was a lame idea, so, Canonical decided to pull the plug of one of the projects. But I think they made a grave mistake in choosing to kill Unity-2D as opposed to the compiz plugin.

Unity-2D was a great idea. Imagine how would things be if Canonical decided to replace the compiz plugin with Unity-2D (and stopped calling it). At this moment Unity-2D has less features than Unity, but that's simply because Canonical put their focus on Unity. This hypothetical new standalone Unity that uses "Unity-2D" as a base and replaces the compiz plugin. Would be developed to have all features Unity has and all the features that get added later. There could also be a compiz compatibility plugin that allows the standalone unity to use compiz for desktop switching and the edge/corner triggers. Since  Ubuntu likes compiz, and compiz can still be used by this standalone unity app, it would be the default. But metacity would be the fallback version and we would also be able to use mutter/etc. Users and also other distributions would have more choices, this would help make Unity look more tempting to other distros to adopt.

I really, really wish things were like that. Instead, when I upgrade to 12.10 or a later version, I will have to make a choice: Either I will get used to dedicate 10% of my computer resources to a window manager (and that's when compiz is idle), or try a different shell that is not unity.
