Infection.... Again it's a bad word, compromise is much more accurate. Let's understand something first. Infection is usually used to reference malware that is viral or worm like in nature. That being said, there are not currently any live viruses/worms on either Windows or Linux. (Uh oh what's this no viruses on Windows!?) yep you got it.


Malicious code of the type that commonly circulates is entirely spread by the user's inability to properly secure their system. Most is spread through flaws in a browser or dynamic content object like Flash, or PDF. This can effect Linux users just as easily, for instance not too long ago Firefox 7 had a vulnerability on Ubuntu that could have easily been used to compromise a poorly secure Ubuntu system. 

Most exploit code is written for Windows, it's more popular, so there is no point in targeting the 2% of the population that uses Linux (this doesn't mean it can't be done, it's trivial to do so).

"bogging down" is a side effect noticed in a system that is overloaded, it's more easily noticeable in Windows because of how it processes threads. Linux processes them more efficiently, whether they are good threads or bad threads. It makes no distinguishment between the two. Yes even malware runs more efficiently on Linux (thank you caching)

That being said, compromising or "rooting" a linux machine is often done very differently than it is on Windows. Windows has very mature AV solutions that need to be bypassed. It's been dealing with malware on the desktop front for a very long time. Only recently has Linux become even somewhat popular as anything other than a server OS.

Often times a Windows rootkit is a very complicated creature which must hook many processes, whereas a Linux based rootkit may only hook a few. Let's face it the majority of you aren't running file integrity checking. So if say... sudo were to get wrapped you probably wouldn't even notice all of your stdin/err/out were being logged and sent off somewhere else. The anatomy of a linux compromise is much more infantile than a Windows compromise in this sense.

Even skilled system administrators have a hard time tracking down and reacting to a root level compromise, why? Because Linux's file system is considerably more convoluted than Windows' (among other reasons). Hence the saying the only way you can trust it is to do a reinstall which I'm sure you've heard before.

Hopefully this helps you understand a little bit better that things are not always what they seem in terms of system compromise.

Steps to Virus Protection in Windows:
1. Install the latest Microsoft updates.
2. Purchase a reputable antivirus program.
3. After installing an antivirus program, scan your entire computer to be sure it is free of malware.

Interesting thread ;)

**For worms:** Linux requires latest security updates as much as windows requires latest updates. Otherwise both could get infected easily (if they were targeted). But Linux security updates usually become available faster than windows updates. Windows updates are also delivered in relatively more huge packs (and so harder to download and ignored by users sometimes to not slow down network connection). This seems enough to argue that Linux is more **secure** against worms in respect to Windows (not because design but because the method of patch distribution; while they are equally vulnerable to yet non-patched holes).

**For viruses:** Viruses are not designed to spread by themselves (otherwise we call them worm). So user interaction is required for a virus to spread. It maybe from downloading and running an email attachment till an USB autorun malware.... In this case Windows users suffer more from autorun. Linux does not run "autorun" files without asking/warning user about it (at least my Linux box does so by default). Also in Linux, users have less incentives for helping viruses to spread ;) because most software are installed from well-known repositories while in Windows a lot of users have used to download programs from everywhere (like the recently revealed insecure download dot com as reported [here]("http://insecure.org/news/download-com-fiasco.html")). It's OK to say users' habits will rule this case but OS affects users' habits ;) It seems reasonable again to argue Linux is more **secure** than Windows in this case too (not because viruses can not infect it, but because it drives its users in a way that viruses have very less chance to infect them; still stupid users could run viruses with root access by their will).

I think above arguments are independent from the **fact** that there are very fewer active malwares for the Linux than for the Windows in the wild.
If not, I'll be happy to correct my arguments.

--PS: Of course above arguments just go for the normal/average cases and any OS could goes from very secure till very insecure spectrum by its user ;)

I think haqking is really right here, there is a lot of difference between "infection" and "security" all of which comes back to the fact that users don't really know what threats are affecting them. To simplify things (which probably isn't a good idea because someone will come through and nitpick it). Let's say for a moment we have a few BASIC avenues of system compromise.

- User induced
- Automated
- Targeted

Now lets look at these different categories. In the user induced category we have everything from you screwed up your sudoers file to more interesting things, like you were taking part in twitter, visited someone's page and it was XSS'ed, thus stealing your twitter authentication token and your credentials. Neither Linux nor Windows are less "secure" in this regard, because this is a weakness in the site you're visiting (and your browsing habits). Truthfully if anything I'd go out on a limb and say that Windows is more secure here (before you lynch me) I'm referring to the anti-xss engine built into Internet Explorer 8+. That being said, there are addons you can get for firefox/chrome that do the same thing. MS just ships IE with that feature out of the box.

You can take it a step farther and utilize an XSS'ed site to attempt to compromise a system. The infamous drive by download comes to mind. Or you can utilize it to compromise a set of browsers (blackhole anyone?) via a scripted exploit pack. Both Windows and Linux browsers (including current versions) have had remote code exec vulnerabilities. These mostly target Windows browsers (why? not because they are more vulnerable, but the bottom line is there are more of them) it's a time money trade off thing there.

Now let's look at the second category automated attacks. These are things like SSH brute force scanners, automated attack tools that are set up on a compromised server and attempt to compromise other servers, things you download that connect your system to a C&C server as part of a bot net, and worms. Real quick, worms are self propogating code that exploit a variety of vulnerabilities including those find in network services (like mysql , samba, sendmail, named etc...)

Again, Windows and Ubuntu are on an equal footing here as they both have the tools necessary to secure against these threats. Both provide mandatory access/integrity controls (depending on if you're MS or nix), ACLs, other controls and of course decent patch management options.

The third category, targeted. If someone is TARGETING your system or network, they will eventually get it. Sorry to say, the only thing you can do in that regard is follow best practices, and safe computing techinques and hope for the best. But truth be told in the case of a dedicated attacker the concept of security by obscurity is working against you. Let's be realistic, odds are the only person linux's workings would be obscure to in that relationship would be you. 

Hopefully this helps explain infection/compromise and all sorts of other misnomers flying around this thread.
