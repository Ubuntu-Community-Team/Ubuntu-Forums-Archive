1- It is not a 100% GUI interface...that is to say, most of required package to run the system are independent and have to look for them here and there, Actually, most of the required packages to run the system are included with the system. And no OS is 100% GUI interface. Ever try to show hidden files in Mac OS X? It all depends on what you want to do.

Yes, there are ways in which Ubuntu (and/or Linux at large, depending on what specifics you're talking about) can add more GUI functionality (I blogged about this recently in [My top ten favorite Ubuntu Brainstorm usability ideas](http://www.psychocats.net/ubuntucat/brainstormtopten/)), so we're not even at 99% GUI interface, but I don't know if 99% is quite as dire as you make it out to be. I'd love to get to 99%, but right now basic functions and even some higher level functions can all be carried out through point and click in Ubuntu.

And the developers are well aware of the need for more graphical interfaces for things. If you don't believe me, compare Ubuntu 8.10 to Ubuntu 5.04 and you'll get an idea. > which makes it difficult and sometimes frustrate users whom are with no programming knowledge...it is designed for IT oriented people (Engineers, IT technicians, Technical People...). Based on the hundreds of testimonials and bad experiences I've read on these forums, I'd have to disagree. It's actually the IT-oriented newcomers who tend to be the most frustrated, because they assume they (as "experts") should have no trouble with Ubuntu, but then they realize all that Windows knowledge has just made it more difficult for them to adapt to a new culture.

Non-technical people have a lot easier time adapting, because they don't consider themselves experts, and they're used to having computer problems no matter what platform they're using.

> 2- It is command driven in majority, Um, actually it's not. Can you cite some specifics about how it's "command-driven in majority"? Based on what you're saying, someone who's never used Ubuntu before would be under the impression that the CD boots straight to a command-prompt, all installation is done from the command prompt, and there are no graphical programs unless you launch only a small handful from the command prompt. Such is not the case at all. It's actually GUI-driven in majority. There are a small number of tasks that still require the command-line, and I hope the developers work on fixing those. > which is something that makes it hard to use and deal with especially when the system fails to login due to ownership problem. I think you'll find that any time there are damaged or corrupted system files, you will have a problem logging in or booting up. That is not unique to Ubuntu. It happens on Windows and Mac OS X as well. > I wish if it is fully GUI which will make it friendly user and I am sure that if it so, WINDOWS & MAC OS are GOOD BUY...will be recalled as OS of the past. They are a good buy? I don't get it. I don't understand what this sentence means.

> 3- It does not support most of the PC hardware, which makes it not preferred by all, when they realize that they have to deal with commands, which is something a user will avoid no matter what great feature the system will or is offering. Well, there are several components to this: [list][*]Your phrasing is wrong. A lot of hardware does not support it. Not it does not support most of the PC hardware. There is a lot of proprietary hardware out there with proprietary or no Linux drivers. The Ubuntu developers are sometimes able to work out some reverse-engineered drivers, but not always. The good news is that more and more these days, hardware companies are releasing Linux drivers (I think Broadcom is next up). Notice it's Broadcom who's releasing the drivers, not the Linux developers who are releasing them. Notice also that Windows did not release drivers either. It's up to the hardware manufacturer to open up specs or port a driver.[*]I've seen bad hardware support on Mac OS X as well. It uses CUPS, just as Linux does. And when my wife and I tried to migrate my in-laws over to OS X, we failed in part because the Canon printer they had would not work with Mac OS X.[*]Mac OS X and Windows have the benefit of being sold preinstalled and properly configured usually. The Ubuntu-preinstalled options are severely limited (only certain models in only certain countries), and as we've seen from the recent Dell Mini Inspiron 9 Ubuntu release, even the preinstallation configuration could use some polishing from Dell.[/list]

> All of the above can change if the LINUX community force its OS by adapting a new MARKETING strategy. Actually, no. All of the above can change if Ubuntu sells well-configured preinstalled machines that are properly advertised as being different from Windows in a good way. That is what has made Apple so successful in the past decade. The big difference would be Apple's computers and operating system being mainly proprietary and Ubuntu's being mainly open source. > In my opinion can be achieved once the H/W manufacturers and S/W developers find LINUX an OS that USER can deal with it in ease. Then you don't know how big businesses are run. It has nothing to do with "ease." It has everything to do with market share. If 10% or more of computers sold are sold with Ubuntu preinstalled, hardware manufacturers would be idiots not to port their drivers over, and software manufacturers would be idiots not to port their software over. But even if you gain some marketshare, you can still miss out.

I know a Mac user who loved her Mac but ultimately had to switch to Windows because she needed AutoCAD for her profession, and AutoCAD is not available for Mac, and she heard that running it in Parallels gave bad performance.

I don't think you appreciate the complexity of factors involved in computing market share and user adoption. It's not as simple as "Let's just make better software and everyone will start using Ubuntu!" Incidentally, I blogged about this recently as well: [Linux-for-the-masses narratives](http://www.psychocats.net/ubuntucat/linux-for-the-masses-narratives/). What you're talking about is narrative #1.

### Post by cariboo on 2008-10-16
I've been a linux user for about 10 years, I first bought Redhat 5.2 and one of O'Rielly's excellent books on unix commands. Unfortunately that experience has made me run away from any rpm based distribution. The difference between now and then is like the difference between Windows 3.1 and Vista. For day to day tasks, I can do everything using a gui application. There is nothing I have to open a terminal for. 

On the other hand I do have a terminal icon on my task bar, as sometimes it is just quicker to do something by clicking on the icon and typing a quick command  to say for example list the contents of a configuration file. I don't expect a newbie to do the same, but nobody should be afraid of the command line.

We have one ISP here in BC, that whenever you have any problems, the first thing they ask a windows user to do is to go to Start-->Run and type cmd and press enter. The first problem a lot of windows users seem to have is the the enter key is one they can't seem to find, they always ask which button to click. Then once the have the terminal open they are asked to type:

```
ipconfig /all
```

then hit enter. I have seem fairly smart people struggle with this command for more then ten minutes. (Telus help-desk people seem to have a lot more patience than i do), Then there is the problem of getting them to read back the results of the output. this is so far outside most peoples computing experience, that it is an ordeal for them.

Using a computer is what you make of it. If all you want to do is get things done, with out learning something new stick with Windows and pay people like me to fix it when it breaks. If you like to learn new things, try different operating systems, there is more out there than just Linux, you might find one that suits you better.
