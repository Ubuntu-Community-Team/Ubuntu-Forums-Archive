### Post by Can+~ on 2008-07-17
Ok, I won't be so blunder about this.

Getting a game done is serious business. I will take years to have the programmer mindset and expierence, and it will take another considerable amount of time to be capable of working with a certain platform or 3d-toolkit.

Nowadays, most games are written on C/C++, but this doesn't mean that you should start immediately by there. In fact, I would suggest you to stay away from them, and from 3d/2d platfroms. The typical advice here, is go with python. Become capable with python and then use pygame. It will not get you a career as a game developer, but it doesn't matter, you'll learn. Later you can learn C and C++ if needed.

Whatever path you choose, learn the language first; later you'll learn GUIs and 2d/3d engines.

### Post by pmasiar on 2008-07-17
Learn basic programming skills in Python first (see wiki in my sig). But be prepared to learn multiple languages, only narrowminded programmers limit to single language (use best tool for the job). Later you may need to learn C or Java or whatever, but learning basics in Python is much easier than in C or Java. And Python is decent small language you can often use for small tasks, even if big tasks are in Java or whatever. And if are you lucky and get into smart company (like Google), you may even use Python as main language.
