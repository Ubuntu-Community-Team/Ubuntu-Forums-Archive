### Post by Old_Grey_Wolf on 2012-02-17
As you increase the number of computers, and the more complex the configuration of servers and workstations, the less often you want to upgrade. **At least that is my experience.**

I have 5 computers in my home computer lab, not including the 100's of computers at work. I use some of the computers in my home lab for self education purposes. I have DNS and LDAP running as a service, printer service, LAMP stacks with Wiki and BB services, and so forth. I also use Ubuntu Enterprise Cloud on a few of them to run virtual machines in a private cloud. The computers are all different brands with different NIC cards, graphic cards, and some I have built from spare parts. **Therefore, I choose to upgrade as infrequently as possible while continuing to get security patches.**

For the computers running Ubuntu, I use the LTS-to-LTS upgrade cycle. 

I try the newest release of any of the distros I am interested in exploring with virtual machines. I try the Ubuntu non-LTS releases as virtual machines just to keep abreast of the changes that are coming.

With Ubuntu, you don't have to stay with LTS-to-LTS or the every 6 month release upgrades. The non-LTS releases are supported for 18 months; therefore, you can skip some of the 6 month release upgrades. You will however need to do a fresh install or upgrade through the intermediate releases if you choose to skip them.

Considering the modification I make to the installations of Ubuntu, upgrades usually fail and I have to do a fresh install of the newest version (upgrade) anyway.
