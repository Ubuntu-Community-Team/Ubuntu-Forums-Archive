### Post by Bruce H on 2009-11-25
Heh. I don't consider Slackware more "advanced" than other systems. There's no package system similar to synaptic, for instance. Now, slackware is one of my favorite distros. It doesn't hold your hand. It does what you tell it to do and nothing else. It won't update itself and break things. It is by far one of the most stable distros I've ever used.

That said, I recently moved to Ubuntu Server for my home server, which was running slack since version 9.1. I'm impressed with the Ubuntu Server so far.

To the OP. If you are looking for a highly customizable system, gentoo is hard to beat. The thing about gentoo is that it does not use binary packages, it downloads and compiles each one, every time. This is good because you can specify options for the compiler. Dual processor support, AMD flags, etc. And you can specify options to leave out if they are unnecessary for you. These options will be applied to every package you compile by default.
