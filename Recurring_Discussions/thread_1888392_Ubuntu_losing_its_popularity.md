### Post by Copper Bezel on 2011-11-29
Yes, the stats are based entirely on user agent strings, and anyone using a browser that isn't packaged by the distributor (such as Chrome or Opera) is going to have a generic string. You're right that Mint probably has either a generic string or an Ubuntu one in Firefox, and it's possible that even if it uses its own, the tracker doesn't recognize Mint and lumps it in with the generic strings.

So yes, that statistic is useless to compare Ubuntu and Mint. It does show Ubuntu's relative ranking with other (real) distros as stable, however, which helps to dispel the myth that Unity is greatly damaging Ubuntu's popularity, which is where this thing starts.

User agent strings are still a much better way of identifying trends than DistroWatch is, of course, but I can only take the idea of watching user agent strings *on* DistroWatch as a bit of humor.

Edit: Simian Man, there's no reasonable way to actually gauge usage, but seriously, there are a lot more Facebook users than DistroWatch users. They're both stupid metrics, but they're certainly not *equally* stupid metrics, and that's part of the point being made there.

And yes, Mint usage is increasing relative to Ubuntu, because it's increasing outright, and that's by all accounts primarily a phenomenon of the tiny fraction of Ubuntu users who feel so personally insulted by the existence of Unity that they feel the need to switch to a different distro.

I realized that it might be read that way, and I edited my post to clarify, so I'm not sure which version of my post you saw. I'll try to address both of these points. And again, I'm taking that we're assuming that a lot of the new Mint users used to be Ubuntu users. If that's not the case, then this is irrelevant.

So you start with a random Ubuntu user on 10.10 or 11.04 looking at the interfaces in Ubuntu 11.10 and Mint 12. He or she decides that he or she prefers Gnome Shell with the Mint extensions over Unity. To get to 11.10, that user can either (risk an) upgrade and add a couple of packages for the shell and extensions, or do a fresh install and add the same packages. To get to Mint 12, the user has to do a fresh install, but there's nothing to tack on after. An Ubuntu user who's already upgraded to 11.10 and can't stand the Unity interface has a different position. Shell and the Mint extensions are a quick install away. There's no practical reason to switch from Ubuntu to Mint (or Mint to Ubuntu) because they're the same OS. But the user might switch for personal or "political" reasons. 

And sure, there's a lot of brand loyalty stuff out there, some of which has practical implications and some of which does not. Mint and Ubuntu have clear goals and priorities for their projects that obviously aren't the same, and users of either one might feel that switching now just means not having to do so later on.

At the same time, putting aside the practical considerations, I don't really expect users to switch distros arbitrarily. I actually do expect that most users have at least a little personal attachment to whatever they're using.

My attitude's a separate thing, and would probably get me an infraction to elaborate on. = )
