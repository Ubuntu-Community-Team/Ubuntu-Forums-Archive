### Post by 3rdalbum on 2009-01-24
I started writing a thorough, point-by-point reply to this message, but in the end I was just repeating myself.

Ubuntu comes with no openable ports, by default. This means no malicious people can just connect to your computer. Bam, this stops 99.99% of threats. If you install server software, this will listen on a particular port; so unless you need a web/ftp server, don't install one! If you do need it, make sure you look at tutorials on how to securely configure your chosen server software.

Don't turn on remote desktop, as this opens a port. Exception: If you have an ADSL/cable router providing your internet service, then this probably already has a firewall built-in. This will close all ports that lead outside your network, but allow those that only travel inside it. Therefore, you can run server software and remote desktop safely, if your ADSL router has a firewall built-in.

Setting your firewall settings will not stop the operating system from booting.

Unless you logged into Ubuntu using the username "root", then you are not running as root.

Windows malware does nothing on Ubuntu. There are no viruses or malware for Ubuntu.

There are keyloggers, but none of them are malicious and you need to install them yourself, if you want one.

In short: Ubuntu is safe out-of-the-box. If you install server software and have it visible to the world, then you could possibly get some problems if you don't have the software securely configured.

It sounds like you've worried yourself more than anyone else here on the Ubuntu platform. Relax.

### Post by druellan on 2009-01-24
A lot of good answers but perhaps I can contribute with this to anyone reading this thread:

What Ubuntu has as a Linux OS is a strong security scheme based on permissions and users. You can't toy with the system unless you have administrative/root permissions, and the same rules applies to all the apps and software, that means, NO app can write information anywhere outside the user home folder without being prompted for a permission (experienced users please clarify this if necessary).

Microsoft got two main security schemes. Wide open as on Win95/98, and secured by file permissions and roles as in 2000/XP. The first prove to be outdated, so MS jumped over NT technology, sadly, most 2000/XP users work as admins (without restrictions) and apps still can write everywhere they want on the system. For a virus thats like a holiday, it can spread everywhere without any prompt on the user side.

On Vista, MS started to implement a system somehow similar to *nix. An user can't be admin by default (and the profile is well hidden), and to perform any disk operation outside the users folder you need a password. Thats why Vista have so many problems with outdated apps that still insist on write crap on the install folder. Users get annoyed about the prompt asking for confirmation here and there, but thats the way should be from start.

So, Windows and Linux are starting to be in par, but historically Linux was way better than Windows on this matter.
