### Post by Copper Bezel on 2011-10-17
Serious answer abridged: Unity is meant to maintain some consistency in look and feel with prior versions (moreso than Gnome Shell would, as it's a very sharp break from Gnome 2) while offering some new usability features, largely inspired by the work with the Ubuntu Netbook interface. 

Versus Gnome 2 at default, Unity gives the current application a bit more screen space and organizes all panel objects into strict categories - the top panel is only menus (and the window controls) while the left Launcher is only buttons, which represent applications, actions, or Dash search features.

Docks, the pseudo-tiling, the screen-space maximization features, and search-based launching are, as has been stated many times in the past, actually bits that the nerdier types have been using under Gnome 2 for quite some time. While a number of features in Unity are designed to be "inviting" to new users to a point that might seem almost obtrusive, there's quite a bit more in the way of "power user" features being pushed into the mainstream (and they're mostly present in Gnome Shell as well.)

Unity provided a relatively soft transition; it was an experimental option in 10.10, default in 11.04, and little changed from 11.04 to 11.10 despite the shift on the backend from Gnome 2 to Gnome 3.

While there's some inspiration from touch devices and perhaps some preparation to support them in future releases, Unity in its present state is not in any way designed as a touchscreen interface. Gnome Shell is, in fact, much further along on this front, as I understand it.
