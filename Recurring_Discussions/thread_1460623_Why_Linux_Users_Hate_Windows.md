### Post by aysiu on 2010-04-23
*Hate* is a rather strong word. I would simply say I *prefer* Linux and Mac OS X to Windows.

I have to use Windows at work. And I have a netbook with Ubuntu and a laptop with Mac OS X. I use all three platforms pretty much every day.

The one conclusion I've come to is no operating system is perfect. "Just works" is a meaningless marketing ploy. There are a lot of great things about Windows. For example, Windows runs a much larger set of consumer-targeted software than Mac or Linux. Windows allows you in an open or save dialogue to do pretty much anything you could do in Explorer anyway. Windows 7 has finally brought about a sane way to run a limited user account you can password-authenticate to an administrator account when necessary.

At the same time, Microsoft seems to deliberately make life difficult for no good reason. At my workplace, we use CloneZilla to mass deploy Windows images to hundreds of laptops. In Windows XP, we used to have an easy way to copy the configured user to the Default User profile. Now in Windows 7, there appears to be no such way to do it. In Linux, it's as easy as just copying settings folders to /etc/skel and then doing a recursive *chown* to root for the whole folder. With Windows, it seems I have to do endless searching to figure out how to do anything, and the forum solutions are usually dead ends. There also seems to be no way to really make the system tray area uncluttered or to turn off all the useless messages that keep getting displayed there. I try to disable as many tray icons as possible, but in Windows 7 many of the tray icons cannot be taken out completely, only "hidden" (which means they're still there, but you have to click an annoying arrow to see them). I want them gone.

Managing Active Directory and Exchange is a pain, too. Expanding cryptic folder trees to get a gazillion tabs in property dialogues. It's a headache.

Just because you have to do everything the GUI way doesn't mean your GUI has to be hard to use, but it seems the Microsoft way is to just add in extra clicks. And don't even get me started on the ugly ribbon interface in Microsoft Office 2007, which has hampered my productivity. Those menus in 2003 worked just fine with the Alt keys. Now I can still use *some* of the keyboard shortcuts (not all of them), but I have to have them memorized. Otherwise, I have to resort to mousing through a total reorganization of the menus.

And the worst part is that it doesn't "just work" as some Linux bashers like to claim. I'm not official tech support, but I do help people out with their problems every now and then just to be nice (especially since our official tech department is a little understaffed), and people are getting cryptic error messages all the time. *OneNote won't load. I can't save this file. I'm always getting my document recovered every time I open Microsoft Word. I have a virus. Why isn't my CD recognized? If I insert a USB key, I keep being asked to format it.* The list goes on and on. Anyone who thinks Windows "just works," I ask you to work in tech support some time.

At the same time, Linux and Mac are not perfect. In Linux every now and then there is a hardware regression or some weird glitch that seems to never get fixed, even if the original bug report was filed back in 2002. Every now and then I like to use MuseScore to make some simple vocal arrangements. In Windows, I know if I install it I can get playback. Some with Mac. If I install MuseScore on Linux, nope... suddenly I have to look up a tutorial on how to use some weird thing called Jack (which I've never used in five years of using Linux... only Alsa, OSS, and PulseAudio, and I don't even know what those are). One of the great things about Linux, though, is even if you have graphical interfaces for changing settings, almost everything is a text file. So if you want to copy settings, back up settings, or change settings, just fine the text file that controls that stuff. And the error messages tend to be a bit more helpful. First of all, I can highlight them (in Windows, I can't) to copy and paste them into Google, and the answers are about 50-70% there in the search results (I find about a 10-20% success rate with Googling Windows error messages).

Mac OS X doesn't always function well. I've blogged about all these crazy issues:
[Macs are just computers, not magic](http://www.psychocats.net/ubuntucat/macs-are-just-computers-not-magic/)
[Macs are computers, not magic (part 2)](http://www.psychocats.net/ubuntucat/macs-are-computers-not-magic-part-2/)
[Made the move to Mac](http://www.psychocats.net/ubuntucat/made-the-move-to-mac/)
[Using a Mac full-time: the good, the bad, and the pretty](http://www.psychocats.net/ubuntucat/using-a-mac-full-time-the-good-the-bad-and-the-pretty/)

The latest episode is trying to get Time Machine to work. It is a nice idea. I like it in theory. In practice, it just takes forever to back up, and then sometimes I just give up (three hours is too long to wait to do a simple incremental and/or differential backup). So I now have a launcher for a custom bash script that runs rsync to back up my Macbook Pro.

And it seems about every other day I, for no obvious reason, get a rainbow circle of death in some program.

Computers are great things. They're human-made, and so flawed. They have bugs. They sometimes break down. But they're fun and help us get our work done. I don't hate on anything. I just use what I prefer and try to enjoy the perks and just deal with the annoyances.

Even finding Startup entries in the registry is a matter of looking in 3 different places.

All the programs I used to install after initial installation & updating. 3.) Files - Scattered everywhere on the hard-drive. Backup to do a reinstall was always a nightmare. Endless hours of searching for all the places my files had found their way to on the hard drive. 5.) Windows Media Player - Without an internet connection, when ripping a CD to hard drive, Windows Media Player would make all of the songs 'Track 32'. I still haven't found the time to sort all the way through every file in my Music folder. 7 or 8 years after banishing Windows to the wastelands, I still have albums in my collection where all the songs are track 32.

6.) Separate updates for every file - Everyone will agree with me on this one. Synaptic = Reload, Mark All Upgrades, Apply > Simple (or simply Check, Update for Update Manager)
Windows = Separate update procedure for windows itself & every single other program installed > A complete pain in the rectum.

To further this point, as far as updates go, Linux even has repository mirrors everywhere. I simply use my Service Providers repository. Thus, under Linux, updates don't even count toward my bandwidth usage.

7.) thumbs.db files - I could start swearing about this, really. A thumbs.db file for every single folder that has an image in it. Let's just say I'm still finding them on some of my hard drives.

8.) Calling Microsoft every time I re-installed - I know you've all done it. At one stage I literally was calling Microsoft on the phone for activation **every** time I did a reinstall. "This one time at band camp..." I was installing on a slightly old hard drive and it was too much for the hard drive to take. It buggered up & I had to reinstall again on a different drive. So I actually had to call Microsoft twice in one day.

9.) Encoding/Decoding - Almost impossible to find an mpeg2 codec without paying for it. DVD players that won't play commercial DVDs, etc, etc. And later versions of Windows Media Player started omitting the option/codec to rip to an .mp3.

10.) Security - I'm not even gonna talk about this. We all know about the constant cleanup & monitoring that is necessary to maintain a windows install in near mint (slightly compromised) working order.

I would never go back to Windows. Not in a million years. Not if it was free. Not if you paid me. There is not one thing Windows can do that I can't do (more easily/with better results/a better way) with Linux.

Of course, I would never throw out my Windows disc. When you pay that much for a drink coaster...
