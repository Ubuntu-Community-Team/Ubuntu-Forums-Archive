### Post by kaldor on 2011-04-27
I might be wrong, but I think the only thing that can affect Linux at the moment is a trojan. But, trojans can affect any OS. 

If you're smart, you won't have any issues. Before Ubuntu, I ran Windows from 95 to Vista (Vista very briefly though) and only had two bad infections. I chalk that up to using a family PC.

Linux and UNIX dominates the server world. Still, the minority of Windows machines in the Server sector still get attacked more.

Mac OS X is based on FreeBSD (reputed to be one of the most secure operating systems, and a very popular Web server OS). It has malware, but only Trojans due to the market share.

Ubuntu has repositories. If you install all software via the Software Centre, I'd be *extremely* surprised if you got infected.

Google's Android OS is based on Linux. It's now a huge magnet for trojans on smartphones. But again, if you are smart and don't *directly allow stuff to go wrong* you'll be perfectly safe.

Also, Windows allows people to log into Administrator. In fact, Administrator is the default account. For non-experienced users, that is a HUGE security issue in itself. Linux doesn't allow root (admin) logins via the GUI by default. This means only one user can get affected if a malware strike occurred; think about a multiple user setup on a server, or family PC, school network, etc.

Have fun :)

### Post by yetiman64 on 2011-04-27
> **brawnypandora0 said:**
> ...since new viruses and malware would be created especially for Linux?

There are already viruses/malware that can affect Linux, they are very rare and NONE are known to currently exist "in the wild". It has nothing to do with user numbers of the OS as your question suggests, but more to do with the underlying security structure of the OS itself. 

File and folder ownership/permissions are used in Linux compared to file properties only in Windows. The root user and its files and folders are more thoroughly separated in Linux than Windows. Hence the complaints of new Linux users from a Windows background "wanting root access" for convenience without fully understanding the implications for doing so that crop up regularly in these forums.

A good quote I read here recently, that after ~16 years of Windows use and ~3.5 years of Ubuntu use, rings VERY true to me,
> [B]All Windows security is like Swiss cheese with bullet holes.
An insecure Linux installation is like cheddar cheese dipped in mutton steel.
A secure Linux installation is like solid titanium[/B].Regards from yetiman64
