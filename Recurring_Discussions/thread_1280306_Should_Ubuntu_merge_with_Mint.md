### Post by Luke has no name on 2009-10-01
Linux Mint is an Ubuntu-based distribution. It is the 3rd most popular distro on distrowatch.com, and has caught on here in the Ubuntuforums community. It is known primarily for its default inclusion of many useful, but not *libre* media codecs, such as mp3, flash, and others. There are several other improvements, notably in the area of usability, that Ubuntu should take note of and incorporate in future (post-Karmic) Ubuntu.

Mint has a better artwork team.
Mint has a better "Start" or "Application" menu than Ubuntu.
Mint has a "Control Center".
Mint does away with the top Gnome panel.
Mint has an improved set of Software management tools.
Mint includes closed-source software.

-**Mint has (arguably) a better artwork team. **
One of the things one notices soon after Mint installation is the number and quality of desktop backgrounds provided by default. I do think they include too few varied themes (like Ubuntu), but this can be fixed.

UPDATE: Karmic is a definite improvement over previous releases in the artwork department.

-**Mint has a better "Start" or "Application" menu than Ubuntu.** 
Ubuntu divides its Menus into three drop-down panels, with (AFAIK) no hotkey for easy keyboard access, no instant text search of programs (ala Gnome-Do) and no "Control Panel/Control Center" for easy-to-navigate, windowed view of system configuration tools. Mint fixes all of this. The menu is accessible through the <Ctrl><Super> key combination. It has program searching, similar to Windows 7 Start menu or Gnome-Do's function (though not *quite* as good as "G-D"). Finally, it maintains the same level of clarity and separation of Applications/Places/System that Ubuntu has, with an even better layout.

-**Mint has a "Control Center".**
For those who have used several Linux distros, and for almost all of us who have used Windows, we are familiar with the idea of a Control Panel or Center, which centralizes all navigation of System settings and preferences to one location. Ubuntu noticeably lacks this central configuration window. Why? I can only guess the developers see no need to duplicate the navigational functionality of the "System" drop down menu in Ubuntu with a link to a windows that provides the same links.

I assert that Ubuntu's way of handling access to configuration options is *inferior* to almost every other desktop OS in existence. Mint fixes this usability mistake by creating a "Control Center", a windowed version of the Ubuntu "System" tab. As with the Mint Menu, it allows for instant keyword search on the menu item you are looking for, as if having them in all in one window isn't easy enough. It is rather rudimentary, as the configuration windows themselves do not open inside the "control center" window, but in their own popup. Fixing this would take a lot of effort, so I do not hold it against the Mint devs.  

-**Mint does away with the top Gnome panel.**
I suspect it would be controversial to call this an improvement, because to many, the two-panel layout is superior to a one-panel layout. My rationale is this: Most of the time (at least on a default Ubuntu install), the top panel is at least 60% blank. Wasted space. Remove the bloated applications menu like I've suggested, and the top panel becomes even more wasteful. With most people using widescreen monitors, and many people on small laptops where vertical space is limited, the top panel becomes a "valuable resource" hog. Mint does well with its simpler design: The only things that are lost in the conversion (that I can tell) are a few panel shortcuts and the trash logo. What you gain is screenspace and a "show desktop" hotkey that I find more useful than a shorcut to Evolution.

Bottom line: The bottom panel is all most people need.

-**Mint has an improved set of Software management tools.** 
Mint's update manager is more readable than Ubuntu's, in my opinion. It is larger, it shows the relative security of each proposed update, and shows the version change for each piece of software to be updated. It does this without eliminating any functionality of the Ubuntu Update manager. Therefore, I believe it is a welcome improvement.

Mint's Software Manager, mintInstall, is absolutely impressive. It is similar to Synaptic on the surface, but it has several notable improvements: Community ratings and comments on packages, and screenshots of applications. My only concern is that it doesn't show all packages, only "applications" (i.e. the "important packages"), which could be an issue for power users, or confusing for beginners. This is a fixable issue, though. 

After trying out the Ubuntu Software Center, it only seems to be a remix of PackageKit, which I tried many moons ago, and disliked. It doesn't search all packages, only those considered 'applications'. This bugs me as an advanced user, but then again, I use the command line more than I use Synaptic. Perhaps an option to "show all packages" would be good, rather than blocking them from view.

-**Mint contains closed source software and codecs.**
We all love free, open source software, but open source software is not inherently superior in quality to closed source, nor is closed source software always evil or avoidable.

Mint includes Flash (inc. native 64-bit!), MP3, WMV, and many other codecs that are patent protected or otherwise non-free. Some of these codecs are necessary for the everyday use of desktop users' systems, and ignoring that fact will not gain a distro any traction in the mainstream market. Mint makes its system more usable by including them. Also, Mint has some closed source software, like Opera, included in their repos by default. 

Note that there is a fully-free version of mint available, with all its improvements sans the closed items.

**This is great, but why should Ubuntu do this? Let Mint do its thing.**
Ubuntu is the forefront of consumer Linux. Its constant mission has been to bring Linux to the masses, and to make Linux an easy-to-use, intuitive system. Mint has taken extra steps in its own distro to further the aforementioned achievements. Some of Mint's tools need refinements and tweaks, and Ubuntu can definitely improve upon Mint's ideas. That's the beauty of this open ecosystem: groups can feed off each other's innovation to create ever improving products. It is time for Ubuntu to seriously look at its child distribution, Linux Mint, for ways to improve the user experience.

P.S. I tried running gnome-shell in karmic without my ATI drivers installed and it moved at 10 seconds/FRAME. I'd like to see what Gnome3 is about, but in the mean-time, it's never too late to improve on Gnome2. Besides, I get the feeling some people will be sticking with Gnome2 for a while...

### Post by earthpigg on 2009-10-02
so,

Let me know what you folks think of this tentative list of goals and philosophy for the nascient Wintergreen Project that currently only exists in my head...

The objective of the Wintergreen Project is to  port many of the unique features of Linux Mint upstream to Debian Testing from whence they will eventually find their way into the official Ubuntu repositories. First up on that list, to test the waters, will be Linux Mint's outstanding menu, ported to become the wintergreen-menu package.

The tentative philosophy and goals are: 

    * ***Giving Credit Where Due***. The only way to have Linux Mint installed is to install Linux Mint. This project will not be a replication of effort.
    * ***Needless Modifications Will Be Avoided***. At this stage, the goal is not to improve anything. Unless there is a specific reason for a given modification, there will not be one. Suggestions for improving any given port and individuals volunteering their coding efforts towards possible improvements will be directed to the folks at Linux Mint.
    * ***Each ported item should have as short a list of dependencies as realistically possible.***
    * ***Debianisation.*** Each ported item will become a source tarball and a 64 and 32 bit .deb that is compatible with Debian Testing. Once our presence is established at Debian, each ported item will have a package maintainer.
    * ***Transparency.*** There will be as few private discussions about this project as possible. A list of all discussions that pertain to this project will be established to allow anyone full access to all information.
    * ***Community-centered development.*** Mailing lists are great, but supporting the above goal would be best accomplished by keeping the discussions at the already-existing discussion forums of Debian, Ubuntu, and Linux Mint.
    * ***Release Cycle.*** After each release of Linux Mint, efforts will be made as quickly as possible to begin porting the latest improved Linux Mint packages upstream to Debian Testing. Given how the Ubuntu, Linux Mint, and eventually Wintergreen Project release cycles work, the end-state will be that 9.10-based Linux Mint's ported wintergreen packages will appear in 10.04. As neither the Debian, Ubuntu, or Linux Mint release cycles can be expected to be radically modified just to support this project, wintergreen packages in Ubuntu will always be one release behind the latest-and-greatest from Linux Mint.
    * ***Clearly Named, Intuitive, and Non-Invasive wintergreen packages.*** As an example, the first package completed and submitted to Debian will be the wintergreen-menu package. Once installed, this package will add an entry in the right click on gnome-panel -> add to panel interface called "Wintergreen Menu". If a user adds this to the gnome-panel, the result will be a ported and Debianised menu nearly identical to the one featured in Linux Mint.



What is your opinion on this?

### Post by mpt on 2009-10-07
> **Luke has no name said:**
> -**Mint has a "Control Center".**
For those who have used several Linux distros, and for almost all of us who have used Windows, we are familiar with the idea of a Control Panel or Center, which centralizes all navigation of System settings and preferences to one location. Ubuntu noticeably lacks this central configuration window. Why? I can only guess the developers see no need to duplicate the navigational functionality of the "System" drop down menu in Ubuntu with a link to a windows that provides the same links.

I assert that Ubuntu's way of handling access to configuration options is *inferior* to almost every other desktop OS in existence. Mint fixes this usability mistake by creating a "Control Center", a windowed version of the Ubuntu "System" tab. As with the Mint Menu, it allows for instant keyword search on the menu item you are looking for, as if having them in all in one window isn't easy enough. It is rather rudimentary, as the configuration windows themselves do not open inside the "control center" window, but in their own popup. Fixing this would take a lot of effort, so I do not hold it against the Mint devs.

You have discovered for yourself part of the reason we have repeatedly considered and rejected the Control Center for Ubuntu: opening and closing two windows makes changing settings even slower, most of the time, than the menus do. To solve this problem (and other problems) I sketched out [a unified System Settings window]("http://live.gnome.org/SystemSettings") a year ago, but no-one has been interested in implementing it so far.

> [i]-**Mint has an improved set of Software management tools.** 
Mint's update manager is more readable than Ubuntu's, in my opinion. It is larger, it shows the relative security of each proposed update, and shows the version change for each piece of software to be updated. It does this without eliminating any functionality of the Ubuntu Update manager. Therefore, I believe it is a welcome improvement.[/i]

I plan to redesign Ubuntu&#8217;s Update Manager for Lucid. It will be smaller and less intimidating by default. If all goes according to plan, it will also tell you how much needs to be downloaded, whether any of the updates will require you to restart or to log out, and if so which ones.

> *After trying out the Ubuntu Software Center, it only seems to be a remix of PackageKit, which I tried many moons ago, and disliked. It doesn't search all packages, only those considered 'applications'. This bugs me as an advanced user, but then again, I use the command line more than I use Synaptic. Perhaps an option to "show all packages" would be good, rather than blocking them from view.*

That&#8217;s exactly [what we plan for version 2]("https://wiki.ubuntu.com/SoftwareCenter#April%202010").

> [i]-**Mint contains closed source software and codecs.**
We all love free, open source software, but open source software is not inherently superior in quality to closed source, nor is closed source software always evil or avoidable.

Mint includes Flash (inc. native 64-bit!), MP3, WMV, and many other codecs that are patent protected or otherwise non-free. Some of these codecs are necessary for the everyday use of desktop users' systems, and ignoring that fact will not gain a distro any traction in the mainstream market. Mint makes its system more usable by including them.[/i]

Do you seriously think the reason Ubuntu doesn&#8217;t include those codecs is that we have been &#8220;ignoring that fact&#8221;?

(And did you know that Mac OS X doesn&#8217;t ship with a WMV codec either?)

> *Also, Mint has some closed source software, like Opera, included in their repos by default.*

[So does Canonical]("http://archive.canonical.com/dists/jaunty/partner/binary-i386/Packages").

> **earthpigg said:**
> It would be quite interesting to see a Free Software Foundation vs Linux Mint civil case... but I don't think the FSF feels so inclined.

It wouldn't actually be the FSF, it would be a shell organization funded by Apple, Microsoft, etc, that ***claims*** to be acting in the defense of Free Software and the GPL.

You are confusing software licensing with patent licensing. It is not the software license that discourages people from redistributing, for example, [Fluendo&#8217;s MIT-licensed MP3 decoder]("http://www.fluendo.com/shop/product/fluendo-mp3-decoder/") or [the MAD GPL-licensed MP3 decoder]("http://www.underbit.com/products/mad/") in some countries; it is Thomson&#8217;s active enforcement of MP3 patents that discourages it. As [Thomson says]("http://mail.gnome.org/archives/marketing-list/2005-February/msg00040.html"), &#8220;We are not prepared to issue an[] irrevocable, free patent license to GPL and LGPL software using our patents.&#8221;
