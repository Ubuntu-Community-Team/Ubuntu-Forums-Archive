### Post by aysiu on 2008-05-19
A few things to clarify here: [list][*]Linux servers are a huge target for malware, and yet self-propagating viruses aren't a threat to Linux servers. So shear size of target alone does not equal vulnerability.[*]Open source code is open to everyone - both crackers *and* patchers. The patchers patch more quickly (generally speaking) than in closed source. [*]Viruses are not the only form of malware out there.[/list] The first and last points go together, though, as the rise of Linux-using (or Ubuntu, in this case) home users (not server administrators) would make Linux a bigger target for malware based on social engineering (not self-propagating viruses).

If, say, the home user market share for Linux ever got up to 20%, trojans would go on the rise, and "Download this cool program" or "You need a plugin to view this video - click here to install it" malware scams would increase significantly.
