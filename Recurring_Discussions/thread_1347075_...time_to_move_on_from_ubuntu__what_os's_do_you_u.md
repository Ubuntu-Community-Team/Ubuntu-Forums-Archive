### Post by ve4cib on 2009-12-05
But just because a GUI exists doesn't mean that you must use it.  And just because something is auto-configured doesn't mean you can't open up the configuration files and tweak them to improve performance, or just see what happens when you change values.  Yes, Ubuntu has a lot of beginner-friendly tools, but underneath it all the raw-text configuration files and command-line tools still exist and can be used.

As for the questions before about Debian, yes, Ubuntu is based off of Debian.  But Ubuntu adds a lot of extra configuration tools, a different kernel, some closed-source modules, and the like.  They both use Gnome by default, and they both use .deb packages and apt/synaptic to manage them, but they are very different in a lot of ways.  The Debian and Ubuntu repositories are largely compatible (in that you can add one to the other), but specific versions and dependencies may clash, resulting in lots of fun repairing broken packages.  It's even theoretically possible to install Ubuntu, change all of the repos over to Debian-Testing, and do a dist-upgrade to turn your system into Debian, while preserving all of your Ubuntu-specific configurations.
