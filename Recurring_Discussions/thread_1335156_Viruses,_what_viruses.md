To answer you points in order:

I have an MSc in computer science (Distributed Systems and Networks), have almost 20 years in the IT industry and have been involved in Software engineering, databases and application development, web development, network management etc ...

Linux is built with security in mind.  For example, to change permissions on a system directory (folder) you need superuser privileges. In windows for example, as the default user is the administrator (in linux it is not), the user simply right-clicks on the folder to change permissions - no password required.  If a user can do it, so can a malicious bit of code. Obvious really.  Also, in the Windows world, a file can be run by it's file type, defined by it's extension (.exe, .com. .bat, .scr).  In the linux world, the permissions of any file has to be set by using chmod to change the file's execute (x) permission for either user, group or other - and to do this you have to have permissions to use chmod to set the executable permission.  Only the superuser can do this to system files.  Thus it becomes much more difficult to write a virus for a linux system that will do the same damage on as it would on windows system.  You are right about the deployment issue, which verifies my point.

No, I'm not going to suggest that Windows is flawed (although it used to be), it's different, as MS didn't really focus on security issues until the later versions of windows have arrived.

What I am arguing is that the main reason why linux doesn't seem to be so vulnerable to virus attacks is because it is much more difficult to do so, not because there are less users than windows users (which is not always true, as when users connect to web servers they are quite often connected to linux or non windows servers).  I believe google has the biggest linux cluster of servers in the world.  When was the last time you heard of google itself suffering a virus attack?

A bit of research on Google will explain how linux applies security compared to Windows.
