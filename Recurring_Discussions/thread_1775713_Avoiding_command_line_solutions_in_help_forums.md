### Post by aysiu on 2011-06-05
There seems to be a bit of a false dichotomy here.

It isn't either CLI instructions or GUI instructions exclusively.

You use what's most appropriate for the situation.

If someone is asking "I'm new to Linux. How do I install software?" A GUI solution is most appropriate. If you don't have time or energy to answer, don't. Someone else will. If you don't want to take a bunch of screenshots, I've already created [a screenshot-laden tutorial on how to install software in Ubuntu.](http://www.psychocats.net/ubuntu/installingsoftware) No need to reinvent the wheel.

If someone needs to troubleshoot a problem, a CLI solution is probably most appropriate. Just tell people how to open a terminal and then tell them to *copy and paste* commands instead of retyping them. Copy and paste is easier for them, and they're also less likely to mess things up (wrong case, not enough spaces, misspellings).

P.S. In the example article in question, the author was using an eHow tutorial (not Ubuntu Forums) to try to install iTunes in Wine (a futile endeavor).
