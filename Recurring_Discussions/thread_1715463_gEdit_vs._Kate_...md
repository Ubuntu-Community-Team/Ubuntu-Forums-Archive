### Post by fabioxxxx on 2011-11-05
kind of late but.

I used gedit for a long time (six years), but there is a really annoying bug with samba, that for me became a deal breaker, since all my files are in a samba server.

[https://bugs.launchpad.net/gedit/+bug/34813](https://bugs.launchpad.net/gedit/+bug/34813)

every day i work with several txt documents (contacts list,task list,commands, tutorials, info about something) , and edit some html and php.

mem usage ubuntu 10.04 x64 
GEDIT :  FILES OPENED :6  : MEM USAGE 232MB
KATE  :  SAME 6 FILES OPENED PLUS OTHER 2 FILES : MEM USAGE 32MB

KATE
cons
-a bit slower to open files.
-different keyboard shortcuts. (if you use gnome)
-interface has a lot of options , be careful to not unwittingly tweak the wrong one.(like read only)

pros
+no samba bugs
+auto complete enabled by default
+better session save
+uses less memory
+better programming tool 


the interface and shortcuts where a little bit different, but easy to customize. i am happy with kate for now.

if you are using gnome, and don't have the need to edit files in a samba share i think gedit it's a better choice.

for system files i always go with vim.
