### Post by holadebob on 2011-05-13
The General Help thread seems to be dominated by..

1. Those that understand that Ubuntu is a DIFFERENT system, they take the time to learn the new system and stick with it until they have learned it good enough to kick back and use it as the tool they wanted. They seem generally interested in reliability and stability, so they generally stick with the conservative LTS versions, and accept any limitations the system has.

2. And then there are those non savvy computer users, or dedicated Windows users who are trying to set up the newest Ubuntu beta versions, have the problems that are normally associated with a beta, then threaten to run back to Windows when they get frustrated. (I can just picture the Ubuntu developers cowering in fear :) )

3. Finally there is a group of savvy computer users who can write code, use the terminal like a programmer, and who are looking for answers to solve the vexing problems of the beta versions.

My humble suggestion:

  1. Since General Help seems to contain mostly posts that have problems with beta versions, why not have a Forum labeled "General LTS Release Help", and relabel General Help as "Beta Release Help", which could be prefaced and described as a forum for computer savvy users.

  2. Initiate a stronger caution for non savvy users to avoid the beta releases.

Thank you for your time....

### Post by 3rdalbum on 2011-05-16
As far as I know, very few Windows users install beta versions of Ubuntu. They're usually in the mindset of "the beta versions are kept a secret" like they are in proprietary software.

They do, however, use the latest release version of Ubuntu; like most home Ubuntu users should. The latest release version is 11.04. The latest LTS version, 10.04, is about a year out-of-date for regular users.

I would, however, welcome an effort to get savvy Ubuntu users to test the alpha and beta snapshots of new versions of Ubuntu, so they can report bugs. Obviously not non-savvy users, at least until the betas or release candidates, but there are a whole heap of people who could and should help test, who don't!
