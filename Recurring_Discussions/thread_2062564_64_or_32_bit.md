### Post by drmrgd on 2012-09-25
If your CPU supports it, there's really no reason to not install 64-bit.  With 64-bit you'll be able to use more RAM in the system, and the OS should run a little faster overall with tasks as it will be able to address larger chunks of memory.  

One quick way to find out if your CPU will support 64-bit is to run the following in the terminal:

```
lscpu
```

On this VM that I have running, I can see that my CPU will support 32-bit and 64-bit mode (no surprise about the 32-bit!):

```
Architecture:          x86_64
CPU op-mode(s):        32-bit, 64-bit
Byte Order:            Little Endian
CPU(s):                4
On-line CPU(s) list:   0-3
Thread(s) per core:    1
Core(s) per socket:    1
Socket(s):             4
NUMA node(s):          1
Vendor ID:             GenuineIntel
CPU family:            6
Model:                 30
Stepping:              5
CPU MHz:               2926.030
BogoMIPS:              5852.06
Hypervisor vendor:     VMware
Virtualization type:   full
L1d cache:             32K
L1i cache:             32K
L2 cache:              256K
L3 cache:              8192K
NUMA node0 CPU(s):     0-3

```

You could also look for the string lm in /proc/cpuinfo as that shows that it is 64-bit compatible:

```
cat /proc/cpuinfo | grep lm
```
