### Post by matthew on 2010-05-04
I have a UNIX book from 1984 sitting on my shelf (one of many) called [The UNIX Programming Environment]("http://en.wikipedia.org/wiki/The_Unix_Programming_Environment") by Kernighan and Pike. In it they state that the term "UNIX" could mean several things, depending on context:
[LIST]It could mean the operating system kernel by itself
[/LIST]
[LIST]It could meant the kernel along with essential programs like compilers, editors, programs for copying and printing files, etc.[/LIST]
[LIST]It may even include programs developed by you or other users to be run on the system[/LIST]
Which they used the term "UNIX" in the book, they said that context would make it clear what they meant. (pg 1)

Stallman was a part of that culture. The GNU project was an attempt to replicate that early culture, which was also open and code sharing, but didn't have a free license other than specific permission from specific entities that was given to hackers to use the code. That's where the FSF, GPL and so on came from.

However, he as much as anyone would know this history that predates his. Yes, the GNU tools are a vital part of the system, but they would be worthless without a kernel on which to run them. Calling the system "Linux" even though the GNU tools are vital is not disrespectful, it is following a tradition handed down from UNIX.

I won't call it GNU/Linux, not because I disrespect the GNU project, but because the demand is selfish and unreasonable in the context of a greater history.

Anyway, that is my opinion. Feel free to have your own.

### Post by prions on 2010-05-04
"What you're referring to as Linux, is in fact, GNU/Linux, or as I've recently taken to calling it, GNU plus Linux. Linux is not an operating system unto itself, but rather another free component of a fully functioning GNU system made useful by the GNU corelibs, shell utilities and vital system components comprising a full OS as defined by POSIX.

Many computer users run a modified version of the GNU system every day, without realizing it. Through a peculiar turn of events, the version of GNU which is widely used today is often called "Linux", and many of its users are not aware that it is basically the GNU system, developed by the GNU Project.

There really is a Linux, and these people are using it, but it is just a part of the system they use. Linux is the kernel: the program in the system that allocates the machine's resources to the other programs that you run. The kernel is an essential part of an operating system, but useless by itself; it can only function in the context of a complete operating system. Linux is normally used in combination with the GNU operating system: the whole system is basically GNU with Linux added, or GNU/Linux. All the so-called "Linux" distributions are really distributions of GNU/Linux."

If Richard Stallman says it, it must be true.
