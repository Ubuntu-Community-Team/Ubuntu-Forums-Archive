### Post by handy on 2009-10-26
I use Arch/Openbox with the xfce4-panel.  

As long as you don't use xfce4-session then you will keep the Openbox speed. You can use gtk-2 themes, & (from memory - as I don't use desktop icons) if you choose to install & run the xfce4-settings, you can have icons on your Openbox desktop.  My setup has all of the looks & functionality of Xfce that I desire.  Though I consider the Openbox menu to be easier to configure, more visually appealing & more practical than that of Xfce4.

I use the following tools with Openbox to make life easier:

OBMenu
Openbox Configuration Manager

If you want to use a screen background, feh in combination with GQView makes it really easy to set persistent background & to easily change them via GQView.

### Post by etnlIcarus on 2009-10-27
Xubuntu comes with a lot more Gnome-centric services and tools. The end result is it only ends up being 0-5% 'faster' than Ubuntu/Gnome, for most people.

You can *probably* get 'xubuntu' to be pretty speedy but it'll involve installing only a base system with the mini.iso and installing the xubuntu-desktop meta-package with the -R (aptitude) or --no-recommends (apt-get) arguments.

Optionally, you can go full-geek, installing only the components of Xfce you want on your base system.

I opted for the last option. There's a lot of dicking around with things like (insane) policykit defaults but the end result is very slick.
