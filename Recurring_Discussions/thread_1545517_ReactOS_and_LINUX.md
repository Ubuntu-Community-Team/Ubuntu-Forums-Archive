### Post by FlyingIsFun1217 on 2010-08-04
A few things to clear up.

There is indeed a future for both Linux and ReactOS. What it entails is not known, nor can it be by anyone.

ReactOS is aimed to be a NT clone, as Linux was developed as a Unix clone. Each share a different purpose, and do not necessarily aim to compete with each other, so there is no reason that they can't both live together. Windows is not an alternative to Linux, they are very different types of operating systems.

With that, ReactOS does NOT use "linux like policies", it aims to replicate Windows down to the function name, which is also why many parts of WINE are not used. WINE is built as a sort of server on linux, one which can run Windows programs through proxy. ReactOS aims to have binary compatibility, meaning it would be like running an actual copy of Windows.

There are bound to be varied opinions on the project, but honestly, what they are doing is in no way competing with Linux, only to create an open-source Windows.

FlyingIsFun1217
