### Post by user1397 on 2016-06-02
Learning linux has nothing to do with learning programming, although you can definitely learn programming in linux.  In fact, often times many developers prefer linux over other systems because it is usually easier to get development tools on linux than windows for example.  Then again, you can perfectly program on windows or OS X as well (lots of devs prefer OS X too, especially if you want to develop apps for iOS or OS X).

If you want to get started with linux basics check out this new site that just popped up [https://linuxjourney.com](https://linuxjourney.com)  Seems to be pretty concise and easy to navigate (found it on [www.omgubuntu.co.uk]("http://www.omgubuntu.co.uk") )

If you want to learn programming, there are a myriad of ways to go about it.  Google is your friend, as is stack exchange.  Wikipedia is actually a great resource as well, their articles on programming seem to be quite thorough.  

You could go to a 4 year university and learn from the basics to the advanced computer science concepts and become a good programmer, but only if you take your classes very seriously and also do a lot of side learning apart from your homework.  Otherwise, you can totally learn programming on your own time by using tons of free tools and guides online (or you could always buy a programming book).

You can go as hardcore as trying to read The Art of Computer Programming by Donald Knuth, which goes from the absolute basics of computer science all the way to the most advanced concepts, or you can jump in to a random language and learn basic programming concepts like variables, functions, arrays, loops, iterations, objects, etc.  

The programming language that you start with doesn't really matter, although some have a much higher learning curve than others.  Once you understand one language you can pick up almost every other language easily as most concepts transfer over and all you have to change is a bit of syntax.  Many people would recommend python to start out as it is more human readable and has less verbosity than something like Java or C++.  If you start with python, I'd go ahead and start with python 3 and not python 2 because eventually everything will be python 3 and python 2 will be phased out (this is taking a long time, but eventually it will happen).

One of the most important points about programming that I can't stress enough, is that to truly learn it is important to have some kind of project you can work on, ideally something that really interests you and keeps you interested.  If you just go on codecademy and go through one of their tutorials and do nothing else, you won't learn how to program.  You might learn some basic concepts but to retain them and to practice you have to work on something. Not to bash codecademy or any other guide like it; they're actually really awesome and have made introducing programming to average users very accessible.  You have to tinker, get in there, break your program, google for answers, and then as you keep learning you will start becoming more proficient.  Think of some problem you run into in your daily computer usage, something that annoys you or isn't efficient, and try to come up with a way to fix that or get around it.  Games are popular projects (although you'll start out with console based games at first, since GUI applications are more complex and are more advanced, and you have to learn console basics first before starting GUI development).  

By far the languages used the most nowadays (in no particular order) are javascript, java, python, ruby, php, C, C++, C#, and Objective-C but there are dozens if not hundreds of languages.  Some are general purpose, some are suitable only for specific tasks, some are web oriented, some are desktop oriented, some are server oriented.  All the major ones are general purpose and object oriented, with the exception of C (general purpose yes, object oriented no; that was the whole premise behind creating C++).
