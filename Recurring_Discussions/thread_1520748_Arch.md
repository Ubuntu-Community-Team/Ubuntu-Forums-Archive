### Post by Windows Nerd on 2010-06-30
> **Timmer1240 said:**
> Ior Im just gonna have to get really geeky for a day and try to do an install!
I am afraid this is what you are probably going to have to end up doing: a non-transparent user install is directly against the Arch Way: Arch is designed to have the user configure the system so that he/she has a better understanding of how the OS works and can configure the OS to exactly his/her needs. What you end up for all that extra work on install is a lightweight system with exactly what you want on it. 

It is not insanely difficult to install; follow the beginners guide and consult Google or the Forums if you have some questions about it. I did it in a test VM: it took me about 3 hours, but I learned a lot more about Linux and Arch in that time than I would in a month otherwise.

[For the Beginners Guide]("http://wiki.archlinux.org/index.php/Beginners_Guide")

[The Arch Way]("http://wiki.archlinux.org/index.php/Beginners_Guide#The_Arch_Way")

I myself am going to be installing Arch on my new desktop when it comes in in the next week or so. It does require a bit of manual maitainance to keep working (especially with updates or installing new programs - they may ask you to modify a configuration file accordingly)

Well arch is designed around the principals of extreme simplicity. By simplicity, though, I mean not necessarily user friendliness (though it's fairly easy to use by my standards) but *lightness*. It comes installed with almost nothing- any packages you want, you install yourself. The upshot of course is that you have all the software you want with no fluff. The downside is lack of out of the box style ease of use. If someone designed an arch based distro to be like ubuntu than it would cease to be arch- it would become yet another redundant distro. Don't fear arch- if you're comfortable enough in the world of linux, it'll work out. Testing it out in Qemu or virtualbox for a test-drive is something I'd highly recommend.
