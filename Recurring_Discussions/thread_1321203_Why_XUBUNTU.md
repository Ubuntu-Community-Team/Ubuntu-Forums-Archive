I did not "time" xfe since it would not be fair considering VirtualBox, however, from a purely theoretical stand point, it seems that it would not be faster at all.

Any comments/thoughts?

People hear reports of Xubuntu or xfce being so much faster than KDE or Gnome, but then when they switch to it, they complain about not having all of the features that their old desktop had.  What do the devs do?  Do they risk loosing users by keeping the desktop light?  Or, do they load it up and hope that people don't notice the extra load times?  This kind of situation is good, though, because the developers will strive to add features as efficiently as possible to keep the 'fast' reputation; but there is obviously a limit to what they can do.

My suggestion, if you like KDE better than Gnome, is that you stay with KDE.  I've never tried it, but from what I've read I get the impression that it's the most sparkly, fully featured DE out there out of box.  If you like that, stick with it. ### Post by XubuRoxMySox on 2009-11-10
I just ended an obsessive quest for "super ultralight Ubuntu" for a couple of reasons:

First, **if it's Ubuntu *at all*, it isn't going to be lightweight.** Ubuntu's efforts to be all things to all users simply makes it "bloated" compared to many other Linux systems. But compared to Windows (XP anyway, which is all I had ever known 'til I started Ubuntu last March), all the 'buntus are lean and mean!

Secondly, **"lightweight" is over-rated.** At least for me it was. I felt I was missing out on something awesome when I began reading thread after thread in UF about "lighter, faster, leaner, less 'bloated,' more speed, gimmie less weight and more speed!!" From power-button on to ready desktop in 0.68 seconds (why does that matter? I'm never in such a hurry)!

So I tried the "lighter, leaner, faster" variants; dumped Gnome for the ultralight (but *very* buggy) LXDE environment, and I tweaked and tuned and trimmed and toned to the Nth degree. And guess what? Even my most minimal, most lightweight remix (minimal Ubuntu + LXDE) was not noticeably faster than big ol' supposedly "bloated" Ubuntu Jaunty had been on my 5-year-old Dell Dimension. What had I been missing? Hype. Bragging rights to the fastest boot-up speed, lowest CPU and memory usage? Pft. I blogged ([here]("http://robinzrants.wordpress.com/2009/11/04/lightweight-is-over-rated/")) about my obsessive pursuit of ultralight Ubuntu and why that quest is now over.

So it came down to just choosing a favorite desktop environment and favorite applications. I share my computer with lots of kids (at a dance studio - we're artists and athletes, not computer geeks), so it has to have a supersimple interface that doesn't require "coaching" on how to use the 'puter. LXDE fit the bill nicely for simplicity, but it's so dang buggy (and *more so on Ubuntu than any other distro* for some reason). I've learned that LXDE takes most of its code from Xfce anyway, so I decided to try Xubuntu when Karmic was released.

Whoa! It's as fast as my old minimal Ubuntu/LXDE remix, but so much more functional. It has all my favorite applications already (all I added was Skype). It automatically fixed a video problem I had given up on, and it seems to offer the best of both worlds: Simplicity, speed, beauty, and all the power of Ubuntu under the hood. I like it much more than Gnome Ubuntu, and I think the devs have achieved an ideal balance between "lightweight" and feature-rich. Xubuntu Karmic is just right for this simple non-geeky boy and runs fast on the 5-year-old Dell. It's more than "just Xubuntu with Xfce tied on." It's an awesome balancing act, wonderfully done and practically flawless on my older hardware.

-Robin

Many people say that XFE is "light-weight." However, all the default applications for xubuntu seem to be simply the corresponding Gnome applications (Firefox, Pidgin, Transmission ...) AFIK Gnome applications depend on common Gnome libraries and KDE applications depend on common KDE libraries (GTK and QT). So KDE applications load faster under KDE and Gnome applications load faster under Gnome. Under this logic I expected to see xfe applications that load fast under xfe. However, Gnome applications would in fact load slower under non-Gnome environment so in fact xfe should be slower. The only advantage that I can think of is a potential decrease in memory use, since libraries that are not used by any program at the moment would be removed from memory (or marked as disposable).

I did not "time" xfe since it would not be fair considering VirtualBox, however, from a purely theoretical stand point, it seems that it would not be faster at all.
