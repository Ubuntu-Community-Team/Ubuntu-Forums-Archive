### Post by aysiu on 2010-12-17
Most of those "misconceptions" are just exaggerations of the truth.

> Linux is a command line OS There is a graphical interface you can put on top of the command line (and most major Linux distros offer that option by default), but it is at its core still a command-line OS.

> Linux is for geeks only You don't have to be a geek to use Linux, but you are likely a geek if you use Linux, because it's not mainstream, and it's not easy to get on a preinstalled computer you actually want.

> Nobody is using Linux Relatively few people use Ubuntu (or Fedora or PCLinuxOS, etc.) as their main desktop/laptop/netbook OS.

> There are no games for Linux A lot of high-end commercial PC games either don't run in Linux, don't run well in Linux, or require weird Wine hacks to run in Linux. That doesn't mean there are *no* games in Linux, but if you are heavily into PC (as opposed to console or no) gaming, I'd recommend using Windows.

> A lot of hardware doesn&#8217;t work under Linux If you use consumer-targeted hardware for desktops, laptops, and netbooks, you will get a lot better driver support (usually third-party) for Windows and Mac than for Linux. The good thing about Linux is that it'll support more out-of-the-box. So if it works immediately in Linux, great. If not, not great.

> You have to ditch Windows to use Linux Dual-booting and virtualization are options, but neither is ideal. I would highly recommend ditching Windows if you plan to use Linux extensively. Dual-booting and virtualization are good for just test-driving Linux.

> 'linux doesn't get viruses/malware' It doesn't get viruses, but it can get malware. It pretty much doesn't get malware, though, either. Read up on social engineering, and you'll be fine.
