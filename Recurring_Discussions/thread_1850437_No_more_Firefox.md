### Post by Mr. Picklesworth on 2011-09-28
> **Copper Bezel said:**
> None of this is true. Chrome at present installs from a PPA in Ubuntu, so it's nothing to do with the maintainers, and you always have the choice between the current stable and unstable versions. Of *course* it updates with the system instead of updating itself - you're using Linux.

Just to clarify a little bit, Google *does* officially package Chrome for Ubuntu, using native 64-bit and 32-bit deb packages. Those packages automatically subscribe you to an apt repository for regular updates, too.

And, yeah, I'm not LinuxFan999 would be saying this if he'd tried Chrome on MacOS :b
(I think it's worth noting that ChromeOS is pretty effectively ChromiumOS, which is effectively Chromium + some fancy patches, so I think they have a pretty significant investment in their Linux build).


> **collisionystm said:**
> That would be rediculous. Ditch firefox but yet integrate thunderbird? I think Ubuntu and Google are doing dealings with eachother. The 2 seem to be more a-like everyday. The new droid center and the new ubuntu software center. The settings gear in 11.10 and gmail. The rumors of google using Ubuntu.... and now making chrome the default browser. It seems like Ubuntu is becoming Google's mothership.

Perhaps I'm misunderstanding you, but surely if there was a secret dealing going on, Thunderbird *wouldn't* be the new default mail client? I don't know what goes on at Canonical but I do pay a bit of attention to Ubuntu, and I, for one, have noticed no shady underhand dealings there. That said, yes, Google tend to be good to Ubuntu. They do use Ubuntu on lots of workstations, after all, and they let us participate in Summer of Code every now and then.

I guess, when you mention &#8220;the settings gear,&#8221; you're talking about the new System Settings application and its Online Accounts panel. That's part of [Gnome Online Accounts](http://live.gnome.org/OnlineAccounts), which is a project that will eventually give us something like Palm's Synergy feature. You authenticate all your online services in one place and any application on your desktop can (with permission) access them. You don't need to enter your account details more than once, authentication is handled sensibly, and app developers can do cool stuff a little more easily. Today, the feature is limited to Google accounts, but it's designed so all sorts of accounts can be supported. In the future, it'll be way more powerful.
