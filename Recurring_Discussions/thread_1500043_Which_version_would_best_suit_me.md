I just wanted to make a suggestion to you as well!  Download the ISO from the link above, referenced by wojox.  Once you've burned the ISO image, there are a couple ways to dual boot Ubuntu with Windows.  You can boot directly from the CD and install Ubuntu that way. If you choose to leave the Windows partition on the disk, it will automatically install the GRUB loader (Ubuntu's boot menu for OS's), and it will include a choice to boot into Windows.

The other way is to insert the CD rom into your drive while you're still in Windows, and you can install Ubuntu right inside Windows!  You will need to create a separate partition to install Ubuntu if you choose to install it that way.  If done this way, it does not install the GRUB loader.  You'll see your normal boot up screen, but you'll see Ubuntu as a listed choice underneath Windows 2000.

Hope that helps!

### Post by JoelOl75 on 2010-06-03
I'd recommend at least 20GB for the ubuntu partition. Also note you will need a swap partition as well. I usually set the swap partition at 1MB or however much system memory you have, whichever is greater. The ubuntu installer should shrink the windows partition and automatically create these for you.

Also if you have a 64bit capable processor there's no reason not to run the 64 bit version of Ubuntu.  64 Bit capable processors are the Core 2 Duo (Not the core duo) and better from Intel. Some newer Celeron processors are 64 bit.  On the AMD front anything Athlon 64 will do (Athlon XP processors are 32 bit)

If your not sure run this utility in Windows and it will tell you if your processor is 64 bit or not.

[http://www.grc.com/files/securable.exe](http://www.grc.com/files/securable.exe)

BTW, If you have 116GB free, I'd probably go for the slightly more complex '2 partitions' way of doing things. (/root and /home). /root is like C: on windows, all the system files are there. /home is like the desktop, all your downloads, etc go there. You can run from asingle partition, but 2 is much nicer. 

The advatage over a single partition is that if you decide to format and reinstall ubuntu (or change to a different linux distro) you can keep all the data in /home, as you only have to format /root. 

I'd make /root about *GB (should be more than enough space) and home...well, as big as you want ;) Probably I'd go for at least 5GB, up to about 40GB.
