This was originally a post at the end of a support thread, and the OP made no mention whatsoever about Wine or running Windows programs in Ubuntu, so I've splintered this off to its own thread.

Personally, I think this isn't a great idea, but if you think it is and want to vote it up, here it is on the Brainstorm site:
[Idea #54: Make WINE a part of the default install ](http://brainstorm.ubuntu.com/idea/54/)

This is what Mark Shuttleworth has to say about it: > They both play an important role, but fundamentally, the Free software ecosystem needs to thrive on its own rules. It is *different* to the proprietary software universe. We need to make a success of our own platform on our own terms. If Linux is just another way to run Windows apps, we can't win - os/2 tried that.

### Post by aysiu on 2010-10-27
Since some people have brought up space on the installer disc and the need to find native Linux applications, I'm just going to reiterate that my primary objection is to false expectations. If Wine is installed by default, it's really sending the message to new users "Worried your Windows programs won't work on Linux? Don't worry. Just use this program to run them" when the truth is that many Windows-only programs either do not run in Wine, run terribly in Wine, or run in Wine only after complicated workarounds.
