### Post by mcduck on 2011-03-04
To make a operating system lighter you need to use lighter programs and lighter desktop environments, which pretty much means less features, less things happening automatically and more manual configuration etc.

In other words, making a system that's light, has all the bells and whistles of a modern OS, and is usable by anyone, is really hard.

It's pretty much that you can pick two out of three from light weight, ease of use and features. Getting all three is where the problems start, because adding more features, and especially the kind of features that make things easier, quickly also makes the OS and programs more resource hungry.

...and of course if you want to run the whole OS from RAM that means it has to be very light on RAM usage, so you can load it all into RAM and still have enough RAM left to run the programs you want to use and handle all the tasks you do on the computer. Doing such thing with a distribution like Ubuntu would require a lot more RAM than what Ubuntu's current minimum requirements are. And you'd end filling your RAM with loads of stuff you'd most likely never actually use...

(in a technical sense *all* operating systems and programs run off of RAM. Processor can only handle data that's in its own cache or in RAM. The difference is just if you simply load everything into RAM when you start the computer, or if you load things into RAM when you need them.)
