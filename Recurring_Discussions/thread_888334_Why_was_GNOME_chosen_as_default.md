### Post by samjh on 2008-08-13
> **jittopjose said:**
> i heard that there is some licensing issue with kde as it used qt libraries. i think that is the main reason for starting gnome project, a 100% open source Desktop. i dont know much about it.That was the reason for starting Gnome.

However that is no longer the issue now, as the Qt libraries are released under GPL for open-source projects.

The key reason, in my opinion, is that Gnome is generally simpler and perhaps easier to use than KDE.  I say "perhaps" because there is no objective and scientific evidence to support such a claim, but that claim is touted by the Gnome project as one of its strengths.  As Ubuntu is aimed at being as user-friendly as possible for beginners, Gnome's simplicity suits it well.

Keep in mind that Ubuntu also has Kubuntu for KDE, and Xubuntu for Xfce, not to mention other derivatives with alternative desktop environments.

Linus likes KDE because Gnome lacks customisation features that he wants.  He also complains about the Gnome developers' lack of willingness to expose functionality to users via the GUI.

Gnome is quite customisable, but to really customise Gnome, you need to be familiar with GConf options, which is quite difficult.  KDE allows almost everything to be configured via GUI.

As for "KDE is too Windowish" argument, KDE is as Windows-like as Gnome is Mac-like.  In other words, KDE is not like Windows!  It looks, feels, and behaves very differently.

I'm a Gnome user, BTW.  But I like KDE for its GUI-based options.  I wish that Gnome project leaders would allow Gnome to be easier to configure via GUIs.  Exposing options via GUIs does NOT reduce usability, no matter what some Gnome developers might claim (usability is a subject I studied in-depth at university, so I'm not just talking trash here).
