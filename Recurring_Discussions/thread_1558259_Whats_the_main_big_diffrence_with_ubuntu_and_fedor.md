### Post by Ctrl-Alt-F1 on 2010-08-22
Fedora leans more strongly toward the FOSS philosophy.  As such they don't package all the proprietary video drivers and stuff.  However, the version of their open source drivers is usually a little bit further along then Ubuntu's (because it is a more experimental distro).

Ubuntu is the distro of choice if you don't want to have to know how to do anything.

Fedora is a close match in terms of usability but I think it is slightly more complicated in terms of not everything being done for you.

### Post by WalmartSniperLX on 2010-08-22
I have so much to say as I am a long time Fedora fan and user. The base of Fedora is backed by Red Hat. 

Fedora [Core] was originally designed to be a community test bed for the following releases of Red Hat Enterprise. It uses the RPM package management, YUM repo package handler (as opposed to apt-get) and demonstrates literally the latest builds of program packages available. It is far more 'cutting or bleeding edge' than Ubuntu. This is great, however because of how young the code is, it can have stability issues as it truly is an 'experimental os'.

I would say Fedora and Ubuntu are the top 2 desktop ready, large distributions. It terms of usability, Ubuntu is a little more user friendly (slightly) and I've noticed that Ubuntu as a whole is a little more stable than Fedora out of the box. 

I actually tried running Fedora 13 on my main system but reverted to Kubuntu because I need an OS that is as close to the term 'just works' for this primary machine. Fedora pretty much 'just works' as well but compared to this latest launch of Ubuntu, Fedora didn't impress me so much. Maybe next release will get them ahead as (IMO) they once were.

### Post by MCVenom on 2010-08-22
Fedora's open source drivers don't suck. :lolflag:

Basically, while Ubuntu is an attempt at a stable, easy to use desktop system based on Debian and aimed at the 'mainstream user', Fedora is the testing base, if you will for features that will make it into Red Hat's Red Hat Enterprise Linux distribution. Fedora is based on a different package manager and format than Ubuntu (Fedora's Yum & RPM vs Ubuntu's APT & *.deb), and Fedora also tends to be more bleeding edge than Ubuntu. It's suitable for desktop use (if you're somewhat used to Linux OSes), though it's not particularly marketed as such. It also has a policy about most, if not all of the software available by default being free and open source.

I think Ubuntu's rival, if you were to be looking for one (and if and OS must have a rival) in the Linux space would be Mandriva. It also aims at being user friendly and stable, and is RPM based.
