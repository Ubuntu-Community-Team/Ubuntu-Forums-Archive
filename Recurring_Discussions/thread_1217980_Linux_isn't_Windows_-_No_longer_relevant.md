### Post by aysiu on 2009-07-21
I don't like people linking to that article on the Ubuntu Forums.

A lot of the specific points are interesting and relevant (the motorcycle/car analogy, for example), but the piece is actually about Linux in general and not Ubuntu Linux specifically.

It is also not just about the technical details of Linux v. Windows but also the sociological aspects of each operating system as well. After all, much of the supposed learning curve of Linux comes from 99% of Linux users having to install and configure Linux themselves and 99% of Windows users buying Windows preinstalled from OEMs.

So the typical course of events for a Linux user is:
1. Buy Windows preconfigured
2. Use Windows
3. Try to install Linux
4. Try to configure Linux and troubleshoot installation problems
5. Use Linux

And the typical course of events for a Windows user is:
1. Buy Windows preconfigured
2. Use Windows

So if you were just comparing *Use Linux* with *Use Windows*, then it wouldn't make sense to say there's a learning curve. But we don't live in a cultural vacuum. We live in a Windows-dominated computing world, so the likelihood that someone has grown up using Linux exclusively is far lower than her having grown up using Windows exclusively and having to de-learn Windows and learn Linux.

The command-line bit again has a sociological component reinforcing it. Even if a GUI solution is available to a problem, you will probably get a CLI solution even on the Ubuntu Forums because with Linux you are more likely to have to get help online through a text-based forum or mailing list, and with Windows you are more likely to get help from someone in person (a friend, co-worker, family member, the Geek Squad).

If I'm standing over your shoulder, I can easily say "Click on that button. No, not that one. The one up there." But if I'm typing instructions on a text-based forum like this one, it's a lot easier to say "Copy and paste this command," and it'll actually yield an error message if something goes wrong instead of "The window just disappeared" or "Nothing happened" or "That didn't work."

Linux as we know it now will not become popular enough in the near future so that most Linux users will also know in person someone who can help with Linux problems, so I think that culture of seeking help online will stay. Even if Google Chrome OS becomes successful, people will then learn how to use Chrome OS and not Gnome, KDE, or .deb and .rpm.
