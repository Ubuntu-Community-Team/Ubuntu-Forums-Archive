### Post by perspectoff on 2011-12-09
Windows 7 is sooo much slower than either Kubuntu or Ubuntu.

Kubuntu is far more user friendly and customisable for a wider range of desktops at the moment (especially laptops, netbooks, and desktops). Ubuntu from Natty onward (using the Unity user interface), on the other hand, is more geared towards the smartphone and tablet markets (and to me is irritating on other platforms).

We have Windows 7 and Kubuntu on all our computers (dual-boot) but about 95% of the time is spent in Kubuntu.

Having said that, Windows 7 has built-in voice recognition, and it is better than Dragon Speak or ViaVoice. That is a game changer. It is possible (in a quiet room, anyway) to control Windows 7 hands-free and dictate into documents with this built-in accessory of Windows 7.

The Ubuntu team is working on voice recognition and integration into the desktop, but Microsoft has clearly bought a heckuva package somewhere to include in its latest Windows OSs.

Microsoft has now bought Skype, too, and I imagine it is a matter of time before Skype support for the Mac and Linux is dropped or hobbled. (For the time being, Skype works fine on my Kubuntu systems.)

Of course, Google is a Linux system and currently Google is driving a lot of Linux-oriented app development (much of the Google Summer of Code revolves around Linux projects).

For a parent choosing an operating system, Linux is hands-down the way to go, because Linux is the operating system (with Google's ascendancy) that is now the world's dominant OS. Further, for a kid who wants to be involved in the software world (and Google Summer of Code is largely oriented towards students), Linux is the way to go.

Google is an ecosystem in which there is no privacy, however, and it is still possible to maintain privacy in the (K)Ubuntu/Debian ecosystem, and that is a major reason, IMO, to choose (K)Ubuntu over Google, currently.

As to schools having only Windows experience... where are you? Some inner city ghetto? I haven't come across a school with "only Windows experience" for many years. By the way, most university computer labs are Linux based; there are almost no computer labs in universities that use Windows. (Our local university uses exclusively Slackware, to my chagrin, although I am trying to change that).

Don't forget that most of cloud computing these days is Linux-based (cloud computing was always Unix or Linux-based from the start) and UC Berkeley and UC Santa Barbara are development nexi for cloud-computing.

Most supercomputing clusters use a Linux OS (although most are OpenSuse-based), now, as do many movie-making studios (which use a Kubuntu-based system). The point is that while you are focusing now on lower-level education, what you should be doing is focusing on what skills your child will need in the future, and that is increasingly Linux-oriented. 

Windows is a platform for games, largely. If you want your kids to merely be able to play games and log into social media sites, then by all means use Windows. If you want them to be a little more educated about computers in the long run, and perhaps be prepared for advanced education, then it is best to start them on Linux now.

BTW, there's nothing wrong with a dual-boot system. You can have both. I don't have a single computer that has only one OS or the other exclusively; all the computers I own still have Windows (and some even have Mac OSX) on them. I just squeeze Windows (or Mac OSX) into a small partition (an advanced skill, admittedly) since I use those OSs so infrequently.

Bottom-line recommendation:

Kubuntu, but with attention towards Google
