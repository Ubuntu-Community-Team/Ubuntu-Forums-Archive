### Post by Frak on 2010-09-05
Too many startup applications, almost all the time.

Filesystem fragmentation? Unless you're running a high-demand system, fragmentation is a non-issue in terms of speed (you should defrag before partitioning to keep contiguous blocks compacted, avoiding corruption).
Malware? Any malware that slows down your system was poorly designed. A good virus is one that goes undetected.
Left-over registry values/files? Linux and Mac OS X have the same issues. Linux (DEB/RPM) will retain configuration files and GConf registry settings after removal, without purge. Mac OS X keeps configuration values after removal (without special tools).
Small files being left all over the disk? Has very little effect on application run-time. Depending on size, could affect application load-time considerably or not-at-all. For example, loading settings from n-amount of configuration files is much less efficient than loading settings from a registry hive.
Registry Errors? For one, it's dangerous to mess with the registry, even using CCleaner could hose your installation. Even so, fixing errors in the registry give minimal performance increases.

If Windows is slow, press Super + R and type "msconfig" (without quotes) and disable unnecessary services under the "Services" tab (Check the "Hide All Microsoft Services" box!) and all unnecessary startup items under the "Startup" tab. It will ask you to reboot, if you want, do it, if not, go about your business and just reboot later. If an application is running slow, like Firefox, clear your browsing history, caches, etc.

### Post by murderslastcrow on 2010-09-05
I know it's mostly been said, and people are getting a little uppity here, but I'll rephrase for them.

The most prominent reasons for slow downs in Windows are the following.

1. NTFS filesystem needs defragmentation when files are created or moved. If things move around a lot, or you download a lot, it will increase the time taken to move around on the disk, making it take longer to load files/programs. Until the user defragments, this can't be fixed. So basically, the more you use it the slower it gets- doesn't have so much to do with time.

2. Spyware encountered innocently on the internet which takes up RAM and CPU cycles in order to do its nefarious work.

3. Duplicated installation of libraries for programs that will accumulate over time, and the RAM cost taken up as more of these programs are opened or used. You see, in Windows, this accumulation of RAM-suckers occurs because the motivation to create or use software is for profit, to send ads, get more users, among other things.

So, rather than being as light and feature-filled as possible, and utilizing memory well, sometimes programs gain dominance that are not made well, or you need several poorly-coded applications to do what one Linux program could do, etc. There are many reasons for this, but in general, Windows programs are messier and you need more of them, or they decide to run when they don't need to. This, again, depends on the time spent on your computer and what you need to do, but applies to most tasks beyond web browsing.

To blame it on the user when there are obvious issues with the platform itself, as well as the typical low quality of applications made for that platform, is unfair and a very closed, biased statement. It's not ALL the user's fault, and what they user does isn't only motivated by them, but by the restrictions of the platform as well.

### Post by blueturtl on 2010-09-06
> **1492 said:**
> Why does Windows slow down you computer over time but other OS don't?

I'm just asking out of curiosity.  I'm not asking how to speed it up, just why it does that.

There are many reasons for this effect.

1) Early versions of Windows (especially in the FAT16/32 days) were very prone to hard disk fragmentation. In the days before DMA (direct memory access) disk throughoutput was even more meaningful than today and load times for any task involving the hard disk would increase significantly simply due to file parts being scattered across a disk. These days both Windows and the hardware compensate for this effect, so it's no longer a major source source of slow down -- except maybe in extreme cases.

2) Free-for-all system start-up items. A lot of programs want to compensate their heavy footprint by preloading parts of themselves into RAM when the system is started. If a user installs many of these types of programs he will a) notice the system tray clutter up and b) witness start-up time go through the roof. This applies to Linux too, it's just that most Linux daemons are better behaved and start in succession rather than at the same time (which will kill any system that relies on hard disk).

3) Everyone's favorite pick: the registry. The registry _appears as a single file to the user_ (but is actually two files) and hosts settings for Windows but also practically all the programs installed on the computer. It dates back to the innocent DOS days when a _central_ settings file was a reasonable trade-off (instead of reading many settings files scattered around the disk, you'd only have to read one _or two_). However the design quickly became obsolete as the complexity of Windows grew and the amount of applications people use skyrocketed. Not only is the registry completely undecipherable to a normal person, it is practically unmaintainable. If you install a program once, you can rest assured it will leave something in the registry after you uninstall it. This means the registry just keeps getting bigger and bigger despite the spring cleaning users sometimes like to do by removing unused applications. Every time an application is run, the system has to parse through that immense pile of completely unrelated settings (as opposed to a single small text file containing only settings for the program that is being run). The increased speed of computers has kept the whole thing from falling on itself, but as any Windows user will know, there is no fix for the registry problem other than regularly timed fresh installs or image backups.

In other words, Windows has simply outgrown it's original intended purpose. The design decisions were made with no sight of the future and are now impossible to remove without breaking compatibility. It's kind of like if you wanted to make an OS for an early 90's cellphone and later decided that the same OS should be used in today's supercomputers.
