### Post by Colonel Kilkenny on 2008-11-24
> **SomeGuyDude said:**
> 
3) Opera renders a lot of fonts horrendously

Opera uses Qt to font rendering so it's your Qt settings which are messed up.
> **SomeGuyDude said:**
> 
4) I like Extensions, and Opera's widgets is such a terrible equivalent it's like asking for a back massage and getting a shovel to the crotch.

Widgets aren't there to do the same stuff as extensions. Opera is cross-platform product (desktops, mobile) and widgets may be handy eg. in mobile phone. Opera was the first to bring widgets to browser, now Apple (iPhone), Firefox (Mozilla Prism), Google (Google Gears), Adobe (Air), etc. are doing basically the same thing. Just couple years behind Opera.

> **mips said:**
> I don't like the way Opera puts the tab bar at the top, I know this can easily be changed but it is a pita.

Offtopic, Linux GUI-rant:

Actually, IMHO, Opera does it the right way. One tab should contain all the controls that are used to control that tab (address, buttons, etc.) and the page itself. Not the other way around. 
It doesn't make any sense that when you change tab something above the tab changes. The "Firefox-way" also means that there is only one addressbar in whole application which is then being modified by child elements (tabs). You have multiple pages open from different addresses but those pages and their controls share the same addressbar, same searchbar, etc. That means that you can't change tab2 to use temporarily Yahoo search because search engine isn't tied to tab even though address next to it is.
IMHO, Firefox GUI is full of stupid things and that is partly because they choosed to use tabs in the wrong way. And Nautilus decided to do that as well. Tabs should have address, address should not have multiple tabs which then change the address.
That approach works fine with text editors etc. but those applications do not have tab-specific controls (or address or zoom level or...)!

> **mips said:**
> 
I still have not figured out how to make the tabs behave like those in FF where it always opens the next one to the right when you close the present tab. Probably some simple setting?

Like this?
Ctrl+f12 -> advanced -> tabs -> When closing a tab: Activate the next tab.
