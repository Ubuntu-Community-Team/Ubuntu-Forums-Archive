### Post by doorknob60 on 2011-03-30
Of those three, I'd go with Fedora, but you don't have to limit yourself. If you're up for a little challenge (not much harder though), then try Debian. It's very similar to Ubuntu (same package management system), but it's more barebones and you kinda set it up yourself (though you can have it install a DE like Gnome by default). When set up correctly, you can have all the functionality you need, but while using significantly less resources and have a faster system.

Arch is sorta like Debian, but to the next level, I guess you could say :P It doesn't use apt, but pacman is much faster than apt in my experience, and overall I like it more than apt, so if you feel comfortable, then try out Arch. As long as you can read the Wiki pages you'd be fine. When you install it, you jet get a root terminal prompt, and you use pacman to install everything you need. It's a little daunting at first, but the Beginners' Guide outlines it amazingly well. If you're worried, but still curious, try it out first in a virtual machine.
