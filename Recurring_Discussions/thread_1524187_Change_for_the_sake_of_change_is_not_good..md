### Post by Xianath on 2010-07-05
Six months is a lot. On my previous job we ran two-week iterations and released every months (to paying customers, no less). I've heard the "there's not enough time" argument countless times but the truth is, there is exactly HALF the time in two weeks as there is in a month, so it's enough to do HALF as much work. The benefit of twice faster feedback is that it helps catch problems earlier before they've had a chance to bury deep, and thus improves quality.

If it were up to me, I'd cut the release cycle in half or even have it bi-monthly (as the kernel team does), but it's not up to me :) and it obviously works well for Canonical.

Change for the sake of change is inevitable in open source. FOSS people code for the fun of it, and there's less fun is one is told what to do. Ergo, FOSS lacks, and at the same time could not survive, a well-defined and strictly enforced direction.

A thought provoking post. I agree with some of what you say. :)

Please understand that Ubuntu is a fast-moving distribution, meant to showcase the latest innovations in Linux development. It is also the #1 desktop Linux distribution by many estimations. These two distinctions go hand-in-hand, IMHO.

Ubuntu does not develop their own applications and code. Gnome comes from the Gnome Foundation, OpenOffice comes from Sun, Firefox comes from Mozilla, etc. And the majority of this code passes through Debian before it becomes a part of Ubuntu. 

Debian has been around for a long time. It's a super-stable distribution that releases roughly every 2 years. Debian is popular for servers and for users strongly committed to the Free Software belief system. But it is much less popular than Ubuntu for the casual desktop user crowd. You don't see a lot of "I'm a noob ditching Windoze for Debian" testimonials--it just isn't as "exciting" as a distro whose name means "I love everybody" and that has a new fuzzy animal mascot every 6 months.

So basically what you are proposing is that: 1) Ubuntu becomes more like Debian by having more stable releases less frequently; 2) Ubuntu sticks closer to the "upstream" code (for example by not moving the buttons to the left side, or not developing "user friendly" new features like the Software Center).

Which raises many questions: If Ubuntu stops releasing every 6 months, and stops developing unique features like buttons-on-the-left and the Software Center, what do they put in their press releases? What distinguishes Ubuntu from Debian and other stable-because-we-stay-close-to-upstream distributions? How does Ubuntu stay Number One in this scenario?

I had the same experience with Microsoft Windows, so I switched to Gentoo Linux. I suggest that you try it. It is a source-based rolling distribution that lets you decide what is installed and how it is configured, so people using it have a fair amount of independence in that dramatic changes such as the 9.10 to 10.04 upgrade really do not happen unless you want them to happen. With Gentoo Linux, it is likely possible to keep using the same version of GNOME for years, while upgrading everything else. Just to give you an idea of how long things can stay available on Gentoo, I believe that KDE 3.5.10 is still available as long as you use the kde-sunset overlay.

Currently, if you want an older version of GNOME, version 2.26.3 is available on Gentoo Linux and to prevent it from being taken away from you, you can move it into an overlay such that you could continue using it indefinitely as long as the system interfaces on which it depends do not change. If they do change, you will need to either start specifying versions of dependencies or patch the dependencies to keep things working. Upgrading works too. Gentoo's package maintainers really do not do much customization with desktop environments, providing them basically with the same sources that upstream releases, so Gentoo Linux users do not encounter any distribution specific stuff in their desktop environments unless they want to encounter it. I think that the things you mentioned are Ubuntu-specific changes to GNOME, so if you were to use a distribution that left GNOME unchanged, you would not have to deal with such things.
