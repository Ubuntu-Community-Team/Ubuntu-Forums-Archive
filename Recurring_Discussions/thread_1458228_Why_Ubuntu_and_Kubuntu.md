### Post by MCVenom on 2010-04-19
I think it's because each version comes with specific software tailored to what's standard in the desktop environment; you couldn't fit all that on a normal cd. For example:

Kubuntu comes with KDE & Qt based applications (the only non-Qt one I know of being Firefox, which is only 'included' as a quick-install option.), not even the Ubuntu Software Center is in there (or at least it isn't present in Karmic, might be in Lucid.)

Xubuntu comes with lighterweight, but still well featured GTK+ applications, such as Thunar for the default file manager. Normal Gnome apps aren't used b/c the Xfce environment tries to cut down some of the dependencies and bloat Gnome has.

And finally (coming soon!) there's the up-and-coming Lubuntu with LXDE, which features has its own ultra-light, even-lighter-than-Xfce GTK+ apps.

Do you see the common thread here? Each of the variants concentrates on staying true to its desktop environment, and there's no way you could package all that software onto one CD :p

(And before we hear "We should really switch to DVDs..." remember, there are still some places (or at least enough old computers), which may not have a DVD drive, or be able to handle one gracefully :p)

P.S.: I'd really like to throw this out there; there's an idea to get rid of the 'Kubuntu, Xubuntu, etc.' names and focus on a more unified branding scheme: (i.e.: Kubuntu becomes Ubuntu KDE Desktop) [https://wiki.ubuntu.com/UnifiedUbuntuBranding](https://wiki.ubuntu.com/UnifiedUbuntuBranding), I think this would be a very good idea.

Well, it could be possible with a Arch-style netinstall option, but my point is that the way the variants package different software as compared to each other and plain old Ubuntu, good luck getting just Ubuntu and Kubuntu to fit on one CD.

Plus I also think that it has to do with the fact that Canonical focuses on marketing Ubuntu w/ Gnome; it goes more with their goal of offering an easy to use Linux distribution. KDE can be a bit too complex for some, and it may not be the most comfortable environment for many new users. And to be perfectly honest, Canonical **didn't** have to accept Kubuntu as an official variant and support it -- Kubuntu was started as a project independent of official Canonical support.
