### Post by nkskjames on 2008-05-07
Before you take this thread into the weeds, let me define mainstream.  Mainstream = > %30 market share on home desktops.

Also please don't turn this thread into a Windows vs. Linux battle or a Linux distribution debate.  

So the question stands:  Will Linux ever be a mainstream O/S?  Let me list what I think needs to be addressed with Linux in order to answer this question as "Yes".  

1. **Problem:** Software developement companies need to be able to make money off of applications written for Linux.  
**Solution:** The package manager (in Ubuntu) is great in my opinion.  It is a centralized repository for applications and makes installation and removal almost idiot proof.  So now what we need is a way for the repository to also manage paid-for applications.  Have a way for developers to charge for apps in the repository.  Now of course many apps will be free, but if one day Photoshop is written natively for Linux or if Id Software wants to charge directly for their native version of Quake4 (today they make you buy the Windows version of Quake4 to play the full Linux version), then they can use the idiot-proof nature of the package installer instead of having non-Linux savy users go through a non-standard install.  This can make Linux mainstream faster than relying on box stores to carry boxed software versions.

2. **Problem:**  Non-standard user policies/software settings database/API
**Solution (or lack there of):**  This is a tough one.  The "registry" solves this problem for Windows, although I'm sure some people hate it and may say it's more a problem than a solution.  Nevertheless, I sometimes spend hours trying to figure out where an application is storing my preferences in the Linux filesystem.   That is not exactly the issue I'm addressing because if you think of mainstream users, they don't care where the settings are.  But mainstream developers need this because if Linux becomes mainstream, users could have hundreds of apps installed and .directories/files would become a mess and there could be collisions.  Need a standard.  You could also lump in here a "My Documents" standard as well.  I think this concept works well in Windows.

3. **Problem:** Too many choices
**Solution:** I know everyone has their preference for window managers/distributions, but all the choices confuse the mainstream user and turn them off.  

4. **Problem:**  Install programs
**Solution:**  As mentioned earlier, the package managers are pretty good, so all software should be installable via this method.

5. **Problem:**  Not enough mainstream native apps
**Solution:** Related to #1.  I know WINE is out there, but I think that adds too much complexity and still keeps focus on Windows. OpenOffice is a good start, but we need a few more major apps ported to Linux: Photoshop, AutoCad, etc.  This will start the revolution.  Once business start using Linux as mainstream, it will migrate to home.  Games are similar discussion.  

6. **Problem:**  Driver support
**Solution:**  I don't think this is as big of a problem as some may believe.  The main issue is probably with video cards.  This problem naturally will resolve itself as Linux becomes more mainstream.  

**The positive:**
1. Window managers are very good and have look/feel of Windows so mainstream users are comfortable with that.

2. HAL - It seems that hardware abstraction layer is very good at dealing with a wide variety of hardware and plug-n-play works well.  Yes, I know video cards can be painful, but that is true on Windows too.

3. MythTv - I like MythTv and it seems to compete well with Media Center.  Home theater PC's are the future and Linux seems to do well in that arena.

4. Major computer companies do offer Linux - Good press.

5. Distributions - the various distributions have made the install process very doable for novice computer users.

6. Firefox - we need the internet

7. It's Free...
