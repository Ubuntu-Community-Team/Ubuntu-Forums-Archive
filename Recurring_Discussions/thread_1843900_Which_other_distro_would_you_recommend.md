### Post by ninjaaron on 2011-09-14
DO NOT try a rolling release distro!

They ruin you for anything else because you always feel as if you're behind the curve for up-to-dateness.  That's the main difference between distros, at the end of the day, release schedule.  You can use basically use all Linux software and environment stuff on any distro.  It's sort of a question of stability v.s. freshness.  Ubuntu is somewhere is in the middle on that scale, but it probably offers the most polished environment out-of-box than any other distro.  Unity had some rough spots (especially at release), but still, Ubuntu does better at integrating application and web services into the OS

But if your inclined to build your own environment, and don't mind a little pain in the process, you could try some of the more minimalist install distros like Arch.  Ubuntu actually also has a minimal installation image so you can build your own system, if you like.

### Post by sanderd17 on 2011-09-19
My favourite is Arch, but I would almost never recommend it to someone.

Arch is based on the KISS principle: Keep It Simple Stupid. That means when you install Arch, you only have a package manager (pacman) and the basic command line tools (although you can install Arch without those).

From that point, you build your own Arch setup. Because only few people are able to build their own system, I never recommend it. It took me a few years in the Linux world before I was able to do it.

Arch is rolling, so if you have a solid setup, you don't need to upgrade. And you always have the most recent vanilla versions of all packages.
