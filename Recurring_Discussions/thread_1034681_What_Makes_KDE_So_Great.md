### Post by Hooya on 2009-01-10
With all this talk about KDE3.5 v 4, I wonder if there is a way to have both installed, and let you choose which one you go into when you start a new X session.

I can start a KDE and Gnome sessions in addition to my usual fluxbox session by opening the tty1 or tty2 terminals and typing:
startx /usr/bin/startkde -- :1
startx -- :2

Can I have 3.5 and 4 installed simultaneously?  I'd like to try 4 out, but I want to be able to just apt-get uninstall to get rid of it if I find I don't need it any more.

Edit:
Actually, way easier than I thought.
$sudo aptitude install kde4-core
Then from a new session
$startx /usr/lib/kde4/bin/startkde --:X
where X is your display number.

Doesn't affect your KDE3 installation at all.  I have both running simultaneously right now.

### Post by Montblanc_Kupo on 2009-01-12
I decided to make the jump to lightspeed...er... linux this year fully after hedging around it for a long while. I've used windows and mac both since ... well forever. heh. I gotta say... having the "desktop" be a window is rediculous though. The concept of a desktop is that it's well... the desktop... it's not part of the file structure in it's interativity. Having it function as a widget folder was just plain stupid IMHO... we ditched that functionality in windows 3.1 if anyone remembers... and good riddence. 

The famous "configurability"... honestly... I had a hell of a time making the couple of simple tweaks that I wanted to make. Things like I wanted to take that stupid widget swirl in the upper right and move or hide it... never could do that. Moving the main panel to different locations on the screen proved more difficult than it should have been. These were all things that gnome, os x, and windows all let me do pretty quickly within their own methods (gnome is particularly easy with it's panels).

Little things like having the bouncing icon on the mouse to tell you what is loading are cute and I can see where they are nice... but I eventually got sick of it and wanted to turn it off... couldn't find that either.

A complaint I've had about all the linux distros I've used to far is that I would like a centralized hardware device manager tool. KDE is no different about this... couldn't find anything. I don't know if that's the DE or just linux though.

Over all I've installed it a couple times and run it off the livecd a few times... I really want to like it because I agree that it's "forward thinking" and could be great... but in practice it just feels... broken. I'm all for innovation, but I think sometimes in linux development they think "well... we want to not be mac or windows... so we're going to do it different"... which is nice... but sometimes the conventions that are in mac and windows are there because they work. It's a fine line. Linux does give us a lot of choices of how to tweak things (once you figure them out)... certainly more than any other *modern* OS seems to.

Diehard hacker that I am though, I'm planning on reinstalling KUbuntu on some unpartitioned space I have left to play with it some more.

$0.02

Regardless of what any tech documents or blogs or articles or crystal balls might say... it's a *released product*. It isn't released with any caveats about being in beta or being in testing or anything like that in places like kubuntu. If you go to download kubuntu for example it asks if you want to download the "cutting edge" version 4. That's it. For most people... if it's listed as the **main** download... that means it's a release version. I tend to agree with that. So it should be taken as such... no grain of salt necessary. It's released... and intended for mass use... and should be judged by those standards. Would you install windows on a machine and forgive it's bugs and crashes because it's "unfinished"?... even though that is exactly what microsoft does. 

> I also dont think that "I removed it within 20 minutes" constitutes as a good informed opinion and coming out with such comments.

That's a two sided argument. 

1. It's not a fair test of the system... giving it long enough to learn about it.. get familiar with it....etc. etc. etc.

2. If it takes more than 20 minutes to get used to an OS... the OS isn't designed intuitively enough. Especially for linux users who, because of things like this, tend to not be the masses but are still the more computer savvy people. I have been using computers for 30 years... as a user... as a technician for large corporations... as an administrator... as a developer... and as an artist. I installed Kubuntu and within about the same time frame nuked it because I couldn't get it to do half of what I wanted. I am going to mess with it again, but I have to say, it was pretty quick that I realized that it did a lot of things very strangely and took a lot of getting used to to even make it work.

If you're looking to "thouroughly test" something... say you're writing a technical review... then of course you need to spend time learning all the features whether they work well or not. If you're looking to use something for personal use... it's usually pretty obvious if it's comfortable or not and you would have to have a compelling reason to use it if it weren't immediately comfortable. Do you buy shoes that don't fit or pants that are too small assuming that eventually you'll get used to them? Would you buy a snowmobile as your main vehicle in hawaii just because it had a good feature set in some conditions? If you were allergic to peanuts, would you eat peanutbutter cups just because everyone else said they were really good? Hyperbole, granted, but the point being that there are different reasons and needs for using something. The ideal for a *user interface* should be that it is user friendly and does what the user needs without getting in the way or having to be learned. The closer that they get to that ideal, the less time will be needed to judge whether the UI is something useful to you *personally* or not.

> If you're looking to "thouroughly test" something... say you're writing a technical review... then of course you need to spend time learning all the features whether they work well or not. If you're looking to use something for personal use... it's usually pretty obvious if it's comfortable or not and you would have to have a compelling reason to use it if it weren't immediately comfortable. Do you buy shoes that don't fit or pants that are too small assuming that eventually you'll get used to them? Would you buy a snowmobile as your main vehicle in hawaii just because it had a good feature set in some conditions? If you were allergic to peanuts, would you eat peanutbutter cups just because everyone else said they were really good? Hyperbole, granted, but the point being that there are different reasons and needs for using something. The ideal for a *user interface* should be that it is user friendly and does what the user needs without getting in the way or having to be learned. The closer that they get to that ideal, the less time will be needed to judge whether the UI is something useful to you *personally* or not.That's exactly what I think about KDE4 - or software in general.

There has been a lot of talking about the early release of KDE4, and the main reason - AFAIK - was to give software developers a way to port their software on KDE4. The second reason was: give users the chance to thoroughly testing it and reporting bugs. Both could have been done without making a stable release. They could have released the API for developers, and a lot of users surely would have installed a beta. So I think they've done wrong. 

By now, many "userfriendly" distros completely switched over to KDE4 without giving users a choice. Suse 11.1 is the only one I know so far (but there may be others). By now, one has to chose between up-to-date distros (latest kernel etc.) or older distros, that still support KDE3 environment. That's sad but true!

So I installed KDE 4.2 rc1 to give it a try. Needs work. They're moving along, but this should NEVER have been released (KDE 4.x) as a stable version that could be adopted by distros... it's still very very beta and should be treated as such.

So I found a way to install KDE 3.5.10 into kubuntu 8.10. I can see where people might like using this. It's polished, works well, and didn't really have any stability issues. I still don't care for some of the KDE conventions though, but that's personal taste. What I did see though was that KDE is slower than Gnome for Me. Noticeably slower. The windows manager is slower to redraw things, making it feel clunky. Compiz had a hell of a time getting installed and integrated, so I couldn't even really use that. Good luck getting Emerald to even run (not compatible with 3.5 AFAIK and iffy / crashy on 4.x). Processes even through konsole seemed to take longer. Installing applications and doing updates took much longer and had more problems.

Overall... not impressed. I haven't seen any of the "more power" and "more custimizability" that I keep hearing about, in fact, quite the opposite. I would love for someone to prove Me wrong... I really want to like KDE... I can see there is potential under the surface, certainly in the development model... but it **just doesn't work**. :(

On a tangent... after I got frustrated with KDE, I started installing every desktop environment I could get My hands on to see what other options there were. XFCE really doesn't seem any different from Gnome from what I can tell... it's like Gnome lite heh. Fluxbox... made Me twitchy. Enlightenment was... interesting... I can see it being cool on like... a palm pilot or iPod touch where you could interact with a stylus. I tried to get Sun's Looking Glass installed but it kept giving Me and "error code 1" every time I tried. I've run into this before when there wasn't enough space on the boot volume... but I had 18GB left... so I don't think that was it. heh. I finally gave up.

Anyway... done rambling for now. $0.02 ymmv.
