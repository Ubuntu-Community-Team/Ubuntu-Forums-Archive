### Post by cartman640 on 2009-11-11
Real world work is where Windows 7 shines above XP. Boot up times might be slower in some cases, but they're not really important, it's what you can do in a given amount of time once the computer is running that matters. Anyway, boot times can be ignored if you don't turn your computer off (or just put it to sleep), my Win7 workstation has been running for the last 55 days without any problems.

Comparing RAM usage isn't really a fair test either, yes XP uses considerably less RAM, but RAM is there to be used. It's so cheap these days it's not unreasonable to expect 4Gb in a system. I don't really understand why people get so worried when an OS actually uses that memory either, why else do we have it? Currently I'm sitting on 93Mb out of 8Gb RAM free in Windows 7. This is where you really see the speed difference, many of my apps are cached in RAM, Firefox opens in less than a second, same with office, Foobar, media players, etc. If that memory is needed, the caches are just cleared out, no problem.

Throw some resource hungry applications into the mix, maybe half a dozen vm's, or some video encoding and XP will soon grind to a halt, where as Windows 7 will keep running at a good speed. Much of this can also be attributed to 7's far better support for multiple cores than XP. XP was released into a single core world, where only servers and very high end systems had two cpus, a quad is a complete waste on XP.

Oh, and I used to run Vista x64, this was also much faster than XP (for many of the same reasons 7 is), not as snappy as 7, but still an improvement.

So yes, on current generation systems, Windows 7 is faster than Windows XP.
