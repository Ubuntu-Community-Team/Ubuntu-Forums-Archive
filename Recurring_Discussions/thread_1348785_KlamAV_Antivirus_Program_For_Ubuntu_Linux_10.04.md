### Post by t0p on 2009-12-07
Hmm.  This video is not needed, for ***two*** reasons:

[LIST=1]
[*]There ain't no* viruses out there that will affect Linux
[*]KlamAV is pretty self-explanatory
[/LIST]
And it's pretty lucky that KlamAV is so easy to figure out, cos this video doesn't explain how to use it at all.  All you did really was read aloud a bunch of text about the app, then go along telling us what various menus are called - which we could read for ourselves anyway.  As an instructional video, this scores very poorly.

KlamAV is about as much use as a chocolate teapot anyway.  Time and again I've seen posts in these forums where users are saying that KlamAV has found such-and-such a virus... and it turns out it's just a system file that the program couldn't identify.  You'd think an AV program for Linux would be able to recognise a Linux system file, right?  All it can find that's of any use at all is Windows viruses.  As you say, it might be nice to find them so you don't pass them on to Windows-using friends.  But Windows-using friends should take responsibility for their own systems, shouldn't they?  They choose to run an OS vulnerable to viruses, so they should scan all files themselves.  It isn't my responsibility to keep their machines malware-free.

As for your comment: "there are a few Linux viruses out there" - could you elaborate?  I know there are some proof-of-concept things kicking about in virus labs, but are there any Linux viruses in the wild? No*.

[* I'm not up on the latest virus news, cos I don't need to be.]
