### Post by aysiu on 2009-05-21
If you're talking about servers, I think the Linux kernel is just better designed than the Windows one (NT, is it?).

But if you're talking about desktops/laptops, I haven't seen any evidence of statistical significance in terms of stability differences between Windows, Mac OS X, and various Linux distributions. I've seen X crash. I've had Ubuntu freeze up on me. My wife has had Mac OS X crash on her. My Windows computer at work occasionally has glitches. For the most part, though, they're all stable.

Maybe the Windows installations you see crashing are crashing because of malware? My Windows computer at work has a limited user account and Firefox with NoScript, so it's highly unlikely it'll get infected. It runs quite well.

### Post by monsterstack on 2009-05-21
I find myself freezing Windows on the odd occasions I have to use it nowadays. It's the waiting-for-stuff-to-load that always gets me. I have never encountered a kernel panic. But I remember back when I used Windows regularly I rarely encountered such problems. It's just a matter of knowing what you're doing. Linux is the same: I have had X fail to start properly, but only because of my poor amendments to the xorg.conf file.

One of the reasons Linux *can* be a little bit more stable is because you don't have to nuke everything to stop a misbehaving process. It doesn't matter if it's X, mpd, apache2 or even some badly-written python script you just whipped up. It's just a matter of stopping it and restarting it. On Windows, there are a few things that are intimately tied up to the OS in such a way that to restart them would require a reboot.

### Post by BuffaloX on 2009-05-21
It's actually quite funny IMO.
Since Linux is based on a monolithic kernel, and NT is based on a micro kernel. ( based on = not pure )

Micro kernels in theory are more solid, because drivers work outside the kernel, and cannot easily crash the kernel, but they are also slower.
Monolithic kernels are in theory less solid, because drivers can easily crash the kernel, but they are faster.

Windows NT 3.5 and 4 crashed in bundles, unless you were very careful with both your choice of hardware and drivers. Windows 2000 and XP got better, but nowhere near the level of Linux for stability. Vista and 7 are supposedly better than XP, but I'm off that wagon now, so I lack first hand experience.

Many factors are crucial, and many design decisions are trade-offs between speed or stability, and of-course code quality is important too.

I think Microsoft has had problems with some bad decision making, causing Windows to be crash prone and insecure, they are now trying to fix those problems, while keeping the API intact, except for some strategic breakage.
