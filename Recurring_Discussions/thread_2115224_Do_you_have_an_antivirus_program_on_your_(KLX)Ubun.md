If your goal is to obtain a higher level of security on a linux box then there are many other, far more effective tools to use. Namely apparmor, firewall, nids, hids, logwatch...

### Post by lykwydchykyn on 2013-02-12
> **Ms. Daisy said:**
> 
If your goal is to obtain a higher level of security on a linux box then there are many other, far more effective tools to use. Namely apparmor, firewall, nids, hids, logwatch...

I'll second this.  Every time this discussion comes up, people assume that not running an Anti-Virus means not having any kind of security software.  

You have to protect against the threats that are relevant to your OS.  For Linux, AV is not relevant and I don't bother with it.

But I do make sure other security measures are in place relative to how I use the machine.  For instance, I'd never have a public SSH box without something like Fail2Ban, AppArmor, and probably PK-only authentication.

An antivirus does nothing on Linux but waste resources and give you a false sense of security.

### Post by Stonecold1995 on 2013-02-13
Yes, I have ClamAV.  I almost never use it though because in general Linux is far more secure than Windows, and its users often have better sense than to run "hawt_pr0n.avi.exe". ;)  The only times I use it are when I'm about to run a Windows program through Wine (yes, viruses will try to run in Wine, although they have limited damage potential because they are run with Wine's permissions, and because any low-level exploit or infection technique is likely to fail in a non-perfect environment like Wine), or when I'm scanning a mounted VirtualBox image (of Windows).  Every once in a while I'll run "clamscan -r ~/.wine" just to make sure Wine isn't infected, but it never has been.
