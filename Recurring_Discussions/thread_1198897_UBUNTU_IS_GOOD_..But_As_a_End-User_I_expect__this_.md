### Post by 3rdalbum on 2009-06-28
> **rushikesh988 said:**
> 2.
Wine-doors should be packed to it.So that its reputation will be the OS which run Linux as well windows programs too.

You're not familiar with OS/2, I'm guessing. That had a reputation of being an operating system that ran Windows programs. It died through lack of users - I guess people couldn't work out why they should run Windows programs on OS/2 rather than run Windows programs on Windows.

Mark Shuttleworth is very much against including Wine, and this is one of the reasons he gives. Another good reason is that it encourages people to use proprietary software rather than open-source software. It also encourages people to use non-native software that won't integrate well into the system (and may feel buggy in Wine). And, Wine does potentially pose a legal threat (patents) from Microsoft.

When Bill Atkinson wrote MacPaint in the early 1980s, he implemented a function that allowed you to put text into your pictures, and once the text was rasterised into the image you could re-edit it again, as long as all the pixels were still in their correct positions (if one pixel was out-of-place the feature wouldn't work). He removed the feature before release, because he was afraid that people would overuse the text editing feature, and that people would regard MacPaint as a useless buggy word processor rather than an excellent image editor.

Well, if you include Wine in a default Ubuntu install, people may regard Ubuntu as a useless buggy Windows clone rather than an excellent Linux distribution.


> 4.I think all linux distros should use same installer so that software manufacturers will make softwares for linux easily..(not rpm ,not deb only one common just like .bin files )

Great idea. Now, which package format? Deb, RPM, or something else? How do you decide which package format wins and which lose, and why would the distributions who use the losing package formats change? How do you solve the problems of dependencies having different names on different distributions?

Oh, it's easy to say that they should all switch to one package format... but you might as well say that all people should speak one single language.

The variety of package formats doesn't pose a problem for a software developer anyway. It's trivial to create a Debian package. I'm sure it's trivial to create an RPM. Takes a couple of minutes, excluding compilation time. It's also trivial to make a binary installer, or you can use a prebuilt one.

Having a couple of packaging formats is not really a problem. Got an RPM? Convert it using Alien.

Just re-reading your post - are you saying that everything should be binary installers? That would be a step backwards my friend. Most Linux users dislike binary installers, myself included. I'd rather have the niceties and efficiency of package management rather than the nightmare of binary installers for everything.

### Post by aysiu on 2009-06-28
The lack of proprietary codecs in Ubuntu by default is **primarily** ideological (only *secondarily* legal):
[http://www.ubuntu.com/community/ubuntustory/philosophy](http://www.ubuntu.com/community/ubuntustory/philosophy)

[Mark Shuttleworth on including Wine by default](http://www.eweek.com/c/a/Linux-and-Open-Source/Ubuntu-Founder-No-Emulation-Software-for-Dell-Systems/): > I do not want to position Ubuntu and Linux as a cheap alternative to Windows. While Linux is an alternative to Windows, it is not cheap Windows. Linux has its own strengths, and users should want it because of those strengths and not because its a cheap copy of Windows. I happen to agree with him. If you use Windows-only programs, you should use Windows. If you're really open to exploring alternatives, then actually explore those alternatives before resorting to a not-fully-functional compatibility layer. Does Mac OS X include Wine by default? No, because they expect you to run Mac software. Likewise, Ubuntu expects you to run Linux software.

I don't really see why Ubuntu should come as a DVD with a lot of packages. My internet connection is fairly fast, but not so fast that a 4 GB download takes seconds. I'd rather download a 700 MB .iso. Also a lot of people do not have DVD burners.

If people have fast internet connections, they can install packages off the internet instead of off the installation media. I do think people should vote up these Brainstorm ideas, though:
[Idea #7792: Use BitTorrent as primary protocol for apt-get](http://brainstorm.ubuntu.com/idea/7792/)
[Idea #142: Optional add-on CDs (not advertised heavily, but still available)](http://brainstorm.ubuntu.com/idea/142/)

#7792 is for making *apt-get* faster, basically, especially on the few days after a new release. Why slam the servers, when we can all share the downloads with each other?

#142 is for those who do not have working or fast internet connections on their Ubuntu computers. They can then download the add-on CDs on another computer or order them to be mailed. Right now, Ubuntu without an internet connection isn't very handy.

As for package management, it's quite easy for some random person on the forums to demand that all package management be unified, but that's not how the world works. I want world peace, but do you really think the governments of the world will just give up their guns and nuclear weapons all of a sudden? Luckily, no one gets killed over package management.

The only way the package management unification will happen is if one distro gets really big with the general public (could be Ubuntu, since Red Hat has basically given up on "Linux on the desktop"). If that happens, by default all third-party software will come in the form of .deb files, and .deb will become the standard (just as .doc is the standard for word processing files, even though it isn't the agreed-upon open standard--just the *de facto* standard).

Bottom line, though: Ubuntu is one distro with certain goals and a certain vision. If you don't like it, you can use something else or even make your own Ubuntu remix (using [Remastersys](http://www.psychocats.net/ubuntu/remastersys), it's really easy--you do not have to be a programmer).
