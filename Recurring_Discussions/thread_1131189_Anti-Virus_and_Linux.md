### Post by aysiu on 2009-04-20
Any files I share with Windows users are files I've created, so I know they don't have viruses on them, so I will not scan them. And if those Windows users aren't protected, they'll get infected sooner than later, whether I send them an infected file or not.

Based on what I've seen and based on simple logic, antivirus is pretty much useless protection anyway. If a new threat appears, antivirus can't possibly know about it, and if it does, then it also detects so many false positives that users don't know what's malware and what's safe.

Conficker took advantage of a Windows flaw that Microsoft had already patched, but many Windows users are too lazy to install Windows updates regularly.

The latest Mac botnet was formed out of trojans in pirated versions of iWork and Photoshop, so those people pirating the software knew they were installing untrustworthy software and did it anyway.

Malware these days generally comes from this mindset "I have antivirus software installed, so I'll just do whatever I want, since I'm protected."

Avoidance of malware generally comes from this mindset "I'm not invincible, but I will do every practical thing I can to educate myself about safe practices and lock down my system as much as possible."

If you are true paranoid, turn off all extraneous networking services, use strong passwords, use Firefox with NoScript, install OS updates regularly, install software from only trusted sources, and don't randomly click on attachments. Be smart.

If you're smart, antivirus will do nothing additional to protect you. If you're dumb, antivirus will do nothing to make you smarter.

Here's my take on this interesting topic:

1. Just because there isn't much in the way of Linux malware now doesn't mean it's not a threat.

2. There is nothing to my knowledge that makes Linux immune to malware. Sure, the typical user doesn't run as root, so the damage might be contained.
