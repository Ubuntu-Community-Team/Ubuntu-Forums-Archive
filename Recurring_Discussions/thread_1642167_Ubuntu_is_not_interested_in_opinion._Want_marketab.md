> Something is broken and when I looked to xorg.conf and other configs they had changed or no longer existed. My "standard" nvidia driver apparently was loaded automatically with no NVidia interface and this was the root of multiple other problems

..indicates that you most probably have a graphics/hardware problem and have been trying to troubleshoot it from within the xorg.conf file.

 I would strongly recommend you start a thread in the 'general Help' section detailing your problems as you have above.

Ubuntu no longer uses a static xorg.conf file but generates it dynamically at boot. This allows changes in hardware to continue to boot without the user having to drop to a command line and reconfigure X.

95% of the time this works just fine and for the other 5% where hardware or drivers are problematic  there is a method where one can generate an xorg.conf and edit it to suit.

Here are a couple of links that detail the process and if you are used to editing the xorg.conf file then this may be all you need to get you up and running.
