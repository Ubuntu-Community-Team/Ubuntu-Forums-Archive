### Post by Lucradia on 2012-01-19
> **Sunships said:**
> Don't these have the same problem, except that things may break at any time, rather than when you choose to upgrade?

Rolling releases, however, push out bug fixes faster, since when the program is updated successfully in the distro'd repos, it immediately is set for upgrade, no matter what. Same with all drivers, everything. Technically, this is how software updates works, but you may have noticed that the programs have a suffix tied to the specific version of ubuntu they're tailored for.

ubuntu can be configured to be "Rolling Release" as well. However, even if you search google with the below search terms, you will not get a tutorial even on the first two pages of search:

```
ubuntu rolling release -proposal -becoming -rumor -"to become" -moving
```

I have seen a topic here on the ubuntuForums detailing how to change your ubuntu version to testing, so that it's always updated to the next version when the next release comes.
