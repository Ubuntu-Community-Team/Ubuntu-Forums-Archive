### Post by Zill on 2010-11-14
It is often useful to read the "Help" (hit F1 while in OOo)...
> **Saving as a Microsoft Office File**
1.Choose File - Save As.
2.In the File type box, select a Microsoft Office file format.

**Saving Documents by Default in Microsoft Office Formats**
1.Choose Tools - Options - Load/Save - General.
2.In the Standard file format area, first select a document type, then select the file type for saving.
From now on, if you save a document, the File type will be set according to your choice. Of course, you still can select another file type in the file save dialogue box.

### Post by Hagar Delest on 2010-11-14
Please remember that until recently, the .doc specification was not available. The import/export filters have been reverse engineered. Hence some flaws sometimes.

Basically, only work in native ODF and export only when needed.

Note also that the Ubuntu version is not the vanilla one, it's a version based on the Novell go-oo build. That one should however have a better .doc export filter.

You can try the official version: [[Ubuntu] Installing OOo on Debian and Co.](http://user.services.openoffice.org/en/forum/viewtopic.php?f=16&t=68)

Have you tried your backup folder in the OOo profile (see [[Tutorial] The OOo user profile](http://user.services.openoffice.org/en/forum/viewtopic.php?f=74&t=12426))?

### Post by arvevans on 2010-11-14
Since I don't have any MS OS available to me, I do all my word processing in Open Office.  I have made many large documents using Open Office, and have successfully shared them with my colleagues who are still using MS-Windows.  In my case, I do all the work and saving using <filename>.odt as a routine work method.  Then when done I save the file once again using <filename>.doc or <filemane>.docx and email this Microsoft compatible file to those who need it.  I have also saved the <filename>.doc version on a pen drive and found that MS users can always read it.

Open Office by default does periodic automatic saves (about every 5 minutes) using it's <filename>.odt format.  So, even if you lost the .doc copy, you should still have the file saved in .odt format somewhere on your hard drive.  You can recover the information from that file and then re-save it as <filename>.doc if needed.
