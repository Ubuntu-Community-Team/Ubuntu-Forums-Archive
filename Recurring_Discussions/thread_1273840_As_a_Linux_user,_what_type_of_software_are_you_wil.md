### Post by moe_syzlak on 2009-10-01
> **JDShu said:**
> On the other hand, when software has a price tag, its probably proprietary. Its not commercially viable to try to sell software thats under a GPL license.

Anyway, I'd pay for games as well, since there is no good model yet to profit from open source games.

Yes absolutely. There needs to be a compromise with the GPL license.

The LGPL allows developers to re-distribute and ship formerly GPL-ed code with their code under their license if they:
1) only link via object code with a GPL library and not just deriving the work [program] and putting their license on it
(2) They must also distribute the GPL library that they used WITH a copy of the GPL, but hey don't have to release the code that made their object libraries.

The LGPL is a de-facto sign that the GPL is nearing its bounds. Nonetheless, that does not mean the GPL is rubbish, this just means that the Free Software Movement is recognizing that all software cannot be open source, but it can be "free as in speech," according to Richard Stallman in Revolution OS, [http://video.google.com/videoplay?docid=7707585592627775409#](http://video.google.com/videoplay?docid=7707585592627775409#) . Open Source must be done properly.
