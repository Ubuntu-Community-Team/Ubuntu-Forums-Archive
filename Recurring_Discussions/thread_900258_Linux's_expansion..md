### Post by LeadHop on 2008-08-25
First, a bit of gushing over how much i love my ubuntu:

Hi all, a relatively linux noob here.  been using ubuntu for about a year now  after a good convincing by a friend and been in love with it since then, and now i'm convincing my friends to switch to linux!  he made me make the whole thing look like a mac, but he fell in love with linux faster than i did.

I enjoy ubuntu because of its ability to do much more with my hardware than xp.  I dual boot xp mainly because of insecurity reasons (not that any of my insecurities have become real - ie programs not working, not finding a way around the problem, etc.), but use ubuntu 99% of the time, the 1% being dl'ing podcasts (6gb allotment for linux, 92gb for xp holding over 60gb of podcasts... too lazy to move! lol), skype usage (although i'm not sure if webcam compatibility got resolved on hardy), wireless connections (couldn't connect to some wifi networks in europe) etc.  Plus i haven't had the nightmarish problems most windows users report, having crashed twice in the 4 years i've had my laptop.

But all the wonderful things aside, i do have a few questions on the awesome shelf life on linux.  They're not subtle shots at linux or theories; just some thoughts i had.  i'm not much of a tech-head, so apologies in advance if some of these questions seem obvious!

1) Linux boasts security and anti-virus measures compared to linux.  However, i'm wondering if this is more that there are more viruses and security hacks aimed towards windows users rather than linux security.  If 95% of all pc's had linux, wouldn't someone invent a virus/hack for linux and put linux in the same shoes as windows?  Or is linux just harder to "crack"?  if more and more people used linux, is it only a matter of time before someone starts a virus for linux and we start having to get virus scanners like windows?  not a computer expert, so i might be saying something stupid here, but my thinking is that a lot of people can track down ip addresses and personal info easily through windows because people learned that most people use windows, learned the code, and learned how to crack it.  couldn't linux be put into the same position?

2) in terms of personalizing, i think windows has just as many options available for themes, etc. as linux does, although if you search now, you'll find many a bug/virus-laden applications spread throughout the web.  with linux, i trust that i'll be safe whenever i get applications.  However, will this be something to say goodbye to when everyone starts using linux?

3) while mac users had their moment of glory from coming from a relatively small number to a boom of users, they're starting to notice how mac is becoming like microsoft now - releasing things that are not ready, buggy, and not what is promised.  if linux goes through a similar boom, will it suffer the same problems, or are the previous two suffering problems because they wanted to make money with products that are not ready?

overall i see linux as a "virgin" land that's not particularly secure in defense, rather that it's grounds that are not treaded upon very often by bots that spread viruses; that is, it's not particularly secure, it's just that no one thinks to get us because we're such small numbers.  I'm wondering if that analogy would be true if linux was used by numbers similar to windows users.  

just some late night thoughts as i was writing a paper!  hope to hear your thoughts!

A self-replicating and self-installing virus without user intervention would probably not make it very far in Linux land because of how Linux is designed (has nothing to do with marketshare). And if one magically appeared, having antivirus installed prior to its appearance wouldn't stop it in any way.

Any malware based on social engineering (tricking the user into installing it) could be quite dangerous is Linux attracts the same current Windows-using population *without proper security education*. So if people visit a website that says "You need to install this .deb file to view the videos on this website" and then they double-click-install the malicious .deb, the .deb has the power to fully compromise the system.

Antivirus for workstations (as opposed to mail servers), in my experience, is completely useless, even on Windows. It's a resource-hog that offers you complacency instead of security. For more details, read [Does Ubuntu need antivirus?](http://www.psychocats.net/ubuntucat/does-ubuntu-need-antivirus/)

> 2) in terms of personalizing, i think windows has just as many options available for themes, etc. as linux does, although if you search now, you'll find many a bug/virus-laden applications spread throughout the web.  with linux, i trust that i'll be safe whenever i get applications.  However, will this be something to say goodbye to when everyone starts using linux? All of the personalization options for Windows involve installing software that hacks the registry and/or kernel and requires at least one reboot. In Linux, you can just download a theme and drag and drop it to the theme manager window to change themes. Far less involved.

I remember trying to install FlyaKite in Windows XP once. When I rebooted, it gave me a blue screen of death. That won't happen if you install a theme in Linux, as the themes are just configuration files and images. They don't touch the kernel or any system files.

In Linux software distribution goes through trusted and authenticated repositories. You don't surf the web and hope the application is trustworthy. The major hole in this system is it's possible (though it happens only infrequently) that the software repositories themselves can be compromised.

> 3) while mac users had their moment of glory from coming from a relatively small number to a boom of users, they're starting to notice how mac is becoming like microsoft now - releasing things that are not ready, buggy, and not what is promised.  if linux goes through a similar boom, will it suffer the same problems, or are the previous two suffering problems because they wanted to make money with products that are not ready?

overall i see linux as a "virgin" land that's not particularly secure in defense, rather that it's grounds that are not treaded upon very often by bots that spread viruses; that is, it's not particularly secure, it's just that no one thinks to get us because we're such small numbers.  I'm wondering if that analogy would be true if linux was used by numbers similar to windows users.   In my experience, buggy software and releases are not the exclusive domain of any platform. In Mac, Windows, and Linux, there are solid releases, mostly solid releases, and buggy releases. Bugginess is a fact of software life.

1. Linux is more secure for two main reasons (in my opinion):

a) It doesn't use administrator by default.  Running as admin is a great way to hose your system.

b) Most Linux users are a bit more tech-competent.  If you don't do stupid things, you won't get infected on ANY operating system.

2. Windows is nowhere near as customizable.  You can choose different window managers, panel layouts (I don't use a panel), and almost anything else you want on Linux.  If you use standard GNOME Ubuntu, visit [GNOME-look ]("http://www.gnome-look.org/")for themes.  KDE, Xfce, and *box have sister sites.

3. Because Linux is open source, there are many different distributions.  You probably had to look through them when you chose Ubuntu.  If you don't like how one distro behaves, just switch to another.
