### Post by bgerlich on 2009-01-24
Maybe I wasn't clear enough. What I'm getting at is, that there is a good reason why the system of downloading programs as an exe package was never implemented. It is flawed and dangerous. A user should never have to worry where to put a program, user should never start a program downloaded off the internet without at least some sanity check that archives provide. If a package is corrupt in Ubuntu it simply won't install, a corrupt executable at best will fail, at worst take the whole system with itself. Of course you can set the executable bit yourself, but nowadays it is only handy for scripts, a normal user will never have to do that.

User should never also worry about library versions the system is using, if a package is incompatible with the installed libraries the package manager will either handle the dependencies gracefully or fail to install the package. The user will never have to wonder whether to overwrite system libraries or not.

That is why the package system is superior to the free-for-all Windows offers. Handling new programs in windows is similar only to what you get when you compile your linux from scratch and I'm not talking about Gentoo which actually uses a packaging system.
