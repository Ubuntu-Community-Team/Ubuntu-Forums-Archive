### Post by jhsu802701 on 2010-10-12
Overall, I like the Linux community.  However, there are elitists who think everyone should be a Linux expert and shouldn't have things spoon-fed to them.  In fact, many of these elitists pan Ubuntu and Mint users as the "great unwashed masses".

Yet at the same time, everyone wants Linux to spread and Windows to fade away.  However, in order for this to happen, this means that the "great unwashed masses" need to start using Linux.

Many people have complained that Linux has too many distros, but I don't.  Having so many options is a good thing.  The hardcore Linux users can use Debian.  The even more hardcore users can use Slackware, Gentoo, and ArchLinux.  The ultra-hardcore can use Linux From Scratch.  The "great unwashed masses" also have a number of great distros - Ubuntu, Mint, antiX Linux, Puppy Linux, and others.  Every major opinion in the Linux world is well represented in at least one distro.

That said, I'd love to see antiX Linux become the go-to distro for newbies.  It's much lighter than Ubuntu and Mint and can thus extend the useful life of a computer for even longer.

### Post by XubuRoxMySox on 2010-10-13
For alot of us, a 'puter is like any other household appliance. We don't mind learning how to *maintain* it, but we have no need nor wish to become Certified Repair Technicians for each of our household appliances. It's a tool. A wonderful, versatile, and very necessary one, but a tool nevertheless. Maintaining it is easy with "friendly" distros like the 'buntu family, Mint, Mepis, Opensuse, Fedora, etc.

For tinkerers, there are the others the OP mentioned, and many more. Naturally a tinkerer's machine will be faster and more super-efficient than mine. But that's expected and accepted. Mine is convenient, pretty, kid-friendly, simple, and "just works."

I've said it before, but since this is a recurring discussion:

I wonder if some of the elitist supergeeks have become victims of their own success in a way. Alot of wonderful supergeeks have worked very hard for a long time to make Linux easy enough for their 7-year-old to use. They've done a great job! But now they complain that too many 7-year-olds are "dumbing down" Linux by not contributing new code or something? Um, what did you *expect* to happen when you made a Linux easy enough for even this silly little dixiedancer to use?

Thank God for supergeeks! And I'm very grateful to have had the awesome power of Debian Linux "tamed" and brought to my desktop on a silver platter, simplified and neatly packaged with a pretty purple bow, so I can just unwrap it and use it right away! But don't get mad at the 7-year-olds you did all that work for. If we had known that we were expected to become experts in order to be "worthy of Linux," we would have stuck with Windows or Mac.

Actually it's fun to learn about Linux, discovering how to customize it and make it "just perfect for me." I can make mine a little faster (just by turning off some unneeded stuff like bluetooth), use different software than the default (I like Opera instead of Firefox), explore, tweak, stuff like that. That's *maintenance* though. I'm not ever going to be a Linux Guru. But I've learned enough to make my own customized Linux remix and share it with other kids from school, and I can even support them while they learn. It's all good.

-Robin

### Post by Peter09 on 2010-10-13
I think we should understand the Ubuntu (as an example) is able to stand up as an excellent product when sold as part of a PC (just like windows is). What is more it is probably more stable than windows.

The issue really arises because most people have never tried to install an operating system onto hardware not specifically designed to support it. They cannot understand why it should be a problem.

With a set of hardware that has not been configured to work together Windows will have as much (if not more) problems that Linux. Couple that with the failure of hardware manufactures to make compatible drivers for Linux and you get the sort of issues that we find.

I think we compond this problem by, every six months, issuing a latest and greatest update, this often leads uniformed users into updating a system that meets their needs 100% to a system that has bugs.
