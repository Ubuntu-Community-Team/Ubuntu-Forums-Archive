### Post by Copper Bezel on 2011-08-31
> This is kind of an issue, Linux is boring. It does nothing extraordinary, so why do so many people love it to the point they ditch Windows and OSX?
You're really sure you're not trolling?

They're all just operating systems, and there's very little that's inherently interesting with that, but it's not possible to be an OSX or Windows enthusiast or even hobbyist because the systems are so locked down. In terms of getting things done, Windows has advantages and OSX has advantages and various Linuxes have advantages and you're almost certainly aware of a number of them. In terms of tricking a machine out with all the special bells and whistles only you have, you need an open platform, and GNU/Linux is the only one really in the running. So if a person is enthusiastic about computers, they're probably using Linux, so they're most likely to be enthusiastic *about* Linux.

Fairly simple.

As for "ditching" OSX and Windows - for a lot of us, it's more like finally having been able to be rid of them at some point or another. Windows, particularly, is a bunch of hassles I don't need. A lot of people would feel the same way about Ubuntu. Strokes and folks and things. But I don't *need* Windows or OSX, so I don't have them. It's not as if I was born with them and had them removed.

### Post by jwbrase on 2011-09-11
> **uRock said:**
> Removing the burning software fixed the issue, therefore the problem was with the 3rd party software, not the OS.

Wrong. The *immediate* cause of the problem was the third party software, but in a properly designed OS, bugs in an application are confined to that application.

An analogy: If I go away for a week and leave my door wide open, and a thief waltzes in and takes everything, is it my fault or the thief's that my stuff got stolen?

If something wasn't wrong with the OS, you'd get an error along the lines of "Easy CD Creator 5 has encountered a problem and needs to close" rather than a bluescreen. That's why Win9x was so notoriously unreliable: Badly written applications could fairly easily scribble over data critical to the OS, whereupon you'd get a bluescreen.

WinXP is generally better about that, thus bluescreens are rarer, but a bluescreen always indicates an OS-level problem. Either Easy CD Creator had a pointer go wild and it overwrote OS data (in which case the kernel wasn't controlling where applications could write to well enough), or (more likely) it passed bogus data to the OS in a system call, and a bug in the OS caused the OS to screw up its own data in response instead of realizing that the passed data was bogus and telling the application that it couldn't complete the syscall. The worst case scenario (although fairly unlikely) is that Easy CD Creator made a syscall, passed perfectly valid data, and that a bug in the OS caused it to choke on it. 

About the only scenario in which XP isn't at least partially to blame is if Easy CD Creator had installed some kernel-level driver (effectively making itself part of the OS) which was buggy. (Of course, microkernel advocates would tell us that the OS is still to blame for allowing drivers in kernel space, and kernel crashes from buggy drivers are what you get for using Windows/OS X/Linux rather than Minix).

> It was XP Home, there was no root account, just mine.

Well, what I was alluding to there was the fact that Windows user accounts pre-Vista defaulted to being administrator accounts and tended to have privileges pretty close to what root has on Linux.

### Post by Copper Bezel on 2011-09-11
jwbrase, as I'd said earlier, on a desktop system, there's no effective difference between hanging the kernel and hanging, say, the GPU hanging instead, or crashing the X server. They all result in exactly the same loss of data. Now, Linux isn't immune to kernel panics, but they're usually not something a particular badly-written enduser app can evoke, sure. GPU hangs and X server crashes, however, most certainly are.

The philosophy you're describing makes intuitive sense, and I'd generally agree with it. An operating system is responsible for its own stability regardless of what's running on top of it, which it ought to have a handle on. Sure. But Linux does not have a clear advantage over Windows in this regard.

The only real exception is in the desktop environment itself - unless the window manager crashes, it's still *much* more difficult for an individual nonresponsive program to hang the entire desktop environment on a typical Linux than it is on Windows. That's a clear advantage to Linux, honestly one of the most comforting things about it for me - but it's meaningless on the philosophical scale you're using to approach this.
