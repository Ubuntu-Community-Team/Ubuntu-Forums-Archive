I've used Gnome 2.24 and KDE 4.1 at length.  Obviously at this current point in time Gnome is more complete than KDE because KDE just underwent a major transition step.  The counter to that is that if you must, KDE 3.5 is still vaguely supported; otherwise, wait 3 weeks for 4.2 (or a few months if you want to stick to the Ubuntu repos).

Qt is GPL as opposed to GTK's LGPL, so you can't write proprietary software using KDE's primary toolkit unless you purchase a special license from Trolltech/Nokia.  If that has any impact on your ideals one way or the other (maybe you like enforced open source, or maybe you like the freedom to issue your GUI based software under your choice of license), there you go.

So really, it's just a matter of preference.  Try out both.  See what you like.  I've found a few features in KDE that I like over Gnome, but there are features that I miss from Gnome as well.  Everything is a trade off.

### Post by Mr. Picklesworth on 2009-01-07
> **aaaantoine said:**
> On the other hand, Gnome has been employing Mono to great lengths, which is a controversial design direction.

GNOME has not been employing Mono at all. Popular apps that work well in GNOME (and distros like to have by default) are developed in Mono. I doubt that Mono will ever be in an official GNOME release because that would open up a lot of new and redundant dependencies.

It will continue to be a great language, though, so chances are high that distros will continue to use Mono quite happily. (I just recently learned, for example, that Unity 3D is an amazing graphics engine. With it, Mono is being used for commercial software on platforms like the iPhone and Nintendo Wii, thus annihilating Microsoft .Net on its own turf by being so pleasantly portable). It's quite unfair to label GNOME for the actions of a third party, when really what this shows is that GNOME has a diverse selection of software that fits the environment.
I've been happy to see a lot of proprietary vendors use technologies native to GNOME - GTK, most often - for their Linux software. Amazon has recently done this, for example. It makes things easier for everyone. It may be proprietary software, but the important part is that it works well and feels consistent with the rest of the desktop, which using GTK allows.

PS: I will jump out of the Mono discussion now because I vowed never to enter one again!
