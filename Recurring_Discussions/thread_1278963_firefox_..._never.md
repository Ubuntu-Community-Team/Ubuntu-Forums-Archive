### Post by Tux Aubrey on 2009-09-30
I can't answer the security question, but I agree that Chromium (I assume you mean the native Linux version of the Google Chrome browser) is a wonderful lightweight browser.  I use it on my customized netbook installs in preference to the others I've tried (like Seamonkey).  It certainly outperforms Firefox for regular web surfing on light machines.

There's absolutely zero chance of it being included by default in any forthcoming Ubuntu distro.  Chromium is still under heavy development, has several known killer bugs and Firefox is too good a selling point for converters from Windows for that to happen.

Firefox is really the most fully featured browser around (just look at all the extensions and plugins that have no equivalents in Chromium or most other browsers).  It does run like a drugged slug for me at times, especially on shutdown, but I still prefer it on my main machine because of the plugins I use regularly.

I would personally like to see Xubuntu come with a lighter weight browser (and "productivity" apps too) but I don't see much interest in that happening either.

The moblin netbook remix does come with the Moblin browser but I don't like that one at all.  Maybe it will develop in time.

Epiphany (the Gnome browser) has apparently undergone a lot of development recently and is reportedly very fast too. 

We are blessed (or cursed) by the amount of choice right now - and may it continue!

### Post by lovinglinux on 2009-09-30
Chromium is really fast, but that's it. It doesn't have a fraction of Firefox features. Besides, you should see how fast Firefox 3.6a1. It's really impressive. If you compare a vanilla Firefox 3.6 with Chromium, they are not very different.

What usually degrades Firefox performance is database junk and extensions. Database performance can be improved with a simple script to optimize the databases and get rid of the junk. Extensions are a different story. If you are experiencing delays when opening and closing Firefox is probably because you have some extensions that perform tasks when loading and unloading Firefox, specially if these tasks involve database reading/writing. When Firefox starts it loads all extensions pages and if they have some functions that are automatically performed during the extension loading time, then Firefox be will only fully opened when these tasks are completed. 

If you want to optimize Firefox see [**[COLOR="DarkRed"]Firefox optimization and troubleshooting thread[/COLOR]**](http://ubuntuforums.org/showthread.php?t=1193567).
