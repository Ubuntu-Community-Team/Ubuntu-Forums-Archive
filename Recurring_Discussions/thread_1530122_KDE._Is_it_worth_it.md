### Post by prshah on 2010-07-13
> **Nick_Jinn said:**
> Do all these environments play well with each other? 

Unfortunately, no. (Inspite of the other posts). Gnome uses pulseaudio, KDE uses Phonon. Putting them both on the same install is a sure-fire way to audio grief.

GNOME uses compiz, KDE uses a different compositing manager... can't recall the name right now... ah, KWin.

Handling of dynamic (USB stick) devices differs from Gnome and KDE. It's less problematic than before, but sometimes...

There are some permissions problems for samba network shares.

I _used_ to experience random lockups when I had KDE and Gnome installed together (but that was over a year back), so don't know about now.

Bottom line: It's better you select one and stick to it. KDE is nice for a fling, but I generally end up switching back to GNOME. Not that this is a judgment against KDE, I've never really stuck with it long enough. KDE _is_ very good; gnome is familiar, and works well on "challenged" hardware.

### Post by ankspo71 on 2010-07-13
Hi again,
As for customization...
Basically, Gnome uses GTK + Metacity themes, and optionally Compiz with Emerald, and there is a huge selection of themes at gnome-look.org. 

With KDE4, I find there is a decent selection of themes but there are less quantity to choose from. However, there are different categories of theme styles that can be used to get the look you want. See kde-look.org for what they have listed there.

Conky, Docks, Clocks, Widgets, can all be used for the KDE desktop. If you already use conky and want to use it in KDE, you will see conky's transparency isn't 100% transparent though. 

If you end up not liking KDE's 'Kwin' 3D effects, it is still possible to use Compiz and even Emerald. I've always had a bit of trouble getting it to appear effortlessly on startup though. (I think gnome does it better)

If you like using nautilus scripts, you might like KDE's 'service menus'.

Colors and theme styles can be altered easily to your liking, one of the built in features of KDE4. There is a built in wallpaper changer in kde4 too. 

I like KDE's panel and menu transparency better because it doesn't make the icons transparent too.

All in all. I think gnome and KDE are about tied is terms of customization.

### Post by prshah on 2010-07-14
> **Nick_Jinn said:**
> I am seeing some cool themes on gnome-look.org/ but I cant figure out how to install them. 

Open System-Preferences-Appearance; then drag and drop the downloaded .tar.gz file onto the window. Your theme will be automatically installed and will show up immediately (ie, no need to close/refresh the window).

One important note: SOME gnome-look themes have been found to be malicious; please be alert and careful when installing themes; multiple comments will generally give a good idea, as will eyeballing the theme file contents.

### Post by ankspo71 on 2010-07-14
Hi,
Here is a link that shows how you can install themes in Ubuntu.
[http://www.simplehelp.net/2009/05/26/how-to-install-ubuntu-themes/](http://www.simplehelp.net/2009/05/26/how-to-install-ubuntu-themes/)

Sometimes themes are packaged differently causing it to be not installable using the above method though. So that is where we need to manually extract the files then place them in a hidden folder inside your home directory.

/home/<username>/.themes  (for GTK,GTK2, and Metacity themes)

If you can't find a hidden folder named ".themes", you will have to create it yourself, then place the extracted contents into it. I think Ctrl+H is the shortcut to reveal hidden files and folders. The period in front of the folder names is what makes the folders hidden.

After you place your theme files in the hidden .themes folder, you will have to apply the theme by going back to the appearance preferences, find the new theme in the theme preview window and select it, then click apply. 

If the theme doesn't offer a theme preview to look at and select from, you will need to click on the customize button, choose a 'control' to use and then a window border, and then apply it.

It has been a while since I have used the Gnome desktop, so I hope I have explained this well enough.:P

### Post by Xianath on 2010-07-17
Unfortunately for KDE, a major portion of Linux users switched from Windows because a) they hated it, b) they had outdated hardware and c) they like open source ideals. KDE follows the Microsoft UI paradigms, it has historically been slower on minimal spec machines, and its UI framework is dual-licensed. All reasons for people to jump to conclusions based on prejudice and not actual reason.

I like both for different reasons, but tend to stick with KDE because it gives me more control. A number of things I've been able to customize in KDE are either not available in Gnome or need to be tweaked in gconf (hardly user-friendly). That said, I follow KDE development closely (running 4.5 RC2 right now) and obviously some things break with development releases, so I keep going back to Gnome when they do. The time I stay with Gnome is longer every time, but I still end up using KDE in the long run.

The two DEs  play along nicely though, so no issues there. In KDE you actually have a choice about your audio framework, and more control over what goes where so for example you can use your onboard card for voice calls and your 5.1 for multimedia.

KDE is certainly worth a shot. I don't know why people find it unfriendly, probably because they're used to Gnome (like Mac users trying Windows). The Plasma workspace is a somewhat different concept but desktop widgets shouldn't be news to anyone these days. There are also more things to tweak so more areas where people can shoot themselves in the foot. The out-of-the-box experience though should be familiar especially to Windows users.

Here are some of the things I like about KDE:

1) Customization -- no other DE beats it on this ground:
* download widgets, wallpapers, themes etc. from within the actual dialog you're in
* stick different applications on their own desktops or screens
* configure global shortcuts that work even if an application is not currently in focus
* configure notifications (sound, pop-up, taskbar etc.) for multiple application events

2) Integration
* receive an email from someone in and you see their picture and their IM status
* protocol-agnostic file access -- edit a file on your FTP server, attach a picture to an email via SFTP, etc. Gnome has GVFS but it's simplified to average joe level and gets in the way of power users

3) Amarok :) Even with what they did to it in 2.x, it's still the best music player around

4) Visuals. It just looks better to me, always has. Gnome has always taken a more minimalistic approach, and before Ubuntu happened to it, it was plain ugly. It is now both sleek and lean, but I still prefer the cold, professional look of KDE.

5) Bug reporting is really easy.

So is it worth it? Certainly. Give it a couple of days, see how stuff is done and what you can do. If you don't like it, you can always go back. If you end up liking it, try the development release and help reporting bugs. Like I said, it's easy :)
