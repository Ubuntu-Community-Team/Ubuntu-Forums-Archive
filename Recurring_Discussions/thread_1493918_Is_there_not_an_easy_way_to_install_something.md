### Post by NMFTM on 2010-05-27
> **TheUnwiseman said:**
> But understanding how your computer works beneath the point-and-click interface level is important to being an educated user.  You'll find that your entire computer experience will make more sense if you learn these "arcane" (I think you meant "archaic") commands and processes.
I don't know if using the CLI instead of the GUI is exactly "understanding how the computer works". For example, using a GUI frontend for GPG to encrypt things is just selecting the file you wish to encrypt and than selectin the key you wish to use. Using the CLI is just typing in "gpg -e <file_name>" than selecting the key. In both instances you can encrypt and decrypt files without really knowing how all of the advanced processes and algorithms behind it work.

The only people who truly know how a computer works are the scientists who work for companies like Intel and AMD designing the processor architectures and the people who write applications in assembly language.

The only thing more hardcore than assembly language is coding directly in machine language (1's and 0's). And you could probably fit the number of people in the world who know how to do that into my living room.
