### Post by Dr. C on 2010-06-05
Free Software refers to software that respects the basic 4 freedoms ([http://www.gnu.org]("http://www.gnu.org")):>     *  The freedom to run the program, for any purpose (freedom 0).
    * The freedom to study how the program works, and adapt it to your needs (freedom 1). Access to the source code is a precondition for this.
    * The freedom to redistribute copies so you can help your neighbor (freedom 2).
    * The freedom to improve the program, and release your improvements to the public, so that the whole community benefits (freedom 3). Access to the source code is a precondition for this.
  It is about freedom and not price so Free Software can be sold for a price. It is also referred to as "Free-Libre Software", Software Libre" or "Free as in Speech Software" to distinguish it from "Freeware or "Gratis Software" or "Free as in Beer Software" 

Open Source software refers to software that is licensed under an Open Source License as defined by the [Open Source Initiative]("http://www.opensource.org/licenses/alphabetical") 

Most popular licenses including GPL, LGPL, AGPL, Apache, Mozilla, and many more are both Free Software and Open Source so the terms can actually be used interchangeably ***when describing the software***. The terms however are very different when describing the values behind the respective Free Software and Open Source communities as RMS describes in his essay [Why Open Source misses the point of Free Software]("http://www.gnu.org/philosophy/open-source-misses-the-point.html")

Now here is the interesting part. Take a commonly used Free Libre Open Source Software program and chances are that there are contributions from both strong Free Software proponents and strong Open Source proponents. For a modern GNU/Linux distribution such as Ubuntu this is guaranteed.

### Post by mickie.kext on 2010-06-05
Open source necessarily **do** include Freedom, and Free Software necessarily **do** include source code. Free software definition is similar in spirit to open source definition, even though the wording is completely different. [Open source definition]("http://www.opensource.org/osd.html") is modeled around [Debian Free Software guidlines]("http://en.wikipedia.org/wiki/Debian_Free_Software_Guidelines") which are in turn inspired by [Free Software definition]("http://www.gnu.org/philosophy/free-sw.html"). **So it's the same software.**

Difference is in ideology that people use to advocate that software. 

Free Software advocates talk about Software Freedom as prerequisite for having good software which respects users Freedom rather than subjugating users. They do not talk about technical merits too much because Freedom is more important then technology - what good is advanced technology if you have to agree to ridiculous EULAs and promise that you wont share it with anybody, in order to be able to use it? They also figure that technological factors come of its own if Freedom is there. As in, if source code is under free license, people will improve upon it and make it near perfect. 


When talking with business people, they tend to understand Free in context of price which is wrong understanding. Free Software stand for much more that just $0-priced software. That is why term 'open source' has been coined, to emphasis openness of the code and accelerate corporate adoption of FLOSS (Free/Libre and open source software) with those people who are initially put of by word "free". Also, open source advocates do not talk to much about freedom, they talk about technical merits of open source software and superiority of open source development model. 


Both are arguments to the same end, and are intertwined and parallel in same time. They are not opposed to each other, it is more like "Who is more right".  For those who think that both camps are in right, there is a term FLOSS which means Free Libre Open Source Software. Word Libre is there to emphasize that Free means Liberty, not price. So FLOSS means: No discussion, they are both right. 


PS: There is also a term Freeware which has nothing to do with neither open source nor Free Software. It is merely name for zero cost binaries which are in fact proprietary software.

### Post by chessnerd on 2010-06-06
Free Software - software which the user can, for any reason, use, modify, copy, distribute, etc.

Open Source - software that has the source code openly available for anyone to view.

Depending on how loosely Free Software is defined, closed-source programs that can be freely used, copied, distributed, etc. can be considered Free Software. However, most feel that Free Software must have the source code open so that users can modify it.

Open Source doesn't have to be Free Software at all. There are programs that have the source code open to the public, but that the end user is not given any freedom to modify, copy, or distribute. An example of this would be Minix before version 3.
