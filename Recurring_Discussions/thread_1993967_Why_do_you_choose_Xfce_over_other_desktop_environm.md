### Post by Morbius1 on 2012-06-02
This is an example of why I use XFCE:  [http://blog.xfce.org](http://blog.xfce.org)

It has to do with a technical matter but I'm using it to illustrate their priorities:
> Technically gtk3 is nothing different then gtk2 when it comes to  programming. The hard parts are porting of some custom widgets (drawing  and size allocation), replacements of some deprecated symbols and link  to gtk3 libs. [COLOR=Blue]All things a *user* is not going to notice if we do it right.[/COLOR] 

Gtk3 is also not faster than gtk2, maybe there are some areas were it  got a bit faster, but so there are areas where performance decreased a  bit. Nothing shocking here.

 An issue I’m aware of is theming issues in gtk3. From what I  understand this changed back and forward in gtk 3.0, 3.2 and 3.4. So we  need to decide which version we require to get this working  consistently, because people will complain if only the *Raleigh* theme can be used :).

 From the Xfce point of view there is (again) the resource problem for  porting all plugins, because if for example the panel is ported to  gtk3, also the plugins need to be ported. Not all goodies are  maintained, but usually they work and distros can compile them. [COLOR=Blue]If in  4.12 suddenly 50% of the external plugins are not working that will be  another thing *users* will notice.[/COLOR]
Consistency. Change the technology if you must but don't remove function. Change the way something is done if the technology demands it but do not remove function. It's highly configurable ( and some would argue it has to be :wink: ) in it's current state and that is something Gnome3 is not regardless of what you slap on top of it like Unity or Cinnamon.
