### Post by pp. on 2008-11-25
> **Soldier4Christ said:**
> What do you do if you are programming?

A program is - as others have said - a complex set of instructions which tell a computer how to go about certain tasks.

Provided that the program is to be 'useful', i.e. meant to assist you in performing some tasks or solving some problems, there often is an ordered sequence of steps you take in order to 'make' a program.

Study the problem you want solved or the task you want performed. Write down in an orderly manner some aspects of the problem or task. There are some notations which have been designed to do exactly that.

Construct a kind of blueprint or overall structure of the program you are going to make. Decompose the problem to be solved into different parts or pieces which you can solve individually. Also decompose the program-to-be into different 'blocks' which are dedicated to different kinds of tasks needed for the working of the program, such as showing things on the screen, saving data to a database, showing a help screen to the user which is adequate to the actual situation the user is trying to cope with in your program and so on.

For all those tasks there are also some notations which have been designed to express exactly what you need to express at this point in time. The result might remotely resemble a sewing pattern or a longish grocery list, depending on the tools you use for that part of the work.

Proceed to code the individual instructions and some housekeeping data which faithfully implement your overall blueprint from the last step.

After carefully checking all your work done so far, design some sample data with which to test run your newly made program. Be prepared to restart from the beginning and/or to spend much time looking for the causes why your program is not quite doing what you intended it to do.

Somewhere in between, also write down what the prospective user needs to know in order to use your program.
