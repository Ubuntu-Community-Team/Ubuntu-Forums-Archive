I have here 1 Laptop for my Job with a fresh install of Windows XP, 1 Laptop with Ubuntu Jaunty 32 bit and 1 Desktop with Kubuntu Jaunty 64 Bit, with KDE 4.3.2

The job-Laptop is a HP mobile workstation (specs: Intel Centrino Dual core, 2x 2,86 GHZ, 3 GB RAM, 800 MHZ front side bus, Nvidia Quadro graphics)

the Ubuntu Laptop is a normal Lenovo 3000 N200 (spec: Intel Pentium Dual Core, 2x 2 GHZ, 2 Gb RAM, 533 MHZ front side bus, Nvidia Geforce 7300 GO graphics)

the desktop specs: AMD Athlon 64 X2, 2x 3 GHZ, 2 Gb RAM, 800 MHZ front side bus, Nvidia Geforce 8400 GS

when I press the respective on/off buttons at the same time, I see that the Bios of the Lenovo takes twice the time to be processed as in the HP, then there is Grub, but on the HP you have something similar: the menu to choose either "normal Windows XP or Windows XP for 3 Gb RAM...

to make a long story short; Jaunty catches up very quickly when it comes to loading the system. So it comes that I can surf the internet already in Jaunty on the Lenovo when Windows XP on the HP is ready for login. Believe it or not: it is a fact.

I did similar tests between the Lenovo Laptop and the Desktop (where KDE is installed), as the Desktop still was running a 32 bit system. Result is: with KDE 4.2 (the original setup in Kubuntu Jaunty) the Desktop was slower than the Laptop. Although the desktop has better specs than the Laptop.
But this changed hardly when KDE 4.3 was released. From now on, KDE is as fast as Gnome. From the personal feeling I would say that KDE 4.3.2 is a little bit faster than Gnome.

Now, the desktop running 64 bit... this speed is just incredible :guitar: :D

when booted, both, Gnome and KDE run at nearly the same speed. In Windows, some applications need longer to load. For example the Firefox. Its just insanely slow to load. Or, lets say Office: OpenOffice in Linux is much faster than Microsoft Office in Windows in terms of startuptime and also in saving files.
The biggest difference is the disk usage. Windows does excessive "swapping" or whatever this is. So the HDD is used lots of times. To me it looks like that these in/out operations take time and cost energy. 

the two Linux machines don't do that. The user has the impression that they just sit there and wait for the next job, while the windows machine has trouble to catch up to the wishes and the working speed of the user.

### Post by Squonk07 on 2009-10-20
This is all very academic because the concept of "speed" is difficult to quantify. These sorts of things can be benchmarked, of course, but internally relevant results (e.g. benchmarks) are rarely externally relevant (in actual practice). Further, there are few (if any) benchmark standards across all these platforms (correct me if I'm totally wrong).

I think a more important metric here is consistency. As has been noted before, I've noticed that Windows XP starts out feeling subjectively quick, but after being in use for a month and then whenever it's been running without a reboot for about a week it feels subjectively slower at times. The effect is inconsistent, so it's jarring when it happens. I've noticed a similar pattern in KDE (in the few times I've used it for any extensive period of time). For me at least, Gnome is the most consistent of the three options, though the Menu tends to hang the first time I try to open it after a reboot. At least that's a consistent occurrence, which I have come to expect.

In the poll I voted *Other* because, without actual statistical data, this is all highly subjective. That having been said, for me Gnome offers the most consistent performance. Something tells me that even if a definitive answer to this came up, it wouldn't stop people from making subjective judgments. ;)
