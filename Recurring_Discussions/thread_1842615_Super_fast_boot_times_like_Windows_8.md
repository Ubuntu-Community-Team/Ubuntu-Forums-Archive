Actually, the boot to OS stage of start is fairly constant for most OS's and is very dependent on the hardware. What takes the time is the loading up and starting of all the OS running processes and start up apps like the GUI, the AV the firewall and anything else that the User chooses to initiate 'at Startup' etc. Windows does a bit of sleight of hand by opening the desktop visuals before any of the startup apps are actually ready to run. Most (all?) GNU/Linux desktop distros insist on opening the desktop after the system is fully started when the system is ready to run. Windows does not and, depending upon your application configuration can remain unusable for minutes thereafter. (The worst case I have ever met was a Windows box which took 45 minutes to be ready to use after the Desktop appeared - and that was a Win 7 system)

When the Ubuntu desktop appears the system is ready to use immediately, in my experience this is always much, much quicker than any Windows system I have ever worked on. I don't know how Ubuntu compares with any other GNU/Linux desktop distro, never needed to try comparisons. Slow startups aren't an issue with Linux, never have been.

As a community, I think we should realize that improvements can always be made and take windows 8's fast boot times as a challenge for us to match (an exceed!). I think EFI is going to help a lot, but also booting to a light-weight compositing UI (wayland???) is another way forward.
