### Post by Erik Trybom on 2008-01-19
Poor research. The reason why Ubuntu doesn't offer different desktop environments at install time is because the whole OS should fit on one CD only (Ubuntu policy). There should be no need for a fast Internet connection or a whole bunch of installation media - which is required if both Gnome and KDE are to be shipped with each installation.

I agree that naming the different versions "Ubuntu KDE Edition" etc would be better for brand recognition, but that's a minor point. I definitely do not agree that too much choice is hurting Linux/Ubuntu/Vista/whatever. Choice is great, and if you have no idea or don't want to choose, just go with the standard - Ubuntu.

### Post by Praadur on 2008-01-19
Same here.

Honestly, what is this drive to change just for the sake of change?  There's rarely ever a logical reason behind it (I have asked around and I've never seen one yet), and the environment of the distros is just fine if people can pick what they want.

Why should I have to have a DVD with Gnome, KDE, XFCE, and Enlightenment on it just because it apparently makes it a little easier for some suit to understand?  The same is true with Edubuntu and Medibuntu.  If it's really required to generalise, then create a new distro named something like Ubuntu Easy or Omnibuntu and toss everything on that.  That way the freedom of choice is preserved.

"You must destroy in order to create." is a myth, and one I'm not fond of.
