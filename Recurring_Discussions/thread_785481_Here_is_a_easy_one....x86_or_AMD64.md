### Post by studo77 on 2008-05-07
Since you are familiar with Ubuntu, go for the 64bit to get the most of your hardware.

BUT since you say browser (and mail) then you should be aware that geting Sun Java is next to impossible, very tricky. So if you have sites you use, that require Suns Java, then x86 is the way to go. Flash can be somewhat slow, but not that tricky.

What i would do is get both versions. Start by installing the 64 bit, and not setting everything up _your_ way, and test the browsers facilities, and consult the 64bit forum.

If you then find that it does not work for you, install the x86.

All other programs you can install. Base files for 32bit is easily installed, and you can install programs that is not 64bit (yet).

### Post by aaaantoine on 2008-05-07
The only setback with AMD64 on 8.04 is that you can not use Sun's Java Runtime libraries.  You can only use IcedTea, which - as functional as it is - doesn't work with all sites (i.e. pogo.com).

Although Flash is also 32-bit only, you still have the option to install it with ndiswrapper.  Ubuntu 7.10 made it trivial to install, and this feature is carried over to 8.04.

Other 32-bit apps can be run as long as the necessary packages are installed.

### Post by Phenax on 2008-05-11
I would use x86 for sure.

AMD64 provides a 'slim to none' performance increase. There is more memory usage in AMD64. There are often compatibility problems with closed-source software and software that uses x86 assembly (such as emulators). Many video encoders and other software have fast x86 assembly ways of doing things which must be switched to slower C code because you are using AMD64.

Just isn't worth it to me.
