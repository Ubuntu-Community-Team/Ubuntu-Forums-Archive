### Post by phrostbyte on 2010-05-09
A stable kernel level API has a number of [serious technical disadvantages]("http://git.kernel.org/?p=linux/kernel/git/torvalds/linux-2.6.git;a=blob;f=Documentation/stable_api_nonsense.txt;h=847b342b7b20bd73758c603d833811c5ac738930;hb=HEAD"). 

That linked document is worth a read (please read it), but I will summerize the general idea from that document that is missed in this conversation: there is no one Linux kernel for a stable API to even exist in. I repeat: **there is no one Linux kernel**.

Windows doesn't have this issue. Windows is used on a very limited set of hardware and a very limited set of circumstances. Linux scales down to microcontrollers and up to supercomputers on dozens of different computer architectures. It is quite impossible to have a stable API unless you want to get rid of make config, which will make Linux useless for 99% of what it is used for these days (ie. not the desktop).
