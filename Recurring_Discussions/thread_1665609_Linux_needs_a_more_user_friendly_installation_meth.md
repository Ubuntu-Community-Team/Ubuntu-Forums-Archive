### Post by ivarn on 2011-01-12
Hello. I wanna discuss this with you because the other day I bought H-Craft, but it doesn't include an installation package, nor script or instructions. I wanna build a deb package so I can control it from synaptic.
I sent them an email about this and asked them if they could make a package, a script or atleast installation instructions. I mean I payed for the game, and the windows version includes an installation file.
The reply I got wasn't actually very helpful.
He told me to figure it out myself, and this is the price I have to pay when I'm dumb enough to use such a geeky thing as linux. that was a quote btw.
Oh well, I can't agree with his attitude but when it comes to installation, linux isn't actually very user friendly. And I guess that is one of the reasons why commercial software doesn't sell.
Linux needs to decide once and for all what package format they should go for and stick to it. We seriously do not need 100 different package methods (deb, rpm etc).
And more important, we need some soft of software that can automatically sort out the right files on the right places in the file system and make the software you installed appear in the package manager.
Yes I know you can build a deb file with dpkg -b but you still have to know where the folders and files belong in your system, and that is not a job for most pc users.
And do not try to give me that "visit the forums" or "learn how it works" crap.
Sure the forums are great, but most people do not use their pc's to learn a bunch of geeky things. These things are for people with pc hobbies. Just because you own and use a pc doesn't mean you are interested in learning how everything works on there.
Most people use their pc to either work, play games and have fun. maybe watch some films, listen to some music, surf the web, chat and stuff like that. This is normal everyday stuff. That does not include rebuilding packages.

I think it's about time that someone takes this seriously.
I think we all here want Linux to grow, right? So let's make it more user friendly on the few places where it still isn't that user friendly.

I mean sure I can blame all the people who doesn't make installation scripts or packages but in the long run that's just stupid. 

Summary:
1. Let's find a way to let deb, rpm etc... work on all the linux distros since repackaging 10 times is just stupid and "scares" people away from making software and games for linux since they will probably get a lot of emails from frustrated users.
2. Let's find a way to auto-build packages from archive files so everyone can install their games and apps without taking nerd courses.

Maybe this should be posted on the brainstorming pages instead, but oh well.
I think it's important to discuss and solve this problem once and for all.
