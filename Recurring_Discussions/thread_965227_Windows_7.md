### Post by Mr. Picklesworth on 2008-11-09
Here is that issue summed up:

Metacity is a window manager. It follows [standards]("http://standards.freedesktop.org/wm-spec/wm-spec-1.3.html") and it works efficiently.

Compiz is a graphics tech demo. It makes pretty effects, many of which have no place inside a window manager and are applied with incredible kludges. It breaks standards such as fullscreen and utility windows to very poor effect. It breaks Fitt's law. But it looks pretty.

What a lot of people don't understand is that the window manager doesn't need to do *anything* for us to have a pretty desktop except for supporting the compositing extension. For example, ARGB lets windows be transparent in certain places. It is not "compiz" that makes GNOME Do transparent in places; it is compositing in general, which Metacity supports (and would support better if distros stopped swarming to the functionality-impaired Compiz).

### Post by Mr. Picklesworth on 2008-11-10
> **phoenix_snake said:**
> please state linux advantages over other OS's?

Well since we're talking about user interface, here are some truly unique ones to get the thoughts going:

[LIST]
[*]Configurable menu accellerators. Hover over an item in a menu in most applications (note: Not Firefox, because it is not using GTK) and press a keyboard shortcut or function key of your choosing.
[*]Completely flexible choice of fonts and font sizes.
[*]Capacity to apply themes that change the entire visual style including sizes of GTK widgets. Very handy for accessibility themes like the High Contrast theme.
[*]User interface tools, as a rule, work on fluid layouts instead of fixed positions for their widgets. Very important for localization; translators don't need to give a single thought to user interface since they can expect their strings, no matter how big or how small, to magically fit.
[*]Our built in card game uses SVGs and can be resized ;)
[*]Linux has really cool and intuitive functionality for the Compose key. For example hold down right Alt (I think that's the default...), press O and then press C to get ©, or press ' and e to get é. (You may have to turn on the Compose key in Ubuntu under System -> Preferences -> Keyboard -> Layouts -> Other Options). Full list [here]("http://www.hermit.org/Linux/ComposeKeys.html"), but one shouldn't need it; if you know the special character you want, it's really easy to guess the appropriate shortcut without any digging.
[*]Very easy application descriptions in /usr/share/applications. Can have descriptions, keywords, specific names for different locales and detailed information about functionality (including MIME types). All standardized, and developers can create these .desktop files to define their applications extraordinarily easily. The result is great for the user; he can search for applications really easily via a tool like Deskbar or Beagle. Menus are also generated automatically thanks to this standard.
[*]Distributed source code control -- GIT and BZR.
[*]dbus, which is a wonderful IPC technology. The D-Feet dbus debugger is a nice way to get a feel for how cool this is. It will be even moreso with some new technologies coming from the GNOME side: [dconf]("http://live.gnome.org/dconf") (desktop neutral dbus-powered configuration system to replace gconf) and [gbus]("http://live.gnome.org/GBus") (glib/gobject bindings for dbus, making dbus really really easy).
[*](Upcoming) Telepathy, with libtelepathy (at the moment libempathy). Those are the underlying instant messaging frameworks currently used by the Empathy instant messaging client. With libtelepathy, applications can set up "pipes" for online collaboration, voice recording or games, for instance. Libtelepathy completely abstracts the method and medium of transmitting messages and makes things dead easy for everyone. The user runs the appropriate application (eg: Chess), then selects an instant messaging contact that he wants to do an online activity with and it *just works*. In the background Telepathy sets up a link using the most appropriate IM protocol (Jabber, IRC, Bonjour, MSN -- whichever works for that contact). The application then pipes its network stuff via that protocol. This way the user can do anything via one single contact list and developers don't need to create complex network code.
[/LIST]

Really, there are lots of things to explore. They are just in different places :)
