Ubuntu is, indeed, supposed to compete with Windows. However, ever since their new UI direction (especially on the newest beta) I question Canonical's ability to push Linux in the right direction as a viable alternative to Windows.

The new UI is supposed to be keyboard-oriented for quick launching of programs (type "fir" and you get Firefox). You can revert to the traditional Ubuntu desktop by logging out, clicking your name, changing the "Session" to "Ubuntu Classic" and then logging in. It's more like a traditional desktop that way.

Batch files are Windows-specific. I suspect you'd want a shell script.

Linux systems *do* use less resources in many cases. But, you're comparing a 10 year old Windows OS to a less than 6 month old Ubuntu release. Compare Ubuntu to Windows 7 (or Vista) and it'll make more sense. Would you install Windows 7 on a PC that weak? 

In the end, just use whatever works for you best. Maybe you'd rather take a look at distros such as [_Fedora_]("http://www.fedoraproject.org"), [_Scientific Linux_]("http://www.scientificlinux.org"), or [_Linux Mint's "Main" Edition_]("http://www.linuxmint.com"). Fedora pushes the latest and greatest open source software every 6 months. Scientific Linux is an enterprise-quality OS. Linux Mint is meant to be an "improved" Ubuntu. Lots of things to choose from. Since it seems like you actually do want to move away from Windows, I suggest you take the time to learn about how Linux works. It's a rewarding experience even if you don't stick with it in the end anyway.

Good luck :)

Edit regarding Java:

OpenJDK is supposed to officially replace Java, so Ubuntu packages this by default. In the past, the "Sun Java" packages were easily installable. It's pretty inconvenient, but OpenJDK works for everything I need. Install the "ubuntu-restricted-extras" package in the Ubuntu Software Centre if you want that. The installation method for the Oracle Java is the way it is because Oracle doesn't package it for Ubuntu- it's not supposed to be that hard, and it should not be.
