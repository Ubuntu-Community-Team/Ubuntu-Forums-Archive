I'm more of a "let's look at some statistics" guy than a "let's have everyone state his or her opinion" guy.

According to [this Wikimedia traffic analisys for October 1-October 31, 2012 ]("http://stats.wikimedia.org/wikimedia/squids/SquidReportOperatingSystems.htm"), the breakdown is as follows:

Linux Other  1,552 M  0.90% 
Linux Ubuntu 1,189 M  0.69%  
Linux Fedora  26.3 M  0.02% 
Linux SUSE  22.7 M  0.01% 
Linux Debian13.6 M  0.01% 
Linux Mandriva  11.1 M  0.01% 
Linux Mint  11 M  0.01% 
Linux CentOS  5.3 M  0.00% 
Linux Red Hat  4.3 M  0.00% 
Linux Kubuntu  4.1 M  0.00% 

I'd assume that "Linux other" would be people who  use a browser that identifies the user's OS as Linux, without specifying  a particular distro.  For instance, if you go to [useragentstring.com]("http://www.useragentstring.com/") using Firefox, it will show you this:
> Mozilla/5.0 (X11; Ubuntu; Linux x86_64; rv:16.0) Gecko/20100101 Firefox/16.0whereas a visit to the site using Chrome shows this:
> Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.4 (KHTML, like Gecko) Chrome/22.0.1229.94 Safari/537.4
I  haven't seen any compelling evidence to suggest that users of one  distro use Chrome more than users of another, so I'd assume that the  proportions of users in the "Linux Other" group is similar to what is  shown in the identified group.

Also, Distrowatch is not a useful metric by any stretch of the imagination.  [They say so themselves]("http://distrowatch.com/dwres.php?resource=popularity").
