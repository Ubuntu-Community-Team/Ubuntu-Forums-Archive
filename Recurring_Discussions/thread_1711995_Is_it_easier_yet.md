### Post by Seq on 2011-03-24
> **tomazzi said:**
> - chdisk: How this is possible that, huge, commercial software developer cant't write program for fixing their own FS? chdisk will mess/delete user files while in fact they are recoverable with: testdisk, GetDataBack and more.

In defense of filesystem check & repair utilities (chdisk and fsck.* included), the goal is different than data recovery tools (testdisk, etc).

[LIST]
[*]Using a filesystem utility, the end result is a 'clean' and mountable filesystem, possibly losing some data.
[*]Using a data recovery utility, the end result is a bunch of recovered data (whether or not that data was previously deleted or simply 'lost') and are left with a broken filesystem.
[/LIST]

Both utilities are needed, but the system can only know when it *needs* to use the first.

### Post by 3rdalbum on 2011-03-27
Sir, you're making the mistake of believing that this is Ubuntu's fault.

Ubuntu's native package format is .deb. You double-click a .deb package, it installs. Even better than these packages are repositories - you tell your system about the repository, and then Ubuntu can retrieve the package and get new versions as soon as they are available.

VMWare uses some silly binary installer rather than providing a proper package, so it's VMWare's fault. Most other software vendors like Google, Skype, Oracle (Virtualbox), Opera etc know to provide a native Deb package or even a repository.

Windows doesn't "make software installation easier". Windows doesn't mandate anything as regards software installation. It's just that, through lack of anything else, binary installers have become the de-facto standard method of installing anything on Windows. These work through a simple double-click on Windows, which is a bit worrying because a malicious file can hide as any other innocent file, and doesn't require explicit execute permission.

So, if the VMWare "bundle" is hard to install, blame VMWare for making it harder for you.
