### Post by tcelgen on 2010-10-26
I think both the command line and graphical interfaces are flawed, and will become obsolete at some point in the future. Maybe in 20 years, maybe in a 100.

I see the problem like this:
Command line requires you to know the command(s) to do tasks, and to understand to a certain degree how these commands function. The problem is a steep learning curve and being able to remember the correct syntax for each command. This often requires people to read and review commands, which can waste time or discourage people from using certain complicated software or commands. Also some visual based software does not work well with a command line. The plus side is understanding system function better and being able to do certain tasks quickly once you have learned the commands and proper syntax.

Graphic interfaces represent function through visual means. While you don't have to remember commands, function and speed can often be limited. Copying a large amount of files or renaming can be cumbersome compared to the command line. Graphic interfaces can also obscure function and keep the user in a constant inability to understand how the computer functions, or how to resolve problems. On the plus side graphic interfaces can appear more visually appealing and require less  time spent remembering commands. Certain programs work better with a visual interface, for example art/modeling programs.

The better solution would be to implement a system that recognizes natural speech, through various inputs. However the ultimate solution would be to create machines that can recognize human thought. You sit in front of a computer and think what you want to do and the computer does exactly that. Of course this could raise some issues. For one it would be very difficult to construct such systems and they could face tons of technical issues. Furthermore having machines understand what we think can bring ethical and privacy problems. 

But we are ultimately striving towards a machine that can understand what we want, graphic interfaces and command lines are clearly not the perfect solution(s). What are your thoughts? What is after the Command Line and the GUI?
