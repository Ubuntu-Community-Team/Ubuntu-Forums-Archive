### Post by YesWeCan on 2011-06-17
Grub in the MBR may be a problem in three circumstances:
1) If you delete or corrupt your Ubuntu partition you won't be able to boot Windows either
2) If Windows has occasion to check the MBR and finds Grub's code there it will over-write it with the standard MBR. You will only be able to boot Windows.
3) You are using an application that happens to write to the same "unallocated" sectors at the start of your disk as Grub uses.

I recommend installing Ubuntu on its own drive and make sure the Ubuntu installer puts Grub on that drive - it will default (grrr) to putting Grub on /dev/sda which will probably be your Windows drive.
If you have to install Ubuntu to the same drive then install Grub MBR on a separate drive.
If you have to use the same drive for everything make sure you have a method to replace the original MBR if you need to, eg: you have your Windows CD (or you can back up the MBR using the dd command, or you can restore an MBR using lilo).



A handy approach is to install Ubuntu to a USB stick. 4GB is enough. Then use the Grub on this stick to boot Ubuntu and Windows on your hard drive. This way no Grub MBR is required on your hard drive. You will be able to boot Windows directly from the hard drive as normal, if you need to. And you have a second Ubuntu that you can use for maintenance of your main Ubuntu.
