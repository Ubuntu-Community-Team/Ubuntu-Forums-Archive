### Post by matt_symes on 2011-01-03
Hi

> Chromium is the open source project.
Google Chrome is the brand version based on it. Google adds some things of their own.
However both use the same rendering and JavaScript engines, so developing sites for one is likely going to work for all of them.
However, extensions are kind of a different story, as those are still being added to and reworked slightly (I think) and stuff.

Also Chrome is more stable..generally. Sometimes unstable things get pushed into a Dev build, and they will likely be fixed in Chromium a few days before the new Dev version of Chrome.

From post 6 here.

[http://www.chromeplugins.org/google/chrome-talk/whats-difference-between-google-chrome-chromium-8318.html](http://www.chromeplugins.org/google/chrome-talk/whats-difference-between-google-chrome-chromium-8318.html)

### Post by cascade9 on 2011-01-03
Chromium- open soruce.

Chrome- some closed source- basicly, all the parts that google released as chromium become closed source when google takes the code from chromium. Other differences are chrome has intergrated abode flash, auto-updater, RLZ tracking (tells google when and where chrome was d/led from, and when it was installed), and a 'token' in the installer. There are some other tracking mechanisms that I've heard are 'optional' but I dotn know if they are enabled by default and need to be turned off, or vice-vesa (they are clientID, suggest, 'page not found' and a bug tracker)
