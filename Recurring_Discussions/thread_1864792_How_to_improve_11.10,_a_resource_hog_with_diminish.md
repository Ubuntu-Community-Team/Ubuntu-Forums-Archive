### Post by ARooster on 2011-10-19
Unity needs improvement. Add functionality to the desktop. In the oldest versions of visual operating systems the right click on the desktop brought up a pop-up menu with more functions than this ghastly excuse for a desktop environment. I do not wish to put a folder on the desktop and I do not wish to make a new file on the desktop. And frankly, given the fact every visual desktop environment for over a decade has had the option of adding a link to an application on the desktop I believe so should Unity.

Perhaps something like Widgets, Gadgets or whatever you wish to call them, would not be a bad idea, but the first thing that needs to be addressed in my opinion is the fact that having run only Mozilla (with this forum open) and no user-run applications (on a practically fresh install of Ubuntu 11.10) the processor usage for both cores on my Core2Duo E6600 was jumping between 9% and 30%. I am currently on Windows XP where I've got a running firewall and antivirus plus some other applications in the background, and the processor cores' activity is jumping between 0% and 2%.

The startup time of my 11.10 is more than double that of 11.04, the shut-down time is also.

So, what I'd like to see is:
- improved resource usage (come on, even Windows are going to allegedly shrink to fit onto netbooks and such),
- a less choppy user interface. My graphics card is not the newest (nVidia GeForce 8800) but it works fine on some much better looking 3D graphics applications with more shading and certainly a lot more bang than Unity, so whatever is slowing things down should be sorted out, I think,
- the option to easily add icons on the desktop (preferably with the right click, because if you don't only use 10 apps the silly Dock just doesn't cut it),
- if it doesn't make Ubuntu grow even further in the form of resource requirement, perhaps you could consider some form of widgets, to stay in touch with the new trendy looking OSs.
