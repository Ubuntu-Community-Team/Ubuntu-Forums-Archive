### Post by docus on 2010-02-10
I've tried:

Evolution ... very good overall - fast, stable, nice calendar, very customisable, good Gnome integration, surprisingly light on resources.  HOWEVER... it can't render all HTML messages perfectly.

Sylpheed ... fast, lightweight, customisable - but poor HTML support, and a few other small niggles.

Claws Mail ... very good, VERY customisable, lots of plugins, lightweight... but so-so HTML support.

Balsa ... a nice surprise - simple, light, clean interface - but a bit slow.  Previous versions were quite buggy, but the latest version (in the Arch repos) is stable for me.  One to watch?

Thunderbird ... works perfectly - the fastest of them all, very stable, perfect HTML support, good calendar (with Lightning).

Spicebird ... based on Thunderbird, so it's fast, but I found it a little buggy.  It has a front page which you can upload Google applets to, i.e. weather, news, etc.  A bit gimmicky tbh.  So far as I know it's not being actively developed.  I can't see any reason to use this ahead of Thunderbird.

Opera Mail ... I like a discrete mail client, so this wasn't for me.  But it was very solid overall - fast and stable.  A great option for an Opera user.  (I use Midori :/ )

-----

Overall: Thunderbird is the best IMO.  Evolution is a great client too, but I wish it could render HTML as well as Thunderbird.  Claws Mail is probably my third favourite; it's a fork of Sylpheed, and the two are very similar, but Claws is more customsable than Sylpheed and has better plugins.  Balsa is one to keep an eye out for in future, if they can just speed it up a bit (on my Gmail imap setup it took a lot longer than the others to load the headers on a big folder with 1000+ emails in it).
