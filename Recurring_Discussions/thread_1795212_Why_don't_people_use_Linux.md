Here's a tip to help out with 'misconfiguration' of you system:

1. When doing the OS install, make a username that will be used as a "System Administrator"
   a. You call call it anything you like
   b. example "sadmin" or "nadmin"
   c. Avoid using "admin", "adm", "sys", etc...

2. After the install goto *System > Administration > Users and Groups*
   a. Change your "System Administrator" account from *Custom* to [I]System Administrator
   [/I]b. Update your system and install all of your favorite programs using this account
   c. Remove any services or programs that you will never use
   d. Reboot

3. Create a new personal user account for yourself (using a variation of Step 2)
   a. Make your new user account a *Desktop User *(e.g. yourname, yourinititials)
   b. Set your personal settings for the new *Desktop User* you've created
   c. Reboot

4. Login as the *Desktop User* account.
   a. If you try to make system changes under this account, you will require [B]sudo
   [/B]b. However, the *Desktop User *account is not in the **sudo**ers list
   c. You will not be able to make any critical changes to your system under this account

5. If you need to make system changes, you can do one of the following:
   a. Login using the (original) *System Administrator* account via the login screen
   or
   b. Login using your *Desktop User* account and open a virtual terminal.
      1. Use the switch user command to switch to your *System Administrator* account
         $ [B]su *admin_login*
      [/B]2. You will be prompted for the *System Administrator *password
      3. Now you will be able to administer super-user privileges via your [I]Desktop User account

[/I][B]This may sound like alot of work, but it pays off in the end. It is not unwise to create certain account just for certain programs involving network communications.

Also, keep in mind that Windows XP-Vista-7 Professional Edition uses a THE SAME IDEOLOGY. The only difference is that in Linux you are much more in control of your system, and with less risk of having an application exploited for admin privileges.
