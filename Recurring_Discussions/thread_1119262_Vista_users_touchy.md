### Post by bcschmerker on 2009-04-07
For what it's worth, I'm a bit touchy about ALL operating systems, ha.  It's easier to find accurate information about building new applications in LinUX than in the competition; but commercial software developers with monster product lines favor well-recognized OS platforms.  I like to build my own LinUX boxes, and my Everex mid-tower is one such build; originally packing an all-VIA Technologies rig arrayed around a 1.5 GHz C7-D (which, I found, could barely keep up with a slower Intel RK80546PE-series Pentium 4), it now has a 2.9 GHz AMD Athlon X2 and nVIDIA GeForce6100/nFORCE 405 chipset (I ended up using the VESA 2D video driver, as I had uncommanded shutdowns sans disc-volume unmounts with the nVIDIA 3D driver).

My eMachines/Gateway T3882, which packs an all-Intel chipset based on an RK80546RE Celeron D, is coming due for replacement; it ran well enough under Microsoft Windows XP Service Pack 3 (which I needed for commercial software), but Vista proved resource-hungrier than XP and I'll probably end up building a Vista box around an Intel Core i7-based system unit with Cyclades PCI-Express x1 multiple-EIA232D (and single IEEE 1284?) card, to-be-determined PCI-E x16 dual-VDA, and probably Adaptec PCI-E x8 SASI or SATA-RAID host adapter.  Much to my chagrin, Microsoft persists in keeping user documents and application data in the root volume (as C:\Documents and Settings), a serious vulnerability in the event of a primary-hard-drive crash, whereas LinUX can branch them off (as /home) to a separate hard disc assembly.

> **Giant Speck said:**
> But what about users (such as myself) who use both Linux *and *Windows and don't just use Windows for games, but because they actually still like it?As I noted on [Post 15 of this thread](http://ubuntuforums.org/showthread.php?p=7033307#post7033307), I work with two OS's, and each has specific strengths.  Commercial vendors favor commercial OS's such as [Microsoft Windows 2000 Professional/XP/Server 2003/Vista/Home Server/Server 2008](http://www.windows.com/) (my eMachines/Gateway T3882, running XP Service Pack 3, is coming due for replacement and I'll probably build an Intel Core i7-based Vista box), and almost all hardware manufacturers support their products therein; the primary downside, I found, is Microsoft's poor handling of what goes where on the software side, with no provision to put different primary subdirectories (e.g., C:\Documents and Settings (cf. /home, /var), C:\Program Files (cf. /usr), et al.) on separate hard discs even under NT File System 5--a serious liability in the event of a hard-drive crash.

I first cut my LinUX teeth on a system I built around an [Asus SP97-V](http://www.asus.com/) (CPU:  200 MHz [Intel A80503-200 Pentium Processor with MMX Technology](http://www.intel.com/)) in the 1990's, running [Red Hat LinUX 5](http://www.redhat.com/) (last known Kernel: 2.2.38); with one large (for the era) hard drive, I set up multiple partitions, each with a specific primary subdirectory (/boot in /dev/hda1; /, /usr, /home, /var, /opt, et al. across logical extensions of /dev/hda2) assigned to it.  It proved a rehearsal for my now-nVIDIA-equipped Everex gPC, which packs an [Elitegroup GeForce6100SM-M V1.0](http://www.ecsusa.com/) and 2.9 GHz [Advanced Micro Devices Athlon X2](http://www.amd.com/) (in for an all-[VIA Technologies](http:/www.via.com.tw/) motherboard with 1.5 GHz C7-D), under Ubuntu 8.04 LTS (LinUX Kernel 2.6.24-23.52rt as of this post).  LinUX has a device-handling system that allows me multiple ext3 or ext4 volumes across multiple hard drives for routinely-used subdirectories (a safety factor in the event of hard-drive crashes), plus a modular kernel that supports commercial filesystems on media mounts (e.g., Microsoft FAT12 on disquette, Microsoft VFAT/FAT32 on USB memory plugs, Microsoft NTFS and/or Apple HFS on e-SATA, ISO 9660 on optical).  Ubuntu puts media mounts in the logical subdirectory /media.
