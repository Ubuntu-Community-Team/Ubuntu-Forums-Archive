### Post by Macskeeball on 2010-08-10
1. Would you say that your reasons are more philosophical or more practical? If both, which of the two do you feel is more important? 
They're definitely much more practical than philosophical. For me, it's about using the right tool for the job.

2. What do you like about the Linux philosophy? Specifically, what do you like about the Ubuntu philosophy?
For Linux/Unix, I like the idea of everyone building on top of the work of others and combining things in interesting ways. For Ubuntu, I like the effort to make this stuff easy to use ("Linux for human beings").

3.  What are your favorite things about how linux works? What are some of your difficult moments and what makes the overall experience worth it?
I like how it extends the usefulness of hardware, such as running well on old computers or adding functionality to my Linksys router. I also like Live CDs and not having to micromanage updates to applicationst. My two most difficult moments would be a failed Yellow Dog Linux install and configuring the sudoers file on Debian, which doesn't use sudo by default.

What makes it worth it? Tomato added some nice features to my Linksys WRT54G, including the ability to run my own custom shell scripts. I used that ability to effectively add my own feature: ad-blocking on the router (and thus for the entire LAN) with automatic updating of the ad server list and with zero configuration on the client-side. Also, my neighbor gave me his old 800MHz PIII Dell. I upgraded it with some spare parts I had lying around and bought a gigabit ethernet card, and after spending only US$32 (and only on the ethernet card) I have a machine that is sharing my USB printer on the LAN, hosting web content, acting as a NAS, and helping medical scientists in their research to cure cancer, Alzheimer's, and other diseases (Folding@Home).

4. Do you have any special reasons, moral or practical, for not using the primary alternatives (Windows, Mac, etc)?
Actually I do use Mac OS X and Windows. I've used Mac OS the most (since 1996) but am trying to become more platform agnostic, so I have Ubuntu and Windows 7 in virtual machines on my Mac. Originally I started messing with Linux to prepare myself for a plan B in case I stopped liking the Mac. Now, I'lm learning all three major OSes because I I want to become a good computer teacher or tech journalist.

On the computers that I've run strictly Linux on, its because they've been older hardware and Linux is more suited to the job.

5. What is your favorite linux distro and DE? What do you like about it specifically?
For the desktop, Ubuntu (Gnome) or Xubuntu (XFCE). Easy to use and XFCE makes *buntu run well on old hardware. For servers, like Debian (no GUI, pure CLI) because it's so well tested and reliable and also uses apt. Once I set up my server with Debian, it just sits there and works so I don't have to mess with it. It's an old machine but its been my most reliable computer ever- flawless.
