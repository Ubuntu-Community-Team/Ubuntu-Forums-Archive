### Post by Paqman on 2011-12-17
Ubuntu will just keep trucking along in it's little niche for the foreseeable future IMO. The big changes in the way people use technology are made by major players like Google, Apple and Amazon. Ubuntu will adapt to whatever landscape these greater forces map out, and will continue to be a popular minority choice for computing on x86 and increasingly ARM devices.

I know Mark Shuttleworth has been making noises about Ubuntu shipping on all sorts of digital devices lately, but TBH making that happen on a major scale would require more clout with OEMs than I think Canonical is able to wield. It would only happen if Ubuntu partnered with one of the big boys to ship a line of products. That could happen, but I'm not holding my breath.

Home servers and server-like machines are going to become much more ubiquitous than they are now, and I think that's a huge opportunity for Ubuntu to make some actual money, even if the desktop isn't setting the world on fire.

### Post by Dangertux on 2011-12-18
I agree with haqking here. If you look at the side by side features of each OS (I'm talking current versions,  stop comparing Ubuntu 10.04 LTS to Windows XP SP0, that's silly). They add up pretty equally, let's run down the list of actual features for a moment.


Windows : Data Execution Prevention, this is a measure to keep a payload from being pushed into executable memory from a compromised application or service

Linux : Has the NoeXec bit (Abbreviated NX) this fucntions identically to Windows DEP in most cases.

Both can be bypassed by injecting payload into the memory addresses of non protected dynamically linked libraries.

Windows : ASLR , Windows uses Address Space Layout Randomization to prevent executable code from running, by randomizing return addresses.

Linux : ASLR in the form of /proc/sys/kernel/randomize_va_space, it does the same thing.

Both can be bypassed by heap spraying and utilizing return oriented programming.

Windows : Supports full drive encryption

Linux : Does too.

Not really easily bypassed , but it can be done through various physical memory forensics techniques, results are still iffy.

Windows : Has an integrated kernel level firewall

Linux : does too  (netfilter/iptables)

Firewalls are a measure for mitigation. Not necessarily a preventive measure, they are easily bypassed.

Windows : Supports use of limited user permissions through UAC

Linux : Does too through the use of sudo

Most malware does not need root permissions to accomplish it's mission. Even still both UAC and sudo are trivial to bypass by escalating privileges through a vulnerable service or application running on the victim system. Or more traditionally by keylogging.

Windows : Has an array of anti-malware solutions, some are better than others. 

Linux : Has no decent anti-malware solutions.

AV/M can be bypassed by randomizing payloads, or changing executable or encoder stubs.

Windows : Supports automated patch management.

Linux : Does as well.

Updated systems are always critical to mitigating potential risk.

Windows : Integrated Mandatory Access controls via MIC in Windows Vista, it is alive and well today in Windows 7 

Linux : Has a variety of alternatives for MAC, including Apparmor and SELinux.

MAC can be bypassed, though it's very targeted, and is not guaranteed to work. These provide the best protection for a known vulnerable service, or for 0 day exploit code. The alternative is always having a fully patched system that is never vulnerable (that condition doesn't exist).

So there you have it, the security features of each group of OS'es side by side. They look pretty similar to me. As far as security by obscurity, the only group of people linux is obscure to is new linux users. Malware developers are neither afraid or unaware of its existence.

Hope this helps.
