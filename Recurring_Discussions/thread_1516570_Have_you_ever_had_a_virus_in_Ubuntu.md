### Post by dtfinch on 2010-06-24
The last virus I got was over a decade ago after letting someone else check their email, though I can't really fault them because my Outlook Express was unpatched and they only had to view an infected (but otherwise authentic) email to get it.

[http://www.symantec.com/security_response/writeup.jsp?docid=2000-121908-3951-99](http://www.symantec.com/security_response/writeup.jsp?docid=2000-121908-3951-99)

On my own Windows systems, I generally don't run antivirus software except for a full scan ever year or so, which never finds anything. I ran Microsoft Security Essentials for a couple months this year, but it made things painfully slow, despite all the glowing reviews from people who are used to worse.

My last close call I can think of on Windows (late 2008 or early 2009) was a pdf exploit attempt via a banner ad on a regular website, which only failed because I had javascript disabled in Reader. I was using Chrome which at the time had no mechanism to disable unwanted plugins. And Adobe Reader 9 had changed the behavior of unchecking "display pdf in browser" so that rather than disabling the browser plugin (as 8 did), it left the plugin enabled, but opened pdfs in a new window. Another factor was that Adobe's Reader download page almost always has an outdated, vulnerable version, and they only serve the latest patched versions through Adobe Update, which I had disabled because I don't like having tons of third party software running in the background all the time doing things without asking.

But yeah, never encountered any malware on Linux.

### Post by Chronon on 2010-06-24
> **Revolutionary101 said:**
> Yeah it has them. This is an article about one. 

[http://www.pcworld.com/businesscenter/article/198686/linux_trojan_raises_malware_concerns.html](http://www.pcworld.com/businesscenter/article/198686/linux_trojan_raises_malware_concerns.html)

That isn't a virus.  It doesn't replicate, so it fails the condition for being a virus.  

Any system can have malware written for it.  A huge part of the security in Ubuntu comes from using trusted repositories in which each binary is digitally signed, preventing tampering by any 3rd parties.  The case you linked to resulted from a tampered tarball getting swapped out without anyone's knowledge (because digital signatures were not used).
