### Post by XubuRoxMySox on 2010-11-02
I was a Xubuntu fan until version 9.04 appeared, now shipping with PulseAudio (even though none of the default apps depend on that troublesome software). PulseAudio is a deal breaker for me. A dancer has to have sound! And removing it was a bigger pain than it was supposed to be. I was reeeeeally ticked off that they chose a LTS release to introduce new and buggy Beta software, too. So I started looking for a new distro, when I heard that Linux Mint 9 Xfce (which is based on Xubuntu 9.04) does *not* ship with PulseAudio! Yaaaay!  :D

So I can have my LTS Xubuntu-like wonderfulness after all. **That is the *only* reason** that I switched to Linux Mint. Especially after [this]("http://www.extremetech.com/article2/0,2845,2346637,00.asp").

Now that I've been using it for awhile, I've discovered a few other way-kewl things about it:

The Mint Updater allows me to ignore "dangerous" updates. I can select "Levels 1 and 2 updates only" or take any or all. It's a nice li'l "safety net" for non-geeky kids. It makes it much less likely that an update will b0rk a perfectly good working system. When you open Synaptic in Mint you see the same kinda thing. The "Mark All Upgrades" button isn't there! 

For Linux gurus, that's pro'lly a pain. I've heard people describe it like, "they've crippled Synaptic," but it *is* still possible to upgrade everything if you know what you're doing and can fix stuff that breaks in an update. But Mint is for geek-challenged kids like me who prefer to run applications instead of running the OS. The LTS version with only "Levels 1 and 2 updates" enabled is rock-stable and pro'lly will stay that way.

-Robin

### Post by TBABill on 2010-11-02
I had a tendency to like Mint more when I started with Linux because I didn't always understand things in Ubuntu and I tried to go it alone till I got into a jam, then I hit the forums. Now I seem to like Ubuntu better because of Ubuntu's Gnome's looks....call me crazy. And I am much more comfortable with the way the system works from a perspective, not from a look. Mint did simplify things and helped me over the hurdle of getting comfortable with the difference of Linux versus another OS and I do like it still.
 
I guess for me Ubuntu just seems to be "old reliable" in terms of where I am most comfortable operating and it has the look and functionality I most appreciate. I actually like the Gnome menus over the Mint Menu, a point where most would disagree. Either system works well, but Ubuntu makes it more "Folder>>>sub folder>>>sub sub folder" to me. The Mint menu seems to mix the KDE or Windows style with the single button that expands to what you select with the Gnome style. It's difficult to describe because in the end it is just personal preference, but for me it is a strong reason why I feel more comfortable in one over the other. And that comfort is not in my ability to use it, but what puts me more at peace while I use it...strange way to say it but conveying it is not easy.
 
Why people break down the comparison to one is easy and the other is techy...different perspectives due to different skill sets, experiences, knowledge, etc. I don't see understanding my OS as techie because my understanding Windows doesn't make me techie. I do think, from my own experience with both, that Mint made the hurdle of learning and gaining an understanding of Linux a little easier. Caring for a users needs to a higher degree up front is always easier for a new person than getting a pretty good system that then needs things added, regardless if it is easy or not. By placing all the tools at the users hands just makes it easier than getting close to a fully functioning system and having to seek out more answers to get it to see a video or hear sound from a device. Not a big deal by any stretch, but some users want more usefulness right at install and with Mint I think they get that. PCLinuxOS does that as well. Caring for the new user is wonderful so long as it does not cross a line with that OS's philosophy, and for Ubuntu adding all those things does do that so they leave it to the user to add them post install (didn't 10.10 add that option to the install instead of forcing additional work later? I did upgrades when I used it so I am unsure).
 
Anyway, that's my perspective on why you may get different answers on very similar distros. There are more reasons, but I understand the users who post for or against one or the other based on useability right after install. For me I just add all the codecs, java, flash, etc., but expecting a new user to just know to do it and HOW to do it is a stretch when some users are just not as adept at computer usage in general as others.
