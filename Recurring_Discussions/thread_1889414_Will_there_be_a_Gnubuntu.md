### Post by grahammechanical on 2011-12-01
We already have one. Ubuntu is built on what some called Gnu/Linux and what others simply call Linux. So, we already have a Gnu/Linux/Ubuntu.

You are talking about a Gnobuntu. A Ubuntu minus Gnome. But Ubuntu is built upon the Gnome desktop environment. That is not predicted to change.

If you want it to remain easy to replace the Unity user interface with Gnome shell user interface then you must put pressure on the Gnome developers to be more cooperative with the Ubuntu developers.

Canonical has the responsibility to make Unity work on top of Gnome. It does not have the responsibility to make Gnome shell work with Ubuntu/Unity. At the moment Canonical is taking responsibility for that work.

I recently read a transcript of an irc session with Mark Shuttleworth and he makes an interesting comment about Ubuntu supporting Gnome shell but I cannot now find the transcript to link you to it.

### Post by lykwydchykyn on 2011-12-01
Kubuntu, Xubuntu, and Lubuntu all began as independent community-run projects, and while they are officially "recognized" by Ubuntu/Canonical, they are still community projects.

The reason these desktops are easy to install and readily available on Ubuntu are due in large part to the efforts of those communities.

So what I'm getting at is that, if Ubuntu decides not to keep Gnome easily installable at some point, it may be necessary for some independent community of developers to step up and create a "Gnobuntu" project to provide a focal point for packaging vanilla Gnome on top of ubuntu.

Until then, there probably won't be such a project.
