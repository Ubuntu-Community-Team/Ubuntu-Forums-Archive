### Post by sanderd17 on 2011-05-10
Mint and Ubuntu are almost equal in ease of use. But the default settings of mint (window list, menu ...) are more windows like and the default settings of Ubuntu are more mac-like (global menu, close button on the left ...). You can transform any ubuntu installation into a mint installation or the other way around by just changing some settings and some applications.

So it really doesn't matter that much. 

If you go for one of the two, I would take the gnome version. Both have a KDE version, an XFCE version (they are called Kubuntu or Xubuntu for the ubuntu derivations and Mint KDE or Mint XFCE for the mint derivations). But the gnome version is of both their "flagship" and thus the best maintained. 

If you go for opensuse, that has no real flagship and all versions are maintained equally well.

### Post by I2k4 on 2011-05-10
I'll echo Mikewhatever above, and suggest you don't install anything on your **hard drive** until you've experimented with Persistent Live USB. There are big differences between even Ubuntu versions in user interface, performance on hardware, and the amount of stuff that comes preinstalled.  I'm not a believer in repeatedly mucking around with complicated hard drive installations unless/until I'm absolutely sure what I'm doing, and Live USB is risk free to your existing installation.

The nicely illustrated "Show Me How" here is bullet-proof for a Live USB:

[http://www.ubuntu.com/download/ubuntu/download](http://www.ubuntu.com/download/ubuntu/download)

The "Persistent" feature is easy to see on the PenDrive installer and is set at the end of Step 2.  I suggest 4GB USB drive with about 2.5GB set as "Persistence" on the installer.

As to versions, I learned a lot from Ubuntu 10.10 Desktop, found Xubuntu didn't recognize my netbook hard drive from the Live USB, so dropped it.  I dislike the Unity interface but certainly suggest you try it out on 11.04.  

I don't want most of the preinstalled software for my purposes, so have been running the very light and fast Lubuntu 10.10 happily for a couple of months, and still learning.  With all the enthusiasm for Mint above, I'm likely to stick it on a drive and give it a spin.  It took me a couple of years to get to know MS, and I've got time for Ubuntu.

### Post by beew on 2011-05-10
Many people suggest live CD but it is absolutely useless for test driving Ubuntu (or any distro for that matter). It is slow, clunky and the worse part is you cannot save any setting or datum so how are you going to actually use it? 

At the least you would need to create a live usb with persistence. For those who are using Windows an excellent tool would be LiLi.[ http://www.linuxliveusb.com/ ]("http://www.linuxliveusb.com/")

The drawbacks of live usb with persistence are that it is rather slow (but still a lot faster than CD) , it doesn't support things like hibernaton,  it doesn't use real Linux file systems, it doesn't support system level upgrades (like kernel updates) and obviously, it is very limited in size. But you can install programs and store data and settings without losing them on reboot. The speed may improve if you have usb3. I don't know whether you can install proprietary graphic card in a live usb or not. It used to break mine for 10.04 and 10.10 but I heard it is possible with 11.04.

But by far the best way to test out a distro is to create a full install in an external hard drive. That way you get the real performance and are free to do whatever you can do with an internal install because it is in every sense a real install.
