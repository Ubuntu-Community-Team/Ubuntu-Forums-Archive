### Post by Shining Arcanine on 2010-05-30
> **mosshorn said:**
> I'm currently planning on putting Arch on my laptop, but I was wondering exactly how difficult IS it to configure things? I've been an active Ubuntu user for around 4 years now, and although I know my way around common terminal tasks and whatnot, I'm not an expert by any means. 

Is Arch too advanced for a mildly linux savvy user, or a useful learning distro? Discuss and thank you :)

Personally, I would use Gentoo Linux instead of Arch Linux, although there seem to be some people who like Arch Linux. Because it is a binary distribution, it is not possible to learn as much from Arch Linux or customize Arch Linux as much as you can with Gentoo Linux, but you could still make it work.

Keep in mind that you will be almost as entirely dependent on other people to fix things with Arch Linux as you would be with Windows, because the procedure required to apply patches to packages will require you to go through the same process the package maintainer does and there are many more obstacles to applying patches with a binary distribution than there are with a source based distribution, not the least of which being that the source code is not provided with the system.

If you decide to try Gentoo Linux in place of Arch Linux, you would want to read the Handbook and also ask questions on the forums, most of which would likely send you to pages documenting various things that are not immediately obvious.

[http://www.gentoo.org/doc/en/handbook/](http://www.gentoo.org/doc/en/handbook/)

On the Gentoo Linux forums, I recently directed a Ubuntu user who is considering switching to Gentoo to the following resources, which were needed in addition to the handbook's installation instructions to get a fully fledged desktop environment running on his system.

[http://www.gentoo.org/doc/en/xorg-config.xml](http://www.gentoo.org/doc/en/xorg-config.xml) 
[http://www.gentoo.org/doc/en/gnome-config.xml](http://www.gentoo.org/doc/en/gnome-config.xml) 
[http://www.gentoo.org/doc/en/nvidia-guide.xml](http://www.gentoo.org/doc/en/nvidia-guide.xml) 
[http://www.gentoo.org/doc/en/handbook/handbook-x86.xml?part=4&chap=4](http://www.gentoo.org/doc/en/handbook/handbook-x86.xml?part=4&chap=4) 
[http://www.gentoo.org/doc/en/alsa-guide.xml](http://www.gentoo.org/doc/en/alsa-guide.xml)

Although it is not immediately obvious to new Gentoo Linux users, all of Gentoo Linux's official documentation is listed on a single webpage:

[http://www.gentoo.org/doc/en/list.xml?desc=1](http://www.gentoo.org/doc/en/list.xml?desc=1)

There is also an unofficial wiki which supplements it:

[http://en.gentoo-wiki.com/wiki/Main_Page](http://en.gentoo-wiki.com/wiki/Main_Page)

If you decide to go with Arch Linux, the only documentation you will find is on their Wiki, which is either a good or a bad thing, depending on how you look at it:

[http://wiki.archlinux.org/index.php/Main_Page](http://wiki.archlinux.org/index.php/Main_Page)

As far as I can tell, installing Arch Linux takes roughly the same amount of effort as installing Gentoo Linux, although you do have the benefit of not having to wait for packages to compile, at the expense of not having as many options as you would if you had to wait for packages to compile.

You do not get the full range of customization with Arch Linux because some customization can only be done at the source level with options that must be provided to the build system. Gentoo Linux provides USE flags, which can be used to turn various dependencies on and off at a whim, among other things. > **Dark Aspect said:**

I cannot consider that to be funny when it is true. Users of Gentoo Linux, or more specifically, users of Gentoo Linux's testing tree, are usually on the front lines of new code from upstream. They often try out the latest software from overlays, even though it is not considered stable, which makes them assume the role of being the beta testers for the rest of the Linux world.

I am not sure if you noticed this, but upstream bug trackers seem to have a disproportionately high number of legitimate bugs reported by users of Gentoo Linux. The bugs are then fixed by upstream, after which point, users of other distributions such as Ubuntu Linux then encounter the very same bugs because their distributions run on static release schedules, such that when they receive the latest software, it already had several months of quality assurance testing during which time issues were identified that were then patched in newer versions of the software that were not included in their distributions. Users of static distributions then proceed to suffer from already fixed bugs for years.

Arch Linux is an in-between kind of distribution, because it is a  rolling distribution like many source-based distributions, but it is also a binary distribution like many static distributions. It is possible to have source-based static distributions, but for obvious reasons (precompiling software is easier when the software does not change) and with few exceptions, they do not exist. The closest thing to a static source-based distribution of which I am aware is a fork of Gentoo Linux for the Sheeva Plug, which uses a snapshot of the portage tree:

[http://plugcomputer.org/plugwiki/index.php/Gentoo_20090217_Plug_Computer_Distribution](http://plugcomputer.org/plugwiki/index.php/Gentoo_20090217_Plug_Computer_Distribution)

Anyway, since Arch Linux is a rolling distribution, it does not suffer from already-fixed bugs for years like other binary distributions do, although being binary does come with drawbacks (and advantages) as I mentioned in my other posts.

### Post by handy on 2010-05-30
To the OP: 

It very easy to see the chauvinists in any thread, especially when it comes to talk on the use of other distros in the Ubuntu forums. 

Naturally there are the team players here that feel that anyone finding something better than Ubuntu/or some other distro, has taken something away from them personally, they fight against this so as not to feel devalued. Thankfully they are only a small minority in this great forum community.

A personal query regarding a different distro/BSD/other system, can quickly get turned into a forum for an argument on which is the overall best system. Which is a complete perversion of the founding question on which the thread was supposed to be based. In this case the OP basically asked will I be able to handle the installation & maintenance of Arch with my level of knowledge?

It is great to try different distros, it is educational & some of us find it fun too. During the hopping some of us discover which distro/system suits our particular needs & taste best of all & stay with that distro, perhaps having a look at other new distro/system developments from time to time.

Sometimes I wonder why some people think so many distros exist?

As far as the Gentoo vs Arch installation is concerned, having used both of them, Arch is much quicker & easier to install, & also quicker & easier to maintain. The dyed in the wool Gentooer that hasn't tried Arch will argue that isn't true, but they are having themselves on & are blinded by their loyalty to their chosen distro.

I distro hopped for some time, tried various BSDs, Solaris, Nexenta, AROS, Haiku (those the last two are still working towards their first release) & found that Arch suits me. I've been using it for nearly 2.5 years on my prime desktop, (still that same 2.5 year old install which I upgrade on a daily basis). My No.2 box has had various distros & Haiku on it, it currently uses both Haiku & primarily Arch.

If you choose to install Arch, follow the Beginners Guide to the letter. Don't use one of the Arch based distros that install Arch for you the easy way. They are great, but not for a first installation. During your first install, you will learn how Arch does things. Arch does things differently in its effort to find a more simple way to make a Linux kernel based system work.

You have to learn to edit configuration files. Which becomes 2nd nature after a while & because there are so few of them that you have to deal with, you learn to know where to go to do whatever, if/when the need arises.

The Arch documentation on their website is absolutely superb as is Gentoo's for that matter. It really makes doing things so easy as you just follow the steps. The Beginners Guide is on the installation CD, so learn how to access it if you don't have a 2nd computer that you can be referring to whilst you are installing, which in my view is the ideal way to do it.

I love the rolling release system, it has caused me to upgrade into trouble 3 or 4 times in 2.5 years. Getting out of trouble is really easy, once you know how. :)

I'll put a link below that has some highly useful Arch related how-tos that I wrote. These are basically essential knowledge for anyone who uses Arch, & I really wish I had of found this around the same time I started using Arch, instead of later on as the need arose. [http://spiralinear.org/index.php?board=9.0](http://spiralinear.org/index.php?board=9.0)

If you do choose to install Arch, have fun? I loved installing it, it was a great adventure into the new, the documentation just made it so easy. :D

The other thing I just thought of, is what happens with an Arch install: Because it stays on your machine until either you remove it or you have drive failure, it is continually over time being tailored to your taste/needs. As you learn more about the way it works you refine & optimise it. By the time 18 months has gone by you have the most comfortable system that you can imagine, without any extras that you don't need, as only what you install exists on your system. What is on your Arch system will be accessed just the way you want to do it; not the way some distro making team decided is the best way for you to do it.

I really like this level of creativity that allows for the individual to make their own personalised system just the way they like it. =D>

Others don't like it that way one little bit.
