### Post by aysiu on 2009-07-31
> **Johnecash said:**
> I have been trying - I really have for 3 days - this is my 3rd time trying linux (2003 and 2005) and it just doesn't work.
 
does linux work for you - what things can you actually do with it? What does "trying" mean in this scenario?

Does it mean you were using Windows full-time until three days ago, and then three days ago you just popped in the Ubuntu CD and expected everything to work?

If so, then I would suggest you take on another approach. Some people (who just happen to have Linux-friendly hardware and who do not use Windows-only software), the migration to Linux can be just as easy as that (pop in the CD, start using it).

For others (like you), it's probably best to take a slower approach:

1. Assess your software needs first. Do you use any Windows-only or Windows-and-Mac-only programs (certain graphics-intensive games, AutoCAD, Adobe CS3, OneNote)? Ask yourself if you absolutely need these programs in particular or if you can use alternatives that may not have the exact same functionality. Start off by using [open source software in Windows](http://www.opensourcewindows.org) (OpenOffice, Firefox, Thunderbird, GIMP, InkScape, Scribus, Pidgin, FileZilla, VLC). If you find you can rely primarily on open source and cross-platform software in Windows, the software end of things is taken care of.

2. Then install Ubuntu in a non-committal setting ([Wubi dual-boot](http://www.psychocats.net/ubuntu/wubi) or [VirtualBox inside Windows](http://www.psychocats.net/ubuntu/virtualbox)). This is just so you start getting used to the Gnome interface, the Linux way of installing programs, the filesystem hierarchy. The goal of this step is to just get comfortable with navigating your way around in Ubuntu specifically and somewhat in Linux generally.

3. If you're pretty sure Linux can satisfy your software needs and you are starting to feel more comfortable in it, pop in a live CD and see which of your hardware components are recognized or not recognized (wireless card, optimum screen resolution, scanner or printer, portable media player, etc.). Do an inventory. Is it a total disaster? Are there four or five Linux-unfriendly hardware components you have? Well, just take stock of that for the future, and keep using Windows in the meantime. When your iPhone dies, consider replacing it with an Android-based phone. When your Lexmark printer dies, consider replacing it with an HP printer. When your computer gets "too old," consider replacing it with a Linux-preinstalled computer... or building your own computer with Linux-friendly parts.

Some of us are lucky enough to not need Windows-only software and to already have Linux-friendly hardware.

For the rest, a slower migration is best if a migration is appropriate at all.

More details here:
[A home user&#8217;s successful migration strategy from Windows to Ubuntu](http://www.psychocats.net/ubuntucat/a-home-users-successful-migration-strategy-from-windows-to-ubuntu-2/)
