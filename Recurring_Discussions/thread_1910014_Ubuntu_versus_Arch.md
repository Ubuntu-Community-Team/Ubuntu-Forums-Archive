### Post by ratcheer on 2012-01-16
I am currently running both Ubuntu 11.10 and Arch on my PC. Arch has a much smaller memory footprint, but I do not believe it is significantly faster at running tasks. It does, however, boot up and shut down faster than Ubuntu.

I run the OpenBox desktop on Arch and gnome-shell (usually) on Ubuntu. Arch is purposely designed to be more minimalist, so after a default installation, you mostly have to add whatever apps you want. To me, that is neither a plus nor a minus, it is just different.

As cutting edge as everyone says Ubuntu is, Arch is much more so. My arch is currently on kernel 3.1.9-2, and the kernel is upgraded every few days, it seems. I believe it will be on 3.2 before very long.

Ubuntu Forums is much more polite and helpful than Arch Forums. They do warn you when you sign up to research things thoroughly before submitting your problems, and they will bite you pretty severely if they don't think you have done so. So far, I have gotten along pretty well with them, and I have asked a few questions there.

Here is a puzzler for you: I think I prefer Arch, but my main system that I use about 95% of the time is Ubuntu. It is just so much easier to install and configure things on Ubuntu. OTOH, I have been using Ubuntu for about three years, but Arch for only about 4 months, so I am still getting used to Arch. Maybe someday Arch will be my preferred system.

Tim

### Post by terry_gardener on 2012-01-16
the advantages of arch is that it is rolling release and therefore you have all the latest packages within couple of weeks after release usually within couple of days and the OS is always up to date. however because of this make sure you check the forums and the announcements before updating because nwo and again there is additional actions you have to take to get the packages to install properly. 

when you have setup the system which takes a long time (for example all services and software you need) it is stable and slightly faster than ubuntu but i don't think it is a big amount, 

what i found is that i would setup arch and then couple of weeks/months i would need to do something that i have not done yet on the system and therefore i would need to setup and install the service, like cups and lirc for example. 

basically if you want a tailor made distro that you don't mind setting everything up to your liking then arch might be for you, if you want a system that most services are already setup and minimal config then i would say stick to ubuntu.

### Post by Double.J on 2012-01-16
I think the title is a bit misleading...

If you ask most people what they love about Arch, they'll say configurability first and rolling release second... I wouldn't say this applies to Chakra which has been forked from arch - it doesn't dump you in a base system to build up - it focuses on the KDE Software Compilation and has a semi-rolling principle - set updates for the base system, but only updating KDE once new versions are sent downstream

It does agree with the KISS principle, but achieves this in a different way to Arch.

With reference to Ubuntu, I've only tried it once, and can't say I had any problems with it at all - in terms of user interface, it's KDE, not heavily modified, so if you like KDE, give it a go! There's no other desktop support mind - QT all the way!

In my opinion as it comes out of the box- 
Kubuntu Vs Chakra - Chakra
Ubuntu Vs Chakra - Ubuntu

Of course, that's half the story, (k)ubuntu has a LOT more packages, especially more recent ones, Chakra promises to be more stable?

As all things, it's just personal choice :)
