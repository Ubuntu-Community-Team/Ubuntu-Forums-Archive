Only debian based distros (ubuntu, mint) use deb.
Not much compiling needs to be done in Arch - its package manager (pacman) is as good as apt.
The only difficulty with Arch is installing your system and configuring it.
A standard installation of Arch installs just the kernel, coreutils, gcc and a few other packages. You have to install X and/or a window manager yourself.
So, whereas ubuntu takes a couple of hours to install and configure, arch takes about... a weekend.
But once it's installed, you will find yourself using a system configured right to your needs and nothing more.
If you update daily, you have nothing to fear...
Besides, the arch wiki is perhaps the best linux resource out there.

### Post by handy on 2011-02-25
The Arch installation is fine if you can follow the Beginners Guide instructions closely & nothing goes wrong.

If you have a wireless network problem your installation boat will sink fast & you will then have to stuff around with a LiveCD (if you haven't another internet connected machine) to find out if there is a solution that will allow you to connect to the internet so you can proceed with the install.

If you are lucky you will find a solution; then you have to use bash to apply it so you can continue using bash to install Arch.

Someone in their early Linux days, coming from a point & click environment like Ubuntu & having to deal with an Arch installation problem will require them to have tenacity & plenty of patience & to get over any fear that they have of using the Terminal.

Such a situation does not suit everyone. Many people don't find that kind of challenge stimulating or fun.

I love Arch, it has been my distro of choice for near 3 years now. I don't recommend it to Linux beginners as there are too many things that can cause stress for them when they are still dealing with the loop the loop Linux learning curve.

Once a person has become comfortable with the Linux way, & they feel the desire to distro hop, it is great to try some other distros & the various BSD options & whatever else. All of which furthers our understanding & makes the Terminal based distros easier for our mind to manage.

Re. Arch again; I don't think I've ever had a system that is so easy to manage, (it could even be easier than the Amiga). But you still have to learn the hows & understand the whys. So you can then be happy editing configuration files when need be & managing the Arch package management issues.

If someone does head down the Arch path, I've written a few very useful how-tos which can be found by scrolling through the following linked page, each of the guides has a title that starts with [Arch]:

[http://www.spiralinear.org/forumdisplay.php?fid=17](http://www.spiralinear.org/forumdisplay.php?fid=17)
