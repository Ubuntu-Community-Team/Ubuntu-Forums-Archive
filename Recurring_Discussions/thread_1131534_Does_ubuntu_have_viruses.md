### Post by yaroto98 on 2009-04-22
Nobody explained how iptables is the best firewall out there. By default iptables will drop any and every packet that comes in on a port not tied to a program. So, unless you installed some linux virus (which really only exist in the lab) you're pretty safe. 

> I agree that every network at least should have at least one firewall, be it where the connection comes in, IE at the router, or on each individual computer in the network.

This is handy just so you don't have random script kitties throwing packets at your personal computer. However, if you quick configure your iptables to specifically drop everything on ports you're not using, you can make yourself the best bastion server that doesn't need any extra firewall bogging down the works. The really nice thing is that iptables is part of the kernel. That means one very important thing...FAST, probably the fastest, lightest-weight firewall you've ever had.

> iptables will drop any and every packet that comes in on a port not tied to a program

Are you sure?

I get the following when I iptables -L:

```
imbjr@main:~$ sudo iptables -L
[sudo] password for imbjr: 
Chain INPUT (policy ACCEPT)
target     prot opt source               destination         

Chain FORWARD (policy ACCEPT)
target     prot opt source               destination         

Chain OUTPUT (policy ACCEPT)
target     prot opt source               destination         
imbjr@main:~$ 

```

That suggests all traffic is permitted (i.e. ACCEPTed). But since there is next to nothing listening on the ports (except a printing daemon I believe) and the fact that routers with firewalls are common these days suggests one is relatively well protected.

However, never be complacent about security. As Linux popularity grows, it will become more of a target to the malware authors and if they cannot get in as they do when targeting Windows, then it will be social engineering that will do the trick and there's very little software protection for that.
