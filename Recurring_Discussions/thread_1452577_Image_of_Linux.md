### Post by Meep3D on 2010-04-12
> **red_Marvin said:**
> User friendlyness depends very much on what kind of user the software targets itself to, a steep learning curve does not mean that the software is not user friendly, but that it is not neccesarily intuitive.

Yes, it does.  Emacs is not user friendly, no matter which way you spin it.  Sure you can learn it, but it's unintuitive, non-obvious and confusing and requires a significant investment of time to be able to use it productively.  This is the very definition of 'not user friendly'.  If you subscribe to the idea of "user friendly once you've learned it" then every piece of software is user friendly and as such negate the whole concept.

If someone who has not used the software before can sit down and start using it effectively without consulting manuals and howtos then it is user friendly.  A text editor that requires a manual just to know how to quit it is not - in fact the whole point of the 'user friendly' and GUI approach is to make manuals optional.

As a general rule of thumb - if a manual is a necessity, then it's not user friendly.

### Post by Meep3D on 2010-04-12
> **red_Marvin said:**
> What I mean is that somebody who has never seen a computer before would not know how to navigate the most user friendly gui in the world without instructions either as a manual or along the way*.

Sure, everything has an initial learning curve coming from zero.  The difference is generally you can show someone how to use a GUI based system in a couple of minutes and they'll 'get' it.  Advanced skills can be taught later, but they are not necessary for basic operation - you can use Windows pretty much without a mouse if you want to but not learning the shortcuts doesn't stop you from doing the basics.

A GUI is all about learning concepts, not details.  Once you know what a dropdown box is and does you can use it anywhere, same with a save dialogue, a slider, a button.  Once you've been shown click and drag once you never need to know it again.  Learn one GUI package and you've learned them all.  With CLI tools each has it's own particular switches, parameters and foibles and just because you know one doesn't necessarily mean you can use the other with first consulting the manual.  Essentially the difference is a GUI presents the options and lets you choose, with a CLI requiring you to know all the options before you can use them.

As has been said the claim that vi would be as easy to learn as Word if only they had seen vi first is complete crap as you can summarize how to use Word in 5 seconds 'click on where you want to type, type, click and drag to select and click those buttons to apply an effect'.  vi on the other hand has a non-intuitive, and pretty byzantine list of commands - [http://www.lagmonster.org/docs/vi.html](http://www.lagmonster.org/docs/vi.html) - none of which will help when you go to learn App2, letting anyone that is used to word use Powerpoint without any real issues yet if they wanted to learn something like LaTex they'd be back to the start again.

Personally I have a poor memory and generally use a computer to get work done which is why I tend to despise the CLI - spending half an hour trying to figure out why the drive I just mounted wasn't accessible (oops, left off -umask 0666) when if there had been a GUI mount tool available (there wasn't) I could have done it in 30 seconds with no fuss.

Essentially forcing the user to re-learn from zero every time they use a new program is certainly not 'user friendly'.

### Post by aysiu on 2010-04-14
> **Meep3D said:**
> This is the very definition of 'not user friendly'.  If you subscribe to the idea of "user friendly once you've learned it" then every piece of software is user friendly and as such negate the whole concept. I don't know about the emacs debate, specifically, since I've never used emacs, but generally speaking, I think you're wrong on this count.

I've spent months trying to find the best web browser on Android. I tried out Browser, Better Browser, Dolphin, Steel, Opera, among others. I'd heard of people raving about xScope, but when I tried it out, it made no sense to me. I quickly gave up on it, thinking "What does everyone think is so great about xScope?"

Eventually, though, I gave it another try and I believe it is hands-down the best and easiest to use browser on Android. It is, however, the easiest to use only after you've read the FAQ, which gives you instructions on the interface. The controls are not intuitive at all. They are, however, a lot easier to use than the controls on other browsers.

For example, to open a new tab in Opera, you hold down the link until a context menu pops up, then you select to open in a new tab. The new tab will then come into focus. It's a similar procedure for Browser. In Steel, you hold down the link and then lift up your finger very quickly. All of these procedures are cumbersome. In xScope, you simply hold down the link until the link opens in a new tab *in the background*. Now how would you know to do that? You wouldn't. But once you know to do that, it makes browsing, especially on a slow connection, a lot easier, because you can load links in the background while you're still reading the current page you're on.

Again, not intuitive. I had to read the manual to discover how to open new links in background tabs. But once I did, a lot easier to use than the context menus, and having tabs open in the foreground.

### Post by santq on 2010-04-14
Trying to stick to my original post, I feel people are not aware why they should use Linux (or any other free OS for that matter). They wouldn't actually have to use Linux, but they should at least acknowledge what's inherently better about it than proprietary OS like Windows or OS X.

For example, my local store has brilliantly put Fair Trade oranges next to cheaper non Fair Trade oranges. Every time the customer wants to buy oranges, they are forced to make a moral choice between paying 50 cents more or being part of the extortion of the farmers. Of course if you don't for some reason believe in the principals of Fair Trade, you don't have to go for it, but it's a great way to raise people's conciousness about the subject.

So people are not aware of the benefits, nobody's talking about it in mainstream. Even Linux advocates [don't always]("http://mahesharandara.wordpress.com/2009/11/10/why-you-should-use-linux-over-windows/") bring up this aspect and tend to focus more on the pragmatical side. Maybe this is because free software is quite abstract subject, so it's hard to fully understand and get hyped up. The Tux RPG actually had quite fun story with its obvious references to Microsoft. Maybe because it went so overboard, it made the concept easy to grasp what could be possible if one party has so much control over computer technology.

This site probably has the best collection of the basic arguments: [http://www.getgnulinux.org/windows/](http://www.getgnulinux.org/windows/)
