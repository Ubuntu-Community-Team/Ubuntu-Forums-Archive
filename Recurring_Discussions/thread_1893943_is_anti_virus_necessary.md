Alright this thread again... Someone should make a sticky about this with a general accepted answer...Oh wait... ;-)

Now that, that bit of sarcasm is out of the way.  Anti-Virus and Anti-Malware are NOT for the umpteenth time a DEFENSIVE measure. They are reactionary and utilitarian in nature. 

Truth - malware is like a vampire it can't come in unless you invite it. Now this is where it gets tricky, you might not THINK you're inviting it because, you didn't click, download or otherwise install anything. However, maintaining an unpatched system, with weak credentials and no measures to protect your browser is an engraved invitation (on ANY platform). 

Truth - Anti-malware solutions on ANY platform will not stop code execution that originates in memory. So for instance, if I'm an internet evildoer and I load this post full of firefox exploit code and your browser is vulnerable. You will be compromised and I will have code execution. I can use this to do whatever I want (within the bounds of the privileges of the user running your browser) and Anti-Virus will just smile at me and say that's cool. As a matter of fact AV won't lift a finger until I touch your hard drive. 

Truth - Most AV vendors are sloppy and lazy (intentionally). Why? you ask. Because you make them that way. Too many false positives and you cancel your subscription. Heuristics based detection and signatures have become more specific than ever. For fear of telling you that your favorite application is malware. This specificity ultimately hurts the overall effectiveness because a malware developer can make a trivial change and go undetected until new definitions are released. 

Truth - On access is usually too late. Most AV vendors have reduced their access time sensitivity threshold. Why? Because you don't want it to slow down your  computer, and you need those extra 2 FPS in World of Warcraft. Face it , by the time whatever has been dropped on the hard drive is accessed, whatever was resident in memory already had time to hook your kernel and brick your system, even if AV removes the backdoor application that was dropped. So you still end up reformatting.

### Post by Telengard C64 on 2011-12-11
> **todd500 said:**
> I don't have WINE installed but do I still need an anti virus?

Depends on how you use and maintain your system.

**_Where do you get your software?_**
How much do you trust the sources of your software? 

[list]
[*]If you stick to the Ubuntu repositories for your software, then you can feel very assured that nothing you install from there will infect you with malware. It is extremely unlikely that malware would ever find its way into the standard Ubuntu repos, and I can't recall ever reading of such a thing happening.
[*]OTOH ... if you get software from random web pages, P2P, or any other source outside the Ubuntu.com domain, then you are increasing your potential exposure to malicious software.
[/list]

**_Do you run programs embedded in web pages?_**
Most web pages include Javascript, and many also include Flash and Java programs.

[list]
[*]Javascript is a program designed to run in your web browser. Most modern web browsers automatically execute Javascript. Carefully crafted malicious Javascript programs can theoretically exploit vulnerabilities in the web browser, which opens the door to all kinds of badness.
[*]Flash and Java programs require plugins to execute. These plugins are automatically loaded by your browser for web pages which have such programs embedded within them. The plugins themselves are known to have had exploitable vulnerabilities in the past. All kinds of badness is possible.
[/list]

Firefox, Chrome (and probably other browsers) have optional extensions to block Javascript, Flash, and Java. If those programs don't run, they can't do anything to you. Typically the blocker provides you a whitelist for web sites you trust. I have Ubuntu.com in my whitelist, along with other web sites I trust.

**_Do you run network accessible services?_**
Any program which responds to inbound requests from the network can be a vector of attack.

[list]
[*]Don't run the service: remove it from your system.
[*]Configure required services to run as securely as possible.
[*]Block network access to services by default, allow only what you must (firewall).
[/list]

**_Who has physical access to your computer?_**
Anyone with physical access to your computer can insert a bootable medium and do whatever they like to it.

Sorry to be so brief and vague. Computer (in)security is an enormous topic about which much can be written, and has been.

If you run Ubuntu without Wine, your risk of being impacted by a virus targeted for Windows is so low it might as well be zero. OTOH, that doesn't prevent you from passing along a virus to Windows computers, perhaps via email.

There have been viruses targeting Linux in the past, but none of them have been very successful or wide spread. As far as I know there are no active viruses targeting Linux at this time.

I do agree with KiwiNZ, in that I think anit-virus software has a place in good security. In the Linux realm though, it is a very controversial topic.

Let's try it this way, since an argument about who runs what how is completely and utterly irrelevant to the question at hand.

This is what AV does, if you want to argue that, you can google it and research it for yourself.  AV detects threats on the hard drive, it detects them at access time and when a scan is done. It detects them in two ways the first and most prominent is via signatures. This performs pattern matching to find well known malicious bits of code. Heuristics is also a type of pattern matching but is more generalized in that it matches combinations of functions in code that are generally seen in malicious software.

That being said, it does not actively scan memory.

So now we've identified what it does. Where is it's place? Layered as a remediary function with other security measures. Should you use it? You need to decide that for yourself, company or whomever you are making the decision for.
