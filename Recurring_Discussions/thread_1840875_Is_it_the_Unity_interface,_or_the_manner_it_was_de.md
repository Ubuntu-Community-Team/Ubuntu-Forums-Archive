### Post by rg4w on 2011-09-08
> **_d_ said:**
> I'm going to have to go ahead and say most likely both.
Agreed; the two are inseparable.

Guidance from a strong leader is necessary for almost any significant move forward, and of course almost any project can benefit from strong leadership in resolving tie-breakers that might hold up a project forever in analysis paralysis.

But software design cannot be driven entirely by hunches; those hunches need to be validated or refuted with rigorous user testing.

User testing is expensive, and good user testing isn't easy.  If you know many usability professionals, you've probably heard far too many stories of companies who've used user testing to make a particular executive look good, with far less if any concern for actual discovery of cognitive patterns which could improve the design.

The user testing done at Canonical appears to have been fairly good, but was sorely limited in both scope and frequency, and began far too late in the design process to be as useful if there had been, for example, tests with paper prototyping done long before the first line of code was written.

For all the money Canonical has invested in Unity, IMNSHO it would be a good invesment to hire a team like the folks at User Interface Engineering (UIE.com) to refine Unity for 12.04.

This isn't a quote, this is my analysis of the interface. I see similar interfaces on phones and they work great, however, Unity and especially Gnome-shell are dysfunctional for a desktop.

An interface should let me open/close program/windows, move windows between workspaces, switch between opened programs/windows. I should be able to do that with as little effort as possible.

By the way, you can get rid of the global menu idiocy (which is just a poor implementation of a questionable concept) by opening synaptic and uninstalling certain packages, specifically the "indicator-appmenu" package. Also, you can search for "global menu" in Synaptic and remove the Firefox plugin. After doing this, menus will again show in the windows instead of the top panel.
