### Post by 3rdalbum on 2011-11-07
> **klein de usa said:**
> thank you Jpenguin i took a look at mate looks promising! 
 
Mikeb85 if Canonical wonted to maintain gnome 2.3? they could, they have the resources and programmers to do so

They most definitely do not have the resources to maintain a whole old desktop environment, especially one that has as much cruft as ye olde Gnome 2. Unity, while only being a shell sitting on top of Gnome, doesn't have the development time and resources that it really needs.

> but there trying to keep up with win8 and apple and bid for the tablet market leaving desktop users behind

I believe Unity has been in the works since before the Metro UI was proposed for desktop Windows 8. Unity is not "leaving desktop users behind"; it is definitely a "mouse and keyboard" user interface. Unlike tablet UIs, you can have more than one program visible on the screen at the same time, you can use the mouse and keyboard shortcuts... Unity is actually best with a keyboard.

If you want to see a user interface that's totally wrong for desktop computers, have a play around with Metro. One program on screen at a time, unless it's a classic Windows program (which sits on its own classic desktop). To launch a program you must return to the Metro program launcher, which of course hides all your other running programs. When you've seen Windows 8 in action you'll certainly realise that Unity is a desktop UI that can, with a few more tweaks, work on a touchscreen - not the other way around.
