### Post by irrdev on 2011-10-21
Today I decided to do a clean install of Ubuntu on one of my computers, and was once again greeted with the extra step of selecting 64-bit instead of the "32-bit (Recommended)". I have used 64-bit for three years now, and it struck me as really odd that this is STILL an ongoing issue for many (newer) Linux users - Mac OS X is 64-bit, Windows is offered as 64-bit on ALL new computers from my Best Buy store, and Adobe has just gone ahead and made Flash 11 for Linux available in 64-bit. Enough is enough. Unless you are running an antique computer (in tech time), there is no excuse for not using 64-bit. We might be able to debate the merits, but today's processors are all 64-bit and it is the future, along with ARM. I can't credit Canonical for not already making 64-bit default, because as of now, there are probably many new Ubuntu users who are blissfully keeping the "recommended" option without realizing that they are replacing their 64-bit Windows OS with a 32-bit OS. It doesn't make any sense.
