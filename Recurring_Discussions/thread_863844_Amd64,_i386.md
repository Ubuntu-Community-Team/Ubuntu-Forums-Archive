### Post by PmDematagoda on 2008-07-18
According to [this]("http://packages.ubuntu.com/hardy/zsnes"), there is only an i386 package and no amd64. So if you use Ubuntu 64-bit(which is amd64) then you can't install it normally although you could force it by downloading the i386 package and installing it with:-
```
sudo dpkg -i --force-architecture name-of-package.deb
```
but keep in mind that this does not mean that the application will work, since it is an emulator it may not work at all or will work marginally or terribly.

i386 and amd64 aren't very much different except that amd64 can support a lot more RAM and is also rather good for encoding or any other processes that require a lot of memory. i386 is what people normally use these days although 64-bit is getting more popular.

### Post by ~sHyLoCk~ on 2009-12-14
> **blueshiftoverwatch said:**
> Three newbish questions, but:

1) What is the difference between x86 and i386?

X86 means any intel, amd, duron style 32bit processor. It implies binary compatibility with the 32-bit instruction set of the 80386 (a.k.a. i386). 
[i386]("http://en.wikipedia.org/wiki/Intel_80386") is a CPU family it belongs to. 
An i386 CPU won't run an i686 code but i686 cpu can run codes compiled for i386,i486,i586.
Ubuntu is available in 2versions - i386 and amd64 which means if you have a 32-bit CPU, all your x86 CPUs are compatible. 32bit architecture was introduced by Intel in 1985 in the form of i386, so ubuntu isn't "officially supported" for [i286]("http://en.wikipedia.org/wiki/Intel_80286") which was a 16bit processor.

> **blueshiftoverwatch said:**
> 2)What is the type of 64 bit processor that isn't backwards compatible with 32 bit called? IA64?

I don't think there are such processors in the market yet which have no support for running 32bit codes. Reason- It won't sale. Many apps are still 32bit. [Not just Linux apps, but other as well.] I always thought IA-64 were pure 664bit , but I found out that they have a x86 core to support 32bit codes.

> 3) Is 64 bit only called Amd64 because AMD was the first to create a mainstream 64 bit chip?
