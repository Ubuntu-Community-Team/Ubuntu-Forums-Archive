### Post by -Jeremy- on 2010-05-20
Well KDE does not support GTK applications such as Firefox and Openoffice which causes them to get a pretty basic (and ugly!) graphical representation in KDE. But in Kubuntu this has been somewhat taken care of using qtcurve if I remember correctly. So you might be missing out on that if you install KDE on your Ubuntu system. 

Also please note that all of your Gnome applications will show up in your KDE environment next to their KDE equivalent which I personally find very annoying, although I've heard some other people like it.

In general KDE has way more options and graphical effects than Gnome (Ubuntu compensates this a little by incorporating Compiz Fusion) but also requires more system resources. XFCE is the lightweight among these three and you'll find it very much the same as Gnome (Because Gnome was based on XFCE!). Although XFCE has generally lesser graphical effects and toys it is also much more efficient with system resources and still provides a lot of functionality.

### Post by kaldor on 2010-05-21
Please, do not use Kubuntu to judge KDE. I did; and my love for KDE lasted very shortly after running into problems. 

KDE is done *very well* on openSUSE. openSUSE is definitely not for everyone, but if you don't mind learning give it a shot. 

The problem with the majority of KDE distros is that none are unique. Sure, on the inside they are all different. But, why the hell does everyone use the near-exact same default KDE setup? Aside from the wallpaper (which is usually set to KDE's default anyway) you couldn't tell Kubuntu from Slackware from Gentoo from Arch. 

openSUSE (and Mandriva, but they're nearly dead) actually put effort into KDE; their KDE desktops are actually customized by default and are also more polished. 

GNOME is something I am liking less and less. I do not use it nearly as much, but it has more potential right now. KDE is still a bit buggy due to the switch over to KDE4 while GNOME is rock-solid. But the thing is.. all KDE distros usually have the exact same software and rely on the same crappy software by default. openSUSE mixes stuff up; they gave OpenOffice and Firefox native KDE themes for one. openSUSE does not tell you to use Konqueror. 

The huge difference is that openSUSE made KDE their desktop while others just made an OS with KDE. Compare Ubuntu's GNOME to openSUSE's KDE. 


Anyway, back on topic...

I much prefer KDE these days. I find I can configure it more easily and it conforms to my needs better. I find Ubuntu and GNOME to be oversimplified to the point I can't fully do what I need without messing around. Ubuntu+GNOME is great for beginners, but more and more I find it being aimed at *only* beginners. KDE looks a bit too much like Windows by default, but that is easily changed. Besides, KDE acts nothing like Windows, save the "start" menu on the bottom left. In fact, I think KDE acts more like OS X than Windows in some aspects; look at the System Settings tool, it's almost identical to OS X's.

### Post by kevin01123 on 2010-05-21
> **Shining Arcanine said:**
> If you think that, then please explain why GNOME uses GTK instead of Qt? Being a KDE fan, I would like to know the answer.

GNOME was originally started because KDE used Qt, which at the time had a proprietary license. People within the free software communities didn't want to see the GNU/Linux desktop dominated by a proprietary toolkit.

Out of the desire for a free desktop came two projects:  GNOME, and the harmony toolkit. The harmony toolkit was meant to be an open source implementation of Qt. It's since been phased out. GNOME stuck around, and used the GTK toolkit, which was originally made for the GIMP.

Since then, Qt has become open source, but GNOME has stuck around.
