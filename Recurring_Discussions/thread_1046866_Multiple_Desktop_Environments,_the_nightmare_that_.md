### Post by etnlIcarus on 2009-01-21
The problem in developing for *nix is not the varying environments, it's the tendency for APIs to come and go without long-term support. You build your app around <insert library here>, only for the maintainer to disappear or the project fork or another API to pop up in it's place to satisfy some minor shortcoming of the original design, etc. 

Destroying *nix's polygamous relationship with DEs doesn't solve this problem, it just pisses off users who have diverse needs/expectations and will in all likelihood result in a bloated environment which will only run on the latest hardware (sound familiar?)
