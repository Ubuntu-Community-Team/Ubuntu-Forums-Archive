### Post by ve4cib on 2011-04-11
> **Thewhistlingwind said:**
> Speaking of which, how fast is machine code (If you were insane enough to write something in it.) compared to assembly? (Compared to C Compared to Java Compared to Python Compared to.....)

Assembly code, broadly-speaking, maps 1:1 to machine code.  Every single assembly instruction corresponds to a single CPU instruction (e.g. load, store, add, branch, rti, etc...).  So you wouldn't expect to see any speed difference between a program written in assembly compared with a program written in raw machine code.

However, a human-written program in Assembly would likely be slower than the equivalent program written in C -- unless the programmer is VERY good at assembly programming.  GCC has a lot of compiler optimizations, and will very often produce better assembly code than a human could.  And even if there are minor improvements one could make the time saved writing a program in a real language (like C) is probably worth it.



Back on-topic though...
