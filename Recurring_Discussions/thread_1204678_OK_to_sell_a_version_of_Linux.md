### Post by chessnerd on 2009-07-05
If you want to create a Windows-like version of Linux here are some basics things you will have to change:

The GUI - a new desktop environment that emulates Windows will be needed in order to make the system function like Windows. In the early stages of the product KDE could be used, but eventually you would want a new environment that works almost identical to Windows.

Application support - this is a big challenge that will not be solved overnight. Integration with Wine might be an early step towards making the OS Windows App friendly, but eventually you will need to make an even more robust piece of software that lets the OS natively use Windows software. If not done carefully this could cause compatibility problems with Linux apps.

The filesystem - this is one of the hardest parts of Linux to grasp for new users (and even moderately skilled ones) and changing the system to work like Windows (program files in the Program Files folder and Windows OS files in the Windows folder) will be one of the most important (and hardest) steps in making the OS Windows-user friendly. This will cause compatibility challenges with Linux and will likely lead to many programming headaches.

Making it sellable - Linux distros have been sold for years. Look at Red Hat and SuSE. These distros are built on the Linux kernel but cost money to use and can't be copied. Why? Because they come with proprietary applications. If you make your desktop environment, compatibility software, and filesystem software proprietary then people will have to pay to use your Linux-based solution even if the kernel is under the GNU license. However, you can't integrate these into the kernel itself because that would then put them under the GNU license as well. You need to have them be apart from the kernal to make them proprietary. This will slow down the OS, but will give you control over the software.

### Post by chessnerd on 2009-07-05
> **R.E.A.D said:**
> ok i will try to keep all of that in mind but i want to make it different in the way mac is different from windows you know what i mean

So you want a totally new, easy to use, customizable, sellable yet open-source OS? Wow. :shock:

First, you need to decide what type of GUI you want. GUIs are the most important thing when it comes to helping the everyday user. You have many options such as Gnome's multi-panel design, Mac's application bar and dock, Windows taskbar and desktop and you could even create some ingenious new GUI that no one has ever seen before. If you want ideas, look at lots of window managers (like Openbox, Fluxbox, JWM, and IceWM) and desktop environments (Gnome, KDE, XFCE, LXDE) to get inspiration. Also, go online and put in "Windows 7" in Google Images. It will give you a bunch of screenshots, but also a lot of conceptual designs that have some rather unusual GUIs, such as this one: [http://www.gearfuse.com/wp-content/uploads/2009/06/windows_7_5.jpg](http://www.gearfuse.com/wp-content/uploads/2009/06/windows_7_5.jpg)

Next you need to make sure that the user has options. Options = customization for most people. Sure, I could go out and write my own code to make Linux do some crazy thing, but I don't have the skill, time, or determination to do it. It would be much nicer if one of Ubuntu's prefrences/settings gave me the option to do something similar. I'd be happy and it would save me time. If you are serious about this, go around on the forums and ask people what they would like to see in a distro as far as cusomization options are concerned.

Making it sellable will be hard because if you want it to be open source it will be free on the Internet in no time. The only way to make it sellable while still being open source is to offer something with it, such as a support package or bonus software disks. However, if you want it to be sellable like Red Hat or SuSE, the only option is to load it up with proprietary stuff so that people can't legally redistribute it as a whole.
