### Post by original_jamingrit on 2008-05-14
The terminal is an important tool, not just for every day file browsing, but for logging, diagnostics.  It would actually kind of pointless to simply 'remove' it.  To remove bash (or dash, zsh, etc), you would also have to replace all shell scripts with compiled language binaries.  To make a purely shell-less distro, you would seriously be crippling it unless it was a distro made to fit a specific architecture or set of hardware.  Which would cut down on it's portability, unless there was some automagic self-diagnostic software running with it to make sure whatever problems are handled automatically, and then recompile modified boot binaries or insert new boot modules to make it work properly.

So in short, making a linux distro with no shell would be an exorbitant amount of work for very little gain and possibly a system that is crippled by design.  To do all the cool stuff in linux, you actually need an interpreter and shell scripts.

Now, if you're just talking about 'hiding' the terminal, Ubuntu does this now, very effectively, at least for supported hardware.  If you don't have hardware that you know is completely compatible with Ubuntu, it cannot be your totally GUI panacea.

Very valid point, but it's not just the actual command that needs to be remembered - it's also the options and the syntax for those options. Although syntax is uniform in most cases, there is obviously a need for commands to have their own unique options as well, which does make it tricky for a novice to remember everything. Also, the options used on a particular command (like fsck) will vary depending on the circumstances, and the majority of novice users won't know which option to use, even if they consult the man pages. A GUI with a few check boxes for the most common options and scenarios, would solve the problem. The average user has no need for real "in-depth" file system options, so those can remain with the command line to be used by someone more knowledgeable (sys admins) IF the user can't find/solve the problem with the help of the basic GUI options (which is likely to only be a small percentage of the time).
