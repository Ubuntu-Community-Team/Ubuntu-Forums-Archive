Edit: could I be nit-picky and seperate the BSD's - Whilst I often tell people linux is linux over in other OS.. this isn't the case with BSDs different derivatives do not agree to the same compatibility as with gnu/linux distro's. pc-bsd is derrived from freebsd in a similar way linux distro's do it, but this is the exception, not the rule. for example Dragonfly BSD is said to be forked from freeBSD - but they don't even share the same kernel structure - dragonfly uses a hybrid whereas free uses the monolithic. 

linux distros are really gnu/linux - gnu tools with the linux kernel and then more packages on top - this is common to all distos - bsd nowadays has elements of open source gnu throughout it's userland, but is still independent to each flavor - one flavor may use another's kernel for example and change it, but it doesn't then attempt to keep them compatible.
