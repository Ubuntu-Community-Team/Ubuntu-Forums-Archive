### Post by SunnyRabbiera on 2009-12-07
Well you would think because the code is open makes it insecure, but actually its the opposite for that very reason.
The thing is that being open source encourages people to look for weaknesses and fix them more easy then a closed source one.
In a closed source platform patches come very slowly because closed source does not allow many people to come in and fix issues.
But patches are applied all the time in open source making it more secure in the long term, while closed source patches are only good in the short term as hackers are fast.

### Post by foldingstock on 2009-12-08
With closed source software, if a vulnerability is found, you sit and wait for a patch. There are some things you can do such as blocking ports, limiting access, etc, but ultimately you are dependent on the manufacturer. 

With open source software, if a vulnerability if found, you have multiple options: 
[LIST]
Wait for the dev's to release a patch[/LIST]
[LIST]Wait for third party/hobby dev's to release a patch[/LIST]
[LIST]Attempt to fix the code yourself[/LIST]
[LIST]Recompile code with certain options removed, eliminating the vuln[/LIST]


This doesn't make open source "more secure," it just gives you more options.
