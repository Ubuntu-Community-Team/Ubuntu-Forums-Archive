### Post by rocky2003 on 2014-03-06
I realize this is going to be contentions post to some but as someone who wants to Ubuntu succeed in it's mission as a consumer OS for wide spread appeal I think it has to be said.

_**Stop developing Ubuntu for linux users.**_

_**If your using the command line your doing it wrong.**_

Ubuntu & other linux OS appeal to only two groups the most basic simple user or a linux command line loving expert faimilar with the deep inner workings.

Ubuntu barely even gets the basic user right, in the beginners forum there is a sticky thread for leaning linux command line, no beginner forum should ever have that. 

Windows (& OSX) relegated the command line to advanced trouble shooting and IT shops long ago, both OS are designed for GUI power users whereas Ubuntu still feels very unpolished with command line work popping up for some stuff even I consider basic, this doesn't happen on Windows or OSX.

Then there is application support, outside of the app store even if there is a linux version of an app it is still a nightmare to install incomparison to Windows which has easy installers for apps or OSX just drag and drop the app. Ubuntu needs something similar to Windows if it is ever going to get users onboard and convince developers to make Ubuntu specific apps not just generic linux apps.

One example is Steam, on Windows or OSX installing Steam is a breeze I download Steam installer and off I go, on Ubuntu
1) Go to app store, click Steam and tells me I need an Ubuntu One account
2) Sign up for account and go back and get Steam which downloads a Steam installer
3) Run Steam installer and it brings up command line prompts asking for root access with Y/N confirms then shows the guts of the install all the way through.

Another example is VLC or Handbrake other popular power user apps, all a breeze to install on Win PC but unless you wait for app store update version then installing manually is a pain in the ass. You should be able to download and install apps just like Windows, without that you wont win them over.

It's shabby and unprofessional looking, not to menton other parts of Ubuntu like on a few PC's I have installed on show the command line background workings when shutting down the PC, it's like the OS can barely keep the GUI together in places.

Google's Chrome OS is a lot more simplistic so it does less than Ubuntu but for what it does do it's in a more professional and competent looking manner without a hint of any command line or linux based nonsense that still infests Ubuntu. If Ubuntu doesnt get its act together then Chrome OS is eventually going to be the number 1 PC linux OS, it's already making inroads in business and as it slowly involves into a more featured PC OS, Ubuntu will still be wondering why the mainstream went with Chrome.

### Post by EnglishElectricAndy on 2014-03-13
Well, I'm a 'consumery' type of computer user and I'm pretty well completely won over to Xubuntu. I used to run Vista, worked okay for me for a while but eventually I found myself spending more time on maintenance (defragging, disk-cleaning etc.) than actual productive use.

Decided it was time for a new machine so at the start of 2013 bought an HP laptop with Win8. Big mistake for me, found it hard to use, lots of stuff I didn't need getting in the way. I can see what Microsoft were trying to do by creating a uniform interface across various device platforms but just didn't suit me, despite persevering.

Having a general growl about this over a pint, a beer buddy said he used Linux all the time and why don't I give it a try? Now I'd vaguely heard of Linux as being some geeky, hobbyist outlier so I was reluctant, until this same buddy gave me a USB with Xubuntu 13.04 on it. Being cautious, I fired up the redundant Vista machine, rejigged the BIOS to boot from the external USB, and clicked on 'Try without installing'.  "Wow, this is fast!" was my very first thought. "And it's got folders, and files, a photo viewer, a music player. What's this Firefox thingy? Hey this is much easier than IE! And no popups reminding me that Norton 360 needs updating!" So, I installed alongside Vista (still cautious, just in case I got something wrong somewhere). Vista hasn't been fired up since then and my Win8 machine goes online once a month to get (usually multi MBs of) updates.

Regarding the terminal or command line my first thoughts were along the lines of "I'm not going anywhere near that, I could seriously mess things up there" but after adding the Ubuntu forums, and a few others, to my Bookmarks, reading the threads through and searching, the terminal has far less of a sense of 'Here Be Dragons' on an ancient map. Having said that, a command featuring a chain of 'greps' 'pipes' 'awks' and 'seds' has me rapidly opening a new tab for research and hopefully education.

In summary, I made my own small addition to an oft-repeated meme: Linux is not like Windows, and I didn't expect it to be like Windows.

### Post by erintiransom on 2014-03-14
I'm a convinced Ubuntu user. I've used it sporadically for a few years, and then almost exclusively for around one year. Before completely switching, I used to fall back to Windows because I needed to use specific software. No, I'm not talking about Word or Excel, I already used LibreOffice in Windows. Just two pieces of proprietary software unavailable in Linux, without a good alternative one of them, that I had bought and really like. Then I discovered virtual machines. I barely ever go back to Windows, since that discovery. Later I found an alternative program that read the data from one of the two Windows programs I needed, so now I only fire the VM for one of them, so not very often.

I'm not afraid of the command-line. I'm not afraid to research and fix things. One of the reasons I switched to Ubuntu, apart from ideological reasons and privacy concerns, is having built-in GPG support. I followed some complicated forums to create a 8192 bit key. I've tweaked many, many things.

I'm saying all this to build credibility, I guess. I won't go back to Windows, I'm committed to Ubuntu. I'm not afraid of the command-prompt. Nevertheless, I think there is some truth to what this guy is saying. Every time I start Ubuntu I don't see the nice Ubuntu logo, I see a plain Ubuntu 13.10 text, and when I restart or shut down, there are commands all over the place on top of that same ugly screen. I believe this is because of my system having an NVIDIA graphics card and the proprietary drivers. I'm OK with that, but it had worked fine in previous installations in this same laptop (12.04, 12.10) so they changed something and now it looks crappy. I don't really care, but if a cult-of-Mac friend sees that, he won't listen to anything I say. Same thing for Windows users.

Besides the aesthetics, this same driver problems cause my computer to freeze when waking up from sleep mode, relatively often -several times a month. It is extremely annoying, but I have developed the habit of closing important programs before putting the computer to sleep. Normally I can CTRL-ALT-F1 to halt the system, but sometimes the freeze is bad and I have to power down the computer.

A recent problem, which caused me to come to Ubuntu forums in the first place. Unzipping a big file (1.5 Gb) containing over 150.000 files. Can take up to ten hours from the normal GUI program (Gzip, right?). In the W7 virtual machine it takes 15 minutes with 7-zip, if I move it to a folder inside it. If I try to uncompress from the shared folder (shared between Ubuntu and the VM) it is extremely slow, like with Gzip. Now somebody will say I can use 7-zip port. I did, it's faster than Gzip but still way slower than the 7-zip in Windows. Take into account that 7-zip in Windows is also working slow if the file is in the shared folder, so I assume the problem is with the ext4 file system, or who knows what.

As I said, I don't mind using the prompt, but I really think it is actually faster to have a good GUI, specially integrated in Nautilius. For example, crypto operations are incomplete, comparing to an underfunded open source project like GPG4Win in Windows. I can use the prompt, yes, but it is actually a waste of time when I could right click and get things done fast. And it makes average users averse toward Ubuntu -OK, average users don't use crypto, but I'm sure there are other examples of bad Nautilius integrated functions.

Seahorse is very incomplete. It seems it went to worse in the last version. Yes you can manage certificates and keys from the command line, but it is visually hard to find things there, a good GUI is much faster.

Anyways, it seems the people answering are just not wanting to spread the word about Ubuntu. It seems they want it to be a niche product. Good job, you are condemning millions of poor people in underdeveloped nations to use Windows. Either pirate versions with all the viruses and malware, plus accustoming them to not use open source, or to paid versions that they will find difficult to pay. Even worse for governments (not just in third world nations), civil servants are lazy and want to use Windows, they will never switch if Ubuntu doesn't become more user-friendly. Which means you (all you superior gurus) and I will have to pay those licenses to Microsoft in the form of taxes.
