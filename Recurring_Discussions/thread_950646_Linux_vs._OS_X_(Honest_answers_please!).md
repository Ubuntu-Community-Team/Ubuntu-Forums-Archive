### Post by aysiu on 2008-10-17
I think if you want to get useful answers, a better question would be - what type of computer use and personal situation would most benefit from Linux, and what use and situation would most benefit from OS X.

If you don't want to use Windows, are either rich or able to save up a lot of money for a computer, have the latest iPod or iPhone, buy regularly from the iTunes music store, are a professional graphic designer (i.e., you make a financial living off your graphic design work), and you like buying well-configured preinstalled solutions from brick-and-mortar stores, then Mac OS X would be great for you.

If you don't want to use Windows, don't mind troubleshooting potential hardware compatibility issues *or* have a Linux-using friend who's able to configure Linux for you, like having your programs autoupdate (but don't need to have less-than-six-month-old cutting edge versions), do not own an iPod, mainly use the computer for web browser, email client, word processor, photo organization, and music listening, then Linux would be great for you.

Of course, there's overlap. If you're a person with a lot of money and without an iPod or iPhone, you don't mind troubleshooting potential hardware incompatibilities, and you don't use Mac OS X-only programs, you could easily be fine with both Mac OS X or with Linux.

P.S. Yes, I realize there are preinstalled Linux computers (I bought one six months ago myself), but I didn't mention them for several reasons: [list][*]Most of the Linux preinstalled solutions are online only and not in brick-and-mortar stores[*]Many of the ones that exist are limited to only a handful of countries[*]They are not usually well-configured. Yes, they take a lot less work to get perfectly running than if you'd installed Linux yourself, but based on what I've experienced myself and what I've read, the Linux preinstalled solutions could use more testing and better default options.[/list]

### Post by estyles on 2008-10-17
I feel like, for "naive" users, OS X, Windows, or Linux are all perfectly fine OSes.  Ideally, it needs to be pre-installed, as naive users generally don't, and shouldn't be expected to install their own OS.  That makes it difficult for Linux, since it's hard to find it pre-installed, but the options are growing all the time.  The real choice of OS comes down to price, and what a particular user is comfortable with.

For users with specific professional needs, it depends on what you need.  A lot of professional image, video, and audio software is only available for Mac and Windows.  For servers, Linux is an obvious choice, although some people need Active Directory or ASP for whatever reasons.

For the computer enthusiast, though, I feel like you are missing out if you write off Linux.  There's just so much freedom to modify whatever you want with a GNU/Linux OS, and there are so many groups out there writing new utilities and applications that are only available on Linux, or that are native to Linux and eventually ported to OS X and Windows.  I think these types of applications are going to continue to grow and grow, and GNU/Linux will continue to be the "enthusiast" OS.  Stuff that is essential, particularly for professional applications, will always be available in OS X and Windows, and certain things will only be available in those OSes.  And while I think gaming will eventually be better on Linux than it is now, for the foreseeable future, dual-boot is a must for serious gamers.  But the free software movement is real, and it's happening largely on Linux.

Bottom line:  I see no reason to buy a Mac unless you just don't want to take a few minutes to configure a Linux box or if you are an extreme noob and don't know how to install an OS.  Or perhaps you just want a Macbook to be cool.

### Post by k2t0f12d on 2009-06-09
> **Sublime Porte said:**
> What kind of Unix will terminate an sshfs mount because you close the terminal in which it was initiated??All of them I expect.

In GNU+Linux, any running process is a child of the shell and dies with the shell.
If it were a local hardware mount, that might persist after the shell closes because that is a state in the kernel itself.
Your sshfs program opens the network connection that provides the mount.
When the shell exits, the program dies, the connection closes, and the mount dies with it - exactly as it should.
Write a script that uses sshfs to open the mount, that way it will persist until Mac OS shuts down or the program is closed,
because the script is a child of the window management shell.
This behavior is not a fault.

Wow, I didn't expect this thread to grow so much...

Well, it's been about 2 years since I posted the original question, and after using a Mac every day for a year straight (for work), I have reached a few conclusions.

1. I have much less free time than I used to.
2. Linux is still awesome.
3. *nix (Mac OSX included) is better than Windows.
4. (as someone else said) OSX's weak point is also it's strongest.
5. Most of your opinion due to the method in which you think.

Okay, so point (1) The main reason I posted the original question is that I was becoming tired of tinkering around all the time... Part of that is my own fault-if I start tinkering, I can't stop, and if I have the ability to, I really do want to... But, in any case, it was becoming inconvenient. So, to that end, I think that Mac OSX wins.

Point (2) is most likely going to seem to be a fan raving, so let me explain. The biggest thing I miss in Linux on my desktop (mind you I still use it on servers on a daily basis), is that I can't dig in very deep under the UI in OSX. I can much more than Windows, but there's still a little guardedness, a little protection from the inner sanctum of the kernel on OSX. Compared to Linux, where it's wide open. That same openness in Linux that runs through the entire UI is the same closed-ness that runs through OSX. So, you can customize some things, but not others, where on Linux you can customize quite literally EVERYTHING and then some. Way more than 90% of the _most_ hard core techies would ever want or need. Linux wins by miles here.

Point (3) is mostly off topic, but still important. I definitely prefer OSX over Windows, hands down. It took a while to get used to some of the conventions that didn't make sense coming from a Linux/Windows world (i.e. dragging items off the dock, enter to rename, shortcut to access menu bar, home/end keys, etc.) But all in all, I like both the UI choices and general mode of thinking that OSX has taken over Windows, and in some respects Linux as well. For example, almost every important command in OSX has a Cmd key shortcut. If it doesn't have one, but does have an item in an application menu, you can create a shortcut or change it. This is something that is 95% universal across the entire OS. Linux comes close, especially KDE's environment, but Windows doesn't even hold a candle to either Linux or OSX here. Furthermore, Linux and OSX come with a decent shell (bash)... something else that both made it easier to switch to OSX from Linux and is sorely missing from Windows.

Point (4) is also true. Quoting Wiebelhaus, "The same advantage of OSX is the bane of OSX (For me anyway) is it's proprietary nature, for me that's a deal breaker but for someone who wants to pay the "mac tax" for the support and quality of hardware and the quality guarantee that it comes with everything that's needed and all of the hardware and peripherals with flawless support , then that's what they get for that extra payload." This sums up my point pretty completely. Just to add that, yes, their hardware/software integration makes most things "just work", but when they don't, it's a royal pain, and when they do, it's absolutely beautiful. Either the worst of the bad, or the best of the best. Hardware quality is also top notch and is another plus point for me... Apple always seems to put a great deal of care into how they design their interface, from hardware through software. Things like the backlit keyboard, solid aluminum body, magnetic power cord & lid closure, etc. I particularly like the hinge they use on the lid of the Macbooks, because it's one piece it's very stable.

Point (5) is mostly in response to people who say "Linux/OSX/Windows does X which is fundamentally wrong with Y"... For example (to borrow aysiu's example), enter (return) on a highlighted item doesn't open it, it renames it. To borrow one of my own, the home and end keys don't take you to the beginning of a line or the end of a line in any native app, they take you to the beginning and end of the document. Honestly, that drove me nuts for at least a month, possibly two. But it all depends on how you think. I appreciate that most applications are a simple drag 'n' drop to install because almost the entire application is contained in that one folder... That's a blessing and a curse (as with most things), because any utilities in that package that may be useful from the command line (mp3gain and ffmpeg come to mind) must be either added to the PATH or symlinked to somewhere that the PATH is already at (eg ~/bin or /usr/local/bin).

Honestly, it's a largely personal choice. I still use Linux of a few different types (Ubuntu included) on a daily basis for servers and other applications. But for a desktop? Well, honestly, right now I prefer a Mac. But, I know that I always have my Linux experience to use in case I decide to be frugal or discover something that really bothers me on my Mac. Furthermore, I'm using (mostly) applications with open standards like OpenOffice that make it easy for me to get 90% of my stuff onto whatever computer I want, should the mood strike me. The other 10% can be easily accomplished with a Windows VM, which I've never been able to get away from completely in Linux, unfortunately. I could in Mac OSX if it weren't for my job.
