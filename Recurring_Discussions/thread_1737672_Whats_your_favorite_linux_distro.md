### Post by bonixavier on 2011-04-29
> **boydrice said:**
> the boot process is slow if you just take the defaults like the Huge kernel, default Lilo config, default DHCP polling(not needed on a laptop), etc.  Install a regular kernel, edit lilo.conf and uncomment "compact", disable DHCP polling on boot and presto it boots much faster.  Slackware is what you make it.
Huge _is_ a regular kernel. Perhaps you meant use the generic one? DHCPCD doesn't bother me because I use a static IP. Either way, the problem is that, by default, some daemons are not started in the background (hal for example), fc-cache, gtk-update-icon-cache, update-mime-database, etc. are started in every boot. You can make them cron jobs and it will start faster too. Even after doing all that, you won't boot as fast as distros using upstart or systemd unless you are into heavy editing of the rc scripts.

My point, though, was that, after you boot, Slackware will outperform any distro I've tried (and the list is long).
