### Post by Thewhistlingwind on 2011-06-10
> **brawnypandora0 said:**
> How do experts memorize so many commands?

Associate a command with it's function/purpose. When you can't remember the command for a function, but you've used it before, read through your bash history until you spot commands from the last time you think you used it.

EDIT: Also, all the commands are really just small programs that do one thing well. That's part of the Unix philosophy. The commands themselves are usually abbreviations. cd for example, is "Change Directory" or theres ssh which stands for "Secure SHell" or ufw which is an abbreviation of "Unix FireWall". So you can remember them by what the abbreviations stand for, kind of like a reverse mnemonic.

### Post by practic on 2011-06-24
> **brawnypandora0 said:**
> Wow! That's quite a jumble of letters. No wonder I never heard of it.

N - Network
D - Driver
I - Interface
S - Specification
G - Gimp
T - Tool
K - Kit

GTK is a collection of widgets (buttons, scrollboxes, etc) used to build programs. Most of the Gnome programs use the GTK toolkit, which is why they have a similar look.

NDIS is the specification for how Network Drivers are supposed to communicate with their hardware. It was developed by Microsoft and 3Com, and is implemented in the network drivers for Windows. 

Ndis-wrapper is a software that takes the windows drivers and re-directs their calls so that they work properly with Linux (and other open-source systems).
