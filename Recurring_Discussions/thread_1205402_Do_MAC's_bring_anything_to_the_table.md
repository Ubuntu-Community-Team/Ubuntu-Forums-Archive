### Post by aysiu on 2009-07-06
I'm also a big fan of the magnetic power cord, especially since I am a major klutz.

Here are some other things:

1. Apple trailers always play.
2. Ability to sync iPhone and iPod Touch
3. Ability to *easily and reliably* run the latest Adobe Creative Suite (or FileMaker or other commercial productivity software)
4. Smooth animation - beats Compiz any day
5. High resolution icons always
6. No momentary terminal display before the bootsplash
7. Uninstall of applications by moving icon to the trash - yes, I know GoboLinux can do this too, but [GoboLinux isn't for the faint-of-heart](http://www.gobolinux.org/?page=faq)
8. Sleep light fades in and out instead of blinking on and off
9. Laptop speakers never sound tinny
10. No audio mess (ALSA, OSS, PulseAudio)
11. Write support for journaled HFS+ in the GUI
12. Universal inbox in Mail client
13. GarageBand
14. Time Machine

Now, granted, some of these "advantages" are artificial ones (1,2, and 11), since they are basically Apple just not playing nice in the interoperability department, but to say Macs bring *nothing* to the table--that's ridiculous.

I happen to think Macs are overhyped. They are computers, not magic. In fact, I'm still struggling to get WPA or WPA2 working on my wife's Macbook Pro (didn't work on her old Powerbook either). What we ended up deciding, because she was so frustrated--get a new router and use WEP in the meantime. Now if our positions had been reversed, you can bet a whole bunch of anti-Linux trolls would have said "See? Linux isn't ready for the average user. It can't even handle WPA connections." But since it's a Mac that can't handle it, well... it's just unfortunate, I guess.

I don't find the Mac user interface very intuitive, but I think intuitiveness isn't that important, and it's always trumped by familiarity. The Mac Aqua theme is also quite pretty, and all the Mac-imitation Gnome and KDE themes I've seen just do not compare.

### Post by mojo2012 on 2010-02-27
I'm now a Mac user since 2006 and I can only say, that Macs (both hard- and software) are better in nearly any sense.
Why?

Hardware:
* magnetic power cord/plug
* Power adapter has a mounting to reel up the cord.
* keyboard backlight (I never every want to have keyboard without backlight anymore ...)
* Display becomes darker before it goes off
* Brightness sensores that regulate display and backlight brightness
* Shock/fast movement sensores for harddrive (can be used for other things to, eg. switching to another virtual desktop ;-))
* Apple Remote control can be used to control Frontrow (some kind of simple boxee media frontend), iTunes, browse though PDFs or presentations
* 360° two finger sliding/scrolling (I hate to have to move the finger on the border of a trackpad to scroll
* multitouch trackpad gestures, eg to rotate images
* (old) Mighty mouse: trackball that can be used to scroll in any direction (360°), not just up/down, very useful eg. for photo editing
* target disk mode: connect your mac to another machine via firewire. Then boot it up and hold "t" and it acts as a external firewire harddrive for the other computer - without booting up Mac OS X.
* two graphics cards, one integrated (more battery life) one dedicated (faster, eg for games). Can be easily switched.
* Battery has charge indicator (no, even in 2010 not all notebooks have this!)
* nice grapfical boot manager that even has mouse support. Automatically detects windows and linux partitions (on internal, usb, firewire, cd/dvd) drive and boots them up. Even shows the correct logos. Can be controlled by mouse, keyboard and even the apple ir remote control!

Software:
* Exposé: show all windows or just all windows belonging to a certain app. Fully supports virtual desktops (spaces) and drag and drop, hot corners. Yeah, Compiz can do this as well (Scale plugin). But do hotcorners work with drag and drop: NO! Show Desktop plugin also doesn't work with drag and drop. Expo mode + Scale? Doesn't work either!
* Dock: ok, I have to admit, that the concept of managing apps instead of windows might be weird at first glance, but it's so much easier, imho.
* Stacks: folders that show a submenu with it's contents. So far, so unimpressive. The download folder stack pops up, if a download is finished (or if any new item is added). This is a very elegant way to inform the user that a download has finished. You see a live preview of the contents of the folder too.
* The progress of the running download can be see on the file icon (small progress bar overlayed.
* Files can be commented, tagged and can even be colored. So it's easier to find a certain file in a folder (tagging_coloring_bonjour.png, 1 + 2).
* Spotlight: not just a very fast (much faster than beagle or windows search) indexing and search engine. Searched tags and comments too. Can be used to calculate, search in dictionaries (I love it), bookmarks, browser history. What? Deskbar? Oh, this ugly looking half-baked spotlight clone? Yes, it can do the same things. But it does them worse. Despite the fact, that the display space is used inefficient (what's about all these "run ...", "start ...", "show ..."??), why doesn't it show the search results directly in the search window, eg. look for "tree" in dictionary (spotlight.png)? It just should say "tree: eng. für Baum" or something similar.
* Intelligent and adaptive search: The way results are ordered is not fixed, but is constantly being optimized on base of what results I most often use.
* Quicklook: press spacebar on a file and it shows a preview. Can be used to look at/into presentations, doc files, archives, folders, slideshows, movies, music, images, installation packages, ... What, Gloobus can do the same? Well ... partly! Ok, it's buggy - it's beta (or even alpha?) - that's ok for me. Can I use it to browse though all images in a folder? Not as comfortable as with quicklook. Does it work everywhere (eg on network) volumes? Nope, at least not in nautilus (caused by gvfs or how it is called ...)
* Dictionary search in every native mac os --> system wide keyboard combo ctrl-cmd-d --> a small bubble pops up showing me word definitions/translations. Doesn't even need to open a separate tab/window/app (dictionary_researchable_textbox.png, 1).
* Safari search: highlights the found words, dimms the rest (safari_search.png, 1). In all browsers I have the same problem: a word has been found but I can't see where. In Safari? No problem at all.
* Safari: resizable text boxes (dictionary_researchable_textbox.png, 2)! Forums, blogs, comments, stuff like this: the textboxes are alwas too small. No problem here on mac os, must grab the resizer and make it bigger ;-)
* Drag and drop simply works, everywhere! Dragging a file onto a folder, and the folder opens. Drag it to the Dock and the app comes to foreground. Drag it to a hotcorner, and Exposé/Spaces comes up. On Linux? Hmm, what is drag and drop?
* App installing? Packages managers? Version conflicts? Libraries? Who cares about that all? On Mac OS X you just download a Disk image, mount it with a double click, drag the app, eg OpenOffice.app (actualy a folder called application bundle), to whereevery you want it (most often /Applications/), double click it and voilà: it runs! You say that package managers are easier! Maybe if I want to install ancient software (that is stable). What if I want to have the latest version (this should be a user's choice, imho)? PPAs? DEBs? Manually download keys and setup different respository. If not availble then …? Yeah, I can just grab the sources … Do I really want that? NOOOOOO!!!!! Why should I want to compile something? I want to use/try it, not compile.

* Sound just works. Plug the speakers in, no configuration. No crackling, no PulseAudio bugs. (since jaunty I have problems regulating the volume for my surround system)
Integrated efi bootmanager, that allows
* PDFs can easily be cropped, combined, tagged, commented, compressed without any special/3rd party app. Creating PDFs can be done from every app via "safe as". You even don't need a "pdf printer". It's even possible to "bookmark" certain pages.

Because people see a screenshot (more or less a menu that shows the contents of a folder) and think that's it.

In general: every damn aspect of the GUI is integrated, designed to be part of something bigger. Every linux GUI (and windows too) I've ever seen/used lacks that. OK, it's possible to theme Gnome. But regardless of which theme is used, it simply doesn't look as good as Mac OS X, which in any case is "pixel perfect".

Ok, I've to say here, that if I'm not happy with the default behaviour of something, I even have to go to the terminal in Mac OS X and change Apache configs or stuff like this. But not the "normal" stuff a normal user does. This is only necessary in special cases (servers). 

Please don't missunderstand me! I really like open source. I like and use Ubuntu (more than windows ;-)). I really appreaciate all the effort that is put into open source.

I dream of the day when Mac OS X becomes Open source ... or at least Étoilé becomes usable.
