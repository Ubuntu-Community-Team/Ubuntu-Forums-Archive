### Post by jhsu802701 on 2010-06-24
I'm a former user of Ubuntu.  I had to start out with the minimal command-line installation and add packages, as the computers I use have only 256 MB of RAM.  This setup meant sacrificing Ubuntu's much-touted user-friendliness.  Every year, a segment of the Ubuntu-using population gets cut off.  The hardware requirements increase with each version, and support is terminated for some of the older versions.  So while 512 MB of RAM is enough for today's Ubuntu, that won't be the case in a few years.  I can even see a future in which Ubuntu matches Windows Vista when it comes to hardware requirements.      

The trend towards more bloat in Ubuntu will not only affect Ubuntu users but also users of other distros based on Ubuntu, like Lubuntu, Peppermint, etc.  256 MB may be enough for these lighter Ubuntu-derived distros today, but will that still be the case in a few years?      

Canonical needs an ultralight distro that takes on antiX Linux and Puppy Linux.  256 MB of RAM is not enough for Ubuntu, Kubuntu, or Xubuntu but is plenty for antiX Linux and Puppy Linux.  While Ubuntu may be trying to compete with Microsoft from a marketing angle, it needs to take on antiX Linux from a technical angle.  There are many computers out there from the Windows 98 era that are gathering dust.  Since these computers have little or no resale value, they're affordable for even the poorest people.  (A used computer that can run Ubuntu would cost at least $50.)    

antiX Linux is my main distro, and I consider it to be the gold standard.    Unlike Ubuntu, antiX Linux is FAST with only 256 MB of RAM.  antiX with 256 MB of RAM is probably like Ubuntu with 1 GB or more of RAM.  Like Ubuntu (full installation) and unlike Debian, antiX Linux is user-friendly.  The Rox Pinboard desktop option even offers icons and shortcuts on the screen but is bloat-free.  antiX Linux is compatible with the Debian repository, so it's competitive with Ubuntu in this respect.      

In my opinion, Lubuntu doesn't go far enough, as it requires at least 256 MB of RAM.  Canonical needs an even lighter version of Ubuntu or to create a new separate Debian-based distro.      

The Linux world would be so much bigger if antiX Linux had the marketing power of Ubuntu or if Ubuntu could work just as well as antiX Linux on old computers.
