Ultimately though I don't think the command-line will ever go away.  We'll keep building bigger and better GUI tools that will replace most of the functionality of CLI applications, but the CLI will still be there for those who want to use it.

### Post by chessnerd on 2010-03-17
I have actually seen my command-line usage in Linux go up since I started using it. First it was very little, then it went up whenever I needed to fix anything, then it would go back down. Now it is steadily going up. I've found that for some things, like copying or moving large groups of files, the command-line is much faster. 

Just today my instructor gave us all some code for a project. From the command-line I was able to extract the JAR, cd into the directory with the files, then use cp *.class ~/workspace/Project5/bin and cp *.java ~/workspace/Project5/src and after I hit F5 in Eclipse everything was ready to go! Will the command-line die someday? Maybe. I don't claim to be a psychic, but I don't see that happening anytime in the next twenty years at least.

If you like the GUI, use it. If you like the CLI, use it. If you like using both, use both. And best of luck to you all. ;)

### Post by abohsin on 2010-03-20
Well, interesting thought, and interesting answers.
Depends what you are using your linux box for.
For work, admins and system admins, nope, command line will never end, it is always easier and more flexible than GUI.

For desktops/laptops at home, you may not use the command line at all, personally i don't use it once i reach home, because i have my eyes pointing at commands all day at work, so at home i enjoy a good GUI with minimal typing and lots of mouse navigation.

Command line will never die, simply, because commands have too many switches, which you can control by typing them in or not typing them. But how would you handle these switches in GUI? and how would you choose which combination of switches to use each time you issue a command? It will stuff you if every time you make a move in the GUI the OS displays all switches and asks you which ones you want to associate with your current action. Which is precisely why it is left for the command line flexibility, want switches, type them in.

GUI usually chooses for you which most flexible switches of commands running in the background to use, which is all the point in the GUI of an OS. This is what makes one OS better than another, choices to relax users and free them from hassles of making their own choices. Whereas the command line is there for you to choose and tweak your actions.

So the definate answer is no, it will never die a natural death, it will always be there for you, if you ever need it.
