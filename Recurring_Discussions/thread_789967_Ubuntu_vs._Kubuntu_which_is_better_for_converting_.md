### Post by CSMatt on 2008-05-11
I've gotten several queries about my Ubuntu system in the past few months, and every time I feel like kicking myself for not carrying around a Desktop CD to give to them.  So far, all of these inquirers were Windows users, and all of them are not particularly interested in computers, so I don't imagine that they will have the time and patience to learn a new operating system if it proves to be more difficult than, say, learning Mac OS X.  Problem is, I'm not sure which derivative to hand out to them.  The GNOME-based Ubuntu, on the one hand, is more simplistic and seems to be less cluttered.  It's also the derivative that I run, so I imagine that these people will be expecting something similar to what I am running when they boot up their LiveCDs.  On the other hand, the KDE-based Kubuntu seems to be more Windows-like.  There's a taskbar with the K menu by default, and things like the Konqurer file manager seem to operate similar to their Windows counterparts.  However, it isn't always so obvious in KDE where you should go to achieve certain tasks.  For instance, it's pretty clear that the menu with the programs in it is in the Applications menu in GNOME, but KDE's Start button equivalent is just a big "K" icon, and it easily looks like the other "quick launch" icons if you don't already know that it's a menu.  More common applications, such as Firefox, are also not present in the default Kubuntu install, whereas Ubuntu includes them by default.

Given this, which do you think is better for Windows users, Ubuntu or Kubuntu?  I'm not asking which DE is better, just which one will provide a more seemless conversion for Windows users who would rather spend more time getting work done than trying to figure out how their system works.

PS: I know that Ubuntu is not Windows, so I'm not expecting either DE to give these people that at-home feeling at first, and I'm not expecting them to be able to use the system right away without having to read at least a little documentation.
