### Post by Jon Monreal on 2010-04-17
Really, I don't think this is that bad of a move.

For one, after you close a program, if the window was maximized your cursor is already close to the Applications menu so you can open another program (assuming you use the default layout, which I would imagine the majority of users do and if they're technical enough to move it they're technical enough to change the side the buttons are on).

Having the buttons on the same side of the menus also makes sense for users new to computers because the left side becomes the go-to side for all of the functions related to the program. Perhaps this makes even greater sense in a way, because in Linux (and Windows) windows represent applications, not documents (as in OS X).

One other thing to think about is that the red colour of the close button in the default theme pretty much screams "I'M THE BUTTON TO CLOSE THE WINDOW!" just like the one Windows XP, Vista, and 7. People will be able to find it.

I would guess that the same users that some here consider to be the "unwashed masses" wouldn't have any problem switching over to the new system. The only problem then is to make sure it is clearly explained **why** it is being done, so people don't just say "that's stupid". _Perhaps a slide should be added to the installation slideshow_.

EDIT: For those interested in more about the UI paradigms used in Windows and OS X, take a look at this [excellent Ars article]("http://arstechnica.com/software/news/2009/01/dock-and-windows-7-taskbar.ars").
