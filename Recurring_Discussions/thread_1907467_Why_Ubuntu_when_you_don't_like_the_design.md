### Post by Greg Merchan on 2012-01-11
I'm using Ubuntu because I believe Unity is taking the UI in a direction I want it to go and all but one of the other distributions are going somewhere else. I've been minimally involved, but I see very often people clamoring for changes which go against the direction Unity is going; against, not just off to a side. Two cases come to mind: the menu bar, and options.

I think Mr. Shuttleworth has indicated clearly that the global menu bar will remain in Unity. Some details may change, like how the Dash trigger is now a button, but the basic idea is fixed.

I don't know that there's an embargo against making options, but building on Gnome and going for something "better than Mac OS" (not an exact quote) indicates to me that Ubuntu Unity is not going to be the UI that adds option upon option like other Linux UIs do.

So, I'm wondering, why are people, opposed to the direction Unity is going, flocking to it when there are other UIs more in line with their desires? Why are there open bug report against things that are clearly design choices? Why are Brainstorm solutions of the "add an option" type so popular?

Frankly, toward a Mac-like interface. This includes the global menu bar, the indicators, better use of title bar controls (not yet implemented), and the high barrier to customization for some things. There are a few things I don't like about Macs, but the only things really keeping me away at the point are cost and my concern for software freedom. It feels like Mac OS X Lion has introduced a dozen features that some of us were trying to get on Linux desktops a decade ago, and that's incredibly frustrating.

Against the direction are not only those who want the global menu bar to go away, but also those who want a configuration option for it. I've made my comments on the bug report claiming the menu bar isn't good on large screens.

I use a 1920x1080 screen on a laptop. I would have been using an external monitor too, but the driver support was lacking and it wasn't that important. I don't know why I thought the NVidia mess had been sorted out. Unless that changes, I'm not getting another computer with NVidia.

Call it baby duck syndrome, but my GUI of choice would be OS/2 WPS, before the colored tabs, with a global menu bar, multiple desktops, and some cosmetic changes. I think if I tried to be more descriptive, I'd end up having to write a complete spec; let me not go down that road. Maybe it's what Taligent would have been.

The other distribution I think is going in the right direction is Etoile OS, but I haven't had time to work with it. The installation instructions indicate they're still too much in development, and I need something I can use for work.

I hope there are plans to replace Nautilus soon, because it keeps getting worse. You can only claim it has a spatial mode now if you have a bad sense of humor. Once they removed Background and Emblems, figuring out the themes so my folders were not all blindingly white was a nuisance. A good spatial file manager can easily be used strictly as a browser or two-pane, but the reverse is not true by a long shot.

As usual, I've rambled on. Thanks to everyone for your answers.

### Post by Handssolow on 2012-01-13
I don't like Unity, it feels like going into a shop with very little on display and having to spell out for the shopkeeper the things I want instead of pointing and saying "I want to buy one of those". For the time being I can make 11.10 look like I like it.

1.Thankfully the gnome-fallback-session is available in 11.10 but not by default. In 11.04 Ubuntu Classic was available as an alternative. This time in 11.10 the package has to be downloaded as it's not by default available in Synaptic.

2.In System Tools>System Settings>Appearance, I prefer the lighter look of Radiance to the white on black of Ambiance.

3.Where's the waste bin gone? It's restored by installing the Gnome Tweak Tools package then Applications>Other>Advanced Settings>Desktop then select "Trash icon visible on desktop".

4a.Firefox and Evolution launchers on the top toolbar? Yes please. Alt+right click>Add to Panel>Custom Application Launcher. Commands for me here were firefox %u and evolution %u. The appropriate icons soon appeared.

4b. I like a Force Quit Launcher on the top toolbar. The command is now xkill and the icon is available if you left click in properties and select the Force Quit icon.

5. I don't like being automatically logged off if I leave the computer for 5 minutes etc. If I used my desktop in a public place then things would be different. System Tools>System Settings>Screen and undo the lock.

6. I did remove Thunderbird and continue with Evolution but if I were doing a fresh install I wouldn't bother. There's a bit of sorting out to do to get Evolution working properly again.

My 11.10 here looks like familiar Ubuntu and it's looking like the gnome-session-fallback package will be available in 12.04. If I used a tablet, netbook or mobile phone I'd probably prefer Unity. On a desktop I think Unity is yuc.

It's sticking with Ubuntu on desktops as long as there's gnome-session-fallback or something similar.
