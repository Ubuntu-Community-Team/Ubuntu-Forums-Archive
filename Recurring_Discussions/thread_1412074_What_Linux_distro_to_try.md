Debian is not harder to use than Ubuntu. Try out Debian stable or Testing and just use apt to build a system to your tastes. You can do that on Ubuntu as well by downloading the alternate install CD, and then just installing a command line only system.

OpenSUSE is excellent and the community is great. I got very fast and thorough help on the IRC channels when I needed it. I like it performance wise, but it seems to have more issues than what it used to.

Distro hopping can be fun, but performance wise I doubt you will notice much difference between them.

### Post by snowpine on 2010-02-22
> **swoll1980 said:**
> The op said stability was most important. Anyone that suggested a rolling release distro (Arch) failed automatically. A minimal install of Debian Lenny is what your looking for. While it is incredibly stable it will meet all your other criteria as well.

Yes and no. :)

A rolling release distro, is **by definition** unstable... in one sense of the word. (Meaning "a stone that is rolling down a hill cannot be called stable.")

In another sense of the word stable ("reliable, bug-free, easy to use for everyday tasks"), a well-designed rolling release distro can be more "stable" than a poorly-designed non-rolling release distro.

How can this be true? you might ask. One reason is that rolling release distros get new kernels, drivers, and app versions sooner, so they are more likely to support your hardware out-of-the-box. Another is that bugs get fixed quicker, what with new updates coming almost every day. I'm sure there are other reasons I can't remember.

I personally feel that Arch is more "stable" (in the second meaning of the word) than many non-rolling distros, for example, Ubuntu Karmic Koala. I would even say that a small set of users with hardware unsupported by older kernels/video drivers/etc. would find that Arch is more "stable" than Debian Stable/Lenny *for their purposes*.

While expert mode allows one to tweak to their hearts content.

Arch is almost like debian, except requires a bit more of an attention span to get things up and going. It makes up for it with even greater customizablity, a central configuration file (/etc/rc.conf) that handles almost everything; local, hostname, console font, loaded modules, daemons, and networking all neatly in one place. Arch usually has the most recent versions of packages availible in any distro, but arch packages are basically vanilla (without any distro specific patches) so they are far less prone to breakage and complications then with other distros. Also many customizaholics find the package management (pacman) to be superior to apt. The arch wiki is rivaled only by gentoo's, and the AUR speaks for itself. If one chooses ABS allows one to build any packages they would like (from the repos) directly from source rather then installing the binaries, which allows one to apply his/her own patches etc.

Fedora I would consider just as user friendly as Ubuntu, Mandriva, or openSUSE except the fedora team follows a different philosophy. Fedora is more focused on having the latest greatest packages, rather then stability, since its basically just redhat testing. Fedora is also more security oriented then the others, probably because it forks from redhat enterprise linux. I know a few sysadmins that swear by fedora, but for the average desktop user it might not be the best choice. It comes default with many network security tools.
