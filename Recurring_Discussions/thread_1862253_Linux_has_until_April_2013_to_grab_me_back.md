### Post by chessnerd on 2011-10-16
TL;DR - I'm looking for something good that Linux has to offer before Lucid Lynx loses support. I'm not impressed with Unity, Gnome 3, or Windows 8, so I'd like something good that the open-source community has done. I'm sure it exists.

This is my first post in a while, having last logged into the forums on June 27 (that's longer than I expected, but that's what the stamp says). I haven't really had anything to say about Ubuntu, and I haven't cared for 11.04 or 11.10. The direction of Gnome 3, design of Unity, and my lackluster feelings about KDE 4 have left me uninterested in Linux these past few months. I was content with using 10.04 until April and then just going to Windows, but now that Windows 8 is also having interface issues that Microsoft is die hard on implementing, it seems that every OS wants to change for the sake of change with no regard for what is intuitive or what the user wants. Now that Redmond has gone crazy too, I figure Linux deserves another look.

So Community, show me something good from Linux. I want to see something that gives me a reason to not sit on 10.04 until 2013. Show me that there is some distro that hasn't lost its way, some desktop environment that offers the same combination of simplicity and versatility as Gnome 2, some reason why Ubuntu 12.04 (which was not named something "Penguin") is good enough to switch to, anything.

You guys, and Linux in general, have until April of 2013 (when 10.04 loses support) to give me a reason not to jump ship entirely. Obviously, I'm just one person, you have little incentive to cater simply to me, but I don't think I'm alone here. I doubt I'm the only one clinging to 10.04 until it dies. If you are in the same boat, let others know that you too plan to jump ship. In the mean time, I'll still check to forums to see if anyone posts something good, if there is anyone who needs help with 10.04, and I'll use Lucid Lynx until I need to let Linux go or I see something that offers good computing.

Note: About Xfce, I like it, I use it on my old desktop. On that computer, I replicated Gnome 2's default environment (2 panels, 3 menus, app icons on the top, taskbar on the bottom, etc) and it offers a lot. I can see why Linus Torvalds and Eric S. Raymond switched to Xfce. The issue with Xfce for me is that its default apps (like Thunar) don't seem to play nice with networking and I don't want to spend hours working out those issues. If there is a distro out there that uses Xfce and works well out-of-the-box, I'll give it a try, but I don't want to just be told to uninstall Gnome 3 for Xfce (although that may be the only option).

### Post by angryfirelord on 2011-10-16
> **TeoBigusGeekus said:**
> Go for lucid server edition and install a desktop environment: 5 years of support instead of 3.
No, the packages that make up the server edition are supported for 5 years. Installing the server edition and then installing ubuntu-desktop will not give you 5 years of support on the desktop as there is no "Gnome server edition". Likewise, if you install mysql on a Lucid desktop, mysql will still get 5 years of support.

If you *really* want Gnome 2, install Debian Squeeze, which should get security updates until 2013 or 2014, depending on when the freeze occurs. If that's not long enough, you can install Scientific Linux or CentOS 6, which will get updates to at least 2016.
