### Post by Copper Bezel on 2011-04-07
The Unity Launcher works more like the Windows 7 taskbar than the OSX dock, and it's on the left because that's supposed to be more accessible. The Scale effect *is* a clone of Exposé with the same behavior for clicking on an item with multiple open windows, but the Launcher doesn't have any of the OSX dock's weird application-oriented thinking (clicking an icon doesn't bring all the application's windows forward, minimized windows don't create their own weird little space in the Launcher, etc.)

The top bar has barely changed from the default Gnome settings, and the notification area uses monochromatic icons because notification areas use monochromatic icons now, regardless of the system. The window controls were moved to the left because Ubuntu is putting *everything* on the left for ease of access and because it keeps them out of the way of the notification area. The global menu was an attempt to save vertical space, and it's part of the same line of thinking in which maximized windows plug into that same top bar, following precedents set back in NBR. 

It's one thing to say that Unity looks quite a bit like OSX. It's another to say that it's attempting to copy OSX. Evey feature in the Unity interface was implemented for specific usability reasons.
