### Post by gilgongo on 2009-03-28
If I had to pick the one thing that I think makes Ubuntu (well, Debian) the best OS for me, I'd say it was package management. I use Windows at work, and installing software is a total pain. Here's what you typically have to do:

1. Find the software on the net: Google for "text editor" for example.

2. Plough through stupid ad-encrusted download site.

3. Download the wrong thing becuase you got tricked by some ad for a registry cleaner and hit the wrong button.

4. Find the installer on your desktop, or is it going to be in "My Downloads"? Hunt, hunt.

5. Run the installer. Click "Next" about three to five times as various unintelligble "options" come up (eg "Install for all users?" Huh?)

6. Run the software. At which point you may need to spend another half hour purchasing and keying in a license code or something).

7. Six months later: receive software update notification from vendor. Sometimes this is while using the software, so you thereby get interrupted in what you were doing and (Adobe!!!) you may even have to restart your computer to complete it. Sometimes this is by email, in which case you have to repeat steps 2-5 all over again.

Compare this to Ubuntu:

1. Run synaptic, search for "text editor"

2. Hit "install"

3. Run software.

4: Six months later: you receive a software update automatically one day and hardly notice.

Amazing that Microsoft, being a monopoly and all, haven't been able to centralise Windows software distribution. But then, when you're a monopoly, you have no reason to be interested in things like that because people will still keep buying your stuff because you're a monopoly. And that, my friends, is why monopolies are bad. 99% of people who use computers have no concept of how things could be any different, because 99% of people use Windows.
