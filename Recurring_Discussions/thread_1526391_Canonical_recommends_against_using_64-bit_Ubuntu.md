### Post by Splooshie123 on 2012-03-08
Correct me if I'm wrong:
32 bit is recommended because most processors can only handle 32 bit "chunks" of data. A side effect of 32 bit I think is that you can only use a maximum of 3-4 GB of RAM.

64 bit processors can handle 64 bit chunks of data as long as the software is programmed to use that functionality. It can also handle more RAM than 32 bit (I use 64 bit on a Mac because I have 4 GB of RAM).

In a nutshell: 32 bit is recommended because it is more common and therefore better supported. Most applications are designed for 32 bit.

Check this out: [http://en.wikipedia.org/wiki/64-bit](http://en.wikipedia.org/wiki/64-bit) and [http://en.wikipedia.org/wiki/32-bit](http://en.wikipedia.org/wiki/32-bit)

For all installations, you are better off having /home as a separate partition. That way you can keep all of your data when you do a fresh install.

These guides will help you do that:

[https://help.ubuntu.com/community/Partitioning/Home/Moving](https://help.ubuntu.com/community/Partitioning/Home/Moving)
[http://blog.mattrudge.net/2010/07/16/moving-home-to-its-own-partition/](http://blog.mattrudge.net/2010/07/16/moving-home-to-its-own-partition/)

As regards installed applications, you can output a list of installed software to a file by typing the following:

```
dpkg --get-selections > software-list
```

To then reinstall it on the new installation:

```
dpkg --set-selections < software-list
dselect update
apt-get dselect-upgrade show
```

### Post by grahammechanical on 2012-04-27
A lot of people don't know much about their hardware let alone know if the CPU was 32bit or 64bit.

So, it made sense to recommend 32bit Ubuntu because a 32bit operating system will run on both 32bit and 64bit CPUs but a 64bit OS will only run on a 64bit CPU.

It was thought that it was better to give a potential user a good install experience by recommending a version of Ubuntu that installed on either 32bit or 63bit CPU, than risk giving a wrong impression because they chose the wrong version of the OS.

This recommendation is going to change over the coming months because most machines on the market have a 64bit CPU. So, the recommendation will in future be 64bit Ubuntu.

Please do not read more into this recommendation of 32bit Ubuntu than is necessary.

### Post by Miljet on 2012-04-28
I don't use a separate /home partition on my machines. I know most people recommend it, but I feel that you wind up with configuration files for programs you no longer have or the version has changed. And I usually have a bunch of packages that I tried and didn't like that need removing anyway. So I just copy over my Pictures, Documents, Music and so forth and install everything else. It takes a bit more time, but I feel that I have a "cleaner" system.

When you install the new version, you can let it install Grub and it will automatically pick up the older version. Or you can use the existing Grub and run "sudo update-grub" to pick up the new version. I usually use the first option.
