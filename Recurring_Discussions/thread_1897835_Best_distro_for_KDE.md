### Post by Dragonbite on 2011-12-20
Funny thing is with Linux distributions, there isn't all that much different between them if you are working on the surface.

**_Fedora_**
I like Fedora for KDE because it is pretty much the upstream, with very little "modding".  This means if I go to another distribution running KDE I won't find many, if any, features which Fedora included which isn't available elsewhere.  

Unfortunately for some of those programs that are desktop environment specific, Fedora usually defaults to Gnome.

An example of this is Dropbox.  There isn't a KDE version of Dropbox for Fedora (that I have found), but I did find installing the server version runs just fine, you just don't get the indicators on the folder in the Dolphin, but you get the system tray icon with status, pause and quit abilities, as well as it does still auto-synchronize.


**_openSUSE_**
OpenSUSE is a very solid distribution for KDE and they are very involved with KDE's development.  I remember a long time ago seeing the menu which you find in KDE 4 being included in openSUSE KDE 3 even before KDE 4 came out!

Plus if you need a number of off-the-beaten programs for KDE, there is a good chance openSUSE has it in either their repositories, KDE-apps or openSUSE Build Service (OBS).

And if you want to fool around with a rolling release, their Tumbleweed repository handles that, or you can go traditional.

Plus openSUSE includes Yast, which gives you a GUI  interface for system management and software installation and updates.


**_Kubuntu_**
Kubuntu, the biggest benefit to them I see is their "diet" mode (I can't remember what it's called). It turns off a lot of services to release resources and although I didn't have as much eye-candy it did make a significant improvement on my laptop!

Other than that, I don't have much experience with Kubuntu.  I dont' know if they have a client for Kubuntu to access Ubuntu On but I did hear they are working on the software store for it.
