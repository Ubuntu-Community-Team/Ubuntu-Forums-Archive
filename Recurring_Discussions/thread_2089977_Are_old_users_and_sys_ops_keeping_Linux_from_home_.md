### Post by Morbius1 on 2012-11-26
You are comparing apples to soy beans I think.

Windows and OSX are software products that are shipped. Each component is under the control of their respective organization which employs all sorts of folks ( business analysts, requirements people, QA, etc .. ) to ensure that the user is presented with their vision of how an integrated system should look and function.

A given Linux distribution is a collection of software packages each of which selected at a snapshot moment within their own development cycle and under the central control of no one. A Linux Distro isn't shipped it's released. There are thousands of packages within the repository so there is no way a given Linux distribution can have the kind of integration tests that takes place in the real world to ensure that every one of them works flawlessly within the OS environment. The number of QA people would outnumber the developers. That's what the Linux user is - it's Linux's QA department.

Let's face it folks, those of us that use Linux as our primary OS are part of the lunatic fringe. I would have to plead guilty to having a lower expectation of Linux software than that from Microsoft or Apple. On the other hand as one who has used Linux for more than a decade now the fact that it comes together and functions as well as it does is nothing short of a miracle.

This is all from the perspective of Desktop Linux though not Server Linux. Server Linux in the hands of adults like Red Hat have proven the superiority of Linux over the rest.

### Post by texpat on 2012-11-26
Linux comes from the console, so many commands are focussed on text input. Converting that to GUI will make it a) less flexible and/or b) the GUI too cluttered if you want to take into account all the options a given command has. And no-one wants to loose flexibility. Hell, MS had to introduce Powershell to offer something like Linux's command line interface.

So if you come from Windows, it is inevitable to learn a few commands on the console, because once you start configuring and tweaking, you're bound to run into something you want to do that has no GUI, yet. You know why most Linux help around the Internet is in command line form? **Because it's easier this way**. It's easier to
> Type this command into the terminal:
```
command
```
(then copy and paste the result)
than
> Open the Dash, type "command", click the first result. Or if you are not on Unity, then click the menu, navigate to System > Blabla > Command. Choose the first tab, click this button, click that button click that tab, etc. etc. etc.
