### Post by PhilGil on 2010-02-27
I think Ubuntu is one of the most terminal-free Linux distros available.  There are GUI tools to do just about everything a terminal command can do.

However, online Linux support/tutorials are much more likely to use terminal commands instead of a GUI.  I think experienced Linux users just like using the terminal - and there are good reasons why.  Troubleshooting seems to go much faster when using the terminal.  Even as a Ubuntu newbie I find software installs to be more convenient using the terminal rather than Synaptic (as long as I know the exact name of the package I want to install).

Windows is not entirely command line free, either.  It's not that unusual to have to open a terminal in Windows when troubleshooting devices or network connections.  I'll bet almost every Windows user who has called their ISP about a connection problem has been asked to open a terminal and ping an IP address.

True, but Windows does give (at least in some situations) more error info via the GUI than Linux (or at least Ubuntu) does. When an application gets killed by the OS in Windows, you generally get an error box saying "there was an error", and it will often have a "details" button, which will give you some info as to what the error was. When an application gets killed by the OS under Linux, you tend not to get a bit of information (the application just tends to close without so much as an error dialog) unless you were running it in a terminal, so you have to go back and recreate the error if you can.

So in Windows, you can often get information on an error without having to recreate it. On Linux, you most often can't, unless you were running it in the terminal in the first place.

I still like Linux better...
