### Post by narnie on 2010-04-13
For those who want an install script, I'll include on here.

If you already have a failing setup with dansguardian, tinyproxy, and firehol, run this first:

```
$ sudo aptitude purge dansguardian tinyproxy firehol
```

then run my script if you like with

```
$ chmod a+x dansguardian_install
$ ./dansguardian_install
```

restarter all servers with:

```
$ sudo service tinyproxy restart
$ sudo service dansguardian restart
$ sudo /etc/init.d/ubuntu_ce_firewall restart
$ dansguardian-gui
```

You may have to stop and restart daunsguardian. Barring that, reboot and all should be well.

The scripts work great on my Acer Aspire one netbooks running UNR and my linux mint machines. If you have a problem with it on your setup, please let me know.

Yours,
Narnie

### Post by rickyrockrat on 2010-12-08
I found this thread and ran into the content issue on a Hardy server. This is how I fixed it - the old fashioned way, from source. I couldn't use any of the pre-compiled binaries because they rely on a newer version of the pcre, and *lots* of apps use that on my server, which I don't want to break. It takes too long to set a new one up (Days for me)!


1) Download libpcre:

```
wget http://downloads.sourceforge.net/project/pcre/pcre/8.10/pcre-8.10.tar.bz2?r=&ts=1291802744&use_mirror=iweb -O pcre-8.10.tar.bz2
```

2) untar and build it:
```
tar -xjf pcre-8.10.tar.bz2
cd pcre-8.10
./configure --prefix=/usr/local
make
sudo make install

```

Then download dansguardian, make and install it:
```

wget http://dansguardian.org/downloads/2/Stable/dansguardian-2.10.1.1.tar.gz
tar -xzf dansguardian-2.10.1.1.tar.gz
cd dansguardian-2.10.1.1
./configure --prefix=/usr/local --with-proxyuser=dansguardian --with-proxygroup=dansguardian --localstatedir=/var --sysconfdir=/etc
make
sudo make install

```
Those options above allow dansguardian to run in /var and use configs from /etc...If you want to move everything to /usr/local, then you can omit the --localstatedir and --sysconfdir, but I didn't want it any more confusing that it already is!

And make sure ldd knows about the new library:
```
sudo ldconfig
```

One tiny final change is to change the path to the new binary in /etc/init.d/dansguardian. The top of the file should have this (the local/ was added):
```
DAEMON=/usr/local/sbin/dansguardian
```

That worked for me on a standard Hardy x86_64 (but it will work on any platform with a compiler).  If you put it in /usr/local it won't conflict with the apt installs.
