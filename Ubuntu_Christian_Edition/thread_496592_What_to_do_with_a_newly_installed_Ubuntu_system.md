Searching through my computer found some worthless junk that I must get rid of. I did a few things to take care of these problems.

```
chris@ubuntu:~$ find / -iname '*christian*' -print 2>/dev/null -exec shred -v {} \;
/usr/share/calendar/calendar.christian
chris@ubuntu:~$ 
```

After doing this command, I found that the problem still existed. The file had been untouched. I looked for the cause of this.

```
chris@ubuntu:~$ file /usr/share/calendar/calendar.christian 
/usr/share/calendar/calendar.christian: ASCII C program text
```

I found it! Here's the problem.

```
chris@ubuntu:~$ ll /usr/share/calendar/calendar.christian 
-rw-r--r-- 1 root root 1522 2006-12-20 07:13 /usr/share/calendar/calendar.christian
chris@ubuntu:~$ 
```

Don't worry if you do not understand ll. It is just an alias for ls -l.
Now that Travis and I have found the root of the problem, we began to shred.

```
chris@ubuntu:~$ sudo find / -iname '*christian*' -print 2>/dev/null -exec shred -v {} \;
Password:
/usr/share/calendar/calendar.christian
chris@ubuntu:~$
```
