### Post by cyberdork33 on 2007-10-16
Running natively has it's advantages as the OS has direct access to the hardware. Thus, you have a system that is just like your Dell, where THE OS running is Linux. Running in a virtual machine is just that... virtual. The OS 'sees' the emulated hardware that the virtual machine provides (Which does not include 3D graphics as of yet). A VM is quite a bit easier to setup linux for as there are normally sufficient drivers for all the virtual hardware. If you want to have a linux system that is there to use when you need it, and are not planning to really utilize it as an OS on your system (maybe just to run that one application or something), then I would recommend the VM, but if you want to be able to run a full linux system, then install natively.

There are many people that are only drawn to Apple by the hardware design, not the OS, and 'highly polished GUI' is a matter of opinion. In linux, you make the interface look however you want. Even quite like OSX. I can't say that multimedia apps for linux are quite up to par with those available in OSX, but that is not to say there is nothing to use. Checkout [Ubuntu Studio]("http://ubuntustudio.org/").
