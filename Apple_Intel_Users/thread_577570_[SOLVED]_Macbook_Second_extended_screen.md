### Post by david_edmundson on 2007-10-16
Gutsies Intel graphic drives now support something called XrandR. This allows you to change montiors/resize stuff on the fly. This makes things far far easier, and is possible without any Xorg hacking. 

I suggest you look into XrandR or Grandr(a graphic interface to it) if you want hotplugging to work.

To use Xrandr simply type "xrandr" into a command line, it will list all attached screens. To make the other screen active simply type "xrandr --output VGA --auto" (WHERE VGA is the name of the other screen)

Anyway, as for your crash. Change your driver from i810 to Intel.

Also the Intel graphic driver has limited memory you cannot have a very high res and 3D acceleration. For now, remove or comment out this line:
Load            "dri"

### Post by Zaff on 2007-10-17
okay so here's what I did : 
I first change the resolution of my external monitor using the command line like so : 
```
xrandr --output TMDS-1 --mode 1680x1050
```, then I decided to give grandr an other chance.
actually if you use grandr without clicking on the display icons to the left you can avoid it crashing. So I just used it to change the layout and place my external screen on the left from my macbook screen, just before realizing this is easely done using command line a such:
```
xrandr --output TMDS-1 --left-of LVDS --mode 1680x1050
```
It works well. Last thing is just to move the menu bars around to have them on the monitor you want. For some reason they get put on the external monitor by default.
