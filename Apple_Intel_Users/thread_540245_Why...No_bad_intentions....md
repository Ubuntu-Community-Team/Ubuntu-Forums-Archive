### Post by cryptocoryne on 2007-09-01
A lot of things that I like, old X11 apps, just don't work so well in OS X. Yes, I know I can use Fink. But Ubuntu is faster and right now Gnome (under gutsy) has better eye-candy than Tiger! Strangely, since I upgradeed to gutsy, even the screen fonts look better than in Tiger too!

There is a feeling that all the parts just fit together with Ubuntu and dpkg that I don't get from OS X.  i.e. Tiger's interface seems bloated in some ways (Dashboard) and inconsistent in others (Aqua vs. Brushed Metal). There is no central package manager in OS X and I have no idea how to undo whatever the scripts in .pkg files do.  

It also drives me crazy how un-customizable OS X has always been. Yeah, I know, there are big fat books on this and shareware apps. But the Ubuntu documentation/community is there to help and it does.  Using OS X after getting used to linux is like visiting the planet Ferengi: 3rd parties nickel and dime you to death. Want to change the color of the menubar? Buy a $20 shareware app.   

The Finder has sucked (no nice thumbnails, overall clunky feel) since the beginning of OS X and Apple seems to have no intention of fixing it. The UI looked better in OS 9. It's hard for me to quantify; OS 9 was just more fun.  Actually, that's something I've been meaning to look into, how to make linux X11 desktop a clone of OS 9....

### Post by Greywhind on 2007-09-01
I'm dual-booting Mac and Ubuntu for these reasons:

1. I love Mac's feel and many of the programs that run on it, and I think their hardware is top-notch. Mac also gets support from some games and programs that Linux doesn't.

2. The same is true of Linux - it can be nicely customized and has many programs that Mac does not have.

3. Gaming - I get Mac games, Linux games, and some Windows games (with Wine.) Without Linux, I'd be restricted to Mac games.

4. Programming - With dual-booting Mac and Linux, I can compile code for Mac, Linux, or Windows (cross-compiling in Linux). Wouldn't be nearly as easy/possible with only Mac.
