```

# Comment the next line to disable ACPI suspend to RAM
ACPI_SLEEP=true

# Comment the next line to disable suspend to disk
ACPI_HIBERNATE=true

# Change the following to "standby" to use ACPI S1 sleep, rather than S3.
# This will save less power, but may work on more machines
ACPI_SLEEP_MODE=mem

# Add modules to this list to have them removed before suspend and reloaded

#

MODULES_WHITELIST=""

# Should we save and restore state using the VESA BIOS Extensions?
SAVE_VBE_STATE=true

# The file that we use to save the vbestate
VBESTATE=/var/lib/acpi-support/vbestate

# Should we attempt to warm-boot the video hardware on resume?
POST_VIDEO=true

# Save and restore video state?

# Should we switch the screen off with DPMS on suspend?

# Use Radeontool to switch the screen off? Seems to be needed on some machines

# afterwards
# DISABLE_DMA=true

# Switch to laptop-mode on battery power - off by default as it causes odd

# Spindown time on battery
```

### Post by BIGtrouble77 on 2007-11-20
I have the same machine and these are the only values I had to change:
```

# Should we save and restore state using the VESA BIOS Extensions?
SAVE_VBE_STATE=false # was true

# The file that we use to save the vbestate
VBESTATE=/var/lib/acpi-support/vbestate

# Should we attempt to warm-boot the video hardware on resume?
# POST_VIDEO=true
POST_VIDEO=false # was true

# Save and restore video state?
SAVE_VIDEO_PCI_STATE=true # was false

# Should we switch the screen off with DPMS on suspend?
USE_DPMS=false # was true

```

I just followed this post:
[http://ubuntuforums.org/showpost.php?p=3785681&postcount=20](http://ubuntuforums.org/showpost.php?p=3785681&postcount=20)

This seems to have fixed all of my issues.  I also have the screensaver set to just a black screen.

### Post by ananunes on 2007-12-14
In order for the backlight to turn on, you only have to make this change in file /etc/default/acpi-support:
```

# Uncomment the next line to switch away from X and back again after resume.
# This is needed for some hardware, but should be unnecessary on most.
 DOUBLE_CONSOLE_SWITCH=true

```

I now suspend and resume working perfectly in my 3rd generation Macbook running Ubuntu 7.10 (Gutsy Gibbon) after patching and compiling the kernel as described in [http://www.michelem.org/2007/12/09/how-to-install-ubuntu-gutsy-on-a-macbook-31-generation/]("http://www.michelem.org/2007/12/09/how-to-install-ubuntu-gutsy-on-a-macbook-31-generation/")

Everything is now working :)
