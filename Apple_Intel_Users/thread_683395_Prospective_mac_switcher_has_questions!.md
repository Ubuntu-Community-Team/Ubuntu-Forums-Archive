### Post by cyberdork33 on 2008-01-31
> **wahr said:**
> I've been compiling a suite of apps which, if they aren't too rough around the edges, could replace the ilife apps i'm using. there are already some bundles out there that are designed to be iLife-like. [F-Spot]("http://f-spot.org/Main_Page") is a photo organizer (iPhoto Replacement), [Kino]("http://www.kinodv.org/") is a great movie editor (iMovie), and [Banshee]("http://www.banshee-project.org/Main_Page") is supposed to be very similar to iTunes. F-Spot is bundled with gnome, so should be installed by default in Ubuntu. There is also a media-centric distiribution of Ubuntu called [Ubuntu Studio ]("http://ubuntustudio.org/")that is geared toward media production and has several other audio / video tools.

OK, on to the other stuff, I will try to help where I can:
> **wahr said:**
> -mail (i know there are many clients, but i'm interested in the simple interface, client side spam filter, and 'smart categories' to auto sort  my mailing list subscriptions) Thunderbird is an excellent mail client. It does have a junk filter, but I cannot vouch for how great it is, because gmail filters most of that for me anyway. It has something similar to smart folders. You can create a "Search Folder" by running a search on your mail, then right-clicking and saving the search as a folder, or you can create a search folder from scratch from the menus (File > New > Saved Search...). The best thing about Thunderbird is that it is cross-platform much like firefox, so you can use it on Windows, Linux, and OSX. 

> **wahr said:**
> -icalI am not too familiar with Calendar Programs. I think that the closest match is the upcoming app from Mozilla (again) called Sunbird. It is still not nearly finished though. There are some add-ons for Thunderbird to add calendaring as well. There is a very old linux app called ical out there, but it is not related to Apple's iCal. This is one app that I fell Linux is a bit lacking in (until Sunbird gets together).

> **wahr said:**
> -macpar deluxe I have never used par, but from what I found, the original macpar was based on an [open source engine]("http://parchive.sourceforge.net/"), so I would have a hard time believing that there was not a linux utility that will work with it. (i think gpar might be what you are looking for. It is in the repos).

> **wahr said:**
