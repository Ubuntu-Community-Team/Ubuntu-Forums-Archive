### Post by Herman on 2007-10-08
Hello goldan,
Ubuntu and open source software works in a completely different way to Windows and its commercial software.

It's true that open source software does seem to take longer to load, that's because open source software is 'polite', and it doesn't half load itself in your RAM as soon as you start your computer. 

Most proprietary software that you add to Windows will try to automatically take a piece of your RAM by running in your start menu so they have a head start loading when you click on them. If you add too many programs to Windows, Windows will slow down and you'll eventually need more RAM, or else delete some programs from your startup group, and then you'll find they'll take longer to load, just like open source programs do.
Some of the worst programs are antivirus programs that run constantly in the background. You can't do much about those in Windows.

With Ubuntu you can add any amount of programs and your computer will still be the same speed. No open source programs are 'resource hogs'.
You can add every program in the Ubuntu repositories and Ubuntu will still run the same speed!

Another thing is it won't slow down over time, because it doesn't collect junk from the internet like Windows does. I have lots of friends who call me for help when their Windows gets loaded with junk and slows down to a crawl, (with that flashlight looking around before it will open anything). 
One friend of mine (who has Ubuntu now), had a Windows computer that took about an hour just to boot up! :lolflag:

Ubuntu never suffers from internet junk, it will be the same speed for it's lifetime.

I have also found that I can open much larger files in Ubuntu than I can in Windows. I have a nice big map of my district that I can open in Ubuntu, It's important for my work. It's 4770x4400 pixels and it takes up 12.7 mb of disc space. Ubuntu can open that and I can pan around in it even in my smallest computer, but Windows, even in an average computer, can barely open it at all. When it does manage to open it, panning around in it is painful in Windows. The monitor keeps going blank as the image slowly redraws. In Ubuntu I can just zoom in and pan around my map as fast as I want.

So Ubuntu and Windows are completely different, and they work completely different, the software (programs) behave completely different, and the operating systems perform differently over a time period.

I still prefer Ubuntu, but I do admit that a clean new Windows without too much added software can be very nice and snappy!
The reality is, it's hard to keep it that way for very long if you use it on internet.

Ubuntu is my choice for an operating system.
Regards, Herman :)

### Post by Herman on 2007-10-09
Well it depends what it is you are measuring. 
I myself often do notice that it does take a second or two to open up an Open Office.org spreadsheet, even in a fast computer, when Ubuntu is running normally, (in a perfectly healthy system).
I would guess (although I haven't times it) that an equivalent spreadsheet program in Windows in the same computer would open about 1/4 the amount of time.
But it is only a second or two, and in my opinion it is tolerable when I understand that open source programs are designed to be polite and not use up all my resources.

I have an idea, maybe it would be interesting if we actually timed (with a stopwatch), how long it actually takes to open certain programs in Ubuntu compared with Windows in the same computer, in computers with various hardware and see what the differences are, just for fun.

After closing the spreadsheet, to open the same spreadsheet again for the second time in succession, took 15 seconds, exactly half the amount of time.

Opening a second spreadsheet in another desktop when one was already open took only 2 seconds, so there is quite a remakable difference in speed depending on what you are doing eh? :)

To open a Microsoft Works Spreadsheet takes just 9 seconds for the firsdt time, even when I had to pause to click the EULA 'I agree' button, (this is a brand new clean install).
Second time opening after closing took just 2 seconds.
To open up a second spreadsheet when one is already open took only about 1 second.
