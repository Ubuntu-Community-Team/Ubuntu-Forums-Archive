### Post by kidders on 2007-08-22
Hi there,

Since you've gone a whole day without a response, I thought I'd at least *try* to help. I'm using the old (white) bluetooth keyboard without any problems on my Linux box. Although I can't say so with *100%* certainty, I would be frankly astounded if the new one was any different. 

  When it comes to bluetooth HID devices, anything that doesn't come with its own "bluetooth" dongle should work fine, by and large, in MacOS and Linux (I can't speak to Windows), because they all tend to adhere to established standards. I intend to buy the new Apple model at some stage, and tbh it never entered my head that there might be any compatibility problems. You see, in general terms, Apple tend to do things "right". For example, where some companies might use some silly, made-up, driver-dependent scheme, the media keys on Apple keyboards all emit standard scancodes, making them trivial to configure. I can only assume that the new model works the same way.

Anyhow, sorry I can't provide you with an iron-clad answer, but I did want to at least point out that *not* making their new bluetooth keyboard pretty universally compatible would be difficult for Apple to justify, given that their old model works very nicely.

### Post by incubus on 2008-01-29
Hello,

I'm typing this post from a non-apple computer, with the apple bluetooth keyboard, which I have been using since early October.  For my needs this little thing is perfect.  I'd like to throw in my opinion on it as well.

1. I can't use bluetooth at boot, so if I need to do something in the BIOS I do need a wired keyboard.  I don't dual boot, so in 99.9% of the cases this is fine.

2. Yes, the Fn key doesn't work and it probably requires a patch in the Kernel.  I'm sure somebody's working on this.

3. There's the caps lock thing, which is annoying.  It has a built-in system to prevent accidental hitting.  In my case I use caps lock as escape, so this is horrible.

4. I had a problem with repeating characters (e.g., I'd type something and sommmmething would appear).  I just had to decrease the repeating rate by 1 level and never again did it happen.

Now besides these downsides, I absolutely love this keyboard.  I type much faster and I like the fact that I can easily change where I'm sitting, just have to turn the monitor.  It works fine with Ubuntu, and regarding the special keys, remember you can just reassign the F1-F12 to whatever you want, or change shortcuts.  This is Linux.

Oh and 'command' does work, and so does 'eject', you just have to assign it to something properly.  In my case, I use it as 'control', but you can also make it a 'windows' key.  Search for tutorials on xmodmap for this.  Or ask us and we can try to help.

### Post by cshobe on 2008-05-14
1.  Apple's don't need a BIOS, the EFI works the same.  The EFI on Macs fully supports use of a bluetooth keyboard (Apple or otherwise) at all points of the boot process, however this is usually done by MacOS updating the EFI when the keyboard is paired under that operating system.  I'm not sure what linux users should do if they did not formerly have MacOS and the same keyboard as I did...

2.  Every key on the keyboard works *JUST FINE*.  There is a utility called pommed, which is available from apt, which is specifically for implementing full support for Apple's special keyboard buttons.  I use fn+f11 to volume down (and get a nice screen popup in KDE similar to the one is OS X), etc.

3.  The repeating characters issue is damn annoying but if you do a quick google search you'll find it happens to loads of bluetooth keyboards.  This seems to be a general bug in the bluez stack since it affects lots of keyboards, but it's hard to say.  I just know it doesn't happen on other platforms.  Regardless, a quick increase of the repeat rate delay hides the issue enough to make it never bother you again.  It's unfortunate because I highly prefer a really fast repeat rate, especially for arrow key usage, but it's not the end of the world.

4.  It's not steal, or even steel.  It's aluminum and plastic.

I type on this keyboard, paired with a MacBook Pro running Ubuntu, for 8+ hours every single day and have been for several months now.  It really won't make any difference at all what the host hardware is, except that the EFI or BIOS needs to have native bluetooth HCI support for the keyboard to work at those levels and for grub.  I have not a single complaint about this keyboard - once I became accustomed to a few new shortcuts to deal with the missing keys, it's GREAT.  It's actually funny because I'm actually more efficient now that I'm not reaching over for the insert/home/pageup/etc. all the time but rather using combinations that, even though requiring a combination press, are more efficient since my hands don't need to move.

I have verified that the keyboard does indeed feature built-in capslock repeat protection, which makes the key unable to be used really rapidly.  I imagine that could make using the caps lock as a control key, etc. really irritating but if you just use it as a caps lock it's a non-issue.

I do recommend that linux users reverse the control and command keys in the X keymap, simply because then the keyboard shortcuts will be more familiar to using OS X, and there's a command key on both sides of the keyboard whereas there's not for control...Apple's use of control and meta keys is opposite of the Windows and Linux world (they had it originally to start with, like Solaris, whereas Windows added support for meta as an afterthought).

Anyways no offense to anyone, just wanted to clear up some of the falsehoods and misconceptions in this thread.  Good luck!
