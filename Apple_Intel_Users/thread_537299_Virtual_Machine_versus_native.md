### Post by ripdog on 2007-08-29
In a virtual machine, everything works slower, has a large tax on the host system, ect. 


However, you still have access to your host OS, this can be useful when installing distros with text mode install and you encounter an error, and the emulated hardware is MUCH more likely to be recognized and work correctly then whatever your physical hardware is. 

If you have a Mac, you may know about the USB keyboard on linux boot issue, when USB devices arent started for isolinux/bootloader, and you cant get past isolinux/ bootloader. This problem is bypassed in a virtual machine, as i think Vmware emulates a PS/2 keyboard/mouse, which is natively supported by... everything. :P

because of the serious speed issues, I personally think installing on a virtual machine isnt really a proper install, and should only be used to poke around a OS, or run one or two essential programs that dont run in OS X.

Oh crap, just saw you said apart from performance issues. Oh well. No 3D acceleration is true, so no Compiz-Fusion for you in VM.
