### Post by Henrik on 2006-09-28
Hm, that's an interesting use case. At the moment out sticky key solution does not actually set the shift state of the keyboard so we avoid interfering with other applications. It just shifts they keys you press on onBoard.

We could implement this, though it might interfere with the built in StickyKeys facility.

On the layer that has function keys, there is a button used to change over the mouse buttons (left to right). Perhaps that would be the right place for a shift-click feature (and alt and ctrl)?
