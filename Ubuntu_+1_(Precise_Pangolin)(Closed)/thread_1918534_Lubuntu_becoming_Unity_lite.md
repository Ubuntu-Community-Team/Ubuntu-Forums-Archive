Of course if your machine(s) have only CD or CDRW, that's not an option.  CDRWs seem odd; I've not had much luck with them.  I know a lot of older machines will not boot from USB drives.
You can boot a .iso directly from the .iso file on your hard drive, or a USB.  No CD or DVD required.  Example:
```

gedit 40_custom
#!/bin/sh
exec tail -n +3 $0
# This file provides an easy way to add custom menu entries.  Simply type the
# menu entries you want to add after this comment.  Be careful not to change
# the 'exec tail' line above.
menuentry "Ocelot sda5 /boot/iso" {
set isofile="/boot/iso/oneiric-desktop-i386.iso"
loopback loop (hd0,5)$isofile
linux (loop)/casper/vmlinuz boot=casper persistent iso-scan/filename=$isofile noprompt noeject
initrd (loop)/casper/initrd.lz
}
save 
sudo chmod 777 40_custom
sudo cp -dpuv 40_custom /etc/grub.d
sudo update-grub

```

Works fine for testing typical oversized .iso files.
Now if you want to live dangerously like me, use sudo gparted to prepare a partition, boot the .iso, then use df to find the isofile.  In the above example, /dev/sda5 so do:
sudo umount -r -l /dev/sda5
then try an install.  Alpha installs are famous for blowing up so back up..
