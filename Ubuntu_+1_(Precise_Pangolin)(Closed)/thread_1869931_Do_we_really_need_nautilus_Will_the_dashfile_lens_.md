----Post: **Is the dash becoming ubuntu's new _central hub_**?

I see nautilus converting into a **legacy app** after maybe 1 or 2 years..

**File lens is getting good**:
[http://www.omgubuntu.co.uk/2011/10/new-movie-scope-brings-rich-meta-data-filters-to-movie-files-in-ubuntu](http://www.omgubuntu.co.uk/2011/10/new-movie-scope-brings-rich-meta-data-filters-to-movie-files-in-ubuntu)

-Many dont really like or use this new nautilus that much anymore thanks to the dash/file lens.

-Many are **disappointed** at gnome for taking features away and not wanting to bring them back.

-The file lens still needs a few features (tooltips, show file properties, select/cut/paste and breadcrumbs; yea thats basically all file-lens needs to be a full blown **file-manager-lens** and i see the gap closing in). So **it could well substitute for basic home folder stuff**.

-We dont even need it for mounting drives, as the unity launcher does this already.

-Also **looks out of place** with the dash. The less interfaces new user need to learn the better.

-Would become the best looking file manager out there.

-A **trash can lens** can even substitute the trashcan.

-Even a **system settings** lens can substitute the control panel

It will still be included, but probably as back-end and legacy to handle more **advance operations** ("the advance file manager"), old users who prefer it or browsing the root filesystem. But for basic stuff it will be used less and less. Oh and this is just to complete the unity/dash vision. All other *ubuntu derivatives will stay the same.

Dash may become the **killer app** in itself (the central hub).

some examples of evolution would be:

- File-lens to **File-manager**-lens
- Music-lens to [**Music-player**-lens]("http://lh3.googleusercontent.com/-rktxlGPG6IU/TpwcnTvZFWI/AAAAAAAAGUE/lZsxF1L44HU/sound.jpg")

But depending on your usage, you could install or remove either one, making ubuntu more customizable.

When we look at how modern devices interfaces is evolving, it makes sense that the desktop will also absorb this trend of simplification / unification of activities under one easy / known interface. While I initially hated this "let's make computers desktops look like a freaking iPod" trend, I can now understand the benefits for most standard users. It actually allow more people to use a PC.

A user can type a search for "Settings", "Mouse", "My3DGame", MyWorkStuff.doc", "MyFavoriteSong", "ChildrenPictures", etc all in the same place. No one will get lost trying to remember where a doc was saved or how to adjust accessibility settings. There's nothing else to learn. It's good for the standard users.

And people like us will most likely continue using Nautilus and other of the "complex" stuff. No one will develop a gdb lens :) I need to have my stuff into a rigid file structure. I need to open two or more file navigators side by side and compare / move / copy stuff. But most users need to perform one simple activity. IMHO Nautilus will be legacy (or maybe just an alternative download) soon.
