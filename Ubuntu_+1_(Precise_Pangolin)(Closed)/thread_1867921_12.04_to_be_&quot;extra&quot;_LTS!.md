### Post by ErkDemon on 2011-11-06
> **dcstar said:**
> I dread the end of the Gnome 2 interface with 10.04 LTS that some of us have spent a lot of time using productively just to be offered either a second rate "Fallback" interface or a touchscreen interface because it is the current fad.


Actually, Unity isn't currently //even// a touchscreen interface.

See, the software base for Ubuntu is mostly cross-plaform stuff that's designed primarily for desktops, so it tends to rely on things like F1 help, alt-clicking and shift-clicking, and right-click context menus. So, an OS platform that runs that software and supports touchscreen interfaces needs to have an additional chunk of GUI infrastructure that lets the user trigger those events using a touch interface. It's easy enough to do ... if you have a launcher strip, you tap a few times on its page pad to get to the "modifiers" strip, and then you tap a modifier pad followed by a screen location to get the extended functions.

Unity doesn't have that additional translation feature to allow its software library to be used reliably on a touchscreen device. There's no obvious way to right-click on something. That's why people who care about UI design tend to hate Unity, because Unity is essentially a fake. It's like a graphic designer's mockup of how a touchscreen interface might look, without the actual functionality. 
The other thing that offends UI designers about Unity is that is that its design (or lack of) suggests a totally derivative approach to user interfaces that concentrates on blindly copying distinctive features from other OSes, without any real understanding of why those features were there, and whether or not they still work in different contexts. So the people behind Unity have probably not felt that the ability to trigger modifier-key functions is important, because the associated GUI components don't appear on iOS or Android. But iOS and Android apps are custom-written for an environment in which right-clicks and Crtl/Shift/Alt keys don't exist, so their OS interfaces don't have to bother simulating those functions. Ubuntu is a different sort of platform, with different demands, and when it comes to running desktop software on a touchscreen device, it's supposed to be the job of the OS to make that transition work. It's not supposed to be the app writer's job to rewrite their app for different hardware, it's supposed to be the job of the OS, as a platform, to seamlessly provide the software with equivalent input on the different hardware devices. 

Since Ubuntu/Unity doesn't yet provide this conversion for touchscreens, Ubuntu is not yet a touchscreen OS, and Unity is not yet a touchscreen interface. 

There's nothing instrinsically wrong with the idea of a pad strip, and we could design a brilliant touchscreen interface whose screenshots looked superficially like Unity. But Unity itself, in its current incarnation, is still a bad joke.
