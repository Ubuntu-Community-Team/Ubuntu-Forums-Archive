### Post by cariboo on 2012-02-06
Which files couldn't you find? ~/.xsession-errors is a hidden file in your home directory. /var/log is where all the logs are located, and /var/crash is where all the crash logs are located. If you are using nautilus to find and look at the files, to view hidden files in your home directory, press Ctrl-h.

To go to the /var directory, click on File System in the left pane, then click on the var folder, you'll then see the log and crash directories.

For me it's quicker to use the terminal, using cd to change directories, and:

```
cat <filename> | less
```

to view the files.

I see. It's an odd case, which I don't think I have ever dealt with here at UbuntuForums. Normally one Linux Distribution is not installed over another. I actually didn't even know the installer would accept to write to the partition without wiping it first. Look at the logs: There are errors everywhere. Your logs have too many warnings and errors. 

Honestly, it will take you hours, days or more to find out and fix all problems and restore it to a Ubuntu system. And still you wouldn't be sure to have fixed all of the problems. The safest / fastest solution is to backup your important data to any external media (usb, external hdd/odd, network share, etc) or (not the best solution) to a separate HDD partition and reinstall Ubuntu 12.04 from the ISO. In 10 minutes you'll have a shiny new working system back. I woudn't even preserve the HOME directory (I wouldn't trust keeping dotfiles with settings from the previous OS). 

If you really wanna keep this install, I guess the only possible way would be to:

- Make sure the repositories in /etc/apt/sources.list are valid for Precise Pangolin (12.04): Check at [http://repogen.simplylinux.ch/](http://repogen.simplylinux.ch/)) and remove all invalid ones. Use the following command to edit /etc/apt/sources.list
```
sudo nano /etc/apt/sources.list
```Then run these commands:
```
sudo apt-get update
sudo dpkg-reconfigure -phigh -a
sudo apt-get install -f
sudo apt-get update && sudo apt-get upgrade
```I would also create a new user, so that it has default settings in his new HOME directory.
```
sudo adduser **yournewusername**
sudo usermod -aG adm,cdrom,sudo,dip,plugdev,lpadmin,sambashare **yournewusername** 
```Then reboot... 
```
sudo reboot now
```...login with new user and update/upgrade again:
```
sudo apt-get update && sudo apt-get upgrade
```But, considering the time this will take, I would do a clean fresh install.

Later you can copy your files from old home to new one, excluding .dotfiles.
