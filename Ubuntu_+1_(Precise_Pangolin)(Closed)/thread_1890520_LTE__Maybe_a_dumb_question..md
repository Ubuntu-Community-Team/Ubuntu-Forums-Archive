I think you meant LTS - Long Term Support.

Anyway, here's the thing. Yes, of course the Fall version is more advanced in some ways than the Spring version. But the following Spring version will be more advanced than the previous Fall version. There's nothing magically better about Fall versions over Spring versions. The six-month improved version will always be six-months improved, Winter, Spring, Summer or Fall. At some point, you have to pick a date and call it an LTS :KS

And LTS versions are meant to be very stable, which means a bit less cutting edge and so forth.

Hello All

Just to add to sgage's answer, here is a link to the LTS page on the Ubuntu wiki, a site where you can find out more about the 'nuts and bolts' of Ubuntu.

[https://wiki.ubuntu.com/LTS](https://wiki.ubuntu.com/LTS)

As you will see from the 'roadmap' graphic on that page, 12.04 LTS will have 'sub releases' at various points (12.04.1 and so on) and these sub releases will ensure that any late bugs and some hardware related updates are made.

The rest of the page emphasises the LTS aim for stability and reliability. Large enterprises *like* stability and predictability! Perhaps a few people in a company will need 'bleeding edge'.
