The link given for tuxgarage in another reply looks pretty good.

Chroot is very handy for repairing broken installs.  At least you can keep them update/upgraded until that fixes them.

All it is doing is getting you into the system, in terminal, just as you would be from a CLI in the install.  Very handy.

I am not even sure who inflicted this script on me anymore.  Was back in 9.10-testing.  I think it may well have been philinux, I know he was involved in the thread where I was learning about this.

As I use Debian I have re-edited this to include sudo.  This is for folks that have labeled partitions.
```

#!/bin/sh

echo "Sqeezy"

sudo mkdir /mnt/Sqeezy-root
sudo mount /dev/sda7 /mnt/Sqeezy-root/
sudo mount -o bind /proc /mnt/Sqeezy-root/proc
sudo mount -o bind /dev /mnt/Sqeezy-root/dev
sudo mount -o bind /dev/pts /mnt/Sqeezy-root/dev/pts
sudo mount -o bind /sys /mnt/Sqeezy-root/sys
sudo cp /etc/resolv.conf /mnt/Sqeezy-root/etc/resolve.conf
sudo chroot /mnt/Sqeezy-root /bin/bash

```
The echo command at the top just prints "Squeezy" at the top of the terminal output.  I put that in there so that I know where I am if I get confused.  I usually run 2 at a time when upgrading my installs.

I keep one somewhere I can find it easily on every install to chroot into it.  This is handy if you need to do it from a Live Session for some reason.  Just grab the bugger to the Live Session and you are ready to go.  Great for those grub screw ups.

It is also handy if you want to change grub from where ever you are just to check that they all are working right.

I use different backgrounds on all my installs and use the same image on that installs grub.  That way I know where grub is coming from.

Yes I can get confused easily.

On the subject of partitions.  As far as I know there is no set limit to the number you can have.  16 is certainly not the limit.  I have never had more than 27 on a drive.

I always seem to over lap a couple somewhere in the 20s and then all partitioners just refuse to do more.  This just comes from my being sloppy.

Somewhere there is the story from a guy that installed 100 OS's on one drive.  I read it years ago and have no idea where it came from.  Could be in the archives of this forum for all I know.  I know it was old when  I read it.  He just was seeing if he could do it.  He did.

I do not know if he had a /swap or not but if so that would be 101 partitions.

He did all installs on 1 partition which I can't stand but would certainly do if trying a stunt like that.  They were all the minimum size needed for installation.
