A standard release (like 11.10) is supported for 18 months. I believe 12.04 (a Long Term Support (LTS) ) will be supported for 3 years.

Either way. There is no requirement for you to upgrade every six months. There are, of course, those of us who follow the development of a new release like normal people watch football/soccer/basketball/nascar/etc. but that is a different thread. Enjoy it and don't upgrade if you don't really want to.

That said, don't let your support lapse, because support for upgrading directly to the newest version is only supported for systems that are still in the support window. For example, if you were running 6.04, you would have to upgrade a couple of times in order to safely upgrade to 12.04 and it would be a real pain.

### Post by gunksta on 2012-03-29
I typically offer one simple piece of advice: 

If you have to ask whether or not to upgrade to an Alpha/Beta/RC, the answer is no. You're not ready AND/OR the computer in question is too important to risk.

Options include (in descending order of cost):
- Get a second deskto/laptop for testing.  ( Most expensive option. )
- Get a second hard drive so you can switch easily.  ( I have an old drive specifically for this. )
- Dual ( or triple or quad ) boot. Your drive must be large and/or relatively empty.
- Run it in a VM. Also requires a large hard drive but is easier to setup, delete, etc.

Options 3 and 4 are equal in terms of cost. The last suggestion is arguably the best for a dev release that you will probably want to nuke out soon anyway. But, there are sometimes problems with 3D stuff in VMs so you may not see the environment in its full glory. It will also, necessarily, run slower due to limitations in RAM and access to the processor. A VM is a poor platform for testing video games.
