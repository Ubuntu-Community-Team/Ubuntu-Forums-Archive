That thread was not really about how to have a 'custom command' show up in your context menu. It was really about how some applications are shipping .desktops where the Exec= line does not end in a %<letter>

In that case the app **may** not show in the r. click > other applications menu & will not show in the r. click > **properties** > other app.. menu

 You can create custom command  context entries in a couple of ways, how so can be personal preference & sometimes depend on what exactly you want to do. 

you can create your own .desktop's, create launcher quicklists to execute directly, use nautilus-actions, or there are a number of pre done ones available thru the nautilus-actions-extra ppa's - 
[https://launchpad.net/~nae-team/+archive/ppa](https://launchpad.net/~nae-team/+archive/ppa)
[https://launchpad.net/~nae-team/+archive/daily](https://launchpad.net/~nae-team/+archive/daily)

(note -  nautilus-actions-extra is not nautilus-actions, it uses python-nautilus extensions for the actions

So you may wise to give an Ex. of what you wish to do
