### Post by u2nTu on 2012-03-14
The point of division appears to be between those who use the monitors **together** in one location (as in the linked video) and those who use them **apart**, for example: a TV in the next room.

The current default is for the 'together' users, for whom windows opening unexpectedly on the second monitor is no problem. Ex. No trouble when a 'full screen' button is toggled on one monitor only to have the window appear in the other -- a slight turn of the head or shift of the eyes fixes it.

But not so for those with some distance between the displays, as when trying to serve video to a TV in a different room. One must carry the input device (mouse or keyboard) to a place where the other screen can be viewed and interact there. This kills some applications as an HTPC.

Which brings up the question, **is there a standard method of configuring multiseat that works with 12.04?**
