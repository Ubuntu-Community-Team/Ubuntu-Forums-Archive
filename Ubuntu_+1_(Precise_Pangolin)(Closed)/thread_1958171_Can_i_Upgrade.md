### Post by ubuntu27 on 2012-04-14
Yes there are errors or bugs that could get "corrected" if it is freshly installed. 

Technically, doing updates should get you on the same state as the final release version. But, while we are using the pre-release, sometimes "things happen" such as file or configuration corruption. So, bugs gets fixed by updating, but it won't fix any errors, mistakes, or corruption that we ourself made unknowingly. (Because those are not bugs, but just corruption in user's config files).

Corruptions can be fixed manually though.
If something happens, you could always delete directories where it host the config files such as .local and .gconf

Another trick is to create a new user. A new user should have the default configuration intact.

But, if any "tricks" don't work to fix the apparent "bug" or configuration issues, then perhaps it is best to re-install the OS.
