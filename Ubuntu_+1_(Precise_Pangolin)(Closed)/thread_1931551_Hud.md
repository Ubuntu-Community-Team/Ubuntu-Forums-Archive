HUD is about searching menus, titles of recent files &c and it learns by what you do. To check its working, try this

[LIST=1]
[*]Load firefox and make sure its the current app, i.e. it has window focus
[*]Click File New Window
[*]Close the new window
[*]Tap the Alt key to bring up HUD
[*]Type New Window
[*]You should see 'File -> New Window' in HUDs result list
[*]Click on that entry (or arrow down to it) and your new window appears in Firefox
[/LIST]

You can also type keywords from the file names of recent files, music track titles &c

My problem with HUD at present is that the results are not labelled with what they are: i.e. you get results from any of the menus of the currently focussed app, the indicator apps, and the recent files but these are not categorised or labelled with where they are from. Its a bit random until you build up some history.

I also want HUD to be able to *issue* commands to applications; verb noun type approach. Well I can dream.

This is more paradoxical than you make it sound.

HUD, not being artificially intelligent, does not "guess" anything.
It searches, like google.

The search algorithm is good, but not magical.
It can suggest recent actions, common actions, and find items in the menus.

Unlike google, it doesn't have searchbots probing all the data in the internet.
It's limited to what menus it can get through dbus.

It cannot *understand* what you want.
If you don't know what you're looking for, you'll never find it.

ie, If I were looking for "red eye removal" in GIMP:
I get five entries for "File - Create - Web page themes - etc - etc"

Many would.  The world moved from the CLI of the '70s to the GUI for a great many reasons.

In terms of keystrokes the Ubuntu and HUD implementations are nearly identical:
1. Type a word or phrase
2. The system locations the nearest match
3. If it's the one you want hit Enter

But an additional advantage to OS X's is that it shows you were the command lives in the menu bar, so you have the opportunity to also learn the Command key equivalent for even faster access.

Meanwhile, for mysterious reasons Ubuntu continues to conceal the menu bar from the user, hoping they'll discover it if the mouseOver in the top panel, but greatly slowing down the acquisition of muscle memory by preventing aiming for a given menu in advance of that movement.

> On another topic, I see a lot of people commenting recently that HUD "is entirely optional" (sic) but that was not at all the impression I got from Shuttleworth's original blog post.Yes, this is perhaps the most disturbing thing I've read about Ubuntu's evolution.  I hope and pay it's a misunderstanding of the actual intentions.

I can appreciate the desire to be novel to stand out from the pack, but novelty for its own sake is an anti-feature when it conflicts with 30 years of human interface research.

As an "also", the HUD is a tremendous step forward.  But as a replacement for menus it will fail every honest usability test, and become the butt of endless industry jokes.
