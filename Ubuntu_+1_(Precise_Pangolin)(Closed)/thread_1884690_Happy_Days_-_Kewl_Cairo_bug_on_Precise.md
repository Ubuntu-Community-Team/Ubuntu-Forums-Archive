### Post by TerryNewton on 2011-11-21
I see that effect when running any Gnome/metacity session with compositing enabled under VirtualBox. Happens with Ubuntu 11.10, 12.04 and Fedora 16 and I presume other Gnome 3 systems but haven't tested with other VM's, does not happen with mutter or compiz (but those cause other not-so-cool effects under VirtualBox). Turning off metacity compositing using gconf-editor "fixes" the bug. If using the Ambiance theme I don't really need compositing, I use it mainly so Adwaita doesn't look funky and just ignore the trails when icons are moved (or have fun with them) - this only applies if desktop icons are enabled, moving windows does not cause the effect.

I had assumed this was just a VirtualBox effect but I guess not..
