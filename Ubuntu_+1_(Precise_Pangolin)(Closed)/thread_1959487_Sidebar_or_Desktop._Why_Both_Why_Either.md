### Post by xajeiw9I on 2012-04-15
The side bar is a task bar and a launch bar. You will notice that the programs you start anywhere will have an icon there when it is running, unless it is some special program that is configured to run in the background or have its icon put in the tray. 

If there was a way to remove it, you would need a taskbar running anyway. If you need to modify your desktop to that level, I would recommend you install something else, maybe XFCE or Gnome (run it in Classic mode), all available in Ubuntu repositories.

The way I understand it, the Unity launcher (sidebar) is there so you don't have to place launchers on the Desktop. This way, you can save desktop space for files and folders. Furthermore, it organizes favorite/most used applications into one convenient location. A lot of the functionality found in the bottom panel (from the Gnome 2 days) is recreated in the Unity launcher. It acts as the taskbar, you can switch workspaces, and you can access your programs all in one spot.

### Post by 3rdalbum on 2012-04-16
The topic title is incorrect. In Ubuntu today it is non-trivial to put icons on the desktop. It is Sidebar, Not Desktop, for program icons.

As has been noted, the sidebar (it's actually called the Launcher) also provides window-switching and workspace management capabilities. It's always there for quick access and is much easier than minimising every window to get back to the desktop (or pressing the shortcut for Show Desktop and then getting confused when you press the shortcut again having opened some more windows).

With the desktop alone you can't switch between windows.

### Post by mc4man on 2012-04-17
Maybe not the place or time - 
just to note that quicklists are depreciating the use of 
X-Ayatana-Desktop-Shortcuts=
and 
[Whatever Shortcut Group]
in favour of the 'Desktop Actions' way
Though for 12.04 the old way will still work

Myself tend to carry several custom .desktops thru releases, still using a couple from 11.04 dev

So for new ones or if inclined  to edit current custom ones the new method is (using the tail end of nautilus-home.desktop to example with one added quicklist entry

```
X-Ubuntu-Gettext-Domain=nautilus
Actions=Window;Share;

[Desktop Action Window]
Name=Open a New Window
Exec=nautilus
OnlyShowIn=Unity;

[Desktop Action Share]
Name=Open Share
Exec=nautilus /usr/share
OnlyShowIn=Unity;
```
