### Post by grahammechanical on 2012-03-27
The intention is to reduce the number of reasons for the user to authenticate an action. Some users complained. So, you will find that where on earlier versions we needed to authenticate sometimes on 12.04 we no longer need to.

Updates are considered one of those times that we do not need to authenticate.

This is the way it goes. We download a development release  and we do a daily update and things are changed.

Or we wait until after the distribution if officially released and then see what has been done with Ubuntu in the mean time. Constant change/improvement.

Somewhere there are blueprints describing in advance what the intentions are for each release.
