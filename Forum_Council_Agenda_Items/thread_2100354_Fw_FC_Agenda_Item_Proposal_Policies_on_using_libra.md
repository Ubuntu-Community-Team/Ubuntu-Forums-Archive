Okay, i hate to be an annoying person and bug you, but this is a bugsquad/bugcontrol suggestion that needs to be discussed (give me permission to add this to the agenda, and i'll do so, if you would forward this privmsg to internal areas).

As an example, [http://ubuntuforums.org/showthread.php?p=12430310#post12430310](http://ubuntuforums.org/showthread.php?p=12430310#post12430310) currently has workarounds which suggest *downgrading* the libraries in 12.10 to the libraries in 12.04 manually. From a bugs perspective, doing this introduces all sorts of hell and fury, because downgrading a library such as libcairo2 (in that thread) can affect all sorts of software in 12.10 that need the newer libraries.

If I had any sort of power here, I'd be going in and slapping each of the users who suggests such workarounds. It introduces a whole mess of issues, from new features in software breaking, to bringing old regressions back, to causing system instability.

I'd advise the Forums Council to modify the forums policies to make it against policy to suggest such manual downgrades to older-release libraries. It makes the jobs of bugsquad, bugcontrol, the release team, and the devel team a LOT harder.
