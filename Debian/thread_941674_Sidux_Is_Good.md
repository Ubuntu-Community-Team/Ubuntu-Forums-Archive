### Post by tommcd on 2008-10-29
> **mosestruong said:**
> Are there a way to "upgrade" from debian to sidux through apt-get dist-upgrade or something?

Well, if you wanted to try that, you could just change your Debian sources.list to point to the Sidux repos and run:
```
apt-get update
apt-get dist-upgrade
```
or
```
aptitude update
aptitude full-upgrade
```
Aptitude is now the recommended package manager in Debian. I have no idea what Sidux uses. I would assume it would work similar to the way it works in Debian. You might want to ask about upgrading from Debian to Sidux on the Sidux forums or their IRC channel.
Personally, I would just do a clean install of Sidux. You should probably download and burn a Sidux install disk before attempting this. If you system gets broken, you could then just do a clean install of Sidux.

 This is one thing that still puzzles me about Debian. If you search around the Debian forums, you will find many knowledgeable Debian users who swear by apt-get, and many others who swear by aptitude. Many of these guys run testing and sid. Some even have mixed testing-sid systems. There does not seem to be a solid consensus for apt-get or aptitude. The one thing people agree on is that you should be consistent, i.e., stick with one or the other, dont mix apt-get and aptitude.

### Post by lamegaptop on 2008-11-03
I have sidux xfce4 on my eeePC 900. On my desktop too. This forum post helped me get the very few things that did not work going - 

[http://sidux.com/PNphpBB2-viewtopic-t-7669-postdays-0-postorder-asc-highlight-eeepc.html](http://sidux.com/PNphpBB2-viewtopic-t-7669-postdays-0-postorder-asc-highlight-eeepc.html)

Comment - On the eeePC you may wish to 'apt-get install gnome-core' instead of 'apt-get install gnome' as stated above. Then you will be free to add only those applications that you will really use. But keep in mind that those same apps will work under the Xfce4 desktop as well.

Use sux for root access whilst in the desktop. 

To upgradeand to use the other excellent tools in smxi script -

<ctrl &alt>F1
login as root
init 3
smxi

On the eeepc I find the tools in smxi very useful for cleaning up the bits that I don't need in order to save space and speed things up a bit during dist-upgrades; remove unused locales, xorg modules, kernel modules, and apps. Keep in mind you must first install the script (it's not installed by default) by following the directions in the mighty fine manual (under Sytem Administration > smxi Syntax Command Sets). Have fun!
