Stable does, as stable focusses on core stability and so only uses tried and tested stable packages. ### Post by DjBones on 2008-08-23
after reading your post, I'd say to make the switch to Debian.
You couldnt have picked a better time either, the new stable branch is about to be released (Etch 4.0 to Lenny 5.0) soon.

The basic desktop of Debian is somewhat like Ubuntu, but much more vanilla because it is not much more than straight default GNOME.  Installing themes though will function just like Ubuntu in that regard.

Now, a common misconception is that Debian has "old" packages.  This is a little short sighted because the packages in Testing & Stable are rigorously tested to make sure that nothing breaks thats not supposed to.  Although Unstable (sid) is great and probably wouldn't give you many problems, Testing is much more valuable in my opinion (I am biased because thats what I use haha).

If you want to learn how to use Linux better, using Debian with a gnome desktop will provide you with a vanilla system that will most likely require tweaking to get it into the shape you want it to be in.  But off the bat won't require as much immediate knowledge like Arch, Slack, BSD, or Gentoo.  Although if you install it from the base system in a minimal install and you do everything by hand, you'll be presented with the same challenge of course.

One of hte main disadvantages of Ubuntu is that its slow and more bloated than other distro's by default due to its extensive hardware support and the vast amount of services turned on.  Most people switch for the reason of a more responsive, expedient system that afford the user more control.

To be fair, Ubuntu isn't much more than a differently themed Debian Unstable thats frozen every six months with an enormous kernel. lol
