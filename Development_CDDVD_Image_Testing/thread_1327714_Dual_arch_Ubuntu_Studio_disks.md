### Post by MetalMusicAddict on 2009-11-15
The Ubuntu Studio project is considering creating a single, dual-arch (32/64bit) ISO image for Lucid Lynx.

Poll is yes/no and feel free to express your concerns.

We're considering this because it would mean a single disk to test along with the possibility of manufacturing these disks for sale. (not to make a livin' but we gotta covers costs) As this is a LTS and we will be heavily focused on stability we feel Lucid would be a good candidate.

We do know however that it will mean that a user is essentially downloading both current disks. Hopefully, we'll reduce some of the size by combining disks. Testing will be required.

### Post by Stochastic on 2009-11-16
I answered the poll as an Ubuntu Studio user myself - I would not be put off by this change.  However, I do imagine that a number of low-bandwidth users would be irritated at the change.

For me, a ~2.8GB download for an operating system isn't an unreasonable request (this is about the size of most windows 7 torrent files if one searches) but it is getting a little big.

Also, it might be useful to stress that low bandwidth users can download the Ubuntu image and install the ubuntustudio-* packages on top as needed.

EDIT:  Hmm, as I think about this further, two more ideas come to mind: 1) It seems quites silly to have two versions of each package on the disc for almost no benefit.  2) Combining the discs will NOT reduce the number of test cases needed to check the ISO.  We will still need to try both the 32bit and 64bit install versions as a problem might present itself in just one of the two versions of a given package.
