I must agree with previous posters saying Gutsy is unstable.

Over the last few years I have been winning people across to linux/ubuntu relying heavily on stability arguments.  Gutsy has made me eat my words on several occasions.  The advice I now give is stick with Feisty wait for Heron.

### Post by frodon on 2007-12-14
1) yes. The first thing is to remove the external repositories before the update, many users got failed upgrades just because of this. This should be something to explain to each beginner before advising them the use of an external repo, you have to remove it before starting an upgrade to the next ubunt version.
Then in top of that if the apps you installed using external repositories may have installed some updated libraries at the same time so you may get some dependency issues during the upgrade. In such case you just have to downgrade the library which cause the problem.
Despites this is easy to fix most users who did such mistake will just say the upgrade process is not stable whereas in this case it is the user fault.

2) of course not, they are official repositories.

With main, restricted, multiverse and universe repositories the only external things i had to install were libdvdcss, w32codecs and skype. Obviously i don't count java applications as i consider them as stand alone apps.
