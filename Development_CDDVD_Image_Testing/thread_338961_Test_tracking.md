### Post by Henrik on 2007-01-15
We've had an interesting start, with lots of tests coming in. A challenge now for Herd 3 and 4 is to streamline this process so that it will be easier to put the results to good use during the (hectic) release process. 

I've written a [proposal]("https://wiki.ubuntu.com/Testing/ForumProject/TestTrackingInMalone") for using the Malone bug tracker as a sort of test-tracker. This would make it possible to see at a glance which tests had been performed, which had failed and which were still outstanding.

Comments? Alternative proposals? Does this remove the testing project too far from the forums? I wouldn't think so because all the discussion and coordination would still take place here. Someone doing testing should already have a Launchpad account for filing bugs anyway.
