### Post by Baji P. on 2010-09-06
In BREAKING NEWS from **Planet 64-bit** of the **AMD constellation** :D

Just a little while ago, I downloaded the 64-bit Alternate install CD image and tried it on my Dell Optiplex 740 machine, with Athlon X2 processor & 4 Gig of ram.

First I tried the check CD feature, it worked successfully. This step had failed on all 64-bit images of 10.04 & 10.04.1

Upon rebooting, I selected expert mode and pressed enter to "Install Ubuntu". The install proceeded just fine.

Since I have a functioning 32-bit install on this hard disk, I had to abort the 10.10 installation at the disk partitioning step. But everything worked fine until that step.

Over the next few days (after the Ohio Linux Fest ;)), when I have more time, I'll put in a blank hard disk and try the install all the way to the end.

Anyone else with spare time & a spare 64-bit processor please test out the [10.10 images]("http://www.ubuntu.com/testing/maverick/beta"), it is time we got full use of 64-bit computing.

[http://www.ubuntu.com/testing/maverick/beta](http://www.ubuntu.com/testing/maverick/beta)
