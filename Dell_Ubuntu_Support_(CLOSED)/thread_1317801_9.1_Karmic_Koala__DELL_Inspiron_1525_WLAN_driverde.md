### Post by chenxiaolong on 2009-12-21
Here's how to get the Broadcom STA driver working. You will need to run these commands in a terminal.

1. [COLOR="DarkOrchid"]sudo apt-get remove b43-fwcutter bcmwl-kernel-source ndisgtk ndiswrapper linux-backports-modules-karmic[/COLOR]

   Removes packages that could conflict with each other.

2. [COLOR="DarkOrchid"]ls /etc/modprobe.d | grep blacklist[/COLOR]

   This lists the contents of the */etc/modprobe.d* folder. Check the output of this command and make sure that there isn't a folder called *b43.conf, *bcm43.conf, *ssb.conf, or *wl.conf. If there is, rename the file by typing: [COLOR="DarkOrchid"]sudo mv thefile.conf newnameofthefile.conf.backup[/COLOR]

3. [COLOR="DarkOrchid"]sudo apt-get install bcmwl-kernel-source[/COLOR]

   This actually installs the Broadcom STA driver.

4. [COLOR="DarkOrchid"]sudo cat /etc/modules[/COLOR]

   This lists the contents of the */etc/modules* file. Check the output of the command and see if there is a line that says ***wl***. If not, run [COLOR="DarkOrchid"]echo "wl" | sudo tee -a /etc/modules[/COLOR]. That appends "wl" to a new line at the end of the file.

5. Restart your computer and enjoy your wireless! By the way, I noticed that you said that you right-clicked the network icon --> you're supposed to "regular"-click it.

### Post by chenxiaolong on 2009-12-22
Efflandt: Well, who cares about eth* as long as everything works?:D

Vikas.sood: Intrepid installed the Broadcom STA driver by default, which is why you didn't have to do it manually. Starting from Jaunty, Ubuntu stopped installing the driver on the default install. You had to install it manually from the CD or from the repos.

The b43-pci-bridge driver is the b43 driver, which is a reverse-engineered driver for Broadcom cards. For this card, it's still is development stage. I have instructions on how to use it in this thread: [COLOR="SeaGreen"]http://ubuntuforums.org/showthread.php?p=8385863[/COLOR].

1. [COLOR=DarkOrchid]sudo apt-get remove b43-fwcutter bcmwl-kernel-source ndisgtk ndiswrapper linux-backports-modules-karmic[/COLOR]

   Removes packages that could conflict with each other.

2. [COLOR=DarkOrchid]ls /etc/modprobe.d | grep blacklist[/COLOR]

   This lists the contents of the */etc/modprobe.d* folder. Check the output of this command and make sure that there isn't a folder called *b43.conf, *bcm43.conf, *ssb.conf, or *wl.conf. If there is, rename the file by typing: [COLOR=DarkOrchid]sudo mv thefile.conf newnameofthefile.conf.backup[/COLOR]

3. [COLOR=DarkOrchid]sudo apt-get install bcmwl-kernel-source[/COLOR]

   This actually installs the Broadcom STA driver.

4. [COLOR=DarkOrchid]sudo cat /etc/modules[/COLOR]

   This lists the contents of the */etc/modules* file. Check the output of the command and see if there is a line that says ***wl***. If not, run [COLOR=DarkOrchid]echo "wl" | sudo tee -a /etc/modules[/COLOR]. That appends "wl" to a new line at the end of the file.

5. Restart your computer and enjoy your wireless! By the way, I noticed that you said that you right-clicked the network icon --> you're supposed to "regular"-click it.
