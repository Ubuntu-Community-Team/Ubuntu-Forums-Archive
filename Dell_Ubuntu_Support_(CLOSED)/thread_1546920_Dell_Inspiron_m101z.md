### Post by boozezela on 2010-09-26
The guy on youtube is not a tool: the fan spins up almost randomly and it is loud if you use the laptop in a silent environment or you use it for couch/bed surfing.

In comparison my 2Ghz Core 2 Duo is quieter.

I describe the behaviour as random because the M101z could stay completely silent for 20 minutes after boot or the fan could start as soon as the computer boots. On the plus side, the noise will stay the same no matter if the CPU is at 1% or 100%, because the speed apparently does not increase (but the fan is capable of way higher speeds, as you will be able to hear from time to time between reboots).

The touchpad generally sucks: sometimes it does not register a tap, other times the pointer will jump all over the screen when you lift a finger and put it down on the same spot. I had no problems with gliding though.

The most annoying thing probably is the lower edge between the touchpad and the buttons.
There is a tiny space that does not register for movement, that is good because the pointer will not move, but the touchpad will register the presence of your finger.

This means that if you use one finger to push the left button and another one to drag an icon around, if you accidentally touch said edge with the finger holding the button, the pointer will stop moving or jump up and down erratically.

The display could need calibration, under Windows 7 everything was  washed up to the point that it was not possible to see the gradient of a  html form button at all, but lowering the gamma helped a lot. Under  Ubuntu the display seemed fine though.

About Windows' serial number and rescue media, the sticker is hidden under the battery, and the rescue media can be prepared using the Dell backup utility that comes preinstalled in the system (DVD+/-R or USB key, your choice).

The recovery media will also allow you to recreate Dell's recovery partition in case it gets corrupted.

On the minus side, the recovery partition is about 15 GB, of which about 7 used and 8 free for backing up files from your Windows system (the recovery process allows you to restore the OS and your own files if there is a backup available).

Do not shrink the recovery partition and move the OS partition or install Ubuntu on the gap between the two because you will screw things up.

About Ubuntu, I tried 10.04.1 x86 (yes, the 32bit version) with strange  results: booting from a USB key prepared on my other notebook everything  worked fine (more on that later), but installing it on an external hard  drive I could get past the login screen just once.

During the other tries the computer just froze. Booting in rescue mode I  had to forcefully switch to TTY1, otherwise I would not get anything as  the system seemed to be stuck on the boot messages. Things did not improve after updating the system with apt-get. As already stated you will need broadcom closed driver for wifi.

In any case trying to adjust brightness resulted in a freeze.

I hope that a future BIOS upgrade will fix the fan, on the other end I will try to install Ubuntu 10.04.1 64bit as soon as I have more time.

### Post by gtaluvit on 2010-10-14
VA accelerated video is now working in Maverick using the GMA500 ppa and an XVBA deb from [http://www.splitted-desktop.com/~gbeauchesne/xvba-video/](http://www.splitted-desktop.com/~gbeauchesne/xvba-video/)
I installed the .7.6-pre2 version.

```

sudo add-apt-repository ppa:gma500/ppa
sudo apt-get update
sudo apt-get upgrade
sudo apt-get install vainfo mplayer

```

Inside $HOME/.mplayer/config
```

vo=vaapi,xv,
va=vaapi

```
Note the comma after XV

Now mplayer will attempt vaapi by default.

In case anyone's not sure how to do it (I had to check... damn grub2.)
1. Open a terminal.
2. Enter: **sudo gedit /etc/default/grub**
3. Find the line beginning with **GRUB_CMDLINE_LINUX_DEFAULT**. Add **noapic** onto the end of the line.
4. Close and save gedit.
5. Enter into the terminal: **sudo update-grub**.
