I'm the one who answered on the list. I can add that 802.11n works, but only if you edit /etc/modprobe.d/intel-5300-iwlagn-disable11n.conf to
```
options iwlagn 11n_disable=0
```

I guess the blacklist should only be used for an Intel 5300 card, but they use the same driver. Anyways, I was able to get speeds exceeding 54MB/s afterwards.

Well, guess I have to keep the screensaver off for now in order to avoid the flicker.  So, in screensaver's place I got a script and put that to a launcher on the panel. When I want the screen back, I can move the mouse or touch any key and the screen will come back briefly before it turns off again. #!/bin/bash
screenOffLockFile=/tmp/screen-off-lock

if [ -f $screenOffLockFile ];
then
        rm $screenOffLockFile
else
        touch $screenOffLockFile
        sleep .5
        while [ -f  $screenOffLockFile ]
        do
                xset dpms force off
                sleep 2
        done
        xset dpms force on
fi 
```

### Post by PhilEvans on 2010-12-31
Hi folks,

A few bits of info on this one which may be of use:

First, for the SD card reader there are a few posts floating about point out that it only works if the card is inserted at boot time. There is a fix to this ([http://ubuntuforums.org/showthread.php?t=1309991](http://ubuntuforums.org/showthread.php?t=1309991)) that worked for me (Dell L501X, Kubuntu 10.10), basically add "pciehp.pciehp_force=1" to your kernel arguments. 
e.g. sudo nedit /etc/default/grub ; then add the line above to the end of of "GRB_CMDLINE_LINUX_DEFAULT" string, then "sudo update-grub" -- this shoudl fix your SD card.

Second: the nvidia issue. This is a pain in the ****. Basically, this system uses optimus so (in Windows, where nvidia support optinus) the i915 card does all the work until it decides that something needs the nVidia GPU, at which point it switches to that; but the nVidia card renders and then uses the 915 to display on. Linux doesn't seem to support this, nVidia have no plans to support it in LInux, and you can't try blacklisting the 915 diver because the nVidia card uses it. If you google for "vgaswitcheroo" you will find some info which is supposed to help solve this... but doesn't work for me (I don't have the right path in /sys for some reason. 

Finally, the screen backlight. Basically, surprise, surprise, there's a BIOS bug because the standards have only been around for a gazillion years, which is clearly not long enough for Dell. Good news is that there is a solution, see: [https://launchpad.net/~kamalmostafa/+archive/linux-kamal-mjgbacklight](https://launchpad.net/~kamalmostafa/+archive/linux-kamal-mjgbacklight)    -- The bad news is that this only works properly in Gnome as no-one has (yet) patched the KDE power control. You can "cheat" by echoing a number to /sys/class/backlight/intel_backlight/brightness to set the rbightness level yourself -- or indeed write a couple of scripts like this:

brt_up:
#!/bin/bash
BPATH="/sys/class/backlight/intel_backlight/brightness";
MAXPATH="/sys/class/backlight/intel_backlight/max_brightness";

declare -i max=$(cat $MAXPATH)
declare -i val=$(cat $BPATH)

declare -i step=max/15

val=$val+$step

if [ $val -gt $max ]; then
  val=$max
fi

if [ $val -lt "0" ]; then
  val=0
fi


echo "$max, $val, $step"
echo "$val" > $BPATH

and an equivalent brt_down which you can then run (via sudo) to brighten or darken.

Hope this all helps. In return, if anyone can tell me why Kubuntu 10.10 crashes on me when I try to mount the SD card from my son's Vtech Kidizoon camera (which won't automount for some reason) I'd be very grateful.....

### Post by kongo09 on 2011-01-07
A goodie for those who tried two-finger scrolling in Ubuntu: it doesn't normall work!

However, if you're on the latest Ubuntu 10.10 (or above) you can get a bleeding-edge driver from Launchpad. The details are discussed here: [https://bugs.launchpad.net/ubuntu/+source/xserver-xorg-input-synaptics/+bug/308191](https://bugs.launchpad.net/ubuntu/+source/xserver-xorg-input-synaptics/+bug/308191)

Just download the latest .deb package (for me it was [https://bugs.launchpad.net/ubuntu/+source/xserver-xorg-input-synaptics/+bug/308191/+attachment/1771346/+files/synaptics-dkms_1.1.1_all.deb](https://bugs.launchpad.net/ubuntu/+source/xserver-xorg-input-synaptics/+bug/308191/+attachment/1771346/+files/synaptics-dkms_1.1.1_all.deb) )

It wouldn't install with the Software Center as expected, so use the command line:
```
sudo dpkg -i synaptics-dkms_1.1.1_all.deb
```

It takes some time and I had to reboot. The you can go to System->Preferences->Mouse->Touchpad and enable the Two Finger scrolling.

### Post by slayerdme on 2011-04-27
One member of the hybrid-graphics-linux team has [found a way]("https://lists.launchpad.net/hybrid-graphics-linux/msg00450.html") to shut down the Nvidia card on the XPS 15(L501x).
I've checked that it works, for the Core i5 version.
What i did was:
```

$ lspci -vnnn | perl -lne 'print if /^\d+\:.+(\[\S+\:\S+\])/' | grep VGA
00:02.0 VGA compatible controller [0300]: Intel Corporation Core Processor Integrated Graphics Controller [8086:0046] (rev 18) (prog-if 00 [VGA controller])
02:00.0 VGA compatible controller [0300]: nVidia Corporation Device [10de:0df1] (rev a1) (prog-if 00 [VGA controller])
$ git clone https://github.com/peberlein/acpi_call.git
$ cd acpi_call
$ make
$ sudo make load
$ ./m11xr2.sh off
$ lspci -vnnn | perl -lne 'print if /^\d+\:.+(\[\S+\:\S+\])/' | grep VGA
00:02.0 VGA compatible controller [0300]: Intel Corporation Core Processor Integrated Graphics Controller [8086:0046] (rev 18) (prog-if 00 [VGA controller])
02:00.0 VGA compatible controller [0300]: nVidia Corporation Device [10de:0df1] (rev ff) (prog-if ff)

```

You can see that the Nvidia card changed status.

To check that it is actually off see the battery rate consumption difference:
```

$ ./m11xr2.sh on
$ cat /proc/acpi/battery/BAT0/state
$ ./m11xr2.sh off
$ cat /proc/acpi/battery/BAT0/state

```

### Post by rockorequin on 2011-07-07
> **zonedabone said:**
> Will this make the card reader auto scan until reboot? If so you could just set up a quick cron job that does this on startup. If not, does this use much CPU, because it could just be run once per minute or something.

Once the kernel sees a card, either with one in the reader at boot or with the PCI scan, from then on it remembers the reader hardware and so cards mount and unmount as they would normally.

If you rescan with the card removed, the kernel doesn't see the reader hardware any more so a newly inserted card won't mount. 

The scan itself is very fast. So a crontab job is a good idea except that it shouldn't rescan if the hardware has been detected.

> 
Also, is there some way to just disable optimus and have the graphics card function like a normal card? It sounds like the whole system may be more harm than good without compatible drivers.

Yes, by default you just use the intel card. The nvidia card is by default powered up (ie wasting power) and you can turn it off with the acpi_call module. bumblebee handles that automatically, ie if you install it, it will turn off the card at reboot and so long as you don't use optirun to run anything the nvidia will stay off.

Yep. Try installing ironhide or bumblebee.

I'd try bumblebee first if I were you because it doesn't turn off the nvidia card (so you are less likely to run into problems):

```
sudo add-apt-repository ppa:bumblebee/stable
sudo apt-get update
sudo apt-get install bumblebee
```

and reboot.

If it works, you can try ironhide:

```
sudo apt-add-repository https://launchpad.net/~mj-casalogic/+archive/ironhide/
sudo apt-get update
sudo apt-get install ironhide

```

This will uninstall bumblebee and install the ironhide server instead, which should configure itself to turn the nvidia card on and off (which saves quite a bit of power).

### Post by jocheem67 on 2011-10-19
```
#!/bin/sh

rmmod nouveau

cd /home/jochem/acpi_call

insmod acpi_call.ko
 
echo '\_SB.PCI0.PEG0.PEGP._OFF' > /proc/acpi/call
```

This is my little dirty script I use for disabling my nvidia on a dell xps l502x

first follow this link > 
[http://linux-hybrid-graphics.blogspot.com/2010/07/using-acpicall-module-to-switch-onoff.html](http://linux-hybrid-graphics.blogspot.com/2010/07/using-acpicall-module-to-switch-onoff.html)

Don't blame me if this wrecks your install, mind you. This method has been successfull on arch with xfce and now on mint.
I suggest to firstt do some more reading on acpi_call. It's all over the web.

You can at best, use the SPIDF output for Front audio, Analog output as Rear but there is no way to use center/woofer through Micro/LineIn. (This is how it's achieved in Windows 7).

For now, you can use 5.1 audio through HDMI : HDMI out to some HD TV which is equipped with a 5.1 sound system.

Ubuntu 11.10, 12.04 and 12.10 do not solve the problem.
Once I am done with investigations, I'd probably post a script on my blog which works around the bug.

Note that in ubuntu 12.04, you can still use your onboard speakers in parallel with an external analog sound device, by plugging it into the SPIDF output.
