### Post by Freaky_Llama on 2009-05-11
The graphics chip is blacklisted by compiz.

I run a Dell D630, and have told Compiz to skip checks. However, before we get to that, please make sure you have installed the Intel graphics driver from Jaunty (jaunty proposed version breaks the libmesa fix I note later on SHOULD you want to do dual head). The package name is xserver-xorg-video-intel

Next, here is a grab from my xorg.conf. You will have to tell xorg to use intel as your driver (Section in [COLOR="Red"]RED[/COLOR]) Please DO NOT Copy the [COLOR="Blue"]section in blue[/COLOR] as I have customized it for my dual head function while using a DVI monitor on the Port Replicator:

```
# xorg.conf (X.Org X Window System server configuration file)
#
# This file was generated by dexconf, the Debian X Configuration tool, using
# values from the debconf database.
#
# Edit this file with caution, and see the xorg.conf manual page.
# (Type "man xorg.conf" at the shell prompt.)
#
# This file is automatically updated on xserver-xorg package upgrades *only*
# if it has not been modified since the last upgrade of the xserver-xorg
# package.
#
# Note that some configuration settings that could be done previously
# in this file, now are automatically configured by the server and settings
# here are ignored.
#
# If you have edited this file but would like it to be automatically updated
# again, run the following command:
#   sudo dpkg-reconfigure -phigh xserver-xorg

Section "Monitor"
	Identifier	"Configured Monitor"
EndSection

Section "Screen"
	Identifier	"Default Screen"
	Monitor		"Configured Monitor"
	Device		"Configured Video Device"
	[COLOR="Blue"]SubSection "Display"
		Virtual	2720 900
	EndSubSection[/COLOR]
EndSection

Section "Module"
	Load		"dbe"
	Load		"i2c"
	Load		"bitmap"
	Load		"ddc"
	Load		"extmod"
	Load		"freetype"
	Load		"glx"
	Load		"int10"
	Load		"vbe"
	Load		"dri"
EndSection

[COLOR="Red"]Section "Device"
	Identifier	"Configured Video Device"
	Driver		"intel"
	BusID		"PCI:0:2:0"
	Option		"UseFBDev"		"true"
EndSection[/COLOR]

Section "DRI"
	Mode		0666
EndSection
```

(yes, that virtual desktop size works as I used the libmesa fix from Cavedon's PPA found at the bottom of:  [https://bugs.launchpad.net/ubuntu/+source/mesa/+bug/146298](https://bugs.launchpad.net/ubuntu/+source/mesa/+bug/146298))

Next, tell Compiz to skip checks for blacklisted devices:

```

touch ~/.config/compiz/compiz-manager | echo "SKIP_CHECKS=yes" | tee -a ~/.config/compiz/compiz-manager
```

Reboot, and make sure you can get back to a graphical interface. Once back in a GUI, attempt to enable desktop effects.

Good luck.

### Post by pro003 on 2009-05-14
Installing xserver-xorg-video-intel

1. Open the file /usr/bin/compiz with root permissions in a text editor
2. Go to the section “# blacklist based on the pci ids”
3. Add a # before the PCI ID of your graphics card (Commenting the line excludies it from processing)
4. Save, Close, Restart
5. Desktop RightClick>Change Desktop Background>Visual effects and choose your favourite mode.

How to find PCI ID of your card? Use the following command:

Code:
```

    $lspci -nn | grep VGA
```
