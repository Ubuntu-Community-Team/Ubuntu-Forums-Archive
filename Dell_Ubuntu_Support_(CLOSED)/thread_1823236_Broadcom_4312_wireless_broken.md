### Post by MARP1961 on 2011-08-12
For some reason in Natty the 'Additional Drivers' does not offer the older B43 driver. I could never get STA to work. Try this:  

Open up Synaptic Package Manager and do a search for 'B43 fw cutter installer'. Select and install this. Reboot and left click the Network Manager icon to see if your router has been found.

I actually got B43 working this way but had to do a fresh install first. I think if you let Additional drivers do it's worst it will blacklist B43 and it's simpler to install Natty, connect an ethernet cable for internet, ignore Additional Drivers and use Synaptic package Manager to get and install the B43 firmware.

You need also to edit your router to accept All Users and while you're at it delete that stupid keyring password too:[http://askubuntu.com/questions/867/how-can-i-stop-being-prompted-to-unlock-the-default-keyring-on-boot]("http://askubuntu.com/questions/867/how-can-i-stop-being-prompted-to-unlock-the-default-keyring-on-boot")

### Post by MARP1961 on 2011-08-14
I don't know why, but the STA driver in Natty seems to cause problems with Dell Broadcom wireless. It seems that having STA installed first in some way gets in the way of B43 driver if you try installing it later. 

The simplest thing I found was to do a fresh install of Natty but **not allow STA to install at all**. I actually removed the 'Additional Drivers' application in Software Centre because it kept pestering me to install STA. Then with an ethernet cable plugged in and internet access established I did updates then installed fw B43 cutter and firmware installer via Synaptic. I then found my router had been discovered by Network Manager. Next I edited my wireless setting to accept ALL USERS (make sure box is checked). I also deleted the silly Keyring Password which I have a feeling causes connection problems.
