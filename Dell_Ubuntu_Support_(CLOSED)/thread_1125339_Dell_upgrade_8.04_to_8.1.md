The Ubuntu version of the Mini 9 ships with Dell's lpia custom kernel for the intel atom processor. As you will find out with playing around with the package manager, certain programs may not be available if you use Dell's custom kernel; although, you can install any packages... it just becomes trickier for a linux newbie. Also, I hear that the Dell lpia repositories are not updated nearly as much as the regular ubuntu versions (8.04, 8.10, and 9.04).

I am actually picking my mini up today and have not had first hand experience with Dell's lpia version. I am going to play around with it until 9.04 is officially released on April 23rd. If I don't like what I see, I will install 9.04.

Run some searches on this forum as well as mydellmini.com/forums for some more information on the pros/cons of Dell's custom kernel.):P

### Post by Therion on 2009-04-14
> **s9032g@comcast.net said:**
> The download may be called "vanilla", but it is a problem because Dell has made it so by leaving out some needed files. It will not download in any simple way. It's like Dell wants us to be stuck with what we have, the alternative being a new computer.
Well, to some degree that IS what Dell wants.  OEM's will frequently "lock down" a customized operating system so the end-user (that's YOU) can't screw anything up.  If you do manage to screw with something the chances are quite good that 99% of the system is still as it was when it left the factory; which makes troubleshooting faster, easier and tech-support calls just that much shorter.

When we refer to a "vanilla install" what we mean is an installation that uses the download you get  from Canonical, as opposed to the "Dell flavor" you have now.  What is being suggested is that you download an .iso of Ubuntu from the Ubuntu website and do a fresh install on your Dell Mini.  Then you will taste the freedom that we all enjoy.
