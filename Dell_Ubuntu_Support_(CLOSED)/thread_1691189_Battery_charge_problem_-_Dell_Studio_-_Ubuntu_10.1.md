### Post by citizen-ychoux on 2011-02-19
Hello everyone,

I installed Ubuntu 10.10 on my Dell Studio 17 without problems, except one.

**Problem**
My battery does not charge. The statistic on charging shows that it is at 23% capacity, and never charge more.

Running on Windows, the battery charges fine. I reached the conclustion that this is not a hardware issue.
On windows, there is a way to stop the battery charge. you can find it in an advanced battery menu that has a plane picture. I reached the conclusion that on Dell Studio, this option is similar to the "plane manager" for other Dell laptops.

I've been searching, and I found some similar problems for Dell Minis laptops: battery does not charge. The issue seems to come from the "aircraft manager". The solution here is to install this aircraft manager on ubuntu.
[http://www.ubuntumini.com/2008/12/aircraft-manager-save-battery-by.html](http://www.ubuntumini.com/2008/12/aircraft-manager-save-battery-by.html)

I tried to download an updated version of the aircraft manager from here
[https://launchpad.net/~opensource-subakutty/+archive/ppa/+packages](https://launchpad.net/~opensource-subakutty/+archive/ppa/+packages)

Unfortunately it does not work. I can't find a way to lauch it.
[B]
Summary[/B]
My battery does not charge on Ubuntu. It works fine on Windows.
The problem to me seems to be the following:
The aircraft manager seems to be activated on my ubuntu, and it prevents my battery from charging.
The solution would be to install the aircraft manager and deactivate it.
But this is my humble opinion, and maybe I'm wrong.

**Questions:**
1/ Has anyone experienced the same problem?
2/ Do you have any idea of how to fix it?

I'll continue my investigations and try to find out a way to fix this issue.
