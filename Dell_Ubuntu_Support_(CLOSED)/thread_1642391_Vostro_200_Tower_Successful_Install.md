### Post by pquinnet on 2010-12-10
After 3 attempts, I was successful at loading UBUNTU on a 2 year old DELL Vostro 200 Tower. The first attempt I did a MANUAL install dictating the sizes for : /boot, /, /usr, /home, /var, /opt, SWAP and more and it failed taking 4 hours in total time. My second attempt used the second install option to ERASE and go with the default install and that worked fine. The third attempt I did MANUAL but only dictated /boot, /, /usr, SWAP & /home leaving 20 gig at the end of the drive unused ( 160gig total size ).  It worked like a charm and took 1.5 hours elapsed time.  

I have a standard DELL VOSTRO 200 Tower with 2gb of memory & 2.2ghz speed. The whole 160gig drive was dedicated to UBUNTU and is one of two drives in the machine.  I use the BIOS BOOT MENU to switch between XP on one drive and UBUNTU on the other drive.

I have read a few threads of folks having problems with the install on the VOSTRO 200 so I thought I would chime in with at least one success.
