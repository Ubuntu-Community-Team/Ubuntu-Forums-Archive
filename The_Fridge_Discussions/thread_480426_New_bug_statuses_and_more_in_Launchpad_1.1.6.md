### Post by TheFridge on 2007-06-21
One of the agenda items at UDS Sevilla was to improve the efficiency of Ubuntu&#8217;s bug workflow. If you work with Ubuntu bugs, you may already have noticed that some of the bug status names have changed:
 [LIST] 
[*]**New:** was *Unconfirmed* 
[*]**Incomplete:** was *Needs Info* 
[*]**Invalid:** was *Rejected*. [/LIST] The meaning of these statuses hasn&#8217;t changed. However, there are two new bug statuses:
 [LIST] 
[*]**Triaged:** this bug has a complete report, has been confirmed and is ready to be fixed. 
[*]**Won&#8217;t Fix:** this bug has been confirmed but, for whatever reason, won&#8217;t be fixed. [/LIST] These two new statuses are available only to either a project&#8217;s owners or members of its Bug Contact team. So, in Ubuntu&#8217;s case it&#8217;s available to members of the Ubuntu Drivers and Ubuntu Bugs teams. This means that developers can view the *Triaged* queue knowing that an experienced triager considers those bug reports complete.
 The *Confirmed* status still works, though, so it&#8217;s down to each project&#8217;s bug workflow policy to decide whether to use *Triaged* and *Won&#8217;t Fix*.
 **More people can nominate bugs**

 To relieve the bottle-neck caused by Ubuntu drivers being the only people able to nominate bugs for release, now anyone with the relevant upload permissions for a component can also nominate bugs in that component. For example: if a developer has upload permissions for universe, she can also nominate universe bugs for release.
 **Other highlights for the Ubuntu community**

 [LIST] 
[*]Answer contacts will now receive notification of new questions in their preferred languages only. 
[*]Team members can now renew their own memberships, when their membership is close to expiry if the team is set-up with an on-demand policy. 
[*]Teams can now only join other teams with the approval of the first team’s administrator. 
[*]It is now possible to see how many and which translations diverge from upstream. [/LIST] **Find out more**

 There&#8217;s plenty more to Launchpad 1.1.6, including the fabulous new [Launchpad News blog]("http://news.launchpad.net/")! You can find the [full release notes on the launchpad-users]("https://lists.ubuntu.com/archives/launchpad-users/2007-June/001611.html") mailing list.
