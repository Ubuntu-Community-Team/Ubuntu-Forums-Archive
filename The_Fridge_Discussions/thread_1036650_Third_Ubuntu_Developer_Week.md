### Post by TheFridge on 2009-01-11
Daniel Holbach is happy to announce **the third [Ubuntu Developer Week]("https://wiki.ubuntu.com/UbuntuDeveloperWeek")**.

 From Jan 19th to Jan 23rd we&#8217;re going to have loads of awesome sessions where Ubuntu developers share their secret of success, spend time answering all of your questions, help you to get involved. It&#8217;s an awesome opportunity to get started, get to know a lot of people and it&#8217;s going to be a lot of fun.

 A lot of things are going to stay the same: we&#8217;ll have top-class talkers, top-notch talks and time for asking lots and lots of questions. One thing I&#8217;m totally excited about is this one: we&#8217;ll have a two-hour Getting Started session in English, French, German, Italian and Spanish. Fantastic! If English is not your mother tongue and you&#8217;d like to know a bit more about how it all works to feel comfortable, this is your opportunity to ask your questions.

 The [timetable]("https://wiki.ubuntu.com/UbuntuDeveloperWeek"):

 ** Day 1**

 
[LIST] 
[*]**Getting Started** &#8212; This is a true novelty and is going to be awesome. We&#8217;ll have an action-packed two-hour session to get you started for UbuntuDeveloperWeek and getting you all set up for developing Ubuntu. Be sure to bring questions. Sounds good? It gets better: we&#8217;ll have the session in English, French, German, Italian and Spanish at the same time. (*English*: #ubuntu-classroom, host: James Westby, *French*: #ubuntu-fr-classroom, host: Dider Roche, *German*: #ubuntu-classroom-de, host: Daniel Holbach, *Italian*: #ubuntu-classroom-it, host: Devid Antonio Filoni, *Spanish*: #ubuntu-classroom-es, contact: Nicolas Valcarcel) 
[*]**Packaging 101** &#8212; MOTU Daniel Holbach will talk you through the bare-bone structure of an Ubuntu package and how to get there the easiest way. 
[*]**Working well with Debian** &#8212; Debian is our most important upstream project. In their session Nicolas Valcarcel and Barry deFreese will demonstrate how to best collaborate with Debian, how to send patches to Debian, the workflow and what&#8217;s important. Barry is a Debian Developer as well, so together they&#8217;ll surely be able to answer all open questions. 
[*]**Understanding GNOME Technologies** &#8212; Dbus? Gconf? GNOME Libraries? Huh? TedGould will give a one-hour session to explain which layer is necessary for what and what exactly they do. If you&#8217;re excited about GNOME and want to start participating in the Desktop world, this is a great place to be. 
[/LIST]
 **Day 2**

 
[LIST] 
[*]**Launchpad Bug Tracking** &#8212; Launchpad developer Tom &#8220;intellectronica&#8221; Berger is going to be there to talk about Launchpad Bug Tracking. This is an awesome opportunity to ask questions, understand how Launchpad Bugs, formerly known as &#8220;Malone&#8221; works, how to use it and how it is developed. 
[*]**QA Tools** &#8212; We&#8217;ll stay on the QA side of things for a bit and Ara Pulido will talk a bit about QA tools. If you&#8217;re interested in quality assurance, bug management and making our beloved distro better, this is your opportunity to learn the tricks and help out. 
[*]**Stop Screen-Scraping, Use the Launchpad Web Service API** &#8212; Launchpad Web Service API specialist Leonard Richardson will be there to talk you through using launchpadlib, a fantastic, easy-to-use python module that will make your life a lot lot easier. No more screen-scraping, get the data from Launchpad easily. 
[*]**Creating high-quality updates** &#8212; Ever wondered how the Security team works? What CVEs are, what embargoing is, how security updates are prepared and tested? This is your opportunity to talk to Jamie, Kees and Marc and find out more. 
[*]**Debugging program crashes** &#8212; Ubuntu hacker Martin Pitt is probably in the best position to talk about examining program crashes. He wrote apport which uploads debug information of program crashes to Launchpad, wrote the tool that examines the dump and adds debug info to it and has looked at thousands of crash reports in his life. Start fixing program crashes, Martin tells you how. 
[/LIST]
 **Day 3**

 
[LIST] 
[*]**Pushing out GNOME releases to millions of users** &#8212; He never takes a rest, some think he&#8217;s actually a team of three people. Right, it&#8217;s Sebastien &#8220;sebuild&#8221; Bacher I&#8217;m talking about. He has largely been responsible for keeping large parts of GNOME in shape and he&#8217;s doing a fantastic job. Learn how to help out in Desktop land and learn all his tricks. 
[*]**Fixing Bugs in Ubuntu** &#8212; We all want to make Ubuntu better, but where to start? The answer is Harvest, a great tool to find easy tasks. Daniel Holbach will pick a few easy tasks and demonstrate what to do about them. It&#8217;s not hard to help out and make Ubuntu better. Really! 
[*]**Bazaar for packaging** &#8212; James &#8220;Merge Proposal&#8221; Westby is in a very good position to talk about the topic: he wrote bzr-builddeb, a fantastic tool to automate tasks around building packages, he&#8217;s involved in the Bazaar community and a very active MOTU. Make things easy: maintain packages in Bazaar. [IMG]http://daniel.holba.ch/blog/wp-includes/images/smilies/icon_smile.gif[/IMG] 
[*]**Kubuntu Bug Squishing** &#8212; You love Ubuntu with the big K? So does Jonathan Thomas. He&#8217;s unstoppable, he knows his stuff and he knows how to get you started helping out in Kubuntu land! Awesome! 
[*]**Using VMBuilder to create tests environments** &#8212; Søren Hansen and Nicolas Barcet will entertain you with virtual machines and the fantastic vmbuilder. Need a clean test environment for something? Don&#8217;t want to run the latest development release on actual hardware yet? These two fine men have the answer for you. 
[/LIST]
 **Day 4**

 
[LIST] 
[*]** Automated Desktop Testing** &#8212; Ara Pulido is up again and will talk to you about testing the Desktop in an automated fashion. Write just a few lines of Python code to test your application in an automated way and find regressions quickly. Fantastic! 
[*]**Ubuntu Netbook Remix Q&A** &#8212; Excited about Ubuntu mobile? Heard of great new Netbooks running Ubuntu? Neil Patel (and friends) are going to be there to talk to you about exactly that. Bring questions, get excited, get involved! 
[*]**Boot performance** &#8212; Ubuntu hacker Scott James Remnant will lead this session and give you insights into what&#8217;s happening to make Ubuntu boot faster. Find out more about the plumbing, find out how to test things, measure and improve Ubuntu. 
[*]**Kubuntu Ninja&#8217;s - Packagers in Unicorn mode** &#8212; This is by far the most melodramatic session title. One hour with Kubuntu hackers to find out what it&#8217;s all about! One thing&#8217;s for sure: Harald Sitter and Stephen Stalcup and talk to you about Kubuntu Packaging and how best to do it. It&#8217;s going to be great! 
[*]**Packaging software for Mono, for great justice** &#8212; Jo Shields and Debian Mono team expert Mirco Bauer are going to be around to talk Mono. How it&#8217;s packaged, why it&#8217;s so great and how you can help out. 
[/LIST]
 **Day 5**

 
[LIST] 
[*] **Testing your .deb with piuparts** &#8212; Lars Wirzenius is back again and will talk to you about piuparts, an awesome tool to test not just the installability of your package, but also if it does it all in a clean way. Keep installations clean, use piuparts! 
[*]**Fun with python-apt** &#8212; The unstoppable Michael Vogt brought a lot of awesome programs to Ubuntu. A lot of them use python-apt. Find out why it&#8217;s so great and how it can make your life easier. 
[*]**Bazaar and Launchpad - How to do it** &#8212; There&#8217;s a good reason why Bazaar and Launchpad are so tightly integrated: because it&#8217;s awesome and because it makes maintaining your open source project really really easy. Paul Hummer and Martin Albisetti spent a lot of time to making this integration as great as it is today and they&#8217;ll talk to you about it. 
[*]**Packaging Kernel modules with DKMS** &#8212; Kernel hacker Ben Collins has been around in Ubuntu for a long time and has taken good care of making sure that all hardware runs as seamless as it does today. DKMS is the best thing since sliced bread when it comes to maintaining separate Kernel modules in Ubuntu. He&#8217;ll show you how to do it. 
[*]**Xubuntu** &#8212; Cody Somerville and the Xubuntu team invite you to one hour of Xubunturama. Get involved, find out why it&#8217;s so great and what you can do to make it better. 
[/LIST]
 Do us a favour and help spread the word everywhere. [DIGG THIS]("http://digg.com/linux_unix/3rd_Ubuntu_Developer_Week_announced").
