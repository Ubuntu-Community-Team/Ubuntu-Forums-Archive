### Post by dmizer on 2007-05-10
okay ... your best samba server guide is the one in the first link in my sig.  have a look through it.  you can modify it to allow password free access by changing the following options:
1) security = user
change to
security = share

2) guest ok = no
change to
guest ok = yes

beyond that ... you should follow the entire directions in the post except for adding your windows users to your ubuntu machine.

you should end up with something similar to the following for your smb.conf:
```
[global]
    ; General server settings
    netbios name = [COLOR="Red"]*change this to something unique*[/COLOR]
    server string =
    workgroup = HOME
    announce version = 5.0
    socket options = TCP_NODELAY IPTOS_LOWDELAY SO_KEEPALIVE SO_RCVBUF=8192 SO_SNDBUF=8192

    passdb backend = tdbsam
    security = share
    null passwords = true
    username map = /etc/samba/smbusers
    name resolve order = hosts wins bcast

    wins support = no

    printing = CUPS
    printcap name = CUPS

    syslog = 1
    syslog only = yes

; NOTE: If you need access to the user home directories uncomment the
; lines below and adjust the settings to your hearts content.
[homes]
    valid users = %S
    create mode = 0600
    directory mode = 0755
    browseable = no
    read only = no
    veto files = /*.{*}/.*/mail/bin/

[Storage2]
    path = /drive2
    veto files = /"My Documents"
    browseable = yes
    read only = no
    guest ok = yes
    create mask = 0644
    directory mask = 0755
    force user = katlyn
    force group = katlyn
```

also ... the "My Documents" with quotes may or may not work.  you may have to change the directory name to something without a space and remove the quotes.  My_Documents or My-Documents should work fine, then the veto line would look like so:
```
veto = /My_Documents
```

also don't forget to change the netbios name option to something easily recognizable as your ubuntu machine when browsing the network from windows.

a final note from the makers of samba regarding the "veto files" option:
>  Setting this parameter will affect the performance of Samba, as it will be forced to check all files and directories for a match as they are scanned.

what this means is that you should specify a SPECIFIC directory on /drive2 to be shared so that "My Documents" is not included without having to veto it.  the same goes for "[homes]".  rather than sharing your entire home directory, you should share a specific directory designated for sharing within your home directory so you don't have to worry about veto, and you don't have to worry about some fool mucking about in your home config files.

### Post by dmizer on 2007-05-11
i think you've mistaken the purpose of smb.conf and samba.  configuring smb.conf will allow other computers to view your ubuntu shares.  it will not allow you to view shares on other computers from ubuntu.

since i know that the above smb.conf will work ... the only thing left now is firewall.  xp firewall enabled? how about zone alarm? *groan* norton?  on your linux side, are you running firestarter?  if not, try this in windows:
1) right click on "my computer" and select "map network drive".
2) in the section listed "folder" enter the following line:
```
//[COLOR="Red"]ubuntu-netbios-name-from-smb.conf[/COLOR]/Storage2
```
replace "ubuntu-netbios-name-from-smb.conf" with whatever you actually have listed next to the "netbios name =" option.
3) click "finish" and you should see your network drive listed when you open "my computer"

for information on how to see/mount the other three computers on your network from ubuntu, see the second link in my sig.
