When you enabled the STA driver for you card (driver=wl0  above) it often "blacklists" the b43 and ssb files automatically. Unfortunately the b44 driver for your ethernet ( BCM4401-B0) requires ssb.  You might check

```
sudo gedit /etc/modprobe.d/blacklist.conf

```

and see if that is the case.

### Post by TBABill on 2011-04-14
+1 When you install the STA driver, the system should place entries into /etc/modprobe.d/blacklist.conf that state > blacklist b43
blacklist b43legacy
blacklist ssb
blacklist b44
blacklist brcm80211
 
In your case you need the b44 to be working to support your wired ethernet card so make sure that is NOT listed in that file. You may also need ssb to NOT be blacklisted as well. If it is/they are, just precede with # to see if it reenables the eth0 on restart so at least you get to wired internet.
 
If those entries are not in that file, place them there (as root) by ```
gksudo gedit /etc/modprobe.d/blacklist.conf
``` and then paste them to the end.
 
You may also need to do a ```
lsmod
``` so you can see all loaded drivers. If you see any of those listed above you can temporarily disable them by ```
sudo modprobe -r b43 b43legacy brcm80211 ssb wl
```
 
Then just ```
sudo modprobe wl
``` to reenable only the wl module, which is the actual driver module for the STA driver. Then see if wireless is working again.
