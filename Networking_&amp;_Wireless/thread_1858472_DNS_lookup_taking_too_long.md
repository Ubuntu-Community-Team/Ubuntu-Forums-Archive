### Post by jonobr on 2011-10-12
Hi 

The trace seems to have caught a packet going from
192.168.1.2  > 192.168.1.1   <-- this is the first IP address in resolv.conf


It gets no response, times out and then sends another packet to the second DNS

 192.168.1.2.56793 > 201.17.128.109

It gets a response from this.

You can do one of two things,

Try commenting out the first line in resolv.conf

(put a hash before the 192.168.1.1 address so it doesn't use it.)
This way it will only use 209

Or you can figure out if your router will handle dns resolution and change to that.

Maybe try changing 192.168.1.1 to 192.168.0.1

or try a different dns server altogether which may be faster
For this, replace both lines with 8.8.8.8
(public dns)
