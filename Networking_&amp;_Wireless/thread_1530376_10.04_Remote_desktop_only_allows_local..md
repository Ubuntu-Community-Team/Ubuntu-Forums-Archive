### Post by YesWeCan on 2010-07-13
Some thoughts.

You can do Remote Desktop (aka vino) that allows you to see the same desktop remotely as is displayed on the computer's physical display. Or you can use something like tightvncserver that lets you see one or more remote windows that are different and independent from the one that is shown on the physical display. Vino is useful if you are doing something like giving remote assistance to a person sitting at the computer. With tightvncserver it is not necessary that you are logged on to your work machine at work.

Another thing is that remote desktop is not secure. The connection is not encrypted. So this may be a consideration when attempting remote desktop across the public internet. So most people would use SSH to create an encrypted tunnel for the transmission.

Without encryption, using remote desktop/vino on the work server, you need to create a port-forward on the work router to forward WAN port 5900 packets to the ubuntu server's LAN IP address. On you home client you need to run something like Vinagre to view the remote desktop and tell it the WAN address of the work router and specify port 5900.

However, because it is not secure I would never do this. A fairly simple method is to install SSH server on the work machine and then create a SSH port forward on the home client: [https://help.ubuntu.com/community/SSH/OpenSSH/PortForwarding](https://help.ubuntu.com/community/SSH/OpenSSH/PortForwarding)
An even better system is an openVPN tunnel, which is what I use, but is a little more tricky to set up: [http://www.openvpn.net/index.php/open-source/documentation/miscellaneous/78-static-key-mini-howto.html](http://www.openvpn.net/index.php/open-source/documentation/miscellaneous/78-static-key-mini-howto.html)

BTW I use an openVPN point-to-point tunnel from my home ubuntu client to my remote ubuntu server. I run tightvncserver on the server that exports a desktop on port 5901. I run Vinagre on my home client to access the remote desktop. It works extremely well.

See also: [http://ubuntuforums.org/showthread.php?t=290055](http://ubuntuforums.org/showthread.php?t=290055)
