Hello,

I have set up a content filtering for my child with ubuntu 9.10.

I want the content filtering to be on only when the "child" session is open  and I do not want my child to be able to desactivate it
 
This is how I have done it:

I created an account on opendns (fantastic service by the way)

I wrote the following script:

/usr/sbin/opendns (owned by root with 755 permissions)
--------------------------------
#!/bin/bash

# /usr/sbin/opendns
# /etc/resolv.child
# /etc/resolv.adult
# link /etc/NetworkManager/dispatcher.d/opendns
# line opendns in crontab as root (sudo crontab -e)

if users | grep -c "child" > /dev/null ; then
	wget -O - -q --http-user=opendnslogin --http-passwd=opendnspassword [https://updates.opendns.com/nic/update](https://updates.opendns.com/nic/update)
	cp /etc/resolv.child /etc/resolv.conf
else
	cp /etc/resolv.adult /etc/resolv.conf
fi
--------------------------------
this script will, if "child" is connected, update opendns with my IP address (I have a dynamic IP address) and change the dns (resolv.conf file) otherwithe it will put normal DNS and wont update opendns

I created the two folowinf files :

/etc/resolv.child (owned by root)
--------------------------------
# generated by opendns - resolv.child

# /usr/sbin/opendns
# /etc/resolv.child
# /etc/resolv.adult
# link /etc/NetworkManager/dispatcher.d/opendns
# line opendns in crontab as root (sudo crontab -e)

nameserver 208.67.222.222
nameserver 208.67.220.220

--------------------------------
208.67.222.222 and 208.67.220.220 are the opendns dns
(I list the files I used for futur debugging and in case I want to remove all of it)

and

/etc/resolv.adult (owned by root)
--------------------------------
# generated by opendns - resolv.adult

# /usr/sbin/opendns
# /etc/resolv.child
# /etc/resolv.adult
# link /etc/NetworkManager/dispatcher.d/opendns
# line opendns in crontab as root (sudo crontab -e)

nameserver 194.2.0.20
nameserver 194.2.0.50

--------------------------------
194.2.0.20 and 194.2.0.50 are my normal ISP dns

I added the line : 
ALL ALL=NOPASSWD:/usr/sbin/opendns
at the end of my /etc/sudoers file so opendns can be run as root by anyone without prompting for password

I added a link /etc/NetworkManager/dispatcher.d/ to runthe scrip when the computer connects or reconnects
ln /usr/sbin/opendns /etc/NetworkManager/dispatcher.d/opendns

Also to check and update every 5 minutes I added the folling line 

*/5 * * * * /usr/sbin/opendns

with "sudo crontab -e" command

I also added the "sudo opendns" command in the satrting program of my child session (which he could desactivate, that is why I also use crontab).

Up to now it seems to work correctly 

Any comment on any security or other problem with what I did would be welcome
