### Post by spookybathtub on 2014-01-15
What is the correct format of the known_hosts file?  I thought each line begins with ssh-rsa, followed by a long string, then an optional identifier.
But looking at mine, I see this on each line:
|1|xxxxxxxxxxxxxxxxxxxxxxxxxxxx|xxxxxxxxxxxxxxxxxxxxxxxxxxxx *public-key-string* [I]identifier
[/I]
What are those two 28 character strings at the beginning?  (They are random characters, not X's.)  I'm trying to add a public key from a new computer, which was generated by **ssh-keygen -t rsa**.  It of course does not have this prefix.  And when I try to login from that machine, I get "Permission denied (publickey).

---

### Post by spookybathtub on 2014-01-15
Oops!  I was mixing up known_hosts with authorized_keys.  Problem solved now.  And I discovered that part at the beginning is a hashed version of the hostnames.

### Post by Lars Noodén on 2014-01-16
More details about the *known_hosts* file format are given in the manual page for [sshd](http://manpages.ubuntu.com/manpages/saucy/en/man8/sshd.8.html) down in the section "SSH_KNOWN_HOSTS FILE FORMAT"  And if you want to turn off hashing of known hosts, that is done in [~/.ssh/config](http://manpages.ubuntu.com/manpages/saucy/en/man5/ssh_config.5.html).
