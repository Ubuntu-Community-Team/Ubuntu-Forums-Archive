### Post by nickdbliss on 2008-07-14
From a security point of view, two things i consider more important. First is SSid broadcast and encryption. Since SSID broadcast is off and u have wep and mac filter on. So ur network is safe enough. Even wep and mac filter alone will do. That is what i learned when i was doing course in ethical hacking...

DNS is domain name server. When u type google.com ur browser asks dns where this text points to. Or which IP is associated with [www.google.com](www.google.com). So it shows the relevant page else gives error. It helps since u dont have to remember IPs of various sites then.
And i think using ur ISP's DNS is good option. Yeah it can be slow like u said, when it tries to resolve the web address. Open DNS is free, dont know about their performance but their is privacy issue as u are sharing ur request with third party.
Yes your internet connection goes through ur ISP server.
