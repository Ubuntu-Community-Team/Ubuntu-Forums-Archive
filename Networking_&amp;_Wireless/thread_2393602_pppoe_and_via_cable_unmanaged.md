### Post by danjde on 2018-06-05
Hi Friends,
I'm using Ubuntu 16.04 Gnome edition for my laptop.
Last day, I've installed pppoeconf and added a new pppoe connection for a work network connection, and all works fine.
Today, at home, connecting my laptop, surprisingly I realize that the connection is not established.
From Gnome panel the cable network menu, tell me: "via cable unmanaged".

Using ifconfig I can see that the wireless and cable device are presents.
And using "dhclient" the connection is established correctly, but not automaticaly as before, simply at the normal Ubuntu startup.

I've removed and purged pppoeconf, but with no luck :-( 

Investigating, I've found on etc:


```
/etc/network/interfaces:pre-up /bin/ip link set enx000ec6a172e7 up # line maintained by pppoeconf
/etc/ppp/ip-down.d/0clampmss:# Disable MSS clamping (autogenerated by pppoeconf)
/etc/ppp/peers/dsl-provider:plugin rp-pppoe.so
/etc/ppp/ip-up.d/0clampmss:# Enable MSS clamping (autogenerated by pppoeconf)
```



Do you have any idea how to get out of this impasse? :-)


Many many thanks!

Davide

### Post by danjde on 2018-06-05
..I've found a simple "workaround":

I commented on /etc/network/interfaces:


```
#auto dsl-provider
#iface dsl-provider inet ppp
#pre-up /bin/ip link set enx000ec6a172e7 up # line maintained by pppoeconf
#provider dsl-provider
```

and changed my network device (enx000ec6a172e7 usb eth adapter) from "manual" to "dhcp"

and now the wired connection comes up automatically, 

but on Gnome menu panel remain the "via cable unmanaged".
