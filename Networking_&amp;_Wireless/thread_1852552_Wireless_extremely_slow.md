### Post by dri-ft on 2011-09-30
Hi all,

I recently installed Ubuntu on my friend's old laptop to try and make it useable again. Everything runs relatively quickly, which is great, but there are some extremely unusual problems with using the internet wirelessly.

First of all, the internet works 100% fine when it is wired to the router. (That's how I'm making this post.)

However, when using the wireless, it is extremely slow... FOR CERTAIN SITES. Some sites load relatively quickly - seemingly just as fast as with the wired connection. Others seem to go grindingly slow, taking several minutes or more to load fully, or never loading at all.

One strange thing is that downloading updates for ubuntu on the wireless seemed to work fine and go at a completely reasonable pace, upwards of 50kb/s, which is plenty. Another is that google searches load almost instantly in Chromium - I guess this is to do with some kind of google-specific caching in Chrome/Chromium, as it didn't seem to be the case in Firefox, but it still seems like a strange contrast with the slow speed of most other sites.

One thing that I discovered while searching for a solution to this problem was that the problem of certain websites being slow (while others load fine) could be a consequence of IPv6 lookups, which apparently sometimes get stuck and cause this - however, I don't see why this would be the case for the wireless and not the wired connection.

Thanks all. I hope someone can shed some light on this extremely strange and mysterious situation. Happy to help with providing any more information as much as I can.
