There seems to be a dog-fight going on here between two drivers -> **Kurtis_Williams said:**
> ```

brcmsmac              534235  0 
brcmutil               15066  1 brcmsmac
cordic                 12518  1 brcmsmac
b43                   369639  0 
mac80211              534884  2 **[COLOR="#FF0000"]brcmsmac,b43[/COLOR]**
cfg80211              416271  3 **[COLOR="#FF0000"]brcmsmac,b43[/COLOR]**,mac80211
ssb                    56410  1 b43
bcma                   41297  3 **[COLOR="#FF0000"]brcmsmac,b43[/COLOR]**
 
```
..and you also seem to have manually blacklisted the "wl" driver. Is it or was it installed?

Just to make sure the "wl" driver is out of the scene, please do (in a terminal) -
```
sudo apt-get purge bcmwl-kernel-source
```
Then you may safely remove the "/etc/modprobe.d/blacklist-local.conf" file, it will be of no use after above, will only create confusion. To remove it -
```
sudo rm /etc/modprobe.d/blacklist-local.conf
```

Then to settle the dog-fight, we need to ask one of the fighting drivers to sit back and relax. I think we should ask the b43 first. Please blacklist it to prevent it from loading -
```
echo -e "blacklist b43\nblacklist ssb" | sudo tee -a /etc/modprobe.d/blacklist.conf
```
This will blacklist the drivers b43 and its supporting ssb module.

Reboot and check -
```
lsmod | egrep 'b43|ssb|brcm'
```
You should only see brcmsmac and its supporting drivers, not b43 or ssb in the output. If so, can you connect now? How stable is the connection if you can?

**PS:**
If yours is one of those routers that are listed to be using WPA/WPA2 mixed mode in the "nm-tool" part, please log into your router/access-point, and change the encryption type to pure WPA2-AES(CCMP). No mixed mode, no TKIP. Save the changes and reboot the router. Then try connecting again.
