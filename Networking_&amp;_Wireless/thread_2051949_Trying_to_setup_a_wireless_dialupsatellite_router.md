### Post by virgomoon on 2012-09-02
Good morning, all,

I have a desktop box running and Ubuntu Server setup.  My plan is to have 2 wireless NICs with SSIDs Dialup and Satellite.  Connecting to Dialup will route your traffic to the dialup connection to the internet while connecting to Satellite will route your traffic through the Satellite connection currently provided by a stand-alone router over ethernet.  I would also like to be able to use any services provided by the server (file sharing, print sharing, etc) regardless of which I'm connected to wirelessly.

I figured I'd start by getting the Dialup side to work, which it does.  I use wvdial to an external modem and use iptables to do the NAT and routing from wlan0 to ppp0.  For a while I had eth0 connected to the Satellite but just figured all the traffic through wlan0 would automatically go to ppp0 since that's how I have iptables set up.  But, when I shut down the dialup connection (ppp0) all traffic gets routed through the ethernet through eth0.  This is not only defeating the purpose by subverting the traffic isolation from wlan0 to ppp0 only but it's showing that I'm doing something clearly wrong or have bad assumptions about routing.  I'm sure this has something to do with the 0.0.0.0 default gateway, but I'm not quite sure what to do about it or how to proceed.

Any help or comments would be greatly appreciated.
