### Post by henrik10 on 2015-01-25
Having an elderly laptop and (what I believe is) as good idea, I'm trying to achieve the following:

I want the laptop (a HP Pavilion 9821eo, AMD Turion(tm) 64 X2 Mobile Technology TL-64 2200MHz, 3Gbyte DIMM DDR2 667 MHz) to act as a form of mobil OpenVPN-gateway, to a temporarily set up network. Using a WiFi-adapter I connect to what-ever public access-point is available (i.e. at a café), set up the VPN-connection and share the VPN-connection, through the ethernet-port where a [WAP54G wireless access-point]("http://support.linksys.com/en-us/support/accesspoints/WAP54G") is connected. The other people in the group will then connect to "my" access-point, instead of the public one, and will access our LAN (and the internet) through the VPN-connection. Schematically it looks something like this:

```

--------------------------
                       |

                      (  )
                     (    )
                    wireless
                      users

[/FONT]
```
I have previously used a MacBook instead of the Ubuntu, which is sort of what is my "conceptual base", but I would like to go the Ubuntu-way for security.
My Ubuntu-starting point is the [Internet/ConnectionSharing documentation]("https://help.ubuntu.com/community/Internet/ConnectionSharing") (ICS-doc), which has gotten me really far, as to: Connection is up and running, and sharing is taking place: It's just that performance is really poor.

When using [speedtest.net]("http://www.speedtest.net/") on the Ubuntu, I can achieve approx. 15 Mbps download, and 8 Mbps upload. When I connect one machine to the WAP54G, I get 2 Mbps download and 2 Mbps upload (numbers are consistent over several days, against the same speedtest.net-server), and the Ubuntu-machine drops to 8 Mbps download and 2 Mbps upload. Unplugging the ethernet-cable and flushing iptables- bring me back to 15/8 Mbps. Using the MacBook (2 GHz Intel Core 2 Duo, 4 GB 1067 MHz DDR3) I'm able to achieve 20/10Mbps (L2TP over IPSec, so I believe it should be a bit faster, on the sacrifice of lesser security), but also pretty much push that through to the one machine connected to the WAP54G. My guess is that my Ubuntu-setup is less than optimal, and that I'm sacrificing a lot of performance on that account.

IP-addresses are allocated using isc-dhcp-server (mainly because I'm familar with that, but also I've dumped dnsmasq so far), with [COLOR=#daa520]*option domain-name-servers 8.8.8.8,8.8.4.4;*[/COLOR] instead of a local DNS-cache.

I have tried dnsmasq as described in the ICS-doc, but that also killed speed on the Ubuntu-machine itself. So-far I've wanted it to be as "out of the box Ubuntu desktop" as possible, so I've refrained from disabling NetworkManager, but I guess it could come to that, so I'm open to that as well.

The OpenVPN connection terminates on 84.22.*.* (I don't have access to OpenVPN/PPTP setup there and don't have administrative priviledges there either).
