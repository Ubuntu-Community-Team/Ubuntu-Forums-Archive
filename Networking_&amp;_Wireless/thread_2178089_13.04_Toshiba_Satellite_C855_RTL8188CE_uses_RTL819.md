### Post by varunendra on 2013-10-03
> **brutikk said:**
> Why is it 8192CE and not 8188CE?

Usually drivers are designed so they can control a whole family of cards. rtl8192ce is such a driver and is the correct for the RTL8188CE card.
You can always check if a driver is correct or not for a device by looking at the devie's VID : PID (vendor ID : Produce ID, can be found by "lspci -nn") and checking 'modinfo <driver>' to see if it exists in its "alias" list.

As a quick shot, please try this -
```
sudo modprobe -rv rtl8192ce
sudo modprobe -v rtl8192ce swenc=1 ips=0 fwlps=0
```

This is a temporary change that will be lost at next boot, so try reconnecting without rebooting. If it helps stabilizing the connection, we'll make it permanent. If not, please follow the "Wireless Script" link in my signature, download and run the script as per instructions there, and post back the report it generates.
