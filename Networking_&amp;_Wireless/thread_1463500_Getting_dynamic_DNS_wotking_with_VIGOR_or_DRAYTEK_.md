**[COLOR="Red"][SIZE="5"][/SIZE]THE SOLUTION BELOW DOES NOT WORK [SIZE="3"][/SIZE]but remains an example of how to code to get Dynamic DNS ddclient working with SNMP.  The code below gets the IP address of the gateway i.e. the ISP's network gateway address! But the Draytek SNMP MIBs are broken anyway and dont report the IP address of the external (ADSL interface), so there is no way to make SNMP work with ddclient on this modem unless Draytek fixes their MIBs.[/COLOR]**  

You can probably use "expect" to run an ssh session with the Vigor, or use web scraping, but it would have been SO much easier if the modem did as it should do with MIBs!

PS the MIBs used in my example script are wrong but if you change them to the right MIBs, and put a bit more logic into the script it would work.  I gave up because the Draytek doesn't populate its MIBs properly

That is, look up ADSL interface using information obtained as below (which shows how it **should** look not what the Draytek reports!)
```

We are interested in ADSL interface
snmpwalk -c public -v 2c -O a 192.168.199.1 1.3.6.1.2.1.2.2.1.2
IF-MIB::ifDescr.1 = STRING: LAN
IF-MIB::ifDescr.4 = STRING: ADSL

Then look up the "index number" for ADSL = 4
snmpwalk -c public -v 2c -O a 192.168.199.1 1.3.6.1.2.1.2.2.1.1
IF-MIB::ifIndex.1 = INTEGER: 1
IF-MIB::ifIndex.4 = INTEGER: 4

then look up the IP address entry that has the same index # = 4
snmpwalk -c public -v 2c -O a 192.168.199.1 1.3.6.1.2.1.4.20.1.2
IP-MIB::ipAdEntIfIndex.11.22.33.44 = INTEGER: 4
IP-MIB::ipAdEntIfIndex.192.168.199.1 = INTEGER: 1

Finally look up the actual IP address (probably redundant but anyway...)
snmpwalk -c public -v 2c -O a 192.168.199.1 1.3.6.1.2.1.4.20.1.1
IP-MIB::ipAdEntAddr.11.22.33.44 = IpAddress: 11.22.33.44
IP-MIB::ipAdEntAddr.192.168.199.1 = IpAddress: 192.168.199.1

ADSL address = 11.22.33.44


```

It is easier to just use checkip.dyndns.org instead of SNMP, to do this update your ddclient.conf file
```

# Configuration file for ddclient generated by debconf
#
# /etc/ddclient.conf

# use=cmd
# cmd="/usr/local/bin/get_vigor_addr 199.199.199.1"
# Configuration file for ddclient generated by debconf
#
# /etc/ddclient.conf

# use=cmd
# cmd="/usr/local/bin/get_vigor_addr 192.168.121.1"

pid=/var/run/ddclient.pid
use=web
web=checkip.dyndns.org,web-skip="IP Address"
#
login=mylogin
password=mysecretpassword
protocol=easydns
server=members.easydns.com
myserver.mydomain.com

```

Note lack of quotes in login and password lines, had major troubles until I got rid of all quotes in those lines.

================ Original post below ==============
OK, I've got dynamic DNS working with my VIGOR DRAYTEK 2710 (2710n, 2710vn) ADSL modem/router.

Access the modem's web page, then
[LIST]
[*]Log in
[*]Go to **System Maintenance** menu
[*]Go to **Management** menu
[*]Tick the **Enable SNMP Agent** box
[*]Press the **OK** button at the bottom of the page
[*]Wait while the modem reboots
[/LIST]

Then on your Ubuntu system
```

sudo apt-get install ddclient snmp
sudo /etc/init.d/ddclient stop

```

Then use your favourite editor to put this file called *get_vigor_addr* into the /usr/local/bin directory (you may have to use sudo to do this)
```

#!/bin/sh
# set -xv
# shell script to get ADSL adapter address (ISP side) from VIGOR DRAYTEK ADSL router
#
# enable SNMP in the router (System Maintenance menu, then Management)
# install snmp package which includes snmpget- for Ubuntu you just do "sudo apt-get install snmp"
#
# Author: Indulis Bernsteins
# indulis
# 21Cmail
# com

my_addr=$1
the_rest=$2

/usr/bin/snmpget -Os -c public -v 1 $my_addr 1.3.6.1.2.1.2.2.1.2
[COLOR="Red"]Then look up MIBS 1.3.6.1.2.1.2.2.1.1, 1.3.6.1.2.1.4.20.1.2, 1.3.6.1.2.1.4.20.1.1, and add more LOOKUP LOGIC HERE[/COLOR] 


```

Then
```
sudo chmod a+x /usr/local/bin/get_vigor_addr
```

Modify your /etc/ddclient.conf file to match your own dynamic DNS provider, username and password, and ADSL modem IP address

```

# Configuration file for ddclient generated by debconf
#
# /etc/ddclient.conf

use=cmd
cmd="/usr/local/bin/get_vigor_addr 192.168.199.1"
protocol=easydns
server=members.easydns.com
login=my_own_login_name
password=wouldnt_you_like_to_know
my_server_name.my_own_domain.com

```

Then try your configuration
```

sudo /etc/init.d/ddclient stop
sudo ddclient -daemon=0 -noquiet -debug
```

Once it is working OK (ignore any messages about "uninitialized value in string")
```

sudo /etc/init.d/ddclient start

```
