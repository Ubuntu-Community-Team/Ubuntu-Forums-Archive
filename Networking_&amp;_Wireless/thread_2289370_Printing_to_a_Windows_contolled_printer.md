I have a small home network  - wired and wireless - through a broadband router. The main network printer is connected a Windows XP PC, cabled to the router.  One of the wired PCs runs Lubuntu 14.04 and can also boot a variety of puppy Linuxes, through a Grub4Dos boot manager.
They have all been set up via CUPs and can connect to the network printer. However, while the puppy Linuxes will do a straightforward print, without authentication, Lubuntu always asks for authentication, which requires the input of a password. Since both the user and password are the same for the Lubuntu and the Windows machine, it is no great difficulty to supply the password. However I am puzzled that none of the puppy Linuxes (nor indeed any Windows PCs in the workgroup) need this step. I am somewhat reluctant to input the password permanently  into CUPS during the setup as a one-off measure, since it doesn't seem to be  necessary with other OSes. 
I would be grateful for any suggestions or help in getting Lubuntu to bypass the authentication process, in order to print to the Network printer.
