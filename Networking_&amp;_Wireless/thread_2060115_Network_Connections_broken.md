### Post by SlugSlug on 2012-09-19
You dnt realy need network manager to get online - in fact I tend to remove it!

```
gksudo gedit /etc/network/interfaces
```Pop this in but change for your ip's (get that via ifconfig on working box)

```
auto lo
iface lo inet loopback

auto eth0
iface eth0 inet dhcp

```
or

```
auto lo
iface lo inet loopback

auto eth0
iface eth0 inet static
address 192.168.0.2
netmask 255.255.255.0
gateway 192.168.0.1
nameserver 192.168.0.1
```
```
sudo /etc/init.d/networking restart
```(Am I the only one that hits tab after /et? nearly lost this post)

### Post by steeldriver on 2012-09-19
In your original post you said your wired connection "was disabled", and now you don't seem to be receiving an IPv4 address via DHCP

IMO the most likely reason for that in an office environment - or at least the first thing to check - is that your network admin has removed your interface's MAC from their whitelist

If that's NOT the case then the next thing I would check is that a driver update has not disabled the device

```
sudo lshw -C network
```(make sure a driver is listed and the device is not marked DISABLED)

I would also check that the network-manager service is stopped just in case it is interfering with ifup

```
sudo service network-manager stop 
```
