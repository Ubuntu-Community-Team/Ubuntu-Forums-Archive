### Post by linuxonbute on 2010-05-25
With 2 cards in the machine on the same subnet there will probably be trouble with routing. Also you might need a DNS server set up.
Does your router provide the information?
It's not something I have tried myself but, at least temporarily this might work:

Start up with **wireless ON   ethernet OFF** 

see what route -n says.

If you do not get a line :

0.0.0.0         192.168.1.1   0.0.0.0         UG    0      0        0 wlan0
your machine does not know how to get out onto the internet.

do this:
sudo route add default gw 192.168.1.1 wlan0

then check route -n and the line should be there.

now try:
ping 192.168.1.1
ping 209.85.229.147
ping google.com

If you can ping except for the last one then do this:

sudo echo "nameserver 213.162.97.65">>/etc.resolv.conf

[COLOR="Red"](notice the 2 ' > 'which will add the line to the end of resolv.conf )[/COLOR] 
then try :
ping google.com again.
It should work then.

Let us know how you get on.
