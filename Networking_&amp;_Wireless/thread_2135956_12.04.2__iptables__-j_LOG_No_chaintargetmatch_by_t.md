### Post by Doug S on 2013-04-16
As far as I know iptables in 12.04.2 works fine. Into a previously non-existant iptables rule set, I entered your exact command and it worked fine. Of course, I created the DROPLIST chain first, because otherwise I would get the exact same error.```
doug@virt64-02:~$ sudo iptables -I DROPLIST 1 -s 207.189.109.125 -j LOG --log-prefix "LOGDROP 207.189.109.125"
iptables: No chain/target/match by that name.
doug@virt64-02:~$ sudo iptables -N DROPLIST
doug@virt64-02:~$ sudo iptables -I DROPLIST 1 -s 207.189.109.125 -j LOG --log-prefix "LOGDROP 207.189.109.125"
doug@virt64-02:~$ sudo iptables -v -x -n -L
Chain INPUT (policy ACCEPT 29 packets, 1620 bytes)
    pkts      bytes target     prot opt in     out     source               destination
