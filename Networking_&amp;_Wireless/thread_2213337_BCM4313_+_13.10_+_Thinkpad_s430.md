Because it IS supposed to work with this card and it *usually* does, just with some issues that don't always affect all users.

So if this 'best result' is satisfactory, I think you may safely mark the thread as [SOLVED]. If not, I do have a further suggestion - about trying a newer version of the native driver as follows -

Make sure you have the basic packages to compile a driver are pre-installed -
```
sudo apt-get install linux-headers-generic build-essential
```

Then -

[INDENT]**1)** Download the "backports-3.12.8-1" package from here : [http://drvbp1.linux-foundation.org/~mcgrof/rel-html/backports/](http://drvbp1.linux-foundation.org/~mcgrof/rel-html/backports/)

**2)** Right-click the downloaded package > "Extract here". It will extract all its contents in a directory of same name.

**3)** Assuming this extracted directory is in your "Downloads" directory, open a terminal (Ctrl-Alt-T) and run the following commands to compile and install the driver -
```
cd Downloads/backports-3.12.8-1
make defconfig-brcmsmac
make
sudo make install
```
Watch out for errors and report back if there are any.[/INDENT]

Assuming everything will be finished smoothly, get rid of the blacklist you did manually, plus the sta driver -
```
sudo apt-get purge bcmwl-kernel-source
```
Reboot, and you should now be using the newer version of the native driver. Hopefully, it should deliver the nicer performance that we expect from it. If not, you can always revert to the sta driver.

**PS:**
You don't need to manually do the blacklisting of other broadcom drivers if you install the sta driver. It automatically does that by creating a separate blacklist file. Purging (not 'removing') removes this blacklist file too, so if doing "apt-get purge.." we don't need to separately remove the blacklist file either, but the blacklisting you did manually in the main blacklist file must be removed manually.

### Post by varunendra on 2014-04-04
Long ago we had discovered that using an older version of the sta driver used to fix the problems for almost all users. I had collected some info and wrote up a [post=12629619]**post**[/post] about how to compile and install that one.

It looks like some changes in either the kernel or the driver(s) have brought the same or similar problems back. And unfortunately, that driver version is no more available for newer kernels. Besides, the native "brcmsmac" became much better than the sta one since kernel 3.8.

If you wish, you may try the latest backported version of the "brcmsmac" driver using exactly the same steps as I suggested in post #18 here but the version no. replaced by the latest one (backports-3.13.2-1). Although I think we have tried that one at least once recently and it didn't make much difference.

Since the sta driver does not get updated very frequently, I suggest you keep trying the native one (brcmsmac) after each kernel upgrade to see if it became a better one for you. Of course this advice assumes there are no other factors (hardware, extra tweaks, external factors) that are affecting the performance.
