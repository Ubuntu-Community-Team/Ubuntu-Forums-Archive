### Post by TheFu on 2022-12-29
Ubuntu 21.04 support ended in Jan 2022.

Move to a supported release.

BTW, calling a release an LTS doesn't make it true.  20.04 and 22.04 are the current LTS releases for servers and the Gnome3 DE.  If you are running one of the other desktop versions, then you'll probably want to install 22.04 with that DE, since most desktop ubuntu flavors only have 3 yrs of support, not 5 like Ubuntu Server LTS would get.

Any October or odd year release is never an LTS version.  21.04 ---> April 2021. That's an odd year and it only gets 6-9 months of support, so expect to move to a new OS every 6 months if you install one of those non-LTS versions.

If you want to see which hardware is specifically supported, the best way I know is to get the release you are considering, download it, put it onto an 8GB flash drive (using the required method), and boot into that specific version.  Then you'll 100% know what works.

In theory, 20.04 has support for that wifi adapter: [https://ubuntu.com/certified/component/934](https://ubuntu.com/certified/component/934)   If you run lsusb, the device ID might show that the name and identifier are different from the certified version.  I suspect there's a driver version that is available, just not included. Someone else will need to help with finding it, pulling the code down, compiling and creating the driver file, then showing where to modify the driver loading config files so it loads.

Or just try 22.04 and perhaps everything is fixed in that release.

and suggest one benefit of the LTS releases is the kernel stack choice, with the installation media you use dictating the default kernel stack choice. You weren't specific to Ubuntu product (only release being 21.04), but I'll use 20.04 as example.

Ubuntu has two kernel stack choices; GA or the most *stable* (also *oldest*) & HWE (*newer hardware enablement stack for newer hardware*).

- 20.04 was available with the GA kernel stack, meaning 5.4 linux kernel.  This was the default for Server ISOs, and *flavor* ISOs of 20.04 or 20.04.1

Ubuntu 20.04 LTS media was also available that used the 5.8 kernel from 20.10, 5.11 from 21.04, 5.13 from 21.10 & finally 5.15 from Ubuntu 22.04 LTS; though post install with normal upgrades all kernels updated to the final 5.15 linux kernel.  This HWE stack was the default for *flavor* media of 20.04.2 and later, or Ubuntu Desktop ISOs.

The significance of the kernel stack, is *drivers *are linux kernel modules, thus using different kernels means you have different kernel modules (aka *drivers*). There was also some OEM kernel stack options, but I've omitted them as only some ISOs provide them & they are geared for specific hardware only (may or may not benefit you).

### Post by morrownr on 2022-12-31
> **kc1di said:**
> Kubuntu 22.04 works well here with similar wireless card.  Good Luck. 

As others have advised you need to install a supported version 21.04 is a dead end.  20.04 or 22.04 should work for you.

Good advice though I would strongly suggest 22.04 or later. The reason is that wifi has gone from complicated to really complicated as WiFi 6 and WiFi 6e have shown up... and then there is WPA3. It takes many parts of an os to get all of this right. If you use 22.04, I would recommend upgrading to the 5.19 HWE kernel as soon as it is available.

Background: support for the mt7921 chipset originally showed up in kernel 5.12. Support for the mt7921au, the usb version of the chipset, showed up to in kernel 5.18, before adapters were available for purchase in July of this year. Kernel 5.19 fixed a few bugs and added support for AP mode as previously only managed and monitor modes were supported. My point being that you have very modern technology and the software is catching up to it...so use the most modern software you can.

Another thing to keep in mind is that in-drivers in Linux come in two parts. There is the part we usually call a driver which is part of the kernel and in your case, the driver is called mt7921e.ko. The usb adapters use mt7921u.ko. The other part of the driver, which is not part of the kernel and does not change out as you upgrade kernels is called the firmware. Upgrading firmware either has to be done by the Ubuntu guy who handles firmware updates that flow into your system when you update or you can do it manually. Firmware updates have been coming often for the mt7921* line of chipsets as features are added and bugs are fixed so it can be a good idea to check and update the firmware on whatever you install...22.04 later.

On my github site, [https://github.com/morrownr/USB-WiFi](https://github.com/morrownr/USB-WiFi) , you will see a menu. For information on how to update your firmware, select menu item 8 and go to the section about the mt7921. All 3 types of chipsets, PCIe, SDIO and USB, use the same firmware so don't worry about this primarily being a USB support site You will need to update 3 files. Two for wifi and one for bluetooth. Keep in mind that the file names do not change. You have to look at file size to determine if a new file is available.

The performance of the mt7921 chipset family is very good and the in-kernel Linux driver is good and improving. Below is a link to a speed test I did recently:

[https://github.com/morrownr/USB-WiFi/issues/139](https://github.com/morrownr/USB-WiFi/issues/139)

Latest, I'm not sure if I fully explained why you need 22.04 or later. It has to do with all of the supporting apps and utilities that are vital for wifi. Getting WPA3, WiFi 6 and WiFi 6e working requires very recent versions of Network Manager, wpa_supplicant, iwd and more.

Good luck,

Nick
