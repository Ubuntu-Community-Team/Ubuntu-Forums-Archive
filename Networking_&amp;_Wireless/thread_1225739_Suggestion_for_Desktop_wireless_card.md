### Post by second.exodous on 2009-07-28
I was wondering if anyone has a suggestion for a wireless card for a desktop, I'm building a computer right now and don't want to run ethernet to my room.

My old notebook had a wireless card that was detected and functioned on the Ubuntu install but I sold that notebook and don't even know what card it came with.  Right now I'm using ndiswrapper on my notebook but I've never used it on a desktop

Should I get a dedicated PCI card or the PCI to notebook wireless card adapter?  Should I go N or just stick with G, it looks like N is only 20 bucks more with the dedicated PCI cards.  I have a G router but it's always nice to have the latest.

I was going to get another Linksys WRT54GL and putting it in bridge mode but going card is definitely cheaper.


**UPDATE**:  I went with the Edimax EW-7728In because it had open source drivers and an easy .deb package to install them with, get the drivers here:  [http://www.array.org/ubuntu/dists/intrepid/eeepc/binary-i386/rt2860-dkms_1.7.1.1_all.deb](http://www.array.org/ubuntu/dists/intrepid/eeepc/binary-i386/rt2860-dkms_1.7.1.1_all.deb)

That is the link to the actual file, if it's no good later then just search for 'ubuntu ew-7728in deb package' and it will be one of the first results.

Anyway, it works great, the driver is open source so it will make it into the Linux kernel eventually, so it will just be plug and play in the future.
