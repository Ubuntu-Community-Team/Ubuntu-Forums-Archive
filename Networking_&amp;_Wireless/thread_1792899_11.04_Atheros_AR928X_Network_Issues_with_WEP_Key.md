### Post by Lord.Necros on 2011-06-28
Howdy howdy,

I've been having issues with my wireless connection ever since upgrading to 11.04. Intermittently, sometimes after a few hours, sometimes within a few minutes, after starting up my laptop the connection will drop and attempt to reconnect. After a minute or so it will prompt me for my wireless key, try to reconnect again, fail, and then mark itself disconnected. For months the only solution to the problem was to restart. 

Recently however I noticed that when this occurred, when I ran iwconfig the key displayed did not match my normal key. It was much longer and the characters were different. Nonetheless, it was the *same* wrong key every time the connection dropped. 
Normally I use a 64 bit key, but after some thought realized what was displayed was a 128 bit key. After some testing I also realized that the 'wrong' 128 bit key was being generated using my 64 bit key as a passphrase. For some reason, intermittently, Ubuntu is treating my 64 bit key as a 128 bit passphrase, generating a new key, and applying it. If I issue a command through iwconfig to revert my key back to the correct, 64bit key, the connection almost immediately re-establishes itself.
