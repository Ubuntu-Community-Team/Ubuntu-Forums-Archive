### Post by Morbius1 on 2011-12-31
[1] The smbfs vs cifs-utils thing is very confussing.

* smbfs - the filesystem - has indeed been deprecated.
* smbfs - the package - is a dummy package that only has one dependancy: cifs-utils. It's only job is to install cifs-utils. 
* cifs-utils - the package - is the one you want and can be installed directly or you can install the smbfs package.

The smbfs package is still in the repos in case someone did not get the memo on all this.

[2] This may be a silly question but it appears you are trying to connect to //10.10.1.1/homes.

Maybe it's just a coincidence but are you talking about the standard Samba [homes] share? If you are you don't connect to it that way. [homes] is a special share that creates a share "on the fly" of the users home directory when asked for by the client. If the client is another Linux machine the share must be asked for by user name explicitly in the mount command:
> //[COLOR=Blue]10.10.1.1/morbius[/COLOR]  /media/samba/homes  cifs  credentials=/etc/.smbcreds,_netdev  0  0That will create a share of morbius' home directory on the server.

You might want to post the output of the following command on the server if you need clarification:
```
testparm -s
```
