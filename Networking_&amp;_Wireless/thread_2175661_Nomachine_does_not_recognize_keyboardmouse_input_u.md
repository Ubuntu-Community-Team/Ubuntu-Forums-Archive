### Post by keraneuology on 2013-09-20
I am running 13.04 on a remote machine that does not have a keyboard/mouse physically connected.

The local machine is running Windows 7 Professional with the latest nomachine (beta) version installed.

If the remote (ubuntu) machine has a keyboard/mouse physically connected to the box then everything works as it should.

If the remote (ubuntu) machine does not have a k/m physically connected then I can establish the session and view the desktop but can not interact with the machine in any way as all keyboard/mouse input is ignored.

When the k/m are disconnected I am still able to SSH into the box and the system will respond normally, the only issue is that unless I have a k/m physically attached to the machine the box will not recognize either one.
