### Post by ramjet_1953 on 2010-11-06
There is a problem with Hardware Encryption with the new Intel Driver, when using WEP.

I have found that the following steps work for me:

1. If you don't already have this file [COLOR="Blue"]/etc/modprobe.d/ipw2200[/COLOR]create it.

```
sudo gedit /etc/modprobe.d/ipw2200
```

Enter this line into the file:

```
options ipw2200 hwcrypto=0 qos_enable=0 qos_burst_enable=0
```

2. Save the file

3. Run this in a terminal:

```
modprobe -r ipw2200
```

4. Run this in a terminal:

```
modprobe ipw2200 hwcrypto=0 qos_enable=0 qos_burst_enable=0
```

5. Re-boot

Hopefully, this will cure your WEP problem.

Roger :)

### Post by BcP on 2011-02-28
I had the same problem where the ipw2200 would connect fine with no encryption, but when WEP or WPA turned on it kept prompting for the passphrase even though it was entered correctly in network manager. The problem turned out to be the driver files installed with 10.4 and 10.10. They are the 3.1 version which has a problem with the crypto. To resolve it I downloaded the 3.0 version files from source forge. [http://ipw2200.sourceforge.net/firmware.php](http://ipw2200.sourceforge.net/firmware.php)

I unzipped the files and copied them to the:
/lib/firmware/ directory, overwriting the v3.1 files. I also copied them to the /lib/firmware/2.6.35-25-generic/ folder as well, although I don't think that was necessary. Reboot and it connected with no problem. Very simple, but I am new to Linux so it took me a week to figure this out. I came across some cryptic references, but it seems no one likes to put their solutions in simple terms. But this simple fix worked for me, hopefully it will do the same for others.
