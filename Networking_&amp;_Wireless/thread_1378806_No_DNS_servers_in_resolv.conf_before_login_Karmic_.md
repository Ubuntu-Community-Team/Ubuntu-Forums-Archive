### Post by x33a on 2010-01-11
I don't have much experience with dhcp, but one thing you can do is to put your dns entries in resolv.conf manually, and then do
```
sudo chattr +i /etc/resolv.conf
```

this way, the file won't be overwritten by any application.

### Post by Cetra on 2010-01-21
Seems that resolvconf was destroying this file.

resolvconf was installed with vpnc as I needed a temporary cisco vpn client.  But this has since been removed.

Running "sudo apt-get purge" and then "sudo apt-get autoremove" this has sorted my issues.
