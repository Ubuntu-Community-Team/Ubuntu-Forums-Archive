### Post by idallen on 2010-02-03
I went from Ubuntu 8.10/32 to 9.10/64 and my tunnel scripts stopped working.  Apparently we don't load "tun" any more - we load the *type* of tunnel we want, which in my case is "ipip".  The changed modprobe line below fixed things for me (IP addresses not to scale):

# modprobe ipip
# ip tunnel add tun0 mode ipip remote 1.2.3.4
# ifconfig tun0 5.6.7.8 pointopoint 9.0.1.2

If I run "netstat -ia" I also see a useless "tunl0" device, but I can't remove it:

# ip tunnel del tunl0
ioctl: Operation not permitted
