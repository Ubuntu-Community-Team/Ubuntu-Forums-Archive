### Post by dkinfl on 2009-03-30
I use my Ubuntu laptop both in my corporate environment and in my home wireless network (and in other wireless networks).  When I am away from my corporate network the performance is really slow.  I have figured out why, but cant figure out how to fix it.  It has to do with /etc/resolv.conf.

When I am on the corporate lan my resolv.conf looks like this:

nameserver 1.1.1.1
nameserver 1.1.1.2
search xxx.local

When I take my laptop to my wireless home LAN my resolv.conf looks like this:

nameserver 1.1.1.1
nameserver 1.1.1.2
nameserver 192.168.1.1
search xxx.local

As you can see it successfully adds my wireless router for the DNS, but it puts it below the name servers for my corporate lan.  So, when I go to browse a web site, I have to wait for the name lookup to fail on 1.1.1.1 and 1.1.1.2 before it looks at 192.168.1.1 and successfully sends me to the web site.  

If I am doing serious work from home I can edit the resolv.conf and restart networking and it is fine.  However, is there a way to have Ubuntu wipe this file each time it DHCPs so only network relevant DNS servers are listed?  I also DHCP this information at my corporate location through a DHCP reservation.  Any help is appreciated.

### Post by dkinfl on 2009-03-30
I got it fixed after a little research and reading the post above.  I opened Synaptec package manager and uninstalled a package called resolvconf, which says it is a name server information handler.  Basically if you remove this package from your system, it allows network manager to directly access and write the resolv.conf file, without going through the resolvconf application first.  Now when I move between networks, the only DNS entries are the one relevant to the current network and my new laptop is as fast as it should be.  If you do this correctly, after a reload your resolv.conf will show it being edited by network manager, like below:

dking@dking-ub:/etc$ cat resolv.conf 
# Generated by NetworkManager
nameserver 192.168.1.1
dking@dking-ub:/etc$
