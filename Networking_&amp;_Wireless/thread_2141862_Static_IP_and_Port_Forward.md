### Post by steeldriver on 2013-05-03
Hello and welcome

In systems that use the resolvconf service, you should add a *dns-nameservers* entry to the /etc/network/interfaces file instead e.g.

```

auto eth0
iface eth0 inet static
address 192.168.1.5
netmask 255.255.255.0
gateway 192.168.1.1
**dns-nameservers 192.168.1.1 8.8.4.4**

```

### Post by SeijiSensei on 2013-05-04
The static address you need has to come from your ISP.  It enables there to be a fixed mapping from [www.example.com](www.example.com) to a specific public IP address.  If your ISP doesn't offer fixed addresses, or wants you to upgrade to a level of service beyond your means, you can look into other options like [noip.org]("http://www.noip.com/").  Another strategy is to rent a virtual server from a provider like [Linode](http://www.linode.com/) that offers static IPs by default.
