### Post by bhold on 2008-09-12
Hello,

I am running Ubuntu 8.04 desktop, current on all updates. I remote print to a Windows XP PRO system, the printer on the XP system is HP4215 OfficeJet All in One on a USB port.

Remote printing has worked fine for months. Suddenly a few days ago I started getting the following symptoms when printing from Ubuntu to XP.
1. The printer starts to respond as usual - things start to move, click, and make the usual printer prep noises, the green on light starts to blink.
2. A pause, several more noises from the printer, and then - nothing.
3. The XP print queue shows an active document. But attempting to cancel the document does nothing.

The printer works fine locally when printing from XP, just no longer remotely from Ubuntu.

Thanks for any thoughts on this.

### Post by bhold on 2008-09-12
On the XP system, printer properties, ports - I unchecked the "Enable bidirectional support" box. This fixed the problem.
