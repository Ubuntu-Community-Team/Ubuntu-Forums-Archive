### Post by baffled68 on 2010-08-15
Hi,

I'm running Ubuntu 10.04 from a CD, on a desktop I dual boot with Windows. I'm able to obtain a connection if I use the Network Manager to set up WEP encryption. I can't connect when using WPA/WPA2 - either via the Network Manager or the interfaces file. I can make a WPA/WPA2 connection when running Windows.

With WPA/WPA2 I took the following approach:
1 Run a blacklist command to disable legacy/conflicting drivers
```

echo -e "blacklist bcm43xx\nblacklist b43\nblacklist b43legacy\nblacklist ssb" | sudo tee -a /etc/modprobe.d/blacklist.conf blacklist bcm43xx, blacklist b43, blacklist b43legacy and blacklist ssb

```2 Using Synaptic Package Manager install the ndis wrapper

3 Install the wireless adapter driver using ndiskgtk GUI

4 Generate my psk hex key:
```

 wpa_passphrase <my ssid name> <my_ascii_key>  

```5 Open my interfaces file, and add the generated psk hex key etc. as per the examples below
```

sudo gedit /etc/network/interfaces

```WPA2 AES attempt
```

auto lo
iface lo inet loopback

auto wlan0
iface wlan0 inet dhcp
wpa-driver wext
wpa-ssid myssid
wpa-ap-scan 1
wpa-proto RSN
wpa-pairwise CCMP
wpa-group CCMP
wpa-key-mgmt WPA-PSK
wpa-psk <psk hex key>

```WPA TKIP attempt
```

auto lo
iface lo inet loopback

auto wlan0
iface wlan0 inet dhcp
wpa-driver wext
wpa-ssid myssid
wpa-ap-scan 1
wpa-proto WPA
wpa-pairwise TKIP
wpa-group TKIP
wpa-key-mgmt WPA-PSK
wpa-psk <psk hex key>

```6 Killed the nmapplet via the system monitor

7 then restarted the network:
```

sudo /etc/init.d/networking restart

```which results in no connection and a stream of text ending in:
No DHCPOFFERS received.
No working leases in persistent database - sleeping.

When I attempt WPA2 using a connection set up with a GUI it attempts repeatedly to login and pops a password dialog every 2-3 minutes.

I've scoured the forums for days without finding anything that, as yet,  solves this issue. I'd appreciate any advice on likely causes, diagnostic commands I should try etc.
