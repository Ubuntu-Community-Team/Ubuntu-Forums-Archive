### Post by Agrajag27 on 2008-12-30
I own a Dell XPS 1530 laptop that acts as my tutorial machine for Ubuntu. It dual boots Vista and Vista does little more on the box than run for 2 minutes before locking up so I have learned Ubuntu faster than I expected and do enjoy it.

We went to Scranton this past week to be close to some local ski mountains that dot its surroundings. We found ourselves at a Ramada that had wireless WiFi.

My son popped out his new iPod Touch Christmas present and was on their network instantly. No need to set anything up.

I booted my laptop and couldn't get on. I'd be told I was connected but I couldn't access anything (including getting an Acceptance page to be able to browse on the Ramada system).

I went over to Vista (God I hate it) and before it locked up it got onto the network immediately after I pointed it to it.

I then booted back to Ubuntu and called support. Amazingly they had someone there who used Ubuntu. He informed me that I had to set a static IP to get it to work. I set the IP and it worked flawlessly from there on out.

This happened just after I had a similar issue at my in-laws. At home I have a WiFi setup and Ubuntu has NO problems connecting to it at all. Got to the in-laws for the holidays and couldn't get on. I'd get connected only to be told the connection was interrupted. There's no security. It's an open router so it wasn't a problem with security. No matter what I did I couldn't connect. If I set up security Ubuntu would see that and prompt me for a password before then giving me the same dropped connection message.

This little bit of complexity concerns me. Why is Ubuntu so finicky? It gets on my WiFi setup at home with no problems but can't access other networks that other OS' have no problem with?
