### Post by dbalascak on 2009-08-23
A router handles a network protocol, TCP/IP, IPX, Appletalk etc.  TCP/IP is the protocol that is used by Linux ( UBUNTU ) and Windows.  The router does not care about the OS. 
Is this following the same line of questioning as your previous post?  The information you supplied showed that no interface was set up to use TCP/IP and can't talk to the router. If your system is basically using defaults for the network configuration, and you are sure you are connected to the network, then the router may not be supplying an IP address via DHCP.  

Which interface is connected to the network?
If you go into System-> Administration->  Interfaces  Do you see the interface?
