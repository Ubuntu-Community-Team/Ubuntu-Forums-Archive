### Post by Nickopotamus on 2008-10-09
**Executive summary:** Connecting to a wireless network with a D-Link DWL-G650+ PCMCIA card kills the keyboard under Xubuntu 8.04.

I'm trying out Xubuntu 8.04 as a way to make use of my old (well, not that old, just big and clunky) Travelmate 2701 usable. Install went perfectly and the machine is running lovely and quickly. Though there is one major problem - with no easily accessible wired connection in this house, working wifi is a must, and this is where the issue lies.

The laptop doesn't have onboard wireless, so I am using a D-Link DWL-G650+ PCMCIA card that worked fine with this laptop under Windows. The card is recognised by the network manager (much to my surprise) and offers me the chance to connect to my wireless network. Enter the WEP key, and the network manager icon changes to show it's connecting, before disappearing completely.

With that, the keyboard also disappears (well, the ability to use it does, the thing is still physically there!). Either it is really, *really* slow (as in, waiting minutes for a keypress to appear) or it seems to be constantly providing a signal - it's lucky dip each time you try and connect to the wireless network. If you get the slowdown, and enter a URL into eg firefox, I get an error that there is no valid network connection.

I've tried using the other two wireless adaptors I have in the house, but they're both USB and neither is recognised by Xubuntu. Google reveals lots of other issues with this card, plus a few known bugs for older Ubuntu releases resulting in this problem, but no fixes, so I'm wondering if you guys can help!
