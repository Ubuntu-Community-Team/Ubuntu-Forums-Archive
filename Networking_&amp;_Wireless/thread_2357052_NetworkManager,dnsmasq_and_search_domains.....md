### Post by papibe on 2017-03-31
Thanks.

It seems the authoritative name server for the domain are in the 192.168.0 segment, but your DHCP is giving you DNS on 10.10.10 (plus google, why?)

Let's try this:
```
nslookup something.host3 10.10.10.1

nslookup something.host3 192.168.0.3

host -v something.host3 10.10.10.1

host -v something.host3 192.168.0.3

nslookup something.host3 8.8.8.8
```
(the latter is why google shouldn't be there).

Regards.
