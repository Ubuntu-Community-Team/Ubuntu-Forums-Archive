### Post by kartwall on 2010-04-07
Thank you very much, chill. It works now. I followed your suggestion, enabled the DHCP service on router, then try to connect again. At first time I choose "WEP 128bit passphrase", connect failed and NetworkManager inform me to reconfig the security option. Then I choose "WEP 40/128 bit key", connect succeeded.
So, back to the original of this issue, seems I picked a wrong security option, then because I config the IP/netmask/router/DNS manually, so "connect succeeded"(NetworkManager confirmed connection established). 
After I change the static IP configuration to DHCP, finally I found out the reason of this issue, just as I described above.

Thank you again, chill.
