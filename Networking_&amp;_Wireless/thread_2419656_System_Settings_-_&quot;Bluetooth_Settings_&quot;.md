### Post by anneranch on 2019-05-24
There are two slide switches on   “Bluetooth “ (settings ) form - see attached screenshot.  

 1. **What is the purpose of „Bluetooth“ switch on the left ?**

     Please note that attached „Bluetooth“ text will only be bold when bluetooth devices are detected by OS.  
     When there are no bluetooth devices detected the entire form is grayed  ( inactive) and „  No bluetooth adapters found „  is  ( grayed) entered where “devices” should be .  
       Little inconsistent terminology ( devices / adapters ) and redundant to a degree.
 
 
 2. **What is the purpose of the “visibility” switch on the right ?**

      When no bluetooth adapters are detected it reads “Visibility” ( grayed ).
      When devices are detected , but NOT displayed , then it reads “Visibility of jim-desktop” and the associated switch is also active .  
 
 
 3. Little out of scope of this question / post – the “jim_desktop “ gets displayed as “name” of “visible” bluetooth device which responded to HCI  inquiry.  BUT if it is not set to “visible” my C++ code inquiry will not get “response” for hci_inquiry function. .

 
 
 It would be nice if I could set it to be “visible “ ALL THE TIME.
