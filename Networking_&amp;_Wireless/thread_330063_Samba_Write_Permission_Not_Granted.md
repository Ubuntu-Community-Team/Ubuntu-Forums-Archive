### Post by dannyboy79 on 2007-02-27
please post the output of this command ls -la /home/ricardo/ | grep Backup

you are aware that upper and lower case names in linux really matters correct? we need to make sure that this folder is writable by all. this is also not a good idea. you probably want to have the backup folder somewhere NOT in your home directory as this is normally personal stuff. so either put it within /media/ or make a folder on the root level. or maybe you could create a new user called backup, then you would have a folder called /home/backup and you could have all your users authenticate using the username backup with a certain password. i noticed that you have guest access enabled in your smb.conf, and you have security = user. so when some1 can't authenticate themselve against the password backend which is tbdsam, they will get guest access, which is always read only!! this is probably why. you either need to add that username to your system and then to your smbpasswd file, have them change their username and password to 1 that is already on the system as well as in the smbpasswd file, or you can change your security to share. which then they won't need to be authenticated prior to having access to your box or shares. check out man smb.conf for more info. good luck
