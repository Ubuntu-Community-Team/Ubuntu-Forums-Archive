### Post by Unconscious on 2007-08-04
Unfortunately, for linux users, this is actually a security feature of VPN, in general. It protects the destination network.

If VPN allowed you to continue normal network activity, someone could set up an open tunnel into the destination network. Thus, as a condition of being allowed to tunnel into the destination network, all external network access is prevented.

When you are connected via VPN, it is exactly as if your machine were physically connected to the destination network.

I know this is not the answer you want, but I hope it helps.
