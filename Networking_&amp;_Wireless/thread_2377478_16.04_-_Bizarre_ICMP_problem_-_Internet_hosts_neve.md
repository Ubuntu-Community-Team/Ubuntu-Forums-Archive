### Post by ajayh on 2017-11-13
My new Ubuntu 16.04 installation has full internet access except that I never receive ECHO-REPLYs back from my ECHO-REQUESTs to the internet.  

Ubuntu 16:  
uname -a:  Linux UbuntuHost 4.4.0-98-generic #121-Ubuntu SMP Tue Oct 10 14:24:03 UTC 2017 x86_64 x86_64 x86_64 GNU/Linux
ens36:  192.168.10.75/24
Gateway:  192.168.10.1
ufw Status: inactive

All other hosts on my 192.168.10.0/24 are able to ping sites like google.com, and a tcpdump on my router (openBSD 6.2) shows their ECHO REQUEST leaving and ECHO REPLY coming back in as expected.   

But with Ubuntu 16, I see ECHO REQUEST packets leaving but I never see an ECHO-REPLY come back in, I tried pinging a number of sites.  my PFlog shows the REQUEST being passed out as expected but a REPLY simply never gets returned by the remote server. 

What really confuses me is that I CAN ping and get replies from:  

- Default Gateway (192.168.10.1)
- Other hosts on my network (e.g. 192.168.10.5)
- My Public IP (OpenBSD's public IP, something like 100.200.15.191 (not my real public IP)
- My Public gateway (e.g. the default route for my outside interface on my edge firewall, something like 100.200.15.1) 

Is this really bizarre or am I overlooking something really simple?
