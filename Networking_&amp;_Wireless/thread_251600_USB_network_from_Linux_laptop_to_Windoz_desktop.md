### Post by bodhidharma on 2006-09-05
Here's a situation other people may also be in (and have
found a solution to):

I have a beautiful Dapper installation on my laptop, I use it all
the time from home and on the road, it is just great.

But when I get to my office, things are a little less convenient:
there is a desktop machine provided by my employer (a university,
so they are a little more casual about things than corporations
may be) with a nice screen and keyboard... but running Windows XP.

I can disconnect the ethernet cable from the desktop and plug it
into my laptop (or even use the campus student wireless access)
but (a) it is a pain to crawl back behind the desk to do this; and
(b) my laptop does not have a recognized MAC address, so the
campus network will only give me the shallow, visitor access -- the
same as the wireless connection (albeit faster and less subject to
wireless sniffing, I guess).

What I wonder is if there is a simple piece of software I could run
on the Windows box which would then let me plug a USB-to-USB cable
between the laptop and desktop and get real network access.

There are two things this would be nice for: (a) I would just plug and
it would be on the net, faster and no digging among the dustbunnies
for the cable; (b) presumably I would have deeper network acces (since
my desktop Windows box does).

Of course, you may ask what I use the real network acces *for*.  This
is a good question -- there is some weird Windows software on campus
which does student grading and so on, I would be able to access that
(but not run it!); I would be able to access the printers in my
building... I guess (I've never set up Windows printing from a Linux
box, I assume it is easy enough -- with Samba? -- and I would merely
need the right network access).  There is also some Windows "network
drive" that I would presumably access in a similar way to Windows
printers.

Of course, another option would be to remove the Windows on my desktop
machine and put on Linux; then I could access printer and network drive
as above, but without the Windows software, and without the problem of
getting my Linux laptop to talk to a Windows desktop via USB (presumably
this is easier Linux-to-Linux via USB).  I am unlikely to get permission
to do this, easily, but it might happen.

(Is that true?  Is it easy to set up an ad-hoc network by attaching a
USB-to-USB cable between two Linux boxes?  Is it possible?)

Anyway, any suggestions from others in similar situations would be
greatly appreciated.

(I will cross-post this in Laptops and Networking....)
