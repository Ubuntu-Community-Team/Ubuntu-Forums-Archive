### Post by The Cog on 2009-04-25
Glad you got it sorted.

My guess is that your ISP has several DNS servers, and dishes out addresses at random as a way of load sharing. If that's the case, which DNS server you use shouldn't really matter.

I don't know how you are finding out the DNS addresses at all. Are you looking at the router config? I will assume you are.

If you use DHCP to get an address from your router, it should also give you a DNS server address to use, which moght be the actual IP addresses given by the ISP or the router might just give its own address as the DNS server. That should all just happen automagically.

If you use static IP configuration, then I think network-manager is not the best thing to use. I have seen a number of posts complaining of network-manager and static configuration problems. And to be honest I can't rememver what they are because replacing network-manager with wicd is the very first thing i ever do after an install. wicd does static configuration very nicely thank you.

If you want to do static connfiguration, you might want to edit the proper files rather than relying on network-manager (see below). Just be aware that I have seen reports of network-manager aciively deconfiguring statically confiugred and working machines after a few minutes so you may have to kill network-manager. I can't say if this still happens because network-manager doesn't stay long enough for me to find out.

/etc/init.d/interfaces should look a little like this:
> # This file describes the network interfaces available on your system
# and how to activate them. For more information, see interfaces(5).

# The loopback network interface
auto lo
iface lo inet loopback

# The primary network interface
auto eth0
iface eth0 inet static
address 192.168.1.101
netmask 255.255.255.0
gateway 192.168.1.1

and /etc/resolv.conf somehting like this - use multiple lines if you have multiple nameservers:
> nameserver 192.168.1.1
