### Post by ibutho on 2008-07-06
I wouldn't have expected that problem from a distro using a recent kernel. The problem is caused by the system thinking that the device is a usb mass storage device, so plugin the modem and then run "sudo modprobe -r usb-storage" and "sudo modprobe -r usbserial". After that run "lsusb" (install usbutils if you can't find this command) and make note of the vendor id and product id of your modem and then do "sudo modprobe vendor=0×12d1 product=0×1003" (Change vendor id and product id to match yours). After that unplug the modem, wait a little while and plug it in again. hopefully your modem will be recognised when you run dmesg.
