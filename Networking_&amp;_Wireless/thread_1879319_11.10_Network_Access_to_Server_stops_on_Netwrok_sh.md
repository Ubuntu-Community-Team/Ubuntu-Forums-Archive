### Post by Kalibur on 2011-11-11
Hello guys am having a bit on an interesting problem in my network environment.  I have an Ubuntu 11.10 64bit its a mythtv server running Unity ui IP 10.42.43.1@eth0  My other network hosts are DHCP hosts a PS3, a Dell latitude-xt, a couple of Android phones and another laptop.

I use mobile broadband dongle for internet sharing that is connected to the ubuntu 11.10 server ppp0.  eth0 is shared to the network hence is default gateway@10.42.43.1.

Now the issue is when ppp0 (mobile broadband) is not connected I have full access to 10.42.43.1 (ssh, mythtv, myth remote, http).  But once I activate the mobile broadband I lose all access to the 10.42.43.1 services and but internet connectivity stays.  So to put it in a nutshell its either I have internet or my local network resources never both.

Any help explaining this and maybe fixing it please.

### Post by Kalibur on 2011-11-12
Hi Folks 

Please consider this thread solved the issue that was preventing access to my Lan while the internet connection is active is my firewall.  I had set firestart to start on internet connection (ppp0 mobile broadband).  To resolve it I openned firestarter and on the policy tab I created expections for in bound out bound traffic for the traffic I need such as media sharing, ssh, mythtv etc.
