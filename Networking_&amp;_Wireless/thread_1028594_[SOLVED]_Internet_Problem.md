### Post by mpokrywka on 2009-01-02
I would advise you to return to working configuration, and when you are sure that networking works, run:
**ifconfig**
**ip route**
**cat /etc/resolv.conf**
to save correct settings.

You say that dhcp client seems to be the problem - dhcp can only affect IP address, default route, DNS settings.
Maybe newer version misconfigures some parameters. Run above commands when networking doesn't work and compare.

### Post by BeRReGoN on 2009-01-03
Yes thanks i figured it out when i check the two resolv.conf

The problem was the nameserver and search/domain

On a working archlinux setup before the dhcpcd update i got this:
```
# Generated by dhcpcd for interface eth0
search cablevision.qc.ca
nameserver 24.212.0.8
nameserver 142.217.192.8
```

On the liveCd of Xubuntu:
```
# Generated by NetworkManager 
domain tlb.sympatico.ca
search tlb.sympatico.ca
nameserver 142.217.192.9
nameserver 24.212.0.7
```
I had the same problem with wicd or manual dhclient calling but when i change the resolv.conf to the workking one everything was working fine.
