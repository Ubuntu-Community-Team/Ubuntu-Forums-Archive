### Post by r8a on 2009-04-29
ok, this is how mine worked, hp model dv5 1023es...

installed madwifi drivers as usual (madwifi-hal-0.10.5.6-r3942-20090205)

---- to install wifi on dv5-1023es when kernel updated ----
r8a@r8a-laptop:~$ cd madwifi-hal-0.10.5.6-r3942-20090205
r8a@r8a-laptop:~$ make
r8a@r8a-laptop:~$ sudo make install
r8a@r8a-laptop:~$ sudo modprobe ath_pci
r8a@r8a-laptop:~$ sudo gedit /etc/modules

this is how modules should look:
----------------------------------------------------------
# /etc/modules: kernel modules to load at boot time.
#
# This file contains the names of kernel modules that should be loaded
# at boot time, one per line. Lines beginning with "#" are ignored.

fuse
lp
rtc
sbp2
ath_pci

-----------------------------------------------------------

as each time the kernel is updated, you have to go to the madwifi install directory and run the make and make install commands in order to get madwifi applied to the new kernel.

This works perfectly for users on dv5 on 8.10... hope it works on 9.04...
