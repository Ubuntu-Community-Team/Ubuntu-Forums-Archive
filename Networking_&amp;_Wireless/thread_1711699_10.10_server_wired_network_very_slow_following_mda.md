First of all I've looked through the forum and searched quite a bit for an answer but I could have missed something so apologies if this is already covered somewhere (if it is a link would be great).

I've recently upgraded a PC I used as a file and media server and installed ubuntu server 10.10 64bit.  After setting the server up with samba, PS3 media server and a software raid (4 x 1TB, raid 5 using mdadm) everything seemed to be fine, with good file transfer speeds over a gigabit network and the media files (including HD) streaming to a PS3 with no problems.  A day or so later the network transfer speeds are far worse (~1MB/s to the server from my Windows 7 box and ~100KB/s from the server), that's using either samba and Windows explorer or an FTP.  Also if the media streaming to the PS3 works the video keeps stopping (as if the bandwidth isn't enough, but I may be jumping to conclusions).

I'm not sure if this is relevant but in the interest of completeness - just before the network problem the shared raid array stopped working, after some investigation I found that 2 of the 4 disks were reporting faulty but I managed to get the array up and running again (looks like a couple of loose power cables, serves me right for not being more careful when putting it together...).  The rebuild seems to have finished successfully so as I said I'm not sure if it's relevant, especially since the transfer speed to another (non-raid) hard disk is just as bad but the internal transfer speed between the raid array and the other HD seems fine.

The PCs and PS3 are all plugged into the same gigabit switch and transfers between two Windows boxes on the same network seem fine.  Also as I said the ubuntu box seemed to be fine until a couple of days ago.

The ethernet card is the on board one, I'm using this motherboard (ASRock Penryn1600SLI-110dB):
