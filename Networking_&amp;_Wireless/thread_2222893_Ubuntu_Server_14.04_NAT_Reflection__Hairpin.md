### Post by Doug S on 2014-05-09
Hi and welcome to Ubuntu forums.

I found it somewhat difficult to do what you are wanting. There might be a better and easier way, but I could only get it working by using "mark".
On my system I have an internal test server at 192.168.111.112 and sometimes I want its web pages available on internet while at the same time leaving my production server web pages available, so I use port 8083 and forward that to port 80 of the internal server. For your requirements I did this (I assume you will understand my mappings):```
echo "FWD: Allow all connections OUT and only existing/related IN..."
$IPTABLES -A FORWARD -i $EXTIF -o $INTIF -m state --state ESTABLISHED,RELATED -j ACCEPT
[COLOR=#ff0000]$IPTABLES -A FORWARD -i $INTIF -m state --state ESTABLISHED,RELATED -j ACCEPT[/COLOR]
#
# port forward stuff. see also the prerouting area.
$IPTABLES -A FORWARD -i $EXTIF -o $INTIF -p tcp --dport 80 -d 192.168.111.112 -m state --state NEW -j LOG --log-prefix "PFNEW80:" --log-level info
$IPTABLES -A FORWARD -i $EXTIF -o $INTIF -p tcp --dport 80 -d 192.168.111.112 -m state --state NEW -j ACCEPT
$IPTABLES -A FORWARD -i $INTIF -o $INTIF -p tcp --dport 80 -d 192.168.111.112 -m state --state NEW -j LOG --log-prefix "PFINT80:" --log-level info
[COLOR=#ff0000]$IPTABLES -A FORWARD -i $INTIF -o $INTIF -p tcp --dport 80 -d 192.168.111.112 -m state --state NEW -j MARK --set-xmark 0x4[/COLOR]
[COLOR=#ff0000]$IPTABLES -A FORWARD -i $INTIF -o $INTIF -p tcp --dport 80 -d 192.168.111.112 -m state --state NEW -j ACCEPT[/COLOR]
$IPTABLES -A FORWARD -i $INTIF -o $EXTIF -j ACCEPT
...
$IPTABLES -t nat -A PREROUTING -p tcp -i $EXTIF --dport 8083 -j DNAT --to 192.168.111.112:80
[COLOR=#ff0000]$IPTABLES -t nat -A PREROUTING -p tcp -i $INTIF --dport 8083 -d $EXTIP -j DNAT --to 192.168.111.112:80[/COLOR]
...
$IPTABLES -t nat -A POSTROUTING -o $EXTIF -j SNAT --to $EXTIP
[COLOR=#ff0000]$IPTABLES -t nat -A POSTROUTING -m mark --mark 0x4 -j SNAT --to $EXTIP[/COLOR]

```
