Hi all,

I'm hoping someone can put me out of my misery.
I have a modem router on 192.168.0.1
I have an ubuntu 12.04 server performing dns dhcp and openvpn server on 192.168.0.5
The open vpn server is bridged (br0) and configured to get the ip from the lan's dhcp.
The server 192.168.0.5 has only one physical nic, eth0 bridged with tap0.
That all works perfectly.
I'm trying to work out what iptable commands I need to issue on server 192.168.0.5 to cause an incoming connection to server 192.168.0.5:1234 to be port forwarded to a webcam server on
 192.168.0.22:9022.
I'm trying to do this because I want to be able to switch the port forward on and off using the server.....


It seems the normal method is :

iptables -t nat -A PREROUTING -i eth0 -p tcp -m tcp --dport 1234 -j DNAT --to-destination 192.168.0.22:9022 
iptables -t nat -A POSTROUTING -j MASQUERADE

but this doesn't work on the server 192.168.0.5 (I've tried replacing eth0 with br0).
I've tried it on another machine 192.168.0.13 which isn't configured to act as openvpn server, it just has single nic eth0 set to use dhcp, and the two iptable commands sucessfully allow me to open the camera's web page in a browser using 192.168.0.13:1234 (originally 192.168.0.13 did have openvpn server bridge installed and the commends didn't work, I removed the bridged arrangement and they worked!)

Clearly the presence of the bridged interface in the server is causing an issue but this is my first attempt to use iptables and despite days of googling and learning a lot, I'm lost as to how to proceed.
