### Post by techiebiker on 2009-05-02
made progress, can connect using WPA-PSK but not on boot.

must run sudo wpa_suppicant -c/etc/wpa_supplicant.conf -iwlan0 -B in terminal after boot to connect.

On Netgear site discovered that wg311v2 didn't ship with WPA capability. Netgear released drivers later that provided WPA. Downloaded wg311v2 software v 2.0.0.7 from Netgear.
[http://kb.netgear.com/app/products/model/a_id/2565](http://kb.netgear.com/app/products/model/a_id/2565)

Used /usr/share/doc/wpasupplicant/examples/wpa-psk-tkip.conf to create my /etc/wpa_supplicant.conf file

Followed instructions from these three URLs to figure what is going on and use ndiswrapper to load wg311 drivers.
[http://ubuntuforums.org/showthread.php?p=6183681](http://ubuntuforums.org/showthread.php?p=6183681)
[http://www.linuxquestions.org/linux/answers/Networking/NdisWrapper_The_Ultimate_Guide/](http://www.linuxquestions.org/linux/answers/Networking/NdisWrapper_The_Ultimate_Guide/)
[http://manpages.ubuntu.com/manpages/intrepid/man8/wpa_supplicant.8.html](http://manpages.ubuntu.com/manpages/intrepid/man8/wpa_supplicant.8.html)

Now it connects to the WAP using WPA-PSK but only from command line after boot.

How do I make it connect at boot?
