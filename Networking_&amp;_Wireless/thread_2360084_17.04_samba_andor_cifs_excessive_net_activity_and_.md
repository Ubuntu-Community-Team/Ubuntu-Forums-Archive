### Post by HNmYk3h on 2017-05-01
After installing Ubuntu Gnome 17.04 and setting up shares in both directions to a Windows 10 box, I noticed constant network activity between the two computers. It was in the 10 megabits/s range. Using tcpdump I verified that it was samba traffic, always on port 445 on the Windows box but mostly in the 30000 to 60000 range on the Ubuntu box. What seems to have fixed the problem was to add the option vers=3.0 to the fstab line where I was mounting the Windows shares.  I.e.,
```
 //<remote-computer>/<share> /mnt/local-mount-point cifs other-options,vers=3.0  0  0
```
The odd thing was that the excessive activity didn't start until about 16 minutes after booting (16 minutes of uptime). I also noticed that the mount would be lost occasionally and I'd get "host is down" errors.  I've got only about 6 hours of operation with the vers=3.0 option but so far so good.
