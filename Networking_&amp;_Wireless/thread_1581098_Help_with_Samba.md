>  I have also removed all the samba packages and then installed  samba4-clients but still cannot see the ubuntu machine on my network     (1) If you removed all samba packages then you no longer have a samba server so there is nothing to see from the other machines on your network.

> These packages contain snapshot versions of Samba 4, the next-generation version of Samba. These should be considered _experimental_, and should not be used in production. In particular, no guarantees are made with regard to upgrades between versions.

### Post by Morbius1 on 2010-09-26
OK, the following shares should be accessible by everyone:
> [antoineubuntu]
[stern]
[timeout]The following will not be accessible by everyone despite you telling it to allow guest access:
> [documents]
[my music]
[1023]
[timeoutbookings]
[ubuntu 2ndcopy]The reason for that is how the external drive is mounting. It's mounting such that the only person who has access to it is the owner - antoine. Guest is not antoine and Samba cannot override linux permissions.

To fix this add the following line to the [global] section of smb.conf:
```
force user = antoine
```Save smb.conf and restart samba:
```
sudo service smbd restart
```

There are two completely different methods of using samba to share directories - usershare ( nautilus-share ) and Classic-share. You are using Nautilus-share. Nautilus-share comes with restrictions however that Classic-share does not have:
> The problem is I want to change the permissions so that only some are visible and accessible via sharing.You have control over what is accessible but all nautilus-shares are visible - there is no way to change that without making them all invisible.
>  I also want to make some folders accessible via a password protection.You already have one:
> [my music]
path=/media/External/My Music
comment=
usershare_acl=Everyone:F,
[COLOR=Blue]guest_ok=n[/COLOR]There is one other thing about your shares that you should think about. You have shares within shares which really should be avoided because they will cause you nothing but grief when they get out of sync. For example:
> [timeoutbookings]
path=/media/External/2ndcopy/Documents/TimeoutbookingsIs a child of:
> [documents]
