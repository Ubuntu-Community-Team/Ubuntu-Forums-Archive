### Post by Ayuthia on 2009-09-13
With the warning messages that you are receiving, it appears that you are using Jaunty.  The current issue that you are encountering is that the bcm43xx module is deprecated.  The b43 module is its replacement.  It appears that Jaunty should work with the b43 module, but if you are more comfortable with the NDISwrapper module that is fine also.  

You can test out ndiswrapper to see if it works first by doing the following after you have loaded the bcmwl5.inf file into NDISwrapper:
```
sudo modprobe -r b43 ssb ndiswrapper
sudo modprobe ndiswrapper
sudo iwlist scan
```
If wireless sites show up, then it should be working.  You can then go and blacklist the b43 and ssb modules:
```
echo blacklist b43 | sudo tee -a /etc/modprobe.d/blacklist.conf
echo blacklist ssb | sudo tee -a /etc/modprobe.d/blacklist.conf
```
That should be all that you need to do.

Hope that helps.

### Post by Ayuthia on 2009-09-14
This information shows that ndiswrapper is the only module running and it appears to be active because lshw -C network does not show it as DISABLED or UNCLAIMED.

So, if you just run the modprobe commands only, does NetworkManager work again?  How far are you from the router?

The other thing to try and see is if NetworkManager is causing the problem.  You can temporarily stop it by doing the following:
```
sudo /etc/init.d/NetworkManager stop
sudo modprobe -r b43 ssb ndiswrapper
sudo modprobe ndiswrappper
sudo iwlist scan
```
If it does produce wireless sites, then it might be NetworkManager that is causing the problem or else the wireless driver is working intermittently.  If it does not produce any wireless sites, can you post the results of:
```
dmesg|grep ndis
```It might provide some clues on what is happening.

If you want to start NetworkManager up again:
```
sudo /etc/init.d/NetworkManager start
```or you can just reboot.
