### Post by jeremy31 on 2017-10-08
```
sudo rm /etc/modprobe.d/AR9485.conf
```
As it isn't valid
```
sudo sed -i 's/wifi.powersave = 3/wifi.powersave = 2/' /etc/NetworkManager/conf.d/default-wifi-powersave-on.conf
```
Reboot
If you still have issues after sleep mode try ```
systemctl restart network-manager.service
```

### Post by jeremy31 on 2017-10-08
inxi gives a little more info about the device.  A google search on ID 168c:abcd indicates an issue with BIOS

ID abcd is used by Atheros devices when not properly initialized by BIOS.

[https://ubuntuforums.org/showthread.php?t=2334688](https://ubuntuforums.org/showthread.php?t=2334688)

I am not sure if resetting BIOS to defaults might help.  I use the same wireless chipset but haven't had those issues
