### Post by treat on 2010-08-05
Hi, I guess you are using dns. Try to remove or comment the chroot line and change pid to /var/lib/stunnel4/stunnel4.pid in your stunnel.conf, then add "delay = yes" in your service configuration. Alternatively if you don't change dns server very often you can just cp your /etc/resolv.conf to the chroot dir or just use static ip addresses.

This is what i meant, also the resolv.conf file should go in /var/lib/stunnel4/etc not /etc/stunnel. Don't forget to add delay = yes to the service config sections or it won't work, it will make stunnel delay dns lookup until it's needed otherwise it will do the dns lookup once at start up (before chroot). The problem with the delay option is that when the dns lookup is needed stunnel has chrooted and can't find the dns config file resolv.conf so you need to either disable chroot or make the dns config file available under the chroot dir:

```
sudo mkdir /var/lib/stunnel4/etc
sudo cp /etc/resolv.conf /var/lib/stunnel4/etc
```

Another option is to mount /etc in the chroot by adding this line to /etc/fstab

```
/etc /var/lib/stunnel4/etc none bind 0 0
```
