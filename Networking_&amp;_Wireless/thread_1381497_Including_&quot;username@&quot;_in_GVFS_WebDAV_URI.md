### Post by andrewkk on 2010-01-14
My school allows WebDAV connections to network drives. When I try to connect in Ubuntu using the URI *davs://**username@**msfs-stu1.wwu.edu/stu1/username*, I am prompted for my password and then I receive the error message *Not a WebDAV enabled share*. When I try to connect using the URI *davs://msfs-stu1.wwu.edu/stu1/username* instead, I am prompted for both username and password, then GVFS successfully mounts the network drive.

The former URI is generated by Ubuntu's *Places -> Connect to Server* GUI when the *User Name* field is specified. When that field is left blank, the latter URI is generated.

Why is there a difference between specifying the username in the URI versus waiting to type it into the credentials prompt? Is this just some GVFS nuance, or is there a formal difference between the two methods?
