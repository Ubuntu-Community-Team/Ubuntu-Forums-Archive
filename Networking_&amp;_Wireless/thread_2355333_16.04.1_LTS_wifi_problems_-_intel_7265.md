### Post by chaichy on 2017-03-11
Last week I freshly installed Ubuntu 16.04.1 LTS on my laptop, since then I have problems with an instable wifi connection. Until the new installation the wifi was working quite well on 16.04 LTS. Since the fresh set-up the connection drops, especially when using my university's wifi. When suspending the notebook wifi sometimes reconnects after wake, but not always. Changing networks results in endless tries for connecting after the second network change.

I already did my google research, but it seems like the Problems on the threads I read never actually were solved and eventually no new threads regarding the problem were posted. 
One of the most common pieces of advice was to check 
```
sudo iw reg get

which produces 
```

	(2457 - 2482 @ 40), (N/A, 20), (N/A), NO-IR

	(5735 - 5835 @ 80), (N/A, 20), (N/A), NO-IR
	(57240 - 63720 @ 2160), (N/A, 0), (N/A)

```
but ```
sudo iw reg set DE
``` does not change anything at all, also setting it via /etc/default/crda and rebooting the system didn't change a thing.

Following the example of this  [thread]("https://ubuntuforums.org/showthread.php?t=2335798") I used the wireless-info and uploaded it to [Ubuntu Pastebin]("https://paste.ubuntu.com/24160059/")

All advice is appreciated, since I already ran out of ideas of how to fix this problem. As mentioned wifi was working with ubuntu properly -- until reinstalling the system.

### Post by jeremy31 on 2017-03-11
It shows wifi power management enabled so ```
sudo sed -i 's/wifi.powersave = 3/wifi.powersave = 2/' /etc/NetworkManager/conf.d/default-wifi-powersave-on.conf
```
If you can, change your access point to channel 9 as there are a few other access points on the current channel

Post the result of ```
cat /etc/default/crda
```
