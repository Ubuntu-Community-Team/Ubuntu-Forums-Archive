### Post by TrevP on 2009-06-21
I've recently built a new computer which I am using exclusively as a Linux box with Jaunty Jackalope installed. Everything works (and works really quickly!) apart from the wireless card which uses the rtl8180 driver. Here, the card works OK, but (apart from occasionally) the average data transfer is only around 6Mb/sec (I have a 20Mb/sec optical fibre connection). My son's Windows based computer connected to the same network realises much faster transfer rates. I've tried many tweaks (including using the windows driver under Ndiswrapper) but nothing improves it.

However, (and this is the point of this thread) I found an old Belkin ethernet bridge (model F5D7330) and tried this instead. Here the computer simply thinks it's plugged into a network so no configuration is necessary. This works excellently with average data transfer speeds of more than 16Mb/sec downstream and more than 800kb/sec upstream. I have also tried the configuration wizard for this device and it runs perfectly under Wine! If anyone is just setting up their machine for wireless networking and has not yet invested in the hardware, I would suggest considering this method of connection (not necessarily a Belkin device - any ethernet bridge should do the trick). In many years of using Linux with various wireless devices, I have never has a better connection than I have now.
