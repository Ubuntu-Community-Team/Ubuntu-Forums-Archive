### Post by ectospasm on 2008-12-22
I just (re-)installed openvpn on my home workstation, and I followed the instructions to install resolvconf.  When I start OpenVPN ("sudo /etc/init.d/openvpn start <company>"), I get varying behavior.  Sometimes, parts of what is supposed to go into /etc/resolv.conf gets there, and other times nothing happens.  This is what's supposed to be there, from my home DHCP server, before the OpenVPN connection is started:

```
cat /etc/resolv.conf
# Dynamic resolv.conf(5) file for glibc resolver(3) generated by resolvconf(8)
#     DO NOT EDIT THIS FILE BY HAND -- YOUR CHANGES WILL BE OVERWRITTEN
nameserver 10.20.30.1
search local
```

Right now, the above doesn't change whether I've started OpenVPN or not.  If resolvconf was working properly, I'd expect this:
```
cat /etc/resolv.conf
# Dynamic resolv.conf(5) file for glibc resolver(3) generated by resolvconf(8)
#     DO NOT EDIT THIS FILE BY HAND -- YOUR CHANGES WILL BE OVERWRITTEN
search company.internal
nameserver 10.19.29.252
```

Sometimes, I get a combination of the two files, like so:
```
cat /etc/resolv.conf
nameserver 10.19.29.252
nameserver 10.20.30.1
search local
```

Which doesn't help me, because the search string is wrong when I'm connected to my company's VPN.  And yes, I could overwrite /etc/resolv.conf every time I wanted to connect to the VPN, but then why am I even bothering with resolvconf in the first place?

I've made sure to edit /etc/openvpn/<company>.conf, and add the up/down lines for /etc/openvpn/update-resolv-conf, and it seems to be working, except for the fact that it's not getting the right information into the /etc/resolv.conf link.

Help?
