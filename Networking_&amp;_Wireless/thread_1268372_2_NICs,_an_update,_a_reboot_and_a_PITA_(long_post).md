### Post by aesis05401 on 2009-10-09
Your subnetting is not logical.   

Your subnet establishes network prefix and host - the network prefix is the area filled with ones, the host address space is determined by the section of zeroes at the end:
```

                |--   Network Prefix   --| |-host-|          
255.255.255.0 = 11111111.11111111.11111111.00000000

```
You are using two different network prefixes because your IP addresses and subnet mask do not line up, and your host portions resolve to the same thing.  

Here are your ip addresses translated into binary with the 'Network Prefix' and 'Host' sections that are determined by your subnet mask marked for comparison:
```

                |--   Network Prefix   --| |-host-|
192.168.3.1   = 11000000.10101000.00000011.00000001
192.168.15.1  = 11000000.10101000.00001111.00000001

The above would be acceptable with a subnet of 255.255.248.0 as it
translates below, but that would theoretically allow a massive 
number of hosts on your LAN:

                |-- Network Prefix  --||-- Host --|
255.255.248.0 = 11111111.11111111.11111000.00000000
192.168.3.1   = 11000000.10101000.00000011.00000001
192.168.15.1  = 11000000.10101000.00001111.00000001

Note that the IP addresses are now identical reading left to right up 
until the last digit covered by the subnet mask network prefix section.  
This would place 192.168.3.1 on a the 192.168.0.0 network, 
and 192.168.15.1 on the 192.168.8.0 network.  

Each network would have a max of 2048 valid addresses... Way too many.

```

If you subnet at 255.255.255.128 and transition to the same network prefix (ie:192.168.3.XXX) you will segregate your network for two address spaces that are differentiated by the final binary octet value (1-126 with broadcast on 127 and 129-254 with broadcast on 255):

```

                  |--  Network Prefix      --|| host|
255.255.255.128 = 11111111.11111111.11111111.10000000

#Subnet1 (Notice the last octet never has a 1 in the first position 
to the right of the last decimal):
                  |--  Network Prefix      --|| host|
192.168.3.1     = 11000000.10101000.00000011.00000001
....
192.168.3.126   = 11000000.10101000.00000011.01111110


Subnet2 (Notice the last octet always has a 1 in the first position 
to the right of the last decimal):

                  |--  Network Prefix      --|| host|
192.168.3.129   = 11000000.10101000.00000011.10000001
...
192.168.3.254   = 11000000.10101000.00000011.11111110

```

This will get your addresses out of conflict and should allow you to run both of your nics without trouble.  Does this make sense to you?
