### Post by fwre01 on 2009-09-18
You're not receiving a ping response from inside your gateway, but hey! thats not a problem. The good news is that you're seeing an arp reply.

At first glance it would appear asthough you arent resolving DNS requests. However, because your other machines are working on your network, I find it hard to believe that your DNS server address is wrong. 

Its common for clients to use their default gateways as DNS forwarders (which is what you're currently doing).

But just to be sure, can you do this...

```
sudo gedit /etc/resolv.conf
```

and make the contents of that file look like this... Then save it.

```

# Generated by NetworkManager
nameserver 208.67.222.222
nameserver 208.67.220.220

```

These are open DNS severs that anyone can use. Then can you post the output from...

```
ping google.co.uk -c 5
```

If this again fails, it sounds more like a problem with your wireless IP stack (in software). Which may require a re-install. Unless anyone else has other ideas. Sorry mate.
