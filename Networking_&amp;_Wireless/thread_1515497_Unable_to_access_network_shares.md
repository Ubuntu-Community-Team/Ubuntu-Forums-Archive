### Post by metalf8801 on 2010-06-22
ok my first thought is that it could be a permission problem because thats something I've run into with Samba when I could connect to the share when running Ubuntu but then I couldn't when I was running Windows. However, that of causes isn't what you are dealing with. but if it is a permissions problem one way to check would be to temporarily change the permissions to 777 to which gives access to everyone then if that works its a permissions problem but ether way do not leave the permissions at 777. Also only try that if your doing this at home. 


I did a little googling and I found this suggestion a few places so in case you haven't already tried this here is


sudo nano /etc/samba/smb.conf


change the following line

; name resolve order = lmhosts host wins bcast

into this (delete semicolon and move host to the end)

name resolve order = lmhosts wins bcast host

then restart your PC


it was written best here [http://ubuntuswitch.wordpress.com/tag/nautlius/](http://ubuntuswitch.wordpress.com/tag/nautlius/)

I hope this helps and good luck
