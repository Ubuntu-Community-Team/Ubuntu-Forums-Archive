### Post by SeijiSensei on 2022-09-16
In your config:

If you're using Linux, you should use "device tun".

Rather than relying on DHCP you can specify the local and remote tunnel IPs with an "ifconfig local_ip remote_ip" statement. 

I only use static tunnels with a prearranged key, so I can't help with any of that negotiation stuff.

I'd turn off comp-lzo unless your on a very slow connection,

> Sep 15 23:52:32 Vader nm-openvpn[18827]: WARNING: Failed running command (--up/--down): external program exited with error status: 

Though I don't see it in the config file, something is trying to run a script with "up/down," probably to configure routes.
