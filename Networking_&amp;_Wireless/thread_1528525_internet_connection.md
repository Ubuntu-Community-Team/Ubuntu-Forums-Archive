### Post by Detonate on 2010-07-09
You can install network manager from you desktop or alternate cd. Or USB if that is the way you originally installed Ubuntu.  You can do this by opening Synaptic and select Settings>Repositories>Ubuntu Software  and checking the CD Rom boxes  or you can edit your /etc/apt/sources.list file as with sudo privileges and uncomment the CD Rom lines.

Then enter in a Terminal:
```
sudo apt-get update
```
```
sudo apt-get install network-manager network-manager-gnome
```

### Post by Detonate on 2010-07-09
Easy. I have the same router.  So your gateway for your interfaces file is same as it is in the one in my instructions.  Once you have the interfaces file edited you should be able to connect to the router.  But you may not be able to connect to the internet until we get the DNS's entered in /etc/resolv.conf.  To find out what DNS's are currently entered in your router (you can change these if you want) open the router interface by entering 192.168.1.1 in the url bar of your browser.  If I remember correctly the username should be left blank and the password is "admin" unless you have changed it.  The first page that opens up should be "Setup".  If you look down towards the bottom of the page you should see The DNS entries the router is currently set for.  If you have not changed any of the defaults you will see the router has IP 192.168.1.100 as the starting address for DHCP.  Now you have a choice.  The static IP you choose must be outside the range of the DHCP reserved IP's.  On my network I use 192.168.1.10 thru 192.168.1.99.  This works well.  If you want, you can disable DHCP but that means if a visitor with a laptop happens by, you must help them set up their computer with an IP address of your choosing.  I leave DHCP enabled but restrict the number of users to just a few in the block just below the starting IP address.  If you make any changes, click "Save Settings" at the bottom of the page.
