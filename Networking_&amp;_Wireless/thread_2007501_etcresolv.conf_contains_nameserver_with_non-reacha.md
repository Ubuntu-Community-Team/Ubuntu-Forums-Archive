### Post by sanderj on 2012-06-20
My Ubuntu 12.04 often becomes very slow. I found the reason: the first nameserver in /etc/resolv.conf has a IPv6 address fd00::be05:43ff:fe4f:ea6, which is not reachable and (thus) does no name resolving.

Work around: "nameserver 8.8.8.8" as first nameserver.

Some more info:
- fd00:: seems to be some kind of "private IPv6" address range
- I'm often connected to IPv6 enabled networks

Is this a bug? A feature?
How can I avoid it?

### Post by lemming465 on 2012-06-21
The fd00::/7 address is a "unique local address", the v6-replacement for RFC-1918 private addresses.  Someone somewhere is generating one wrongly (the next 40 bits shouldn't be zero); see [RFC 4193]("http://tools.ietf.org/html/rfc4193").   It's probably getting to you by DHCPv6 from somewhere, though it might be one of the newfangled RFC6106 DNS options in a router advertisement. 

Avoiding it is going to depend on specifics of your machine and its upstream topology and if things mutate due to mobility or VPN connections and exactly how you have things configured.  Is this on a laptop?  A desktop?  Is the uplink a wifi router you control the firmware for, or e.g. a cable modem you don't?  Is your v6 native or tunneled?  Who provides your v6 routing?  Do you have a static v6 prefix or a dynamic one?  Etc.

Not a bug.  I'll bet you are using DHCP and the DHCP client is what supplies the DNS server parameter.  Read the man page ```
man resolvconf
```

This is specifically what I'm referring to```
Normally  resolvconf  is  run  only by hook scripts attached to network
       interface configurers such as pppd(8) (for  ppp  interfaces),  to  DHCP
       clients  such  as dhclient(8), to ifup(8) and ifdown, and to DNS caches
       such as dnsmasq(8) (for the loopback interface).   These  hook  scripts
       furnish  resolvconf  with  information about nameservers.  For example,
       dhclient receives one or more nameserver addresses during its  negotia&#8208;
       tion  with  the  DHCP server; its hook script /etc/dhcp/dhclient-enter-
       hooks.d/resolvconf pushes this information to resolvconf.

```

It is confusing as there is an /etc/resolv.conf file and also a service named resolvconf that is now used to auto-configure /etc/resolv.conf.
