### Post by jdthood on 2012-11-18
It seems that something related to your resolver configuration is wrong.

First make sure that the resolvconf package is installed.

    ```
sudo apt-get install resolvconf
```

If it was already installed, make sure that it installed correctly.

    ```
sudo dpkg-reconfigure resolvconf
```

If /etc/resolvconf/resolv.conf.d/tail is a symbolic link to "original" then make it a symbolic link to "/dev/null".

    ```
sudo ln -nsf /dev/null /etc/resolvconf/resolv.conf.d/tail
```

If /etc/resolvconf/resolv.conf.d/head or /etc/resolvconf/resolv.conf.d/base or /etc/resolvconf/resolv.conf.d/tail contains any "nameserver" lines then use an editor to remove those lines.

Add the static nameserver address information to the "iface eth0" stanza in /etc/network/interfaces on a line like the following.

    ```
dns-nameservers 208.67.222.222 208.67.220.220
```

Note the letter 's' at the end of 'dns-nameservers'. The whole stanza will appear as follows.

    ```
iface eth0 inet static
        address 10.1.45.76
        netmask 255.255.254.0
        gateway 10.1.44.1
        dns-nameservers 208.67.222.222 208.67.220.220
```

Reboot.  Log in and make sure that /etc/resolv.conf contains exactly two "nameserver" lines.

    ```
nameserver 208.67.222.222
nameserver 208.67.220.220
```

If it does not then we need to investigate further.
