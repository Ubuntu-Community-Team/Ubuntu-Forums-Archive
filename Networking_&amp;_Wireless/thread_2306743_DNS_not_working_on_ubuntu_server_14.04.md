### Post by SeijiSensei on 2015-12-18
You're missing an important line that should be at the top of the iptables ruleset:
```

/sbin/iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

```
This tells the machine to accept packets sent in response to requests sent from it.  With your rules the replies from the DNS servers are blocked.

Linode's nameservers should work fine after this change.

```

# host www.google.com 109.74.194.20
Using domain server:
Name: 109.74.194.20
Address: 109.74.194.20#53
Aliases: 

www.google.com has address 74.125.136.103
www.google.com has address 74.125.136.147
www.google.com has address 74.125.136.99
www.google.com has address 74.125.136.104
www.google.com has address 74.125.136.106
www.google.com has address 74.125.136.105
www.google.com has IPv6 address 2a00:1450:4013:c01::6a

```

You may not be using dnsmasq, but Ubuntu runs resolvconf by default to generate /etc/resolv.conf.  If you want to use a different nameserver you have to change /etc/resolvconf/resolv.conf.d/head like this:
```

# Dynamic resolv.conf(5) file for glibc resolver(3) generated by resolvconf(8)
#     DO NOT EDIT THIS FILE BY HAND -- YOUR CHANGES WILL BE OVERWRITTEN
nameserver 8.8.8.8

```
That will put the Google nameserver ahead of the others in resolv.conf.  But I don't see any reason not to use Linode's servers.

The other alternative is to disable resolvconf entirely and create resolv.conf yourself.  On a server that makes good sense to me.
