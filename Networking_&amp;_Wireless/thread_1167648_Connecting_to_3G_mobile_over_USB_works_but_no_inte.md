### Post by GeorgeVita on 2009-05-23
Hi **skatebiker**,

> --> Carrier detected. Waiting for prompt.

To skip above message add the line: **Stupid mode = yes**

Usually 3G providers need to set an APN (if already not stored to SIM profiles). This can be done by adding the line:
**Init3 = AT+CGDCONT=1,"IP","internet"**
Possibly "internet" could be "internet.vodafone.xx" or anything else for your provider (search windows modem log file or give more info about your country/provider).

Finally check with Wireless Disabled and Ethernet disconnected.

EDIT: also why you faced: "*sudo: unable to resolve host klaas-unix*"
Do you have more users and this one is not root?

Regards,
George

### Post by GeorgeVita on 2009-05-23
Hi again,

below I am trying (as I can) to summarise the parameters needed for a successful Mobile data connection:

1- the system recognises the modem, creates a communication port (/dev/tty...) and uses a driver to drive the port (usbserial, option,...)

2- some configuration data initialise the modem and the data account (/etc/wvdial.conf, Network Manager, pppconfig) providing phone number, APN, username and  password.

3- most cases a valid SIM card is used (with data capability)

4- the provider accepts all data regarding type of connection for the specific customer

5- data flow using compatible methods and coding (ppp, TCP/IP)

6- also there must not be any conflict with other similar programs trying to get the data flow and use it (3G modem vs wireless and ethernet)

For your case, and as the wvdial.conf seems to work, you must check if  the APN=live.vodafone.com is used for data and not WAP. Check also the **cost** if you try other APN (some companies charge a lot of money for the **modem** data connection).

For the Bluetooth connection I do not know anything else than [http://en.wikipedia.org/wiki/Tethering](http://en.wikipedia.org/wiki/Tethering)

Regards,
George
