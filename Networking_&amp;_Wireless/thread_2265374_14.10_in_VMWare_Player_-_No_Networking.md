### Post by TheFu on 2015-02-15
Add this line to the interfaces file in the stanza for the specific eth0:
```
  dns-nameservers 8.8.8.8
```
in 12.04 an later Linux versions, this became how to manage dns servers.  Assuming you need a static IP, then editing the interfaces file is the way to do it on a server following the requirements spelled out in the manpage (man interfaces). On a desktop, most people prefer to use the GUI.

If you don't need a static IP, which is more common, s/static/dhcp/ and remove the last 3 lines. This is best done using bridged networking for a VM, not NAT.

BTW - we only needed the ifconfig and interfaces output. There's nothing wrong with the drivers.
[http://blog.jdpfu.com/2013/03/01/linux-troubleshooting-101-networking](http://blog.jdpfu.com/2013/03/01/linux-troubleshooting-101-networking) - how to troubleshoot Linux networking.

NAT for VMs complicated networking, IMHO. I don't use it.
If all your machines use .2 ... then why wouldn't you put that in here?

Also - why not use virtio for the networking setting in the VM settings? It is faster, lighter and well supported by popular Linux distros.
