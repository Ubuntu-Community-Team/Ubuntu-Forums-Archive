### Post by arnaldo on 2011-10-25
Since time immemorial I had two interfaces configured on the same network card, eth0 and eth0:0.  Both auto, static.

Since upgrading to 10.10, only one of these will come up.  Moreover, resolv.conf starts up  with a single, commented out line, saying it was generated by NetworkManager. /etc/network/interfaces has all the needed information and is syntactically correct; it has not changed for a long time and network config worked flawlessly before this upgrade.  Any pointers?

---

### Post by chili555 on 2011-10-25
Manual configuration; that is /etc/network/interfaces, conflicts with Network Manager. It is doubtful you'd ever get a complex arrangement (eth0:eth0) going correctly with NM. I suggest you remove it. Re-populate /etc/resolv.conf, reboot and you should be all set.
