Hi all!

I have a hard time understanding a couple of things.


I have a supported USB 3G-modem that connects me to the web on my laptop (let's call this laptop 'server'). By way of masquerading I want to share that internet connection to a WLAN access point, which in turn shares the connection to a couple of other computers (let's call these 'clients').

Now, the 'server' has a few firewall rules, along with ip forwarding and masquerading. On top of that I've set up a simple dhcp server that gives clients (in this case the AP) what they need to get along. The dhcp configuration is very simple and looks like this:

```
# Temporary DHCP configuration generated by share-the-net.sh
subnet 192.168.2.0 netmask 255.255.255.0 {
        option routers 192.168.2.1;
        option subnet-mask 255.255.255.0;
        option domain-name-servers 80.251.192.244, 80.251.192.245;
        range dynamic-bootp 192.168.2.100 192.168.2.254;
}

```

The AP's LAN address is 192.168.2.2 (I've also tried with 192.168.3.1). I've also turned off the AP's firewall to have it function only as a hub.

The clients have access to the web, which means that ip forwarding works fine from the server to the AP, and from the AP to the clients. However, from the server it's impossible to reach the clients. I can't see their shares, and I can't ping them.

So, finally, any ideas of what's wrong here? Should the AP be on 192.168.2.X or 192.168.[something else].X?

Thanks a lot!
