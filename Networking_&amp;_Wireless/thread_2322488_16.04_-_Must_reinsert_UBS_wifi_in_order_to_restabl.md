### Post by jeremy31 on 2016-05-02
After a reboot with it plugged in and before it is working run ```
nmcli -f all device show | sed '/^GENERAL.DEVICE:[ ]\+lo$/,/^$/d; /^AP\[[0-9]\+\]\./d'
```
Then unplug and plug it back in so it works and you can post the results from the command

I think the driver from [https://github.com/pvaret/rtl8192cu-fixes](https://github.com/pvaret/rtl8192cu-fixes) might help

### Post by praseodym on 2016-05-06
Via
```
sudo modprobe -rfv 8192cu
sudo dkms remove -m 8192cu -v 4.0.2.9000.20130911 --all
sudo rm -r /usr/src/8192cu-1.10
sudo rm /etc/modprobe.d/blacklist-rtl8xxxcu.conf
```
Reboot
