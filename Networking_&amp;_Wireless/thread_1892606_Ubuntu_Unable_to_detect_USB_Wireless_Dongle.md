### Post by praseodym on 2011-12-08
Hi,

install the driver via (Installation for Ubuntu until 11.04):

```
sudo apt-get install --reinstall linux-headers-$(uname -r) linux-headers-generic build-essential dkms
wget http://media.cdn.ubuntu-de.org/forum/attachments/2844083/rtl8192cu-3.1.2590_dkms.tar.gz
sudo tar xvf rtl8192cu-3.1.2590_dkms.tar.gz -C /usr/src
sudo dkms add -m rtl8192cu -v 3.1.2590
sudo dkms build -m rtl8192cu -v 3.1.2590
sudo dkms install -m rtl8192cu -v 3.1.2590 
```
Reboot and check:

```
uname -a
lsmod
iwconfig
dmesg | grep r8
sudo iwlist scan
iwlist chan
```

> uname -a

The results appeared no different than they had when the original installation attempt failed.  That was what I had posted, and was wondering what to do next. It was as if Ubuntu had always seen the dongle, and the wireless had never been missed.  All relevant connections were in place, and I was able to unplug the direct wired connection and use the wireless connection.

So, direct download, and open all at once with Archive Manager.

Open root using your Super User prerogatives.

Drag, and drop the already unwrapped/opened driver folder into /root/usr/src/

Run the installation commands in Terminal;

> sudo dkms build -m rtl8192cu -v 3.1.2590
sudo dkms install -m rtl8192cu -v 3.1.2590 

Reboot.

That should work, at least on a Dell Latitude C600, with a RealTek USB dongle.  ;) :D
