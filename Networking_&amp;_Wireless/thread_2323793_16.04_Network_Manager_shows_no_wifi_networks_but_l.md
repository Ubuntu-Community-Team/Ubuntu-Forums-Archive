### Post by id02 on 2016-10-02
Hi,
I had this problem too, and I created a fix (xubuntu 16.04!)
It looks like nm-applet doesn't like to be owned by the "session and startup" process.

I created a simple script here to automate the task of killing and reinitiating it: [https://github.com/id01/nm-applet-fix](https://github.com/id01/nm-applet-fix)
You can add this script to your "Session and Startup > Application Autostart". It'll work. Disable the original nm-applet starter just in case.
For some strange reason I put "nm-applet & disown" in application autostart and it didn't work. Only seemed to work in some type of script. Strange...

Well, I hope the official fix comes soon but until then, it looks like this problem is for the most part, solved. Unless, of course, this script is broken in a future update.
