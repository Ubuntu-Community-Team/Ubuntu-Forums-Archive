### Post by jpepin on 2011-04-03
I know it's been a while since the original post, but I just ran into the same exact problem recently, and have been trying to find a fix for a couple days.

To clarify, my problem was that web pages were taking forever to load.  This happened regardless of browser.  Windows worked without a problem.

Things that didn't work for me: 
- disable ipv6 in forefox in about:config
- disable ipv6 on boot in /etc/defaults/grub:
  ```
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash ipv6.disable=1"
```
- change dns server setting on router to Google Public DNS (8.8.8.8 and 8.8.4.4)

What worked for me: I changed to following line in /etc/nsswitch.conf

```
hosts:          files mdns4_minimal [NOTFOUND=return] wins dns mdns4
```

To this (removed "wins"):

```
hosts:          files mdns4_minimal [NOTFOUND=return] dns mdns4
```

I'm not sure why this fixed it, or why the problem began in the first place.  I had modified the file months ago to fix another issue I was having.  The change has never caused trouble until now.  I assume some update caused that entry to become troublesome.
