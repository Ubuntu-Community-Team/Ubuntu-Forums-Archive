**====== Wireless Access Point & IP Forwarding Mini-Howto ======**

**Use case:** we have a pc (laptop, for that matter) with UBUNTU 10.04, and with wired connection to the internet. We want to connect iPod/iPhone or any other device to this laptop, and browse the internet, effectivelly turning our pc/laptop to a WI-FI access point. At this stage, we would not implement any encryption, so our network would be open to anyone.

Oour pc/laptop have 2 working interfaces, wlan0 and eth0. Device eth0 is the wired connection to the internet, device wlan0 is going to be set up as the wireless access point for the iPod/iPhone or similar gadget. We want to forward any IP traffic attached to wlan0 through to eth0. Interface eth0 can either obtain the IP addres from DHCP server on the router, or it can be fixed, it does not matter. But we definitelly need to set fixed IP address to wlan0, and configure dhcp server to listen on this interface.

**===== Things to do =====**

[LIST=1]
[*]configure iptables to forward the traffic from wlan0 to eth0
[*]setup wlan0 in /etc/network/interfaces
[*]configure DNS server for wlan0 interface
[/LIST]

**===== Configuring iptables =====**

**==== IP forwarding ====**

First, make sure we have IP forwarding enabled on on the kernel level:

```
echo 1 > /proc/sys/net/ipv4/ip_forward
```

Put this line to /etc/rc.local, so the forwarding is set every time we start the machine.

**==== iptables ====**

In order to achieve desired routing, run following in a console as a root:

```
iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE && iptables -A FORWARD -i wlan0 -j ACCEPT
```

If the command runs without errors, it means that forwarding is actually set in a way we want it. In order to keep the settings and load them every time the interface wlan0 goes up, dump the config file like this, again preferably in root console:

```
iptables-save
```

output looks like this:

```
# Generated by iptables-save v1.4.4 on Sun Aug 15 12:57:00 2010
*filter
:INPUT ACCEPT [7423:2313642]
:FORWARD ACCEPT [36648:53016274]
:OUTPUT ACCEPT [11365:4610810]
-A FORWARD -i wlan0 -j ACCEPT 
COMMIT
# Completed on Sun Aug 15 12:57:00 2010
# Generated by iptables-save v1.4.4 on Sun Aug 15 12:57:00 2010
*nat
:PREROUTING ACCEPT [490:37325]
:OUTPUT ACCEPT [456:57854]
:POSTROUTING ACCEPT [170:25138]
-A POSTROUTING -o eth0 -j MASQUERADE 
COMMIT
# Completed on Sun Aug 15 12:57:00 2010
```


In order to re-use it, let's save it in the proper config file:

```
iptables-save > /etc/iptables.rules
```

We will use it later, when configuring network interfaces.

**===== Networking settings =====**

As written above, we would not bother about eth0 very much, so let's focus on wlan0 setting. Since we are on UBUNTU, it is pretty much about playing with /etc/network/interfaces. Open it up for editing

```
sudo vi /etc/network/interfaces
```

and make sure that wlan0 is set up like this:

```
auto wlna0
iface wlan0 inet static
wireless_essid ADHOC
wireless_mode ad-hoc
wireless_channel 2
address 10.0.0.1
netmask 255.255.255.0
  pre-up iptables-restore < /etc/iptables.rules
```

It is pretty much self explanatory, our ESSID would show as ADHOC on channel 2, with an IP address 10.0.0.1, and last line loads iptables configuration every time the interface goes up. So far so good, if we restart networking and put wlan0 up, our ESSID should be already visible. So let's do the last step.

**===== Configuring DHCP server =====**

Edit the configuration file of the dhcp server, provided by UBUNTU (dhcp3-server package). Open up the root console:

```
sudo vi /etc/dhcp3/dhcpd.conf
```

and insert follwing and only content:

```
subnet 10.0.0.0 netmask 255.255.255.0 {
default-lease-time 600;
max-lease-time 7200;
option domain-name-servers 192.168.1.1;
option routers 10.0.0.1;
option broadcast-address 10.0.0.255;
range 10.0.0.10 10.0.0.50;
}
ddns-update-style none;
```


Again, pretty legible. Note, that option-name-servers has to correspond to the nameserver from /etc/resolv.conf, router is an IP address of wlan0 interface, we have configured before, and range simply limits the array of IP addresses we want to assign. Make it as wide or narrow as you wish. Save the file and check the server:

```
service dhcp3-server status
```

If the service is not running, bring it up

```
service dhcp3-server start
```

or if it already runs, restart it

```
service dhcp3-server restart
```

Read any errors, and try to figure out where you did a typo ;-). If successfull, we have all we need to pull out the iPod and swith the WI-FI. Yu should see your ESSID and, connect to it, get an IP address and browse the net.

### Post by seydou on 2010-10-13
> **Amit Howard said:**
> How do I access root console from my user profile? and can I still use my laptop to access internet through other Wireless points when it's not connected to Ethernet at the house?

Hi, sorry for being a bit late - nevertheless, as for the root console, open the terminal and issue

```
sudo su -
```

and you are there. The latter is a bit tricky - you can do it, but you would most likely need to change /etc/network/interfaces. I have podered a bit on the subject and wrote short script, which does the job (converts the laptop to an AP), but does not impair the ability to connect to your wireless router back at home. So leave this and only this in /etc/network/interfaces

```

auto wlan0
iface wlan0 inet dhcp

```

We will change the config with ifconfig - see the script below. Prepare the iptables setting and dhcp as suggested above and have a look at the script. It needs to be run with root privileages, though, so if you save it under the name 'ap_start' in your path (for example /usr/local/bin), you need to issue

```
sudo ap_start
```

in the console. So here it goes, all steps commented and in correct order:

```

#enabling IP forwarding
echo 1 > /proc/sys/net/ipv4/ip_forward

#Deconfigure network interface wlan0, just to be sure
ifdown wlan0 2>/dev/null

#set the wireless configuration, key is off
iwconfig wlan0 essid "ADHOC" channel 2 mode Ad-Hoc key off

#Reconfigure wlan0 to desired IP address for Access Point
ifconfig wlan0 10.0.0.1
ifconfig wlan0 netmask 255.255.255.0

#Restore iptables configuration
iptables-restore < /etc/iptables.rules

#Start the dhcp service - this can be omitted,
#provided you setup IP address on accessing device manually
service dhcp3-server start

#set the wireless configuration, again in order to wake the AP up
iwconfig wlan0 essid "ADHOC" channel 2 mode Ad-Hoc key off

```

This should be enough to make it running. If you turn the laptop off, it gets discarded, so you can connect at home as usual.
