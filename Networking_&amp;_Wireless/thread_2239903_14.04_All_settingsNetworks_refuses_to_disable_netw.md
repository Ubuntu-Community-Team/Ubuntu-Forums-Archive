### Post by Hertzyscowicz on 2014-08-16
My wired network has been acting iffy lately, so I'm occasionally using my mobile phone for network access. When this happens,  I naturally want my programs to use the mobile connection, so I want to disable the ethernet connection. However, when I go to All Settings/Network and click the slider to switch the connection off, it immediately snaps back on.

This has been happening since update manager prompted me to upgrade to a new version of Ubuntu on the 13th. This update broke or messed up a number of other things, so I suspect that this issue is related as well.

EDIT: to add, it appears that the settings window retains an old IP address (I have manually acquired a new IP via dhclient), and uses it instead of acquiring a new IP by dhcp. I also get that old IP during boot.

---

### Post by Hertzyscowicz on 2014-08-16
Okay, this behavior was because NetworkManager was configured to auto-connect to the network every time it became available, which overrode manual disabling. Also, the odd DHCP behavior is most likely due to a rogue DHCP server.
