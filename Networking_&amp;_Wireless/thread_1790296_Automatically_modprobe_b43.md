### Post by fhgshfdg on 2011-06-24
Hello Linux buddies,

I'm trying to configure my network card using the b43 driver. I'm running 10.04.2 on the kernel version 2.6.35-25-generic-pae.

Surprisingly, I was able to connect to my home wireless network using the b43 driver without a problem. The only issue is that it doesn't automatically connect. I have to use the following command in order to get it to work:

```
sudo modprobe b43
```After typing my password, I wait for 10 seconds and am automatically connected. My question is, how do I configure my system to do this for me permanently (or at least automatically) so I don't have to type it every time I log on?    

Thanks for the help!

---

### Post by chili555 on 2011-06-24
In a terminal:```
sudo su
echo b43 >> /etc/modules
exit
```Enjoy!

### Post by chili555 on 2011-06-24
/etc/modules is a file that the system refers to to load modules automatically, no matter what, so help me, I swear.

Ordinarily, the driver and the card or other device, join up during the boot process by looking at the identifying codes hard-coded into the device and driver. For instance:```
lspci -nn
02:00.0 Ethernet controller [0200]: Intel Corporation 82573L Gigabit Ethernet Controller [[COLOR="Red"]8086:109a[/COLOR]]
``````
modinfo e1000e
filename:       /lib/modules/2.6.38-8-generic/kernel/drivers/net/e1000e/e1000e.ko
version:        1.3.10a-NAPI
license:        GPL
description:    Intel(R) PRO/1000 Network Driver
author:         Intel Corporation, <linux.nics@intel.com>
srcversion:     9F095B7E8A0A830E03B20D0
--- snip ---
alias:          pci:v0000[COLOR="Red"]8086[/COLOR]d0000[COLOR="Red"]109A[/COLOR]sv*sd*bc*sc*i*
--- snip ---
```But sometimes, the system misses or gets lazy and the module and physical hardware don't meet and fall in love.

/etc/modules comes to the rescue and loads the module automatically.
