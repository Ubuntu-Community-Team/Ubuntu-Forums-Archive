### Post by skullmunky on 2011-04-25
My university uses 802.1x Dynamic WEP, with PEAP, MSCHAPv2, and no CA certificate.  Our IT department actually provides instructions for Ubuntu, and they work, but I can't get it to work in Kubuntu.  Here's the instructions using the Gnome Network Manager applet:

   3. Change the LEAP setting to Dynamic WEP (802.1x).
   4. Change the Authentication setting from TLS to Protected EAP (PEAP).
   5. Leave the Anonymous Identity option blank.
   6. Leave the CA certificate setting to None.
   7. Change the PEAP version to version 0 if running 8.10, or Automatic if running 9.04+.
   8. Leave the inner authentication setting to MSCHAPv2.
   9. Enter your username.
  10. Enter your password.
  11. Select OK. A warning message will appear about having no certificate; click Ignore.
  12. Click Connect.

I'm using Kubuntu 10.10.  All the same settings are there for the KDE plasma network manager applet, but the connection always fails immediately.  

How do I go about debugging this?  Is there an error log somewhere, and is there a way to test this out from the command line?
