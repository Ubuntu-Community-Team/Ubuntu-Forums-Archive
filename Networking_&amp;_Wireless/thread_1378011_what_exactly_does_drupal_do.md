### Post by Lars Noodén on 2010-01-11
It's a Content Management System, which is a fancy word for a publishing database for those that want to publish short pages, run forums, or similar functions.  

[list]
[*][Drupal](http://drupal.org/)
[*][OpenCMS](http://www.opencms.org/en/)
[*][Plone](http://plone.org/)
[*][Lyceum](http://lyceum.ibiblio.org/)
[*][Alfresco](http://www.alfresco.com/)
[*][Lenya](http://lenya.apache.org/)
[*][Joomla](http://www.joomla.org/)
[*][Nuxeo](http://www.nuxeo.org/)
[/list]

These are used all over the place and are so common that they usually aren't even mentioned.  But here are two examples

[http://www.itwire.com/content/view/30304/1141](http://www.itwire.com/content/view/30304/1141)
[http://www.theage.com.au/technology/enterprise/white-house-switches-to-open-source-20091026-hfdo.html](http://www.theage.com.au/technology/enterprise/white-house-switches-to-open-source-20091026-hfdo.html)

You can go with simpler tools for blogs, such as Roller:

[http://roller.apache.org/](http://roller.apache.org/)

Or heavier with actual document management systems:

[http://o3spaces.org/](http://o3spaces.org/)

However, don't forget that if all you want is to post a few web pages and maybe a simple personal blog, then maybe all you need is a web server (Apache or lighttpd) with SSI (for standard footers, headers and menus) and CSS and maybe a few of your own document templates.
