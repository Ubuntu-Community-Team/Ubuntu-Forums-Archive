### Post by chili555 on 2011-05-14
How did rt2800usb get loaded? It doesn't apply to your device:```
modinfo rt2800usb | grep 0323
```It returns nothing; the human readable meaning is, "I don't know about this device, sorry."

Did you manually load rt2800usb?

It won't matter at all what got built when you compiled the driver. As I said:> I downloaded the package and looked in rtusb_dev_id.c and your usb.id isn't listed.That means, "I don't know about this device, sorry."

I hate to disagree with the install instructions in README_STA_usb, but...I disagree.

If you are determined to compile this little jewel, here is what you do. Open a terminal and navigate to the directory that was created when you extracted the bz2 file; for example:```
cd Downloads/2011_0407_RT3070_RT3370_RT5370_RT5372_Linux_STA_V2.5.0.2_DPO
sudo su
make clean
```Now open the file common/rtusb_dev_id.c with a text editor. We are going to add your device. Find these lines:```
/* module table */
USB_DEVICE_ID rtusb_dev_id[] = {
#ifdef RT3070
	{USB_DEVICE(0x148F,0x3070)}, /* Ralink 3070 */
	{USB_DEVICE(0x148F,0x3071)}, /* Ralink 3071 */
	{USB_DEVICE(0x148F,0x3072)}, /* Ralink 3072 */
	{USB_DEVICE(0x0DB0,0x3820)}, /* Ralink 3070 */
	{USB_DEVICE(0x0DB0,0x871C)}, /* Ralink 3070 */
	{USB_DEVICE(0x0DB0,0x822C)}, /* Ralink 3070 */
	{USB_DEVICE(0x0DB0,0x871B)}, /* Ralink 3070 */
```***Leave everything before and after unchanged!!!*** Add your device as follows:```
/* module table */
USB_DEVICE_ID rtusb_dev_id[] = {
#ifdef RT3070
	{USB_DEVICE(0x148F,0x3070)}, /* Ralink 3070 */
	{USB_DEVICE(0x148F,0x3071)}, /* Ralink 3071 */
	{USB_DEVICE(0x148F,0x3072)}, /* Ralink 3072 */
        [COLOR="Red"]{USB_DEVICE(0x177F,0x0323)}, /* Ralink 3072 */[/COLOR]
	{USB_DEVICE(0x0DB0,0x3820)}, /* Ralink 3070 */
	{USB_DEVICE(0x0DB0,0x871C)}, /* Ralink 3070 */
	{USB_DEVICE(0x0DB0,0x822C)}, /* Ralink 3070 */
	{USB_DEVICE(0x0DB0,0x871B)}, /* Ralink 3070 */
```***Leave everything before and after unchanged!!!***

Spacing, brackets, commas, etc. are crucial! Be sure to proofread carefully before you save and close the text editor. Now back to the terminal:```
make
make install
modprobe rt5370sta
exit
```Now is your wireless working?

### Post by vitoshka on 2011-08-08
Thanks Chili555 and Tychos

I struggled with my[ Sweex LW165]("http://www.sweex.com/en/assortiment/internet-networking/lan-cards/LW165/drivers_firmware/") -   for the whole day, reading various posts ad half baked howtos. 

The above worked perfectly for me. Here I post the detailed instructions wich will probably work for all Sweex ralink wifi adapters:

1 Download the multi-pack driver from [http://www.ralinktech.com/support.php?s=2](http://www.ralinktech.com/support.php?s=2) (mine is RT3370)

2 Unpack it to a folder, say 'rtdriver'.

3 In terminal move to the rtdriver folder and edit os/linux/usb_main_dev.c  by putting 

```
MODULE_LICENSE("GPL");
```just after

 ```
MODULE_DESCRIPTION("RT2870 Wireless Lan Linux Driver");
``````

cd rtdriver
gedit os/linux/usb_main_dev.c

```4 Get you device ID:

```
lsusb 
``````
Bus 007 Device 001: ID 1d6b:0001 Linux Foundation 1.1 root hub
Bus 006 Device 003: ID 413c:2003 Dell Computer Corp. Keyboard
Bus 006 Device 002: ID 413c:3016 Dell Computer Corp. Optical 5-Button Wheel Mouse
Bus 006 Device 001: ID 1d6b:0001 Linux Foundation 1.1 root hub
Bus 005 Device 001: ID 1d6b:0001 Linux Foundation 1.1 root hub
Bus 004 Device 001: ID 1d6b:0001 Linux Foundation 1.1 root hub
Bus 003 Device 001: ID 1d6b:0001 Linux Foundation 1.1 root hub
Bus 002 Device 001: ID 1d6b:0002 Linux Foundation 2.0 root hub
Bus 001 Device 005: ID [COLOR=Red]177f:0165[/COLOR] Sweex 
Bus 001 Device 001: ID 1d6b:0002 Linux Foundation 2.0 root hub

```5 Edit the rtusb_dev_id.c and register your device ID in corresponding driver list:

```
gedit common/rtusb_dev_id.c
```For me it was:

```
#ifdef RT3370
    [COLOR=Red]{USB_DEVICE(0x177F,0x0165)}, /* Sweex  165 */[/COLOR]
    {USB_DEVICE(0x148F,0x3370)}, /* Ralink 3370 */
    {USB_DEVICE(0x0DF6,0x0050)}, /* Sitecom 3370 */
#endif /* RT3370*/
```6 change HAS_WPA_SUPPLICANT** and **HAS_NATIVE_WPA_SUPPLICANT_SUPPORT[B] from n to y in os/linux/config.mk

[/B]```
[B]
# Support Wpa_Supplicant
HAS_WPA_SUPPLICANT=y


# Support Native WpaSupplicant for Network Maganger
HAS_NATIVE_WPA_SUPPLICANT_SUPPORT=y[/B]
```


7 now just make, make install, and register your module with modprobe (for me it was rt5370sta - look in os/linux/ for sta files to see the name of your module):

```
sudo su
make
make install
modprobe rt5370sta
```

8 To enable the module at each boot add your module to /etc/modules

```
 echo rt5370sta >> /etc/modules 
```
