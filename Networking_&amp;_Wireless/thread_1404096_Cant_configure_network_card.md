
### Post by Iowan on 2010-02-11
Is yours a standard CLI-server - or have you installed a desktop (and Network Manager)? **sudo lshw -C network** should provide some information about the interfaces. **ifconfig -a** will show interfaces - active or not. 
Another thing to consider - multiple interfaces in the same subnet tends to be problematic. One of the other interfaces with an address in the DHCP range might be responsible.

### Post by mdewet on 2010-02-12
Thanks Iowan, I'm using 9.04 Server, CLI, so no network manager.

I know this is not the correct method, but I added the card manually to /etc/udev/rules.d/70-persistent-net.rules, where I kept everything the same except for the MAC address and the NAME (eth3).  

```
# 
#This file was automatically generated by the /lib/udev/write_net_rules
# program, run by the persistent-net-generator.rules rules file.
#
# You can modify it, as long as you keep each rule on a single
# line, and change only the value of the NAME= key.

# PCI device 0x8086:0x104b (e1000e)
SUBSYSTEM=="net", ACTION=="add", DRIVERS=="?*", ATTR{address}=="00:19:d1:1f:28:c7", ATTR{type}=="1", KERNEL=="eth*", NAME="eth0"

# PCI device 0x10ec:0x8139 (8139too)
SUBSYSTEM=="net", ACTION=="add", DRIVERS=="?*", ATTR{address}=="00:30:bd:b9:f6:81", ATTR{type}=="1", KERNEL=="eth*", NAME="eth1"
SUBSYSTEM=="net", ACTION=="add", DRIVERS=="?*", ATTR{address}=="02:f0:b0:d6:36:b2", ATTR{type}=="1", KERNEL=="eth*", NAME="eth3"

```

I also configured the new card on another subnet, so everything is fine now.  I'm just curious now about why that happened..
