### Post by newbie-user on 2013-08-17
Remove the line:
```
-A POSTROUTING -s 10.42.0.0/24 ! -d 10.42.0.0/24 -j MASQUERADE
```
Since both subnets on either side of your HTPC are your own local networks, that line is unnecessary since both subnets are trusted

Also, you should enable ip forwarding in /etc/sysctl.conf. To this, uncomment the line:
```
#net.ipv4.ip_forward=1
```

### Post by Destruct0r on 2013-08-18
I ended up using another solution that didn't involve NATing.

The lines
```
-A FORWARD -o eth4 -j REJECT --reject-with icmp-port-unreachable
-A FORWARD -i eth4 -j REJECT --reject-with icmp-port-unreachable

```
I changed to
```
-A FORWARD -o eth4 -j ACCEPT
-A FORWARD -i eth4 -j ACCEPT

```

(As I understand it, the old rule said: all traffic destined to be forwarded to/from eth4 will be dropped.)

Also I had to add a static route to the 10.42.0.0/24 network on my home LAN Router to the HTPC as gateway.  Now my HTPC is routing traffic to the receiver no problems.

(After removing the forward drop all rule, the NAT started working aswell, but since normal IP routing works, I didn't have to use it in the end.)

So, yeah, my IP forwarding was set from the start anyway.  Now I will just have to install these firewall rules after eth4 comes up.
