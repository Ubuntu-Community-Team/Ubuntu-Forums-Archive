```

iface eth0 inet static
  address 192.168.1.5
  network 192.168.1.0
  netmask 255.255.255.0
  broadcast 192.168.1.255
  gateway 192.168.1.1
  pre-up /sbin/ifconfig $IFACE mtu 1492
  dns-nameservers 192.168.1.1

```When I reboot this computer, I have to restart networking manually before the internet works:

```

sudo /etc/init.d/networking restart

```Why isn't the dns-nameservers line being honored when the machine boots up?  Ubuntu 9.10, 2.6.31 kernel.

### Post by chili555 on 2011-07-18
Does this help? [http://www.madirish.net/justin/where-does-conf-resolve](http://www.madirish.net/justin/where-does-conf-resolve)> So if you're looking to edit your /etc/resolv.conf file you should actually edit the file /etc/resolvconf/run/interface/eth0. Once updated be sure to regenerate the files with:

[root@localhost interface]# resolvconf -uNote in his example, he's root # but you can simply use sudo.
