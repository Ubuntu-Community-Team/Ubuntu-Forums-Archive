Hi all,

I'm having trouble getting working my newly purchased EW-7811Un Wireless nano USB adapter (chipset: Realtek RTL8188CUS)

So far I have tried off of the rtl8192cu module. It can detect wireless networks but is unable to connect (repeatedly rejects (correct) password). I've tried the solutions in the threads below. The significant difference between these threads and my set up is possibly that i'm 64 bit.  On reinserting the USB the computer locks up, or on inserting 8192cu module. Because my password has been wiped, the lock up occurs after entering a password on inserting 8192cu. It may also lock up on re-inserting the stick. My situation seems to be the the same as the bug report at the bottom (not filed by me). As the working solutions are for 32 bit systems, I wanted to ask and see if anyone had any suggestions on how to proceed. Either a kernel downgrade, figure out some further patch of the drivers, or getting ndiswrapper to work. I'm still figuring out linux so apologies if I've missed something obvious!

1) Patching the realtek drivers (kernel lock issue) by compiling against smp_lock.h > smp.h)
Result: kernel panic (computer locks up)

>> I've tried a variety of ways of getting this part to work. In one of the links somewhere I used pre-patched and confident I made the correct changes. 

2) Using ndiswrapper. All windows drivers attempted, some of which ndiswrapper is able to load but non of the loaded ones are able to be loaded on inserting ndiswrapper into kernel. (edit: I didn't have ndiswrapper and the rtl drivers loaded at the same time when trying to get this to work)
Result: wlan1 doesn't appear. 

3) I was going to downgrade to 2.6 which has been suggested but I can't get this to install. I have the correct debs downloaded but dpkg -i doesn't complete. It doesn't appear in synaptic. 

I'd appreciate any help on this. I keep having to network with my laptop to get internet connection on this machine. The USB dongle was supposed to solve this!
