### Post by actionmystique on 2015-05-31
I believe I have found a **few glitches**:
[LIST]
[*]_**ping6 always use the same source address, whatever the destination is**_; it should use one of the IPv6 addresses of the connected interface it is sending the traffic on, by default
[/LIST]
[INDENT]Considering the following internal interfaces, virbr0 being a linux bridge[/INDENT]
```
root@msi-ge60:/# ifconfig tap0
tap0  inet6 addr: fc00:137::1/64 *Scope:Global*

root@msi-ge60:/# ifconfig virbr0
virbr0 inet6 addr: fc00::1/64 *Scope:Global*
```[INDENT]*As a side note, these addresses scope is not "Global", but "Private", but this should not interfere with the current situation.*
Considering the [firewall rules]("https://drive.google.com/file/d/0B5fXyIn0-GDFa3JrdWgtd0F3TzQ/view?usp=sharing") allowing traffic into the previous interfaces only from the subnet of their own interface:
[/INDENT]
```
root@msi-ge60:/# ping6 FC00::1[INDENT]PING FC00::1(fc00::1) 56 data bytes[/INDENT]
[INDENT]64 bytes **from fc00::1**: icmp_seq=1 ttl=64 time=0.047 ms[/INDENT]
[INDENT]64 bytes **from fc00::1**: icmp_seq=2 ttl=64 time=0.038 ms[/INDENT]
[INDENT]64 bytes **from fc00::1**: icmp_seq=3 ttl=64 time=0.039 ms[/INDENT]
[INDENT]
root@msi-ge60:/# ping6 FC00:137::1[/INDENT]
[INDENT]PING FC00:137::1(fc00:137::1) 56 data bytes[/INDENT]
[INDENT]**From fc00::1** icmp_seq=1 Destination unreachable: Address unreachable[/INDENT]
[INDENT]**From fc00::1** icmp_seq=2 Destination unreachable: Address unreachable[/INDENT]
[INDENT]**From fc00::1** icmp_seq=3 Destination unreachable: Address unreachable
```[/INDENT]
