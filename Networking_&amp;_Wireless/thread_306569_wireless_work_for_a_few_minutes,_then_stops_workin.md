Can you check a few things for me:

2. What kind of video card (nVidia, ATI, etc.)?  Please post the output of your xorg.conf file:

I'm checking to see if you're got the proprietary nVidia binaries installed.  You should have a section similar to:
```
Section "Device"

    Driver         "**[COLOR="Red"]nvidia[/COLOR]**"

3. Can you post the output of:
```

```

If you've got an nVidia video card and there is a message in dmesg about disabling irq #177, then the problem is with the proprietary nVidia video driver (a known bug).

Try changing the 'nvidia' driver to 'nv' and then test.

[http://www.ubuntuforums.org/showthread.php?t=295787](http://www.ubuntuforums.org/showthread.php?t=295787) (check post 46 for the solution)

### Post by dbott67 on 2006-11-28
The command 'dmesg', which is used to print kernel messages, is useful in determining if a piece of hardware has been found, and if so, what the system is referring to it as.

'grep' means "search **g**lobally for lines matching the **r**egular **e**xpression, and **p**rint them"

The pipe command (typed as a vertical bar | ) joins 2 commands together.  Suppose you have a really long log file with thousands of lines and you only want to look for a specific error.  You could send the output of the log file to grep to search for whatever term you are looking for.

In your case, the command 'dmesg | grep 177' basically reads (in plain english):

*Send the output of dmesg to the grep command and print out any term with '177' in it.*

I'm thinking that this may not be related to the nVidia driver, but just to be sure, can you post the output of your entire dmesg log:
```
