### Post by Kelly_Yamamoto on 2014-05-20
Managed to solve this with the workaround listed on this thread -- [http://ubuntuforums.org/showthread.php?t=2181558](http://ubuntuforums.org/showthread.php?t=2181558)

Specifically:[INDENT]"[SIZE=3][COLOR=#0000CD]**_Workaround_**[/COLOR][/SIZE]***(Thanks to my respectable senior Mr. chili555 for [finding this fix]("http://ubuntuforums.org/showthread.php?p=12785026#post12785026") and guiding me through it)* :**
[/INDENT]
 
[INDENT]Please open a terminal (Ctrl-Alt-T) and run the following command in it (you may copy-paste it) -
 	Code:
 	echo "options asus_nb_wmi wapf=1" | sudo tee /etc/modprobe.d/asus_nb_wmi.conf 
It will create a file *"asus_nb_wmi.conf"* in *"/etc/modprobe.d"* directory that will load the driver *"asus_nb_wmi"* with a parameter *"wapf=1"* since next boot.

Reboot and the wifi should be active now. Unfortunately, Fn+F2 still  won't work. A workaround, until it gets fixed, is given in the next post  below. I'll update this post if (and when) found a fix to that."
