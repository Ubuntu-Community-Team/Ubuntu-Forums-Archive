### Post by actionmystique on 2015-05-14
I have just found the glitch:

```
sudo sysctl -a | grep ipv6 | grep forwarding
net.ipv6.conf.all.forwarding = 1
...

sudo sysctl -p
net.ipv4.conf.default.rp_filter = 1
net.ipv4.conf.all.rp_filter = 1
net.ipv4.ip_forward = 1
net.ipv6.conf.all.forwarding = 0
net.ipv6.conf.all.accept_ra = 1
net.ipv4.conf.all.accept_redirects = 0
net.ipv6.conf.all.accept_redirects = 0

sudo sysctl -a | grep ipv6 | grep forwarding
net.ipv6.conf.all.forwarding = 0
...
```

That means that some mysterious process reenables some of sysctl.conf values during the boot, including net.ipv6.conf.all.forwarding!
Let's include **"sudo sysctl -p" at the end of /etc/rc.local** to see if we are finally able to release our role as an IPv6 router.

```
          RX bytes:16396588 (16.3 MB)  TX bytes:3600544 (3.6 MB)

```

Despite the warning in /etc/sysctl.conf:
```
#  Uncomment the next line to enable packet forwarding for IPv6
#  Enabling this option disables Stateless Address Autoconfiguration
#  based on Router Advertisements for this host
#  net.ipv6.conf.all.forwarding=1
```

**It is possible to be an IPv6 router and still get its IPv6 address through SLAAC (through RA)**:
```
# cat /proc/sys/net/ipv6/conf/all/forwarding
1
```


**The key is to enable RA reception in /etc/sysctl.conf** (_**which should be enabled by default**_) with:
```
net.ipv6.conf.all.accept_ra=1

# cat /proc/sys/net/ipv6/conf/all/accept_ra
1
```
