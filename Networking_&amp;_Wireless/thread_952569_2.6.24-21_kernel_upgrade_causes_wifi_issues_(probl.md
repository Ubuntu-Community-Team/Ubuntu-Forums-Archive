### Post by andrew.clegg on 2008-10-19
Hi folks,

We have a Dell Inspiron 1525 with a Celeron 550 and Intel 3945 wireless adapter, running Hardy Heron.

After installing the latest Hardy upgrade, we couldn't connect it to our WLAN (a Thomson SpeedTouch ADSL router). Although the interface came up okay, and NetworkManager could see all our neighbours' networks, it couldn't see our own! Even manually entering the details didn't help. After a bit of digging, it turned out that the kernel update to 2.6.24-21 was causing the problem.

Workaround:

I edited /boot/grub/menu.lst and found the entry for the previous kernel version (2.6.24-19):

```

title           Ubuntu 8.04.1, kernel 2.6.24-19-generic
root            (hd0,2)
kernel          /boot/vmlinuz-2.6.24-19-generic root=UUID=95963397-9731-491a-8197-50161d498b20 ro quiet splash
initrd          /boot/initrd.img-2.6.24-19-generic
quiet

```

I copied all of this and pasted it in above the line marked BEGIN AUTOMAGIC KERNELS LIST so that GRUB would choose it as the default kernel on startup.

**NB Don't just copy and paste it from this post because your details may vary, find the right entry in your own menu.lst .**

Obviously though, this is a bit of a temporary hack -- hopefully someone who's working on wifi stuff will read this and be able to look into it properly. I can supply more details about the system if it would help.

Hope this helps anyone else with the same issue...

Andrew.

### Post by wu-lee on 2008-11-20
I also found this problem: after Hardy's upgrade from from 2.6.24-19-generic to 2.6.24-21-generic, my wifi adaptor wouldn't connect to my own access point, but would connect to our neighbours' (unencrypted) connection.

Rebooting back to 2.6.24-19, it works again.

I noticed I'd uninstalled linux-backports-modules-2.6.24-19-generic, but linux-backports-modules-2.6.24-21-generic was installed, and had a vague memory that was related, so I uninstalled the latter.  Rebooting back into 2.6.24-21, it now worked - so my guess is there is a driver in that package which interferes with my wifi's encryption.

I hope this helps.
