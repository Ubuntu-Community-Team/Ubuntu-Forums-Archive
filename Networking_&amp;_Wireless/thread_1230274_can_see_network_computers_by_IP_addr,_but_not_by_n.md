### Post by Iowan on 2009-08-03
Something must tie DHCP and DNS together. Although the router has all the information, it may not be configured (or able) to serve that info via DNS.  DNSMasq is supposed to be able to do that.  You could build the information into the */etc/hosts* file on each computer... but it would only be accurate until DHCP handed a machine a different address.

### Post by Gustavo Narea on 2009-08-03
> **doublempi said:**
> 
Thanks, Gustavo.

Does this get entered in the router, or the hosts file, or where?

Those addresses should be synonyms to the IP addresses in your local network, so the following command pairs should be equivalent in your network:
[LIST]
[*]ping -c 1 192.168.1.2  <--> ping -c 1 dede-desktop.local
[*]ssh dede-desktop.local <--> ssh 192.168.1.2
[*]and so on...
[/LIST]

Again, it has worked out-of-the-box on all the modern routers I've used lately. I don't know if this will work on yours.

[More information about it on the Wikipedia]("http://en.wikipedia.org/wiki/.local").
