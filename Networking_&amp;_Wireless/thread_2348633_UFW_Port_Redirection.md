### Post by john423 on 2017-01-05
Hello,

I'm trying to forward port 443 that's open externally to internal port 30000 on Ubuntu 16.04 with UFW and I seem to be having some trouble. 

I've added the appropriate iptables rule: 

```
iptables -t nat -A PREROUTING -i eth0 -p tcp --dport 443 -j REDIRECT --to-port 30000
```

 and with both ports allowed on ufw (443 & 30000) I can remotely connect to port 443 and it forwards the connection to the internal port 30000 (I'm testing with nc). If I remove the "allow port 30000" from ufw, it won't work. If I disable ufw, the redirect works. Is it possible to get this working with ufw so I can close post 30000 externally and just have port 443 open from the outside? I'm a little confused why 30000 still needs to be allowed in ufw even when I'm just connecting to port 443 for it work..

Thanks in advance!

---

### Post by The Cog on 2017-01-05
By the time the packet reaches the INPUT chain, it has been redirected to port 30000, and UFW is almost certainly policing this chain. So allow port 30000 is needed.
I am less sure if UFW polices the PREROUTING chain. If not then allow port 443 might not be needed.

Use the command iptables-save to see the entire iptables configuration that ufw has set up (I don't use it myself).
