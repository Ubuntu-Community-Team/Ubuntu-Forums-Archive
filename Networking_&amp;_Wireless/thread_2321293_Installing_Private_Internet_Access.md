### Post by QDR06VV9 on 2016-04-21
Here you go.
Enter the following command:
        ```
wget https://www.privateinternetaccess.com/installer/install_ubuntu.sh
```
After that completes the download enter the next command:
        ```
sudo sh ./install_ubuntu.sh
```
Should be in your Home Directory.
This will prompt for your Sudo password, Enter the password used to login to your computer.
I_**f you are prompted to install python 2.7 or network-manager-openvpn, or any other packages, type y and press enter to confirm each**_
When the install completes, it will ask for your Login. Please enter the Username starting with p you received after signup, and press enter. 
This will generate the Configuration files, and add them to Network Manager.
To connect, Click on the Network Manager icon, click VPN connections, then choose the region that you want to connect to.
After choosing the region, you will be prompted for your password.
If you are using Wily(15.10) or Xenial(16.04)
Be sure to have seahorse installed to save you from having to enter password every time you login to PIA.
Kind Regards

### Post by TheFu on 2016-04-23
My system is a server install - so no GUI and no network manager.  Don't recall anything being hard about this.

Installed stock openvpn software, so I knew it hadn't been modified and grabbed a zip file (wget [https://www.privateinternetaccess.com/openvpn/openvpn.zip](https://www.privateinternetaccess.com/openvpn/openvpn.zip)) with the different settings - ca.crt  crl.pem  update-resolv-conf and dropped those into a temporary location first, then moved the modified-for-my-setup versions into /etc/openvpn/.  Seem to remember logging into the PIA site to generate new credentials (wasn't impressed by the complexity or length), put those into login.cf in the same directory. Grabbed a list of their available servers around the world and made a client ovpn file for each that I planned to use - UK_London.ovpn CA_Toronto.ovpn ..... Started up the openvpn pointing at the config file for Toronto. 
My ~/bin/pia-toronto.sh 
```
#!/bin/bash
cd /etc/openvpn
/usr/bin/sudo  /usr/sbin/openvpn  CA_Toronto.ovpn &
```

14.04.4 release.  Probably won't try on 16.04 for a few months. I wanted to configure all of this manually, so I knew what was happening - no magic needed.  If you do this stuff manually, when it breaks it isn't a big deal to fix.  Plus all the knowledge gained in the effort is priceless.

Plus I wanted to ensure that PPTP **was NOT** used.  Want IPSec.

It has been extremely stable and fast enough for hidef video.
