### Post by bapoumba on 2007-08-07
Sorry, I was on vacations.

If eth0 is your wireless card, in /etc/network/interfaces, it should be like:
```
iface eth0 inet dhcp
wireless-essid AMICALE 
wireless-key <your_key_here>
auto eth0
```
You need to be root to edit the file (gksudo gedit /etc/network/interfaces) and restart the network after modification.
(with DHCP, you do not need to assign IPs and such).

I'm not with free, but got my laptop connected at a friends house without any trouble ;)
One more question: is the router set up to filter on MAC adresses? If so, you'll have to enter your card in the router config panel.

### Post by bapoumba on 2007-08-08
For now, edit the /etc/network/interfaces for it to look the way I posted it (for the eth0 section). Remove:
```
address 192.168.0.16
netmask 255.255.255.0
gateway 192.168.0.254
```
and move up auto eth0 under the wireless_key line.

This is what I did to access a freebox at friends house two week-ends ago and it worked. If the router does not filter at all, do not worry about the MAC address. The wireless-key you have to enter there is the "--cut--" part of your previous post.

EDIT: to access the filr, you need to be root:
```
sudo nano /etc/network/interfaces
```
edit the file.
CTRL-O to save the file (with the same name, hit <enter>)
CTRL-X to exit nano.

```
sudo ifup eth0
```
to bring the network up.

### Post by Kobalt on 2007-08-10
The free.fr encryption settings can only be set through the admin console from the free.fr website, after logging in with the subscriber login/password (and validated by a freebox hard reboot). 
When you click on your Network-Manager icon : do you see your network in the list ? When you click on it, what happens ? Don't you get prompted to enter the encryption key ? You may need to specify the encryption method more than just WPA, possibly select either TKIP, AES/CCMP, TKIP+AES  (these are the 3 free.fr offers for WPA). 
If you want to use Network-Manager, you should know it requires an almost blank /etc/network/interfaces file. Only this part must remain in the file : > auto lo
iface lo inet loopback

### Post by siddartha on 2007-09-25
Now I come back with an answer. Sadly nobody could help and the solution was quite obvious.

Although the Gnome Network Manager is nice and quite smart for easy tasks when it comes to more complicated stuff it is the wrong answer. I had to put that down and take it off my configuration.

Also, bothering in the dark with various config files might be fun when you own the access point and can stay and tweak with its settings to get an exotic handshake no other comp could use or something like that. In my case I had no information about the configuration of the access point, just a password. That IS ENOUGH FOR WINDOWS. Sadly for Linux I was asked about my business, about why my network card is not named like in the beginner's guide and so on.

Well, the right answer for me was Kwlan. [http://www.kde-apps.org/content/show.php?content=37041](http://www.kde-apps.org/content/show.php?content=37041)

This toy is in the Ubuntu repository and it will ask to uninstall the Network Manager related packs. After that it's a pleasure working with it. Ok. It needs wpa_supplicant already installed if you want to use encription but you won't have to mess with the config files unless you want to.

From this point you can just select the network - the scan menu is filled with information. I have used before the Wifi Radar. Well, that toy had no use for me as it just shows the networks and the b/g type. Kwlan shows also the type of encription used by the access point. With this information I just went in the menu, added my password and I was online.

Thank you for reading this.
