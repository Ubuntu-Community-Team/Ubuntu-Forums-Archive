### Post by TheFu on 2023-01-09
Could be lots of things.  What have you tried?  

[https://ubuntuforums.org/showthread.php?t=2482696](https://ubuntuforums.org/showthread.php?t=2482696) suggests booting from an older kernel. You might want to watch that thread.

Testing differences over the internet isn't the best, since most of that connection is completely outside your control.  

Don't test the speed of storage, so don't transfer files, for now.  

Do testing between 2 systems on your LAN. Use only Wired Ethernet connections and a network tester like 'iperf3', to avoid other issues and overhead from other protocols.

Essentially, start with the wired network as it is. Remove switches and routers as part of your simplification steps.  Swap the cable. Try a different port in the switch/routers.  Boot into a "Try Ubuntu" environment from any ISO and test.  That test will say if it is hardware or software (usually). Removing the installed OS, but staying with Linux will get a different driver (perhaps) and if it is fast again, you'll know it is something specific to your current install with the issue. 

Don't use wifi.  Wifi even in your own home has many possible interference causes. Neighbors, cordless phones, RC toys, microwave ovens, even humidity and building materials matter.  Too hard for someone not there to troubleshoot. Stay wired.

So, assuming all these tests show slow connectivity, then you'll want to start checking drivers and isolating different parts of the network to simplify what might be causing the problem.  
1. Get your testing process down, consistent.
2. Simplify the 2 system connections.
3. Run the test.  
4. Do you know what caused the issue? If not, simplify further and 
5. Repeat.
