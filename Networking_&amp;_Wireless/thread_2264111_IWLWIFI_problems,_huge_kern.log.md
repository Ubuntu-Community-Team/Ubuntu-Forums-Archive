### Post by chili555 on 2015-02-05
You have one unhappy card/driver/firmware combination there!

Here are some things to try; I would try them in the order I will list them. Please conduct all tests with the USB wireless detached. First, flip the hardware switch or key combination so you stop the driver iwlwifi from spamming the logs and causing you to run out of disk space. Walk your laptop to the router and open a terminal and do:```
sudo apt-get update && sudo apt-get -y upgrade 
```Reboot. Detach the ethernet, turn on the wireless and do:```
tail -f /var/log/syslog
```Get out of tail with Ctrl+c. Is it better or worse?

You might also try, with the wireless turned off:```
sudo apt-get install --reinstall linux-image-`uname -r`
sudo apt-get install --reinstall linux-firmware
```Detach the ethernet, turn on the wireless and do:```
tail -f /var/log/syslog
```Get out of tail with Ctrl+c. Is it better or worse?

If no better, reboot, interrupt the boot process at the GRUB menu and select an earlier kernel version. Then, again:```
tail -f /var/log/syslog
```Better?

I would also open the laptop to find the wireless card and be certain that the card is seated properly and both antenna wires are firmly attached. Generally, there is a small door on the bottom that gives easy access to the wireless card. For example: [http://www.insidemylaptop.com/images/pavilion-dv9000-laptop/05-disconnect-wireless-antenna.jpg](http://www.insidemylaptop.com/images/pavilion-dv9000-laptop/05-disconnect-wireless-antenna.jpg)

If all of these steps are ineffective, we'll compile a later driver version.
