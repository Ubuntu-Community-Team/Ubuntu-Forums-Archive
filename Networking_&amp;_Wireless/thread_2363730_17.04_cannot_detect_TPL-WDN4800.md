### Post by chili555 on 2017-06-13
I don't think the problem is the driver ath9k. I think it is Ryzen. Google up 'Ubuntu  Ryzen' and you'll see there is a problem.

In your case, all of this:> [    0.031267] [Firmware Bug]: AMD-Vi: IOAPIC[17] not in IVRS table
    [    0.031269] [Firmware Bug]: AMD-Vi: IOAPIC[18] not in IVRS table
    [    0.031270] [Firmware Bug]: AMD-Vi: No southbridge IOAPIC found
    [    0.031270] AMD-Vi: Disabling interrupt remapping
     ...
    [    0.262665] ACPI Error: Needed [Integer/String/Buffer], found [Region] ffff95850e0efca8 (20160930/exresop-425)
    [    0.262670] ACPI Exception: AE_AML_OPERAND_TYPE, Could not execute arguments for [IOB2] (Region) (20160930/nsinit-412)
    ...
    [    4.529595] tpm_crb MSFT0101:00: can't request region for resource [mem 0xdb122000-0xdb122fff]
    ...
    [    4.724076] nommu_map_single: overflow 400d29040+1974 of device mask ffffffff...means that the PCI bus doesn't initialize the device correctly, leading to this:> [    4.724080] ath: phy0: dma_mapping_error() on RX init
    [    4.724106] ath9k 0000:24:00.0: Failed to initialize device

I don't know the answer directly but this is not the first example I've seen.

I suggest that you make sure the BIOS is fully updated. Reset the BIOS to defaults. Make sure you are running the latest available 4.10 kernel. Finally, ask the question here: [https://ubuntuforums.org/forumdisplay.php?f=331](https://ubuntuforums.org/forumdisplay.php?f=331)
