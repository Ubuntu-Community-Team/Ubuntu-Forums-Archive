### Post by YorYor on 2007-11-02
I encountered this on my Gutsy Gibbon installation with an Asus A7N8X nForce2 based motherboard that has an onboard ethernet adapter.
The MAC address was changing with every reboot, and there was no way for me to fix it in the BIOS screen. Thus, the eth name was changing all the time, and I could not assign a static IP.

After some research and Googling, I finally found the solution at [http://forums.gentoo.org/viewtopic-t-489863.html](http://forums.gentoo.org/viewtopic-t-489863.html).

1. Execute **udevinfo -a -p /sys/class/net/eth<X>** (where X = current eth that is acting "dynamic")
	and you are looking for **ATTRS{device}=="<DEVICE ID>"** (take note of the <DEVICE ID>)

2. Edit /etc/udev/rules.d/70-persistent-net.rules and change
	```
ATTRS{address}=="xx:0a:cd:13:b7:xx"
```
		into
	```
ATTRS{device}=="<DEVICE ID>"
```
		and change to the desired name (e.g. eth0)

3. Remove all other duplicate lines which assign the eth name based on the MAC address.

Reboot, and viola, the changing MAC address no longer affects the eth name.
If you need to hardcode the MAC address, then do some of the other fixes/workarounds available. For me it wasn't a concern, but do feel free to post any other links which any of you may have found related to this problem and how to further "stabilise" the MAC address.

Great Information! Just a side note - you don't need to reboot. 

/etc/init.d/udev restart 
/etc/init.d/networking restart 

is sufficient.
