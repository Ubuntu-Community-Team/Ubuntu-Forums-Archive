### Post by stefangr1 on 2009-12-02
Can you post what you get when you type:

ifconfig

, what you get if you type:

cat /etc/resolv.conf

and if you type:

cat /etc/network/interfaces

That's probably enough information to determine what the problem is.


If you don't have anything to copy this output, as you're not on a network, check whether your computer has gotten a valid ip-address, and check whether the nameserver specified in your resolv.conf is correct. If, for example the router's address is 192.168.0.1, your resolv.conf should look like:

domain lan
nameserver 192.168.0.1

Your /etc/network/interfaces should contain something like the below:

auto eth0
iface eth0 inet dhcp

If there are multiple interfaces specified, you 'could' temporarily put a # before them, though this disables these interfaces untill uncommented again.

### Post by stefangr1 on 2009-12-03
You have to change this:

auto eth0
#iface eth0 inet dhcp

to this:

auto eth0
iface eth0 inet dhcp

Also, as your ifconfig shows only two network interfaces, which are eth0 and eth1, I suggest you put a # before all the other interfaces, e.g.: #auto ath0

The simplest /etc/network/interfaces would look like this:

#internal loop
auto lo
iface lo inet loopback

#main interface
auto eth0
iface eth0 inet dhcp


and then run:

sudo /etc/init.d/networking restart

When you run ifconfig again, the part below eth0 will hopefully contain something like:

inet 192.168.0.x , 192.168.1.x, or 10.0.0.x

Let's assume it's 192.168.0.101

Now you can run:

netstat -nr
This should contain at least an entry like:

Destination        Gateway            Flags        Refs      Use   Netif Expire
default            192.168.0.1        UGSc           47        0     eth1

This is the routing table of your computer, the default gateway is (in this case) 192.168.0.1

Now you change your /etc/resolv.conf in a corresponding manner. In the above example, it should be like:

domain lan
nameserver 192.168.0.1
