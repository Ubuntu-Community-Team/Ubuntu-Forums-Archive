### Post by RavensKrag on 2010-07-20
Do you have the development headers for the kernel installed?  The package should be something like linux-headers-2.6.34 or something similar.

Also, make sure you have build-essential installed.

If that does not work, try this.
```
cd /lib/modules/`uname -r`/build
```

If that fails, and ONLY if it fails, run this command.
```
sudo ln -s /usr/src/linux-headers-`uname -r` /lib/modules/`uname -r`/build
```

Basically, what that does is it links the files you need to build stuff into the right place.  But this assumes you have the linux-headers but somehow the link was not created.  Happened to me for some reason.  Probably has something to do with using a custom kernel.



You did remember to apply the patch, right?  Just asking, sometimes I forget to do really simple things like that ^_^; The version of the broadcom drivers in the repo won't work, or at least I don't think so, because they do not have that patch.
