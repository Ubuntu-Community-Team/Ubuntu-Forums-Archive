Your wireless seems to be disabled by hardware switch. Please check if you have any physical switch to toggle it on/off. If it is some Fn+ key combination, try that as well. Check the output of command "**rfkill list**" each time you try something. We need to set both hard and soft block states to "no" for interfaces '0' (phy0) and '1' (hp-wifi).

To remove the soft-block, try this command -
```
sudo rfkill unblock all
```

If these don't help removing the hard blocks, check your BIOS to make sure wireless is enabled there. Some laptops also have this setting that automatically disables the wireless as soon as a cable connection is detected. If yours has such any setting, make sure it is turned off, and it would be better to keep the cable unplugged while troubleshooting the wireless hard/soft blocks.

If everything seems okay but the hardware switch still doesn't make any difference, then a BIOS reset or a full power drain may be required. DO NOT reset the BIOS if it is a UEFI based installation, else the boot configuration will also get erased and you may need to fix booting afterwards.

For a full power drain trick, remove the AC adapter and battery, press the power button a few times to make sure any remnant power is also drained. Leave the laptop in this state for a few minutes (usually 30 minutes should by more than sufficient, but I know a few Dell models that needed to be left overnight !).

However, if you have a Live CD/DVD/USB and the wireless does not appear to be hard or soft blocked in the live session, then the above tricks of BIOS reset or Poser drain are not required and there may be something wrong with the installation itself, so a Live session would be a good way to test whether it is an OS problem or hardware/firmware.
