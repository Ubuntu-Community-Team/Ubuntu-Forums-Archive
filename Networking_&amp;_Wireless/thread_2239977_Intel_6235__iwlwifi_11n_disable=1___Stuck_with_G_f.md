### Post by varunendra on 2014-08-17
Regarding these two points -> **dave-6 said:**
> [LIST]
[*]Netgear WNDR3700, reliably getting 50 mbit on 5 Ghz.
[*]Coming from Ubuntu Gnome 13.10, wireless worked perfectly out of the box.
[/LIST]
**1)** Try 2.4 GHz band at 20 MHz fixed bandwidth. 20/40 MHz auto mode is not so good with 2.4 GHz band, not sure about how good it is with 5 GHz band. This has been especially observed with Intel cards by our expert chili555 (I hope I'm not misunderstanding/miscarrying his statements).

**2)** The iwlwifi driver in latest kernels has more than two modes for '11n_disable' parameter. Instead of just 0 and 1, now it also has 2,4 and 8 as acceptable values. Check what they are supposed to do with -
```
modinfo iwlwifi | grep parm
```
..and I think you should try value '**8**' instead of '1'. It seems to '**_force enable_**' Tx aggregation on the card which is supposed to help with some particular cards (we are not yet sure which ones). See the comment #3 on this bug report : [https://bugs.launchpad.net/bugs/1341068](https://bugs.launchpad.net/bugs/1341068)


**PS:**
If you wish to let us see if something other than 11n_disable can be tried with your setup, please show us a detailed report following this post : [http://ubuntuforums.org/showpost.php?p=13024222](http://ubuntuforums.org/showpost.php?p=13024222)
