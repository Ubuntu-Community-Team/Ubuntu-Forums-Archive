### Post by Ayuthia on 2008-11-23
It looks like it is trying to use the b43 driver.  Please try the following:
```
sudo modprobe -r b43 b44 ssb ndiswrapper
sudo modprobe ndiswrapper
sudo ifconfig wlan0 up
sudo iwlist scan
```
If it does not work, please post the results of:
```
lsmod|grep -e b43 -e ssb -3 ndiswrapper
dmesg|grep ndiswrapper
```

Since it is trying to use the b43 driver, it generally means that the b43 driver is still active or else the ndiswrapper module was unable to load for some reason.  The first set of commands will try to unload all the possible wireless drivers for your card and then load ndiswrapper.  The second set will check to see if error messages are being created when ndiswrapper is being loaded.

Also, have you tried using the b43 driver?  I know that some of the 4306 rev 03 cards work with it.

### Post by Ayuthia on 2008-11-23
> **megajosh2 said:**
> Thank you both so much, that got it working!

If that is the case, you will need to make sure that b43, bcm43xx, and ssb are blacklisted.  You will also want to check and see if ndiswrapper is added to /etc/modules.  Here is a way to check:
```
cat /etc/modprobe.d/blacklist | grep -e b43 -e ssb -e bcm43xx -e ndiswrapper
cat /etc/modules | grep -e b43 -e ssb -e bcm43xx -e ndiswrapper
```
The blacklist file should produce:
```
blacklist b43
blacklist ssb
blacklist bcm43xx[code]
and should not contain:
[code]blacklist ndiswrapper
```

/etc/modules should list:
```
ndiswrapper
```
and should not have:
```
b43
ssb
bcm43xx
```

If those files are not correct, you will have to run those previous commands again to get your wireless to work.  If you need help fixing those files, please post the results of the commands in this post and we can provide the commands to update the files.

Glad to see it working!
