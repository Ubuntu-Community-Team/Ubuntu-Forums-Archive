### Post by TBABill on 2011-04-27
I wouldn't fear a virus. What I would look at if it were mine is if the machine received any updates, particularly any kernel updates. It's easy to test by restarting and booting into an older kernel (hopefully you keep at least one older kernel listed in grub2 just for this reason). It could also be something else, but that'd be my first guess assuming power is on the device, it connects fine in XP, etc....cover the very basics then try an older kernel. If you don't have one, and can plug into ethernet, download one via Synaptic and boot into it. If that's not showing any signs of helping, removal of the kernel is just as easy.

```
### Post by vagrale13 on 2011-05-02
I think i found a solution... i test it a couple of boot, and seems ok!
I generate a *hex-key* of my wireless, and add it in  */etc/network/interfaces *

The file is like this
```
auto lo
iface lo inet loopback

wireless-psk your_hex_key_generate
wireless-essid your_essid
