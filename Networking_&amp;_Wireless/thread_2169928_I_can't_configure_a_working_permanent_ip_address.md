On Ubuntu Server Edition you should only need configure the interfaces via the */etc/network/interfaces* as you have done.  And configure the */etc/resolv.conf* file for DNS servers.

The *auto* as shown in the following brings up the interfaces on boot up
```

[COLOR="#FF0000"]auto[/COLOR] eth0

[COLOR="#FF0000"]auto[/COLOR] eth1

[COLOR="#FF0000"]auto[/COLOR] eth2

```

Using DHCP (dhclient) should never be needed on a server that is truly configured for a static IP address.  A static IP address is NOT an address that is configured via DCHP reservation.

Could this be a GUI desktop install with Network Manager and resolvconf?  All of my servers Ubuntu 12.04 and Debian are configured with a static IP address and DNS just as I have described.  Or maybe there is a problem with the OS.  What do the system logs show?  What kind of errors do you get?

Sure I can.  I looks to me like this host is not an Ubuntu Server Edition distro or you have installed a GUI desktop environment.  Both Avahi and Network-Manager are part of the Desktop distro of Ubuntu.  Both Avahi and Network-Manager are sending errors.  Most likely from a misconfiguration.

Let us know exactly what GUI desktop you have installed.  What have you configured or deleted from the Network-Manager setup.  Have you modified any other parts of the boot sequence?

With my desktop (workstation) hosts I remove both the Network-Manager and the resolvconf packages before I start to statically configure that host for a specific IP address in /etc/network/interfaces and configure DNS resolution in /etc/resolv.conf.  I also map the IP address to the hostname in the /etc/hosts file.  

On my last desktop install I left the resolvconf package as it also includes DNSmasq and that causes the localhost to cache recent DNS requests. 

On my laptops I use Network-Manager for immutable (non-changing) wired IP addresses.  The wireless uses Network-Manager for DHCP acquired IP addresses.
