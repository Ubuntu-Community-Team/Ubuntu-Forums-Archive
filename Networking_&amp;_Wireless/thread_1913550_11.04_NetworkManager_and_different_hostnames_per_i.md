### Post by phormion on 2012-01-22
Hi, 

I have Ubuntu 11.04 at work with 2 NICs - each is connected to a different internal network. I don't want to use the same hostname for both connections, because only one of the subnets is accessible from my VPN connection, so I tried using separate hostnames in dhclient.conf. So in /etc/dhcp/dhclient.conf I have: 

```

interface "eth4" {
   send host-name "host2";
}

interface "eth5" {
   send host-name "host1";
}

```

However, I still get host1 assigned alternatively to eth4 or eth5. In /var/run/nm-dhclient-eth4.conf, it seems NetworkManager decided on his own account to "make some changes": 

```

interface "eth4" {
}
interface "eth5" {
}
send host-name "host1"; # added by NetworkManager

```

And then in nm-dhclient-eth5.conf: 

```

interface "eth4" {
send host-name "host2";
}
interface "eth5" {
send host-name "host1";
}

```

So the result is both interfaces compete again for the same hostname. Now, I guess I could swap the directives around, but that doesn't feel like more than a kludge. Any ideas on how to configure this properly? It kinda sucks that the NM GUI doesn't have anything about hostname configuration either...
