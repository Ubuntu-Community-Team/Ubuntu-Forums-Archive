### Post by TheFu on 2015-10-31
Try the easy stuff first.
Looks like the port, cable are bad.  Try a different port on the switch/router. Try a different cable.

If those aren't it, boot off a live CD/Flash and see if that works - try 15.10, 14.04, and a few other distros like TinyCore. Keep track of which work and which don't. Hopefully, none work and we can close  this as  a  hardware issue.

The last issue could be the driver/driver-settings. Check the forums for the specific driver needed for that chip. I recall seeing more than a few similar issues here with [solved] status.  Vaguely recall they had to force a different driver.

### Post by praseodym on 2015-11-01
Change the driver, download these files and save them in "Downloads":

[http://de.archive.ubuntu.com/ubuntu/pool/main/d/dkms/dkms_2.2.0.3-1.1ubuntu5.14.04.5_all.deb](http://de.archive.ubuntu.com/ubuntu/pool/main/d/dkms/dkms_2.2.0.3-1.1ubuntu5.14.04.5_all.deb)

[http://de.archive.ubuntu.com/ubuntu/pool/universe/r/r8168/r8168-dkms_8.040.00-1_all.deb](http://de.archive.ubuntu.com/ubuntu/pool/universe/r/r8168/r8168-dkms_8.040.00-1_all.deb)

Installation:
```

sudo dpkg -i ~/Downloads/*.deb
echo "blacklist r8169" | sudo tee -a /etc/modprobe.d/blacklist.conf
```
Reboot.
