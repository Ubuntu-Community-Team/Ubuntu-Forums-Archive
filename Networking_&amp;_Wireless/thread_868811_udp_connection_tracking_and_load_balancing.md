I've configured a kunbutu PC as a router. It has one LAN interface and three uplinks to the Internet. The load balancing gives each of the uplinks an equal weight.

The goal was to have a UDP stream spread across the uplinks. The UPD stream is generated by a PC on the LAN. The system receiving the UDP stream does not care that the source addresses are different.

It appears the connection tracking kernel module (nf_conntrack) considers the UDP stream to be a connection and treats all the packets as "related" to the first UDP packet. This means they all get sent across the same uplink.

I'd like all the packets to be treated as "new" so they get load balanced across the uplinks.

Is there a way to configure iptables or the nf_conntrack to do this? Or, am I going to have to modify nf_conntrack? Or, is there a better solution.
