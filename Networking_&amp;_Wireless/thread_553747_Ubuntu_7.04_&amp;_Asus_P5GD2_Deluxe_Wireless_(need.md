### Post by noob12 on 2007-09-19
I have to get some sleep, so I'm off.

Here are some further steps for you.  

First determine if you can ping that address.

If you can successfully ping that address, then your problem is probably flaky DNS service.  Basically what is happening is that your router serves as a DNS proxy.  It has one or more actual DNS server addresses which you either configured manually or that it obtained via DHCP from your ISP.  These servers are not always responsive, or the DNS proxy in your router may sometimes fail.

You can get around that by configuring more reliable DNS servers in your router or just going to the DNS servers directly rather than using the proxy on your router.  The thread here provides a couple of ways to do that: [http://ubuntuforums.org/showthread.php?t=543659](http://ubuntuforums.org/showthread.php?t=543659)   Try the first set of instructions there.


If you can't successfully ping that address, then we need to check your routes.  In that case, you should post the output of ```
route -n
```
