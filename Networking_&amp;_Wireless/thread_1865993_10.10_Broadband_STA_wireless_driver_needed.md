### Post by TBABill on 2011-11-03
I tried to go over and over all the posts and understand where in the process you are. Looks like you installed the wl module, but it's not active. The b43 got installed, removed, installed, etc. Last lsmod showed the b43 was listed. Also, one of the dmsg outputs showed b43 firmware failed to install and that's most likely because you have a BCM4312 LP-PHY (low power) and it requires different firmware. What you may need to do is: ```
sudo apt-get install --reinstall bcmwl-kernel-source
``` and you may also need to ```
sudo apt-get install firmware-b43-lpphy-installer 
``` That last one should be the right firmware for your device.
 
Once that's done, you can unload the b43 module if it's still showing in lsmod by doing ```
sudo modprobe -r b43
``` and just in case then ```
sudo modprobe wl
``` in case the wl module wasn't already loaded. After this point, if you installed the driver correctly you should not need a restart for it to begin working. It should work after about a minute or so. 
 
To make the b43 not load again, just ```
gksudo gedit /etc/modprobe.d/blacklist.conf
``` and if you don't see this statement in the file, add it to the end: ```
blacklist b43
```

The module dell_laptop is sometimes a little bit buggy, blacklist it, and reboot:

```
echo "blacklist dell_laptop" | sudo tee /etc/modprobe.d/blacklist-dell_laptop.conf
```
If there are problems, just remove the file:

```
sudo rm /etc/modprobe.d/blacklist-dell_laptop.conf
```
