### Post by Ulysses_ on 2009-11-19
Someone suggested what he called ''dns pooling'' as a countermeasure for those man-in-the-middle attacks that are based on DNS poisoning.  His suggestion was to query several worldwide DNS servers and compare the results.  If there's any difference, alert the user.    Of course that would produce false alarms for legitimate IP changes propagating in DNS networks.  But it's better than nothing.   Are there any tools to automatically compare the results from several DNS servers, and alert for any difference?
