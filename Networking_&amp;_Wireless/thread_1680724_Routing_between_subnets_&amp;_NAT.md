### Post by oohshiny on 2011-02-03
I'm working on a virtual network config with multiple subnets and for a router, I have a VM set up with 3 virtual NICs:

eth0: DHCP to physical network
eth1: 10.10.4.1/24
eth2: 10.10.5.1/24

I then have VMs on the 10.10.4.0 and 10.10.5.0 networks. I attempted to set up a NAT on eth0 (to hide the other subnets from the physical network) but have accidentally ended up hiding the subnets from each other - they can ping but connections appear to come from 10.10.4.1 or 10.10.5.1 when crossing the subnets. My iptables config (from iptables-save) is:

```

*nat
:PREROUTING ACCEPT [2214:428370]
:POSTROUTING ACCEPT [0:0]
:OUTPUT ACCEPT [91:6048]
-A PREROUTING -i eth0 -p tcp -m tcp --dport 3389 -j DNAT --to-destination 10.10.4.100 
-A POSTROUTING -j MASQUERADE 
COMMIT
*filter
:INPUT ACCEPT [30645:6316274]
:FORWARD ACCEPT [218:16383]
:OUTPUT ACCEPT [11581:1072457]
-A FORWARD -d 10.10.4.100 -i eth0 -p tcp -m tcp --dport 3389 -j ACCEPT 
-A FORWARD -s 10.10.4.0/255.255.255.0 -i eth0 -o eth1 -m conntrack --ctstate NEW -j ACCEPT 
-A FORWARD -s 10.10.5.0/255.255.255.0 -i eth0 -o eth2 -m conntrack --ctstate NEW -j ACCEPT 
-A FORWARD -m conntrack --ctstate RELATED,ESTABLISHED -j ACCEPT 
COMMIT

```What do I need to get the two interfaces (eth1 and eth2) talking to each other without NAT, but still have eth0 behind NAT?


[Edit] Ok, so I ended up dropping the "FORWARD" rules and using "POSTROUTING" instead:

```

*nat
:PREROUTING ACCEPT [1714:276473]
:POSTROUTING ACCEPT [348:22988]
:OUTPUT ACCEPT [85:6119]
-A PREROUTING -i eth0 -p tcp -m tcp --dport 3389 -j DNAT --to-destination 10.10.4.100 
-A POSTROUTING -s 10.10.4.0/255.255.255.0 -o eth0 -j MASQUERADE 
-A POSTROUTING -s 10.10.5.0/255.255.255.0 -o eth0 -j MASQUERADE 
COMMIT
# Completed on Thu Feb  3 17:18:23 2011
# Generated by iptables-save v1.3.8 on Thu Feb  3 17:18:23 2011
*filter
:INPUT ACCEPT [26508:9261698]
:FORWARD ACCEPT [11161:6266924]
:OUTPUT ACCEPT [13063:1296970]
-A FORWARD -d 10.10.4.100 -i eth0 -p tcp -m tcp --dport 3389 -j ACCEPT 
COMMIT

```

and this work properly - connections now show up from their real source IPs.
