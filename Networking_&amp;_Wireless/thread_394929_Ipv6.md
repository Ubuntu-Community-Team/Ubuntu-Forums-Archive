### Post by nautilus on 2007-04-15
IPv6 is the *current* Internet Standard Protocol.

By definition, a system which doesn't support it simply doesn't meet up to Internet Standards.

Linux supports nearly the entire specification for IPv6, save a couple obscure addendums, and WinXP supports a fair amount as well.

Linux has had IPv6 support since Kernel 2.2, which was popular a few years ago, at least.

IP (internet protocol) version 6 replaces IP version 4, but requires 'help' in the form of IPv6 capable operating systems, routers, software, and naturally hosts, to connect to.

Migration/transition mechanisms are available if you poke around for them.

If anything, using IPv6 will improve performance rather than harm it.

It's enabled by default for a reason, and disabling it simply won't improve overall performance.  If it does, you need to find a new DNS server.</rant>
