Hi everyone. This one has nothing to do with Ubuntu specifically. I managed to end up in a situation where I must take care of a LAN which forms part of a larger WAN and now have a few sticky issues. The network is mostly for company use, but it has been decided to to give certain people access to Internet only, via Wifi. Since the area is fairly large, we ran one trunk line and connected several wifi routers on this line, to provide coverage. The idea was also that since the wifi now sits only on one trunk line (which is finally connected to the main switch), it would be easier to controll user access. Until now access was done with a WPA key, but we found this leaked out too easily and soon there would be too many users on the wifi, affecting the bandwidth of the rest of the network.

What I am looking for is a method to control the access and bandwith of this line at a single point. I think this is probable a firewall of some sort or a wifi hotspot such as what is used in hotels. Unfortunately I have no experience with this sort of thing, so if anyone has some ideas of how to do, much appreciated.
