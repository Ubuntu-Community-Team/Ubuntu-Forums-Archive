### Post by BloGTK on 2008-06-08
> **Screwtape said:**
> Well, none of this is very helpful. Seriously, is this an exploit, and can it be prevented in the future, and is there a way to detect spyware?

Maybe, yes, and there's no need on Linux.

You should make sure that your copy of Firefox is up to date, as older versions do have security holes.

There's an extension called "Noscript" for Firefox. If you're going to sites that you don't trust, then you should use it to prevent that site from running any scripting at all.

There is no real "spyware" for Linux as Linux has a better security model - provided that you have been keeping up with updates to the system you should be fine. Ubuntu's update manager notifies you of any updates - make sure you install them.

Chances are the malicious site used a simple JavaScript exploit and little else. If you want to be safe, use Firefox to export your old bookmarks, then trash your Firefox profile directory. At a terminal you can use the command "rm -rf ~/.firefox" to start from scratch. You will, however, lose settings and saved passwords, but you'll also get rid of any potential bad code from an exploit. Then you can have Firefox create a new profile and import your bookmarks again.

Oh, and leeching off of someone else's connection is seriously bad karma, and it could be illegal. At the very least, you should ask nicely first...
