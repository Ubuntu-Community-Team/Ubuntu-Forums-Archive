### Post by chili555 on 2010-04-05
I still think that the interaction between the native driver and ndiswrapper is the underlying issue. We have to get tougher. Please do:```
sudo gedit /etc/modules
```If ndiswrapper is in there, comment it like this:```
[COLOR="Red"]#[/COLOR]ndiswrapper
[COLOR="Red"]#experiment05APR2010[/COLOR]
```The comment hashmarks # will cause the system to ignore all that follows it on the same line. The explanation helps us retrace our steps. Proofread, save and close gedit. 

Next, do:```
sudo gedit /etc/modprobe.d/blacklist.conf
```Add two new lines:```
blacklist ndiswrapper
#experiment05APR2010
```Now, while the text editor is still open, open a new terminal and do:```
cat /etc/modprobe.d/blacklist
```Please transfer any listings in blacklist ( the terminal window) to blacklist.conf (the gedit text editor). Proofread carefully, save and close gedit. Close the second terminal. 

Now do:```
sudo rm /etc/modprobe.d/blacklist
```Now, we have one consolidated blacklist file, not two confusing ones.

Now reboot and, if the wireless is not running correctly, let me see dmesg in a pastebin. Thanks.
