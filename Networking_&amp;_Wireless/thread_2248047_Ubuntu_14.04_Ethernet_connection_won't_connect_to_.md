### Post by TheFu on 2014-10-12
If you cannot ping the router, the issue is between the computer and the router. Right?
Everything after that won't work.

I'd suggest that you check the router connection, cables, and verify the router settings. If the PC isn't using the correct network settings (which seems unlikely with DHCP used), we are in guessing mode.

The system is using DHCP, correct?  That is how it got the .4 IP.   The .1 IP for the router is fairly common, so that isn't the problem..  The connection works well enough that DHCP returns the network settings (route, IP, netmask) from the router ... Those look reasonable to me.

Time to check the firewalls and routing ... 

route
sudo iptables -L
