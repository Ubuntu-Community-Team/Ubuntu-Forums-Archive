### Post by nixfaced on 2007-05-26
The only way your dhcpd would give your eth1 a lease is if you've got a dhcp client requesting leases on that interface.  You can verify whether you have an active instance of dhclient by running 'pidof dhclient3'.  If that command returns a pid number, that's the problem -- dhclient is requesting an ip, and overriding your manual settings.  

In any event, I'd recommend configuring eth1 via /etc/network/interfaces, rather than issuing seperate commands manually or through a script.    Check out 'man interfaces' for more info.
