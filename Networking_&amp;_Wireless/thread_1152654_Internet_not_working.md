### Post by t0mppa on 2009-05-08
What you could check up is running 'route' on a terminal and see if there's an ip route equivalent to your nameserver specified in resolv.conf, something along the lines of this: ```
$ route
Kernel IP routing table
Destination     Gateway         Genmask         Flags Metric Ref    Use Iface
...
default         192.168.0.254   0.0.0.0         UG    0      0        0 wlan0
$ cat /etc/resolv.conf
# Generated by NetworkManager
nameserver 192.168.0.254
```

If that's ok, then it should be correctly configured. A simple way to get past this is using the [OpenDNS]("http://www.opendns.com") servers, by adding them to your resolv.conf file. But of course it'd be nice to know why your local DNS server isn't working, since it shouldn't be affected at all by the OS change.
