### Post by Hadaka on 2013-04-05
Hi, glad that helped, hope it holds.


sudo modprobe -r rt2800pci <-> removes module rt2800pci
sudo modprobe rt2800pci nohwcrypt=1 <-> inserts module rt2800pci with "no hardware encryption"

[nohwcrypt=1/off nohwcrypt=0/on] or [nohwcrypt= y/n] as it is bool.
to view:
```
modinfo rt2800pci
```
have fun !

### Post by SeijiSensei on 2013-04-10
Just so I can eliminate the one possible problem I can think of, you are not running BitTorrent are you?  If so, try reducing the number of *simultaneous connections* to a small number like 20 and see if that fixes the problem.  BT over residential routers can cause random disconnections if the number of simultaneous connections to peers gets too large for the router's memory to manage.  If the problem disappears, you can start increasing the limit until the router falls over again.
