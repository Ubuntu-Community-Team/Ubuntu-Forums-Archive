### Post by hiec2 on 2018-10-24
Hello! 

Recently, I did a fresh install of Lubuntu 18.04.1. The USB wifi adapter did not work initially, so I established a wired connection to the internet and installed a wifi driver via the "Additional drivers" pane. All was well, or so it seemed.

Today I unplugged the wired connection and DNS resolution stopped working. If I ping google.com I get
```
$ ping www.google.com
ping: www.google.com: Temporary failure in name resolution

```

If I then reconnect the wired connection I can successfully ping google.com again.

The wired connection has to go because the intended destination for this machine is nowhere near my router, so I need to fix DNS resolution for the wireless connection. Any help would be greatly appreciated!

**Update:**
I noticed in "systemd-resolve --status" that the current scope of my wireless connection is "None" whereas the current scope of my wired connection is "DNS". In addition the wired connection has DNS servers configured and knows the domain of my ISP. This looks wrong, but what exactly is the problem?

[B]Update2:
[/B]While I was collecting the infos below (specifically after displaying the status of systemd-resolve and systemd-resolved.service) /etc/resolv.conf changed. The line "search myisp.xyz" was added. Weird...

**Update3:**
If I plug in the wired connection, the line "search myisp.xyz" is added to my /etc/resolv.conf. If I unplug it again said line disappears. myisp.xyz is the domain of my internet service provider.

[B]Update4:
[/B]:lolflag: As is so often the case the problem was located right in front of the screen. I configured a static IP for my wireless connection, but did not configure DNS servers for it. Oh well, I will leave this here for future generations to laugh at.
