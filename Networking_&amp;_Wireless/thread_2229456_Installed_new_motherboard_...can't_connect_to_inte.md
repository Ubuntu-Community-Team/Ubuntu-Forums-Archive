### Post by TheFu on 2014-06-13
If the networking is on the motherboard, then the old network adapter is probably stuck holding the eth0 device.  That is controlled inside the /etc/udev/rules.d/70-persistent-net.rules file.  It is safe to delete it, then reboot. It will be recreated  and the current NIC will be assigned to eth0 - all should be good.

If not, [http://blog.jdpfu.com/2013/03/01/linux-troubleshooting-101-networking](http://blog.jdpfu.com/2013/03/01/linux-troubleshooting-101-networking) explains how to determine where the actually problem lies, so you can get more targeted help.
