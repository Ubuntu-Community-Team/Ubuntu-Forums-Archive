### Post by varunendra on 2014-07-06
No known culprits in the outputs. Please try these -

**1)** Try blacklisting 'toshiba_acpi' module -
```
echo "blacklist toshiba_acpi" | sudo tee -a /etc/modprobe.d/blacklist.conf
```
Reboot and try the wifi switch (Fn+F8). This is purely guess work, so may break some functionality (e.g., other key combos) instead of fixing something. In that case, remove it from blacklist with -
```
sudo sed -i '/blacklist toshiba_acpi/ d' /etc/modprobe.d/blacklist.conf
```


**2)** If the blacklisting above doesn't help, try an old trick that *sometimes* works in stuck firmware cases - remove battery and ac power connections from the laptop, and press & hold power button for 4-6 seconds so that all the power is drained from the laptop circuitry. In fact, it is also recommended to remove the CMOS battery that powers the system clock and BIOS memory, but that is usually located in an area that can not be accessed easily, so leave it if it is not easily accessible. Re connect the battery/ac power after 10 minutes and check if the Fn+F8 combo works now. If not, try the same again and leave the laptop powerless overnight.

If everything else is okay and it is just the card's firmware stuck in a 'confused' state, this should successfully reset it. If not, then probably we need some acpi related [boot option]("https://help.ubuntu.com/community/BootOptions#Common_Kernel_Options") that I don't know or can't guess right now.

### Post by varunendra on 2014-07-07
> **goethesf said:**
> I had an idea -- I remember I have a **Netgear WNA100M USB WiFi** nano sitting around ... could this be helpful in testing?

It shouldn't have the hardblocked issue, but it seems it uses the rtl8192cu driver (see [here]("https://wikidevi.com/wiki/Netgear_WNA1000M")), which is a bit buggy in kernel 3.13. So you may need to use a patched driver mentioned here : [http://ubuntuforums.org/showpost.php?p=13026049](http://ubuntuforums.org/showpost.php?p=13026049)

For the internal one, you don't need to drain the battery (you can't, it always keeps reserved power). Just disconnect it from the laptop, as well as the AC power adapter so that the laptop doesn't get power from anywhere. THEN press the power on button on the laptop and hold it for a few seconds (4-10 sec) so that any remaining power on the motherboard is drained.

Some blog posts mention that the posters' laptop(s) (those were Dell models by the way) needed to be left overnight to completely reset. So the suggestion to leave the battery and AC adapter disconnected overnight came from that experience of others. Most often, it gets reset within half an hour or immediately after the power drain. So leaving overnight is just a desperate attempt, with little hope.

Hi .... bit of an update here for anyone who crosses this path. I went to the Toshiba support site and ordered a USB recovery disk which takes the laptop to the orig. config (Windows). 

Here's what I did:

[LIST]
[*]I booted into the BIOS and turned on the WiFi (I had previously turned it off to get the Wifi Nano USB to work)
[*]I removed the Nano USB
[*]I rebooted and hit Fn+12 to boot off the Toshiba recovery USB stick (45min) / installed and booted up in Windows
[*]The internal wireless chip was showing disabled in Windows (turned off)
[*]I typed Fn+8 and turned WiFi back on in Windows
[*]This worked and Windows displayed a pop-up "Wireless networks avail" -- I was able to see my home network (didn't login)
[*]I rebooted the laptop hitting Fn+12 to boot off my Ubuntu USB install drive (total time in Windows was less than 5min)
[*]Ubuntu installed booted fine (expected) and the installer found the embedded chipset (problem fixed!)
[/LIST]

So there was something going on in Windows which software locked the embedded WiFi chip. Once I unlocked it in Windows, I'm free and clear in Ubuntu.

The USB recovery disk cost me $34 and to be honest, it's worth it to not have my USB port dedicated to WiFi. I'm assuming this is lower power consumption as well.
