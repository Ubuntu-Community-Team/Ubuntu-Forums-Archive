### Post by varunendra on 2013-07-15
> **pingkong said:**
> 
Broadcom BCM43224 **[14e4:[COLOR="#FF0000"]0576[/COLOR]]** (I have tried **wl** and **brcmsmac** drivers)

Apparently those drivers are the only ones that this device has been officially tested with : [http://wireless.kernel.org/en/users/Drivers/b43#Supported_devices](http://wireless.kernel.org/en/users/Drivers/b43#Supported_devices)

We *may* try the b43 driver as well, but I'm not much hopeful with that. But since it is not going to hurt, let's try it anyway. Please install the linux-firmware-nonfree package to get the firmware that b43 driver requires. If you can somehow connect to internet, then :
```
sudo apt-get install linux-firmware-nonfree
```
If you can't connect to internet, download the above package from here : [http://packages.ubuntu.com/precise/all/linux-firmware-nonfree/download](http://packages.ubuntu.com/precise/all/linux-firmware-nonfree/download)
Copy it to your Ubuntu machine and double-click to install. Then do -
```
sudo modprobe -rfv wl
sudo modprobe -rfv brcmsmac
sudo modprobe -v b43
```
The first two commands will only be needed if the "wl" and/or "brcmsmac" driver(s) are loaded. The third one will load the b43 driver. If it makes any positive difference, let us know.

Otherwise, please follow the "Wireless Script" link in my signature and post back the report generated by the script suggested in the linked post.

The first possible problem is that you are using WPA/WPA2 mixed mode in the router/access-point. It is known as a problematic encryption setting. Please change it to pure WPA2-PSK (AES). No mixed mode, no TKIP.

### Post by varunendra on 2013-07-16
There are not many options available with this card/driver, but there are still a couple of things that you may try -

[INDENT]* Your current signal seems to have too much noise in it (-92 dBm, anything above -65 is not okay). Try changing the channel from 7 to 1 or 11

* If the above doesn't work, try fixing the speed to a lower value -```
sudo iwconfig eth1 rate 48M
```(you can also try other supported values like 36 or 54M)

* Completely purge the proprietary driver and try the brcmsmac ```
sudo apt-get purge bcmwl-kernel-source
``` (Please note that as soon as you purge the wl driver, your interface name will be changed back to "wlan0" or something other than eth1. So if you wish to try the above fix speed thing afterwards, you'll have to use that name then.)[/INDENT]

Keep the encryption to the current (WPA2-AES only) mode anyway. The mixed mode is almost always an additional problem.
