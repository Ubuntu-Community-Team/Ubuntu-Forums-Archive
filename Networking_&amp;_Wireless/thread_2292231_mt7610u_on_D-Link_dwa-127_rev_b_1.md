### Post by chili555 on 2015-08-26
First, I noticed this: [https://github.com/porjo/mt7601](https://github.com/porjo/mt7601)>  If you have Linux 4.2 or later the new driver is already part of the kernel (it's called mt7601u).Since you are running 4.2.xx, check to be sure the module isn't already present.```
modinfo mt7601u
```If it is present, load it and check the log for messages:```
sudo modprobe mt7601u
dmesg | grep mt76
```A possible problem may be firmware. You may be able to get the firmware here: [https://github.com/porjo/mt7601/tree/master/src/mcu/bin](https://github.com/porjo/mt7601/tree/master/src/mcu/bin)

As you have already discovered, if the usb.id of 2001:3d04 isn't included in modinfo, the driver is useless. 

Post back if you need additional guidance.

### Post by hegex2 on 2015-10-08
Hi all!

I'll write this because many google results about this dongle points here. 
So the problem with the mt7601u is there is a driver supported by the kernel (since 4.2.0), but there is no firmware installed yet. [[https://github.com/kuba-moo/mt7601u/issues/38 ]]("https://github.com/kuba-moo/mt7601u/issues/38")

To get this dongle work right you need to compile the firmware and copy it to /lib/firmware directory. 

Shortly
sudo apt-get install git
sudo apt-get install --reinstall linux-headers-generic build-essential
git clone [https://github.com/porjo/mt7601](https://github.com/porjo/mt7601)
cd mt7601/src/
make
sudo mv mcu/bin/MT7601.bin /lib/firmware/mt7601u.bin

And voila! :)
