### Post by varunendra on 2012-11-05
> **Dakster01 said:**
> What could cause dropped packets? It can be caused by many reasons - including packet collisions, network congestion, bad physical connection, low buffer memory etc. But unless it is very large in numbers, it can be safely ignored.

The amount in your output is ignorable (103 against 9799 received ones) unless it is increasing rapidly. So when the connectivity apparently losts, keep it monitoring for a while to see if it stays low or increases rapidly afterwards.

> **Dakster01 said:**
> but I seem to remember it was an ubuntu approved version of [COLOR=Red]**VMware server**[/COLOR]. I might actually try removing that as I don't really use it anymore. Again, it may not be related, but VMware server is an obsolete product now. Get rid of it.
 
> **Dakster01 said:**
> I am unable to connect from any remote machine to the server (which includes samba shares, etc) and [COLOR=DarkRed]**network access from the server console iteself**[/COLOR] (and internet browing) **[COLOR=DarkRed]is very very slow[/COLOR]**.Which network and what kind of access ? LAN or WAN or both? If it is (or includes) the same network on which the 'remote machines' are, then it obviously means the connectivity is not completely lost, it just gets so slow that services like samba fail to work properly. To elaborate this - you are physically and logically connected, but the traffic comes to 'almost' a standstill.

If I'm understanding above correctly, then you should also check if something on the network is generating abnormal traffic, causing it to get jammed. It can be easily monitored if the traffic goes through a firewall. Another way to check is to disconnect all nodes, then connect them one-by-one and check the connectivity (& traffic) to the server.

While the problem may actually be internal, all of the above suggestions are my attempts to pin-point the culprit, since the driver for your ethernet card is already the correct one and is not known for such problems as far as I could search.
