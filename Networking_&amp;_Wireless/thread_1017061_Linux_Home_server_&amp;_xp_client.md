First I want to create a home server intranet thing where I want the sever to do the following:

  - I want to be backup storage server
  - Be accessable from a windows xp client
  - I want the internet connection to go through the server and be controled from the server(I think thats called a internet Gateway?)
  - I want to be able to have good security
  - Be a web application server with appache, php, and mysql.
  - Have a file sharing server
  - And I will a wired/wireless internet router!

 I would like most of the this to be accessable through the xp client. 
That's an ambitious project :)

But if i were you i would change the project a bit.
You want both to offer wifi and put php scripts online and use it as a backup storage machine, and "most of it to be accessible by a xp client". :|
That's tough!
One successful break in from some unknown intruder, and you risk losing everything.

In an ideal situation you would have 3 machines for that.
1) firewall
2) web- and file-server
3) backup-server

But if you only have one machine for this, i would still recommend to make also backups on an external hard disk, which is not always connected to that machine.

If you can't afford to buy another machine, it makes sense to spend some time looking at things like XEN, OpenVZ and the like.
