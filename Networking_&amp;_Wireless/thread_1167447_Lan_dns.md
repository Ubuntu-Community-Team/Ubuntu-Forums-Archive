### Post by helgman on 2009-05-23
There are a few ways to make name resolutions.

My **guess** is that when you ping you use DNS.
When you do files sharing you might be using WINS.

You could try to use dig och nslookup and resolve the host. Just nslookup *hostname* in the terminal an you'll see the information ping uses to make it's echo request. You can also try nslookup *IP-of-the-host-on-your-LAN* to see if that helps you find what you're looking for (there might be a PTR record of it's your DNS).

### Post by helgman on 2009-05-24
Yes. The DNS server belongs to the domain ecompany.ae, sounds familiar?

There is quite some likeness between your forum name (Mosaab) and the domain name you look for (mosab.com) - if you cat /etc/resolv.conf - does it say *domain* or *search* mosab.com (or have you configured it somewhere)?

That domain seems to be "parked" and the names server responsible for it will answer any call for <something>.mosab.com with the same IP.

If you don't have a local DNS responsible for mosab.com and your computer is said to belong to (or that it should search) that domain, every time you query with a name that does not match an existing domain (like ubuntu or perfect instead of www.ubuntu.com or perfect.me.net) your computer will try to add mosab.com (so it will ask for ubuntu.mosab.com and perfect.mosab.com). Since your DNS server doesn't run this domain it will find the DNS server that does and that one will answer you (and the answer is always the same) - hens the strange IP address.

In your case (I guess), you try to reach neda-lap. The computer (or you) add .mosab.com and query the DNS server (213.42.20.20) that don't know the answer to it traverses the "DNS tree" until it finds a server responsible for the domain - that DNS server will return an answer that then is passed along to you.

WINS on the other hand don't bother with domain name and stuff so if you've got WINS up and running (or your computers take care of that by them selves) then neda-lap will resolve locally (since WINS don't go asking around when it doesn't know the answer, I guess).

Longer answer than I was initially aiming for, I hope the explanation is not to nested.

Regards,
Helgman

### Post by helgman on 2009-05-24
Na, that's not really big.

The content in your resolv.conf tells your computer to add mosab.com to any hostname it can't resolve - and that is common practice, especial with larger networks where you just put try and reach the hostname (ex alpha) and then let the OS try one or more domain (ex alpha.company.con alpha.subdomain.company.com).

It all makes perfect sense.

As for setting the domain/search parameters - you can set them to anything.

And you do your DNS query against externals (your ISP) DNS servers, nothing strange with that neither.

As for WINS and how it works in Linux/Ubuntu, I don't have a clue, never had the need. But normally all machines "advertise" themselves on the network and that's how one host picks up the information it needs to connect to other hosts on the LAN.

Hope some of the mist is lifting.
