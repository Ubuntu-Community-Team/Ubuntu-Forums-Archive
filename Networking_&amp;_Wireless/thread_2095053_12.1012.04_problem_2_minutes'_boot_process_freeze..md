### Post by chili555 on 2012-12-15
> How can I force dhcpclient not query non-existent interfaces for IP address?Easy, just like this:```
[COLOR="Red"]#[/COLOR]auto wlan0
iface wlan0 inet static
address 192.168.20.1
netmask 255.255.255.0
network 192.168.20.0
broadcast 192.168.20.255
up route add -host 255.255.255.255 wlan0

```But then, after you insert the USB, you'll have to start wlan0 manually:```
sudo ifup wlan0
```You've told interfaces to start wlan0 automatically; auto wlan0, but your intent is to *NOT* start it automatically. I suspect your real intent is the harder, maybe impossible task of starting wlan0 automatically only upon insertion of the USB.

That sure is a busy and mysterious interfaces file!

### Post by chili555 on 2012-12-16
> should I try to debug upstart boot process?I don't believe so. When there is no bug, there is nothing to debug. You have said to the system, auto wlan0, meaning, start wlan0 automatically no matter what. But, without the USB, there is no wlan0. You are asking the impossible. Therefor the system sits there asking itself what to do and trying to follow impossible requests. I think you have three options:

# Comment out auto wlan0 and start it manually when needed; or,

# Always start with the USB; or,

# Write some code somewhere that only starts wlan0 if the interface exists or the driver is loaded or some such. Where and how to do this is beyond my limited knowledge.

### Post by mafeuser on 2012-12-16
I partially solved the problem by using udev.
I wrote 2 rules to my new file /etc/udev/rules.d/90-myLocal.rules:

```
SUBSYSTEM=="net", ACTION=="add", DRIVERS=="?*", ATTR{address}=="00:60:b3:ff:b8:91", ATTR{dev_id}=="0x0", ATTR{type}=="1", KERNEL=="wlan*", RUN+="/sbin/ifup wlan0"

SUBSYSTEM=="net", ACTION=="remove", DRIVERS=="?*", ATTR{address}=="00:60:b3:ff:b8:91", ATTR{dev_id}=="0x0", ATTR{type}=="1", KERNEL=="wlan*", RUN+="/sbin/ifdown wlan0"
