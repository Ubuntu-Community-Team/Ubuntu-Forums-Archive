### Post by MoebusNet on 2013-06-03
I need to connect my notebook via WiFi to the Internet from inside my RV. As long as my doors & windows are open and I am near the WiFi source at RV parks, I can connect. If I am parked farther away from the WiFi antenna or if it is rainy I usually have a poor to unusable connection. I was thinking that an outdoor-type router/antenna combination might work, but I haven't found one that will work  for me with Ubuntu. I've bought & returned about 4 different routers to include a TP-LINK TL-WA5210g recently because although I could see and even connect to the RV park WiFi signal, I never could successfully connect my Ubuntu 12.04 notebook to the Internet through the router as a WISP Client.

I would appreciate any advice on what is working for you with this type of outdoor-to-indoor connection scenario, because I travel extensively and connect to many different RV park WiFi set-ups. The RV parks usually run an open-WiFi setup that requires a username & password in a browser log-in window before you can connect to the internet (not an issue, WiFi is included in RV lot rental fee). So far, I have never been able to see the log-in window in Firefox or Chrome even when the router appears to be connected.

I would prefer to purchase hardware that is DD-WRT and/or Tomato-capable and is capable of using a directional antenna in Client mode.

I haven't received any advice here, so I picked up a BUFFALO WHR-HP-G54 router to use in Client mode using a Yagi antenna. I guess I'll see how it goes.

UPDATE: This router is working well for me with DD-WRT installed, currently in repeater mode. I haven't needed the Yagi yet; it's kind of awkward unless you have it permanently mounted. Some open-security Access Points have a web-based login page that I haven't been able to get working through the repeater yet (web page never shows up in browser).
