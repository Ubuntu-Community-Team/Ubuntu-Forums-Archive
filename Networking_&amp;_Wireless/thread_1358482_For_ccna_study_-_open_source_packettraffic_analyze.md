### Post by puppywhacker on 2009-12-18
Wireshark is perfectly fine for analyzing traces, any professional that says otherwise is not well informed =) You can make traces with tcpdump, snoop or in any other capturing format and you can load it in Wireshark, ofcourse you can use libpcap to make live captures as well.

You can run IOS (cisco os right?) on a PowerPC, which you can emulate under i386, ofcourse you wouldn't be abled to run real routing cases since you don't have the routing engine (or how is that called in cisco's)
