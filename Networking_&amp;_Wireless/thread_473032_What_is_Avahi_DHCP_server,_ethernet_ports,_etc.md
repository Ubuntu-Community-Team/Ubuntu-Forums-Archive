### Post by spd106 on 2007-06-14
There's no need to remove avahi, that isn't your problem. 

You need an address for your eth1 interface before you can run a dhcp server (or any other for that matter). Usually you would  set a static address in the /etc/network/interfaces file, though you can use a dhcp client as well, though that complicates matters as you would have two dhcp servers on the same network.

The address you chose (192.168.0.0) is the network address and therefore is not a valid host address, so I suggest you use 192.168.0.1 instead. 
e.g.
```
:~$ cat /etc/network/interfaces
auto lo
iface lo inet loopback

auto eth0
iface eth0 inet dhcp

auto eth1
iface eth1 inet static
address 192.168.0.1
netmask 255.255.255.0
```

Because you do not set a static address the default setting is to use dhcp, after this failed avahi-autoip self assigned one in the link-local address space. This is done to allow you to still have a functioning network in the event that the dhcp server isn't present. This behaviour is new in Linux, though it has been like this in Windows and Macs for many years now. As you can tell it benefits desktop users far more than server admins.
