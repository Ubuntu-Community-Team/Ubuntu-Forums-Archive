### Post by papibe on 2017-03-07
Thanks.

Make sure 192.168.0.54 out of the range of the DHCP pool.

I personally prefer to set things up so that the router always knows who (hostname) and where (IP) the server is.

I would set the server as regular DHCP client, and set a reservation on the router. This way the router won't be dropping any server's information from the [ARP table]("https://en.wikipedia.org/wiki/Address_Resolution_Protocol")  (requiring for routing). Also, if the router does DNS, that would be even better as you may be even access the server by hostname.

Hope it helps. Let us know how it goes.
Regards.

### Post by papibe on 2017-03-07
> **milleroi said:**
> The only thing I have done since is go in and turn auto-reservation off.

DHCP lease or reservations last only a certain period of time (usually 24hrs). Once the lease expired, the server will request an extension or a new address to the DHCP server. Since the reservation is not there anymore, your server may be getting a different address.

What does this command show?
```
nmcli device show
```
Regards.
