### Post by nariub on 2012-09-22
I have several Ubuntu boxes at home
plus two laptops that sometimes are at home and sometimes on the road..

I was pulling updates from the internet for each..
not really bad but hey.. i wanted to fix that..

So I installed **apt-cacher-ng** on my server which had plenty of extra disc space

~$ sudo apt-get install apt-cacher-ng

then i stopped it so I could customize

~$ sudo service apt-cacher-ng stop

my big disc is mounted on /home so I created a directory and replaced the default directory with a symlink
~$ sudo mkdir /home/APT-CACHER
~$ sudo rmdir /var/cache/apt-cacher-ng
~$ sudo ln -s /home/APT-CACHER /var/cache/apt-cacher-ng

gotta get the permissions right
~$ sudo chown apt-cacher-ng:apt-cacher-ng /var/cache/apt-cacher-ng
this one was me setting a FACL nobody told me to.. I am just freaky like that..

~$ sudo setfacl -d -m mask:007 /home/APT-CACHER

this one makes the new directory match the one I removed earlier
~$ sudo chmod g+s /home/APT-CACHER

set up the import file we will use it in a minute
~$ sudo mkdir -p /home/APT-CACHER/_import
~$ sudo chown apt-cacher-ng /home/APT-CACHER/_import

here I peeked into the file to see what was what but didn't change anything
~$ sudo vi /etc/apt-cacher-ng/acng.conf

on the server I set up the proxy
sudo echo 'Acquire::http { Proxy "http://localhost:3142"; };' | sudo tee /etc/apt/apt.conf.d/02proxy

started the service
~$ sudo service apt-cacher-ng start

pulled my first update to get the sources.list correct
~$ sudo apt-get update

then i imported..
~$ cd /var/cache/apt/archives/
~$ sudo cp *.deb /home/APT-CACHER/_import/

use a browser to check to see if it is running and do the import
[http://localhost:3142/acng-report.html](http://localhost:3142/acng-report.html)

bottom of the webpage is an import button.. we hit it and populated our cache.

now we don't need the import directory anymore
~$ sudo rm /home/APT-CACHER/_import/*.deb
~$ sudo rmdir /home/APT-CACHER/_import

it says you can remove your archives now
~$ sudo rm -fr /var/cache/apt/archives/*.deb
  I haven't yet.. but you could..


for my static hosts that do not move about the country..
I hard configured them to proxy apt off myserver

on each client
~$ sudo echo 'Acquire::http { Proxy "http://MyServer:3142"; };' | sudo tee /etc/apt/apt.conf.d/02proxy

which creates a file 02proxy with the contents 
----
Acquire::http { Proxy "http://MyServer:3142
----

in /etc/apt/apt.conf.d/
which is where we want it..

each of those do a sudo apt-get update and upgrade

~$ sudo apt-get update; sudo apt-get upgrade

they are good..  google was whining.. I will come back to that one too..

now my laptops need to proxy on my home network but no proxy elsewhere.. so now I create a file in /etc/NetworkManager/dispatcher.d/ to detect what network i am on and set the proxy when at home and remove it when i am anywhere else..

I have set up my dhcp server to give me a domain when i connect and pull a dhcp address
 i have mylocal.dom as my local search order.. 
when connected to my home network i can see a search entry in /etc/resolv.conf 
-------------
# Dynamic resolv.conf(5) file for glibc resolver(3) generated by resolvconf(8)
#     DO NOT EDIT THIS FILE BY HAND -- YOUR CHANGES WILL BE OVERWRITTEN
nameserver XXX.XXX.XXX.XX1
nameserver XXX.XXX.XXX.XX2
nameserver XXX.XXX.XXX.XX3
search mylocal.dom

so i use the presence of my locally defined network in my network search order to determine whether I am connected to my network or not.

I have therefore created a script in 

```

---------------- BEGIN /etc/NetworkManager/dispatcher.d/99SetProxy ------
#!/bin/bash
MyNetwork=`grep search /etc/resolv.conf | awk '{ print $2 }'`
if [[ $MyNetwork == "mylocal.dom" ]]; then
        if [[ ! -f "/etc/apt/apt.conf.d/02proxy" ]]; then
                echo 'Acquire::http { Proxy "http://MyServer:3142"; };' | tee /etc/apt/apt.conf.d/02proxy
        fi
else
        if [[ -f "/etc/apt/apt.conf.d/02proxy" ]]; then
                rm /etc/apt/apt.conf.d/02proxy
        fi
fi
---------------- END /etc/NetworkManager/dispatcher.d/99SetProxy ------

```



which creates the proxy entry if i am connected to my home network and deletes it if i am not..

i can turn my wireless on and off  .. and watch the 02proxy file come and go

if you have multiple home networks or proxy settings.. you can mod my script above with a bunch of elif's to add one and a custom proxy for any network you expect to connect to..  

be careful about connecting to two network at the same time.. i do not know the behavior when VPN'ing across a wired or wireless network.. 



hope this helps someone
