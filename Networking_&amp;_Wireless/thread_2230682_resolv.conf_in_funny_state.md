My work has an internal network that only allows access if you're on the intranet.  When I started I could connect my computer running 12.04 to the ethernet and I could access the internal wiki, source control, etc.  Somehow my resolv.conf got screwed up when trying to use the VPN tools from home.  I couldn't access any domain so I just added nameserver 8.8.8.8 to my address (Google's DNS server, I believe).  That at least got me general internet.  I've now got VPN working using a java applet, but I'm in a weird state that even when I'm at work plugged into the ethernet I can only access external websites.  I have to actually run the VPN software to get access to internal sites.  

Here's my resolv.conf

```

# Dynamic resolv.conf(5) file for glibc resolver(3) generated by resolvconf(8)
#     DO NOT EDIT THIS FILE BY HAND -- YOUR CHANGES WILL BE OVERWRITTEN
nameserver 8.8.8.8
nameserver 127.0.0.1
nameserver 8.8.8.8
search corp.mycompany.com prod.mycompany.com staging.mycompany.com server.mycompany.com
nameserver my.work.vpn.address

```

When I run the VPN tool it replaces my resolv.conf with the last two lines, so I thought they would be required so I added them to the /etc/resolvconf/resolv.conf.d/tail file to look like this.  

```

nameserver 8.8.8.8
search corp.mycompany.com prod.mycompany.com staging.mycompany.com server.mycompany.com
nameserver my.work.vpn.address

```

I'm not very knowlegeable with resolv.conf but I figured if I just add 8.8.8.8 and the other two lines I'd have both the internal and external sites available when running off the ethernet at work.  Any insights to what is wrong with my setup?  Or some way to completely reset my resolv.conf configuration so it at least works the way it used to without VPN?
