### Post by capscrew on 2011-11-01
> **brett- said:**
> Each Linux machine on the network runs Ubuntu 10.04

I had to upgrade my DSL router, after a DSL service change, to a Q1000Z.

Prior to upgrade, any computer connected to my home network could access shared folders on the network.

After the upgrade I have to enter the IP addresses and names of the other machines in a machines etc/hosts file in order for that machine to access shared folders on the other machines.  Machines running widows access shares with no modifications to the windows machine.

If IP addresses and machine names are not hard coded in the etc/hosts file, the machine name is shown under Networks in Nautilus but selecting the machine gives -

"Failed to retrieve share list from server"

This is due to the method Samba uses to resolve hostnames to NETBIOS NAMES and ultimately to IP addresses.  A short description of this process is:  By default Samba looks for the DNS (hosts) name machine and converts this hostname into a NETBIOS NAME.  Samba then builds a Browse List (share list) which maps the NETBIOS NAME to IP address.  This what is queried wehe you browse for shares.

There is an alternative method to accomplish this.  You can *explicitly assign a NETBIOS NAME *to your Ubuntu hosts and then broadcast that name across the LAN.  This method is how Windows does it.  See below [COLOR="Red"](in red)[/COLOR] for what to do to implement this.> 

Here is the etc/hosts file for computer1.  In order for computer1 to access shares on computer2 the LAN IP and machine name for computer2 has to be entered manually -

```

127.0.0.1    localhost
127.0.1.1    computer1-desktop
192.168.0.6 computer2-desktop

...


```Prior to the router change out I did not have to do this.  In fact I did not know anything about the hosts file or the samba.conf file.


Here is samba.conf file
```

[global]
[COLOR="Red"]
### The NETBIOS NAME should be no longer than 15 characters ###

netbios name = <This_machines_hostname>
name resolve order = bcast hosts
[/COLOR]
...

[Public]
    path = /home/computer1/Public
    writeable = yes
;    browseable = yes [COLOR="Red"]<- Remove the ; to be able to browse[/COLOR]
    guest ok = yes


```

Add the items in red to the smb.conf file.  This should make the machine and it's shares visible.  You need to do this on any Ubuntu host that has Samba shares.
