### Post by chili555 on 2016-08-20
I notice this in your wireless script:> [/etc/modprobe.d/asus-laptop.conf]
options asus-laptop wapf=4 wlan_status=1 wimax_status=1 wwan_status=1To which I can only say, "YIIKKKEESSS!!!"

Ah, I feel better now.

If you consult the list of loaded modules:```
lsmod
```...you will see that asus_laptop isn't even loaded. Whether asus_laptop or asus-wmi (and it's brother, asus-nb-wmi) load depends on the DMI number for your laptop. One or the other will load; not both.

Since the driver parameter is not used, let's clean up a bit and simply remove the file:```
sudo rm /etc/modprobe.d/asus-laptop.conf
```Next we see this:> [/etc/modprobe.d/asus.conf]
options asus_nb_wmi wapf=4
...and this:> [/etc/modprobe.d/asus_nb_wmi.conf]
options asus_nb_wmi wapf=4They are duplications, probably evidence of two attempts, both unsuccessful, to do the same thing. Again, let's clean up a bit and remove one:```
sudo rm /etc/modprobe.d/asus.conf
```Now we can address the real issue. There are three possible values that we know of for asus_nb_wmi:  [http://lxr.free-electrons.com/source/drivers/platform/x86/asus-nb-wmi.c?v=4.4](http://lxr.free-electrons.com/source/drivers/platform/x86/asus-nb-wmi.c?v=4.4)>  44  * WAPF defines the behavior of the Fn+Fx wlan key
 45  * The significance of values is yet to be found, but
 46  * most of the time:
 47  * Bit | Bluetooth | WLAN
 48  *  0  | Hardware  | Hardware
 49  *  1  | Hardware  | Software
 50  *  4  | Software  | SoftwareYou currently have the parameter set as =4. Let's try =1.

We will over-write the existing file with =1.```
sudo -i
echo "options asus-nb-wmi wapf=1  >  /etc/modprobe.d/asus_nb_wmi.conf
exit
```The objective is to get the wireless key combination, evidently Fn+F2, to turn 'hard blocked:yes' into no.

We also see this in your script.> Region: Asia/Manila (based on set time zone)


country 00: DFS-UNSET
	(2402 - 2472 @ 40), (N/A, 20), (N/A)
	(2457 - 2482 @ 40), (N/A, 20), (N/A), NO-IR
	(2474 - 2494 @ 20), (N/A, 20), (N/A), NO-OFDM, NO-IR
	(5170 - 5250 @ 80), (N/A, 20), (N/A), NO-IR
	(5250 - 5330 @ 80), (N/A, 20), (0 ms), DFS, NO-IR
	(5490 - 5730 @ 160), (N/A, 20), (0 ms), DFS, NO-IR
	(5735 - 5835 @ 80), (N/A, 20), (N/A), NO-IR
	(57240 - 63720 @ 2160), (N/A, 0), (N/A)I recommend that your regulatory domain be set explicitly. Check yours:```
sudo iw reg get
```If you get 00, that is a one-size-maybe-fits-all setting. Find yours here: [http://en.wikipedia.org/wiki/ISO_3166-1_alpha-2](http://en.wikipedia.org/wiki/ISO_3166-1_alpha-2) Then set it temporarily:```
sudo iw reg set IS
```Of course, substitute your country code if not Iceland. Set it permanently:```
gksudo gedit /etc/default/crda
```Use nano or kate or leafpad if you don't have the text editor gedit.

Change the last line to read:```
REGDOMAIN=IS
```Proofread carefully, save and close the text editor.

Reboot and tell us if the wireless is now working!
