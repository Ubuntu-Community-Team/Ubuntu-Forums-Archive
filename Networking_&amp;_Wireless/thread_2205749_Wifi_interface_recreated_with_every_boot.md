### Post by varunendra on 2014-02-16
Welcome to the forums Nentuaby !

There may be some configuration issue with Network Manager causing this. How do you configure your connection in Network Manager (details of every step).

It may also help if you check your /etc/NetworkManager/system-connections/ directory and see how many duplicate connection files have been created. Compare the contents of these duplicate connection files and post back the contents of any of them if all are similar.

These connections files (called "key files")** contain sensitive information like [COLOR="#FF0000"]your wireless passkey[/COLOR] and MAC addresses**, so if you post their contents here for analysis, **_make sure to obscure or remove these before posting here_**.

Also, since they contain sensitive information, you'll need root privileges to be able to even read them. The terminal way to read their contents with root access -
```
sudo cat /etc/NetworkManager/system-connections/*<the desired key file>*
```

Oh, and you can safely delete your 70-persistent-net.rules file. It will be automatically regenerated on next boot or when a new connection needs to be created.

### Post by varunendra on 2014-02-16
The one that you edited out or the "seen-bssids" ?

If it is the "**mac-address=..**" field, then it shouldn't happen automatically. It is meant to be the MAC ID of _YOUR_ wireless card (not the access-point), and can't change unless the card itself is changed.

This is what I suspected in the first place - unknowingly selecting or otherwise filling in an external MAC address in the Network Manager's "Device MAC address" field - something that can't happen by itself as far as I know. This is the only reason I have seen so far that causes creation of multiple connections IDs for the same connection.

This field tells Network Manager which card to use to establish the connection (thus 'Binding' that connection specifically with that card). What happens is - you save a MAC address which is not of the card you have. So on the next attempt, Network Manager tries to use that connection, but can't find an internal card or adapter with that mac-address to use, so creates a fresh connection. Same error again resulting in the same behaviour again.

That is why I asked you the steps in detail, but you don't seem to be doing this manually? This is mysterious then. Have you recently installed a program that 'Changes' your hardware MAC address randomly? This is possible and is known as 'MAC Spoofing', but is something that takes quite some deliberate efforts to achieve that.

When you create a connection, open it from Network Manager (nm-applet) menu > Edit Connections... > "Wireless" tab > double-click the connection currently in use > under "Wireless" tab, check the "Device MAC address" field.

This field should be either blank, or should be the MAC ID of one of the cards you are using to connect to the network. So if there already is a MAC address there, is it the same as your card's MAC address?

You can check your card's MAC address with "**ifconfig**" command. The "HWaddr" in the output would be the MAC ID of your card, and I'm sure it won't change by itself.
