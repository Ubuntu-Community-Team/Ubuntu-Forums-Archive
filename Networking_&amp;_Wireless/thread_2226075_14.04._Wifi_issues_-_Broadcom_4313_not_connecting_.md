### Post by Tze_Veh on 2014-05-25
Hi all,

I've been using Xubuntu since 13.04 on my current laptop (Lenovo ideapad N581). I had wifi issues to begin with, but managed to get around them by installing a broadcom backport driver that worked like a charm. Upgrading to 13.10 was no problem either, the drivers continued to work. When trying to upgrade to 14.04 however my system crashed beyond repair and I had to do a full reinstall; and now my wifi is acting up again in the same way:

I *can* connect to a wifi when the source signal is not further away than maybe 1 meter, speed is not an issue whatsoever - when using my phone as a mobile hotspot placed next to the laptop, everything works fine. I'm also picking up all other wifi signals in a 100 meter range, and I have access to 3 of them - my phone connects without issues, the broadcom does nothing.

Last time I fixed this precise issue by installing a broadcom backport driver that was recommended in a threat here; unfortunately I can't locate it anymore, and the bookmark died along with my 13.10 installation... I came across this how-to ([https://sites.google.com/site/easylinuxtipsproject/internet](https://sites.google.com/site/easylinuxtipsproject/internet)) where it is mentioned in 1.5. that the backport driver is not available yet for 14.04 - I tried all other tips for good measure, but none worked; neither did one from the countless suggestions here in the forum (installing a variety of different drivers etc.).

### Post by varunendra on 2014-05-27
Welcome to the forums Tze_Veh ! :)

Let's start with some house-cleaning.. Please delete custom blacklist entries and a blacklist file that are incorrect/unnecessary -
```
sudo rm /etc/modprobe.d/broadcom-sta-common.conf
sudo sed -i '/^blacklist b43/ d'
sudo sed -i '/^blacklist wl/ d'
```
..we may have to blacklist "b43" and/or "wl" again, but for now, the above would simply make things as they are by default.

Apart from above in Ubuntu, please change the encryption type in your router/access-point to pure **WPA2-PSK with AES (CCMP)**. Your current AP is using WPA/WPA2 mixed mode with TKIP, and some others (apparently yours, since they have similar SSID) are using WPA with TKIP. The TKIP algorithm is an outdated, inefficient algorithm that may cause performance issues, and WPA/WPA2 mixed mode may confuse the drivers. So they (the mixed mode and TKIP) should be avoided. Since WPA(1) essentially needs TKIP, that means avoid WPA as well. Reboot the router after saving the changes, then reboot your computer too, and try to connect again.

If these two changes don't fix the problem, please follow the instructions in this post to download and run 'wireless_script' (experimental version) and post back the report it generates : [http://ubuntuforums.org/showpost.php?p=13024222](http://ubuntuforums.org/showpost.php?p=13024222)

### Post by praseodym on 2014-05-31
Uninstall the STA driver and install the missing firmware:
```

sudo apt-get install linux-firmware-nonfree
sudo apt-get remove --purge bcmwl-kernel-source
sudo rm /etc/modprobe.d/blacklist-bcm43.conf
sudo rm /etc/modprobe.d/broadcom-sta-common.conf
sudo rm /etc/modprobe.d/broadcom-sta-dkms.conf 
sudo sed -i "s/blacklist b43/#blacklist b43/g" $(egrep -lo 'blacklist b43' /etc/modprobe.d/*)
sudo sed -i "s/blacklist ssb/#blacklist ssb/g" $(egrep -lo 'blacklist ssb' /etc/modprobe.d/*)
sudo sed -i "s/blacklist bcma/#blacklist bcma/g" $(egrep -lo 'blacklist bcma' /etc/modprobe.d/*)
sudo sed -i "s/blacklist brcmsmac/#blacklist brcmsmac/g" $(egrep -lo 'blacklist brcmsmac' /etc/modprobe.d/*) 
```
Reboot.
