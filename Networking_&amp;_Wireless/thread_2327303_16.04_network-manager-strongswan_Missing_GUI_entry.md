### Post by herve-i on 2017-01-12
The solution is to build network-manager-strongswan from source. I tested it on 3 different machines and I now have Strongswan appearing as an option when creating a new VPN in network Manager.

```

[COLOR=#000000][FONT=&amp]# get the strongSwan tarball
wget http://download.strongswan.org/strongswan-5.x.x.tar.bz2
tar xjf strongswan-5.x.x.tar.bz2
cd strongswan-5.x.x
[/FONT][/COLOR]
[COLOR=#000000][FONT=&amp]# build charon with OpenSSL/NM Plugin
./configure &#8211;sysconfdir=/etc &#8211;prefix=/usr &#8211;libexecdir=/usr/lib \
&#8211;disable-aes &#8211;disable-des &#8211;disable-md5 &#8211;disable-sha1 &#8211;disable-sha2 \
&#8211;disable-fips-prf &#8211;disable-gmp &#8211;enable-openssl &#8211;enable-nm &#8211;enable-agent \
&#8211;enable-eap-gtc &#8211;enable-eap-md5 &#8211;enable-eap-mschapv2 &#8211;enable-eap-identity
make
make install
[/FONT][/COLOR]
[COLOR=#000000][FONT=&amp]# get the NetworkManager strongsSwan plugin as a tarball
wget http://download.strongswan.org/NetworkManager/NetworkManager-strongswan-1.x.x.tar.bz2
tar xjf NetworkManager-strongswan-1.x.x.tar.bz2
cd NetworkManager-strongswan-1.x.x
[/FONT][/COLOR]
[COLOR=#000000][FONT=&amp]# build the NetworkManager strongsSwan plugin (if you changed prefix/libexecdir above, set &#8211;with-charon=/path/to/charon-nm)
./configure &#8211;sysconfdir=/etc &#8211;prefix=/usr &#8211;with-charon=/usr/lib/ipsec/charon-nm
make
make install
[/FONT][/COLOR]
```[COLOR=#000000][FONT=&amp]


[https://www.andrieu.us/strongswan-vpn-client-with-ubuntu-16-04/](https://www.andrieu.us/strongswan-vpn-client-with-ubuntu-16-04/)[/FONT][/COLOR]
