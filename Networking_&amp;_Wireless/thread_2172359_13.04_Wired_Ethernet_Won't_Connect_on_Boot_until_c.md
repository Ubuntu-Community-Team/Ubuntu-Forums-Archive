### Post by Shylerbrew on 2013-09-04
Strangeness... On boot, my new install of ubuntu 13.04 (64-bit) will not connect my wired ethernet until the cable is unplugged and reinserted, afterwhich the connection performs normally and stays up until the next restart. 
Ubuntu is installed on a desktop, dual booted with Windows 8, which shows no issues with regards to the connection, thus the hardware appears to be fine. The Windows OS was installed on the system first. 
The system is a (new) ASUS G41 class motherboard, hosting an Intel Core2-Quad Q8200 CPU with 4 Gbs memory. 
When ethernet is up, the Driver listed is:  r8169. 
There is no WiFi. 
The connection tested OK with the LiveCD prior to install. 
Any suggestions as to what to check will be awesome. Thanks!

### Post by praseodym on 2013-09-04
Please install
```

sudo apt-get install ethtool
```

Please show then when it is not working:
```

lspci -nnk | grep -iA2 net
lsmod
cat /etc/network/interfaces
ifconfig -a
dmesg | grep r8
sudo ethtool eth0
```

### Post by houstonbofh on 2013-09-04
Next time it is down, show the output of "ifconfig" and it is it a 169.x.x.x address, try "sudo dhclient" and see if it resolves.  It could be the DHCP client coming up before the network stack is fully up.
