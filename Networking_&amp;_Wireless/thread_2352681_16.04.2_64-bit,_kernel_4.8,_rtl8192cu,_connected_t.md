```

When my wireless got messed up it installed this driver rtl8xxxu as well I blacklisted it then I reinstalled the fixed driver after changing to the path of the fixed driver and doing:
```
sudo make clean

It is dkms that failed you can leave that package out of the command and see if it will build then but you will have to do make clean and run the make commands every time there is an upgrade to the kernel.

I think it is likely that the fixed driver you installed will work good again like mine does if you just blacklist rtl8xxxu, if you look the 8192cu driver is loaded but the other driver is conflicting with it.
