### Post by Lars Noodén on 2014-11-19
> **sognoct said:**
> 
```

Chain FORWARD (policy ACCEPT 0 packets, 0 bytes)
num   pkts bytes target     prot opt in     out     source               destination
1        0     0 ACCEPT     tcp  --  any    any     anywhere             192.168.1.69         tcp dpt:http

```

If that machine is the one that is supposed to be serving up HTTP, then you should probably clear the forwarding chain.  

Your network is like this, if I read correctly?

```

Internet ----- Cisco ----- Ubuntu12.04

```

As to forwarding FTP, it needs two ports forwarded, one of which is determined dynamically.  If you are not absolutely tied to FTP, it might be a convenient excuse to upgrade to SFTP and turn off FTP.  SFTP works well with port forwarding.
