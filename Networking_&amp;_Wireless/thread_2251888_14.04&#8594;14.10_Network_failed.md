But I hesitate to format and reinstall from scratch.I'm not suggesting a reinstall. I'm suggesting you remove Network Manager altogether; it obviously has quite a few problems.```
sudo apt-get purge network-manager
```And then amend a file:```
gksudo gedit /etc/network/interfaces
```Use nano or kate or leafpad if you don't have the text editor gedit. 

Add two new lines so that the file now reads:```
# interfaces(5) file used by ifup(8) and ifdown(8)
auto lo
iface lo inet loopback

auto eth0
iface eth0 inet dhcp
```Proofread carefully, save and close the text editor. Reboot and you'll be all set.

### Post by skygreg on 2014-12-14
Hi ITKBrenger, Hi all,

I've faced the same problem since I've upgraded to 14.10. I finally found where was the problem: a buggy script in /etc/network/if-up.d/

ST;LT:

Execute command "ifup -a" and see if errors occured. Then, fix them.

pre-start script
    echo "pre start script 0" >>/tmp/greg.log
    if [ "$UPSTART_EVENTS" = "stopped" ] && [ "$UPSTART_JOB" = "networking" ] && [ "$EXIT_STATUS" = "100" ]; then
    echo "pre start script 1" >>/tmp/greg.log
        exit 0
    fi


    echo "pre start script 2" >>/tmp/greg.log
    mkdir -p /run/network
    echo "pre start script 3" >>/tmp/greg.log
    ifup -a
    echo "pre start script 4" >>/tmp/greg.log
end script

I've never see step 4 in greg.log, so I've finally execute "ifup -a" and see that a buggy script in /etc/network/if-up.d/ was exiting with an error code > 0. Removed it, restart, and everything was fine !
