### Post by squakie on 2014-02-19
First off, unless I missed it, you don't have any shares defined.  Kind of hard to find something that doesn't exist.

Also, many of us have reported that sometimes we cannot go via the "Network" option on Nautilus, but must instead go to "Files" and then "Connect to server" or "Location".  All of this is *supposed* to work if (1) the systems have fixed addresses and (2) the host files on each system are set up correctly so that each system "knows" about the other.  However, this doesn't seem to work very well for a large number of us.  Even specifying the connection with something like "<userid:password>//xxx.xxx.xxx.xxx doesn't seem to want to work a lot of the time.  I read a reply from one of the "in-the-know" forum admins a while ago that noted this problem and mentioned going via "Files" and "Connect to server".

Don't know if any of that will help or not.   Remember you must define a share or there is nothing to see.  You need to get permissions defined correctly (1) for the share in the smb.conf file, (2) possibly in the samba user/password file, (3) on the actual folder that is the share.  Be sure that the user is part of the group of samba users.

```

#following lines were added to original /etc/samba/smb.conf file
[COLOR="#FF0000"]#netbios name = l05[/COLOR]
#name resolve order = bcast host
[COLOR="#0000FF"]name resolve order = bcast host lmhosts wins
[/COLOR]

```

These type of errors are sometimes a combination of things.  You have part of the answer listed in the code above (in [COLOR="#0000FF"]**blue**[/COLOR]), but you need to remove the # in the netbios name line (in [COLOR="#FF0000"]red[/COLOR]) for it to work.  Once you do that the name resolve is directly handled by the NETBIOS daemon (service).  This means there is no need to have a static IP address to find it by name when browsing the "Network".  It works just like Network Neighborhood in Windows.

I have to assume you have created the shares by using the Nautilus GUI method ( right click>>etc ).  To check that list the shares with the following terminal command```
net usershare info --long
```...you can paste the info in the code blocks using the [SIZE=4]**# **[/SIZE] icon at the top middle row of the editor formatting icons.

To diagnose this further post the output of ```
smbtree -d3
```
