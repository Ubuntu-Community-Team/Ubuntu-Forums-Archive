### Post by shochatd on 2011-10-15
Here is how I fixed it:
1. Edited /etc/NetworkManager/NetworkManager.conf as follows (using sudo gedit):
After: [ifupdown], changed managed=false to managed=true
2. Then had to get a root prompt via sudo bash and then...
# service network-manager stop
# service network-manager start
It did NOT work to say 
$ sudo service network-manager stop etc.
Once I did that, my interface (eth1) showed up as managed. Yipee! The network icon in my menu bar changed from an "empty wifi" icon to a pair of arrows, one up and one down. I had already found something called gnome-nettool which had allowed me to enter my static IP address, DNS servers etc. although that had left the interface configured but still "unmanaged" by NetworkManager.
At this point, I restarted and came up with a fully-populated /etc/resolv.conf which had been written by NetworkManager but with the right stuff this time.

$ping www.google.com

```
$ cat /etc/resolv.conf
# Generated by NetworkManager
```no nameserver, so I followed the instructions above

3. Edited /etc/NetworkManager/NetworkManager.conf 
```
$gksu gedit /etc/NetworkManager/NetworkManager.conf
```After: [ifupdown], changed managed=false to managed=true

4. Then
```
$ sudo service network-manager stop
$ sudo rm /var/lib/NetworkManager/NetworkManager.state 
$ sudo service network-manager start
```Now the network icon changes to a pair of arrows.

5. Right click on the network icon and select Auto Ethernet
Worked for me

### Post by shochatd on 2011-10-18
To Papaapa: NetworkManager will ALWAYS rewrite /etc/resolv.conf. So it needs to know what to rewrite it with. You need to tell it by configuring the interface (Network from the Session indicator menu). If you say to use DHCP (see pa3fat's comment), then the DNS server address(es) should be supplied by your DHCP server, which may be a "cable modem", depending on how you're set up. In my case, I was NOT using DHCP because I wanted to assign a static IP address. So I had to enter my DHCP server addresses in the appropriate tab of the network interface configuration dialog, separated by commas (I'm at work right now, which is why I can't be more specific). NetworkManager pays NO attention to anything you may have put into resolv.conf yourself.
-- David

I ran into this problem when upgrading to 12.04. The instructions above only fixed my problem until the next reboot. However, they did point me in the right direction:

```
sudo stop network-manager
sudo dpkg-reconfigure resolvconf
# ...answer "yes" to "Prepare /etc/resolv.conf for dynamic updates?"...
# ...finish reconfiguration of resolvconf...
sudo reboot
```

The reboot was recommended at the end of the "sudo dpkg-reconfigure resolvconf" process, so I followed its advice. Now NetworkManager and resolvconf are playing well together, even after rebooting :)
