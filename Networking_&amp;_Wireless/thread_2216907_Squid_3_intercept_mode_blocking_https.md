### Post by SeijiSensei on 2014-04-14
I don't see anything in that ruleset that would block HTTPS requests, or any other connections for that matter.  The default policies are all ACCEPT, so all traffic should flow directly through.  

Maybe you didn't enable TCP forwarding in /etc/sysctl.conf?  Make sure the line net.ipv4.ip_forward=1 is uncommented in that file.  If not, remove the hash mark at the beginning of the line and reboot.

If that solves the immediate problem, you need to start adding more security to the iptables ruleset.  You should have DROP as the default policy for INPUT and FORWARD; I usually leave OUTPUT set to ACCEPT.  Then you'll need a couple of rules to handle HTTPS requests:
```
/sbin/iptables -A INPUT   -s 192.168.0.0/24 --dport 443 -j ACCEPT
/sbin/iptables -A FORWARD -s 192.168.0.0/24 --dport 443 -j ACCEPT

```
