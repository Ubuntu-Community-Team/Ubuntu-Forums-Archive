### Post by rabatitat on 2010-06-05
network-manager-openvpn adds an extra route in my case.
10.8.0.0/24 is the VPN subnet. I have 3 routes pushed to me by the server.
server assigned IP 10.8.0.14 with tunnel end point at 10.8.0.13.

the extra route added is 10.8.0.1 gw 10.8.0.13 which then causes all the routing troubles.

removing the route manually solves it for me but is quite a PITA.

network-manger-openvpn worked fine 9.04 then got messed up after.

Didn't have the same routing problem on openvpn-gui for windows.

Also ignoring the routes pushed by server and adding the routes manually works as well. However in my case since my VPN IP is server-assigned, I'd have to change the gateway manually each time it changes.

Should we file a bug report?
