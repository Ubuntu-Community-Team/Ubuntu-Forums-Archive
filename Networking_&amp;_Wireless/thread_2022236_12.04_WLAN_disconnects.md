### Post by Kirk Schnable on 2012-07-10
I'd like to try and determine which drivers you have installed, and whether there might be more recent ones available.  Sometimes, updating your Ubuntu packages doesn't get you the latest cutting edge drivers, most of the time you have to compile those yourself.

For every item I put in a Code box below, please run the command in a terminal window and paste the output to this thread, or link us to the output in a pastebin.

Let's start out by determining what type of hardware you have installed in your system.

```
lspci
```

Then let's see what loadable kernel modules you have loaded up.

```
lsmod
```

Finally let's have a look at your IP and Wireless configurations.

```
ifconfig
```

```
iwconfig
```

This information should shed some light on probable causes for your networking difficulties.

Kirk

### Post by Kirk Schnable on 2012-07-11
> **ddavidd said:**
> Hello Kirk, thanks again for your help.
Always glad to be of some assistance.

> I also tried starting with previous kernel versons 3.2.0.23 and 24 instead of .26, didn't work.
Unless you're planning on booting into the outdated kernels, it does not help to compile drivers for them.  The drivers only benefit you when the kernel is actually running.

> **ddavidd said:**
> Unfortunately, your instructions are more advanced than my Linux capabilities.
Sorry, didn't mean to assume too much.  I will clear up my instructions later on in the post.

> **ddavidd said:**
> Is it true to say that ath9k = problem = driver = kernel module?
The kernel module should be the **ath9k** one, yes.  That appears in your lsmod from before as well.

> **ddavidd said:**
> And if I remove it with rmmod, where do I find the new one?
You just compiled it.  Those .KO files you see are kernel modules.  .O files might also be present which might be modules.  You should see an **ath9k.ko** file in your folder where you compiled the new drivers, correct?

> **ddavidd said:**
> I found, for example, a folder 3.2.0-26-generic.pae/kernel/drivers/net/wireless/ath/ath9k 
with some .ko files in it
This is the location I referenced in my last bit above.

Here's what you should do to test this new module.

Before doing any of this, go ahead and check the version of ath9k running now.  It wouldn't hurt to paste this output somewhere so you have it for reference.

```
modinfo ath9k
```

Now let's test the new module:

```
rmmod ath9k
```

```
cd 3.2.0-26-generic.pae/kernel/drivers/net/wireless/ath/ath9k
insmod ath9k.ko
```

If you have trouble with the insmod command, make sure ath9k.ko exists.  If it doesn't try ath9k.o if that exists.  Otherwise give me a directory listing so I know which KO files actually exist.

After successful loading of the module, use modinfo to verify that the new version is actually loaded.

```
modinfo ath9k
```

If you have trouble, copy this output as well and supply the forums with the output from this modinfo command, and the one you ran above.

I hope I've cleared up my vague instructions from earlier.

Kirk
