The version of NDISwrapper found in the repositories has been an older version with 'issues', don't know if that has changed or not.  People have downloaded the current version source and built it rather than using the repository.  Does the Netgear WNDA3100v2 use the same problematic Broadcom chip that WNDA3100 uses?

### Post by kurt18947 on 2013-04-11
I don't have that adapter so I only know some simple stuff.  I'd start out by opening a terminal and typing "lsusb".  That should tell you the chipset in use.  Assuming it's Broadcom, I'd go to Broadcom's site and see what's available for Windows XP.  Here's an example of lsusb output:

```
Bus 001 Device 002: ID 07d1:3303 D-Link System DWA-131 802.11n Wireless N Nano Adapter(rev.A1) [Realtek RTL8192SU]
```

This tells me this DWA-131 uses a RealTek RTL8192SU chipset.  If I required NDISwrapper,  I'd go to RealTek's site and search for the appropriate Windows driver.  And hope it's not an .exe file which quite a few these days are.  There's probably a way to extract .inf, .sys, and .cat files from an .exe but I don't know it.  If you already know this, sorry for being redundant.
