### Post by madcap_magician on 2006-11-12
I installed Dapper server today.  I had previously run Breezy server. 

The cable modem connects on eth0.  eth1 goes to my internal network.

The first packages I installed were dhcp, ipmasq and dnsmasq.  Everything is working fine for ip masquerading as it was with Breezy.

The big difference I noticed between Breezy and Dapper were that Breezy's iptables started out empty where as Dapper's iptables have a bunch of stuff in them (see below).

In Breezy I ran this simple script to set up iptables the way I wanted them.

```

#!/bin/bash

# set rules for iptables


# allow anything that i open
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# allow ssh
iptables -A INPUT -p tcp -i eth0 --dport 22 -j ACCEPT

# allow anything from localhost
iptables -A INPUT -i lo -j ACCEPT

# allow anything from my internal network
iptables -A INPUT -i eth1 -j ACCEPT

# ping
iptables -A INPUT -p icmp -j ACCEPT
# web
iptables -A INPUT -p tcp -i eth0 --dport 80 -j ACCEPT
# ntp
iptables -A INPUT -p udp -i eth0 --dport 123 -j ACCEPT
# echo (important for dhcp to work)
iptables -A INPUT -p tcp -i eth0 --dport 7 -j ACCEPT
iptables -A INPUT -p udp -i eth0 --dport 7 -j ACCEPT
# aim/icq
iptables -A INPUT -p tcp -i eth0 --dport 5190 -j ACCEPT
# dhcp
iptables -A INPUT -p udp -i eth0 --dport 67 -j ACCEPT
# dhcp
iptables -A INPUT -p udp -i eth0 --dport 68 -j ACCEPT
# dns
iptables -A INPUT -p udp -i eth0 --dport 53 -j ACCEPT
iptables -A INPUT -p tcp -i eth0 --dport 53 -j ACCEPT
# mysql admin
iptables -A INPUT -p tcp -i eth0 --dport 3306 -j ACCEPT

# ipmasquerading
iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
iptables -A INPUT -i eth0 -m state --state NEW,INVALID -j DROP
iptables -A FORWARD -i eth0 -m state --state NEW,INVALID -j DROP

#block all other incoming
iptables -A INPUT -i eth0 -j DROP

```

This all seemed to work fine on Breezy.


On dapper if I flush the default settings and run this script I can't do anything.  It seems like everything I"ve tried gets dropped.  I can't even ping the loopback address.

Is there something I'm missing?  Why was this so straight forward on Breezy but completely fails on Dapper?  I eventually want to run more services but I'm worried I won't be able to control what gets rejected by just sticking with the defaults.
