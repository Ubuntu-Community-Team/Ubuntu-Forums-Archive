If you do not have a wired network you should change your 'map eth0' to read 'map wlan0'.

My own interfaces file looks like this (I've missed out some of the stuff at the beginning...) :

> 
iface wlan0 inet static
address 192.168.0.100
netmask 255.255.255.0
gateway 192.168.0.1
wireless-essid bscbritnet
wireless-nick spare
wireless-channel 6
wireless-mode managed
wireless-ap 00:13:46:9E:FF:FF
wireless-key1 aa11bb22cc33dd44ee55ff6600
wireless-defaultkey 1
wireless-keymode open


Possible significant differences are 1.  I tell the card which channel to use 2. I define the key (and note it is ....-key1) and then select as ...-defaultkey 1  and 3. I tell it the MAC address of the AP I am trying to connect to, and 4.  I also provide some additional information.  

Now, I don't know which of the above might be the critical change that you require but if you can include them all it shouldn't do any harm.  If that doesn't recover the situation, try the following link to identify where the problem lies:  

[https://wiki.ubuntu.com/WirelessTroubleshootingGuide](https://wiki.ubuntu.com/WirelessTroubleshootingGuide)
