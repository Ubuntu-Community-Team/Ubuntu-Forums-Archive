### Post by liskiewicz on 2009-11-12
Hello,

I have AMILO M7440 laptop which requires modprobe fsam7440 to run wireless card: 
[http://fsam7440.sourceforge.net/](http://fsam7440.sourceforge.net/)

Starting from ubuntu 8.10 it had to be additionallt patched according to [https://bugs.launchpad.net/ubuntu/+source/linux-source-2.6.17/+bug/61827](https://bugs.launchpad.net/ubuntu/+source/linux-source-2.6.17/+bug/61827)

That is all ok in regular ubuntu. But today I've tried with ubuntu studio (which has rt kernel). However, I've decided to install generic kernel on it by ```
sudo apt-get install linux-generic
``` to avoid problems with OpenOffice. And in generic kernel created in such a way fsam7440 makefile does not work. Here's the output:

```
grzesiek@grzesiek:~/Desktop/fsam7440-0.4$ make
make -C /lib/modules/2.6.28-16-generic/build SUBDIRS=/home/grzesiek/Desktop/fsam7440-0.4 modules
make: *** /lib/modules/2.6.28-16-generic/build: No such file or directory.  Stop.
make: *** [default] Error 2

```

Next thing I did was just creation of an empty folder "build". After that the result was:
```
grzesiek@grzesiek:~/Desktop/fsam7440-0.4$ make
make -C /lib/modules/2.6.28-16-generic/build SUBDIRS=/home/grzesiek/Desktop/fsam7440-0.4 modules
make[1]: Entering directory `/lib/modules/2.6.28-16-generic/build'
make[1]: *** No rule to make target `modules'.  Stop.
make[1]: Leaving directory `/lib/modules/2.6.28-16-generic/build'
make: *** [default] Error 2

```

The outcome of "make install":
```
grzesiek@grzesiek:~/Desktop/fsam7440-0.4$ sudo make install
su -c "install -d /lib/modules/2.6.28-16-generic/kernel/net/wireless/ && install -m 644 -c fsam7440.ko /lib/modules/2.6.28-16-generic/kernel/net/wireless/ && /sbin/depmod -a"
install: cannot stat `fsam7440.ko': No such file or directory
make: *** [install] Error 1

```

I guess that the issue may be connected somehow with the fact, that in usr/src there is no folder related to "generic" version of the kernel, and this makefile somehow needs that file (some people have succeded in similiar problems by creating empty config.h- this did not work in my case)
[http://ubuntuforums.org/showthread.php?t=404209](http://ubuntuforums.org/showthread.php?t=404209)
