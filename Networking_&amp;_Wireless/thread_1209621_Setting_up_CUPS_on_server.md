### Post by tgalati4 on 2009-07-10
Yes, cups will print over the web with port-forward 631 set on your router.  May be a security risk to leave 631 open all the time.  I would do it only when you really need it.  

ssh into the server (properly safeguard ssh--search the forums), change either the Listen port  or open the 631 pinhole on the router during a remote printing session.

Web printing seems to be a novelty since you normally need to be near the printer to pull the paper jams.

### Post by tgalati4 on 2009-07-10
The test page has complicated gradients that could seize up the postscript transcriber.  I've had it fail on a few printers even though the printer worked fine otherwise.

Try setting browseable = yes and guest OK = yes under [printers].

I don't use samba anymore.  One benefit to "opening the window".
