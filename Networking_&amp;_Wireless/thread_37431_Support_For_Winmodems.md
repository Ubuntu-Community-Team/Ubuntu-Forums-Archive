### Post by pja on 2005-05-27
**This is a request for future versions of Ubuntu.** 

Can we please have 'out-of-the-box' support for winmodems?  Xandros is the only distribution that I have tried that provides such support.

As a Windows user trying to make the move to Linux two of the most frustrating issues are:
[list=1]
[*]Lack of support for Winmodems - many of us have PCs fitted with these modems which work fine in Windows but won't work in Linux (buying a new modem is NOT a satisfactory solution).
[*]Printer support - the support for newer types of printers (such as my Canon LBP-1210 laser) is generally non-existant.
[/list] 

I really don't understand the ideological approach some people seem to take to these issues, particularly to Winmodems.  Like many users who would like to make the switch, I don't yet know how to install drivers and re-compile the kernel.  It might sound trite, but I never had to do that with Windows.

How about Ubuntu take a new look at these issues and give us an operating system that can use our existing hardware.

I don't mean this as a negative to the Ubuntu, it really is a very good effort, but it needs some of these small extras to really make it great.

Regards,
Peter

### Post by az on 2005-05-27
"I really don't understand the ideological approach some people seem to take to these issues"

You really don't.

Putting aside winmodems, free software is more than just a different way to develop software.  Yes, it uses the pooled ressources and problem-solving abilities of many people, but it also provides you with a way to keep your rights safe.

If you do not feel that it is your right to be able to know what is going on inside your box, you are still free to use proprietary software.

Getting back to Modems, the problem is that the manufacturers are not providing the source code, nor the specifications needed for someone to write the a driver.

Many winmodems have binary-only drivers available.  The multiverse repository is there to host packages like that.  It is not for  ideological reasons that you have to compile your driver, if there is one available, it is for technical reasons.  Some drivers are only available for older linux kernels.

Instead of complaining to Ubuntu, file a bug report or add to an existing one regarding your modem.  If you really want to complain, do it to the manufacturer.  Who else do you think will provide you with one?
