My apologies for not coming up with a better title, but it's hard to explain what I want to accomplish in just one simple sentence.

If you've ever used Vine VNC on Apple OS X, you know what I'm talking about.  With fast user switching turned on, you can connect remotely via VNC on one account in the background while another account is currently being used locally.

For example, my son is on his account at home, but I want to connect via VNC to my account (on the same box) from work at the same time, in the background.

This was easily done on OS X with Vine VNC.  The only drawback was the account needed to be already logged in in order to connect.  What was also cool was that I could set up a VNC server for each account, so I could connect to different accounts on the same machine.  It worked really well for the most part.

But, on Ubuntu, if I try to connect to an account while it's in the background, I'll get some funky lookin' colored static.  And, the only way to be able to connect again is to have someone physically switch back to the account I want to connect to.

It seems like there are some alternative VNC servers out there besides the built-in one that might be able to accomplish this, but they all seem extremely intimidating and the threads that I saw on how to install them don't seem to be updated for Hardy.  

So, has anyone found a solution for this that works on Hardy? 

Thanks in advance.
