### Post by MrWestwood on 2011-09-05
> **Alan Stainer said:**
> I can ping the IP address you mentioned, but I get unknown host for  [www.google.com]("http://www.google.com")


Could be the DNS server. My router is my IP address for my DNS. 

if you edit /etc/network/interfaces

you can add the line 

nameserver <IP of router or DNS server>


E.g.

```
auto eth1
iface eth1 inet static
address 192.168.0.229
netmask 255.255.255.0
gateway 192.168.0.1
```should now be 

```
auto eth1
iface eth1 inet static
address 192.168.0.2
netmask 255.255.255.0
gateway 192.168.0.1
nameserver 192.168.0.1
```you can also make the change in /etc/resolv.conf

```
nameserver 192.168.0.1
```Don't forget to restart network afterwards

```
sudo /etc/init.d/networking restart
```Hope this helps.

### Post by Alan Stainer on 2011-09-05
I found the problem. resolv.conf was being overwritten every time the server rebooted, so it was losing the nameserver entry for my ADSL gateway. I have now write protected the file to stop it from being overwritten.
