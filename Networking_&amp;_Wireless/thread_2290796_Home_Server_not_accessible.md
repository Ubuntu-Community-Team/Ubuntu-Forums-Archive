### Post by TheFu on 2015-08-15
Seems like you know about the parts, but are missing some important details.

Is your connection with a static IP or DHCP from Comcast?  That matters.
Is all the networking working today? Can the machine be used to talk on the internet and back or not? Can you **ping google.com**?

While we wait for your answer, a few comments.  I've been running internet servers at work and home since the mid-1990s.

a) I would NOT put a server directly on the internet. That is overly dangerous. **Always use a router and setup filters** so that only the few ports you need open are actually open.

b) I would NOT set a public IP on the server. It needs a static IP on the LAN and the router should forward ports to the static IP on the LAN from the WAN. The WAN **can** have a DHCP address, provided the dynamic DNS service is setup and working correctly. The easiest way to accomplish that is with the built-in Dynamic DNS settings from the router. Most routers have a few different dynamic DNS services in a list.  Of course, you can always use the IP address .... but be aware that some home routers think that access to the WAN IP is an attack, so you may need to disable that protection.

c) I think your public IP in the "interfaces" file is wrong. The **auto** needs to come before the static stanza. Plus I think you need some specific routing entries - because the routing table isn't showing it.  If you change the method used and add a router, this will go away and life will be easier.

d) I would not setup or use any webgui to manage a server. That just provides another attack vector into the system. Sure, you can secure it correctly, but most people who need these webguis are not skilled enough yet to properly secure it. [http://blog.jdpfu.com/2011/10/12/web-based-administrative-tools](http://blog.jdpfu.com/2011/10/12/web-based-administrative-tools) has some ideas to secure it, if I can't convince you to stop using it. 

Anyway, I hope that is helpful and away your answers.
