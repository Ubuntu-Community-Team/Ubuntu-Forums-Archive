### Post by viralnexxus on 2009-05-20
IMO MLdonkey is more powerful and full featured, but requires much, much more patience when customizing and setting it up; the reason being by default it does not come with a user interface, but you can easily download and add one if you wish.  Another noted feature of MLdonkey is that it runs as a service in the background using very little system resources and seeing as how you want to use Ubuntu as your server it would make sense not having it drain your resources.  A con or pro, depending on how you view the command-line and shell is that there is a massive list of commands for using MLdonkey; but you don't have to memorize them all and in case you forget, I think the command is something *longlist* in the prompt.  Anyway, they are both good so it will just come down to preference in the long run.  If you have any questions about any of this, feel free to drop me a line and I will try to help the best I can.  K?  Cool. GL

I use amule as a service. You just have to install amule-daemon packet in ubuntu. It takes the configuration from amule and you can easily connect to amulegui.
If you are just going to use ed2k network, amule is the simplest to configure. And if you'd happen to have two partitions, emule can read with no problem amule's temp files.

I don't know which one is better, but I tried mldonkey and I can say that amule is at least the simplest and easiest to install and setup.
