### Post by habana on 2017-11-13
I have two computers running 17.10. Computer1 is a clean install, computer2 is an upgrade from 17.04. After setting up a shared folder on both computers (all boxes ticked) I can access computer1 files from computer2. On computer1 I can see the shared folder I have set up on computer2 but when I try to access its files I am presented with the "password required" window. I enter the details but they are not accepted. User name and password are the same on both machines so what worked for one should work for the other. The permissions for the shared folders are "create and delete" for all users.

How do I fix this?

### Post by Morbius1 on 2017-11-13
Your [shared] share on computer2 requires a password. If the user name and password on both machines are the same dis you add yourself to the samba password database on computer2:
```
sudo smbpasswd -a bill
```
