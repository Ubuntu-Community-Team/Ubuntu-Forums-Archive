### Post by noob12 on 2007-09-11
I was just asking you to try to put eth1 down to test whether that was part of the issue.

If you've removed network manager, you *will* want a section in /etc/network/interfaces for each interface that you plan to use.  It will be read by **ifup** and **ifdown** to configure the interface.  

Based on what you had before you were using **eth0**, so you will want the stanza for that  in your /etc/network/interfaces and you will also want the stanza for **lo **(the loopback interface).

You can leave out interfaces you don't plan to use ever.  For interfaces that you plan to keep unplugged most of the time but might use at times, you can just leave out the **auto** line.  This means the interface will stay down unless you run **ifup** specifically for it.

Once you've done this, and you are running with eth0 UP (and plugged in) and lo (loopback) UP, you should see just the eth0 routes in **route -n**. 

At that point, If things are still slow I don't have more guesses about what is going on.
