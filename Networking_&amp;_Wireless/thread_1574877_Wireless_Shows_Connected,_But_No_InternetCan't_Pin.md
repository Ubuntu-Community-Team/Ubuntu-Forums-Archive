### Post by dineshs on 2010-09-17
This is what I have learned.
In older linux versions IP configuration was done by editing [COLOR="SeaGreen"]/etc/network/interfaces[/COLOR] and DNS in [COLOR="SeaGreen"]/etc/resolv.conf[/COLOR] (GUI method [https://help.ubuntu.com/community/NetworkAdmin](https://help.ubuntu.com/community/NetworkAdmin))
Latest versions use Networkmanager .The file is found in [COLOR="SeaGreen"]/etc/NetworkManager/system-connections[/COLOR] forexample as [COLOR="SeaGreen"]Auto eth0[/COLOR]
Since you were able to ping 4.2.2.1 (an open DNS)but not google.com a DNS issue was suspected and using an open DNS worked.You can try any DNS(open or ISP maintained or your gateway IP ie 192.168.2.1)
