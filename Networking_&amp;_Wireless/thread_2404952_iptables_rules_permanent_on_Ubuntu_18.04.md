One thing comes to mind though: interface tun0 may not exist as the rules get loaded (tun0 may get created later) Perhaps you should configure these commands in the openvpn configuration file so that openvpn runs them when it connects. Create a script that contains those iptables commands (e.g. something like /etc/openvpn/tun0-up.sh) like this:

```
sudo iptables -I FORWARD -i tun0 -j ACCEPT
sudo iptables -I FORWARD -o tun0 -j ACCEPT
sudo iptables -I OUTPUT -o tun0 -j ACCEPT
sudo iptables -A FORWARD -i tun0 -o eth0 -j ACCEPT
```
Make it executable, and add this line to your openvpn configuration file:
```
up /etc/openvpn/tun0-up.sh
```
