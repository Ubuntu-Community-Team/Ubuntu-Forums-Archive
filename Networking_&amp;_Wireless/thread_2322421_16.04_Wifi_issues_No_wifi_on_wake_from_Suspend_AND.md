### Post by xmbwd on 2016-04-27
So there appear to be major issues with 16.04 and wifi.  Just take a look at [these search results]("http://askubuntu.com/search?q=16.04+wifi") from [askubuntu.org]("http://askubuntu.org").  I would love to start a wiki on this problem, as it appears to exist across network cards and computers.  I'm on an Asus with the Intel 7260 chip.

My issues are twofold.  One I haveresolved.  The other I have not.  

**1) The firs**t issue is that wifi dies after suspend.  The message from the applet is ***device not ready*** and no network connections are available, which suggests that the driver is not loading after the computer wakes from suspend.  I was able to find [a solution to this problem here]("http://askubuntu.com/questions/761180/wifi-doesnt-work-after-suspend-after-16-04-upgrade").  After activating the suggested module, I also issued this command after rebooting to get the wifi to work after waking from suspend:

```
sudo service network-manager restart
```

This fix is persistent both in terms of suspend/wake cycles and reboot cycles.  

**2) The second** issue is that wifi dies whenever I switch wifi networks.  It presents as the same failure as the suspend issue: the applet says ***device not ready*** and no network connections are visible.  I can temporarily fix the problem by issuing the same network-manager restart command above.  

But this is a half-fix, and I would like to have a permanent one.  Attached are the results from the wifi script, one with the wifi working and one with the wifi dead after switching networks:

### Post by mdmoreau on 2016-05-13
I was having the same problem since installing 16.04, but I think I finally figured it out.  I noticed the firmware 17 error message too, so I thought I'd try to get that installed.  I didn't find it on the Intel site, but you can grab it from [https://github.com/OpenELEC/iwlwifi-firmware/blob/master/firmware/iwlwifi-7260-17.ucode](https://github.com/OpenELEC/iwlwifi-firmware/blob/master/firmware/iwlwifi-7260-17.ucode).  If you place that in the /lib/firmware directory it should load version 17 on your next restart - I only had up to version 16 initially.  You can check which is loaded before/after the process by using:

```
dmesg | grep iwl
```

It's worked fine through restarts and suspends for me so far, so hopefully you'll have the same luck.
