### Post by tgalati4 on 2015-06-29
Welcome to the forums.

What modules are you running?

```
lsmod
```

If you are running the *r8169* module, that apparently does not work with your specific Realtek network card.  So you have to download the source code from Realtek, and compile the *r8168* module and install it.  It is painful, but doable.  Just follow the instructions step-by-step.

[https://unixblogger.wordpress.com/2011/10/18/the-pain-of-an-realtek-rtl8111rtl8168-ethernet-card/](https://unixblogger.wordpress.com/2011/10/18/the-pain-of-an-realtek-rtl8111rtl8168-ethernet-card/)

Post the output of:

```
cat /etc/modules
grep r8169 /etc/modprobe.d/blacklist.conf
```

Everything is fixable in Linux; patience is key.

### Post by tgalati4 on 2015-06-29
So let's flip the problem around.  Perhaps the *r8169* module works and the *r8168* does not.

```
sudo modprobe -r r8168
sudo modprobe -i r8169
sudo ifup eth0
ifconfig
```

It's possible that your network card is burned up.  Find the chip and put your finger on it.  If it is really hot, then that's a clue.

There is a debug switch, perhaps turn in on and look at the output:

> parm:           debug:Debug verbosity level (0=none, ..., 16=all) (int)

```
sudo modprobe -i r8169 debug=3
```

### Post by praseodym on 2015-06-30
You need nameservers in your /etc/resolv.conf
```

sudo modprobe -rfv r8169 r8168
sudo modprobe -v r8168
echo "nameserver 8.8.8.8\nnameserver 8.8.4.4" | sudo tee -a /etc/resolv.conf
sudo service network-manager restart
```
