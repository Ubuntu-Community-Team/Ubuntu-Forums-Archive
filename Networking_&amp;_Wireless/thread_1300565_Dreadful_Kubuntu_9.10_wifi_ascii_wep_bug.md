### Post by opensourcederry on 2009-10-25
Hi all,

I use the term 'dreadful' as the Kubuntu team are well aware of it, and also as it is a few days until release if they don't get it sorted then this networking forum will possibly go into overdrive.  Or maybe not lol.

I installed what is likely to be the final RC last night and it still exists there.

Basically, the issue is that if you use wifi with a wep ascii key, the plasma network manager does not provide ascii as an option when entering the key, only hex / passphrase.

This is well documented and reported for quite a while and as such a fundamental bug, it seems amazing that it has been not addressed/sorted.

There are workarounds involving python terminal commands to convert ascii to hex.  Admittedly these work but this is no option for a newbie or for someone who has used wep/ascii perfectly < Kubuntu 9.10

google below and you'll get the drift

kubuntu 9.10 wep ascii
