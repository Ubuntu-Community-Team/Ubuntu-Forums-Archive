### Post by marv_223 on 2010-02-16
Open-DNS.org maintains a list of sites that are DNS servers, or are lists of DNS servers.

Their URL is 

[http://www.open-dns.org/](http://www.open-dns.org/)

On that page, they have a link to their list of open DNS servers, which is at 

[http://www.dnsserverlist.org/](http://www.dnsserverlist.org/)

(The reason to go to [http://www.dnsserverlist.org/](http://www.dnsserverlist.org/) first is to read a little about the organization, etc.)

When you go to the dnsserverlist URL, you will get a couple of IP addresses for dns servers that are geographically close to the IP that you are using.

If I remember correctly, there is also a utility on that page that let's you input a physical location and then it provides a couple of open dns servers that are in the vicinity of the physical location.

sudo vi /etc/resolv.conf will open the file.

ping and host were working fine for me, but the package manager and mozilla were lost.  I replaced the last nameserver in my list with the IP that I got from [http://www.dnsserverlist.org/](http://www.dnsserverlist.org/), and a miracle happened.  "The update manager could now see!" 

By the time I rebooted when the update manager was finished, NetworkManager had deleted my entry, but I'm operational.  

Marv

### Post by mkormendy on 2010-02-18
Problem fixed, I removed the resolvconf package, using this command:[B]

sudo apt-get remove resolvconf[/B]

I then added my nameservers to resolv.conf file, saved it, then rebooted.

No more rewrites of my resolv.conf file.

Thanks for leading me in that direction!

Cheers!
