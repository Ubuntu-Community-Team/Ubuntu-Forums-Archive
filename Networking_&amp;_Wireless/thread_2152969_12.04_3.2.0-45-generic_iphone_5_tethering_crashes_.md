### Post by RichLouth on 2013-06-09
When connecting my iphone 5 via usb, the tethered internet connection will work for a time but eventually will suddenly stop working and may cause a system crash.

If it stops working and I disconnect my phone, by unplugging the usb, and reconnect it the tethering will continue to work.

Eventually though, after one or several reconnections, the whole system will crash. The system becomes unresponsive to both the mouse and the keyboard. If sound is playing at the time it will continually loop over the last half second of playback.

At first I thought it was flash but I have had this occur without flash being involved (I have a flash blocker installed).

I assume the problem is with the kernel module supporting the iphone 5's usb connection.

I'm guessing there's some poorly handled state going on.

Which log do I review to identify the problem?

Is this a known bug and is there an existing kernel patch that I can apply?

Not really relevant, but there are no issues with the tethered connection or its speed when used in osx 10.7.

Thanks in advance.

### Post by RichLouth on 2013-06-09
Also, something I've noticed.

It seems to happen when there is heavy load on my hdd's. Like when running Minecraft (It has happened without Minecraft or Java running).

As though it panics when it is asked to wait for another write opperation to finish.
