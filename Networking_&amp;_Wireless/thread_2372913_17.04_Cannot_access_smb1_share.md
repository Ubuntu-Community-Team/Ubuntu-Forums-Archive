### Post by TheFu on 2017-09-30
I don't have the answer. Sorry.

SMB1 is highly, highly, highly non-secure. Using an ASUS router for storage access AND WAN routing is asking to be hacked, IMHO.  Asus has a history of major issues with the security of their routers.  
* 2014 - [https://arstechnica.com/information-technology/2014/02/dear-asus-router-user-youve-been-pwned-thanks-to-easily-exploited-flaw/](https://arstechnica.com/information-technology/2014/02/dear-asus-router-user-youve-been-pwned-thanks-to-easily-exploited-flaw/) 
* 2016 - [https://www.consumer.ftc.gov/blog/2016/02/got-asus-router-home-read](https://www.consumer.ftc.gov/blog/2016/02/got-asus-router-home-read)
In short - beware.  BTW, any Linux-based router that hasn't been patched since March 2017, has a remote root access problem.  This means the router can be trivial pwn'd from the internet.  Be afraid.

Samba has a client-side and a server-side.  The smb.conf controls the server side.  The Asus system is the Samba server - probably unpatched and really, really, really, old.  There are a number of client-side smb access tools. Each will use a little different client library based on what the specific project team. Each of those project teams will decide which defaults that want.  So, if you use the smbclient program, the defaults will be different than if you use the gnome gvfs methods or any other GUI that leverages those methods. 
[https://askubuntu.com/questions/919967/how-to-tell-gigolo-gvfs-to-use-smbv2-for-windows-shares/926451](https://askubuntu.com/questions/919967/how-to-tell-gigolo-gvfs-to-use-smbv2-for-windows-shares/926451)
