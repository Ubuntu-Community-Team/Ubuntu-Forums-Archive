### Post by aljames2 on 2023-06-04
Do you have the bridge set up on your network?  This would be done either using Network Manager if your host is a desktop, or if you are running Ubuntu Server as your host then this is done using a netplan YAML file.  The bridge must be built before you can ride on it.

### Post by Doug S on 2023-06-04
In addition to having the bridge setup on the host, you need to tell the host kvm/qemu stuff about it.

I left myself the following notes last time I did this on 20.04:

```
doug@s19:~/config/etc/libvirt/qemu/networks$ cat README
2021.11.19. Due to file permissions and other setups, this one is not so easy to backup.
See also: https://ubuntuforums.org/showthread.php?t=2461631&p=14036896#post14036896

doug@s19:~/config/etc/libvirt/qemu/networks$ ls -l /etc/libvirt/qemu/networks
total 24
drwxr-xr-x 2 root root 4096 Apr 30  2021 autostart
-rw------- 1 root root  576 Apr 23  2021 default.xml.original  <<< Saved, just in case we want to go back
-rw------- 1 root root  383 Apr 30  2021 host-bridge.xml       <<< Autogenerated via below commands
-rw-r--r-- 1 root root  104 Apr 30  2021 host-bridge.xml.original <<< Saved original file.


Commands:

doug@s19:/etc/libvirt/qemu/networks$ sudo cp bridge.xml.original host-bridge.xml
virsh net-define host-bridge.xml
virsh net-autostart host-bridge
virsh net-start host-bridge
virsh net-list --all

doug@s19:~/config/etc/libvirt/qemu/networks$ cat host-bridge.xml.original
<network>
    <name>host-bridge</name>
    <bridge name='br0'/>
    <forward mode="bridge"/>
</network>
```

I have never heard of having to use GATEWAY_MAC.
