### Post by theDaveTheRave on 2008-12-09
Ouro,

for setting up of shared folders on a server machine I have done the following.

I created a user called <guest>

I then simply shared the whole of guest's folder with everyone, I was then also able to have a single sign on for all my colleagues to use - i didn't need to worry about giving each a specific username / password.

It is then possible (although I haven't personaly done this as yet) to mount this shared area into each of the desktops that will access it - using the /etc/fstab file to tell each desktop where to find the directory and where to place it on each system. I have performed this however with some large disk storage space to ensure that it looks like it is part of the shared folders.

you could potentially share the whole <etc/> folder to the host machines so as you only need to perform any changes / updates once... I'm not sure how this would set up though??

As each of the users have read / write access to the shared folder system they can add /  create folders and docs to thier hearts content.



If some of your users are on Windows systems you can mount the shared drive using the <net use> command (you'll need to look this up on the net to make it persistent on logon as you need to edit the "autoexec.bat" file or similar??? its a while since I have done this... ! ;)

I realise that this ins't exactly helpful (in terms of a "howto"), but should give you some ideas for solutions that you can search for on the forums, where the advice would be better than if I gave it too you.

for samba setup look at Stormbringer's tutorial [here]("http://ubuntuforums.org/showthread.php?t=202605") I'm using it all the time

for fstab info look at BodhiZazen's [tutorial]("http://ubuntuforums.org/showthread.php?t=283131")

Hope that they help, they certainly did for me.
