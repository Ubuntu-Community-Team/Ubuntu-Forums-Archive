### Post by chili555 on 2013-02-09
> product: [COLOR="Red"]BCM4311[/COLOR] 802.11b/g WLAN
vendor: Broadcom Corporation
physical id: 0
bus info: pci@0000:0c:00.0
version: 01
width: 32 bits
clock: 33MHz
capabilities: pm msi pciexpress bus_master cap_list
configuration: [COLOR="Red"]driver=wl[/COLOR] latency=0
resources: irq:17 memory:f9ffc000-f9ffffffHere comes a rant, so buckle up. It is amazing the downright faulty fixes there are out there on the internet. The Broadcom STA driver is incorrect for your device. It blacklists the module ssb which is required by your ethernet. Moreover, the STA driver doesn't operate your wireless. Great fix!!!

End rant. Whew, I feel better now.

Please open a terminal and do:```
sudo apt-get remove --purge bcmwl-kernel-source
sudo modprobe b44
```Now your ethernet should be working. Hook up to the router and do:```
sudo apt-get install linux-firmware-nonfree
sudo modprobe b43
```Now your wireless should be working.

For the benefit of the searcheers, which driver is correct for which Broadcom device is very specific. You can't throw any old driver at any old Broadcom and expect it to work. Please check here: [https://help.ubuntu.com/community/BroadcomSTA%28Wireless%29](https://help.ubuntu.com/community/BroadcomSTA%28Wireless%29)

### Post by visierl on 2013-03-29
Along the same lines, I found that I could not remove bcmwl-kernel-sources while running multi-user, so I booted into recovery mode, fsck'ed my file systems and removed it in a root shell.  That worked fine, but the modprobe b44 hung, so I rebooted back to multi-user mode and my networks were fixed.  So, if you have trouble getting the above suggestion to work for you, try just removing bcmwl-kernel-sources in recovery mode and then rebooting as normal.
