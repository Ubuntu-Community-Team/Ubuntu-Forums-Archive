### Post by dallas7 on 2008-04-27
I have seen this problem on several of the older (P3, early P4) systems where networking breaks on 8.04 upgrades, or it can't be made to work on clean installs or running off the CD.

The easiest way to view this issue is to do an ifconfig on your nic like "ifconfig eth0" (your nic device may be different).

Normally, it would look like this:
eth0      Link encap:Ethernet  HWaddr 00:50:8B:FB:EA:53  
          inet addr:192.168.0.7  Bcast:192.168.0.255  Mask:255.255.255.0
          inet6 addr: fe80::250:8bff:fefb:ea53/64 Scope:Link
          UP BROADCAST RUNNING MULTICAST  MTU:1500  Metric:1
          RX packets:11964 errors:0 dropped:0 
          etc.

The problem looks like this:
eth0      Link encap:Ethernet  HWaddr 00:50:8B:FB:EA:53  
          UP BROADCAST RUNNING MULTICAST  MTU:1500  Metric:1
          RX packets:11964 errors:0 dropped:0 
          etc.

(Can't see it?  Hint: the inet lines are missing!)

Yet, the resolve.conf and interfaces files will be correct.

ifconfig up and down commands don't return errors.
But other ifconfig commands to set ip, broadcast, etc. may return errors like:
SIOC... no such device
SIOC... no such process

Ping your router and you'll get "Network is unreachable." 

It's my opinion if the problem exists on a system, there is no fix - go back to or stay with 7.0x.

I have viewed a multitude of recent postings here presenting ifconfig output like the one above (with the missing inet lines) but without reading every thread here from the last three days, it seems the Big Beaners here haven't caught on yet.

Before upgrading, run 8.04 as off the CD and find out if networking is OK by examining your ifconfig output.  Well, and getting on the network.

Otherwise, stick with Gutsy.
