### Post by kd7swh on 2007-11-06
if you are using gusty, (the 0.6.5 version of the pptp plug-in) it is hard-coded to only work with the eth0 interface.

change your ppp0 device to eth0 in '/etc/udev/rules.d/70-persistant-net.rules' and the pptp apps should begin working

### Post by kd7swh on 2007-11-06
My mistake on the file name. In your case if you want to get your wireless card to work with your vpn you should change this:

```
# This file was automatically generated by the /lib/udev/write_net_rules
# program, probably run by the persistent-net-generator.rules rules file.
#
# You can modify it, as long as you keep each rule on a single line.

# PCI device 0x10ec:0x8139 (8139too)
SUBSYSTEM=="net", DRIVERS=="?*", ATTRS{address}=="**:**:**:**:**:**", NAME="eth1"

# PCI device 0x8086:0x4223 (ipw2200)
SUBSYSTEM=="net", DRIVERS=="?*", ATTRS{address}=="**:**:**:**:**:**", NAME="eth0"
```

to this:

```
# This file was automatically generated by the /lib/udev/write_net_rules
# program, probably run by the persistent-net-generator.rules rules file.
#
# You can modify it, as long as you keep each rule on a single line.

# PCI device 0x10ec:0x8139 (8139too)
SUBSYSTEM=="net", DRIVERS=="?*", ATTRS{address}=="**:**:**:**:**:**", NAME="eth0"

# PCI device 0x8086:0x4223 (ipw2200)
SUBSYSTEM=="net", DRIVERS=="?*", ATTRS{address}=="**:**:**:**:**:**", NAME="eth1"
```

See all I did was switch your eth0 to eth1 and eth1 to eth0.

the pptp plugin and pptpconfig only work with eth0 so now that your wireless card is eth0 in '70-persistant-net.rules' it will work. 

you may have to reboot after making the change for it to take effect.
