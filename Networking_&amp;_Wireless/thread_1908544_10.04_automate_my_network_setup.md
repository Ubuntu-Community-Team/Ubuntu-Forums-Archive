### Post by User#32423 on 2012-01-13
Currently I need to manually setup my wireless network each time I boot up. I wish to automate it so that the settings are correct on boot up. I have the following /etc/network/interfaces file:

auto lo
iface lo inet loopback

auto wlan0
iface wlan0 inet static
wireless-mode ad-hoc
wireless-channel 4
wireless-essid wifinet
wireless-key xxxxxxxxxx
address 192.168.0.1
network 192.168.0.0
netmask 255.255.255.0
broadcast 192.168.0.255

I cannot ping my 192.168.0.5 laptop until I run the following script as root:

#! /bin/bash

ifconfig wlan0 down
iwconfig wlan0 mode ad-hoc
iwconfig wlan0 channel 4
iwconfig wlan0 essid wifinet
iwconfig wlan0 key xxxxxxxxxx
ifconfig wlan0 up
ifconfig wlan0 192.168.0.1

Now I can ping 192.168.0.5 laptop.

what am I doing wrong in my interfaces setup?

### Post by jsra on 2012-01-13
Hi,
I dont know whats wrong with your settins and I can't test it tonight but you can put that script in ```
/etc/init.d/
```
and then you can use update-rc.d so this script will be executed on boot.
```
update-rc.d script.sh defaults
```
It's not a fix for wrong settings but it is a temporary solution that will work:)

### Post by jsra on 2012-01-15
update-rc.d: /etc/init.d/script.sh: file does not exist

obviously it says that the file script.sh does not exist.

Change script.sh to name of your script in that command, eg. if your script name is wifi.sh.
```
sudo update-rc.d wifi.sh defaults
```
