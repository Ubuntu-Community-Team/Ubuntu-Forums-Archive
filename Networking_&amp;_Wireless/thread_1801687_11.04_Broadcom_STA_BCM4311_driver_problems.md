### Post by chili555 on 2011-07-11
Well, as long as I've been invited...

I think you have three (!!!) problems. The first may be that the installation of bcmwl-kernel-source often blacklists b43 which, I believe, is correct for your device. Let's check:```
ls /etc/modprobe.d | grep b4
```Any files listed? Remove them:```
sudo rm /etc/modprobe.d/file_you_found
```Next do:```
cat /etc/modprobe.d/blacklist.conf
```Any listings for b43 or ssb? If so, remove them with:```
sudo gedit /etc/modprobe.d/blacklist.conf
```Be sure to proofread carefully, save and close gedit.

Next, let's fix this:> $ rfkill list all
0: hp-wifi: Wireless LAN
Soft blocked: no
Hard blocked: yesIn some cases, the module hp-wmi *thinks* the wireless is blocked by the key combination, for example, on my laptop Fn+F5. The symptom is that you press your key combination and nothing happens. In one case I solved a few weeks ago, it turned out that, instead of Fn+F12, it was Alt+F12 that turned on his wireless. Try all the combinations you can and see if the wireless light comes on. Try Alt, Ctrl, Fn and all the F keys from F1 to F12 until it works.

If it never works, post back and we'll dig deeper.

Finally, some have reported that the module b43 won't load automatically on boot. Please try:```
sudo modprobe b43
sudo lshw -C network
``` Does this change?> [sudo] password for rick:
