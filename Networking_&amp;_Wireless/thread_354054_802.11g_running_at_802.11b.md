After some digging, I got my wireless card (truemobile 1300) up and running using the firmware update (not the ndiswrapper).  So, now I can surf wirelessly. ### Post by spd106 on 2007-02-05
The bcm43xx driver defaults to 11mbps a the moment. I'm not sure why this is so, but I suspect that higher data rates mean less stability. You can set the data rate manually by using the iwconfig tool. **sudo iwconfig eth1 rate 54** should work.

The ndiswrapper driver is set to 54mbps as default, though I can't say that I've compared the speed of the two drivers against each other. I find ndiswrapper to be more stable and drops only very rarely compared to almost hourly drops with bcm43xx. Things will probably change when the new dscape stack lands in the kernel.
