### Post by Groodles on 2009-11-01
So I was running 9.04 and my /etc/network/interfaces file looks like this;

```

# The loopback network interface
auto lo
iface lo inet loopback

# The primary network interface
auto eth0
iface eth0 inet static
        address         192.168.1.2
        netmask         255.255.255.0
        broadcast       192.168.1.255
        gateway         192.168.1.254

```

At reboot the eth0 interface would be set to the static ip address and everything was fine.

I upgraded to 9.10 (/etc/network/interfaces is the SAME as before) and on reboot eth0 is being assigned from the network DHCP server.  If I then perform a

```

sudo /etc/init.d/networking restart

```

etho is then assigned with the correct static ip as defined in /etc/network/interfaces file.

Wierd?

### Post by Groodles on 2009-11-03
I found the problem.

For some reason, during the upgrade from 9.04 to 9.10, "NetworkManager" was installed.  (I dont use this, never have and prefer to do things from the /etc/network/interfaces file instead.)

Basically, what was happening was that eth0 would be initialised according to the configuration of /etc/network/interfaces and then later in the boot process NeworkManager would be started and it would try to configure eth0 the way it thought best (ie DHCP).

```

sudo apt-get remove network-manager network-manager-gnome

```Solved the problem for me.  NetworkManager is useful for laptops that switch wireless networks all the time, for a static ip machine it has no value as far as I'm aware.

### Post by pjhsv on 2009-11-14
> **abdusamed said:**
> im tryin to achive static ip...

by runnin this command
```
nano /etc/network/interfaces
``` .i typed this up
```

iface lo inet statoc

```

Okay, you've got a few things wrong here.

1.  You're editing the lo interface, which isn't what you want to do.  You want to be editing the eth0 interface.
2.  Your netmask, broadcast and gateway addresses aren't correct.  They're external addresses.  For your ethernet connection, they need to be internal addresses (192.168.1.x range).  Your netmask should probably be 255.255.255.0 for a /24.

Try something like :

iface eth0 inet static
address 192.168.1.10
netmask 255.255.255.0
broadcast 192.168.1.255
gateway 192.168.1.1 (or whatever the internal IP if your router is - the number you use to connect to it)
