### Post by DjSesso on 2010-07-14
> **gordintoronto said:**
> Most people use static addresses below 100. I doubt if it's important, but have you tried 192.168.1.10?

Are all the computers in the same workgroup?

Is Apache installed on the server? If not, that's why the other computers can't http to the server.

Its because most routers are setup to only give out 50 or 100 addresses. Since for some reason you are using static... Im assuming because you dont ever want the IP to change because you are using it as a server. If your using a Linksys router then the .222 addy is out of the factory default range of addresses given by Dhcp. 

So my guess would be that your router doesnt know what the .222 address is even though you assigned the server to it. Change your IP to maybe .30 unless your planning on having over 30 nodes. 

If it still doesnt work then turn off Iptables and try it without it running. ;)

### Post by YesWeCan on 2010-07-15
That's a good point. I am not expert on routers but it wouldn't surprise me if the right hand doesn't know what the left hand is doing. I think that when the server sends internet bound packets to the router the router notes the source MAC address and then replies get forwarded to that noted MAC address. But this is probably a different process to what happens if another PC on the same wlan pings the IP address of the server. I believe the PC needs to ask the LAN who has the server's IP and then wait to receive a mac address.
