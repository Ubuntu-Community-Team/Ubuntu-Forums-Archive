### Post by Morbius1 on 2018-08-16
All of the samba share applications have either been abandoned by Ubuntu or are destructive. I have a suggestion that will allow you to create a share of a folder through Thunar if you are interested:

Go to: Thunar > Edit > Configure Custom Actions > +

For a Public Read Only share:

_**Basic Tab:**_
Name: **SMB Public R/O**
Command: ```
net usershare add %n %f "" Everyone:R guest_ok=y
```
_**Appearance Tab**_:
Make sure only Directories are checked

Then you can reopen Thunar, right click on a folder, and select **SMB Public R/O **to create a samba share of that folder.

You can make as many of these as you want:

A Public R/W share ( **SMB Public R/W** ):
```
net usershare add %n %f "" Everyone:F guest_ok=y && chmod 777 %f
```
A Private R/O Share ( **SMB Private R/O** ):
```
net usershare add %n %f "" Everyone:R guest_ok=n
```
A Private R/W share ( **SMB Private R/W** ):
```
net usershare add %n %f "" Everyone:F guest_ok=n && chmod 777 %f
```
And finally to remove the sharing from a folder ( **SMB Remove Share** ):
```
net usershare delete %n && chmod 755 %f
```

Takes a bit of work to set up but then you can carry it over to any new install by copying the /home/username/.config/Thunar/uca.xml file to the new install.

[COLOR=#0000cd]**Note**: These are Samba Usershares you will be creating so they will not show up in /etc/samba/smb.conf but in /var/lib/samba/usershares.[/COLOR]

### Post by Morbius1 on 2018-08-17
smb.conf is still in control of all usershares but only at the [global] level. So If you created a writeable guest share for example and want it so the file added by the client is editable by the Linux user you would as on option use something like this in the [global] section of smb.conf:
```
force user = simpos
```

### Post by Morbius1 on 2018-08-19
> **simpos said:**
> But now that everything works, about what you mentioned, what is uca.xml? Does it save time by configuring?
In Thunar all of these context menu items - like the ones you added for samba - are contained in a file: uca.xml.

You can view them all by running this command:
```
cat $HOME/.config/Thunar/uca.xml
```
What I do when I move on to a new install of another XFCE operating system is copy that file and replace the default uca.xml file that came with the install with the one I created on the old system when I used "Configure Custom Actions".

It's not a requirement it's just something that might save some time with a new install.
