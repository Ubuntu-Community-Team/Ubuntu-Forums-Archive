It wasn't (past tense) the fstab file, it was the symlink for Samba that broke it. I had it fixed earlier this morning, but just rebooted again to make sure it would work through two boots. It's now flawless.

```
sudo mv /var/lib/samba/secrets.tdb /var/lib/samba/secrets.tdb.orig
sudo ln -s /etc/samba/secrets.tdb /var/lib/samba
```These lines are listed in the directions here to get your computer working with Active Directory (the link up above, in an earlier post). I used unlink to break the symlink, then restored the Samba secrets.tdb that had been backed up, rebooted the machine, and everything worked perfectly. I believe (again, with no proof) that forcing it to rely on the accounts database that Likewise-open imported caused a glitch. Maybe a user ID was wrong, maybe a password was stored incorrectly - whatever it was, going back to the original Samba file fixed it.

I don't know if there's someplace I can report that? Because following all the steps here broke it, maybe there's a problem with the documentation. Or maybe it was unique to my machine/network. <shrug> I just don't know.

Edited to add:
Just in case anyone else comes in here looking for what I did, and are noobs enough to not know (it took me a while to find it) the commands to counter those up above are:
```

sudo unlink /var/lib/samba/secrets.tdb
sudo mv /var/lib/samba/secrets.tdb.orig /var/lib/samba/secrets.tdb

```
Enter them in that order.
