### Post by SunIT on 2008-05-01
I am very new to Linux but decided to give Ubuntu a shot on a production computer. Basically the idea is for the Ubuntu computer to automatically login, connect to the wireless network, and then open a VNC view of another computer. Everything is working great except for the wireless connection on login (not bad for a first Linux install).

Setup:
OS: Ubuntu 7.10
Wireless Card: Linksys WMP54G (4.1)
Encryption: WEP hex 128-bit

As I stated, the autologin and VNC script work fine but when trying to connect the the wireless I am prompted with a box that says "Unlock Keyring: Enter password for default keyring to unlock. The application 'nm-applet' (/user/bin/nm-applet) wants access to the default keyring, but it is locked" Manually doing an ifdown/ifup restores the wireless connection but it needs to start without any input from the user.

Any help would be greatly appreciated!

### Post by luckyuser on 2008-08-31
another fix, and a bit less complicated, is here:
 [http://ubuntuforums.org/showthread.php?p=5698832#post5698832]("http://ubuntuforums.org/showthread.php?p=5698832#post5698832")

see:
"System>Preferences>Encryption and Keyrings
Under Password Keyrings click o 'login Automatically unlocked when user logs in.'
Click "Change Unlock Password"
Enter your old password, but leave the new password fields blank."
Click OK"
