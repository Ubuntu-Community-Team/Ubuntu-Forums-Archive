### Post by steeldriver on 2015-01-15
Hello and welcome to the forums

First thing to note is that if you have added a GUI desktop, you no longer have a "pure" server and it will likely have installed additional network services (NetworkManager) which may conflict with the settings in /etc/network/interfaces

For the nameserver, instead of editing /etc/resolv.conf directly, you should add a dns-nameserver or dns-nameservers field to your interface definition in /etc/network/interfaces:

```

auto eth0
iface eth0 inet static
address 192.168.67.111
netmask 255.255.255.0
network 192.168.67.0
broadcast 192.168.67.255
gateway 192.168.67.1
** dns-nameservers 192.168.67.1**

```

[Also note you also have a transcription error or typo in your netmask - too many octets]

BTW you should be careful about using plain `sudo` with GUI applications like gedit - use gksudo (Gnome), pkexec, or `sudo **-H**` - or use a terminal editor like nano

```

sudo nano /etc/network/interfaces

```

Notice what I bolded.  How come you get 192.168.**38**.128 when you use DHCP, but are trying to use 192.168.**67**.111 as a static IP?

This sounds to me like your network is actually using 192.168.38.xxx and not 192.168.67.xxx.  Unless there is a router forwarding one network to the other, or there are two class C networks on the same physical network with a gateway on 192.168.67.1, it isn't going to work.  For a class C network the first 3 octets have to be the same, and accessing IPs outside that range goes through the gateway/router.  (actually it's determined by the subnet mask, but for simplicity's sake we won't go there now).

What gateway IP do you get from DHCP?  Chances are it's 192.168.38.xxx, and that's why you can get to the internet when using DHCP.

Assuming your network really is on 192.168.38.xxx and you have a DHCP server, to set up a static IP you'll need to find out what range of IPs the DHCP server is handing out, and then pick a static IP outside that range.  So, for example, if the DHCP server hands out 192.168.100 and higher, you can use a lower number such as 192.168.38.90 and it won't conflict with any machine getting its IP from DHCP.

Where are you getting 192.168.67.xxx from?  Can you ping the gateway at 192.168.67.1?
