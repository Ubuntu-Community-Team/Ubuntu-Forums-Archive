### Post by tacantara on 2010-01-25
After searching this forum and Google, I have not found any useful information on this problem.  I installed Samba on my Ubuntu laptop, and managed to successfully access my Windows XP desktop (files and a shared printer).  While file access is going smoothly, I am having issues with printing.  Any document I send to the printer remains in the queue, and never print.  I've waited as long as 30 minutes, and no output.  The printer recognizes that there is a print job (lights flash, ink cartridges start to move in anticipation of printing), but no output.

On the XP end, when I open the printer folder to look at the print queue, I see the print job listed as:  Remote Download Document/Status: Printing/Owner: Guest/Pages: NA/Size: 64KB/1.0MB/Date & Time info/Port: USB001.  I've tried sending the test page from the Ubuntu printer box (that's the document referred to above) and a small text file, which is 92 bytes, but shows up in the XP print queue as 64KB/285KB.  No matter what I send, XP only shows 64KB received, and an overall file size that is way off (92 bytes < 283KB).

Until I can find a way to get actual output from the printer, I'm transferring anything that needs printing through Nautilus to the XP box.  From there, I can open the file and print.

Printer information:  HP DeskJet f2280.  I have connected this printer to my Ubuntu laptop and successfully printed and scanned with it.  I have also successfully printed with the printer hooked up to the XP box.  The problem lies in printing from Ubuntu, to XP, via Samba.
