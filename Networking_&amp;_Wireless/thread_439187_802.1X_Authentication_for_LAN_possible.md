### Post by spd106 on 2007-05-11
AFAIK this should be possible with wpa_supplicant, though you will have to modify some text files as this hasn't been integrated into network manager yet.

The wpa sticky will be helpful in setting this up, as will the wpa_supplicant home page [http://hostap.epitest.fi/wpa_supplicant/](http://hostap.epitest.fi/wpa_supplicant/)

As wpa_supplicant is on the live cd you should be able to test it with a modified conf file, though you will probably have to disable network manager first.

### Post by crumbfire on 2007-05-12
thanks. i tried it with the live cd and it worked right away. What I did was to edit /etc/network/interfaces and adjust the eth0 settings to this:

```

auto eth0
iface eth0 inet dhcp
wpa-driver wired
wpa-eap TTLS
wpa-key-mgmt IEEE8021X
wpa-anonymous-identity <here i just entered "none">
wpa-identity <youridentity>
wpa-password <yourpassword>
wpa-phase2 auth=MSCHAPV2
```

then I did a /etc/init.d/networking restart and immediately got the correct IP address...
