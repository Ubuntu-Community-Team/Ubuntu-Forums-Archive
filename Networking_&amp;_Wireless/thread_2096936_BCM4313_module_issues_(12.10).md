### Post by rwschneider1 on 2012-12-21
I have recently installed Ubuntu 12.10 on a Dell Latitude E6520 with a Broadcom BCM4313.  
```

Linux rubuntu6246 3.5.0-21-generic #32-Ubuntu SMP Tue Dec 11 18:51:59 UTC 2012 x86_64 x86_64 x86_64 GNU/Linux

```

After hours of digging through various threads, blogs, and documentation detailing how different people were able to get their BCM4313 to work on 12.10, I stumbled across the following commands, which worked like a charm after I had removed all other broadcom-related packages:

```

apt-get install --reinstall linux-headers-generic
apt-get install --reinstall bcmwl-kernel-source
rmmod wl
modprobe wl

```

The problem is that I need to perform these steps every time I reboot the machine, which is quite an inconvenience.  

Here is the output that is displayed when re-running the commands.  Shortly after the final *modprobe* command, the Network Manager icon indicates that it is negotiating a DHCP lease and connects to the local wireless network.
