### Post by TheFu on 2020-07-21
```
net.ipv6.conf.all.disable_ipv6 = 1
net.ipv6.conf.default.disable_ipv6 = 1
net.ipv6.conf.lo.disable_ipv6 = 1
```
**doesn't disable ipv6** on any recent Linux running systemd.  You'll need to use the correct method for the version of the OS that you have.  But it appears that nothing is listening on 80/443/tcp on IPv4.  Check the config file for the daemon that should be listening on those for a way to control which type of networking is used.  I use the GRUB option method.  We used to use sysctl and/or blacklisting the ipv6 modules, but that stopped working someone last year.
```
$ egrep -i ip  /etc/default/grub
GRUB_CMDLINE_LINUX_DEFAULT="ipv6.disable=1"
```
Be certain to run **sudo update-grub** after that change.

Is there a reason you don't say the name of the tool to get certificates?  I use acme.sh, after leaving the earlier version.  Since January, Let's Encrypt enabled pickier validations from places all over the world.  When it comes time to update my certs (every 2.5 months), my solution is to completely disable all firewalls before any attempt, otherwise the requests fail.  Also, before I disabled the firewalls, updates would take 90 seconds for each cert.  After disabling the firewalls, I think updating all 20-something certs takes less than 30 seconds total.  My certs don't expire for about 6 weeks, so in about a month, I get to go through this process again and will time everything.
