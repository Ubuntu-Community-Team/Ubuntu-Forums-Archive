### Post by freshtoast on 2011-04-01
Hello all,

I've managed to set up an openssh-server using a non-standard port, and I have a dynamic DNS service.

All works well, for example:
```
ssh -L 5900:localhost:5900 *username*@*hostname.org* -p 1223
```

So, in the pursuit of laziness I have set up a .ssh/config file as follows:
```
Host myserver
HostName *hostname.org*
User *username*
Port 1223
```

```
chmod 600 ~/.ssh/config
```

(~/.ssh/ is chmod 700)

Alas, when I wish to revel in my lazy glory and type:
```
ssh -L 5900:localhost:5900 myserver
```
I am faced with:
```
ssh: Could not resolve hostname myserver: Name or service not known
```

I've checked and I've double checked that I didn't make an error in the config file... but I'm thinking it might be something to do with permissions?

Any help gratefully received!
