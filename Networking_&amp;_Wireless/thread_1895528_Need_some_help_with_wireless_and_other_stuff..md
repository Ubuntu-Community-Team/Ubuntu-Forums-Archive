### Post by praseodym on 2011-12-15
Hi there,

please show the output of

```
lspci -nnk | grep -iA2 net
```
too. Did you install the Broadcom-STA driver via "additional drivers" because it was recommended? If so, you need to prevent several drivers from being loaded at boot-up, too:

```
gksu gedit /etc/modprobe.d/blacklist.conf
```
Add the following lines to this file:
```
blacklist bcma
blacklist brcmsmac
blacklist brcm80211
```
save, close the editor, and reboot your system.

### Post by praseodym on 2011-12-15
Try to reinstall all of the stuff:

```
sudo apt-get install --reinstall linux-headers-$(uname -r) build-essential dkms nvidia-current
```
Reboot and (re)activate the driver again in "additional drivers".

If this does not help, change with [COLOR="Red"]CTRL+ALT+F1[/COLOR] to the 1st terminal, login there and:

```
sudo service gdm stop
sudo nvidia-xconfig --composite
```
Check with
```
sudo nano /etc/X11/xorg.conf
```
if the driver is there in the "Section 'Device'" (if not add it like above), save and close the editor with [COLOR="Red"]CTRL+X[/COLOR] and
```
sudo service gdm start
```
