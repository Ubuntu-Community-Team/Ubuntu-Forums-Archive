I am not able to ping any machines on my network by their FQDN, but I can by their shortname. For instance:
```

[username@kawasaki ~]$ ping dns.internal.local
ping: unknown host dns.internal.local
[brhodes@kawasaki ~]$ ping dns
PING dns.internal.local (192.168.1.10) 56(84) bytes of data.
64 bytes from dns.internal.local (192.168.1.10): icmp_seq=1 ttl=128 time=0.126 ms

2 packets transmitted, 2 received, 0% packet loss, time 999ms
rtt min/avg/max/mdev = 0.126/0.133/0.140/0.007 ms

```

Here is /etc/resolv.conf:
```

# Generated by NetworkManager
domain internal.local
search internal.local
nameserver 192.168.1.10
nameserver 192.168.1.12

```

/etc/hostname:
```
kawasaki.internal.local
```

Any ideas? I cannot seem to figure this out. The only thing I can think of is that NetworkManager is messing something up, which wouldn't be surprising :)
