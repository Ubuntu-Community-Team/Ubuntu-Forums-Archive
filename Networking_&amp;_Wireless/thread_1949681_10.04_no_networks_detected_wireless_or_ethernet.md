### Post by dandnsmith on 2012-03-31
OK, that's progress
- the ifconfig shows that the ethernet port isn't being recognised and configured for use
- the lspci shows we have this RTL8101E/RTL8102E with the r8169 driver module.

I don't know about this particular model ethernet port, but others using the r8169 have had problems which vary in effect (mine went to sleep and were difficult/impossible to wake up).

My resolution with ub10.04, ub10.10, and ub11.04 was to replace the r8169 module with the r8168 (which worked for those with interfaces like mine) - but I've no idea if it would work for you.
I advise google to find ubuntu problems with your interface hardware model, and if that fails look for problems relating to the r8169 kernel module.

### Post by dandnsmith on 2012-04-02
There still seems to be a certain amount of confusion here.
If your ifconfig results do not show a real ethernet port configured with a usable address (as far as I can see you haven't said they don't), then the driver isn't doing you any good, so may well be the wrong one.

the way to do the driver replacement is summarised in this
```

# download driver for r8168, and unpack
http://www.realtek.com.tw/downloads/downloadsview.aspx?langid=1&pnid=13&pfid=5&level=5&conn=4&downtypeid=3&getdown=false


# go to driver location
cd r8168-8.020.00/
sudo rmmod r8169
sudo mv /lib/modules/`uname -r`/kernel/drivers/net/r8169.ko ~/r8169.ko.backup
make clean modules
sudo make install
sudo depmod -a
sudo insmod ./src/r8168.ko
sudo mv /initrd.img ~/initrd.img.backup
sudo mkinitramfs -o /boot/initrd.img-`uname -r` `uname -r`
sudo echo "r8168" >> sudo /etc/modules
lspci -v
sudo reboot

```
Which I extracted from [this thread](ubuntuforums.org/showthread.php?t=1465703), so you can see what I did (more or less)
However, I'm puzzled as to why no port at all shows, as I had one which worked (but unreliably) on two different PCs, and then replaced the driver to achieve full working status.

No amount of fiddling with measures for DNS lookup and routing are going to help in the face of a non-existant ethernet interface over which to operate.


Good luck in your endeavours.
