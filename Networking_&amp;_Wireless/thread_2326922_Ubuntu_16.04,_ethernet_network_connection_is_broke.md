### Post by TheFu on 2016-06-10
Ok ... so if that ping 8.8.8.8 doesn´t work, can you ping the router by IP?  This was requested in that link posted above with a few other commands not yet posted.  

There are 2 possible issues based on the information provided thus far:
a) wrong driver
b) incorrect settings

The NIC seems to be an intel PRO/1000 - that´s the drive loaded. If that isn´t correct, then you´ll need to find the correct driver. I´ve never had issues with mine and I prefer the Intel PRO/1000 NICs over all others. These are THE INDUSTRY STANDARD NIC.

I completely agree about wifi.   Prefer to ignore it myself as well.  Wifi vs GigE wired is no contest even to the point of using a USB-to-GigE adapter on wifi only computers to avoid wifi.

I never use automatic network settings or a GUI to manage them, so don´t know how to deal with that.  If you are interested, I can help with manual settings for a static IP, but this assumes you know a little about networking.  Another option is to setup a ¨DHCP reservation¨ on your router so the system will still use DHCP, but will get the same IP every time.  Not all home routers support this, but many do.  This link will attempt to explain: [http://blog.jdpfu.com/2011/07/18/use-your-router-to-centralize-your-network-device-management](http://blog.jdpfu.com/2011/07/18/use-your-router-to-centralize-your-network-device-management)
I use DHCP reservations for all my portable devices so they work correctly both at home and elsewhere.

For stationary systems, I manually setup the /etc/network/interfaces file.
