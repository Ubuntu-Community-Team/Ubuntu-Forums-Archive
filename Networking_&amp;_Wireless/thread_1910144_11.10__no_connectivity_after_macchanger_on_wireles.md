### Post by AnotherBrian on 2012-01-16
Using system settings network menu, I configured  the wireless interface to off. 
Using terminal I changed the mac address using macchanger (sudo macchanger -a wlan0). 
I revisited  system settings network menu having these problems:
1) menu displays hardware address as the original mac address and not the new mac address - so its pulling it from the wrong place
2) the menu claims wireless is unavailable. 
3) the configure button will not respond 
Even after rebooting I could not get this stuff work.   

After maybe a half hour, I was able to get the menu to fire up and offer me the local wireless networks within range.  It was as if an audit program detected and corrected an  inconsistency.  

If found an old macchanger post stating that you needed to restart NetworkManager using /etc/init.d but apparently all this stuff has moved around and uses the commands start/stop followed by the service.  I attempted using them, don't recall the exact responses but it didn't help. 

Suggestions?  Thx!

### Post by AnotherBrian on 2012-01-16
The following does the job without macchanger; 

sudo stop network-manager
sudo ifconfig wlan0 hw ether xx:xx;xx:xx:xx:xx
sudo start network-manager
