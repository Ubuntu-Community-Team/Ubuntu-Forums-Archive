### Post by matt_fussell2 on 2014-11-21
This is based on a solution found [here on the forums]("http://ubuntuforums.org/showthread.php?t=1538264"), and luckily, it still works for Ubuntu 14.04.

There's an issue with avahi (also known as zeroconf) and .local address resolution

On the client, edit the hosts line /etc/nsswitch.conf as follows:

```
sudo nano /etc/nsswitch.conf
```

```

# /etc/nsswitch.conf
#
# Example configuration of GNU Name Service Switch functionality.
# If you have the 'glibc-doc-reference' and 'info' packages installed, try:
# 'info libc "Name Service Switch"' for information about this file.

passwd:        compat
group:        compat
shadow:        compat

#hosts         files mdns4_minimal [NOTFOUND=return] dns
hosts:        files mdns4_minimal dns
networks:    files

protocols    db files
services:    db files
ethers:        db files
rpc:        db files

netgroup:    nis
```

So if you have a domain that is something akin to mynetwork.local, network.local, mycompany.local, etc, when connected to your properly configured DNS server, local host names will resolve properly.
