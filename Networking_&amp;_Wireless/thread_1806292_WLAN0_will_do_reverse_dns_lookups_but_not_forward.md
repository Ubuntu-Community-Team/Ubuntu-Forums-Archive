### Post by jmoorse on 2011-07-17
Ok, something is up with your DNS queries to internal servers.  External servers seem to be responding fine.


As an interim fix I recommend you add 8.8.8.8 ( Google honestdns ) and/or 208.67.222.222 ( Opendns ) as name servers.  

Go to your network settings (click on network icon in panel, edit connections, ipv4, select adapter, edit, set to automatic (dhcp) addresses only, enter 8.8.8.8, 208.67.222.222 in 'dns servers')

What did wireshark show?

### Post by GTrip on 2011-07-20
**[SIZE="4"]*** SOLVED ***[/SIZE]**
Tenda W311U Wireless Adapter didn't work well with the DLINK DIR-625 wireless router's enabled settings of:

- Multicast Streams
- UPnP
- QOS Engine

Once these 3 items were **_disabled_**, forward and reverse DNS functioned and normal network traffic passed without fail.  The Tenda WiFi adapter must be more sensitive than other adapters.  Other computer's WiFi adapters never have had difficulties connecting to this DLINK router with these settings enabled as they were.
