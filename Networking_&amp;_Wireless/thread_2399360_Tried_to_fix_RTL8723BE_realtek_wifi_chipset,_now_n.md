### Post by PippoPalmieri on 2018-08-24
I suppose you have installed this module [https://github.com/lwfinger/rtlwifi_new?](https://github.com/lwfinger/rtlwifi_new?) Required key not available is usually a problem when secure boot is enabled and you're trying to install third party modules that are unsigned.

Try to follow the instructions here 

[https://askubuntu.com/questions/762254/why-do-i-get-required-key-not-available-when-install-3rd-party-kernel-modules](https://askubuntu.com/questions/762254/why-do-i-get-required-key-not-available-when-install-3rd-party-kernel-modules)

and then reload the module with 
sudo modprobe -v rtl8723be

### Post by PippoPalmieri on 2018-08-24
What i think it's happening, correct me if i'm wrong, is that in order to fix your issue you tried to install a third party module. Now the module is installed but it's not loading because of secure boot. Since the module is not loading, it's as if the wifi card was not installed therefore it won't appear anywhere and it will say there's no wifi adapter.

If mokutil is not working try disabling secure boot completely from the bios.
