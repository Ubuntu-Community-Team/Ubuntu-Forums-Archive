### Post by imdano on 2007-07-11
Ok, a bunch of points here
1) WPA being misinterpreted as WEP is a known issue with ralink drivers.  However, you might get the correct info if you use the command ```
iwpriv rausb1 get_site_survey
```.  I'm not sure if that command is supported by the default ralink drivers, or just the enhanced ones you can get from serialmonkey.

2) You can find out the current drivers you're running by using this command ```
sudo lshw -C network
```You should see an entry in there for your wireless interface, there should be another entry under that which says "driver=<some name>".  That's the driver you're currently using.  It's probably something like rt2500.

3) The problems with Feisty that HOWTO noted were with people who didn't have network-manager in Edgy, altered their /etc/network/interfaces file, then upgraded.  When they upgraded, network-manager was installed by default and conflicted with the changes they made.  Since you're not using network-manager you shouldn't have this problem.

4)  I think a considerable amount of your trouble is due to ralink cards not having proper wpasupplicant support.  Upgrading to the serialmonkey drivers is your probably best chance to get things working.  They have two options, one is an enhanced version of the legacy drivers, which still don't support wpasupplicant, but are more reliable and use iwpriv to set the wpa info.  That's what the HOWTO I posted covers.  The other is to get the next gen rtx00 drivers that are going to be included in future kernel releases. They support wpasupplicant and wext (which means it will act more like the majority of drivers).  The downside is that until they are integrated into the kernel by default I *think* you'll have to reinstall them after every kernel upgrade.  You could try connecting before upgrading by entering this into the console:
```
ifconfig rausb1 down
iwconfig rausb1 essid "your essid"
iwconfig rausb1 mode managed
iwconfig rausb1 channel <channel number>
iwpriv rausb1 set NetworkType=Infra
iwpriv rausb1 set AuthMode=WPAPSK
iwpriv rausb1 set EncrypType=TKIP
iwpriv rausb1 set SSID="your essid"
iwpriv rausb1 set WPAPSK=<your passphrase>
iwpriv rausb1 set SSID="your essid"
sudo ifup rausb1
```I'm not sure how well this will work (if it at all, iwpriv commands are driver specific) without the enhanced drivers, but you can give it a shot.  If it doesn't work just remove the encryption and connect like normal through wicd (or whatever method you use).  Make sure that passphrase is in hex format, not ascii.  You can generate it using the command ```
wpa_passphrase <youressid> <your ascii passphrase>
```
