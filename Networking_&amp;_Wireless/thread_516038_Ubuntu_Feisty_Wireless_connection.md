### Post by john.klungle@hotmail.com on 2007-08-02
Have been trying to connect a Linksys WMP54Gv4.1  card for weeks. The Ethernet connection using a cble worked fine right off the install but could not get the wireless to work. Looked in the /etc/network/interfaces file and there was a configuration generated by the install for RA1. Down loaded drivers from RaLink, and several other places but none seemed to work. Would get a signal level but no connection. 

Finally I inserted a entry for RA0 into the interface card. Just copied the RA1 entry and changed RA1 to RA0. Surprise the RA1 interface made the connection.

Deleted all the RA0 data except -"auto ra0" and "iface ra0 inet dhcp". rebooted and connection still made. 

From all the GOOGLing there was only one article that mentioned ra0, ra1, ra2, ... and it said only that the interface looks uses ra0, then the next one uses ra1, etc.

"MUST HAVE RA0 TO GET TO RA1". Why Ubuntu did not but an entry into interface for ra0 would be a good question for the developers.
