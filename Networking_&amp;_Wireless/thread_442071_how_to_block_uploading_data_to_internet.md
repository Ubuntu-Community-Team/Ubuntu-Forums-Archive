### Post by heimo on 2007-05-13
You can't block all outgoing traffic without shutting yourself off the internet. But if you're not deliberately uploading - like using some p2p client - outgoing traffic should be fraction of the incoming. If you're using some p2p client, you can usually limit the used bandwidth in settings.

### Post by tact on 2007-05-13
install firestarter > set the outgoing default rule to be "restrictive by default".

when in this mode all outgoing is blocked, as is all incoming.  so you are effectively totally blocking all in and out traffic.   observe the events tab to see what was blocked, and if its something you want to let thru - set the policy to allow that connection...  etc etc...

then you know that only what you explicitly allow can perr thru
