### Post by three_jeeps on 2012-08-17
Ubuntu server 8.04 lts has been working fine since its comission about 1.5 years ago.  It serves a few virtual sites and fax server. Recently, had a number of power outtages, lost my battery backup and everything shut down.  Upon powering up the router, the server, and several other PCs on the LAN, I discovered that the router does not have the server in its DHCP subscription table, e.g. it does not see the server.  
Checked the router with ifconfig and sure enough, no ip address.
The server was initially set up as eth1, and configured to have a static address of 192.168.1.50 and configured to accept dhcp from the router, and the router reserved 1.50 as the address for the server.
I tried rebooting the server and router, did dhclinet -r to release and renew the lease, and still nothing.  
Any suggestions on how to resolve this issue is appreciated.
TIA!
John
