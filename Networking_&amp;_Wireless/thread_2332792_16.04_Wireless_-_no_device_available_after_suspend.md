Second, if your router is capable of N speeds, you may have better connectivity with a channel width of 20 MHz in the 2.4 GHz band instead of automatic 20/40 MHz, although it is likely to affect N speeds. I also have better luck with a fixed channel, either 1, 6 or 11, rather than automatic channel selection. Also, be certain the router is not set to use N speeds only; auto B, G and N is preferred. After making these changes, reboot the router.

### Post by wigginbalz on 2016-10-07
I have the same problem on my Alienware 15R2.  After resume, both the ethernet and wifi devices were gone.  I had tried all the different fixes:  restarting the network-manager service, upgrading the kernel to 4.6, rfkill before/after suspend, restart networking service, and rmmod/modprobe my device modules.  Nothing worked except a reboot.  Then I finally found something that worked.

sudo su
echo 1 > /sys/bus/pci/rescan

This worked immediately.  So I wanted to automate this on resume. So I copied /lib/systemd/system-sleep/wpasupplicant to /lib/systemd/system-sleep/z_pci_rescan so I had a working template to work with.

then modified z_pci_rescan to this:
#!/bin/sh
set -e

if [ "$2" = "suspend" ] || [ "$2" = "hybrid-sleep" ]; then
    case "$1" in
        #pre) /sbin/wpa_cli suspend ;;
        #post) /sbin/wpa_cli resume ;;
        post) echo 1 > /sys/bus/pci/rescan ;;
    esac
fi


Saved the file, made sure it was executable and tested.  Now all my network interfaces are back after a resume/wake.

Note:  I named the file z_pci_rescan so it would be run last.

Alienware 15r2 network config:  Killer E2400 Gigabit Ethernet (alx module), Intel 8260 wifi (iwlwifi module)

Hopefully this will help someone else with this problem!!!
