When I make a copy of the VMWare machine, keeping the original MAC address, everything works OK. However, if I let VMWare generate a new MAC address when I move or copy the machine (which is the VMWare default and usually the best choice), Ubuntu won't find the network card anymore. 

Does this mean that the MAC address of the network card has to match a MAC address in some config file somewhere?

### Post by netztier on 2007-06-26
> **xrvjorn said:**
> Is that file necessary, or can I delete it and get Ubuntu to be happy with whatever MAC address it finds on the NIC (sort of like the Windows behaviour)?

To the best of my knowledge, deleting the file's content does no harm - and in general, it should be the first interface found in the system that gets eth0.

I do not know how and why some entries seem to make it there "by magic" - but whenever you start swapping cards, make sure to check it's content for wrong or stale entries. It took me a while to find the file and with it the reason why I just couldn't aggregate eth0 and eth1 to bond0 - the bonding module kept shouting and yelling that some interfaces were not available -  it turned out that the cards i wanted to use were eth1 and eth3, right because of the statements in /etc/iftab.
