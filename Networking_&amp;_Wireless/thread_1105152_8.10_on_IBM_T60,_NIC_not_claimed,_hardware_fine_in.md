### Post by chili555 on 2009-03-26
> Remaining tasks as I see it are to:
1. Map eth0 to lan
2. Map wlan0 to wireless cardThe key to mapping devices is shown in */etc/udev/rules.d/70-persistent-net.rules*. Let's back yours up first, before we make any mistakes. Please do:```
cat /etc/udev/rules.d/70-persistent-net.rules > pers-net.rules.txt
```Now, please amend yours to map the interface names to the MAC addresses of the devices. You can determine the MAC addresses in:```
sudo lshw -C network
```Here is an excerpt from mine:```
*-network
       description: Wireless interface
       product: PRO/Wireless 3945ABG [Golan] Network Connection
       vendor: Intel Corporation
       physical id: 0
       bus info: pci@0000:03:00.0
       logical name: wmaster0
       version: 02
       serial: [COLOR="Red"]00:19:d2:c2:1b:8d[/COLOR]
```Here is my entire */etc/udev/rules.d/70-persistent-net.rules*:```
# This file maintains persistent names for network interfaces.
# See udev(7) for syntax.
#
# Entries are automatically added by the 75-persistent-net-generator.rules
# file; however you are also free to add your own entries.

# Converted from /etc/iftab on upgrade
SUBSYSTEM=="net", DRIVERS=="?*", ATTRS{address}=="00:16:41:e6:cb:ef", ATTRS{type}=="1", NAME="eth0"

# Converted from /etc/iftab on upgrade
SUBSYSTEM=="net", DRIVERS=="?*", ATTRS{address}=="00:19:d2:c2:1b:8d", ATTRS{type}=="1", NAME="eth1"
```You can see the wireless address (MAC) is the exact same as shown above. Incidently, my wireless is *eth1* but you can map it to wlan0 if you wish.

Please amend your */etc/udev/rules.d/70-persistent-net.rules* to map the correct devices to the correct interface. The stuff behind # is comments to help human readers understand it. Yours may be slightly different. That's OK.

You also need to amend your */etc/network/interfaces* as follows:```
auto lo
iface lo inet loopback

auto wlan0
iface wlan0 inet static
address 192.168.1.9
netmask 255.255.255.0
gateway 192.168.1.1
wireless-key i've edited the key
wireless-essid sol

#auto eth0
iface eth0 inet dhcp
```Check your work carefully and *immediately* reboot. I think we will have knocked out the first two items on your list. Then run the vidalia script and let's hear how it went.
