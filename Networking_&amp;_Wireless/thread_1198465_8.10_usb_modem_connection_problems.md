### Post by sanderj on 2009-06-27
I agree with ajgreeny that putting in an ethernet card is probably much easier. Having said that, you could do this:

With the usb-ethernet-stick + ethernet-cable plugged in, in Network Tools -> Devices, check whether your usb-ethernet-stick in the dropdown list. If so, select it, and see if you can activate it to have an IP address.

If that does not work:

plug out the usb-ethernet-stick
Open a terminal
type 'lsusb'
then put in the usb-ethernet-stick
type 'lsusb' again, and check which new line is there: that must be your usb-ethernet-stick. Report the complete line here.
you can start googleing for that usb device id.

EDIT: I now realize that the USB-device *is* the (ADSL?) modem. Instead of "usb-ethernet-stick" read "usb cable between computer and USB-modem"

### Post by sanderj on 2009-07-05
I googled, and found this: [http://www.linuxquestions.org/questions/linux-newbie-8/ambit-cable-modem-drivers-77412/](http://www.linuxquestions.org/questions/linux-newbie-8/ambit-cable-modem-drivers-77412/)

> 
open an xterm window, and as root type

rmmod acm
modprobe CDCEther
ifconfig -a

After this last one, it *should* say something like

eth0: Ambit Cable Modem blah blah blah
inet add: 0.0.0.0.0 (dont worry about this bit)

If so, type

dhcpcd
ifconfig

Now it should say

eth0: Ambit Cable Modem
inet add: 80.98.22.1 etc

Leave the window open and try an internet browser.Hopefully you'll have internet access.
These instructions *wont* however give you interent access every time you boot up



So, maybe you need "sudo modprobe cdc_ether" ((Communications Device Class Ethernet)). I don't know. USB-modems are a pain in the ***, and should be avoided. As said before, getting a plain ethernet card is 1000000 times easier ...
