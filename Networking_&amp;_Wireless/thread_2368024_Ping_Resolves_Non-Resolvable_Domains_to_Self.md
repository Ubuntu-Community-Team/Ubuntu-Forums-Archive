### Post by own3mall on 2017-08-05
I'm having a very strange issue when attempting to ping hosts on one of my Ubuntu 14.04 servers.  Non-resolvable domains and hosts are resolving to itself, and I have not configured the server to do this anywhere!

```

ping dsafadsfjdsadsafds.com
PING dsafadsfjdsadsafds.com.mydomain.com (1.1.1.1) 56(84) bytes of data.
64 bytes from www.mydomain.com (1.1.1.1): icmp_seq=1 ttl=64 time=0.026 ms
64 bytes from www.mydomain.com (1.1.1.1): icmp_seq=2 ttl=64 time=0.056 ms

```

Anyone know why that's happening?

Resolv.conf has the following:

nsswitch has:

### Post by own3mall on 2017-08-06
I have no idea what happened, but after looking at some of my other Ubuntu servers, I decided to change the /etc/nsswitch.conf setting from:

```

hosts: files dns

```

To:

```

hosts: files mdns4_minimal [NOTFOUND=return] dns mdns4

```

I had to reboot for it to pick up these changes.

Anyone know why that fixed it?  What does mdns4_minimal do?  What does the NotFound entry do?

Oh well... the problem is fixed for now until it randomly starts happening again.

### Post by own3mall on 2017-08-06
Ok, I've finally traced it down.  If you use a FQDN as your hostname such as (serv1.mydomain.com), for some reason, utilities such as ping will append the domain name (.mydomain.com) when performing a lookup on something as silly as asdfasdsfadsfadsffadsfads.com which becomes asdfasdsfadsfadsffadsfads.com.mydomain.com.  Why??  I have no idea.  This appears to be a bug in Ubuntu?  I did the same thing in CentOS 7, and it doesn't do this incorrect behavior.

---

### Post by own3mall on 2017-08-06
After more toiling, I figured out it's a problem with wildcard DNS set on the main FQDN DNS server!

[https://bugs.launchpad.net/ubuntu/+bug/1708905](https://bugs.launchpad.net/ubuntu/+bug/1708905)

It appears to affect all Linux distros (tried on CentOS 7 as well).  Why would wildcard DNS cause the base domain to be appended to a queried unresolvable / unknown hostname?
