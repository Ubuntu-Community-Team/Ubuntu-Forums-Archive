### Post by steveontario on 2016-05-15
After six months of wonderful hassle-free experience with 14.04 LTS, I updated last Friday as per standard procedure; when I rebooted, my wifi and ethernet were not working.

My computer is an Asus Zenbook UX305, with Windows 8 and of course Ubuntu 14.04 LTS installed.

Researching the issue, I came across [this Ubuntu forum post]("http://ubuntuforums.org/showthread.php?t=2238503"), in which the original poster's issue appears very similar to what I'm dealing with. However, the OP's output from the terminal command

```
lspci -n | egrep '0200|0280' | awk '{print$3}'
```

is different from mine, which is: 
8086:095a

Further research tells me my network controller is made by Intel, specifically that it is an Intel 7265 driver (or is it module, I'm not sure).

I cannot figure out how (or whether) to modify the community member Hadaka's suggestion to the OP, which is to run the following commands: 

```
sudo apt-get remove --purge bcmwl-kernel-source
sudo rm /etc/modprobe.d/blacklist-bcm43.conf #its ok if this command does nothing or fails
sudo modprobe b44
```
... because I figure they are tailored to a situation in which the network controller is made by Broadcom.

Does anyone have any idea how I could use the above info to get at least my ethernet internet connection working again?

thanks,

### Post by steveontario on 2016-05-15
ahhh... much obliged. I followed [tamkderplar]("http://askubuntu.com/users/543259/tamkderplar")'s suggestion of downloading network-manager and putting it onto a USB thumb drive, then copying it into the affected machine's Home directory then running 

```
sudo dpkg -i network-manager_*.deb
```

then running 

```
sudo service network-manager restart
```

Everything works now. What a relief.

Thanks
