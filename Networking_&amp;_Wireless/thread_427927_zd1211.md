### Post by MarcoBertolini on 2007-05-01
First of all, now you can use the zd1211rw driver that works fine since kernel 2.6.18. 
The only problem is that the usb-ids of some cards that use this driver are not listed in the driver, so it doesn't work out of the box. For example, to have my Zyxel ZyAir G-220v2 back working, I had to slightly modify the sources (only a one-liner) and recompile all. The basic step I followed are (probably there's a faster method)

1- Download the sources of the kernel
2- Modify them
3- Compile a new kernel (make-kpkg) and install it
4- Copy the relevant modules (zd1211rw.ko and zd1211rw-mac80211.ko) from the new compiled kernel to the old one
5- Reboot

You can find some details of the process here

[http://ihome.ust.hk/blogs/home/josephwu/System_Security/2007/03/31/ZyAir-G-220-V2-Wireless-USB-adpater-and-Kismet.html](http://ihome.ust.hk/blogs/home/josephwu/System_Security/2007/03/31/ZyAir-G-220-V2-Wireless-USB-adpater-and-Kismet.html)
[http://www.howtoforge.com/kernel_compilation_ubuntu](http://www.howtoforge.com/kernel_compilation_ubuntu)
[http://www.mail-archive.com/netdev@vger.kernel.org/msg31827.html](http://www.mail-archive.com/netdev@vger.kernel.org/msg31827.html)

Finally, your problem with compiling is due to the fact that config.h as been changed sometime during the kernel development to autoconf.h. Alas, changing the header name everywhere in the zydas driver wasn't enough for me.

HTH
