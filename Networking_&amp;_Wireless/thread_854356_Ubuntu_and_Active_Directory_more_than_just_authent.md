Greetings,

I work for a school district running an Active Directory network. Over this summer (as we tend to have more time for projects in summer) We've set aside a project see how well we can integrate Linux into our AD network. We already have quite a few Macs pretty well integrated (actually I'm the "Mac guy" around here heading most of that up), and hoping to get to a similar stage on linux.

Basically what we've got so far: Authentication. I've set up Likewise and bound to the domain and authentication is working. Sweet. Also tested out Evolution with our Exchange server some and that seems like it's going to work acceptably.

Now, what I'm REALLY looking to accomplish overall is the following:

    1) Exchange support - Check, seems to be going well.

    2) AD Authentication - Check, got this one working.

    3) Home directory ("I" Drive) mounting at login for user's shared files. Similar to what [this guy is asking (and didnt get responses)]("http://ubuntuforums.org/showthread.php?t=842687").

    4) Proper kerberos SSO, aka can browse and add network printers and drives without requiring authentication. Or at least a way to script the mounting of certain drives based on login credentials (not hardcode).

    5) The ability to assign "admin" priviledges on the machine based on AD groups (for example a member of the "Technology" group logging in and being recognized as an admin).

So, any thoughts? any steps or nudges in the right direction would be great also. My biggest frustration with doing these types of things on linux is even when you search and seem to find a solution, it may be extremely old or out of date or may have changed with the current version of the software, so I'm hoping here to get some current solutions and answers (and will hopefully get to publish the results either in a how-to or a blog to share it in an organized manner for others).
