### Post by matt_symes on 2012-06-18
Hi

You connections should be stored in the folder

```
/etc/NetworkManager/system-connections/
```

Check your connection information is valid using

```
sudo cat /etc/NetworkManager/system-connections/<connect_name>
```

It requires sudo privilages to view these file.

I assume you are using the latest version of Lubuntu ? Can you state what version you are using please.

You should not need to manually edit /etc/resolv.conf and it should look like...

```
matthew@matthew-Aspire-7540 ~ % cat /etc/resolv.conf 
# Generated by NetworkManager
nameserver 127.0.0.1
matthew@matthew-Aspire-7540 ~ %
```

On the newest version it should use the localhost ip address and dns is handled by resolvconf.

This is certainly true of the newest version of Ubutnu are i would imagine it would be true with lubuntu.

### Post by rukiaEnix on 2012-06-18
Hello, you can block your resolv.conf to be changed again with this command with sudo or as root:

```

chattr +i /etc/resolv.conf

```

and to unblock it again:

```

chattr -i /etc/resolv.conf

```

This is what you want as network manager handles everything higher up. If you are still not getting the dns server you expect then there is something wrong at a higher level.

You can check the dns server being used with

```
nm-tool | grep -i dns
```

for both connections.

That will tell you the current dns server being used for the connection.

```

nameserver 127.0.0.1
matthew@matthew-Aspire-7540 ~ % nm-tool | grep -i dns
```
