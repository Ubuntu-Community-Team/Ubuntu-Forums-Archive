### Post by Morbius1 on 2010-06-21
There are two completly different methods of using samba to share folders: Nautilus-share and Classic-share. 

** Nautilus-share comes with restrictions but is far easier to use so let's start with that:**

Open Nautilus
Right click on a folder
Select "Sharing Options"
For a guest share select: "Share this folder", "Allow other people to write", and "Guest Access"
For an authenticated share select: "Share this folder" and "Allow other people to write"
Select "Create Share"
It will ask you if you want to modify permissions - you do
That's it.

The share definitions it creates are located at [COLOR=Blue]/var/lib/samba/usershares.[/COLOR] And you can always get a list of the Nautilus-shares you have created as well as how they are defined by issuing the following command:
```
net usershare info
```If you have an authenticated share then only those who have the proper usernames and passwords will be allowed to access the share so you will need to add then to the server and that is a two step process. It appears from your post that you did that already so you should be good to go.

**Classic-share is far more configurable but can be more difficult to set up depending on your requirements.**

I usually set these up manually using a text editor on smb.conf directly. Most of the GUI's available for Classic-shares are almost guaranteed to make a mess of things but the one with the least risk is: **system-config-samba. **Once installed it will show up under Administration > Samba.

The shares definitions it creates are located in [COLOR=Blue]/etc/samba/smb.conf[/COLOR]. Lauching Administration > Samba will show you the list of classic-shares you have created and you can issue the following command to see all your classic-shares and how they are defined by issuing the following command:
```
testparm -s
```[COLOR=Blue]One word of caution[/COLOR]: There is no rule that states that you cannot use Nautilus-share and Classic-share at the same time but I would suggest you don't. You don't want to end up using both methods on the same folder, or one method on a parent folder and another on a child folder. It's more of a user manageability issue than a technical issue.

**One big difference between Nautilus-share and Classic-share.**

The biggest difference between the two methods concerns authenticated shares. If I have two remote users - john and mary - and I create two private shares using Nautilus-share then both john and mary will have access to both providing they have a proper username and password. If I want to create one private share that only john can access and another that only mary can access then choose Classic-share because Nautilus-share is incapable of doing that.
