So in detail what I did was:

install ubuntu 12.10: sda1 as efi boot partition (256 mb), sda2 as / (root) partition, sda3 as swap partition and installed bootloader on sda (no boot-repair needed). After the install I got the ubuntu entry in bios again but wifi didnt work when i changed the boot priority as mentioned in my previous posts. Next I tried all this blacklisting options mentioned 100x times and none of them worked, so I deleted all this blacklists in blacklist.conf and rc.local. 
After all that frustration I changed the realtek card to first boot place in front of ubuntu and it didnt work either (it just hangs on boot) UNTIL I disconnected the Lan cable. Some data was displayed and ubuntu booted with enabled wifi streight away.
After that I upgraded to 13.04 via network with my fully funcional wifi. After upgrade  I changed the boot priority to realtek card again.
This way I cant boot with the lan cable inserted but who cares, I have a working wifi :D.

My assumption is that somehow you must load or enable the wifi card during boot process, some also mentioned that they turned it on, off and on again in windows so that it could function in ubuntu again, because of this hardware switch thing :D . Maybe a cmos reset would help too.
