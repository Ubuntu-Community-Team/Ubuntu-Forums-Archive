### Post by dmizer on 2009-07-14
Well, just because the results of an error were huge, doesn't mean that the problem was huge, or that the fix would have been difficult. I suspect that there may have been a typo or some such in your /etc/fstab file. No matter what the problem, you could have used a live CD to find and fix it.

> **MacDuff said:**
> Where does 9.04 mount network shares by default?  
It doesn't. That's the whole reason why I wrote the CIFS mount howto. However, I recently discovered this: 
[http://ubuntuforums.org/showthread.php?t=1186877](http://ubuntuforums.org/showthread.php?t=1186877)

Otherwise, you can just install smbfs and add the line I gave you above to your /etc/fstab file. Make a backup first:
```
sudo cp /etc/fstab /etc/fstab.backup
```
Then, if something goes wrong, you can boot to your live cd and run this command to restore it
```
sudo mv /etc/fstab.backup /etc/fstab
```

### Post by dmizer on 2009-07-15
Ok, first of all, you should try enabling netbios name resolution as outlined in the "Pre-work" section. This sometimes solves writability problems. Again, if this causes problems, you can use the live CD to remove the changes.

When the share is mounted, please run the following command and post the output.
```
sudo ls -la /media/RAID
```

Samba only reads one smb.conf file: /etc/samba/smb.conf. It is NOT necessary to manipulate the smb.conf file in order to get access to your NAS shares.

You won't need to make any changes to your Ubuntu machine. Since Samba doesn't appear to be doing the job correctly for you, then you need an alternative. Since your NAS device supports FTP access, you can use it instead of Samba.

You need to set up FTP on the NAS. You can refer to the user's manual if you don't know how. If you don't have it, you can find it here: [http://www.dlink.com/products/default.aspx?pid=DNS-323&tab=3](http://www.dlink.com/products/default.aspx?pid=DNS-323&tab=3)

Once you have the NAS set up for FTP access, then all you have to do on your Ubuntu machine is click on places > connect to server and enter the correct information for FTP.

One caution, the owners manual talks about forwarding ports through the router to allow access from the internet. You won't need to do that.
