### Post by lloyd_b on 2007-05-24
One point - for that static route, try using "route add -host..." rather than "route add -net..." and see if that helps.

Other than that, could you post the following info, one set for the DHCP version, and again for the static IP version, so we can do a comparison between the two:
1.  The output from the "ifconfig" command.
2.  The output from the "route -n" command.

A comment: You should really segment your network differently.  The fact that you need a static route to reach a host in the 10.1.1.x range, while your IP and netmask indicates that such an address should be directly accessible, is just a bit weird.   I realize that you may not have any control over this, but if you do, having a cleanly segmented network (with each segment in it's own 10.1.x.x range) will save you a lot of headaches in the long run.

Lloyd B.

### Post by lloyd_b on 2007-05-25
First off, I'm a bit confused as to your actual problem.  From the initial message, I was under the impression that you were just trying to get reliable access to the FTP server (10.1.1.13) to work, but that everything else is fine.

From you last message, you introduce the 169.254.0.0 issue.  I have no clue where this route is coming from, or what its purpose is.  It doesn't appear to relate to the 10.1.1.13 issue at all.

For the 10.1.1.13 issue - after booting with static IP, in a terminal window do a "traceroute 10.1.1.13" and see what path it is following.  This might provide a clue as to what's going wrong.  I'm not sure if the "-net" vs "-host" will make a difference (I suspect that "-host" is simply providing a default netmask of 255.255.255.255, but there may be other differences as well).

From the info you posted, I see no difference whatsoever between the DHCP and static configs, as far as the FTP server is concerned.  You highlighted the 169.254.0.0 issue, but I don't think that relates to the FTP server issue at all (at least it *shouldn't*).  Hopefully the traceroute will shed some light on the situation.
