### Post by jonobr on 2012-03-23
Hey


I think you have quite a few problems here,

Its better practice to drop the leading zero's where you dont need them
from

> 
auto eth0
iface eth0 inet static
address 172.030.014.038
netmask 255.255.255.252
network 192.168.2.0
broadcast 192.168.2.255
gateway 172.030.014.038
[\QUOTE]
and 

[QUOTE]

# Generated by NetworkManager
nameserver 010.050.001.001
nameserver 010.050.001.002 

to


> 
auto eth0
iface eth0 inet static
address 172.30.14.38
netmask 255.255.255.252
network 192.168.2.0
broadcast 192.168.2.255
gateway 172.30.14.38
[\QUOTE]
and
[QUOTE]
# Generated by NetworkManager
nameserver 10.50.1.1
nameserver 10.50.1.2 
[\QUOTE]
Second

for a 252 subnet in the 172.30.14.x network , your broadcast and network need to be changed from
```

network 192.168.2.0
broadcast 192.168.2.255
```

to most likely 

```

network 172.30.14.36
broadcast 172.30.14.39
```

Lastly your ip address is matching your gateway,
are you sure thats correct?

### Post by praseodym on 2012-03-23
Hi,

if you added the nameservers by hand, remove all the "leading zeros":

```
nameserver [COLOR="Red"]0[/COLOR]10.[COLOR="Red"]0[/COLOR]50.[COLOR="Red"]00[/COLOR]1.[COLOR="Red"]00[/COLOR]1
nameserver [COLOR="Red"]0[/COLOR]10.[COLOR="Red"]0[/COLOR]50.[COLOR="Red"]00[/COLOR]1.[COLOR="Red"]00[/COLOR]2
```
Also in the "interfaces"-file.
How do you connect? Router, modem, or both (somehow combined)?
