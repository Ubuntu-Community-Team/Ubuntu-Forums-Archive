### Post by steeldriver on 2012-07-14
This is a fast moving forum and things WILL slip through the cracks - these intermittent problems are notoriously hard to diagnose as well - I've only been here a short time but I can assure you people are VERY willing to help if they can

A quick search throws up a number of reports of similar (although possibly not identical) intermittent dropouts with the ipw2200 on a variety of Linuxes / kernel versions. Some people have suggested removing network-manager in favor of wicd but I don't know if there's any basis to that. AFAIK the firmware is no longer in active development so ultimately you may have to suck it up and either replace the card with a better supported one (or a USB / CardBus device) or use the wired interface.

If you do want to carry on and troubleshoot I would start by taking a harder look at dmesg - especially for things related to wireless association - some or all of these greps for example:

```
dmesg | grep -i -e ipw -e assoc -e auth -e eth[0-9] -e phy[0-9] -e wlan[0-9] -e firmware
```What protocol are you using (WEP / WPA / WPA2)? (not even sure it supports WPA2) Is it stable if you change that? do you have any power management enabled that might be messing with things?
