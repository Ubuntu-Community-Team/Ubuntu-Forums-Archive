### Post by belnac on 2014-09-08
I'm attempting to set my server's ethernet port as static and could appreciate some help as I'm not getting anywhere. After applying the following changes I am left with no networking at all.

Here's what I've done.

1. Created [FONT=Courier New]/etc/init/network-manager.override[/FONT] with [FONT=Courier New]manual[/FONT] in it.

2. Adjusted the [FONT=Courier New]/etc/network/interfaces[/FONT] file so as to contain:
```
# interfaces(5) file used by ifup(8) and ifdown(8)
auto lo
iface lo inet loopback

face eth0 inet static
address 192.168.0.100
netmask 255.255.255.0
network 192.168.0.0
broadcast 192.168.1.255
gateway 192.168.0.1
dns-nameservers 8.8.8.8 8.8.4.4
```

Upon rebooting, [FONT=Courier New]ifconfig[/FONT] shows no connection:

```
eth0      Link encap:Ethernet  HWaddr 48:5b:39:49:f4:dc  
          inet6 addr: fe80::4a5b:39ff:fe49:f4dc/64 Scope:Link
          UP BROADCAST RUNNING MULTICAST  MTU:1500  Metric:1
          RX packets:14 errors:0 dropped:0 overruns:0 frame:0

          collisions:0 txqueuelen:1000

```

What am I doing wrong?

### Post by belnac on 2014-09-08
Nevermind, found the solution.

In the [FONT=Courier New]interfaces[/FONT] file, [FONT=Courier New]face[/FONT] should be [FONT=Courier New]iface[/FONT].

### Post by lisati on 2014-09-08
That doesn't look quite right to me. Assuming that 192.168.0.x is the correct mask for your network, broadcast should be 192.168.**0**.255

```
# interfaces(5) file used by ifup(8) and ifdown(8)
auto lo
iface lo inet loopback

face eth0 inet static
address [COLOR="#FF0000"]192.168.0[/COLOR].100
netmask 255.255.255.0
network [COLOR="#FF0000"]192.168.0[/COLOR].0
broadcast [COLOR="#FF0000"]192.168.1[/COLOR].255
gateway [COLOR="#FF0000"]192.168.0[/COLOR].1
dns-nameservers 8.8.8.8 8.8.4.4

```
