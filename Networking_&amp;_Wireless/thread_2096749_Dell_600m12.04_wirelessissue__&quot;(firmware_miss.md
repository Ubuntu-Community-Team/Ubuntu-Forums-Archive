### Post by tgalati4 on 2013-01-04
I have a Dell 600m and I'm running Linux Mint14 Mate based on 12.10 and I'm using wireless as we speak, so I know it works.  One issue that I have noticed is that the wireless tends to shut down with static electricity when picking up the laptop and moving around.  Fn-F2 does not wake it--must be a BIOS, safety feature.  Anyway you might have to go into BIOS and turn Wireless ON and set to Fn-F2/Application.  

When not zapped by static electricity, the wireless works and you can turn it on and off with Fn-F2 as expected.  When you lose wireless with a static shock, you have to shutdown completely and reboot.

I loaded the standard b43 driver, and it looks like I have a 4318:

firmware-b43-installer:
  Installed: 1:015-14

You are running a 4306.  I'm not sure of the differences between a 4306 and 4318.

Perhaps you can use the firmware cutter and manually install a windows binary driver:

tgalati4@Dell600m-mint14 ~ $ apt-cache search b43
b43-fwcutter - Utility for extracting Broadcom 43xx firmware
firmware-b43-installer - Installer package for firmware for the b43 driver
firmware-b43-lpphy-installer - Installer package for firmware for the b43 driver (LP-PHY version)
firmware-b43legacy-installer - Installer package for firmware for the b43legacy driver

The basic procedure is to have a CD with the Broadcom windows XP drivers on it, or a Windows partition with the wireless drivers installed and point fwcutter to that directory.  It will extract the binary blob and insert it into a wireless module that wakes up your card.  It generally works.
