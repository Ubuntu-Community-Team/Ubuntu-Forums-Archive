At home I have a network of three computers. Two of them are connected to my router with regular CAT5 cables, using IP's 192.168.2.2 and 192.168.2.3 respectively. On the last computer, a shared printer is installed and working fine with the other ethernet-connected computer. The same goes for remote desktops with vinagre. I can access the desktop of computer 2 from computer 1 and vice versa.
On my netbook using wifi on IP 192.168.2.4 it doesn't seem to work at all. (My router has a built-in wifi access point, so it's connected to the same device.) The shared printer cannot be found and the shared desktops don't pop up in vinagre's bookmarks. The other way around, my netbook's shared desktop does pop up on the ethernet computers, but when I try to access it, the connection closes within a second.

I tried connecting my netbook directly to my router using a third CAT5 cable, and everything was working perfectly. I was able to print to the shared printer and I could access my netbook's desktop using either one of the 'real' computers. (And that can come in quite handy when you have to do administrative tasks.) 

But fiddling around with a big ethernet cable in my netbook isn't really what I was hoping for when I bought it, so I would really like to get this working using wifi. I'm just not sure of what I'm missing here.
