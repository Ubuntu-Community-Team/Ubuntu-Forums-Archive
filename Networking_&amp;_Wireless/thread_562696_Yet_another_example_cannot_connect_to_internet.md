### Post by Thunar on 2007-09-29
> **figo2476 said:**
> 
em... I can ping 127.0.0.1 & 169.254.3.164. That means I can talk to the router...?

I'm no expert, but i think you're pinging yourself, not the router...

It also seems like you have a lot of "Unknown Device" going on, but that may or may not be the problem...

If you are directly connected to the router, one thing you may want to try is this:

Make sure your Linux machine is the only one connected, and your router is connected to the internet directly.

Reboot your router (either by turning your router off and on physically, or preferably by connecting to it via your browser by typing in something like 192.168.100.1, going to the configuration page and selecting 'reboot' if available)

wait for the router to boot up and try, try again.

The reason: Some routers, and particularly modems, limit the number of MAC addresses they can 'learn'. So if you reboot the modem with your Linux machine connected to it, it might 'learn' the MAC address of your Linux NIC card. This is true for both Ethernet and USB network connections.

If you are connecting via a Windows machine, you'll have to set up Internet sharing on the Windows machine before you do anything.

Hope this helps!

### Post by Rui Pais on 2007-09-29
> **figo2476 said:**
> 
* how do I find out the ip of my router, then connect to it via browser, in linux? (or I have do something in windows)

Hi,
your router IP should be written on the hardware documentation. Whats the model you have?

Anyway, NetworkManager seems to be detect a 10.1.1.1 (another usual IPs ate 10.0.1.1 or 192.168.1.1). 
You enter the configuration window using a browser (firefox, epiphany, whatever...) and type on address box the router IP.

Your actual problem is that your net card don't seems to get a correct IP (set by dhcp). Avahi-daemon (on linux side) is giving an incorrect one.
That happens sometimes, and the simpler solution seems to be stop that daemon ([here a link for a thread on that]("http://ubuntuforums.org/showthread.php?p=3082884#post3061592")).

Before you try anything harder, get sure of your router IP first, then do on a terminal:
```
sudo mv /etc/network/interfaces /etc/network/interfaces.BAKUP
sudo kedit /etc/network/interfaces

```(i think kubuntu use kedit, or is kate?)
and on new /etc/network/interfaces copy+paste this:
```
auto lo
iface lo inet loopback
```
to be make sure that you get rid of any wrong net confs, reboot.

Check then how your connection is and post the output of ifconfig you problems persists.

Good luck

the new interface file with the "light" config is the equivalent of roaming mode on Network Manager. That usually enable automatic set of IP with less trouble.
If that still don't happen, either you have a dhcp issue or ipv6 is preventing the correct assign of an IP.
Fot ipv6 issue, do:
```
sudo sh -c "echo 'blacklist ipv6' >> /etc/modprobe.d/blacklist"
```
and reboot (the only thing i know that can unload ipv6...)

Any other suggestions would require a little more info on your router.
