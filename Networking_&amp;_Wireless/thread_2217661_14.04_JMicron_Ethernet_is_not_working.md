### Post by varunendra on 2014-04-20
> **robi-hipnos said:**
> It does help!

Let's try making it permanent then. Please run the following code to add the command to your /etc/rc.local file -
```
sudo sed -i '/^exit 0/i /sbin/ethtool -s eth0 speed 100 duplex full autoneg off' /etc/rc.local
```

Reboot and check if the Ethernet is working now. If not, we may have to try another method to make it permanent.

Your card is capable of Gigabit speed (1000 Mb/s), so "speed 100" option is a compromise with speed. We may try forcing 1000 Mb/s, but I doubt its success. "Autonegotiation" is essential for gigabit speed link, so the ethtool command will have to be changed a bit to try that. But if your router is 10/100 Mbit only, there is no point in trying that and we're good with 100 Mb/s.

### Post by varunendra on 2014-04-21
> **robi-hipnos said:**
> So this seems to be working too :)
That's a really great news! Have you checked the practical download/upload speeds too? Sometimes the link speed may appear to be good, but the actual data transfer speed may suffer. Hope that is not the case with us here.

> **robi-hipnos said:**
> And now the big question, why it does not work out of the box? Kernel driver seems to be ok.
I don't have any solid clue on this, but I *think* it may be a bug or some kind of discrepancy in the implementation of driver/firmware due to which _the card can not negotiate the best possible link with the router/switch in a timely fashion_. This discrepancy may exist in either the Linux driver or the router's firmware. Again, I can't say with confidence, but most often it is the device vendors who love 'patches' and don't follow the standards properly. The proprietary drivers work because they are ready with the required 'patches'.
