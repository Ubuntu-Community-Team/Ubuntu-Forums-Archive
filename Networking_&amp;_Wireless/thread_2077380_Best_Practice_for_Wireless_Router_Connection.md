### Post by Qwirty on 2012-10-28
New member here saying Hello World and glad to be aboard.
Unlike most people in this sector I am lucky to be having no network problems but I would appreciate advice on "best practices".

I have just converted my Windows peer-to-peer workgroup to a Ubuntu server workgroup  based on Ubuntu desktop. The layout of the old system was, DSL line, Actiontec gateway and Netgear wireless/wired router. Two desktop computers were hard-wired to the Netgear router and other computers could join the wireless network as required. The DSL modem and Netgear router have hardware firewalls and the desktops and wireless linked computers all ran Microsoft Security under W7.

I changed the system layout so that there is a single wired connection to the Ubuntu server from the Netgear router and the desktop clients now receive the web through the server, works great. Of course, the hardware firewalls still exist and the clients still run MS Security. Here is where I see that there may be a problem. On my laptop, and probably any other computer receiving a wireless signal, I am accessing the web directly from the Netgear router not through the server. I can also, in some way I do not understand, access the server file system from the same laptop. This would seem to constitute a potential security leak.

It is convenient to connect to the web directly through the router because I can use the laptop for browsing with the server shut down but I do not want to create a security breach. Should I move the wireless router downstream from the server from the network switch or do the experts feel that my present hook-up provides adequate protection?
