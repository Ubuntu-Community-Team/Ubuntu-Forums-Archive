### Post by grahammechanical on 2013-12-26
Your interfaces file looks correct. If an interfaces file contains anything other than what you have then it will prevent Network Manager from working. You could try

```
rfkill list
```

You should see something like this

> 0: phy0: Wireless LAN
    Soft blocked: no
    Hard blocked: no




If there is a "yes" in the output of that command then the wireless adapter is switched off either by software or by hardware. The Network Manager icon should give you an option to enable wireless. On 13.10 we can also go to System Settings>Network and switch on wireless from there.

Regards.

### Post by grahammechanical on 2013-12-26
Yes it should all automatic. I am now moving into fantasy guess mode. Is the wireless adapter turned off in the BIOS? Is the wireless adapter still in its socket in the motherboard. Is it in range of the wireless access point - the router?

SSID or ESSID would be the name of your Internet Service Supplier that the router connects to when it is switched on. It is the wireless name. The device MAC address is a code that identifies that router from other network capable devices on a network. It is also known as a hardware address. The ethernet adapter will have one. And it is stored in the electronics of the device.

You could try

```
nm-tool
```
