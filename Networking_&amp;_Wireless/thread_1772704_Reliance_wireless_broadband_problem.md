### Post by sidfadnis on 2011-06-01
I have a Reliance Netconnect BroadBand+ datacard (using USB). It is a wireless internet device that runs on CDMA network. My plan is such that this device runs on CDMA EVDO network (speeds comparable to 3G) till a limit of 5GB data transfer. After this limit is crossed the device automatically switches to low speed on plain CDMA network.

The "new wireless device" wizard in ubuntu network manager, supports this device but only in CDMA EVDO mode. So as soon as it switches to plain CDMA, I can't connect to internet with this device in Ubuntu. 

I searched forums/internet for my problem, and someone has given a workaround. Login to windows, connect this device in plain cdma mode, and restart, only this time start Ubuntu (I have a dual boot laptop). The device automatically connects to internet on plain CDMA mode (with same connection entry created earlier in network manager)

The device comes with a connection dialer software and GUI that can be used in Windows to select either CDMA evdo, or plain CDMA mode. And so the problem is not there in Windows. The same software binaries and a PDF guide are also provided for fedoracore 6.0 linux.

I was hoping some one technical enough could recompile the binaries for Ubuntu, so that I could use it with Ubuntu OS, or let me know how to do that.

The problem is there with both Ubuntu 10.10 and 11.04. 

My device is Huwei EC150. Any help will be appreciated. Kindly reply to this post or send me an email at sidfadnis[at]gmail.com
