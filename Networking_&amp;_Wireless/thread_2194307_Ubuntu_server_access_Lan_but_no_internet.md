### Post by papibe on 2013-12-17
Thanks.

It looks like this is a VM, right?

If so, it looks like you chose IP addresses and network outside what VMware is offering as a network to support its VMs.

I'm not familiar with VMware, but in VirtualBox, you can define the addresses of the network and solve the problem that way. Another alternative that is available on Virtualbox, is to bridge your network adapter so that works as if it is on the outside physical network as the VM host is.

I hope that points you in the right direction.
Regards.
