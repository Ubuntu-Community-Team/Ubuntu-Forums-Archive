### Post by graysky on 2009-02-01
To answer your question: samba would be just fine.  Further, if you're not partial to Ubuntu 8.10 at all, and if the sole purpose of one or both of your machines is file server, why not give [freenas](http://www.freenas.org) at try?

To get samba working on Ubuntu 8.10 simply install it, configure it, make a samba user, start it.

```
$ sudo aptitude install samba
$ sudo nano /etc/samba/smb.conf
```

Use my smb.conf as a guide:

```
[global]
	load printers = yes
	guest account = nobody
	name resolve order = lmhosts host wins bcast
	passwd chat = *Enter\snew\sUNIX\spassword:* %n\n *Retype\snew\sUNIX\spassword:* %n\n .
	socket options = TCP_NODELAY
	preserve case = yes
	obey pam restrictions = yes
	encrypt passwords = true
	passwd program = /usr/bin/passwd %u
	dns proxy = no
	printing = bsd
	server string = tripwire
	invalid users = root
	unix password sync = false
	workgroup = tripwire
	syslog only = no
	os level = 20
	printcap name = /etc/printcap
	syslog = 0;
	security = user
	panic action = /usr/share/samba/panic-action %d
	short preserve case = yes
	max log size = 1000; By default, the home directories are exported read only. Change next
;	parameter to "no" if you want to be able to write to them.
   read only = no

; File creation mask is set to 0700 for security reasons. If you want to
;	create files with group=rw permissions, set next parameter to 0775.
   create mask = 0755

; Directory creation mask is set to 0700 for security reasons. If you want to
;	create dirs. with group=rw permissions, set next parameter to 0775.
   directory mask = 0755

[home-squeeze]
        writeable = yes
        valid users = squeeze
        path = /home/squeeze

[share]
	comment = share
	writeable = yes
	valid users = squeeze
	path = /share
```

Now make a user for samba (I used squeeze):

```
$ sudo smbpasswd -a squeeze
```

Enter the password when prompted.  Now start samba:

```
$ sudo /etc/init.d/samba start
```

To have it start at boot time, issue the following:

```
$ sudo update-rc.d samba defaults
```

### Post by jmartrican on 2009-02-01
I have a similar setup with the same goal.  While all the comp's were on the same LAN is was going with Samba.  It worked with the Ubuntu's and the Windows, but the MAC never found the computers.  Before I had a chance to troubleshoot I changed the network and now the MAC is not on the same LAN, so I am going to switch to NFS.  

I might be able to help with the Samba.  

In the smb.conf file I added a shared section with the following lines:
```

[your_shared_drive]
   guest ok = yes
   comment = shared drive
   path = /your/saherd/path
   browseable = yes
   read only = no
   create mask = 0755
   directory mask = 0755

```
