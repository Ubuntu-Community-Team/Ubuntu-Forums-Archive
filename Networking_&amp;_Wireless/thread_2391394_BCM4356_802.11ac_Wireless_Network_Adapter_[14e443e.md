### Post by chili555 on 2018-05-21
I'm embarrassed to say that googlesource played a little trick on me and I didn't catch it until today. Above, when we used wget to retrieve the missing brcmfmac4356-pcie.txt file, we retrieved an HTML web page!!! Doh!! I regret my mis-step.

Let's try again. First, delete the wrong file:```
sudo rm /lib/firmware/brcm/brcmfmac4356-pcie.txt
```Now go to this page: [https://chromium.googlesource.com/chromiumos/third_party/linux-firmware/+/f151f016b4fe656399f199e28cabf8d658bcb52b/brcm/brcmfmac4356-pcie.txt](https://chromium.googlesource.com/chromiumos/third_party/linux-firmware/+/f151f016b4fe656399f199e28cabf8d658bcb52b/brcm/brcmfmac4356-pcie.txt) Copy and paste the actual text starting at # Sample variables file for BCM94356Z... and ending at, exactly, rssicorrnorm5g_c1=1,2,3,2,2,2,7,7,8,7,7,8. Paste it into a text editor such as gedit. Save the file as anything convenient, such as brcm.txt.

By default, the file should be saved into your /home/user directory, often abbreviated as ~. Now let's copy it to the firmware directory:```

sudo cp ~/brcm.txt  /lib/firmware/brcm/brcmfmac4356-pcie.txt
```Check:```
cat /lib/firmware/brcm/brcmfmac4356-pcie.txt

```Does the file start and end as above?

Reboot and show us:```
dmesg | grep -e brcm -e wlp
```

### Post by chili555 on 2018-05-23
> Current Frequency:2.422 GHz (Channel 3)Channel 3?? It is very rare for anyone to intentionally set the router to channel 3 as it is an overlapped channel: [https://www.metageek.com/training/resources/why-channels-1-6-11.html](https://www.metageek.com/training/resources/why-channels-1-6-11.html) This leads me to suspect that the router is set to auto channel select. Most Linux drivers hate hate hate auto channel select. Here is a fascinating thread about the subject: [https://superuser.com/questions/1311149/why-do-wifi-routers-do-such-a-bad-job-of-channel-selection](https://superuser.com/questions/1311149/why-do-wifi-routers-do-such-a-bad-job-of-channel-selection)

Please check the settings in the router. WPA2-AES is preferred; not any WPA and WPA2 mixed mode and certainly not TKIP. Second, if your router is capable of N speeds, you may have better connectivity with a channel width of 20 MHz in the 2.4 GHz band instead of automatic 20/40 MHz, although it is likely to affect N speeds. I also have better luck with a fixed channel, either 1, 6 or 11, rather than automatic channel selection. Also, be certain the router is not set to use N speeds only; auto B, G and N is preferred. After making these changes, reboot the router.
