### Post by SeijiSensei on 2019-12-13
First, a reverse domain maps from IP addresses to hostnames.  All the entries need to be of the form
```

1      IN     PTR      myserver.example.com.
10     IN     PTR      myclient1.example.com.
11     IN     PTR      myclient2.example.com.
[etc.]

```
There are no A records in this file. 

You don't need a forwarders directive. The server will direct queries for other domains directly to the root nameservers (though see below).

I believe the listen-on directive takes only host IP addresses, not names like "localhost" or IP networks like 192.168.1.0/24. You should simply list the server's own IP addresses.  

On my server I have this entry in named.conf:

```

zone "." {
     type hint; 
     file "named.root";
};

```
where named.root contains the A records for the root nameservers like
```
A.ROOT-SERVERS.NET.      3600000      A     198.41.0.4
```

You might not need this if you forward unresolvable requests, but I've always done things this way.
