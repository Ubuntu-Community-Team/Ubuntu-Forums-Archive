### Post by crisko on 2009-04-28
I've recently migrated our Zenoss application to a new server. However I am having some trouble getting both NICs up and running.
 
Eth0 is issued a global IP for monitoring various servers. Eth1 has an IP internal to our network for monitoring the infrastructure.
 
/etc/network/interfaces
 
```
auto eth0
iface eth0 inet static
address 202.x.x.11
network 202.x.x.0
broadcast 202.x.x.255
netmask 255.255.255.0
gateway 202.x.x.1
 
auto eth1
iface eth1 inet static
address 10.20.7.99
network 10.20.7.0
broadcast 10.20.7.255
netmask 255.255.255.0

```
 
I am running into two problems - the first is that when eth1 is up, I cannot resolve addresses outside of our network, however, SSH, FTP and such other services are available internally. The tells me there is a routing problem, but I am not sure how to address it (seeing as they are on seperate subnets / networks)
 
The second issue is the eth1 is unable to resolve anything on it's assigned network. I am sure I am missing something, something small. This configuration was working in the past, and I am using the same NICs.
 
dmesg | grep eth results in an error that I have not seen before..
 
ADDRCONF(NETDEV_UP): eth1: link is not ready
 
I just can't get my head around this one..
