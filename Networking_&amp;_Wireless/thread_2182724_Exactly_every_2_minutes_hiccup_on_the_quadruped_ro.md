Hello,

The problem: I have a hiccup (latency spike of 2000ms) exactly every 2 minutes over both of the wireless interfaces, but not over the ethernet interface.
The setup: this linux is running a quadruped robot. ([http://www.youtube.com/watch?v=hPXLpKsiSdI](http://www.youtube.com/watch?v=hPXLpKsiSdI)) That's why some of the things might be unusual. It is an ubuntu nonetheless.
We are running on a roboard RB-110, a xenomai kernel, with ubuntu 10.04.

On this roboard, we have 3 network interfaces: 1 over ethernet with a plain RJ45 cable connected to the university network. 1 over wifi with a pci-card, 1 over wifi with a usb. Both of the wifi-connections provide an ad-hoc network.

The problem: Every 2 minutes (exactly), the latency jumps up to 2000ms, for about two seconds, but only on the wireless interfaces. It is as if for about 2 seconds no packets are being processed. No packets are dropped however.

What I have tried:
- I can detect nothing abnormal in the radiospectrum here.
- when I turn things around, and have the robot connect to another ad-hoc network, it has the same issue.

Does anybody know where to start debugging this? Any help would be greatly appreciated. I just recompiled the kernel in order to compile the driver for the wifi-usb-key, but it appears to have the same problem.

[INDENT]**1)** Is there a specific reason for using an older version with older kernel instead of current ones? If not, it is recommended to upgrade to the latest one available with the latest stable version. That is, 12.04.3, that comes with kernel 3.8.

**2)** I don't know anything about that vt6655 driver. Where did you get it from? Maybe you can try a better supported one. For more info about the driver, please post back the output of -
```
modinfo vt6655
```

**3)** You already are using the recommended proprietary driver for your USB adapter. Quoted it just to let you know that this driver won't compile normally on the latest kernels. It would need some patching, and probably [post=12790346]this one[/post] would do (in case you try one of the latest kernels).

**4)** Could you try channel 1 or 11 instead? I see you are already using channel 11 on the usb one, and since you say the 'hiccup' is same for both, it may not be related, but still 1 or 11 usually work better with Linux drivers (maybe due to less overlapping of available frequency bands).

**5)** Again, this is clearly not related once the device starts working, so consider it a side note. It is interesting to see that while your udev rules file contains double entries for two of the cards, and one top entry for probably a non-existent card, it has none for the usb (ra0). You should probably delete or move it so that a fresh one is generated with only relevant entries, thus not confusing the system with names to assign.

**6)** Since you don't seem to be using IPv6, try disabling it completely as mentioned in this [post=12640479]post[/post].[/INDENT]
