### Post by varunendra on 2014-07-14
> **Mr_Mags said:**
> ```
nm-tool ~~~~~~~~~~~~~~~~~~~~~~~~~~~~

**State: [COLOR="#FF0000"]asleep[/COLOR]**
....
....
NetworkManager.state ~~~~~~~~~~~~~~~
[main]
**NetworkingEnabled=[COLOR="#FF0000"]false[/COLOR]**
```

As highlighted above, it seems the "network-manager" service (or "networking" service) failed to resume after suspend. This may be a bug in network-manager or the scripts handling suspend/resume of services.

Check the "nm-applet" menu (the Network Manager's icon at the top right corner) and make sure "Enable Networking" has a tick mark beside it. If not, click it to enable it.

If it already appears to be enabled, or if clicking it fails to enable it, try to restart the service -
```
sudo service network-manager restart
```
If this does nothing, try restarting the entire networking service -
```
sudo service networking restart
```
..followed by the previous command to restart network-manager service AFTER this, if required.

I am not sure what we can do to make sure the service, whichever is failing to resume (the **/var/log/pm-suspend.log** file may have clues), suspends and resumes properly. But if restarting one of the services manually does the job, we may be close to a permanent solution.

### Post by varunendra on 2014-07-29
> **Mr_Mags said:**
> I ran the "network-manager restart" command and that worked.  What do you suggest I do next?

I suggest you try this script -
```
case "${1}" in
    hibernate|suspend)
        # Stop network-manager service
        service network-manager stop
        ;;
    resume|thaw)
        # Restart network-manager service
        service network-manager start
        ;;
esac
```
Copy-paste the above code into a plain text file > save it as "**nm_restart**" onto your Desktop (no special characters, no extension name in the file) > copy it to "**/etc/pm/sleep.d**" directory and make it executable with the following commands -
```
sudo cp Desktop/nm_restart /etc/pm/sleep.d
sudo chmod +x /etc/pm/sleep.d/nm_restart
```
Reboot > suspend/resume to make sure it works. Report back if it doesn't.

The above script will cause the network-manager service to automatically stop while going to suspend/hibernate, and will start it again as soon as you resume.
