### Post by candtalan on 2011-03-10
This posting is for information:

This laptop has a physical slider switch on the top  edge of the key board which switches wireless function on and off. 
The wireless driver functions for Ubuntu (8.04 and) 10.04.2 out of the box (along with all other functions afaik). 

However, with 10.04.2 the enable/disable functions from the slider switch seem to be asymmetrical in that when running with switch at wireless 'on', and then moving the switch to wireless 'off', immediately disables wireless, as expected. 

But, if the switch is then moved back to wireless 'on', the 10.04.2 does not then re enable wireless, which remains disabled.

This asymmetry was not present in Ubuntu 8.04.x, and the switch operation in this machine then, both enabled and disabled wireless as expected.

The Network Manager top panel icon indicates correctly the current status. 

When attempting to re enable the wireless following a switch to 'off' condition, then to 'on' as described above, then it was not possible to sucessfully re enable the wireless function (without a reboot). 

A re boot of the laptop after the switch was put into the 'on' position resulted in wireless enabled ok.

Details:
Ubuntu 10.04.2, Wireless switch on keyboard.
Vaio laptop:
VGN-SZ1XP/C  
Model PCG-6J1M

Install note:
Machine previously had 8.04.x installed and was reinstalled with 10.04.2 by manually nominating the partition and leaving the target partition format box unchecked (that is, not to be formatted), so that existing user data was retained.
