### Post by e79 on 2010-09-14
Does your **/etc/resolv.conf** file shows proper DNS entries (either your router or ISP DNS) ?

Here's and example from my machine :
```
$ nano /etc/resolv.conf

# Generated by NetworkManager
nameserver 172.25.16.1

```What is your output when typing '**route -n**'  ? Here's the reply I get on my machine :
 ```
$ route -n

Kernel IP routing table
Destination     Gateway         Genmask         Flags Metric Ref    Use Iface
192.168.19.0    0.0.0.0         255.255.255.0   U     0      0        0 vmnet1
172.25.16.0     0.0.0.0         255.255.255.0   U     1      0        0 eth1
192.168.1.0     0.0.0.0         255.255.255.0   U     0      0        0 vmnet8
169.254.0.0     0.0.0.0         255.255.0.0     U     1000   0        0 eth1
