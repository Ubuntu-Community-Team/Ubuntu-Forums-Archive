### Post by tomearp on 2014-01-25
I would move the line that starts use=web... into the global section (top section) and then change the protocol section to match the [example on the namecheap website]("https://www.namecheap.com/support/knowledgebase/article.aspx/583/11/how-to-configure-ddclient") (i.e. lose the commas at the end of lines).

### Post by ssk-copan on 2014-01-25
SOLUTION TO WARNING MESSAGE:  "WARNING:  skipping host: &#65279;: 'login=' is an invalid login."

Since it was already giving me a warning, I never thought to run

```
ddclient -daemon=0 -debug -verbose -noquiet
```

Then you get this message at the bottom:

```
Use of uninitialized value in string ne at /usr/sbin/ddclient line 2102.
```

[SIZE=6]The file /usr/sbin/ddclient [/SIZE]will reveal that there are default settings outside of the ddclient.conf file.  Ddclient is trying to read settings from this file, and they are not set.

See line 335 of /usr/sbin/ddclient[SIZE=6]
[/SIZE]```

    'service-common-defaults'       => {        'server'              => setv(T_FQDNP,  1, 0, 1, 'members.dyndns.org', $
        'login'               => setv(T_LOGIN,  1, 0, 1, '',                  u$
        'password'            => setv(T_PASSWD, 1, 0, 1, '',                  u$
        'host'                => setv(T_STRING, 1, 1, 1, '',                  u$

```

You can then change it to a dyndns account if you have one, or something else.  This will clear the warning.
