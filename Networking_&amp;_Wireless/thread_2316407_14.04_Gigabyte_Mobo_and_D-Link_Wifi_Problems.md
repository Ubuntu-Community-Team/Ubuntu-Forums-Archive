### Post by malikov.ca on 2016-03-18
Finally my situation got resolved. It turns out that my motherboard GA-970A-D3P (rev. 2.x) doesn't fully support 64-bit Linux.

**Solution**: 32-bit Ubuntu (I wanted to try Arch, but it doesn't have a 32-bit version anymore. I decided to use Ubuntu Studio instead and it worked right out of the box, USBs and all.) 

Please let me know if there are issues I may encounter with running a 32-bit Ubuntu nowadays. Also, please let me know if there is a way that I overlooked to run a fully capable 64-bit Linux on my system, since support for 32-bit is disappearing like Arctic ice.

**Problem**: the main problem - why it took a long time to get the system fully working was *not seeing the forest behind the trees*.

The trees that blocked the view:

1. The motherboard has partial support of 64-bit Linux. Everything was working perfectly fine, but not some peripherals, like USBs, ethernet and wifi adapter. Given the modular nature of Linux the partial support implied the possibility of resolving the issues with proper drivers/modules/configs.

2. The USBs didn't properly work - USB 3 worked with IOMMU disabled, but not USB 2, and USB 2 worked with IOMMU enabled, but not USB 3. This was fixed by setting IOMMU=soft (or something like that) in a GRUB config file, and the fact that it was fixed helped further obscure the main issue.

3. Trying to resolve the wifi issue through fixing the network adapter. That was the obvious most common trouble people run into, but a wrong one in my case. Moving my original post to "Networking" from "Hardware" (where I posted it originally) was a mistake. I understand the need to categorize problems, but we need to figure out what the problem actually is first to categorize it properly. I suggest changing the protocol for moderator actions on this forum to avoid such situations.

4. Strange behaviour of Google. It seems to think that I always want something else instead of what I actually type in. (Frankly, I think Google was never better than around 2005--2006. After that it started doing that personalization thing, which is stifling the search experience, in my opinion.) I found the adequate description of my problem with the query "South Bridge: AMD SB950" where I found a thread on a Gigabyte UK (!?) forum with links back to Ubuntuforums, yet those pages had many keywords and phrases I used to identify the problem before.

5. Being new to Linux didn't help, obviously. Besides, I hate this administration-type stuff and procrastinated a lot.


I hope that somebody will find my experience in this matter useful, and I thank Bucky Ball and chili555 for trying to help.
