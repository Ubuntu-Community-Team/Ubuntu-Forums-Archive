### Post by steeldriver on 2014-06-13
You should let resolvconf control the /etc/resolv.conf file for you and not try to modify it by hand - the 127.0.0.1 entry means it's referring DNS queries via a caching nameserver (dnsmasq) running on your machine (via the localhost interface). 

It is dnsmasq that should be picking up the upstream DNS information - and that should all happen behind the scenes, all you should need to do is add your DNS servers via the nm-applet GUI - either by setting the 'IPv4 Settings' --> 'Method' to 'Automatic (DHCP)' (which gets DNS settings via DHCP from your router) or 'Automatic (DHCP) addresses only' (which allows you to enter your chosen DNS server IPs manually in the boxes provided).
