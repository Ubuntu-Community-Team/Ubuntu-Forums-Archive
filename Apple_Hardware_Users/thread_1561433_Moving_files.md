### Post by dngfng on 2010-08-26
By using a File manager - the default File manager for ubuntu is Nautilus. 

This offer drag and drop functionality just as any other File Management system in windows or OSX.
[http://en.wikipedia.org/wiki/Nautilus_%28file_manager%29](http://en.wikipedia.org/wiki/Nautilus_%28file_manager%29)

Also I prefer Gnome Commander - this offers Total Commander like functionality for file management.
[http://www.nongnu.org/gcmd/](http://www.nongnu.org/gcmd/)

Back then, I would only use Linux on my Apple internal file and web servers, which ran headless anyway and did not require a GUI. So I am new to the Linux GUI and prefer the Gnome desktop myself. I am very impressed with what I have seen so far.

I feel to truly enjoy the GUI, there must be support for full hardware acceleration. Even the first OSX release, (10.0) was a terrible experience for me because it lacked HW acceleration on my B&W's rage128 card. Simple window resizings were painfully slow as they were all software rendered.

Today, it's a different story. Many graphics cards are supported and the Debian distributions have the best support for such a niche hardware market. I come from a RedHat/Yellow Dog background and am familiar with the limited support for the Apple hardware I owned.

After successfully replacing my WinXP and RedHat installs on several x86 desktops, netbook and servers with Ubuntu,  I decided to give my older Macs (pre-AGP graphics) a 2nd shot at Linux.
