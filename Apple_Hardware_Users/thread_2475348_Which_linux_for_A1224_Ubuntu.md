### Post by guiverc on 2022-05-24
I have no experience on using GNU/Linux (or Ubuntu) on Apple hardware, but you can try Ubuntu without install.

See [https://ubuntu.com/tutorials/try-ubuntu-before-you-install](https://ubuntu.com/tutorials/try-ubuntu-before-you-install)

I used many C2D devices in QA-testing (*Quality Assurance*) releases up to the *latest* release, from t6570, u9400, e6320, e8400, 6600, e6850, e8300 etc. and am using a 2009 dell desktop right now in this reply, but my most used box used in QA-testing is possibly a 2005 hp.

The RAM however makes a difference, with 4GB being the [minimum for Ubuntu Desktop]("https://help.ubuntu.com/community/Installation/SystemRequirements") (*though lighter flavors can be run with less RAM, eg. my c2d-t6570 has only 2GB RAM*), but with limited RAM you need to match final apps you'll use with your desktop; so they are sharing resources; otherwise as they say in the windows & apple worlds - you need more RAM (*avoiding the technical discussion on libraries & toolkits** & sharing ram*)

I'd boot & Try various *flavors* and see.  As stated I'm not familiar with your hardware and thus don't know what keys need to be pressed; but I've seen this done many times on apple hardware (*at a recyclers*) so we can test devices before selling them on eba*y*.
