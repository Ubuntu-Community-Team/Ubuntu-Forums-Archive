### Post by BlackRockCity on 2009-10-26
I solved the problem!

Reading the man page for ybin got me on the right track.

I changed /etc/yaboot.conf to point to the firewire drive (see below) but I had not run the following command: 

```
sudo ybin -b /dev/sda2 -C '/media/disk/etc/yaboot.conf' -v

```Which resulted in...
```

ybin: Finding OpenFirmware device path to `/dev/sda2'...
ybin: Finding OpenFirmware device path to `/dev/hda3'...
ybin: Installing first stage bootstrap /usr/lib/yaboot/ofboot onto /dev/sda2...
ybin: Installing primary bootstrap /usr/lib/yaboot/yaboot onto /dev/sda2...
ybin: Installing /media/disk/etc/yaboot.conf onto /dev/sda2...
ybin: Setting attributes on ofboot...
ybin: Setting attributes on yaboot...
ybin: Setting attributes on yaboot.conf...
ybin: Blessing /dev/sda2 with Holy Penguin Pee...
ybin: Updating OpenFirmware boot-device variable in nvram...

```Here is my /etc/yaboot.conf after I added the firewire drive device (see my second post in this thread for how I obtained the fully qualified OpenFirmware device name)
```

## yaboot.conf generated by the Ubuntu installer
##
## run: "man yaboot.conf" for details. Do not make changes until you have!!
## see also: /usr/share/doc/yaboot/examples for example configurations.
##
## For a dual-boot menu, add one or more of:
## bsd=/dev/hdaX, macos=/dev/hdaY, macosx=/dev/hdaZ

boot=/dev/sda2
#device=/disk@0:
device=/pci@f4000000/firewire@e/node@0050c502e0002265/sbp-2@c000/disk@0:
partition=3
root=/dev/sda3
timeout=100
install=/usr/lib/yaboot/yaboot
magicboot=/usr/lib/yaboot/ofboot
enablecdboot
macosx=/dev/hda3

image=/boot/vmlinux
    label=Linux
    read-only
    initrd=/boot/initrd.img
    append="quiet splash"

image=/boot/vmlinux.old
    label=old
    read-only
    initrd=/boot/initrd.img.old
    append="quiet splash"
```When I rebooted holding the option key, the Linux Firewire drive appeared to have been 'blessed' and was now listed before the internal OS X drive in the graphical boot disk selection screen. 

Selecting the FW Linux drive sent me to the first boot stage where I hit enter and reached yaboot (the second stage).

Hitting enter again booted Ubuntu but displayed a very messed up psychedelic splash screen for 15 seconds (as though a primitive video driver had loaded) then loaded the User login screen without any graphical problems. Upon login, it says new drivers are available and that I needed the language packs after all. I'll install them and see if the video driver is fixed in the boot splash.

Apple is still reigning champion for plug and play installation! Why didn't the Ubuntu live CD configure yaboot.conf for me? 

Luckily my grandma knows the arcana of OpenFirmware syntax... so I guess most  people will be just fine.
