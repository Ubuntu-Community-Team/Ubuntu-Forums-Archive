### Post by pafufta503 on 2010-06-02
> **funkwurm said:**
> Seeing as Mac OS X is Unix-based and runs X11 (might need to install that from the install DVD) you can run virtually every Linux app on Mac OS X (you talk about wanting to do that somewhere).

Technically, QT4 applications run without any issue.  Let's take Banshee for example, for it to work under OS X you would have to do several odd things.

Firstly you'd have to kill Aqua, OS X's compositing WM, in order to run Gnome.  This means that you are restrained to use only EITHER Aqua or Gnome, but not both.  For some reason they can't run simultaneously.

After that you are still stuck with a mess.  The sound card drivers are different for both OS's, so you'd have to figure out how to get Banshee to work with CoreAudio.  I don't know if this is possible.

To be more precise in your statement, most Linux apps will compile and run under OS X, but each must be ported (requiring writing new code or rewriting old code) to work properly.  MacPorts and Fink do a wonderful job at this, but their repositories are not perfect.  In many cases you will have to manually edit a configuration file, use DBUS from terminal.

It's not totally feasible yet, and most users will find themselves in over their heads in trying to run most Gnome or KDE software apps in OS X.

### Post by sha.goyjo on 2010-06-02
Crozar, I feel like you are missing the very obvious (and indeed essential difference) between Linux, OSX, and Windows.

Linux is free (as in freedom) software.

I know this seems like a basic (and perhaps unessential) point to the end user, but in all actuality, it is exactly what makes it what it is today. Open source has moved faster, further, and easier than any other software philosophy any organization has ever employed. The reason to use Linux is not simply because of what services it can offer you, but because of what it means in the terms of building a community, helping your neighbors, and other very important things of that nature.

Open source software is an sociological infrastructure that allows us to work together, partially in a direct fashion, and partially as a foil to large proprietary software vendors. It's not so much about "what is the feel of this operating system NOW compared to other operating systems NOW", but rather "What can I help build if I choose to support this operating system over another".

Thats what I think. I would rather choose Open Source because I like to build my society in a way that allows people to help other people.
