### Post by linuxopjemac on 2010-05-17
You have to keep PCI:0:16:0 and you should keep the refresh rates of your monitor. Maybe the modelines are not correct.

Try to find the correct ones with cvt (so try to find the modelines for 1152x768 with
```
cvt 1152 768
```

this is an example of some computer with 800x600:

```
 $ cvt 800 600
  # 800x600 59.86 Hz (CVT 0.48M3) hsync: 37.35 kHz; pclk: 38.25 MHz
  Modeline "800x600_60.00"   38.25  800 832 912 1024  600 603 607 624 -hsync +vsync

```
You have to replace that modeline with the one in the config file

You could also try to add in the Device section
```
Option "ForcePCIMode" "True"
```

Just an option, Try setting the render mode to XAA, I believe it is 
```

Option    "RenderMode"   "XAA"

```

But don't count me on that. You can look it up in the xorg.conf manpages.

This option has fixed a LOT of my rendering problems with the mobility radeon cards in the newer versions of Ubuntu, as a switch was made from XAA rendering as the default.

Give it a try, and let us know of your success.
