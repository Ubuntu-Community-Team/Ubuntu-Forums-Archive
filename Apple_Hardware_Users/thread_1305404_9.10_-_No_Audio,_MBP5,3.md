### Post by PhoenixtheII on 2009-10-30
You need to compile alsa on your macbook (Well this has worked for me).
The reason is quite simple, karmic has alsa 1.0.20 while our macbook sound card has been fixed in 1.0.21 :)

Copy paste in console and input password for sudo when needed:

```

sudo apt-get install build-essential

cd
wget ftp://ftp.alsa-project.org/pub/driver/alsa-driver-1.0.21.tar.bz2
tar -xjvf alsa-driver-1.0.21.tar.bz2
cd alsa-driver-1.0.21
./configure
make
sudo make install

cd
wget ftp://ftp.alsa-project.org/pub/lib/alsa-lib-1.0.21a.tar.bz2
tar -xjvf alsa-lib-1.0.21a.tar.bz2
cd alsa-lib-1.0.21a
./configure
make
sudo make install

cd
wget ftp://ftp.alsa-project.org/pub/utils/alsa-utils-1.0.21.tar.bz2
tar -xjvf alsa-utils-1.0.21.tar.bz2
cd alsa-utils-1.0.21
./configure
make
sudo make install

```

After this, reboot.

Goto to your favorite alsa-mixer and unmute your channels. Play around with your sliders and viola you can hear sounds! :)

### Post by PhoenixtheII on 2009-10-30
Add these 2 lines into /etc/apt/sources.list

```

deb http://ppa.launchpad.net/mactel-support/ppa/ubuntu karmic main
deb-src http://ppa.launchpad.net/mactel-support/ppa/ubuntu karmic main

```

Then

```

sudo apt-get update
sudo apt-get install mbp-nvidia-bl-dkms isight-firmware-tools applesmc-dkms hal-applesmc bcm5974-dkms pommed

```

(You probably get nagged about a PGP key, which you can ignore. But you can also add that if you know how, else you can search around for 'apt-key' around the forum :))

This should upgrade/install macbook pro packages.

After that reboot.

Try out your brightness/volume keys:

If F1 & F2 (The brightness keys) now control your brightness you're ok i assume. Fn + F1 should generate a normal F1 keystroke then.

to switch that around (as root):

F1,F2 ect.. = Brightness, Volume ect...
Fn + F1-12 = F1-12
```

echo 1 > /sys/module/hid_apple/parameters/fnmode

```

Or this

Fn + F1,F2  = Brightness, Volume ect...
F1-12 = F1-12
```

echo 2 > /sys/module/hid_apple/parameters/fnmode

```
