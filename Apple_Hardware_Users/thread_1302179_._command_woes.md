### Post by fancypiper on 2009-10-26
It means "in the current working directory".  If you don't know what your directory is, pwd (print working directory).

I modify my bash prompt so that it tells me.  From my .bashrc file:
```
# Change bash prompt. See the article
# http://www-106.ibm.com/developerworks/linux/library/l-tip-prompt/
export PS1='\d \@ \[\e[32;1m\]\u\[\e[34;1m\]@\[\e[36;1m\]\H \[\e[34;1m\]\w\[\e[32;1m\] $ \[\e[0m\]'
```

### Post by ad_267 on 2009-10-26
. means the current directory.

So replace . with the full path to where the executable is.

Eg, I have an executable at ~/Stuff/executable.sh

I can do:
```
cd ~/Stuff
./executable.sh
```
or
```
~/Stuff/executable.sh
```

### Post by x C0MMAND0 x on 2009-10-27
instead of running
```

cd /home/user/scripts
./executable-in-sctips-folder.sh

```

or

```

cd ~/scripts
./executable-in-sctips-folder.sh

```

Run the following:

```

./scripts/executable-in-sctips-folder.sh

``` (since you are already in /home/user/ folder.)

Don't forget, the TAB button is your friend in terminal and is VERY useful for finishing off commands to make sure they are correct and recognized by the system.
