### Post by psusi on 2010-10-05
This is a MAC, not a PC, so grub needs to be installed in the partition, not the MBR.  MACs don't boot the MBR, they load EFI modules from the EFI system partition, which includes rEFIt, which can then boot grub from the partition.

Not a MAC user.

Since it still is installed on a partition, the issue of blocklists and the possibility of the Grub files being moved and no longer being found by Grub remains an issue. That being the case, I would probably recommend reinstalling Grub2 to the partition, since it will update the location of the Grub files:

After booting to the normal Ubuntu OS:
```
sudo grub-install --force /dev/sda4
sudo update-grub
```
Note you will still get warning messages, even using the "force" option.
