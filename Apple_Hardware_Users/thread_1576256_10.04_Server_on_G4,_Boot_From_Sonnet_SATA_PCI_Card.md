### Post by linuxopjemac on 2010-09-17
If you look carefully at the yaboot.conf file you see that the device section is wrong:

```
device=/disk@0:
```

You have to find out the correct path to the device. What you have to do is boot in a live environment and issue the ofpath command:

```
ofpath /dev/sda3
```

I am now not so sure if you have to find the correct path to /dev/sda3 or to /dev/sda, you can try both options.

This will give you the correct device path to /dev/sda3

With an editor you have to then change yaboot.conf. I would chroot into your system from the liveCD, edit the file and then issue ybin:

```
sudo ybin -v
```

to write the changes into the bootloader


Reference:
[http://mac.linux.be/content/yaboot](http://mac.linux.be/content/yaboot)

### Post by linuxopjemac on 2010-09-18
you forgot a few things chrooting into the system:

> ubuntu@ubuntu:~$ sudo mkdir /mnt/root
ubuntu@ubuntu:~$ sudo mount -t ext4 /dev/sda4 /mnt/root
ubuntu@ubuntu:~$ sudo mount -t proc none /mnt/root/proc
ubuntu@ubuntu:~$ sudo mount -o bind /dev /mnt/root/dev
ubuntu@ubuntu:~$ sudo chroot /mnt/root /bin/bash

then :

sudo ybin -v

It seems like the ofpath entry in yaboot.conf is correct.

You can indeed try to find the name of the device with devalias and dev / ls in open firmware. It should have the same address.

It could also be that you suffer from a problem that is known to G5:
[http://mac.linux.be/content/ofpath-fails-detect-correct-openfirmware-path-some-powermac-g5](http://mac.linux.be/content/ofpath-fails-detect-correct-openfirmware-path-some-powermac-g5)
