### Post by Jibize on 2009-01-19
Hi there,

  I've got a problem which is driving me totally mad! I did a fresh install of Ubuntu Hardy on my ibook G3 v2.2 with a Radeon Mobility M7 LW [Radeon Mobility 7500] AGP 2x, and everything worked great excepted that it froze time to time (had to do a hard reboot) and the glxgears gave me an error saying 

```
Error: Could not get dma buffer... exiting ...
```

ending with a the same type of kernel panic, couldn't do anything but moving the mouse, hard reboot needed... So I edited my xorg.conf file, and had the classic tweaks for radeon driver, but it didn't change anything. Desperate I used the command 

```
sudo dpkg-reconfigure -phigh xserver-xorg
```

and tried glxgears again and it worked!! So I added a few lines in my xorg.conf and installed DRIconf to get a better FPS and reached 970 FPS, which looked fine to me. I then decided to move on and put the ibook on my home network, so I installed samba, smbfs, and winbind, restarted my laptop and noticed that the Mac was going a bit slow again, tried glxgears and I was back to scratch. Did a reset of the xorg.conf to default with the same command line, and it didn't change anything that time!!! it's now my 2nd days looking on the web for a solution, but I'm stuck!! the only interesting thing I found is that if I set up my xorg.conf file in a manner that it makes the 3d performance very slow the system is stable and I can run glxgears but with an output of 85 FPS (see the BadWorkingXorg file).

  I've got something else, which I'm not sure is related to my problem, during the boot (it actually appeared since the first boot after the fresh installation), between the Ubuntu splash and the login screen, I've got a very strange blurry white screen, and then the login screen is normal.
