### Post by GuyMontag on 2010-12-10
First off I am definitely a newbie when it comes to Ubuntu/Linux ect, so I apologize if this has a ridiculously simple fix. I have Macbook 2,1 that's currently dual booting OSX 10.6 and Ubuntu 10.10 using ReFit. I've successfully been able to troubleshoot the sound and a few other issues, though this one has stumped me for the past few days. 

I'm trying to access files (specifically music and photos) that are stored on my OSX partition. When I've booted in Ubuntu and try and access the files I'm told that I don't have permission to access the files. After rebooting in OSX I turned on file sharing via sys preferences, I also have SMB file sharing enable and have added the folders I wish to share to the shared folders list. However when I go back to Ubuntu and mount my OSX disk I'm still unable to access the folders as they are x'ed out. I tried the fix featured here [http://ubuntuforums.org/archive/index.php/t-958457.html](http://ubuntuforums.org/archive/index.php/t-958457.html) but still end up getting the same error after trying to access the folders. 

Edit: After experimenting some more using 'gskusudo' in the terminal then running nautilus I'm now able to browse the files on my OSX partition through Ubuntu. I'm un-restricted in the folders/files I can view, though I still get the same error if I attempt to copy/view any of the files. 

Any help or hints would be greatly appreciated; if I've left anything out that's important let me know. Thanks!
