### Post by stream303 on 2008-07-25
I don't know how I missed this one in Hardy - I normally never use the keyboard audio controls, and rely solely upon the software volume controls on my G5 iMac, be it the master gnome volume, or alsamixer in a terminal.

I left pulseaudio activated, and once I started using the keyboard controls, I got into some weird interaction, that even alsamixer couldn't fix on it's own.  I had to use a combination of keyboard and alsamixer to get things back to normal temporarily.

Ok, PulseAudio had to go.  Disabled it in System > Preferences > Sessions (and reboot) to get back to the older Alsa audio.

So far so good.  The keyboard controls work better, but still not 100%, so I again rely mostly on Alsamixer in a terminal.  It is still working better than with PulseAudio enabled.  Maybe I'll try PulseAudio again in Intrepid.

In any case, the weird interaction between the keyboard setting not letting Alsamixer properly balance the audio freaked me out - but for now I'm running without PulseAudio and life is better.
