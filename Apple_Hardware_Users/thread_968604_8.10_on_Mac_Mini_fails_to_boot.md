as you can see from the output... the first command shows the contents of the GPT. It shows 2 partitions (as it should). The second command shows the MBR partition table. It is not correct. GRUB relies on the MBR partition table to boot, and since it is broken, it cannot boot.

You have three options.
1. boot from a livecd, and use the gptsync tool to sync the partition tables (or install refit to to it). You can also manually build the MBR partition table with fdisk, but i wouldn't recommend that.
2. look into booting from EFI directly. I think this is better supported on the minis than other macs.
3. Do a complete reinstall, but use gparted first to convert the disk to a normal, non-GPT disk.

Just some amplification -

The installer seems to be installing grub to the mbr ok, but it is getting the mbr partitioning (above 446 bytes) wrong. (In the case where you have a GPT partitioned drive).

A refit usb stick or CD, and running the partitioning tool (gptsync) will fix this, but if you dont have one -

From ubuntu, gptsync corrects this -

Get gptsync for ubuntu by installing the refit package.

```

sudo apt-get install refit

```
(or use synaptic)

Then

```

$ man gptsync
$ sudo gptsync /dev/sda

```

And restart with the Option key.
