### Post by mchladek on 2008-06-05
Before you make changes to your xorg.conf file, execute the following:

```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.bak
```

This way you'll have a backup of a config file you know works.  Now, here's my entire xorg.conf file:

```

Section "InputDevice"
	Identifier	"Generic Keyboard"
	Driver		"kbd"
	Option		"XkbRules"	"xorg"
	Option		"XkbModel"	"pc105"
	Option		"XkbLayout"	"us"
	Option		"XkbVariant"	"mac"
	Option		"XkbOptions"	"lv3:ralt_switch"
EndSection

Section "InputDevice"
	Identifier	"Configured Mouse"
	Driver		"mouse"
	Option		"CorePointer"
	Option		"Device"		"/dev/input/mice"
	Option		"Protocol"		"auto"
	Option		"ZAxisMapping"		"4 5"
	Option		"Emulate3Buttons"	"false"
EndSection

Section "Device"
	Identifier	"ATI Technologies Inc M9+ 5C63 [Radeon Mobility 9200 (AGP)]"
        Driver		"radeon"
	BusID		"PCI:0:16:0"
	Option		"UseFBDev"		"false"
        Option		"AGPMode"		"4"
        Option		"AGPFastWrite"		"true"
	Option		"AccelMethod"		"exa"
	Option		"RenderAccel"		"true"
	Option		"AddARGBLXVisuals"	"true"
	Option		"XAANoOffscreenPixmaps"	"true"
	Option		"DynamicClocks"		"true"
EndSection

Section "Monitor"
	Identifier	"Configured Monitor"
EndSection

Section "Screen"
	Identifier	"Default Screen"
	Monitor		"Configured Monitor"
	Device		"ATI Technologies Inc M9+ 5C63 [Radeon Mobility 9200 (AGP)]"
EndSection

Section "ServerLayout"
	Option		"AIGLX"	"True"
	Identifier	"Default Layout"
	Screen		"Default Screen"
	InputDevice	"Generic Keyboard"
	InputDevice	"Configured Mouse"
EndSection

Section "DRI"
	Mode	0666
EndSection

Section "Extensions"
	Option	"Composite"	"Enable"
EndSection

```

Yours will look a little different since you have a different video chip and I did a bit of tweaking to get compiz-fusion working.  I think the main change that will be relevant to you is to make sure under "Configured Video Device" you specify "radeon" as the driver.

Note, however, that this change alone did not get the brightness keys or suspend working for me.  As I stated elsewhere, I had to change my yaboot.conf file to the following:
```

boot=/dev/hda2
device=/pci@f4000000/ata-6@d/disk@0:
partition=3
root=/dev/hda3
timeout=50
install=/usr/lib/yaboot/yaboot
magicboot=/usr/lib/yaboot/ofboot
enablecdboot

image=/boot/vmlinux
	label=Linux
	read-only
	initrd=/boot/initrd.img
	append="nosplash video=radeonfb:1024x768-24@60"

image=/boot/vmlinux.old
	label=old
	read-only
	initrd=/boot/initrd.img.old
	append="nosplash video=radeonfb:1024x768-24@60"

```

Don't forget to backup your original yaboot.conf file before making changes!
```
sudo cp /etc/yaboot.conf /etc/yaboot.conf.bak
```

Also don't forget to excute the following to make your changes permanent:
```
sudo ybin -v -C /etc/yaboot.conf
```

### Post by stream303 on 2008-06-07
As a temporary workaround to keep your eyesight until the real fix is found, you can try to adjust the gamma and fake it:

```
xgamma -gamma 0.5
```

Try values between say 0.5 and 1.5, 1 is the default.

I definitely wouldn't do any photo editing with a funky gamma setting, but this might help.

It isn't a real fix.  Xubuntu has a similar control that might prove useful, but obviously a real fix for brightness is needed.
