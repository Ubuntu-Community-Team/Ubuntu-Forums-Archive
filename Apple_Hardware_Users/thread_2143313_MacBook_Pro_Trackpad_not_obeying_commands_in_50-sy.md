### Post by pvasudev on 2013-05-08
I'm running Ubuntu 12.04 LTS on a MacBook Pro, and (as it seems) many of you know that the trackpad mouse is annoyingly over-sensitive.  I wanted to change that, and enable some other features,  I've done the following:

1) Tried creating a file xorg.conf in /etc/X11/ which looked like: 
```

Section "InputClass"     
Identifier         "Touchpad"     
Driver             "synaptics"     
MatchIsTouchpad    "on"     
Option         "ClickFinger3" "2"     
Option         "HorizTwoFingerScroll" "1"     
Option         "VertScrollDelta" "85"     
Option         "HorizScrollDelta" "85"     
Option         "TapButton1" "0"    
 Option         "TapButton2" "0"     
Option         "TapButton3" "0"    
 Option         "FingerHigh" "8"     
Option         "FingerLow" "8"
 EndSection

```
After this, Ubuntu would not reboot and so had to go into recovery mode prompt to delete this file.  Then Ubuntu started up.  (Thus, making an xorg.conf file in /etc/X11 is NOT the answer.)

2) Edited /usr/share/X11/xorg.conf.d/50-synaptics.conf.  Made it look like:
```

# Example xorg.conf.d snippet that assigns the touchpad driver
# to all touchpads. See xorg.conf.d(5) for more information on
# InputClass.
# DO NOT EDIT THIS FILE, your distribution will likely overwrite
# it when updating. Copy (and rename) this file into
# /etc/X11/xorg.conf.d first.
# Additional options may be added in the form of
#   Option "OptionName" "value"
#
Section "InputClass"
        Identifier "touchpad catchall"
        Driver "synaptics"
        MatchIsTouchpad "on"
# This option is recommend on all Linux systems using evdev, but cannot be
# enabled by default. See the following link for details:
# http://who-t.blogspot.com/2010/11/how-to-ignore-configuration-errors.html
      MatchDevicePath "/dev/input/event*"
      Option "SHMConfig"  "true"
       Option "VertTwoFingerScroll" "1"
       Option "HorizTwoFingerScroll" "1"
       Option "MinSpeed" "1"
       Option "MaxSpeed" "1.8"
       Option "AccelFactor" "0.03"
       Option "TapButton1" "0"
       Option "TapButton2" "0"
       Option "TapButton3" "0"
       Option "VertScrollDelta" "-65"
       Option "HorizScrollDelta" "-65"
EndSection

Section "InputClass"
        Identifier "touchpad ignore duplicates"
        MatchIsTouchpad "on"
        MatchOS "Linux"
        MatchDevicePath "/dev/input/mouse*"
        Option "Ignore" "on"
EndSection

# This option enables the bottom right corner to be a right button on
# non-synaptics clickpads.
# This option is only interpreted by clickpads.
Section "InputClass"
        Identifier "Default clickpad buttons"
        MatchDriver "synaptics"
        Option "SoftButtonAreas" "50% 0 82% 0 0 0 0 0"
EndSection

# This option disables software buttons on Apple touchpads.
# This option is only interpreted by clickpads.
Section "InputClass"
        Identifier "Disable clickpad buttons on Apple touchpads"
        MatchProduct "Apple|bcm5974"
        MatchDriver "synaptics"
        Option "SoftButtonAreas" "0 0 0 0 0 0 0 0"
EndSection

```

But the problem is the computer doesn't seem to recognize ANY of the changes I specified above except for the cursor speed and acceleration.

I know I can set these parameters this way because going
```

synclient VertTwoFingerScroll=1

```
(and so on and so forth) sets up the parameters right and it works.  But I don't want to have to type it in everytime I boot up.

3) Like at the top of 50-synaptics.conf file said, I put an original copy of 50-synaptics.conf into /usr/share/X11/xorg.conf.d/ and copied the edited version I have above into a directory /etc/X11/xorg.conf.d/ and it still doesn't work.

Can someone please help me to make Ubuntu recognize the 50-synaptics.conf file, either in /usr/share/X11/xorg.conf.d or in /etc/X11/xorg.conf.d?

Thank-you!

### Post by ole01 on 2013-05-23
I just have had the same issues while trying to configure natural scrolling. The way I solved this, is to create a bash script which sets the parameters with the help of synclient. This script is executed every time the touchpad is "hotplugged" (after reboot, wake from suspension, relogin ...). Due to this behaviour, it overwrites the hard coded default settings, which can't be reconfigured by using the xorg.conf files (as I found out).

The script named naturalscrolling.sh is located in ~/bin (which is /home/ole01/bin for me) and has the following contents:
```
*#! /bin/bash*


*synclient="/usr/bin/synclient"*

        
 *$synclient VertScrollDelta=-***243**
*$synclient HorizScrollDelta=-***243**[I]
$synclient HorizTwoFingerScroll=1  # activates horizontal scrolling with two fingers[/I]


```

Here you can find many other options for synclient: [https://wiki.archlinux.org/index.php/Touchpad_Synaptics](https://wiki.archlinux.org/index.php/Touchpad_Synaptics)

The script file has to be made executable. Then start a terminal and enter the following:
```
*gsettings set org.gnome.settings-daemon.peripherals.input-devices hotplug-command "/home/ole01/bin/naturalscrolling.sh"*
```

I think your desired options should work in the same way as mine do.
Then try a reboot and it's done.
