### Post by OrangeJuicer on 2012-02-25
I needed to install ubuntu on an external drive and I thought it would be easier from my live CD than OSX, my normal OS. I restarted my computer and held down c so it would boot from the CD. It waited and I could hear the CD running then showed a file with a ? on it and booted OSX. I tried again but got the same result so I went to boot options in System Preferences and selected the CD. I did what I needed to do but, when I tried to boot back to OS X neither holding down option or using arrow keys in BIOS had any effect (I'm guessing this is because of a BIOS update that does not support USB keyboards). Since it kept timing out and defaulting to a memory test I booted back from the live CD I am posting this from. The only ways of changing BIOS defaults I can find involve access to locked files or using my keyboard before the step in booting where it asks if I want to install Ubuntu or try it. Any suggestions???
EDIT: forgot to mention, /boot/grub/ only contains grubenv

---

### Post by OrangeJuicer on 2012-02-25
Fixed it! Since I last opened Ubuntu I had replaced my keyboard with a new apple one which mac boot settings don't work with.:confused: Pulled out the old one and it worked fine.:P
