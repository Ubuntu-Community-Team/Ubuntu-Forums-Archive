So back to your yaboot fun! I don't dual boot because I'm lazy and I have enough scrap machines I can play with... :) Anyway, from what I was reading earlier: > [COLOR=#000000][FONT=Times New Roman]In order for OpenFirmware to automatically boot Ubuntu the bootstrap partition should appear before other boot partitions on the disk, especially MacOS boot partitions. [/FONT][/COLOR]

Your /dev/sda11 bootstrap is after the /dev/sda10 osx partition, so open firmware is never going to see yaboot. It will just boot osx. So according to the doco: > [COLOR=#000000][FONT=Times New Roman]you can use [/FONT][/COLOR][COLOR=#000000][FONT=Times New Roman]**mac-fdisk**[/FONT][/COLOR][COLOR=#000000][FONT=Times New Roman]'s [/FONT][/COLOR]**r**[COLOR=#000000][FONT=Times New Roman] command to reorder the partition map so the bootstrap partition comes right after the map (which is always partition 1). It's the logical map order, not the physical address order, that counts.[/FONT][/COLOR]

I have not done that before so YMMV. It does make sense from the times I have messed around with open firmware though. Once you get the order correct, it should load the yaboot file and away you go, (hopefully). I also have found that it helps bunches to make changes to yaboot.conf by booting linux single, edit the file, then issue your ybin -v.
