Below is the general outline for rebuilding a package, you can google for it. Glib is a bit different so I am currently rebuiling it myself as a lot of gnome software will use it.

Find the package that offending library is in using search in the package manager. Scratch that does not list it, through google it is libglib2.0-0 and a big sucker it is. It houses many C cross-platform utility functions, I have looked at it before but used the Boost libs instead but that does not fix your problem. 
Exit the package manager, create a build directory and cd into, then to get the source
apt-get source <package>
Get the build dependances for building the package
sudo apt-get build-dep <package>
Apply the Ubuntu specific patches to the original sources
dpkg-source -x <package>-ubuntu-verson.dsc
This should extract all the sources and apply the patches.
Change to the source directory.
Check in debian/rules for any altivec or runtime-detection options. Set any altivec options to no-altived. In your case there is none.
To build it
fakeroot debian/rules binary
The new deb package will be created above the current directory so cd ..
and can run dpkg -i <package>.deb
