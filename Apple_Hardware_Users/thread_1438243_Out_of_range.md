I don't use contrib or non-free in Debian, but do use software that may have patent "issues" I play DVD's, watch h264 video, can listen to mp3 etc. But use free software to do it.

From the link in my sig: This example refers to "squeeze" but could equally apply to "lenny" etc 

> deb [http://www.debian-multimedia.org/](http://www.debian-multimedia.org/) squeeze main
deb-src [http://www.debian-multimedia.org/](http://www.debian-multimedia.org/) squeeze main
#comment: This is a third party repository. It contains possibly patent encumbered code that is not included in Debian by default. Even though if may be free-software. (Were talking patents here not copyright.) You will also note that like many third party repositories, it does not respect the main, contrib, non-free protocol (even though I've not included them above, I could have). With all third party repositories it is up to you to check their status as to free-software vs. non-free, (There is an application in the repositories called "vrms" that can aid you in this.) It's up to you to check the license, security, and quality etc., of third party packages. The "Backports" repository falls under third party repository also. 

Once You've added debian-multimedia or the backports repositories to your debian system you could install say... k9copy

k9copy in debian main will not rip encrypted DVD's
K9copy from debian-multimedia will. Because it includes the libdvdcss2 and libdvdread3 library's. See dvd-john:   [http://en.wikipedia.org/wiki/Jon_Lech_Johansen](http://en.wikipedia.org/wiki/Jon_Lech_Johansen)  and [http://en.wikipedia.org/wiki/DeCSS](http://en.wikipedia.org/wiki/DeCSS)

Both versions are free software, but the version from debian-multimedia may have code that in-fringes a patent (no-matter how crap) in SOME countries. 

As Debian is USA based any code that MAY infringe on a patent will not be in Debian main repo. 

Finland-Norway for example and countries with less draconian laws, may not recognise the patent so it be perfectly legal. Adding debian-multimedia repos offers easy to access free-software with possible patent issues, at the expense of less clarity over free vs non-free than you get from the official Debian sources.

If your running ubuntu DO NOT add Debian repos to your sources.list you WILL screw up your system.

### Post by oswaldkelso on 2010-04-18
This is Debian. Login as root not user to configure your sources.list. 
Also note sudo is not used by default like ubuntu.

as root:

cd (change directory) to your sources.list

```
cd /etc/apt
```

Make back-up copy 

```
cp sources.list sources.list-bu
```

Open the sources.list file and edit it. 
i.e. add the the repo to it.
