### Post by mfox on 2008-11-26
I think that if Linux was my primary OS and I only had occasional need for the Mac OS, I would go for dual boot rather than virtual Linux running on the Mac side.  There is the reason you gave (saving RAM for what you need it for), plus virtualization is going to give you some speed hit.  But if my need for the Mac OS is regular, virtualization has a real advantage of giving access to both at the same time.  If you do your virtualization on VMware Fusion, you have an extra advantage - drag and drop movement of files between OSes.  That's a real time saving.

If Linux is your primary OS, I would think the way to go would be running the Mac OS on a virtual machine, but I don't think you can do that on an Intel Mac.  On a PPC Mac, you do have that option, with the Mac-on-Linux program (runs on Debian, though not Ubuntu newer than 7.04).
