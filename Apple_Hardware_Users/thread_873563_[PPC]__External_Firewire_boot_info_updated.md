### Post by stream303 on 2008-07-29
Anyone wanting to boot from an external firewire drive, and needing to modify the yaboot.conf on the firewire drive, may have been reading my very old thread about it:

[http://ubuntuforums.org/showthread.php?t=314237](http://ubuntuforums.org/showthread.php?t=314237)

I have a much easier way to find the firewire node: just use lshw.

```
sudo lshw | less 
```

and look for the *-scsi to see it.  In my case, it shows up like this:

     ```
*-scsi
          physical id: 6
          **bus info: firewire@0001d202e0a60004**
          logical name: scsi4
```

My old method of using of using a find /proc/device-tree sometimes didn't work with install images.

Save your sanity and use lshw instead. :)

download and gunzip the attached ofpath revision to a directory.
```

$ gunzip ofpath.gz

```

check that ofpath is executable, it should be.
If not
```

$ chmod +x ofpath

``` 

Then you can run the unzipped ofpath to get the correct external path

```

$ ./ofpath /dev/sda

```

Then use that path in yaboot.conf.



ofpath-

Now that you have a working revised ofpath (pxw) utility, you should replace the broken original  /usr/sbin/ofpath, (rename and save the old one first, it was part of the yaboot package). Then check it by running ofpath -V.

Then the yaboot installer utilities will have correct path info, for example you can use yabootconfig  to create a minimal yaboot.conf and write the boot partition files from scratch, enough to get booted into ubuntu, from where you can add the minimum necessary options (e.g. macosx, enablecd, ... ) to yaboot.conf and rerun ybin, without needing to bother with openfirmware path information ( see man yaboot.conf, ybin, mkofboot, bootstrap).
