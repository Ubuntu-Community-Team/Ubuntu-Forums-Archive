### Post by pteryges on 2017-09-27
> **ardouronerous said:**
> Mmmmm, I wonder why Linus was able to install Ubuntu on a Macbook then?

Totally different beast. Since the switch to Intel, Macs are just Apple's PCs. They use (mostly) the same hardware as any Windows machine. An iPad is a custom piece of Apple-designed hardware, including the CPU. While they use some of the same components as other tablets, the CPUs are Apple's version of ARM, and the integration is Apple's own.

### Post by kevin160 on 2017-09-27
I've tried running various custom Android and Linux distros on an HP Stream7 tablet.  It works, but there are wireless and touch screen driver issues and it simply isn't worth the trouble.  Yes, Win10 is better than Ubuntu on it, sad to say.  And that's a pretty standard Intel/EFI machine which is as easy as it gets.  

iPads and iPhones are iPod touches don't have Intel processors in them and don't run macOS.  They run on ARM processors and run iOS, and are locked down pretty tight compared to most other tablets.  

While it is *technically* possible for someone to run an ARM linux distro on an iPad, I haven't seen anything like this other than old projects like iDroid that put Android on really old hardware.  

There was a project called iBox that let you run virtual guests inside bochs on iOS although that project seems to be pretty much dead.  I imagine it's painfully slow.  You could try [https://github.com/Enthri/iBox-2](https://github.com/Enthri/iBox-2) though.  

Your best bet at this point would be to use your iPad to remotely connect to a linux machine with vnc or whatever and share screens.  Spend your time finding the best way to get touch support working well.
