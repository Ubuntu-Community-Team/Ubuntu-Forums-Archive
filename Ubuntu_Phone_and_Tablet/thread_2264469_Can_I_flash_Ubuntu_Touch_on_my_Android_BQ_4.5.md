### Post by grahammechanical on 2015-02-07
It would be an experiment. You do understand that? Do you not?

[https://developer.ubuntu.com/en/start/ubuntu-for-devices/installing-ubuntu-for-devices/](https://developer.ubuntu.com/en/start/ubuntu-for-devices/installing-ubuntu-for-devices/)

To get the same version of Ubuntu as going on the BQ Aquaris E4.5 you would need to set the channel to the RTM channel. I believe that the RTM (Release To Manufacturer) channel is the location for the code that goes specifically to the OEMs.

[https://developer.ubuntu.com/en/start/ubuntu-for-devices/image-channels/](https://developer.ubuntu.com/en/start/ubuntu-for-devices/image-channels/)

I do not think that you will get the BQ specific apps but I do not know. There should be a Ubuntu app store and a BQ app store from what I have heard. ubuntu-touch/ubuntu-rtm/14.09 is the code going into the BQ phone.

Canonical will provide updates to the OS. Twice a month images will be built and tested and once a month the OS will be updated. When 15.04 is released then the RTM image will be re-based on the Ubuntu 15.04 code which will be pushed onto the phone. Right now it is based on the utopic code. These phones will not be running old versions of the OS. Unlike Android phones.

> - Custom Ubuntu image channels are based on the regular Ubuntu releases but are either customized or separate from the project's six-month cycle. An example is the Ubuntu RTM, which was branched off the utopic channel to create the Release To Manufacture (RTM) distribution that will become the first release preinstalled on devices.

Regards.

> [COLOR=#333333][FONT=Ubuntu Beta]I still have an Android Version of the bq phone reflashed with Ubuntu. Still a bq phone, but you actually have even before anybody at Canonical the very first production of Ubuntu Phones.[/FONT][/COLOR]

> 
[COLOR=#333333][FONT=Ubuntu Beta]**A few people asked me, if they can buy Android Version of the bq phone and flash it with Ubuntu?**[/FONT][/COLOR]
[COLOR=#333333][FONT=Ubuntu Beta]They could. The problem is that &#8230; obviously we have a version of Ubuntu which you can always download from the web. Unlike on the desktop where you encourage people to download and use it every day, on the phone side it's for developers. It is the same codebase, but what a manufacturer does with scopes is that they create an experience thats unique to the device. So for example the bq device is launching with an experience that has a news scope, photos scope, video scope, music scope, NearBy scope for location services, that has a much richer experience than the version you download from the web that has an app scope and a whole bunch of blank scopes.[/FONT][/COLOR]
[COLOR=#333333][FONT=Ubuntu Beta]It's a developer build which we have on the web, while the user build is the one that goes into the devices. So we actually can't fully appreciate what the Ubuntu Phone is all about unless you get your hands on a preloaded Ubuntu Phone from bq.[/FONT][/COLOR]
