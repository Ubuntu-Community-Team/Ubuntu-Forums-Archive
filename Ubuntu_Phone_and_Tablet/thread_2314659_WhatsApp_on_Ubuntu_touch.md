### Post by branau on 2016-02-22
I think if we sit around waiting for WhatsApp to release an app for Ubuntu touch, it'll never happen. Luckily for us, I think there's a way to build an unofficial client for Ubuntu touch, and I think it would be a hit because I've seen plenty of requests for. I myself find its absence to be more than a slight annoyance, and would greatly appreciate a client for it. Would anyone be willing to undertake this task with me? I tried out FirefoxOS for a little while, and some guys there have put together a [web-based client]("https://github.com/mozillahispano/openwapp") for it, and it worked just fine for me. They don't have any license listed for their project, so I've requested that they add one, because it could be as simple as wrapping it up in a web-container and porting it to Ubuntu Touch. If they don't put a license on it or put a restrictive license on it, we can at least take a look at their code for ideas and build a native client. A native client may even be better, because I'm not sure if Ubuntu Touch has compatibility with notifications from web sites. Any thoughts?

EDIT: I've also stumbled upon a [Python library]("https://github.com/tgalal/yowsup/") that looks like it might still be in active development.
