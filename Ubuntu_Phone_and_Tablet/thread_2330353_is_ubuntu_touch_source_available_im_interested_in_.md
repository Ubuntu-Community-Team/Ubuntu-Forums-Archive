Id like to get a group together and start working on an entire touch based distro from scratch, optimized for the newer core m-m3 and broadwell/skylake i5/i7.

I think the reason you guys arent massively distributing it is probably because its experimental without good hardware support.

but the way linux is, tablet linux will not grow if you are simply selling pre built tablets with it on there. the REAL storm will come when someone applies my idea and creates a free-for-download tablet distro that can be installed on some or most (however many possible) already existing x86 and x86_64 tablets.

most people wanting linux on a tablet will not pay 300$-600$ for a new tablet. they want the asus transformer book they got for $150-$350 to run faster, more efficiently and more like a tablet than a PC, while still allowing them the flexibility of a 2 in 1. I know that myself and a LOT of other people are waiting for something like this. but until it happens, android_x86, phoenixOS, and remixOS (all android based Operating systems) are the only serious OS alternatives for these new 2 in 1s. a

while I dislike the unity desktop... a lot might already be done in the touch version that would make the project easier to work with or learn from at the least.

im honestly more influenced by iOS, android and remixOS than any linux desktop currently existing. I basically intend on recreating an android like OS, but instead of creating a "walled garden" I want it to be full linux, full freedom. I think in theory, the linux community would be an amazing place for tablet development, I think we can achieve greater design successes than corporate entities because we are users, and the bottom line is effectiveness and LACK of cost.

where corporations need to limit user experience to make money, sell more devices, software. all we want as a community is to write good software that does what we want, how we want. the linux sales model is that less sales means more successful. we dont need to create a tablet OS that still leaves the need for a desktop. the greatest success of linux would be to absolutely destroy commercial sales models by taking all the succesful designs users want in one place, and putting them there. 

to make a tablet as fast and efficient and touch friendly as android or iOS that still allows currently existing code to run.

imagine a cross between remixOS (multitasking android, taskbar, windows) and iOS -- now run this imaginary desktop on the linux kernel. it can have the simplicity and speed of a mobile OS, while still offering full 100% compatibility with any existing linux apps and all the hardware too.

its a simple idea really. forget the "full desktop" nobody with a tablet needs or wants it. they just want the tablet, with the addition of multitasking, and software/hardware compatibility. this is the ultimate tablet. one that can run any desktop app the same way its meant to be run, but at the same time, has an entirely mobile interface. I imagine it could be more successful and ambitious than currently existing mobile OS software, simply because the linux community doesnt give a damn about profits, selling apps or devices. we just want solid, secure computing experiences.

whats more is with limited use of resources and a minimalist window manager -- it could still look absolutely beautiful if influenced by current simple mobile designs, and be extremely fast on even cheap hardware. it would be the equivalent of putting slackware on an 800mhz pentium from 15 years ago and running only icewm with no desktop compared to windows ME.
