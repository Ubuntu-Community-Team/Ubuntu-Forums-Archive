### Post by grahammechanical on 2015-08-04
There should be a feed-back app in the app store. Why did you buy the phone when it clearly does not meet your expectations?

At the moment this phone is for early adopters. The OS is new. It is different. It is not meant to compete with Android. To put it another way. Canonical does not have the financial resources to compete with Google, Apple or Microsoft when it comes to the number of programmers it can employ. The development of the Ubuntu phone OS and the applications on it is very much a community project as well as a product developed by Canonical.

This OS gets transactional updates. There are no Alpha or Beta stages. What you have is an OS that will continually be updated as long as Canonical is able to update Ubuntu. This OS does not use the Debian package method. So, if software such as Firefox is to become a Ubuntu phone app then the developers of Firefox will have to take responsibility for porting Firefox to the Ubuntu phone and that means it will have to be packaged as a Click app and run in confinement and meet all the other security requirements that all Ubuntu phone apps comply with. And Firefox will have to meet the design standards for Ubuntu apps as well.

And it gets worse, depending on your point of view. The underlying software technology of the Ubuntu phone is coming to the Ubuntu desktop. Mir will replace X and Compiz. Snap packaging replaces Debian packaging. Apps will run in confinement. The OS will get transactional updates with an automatic roll back should an update fail in some way. The future is a different country. 

The Ubuntu phone OS is a proving ground for future Ubuntu desktop technology. There will be one code base for all devices. Applications like Firefox will no doubt be there on a Ubuntu desktop but if they do not scale to the phone or tablet form factor then they will not be available on the phone or tablet.
