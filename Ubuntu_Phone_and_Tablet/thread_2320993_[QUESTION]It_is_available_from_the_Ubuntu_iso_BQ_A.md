### Post by grahammechanical on 2016-04-19
Ubuntu for Devices, which is installed on Ubuntu phones & tablets is not available as an installable ISO image in the same way as desktop Ubuntu is available as an installable ISO image.

The BQ images are written specifically for those BQ devices. Which by the way use ARM processors. And that may not be the case with your Windows 10 tablet. 

There is talk of the phone/tablet code being converged into the Ubuntu desktop code. The intention is to have one code base that will work as a phone OS when installed on a Ubuntu phone, a tablet OS when installed on a Ubuntu tablet and a desktop OS when installed on laptops & desktop machines. This should come as an installable ISO image but what phone or tablet devices it could be installed on is not known. We will have to see what the next 12 months brings.

Last year some work was done on something called Ubuntu Personal. It was not an installable ISO image but an image file. I was able to use the Ubuntu Disks utility to restore the image to a hard drive. It took over the whole hard drive wiping out everything else on it. Which I did not mind as I was experimenting with a spare drive. And I knew that would happen.

This was like having the Ubuntu phone OS on a desktop machine but it was built on Snappy Ubuntu core. It had limited usefulness and development work was stopped six months ago. We have been told that development work will restart in May once Ubuntu 16.04 has been released. This project may develop into an installable ISO image that will install on Intel & AMD x86 CPU machines. It could be the Ubuntu of the future. Time will tell. 

If you are interested there are sessions on convergence at the Ubuntu Online Summit 3rd to 5th May. See what the Ubuntu developers are talking about on this subject.
