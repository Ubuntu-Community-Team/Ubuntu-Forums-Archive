### Post by grahammechanical on 2015-08-07
I do not think that we should consider the Ubuntu phone OS as following the same development schedule as the Ubuntu Desktop OS. The first Ubuntu phone code was based upon Ubuntu 14.10 and it was only after desktop 15.04 was released that work began to migrate the phone OS to the 15.04 code base. I think a similar situation will apply when Ubuntu desktop 15.10 is released.

The Ubuntu phone uses a different method of updating and the Click package method of packaging the apps. But Canonical has evolved Click into Snappy packaging and at some point the Ubuntu phone OS will switch from Click to Snappy but Ubuntu desktop will still be using the Debian (deb) package method.

I think this shows that the phone OS and the desktop OS are following different paths even though they are supposed to be converging paths. So, I do not think we should be trying to co-relate OTA updates with Ubuntu release dates. OTAs are more frequent. The OS needs continuing improvement.
