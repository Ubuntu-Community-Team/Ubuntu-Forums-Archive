### Post by wlbi on 2017-01-11
if it is a i386/amd64 Tablet, You can install the regular Ubuntu Desktop Version.
If it is a ARM tablet, you can't do this, because ARM devices not including hardware detection for the operating system.
That's why you can't install Ubuntu Touch just on any other Android device. The operating system will work only for one specific device with a specific hardware. If you change some slight things at the hardware, you need a new operating system, fitting to this hardware.

### Post by Bucky Ball on 2017-01-11
> **wlbi said:**
> The operating system will work only for one specific device with a specific hardware. If you change some slight things at the hardware, you need a new operating system, fitting to this hardware.

Not actually true and slightly 'Windows world' thinking. As long as the target device is the same architecture as the original device (for instance i386) you can copy the installed OS and install it on the other device without issue (for instance, if you have an install on a desktop and you want the same install on a PC, copy and clone). 

As Ubuntu has all the drivers contained in the OS kernel, unlike Windows where you need to install drivers for different hardware, first boot on the new device will pick up the hardware and load the drivers required, providing they are in the kernel. RAM, hard drives and most other hardware don't matter if the processors are the same architecture. ;)
