### Post by andrew-q2 on 2015-11-22
I noticed the app Wobzip which unzips files including password protected zips, so I thought I’d try it.  Needing a zipped file, and with password for good measure, I emailed myself one.
 
First difficulty – how do you just save an attachment?  TOUCH offered to open it with an app, but of course there was none.  I suppose Doc Viewer might handle this one day, but offering the photo viewer etc. wasn’t going to help.  I could see no way to simply save the attachment – does anyone know how please?  It was not appearing in Downloads or amongst trojika attachments.  Why does Dekko not just give the option to save the file somewhere, like desktop email would?  People shouldn’t restrict what we can do by allowing only for the user journeys the developer can think of.
 
I connected the MX4 to my ‘puter via usb and copied the zip file over.  I went into File Manager and tapped it.  It offered to extract the contents, but this failed.  A second option was to open with an app, but it said there were no suitable apps, even though WOBZIP was installed, and .zip is amongst the file types it handles.
 
So I went into WOBZIP, with the internet connection off, and all I got was a blank screen.  I don’t see why I need the internet to unzip a file!  Why is this app linked to Google Adsense?  I thought Touch wouldn’t have the data stealing nonsense you get with say Android apps.  (Ok, I gather the HERE apps take data, but these are sophisticated bits of software, I don’t expect to be given these for nothing – though I suppose you could say they’re part of the overall phone purchase/deal)
 
I enabled wifi, repeated the test, and it uploaded the file somewhere and said it had done that “about 5 hours ago” so I think some time zone bug here?  It seems the file was then unzipped on the website and I was given these choices:
- delete upload
- download as zip (what!!!)
- send to dropbox.
There seemed to be a link to the unzipped file itself, but nothing happened on tapping it.
 
All this is nonsense.  The function of the app (I thought) was to unzip, but you don’t seem to get the contents back once they’ve been spirited away for who knows what purposes.
 
WOBZIP says the max size is 200Mb, so if I unzip a 199Mb archive, it seems we have 199Mb to upload, and who knows how much to come back (assuming you can ever get it back).  This is silly technology.  Canonical need to get a grip surely?  The apps should be sensible and they should integrate properly.
 
System details:  MX4, OTA-8.
 
On a positive note, I think some things are snappier since OTA-8, an initial reaction.

Andrew
