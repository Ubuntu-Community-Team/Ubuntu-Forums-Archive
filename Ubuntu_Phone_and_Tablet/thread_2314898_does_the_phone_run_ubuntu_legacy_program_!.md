### Post by grahammechanical on 2016-02-24
The answer is no.

At present all apps on Ubuntu phones have to be packaged as Click apps. In the near future apps on Ubuntu phones and tablets will be have to be packaged as Snap apps. Click packaging is a Ubuntu development of the Debian packaging method. And Snap packaging is an evolution of Click packaging.

Ubuntu convergence devices such as the soon to be available BQ M10 tablet will be able to run Debian packed X apps. They will run on Xmir inside a Linux container for security.

If a Ubuntu phone model has the hardware to make a HDMI connection to a monitor then the screen display will switch modes and the Ubuntu phone can be used as a PC if a bluetooth keyboard and mouse are connected. The BQ M10 Ubuntu tablet will do the same.

Watch the video. See convergence in action on the BQ M10 Ubuntu tablet and on a Nexus 4.

[http://www.omgubuntu.co.uk/2016/02/ubuntu-tablet-convergence-xda-video](http://www.omgubuntu.co.uk/2016/02/ubuntu-tablet-convergence-xda-video)

If the phone hardware does not support HDMI connections to a monitor then a user will have to wait until a method for connecting to a monitor using WiFi is developed. It is being planned.

The code for convergence is in the Ubuntu for devices OS but is all depends if the phone hardware can make use of it. Remember, all of this is still very much early days and it is being done in the open.

HDMI over WiFi. Yet to be implemented on Ubuntu phones & tablets.

[https://en.wikipedia.org/wiki/Miracast](https://en.wikipedia.org/wiki/Miracast)
