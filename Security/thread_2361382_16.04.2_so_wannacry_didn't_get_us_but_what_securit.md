### Post by &amp;KyT$0P# on 2017-05-15
That's not the best way to think about Linux security.  More like, what steps should you take to maximise security of your system?

DuckHook and TheFu have done a lot of good posts on Linux security.  Here's one - [https://ubuntuforums.org/showthread.php?t=2184758&p=12833795#post12833795]("https://ubuntuforums.org/showthread.php?t=2184758&p=12833795#post12833795")

I would have only a few things to add to that.

1) For hardening Firefox, the available extensions have changed a bit since DuckHook wrote that post.  I recommend installing [NoScript]("https://addons.mozilla.org/addon/noscript/") and [uBlock Origin]("https://addons.mozilla.org/addon/ublock-origin/").  That combination alone should give you good security.  If you're an advanced user you might also want to add [uMatrix]("https://addons.mozilla.org/addon/umatrix/").

2) Sandboxing is sometimes better with AppArmor and sometimes better with firejail.  It depends on what you're sandboxing and what sort of sandbox you need/want.  For example, I use AppArmor profiles for VirtualBox and various system services, but I use firejail for web browsers.

3) Once you have applied all the good security practices on your system, keeping backups (yes, plural) can be a form of security - [https://ubuntuforums.org/showthread.php?t=2361182&p=13644816&viewfull=1#post13644816]("https://ubuntuforums.org/showthread.php?t=2361182&p=13644816&viewfull=1#post13644816")

### Post by TheFu on 2017-05-31
Firefox has some insecure architecture choices that were made long ago related to scripting and addons.  Not much we can do about it.  Basically, any addon can access private data from other addons.  "Can" doesn't mean that anyone does, at least that we know.

I use firefox as my daily use browser, when on a network I trust and only for visiting sites I semi-trust.
I use Chromium-browser inside a firejail for all online financial stuff.  If I'm connecting to my banks and broker, then I won't restart the session.  If I need to deal with ugly sites that DEMAND too many trackers and javascript, then I'll restart a firejail --private chromium session - necessary for hotel and airport reservations.  You get the idea.

If I want the fastest, quickest, lite-est browsing, but still with images, then I use dillo.  Dillo solves all sorts of web security problems because it doesn't support javascript.  It also doesn't validate SSL certs, so don't use it if you need that point-to-point security.

Sometimes I don't want images.  Lynx is my browser for that.

When it comes to system security
 - versioned backups that are "pulled", never pushed or directly accessible by the client. 
 - stay patched on currently supported OS releases.
 - nothing can replace a smart, aware, human, using the computer.

There ain't no security software that can replace a human brain.
