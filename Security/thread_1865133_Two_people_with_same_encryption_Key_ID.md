### Post by movieman on 2011-10-19
If I remember correctly, the key ID is a 32-bit hash of the key (or 32 bits taken from a longer hash). So you'd expect to see a match about every 10,000 keys. That is, with 10,000 keys in the database you'd expect about one match between two keys.

Of course I may be remembering wrong :).
