### Post by newhere_m on 2012-06-08
In my ubuntu 10.04 OS, evince is in apparmor-enforced mode as seen from the output of command, sudo aa-status:
```

13 profiles are in enforce mode.
   /usr/bin/evince
   /usr/bin/evince-previewer
   /usr/bin/evince-thumbnailer

```I sometimes notice in the messages section (opened with system log viewer) these comments:
```

type=7854 audit(3394434117.372:10):  operation="open" pid=4260 parent=7165 profile="/usr/bin/evince-thumbnailer" requested_mask="::r" denied_mask="::r" fsuid=1000 ouid=0 name="/etc/texmf/dvips/"

type=7854 audit(3394434117.372:10):  operation="open" pid=4260 parent=7165 profile="/usr/bin/evince-thumbnailer" requested_mask="::r" denied_mask="::r" fsuid=1000 ouid=0 name="/etc/texmf/"

```Surprisingly these appear even before I try to open any pdf file with evince in that particular session. Can anybody please tell me if these are generated by apparmor and if so, what I should do?

(I guess all are familiar with dvips, the command used to generate  postscript file from a dvi file. This command is needed at a step to  create printable documents starting from input file written in latex.)

### Post by Dave_L on 2012-06-08
Those are apparmor messages.

Maybe evince-thumbnailer uses those files to generate nautilus thumbnails?

> requested_mask="::r" denied_mask="::r" fsuid=1000 ouid=0

That means that a process owned by uid 1000 (normal user) is trying to read a file owned by uid 0 (root), and the apparmor profile has no rule to permit that. 

If that's a problem, you could add a rule to the evince profile to permit read access to the directory /etc/texmf.
