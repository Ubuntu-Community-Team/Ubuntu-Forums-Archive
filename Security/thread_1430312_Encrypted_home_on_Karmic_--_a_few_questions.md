### Post by ibuclaw on 2010-03-15
> **maplon said:**
> Hi

Sorry if this has been answered elsewhere.

During the installation of Ubuntu Karmic, I picked the option that encrypts my home directory. 

A few questions:

(1) Shortly after installation, I was asked to run a command to print a key necessary for data recovery from a rescue CD. I didn't run it at the time and am now looking for the command to run. What is it?

```
ecryptfs-unwrap-passphrase
```
Then type in your user password, and the autogenerated one will unwrap and print on the screen - this is what you need to make a note of incase you require recovery.

The autogenerated passphrase is quite long (32 characters, last time I checked).

> **maplon said:**
> 
(2) I think I read somewhere that this also encrypts swap. Great. Correct me if that's wrong. 

I don't believe that is true with the encrypt option from the installer. Though it can be done, that requires some extra work, but simple to implement.

(edit):
Check the /etc/fstab file. If the swap line shows a "/dev/mapper" location, then chances am it is encrypted. If it instead shows a "UUID=" location, it is not.
> **maplon said:**
> 
(3) If I suspend the machine, is my home directory encrypted? That is, if I have this on a laptop and travel with the suspended laptop and someone steals it, are my data safe, or not?

Suspend bypasses encryption. In the corporate world, we disable suspend on encrypted devices for this reason, and advise users to instead hibernate.

### Post by Agent ME on 2010-03-15
> **ibuclaw said:**
> Suspend bypasses encryption. In the corporate world, we disable suspend on encrypted devices for this reason, and advise users to instead hibernate.
On a computer with a completely encrypted hard drive, hibernate would be the best option, but on a system with an unencrypted hard drive and user(s) with encrypted home directories (which I assume is the setup of the original poster), I would think that both suspend and hibernate are just as vulnerable as each other.

Hibernate on an unencrypted hard drive will dump all ram to an unencrypted swap space, and actually might even be easier to exploit than suspend because an alternate boot disk could be used to read the swap space, where as reading the ram from a suspended computer takes special equipment.

Best thing to do is to log off your user account when you're not using it. If you're not logged on, your encrypted files should be completely safe because they aren't mounted with their key in memory.
