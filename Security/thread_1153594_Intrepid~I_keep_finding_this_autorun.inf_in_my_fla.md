### Post by bobince on 2009-05-10
It's definitely a Windows virus. As well as the autorun.inf file you will find virus files hidden inside the RECYCLER folder on the device. (May not be visible under Windows as the Recycle Bin is a ‘special’ folder. Also many of the current wave of autorun-infectors install use rootkit techniques to hide their own files.)

If the files keep reappearing, the WinXP box you are using it with is certainly infected. Whilst you can try running anti-virus on it, the sad fact is today's AV tools are getting less and less effective. If you've been getting multiple infections on that box it is quite possible no combination of AV will be able to clean it automatically for you, and the only ‘safe’ response is to reinstall the operating system on that machine.

You should also disable Autorun on all Windows PCs you are responsible for. It's possibly the stupidest, most insecure feature ever to grace Windows. After years of complaints, and finally the last year of rampant autorun-infectors, Microsoft are finally [nerfing it]("http://blogs.msdn.com/e7/archive/2009/04/27/improvements-to-autoplay.aspx") in Windows 7 - though sadly, not completely removing it: you'll still be vulnerable to Autorun on optical media (thanks Sony).

The particularly bad part is that until a [recent patch]("http://support.microsoft.com/kb/967715"), both of Microsoft's published ways to disable Autorun were [dangerously incomplete]("http://www.us-cert.gov/cas/techalerts/TA09-020A.html"). The ‘Services\cdrom\Autorun’ value wouldn't affect pen drives, and the ‘Turn off Autoplay’ group policy, though it would stop the code on a removable device from executing automatically, still allowed the rest of the Autorun infrastructure to replace actions on the device. So attempting to open the drive would still immediately infect you.

The reliable method to properly disable Autorun is given at [CERT's vuln log]("http://www.cert.org/blogs/vuls/2008/04/the_dangers_of_windows_autorun.html").
