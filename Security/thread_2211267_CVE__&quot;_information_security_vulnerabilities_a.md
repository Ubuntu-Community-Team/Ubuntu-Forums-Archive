 CVE Now Available in CVRF Format
  The [CVE List]("http://cve.mitre.org/cve/cve.html") is now publishing CVE content using the  [Common Vulnerability Reporting Framework (CVRF)]("http://www.icasi.org/cvrf"). Developed by the Industry Consortium for Advancement of Security on the Internet ([ICASI]("http://www.icasi.org/")),  CVRF is an XML-based standard that enables software vulnerability  information to be shared in a machine-parsable format between  vulnerability information providers and consumers. 
 CVRF is currently used by major vendors, including  [Red Hat, Inc.]("http://www.redhat.com/"),  [Microsoft Corporation]("http://www.microsoft.com/"),  [Cisco Systems, Inc.]("http://www.cisco.com/"), and  [Oracle Corporation]("http://www.oracle.com/"), which issue their security advisories in CVRF format.
 Having vulnerability information in a single, standardized format  speeds up information exchange and digestion, while also enabling  automation.
