At a glance, it didn't look like the default filters are supposed to ban based on those events. What makes fail2ban really powerful, though, is you can configure it to ban based on any logged event.

This should work, but I don't make any guarantees you won't get any false positives.

create the file /etc/fail2ban/filter.d/apache-proxy.conf
```

[Definition]
failregex = ^<HOST>.* "GET http://
            ^<HOST>.* "POST http://
            ^<HOST>.* "CONNECT
ignoreregex =

```

add this to /etc/fail2ban/jail.conf
```

[apache-proxy]
enabled = true
port    = http,https
filter  = apache-proxy
logpath = /var/log/apache*/*access.log
maxretry= 4

```

restart fail2ban
```

sudo /etc/init.d/fail2ban restart

```
