### Post by TheFu on 2017-07-23
ssh is picky about file and directory permissions. Check those.

Sometimes an example is clearer.

```
host  osmc
  user osmc
  hostname 172.22.22.28
  port 22
```

I use **$ ssh osmc** (the host alias) becomes **$ ssh osmc@172.22.22.28**.
If the port was non-standard, that would be filled in.  All ssh-related commands will do this. rsync, scp, sftp, rdiff-backup, anything using librsync ...
