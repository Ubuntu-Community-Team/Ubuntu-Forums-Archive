### Post by OpSecShellshock on 2010-07-25
Every so often, any software or OS can exhibit signs of a bug that can, under certain circumstances, lead to information disclosure, arbitrary code execution, privilege escalation, or use of unintended memory space. Most of the time it's not a big deal on this particular OS given that it'll either only be exploitable by an already-legitimate local user or involve non-default or otherwise not recommended configurations. Either way, though, fixing it is better than not fixing it. Not every development team insists on waiting for active wild exploitation before fixing bugs.
