### Post by lemming465 on 2009-01-01
At a guess, it means that there is a character set mismatch between your terminal and the chkrootkit output, possibly due to stuff in log files.  Try **LANG=C sudo chkrootkit** and see if you get better looking output.  Or pipe it into *less*.

Naturally, when I run chkrootkit on the system I'm typing this from, I get output in plain ASCII.  But your output is typical of the kind of stuff that happens when you cat binary files to a terminal emulator with character set options; I've had similar things happen to me in other contexts.
