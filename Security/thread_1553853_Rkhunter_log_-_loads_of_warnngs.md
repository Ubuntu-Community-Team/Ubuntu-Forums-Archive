### Post by cdenley on 2010-08-16
```

[23:39:29]          Found string 'hdparm' in file '/etc/init.d/bootlogd'. Possible rootkit: Xzibit Rootkit

```

Well, obviously it suspects /etc/init.d/bootlogd might be infected with that rootkit because it contains the string "hdparm". You can either compare that script with a known good copy, like from an initscripts package with a matching version and architecture, or simply check anywhere that string appears in your script and see if it was used legitimately.
```

grep hdparm /etc/init.d/bootlogd

```
In my script, it only appears in one of the comments.

And for the warning:
```

[23:36:25] Warning: The command '/usr/sbin/adduser' has been replaced by a script: /usr/sbin/adduser: a /usr/bin/perl script text executable

```
It tells you exactly what it means! /usr/sbin/adduser is a PERL script. That is all.
