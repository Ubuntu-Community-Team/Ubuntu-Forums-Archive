Short answer: the defaults for ufw/gufw are overkill for the vast majority of desktop users.

[ZeroConfPolicySpec - Ubuntu Wiki]("https://wiki.ubuntu.com/ZeroConfPolicySpec") 

You can test it yourself with nmap =)

zenmap if you want a graphical interface

[http://nmap.org/bennieston-tutorial/](http://nmap.org/bennieston-tutorial/)

Otherwise you would need to give us a lot of additional information about your network, what services you are running, and what it is you are expecting to accomplish exactly with a firewall.

### Post by jhayes on 2010-09-20
man that was hard to look at. you said you were headed to firestarter? 
i would do something like this 
"Save your firewall rules to a file

# iptables-save > /etc/iptables.rules

At this point you have several options. You can make changes to /etc/network/interfaces or add scripts to /etc/network/if-pre-up.d/ and /etc/network/if-post-down.d/ to achieve similar ends. The script solution allows for slightly more flexibility. " [https://help.ubuntu.com/community/IptablesHowTo](https://help.ubuntu.com/community/IptablesHowTo)

then sudo /etc/init.d/networking restart
to clean up the iptables rules. or simply iptables -F
then turn on firestarter and go from there.
