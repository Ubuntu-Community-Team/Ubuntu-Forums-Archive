### Post by oldsoundguy on 2011-02-11
In a nutshell .. unless you are running a front end server that will also serve Windows machines, there is no need for anything other than a firewall.  As you get used to the system, you will realize that there is no way to install something hidden or by accident.  The steps need to install anything are the primary reasons that the script kiddies don't bother writing virus/malware files for Linux.
(the argument that there are not enough Linux computers out there to justify writing is FUD .. over 58% of all servers now run Linux .. so it IS a big target .. just that up to now, unreachable.)

Remember that AV programs detect only programs that they KNOW about .. so really, there is no protection from a NEW and unknown virus in Windows.

Having said all of this .. there is a lot of crud out there that is BROWSER specific.  So using common sense is called for there! (but, unlike Windows, it can't get past the browser!)

### Post by rookcifer on 2011-02-13
Let's put it this way.  In my years of reading Linux security forums (like this one), I have never seen a **single** case of someone being infected with malware.  A lot of people think they have malware, but I have never seen a single case where it were actually true (most of the time it is a hardware issue or a false positive).

The only time I remember hearing about any Ubuntu machine being compromised by malware is the famous Gnome-look trojan hidden in a screensaver.  This just goes to show that you should be very careful installing something that's not in the official repos.  But I have never seen a single case of "drive-by" malware in any Linux distro.
