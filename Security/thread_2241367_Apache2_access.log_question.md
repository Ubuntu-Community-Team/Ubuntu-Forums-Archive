### Post by SeijiSensei on 2014-08-27
Do you have squid listening on port 80?  Why not the default port 3128?

If you don't want to specify a custom port, use transparent proxying.  It's not that hard and well documented on the web.

Also if those requests are coming from outside via squid, that's a pretty major security hole.  You need to use ACLs in squid.conf to limit the network ranges to limit access to the proxy to just machines behind the firewall.  If you must allow access from outside the network, use authentication.
