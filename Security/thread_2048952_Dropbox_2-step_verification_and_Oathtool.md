### Post by Erik1984 on 2012-08-27
Dropbox now offers 2-way verification, just as Gmail does. To login you now need your username and password + a one time key. That is either generated by an authenticator app on your smartphone or generated by Dropbox and sent through SMS. 

In their help page ([https://www.dropbox.com/help/363/en](https://www.dropbox.com/help/363/en)) they describe the process, and here is one interesting part:

> **For our advanced users**

 
**Generating a security code from the command line**

 Those of you using a Unix or Linux shell might consider generating a security code using the [OATH tool]("http://www.nongnu.org/oath-toolkit/man-oathtool.html"). This way you can generate a security code from your computer safely within the comfort of the command line.


How does this work? I guess you need something as input for generating a security code, and that something must be only known to you. Else an attacker can just use oathtools as well to generate a valid code. Dropbox seems to provide noting that could serve as input for security code generation. 

My question is: how to use oathtool for Dropbox and is it safe?
