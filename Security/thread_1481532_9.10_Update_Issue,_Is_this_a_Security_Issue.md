### Post by schwascore on 2010-05-12
> W: GPG error: [http://ppa.launchpad.net](http://ppa.launchpad.net) karmic Release: The following signatures couldn't be verified because the public key is not available: NO_PUBKEY 6AF0E1940624A220
W: A error occurred during the signature verification. The repository is not updated and the previous index files will be used.GPG error: [http://security.ubuntu.com](http://security.ubuntu.com) karmic-security Release: The following signatures were invalid: NODATA 1 NODATA 2

These lines indicate that for the first repository, [http://ppa.launchpad.net](http://ppa.launchpad.net) karmic, there is no public key available. This is typically a repository you added recently but you did not import the public key. The key should be available from the ppa you are trying to use. Come to think of it, your repository shouldn't just be [http://ppa.launchpad.net](http://ppa.launchpad.net) karmic, it should have the ppa project directory on the end of the URL. 

Here's an example of a legitimate ppa repository URL: deb [http://ppa.launchpad.net/chromium-daily/ppa/ubuntu](http://ppa.launchpad.net/chromium-daily/ppa/ubuntu) jaunty main
