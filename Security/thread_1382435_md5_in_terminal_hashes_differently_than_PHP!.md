### Post by medicdave on 2010-01-15
Hi Friends-

I'm trying to generate MD5 hashtext within gnome-terminal that will match that generated by PHP running on a live web server. However, when I hash the same text I get completely different results!

```
<?php md5('testing123') ?> ===> 7f2ababa423061c509f4923dd04b6cf1
bash$ echo 'testing123' | openssl md5 ===> bad9425ff652b1bd52b49720abecf0ba
```

Could this be a character set problem? My terminal is running UTF-8, but even if I change it to ISO-8859-1, the hashtext that's generated is the same. Also tried using md5sum with a text file, saved in various character encodings, but still got the same [wrong] hashtext.

I'm running Ubuntu 9.10, tried running the local hash in both gnome-terminal 2.28.1 and the CTRL+ALT+F1 console.

TIA,
[medic]dave

---

### Post by FuturePilot on 2010-01-16
By default echo does a \n (new line). Try it with echo -n. It should work.
```
$ echo testing123 | openssl md5 
bad9425ff652b1bd52b49720abecf0ba

$ echo -n testing123 | openssl md5 
7f2ababa423061c509f4923dd04b6cf1
```
