### Post by toofun on 2012-09-20
[B]Ubuntu 12.04 usermod and first default user
[/B]             
                                                                I locked myself out of root access by issuing the command:
usermod -G bind myuser
instead of 
usermod -a -G bind myuser   (this appends the groups instead of using only this group)

By doing so I no longer have root access, and  oddly enough the admin group is no longer in the /etc/group file. I attempted to use the instructions at
[http://ccollins.wordpress.com/2007/0...ubuntu-groups/]("http://ccollins.wordpress.com/2007/07/02/restore-default-ubuntu-groups/")

only to find out that beginning with Ubuntu 12.04, the sudo group is used to give the first user created on the system root access instead of the admin group as in the past.

Question is, does the admin group even get used any more?

### Post by CharlesA on 2012-09-20
The admin group is still in 12.04 due to backward compatibility, but I do not believe it will be used starting in 12.10 and on.

The admin group should be listed as adm:

```
charles@Precise:~$ id charles
uid=1000(charles) gid=1000(charles) groups=1000(charles),4(adm),24(cdrom),27(sudo),30(dip),46(plugdev),109(lpadmin),124(sambashare)

```

Debian doesn't use an admin group either.
