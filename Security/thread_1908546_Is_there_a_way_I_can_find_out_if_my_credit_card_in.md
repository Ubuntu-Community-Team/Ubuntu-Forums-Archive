### Post by unspawn on 2012-01-14
> **venom104 said:**
> Is there some type of program I can use on Ubuntu to check for malicious scripts (even browser ones) so I can see if they stole my credit card information from ubuntu?
As the replies in this thread do not answer this question directly and the implications of this question encompass more than "running a script", allow me to focus on this for sec.

Linux by default offers user (lastlog), current login (utmp), logins (wtmp), shell history, PAM, service and Desktop Environment-based (think IMAPS or OpenSSH daemons, GNOME, KDE and other application) and kernel logging. 
While a more thorough approach (less suitable for new Linux users), trying to make sense of what the Linux architecture offers in terms of separation of privilege, time stamps, correlating disparate information from what is logged and looking for unlinked remnants on disk may provide more but the results (depending on the nature of the threat of course) of just a reactive response ("running a script") without prior configuration will likely yield a less than expected amount of clues. And anything the user can easily unset or evade like shell history, kludges like using 'script' can not be trusted.

In short: no matter what recent major distribution you use, it does not come out-of-the-box with the kind of exhaustive logging enabled that creates all-encompassing audit trails, or even shorter: 
*you don't know what you don't log.*

More elaborate logging is possible layering a combination of MAC (SELinux, TOMOYO, GRSecurity, AppArmor), Audit service, network filtering (NFQUEUE-like, IDS or proxy) and shell logging no unprivileged user can evade using for example Rootsh on top of a base system. This however comes at a considerable cost (more than average system knowledge, more restrictive user movement, properly hardened system, performance drain, disk space requirements or remote syslog). This short list of additional measures implies more work than it seems at first glance and isn't all-encompassing at all but I hope it at least provides some hooks for those interested.

HTH

### Post by OpSecShellshock on 2012-01-14
Yeah I guess the main reason it's hard to answer the original question is that there are so many different ways to get credit card information, most of which don't really need the local desktop computer itself to be compromised. Credit card info couldn't be copied from the system itself unless it had been put there manually in the first place. Most browser-based mechanisms for copying it off site involve phishing or compromise of a site that is storing the details, neither of which are located on the user's system and so consequently couldn't be detected there. It's possible that some sort of browser extension could be causing it I suppose, but most of those get exposed eventually, and anyway I think Mozilla and Google both check their browser add-ons that are delivered by their official channels (no guarantees with regard to how rigorously they do so, of course).
