### Post by unspawn on 2011-12-27
```
operation="open" (..) requested_mask="::r" denied_mask="::r" fsuid=1000 ouid=0 name="/etc/texmf/dvips/"
```

> **arroy_0209 said:**
> Can anybody please explain what this message signifies and if this is generated by apparmor, what should I do about this?
I don't use AppArmor much (more familiar with SELinux) but since nobody posted a response yet here's what I make of it: a process tried to open /etc/texmf/dvips/ for reading but was denied access based on access permissions. Unless the directory holds privileged information (I doubt that but you decide) check if the access permissions need changing to allow unprivileged users read access. 'ls -ld /etc/texmf/dvips' or 'stat /etc/texmf/dvips' should show what it's now (may be 0700 octal), you may want 0755 for directories and 0644 for files: 'find /etc/texmf/dvips -type d -print0|xargs -0 -iX chmod 0755 'X'; find /etc/texmf/dvips -type f -print0|xargs -0 -iX chmod 0644 'X';'. 

*'man chmod' for more nfo.
