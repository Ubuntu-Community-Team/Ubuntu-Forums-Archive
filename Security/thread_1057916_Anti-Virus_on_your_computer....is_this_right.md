### Post by e1mer on 2009-02-06
Don't be too caviler about Linux being virus proof.

I have been getting attacked by a trixbox (linux based) system that is
compromised.

The deal is that by running your computer as a user that can't install programs, you reduce the risk of system level compromise.  

That is a big part of why Ubunto discourages logging in as root.
You can still get malware, but you have to actively try to.

The idea is that with Windows, there are so many systems out there,
and so many security holes, the malware writers primarily target them
because they are easy.  As Linux use goes up, the bad guys will turn
to us.  
It is very important that you keep your Ubuntu system up to date
with the most recent security updates.

### Post by cariboo on 2009-02-06
ClamAV is a command line program. If you need a gui you can install ClamTK. To update clamAV open a terminal and type:

```
sudo freshclam
```

This installs the updater as a cron job, freshclam will start up every night and check for and download update.

To scan a directory and it's subdirectories in a terminal type:

```
clamscan -r
```

There are many options to clamscan the easiest way to see them is to type:

```
clamscan --help
```

### Post by Chayak on 2009-02-07
I really need to just write a generic response in a text file and paste it in whenever this comes up.

My job is malware analysis.  That's malware as a broad category ranging from a virus, trojan, bot, worm, etc.  I will say that having anti-virus is better than nothing but not by much.  The flaw in most AV is that they rely on specific signatures for detections.  It's really trivial for someone knowledgeable to go into a PE (windows format .exe) and add a few bytes to throw off all of the AV engines or simply pack or armor it with custom software.  Then there are no detections what so ever.  If you can run a file through something like VirusTotal then so can malware authors and they can tweak until nothing is detected.

The point is that there is a huge amount of malware that isn't detected by any AV engines at all so blindly believing your AV scan is rather foolish *especially* if you use P2P which is where I find a lot of undetected samples as that gives a malware author the best bang for his buck in distributing his creation.

If signature based AV is a loosing game then what isn't?  Well the best detection rate I've seen in the lab, and we run thousands of samples through our AV gauntlet every day, is something with behavioral based detections as well such as Kaspersky and to my surprise Norton's 2009 suite.  It's not perfect by far but they do catch a huge number of samples that get past the signature based scans.

There's a reason linux is the base OS of all of our lab systems as well as my own at home.  Linux doesn't run Window's PE format executable which makes it perfectly safe for me to have GB of malware sitting on my drives or on the lab network share to analyze in WinXP virtual machines as we get time.

There is malware for linux but most if not all requires someone to root a box to install it.  I've examined a lot of popped unix and linux boxes and 95% of the time it's due to weak passwords or not updating the box.

The most recent outbreak of malware, a trojan to be specific, for a unix based system was actually on OS X.  The trojan required the user to install it and most did because it was hidden in a pirate copy of Apple's office suite that was being shared on P2P.

The moral of that story is don't install software from untrusted sources.

### Post by diabolik on 2009-02-08
I fully agree with Chayak.

To say this a slightly different way, most virus scanners rely on matching the signatures in their virus pattern files.  All vendors have been "moving towards" rule-based behavior detection - and it sounds like a couple may finally be making some progress after all these years - but by and large, AV will only detect viruses for which a signature has been written.

So, if you want to know how much good that AV scanner will do you in Linux, a good question to ask is, "How many signatures does it have for viruses that attack Linux?"  The answer is very few to none, depending on the vendor.

If you want to make your Linux box more secure, follow the general security advice many have offered:  set strong passwords, don't expose services to the network unless absolutely necessary, and browse carefully.  A software firewall and something like SELinux will also help.  But A/V just ain't gonna do much for you.
