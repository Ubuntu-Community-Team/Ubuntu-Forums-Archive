### Post by lovinglinux on 2009-06-23
> **philcamlin said:**
> why so meny to 51413 ?

These are probably ghost packets, specially if you have a dynamic IP. What usually happens is that when you connect to the Internet your ISP assigns you a new IP. This IP is not yours, you are just using it for that connection. The ISP has a limited number of IPs that usually do not correspond to the number of their costumers. They can do that because not every costumer is connected 24/7. So you are actually getting an IP from a range of addresses that are assigned for different costumers all the time. If you get for example an IP that was previously used by someone running a p2p program, lots of users my still trying to connect to this IP to share files, because they don't know the IP was assigned to you now.

If you disconnect and connect again, these connections attempts will stop. They tend to stop after a while anyway.

You should not be worried about those attempts, because they are probably blocked connections by the firewall right? Even if they come through, you won't be in jeopardy, because you probably don't have any service (program) listening (accepting remote connections) on that port. So the connection will simply be ignored or rejected when it reaches your machine. Nevertheless, you can check if you have a service listening to that port.  Open "Applications >> Accessories >> Terminal", then paste (CTRL+SHIFT+V) the code below in the terminal window and hit enter:

```
netstat -plntu
```

Post the output here so we can check it out.

Even if you have a service listening on that port, it only would be a security risk if the program listening to it has a security flaw. So, in fact the firewall would not give you any additional security, unless you have a service listening to a port and you don't want the external world to be able to connect to it. So you would use the firewall to block all connections on that port and allow only the IPs you want. For example I have a ssh server on a computer on my lan, so I can login remotely and perform maintenance on it. I don't want anyone to be able to log into that machine, so I use the firewall to allow ssh connections only from my machine. This way, any attempt to connect to the ssh server coming from the Internet will be blocked by the firewall. Additionally, I have a router that block connections and do not forward remote connections to the ssh server port on the other computer, so the firewall is there just to reduce my paranoia, because it is not even needed, since nobody would be able to reach it from the Internet anyway.

You should try [gufw](apt:gufw) [apt-get], which is the recommended firewall manager now. BTW, since gufw and Firestarter are just firewall managers, which means they allow you to create rules for iptables (the real firewall), you don't have to run them all the time to be protected. You configure the firewall rules and close Firestarter, because the iptables will be still effective on the background.
