### Post by slater86 on 2008-08-27
I had the same problem.

I fixed it by changing it from /proc to /proc/net.
```

       #/proc           -> $(Device) ;
        /proc/net       -> $(Device) ;

```
files in /proc are dynamically created anyway so it shouldn't matter but /proc/net contains things like realtime values for ip_forwarding which can be useful to crackers.(maybe i'm just paranoid)

I also commented out any other no existent files such as /etc/rc.boot

basically rinse and repeat until you get no errors left.
