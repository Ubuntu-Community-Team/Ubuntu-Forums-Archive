### Post by dualityim on 2010-06-11
Hi everyone,

I'm trying to automatically load some iptables rules by adding the below script in the /etc/network/if-up.d directory:

```
#!/bin/sh

IPTABLES=/sbin/iptables

if [ ! -x $IPTABLES ]; then
    echo "iptables not found"
    exit -1
fi

$IPTABLES -F INPUT

$IPTABLES -A INPUT -i lo -j ACCEPT

$IPTABLES -A INPUT -p tcp --dport ssh -j ACCEPT
$IPTABLES -A INPUT -p tcp --dport www -j ACCEPT
$IPTABLES -A INPUT -p tcp --dport https -j ACCEPT
$IPTABLES -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
$IPTABLES -A INPUT -j DROP
```

It seems to work fine, but for some reason when I list the rules with iptables -L some of the rules are repeated twice. I don't know much about iptables so it might be something obvious that I'm doing wrong. I'd appreciate if anyone has any ideas about what might be causing this. The rules are repeated like this:

```
# Generated by iptables-save v1.4.4 on Fri Jun 11 03:41:08 2010

:INPUT ACCEPT [0:0]
:FORWARD ACCEPT [0:0]

-A INPUT -m state --state RELATED,ESTABLISHED -j ACCEPT
-A INPUT -j DROP
-A INPUT -j DROP

```

Thank you.

### Post by kidders on 2010-06-12
Hi there,

My guess is either (a) your script is running when more than one network interface comes up, or (b) one of your interfaces flaps very rapidly for a short period. Either could cause the script to execute more than once, in such quick succession that one instance of it doesn't get a chance to finish executing by the time the next one starts.

I *think* there should be a few interface-specific environment variables set for these scripts, so you could test for my first suggestion with something like ...```
if [ "$IFACE" != "eth2" ]; then
        exit 0
fi
```

Testing for flapping would be more difficult. You may have to modify your script to check for the presence of a lock file when in runs.

I hope that helps.
