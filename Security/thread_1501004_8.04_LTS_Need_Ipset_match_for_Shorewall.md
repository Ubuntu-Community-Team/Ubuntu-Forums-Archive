### Post by bodhi.zazen on 2010-06-03
ssh -X will forward the graphical interface.

Otherwise, use iptables

```
sudo iptables -A INPUT -s 53.02.5.0/24 -j DROP
```

And on ...

Save your rule set

```
sudo iptables-save > /etc/iptables.save
```

And add this line to /etc/rc.local

```
iptables-restore /etc/iptables.save
```

Then all will be well in the world, drink some tea, and read up on iptables

[http://bodhizazen.net/Tutorials/iptables](http://bodhizazen.net/Tutorials/iptables)

### Post by BoneKracker on 2010-06-03
First of all, sorry if I am misunderstanding your problem.  I've been using ipset and shorewall for a couple years.  Things could be different on Ubuntu, but from my experience here are some basics:

ipset is not in the mainline kernel, you have to install it separately

There should be a package (probably called ipset).

The package, during post-install configuration _used to_ patch the kernel sources and you had to then set kernel configuration options (to enable or disable the various types of ipsets and ipset-oriented iptables matches and targets) and then recompile the kernel.

_Now_, the package just automatically builds kernel modules for all the various ipset types and the related iptables matches and targets, without touching the actual kernel sources or configuration.

I do not recall at what version numbers that transition was made, and it's possible that Ubuntu is using either of these two.  I would expect they are using the latter, which is appropriate to a binary distribution (otherwise users would be forced to build their own kernels).  So, my guess is that you have to install a package named 'ipset' or something similar.

If your system is not set up to autoload modules, then you'll need to [do whatever Ubuntuans do to get modules to load] for those ipset-related modules you need (I'd try it first without doing anything, to see if they'll just automatically get loaded).

Then, of course, you have to actually create one or more ipsets (of the type appropriate to your purpose).

Until you've done these things, you can't start a firewall that contains references to ipsets.

What does this produce:
```
sudo ipset -v
```

Well, at some point you are going to need to upgrade Ubuntu 8.04.

I would suggest you install Ubuntu 10.04 on a test server, either a separate test box or in a VM, and learn how to configure Linux and the servers you use. It will take some time, but I assume you are compensated for your time.

When you are ready, migrate the old 8.04 server data to the new test server. Then use the test server as you upgrade the old server.

In a nut shell, you need :

1. To understand how to sys admin your server.

2. You do not need to upgrade now, but you need an upgrade plan.

For dedicated server I suggest you use Ubuntu LTS (10.04), Debian, or Centos.
