### Post by cdenley on 2009-09-30
It looks like somebody/something was trying to connect to a web server on 192.168.1.64. It was filtered. If it weren't being filtered, then the connections would only succeed if there was a web server running on that IP. Judging by the frequency of the packets, it is probably an automated script which wasn't scripted well enough to check for the presence of a web server before bothering to attempt sending one request after another.

Incoming connections on your router with TCP port 80 appear to be routed (forwarded) to 192.168.1.64 by your router. Did you configure port forwarding for a web server? Is it in a DMZ? Typically, by default, routers will drop or reject incoming connections. Is the ubuntu computer with firestarter configured as the router for your LAN?

I do not recommend using Firestarter. It is no longer maintained, probably creates security vulnerabilities while having the GUI opened, and seeing the filtered packets from random, automated attacks which would've never succeeded anyway is kind of pointless.
