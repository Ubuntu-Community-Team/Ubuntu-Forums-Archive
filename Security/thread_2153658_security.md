Also, just because something comes from a legitimate media outlet does not mean it is the truth. Our government has used the media since it was created to spread all kinds of misinformation to sway public opinion. For better or for worse, you have to take anything you read with a grain of salt and be informed enough to read between the lines and draw your own conclusion. This is why the large majority of the population is so grossly misinformed said:**
> ....it is open source.
This argument has always been a silly one. Just because something has its source code available does not mean it is free of malicious programming. Yes, if you build all of your packages from source, then you will be using the code that was inspected by up to millions of people and it should, in most cases, be safe. However, if you're using pre-compiled sources (As most people do) then the fact that it is open source means nothing. A compiled version could have additional code that is malicious and people wouldn't even notice, especially when the clean code is up for grabs online and people happily review it.

Open source is only going to protect you if you're actually building from source and reviewing the code to some extent. Otherwise, anyone could slip any type of malicious code they want into a project before they compile it and pass it out to the masses. 

Think that's paranoid? Well, not too long ago this did happen with OpenSSH. Malicious compiled versions were wondering around the Internet and one of them ended up in a distributions repository. So, it is very possible for a maintainer that is not closely involved with the developers of a package to be duped, or for the maintainers themselves to add in malicious code into an otherwise great application without even the developer knowing (unless he signs it in advance, but this is not usually the case. In most cases the maintainer will sign the package once it is ready to be pushed into the repository and it is trusted from there).

If the maintainers of Ubuntu really wanted to be apart of the bandwagon, they could do so easily regardless of their open source status. They would just have slightly different versions. One that is pushed to the public in a ready to use state that has malicious code and the clean source for people to review.
