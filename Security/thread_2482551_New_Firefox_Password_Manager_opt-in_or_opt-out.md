Smarter people might be able to keep the completely unimportant credentials/accounts separate and use a minimally-secure password manager. Just be certain that all logins require positive action by you. Don't let them be automatic.  Of course, there are different opinions on this.

### Post by TheFu on 2023-01-04
> **Daveski17 said:**

There are lots of free and commercial password managers.  Google can help to find them.  Lastpass is "cloudy".  If they are putting your passwords in a DB that isn't on your local storage, run away!  

On Linux, I use KeePassXC.  The same encrypted DB can be shared to other platforms and opened by tools on them.  The encrypted DB is a well-known format, shared by many password managers.  In my mind, call it a social contract, I can only change entries on my main desktop.  All other copies of the password DB are read-only.  In my mind.   Nightly (a little after midnight), I push copies of the password DB to 6 different locations using rsync.  This gets the data to many storage devices and different computers, including an android tablet and my phone.  If I do need to make any change, I simply need to get the updated DB back to the main desktop before the nightly "push" script runs.  I've been working this way for a very, very, long time, perhaps 20 yrs.

Podcasters routinely get sponsors from cloudy services.  They have a conflict of interest.

Before I'd put my passwords on someone else's storage, connected to someone else's computer, at the end of someone else's network, inside someone else's building, I'd switch to writing them all down on paper missing the half that only I know.  Then I'd treat that paper more carefully than my debit/credit cards.

On the paper:
```
# Site - Login - random - 
goog - TheFu83342 - 965a1c053e92ed8d4e599ad0c 
UF - TheFu9954 - c1030a0d86fbe870cd493a0b
...
other 200 logins

```
Then I'd have 3 memorized suffixes for the passwords, that I'd add.
Something I have: the paper
Something I know: those memorized suffixes.

Say one of the memorized suffixes is "brusselS", so the full password for the google login would be:
Username: TheFu83342
Password: 965a1c053e92ed8d4e599ad0c$brusselS

Simple. Effective. Easy to implement.

You can make up a complex way to decide on the separator ... or use a space.

It is about random, long (13+ random characters), passwords that are easy to build, but are never written down while still being unique for every different site.
Avoid using SSO options.  Those allow tracking, take away privacy, and mean that 1 good hack provides access to more than one website.
I know this paper method works.  Mom used it for years, before she died.

For websites that don't allow sufficiently long passwords (cough - my bank!), I'll use a random username too.  I don't have a clue what my username for the bank is.  I think it is 25+ characters and I know they were random, since I used a random character generation tool to create it.  Since I use keepassxc and never type that username or password into the bank's website, I can say I honestly don't know what it is. Heck, I don't know the character it starts with either.  If someone tries to get it using "rubber hose" decryption, I can't help them without the kdbx file - and that requires a specific file to be used in addition to the unlock passphrase.  That means they would likely only get $400 in cash (the daily max allowed), which would be replaced by the bank after I file a police report.  If only kdbx allowed multiple authentication types, I'd setup a 2FA device too.  90% of the time, I could use my 2FA device, but not all my devices support it, so I need an alternative unlock method.

I've been consistent about not trusting cloudy services for a long, long, time.
> “I suppose many people will continue moving towards careless computing, because there’s a sucker born every minute. 
...
However, as long as enough of us continue keeping our data under our own control, we can still do so. And we had better do so, or the option may disappear.”
- RMS

Look into the Fediverse, if you'd like to know more.  This is a movement for people to control their own data.
