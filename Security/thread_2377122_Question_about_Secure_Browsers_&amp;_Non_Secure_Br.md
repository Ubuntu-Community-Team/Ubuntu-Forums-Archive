So any programmer is human which means mistakes can be made, you can have an awesome web browser dev who knows little about some aspect of hacking hardware, like the Intel Management Engine, or how to secure some passwords in memory.

If you treat everything you use as having a zero day, then what can you do to minimise the effects of these exploits? Backups. Get used to wiping your device's and reinstalling your data from backups assuming your backups or hardware is not compromised.

From a tracking perspective, using lots of different browsers and using configurations or addons/extensions in the browser to help reduce tracking can help, but even doing something as simple as changing the monitor resolution, enabling and disabling addons in a browser, just to mix things up can help. 

Using lots of VPN's instead of one, can help, the more you change your online fingerprint/footprint/appearance and keep changing, the harder it is to be identified. So this can mean using different versions of Linux live distro's, different versions of OS's, even different hardware.
