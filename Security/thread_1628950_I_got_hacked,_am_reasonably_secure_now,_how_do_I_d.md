### Post by magnatron on 2010-11-23
I agree it's very difficult to hack into any system that has had it's security tightened. If you caught the hacker in the act with say rootkit hunter or have tcpdump or wireshark logs of the hack actually taking place thats different as then you have something concrete to go on.  Although it has to be said a lot of people download and install services like SSH and then promptly neglect to tighten the security by disabling root access.  If you have services enabled what are they and on which port, use nmap on your own machine and type: nmap -v -A localhost and look at which ports you have open, if there is a port open with a service you don't recognize that can sometimes narrow down if a hack has actually taken place.  Also try the following: sudo apt-get install rkhunter then check your own security by typing sudo rkhunter -c  hope that helps  Also be advised that rootkit hunter can sometimes report false positives or report suspicious activity when a system package upgrades or changes as the salt thats generated by the command rkhunter --propupd will have changed during that period Another thing to check is the state of your Bourne shell history file by typing: history at the command line, some attackers are so stupid they overlook it and it records everything, other attackers are so brash they send it to /dev/null and forget to put it back!
