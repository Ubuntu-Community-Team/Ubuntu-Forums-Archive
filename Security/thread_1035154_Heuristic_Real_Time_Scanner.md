### Post by khedron on 2009-01-29
I'm against this idea. Virus scanners are weak in that they only search for known viruses. Heuristic scanners try to solve this by being more general in what they look for, but they can still easily be fooled.

For example, consider a program that, in the course of its execution, deletes a user's file. Is this malicious? Maybe the user requested it. Maybe the application asked and the user said yes. Or maybe the application asked a completely different question but deleted the file anyway.

These different scenarios are very hard to tell between for a program. In the last two cases, the program would have to analyse the English text of the message to work out whether the program really was asking for permission to delete a file.

My point is really that **whether a program is malicious really means whether it behaves as the user expects it should when given a particular input.** This is impossible for a simple heuristic scanner to detect.

For example, many virus scanners detect open source games as viruses because they include a server program in their install. Many bot programs also use such open-source servers and there is no practical way of telling the difference, so virus scanners also blacklist these harmless games.
