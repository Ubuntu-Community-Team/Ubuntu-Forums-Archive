### Post by noxvirtox on 2018-03-10
Imagine the following scenario. You have a computer that runs on Windows. You suspect that you may have caught something nasty. You run all the standard scans and they all return nothing, but it still feels like something was off. Because you use this computer for things like accessing your back account, you decide to not take any chances and just replace Windows with Ubuntu, since you've already been using it on another device and you're very happy with it. So you just install Ubuntu in place of Windows on this computer as well and the problem seems solved finally. My question is, is it technically possible that, if the Windows system actually was infected with something and it really was malicious enough, it could still pose some threat even if the computer in question only runs on Ubuntu now? I know that in theory installing a new OS on top of the old one *more or less* wipes out the old one and that Windows threats technically can't work on Linux anyway, but you know all those stories you hear about dangerous new types of malware that can literally install themselves in the BIOS and run independently of the OS, or infect the very beginning of the hard drive which remains untouched during a format or reinstall, and how it can still infect external data storages through the USB port or other things like that. I know I may be a little paranoid, but like I said, I'm talking about a computer that is used for serious things so I thought it would be better to make sure.

### Post by mikewhatever on 2018-03-10
Generally, OSs like Windows or Ubuntu run on some hardware. Reversing that relationship doesn't make any sense. It is similar to saying "cars drive people", which is nonsense.
Now, to the malware part, the question is kind of vague, and I am not sure what's the point, but yes, what you describe is not impossible. Theoretically, any OS can be infected, and yes, some malware can infect HDD's firmware or the BIOS, and thus survive full OS reinstallations.
