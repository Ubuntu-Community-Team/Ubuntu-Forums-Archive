### Post by TheFu on 2020-08-13
No, not typically. Just have daily, versioned, backups and don't allow physical access to anyone untrusted. Every different user should have their own account with their own password. This way, they can only harm themselves. The system won't be harmed by a normal user being hacked.

There is a sticky thread in this subforum which explains normal security for Linux systems.

There are 2 exceptions to when I'd load AV on a linux system:
[LIST=1]
[*]Contractually mandated, usually by corporate insurance
[*]If it is the only Linux desktop/server in a LAN full of dangerous Windows systems. Linux won't care, but it can be a good neighbor. Those Windows users may need help catching Windows viruses in scans so they are quarantined on the Linux system.
[/LIST]

In general, A/V on Linux searches for Windows viruses.  Linux attacks are different and designed differently.
