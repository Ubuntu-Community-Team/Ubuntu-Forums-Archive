### Post by trinitydan on 2010-12-25
> **Pithikos said:**
>  
There are a bunch of books that are worthless or just too old so a recommendation is never bad..
Exactly why the information you are seeking is probably not best presented in a book.  If you want to learn penetration testing, I would suggest you get a penetration testing OS, like Backtrack, for example, or any of its variations.  Then set about learning how to run the hundreds of penetration testing applications that come with it which are basically all run from the command line.  the instructions you need are right there in the man files.  There is more than likely a ceiling for how far you can go with this and remain "ethical".  It's good to know how easy it is for someone to get your passwords or watch your internet activity, but if you aren't really out to rip anyone off it is mostly just a novelty and you can show off by "hacking" your friends MacBook or other such things...

I think if you really want to make a difference you would do better to learn coding.  Most of what people refer to as hacking is just running scripts.  Hacking IMO should involve a fluent knowledge of the operating system.

### Post by Jive Turkey on 2010-12-27
I'm certainly no expert on this stuff but I like to keep up with the news in this area.  One method for experimentation I can recommend is to create virtual machines with the target OS/application(s).  In virtualbox it is really fast and simple to save a snapshot of the OS state before trying out different attacks that could damage it.

You asked specifically about books, I have read and learned quite a bit from the following ones:

[Steal this Computer Book]("http://rcm.amazon.com/e/cm?t=kerfuffles-20&o=1&p=8&l=as1&asins=1593271050&fc1=000000&IS2=1&lt1=_blank&m=amazon&lc1=0000FF&bc1=000000&bg1=FFFFFF&f=ifr")
