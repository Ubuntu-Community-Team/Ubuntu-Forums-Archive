### Post by japzone on 2012-01-30
This is very odd, it doesn't seem to be a Problem with your Mac's setup or VirtualBox's Setup. Make sure you aren't using Scaling mode in the VM(View-->Scaling Mode) and try only using Windowed Mode or Fullscreen(Any other mode I've noticed decresed performance) as for Remote Folder Sharing you can try just using SMB(Windows File Share) Directions for Setting up Ubuntu for SMB are [**HERE**]("http://ubuntuforums.org/showpost.php?p=11631886&postcount=2") and Directions for Setting up OSX for SMB are [**HERE**]("http://www.makeuseof.com/tag/how-to-easily-share-files-across-mac-windows-computers/"), just make sure the workgroup name is set to "workgroup" on both Ubuntu and OSX("workgroup" is usually set by defualt)

If you continue to have problems it may be wise to back up any important VMs(Use the "File-->Export Appliance" option) and then Reboot and try reinstalling VirtualBox from scratch. Then try Installing Ubuntu 11.04 in a VM and use the "Ubuntu Classic(No Effects)" desktop UI(Found on the Bottom bar after selecting a User). Hopefully this'll clear up any problems with VirtualBox and fix the Problem with Ubuntu VMs. Also when setting up the VM make sure you Select "Linux--Ubuntu" in the Drop-Downs and then keep all the Default Settings in the Wizard except use a Fixed-Size VDI. Then before booting the VM make sure in the settings under "System" you Turn on all Hardware Virtualization features and under "Display" you turn on all Hardware 2D and 3D settings.
