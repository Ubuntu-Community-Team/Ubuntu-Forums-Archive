### Post by memilanuk on 2011-11-12
I've been messing with some other distros in Virtualbox lately.  Most of the more 'mainstream' ones like Ubuntu, Debian, Fedora, etc. are reasonably easy to get up and running.  After I get them running, then you have to go in and install the Guest Additions if you want things like shared folders, etc.  If you want to have the guest screen resolution be such that you don't have to scroll around the window to find the task bar, you generally have to download some things like build-essentials, kernel-headers, etc. and build the kernel modules.  

Nothing too difficult, really... but when I fire up an OpenSuSE DVD (virtually) to install it in VirtualBox... it *automatically* goes full-screen - which is a real treat compared to squinting at the little 800x600 VM screens I normally see on my laptop when installing guest OSes.  And once it finishes the install, no more of this 1024x768 or whatever and having to scroll around to find the Start Menu button or the app bar or what have you at the top or bottom or side of the screen... it automagically resizes to full screen - perfectly - on its own.

What would it take to get Ubuntu to move that direction - to have the VirtualBox guest additions already available during installation?  I can understand if it wouldn't be available on the LiveCD version - there is only so much stuff you can cram in a 5lb bag, so to speak ;)  But on an install DVD... this would be sweet (in my opinion)!

What do you think?
