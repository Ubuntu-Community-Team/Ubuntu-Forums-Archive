### Post by tylerspaska on 2008-01-04
i think any c2d processor (or equivelant) would be good enough, and probably 1 gig of ram. you have to split your ram between two machines and 512 is a pretty good ram size for xp and ubuntu. i run c2d 1.8GHz with 2 gigs of ram and i don't notice any slowing when i open a virtual machine.

i use VMware. it's not a bad idea to always have xp running virtually, but VMware saves the state of the machine when you close it. so 'boot' time is only the time it takes to start the program. i use it and love it. try it on windows and see if you like it or not. i'm pretty sure it's not open source, but it is free...

edit: you have to split your ram, but you can choose how much goes to each environment.
