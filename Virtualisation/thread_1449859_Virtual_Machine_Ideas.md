### Post by Micronic on 2010-04-08
I was searching the internet for ways to play games on virtual machines, but had no luck.  I have a 16 core AMD opteron server running windows server 2008 x32 and want to run Windows XP for gaming and Ubuntu 9.10 with compiz fusion in Virtual Machines.  The only problem is, you cannot dedicate an entire graphics card to a virtual machine, even if you have 3 seperate graphics cards in your rig.  

Ideas:
   1.) Make the ability to dedicate an entire graphics card to a virtual machine.
   2.) Ability to boot into Virtual Machine Hard Drives from GRUB2, (making a listing in grub2 when the virtual machine is created) as a separate OS such as a multi-boot device.  Letting you use all your hardware just as it would in a multi-boot system.  This ability will let you run the OS as separate and dedicated, but with the ability to open it up as a Virtual Machine in your other Operating System to make changes to it or install new programs instead of restarting your computer.
   3.) The ability to "Drag and Drop" files from your Desktop to the Virtual Machine, allowing you to move data from one OS to the other seamlessly.  
   4.) (this can go for anything, not just Virtual Machines)  Make a Remote Desktop/Server Service, with the ability to have multiple users login to the same desktop with separate cursors and keyboards. (using MPX)  With each user using a separate Virtual Desktop with the ability to change to the other users Virtual Desktop.  This can be used to help the other person (or coworker) if they are having trouble, or just playing around with a friend on paint and having fun.  This ability could also take advantage of microphones and web cams, letting users communicate via their remote connection.

   I would love to see these programs in the near future!  However if you do take my ideas to heart, the only thing I ask is you publish them freely, and the credit goes to those who are involved and is distributed accordingly.
