### Post by rodda on 2008-06-17
I would like to propose the idea that Ubuntu setup a forum specifically for KVM virtualization as it was announced by Soren Hansen that KVM is the preferred virtualization platform for Ubuntu/Cononical.

[https://wiki.ubuntu.com/UbuntuWeeklyNewsletter/Issue76#head-ae07153a350b0c9e5da6b4c84d8fca9fa1d7ab7a](https://wiki.ubuntu.com/UbuntuWeeklyNewsletter/Issue76#head-ae07153a350b0c9e5da6b4c84d8fca9fa1d7ab7a)

(Additionally, since KVM has been chosen as the standard for Ubuntu server virtualization, it should also be included in the list of server 'roles' during installation)

Nothing against the other virtualization platforms, but I'm planning a future deployment of Ubuntu/KVM in a production environment with a Conanical support contract and it would be nice to concentrate on the officially supported package without postings regarding extraneous virtualization platforms (most of which I've used in the past).

Again, this is just a suggestion by an end-user but I would appreciate your feedback...

### Post by seetho on 2008-06-19
I agree with rodda here.  I use VirtualBox for OS testing because it is easy to setup and use - it has a reasonable clean and user friendly GUI.  However for production environment servers (without GUIs) I much prefer KVM.  

The kvm setup document on the Ubuntu site is great to get you started but is not nearly enough for anything more.

A separate forum would certainly help kvm users to iron out a lot af issues not properly documented in a single place.
