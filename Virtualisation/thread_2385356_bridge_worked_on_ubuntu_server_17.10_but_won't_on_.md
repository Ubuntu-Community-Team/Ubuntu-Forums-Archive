### Post by KillerKelvUK on 2018-02-20
If your problem is because you cannot "ping google" then this is DNS related because you have no DNS configuration in your interfaces file.

The below amended file uses google's own DNS servers for resolution however you are free to choose whichever you like just replace the "8.8.8.8" and "8.8.4.4" with the IP's for your preferred service.

```

auto enp0s25
iface enp0s25 inet manual




auto br0
iface br0 inet static


  address 192.168.2.67
  netmask 255.255.255.0
  broadcast 192.168.2.255
  gateway 192.168.2.1
  dns-nameservers 8.8.8.8 8.8.4.4
  bridge_ports enp0s25
  bridge_stp off
  bridge_fd 0
  bridge_maxwait 0

```

A further validation that this is DNS related would be to inspect the contents of /etc/resolv.conf, this file should incorporate a line per DNS server specified/known.  Below is an example...

```

$ cat /etc/resolv.conf 
# Dynamic resolv.conf(5) file for glibc resolver(3) generated by resolvconf(8)
#     DO NOT EDIT THIS FILE BY HAND -- YOUR CHANGES WILL BE OVERWRITTEN
nameserver 8.8.8.8
nameserver 8.8.4.4

```
