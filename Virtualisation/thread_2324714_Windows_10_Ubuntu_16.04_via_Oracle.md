### Post by QIII on 2016-05-15
You will have to enter your BIOS/EFI setup when you boot the machine.  How that is done depends on the manufacturer.  You may want to consult your machine's documentation or see if you can find it on line.

Most BIOS setups list something labeled "Virtualization" or "VT-x" (for Intel.  It's AMD-v for AMD) which you can enable or disable.  Where that is in your settings menu is again something you may have to find by referring to your machine's documentation.

Try chkdsk first.

### Post by MAFoElffen on 2016-05-16
That has an Intel i7 CPU.

Explanation to OP: You have an vt-x (IMMOU, virtual hardware support, etc.) capable CPU, but virtual support is not enabled by default in your host system. To run a 64-bit OS in Oracle's Virtualbox, you need to turn this on in your BIOS.

On system boot of your host machine (not the virtual), hit esc > then find this option to set from disabled to enabled:
[IMG]http://psg.i.lithium.com/t5/image/serverpage/image-id/68473iA6F59016F87CFB73/image-size/original?v=v2&px=-1[/IMG]
(F9 on HP's is the advanced boot menu.) Notice that HP is not under the main or advanced setting, but is located in the security section ... this is a reoccuring question about HP's up on SPICE (the HP community forums).

I am assuming you are using Oracle VirtualBox. This will install with this option tuned off, but with that option turned off, willl limit you to installing 32 bit OS'es. Once you enable that option, you will be able to install 64 bit OS'es. If you go into vbox VM Manager > Prefrences ... it will also tell you there if VirtualBox sees that capability enabled or not. You cannot change it from there, but is a quick way to check that.

Note: If you have a Windows Host and have Hyper-V turned on as a feature ... and consurrently also have VirtualBox installed, nothing other than Hyper-V will see that PC as vt-x enabled ... meaning VirtualBox and/or VMware Player will never see that PC as vt-x enabled, even though it is really, truly enabled in the BIOS, IF Hyper-V is present. Hyper-V takes over control of all hypervisor functions and thinks it should be the only meaningful user of that resource... So if you want to use VirtualBox or VmWare Player as your Hyperviosr, ensure that Hyper-V is turned off as a Windows feature.
