### Post by &amp;KyT$0P# on 2017-05-22
If you are using NetworkManager inside the VM:

1) Check your DNS settings in "Connection Information" and make sure it's what you expect.

2) Compare the output of these commands to what you get on your VM...
```
$ ls -la /etc/resolv.conf
lrwxrwxrwx 1 root root 29 Mar 26 16:17 /etc/resolv.conf -> ../run/resolvconf/resolv.conf
$ cat /run/resolvconf/resolv.conf
# Dynamic resolv.conf(5) file for glibc resolver(3) generated by resolvconf(8)
#     DO NOT EDIT THIS FILE BY HAND -- YOUR CHANGES WILL BE OVERWRITTEN
nameserver 127.0.1.1

```
... and if the first one doesn't match, run
```
sudo dpkg-reconfigure resolvconf
```

### Post by SeijiSensei on 2017-05-22
Edit the contents of /etc/resolvconf/resolv.conf.d/head to include the nameservers you want.  That file is used to build resolv.conf and its contents will come ahead of any other servers that the resolvconf process might identify.
