### Post by dcwangmit01 on 2010-11-22
Hi,

I originally had the same error, while trying to migrate a machine from 10.04 to 10.10.  I actually had several levels of problems.

Here's how I fixed it:
* Enable virtualization in the bios to allow a /dev/kvm device to be created
* Disable AppArmor
       sudo /etc/init.d/apparmor kill
        sudo update-rc.d -f apparmor remove
  * virsh edit <machine> to set the disk format to "qcow2", if you are using the qcow2 image.
      The new 10.10 kvm/qemu does not autodetect the disk image format.
* if it is a windows machine, disable virtio

-dave
