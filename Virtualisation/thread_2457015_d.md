### Post by ajgreeny on 2021-01-24
The 5.8 kernel series seems to be causing a fair number of problems so I suggest that you boot to the older 5.4 series kernel from grub (I presume you still have that version) and then remove the package ***linux-generic-hwe-20.40*** and the problem 5.8 kernel, ie ***linux-image-generic-hwe-20.40*** along with any headers of the same version.

The simplest way to remove such problem kernel versions for you may be with synaptic package manager, where you can use the search term ***hwe*** and remove what is installed, though if you're familiar with command-line you can also use that to find the actual kernels installed with [code]dpkg -l *linux-image* | grep ii[.code] and remove all that are listed.

No further 5.8 kernels will by installed following this.
