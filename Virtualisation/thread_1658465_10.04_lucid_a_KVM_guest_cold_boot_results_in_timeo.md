### Post by egerlach on 2011-01-03
[B]Solved! Its the fault of the kernel update 2.6.32-27  (or maybe a fault in KVM in 2.6.32-bpo.5-vserver-amd64 kernel of lenny that can't cope with latest optimizations of Mike Shuttleworth's crew)!

[/B]I installed ubuntu once again. I installed NX. But no Update. All works, reboot and cold boot (halt & "run"). 

Then I installed online update like proposed by ubuntu - except the new kernel and except grub update. After all works fine: reboot and cold boot. 

Then I installed the kernel update only, not grub update. The result is that ubuntu is NOT booting as KVM machine. The "console" rests black, ubuntu hangs, not freezed.  After about 5 to 60 minutes the machine is booting .... same as twice before. 

So: don't update kernel 2.6.32-24 to -27  if you want to run lucid as KVM machine. 
If it's too late, then start old kernel -24  by installing startupmanager. Select old kernel to start. 

Your  2.6.32-27  won't start at all? -> reboot your host, "run"  ubuntu. In about 10 min ubuntu will start. But only once. You have to reboot your host before a second start of ubuntu works within 10 min.

Eckard
