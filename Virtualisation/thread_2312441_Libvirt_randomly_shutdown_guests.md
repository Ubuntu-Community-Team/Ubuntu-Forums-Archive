That log message is generated by the upstart script *libvirt-bin.conf*, when the libvirtd daemon is stopped in the *pre-stop script* stanza.

```

media-server1:/etc/init:4 % grep "attempting clean shutdown of" *
libvirt-bin.conf:                   log_msg "libvirt-bin: attempting clean shutdown of $domain at $(date)"
media-server1:/etc/init:4 
```

Was the libvirtd daemon running when you found the guests had been shutdown ?

### Post by MAFoElffen on 2016-02-04
That would have been normal *if* there was a powerdown or reset on the host ... But you said there was not? Libvirt has an upstart trigger on shutdown or reset to try to cleanly shutdown any running guests. This fixed a previous bug where it didn't do that (did not allow enough time for that.)

The syslog has nothing on the host that indicated any shutdown or reset condition that could have triggered that process? And not updates going on at that time? (such as with KVM or libvirt?) I would be looking through your syslogs, just previous to that guset shutdown at "[COLOR=#000000]*Wed Feb 3 11:24:14 EST 2016*[/COLOR]"... The event above starts with a message is this format:
```

($DateTimeStamp) kvm libvirt: libvirt-bin: starting wait_on_vms at ($ShortDate) 

```
grep for that pattern and show 25 lines previous...
```

dmesg syslog | grep -B 25 'starting wait'

``` 
EDIT--
Just in case they might have recently changed that message format, if the above shows no output, then search (the same command) on libvirt...
```

dmesg syslog | grep -B 25 libvirt

```

### Post by matt_symes on 2016-02-04
Hi

As well as MAFoElffen's suggestions above, you may want to look at

```
last reboot
```

Something caused that script to be triggered and call the pre-stop stanza.

How have you verified that a reboot didn't happen ?

Do you have automatic updates configured for your server ? If so then an update to the libvirt package may explain why the guests were shutdown.

Check your dpkg logs. I recently had an update...

```
media-server1:/home/matthew:5 % grep " installed libvirt" /var/log/dpkg.log{,.1}
/var/log/dpkg.log.1:2016-01-17 15:11:34 status installed libvirt0:amd64 1.2.2-0ubuntu13.1.16
/var/log/dpkg.log.1:2016-01-17 15:12:19 status installed libvirt-bin:amd64 1.2.2-0ubuntu13.1.16
media-server1:/home/matthew:5 % 
```

Look to see if you have an update to libvirt-bin.

```

media-server1:/home/matthew:1 % apt-file search libvirt-bin.conf
libvirt-bin: /etc/init/libvirt-bin.conf
media-server1:/home/matthew:1 %
```
