### Post by reiki on 2010-08-31
This was mainly a personal challenge for me. 
I have a Dell Zino with an AMD processor (X2 6850e) and an ATI HD4330 in an MXM slot (like a laptop). I have no idea what motherboard they use in this little box. (The Zino is the size of a Mac Mini)

Using a Hazard OSX iso I successfully got Snow Leopard running as guest in a VB VM on an Ubuntu 10.04 host. No sound, but I haven't even BEGUN to tweak this. It does work. Boots reliably if a bit slowly. Runs OK. Has to be brute forced into shutting down. 

But it starts back up fine. Now all I gotta do is figure out what to DO with it. :)

### Post by reiki on 2010-09-02
Just FYI, I've been tweaking my install procedure and last night I was successful in doing the install from a retail Snow Leopard DVD. Not one of the "customized" images. I did use a modified boot loader that was customized for AMD processors. It's called EmpireEFI for AMD or something close to that. (I'd have to look it up again myself)

In terms of usefulness, right now I'm going to say this has extremely limited practical use. I installed iLife last night onto teh Snow Leopard VM and while it installed fine, many programs would not run because the VM was not replicating a graphics card with the capabilities required by the programs. So if I can't run iDVD or iMovie then.... what's the practical use?

A practical use of this experiment would be to get a look at how the basic OS X installation works. How to navigate around and learn about OS X. What does it call things? How do I do this? So as a learning tool it might be useful, but beyond that I'm just not sure.

I have a Windows 7 VM because I have a need for that OS as I have medical equipment that requires it and there is no linux equivalent for them. I don't, however, have a practical need for OS X so except for the novelty of HAVING it, I doubt I'll use it much if at all.
