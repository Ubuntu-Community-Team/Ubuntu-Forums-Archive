### Post by MAFoElffen on 2023-10-23
.ova format is a tar compressed file with the VM machine definition and disk image inside of it. You can use VBoxManage clonehd to create the raw, but since I don't have VBox installed any longer... I just to
```

cd /path/of/file
tar -xvf ./filename.ova
## Sample Output:
#x filename.ovf
#x filename.vmdk
#x filename.mf
file ./filename.vmdk
#filename.vmdk: VMware4 disk image
qemu-img convert -O qcow2 filename.vmdk filename.qcow2

```
With that last command, you could also change the type to raw, and the extension to img...

KVM usually works best with raw and qcow2 types of image files. The docs say that it will work with vmdk images, but why not make it easy on it and read something more native to it.

The other image formats the KVM can work with are:
```

bochs - Bochs disk image format.
cloop - Linux Compressed Loop image, useful only to reuse directly compressed CD-ROM images present for example in the Knoppix CD-ROMs.
cow - User Mode Linux Copy On Write image format. The cow format is included only for compatibility with previous versions.
dmg - Mac disk image format.
nbd - Network block device.
parallels - Parallels virtualization disk image format.
qcow - Old QEMU image format. Only included for compatibility with older versions.
qed - Old QEMU image format. Only included for compatibility with older versions.
vdi - Oracle VM VirtualBox hard disk image format.
vhdx - Microsoft Hyper-V virtual hard disk-X disk image format.
vmdk - VMware 3 and 4 compatible image format.
vvfat - Virtual VFAT disk image format. 

```
But I just take those formats and convert them with qemu-img convert to raw or qcow2.

As I tried to explain, to work with that .ova file with KVM... you are going to have to extract the image file from within it, out to "somewhere". Convert the .vmdk image file to either .img or qcow2. type format. KVM cannot work with the .ova compressed image directly.

To use KVM as a a user, you need to be added to the lib-virt and kvm groups. they are owned by lib-virt-qemu:kvm, so as a member of the kvm group, you can work with them. I set the permissions of them (wherever I place them) as 774, which shows up in ls -l as -rwxrwxr-- 1 libvirt-qemu kvm <blah, blah, blah>...
