### Post by dawynn on 2008-08-28
YMMV.  My setup: Ps/2 Mouse and Keyboard.  Note: Other users have seen issues with USB mice / keyboards.  That may or may not be related.

I also upgraded from HH to II.  Having been burned in the past, I no longer just do a blind dist-upgrade.  Instead, I mark blocks of modules to upgrade, and verify each round of updates / deletes / etc.

When it came time to do xorg, I saw that xorg again wanted to place code for just about every input and output device on my system.  I blocked it from doing so, and ended up with the situation you describe: no usable keyboard or mouse.  But the system *would* allow alt-ctrl-f2 (and such like), so the system was recognizing the keyboard, but X wasn't.

Checked /var/log/Xorg.0.log.  There were a number of complaints regarding "evdev".  An aptitude search found xserver-xorg-input-evdev was not installed.  Strangely, the description for this package indicates it lets *multiple* keyboard / mice be recognized as separate devices.  That wouldn't *seem* to apply to my system since I only have one keyboard, and one mouse.  But, installing this one package cured this particular problem for me.

As Nullack said -- check your /var/log/Xorg.0.log.  If you're not familiar with this file, each message is tagged with one of the following: (II) -- Informational (ignore when diagnosing issues), (WW) -- Warning (possibly a cause of your problem), and (EE) -- Error (definitiely review these).

### Post by ncubede on 2008-09-11
The solution for my system:

I kept the system up to date using incremental upgrades.  One of the actions of the gdm postinstall script is to move GDM startup from S21 to S30.  It did not do this for me, so gdm started after hal.  As X needs hal for evdev_drv to load, neither keyboard nor mouse worked until I restarted gdm.  

I opened a bug against the upgrade, but this issue can be recognized by looking in the X log for an (EE) error where hal/config fails.  /etc/rc2.d/S30gdm should exist in intrepid.
