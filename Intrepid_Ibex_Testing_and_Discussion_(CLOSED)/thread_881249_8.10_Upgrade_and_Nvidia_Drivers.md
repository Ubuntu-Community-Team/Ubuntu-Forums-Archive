### Post by AnarchyCow on 2008-08-05
I installed my nVidia drivers on 8.04 using the following method:

CTRL+ALT+F1
```
sudo apt-get remove --purge nvidia*
sudo apt-get install linux-headers-$(uname -r) libc-dev build-essential
sudo /etc/init.d/gdm stop
sudo sh NVIDIA-Linux-x86-###.##.##-pkg#.run
sudo /etc/init.d/gdm start
```

But, after upgrading to 8.10 using "sudo update-manager -d" this installation method will not work (Atleast if I did it when I installed 8.04). I'm trying as we speak with not installing the drivers on 8.04 and just trying to install them on 8.10, because the last time I tried, it gave me a error saying that it was already installed, and it said it sucessfully removed them, then installed them again, but, following that, when I rebooted, I didn't have the display drivers setup. And when I pressed CTRL+ALT+F1 again, I was met with a intimidating screen with a lot of "periods" filling the screen with columns of different background color (Don't remember the two colors, think it was a blue and something else) shifting between two every other column of periods. (like {.}.{.}.{.}.{.})
BTW, When I did this, I did type the "sudo apt-get remove --purge nvidia*" before installing the drivers using the .run file. But, now that I start to think about it, I'm not 100% sure I typed it right...

When I tried to upgrade before, It wouldn't boot into the new kernel (no option for the new Kernel in Grub), so I setup for it to boot into the new kernel through grub's "menu.lst" by simply changing the Kernel, but I was met with the whole period issue like I explained above on the first boot when trying to CTRL+ALT+F1.

I really want to be on the newest platform available, and I am willing to do some troubleshooting to do so.

One other thing I was wondering..
