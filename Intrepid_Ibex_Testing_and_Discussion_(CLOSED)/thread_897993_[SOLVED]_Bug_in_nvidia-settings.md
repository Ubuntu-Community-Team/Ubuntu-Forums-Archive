### Post by gmxgeek on 2008-08-22
I would like to apologize in advance if this is in the wrong place, but i looked for a bugs section and couldn't find one, and thought this might be more helpful to those with the same error.

Having just upgraded from hardy to the ibex alpha, I had to reset all of my video settings. No big deal, just reset the xorg.conf and run nvidia-settings on it.

So, here was my process
```
sudo dpkg-reconfigure -phigh xserver-xorg
sudo nvidia-xconfig
sudo reboot
```

after rebooting, things came up fine, good resolution, everything.
now, i have two monitors, so, i tried to fire up nvidia-settings to fix the second one. i configured everything, and saved the config file to my desktop, backed up the old config, and copied over the new one. A quick reboot and....

Everything is broken.

So, after much debugging and comparison of configs, i found out that there was one line that was causing all my problems.

```
Section "Files"
        Rgbpath         "/usr/X11R6/lib/X11/rgb"
EndSection
```

It turns out that that file does not exist. So, I deleted that line, rebooted, and everything works fine. I don't know if this happens in hardy too, or if it is just an ibex issue, but I thought i would post either way.
