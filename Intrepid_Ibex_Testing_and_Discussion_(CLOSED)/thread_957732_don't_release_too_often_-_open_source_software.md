### Post by edd07 on 2008-10-24
Actually, releasing early and often is part of the essence of open source. How else would people get interested in helping to develop something they haven't seen?

And how would bugs get caught if the product isn't released to be tested? Internal testing can only help so much, the developers can't test every single configuration.

I understand using an unstable system can be very frustrating, but you can wait a couple of weeks after the release. Hardy, for example, was a bit buggy for me, but since the first couple of updates it's been rock-solid.
