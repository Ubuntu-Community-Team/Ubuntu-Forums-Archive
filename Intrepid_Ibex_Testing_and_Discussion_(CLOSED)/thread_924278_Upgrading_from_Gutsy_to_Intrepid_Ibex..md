### Post by ehbursey on 2008-10-21
> **nyroshan said:**
> so does would anyone want to return to the original question and let me know how id be able to use my home partition in another system install such as by putting it on a different partition?
Your hard disk drive can be broken up into individual sections.  These sections are called partitions.  Although they are on the same physical hard disk drive, you can use these partitions as if they were separate disk drives.  This can be very helpful if / when you need to re-install your operating system.  For instance, if you have your operating system on partition 1 and your user data on partition 2, you can format partition 1 and reinstall the operating system without affecting the data on partition 2.  

Repartitioning your hard drive isn't difficult, but it is probably more complicated than could be covered here.  Consider this an overview.  The general idea is to make room on the hard drive and then section that room into individual partitions.  You can make room by 1) deleting existing partitions (note, you'll lose any data on those partitions!) or 2) changing the size of an existing partition.

You can use a command-line tool like "fdisk" to repartition your hard drive.  As far as I know, fdisk does not allow partition resizing, so making room on the hard drive with fdisk would consist of deleting existing partitions.  If you want to know more about fdisk, open a shell and type "man fdisk".  If you google fdisk, make sure you're reading about the Linux fdisk and not the Windows/DOS fdisk.

You can also use something like "gparted."  This tool has a graphical interface and I would guess would be somewhat easier for a beginner.  Google "gparted".

Whenever you repartition the hard drive, you run the risk of losing data.  Before doing anything else, you should back up all of your data.

I don't think you can resize a partition you're currently using, so if your hard drive has just one partition (containing your OS and your data), you'll have to boot from some other disk.  I think that the Ubuntu Installation Disk will work (perhaps the experts would stop arguing and offer a suggestion about this).

Once you've resized and repartitioned the hard drive, you'll need to format the partition.  I think this can be done from within gparted.  You'll probably want to format the partition using the ext3 filesystem.  If you dual-boot, you might consider partitioning it using the FAT32.  That would allow Windows to use the partition too because Windows can read FAT32, but not ext3.  If you dual boot, it can be nice to have all of your data in one place that can be accessed by both operating systems.

Once the partition is formatted, you have to tell the system to mount the new partition.  This is easily done by setting the "Mountpoint" in the gparted program.  Typically, you might set the mountpoint of your data partition to "/home"  If you have a pre-existing /home directory, you might want to rename it to something else (/home_old would work) and then mount the new /home partition.  Then you can copy your data from the /home_old directory to the /home directory.

Again, this is just an overview, but hopefully now you see what's involved.  Use google, read a lot, and back up your data before you start. 

Good luck

### Post by scaine on 2008-10-22
> **DrMega said:**
> What about those of us for whome Hardy was a disaster?

I just don't see how you can advise people not to upgrade based on fear that some things might break.  It sounds like you (and one other on this post) have bad experiences, but that shouldn't stop people trying out Hardy (or Intrepid soon) - things could well (and are likely to) be just fine.

The thing about these forums is that they're largely here to troubleshoot problems, so 95% of the posts tend to be people griping that stuff doesn't work.

When it does work, as I'm sure it does for masses of people, do you think people actually take time to pop onto these forums and start a thread saying "Everything's great!".  It just doesn't happen.  Or at least, not very often.  And it would get old pretty quickly if it did.

My advise for upgrades is always the same - don't bother moving your home onto a new partition just so that you can use it after the upgrade.  All you'll do is carry over a bunch of legacy and possibly conflicting settings from an old install.  I speak from experience, although it was admittedly a few years ago.

DO MAKE A BACKUP of your home, however.  Simple as that.  Just launch nauitlus, show hidden files and copy your home onto a big enough USB drive to take it all, then get installing.

Sure, it takes a while to build everything back the way you like, but if you're going to stick with a release for 12 months, it's not that big a deal.

Oh, and despite saving your home directory, you might also benefit from popping into Firefox and doing a manual export of your bookmarks.  It wouldn't hurt to pop into Thunderbird and do a manual export of your contacts too.  Firefox has a brilliant plugin called FEBE which makes restoring your bookmarks, cookies and such very easy, but I'm not sure it will help you here because you'll be moving from Firefox 2 to Firefox 3.  That's an unknown for me.

Whatever you decide about your home folder and your upgrade, good luck!
