### Post by wgrant on 2008-10-07
You really shouldn't be using xorg.conf to configure input devices any more - in many cases it will just break things. See the Input Configuration section of [this wiki page](https://wiki.ubuntu.com/X/Config) for details on configuring input devices in Intrepid.

As for displayconfig-gtk - XRandR 1.2 is the new way to set resolutions, and displayconfig-gtk didn't support it. It also ate people's xorg.confs, and is generally pretty useless now (except for selecting a display driver).

### Post by wgrant on 2008-10-09
> **jackbusch said:**
> I'd like my touchpad to act more like it does when I'm using OS X.

Particularly, I'd like to be able to type without having my elbows out to the side and my fingers arched away from the keyboard (which I suppose is how you're supposed to type anyway, to prevent carpal tunnels).

So, I guess I'm looking to disable touchpad while typing.

And two finger scrolling.

Turning the touchpad off is done by syndaemon, which I yesterday [ported to input device properties](http://ubuntuforums.org/showpost.php?p=5933365&postcount=17). You can find and test it there.

Two-finger tapping for right click is easily set through xinput:

```
xinput set-int-prop "SynPS/2 Synaptics TouchPad" "Synaptics Tap Action" 8 2 3 0 0 1 3 0
```

(the 8 is indicating the number of bits for each integer, and the rest the button numbers to register clicks for the following actions respectively:
top-right corner tap, bottom-right corner tap, top-left corner tap, bottom-left corner tap, 1-finger tap, 2-finger tap, 3-finger tap.)

For two-finger scrolling, it'll hopefully be exposed in the GUI some time next week (if I can obtain a UI freeze exception). For now, you can set it with:

```
xinput set-int-prop "SynPS/2 Synaptics TouchPad" "Synaptics Two-Finger Scrolling" 8 1 1
```

That will enable it in both directions - the vertical value is former, and horizontal the latter.

You will need to run these at the start of each X session.
