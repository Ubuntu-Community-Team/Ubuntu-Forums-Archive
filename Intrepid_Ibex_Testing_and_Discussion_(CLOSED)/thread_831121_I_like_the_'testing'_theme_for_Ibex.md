### Post by Raptor45 on 2008-06-17
> **Exsecrabilus said:**
> OMG YOU CAN DO THAT?????
How do you upgrade?

I'm gonna give you a warning: If you have to ask this, the dev release may not be suitable for you. Be very careful, and don't do this on your main install.

To upgrade, boot into a fresh install of hardy (no updates). Edit /etc/apt/sources.list and replace "hardy" with "intrepid". Then start up update manager and run though all the updates you can (do NOT do the partial upgrade). You will have to uncheck debconf the first time, as it needs another update applied before it will work. From there, start up synaptic and manually apply the remaining updates, skipping those which would remove important packages.

That's not what I'm saying at all. Let's imagine you perform a dist-upgrade from Hardy to Intrepid on 30th of October (or whenever the final release occurs). Unless you've treated your system badly (with 3rd party packages etc.), you can be sure a "dist-upgrade" will be successful - why? Because the repository has stabilized, all packages are built and everything has been tested thoroughly (well, hopefully).

During the development process, here's an example of what can happen: a maintainer will upload "evolution-data-server" to get built and uploaded to the repository. Pretty soon it will appear in our repository list, but if one of its dependencies are not in the repository yet (the package "evolution", for example) an "apt-get dist-upgrade" will cause most of evolution to get uninstalled. This happens surprisingly regularly during the development process (perhaps every day?) and I would bet that 80% of threads in this subforum will be due to users not understanding how to handle this situation properly.

So my point is that "apt-get dist-upgrade" is bad unless you know exactly what's happening, while "apt-get upgrade" is safer. It's explained in the stickies, if you read them (I believe I gave you the link in your other post).
