### Post by bcat on 2008-10-22
I recently installed Intrepid on my home server. (It was previously running Hardy.) The installation was quite smooth, and I'm generally very impressed. There's one small problem, however. I can't seem to set my display resolution to my LCD monitor's native resolution. The highest resolution the screen resolution applet allows me to select is 1280x960, but my LCD's resolution is 1280x1024. Needless to say, the resulting stretched picture is quite ugly. How can I get Intrepid to output 1280x1024? I was thinking I could just reconfigure xserver-xorg, but I'm not sure how this works with the new, stripped-down xorg.conf.

If it helps, my video card is an nVidia GeForce2 MX/MX 400 and my monitor is an Envision EN7410e. I'm currently using the open-source nv driver. (I'd like to try nVidia's binary driver, but my card requires nvidia-glx-96, which doesn't work with Intrepid's X server 1.5 yet.)

I've attached /var/log/Xorg.0.log in case it's useful. (I had to gzip it to get around the forum's attachment size limit.) It looks like the driver is properly detecting that my card supports 1280x1024, but the log also contains the following message repeated a few times:

```
(II) NV(0): Not using driver mode "1280x1024" (height too large for virtual size)
```

This sounds like it might be related, but what's causing the problem? Is my monitor reporting bad horizontal sync/vertical refresh limits or something? If so, can I override them?

### Post by bcat on 2008-10-22
OK, it looks like I can fix this by adding a display subsection to my xorg.conf as follows:

```
# xorg.conf (X.Org X Window System server configuration file)
#
# This file was generated by dexconf, the Debian X Configuration tool, using
# values from the debconf database.
#
# Edit this file with caution, and see the xorg.conf manual page.
# (Type "man xorg.conf" at the shell prompt.)
#
# This file is automatically updated on xserver-xorg package upgrades *only*
# if it has not been modified since the last upgrade of the xserver-xorg
# package.
#
# Note that some configuration settings that could be done previously
# in this file, now are automatically configured by the server and settings
# here are ignored.
#
# If you have edited this file but would like it to be automatically updated
# again, run the following command:
#   sudo dpkg-reconfigure -phigh xserver-xorg

Section "Device"
        Identifier      "Configured Video Device"
EndSection

Section "Monitor"
        Identifier      "Configured Monitor"
EndSection

Section "Screen"
        Identifier      "Default Screen"
        Monitor         "Configured Monitor"
        Device          "Configured Video Device"
        SubSection      "Display"
                Modes   "1280x1024"
        EndSubSection
EndSection
```

That lets me use my monitor's native resolution, but I didn't think I had to configure this by hand anymore. Isn't X supposed to autodetect the maximum resolution using my monitor's EDID? Should I file a bug?
