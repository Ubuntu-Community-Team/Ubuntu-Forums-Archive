### Post by engineer on 2008-10-08
I'm not completely sure, if this is the same behavior like others describe it here. Therefore I opened this new thread.

I have a GeForce 8400GS GPU, connected to a 13" laptop display.

These are my symptoms:
The X server starts and I see the brown background and the rotating busy cursor of the mouse, which also can be moved. From it's size I derive that the display is run at it's native resolution.
This is a steady state, so no form where I could enter username and password appears.

Additionally the fan of my laptop increases to highest speed every few seconds for at least a second. This stops as soon as I change to a terminal (Ctrl + Alt + F2 e.g.).

I installed the newest nvidia drivers (something.177.80 I think) and the newest versions of all other applications. There were no errors during the update process. Also /var/log/Xorg.0.log and /var/log/gdm/:0.log don't show any errors, though the gdm log contains some Xkb warnings described as not fatal for the X server. Sorry for not posting the logs, but I'm writing this post from the other operating system, which I luckily still have installed.

I also tried to change the Xorg driver to vesa as suggested by somebody in this forum.

Thanks for any help!

### Post by engineer on 2008-10-08
Thanks for the tip, however it didn't change too much.

Then I was back to my old state, meaning no more errors in the log files and the busy cursor but no gdm login.
