Hmm, in this day and age I don't know how you can possibly mess up a download, especially for a Linux distribution. For example: you grab a DVD image by http/ftp transfer, the download completes and the md5sum doesn't match. In the past such a scenario would have meant re-downloading from scratch - but we're living in the bittorrent age :).

Keep the corrupted ISO, then download and open the torrent for the same file. When it asks where to save files, choose the directory containing the ISO you downloaded via http/ftp. Any bittorrent client worth its salt will verify the ISO image you already downloaded* and download only corrupt chunks. This way you'll have a working ISO in a matter of seconds/minutes rather than spending hours re-downloading.

*Caveat: if the torrent creates a directory structure or the ISO file is named differently in the torrent, you'll need to close the bittorrent client, move/rename the corrupt ISO accordingly, and then open the bittorrent client again.
