### Post by mc4100 on 2008-08-13
> **olskar said:**
> I guess he meant that when you click on the systemmenu, the administration settings and normal settings has switched places.
Actually, I was referring to the so-called [Exit Strategy]("https://wiki.ubuntu.com/DesktopTeam/Specs/ExitStrategy").
Where Intrepid seems to have 3 entires in the "System" Menu that replace "Quit...", this was an upstream change:
> What other systems do

Gnome upstream

Three menu items:

    * “Lock Screen” locks the screen immediately.
    * “Log Out” leads to an alert: “Switch User”, “Cancel”, “Log Out”. It automatically logs out after one minute.
    * “Shut Down” leads to an alert: “Suspend”, “Hibernate”, “Restart”, “Cancel”, “Shut Down”. It automatically shuts down after one minute. 

You can try these options in Ubuntu by setting the gconf key /apps/panel/global/upstream_session (and then either restarting the panel or adding a new menu bar).
Although the link hints that Ubuntu might be changing it further, for example:
> Quit

The last item in the System menu (and the Main Menu) should be “Quit”, without ellipsis. (Sometimes quitting does require more input, which implies ellipsis, but currently we can’t tell that ahead of time.)

Choosing “Quit” should log out as quickly as possible. If an application cancels the logout, this should be explained using an error alert, using the application's human-readable name if known.

Also, there have been issues with clicking Shutdown and being taken only to the login screen, but as of now, the Shutdown option doesn't seem to work at all:
[http://ubuntuforums.org/showthread.php?t=885176](http://ubuntuforums.org/showthread.php?t=885176)

### Post by BwackNinja on 2008-08-18
After reading that mailing list, I kind of agree. Pulseaudio is great, but it is an incomplete solution to the problem because it specifically avoids the problem and causes a few of its own. Because pulseaudio uses ALSA, I wouldn't think there would be any reason why its features could not be integrated into ALSA. Fixing ALSA should be the priority. Pulseaudio can do fun network audio stuff, but when you think about it in that instance, you're sending audio from an application to pulseaudio, to another pulseaudio and then to ALSA when you could send it to ALSA, ALSA could send it over the network rather than to hardware, and another ALSA could intercept it and send it to hardware. Less extra steps, more direct connection to network hardware, less latency. Pulseaudio tries to reduce ALSA from a sound server to a hardware interface, it works, it adds cool features, but its not the right approach. Pulseaudio has added regressions (audio in this app doesn't work, etc) that are **not its fault**, things that it won't fix because it makes itself supreme and cuts out some ALSA interface.

Gnome is working on integrating pulseaudio tools into volume control, but you're controlling two sound servers at once and hoping they play nice together -- not ideal. What I find strangest about pulseaudio is that you are then unable to interface with alsa directly, so pulseaudio takes over rather than sending just one stream that would then be able to be mixed with other streams going to alsa. (But then again, mixing multiple times would just be frightening).

Despite how it may sound (lol), I like pulseaudio, but I'd like a cleaner ALSA better.

If you want a newer version of an application that is NOT in main on Ubuntu, you don't report a bug against the Ubuntu package.  You post it on the Debian package!
