### Post by Nullack on 2008-08-19
Folks, I think we need another sticky thread beyond the release schedule thread. Especially as were moving through the alphas and approaching beta. I thought that we as the community could do this. :)

The purpose of the thread in my view should be to guide people into making the best contributions possible to the Alpha / Beta process. In turn, save us some time as well :)

Some ideas I had about what information to include are:

1. Guide To Reporting Bugs

Something distilled from:

[https://help.ubuntu.com/community/ReportingBugs](https://help.ubuntu.com/community/ReportingBugs)

Especially info on searching for duplicates and providing useful info. Make the point that it shouldnt take heaps of time and to ease any anxieties about doing a report - its not expected they will know everything. Bugs reports can be added too.

Also here is a link that I think should be included as well:

[https://wiki.ubuntu.com/Bugs/BestPractices](https://wiki.ubuntu.com/Bugs/BestPractices)

It has some best practices about bugs

I think also we should include some info on when to raise a bug upstream and when to raise it in launchpad. I was discussing this issue on IRC with a dev and his comments were:

1. More advanced users should raise it upstream
2. Most of the code isnt patched
3. Its ok to raise a bug in launchpad to give notice to the Ubuntu devs that youve already raised upstream
4. Newer users should raise within LP as upstream can be a bit unforgiving (ever try the mplayer list? haha) / expect more

Include some info on setting up a bug watch upstream:

[https://wiki.ubuntu.com/Bugs/Watches](https://wiki.ubuntu.com/Bugs/Watches)

2. Guide to Debugging

Something distilled from:

[https://wiki.ubuntu.com/DebuggingProcedures](https://wiki.ubuntu.com/DebuggingProcedures)

Make the point that more advanced users can conduct debugging to expidite the bug lifecycle.

3. A possible "standard" that help wont be given unless its in reference to a bug or a discussion about a possible bug

This might be controversial but I don't like the attitude I sometimes unfortunately see where a user in the alpha is, well..... a bit selfish where the minimum amount of effort required to fix the particular problem is applied without getting that fix into Ubuntu or sharing it around for the benefit of everyone.

These are just my initial thoughts and I fully expect people to agree, disagree, come up with others etcetc :) The objective is we together work out a sticky thread that helps us all.

Lets really make Intrepid a great release and help people to do so

EDIT:

Maybe some stuff on bug status:

[https://wiki.ubuntu.com/Bugs/Status](https://wiki.ubuntu.com/Bugs/Status)

And some stuff on finding the right package:

[https://wiki.ubuntu.com/Bugs/FindRightPackage](https://wiki.ubuntu.com/Bugs/FindRightPackage)
