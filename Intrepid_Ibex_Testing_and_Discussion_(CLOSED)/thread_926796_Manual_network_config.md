### Post by Miguel on 2008-09-22
You can insert your own DNS's in /etc/resolv.conf and the syntax is similar to this:

```

# Dynamic resolv.conf(5) file for glibc resolver(3) generated by resolvconf(8)
#     DO NOT EDIT THIS FILE BY HAND -- YOUR CHANGES WILL BE OVERWRITTEN
nameserver 111.222.33.44
nameserver 111.222.33.16
search my.domains.com

```

Where my DNSs and the search domain have been substituted for something else. Note that if you connect to a DHCP network (LAN, WLAN or whatever) network-manager will erase your values and insert the IPs obtained via DHCP. In my case, I have to reload my Office settings every time I boot due to a Hardy bug, and it's unbelievably tiresome.
