I've the same problem and solved it pressing e at GRUB menu prompt select the kernel line and pressing e again remove the last options at the end and then press b to boot.

It boot then fine, after edit the menu.lst to fix permanently the kernel line in order to be as is below.

[COLOR=Red]kernel        /boot/vmlinuz-2.6.27-3-generic root=UUID=0586fb66-fee7-473f-a3d3-39d526d6c800 ro

[COLOR=Black]This is really a workaround not a definitive solution but it works, seems that something related with splash is the reason, also I have messages in DMSG pointing to a problems with vesa but with this workaround I've no problems and all I've tested runs reasonably.
