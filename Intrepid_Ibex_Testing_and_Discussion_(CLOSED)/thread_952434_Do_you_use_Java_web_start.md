### Post by LexRoss on 2008-10-21
I have noticed that banks in Russia require Sun Java Web Start with crypto modules to be installed in order to use their Internet banking services. Unfortunately, Sun Java Web Start is only available in 32-bit version. On 64-bit Hardy it fails to connect to my bank with "Cannot connect to server" message. Open Web Start actually logs into the server but fails to retrieve account information waiting forever for the reply from server. And the fonts are wrong and are misplaced. Hope this will be fixed in Intrepid. For now, I have to reboot into 32-bit Gutsy whenever I need to do online banking. I do not need Java Web Start in my application menu though as I would rather use set up a launcher for the specific program (online banking in my case). Not to mention that Java plugin can be used on most sites as an alternative method to launch the application.
