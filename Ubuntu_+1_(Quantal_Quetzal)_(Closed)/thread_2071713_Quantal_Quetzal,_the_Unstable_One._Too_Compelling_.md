### Post by Hedgehog1 on 2012-10-16
> **mmasroorali said:**

...

I am coming back to the original question. Is it really that necessary to release a new version every six months only for the sake of release?

I think that it now time to stop and ponder.

I understand your concern here.  A new release every six months with the level of change happening in Ubuntu is both a blessing and a curse.

Blessing: Agile/Rapid release to the field allows feedback and gathering of real world data on changes.

Curse: The tendency to take 'risks' to move things forward as quickly as possible can create issues for users of the latest release.

**Full Disclosure:** I am a software developer myself, and I suffer from the 'take risks' tendencies myself.  We have folks on staff whose job it is to tell me 'No - not on this release'.

The ideal situation is that every release, be it long term support or not, be rock solid.  I think I can be honest here and tell you that is not likely to happen for a while with Ubuntu.  The scope of changes and short time frames are generating releases with high er defect counts than we as users would prefer.

Dropping the releases to one every 12 months instead of every 6 months would not change the bug rate - you would have twice the changes in each release.

This is the reality at many other software development houses too.  It is not a good trend from the 'early adopters' perspective, but the rate of change demanded for software to work on more & more kinds of hardware is making this happen all over.

It takes deep pockets and closed systems to get a clean upgrade like the last OSX was.  But that same company released Maps for iOS that were not even close to ready for the real world.

Really, software should not be released before it is stable.  The reality is that rarely happens much in any area now-a-days.

I use the same rules for Ubuntu that I do for Windows and OSX:  Run the 'just released' & 'beta' versions on a system I can swap back expecting it to be troublesome.

So Demo from 12.04.02 for now.  You can show off 12.10 in a month or two; until then please keep being a tester of 12.10.  It's not perfect, but it is the world we live in.
