### Post by kurt18947 on 2012-08-30
You're probably running into the version problem.  Sometimes you can modify the metadata.json file associated with the extension to the current shell version and the extension will work.  Sometimes modifying the .json file doesn't help.  Here are the contents of one of my extensions installed in 12.04:

"_generated": "Generated by SweetTooth, do not edit", 
  "description": "All-in-one quick navigation to places and devices with a few extra configuration options to customize and change the look and feel of the extension.", 
 [B] "name": "All-in-one Places", 
  "shell-version": [
    "3.4"[/B]
  ], 
  "url": "http://jferrao.github.com/gtk", 
  "uuid": "all-in-one-places@jofer", 
  "version": 11

I have also gotten some extensions to install in 12.10 even though the shell-version is not the same as the shell-version in 12.10.  I'm not a coder, I guess it depends on how the extension is written.

Same problem here; most (or all?) of the extensions on the gnome.org website aren't available for the Gnome-Shell version in 12.10. Fortunately, I already had a few favoured extensions to version-edit. Others can be found at this PPA:

[https://launchpad.net/~webupd8team/+archive/gnome3](https://launchpad.net/~webupd8team/+archive/gnome3)

They will be installed in /usr/share/gnome-shell/extensions. If I recall correctly, many (most? all?) already work in Gnome-Shell 3.5.4.

Note that the PPA doesn't support 12.10 (Quantal) (yet?). So if you add it with a sudo apt-add-repository command you'll have to manually replace "quantal" with "precise" in the generated entry.

It's all a bit fiddly and I don't know if there's a better way but at least I have a usable Gnome-Shell back.
