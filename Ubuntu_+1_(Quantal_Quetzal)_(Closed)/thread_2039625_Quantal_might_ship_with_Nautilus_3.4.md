### Post by Mr. Picklesworth on 2012-08-12
> **forcecore said:**
> this has to be joke because new nautilus is so slim and good just like Elementary version was exactly.

I think there are a lot of people hopping on a bandwagon without giving this version a real shot. The removal of Compact view makes sense given the improvements to other views. The *addition* of type to search is brilliant, particularly when working with heaps of files with imperfect naming conventions. While it still has some problems, this isn't 3.6 yet: those problems can be ironed out, and they will be if there's enough positive energy.

Removing split pane is kind of a shame, especially since we only got it a few releases ago, but I guess I'm not as upset about that as I should be because the breadcrumbs work better now, the tabs _look_ better (yay, Ambiance!), and I'm an obsessive-compulsive tidier who appreciates the rationale: it's a job that should be handled by the window manager. I wish the window manager *already* handled that job (alas, "should" seems to be good enough in a lot of circles), but maybe this will inspire somebody? The window manager in Haiku is a nice example of just how much awesome you can stuff into these things: [http://www.haiku-os.org/docs/userguide/en/gui.html]("http://www.haiku-os.org/docs/userguide/en/gui.html").
