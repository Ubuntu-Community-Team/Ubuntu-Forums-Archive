### Post by DoktorSeven on 2008-02-03
Xorg is the program that does the base configuration and implementation of your graphical interface.  It's at a lower level than Gnome and KDE, which provide the look and feel -- Xorg does all of the handling of mouse, keyboard, and other functions like telling the graphics card how to draw it all.  Basically without it, you'd be stuck on the old DOSlike text screens. :)

Try pasting your /etc/X11/xorg.conf file here, that's the configuration file for it.

### Post by DoktorSeven on 2008-02-03
Your xorg.conf looks okay except that it has far fewer Modules loaded, but I'm not sure that's the issue or not.

My Section "Module":

```

Section "Module"
        Load  "dbe"
        Load  "glx"
        Load  "xtrap"
        Load  "record"
        Load  "extmod"
EndSection

```

You could try to add all of these lines that's missing to yours, restart, and see if that helps.  If not, I'm leaning toward some sort of weird hardware issue or something.

To edit:
Backup your xorg.conf: **sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.backup**
run: **gksudo gedit /etc/X11/xorg.conf** (kubuntu: **kdesu kwrite /etc/X11/xorg.conf** )
Add the missing lines to your Section "Module" as shown in mine
Save and quit the editor
Reboot (easiest way to restart X, though there are other ways that I won't go into).

If X doesn't start because of some misconfiguration or mistype, log into the textmode screen and:
**sudo cp /etc/X11/xorg.conf.backup /etc/X11/xorg.conf**
then restart.
