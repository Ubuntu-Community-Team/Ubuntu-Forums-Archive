### Post by Simian Man on 2009-11-13
If it's Flash, then it really doesn't matter, because the flash plugin is the same across browsers and will be doing all the work.  If you have constrained resources where the browser itself is consuming enough resources that it competes with the flash plugin, then I suppose it would make sense to go with a lighter browser like Midori.

If it's Javascript-based, then Chrome is supposed to have improved performance over most other browsers out there, but I haven't really investigated this.

### Post by Hungry Man on 2012-08-19
Google Chrome.

PPAPI Flash should allow for improved performance.

[http://blog.chromium.org/2012/08/the-road-to-safer-more-stable-and.html](http://blog.chromium.org/2012/08/the-road-to-safer-more-stable-and.html)

> 
Beyond the security benefits, PPAPI has allowed us to move plug-ins forward in numerous other ways. By eliminating the complexity and legacy code associated with NPAPI, we’ve reduced Flash crashes by about 20%. We can also composite Flash content on the GPU, allowing faster rendering and smooth scrolling (with more improvements to come). And because PPAPI doesn’t let the OS bleed through, it’s the only way to use all Flash features on any site in Windows 8 Metro mode.
