### Post by cogadh on 2007-08-16
This has already been debated so many times now. Linux represents less than 2% of the entire desktop PC market (it does have a larger percentage of the server market). That's approximately 13 million Linux desktop PCs compared to the approximately 605 million Windows desktop PCs worldwide. Even if every single Linux user is a paying gamer, it is still not worth the additional expense and time required to make a Linux version of a AAA Windows game. Especially when you consider that most Windows games require some DirectX functionality and there is no DirectX on Linux. Game developers would either have to switch to all open source/standards APIs (OpenGL, OpenAL, SDL, etc.) or code two separate versions of the game. Most of them are not willing to do that since it is so much easier to just use DirectX and stick with selling games for the majority market.
