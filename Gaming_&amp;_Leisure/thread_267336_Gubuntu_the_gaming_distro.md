### Post by linuxlizard on 2007-06-08
Here is a problem with games on linux that I never see anyone talk about and I believe is an obstacle for professional game development. 

Different distros use different kernels and libs. Even a single distro may have significant changes every few months when a new version of the distro is released. For example, I purchased a several of the old loki games. All installed fine 3 years ago without extra tweaks. By a little over a year ago, when i was using breezy badger, I could still get most of them to install fine, but they required tweaking to get them to run and one or two I couldn't get going any more. Today, I can get only one to install (heroes3) and can't get it running full screen or over a network. I haven't found tweaks to get it working right or to get the others working at all. These games have had an effective lifespan of less than 3 years. On the other hand, I've got windows games 10 years old that are still running fine even though I changed from 98 to xp several years ago.  

Neverwinter nights was running fine in edgy, but I'm not sure I want to go through the hassle of tweaking it to run in feisty when I can just switch over to my xp side and install it in a few clicks. 

Somehow linux needs to be a little more accommodating so if a game requires certain libs, they aren't going to have problems in a few months. Maybe the way they link to the libs or maybe some buffer between the libs and the game that finds the correct libs regardless of version numbers, etc. What's the point in paying for a game  that will be unplayable in 2 years if one can buy the same game on windows and play it for several? As a game developer, would you want to have to figure out how to make your game compatible with several linux distros and then on top of that spend extra time and money coming out with a new installer every few months to keep your game compatible with several distros, or spend your time and money developing a game for 98% of the people that will pay for it who also happen to be those who will run it on windows and only have to make patches to fix errors in the game code?

I'm not saying I'm not a huge fan of linux - I am, but as someone who has some commercial linux games, there needs to be a better way to keep games running in spite of new linux versions coming out every few months. Linux changes versions constantly.

Maybe this isn't as much of a problem as I think, having a little bitterness myself about my loki games not running anymore.:(

### Post by cogadh on 2007-06-08
frankiekp, nVidia already supplies constantly updated Linux drivers for all of their hardware under the same type of unified driver structure they use in Windows. They have always been very strong supporters of the Linux community and are already "behind us" on this front.

ATI, on the other hand, barely supplies good Windows drivers, let alone good Linux drivers (just ask anyone who has tried to play an OpenGL-based game on the recent Windows Catalyst drivers). Anyone who wants to game seriously on Linux will have to avoid ATI products until this changes, or put up with the hassle of GLX configuration.

What Linux really needs is a compatibility layer that replicates DirectX on Windows. Before DirectX, getting games to work on a Windows PC was a hit or miss process, much like what linuxlizard describes as his Linux gaming experience. Install a new driver or update a Windows component and suddenly games that used to work fine were no longer functional. With DirectX, game developers don't have to try and make game compatible with every possible hardware/software/driver configuration out there, they just need to make it compatible with whatever the current version of DirectX is. Since DirectX is backward compatible, they never have to worry about the game failing if DirectX is updated. The same is true of device drivers. New drivers just need to be able to communicate with DirectX, and generally speaking, the games will work fine.

Unfortunately, creating a DirectX-compliant compatibility layer for Linux would be no small feat. Wine/CrossOver/Cedega come close in some ways, but they are not good enough (yet). Linux has finally got to a point in recent years where it is truly a viable desktop replacement OS, maybe now some of those geniuses who made that happen can work on making it a replacement gaming OS.

Example bash script contents:

installgames () {
	clear

	# package names to be installed
	GAMES_NAME_LIST=""
	GAMES_NAME_LIST=$(cat gameslist | grep -v -e "^#" | cut -f1 -d' ')

	echo ""
	echo "Installing games:" ${GAMES_NAME_LIST}
	echo "--------------------------------------------------------------------------------"
	sudo apt-get install -y ${GAMES_NAME_LIST}
}
