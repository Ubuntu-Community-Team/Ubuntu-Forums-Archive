### Post by RichardLinx on 2010-03-01
I think castlefox is a little bit too optimistic with his views on gaming with Ubuntu/Linux. 

@OP: There is evidently a lot you don't understand about Linux if you are considering it as a viable gaming platform, it's not. You have very modern hardware - The Linux kernel doesn't typically support "new" hardware for a minimum of six months, and even when it does, it's not at it's optimal performance.

Linux is ideal for modern computing tasks. It is horrible for modern, high-end gaming.

My suggestion to you would be to dual-boot. Make your Windows partition exclusive to gaming only and use your Linux partition for your computing needs.

If you're into retro gaming then emulation on Linux is fantastic. (Except Dreamcast, lxdream is the only decent dreamcast emu I know of for Linux, and it's development is slow because there are very few people working on the project.)

PCSX2: (You can run pretty much all PS2 games at full speed and higher resolutions then they were originally designed for.)

Dolphin: (Gamecube/Wii Emu) [This actually runs many commercial Wii titles better then the Wii does - Metroid Prime 3, Zela TP, Mario Galaxy in HD 720p. Also runs most Gamecube games. Full speed.]

N64 Emulation on Linux is great. (The latest Mupen64Plus looks very promising)

ePSXe (PSone emu) has already proven its self as a fantastic emulator.

NES, SNES, SEGA Mega Drive, etc are all great on linux too.

Honestly though, you sound like a modern hardcore gamer who would rather be playing Doom 4, Rage, Crysis 2, etc so yeah stick to Windows for gaming. Linux for computing. Setting up a dual-boot system is easy (Plenty of instruction on the web) so you shouldn't be too lost there.

Linux distros I recommend for you:

Linux Mint
Ubuntu
openSUSE
PCLinuxOS

... Seriously, I don't think Linux even has proper support for Core i7 yet.

### Post by IceDoE on 2010-03-03
I like gaming. I used to keep Windows around on a dual-boot just for a few games. I no longer keep it around.

Here is the thing: you will not be able to play every game designed for Windows in linux, at least not right away (you can do this even less in Mac OSX, and good luck trying to get Windows to run something designed specifically for a linux system). When a program is designed for specifically one thing, it is difficult to make it run on something else. 

But that is not where it ends.

Many games, designed for Windows (because it is currently the most popular OS, mind you), will run on linux under programs like WINE, or through other methods. Some of these will work beautifully and without a problem, some will need some tweaking, some will just have problems. My brother plays WoW (full expansions) on this machine without any problems, and while a year ago there were MINOR issues installing everything, even those are gone now. Interestingly, some areas have lower FPS in that game, some actually have higher under WINE. Lots of other games will work. The only ones I haven't been able to get fully working are my old Star Wars games, and they apparently used to work, but I think no one has bothered trying lately because they are now pretty old (I haven't tried that many newer games though, mind you).
I do personally play Heroes of Newerth, which, though in Beta, is a commercial game that can push things. Luckily, it has a linux version. The number of commercial games made with a linux version is still small, but it is growing, and will grow faster the more people are using linux (similar with other kinds of software).

But, on another side, there are alot of Open Source games, and alot of older games or freeware or just linux games out there that will work great on linux too. The quality isn't always as high, but often you aren't paying for them either. I like to go through the repository and some other sites or libraries and try out various games every once in a while. Sometimes I keep them and have alot of fun, sometimes I get bored and very quickly and easily remove them. If I get it from Add/Remove or the Software Center, all this is extremely easy. 

As for your hardware: Well, I can say at least that I have a nVidia 9800 GT, which isn't actually that advanced anymore but is still a better card, and have no problems with it. Ubuntu, or something, knows what it is right off the bat and will get you nVidia's settings software to do your own adjustments with it. And, like I said, I play some graphic intensive games with it (though not Crysis level I don't think, maybe if I set everything to Max in HoN...). The rest of your setup I don't think you really have any reason to worry about, the i7 does work (and has since the introduction as far as I'm aware) with linux, and I've seen specifically with Ubuntu. The benchmark tests I saw in fact used an i7 and the 9800 GTX, and overclocked it, running some games like Nexuiz with Ubuntu 9.04 -64 (beta) (old, obviously).

In the end, if you are really tied to particular games you want to play that are designed for Windows, check around to see if they can work, or if there is an acceptable equivalent for linux. If not, you'll need Windows to use it still. Though, as mentioned above, dual booting isn't very difficult to do. At that point, you might like the aspect of playing the high-end windows games, but also trying the other quick or extensive games you might find in the linux repositories or some other place.
