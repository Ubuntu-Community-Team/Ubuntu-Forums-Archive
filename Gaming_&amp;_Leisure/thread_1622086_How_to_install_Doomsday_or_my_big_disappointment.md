### Post by staim on 2010-11-15
I'm Linux user since 2003. I've tried many distros (Red Hat, Slackware, Gentoo, Debian etc) and I really love it. But in recent 2 years I'm working with a computers a lot on my job and I have a family at home, so I don't have time for configuring, compiling etc

That was the reason, why I've switched from Gentoo to Ubuntu. It looked to me as out-of-the-box working system (like Mac Os X). But I was dissapointed. 

My story begins when I have tried to install DooM (that a game I like since 1995). But not a simple DooM, but a nice-looking DooM with 3D models and cool effects. In the past I was playing jDoom or Doomsday on my friend's Windows PC (but I have Windows) 

At first when I was a Gentoo user I used portage. It contained every program or service I may even think of. It was just enougth to type "emerge <program name>" to install it. in Ununtu we have a nice looking Software Center, but I was not able to found jDoom or Doomsday there. 

After that I gone to oficial site: [http://www.dengine.net/](http://www.dengine.net/) and it suggested me to add a repository. And after that I was asked to download archive signing key and install it manually, hopefully I know how to do it, I'm a programmer, but Ubuntu is a OS for normal people, isn't it???

I was able to install deng engine. But there was no GUI utility, no 3d models, no hi-res texutures. I was offered to use SVN to install snowberry (GUI utility to configure it). I'm a programmer and I know how to use SVN (don't think normal user does), but svn was not working and I got an error message, so I was not able to install snowberry.

I've launched deng from console. I has started 640x480 without 3d models, effects texutre packs and when I selected exit - it crashed my X Server. 

I've got tired with it and I downloaded setup for windows. And run it under WINE!!! And it installed everything. Deng, snowberry, texture packs. It is working perfectly UNDER WINE!!! Damn it!!! I can understand such things with a propietary software. But not with our own Open Source projects.
