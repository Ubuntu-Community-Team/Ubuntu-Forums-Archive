### Post by Asacolips on 2009-02-06
I've recently found some work arounds to get  Game Maker 6.1 made games working fine under Linux.  It's main two issues before were no sound and no fullscreen scaling, both of which I've found how to fix.  I used a dll and some surface functions, however, which means you have to have the registered version.

Anyways, I was wanting to test it out and see how it fares on different peoples' pc's.  I'm testing using Linux Mint, but it should seemingly work on any Linux distro with Wine 0.9.49 or above.  Again, I'm testing using 1.1.14, but 0.9.49 was the oldest known version to work with Game Maker 6.

If you wouldn't mind, download from [http://www.rocketsoft.gm-school.uni.cc/uploads/gm6-linux-0.1.zip](http://www.rocketsoft.gm-school.uni.cc/uploads/gm6-linux-0.1.zip)

I'd really like to see some results.  In order to test, run sample.exe
If you want to try out making games, get a registered copy of gm6.1 (a little hard since they no longer offer that) and run sample.gm6

Lastly, here's some screens showing it in action.  Note that the border on the right is because of my widescreen monitor, something that can be fixed by changing the game's resolution when making it.
