### Post by R33D3M33R on 2011-07-30
> **Shazzner said:**
> Hammerfight doesn't seem to register my mouse in fullscreen, unfortunately I have no idea how to turn it back. :(

There is a config.ini in the game directory. open it with a text editor, find:
```

GFX_WINDOWED=false
```

and set it to:

```
GFX_WINDOWED=true
```

### Post by ZarathustraDK on 2011-08-10
> **dreadpiraterobert said:**
> 
Sorry for being new, but how do I install these games?  What version do I install (some games have 4 options)?  I've only ever installed applications from the Ubuntu Software Center so these .bin, .tar.gz, .deb, i386 and amd64 files mean nothing to me.

deb-files are your normal installation-packages like what the Ubuntu software center uses, and probably what you'd prefer on Ubuntu. Install it by double-clicking the downloaded file. If it complains about a dependency (it'll specify which one) open up a terminal and write "sudo apt-get install *name of missing package*", rinse and repeat until the game works as there may be more.

tar.gz-files, in this instance, are generally just the game-files dumped into an archive (called tar or tar.gz, just like windows-zip-files). Extract them by rightclicking the archive and choose extract here. Start the game by going into the extracted folder and doubleclicking the game-launcher, usually a .bin or .sh file. These can be called anything, use common sense.

bin-files (as in a file you downloaded containing a game) are binary files that usually contains a setup/install-program for the game; run it to install the game, either by double-clicking or do this in a terminal:
```
cd "path to where the downloaded file is"
./"name of file"
```

both tar- and bin-files are sometimes not able to run right after download, that's because they got the "allow execution"-flag unticked in their properties for security reasons (you don't want stuff from the internet to be able to run off the bat unless you permit it). Right-click the file, choose "properties", then "Permissions", then tick the "allow execution" box and apply changes, now you should be able to run it. Alternatively do this in a terminal:
```
cd "path to file you wish to run but can't"
chmod -X "name of file you wish to run but can't"
```

That's the cliff-notes on these filetypes. I hope you can see why you would always prefer deb-files over the others ^^

EDIT: oh, and the i386 and amd64 specify which architecture your OS is running 32-bit or 64-bit. Most likely you're running 32-bit (i386) as running 64-bit is usually a conscious decision that would require you to know the difference.
