About right, but slightly misleading - there is a misconception amongst some that because you have developed open source software, anyone can demand the source from you (an example I heard recently was someone who believed that Google would be obliged to hand over the source to Goobuntu should someone ask for it).

As another example, imagine a company was selling an open source game but, unlike the Yo Frankie guys, they did not offer a free download.  You could not say to them "I don't want to buy the game from you - just give me the code".  The obligation to provide source code to you is only applicable if they have supplied a copy of the game. Of course, Open Source being the way it is, as soon as one copy is sold it is out there and can be redistributed by others at will.

Your post seemed to imply the misunderstanding I described, but I apologise if that was an error in my reading of it.

### Post by ajcham on 2009-02-04
> **Firestem4 said:**
> how would an Open Source company make any money if they had to release their source code if it was demanded?

Well, that is still an issue - as I said, the people who have bought the software from them *are* entitled to the source code (it's all the other Toms, Dicks and Harrys that are not). Having received the program and code under an OS licence, the customer may then copy and redistribute it as he pleases.

A good example is Red Hat Enterprise Linux.  Red Hat only provide this software to paying customers - however, members of the Linux community are then able to repackage it (removing certain non-free elements) and we get CentOS, which is freely available.

Companies such as Red Hat, Novell and Mandriva which strive to turn a profit from Open Source software typically do so with the Enterprise support packages they offer their customers.
