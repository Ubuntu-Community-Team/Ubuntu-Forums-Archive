### Post by nexoncore on 2010-07-14
Before I start, this is a request / suggestion and if its in the wrong forum then please move to the correct one.

The following link shows augmented reality using a chibi character, the video itself does not show the true program's nature but i have been unable to find the name of the program.
[http://videosift.com/video/Augmented-Reality-Anime-Girls](http://videosift.com/video/Augmented-Reality-Anime-Girls)

My suggestion / request is for something along similar lines to be developed for and intergrated into ubuntu, but not the augmented reality side, more of a desktop mate. A selection of character's such as chibi or western characters which can be interacted in like they were a desktop pet. But also intergrate it into the system allowing it to perform certain action's or animations when performing tasks.

For instance, if you open up the terminal, the character could pull out a laptop and start typing away. It could also be used to convey system messages, the messages could also influence the expression of the character, such as a low level error might result in a hiccup, but a majour error such as a driver failure could result in the character acting like the world is about to end.

I feel this would be a good project for the following reasons
- This could be an integral way to invite new audiences
- Would provide a new way of communicating system messages with severity
- Could be intergrated into other programs and features such as Mail, Wifi, Rss Feed
- Could also be used for Help, FAQ, Reporting Bugs

Basically like a desktop assistant, and for linux lover's, we could always have the penguin.

Would love to hear your views on this idea, or if anyone would like to pick it up. I would love to make it myself but I know nothing about linux programming.

### Post by frostwork on 2010-08-10
Hi!
*(let me introduce myself quickly, as this is my 1st post. I'm frostwork, a long time gentoo user and I like searching for new and exciting opensource programs, patch them for linux (and maybe also create gentoo ebuilds for them). I always thought that the ubuntu community is very innovative and excellent at brainstorming and teamwork and therefore I finally registered here :) )*
but now back to topic ;)
---

The application shown in the video is very likely artk-mmd
( [http://ppyy.hp.infoseek.co.jp/artk_mmd.html](http://ppyy.hp.infoseek.co.jp/artk_mmd.html) ) which is even opensource and only
requires just a few patches to work on linux.

For using mikumikudance models as "desktop models" you could try my little program
dunan, which can be found here:
[http://www.frostworx.de/?p=64](http://www.frostworx.de/?p=64)
I took the main mmd code from artk-mmd (and/or from ardesktop, which can be patched to linux too btw - [http://wiki.livedoor.jp/wah_wah_hawah](http://wiki.livedoor.jp/wah_wah_hawah) ), applied some patches and made a shared lib from it, which can be found here:
[http://www.frostworx.de/?p=43](http://www.frostworx.de/?p=43)
dunan depends on this lib.
It is in early stages and I haven't touched the code since some time, but this might change, when I'm "done" with my main project "typhon" - [http://www.frostworx.de/?p=1](http://www.frostworx.de/?p=1).
help, patches and suggestions are very welcome :)
cheers,
frostwork
