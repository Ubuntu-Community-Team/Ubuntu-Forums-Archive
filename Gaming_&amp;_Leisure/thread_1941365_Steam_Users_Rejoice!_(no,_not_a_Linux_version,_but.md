### Post by lite1979 on 2012-03-15
For what seems like forever, I've had to run steam using the "-silent" option at the end of the string to prevent a steam browser launch that ultimately crashed steam.
During this time, I was still able to play half-life 1 (which is the only game I play on steam, but keep reading) by starting steam in wine using the -silent option, waiting for the little icon to appear in my upper right corner (yes, I'm using 10.04 LTS on a x64 machine), and clicking it to select half-life. Usually, once I ended my gaming session, a steambrowser would pop up (the steam news advert, IIRC), and this would crash steam.
Recently, however, I've been getting a pop-up from wine saying "steam has encountered a problem and needs to close." This isn't true at all, though. I click "ok" on the pop-up, and steam continues to run just fine.
So, I decided to remove the "-silent" option from my command string. The steambrowser no longer crashes steam, but for the past few weeks it didn't display anything; just a blank page.
After last night's update, the steambrowser works like it should have all along:
