### Post by kegie on 2007-05-30
At first I wrote this as a reply to the petition thread, but it wasn't really related so I decided to post it as a new topic instead..

Unfortunately, I think getting commercial games natively for linux is a long way off. First of all, commercial software and linux doesn't really mix well. The commercial software that is out there already sells disproportionally few copies compared to, say, Mac OS. The explanation for that, I think is simple: in most cases, there are free alternatives that work better in linux than the commercial ones, and since most people already have these alternatives easily available to the distribution, there is little incentive to seek out and buy the commercial stuff.

The biggie as I see it though, is this: Anyone who makes a game for linux now has to be a pioneer. There simply isn't a well-defined standard way of releasing software on linux outside of the distribution channels. To get things streamlined, you'd need to package the game for APT, RPM, whatever mandriva uses, SUSE and so on. There are some efforts to make it easier to release software as standalone packages, but nothing mature enough that, if you went that route, you'd have to spend as much time debugging and fixing those systems as in actually porting the game.
Just look at Loki, and everything they had to write themselves to get games onto linux. I think that was a big part of what made it so hard for them. It is kind of like the situation on windows before DirectX 5 came out. The first game that I remember really working well on windows was Diablo.

The games we do have ports for, like UT etc, are enthusiast ports done for no expectation of commercial gain whatsoever beyond goodwill. Most gaming companies just won't get permission from their publisher to spend money on something where commercial benefit is unlikely. Sadly.

However, I do think we're getting to the point where it won't be too expensive to release games on Linux compared to the rewards. Especially if the Ubuntu Dell deal takes off, and we could get APT as a kind of standard for distributing software - then you could just set up something like steam but through APT.

Really, I think all it takes is one commercial success under linux, where someone who is just a user with little system administration skills can buy the game and get it working without help. There are a lot of game developers out there who run linux themselves, and would love to be able to release their games for it.

This is probably something the community could do.. set up something kind of like steam, maybe under games.ubuntu.com? And have some actual commercial software there - not really with the intention to make money, but as an example of a distribution system, setup/install pipeline, using technical support things like SDL etc. That way, a developer could point to that website and tell the publisher, look, it's not a huge risk.
