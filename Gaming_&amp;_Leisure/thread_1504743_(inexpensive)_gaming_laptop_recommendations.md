### Post by Progressive on 2010-09-29
radeon and radeonhd are DDX drivers that hook into the X Server, but going forward we might not need either of them.

The reason is that now there is modesetting support within the linux kernel for all ATI cards. This is a prerequisite for Gallium3D.

Gallium3D is a cross-platform abstraction layer which supports a number of "state trackers" which are like plugins. That DirectX support I was talking about is in the form of a state tracker.

Other state trackers include EXA (fast 2d rendering), OpenVG (2d vector rendering), EGL (for mixing OpenVG, OpenGL, etc), [and so on]("http://www.x.org/wiki/GalliumStatus").

Once a gallium3D driver is written for a card, it can gain support for all those different state trackers with minimal coding on the part of developers. This also allows Intel, ATI, and Nvidia to share a lot of the same codebase (state trackers and gallium platform), so when one of them improves a state tracker or something, everyone benefits.

There is also an Xorg state tracker in the works which is likely to make the DDX drivers obsolete in the future, but for now they seem to have bigger priorities.

OpenGL 3 and 4 state trackers are in the works, and I would say it is ironic that we have DirectX 11 support before OpenGL 4, but from what I hear OpenGL is a very messy platform.

Catalyst is buggy as far as I know, especially with WINE, but when it works it is typically better at 3D rendering for now.

The 5xxx series is known as R800, but they use the R600 3D architecture, so the open source mesa drivers for it are known as r600c and r600g. (c stands for classic and g stands for gallium)

r600c is a bit more feature-full and stable at the moment, but r600g is progressing quickly. 

I believe you can play Nexuiz, Alien Arena, and all that no problem on either card. DirectX stuff is still new and only works on r600g.

I don't have a R600 or newer card, I have a Radeon X1800 which uses the more mature r300g driver (I can play Oblivion). So my experience doesn't translate so well and you just have to try r600g yourself.

To test out r600g without compiling everything yourself you can just get [this repository]("https://launchpad.net/~xorg-edgers/+archive/ppa/"), and do a ```
sudo apt-get upgrade
``` 

The follow this instruction to switch from r600c to r600g:

> Gallium drivers for r600, i915, and swrast (llvmpipe) are in /usr/lib/dri/gallium and must be manually selected with a LIBGL_DRIVERS_PATH="/usr//lib/dri/gallium" env variable or moved to /usr/lib/dri if you wish to use them because the names conflict with the classic mesa drivers.

It is updated basically daily, so you should see it progress right before you eyes. If it starts acting up for any reason, since you won't be running a stable release, to remove it do ```
sudo ppa-purge xorg-edgers
```

Though I haven't had desktop freezes or anything in a long time.

To get the DirectX stuff to work you'd have to head over to [phoronix]("http://www.phoronix.com/scan.php?page=news_item&px=ODYzMA").

### Post by efikkan on 2010-09-30
Remember we are talking about gaming, both fglrx and nVidia's proprietary drivers got more than enough 2D-performance, so 3D performance is what matters. I'm not trying to force anyone to use one driver, but I want to inform you about the benefits and disadvantages. I'm working with OpenGL and graphics on a daily basis. But first of all I have to tell I love open source and free software, but I don't use free software at any cost. So you'll have to consider what is the most important:

Free ATI drivers:
+ Free and open source.
+ May be preinstalled(saves you three mouse clicks!).
+ May have decent 2D performance.
* May have acceptable stability for some uses.
- Very low 3D performance.
- Lack of support for newer OpenGL and any OpenCL spesification (currently lagging 4 years behind in OpenGL support and does completely lack the core features of modern graphics).
- Poor power management.
- Slow support for new hardware/lacking support.

Proprietary nVidia drivers:
+ Enterprise level quality.
+ Excellent performance (equal to windows drivers and Direct3D).
+ Good power management (important for laptops).
+ Complete support for OpenGL and OpenCL specifications.
+ Video acceleration.
+ Working GUI for multiple monitor set up.
+ DeepColor support (only for cards with DisplayPort, wich is mostly Quadro and some Zotac cards).
- Proprietary.

Proprietary ATI drivers:
+ Usable performance (but not good compared to Direct3D performance on windows drivers or OpenGL performance of nVidia drivers).
+ Working power management (still nVidia is better).
+ GUI installation.
+ Support for new OpenGL and OpenCL specifications (but see below).
- OpenGL implementation may be unstable/buggy. It might work, but fails in some releases. (*)
- Low OpenCL performance compared to nVidia.
- Lacking video acceleration.
- Multiple monitor setup tends to crash X or fails to detect the correct screen resolutions.
- Proprietary.
? DeepColor support, I don't know.

(*)I have to say the proprietary ATI drivers have come a long way the last two years, but nVidia is still superior i most cases. But AMDs lack of interest in actually creating a working implementation of OpenGL and the performance annoys me. AMD actually managed to ship two driver releases without a working GLSL compiler (Catalyst 9.12 and 10.1), I don't even understand how this is possible. The quality of the driver releases may vary a lot, so you might want to try a few versions. As of today you might experience 50-80% of the Direct3D performance (of the same card with the Windows driver), so it's fully playable but nVidia is the better choice. Especially on a laptop where power management is very important, you would have to get a more powerful ATI chip to compete with nVidia, so it's going to be hotter and more expensive. Currently nVidia offer more "Windows-performance" for the money, and since nVidia's GNU/Linux performance equals their windows driver, and ATI does not, I would stick with nVidia.


Performance wise, let med put it this way: a high-end graphics card with the open drivers will perform like a low-end card with the proprietary drivers (nVidia might even perform better), and it will get very hot. So it's a waste of money at best. It's up to you if a free driver is that important.

I really dislike those who almost forces new GNU/Linux users to stick the crappy free graphics drivers, quite a few uses has been scared away. Proprietary drivers _are not evil_, and are indeed acceptable when they offer superb performance, stability and features. Proprietary operating systems, APIs and file formats are however "evil", _because these locks people in_. I would prefer that every piece of software was free, but for now that is not realistic. Maybe someday in the distant future...
