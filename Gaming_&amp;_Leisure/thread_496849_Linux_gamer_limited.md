### Post by MonkeyBoy on 2007-07-10
wDude it just took me less than a minute to download, configure and run Mupen64 with no cli use required.

Step 1:  Go [here]("http://mupen64.emulation64.com/down.htm") and download the Linux version.
Step 2:  Go to wherever you downloaded it to and extract it wherever you want it.
Step 3:  Go to the directory and double click Mupen64 with the blue cog diamond icon thing above it.  Hurrah!  It should run.
Step 4:  Now click "options" and try out some of the audio and input settings until you find ones that work for your setup.
Step 5:  Click "open rom" and point it at a rom.
Step 6:  Don't get cross on the forum 'cos you don't know how to do simple tasks like this.

### Post by DoktorSeven on 2007-07-10
One reason there is no debian package is because it writes its configuration files into the install directory, and if a debian package installs it in the usual location then it would either not be able to write its configuration or would have to share configs with multiple users.

It's thousands of times easier to just download the thing, put it in your home directory, and run it.  Think of it as being like a program that comes in a ZIP file for Windows that you have to extract and make a link to instead of it being an installer.
