### Post by zitch on 2010-10-18
I've been playing this game just fine on my Ubuntu 10.04 64-bit laptop for the past weekend (and yes, it is quite addicting if you're into sandbox-style gaming).  I have Sun's (now Oracle's) Java 1.6 installed (*apt-get install sun-java6-jre*, I think), and use the following script to launch the game without any issues:

```
#!/bin/bash

java -Xmx1024M -Xms512M -cp Minecraft.jar net.minecraft.LauncherFrame
```

I even have a private minecraft server running on a CentOS box (with OpenJDK).  I use the following script (not complete, shutting down isn't working properly unless you use "connect" to the server and then enter "stop") to have a daemon-like instance of the server running on it:

```
#!/bin/sh
SERVER_PATH=/home/xxx/bin/minecraft
SCREEN_NAME=mc_server
MEMORY=1024

LAUNCHER=$0

function start_minecraft 
{
	echo "Starting Server"
	if [ -e $SERVER_PATH/*.log.lck ]
	then
		echo "Server already running."
	else
		screen -dmS $SCREEN_NAME java -Xmx${MEMORY}M -Xms${MEMORY}M -Djava.net.preferIPv4Stack=true -jar $SERVER_PATH/minecraft_server.jar nogui
		sleep 5
	fi
}


function stop_minecraft 
{
	echo "Stopping Server"
	if [ -e $SERVER_PATH/*.log.lck ] 
	then
		screen -S $SCREEN_NAME -X stuff $'stop\015' 
		sleep 5
	else
		echo "Server already stopped."
	fi
}

function connect_minecraft
{
	echo "Connecting to Minecraft Server"
	if [ -e $SERVER_PATH/*.log.lck ]
	then
		echo "NOTE: To detach from server, press CTRL-A-D!"
		sleep 5
		screen -r $SCREEN_NAME
	else
		echo "Server not running!"
	fi
	
}



if [ $1 == 'stop' -o $1 == 'restart' ] 
then
	stop_minecraft
fi

if [ $1 == 'start' -o $1 == 'restart' ] 
then
	start_minecraft
fi

if [ $1 == 'connect' ]
then
	connect_minecraft
fi

exit 0

```

Hopefully, this helps others get some ideas to setup their own server.  Feel free to make improvements to it.

UPDATE: if you're loading it from the website, It looks like java is set to use up to 512 MB on the site.  My own script moves the limit up to 1 GB for java (command line option "-Xmx1024M" sets the max.  A "-Xms512M" option sets the starting memory size for Java at 512M).  By default, I think java is set to max out at 64 MB (which isn't enough for the game, from what I remembered)


UPDATE 2: jconsole is a neat tool to use to check on memory usage of Java apps (might be a Sun JRE tool, though; not sure if there's something similar for OpenJDK).

Basically, run "jconsole" in a terminal.  On the New Connection dialog (or go to the menu Connection->New Connection), pick "Local Processes", the select the process that corresponds to the minecraft game (if running off of the website, it looks like it starts with "sun.plugin2.main.client.PluginMain ".  When I run my local copy from the scrip I use, it shows up as "net.minecraft.LauncherFrame").  From here, you can watch the memory usage, threads, and all other kinds of info.  Have fun... :)
