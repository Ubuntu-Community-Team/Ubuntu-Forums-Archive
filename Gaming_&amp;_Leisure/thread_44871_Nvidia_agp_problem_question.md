### Post by skoal on 2005-06-27
Omnios, everything checks out pretty good that I can see.  You could try adding "RenderAccel" "on" (option) to your 'xorg.conf', but that won't help your 3D out any.  * warning! that option has been known to cause hard locks on older GeForce cards.

By the way, you should have "intel_agp" loaded as a module if you're using the agpgart kernel interface, instead of Nvidia's.  Just for shiznitts and grins, if you want to enable the Nvidia GART:

1. (maybe not entirely necessary) Add these 2 modules to '/etc/hotplug/blacklist':
# To use Nvidia supplied agpgart
agpgart
intel-agp

2. "Flip the switch" on which agpgart interface gets loaded by setting the following option in red:
```
Section "Device"
        Identifier      "NVIDIA Corporation NV25 [GeForce4 Ti 4600]"
        Driver          "nvidia"
        BusID           "PCI:1:0:0"
        # Insert Clocks lines here if appropriate
[...]
        # NVIDIA Options
        [COLOR=DarkRed]Option          "NvAGP" "1"[/COLOR]
        Option          "RenderAccel" "on"
[...]
```

3. logout, kill X with ctrl-alt-bkspc, fire up a terminal and check which one you're using:
```
skoal@morpheus:///data/ubuntu/dev/libfmod-3.74.0 $ cat /proc/driver/nvidia/agp/status 
Status:          Enabled
[COLOR=DarkRed]Driver:          NVIDIA[/COLOR]
AGP Rate:        4x
Fast Writes:     Disabled
SBA:             Disabled
```
Unfortunately, I don't think there's a magic bullet to be found here.  YOu could play with your AGP aperture size in BIOS.  The nvnews.net forums have a good FAQ on what to set that to.

### Post by wylfing on 2005-06-28
Blacklisting agpgart itself is not always the right solution, and can prevent X from starting. I'd start by just blacklisting intel_agp. That's probably the real culprit.
