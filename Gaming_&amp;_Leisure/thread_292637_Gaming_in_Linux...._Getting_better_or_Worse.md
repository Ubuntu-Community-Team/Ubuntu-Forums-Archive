### Post by mongrol on 2006-11-20
> **Artificial Intelligence said:**
> I second that. Would like an explaination as well.

I think it's fairly obvious when you think about it. Games developers these days demand componentised development frameworks. They buy 3d engines, physics engines, scripting frameworks, sound engines from 3rd parties. These are specialised off the shelf products produced specifically for the industry and allows them to reduce development time, churn out product quicker and maintain a relatively stable development environment.
DirectX (now XNA) provides one of these development frameworks that is very well supported, very popular with developers and gives them out of the box solutions to 3D, sound, gameflow scripting, networking, workflow management and now even provides XNA Express which is an introductary environment for up and coming developers.
Linux has none of this game developer support. OpenGL is a graphics library, OpenAL is a sound library developing at a glacial pace.
SDL is the closest we have to DirectX but it's tiny in comparison. Without a major project, and I mean major in an openoffice type of sized project, to produce a similar well supported framework for game developers then they are continually going to go with the easiest route to making money.

### Post by curvedinfinity on 2006-11-26
As it is, any effect that can be done in DirectX can be duplicated in OpenGL. This is because they both have access to the programmable pipelines of modern graphics cards. The difference is that DirectX often abstracts common effects above the raw shader pipeline, so in effect it does them "automatically," but in reality, OpenGl will run the effect at the same speed as DirectX, just you need to make it yourself.

As it is though, I don't believe DirectX has a feature for a water refraction shader, so in all likelyhood, the game you are playing wrote that shader in HLSL, which is the DirectX-exclusive shader language. Alternatively, they could have written it in another median language like CG which allows them to compile the shader into both HLSL and GLSL (the OpenGL shader language).

Shaders are a real mess as it is. I can't wait until there is a more established abstraction layer over them so making one becomes simpler.
