### Post by DarkAmbient on 2013-09-07
PoL is "just" an manager that does Wine-stuff for you.

Normally with Wine you have a .wine -folder in your home-folder. Everything you install in Wine ends up in the same Windows-env. under /home/user/.wine.  
PoL creates a .playonlinux -folder, and with each new Windows-app installed thorough POL a new Windows-env (in a new folder) gets created below .playonlinux.

Each Windows-env. is called a Wineprefix, and using wine you can create any amount and anywhere, but it's more complex using wine. So you'r best of using PlayOnLinux, or CrossOver (not free, but good).

In PoL make sure you install each new game in it's own wineprefix, you can choose that during the install-process. Having different games (that req. different settings to work) in the same prefix can mess up things for you. 
Think of each Wineprefix as yet another Windows-computer, it has C:\ and perhaps other partitions as well. Just like in Windows.


And you'r computer is more than capable to play any of those games you'r listed, old age can be good as the chance is higher that someone has written a PoL-script for it.

Good luck!

### Post by DuckHook on 2013-10-09
Since all of my gaming needs are satisfied by older PC games, I've developed the following strategy to get my fixes:

1. I first try to run every game I can using WINE. I've found that WINE will run about half of my old games well enough that I'm happy. Equivalent to Platinum or Gold on WINEDB. It takes some tweaking, and on some games it takes a lot of tweaking, but you only have to do it once and then it's set. One advantage of old games is that they don't get updated anymore. If you can get it to run, it stays running.

2. Everything else is run in a Windows VM. I've now got a very fast box, but even on my old boxes, this was usually quite acceptable.

3. Until about four years ago, I kept a dual boot partition alive for the sake of four games. Then WINE improved to the point where one game ran seamlessly, and VBox improved enough that two others could run acceptably in a VM. I decided that it wasn't worth the time and trouble to keep a whole Win partition alive for the sake of one game. Haven't missed that game in four years, and haven't missed the AV updates, Malware scans, disk thrashing, bloated overhead, scheduled defrags, BSoDs, blah-blah-blah of Windows either. I *have* enjoyed a simplified HDD structure, rock-solid stability, resource efficiency and pure freedom of Linux. Point is, there's always this third option for those who simply can't let go.

4. I'm beginning to explore Xen. To my knowledge, this is the only VM solution that has properly worked out VGA passthrough. Since graphics is by far the biggest bottleneck to gaming on a VM, a well-behaved VGA passthrough would mean that even the most resource-hungry monster of a game could run at native speeds on a Xen VM. Not only that, but Xen is also capable of running a paravirtualized Linux session. This means that not only would my Ubuntu sessions also run at native speeds concurrently with Windows, but each OS would enjoy the added security and backup advantage that comes with having their snapshots taken at any time I choose. It's almost too good to be true. The one caveat is that this requires some heavy-duty HW, but a gamer has that anyways.
