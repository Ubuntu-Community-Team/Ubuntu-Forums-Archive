### Post by dericke on 2011-01-15
Could anything be accomplished by a collaborative terrain project with a gaming focus? There's plenty of proprietary models, like Google Earth, floating around. It seems that Flight Gear has a complete, free terrain set. And OpenStreetMap already exists for basic maps. But I'd like to see something where any future mapping, by anyone interested in making their data, could be upstreamed into an increasingly-detailed model. I thought of this first as relevant for games. But it could be useful for other projects, as well. There would be crossover with OpenStreetMap, using the data they already have. It would have be scalable - someone might wish to use an entire country on a low-end system and would want lower-resolution data. Someone else might want the interior of a public building. And there would be layers, where historical or fictional versions of a given location would be available. For example, a developer could select France as it is in 2011, or as it was in 1944.

Game makers would have a huge cache of free data available. The data should be available under a modular share alike/GPL style license, requiring derivative map data of a program to be shared, but not other code like engines, non-map textures, sound assets, etc. This would encourage commercial devs to contribute to the project, rather than work on proprietary models.

I’d like to see this project make development easier, especially for ambitious, large-scale games. I’d like to see a FOSS Oregon Trail game made, and see terrain developed for that project appear in Flight Gear. Or maybe a commercial studio is making a WWII strategy game. They grab some base terrain from the project, use some reconnaissance photos from a public archive to improve the detail, time-encode their modifications, and then upstream it. Their unit models, sounds, engine and other code remains proprietary.

Does this idea have merit, or is unneccessary?
