### Post by Jovec on 2007-07-09
Not trying to discourage, but you'll also want a relatively powerful computer and a NVIDIA video card for best results in categories 2 and 3 listed below.

As for details, how about a summary?

1) You'll find plenty of decent native Linux games, many of which install easily into Ubuntu through the repository and the Add/Remove programs tool.  The majority of these games do not compete with commercial games for graphical prowess and scope, but nonetheless can be very fun.  You'll find both a mix of original games and clones of popular Windows games.

2) Some commercial games, especially those based on id software (Doom/Quake) and Epic (Unreal) engines, have native Linux binaries available for download and are virtually identical to their Windows counterparts.  Also Neverwinter Nights 1.

3) Many windows games are playable through WINE, which technically is a port of the Windows APIs to Linux, but it's easier to think of it as an Windows emulator.  Mileage may vary, but many apps/games get good results.  Look at the App DB at [www.winehq.org](www.winehq.org) to see how well a given app/game runs.  Cedega is a branch from WINE that focuses on gaming, and also provides some licensed code to legally use some forms of CD copy protection.  Cedega requires $5/mo sub for the complete feature set.  Results with Cedega vary, some games run better than WINE, some worse.  As a generallrule, games that can use OpenGL rendering and/or less advanced DirectX features work better here.

Edit:  If you are used to just buying the latest games every month and playing them on Windows it's just not going to happen like that on Linux, regardless of what anyone tells you, but you can generally find a few great Linux games per genre that will run either via Wine of with native Linux support.
