Maybe there is a way to get past HDCP Compliance all I Did was an x-server restart.
here is the information on my box which is having some trouble using a dvi chord hooked up to a non hdcp compliant monitor

1gb ram
160gb HD
E4300
8800gts a HDCP Graphics card
I Used Envy to install the graphics adapter
NON-HDCP compliant monitor

Things that make the DVI connection work

1.Regular boot with dvi-vga adapter then wait for it to finish booting then plugin the dvi chord in its place and restart the x-server (ctrl-alt-backspace)
(this is the only way I got ubuntu to work with a dvi chord even before I used envy to install the graphics adapter)
2.Belenix works on first boot
3.So does open-solaris
