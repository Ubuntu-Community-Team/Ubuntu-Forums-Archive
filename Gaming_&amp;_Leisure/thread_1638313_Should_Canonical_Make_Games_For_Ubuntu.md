### Post by Ellipsis on 2010-12-05
I am wondering why Canonical hasn’t started making commercial games for Ubuntu… it is one of the most requested features and there is a huge opportunity to profit from it. 

Nintendo, Sony (now that it has acquired a bunch of small studios) and Microsoft all make games for their own platforms. Even Valve used games like Counter Strike, Portal and Half Life to draw people to Steam.

Canonical could really set a standard to which games for Ubuntu/Linux could be measured. Moreover, Canonical having in house experience building games for Ubuntu could support other companies thinking of porting games.

Should they do it? How should they do it? 

Ubuntu One integration would be great. Other ideas are releasing open source games but with "Premium" content sold (levels, units, etc.) or charging for server access for multiplayer games (like xbox live. Releasing plain proprietary games would also be an option but I am not sure if the community would be happy about that.

Likewise, I do not believe building the "next call of duty" or any extensively large project is a good idea (at first). Something with nice (2D) graphics and built in less than a year would be highly effective if done well.

Moreover, it adds value to the platform especially if the games integrate well into Ubuntu. Likewise, Canonical developing games internally could bring greater attention internally to what challenges face other game developers. 

I realize there are in fact a thousand other things Canonical needs to help fix and improve in Ubuntu. But of these things few have direct ways to generate a profit. Likewise, few of these features and bugs have been demanded as much as making Ubuntu into a proper game platform.
