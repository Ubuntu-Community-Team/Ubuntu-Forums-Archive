### Post by Benjamin_Vedder on 2008-12-10
Hi

A while ago when i was new in linux i wrote something about problems with quake4 since gutsy. And i still have the same problem, and i have confirmed this many times. In fact, it deos not matter on what computer i run the game, its always the same problem. I have tried quake4, doom3 and prey (they all use the idtech4 engine). You can download the prey demo from [http://icculus.org/prey/]("http://icculus.org/prey/").

The problem is: the performance is not good. The framerate jumps between 42 and 60 fps no matter what settings i use, even in the menus. This was not the case with ubuntu 7.04 (kernel 2.6.20). I have even tried this on a friends computer with ubuntu 8.10, he has an core2duo at 3.8 ghz and an overclocked gtx280 graphics card, and the behaviour is the same there.

Today i tried quake4 in wine just for fun on my laptop (i had to use the cracked version), and it runs smooth, better then the native version. So i decided to post about this again.

Try this (if you want to):
1. Download the prey demo from the link i posted above and install it.

2. Make sure desktop effects are disabled. (run "metacity --replace" from the terminal).

3. Start the game and type "com_showfps 1" in the console. (pull down the console with ~).

Your framerate should be rock solid at 60 fps, at least in the menus.

If you can confirm this problem or if you have an solution for it i would really appreciate it.

Thanks in advice.
