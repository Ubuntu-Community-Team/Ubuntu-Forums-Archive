### Post by eiskalt on 2007-08-10
I'm using wine 0.9.42 to run all my windows stuff.  I have all the steam games running, warcraft III, ventrilo, Command and Conquor 3, Macromedia flash 8, and a few nintendo emulators.  Wine is great, they update it usually every 15 days.  If you decide to use wine, do not use the deb's, download the source from winehq, and compile it.  Its a bit more work, like finding packages and stuff in synaptic package manager, but once its all there, and installed; you'll have next to no problems.

### Post by cogadh on 2007-08-10
I dual boot XP/Ubuntu just for the few games that won't work in Wine or DOSBox. As for the games you want to run in Wine, I checked Wine's AppDB ([http://appdb.winehq.org/](http://appdb.winehq.org/)) and found this: 
[LIST]
[*]Planescape won't work with Wine
[*]Sanitarium might, though it appears there may have been a regression in Wine around 0.9.30 that broke it. It hasn't been tested since then
[*]No one has tried to run M:UA with Wine, or at least there is no test data for it on the AppDB, so it might be worth trying
[*]Fallout will definitely work
[*]Starcraft will definitely work
[/LIST]
