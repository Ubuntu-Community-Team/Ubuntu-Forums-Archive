### Post by ubuntu_demon on 2005-06-28
This is the thread for the European Ubuntu Clan

We play free (as in cost) games that run native on Ubuntu (GNU/Linux). As soon as someone is willing to be team leader for a game then [ubu-e] will play that game.

Currently we only play Enemy Territory (ET) and its mod, True Combat: Elite (tc:e). True Combat: Elite is a free tactical shooter which resembles CS and Urban Terror. ET is a WWII-themed, somewhat less tactical shooter based on the quake3 engine.

Clan name : European Ubuntu Clan
Clan tag is : [ubu-e] (The e stands for europe :-P)
Clan irc : #ubu-e @ quakenet
Clan spirit : We are a clan of Ubuntu fans who live in Europe and who want to play tc:e. This is a passive way of promoting Ubuntu.

### Post by Rashid584 on 2006-10-01
OK, so what does everyone think of the game so far?

I definitely like the new textures, even though I have to run on low detail textures, medium geometry (everything else medium, except resolution @ 1024x768 ) Also, I like the new routes they've added a lot, it definitely balances the maps out more (a LOT for some maps). The new railhouse crate room looks AWESOME :D

Some common issues for me were: obj_railhouse and obj_snow gave HunkAllocate fail errors. And all maps had MASSIVE fire fight lag.

Solutions:

Go to Options > View > Wallmark Lifetime and set it to OFF.

goto ~/.etwolf/etmain/profiles/your-profile/etconfig.cfg and find the following lines:

seta com_hunkMegs XX
seta com_zoneMegs XX
seta com_soundMegs XX

Change the values to a higher value, depending on your system.

I have 256MB of RAM, so I set it to: 192, 64 and 64 respectively.

That won't work for everyone, apparently. Try it anyway though, should work. 

You might think the second one already works because your autoexec.cfg changes the values, but they don't. the etconfig overrides everytime you start up.

Hope that helps someone :D

-Rashid
