### Post by leech on 2006-03-09
I don't think I've had the problem with Ut2k4, but I know for the Original Unreal Tournament, if you played it on a system (windows or linux) that has speed stepping (so that your CPU changes speed based upon what you are doing) it'll run faster than midget from a bear if you run it normally.

The problem with that game is that it checks your CPU speed when it first loads up and tries to calculate how fast it should go based on that.  But if your CPU is running at 800mhz initially, then as soon as the video hardware is utilized, kicks up to 2 ghz, you'll have super sonic speeds.  The fix for UT is to have a script start up and do some heavy math right before running UT, so that the CPU kicks into gear and UT can calculate how it's supposed to run.

I am pretty sure they fixed that in UT2k4, but I could be wrong.
