### Post by Mr.Linc on 2007-04-26
Hi,

I've been on Edgy for months, and my Quake 3 worked fine (Linux native version, point-release 1.32).

Now, a few days ago, I did a format and a clean install of Feisty, installed the game again, and everything looks fine except for one thing: mouse doesn't move. Can't move the cursor on the menu screen, and when I load a map, I can move forwards, backwards and to the sides with the arrow keys, but I'm stuck looking always to the same place; I cannot use the mouse to freely look anywhere. I can use the mouse buttons and the wheel just fine, though. But the freelok doesn't work.

Mouse is an optical Genius Netscroll. As I said, it worked fine on any game on Edgy.

The mouse section in my xorg.conf (in case it is needed):
```
Section "InputDevice"
    Identifier     "Configured Mouse"
    Driver         "mouse"
    Option         "CorePointer"
    Option         "Device" "/dev/input/mice"
    Option         "Protocol" "ImPS/2"
    Option         "ZAxisMapping" "4 5"
    Option         "Emulate3Buttons" "false"
EndSection
```
Small note: "Emulate3Buttons" was originally set to "true" and I changed it to "false" to solve the problem of pressing the two buttons at the same time (so I could jump and shot at the same time), although it didn't affect the main issue on either state.

Thank you in advance for your help.
