### Post by eggsome on 2011-11-22
Ok so I lied, we ended up using Cairo rather than OpenGL.
However, when I originally went searching for this I thought that I wanted an OpenGL app, so I've given it that title to help searchers find this post.

Many years ago when I first started playing ADOM on my Win95 PC, I found it nice to run using the DOS fullscreen mode.
I wanted to recapture that feeling, but with the added convenience of being able to switch between workspaces (1 for ADOM, 1 for Guidebook and 1 for Notepad).
This is complicated by the fact that all computers use LCD screens with different resolutions.
In essence what I thought I wanted was a fullscreen OpenGL XTerm, that would stretch the 80x25 chars to whatever the current res was.
Nothing existed to fill this need, so I commissioned someone from ELANCE to do the job.

May I present:[SIZE="3"]
[vga-terminal_0.3.0_i386.deb]("http://www.users.on.net/~kirin_other_acc/vga-terminal_0.3.0_i386.deb")[/SIZE]
And for those with the inclination or interest here is the source code:
[SIZE="3"][vga-terminal_0.3.0_source.tar.gz]("http://www.users.on.net/~kirin_other_acc/vga-terminal_0.3.0_source.tar.gz")[/SIZE]

It features:[LIST][*] Fullscreen operation (stretched to your desktop resolution)[*] Oldskool VGA font for that classic feel[*] Compatibility with workspace switching (CTRL+ALT Right arrow, ect.)[*] Packaged conveniently as a .DEB for Ubuntu.[/LIST]

I've only tested this with Ubuntu 10.10, however the developer tells me that it should work with 11.04 and 11.10 too.
Notwithstanding my irritation with others hosting apps and projects from forums, I think this is the most viable option. Both because it will reach the intended audience and the lack of time I have for maintaining a proper website.
I hope this is useful to a number of people. I know I *love* it.
