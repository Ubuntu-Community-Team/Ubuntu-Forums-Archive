### Post by m0j0j0j0100 on 2008-01-05
You can use a program called WINE which lets you install and run a lot of windows applications, including Steam.  You should be able to download and install the latest version of WINE from the Add/Remove box.
To install Steam, go to steampowered.com and download the SteamInstall.msi file.  Open a terminal, cd to where you downloaded it to, and type:
msiexec /i SteamInstall.msi

### Post by mikedep333 on 2008-01-18
> **sx66gns said:**
> I'm sorry please explain , I'm not sure if I get what your saying.

I know that the Wii games are protected but what does that have to do with FOSS? , I'm not trying to argue or contradict you , I'm Honestly interested in what you have to say.

Consider a Windows XP computer.
You go to install a commercial game, and Securom/Safedisc/Starforce copy protection/restriction drivers are installed in the process (or when you run the game for the 1st time.) They control your usage of the commercial game, but you are free to install any other game you want on it. You can run everything from the proprietary freeware game Wolfenstein: Enemy Territory to the open source game Nexuiz, to a commercial game from a competing company with another DRM system on it. This can happen despite the DRM from the original commercial game being on your computer.

Now consider the Wii.
Not only are official Nintendo-Santioned Wii games protected/restricted, but the console's DRM goes beyond that. As with most consoles, the DRM on it prevents you from being able to run any game or program that does not have the Nintendo DRM on it. So you can write a program/game or port an open source program/game to the system such that is perfectly compiled for the Wii hardware and is completely compliant with the Wii operating system's API , but Nintendo will not let you run it on the system. You, or any other developer, needs Nintendo's approval and license to run a program for the Wii, even if you do not intend to distribute it or commercialize it. Needless to say, the license if applied to even permissively licensed code would make that code proprietary. This approval/license is not easy to obtain too.
It's as if you don't own your Wii at all, and according to Nintendo you do not. You are merely licensing it, just like a proprietary piece of software. Your rights are thus extremely limited.

I myself recognize that the proprietary/commercial model for games leads to more high-quality games than the open-source model. I buy lots of commercial PC games as such. But I strongly object to the way DRM is built into computers such as the Nintendo Wii to turn them into restricted gaming machines that you license rather than own.

This DRM is not merely a threat to gamers. Consider Richard Stallman's essay, The Right to Read. It describes what things may be like in 100 years:
[http://www.gnu.org/philosophy/right-to-read.html](http://www.gnu.org/philosophy/right-to-read.html)
> Dan would eventually find out about the free kernels, even entire free operating systems, that had existed around the turn of the century. But not only were they illegal, like debuggers&#8212;you could not install one if you had one, without knowing your computer's root password. And neither the FBI nor Microsoft Support would tell you that.
...
Author's Note:
The proponents of this scheme have given it names such as &#8220;trusted computing&#8221; and &#8220;palladium&#8221;. We call it &#8220;treacherous computing&#8221;, because the effect is to make your computer obey companies instead of you. This was implemented in 2007 as part of Windows Vista; we expect Apple to do something similar.

And don't forget, that the line between computers and gaming systems is starting to blur as Nintendo Wiis have web browsers.

I fear for the day when all computers are licensed to you from a bunch of colluders that Microsoft is among, and you will truly not be able to own a computer at all.
