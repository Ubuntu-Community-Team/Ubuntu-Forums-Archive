### Post by Nevon on 2010-03-15
I don't know what the case is for more advanced games, but at least for 2d games, learning to make them isn't about reading tutorials or anything like that. What you really need to learn is how to work with graphics and sounds, and learning about how games are generally structured, the rest comes with experience.

I'm currently toying around with something called [Löve](http://www.love2d.org), which is a framework for making cross platform 2d games in Lua. If you're more interested in Python, then there's [Pygame](http://www.pygame.org/). Or, if you're more hardcore, you can always use libraries like SDL (though I'd recommend OpenAL over SDL_mixer for audio) or OpenGL. Those have bindings for most languages.
