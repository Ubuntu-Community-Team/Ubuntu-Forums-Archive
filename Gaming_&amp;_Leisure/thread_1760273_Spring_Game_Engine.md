Well depends on the style of RPG you would like. Second Life is more FPS so I presume you want an FPS style engine. In that case Spring may or may not be the approach for you. Spring does FPS, not sure how well but it does. If you wish to test it download Gundum (which is a mod for the spring engine). You can flick between overhead view or fps via the "C" command (and some others, check keyboard mappings). FPS mode works quite well, but not sure how suitable it would be for a RPG, though I'm sure you could experiment.

Maybe post a question on the Spring forum. They do use LUA for scripting quite a bit now within spring. About the only thing you wouldn't be able to do that I can forsee is enclosed environments. You cannot do tunnels or bridges per say, but you could in theory render a roof via the ceiling environment. just dont know how that would work out.
