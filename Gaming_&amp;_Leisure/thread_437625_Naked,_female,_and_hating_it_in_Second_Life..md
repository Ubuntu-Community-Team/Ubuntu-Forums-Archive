### Post by eye208 on 2007-05-23
It's a server-side bug that happened during account creation. It's unrelated to the Linux viewer. The appearance option was probably greyed out because there were no clothing items in the inventory to be edited. A possible workaround would be to grab a fresh set of clothes from the library->clothing folder and drag it onto the avatar.

In fact the Linux viewer, although officially considered at alpha stage, has proven to be more stable than the Windows viewer many times. Crashes usually happen because of faulty OpenGL implementations. The Linux version of the viewer allows to control the usage of OpenGL extensions by blacklisting each of them separately. That way you can make Second Life run even with video adapters that are officially listed as unsupported. See the viewer startup script for details.
