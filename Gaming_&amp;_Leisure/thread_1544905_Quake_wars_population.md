The gameplay is different to Quake Live, which is a reimplementation of Quake 3: Arena in your browser. From a pure Linux games market perspective, it's best to invest a game of Quake Live once a month on Linux, as Carmack mentioned the stats for Linux users in Quake Live can give id a nice indication of Linux usage, thus bringing more id games to Linux. They count players who have logged in at least once a month.

So, even casual Linux players can play once a month, and this sends a message to id that Linux users are willing to support future id software games on Linux. Quake Wars, not so much. If you prefer Quake Wars, then play that. It's up to you:D They both rock 100%

Regardless of the stats, I am pretty sure Rage for Linux is on the way, as TTIMO/Carmack states the code quality advantages from porting to Linux justify the ports, regardless of the financial projections. Basically, they do it because of sound, technical authorship. Very Clean code I suspect when it comes to id code. It's interesting to note the Linux ports of id games came before the Mac ports for the last few years.
