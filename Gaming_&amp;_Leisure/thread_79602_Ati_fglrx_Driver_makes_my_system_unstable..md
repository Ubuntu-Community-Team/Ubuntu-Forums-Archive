To fix this, I replaced /etc/X11/xorg.conf with a backup I had made. My system is stable again, but I have no 3D acceleration and I'd like to play games on it.

### Post by jecos on 2005-10-20
Post your X.org log..
Install linux-restricted-modules , usually its wise to have them with the driver.. 
Also run fglrxconfig and it will supply an xorg.conf with lots of driver options, thats most likely what it needs.

Theres also an open source driver for r200's incase fglrx won't work this time around..

### Post by jprupp on 2006-11-14
I think the correct answer to this question would be:

The ATI fglrx driver is a proprietary driver. It's source code isn't available to the open source and free software communities.

This driver is particularly unstable, and it runs at kernel level, so any problem can affect the kernel and other subsystems, very much like what happens with Windows. This driver can hard-lock an entire computer.

I recommend against using this driver in a production machine that is experiencing stability problems with it. Just use the open source ati driver that is enabled by default.

If you definitely need 3D acceleration or if you would like to test Xgl and Compiz/Beryl, you might need to install this driver, or replace your video card with an Intel-based one, that includes an open source driver that is much more stable.

Nvidia cards also have a proprietary driver, but it works much better than ATI's.

There is a 3D accelerated driver for some ATI Radeon boards that is called radeon. It could possibly work with your card. Chances are that it is enabled by default in a Ubuntu installation, so if your card is not accelerated, it might mean that it is not compatible with the open source radeon driver.
