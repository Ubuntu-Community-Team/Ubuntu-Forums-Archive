### Post by perillux on 2008-08-11
I sent a message to the above poster as he requested detailing where we stand on the current major issue.  I'll copy it below for all to see:  (NOTE:  DT is "Defense Turret" the anti-cheat program that most online servers require you to run)

> Hey, you posted on my thread at Ubuntu forums:
[http://ubuntuforums.org/showthread.php?t=709780&page=4](http://ubuntuforums.org/showthread.php?t=709780&page=4)

ok... I did a fresh install of the tribes 2 on your site.


So here is where we stand on this issue as I am so far aware:  (sorry to
over-explain things :) )

+ DT (Defense Turret) itself works just fine.

+ tribes2 doesn't work with the newest libraries that are on most Linux
distros.

+ So tribes2 needs to be supplied with the older libraries
instead.  &#65279;(&#65279;not DT, but it won't hurt DT to do so anyway).  So far we
have discovered 1 solution and 1 semi-solution.

+ The absolute solution is to overwrite your newer libraries with the
older ones.  However this will cause all applications on your system to
use those older libraries...  might cause problems, and seems a little
more risky but it should completely fix the current issue.

+ The semi-solution is to make a folder called "Loki_Compat" (or
whatever) and put the required old libraries into it.  Then when you run
the game from the command line you tell it to load those Loki_Compat
libraries.  The problem with this though is that I can tell tribes2 to
load the libraries from the terminal and tribes2 runs.  BUT, if I start
the game with DT there is a problem.  As I'm sure you know, once DT is
started it then calls tribes2 to run.  The problem is that although I
told DT to use the old libraries DT itself does not tell tribes2 to use
the old libraries.  So we end up with DT starting with the Loki_Compat
libs and the actual game tribes2 still uses the newer libs.  and I get
the error we were discussing on my thread:

&#65279;&#65279;tribes2: ../../src/xcb_lock.c:77: _XGetXCBBuffer: Assertion `((int)
((xcb_req) - (dpy->request)) >= 0)' failed.
Failed assertion in cleanup -- aborting
Unable to execute loki_qagent - exiting




So using the above solution (semi-solution) you can run tribes2 without
changing any system files, but you cannot (so far) run tribes2 with DT.
If you want to I can find the link to download the Loki_Compat files and
tell you how (in detail) to do the second solution if you want.  Or I
could send you the libs.
Also, even though I can run Tribes2 without DT I still have another bug
in T2 that prevents me from playing.  I described it on the second post
on that link I gave you at the top.

I really hope we can get this sorted out!
