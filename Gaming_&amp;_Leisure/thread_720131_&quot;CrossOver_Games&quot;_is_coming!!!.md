### Post by handy on 2008-03-16
**Here is a copy of a post in the closed forum at Codeweavers from Stefan Dosinger, a dev' who has done so much for us:**

[I]The main difference between CrossOver Games and regular CrossOver will be the release cycles. Crossover has a rather long release cycle, with a lot of testing for stability, etc.

This imposed problems among games. The game support in Wine is improving rapidly, for a big part due to my work on Direct3D, and our customers had no way to get access to those new features because game and office support had to be synchronized in one release. So regular Crossover will still run the games, but Crossover games will be more bleeding edge.

For CrossOver Games we're also considering offering unsupported builds made from the nightlies if some new feature was added to the code, to give it to those users who want to use them. For example, I am currently working on some performance improvements for Guild Wars(not much, maybe 5-10% faster, or even less), which will not be included in the initial cxgames release because the work is not finished yet. I might make this available shortly after the cxgames release as an unsupported build.

[/I]
