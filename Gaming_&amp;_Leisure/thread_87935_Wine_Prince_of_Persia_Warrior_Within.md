### Post by indulis on 2006-01-07
This is the game designer trying to stop the game crackers from looking at the game as it runs, and figuring out how to change how it installs and/or runs by uding a debugger (this lets you single step the lines of code and see what they are doing).  The game detects that that there is something else running "underneath it", not straight hardware+Windows, and refuses to run.  Of course you're not running a debugger, you're running Wine which is a Windows simulator, but it is detecting that and saying "nope".

Strangely enough, you may be forced to use  a "cracked" version of the game you legitimately bought, just 'cos you want to run it on Linux not Windows.  So, by a strange and twisting path, the game designers have made you dependent on the game crackers by trying to stop the game crackers.  Ironical, huh?
