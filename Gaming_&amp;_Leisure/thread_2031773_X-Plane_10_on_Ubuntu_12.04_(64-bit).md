### Post by MdMax on 2012-08-19
Hello !

*1. Is there any reason to avoid going 64-bit (Ubuntu 12.04) for X-Plane 10 ?*

I see a lot of reasons to avoid a 32-bit operating system for X-Plane. As you said, X-Plane 10, depending on your settings, will need a lot of memory address space, and future versions of X-Plane ([version 10.20]("http://developer.x-plane.com/2012/06/were-still-here-really/")) will use the 64-bit architecture.

For now, with the 64-bit version of Precise Pangolin, you only need to install this package: ia32-libs-multiarch:i386

But there's one tool, the Overlay Editor for editing X-Plane sceneries, that uses the 32-bit version of the python-opengl library. If you need this program, you can run it in a 32-bit virtual machine, or use the Windows version with Wine.

*4. In Windows, an X-Plane installation is totally self-contained in its own folder (directory), independent of any other Windows folder and folder hierarchy location.*

It's the same in the GNU/Linux version.

*5. Does X-Plane management (in Ubuntu) differ much (versus Windows) as far as adding new aircraft and scenery ?*

It's the same in Windows and GNU/Linux. But folders and filenames are case-sensitive. If you're using a plug-in that can't find a file, the case of the filename may be the problem, just read your Log.txt file.

*6. Most subject matter experts advocate using Nvidia GTX GPUs with Ubuntu, as opposed to AMD Radeon HD GPUs; because OpenGL Drivers from Nvidia deliver superior (and more bug-free) performance. Would anyone want to confirm or deny this ?*

I'm happy with my GeForce 8800GT and the proprietary Nvidia driver from this PPA:
[https://launchpad.net/~ubuntu-x-swat/+archive/x-updates/?field.series_filter=precise](https://launchpad.net/~ubuntu-x-swat/+archive/x-updates/?field.series_filter=precise)

*7. How much Terminal Command Line expertise is needed for the care and feeding of X-Plane on Ubuntu ?*

This can be helpful if you want to understand why a plugin or a plane add-on that has just been released does not work on your system. The [ldd command]("http://manpages.ubuntu.com/manpages/hardy/man1/ldd.1.html") can show you the libraries you may have to install for an add-on, and you have to know where and how to install 32-bit libraries on your 64-bit OS. But most of the time, you don't need this, or you just have to ask for support, here or on the X-Plane.org Linux forum, and copy/paste commands.

Happy flying.
