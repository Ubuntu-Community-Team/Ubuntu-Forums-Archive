I just wanted to take a second to let you know about the awesome, open-source, cross-platform, game development framework that is Löve. I just recently tried it out, and it really makes developing 2D games so much simpler! 

[img]http://www.blastfromthepast.se/blabbermouth/wp-content/uploads/2009/09/screenshot_001-500x387.png[/img]

As an example, I created a soccer styled pong game in little over a day:
[img]http://www.blastfromthepast.se/blabbermouth/wp-content/uploads/2009/09/screenshot_0011-500x387.png[/img]

It's not like those game makers that just allow you to create cookie-cutter games by pressing a bunch of buttons and dragging some boxes on the screen. Löve does require you to actually code things. What it does is that it simplifies things such as displaying graphics, playing sounds and music, write to the file system, model physics, etc. to make the development much easier and faster. It's very much like Pygame in that sense.

If you are interested in reading more about Löve, you can read and article I wrote about it and some of the games that have been made (or are being made) using it:
[http://www.blastfromthepast.se/blabbermouth/2009/09/game-development-with-love/#more-718](http://www.blastfromthepast.se/blabbermouth/2009/09/game-development-with-love/#more-718)

Or you can go straight to the Löve homepage:
[http://love2d.org/](http://love2d.org/)

If you've made any games using Löve, I would love to hear about them (or even better, play them). It could also be interesting to hear about your experiences with frameworks such as these. Are the necessary? Do they help at all? Do you think Pygame is much better? If so, why?
