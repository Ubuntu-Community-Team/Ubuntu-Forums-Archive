Usually a midrange graphic adapter requires most likely 120-150W (with extra PCIe power connector).

>

Depends on GPU clock, GPU cooler design, air troughput in your case and so on. 

> 
Would an AMD GPU be better? (There support has improved hugely over the last few months; on my laptop anyway)

To my best understanding for the native Linux blob both AMD and NVidia play nicely if you have a recent card. For open source currently ATI is the leader of the pack.
NVidia is "know to be more stressfree" in regard to installation, WINE compatibility and so on. YMMV

> 
Are there discreet GPU's better suited to my requirements?

Clarify this question !

>

If you get blue spray paint this is subject of change *ggg*

In the end lets stop the joking:
In the end no integrated GPU will be able to run games similar to Skyrim, Crysis 2, etc as fast as a midrange card of the current GPU familiy. I can hardly imagine that this changes soon. Most midrange cards (thats around 100€ for me) do not produce immense amount of heat (often there are passive variations). They are able to run medium demanding games at resolutions up to 1280x1024 (if you can live with medium detail level). For the latest top notch title in 1080p at maximum details you always need a beefy GPU which eats cheap power supplies for breakfast and produce huge amount of heat.
