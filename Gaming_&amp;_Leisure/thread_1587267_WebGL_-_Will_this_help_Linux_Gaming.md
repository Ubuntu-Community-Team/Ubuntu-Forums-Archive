### Post by naikon89 on 2010-10-03
Just tried out multiple web WebGL demos last night. Quake 2 is technically the most advanced web game so far(even though it's java based - blech!)
and this is leading  me to speculate that Web gaming(not talking about Onlive type services, though they may gain steam too)will really play an important
role in gaming for Linux in the coming years. The mere idea that a web app has 3d accel through markup is a bit crazy tbh. I don't think we will be seeing 
the most technically advanced games being ported, but I do see the possibility of certain fps games/puzzle games coming to the web. Think about it, the 
web is suppossed to be one the most platform agnostic tech out there. 

In the true form of the Internet and all it's associated protocols, I think the future is not in waiting for big companies to release ports which require future 
fixes, but a t controlled entity to whichanyone having web access can play. It won't matter anymore what OS you are using. There will always be a need
for native local ports I think, but it will become less important as time advances.

Thoughts welcome:)
