### Post by Sofox on 2012-04-10
There's been a bit of talk about Game Engines, so I figure we should make a list of Open Source ones for easy reference (thanks [juancarlospaco]("http://ubuntuforums.org/member.php?u=784640") for suggested additions):

[Panda3D]("http://www.panda3d.org/")
3D engine using Python and with web plug-ins. Good for fast development and used by Disney for various games.

[Darkplaces Engine]("http://planetquake.gamespy.com/View.php?view=Quake.Detail&id=342#Files")
Modified id  engine with advanced features, used in Steel Storm.

[ioQuake3]("http://www.ioquake3.org")
Advanced version of Quake 3 engine

[Cube Engine]("http://cubeengine.com/")
FPS Engine with in game Level Editing

[Irrlicht Engine]("http://icculus.org/twilight/darkplaces/")
Provides a set of tools and utilities for making 3d games

[Blender Game Engine]("http://www.blender.org")
Integrated into the Blender 3d modeler. Allows for fast building of games instantly using assets created in Blender. Games can be build using "logic blocks" or Python code.

[Ogre3d]("http://www.ogre3d.org/")
Powerful for game graphics, but not much else. Used in various commercial games.

[ioDoom3]("http://www.iodoom3.org/")
Open source Doom 3 engine, supported by the guys behind ioQuake3

On the 2D side of things:

[LÖVE]("https://love2d.org/")
Allows Lua scripts to make games. Used for Not Tetris 2 and [Mari0]("http://stabyourself.net/mari0")

[Pygame]("http://www.pygame.org/news.html")
Python libraries to assist game making via SDL.


Incomplete list, but I wanted to hit the main ones. Any suggestions?

### Post by Sofox on 2012-04-11
Hey Arthur, nice list.

I'm actually finding making this list rather difficult. I obviously don't want to put down every open source game engine out there, as the idea of this list is to provide a quick list of the important OS engines for handy reference. It's hard to keep an objective standard as to what engines should be on this list, it's too easy to let my personal bias come into play.

I want to keep the engines to ones that are (relatively) easy to develop for, mature, well supported, and preferably have one more more solid games made using it that demonstrates its worth.

If you feel an engine I've mentioned fails to meet that standard, or I'm missing an engine that does. Let me know, and we'll try to determine what selection of engines to put down.

Juan, I didn't put down the SilverBack Engine because it seems like an engine for a single game rather than something for general game development. Prove me wrong if you can. Also, I didn't put down the Pilas engine because while it seems cool and interesting, I can't find enough info on the engine or see if it has been used enough to be stable and reliable.
