Hi,
I am a cricket lover and Linux enthsiast (though not a programmer).
My son is also a cricket enthusiast, and I'm introducing him to the world of cricket scoring, especially as a way of following a match. (He also likes to stage fictional matches in his bedroom involving a kind of subbuteo-style game! But this requires keeping score too.)

As far as I can make out, there is CURRENTLY no Linux software for cricket scoring.
(In fact, even for PC / Mac, there is virtually nothing except quite expensive commercial software.)

BUT there used to be quite a nice-looking program called kricketscoreboard.
[http://kricketsb.sourceforge.net/](http://kricketsb.sourceforge.net/)
It looked great, with excellent functionality, but still keeping things fairly simple. Perfect.

Its last development activity seems to have taken place about a decade ago, but it seemed pretty functional at that point.
It seems to have been written in C++, with XML data storage.

The old source code won't compile, because it's depending on ancient versions of kdelibs and Qt, and maybe other stuff too.

Given the popularity of cricket in places that have a lot of programmers and Linux enthusiasts (e.g. India, Pakistan), I am hoping there might be some prospect of getting the software resurrected so that I could compile and run.

How could one set about attracting attention to the project and seeing if it could be resurrected?
Any ideas / suggestions?

If it was done somehow in a way that was platform-neutral, it could be a great resource for cash-strapped cricket clubs around the world!
And it could be fun for people like me and my son who enjoy using scoring as a way of following a cricket match.

All help much appreciated.
Cheers, Jamie
