### Post by FrozenFox on 2008-03-30
Your card (from what I understand, I own an older card) is not supported by the driver version from the restricted drivers manager. Please use [http://www.albertomilone.com/nvidia_scripts1.html](http://www.albertomilone.com/nvidia_scripts1.html)   envy to install the newer nvidia drivers instead. It is pretty straight-forward: download it, run the .deb to install it, and then run envy to pick your drivers and let it do its thing. If you have problems, please post back.

WARNING: If you install using envy, it is not "officially supported". As such, it may not work or may do strange things to your ubuntu partition. Proceed with caution (though it's usually not problematic). Also, if you install that way, you MUST uninstall the drivers using envy before you update CRITICAL system pieces like xorg-server or the linux kernel,  linux modules, so forth, then reinstall them after your system update. I would HIGHLY recommend you fully update your system using the update manager before you use envy. Failure to follow this procedure prior to updating critical system components will result in a system that will not fully boot (it will drop you to the command line, and you will have to use envy from the command line to reinstall the drivers). Note that in 8.04 Hardy and above, xorg will boot into graphical mode despite problems if you give it a minute, but will be limited to 800x600 or so and will obviously not be in correct working order. You'd just need to install the drivers via envy again. In 7.10 Gutsy and below, you'll just be dropped to command line as previously stated.

### Post by forrestcupp on 2008-03-31
When you tried Envy, did you uninstall the old drivers first?  Envy has the option to automatically uninstall them for you.  Then you can use Envy to install the latest drivers.  I'm using a geforce 8600 with Envy installed drivers, and it works perfectly.

Also, if you don't have the Universe and Multiverse repositories enabled, Envy won't work.  Go to System->Administration->Software Sources and in the 'Ubuntu Software' tab, make sure all boxes are checked except the CD box.  Then type in a terminal
```
sudo apt-get update
```
Then try Envy again.

Edit: 
Also, the new drivers won't take effect until you restart.

### Post by wolterh on 2009-02-01
Wow, I finally got it fiex too! =D Hey ario, now you don't have to put the guide, I will.

First of all, navigate into [http://www.libsdl.org](http://www.libsdl.org) and download the latest stable source-code. This is all you need to download.

Afterwards, follow this guide, which I summarized from [http://cone3d.gamedev.net/cgi-bin/index.pl?page=tutorials/ogladv/tut1b](http://cone3d.gamedev.net/cgi-bin/index.pl?page=tutorials/ogladv/tut1b) in case the page was moved or something, making it non-available.
```

1. Decompress the downloaded source code file
2. Navigate into it's folder (where you uncompressed it)

3. run $ ./configure
4. run $ make
5. run $ sudo make install

# I recommend you to now open two nautilus windows with root privilege (sudo nautilus). If you use this nautilus method, then you just have to drag and drop files
6. Copy from /usr/local/lib/ all the SDL relative files into /usr/lib/
7. Copy /usr/local/include/SDL into /usr/include/ (You will have to merge the folders, replacing all the old files with the new ones)

```

I did a restart, but perhaps a relog will do. Next time you login, you will be able to use your opengl!
