### Post by flowtron on 2008-07-16
Why don't you go with a problem like this to the <a href="http://assault.cubers.net/forum/">official forum</a> first?
And - especially if you _did_ install from source (which I'm currently reading as "I got it from the sf.net-site" ..) instead of installing some repository-deb that's usually hopelessy out-of-date and installs itself all over the place so the developers can't support it anymore *cough*.

Anyway.
The issue with the textures usually arises if you don't run the game-executable from the base folder .. some people try changing into bin_unix ... Don't ..
If you really did simply unpack the sf.net-package then just change into the AssaultCube-folder and enter "./assaultcube.sh" .. that should eliminate any and all error-messages.

For further help - go to the official forum for AssaultCube.
