### Post by Tony Flury on 2009-08-04
Firstly, the games producers produce games mainly for Windows because Windows has a massive market share, and Windows is seen as a gamers platform - amongst other things. Also Windows is relatively easy to develop for (not saying that linux isn't). Finally most people don't see Linux as the platform of choice for gamers (partially at least due to the perceived lack of games).

Essentially the cost of doing the development for a given game must be outweighed by the revenue gained by selling it - I don't think that the linux gamer market is big enough (nor will it be soon) to justify the costs of development (or even porting an existing game). Note that some producers don't even provide Mac versions of their games.

At the end of the day if you have linux - and you want to run games you have a number of choices : 
[LIST]
[*]Use Wine
[*]Dual Boot
[*]Use a virtual machine (although 3d struggles here at the moment)
[/LIST] 

The biggest incentive (I would guess) would be a far bigger market share - i.e. people running linux on its own without dual booting - and most people who want to play games wont run linux on its own (cos the games wont run).

How do we drive up market share of Linux amongst Gamers - maybe someone could produce a block busting awe inspiring game - with all the advertising etc but which only runs on Linux - but that costs money (for most games we are talking multi-million dollars), and would probably flop (due to lack of a market).

### Post by Ferrat on 2009-08-05
Well the biggest problem according to many bigger companies isn't the lack of users or market shares, many developers use linux in their development for servers and some even do beta clients which are Linux but never moved to retail. 
THe biggest problems that is a fairly uphill battle for Linux is the fact that it's open source and develop at a much faster rate than other OS:s because of it, also Linux develop forward and even though backwards compatibility now is fairly good, in general it's not really taken in to consideration when developing under Linux, also since it's more a "living" thing adapting to the new demands and possibilities rather than sticking to one standard that runs through out game developers can't support a Linux version (unless they did something like only support one distribution and even then one release might break game play and the fix for this breaks game play on the earlier version and so on). 

With this in mind and a smaller group (no as insignificant as some make it sound) the work needed for a Linux port added to the problem with supporting it and making sure it works and so on is too much for most companies. 

Most retail game like IDs game that have Linux port give them out totally unsupported with no guaranties but Linux "standards" are moving forward looking at developers like S2 games (Savage, Savage 2 and now HoN) give out a Linux client with some support, however it's more than once during updates etc that their client has been broken and needed a fix, luckily they have a great fixer for Linux but still it breaks some times.
S2 use VOIP in their games for ex. which is great but since many Linux system use different ways for sound, pulse-audio, ES, OSS, ALSA and then on top of that some distros use completely different paths and versions etc getting this to work is some times more than a challenge.
Linux as a base for gaming is there for a tough sell to most companies that doesn't have a lead dev or someone sitting and pushing it through which is sad but as Linux grows and some distros like Ubuntu start reaching the mainstream users I hope there will materialise some what of a "standard", not a linux wide one because that would work against all the good things about Linux being open source but something that the major distributions all had in common as a base which made a program developed for red hat work just as well on debian systems or suse or any other without tweaking needed.

Edit:
DirectX and OpenGL isn't the same thing, also according to many OpenGL 3.2 is easier to work with than D3D under DX10.1, still DX is the dominate API for games but many games, specially bigger ones still have OpenGL, alternative sound and so on since for ex. Mac doesn't have DX either.
