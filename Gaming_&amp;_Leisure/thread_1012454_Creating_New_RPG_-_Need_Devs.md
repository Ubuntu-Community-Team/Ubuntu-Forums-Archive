### Post by farrell2k on 2008-12-16
Holy noobness, Batman!

Your plan, as is,  is doomed to failure.   You're never going to attract anyone without a half decent plan and some code that at least does something.

O.K.  I know it'll be an rpg.  That's cool.   

1.  You need an idea for a story and game world.
2.  You then need to develop a decent story with your protagonist, antagonist, and supporting characters.
3.  After you have something cohesive, you need to work on a script that actually makes sense, while playing out the story on paper.

When you have all of these, you then need to decide whether it'll be 2D or 3D.  You obviously have no programing experience, so I will suggest that you not make the mistake of jumping straight to 3d.   Actually, I recommend learning a language, first.  

I recommend a top-down, 2D game done with either Python and pygame or Java2D, as they're going to be the easiest.

When you have that, write some code to get yourself started.  

This will obviously be a learning experience for everyone involved, so I'll only be interested in contributing if you can show me that others have basic game writing skills like:  game loop, movement, and collision detection basics.

Good luck.  I look forward to seeing if this actually develops into more than an idea.
