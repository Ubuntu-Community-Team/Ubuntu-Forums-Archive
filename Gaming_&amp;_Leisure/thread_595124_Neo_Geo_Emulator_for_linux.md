### Post by FranMichaels on 2009-05-14
> **BeBoBli said:**
> Console based? Why does that even matter? Most people are coming from a Windows background and games being from whatever platform shouldn't matter.

And MAME is not based on home console hardware. Hence the abbreviation making up the name: Multiple Arcade Machine Emulator

This thread is from last year... But anyway.

Console based in this case is referring to Command line interface. 
[http://en.wikipedia.org/wiki/Console](http://en.wikipedia.org/wiki/Console)

The MAME executable or binary, takes command line arguments. With that context, it is console based. One can just use a front-end, such as [gelide]("http://gelide.sourceforge.net/index.php?sect=screenshots&lang=en") if one loves pointing and clicking.

Lastly, MAME supports a plethora of arcade machines, and guess what, it does support some NES (home console) hardware with play choice 10.
[http://en.wikipedia.org/wiki/PlayChoice-10](http://en.wikipedia.org/wiki/PlayChoice-10)

So don't sell MAME short.
