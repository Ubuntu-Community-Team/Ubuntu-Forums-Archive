yeah, its alright. Run the following code in a terminal, and post the log file that appears on your desktop.

```
cd /opt/steelstorm-episode2; ./steelstorm > ~/Desktop/steelstorm-log.txt & sleep 10; killall steelstorm; exit
```

this just changes to the directory where steelstorm is stored, runs steel storm and outputs all the terminal text to a file on your desktop, it waits ten seconds so that steel storm will give us the data we need, then it kills it and closes the terminal. All you have to do is paste that into a terminal and it'll do the rest. That should contain some useful information about what's going on with the audio, if not, that'll be even weirder.
