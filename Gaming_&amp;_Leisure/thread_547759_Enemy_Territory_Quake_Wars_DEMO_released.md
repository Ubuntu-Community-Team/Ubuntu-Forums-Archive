### Post by eye208 on 2007-10-17
> **Tux0r said:**
> Because he wants to actually install the game? :P
This is not Windows. You can install anything anywhere. Running random executables as root is the best way to hose your system. Package managers and code signing exist for a reason.

This is the proper way to install a game that does not come as a .deb package:

[LIST=1]
[*]Run the installer as normal user and point it to a target folder in your $HOME.
[*]When the installer has finished, run the game to see if it works.
[*]If it does, "sudo cp" the game folder to /usr/local, /opt or whatever. You could even turn it into a .deb package at this point.
[/LIST]

Step 3 is optional and only required if several users on the machine should be able to access the game.

Rule of thumb: _Never_ run untrusted code as root unless you absolutely need to do so.

```

```

The "./" is important.

Full explanation: Binary files will only run if they are in your $PATH global variable.  You can see this with:

```
$ echo $PATH
```

If the directory containing the binary is not in your PATH then you need to give the path to that binary when calling it -- in this case the current directory "./".

2 ways to fix this:

1. Add the current directory "." to your path.  This way you can run programs from whichever directory you are currently in without appending the "./"

2. Adding /usr/local/games/etqw to your PATH.  Then you can just run "etqw" from wherever.

To do this, edit /home/yourname/.bashrc and add this to the very bottom:

[code]
export PATH=.:$PATH
[code]

or

[code]
export PATH=/usr/local/games/etqw:$PATH
[code]

This will take effect next time you open a terminal window.

### Post by eye208 on 2007-10-25
"Official" trackers can be hacked just like any other. In fact it's much easier to manipulate a torrent because the .torrent file is just a few KBs large. By replacing it with his own one, a hacker can make himself a seeder of a modified installer file, no matter how large. All the leechers will happily download the code from him and upload it to others without knowing.

There have been incidents of trusted parties distributing malicious code by mistake. I quoted some examples here, but someone marked my post as spam because he did not quite get it. Anyway, I'm not taking issue with how you, Tux0r, administrate your own machine. In fact I couldn't care less.

But telling newbies that "sudo <installer>" is the proper way to install things, that's what I'm taking an issue with. If this turns into common practice, it will be an invitation to script kiddies. Installers on Linux are really just self-extracting archives. There is NO REASON AT ALL to run them with root privileges. Unlike Windows, Linux has no registry where COM DLLs or uninstallers or file types or protocol handlers or context menu extensions have to be registered. Yellowbelly, that sounds like the conflict with Desktop Effects (Compiz) that myself and others have mentioned. Disable these before playing (there'sa script in the thread in my sig for doing this automatically).

To enable switching around while the game is running, Open a terminal and do this:
```

gedit .etqw/base/autoexec.cfg

```
(the path will be .etqwcl/base for the full version)

Put this line in:
```

com_useFastVidRestart 1

```

Save, exit, and run ETQW. You should be able to press ALT+Enter to switch into windowed mode. From there you can do your desktop stuff, then come back and press ALT+Enter to go back to fullscreen

Caution though, this was disabled by default because certain graphic drivers don't handle it gracefully. So, "your mileage may vary".

### Post by william_nbg on 2007-11-21
Hey folks!

I've seen a lot of complaining here about compiz and gaming.

Why don't you simply make your own start up script to start the game - and then change your menu launcher to your script. A script which shuts down compiz and restarts it after the game.

My script:

> #! /bin/bash

metacity --replace &
sleep 2
/home/name/.etqw/./etqw &&
sleep 2
compiz --replace &

Stick this in a text editor, change the paths with your path-to-game, and name it whatever.sh, make it executable.
