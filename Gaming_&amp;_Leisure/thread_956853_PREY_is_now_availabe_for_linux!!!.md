cp ~/prey-demo/libSDL-1.2.so.0 ./ .... since you never left the ~/prey demo you can copy the libSDL-1.2.so.0 from the ~/ or /home/chriscri/prey-demo directory to the current directory (remember ~/prey) by putting this ./ which represents the current directory.

### Post by PoHandle on 2008-12-11
Regarding the sound issue, the export SDL_AUDIODRIVER=esd command worked wonders!  I was having trouble getting the game to run from the menu, but googled up a solution, so my launch command in System>Preferences>Main Menu looks like this:

```
sh -c 'export SDL_AUDIODRIVER=esd && playgame /usr/local/games/prey/prey'
```

This allows you to launch the game without that pesky child process error.

The **playgame** command is a custom script I read somewhere that runs the game under metacity so compiz doesn't make it look ugly.

If you run compiz and don't have the script, I highly recommend you use it for convenience.

Just do the following:
```
gksu gedit /usr/local/bin/playgame
```
and copy and paste the following into gedit:
```
#!/bin/bash

if [ $# -lt 1 ]; then
	echo "Usage: playgame game_name game_options" 1>&2
	exit
fi

# Replace compiz with metacity to prevent issues
metacity --replace &

# Run game
$@

# Replace metacity with compiz
compiz --replace
```

Save and quit, then:
```
sudo chmod +x /usr/local/bin/playgame
```
To run any game that compiz doesn't like, just run the following:
```
playgame */path/to/game*
```

Hope this helps.

### Post by Macubus on 2008-12-27
I'm not sure if this has been addressed before, but you can likely fix the sound issues for the native game as follows (at least it worked for me)..

First, ensure you have libopenal installed..

To do this, type **sudo apt-get install libopenal1**

It will either ask you to install it, or it will say it is already installed.

Next, cd into your prey directory (for me I installed in /usr/local/games/prey so to do this I typed **cd /usr/local/games/prey**) and type **sudo ln -s /usr/lib/libopenal.so.1 ./libopenal.so**

Next, start the game..  the sound will still be choppy.  You next need to go to the sound options and switch the sound from "default" to "openal"

After this the sound should improve greatly.

Hope this helps.

### Post by jhansonxi on 2010-02-06
I was able to get Prey working on Ubuntu 9.10 Karmic Koala x86_64.  I renamed the included libs and made symlinks to the system libs.
/usr/local/games/prey/openal.so -> /usr/lib32/libopenal.so
/usr/local/games/prey/libstdc++.so.5 -> /usr/lib32/libstdc++.so.5
/usr/local/games/prey/libSDL-1.2.so.0 -> /usr/lib32/libSDL-1.2.so.0

Command line:
env SDL_AUDIODRIVER=alsa /usr/local/games/prey/prey

Set audio to OpenAL.  If it hangs when you try to change it then edit the config file directly:
~/.prey/base/preyconfig.cfg

Change/add:
seta s_useOpenAL "1"

OpenAL also **[needs to be adjusted]("https://bugs.launchpad.net/ubuntu/+source/pulseaudio/+bug/356206")**:

echo 'drivers = oss' >~/.alsoftrc
echo 'refresh = 2048' >>~/.alsoftrc


Sound drops out during level load but works during play.
