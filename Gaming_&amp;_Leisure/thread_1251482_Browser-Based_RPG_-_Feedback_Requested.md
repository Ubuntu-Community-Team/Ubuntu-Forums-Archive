### Post by Bios Element on 2009-08-27
Hello there,

I've been toying with the concept of starting a python+turbogears based RPG and would appreciate some constructive feedback on my ideas.

For the basics, The game would be browser-based with the ability to both play online and download an offline 'sandbox' game. This allows it to be both a MMORPG and RPG while still using the same codebase. Major advantages include the ability to play offline, create single player games and accomplish development work without requiring an Internet connection.

The reasoning behind browser-based is quite simple. It puts the game within reach of low-spec systems and vastly simplifies development time. Also because python is somewhat simpler to learn it will be easier for others to contribute.

Core gameplay will be to encourage user-creation and will have no "classes" or smiler systems. Players will be able to own cities, drive vehicles, fly to other planets, etc. I hope to create a mesh of both fantasy/realistic/sci-fi while maintaining a believable balance.

Most important of all, Roleplay will be absoutly central to the game. Players will be able to create a world togeather with other players. Ideally there will be no need to force anything besides what the players desire.

As the game is open-source there is always the ability to reward groups of players who prove they are capable with things such as a hardcoded government. Please note however that the code would only be a way to simplify things for the players, not to give players things for free. The administration will work to enable the players to do anything within Reason.


Any thoughts or suggestions are greatly welcome.

Thanks for reading.
