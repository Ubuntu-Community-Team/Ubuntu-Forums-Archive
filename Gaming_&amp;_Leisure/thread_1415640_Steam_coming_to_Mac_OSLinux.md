### Post by Objekt on 2010-02-25
Steam?  Meh, not interested.  If I want to rent my games, I can always buy a console.

There's no technical reason to not have more Linux games.  The only real reason is the loop which hardware manufacturers get stuck in, much to our chagrin as Linux users.  It goes something like this:

1) Hardware mfg./game publisher reasons, "Hardly anyone uses Linux, so it isn't worth our time to produce Linux drivers/Linux port of our game."

2) Windows users who might switch don't, because hardware/game they want to use won't work under Linux.  Existing Linux users who might have bought the product, don't, for the same reason.

3) Number of Linux users barely increases because of #2, and hardware mfg./game publisher finds further justification for belief in #1.  

4) Back to step #1.

I don't see any way out of it.  Virtualization works for some things, but so far it is not a solution for gaming.
