### Post by dfreer on 2007-07-10
Cedega basically is wine with a GUI interface that they charge you for. It uses wine, so anything wine can do Cedega can do (at a price! :D ). Although to be fair cedega does make things a lot eaiser, and I don't have a problem paying a reasonable amount of money to get things to work.

Cedega Pros: 
(1). Cedega just included pixel shader 2.0 support, which so far I do not see replicated in wine (I believe it's there, just not enabled by default and a regedit enables it AFAIK).
(2). It has some specialized hacks to make really popular games (WoW, Steam, etc) work without much fuss.
(3). It has an GUI which manages which games are installed on the system, and uses "bottles" or seperate windows enviroments for each game, basically each game has it's own C:\ directory. This GUI also makes installing games a "bit" friendlier.
(4). Supposedly can handle games with exotic copy protection schemes that don't work in wine (although so far most of my games haven't had any problems in wine).

Cedega Cons:
(1). Is more focused on making specific games work with various workarounds, instead of working on actually getting directx API figured out, so it isn't really solving the problem I guess (frankly I don't think this is much of a con).
(2). Although they say it will contribute back code to wine, this seems to be under discussion.
(3). Pretty much every game that works on cedega works with wine anyways
(4). Poor tech support (IMO), and the application database needed sorely updated in 5.X although this may have changed in version 6

Wine Pros:
(1). I think it runs faster than Cedega, especially with Steam. I haven't tested this so I can't say for sure, definitely seems that way. Not FPS so much, but steam's main window took FOREVER to load in cedega.
(2). Is updated QUITE regularly, more so than pretty much any linux software I've seen. Expect to see new releases pretty much every month if not more regularly.
(3). AppDB is awesome, with tons of advice and step-by-step guides (in some cases). If you want to know if a particular game will work, and if so instructions to do so, check out winehq's appdb first!

Wine Cons:
(1). Is updated QUITE regularly, as said above. These updates, although often fix one thing, can quite often introduce new problems (lol).
(2). Can require register edits and uploading native .dlls in order to get some games to work. Although if it is well known you can find instructions in the AppDB.
(3). Resizing of windows and resolutions seems buggy, Cedega seems to handle these sort of issues a lot better.

That's what I can think of off the top of my head. Starcraft's Battle.net should work with wine, check the appdb. I have other problems with Starcraft, mainly because I have a widescreen laptop and starcraft is fixed at 640x480.

### Post by beeldings on 2007-07-13
As an Ubuntu gamer, I use all three Wine implementations: Cedega, Crossover Office, and Wine. I use Cedega for certain titles while I use Crossover and Wine for others. I use Cedega only for those titles that are officially supported by Transgaming (which isn't many). If I can't get a certain title to run under either Crossover or Cedega, then I turn to Wine. I've run into my fair share of problems running games with all three programs, and I found that I have had more success correcting problems with the Wine AppDB and by perusing these forums. However, with Crossover, the developers actually review the support forums and post responses, I have yet to see an official post from anyone at Transgaming on the Cedega forums. The Crossover forums come off as a more mature, informed community than the Cedega forums, but I guess that's just a result of each company's niche (Crossover=office and productivity applications, Cedega=gaming and nothing else).

I really don't know what else to add except that my PC game library has expanded thanks to Wine and its commercial implementations. If there's anything specific that you want to know about Crossover and Cedega, shoot me a private message or just ask here.
