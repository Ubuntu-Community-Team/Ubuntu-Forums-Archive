Well this is from wine FAQ.
>  
Is Wine an emulator? There seems to be disagreement.

There is a lot of confusion about this, particularly caused by people getting Wine's name wrong and calling it WINdows Emulator. 
When users think of an emulator, they tend to think of things like game console emulators or virtualization software. However, Wine is a [compatibility layer]("http://en.wikipedia.org/wiki/Compatibility_layer") - it runs Windows applications in much the same way Windows does. There is no inherent loss of speed due to "emulation" when using Wine, nor is there a need to open Wine before running your application. 
That said, Wine can be thought of as a Windows emulator in much the same way that Windows Vista can be thought of as a Windows XP emulator; both allow you to run the same applications by translating system calls in much the same way. Setting Wine to mimic Windows XP is not much different from setting Vista to launch an application in XP compatibility mode. 
There are a few things that make Wine more than just an emulator. 
[LIST]
[*]Sections of Wine can be used on Windows. Some virtual machines use Wine's OpenGL-based implementation of Direct3D on Windows rather than truly emulate 3D hardware. 
[*]Winelib can be used for porting Windows application source code to other operating systems that Wine supports to run on any processor - even processors that Windows itself does not support. 
[/LIST]**"Wine is not just an emulator" is more accurate.** Thinking of Wine as just an emulator is really forgetting about the other things it is. Wine's "emulator" is really just a binary loader that allows Windows applications to interface with the Wine API replacement.

### Post by armahillo on 2012-01-18
FWIW, I am currently running Skyrim on my Ubuntu machine, via Wine. I submitted test results. It runs decently that I've been playing (absurdly too much) for two weeks now and am at level 48. So... yeah, it plays fast enough for combat.

Getting it set up correctly was a bit of a pain. You have to install DX10 (on the disc), and then use winetricks to load XACT, VCRUN2008, and the D3d9_42 (look in the list, I forget the exact name). Remove the intro video, disable Gamepad (found in the skyrim prefs, which are in your My Games folder, *NOT* in the skyrim folder). I also curiously have to alternate between setting "Windows XP" and "Windows 7" modes for TESV.exe -- don' tknow why. sometimes the sound just doesn't work when I load it -- but if it works when I load, it will play fine.

There are some speed considerations, but I read on Toms Hardware that the game's AA an Isotropic Filtering bits are not so good, and that you should override them with those settings as provided by your video card (so use your ATI Catalyst or Nvidia tuning software to set your video settings manually, rather than leaving it at "use application settings"). I have yet to try that, so I don't know if it will make a difference.

Also, having an Intel Core i5 / sandybridge-based system seems to make a big difference too; I'm running an AMD Phenom II x6 and I've heard skyrim leans more heavily on the CPU than previous versions.

My specs (if you care):
AMD Phenom II x6
8 GB RAM (DDR3, IIRC)
NVidia GTX 550 Ti w/ 2GB DDR5
