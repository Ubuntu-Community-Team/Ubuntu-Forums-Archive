### Post by dorath on 2007-07-11
If there was a product comparable to DirectX, but with open components, then we'd probably see more games for linux (and Mac too).

You can't really compare openGL to DirectX, because DirectX is more than just graphics.  You *can* compare openGL to Direct3D, because they're both used for graphics.

For the same reason you can't really compare openAL to DirectX, because DirectX is more than just audio.  You *can* compare openAL to DirectSound, because they're both used for audio.

Here's a couple other things that DirectX includes, courtesy of [Wikipedia]("http://en.wikipedia.org/wiki/Directx"):
DirectInput, for keyboards, mice, controllers, etc
DirectPlay, for networking

Having all of those bundled in to one neat, easy to use package means developers have an easier time putting everything together.  If it's easier, it's quicker.  If it's quicker, it's going to *cost less* to make the game.  Remember, they're in this to make money.

So while there are definately alternatives to using DirectX, they're not integrated into one neat, easy to use package.

So what would it take for some enterprising souls to create the equivelent of openInput and openPlay, then roll them up with openGL and openAL into a neat, easy to use package we could call OpenX?  A suite of tools that could be used to create games not only for Windows, but for Macs and linux as well.  That's the sort of thing that I think could really sway developers.
