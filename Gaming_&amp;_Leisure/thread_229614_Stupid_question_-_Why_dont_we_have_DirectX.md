### Post by leech on 2006-08-05
It's actually kind of a good thing in my opinion that Linux does not have DirectX.  Mainly because that way we have a core API for 3D graphics, rather than the two competing ones on Windows.  Since OpenGL is pretty much universal (Mac, Linux and Windows all support it) I think DirectX was one of those 'Embrace and extend' type things where they created a proprietary API so that they could lock developers into their platform.  If anyone recalls long enough ago, DirectX really didn't become decent until about version 7.  Now DirectX 10 is going to be only available for Windows Vista, hello lock-in.  

Here are the reasons I think Linux doesn't get that many ports, and I don't think that low market share has anything to do with it (since no one can accurately tell how much of a market share Linux has, and it's growing exponentially, since MS hasn't released a new OS for a long time, and people are getting sick of malware)

1) Locked into DirectX.  This is simple, since the Xbox supports this, and a lot of game developers are concentrating on consoles (sad, really).

2) Support.  Most game companies feel they have to support whatever operating systems their game is available, naturally.  But with how many different Linux distributions out there, it's hard to hire someone that could support all of them.  This is why right now most of the native linux games out there are 'unsupported', but some companies won't release something that isn't supported.

3) Proprietary drivers.  Not everyone has a graphics card that can handle the newer games, or will even work properly on them due to driver issues (I'm looking at ATI here).  I think Windows users would be mighty pissed if there were games out there that only worked on one video card (well, there are, and some games will only support some level on detail on the SM3.0 cards.)  The legality of these keep coming up too, but I think that's just stupidity....

4) Copy protection schemes.  Publishers are usually the ones who worry about this, and quite frankly I think they worry too much about it and in the end it hurts the legitimate users more than the pirates.  Think Star Force for linux, do you think any self-respecting linux user would allow that crap on their system?

5) Everything is Free.  I know we don't think that way, but some developers think that since we more than likely didn't pay for our linux distribution, which came with all the normal neccessities of computer life (Browser, Email, Office, etc.) then we probably as a community won't spend money on their games.  I don't think many developers think this though.

6) Publishers.  Again, these guys are the ones who set the prices, distribute the games, etc.  These are the ones who, even when there IS a linux port, don't want to sell it.  Take a look at Cold War for example.  The port was completed the same time that the Windows version was.  But Dreamcatcher wouldn't publish the game for Linux, and instead made Mindware studios sit on the code for it.  Finally after 8 months of it floating around, and LGP doing testing, printing everything up and distributing it, Cold War for Linux is out.


There were some other reasons I had thought of before, but considering it's 4:15am, I think I'm going to go to sleep :D

### Post by figaro on 2006-08-06
DirectX is "[a collection of API's]("http://en.wikipedia.org/wiki/Directx#DirectX_APIs")" which covers graphics, sound, input, networking, encoding/decoding media.
OpenGL is just [graphics]("http://en.wikipedia.org/wiki/Opengl").

DirectX is a complete HAL (Hardware Application Layer). OpenGL is not.
You can see how this might be more attractive to developers - it is a standard set of tools for doing practically everything they need. 
Whereas with Linux you have ALSA for sound, OpenGL for graphics, CUPS for printing, blah, blah, blah. Different projects for each thing can get messy for a developer. People find it easier to learn one thing (language) and stick to it. Also DirectX is a lot more binary driver friendly which is ideal for hardware companies who dont want to reveal their code (not that that's a good thing, but if they won't budge on the issue and you have to use their hardware...).

I was looking at the Linux Standards Base to see whether they were implementing a proper HAL for Linux a la DirectX but its a bit unclear to me what they are actually doing.

I personally think Linux *needs* something similar to DirectX (an open source project of course) if it wants to go to the "next level" of public acceptance, but it might be better to tough it out with getting hardware manufacturers to release open drivers - still that might also be made easier for them to do by getting the LSB in order.

I just want to throw a quick comment in here:

**A game/engine is only easy to port to a different platformed if it was developed with that in mind.**
