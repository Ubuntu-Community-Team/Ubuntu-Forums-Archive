### Post by shaft350x on 2007-10-31
I read somewhere before about the idea of sort of a gaming specific platform through Linux.  Then I read this article and it made me start thinking about it again.

From: [http://diary.bluemango.in/?p=162]("http://diary.bluemango.in/?p=162")
> Even some game makers could conceivably abandon Windows by releasing custom Linux LiveCD versions of their games. Granted, there might need to be some embedded graphics support, but this need not be an insurmountable problem since many games only support a limited number of graphics adapters anyway.

Now I know that running from a LiveCD would be slower and probably not work, but I just thought it sounded like a really good idea.  Now I'm not a programmer so feel free to correct me on the ideas (just don't flame me please).

I see it working in one of two ways.

First way: every game is shipped on a super stripped down "distro" that is dedicated to running ONLY that game.  Since the game would ship with its own OS it would have only hardware requirements.  One would have to boot into each game (kind of reminiscent of old DOS days)  Then you'd probably have to go through and partition for each game which would be a pain.

Second way would be to have a universal gaming distro that all gaming companies would program on.  Notably the distro would be open-source and not MS.

I have no idea how it would actually have to work, so maybe I'm just dreaming big.  Let me know what you all think.  I know there are people out there who know more about this than I do and would have ways to easily fix the holes in how I am perceiving this.

(I know this isn't all that technical, but I just wanted to generate some discussion)

### Post by cogadh on 2007-11-01
This has been done many times already (FreeDuc Games, Games Knoppix, Kurumin, LinuxConsole, a few others). The best one I have found is the Linux-Gamers.net live sample DVD, it has a very good selection of Linux games on it:
[http://www.linux-gamers.net/modules/wfdownloads/singlefile.php?cid=42&lid=530](http://www.linux-gamers.net/modules/wfdownloads/singlefile.php?cid=42&lid=530)

Basically, it boots a stripped down version of Arch Linux, installs any proprietary drivers you might need into the virtual system and uses a memory stick for save games and config files. I think it meets pretty much all of the basic ideas you hit upon.
