### Post by dickohead on 2007-10-09
The correct question to ask here is:
"Which games support Linux"
And the answer, unfortunately, is: Bugger all.

You can however get yourself a copy of Cedega from [http://www.transgaming.com](http://www.transgaming.com) and see if they support the games you wish to play.

Cedega emulates a windows environment by providing the game all the files it needs to operate in windows, but while running on a Linux desktop.

Oblivion works quite nicely (not perfectly though)

A list of games can be found here:
[http://games.cedega.com/gamesdb/](http://games.cedega.com/gamesdb/)

There are also the native games like Quake that will run locally, guild wars too I believe.

### Post by Jovec on 2007-10-11
> **tomjennings83 said:**
> Well im a really serious gamer, thats all I do so im not sure if ubuntu would be the ideal OS for me.

Absolutely not.  Stick with Widows if you are a serious gamer, and you'll be able to play virtually any game you want, without complications.  However, you can usually find a suitably fun, but often older, alternative game to play in a given genre for Linux.  And there are a few top quality games for Linux either dedicated (Quake 4 and Quake Wars:ET) or with excellent support via Wine (WoW).

You can also dual-boot, or buy/build a second computer to game in Windows and use Linux for everything else  (A KVM switch works well here to save desk space).  Many serious gamers upgrade the core of their computer (MB, CPU, GFX) every 12-24 months, so often your hand-me-down parts can be used to build a dedicated Linux box for only a little extra while your gaming rig stays current.

Another alternative is to game when you can on Linux, and buy a 360 or PS3 for the rest of your gaming needs.

There are other factors to consider here too.  Nvidia cards are still much better supported in Linux than ATI.. if you dropped $500 on the latest ATI card I wouldn't use it with Linux.  Voice communication software like Ventrilo can sometimes not work at all for some users via Wine, although you can try Teamspeak with its native Linux client or use basic VoIP.  Level editors may not work even if the game does (like NWN).

### Post by Jovec on 2007-10-11
There are many reasons to use Linux.  Gaming isn't one of them, and properly setting expectations is key to preventing potential Linux users from being disappointed in one area which can prevent them from seeing its benefits in another.

For many of us however, there are enough native and emulated games, both commercial and open source, to satisfy our gaming needs while we enjoy the other benefits of Linux.  Very rare is the "can't miss" game that only runs on Linux.  Even Tremulous from the example above has Windows binaries, as do many of the open source games we play on Linux.
