Compatibility wise I had no issues with finding software on the net, for example I'll always use VLC player for media files, so I looked on their website and sure enough theres a linux version for download. ### Post by 3rdalbum on 2013-02-08
Nvidia GPU, definitely. ATI GPUs are total rubbish on Linux because ATI's drivers lack features. ATI also removes support for anything more than a couple of years old. You are probably safer with just one Nvidia card, not an SLI setup.

As far as distros goes, "Windows-like" shouldn't be your main concern. The quality of the distros, the hardware support, how easy is it to install Nvidia drivers, how easy to use generally, how much software is in the repositories, whether there is a big community who can help you if you get stuck... that's the things that matter when choosing a first distro. Not "does it look like a Windows desktop". 'Cos it certainly wont work like one!

Ubuntu is a good choice for all the factors I listed. You couldn't mistake it for Windows but it is pretty consistent and easy to use.

Couple of words of advice here; I'm a hardcore gamer myself; have been since DOS days and, as per my previous post, I jumped ship to Linux a week ago.

Desktop Environments, aka, DE's. Be a bit careful with them if you're looking to game. This is why I've been undecisive on my own build.

Ubuntu uses Unity (Gnome3 and Compiz)
Xubuntu uses XFCE
Kubuntu uses KDE
Lubuntu uses LXDE
Mint (standard) uses Gnome3 and Cinnamon.


Reason for that? Do you remember talk of things like Steam Overlay, or overlay's in general under Windows slowing down games? If you do this'll make sense, if not then just know this:

Gnome, Compiz, Cinnamon and KDE were designed to make the desktop look pretty. That is, they use the GPU, and weren't originally designed to play too nicely with games requiring all the GPU grunt.

XFCE is far more light weight, as is LXDE (and a myriad of others such as OpenBox etc but that's beyond the scope of this)

TL;DR - If you want great performance from linux in games, use a lightweight DM. If you don't mind suffering a NOTICABLE performance hit, then go ahead and use a standard DM; you'll get an (arguably) better desktop experience out of it.

I've got Heaven benchmark on my system atm actually and have been playing around with a few DE's, still trying to work out what I want; I can post some results up if you're interested.

For the record my system is a Core i5 Wolfdale, 8Gb ram, nVidia GTX 670.

Also - Be aware that you won't be able to play silverlight based websites, so no blinkbox, and (if you're going to seriously use linux for gaming) no Netflix. (you CAN watch Netflix, but you need a customised Wine installation to do it which you can't then mix'n'match with a Wine installation you might want for running Windows games.)

### Post by Am Elder on 2013-02-10
> **JMerithew said:**
> From what I was reading it sounded like people have had success with KDE for gaming, but with what you said about overlays and stuff I'm a bit hesitant.

The KDE folks certainly [claim their desktop environment is better for gaming]("http://www.muktware.com/4894/kde-plasma-should-be-your-primary-choice-linux-gaming") because of, er, something to do with compositing.

However, [recent tests by Phoronix]("http://www.phoronix.com/scan.php?page=article&item=ubuntu_1304_desktop1&num=1") of OpenGL gaming using various desktop environments showed KDE to be the slowest and the light desktop environments to offer the best performance for gaming on their test machine.  Phoronix periodically publishes [informative articles]("http://www.phoronix.com/scan.php?page=news_item&px=MTI5NjU") and benchmarks, so you might want to search the site (and of course the web generally) to look for evidence on the hardware you're thinking of for your new Linux gaming rig.

Personally, I'm happy with a light desktop environment.  The bells and whistles of fancy DEs are envy-inducing, but I notice a slowdown on my four-year-old laptop, so prefer to go with LXDE and Lubuntu at the moment. 

Of course distro-hopping is part of the Linux experience.  You can do your own testing and change distro at will.  You don't have to make a decision now, and you can change your mind later.

### Post by holastickboy on 2013-02-11
Please note that the version of KDE that they are benchmarking is 4.9.9.8 which is not the current release version. Also, are they suspending desktop effects? I can't see that (they usually list KDE as KDE Sc when they do) and can't see anything written about it. If you look at the ACTUAL differences in frame rates, they were only a few at the most, so you could still use want you want and game.

Honestly, I think all DE's are pretty good with gaming currently, as long as you disable compositing, which make a noticeable difference in performance.  One good thing that KDE has, that I haven't seen on any other DE's is the ability to disable compositing via a hotkey on the fly (eg, if the game doesn't support full screen, you can hit alt-shift-f12 and it disables compositing).  Currently in DE's like Unity, disabling compositing is only working in full screen apps (so performance decrease in windowed games).

While this might not be an issue for everyone, if you are running more than one monitor, you tend to run games in Linux so it doesn't attempt to stretch over multiple monitors.

While we are talking about having more than one monitor, I can tell you that there is NO difference in frame rate tests in Unigine Heaven with my dual screen setup over single screen (both in seperate x sessions and twinview).
