### Post by farrell2k on 2009-12-30
So I have decided to do an top-down, ASCII in Java via console, done with print() statements, or maybe in swing with drawString() commands(likely).  It needs to be something we can all play on windows, mac, or linux.

Game play will be like Zelda for exploration, where you moved from screen to screen, unless I can get decent performance in a console with smooth scrolling.  I doubt I will be able to, unless I use swing.  If I end up using swing, which is likely, I will make overworld movement like Final Fantasy and Dragon warrior for the NES.   

The game may be space-themed like Phantasy Star.   The whole dungeons and dragons fantasy theme has been done to death.

The game will be single player only.  I have no motivation to learn any network code beyond the basics I already know.   If you're a pampered gamer who thinks single player is pointless, this will not be for you.

Battles will be random, turn-based battles like Dragon Warrior for the NES.   You'll be able to control more than one player, and you will mostly be attacked by more than one enemy.

Players will be able to attack, defend, cast magic, run, etc.

Enemies will drop loot and items, and will also add to your experience when they're defeated.

You will be able to parlay with enemies who will react with you based upon your attributes like charisma.   They may attack you, beg for mercy, run away from you, etc.   

Players will level via experience points.  I may handle leveling automatically, because it's easier :), or I may require players to pay for training like we did in the 80s and early 90s with the SSI Bold Box AD&D RPGs.  This is easy as well, but some may not like it.

Each player will have his/her own inventory, and will be able to trade with one another, as well as buy and sell items in shops.

Players will be able to use different weapons and armor, each with the own attributes.  Certain classes will be restricted to certain weapons and armor.

You will be able to make camp, rest, and save anywhere (I hope.  I'll need to see how I will work this.), and you'll be able to sleep at inns as well.

NPCs may join you for hire, or for the completion of a quest.

There will be some sort of main quest, as well as side quests.  Maybe the community can help me with story ideas?

There will be three player classes, but I may make more, depending on motivation.

Fighter
Cleric
Magic-User

Maybe I'll add a thief class as well.

Players will probably look like this: @
Enemies will look probably like this: x or X, or maybe an odd looking ascii character.  
Buildings and other structures will probably be made with...I have no idea, yet.  I'll worry about it later,  when I actually start working on the visuals.

The game will probably not have audio, unless it turns into something half-decent with real graphics.

I'd like to make a nice story, something a little more complex than "save the princess", not that there is anything wrong with that. :)

Despite the game's simplicity compared to games today, believe it or not, it is still a very ambitious undertaking by one person.  I doubt the game will ever really be *finished.   I am sure there will always be something I'll be poking around with.   I will make it playable to the end, however.

I already have a few main player, monster, and item classes made, a small inventory system, as well as a silly little battle loop that I have been goofing around with for a bit.

Are there any game idea that you guys would like to see, or anything that you think would work better than what I am currently thinking?
 
Wish me luck.
