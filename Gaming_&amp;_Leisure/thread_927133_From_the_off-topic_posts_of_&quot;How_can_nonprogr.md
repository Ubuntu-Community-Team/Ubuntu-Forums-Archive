### Post by curvedinfinity on 2008-09-22
From page 2-ish of [http://ubuntuforums.org/showthread.php?t=916502](http://ubuntuforums.org/showthread.php?t=916502)

[QUOTE=curvedinfinity]In keeping with the ID theme, my model is having "Quake" (including the first campaign) open source from the get-go. Released along side it is an expansion like "Quake Mission Pack 1: Scourge of Armagon," which is available for sale.

A pessimist would look at this and say that the open source part is a "demo" while you have to pay for the real game. The yard stick we use to prevent the open source part from turning into a glorified demo is that the expansion should have no more new content than there was in the original open source game.

-- Just to note, this is just the model I am personally using for the games I make. The previously mentioned project, which is a publishing platform, doesn't require that other games follow this model. However, I'm going to encourage this model's adoption if feedback is positive.[/QUOTE]

[QUOTE=Mickeysofine1972]I like your idea curvedinfinity, but how would you apply that to a game that doesn't have levels such as a soccer game?

Mike[/QUOTE]

Mike, there are a number of ways you could work it. Your expansion could have new teams, players, stadiums, balls, and maybe even some arcade-styled "challenges". Maybe one mode has you on a stadium on the moon, where everyone is very light and the ball floats, not to mention the stadium is a moon-rock crater.

For any game there can be any number of things that people would enjoy purchasing, especially because they are directly supporting their developer.
