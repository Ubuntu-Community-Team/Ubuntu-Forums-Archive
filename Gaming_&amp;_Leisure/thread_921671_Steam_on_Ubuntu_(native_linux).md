### Post by DeadSuperHero on 2008-09-17
I'd love to see someone actually develop a good Linux Steam client.

A few ways to work with this is to first find out what programming language Steam is written in. What calls does it make? How? Why?

What you then need to do is find some devs who happen to know the internals of Linux, which libraries to work with, compilers, etc.

And finally, you'd need to have people actually code the client, port various Steam engines, etc.

There's actually some great Linux ports from Id Software.

As for the community though, I have no idea.
