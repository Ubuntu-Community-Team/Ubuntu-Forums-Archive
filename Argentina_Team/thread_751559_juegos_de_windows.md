### Post by Mauro22 on 2008-04-10
VMware es para virtualizar un SO y WIne emula la API de windows para correr aplicaciones.


O sea, con VMware emulas el sistema operativo completo y con wine una aplicacion en particular.


Con respecto a los juegos, generalmente los sistemas virtualizados no "levantan" la placa de video, sino que en lugar de la que tenes, crean una generica de 64mb (o cualquier otra cantidad que quieras) pero sin acceleracion grafica.

### Post by Mister X on 2008-04-10
el VMware es un emulador de hardware, lo que hace es crear una maquina virtual en donde podes instalar un sistema operativo completo desde cero y las aplicaciones como si se tratara de una PC aparte, siempre corriendo dentro de un sistema operativo host (anfitrion)

el wine es una implementacion de las APIs de windows que te permite ejecutar en linux aplicaciones hechas para windows

### Post by Diabolis on 2008-04-12
> con VMware emulas el sistema operativo completo y con wine una aplicacion en particular.

wine no emula, su nombre de ahi viene Wine Is Not a Emulator. Es una implementación del API de windows, como ya mencionaron en otro post. No es recomendable jugar dentro de vmware porque este carga un sistema operativo dentro de otro que ya esta corriendo y consume bastantes recursos de la computadora. En cambio lo que hace wine es permitir la comunicación entre windows y linux, como si fuera un traductor.

Los juegos que he corrido son warcraft III, world of warcraft y diablo 2. No he tenido níngun problema con ellos y corren muy bien.
