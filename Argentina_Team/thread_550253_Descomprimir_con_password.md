### Post by marianom on 2007-09-14
La mejor forma de hacer que esto cambie es la siguiente:

1- identificar si el programa que se está usando es soft libre o no.
2- a- si no es soft libre THE END
2- b- si es soft libre ubicar la página del proyecto.
3- una vez allí verificar la documentación para confirmar si hay o no solución establecida.
4- si no la hay, buscar en su sistema de bugs para ver si alguien lo reportó, si se lo solucionaron o no.
5- si está reportado pero no solucionado, arrimar un reclamo más al respecto para hacer bulto.
6- si ni siquiera está reportado, armar un caso y subirlo: se puede hacer con la terminal para que queden registrados los errores, con strace, con bug-buddy, recompilando con soporte para debug, etc para obtener pruebas claras y contundentes del issue así no cierran el ticket por falta de elementos.
7- hacerle la marca personal al problema hasta que esté solucionado,

Si, es una terrible molestia e insume bastante tiempo pero es una forma muy buena de colaborar para que los programas que usamos mejoren.

Confirmo lo que dice kowal, para mantener los archivos hay que abrir en una terminal dentro de la carpeta en donde está el .rar y poner:


Con esto de descomprime el archivo en la misma carpeta aunque tenga errores.
