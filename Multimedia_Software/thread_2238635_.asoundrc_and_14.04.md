### Post by f1ebk on 2014-08-09
Hi,
I used a .asoundrc file to do some re-sampling of a 48Ks/s card to a 11Ks/s program, with Ubuntu 12.04 everything was fine, and the .asoundrc looked like : 
--------------------------------------------
pcm.radio {
type hw
card 1
device 0
}
pcm_slave.radioslave {
pcm radio
rate 48000
}
pcm.radioconv {
type rate
slave radioslave
}
--------------------------------------------

Now, on Ubuntu 14.04, the radioconv device do not appear any more, so I cannot use it. 

I checked the alsa.conf file, and the .asoundrc hasn't moved, but it did not. So is there a new syntax to do this simple thing ? 

Thanks for your help

### Post by Glen_Overby on 2015-02-04
I recently upgraded to 14.04 and encountered the same problem.  My .asoundrc file was identical to yours, so I assume we're both running wsjt.

I downloaded the latest alsa-lib (1.0.28 today) from the ALSA project ([www.alsa-project.org]("http://www.alsa-project.org")), built and installed it to my home directory, and was able to run wsjt after removing the "device 0" line from the pcm.radio definition.

Before running wsjt, I have to set:

  export LD_LIBRARY_PATH=$HOME/os-fixes/lib

where there is a new libasound shared library.
