### Post by pssturges on 2011-06-12
Hi,

I have been having this issue for quite some time with many different sources. I have a very basic script that takes a given video source file and creates a pal dvd version of it. The script calculates the required bit rate in order to produce the the highest quality that will fit on the disk. I use ffmpeg to do the conversion, but have also tried mencoder and had the same results.

The problem is, that ffmpeg and mencoder seem to disregard the bitrate I specify and the resulting video is way smaller (and lower quality) than should be. Videos that should fit on a DVD5 end up anywhere between 2.5gb & 3.5gb.

Now I have read other people have had similar issues and the responses have said that it is due to the quality of the source material. If the source is of low quality, then ffmpeg will only use the required bits to reproduce that quality. That seems quite reasonable to me. However, I wouldn't think that would apply if the source was a blu-ray rip of a very recent blockbuster film.

Here is the ffmpeg commond used:
```
/usr/bin/ffmpeg -i movie.m2ts -f dvd -target pal-dvd -aspect 16:9 -b 7848kb -mbd rd -trellis -mv0 -cmp 0 -subcmp 2 -y outputfile.mpg

```
