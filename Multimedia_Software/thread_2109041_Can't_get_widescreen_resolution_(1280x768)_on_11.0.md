### Post by cwsnyder on 2013-01-26
Have you tried either removing or commenting out the line: > Option         "metamodes" "1024x768_75 +0+0 {viewportin=1280x768}"

That looks to be causing your problem.  The error you are reporting is because you tried to create a **--newmode** which is the same as an already existing mode.

### Post by cwsnyder on 2013-01-27
> **sword_guy said:**
> 
user@computer:~$ xrandr --addmode VGA-0 "1280x768_75.00"

I usually keep quotes around the mode name when I have an underscore or period in the name, otherwise it does not interpret the name correctly.  If I don't want to keep using quotes, I use just the first portion of the resolution name, ie. 1280x768
