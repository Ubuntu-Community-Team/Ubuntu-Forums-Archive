### Post by cariboo on 2009-04-04
I would suggest you start up synaptic and remove all references to nvidia, then backup your /etc/X11/xorg.conf file:

```
sudo mv /etc/X11/xorg.con /etc/X11/xorg.conf.bak
```

THen restart and use System-->Administration-->Hardware Drivers to install the restricted drivers. I have an 8400GS that work flawlessly. See screenshot.

If you can get a working desktop with Live CD, there is no reason it wouldn't work on your normal installation. Backup and remove your xorg.conf, a new one will get generated. Next, install the recommended restricted driver via the restricted driver manager. You should now have a working desktop running nvidia driver and 3D graphics. Note the new line "driver=nvidia" in xorg.conf.

Now you can customize your xorg.conf, one step at a time. Once the nvidia driver is specified in xorg.conf, there is no fallback if it can't be loaded. I noticed you had 2 screens defined and some other options. The error message "but none have a usable configuration" gives a hint that some options are causing trouble. You may have carried over options that are no longer relevant.
