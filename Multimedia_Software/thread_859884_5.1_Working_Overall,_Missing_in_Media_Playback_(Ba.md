### Post by actionscripted on 2008-07-15
I'm having a problem getting 5.1 to carry into the various media players. I have the channels configured using .asoundrc:

```
pcm.!default {
        type plug
        slave.pcm "surround51"
        slave.channels 6
        route_policy duplicate
}
```

In most places this works perfectly, Pandora in Firefox being one example. However when playing back music using Rhythmbox, Banshee, Totem, et al. there are only two channels being sent out.

I realize that the files I'm playing back aren't necessarily 5.1-capable, but is there a way to force the 5.1 cloned shenanigans to carry into these media playback applications? It seems odd that Pandora in Firefox would work fine, but a stand-alone media playback app would not.

### Post by mc4man on 2008-07-15
I've pretty much found that changing the audio output to alsa will give you 5.1 on music files with most players. The exception was amarok which seemed to need a couple of add. packages.
Start with preferences -> sound and change the dropdown for music... from autodetect to alsa. Also ck. each player in setting, preferences, wherever and if there is a setting also change from autodetect or default to alsa.
