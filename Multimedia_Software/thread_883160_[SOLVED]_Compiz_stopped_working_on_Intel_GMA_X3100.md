### Post by Frem on 2008-08-07
So, when I first installed Ubuntu, my Intel GMA X3100 worked out of the box, I didn't have to do anything.

I decided to try KDE 4.1 this past week, so I installed it via apt-get using the instructions on the Kubuntu website. Apparently Aptitude is the preferred method. Oops.

I decided KDE wasn't for me, so I went to uninstall it. I ended up doing a lot of it manually, starting with the full dependencies of the KDE 4 metapackage, and removing all the items that ubuntu-desktop didn't depended on. I may have removed a few too many, I'm not sure. There were also several stragglers including the linux headers for my kernel that aptitude uninstalled (though I promptly reinstalled the headers).

But anyway. The Kubuntu KDE 4.1 packages need a *lot* of work; there are a bunch of errors I see during boot referencing KDE scripts, and it didn't reconfigure GDM as my default login manager, printing an error about KDM and dumping me to the command prompt. 

Also, _Compiz has mysteriously been disabled_. When I try to activate Desktop Effects, the windows flash a few times and I get the message "Desktop effects could not be enabled."

Edit: Ok, I'm sorry. I accidentally removed the package libgl1-mesa-dri. Installing it instantly gave me 3D acceleration, didn't even have to restart X. Gotta love good open source drivers. 
If you find yourself in a similar situation, run the command ```
cat /var/log/dpkg.log | grep remove > removed.txt
``` to see what has been uninstalled recently.
