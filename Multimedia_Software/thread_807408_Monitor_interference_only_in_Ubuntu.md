### Post by blorch on 2008-05-25
Hey everyone. I recently upgraded to an nVidia GeForce 6800 on my Hardy Heron box. I installed the driver from the restricted drivers manager, and all seemed to be working well.

The video card has two outputs--one analog VGA and one DVI. I have LCD monitors plugged into both of these outputs. The problem is that the monitor attached to the VGA output is displaying some weird visual noise. It looks like a bunch of moving diagonal bands, and if I didn't know any better I'd say it was interference from a poorly shielded cable. This monitor and cable worked just fine with the old video card, however, and when I switched cables, I determined that the VGA port itself was the culprit.

This is where it gets strange, though. I switched to Windows, and I experienced the same problem with the moving bands on the VGA monitor. When I increased the monitor's refresh rate from 60 Hz to 70 Hz, though (which, if I'm not mistaken, really shouldn't do anything to an LCD), the problem disappeared. My display went back to being as clear as it had ever been. So... it seems my hardware's fine; it just takes a little finagling to get it working now with this new video card.

Whatever changing the refresh rate did in Windows, I can't seem to find the equivalent solution in Ubuntu. I manually edited the horizontal sync and vertical refresh values in my X configuration to exactly what the monitor's manufacturer specifies, and still no difference. Oh, that's another thing: this problem only presents itself in X sessions, which pretty much convinces me this is a software issue. I'd be thankful for any ideas people come up with.
