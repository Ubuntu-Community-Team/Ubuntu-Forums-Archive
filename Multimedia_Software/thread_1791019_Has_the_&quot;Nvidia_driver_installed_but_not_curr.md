### Post by jocko on 2011-06-26
> **fpgdu said:**
> Jocko how did you get nvidia-current to work? I cannot run Unity because  of the problem and this is the screen I get in System, Administration,  Additional Drivers:
...

First I made sure the restricted repos were activated (open up synaptic; Settings-->Repositories. Activate the line that ends with "(restricted)").
Then I started the driver manager, highlighted the recommended driver, clicked "Activate" and let it do it's thing. Then I rebooted and it worked. Just as it has for the last few years.

To fix yours, first make sure the driver is really installed. Search for it in synaptic. If it is marked as installed, it is installed.
Second, have a look at your /etc/X11/xorg.conf.
If you do not have a xorg.conf, try this in a terminal:
```
sudo nvidia-xconfig --force-generate
```
Note if the commands generates any errors, otherwise reboot (or just log out and in again, or restart your xserver by closing all applications and pressing Atl+SysRq+K).

### Post by mikewhatever on 2011-07-12
Right, making Compiz/Unity work is a bit tricky and might not work at all. Due to some changes to Compiz in 11.04, all the Geforce 700s displayed bizarre behavior with the nvidia-current driver. The bug is still unfixed (probably won't be for 11.04), and the devs had decided to blacklist the problematic hardware.

What worked for me with the Geforce 7400go was installing the legacy nvidia-173 driver instead of the recommended nvidia-current, and to override the blacklisting, add UNITY_FORCE_START=1 to /etc/environment.

I've used both the command line and the Additional Drivers Utility to install and remove different nvidia drivers, no problems there. If you are stuck in the recovery mode, use the following commands to purge nvidia-current and to install nvidia-173:
```
sudo apt-get purge nvidia-current

sudo apt-get install nvidia-173
```
