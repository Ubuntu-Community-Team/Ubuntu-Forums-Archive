### Post by zgornel on 2008-07-16
Well, you are using the open-source ati driver. Depending on what ati you have it may not be fully supported (especially for the X1000 cards). If you want to use the ATI drivers you should have that portion of xorg.conf looking like this
```
Section "device" # 
	Identifier	"device1"
	Boardname	"ATI Radeon"
	Busid		"PCI:3:0:0"
	Driver		**"fglrx"**
	Screen	0
	Vendorname	"ATI"
	Option		"MergedFB"	"off"
```

By the way, fglrx does not work extremely well with compi because of flickering of xv and opengl windows. The open-source driver works perfectly just that is highly unstable (and it requires Xserver 1.5 that will be available in Intrepid).

### Post by Rocket2DMn on 2008-07-16
xorg.conf has you using the failsafe "vesa" driver, you should attempt to reconfigure X with
```
sudo dpkg-reconfigure xserver-xorg -phigh
```
Then restart X with CTRL+ALT+BACKSPACE.  You may need to use aticonfig (or whatever the ATI restricted driver pacakge is) to configure from there, the restricted drivers may not enable by default after reconfiguring, though they might.

### Post by markbuntu on 2008-07-17
You can run aticonfig --help and it will give you a list of options you can run. You could probably force the higher resolution on your monitor with these commands but you need to be aboslutely sure the monitor can handle it. 

As an altenative, you can use the Catalyst Control Center, amdcccle from the command line and tell it to "detect display" and see what it tells you. It may let you use a higher resolution than x. Usually, Catalyst CC is hiding in Applications/Other. If you do not have it, it is in the repos as amdcccle.
