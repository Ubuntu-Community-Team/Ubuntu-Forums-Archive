### Post by hammeraxe on 2009-07-16
My EeePC 901 is running an almost fresh Ubuntu NBR 9.04. It all worked incredibly well until I decided to plug in my 19" external LCD. I could not get the right resolution and had to use xrandr to make it available. I set the screens up so that my laptop screen is off, but the external display runs at its max resolution (1280x1024). I was asked to increase my virtual resolution, which I confirmed. At that point both Totem and VLC started crashing immediately after launch (i.e. when a video file is opened). I took a random tip from the internet and set my default video output to X Window System (No Xv) in Multimedia Systems Selector. Now Totem plays video, but it is extremely laggy and CPU usage skyrockets. VLC works best with X11 Output set in its preferences, but it is still unusable.

Any help would be greatly appeciated.

### Post by hammeraxe on 2009-07-17
Ok, I managed to figure it out myself. The EeePC video card apparently cannot handle large video surfaces. As I changed the virtual resolution in xorg.conf back to the laptops native 1024x600 all the problems were gone. :)
