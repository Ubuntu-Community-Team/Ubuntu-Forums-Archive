### Post by mc4man on 2010-11-18
The drive isn't one that would prevent playback if there was no region set or a region mismatch.
you could check if a region has been set, though try this first.

Insert a dvd, close out any player if it opens.
Then open up your home folder - view - show hidden files. Scroll down and delete the .dvdcss folder.
(players always look there first for keys for the dvd  and will use them if found, even if incorrect so always delete the folder before trying to troubleshoot

Then open a terminal and run these 2 commands. Look in home folder for vlc_output.log and post the contents if the movie doesn't start

```
export DVDCSS_VERBOSE=2
``` 
  ```
export DVDCSS_METHOD=title && vlc dvd:// > vlc_output.log 2>&1
```

### Post by Stigmata13 on 2010-11-20
ubdoobdu, youtube videos use adobe flash which is restricted software. An easy fix would be to open the terminal and type:
```
sudo apt-get update && sudo apt-get install ubuntu-restricted-extras
```
This will install flash, java, and various codecs which could prevent future problems.
If you later find yourself looking to play windows format videos, you could always add the "Medibuntu" repository, which contains restricted software such as windows codecs and even libdvdcss2. You can add the repository by using this code, in the terminal:
```
sudo wget --output-document=/etc/apt/sources.list.d/medibuntu.list http://www.medibuntu.org/sources.list.d/$(lsb_release -cs).list && sudo apt-get --quiet update && sudo apt-get --yes --quiet --allow-unauthenticated install medibuntu-keyring && sudo apt-get --quiet update
```
You can look here for more details about the packages in the Medibuntu Repository:
[https://help.ubuntu.com/community/Medibuntu]("https://help.ubuntu.com/community/Medibuntu")
