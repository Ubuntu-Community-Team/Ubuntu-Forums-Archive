### Post by sam-korson on 2014-07-25
Hi, I've been recently having a problem with getting my native screen resolution on my monitor with the proprietary AMD drivers.  I was able to get it in Ubuntu 13.10 with the open-source drivers, but now I need to use the "nomodeset" option due to a [bug in 14.04 with my hardware]("https://bugs.launchpad.net/ubuntu/+source/linux/+bug/1309578"), and this makes the open source drivers unable to detect my screen's supported resolutions.  Plus, only the proprietary drivers can provide me with HDMI sound to the monitor, it seems.  So now that I've switched to the "fglrx-updates" AMD drivers, my resolution can't go any higher than 1920x1080.  It's supposed to be 2560x1440.  I've been messing around with a couple different methods to add this resolution, but they aren't working for me.

The first thing I tried was adding it with xrandr.  I used the command "cvt 2560 1440 60", and got the modeline from that.  Then I did "xrandr --newmode "2560x1440_60.00"  312.25  2560 2752 3024 3488  1440 1443 1448 1493 -hsync +vsync", that seemed to work fine.  Next, I did "xrandr --addmode DFP1 2560x1440_60.00", and got an error:
```
X Error of failed request:  BadMatch (invalid parameter attributes)Major opcode of failed request:  141 (RANDR)
Minor opcode of failed request:  18 (RRAddOutputMode)
Serial number of failed request:  39
Current serial number in output stream:  40
```
So, I eventually tried a new xorg.conf file, which I had generated by using the command "aticonfig --initial", modifying it to match my desired settings.
