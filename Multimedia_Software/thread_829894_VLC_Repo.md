### Post by Pjotr123 on 2008-06-15
What you want is understandable, but it goes against the repo system. In Ubuntu you're guaranteed a secure and stable system, as long as you stick to the apps in the repo's. This means that you don't always have the latest version of an app. 

Sometimes the package maintainers from the repo's hack an app, to make it more compatible with Ubuntu. Or to plug a security hole. It's all open source, so that's no problem for them.

Which means that the version number of an app may be older, but that doesn't mean that it's still the same as the identical version number made originally by the upstream developers.

Greeting, Pjotr.

### Post by cozmicharlie on 2008-06-15
The repos can be a little slow about updating.  The philosophy is they want the most stable versions so they tend to run behind in their upgrades.

The vlc page has a nice tutorial on adding the vlc repos either gui or command line.  Go here [http://www.videolan.org/vlc/download-ubuntu.html](http://www.videolan.org/vlc/download-ubuntu.html) and in the right column it has "How to add to repositories".  Note though the page does not show Hardy so my guess is they have not updated vlc for Hardy yet (or they have not updated the web page).  You could try the debian version ([http://www.videolan.org/vlc/download-debian.html](http://www.videolan.org/vlc/download-debian.html)) - Ubuntu is based on Debian so it should work. Some advice - I set up a Ubuntu install in Virtualbox just for situations like this.  That way I can play around with it and make sure what I do does not hose my system before I try it on my main Ubuntu box. 

Good luck
