### Post by markbuntu on 2009-04-24
You have no kernel modules installed so the driver has defaulted to indirect rendering and that is why it is so slow. You need to remove any old drivers you have using the script in usr/share/ati and then you can try installing the new driver.   If that fails you need to install the driver manually. The directions are here

[http://wiki.cchtml.com/index.php/Ubuntu_Jaunty_Installation_Guide](http://wiki.cchtml.com/index.php/Ubuntu_Jaunty_Installation_Guide)

The most likely cause of this is failure to remove fglrx before upgrading which will leave the old driver in place and the new kernel build will try to use it. Drivers before 8.602 are incompatible with the Jaunty xserver. This is why x failed.
