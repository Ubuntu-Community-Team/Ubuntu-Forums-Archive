### Post by tgalati4 on 2010-01-31
Responding to your PM:

Post the audio-related output of:

sudo lsusb -v

Your USB sound card looks like a nice device, but perhaps there is not a decent linux driver for it.  If that is the case, then you need to send an email to the manufacturer and ask for a linux driver.  

It sounds like the volume control registers are set at maximum and the usual controls are not changing them.  Most likely because the driver is not changing the correct registers.

I don't have any USB sound cards in my collection of stuff, so I can't provide any better tips on how to troubleshoot.

If your machine dual-boots into windows (or find a windows machine), load the windows driver and work through all of the control settings and take notes as to the behavior.  You obviously want the the same response in linux.

Once you discover the basic sound chip that is being used in your device, you can google search for the appropriate ALSA settings--if a valid ALSA module exists.

In Ubuntu up to version 9.04, the Gnome mixer applet was configurable to use any volume control as the main volume control, regardless of its name.         control {

        }

            type pulse
        }
    }
    capture.pcm {
        type pulse
    }
}
```
