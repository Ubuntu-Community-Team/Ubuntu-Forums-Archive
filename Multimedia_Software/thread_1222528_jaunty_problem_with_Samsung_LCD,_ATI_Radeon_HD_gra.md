### Post by hyperyoda on 2009-07-25
Having trouble getting my monitor (LCD) and video card working correctly in Ubuntu Jaunty.

When I use the open source X.org driver it correctly detects my monitor as
a Samsung and I can quickly change display resolutions using System->Preferences->Display however when I play a game using 2D acceleration such as Nexuiz there is no hardware rendering apparent.

However when I use the proprietary fglrx driver the hardware acceleration
works great but it does not detect my monitor (when I go into Display) and if I try to change resolutions it freezes, the window goes gray and my X server
process jumps to 99% utilization! So I think there is some bug in the newer
version of the X server and the fglrx driver.

So I was wondering if you could help me manually setup my /etc/X11/xorg.conf so it uses that and does *not* attempt dynamic configuration which it is currently doing.

Here is various info, let me know if you need any more.
The 4 modes I want are 1920x1080, 1440x900, 1024x768, 800x600.

My Samsung monitor says it should run at refresh rate of 60Hz.

### Post by jbrown96 on 2009-07-25
I would try to install the ATI driver from ati.amd.com and you will get the Catalyst Control Center application, which should allow you to change the resolution. The Display section doesn't typically work with fglrx. Once you download the driver, install it with ```
sudo sh DRIVER/FILE
``` obviously replace with the actual file. That will take you through the installation process. Reboot, then you should be using the driver. There will be an entry in Applications for the control center; this should allow you to change the resolution.

### Post by markbuntu on 2009-07-25
The fglrx driver in the repos does not support the 4550, it is too new. The latest driver from ati should work just fine for you though. be sure to remove whatever fglrx driver you have installed before installing a new one
