### Post by BicyclerBoy on 2011-07-31
When you refer to your custom modeline you must use the complete name string.
(This is different to the xrandr nonsense)

So in the metamode line 
Option         "metamodes" "DFP**O**-0: 1440x900"
should be
Option         "metamodes" "DFP-0: 1440x900**_60.00**"

Option "UseEDID" "False"
should be in the screen section.

Are you using DVI or VGA connection ?
You refer to DFP-0 (DVI) but have a name (just a name) of CRT in the monitor section.

There is lots of good debugging info in the file
/var/log/Xorg.0.log

### Post by realzippy on 2011-08-01
Hasn't this monitor 1680 x 1050 native ?
Anyway,here you go:

1.move eventually existing file *monitors.xml* out of the way:
```
mv ~/.config/monitors.xml ~/.config/monitors.xml.old
```

2.Back-up existing *xorg.conf*:
```
sudo mv /etc/X11/xorg.conf /etc/X11/xorg.conf.old
```

3.Create new *xorg.conf*
```
sudo nvidia-xconfig
```

4.edit created *xorg.conf*;
```
gksudo gedit /etc/X11/xorg.conf
```

Add or change in the "Monitor" section:

```
HorizSync       28.0 - 83.0
VertRefresh     56.0 - 75.0
```

**Edit:**
For others than OP,do not copy these values.Search for the correct values in your monitor manual.

5.Close file saving changes.Reboot or restart Xserver.

6.post if it works;resolutions should be available in 

```
nvidia-settings
```
