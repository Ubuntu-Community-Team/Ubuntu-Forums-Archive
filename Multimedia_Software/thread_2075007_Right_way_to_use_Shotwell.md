### Post by JanneM on 2012-10-22
Hi, I'm a long-time F-spot user that switched to Shotwell last year. It's not been very smooth, unfortunately, and I can't seem to figure out how to actually use Shotwell correctly. It's quite likely that I'm simply thinking about the whole thing wrong in one way or another, so I'd like to hear what workflow it's actually meant to support.

Here is what I try to do currently, and what the problems are:

* I import files from the camera; often I have both a RAW .dng file and an in-camera produced .jpg file.

- Problem: sometimes I want to use the in-camera jpeg even though I have a raw file, but the in-camera file is not visible or findable in Shotwell even though it's imported along with the dng file. 


* I develop the raw file using Darktable (open with raw editor), and save it back to the same directory with the file name suggested by Shotwell (had to stop darktable from using its own different conventions).

- problem: the file ends up in its own event and is not associated with the original in any way.


* I open the jpeg file with gimp (open with external editor), edit and crop, then save it. 

- problem: it seems to become the visible version for the image (good!) but there is no indication that I have edited the image. I have a whole folder/event with a hundred images and I have no idea which ones are done and which ones are not.

- problem: I may sometimes want to do more than one edit using gimp and compare them, but there seems to be no way to actually do direct comparisons among versions of an image.


So, what am I doing wrong? Using external editors for raw and finishing is necessary, not optional. Would it be impossible to at least have some kind of manual tool for marking a set of images as actually being versions of the same image? And to switch between seeing just one version in the catalog; and see all the versions? Am I meant to import each image as its own event of something like that?

What I end up doing in practice is to import the files completely separate from shotwell, do all the editing and finishing, then import just the finished images to shotwell after the fact.  This is pretty suboptimal and I'm sure shotwell is meant to actually support editing workflow as well, not just be a database of finished images.
