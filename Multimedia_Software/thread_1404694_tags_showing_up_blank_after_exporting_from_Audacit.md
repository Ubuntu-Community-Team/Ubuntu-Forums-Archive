### Post by Robertjm on 2010-02-11
Strange thing happening. I edit a program in Audacity 1.39.11 and then export it as an mp3 file, filling out all the fields with information as necessary. Then I upload it to my podcast website.

We have a page which will show file name, size and running time, which is information pulled out of the tags. Recently, I notice there's a blank that shows up where the subject tag should be, followed by the size and running time tags.

I wasn't having this problem when I was using the OSX version of Audacity, but only when I switched to the linux one about three weeks ago on my other laptop.

When I download the file and check the properties in Windows Explorer (this laptop is a dual-boot machine) and go to the Details tab, all the fields I'd expect to be filled in are blank. If I download a file that I'd uploaded using the OSX version and go to properties there's information in there as expected, so something is effecting the tags.

What is odd is the file shows up OK if I open the file with iTunes and select the Get Info link, all the information is there.

UPDATE: When I was back in Ubuntu I did another export, but then opened the file in Rythmbox. Right-clicking on the file and selected Properties. The fields "Disc" and "Year" had the words UNKNOWN in there. While I had left "Disc" blank when exporting, I definitely put 2010 in for "Year". Once I put 2010 in through Rythmbox and re-uploaded the file is now visible on my website, so I'm guessing there must be a problem with the export tagging feature.
