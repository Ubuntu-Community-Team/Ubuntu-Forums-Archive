### Post by Chronon on 2010-09-24
I'm glad it works, though it indicates that something is not quite following standards.  Usually, M4A should label audio streams (usually LC-AAC) inside of an MP4 container.  AAC should normally label an unwrapped stream (similar to how FLAC can be bare or can be put into an OGG container).

Possibly, the DSi supports bare AAC streams, regardless of file extension.  In this case, renaming them was unnecessary.  Another possibility is that the converted files were already inside of an MP4 container, which makes the file extension initially incorrect (MP4 or M4A would both be valid).
