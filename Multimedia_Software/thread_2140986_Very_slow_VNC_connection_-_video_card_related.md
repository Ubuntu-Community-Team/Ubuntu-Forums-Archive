### Post by gfunkdave on 2013-05-01
Ever since upgrading to Ubuntu 12.04 I've had problems with VNCing to the machine from my Windows laptop over my LAN. There is a lag of several seconds from when I click or type something to when it is reflected on my laptop. I use the buiilt-in Vino VNC server, enabled from the Desktop Sharing app. I use TightVNC Viewer on Windows and have the quality settings set to use the highest compression, which doesn't seem to make a difference either way.

I know it's video card related because when I unplugged the HDMI cord from the computer and replaced xorg.conf with a bare-bones one, suddenly everything worked great. Unfortunately, my Ubuntu install got corrupted a while back and I reinstalled 12.04. Since the reinstallation, the bare bones xorg.conf results in the machine not booting up.

The machine in question is a little eMachines PC with an AMD Athlon LE-1640 processor and an nVidia GeForce 8200 graphics card. I am using the current "recommended" proprietary driver, which the proprietary drivers control panel applet lists as "version-current".

I'm currently using the xorg.conf file auto-generated by nvidia-xconfig.

Does anyone have any ideas?

c
