### Post by wilkenw on 2008-08-17
I am running Hardy Heron with all current updates.  If I use the web interface to my networked HP 72xx All-In-One unit, I can scan documents with no difficulty.  If I attempt to use xsane, however, the application reports that it is unable to find any attached scanner.  Xsane, however, worked properly with the initial release of Hardy Heron.  Re-installing xsane and its various components provides no relief.

This failure of xsane is the latest in a string of failures following the same pattern:  xsane works properly with an initial release of a major Ubuntu version, then is broken by subsequent updates to the OS.  Given this pattern, it strikes me that it's time for Ubuntu to review its QA processes.
