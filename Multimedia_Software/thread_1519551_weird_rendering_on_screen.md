### Post by ajgreeny on 2010-06-28
Oh dear, Oh dear, an ati 9200SE card, the same as mine!

I have tried and tried as long as I have had 10.04 on my machine to get this card to perform as it did in all previous Ubuntu versions, but not with total luck.

However, as a start I always run this card now at 16 bit colour, and had to in 9.10 as well in order to get a desktop background image with compiz running.  In 9.10 I managed to get about 8000 frames per 5secs in glxgears, but in 10.04 it is down to about 3200.

Quite by accident I also have an /etc/X11/xorg.conf file that I found somewhere from a previous version of ubuntu, where the file was needed for any x output.  Lucid does not have this file by default, but it will use the file if it exists, so perhaps it is worth your trying my version of xorg.conf, just to see if it helps.  Backup your own xorg.conf first.  If it does not help you out, just remove it and restore your old version.

```

#
# This file is automatically updated on xserver-xorg package upgrades *only*
# if it has not been modified since the last upgrade of the xserver-xorg
# package.
#
# Note that some configuration settings that could be done previously
# in this file, now are automatically configured by the server and settings
# here are ignored.
#
# If you have edited this file but would like it to be automatically updated
# again, run the following command:
#   sudo dpkg-reconfigure -phigh xserver-xorg

    Identifier     "X.org Configured"
    Screen        0  "Screen0" 0 0
    InputDevice    "Keyboard0" "CoreKeyboard"
EndSection

Section "Module"
    Load  "dri2"

    Option        "RenderAccel"        "True"
    Option        "VGAAccess"        "True"
```
