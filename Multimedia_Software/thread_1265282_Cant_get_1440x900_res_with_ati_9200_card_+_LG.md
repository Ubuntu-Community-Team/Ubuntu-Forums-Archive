### Post by tgalati4 on 2009-09-14
If you are using a standard VGA connector, then you may have trouble getting some modes to work.  Try 1600x1200 at 60 Hz:

tgalati4@tpad-Gloria7 ~ $ gtf 1600 1200 60

  # 1600x1200 @ 60.00 Hz (GTF) hsync: 74.52 kHz; pclk: 160.96 MHz
  Modeline "1600x1200_60.00"  160.96  1600 1704 1880 2160  1200 1201 1204 1242  -HSync +Vsync

Sometimes, older video chips are limited to a syncing clock speed that is an even multiple of common resolution modes such at 1600 x 1200 or 1280 x 1024.  Only in the past few years as wide screen and High Def monitors have become popular have the newer video cards supported more modes to include the wide sceen resolutions.

Sometimes the limitation is in the VGA output module or with the VGA cable.  If you have DVI then use that to get the higher resolution modes.

If this machine is dual boot with Windows XP then try the ATI proprietary driver in Windows and see if you can drive the display with its native resolution using a VGA cable.  If so, then you can be sure that the hardware and cable is capable of driving the native resolution.  If you can't do it in Windows, then you can be confident that you can't do it in linux.

You can upgrade the video card, or return the monitor for something else that you can drive the native resolution.

LCD panels driven at resolutions other than native tend to look like crap.

As you have already discovered.
