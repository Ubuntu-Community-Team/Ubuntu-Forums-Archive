### Post by jcoles on 2011-10-30
You're right, wolfen69, that manufacturers write the drivers. They write them specifically for the Windows OS. In Linux distributions this is a reverse-engineering / hacking effort. If the result only works on certain hardware, list the hardware. 
It might be good for particular PC manufacturers if Linux geeks preferred their products. My PC is an HP. HP printers always work well with Linux. But I've read that's due to some cooperation from HP.
Ubuntu Linux is good for many general-purpose functions. I use it for all my browsing, writing, financial planning, etc. But when sustained high-speed transfer of data is required, it fails. For example, USB 2.0 is supposed to support 480Mb/s (60MB/s) transfer. I'm lucky to see even half that speed. Over sustained operation, to an external HDD for example, the speed drops -- 16, 8, 4, 2 MB/s. Similarly, HD video requires sustained timely transfer of data. 
Perhaps I have consistently chosen the wrong chipsets.
