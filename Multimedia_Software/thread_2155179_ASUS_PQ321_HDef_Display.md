### Post by mike acker on 2013-06-17
the PQ321 is 3840x2160 px on a 31 inch screen for 140 px/inch

[http://www.asus.com/News/L9xTPmmMwTlPMq5l](http://www.asus.com/News/L9xTPmmMwTlPMq5l)

interesting, but: what we are looking for is higher px/inch mainly for images: a 31 inch monitor is getting to be TV size rather than desk-top

this leads to some interesting presentation questions,--  11 pt type should still present at 11pt size -- while in terms of images we would like the higher resolution, --
96 px/inch is common today on 1920x1280 monitors.  i think the so called 'retna' displays are going to something closer to 300 dpi -- more like standard quality print images

i suppose the necessary re-sizing will be done in the end-point system, pretty trivial for the puters running around these days

the re-size can't assume 96 px/inch though -- it will need to ascertain the total px in the view window and re-size to that
