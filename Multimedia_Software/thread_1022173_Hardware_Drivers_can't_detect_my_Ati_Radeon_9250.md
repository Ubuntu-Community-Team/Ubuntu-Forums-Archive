You should be using (with that xorg.conf) the open source drivers with full 2D/3D support.  You don't need to change anything.  If you want, you may further tweak the driver; typing "man radeon" (no quotes) on a terminal will display the driver's manual with several options that you could use for performance tweaking.  
One thing you may want to try to speed up 2D (it only works for certain chips apparently and is mostly useful when running Compiz) is use EXA acceleration:

```
Section "Device"
	Identifier	"Configured Video Device"
        Option          "AccelMethod"     "EXA"
EndSection
```

You may also want to reduce the color depth as that sometimes speeds things up when dealing like large resolutions:

```
Section "Screen"
	Identifier	"Default Screen"
	Monitor		"Configured Monitor"
	Device		"Configured Video Device"
        DefaultDepth    16
EndSection

```

### Post by Blue Beard on 2009-01-27
I had problems with an HD3450 installed on an RS690 motherboard.

Install the radeon driver.
Use lspci -vvnn to get pci bus address.
Edit /etc/X11/xorg.cong adding the lines
Driver "radeon"
BusID  "PCI:1:0:0"
The 1:0:0 should match the address from lspci output.

### Post by Blue Beard on 2009-01-29
Your /var/log/Xorg.0.log indicates:
Intel 82845 at BusID PCI:0:2:0
ATI RV280 at BusID PCI:0:10:0

Only the Intel driver is loaded.

Manually install the radeon driver on your system
sudo apt-get install xserver-xorg-video-radeon

Make sure Driver "radeon" and BusID "PCI:0:10:0" are in your driver section of /etc/X11/xorg.conf.  Reboot to restart x-server

cat /var/log/Xorg.0.log | grep RADEON to verify is loaded.
