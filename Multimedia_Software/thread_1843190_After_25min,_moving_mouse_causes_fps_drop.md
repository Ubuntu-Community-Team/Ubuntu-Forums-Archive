### Post by boubbin on 2011-09-13
When playing 3d-games like "Savage2" or "0 AD" i get steady 60fps for the first 20-30minutes, after that, moving mouse cursor causes massive FPS drop.
This also happens in ingame menus/lobby where 3d-graphic is not present. 
If i don't move mouse i get 60 fps again.
Nvidia settings reported 70C and CPU COREs were around 60C temperature.

Restarting the game resolves the issue for the next 25mins until it begins again. 
I always play with the maximum graphic settings, and lowering them does not affect to this issue.

After the lagging starts i can still watch youtube in fullhd fullscreen or browse internet/computer normally.
Its only the ingame cursor.

This problem has been reported on [Savage2 forums]("http://forums.savage2.com/showthread.php?t=26447") with the following hardware specs:

(Users have reported NOT to have the same issue from windows, or other linux distros like Debian 6.0)

My setup:
Ubuntu 11.04 64Bit
RAM: 4GB DDR2 - 10GB swap
CPU: E6700 @ 2.66GHz
GPU: Nvidia GTX 285

Other reported #1:
Ubuntu 11.04 64Bit
RAM: 6GB DDR3 - No swap partition.
CPU: I7 920
GPU: Nvidia GTX 580

Other reported #2:
Ubuntu: ubuntu 11.04 64-bit
CPU: triple core 3 ghz
GPU: gtx 460
