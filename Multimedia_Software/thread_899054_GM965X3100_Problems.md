### Post by Alaric on 2008-08-23
Hey guys,

I'm currently running a Dell Inspiron 1525 Laptop with Ubuntu 8.04 upgraded from 7.10.  Inside this machine comes the integrated Intel GM965 (a.k.a. X3100) graphics chip.  Apparently, this card can support direct rendering and Compiz.  Unfortunately, when I first installed Ubuntu with 7.10 back in March, the best drivers for this card weren't released in the repositories yet.  I proceeded to complie from source on intel's linux dev site ([http://www.intellinuxgraphics.org/](http://www.intellinuxgraphics.org/)) and, if memory served, managed to get Planet Penguin Racer and Tremulous up and running.  Unfortunately, Nexuiz and wine were unable to run and error'd out and died (see [http://www.linuxforums.org/forum/peripherals-hardware/122564-i915-driver-troubles.html](http://www.linuxforums.org/forum/peripherals-hardware/122564-i915-driver-troubles.html)).  

Recently, I decided to revisit the issue and try to obtain full wine support so I can play my Half Life 2, WoW, and Return to Castle Wolfenstein (yes, this too errored out, even native Linux Enemy Territory) without having to reboot into XP.  I got myself into trouble immediately.  I couldn't even run PPRacer at 10+ FPS rates because I hadn't recompiled the source for the new kernel updates from Ubuntu.  I decided to go with the repo versions so I wouldn't have to worry about this anymore. 

Unfortunately, it doesn't seem to be as easy as downloading xserver-xorg-video-intel and popping a few lines into my xorg.conf, because I can't seem to get direct rendering enabled and I still haven't gotten to try compiz.  I also tried installing the stable version of the driver on intel's site, but I couldn't figure out how to compile drm and mesa because the Makefile complains about not having any targets.  Has anyone gotten this card to work?

### Post by Stemp on 2008-08-24
From [http://dri.freedesktop.org/wiki/DriTroubleshooting](http://dri.freedesktop.org/wiki/DriTroubleshooting)

> If you get:

libGL error: failed to open DRM: Operation not permitted
libGL error: reverting to (slow) indirect rendering

then you are trying to run as a user that doesn't have permission to use the DRI (root is the default allowed user). To let all users access the DRI, add the following section to your xorg.conf:

Section "DRI"
    Mode 0666
EndSection
