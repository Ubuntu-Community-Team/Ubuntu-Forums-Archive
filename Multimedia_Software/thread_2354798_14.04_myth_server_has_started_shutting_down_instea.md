### Post by JasonEde on 2017-03-06
I've a mythbuntu server that has been working fine for a while. Recently however, it has not been recording correctly all the time. It starts up correctly before the program is due to start, but the idle shutdown kicks in and it shuts itself down again and doesn't start recording. If I look on the mythwelcome screen then it's stopped showing that it's about to start recording which it used to.

For example. Tonight it should have recorded broadchurch on UK ITV starting at 21:00. It started up at 20:50 and then shut itself down just after 21:00

In the mythbackend log I can see it setting the next wakeuptime correctly. When it shuts down after not recording I can see it setting the timer for the next time that the recording is due to start so it looks like it's detecting all the programs ok. I've increased the idle time to shutdown to see if that affects the system, but it's not made any difference. It's normally been starting up 10 minutes before programs due to start and then 5 minute idle shutdown.

So far I've been unable to find any reason why it's not recording. Any suggestions on where to look?
