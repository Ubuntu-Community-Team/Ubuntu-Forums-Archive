### Post by dmakfan on 2008-04-30
I thought I had posted this yesterday, but I must have forgotten to submit it.

Anyway, after googling, I found various fixes for how to get the restricted nvidia_new drivers to work with 8.04, but the solutions varied depending on what version of Hardy Heron you were using, so I thought it might be useful to post what I did for a clean install of 8.04 LTS to get my nvidia 5200fx (without having to compile or use any tools other htan what ubuntu provides out of the box).

After the installation, under System/Admin/Hardware, it came up with nvidia_new as Enabled, but not in use.  Checking xorg.conf and glxinfo confirmed that nvidia drivers were not in use.  

I unchecked the "enabled" box, and restarted my machine.  Once it had restarted, I checked the "enabled" box again, and Ubuntu then offered to install the drivers for me.  After it was done installing, it told me to restart to get the changes.  I did, and voila:  Ubuntu booted up using the nvidia drivers.  You get the nice nvidia splash screen on startup, and xorg.conf and glxinfo say the right thing.  If I didn't restart between unchecking and checking the "enabled" box, Ubuntu didn't try and install the drivers for me.

Hope that is of some help to someone out there.
