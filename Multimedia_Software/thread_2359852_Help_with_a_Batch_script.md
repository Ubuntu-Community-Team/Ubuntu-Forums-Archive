### Post by papibe on 2017-04-28
Hi uwe-bungto.

Assuming your files have a RAW extension, like move001.RAW, something like this should work:
```
#!/bin/bash

for file in ./*.RAW
do
    echo ffmpeg -i "$file" -pix_fmt yuv422p10le -c:v prores_ks -profile:v 2 -q:v 4 -acodec pcm_s16le -ar 48000 -y -f mov "${file%RAW}"MOV
done
```
Run that and check that the output is correct. When you are sure that is want you want, remove the work **echo**, to actually execute the ffmpeg commands.

Hope it helps. Let us know how it goes.
Regards.

I change it a bit 
My input is a .MOV and so is the prores output so i made a dir called chan where the output can go
```
#!/bin/bash
mkdir chan
for file in ./*.MOV
do
    ffmpeg -i "$file" -pix_fmt yuv422p10le -c:v prores_ks -profile:v 2 -q:v 4 -acodec pcm_s16le -ar 48000 -y -f mov "chan/${filefile%MOV}"MOV

```

Some time I would like to encode to MXF format so the one below is handy
```
 
#!/bin/bash
# make a dir called mxf
mkdir mxf
for file in ./*.MOV
do
    ffmpeg -i "$file" -pix_fmt yuv422p -c:v mpeg2video -q:v 0 -qmin 1 -intra -c:a pcm_s16le -ar 48000 -y -f mxf   "mxf/${file%MOV}"MXF
done

```
