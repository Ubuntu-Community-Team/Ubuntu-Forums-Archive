I'm having a problem trying to find the right podcast solution for my needs.  My system is a home server box that I've connected to my TV, essentially an HTPC but I use it as a 24x7 home/file/whatever server.  I currently run XBMC as the media center application for managing my audio and video library.  

Although XBMC has the ability to subscribe to audio podcasts I control it from the web interface which can't see podcast sources properly and XBMC doesn't check individual podcasts when you've listened to them.  There is a third-party addon, but it is also inaccessible from the web interface.

My current solution is I use gpodder on my desktop machine to subscribe to podcasts and download new episodes to a shared directory on the HTPC and then use XBMC to play the podcast and manually delete it through gpodder when it's finished playing.  Although it's very little effort and simple to do, I've been able to automate so many other things that this one exception just drives me up the wall.

My best case scenario which I can't figure out how to reach is:

1) Be able to subscribe to a podcast.
2) Flag podcast entries as listened, including old entries from the time I subscribe to the podcast.
3) Tell the HTPC box to play it via the TV (HDMI) without the use of a mouse/keyboard/remote.  Web interface is my preferred method, but iPad works as a backup.
4) If the solution downloads the podcast instead of streams, the podcast will automatically be deleted after listening to it.  If it doesn't automatically delete it but does download the podcast, then the ability to just stream rather than download.

Does anyone know how I can get closer to my ultimate goal if not entirely there?  I'm not married to XBMC, but I do want to stick with Linux (which it's even extra annoying that this is incredibly easy to do with Windows or Mac), and if I replace XBMC then whatever I replace it with has to fill that void.  I haven't tried MythTV since I don't have any cable service or even over the air channels to record from but Plex won't work because while the backend works fine the frontend client will not work on my HTPC machine.
