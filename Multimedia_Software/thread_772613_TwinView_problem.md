### Post by MrHippocampus on 2008-04-28
From the output you've posted it seems Xgl is running on top of X. If you want WoW to work your going to have to disable Xgl and just run normal X. 

After a quick search of the forums, aparently doing this in a terminal should work:

touch $HOME/.config/xserver-xgl/disable

Just to explain what 'ps aux | grep X' did, 'ps aux' lists all of the running processes (or programs) on your PC, the "| grep X" sends the text output of 'ps aux' to a program called grep which will only show you lines which contain the characters you've given to grep, so 'grep X' will only output lines which contain an X. Only two of the ones shown are X servers ('/usr/bin/X and Xgl'), the other three simply contain the letter 'X'.

### Post by MrHippocampus on 2008-04-28
You can force the nvidia driver to recognise a TV if the hotplug fails. Adding the following to the device section should help:

```
Option "ConnectedMonitor" "DFP,TV"
```

That forces the driver to think that there is a DVI monitor and a TV connected.

Also, for a TV you may also need to set the following options (also in the device section).

```
Option "TVOutFormat" "SVIDEO"
Option "TVStandard" "NTSC-M"
```
The TVStandard option is dependant on where you live, [this]("http://us.download.nvidia.com/XFree86/Linux-x86_64/169.12/README/chapter-16.html") page details the different possible standards.
