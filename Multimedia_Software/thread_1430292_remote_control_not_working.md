### Post by mhgsys on 2010-03-15
btw, 
```
sudo apt-get install lirc
``` will do

I've messed and played around with lirc more then is good for a human this weekend. 
Finally got my remote working with freevo. 
[SIZE="1"](controlling the volume with my remote was a pain in the ... until I understood how to edit the lircrc file for freevo... turns out button VOLUP needed to be changed to VolUp and the command needed to be Vol+ instead of volume_up in the /etc/freevo/lircrc file)[/SIZE]

Anyway; lirc will be loaded on boot, so there's no need to start lirc with the command /etc/init.d/lirc .


After boot, just open up a terminal and type
```
irw
```

Press some buttons on the remote and see if it gives you any output.
If it doesn't work try reconfiguring lirc with sudo dpkg-reconfigure lirc
If it works you can use the info from lircd.conf to create your own .lircrc files to work with programs.

f.e 

Mplayer looks at ~/.lircrc

So you create a file in ~/ called .lircrc (sudo nano ~/.lircrc) and put in something like 
```

begin
     button = VolUp
     prog = mplayer
     config = volume +1
     repeat = 1
end

begin
    button = VolDown
    prog = mplayer
    config = volume -1
    repeat = 1
end

```

**But like I said, use the the info from lircd.conf for YOUR remote, f.e VolUp on my remote could be something like volume_up on yours...and maybe you want to use other buttons
