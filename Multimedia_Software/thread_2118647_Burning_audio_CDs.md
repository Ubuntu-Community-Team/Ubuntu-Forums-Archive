### Post by Barney-R on 2013-02-21
I'm sure this has been answered a million times already, but all I've found from a quick search is mainly very old threads advising people to do exactly the same thing I've been doing.

I'm trying to burn a (mainly) speech recording to an audio CD that will work in any player, which means .cda.

I'm aware that .cda is not an actual file type, but merely a link, but for this particular purpose, the result **must** be playable in **any** player.

Brasero converts directly from .mp3 but burns the file as .wav.

K3b insists on .wav for the input file (no problem using sound converter), but again the burned track shows as a .wav file.

How do I create a CD that shows the file as a .cda?

This is a special one-hour file with no copyright issues because the author (copyright owner) has **asked** me to burn it to CD, but it **has** to be compatible with **any** player.

No hardware or software problems. New installation of Ubuntu 12.04 (64-bit). 8-core processor. 6GB RAM. Plenty of HD space. Twin LG DVD writers no more than two years old.

### Post by evilsoup on 2013-02-22
When you insert an audio CD into a computer, it normally represents it so that it looks like a bunch of WAV files to the user. That's probably what's happening here.
