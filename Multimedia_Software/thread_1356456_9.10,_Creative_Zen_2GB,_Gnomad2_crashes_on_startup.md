[FONT=Courier 10 Pitch]> **geoffd8135 said:**
> I also got gnomad2 crashes with a Creative Zen.

Thank you for this geoff.

I'm going to revive my old thread as I have now solved this problem, and from the number of views there seems to be a substantial amount of interest in this issue.

First off, there is something wrong with the gnomad2 package in the Ubuntu repositories for 9.10 and 10.04 (maybe other versions as well, but these are the only two editions I have tried installing under). I don't know what the issue is, and I don't care, but here is how you can install gnomad2 so that it works correctly:

!. Uninstall gnomad2 from your system (sudo apt-get remove gnomad2). Clean out any left over dependencies (sudo apt-get autoremove).

2. Open a terminal and CD to your desktop (cd ~/Desktop (or wherever you want to download the source files)). Attempt to download the source files for gnomad2 (this will most likely fail the first time) using the command "apt-get source gnomad2" (note "apt-get" not "sudo apt-get", you don't need root permissions here as we're not installing anything. Using the sudo command will cause permissions problems later when trying to delete the source files).

3. If the attempt to download the source files fails because of dependencies, note the required dependencies listed and install them (sudo apt-get install *whatever was listed here*).

4. When you have downloaded all necessary dependencies run the command "apt-get source gnomad2" again. This will download the source files for gnomad2 to whatever directory you are currently in (i.e. they will download to your desktop if you CD'd to ~/Desktop as I recommended in step 2).

5. CD into the newly downloaded source directory (cd gnomad2-*something*). If you already know how to install software from source, just configure, make, and make install. You may have to read the readme if you are concerned about certain options (i.e. gnomad2, by default, installs to /usr/local instead of /usr/share). If you don't know how to install from source, it isn't that hard. Use the command "./configure" to run a configuration script. After a moment this will finish. When it does type "make", wait for that to finish, then type "sudo make install". If you see warnings anywhere along the way, that is perfectly normal and nothing to worry about. Continue on with the install.

6. Assuming there were no errors, gnomad2 installed successfully! Delete the files and directs apt-get downloaded, and you're finished![/FONT]

### Post by netlaptop on 2011-07-27
I use Gnomad2 to transfer stuff to my Creative Zen. On my Ubuntu 10.04, the problem is that Gnomad2 quits itself and says something like that:

Device 0 (VID=041e and PID=4157) is a Creative ZEN.

PTP_ERROR_IO: Trying again after re-initializing USB interface
Queried Creative ZEN
Segmentation fault
Here is my work around:

1- Remove libmtp8

First, just install Gnomad2 on your Ubuntu 10.04 (if you have not installed Gnomad2 yet), then go to Synaptic Package Manager, remove “libmtp8&#8243;.

Then downl oad and reinstall libmtp8 from Karmic (go to this link and do wnload the .deb file)

[http://packages.ubuntu.com/karmic/libmtp8](http://packages.ubuntu.com/karmic/libmtp8)

2- Reinstall Gnomad2:

sudo apt-get install gnomad2

3-Connect your Creative Zen to the computer and unmount it (R-click on the device and choose Unmount), please remember unmount your creative device before you run gnomad2

4- Now start Gnomad2 with sudo privileged :

sudo gnomad2

That’s it!
