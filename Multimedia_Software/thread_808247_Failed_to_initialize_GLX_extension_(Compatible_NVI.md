### Post by clarknova on 2008-06-20
Before you installed nvidia's beta driver were you using the restricted nvidia driver that ships with Ubuntu?

I tried installing the latest nvidia video driver from their web site and experienced symptoms similar to what you have described (although no nvidia splash screen here). I eventually discovered that I had to use apt or synaptic to remove the nvidia-glx and linux-restricted-modules packages. I then reran nvidia's installer and started gdm--success. I clued in only after checking dmesg:

```
[   43.824958] NVRM: API mismatch: the client has the version 169.12, but
[   43.824961] NVRM: this kernel module has the version 71.86.04.  Please
[   43.824962] NVRM: make sure that this kernel module and all NVIDIA driver
[   43.824964] NVRM: components have the same version.
[   44.310261] NET: Registered protocol family 17

[   49.516265] NVRM: API mismatch: the client has the version 169.12, but
```

ps: If you are using other restricted drivers then you won't want to remove the linux-restriced-drivers package; you'll need a more sophisticated solution to ensure the the nvidia-provided module is loading rather than the Ubuntu-provided nvidia module.
