### Post by TheWhiteDemon on 2010-01-03
I used to have this issue the majority of the time (but not all the time) while using Firefox.  I ran across a forum post which was kind of related and a trick listed there seemed to help out.  

Disable Global Tooltips:

Run: `gconf-editor /apps/panel/global` and then uncheck tooltips_enabled

Hope this helps.

### Post by sleepee on 2010-01-04
i have the same problem and it seems to be a problem with flash support for 64-bit systems...  at least in my case..
so if you have 64 bits then that might be the cause...

i'm about to try the solution here:

[http://ubuntuforums.org/showthread.php?t=1358591&highlight=youtube+flash+64](http://ubuntuforums.org/showthread.php?t=1358591&highlight=youtube+flash+64)

ill let you know what happens..

edit:  

i tested that that solution for 64 bit flash on youtube, and it worked for the most part...
i tried out a few vids, and all the buttons worked..  volume control, play/pause, the slider, HD, etc..

BUT the one button that screwed everything up was the one that plays the video in a new window..  when i clicked on that, i got the same problem again..  all the buttons in the video would be unresponsive to my mouse clicks.  i would have to reload the page, or sometimes even restart the browser.
i think that this plugin is still in its alpha or beta stage, so that would explain why it wouldn't completely work, but at least it works for the most part.. at least on youtube..
