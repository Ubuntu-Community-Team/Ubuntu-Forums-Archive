Well might be, but its comprised of many services/plugins/modules/packages that have to be installed separately, if one of them has a problem, the rest has a problem, sometimes mission-control just hangs and cannot use other programs because it uses certain ports etc etc. The many components might allow for expandability and whatnot, but in its current state are just confusing.
When it will be stable and i will be able to transfer files over XMPP/Gmail/Yahoo and use SIP accounts with success (and have options to configure everything i want) i will consider it a viable option. But it seems that is far away. Until then i will be using Sip Communicator (and i suspect after that too if all the promises are fullfilled).

I dont know why was the decision made (removing Pidgin+Ekiga that is), but i suspect that the Ubuntu devs think that every Ubuntu user is there to beta test...
In Debian, there is Ekiga, Pidgin and Empathy preinstalled (also there is Remmina too - a remote desktop client that blows the default "Remote Desktop Viewer" out of the water + no default PulseAudio 'n Compiz - talk about common sense...).
So you can take your pick from the get-go, no installing necessary.
