### Post by BicyclerBoy on 2011-04-08
Just edit the file directly..
sudo kedit /etc/X11/xorg.conf

(kedit for kde ?  gedit for gnome)

The modes listed in the metamode list (or anywhere) need to be internally defined or user defined in a modes section.

Some of your modes listed look wrong/incorrect (AFAIK) I could be wrong.
I would set xinerama off "0" , twinview off "0" so you have separate X screens; this is best for full screen TV video on big screen (IMO).
Separate X screens allows easy launch to screen & independent resolution & refresh rate & no panning nonsense..
Downside is no icons on second screen since 8.04..& no dragging to/from screens.

Delete the metamode cruft except for:
Option         "metamodes" "CRT: nvidia-auto-select +0+0".

Reboot run nvidia-settings check what you have..check you have separate X screens & configure the resolution/refresh for each.

Post your:
/var/log/Xorg.0.log    as attachment .txt file.

You could try to run nvidia-settings from terminal with sudo..this might let you save back to xorg.conf file...This is not necessary with 10.04 or 10.10..

### Post by BicyclerBoy on 2011-04-09
There is something badly wrong if you can not edit the xorg.conf file.

You need to disable xinerama..it is stopping glx & RandR modules loading..
The vertical freq range in the samsung monitor needs to be 23-73 Hz.

You could backup/rename/delete /etc/X11/xorg.conf   (11 is eleven not lower case LL)
reboot & run
nvidia-xconfig
This will recreate a new xorg.conf


The X server does not need any xorg.conf to boot up..
