### Post by Bramkaandorp on 2011-11-14
For some time now I've been looking for a feed reader with good podcast support (which includes a good downloading system).

My main gripe with Liferea is that the downloaded files aren't easily accessed after downloading. I can't double-click on the file name in Liferea to open it, so I have to go into Nautilus (Oh the humanity, he has to click three more times...). Another problem is that Liferea is primarily a news reader, and podcast support is nothing more than attachment downloading. I can't assign each podcast to a different folder (at least not that I could achieve it).

I've used Rhythmbox before this, but stopped using it because I couldn't get the show notes for any of the podcasts, but also because Rhythmbox has tag-reading issues with other files (unrelated to podcasts, so not really relevant here).

For the time being, I'm using Gpodder, which I really like, but still, having two programs for something roughly the same just doesn't seem all that logical to me, especially since some of the podcast feeds I follow also publish posts without files in-between 'casts. If only it had some news feed support, and I'd be out of trouble...

Then I saw Songbird, and I thought that, if I couldn't combine a feed reader and podcatcher, then at least I could do my podcatching in a media player again. Sadly, I found out that it isn't officially supported for Linux any more, and to wait for Nightingale would take too long (I will keep an eye on it though)

So, to get to the question, does anyone have any suggestions?

### Post by aravinthrk on 2012-01-02
Can't comment much about Podcast support and such. However, I found the latest version of Liferea (1.9.0) compiled against GTK 3 libraries in Ubuntu 11.10 has made it very efficient in terms of performance. I'm not that into podcasts. However, my testing of some audio and video files did play natively in the Liferea when it is set to use its native browser instead of the OS default browser. 

Don't give up faith on Liferea :) I think it will evolve into a much better software in the near future...
