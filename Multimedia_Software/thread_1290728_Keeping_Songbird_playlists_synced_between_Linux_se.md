### Post by tehschulman on 2009-10-13
So I'm a DJ and I use Ableton Live and iTunes under Windows to make most of my mixes. Under iTunes I keep a bunch of constantly updated playlists for different genres of music I listen to and using these playlists I can simply drag and drop songs to sample from iTunes to Ableton without a problem.

Currently I have all of my music and media sitting on my new home server, which runs Ubuntu Server and uses Songbird for music and playlist management. Now, here in lies the problem: I want my client computers, which access the media server's media via Samba, to not only be able to be able access the data but I'd also like to be able to have my Songbird playlists sync up with my music management programs on my other machines.

Initially I tried exporting my Songbird playlists from my server with a plugin and importing them into iTunes on the Windows machine, but this did not work since the paths to the files in the playlists on the server were not the same paths on my other machines (obviously). My next strategy was to install Songbird on my Windows machine and see if there wasn't an easier way to share playlists on my network. This was another false start.

Basically I want the playlist files to live on my Ubuntu server and be able to add these playlist files to programs like iTunes or Songbird on my other machines. On top of that I'd like to be able to add a song to the playlist from a client machine and have that change be reflected in the master playlists on the server. Is this possible with my current programs or do I need to figure out a different solution?

Thanks!
