### Post by raghu1111 on 2009-10-14
Using ubuntu 9.04 (64 bit)

Problem: 

The driver detects two monitors but there is signal on the second (right side) monitor. The problem exists both with or without "mirror screens" checked.

hardware : "GeForce 9800 GT (rev a2)"
Drivers : "nv" or its newer replacement "nouveau" (nvidia driver didn't work at all).

xrandr shows two monitors :
```

$ xrandr
Screen 0: minimum 320 x 200, current 1920 x 1200, maximum 3840 x 1200
DVI-0 connected 1920x1200+0+0 (normal left inverted right x axis y axis) 519mm x 324mm
   1920x1200      60.0*+
   ....
DVI-1 connected 1920x1200+0+0 (normal left inverted right x axis y axis) 519mm x 324mm
   1920x1200      60.0*+
   ....
```
Also xrandr can detect if I disconnect a monitor.

So every thing seems to be fine except that there is no signal on the second monitor (second DVI). This is not a monitor issue since it show the picture if I connect it to first DVI.

xorg.conf is pretty simple:

```

Section "Monitor"
        Identifier      "Configured Monitor"
EndSection

Section "Screen"
        Identifier      "Default Screen"
        Monitor         "Configured Monitor"
        Device          "Configured Video Device"
        DefaultDepth    24
        SubSection "Display"
                Virtual 3840 1200
        EndSubSection
EndSection

Section "Device"
        Identifier      "Configured Video Device"
        Driver  "nouveau"
EndSection

```

Please let me know if you need any more info.

### Post by raghu1111 on 2009-11-03
With help of a friend I got this working. I used his xorg.conf to start with. envyng package is useful for installing nvidia binary packages or you can search nvidia in synaptic package manager. 

I started with nvidia driver using working xorg.conf and then made a few changes using nvidia-settings. This was on ubuntu-9.04 and 9.10 upgrade worked smoothly.. great! I didn't have to change anything. Just in case, if Xserver fails to start, you can 'ssh -Y' from another machine and run nvidia-settings remotely.
