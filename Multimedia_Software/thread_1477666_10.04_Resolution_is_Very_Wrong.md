### Post by pbhj on 2010-05-27
My older system has the resolution all wrong - it appears that whatever was reading the EDID code from the monitor before isn't doing it now or is messing it up. Also the nouveau Nvidia driver (not from Nvidia, for their legacy cards) crashes often for me.

That said I'm using xrandr to switch the resolution. Run it with no switches from a console to get the possible resolutions for your system.

I use KDE so I made a little startup script to run after KDE starts to reset the resolution/refresh rate to my liking:

~/.kde/Autostart/xrandr-fix (chmod the file to u+x or 755)

```
#!/bin/bash

xrandr --size 1152x864 --rate 75

```

HTH.
