### Post by null_pointer_us on 2010-05-15
Solution:

Open winecfg and change audio driver to OSS.


Problem:

Wine cannot find a mixer device:

```
fixme:win:EnumDisplayDevicesW ((null),0,0x32f358,0x00000000), stub!
fixme:mixer:ALSA_MixerInit No master control found on HD-Audio Generic, disabling mixer
fixme:dsalsa:IDsDriverBufferImpl_SetVolumePan (0x1412f8,0x141268): stub
```


Symptoms:

An older (~Win95) game I run though Wine has a problem with audio out.

Background music and most speech plays fine.

Sound effects are usually distorted. By this I mean that they are replaced with a sort of low, roundish popping noise; it is not loud enough to be painful, just really annoying.

Occasionally, sounds effects are terminated early, ending with the same weird noise.

Is there something I need to use to make it work fine with pulseaudio?


Configuration:

This PC has a mostly-stock setup. I have made no changes to pulse/alsa/mixer settings.

Wine is from the Lucid PPA. Normally, I would expect things to be broken in some development releases, but this game is so old that it literally uses standard wave out (w/ .wav files). I think my audio problems would have to be something more fundamental, like a Ubuntu driver bug or mixer setting.


Thank you in advance.
