### Post by bobdobbs on 2009-01-16
Finally found the cause and fixed the problem.

First I used dpkg and apt to remove the nvidia kernel modules:
apt-get remove --purge nvidia-glx*
Then I downloaded and installed the latest version of the driver from nvidias site.
I installed it and rebooted.
I was still stuck in low-graphics mode, but compiz worked, and X actualyl started without any errors.

My monitor is a Chimei CMV221D

I discovered that the video card has to get information from the monitor in order to set some settings properly.
It probes for something called an EDID.
Google tells me that the EDID, which is a binary file, should be presented in a standard format.
The CMV221D fails to present its EDID in the standard format, 
This failure result in nvidia cards being limited in what resolutions they can offer.

The fix lies in the configurability of xorg.conf.
You can put a copy of the EDID somewhere, and via xorg.conf, point to it's location.

I discovered a copy of the EDID for the chimei. 
(I'm sorry, I can't recall how, but there is a linux utility for extracting it. Of course, if the chimei can't present teh EDID in a standards compliant manner, then 'read-edid' might be useless).
I put a copy of my EDID in /etc/X11.
Then I added a line to my xorg.conf in the "Screen" section"
```

Option "CustomEDID" "DFP-0:/etc/X11/edid.bin"

```

Rebooted.
(although I probably only needed to restart X.

Bingo! It works!
I now have high resolutions and compiz working.

Ok - there's at least one caveat here:
I understand that if you download the package from nvidia drectly, and install it, problems follow when you recompile or upgrade your kernel.
It seems you will be dropped back to low-res.
This can apparently be solved running the installer package file  provided. by nvidia again.

Attached is my xorg.conf, in case it might be useful.
Bear in mind that it can be further optimised. This is pretty minimal.
```

Section "ServerLayout"
    Identifier     "Layout0"
    Screen      0  "Screen0"
    InputDevice    "Keyboard0" "CoreKeyboard"
    InputDevice    "Mouse0" "CorePointer"
EndSection

Section "Files"
EndSection

Section "Module"
    Load           "dbe"
    Load           "extmod"
    Load           "type1"
    Load           "freetype"
    Load           "glx"
EndSection

Section "InputDevice"
    # generated from default
    Identifier     "Mouse0"
    Driver         "mouse"
    Option         "Protocol" "auto"
    Option         "Device" "/dev/psaux"
    Option         "Emulate3Buttons" "no"
    Option         "ZAxisMapping" "4 5"
EndSection

Section "InputDevice"
    # generated from default
    Identifier     "Keyboard0"
    Driver         "kbd"
EndSection

Section "InputDevice"
    # generated from default
    Identifier     "Keyboard0"
    Driver         "kbd"
EndSection

Section "Monitor"
    Identifier     "Monitor0"
    VendorName     "Unknown"
    ModelName      "Unknown"
    HorizSync       28.0 - 33.0
    VertRefresh     43.0 - 72.0
    Option         "DPMS"
EndSection

Section "Device"
    Identifier     "Device0"
    Driver         "nvidia"
    VendorName     "NVIDIA Corporation"
EndSection

Section "Screen"
    Identifier     "Screen0"
    Device         "Device0"
    Monitor        "Monitor0"
    Option "CustomEDID" "DFP-0:/etc/X11/edid.bin"
    DefaultDepth    24
    SubSection     "Display"
        Depth       24
    EndSubSection
EndSection


```
