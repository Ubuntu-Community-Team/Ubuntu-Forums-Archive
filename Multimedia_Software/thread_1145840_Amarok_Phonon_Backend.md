### Post by Niniel on 2009-05-02
I'm not sure if everybody knows this already, but when I contacted the developers about an issue - no track information in streaming radio like it used to be in v. 1.4 - I was told to use the Phonon-Xine backend since the GStreamer one had bugs. 
And simply after installing the Xine-BE (with Synaptic) alongside the GS-BE, this is now working. 

> Make sure that you are using the Phonon-xine backend. The Gnome Ubuntu flavor uses the Phonon-GStreamer backend by default, which has all
sorts of bugs.

-- Mark Kretschmann Amarok Developer [www.kde.org](www.kde.org) - amarok.kde.org
