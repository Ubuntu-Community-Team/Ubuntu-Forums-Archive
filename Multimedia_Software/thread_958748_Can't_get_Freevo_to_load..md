### Post by scalpel4hire on 2008-10-27
I was following the guide at [http://doc.freevo.org/FreevoAptbuntu](http://doc.freevo.org/FreevoAptbuntu) (familiar?) & got to the same error point you had.

I'm certainly no expert, but muddling through, have made some progress...


Just before the end of the install howto, there are the lines:

> cd ~/freevo-1.8.1
PYTHONPATH=~/freevo-1.8.1/lib/python2.5/site-packages:/usr/lib/python2.5/site-packages/oldxml bin/freevo

This seems to put the freevo executable into ~/freevo-1.8.1/bin

so:

> $ cd ~/freevo-1.8.1/bin
$ ./freevo setup --help

should explain the options for the setup program.


Then,  

> $./freevo setup [options]        #that apply to your location ie   

creates
 
freevo.conf (in ~/.freevo/ ) & 
freevo_config.py files (in ~/freevo-1.8.1/share/freevo/ )


Now you need to copy the contents of the freevo_config.py file into a new file called local_conf.py, in ~/.freevo/  

(This is the config file you edit, and changes then get mirrored in freevo_config.py ,I think.)

NOW freevo has the 3 config files it wants, so life should be sweet.

In my case, however, 

> ~/freevo-1.8.1/bin$./freevo 

still failed, with an error about not having consistent 'CONFIG_VERSION' entries.

In my case, I had to put the line 

> CONFIG_VERSION = 5.25

in my local_conf.py file (being careful that this entry is at the start of the line, as Python apparently demands). 
I just put it near the beginning of the file, under "LOCAL_CONF_VERSION = 5.25"

This then got freevo to launch at least, but I've yet to actually cnfigure it to view TV etc.

Hope this helps. There may be typos in here - it's late where I am at the moment - but I hope at least the gist makes sense.

### Post by ugm6hr on 2008-12-31
Is the config.py file created?

Look here: /home/jim/freevo-1.8.1/share/freevo/freevo_config.py

If not, try:

```
PYTHONPATH=~/freevo-1.8.1/lib/python2.5/site-packages:/usr/lib/python2.5/site-packages/oldxml ~/freevo-1.8.1/bin/freevo
```

Once created, you have to copy that file:

```
cp ~/freevo-1.8.1/share/freevo/freevo_config.py ~/.freevo/local_conf.py
```

Then again:

```
PYTHONPATH=~/freevo-1.8.1/lib/python2.5/site-packages:/usr/lib/python2.5/site-packages/oldxml ~/freevo-1.8.1/bin/freevo
```

This will give you an error again about versions not matching (or something like that), with a suggestion about what to do.

So, edit the local_conf file as advised:

```
gedit ~/.freevo/local_conf.py
```

Add the line (change 5.26 to whatever is suggested):
```
CONFIG_VERSION = 5.26 
```

Then to launch freevo:
```
PYTHONPATH=~/freevo-1.8.1/lib/python2.5/site-packages:/usr/lib/python2.5/site-packages/oldxml ~/freevo-1.8.1/bin/freevo
```

freevo configuration requires editing of that local_conf file - I'm working it out myself.  Setting directories and plugins seems easy.  Setting the xine engine as default is also straightforward.
