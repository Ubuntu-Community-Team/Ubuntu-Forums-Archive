EDIT: hey more info - was messing around some more, and noticed if i close my folder containing all my sounds it doesnt appear anymore...so perhaps this isnt a problem with FL or ASIO...maybe its just a case of defective user :P


EDIT 2: ahhhh i figured it out...apparently, not while in windows...but just under wine...filenames (even for folders) can cause FL to do that...not sure exactly why...but it seems that a few folders i had that contained underscores and spaces didnt want to work...after renaming the "a info" has dissapeared!

EDIT 3: um no, actually the problem came back, apparently fruity loops generates a file with a list of the available kits for it, however, the reason it appears there, is it is actually in the directory...so i deleted the one in the directory, and as far as the samples/loops that wouldnt load for me, all working no problem now! and as an added note...if your filenames or folder names are longer than the browser is wide...it wont allow the folder to open...so just stretch out the width of your browser and things should act normally :)
