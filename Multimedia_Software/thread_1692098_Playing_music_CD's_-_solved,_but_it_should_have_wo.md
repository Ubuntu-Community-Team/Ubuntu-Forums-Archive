### Post by pwabrahams on 2011-02-20
I'm working with Kubuntu 10.10 Maverick.  I inserted a music CD in the drive and expected it to play.  It didn't.  I got routed over to Amarok, which just sat there.  No complaints, just no action.  I soon discovered, however, that I could rip the CD easily enough with k3b (though flac was not available until I installed the package for it).  So I knew that the problem was not with the CD.  I still don't know why Amarok doesn't work for this very common task, but I solved the problem by installing the kscd CD player, which I found out about from a forum post.

Even though I've solved my problem, this points up something wrong with the Kubuntu packaging.  It doesn't help recruit fans for Kubuntu when a very common task like this doesn't work out of the box with the initially installed package set.  Sure, there's a lot of competition for spots on the installation CD (I assume).  Perhaps kscd is on the DVD -- the blurb on the installation page of the website suggests that if you're not looking for non-English support, the CD should suffice.  If there isn't room for kscd, then perhaps there's at least some way to alert users that it's needed, possibly as a result of mounting a music CD.  And I still don't know why Amarok gives music CDs the cold shoulder.
