### Post by BicyclerBoy on 2011-09-17
S/PDIF output was the deprecated audio method for HDMI before azalia HDA intel.
You can still output digital pass-thru' (for AC3/DTS) over HDMI HDA dependent on codec & external h/w.

Now we have the working h/w codec you can edit one of the pulse audio files so it (pa) can control & use  it.
By default alsa only enumerates the first device on each soundcard device, so PA does not see the h:0,8.
And alsa will allow (2) only. (alsa1.0.24)

There are (2) solutions:
- snd_hda_intel probe_mask in /etc/modprobe.d/*.conf
- edit /etc/pulse/default.pa

The latter is my pick..
The PA control file is:
/etc/pulse/default.pa
add at the end:
load-module module-alsa-sink device=hw:0,8

make sure this is the only "module-alsa-sink" because alsa only allows (2) & the first alsa device is always one of them by default.

Install
pavucontrol
Try to set this new output device as default etc..

Check the /proc/asound/card0/eld#0.2 file for your monitor info..

### Post by BicyclerBoy on 2011-09-17
Good question..unfortunately I have no answer..

The problem stems from alsa only allowing the number of devices that equals the number of audio streams the h/w supports.
Pulse audio allows only 2 because alsa does..

e.g. some nVidia GPUs have 4 codecs but only allow 2 streams. Alsa only enumerates the first two. The first two are almost always wrong.

The situation will have to get cleaned up at some point..

It is possible that the module probe_mask method will become standard work-around. But the probe_mask is really confusing when debugging.
