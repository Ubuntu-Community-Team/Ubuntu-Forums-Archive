### Post by a_flj_ on 2010-12-06
I'm not that much of an expert with ubuntu either, but you could try:
```
apt-get -f --fix-mising  install build-essential linux-headers-`uname -r`
```

It seems like a kernel module wants to get built when running apt, but something is missing and prevents it from building.

build-essential should contain what's needed for building stuff on Debian-based systems, linux-headers is all that most kernel modules need for installation.

In case you still don't get the problem fixed, try also installing linux-source - some modules (I haven't seen any lately) rely on the full kernel source to be installed, not just the headers.

### Post by a_flj_ on 2010-12-06
As for updating your system while keeping your home and all of your personal settings: there is a way.

Just been there for the last week or so, so I have it all fresh in my mind.

```
dpkg --get-selections > ~/package_list
```

writes all packages currently installed into the file package_list in your home dir.

You can then do a clean install without formatting your disks, which will leave your profile in place.

After the new install is over, you can:
```
sudo dpkg --get--selections < /home/<your home>/package_list
apt-get dselect-upgrade
```

Please be advised, however, that this is risky. If the bug is in some settings file in your home, it will propagate to the new installation too.

Also, the process won't keep your overall, non-user-specific settings, other than the ones about what software should be installed. In order to do this too, at least to some extent, you need to play a little with debconf - it also has something like --get-selections and --set-selections. But this I haven't done - most setting on the system I had to rebuild were the default ones.
