### Post by Yellow Pasque on 2010-07-15
```
HorizSync       28.0 - 33.0
```
In the interest of not damaging monitors, X defaults to a very safe set of values when it can't read the Monitor's EDID properly. Specifically, the default horizontal sync precludes higher resolutions. You need to edit the 'HorizSync' line in xorg.conf, preferably to values specified in your monitor's manual/support web page. Failing that, use cvt to get an idea of the maximum horizontal sync needed for a given resolution, e.g:
```
cvt -v 1680 1050 60.0Hz
```
returns
```
# 1680x1050 59.95 Hz (CVT 1.76MA) **hsync: 65.29 kHz**; pclk: 146.25 MHz
Modeline "1680x1050_60.00"  146.25  1680 1784 1960 2240  1050 1053 1059 1089 -hsync +vsync
```
So the maximum value needs to be slightly greater than 65.29
```
HorizSync       28.0 - 65.5
```

### Post by efflandt on 2010-07-15
Not sure how you installed nVidia 195 driver, but maybe there are things in your xorg.conf that are no longer supposed to be there.  I just used System > Administration > Hardware Drivers to activate it (which installs it).  And my complete default and current xorg.conf for a GT220 consists of:

```
Section "Screen"
	Identifier	"Default Screen"
	DefaultDepth	24
EndSection

Section "Module"
	Load	"glx"
EndSection

Section "Device"
	Identifier	"Default Device"
	Driver	"nvidia"
	Option	"NoLogo"	"True"
EndSection
```

That automatically worked when initially DVI connected to an old 720p (1280x720) HDTV, and currently for DVI or HDMI to 1080p HDTV (1920x1080).  But I imagine for VGA you would need to give it more info about frequencies or modelines.
