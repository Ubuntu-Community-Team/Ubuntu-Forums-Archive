### Post by markbuntu on 2009-02-20
Well, you need to have pulse running before you start jack and load the pulsejack modules. That may be part of your problem.
You also need to make yourself a member of the groups 
pulse
pulse-rt
pulse-access

That should take care of the pulse issues. For jack

Edit /etc/security/limits.conf and add or modify these lines

@audio - memlock unlimited
@audio - nice -10
@audio - rtprio 99

Ayway, there are problems with the rt kernel in Intrepid so to answer your orginal question, yes, you would be better off with Hardy for UbuntuStudio. It will be supported longer and has all the bugs worked out so the only ones that crop up are from updates. If You really need something like hardware support that Hardy does not have, you should wait for jaunty, or try Studio64.
