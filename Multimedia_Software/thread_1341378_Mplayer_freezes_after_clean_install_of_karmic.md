### Post by nstgc379 on 2009-11-29
The symptom is that MPlayer freezes up during video playback requiring me to use "alt+F2" then "killall mplayer". There may be another way to unfreeze it aside from killing mplayer, but I don't know of any. I'm more interested in keeping it from happening at all.

I don't know what the problem is. I checked the system logs and there wasn't any indicator that any services failed. I was specifically looking ALSA, OSS, and pulse, the three audio outputs I tried. I included the console print out which suggests its video problem. I assumed audio first since I've had problems with that before.

This is in a clean installation of Karmic Koala (I would like to mention that I think that is an awesome name for an OS by the way). I installed mplayer from source last week (20th) and just updated from svn an hour ago. I included the results of ./configure below.

I have tried using the XV output as well as X11, but the result was pretty much the same. The only output included in this post was for the output specified in the mplayer config file which is included. The errors all looks the same to me.

While not my main concern, I can't help but think there is a connection. Amarok will not play, it just goes to the next song, which cannot be played. I don't get any error messages and the logs show nothing. I don't have any output to post. No other audio players (VLC, Rhythem, movie player) have this problem.

I don't understand a lot of stuff about linux, but I'm kind of thinking this is related to the use of the pae headers instead of normal generic. I haven't tested that out because I don't want to do anything that would involve the kernel.

Any time you see *** in a weird place it should probably be replace with the acronym for "advance sub station".

Perhaps you could try compiling with gcc 4.3 rather than the default Karmic compiler, there have been many reports of problems when compiling with 4.4.1. Install it as follows:

```
sudo apt-get install gcc-4.3 g++-4.3
```

and when compiling MPlayer use the following option:

```
./configure --cc=gcc-4.3
```

All the best,
