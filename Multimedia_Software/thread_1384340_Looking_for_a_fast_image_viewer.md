### Post by motumboe on 2010-01-18
My hobby is photography.
I have 40,000 pictures on my HD and a lot of them are 4 MB or more in size; so I'm looking for a very fast image viewer and rotator.

My best pick at the moment is GPicView.

It is light-weight, and with it I can rotate images with a single keystroke (when you have to check and rotate 2000 images at a time, these details are important).
Qiv looks rather faster at loading images, but (perhaps I haven't got it right) when I press k or l to rotate, it doesn't actually save to disk the rotation: the rotation happens only on the screen... I need to have it saved on the disk.

Are you aware of faster image viewers?
I'm developing a clutter-based simple image viewer with a minimal set of features in order to match my needs.
My most important goal is that it must be fast.
It is very preliminary but it already loads a 23MB picture in 2,7s on my test machine (compared to the about 12s needed by gthumb, eog, gpicview). Nothing special indeed, I achieve it by using djpeg with fast options via system calls :oops: ...but it tells me that there's some space to get some significant improvements in speed.

But before dedicating my time to it, I ask you if someone has already developed a fast image viewer.

Cheers, 
 Marco
