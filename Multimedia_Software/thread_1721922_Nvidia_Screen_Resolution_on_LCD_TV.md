### Post by BicyclerBoy on 2011-04-06
If the VGA -PC input worked okay with windows then it should be possible to make it work now.
Some TVs work better with singlelink-DVI-D/HDMI connection because their VGA modes are a subset of the HDMI ones & DVI/HDMI allows CVT-RB timing.

Have you disabled overscan on the TV ? some TVs will only do this for some inputs..

Try using separate X screens setting & not twinview.
Twinview forces a meta-mode that includes both monitors/screens & forces the same refresh rate..

Post your /var/log/Xorg.0.log as an attachment.

### Post by BicyclerBoy on 2011-04-06
Your log file shows that CRT-1 (wired port) has no valid EDID so the driver makes a guess.
You can try to fix or find a work-around..

You could use a monitor section & a specified mode & possibly a custom modeline entry..
(The custom modeline should not be necessary but...)
I would use separate X screens & not twinview just to make it easier to get working..
I prefer separate screens for full screen  video playback.

There was a lot of EDID problems a few months back & the nvidia driver version or xorg/kernel all could have been the cause..

The nvidia driver you are using was not a very good release.
The x-swat ppa has nvidia driver 270 pre-compiled packages for easy consumption.

Post your TV model specifics if you want a monitor section or to try a custom modeline calculated (or use CVT)..

All these problems may evaporate with a $5 singlelink DVI-D to HDMI (19pin) cable..
