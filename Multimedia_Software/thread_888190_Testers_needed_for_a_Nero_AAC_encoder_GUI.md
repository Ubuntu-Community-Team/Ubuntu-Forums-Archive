Sorry for the cross-posting, but I don't think anyone here is visiting the PT forum often, and I guess most people who would be interested in it are here. And besides, I'm staff, so I can do whatever I want ;)

So, as you might or might not know, Nero has a free (beer) Linux version of its Nero Digital Audio AAC encoder, but it's a command-line tool only. So since I wanted to learn GUI programming with PyQt, I decided to code a GUI for it.

Here's the thread about it in PT: [http://ubuntuforums.org/showthread.php?t=888078](http://ubuntuforums.org/showthread.php?t=888078)

Why you might want to use Nero's AAC instead of Vorbis or FAAC for audio encoding despite the fact that it is non-free is a bit off-topic, but I can see at least three reasons:

* It performs better at low bitrates, which might be nice if diskspace is critical for you.

* Most hardware multimedia players like portable devices (iPods and such) and game consoles (PS3, Xbox...) can only play video in an MP4 container, for which Vorbis audio is out of the question.

* The AAC format is patented, and unlike FAAC, the Nero encoder is compliant with it. In other words, if you happen to be in a jurisdiction that recognizes software patents and want to "play it safe", it's a solution.

P.S.: the encoder is not Free, but the GUI certainly is ;)
