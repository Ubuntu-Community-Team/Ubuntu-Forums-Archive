### Post by TheFu on 2023-12-04
We've all been frustrated before.  Expertise in non-Unix OSes doesn't translate much to Linux.

As with all things, the more you do it, the easier it becomes.  In the 1990s, doing an install would take me 5+ hrs.  These days, if an install takes over about 15 minutes, I consider it too long.  The install has nothing to do with configuration beyond the initial config of the storage. Nearly everything else can be changed later.

Yes. Logitech does not support Linux most of the time.  Keep that in mind with your purchases.  Look for hardware that clearly supports ChromeOS/Chromebooks for a good chance that Linux support exists, at least for normal peripherals.

I have a 5.1 Logitech speaker system.  It uses 5 analog 3.5mm connectors and works well enough, though multi-channel output is not well supported with Linux.  It isn't just Logitech. I have Plantronics headphones with fake 7.1 surround.  Sadly, it requires special drivers on Windows to work, which means I get stereo only on Linux.  Besides that the headphones are plug-n-use.  On my other system, I have 2 Harmon Kardon speakers, using a single stereo 3.5mm jack. Those work fine.  I use the built-in motherboard audio chips.
If I want HDMI audio, that's when using the GPU audio chips happens.  For something that should be simple, having different speakers, different microphones, and different systems can make things complex.

In general, PulseAudio will usually switched to the most recent connected audio device for input or output.  For USB/digital connections, give it 5 seconds to recognize any plug/unplug changes. For analog connections, it can take longer. I usually restart the playback program after any changes. Many programs only look for the hardware to use at start up, it seems.

For controlling audio devices, I use **pavucontrol**, which isn't installed by default in Ubuntu.  I don't know if 22.04 uses pulse audio still, so that tool may not be the "new hotness". 22.04 is too new a release for me.  Lots of controls for hardware, gain, and which programs use which outputs. It can get complex.  I simplify it in the far-right tab to just 1 output device and 1 input device enabled. Helps a little with my sanity.  Then I move left one tab at a time, ensuring those specific devices are set to be used.  There are command line methods too.  These days, I have scripts to force defaults for specific devices to be used, so I don't have to spend 30 seconds in pavucontrol checking and rechecking the multitude of settings.
