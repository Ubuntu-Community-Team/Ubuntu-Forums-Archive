### Post by damien_d on 2008-12-05
Hello all,

I have recently upgraded to 8.10 (a harrowing experience to say the least!), and I have finally gotten my X-Server working again.  It worked well for 8.04, but completely broke upgrading to 8.10.

The problem is that I have had to turn off Xinerama, or x completely breaks - see below.

The questions I have are:
1. Is this a bug and should be lodged?
2. Is there a workaround?
3. What is xrandr ? Is it a replacement for xinerama, and if so, how should I be using it, and what should I expect.   Not up with this new-fangled technology thing :P

My setup:
I have (well, had) a triple-screen setup, with an NVidia 8600GT as the primary card, and a NVidia 7300GS as a secondary card.  There is also an embedded NVidia 750 graphics with the chipset.

The primary display is on the 8600 on DVI.  The screen on the left is on the 8600 on VGA, the screen on the right is on the 7300 DVI.

The Journey:
When I upgraded, it stated that it could not start the x-server.   I played around for some time, and eventually got to a a point where it attempted to start 6 times, failed, and popped up a message saying:

"The display server has been shut down 6 times in the last 90 seconds. It is likely that something bad is going on. Waiting for 2 minutes before trying again on display :0"

At which point I felt like crying.

To solve, I performed the following:
1. Attempted to get a basic x-setup by doing the following:

```

sudo dpkg-reconfigure xserver-xorg

```

Which gave me back a simple single screen on the primary monitor.

2. Re-enable the nvidia drivers

Using the GUI interface, I re-enabled the NVidia proprietary drivers (v177), and then rebooted.

System -> Administration -> Hardware drivers

3. Reboots, but thinks the primary monitor is the embedded 750a card

This was a trial and error thing.  In the end, I had found that it now thinks the primary monitor and had to put up with using it.

4. Use the nvidia-settings gui for reconfiguring the graphics setup.

This has to be a multi-step process.  For those comfortable with hacking xorg.conf files, you may do that instead.

a) enable x-servers on all the screens you wish to use.  Arrange them so that all are reachable.  Once this is done, disable the screen on the 750a
b) restart x (logging out then back in)
c) using the nvidia-settings gui, reconfigure the monitors again.

In my case, becasuse I wanted the middle as my primary monitor, not my left, I needed to set the middle as Absolute +1280+0 and the other "LeftOf" Screen 0 and the other "RightOf" screen 0.   I had to manually hack the xorg.conf file to recognise my middle monitor as the primary display.

d)  restart x again
e) Fiddle with the screen placement again.

Because I was doing it one step at a time, i had NOT enabled xinerama at this stage.

5.  Enable Xinerama and watch it crash

I then enabled Xinerama through the nvidia-settings gui.  Restarted X and got the "The display server has been shut down 6 times..." error again.  SSH'ing in from another machine.  I modified the xorg.conf file to disable xinerama:

```

Section "ServerFlags"
    # Disable xinerama
    # Option         "Xinerama" "1"
    Option         "Xinerama" "0"
EndSection

```

At which point x was happy again.   This is repeatable.
