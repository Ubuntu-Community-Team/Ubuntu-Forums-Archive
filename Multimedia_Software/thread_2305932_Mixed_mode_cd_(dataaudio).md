Many I see asking but few I see posted responses, even as far back as 2008.I hope yall got an anwser cuz this is just plain unacceptable.

Most games from the 90s /2000s used this format. All the way up to Dreamcast. Beyond this, codecs were used instead.

Some emulations allow for direct disc access but hiccup due to the way linux mounts things.For some reason, linux refuses to mount a mixed-mode disc. In some cases, you only get the audio track.
On occasion linux can be forced into reading the disc anyways. Gens SegaCD mode is an example. That app will work under wine if you have issues.Im getting mixed results with my twin bdrom drives.

Sometimes, with some apps, they wont fudge the mode from audio back to data, or something weird happens and linux wont let it.What happens is you get the audio tracks and no data tracks mounted.
dmesg says there is error reading track 1, skips it.

So heres an alternative.Unfortunately with this method, you ONLY get the data track in a useable format.
Some say CDEmu works, I am not having luck with it on Fedora. Maybe it works for ewe.
Similar windows apps dont seem to work.
