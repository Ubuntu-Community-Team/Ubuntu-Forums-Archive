### Post by forger on 2008-08-25
monitor brand/model?
also post the output of this command:
```
sudo lshw -C display
```

Take a look at the file **/var/log/Xorg.0.log** - try to locate any information about this behaviour
Finally, your xorg.conf shows that you haven't configured the driver, try configuring it using displayconfig-gtk and use the driver intel if your graphics card is intel
```
sudo apt-get install displayconfig-gtk
gksu displayconfig-gtk

```

### Post by luckylucky on 2008-08-28
I solved this problem, and here is how I did it in a nutshell.  The following won't help a newbie, but if you are familiar with hacking your machine then the following might be helpful.

First of all I found Forger's suggestion in post #8 above useful with the codes:
sudo apt-get install displayconfig-gtk
gksu displayconfig-gtk

What I then did is I booted the live CD of Ubuntu 7.10 (older version).  I booted twice to see how it configured each of my monitors.  I saved the xorg.conf files then I merged them with the existing xorg.conf file on the main machine, keeping the parts that were correct.  Now the machine works as I expect it to.
