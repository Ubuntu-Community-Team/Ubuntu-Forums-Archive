### Post by BicyclerBoy on 2013-01-29
Backup/Delete /etc/X11/xorg.conf
Then logout/login
Run nvidia-settings & configure screens..
Save to new xorg.conf if you want (but you should not need to) .

Separate X screens does not work properly with unity, same could apply to kde..I have not used kde with >1 monitor..

Try twinview.
Don't be tempted by xinerama as this will wreck havoc on openGL & VDPAU.
Twinview can cause problems with video modes/tearing & confused full screen.

There have been some users finding that driver 304 causes black screen on VDPAU video playback.

### Post by OrenG on 2013-01-30
Success!

First, I've tried to restore to the previous xorg.conf file but it 2nd monitor isn't detected now at all. Than I've updated the nvidia drivers to 310.14.

Than, I went here to post this and for completeness I've added the xrandr output:
```
DVI-I-0 disconnected (normal left inverted right x axis y axis)
VGA-0 disconnected (normal left inverted right x axis y axis)
DVI-I-1 connected 1680x1050+0+0 (normal left inverted right x axis y axis) 433mm x 271mm
   1680x1050      60.0*+
   1440x900       75.0     59.9  
   1280x1024      75.0     60.0  
   1024x768       75.0     60.0  
   800x600        75.0     60.3  
   640x480        75.0     59.9  
HDMI-0 disconnected (normal left inverted right x axis y axis)

```

Realizing that the HDMI isn't connected. I've checked and sure enough the plug wasn't inserted all the way inside... Probably since I reconnected it to my previous PC between tries.

After pluging it all the way xrandr displayed it and the other screen is both detected and working in twin.

Although I can't be sure, I believe that the problem was with the driver 304, and upgrading it to 310 fixed the problem.
Here is the procedure I used to upgrade:
```
sudo apt-add-repository ppa:ubuntu-x-swat/x-updates
sudo apt-get update
sudo apt-get install nvidia-current

```
