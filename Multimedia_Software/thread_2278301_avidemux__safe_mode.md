I'm not convinced you need CLI avconv here.

I'm not seeing that screengrab but "safe mode" in avidemux when you open a file tells me it's AVC ie.h.264.  You'd need that to edit but that's not avidemux 2.5's strong point.

However, to *convert* the file ... which you'll need to do if you rotate ... you don't need to use safe mode.  I've done this many times.  Open the file without using safe mode and try it again.
