Gtk-gnutella is far from being obsolete, as it is one of the few existing Gnutella clients today that is still actively developed and it has the most up-to-date Gnutella feature set.

The problem you're having (search pollution) can be easily overcome with suitable search filters.  Even though you may be searching for "Audio" files, you can easily add a search filter (make a named one so that you can select it at search creation time) that will drop anything which is not an MP3 file.

The version you have (0.97) supports GUESS searches, so it would scan the network much more widely than your actual Gnutella horizon.  However, these searches take time to complete (can take up to 6 hours to crawl the whole GUESS host set) unless you crank up the bandwidth dedicated to GUESS searches (Preferences -> Bandwidth).

You'll find that you have the best experience on Gnutella (regardless of the vendor) if you are patient -- leave the client running unattended 24x7.  Eventually, if the file is shared in Gnutella, it will be found.  Otherwise, it just means the file is not available at the moment.
