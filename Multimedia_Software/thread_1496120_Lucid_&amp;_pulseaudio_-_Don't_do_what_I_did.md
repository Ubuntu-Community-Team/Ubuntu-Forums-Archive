### Post by warsev on 2010-05-28
I'm not sure exactly which forum to post this, so I'll start here. My  hope is to prevent someone else from spending two days pulling his hair  as I've just done.

Two days ago I got fed up with the wacky sound system on Lucid that  seems to combine elements of pulseaudio with elements of ALSA. I  couldn't get the interactive settings right.

So I got the bright idea to completely remove pulseaudio and then  reinstall it, hopefully starting all over with new settings and all  components matched. That's what I did. Not my best idea. [COLOR=Red]_**DON'T DO IT !!!**_[/COLOR]

Upon reboot I was not able to log into my computer except by recovery  mode into a console. Attempts to log into the gnome desktop were met with either a  hang/freeze at the login screen or a blank/black screen immediately  after entering login credentials. I spent two days trying to discover what little entry in one of myriad something.conf files wasn't set right. Then it occurred to me to check the update log. 

Upon checking the log I found that the removal of pulseaudio had not just removed it and maybe a couple of ancillary libraries. The removal of pulseaudio had cascaded into the removal of _**97**_ different core system binaries, libraries, and applications. Apparently everything and anything that could possibly talk directly to pulseaudio was removed. That includes core components like the gnome desktop system, a skazillion shared libraries, and applications like GIMP, Okular, VLC, medial player, on-and-on.

What I'd done was a complete removal of pulseaudio via the synaptics package manager GUI. That's about the same as:
```
~$ sudo apt-get purge pulseaudio # DON'T ACTUALLY DO THIS!!!!
```Followed by re-install of same.

OK. So, if you're reading this because you were as foolhardy as I was, here's the way out.  First, find the log kept by the apt application. In Ubuntu it's here:
```
/var/log/apt/term.log
```Scroll through it to find the point at which you made the truly unfortunate decision to remove pulseaudio. You will find a long set of lines like "Removing <something> ...".  Copy all those lines into a new file, and then replace the "Removing " with "apt-get install ", then delete all of the " ...". Save as a bash script. Then execute the script. You will probably to use a lot of 'sudo' in the above operations to get the right permissions. The result will be that most of the stuff that was unintentionally uninstalled will be reinstalled with minimal residual problems. You should then be able to log into the gnome desktop environment and have most of your applications and libraries back.

I don't know if this behavior is a 'bug' or a 'feature' of pulseaudio or of apt. In any event, more dumb folks like me should be aware of it and how to fix the unintended consequences.
