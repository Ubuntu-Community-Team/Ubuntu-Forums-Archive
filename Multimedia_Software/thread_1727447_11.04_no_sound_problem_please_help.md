### Post by khelben1979 on 2011-04-12
So you got Intel sound chips over there. The reason why it's not working could be because the Ubuntu installation have chosen a different version of the Linux kernel, minor difference in kernel version.

You can search for kernel in [Ubuntu software Center]("http://en.wikipedia.org/wiki/Ubuntu_Software_Center") to see the alternatives for Linux kernels which are available and you can see which one you got installed at the present, just logging in to a shell from the tty terminals shows that as well (go there by ctrl + alt + f1).

From personal experience, upgrading from one kernel to another can cause a Linux system to loose the sound, and then it can be solved by reinstalling the [ALSA]("http://en.wikipedia.org/wiki/Advanced_Linux_Sound_Architecture") system, even compiling up the whole sound system can solve it.

Quick fix: _try another Linux kernel_ to see if it fixes it. If the sound setting program cannot find your sound chips, the driver does not get initilized after a reboot.
