**EDIT** - sorry about the subject.  It's 10.10, not 11.04.

My 10.10 PC doubles as a recording studio.  I have an 8x8 IO unit attached, which I also use for the system audio in out.  So I want audio to go through jackd, with pulseaudio configured to route audio to jackd.

When I start it all manually, it works beautifully.  I use something like this:

```

> jackd -dalsa -dhw:2 -r44100 -p256 -n2 -i18 -o18 -Xseq -H -M
> pactl load-module module-jack-source
> pactl load-module module-jack-sink
# Manually set the PulseAudio input/output to the jack source/sink

```I've tried to automate it, and here I run into trouble.  I've modified /etc/pulse/default.pa to this:

```

load-module module-jack-sink
load-module module-jack-source
set-default-sink jack_out
set-default-source jack_in

```I've added ~/bin/jackd:

```

#!/bin/bash
/usr/bin/jackd -dalsa -dhw:2 -r44100 -p256 -n2 -i18 -o18 -Xseq -H -M >> ~/.jackdlog

```And added ~/.config/autostart/jackd.desktop:

```

[Desktop Entry]
Type=Application
Exec=/home/family/bin/jackd
Hidden=false
NoDisplay=false
X-GNOME-Autostart-enabled=true
Name[en_GB]=JACK Server
Name=JACK Server
Comment[en_GB]=
Comment=

```But it doesn't work.  Jackd doesn't start correctly, and it seems to be because there is already one running:

```

> ps auwx | grep jackd
family   13055  0.3  4.3 106312 90080 ?        SLsl 18:11   0:23 /usr/bin/jackd -T -ndefault -T -d alsa

```I've wracked my brains, I've grepped, I've googled and I can't figure out what on earth is starting this jackd instance.  It happens on two separate user accounts, too.  It does start, and it 'works', because there is a default soundcard that it connects to - but that soundcard has no speakers or microphone connected to it.  This jackd instance then prevents the one I want to start from starting.

What is starting this, and how do I stop it?
