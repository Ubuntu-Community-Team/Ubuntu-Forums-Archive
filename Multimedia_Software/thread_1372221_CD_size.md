### Post by SuperSonic4 on 2010-01-04
It's because audio CDs are based on time whereas data CDs are based on file size. If you ripped a CD to flac you'd get much the same amount of songs on.

Audio CDs have universal compatibilty in CD players whereas, as you said, mp3 data holds more room but a lot of modern devices now support mp3 cds (ie mp3 as data - dvd players tend to have it already)

### Post by ajgreeny on 2010-01-04
When you burn an audio CD from mp3 or any other audio file it is encoded to the file type needed for an audio CD, which in effect is the same as a .wav file.  In MS Windows thet are called cda files, I think, but are basically wav.

In karmic, if you put an audio CD into the drive and open it in your file manager you will see a list of wav files, which shows the point I am making.

Some audio CD players can play mp3s direct, but most can not (yet?), so it makes sense usually to burn as audio CD not data mp3 CD.
