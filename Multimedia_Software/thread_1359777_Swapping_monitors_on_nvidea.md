### Post by chewearn on 2009-12-20
Refer to:

```

Section "ServerLayout"

```Change these:
```

Identifier     "Layout0"
Screen      0  "Screen0" 0 0

```to:

```

Identifier    "Layout0"
 Screen    0    "Screen0" 0 0
 Screen    1    "Screen1" RightOf "Screen0"

```

### Post by chewearn on 2009-12-20
My apologies, the changes I described above is only applicable for "Separate X Screen" setting only.  I gather you have "Twinview" instead.  The changes required for Twinview are:
[FONT=monospace]

[/FONT]Add these lines under Section &#8220;Device&#8221;:

```

Option "TwinViewXineramaInfoOrder" "DFP-0, CRT-0"[FONT=monospace]
[/FONT]Option "UseDisplayDevice" "DFP-0, CRT-0"[FONT=monospace]
[/FONT]Option "TwinViewOrientation" "DFP-0 LeftOf CRT-0"

```

### Post by chewearn on 2009-12-21
Yes, I did understand what you are trying to accomplish, mainly: to change the priority of VGA vs DVI.  By default, VGA always takes priority over DVI in being assigned as the primary screen (probably due to legacy reasons).  The two different settings I mentioned overwrite the default and swap it around.

Your difficulty in getting the settings to work could be due to the resulting messy xorg.conf after a few tries and errors.  At this point, I am not sure whether it would be more beneficial to just back-up your current xorg.conf, and then start from a fresh one.

For nvidia graphics, there are two ways to achieve multiple screens:
1. separate X screens
2. Twinview

It's too long to go on with the explanation in this thread, and I did make two blog posts a while back, that explain the set-ups in details.  Some careful readings will be required, see [here]("http://blog.chewearn.com/2008/10/08/nvidia-twinview-in-intrepid-ibex-beta/") and [here]("http://blog.chewearn.com/2008/10/09/nvidia-separate-x-screen-in-intrepid-ibex-beta/").
