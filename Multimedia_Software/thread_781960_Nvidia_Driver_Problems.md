### Post by ali999 on 2008-05-06
I'm not really sure why you are doing it like that...the procedure i used is much simpler and works perfectly with this card. 
First download the 171.06 driver from: [http://www.nvidia.com/object/linux_display_ia32_171.06.html](http://www.nvidia.com/object/linux_display_ia32_171.06.html). Then navigate to the directory u installed it in and give it the proper permissions by doing ```
chmod +x NVIDIA-Linux-x86-171.06.pkg1.run
``` However I am not sure if this step is neccessary because I forgot to do this and it still worked...but do it anyways. Then do ```
sudo apt-get install build-essential
``` to install the proper libc headers. This may not be neccessary for you but I kept getting an error until this was done. Then press ctrl+alt+f1 to exit gnome and login on the command prompt. Then run ```
sudo /etc/init.d/gdm stop
``` to stop the X server. Then simply do ```
sudo sh NVIDIA-Linux-x86-171.06.pkg1.run
``` to install the driver and follow the on-screen instructions. The setup process reconfigures all your xserver xorg files so you don't have to do anything. Then simply type ```
sudo gdm
``` or ```
sudo reboot
``` and you should be good to go. I used this method and got it working perfectly in 32-bit Hardy.

### Post by Zorael on 2008-05-07
Reconfiguring xserver-xorg restores it to defaults, the sole exception being whatever you picked in the guide the command itself evoked. So it doesn't detect and configure it to use your preferred driver.

Running the nvidia-xconfig command "enables" the nvidia driver, in the sense that it edits said xorg.conf file and places **Driver "nvidia"** in the videocard device section.

If X won't start with the driver enabled in xorg.conf, instead going 640x480 on you or ending up in the bulletproof menu (where you pick fix X etc), we'd need to see your X logs to see **why**. And hopefully we can then tell what caused that.

The log file is located at and named **/var/log/Xorg.0.log**. *However*, if you try to start X and it doesn't work, and you then pick fix X, *that file is overwritten when it *does* work in the fixed, failsafe mode*. Hence, unless you drop to a root shell instead of fixing X and make a backup of the log file, we're likely more interested in **/var/log/Xorg.0.log.old**; namely, the backup of Xorg.0.log.


One thing you could preemptively try is to see if the nvidia module is blacklisted.
```
zorael@sunspire:~$ grep nvidia /etc/modprobe.d/blacklist*
/etc/modprobe.d/blacklist-framebuffer:blacklist nvidiafb
```
It should *only* output the above line, listing the occurence of "nvidia" in /etc/modprobe.d/blacklist**-framebuffer**. If there are more lines output, well, that could be the answer.
