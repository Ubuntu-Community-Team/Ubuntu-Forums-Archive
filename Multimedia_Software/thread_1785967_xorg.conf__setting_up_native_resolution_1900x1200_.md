### Post by BicyclerBoy on 2011-06-19
You don't need anything in the modules section.
The nvidia driver will load all required extensions.

You need to be sure what the wired ports are called. You are using DVI & VGA but referring to (2) DVI/HDMI displays in the xorg.conf.

You can use twinview or separate X screens.
Twinview makes a meta-mode screen & pans the screen margins to make a virtual rectangle.

If you post your
/var/log/Xorg.0.log
file as an attachment it will reveal the wired port names & any other errors.

Have you run
nvidia-settings
GUI app ?

then reboot

reboot or logout/login GUI (gnome)

### Post by BicyclerBoy on 2011-06-20
Linux nvidia twinview is not as nice as windows version.
The linux flavour pans & scans with mouse at virtual margins...you need to try before you buy..

Twinview use a meta-mode the size of a virtual rectangle that include both screens.
i.e. 1680x1200 (4:3) & a 1920x1080 (crap 16:9) would become a (1680+1920) x 1200..

The extra (1200-1080) on screen 2 becomes virtual pan & scan.

Twinview forces same refresh rate on both...could be bad for full screen video outside NTSC land.
No openGL or vdpau performance losses.
Can drag between screens & have icons on 2nd screen.

Xinerama has no openGL & consigned to history AFAIK.

Separate X screen you have now but the keyboard should work on both..
Independent refresh & vsync-2-blank.
No dragging or icons on 2nd screen. Menus work fine.
