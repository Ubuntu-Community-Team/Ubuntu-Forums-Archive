### Post by Gen2ly on 2009-05-29
This is called tearing and it can happen when either the video card can't process the information quickly enough or couldn't be a problem with the nvidia drivers.  I Got the 185 (beta) drivers too but I'm not getting the same problems you're getting and your video card is plenty-good enough to handle this.  You could try to revert to the non-beta drivers but more likely I think something in your xorg.conf is at fault.  If you're doing Jaunty then a more-minimalistic xorg.conf is a better thought.  E.g., here's mine:

```
Section "ServerLayout"
    Identifier     "Layout0"
    Screen      0  "Screen0" 0 0
EndSection

Section "Monitor"
    Identifier     "Monitor0"
    Option         "DPMS"
# Since most monitor's are never calibrated i belive images on the web are
# darker then they should be but 
# Because most monitors are never calibrated images are darker than they 
# should be.  The first value is correct but too dark for most sites
# Gamma            0.86 0.85 0.87
Gamma              0.91 0.90 0.92
EndSection

Section "Device"
    Identifier     "Device0"
    Driver         "nvidia"
#    Option         "DynamicTwinView"    "False"
    Option         "NoLogo"             "True"
    Option         "NvAGP"              "2" # 0 disable, 1 nv, 2 agpgart
EndSection

Section "Screen"
    Identifier     "Screen0"
    Device         "Device0"
    Monitor        "Monitor0"
    DefaultDepth    24
    Option         "AddARGBGLXVisuals" "True" # For Compiz
    SubSection     "Display"
        Depth       24
    EndSubSection
    Option          "OnDemandVBlankInterrupts" "True"
EndSection

Section "Extensions"
    Option         "Composite" "Enable"
EndSection
```
