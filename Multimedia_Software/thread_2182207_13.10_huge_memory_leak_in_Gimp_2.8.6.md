Every time a new filter, plugin, tool, etc is used - the used memory goes up and the free memory goes down.  When the tool/filter/plugin is finished, this memory is never freed back up. What is the safest way to downgrade to the Gimp 2.8.4 that was in Ubuntu 13.04?  It was rock solid for us, but this new version is really slowing progress!

### Post by SeijiSensei on 2013-10-20
Try removing the current version from your machine with "sudo apt-get remove gimp*".  This will remove gimp itself and any dependent packages like gimp-data.  Next, visit [http://mirrors.kernel.org/ubuntu/pool/main/g/gimp/](http://mirrors.kernel.org/ubuntu/pool/main/g/gimp/) and download the 2.84 releases for gimp, gimp-data, and libgimp2.0.  Make sure you pick the packages that match your machine's architecture.  (If you don't know your architecture, run "uname -a" and look to see whether it is i386 or x86_64.  For the latter use the amd64 packages, otherwise use the i386 versions.)  Put them all in a directory, then run "sudo dpkg -i gimp* libgimp*" to install them.

There some chance that you'll encounter dependency problems, but I doubt it.  If so, you'll be warned and the installation will not go forward.

Have you filed a bug for this at Launchpad? I did a search for "memory leak" and came up with nothing.  If not, go to [https://launchpad.net/](https://launchpad.net/) and create an account for yourself.  Then go to [https://bugs.launchpad.net/ubuntu/+source/gimp](https://bugs.launchpad.net/ubuntu/+source/gimp) and file a bug report.
