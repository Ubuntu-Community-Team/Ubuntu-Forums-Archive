### Post by evilsoup on 2012-11-27
*Sometimes* re-encoding the audio can fix problems like this. I use avconv, a commandline tool, for this kind of thing; there is a GUI frontend called WinFF in the repos as well.

```
avconv -i input.ogv -c:v copy -c:a libvorbis output.ogv
```
Will produce a mostly-identical file. Again, sometimes this works, sometimes it doesn't, depending on exactly what is wrong with the sound.
