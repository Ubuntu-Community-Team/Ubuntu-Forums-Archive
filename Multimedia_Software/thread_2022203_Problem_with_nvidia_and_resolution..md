The monitor is not given its secrets. In other words, the card is not able to get the EDID data from the monitor (resolutions supported, timings, etc). It is happening very often these days over an analog connection (VGA cable).

My first suggestion would be to try to use a pure DVI or HDMI cable (not a conversion VGA to DVI). That usually solves the problem.

If that not possible, it may be possible to get the EDID data from Windows (as the manufactures drivers have it to cover these cases), and then put it on a file and hardcoded in Ubuntu.

### Post by papibe on 2012-07-12
First thing to do is calculate the correct checksum. According to the documentation the checksum works like this: the sum of all bytes must be zero.

First thing is to replace the last byte with a 0:
```
CS -> 00
```
Then using this little program over the binary data obtain using the code [here]("http://ubuntuforums.org/showpost.php?p=11331938&postcount=7"):
```
include <stdio.h>

int main()
{
  unsigned char sum;
  int i;

  sum = 0;
  for( i = 0; i < 128; i++)
  {
    unsigned char byte;

    scanf("%c", &byte);
    sum += byte;
  }
  printf("%x\n", sum);
  return 0;
}

```
I got that that your sum is 75. Then in hex:
```
100 - 75 = 8B
```
Replacing the last byte with 8B will result in a total sum of 0, thus making correct the checksum procedure.

To get the final binary edid, you could run the linked code again over the corrected text file, or use an hex editor.

Attached is the new corrected edid file (compressed).

Let us know if you have a any problems with the next steps.
