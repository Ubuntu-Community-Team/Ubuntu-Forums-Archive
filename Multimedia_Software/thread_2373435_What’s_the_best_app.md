### Post by TheFu on 2017-10-07
A chromecast is pretty picky about what it will play.  Basically, it likes VP8/ogg and h.264/AAC content.  Almost anything else, and it refuses to work.  It does handle mp3 audio too, but it doesn't handle any mpeg2, flv, mpeg4, xvid, divx, h.265 videos.  Those must be transcoded, at least on mine.

So .... the answer to the question depends on whether the multimedia content is streamed from the internet using a service provider 
  or
if the content is local and **not** in one of those formats that a chromecast supports.  

I hate to say this, but compared to other options, many are cheap too, a chromecast is a deficient device, unless the content comes from youtube.

I've had a chromecast for 3 yrs now and have used it about 10 hours total.  It isn't a very good playback device, IMHO.

I understand that only Google-Chrome (their browser) will "screencast" from a PC. I've never used it due to privacy considerations.

For local videos on my LAN, I have to use 3 devices:
* chromecast (DLNA renderer)
* Android phone/tablet (DLNA Controller)
* Plex Server (DLNA Server / Transcoder)

The Android device uses a program that lets me browse the content on the Plex server and set the "renderer" to "chromecast" for display.

Besides the google-browser, I don't know any program that will throw videos at a chromecast.  I find a Roku to be 100x better device for that sort of thing and a Raspberry Pi ($45 all in) running OSMC to be 1000x better than both those options for playback of local content.

Sorry, I'm not any more help.
