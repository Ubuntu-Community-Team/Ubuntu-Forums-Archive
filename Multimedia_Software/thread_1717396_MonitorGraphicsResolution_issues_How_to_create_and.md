### Post by WindowsXeyeL on 2011-03-29
You might be able to solve an array of resolution and graphics issues    you may have due to a hindered EDID signal. I've had great success in    creating and modifying an xorg.conf file to manually configure my    ViewSonic monitor and for the most part **utilize the full potential of   both the graphics card and the monitor**  (I'm still studying the various   options available to both, but  nevertheless it now runs infinitely   better with a crisp, vibrant  display). I did this because I have an onboard graphics card and only  have   the capacity for connecting to my monitor via VGA cable, which    apparently is notorious for inhibiting the EDID signal from HDMI/DVI    devices (EDID is configuration details that your monitor sends to your    graphics card to synchronize with it) (there are other reasons this  data   may be blocked besides using a VGA cable that I'll investigate,  but   nonetheless, **following the steps below should allow you to workaround   nearly any graphics issues you may have **(with  enough research that is)).   When your EDID is not being read for some  reason, newer versions of   Ubuntu will not be able to utilize the new  feature &#8220;kernel Modesetting&#8221;   that automatically sets the parameters  needed, and thus your system  will  safeguard itself by setting default  values that can safely be used  by  most systems (it can't guess what  your configurations are!). This  seems  to be a common issue due to the  complex nature of video systems  and  hardware incompatibility, one that  might not completely disappear  for a  while, so I just wanted to pass  along the methods I've discovered  for  quickly creating and modifying  an xorg.conf in the event your  graphics  card may not be reading the  EDID information sent by your  monitor (okay, so it may not be *super *quick, some work will have to be done, but it is well worth it in the end). **When  you create this file it becomes the primary path your  computer uses to  set up your graphics**,  and by gaining access to these  configurations you  will likely be able  to solve an array of issues that  you may have  because of a hindered  EDID signal. This can also just be a  temporary  workaround if you  absolutely must find a way to make it read  on it's  own. 

I'm no guru by any stretch of the imagination, however, I have studied    this topic intensely and Know my way around Ubuntu Well. I've also    learned a great deal about what to do and what not to do and understand    the frustrations that are often encountered in this great expanse   called  Linux, so you can trust that I've covered the bases and I   personally  assure you that** what is done here can be completely reversed  if for some  reason it doesn't work for you **(once  you start changing other options than stated here you're on your own!).  I'll try to be as  open-minded as  possible pertaining to different  display layouts (i.e. multiple monitors, screens, etc...),  however the   information here should be able to point anyone in the  right direction   who's system uses the X window system. Anyway, I've  compiled all the   steps that *eventually* worked for me (don't  worry though, this   tutorial should save you a great amount of time),  along with my  mistakes  too. If you have success with this and would  like to see  more, I plan  to make tutorials as I progress so please look  for me.

[SIZE=3][COLOR=#006400]**BACKUP:**[/COLOR][/SIZE]
Last but definitely not least, there is a good chance you will need   to  use your backup file (I quickly became good at this!) because if you    enter something wrong your system will likely boot into a command line    instead of the way you're used to. But don't fret, your system is not a    brick, you simply need to replace the corrupted xorg.conf with your    backup. To do this you need to login at the terminal with your username    and password, then change the path of each file and reboot. I usually    move the active file to another location and then move the backup  into   the active spot. The command to change a file path is: ```
sudo cp /from /to
``` (where cp is &#8220;change path&#8221; or "copy???", and /from  and   /to should be self explanatory) [COLOR=#ff0000]

// Note: Using the CP command can have unwanted results if you don't pay    attention to the destination because it overwrites existing files. [/COLOR]

So you'll type: ```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.corrupt
``` hit enter, give password if asked, and    then: ```
sudo cp /etc/X11/xorg.conf.backup /etc/X11/xorg.conf
```  hit enter, reboot and try again.```
sudo reboot
```[COLOR=DarkOrange]
[/COLOR] Once you have repeated these steps to your satisfaction and you  are comfortable  with your configuration run:   ```
nvidia-xconfig
``` and it will  create an updated file and   backup.
