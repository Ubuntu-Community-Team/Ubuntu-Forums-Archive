### Post by pytheas22 on 2008-09-05
I got a chance to google a bit and am still not positive what's wrong, but there are several avenues to explore.  First of all, let's edit xorg.conf so that we tell the video card explicitly to use GLX rendering--it should know to do that automatically, but it's obviously not for some reason.

First, backup xorg.conf in case something goes wrong:
```

cp /etc/X11/xorg.conf ~/Desktop/xorg.conf.backup
```

Then open up your xorg.conf file for editing by typing:
```

sudo gedit /etc/X11/xorg.conf
```

Inside the file, you need to change a few of the sections.  First, make the "device" section look like this:
```

Section "Device"
	Identifier	"Configured Video Device"
	Option		"UseFBDev"		"true"
        Driver          "nvidia"
EndSection
```

Also change the "screen" to:
```

Section "Screen"
	Identifier	"Default Screen"
	Monitor		"Configured Monitor"
	Device		"Configured Video Device"
        Option         "AllowGLXWithComposite" "True"
        Option         "RenderAccel" "True"
        Option         "AddARGBGLXVisuals" "True"
EndSection
```

Then logout of X and log back in, and see if this has made a difference.

If for some reason X doesn't start at all, you can revert to your original configuration by pressing control-alt-F2, which will bring you to a terminal where you can log in.  After logging in, you can run this command to restore the default xorg.conf:
```

sudo cp ~/Desktop/xorg.conf.backup /etc/X11/xorg.conf
```

Please let me know if this helps.

### Post by pytheas22 on 2008-09-10
Yes, at least glxinfo looks a lot healthier.  Is video playback still slow?  If not, then at least we've solved that problem.

As for the resolution, first of all, did you try changing it using the utility at System>Preferences>Screen Resolution (I assume you have but just want to check)?  If that didn't help (not surprising), you can try to force a higher resolution by adding a monitor modeline to /etc/X11/xorg.conf.  To get a modeline, first run the command:
```

xrandr --query
```

The output should list the resolution and refresh rates supported by your monitor, e.g. (for my monitor):

```
1440x900       59.9 +   75.0*    59.9  
   1280x1024      75.0     59.9  
   1280x960       59.9  
   1152x864       75.0     74.8  
   1024x768       75.1     70.1     60.0  
   832x624        74.6  
   800x600        72.2     75.0     60.3     56.2  
   640x480        75.0     72.8     66.7     60.0  
   720x400        70.1  

```

As you can see, the highest resolution supported for me is 1440x900, with refresh rates from 59.9hz-75hz.

Next, to generate the actual modeline, run the command:
```

gtf  <xresolution> <yresolution> <refresh rate>
```

So if I wanted to generate a modeline for 1440x900 resolution at 60hz, I would run:
```

gtf 1440 900 60
```

which outputs:
```

Modeline "1440x900_60.00"  106.47  1440 1520 1672 1904  900 901 904 932  -HSync +Vsync
```

(There may be a line above that in the output beginning with a # sign; you can ignore that line.)

Finally, you need to add the modeline to the "Monitor" section of xorg.conf, so that it looks like:

```
Section "Monitor"
	Identifier	"Configured Monitor"
        Modeline "1440x900_60.00"  106.47  1440 1520 1672 1904  900 901 904 932  -HSync +Vsync
EndSection
```

Make sure you generate a modeline for your monitor using the instructions above (that is, don't just copy mine), because there's a remote possibility that forcing X to use the wrong modeline could damage your monitor physically.

### Post by pytheas22 on 2008-09-11
> jonrserv@jcrserver:~$ xrandr --query
Screen 0: minimum 320 x 240, current 640 x 480, maximum 640 x 480
default connected 640x480+0+0 0mm x 0mm
   640x480        50.0* 
   320x240        51.0

That's interesting.  xrandr should return the same values regardless of the video driver being used by the card or the values of xorg.conf--xrandr queries the monitor itself to ask it which resolutions are supported; it doesn't ask the video card or driver.

I wonder if your cables are secured tightly?  A loose connection can cause the computer to think that higher resolutions aren't supported by the monitor (I know from experience).  I'd check the connection, and if possible try using a different cable.  Or backup your current xorg.conf, go back to the original one (of which you should have a backup copy on your desktop), and see if for some strange reason xrandr reports different values when using the original xorg.conf, since you had higher resolutions there.
