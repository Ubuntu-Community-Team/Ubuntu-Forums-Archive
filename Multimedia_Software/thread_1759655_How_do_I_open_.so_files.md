### Post by bouncingwilf on 2011-05-16
You do not generally open .so files, they are usually stored in /lib or /user/lib and used by programs that reference library routines from within them. They are not programs per se but shared libraries of object code linked to specific programs at runtime ( like windows DLL's).  They're dependencies of the programs that install them.


Bouncingwilf
