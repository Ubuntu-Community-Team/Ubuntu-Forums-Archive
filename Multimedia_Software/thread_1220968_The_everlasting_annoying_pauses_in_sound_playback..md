### Post by michel_iwaniec on 2009-07-23
So I have once again installed Ubuntu on my Znote 6224W laptop. This being the 64-bit 9.04 version with a reportedly improved Pulseaudio, I was hoping it would work better than a few years earlier. But actually, the opposite is true. Sound performance is worse than ever before.

As usual, sound playback pauses when system is under load. This is the case even for having too many browser windows open, which should logically not interfere with realtime tasks such as audio playback.

Is there really no solution for this problem that actually works? If anyone can give me some insight into why this problem never goes away I would be grateful. Being a programmer myself I really cannot see why this is a problem at all. It's well-known that the most disturbing user experience is glitchy audio, then comes glitchy video and only then do you deal with non-critical tasks such as loading web pages, so sound should have the highest priority of all. But with Linux, everything seems to be the other way around... it's ironic how Unix systems were way ahead in multi-tasking in their time, yet it's practically unusable for me today since the audio experience is unbearable as soon as you run any other programs along with Rhythmbox.

Granted, I had less problems when running Ubuntu on a Dell laptop from work before I started using this private laptop again, so I'm aware that the Znote 6224W is not an ideal Ubuntu laptop. But still, I can't begin to understand why a problem that has existed for so many years is so non-trivial to fix. Please enlighten me if there's some fundamental limitation in the Linux kernel I'm unaware about that causes this awful problem to persist throughout every new Ubuntu release.
