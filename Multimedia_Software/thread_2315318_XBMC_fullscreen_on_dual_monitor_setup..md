### Post by TheFu on 2016-02-27
I use kodi + plex server now.  Used xbmc for a few years before that, but wanted easier centralized library management and server-side real-time transcoding to support some old files on new hw that is notoriously picky about codec support (cough, chromecast, cough, roku ).

In Kodi, is an option to choose which monitor it runs on in the video settings. It works. I haven't tried the new kodi version that came out this week - plan to wait a month or 2 before that. The version from last fall has that option and it works. Switched to Kodi on Windows, Linux and Raspberry Pi v2 systems last June. It was great until an August update that pushed a 4.2 kernel with systemd. Had to make work-arounds on the raspberry pi for all sorts of things (remote/audio/skip buttons and skip amounts) and finally got it stable around December provided every video is transcoded to h.264 on the r-pi. I'm not willing to change just yet. On the Pi, I'm running OSMC - a specialized distro just for Kodi.

The settings stays for me with whatever the last setting was. No need for scripts, that I can see. Don't think it was required on XBMC last year either.
