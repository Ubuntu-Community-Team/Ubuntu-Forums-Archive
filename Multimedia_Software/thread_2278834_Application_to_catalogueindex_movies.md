I use Plex Media Server and text files with **ls -lR** outputs which can be grep'ed easily.
I've tried a webapp to manage this in the 1990s and found it too cumbersome to bother and ended up using a Java application - McDe's Movie Manager - but couldn't take how slow and heavy it was.  

Oh - and XBMC is no more. It has been renamed to Kodi.  There are many reasons to use Plex Media Server, provided you aren't under a 100% F/LOSS mandate.  It isn't open source, but it is free.  Also, since it is a DLNA server, it doesn't have any security. Plex is amazing as a household media server and manager. Any DLNA client can be used from any platform plus there is a nice web interface for most things.

Let me google a little to get the names of those tools I used all those years ago. Couldn't find them anymore. Guess they died. Did find this: [http://www.junauza.com/2013/01/best-movie-collection-managers-for-linux.html](http://www.junauza.com/2013/01/best-movie-collection-managers-for-linux.html) which seems to have a few options.
 
My brother-in-law wrote a little php web-app to manage his collection. It does about the same things as my ls -lR | grep solution, just with a webapp and mysql DB.

If you are interested in the details around my little script and ls-lR stuff, I'm happy to post it and samples. If you demand a GUI, don't have one for that.
