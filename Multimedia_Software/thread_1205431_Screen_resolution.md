### Post by CatKiller on 2009-07-06
Normally, resolution problems are caused by one of the monitor, the graphics card, or the graphics driver not passing the EDID information to X that lets it automatically set the correct refresh rate, and so it defaults to really low values that won't damage any monitors. You can manually set the values for your monitor by putting the VertRefresh and HorizSync ranges in your xorg.conf. You should be able to get these numbers from the manual or specifications for your monitor. These values go in the "Monitor" Section of xorg.conf.

You can edit xorg.conf with ```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.backup
gksudo gedit /etc/X11/xorg.conf
``````
Section "Monitor"
        Identifier      "Configured Monitor"
        HorizSync       aa-bb
        VertRefresh     cc-dd
EndSection

```You'll need to replace aa-dd with the values for **your** monitor.

### Post by CatKiller on 2009-07-07
OK. X is picking up the values that you've specified alright. I just had a look at your monitor's manual and 1024×768 **is** the recommended resolution for that monitor, although it is physically capable of 1152×864 and potentially 1280×1024 too.

One possibility is that you can tell X to ignore everything that your monitor tells it and just use the information that you've given it. You can do this by putting

```
        Option          "UseEDID"                       "False"
``` in the Device section.

Another option is to tell X that you want it to try 1280×1024 by putting the following in the Screen section.

```
        SubSection "Display"
                Depth           24
                Modes           "1280x1024"
        EndSubSection
```

A third option is to manually generate a modeline for 1280x1024@60Hz, giving all the timings for that mode. I've never used this method, but you'll see the syntax in other monitor resolution posts. Some people swear by it. There's an online modeline generator [here]("http://xtiming.sourceforge.net/cgi-bin/xtiming.pl"). It will look something like this and will go in the Monitor section.


```
Modeline "1280x1024@60"    114.98  1280 1312 1744 1776   1024 1045 1055 1076
```

### Post by CatKiller on 2009-07-16
> **deeaycee said:**
> there's no way to "undo" an upgrade is there?

Yes. Find the package in Synaptic that you wish to keep at the older version and select Package -> Force version to pick the older version. Then you can select Package -> Lock version to keep it there through updates. Depending on the dependencies of the package that you want to keep locked you may need to change some other packages too. When there's a new version of that package that you do want to upgrade to just remove the lock and upgrade normally.
