### Post by aakaasha on 2010-01-13
The solution can be found here: [http://www.mombu.com/gnu_linux/mandriva/t-grip-wont-rip-track-1-2909592.html]("http://www.mombu.com/gnu_linux/mandriva/t-grip-wont-rip-track-1-2909592.html")


> SOLUTION: On the green bar below the track listing, before ripping
sometimes the time displayed is not 00:00, in which case it's usually
the total time of the CD. If it's 00:00 and track 1 is selected, track
1 gets ripped. If it's anything else, track 1 (or whatever the first
selected track is), is always skipped. This was the key. If the time
displayed is *not* 00:00, then press the stop button (bottom left) and
then reselect the tracks, and then wait a second to make sure track 1
doesn't get deselected. Repeat if time is still not 00:00.
