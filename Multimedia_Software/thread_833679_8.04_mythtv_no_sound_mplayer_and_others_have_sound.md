### Post by kansei on 2008-06-18
So since we got an HDTV for the living room that has 1080p native resolution, I needed to switch to a computer with a DVI out to power it. No problem, I have a bunch of those laying around.

Instead of Debian Etch (which was on the old frontend computer and well.. takes a ton of work to get 90% of everything working, nearly impossible to get 100%) I decided to give MythBuntu 8.04 a try as I was so impressed with my Ubuntu 8.04 experience on my thinkpad. 

After spending the evening fooling around with vid card drivers and modelines to get it to output the proper resolution, I reached a trouble spot. MythTV has no sound when watching tv or recordings. When watching videos with the mythvideo plugin (just runs mplayer) things work excellently. 

I can't even run alsamixer, I'm getting the dreaded "why is my system further complicated via pulseaudio" error:
```
chris@57may-htpc:~$ alsamixer
*** PULSEAUDIO: Unable to connect: Connection refused

alsamixer: function snd_ctl_open failed for default: Connection refused
```

From running mplayer from the console, it looks like it also gets the same pulseaudio error, then tries ALSA, gets the same connection refused error, then uses OSS which is working flawlessly.

This is a dedicated home theatre computer which IMO will never ever benefit from pulseaudio, I don't care how cool the gimmicky little new features are. I disable all software mixers on the system to output basically line level audio and use my home theatre amp to do the audio control.

At this point I have a few options, none of which have panned out successfully so far:
1. Get pulseaudio off the system
2. Disable pulse audio so I can just go back to good ol' ALSA
3. Have myth use OSS since it seems to be working fine for mplayer.

A further nugget of information: when the system first had mythbuntu installed, it had two audio cards installed and I removed the PCI one to just use the onboard nforce audio. /proc/asound/modules lists some intel module.. wth is that, this is an old AMD Athlon (mobile 2500+ overclocked to 2.4ghz) system with an nvidia motherboard... intel what?

```

chris@57may-htpc:~$ cat /proc/asound/modules 
 0 snd_intel8x0
 1 snd_mpu401

```

Edit: I set mythtv to use /dev/dsp and yay at least I have sound temporarily.
