I have recently setup a dual head Xubuntu 11.10 with PCI ati and intel integreted graphics ports.

(Please see enclosed screenshot; the terminal on the left monitor is half off the edge and should appear on the right monitor, but doesn't).

### Post by prshah on 2012-05-10
OK, issue is solved, but please don't ask how; I am not certain myself.

I came across a tip to use ```
Option "ZaphodHeads" "VGA"
``` in each of the driver sections.

I added the line, enabled Xinerama and restarted lightdm. It worked!

Wondering what "ZaphodHeads" was all about, I checked /var/log/Xorg.0.log; that's when I came across this little gem```
[  1449.501] (WW) MACH64(0): Option "zaphodheads" is not used
[  1449.520] (WW) intel(1): Option "zaphodheads" is not used
```

Confused, I commented out the lines, and restarted lightdm; it still worked! I then deleted the lines and restarted lightdm; it STILL WORKED! I rebooted the computer; yes, it's still working.

There is no other change in the xorg.conf; it is exactly the same as listed above (Except the xinerama option is UNcommented now).

So, i'm not going to look at a gift horse too closely in the mouth.

Solved. God knows how.

### Post by prshah on 2012-05-10
> **prshah said:**
> Solved. God knows how.

OK, a quick "diff" has shown the difference; the option for xinerama was placed in the wrong section.

Originally, the option was placed in it's own "ServerFlags" section. That's wrong.

In the working version, it's placed with the "ServerLayout" section:```

Section "ServerLayout"
Identifier "Default Layout"
Screen 0 "screen1" 0 0
Screen 1 "screen2" RightOf "screen1"
Option "xinerama" "true"
EndSection

```

One can use either "true" or "on"; both work.
