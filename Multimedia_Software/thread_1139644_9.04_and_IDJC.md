### Post by bobince on 2009-05-02
> **trucker33377 said:**
> ALSA: cannot set period size to 2048 frames for playback

Yes, that's jackd not starting up; seems a lot of people can't get 2048 frames/period to run. 1024 is more reliable.

Try starting JACK manually using qjackctl. You can then make sure the frames-per-period and sample-rate settings are set to the values you want (usually 1024 and 44100 respectively) in the Setup window. If you save the settings here (to .jackdrc) I think jackd-as-started-by-idjc should also pick up the changes.

Incidentally I personally prefer 0.7.13 to 14, as the new listener-numbers feature causes the UI to block briefly every so often. (Either version is clearly better than the 0.7.7 bundled with Ubuntu, whose age and lack of MP3 support make it pretty useless.)

### Post by StephenF on 2009-05-04
> **trucker33377 said:**
> thanks ive got this to work with the 7.14 and man starting jack when i start Idjc I was getting the popup saying to try to start jack manually so I used the first line and changed to 2048 to 1024.. 

The other issue i still have is the stream stops at the end of each song is this what 7.13 would fix?

now this 7.14 plays ok if im on a windows OS so i dont know whats wrong.
It's not playing right because you are streaming a chained ogg file. This is one that creates a new ogg stream with new metadata for each song. Either dispense with the metadata by selecting None for the metadata source in the main window or switch to a player that can handle chained ogg like Audacious. Mplayer has been tested and works fine too but VLC is patchy since an audible one second gap is inserted into the audio (not good). Note that there are no problems with chained streams when using Icecast (the server accurately reports the metadata when using the web interface). It's the players that are the problem, and so many.

Bad players would be all the ones based on G-Streamer which is another way of saying all the Gnome media players. Hardware ogg players (yes they exist) have problems too. A large percentage of KDE players work perfectly but notably not Amarok.
