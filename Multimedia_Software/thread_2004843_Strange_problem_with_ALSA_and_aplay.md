### Post by Krupski on 2012-06-16
Hi all,

I have a small program that outputs 8 bit unsigned .WAV style data (no headers - just sound data) at 22.050 KHz. that is generated by the program.

The program writes to STDOUT and I pipe it to aplay like this:

./tick | aplay -r 22050

My strange problem is that there are all kinds of pops and clicks in the sound... BUT if I open the "Pulse Audio Volume Control" while the program is running, the noise goes away.

I verified that my code outputs clean data by graphing it... so I know that's not the problem.

Another note is that I have two different audio systems... the motherboard Analog Devices AD1882 chipset and the digital HDMI/SPDIF audio from the NVidia sound card.

Both audio systems work the same and both have the "popping" problem, so I know it's not a hardware or driver problem.

More info that I *just* discovered... I was going to try to record a snippet of the audio as a sample, but when I opened "gnome-sound-recorder" and pressed the "record" button, the noise stopped!

It seems that any program that accesses the audio system "fixes" something that makes the noise go away (and obviously something that "aplay" is NOT doing).

Any ideas? If anyone needs more hardware or software info, I'll be glad to provide it.

Thank you!
