### Post by wizardStan on 2008-10-15
Today's (yesterday's?) update to something caused X to go crazy and stop working.  Had to reset xorg, uninstall drivers, and reinstall everything from scratch.  Just so there's a record, in case anyone else has a similar problem, here's exactly what I needed to do to get my old CRT to have resolutions other than 640x480 when running dual screens.
1) Install the nvidia drivers by whatever means, either via package, or envy.
2) run "sudo nvidia-xconfig".  This may not actually be necessary.  Installing the drivers may accomplish this step automatically, but why take the chance.
3) reset X (ctrl-alt-backspace)
4) run "sudo nvidia-settings".
5) Highlight "X Server Display Configuration" and select the secondary monitor.
6) Configure, select "separate X screens", and "save to X configuration file"
7) reset X again (ctrl-alt-backspace)
At this point, if your second screen is of reasonable resolution, then you can skip straight to 14, else, carry on.
8) edit xorg.conf
9) find the Monitor section referencing your second monitor.  Mine has "Identifier     "Monitor1"" for example.
10) Add the following to it and save (this step was the key)
```
    HorizSync       31.5 - 81.0
    VertRefresh     50.0 - 75.0
    Gamma           1
    ModeLine       "640x480@60" 25.2 640 656 752 800 480 490 492 525 -hsync -vsync
    ModeLine       "640x480@72" 31.5 640 664 704 832 480 489 491 520 -hsync -vsync
    ModeLine       "640x480@75" 31.5 640 656 720 840 480 481 484 500 -hsync -vsync
    ModeLine       "800x600@56" 36.0 800 824 896 1024 600 601 603 625 +hsync +vsync
    ModeLine       "800x600@72" 50.0 800 856 976 1040 600 637 643 666 +hsync +vsync
    ModeLine       "800x600@75" 49.5 800 816 896 1056 600 601 604 625 +hsync +vsync
    ModeLine       "800x600@60" 40.0 800 840 968 1056 600 601 605 628 +hsync +vsync
    ModeLine       "832x624@75" 57.3 832 864 928 1152 624 625 628 667 -hsync -vsync
    ModeLine       "1024x768@75" 78.8 1024 1040 1136 1312 768 769 772 800 +hsync +vsync
    ModeLine       "1024x768@70" 75.0 1024 1048 1184 1328 768 771 777 806 -hsync -vsync
    ModeLine       "1024x768@60" 65.0 1024 1048 1184 1344 768 771 777 806 -hsync -vsync
    ModeLine       "1152x864@75" 108.0 1152 1216 1344 1600 864 865 868 900 +hsync +vsync
    ModeLine       "1280x1024@75" 135.0 1280 1296 1440 1688 1024 1025 1028 1066 +hsync +vsync
    ModeLine       "1280x960@60" 102.1 1280 1360 1496 1712 960 961 964 994 -hsync +vsync
    ModeLine       "1280x1024@60" 108.0 1280 1328 1440 1688 1024 1025 1028 1066 +hsync +vsync
    ModeLine       "1280x960@75" 129.9 1280 1368 1504 1728 960 961 964 1002 -hsync +vsync
    ModeLine       "1400x1050@60" 122.6 1400 1488 1640 1880 1050 1051 1054 1087 -hsync +vsync
    ModeLine       "1600x1200@65" 175.5 1600 1664 1856 2160 1200 1201 1204 1250 +hsync +vsync
    ModeLine       "1600x1200@60" 162.0 1600 1664 1856 2160 1200 1201 1204 1250 +hsync +vsync
```
11) reset X again (ctrl-alt-backspace)
12) you should now be able to choose System->Preferences->Screen Resolution on the second monitor (running it's own X server) and choose an appropriate screen resolution.  1024x768 is what I was going for.
13) Probably not needed this time, but just in case, reset X again (ctrl-alt-backspace)
14) run "sudo nvidia-settings" again
15) Highlight "X Server Display Configuration" and select the secondary monitor.
16) Reposition it as needed and check "Enable Xinerama"
17) Save to X configuration file
18) and one last time, reset X (ctrl-alt-backspace)

That is what it took to get my second monitor (an old CRT) to work with resolutions other than 640x480 and 320x240.  If I want to change my resolution again, I need to disable Xinerama, change it with the Screen Resolution, and then re-enable Xinerama, reseting X after every step.
Hopefully someone finds this information useful.  Even if they don't, I will no doubt have need to come back to it myself at some point in the future.
