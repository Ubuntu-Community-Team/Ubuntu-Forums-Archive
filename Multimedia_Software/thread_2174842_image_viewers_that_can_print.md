### Post by raydar on 2013-09-16
It must take a lot of code, or use a lot of overhead, or be onerous in some other way to include a print function in an image viewer; so many don't have it. 

Setting some elderly friends up with Ubuntu (12.04 LTS) because, especially with Unity, it's more foolproof than W!ndows, I was really surprised when they called me up and said they couldn't print any photos. I found this closed thread, [http://ubuntuforums.org/showthread.php?t=1951155](http://ubuntuforums.org/showthread.php?t=1951155) , entitled "image viewers which can print," helpful in choosing alternatives, as it lists quite a few, but unfortunately the thread gets lost on its way to the goal, recommending in the end two different applications which do NOT print: the Mirage viewer (which I like very much otherwise) and the Pinta editor (which, when it opens, looks way more complicated than a mere viewer and would scare off my elderly users). The Mirage is very fast and otherwise likeable, and I wish we could've kept it.

I wound up just making Ristretto (which was already installed in 12.04) the default viewer. It PRINTS, after all.
