### Post by ARam1024 on 2009-07-18
After I installed a fresh copy of Kubuntu 9.04, I tried to upgrade to version 180 of the nvidia proprietary drivers for my nvidia 9600M GS graphics card.  I tried to install the drivers with the proprietary hardware drivers utility several times.  Sometimes the utility said the install completed successfully.  Sometimes it wouldn't.

Whenever I would restart the computer, I would get past the splash screen, but then the screen would flicker briefly, report that the nvidia module could not be loaded, and ask me to login via the console.

Thankfully I could get back to my previous settings with:

```
sudo dpkg-reconfigure -phigh xserver-xorg
```

I've tried multiple solutions, from experimenting with other versions of drivers to editing the xorg.conf file.

I've run:

```
sudo nvidia-xconfig
```

This reported;

```
VALIDATION ERROR: Data incomplete in file /etc/X11/xorg.conf.
                  Device section "Configured Video Device" must have a Driver line.

```

I looked at the file, but it's driver line was: Driver "nvidia"

Rebooting also resulted in being dropped back into the console.  I followed instructions [here]("http://ubuntuforums.org/showthread.php?t=1206180&highlight=nvidia+xorg") to edit the xorg.config generated by nvidia-xconfig.

```
sudo lshw -businfo | grep -i display
pci@0000:01:00.0              display     GeForce 9600M GS
```

```
lspci | grep -i nvidia
01:00.0 VGA compatible controller: nVidia Corporation GeForce 9600M GS (rev a1)
```

```
sudo dkms status
nvidia, 180.44: added
```

I used this information to add the following line to xorg.conf:

```
BusID    "PCI:0000:01:00.0"
```

Rebooting resulted in device no found errors when loading from xorg.conf.

I've read lots of threads with similar issues, but I haven't been able to find a good solution to this.  Any help would be appreciated.

The computer is the [Asus M50VM-B1]("http://usa.asus.com/products.aspx?modelmenu=2&model=2383&l1=5&l2=132&l3=672&l4=0")

### Post by dagrump on 2009-07-18
Well the BusID line looks wrong. Log in on the terminal & edit xorg.conf try changing that line to; BusID    "01:00:00". 
You should run lspci at logging in to make sure that is actually correct for your hardware so I'm not sure where that addition came from. I have never had to assign a address to the card that wasn't a SLI setup.
Then a good old "sudo shutdown -r now".
I really I'm not sure that you need to designate a BusID with a single card.
