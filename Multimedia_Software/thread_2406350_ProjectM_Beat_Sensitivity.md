### Post by hubrisonics on 2018-11-19
I am using the Canonical version of projectM for PulseAudio on Ubuntu 18.04. With a small amount of research, I have resolved the known issue of invalid paths to two fonts. Now I have an issue that I have not been able to find documentation for.
 
 The value of the projectM parameter that is called “Beat Sensitivity” in “projectM Settings” is saved as “Hard Cut Sensitivity” in “config.inp.” When projectM is launched, the value of “Beat Sensitivity” is not affected by the value of “Hard Cut Sensitivity” in “config.inp.” Instead, the value of “Beat Sensitivity” always defaults to 10.00.
