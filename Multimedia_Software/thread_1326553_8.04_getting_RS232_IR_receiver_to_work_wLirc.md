### Post by AngusM on 2009-11-14
I'm having a devil of a time trying to get lirc to work with my homebrew RS232 IR receiver. I've been following [http://www.mythtv.org/wiki/Ubuntu_Serial_Lirc_Install](http://www.mythtv.org/wiki/Ubuntu_Serial_Lirc_Install), but the instructions don't seem relevant (events they expect to happen don't).
   I've run:
```
sudo setserial /dev/ttyS0 uart none
```
(according to dmesg, the serial port *is *ttyS0) and
```
sudo modprobe lirc_serial
```
and they don't give errors (and lsmod shows lirc_serial and lirc_dev are there). But whenever I try to run something like irrecord or irw, the serial port shows no indication of being read.
When I try
```
irrecord -d /dev/ttyS0 bogus
```
I get
> irrecord: could not get hardware features
irrecord: this device driver does not support the new LIRC interface
irrecord: major number of /dev/ttyS0 is 4
irrecord: LIRC major number is 61
irrecord: check if /dev/ttyS0 is a LIRC device
irrecord: could not init hardware (lircd running ? --> close it, check permissions)

lircd is not running.
```
irrecord -d /dev/lircd bogus
```
gives me
> irrecord: could not connect to unix socket /dev/lircd
irrecord: default_init(): Connection refused
irrecord: could not init hardware (lircd running ? --> close it, check permissions)
and from
```
irrecord -d /dev/lirc0 bogus
```
I get
> irrecord: could not open /dev/lirc0
irrecord: default_init(): Permission denied
irrecord: could not init hardware (lircd running ? --> close it, check permissions)
which is odd, because the /dev/lirc0 looks like this
> crw-rw-r-- 1 root root 61, 0 2009-11-14 13:58 /dev/lirc0
but
```
sudo irrecord -d /dev/lirc0 bogus
```
doesn't give me any error messages until I begin recording, and I spend 10 seconds pressing remote buttons (trust me, the remote's xmit light goes on) and then I'm told that it didn't receive anything in the last 10 seconds so it's quiting. I don't know what to try now.
