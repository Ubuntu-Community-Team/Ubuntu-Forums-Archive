### Post by Tom Morris on 2008-12-22
I've been getting nowhere with the Realtek sound card in my Acer AspireOne. The audio playback volume is very quiet - about a quarter of what it should be. I've checked all the volume controls both in GNOME and by using alsamixer, and by using the volume control on my keyboard. (I tend to use mpg123 as my audio player as I spend most of my time using mutt/vim/slrn - I may not be the typical Ubuntu user in this regard).

I recompiled the modules the other day by running
```
sudo aptitude --purge reinstall linux-sound-base alsa-base alsa-utils linux-image-`uname -r` linux-ubuntu-modules-`uname -r` libasound2[/quote]
as per [SoundTroubleshooting](https://help.ubuntu.com/community/SoundTroubleshooting). It worked for a day or two, but I'm now back to where I was a few days ago.

I've put up the [alsa-info.sh results](http://www.alsa-project.org/db/?f=f78a64e758a625be2d5a04695e7512059451a646) and here is what's in my /proc/asound/cards.

[CODE]tom@gadamer:~$ cat /proc/asound/cards 
 0 [Intel          ]: HDA-Intel - HDA Intel
                      HDA Intel at 0x38540000 irq 16
```

Anyone got any suggestions on how I can fix this? I can always just reinstall alsa using aptitude, but would prefer it if I can get it fixed.

### Post by wolfen69 on 2008-12-22
try this: 
```
Try adding this to your /etc/modprobe.d/options (or alsa-base) file:

options snd-hda-intel model=XXX

With XXX from this list:

3stack 3-jack
3stack-dig 3-jack with SPDIF I/O
6stack-dig 6-jack with SPDIF I/O
auto auto-config reading BIOS (default)
```

then reboot.

### Post by Tom Morris on 2008-12-22
I've solved the problem. I saw mention of the sound problems on [the ArchLinux Wiki page on the AspireOne](http://wiki.archlinux.org/index.php/Acer_Aspire_One). I downloaded and compiled 1.0.18a of the ALSA drivers, rebooted and it works. I see that 1.0.18 is being rolled into the 2.6.28 kernel, which I'm guessing will be rolled out in Jaunty Jackalope in April. I'm now rocking out to my MP3s again.
