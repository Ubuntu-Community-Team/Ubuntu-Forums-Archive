### Post by Norwegiandude on 2012-01-15
You've been a lot of help inobe. I'm not sure if the new driver did the trick, but having the newest drivers is always important.

However, I won't connect it to the TV all the time, so  for my use this is sufficient. I've managed to get it working every time in Twinview now, so I'll just stick with that.

So just to sum it all up (if anyone else has the same problems):

1. Connect the PC to the TV (By VGA cable)
2. Detect the other screen (in my case the TV) and choose Twinview as your way to display the screens
3. Choose the TV as your main preferred monitor, and then apply.

It should now work with twinview, and you can at least watch movies by dragging for example VLC player from one screen to the other and have it fullscreen on one of the screens.

I've had some problems with the xorg.conf file being misconfigured by changing all the settings, making the X server unable to start, so if you have that kind of troubles, remove the xorg.conf file in the terminal and make a fresh one. That should be enough.

### Post by Norwegiandude on 2012-01-15
And how to disconnect it:

1. Do NOT disconnect the VGA cable first, that's the last thing to do.
2. Click on the other screen (in my case the LG TV) and disable it in the nvidia-settings and then  click apply. The TV should go black (typically displaying "No signal") and the computer should return to it's normal state. You can now disconnect the VGA cable
3. I'm not sure if it's necessary, but i would suggest that you click the "Reset" button (after you've disconnected the VGA cable), so that the xorg.conf file is fresh when you want to connect it to a TV.

You will also need to run the nvidia-settings as root to make any changes to the xorg.conf-file and the settings as well.

All in all, I'll say this is (partially) solved :)
