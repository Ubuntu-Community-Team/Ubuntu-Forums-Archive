### Post by TwoD on 2008-05-07
I've been able to get my two monitors up and running at different resolutions.
I'm aware of two ways to do it, TwinView and Xinerama.

EDIT: Forgot to mention I'm running the latest nvidia-glx-new driver (version 169.12). Had a problem with the driver being listed as active, but not in use in the driver manager, but that went away after I used the settings below.

TwinView is made by nVidia, which should be no problem for you, and is said to perform slightly better in pure [fps benchmarks](http://www.phoronix.com/scan.php?page=article&item=387&num=1). A downside I noticed was that you're able to move the cursor and icons etc outside the smaller screen which is a bit annoying when having many icons.

Xinerama doesn't have the cursor/icon issue I mentioned above and that's what I'm using at the moment. I don't have much experience with it but it seems to work just fine now.

I was able to easily switch between Xinerama and TwinView (Xinerama option is only visible when screens are set to "Separate X scrrens") only when NOT merging the settings with my old xorg.conf file. (I run nvidia-settings with sudo or it won't be able to save, but you might as well copy/paste). Unfortunately that meant I lost my mouse and keyboard settings between switches. But on the other hand it was a lot easier to just copy that back in to xorg.conf when I knew all was well, rather than worrying about messing those settings up too.

Here are my complete xorg.conf files for both Xinerama and TwinView. Note that I'm using two LCD screens (One 4:3 on DVI and one 16:10 on VGA), so you'll obviously have to modify it to suit your screen/TV.
