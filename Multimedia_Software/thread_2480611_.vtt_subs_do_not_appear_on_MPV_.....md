### Post by TheFu on 2022-11-04
```
$ mkvmerge -o newfile.mkv   oldvideo.mp4               old_video-captions.vtt
```

I use this all the time because the way my players work is they get a file streamed and trying to get 2 files to be available over a stream doesn't work most of the time on most playback devices.  Embedding the captions (subtitles are completely different) makes it 1 file that needs to be tracked and moved.  Also, if you have 20 different languages, just add all of them to the end of the 'mkvmerge' command above.  Probably want to tell mkvmerge the specific language ... it doesn't pick up .en. or .es. or .ru.  .... or any of the other hints that humans use in filenames.  There's also 'mkvtoolnix-gui' if you like a GUI. I don't. Too slow and impossible to script.

Just for clarification.

**Subtitles** use image glyphs, not text.  Basically, little pictures for each character displayed.  By default, DVDs use subtitles so they can support any language.  Remember that unicode didn't really exist until many years after DVDs.

**Captions** use text, so SRT, VTT, ASS are all captions, technically.

There's also **teletext**, which I'm unsure of the difference between it and captions.  Probably in the way is it embedded in broadcasts. IDK.
