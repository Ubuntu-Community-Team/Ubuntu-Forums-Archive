### Post by Valeryan_24 on 2010-01-14
Hello,

I have a little question regarding Totem use on Ubuntu, nothing really wrong, but a wish that would make life easier and could be added to papercuts if it does not already exist.

It is possible to select many video or audio files and run them (right click mouse) to be read with Totem.

In Nautilus, if selected view is "icons", the first file up to the left will be read first by Totem.

If view is in list mode, it is the file on the top which is read first, etc...

Usually, in list view, I order to have the most recent files to the top of the window. Problem, with podcast shows for example, files are read in inverse chronological order (the most updated first, the latest at the end), which is not logical.

In Nautilus, you only have to click on the "date" column to invert order so in Totem all is right.

My problem is that I run many audio / video files to be opened in Totem directly from Gpodder, which is a great automatic aggregation program for podcast management.

[http://gpodder.org/](http://gpodder.org/)

In Gpodder default view is list, most recent episodes at the top, and it is not possible to invert order. So here, when I select many files, Totem plays first the most recent instead of the oldest.

In right panel of Totem we can move files one to one, but if you have a lot in your reading list, it's annoying.

Other solution is to play files individually, but for podcasts during less than 2 minutes, it makes to do the manual run very often. Or it is possible to open in Gpodder the Nautilus folder where podcasts are, have list with oldest files on top, select them and run with Totem.

So that is my question, in order to make some "clicks" economy:

**Is there, in the hidden options of Totem / Ubuntu or Gconf ... a parameter to modify to make Totem reading selected files in "inverted order" as usually, or a way to switch completely reading list order in one time ?**

I keep Totem for podcasts and videos, because one of its main features is to remember exactly where we stopped reading a file, and begin at the same place the next time we run it, it is very useful.

Thanks in advance if someone has a solution. :D
