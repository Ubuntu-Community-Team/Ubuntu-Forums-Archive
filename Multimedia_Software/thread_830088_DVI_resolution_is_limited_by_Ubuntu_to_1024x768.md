### Post by Clive McCarthy on 2008-06-15
I've seen several posts from people with screen resolution problems where the resolution is limited by Ubuntu to 1024x768, though the monitor can do better.

I have encountered this problem on computers with the Intel 945 chipset (950GMA graphics). If I install Ubuntu with the analog port (VGA) connected then Ubuntu 'likes' the display (a Samsung 245) and adopts the correct resolution (1920x1200). However, with just the DVI port connected at boot-up, Ubuntu gets nervous and limits the resolution to 1024x768. This is doubly odd because the VGA interface is not really good above 1600x1200 whereas DVI is designed for this higher resolution.

All the solutions I have seen posted involve editing sundry scripts. There seems to be multiple solutions which is frequently not a good sign.

I have been pleasantly surprised at the robustness of Ubuntu. I even managed to start an installation with a wired network connection and finish it later with a newly installed WiFi connection!

This display resolution problem needs a similarly robust fix. Is the issue with the monitor, the graphics chip-set, its driver or the OS ?
