### Post by Maelor on 2008-08-15
This was indeed a driver problem.  One of many actually.  With this 8800 GT video card, the drivers in the repos wouldn't work.  Neither would the latest binary downloaded from nVidia.

This is what worked:  Uninstalled everything nVidia, then installed driver 171.06.01 from nVidia's archives.

Specifically:

ctrl-alt-f1 then stop the gdm, then ran this:

sudo apt-get remove nvidia-glx-new && cd $home && wget [http://us.download.nvidia.com/XFree86/Linux-x86/171.06.01/NVIDIA-Linux-x86-176.06.01-pkg1.run](http://us.download.nvidia.com/XFree86/Linux-x86/171.06.01/NVIDIA-Linux-x86-176.06.01-pkg1.run) && sudo sh NVIDIA-Linux-x86-176.06.01-pkg1.run && sudo reboot

Note that at this point, the glx-new stuff was all that was on my machine.  You might want 'apt-get remove nvidia*' instead.

After the reboot, my monitor had good resolution, but it had a little warning box bouncing around saying it wasn't detected properly.  THat was fixed by lowering the refresh rate a little.

Hopefully this will help somebody else too.  Much thanks to the people who contributed to this thread:  [http://ubuntuforums.org/showthread.php?t=699693](http://ubuntuforums.org/showthread.php?t=699693)

Oh...after removing the old nvidia stuff, I also started with a newly generated xorg file, which may not have been necessary... sudo dpkg-reconfigure xserver-xorg
