okay



so as of Feb 11 2018 the morituri linux route still fine on 16.04
=======================================


```
sudo apt install python-musicbrainz2 libcdio-cdda1 libcdio-utils libcdio-cdda-dev python-gst0.10 python-gst0.10-dev python-cddb
```



THEN YOU NEED

libcdio and pycdio *to see drive*

```

wget ftp://ftp.gnu.org/pub/gnu/libcdio/libcdio-2.0.0.tar.gz
wget ftp://ftp.gnu.org/pub/gnu/libcdio/pycdio-2.0.0.tar.gz
```


and install both

```
./configure   [this one not always needed but run it anyway]
sudo make
sudo make install
```


=============


* then link given by mc4man*


```
wget http://launchpadlibrarian.net/188238106/morituri_0.2.3-1_all.deb
```

and install

```
sudo dpkg -i morituri_0.2.3-1_all.deb
```


=========


*you will need to run*

```
rip offset find
```


to find offset   [unless you already know from EAC or elsewhere]


*also*

```
rip drive analyze
```

*which produces*


```
cdparanoia can defeat the audio cache on this drive.
Adding drive cache behaviour to configuration file.
```


* then*

```
rip  cd  rip  --offset 6 --output-directory=Music
```

* or example for external drive:*

```
rip  cd -d /dev/sr1/  rip  --offset 6 --output-directory=Music
```
