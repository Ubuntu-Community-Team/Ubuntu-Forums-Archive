### Post by L4G4RT0 on 2010-12-13
I use Amarok, but I am about to change from KDE to Gnome due to better performance and being more stable. Amarok looks great (some may not agree with me), but yes, it's the best looking music player for Linux - the drawback comes from being a bit heavy, but that's because Amarok is not a simple "music player" - to play music you can really try lighter things: one of the best features is that Amarok can use a MySQL database to fasten it's performance and permits a huge number of music files without "complaints" - i've tried different players before, one of them the popular Rhythmbox and it got into my nerves with it's constant rescan. Amarok is not perfect, but to me it's the most complete music player AND music database!

An example of how good it can be, you can use a mysql query and then script the output so you can for example, get average rating for an album. This is the output of a little script:


mysql -u amarokuser -p<password> -D amarok -e "select tracks.title,albums.name,statistics.rating from tracks,albums,statistics where tracks.album=albums.id and albums.name like 'Elegy%' and statistics.url=tracks.url;"




Query the album Elegy

| Better Unborn.............................. | Elegy |......7 |
| Against Widows..............................| Elegy |......9 |
| The Orphan..................................| Elegy |......9 |
| On Rich and Poor............................| Elegy |......9 |
| My Kantele..................................| Elegy |.... 10 |
| Cares...................................... | Elegy |......8 |

(this output is not well aligned but when you do the query it is correct)

Now imagine how much scripting you can do to query your music database. In this area, Amarok is fantastic, so I vote for it as the top music player for Linux since the main drawbacks come for things like "personal taste" and not really concrete things... :) (just my 2 cents)
