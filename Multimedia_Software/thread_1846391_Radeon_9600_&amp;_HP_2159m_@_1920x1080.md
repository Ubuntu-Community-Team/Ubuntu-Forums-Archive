### Post by varunendra on 2011-09-19
AFAIK, xorg.conf is no more used by default. Although there is that X11 directory in /etc where "xorg.conf.failsafe file exists, but I think it is used only with failsafe graphics. But that's kinda "shot-in-the-dark" - sorry for that ;)

Something definite: 'You need a better driver' - you already know it, don't you?

To get one, open System Settings > Additional Drivers to see if a better driver is offered there. I hope there should be. If not, make sure "Restricted" repository is enabled in **Synaptic Package Manager > Settings > Repositories**. It is not enabled by default. So put a tick in the check box, close the repository dialogue-box and click "Reload" in synaptic. Then close synaptic and reopen "Additional Drivers" to see if a suitable driver gets listed for your graphics.

That was the easier approach. But if that doesn't help, here's the online 'how-to' for you: [https://help.ubuntu.com/community/RadeonDriver](https://help.ubuntu.com/community/RadeonDriver)

Hope I didn't *led you down the garden path with *ONLY* shot-in-the-dark guesses*. ;)

### Post by .... on 2011-09-19
> S/he probably just needs hsync/vertrefresh manually set in xorg.confMake an xorg.conf file with this command:
```
gksu gedit /etc/X11/xorg.conf
```Copy/paste:
```

Section "Monitor"
   Identifier    "Monitor0"
   HorizSync 30-82
   VertRefresh 50-75
EndSection

Section "Device"
   Identifier    "Device0" 
EndSection  

Section "Screen"     
   Identifier    "Screen0" 
   Device        "Device0"     
   Monitor       "Monitor0"
EndSection
```Save. Quit. Log out and back in.
