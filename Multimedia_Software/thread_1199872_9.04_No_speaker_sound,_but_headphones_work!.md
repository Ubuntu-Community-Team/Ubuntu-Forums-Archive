EDIT: To summarize how I did things in case someone with a similar case and laptop has this problem, this is what I did:
I went and downloaded the new ALSA driver according to [https://help.ubuntu.com/community/SoundTroubleshootingProcedure]("https://help.ubuntu.com/community/SoundTroubleshootingProcedure").
 THEN, I went to [https://answers.launchpad.net/ubuntu/+source/alsa-driver/+question/78023]("https://answers.launchpad.net/ubuntu/+source/alsa-driver/+question/78023") and followed Mark Rijckenberg's solution (post number 4).
In case it(solution) ever  gets removed and this problem still exists, this is what Mark said to do:
> Hi,

Please try this procedure:

1. copy-paste the following command into the Terminal:

gksudo gedit /etc/modprobe.d/alsa-base.conf

2. and add these lines to the end of the file:

# Keep snd-pcsp from being loaded as first soundcard
options snd-pcsp index=-2
alias snd-card-0 snd-hda-intel
alias sound-slot-0 snd-hda-intel
options snd-hda-intel model=hp-dv5
options snd-hda-intel enable_msi=1

3. Then navigate to System>Preferences>Sound and change everything to ALSA

4. reboot and retest sound

5. If sound still does not work, then replace hp-dv5 with one of the following 4 model options:

 STAC92HD71B*
   ref
   dell-m4-1
   dell-m4-2
   dell-m4-3

6. You will have to reboot and retest sound after every model option change to the alsa-base.conf file

See

[https://answers.launchpad.net/ubuntu/+source/alsa-driver/+question/75353](https://answers.launchpad.net/ubuntu/+source/alsa-driver/+question/75353)

for more info.

Mark
