### Post by wayover13 on 2011-03-20
Hi. I need recommendations for a media player that will be usable for some really dead-stupid users (at least when it comes to computers). The main uses of the media player will be: 1) to play internet radio stations like shoutcast streams; 2) to play mp3's located on the local hard drive; and 3) possibly to play last.fm or pandora streams. There won't be any need to viewing video: the machine on which the player gets installed doesn't have enough oomph for that anyway. So my need is really for a strictly audio-capable player.

I personally don't do much with media players--just play an occasional track using xmms, mplayer, or vlc. So I don't have the requisite personal experience to know which player might work best given the need for maximum simplicity in the proposed usage scenarios.

I thought vlc would be best since it seemed farily simple to play shoutcast streams using it (in your browser right-click on the stream's url and select "save link location," then select Media > "open location from clipboard" in vlc). But the experimentation I've done with creating playlists and finding media located on the local computer using vlc has been a bit of a challenge for me--a fairly experienced Linux user, apart from using media players--so I think vlc is just going to be beyond the capabilites of the target users. To put it in technical terms, I just don't think vlc will be sufficiently intuitive for my users.

Can anyone suggest another media player that might fit the criteria I'm aiming for? Which media player(s) do you think are most intuitive and might be most approachable for the technically-challenged?

Thanks,
James

### Post by SeijiSensei on 2011-03-21
[Clementine]("http://code.google.com/p/clementine-player/"), the fork of KDE's Amarok, is a nice player.  It builds a database of all the music it finds, lets you manage playlists, etc.

It's a KDE application, though.  So either you'd want to be running Kubuntu or, if you install Clementine from the repositories into GNOME, be prepared to see a lot of dependent libraries installed as well.
