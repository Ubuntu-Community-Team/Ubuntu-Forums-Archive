For the most part I use Kaffeine, which is a front-end GUI for Xine, and gmplayer. All of the players have their "quirks" so many times I will switch from one to the other if something does not run correct.

Now the problem is not the players themselves but the available codecs for each player and the interaction between the codecs. The order I use these players is usaually like this:

Kaffeine
Gmplayer
Vlc
Xine

> Kaffeine is for KDE what is the best front-end for gnome?
TNX
Well totem-xine is available instead of using totem-gmstreamer, so that would be the 'equivalent' of Kaffeine for Gnome. You do know that you can install KDE apps on Gnome is you like so it's not a problem since the package manager will install the additional kde-libs for Kaffeine to work

Now of course you can always install xine-ui, which is skinnable, for the same result without having to deal with kde-libs if space is a problem.
