### Post by doas777 on 2010-01-05
I think you are overthinking it.

1) converting an mp3 to another audio file is not reverse engineering. RE deals with breaking and disassembling application code, so that a new application that does the same thing can be created. 

2) there should be no reason that ubuntu cannot play an MP3. the LAME mp3 codec is opensource.  if there were an issue reading the file, and you were to re-encode it,  encoding problem is not circumvention, since the problem is encoding, not a DRM mechanism.

3) you can play .mov's in VLC, or in totem if you install the proper codecs. the mediubuntu repositories and the ubuntu-restricted-extras packages should be all you need for most video.

regardless it is your responsibility to vote with your wallet, and not support DRM'd technology.

### Post by ajgreeny on 2010-01-05
As I understand the situation, you will not be breaking any US laws by using any of the applications packages that are available for free download, whether covered by the gpl license or any other license.  I think the illegality only rests with any organisation that markets for gain any of the applications/codecs that are covered by a software patent.  This explains why UK magazines that supply distros on cover DVDs, never include in those distros some of the packages that would break the patent laws in US and other countries.

As an example, Linux Mint, which by default comes with all the codecs and applications needed to play mp3s, flash videos, encrypted DVDs, just about any video file you can find etc etc, will always be shipped on the magazine with none of those codecs, but the end user can still legally and very easily install them by using the standard package manager.

Don't quote me on all this because I may have got some of the detail wrong, but basically I think you are quite safe to download and use anything in any distro, what you can't do is sell any of those packages for gain in some countries.
