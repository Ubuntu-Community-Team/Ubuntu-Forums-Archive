### Post by Selanit on 2010-05-11
I've got the problem as well: any time I play a video, the system crashes, with the screen flicking back and forth between empty blackness and error text which flashes away too fast to be read easily.

However, I don't think it's a video problem, because I get the same behavior when I try to play a sound, any sound.  I think it's a problem with PulseAudio, specifically.  I've deliberately crashed my system over a dozen times now just to study those fleeting error messages, and some of them definitely mention pulse.

Here's my LSPCI -v for my audio card:


It seems to be related to this bug: 

[https://bugs.launchpad.net/ubuntu/+source/dbus/+bug/434436](https://bugs.launchpad.net/ubuntu/+source/dbus/+bug/434436)

No magic bullets in there that I can see, but it did have some instructions for getting a more verbose error log:

[https://wiki.ubuntu.com/PulseAudio/Log](https://wiki.ubuntu.com/PulseAudio/Log)

I'm going to try that and see if it tells me anything useful.

I suspect that purging PulseAudio and replacing it with esound (or whatever) would fix the problem.  I strongly suspect that that would also have undesirable effects on other parts of the system since esound was completely dropped in 10.04 in favor of PulseAudio.  I've had my share of tribulations with esound, but at least it didn't crash my system ...
