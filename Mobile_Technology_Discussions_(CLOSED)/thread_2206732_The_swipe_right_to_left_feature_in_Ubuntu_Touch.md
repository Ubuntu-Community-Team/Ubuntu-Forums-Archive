### Post by branoic on 2014-02-20
Hi!

First time I post here so if anything is weird or if this has been discussed before (I tried to search), I apologize now :)

Anyway!

I've tried out Ubuntu Touch for a while now and overall I am very impressed by a lot of the things that they've managed to do.
I do like the swipe gestures from left to right(bring up the left menu), bottom and top. But when you go backwards(swipe right to left) to the
applications that you used before. It does, at least for me, feel quite messy.
It is often that I want to go back to a previous application but it is hard to remember which the previous application
were or that there are several open applications that needs to be scrolled through.
So I usually tend to go out to the "home screen" and open the application the "normal" way.

Does anyone else feel the same? Or have problems with the current approach?

This has been bugging me and I think a better, easier and more understandable behaviour would be that when you swipe right to left with your finger a vertical bar would follow out (not all the way, but maybe the same width as the bar on the left side(when you swipe left to right)).
This bar on the right side could contain the icons of the currently open applications. With the current open application on the bottom and each older application follow upwards.
This does give:
1) Much easier, more intuitive and faster way to go back 2 or more applications in the history
2) An easy way to see how many applications that is open (If there are more open applications than screen height there could be a number on the top of the screen with remaining open applications or similar)
3) This opens up to the possibility to interact with the open applications inside another application (like holding on an icon or wripe left to right on the icon would terminate it)
Another more crazy idea with this is also
4) It could be used for multitasking with several windows (if I have two instances of firefox open and I want to switch between these) -> Example: If you swipe right to left on the firefox icon and there are several instances open it could bring up a miniature of the last screen on both (much like unity does now if you alt tab over a program with several instances)

So the bar on the left side is "safe haven" with your favourite applications that you can pin and the bar and on the right would be currently active applications.

Does it sound like a better way to interact with the phone? 
This is just a discussion and I would like to hear other peoples thoughts on both the way to work with the current implemtation and my idea 
If you feel I have missed something (or if I should try to explain it better, maybe with some pictures :)) and feel you can improve my suggestion, go ahead!

If people tend to like the suggestion better it could be worth bringing up to the Ubuntu Touch team.

Henrik

I think that the developers have got there before you. What you are discussing is called the Side Stage, I think. It already exists but is not included in Ubuntu Touch version 1.0 (based upon Ubuntu 13.10) as that is optimised for the phone form factor. The Side Stage is now being included in Ubuntu Touch version 1.5 which is being optimised for the tablet form factor and is based upon the Ubuntu 14.04 code base.
