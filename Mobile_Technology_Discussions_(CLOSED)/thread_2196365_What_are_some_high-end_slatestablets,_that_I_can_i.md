### Post by grahammechanical on 2013-12-29
What are you asking about? The hardware specifications or the restrictions put on the device by the Company providing the OS to be pre-installed and the manufacturer?

A Windows 8 tablet might have a good hardware specification but be restricted by the conditions agreed between Microsoft and the device manufacturer. I would say that the same thing applies to any Apple device.

For myself, I will wait until I can buy a Ubuntu tablet. Then I will know that Ubuntu works on the device. I know the quality of Ubuntu. It is the quality of the hardware that I will examine before I hand over my money.

It is only recently that Canonical has reached agreement for Ubuntu Touch to be pre-installed on a mobile phone device. Ubuntu Touch is still to be optimised for tablet devices. So, we have to continue waiting for a tablet product to be offered for sale with Ubuntu Touch pre-installed. Whether the hardware of these prospective Ubuntu mobile devices will be capable of connecting to a monitor remains to be seen. But Ubuntu Touch already has the capability of running a Ubuntu desktop on a monitor connected to a phone/tablet with Ubuntu Touch installed.

At the moment we install Ubuntu on desktop machines by downloading an image and burning it to a DVD. I will be very surprised if we will ever be able to install Ubuntu on mobile devices the same way we install Ubuntu on desktops and laptops. That is not at present the direction that Ubuntu development is going in.

Note this recent announcement but also notice that it is for developers only.

[http://developer.ubuntu.com/2013/12/announcing-ubuntu-and-android-dual-boot-developer-preview/](http://developer.ubuntu.com/2013/12/announcing-ubuntu-and-android-dual-boot-developer-preview/)

> [COLOR=#333333][FONT=Ubuntu Beta]Dual boot is not part of the regular Ubuntu release.[/FONT][/COLOR]

Will it ever be? Who knows.

[https://wiki.ubuntu.com/Touch/DualBootInstallation](https://wiki.ubuntu.com/Touch/DualBootInstallation)
