### Post by grahammechanical on 2013-11-29
As far as I know the situation is the same as it always has been.

1) Canonical developers are running Ubuntu on 4 Nexus reference devices.
2) Community developers are free to port the Ubuntu code to any device they wish to port the code to. It is open source after all.
3) Canonical will work with any Original Equipment Manufacturer (OEM) to help them release a Ubuntu phone/tablet product.
4) We do not yet have the option to purchase a Ubuntu phone from a retailer.
5) The goal is not just to produce a Ubuntu phone OS but one single converged phone, tablet and desktop code base. The desktop platform is already there. The phone platform is almost there. The tablet platform is a little bit behind the phone platform. The single converged code base will not be ready for for 12 or more months.

It is up to the Fairphone people to bring this about. To be fair Ubuntu phone has only just reached version 1.0 running on Ubuntu 13.10. And it is not yet optimised for tablets. The next six months will see the Ubuntu phone/tablet code reach version 1.5 as it is developed and built on the forth coming 14.04 code. So, I can understand the reluctance of the Fairphone people to be offering a Ubuntu Fairphone product at this time.

I have heard some talk of how in the future Ubuntu phone/tablets will dual boot between Ubuntu and Android. Desires, intentions, goals are one thing. Reality is something else. We wait and see if intentions  become reality.
