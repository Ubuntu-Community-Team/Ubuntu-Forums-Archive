### Post by grahammechanical on 2014-12-11
Ubuntu phones and tablets will have a set of core apps and the ability to download apps from an app store.

Skype is proprietary software and is not in the desktop Ubuntu Software Centre because the owners of the software have not applied for Skype to be included in the software centre and that is because it will mean allowing the code to be audited to identify any malicious code. Skype will not be on Ubuntu phones unless the Skype developers package it for Ubuntu phones. The same applies to lots of other applications. Then there is the question of the application not giving a suitable user interface for a phone.

Ubuntu for devices uses a special packaging method called Click. The process of packing the app and applying for inclusion in the app store is said to be easy and fast. But the Click package method is designed to prevent apps from doing malicious things and from accessing data without user authentication. For example, a music app should not need to access the address book and it will be prevented from doing that without user authorisation. On the other hand an email app would need to access an address book but even so the user will have to authenticate the request. So, developers who want their applications in the app store will have to work with the conditions imposed by the Ubuntu phone developers.

We should not think of Ubuntu phone as Ubuntu on a phone. The Ubuntu for Devices developers have a different approach to an operating system to what we have on the desktop. There will be a terminal but it will not be a default application and to use it may require putting the phone into "developer mode." The desktop method of using apt-get to update package by package is not used. The system will have image updates that download only the differences between the old and new system image. Click apps will be updated when the app developer pushes an updated version into the app store. The apt-get utility will still be available but again it may be necessary to put the device into developer mode to use it.

Unrealistic expectations lead to disappointment, which leads to complaining.
