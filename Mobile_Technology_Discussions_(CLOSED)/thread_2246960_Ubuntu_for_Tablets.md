### Post by grahammechanical on 2014-10-05
Ubuntu for Devices, as it now seems to be called, is special in the sense that it is being developed for OEMs to install on mobile devices for retail. I do not think that we should expect to install Ubuntu for Devices on all kinds of hardware like we do Ubuntu Desktop. It is not the intention of Canonical to provide that kind of ISO image.

Three Nexus devices are used as references devices. If we have one of those and we are willing to take the risk, then we can install it. Some community developers are porting the code to other devices but how far they have got I cannot say. Here are the links

[http://developer.ubuntu.com/start/ubuntu-for-devices/devices/](http://developer.ubuntu.com/start/ubuntu-for-devices/devices/)

[https://wiki.ubuntu.com/Touch/Devices](https://wiki.ubuntu.com/Touch/Devices)

Ubuntu for Devices is not special in the sense that by the release of Ubuntu 16.04 (hopefully) the code for the mobile devices will be merged into the code for the desktop. So, that there is one code base for Ubuntu. That will not mean that it will be possible to install Ubuntu on any mobile device. It will mean that whether we have a Ubuntu phone, tablet or desktop we will have a similar user interface that is appropriate for each of those form factors and applications that scale for each form factor. And we may just have a Ubuntu superphone on the market that can become a tablet and a PC when connected to the right hardware.
