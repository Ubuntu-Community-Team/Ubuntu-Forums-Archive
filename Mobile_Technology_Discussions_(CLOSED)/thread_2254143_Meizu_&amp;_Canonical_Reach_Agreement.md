### Post by grahammechanical on 2014-11-25
I agree. Apps are key to the success of any mobile device these days.  Work has been going on and it continues so that app developers can easily develop apps for Ubuntu for Devices.

[http://developer.ubuntu.com/apps/sdk]("http://developer.ubuntu.com/apps/sdk/")

That SDK will not only be useful for writing an app but it also makes it a one click effort to package and submit the app for inclusion in the Ubuntu app store. The time taken for acceptance is a matter of minutes and that compares to the many days a developer has to wait to get their application in the desktop software centre.

Then there is the Ubuntu Developer Tools Centre (which is looking for a new name).

[http://blog.didrocks.fr/post/Ubuntu-loves-Developers](http://blog.didrocks.fr/post/Ubuntu-loves-Developers)

This takes out all the hard work of installing on Ubuntu the Android SDK. And then there is NDR to tempt Android developers to port their apps to Ubuntu for Devices.

[http://mhall119.com/2014/04/make-android-apps-human-with-ndr/](http://mhall119.com/2014/04/make-android-apps-human-with-ndr/)

Then there is the idea just being discussed of a component store where developers can find re-usable code.

[http://summit.ubuntu.com/uos-1411/meeting/22338/ubuntu-touch-component-store/](http://summit.ubuntu.com/uos-1411/meeting/22338/ubuntu-touch-component-store/)
