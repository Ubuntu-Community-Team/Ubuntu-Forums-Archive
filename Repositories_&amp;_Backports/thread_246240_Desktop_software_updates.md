### Post by Animortis on 2006-08-29
This is more of a request than a question, but is there anyway developers can look into updating important desktop software faster? 

While Ubuntu is the best desktop distro overall, I think, in terms of polish and functionality, almost all the useful software is a version behind. When I used Fedora, updates took two weeks at most.

Firefox is 1.5.0.5 when the newest version is 1.5.0.6 (Update was weeks ago)
OpenOffice.org is 2.0.2 when the newest version is 2.0.3 (Update was a few months ago)
Scribus is 1.2.4.1 when the newest version is 1.3.3.3 (Update was a few weeks ago, but the Ubuntu version is months old in itself.)
Gimp is 2.2.11. Newest is 2.2.13 (Update was a few days ago, but the preceeding version was updated a few months ago)
Inkscape is .43. Newest is .44. (Update was a few months ago, and the current version gives a silly error message that affects nothing)

The list goes on, and I'm sure others could name more. Some of these were incrimental updates, and some were rather big (Inkscape and Scribus being two). These are important desktop softwares, at least for me as a graphics designer who does some work at home on Ubuntu. While security updates are good to keep up with, this functionality software thing is rather discouraging, especially since Ubuntu is my favorite distro in terms of its polish and development, but new software is generally very important to keep an OS functional and friendly. This is my request to Cannonical - Better functionality package updates! This extra polish can help Ubuntu continue to have the best opportunity to become "The Linux Standard."

### Post by mlind on 2006-08-29
Most of the software will only get security updates once the distributions has been released, I guess that's been the drill from the beginning. Updates are usually backported patches, rarely new version (synced from Debian) gets released.

You may have luck with ubuntu backports though [http://ubuntuforums.org/showthread.php?t=153402](http://ubuntuforums.org/showthread.php?t=153402)
