### Post by Clopper Almon on 2010-03-28
Ubuntu 9.10 comes with GIMP 2.6 but the latest help files in the repositories are for 2.4 and a lot has changed since then. Moreover, the help files for 2.6 are now available on the Internet but getting them installed is tricky if they are not in the repository. I have been using the new help on line, but sometimes I need to work where the Internet is not available.

There is a similar problem with KompoZer, the open software alternative to Dreamweaver for creating websites. The KompoZer website says plainly that version 0.7.10 WILL CRASH when used with Ubuntu 9.10 and urges the use of KompoZer 0.8b3. (The problem seems to be in the use of a more recent GTK version in 9.10.) Yet only 0.7.10 is available in the repositories.  It would seem to invite trouble for Ubuntu 9.10 users to pass out only KompoZer 0.7.10 via the repositories. Granted, 0.7.10 is said to be the last "stable" version, but if it is guaranteed to crash, shouldn't 0.8b3 at least be available?  I have used 0.8b3 a little and have had no problems.

The repositories are a wonderful resource.  Many thanks for maintaining them.  I hope you can soon fix these two fairly straight-forward matters.
