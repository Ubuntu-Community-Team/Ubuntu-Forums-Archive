### Post by fragos on 2008-01-04
Ubuntu will use what ever version of stable software is available when the ditribution contents are frozen.  This gives them time to test the system before final release.  Individual packages have different release schedule from each other.  They aren't timed for any distribution release.  After the code freeze the focus is on a stable system.  The network manager used works well.  There is no security or bug problem that warrents releasing a newer version.  This is all quite normal.  As an open source user you have the ability to intigrate a newer release.  In this case I see no reason to do so.
