### Post by WSmart on 2008-03-03
I'm new to Linux, Ubuntu, too.  I built an upgrade computer for Ubuntu and I also have my old machine which runs XP connected via KVM switch.  Highly recommended, if you're moving from a different platorm, to use them both until you're more confident with the Linux system.  It's the difference between walking, one foot at a time, or jumping with both feet.

Graphics drivers have limited support in Linux.  There's a lot of good information at Phoronix.com, a Linux hardware site, about the state of the art Linux graphics driverrs.  It's an exciting time, but you need to be aware of what has support, what doesn't, and who's more committed to the open source community going forward.  Newer cards have much better support, but the very latest cards are still maturing.   ATI didn't have much support for Linux until more recently, and now they seem to be really turning that around nicely, since the AMD union.  But even with the best of driver support that's available, I think there are still some limitations, like with little features such as the one you're talking about.  The biggest issue seems to be the API, applications programing interface, if you want to run gaming software.  Most games are being made with the proprietary Windows API, DirectX.  In Linux we have Open GL 2.0, which sort of got left by the wayside, seems like.
