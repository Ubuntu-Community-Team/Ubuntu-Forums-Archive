### Post by Erol512 on 2007-07-20
If anyone's interested in instructions on how to get this thing set up, I have just got _everything_ working with Debian Lenny. Even the sound, which was a fix originally designed for feisty, and should work on any computer.

Just make a file /etc/modprobe.d/lanai-sound with this in it:

options snd-hda-intel model=3stack

It also works with 5stack instead of 3stack. To do this from the command prompt, run as root:

# cd /etc/modprobe.d/
# echo options snd-hda-intel model=3stack > lanai-sound

Restart, and it should work perfectly.

Alright, I've got network to come up after hibernate, sort of :)

```

 cd /etc/default/
# cp acpi-support acpi-support.orig

# vi acpi-support
Nice size file, so here are the changes.

# diff acpi-support acpi-support.orig |grep "<"
< MODULES_WHITELIST="ipw3945 nvidia"
< SAVE_VBE_STATE=false
< POST_VIDEO=false
< DOUBLE_CONSOLE_SWITCH=true

```

And then create a restart networkmanager.sh

```

$ cat networkmanager.sh 
#!/bin/sh

su -c "/etc/init.d/ipw3945d restart

/usr/bin/dbus-send --system \
      --dest=org.freedesktop.NetworkManager \
      --type=method_call \
      /org/freedesktop/NetworkManager \
      org.freedesktop.NetworkManager.sleep

/usr/bin/dbus-send --system \
      --dest=org.freedesktop.NetworkManager \
      --type=method_call \
      /org/freedesktop/NetworkManager \
      org.freedesktop.NetworkManager.wake"

```

then just rrun that when you come out of hibernation.

### Post by DaveTRG on 2007-07-25
Here's a workaround for the cd-rom drive (really a dvd-rw).

However, this isn't a good fix as it doesn't support dma so burning cd's/dvd's is painfully slow.  Please someone help with finding the right driver (module) for the ide controller on these laptops ;-)

```
sudo modprobe -v ide-generic
```

If it worked, write in the terminal
```
sudo gedit /etc/modules
```

Add this line in it and save:
```
ide-generic
```
