### Post by Superfish on 2008-01-25
All right, I think I fixed it.

First of all, here is what my "hsfconfig -i" said when I first checked it:

[FONT="Courier New"]Conexant HSF softmodem driver, version 7.68.00.06tst1oem

If you need assistance or more information, please go to:
        [http://www.linuxant.com/](http://www.linuxant.com/)

When reporting a problem for the first time, please send
us the file generated by "hsfconfig --dumpdiag".

Note: kernel module snd-via82xx-modem overridden by hsfmc97via
Note: kernel module snd-intel8x0m overridden by hsfmc97ich hsfmc97sis
Note: kernel module snd-atiixp-modem overridden by hsfmc97ati

Warning: no device detected by hsf driver - HDA modems may require reboot

Note: kernel module snd-via82xx-modem overridden by hsfmc97via
Note: kernel module snd-intel8x0m overridden by hsfmc97ich hsfmc97sis
Note: kernel module snd-atiixp-modem overridden by hsfmc97ati[/FONT]


After running that, I tried running just "sudo hsfconfig". It said I already had generic device drivers, do I want to keep using those? I said no, but then it wanted to recompile, so I stopped it.

I then restarted the computer, and during boot, it said "loading hsf modules" or something similar. Then, it said, "No module found, compiling a new module"
It proceeded to compile successfully, and when my Ubuntu splash came up, it was accompanied by a welcome sound effect! Now, both the sound and the integrated microphone are working.


So in summary, running "sudo hsfconfig" and choosing not to use the generic device drivers (and letting it compile some new ones) appeared to fix the problem. Thanks for all the help!
