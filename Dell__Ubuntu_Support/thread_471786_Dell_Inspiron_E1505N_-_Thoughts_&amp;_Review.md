### Post by Stink Hook on 2007-06-19
> **krull_etc said:**
> It runs so fast that I'm questioning the need for more than 512mb RAM.  Guess I'll need it when I get around to adding a partition to boot Vista, though :)

512MB is fine when your not doing a whole lot. It doesn't take me long to be on the computer before I have two browsers open, at least 10 tabs in each, remote sessions going, Gimp, music running, and so on. Maybe not everyone does that, but I multi task to the extreme on my computer. That's a big reason why I love having multiple desktop. The idea seemed silly until I tried it out. Beryl seemed silly as well until I realized how natural it feels to have a 3D cube to manage these multiple desktops.

Another good reason to have more RAM is for virtual machines. Granted, your going to run a faster OS off of a native boot than off a VM, but is it worth it? Do you really want a Windows partition to dirty up your hard drive? Do you really want to shut down everything your working on just to boot a Windows environment to run one application? I can see how you might want to if you still use Windows a lot for specific applications or games, but I use Vista every day at work and I hate being on it. I can do 95% of what I can in Windows on my Linux machine. That's why I choose a VM. I can just boot up Windows XP inside of Ubuntu, do what I need to do, and get out. Not to mention the fact that I can create a restore point from VMware and completely destroy Windows, then restore it right back to where it was. To each his (or her) own, but I've monkeyed with this for quite a while and I think my decision is a justified and sound one.

P.S. If your going to install Windows at all, I _highly_ recommend XP. It's way cheaper, much more stable and tested at this point, and who cares about the eye candy of Vista when you have Linux. I love eye candy as much as any other artistic natured anal retentive nerd (that's a dig on me, so no offence to anyone else), but even Vista's Aero environment got stale after a week. So unless you are convinced that you NEED DirectX 10 or something, stick with XP.

### Post by jordon on 2007-06-25
> **elijlyn said:**
> how do you change the root passward?? I don't recall being asked to input one??

For security reasons, you can't login as root in Ubuntu. The user created during the setup of the OS (most likely you) can use sudo to do important root things. Sudo asks for your own password - not the root password - to be entered before it executes a command.

See also: [https://help.ubuntu.com/community/RootSudo](https://help.ubuntu.com/community/RootSudo)
