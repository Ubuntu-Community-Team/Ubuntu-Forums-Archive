> **qingshi said:**
> On the terminal, there is no error message. If I run 'emacs a.c &' in the terminal, no response, then enter, show that emacs stopped.


I'm not entirely sure that this is an error.  After pressing enter, and the 'stopped' message is displayed, can you bring emacs to the foreground?  One way to bring a process to the foreground by by running the 'fg' command.

Before debugging X11, make sure that the emacs instance is definitely not working as intended.   Do a quick web search to try and find out if the minor version difference can explain the inconsistent behaviour.
