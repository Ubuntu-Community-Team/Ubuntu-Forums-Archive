### Post by dacap06 on 2007-11-21
The good news is that you have the correct display card listed in your Xorg.conf file.  The fglrx driver is the correct one for the ATI Radeon card.  The "Identifier" field in the display device section is an arbitrary string of characters, so far as X-windows is concerned, and is only meaningful to humans.  FWIW, my nVidea card is listed as a "Generic Video Card" too.

As for the resolution conundrum, that's a good question!  The underlying X-windows configuration as documented in your xorg.conf file is OK.  The only three resolutions I'm seeing in your configuration file are the three you specifically want - "1920x1200", "1680x1050", and "1440x900" - so long as your video card supports them you should see them as selections.  I know darn well that that the fglrx driver supports them.  I run Kubuntu, rather than Ubuntu, so my knowledge of its software to select resolution is too limited to help with that.

The only other thing I can think of to try is using the Gnome display software to select the ATI Radeon driver specificially (and keep your generic 1920x1200 16:9 display) and see if that makes a difference.

DaCAP

### Post by derby007 on 2007-11-23
Did ye try to get the 'restricted driver' installed. It should be in the menu, System>Administration>Restricted Devices Manager.  It worked for me, I have an ATI Radeon X1300 Pro graphics card. 

When you click on RDM it might ask to install 'fglrx', this will then be your driver, ie. in your /etc/X11/xorg.conf.
Like: 
Section "Device"
	Identifier  "ATI Technologies, Inc. ATI Default Card"
	Driver      "fglrx"	
	BusID       "PCI:4:0:0"
EndSection
