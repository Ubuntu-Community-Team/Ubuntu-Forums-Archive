### Post by 67GTA on 2008-06-21
If you are used to Ubuntu, then openSUSE is a different beast. I use Linux Mint and openSUSE. There is a small learning curve, but it shouldn't be too hard. It uses Yast to configure just about any aspect of the system. It is a BIG graphical configuration app. You will ocassionally have to use the terminal in any Linux distro, but if you dig around in Yast, you can almost keep from it. Ubuntu is just a tad easier to setup than openSUSE for begginers, but openSUSE KDE seems to have always been more professional and stable after setup. I would recommend it to anybody learning to use Linux. KDE4 is still under development, and a little shaky, so I'm not sure about it. Opensuse KDE3 is rock solid. Opensuse has one of the best KDE distros out there. I'm sure once KDE4 is more stable, it will rock also.

### Post by some-guy on 2008-06-21
You take the same risks with suse as you do with ubuntu.

The limits are pretty much do what you want with it, however, if you make a derived distro, it can't have **any** Novell/Suse branding

The whole novell/ms thing, novell has almost no effect to suse, it tried to push in ZENworks, which caused horrible consequences in 10.1/10.2, but has been fixed now.

Zypper is as fast/faster than apt, Synaptic is probably slightly faster than  YaST, but it's very slim.

Gnome is also extremely polished, and you may feel more at home in it.

[openSUSE FAQ on M$/Novell]("http://en.opensuse.org/FAQ:Novell-MS")

Have fun with openSUSE :)
