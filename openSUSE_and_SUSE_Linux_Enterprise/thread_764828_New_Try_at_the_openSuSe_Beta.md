### Post by Down8ve on 2008-04-24
This time I used the Reiser file system (read Ubuntu blog about them being so surprised at its speed), and selected KDE4 as the default desktop.  I then selected all of KDE3 from the software management dialogs.  The machine booted in KDE3 after installation, not sure why.

At any rate, I added the opensuse community repos (oss, non-oss, KDE) and updated via the software manager tool in Yast, choosing to upgrade available packages.  Note I did not use the software updater.

About 1/4 of the way through, one of the repos became unavailable.  Hitting "retry" didn't work.  I copied the url from the error dialog and opened it in Konqueror, and then clicked the "retry" button, and it worked.  Not sure why.

So far so good.  It does seem much faster using the Rieser FS when copying large amounts of data over from other partitions.
