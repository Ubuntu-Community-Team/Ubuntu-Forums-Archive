### Post by cogadh on 2007-09-26
If the mouse problem you are describing is a limited rotation range, then there is nothing you can do about it, there are several games that have problem with that. It apparently has something to do with a fault in Wine's implementation of DirectInput that has not been corrected (yet). I worked around it a bit by increasing the mouse sensitivity in the game's settings. It still won't get you a full 360 degree movement range, but it will be much better than the default. Also, you can use the keyboard functions to control your character's movement, which practically eliminates the mouse problem.

As for the screen resolution, KotOR's maximum resolution is 1600x1200, 4:3 aspect ratio. It cannot do wide screen resolutions like the default resolution of your monitor. There is a fan-made hack that will expand the game's resolutions to wide screen format, but I don't know if it will work with Wine:
[http://www.widescreengamingforum.com/wiki/index.php/UniWS](http://www.widescreengamingforum.com/wiki/index.php/UniWS)

Unless you disable the "Emulate virtual desktop" option, the game will never run in full screen anyway, it will always be in a Wine window. When you try running the game without that option, what errors are produced in the terminal?

### Post by cogadh on 2007-09-26
Almost forgot about KotOR's other mouse problem; the disappearing cursor. That one can be corrected by simply turning off "Hardware Mouse" in the game's configuration menu.
