First off I'm not sure how to force an install regardless of processor. Dose the 64bit version of the live CD work?
Second let me go over the 32bit programs issue in simple terms. 64bit processors can run 64bit and 32bit applications. In a perfect world people with a 64bit system wouldn't need to run any 32bit applications. But some applications, even those on an open source operating system are not open source. That or the programs rely on closed source formats or codex.
An example of this is Firefox. It is a open source browser. But it relies on the Macromedia Flash plugin and other plugins that are owned by someone to deliver content. If we could do without the closed source parts all would be good. Or if the companies that make the closed source parts made 64bit versions we would be OK. But they haven't yet.
So being able to run the 32bit applications on 64bit processor's comes in handy. We are able to use some 32bit applications while we wait for the population of 64bit users to grow to the point the people who own the closed source applications create 64bit versions. This isn't forcing the system to do something it isn't designed to. 64bit processors are designed to run both.
lastly there isn't a large number of applications we need to do this with. The number can be counted on your fingers. You can see them listed in the [first post in the sticky.]("http://www.ubuntuforums.org/showthread.php?t=191205") It isn't hard to install them either. Most of them have easy cut and paste howto's or automated setup scripts to install them. 
This way we get the best of both worlds. The 64bit system and applications that are already done. About 98% of the applications we need are 64bit. Then we get to run the 2% of applications in 32bit while we wait for the population to grow to the point the developers consider it worth making the closed source 64bit applications.
Is it worth it? Yes, IMHO its worth installing a 64bit system. After all you paid for a 64bit system. You should use it as it was designed. It was never designed to run pure 32bit code. Not only that most 64bit systems will run a 32bit os, but we are learning there may be problems doing so. 
Of coarse there are people that have failed to get this or that working, they complain that something isn't available or ready because they cant get something to work. This then is a rationalization of their inability to install the 64bit system to make themselves feel better. In essence they are making excuses in their head to convince themselves they are doing the correct thing. Sadly some people will do anything to take the "easy" way out of any situation. Even if it means admitting they failed. But they don't admit they failed, they just tell themselves the OS just wasn't ready.
No version of Linux is 100% easy with no work involved or setup. If they want that my sugestion is to pop the Windows cd back in.
