### Post by rivera151 on 2007-06-10
So during late March I decided to give Linux one serious try.  Being an absolute Windows hack, I though my "hacker" stigma suffered from not having any linux experience.  My solution: wipe my entire laptop (backing up some important stuff, of course) and reinstall OS a single-boot linux machine.  The result:

- First thing to notice is that my machine is a 64-bit Athlon.  So I decided to do the 64-bit OS thing.  Mind you, I had tried 64-bit XP, so this was a bit risky for me.  In reality, 64 bit XP was quite a well-oiled machine, but the lack of support for hardware and software vendors killed it.  But I found 64-bit support for Ubuntu is amazing.  The Ubuntu team should be proud of having done the 64-bit community a HUGE favor.  Kudos!

- The ONLY painful bit I would say are a bit of a hassle are kernel modules.  Even then, with a little scripting it's not half bad, and reduces to running your uninstall and reinstall scripts.  So far the only thing that requires any frequent removal and reinstall is the video drivers.

- Other than that, I love the speed and the stability.  Wow.  This is where Linux really stands above Windows,  I love the fact that I don't need a virus scanner always running in the background makes a huge difference.  And adware and spyware gone forever!  My machine never, ever, ever slows down.  I can't remember a Windows machine that doesn't slow down considerably after 3 weeks of a fresh install.  This I think is why people should switch to Linux.

- And the applications.  There's a subtle difference between apps from Linux and apps from Windows.  In windows, applications focus on trying to have every feature available on that single app.  The result is a bloated application and in my opinion slows down your productivity.  I have noticed that Linux applications, in general, tend to focus on one aspect of the process at a time (reflecting the modular build up of the entire system), and that translates into more productivity when you take out all the other stuff and focus on that substep.  At least that's my experience authoring DVDs from my hard disk camcorder, but I think all media applications kinda follow this type of pattern

- The 64-bit browser and lack of essential plugins.  I have Iceweasel installed, but I would rather just have 1 browser.  Especially if its Firefox 64.  What an absolutely awesome browser.  Did I mention pop-ups already?  I never get them, except when I sit at a Windows terminal somewhere else.  And then, I get really annoyed and wish I could go back to my computer.

- The X subsystem.  Pretty neat idea to have it modularized like that.  I still can't believe how much I can customize, especially with XGL and Compiz.  AIGLX is off-limits for me, by the way, but I can get just about as much done with XGL.  

I hope I don't forget something or anything, If I do and I think its relevant, Ill edit.

It's been a great time so far.  If you are thinking of switching, I say do it.  I can't be happier with my linux box.

### Post by Kilz on 2007-06-10
> **Kuraudo said:**
> About Firefox 64-bit and 32-bit plugins. Try a search for "nspluginwrapper" and you will get some nice results. With nswrapper you can run 32-bits plugin on 64-bit Firefox.

You can run some 32bit plugins. But you cant run a 32bit java plugin under nspluginwrapper. For those that suggest running java-gcj-compat-plugin. Be aware that that java dose not have a security manager. Per the java page on the wiki


Starting with Ubuntu 6.10 (EdgyEft), you can install the experimental java-gcj-compat-plugin from the universe archive. It is available on all architectures. Note however, that **the plugin currently runs with [COLOR="Red"]no[/COLOR] security manager.** This means that applets you load can do anything a java application that you download and run can do. Be **very** careful which applets you run.
