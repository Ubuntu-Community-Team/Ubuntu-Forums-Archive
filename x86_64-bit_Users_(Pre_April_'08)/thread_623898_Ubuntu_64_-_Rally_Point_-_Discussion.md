### Post by jdos2 on 2007-11-30
I love the discussion.

My perspective might be useful for some of the documentation needs.

FIRST of all, I'm a Unix admin- it's what I do. Solaris, Linux (Red Hat and Suse), AIX. All of 'em, including the different versions of 'em (Solaris 2.6 to current, AIX 4.3.2 to current, Suse 8.2 to current).

Semantics.

I installed Ubuntu knowing I was in for a pretty typical ride. The Toshiba Laptop was inexpensive and my hopes weren't high. I did solve my issues, and none of them were related to 64 bit'dness of the operating system. Finally got audio working, finally got the ATI drivers working. Wireless is bum until the next kernel version, but knowing that I got a USB wireless adapter that works "well enough."

Then...

My measure of a successful installation for Linux these days is Youtube. 32 bit Ubuntu is simple. For me, 64 bit is NOT.
FIRST of all was getting sound. Alsa drivers I've had to compile and install myself. Not a problem- I'm comfortable with the process, though it did take some research. Now that I had sound in a 64 bit system, it was a surprise that I did NOT for 32 bit applications. Seems the Alsa support has removed the 32 bit compatibility, meaning my test fails- I can't play Youtube videos in an acceptable way without using native 64 bit applications, and they are dubious at best- messy software that renders poorly (when it works!) the test videos.

The above was a discovery process. Hours and hours of work, research, paging through posts here and elsewhere. Ubuntu re-installs because I installed in-compatible plugins, &c. What really sucks is that there might be a better way, and I could have good compatibility with sound, but I've still not gathered enough information. 

Linux has changed quickly enough that a web search is really a poor way to do research, though it's a great way to get an idea of "from where we came." There are a zillion ways to do the same thing under Linux distributions, and usually only one (or a few) "best ways." THAT is rarely clear.
