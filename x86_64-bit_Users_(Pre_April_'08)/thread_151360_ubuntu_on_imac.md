### Post by Doobie9 on 2006-04-27
Unfortunately, you'll run across this a lot. Like Skype, and stuff won't work. But there are some workarounds that work quite well most of the time. Make sure you have mplayer and vlc installed, this will take care of most of your video-playing needs. You have to be able to download the files though, so not much luck with embedded stuff (Unless you can get the mozilla mplayer plugin working, which I couldn't quite do). Then there's Helix. If something is open-source you wont have too much difficulty getting it to work -- even if they don't have a binary you could try compiling it, which I have done in MaxOSX but not much in Linux. I've ported a few apps to OSX like this. In fact, I use so much ported Linux software in my OSX machine I decided to try what it would be like full-blown.

Some of the workarounds are ones you'd have to do in Mac anyway. Like Divx isn't supported on a mac as well as Linux etc. etc. So just get creative, and learn how to compile software (Much easier than writing it, mind you!). 

The most frustrating software issues in Linux have to do with proprietary hardware drivers, which really ticks me off :(. If you have good hardware support on your Linux machine, consider yourself lucky and pretty well-off, because the rest of this stuff isn't so hard :D.
