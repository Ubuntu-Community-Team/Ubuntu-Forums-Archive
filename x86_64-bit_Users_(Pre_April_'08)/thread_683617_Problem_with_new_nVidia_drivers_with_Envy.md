### Post by dabl on 2008-01-31
Although this happened at the time you installed the new driver, I don't believe it is the new driver itself, but perhaps the new xorg.conf file that was written, or perhaps some change to the screen saving/power saving settings.  But it's possible there's a new interaction between the new driver and the power-saving stuff that the older driver didn't do.

A backup copy of your previous /etc/X11/xorg.conf file should have been made at the time the new one was written.  I'd look at them side-by-side, and see what was changed.  If it was working correctly before, you might want to try renaming the new one and then renaming the backup to "xorg.conf" again and try running with it.

If that doesn't shed any light on the issue, you could disable the screen saving and power saving settings totally (and verify that they are indeed disabled), and then start turning them on one at a time, and observing the performance.  That's not a quick process, but it should ultimately reveal what specific feature is killing off the x server.  :)

### Post by dabl on 2008-01-31
Just be careful that you rename your NEW xorg.conf file to something useful, like "xorg.conf.31jan08".  Then rename your most recent backup (the one just prior to installing the new driver) to "xorg.conf".

Then restart the X server with Ctrl-Alt-Backspace.  You can't hurt anything (that I can think of) by merely running a different xorg.conf file.

Another possibility that occurred to me is that there might have been an update to one of the power-saving packages (powernow, lapic, etc.) that changed the performance on your laptop.  I haven't noticed, but I run a desktop rig and don't think about those things often.

### Post by generalj on 2008-02-06
I had this problem after upgrading my kernel. I also used Envy to install my nvidia driver. I have a hp tx1320us laptop which I believe is similar to your laptop.

What I did was download the nvidia driver from their website, drop out of X and killall gdm, I then had to --purge all nvidia drivers. I think the command is sudo apt-get remove --purge nvidia** 

but don't do it, make sure that is the command, I googled how to --purge and found the correct command. 

I then ran the nvidia installer from their website with X killed and then everything was working again.

Apparently this is a problem with the way Envy installs the driver. Someone should know more about it then I do.

The two important steps though is --purge all nvidia related drivers etc. then install the new driver with X killed.

If I find the correct steps I will post them but this should get you going in the right direction.
