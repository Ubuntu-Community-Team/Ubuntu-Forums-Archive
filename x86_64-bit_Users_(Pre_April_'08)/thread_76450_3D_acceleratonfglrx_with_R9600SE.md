### Post by daller on 2005-11-14
To start with, take a backup of the file:

sudo cp /etc/X11/xorg.conf /etc/X11/xorg.back

Change this:

```


Section "Device"
        Identifier      "ATI Technologies, Inc. Radeon 9600 (R300 AP)"
        Driver          "ati"
        BusID           "PCI:1:0:0"
EndSection


```

into:

```


Section "Device"
        Identifier      "ATI Technologies, Inc. Radeon 9600 (R300 AP)"
        Driver          "fglrx"
        Option         "UseInternalAGPGART" "no"
        BusID           "PCI:1:0:0"

EndSection

```

Afterwards try to replace "PCI:1:0:0" with "AGP:1:0:0", but I suggest you try the above at first!

### Post by jocko on 2005-11-14
There's one more thing you need to do _before_ you change driver from "ati" to "fglrx":

Install the 'linux-restricted-modules' for your kernel (this package contain the actual fglrx driver module):
First check which kernel you have:
```
uname -r
```
This will probably say '[COLOR="Red"]2.6.12-9-386[/COLOR]'

If you have the standard breezy kernel ([COLOR="Red"]2.6.12-9-386[/COLOR]):
```
sudo apt-get install linux-restricted-modules-[COLOR="Red"]2.6.12-9-386[/COLOR]
```

This should do it, but you may have to reboot before it works.

### Post by jocko on 2005-11-14
Daller is right... the restricted-modules should be installed when installing xorg-driver-fglrx...

If that's already installed I don't think I'm of any help here...

But there are some good how-to's in this forum on how to get the drivers from ati to work (by some reason ati's installer doesn't work as it should), for example:
[URL="http://ubuntuforums.org/showthread.php?p=423589"]
This worked for me with the latest ati drivers (8.19.10). If you decide to try it there are a few things that at least I had to change to get it to work:
the .deb files created by the installer ended up in /tmp and not in the directory where i downloaded and ran the installer, so before the 'dpkg -i' commands it _may_ be necessary to 'cd /tmp'. The 'fglrx-control' package should be installed last, not first otherwise it will cause dependency errors and the install will fail.
