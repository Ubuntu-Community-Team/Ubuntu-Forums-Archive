### Post by ihopetc on 2007-07-22
Forgive me if anyone has suggested this, I tried several searches but abstract ideas are a bit tougher to find by searching.

Reading this forum for a few months I noticed the vast majority of posts (and stickied FAQs) are either  1) Questions about the roadblocks are to running 64bit linux and  2)  Install/workaround guides and howtos for unsupported applications/drivers.

One post described the barrier to 64bit linux adoption and support fairly well:
People are less likely to install/adopt 64bit because it's not easily supported by applications they need/want to run (flash, java, etc.); developers are less likely (or less fast) to support 64bit linux with their applications because they don't think there's enough demand (installed users)... a vicious cycle.

The obvious solution for the users is to take the plunge, install 64bit linux and create the demand for developers to respond to.  It's here that all the workaround install guides and scripts work both for and against the community:  On the positive side they allow an installed userbase to develop by giving people functioning desktop environments.  But on the negative side if people are running viable workarounds, they're less likely to be vocal and communicate their presence & demand to straggling developers.

_What seems to be missing from the stickies, FAQs, and guides are these:_
[LIST=1]
[*] A single post where the author collects a list of proprietary (or even open) software that's slow to adopt 64bit linux as a platform, the latest news about project status, and links to feature request or feedback pages where people can (politely) request 64bit support.  The post would need someone dedicated to maintaining it and to getting it linked to in the stickies.

[*] When newcomers ask about the disadvantages of 64bit vs 32bit, the 64bit community need to emphasize that the biggest disadvantage is that not everyone who can install 64bit has installed it, and that widespread use is the best way to improve the situation.

[*] Every howto & guide to get problem programs running under 64bit should begin with a link to the feature request or feedback page for that program and encouragement for people to communicate with devs.
[/LIST]

Someone had the right idea 2yrs ago with flash, but as a single post without someone to keep track of things it ended up as a 3 page thread buried in the forums:
[http://ubuntuforums.org/showthread.php?t=51211](http://ubuntuforums.org/showthread.php?t=51211)
[http://www.adobe.com/cfusion/mmform/index.cfm?name=wishform](http://www.adobe.com/cfusion/mmform/index.cfm?name=wishform)

I think the feature request page is probably the best communication medium, petitions seem kind of useless because they don't go directly to developers.

Basically if people don't **install 64bit linux**, then **directly communicate to deficient developers that they have done so en masse**, it will continue to be a low priority to developers.  What's missing are the visible tools to do so and a push to get everyone to use them.  Any thoughts?
