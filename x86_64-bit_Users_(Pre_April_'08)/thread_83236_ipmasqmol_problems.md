### Post by calum on 2005-12-07
[QUOTE=calum]
I presume at the root of all this is a need to configure ipmasq properly, but I haven't the faintest idea how.  Any suggestions?[/QUOTE]

If anyone cares, I just found the answer [here]("http://open.mcgill.ca/pipermail/mclug/2005-April/000171.html"): you need to modify one of the ipmasq rules to allow traffic through on the vpnlink interface (not tun0, as referred to in that post).  So, just add the red lines below to /etc/ipmasq/rules/A01interfaces.def, and resave it as /etc/ipmasq/rules/A01interfaces.rul:

```
# find interface names
EXTERNAL_OUT=$(default-if | sort -u)
EXTERNAL_IN="$EXTERNAL_OUT"
INTERNAL=$(enumerate-if | sort -u | grep -v lo)
if [ -n "$EXTERNAL_OUT$EXTERNAL_IN" ]; then
[COLOR="Red"]    if [ $EXTERNAL_OUT = "vpnlink" ]; then
        EXTERNAL_OUT="vpnlink eth0"
        EXTERNAL_IN="$EXTERNAL_OUT"
        INTERNAL=$(enumerate-if | sort -u | grep -v lo | grep -v eth0)
    fi[/COLOR]
    for i in $EXTERNAL_OUT $EXTERNAL_IN; do
        INTERNAL=$(echo $INTERNAL | sed -e "s/\(  *\|^\)$i\(  *\|$\)/\1/")
    done
fi
```

Connect via vpnc, restart ipmasq, fire up MOL, and you should be laughing...
