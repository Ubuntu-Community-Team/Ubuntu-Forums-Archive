### Post by mmmmna on 2007-01-28
Downloaded ISO of 6.10 for EM64T.
MD5 checks correctly on my system, CDR burn was made at 16x on a 56X capable drive...
Nice and slow for nice clean fat CDR pits!!

Booted 6.10 EM64T disc.
Did not know to explore graphics options since other live distros get 1024x768 and I do nothing.
Elected to install, CD booted to Gnome desktop.

My system has the following disk partitions:
/dev/hda1 = WinXP Pro 64 bit, NTFS.
/dev/hda2 = Extended partition containing 3 partitions:
  /dev/hda5 = target partition for this install, intended for /
  /dev/hda6 = target partition for this install, intended for swap
  /dev/hda7 = unused partition intended for a second distribution of Linux.
/dev/hdb1 = my audio collection, to be mounted at /home/usernamehere/40g

Partitions /dev/hda1 and /dev/hdb1 are vulnerable, I cannot 'back up' these partitions because this system is only booting XP 64, Windows has no ability to read EXT3 filesystem in its present configuration.

In my attempt to install, here is what happened, in linear progression, top to bottom is chronologically sequential.

[COLOR="Sienna"]Desktop is 640x480.[/COLOR]
I would think that in this day and age, 600x800 should be the default, but whatever, developers set that value, this was not my decision since nothing told me I needed to make any choice for display resolution.
I ran the desktop icon "Install".
[COLOR="Sienna"]First issue: The installer window has buttons on the lower edge; the buttons are covered by the desktop panel.[/COLOR]
The user must select panel properties and enable autohide before the user can read the text on the buttons of the first screen of the installer window.
After enabling autohide for top and bottom panels,
Click forward, => "Where are you".
Selection on screen is correct.
Click forward, => "Where are you".
No screen change.
Click forward, => "Where are you".
No screen change.
Click forward, => "Where are you".
No screen change.
[COLOR="Sienna"]The button seems to ignore user clicks, although the dotted line around the "Forward" button became active after the first mouse click.[/COLOR]
Move the mouse but deliberately do NOT leave the dotted line around the "Forward" button.
Click forward, => "Where are you".
No screen change.
Click forward, => "Where are you".
No screen change.
Click forward, => "Where are you".
No screen change.
[COLOR="Sienna"]Wait... this has a solution!![/COLOR]
Move the mouse and deliberately LEAVE the dotted line around the "Forward" button, then return the mouse to the "Forward" button.
Click forward, => "Keyboard Install"
[COLOR="Sienna"]Mouse operation has to first enter the "Forward" button, click the "Forward"button, leave the "Forward" button, then re-enter the "Forward" button and now the "Forward" button will work as expected when user clicks the mouse.[/COLOR]
Keyboard is correct.
Click forward, => "Select a disk"
Click to Manually edit the partition table.
Click forward => Prepare Partitions.
Nothing seems incorrect... Am I ok?
Click forward => Prepare Mount points.
[COLOR="Sienna"]Current window does not show all the systems partitions on the left side.[/COLOR]
I do not see mount points for 4 partitions, hda1, hda5, hda7, hdb1 (hda6 = swap).
I have lost drive contents in the past, when installer scripts on another distro didn't seem to be making all the sense it could make.... I decided to abort this install to perform more research.
Click cancel => Get a popup: "Do you really want to abort the installation now?", that popup has 2 options: "Cancel" and "Quit".
Click cancel => Prepare mount points.
[COLOR="Sienna"]Syntax in the buttons leads to a context issue in the popup: Cancel what? Quit what?[/COLOR]
Click cancel => Get a popup: "Do you really want to abort the installation now?", that popup has 2 options: "Cancel" and "Quit".
Click quit => Back to the desktop.


I felt really afraid, there.....
