### Post by az on 2005-10-30
Software that is packaged is in the repositories.  You install the packages by using synaptic or the add/remove package tool.  Software you get from other sources, you need to compile yourself.  Also, if it is not a linux application, you may not be able to run it.

In that case, there most probably is a linux-equvalent to it.  Just search the repositories using synaptic.


The person who originaly writes a peice of software is called the upstream developer.  That software gets packaged by each linux distribution, that way you do not have to copmile everything.  Also, it is tweaked to work well with the other software in the distribution.   Although, you can if you want to.  The developement model by which all this great software was made is that everyone can get the source code (from upstream) and improve it if they want to.  You do not have to re-invent the wheel to get somethig done.  You can build on the work of others.  

For users who do not care about all that, you can just use the binary packages.  But that is why there are software repositories and you don't really go around installing things you get off the internet willy-nilly.

The easiest (and best) way to install software for Ubuntu is to use Synaptic. It's in the System menu, under Administration.
