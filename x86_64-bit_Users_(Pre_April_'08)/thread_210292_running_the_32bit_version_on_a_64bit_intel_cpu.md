### Post by Lil_Eagle on 2006-07-06
I have no trouble with the 64-bit version.  Nor do most of the people in this forum.  I liked it so much that I upgraded to edgy (getting bored with everything working).  

The best way to learn linux is not by reading books, or any web page.  The best way is to use it, and fix problems.  Since the dapper version works so well....

For the day to day tasks, it really doesn't matter which version you use, but what is the point of having a 64 bit processor if you only run 32 bit programs?

Like kilz said, almost all apps are 64 bit, and those that aren't have work arounds.  What is more frustrating is the lack of certain codecs.

If you had trouble with the beta, you may want to download the alternate install CD.  The Live CD that you will get from ship-it will possibly work, but the alternate install almost surely will.  It also gives you more options.  Do give it a try.  Surely things have improved from the beta you used.

Of all the 64-bit versions of Linux, Ubuntu is by far the most complete.  I have tried several.  SuSE is the only one that compares, but if you disagree with Microsoft's license agreement, most likely you will also disagree with Novell's.  To be honest I don't understand why they are so well regarded in the Open Source community.  If you want to watch your computer compile things, and aren't afraid of the command line, Gentoo is a good option.  Of course, this paragraph is not relevent to your question, nor does it really belong in this thread.  Shame on me.
