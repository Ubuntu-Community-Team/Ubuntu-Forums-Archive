### Post by the_it on 2007-06-02
Okay there's a noticeable change in swiftweasel 2.0.0.4

It now keeps its settings in .mozilla/swiftweasel rather than using the .mozilla/firefox settings.  This is cleaner and all, but I was used to having swiftweasel use my firefox (same tabs) every restart.  The installer is smart enough, however, to copy all of your firefox extensions and whatnot to your default swiftweasel installer.

You shouldn't need to do this if you've completely replaced your firefox, but if you want swiftweasel to still use the same stuff firefox does, you can symlink your .mozilla/swiftweasel to your .mozilla/firefox directory.

===
Edit:
By same stuff, I mean the same session, open tabs, etc.  There shouldn't be a need to do this if you just want to keep your extensions and passwords.

===
Edit:
My mistake, I got my swiftweasel directories mixed up.
Swiftweasel startup is smart enough to migrate all your extensions, but your passwords and search engines, as well as your extension settings are not migrated.

@stickk - it would be nice if the default startup for swiftweasel was to just copy the firefox subfolder, or to ask a dialog to copy your firefox settings (much like FF asks about IE settings).  I think it's a very simple dialog to start up. ^__^

### Post by stickk on 2007-06-05
No, no secret, its all optimizations in the code and the build process. The code is available , so anyone can see what has been done to the code. The graphics are also open source. Anyone can use them.
What is different is that I wanted to have some control over what was being done. I wanted to work on the current releases from Mozilla as well as development code. Iceweasel's last release used the code for Firefox 1.5.0.7. I also wanted to provide a 64bit version as I use a 64bit computer.

### Post by johnwillyums on 2007-12-04
Hi All. The problem I was having with flash in 64x Swiftweasel is now sorted thanks to "daflame" over on an absolute beginners thread.
He worked out that my flash was tied to gnash when it should have been using flashplugin-nonfree.

Therefore he suggested that I remove all gnash items in synaptic and then put this command into terminal:

sudo dpkg-reconfigure flashplugin-nonfree

I've done that and now have flash functioning well in 64x Swiftweasel.
I'm very pleased with this as I think it's a tremendous browser and I'm very grateful to Kilz and others who have spent time thinking about this and making suggestions.
