### Post by LTG1 on 2008-02-21
It seems as of 2008, it's possible to do most computer tasks with open software that is equal to if not better than what you can buy in a store as closed software.

However I wonder if the most glaring "hole" in the open software world is the lack of an effective competitor for the flash player?

It used to be not using flash on a web site was a good design practice that didn't exclude certain users.  However now its somehow become so ingrained into the web application universe it seems inextricable.

I know there are some related open software efforts, but does any have any serious momentum?

Sorry if this is a bit off topic, but it started making me mad when I realized Adobe won't even make an x64 flash version for Windows let alone an open system.

### Post by cdean on 2008-02-24
Flash is accessible, but few developers actually integrate the accessibility features into their applications. It's also overused and often breaks the browsing model since developers write entire sites with it.

SVG is a decent competitor, but there's not an editor for it such as Flash has. Silverlight is up-and-coming, too, but it's more proprietary than Flash and even less supported. It might get better once more people begin to use it, since it runs on Mono and Mono is fantastic for Linux.
