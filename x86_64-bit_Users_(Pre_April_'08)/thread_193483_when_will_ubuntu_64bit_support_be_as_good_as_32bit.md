I don't think its drivers that make people stay with a 32 bit OS. I think what keeps people in a 32 bit OS is applications. Mainly flash, wine, multimedia and codecs. But there are others. There are simply no 64 bit versions of some applications. What I read a lot is "But I want to use <install multimedia setup script> and it wont work on 64 bit because the apps are 32 bit". With Breezy it was a huge problem, 32 bit applications just didn't run at all on breezy for the most part without a complicated to setup chroot. Dapper is a little better, you can force install 32 bit app's. Edgy from what I've read will be a even more of a mixed arch environment. Hopefully it is like 64 bit SuSE in that you will be able to install 32 bit apps with the package manager with all dependencies. Some distro's are slightly ahead of Ubuntu in 64 bit development, mainly because Debian is its base and its behind. But hopefully Ubuntu can catch up and then surpass them. :D

[QUOTE=equal]Well, Linux is only as good as it's community. There aren't nearly as many 64bit users, so that accounts for the slower progress. What can you do? Contribute! Every advancement makes more 64bit users come along, and that adds more advancements, for a constantly growing cycle.[/QUOTE]
I think with amd selling mostly 64 bit possessors the number of 64 bit users increases each day. The problem is that its a catch 22 situation. It appears that there are more 32 bit users because 64 bit users take the easy way out and just install the 32 bit system waiting for the 64 bit OS to get better. But the 64 bit OS will not get as many resources if there doesn't appear to be more 64 bit users. Linux has a real opportunity right now. Windows SUCKS big time in the 64 bit OS. More people are willing to give Linux a try because of this. Who knows what quality people Linux may attract because of this., but the population should grow.

### Post by wmcbrine on 2006-06-11
Most of what holds back the 64-bit version is proprietary software -- either companies that can't be bothered to compile for 64-bit, or in some cases, binaries borrowed from Windows. The more of this stuff we can replace with Free Software, the better 64-bit Ubuntu will be, not just ideologically, but functionally.
