Just finished reading [30 days with Linux]("http://consumer.hardocp.com/article.html?art=MTI5OCwxLCxoY29uc3VtZXI=") by Brian Boyko.  In the article, he writes the following:
> If you've read so far, I've had a really, really hard time trying to get things to work in 64-bit Ubuntu. Truly, 64-bit Ubuntu isn't worth it for the desktop user, and should stay in the IT departments and programmer lairs until development catches up.  Whether we should install 64-bit Ubuntu on our 64 bit machines, or go with a 32-bit OS because it is easier to install, has been a topic on this forum for ages (seemingly forever).  However, there are_ other_ reasons to use 64-bit Ubuntu.  If you've read the article by Eric Steven Raymond called [World Domination 201]("http://catb.org/~esr/writings/world-domination/world-domination-201.html"), you'd know why.  Kilz has been complaining for ages about speeding up the development of 64 bit programs for 64-bit Ubuntu.  He's right.  We should all encourage users with 64 bit machines to use 64-bit Ubuntu (the more users, the more influence we have), and we should complain to Ubuntu developers/programers at the same time.

### Post by Kilz on 2007-03-11
> **Ocxic said:**

To each their own, but Automatix tends to install a lot of proprietary applications. For those that want to use free as in freedom code whenever possible it kind of defeats the purpose. Ubuntu is a open source operating system with a great [philosophy of using Free as in freedom]("http://www.ubuntu.com/ubuntu/philosophy") (read the Free software section of that page) applications whenever it can. 
A prime example of whats wrong IMHO with Automatix is one you mention. Swiftfox is proprietary, it is not free as in freedom. It takes Free as in Freedom code, then changes the license to make it non-free. Taking away freedom the code and application (Firefox)its based on have. Without adding anything but a few graphics, common settings, extensions anyone can add, and a catchy name.
Some people that come over from Windows dont pay attention to freedom. They tend to focus in on the cost. But the two are tied tougher in my mind. Without the freedoms of free software someone would charge you for it. IMHO we (Linux users ) should support free as in freedom software wherever possible. Because when we act like its ok to lose those freedoms others will see it as an opportunity to take them.
Secondly, while it does install a lot of things, I wouldn't recommend it to a new user. Why? Well I think we do a disservice to a new user by not letting them gain the knowledge of how to install some things themselves. Granted some things are hard, but automatix installs things that are easy. The install flies by, giving the user no knowledge, and little info if a problem occurs.

### Post by hizaguchi on 2007-03-14
I use the 64-bit version on my laptop, but there's no way I'd install it for a less computer literate friend or relative.  I've run into too many dependency issues.  Codecs and Flash and Java are tiny issues that can be predicted and only need to be tackled once, preferably right after installation.  The big problem is when you find yourself hunting for or compiling a newer version of a library because a package you want to install, that is in the repositories, depends on a newer library version than is available to go with it.  And even then, it isn't that I've had to do that once, but that I've had to do it many times, that breaks the experience for the average "desktop user".  I'm fine with dealing with these issues, but I wouldn't install a system like that on someone else's computer.
