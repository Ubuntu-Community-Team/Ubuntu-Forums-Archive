### Post by gnusci on 2007-08-31
Change form 32bits to 64bits does no means always an improvement of performance. 64bits platform means a longer register and in consequence more precise and longer number capacity. Anyway this feature can be exploited in different ways.

The old 32bits platform have many frustrating limitation for developer of large application such as database, big files, virtual memory, special hardware, cryptography, scientific calculations, an there must be many more cases where 32bits become a problem.

In summary, 64bits have a lot of advantages, but they are not for everyone needs. Anyway, what it is undoubtedly true, it is that the 64bits will become the common used platform during the next few years.

And if you were to ask my kids what was important to have working, they would tell you you that if the computer doesn't work with YouTube, then it ain't worth it...Yes I know there are scripts and wrappers for this, but it still isn't 100% yet, judging by last night's upgrade...

Running two identical systems side by side, one with Feisty (32) and one with Feisty amd 64, for the most part, doesn't make a whit of difference in normal usability, except for perhaps video rendering.  And that the 64 bit version chews up  more RAM, with absolutely no benefit to the end user.  I have both versions running on several 100% identical systems.  If you sat down to do normal work on one system or the other, and didn't know which was amd64, 90% of the time can't really tell. Most software apps really are not designed from the ground up to efficiently use the full dual-core register system yet,usually the sources are just recompiled with 64-bit memory pointers.  But that will change as you see more software made available for 64-bit _only_, and the software truly makes use of all the CPU 64 bit/ dual core  features.

Unless you -needed- to do a lot of video editing, and you had a more modest computer in the RAM department: then 32 bits still offers more compatibility with drivers &  plugins, and will be a bit less hassle to install, and just runs better on a more modest box.  If you have a newer system with more RAM, say 4~8Gb or more, and are willing to  to hassle a little bit more with drivers and plugins, and wait a little longer for the repos (at this point repos usually upgrade the 32 bit stuff first)  then running 64 bits is a good choice.  

There isn't any wrong or right way, its just what works best for you and your box.

As I said, it is up to each individual user. If you look at your memory resources running identical programs in both 32 and amd64 bit version, you'll see that more memory resources are used on the 64 bit version.  In every case I've seen, and sometimes the memory resources used area substantially more (if storage of memory pointers is required, remember that this take double the space as before).  Not that that is a huge problem in most cases.

And you certainly don't _need_ a 64 bit OS if the software you're using doesn't make use of it or if you have a small amount of memory.  Anything under 4GB RAM, in the 64-bit world, is small memory.
