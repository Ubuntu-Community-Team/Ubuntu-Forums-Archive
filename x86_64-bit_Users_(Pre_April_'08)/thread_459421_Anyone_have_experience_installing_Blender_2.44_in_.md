### Post by Unterseeboot_234 on 2007-05-31
Yes, I had a Ubuntu update notice for Python 2.5. I did that. While I was on the Blender site I almost downloaded the 32-bit Blender, realized I wanted the 64-bit. On the 64-bit download are two versions for the Python. Using that binary (already built) from Blender worked like a charm first time. I was very impressed how fast the UI loads. Boom, it's there, ready to go. I recall you have to have glibXXX and some other library, check the Blender download page. Those libs are readily available from Ubuntu repositories.

+++++++++

Now, I do recall something... execute Blender from a Terminal prompt. If any of the libs for Blender are missing it will report back to the Terminal that fact. I have a copy of Dapper on its own partition and a copy of Fiesty on another. I now remember Dapper is an odd duck in missing some libs the other Ubuntu distros have included. That's perhaps why I didn't have any problems with the new Blender on either Ubuntus. Fiesty seems to be more 64-bit complete, more polished, but I still have preference for Dapper.
