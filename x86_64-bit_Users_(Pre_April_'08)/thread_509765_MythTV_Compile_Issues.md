### Post by praxis22 on 2007-07-26
Hi,

This is your answer:

> 
`a local symbol' can not be used when making a shared object;** recompile with -fPIC**


PIC is Position Independent Code, means it can be located anywhere in memory, (more or less) and depending on how you're building it, it means you either have to add -fPIC to your CFLAGS, append it to your compile options on the command line or edit the Makefile. if you stick it either at the end of your compile flags or after your optimisation (I'm English) flag, then you should be fine.

eg: ```
gcc -O2 -fPIC 
```

### Post by praxis22 on 2007-07-27
Hi,

Before you run any **make** commands make sure you type the following:

```

echo $CFLAGS

```

If this returns any result, then you'll need to do this:

```

CFLAGS=$CFLAGS -fPIC;export CFLAGS

```

If you get no results from the echo line above then simply do the following:
```

CFLAGS=-fPIC;export CFLAGS

```

What you're doing here is adding base C compilation flags, ones that will be inherited by any command you type in this shell. If you edit a Makefile, it will only affect the object code generated by that Makefile, and not ones in any other directory.

The reason why your attempt didn't work is you missed of the S it's CFLAGS, not CFLAG. This affects the runtime environment of the C compiler so it'll work with any build.
