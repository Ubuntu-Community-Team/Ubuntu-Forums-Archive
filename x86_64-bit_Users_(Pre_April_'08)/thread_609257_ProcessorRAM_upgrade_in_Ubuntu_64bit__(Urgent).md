### Post by orthopod on 2007-11-11
Should have bought the 5600 instead.  The clock multipliers works out best with the 5600 chip and 800Mhz memory - since 7x800=5600.  Now with the 6000 chip you won't be using your memory at their fastest setting.  6000/800 = 7.5  = clock multiplier can't do a half step, so the memory speed gets switched down to 750 Mhz and uses the clock multiplier of 8

At least for memory intensive procedures , the 5600 should be faster.

Toms hardware had a nice artilcle comparing the 5600 X2 and the 6000X2 chips, and saw in real world applications that the 5600 often was faster.

Having said all that, I did the same thing, moving from a 32 bit Suse system to a 64 bit Ubuntu as well - I just kept my hard drives.  Couldn't have been easier - everything recognized/worked without a hitch.
