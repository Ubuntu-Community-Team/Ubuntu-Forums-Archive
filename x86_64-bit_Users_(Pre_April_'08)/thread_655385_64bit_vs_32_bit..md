Nope.  64 bit ubuntu is what you want.  In Windows land, most programs, drivers, etc. are closed source and most developers make a 32 bit version of the software to save on development costs. Let me get rid of a few myths you may come across.

1. There are problems with the 64bit version. 
Answer - There are problems with every version of Linux. There is no such thing as a version of linux that requires no setup or is problem free. If you are looking for something that doesnt require you to do a little learning and setup, search for your windows restore disk.

2. Something is missing from the 64bit version.
Answer - According to launchpad, the ubuntu development site the number of avilable packages in the 64bit version is exactly the same as the 32bit version. All the common applications have 64bit versions. 
Granted if you want some ancient, or rarely used application you will have to mess around with it to get it to run sometimes. This is common in all versions of linux, even the 32bit version.

3. But the 32bit version "Just works" for new users.
Answer - [Please visit this section of the forum]("http://ubuntuforums.org/forumdisplay.php?f=73"). It is for new users, 99% of wich are running the 32bit version. In fact look in any section of the forum but this one. All of the posters have problems. There is a learning curve. You and the people in that section have a lot to learn when moving from Windows. Most of us (me included) went through the same thing. What bit the operating system is isnt going to make it any easier.

To sum it up, All versions of Ubuntu require setup. None is any more difficult than the others. There is no easy version or one that doesnt have issues. If you have 64bit hardware run the version that makes the best use of the hardware you bought.
If you have any problems, post questions to this section. Some of the issues with 64bit come up over and over. Most of us will be happy to help you solve them or point you in the right direction.
Lastly, I will make one friendly word of advice. If you come across Automatix or someone suggests using it. Dont, or at least [read this blog post]("http://mjg59.livejournal.com/77440.html") from a member of the Ubuntu Technical Board before you do.
