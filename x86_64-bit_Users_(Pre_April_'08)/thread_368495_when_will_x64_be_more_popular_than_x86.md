### Post by JKester on 2008-02-16
There's a big difference here.

Ubuntu could easily say that although they are not going to come out with a new 32 bit version, the it will be kept up to date until 2009 and be available as long as there is a demand for it.

With this in mind, there is no reason not to move forward.  This wouldn't be a Microsoft XP/Vista thing where someone would say, 'Ok, go buy new hardware if you want to keep up to date, we're taking away the old version'.  It would be more like 'Ok, while we realize that 32 bit processors are still around, we're pushing forward to 64 bit as a step of progress and standardization.  Keep in mind that the 32 bit versions will be kept available as long as there is a demand for it'.

Even when we reach a level (lets say 10+ years from now), were something new like a 128 bit version or some other type of processor comes out, there is nothing that will keep the 32 bit version from being available.  It's still open source.  

I wont say anything about the CD-ROM vs DVD support.  So long as the software doesn't require 4+ CD's to install the CD is still functional.  Anyone remember installing an OS from 10+ floppies?  There is no reason for this to happen with the CD-ROM though as DVD is already available and is functional on the tried and true IDE standard.

### Post by Yellow Pasque on 2008-02-17
I think we should realize a little more about the software develeopment process before jumping to conclusions that the Linux developers aren't focusing on 64-bit. The main difference between the Ubuntu-x86 and Ubuntu-amd64 distros comes from the proprietary programs (e.g. Java, Flash) because the source isn't available to compile on a 64-bit platform. That may be because the companies don't want to put out two versions to support or because they've written poor, non-portable code.

Other than those proprietary programs, there shouldn't be any difference in the stability of the 32-bit and 64-bit versions assuming we have equal compilers. And that's where the real gains in performance are going to be made: compiler algorithms. I believe we're just starting to really take full advantage of the extra processor registers and instructions available with the AMD64 architecture.
