### Post by Mongoose on 2007-02-02
Well, you didn't need to post the entire listing from glxinfo.  My bad on that. Anyway it looks like your driver is working fine.  =)

Try these options in your Device section in /etc/X11/xorg.conf:

        # Composite
        Option  "XAANoOffscreenPixmaps"
        Option  "TripleBuffer"  "true"

/etc/apt/sources.list:

# beryl
deb [http://ubuntu.beryl-project.org/](http://ubuntu.beryl-project.org/) edgy main 
deb-src [http://ubuntu.beryl-project.org](http://ubuntu.beryl-project.org) edgy main

Then just use xorg as your server, and make sure you have the latest packages.  The beryl wiki can hook you up.  They just updated packages to 0.1.9999 yesterday iirc.   

[http://wiki.beryl-project.org/wiki/Install_Beryl_on_Ubuntu_Edgy_with_AIGLX](http://wiki.beryl-project.org/wiki/Install_Beryl_on_Ubuntu_Edgy_with_AIGLX)

### Post by Snookered on 2007-02-22
Had trouble running Beryl myself at first. On the Beryl website install instructions there was a note in troubleshooting that said to go to Beryl Manager>Advanced Beryl options>Rendering path> and select "Copy".
This solved a lack of memory problem. My onboard Nvidia 6150 has 256MB memory and appearently needed more, I think. At least no crashes or falling back to Metacity.
