### Post by jespdj on 2007-04-25
I have an Intel Core 2 Duo and also an ATI X1600 video card.

I am using Compiz (not Beryl) and it works fine. Beryl is a fork of Compiz.

I have installed the restricted ATI driver (fglrx). I am not using the AIGLX xserver, because [the Compiz documentation](http://compiz.org/ATI) says: **Fglrx doesn't support AIGLX. That means you must use XGL with it. AIGLX support is expected to be added in next few versions (few months).**

I see you are using AIGLX and fglrx. Switch to XGL instead of AIGLX.

Info on how to install Xgl here:
[https://help.ubuntu.com/community/CompositeManager/Xgl](https://help.ubuntu.com/community/CompositeManager/Xgl)

### Post by blazercist on 2007-04-25
Ok, here's what you do: uninstall everything xgl and beryl related that you've done so far.

sudo apt-get autoremove xserver-xgl beryl beryl-ubuntu beryl-manager emerald emerald-themes

restart the X-server: crtl+alt+backspace

That should undo all the crap you've already messed up.

Now, in order to make sure you have the correct graphics driver running...

fglrxinfo  

_***It appears according to your post with your xorg.conf that this is already correct [COLOR="Red"]however the output of fglrxinfo give you some crap about mesa THATS BAD YOU MUST FIX THIS[/COLOR]!***_

_**It should say something about ATI driver 8.33.5 or something similar**_, [COLOR="Red"]_**IF**_[/COLOR] it says ANYTHING about vesa/mesa/radeon or any other crap other than fglrx then you need to go back and fix this:

sudo apt-get remove xorg-driver-fglrx

then

sudo apt get install xorg-driver-fglrx

sudo aticonfig --initial
sudo aticonfig --overlay-type=Xv

sudo gedit /etc/X11/xorg.conf

find the section for your Screen Driver and replace "vesa"/"mesa"/"radeon" with "fglrx"

restart X: ctrl+alt+backspace

now everything should be nice and pretty, confirm its using the fglrx driver by typing

fglrxinfo

into a console.... NOW, we are sure that we've removed all the F***ED up junk we've done to beryl and flgrx and we have the driver working properly we can setup XGL and install beryl.

To do so you need to follow this guide  [http://wiki.beryl-project.org/wiki/Install_Beryl_on_Ubuntu_Feisty_with_XGL](http://wiki.beryl-project.org/wiki/Install_Beryl_on_Ubuntu_Feisty_with_XGL) TO THE LETTER! Make sure you read the damn thing before you copy and paste the commands into the terminal!!!!  Pay attention some of the stuff is for NVIDIA cards you need to ACTUALLY READ!!!! You can start reading from the Section "Migrating from Compiz-QuinnStorm"  

Good luck and don't F**K it up.
