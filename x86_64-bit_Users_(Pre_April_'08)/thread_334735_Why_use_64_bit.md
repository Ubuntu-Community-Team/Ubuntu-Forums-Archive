I think the problem with 32 vs 64 is that not all programs have been recompiled with for 64 bit processors, so you can encounter programs that won't run. Then you have to muck around using various techniques to make it work, ie, fool the program into thinking it's running on a 32 bit chip, etc. 

You'll probably find more solutions to problems using 32bit, as there are more people using 32bit. For a newbie, I'd recommend the 32bit until you're comfortable with installing, configuring, troubleshooting, etc. Even then, if it's not broke, don't fix it, ie, if the 32bit does what you need, then just leave it there.

That's my .02...:)
