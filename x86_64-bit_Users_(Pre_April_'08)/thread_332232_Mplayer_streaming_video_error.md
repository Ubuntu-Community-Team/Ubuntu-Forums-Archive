### Post by Aega on 2007-03-28
I was having a similar problem every time I tried to open video/audio in firefox. This fixed it.

I had the 64-bit version of mplayer installed, but everytime 32-bit firefox tried to open a multimedia streaming file, it would result in 

Error: mplayer execv: No such file or directory

Check to see if you have mplayer installed by

ls -l /usr/bin/mplayer*

My system reported this

-rwxr-xr-x 1 root root 8212848 Oct 12 21:25 /usr/bin/mplayer64

(If you don't see mplayer installed, then you'll have to run sudo apt-get install mplayer)

Apparently firefox was trying to call mplayer, not mplayer64. So I did a 

sudo ln -s /usr/bin/mplayer64 /usr/bin/mplayer 

Afterwards running ln -l /usr/bin/mplayer* yields

lrwxrwxrwx 1 root root      18 Mar 28 11:44 /usr/bin/mplayer -> /usr/bin/mplayer64
-rwxr-xr-x 1 root root 8212848 Oct 12 21:25 /usr/bin/mplayer64

And then everything worked.
