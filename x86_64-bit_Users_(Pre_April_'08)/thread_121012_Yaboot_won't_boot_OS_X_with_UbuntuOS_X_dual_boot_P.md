### Post by RoadTripDK on 2006-01-24
OK, now I have gotten Ubuntu (Dapper Flight 3 PPC) installed and running on my Power Mac G5. You can see how I got to this point by looking at the thread "Ubuntu/Kubuntu on Mac G5." on the ubuntu-users list.

Unfortunately, I have run into an OS X specific problem with Yaboot. Yaboot simply can't load OS X. it tries, and the screen flickers, but nothing happens. After three tries, it says something like "booting Mac OS X..." and nothing works, so I have to do a hard shutdown and then unplug my #1 SATA drive, which is where Ubuntu PPC is installed..

Any ideas will be appreciated. I am glad Ubuntu can boot, but I need to be able to boot both OS's and unplugging the hard drive every time I switch OS can't be an option.

I was hoping that there are more people on this forum that have PPC hardware (duh!) than on the ubuntu-users list. Has anyone run into this before?

Oh yeah, my path for yaboot.conf was as follows:

/ht@0,f2000000/pci@3/k2-sata-root@c/k2-sata@0/disk@0:

This is what has finally got Ubuntu to boot. I got this info by running the following command from an Ubuntu live-CD:

ofpath /dev/sda

Here is a full copy of my yaboot.conf file:

## yaboot.conf generated by the Ubuntu installer
##
## run: "man yaboot.conf" for details. Do not make changes until you
## have!!
## see also: /usr/share/doc/yaboot/examples for example configurations.
##
## For a dual-boot menu, add one or more of:
## bsd=/dev/hdaX, macos=/dev/hdaY, macosx=/dev/hdaZ

boot=/dev/sda2
device=/ht@0,f2000000/pci@3/k2-sata-root@c/k2-sata@0/disk@0:
partition=3
root=/dev/sda3
timeout=100
install=/usr/lib/yaboot/yaboot
magicboot=/usr/lib/yaboot/ofboot
enablecdboot

# to boot OSX by default
# to boot linux by default, change to defaultos=linux
defaultos=macosx

macosx=/dev/sdb3

image=/boot/vmlinux
	label=Linux
	read-only
	initrd=/boot/initrd.img
	append="quiet splash"

image=/boot/vmlinux.old
	label=old
	read-only
	initrd=/boot/initrd.img.old
	append="quiet splash"
