### Post by jammastercd on 2012-05-17
I have a MythTV system in my home with a single master backend running 24x7 in the basement.  I then have a frontend in our family room and another frontend in the home gym.

I recently upgraded all of the units to version 0.25.  The backend and family room frontend work great.  In fact, I have a Zotac with an SSD in the family room, and I've noticed the startup time for this unit has improved quite nicely compared to version 0.24.

The home gym frontend is a different story.  During the startup process, it appears to fail connecting to the backend, causing it to bring up the dreaded screen with the select a country and language options.  I leave the US and english options as is and click save.  From there it takes me to a backend connection configuration screen.  The settings are all correct, matching what I did on the family room frontend.  I don't make any changes, click next and then finish on the second screen.  At this point the frontend connects to the backend and brings up the MythTV main menu.  I can then watch live TV, recordings, etc.  Since I don't make any settings changes, I don't know why it fails to connect during initial startup each time.

Thoughts?  If there is a log file that would help, could you point me to the proper directory and file?  Thanks!

### Post by jammastercd on 2012-05-17
I'm using a wired connection, but for some reason the connection doesn't establish before MythTV starts.  I added a delay (sleep 5) to the startup script (/usr/bin/mythfrontend).  That worked!

Thanks!
