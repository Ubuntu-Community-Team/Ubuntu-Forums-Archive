Your logs/build stdout clearly show the install path as /usr/local/bin/.

In a terminal:
which mythbackend

Normally mythbackend is started by upstart (ubuntu)..
sudo service mythbackend [status|start|stop|restart]

Because you build from source there lots of configuration required to:
- mysql
- upstart job

But you should first address/fix the compilation problems..
- python bindings
- perl bindings
- VDPAU libs (for nVidia & FOSS AMD h/w video decode)
- vaapi 
- missing alsa support


You could be better off using the mythbuntu ppa (compiled packages & install scripts) for your ubuntu version.
[http://www.mythbuntu.org/repos](http://www.mythbuntu.org/repos)
You can use the mythbuntu mythtv ppa on ubuntu without having to install the mythbuntu desktop etc..

### Post by bric on 2014-02-07
I found this somewhere on the internet.  It looks like it is where you are stuck (because you get a can't connect to database message).

Q: When running mysqlcheck I get the error message "Access denied for user ..."
A: The problem may be that you are using the wrong password. The following commands will change the mysql password to "mythtv":


Open a command prompt (ctrl - alt - t)

type:
mysql -u root -p mysql

<press enter for the password prompt since default root password is blank>

at the mysql> prompt type:

GRANT ALL PRIVILEGES ON *.* TO 'mythtv'@'localhost'
IDENTIFIED BY 'mythtv' WITH GRANT OPTION;

then type:
UPDATE user SET Password=PASSWORD('mythtv') WHERE user='mythtv';

then type:
FLUSH PRIVILEGES;

finally type:
quit;

now run mythtv again.

If you have installed from the repos and are stuck at that point (I haven't read this forum thread carefully) then you should run the mythtv backend setup first.  I'm no expert with that sorry.  Then run the frontend.
