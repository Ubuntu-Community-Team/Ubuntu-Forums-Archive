### Post by suv-ajax on 2008-03-01
Well, after struggling for several hours... I got a GV47L to work at 1080p.

System setup:
Radeon 9600 256mb
flgrx driver ( Not open source driver)
Abit NFS-7
Linux edubuntu 2.6.22-14-generic

GV47L panel <->HDMI cable <-> HDMI-DVI adapter <-> Radeon 9600

Also worked in Windows XP too... but:
[http://www.avsforum.com/avs-vb/archive/index.php/t-818988-p-5.html](http://www.avsforum.com/avs-vb/archive/index.php/t-818988-p-5.html)
You have to UNCHECK "reduce DVI frequency on high-resolution displays"

The following is my xorg.conf:
-----------------------
Section "ServerLayout"
	Identifier     "single head configuration"
	Screen      0  "Screen[0]" 0 0
	InputDevice    "Keyboard0" "CoreKeyboard"
EndSection

Section "Files"
EndSection

Section "Module"
EndSection

Section "InputDevice"
	Identifier  "Keyboard0"
	Driver      "kbd"
	Option	    "XkbModel" "pc105"
EndSection

Section "InputDevice"
	Identifier  "Configured Mouse"
	Driver      "mouse"
	Option	    "CorePointer"
	Option	    "Device" "/dev/input/mice"
	Option	    "Protocol" "ImPS/2"
	Option	    "ZAxisMapping" "4 5"
	Option	    "Emulate3Buttons" "true"
EndSection

Section "Monitor"
	Identifier   "Monitor[0]"

	ModelName	"Vizio GV47L"

	Option	    "VendorName" "ATI Proprietary Driver"
	Option	    "ModelName" "Generic Autodetecting Monitor"
	Option	    "DPMS" "true"
EndSection

Section "Device"
	Identifier  "Device[0]"
	Driver      "fglrx"
	Option      "ExactModeTimingsDVI" "True"
EndSection

Section "Screen"
	Identifier "Screen[0]"
	Device     "Device[0]"
	Monitor    "Monitor[0]"
	DefaultDepth     24
	SubSection "Display"
		Depth     24
		Modes    "1920x1080" "1333x768"
	EndSubSection
EndSection
-----------------------------------
xorg.conf was partially created from gnome/edubuntu, aticonfig, and by hand.  

The GUI tools didn't seem particularly reliable, causing either lockups(needing remote/button reboots),  or else not having the resolution available(1920x1080) even if I chose various Generic panels.  Once the GUI tools produced 1080p resolution but after I rebooted and unplugged a CRT I had in the vga port the screen went back to 720p (Resolutions as reported by tv "info" button).

 I added the 1333x768 mode to the screen because certain movies seem to look rather blocky/aliased at 1080p but I don't know how to change resolution in edubuntu (ctrl alt - doesn't seem to work).

Somethings I noted while setting this up
radeon(open source) drive sometimes worked for my GDM login page at 1080p but once I log in it switches down to 720p.
Your home folder might have some old files which will overwrite your new setup every time you login/logout of Gnome---Thus creating a new user expressly for testing the X setup might be a good idea.

Connecting SPDI/F for me... well I just connected the cable and found out after several hours of troubleshooting I just had to specify the program to use SPDI/F out for audio.
thus for mplayer configuration in ~/.mplayer/config
ao=alsa:device=iec958
and as mentioned you can check in alsamixer that IEC958(Another name for SPDI/F) is not muted/volume down.
