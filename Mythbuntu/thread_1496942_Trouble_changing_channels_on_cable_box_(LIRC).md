### Post by chiron80 on 2010-05-29
I'm having some weird problems changing channels on a cable box (using LIRC).

1) I can change channels by directly calling:
```
/usr/local/bin/change_channel_lirc.pl XX
```
This works just fine 100% of the time.
2) ~90% of the times the backend changes channels for a scheduled recording, it works just fine.
3) The other ~10% of the times, the first digit in the channel is ignored by the cable box. For example, if I want to change to channel 56, I get channel 6. If I change to channel 45, I get channel 5, etc.
4) 100% of the times the *frontend* changes channels, the first digit is ignored by the cable box.

The odd part is that, as far as I can tell, the digit *is* being sent, it just seems to be ignored by the cable box. I determined this by testing using irw. For example, here is the output of irw when trying to change channel to 56 using the remote and frontend:

```

000000037ff07bfa 00 Five mceusb
000000037ff07bfa 01 Five mceusb
000000037ff07bfa 02 Five mceusb
000000037ff07bfa 00 Five_UP mceusb
000000037ff07bf9 00 Six mceusb
000000037ff07bf9 01 Six mceusb
000000037ff07bf9 00 Six_UP mceusb
000000037ff07bdd 00 OK mceusb
000000037ff07bdd 01 OK mceusb
000000037ff07bdd 02 OK mceusb
000000037ff07bdd 00 OK_UP mceusb
0000000000005ff2 00 5 DCT2000

0000000000009ffa 00 6 DCT2000
0000000000009ffa 00 6_UP DCT2000

```

You can clearly see my remote (mceusb) send 5-6-OK. You can also clearly see the 5-6-OK generated by the transmitter. Despite this, the cable box still changes to channel 6 instead of 56. All sources are using the same change_channel_lirc.pl, and (obviously) I'm not moving the cable box, transmitter, or receiver throughout.

Here's the setup:

Motorola Cable Box (Comcast Cable)
MCE Remote/Transmitter (came with Hauppage PVR-500)
(remote has model RRS9002-86XXF)

I'm using the default Mythbuntu configurations and Mythbuntu Control Center to set up the following remote/transmitter:
Windows Media Center Transceivers/Remotes
Microsoft Windows Media Center V2 (usb) : Motorola Cable box transmitter

Here is my "change_channel_lirc.pl" script below.

```

#!/usr/bin/perl

# make sure to set this string to
# the corresponding remote in /etc/lircd.conf
$remote_name = "DCT2000";

sub change_channel {
        my($channel_digit) = @_;
        system ("irsend SEND_ONCE $remote_name $channel_digit");
}

$channel=$ARGV[0];
sleep 1;
if (length($channel) > 2) {
        change_channel(substr($channel,0,1));
        change_channel(substr($channel,1,1));
        change_channel(substr($channel,2,1));
} elsif (length($channel) > 1) {
        change_channel(substr($channel,0,1));
        change_channel(substr($channel,1,1));
} else {
        change_channel(substr($channel,0,1));
}
system ("irsend SEND_ONCE $remote_name OK");

```

### Post by chiron80 on 2010-05-29
I think I got the problem resolved.

I did a little further testing, and discovered that as long as I blocked the cable box from seeing the output from the remote, the cable box changed channels properly. Therefore, I guessed that the combination of the IR signals was somehow confusing the cable box. I added a line to change_channel_lirc.pl to send "EXIT" before sending the channel, and now it works.

Here is my new change_channel_lirc.pl:

```
#!/usr/bin/perl

# make sure to set this string to
# the corresponding remote in /etc/lircd.conf
$remote_name = "DCT2000";

sub change_channel {
        my($channel_digit) = @_;
        system ("irsend SEND_ONCE $remote_name $channel_digit");
}

$channel=$ARGV[0];
sleep 1;
system("irsend SEND_ONCE $remote_name EXIT");
if (length($channel) > 2) {
        change_channel(substr($channel,0,1));
        change_channel(substr($channel,1,1));
        change_channel(substr($channel,2,1));
} elsif (length($channel) > 1) {
        change_channel(substr($channel,0,1));
        change_channel(substr($channel,1,1));
} else {
        change_channel(substr($channel,0,1));
}
system ("irsend SEND_ONCE $remote_name OK");

```
