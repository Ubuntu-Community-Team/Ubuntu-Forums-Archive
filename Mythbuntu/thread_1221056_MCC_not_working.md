From your log it appears you're running ATI Radeon graphics. There's a bug with Radeon graphics and mythfrontend in 9.04.

You may be able to run mythtv-setup directly from a terminal (not from MCC) with sudo. mythfrontend may start correctly from a terminal with:

XLIB_SKIP_ARGB_VISUALS="1" mythfrontend

If sudo mythtv-setup still gives you the visuals problem with backend setup, the same workaround applied to it may succeed:

XLIB_SKIP_ARGB_VISUALS="1" mythtv-setup 

If you have another Linux machine on your network with X enabled, you can also configure your backend remotely via ssh:

ssh -Y -l [user-name] [ip address of the myth box] and then run sudo mythtv-setup on the terminal.

Have you set up the backend first with mythtv-setup? Until the backend is set up, at least partly, there's no mythconverg for the frontend to log into. Until it finds a mythconverg (whose user will be "mythtv" and whose password will be generated by the backend and will be found in the file /home/mythtv/.mythtv/mysql.txt) you never get out of the frontend config page.

Usually a normal mythbuntu install takes care of most of this behind the scenes. But, having installed myth on top of plain vanilla Ubuntu, and being afflicted by the Radeon bug, you're forced to configure some of this manually.

OK, it appears you haven't gotten a backend configured. Until a backend is at least minimally configured **and is running**, and a mythconverg.sql is in place, you will not be able to configure your frontend, and will end up in that frontend configuration loop.

Make sure no mythfrontend process is running. Run your mythtv-setup command sudo, as most backend processes will depend on "root", i.e. :

sudo mythtv-setup 

or sudo XLIB_SKIP_ARGB_VISUALS="1" mythtv-setup

whichever is working for you. When the backend is configured with at least 1 tuner card, in accordance with [http://www.mythtv.org/wiki/User_Manual:Detailed_configuration_Backend](http://www.mythtv.org/wiki/User_Manual:Detailed_configuration_Backend)

(Note that even though this is a single/machine backend/frontend, you'll still want to set the PIN number to 0000, as described in the wiki manual, just to avoid future frontend connection problems.)

After getting so much as a single tuner added to the backend configuration (as described here: [http://www.mythtv.org/wiki/User_Manual:MythTV_structure](http://www.mythtv.org/wiki/User_Manual:MythTV_structure)  ), and exiting, you can see whether you can start the backend in a terminal with:

 sudo mythbackend  

(as described in the wiki manual; mythbackend has no graphical component, requires no XLIB workaround). If it runs and **is still running**, then you can harvest the password Ubuntu has generated for you in /home/mythtv/.mythtv/mysql.txt and use this password as the mysql password in the configuration screen that will pop up  when now from another terminal you run (as your regular user) 

XLIB_SKIP_ARGB_VISUALS="1" mythfrontend


And hopefully now (if your backend's running) with IP = 127.0.0.1, user = mythtv, password = [whatever the file says it is], and PIN = 0000, you should finally be able to  escape that maddening frontend config screen and proceed to the initial frontend setup. (Described here: [http://www.mythtv.org/wiki/User_Manual:Detailed_configuration_Frontend](http://www.mythtv.org/wiki/User_Manual:Detailed_configuration_Frontend)  )
