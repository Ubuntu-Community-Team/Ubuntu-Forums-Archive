Since upgrading to 16.04 LCDd (for my imon LCD screen) stopped starting on boot. However, it runs fine when I manually start it from the command line (/etc/init.d/LCDd start).


Any ideas how to fix this?

### Post by 74corvette on 2016-06-01
Not sure if this is your exact issue but when I upgraded from 14.04 to 16.04 I also had a similar issue with LCDd not starting up at boot.  If I manually started it everything worked fine.  I traced the issue to Ubuntu switching from upstart to systemd.  When rebooting in your grub menu you can try selecting the kernel with (upstart) that should boot with the old upstart and LCDd should work.  Since systemd is here to stay with Ubuntu I run this command and now I can boot up in the default systemd environment and LCDd starts up like before.

sudo systemctl enable LCDd.service
