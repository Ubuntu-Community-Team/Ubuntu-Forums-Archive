There is an effort to move everything to a web-based setup, mostly this is to make it so you can have a gui-less backend. If I'm to take your list point by point

1) While installing the software from the CD, I selected 'Master Backend' as this machine's role, but when I get to this screen the Local IP Address defaults to an address that is incompatible with a Master Backend configuration.

This isn't necessarily true. A master backend can work on both localhost and the private IP address, depending on how you are configuring things.

2) The instructions for this field say "Enter the IP address...", but should say "Select the IP address from the drop-down list."

This seems like it might be a theming issue, I'll take a look.

3) If the user tries to type the IP address, there is no feedback to say "Oops, you must use the drop-down".

True, there isn't any feedback. I'm not sure if this is a theming issue (as there is an option for some things to provide a help-text). This might be solved by solving #2 above

4) The Master Backend IP Address field does not offer you a drop-down list; you must type in the IP address. This isn't good UI design; the fields look pretty much the same, but one requires that you type, and one requires that you use the drop-down.

This is because one is the local backend and one is for the master backend. The local backend can be a drop down as you can query what ip addresses are on the local machine. The master backend could be on a different machine though, so it cannot be a drop down (it might be able to be a drop down, as you can sometimes query the network for the backend)

5) The Master Backend IP Address _could_ offer a drop-down, or could just default to the machine's current IP address.

See #4

6) Since I already selected the role as "Master Backend" at install time, this screen could change and have only one IP address. We know they must be the same, so why force the user to do it twice? If there is only one, then you can't make the mistake of making them different.

The master backend setting during installation is a Mythbuntu addition, meaning that certain packages get installed. This could be easier, but requires developer time. 


To answer your question "How do we interest the developers in investing effort on the installation process, to make it much simpler for newbies", you can either pay a developer or submit a patch. I've written a lengthy install/setup guide that if followed should get around some of the issues you found, but the issue we usually have is one of manpower.
