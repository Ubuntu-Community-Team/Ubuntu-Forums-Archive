### Post by phillfri on 2008-02-26
I'm totally new to Ubuntu, so take my comments with a grain of salt. I might just well have no clue what I am doing :>)

First a couple MythTV issues without the propietray ATI driver installed.

(1) During the installation process I goofed some stuff up when defining cards (I have a PVR-150 and an HDhomerun), so I deleted them and recreated them. When I view the Information screen in MythTV my cards are numbered 3 and 4 respectively. It seems that deleting a card definition does not recalibrate the card numbering. I wonder if this might not cause MythTV to think it has 4 cards installed.

(2)  When I try to view television with the default graphics driver installed I get nothing. It just drops back to the main menu.

Now the propietary ATI driver:

ATI driver installation produces no output on x1250. Nothing but a black screen.

Now, I have been working on a PCLinuxOS MiniMe 2008 MythTV setup  also and managed to get a somewhat working xorg.conf file set up for my dual-head display devices using the ATI 8.1 Catalyst release, (1) a 15" Ilo monitor with a VGA>VGA connection and (2) a 27" Ilo HDTV running on a DVI>HDMI connection.

After recovering with a safe boot, I looked at the xorg.conf files that were in the X11 directory and none of them look anything like the xorg.conf file I used in my MiniMe MythTV. Its as if aticonfig isn't even being run. There are absolutely no entries in any of these files that are typical for the ati driver.

I also took a look at xorg.0.log and noted that even when the system is using the default driver it assigns a vertical or horizontal frequency value in the 6,000,000 plus range (can't remember exactly - but its definitely not a correct reading).

The Screen and Graphics Preferences program has no entries available in any of its dropdown boxes, no graphics card listed in the graphics tab, and bombs out if you try to make any selection in the dialog box.

I think I'll check the forums for ATI compatability and maybe drop back to the last version if others have managed to get ATI to work on it.

### Post by foxbuntu on 2008-02-28
1) I am not sure what you mean by: > It seems that deleting a card definition does not recalibrate the card numbering But, if you deleted the capture cards from the backend config and re-created them, the current config should be fine and whatever you deleted before should not effect it.

2) If you did not change anything during the install you proably got the vesa driver which is just generic and would cause this issue. 

3) I would suggest you stick with the most current open source driver relase of fglrx as it will be the most stable for your Mythbuntu system. You can enable it via the Restricted Drviers manager, which is also listed in the Mythbuntu Control Centre (Utilites/Setup > Setup > Mythbuntu)
