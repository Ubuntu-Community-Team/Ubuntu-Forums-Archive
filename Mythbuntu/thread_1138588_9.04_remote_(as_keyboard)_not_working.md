I don't have a solution, but I think I know what the problem is, since I did a clean install of 9.04 yesterday on a machine that formerly ran 7.10 and was configured similar to yours in this regard.

xmodmap is kaputt.

Which is to say that you can still invoke xmodmap, and load text files of keycodes with xmodmap, but they will be cheerfully ignored by the Xserver. If you were using xmodmap as a startup application to load extra keycodes so that an IR remote similar to the Firefly mini could function as a keyboard, this attempted xmodmap at startup seems to have one of three results. 1) Cheerfully ignored and the standard keyboard is loaded with no extensions. 2) Hangs the keyboard for a while until xmodmap (apparently) times out, whereupon the standard keyboard resumes. 3) Hangs X completely.

Now I'm not sure, but in the research I've attempted to do, I think xmodmap is being phased out by design, so that this is a feature, not a bug. But I wish someone could confirm this.

Nor am I sure whether one now is expected to compile a config file for a whole new keyboard for Xkeyboard extensions if one just wants to activate a few extra keycodes on his current default keyboard (Aarrgh!). If this is in fact the case, I haven't found a really good (or even moderately bad) tutorial to help with such a project.

In the meantime, unless someone has a good simple solution, it may be time to swallow hard and LIRC up. :(
