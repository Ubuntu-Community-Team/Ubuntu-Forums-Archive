### Post by SiHa on 2009-09-18
WTF!!

At first glance it looks very bad, they talk about encrypting the EPG data via lookup tables which would only be supplied to hardware vendors who agree to implement 'content management' in their equipment. Also there will be a requirement on the vendors to build in software upgrade capability to the boxes for when the tables are (inevitably) broken. So the material itself would still be Free to Air and thus meet the public broatcasting license requirements.

But (and correct me if I'm wrong) this would only affect people, like me, who use the broadcast EIT data. I think if you're using an xmltv grabber, then it is no different (from a broadcast EPG point of view) to simply setting up a manual recording, as long as you know how to find the channel on the multiplex. Of course it would rather depend on xmltv data being available for BBC HD...

Then again, neither of my systems is HD-capable (or DVB-T2 for that matter) at the moment, so I don't think it'll affect me too much. It'll be interesting to see what ofcom does though.
