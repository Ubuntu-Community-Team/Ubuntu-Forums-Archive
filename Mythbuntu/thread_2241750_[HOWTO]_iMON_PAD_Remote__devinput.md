### Post by JamesNorris on 2014-08-28
Every time I reinstall Mythbuntu (or sometimes after updates) I lose functionality of my iMON PAD remote control. Here's how I got it working this time; hopefully others will find it helpful too. 

sudo irkeytable
will give details of the remote, the important line is this one:
**Name: iMON Remote (15c2:ffdc)**

We now know the location of the remote in /dev/input so it is time to edit the lirc files. 

The details from irkeytable are used to point the REMOTE_DEVICE section to /dev/input/by-id/usb-15c2_ffdc-event-if00
REMOTE_DRIVER needs to be set to devinput and unlike other tutorials on this topic, the LOAD_MODULES needs to be true to load the kernel modules for the remote, otherwise all we have working is the mouse part of the remote. 

```

# /etc/lirc/hardware.conf
#
REMOTE="Soundgraph iMON PAD IR/VFD"
REMOTE_MODULES="lirc_dev lirc_imon"
REMOTE_DRIVER="devinput"
REMOTE_DEVICE="/dev/input/by-id/usb-15c2_ffdc-event-if00"
REMOTE_SOCKET=""
REMOTE_LIRCD_CONF="imon/lircd.conf.imon-pad"
REMOTE_LIRCD_ARGS=""

START_LIRCD="true"
#START_LIRCMD="false"
LOAD_MODULES="true"
LIRCMD_CONF=""
FORCE_NONINTERACTIVE_RECONFIGURATION="false"
START_LIRCMD=""

```

lircd.conf needs to be edited with only the following line:
```
include "/usr/share/lirc/remotes/devinput/lircd.conf.devinput"
```

A reboot (maybe unnecessary) should load the kernel modules for the remote, so irw* should *now work. 

The ~/.lirc/mythtv file generated by the MythTV installation and Myth Control Centre gives all kinds of key mappings that my remote doesn't have, so working through the remote with irw and mapping the correct keys in ~/.lirc/mythtv should get it working.
