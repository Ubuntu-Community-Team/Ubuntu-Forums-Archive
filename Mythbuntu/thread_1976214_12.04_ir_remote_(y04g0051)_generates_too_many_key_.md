### Post by declanmullen on 2012-05-08
Hi 

I've got a Leadtek dtv2000ds and its bundled infrared remote (remote model is y04g0051). 

The remote is working absolutely fine with 'buntu 10.10 (see post [http://ubuntuforums.org/showpost.php?p=10771599&postcount=7]("http://ubuntuforums.org/showpost.php?p=10771599&postcount=7"))

I've done a fresh install of 'buntu 12.04. The remote is detected by the kernel and the dmesg output shows that it seems to recognising it and assigning a y04g0051 keytable and ir-keytable lists the remote too. 

However sometimes a press of one of the remote's keys will be interpreted as more than one key press, even when the key is pressed very briefly. This can be seen when using "ir-keytable" with it's "-t" option or when using the remote with an application (via lirc) like the mythfrontend. Increasing the repeat timing parameters (via ir-keytable and its "-P" and "-D" options) does modify the repeat behavior when you continue to hold down the key (as expected), however it does not prevent some of the brief single key presses from being interpreted as multiple key presses. 

This behavior is mentioned by Cloggy at [http://www.linuxtv.org/wiki/index.php/Leadtek_WinFast_DTV2000DS#Remote_Control_Support_.282.29]("http://www.linuxtv.org/wiki/index.php/Leadtek_WinFast_DTV2000DS#Remote_Control_Support_.282.29") when he was using the remote with 'buntu 11.04. (Cloggy's advice is to not use the remote.) 

Does anyone know how to get this remote to not generate multiple key presses in 'buntu 12.04 when the key is actually only pressed once ?   

Thanks,
Declan
