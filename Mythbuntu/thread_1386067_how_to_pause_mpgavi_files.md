### Post by klc5555 on 2010-01-20
At least as recently as mythtv 0.21, if the file is in your mythtv videos directory along with your ripped DVD files, then you should be able to build a standard mythtv frame index for it from a prompt with mythtranscode, i.e.:

mythtranscode --mpeg2 --buildindex --video -i [path/and/filename]

In addition to building an index, this command should leave behind a zero-byte tmp file, (filename.avi(or.mpg).tmp), which zero-byte file may then be deleted.

Even with ripped DVD files successfully done with myth, I find that about half the time the automatically generated frame index generated by the myth ripper is wrong (absent entirely or seriously too short). The above procedure works for these DVD ripped files as well.

### Post by klc5555 on 2010-01-20
Are you using the mythtv internal player to play a video which you've placed into your /mythtv/video directory? The internal player can't do much of anything properly (pause/play/skip/jump) unless the video or rip has a frame index in mythconverg. The myth ripper (usually) tries to build one for DVD rips. But if you're just dumping some video file or other from elsewhere into your /mythtv/video directory, you have to build the index yourself.

You did not mention which version of Mythtv/Mythbuntu you have installed. For Mythtv versions through 0.21, mythcommflag would not generally build seektables correctly for dvb recordings. Also dvd rips, or manually added videos. For versions of mythtv through 0.21, the only way to build a seektable that would actually work on a digital recording of any sort was mythtranscode --mpeg2 --buildindex --video -i [filename]

Mythcommflag's shortcomings may have been corrected in Mythtv 0.22. The wiki page for rebuilding seektables has since been emended and the alternative method of using mythtranscode removed.
