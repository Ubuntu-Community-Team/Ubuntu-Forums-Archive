### Post by klc5555 on 2010-02-01
Covered in various places, including this thread here: [http://ohioloco.ubuntuforums.org/showthread.php?t=1377238&highlight=1600+drivers](http://ohioloco.ubuntuforums.org/showthread.php?t=1377238&highlight=1600+drivers)

Basically the standard cx18 beta driver for the 1600 in Linux is not up to snuff with regard to QAM and signal strength. To improve this, for the foreseeable future, you'll need to compile and install Devin Heitmuller's optimized driver yourself.

Eventually, I suppose, the optimized driver will make it into the standard kernel tree. But not yet.
