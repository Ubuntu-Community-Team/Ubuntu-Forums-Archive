### Post by thrytis on 2011-07-08
I've recently upgraded from 10.10 to 11.04, and i'm in a similar situation where arrow and select keys work, but lircd doesn't appear to be processing any of this.  I've even tried without lircd running and everything functions the same way.

One thing with your configuration is that it appears to be using the old lirc_imon driver, where as now it should be using /dev/input.  See [this page]("http://wilsonet.com/?page_id=95") for more information.  You would need to change the module (REMOTE_MODULES) from lirc_imon to imon, and then change your device to the input device, e.g. /dev/input/by-id/usb-15c2_0038-event-mouse.  That said, it still doesn't work for me under 11.04.  I'm hoping i can glean some useful help out of your thread for my problem.

### Post by thrytis on 2011-07-08
Mine is back working, so i can give some more help here.

In your /etc/lirc/hardware.conf file, change the REMOTE section as follows:

> 
#Chosen Remote Control
REMOTE="Linux input layer (/dev/input/eventX)"
REMOTE_MODULES=""
REMOTE_DRIVER="devinput"
REMOTE_DEVICE="/dev/input/by-id/usb-15c2_0038-event-if00"
REMOTE_SOCKET=""
REMOTE_LIRCD_CONF="devinput/lircd.conf.devinput"
REMOTE_LIRCD_ARGS=""
I'm guessing here at the REMOTE_DEVICE.  It'll be whatever is valid for your device "iMON Remote (15c2:0038)".  You can find that out by looking at your /proc/bus/input/devices for the event number, then finding out which device in /dev/input/by-id/ links to it.  You don't want to use /dev/input/eventX directly because that will change.

Next update your /etc/lirc/lircd.conf to include "/usr/share/lirc/remotes/devinput/lircd.conf.devinput" rather than lircd.conf.imon-antec-veris.

Check if lirc_imon is loading (lsmod | grep lirc_imon).  If it is, blacklist it by creating a file blacklist-lirc.conf in /etc/modprobe.d, containing the line "blacklist lirc_imon".  I think Ubuntu is already doing this though.  Restart if you have to make this change, and in any case make sure the imon module is loading (lsmod | grep imon).

You'll probably need to update your ~/.lirc/mythtv file to reflect the codes from the lircd.conf.devinput file.  You can use irw if you need to see how the keys map.

The thing that was causing a problem for me in the upgrade is that mythtv was using /dev/lircd, while lircd was outputting to /var/run/lirc/lircd.  I don't know which changed, but i updated mythtv through the settings to point to /var/run/lirc/lircd and it started working again.

The only issue i have left is the power on button doesn't work after the upgrade.  I would think that would be independent of the software, but i have to investigate it further.
