### Post by Cheesehead on 2013-01-13
It's unlikely to be a typing error. The various build-and-package systems are automated in order to prevent typos.

It's much more likely that 2.1 existed, and has been superseded and replaced by 2.2. If so, then your package manager has out-of-date information in /var/lib/apt/lists and needs to be refreshed.

The command [FONT="Courier New"]sudo apt-get update[/FONT] will refresh (update) /var/lib/apt/lists, and usually runs automatically when you start Software Center or Update Manager or Synaptic. But you do need to run it manually when you update on the command line.

The #1 cause of update failures is an out-of-date /var/lib/apt/lists.

The #2 cause of update failures is interference from a PPA. Disable them and then try updating again.
