### Post by cabbage on 2010-06-03
Did some more testing and had the same backend errors when using either "mythshutdown" or the normal "shutdown" to stop the system. Also noticed that mysql is never stopping cleanly as there is nothing in syslog for it. However, manually stopping the backend and then mysql works.

So, I created a little script to do this:

> #!/bin/bash
# mythcleanshutdown
#
# Script to stop mythbackend and mysql cleanly before initiating a system shutdown.

trap "" 1
trap "" 15

stop mythtv-backend                                                                                 
/etc/init.d/mysql stop                                                                              
                                                                                                    
shutdown -h -P now

I added this into visudo and changed the mythwelcome and backend shutdown/halt commands to run "sudo -H mythcleanshutdown" and all is good!

So I'm guessing this isn't really a mythtv issue, more a karmic one... I wonder if Lucid has the same issues?

### Post by cabbage on 2010-06-08
Actually, all was almost good. There is no need to replace the "mythshutdown --shutdown" command in the backend as this is needed to write the BIOS wake up alarm time. It then calls the shutdown script defined in mythwelcome.

So just change the mythwelcome shutdown command and all is good..!
