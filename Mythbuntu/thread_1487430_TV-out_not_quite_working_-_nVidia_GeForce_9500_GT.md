I've got a brand new build that I'm trying to get Mythbuntu working on.

The computer has an nVidia GeForce 9500 GT (EVGA) PCI Express video card with two DVI outputs and one S-Video output. I have a VGA monitor plugged into one port (with a DVI->VGA adapter), and I have a CRT TV plugged into the other port (with an s-video to composite cable).

I'm having trouble getting the TV working the way I expect. nvidia-setup is detecting the TV, but no matter what it does the screen remains blank. I've tried auto-configuring it with either "TwinView" or with Separate X screens, and neither seem to get the display working. I've included the xorg.conf files created by nvidia-setup for both TwinView and Separate below.

I was able to get it working by modifying an xorg.conf I found online (see below), but it was a few years old. I'm not sure if this is the "correct" way to do it, and it is set up using a separate X server (where as I think TwinView is considered the way to go these days). I can't seem to find any more recent xorg.conf files, as most recent instructions say to just use nvidia-setup to do it automatically.

Can anyone suggest how to get TwinView working, or should I continue using separate X servers? (I do like the idea of keeping the MythFrontend isolated so I can continue using my desktop while watching TV, does that change things?)

One other thing, even with the separate display "working" (using the configuration below), I can't seem to get anything to display on the separate screen, and when Ubuntu starts, a warning pops up saying "Notification Area" has quit unexpectedly".

I'm running Ubuntu 10.04 (Lucid)
