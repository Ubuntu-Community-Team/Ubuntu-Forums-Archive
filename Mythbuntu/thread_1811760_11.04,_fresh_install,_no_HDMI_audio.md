```

      type plug
      slave.pcm {

              card 0

      }
```

I've unmuted and put to 100% *all* sources that I find in both alsamixer and the GUI mixer (the default one in xfce).

That exact line works on the exact same hardware in 10.04, and yes, when I switch back to 10.04, with *everything* else the same, same volume levels on the TV, etc., it works fine in the old environment (I've got my legacy system running on one HDD and the new one on another).

I should also explain, I'm running a Biostar [TF7050-M2]("http://www.biostar-usa.com/mbdetails.asp?model=TForce%20TF7050-M2") board that's a bit old but has been chugging along like a champ for me.  It has an NVIDIA GeForce 7050PV chipset with onboard HDMI which works nicely, when you have things configured properly.

It's hooked up to a 42" LCD TV via the HDMI, which provides video *and* sound for me in 10.04, and I think I even had that working in 9.04.

So, after that long setup (thanks to those of you still reading), any one have any suggestions?  Does anyone with a similar board (NVidia, at least), have audio through HDMI working?

### Post by thevilla on 2011-07-29
I had the same issue on my Dell XPS 15.  I hope this will helps you.  The information was condensed from    	 	 	 	 	 	  [ftp://download.nvidia.com/XFree86/gpu-hdmi-audio-document/gpu-hdmi-audio.html](ftp://download.nvidia.com/XFree86/gpu-hdmi-audio-document/gpu-hdmi-audio.html)
 :

1.  aplay -L

This listed various devices.  I was looking for NVidia devices like

plughw:CARD=NVidia,DEV=7
    HDA NVidia, HDMI 0
    Hardware device with all software conversions

2. With the HDMI cable connected to my TV I went through each NVidia device testing with speaker-test.  1,7 ... 1 because this is common for GPUs apparently and 7 from the device above.  For me device 7 produced a test tone.

sudo speaker-test -c 2 -r 48000 -D hw:1,7

3. edit /etc/pulse/default.pa (after backing it up).

Find the line containing 

load-module module-alsa-sink

change to (match the device above)

load-module module-alsa-sink device=hw:1,7

4.  In sound preferences / hardware select the correct hardware device for hdmi and select the profile which produces sound.  In my case this was "Digital stereo (HDMI) nr 2"

Worked for me.  I hope it helps others too.  Someone who knows more about alsa / pulse might explain it better.
