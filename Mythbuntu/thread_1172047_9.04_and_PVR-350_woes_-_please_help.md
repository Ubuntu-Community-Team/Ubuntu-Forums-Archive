Mythtv should also be the group of the storage directory as well as the owner, e.g. the command ls -l /var/lib/mythtv will show the ../recordings directory as "mythtv mythtv". Your username should be part of the mythtv group (first time you run mythcontrol center > configure mythtv  from the frontend, after the installation is over, mythtv should ask if you want to be added to the mythtv group). Your storage permissions look ok, but non-members of the group don't have write access to your storage.

The /dev ownership looks ok --root should own the cards, but I'm not at a myth machine at the moment, and would have to check when I get home for certain.

Despite not having livetv, have you tried scheduling some kind of recording on some channel? If it can do this (since root runs the backend and handles scheduled recordings), and you can play the recording in mythtv, then I think you can be pretty sure that the livetv problem is a permission/owner/group problem somewhere between your username, the frontend that runs under your username, and the mythtv group.

But if you can't do a scheduled recording at all, then it must be something else.

### Post by klc5555 on 2009-06-03
I dunno --depends first on whether mythfrontend.real isn't completely exiting (and so xfce4 and X are still running under the blank-looking screen), or xfce4 has died (while X itself is running), or X itself has hung up/died and failed to respawn (which seems a bit odd, but I suppose is possible with the spiffy new version of Xorg that 9.04 uses).  

For diagnosing, anyway, if it's the 1st possibility, then killall mythfrontend.real (or maybe killall mythfrontend) from your prompt ought to restore the xfce4 desktop. If the 2nd or 3rd, then startxfce4 from the prompt may restart the environment/X. 

If you have installed the "dontzap" package and then disabled dontzap (or simply added the correct line to your xorg.conf), you'll also be able to kill/restart X and the frontend with the good old ctl-alt-backspace. Not a solution, of course, but helps keep everything usable while you're diagnosing.
