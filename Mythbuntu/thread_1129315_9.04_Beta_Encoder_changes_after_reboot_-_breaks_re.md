### Post by Dewey_Oxberger on 2009-05-02
Okay, that makes #4 and #5.  Now I know we have bug here.  I have the same problem.  The fix described in the bug report doesn't help for me.  The only fix that works perfectly is to not autostart mythbackend and to not autostart mythfront end.  Let the system boot then manually start them both.

The next best thing is to go to /etc/rc2.d and move S31Mythbackend to S99zMythbackend.  That mostly works for me.  Once in a while I'll get a bad query but not all the time.

It looks like mysql is not taking connections about 5-7 sec after it starts and it recovers on its own after several more seconds.
