0.25 -> 0.27 upgrade failed

 I recently upgraded Ubuntu from 11.10 Oneiric to 12.04.3 Precise.  This took most of a day.  Next on the agenda was MythTV so I changed repository to 0.27 and steamed ahead.  The upgrade failed at configuring *mythtv-database*.
 
 As far as I can tell, this seems to be caused by the script (/var/lib/dpkg/info/mythtv-database.postinst) trying to get the database details from somewhere and failing. I’ve read on the mailing list that this info is supposed to come from /etc/mythtv/config.xml but this file didn’t have any useful info in it (only a password). This is confusing as I expected this to be ~/.mythtv/config.xml which is where MythTV itself writes this information.

~/.mythtv/config.xml should be a symlink to /etc/mythtv/config.xml

The backend runs as the mythtv user, so it will look at /home/mythtv/.mythtv/config.xml which should also be a symlink to /etc/mythtv/config.xml

IIRC, it falls back to /etc/mythtv/config.xml if the one in the home directory doesn't exist.

It looks in the home directory first, because not all users have write access to /etc/mythtv.
