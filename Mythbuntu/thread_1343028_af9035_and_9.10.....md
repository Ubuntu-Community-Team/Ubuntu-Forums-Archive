### Post by Shihan74 on 2009-12-02
ended up solving my own problem as luck would have it by accident... If your going to compile as root, dont use fakeroot..

anyways, if anyone else has the af9035 with the ml5007 tuner (avermedia twinstar). It is possible to get the kernel source, patch it using the terratec drivers and then just grab the modules from the compiled tree and use them straight in the binary-generic kernel (or whatever kernel your using)..

Grab the stuff from here: [http://patchwork.kernel.org/patch/61950/](http://patchwork.kernel.org/patch/61950/)

Compile using this: [http://blog.avirtualhome.com/2009/11/03/how-to-compile-a-kernel-for-ubuntu-karmic/](http://blog.avirtualhome.com/2009/11/03/how-to-compile-a-kernel-for-ubuntu-karmic/) (you dont have to create a new config like he does with core2, just re-configure binary-generic or whichever kernel your using)

then grab the af9033.ko, dvb-usb-af9035.ko (put it in /lib/modules/<your-kernl>/misc) and mxl5007t.ko (goes in, and replaces /lib/modules/<your-kernel>/kernel/drivers/media/common/tuners/mxl5007t.ko) from the compiled source.

Theres probably an easier way to do this, but the kernel modules required for the avermedia twinstar dont come as compiled-outside-the-kernel modules (unlike the terratec ones).
