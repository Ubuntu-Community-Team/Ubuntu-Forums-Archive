### Post by Lepy on 2010-02-06
Try editing /boot/grub/grub.cfg and adding "vmalloc=256M" to the end of your kernel boot line. This is especially crucial if you are running the 1600 with an nvidia card.

Also, you could try updating V4L-DVB according to the this [link](http://www.linuxtv.org/repo/).

### Post by Lepy on 2010-02-07
Search for an entry that looks similar to this
for /boot/grub/grub.cfg
```
menuentry "Ubuntu, Linux 2.6.31-19-generic" {
        recordfail=1
        if [ -n ${have_grubenv} ]; then save_env recordfail; fi
        set quiet=1
        insmod ext2
        set root=(hd0,1)
        search --no-floppy --fs-uuid --set 8ec7639c-4bb8-4029-8db1-cb9c26419975
        linux   /vmlinuz-2.6.31-19-generic root=UUID=efefcbc0-a8e2-499e-aee2-631de3426c8a ro   quiet splash
        initrd  /initrd.img-2.6.31-19-generic
}

```

or /boot/grub/menu.lst
```
title           Ubuntu 9.10, kernel 2.6.31-19-generic
uuid            08aba5f1-ac04-45f9-a6be-6bcb73b13358
kernel          /boot/vmlinuz-2.6.31-19-generic root=UUID=08aba5f1-ac04-45f9-a6be-6bcb73b13358 ro quiet splash
initrd          /boot/initrd.img-2.6.31-19-generic
quiet

```

and edit the line "linux" or "kernel" line 
```
/vmlinuz-2.6.31-19-generic root=UUID=08aba5f1-ac04-45f9-a6be-6bcb73b13358 ro quiet splash
```

to
```
/vmlinuz-2.6.31-19-generic root=UUID=08aba5f1-ac04-45f9-a6be-6bcb73b13358 ro quiet splash vmalloc=256M
```

I used two different machines for this example, and your UUID will be unique for your HDD, so make sure to only add the vmalloc=256 to the end of the line.
