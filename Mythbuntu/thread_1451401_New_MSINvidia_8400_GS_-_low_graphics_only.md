At least through Ubuntu 9.04 which still uses the old version of grub, you'll append vmalloc=256M (or in some cases vmalloc=512M, if necessary) to the end of the kernel line in the menu.lst. As described in this basic editing guide (and very many other places): [http://grumpymole.blogspot.com/2007/05/ubuntu-how-to-edit-grub-boot-parameters.html](http://grumpymole.blogspot.com/2007/05/ubuntu-how-to-edit-grub-boot-parameters.html)

_Maybe_ after the thing works and is known to survive a reboot you can come back at some later time and edit the "kopt=root" line to make it survive kernel version upgrades too. But that's the last thing on the agenda.

My own menu.lst from the (admitedly Slackware) machine that holds my hauppauge 1600 looks like this: 

# GRUB configuration file '/boot/grub/menu.lst'.
# generated by 'grubconfig'.  Sun Dec 28 12:24:22 2008
#
# Start GRUB global section
#timeout 30
#color light-gray/blue black/light-gray
# End GRUB global section
# Linux bootable partition config begins
  title linux on (/dev/sda1)
  root (hd0,0)
  kernel /boot/vmlinuz root=/dev/sda1 ro vga=normal vmalloc=512m
# Linux bootable partition config ends
title --- For help press 'c', type: 'help'
root (hd0)
title --- For usage examples, type: 'cat /boot/grub/grub.txt'
root (hd0)

Barring that, I seem to recall that I added the vmalloc parameter to the default options section where you did (so it gets picked up during kernel upgrades is my understanding), but also at the end of the menu.lst file in the actual kernel line for the boot entry (under "End Default Options").

I don't have any 9.04 systems around, but here is the end one of my old old 8.10 menu.lst files I had a copy of.  Note the vmalloc at the end of the first line of kernel parameters.

```
## ## End Default Options ##

title		Ubuntu 8.10, kernel 2.6.27-7-generic
uuid		d458c578-454e-4833-8984-8d910e76bcef
**kernel		/boot/vmlinuz-2.6.27-7-generic root=UUID=d458c578-454e-4833-8984-8d910e76bcef ro quiet vmalloc=256M**
initrd		/boot/initrd.img-2.6.27-7-generic
quiet

title		Ubuntu 8.10, kernel 2.6.27-7-generic (recovery mode)
uuid		d458c578-454e-4833-8984-8d910e76bcef
kernel		/boot/vmlinuz-2.6.27-7-generic root=UUID=d458c578-454e-4833-8984-8d910e76bcef ro single
initrd		/boot/initrd.img-2.6.27-7-generic

title		Ubuntu 8.10, memtest86+
uuid		d458c578-454e-4833-8984-8d910e76bcef
kernel		/boot/memtest86+.bin quiet

### END DEBIAN AUTOMAGIC KERNELS LIST
```
