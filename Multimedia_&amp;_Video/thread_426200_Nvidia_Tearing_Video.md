### Post by FuturePilot on 2007-04-28
Open the Nvidia Settings tool. It should be under System Tools, but if it's not you can open a terminal and do ```
nvidia-settings
```
Go to Xserver Xvideo Settings and make sure that both boxes are checked that say Sync to VBlank 
Then go the OpenGL Settings and make sure the box for Sync to VBlank is checked.

If you're using XGL though, there's no hope. XGL tears videos apart like no tomorrow.

### Post by Tristan Schmelcher on 2008-03-06
herbster:

If you have turned on all of the Sync to VBlank options (and selected the right monitor for the blitter adapter sync) and you still have tearing, I recommend that you switch from TwinView to a multiple X servers set-up. This is very easy to do, you can just go in to nvidia-settings and choose that option in X Server Display Configuration -> Display -> Configure, then click Save to X Configuration File and restart. After you restart, go back in to nvidia-settings and there will now be a new section for the new X screen ("X Screen 1") with its own sync options; turn them all on too. The tearing should then be gone.

I was experiencing tearing on my second monitor, a Sony HDTV, and the sync options were't enough to make it go away; I had to do the above too. My theory is that the texture sync option and/or the OpenGL sync option always sync to the default/primary monitor in TwinView, whereas with separate X servers it's possible to have the apps on each screen sync to the right monitor.

After having used the separate X server mode, I now actually prefer it. The second monitor gets its own taskbar and program menu, so I don't have to walk back to my PC in order to start new programs. The only drawback is that only TwinView lets you drag windows across monitors, but for viewing movies I don't really need that.
