### Post by pytheas22 on 2007-08-13
Hi all,

I just got a new LCD monitor whose native resolution is 1440x900.  I edited xorg.conf so that that resolution would be available.  However, now when I reboot my computer, the screen is displaced by about four inches as soon as the login manager comes up, so that about 1/5th of my monitor's screen on the left is black, while the display gets cut off on the right.  It is in 1400x900 resolution when it does this.  It remains offset when I log in to gnome.

The problem can be fixed by changing to another resolution using preferences>>screen resolution and then switching back to 1440x900--if I do that, the screen is drawn perfectly.  Also, if I restart the X server again without rebooting, the screen is drawn correctly.  It's just the first time X is launched after a restart that the problem occurs.

If anyone has ideas on why this happens, I would appreciate them.

Alternatively, I was thinking that a second-best solution to the problem would be to automatically run a command that would reset the screen resolution whenever I log in to gnome, but I can't figure out what the command-line utility would be that could do that.  That would at least save me the trouble of having to go to Screen Resolution, change to 1024x800 resolution, and then back to 1440x900 every time I start the computer again.

My xorg.conf is the default one, except I changed the driver to "intel" from "i810," because in the past intel has been more stable for me, and I edited in the 1440x900 modeline.

Thanks a lot for any help.
