### Post by Yellow Pasque on 2008-01-25
type gconf-editor in a terminal and follow this path: /apps/gnome-power-manager/timeout
Make sure the sleep_display keys are set to 0.

If that doesn't work, it's probably X Windows blanking your screen. In the ServerLayout section of /etc/X11/xorg.conf, add this:
```
Option "StandbyTime" "0"
Option "SuspendTime" "0"
Option "OffTime" "0"
```

### Post by pja on 2008-05-19
Hi!  I have Ubuntu 8.04 installed on a Toshiba Sattelite A10 and it has been having this "hibernation" problem ever since I installed 8.04.  I'm not sure that previous versions had this problem but I may be wrong.  I have looked at this and several other forum threads and made the following changes which seemed to have fixed the problem.

```
Type "gconf-editor" in a terminal and when the Configuration Editor opens
follow this path: /apps/gnome-power-manager/timeout

Make sure that the sleep_display keys are set to 0.

    Name                    Value
    ----                    -----
    sleep_display_ac        0
    sleep_display_battery   0


If that doesn't work, it's probably X Windows blanking your screen.

In the ServerLayout section of /etc/X11/xorg.conf, add this:

Code:

Section "Monitor"
# Only change the DPMS value; leave all other values as set
    Option "DPMS" "false"
EndSection

and add this section

Section "ServerFlags"
   Option "BlankTime" "0"
   Option "StandbyTime" "0"
   Option "SuspendTime" "0"
   Option "OffTime" "0"
EndSection
```

Regards,
Peter :)
