### Post by henrykeskus on 2006-11-11
Hi,

  I don't really know if I'll be helping much with this comment, but I just want to tell you what I have found out so far.

  It (still) is impossible for me to get to use the 1280x800 resolution in Linux in a straightforward manner as I can in XP. BUT, I have been able to do it with a very weird tip:

  - If I leave connected an external VGA cable tied to a monitor (which doesn't need to be turned-on) in the very moment that I turn on the computer, then Linux seems to be aware that the 1280x800 resolution exists. Thus, KDM starts in native resolution and I just need to log in and set the KDE resolution to the native one once (and will be kept from that moment on).

  I changed also small things before getting the result above which I cannot tell for sure if are completely necessary for it to work, but I just use them:
  - the 915resolution script is executed everytime I boot,
  - the BIOS settings that I have changed: i) disabled the "autoscaling image" feature , ii) and enabled the "choose automatically between LCD or External VGA".

### Post by creigscofield on 2007-01-25
I had this problem for a while also.  First modify your 915resolution script to override all the 1024x768 modes and make them 1280x800 modes, I believe it Modes 34, 45, and 54. (For some reason when ubuntu would see a 1024x768 mode it would decide that that must be the correct mode and ignore my others)  Next change your /etc/X11/xorg.conf in a similar manner, removing all reference to 1024x768 modes.  Restart X and cross your fingers.  If it doesn't work try removing all resolutions that are not 1280x800.  Hopefully this works for you.
