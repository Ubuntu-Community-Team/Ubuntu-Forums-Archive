### Post by Zimmer on 2006-01-02
What looks wrong to me is your xorg.conf file where you have 2 device sections (and 2 monitor sections). One is set to use ati drivers and the other the fglrx from xorg...
The messages showing MESA means that the 3d drivers are not loaded properly...
Apparently this site will generate a config file for you, and is less pain than fglrxconfig....
I got my ATI 9600 to work with the xorg (fglrx) drivers following this wiki.
[https://wiki.ubuntu.com/BinaryDriverHowto/ATI](https://wiki.ubuntu.com/BinaryDriverHowto/ATI)
I do not use the ATI drivers from the ATi site.
There may still be issues with the AMD64 and ATI drivers but..I don't have an AMD64 so I can offer no advice there..
a quote from the wiki to give you hope of an eventual solution:

Common issue: If you have an nforce2 chipset motherboard or if your Xorg.0.log reveals troubles when initialising AGP, put this in the Device section of /etc/X11/xorg.conf: Option "UseInternalAGPGART" "no"

Note If you are having problems related to DRI or 3d acceleration and the following lines show up in your /var/log/Xorg.0.log

(WW) fglrx(0): Kernel Module version does *not* match driver.
(EE) fglrx(0): incompatible kernel module detected - HW accelerated OpenGL will not work

then make sure you have linux-restricted-modules installed for your kernel (type sudo apt-get install linux-restricted-modules-$(uname -r)). 

..and the important bit is to use "fglrx" in the config , as far as I can see..
EDIT....
Check in the Synaptic Package manager whether xorg-driver-fglrx is loaded...

Regards
