### Post by caintona on 2007-02-21
Im running an ATI x1650 pro agp and had a similar problem.  I have noticed after working with nvidia at work and ati at home.  ati sucks for linux.  plain and simple.

it looks like ati didn't look at the settings of the drivers when the install set it up.

Are they purposefully releasing broken drivers from their site?  I mean do they want me to only use a windows computer if i buy the thing from them?

what you need to do is (you should do all of these things in recovery mode or whatever it is called)

1.  use envy and uninstall the old ati drivers.
2.  restore the original /etc/X11/xorg.conf.  In my install the /etc/X11/xorg.conf~ had the original configuration(there were several in this dir.  I found one that worked then rmed all the other ones, then backed that one up make sure not to touch it again).  I knew so because it DID NOT have the "Identifier "aticonfig-Device[0]"" line.  it said something else.  if it does make sure you make a copy of this file )
3.  use envy to install the ati drivers again.

It worked after doing these three steps.

It looks like there is a gap in the envy program to remove the old settings from the old ati installation specifically with the "aticonfig" line in it.

### Post by NT4usB on 2007-02-21
easy way:
sudo dpkg-reconfigure xserver-xorg

some reading:
[http://users.bigpond.net.au/hermanzone/p7.html](http://users.bigpond.net.au/hermanzone/p7.html)

Hard way
Ctrl+Shift+F1 if you're not already at a black screen.
cd /etc/X11/ (note the capital X)

oops... are you familiar with a non graphical editor like nano or vi?
If not, google vi or nano and print a cheatsheet of commands and find the refresh rates of your monitor

sudo cp xorg.conf bkup.xorg.conf (make a backup!)

sudo vi xorg.conf

edit the vert and horiz refresh rates to suit.
save and close.

sudo /etc/init.d/gdm restart
