### Post by andrew.46 on 2008-02-26
Hi,

There is a way:


> **phoenix5002 said:**
> ok...well then are there any free programs that can convert from a different type into .wma available for linux?.

Try either:

```
$ ffmpeg -i input.wav -acodec wmav2 -ab 192k  output.wma
```

or

```
$ ffmpeg -i input.wav -acodec wmav1 -ab 192k  output.wma
```

Not sure if the repository ffmpeg can do this, mine is compiled from source with all the extras.

               Andrew

The ubuntu repository will never have a *ful*l ffmpeg for political / legal reasons. Try svn if you are keen:

```
$ sudo apt-get install build-essential subversion libogg-dev \
libtheora-dev libvorbis-dev liblame-dev libfaac-dev libfaad-dev \
libxvidcore4-dev liba52-0.7.4-dev libx264-dev checkinstall \
texi2html  info2man
$ cd Desktop
$ svn checkout svn://svn.mplayerhq.hu/ffmpeg/trunk ffmpeg
$ cd ffmpeg
$ ./configure \
  --prefix=/usr \
  --mandir=/usr/man \
  --disable-debug \
  --enable-shared \
  --disable-static \
  --enable-pthreads \
  --enable-libtheora \
  --enable-libvorbis \
  --enable-gpl \
  --enable-pp \
  --enable-swscaler \
  --enable-x11grab \
  --enable-libmp3lame \
  --enable-libfaac \
  --enable-libfaad \
  --enable-libxvid \
  --enable-liba52 \
  --enable-libx264
$ make
$ sudo checkinstall -D
```

Naming of the dev files may differ in your version of Ubuntu, these are for Hardy Heron, but I suspect only the liba52 file may be different.
