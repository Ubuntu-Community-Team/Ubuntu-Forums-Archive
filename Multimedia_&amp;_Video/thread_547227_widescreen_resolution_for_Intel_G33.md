### Post by dcstar on 2007-09-12
> **pfeifferaj said:**

Any thoughts on how to fix this issue? -   Thanks

When I had an old Nvidia card I had to manually add in some modelines into xorg.conf (created by the **gtf** utility) to get it to work at 1440x900, you may have to do something similar.

The gtf command string is pretty simple:

```
# gtf 1680 1050 60

  # 1680x1050 @ 60.00 Hz (GTF) hsync: 65.22 kHz; pclk: 147.14 MHz
  Modeline "1680x1050_60.00"  147.14  1680 1784 1968 2256  1050 1051 1054 1087  -HSync +Vsync
```
If your monitor res doesn't use 60Hz, then run it with the appropriate number.

IIRC you then put the modeline string in the "Monitor" section of xorg.conf, and then the various res entries in the "Screen" subsections.

Smileypaul, I had the same problem (left and right side of the desktop was off the screen in 1680x1050 - the actual width of the screen in pixels measured with kruler was about 1380 pixels).

I've found a solution - X picks an incorrect modeline.

Here's the bug I've filed in Launchpad that contains a workaround:

[https://bugs.launchpad.net/ubuntu/+source/xorg/+bug/212018](https://bugs.launchpad.net/ubuntu/+source/xorg/+bug/212018)

Search **/var/log/Xorg.0.log** for the proper modeline and force it using option "PreferredMode". The proper modeline should be logged among autodetected ones, the problem is X server picks a different one by default.
