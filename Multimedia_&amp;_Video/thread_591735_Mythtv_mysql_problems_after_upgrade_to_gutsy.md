### Post by PyotrG on 2007-12-26
I had a similar problem to yours and I found a solution:

When I installed MySQL I was invited to enter a password for the root account. Note that this is the MySQL root user (administrator) as opposed to the Linux system root user. They are different, unrelated types of user account. I entered a password.

I subsequently installed the MythTV packages. I believe one of them requires the MySQL root account to install the MythTV database (mythconverg). It presumes the password is not set. The database installation failed although I did not notice at the time.

When I tried to run MythSetup I saw a similar error message to that given in your latest post.

I uninstalled the MythTV packages, set the MySQL root user password to nothing using the MySQL Adminstrator utility and then re-installed MythTV. It worked. The mythconverg database had been created and I could continue with setting up MythTV.

So, in short, set the MySQL root password to nothing and try uninstalling and then reinstalling your MythTV packages.

Please let me know if this works for you.
