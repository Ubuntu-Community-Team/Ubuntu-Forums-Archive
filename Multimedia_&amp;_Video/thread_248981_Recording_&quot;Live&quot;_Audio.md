### Post by capsid on 2006-09-02
> I would love to build a computer to do the recording. The recording software I would like to use, should be able to record tracks when I need them, examples, opening song, sermon, etc. should all be seperate from each other.
In my experience, you should separate the editing stuff from the recording stuff.  You don't need to build a computer just to keep in the church.  You might want to use a camcorder with a good mic, or an ipod-ish thing that has a mic. Even your cd burner would work if it has a line in.  The point is that when you're recording, just record one long file.  Then, transfer it to your computer in whatever way you can that preserves the fidelity. THEN, you start chopping it up into separate tracks using an audio editor like Audacity.  

Even if you want to bring a desktop computer to the church and record with it (which you can do in Audacity), you would still probably want to just set the levels, hit record, and leave it alone until the service is over (making sure it doesn't get too loud and distorted).   

> Also, after recording, I would be able to burn a CD or convert the files or tracks to other formats like mp3s.

No matter what you record it on, even if it's a cassete tape player/recorder, as soon as you transfer it to the computer you can turn it into whatever kind of file you want. Again, Audacity alone can do all of this.  I don't know what your mastery of digital audio workstations is, but it sounds like you want to keep it as simple as possible.

I have an Acer laptop which I record the sermons live in through the mic input, this is mono. I use Audacity at the moment, (but looking into Ardour). I save as an Audicity project. Then later when I have a little more time I edit the file in Audicty and then save as an mp3 ready for uplaoding to the net. With audicity you can add as well as subtract audio. 

In stead of using a laptop you could use an mp3 recorder (like iRiver or iAudio) and then copy to PC for editing and burning.

### Post by NigO on 2006-10-27
> I would love to build a computer to do the recording. The recording software I would like to use, should be able to record tracks when I need them, examples, opening song, sermon, etc. should all be seperate from each other.


I am working on this at the moment, not complete by any means but the basic system is an old (500MHz) PC running Ubuntu, boot into audacity with a project name derived from date and time, record the service adding labels live.  

At the end of the service export the multiple WAV files then compress them in the background and optionally upload them to the web.  Also offer an option to burn a CD of the current or previous services.

It's for regular use, others have detailed options for 'specials', where you will want to do some editing afterwards, this will have that option too, but is designed to replace existing cassette recordings, some added features but little or no operator interaction required.

I've done it in the past using a customised version of the text interface 'record' program, but the graphical option is more appealing to non-technical users!

One thing I haven't sorted but would like to- if a USB stick or drive is inserted, find and delete the previous service and dump the new one onto it without user interaction.  Any pointers on an efficient way to automatically run a shell script when any USB device is inserted would be welcome!

Unlike the mainstream, I stick to desktop PCs for audio recording, less attractive for theft, easier to upgrade and repair, cheaper (or free at this spec) etc etc.
