### Post by Zorael on 2008-04-12
Okay. That Envy didn't work is alarming, so let's try it with the supported driver. This will involve several terminal commands, so prepare yourself. Here Be Dragons.


```
$ sudo apt-get autoremove --purge nvidia-glx*
```
This will remove the driver and any remnants, which seems to have caused the issue in your posted Xorg.0.log file (kernel mismatch = remnant).

```
$ sudo apt-get install nvidia-glx-new
```
Obviously, will install the nvidia-glx-new package with the driver therein.

I would suggest you use nvidia-xconfig to configure your /etc/X11/xorg.conf file, even if it messes up tabbing architecture. That irks me something fierce, so I do it the manual way. I'll describe that further down.
[list][*]**Automatic procedure:**```
$ sudo nvidia-xconfig --allow-glx-with-composite --add-argb-glx-visuals --composite
```
This will primarily add support for Compiz, but also enable the nvidia driver.



[*]**Manual procedure:**
If you didn't run the nvidia-xconfig command previously mentioned and would instead like to configure stuff manually, read on.

```
GNOME (Ubuntu): $ gksudo gedit /etc/X11/xorg.conf
KDE (Kubuntu):	$ kdesu "kate /etc/X11/xorg.conf"
Xfce (Xubuntu)	$ gksudo mousepad /etc/X11/xorg.conf
Terminal:	$ sudo nano /etc/X11/xorg.conf
```
Pick your poison. I'm not too sure about the Xfce command, but should work.

Make changes so it looks something like this:
```
Section "Device"
	Identifier	"Generic Video Card"
	Driver		"nvidia"
EndSection

[i]Section "Monitor"
	Identifier	"Generic Monitor"
	Option		"DPMS"
	Horizsync	30-70
	Vertrefresh	50-160
EndSection[/i]

Section "Screen"
	Identifier	"Default Screen"
	Device		"Generic Video Card"
	Monitor		"Generic Monitor"
	DefaultDepth	24
	SubSection "Display"
		Depth		24
		Modes		"1680x1050" "1280x1024" "1024x768" "800x600" "640x480"
	EndSubSection
	Option		"AddARGBVisuals"	"True"
	Option		"AllowGLXWithComposite"	"true"
	Option		"AddARGBGLXVisuals"	"True"
EndSection
```
The monitor part in italics is optional. Since you seem to have the same machine as I do, I thought I'd list it as well. Other people reading this should omit that, *AND* make sure only supported resolutions are listed under Modes.

The Identifier, Device and Monitor fields ***MUST*** match the other identifiers for the relevant sections (so if you have "Configured Video Device" instead of Generic, it must also say so there). Else; no screens available, safe mode.

Also, you might want to comment out EVERYTHING else listed under these sections. Especially long and weird Modes lines. This is, in essence, everything needed for a single monitor setup.[/list]


After this, you should have the nvidia driver installed (with the proper kernel), you should have your xorg.conf set up to use it, and you should have the extra settings needed for the Compiz window compositioner, which I guess is standard in Ubuntu nowadays.

If it somehow still doesn't work, let us see your /var/log/Xorg.0.log file. It will likely say "no screens available", and hopefully why if so.

### Post by Zorael on 2008-04-12
The GUI way would be to open up Synaptics (Gnome) or Adept Manager (KDE), find your nvidia-glx-new package (or just nvidia-glx, whatever is there), right-click and choose **Complete Removal**/**Purge**. This is the same as the first command listed in the earlier post. It will remove the driver and the remnant kernel.

Then find and install the nvidia-glx-new package in your chosen package manager.

Then enter the nvidia-xconfig command listed earlier.

If you end up in fail safe mode, again, show us your /var/log/Xorg.0.log file.
