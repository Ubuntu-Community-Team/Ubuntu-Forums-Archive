### Post by turerkan on 2007-12-23
i have spent a whole day trying to find out why i can't get the native resolution of my 205BW screen. I collected the information from various (means A LOT) forum posts and howtos, finally i got it working. Read on for the reason and solution of this problem.

**Reason:**

Blame samsung:P Monitors normally report their available settings to the graphics card, this is why most monitors work out-of-the-box when you just plug the VGA cable. But this 205BW DOES NOT report its modes correctly.

Because of that, you will not be able to configure your monitor through the graphical interface.


**Solution:**

You will have to edit your xorg.conf manually. To do this first open a terminal and navigate to /etc/X11

```
cd /etc/X11
```

then open xorg.conf in a text editor.
```
sudo gedit xorg.conf
``` 
(you can use whatever editor you wish, also gedit may not be available in your installation)

Find the section which goes like this:

```
Section "Monitor"
(......)
EndSection
```

Replace the section with this:
```
Section "Monitor"
       Option "DPMS"
       Identifier "Samsung205BW"
       VendorName "Samsung"
       ModelName "SyncMaster 205BW"
       HorizSync 31.4 - 80.0
       VertRefresh 56.000 - 75.000
       Modeline "1680x1050" 146.25 1680 1784 1960 2240 1050 1053 1059 1089
EndSection
```

This monitor section will correctly identify your monitor to X. It only defines the 1680x1050 resolution however. I didn't try other resolutions but my experience tells me they aren't going to work. If you happen to know the correct modeline for any resolution, feel free to post here.

Now lets assign our monitor to the screen. Find this part:

```
Section "Screen"
(.........)
EndSection
```

And replace it with the following.

```
Section "Screen"
	Identifier	"Default Screen"
	Device		"DEVICE ID"
	Monitor		"Samsung205BW"
	Defaultdepth	24
	SubSection "Display"
		Depth	24
		Modes		"1680x1050"
	EndSubSection
EndSection
```

We are not done yet. You have to update the *Device* field with the correct value from the Device Section.

Forexample, if your Device section goes like this:

```
Section "Device"
	identifier	"X1200"
	Busid		"PCI:1:5:0"
	Driver		"fglrx"
EndSection
```

Then your Device ID is "X1200". So your Final Screen section would be:

```
Section "Screen"
	Identifier	"Default Screen"
	Device		"X1200"
	Monitor		"Samsung205BW"
	Defaultdepth	24
	SubSection "Display"
		Depth	24
		Modes		"1680x1050"
	EndSubSection
EndSection
```


If you filled your device ID correctly, it is done!
Save the xorg.conf file. 
Close all running applications.
hit CTRL+ALT+Backspace to restart X.


Here is a copy of my entire xorg.conf:

```
# xorg.conf (xorg X Window System server configuration file)
#
# This file was generated by failsafeDexconf, using
# values from the debconf database and some overrides to use vesa mode.
#
# You should use dexconf or another such tool for creating a "real" xorg.conf
# For example:
#   sudo dpkg-reconfigure -phigh xserver-xorg
Section "Files"
EndSection

Section "Module"
	Load		"GLcore"
	Load		"glx"
	Load		"dbe"
	Load		"v4l"
EndSection

Section "InputDevice"
	Identifier	"Generic Keyboard"
	Driver		"kbd"
	Option		"CoreKeyboard"
	Option		"XkbRules"	"xorg"
	Option		"XkbModel"	"pc105"
	Option		"XkbLayout"	"tr"
EndSection

Section "InputDevice"
	Identifier	"Configured Mouse"
	Driver		"mouse"
	Option		"CorePointer"
	Option		"Device"	"/dev/input/mice"
	Option		"Protocol"	"ExplorerPS/2"
	Option		"ZAxisMapping"	"4 5"
EndSection

Section "Device"
	identifier	"X1200"
	Boardname	"vesa"
	Busid		"PCI:1:5:0"
	Driver		"fglrx"
	Screen	0
EndSection

Section "Monitor"
       Option "DPMS"
       Identifier "Samsung205BW"
       VendorName "Samsung"
       ModelName "SyncMaster 205BW"
       HorizSync 31.4 - 80.0
       VertRefresh 56.000 - 75.000
       Modeline "1680x1050" 146.25 1680 1784 1960 2240 1050 1053 1059 1089
EndSection

Section "Screen"
	Identifier	"Default Screen"
	Device		"X1200"
	Monitor		"Samsung205BW"
	Defaultdepth	24
	SubSection "Display"
		Depth	24
		Modes		"1680x1050"
	EndSubSection
EndSection

Section "ServerLayout"
	Identifier	"Default Layout"
  screen 0 "Default Screen" 0 0
	Inputdevice	"Generic Keyboard"
	Inputdevice	"Configured Mouse"
EndSection
Section "ServerFlags"
EndSection
```

### Post by Joren on 2008-06-02
Thank you very much for this!!!
I've spent a lot of time with my Monitor in stupid 640x480 resolution. It happened after I used Windows one time (I have dual boot) and I think Windows made a small update or installed something (but I wasn't really paying attention), and when I booted Ubuntu again my resolution was gone!

Your thread was really usefull for me because I didn't know what to do and to search for. Finally I saw your post and used your xorg settings and this is my working perfectly xorg.conf. (Identifier is different but the rest is equal)


Section "Device"
	Identifier	"Configured Video Device"
	Driver		"nvidia"
	Option		"NoLogo"	"True"
EndSection

Section "Monitor"
	Option		"DPMS"
	Identifier	"Samsung205BW"
	Vendorname	"Samsung"
	Modelname	"SyncMaster 205BW"
	Horizsync	31.4	-	80.0
	Vertrefresh	56.000	-	75.000
  modeline  "1680x1050" 146.25 1680 1784 1960 2240 1050 1053 1059 1089
EndSection

Section "Screen"
	Identifier	"Default Screen"
	Device		"Configured Video Device"
	Monitor		"Samsung205BW"
	Defaultdepth	24
	SubSection "Display"
		Depth	24
		Modes		"1680x1050"
	EndSubSection
	Option		"AddARGBGLXVisuals"	"True"
EndSection
