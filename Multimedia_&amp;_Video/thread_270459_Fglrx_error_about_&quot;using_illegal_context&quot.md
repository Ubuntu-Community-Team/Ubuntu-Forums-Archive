### Post by misha680 on 2006-10-03
Hi, I have recently switched to edgy and am now having some problems with my Xgl/fglrx setup on my Mobility Radeon 9000 (yes, I tried AIGLX/r200 open source drivers, but it is _really_ _really_ slow on my computer at 24 bit depth, whereas fglrx with Xgl and beryl is quite speedy - i guess mileage varies with fglrx vs open source depending on graphics card).

Anyhow, I am trying to solve some problems I've been having with Xgl in edgy. At first it was crashing randomly, but I think (I hope) I have fixed that now by adding some options in Xorg.conf. However, a couple of minutes ago I did a:

```
sudo /etc/init.d/dbus restart
```

because network manager (CVS) was doing some weird stuff, and this is what I do to reset it sometimes. For some weird reason, a nautilus window opened, and when I tried to close it by clicking on the X button in the top-right hand corner of the window, it did not close, but rather seemed to just ignore my command and get stuck in the "hand cursor" mode (like the cursor you get when clicking on the close button or moving a window) and I could not get out of that mode and get back to using any windows (and the window I was trying to close did not close).

Finally, I went to a text VT CTRL-ALT-F1, killed the nautilus process, back to CTRL-ALT-F7 and everything back to normal (no Xgl crash this time, which I am happy about, as before I messed with Xorg.conf it would usually  crash after getting stuck like this). I also saw the following error in my VT:

```
[17180704.632000] [fglrx:firegl_lock] *ERROR* Process 7913 is using illegal context 0x00000005
```

I think it is related. I used to see these errors before in Dapper (I think?) and ignore them, but I think this error might be the key to my problems with Xgl under edgy. Anyone seen this error before and, more importantly, know what it means or what to do to correct it?

Thanks
Misha
