### Post by meist3r on 2006-08-26
I had the same issues with the 8.26.18 drivers. Please do use the 8.27.10 drivers or later (latest 8.28.8 gives me troubles on video playback).
So here's how I install my ATI drivers all the time and I am having no problems except a stupid video playback bug with the 8.28.8 drivers. I use a 19" (DVI) and one 17" (VGA) monitor both running 1280x1024 combined to a big desktop of 2560x1024. My video card is a RadeOn X850 (R480) Pro 256MB AGP

So here's what you do:

Skip steps that you have already completed succesfully but please try the whole procedure once.

```
<!-- Howto compile ATI drivers from .run package.
Derived from http://www.ubuntuforums.org/showthread.php?t=204910 -->

1. Download ati-driver-installer-<version>.run file from www.ati.com or any other trustworthy source 
(ati.com frequently gives me false checksums so I have to download that file two or three times until it works).
#skip to step 3 if you have already built the drivers from the package once

2. Install necessary Tools:
	sudo apt-get update 
	sudo apt-get install module-assistant build-essential 
	sudo apt-get install fakeroot dh-make debconf libstdc++5 gcc-3.3-base

3. Create .deb Packages
	chmod +x ati-driver-installer-<version>.run
      <!-- This is when md5sum errors of the package show up -->
	./ati-driver-installer-<version>.run --buildpkg Ubuntu/dapper

4. Install .deb Packages
	sudo dpkg -i xorg-driver-fglrx_<version>-1_i386.deb 
	sudo dpkg -i fglrx-kernel-source_<version>-1_i386.deb 
	sudo dpkg -i fglrx-control_<version>-1_i386.deb

5. Remove old fglrx packages from /usr/src/
	sudo rm /usr/src/fglrx-kernel*.deb

6. Compile the new kernel module:
	sudo module-assistant prepare,update 
	sudo module-assistant build,install fglrx 
	sudo depmod

```

Now here is what took me quite a while to figure out the options: 

```

7. Update the xorg.conf:

	sudo aticonfig --initial=dual-head --dtop=horizontal --screen-layout=right

```

You may have to enable the second screen manually so use

```

aticonfig --query-monitor

```
to verify which screens are already activated. 

My machine shows this:
```

maggus@412XPC:~$ aticonfig --query-monitor
  Connected monitors: crt1, tmds1
  Enabled monitors: crt1, tmds1

```

(initial=dual-head will tell the driver that you are actually running two screens and will drop additional "Screen"/"Device" and "Monitor" sections to xorg.conf, dtop will tell it were to put the desktop layout, screen-layout is another option to tell the driver were the second screen is)
