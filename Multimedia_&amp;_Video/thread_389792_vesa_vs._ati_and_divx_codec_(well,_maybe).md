```
Section "Device"
        Identifier      "Scheda video generica"
        Driver          "vesa"
        BusID           "PCI:1:0:0"

```
I'm happy with this solution, it works well, 2d is smooth and about 3d... I will wait for ATI; I overlooked or simpli forgot about the vesa driver, and the fglrx - both with [kx]?ubuntu precompiled restricted drivers and with manual compiling - never worked for me: glitches, missing background of windows and so on.

There is only one problem: video rendering.
Maybe it's the vesa driver not supporting some key function or extension, or I did not install some key codec but below you can see how kaffeine plays a divx video:
[img]http://www.massimilianodellarovere.eu/bildoj/ceterajxoj/snapshot2.png[/img]
This is the output of kaffeine-xine; kaffeine-mplayer shows correctly the video/images part but there is no sound!
It occurs only with divx files: for example a XVid works perfectly.

I installed all gstreamer plugins, everything listed in [_[COLOR="Purple"]Medibuntu's package page[/COLOR]_]("http://medibuntu.sos-sts.com/packages.php") and libxine-extracodecs.

Can anybody help me?
