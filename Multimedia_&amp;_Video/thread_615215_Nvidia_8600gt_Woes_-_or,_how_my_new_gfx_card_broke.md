### Post by Shazaam on 2008-01-23
This post might seem like a rehash of what's been tried before but there might be a few gems in here  :)
This is the way I got the 169.07 drivers to install and work in Gutsy (it should work for ANY nvidia card)-
1. Uninstalled ALL nvidia related files
2. Used this from the nv forums......
```
edit the /etc/default/linux-restricted-modules or /etc/default/linux-restricted-modules-common configuration file and disable the NVIDIA linux-restricted kernel modules (nvidia, nvidia_legacy) via:

    DISABLED_MODULES="nv nvidia_new"

Additionally, delete the following file if it exists:

    /lib/linux-restricted-modules/.nvidia_new_installed
```
3. Installed the kernel headers for my kernel=
```
sudo apt-get install linux-headers-$(uname -r)
```
4.Downloaded the driver from nvidia to my desktop
5.  build-essential (from Synaptic)- installed it.
6. Made sure pkg-config was installed with Synaptic
7. Hit control+alt+f1 and dropped to a prompt. Logged in
8. Shutdown X-
```
sudo /etc/init.d/gdm stop
```
8. Typed this in...
```
cd ~/Desktop
```
You should be in your  desktop folder. Type in ls and if your driver is listed you are in the right place
9. Typed this in...
```
sudo sh NVIDIA-Linux-x86-169.07-pkg1.run
```
(substitute your driver version number)

10. Answered yes to all questions.
11. Once the driver was done installing typed in  sudo reboot

There is a problem with the new update for xorg causing reboots while trying 3d apps. Look at post #6 by oscarsfriend to fix the symlink problem...
[http://ubuntuforums.org/showthread.php?t=671091&highlight=glxinfo](http://ubuntuforums.org/showthread.php?t=671091&highlight=glxinfo)

Links used....
[http://us.download.nvidia.com/XFree86/Linux-x86/169.09/README/index.html](http://us.download.nvidia.com/XFree86/Linux-x86/169.09/README/index.html)
[http://www.nvnews.net/vbulletin/showthread.php?t=72490](http://www.nvnews.net/vbulletin/showthread.php?t=72490)

### Post by estark on 2008-04-29
I found this thread through Google after having continuous problems with Hardy Heron on a new box.   I experienced the same black (power saving) screen with eventual hard lock-up that everyone else saw.  I tried (almost) every piece of advice in this thread, without any resolution.

I have:

CPU: AMD Athlon 3700+ San Diego
VGA: GeForce 8600GT 256
MB: Abit AN8 Ultra (nForce4)

Eventually, I gave up on the 8600GT and replaced it with an old 6600GT that I had lying around.  To my surprise, I had the exact same problems with the older card.  Obviously, my problem wasn't with the video card.  Looking back through the thread, I saw a pattern: The people reporting black screens and hard-locks all seemed to have motherboards based on the nForce4 chipset. 

As someone else has pointed out by now (I think), everyone experiencing this will want to go and check with their motherboard vendors.  Looking at Abit's website, I found that quite a few updates had been made to the AN8 BIOS over the last three years and a few of them addressed video card issues.

I took out my boot flash drive, put the AN8-21 BIOS update on it and after doing an X11 recovery and then a force reinstallation of the nvidia-glx-new drivers, the black screen disappeared.  The 6600GT and the 8600GT are now fully functional and running glxgears from within compiz without crashing and with good framerates.

So again, if you've got the black screen and hard-lock, check your motherboard and update the BIOS, especially if it uses the nForce4 chipset.
