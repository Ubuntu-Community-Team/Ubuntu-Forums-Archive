### Post by ItalOz on 2008-02-14
Hi,
since I had some trouble obtaining Direct Rendering with this video card I share my experience.

This is valid for

ATI Technologies Inc RV505 CE [Radeon X1550 64-bit]

and Ubuntu Gutsy 64-bits

I had compiz and desktop effects working but I could obtain them without direct rendering, I think that AIGLX is the name of the technology that allows to simulate 3D effects with indirect rendering.

Anyway the command
```
glxinfo | grep rendering
```

gave me something of the kind
direct rendering: No

and then "if you want to know why..." bla bla bla.

I tried to reconfigure the xorg.conf file several times 
I found out that the "radeon" drivers do not work with this graphic board.
I also tried issuing the command
```
sudo dpkg-reconfigure xserver-xorg
```

and tried to install the drivers with envy but nothing actually happened so finally I uninstalled completely the gnome desktop

well actually this was kind of a mistake :-$ but I run the command
```
sudo apt-get remove libgl1-mesa-glx libgl1-mesa-dri
```

so [COLOR="Red"]**DON'T DO IT**[/COLOR] unless you want completely uninstall the graphic server, by the way I was left with a basic B&W prompt and a sort of desperation.
But I found out that reinstalling is pretty easy with
```
sudo aptitude install ubuntu-desktop
```

that made the deal and I got my graphics back, this time of course with the basic Vesa capabilities and nothing else.

Please also note that all my KDE applications and Third party software were gone, my GNOME applications were doing fine. After reinstalling the software all the set-up was back because I never touched my home directory.

Then I installed envy and I asked it to install manually the oldest release of the Ati drivers.
When it came the time that it asks if I wanted to auto-configure my xorg.conf file I answered yes.

So I do post here my xorg.conf file in case someone might need it
