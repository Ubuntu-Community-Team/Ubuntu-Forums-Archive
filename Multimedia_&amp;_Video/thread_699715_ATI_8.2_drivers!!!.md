### Post by sloggerkhan on 2008-02-20
yes. You have to remove fglrx and then make sure dkms is clean before trying to reinstall fglrx. 

Then, after you build the debs from flgrx installer and install them,
you reconfigure your x server to use fglrx and then do aticonfig --initial.
At this point you will most likely need to check that your xorg.conf is clean of anything with an 'ati' driver and also add the 'module' section: 
```

Section "Module"
	Load  "bitmap"
	Load  "ddc"
	Load  "dri"
	Load  "extmod"
	Load  "freetype"
	Load  "glx"
	Load  "int10"
	Load  "vbe"
	Load  "GLcore"
EndSection

```
if it's missing.

Finally, restart xserver.

### Post by Melcar on 2008-02-24
Clean up your xorg.conf a bit.  You have three Device sections; you should only have two if you used aticonfig (it duplicates sections).  You should just leave the ones with aticonfig in them (comment out the rest just in case you need to revert).  This is how I would leave it:


```

Section "Monitor"
	Identifier   "aticonfig-Monitor[0]"
	Option	    "VendorName" "ATI Proprietary Driver"
	Option	    "ModelName" "Generic Autodetecting Monitor"
	Option	    "DPMS" "true"
EndSection

Section "Device"
	Identifier  "aticonfig-Device[0]"
	Driver      "fglrx"
        BusID       "PCI:1:0:0"
	Option	    "VideoOverlay" "on"
	Option	    "OpenGLOverlay" "off"
EndSection

Section "Screen"
	Identifier "aticonfig-Screen[0]"
	Device     "aticonfig-Device[0]"
	Monitor    "aticonfig-Monitor[0]"
	DefaultDepth     24
	SubSection "Display"
		Viewport   0 0
		Depth     24
	EndSubSection
EndSection
```

Just comment out the extra sections (don't delete them) in case something bad happens.

Also, check what video extension your video player is using.
