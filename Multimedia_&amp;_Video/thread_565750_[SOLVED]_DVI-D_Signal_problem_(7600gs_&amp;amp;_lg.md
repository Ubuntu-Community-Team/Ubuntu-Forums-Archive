### Post by Skerit on 2007-10-02
(In short) My screen only accepts DVI-D signals but I believe my graphics card is sending out something else, which makes it go into energy saving mode all the time ...

I bought an LG L204WT screen and an nVidia 7600 GS graphics card 8 months ago, the only problem is I never got the DVI to work...

**Solution**
There are some monitors which fail to provide valid EDID data to the computer. On my setup this is caused by faulty firmware on my monitor, itself! The solution is to find the valid EDID bin file on the internet. With a lot of luck I found mine!

Here's the EDID file: [http://www.nvnews.net/vbulletin/show...9&postcount=11](http://www.nvnews.net/vbulletin/show...9&postcount=11)

Then copy the file to, for example, /etc/X11 and add this to the xorg.conf file (in your device section)

```
Option "CustomEDID" "DFP-0:/etc/X11/edid.bin"
```

I'm also still using this option to indicate it needs to send out a digital signal instead of an analog one. (also in the "Device" section)
```
Option "ConnectedMonitor" "dfp"
```

Restart X and rejoyce. Do a happy dance!

Note: to find out if it really is your EDID which is wrong, execute following command:
```
sudo get-edid
```
If you get something in the line of:
```
The EDID data should not be trusted as the VBE call failed
EDID claims 255 more blocks left
EDID blocks left is wrong.
Your EDID is probably invalid.
```
Then you have found your problem!

### Post by Skerit on 2008-05-13
I've finally been able to fix it.
I read a bug report in launchpad and, apparently, some monitors fail to give the proper EDID data. My monitor, LG L204WT, does this because it has a firmware flaw when you try to connect it to a DVI port. Strange, but anyway ...

I searched the net for the correct EDID file, which you can save to your computer and, in an xorg.conf option, you can tlel it to use this information instead of the faulty info it gets.

Here's the EDID file: [http://www.nvnews.net/vbulletin/showpost.php?p=1389689&postcount=11](http://www.nvnews.net/vbulletin/showpost.php?p=1389689&postcount=11)

Then copy the file to /etc/X11 or where-ever and add this to the xorg.conf file (in your device section)

Option "CustomEDID" "DFP-0:/etc/X11/lgl204wt.bin" 

I renamed my edid.bin file to lgl204wt.bin
