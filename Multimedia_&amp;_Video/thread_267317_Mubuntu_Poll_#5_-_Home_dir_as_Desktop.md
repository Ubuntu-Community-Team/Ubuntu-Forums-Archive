### Post by dolson on 2006-10-07

### Post by markwren on 2007-01-21
I use ~/desktop because I like to have as FEW non-hidden files and subdirectories on HOME as possible. Using HOME would mean having all sorts of stuff lying around there. In fact I prefer to have almost everything I use hanging off HOME/work rather than HOME.

This is because I like to have a periodic Spring-clean, especially on upgrades. 

It's good to have /home on a separate filesystem in case you are forced to re-install on root if the upgrade goes wrong, but I often re-install anyway (first backing up a few key /etc configuration files).

And usually Gnome, KDE or Xface are upgraded with a new Ubuntu release. So on my personal workstation I create a fresh login and copy over my files from the single directory HOME/work. Of course I also salvage just the bare essential few hidden files and subdirectories too (email and bookmarks for instance). But the idea is to ditch the lot and start again, letting the new sytem re-initailise my desktop with all the new features clear and unconfused and losing all the obsolete bits of config and obsolete dot files.

In my long Unix and Linux experience this is easier if I keep my HOME directory as clear as possible.
