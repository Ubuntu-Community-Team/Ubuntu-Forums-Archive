### Post by crazybilly on 2007-07-11
One of things I miss most about Windows is foobar2000.  It was great for 3 reasons:

1.  the file browser panel used windows' file browser, which is how I want to browse my music.  there's no overhead b/c it's just nautilus, er windows ;), looking at the files for me
2.  tagging, particularly live bootlegs from etree, for example, was simple (with a plugin)
3.  converting files was simple
4.  (ok, 4 reasons) it would play anything (with the right plugin)

I've tried installing it under wine without much luck--columnsUI, at least the way I'd like to configure it, crashes and burns.

I've looked at lots of other music players on Linux without much satisfaction:

Amarok is too bloated (and runs a db)
MPD also runs a database
rythymbox (i think) was close but still didn't use nautilus to browse files
everything else seems a bit too simple 

Is there anything out there that would allow me to add files to a playlist w/o opening another nautilus window, but at the same time, uses the already-running nautilus to browse files?  I can use nautilus scripts for coverting, I think, if it comes to that. For that matter, it looks like there's [scripts to do the tagging w/in nautilus]("http://sourceforge.net/projects/etree-scripts/") too.

The real question is: is there a simple music player that wraps around nautilus somehow?
