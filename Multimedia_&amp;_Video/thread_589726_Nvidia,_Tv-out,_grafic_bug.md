### Post by 505 on 2007-10-24
Okay, I'm home and in a good mood, so you'll get the long reply. I use the Nvidia binary drivers 9631 from Nvidia's site. You need to install those, and during install you need to compile your own driver. For this you need the linux-headers package. The disadvantage of this is that any time a new kernel gets installed, X will fail, and you need to reinstall the driver.

As for my setup, I have my computer connected to my TV using a s-video cable and a scart converter. The TV is a PAL system. Using Linux/X is the only way I can get TV out in colour, switching to the terminal or in Windows only results in back and white.

When you install the drivers, you also get the Nvidia config tool. My advice, don't use it. You can indeed enable your TV with that, but it srews up your xorg.conf file.

Now for the file itself:

Default, the signal is only displayed on the computer. To change that, open xorg.conf, and change "ServerFlags". Uncomment the second one, and comment the first one. Save and press control+alt+backspace. X will restart and your TV will have a signal.
There is one big bug with using cinerama, the terminal does not work. You can however use 'xterm' as an alternative terminal.
