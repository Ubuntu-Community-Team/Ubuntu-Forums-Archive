### Post by DigitalJunkie on 2007-07-24
Hi everyone,

I've just installed a new LCD monitor, but after editing xorg.conf to use the native resolution and supported refresh rates, I noticed that OpenGL screensavers and games are not running smoothly. The screensavers show flickering horizontal bars, as if the image is being updated halfway during the rendering of a frame. OpenGL games (I've only tested this with Sauerbraten so far) don't have this flickering problem, but seem to simply drop a few frames every second, resulting in choppy motion.

I assume this has to do with the vertical refresh rate of my monitor and the video card output not being properly synced, but how do I fix this? I've tried adding a custom modeline, but that doesn't seem to solve the problem. According to the monitor's display menu, it's operating at 60 Hz, like it's supposed to.

### Post by MrHippocampus on 2007-07-24
What will probably solve this problem is turning on "sync to vblank" in the nvidia driver. Run "nvidia-settings" (its in the main menu or it can be run from a terminal), go to the OpenGL page and tick "Sync to VBlank"

---

### Post by DigitalJunkie on 2007-07-24
Wow. Thanks for the quick reply. I tried Sync to VBlank, and it makes all the difference in the world. Everything runs perfectly smoothly now.

Thanks for helping out, I never thought it'd be this simple. :)
