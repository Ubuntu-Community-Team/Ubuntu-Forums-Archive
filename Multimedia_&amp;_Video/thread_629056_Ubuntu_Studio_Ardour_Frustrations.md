### Post by athianos on 2007-12-01
Hey folks!

I am an Ubuntu fan, big time. I've 100,000 people working to make my life easier rather than 100,000 working to make it more difficult in microsoft land. That's all great.

My frustration lies in using ardour in ubuntu studio. I invested time in ardour because it seemed to be able to handle the basic (<8trac) demands I was throwing at it. Ardour tended to crash after I reached a fairly large archive of recordings that I was losing from my mix and I tried to clean up. OK. Fair enough. Ardour didn't want to load after I cleaned up a load of files it still wanted to refer to post clean up. Minor glitch. Save prior to clean up, right?  

Why oh why does Ardour not want to open the third mix I have created following after two major crashes of the same project? Ardour renders my Ardour files unopenable. I've uploaded the .avi files onto 2 new projects, Both files surpassed my expectations until, post saving, they refused to open...

How is it possible to create anything of value with a system that, in practice, locks you out of your work the moment you reach  the complexity required to be marketable? 

Help me out guys. Should I not be using ardour? Is there a more reliable system? I'd endorse Ubuntu all the way, but surely we can stay on top of this on a high profile release like Ubuntu studio.

Athianos
