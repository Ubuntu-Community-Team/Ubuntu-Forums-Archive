### Post by Jive Turkey on 2008-01-09
If you have an ATI card running the proprietary driver you xorg.conf device section might look like this:

```
Section "Device"
	Identifier  "ATI Technologies Inc Radeon R350 [Radeon 9800 Pro]"
	Driver      "fglrx"
	Option	    "DesktopSetup" "horizontal"
	Option	    "Capabilities" "0x00000800"
	Option	    "PairModes" "1920x1440+1600x1200"
	[COLOR="Red"]**Option	    "EnableMonitor" "crt1,crt2"**[/COLOR]
	BusID       "PCI:1:0:0"
EndSection
```

The "crt1,crt2" might say something else, but to get back to single monitor mode, change it to only the one monitor you want.

Alternatively, type:
```
sudo aticonfig --enable-monitor=crt1
```
where crt1 is the monitor you want, the other one will be disabled, if you have some other videocard I'm not sure.  Others may be able to help you if you post your xorg.conf file here.

### Post by luisito on 2008-01-09
First, let me tell you that you can see what went wrong if you look at the file  /var/log/Xorg.0.log

I don't have an intel card, so I cannot test. But I believe if you want it to work single monitor you should comment out your red line and also the last four sections: device1, screen1, monitor1, and Xinerama.

By the way, you do not need to reboot to test. If you are in the console and there is no X running, try startx. If X is running (but not well) you can kill it with Ctrl-Alt-Backspace.
