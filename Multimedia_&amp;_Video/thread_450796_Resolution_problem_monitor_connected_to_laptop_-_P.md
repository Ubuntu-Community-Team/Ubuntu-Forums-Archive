```

So the first thing I tried was running the new Intel driver, something I saw recommended to a few other people here on the forum:

```
sudo aptitude install xserver-xorg-video-intel
```

That uninstalled the i810 driver, and then I rebooted with my regular xorg.conf file (after changing the Driver from i810 to "intel". Both my laptop and the Samsung come alive when X starts, but the resolution of 1280x960 is chosen for some reason. At this point, I can choose from a number of resolutions in the KDE System Settings controls, including the 1680x1050 native mode. The problem is, it doesn't work. The monitor doesn't recognize the signal and the desktop runs off the screen. Pressing the button on the monitor to display the info shows that it doesn't recognize the resolution, and refresh rate is seemingly 58hz.

However, setting the resolution to 1280x1024 60hz does work, and the monitor picks this up and will display the setting when I push the menu button on the screen.

The strange thing is that the KDE settings seem to ignore the xorg.conf file when using the intel driver. I can remove certain resolutions from xorg.conf and they will still appear as options!

The next option was to re-install the i810 driver, and attempt to follow the advice about using the 915resolution hack...

```
sudo aptitude install xserver-xorg-video-i810
```

With the monitor plugged in and the xorg.conf reset back to it's original state, I rebooted. Now both screens are alive but the KDE System Settings do not allow me to set a resolution higher than 1280x800. This, I guessed, was the symptom which required the 915resolution hack to fix.

So following some advice I read about creating a ModeLine, I removed the references to refresh rates from xorg.conf and restarted X in order to look at the log files for the output of the DDC module (which runs when X starts up and queries the monitor to see what modes it supports - just including all info in case someone new like me reads this ;).
