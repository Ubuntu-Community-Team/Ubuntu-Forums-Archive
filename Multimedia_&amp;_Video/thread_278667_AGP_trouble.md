### Post by teiresias on 2006-10-17
I had to do some very roundabout debugging to get my AGP to enable.  Just to let you know what I did.

First I noted what AGP stuff was loading with:

```
lsmod | grep agp
```
(I apologize for any wrong code, I'm working off of memory here at the moment, not at my PC).

That came up showing I had agpgart, nvidia, and nvidia_agp.

agpgart just DID NOT seemt to like my setup, not sure why, nothing I did could get it to work, so I tried blacklisting it and nvidia_agp to use the nvidia agp support.  Actually, blacklisting nvidia_agp seems counter-intuitive now that I think about it, so maybe I blacklisted nvidia, I'd have to check when I get home.

However, that turned out to be picky too, because in the blacklist file I had to make sure that nvidia_agp was blacklisted in the file before agpgart was in order for it to work.

Then, as tseliot is saying, I had to change my NvAGP option in xorg.conf, I can't remember the options off the top of my head though, but I believe the "1" he's suggesting forces it to use the Nvidia agp, but it might be "2".

After that AGP worked fine and even had SBA enabled by default - I haven't messed with getting fastwrites enabled though, not really doing anything that requires either one.

Of course, if your system likes agpgart you won't need any of this. :D

Blackist the via_agp & agpgart modules by adding this to your /etc/modprobe.d/blacklist (don't worry if the file is blank or doesn't exist):

```
sudo nano -w /etc/modprobe.d/blacklist
```

Copy and paste the following lines:
```
blacklist agpgart
blacklist via_agp
```

CTRL+X to exit (save)

```
sudo nano -w /etc/X11/xorg.conf
```

Put this option in the Section Device of your /etc/X11/xorg.conf:

```
Option          "NvAGP"                 "1"
```
 
Now logout and restart of X
