### Post by zeesson on 2008-03-08
Media Monkey has done for mediaplayers what Ubuntu has done for Linux distros.  I think you would be suprised how many people, (myself included) took so long to make a permanent switch to linux or would have already done so if not for one program... MediaMonkey.  Getting to the point....  I know that there are ways; mostly involving virtual machines or Wine that will allow you to run MediaMonkey but all of these methods sacrifice functionality.  Some people will unnecessarily  immediately try to "defend" Linux (which i am certainly not attacking) by pointing out exaile and Amarok as suitable alternatives.  This is simply not the case, NO media program on any platform does ALL the things Mediamonkey can do, in the smooth seamless user-friendly way that mediamonkey can do them.  I believe that if we pool our resources and realy make an effort to convert mediamonkey into a native linux application 10's of thousands will flock to Linux.

YOUR Thoughts?

### Post by Aluminio on 2008-07-06
I certainly TRIED to use Amarok. I even used Songbird, and managed to comment to the developers what I saw missing there. No luck. In turn, I learned something awful: nobody cares to come up with some BETTER program, no, the game seems to be just to be there, repeating the same oldies over and over, with a new cool look. 
MediaMonkey really doesn't look that great. It just IS a wonderfully usefull, feature packed utility pared to none.
Why is it impossible to port? 
Answer 1: Well, that happens when you code with Visual Basic and with Jet, the DB from Access. This means that if you want the final result in Linux, you cannot "port" it, you must code it from scratch. Incidentally, given the tools they used, I seriously doubt that the developers from MediaMonkey have any low level programming skill... so forget about it.
Answer 2: I will not believe that anybody with the programing skill to do something like MediaMonkey in Linux would care to do it. Examples are at hand: at Songbird, they told me that one particular function I wanted, i.e. to include a collection of AUDIO CDs and DVDs, or CDs filled with mp3, in the library, was not usefull to them. This means that, for them, all the new music organizers should perpetually follow the lead of iTunes: Thou Shall Only Catalogue Your Hard Disk... Jesus! What is so good about iTunes? Beats me.

By the way, I am one of those who is stuck with Windows because of MediaMonkey, too. Isn't a handful of people already too much?

Hugs to all.
