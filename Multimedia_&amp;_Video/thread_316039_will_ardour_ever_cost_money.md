### Post by stbub on 2006-12-12
> **o_fortuna said:**
> They could go commercial, but they are under the GNU GPL, which means that it would still be open source (ie, you could get the source freely and redistribute it freely). They currently request donations, though.

That's not quite accurate.  The author of the code can relicense their own code.  All you need to release a GPL program under a proprietary license is to get the agreement of all contributors.  That's easier to do if the number of contributors is low, and nearly impossible for projects which has a large number of contributors.

So, it follows that if you want to make sure that it stays GPL:

1) make it worth to the developpers by making donations

2) make code contributions to an extent that ardour can't live without what you've contributed.  At that point you can prevent ardour from going proprietary by refusing that the code YOU contributed be licensed under something other than the GPL.

I can't predict the future, but I don't think ardour is going to turn proprietary.

Also, if the developpers where to go proprietary, the already released version will remain, and there could then be a continuation of the project using the last GPL release (... provided you can find people to work on the software)
