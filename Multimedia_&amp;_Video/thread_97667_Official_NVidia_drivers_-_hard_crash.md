### Post by hornett on 2005-12-01
Hello, I have recently moved from Fedora Core (and before that from just about every distro) to Breezy on my desktop machine and am now having some troubles with the official NVidia drivers.

I think it is worth mentioning that I have never had any problems with the drivers (even the same version) before I installed Breezy, so the hardware seems fine.

I have tried following the wiki to use apt to install the .debs also using the 'normal' way of installing them using the Nvidia installer.

When I run the X server, the screen goes blank, flashing text cursor stops dead and I can no longer SSH into the machine or ping it. 

Doing:
```
cat /var/log/Xorg.0.log | grep -i \(ee\)
```
reveals no errors, neither are there any in the kernel logs.

Also, the module loads fine into the kernel (aside from tainting it, of course!)

I have also tried using various options with the NvAGP option to no avail.

Please please please help! I don't really want to go back to another distro :(

PS. I am running a fresh but updated copy of Breezy, with kernel 2.6.12-10-686. 

:confused:

### Post by daller on 2005-12-02
I've had the same problem the other day...

I removed all the relevant packages (linux-restricted-modules($uname -r) and nvidia-glx)

I rebooted (to get it out of the memory!)

Reinstalled the mentioned packages, and ran "dpkg-reconfigure xserver-xorg" chose "nvidia", and now everything is running smooth again!

Good luck!
