twinview can be quite hard to configure. just took me about 20 hours to finally have all twinview problems solved for me.

[LIST=1]
[*]what are the displays you are connecting. (size, vendor, resolution)
[*]how did you connect them with your graficcard. (analog or digital)
[*]what video card do you have. (which connectors do they have?)
[/LIST]
some things that may already help you:
[LIST]
[*]DFP (digital flat panel) as Primary Display:
i recently got a new tft which i connected to my video card via DVI. i couldnt get xorg to use it as primary display until i found out that the nvidia driver has a fixed way of recognizeing displays in the following order:
"CRT,DFP,TV". (analog TFTs are treated as CRT!!).
to get around this there is the option TwinViewXineramaInfoOrder. i think is is just present in newer drivers...
you need to add the following line to your device section of the xorg.conf:
```
Option      "TwinViewXineramaInfoOrder" "DFP"
```
[*]there is a bug in kde which may cause some real aggression. if your configuring xorg correctly and both screens are used properly the screens might switch back to just one screen after login.
[http://bugs.kde.org/show_bug.cgi?id=131197](http://bugs.kde.org/show_bug.cgi?id=131197)

most issues have already been discussed. i suggest reading:
the readme in /usr/local/doc/nvidia-glx/ (or /usr/local/doc/NVIDIA_GLX-1.0 if you installed on your own)

### Post by Inigo Montoya on 2007-04-17
I guess you want your 19" TFT to be your primary screen. As i stated before the nvidia driver handles display priority in the order "CRT,DFP,TV".
So you need to set the option
```
Option      "TwinViewXineramaInfoOrder" "DFP"
```
in your Device section of the /etc/X11/xorg.conf to tell Xorg that you want your digitaly connected 19" TFT (=>DFP) as primary dispay.
The Nvidia driver that comes with k/x/ubuntu 6.10 in package  linux-restricted-modules-common/nvidia-glx doesn't support this option as far as i know so you might need to install a newer driver. you can download it at [www.nvidia.com](www.nvidia.com). in the resources i mentioned before there is a guide how to install it properly.
if you have never compiled your own kernel source before that might look a bit frightening...
