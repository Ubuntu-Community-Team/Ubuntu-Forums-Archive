### Post by gerhard on 2006-11-03
I am trying to use a realtime kernel with Dapper as defined here:
[https://help.ubuntu.com/community/HowToVanillaKernelWithRealtimePreemption](https://help.ubuntu.com/community/HowToVanillaKernelWithRealtimePreemption)
There are also some hints here (should likely be linked at the Ubuntu page)
[http://rt.wiki.kernel.org/index.php/RT_PREEMPT_HOWTO](http://rt.wiki.kernel.org/index.php/RT_PREEMPT_HOWTO)

The current kernel is 2.6.18 and Ingo Molnar realtime patch is 2.6.18-rt7 (I also include the bd-claim patch). I did not see any problems when appling the patches. However I do get problem at booting when high resolution timers are enabled. The kernel oops just after grub leaves control to linux, before "normal" printouts appear.

Update: 2.6.18.1-rt7 (minor manual merge required) does not give High Resolution timers at all, so this version is not compatible. 
2.6.18-rt6 works fine.

Some examples of what I tried:
 * 2.6.18 with default dapper config (+make menuconfig without changes) -> Works OK
 * 2.6.18-rt7 (+make menuconfig) -> Works OK
 * 2.6.18-rt7 (+make menuconfig, enabling High Resolution Timers, setting CPU type, CONFIG_HZ_1000)
 * 2.6.18-rt7 with some random settings, possibly not cleaning up properly between builds. However, I could never boot with hrtimers enabled.

As the oops occurs very early, it should be in low level code and not really be dependenton Dapper. Hard to investigate...

Any thoughts how to investigate this?
