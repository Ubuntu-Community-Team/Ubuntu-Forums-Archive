### Post by michael37 on 2007-08-08
The keyword is your need AIGLX working.  Your xorg doesn't even enable AIGLX.  Feel free to google "AIGLX" and read more about it.

Here is more info on the same subject from highly cryptic (and not quite applicable to Ubuntu gentoo howto) [HOWTO_AIGLX](http://gentoo-wiki.com/HOWTO_AIGLX)

> 
In "Module", make sure GLcore is disabled and you might want to disable vbe as it's been known to slowdown Compiz, and be sure dbe is enabled:

```

#    Load            "GLcore"
#    Load            "vbe"
    Load             "dbe"

```

In "Device" you need DRI and this other thing:

```

   Option          "DRI"     "true"
   Option          "XAANoOffscreenPixmaps" "true"

```

### Post by michael37 on 2007-08-09
A couple of suggestions.

[list]
[*]Get rid of  [highlight]Load "vbe"  [/highlight]
[*]Get rid of   [highlight]Option "UseFBDev" "true"  [/highlight]
That's a very dangerous option for some drivers.
[/list]

Then, restart the Xserver (Ctrl-Alt-Backspace) and re-login.
In terminal, run these commands and give us the output:
```

glxinfo | grep direct
cat /var/log/Xorg.0.log | grep -i aiglx

```


[list]
[*]Change [highlight]Driver "intel" [/highlight]  to [highlight]Driver "i810"[/highlight]
[/list]

Reboot computer after this change, log in and run the same commands (glxinfo and another one).
