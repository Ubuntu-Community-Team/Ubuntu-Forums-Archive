### Post by jonkulp on 2008-02-10
I haven't done this on my Linux machine yet, but using Audacity on my Mac I just run an audio cable from the headphone jack to the audio input jack and hit "record" on Audacity, making sure that Audacity's preferences are set to use line-in as source.  It's kind of crude but it works.  The trouble is "hearing" it while it's recording.  I had to wait until it was finished to play it back and see how the recording went.  You could use a splitter in the headphone jack and have one side go to the audio input and plug headphones in the other to monitor the sound?  I haven't tried this but it seems like it should work.  There might be an audio capturing application that would do this better, but with Audacity this is the only way I've found to do it.  If it's streaming audio from the web, then Firefox might have a plugin that will handle it.  I know there's a plugin (or is called an "add-on"?) for grabbing streaming video that works very well.

### Post by Dandeloreon on 2008-02-11
first off, do you have audacity installed?

Press ALT + F2 and type in the box 'audacity' ( without the ' )

this should bring up audacity, if it is installed, otherwise you will need to run the following command:

sudo apt-get install audacity


after you are sure it's installed you mainly just need to follow the tutorials section of their website:

[http://audacity.sf.net/help/tutorials](http://audacity.sf.net/help/tutorials)

or if that is no help, check the audacity team site:
[http://www.audacityteam.org/](http://www.audacityteam.org/) ( namely their forums and wiki )
