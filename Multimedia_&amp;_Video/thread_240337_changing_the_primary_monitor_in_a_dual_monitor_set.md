### Post by rumli on 2006-08-20
I solved this by using the "UseDisplayDevice" option in my Device sections in /etc/X11/xorg.conf to specify which monitor connection to associate with which device.  
See [this page]("http://download.nvidia.com/XFree86/Linux-x86_64/1.0-8762/README/appendix-d.html") for more information about the UseDisplayDevice option (and the related ConnectedMonitor option).

This solution is specific to the nVidia driver (which is to be expected I suppose since monitor detection appears to be a function of the driver).

For the benefit of others, here is my new /etc/X11/xorg.conf file which does the trick:
```

# /etc/X11/xorg.conf (xorg X Window System server configuration file)
#
# This file was generated by dexconf, the Debian X Configuration tool, using
# values from the debconf database.
#
# Edit this file with caution, and see the /etc/X11/xorg.conf manual page.
# (Type "man /etc/X11/xorg.conf" at the shell prompt.)
#
# This file is automatically updated on xserver-xorg package upgrades *only*
# if it has not been modified since the last upgrade of the xserver-xorg
# package.

# again, run the following command:
#   sudo dpkg-reconfigure -phigh xserver-xorg

Section "Files"
	FontPath	"/usr/share/X11/fonts/misc"
	FontPath	"/usr/share/X11/fonts/cyrillic"
	FontPath	"/usr/share/X11/fonts/100dpi/:unscaled"
	FontPath	"/usr/share/X11/fonts/75dpi/:unscaled"
```

The key point was to add:
```
Option "UseDisplayDevice" "DFP"
```
to the Device section associated with the LCD (plugged into the DVI slot)
and to add:
```
Option "UseDisplayDevice" "CRT"
```
to the Device section associated with the CRT (plugged into the analog VGA slot).
