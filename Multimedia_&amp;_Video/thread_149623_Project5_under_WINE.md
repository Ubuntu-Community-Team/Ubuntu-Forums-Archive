### Post by dolson on 2006-07-27
Yes, metasymbol has given wise advice. Honestly, the point is for Linux users to run Linux applications, not Windows applications with a "compatability layer."

If you spent money on a Windows application, then you have also got Windows around, so use them together. Sure, we hate Windows and all that, but it's going to give you good results, regardless of how much better you think Linux really is. Wine still, largely, sucks, especially when it comes to something so critical as timing for MIDI and other audio stuff. You are just way better off using Linux apps under Linux, and Windows apps under Windows at this point in time. And that's how it should be. I don't know about you, but I would not spend money on an application for another platform and then hope that it will run on Wine.

And if you spent money on an application hoping it would work in Linux and you DON'T have Windows, then I'm sorry, but I have to ask, what the hell were you thinking? You don't buy a PSone game and then put it into your modded Xbox hoping that psxbox will play it as well as a PSone would... You don't put an AMD CPU in an Intel motherboard... You don't put an Epson ink cartridge in an HP printer... You don't put a Betamax tape into a VCR... You don't wear white pants after labor day... You don't not put gravy on your mashed potatos... You don't eat yellow snow... And so forth. Just common sense.
