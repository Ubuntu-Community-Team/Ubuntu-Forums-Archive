Hi there,

Okay,

please type the following command into the terminal and see what it says:

```
fglrxinfo
```

It should output:
```

display: :0.0  screen: 0
OpenGL vendor string: ATI Technologies Inc.
OpenGL renderer string: RADEON 9600 Generic
OpenGL version string: 2.0.5814 (8.25.18)

```

If it does say something about mesa than you have a problem. then i need you to post your xorg.conf files. 

in the terminal copy and paste the following:

```
sudo gedit /etc/X11/xorg.conf
```

In gnome editor, create a copy of this file by saving it in the same directory and nameing it xorg.conf~

now, close gedit and type the command in again. copy and paste the ENTIRE xorg.conf file to a reply and I'll check it for errors.

as a benchmark, before you copy and paste anything, type in 

```
glxgears -printfps
```

### Post by aztektum on 2006-10-25
I'm no super genius but, I'm having my own ATI drive issues, but I noticed something in your xorg.conf

```

Section "Device"
    Identifier  "aticonfig-Device[0]"
#    Driver      "fglrx"
#    Driver        "radeon"
    Driver        "ati"

```

In all the spots where it specifies the driver to use you have the fglrx and radeon drivers commented out. Driver "ati" is the X11 standard driver.

So as it stands here, it's not even trying to use anything but the standard "non-accelerated" driver.
