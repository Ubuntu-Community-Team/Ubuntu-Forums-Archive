### Post by peterx14 on 2007-10-18
I've upgraded my embedded Intel graphics to a new [PNY GeForce 6 6200 PCI 256MB](http://www.misco.co.uk/applications/SearchTools/item-details.asp?EdpNo=246192&Tab=0&NoMapp=0) card. This is pretty much working perfectly... except for a few niggles! Here's what I did to upgrade:

Installed PCI card.
Booted Ubuntu. X fails to start properly.
sudo dpkg-reconfigure -phigh xserver-xorg
I selected the "vesa" (default) driver and NOT "nv".
sudo /etc/init.d/gdm start
but this didn't seem to start X for me (perhaps I should've used "restart"?). Anyway, I just rebooted (sudo shutdown -r now)

When I booted (if my memory serves me right) the refresh was a bit low (65Hz I think) and the restricted drivers icon was showing. I clicked on that and enabled the nvidia drivers which caused them to download and I think I rebooted again.

Okay, so aside from ending up with a "us" rather than "gb" keyboard (which I have now corrected), everything is good. My resolution is 1024x768 as before and I am getting 85Hz refresh.

But, when I start VMware Workstation and switch to full-screen, I get a *horrible* refresh. The monitor (Dell E773p CRT) says its "New mode    36K / 87Hz", but it has thin black horizontal lines between each pixel row... I think it's interlaced... or something (I don't really know -- I'm just guessing). The guest OS is Windows 2000 and it thinks it's running at 1024x768 at 85Hz.

I've Googled a bit, and VMware is supposed to only use the modes available to the host. So I looked in Gnome Prefs/Screen Resolution and it shows the refresh rate as 50Hz! And the only options I have are 50, 53, 54, 55 and 56Hz. So something isn't quite consistent!

I'd hazzard a guess that VMware is using 50Hz as the refresh for full-screen mode; the way it flickers is probably consistent with this.

So I think my question is: how do I get the Gnome refresh rate to agree with what I am apparently getting through the nVidia drivers?

### Post by peterx14 on 2007-10-22
I've fixed it now!

First I decided to upgrade to Ubuntu 7.10 (Gutsy) in the hope that that would magically solve all my problems. Predictably [in retrospect!] is didn't; and I then had a bundle of fun trying to install VMware Workstation 5.5.5 on Gutsy.... it *is* do-able!

I then found with Compiz enabled, that when switching VMware to full screen, the refresh was completely wrong (as before), but now, when I switched back, it remained completely wrong.

The problem was, the nVidia drivers report the wrong refresh rate. They do this deliberately [as a bodge?] to track twin-view screens (allegedly, this will be fixed in a future driver release). However, since I didn't need twin-view, I could disable this and have the correct refresh rate reported!

The trick is to add ```
Option "DynamicTwinView" "False"
``` to the Device section of the /etc/X11/xorg.conf file.

NOTE: If you go into the Gnome "Screens and Graphics" application and change anything, it seems to re-write the xorg.conf file killing of the newly added option and thus breaking VMware. So it is sensible to keep a backup copy of your working xorg.conf file handy in case of any problems.

The only remaining issue I have is that where I normally run my screen at 1024x768@85Hz, under VMware full screen, I get 1024x768@75Hz... and I *do* notice the slight flicker! But it is completely usable, so I'll ignore that gripe for now.
