### Post by simonn on 2007-04-25
Don't use envy!

Post your xorg.conf here and, just to get you a GUI, edit xorg.conf (sudo nano /etc/X11/xorg.conf) find the section "Device" below (it may or may not say NVIDIA and the ????? will be replaced by the name of a video card if NVIDIA is there) and replace Device "nvidia" with "Device "nv":

```

Section "Device"
    Identifier     "NVIDIA ??????"
    Driver         "nv"
EndSection

```

nv is the opensource 2D driver.

Then CTRL+ALT+F1 and log into the terminal.

```

$ sudo /etc/init.d/gdm restart

```

If this does not start, post your xorg.conf here again. Do not change things and/or reboot randomly, this never works. Otherwise you should get a login screen.

If you currently have the nvidia drivers installed via synaptic, remove them - all of them.

Assuming you are not using 64bit ubuntu, download (as of today) [http://us.download.nvidia.com/XFree86/Linux-x86/1.0-7185/NVIDIA-Linux-x86-1.0-7185-pkg1.run](http://us.download.nvidia.com/XFree86/Linux-x86/1.0-7185/NVIDIA-Linux-x86-1.0-7185-pkg1.run)

Then CTRL+ALT+F1 and log into the terminal.

```

$ sudo /etc/init.d/gdm stop
$ sudo sh /path/to/NVIDIA-Linux-x86-1.0-7185-pkg1.run

```

Obviously, /path/to/ should be replaced with the path to NVIDIA-Linux-x86-1.0-7185-pkg1.run.

The questions should be pretty straight forward - you can just answer yes to most of them. If you are not sure post here. DO NOT LET THE INSTALLER MODIFY YOUR xorg.conf - IIRC it asks you if it should modify the configuration, it is easier to do it manually.

Once the drivers are installed:

Edit xorg.conf once more and change it back to 

```

Section "Device"
    Identifier     "NVIDIA"
    Driver         "nvidia"
EndSection

```

then:

```

$ sudo /etc/init.d/gdm start

```

If this does not start, post your xorg.conf here again. Do not change things and/or reboot randomly, this never works.

Once we have it running we can then try optimizing it.
