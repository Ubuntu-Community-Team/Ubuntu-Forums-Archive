### Post by dixonstalbert on 2006-09-24
I am wondering if there is a conflict between the device sections that have the same BusID sections but different identifiers...


maybe try commenting out the 2nd one like this 

```
#Section "Device"
	#Identifier  "aticonfig-Device[1]"
	#Driver      "fglrx"
	#Option	    "VideoOverlay" "on"
	#Option	    "OpenGLOverlay" "off"
	#BusID       "PCI:5:0:0"
#EndSection
```

also  add     Option "HSync2"  "75" line to aticonfig-Device[0] section

Prior to hooping myself by trying the newest fglrx driver, I was able to get Horizontal Desktop running on my ATI by only using a single "Device" section, so you should by okay commenting out the 2nd one like I've shown

good luck!

### Post by dhgilmer on 2006-09-24
Thanks for the advice, it didn't quite work but it put me on the right track. All it needed was to change:

```

Section "Device"
	Identifier  "aticonfig-Device[1]"
	Driver      "fglrx"
	Option	    "VideoOverlay" "on"
	Option	    "OpenGLOverlay" "off"
	BusID       "PCI:5:0:0"
EndSection
```

to 
```
Section "Device"
	Identifier  "aticonfig-Device[1]"
	Driver      "fglrx"
	Option	    "VideoOverlay" "on"
	Option	    "OpenGLOverlay" "off"
	Option	    "Centermode" "off"
	Option	    "VRefresh2" "55-75"
	Option	    "Hsync2"	"30-83"
	Option	    "Mode2" "1280x1024"
	BusID       "PCI:5:0:0"
EndSection
```


For some reason even thought I had the refresh rate set for Vrefresth2 in the Device[0] area it only picked it up when it was in the Device[1] area. 

Woot for dual monitors, and changing my default boot target from Winxp to
/boot/vmlinuz-2.6.15-27-386 root=/dev/sda4 ro quiet splash :)
