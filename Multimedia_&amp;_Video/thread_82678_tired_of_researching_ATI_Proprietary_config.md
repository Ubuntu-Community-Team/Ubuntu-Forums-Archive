### Post by daller on 2005-10-27
Well, buy an nVidia-card! :) - Simply the best choice in Linux!!!

But back to the ATI...

sudo apt-get install xorg-driver-fglrx

sudo dpkg-reconfigure xserver-xorg (write this down, so that you can reverse it, when X wont start :))

When you come to the driver section, find "fglrx", and click enter 

Answer all the other questions (most of them just click enter)

Restart X

...is it working?

Actually ubuntu was the first distro I've tried succesfully with my ATI 9600XT card.

The easy solution (using the ATI fglrx package from ubuntu)
System -> Administration -> Synaptic Package Manager
Install:
fglrx-control
xorg-driver-fglrx
xorg-driver-fglrx-dev

Here's my xorg.conf - The keyboard is setup for danish, you can replace this part with whatever your own xorg.conf says. Otherwise it's pretty much the basic with ATI fglrx. The driver from ubuntu is an old version but it works fine. Run fglrxinfo to check the output, if it says something about mesa, then you're not using ATI fglrx driver.

### Post by Robocoastie on 2005-12-11
Once I learned mlonkers how to I didn't find setting up an ATI card any harder than an NVidia one is. The only difference is that Nvidia has precompiled mini kernels that it can d/load from its site when it detects a lack of kernel sources on your machine.

That step I'm finding to be a very commone error people are having - they forget to enable the universe and multiverse repositories which is needed for one of the programs you have to d/load via apt-get. Without that the ati kernel compile won't work and you'll keep getting the "mesa" drivers message when you do fglrxinfo.
