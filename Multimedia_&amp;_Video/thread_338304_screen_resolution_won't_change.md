### Post by floke on 2007-01-14
No idea - have had strange things like this happen to me before.

Firstly, back up your xorg.conf file (run nautilus as 'gksudo nautilus', enable view hidden files, move up to file /etc/X11/xorg.conf, and copy the xorg.conf file to the same directory - i.e. make a back-up in case anything goes wrong).

Then you can try to reconfigure the file - sinec you have switched monitors it might be having trouble detecting the right settings for the new one. So open a terminal and type

sudo dpkg-reconfigure -phigh xserver-xorg

and follow the on-screen instructions to select resolution etc. - use space bar to select the resolutions you have, ubuntu will try to use the highest one

If this doesn't work then you might have to manually edit your xorg.conf file

to do this type in a terminal

sudo nano /etc/X11/xorg.conf

you might then need to enter the details of your monitor etc. 
It might help with this if, after you have tried the dpkg-reconfigure option - you post the output of your xorg.conf file here so that we can see what's going on

hope this helps
