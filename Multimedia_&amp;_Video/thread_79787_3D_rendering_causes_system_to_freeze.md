System Specs:

ATI Radeon 9800 Pro - 128 Mb (AGP)

Asus K8N-E Deluxe
AMD64 3000+
1 Gb Corsair RAM

Ubuntu 5.10 (32-bit)


I originally had the ATI drivers installed from the repository. Now, I had everything working well in Hoary, so I can only guess that perhaps I am misconfigured in some way. I configured using fglrxconfig, accepting the defaults. I had originally tried using: dpkg-reconfigure xserver-xorg but the generated xorg.conf didn't work. Any suggestions are more than welcome...

### Post by WilliamAnderson on 2006-01-31
Hi Newclique,

The package Ubuntu desktop is a meta package that contains Xorg, Gnome, and all of the other components needed to run the graphical user interface in Ubuntu. This includes the ati and radeon open source drivers. When you remove these drivers, you break the meta package, which requires that the meta package be removed to keep the package management system in sync. Removing Ubuntu desktop is safe and will not result in any harm.

When installing the fglrc package, you will also need to install the restricted modules package that matches your kernel. To determine which one to install, run the uname -a  command which should return something like this:

```

$ uname -a
Linux odin 2.6.12-10-k7 #1 Mon Jan 16 19:20:45 UTC 2006 i686 GNU/Linux
$

```

The string you are interested in is the version number, here “2.6.12-10-k7“ which includes the kernel version number, the package number, and the processor that the package is compiled for. For my system, the correct module would be “linux-restricted-modules-2.6.12-10-k7”, yours may be different.  Find and mark this restricted modules package  and the xorg-driver-fglrx package  (and the fglx-control package to include the ATI control applet) for installation, install, and reboot. 

After this, you should find that you are running the 3D accelerated driver. confirm this by running the command “glxinfo” which should include lines with “direct rendering: Yes”, “OpenGL vendor string: ATI Technologies Inc.”, and “OpenGL renderer string: RADEON 9550 Generic” (this last should have your installed card model number in it). fgl_glxgears should run and display the frame count in the terminal window as the cube with gears on the surface spins.

More detailed instructions can be found on the ATI binary driver page in the Ubuntu Wiki at wiki.ubuntu.com/BinaryDriverHowto/ATI . If you are interested in installing the ATI binary driver 
supplied by ATI, see the start of this thread: ubuntuforums.org/showthread.php?t=75378 .

Let us know how it works,

William
