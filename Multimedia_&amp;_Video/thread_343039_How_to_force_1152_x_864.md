### Post by Niko38752 on 2007-01-21
You could check if xrandr has the resolution. Just type xrandr in the terminal. It should print a list of available resolutions, such as..
```
 SZ:    Pixels          Physical       Refresh
*0   1024 x 768    ( 347mm x 260mm )  *75  
 1    800 x 600    ( 347mm x 260mm )   75  
 2    640 x 480    ( 347mm x 260mm )   75  
Current rotation - normal
Current reflection - none
Rotations possible - normal 
Reflections possible - none
```
If you see the resolution there, just
```
xrandr -s %
```
..where you replace % with the number you see before the resolution. However, if you don't see it in there, you could always reconfigure xorg with
```
sudo dpkg-reconfigure xserver-xorg
```
and when it asks for the resultions, check the resolution from the list with spacebar:
[[IMG]http://img259.imageshack.us/img259/9571/resolist7op.th.png[/IMG]](http://img259.imageshack.us/my.php?image=resolist7op.png)

After you're done reconfiguring, restart x with alt+ctrl+backspace and then check if you can select the resolution anywhere.

### Post by phansiizwe on 2007-01-28
I have forced the resolution by adding a ModeLine to the monitor section in /etc/X11/xorg.conf

My section looks like:

```
Section "Monitor"
    Identifier     "A705MT"
    DisplaySize     324    242        #For solving font issues: [mm]
    HorizSync       30.0 - 86.0
    VertRefresh     50.0 - 180.0
    ModeLine       "1152x864@85" 129.4 1152 1216 1376 1672 864 864 867 910
    Option         "DPMS"
EndSection
```

Now this data is specific for my monitor, you will probably need another line. One tool to calculate it is found under [http://www.bohne-lang.de/spec/linux/modeline/](http://www.bohne-lang.de/spec/linux/modeline/), but there are some others (with more options) around.

Then under: Section "Screen"
you will need to specifiy the ModeLine using:

```
SubSection     "Display"
        Depth       16
        Modes      "1152x864@85" "1152x864" "1024x768"
EndSubSection
SubSection     "Display"
        Depth       24
        Modes      "1152x864@85" "1152x864" "1024x768"
EndSubSection
```

Hope this helps you a little further.

### Post by aeiah on 2007-02-01
using the modeline calculator your horizontal refresh rate is 92.9Hz. this gives a horizontal refresh of exactly 85KHz at your resolution but the calculator isnt specific to your monitor since the electron gun spends approx 5% of its time moving back up to the top after each cycle.

so try using 92.9 and 93 in the calculator and see if it does anything. if you poke around google for long enough you may find either a windows program that can detect your current vertical refresh rate (since you cant achieve it in linux yet) or full specs for your monitor that has the refresh rates supported for each possible resolution. maybe something like pstrip.exe (power strip) for windows may help, if it's still available. poke around its faq to see if it can detect what you need.
