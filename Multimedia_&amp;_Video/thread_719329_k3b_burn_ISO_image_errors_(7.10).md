### Post by candtalan on 2008-03-09
I have recently started having a lot of failed ISO burns using k3b (gutsy 7.10, Kubuntu) in my desktop machine and also a laptop machine. 

I initially suspected the media, and burned a slower speed. Then I suspected my (desktop) burner drive so I bought a new one, and I still got errors when burning CD ISO images (use md5checks). I burn a lot of ISO images because I give out Live CDs at local events, so I know what I usually expect from burning.

In desperation, I used the gnome desktop with the existing installation (ubuntu-desktop is already installed for occasional use) and voila! All my burns did not have any errors! I was using the gnome native burn app, not k3b here.

I am surprised about this. It could be the result of coincidences, still some effect with media maybe, and I will continue to be vigilant, but I thought I should post here to see if anything else is being noticed?
