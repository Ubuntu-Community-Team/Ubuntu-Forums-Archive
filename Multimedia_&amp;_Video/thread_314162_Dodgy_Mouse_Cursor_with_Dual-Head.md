I may have found a fix for this (just in case anyone's found this thread with the same problem). Appending the following to your xorg.conf should sort it:

```
Section "Extensions"
    Option "Composite" "Disable"
EndSection
```

As far as I am aware, composite mode is enabled by default and that's what's causing the problem. I found this mentioned on another website, and will have to wait until I get home to try it. Will update with what happens. :P

### Post by EddM on 2006-12-08
Just to let everyone know, I fixed it. Nice thread btw. :P

1) Completely reset xorg.conf
```
$ sudo dpkg-reconfigure xserver-xorg
```

2) Download proprietary ATI Linux drivers ([Here]("http://ati.de/support/drivers/linux/linux-radeon.html"))

3) Follow the ATI Installer Instructions ([Here]("https://a248.e.akamai.net/f/674/9206/0/www2.ati.com/drivers/linux/linux_8.31.5-inst.html"))

4) Get the hell out of X (this is probably not the best way to do it, but it's how I did -- thought it best to completely take the runlevel down if I was going to do this cleanly. Someone let me know if this was a bad idea :P)

```
$ sudo init 1
```

5) Exemacute these here commandingz:
```
$ aticonfig --initial=dual-head --dtop=horizontal --screen-layout=right -v
$ /etc/init.d/gdm start
```

Worked pefectly for me, and the cursor's all fine and dandy. I guess it helps to just wipe the slate clean and start again more often than I thought.

### Post by dukat on 2006-12-14
Thanks, worked perfectly for me. Although on my Edgy 6.10 setup fglrx was already included. 
Note, that at least from fglrx 8.28.08 (the one from edgy) the *aticonfig --sreen-layout* flag isn't supported anymore. 
Use *--dtop=horizontal,reverse* if you need the second monitor to the left of your primary one.

Having read aticonfig's help carefully, --sreen-layout is still quoted in version 8.32.5 and has successfully set up dual-head support on my laptop. To my humble knowledge, version 8.32.5 is currently not in the Ubuntu repositories which show v8.25.18 for my system. (I doubt that the use of --screen-layout is responsible for the dodgy mouse.)

NOTE:
ATI points out that updating its proprietary driver successfully is heavily dependent on the mode of initial installation. 
1) In case you are planning to switch from current Ubuntu-supported versions TO a more up-to-date version from the official ATI homepage
2) OR intend to update your driver from ATI's homepage
DO READ the installation instructions carefully! 

In short, uninstalling the previous version of fglrx is advised. Therefore, launch a Terminal and issue
```
cd /usr/share/ati
sudo sh ./fglrx-uninstall.sh
```

I can indeed confirm that using the latest proprietary driver from ATI's homepage works out fine on Dapper 6.06 and my radeon X300. Since I have come across the dodgy mouse cursor among other problems (suspend to RAM etc.) in previous versions of Ubuntu, Xorg and fglrx - I have been reluctant to move to ATI's proprietary driver (fglrx).
