### Post by kc4mts on 2007-12-02
Unfortunately the Veo Advanced Connect ( model 642v ) ( ID 0x0545 : 0810 ) has some proprietary firmware inside on the chip , and so far there has been limited success in making a proper driver for it.
   You could wait and see if some one does get a driver out for it but I have been looking for 2 years and not had much luck. I would recommend trying to get another web cam that is compatable with linux, or that already has drivers available.
   If you are going to get another web cam ask what others are using before spending any money .... I am a perfect example of what not to do ... there are 3 web cams sitting on my desk and none of them are responding under linux although there are some drivers that I am trying to experiment with to see if I can get anything to work.
   I can give you a short list of what is not working for me yet so you will have a head start in your search.
   Aiptek pencam trio (worked under KDE desktop and Mandriva linux but I havent found the correct driver for it yet under ubuntu)
Kensington Video Cam ( still experimenting with this one... there are drivers for it)
Veo Advanced Connect (havent found a driver yet and the  MFGR ID is not recognised  for some reason)
    I hope this helps in some small way

Alan
