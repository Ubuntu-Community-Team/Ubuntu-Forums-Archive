### Post by keforex on 2007-10-28
I found the solution for nvidea card users:

ASSUMING you already have the nvidea drivers installed:

1) Create a desktop launcher with the following command: gksudo nvidia-settings

2) After creating this launcher (called shortcut in Windows) double click on it

3) Type password if asked

4) On the left pane, select X Server Display Configuration

5) Click on the square that represents the screen you want to be to the left (main) and click on "configure" and then select twinview.

6) Set Position to "absolute" on this first screen.

7) At the bottom, click on "Save to X Configuration File"

8) Reload xorg.conf by pressing ctrl-alt-backspace

After that you can start playing with the configurations on the nvidia X Server Settings (through the launcher you created) and see what is best for you.

If you don see certain options mentioned above, try clicking on the button "advanced" on the nvidea settings application.

### Post by pennygov on 2007-10-29
For intel users and I think ATI read [http://www.thinkwiki.org/wiki/Xorg_RandR_1.2](http://www.thinkwiki.org/wiki/Xorg_RandR_1.2)

I have a Toshiba notebook using an Intel Mobile 915GM graphic controller on Kubuntu, Gutsy. The xrandr info enabled me to have a big desk across an external monitor and the notebook's screen (Before Gutsy and xrandr v. 1.2 I've not been able to get an extended desktop across screens no matter how I edited my xorg.conf file). I only changed my xorg.conf file to define the size of my desktop. That config file works when I'm not plugged into an external monitor so I don't have to restore any backup files when I'm unplugged. I do have to run a script to extend the desktop though (which is fine) or run an xrandr command. The script has the xrandr command that works for the settings that I like. You can experiment with the commands to see what resolutions work best for you.
For example: xrandr --output VGA --left-of LVDS --mode 1024x768 is what I have - the resolution on my notebook stays the same with this because it is fine. I could change it though with xrandr if I wanted to. It is all on the fly, nothing "permanent."

My xorg.conf file has an added line in the SubSection "Display" that defines the desktop size:
<snip>
Section "Device"
	Identifier	"Intel Corporation Mobile 915GM/GMS/910GML Express Graphics Controller"
	Driver		"intel"
	BusID		"PCI:0:2:0"
	[B]#Add this if your laptop does not have a TV connector or Docking Station
	Option		"monitor-TV"	"TV"[/B]
EndSection

Section "Monitor"
	Identifier	"Generic Monitor"
	Option		"DPMS"
EndSection
	[B]#Add this if your laptop does not have a TV connector or Docking Station
Section "Monitor"
	Identifier	"TV"
	Option		"Ignore"	"True"[/B]
EndSection

Section "Screen"
	Identifier	"Default Screen"
	Device		"Intel Corporation Mobile 915GM/GMS/910GML Express Graphics Controller"
	Monitor		"Generic Monitor"
	DefaultDepth	24
	SubSection "Display"
		Modes		"1440x900"
	[B][COLOR="Red"]#Add a virtual line to provide for the largest screens you will hotplug
	Virtual		2720	1860[/COLOR][/B]
	EndSubSection
EndSection

Section "ServerLayout"
	Identifier	"Default Layout"
	Screen		"Default Screen"
	InputDevice	"Generic Keyboard"
	InputDevice	"Configured Mouse"
***************

The bold additions about the TV make it so the monitor doesn't click on and off a few times when adjusting (I guess it is looking for a TV)

The bold red line is the important one. Without it the default "desk size" is too small to accomodate the two screens.

Hope this helps. It was the answer for me. Play with the xrandr command and see what you can do. I don't know how this works with other eye candy like compiz or enlightenment (don't think I need it now that I've got a big desk!)  :guitar:

In KDE I can have the same wallpaper on each screen, stretch it across screens, center it, or have different pics on each!

### Post by keforex on 2007-11-02
I found a very easy and definite way to activate 2 screens under Ubuntu, Kubuntu, Ubuntu+KDE, etc...

Just Download the nVidia installer called ENVY from Alberto Milone (google it). Go half way down his web page and click on "envy_0.9.8-Oubuntu10_all.deb" - you can choose the "run" option instead of saving it to your disk.

After the install, it will be on the menu "Sytem Tools->Envy".

Run it, and uninstall the messed-up drivers you might have (nVidia or ATI) and then reinstall them (all through Envy options).

Reboot and go to the menu "System Tools->nVidia Settings" and play with the options there. You'll find out how to activate 2 screens, it's pretty easy.

Inside the nVidia Settings tool, don't forget to click on "save to X configuration file" before exiting. This button is available at the bottom right when you select "X Server Display Configuration"  on the left pane.

If this saving process can't happen because you don't have permissions, logout of ubuntu/kubuntu and restart the system. At the boot menu, select recovery mode.

When at the prompt (you're now as root) type "startx" and go configure your nVidia settings again. You should be able to save the settings through the "Save to X configuration file" button without any problems. After that, logout. When back to the prompt either type "exit" or restart your machine. You're done, and you should see the nVidia splash screen.

### Post by 11touche on 2007-11-26
Well I finally managed to make it work on a Ati Radeon 9600 with the Ati driver (Open Source)
At first, I added a line to my /etc/X11/xorg.conf under "SubSection "Display" :

Virtual  2048x768

Then I restarted the X server ( /etc/init.d/gdm restart )
Now, this was the part which made me go crazy.
I didn't know that I had to turn off both monitors with xrandr, and then turn them back on.
I used my laptop under a ssh session, so I wasn't doing it blindly.
Here's the two commands I used, which worked for me
```

xrandr --output DVI-0 --off --output VGA-0 --off
xrandr --output DVI-0 --mode 1024x768 --output VGA-0 --mode 1024x768 --right-of DVI-0

```

This was successful for me, on Ati hardware, but I don't know if it'll work for you.
Good luck all !!!
