### Post by majoridiot on 2006-12-08
it looks like you need to define both monitors, set up some metamodes
and tell x how your monitors are arranged.

start by making a backup of your xorg.conf.  then edit xorg.conf...

rename your existing monitor by changing the identifier from:

```
Section "Monitor"
Identifier "Generic Monitor"
Option "DPMS"
HorizSync 28-51
VertRefresh 43-60
EndSection
```

to:

```
Section "Monitor"
Identifier "Generic Monitor 1"
Option "DPMS"
HorizSync 28-51
VertRefresh 43-60
EndSection
```

then define the second monitor by adding:

```
Section "Monitor"
Identifier "Generic Monitor 2"
Option "DPMS"
HorizSync 28-51
VertRefresh 43-60
EndSection
```

next, change both the "Screen" Identifier and Monitor from:

```
Section "Screen"
Identifier "Default Screen"
Device "nVidia 6200"
Monitor "Generic Monitor"
```

to:

```
Section "Screen"
Identifier "Screen 1"
Device "nVidia 6200"
Monitor "Generic Monitor 1"
```

and add the following line between DefaultDepth 24 and "SubSection "Display":

```
Option         "Metamodes" "1024x768,1024x768; 800x600,800x600"
```

then create a "Screen" section for the second monitor by duplicating the
existing one (cut and paste that entire Screen section), changing the 
Identifier to "Screen 2", the monitor to "Generic Monitor 2", and deleting
the "option metamodes" line (so as not to confuse things), which results 
in:

```
Section "Screen"
Identifier "Screen 2"
Device "nVidia 6200"
Monitor "Generic Monitor 2"
DefaultDepth 24
SubSection "Display"
Depth 1
Modes "1024x768" "800x600" "640x480"
EndSubSection
SubSection "Display"
Depth 4
Modes "1024x768" "800x600" "640x480"
EndSubSection
SubSection "Display"
Depth 8
Modes "1024x768" "800x600" "640x480"
EndSubSection
SubSection "Display"
Depth 15
Modes "1024x768" "800x600" "640x480"
EndSubSection
SubSection "Display"
Depth 16
Modes "1024x768" "800x600" "640x480"
EndSubSection
SubSection "Display"
Depth 24
Modes "1024x768" "800x600" "640x480"
EndSubSection
EndSection
```

the last thing you need to do to get it initially running is to explicitly
define how you want the screens laid out.  change your server layout
section to read:

```
Section "ServerLayout"
Identifier "Default Layout"
Screen "Screen 1" 0 0
Screen "Screen 2" LeftOf "Screen 1"
InputDevice "Generic Keyboard"
InputDevice "Configured Mouse"
InputDevice "stylus" "SendCoreEvents"
InputDevice "cursor" "SendCoreEvents"
InputDevice "eraser" "SendCoreEvents"
EndSection

```

make sure the 17" is on the main video output and the 15" is on the second 
output and it should hopefully work ok.

you will probably need to adjust the frequencies for the second monitor
definitions and the metamodes which define the virtual desktop twinview
uses.  i would not remommend mixing resolutions (e.g. 1024X768 on the 
17 and 800x600 on the 15, etc.) as i have read that people have problems
doing that sometimes... but feel free to experiment with it.

good luck!

### Post by majoridiot on 2006-12-09
in my experience, the VGA out of nvidia cards will be your main monitor
and the DVI will be the second monitor.  so, if you want the 17" to be
the main desktop anchor, it should be on the VGA out.  switch the cables
and the orientation should be right.

the resolutions right now are set fot 1024x786 on each monitor with a 
virtual desktop of 2048X768 (the sun of the horizontal x vert. resolutions
of both monitors.  if you want additional resolutions, you need to add them, 
which requires adding metamodes and modes for BOTH monitors.

e.g...

if you want 1280x1024 on both monitors, you need to change the metamode in
Section "Screen", Identifier "Screen 1" to read:

```
Option "Metamodes" "1280x1024,1024x768; 1024x768,1024x768; 800x600,800x600"
```

you should check to be sure the 15" can handle 1280x1024 before trying
this.  if it can't, i would recommend running both monitors at the
higest resolution the 15" can handle... because i have read many, many
complaints of getting dead spots, etc. worked out when mixing resolutions
on the same desktop.  if you want to try running mixed resolutions, such
as 1280x1024 on the 17" and 1024x768 on the 15", then your metamode would
read:

```
Option "Metamodes" "1280x1024,1024x768; 1024x768,1024x768; 800x600,800x600"
```

or whatever mix you want.  note that the format is "<main monitor res>, <second monitor res>" semicolon delimited per desktop setup.

you also need to add the new resolution(s) to your screen definitions for
both monitors.  change each (6 total) mode line in the "Screen 1" 
definition from:

```
Modes "1024x768" "800x600" "640x480"
```

to:

```
Modes "1280x1024" "1024x768" "800x600" "640x480"
```

if your 15" will handle 1280x1024, then make the same six changes to the
mode definitions of "Screen 2" as well.  if you intend to run mixed modes
with 1024x768 being the highest res. on the 15", then no changes need to
be made.
