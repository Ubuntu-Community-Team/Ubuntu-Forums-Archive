### Post by Sam Lars on 2007-07-01

I threw a png in there and it showed up under predefined.  With caps, I think it puts it at the top of the list, too.

### Post by fuzzy_umbra on 2007-12-31
This seems to have changed along the way.  I'm now using Ubuntu 7.10 and F-Spot version 0.4.0.  The location listed above contains the icons that used to be available.  e.g. I used emblem-camera.png for all keywords, and that is in the directory mentioned, but that image is no longer listed when editing keywords.  The result is that new keywords will have to have a different image.   It would be great to have the option of *no* image, since text looks cleaner and is good enough for me.  e.g. I don't need a picture of my family to remember what's attached to the "family" keyword.  A workaround is to just create a small, blank image, add it to every tag, and choose it as the image for each tag to use, but that is a little clunky.  Thanks.
