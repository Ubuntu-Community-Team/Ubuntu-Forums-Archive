I then followed both methods in the Ubuntu wikis, but neither results in a picture on the TV. The monitor keeps working fine though. In XP, the TV-out works, so the setup should be alright.

### Post by WakkiTabakki on 2007-01-22
This one works for me on a laptop with an nvidia card. If you're on an external monitor, you may need to replace 'dft' with 'crt' and change to your own monitor resolutions...
Also, check the identifiers for your input devices in the SeverLayout section

Don't forget to backup you conf-file before trying :p

Good luck
/N

Note! This is only the parts of the xorg.conf that appy to the monitor.
```

Section "Device"
	Identifier     "NVIDIA Corporation NVIDIA Default Card"
    	Driver         "nvidia"
	Option		"AddARGBGLXVisuals"	"True"	
	Option		"TripleBuffer"		"True"

### Twinview config
	Option		"ConnectedMonitor"	"dfp, tv"
	Option		"TwinViewOrientation"	"DFP-0 LeftOf TV-0"
	Option		"TwinView"		"1"
	Option		"NoPowerConnectorCheck"
	Option 		"Metamodes" 		"DFP-0: 1280x800, TV-0: 640x480"
	Option		"SecondMonitorHorizSync"	"30-50"
	Option		"SecondMonitorVertRefresh"	"60"
	Option		"TVStandard"		"PAL-B"
EndSection

Section "Screen"
	Identifier	"Default Screen"
	Device		"NVIDIA Corporation NVIDIA Default Card"
	Monitor		"Generic Monitor"
	DefaultDepth	24

	SubSection "Display"
		Viewport		0 0
		Depth		24
		Modes		"1280x800" "1024x768" "800x600" 
	EndSubSection
EndSection

Section "ServerLayout"
	Identifier	"Default Layout"

	Screen 0 "Default Screen" 0 0
	Option		"Xinerama"	"Off"

    	InputDevice    "Generic Keyboard"
	InputDevice    "Configured Mouse"
        InputDevice     "Synaptics Touchpad"
EndSection
```
