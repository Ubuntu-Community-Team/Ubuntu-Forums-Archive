### Post by Napoleon XIV on 2007-07-23
I just built a new machine for my home theater, and I'm using the room's projector as my primary (and only) monitor.  I have an Asus Nvidia 7600GS PCIEx16 video card with HD component out, which is plugged into the A/V hub.

Now, this set up works decently (it's a tad blurry and maxes out at 50hz and 1024x768, but usable mostly) while I'm using the vesa driver, but the second I enable the Nvidia driver the screen basically turns everything shades of blue.  The machine is still usable, but blue - very, very blue.  Also NvTVOut doesn't open at all.  (Note:  this blueness only shows up AFTER the Ubuntu loading screen, which is in perfect color.)  The latest Nvidia *nix drivers from their site don't help either.

Another probably related problem:  My projector's specs say that its horizontal refresh rate range is 30 to 70, and its vertical refresh is 40 to 87, if memory servers.  The stated max resolution is 1280 x 720 (which seems to me like an odd number, but whatever), and it's capable of 1080p (at present the machine is, according to the projector, only outputting at 480i).  I can't get X to go beyond 50hz and 1024x768, with 75 x 75 dot density (or something to that effect - in any case, the projector's specs say it's capable of 150x150).  I've already gone through xorg.conf adding the new resolutions manually (and killing X a few dozen times ;) ), and the reconfigure script a few times, but the entries never show up under Preferences --> Screen Resolution.

Anyone have any ideas?  I've been trying to figure this oddness out for the past two days.  Hopefully the above makes some sense. :)
