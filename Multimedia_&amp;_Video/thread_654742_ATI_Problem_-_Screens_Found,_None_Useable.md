I had the drivers from the repositories. Following your suggestion, I've actually downloaded the latest drivers (version 8.443.1) from the ATI site and followed the instructions [here]("http://wiki.cchtml.com/index.php/Ubuntu_Installation_Guide").

This got me very close to working. The output of fglrxinfo told me I was still using Mesa so I altered the listed driver via Kubuntu's System Settings->Monitor and Display->Hardware option to be my graphics card. Unfortunately I also commented out the vesa device entry in my [new, not the one above] xorg.conf file at the same time, so after restarting xorg, I don't know which of these fixed it. One of them did though as fglrxinfo now reports I'm using the correct driver and the restricted driver option under Advanced in System Settings shows that I have the proprietary driver enabled.

Not quite there yet, as when I try to run a game with 3D under Wine, the application consistently crashes. I don't know what causes that, but I do *appear* to be using the proprietary drivers, now.
