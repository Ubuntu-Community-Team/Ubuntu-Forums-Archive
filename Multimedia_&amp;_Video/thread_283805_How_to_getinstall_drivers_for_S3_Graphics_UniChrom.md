### Post by cagouilles on 2006-10-24
Hello,

(I'm rather new to Linux and Ubuntu so please be indulgent - however I turn around this problem for some days so I think a new thread will *at least* be useful for beginners !)

I've installed Ubuntu Dapper Drake on a new (low cost) PC, which has a VIA motherboard PM8M3-V. This board uses a VIA chipset P4M800CE, whose graphics ID is 3344 (more info : [http://www.viaarena.com/default.aspx?PageID=5&ArticleID=68&P=5)](http://www.viaarena.com/default.aspx?PageID=5&ArticleID=68&P=5)). A lot of people have a similar problem, indicated by the following line in the response to "lspci" command :
```
0000:01:00.0 VGA compatible controller: VIA Technologies, Inc.: Unknown device 3344 (rev 01)
```
An often proposed solution is to change the lines in /etc/X11/xorg.conf (either by editing or using "dpkg-reconfigure xserver-org") from
```
Section "Device"
	Identifier	"Carte vidéo générique"
	Driver		"vesa"
	BusID		"PCI:1:0:0"
EndSection
```
to
```
Section "Device"
	Identifier	"VIA Unichrome Pro"
	Driver		"via"
	BusID		"PCI:1:0:0"
        *Option          "EnableAGPDMA"*
        *Option          "DisableIRQ"*
EndSection
```

But in most cases the X server will not start after a reboot. A manual restore of the original driver "vesa", with vi from the command line, is then mandatory to rescue the sytem.
There is a driver proposed by VIA for Xfree86 but it is not clear for me if it can work with Xorg or even if it is the same.
The subject is discussed in several previous threads (search for 3344) but there is no clear statement if it is possible to solve it. One suggestion was to check if Xorg was on version 10.4, so I did check, the version is correct but the problem remains unresolved.

Now the questions :
- did I forgot something in the description of the problem ?
- is there a statement that the problem can or can not be solved ?
- is the Xorg driver "via" usable with this chip ?
- is the xfree86 driver, if different, usable - but in this case is it needed to install xfree86 and how ; or how to install a xfree86 driver on Xorg ?
- a detail : dpkg has a problem with the accent "é" of the french description of the card, it's impossible to change the name - but I don't think it is relevant here.

Thanks for help,

### Post by Jasper Houtman on 2006-11-02
Just editing the xorg.conf won't work.
Open your terminal and type:

```
 sudo dpkg-reconfigure xserver-xorg 
```

Select the unichrome driver in the list, and set it up (resolutions, etc)
If it's not in the list than download and compile it from sourceforge (unichrome project).

Then restart GDM:

```
 sudo /etc/init.d/gdm restart 
```

Should work fine after that.
