### Post by flacharite on 2007-12-05
Hi, I had some trouble getting my new monitor to give me a 1680x1050 resolution and I want to post the solution for my particular software/hardware as I did not find any post that fully explained what to do.

Here is my environment :

Ubuntu Dapper
Monitor  : Dell E228WFP LCD
Display controller : 915G (lspci : 0000:00:02.1 Display controller: Intel Corporation 82915G Express Chipset Family Graphics Controller (rev 04))

The solution for me was the addition of Modeline to the xorg file and to tweak 915resotion.

    * Reconfigure xservser-xorg to add the 1680 x 1050 resolution and set the driver to i810.

$ sudo dpkg-reconfigure xserver-xorg

Keeps defaults exept for the driver and the resoution

Set driver i810
Choose at least 1680*1050 from the list.

    * Add Modeline in the file /etc/X11/xorg.conf at the Monitor section :

Section "Monitor"
        Identifier      "DELL E228WFP"
        Option          "DPMS"                
        HorizSync       30-83                 
        VertRefresh     56-75                  
        Modeline "1680x1050" 146.2  1680 1784 1960 2240  1050 1053 1059 1089 +hsync -vsync # Added manually
EndSection

Numbers in the modeline are from the /var/log/Xorg.0.log :

(II) I810(0): Supported additional Video Mode:
(II) I810(0): clock: 146.2 MHz   Image Size:  473 x 296 mm
(II) I810(0): h_active: 1680  h_sync: 1784  h_sync_end 1960 h_blank_end 2240 h_border: 0
(II) I810(0): v_active: 1050  v_sync: 1053  v_sync_end 1059 v_blanking: 1089 v_border: 0

    * Install 915resolution

$ sudo apt-get install 915resolution

    * Change /etc/default/915resolution. Set the following values :

MODE=5a
XRESO=1680
YRESO=1050
HTOTAL=2240
VTOTAL=1089

Explanations :
This is the step that was not well documented on the net. I had to override an existing mode because the 1680 res is not in the list (915resoution -l) by default. All modes were 4x3 instead of 16x9. This is the default list :

---
$ sudo 915resolution -l
Intel 800/900 Series VBIOS Hack : version 0.5.2

Chipset: 915G
BIOS: TYPE 1
Mode Table Offset: $C0000 + $269
Mode Table Entries: 27

Mode 30 : 640x480, 8 bits/pixel
Mode 32 : 800x600, 8 bits/pixel
Mode 34 : 1024x768, 8 bits/pixel
Mode 38 : 1280x1024, 8 bits/pixel
Mode 3a : 1600x1200, 8 bits/pixel
Mode 3c : 1920x1440, 8 bits/pixel
Mode 41 : 640x480, 16 bits/pixel
Mode 43 : 800x600, 16 bits/pixel
Mode 45 : 1024x768, 16 bits/pixel
Mode 49 : 1280x1024, 16 bits/pixel
Mode 4b : 1600x1200, 16 bits/pixel
Mode 4d : 1920x1440, 16 bits/pixel
Mode 50 : 640x480, 32 bits/pixel
Mode 52 : 800x600, 32 bits/pixel
Mode 54 : 1024x768, 32 bits/pixel
Mode 58 : 1280x1024, 32 bits/pixel
Mode 5a : 1600x1200, 32 bits/pixel
Mode 5c : 1920x1440, 32 bits/pixel
---

I don't know why 1680x1050 is not on that list is.

Override on modes 58 and 5a will work. Override on 54 and 5c will not.
With 54 and 5c, the screen will not be centered correctly.

The HTOTAL and VTOTAL value are are also very important. Without them the image quality will be very poor. You must set them to the same values as the modeline. See my example for the correlation.

    * finally, modify the init script (/etc/init.d/915resolution) to take the HTOTAL et VTOTAL into account.

Replace

$PROG $MODE $XRESO $YRESO $BIT

by

$PROG $MODE $XRESO $YRESO $BIT $HTOTAL $VTOTAL

This will allow 915resolution to patch the bios at bootup.

To test without having to reboot, use 915resoltion in command mode with the file or with parameters directly. Ctrl Alt Backspace to restart X. Its best that you test with parameters before you set the /etc/defaults/915resolution in case something goes wrong. If the HTOTAL and VTOTAL are not correct, you may loose the display completely and be forced to reboot.
