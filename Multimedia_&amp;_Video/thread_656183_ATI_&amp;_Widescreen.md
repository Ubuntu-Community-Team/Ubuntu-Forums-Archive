### Post by Yellow Pasque on 2008-01-02
Try setting your resolution to 1680x1050, restarting, and when you get to the part where your screen is messed up, press Ctrl+Alt+F1. Login to the terminal and do a:
```
cp /var/log/Xorg.0.log ~/Desktop
```
Note:(That's a zero between the Xorg and log)
Then you can restart and get your screen back to whatever you're using now. Post the file that's now on your Desktop to this thread (use the # sign to post it between CODE blocks, PLEASE!) and we'll take a look at it and see exactly what's going on.

Regardless of this issue, you should add the following to the end of your xorg.conf
```
Section "DRI"
    Mode 0666
EndSection
```

I'd also try giving it a run with AIGLX disabled and see if that resolves the problem:
```
Section "ServerFlags"
        Option "AIGLX" "Off"
EndSection
```

I had this same problem for awhile.  My fix was in System<Administration<Screens&Graphics.  Once you navigate to that menu click on "MODEL", then at the bottom of that window that gets displayed is a tick box that says "widescreen monitor" tick that then in "manufacturer" click "generic" and finally at the bottom of the "model" pick "plug n play".
