### Post by The Miker on 2007-06-02
I've looked everywhere and I can find no working solution for this.

This only happens when I'm playing a 3d game, mainly the ones with advanced graphics. It's happened in Nexuiz (I think that's how you spell it) and Prey within Cedega so far...but sometimes Open Arena even causes my system to hardlock. By hardlock I mean not being able to do anything except using sysrq to reboot. I can start the game and everything just fine, but I go to walk around and usually about 5 seconds later the entire system hardlocks. I've never had this problem under Windows so I know it's not an overheating problem.

Other odd things I've noticed is poor performance with glxgears and fgl_glxgears. I know, I know, they're not a benchmarking tool, but consider that I used to get 1000+ frames per second rendered with my old 9600XT under Linux and now I'm not even getting 1000 with this brand new card. A quick check of fglrxinfo and glxinfo | grep render confirms that 3d acceleration is working properly, so what's the deal?

Another weird thing is that I'm able to play UT2004 for hours without one single problem, but the games I described above hardlock my system within seconds. What really confuses me is that Open Arena is based on Quake 3 Arena, and such an old game should NOT hardlock my system, especially if a much newer one such as UT2004 can run for hours with no problems.

Here's my specs:

AMD Athlon XP 2500+
1024MB RAM
ATI Radeon X1650 512MB (although it detects as a 256MB card....)
Kubuntu 7.04
