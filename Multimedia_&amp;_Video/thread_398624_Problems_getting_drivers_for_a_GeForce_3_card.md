### Post by RossMM on 2007-04-01
Hi, I'm installing Linux for the first time (I've never used this OS much before) and I've come across two problems that myself and a relatively experienced Linux/Ubuntu user can't resolve. The first was an issue with my motherboard not being supported in newer versions of the Linux kernel ([here]("http://ubuntuforums.org/showthread.php?p=2385719#post2385719")) which might possibly be relevant in fixing the problem I'm having with my graphics card, but as it is partially sorted I think it's unlikely.

My problem with my graphics card is being unable to install any drivers for it. It is an NVIDIA GeForce 3 series card, and I'm using the 6.10 Ubuntu release with the newest kernel (2.6.17-11-generic).

After installation I could set my screen to resolutions up to 800x600, but anything 3D-accelerated did not run at all (I have been using the 3D screensavers to test this). So to install my drivers the first thing I did was download them from the NVIDIA site, and after adding all the dev tools for things such as libc the installer informed me that I would need the legacy version (something that other sources seem to say is only necessary for GF2 cards and before). However after downloading and installing the legacy drivers it still did not work, so I removed them and all related files using Synaptic, and tried to update them using that application. I did also try editing the xorg.conf file, adding / commenting out the sections listed in the stickies for this sub-forum, but none seemed to have any effect on the rendering or options for screen resolution apart from one. This was (I think) using the vesa driver, which displayed the screensavers but very slowly, indicating that the 3D acceleration was working but running poorly.

Finally I used the Envy script that tseliot of these forums has produced. This has installed the drivers and set the xorg.conf file to the same options as I had done manually before, but the screen is now limited to 640x480. However the 3D screensavers are animating and are faster than before, but as my friend pointed out this may be a consequence of the resolution being set so low and not the drivers working properly for the card.

So how do I allow settings better than 640x480 and 50Hz, as it seems my xorg.conf file should allow this but the option is not available in either the basic screen settings or the one added by Envy?

### Post by Huffers on 2007-04-01
We found a sort of fix:

put:

Option "ModeValidation" "NoHorizSyncCheck"

In the /etc/X11/xorg.conf file's "screen" section.

Now it works with 3d acceleration, and at any resolution

----------------

RossMM and I figured this out by:

stopping X:
(ctrl+f1), log in, *"/etc/init.d/gdm stop"*

starting X in log mode:
*"startx -- -logverbose 6"*

then checking the X-windows log file in /var/log/ to see why it was rejecting all the other resolutions (it said something was wrong with the horizontal-sync for all of them)

then looking at the NVidia drivers xorg.conf options
( [http://us.download.nvidia.com/XFree86/Linux-x86/1.0-9755/README/appendix-d.html](http://us.download.nvidia.com/XFree86/Linux-x86/1.0-9755/README/appendix-d.html) )
