### Post by ArtInvent on 2008-02-07
[SIZE="3"]iRiver Clix2[/SIZE]

If you're looking for a pretty sweet little media player with great Ubuntu compatibility, the Clix2 is highly recommended. I recently got an 8GB model and am pretty happy with it. I really wanted to be able to use a music subscription service like Rhapsody or Napster, as it's now possible to at least stream music through a browser in Linux with both these services. I like the service so much that I'm even willing to use my old WinXP laptop on a limited basis to feed the Clix with Napster-to-go tracks. However, I'd still like excellent functionality back on Ubuntu, because my own music collection is on that computer and I really like Rhythmbox. I even have an old Creative Nomad 40gb and a Muvo that I still like to use, and Gnomad2 syncs beautifully between my music collection and the Nomad. But it's kind of a tank and the new mp3 players are so much better.

[SIZE="3"]
Music Subscription: Still have to Sync on XP[/SIZE]

Basically with Napster or Rhapsody I have to download the tracks to my XP laptop and sync it to the Clix, and go back to Ubuntu with relief. Using Windows Media Player, even version 11, is painful. It's got a nice looking interface but it just doesn't behave right and organizing and finding music on it is torturous. However, it's necessary at the moment if you want to put Napster-to-go tracks on. Actually it didn't even work right when I plugged in the Clix to the usb port. I read that it might help to uninstall and re-install Media Player 11, so I did and it finally worked. (Ubuntu actually recognized the player immediately, as I had already had MTP packages installed, I guess by Rhythmbox.) Seems to work well enough so  far. But, I was concerned that you must use MTP mode on the player and not UMS, and you can't switch back and forth without re-formatting the Clix. So I was wondering if it would be possible to use it at all in Ubuntu, since MTP compatibility has been quite limited.

[Addendum: since I first posted this, I've gotten the USB connection working quite well in an XP virtual machine running in VirtualBox. It can sync quite well through WMP 11, so now at least I don't have to go to a separate computer running XP just to sync.]

[SIZE="3"]MTP is Getting Good[/SIZE]

Well, MTP mode in Linux has actually recently improved substantially. Gnomad2 just today came out with an update release 2.9.1 I got from getdeb.net that pointedly has MTP improvements both to sync music and to browse the data directories on the device, which it couldn't before. So it should be possible to add themes, Flash games, photos, video files, etc. to the appropriate directories on the Clix while still in MTP mode. You shouldn't ever need to use USM mode if you don't want to. Right now I've only transfered music and jpg photos over, but it seems to work great and speedy, and the photos and music all show up as expected. There doesn't seem to be any problem syncing the player alternately on Windows and then on Ubuntu. 

[SIZE="3"]Rhythmbox
[/SIZE]
Within Rhythmbox itself, using MTP with the Clix seems to be still not working quite right. You can plug in the player and Rhythmbox detects it, but it seems to hang when transferring. I managed to transfer over an album okay but the album art photo looked very screwed up and some of the tracks weren't complete. For the moment I'm happy using Gnomad2 so it doesn't bother me, but it would be great to get Rhythmbox connectivity perfected as well.

Also, when I plug the player in, Rhythmbox starts up whether I want it to or not, which I would like to disable.

[SIZE="3"]Remaining Issues[/SIZE]

The only issues I have so far: not sure how to transfer over album art. No biggie. And of course it would be great to have a fully functional subscription service with to-go tracks working completely in Ubuntu. I realize this necessitates dealing with DRM in some way: frankly I am cool with DRM on subscription tracks, because I really don't see any way the subscription model works without it. When I buy music, it's either on CD or Amazon mp3's with no DRM.
[SIZE="3"]
The Clix Itself[/SIZE]

The Clix2 itself is pretty awesome. The OLED screen is great if a little small. The menus and navigation are easy and attractive. The FM radio, which I MUST have on an mp3 player, is one of the best radios I've ever used. The reception is awesome, there are tons of presets, and the radio 'dial' is very easy to navigate. I have a set of Etymotic in ear phones, the good ones, and it sounds fantastic with plenty of volume. It sounds better than the Nomad Jukebox and better than the Creative Muvo I have, and they're no slouches in fidelity. The included earbuds are much better than anything that came with the Creative players. Sometimes I don't want the isolation and hassle of sticking the Etymotics in, and the iRiver buds are definitely listenable. There is a little problem in that the Etymotic plug is a super-tight fit into the player. It can be hard to get it out. Weird.

This player seem unusual in that you can customize all kinds of things, make your own themes with fonts and background photos and use Flash for animated effects. You can even make your own Flash games. 

This is also one of the few players with good Ogg support, so if your music collection is in Ogg this thing is a 100% no-brainer. Compared to an iPod with no radio, no Ogg, no subscription or wma support etc. this thing blows them away.

[SIZE="3"]Napster and Rhapsody in Linux[/SIZE]

So this player works well with either music subscription service, but which service is better?

Rhapsody got some good press with Linux types recently for having made some efforts to allow you to stream tracks through any web browser like Firefox with the appropriate plugin. (Unfortunately as I mention above, you still have to use Windows and Media Player if you want to put tracks 'to go' on your portable player, so all is not rosy.) I'm convinced music subscription is the way to go, you have almost any album you want to listen to at your fingertips and it's great for discovering new music, and frankly it seems like a bargain for 12-15 bucks a month. 

So I signed up with Rhapsody. Hmm. Once you get past the awsomeness of having like 4 million tracks to choose from, some chinks start to appear. At the moment, Rhapsody's player window is very buggy. It just stops playing randomly. The album art doesn't show up, or it indicates you're still on the first track you played, even though you've moved on. It signs you out and you can't sign back in and play something unless you log out of Ubuntu and log back in once a day. They seem to have the odd occasional case of missing music files; click on something and it doesn't play, ever. They're very lacking in the album info department. I kind of miss not having extensive CD album notes. I'm not asking for that kind of thoroughness, but on Rhapsody there's often noting at all. You pretty much have to go to Amazon or something to figure out what the album's all about. Not good.

Overall Napster definitely has them beat (though these things tend to change over time.) Napster's web player seems to work better and have more information and album art. Rhapsody's player tends to play a track or two and then stop. I had to constantly go back and hit play for the next track, which as you can imagine is quite annoying. Napster player just works. The two services are fairly equal in selection of pop and contemporary and alternative etc. but I tend to like some rather esoteric classical music and Rhapsody had a better selection. Both services have some holes in their catalogs. Neither service supports gapless playback, which with classical music is often quite annoying. And it's annoying that both services show you tracks that you can't even play, you must buy. Why some and not others, I've no idea.

At this point I would definitely go with Napster. However, if either service or some other service (are you listening Amazon?) offered integrated Linux compatibility, I would switch in a heartbeat.


Anyone else have any other experiences or problems or tips for using the Clix?

### Post by ArtInvent on 2008-03-18
If you insist on using Dapper, and not using any other packages not in the official Dapper repos, you will   almost certainly not be able to use MTP mode with the Clix. 

You'll have to just use UMS. It's a little confusing. In the Clix menu, go to Settings, Advanced, Connection Type. The two choices are MSC(UMS) and MTP. Whenever you switch from one to another, it will reformat the Clix and you'll lose anything you have in memory. And of course, under Select USB Mode you will have to set it to "Power and Data" whatever the Connection Type is set to. If it's set to "Power and Play" it won't connect so you can hook it up for charging only while still being able to use it as a player.
