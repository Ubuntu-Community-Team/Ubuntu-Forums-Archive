### Post by Yellow Pasque on 2008-01-14
Have you completely uninstalled the 7.12 drivers? Press Ctrl+Alt+F1 to go to a terminal, then:
```
sudo sh /usr/share/ati/fglrx-uninstall.sh
```
Now you should be able to install 8.37:
```
sudo apt-get install xorg-driver-fglrx
sudo aticonfig --initial -f --overlay-type=Xv
```
Reboot and see if that works.

### Post by BMWDriver on 2008-01-15
If I may suggest installing the Catalyst 7.11 Drivers. Otherwise you can install previous drivers with Envy by selecting the manual installation option, thus installing a previous driver such as 8.42x. This may be the easiest way. Envy also has an uninstall.

Unfortunately though, the Envy I used did not offer 7.11, so I had to download it from ATI, and install it via console, which was not that difficult after all. Catalyst 7.11 has driver version 8.433. It resolved the issue of display resolution.

In the release notes of Catalyst 7.12, it states that resolutions higher than 1280x1024 may not be recognized. I could not enable my 1680x1050 even with editing xorg.conf by adding modelines. There are no known workarounds to that "bug".

If you choose to install Catalyst 7.11, here is the procedure (link to get Catalyst 7.11 [http://ati.amd.com/support/drivers/linux/previous/linux-r-cat711.html]("http://ati.amd.com/support/drivers/linux/previous/linux-r-cat711.html")):

```
In superuser mode, first reset the xserver (as a precaution):

$ dpkg-reconfigure -phigh xserver-xorg

Then switch to the directory where the 7.11 Catalyst was downloaded, and :

$ chmod +x ati-driver-installer-7-11-x86.x86_64.run
$ ./ati-driver-installer-7-11-x86.x86_64.run

DON'T REBOOT YET ! just follow the instructions for automatic installation for X Server 7.01...
Then go back to the console and initialise the driver, then set the resolution of the screen:

$ aticonfig --initial
$ aticonfig --resolution=0,1680x1050

```
