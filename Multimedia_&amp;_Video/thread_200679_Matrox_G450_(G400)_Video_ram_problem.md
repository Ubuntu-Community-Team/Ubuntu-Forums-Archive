Have you tried glxgears to check wether your video acceleration is working? If its not you may want to use mgapdesk to change the defaultdepth from 24 to 16 bit.
use terminalcommands: sudo aptitude install mgapdesk

(click left to the "roundgoing arrows" option to do this somewhere)

### Post by sp4rd on 2006-10-16
I could only use gl on both screens if I used MergedFB.  I could get gl working with only one screen when using xinerama.  Try a MergedFB if you want to use some program like mplayer with gl output.  I think you might need the mga with the mga_hal driver from matrox's site too.  I think the Option "MGASDRAM" is recommended if you experience screen corruption.  You could also try setting AGPSize in your Section "Device" area to one that corresponds to the size set in your bios.  But the size of your AGP shouldn't affect XV output.
This is my xorg.conf for a MergedFB with the matrox mga/mga_hal driver.  Just use mga for the driver and that will load the mga_hal if you decide to use their driver.  I didn't need the AGPSize option so I left it commented here but I think you do need the MGASDRAM one along with VideoRam.
