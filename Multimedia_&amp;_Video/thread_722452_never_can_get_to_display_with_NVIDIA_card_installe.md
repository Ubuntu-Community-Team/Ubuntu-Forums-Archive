### Post by justsomedude on 2008-03-13
Backup your xorg.conf:

```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.backup
```

Then open it

```
sudo gedit /etc/X11/xorg.conf
```

Switch to vesa driver:

```
Section "Device"
    Identifier     "Graphics"
    Driver           "vesa"
    VendorName     "nVidia Corporation"
    BoardName      "MCP51 PCI-X GeForce 8600gts"
    BusID          "PCI:0:5:0"
#    Option         "NvAGP"       "1"
#    Option         "AddARGBVisuals"    "True"
#    Option         "AddARGBGLXVisuals" "True"
#    Option         "RenderAccel" "true"
#    Option         "NoLogo" "true"
EndSection
```

The above is just an example. Uncomment everything in Section "Device" except for Identifier, Driver, VendorName, BoardName and BusID. Change Driver to "vesa".

This should allow you to boot into a graphical environment with your new card plugged in.

If not, you can restore your previous settings with:
```

sudo cp /etc/X11/xorg.conf.backup /etc/X11/xorg.conf
```
from a root shell.


You can then try to uninstall your current driver and install a new one with the restricted drivers manager. Not sure if it supports the 8600GTS, but it's worth a try.

### Post by scottbporter on 2008-03-24
Solved

I finally got my NVIDIA GTS 8600 installed. I upgraded to Hardy alpha 5, which I assumed by now is the Hardy Beta. But I don't think this was the problem. I went to Nvidia's site, downloaded the driver.run file and xconf file. I backed up my home directory and xorg.conf files in case something bad happened. I utilized the installation instructions at [http://ubuntuforums.org/showthread.php?t=57368](http://ubuntuforums.org/showthread.php?t=57368). 

I ran in recovery mode and went to command line and followed the instructions, basically running the .run file from nvidia and the nvidia xconf utility. Then rebooted. 

On reboot I noticed an error line stating NVIDIA card doesn't have enough power (which I never saw before. ) Note that the card would work, the fan was cooling and it would display in low graphics mode. I found a slot on the card, got a power cable to fit it and connected a power cable splitter and hooked up the card to more juice. Rebooted and everything worked great.
