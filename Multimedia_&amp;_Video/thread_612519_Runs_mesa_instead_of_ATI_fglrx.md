### Post by ephioz on 2007-11-15
Try this:
```
sudo rmmod fglrx
cd /lib/modules/*/misc
sudo insmod fglrx.ko
modprobe fglrx
```

It worked for me.

### Post by Qrikko on 2008-06-19
for the noobiest of us :) that still want to run glxgears (and also maybe other 3d :P) in reasonable fps.. I agree that Mesa is not an option.

well what I would do if I was in your position is to build the kernal with the latest envyng version (I am guessing you are building 8-5 which is the latest ATI driver right now).

I am almost sure that EnvyNG run 8-4 right now and should any day be compatible with 8-5.. but enough of that :P

do:
sudo apt-get install envyng-gtk

this should install a shortcut in your system tools menu (and if you don't use the menu "sudo envyng -g" should also bring up the interface (the -t flag for text mode which is almost identical to the graphic one just without gui.. or something)

Then let this script do the work and it should get it working, and if you really want the 8-5 then if nothing else it might at least fix what went wrong in your previous build..


So SHORT version if you feel lazy and actually jumped to this line the first you did.

try using envy to install the drivers.
get it from the repos: sudo apt-get install envyng-gtk
and use it.

(more information: [http://albertomilone.com/](http://albertomilone.com/) )
