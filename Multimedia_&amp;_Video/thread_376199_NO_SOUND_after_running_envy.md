### Post by bobbob1016 on 2007-03-06
To the best of my knowledge, xorg.conf, the file you pasted in your second post, has nothing to do with your sound.  There are a few things you should try BEFORE trying what I have at the end of this post.  First make sure your sound isn't muted, I know this might sound insulting, but I am just covering everything I can.  Then make sure the speakers are plugged in and working, try them on other devices to check.  If you installed anything before the reboot, or changed anything settings, or removed anything, try to undo those changes.






I would wait for someone else to help before even attempting this.
The only way I could see it having something to do with your sound card is if they are integrated somehow.  I HIGHLY doubt this is the case, and you should ONLY follow these steps if you are brave, and have a reasonable amount of knowledge in linux.

AGAIN, THIS IS PROBABLY NOT GOING TO GET YOUR SOUND WORKING, THIS IS ONLY A WAY TO TEST IF THE SOUND IS RELATED TO YOUR VIDEO CARD.

MAKE SURE YOU ARE SURE YOU WANT TO DO THIS.
Type this in the terminal:
sudo mv /etc/X11/xorg.conf /etc/X11/xorg.conf-nosound
cd /etc/X11/
ls
DON'T TYPE THIS LINE:  You should see xorg.conf_backup_2007something, if you used envy DON'T TYPE THIS LINE:  this year.
sudo mv /etc/X11/xorg.conf_backup_2007something /etc/X11/xorg.conf


IF YOU INSTALLED BERYL AND ADDED XGL, BUT DIDN'T DO IT IN ANOTHER SESSION, I DON'T KNOW WHAT THIS WILL DO, IT SHOULD BE REVERSIBLE, BUT I DON'T KNOW.  If you did install XGL in a different session, meaning if you did something like this: [http://wiki.beryl-project.org/wiki/Install_Beryl_on_Ubuntu_Edgy_with_XGL#Configuration](http://wiki.beryl-project.org/wiki/Install_Beryl_on_Ubuntu_Edgy_with_XGL#Configuration) or if you didn't install XGL or Beryl, you should be able to follow these steps too.
If you installed beryl, and added it to your startup, disable it for the time being.  Click System-> Preferences -> Sessions.  Go to the startup programs tab, and click "beryl-manager" and click disable.  Now reboot.  If you did install XGL in a different session, just select the non-XGL session after the reboot.
What that did was backup your current xorg.conf and give you your old one back.  I included the beryl line, because it would cause problems if you changed your xorg.conf, and had beryl.

If you have sound now, then the sound card is on the video card, if not, follow these steps:
Open a terminal, and type this:
sudo mv /etc/X11/xorg.conf /etc/X11/xorg.conf-pre-envy
sudo mv /etc/X11/xorg.conf-nosound /etc/X11/xorg.conf
Also undo anything you did before, as in enable beryl in the startup, or select the XGL session,
