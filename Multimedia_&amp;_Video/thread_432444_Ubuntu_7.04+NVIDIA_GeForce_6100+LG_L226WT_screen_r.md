### Post by mcook2 on 2007-05-04
Ubuntu 7.04+NVIDIA GeForce 6100+LG L226WT screen resolution

When running at it's native resolution 1680x1050 the screen extends beyond the physical bounds of the monitor to both left and right.  Running at 1440x900 or less is Ok.

I tried various drivers and changes to /etc/X11/xorg.conf but the problem remains.  Only driver "nv" seems to work at all.  I also tried using Envy but I still get an unusable configuration. 

See also [http://www.ubuntu-es.org/index.php?q=node/46245#comment-114712](http://www.ubuntu-es.org/index.php?q=node/46245#comment-114712)

Can anyone suggest a resolution to this problem? (No pun intended).

### Post by mcook2 on 2007-05-06
I was able to resolve this as follows:

First, you might need to use System/Administration/Software Sources to  turn on "Proprietary drivers for devices (restricted)"

Looking under System/Administration/Restricted driver manager I installed "Nvidia accelerated graphics driver".  The messages showed it was installing nvidia-glx. 

Upon rebooting the new automatically generated /etc/X11/xorg.conf shows my NVIDIA GeForce 6100 graphics card and LG L226WT monitor have been auto-detected correctly for the first time, and I can now use System / Preferences/Screen resolution to switch to 1680x1050 without getting the width problem.

I was uncertain how to check which version of the driver I was using.  Eventually running nvidia-settings confirmed the active driver version to be 1.0-9631 as below.  It also shows the graphics card VBIOS version number for my GeForce 6100 nForce 405 is 05.61.32.25.04.
