### Post by tommcd on 2008-01-27
After you install the nvidia-glx-new run "sudo nvidia-xconfig" to enable the driver. Then edit the /etc/default/linux-restricted-modules or /etc/default/linux-restricted-modules-common configuration file and disable the "nv" driver by adding this to the file:
```

DISABLED_MODULES="nv"

```
I have read that "nv" sometimes conflicts with "nvidia" so disabling nv may help.

### Post by orionsbelt on 2008-02-05
Sorry for the late response!  I managed to get it working (with the help of my roommate), although not quite through your excellent suggestions.  I think they would have worked, but she did a clean install and followed the instructions here:  [http://albertomilone.com/latest_nvidia_udsf_feisty.html#METHOD_1](http://albertomilone.com/latest_nvidia_udsf_feisty.html#METHOD_1).

Then came the next problem:  my screen still blacked out right before the login screen.  However, my monitor is a Dell 1907FP, which is an LCD flat panel monitor.  I also have a nvidia 7600 GT card, which has both a DVI and VGA input.

My screen wouldn't display on the DVI screen.  To get it to work, I had to do the following:

(1) Connect the VGA cable and switch to Analog Input.
(2) Go to Applications -> System Tools -> NVIDIA Settings (I used the one with the fancy icon, but I think you could use both).
(3) Click "X Server Display Configuration" on the left-hand toolbar.
(4) It would show two screens, one of which was disabled (for me, the one on the right).  I click on this one, and click the "Configure" Button below it.
(5) Choose the "Twin View" option.
(6) Change the resolution to my preferred choice.
(7) Click "Position", choose "Clones".
(8) "Apply" and save changes.
(9) Switch my input back to Digital.

And voila!  It works!  The only problem now is that I have to do this every time I boot into Ubuntu.  Despite the annoyance, I'm just glad it's working.
