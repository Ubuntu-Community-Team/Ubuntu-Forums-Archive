### Post by openback on 2006-06-11
I'm on an Inspiron 710m with an 855 video adapter and had the same problems as everyone else here. After going crazy, I spent a RIDICULOUS amount of time searching the Xorg log and found that the driver (i810) didn't like the refresh rates in my xorg.conf.

I ended up commenting out the "HorizSync" and "VertRefresh" lines in my "Monitor" section (just add a "#" to the beginning of the line) and changed my Modes line to say "1280x800@60" so that the refresh rate was in there. And all was well again. Then I just removed those lines.

So now my xorg.conf has this snippet in it:

```

Section "Monitor"
	Identifier	"Dell Display"
	Option		"DPMS"
EndSection

Section "Screen"
	Identifier	"Default Screen"
	Device		"Intel Corporation 82852/855GM Integrated Graphics Device"
	Monitor		"Dell Display"
	DefaultDepth	24
	SubSection	"Display"
		Depth		24
		Modes		"1280x800@60"
	EndSubSection
EndSection

```

and in my /etc/default/915resolution I have:
```

	MODE=7e
	XRESO=1280
	YRESO=800
	BIT=32

```

Mode 7e already had these values initially, but I still needed to use it anyway. I don't know why, but my display only supports 24-bit color, but I needed to put 32 in that file in order for it work. xorg needed the value 24 in order for the setup to work also, so play around with those if you have issues.

### Post by hamamelis on 2006-10-07
I forgot to put in the crucial line, when I did my last post.

Do the following:

Install the i800/900 VBIOS Startup Script for Ubuntu by Robert Pectol as in README then edit the scipt in the following way:

sudo gedit i800_900_res_patch.sh

Remove anything that may be there and insert the following 4 lines

#!/bin/bash

915resolution 30 1280 800
exit

then save this as /etc/init.d/i800_900_res_patch.sh

The reason is that the patch supplied by Ubuntu to correct the i810 resolution problem uses modes with high numbers and in some way these are not picked up by the Acer Travelmate 3004. If the 1280x800 resolution is put in the lowest no 30 it is detected. 

Then reboot the computer, then once in Xwindows, run in a terminal:
sudo dpkg-reconfigure xserver-xorg

THis reconfigures the X system. I use the defaults and select resolution 1280x800. In fact it only ever acheives 1280x768 for some reason but that is fine.

Restart X11:
sudo  /etc/init.d/gdm restart

You will have to run the same command again at the system prompt probably to start X, or you could just reboot.

Now the resolution should be 1280x768, 800 if you're luckier than me.

I think these changes are more important than exactly what goes into xorg.conf. Sorry I forgot all this earlier. There is probably an easier way of doing all this, but I don't understand permissions ans run levels too well.

Good Luck, Ian.
