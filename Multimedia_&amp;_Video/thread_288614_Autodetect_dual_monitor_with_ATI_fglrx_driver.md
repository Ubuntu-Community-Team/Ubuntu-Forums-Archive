Hi,
I apologize if I'm posting in the wrong section, or if my question was answered previously, but I've been searching all the forums (and I googled for hours) without being able to find any solution. ](*,) 

My problem is the following: I have configured BigDesktop mode for my Radeon 9700 Mobility-equipped laptop, and it worked perfectly on Dapper Drake. It was also able to detect when the second monitor was connected, and it activated dual monitor mode **only** when a second monitor was available, otherwise X started in normal mode (only LCD panel).

This stopped working since I upgraded to Edgy Eft: now X starts **always** in dual monitor mode, even when it's not required. Besides being annoying, this causes some problems with many programs (e.g. Firefox 2.0 is unable to start at all, console windows start on the wrong, disconnected, monitor, and so on).

I tried to force auto detection as suggested [here]("http://gentoo-wiki.com/HOWTO_Dual_Monitors#With_the_proprietary_binary_driver"), but it didn't work (Option "MonitorLayout" is not used anymore, and ForceMonitors doesn't obviously support any "auto" parameter).

So far, the only solution I managed to use is using xrandr to switch back and forth between the dual head resolution (2560x800) and the single one (1280x800), and then using aticonfig --enable-monitors=lvds,crt1 or lvds only to manually toggle the VGA output.

Any suggestion would be greatly welcome, and thank you for reading so far.

While it might not work for you, my xorg.conf is "more simple" and I can control how the second screen should behave - expand desktop or mirror primary display - without reboot of X.

Basically I've only 1 Device section and 1 Screen section. The rest is managed through System -> Preference -> Screen Resolution (for this to work you need fglrx 8.28.8 or higher)

What did the trick is option DesktopSetup horizontal.
