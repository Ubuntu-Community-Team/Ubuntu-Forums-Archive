### Post by rotten on 2006-09-11
This helped:
------------

After you install a new monitor on your Ubuntu system, log in remotely, set your http_proxy environment variable, and then run this command:

$ sudo dpkg-reconfigure -phigh xserver-xorg 

After running it, try rebooting.
If you get an error on your display, you'll need to log back in remotely and edit the /etc/X11/xorg.conf file.

Try different video modes, then restart X:

root@edea021:/etc/X11# ps -ef | grep X
root      5031  5028  1 09:42 tty7     00:00:01 /usr/bin/X :0 -br -audit 0 -auth /var/lib/gdm/:0.Xauth -nolisten tcp vt7
root      5053  4893  0 09:43 pts/0    00:00:00 grep X
root@edea021:/etc/X11# kill 5031

iterate until you get a display.

### Post by NobodySpecial on 2006-09-11
It is impossible to give one set of instructions for everybody because of various video cards, but this worked for me with DVI connection and nvidia drivers:
```

Section "Monitor"
	Identifier	"Generic Monitor"
	Option		"DPMS"
	HorizSync	30-81
	VertRefresh	56-75
	DisplaySize	518.4 324
EndSection

Section "Screen"
	Identifier	"Default Screen"
	Device		"NVIDIA Corporation NVIDIA Default Card"
	Monitor		"Generic Monitor"
	DefaultDepth	24
	SubSection "Display"
		Depth		16
		Modes		"1920x1200_60" "1600x1200" "1280x1024" "1024x768" "800x600" "640x480"
	EndSubSection
	SubSection "Display"
		Depth		24
		Modes		"1920x1200_60" "1600x1200" "1280x1024" "1024x768" "800x600" "640x480"
	EndSubSection
EndSection

```
Notice I didn't need to do the "modeline" thing at all.  Don't forget to set Subpixel Smoothing under System -> Preferences -> Fonts after you get the resolution thing working.
