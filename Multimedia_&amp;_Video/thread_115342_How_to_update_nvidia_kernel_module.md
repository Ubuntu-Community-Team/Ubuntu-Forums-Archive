### Post by SolidAndShade on 2006-01-10
I'm running Breezy on an AMD64 system with a Geforce 6150 motherboard, and I don't have X working yet. I just installed the nvidia 1.0-8178 driver and it appeared to compile successfully, following all instructions in tseliot's nvidia driver installation thread. However, when I reboot the system X fails to start and I get a message saying that the installed nvidia kernel module is version 7667 while the X nvidia module is 8178. So it appears that the kernel module was not updated. I thought that the nvidia installer created a new module, but apparently that's not the case. Does anyone know how to replace the kernel module with the latest version? Thanks.

---

### Post by MartinG on 2006-01-10
You need to remove the linux-restricted-modules, as otherwise on boot the system seems to prefer the module in that to the one generated by the NVidia installer.  (Some people say that they still can't get it to behave properly even then on an AMD64 -- try it and see.)

See also this thread:
[http://ubuntuforums.org/showthread.php?t=114853](http://ubuntuforums.org/showthread.php?t=114853)
