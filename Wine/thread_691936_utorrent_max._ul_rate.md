Anyone come across this oddity? I use utorrent/Wine in Gutsy - I tried Deluge but found it a bit too buggy to be useful right now. It works fine in most respects except that the setting 'Alternate rate when not uploading' seems to be ignored. Instead the 'Maximum u/l rate' is used whatever combo of uploads and downloads is occurring.

This is a bit of a nuisance as an unlimited u/l rate kills the d/l speed quite effectively but I like to revert to best seeding rate once the downloads have completed. I'd guess that rather than ignoring the setting utorrent is actually not registering the fact that it's actually downloading. I've got a similar setup on a backup Feisty box and this problem doesn't manifest on that.
