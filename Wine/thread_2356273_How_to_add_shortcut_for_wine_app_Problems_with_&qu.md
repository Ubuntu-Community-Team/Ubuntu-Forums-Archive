### Post by Dennis N on 2017-03-21
How did you create the shortcut? If I right-click on a windows executable, and select **Send to.. > Desktop (create link)** this works, provided you right-click on the created link and select "Open With Wine Windows Program Loader".

A better way (in my opinion) would be to add a .desktop file (serves as a launcher) for the program in your Desktop folder. How to: The Wine installer may have created a new menu entry (launcher) in your menu, in which case you can send that launcher to the Desktop from it's right click menu. If there is no menu entry, you can manually create a .desktop file, and place it in your Desktop folder. Using this method has the benefit of displaying the program's icon, and allows you to specify a working folder.

Help about .desktop files:

[http://askubuntu.com/questions/281293/creating-a-desktop-file-for-a-new-application](http://askubuntu.com/questions/281293/creating-a-desktop-file-for-a-new-application)
[http://askubuntu.com/questions/64222/how-can-i-create-launchers-on-my-desktop](http://askubuntu.com/questions/64222/how-can-i-create-launchers-on-my-desktop)
[https://developer.gnome.org/integration-guide/stable/desktop-files.html.en](https://developer.gnome.org/integration-guide/stable/desktop-files.html.en)

Example of a Exec= line for a Windows application run under wine:
```
Exec=wine home/username/.wine/drive_c/"Program Files/Litsoft/Across Lite/ACROSSL.EXE"
```
Note this starts with **wine**, then the path to the executable. The part containing spaces was enclosed on quotes, or it won't be parsed correctly in Linux.
