### Post by donkyhotay on 2009-02-05
Open source means the software is free, as in speech. Not free, as in beer. Although there are many people that donate time and knowledge to open source projects for nothing more then community recognition, at the end of the day most developers still need to put food on the table. Admittedly cedega isn't completely open source, but much of it is and the project does provide benefits to community. If the program is worth using then it's worth paying for. If it isn't, use wine instead! This is true whether it's proprietary or not. There are many open source projects I've seen that are honestly worth at least worth a couple bucks and if everyone that liked (much less used) it did so then we might get more devs working on less business applicable programs (like games). I'll admit I'm a bit of a cheap-skate myself (getting mad about having to buy *another* copy of windows is what forced me to linux in the first place) and have never paid money to a project but I do try to support projects I like in other ways. I make it a point to send in useable bug reports, code audits, updated documentation, and on occasion bugfixes and feature enhancements in the code itself to projects that I actually use. You get what you pay for and this is true with linux. The nice thing about linux is that you don't always have to pay with cash or credit cards.
