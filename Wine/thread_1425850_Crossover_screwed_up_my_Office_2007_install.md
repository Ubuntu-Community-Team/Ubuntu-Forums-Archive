### Post by UncleChicken on 2010-03-09
Hello,

 I am running Office 2007 in Wine (1.0.1) in KK 9.10, and it has been working great. However, I've been trying to get Bibleworks 8 to run better (it runs in Wine OK, but has a few really annoying glitches), so I tried to install it with Crossover. It did not work very well, so I uninstalled Crossover (I went into terminal and uninstalled it completely). 

 However, I just tried to load Word, and after it loads, I get an error message saying, 'Word is not installed for this user - please run setup and install this program.' Now I imagine this is a result of Word's believing it is running in a Windows environment, but I am the only user on my machine. I tried restarting my machine and like all easy fixes, that of course did not do anything. 

 I imagine this is just a matter of pointing the program to different directories that Crossover may have switched; but honestly, I don't know. I could just re-install, but that is such a pain in the neck.

 Any suggestions?

### Post by UncleChicken on 2010-03-09
OK, figured it out; For some reason crossover changed my Wine Version to 1.0.1 - I thought that didn't look right when I posted it in my previous post. I have updated to the latest version, which is what I had been running, and everything is good again.
