Some interesting things to note:

The crash seems to happen while it is loading standard profiles or while it is moving first run files. I am installing it fresh in my windows box today to see which files are being moved exactly and seeing if I can't just move those files manually to see if that makes a difference.

Deleting the configuration files in Windows seems to stop it from crashing. It doesn't stop it from crashing in linux, but it is interesting to see that Flash does rebuild the configuration files as it should before it does crash.

Running it as Windows 2000 in winecfg got it to display the "Help make our product better" dialogue before crashing. However, I've been unable to reproduce this.
