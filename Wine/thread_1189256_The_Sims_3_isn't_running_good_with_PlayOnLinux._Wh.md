Not to threadjack, but it isn't exactly ironic. Wine isn't an emulator, its a compatibility layer. It doesn't emulate the programs code - it actually let's the program run on your cpu. If you write a program that does absolutely nothing windows specific (just prints stuff to console) and let it run under wine, the performance will be the same as compiling the same program on windows.

To explain why you need a no-cd crack - wine doesn't support the copy protection the game uses. It does some bad mojo that wine hasn't implemented yet.
