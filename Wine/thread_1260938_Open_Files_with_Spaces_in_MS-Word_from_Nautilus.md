### Post by ignasigarcia on 2009-09-08
Hi there. I'm trying to figure out how to open a *FILE THAT CONTAIN SPACES.doc* using the file-association in nautilus. By default, this is the command generated by wine for MS-Word.

wine start /ProgIDOpen Word.Document.8 %f

the file I try to open is called *FILE THAT CONTAIN SPACES.doc*

When I open it Word claims that the file /home/ubuntuuser/Documents/FILE does not exist. (you can see it's ignoring everything else after the first space)

If I go to the shell and add a couple of " to the command, between the whole route of the file, the same thing. If I add \" then it won't even open it.

Any suggestions on how to fix this bug?

TIA

Ignacio

### Post by ad_267 on 2009-09-09
Or alternatively, "FILE THAT CONTAINS SPACES.doc" (in quotation marks):
```

```
