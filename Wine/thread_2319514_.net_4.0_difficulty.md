### Post by montag dp on 2016-04-06
I'm not clear how you installed .net 4.5 in the first place.  Did you use winetricks or copy it from your Windows installation?

If you haven't already, try a fresh wineprefix and use winetricks to install .net 4.0:

```
WINEPREFIX={path to your wineprefix here} winetricks dotnet40
```

 You will need to install winetricks if you don't already have it.

### Post by montag dp on 2016-04-07
Joe,

When you use Wine, it sets up a fake Windows directory where your Windows programs are installed. This directory is called the wineprefix. If you look in it after installing a program, it will probably look familiar to you -- there is drive_c which contains Program Files, Users, etc. The default wineprefix is located in your home directory and has the name .wine (the . in front signifies that it is a hidden directory). To create a fresh wineprefix, you can simply delete this directory, and a new one will be created when you set up Wine the next time. You can also use a different directory as the wineprefix by putting WINEPREFIX={your desired directory} at the front of any wine commands.

Now, about your problem. The *actual* Windows 7 might have .net 4.0 installed, but that doesn't mean that Wine does. You will need to install it yourself. That's where winetricks comes in. It is a tool to automatically download and install (using Wine) common Windows programs and libraries. The command I posted previously will install .net 4.0 for you, at least in a perfect world. If you don't have winetricks, you will need to install that first from the Ubuntu repositories.

All that being said, I have tip I have for you. Wine can be kind of difficult to get working correctly, particularly if you are new to Linux. There's a graphical front-end for Wine called PlayOnLinux which will probably be simpler for you. It can also be used to install the libraries (.net 4.0) that you need -- you simply tell it that you want to install some libraries while you are installing your Windows program, and then select them. In fact, I've had better luck with that than I have with winetricks in some cases. I recommend you check it out. It is also available in the Ubuntu repositories.

Finally, before installing your Windows app in Wine, you should check out appdb.winehq.org to see if someone has already tried.  This site can be helpful to see if the program is already known to work / not work, and what steps to take to make it work.

### Post by montag dp on 2016-04-13
A PlayOnLinux virtual drive is pretty much the same thing as a wineprefix.  I think the only difference is that PlayOnLinux puts in a couple configuration files, but otherwise it's the same thing.  You can actually install a program using PlayOnLinux and then run it with plain wine.  I've done it before.

About your problem.  When you used PlayOnLinux, did you try to install the program, and click "install some libraries," and then have PlayOnLinux download and install dotnet40?  (It sounds like you did, but I'm just checking to be sure.)  If you did that, and your program is still giving you the error, it might just not work with Wine. Many programs don't, unfortunately.  You should check appdb.winehq.org to see if anyone has tried installing your program before.

As for using MS utilities to install/uninstall things in wine, I wouldn't do that.  If you want to uninstall and start fresh, simply delete the wineprefix (or PlayOnLinux virtual drive) and then start over.
