### Post by starblessed on 2012-09-25
Hiya.

Its not bad.  GW2 is highly CPU intensive.  The graphics are secondary.

The game will only run on a single core, so that core needs to be fast to get any usable result.

I run a 1090T Hex Core that tops out at about 4GHz.  It runs relatively well, though as of writing, the game is still much more playable on Windows with the same config.

Windows I get around 120fps @ Max settings

Linux seems to be limited to about 30fps and drops dramatically in heavily populated areas.

Playable, but nowhere near as smooth.  There also seems to be stuttering, as the game constantly loads content from the hard drive, without caching.  Therefore its always loading and seems to freeze briefly every time it does.

As the Wine dev's catch up, the fps should increase.  If I can get at least 60fps via Linux, Ill switch completely.

Ive also heard of people with nVidia cards getting better results than ATi users like myself.

Hope this helps!

Star

### Post by pf2 on 2012-12-05
Hello everyone. 

Recently I completely switched to xubuntu 12.04 from win7. After installing Guild Wars 2 through PoL, I managed to play with  ~ 20-35fps (about 1/2 of fps at win7).

Trying to find a possible improvement of fps, I installed cpu frequency scaling, in order to set it manually ([http://askubuntu.com/questions/142688/cpu-frequency-scaling-for-12-04](http://askubuntu.com/questions/142688/cpu-frequency-scaling-for-12-04)).

After installation of this, I scaled cpu frequency to highest possible (mine i5 3570k=3.4Ghz), and discovered that fps rose to 45-60 during pve with the same in game settings.

Hope to help others too
