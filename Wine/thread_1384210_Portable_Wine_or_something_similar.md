### Post by labinnsw on 2010-01-25
This is an interesting topic. I am not going to view this as a problem that requires a solution, but more like a request for an optional extra.

Before I carry on my rant, the direct answer to your question is, I know of no portable wine, never seen anything related to the subject before today and when I did a search, I could not find anything.

Now for my rant.
Why wouldn't you want to install wine? It works in the background as if it is not there and it is a linux application. This is not an exact comparison, but, if you want to run java programs, you install java. You want to run python programs, you install python and so on and so forth. This is true on Windows, Mac Linux and any other OS you can think of. You want Windows programs to work on Linux without having to port them, just install wine. Notice I did not say if you want to install Windows programs on Linux. Sure you can install them if you want to, but there is no need if you don't want to. If you prefer to use portable applications on linux, You can do so provided you have wine installed.

As a Linux thinker, you are on the right path. When I install Ubuntu on my Machine, the process takes a few minutes and the result is a machine that is ready for the most part to do any task I might need to. When I install Windows (XP) it takes hours and once that is finished I have to secure it and start reaching for CDs left rignt and center if I want to do anything useful on it. Yes, your innovative thought pattern is admirable. Windows does not even have a wine equivalent, but you are already thinking of making the compatibility layer portable!!!

I am interested to see what the developers think about that and even though I have no use for a portable "Windows compatibility layer," it would be nice if they would get cracking on it just so we are able to say we can.

In the meantime you can use virtual box. It will allow you to not just run Windows applications, but Windows itself on Linux.
(the portable version will require wine.)

**Correction:** 
There are some Linux compatibility layers for Windows.
