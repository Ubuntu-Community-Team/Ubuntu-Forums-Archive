### Post by KhaaL on 2007-12-01
I'm a wine lover (litterarly!) and it hurts me to see that though all the development strides in the wine project, none is noticed on the appdb and the bugs.winehq sites - who are important places for the users to exchange information to eachothers and wines functionality to the developers.

I don't know how many missing bug reports the disintegration has made itself guilty of, the bug tracker and the appdb need seperate logins. And often the bugs are submitted by a user who is also a appdb user. 

Not to mention that bug submitting is a bit clumsy:if you submit a bug for a application, then you'll need to go back to "known bugs" section in appdb  and link it there. Why not make it work seamless?

I also have a lot of critism towards appdb looks and how it behaves. If you want to submit test data, you have to be logged in. Why can't anonymous users send in test data? they need to be approved by the (super)maintainer anyway so there is little risk that junk reports would be filed. Appdb could start using OpenID or image verification to let anonymous users submit testdata.

And personally, I'd really like to see the text editor to go for the "add test data" section. It's a overkill when all you need to do is to submit plain, unformatted test data. We're not building a myspace page! :-)

Another thing to encourage appdata submission is if wine had a "data submission" program running after the current exe has finished (or crashed). It would be convinient because the user wouldn't have to go to the site to submit the report, and the application would detect and pre-fill certain fields, like version number, the OS run, wine version... 


Some of these things are complains mixed with suggestions, but they're all coming from someone who's sad to see one of wines backbones getting old in comparision to the other aspects of the project.


EDIT: Oh, and winehq really needs a forum too!

### Post by cogadh on 2007-12-01
The reason you need to log in in order to submit AppDB data is because the Wine devs may need to contact you regarding the content of your testing report. They can't do that without some kind of e-mail address and forcing submitters to register ensures that they will get valid e-mail contacts. The same is true for Bugzilla, though I agree, AppDB and Bugzilla should have combined logins.

Wine is old school, they don't have a forum, they have a mailing list. It is really meant for developers and not users, but you can join it to watch the Wine progress. There are official IRC chat rooms you can take advantage of as users.

I believe a "error submission" system is already in the works (I think I remember seeing something on the mailing lists about it), but it is not a high priority item when there are still so many basic functionality items to implement or fix first.

While I do agree that the state of the WineHQ site is rather dated, I personally would rather the devs spent their time working on Wine, rather than updating the website. Websites are incidental, the application is what matters.
