### Post by cogadh on 2010-07-16
It installs the latest *stable* version, but that is not the latest version. Wine, like many open source projects, maintains two versions, a stable version and a development version. The stable version generally does not get updated, except for security fixes, and is meant to give everyday users consistent performance. The development version is the "work in progress" version that will eventually become the next stable version. It may contain bugs and regressions that the stable version does not have, but at the same time, it has tons of functionality that the stable version is lacking. Many people find that the extra functionality of the development version is worth risking the possible bugs. Currently the development version has reached "release candidate" status for the new stable version.

See the sticky topic at the top of the forum for instructions on getting the latest Wine version.
