### Post by JGJones on 2007-12-18
As an avid EVE player, I stopped playing EVE about 2 years ago when I switched full time to Linux instead of dual booting with Windows.

However since CCP now support Linux, I've started playing it again.

The Linux client does play very well - it's wrapped in Cegeda so it's still using Wine, but optimised for EVE and I'll recommend this over Wine because it's a supported version by CCP.

There are some minor issues - I would prefer to play in a window rather than full screen - however you cannot set this as an option in EVE (I do this with Team Fortress 2 with great results) so I've had to tweak the Cedega settings so that it display as a window (whose size is larger than graphics options set within EVE itself)

In my case - window size is 1600x1200, and EVE plays as "full screen" within that at 1400x990 (you get black areas around EVE where it doesn't fill the screen)

It works very well, even if you are using Compiz Fusion (I use 3D cube, wobbly windows etc etc - no effect on EVE).

Finally for EVE's Trinty graphics update - sadly so far we can't get it on the Linux version...yet! However I've played it with upgraded graphics on Windows - it's very impression, but does have performance hits. Having said that...the "old graphics" is still very decent and does obviously give higher FPS.

Hope this answer most of your questions, but please do ask away.

Cheers

Another in the very long list of issues with it is that you must turn off Shadowing and set many things in regedit such as fbo for DirectDraw, which slows things down inherently. Suns will not flare with settings like these and there are rendering issues with the Maps. In short, if all your going to do is sit in a station, then you'll be one happy camper with the state of Premium content via wine right now. Mining might be OK too. For the rest of us however, who play the game to its fullest, it is a non working solution at best. Stability and rendering issues make playing via wine extremely problematic. Thats life with Premium via wine right now. Development continues and should produce a more working solution in time. Most likely AFTER CCP releases the Official TG release, but just a guess. Now on to the many and varied issues with the Classic client via wine..

1. Static objects do not render in mission "rooms". Aweful hard to complete many missions if you cant see your objectives. You can ferret them out via the overview, but meh, id rather see what im after.
2. In game Map issues. The Map Browser DOES NOT WORK at all. This is HUGE! You learn to rely on that thing quite a lot as your experience in EVE grows. It is an extremely valuable tool for planning and logistics. Also, the mission maps fail to work at all.
3. Combat related graphics fail to render properly. Instead, what you get most often is large blotches where hits occur instead of the explosion graphics.
4. Warp Tunnels are completely hosed. Large blotches and weird geometrics is what you get instead.
5. Sound/ALSA issues. This relates to the EVE Voice issues already well reported most likely. In fact, right now its RECOMMENDED to turn sound off completely if you get sound error dumps. Again, totally unacceptable.

Not one of these issues exists in the Official Client. So, in short, the TG Wrapper just works, wine does not. It is possible that in time, perhaps even before the official release, that these issues will be fixed in wine, but right now, this is not the case.

### Post by buntunub on 2008-01-11
[This]("http://myeve.eve-online.com/devblog.asp?a=blog&bid=517") should clear up alot of questions and cuts to the heart of what everyone seems to be all wrapped up about with that TG Wrappered Client vs. wine. Strait from the CCP Devs keyboard. What I find to be most interesting:

> Q. Will EVE: Trinity break on Linux using Wine?
A. Maybe. We have implemented a number of new functions and code that is needed by both the EVE: Trinity Classic and the EVE: Trintity Premium client. Most notable is Asynchronous IO and IO Completion. This is not currently supported by Wine, so EVE: Trinity will not run on Wine at the initial release.
