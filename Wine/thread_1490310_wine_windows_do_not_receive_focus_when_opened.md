### Post by rabbitoid on 2010-05-22
I'm not sure if this is a wine or Gnome issue. Anyway, it's just a minor annoyance.

I'm on Ubuntu 10.04.

I notice that when I click on the minimized icon of a wine application it doesn't behave as other windows. Normally, when I click on a minimized icon, the application re-opens and has the focus. for wine application a click on the minimized window opens it, but the window does not have the focus. The focus stays with the application that has it before. A second click on the now unminimized icon gives it the focus.

It's the same if I switch application with Alt-Tab, if I go to a minimized wine application it comes to the foreground, but without the focus. 

This is a small pain, because my fingers remember what they usually do: do something in an application, copy, switch to another, control-v to paste, oops... :(

This behaviour is new, I think, since the update to 10.04. since I updated everything at once, I don't know what caused it: the new gnome, maybe wine. Of my usual applications, only wine windows behave like this.
