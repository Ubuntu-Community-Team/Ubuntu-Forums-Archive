### Post by tghe-retford on 2008-06-14
I don't know how long this has been implemented for, but the way that you browse applications has changed. Instead of browsing through categories to get a list of games like before (it still exists):

[http://appdb.winehq.org/appbrowse.php?catId=0](http://appdb.winehq.org/appbrowse.php?catId=0)

You now have to select filters to find the application you want:

[http://appdb.winehq.org/objectManager.php?sClass=application&sTitle=Browse%20Applications&sOrderBy=appId&bAscending=false](http://appdb.winehq.org/objectManager.php?sClass=application&sTitle=Browse%20Applications&sOrderBy=appId&bAscending=false)

I am getting used to it, but it was fiddly at first because you have to select the individual options before you can browse down the list of applications. It's set to submission date and lowest number per page as default so if you don't want to click every few applications to the next page, you have to set these options.

Nonetheless, it is a bit quicker than the category function, once everything is setup.

What are other people's thoughts? Step in the right direction or more confusing to browse?
