### Post by cwwilson721 on 2010-09-04
> **5dolla said:**
> ok dont flame me but.... which version of ubuntu is better for wine 
gaming and why?
As long as Compiz is disabled, it really doesn't matter which version of the distro you use. And, as far as wine running speed (except for VERY small differences), it really doesn't matter what Distro of Linux you use (DSL, Unbuntu/Kubuntu/Xubuntu, Slackware, Debian, etc...)

As for 'why', it goes to the very basics of what the Desktop GUI is, and what Linux is.

The GUI you use (gnome, KDE, XFCE, et. al.) is just a pretty interface that runs on top of Linux, that basically allows you to run programs via a mouse, rather than typing in commands (I know this is VERY simplified example of what happens, but...). wine is just a program that pretty much runs the same on all versions of Linux. 

What you are actually seeing is the xserver. And THAT is fairly standard. The differences are the desktops. Whether it's Gnome, or KDE, or Fluxbox doesn't really matter much to wine. As long as you have plenty of memory, and a videocard that will work good, wine will happily plug along in any environment that it can run in.

But in 'buntu distros, make sure that Compiz is disabled. That will give you a performance/operative hit in wine most of the time.

So, pick your favorite, and have fun with wine!
