### Post by gsanders99 on 2009-02-23
I have a dual-boot system.  I decided to run the standalone SplashID in Linux using Wine and let it sync with the Windows version's database in the Windows partition of my hard drive.  It means I have to load XP to sync the Palm to SplashID, but I don't make changes in SplashID often enough for that to be a problem.

I wrote out how I did it on my site: [http://www.gasanders.org/projects/splashid_linux.html](http://www.gasanders.org/projects/splashid_linux.html)

What this does for me is lets me have SplashID available when I'm using Kubuntu (which is most of the time), and I can make changes there, too.  When I exit SplashID, it prompts me to sync to the Windows partition.  If I made a change, I say yes.  If no change, I say skip.  If I want the changes on the Palm, I log out, load Windows and sync the Palm.

A kludge, true, but it works for me.
