### Post by euler_fan on 2008-03-03
Given NIST 800-53 is a guide to building a security policy in general which addresses your needs. Ergo, there can be by definition no definitive guide to making Ubuntu NIST 800-53 compliant. I think you would find it more helpful to go through the process it outlines and come back later with a list of what you need to do.

For instance, we are talking about very different sets of requirements to be NIST compliant if you are running Ubuntu as a mail server as opposed to a desktop. Likewise, some desktops/laptops will require CAC card readers, etc.
