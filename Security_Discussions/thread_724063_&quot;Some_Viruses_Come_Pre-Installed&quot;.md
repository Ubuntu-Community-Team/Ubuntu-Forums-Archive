### Post by tubezninja on 2008-03-14
If it's a deliberate attempt, it's as likely as the malicious actor is skilled.  If they can write malware targeted at linux boxes and can successfully convince people that it needs to be installed, then it'll be installed.  This is a vector that targets the weakest element of a linux box: the user at the keyboard.  Though I imagine once the malware is detected and word gets out, a fix will come out in short order for most distros.

How likely is it for a hacker to deliberately target linux users?  I'm not normally a fan of security by obscurity, but in this case, the likely target of malware is still Windows machines, because a hacker wanting maximum effect will target the largest installed base.  It's also just plain easier to write malware for Windows, or re-engineer and re-deploy any of the countless existing specimens already.  I'm also willing to speculate that someone with the skills to write and deploy viruses might actually use Linux or some other OS for their own non-virus-writing activities... the whole don't-go-to-the-bathroom-where -you-eat thing.

Of course, if someone has an OS-specific vendetta for whatever reason, then all bets are off.

For accidental infections, the chances are slim to none.  The viruses that are out there are by and large written for Windows machines.

### Post by HermanAB on 2008-03-14
Linux viruses are mainly a problem on web servers, since web servers on optical fibre backbones make fantastic high volume spam servers.

I fix one or two compromized Linux machines per year.  Invariably the trouble was caused by some hyper intelligent schmook who thought that a four character password is way more cool than a 16 character pseudo random one.

If you are running a Linux desktop system behind a little home router NAT box, then you have nothing to worry about.
