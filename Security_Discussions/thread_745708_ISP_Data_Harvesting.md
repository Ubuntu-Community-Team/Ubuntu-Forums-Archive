> **deh1963 said:**
> In light of this article[COLOR="DeepSkyBlue"][ Every Click You Make]("http://www.washingtonpost.com/wp-dyn/content/article/2008/04/03/AR2008040304052.html")[/COLOR], I wanted to know if there was something I could do to prevent my ISP from harvesting my internet data for their advertisers.

SSH will encrypt the data stream but not all sites will accept an encrypted connection.  For instance [https://ubuntuforums.org/](https://ubuntuforums.org/) will not work.

This is good that you are aware of your privacy.  I just skimmed the article, but I'll give some recommendations.  I'll start with some privacy related Firefox extensions that I use: [Adblock Plus]("http://adblockplus.org/"), [CookieCuller]("http://cookieculler.mozdev.org/"), [CustomizeGoogle]("http://www.customizegoogle.com/"), [NoScript]("http://noscript.net/"), [RefControl]("http://www.stardrifter.org/refcontrol/"), [SafeCache]("http://www.safecache.com/"), [SafeHistory]("http://www.safehistory.com/"), [SwitchProxy Tool]("http://mozmonkey.com/"), and [Torbutton]("http://freehaven.net/~squires/torbutton/").

Another useful tool is a SSH tunnel.  A SSH tunnel will encrypt all data between your machine and a proxy machine you are connecting to.  This will make the web site you are visiting think that you are actually the proxy machine and it will log the proxy machine IP address and not yours.  I use this for public wireless connections to my home cable router (wrt54gl with DD-WRT) when I don't want other wireless users snooping my connection.  However, depending on your browser configuration, the DNS requests may go through the ISP and not through the proxy, so keep that in mind.  When using an SSH tunnel with Firefox, open about:config and set:
```
network.proxy.socks_remote_dns = true
```

This will tell Firefox to resolve DNS requests through the tunnel instead of on the local network where other LAN users may be able to see what you're doing:
[URL="http://wiki.freaks-unidos.net/weblogs/azul/firefox-ssh-tunnel"]
Tunneling Firefox traffic over SSH[/URL]
[Encrypt your web browsing session (with an SSH SOCKS proxy)]("http://lifehacker.com/software/ssh/geek-to-live--encrypt-your-web-browsing-session-with-an-ssh-socks-proxy-237227.php")
[How can I secure my computer when using public wifi?]("http://ubuntuforums.org/showthread.php?t=712919")

You can also run your own DNS server if you don't trust your ISP or a third-party DNS provider such as opendns.  My ISP for example will reroute requests to non-existent domains to an ad heavy "search portal".  There is no way to opt out besides using a different DNS server.
