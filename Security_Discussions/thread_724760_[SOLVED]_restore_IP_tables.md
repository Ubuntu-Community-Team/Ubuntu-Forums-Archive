### Post by p_quarles on 2008-03-15
Easy enough. First, save the rules to a file:
```
sudo sh -c "iptables-save > /etc/iptables.up.rules"
```
Then tell the system to load it on startup:
```
sudo nano /etc/network/interfaces
```
Find the line for your main network interface, and include the following new line:
```
iface eth0 inet dhcp
   **pre-up iptables-restore < /etc/iptables.up.rules**
```
Save the file, and you're set.

### Post by p_quarles on 2008-03-15
Definitely don't replace the loopback device. 

As for adding those lines, yes, that's what I would do. Since your main network interface is apparently being managed at the GUI level, though, this could potentially cause problems (I'm honestly not sure). So, make sure that you have a backup copy of the interfaces file that you can easily restore in case of any problems.

### Post by The Cog on 2008-03-17
If you are having to undo the fireall rules every time you boot, it would make sense to try and find out where the script is that sets the rules every time you boot, and remove that script. Try this:
**grep iptables /etc/***
and see if there are any scripts in /etc that do iptables things. another possible hiding place is
**grep iptables /etc/init.d/***

### Post by The Cog on 2008-03-18
So /etc/firewall is the script that's installing all the rules you don't want any more. It's almost certainly being run from /etc/init.d/guarddog. You can safely delete these files if you have deleted guarddog. You can also safely delete the link to /etc/init.d/guarddog that lives in /etc/rc2.d/*guarddog.
