### Post by David D. on 2014-02-20
If you install the daily build of 14.04 and keep up the updates (either with Update Manager or the command line), you will have 14.04 LTS when it comes out in April.

Understand that it is still in development, so there will be many updates between now and the final release in April.  Also, as you stated, there may be some problems along the way.

### Post by Bucky Ball on 2014-02-20

If you're wanting a reliable machine for daily use, don't go there. If you are up for testing, don't mind unexpected breakages and have the know-how to troubleshoot, discuss and report these things with community, go there. 

I would strongly advise you go with 12.04 LTS or 13.10, both of which are directly upgradeable to 14.04 LTS when it comes out in April, and if you want to help the community by testing and reporting in ***Ubuntu+1***, install 14.04 on another partition. (Incidentally, any and all support requests regarding 14.04 should be posted in ***Ubuntu+1***.)

Keep it updated and you will have the released version in April, as mentioned.

PS: You might want to have a look [HERE]("http://ubuntuforums.org/forumdisplay.php?f=427") to get a feel for how it rolls. The daily build is pretty much bleeding edge so you are face first to the wind. ;)

### Post by buzzingrobot on 2014-02-21
Development releases follow something of an arc, more or less.  Early on, they're probably pretty stable since they vary little from the previous release. Late in the process, after a beta or two, things are pretty stable.  It's the large middle part where things can get interesting. 

If you are interested in using a development version, and can afford to risk breaks and re-installs, I say dive right in.  If not, go with 13.10 or the current 12.04.04 version of the last LTS for the next couple of months.

### Post by grahammechanical on 2014-02-21
If you do install Trusty Tahr and update you may get a dialog offering a Partial Upgrade. Do not accept the offer. Just click Continue and the update/upgrade will take place as normal.

Those of us who run the development branch as a habit take at least two precautions. 1) have a stable release as a fall back OS. 2) keep data and Docs on a separate /home partition at the very least. Then when the worse happens we can re-install without the risk of loosing data.

Regards.
