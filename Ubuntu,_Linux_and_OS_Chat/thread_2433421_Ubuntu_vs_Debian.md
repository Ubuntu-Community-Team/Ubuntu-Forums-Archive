### Post by guiverc on 2019-12-19
I've used debian far longer (1998 or 1999) than Ubuntu (Ubuntu hadn't started back then), but for the most part I see Ubuntu as an easier Debian.

This box runs Lubuntu 20.04 (it also has GNOME (ubuntu-desktop), XFCE (xubuntu-desktop) & MATE (ubuntu-mate-desktop) installed); and thus allows me to compare with a debian bullseye/sid box (which has the same desktops & more).  The Ubuntu box has newer software generally than the debian box (though not always; some are ~identical).  For newer software I'd say Ubuntu wins though.

As I don't mind the bloat of multiple desktops on my systems (prior paragraph), I'll note I have far less problems with multiple desktops on debian when compared to Ubuntu, so if this matters - debian is a win here.  (I doubt it'd matter to many; but I gave up trying to have KDE (kubuntu-desktop) on this system, yet no problems with the debian bullseye/sid, which also took...)

If you're only using a LTS release - debian usually releases in odd years (less predictable; released when "it's ready") where as Ubuntu releases it's LTS in April of the even year.  So for corporate planning - Ubuntu wins.

Debian LTS support is somewhat similiar (~5 years) to Ubuntu.  Ubuntu flavors only get 3 years of support; however most packages used by desktops don't get updated once Debian reaches it's LTS support anyway (this could be just my opinion reading debian LTS posts on planet debian & I could be missing something as I generally only glance at it).  I'll say equal here.  (*If you include Ubuntu ESM - Ubuntu has an advantage*)

Whichever you're happiest with is what I'd go for. Each has pro's & con's.   I mostly see them as variations of the same thing, and my actual use will dictate which I'll install.  For desktops I usually go with Ubuntu, servers possibly debian.

### Post by TheFu on 2019-12-20
> **j2ee said:**
> Which one is the best and why?

It depends.

What is the purpose of the system?
Are there things that Canonical is doing which you dislike?
Are there things that Debian is doing which you dislike?

It depends.

When I need a tiny, server, for 1 purpose, it will be Debian.

When I need a bloated Desktop with lots of "cheese" pre-18.04, it will be Ubuntu. 

I haven't decided what I will do for 18.04 and later releases.  Snap packages, as they are current deployed, are an abomination IMHO. So far, I've been able to avoid them completely.  Canonical is really pushing them for a very few good reasons, and many, many, more not-good reasons.  People break-up over a difference of opinion all the time.  If 20.04 forced snaps like 19.10 does, I'm gone as much as possible.  A few of my servers might be stuck running Ubuntu Server because that is the OS the project team prefers. Some others run CentOS for a similar reason.

There are political considerations for some users too.  I've met a few who don't want to use **any** proprietary code with proprietary licenses.  Most of the time, they have to give in if they want wifi to work.  Only one of those friends refuses to use wifi.  I much prefer F/LOSS licenses, but will stray for a few needs.  Then there are users who don't care about licenses at all and just want a system that is easy, works, doesn't crash.  We each have our limits and priorities for what is important.

It depends.
