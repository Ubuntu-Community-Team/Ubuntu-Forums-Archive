### Post by QIII on 2017-07-03
Are you running RHEL 6.x or CentOS with the 2.6.32 kernel?  That is the supposed target.

Most reports about this use the words "apparent" and "alleged."  But there is no reason to believe that State-level agencies with the resources of nations behind them *can't* hack pretty much whatever they have a mind to.  Neither whether they *should* have them nor what their *intent* for having them are discussions we will allow on the Ubuntu Forums.

If and when something becomes a danger in criminal hands, as with WannaCry, etc., then we can discuss it in terms of the threat it poses in normal use and how to mitigate that threat.

That is the myth of "security by obscurity."  I'm quite sure that both State agents and criminals know how lucrative Linux targets are.  Don't fool yourself into thinking Linux is somehow "safe".  It's not.  A lot of the hackers doing this stuff are using Linux themselves.  Those "millions of eyes continuously reviewing the code looking for problems"?  Half of them are probably hackers.

### Post by mörgæs on 2017-07-03
It only takes one couple of eyes out of the million to create a bug fix and that is what happens: Every time a major security bug is discovered, being through Wikileaks or otherwise, a fix is released shortly. This makes GNU/Linux a more secure (though not bug-free) operative system than others. 

I haven't seen anything in the [security notices]("https://www.ubuntu.com/usn/") regarding this bug so it only seems to be relevant for RHEL/CentOS running the kernel mentioned. Some bug fix or code rewrite back in the 2.6.32 days probably solved the problem.

General advice: Just apply security updates on a daily basis and you have a good foundation.
