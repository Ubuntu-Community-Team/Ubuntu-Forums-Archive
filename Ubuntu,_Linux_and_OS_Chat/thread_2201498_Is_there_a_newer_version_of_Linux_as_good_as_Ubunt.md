### Post by buzzingrobot on 2014-01-24
Yes, it's telling that XP is also slow and sluggish on that hardware. Time may have caught up with it.

Many HP printers need a proprietary HP driver. Installing and running "hplip-gui" from the repos will locate and install a driver, if still available from HP.

The OP should understand that 10.04 was based on Gnome 2, which was not contolled by Canonical, and which was abandoned about 3 years ago. Whether or not Unity is a "smartphone" interface that doesn't work is very debatable (I think it's a very usable desktop). But, non-desktop devices will clearly dominate in the very near future so any vendor, like Canonical, would be fooolhardy to ignore them.  Porting Gnome 2 to that environment is a nonstarter as it, a spin of the Win95/98 interface, would be unusable there.

Gnome 2 remains the default desktop on CentOS, currently at 6.5. Essentially a rebuild of Red Hat Enterprise Linux, it has a well-deserved reputation for stability and reliability.  However, those attributes come at the price of 2010-era libraries and programs. With care, some current apps and multimedia support can be added from independent sources. Anyone considering CentOS as a desktop distro should first do some research. (i used it for a long time, but the aging GTK it uses eventually became a headache.)

### Post by buzzingrobot on 2014-01-26
hplip-gui is in the repos, installable with Synaptic or Software Center, etc. The HP drivers are not.  

hplip-gui is a program from HP that identifies the printer and then downloads and installs the appropriate driver. I.e., it automates what you do manually at hplipopensource.com.
