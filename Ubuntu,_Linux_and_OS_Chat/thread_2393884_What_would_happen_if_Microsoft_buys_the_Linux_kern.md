Let me put your mind at little more at ease if I may...

First, let's actually look at Github. A big difference between Github and the Linux kernel is how they are managed. The Linux Kernel is managed by a foundation, where as Github is managed by a company. From my understanding Github has not been that profitable which is what put Microsoft in a position to afford the platform.

The Linux Kernel however has a very well funded foundation backed by very large companies with an interest in a open kernel development. Some of these companies include but not limited too Cisco, IBM, Samsung, Intel and Oracle to name a few, these are all platinum members of the Linux Foundation. The benefit of being a platinum member to the foundation is a seat on the board, which means all these companies have a say in large decision like whether the foundation needs a new parent owner or who that owner may be. Not to mention that the kernel has something like over 15 million lines of code contributed by god knows how many people from who knows where that would have to be re-licensed.

Here is the current list of companies who are apart of the foundation...

[https://www.linuxfoundation.org/membership/members/](https://www.linuxfoundation.org/membership/members/)

Do have to say though, after looking through the list I have to admit I am little disappointed in RedHat for only being a Silver Member, especially since they have been crushing their financial expectations as a company for the last two years. Figured that would at least be a Gold Member with Suse...

### Post by lostmoonofsaturn on 2018-06-13
Important to note that Github has been unprofitable and had been seeking a buyer. Microsoft devs, in aggregate, it seems, are the biggest users of Github. While anything can change in the long run, I read this Github acquisition as in line with Microsoft adding the WSL Linux layer to Windows:  An effort to keep Windows developers developing products for Windows by assuring those devs can use the open source tools they want to use without running Linux rather than Windows. And remember that Microsoft, like everyone else, has placed its desktop product in unspoken legacy mode and sees its future resting on Azure and other cloudy products where easy integration of Microsoft and open source pays off big for the corporation. This is an acknowledgement that open source is not going away.
