### Post by SeijiSensei on 2014-04-11
For someone new to Linux, I'd stick to either Ubuntu or Fedora.  Since you're already here, try Ubuntu first.  In addition, I'd try a couple of different "flavors" of Ubuntu besides just the stock installation with the Unity desktop.  [Kubuntu]("http://www.kubuntu.org/") uses a different desktop environment called KDE which may look more familiar to Windows users.  The other two main flavors, [Lubuntu]("http://lubuntu.net/") and [Xubuntu]("http://xubuntu.org/"), work well on machines with weaker hardware as well as the most modern machines.

You can "test drive" a distribution by downloading and creating a bootable device, either a CD/DVD or a USB drive if your computer supports booting from USB.  Every Ubuntu version allows you to pick "Try Ubuntu" after booting which will leave your computer's hard drive unaltered while you browse around in Linux.  You'll find that it's rather slow running this way; rest assured that it will run a whole lot faster when installed to your computer's hard drive.

In my case experience and patience were key.  I've been using Linux since 1995, originally on gateways and servers, but now on the desktop as well.

### Post by RadicaX on 2014-04-11
Hello, Welcome to Linux! Really, We all would have different stories of how we got here, and our motivations (Mine involves me being a kid on a now defuct Megaman Forums). I read about it, I looked up things about it, tried it, and now I use it regularlly, especially when the Windows Vista OS I had completely killed out on my computer, that made me change quickly for need of a computer. I will now do my best to answer some of the other questions. 

I would Recommend Bash as a Language you learn, and using the Terminal, as you learn commands, and you come up to a problem, you will then have to problem solve. A good example of when I was starting out, was that I needed to find an .rtf file somewhere in my documents, the problem is it was so full of different document types that it would take forever to just go through the list with the ls command alone. (which list all the things in that directory). So stopping, and thinking about it, I remembered learning about the Linux wild card command, and used it with ls to find what I was looking for much quicker. (I had no clue the name of the file, only the extension). 

```
ls *.rtf
```

That little code right there displayed only the files that ended with .rtf, which helped me to find the file much faster. 

Feel free to learn any programming language though, python is powerful and easy to learn, C is another powerful one. And as you learn commands, and logic, you then problem solve. Reading around these forums, and watching how others solve common problems is a good way to learn on that.

Feel free to Use any Linux you want, some force you to learn certain things faster you could say, especially with the need to get it up and working. (Any Linux without something like the Software Center, or Synaptic), because you have to learn the local language and use terminal most of the time to get it. (Not quite as hard as it sounds, I will still use Apt-get on here for example), some Linux Distros use Apt-get, some use Yum, some use random other package managers. This does not often involve a lot of learning over and over, more like just revising some of your commands to do it a bit. Using something like Arch Linux, which you will have to build up, is a good way to learn the insides and such of your distro, and how it works. But it takes a lot of work, and a lot of reading. If you just want something that works, Ubuntu is fine, if you want to learn more, you can keep using Ubuntu, and if you really want to get down and dirty and learn a lot, feel free to try something like Linux from Scratch. 

Feel free to study and or learn programming on any distro, Debian/Ubuntu Based ones are pretty good for that.
Some come with more stuff in them already ready for programming, (Like IDEs and such already installed), but that is nothing you can not do on your Linux already. 

An OS, as you ask, is a little hard to define, because people do argue when something can be called an Operating system, and when it is not. I would say, Linux, is not an Operating System, but is a core part for it, an Operating System handles all the things between the computer, and the person, including programs, apps, and daemons. Every little bit of that. Different Operating systems, are built differently, handle programs differently, and act differently. While if you strip everything away from Fedora, and Ubuntu, you have the same thing under the hood, that does not mean things made for one, will work on the other, without extreme modifications. (Fedora for example, packages things with RPM, while Ubuntu uses .deb Packages, something made for fedora will not work for Ubuntu, you would have to recode it for it to work).

There are different sites you can use to read about the subjects. I will give some links to some I use.

[http://linux.oneandoneis2.org/LNW.htm](http://linux.oneandoneis2.org/LNW.htm)

An interesting Article on how Linux Is NOT Windows.

[http://linuxcommand.org/index.php](http://linuxcommand.org/index.php)

Great sight for learning Shell commands, and even some programming later on. 

[http://mywiki.wooledge.org/BashGuide](http://mywiki.wooledge.org/BashGuide)

A site highly recommended for learning Bash, I liked the second one a bit more for learning the commands. Many people use the third one.

[https://help.ubuntu.com/community/CommandLineResources](https://help.ubuntu.com/community/CommandLineResources)

A whole list for learn the Command Line, that is the Terminal.

Now really, you can live most of your Linux life without going into Terminal too much, but if you learn it, it makes your life so much easier, and allows you to use your computer if your Gui for whatever reason fails. Setting up the ufw firewall is a breeze in the commandline for example, and saves you from having to deal with a GUI. But if you are dead set on pointing and clicking everything, Ubuntu and things like it, are more for you. 

Anyway, I hope this helps you out, be sure just to look around the forums and feel free to ask questions like you just did! You will see different people feel different about somethings (some like to use an AV, some do not like to use an AV, some like Unity, some like XFCE like me).
