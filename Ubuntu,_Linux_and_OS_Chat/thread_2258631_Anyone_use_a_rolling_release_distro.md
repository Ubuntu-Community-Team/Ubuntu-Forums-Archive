### Post by malspa on 2014-12-29
I'm running Arch. In the past, I've run PCLinuxOS and Sabayon. I like Arch better than those two; package management with pacman is great, and I haven't run into any major problems so far (after a little over one year). You just have to keep an eye on the announcements at Arch's main page and follow the instructions there, if applicable. The Arch Wiki is superb.

I'm also running a couple of Arch derivatives -- Bridge Linux (since around Feb 2013) and ArchBang (since around June 2013). Those two have worked out just as well as Arch for me, no major issues yet. With both, it's about the same as running Arch, once installed, but I think ArchBang is closer to "pure" Arch than Bridge is. Just don't go to the Arch forums expecting help with any Arch derivatives! The derivatives have their own forums... Most of the info in the Arch Wiki will still apply for the derivatives, though.

I'm not gonna make any recommendations because each user is different. I'll just say that an Arch derivative is nice for if you want to get a feel for pacman, etc., but in the end I think it's worth it to take the time to install "straight" Arch. For info about NVIDIA in Arch, see: [https://wiki.archlinux.org/index.php/NVIDIA](https://wiki.archlinux.org/index.php/NVIDIA)

### Post by TheFu on 2014-12-29
Something breaks after every patch.  They figure it will be fixed quickly, so no big deal ... until it is and you can't work for a week until the next update.  It has been a few years since I bothered with arch, so things may have changed.  I only run LTS releases now. I did my beta/alpha testing time with Linux in the mid-1990s. Don't feel the need to keep doing it now.

Of course, things that break often aren't of any concern too, but we don't usually notice those.

I stopped using proprietary video drivers except on an XBMC machine years ago.  There were just too many issues.  On the XBMC box, it has an ATI GPU - I only patch it quarterly and expect to spend 6 hrs to fix any issues that arise.  After all, the [WAF]("http://en.wikipedia.org/wiki/Wife_acceptance_factor") for the primary movie/TV watching box needs to remain high.

Rolling releases are bad for most people who just want a system to work.  They are especially bad for developers who force their code to be bleeding edge and leave all their users behind.  Developers that are not core for linux kernel or Gnome should be 2 releases behind, IHMO. That is where most clients are.
