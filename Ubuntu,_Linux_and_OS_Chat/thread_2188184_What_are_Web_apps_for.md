### Post by Copper Bezel on 2013-11-16
I have Gmail and Play Music as webapps. I use Chrome, not Firefox, but Chrome has the functionality built in.

It's a bookmark per a sense, but when the web app opens in its own separate window, has its own launcher icon and the same separate icon in the Alt+Tab switcher, and clears out the extra chrome of the location bar, it's almost like fighting back the "everything is in the browser" tendency and making the desktop useful again. If I used a local mail client, I wouldn't want it weirdly squished into my browser.

Chrome uses nasty low-res favicons for the web apps in the dock, but they're simple enough to change out for higher-res alternatives. Because, you know, Linux. (And I think this can be done in Windows, too - I never bothered to check the last time I had a working Windows install.)

I hate this idea. I use Chrome on my Nexus 7, and it's as straight a port of the desktop browser as possible. Firefox for Android is a phone-centric browser instead, with no a conventional tab bar, a hamstrung omnibar, and limited features - and it's very frustrating on a 7" tablet. While you're right on the main topic here - webapps are one way of separating services out from the browser on a mobile device, like, say, Google Maps, that really ought to be native apps - I do find this perception of what a mobile browser should and shouldn't be able to do extremely misguided.
