Your installed Ubuntu System does not register itself with any central server, so total usage of each release is unknown and unknowable. Several widely varied estimates exist.
See [http://popcon.ubuntu.com](http://popcon.ubuntu.com)
See [https://errors.ubuntu.com](https://errors.ubuntu.com)
See [https://insights.ubuntu.com/2015/12/22/more-people-use-ubuntu-than-anyone-actually-knows/](https://insights.ubuntu.com/2015/12/22/more-people-use-ubuntu-than-anyone-actually-knows/)

The best choice for a new Desktop user of Ubuntu should not be based upon any of those. Number of users does not imply 'better'.
Desktop users who want the latest stable versions of software should use the latest release of Ubuntu (15.10).
Desktop users who are annoyed by the constant changes of updating software should use the LTS release of Ubuntu (14.04).

And, frankly, if you install the wrong release of Ubuntu...so what? Reinstalling is free and not difficult.

### Post by bashiergui on 2016-04-01
Agreed: it's unknowable. Thought exercise - how could you try to count? You couldn't do a masscan of the internet to find out because that would only find Ubuntu computers facing the internet. And desktop users typically don't.

You could count the unique IPs that pull updates from canonical, but that wouldn't count how many computers were behind a NAT. Plus it would count the same laptop many times if they're moving around a lot. And people getting DHCP from their ISPs would get counted multiple times. 

You could count how many requests canonical gets for core packages, but that doesn't differentiate between desktops and servers.

You can't count how many times iso's are downloaded because 1 could be installed on 0 or a million computers.

Can you please tell me how can I do that?
I found the script for PPAs but does not work for core packages.
```
#!/usr/bin/python
#usage python ppastats.py PPATEAM (ex: webupd8team) PPA (ex: gthumb) DIST (Ubuntu version eg maverick) ARCH (ubuntu arch eg i386 or amd64)

#example - highest downloaded file: python ppastats.py webupd8team y-ppa-manager maverick amd64 | tr '\t' ',' | cut -d ',' -f3 | sort -gr

import sys
from launchpadlib.launchpad import Launchpad

def stats(distro, arch):
    PPAOWNER = 'mc3man'
    # PPA = 'ffmpeg-test1'
    PPA = 'mpv-tests-'
    desired_dist_and_arch = 'https://api.launchpad.net/devel/ubuntu/' + distro + '/' + arch

    cachedir = "~/.launchpadlib/cache/"
    lp_ = Launchpad.login_anonymously('ppastats', 'edge', cachedir, version='devel')
    owner = lp_.people[PPAOWNER]
    archive = owner.getPPAByName(name=PPA)

    for individualarchive in archive.getPublishedBinaries(status = 'Published', distro_arch_series = desired_dist_and_arch):
        print "%(distro)s\t%(arch)s\t%(package)s\t%(version)s\t%(download)s" % {'distro': distro,
               'arch': arch,
               'package': individualarchive.binary_package_name,
               'version': individualarchive.binary_package_version,
               'download': str(individualarchive.getDownloadCount())}

def print_distros(distro):
    archs = ['amd64']
    for arch in archs:
        stats(distro, arch)

distros = ['xenial', 'wily']
for distro in distros:
    print_distros(distro)


```
As you know some packages uses only for desktop so maybe can retrieve a rough estimate!
