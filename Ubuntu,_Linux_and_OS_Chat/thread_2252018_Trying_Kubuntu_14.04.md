### Post by mastablasta on 2014-11-10
> **sports fan Matt said:**
> Besides all the KDE packages and a different look and feel, are there any major things I should look out for that are widely different from Gnome as a desktop?

the difference is in basic principle/philosophy of the desktop environment. With Gnome the idea is not to overwhelm the user with various data, settings etc, so they would usually give only the most important things in the GUI. white the rest is setup via terminal or config file. while with KDE the idea is to offer as much configuration as possible to the user via GUI. around this there are also some good programs that are based on KDE. there is even a whole new office suite (still under heavy development).

there is also journaling and desktop search, so upon fresh install KDE will do a disk scan. this can take time. it used to be quite bad but they fixed many of that issue. now it will make computer slow for about an hour or two after install after that it doesn't really hog the resources as much as it did before. I have it installed on an old Celeron dual core with 1.25 GB ram. it's very responsive. even with default desktop special effects turned on.
