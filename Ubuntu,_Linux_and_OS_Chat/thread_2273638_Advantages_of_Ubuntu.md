I'll suggest going slow and getting your feet wet before jumping into trying to play Windows games on Linux. When you add tools like Wine/PlayOnLinux you are essentially adding on complexity in an effort to get 'foreign' incompatible software to work.  Sometimes it will, sometimes it won't, and often a bit of both. Windows software is intended to run on Windows.

Video drivers from AMD and Nvidia for Linux are released without source code.  While they are usually needed to get adequate game performance, their use can be problematic for various reasons. The safest way to install them on Ubuntu is to use the "Additional Drivers" tool.  (It's also the safest and easiest way to remove them and rollback to an earlier driver.  These proprietary drivers need to be completely and cleanly removed before trying to install another version, but manually removing them is error prone.)

Do not immediately jump to the very latest proprietary drivers as soon as they are released unless your current driver is not working correctly and you know that your specific problem is fixed by the new release *and* you know that it works with the Linux kernel version on your system. Any number of Ubuntu-related sites and blogs routinely recycle how-to posts that briefly outline how to install just-released drivers from various unofficial and unsupported sources.  Be cautious.

### Post by buzzingrobot on 2015-04-15
> **rich1912 said:**
> 

Now in order for Ubuntu to be a viable option for me it must be able to do everything Windows could do without too much headache. This has not been the case. For example I was trying to install Diablo 3..... Linux/Ubuntu need to provide this support more effectively if they want people to consider switching to their OS.

Programs written for Windows cannot launch and execute on Linux, and vice versa. They are two different operating systems.

That you can execute *any* Windows software on Linux is due to the diligence of some developers who have created tools that, in effect, intercept requests made by Windows software to the Windows OS and reroute some of them to equivalent Linux facilities. Oversimplifying, these tools create an isolated environment that tries to fool some programs into thinking they are running on Windows.

No amount of "support" from Linux developers can make Windows software work seamlessly and natively on Linux.  That's down to the software vendors.  If you want Windows software to run on Linux, then the developers of that software need to rewrite it to run on Linux. That is, they need to make Linux software.

If you are committed to using Windows software, then stay with Windows. Linux is *not* trying to be a direct replacement for Windows.
