### Post by TheFu on 2018-02-24
Seems there are lots of misconceptions.

The GUI on any Linux is just another application.  They are easy to replace or have 10 of them loaded.  Your choice.  Linux is the king of customization too.  You can do anything you have the time, knowledge and desire to make happen.  The code is there to modify, but many options are managed by a config file or DB.

I find that most people who claim to want to learn and customize Linux really don't want to do that.  They really just want to find a pre-configured variant that works good enough for them.  Assuming you are different and really want to learn as much about Linux as possible, then you can go with gentoo or Linux from scratch.  The learning curve is steep, but you WILL LEARN. No choice with those distros.

Canonical is pushing commercial offerings for servers, cloud, and infrastructure, not on the desktop.  It is important that Canonical be profitable to be able to keep doing all the things they do for all us moochers.

If you want to stay with Ubuntu, and there are many great reasons to do that, and you want to limit bloat, then start with a minimal Ubuntu install (no GUI) and build up what you want.  I start with Ubuntu Server and add the 15 GUI programs and a window manager to my systems.  Since I run Ubuntu Servers, having the default server tools on my desktop is good, without all the extra stuff like network manager or office suites or video editing or photo editing automatically installed.  I've learned which packages I like and which I don't.  A "desktop distro" is more for the grandmas or people who just want everything preinstalled for them so it is already in the menu.  That isn't me. Perhaps it isn't you either? 

I've been using nearly the same custom setups for my desktop for 10 yrs.  I have some custom settings from the mid-1990s that work still as well, just with a different WM - fvwm.

You can customize pretty much everything on a Linux system.  Worst case, it just takes time to learn how. That really is the only limitation.  No need to put up with a GUI that Steve or Bill say you should have and like it. With Linux, YOU can make your own GUI, if you like.  There's a group that made their own in about 1200 lines of code, so you certainly can do that too.  [https://suckless.org/](https://suckless.org/)
