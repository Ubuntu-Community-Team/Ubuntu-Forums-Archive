### Post by d-cosner on 2014-05-10
A week ago I installed Ubuntu Gnome 14.04 x64, I had been hearing some good things about Gnome 3.10 and wanted to try it out. Previously I was running Ubuntu 14.04 x64 with Unity and I liked it. I wanted to see the differences between Gnome Shell and Unity, they are very different! Gnome Shell is slightly faster on my hardware and I do mean slightly. The applications menu populates a little faster than the Unity applications lens.

Application launch and everything else seems pretty equal between the two. Gnome Shell always reminds you that you have multiple desktops available, I like that because I tend to use them because of it. Gnome Shell is very customizable, more so than I had ever imagined! Between the provided Gnome Tweak tool and all of the extensions that are available you can make the system just about any way you could possibly imagine!

Unity also has some tweak tools available too and you can use CCSM to tinker with things too. I find Gnome Shell to be the more customizable of the two though because you can customize it all the way to the point of having a traditional desktop layout if you so desire. Ubuntu Gnome also provides a Gnome Classic session by default, you simply select it at the GDM log on screen.

I went through the 14.04 development cycle and I know that a great amount of work and testing was done in order to provide the Gnome Classic session. If you miss the look and feel of Gnome 2x the Gnome Classic session provides that look and feel with a few modern touches! Both Ubuntu and Ubuntu Gnome are very refined releases, all of the LTS releases are for that matter!

I like the way Unity and Gnome Shell work, the look and feel of both of these desktop environments is great! I have been favoring using Ubuntu Gnome lately because I like the way I can get work done with it, it gets the desktop out of my way when I am trying to get things done and it reminds me to make use of multiple desktops. I use the launcher in both Unity and Gnome Shell for the applications I use every day but I like how Gnome shell uses pages now in the applications menu.

Using pages instead of scrolling through my applications is a minor thing but I really like it! Both Unity and Gnome Shell use compositing but Gnome Shell has a little more bling to it in my opinion. Memory and processor use is pretty similar between Unity and Gnome Shell, for full featured desktop environments both are reasonable as far as resource use. 

I installed the new Gnome Weather and Maps applications, both are very nice! I should mention though that these apps have no icons when installed under Unity and they do not show up as recently used in the applications lens... All of the complaints about Gnome Shell in the past seem unfounded now. Gnome Shell looks more like a finished product and it is extremely customizable!

### Post by monkeybrain20122 on 2014-05-10
I have been using gnome shell in Fedora since 3.4 or 3.6, now it is 3.10 on Fedora 20 (dual boot with ubuntu). While I prefer GS over all the 'classic' desktops, I find Unity comes out on top.

> **d-cosner said:**
> The applications menu populates a little faster than the Unity applications lens.
It only populates faster at first launch in a session, afterwards, no.But it also does a lot less, gs's dash doesn't search or log files.

> Application launch and everything else seems pretty equal between the two. Gnome Shell always reminds you that you have multiple desktops available, I like that because I tend to use them because of it. Gnome Shell is very customizable, more so than I had ever imagined! Between the provided Gnome Tweak tool and all of the extensions that are available you can make the system just about any way you could possibly imagine!

I find it actually less customizable. 

In gs I feel more restricted because I cannot use the edge and corner to switch and expose workplaces, everything has to be triggered in the upper left corner, which is quite awkward for multiple desktops since they are exposed on the right edge. Also having gotten used to Unity's multiple desktops arranged in a 2d array  so I can reach another workplace in any of the four directions the way that GS stacks up the virtual desktops vertically feels constrictive.

launcher icons get smaller and smaller if you pin more applications to  g's dock, unlike Unity's accordion dock which adjusts by "folding up"  the icons. I also  don't think you can change the size of gs's dock. 


> Using pages instead of scrolling through my applications is a minor thing but I really like it! Both Unity and Gnome Shell use compositing but Gnome Shell has a little more bling to it in my opinion. Memory and processor use is pretty similar between Unity and Gnome Shell, for full featured desktop environments both are reasonable as far as resource use. 

I doubt that it can beat compiz when it comes to bling.

Gnome Shell is only customizable with extensions, but the extensions that work for one release often stop working for the next, many of them become deprecated because the devs have lost interest etc. I used to use an extension for 'edge flipping' between workplaces but it doesn't work for gs 3.10 anymore

I also find animation to be a lot smoother with Compiz than with gnome shell. e,g application switcher (in gs used an extension coverflow altab, in unity static switcher in ccsm) In gs no transparent windows (there is an extension but badly implemented) and its version of wobbly windows (via extension) tears so badly that it is unusable (regardless of graphic cards, tried it on Intel, AMD and Nvidia machines)

Finally, what really annoys me is that when I click something on the gs dock there is no visual cue of any kind that it is launching (in Unity icon will flash or bounce) or just dead in the water.
