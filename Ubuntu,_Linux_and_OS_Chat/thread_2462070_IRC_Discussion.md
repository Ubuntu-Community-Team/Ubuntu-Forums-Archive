 A couple of years back [I added]("https://wiki.ubuntu.com/IRC/ChannelList/Support?action=info") *Web Links* for support channels... those links could always be added beyond just support...  ie. on page [https://wiki.ubuntu.com/IRC/ChannelList](https://wiki.ubuntu.com/IRC/ChannelList) (*though it's in sections so there are sub-pages*)

---

I do love the way IRC is linked with other *mediums*, in fact a lot of Lubuntu IRC traffic is coming from matrix or telegram  .. the *younger*/*newer* users seem to prefer the newer methods, which allows easy reading replies later whilst *offline* (no  IRC bouncers or irclogs required). To have reliable bridges between the various options would be great (*they tend to go down a lot*)... but personally I'm mostly in *flavor* rooms like Lubuntu which may not be covered by Canonical IS (and what you're asking about @Madhens), and for the Ubuntu rooms I'm in, everyone is on IRC anyway so it doesn't matter there.

### Post by linuxyogi on 2021-05-24
If I want to try a different distro other than Ubuntu the first thing I do I login in to Freenode and visit that distro's IRC channel. I do this to check 2 things, firstly how many users are present which gives me an idea about the size of the community & the second thing is how helpful the community is which I try to find out by asking some basic questions. 

Recently I tried GhostBSD which is based of FreeBSD. IRC is not dead. Just visit #ubuntu on Freenode and see how huge the userbase is. The only problem that I face is timing. When I am awake (in India) almost everyone is sleeping in USA but still I get a descent number of replies.
