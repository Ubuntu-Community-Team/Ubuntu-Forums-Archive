### Post by jadaube2 on 2022-01-26
The LiveCD simply didn't  work in any of my computers. One computer was ridiculously slow and the other two crashed after the splash screen. I kept getting a number of errors including "failed to unmount cdrom" and "failed to start snap " in the CLI after the splash screen.  I verified the iso file reburned on different discs and in different  computers with the same result. Finally, I just imaged the iso to a thumb drive and it worked just fine.

Once I was satisfied with the live USB copy, I did a fresh install on an old Latitude E5430 with 16gb of ram and a new SanDisk ultra 1tb ssd. Went off withhout a hitch and I am very satisfied with the results. The os recognized and worked with both the printer and scanner functions on my Brother MFC-J885DW without needing to mess with CUPS or install additional  software. That was a first.

### Post by SeijiSensei on 2022-01-27
This is a post about Jammy, but I had the same experience trying to install Impish.

The time between the software selection and disk layout phases is extremely long, and there is no indication that the computer has not hung.  On one install it took five minutes for the disk layout page to appear.  That's way too long for the installer to provide no updates on its progress. First time through I thought there was a problem with it reading the disks and powered down the machine after waiting three or four minutes.  Tried again and just waited until the next step.  I was installing from the desktop icon, so I could open an terminal and run top to watch installer's progress.  It kept doing things so I knew the process wasn't hung.

I'd hate to be a novice in this situation.

Kubuntu 22.04
