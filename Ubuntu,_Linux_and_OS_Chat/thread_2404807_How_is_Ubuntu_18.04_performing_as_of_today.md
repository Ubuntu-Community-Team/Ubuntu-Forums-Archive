### Post by DuckHook on 2018-10-28
I have found Bionic to be stable and problem-free, but one person's anecdotal experience such as mine counts for very little. I will say that the bigger issue has proven to be some pretty big changes in how things are now done: Gnome took some getting used to after putting so much time into getting proficient with Unity. A lot of admin stuff was also nerfed by default in Bionic, like gparted, synaptic, etc. They can be re-enabled, but it's a pain. I also dodged Wayland and switched back to X11, although that is now default, so less bad. Other changes both major and minor—too long to list.

So my issue is not with the "bugs", but with the "features". However, now that I've made the switch, getting used to the new way is not actually that big a deal.

I can't speak to the 24/7 issue. All of my 24/7 machines are servers and they still run Xenial.

Probably haven't helped you all that much, but those are my 2 cents.

### Post by kansasnoob on 2018-10-29
I'm running Ubuntu 18.04 on two machines at home (as well as a few others strung about between friends and family) and I've encountered zero issues. The two at home were both upgraded from 16.04 in August. Something I always do before upgrading is actually purging (not just disabling) any PPA's in use so all packages revert to their natural state. I should probably mention that I was already a fan of the default GNOME desktop so I didn't have to deal with any Unity -> GNOME Shell learning curve.
