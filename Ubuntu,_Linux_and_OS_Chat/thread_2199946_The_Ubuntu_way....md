### Post by BBQdave on 2014-01-16
Unity and Gnome 3 are definitely a different DE flow from traditional environments, such as Gnome 2 or KDE. As you read up on these new flow designs, I'm sure you'll come across a lot of folks saying "give it time." And straight forward, that's about the best to get familiar with the DE flow and then decide if it works for you or not.

Gnome 3 uses expanding desktops and the Super Key for quick access to applications. The idea (my rough take), is that you are focussed on one application, while all others are in the background (or hidden). Pop-up notifications do a great job of information on applications not in focus. And the bottom notification bar holds more in depth information on applications not in focus. You can easily move around to applications with the Super Key or clicking on Activities (top left). Search is another strong component of Gnome 3's flow. Hit Super Key and start typing, usually what you want appears before you can finish typing the search query.

Unity is a little different with it's DE flow, but I think does a great job with Global Menus and Dash. The more time you spend using Dash, I think you will find it a powerful component of Unity. Dash can be tuned in System Settings > Privacy. And the left side Application Bar does a nice job of displaying information on running applications not in focus (similar to Gnome 3's pop-ups and notification bar). Pop-up notifications also are key in relaying information on applications not in focus.

Coming from Fedora Gnome 2 and Debian 6 Gnome 2, I was a little *shell shocked* with Gnome 3 and Unity. But giving the two time, I was able to understand the flow. My comfort zone is F20 Gnome 3, and I can't imagine using anything else (unless maybe Unity) :)

### Post by buzzingrobot on 2014-01-17
1.  Conflating the panel- and menu-based designs rolled out with Win95/98 with "established" design principles is wrong.  That approach is, obviously, familiar to many users, but only because of Windows' position in the market.  It's a bit like declaring that Twitter and Facebook set "established" principles of journalism. (Several Linux efforts mimic that Windows approach, at least partly in the expectation that imitating Windows will lure Windows users to Linux.  I think the numbers show it hasn't worked.)

2. The width of Unity's Launcher varies from 32 pixels to 64 pixels.  That does not merit the label of "HUGE".

3.  No computer interface is intuitive.  We should stop using that word. Interfaces must be learned.  Some are easier to learn than others, depending on an individual's previous experience. What we typically mean when we say "intuitive" is "familiar". If a user is unwilling or unable to explore an unfamiliar interface, that's fine, but it should be recognized as a personal and subjective choice and not an objective assessment of that design.
