### Post by Shibblet on 2019-01-24
I've been a distro-hopper since Ubuntu 8.04 came out.  I tried all the 'buntu's.  Currently, my distro is Kubuntu, and has been since 16.04.  We all know that choice of distro comes down to personal preference.  However, I am not going to get into all of the distro differences.  What I am going to discuss, is stigma that has come down on Linux in general, because of Windows.

I think that many Linux users call Linux OS's a "switch" from OS's like Windows.  In other words, the issues that face Linux as an OS, stem right from the beginning.  Anyone who is interested in running any Linux OS, immediately has problems with the fact that it is not Windows.  Whether it's differences in the interface, that it doesn't run all of the same programs, HD Partitioning, dual-booting, and the like.  They all seem problematic for the Windows "transitioner."

As a matter of fact, I have NEVER had any issues with running software in Linux, as long as it comes from the software store, or is compiled to run natively in Linux... I honestly cannot think of a single app or program that doesn't work directly from whichever repo you download your software from.  Fedora apps run perfectly on Fedora, OpenSuse apps work flawlessly on OpenSuse, and Ubuntu apps are no different.  They work GREAT in Ubuntu.

The only time I cannot get certain software to function, is when I am "trying" to make it work in Wine, or the like.  Windows software running on Linux has turned into the major problem.  As a matter of fact, this is the largest problem that seems to plague the Linux community.  We are doing everything we can to make Windows software run in Linux.

I think "We" as Linux users are going about this issue the wrong way.  This is where our focus is at.  I mean, don't get me wrong, projects like Wine, Proton, and SteamPlay are absolutely GREAT ways to assist end users and developers in getting their programs, media, and games to run in Linux.  But ultimately, it is still "us" as a community trying to get software made for another platform to work on "ours."  This is akin to making an iPhone app, and just trying to make it work natively on an Android phone.

There are even distros, such as ZorinOS (which is a great distro, btw) that try to make it's interface look like Windows 7.  Zorin even goes so far as to include Wine in the distro, so that users can run Windows Apps without having to learn how to use Wine.  And this is their "schtick."  They want to ease the user into Linux, from Windows.  Distro's like this are VERY useful to assist in transitions.  But ultimately, again, this is an entirely different OS, and shouldn't be expected to run software from other OS's.

Most of the Linux "tryouts" that I am aware of, end up complaining about lack of Windows compatibility.  It's a bit like complaining to the manager of Taco Bell that they don't have any cheeseburgers.  And not only that, but continuing to come to Taco Bell, and continuing to ask for cheeseburgers EVERY TIME YOU COME IN.

Linux is not Windows.  It was never intended to be Windows.  And I wonder if Windows compatibility is really the future of Linux. 

I always question solutions like "Proton" for SteamPlay.  It's a great idea for current software, but dioes it truly help developers for future titles.  If a game can be written for Vulkan to run on both Linux and Windows without having to change the API, why then would it take that much more time to make the game Linux native?

If cloud apps, like Adobe CS are available for both Mac and PC, why then is it so much more difficult to make it native for Linux?

What we need for Linux is support, and not necessarily compatibility with other OS's.

I'm hoping this becomes a good open discussion.

### Post by SagaciousKJB on 2019-01-30
> **freemedia2018 said:**
> Or if Linux compatibility is really the future of Windows.

You know honestly, I feel like that would have more utility.

I've been using Linux now as my sole operating system for a long time.  I don't really remember the last time I absolutely couldn't avoid a Windows application.  I think it was for something pretty quirky or niche, and I seem to remember just installing Windows 7 in a virtual machine to run it.

Now conversely, in all that time, I've gotten really used to the way things work on Linux, and any time I have to use a Windows OS I feel quite limited and like I don't know what's going on.  I think that's kind of interesting since when I transitioned away from Windows, they were in the XP->7 (well most users skipped Vista haha ) phase and that was a pretty straight-forward switch to me for most things.  Meanwhile, I look at Windows 10 and I'm kind of like, "Where the heck is anything?"

What I think holds Windows back is the graphical interface.  Yeah you know, that thing that was supposed to make using a computer so much easier.  Everything is point and click, point and click, over and over.  You gotta find a menu, find the new place they put the menu, etc. and so on.  It's not as if Linux interfaces haven't changed drastically over time, but because so much of the tasks needed to run a computer on Linux can be done from the command line, and because they don't change a lot of over time, I haven't really had to learn a lot of new ways to do things from one Linux to another.  I think the biggest change-up has been the switch to SystemD.  However, I found finding the difference between being told that "/etc/init.d/whatever start" was now "service whatever start" was a lot more succinct than being told how to navigate my way through an endless string of "This Menu->That Menu->This Option->That Option->This Box->That- Box" type mappings which you need to follow to do anything on Windows.

In any case, I think after years of doing things the "Linux way" of using commands, configuration files, etc. everything is a little more universal, save a few caveats here and there.  I think that's more a byproduct of GNU than Linux itself, but anyway...  Just trying to give credit where credit's due.

Point is when it comes to Windows, there really isn't the same kind of standard way to set things up.  Every software package has just a little bit different way of doing things, and it doesn't always play well together, and it's not always obvious.  It's not really ever very efficient either, to be honest.  I remember nightmarish scenarios with "Windows Networking" that I eventually just resorted to FileZilla as an FTP server fore file sharing.  

Meanwhile on Linux, you can use the far superior NFS with ease, and even Samba is more straight-forward to setup than Window's own filesharing setup. Oh and not to mention an SFTP server coming with basically every installation of openssh-server.  By comparison, Windows feels like some kind of movie-rental kiosk's operating system.
