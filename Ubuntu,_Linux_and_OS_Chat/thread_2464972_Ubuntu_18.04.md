### Post by guiverc on 2021-07-18
Yes, if you *distro hop* you'll learn more about your base GNU/Linux system.

I have here a number of boxes; on which I run

- **Ubuntu** (various flavors, this box is *impish* (will be 21.10 on release) though it dual boots to *focal*/20.04) and I'm using Lubuntu's LXQt desktop currently (my norm), though I've also got Ubuntu (GNOME) & Xubuntu (XFCE) which I can select at login; ie. to use another desktop you can install them on the same box & select at login which to use.  The other system on the box is likewise a multi-desktop system, but also has LXDE (what Lubuntu used in 18.04) & MATE (Ubuntu-MATE variety).

- **Debian** (bullseye, buster & stretch). I tend to use *testing* the most; using the other *stable* releases for specific purposes/reasons

- **Opensuse** (*tumbleweed*) - a RPM based system

- **Fedora** (*34 I think; whatever is latest*); another RPM based system

plus other Ubuntu systems (*and a CentOS I've not moved yet*)

**To me they are all pretty much the same**... All systems are built from upstream code; the code is the same for each though aren't always grabbing the code at the same time, so at rare times my debian *testing* is ahead of my Ubuntu *development* box.  The opensuse I have is often ahead (*given I'm using the rolling opensuse*)*, *Fedora rarely ahead (close to release time or if I bump to *rawhide* which I use on rare occasions it's different).

All the distros I've mentioned will run KDE. All can be used with multiple desktops (*in fact all mine have multiple desktops installed unless used as a server; box to my right has Lubuntu (LXQt) installed with Kubuntu (KDE) the second desktop; also Xubuntu (XFCE) as a third*..).  There are costs to having multiple desktops installed (*complexity, larger updates, ....*)

Anyway my point is; of those I use - **none stand out**. For specific roles I'd likely pick one over another (*like whether to use a rolling, stable, LTS etc release*) but it depends on use-case. To me the big thing with Fedora you mention is the RPM package management system, thus you'll be learning new commands to handle packages (*and upgrade it at least once per year*!)....

I'm most at home on Debian or Ubuntu - as I know the `apt` , `dpkg` & like tools best.  But I'd be struggling to think of anything I can do in one of them, that I cannot also do in others.
