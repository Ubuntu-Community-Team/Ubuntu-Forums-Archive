Welcome. Part right, part wrong. Yes, you can install another DE in Ubuntu, but you then have another DE with the Ubuntu default apps, bells and whistles. The idea of the other flavours is that they are complete units in their own right, with their own default apps which are not necessarily the same (especially Kubuntu). Because of this, Xubuntu (xfce4), Lubuntu (lxde) and Ubuntu Mate (mate) are all considerably lighter and take up less space than Ubuntu (and less resources). 

Just a word: if you do want to try another DE, don't install *ubuntu.desktop (xubuntu-desktop for instance). That will drag in the desktop PLUS all the defaults apps and software that go with that flavour. Thus, you will have Ubuntu and Xubuntu installed together effectively. Messy, bloaty, redundancy and you don't normally want that. If you want a specific package from another flavour you can install just that (run Ubuntu's Nautilus file manager in Xubuntu, for instance).

Bottom line: It's not quite as simple as you're hypothesizing. Every DE will pull in a few bits and pieces and the more DEs you accumulate, the more chance of conflicts and redundancy. Pretty sure xfce4 drags in Thunar and a few other things. Other DEs do the same. Hopefully it is self-explanatory as to why there is a difference between a Xubuntu install, say, and plopping xfce4 on Ubuntu. The experience will not be Xubuntu. It will be Ubuntu with xfce4 and Ubuntu defaults. The lighter flavours can also go where Ubuntu and Unity can't when it comes to poor spec and older machines.

Personally, I use a [minimal install]("https://help.ubuntu.com/community/Installation/MinimalCD") and xfce4 then add just the apps I need/want. No bloat, no redundancy, no resource hogging. Some people use the mini with a window manager only and no DE. 

Hope that helps and good luck. :)

PS: If you want to know the differences in the 'real-world', why not download some ISOs and try them as virtual machines or create boot media, boot from them and try the distro in a 'live' session which makes no changes to your hard drive or configuration as you run the OS from the USB or disk install media.

### Post by Dennis N on 2015-08-26
Xubuntu is not the result to just the XFCE desktop added. For example, I am using Ubuntu MATE here, and adding the XFCE desktop environment (package: xfce4) adds 42 packages. But this is not giving you Xubuntu. To get Xubuntu, you would add the Xubuntu Desktop. If I did that instead, I add 192 additional packages. So doing the former does not let you see what Xubuntu really is. Xubuntu could be fine tuned to work with XFCE, and have special packages added, such as their own utilities or other features.

You get lots of duplication of packages (as to their function) when you mix Desktops. How well it all fits and works together can be problematic. I always do a pure install of the OS.

### Post by Dennis N on 2015-08-26
I like both and could live with either one, and actually use both on different machines. In MATE, I delete the bottom panel, and move the top one down to the bottom to get a one panel setup. That said, the MATE Tweak utility that comes with it has an "Interface" tweak that offers pre-defined panel setups.

I am more and more preferring MATE for its ease in customizing themes and changing general appearance. The developer (Wimpress) is interactive with the community as well - there is a Google+ group and also a Discourse forum.

Just a few packages in MATE have Spanish names - you quickly get used to them.
