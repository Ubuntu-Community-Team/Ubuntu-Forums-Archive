### Post by DuckHook on 2019-07-13
Perhaps explain things to your dad the way I explained things to Mrs DH:

> Though it may not be evident to you, the command line interface is always there, lurking under the surface, whether it's Linux, Windows or Apple. For all OSes, the command line is unnecessary for typical use… but it is not possible to do the really powerful things without it. In every OS, the foundations are the command line; the pretty graphics are then layered on top to present the general user with a friendly but limited interface.

Therefore, it is not really necessary to learn the command line for everyday things and many people (i.e. Mrs DH) are never exposed to it.

But for power users, facility with the command line is both indispensible and, truth be told—especially within the geeky Linux environment—a point of pride. There are shortcuts and complex actions that can be undertaken in the command line that are just not possible in a graphical environment: for example, stringing several disparate commands together through the process of piping. Such actions are not possible in a graphical environment because the nature of a graphical interface constrains a user to only such actions as are prebuilt into those pretty dialogue boxes. Actions that are outside of or amalgamations/combinations of those dialogue boxes are impossible because they are just not on offer.
