### Post by TheFu on 2015-08-10
There are two ways to deal with inbound attacks from the outside world.
a) don't allow any services to listen on public interfaces
b) run the firewall to block any unwanted inbound connections

For a typical home user, "a" is probably sufficient.  OTOH, if you run anything like BT, a web server or an ssh server, then you'll want to enable the linux firewall.  It is part of the kernel and already there. You just need to pick which front-end you'd like to use to manage it.  This is an extremely powerful firewall - unlike you've ever seen on that other OS.  For most normal people, something like **gufw** is the best answer.  It you need more power, you can use iptables directly. There are multiple books written just about Linux firewall techniques and all the methods available for all sorts of very cool solutions.

AV?  Well - that depends.  Do you exchange files with Windows people who are likely to be spreading virii?  The AV tools don't really scan for Linux problems - just Windows, so if you elect to run AV, it is mostly for your friends benefit.  If you run an email server, it is smart to run AV.

I don't know those programs, so cannot help with alternatives.  Often, looking for a 1-for-1 replacement doesn't work. However, there are ways to solve pretty much any problem, it just might taking thinking about the problem differently. An open mind is a power solution creation tool.

Alwaysync - sounds like rsync, or owncloud or seafile or if you want something like dropbox + tor + ... demonsaw is available.
Blowfish is an encryption algorithm - it is available and many encryption tools are available which make use of it.
120% ... vobcopy and/or dd.

Here's some other things for your consideration ... [https://github.com/sovereign/sovereign](https://github.com/sovereign/sovereign)  ... with Linux, you have much power at your fingertips. Things that you used to use a cloud service for just aren't necessary anymore. Be your own provider.
