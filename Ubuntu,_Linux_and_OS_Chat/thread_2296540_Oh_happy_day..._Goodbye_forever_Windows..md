### Post by Geoffrey_Arndt on 2015-10-12
While data on the "Cloud" is generally more accessible and secure . . . extremely sensitive data must be "pre-encrypted" (secured prior to any Cloud upload).

The tool used MUST be a strong open-source encryption program such as 7zip using AES256 or even better, GnuGpg backend with various front-ends (KGpg, Kleopatra, etc.).    Using unique (different) and strong passwords (64 mixed characters) with 4096 ***_twofish algorithm_*** will present a decryption problem for anyone.    Minimum 2048 bit is still suitable if using low-end cpu's or wanting quicker master keys.

So, probably 75 to 95% of typical data needs either no encryption (e.g., media files) or AES256 (CC receipts, utilities, contacts) , the remaining high value data (IRS, KeypassX DB) can be locked down via twofish as described.
