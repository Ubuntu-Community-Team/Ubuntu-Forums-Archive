### Post by Mike_Walsh on 2014-11-17
Evening, all.

Don't get the wrong idea from the title; this isn't going to be a rant, so much as some personal thoughts about MATE.

I've been using Ubuntu since May this year, and in the seven months since, have explored many of the Linux world's distros. However, I've pretty much settled on the 'buntus; I just really like them.

I have Ubuntu 'Trusty' as my main workhorse OS. I have Lubuntu on an external HDD, and Kubuntu & Xubuntu on flash drive installs.....along with Puppy Linux 'Tahrpup' 6.0 (the latest release of this little gem).

I've heard quite a bit in recent weeks about Ubuntu Mate; utilising the old Gnome2 DE on top of 'Trusty Tahr'. I thought I'd try out the DE that for many years was the mainstay of Ubuntu, and, as I understand, of quite a few other distros, too. Accordingly, I downloaded Ubuntu-MATE, installed to USB with UNetbootin, & gave it a try.

It wasn't what I expected, I have to confess. At the risk of offending some of the longer-standing members of the community (which I don't want to do), I found it to be awkward to use. I guess it's like everything, though; it's a case of becoming comfortable with what you've used for a period of time.....and then it's suddenly taken away, and replaced with something alien, like Unity!

I use Unity, yes. I also use KDE, Xfce, and LxDE.....and enjoy using all of them. I don't think Ubuntu MATE is for me, although I DO like the Gnome2 'look', with the twin panels, top & bottom; to which end, I've incorporated that look into both Xfce & KDE:-

I understand (perhaps wrongly) that Ubuntu Mate is still in beta? It may explain some of the negative aspects of the experience.

I found that:-

1) The Software Centre would install software, but gave no option to remove, if required.
2) The Epson scanner-data and scanner utilities which I've used successfully in the other 'buntus just would NOT install; I kept getting error messages about broken dependencies.
3) Whenever I minimized a window, it would not maximize again correctly. The frame would appear, but NOT the contents.

These were just a few things, among others, that didn't seem to work as you might expect them to! I could understand it if the download was faulty, but I downloaded the .iso image 3 times, checking the md5# sum every time; it was always correct.

Perhaps I've simply tried it at too early a stage in its development.....I don't know. I WOULD like to try it again, perhaps in a few months from now; maybe the problems will have been resolved by then. ### Post by buzzingrobot on 2014-11-18
> **vellon said:**
> ... it is my impression, as an outsider looking in, that Mate is maintenance-mode Gnome 2 - so you may get fixes but no new features.

The Mate folks explain their intent over at their site rather well.  It's to maintain the Gnome 2 desktop metaphor while keeping it compatibile with comntemporary systems.  This involves more than just fixing bugs in the old pre-2011 Gnome 2 codebase.  Probably the most obvious compelling change is making Mate fully compatible with GTK3 and not just the GTK2 base Gnome 2 was built on. 

Few of us would find a Gnome2 desktop from 2011 to be useful today.  The RHEL/Centos 6 distributions retain that Gnome 2 desktop and, from personal experiences, the underlying code it relies on is no longer capable of supporting a contemporary desktop. The  uptodate dependencies needed to run many current applications are not, and cannot be made, available. 

Mate currently does not require compositing, and the Mate team says they will try to keep it that way.  In that sense, it's a candidate for use on old hardware that doesn't support compositing. Other than that, I don't think it particularly targets being "lightweight".    Web and blog posts positioning it that way have things wrong.

Personally, I've found little significant variation in memory use *once applications are loaded* between distributions. Memory use varies from approximately 1.2 gigs in the so-called lightweight distributions to 1.6 or so in Gnome Shell. The differenes between Mate, KDE, Unity, and Cinnamon here are insignficant.  In all cases, the memory imprint of Firefox alone far exceeds that of anything else in use, on any distribution.

### Post by qyot27 on 2015-01-01
> **monkeybrain20122 said:**
> 3) Mate somehow is lighter than gnome2,--which seems unlikely as it includes more patches just to work with gtk3.
# of patches =/= higher resource requirement.

The difference in performance between running MATE nowadays vs. GNOME 2 back then is probably mostly due to whether you have Compiz attached to it.  The last few releases of Ubuntu when GNOME 2 was used also ran Compiz, and my memory's not that good on whether you could turn that off easily (but GNOME 2 was not itself tied to Compiz).  MATE doesn't do that by default.

I've never understood the need to install the separate *buntu distros just to use a different DE; even back in 2005, I just installed all of the ones I wanted to use under my standard install and picked which one I wanted at login.  I still do.

Running LXDE and MATE on the same hardware, with Compton active for light window effects, results in ~145MiBs of RAM usage (as seen by Conky) under normal load for LXDE, and maybe ~185MiBs for MATE (and that also includes running Cairo-Dock and Conky as well).  Which in some regard, is similar to how GNOME 2 would be in a standard Ubuntu release from the pre-Compiz-by-default era (post-Compiz, it hovered somewhere around ~225-250 MiBs with AWN).

The measure of 'old hardware' wouldn't go into this either: my normal desktop setup is an eMachines T1110 (with the only processing-relevant changes being that I doubled the PC-133 SDRAM to 512 and added a GeForce 6200).  MATE runs at least as comfortably as GNOME 2 used to, if not better (see the unscientific measurements from the last paragraph).  I still mostly use LXDE, though, but usually because a lot of what I end up doing is software compilation, and I'd prefer to save resources while I'm compiling things.

It's probably more correct to say that Ubuntu Mate is *not yet* an official Canonical spin. The team behind it, with participation by Canonical personnel, intend it to become official.  They're working to meet the requirements Canonical sets for granting that status.

Meanwhile, it is incorrect to say that Canonicial abandoned Gnome 2. That's an old canard that merits a deep, deep burial. The Gnome project itself ceased development *and* maintenance of Gnome 2 in 2011. Along with every other distribution that used Gnome 2 at that time, Canonical faced a choice of continuing to use a dead and unsupported desktop envirionment, moving to another exisiting desktop environment, or creating their own environment.  Canonical chose the third option, while continuing to provide and support spins from all other major so-called traditional (panel-and-menus) interfaces.  

As to why Ubuntu Mate when there's Mint Mate? You'd need to ask the developers and packagers. I suspect, though, that not everyone who wants to use Mate on an Ubuntu platform buys into Mint's way of doing things, especially it's default approach to updates or it's decision to base itself on 14.04 for the lifetime of that release.

Except for minor theme differences, minor version differences, the odd few of Mint's own applications, and Mint's different (debatedly useful) default approach to updating, they're much the same.
