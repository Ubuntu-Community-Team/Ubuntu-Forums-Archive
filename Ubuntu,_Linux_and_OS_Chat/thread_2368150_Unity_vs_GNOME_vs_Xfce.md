### Post by thenh813 on 2017-08-08
Yes, this is turning into a XFCE fan thread. XD
First off, let me say XFCE = best DE I'v ever used.

Gnome3 is a good environment, but, I don't use it at this current time, I just don't like the layout. I prefer the looks of Unity and XFCE. Unity has always felt "right" whenever I'v used it. When I think of Ubuntu, it's the first thing that comes to mind. After all, it's the way Ubuntu was meant to be out of the box. The search feature is very useful, and the Win+<NUMBER> keybinds to launch pinned applications is also useful. It is a bit resource heavy though, and on newer versions, Wayland gives me never ending problems (more on that later).

 XFCE is quite light on resources and functions exactly how I want it to, nothing is out of place. As a few other mentioned being able to open many instances of applications without getting sent to an existing instance is pretty much necessary to me. Another thing I like is that the interface just looks so clean, and the simple categorized application menu helps me find software fast. Press and hold left click on the menu, slide over to the application, and release. I do this on pretty much any menu. It's one of the things about Windows that annoys me to no end. As far as I know, it works on most GTK and QT based application menus. Yet another thing I like is how the taskbar is set up, it's simple, and effective, and puts the tray where I expect it. I always delete the bottom panel and stick everything on the top. XFCE is definitely my favorite.

Now... about Wayland... To be honest, I despise Wayland. It has never worked right for me. I converted my regular Ubuntu into Xubuntu. Purged Unity off completely due to the  issues it caused me. Sure, the desktop looks fine,  and sure, web browsing works fine, but that's where the good parts about  it begin to end. As soon as I launch any OpenGL accelerated  application, things go wrong, very wrong. It's even worse if the  software is running in Wine. 

Sometimes after going fullscreen the system  and software both loose kb+mouse capture. This requires a hard poweroff  or a bit of sysrq magic and log in text mode on another VTY and restart the desktop environment service. 200ms of input  lag, <30FPS in most games on Unity, even an old version of Minecraft (1.5.2). Older Unity  interface versions that used X.Org never gave me problems.

There's no reason it should perform so badly. XFCE with X.Org  works perfectly. Input lag then around 9ms and FPS usually stays  at VSync (75FPS) in less intensive games. I have to say Wayland is heavily lacking optimization, or it's anti-optimized. The  graphics processor in my APU is roughly equivalent to a GTX  650.

Motherboard: Gigabyte A68H?? microATX form factor.
APU: Socket FM2/FM2+ A8-6600K, Quad@3.9-4.2GHz, HD-8570D@844MHz.
HDDs: 3 x 320GB. They were very cheap. I need to configure it into RAID someday.
RAM: 8GB of Kingston RAM. Cheap low profile sticks.

### Post by yoshii on 2017-09-07
I'm a long-term XFCE fan.  I also like Lubuntu's interface (I forget what it's called).  The nice thing is, you can run XFCE and also download and install PCManFM (same as in Lubuntu) and run it casually like any other program.  It's clean and simple.  And although it's not always safe to do this, you can run "sudo -H pcmanfm" and browse all your stuff as root and edit your system files without complaints from the OS.  (Useful if you need to edit/create/delete .conf or .override files).  Similarly, but not related, if you carefully do "sudo -H catfish" you can search for files and then delete them with the contextual menus instead of the command prompt.  But I am digressing.  Catfish can be installed wherever also.  That's the nice thing about Linuxes; they are so modular.  You aren't locked into any particular format.  You can often mix and match.  

And if you do like Xubuntu, you might like Ubuntu Studio, which also uses XFCE.  It might be overkill if you don't need all those video and audio codecs and programs, but if you do prefer or need them, it's good.  Similarly, if you want to build a lighter Ubuntu Studio, you could try a custom setup starting with Xubuntu (fewer fonts and bloat).  

Technically, according to some old reviews, Xubuntu uses less RAM than KDE and Gnome.  Sorry I can't find the link to cite my sources, though.  

EDIT:  I hope everyone knows that although you can bypass sudo as described above and using other techniques, it's a bit risky if you don't know what you're doing.  
But if you're just doing basic stuff and know what you're doing (not taking risks), then it's alot less fussy than typing sudo over and over again after the system complains about every minutiae.  The mods edited my post to correct my syntax and to remind us of the risks, which is fine; yet really, in this day and age, a user should have the right to take the risks if they want to as long as they know what they are doing.  For example, if I want to add a boot image to my boot screen, I might need to be able to put that custom image of mine into the root directory via copy and paste (after editing the grub entry to allow for a link to that boot image).  It's typically a lot easier to copy and paste a file with a file manager, than to type out long paths in the terminal.  Some people are good at that, and there's nothing wrong with that either.  It's just that if I have 50 tasks to do for 50 files, maybe I'd rather drag and drop 50 files after unlocking the area temporarily rather than typing 50 longwinded command strings.  See what I mean?  Anybody who has used Puppy Linux realises this advantage (and implicit risk) also.  Puppy Linux normally runs as one user with admin root privileges already without any need for special syntax most of the time.  It's not a problem because those who work that way are used to it (and we usually have a backup strategy also).
