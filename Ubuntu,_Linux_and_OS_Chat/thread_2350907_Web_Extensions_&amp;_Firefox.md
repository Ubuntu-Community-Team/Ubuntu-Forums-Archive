### Post by yoshii on 2017-10-07
One funny thing I learned...

If you use a User-Agent Extension AddOn to modify your User-Agent string, you can download and install older AddOns than FireFox would otherwise allow.  Just backtrack to some of the older AddOns that were still stable and not too old.  Yeah, there's risk of course of something going wrong, but I've tried this without issues and installed from AddOns which haven't changed since about v39 of FireFox and still work.  They are labeled as "Legacy" but I don't care about that, I want extensions that are reliable, which is why I've been using them all for about 8 years plus.  

This is similar to how there's a way to edit the internal component of an .XPI file to change it's Maximum version of compatibility.  Sometimes you can change this to an asterisk and it will load into all FireFoxes no matter what their version number is instead of being limited.  Of course, FireFox might still reject it, but at least you can try this.  I forget exactly how this is done, but I learned about this on the internet of course, and tried it for several .XPI's.  Of course there are risks with that too, but I've never had any significant problems.  The setting that gets overridden is kind of like an "expiration date" for the AddOn and is somewhat arbitrary.  FireFox is stable enough that if an AddOn is really incompatible, FireFox will probably detect that incompatibility by itself and disable or prevent loading of the .XPI.  

All this stuff about forcing people to upgrade is an old hat trick that keeps coming back, perfected by the likes of Microsoft.  

But I'm a Linux guy, so yeah, that's not my bag, baby.  :D
