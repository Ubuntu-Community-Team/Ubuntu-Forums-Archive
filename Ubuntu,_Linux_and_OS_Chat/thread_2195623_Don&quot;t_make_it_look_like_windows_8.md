### Post by grahammechanical on 2013-12-26
Linux developers have not worked hard (usually in their own time and without being paid) to produce a cheap (free) Windows clone operating system. I credit Linux developers with having their own ideas and aims and I am appreciative of their efforts on my behalf.

And yet we sometimes get people coming on this forum and asking: "How can I get Ubuntu looking like Windows." And then there are others who insist that their father, mother, granny and granddad, and anyone else that they know, could not possibly learn to use Ubuntu because they only know how to use Windows. And that Ubuntu should be more like Windows if we want people to move away from Microsoft products.

I am confident that Linux developers do not need to get their ideas from Microsoft or Apple, for that matter. And yet I recognise that certain styles of user interface that work well on the desktop do not work so well on a mobile device, and also the other way around. Ubuntu developers have this aim of one Ubuntu code base that will work on PC, phone, tablet and TV. That requires the convergence of the recently developed Ubuntu mobile platform code with the existing desktop platform code but with a user interface appropriate for each device form factor.

With Windows 8 Microsoft is attempting its own version of convergence with a user interface that will be familiar to both its desktop users and its phone users. I have spent just a little time using a Windows 8 preview edition and I think Ubuntu users are getting the better portion.  But then again I am biased in favour of Ubuntu.

Regards.

### Post by lykwydchykyn on 2013-12-27
When Microsoft created the ribbon interface for MS Office, there was a great hew and cry about how awful it was.  We FOSS/Linux fans were quick to point out that OpenOffice had the "classic" interface, and pushed it as the office suite for people who hated the ribbon.

That was 2007.

Now almost seven years later, Open/LibreOffice still has the "classic" office suite interface, but I hear very little complaining about the ribbon, and more complaining that Open/LibreOffice looks like something from the 1990s.  OpenOffice got typecast as the office software for people who hate change. 

The point to contemplate is that appealing to people who like "the good old way" is a short-term gain, long-term loss.   It sounds like Microsoft is going to keep hammering away at the Windows interface, so it may be a few versions from settling in.  But eventually it will settle in to some new arrangment and paradigm, and those trying to capitolize on a "classic desktop experience" will be left behind.

The only way to beat that is either to try to second-guess what the new desktop standard will be, or to create your own version of a user-friendly experience.  So far, I don't see any Linux desktop environment trying to be anything remotely like Windows 8.  I see a few trying for the "classic desktop" UX, a few emulating the Mac, and the rest pushing boldly ahead in some new path.

But rest assured that someday, there will be a demand for a desktop like "modern Windows", and someone will try to meet it.
