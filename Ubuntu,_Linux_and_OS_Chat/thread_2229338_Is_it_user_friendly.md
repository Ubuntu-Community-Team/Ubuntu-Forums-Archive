### Post by robin7 on 2014-06-12
As long as you know going in that Ubuntu is not a drop-in replacement for Windows and that things are done differently, it's great!  Installing software, for example:  In Windows you go to some web site and download the .exe file and hope it's safe.  Then you install it and clean up all the icons and tool bars it automatically adds to your desktop and browser.... in Ubuntu, we open the software center or Synaptic, select the "package" from the Ubuntu repositories (thousands of softwares!), and click "Install."  It downloads and installs automagically and leaves no mess to clean up in the wake of installation.

The interface in Ubuntu is different from what you were used to in Windows.  Something a little more familiar looking - the one I always recommend for new Linuxers coming from Windows - is [Xubuntu]("http://xubuntu.org").  It has a "menu" button right where you would expect to find the Windows "Start" button, that brings up an easily navigable menu of installed applications and the Settings Manager where you can change fonts, wallpapers, borders, panle items and all that.

I also recommend doing some READING before jumping in.  Even as much as one can make Ubuntu (or Kubuntu, Lubuntu, and Xubuntu) "look like" Windows, it's still Linux and still works that way.

### Post by LastDino on 2014-06-13
Put in the live DVD or Usb and watch the magic. 

Things to note:a
*Linux is not windows, so you wont find xyz working on windows working on Linux and vice-verse.
*Don't be scared or get overwhelmed by an idea of you needing to learn something completely different, it really isn't that different as far as basic tasks you mentioned goes.
*You don't need to be a programmer to work with linux.
*Linux operates on principle of opensource; so there will be support for it by the same free software you used on Windows. Like; Firefox.
*You should stick with software provided in Ubuntu software center or Synaptic until you get used to things. *However*, you can very much install anything, given it supports the platform. Don't worry, there are thousands of software available in software center, you might never even need to install something different.
*Get used to using Synpatic. Read up on that.
*Read up on partitioning and file system support in Linux, look for latest guides.
*Make a list of things you do on windows and software you use for it. Then google if there are valid counterparts available on Linux. At the end of the day, it is your decision.
*Google is your friend, if you stick with major distro like Ubuntu, its likely that you will find every problem you might have already addressed by someone somewhere. 
*Just like how you listed your software, list your hardware and see if there are any issues running Linux or a particular distro running on it.
*Getting hang of nomenclature is useful too. So, read up as much as possible.




Things to not do:
*Don't watch Youtube vids and get swayed to try out every single so called ''XP-refugee'' distro out there, there will be time when you will have enough experience to do that, so save it for that time.
*Don't get swayed by people simply stating ''I don't like Unity'' or ''I don't like Gnome'', try them yourself and then decide, of course, no body is asking you to install all of them on the go. Stick with the default until you get used to things.
*Don't get frustrated if you don't find a solution easily.
*Lastly, don't be afraid to ask, no body is born Linux genius. 

If you posted system config, we could tell you suitable Desktop Environment. Also, using Live mode to test things out is good but don't compare its performance to the installed version of same thing, there is considerable difference due to former being on the removable media which read/writes slow and latter being on HD.
Good luck!

### Post by andrea b on 2014-06-15
Niora - much wisdom in this thread. I'm an XP refugee (lots of us flocking to Linux these days).  My two cents: Ubuntu is user friendliest when you really see yourself as a user, rather than a consumer. You are not buying it, you are using it.  It's a beautiful thing, and much of the time, works without issues - but when issues arise - and they will - you 'pay' for your system by learning it.  This seems a reasonable exchange to me, *and* it can also be incredibly frustrating.  Therefore, you're asking a great question; the answer seems to be that each release can mean different issues for different hardware.  If you're a 'user', and you keep everything backed up, you gird yourself and come to the forums.  If you're a 'consumer' - which is also a perfectly *valid* thing to be - you want it to just work! - you can feel really panicked or helpless. 

Sort of like manual transmission vs automatic - you are really in control of this machine, and driving itself puts you in touch with what's under the hood.

It seems to me that the main problem with Linux is ... the sovereignty of Microsoft. That is, compatibility issues.  There are workarounds, and ways to do things simply.  And then there are problems that keep popping up and are inexplicable to the new user (me, for example).  On the one hand, I feel quite committed to learning the thing, and understanding how to fix what comes up; on the other hand - I also need my system to work, and have enough on my hands learning and doing all the other things I need to learn and do without spending time tweaking and working-around.
