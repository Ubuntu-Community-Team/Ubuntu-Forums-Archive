### Post by endlessinstant on 2014-03-05
Unfortunately we don't live in a world where all software is free and there simply aren't alternatives to various hardware/firmware that are setup to work fine in a Windows environment.   The kernel and various distro maintainers do a fairly good job at supporting major hardware iterations but I suspect it will always be the case that Linux will not be a fit for any and all computers.  In the case of some specific apps you mentioned, moving away from their native platform or from a platform that their vendors/maintainers support is simply not going to be viable if you ~must~ run that software.
 
Ubuntu isn't a product.   Canonical does not make any profit off of the proceeds or development of Ubuntu.  They do make money off of services they provide in relation to Ubuntu (eg paid tech support).

### Post by QIII on 2014-03-05
As for printers, graphics cards, wireless adapters, etc.: OEMs write drivers -- not OS distributors.  Microsoft does not write drivers for printers.  Windows drivers work with Windows because the OEMs write them.  Those drivers work with an OS that commands 95% if the market because if they didn't the OEMs would go out of business.

Linux doesn't rate high enough for even a second thought.   That's just how it is.  And there isn't much the Linux community can do about it except try to reverse engineer drivers into a black box because OEMs don't give out proprietary information on their products.

### Post by JeQhdMD on 2014-03-06
If you want to do a fair, "apples to apples" comparison, get your PC with Linux pre-installed, just like you got your Windows PC's.  In many countries, you can do that at the local retail big box, but in the US & Canada, you must go online.

[https://www.system76.com/](https://www.system76.com/)

Also, since Linux is THE major OS for business servers (65%+), mainframes, super-mains (95%+), typical business hardware like scanners, printers, storage devices, are supported because the business & Gov't world requires it.   Just stick with Epson, HP for consumer grade hardware.

Netflix support is in the works as Netflix is looking at alternatives to existing software drivers.   

Also, ANY windows apps, drivers can be run inside Linux via Virtual Machine technology such as Oracle's "Virtual Box" or VMWare's "VMWare Player" . . . just need enough ram, the install media, and your OEM/MS licenses.

In time, you learn that some hardware is actually better supported in Linux - when the OEM's release the code (as in Atheros wireless).   Don't even need to install any drivers, as the Linux kernel has the required code to setup (Plug & Play).

BTW, install the program "Simple Scan" and test it on your scanner.  There's a decent possibility it will work without the need to get the manufacturers driver software at all.

### Post by buzzingrobot on 2014-03-06
Folks who rely on Windows-only hardware and software ought to stay with Windows, rather than bringing unrealistic expectations to Linux. Linux is not Windows.  Windows is not Linux.  Neither are OS X or iOS.  Hardware and software deliberately made to work on only one of those platforms shouldn't be expected to work on any of the others.

Three of the OP's issues are hardware-related. The other two are about software that is also Windows-specific. If commercial hardware manufactuers do not create and release Linux drivers for their products, it isn't realistic to expect Linux to support them. Developers are often able to figure out enough about such a device to create a driver, but there are no assurances if the vendor does not cooperate. (Cheap printers and all-in-one devices, etc., often leverage Windows code on the machine their attached to, making them effectively Windows-only. I.e., if someone wrote a printer driver that relied on code in the Linux kernel, that printer would work only on Linux.)
