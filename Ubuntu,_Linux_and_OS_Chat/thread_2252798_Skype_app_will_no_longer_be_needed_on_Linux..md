I doubt it. the webRTC technology is basically OS agnostic, the only platform that matters is a web browser that can support it. Everybody can access the Skype website, Outlook, Onedrive ect using a Linux or Mac system there is no 'blocking' there.

> **Mike_Walsh said:**
> It's a wee bit suspect saying that ONLY the *Windows* version of Chrome will work with the beta version. I'll bet development for the Linux version of Chrome will be put on the back burner...and turned down SO low, it might as well be off.

That's because WebRTC is not currently enabled in this beta. Therefore it requires a plugin to install which is only available for Windows and Mac. Chrome **for Windows** is specified, meaning that the beta won't work with chromebooks or Linux system...**yet**. Once WebRTC is enabled and fully functionning, it will work in any web browser that can support it regardless of the OS. That's the beauty of this technology, as it makes third-party plugins unecessery and we know that these plugins often disregarded Linux in their system requirements. All it takes is IE, Firefox, Chrome, Opera and their derivatives. All of these browsers support WebRTC.
