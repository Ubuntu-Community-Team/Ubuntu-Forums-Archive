### Post by Rob Sayer on 2014-10-12
Lubuntu/LXDE is cryptic when it comes to config issues, and there are a lot of things you can't do at GUI level as you can with the other ubuntu DEs.


That's because it's basic when it comes to features and doesn't have the integration that the other ubuntu DEs have.  This is why it's so fast and uses as little memory as it does.  No free lunch there.  Don't blame ubuntu or linux.

Linux takes more configuring than windows, but it's more reliable once you do.  It's always been like that and I suspect it always will.  But it's not 100% fair to compare it to pre installed windows.  Anyone who's bought a copy of windows and installed it can tell you that they had to do a bunch of config too.

Do you have multiple GTK based ubuntu desktops installed?  They're all GTK based except for KDE.  Doing this is a common noob mistake, and it often causes difficult to diagnose problems.  I've done it myself in the past but I won't do it any more.

If you're having firefox problems try google chrome (not chromium).  Chrome isn't FOSS but it has better flash and html5 support.  I actually only started using Chrome myself since Mozilla ruined Firefox by turning it into a bad imitation of Chrome ... I'm not alone in thinking this, believe me ... and, while I don't like the fact that I have to install chrome plugins to do things I consider basic, it seems more reliable.
