### Post by QIII on 2013-11-13
Hardware is not an issue with Linux itself.

OEM driver support for Linux is the issue.  No OS can use hardware for which drivers are either unavailable or shoddy.

That does present the practical problem that if you are running Linux and have certain hardware, life will be difficult.  But it is not Linux that is at fault.

### Post by SeijiSensei on 2013-11-13
Linux over dialup was the first type of Internet routers I built back in the mid 1990s.  The [PPP HOWTO]("http://www.tldp.org/HOWTO/PPP-HOWTO/") provided the key information.  

We also supported inbound dialup for a while.  Some connections used PPP, but we also provided email relaying over UUCP to people with old mail systems.  There was a short-lived business in email gateways that used UUCP to get Internet mail and pass it on to old mail systems from Microsoft and Novell.

I'm guessing the OP never has had to set up routing or DNS or any of the myriad of other network services that Linux handles easily.  Doing the same things on the Windows platform would be a lot harder.
