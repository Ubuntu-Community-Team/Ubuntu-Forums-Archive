### Post by CatKiller on 2021-03-15
The thing to do, which is reflected in the default settings, is to use LTS releases, and update to the next LTS release after its first point update, which is when the upgrade starts getting automatically offered. If you choose to participate in testing by using the interim releases or by upgrading early, then we thank you for your service. The releases are generally high quality either way, but it's the wide testing that uncovers any issues.

### Post by TheFu on 2021-03-16
If it isn't an LTS, I will not trust production data on it.  Recently, it feels to me like the LTS versions are less *finished* than what I recall from 10.04 days.

I'm still testing 20.04.x ..... let me see what the exact version is. 20.04.2.  20.04.0 required re-loading 2 times, then again a month later for me.  Sometime in the next month or so, I have to move my last 16.04 server to something newer.  The software system that runs on it - the only reason that server even exists - doesn't support 20.04 yet, so it looks like 18.04 will be the path forward.  I have very little interest in running ESM support stuff.  That just further delays a needed migration that is already too long.  That's why some businesses are still using WinXP - they didn't get into the habit of migrating systems every 2-5 yrs, so they have cement systems that everyone is afraid to touch.

As for non-LTS releases, I see lots of good reasons for those, but mainly for developers and people with a pathological need for "new".  I was like that in the 1990s. New capabilities and entire subsystems were being added every 6 months back then.  I haven't seen anything like that not tied to specific hardware in 15 yrs.  In 2015, I got a new laptop and only the latest non-LTS was easy to load due to the new hardware.  I moved from 15.10 to 16.04 much earlier than normal and that system never ran any other release before the HW broke.

So the real question is whether you are willing to have a few more problems to have newer software and the latest GUI available?
The older I get, the less I want change for the sake of change. Also, I'd rather not have the GUI change ... er ... ever.

**New is the enemy of stable.**
