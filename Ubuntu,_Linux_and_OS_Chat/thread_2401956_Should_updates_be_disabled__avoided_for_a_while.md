### Post by ZoiaGuyver on 2018-10-02
TL/DR: Don't stop doing updates because of this. Security/Stability/Usability > Drama

This is getting way too much cover. Plus the fact you have Stallman saying it isn't a worry while E.S.Raymond says it does have "teeth".

If you actually read the GPLv2 which is what the Kernel is licenced under they cannot "remove" the code as it is. The worry a lot are "toting" as this big problem is actually due to some of the older drivers/modules which are maintained by one or two maintainers that "could" cause an issue if they stopped working on it (from a security/support stance). In theory, this could cause major issues from a business standpoint (if they had some old drivers that were "critical"), but from a personal user standpoint I don't think it's going to be an issue and may actually push the Kernel teams to speed up what it's been doing already with removing old out of date code and generally cleaning it up.

Overall personally I don't think this will be an issue for most of us. Even though people are making a lot of noise about it, just to be clear though I think the CoC is a "good" thing if it's changed wording wise to be more specific/focused and less "catch-all".

Really they should have looked at something like Ubuntu CoC or KDEs I think they are some of the better ones.
