### Post by SeventhGear on 2016-09-20
My first home project with Ubuntu was a HTPC. I started with Plex & XBMC (Kodi now) and just grew from there. Since you are hands on I suggest you use the command line as much as possible. For me it forced me to think of what was being done so I understood the background processes more (rather than just clicking on GUI links & buttons). It also helped with scripting since you can copy your history and create beginner scripts. 

My simple HTPC now has become the center of the house, DNS, Plex, Samba, OwnCloud, Print, etc... It kinda took a life of its own, but I enjoyed the learning and actually being able to use what I built.

What are your current skills and what would you like to learn?  Are you a desktop user or a server admin?  Do any cloud work? Security consulting?  What areas do you want to start?  Even a vague idea is better than none.

For example, everyone needs to know about directory and file permissions on every Unix OS.  I need to know how to setup a printer every 2 yrs - not worth studying for me. I do mostly virtual machine management, so learning and relearning the Ubuntu GUI is a waste of my time - so I don't use it.  More and more, I administer systems using DevOps methods and a tool called Ansible. If I hadn't learned how to manually do the things that ansible does easily, I'd be lost.

What skills do you need to learn?  Prioritize and realize some skills like graphical game programming really need lots of background in other skills before you can begin.
