### Post by buzzingrobot on 2014-01-10
> **alexander.gysi said:**
> 
I was wondering if its worth to install a dual boot on the mac, anyone some positive experience to share? Did someone switch from mac to linux? Do you use both for work (e.g. office, scientific plotting)?

  

I used Macs alongside Linux for a long time.  So, here's my advice:

1.  If you're doing this for fun, dive right in. 

2.  If you're doing it to get some work done or accomplish some specific task via Linux-on-Mac, look first to see if the software is there to do that and if it isn't available for OS X.

Installing Linux on a Mac can be simple and easy, or complex and difficult. It depends on the Linux distribution and on the specific Mac model.  Macbooks with dual graphic cards, in particular, can be problematic.  If the machine has an Intel GPU and an Nvidia GPU, or Intel and AMD, OS X can seamlesly switch between them as demand warrants. Linux, at present, can not. Typically, you must decide which GPU you want to use all and disable the other. The catch is that disabling the card you don't want to use can, sometimes,  be very difficult.  There is no hardware switch. (Fan control can also be an issue.)

Also, Mac's do not have a BIOS. They boot in straight EFI mode.  Some Linux distributions, including Ubuntu, provide install images that boot up, install and run in a BIOS-compatibility mode. (The same thing Apple uses to get Windows to run on Mac hardware.)  The choice of using BIOS-compatibity mode or doing an EFI install is entangled with the video issue because one or the other GPU can be invisible to Linux depending on which approach you took.  You can't enable a video card in Linux if Linux does not know it is there.

Do your homework, then, before trying to install Ubuntu or any other Linux.  Do some research keyed to your specific Mac model, i.e., the version ID string that is shown in the "About This Mac" display.

Also, OS X is a Unix. Underneath the GUI is a character-based toolset that is essentially equivalent to that found in Linux.

Second also:  Many, many Linux programs are ported to OS X and made available from several projects, including packaging and dependency-resolvers.

### Post by 1clue on 2014-01-13
OK well XQuartz is something you REALLY want on your Mac if you're using a Linux box with a GUI.  It's actually pretty decent, but you might want to install a window manager and other gui-related apps to make it look nice.  I don't bother.

From your mac:

ssh -X joe@yourlinuxbox
firefox

That will start the firefox application on your linux box, but the display will be on your mac.  You need good fast networking if you're going to do this.  It's not nearly as efficient as Remote Desktop, and it's an encrypted interface so that makes it slower.

It's lighter weight if you're not doing a media-oriented app.  For example, running sublime text 2 is pretty light weight, although it works on both Mac and Linux with the same license.

The -X is a capital X, and it allows xorg forwarding through the ssh tunnel.  I've done it across the net but it's amazingly slow.  IMO when the new display techs trickle down to here it should be better.

### Post by 1clue on 2014-01-19
The OP already listed a couple apps which are critical to him that there is no happy analog for in Open Source.

FWIW I can't imagine why anyone would ever dual boot anything anymore.  You can install in a VM and get full use of both operating systems simultaneously, copy-paste between them, share apps, use them as client/server -- or you can shut down the one you're in for the 5 minutes worth of work you wanted to do on the one and save it to a file before you reboot back into the other and hope the formats are actually compatible.  Dual boot is a deliberate handicap to prevent two operating systems from interacting.

Again I ask the question:  Why would you have to choose one over the other?  I've been using Mac and Linux side by side for years.  Won't have it any other way.  My mac has Linux VMs on it AND a whole lot of Open Source software, my Linux box has Linux VMs on it, and the mac takes care of the things that Linux sucks at.  And Linux takes care of the things Macs suck at.

This push to use only Linux is just peer pressure -- which is indistinguishable from bullying -- there's not even a market reason for it.

Depends on what you want.

If you install it over Ubuntu, you will still have Unity and all the default Ubuntu applications on the system; you can just choose your desktop environment at login time and swap back and forth.

If you install Xubuntu, you'll get only XFCE on the core Ubuntu system, so Unity won't be present and the default applications will be different.

### Post by 1clue on 2014-03-01
If you're an average guy using photoshop then maybe gimp and inkscape are enough.

My sister is a professional photographer. She uses that huge imac.  When she got it she blew her whole budget getting the mac, didn't have enough for photoshop, she was using photoshop on her old laptop still, which I thought was a waste.

I put gimp on her mac.  She used it for a few months until she could get photoshop.  She DID get photoshop, because gimp just didn't have it.

Feature for feature, almost everything is there.  The problem is that gimp's controls are not set to realistic ranges.  They'll go from 0 to 100, the full range of possible values.  But in real world editing, maybe the only rational values go from 30 to 50.  So the photoshop control is easier to manage because it's a more gradual change for the same mouse movement.

The other big thing was the arrangement of controls in relation to each other.  When you're doing a certain type of thing to an image, photoshop places all the tools you might need for that activity together.  Gimp has the controls distributed differently, causing you to go from one place to the other, meaning you go through a lot more steps to get the same job done.

I'm not dissing gimp.  It's what I use when I need to do something like that.  But if you're a trained professional and use most of the features in photoshop to do real work, then you're probably going to want photoshop.  I'm a complete novice with that sort of thing, so while I've never used photoshop I can imagine I would be every bit as incompetent with that as I am with gimp.

Inkscape for me is another story.  I like it a lot.  I use it on Linux and Mac both.  That said, I am not a professional with that either, so your mileage may vary.

It's like this:  You get the right tool for the job.  The right tool for photo editing is photoshop.  The right tool for what I do is a linux box.

You might consider installing Mac OS on a VM inside of your Linux partition.  If you're using your Linux partition most, you could just fire up the Mac vm the few times you need it.
