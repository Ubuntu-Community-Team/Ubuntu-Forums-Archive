### Post by oldfred on 2020-09-09
I am most guilty one.
My standard copy & paste to use Boot-Repair has for years suggested to only use the ppa as that has been the most current. 
And users typically already have the Ubuntu live installer, so usually easier to just add Boot-Repair to Ubuntu live installer. 
Not sure about other distributions as Boot-Repair expects certain supporting software and then ISO may be better option.

Most also do not understand that Boot-Repair is not for fixing Windows issues. With BIOS it could install a generic Windows "type" boot loader, syslinux, but really could not fix Windows.
And it does not do partitioning, which sometimes is the issue.

But the report it generates is very useful for resolving boot issues.
