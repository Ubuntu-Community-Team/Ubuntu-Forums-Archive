### Post by grahammechanical on 2016-04-28
The proposed repository is a way of allowing users to test updated packages before they are put into the normal update channel. If users start screaming the package can be withdrawn until the bug is fixed. It is all part of the method of keeping even the development version stable through its 26 week development period.

There is a risk with running the development version and there is greater risk with running the development version with proposed enabled. This is why it makes sense to have more than one install of Ubuntu when running the development version. The fact that it did not rain at all last summer does not mean that it will not rain at least once this summer. Just keep the umbrella close by.
