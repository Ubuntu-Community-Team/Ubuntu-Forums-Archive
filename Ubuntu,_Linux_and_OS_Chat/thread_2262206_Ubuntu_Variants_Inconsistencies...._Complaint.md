### Post by jp42simm on 2015-01-23
In my view one of the major reasons that Linux will never be a true competitor to Microsoft or Apple (note I did not say "Windows" or "Mac" because those are not the company names) is embodied in what we find among the various Ubuntu variants.  I suppose there is a kernel variation that they all might have in common.  Beyond that there are numbers of desktops that one may choose from.  It all seems so nice.  Don't like Unity?  OK just go to KDE or Lubuntu.  But there are down sides to these decisions.  The one that I find, and I hate it, is the wide variation in the software repositories.  For some unknown reason each desktop has it's own repository.   Try Ubuntu then move to Lubuntu and see what I mean.  More than just the desktop is different.  Under Lubuntu don't expect to be able to logon to your software account and make a purchase as you did with the other desktop.  Under Unity I purchased "Cave Story" but under Lubuntu it is not available.  Under Unity "Neverputt is an older version.  Under Lubuntu a newer and better version is available.  Under Unity "Monkey Bubble" isn't there.  Under Lubuntu it is.   Plainly the desktop enviornments are about much more than just the way things look.   This doesn't have to be.  When the Ubuntu family decides to get together and cooperate we can change at will from one desktop to another without being afraid of losing the software that we have come to be comfortable with.   Change all the desktop variants so that they all are using the exact same repository as the main Ubuntu flavor and then improve that repository so that it includes all the programs that other desktop repositories have including the latest versions of each.  As long as all of you developers refuse to do this Microsoft and Apple will continue to reign supreme and Linux will be riding at the rear of the computer bus wondering why, when our softeware is free, this could be.

I do not expect an answer and do not want to read excuses about why this would be so hard to do.  It's doable... period.  In the world of (free and open source) computer programming anything can be done.  All that is missing is the will.

Thanks for reading my long overdue complaint about Linux.

### Post by TheFu on 2015-01-23
Dude, Linux is an "idea" that it appears you haven't learned.
It is about freedom, not some GUI.

The GUI is NOT the OS.  It is just another program. If you don't like a GUI, don't use it.  You can still load any Linux program that you like. If it isn't in the base repository on your box, add the repository. If it isn't there, use a PPA. If it isn't there, get the source and compile it yourself.

Freedom. There are many articles, books written on this.

Linux is also about the [UNIX philosophy]("https://en.wikipedia.org/wiki/Unix_philosophy"), which I don't think you know.
* Make each program do one thing well
* Small is beautiful
* Choose portability over efficiency
* Store data in flat text files.
* Avoid captive user interfaces
* Make every program a filter.
.....

You can read the rest.

Canonical is not "Linux."  The Linux distros they create - Ubuntu ... are not Linux.  Linux is an ideal, and a kernel.  As an end user, you've probably never interacted with "linux" directly.  How can you blame something you don't know?

OTOH, if you want to complain about Canonical (we all do at some level), fine.  I'll start - I think Canonical has rejected the UNIX philosophy too much. ;)  Am I correct?  Probably not.  Heck, the UNIX philosophy isn't correct for most end-users either, at least not for noobs.

Small clarification.

NASA uses almost every OS there is.  I worked at JSC for 8 yrs inside MOD and in the STL for ISD.  We had one of everything in the STL - pretty much.
