### Post by grahammechanical on 2014-06-30
Anyone can handle Ubuntu whatever their age. Every new thing requires learning new things. That is a fact of life. The most scary thing about Linux distributions is the installation.

In my case I installed on to a new, blank hard disk in a machine I had built myself. Not so scary. But when I came to dual booting with other installations of Ubuntu, then due to inexperience, I found it scary. That is not necessarily a bad thing. Overconfidence can lead to users making stupid mistakes. So, I would say that buying a machine with Ubuntu pre-installed is the least scary and troublesome way of getting and using Ubuntu.

On this forum we see posts from people who have had problems installing Ubuntu on machines with another OS pre-installed. But that is what the forum is for. So, by nature we see the problems and not the success stories. A person does need to research what could go wrong and what should be done to fix it. Prepare well if you decide to dual boot.

If your machine is running Windows 7 then it should run Ubuntu very well. Linux distributions need a lower hardware specification than Windows does. But there is one thing that you should know about since you mention it. Do not expect a lower power consumption with Ubuntu. It may happen. It may not. Linux developers have had the luxury of making Linux compatible with ever more powerful hardware and that includes improved laptop batteries. They have not had to worry so much about battery life.

A little more than a year ago it was decided to work on a form of Ubuntu that would be pre-installed on phones and tablets. So, the Ubuntu developers have had to worry about power consumption and battery life expectancy. All this work on Ubuntu for phones and tablets will filter down on to Ubuntu for the desktop. Things will get better as regards laptop battery life.

### Post by buzzingrobot on 2014-06-30
Yes, definitely backup any files you can't afford to lose. That's always good advise, but especially true when you are changing operating systems.

If you use Windows programs that you cannot do without, or do not want to do without, perhaps the better option is to remain on Windows.  Ubuntu offers many programs that are equivalent or superior to those in Windows in capability, but they are *different* programs.

Here is a wiki page on setting up dual-booting: [https://help.ubuntu.com/community/WindowsDualBoot](https://help.ubuntu.com/community/WindowsDualBoot). The catch to dual-booting is that setting it up correctly is very likely the most complex thing a new user migrating from Linux will do, and they do it, by definition, before they have any experience with the new OS. Things can, and do, go wrong.  If you scan these forums, you will see  plenty of pleas for help from new users whose efforts at dual-booting went awry, leaving them without (real or apparent) access to Windows.

As you may know, there are several "flavors" of Ubuntu that use different interfaces on the same infrastructure:  Kubuntu, Xubuntu, Lubuntu.  The latter two are generally considered to place the lightest load on hardware.

All these, as will as straight Ubuntu, can be downloaded, burned to a DVD or USB stick, and booted on your current machine and run in live test mode with no impact on the hard drive(s).  It's a good way to uncover hardware support problems and just to see if you like it.

If battery life is a critical, or the critical, issue for you, be aware that Linux is unlikely to extend it in any significant way, and you could, in fact, have a shorter battery lifetime. Maybe a new battery?

Linux is not immune to malware.  But, malware is a business these days and Windows is the overwhelming target because that's where the greatest return will be found. Viruses designed to attack or exploit Windows code won't affect Linux code.  However, attacks that rely on the user's gullibility -- clicking links in email and blithely filling in account data, etc. -- are just as possible in Linux as elswhere.  (Malware does not float in on the wind.  While I haven't used Windows in a very long time, I know plenty of other people who do use it much as you do without being routinely pestered by viruses and malware.  They use an effective anti-malware program and the keep their wits about them as they use the internet.)

To be a bit heretical, I'll suggest being in no hurry to move to Linux. Check out some of the live DVD/USB images.  See how they deal with your hardware. See how you deal with them. If you decide to go with Linux, or to try dual-booting, *ensure* all irreplacable files  are backed up somewhere else, even on a batch of CD/DVD's, if necessary. (And verify that the backup worked and the files can be accessed.) You don't need a fancy backup program for this.  Just copying files is sufficient.

### Post by Mark Phelps on 2014-06-30
You should read the linked article BEFORE you make the switch to Linux:  [http://linux.oneandoneis2.org/LNW.htm]("http://linux.oneandoneis2.org/LNW.htm")

Too many people come here thinking Linux is a free Version of Windows and then get very frustrated when they find out it is NOT.

Except for the few cases in which you're using Windows apps which have versions that also run in Linux (for example, Firefox browser), in the other cases, you will have to use entirely new apps -- which will require learning new ways to do things you already know how to do with Windows apps. It's not a huge learning curve, but it is there -- and it can prove daunting to some folks.

You should boot from a Linux liveDVD or liveUSB to get a feel for the differences.  It will run slower, but the desktop experience (look and feel) will be the same as if you had installed it.  Then, you can see for yourself if the Linux apps fit your needs as replacements for Windows apps.
