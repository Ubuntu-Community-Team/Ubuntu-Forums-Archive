### Post by buzzingrobot on 2014-11-30
> **ronniepinsky said:**
>   Will we ever reach a situation where we can run Linux on cutting edge hardware? 

Windows only works on "cutting edge hardware" because developers are paid to write drivers for the "cutting edge" devices in it before the hardware is released for sale. If they wrote them for Linux, then Linux would run on that hardware the day it hit the market. If the device vendors released specifications and data about their hardware, then Linux developers would be quicker to write their own drivers. 

I'm not trying to set up proprietary vendors as villains.  But, it is as fact that all of them want to profit from selling into the Windows market, while few of them give a thought to the Linux market.  They just spend their money where they think they'll get the biggest payoff.

We have much the same experiece installing something like Win7 on current hardware.  You'd better have the driver for your wifi or ethernet card handy on a USB stick, because Windows 7 won't have one.

That said, it's been, I'd guess, about 10 years since I put Linux on a machine that had hardware it didn't know how to use. These days, hardware must be known to work with Linux or I don't buy it.  If it isn't the absoute latest "cutting edge", I don't care. Computer hardware is commodity stuff and progress is steady and evolutionary, not explosive and revolutionary. The payoff from this month's "cutting edge" is always way smaller than the extra cash you pay for it.

Hmmm. Well, think about "cutting-edge" with regards to consumers. 

A consumer can get a 17" ASUS maxed-out laptop for $2278.99, or he can opt for a Dell Inspiron 17" with more conventional hardware specs for $432.99. (Both on Amazon.) Regardless of which machine YOU would buy, which do you think sells the MOST units?

I bought a couple of new computers in the last few years, and both times I opted for middle-of-the-road designs. They delivered all the features I need at a very reasonable prices. And I've been able to install various Linux distros on them without difficulty.

So the point is, if I were smart enough to develop drivers for Linux but had limited time and money to devote to the project, I would probably concentrate on making drivers for the hardware used by the largest percentage of buyers. And, actually, the other side of the same coin is that hardware manufacturers, who also have finite development resources, concentrate on making drivers for the operating systems used by the largest percentage of buyers. Just as others have noted in their comments.

On the third hand: virtually ALL the highest-end computers run Linux. It rocks the super-computer world. But that's not really what the OP meant in this case by "cutting-edge"!
