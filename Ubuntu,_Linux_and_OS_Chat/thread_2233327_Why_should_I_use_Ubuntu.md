### Post by buzzingrobot on 2014-07-08
> **victor_queiroz2 said:**
> 

I am a Fedora user...

  


The pool of packages available in Fedora or Ubuntu or most any other mainstream distribution is essentially identical because distributions are primarily packaging efforts. At any given time, point release versions may be a bit different, or not. 

Fedora's primary interface is Gnome.  It's Unity on Ubuntu.  KDE, XFCE, LXDE and Mate spins are available on both.  Cinnamon can be installed after-the-fact.  Ubuntu has PPA's.  Fedora has koji. Fedora uses sysemtd.  Ubuntu is transitioning to systemd.  Both have a 6-month release cycle; Fedora has no LTS equivalent. Fedora is RPM based; Ubuntu is deb based.

In other words, the differences are around the edges.

But, those edges can be significant depending on what you like and what annoys you.  Burn and boot a live image and see what you think.

I use both Ubuntu and Fedora and find neither is particularly buggy and neither is more or less buggy than the other. Are there bugs?  Yes, there are bugs in all software. 

Changes between releases, and the risk of upgrading to a new release, seem to me greater in Fedora because its purpose in life is innovation.  It's an early, often the first, adopter.

### Post by buzzingrobot on 2014-07-10
> **Rob Sayer said:**
> I tried Fedora 19 for a short while.

The repos stink.

That's not very specific.  Pretty much the same software that's in Ubuntu's repos and PPA's is in Fedora's repos, and in their koji and copr collections, and the RPMFusion site.   Specific versions often vary a little depending on where each distro is in its release cycle.  Fedora seldom updates major package collections (think Gnome, KDE, etc.) after a distro release.  But, updates to kernels and individual packages are frequent. 

> Plus, the package management tools in fedora 19 were broken.

Not that I noticed.  They have been rolling out early test versions of a new Software tool, and I believe it saw its first appearance in F19.  It wasn't intended as the primary package management tool, and it still isn't.  A key objective for Fedora is as a platform for debuting and wringing out new software for potential use down the road in Red Hat Enterprise Linux. New users need to be aware of that.

> When I tried to diagnose this problem the people on their support forum certainly seemed knowledgeable.  However, they came across as a bunch of passive aggressive pompous geeks who act like it's your fault because you didn't read the bugzilla report first.

FedoraForum is not an official Fedora/Red Hat support site.

>  So I'm wondering: why Fedora?

Ahead-of-the-curve Linux technology, excellent documentation, unadulterated Gnome and strict FOSS adherence for those who want those, spins for KDE, XFCE, Mate, LXDE; Cinnamon available and well supported.

> **abilbrou said:**
> Last time I used Fedora (I think 19 or 20) it refused to accept the password given in the installer.  After doing some configuration I managed to get in, but I pretty much stopped using it after that.

You typoed one of the passwords.  

If Fedora actually failed to recognize correct passwords, *every* user would be affected and the release would have been held back. 

> ...you should use Ubuntu because it is not a sandbox used for testing ;)

Fedora is not a "sandbox used for testing", which implies releasing untested code directly to users. Fedora follows a 6-month release cycle just like Ubuntu.  New code in each is subject to the same amount of internal testing before release.

Fedora is fine and stable for use as a desktop.  It is not, however, deliberately shaped to appeal to Windows users with no Linux experience, as Ubuntu is. It also strictly adheres to free software requirements, as does Red Hat (for reasons of principle and legal practicality).  The resulting lack of some drivers and codecs in the default install can annoy some users, even though procedures to add them from reliable third-party sites are simple and well documented.
