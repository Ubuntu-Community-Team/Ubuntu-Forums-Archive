### Post by buzzingrobot on 2016-09-02
"Learning Linux" is something that depends on what you want to learn.

Ordinary users who need to launch and use GUI applications should seldom, if ever, need to open a terminal. The web is full of copy-and-paste terminal commands. That is misleading. Terminal commands are not magical incantations. 

If you want to use the terminal effectively, however, for whatever reasons, you need to understand that it is program like any other program. It takes input (as text rather than clicks), interprets that input, and then dispatches "commands" to the appropriate portions of the operating system. Typically, when a command completes, the results are displayed.

When you use a terminal you are, then, interacting with a single program that interprets your input,  This is traditionally called a "shell" program.  Several exist, including Bash which is commonly used in Linux. Bash is effectively a rather powerful programming language. So, learning to use the "terminal" effectively really means learning to use the programming language called Bash.

Again, though, users who simply want to use GUI programs should seldom have a reason to do that beyond curiosity.

If you are interested in learning how Linux works, in depth, as an operating system, then you've taken on a considerable course of study.  Begin with books on operating system principles and design, then books specific to Unix, and then Linux. Several of the original designers of Unix have written books describing that experience.  That historical perspective can be useful to understanding how decisions they made continue to shape Linux today.

### Post by sudodus on 2016-09-06
**~/.bash_history** is also a good reference. The standard way to access previous commands (in order to use them again) is via the arrow-up and arrow-down while in a terminal window. But you can also search it via grep, or like I do, search for commands containing a search string via a one-line shellscript line, ***HI*** (all such commands) and ***HIT*** (the ten last such commands), in my own bin directory.

```
$ cat ~/bin/HI

[SIZE=3]history|grep "$*" |sed 's/^ *//' |tr -s ' ' ' '|cut -d ' ' -f2-[/SIZE]

$ cat ~/bin/HIT

[SIZE=3]history|grep "$*" |sed 's/^ *//' |tr -s ' ' ' '|cut -d ' ' -f2- |tail[/SIZE]
```

That way I can easily find a command line, if I remember part of it, for example my previous commands containing **mkusb.*iso** (**mkusb** followed by some characters and **iso**
