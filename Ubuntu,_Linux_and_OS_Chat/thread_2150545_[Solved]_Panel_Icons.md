### Post by Copper Bezel on 2013-06-03
Yeah, this is an old complaint about Unity that goes back to when it was introduced in 11.04. Mark Shuttleworth has weighed in on it and claimed that no matter how old and established that behavior of minimizing when the icon (or taskbar item) is clicked, it doesn't make a lot of logical sense to click the program to make it go away. For consistency's sake, clicking the icon always means accessing the application - launching it, switching to it, or spreading open windows - not putting it away.

There *is* a complication in that in Unity, clicking the Launcher icon when an application has two or more windows running spreads out the windows. Even if the minimize function was in there somewhere, it would only apply to single windows, which would be weird and inconsistent.

### Post by gordintoronto on 2013-06-03
If you right-click on the desktop and select "Change Desktop Background," the Appearance setting will open up. If you select the "Behavior" tab, you can click on "Add show desktop icon to the launcher." Basically, that means one click can minimize everything. Perhaps that is useful for you, maybe not.
