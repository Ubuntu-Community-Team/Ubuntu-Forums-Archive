I have an HP dc7600 that I bought as a refurb a few years ago with the same processor, 2GB RAM and XP on it.  I installed Ubuntu (dual-boot with XP) without issue and then installed the Edubuntu packages later, again without issue.  However, as web pages became more RAM-hoggy and the computer started to attempt to use swap space for RAM the computer slowed to a crawl.  A RAM upgrade to 3.5GB (the computer has four slots for RAM, since it is only a 32-bit CPU it can only recognize 3.2GB RAM anyway so further upgrading that is worthless) took care of that issue for the most part.  The most RAM I have seen the computer using is 2.6GB, in general it uses about 1.5-1.8GB in average use so I don't foresee this being a problem in the near future.  Occasionally my CPU will max out and cause an issue for a couple seconds but overall the computer still works well.  Four 1GB RAM sticks (PC2-4200 533MHz) cost $42 shipped on Amazon.  I only replaced three as the fourth one was directly under the power block and I didn't feel like disconnecting/reconnecting it and the computer cannot recognize any more RAM anyway, using different RAM capacities doesn't seem to hurt the computer any.  

The documentation for my computer claims that it can also use PC2-5300 667MHz memory but it is a bit more expensive and from what I have gathered it does not increase computer speed that much.  Obviously if you use a different speed RAM stick from what came with the computer you would need to replace all of them to get any benefit from the faster RAM.  Otherwise sometimes the computer will run but at the slower speed.

As for the OP's original question, if you do not have at least a GB of RAM even Lubuntu will give you problems.  I have been using my laptop with Lubuntu on it more lately, the computer routinely uses 800GB of RAM (as I type I am at 807MB use with Firefox using 438MB of that) and with RAM-hog websites goes as high as 1400MB (1.4GB) with several tabs open in Firefox, Rhythymbox running, the Task Manager (to measure RAM and CPU use) and Psensor to monitor temperatures of the CPU and graphics card.  Firefox takes up 1.0-1.1GB of that with the RAM-hog websites.  It may technically install and run on 512MB (with nothing running it uses about 300MB) but even just starting Firefox brings that up in the 475-500MB range, with one tab open you would be looking at about 600MB -- enough to put you into swap which is so slow that it is unusuable IMO.  Even if you use Lubuntu with 512MB RAM you really need to increase that substantially, if your HP has four slots like mine an increase to 2GB (four 512MB or two 1GB sticks, Google what slots you need to use if you only have two sticks) would make the computer much more usuable.  If you use the full Ubuntu OS, you need to max it out at 3.5GB.  Good luck whatever you do.
