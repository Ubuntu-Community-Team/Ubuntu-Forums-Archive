It's got nothing to do with the actual Linux desktop environments we use. Software vendors will come to the Linux software market (which barely exists) as soon as they decide they can get a better return on investment there than from putting the same resources into doing what they've always done.  The reasons they don't have nothing to do with Unity or Gnome or KDE or whatever.  The licensing requirements put them off.  The reputation, deserved or not, of Linux users as unwilling to pay for software puts them off. (No corporation has ever succeeded in sustaining a business selling a Linux distribution in the retail market.)

I think there is, however, an issue with Linux desktop software:  It lacks variety and it isn't innovative enough. When Linux software allows people to do things they want to do that cannot be done on Windows, then those people will start to think about using Linux.  Without those kind of applications, the best desktop imaginable is just an empty shell. (One big reason people go to OS X and stay there are the applications for that platform.)
