Mint does indeed have more packages since it uses the Ubuntu repositories as well as it's own repositories on top of those.
But mint also uses a different software store which probably accounts for something.

Ubuntu Software (which in turn is really gnome software) is a desktop program for desktop programs.
So it exclude a lot of non-desktop packages.
With the exception of snap packages, which also show up in Ubuntu Software, you'd be hard press to find many terminal-based packages in Ubuntu Software.

I don't use mint so I don't know if it too excludes non-desktop packages or if it just lists anything and everything it can see.
But I don't know of any other than gnome software that doesn't list everything.

(even the older software-center Ubuntu used to use could list everything if you toggle it to show technical items.)

### Post by SagaciousKJB on 2018-10-18
I kind of like the balance between new and stable that Ubuntu achieves with its support cycles.  Something like Debian can become a bit outdated rather fast because it's designed to be as "stable" as possible.  That makes updating to newer packages kind of difficult when something like your kernel or gcc is too outdated for the software.

Then on the other hand there's distributions like Arch which tend to stay so new that they often break hardware support, or break backwards compatibility between older software and newer libraries.

I like that Ubuntu comes kind of in the middle, and has their Long Term Support versions.  Those will receive back-ports, security and bug fixes for three years which means that for a whole three years between system upgrades you can pretty much rest assured that when you need something to be installed or upgraded through apt that it will work without any issue.

I personally never really understood why people needed the newest bleeding edge editions of software, like as if it had some kind inherently better capability versus older/stabler software, but ubuntu also has the .10 versioned releases to try to keep up with stuff.  I may not be understanding the point of the .10 releases though.

### Post by TheFu on 2018-11-14
For a desktop, Mint is fine, especially if you like the GUI.  Ubuntu has a much larger community, which can make finding answers to those little questions and issues easier.  Much more documentation for Ubuntu releases too, assuming you only use LTS versions.
For a home user with a few computers, I doubt there is much difference between Mint and Ubuntu that matters. To me, the GUI is just another program, so I'd never pick a distro just because of some GUI.

Some commercial software only supports Ubuntu. If you call up a vendor to get help, they won't help on Mint if they only support Ubuntu. While it shouldn't matter, some vendors just won't help on a different OS.

I've never seen a Mint Server release, so it is mostly useless to people like me.  I manage 20-30 systems, but only 2 are desktops.
Do I want to deal with 28 Ubuntu servers and 2 Mint systems?  If you want to be an Admin, then learning RHEL, Deb, and Ubuntu servers will make you much more useful to employers.
