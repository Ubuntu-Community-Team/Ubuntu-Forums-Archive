### Post by Mike_Walsh on 2016-01-24
Thought I'd throw my two-penn'orth into the ring.

I run Xubuntu 14.04 alongside half-a-dozen Puppies. My old Compaq desktop is 64-bit capable (AMD Athlon 64 X2 3800+ dual-core), but I choose to run 32-bit Pups because, quite simply, they are so much faster and more responsive. With the max of 4 GB RAM my machine can support, I've no real need for 64-bit anyway.

I'm guessing the purists amongst you will already be shuddering.....or disdainfully sniffing! Some of us have budgets we have to work within.....and new hardware rarely comes into the equation.

I also run an even older Dell Inspiron laptop. 2.6 GHz P IV. Can only run 32-bit Pups. It currently runs Chrome 47. Haven't seen any of these warning messages at all. I also run Chromium on the Dell, but quite an old version.....37; quite simply, it just works on this hardware. I've written a tutorial on the Puppy Linux forums as to how you can run the newest version of FlashPlayer with an older browser. It's perfectly possible; I've been doing this for over a year. There's no complaints from Google about what I do.....and kernel version appears to make absolutely no difference whatsoever.

If anyone's interested, you can find it here:-

[http://www.murga-linux.com/puppy/viewtopic.php?t=101275](http://www.murga-linux.com/puppy/viewtopic.php?t=101275)

I've not yet tried this in Xubuntu, but in theory it should work. Directory structure may be slightly different, and permissions may need modifying, but the basic principle remains sound. You're only interested in the 'libpepflashplayer.so' module. Rather than the method outlined in the original post, we now use the Chromium flashplayer module compiled by a guy named 'AlienBob', which can be found here:-

[http://www.slackware.com/~alien/slackbuilds/chromium-pepperflash-plugin/pkg/14.1/](http://www.slackware.com/~alien/slackbuilds/chromium-pepperflash-plugin/pkg/14.1/)

Don't worry about the fact it's compiled in Slackware.....this works just as well in the 'buntu-based Pups, such as Tahrpup. If anyone fancies giving this a try, be my guest; I'd be curious to know if it would work with the 'buntus.

The only 'fly in the ointment' is that Bob can be a little bit slow coming up with the new version.....but it's never more than one version behind. However, since I don't use any of the major streaming services like NetFlix or YouTube, it's never really been a problem for me. I'm a graphic artist as far as my hobby goes; I only use browsers for general browsing.

Of course, for those of you who are sticklers for doing things 'by the book', this post will be of no interest whatsoever. Your other option is, of course, HTML5; there are plenty of posts on the Forums about how to enable it.
