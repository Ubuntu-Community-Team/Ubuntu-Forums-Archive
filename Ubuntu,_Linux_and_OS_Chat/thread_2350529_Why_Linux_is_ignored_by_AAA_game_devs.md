### Post by schmied on 2017-01-25
So as we all know, Valve fully supports Linux gaming, but that's mainly because their own OS (SteamOS) is based on Linux. They don't even have to do that much work, either - since Windows XP doesn't support DirectX 10, 11 or 12, Valve have to make OpenGL ports of their games anyway. Why? Because 18.21% of Chinese desktop users still use Windows XP (for obvious reasons). Now that is a lot of people who can't buy your products. Recently, the numbers have begun dropping with the release of NeoKylin, China's own Linux-based system, but Linux can't use DirectX 11 or 12 either, so OpenGL ports were a no-brainer. So since Valve have the OpenGL ports anyway, it was easy for them to release their games on Linux.

id Software is another prominent example. Ever since the first release of Doom 1993 (I hate the fact that whenever I want to address the game, I have to write the year it was made.), id have always supported Linux gaming and treated it with a lot of respect. After the famous Carmack rant, id Tech has used OpenGL as it's main API (and it still does today, along with Vulkan) and everything was going well.
Then, after Bethesda's acquisition of id, id stopped supporting Linux altogether. Quake Live can no longer be played on Linux. And there were exactly ZERO reasons as to why that happened. But after seeing Bethesda's other anti-consumer practices, I accepted that id was pretty much dead. 


Supporting Linux is a great idea. Linux users have the tendency to buy software they like (such as the Lightworks video editor), and from my personal experience, appreciate developers and the effort that went into their software. So, if you have the option to sell you products to people who are less likely to steal it, why not take it? If you're developing your game in OpenGL (which you should, because DirectX is a pain in the ass), it doesn't take that much effort to port it over to Linux (unless you use hundreds of proprietary extensions, in which case RIP)



TL;DR: There isn't much of a point to this article, rather it's supposed to start a serious discussion regarding gaming on Linux.
I'm currently co-developing an IDE for game development and fully intend to support Linux and it's community.

Schmied.
