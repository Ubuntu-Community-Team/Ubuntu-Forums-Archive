### Post by Claus7 on 2022-02-18
Hello,

every single time I'm trying to use the default gdm, I revert back to lightdm: as name suggests it is lighter and boot time decreases a lot (using an hdd). Also, it is easier to configure e.g. change its theme/background.
With gdm, on top left, you are able to see which applications are open, and if for example you have gedit, then you can click in order to open a new instance. This instance won't be accessible until you actually log-in. I haven't checked if this can be modified.

For latest gdm configuration see: [https://www.omgubuntu.co.uk/2022/01/change-ubuntu-login-screen-background](https://www.omgubuntu.co.uk/2022/01/change-ubuntu-login-screen-background)

Regards!

### Post by &amp;KyT$0P# on 2022-02-23
I use lightdm mainly for these two reasons:

1) With lightdm & lightdm-gtk-greeter in Xubuntu, the lock screen can be set up to A) hide the username and B) require entering both username and password to unlock.  I was unable to achieve this with other display manager nor other desktop environment.

2) lightdm is lightweight and doesn't pull in dependencies that seem unnecessary to me for the function of a display manager.
