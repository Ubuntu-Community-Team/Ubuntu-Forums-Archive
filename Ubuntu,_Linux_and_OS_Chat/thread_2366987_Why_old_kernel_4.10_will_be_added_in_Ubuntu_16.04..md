I don't understand what you think the problem is.

It seems you are bit confused. Ubuntu always uses a slightly older kernel, but it is a supported and stable kernel. Ubuntu 16.04 originally uses the 4.4 kernel, which is old. Is that a problem to you?

What is your complaint exactly? Should Ubuntu use always the newest kernel? Why? Using the newest kernel may introduce instabilities, and you don't want that in a long term support version. You want a tested kernel, with enough hardware support, even if you miss the newest features.

Canonical does contribute directly to the original kernel development, just like many companies do. They don't develop their own kernel, they just take a snapshot of the main kernel and do the appropriate patches for Ubuntu.
