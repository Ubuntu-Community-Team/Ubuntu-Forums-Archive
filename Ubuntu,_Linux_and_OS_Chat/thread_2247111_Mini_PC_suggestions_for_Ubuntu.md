### Post by endlessinstant on 2014-10-09
As above, Intel NUCs are pretty rad.   If you do [something like this]("http://www.newegg.com/Product/Product.aspx?Item=N82E16856102055&cm_re=Intel_NUC-_-56-102-055-_-Product") you get an i5 system for dirt cheap.   That is maybe overkill as you could get by with a Celeron or Pentium for what you want but might as well to get the iSeries is its in budget for you.   iSeries integrated graphics are not incapable either which is nice if you decide to install games later on.   2 GB RAM is fine for starting, just put in more later when you get more money.   My box runs an Ivy Bridge i5, HD 4000, and 16 GB of RAM.  There is very little it can't run, though granted I have to turn visuals in games way down.   If you're not doing gaming or CPU intensive, save money and look for an i3 or maybe a Pentium.  I'd avoid a Celeron, though Haswell Celerons work well enough (especially under Linux) for basic use. 

Best speed is going to come from installing your own SSD.  You want to factor in your own RAM and storage for these Intel kits.  That is why, [this product listing]("http://www.amazon.co.uk/Intel-Compact-Barebone-SO-DIMM-DisplayPort/dp/B00HVVKRRW/ref=sr_1_2?ie=UTF8&qid=1412883043&sr=8-2&keywords=Intel+NUC") on Amazon.co.uk is showing the RAM and SSD being purchased with the kit.   If you have an SSD + 4 GB of RAM you will have a very speedy little machine.   If you need to save money initially, you can get a slower storage drive and less RAM.  Linux in general runs well on lower system requirements and you can tweak easily (run Xubuntu initially?) until you scrounge up the funds to do upgrades.

### Post by vtpoet2 on 2014-12-23
And if you tried, you would discover that you can't use Netflix on an ARM processor (unless you're using ChromOS), that Dropbox doesn't provide ARM compatibility (so no Dropbox on ARM unless its through the browser), that Wine won't work on an ARM processor (for obvious reasons) and that _**NONE**_ of your PPAs will work with an ARM-based Linux desktop installation because [askubuntu]("http://askubuntu.com/questions/181409/what-ppas-support-the-arm-architecture-armel"):

[COLOR=#333333][FONT=UbuntuRegular]**"NONE.**[/FONT][/COLOR]

[LIST]
[*]This is because Launchpad currently [only supports the i386/amd64 architectures for PPA packages.]("https://help.launchpad.net/Packaging/PPA#Supported_architectures")
[*]armel packages *can* be [built after setting a special flag]("https://bugs.launchpad.net/launchpad-buildd/+bug/379592") to use a non-virtualized build environment, but because of security reasons this access is restricted to the core Ubuntu teams.
[/LIST]
[COLOR=#333333][FONT=UbuntuRegular]
[SIZE=2][FONT=arial]That's not to say there may not be third-party repositories offering armel versions, but there are no PPAs, per se."

Unless, for smaller packages, you build the ARM deb yourself (which I've done).  I'm not new to this. :-) And you will discover that Chrome doesn't work on the ARM Linux Branch, only Chromium (and only on a good day) and Firefox is iffy, and the Ubuntu ARM repositories (as with the other distros) remains largely an afterthought (Fedora claims to be changing this but time will tell). Unity doesn't work on ARM and much compositing support is hit or miss.

So, to make a long story short, ARM Linux desktops are best left to hobbyists and tinkerers -- and I am one of them (when time permits).[/FONT][/SIZE][/FONT][/COLOR]
