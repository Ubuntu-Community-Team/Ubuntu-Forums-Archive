### Post by buzzingrobot on 2015-08-05
> **mhi2 said:**
> 
I may be wrong here, but in my view, Ubuntu has followed Windows in its desktop design.

Ubuntu did not use an interface of its own design until Unity. 

> In the early days, Ubuntu had a "start menu" similar to Windows... 

Ubuntu used the Gnome 2 environment at that point. Gnome is an independent open source project not affiliated with Canonical. Canonical adopted Gnome 2 for its primary release, just as KDE was adopted for Kubuntu, XFCE for Xubuntu and LXDE for Lubuntu.

All of those interfaces are panel-and-menu interfaces.  They are typically called "traditional' but I'd argue that's only because the computer experience of most consumers began with Windows, and they allowed themselves to think that Windows design choices set the boundaries for all interface design efforts. 

In fact, panel-and-menu interfaces were in use on other platforms before Microsoft adopted it for Windows. Remember that the first three versions of Windows were very icon-centric and, in fact, Microsoft threatened legal action against Apple when the Macintosh was released (a design that, also, was based on earlier interfaces.)

> We then saw Ubuntu move to the Unity desktop as Microsoft went with Metro.

Look at the release dates: Windows 8 was released in the fall of 2012.  Alpha versions of Unity were available in 2010, and the first release arrived in 2011. 

The Gnome project terminated Gnome 2, moving to the entirely new Gnome 3 interface in 2011. Development and maintenance of Gnome 2 ceased, making it an untenable choice for distributions. Gnome 2 had been a very popular choice for distributions.  Now, all distributions that had used Gnome 2 had to find an alternative.   Very few distributions have the resources to develop and sustain their own desktop environment.  Canonical does, being more than a simple packaging effort like most distributions, and chose to use Unity.

> I'm just wondering if there has been any thought by Canonical to phase out Unity and bring back their own start button.


I'm certain the answer is: No. A start button is a perfectly fine design choice, but it does not define computer use.   The majority of interfaces available for Linux still use a conventional panel-and-menu approach, including a "start" button that triggers a menu display. They're in the Canonical repositories and can be installed and used on Ubuntu.

### Post by buzzingrobot on 2015-08-06
> **monkeybrain20122 said:**
> The majority of Windows users probably think MicroSoft invented computing.

For most people, Microsoft was their first exposure to computing and remains their only exposure to computing.  Linux is invisible because it is not available in retail stores or from online retailers. The FOSS licensing requirements are essentially a poison pill that allows any retail Linux product to be copied and redistributed by someone else.  No one can successfully sell a Linux product in those circumstances.  If you can't sell Linux, consumers won't know it exists.

More fundamentally, all consumer PC operating systems -- Windows, OS X and Linux -- rely on a hardware architecture created and fixed in place more than 20 years ago. Hardware capabilities determine software capabilities.  No OS will be able to offer fundamentally different capabilities until that changes.
