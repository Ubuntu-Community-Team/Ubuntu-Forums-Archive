### Post by mystics on 2015-09-09
If you want to get into programming, you'll want a good text editor. Right now, my personal favorite is GitHub's Atom. It takes a lot of inspiration from the already solid Sublime Text in terms of being a good editor, but it also has good GitHub integration and is very easy to modify to your desires. It also has the Github community behind it, which gives it a solid stream of extensions and themes to expand it to your needs. Even as a relatively new text editor, I find it to be the best. Here's some info on how to easily install it on Ubuntu:

[http://itsfoss.com/install-atom-text-editor-ubuntu-1404-linux-mint-17/](http://itsfoss.com/install-atom-text-editor-ubuntu-1404-linux-mint-17/)

(Note: That article is a little outdated, but the installation instructions are still good.)

If you don't mind some proprietary text editors, a couple other good ones to look into are Sublime Text and Microsoft's Visual Studio Code. I still do prefer Atom, but all three of those are excellent and pretty much unmatched as far as I've been able to find so far.

After that, for programming itself, you could always start with Python. For a programming language, it is relatively easy, and it already comes installed on Ubuntu. Google has a decent tutorial to get you started:

[https://developers.google.com/edu/python/](https://developers.google.com/edu/python/)

Beyond that, you already have FireFox, Thunderbird, a media player, and a music player. If you want Spotify, installation instructions are here:

[https://www.spotify.com/us/download/linux/](https://www.spotify.com/us/download/linux/)

LibreOffice can help with most of what you'd need Microsoft Office for. PlayOnLinux is good if you start missing some of your Windows games, and Steam and GOG both have plenty of Linux games. Also, be sure to take advantage of the Ubuntu Software Center when you can.

If you want to learn some Linux-specific stuff beyond just what can be used to serve your daily needs, then this is a free book that, as far as I've read, gives a good overview of some command-line stuff:

[http://linuxcommand.org/tlcl.php/tlcl.php](http://linuxcommand.org/tlcl.php/tlcl.php)
