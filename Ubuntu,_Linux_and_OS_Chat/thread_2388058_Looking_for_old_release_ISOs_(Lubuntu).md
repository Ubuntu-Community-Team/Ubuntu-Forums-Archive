### Post by sudodus on 2018-03-28
1. Please be aware that it is a security hazard to run end of life releases, if the computer is connected to the internet.

2. I think the first official Lubuntu version (officially accepted as an Ubuntu community flavour) is 12.04, and this may be the reason, that it is the first version listed at

[cdimage.ubuntu.com/lubuntu/releases](http://cdimage.ubuntu.com/lubuntu/releases/)

3. I could find Lubuntu 10.04 alias 'Lucid Lynx' at the following link

[phillw.net/isos/lubuntu/lucid](http://phillw.net/isos/lubuntu/lucid/)

4. Finally, the i386 (32-bit) iso files are still produced except for Ubuntu Desktop. The Ubuntu Server and Ubuntu mini.iso as well as all the Ubuntu community flavours (Kubuntu, Lubuntu, ... Xubuntu) create amd64 (64-bit) and i386 (32-bit) iso files for the present developing version, Bionic Beaver, to be released as 18.04 LTS in April 2018. See 'Bionic Daily' right now at the [ISO testing tracker](http://iso.qa.ubuntu.com/).

### Post by sudodus on 2018-04-01
Some help with md5sums:

1. **gilir** is the developer of Lubuntu. It is very likely that his web page has correct md5sum values. See this link,

[people.ubuntu.com/~gilir/md5sum.txt](http://people.ubuntu.com/~gilir/md5sum.txt)

I downloaded the iso files you found and could verify the md5sums according to gilir's uploaded md5sum values.

```

$ ln lubuntu-10.10-desktop-i386.iso lubuntu-10.10.iso
$ <<< '098254aeb0153b10bcfce948c43a0df6  lubuntu-10.10.iso' md5sum -c
lubuntu-10.10.iso: OK

$ ln lubuntu-11.04-desktop-i386.iso lubuntu-11.04.iso
$ <<< '76e5865ce8d0d08fa9f833d781016fe3  lubuntu-11.04.iso' md5sum -c
lubuntu-11.04.iso: OK

```

2. I have saved iso files from Oct 21 2015 (for ISO-testing), and I can verify the md5sums at phillw.net ('Wily' alias 15.10).

```

$ <<< 'd88846ba3a7493cc4b792680c43ad8ef  wily-alternate-i386.iso
> 85987b5e0755822ba54f4068017b3ab0  wily-desktop-i386.iso' md5sum -c
wily-alternate-i386.iso: OK
wily-desktop-i386.iso: OK

```

3. I have saved iso files from April 29 2017 (for ISO-testing), and I can verify the md5sums at phillw.net.

```

$ <<< '3d3484845bd82e0653c7a1aeb32f5564 *lubuntu-17.04-desktop-amd64.iso
> e98bd5f4b750888bcc77f16df79bc775 *lubuntu-17.04-desktop-i386.iso' md5sum -c
lubuntu-17.04-desktop-amd64.iso: OK
lubuntu-17.04-desktop-i386.iso: OK
```
