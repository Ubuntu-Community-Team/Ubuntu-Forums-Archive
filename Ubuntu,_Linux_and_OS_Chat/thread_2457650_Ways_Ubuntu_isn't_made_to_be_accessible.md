### Post by zacnelson256 on 2021-02-06
I was eager to finally try an open OS but I regret to say I was really disappointed by how inaccessible I found it to be. I want to promote a free and secure OS, but these things prevent me from recommending it. Some of these I may have been able to resolve or did, but if I did, it required research, which a user shouldn't be required to do to have a positive experience.


---------------------------------
Wifi usb was only made for windows so for it to work I would have needed to install drivers that I need to connect to the internet, but I have no ethernet cable so I had to buy a new one. If it were possible to install it from a usb, it would make it a lot easier for people.


Takes longer  connect to my new wifi external usb
Forgets my headphones and won't recognize them or give an option to search for devices without entering sudo alsa force reload in the terminal.


mmb Scroll wheel is copy and paste instead of Ctl+C Ctrl+V and auto-scrolling can only be used if it's an option to change in the program. This effects how some programs use the alt key, liken in blender which has to reassign one of the , short cuts to shift+alt, which matters because I had to take extra time to search why and it will effect how others learn the program.


Needing to convert rpm files using the terminal using alien, can be really tedious entering each of them, especially when the files have a longer name. This should just be possible with a double click and maybe a confirmation message.


Tablet worked immediately but there was nowhere I cold find to change the settings. This matters for left handed users that have tablets with buttons on the pad and button preferences on the pen.


Scrolling on a window puts the window in front, which is often times not what I want and requires me to go down to the task bar to bring it back up


GUI for the terminal
Dragging a window to the corner then gives you an option for how to fill the other side.
I can't preview windows when putting my mouse over them in the taskbar.
Opening another instance of a program isn't an option on the toolbar.


Naming the taskbar "Whisker" isn't intuitive


No help hotline where i can talk to a person on the phone


Positives:


Starts up super fast
I like having a customizable desktop while it does make making tutorials more complicated.
Having apps organized by function and not company
Security

### Post by Tadaen_Sylvermane on 2021-02-06
Just because something is different doesn't mean it's inferior. In some cases what you are talking about isn't Linux's fault at all, rather hardware providers who don't provide drivers. You also have to remember that you are coming most likely from a Windows experience. You didn't just sit down at a Windows machine and know how to do all kinds of things, and they are far from intuitive imho. That took time. This adjustment will to. You may find that this way is better for you, but you can't expect it to just automatically work the same as what you already know. 

I saw someone's forum signature that said "Windows assumes you are an idiot. Linux makes you prove it." Linux doesn't spoon feed you. It makes you learn. If you just want a idiot box that you don't even have to think (which I don't blame you. I use Windows happily these days for some things) then stick with Windows. If you want to learn something new, and what kind of things you can easily accomplish in linux that are difficult as hell in Windows then boss it up and own it.

### Post by grahammechanical on 2021-02-07
> If it were possible to install it from a usb, it would make it a lot easier for people.

I have installed Ubuntu on my machine using a CD-Rom R/W: a DVD - R/W; and a USB memory stick. At present I have 7 installations of Linux on my machine. All of them were installed from a USB memory stick.

> Naming the taskbar "Whisker" isn't intuitive

You must be using Lubuntu. It is a long time since I installed Lubuntu even out of curiosity. I would have to spend some time getting used to the different way that Lubuntu does things when compared to Ubuntu. Linux distributions are developed by many different groups of developers. They are not controlled by one monolithic commercial organisation. Each group of developers is free to design their distribution and the utilities that are part of the distribution according to their own preferences and this includes the names they give to sections of the code base. It does take some getting use to.

> Needing to convert rpm files using the terminal using alien, can be  really tedious entering each of them, especially when the files have a  longer name. This should just be possible with a double click and maybe a  confirmation message.

You need to speak to the developers to the Alien application about this matter. There are distributions based upon Ubuntu. Lubuntu is one of them. Ubuntu itself is based upon Debian. All distributions that are ultimately built on Debian do not run rpm binary files. They run deb binary files. If you prefer the applications that come in rpm format it might be better if you installed either RedHat or a distribution that is based on RedHat. In Linux we have to the freedom to make that choice.

Because of the freedoms that we have in Linux if we think that there are ways in which an application or a distribution can be improved then we can get involved with the developers. We do not need to know how to write code to be able to do that. There are others ways to assist that might gain us credit with the programmers and they might listen to our suggestions.
