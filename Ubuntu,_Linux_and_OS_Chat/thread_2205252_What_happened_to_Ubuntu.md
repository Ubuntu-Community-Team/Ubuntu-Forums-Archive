### Post by dave2001 on 2014-02-13
I hate the Unity interface, I agree with you on that. However, the rest of you're "rant" is a bit unfair. You can still configure Ubuntu however you like, especially given that you can change you're entire desktop, unlike windows. There are several flavors of Ubuntu with very different desktops, Kubuntu, Xubuntu, Gnome Ubuntu, etc.

For a new desktop, I would suggest KDE. It is exceptionally configurable, and can be either simple or have plenty of "eye candy". Xfce would be my second choice, it's a very nicely designed desktop and has a smaller resource footprint than KDE.

Power management for laptops has improved a lot since Hardy Heron, however, it is not enabled by default in newer versions. You have a couple options:
1. enable and configure the default Ubuntu solution, called "laptop-mode-tools"
2. install another utility many people use, TLP   [http://linrunner.de/en/tlp/docs/tlp-linux-advanced-power-management.html](http://linrunner.de/en/tlp/docs/tlp-linux-advanced-power-management.html)
-- keep in mind you want to use one or the other, do not have both installed or they will conflict with each other.

### Post by buzzingrobot on 2014-03-07
> **SonWon said:**
> 

... this is 2014 so why do we still have command line tools for power management?  I can deal with it I just find this unacceptable in 2014.

For starters, for people who can't, or choose not to, run a GUI.   Command-line tools are typically faster, lighter, and more capable. If nothing else, GUI's tools are limited by space and design constraints from offering as many options as some command-line tools do, because when options are simply delimited text characters, limits to options and their combinations are unrestrained.  Do a 'man ls', a 'man mv' and a 'man cp',  look at the options and combinations available to those commonplace tools, and then design a GUI file manager that incorporates all of that while still being usable and comprehensible.

Even Windows has Powershell for the same reasons.

The Unix tradition began in a text-only environment and incorporates simple but powerful concepts like a scriptable shell, input-output redirection, pipes and tees, pattern matching, etc.  None of that can be sensibly provided in a GUI. (A number of capabilities we take for granted, e.g., spell checking, began as simple scripts leveraging those strengths.)

That Unix tradition opens up possibilities for users to perform tasks developers never considered.  GUI's close off those possibilities.
