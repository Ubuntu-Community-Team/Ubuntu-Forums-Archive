### Post by yoshii on 2015-12-12
I have Puppy Linux installed into a tiny little partition of it's own alongside Ubuntu Studio.  
It's just there so I can troubleshoot/recover my main partition if for some reason it gets corrupted.  
It has pretty much everything I need to do that and Puppy Linuxes boot up into root user so it's easier in some ways to do things.  

An added benefit is Puppy's GRUB4DOS which I use instead of GRUB2.  
It requires MBR partitions instead of GPT, but I already planned that ahead of time when I first formatted/installed both partitions.  
The nice thing about GRUB4DOS is that it's easier to edit the boot menu and it boots up to a text menu.  I like it better than GRUB2 but it's almost the same thing.  
It uses legacy GRUB commands is what i've been told.  And Linux kernel boot up procedures are generic enough that it all works.  
GRUB4DOS just has to find some files which are normally there and it boots with the kernel settings custom specified in the menu.lst file. 

The other nice thing about this type of dual boot is that if I want to surf the net without affecting my main workhorse area, it's fine using Puppy.  
I use an older version of Puppy called X-Precise.  I was able to update some things but it's still pretty old.  I might try a newer and different Puppy later on this year.  
Some of the Puppies are based upon different forms of Linux so cross-compatibility is something to notice.  But the frugal installs aren't very invasive which is nice if you need that.
