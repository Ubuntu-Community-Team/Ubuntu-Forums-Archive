### Post by coldraven on 2014-11-03
All the ones you listed derive originally from Debian. The common thread in all of them is the package manager which makes it easy to install programs with all the correct dependencies to make it work. They all have .deb packages, other versions of Linux use different methods. For example Red Hat uses .rpm packages.
The difference between all the 'buntus is the choice of the desktop environment (DE). Ubuntu uses the Unity DE, Xubuntu uses the Xfce DE and so on.
It's all about user preferences, some people like a DE that looks and behaves like Windows or Mac or whatever suits their working style.
The good news is that it is easy to try out the different DEs using a Live CD or USB stick. You can even install more than one on your computer and switch between them at the login screen but don't try that unless you have plenty of space on your drive and enjoy experimenting.

Edit: Mythbuntu is a home theatre media centre for playing movies etc.

### Post by The Cog on 2014-11-03
Coldraven is right. They all have the same base OS, but their appearance can be quite different. Their choice of apps to install by default varies a bit, too (although any app will run on any of *buntu flavour). The difference is perhaps a little like the differences between windows, XP, Vista, 7 and 8. Much the same underneath, but with a different look and feel. 

I would say that Ubuntu doesn't look like any version of windows, but maybe a bit like a Mac. Xubuntu (my preferred flavour at the moment) probably looks more like XP than win 7 or 8. Each of the flavours has its own fans and haters. 

Perhaps the best way to appreciate this is to try the live DVD/stick of one or two different flavours. You can run them straight from the DVD/stick without installing anything. Apart from being a little sluggish because of the speed of the media, they are the real thing.

### Post by grahammechanical on 2014-11-03
Remember, because Linux and the software components that make up a Linux distribution are Free and Open Source Software (FOSS) anyone can take the source code and modify it into something different. So, when a developer does not like what he gets with one distribution he takes the source code and he makes it into something else. Perhaps it is a different user interface. In some cases it is a complete distribution.

The developers have the freedom to develop in the direct of their choice  and that in turn gives us users freedom of choice.
