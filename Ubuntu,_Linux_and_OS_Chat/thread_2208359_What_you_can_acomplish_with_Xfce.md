### Post by Rob Sayer on 2014-06-01
I returned to Xubuntu and XFCE last weekend when I installed 14.04 on my 1Gb netbook.  Previously ran Lubuntu 13.10 but I find LXDE too clunky and they seem to take too long to fix bugs.  Lubuntu is definitely faster, though less features means it slows *me* down.

There's not much point in me posting a screenshow of my xubuntu.  It's too boring.  Generally the first thing I'll do after install and basic config is turning off all effects and compositing (except with KDE ... then it's the sencond thing after turning off that damn bouncing cursor).

What I'm shooting for is "works like a Mac more or less but looks like 90's Windoze".  I don't care about eye candy, I want to crank the speed up.

One thing that surprised me this time is that it seems to run every bit as fast using dolphin rather than the default file manager.  I love dolphin ... actually I won''t use anything else anymore unless necessary ... but it loads in a ton of dependencies from KDE.  If anyone else wants to try it, also install konsole and ark (which needs rar) so you have easy access to terminal and file unzipping in the open folder.  You can probably configure dolphin to use the xfce defaults but it's much simpler this way.

I love how you can mix and match software from different DE's that way in linux.  But I just have Xubuntu on here at the moment.  I've used multiple DEs on one machine before but it's not a good idea unless you seriously know what you're doing.

It's fun playing with themes like some have here but I don't bother anymore.  Besides I don't want 3rd party themes.  Or skins.  Or anything else related to something really important.  You're looking at too many potential problems.

Something newbies don't seem to understand is that if you take a lightweight DE like XFCE and LXDE and load it up with compiz and conkys is that it won't be a particularly lightweight DE anymore.

I've been looking at some conky launcher things just for the hell of it.  But I don't really care about desktop search.  And if I want a convenient app launcher I can simply create a new auto hide panel and put a launcher on it.

If you look at gOS it used computers that had ULV CPUs and only 512MB-1GB RAM. Sylvania G Netbook is a good example. Now yes, there are some which gobble up resources like some of KDE's widgets, but you can pick and choose to make it look awesome.

3rd party eye candy apps on the other hand, eh, if you're talking about themes and skins, those go off of the compositor/window manager and are just jpg, svg, or png images. So they don't really take up that much. If we look at things like compiz, yes that does take an extra amount, but that's if you really want graphically intensive items like drawing with fire. I used to use compiz on 512MB of RAM with a 64MB video card.

Personally, I believe the argument is somewhat valid, but invalid as well. It's all dependent on what people use as their "eye candy".
