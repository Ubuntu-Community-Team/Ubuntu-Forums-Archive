### Post by ajgreeny on 2016-11-15
You are correct that there are many different ways to deal with this situation.

I can not speak for other users but like many, I only install the LTS versions and I wait for the first point version of those before moving from the previous LTS to the current one, ie, I have only just gone from 14.04.5 to the current LTS 16.04.1.

I have a separate /home partition and two 20GB partitions on my disk for root partitions of OSs on my hard disk, so at present I have both OSs available to me from grub, but no Windows OS except for a VM of XP only ever used to update a TomTom Satnav GPS navigation device.

I have never updated from one OS version to another; I clean install which with the separate /home partition takes only 15 minutes or so, plus the extra time to update and add packages I need that are not installed by default.

### Post by Bucky Ball on 2016-11-16
I always use LTS release and always clean install the next one on another partition to play around with. Once I have it as I like, I migrate to that fully. 

Depends on your setup. I have all my data on one partition and all my installs share that for data so it is easy for me to do it this way (and all installs use the same /swap partition). 

Thing to remember if you are going to net upgrade: you can upgrade from LTS to LTS, say 16.04 LTS to 18.04 LTS directly, but you can't leap-frog an LTS, 16.04 LTS> 20.04 LTS. 

Going LTS to LTS upgrade means you can leap-frog the interim releases in between at least.
