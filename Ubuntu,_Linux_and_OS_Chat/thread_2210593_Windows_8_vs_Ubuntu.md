### Post by oldfred on 2014-03-11
Linux is not Windows:
 [http://linux.oneandoneis2.org/LNW.htm](http://linux.oneandoneis2.org/LNW.htm)

   Ubuntu is not Windows
[http://www.psychocats.net/ubuntucat/a-home-users-successful-migration-strategy-from-windows-to-ubuntu/](http://www.psychocats.net/ubuntucat/a-home-users-successful-migration-strategy-from-windows-to-ubuntu/)
[https://help.ubuntu.com/community/SwitchingToUbuntu/FromWindows](https://help.ubuntu.com/community/SwitchingToUbuntu/FromWindows)

Often wine does not fully run many Windows applications. I use it for Picasa, but Google worked with Wine developers to make it work.
You should long term plan on finding equivalent native Linux applications. A few are better than the Windows equivalents, many just as good and some not so good. Anything from Microsoft, Adobe or Intuit will only work in limit modes if at all.
For games, more & more are being ported to Linux, but if your favorite game is not native to Windows do not expect it to work or work well.

We use terminal in forums as the underlying operating system is the same for all flavors of Ubuntu. To try and tell you to click on icons and do things is very difficult to explain. Much easier to have you copy & paste a few commands.
 But you just about do not need terminal at all to run or use system. It just is sometimes once you have learned a bit terminal is quicker & easier than drilling thru levels of menus or finding correct gui based app.

### Post by lykwydchykyn on 2014-03-11
Any migration to Linux that hinges on WINE working flawlessly usually ends in failure.  

Probably a better approach is to consider whether Ubuntu or some other distro can fill some other computing niche in your life and start there; e.g., put it on a little netbook for web surfing/note taking, etc.  

I'm told dual-booting is a kind of hairy affair now with UEFI and Windows 8, but if you can work that out it might be a reasonable approach.

Installing software from the repositories is as easy as it ever was.  Installing software some other way is pretty much what it's always been.  I guess steam is available now, so that's a plus.

As for the CLI, while there are more GUIs for things, there will always be things that can only be done via CLI.  Asking when you'll be able to do everything in a GUI is like asking when you'll be able to cook any dish in a microwave oven.

### Post by slooksterpsv on 2014-03-12
The best thing to do for Linux Gaming is:
1. Support companies that develop games for Linux - Crytek is making a Linux version of their Engine WOOT!
2. Research the WINEdb for compatability.
3. Consider that where WINE is a layer (not emulation) it won't work perfectly and some games may crash right in the middle, dual-booting.
4. See if there's a Linux version of your game - if your games are on Steam you can do this easily, otherwise generally not. You could also research and see if there is a variant like said game on Linux.
