### Post by papibe on 2013-08-12
Hi all.

A few of us are experiencing a problem with searches on [help.ubuntu.com]("https://help.ubuntu.com/").

The results are blocked because they come from mixed content (HTTP and HTTPS). See here: [How does content that isn't secure affect my safety?]("https://support.mozilla.org/en-US/kb/how-does-content-isnt-secure-affect-my-safety?as=u&utm_source=inproduct").

If you select 'Disable protection on this page', you are in fact able to see the results, but subsequent searches are blocked any way. This is very inconvenient, as you are forced to do it for every search. 

I created this bug report:  [Bug #1211375]("https://bugs.launchpad.net/ubuntu/+source/ubuntu-docs/+bug/1211375").

If you also experience this problem, I would appreciate if you add yourself as 'affecting me' (and may be also adding a comment).

Thanks in advance.

BTW: Chromium seems to be not affected by this situation.
