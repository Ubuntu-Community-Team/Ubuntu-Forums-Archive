### Post by buzzingrobot on 2013-06-07
You know, I'd prefer to see the whole LTS vs non-LTS distinction eliminated. Instead, let's have a new release that requires a new install only when there's an actual redesign, like the Gnome 2 to Gnome 3 transition.  Otherwise, just roll out update packs that are analagous to the current bumps from, say 12.10 to 13.04. And, do security patches back to the original release in the line. (Note: This is not a rolling release. You'd have 3 distinct kinds of updates coming: Security patches, upstream  and inhouse updates as now, and the, new, update packs to deliver the broader new features we see now in the non-LTS bumps.)

Ubuntu would need to fix updating so it goes from problematic to dead reliable. But, that's way overdue, here and almost everywhere in Linux.

Unfortunately, they only do minimal updates to stable after its release.)
