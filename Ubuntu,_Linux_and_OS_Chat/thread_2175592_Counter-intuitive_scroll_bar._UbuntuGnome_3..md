### Post by mike_smith2 on 2013-09-19
I've recently come back to Ubuntu after using Xubuntu for the last couple years and one thing I have noticed is the user "un-friendly" scroll UI.

If a window (Nautilus for example) is not maximized and you wish to scroll it's contents, you must first make the mouse cursor cross over the scroll bar so the scroll "handle" appears, this is not ideal but if you come from the left of the screen it's not a big issue, if you come from the right of the screen is means the user needs to first hover/cross the scroll bar then go back again (from the left) to grab the scroll "handle".

This is counter-intuitive, time consuming and bizarre.
 
If someone uses Ubuntu (or other distros with this "feature") on a daily basis they will come across this annoyance dozens of times every week, or even every day.

### Post by Copper Bezel on 2013-09-22
I exaggerated. But yes, part of the idea is that the scrollbars are simply not needed as much as they once were, because every modern computer that doesn't have a scroll wheel has touch scrolling in some form. That was part of the stated reason, along with the objective of reducing irrelevant UI clutter (which is as much substance as it is style, because readability is not an aesthetic preference.)

And it's not a *mistake*, because scrollbars are *not *needed. In the (relatively much less common) instance that the user needs to jump to a specific point in the page, the overlay scrollbars are sufficient, and the visible bar indicates the present position in the document, but for simply scrolling down while reading a document, scrollbars should *never* be necessary and really *ought* to be actively discouraged.

If you're on a computer with a trackpad that doesn't support touch scrolling, even edge scrolling, then that trackpad is broken. Either fix it, if possible, or implement whatever workaround is required, including removing the overlay scrollbars.

Yeah, I did. I was there.*

I don't use them except as an indicator of the current position in the document, which they do quite well and unobtrusively. I don't have to interact with the overlay at all.
