### Post by TheFu on 2019-02-14
Google "_Ubuntu Desktop Guide_" if you want to point-n-click.  That's about 20% of what a Unix system can do.

Google "_The Linux Command Line_" if you want to learn as much as possible and gain a deeper understanding. This will find a foundation-level book and introduce topics in the correct order.  In Unix, skills build and build and build on prior learning.  If you don't have the fundamentals down, then you will always be a point-n-click end-user.  OTOH, the more you learn, beginning with the fundamentals, then the more of the pieces to the total system will "fit" and concepts which seem unrelated will connect.  Then you'll be able to use those connections in ways that nobody can predict.

There are other well-written blogs with "Linux for Windows users" as the topics. These usually provide an overview of the key ways things are different so you can begin to unlearn things from Windows that do not apply.
