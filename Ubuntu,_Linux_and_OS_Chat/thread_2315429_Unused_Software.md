### Post by him610 on 2016-02-28
Your discussion topic should probably be moved to another forum since you seem to not have a real problem.

The only applications I ever removed are maybe some I downloaded and later decided did not suit my purpose. One never knows when one might possibly need an obscure application that was originally included in the distribution. Take editors for instance, some folks are completely satisfied with nano, leafpad, or gedit while others may prefer emacs, vim, or vi. It all depends on the individual. 

Other possibilities - replace libreoffice/openoffice with abiword and gnumeric. Remove all *office type applications and use only cloud-based applications like Google Docs. Remove all internet browsers except one.

If you are running short of disk space, you probably will not save that much by deleting appplications. You might consider a lighter distribution, ie xubuntu or lubuntu. Personally, I have never come anywhere near to filling up any of of my disks with any of the ?buntu distros including a 8GB SSD on a netbook. You might consider acquiring a larger capacity disk drive - they are really cheap now.

What's nice about Linux is that if you don't like some application because it is resource hungry or has a sparse interface, you are free to remove and replace it with something else, or not. ;)

### Post by montag dp on 2016-02-28
The best way I've found to free up disk space is to use a graphical tool that shows what is actually using it all up.  Disk Usage Analyzer (Gnome-based DEs) or Filelight (KDE) are good for this.  For example, after awhile you may get lots of log files or old versions of packages taking up space on your root directory, and these tools will reveal those things.
