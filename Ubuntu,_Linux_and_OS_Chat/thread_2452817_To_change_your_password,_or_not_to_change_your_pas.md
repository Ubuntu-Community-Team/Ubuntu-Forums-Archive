### Post by guiverc on 2020-10-28
I'll give my thoughts, but I have no special knowledge.

I don't bother with my own systems, I really rely on physical security (*who can access the hardware*), thus my password has remained pretty much the same for a long time  (*I  do change it every so often, so an LTS upgrade is probably as good a  reason as ever, and a easy clue to remind me how long ago it was last  changed!*)

On laptops or devices I move around (*not tied to my home or home-office*), I rely on encrypted disks. If I was doing a clean install (ie. format) for that, I'd possibly change the password (*as long as I can cope with another one to remember.. I've already a few*). The actual user password I'd treat the same as my home/main PC & servers.


I've replaced passwords before on disks where it had been forgotten (both in windows and GNU/Linux) by replacing the lost hashed password with a known hashed password. I've not done that recently, but I suspect nothing much has changed, and maybe it's why I put more trust into physical security OR an encrypted disk (rather than the user password itself)
