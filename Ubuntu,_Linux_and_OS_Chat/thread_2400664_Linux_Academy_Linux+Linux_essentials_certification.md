### Post by TheFu on 2018-09-08
Structured, active, learning is best.  Don't expect to watch a few videos and be ready for a Linux position.
Nothing can replace time using the OS and time spent being an admin around other admins. For Unix-like OSes, mentoring is really the only way to learn more than "type this here" knowledge.  A mentor can explain WHY, since many problems to be solved require doing things that nobody has done before.

I use some free LPI documents for my Intro to Linux and Beginning Linux User sessions. These are usually packed, standing room only.

When I've attempted to teach the 1st Linux Admin using the LPI materials, only about 2% of the attendees came with the prerequisite knowledge to be successful in the course. At the 2nd class, less than 5 people come because there ain't no point-n-click.  For every hour of class time, I expect 3 hours will be needed outside class.  That is usual for college coursework.

But my experience only includes a few hundred students.

EdX has free Linux training. There are others, but they are the most reputable that I know.

I would specifically ask the recruiter if the desktop support was for Windows or Linux.  Just don't come across Linux desktops that often.
