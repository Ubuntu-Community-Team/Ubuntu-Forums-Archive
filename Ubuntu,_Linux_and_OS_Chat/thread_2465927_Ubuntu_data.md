Is there a question?

In the 20.04 installation, there is a question about reporting bugs automatically. There are good reasons to allow this, but we each need to decide for ourselves. I choose NOT to report for a few different reasons.  There are also tools that track how often a program gets used.  Programs used all the time, by millions of people, should probably have more attention paid than an old program used by 100 people, once a year. Providing the data can help Canonical and upstream projects know how used/important a program is for how many people.

Think of how the Gnome project must feel when they see millions of people using some applications, but only 5,000 using Gnome3 ... people choosing to run KDE, LXQt, XFCE, Mate instead of Gnome3+?  That's good feedback for "improvement needed."

At least on Linux, we get to choose if anything is sent or not, unlike MSFT where no choice is provided. I'm amazed that more people didn't stop using Windows immediately.

### Post by deadflowr on 2021-08-15
Ubuntu has for a few releases done a survey based on data collected during an installation.
This is opt-in.
The information is the basics of what your system setup is, with items such as type of cpu, type of storage. etc etc.
The report is usually found  in /home/username/.cache/ubuntu-report.
It'll list any reports generated for the system.
Read more here: [https://ubuntu.com/blog/a-first-look-at-desktop-metrics]("https://ubuntu.com/blog/a-first-look-at-desktop-metrics")
and here [https://ubuntu.com/desktop/statistics]("https://ubuntu.com/desktop/statistics")

With Ubuntu we can open System Settings>Privacy tab and make some adjustments. For example, under Diagnostics we can set Send Error Reports to Canonical to be Never, Automatic or Manual. There are other options under the Privacy tab.
