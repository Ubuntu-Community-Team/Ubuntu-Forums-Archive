I use Xubuntu with the Cairo dock on the bottom to launch programs.  But it's just as easy to create a new panel on the bottom without installing Cairo, and put all your launchers there. IG . . in addition to prior pm, I want to offer a couple other insights:

a)   ALL of the official "Buntu's" (Ubuntu, Xubuntu, Lubuntu, Kubuntu, Edubuntu, Ubuntu-Mate (soon), and even 1st cousin OS's such as Mint (all flavours), Bodhi, yada, yada, yada re supported in two ways)):

1)   Security updates
2)   Bug updates/fixes.

The flow of updates is generally:   Mother Debian>Ubuntu Official Distros>Ubuntu Related Distros

b)  ONLY Ubuntu is assured of getting the new components of MIR (new display platform), Systemd (init/start manager), and Unity8

c)  ONLY Ubuntu will have integration with Ubuntu Touch-Phone-Tab-Droidbuntu-etc.

d)  Only Ubuntu will have primary access to customized toolsets and dev platforms for above, in addition to integrated cloud mgmt tools implemented by Canonical (without other changes).

SO, in a nutshell, all systems are supported, but if you are in a business environment, govt, or a systems admin, developer, etc., you are BETTER supported with Ubuntu Unity (excluding Redhat/Suse).

Note that "Gnome" is not a distro . . . it is a DE (desktop environment) using GTK2 or GTK3.     KDE is likewise a DE, using "Cute" (QT4 or QT5)

In my view, the desktop that is using current technology (GTK3 or any QT) is at the top of the lists . . . after Unity DE, Cinnamon and customized Gnome3 (GTK3) desktops look the most polished and professional, although you can make XFCE look good.    KDE desktops using the latest versions also look very polished (the OpenSuse distro does the best job there IMHO)

I've seen "Fences" demo'd a couple times at our local computer club, but it really  makes a desktop look antiquated and cluttered in my view.   The search and filtering capabilities in Unity-Dash does a much better job of organizing apps quickly, not even considering the benefits of the HUD (heads up display) for searching Application menus.

In summary, the best way to really know what works best for you is to try these various desktops - - but you don't have to do a complete install of each.    Use Virtual Box (considering you have 16 GiB ram on each machine and 8 on the Shiba).   OR, you can create a "persistent USB Flash Stick" with the various desktops.

One more quick note - - - after Unity, the Cinnamon desktop is something that needs to be looked at (which is why Mint is at the top of the charts).

PPS:   not being familian with Linux distros "yet" . . . . you should really check-out "virtual workspaces" in linux.   Again, I like the way Unity provides for a Quad-Screen there to assist your workflow.   That's why Win10 is copying Linux (and Mac) in that respect.

> [COLOR=#008000]**CE?**[/COLOR]

The main reason I fear stepping away from Ubuntu is 1) lack of support and longevity of the OS; 2) More programs are written for Ubuntu than for other flavors??


All the official Ubuntu spins -- Xubuntu, Kubuntu, Lubuntu, Ubuntu Gnome, Ubuntu Mate -- are built on the standard Ubuntu base.  The difference is the "desktop environment" used on that spin. The vast majority of the packages on each are the same, and all the packages in the standard Ubuntu repositories are available by default in each.  Any updates and security patches released by Canonical will be installed, as needed, in any of the spins in the course of normal system updates by the user. 

While there may be a few edge cases where an application written to support Ubuntu Unity does not run at all on another desktop environment, in my experience I've never seen it.

That said, applications created as part of a desktop environment typically require -- have dependencies on -- other software packages within that desktop environment.  These dependencies need to be installed (and typically will be automatically) to ensure that specific application runs and displays correctly. The number of these dependent packages can be very small or very large, or somewhere in between.

Ubuntu's default Unity interface and the Gnome 3 interface used by Ubuntu Gnome do not use the traditional panel-and-menu Windows-esque design. The other spins do. Zorin, mentioned earlier, is an independent project also based on Ubuntu and using the Ubuntu repositories that makes a deliberate effort to mimic the traditional look and feel of Windows.

As grahammechancal suggested, you can check out any of these by booting an install image and running it in live mode. (You're given that option when it boots.) You should be aware that each environment is configurable, and that you are not limited to the specific layout used by default in any of the the spins. I.e., a panel may be placed at the top or bottom of the screen by default, but easily moved to another edge of the screen, or, in fact, you can use multiple panels, or replace the default menu with another, etc.

[EDIT:  The standard Ubuntu Server install image does not include a GUI interface, and, therefore, lacks the GUI desktop applications. A specific desktop environment can be installed after the fact, along with any other applications. Likewise, server components are typically not included in Ubuntu Unity and the other spins, but can be added. Otherwise, it's all the same.]
