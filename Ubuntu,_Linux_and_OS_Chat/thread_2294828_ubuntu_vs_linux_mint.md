### Post by tgalati4 on 2015-09-13
The only possible difference that I see is that Linux Mint assigns "Levels" to updates.  Kernel and Xorg updates get Level 4 or 5, which means high risk of system breakage if you apply those updates and then suffer a regression.  For a new user, these Level 4 and 5 updates are hidden so that only applications and support libraries get updated routinely.  You have to unhide the Level 4 and 5 updates to see them and apply them.  This could be a security risk if there is a serious vulnerability in either the kernel or Xorg that needs to be patched right away.

So you have to weigh the new user's computer experience level.  If your new user has some computer experience and knows how to search the web to solve a problem, then you can unhide the Level 4 and 5 updates and apply them.  If you have a new user that is a complete noob--no real computer experience, then I would leave the default settings.  So the risk of a security issue is slightly higher for this user.

With Ubuntu, there are no Level ratings, so users can blindly apply all updates and (rarely) get a broken system because of a graphics driver or a kernel change that won't allow a particular motherboard to boot properly.  In my several years on these forums, I see this scenario a lot, but statistically, for a single user, this occurrence is rare.  

The more important selection is the Desktop Environment:  Unity vs Cinnamon, Gnome Flashback vs Unity, MATE versus XFCE, XFCE vs LXDE.  Some DE's work better with older hardware than the default Ubuntu Unity. so if installing on an older computer, you need to test the Live Session to make sure the system will boot properly and the graphics works correctly.  Putting Unity on an old computer can lead to slow response and send you right back to this forum for answers.

I have used Linux Mint MATE for several years on several desktops and laptops.  The Gnome2 interface is familiar and works fine.

### Post by Sweet_Baby_Jamie on 2015-10-31
One of the main differences, according to many comparisons I have read, is the way they are updated.  Linux Mint's updater lets you choose "levels" from rock-stable and *completely* safe and Mint-tested and proven *generally* safe, to untested but *probably* safe and high-risk.  

With the Ubuntu updater you can choose to get only security updates, and so probably avoid most updates that break things.  The Mint team has people that *actually test* other updates on different kinds of hardware and classify them by risk.  I don't know if Canonical has a testing team for updates like they do for development versions of their OS.   

As frequently as "broken after update" threads appear in these forums, I would say not many Ubuntu users select "only security updates" in the Ubuntu updater software.  I have my computer and my parents' computers set up to only get security updates since I probably wouldn't have a clue how to fix stuff broken by updates!

Not the best advice as this is not installing a desktop environment but the entire flavour. This method is equivalent to installing Xubuntu, or any other flavour, on top of the flavour you are currently using as it pulls in ALL the default applications associated with that flavour. xubuntu-desktop is NOT the desktop environment, xfce4 is! xubuntu-desktop will bring with it all the default Xubuntu apps. :|

If you have a low-end machine, this is really going to clag up your machine, possibly cause conflicts, will cause redundancy as you'll have duplicate apps that do the same thing (Nautilus from Ubuntu and Thunar from Xubuntu as file managers, then PCmanFM as well if you add lubuntu-desktop). And on it goes. A mess. Don't go there. 

If you're after just the desktop environments, just try the desktop environments. There's plenty of them. They'll drag in a small amount of bits and pieces, perhaps an app or two, there may be a conflict on the odd occasion (there are a million possible combinations and therefore possible clashes), but they are easily rectified. Dragging out a *buntu-desktop is not. 

xfce4, lxde, lxle, cinnamon, mate ... the list goes on. See how you go. But avoid installing *buntu-desktop unless you are aware of and happy with the consequences. :)
