Lots of great advice above.

Nothing will answer your questions as quickly as running each of the OSes will. 

As a coder, I'm sure you have plenty of bandwidth.  Find a repo with all the distros and setup a **wget** job to grab each of the ones you are interested in.  Don't worry about the pre-installed applications,. You can install almost any app on any mainstream Linux distro trivially.  Then you can use a multi-boot flash drive to load up 5, 10, 15 different ditros and TRY THEM OUT without risking anything on the machine.  Any system with 6G of RAM can handle all the different GUIs, so it really comes down to preferences.  On Linux, the GUI is just another program - is it NOT the OS.  Swap them in and out as you like. Customize any of them as much as you like, except Unity.  You can make almost any Linux DE (desktop environment) look like anything you want. Linux is nothing, if not highly, highly, highly, flexible.

Try:
[LIST]
[*]Ubuntu-Mate
[*]Kubuntu - kde version
[*]Lubuntu - LXDE version
[*]Xubuntu - XFCE version
[/LIST]
To get started.  I'm surprised people didn't mention Mate - it is a fairly friendly UI with settings sorta like control panel.  Just know that if you only use the GUI, you are leaving 90% of the power untouched - basically, it is like Windows.

If you are really lazy, there are thousands of youtube videos with people showing off their Linux desktops.

Eclipse for C++ devel?  You are the first C++ coder I've met ---- ever ---- who uses that.  I used vim and 3 xterms for my IDE the decade I wrote C++ code. I actually don't use any DE. Prefer the simplicity of a pure WM setup. No menus. No bloat. Enough GUI to do what is necessary, but stay out of the way.  My desktops have 1.5G and 4G of RAM.  If I need performance, there are servers for that, but since Linux is much lighter when there isn't any GUI at all, most servers only need 512MB-1GB of RAM unless the program(s) running are highly bloated or handling many thousands of users.

As someone knew to linux and coming from Windows, there are a few short articles that will help you understand some critical Linux distro ideas that are VERY different from Windows.  Search for "*Linux for Windows Users*"  and "*Linux is not Windows*" to find the 1 most helpful articles.

Should also say that my very old mother used lxde happily for years. Oddly, she couldn't tell the difference between a Pentium4 CPU and a Core i7 CPU running the same OS.
