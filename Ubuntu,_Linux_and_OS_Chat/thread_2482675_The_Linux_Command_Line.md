### Post by The Cog on 2023-01-07
I've read most if not all of it. You might find the beginning a bit slow because it really starts from very basics, but read it through anyway. It's very well written and it's good to follow the author's train of thought. Its style makes it easy to re-read if you think a bit needs refreshing.

The later chapters can be treated as separate subjects and it's OK to jump ahead if there's a need to read up on a particular subject, but it's all worth reading at least once.

Don't try to memorise it all at the first read. Remember the interesting bits, make mental notes of other bits that you may want to return to later. First time through, you are trying to get a general flavour, not swotting up on every fine detail. Enjoy the learning.

I used it as the text for a Beginning Linux class a few years ago.  We covered 1 chapter each week.  It is a good introduction. Those first steps, the ways that different aspects of the OS neatly click together, presented in the correct order, so knowledge builds on prior knowledge is important. Linux and Unix aren't like the 2 other major OSes.  Jumping into a GUI guide clicking on  A, B, C, <Ok> and done doesn't work. As the student sees the different connections between permissions, users, groups  and how they all actually fit, is fun.  Over the next few years, more and more connections will come to you that aren't specifically covered in any book, but have to work in certain ways based on the underlying security and permissions model that is shared by all Unix-like OSes.
