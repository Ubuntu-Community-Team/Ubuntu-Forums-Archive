### Post by buzzingrobot on 2014-08-15
Using the desktop as a launchpad for icons and shortcuts is primarily a Windows-oriented design choice.  In Windows, with no multiple virtual workspaces and no docks or similar features, users are left with constantly minimizing windows to the panel so they can create desktop space to open/use another application.   They have only a single workspace, so everything is open on that single workspace, with the inevitable mess of windows overlaying and obscuring other windows.

The effective way to use Unity, or any similar interface, is *not* to clutter up a single workspace.  Rather, take advantage of the availabilty of multiple workspaces. You don't need to minimize windows that are obscuring other windows.

Personally, I seldom see the desktop, so it's pointless to place icons on it. That's been my style for years.
