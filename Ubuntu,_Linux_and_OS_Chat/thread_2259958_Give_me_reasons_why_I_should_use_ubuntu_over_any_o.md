### Post by PondPuppy on 2015-01-08
As TheFu wrote, no distro is perfect. All have good points. No particular reason to convince you of anything, really. 

It depends on what you do with your desktop. There are specialist distros for graphic artists, multimedia, penetration testing -- as you probably know, these have somewhat different pre-installed software selections, and may have specialty hardware drivers included. But most of these packages and drivers can be installed on general-purpose Ubuntu or Mint computers if you want.

Two advantages of Ubuntu-*based* distros is the extensive software repositories available, and the frequency with which the system is updated. After the recent Bash-bug kerfluffle, patches for Ubuntu and Mint came out very fast. To be fair, patches came out almost as fast for the Arch- and Slackware-based distros I use. 

But for a general-purpose desktop PC, a common recommendation is to start with Mint. The desktop is easy to use, it shares Ubuntu's package manager and software repositories, it's stable and popular. 

Choose whatever you feel most comfortable with.

> **remmas-sido said:**
> I want a distro for desktop

That is awfully vague.  What do you want to do with it?  

Of course, you realize that Linux systems can be trivially customized, so changing distros really isn't needed.  Heck, I add a window manager to Ubuntu Server and use that as my desktop.  This keeps it light, dumps the panels, other crap, keeps the noobs off my box (no menus).  People looking to point-n-click on my system will be disappointed.  Point-n-click is just too slow.  OTOH, a GUI is just a way to have more xterms open for me.  Any distro can be customized.

If you want someone else's distro to have everything you want, you will never be happy.  Perhaps you should make your own, custom, distro?  [http://www.linuxfromscratch.org/](http://www.linuxfromscratch.org/) will get you started.

Or perhaps you should load gentoo if you really want to **learn linux**.  I ran that for a few months until all the compiling got in the way of being productive.

### Post by zombifier25 on 2015-01-12
Well, instead of telling you to "play around", I'll tell you some reasons why you should use Ubuntu.

1. It is the most well known and widely used, so there's more support, software, hardware compatibility etc.

2. It's easy to use and works out of the box (but then, the same can be said of various other distros)

That's it actually. On second thought, maybe you should play around. Other distros have other advantages as well as subtle differences. Arch is must harder to set up, but also extremely lightweight, easy to config and on a rolling release model (no need to upgrade). Debian is utra stable while being mostly binary-compatible with Ubuntu. Fedora offers a solid GNOME desktop with yum as the package manager, if for some reason you hate apt.

So on and so on.
