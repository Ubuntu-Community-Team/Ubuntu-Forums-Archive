### Post by Lars Noodén on 2014-08-27
You're more likely to find guides on setting up and using the individual services you'll be adding.  What are you planning on doing with the server?  Printing queues, file sharing (NFS, SMB, AFS, etc.), web services are some of the more common options.  Things like DNS and mail are usually outsourced.

### Post by ian-weisser on 2014-08-27
One way is to jump in the sandbox and get dirty.
Open a VM, create an experimental server, and start installing the accounts and services you want onto it.
Carefully take notes on each service setup and config change.

Very quickly, you will discover your detailed questions.

When you're ready to do an operational server, your notes will tell you exactly how you did it.
