### Post by buzzingrobot on 2015-12-29
If I used Windows, most of the time I'd be using the same apps I use on Linux. 

I've had more personal exposure to Linux and OS X than to Windows, which has been a workplace tool someone else bought.  So, rather than judging Linux from a Windows frame of reference, as most people do, I tend to judge Windows the other way around. I have a bias favoring docks, virtual workspaces, and a top panel, and not favoring bottom panels, menus and lots of maximizing and minimizing.

I was in the Win10 Insider's Program because I had a machine to try it on and was curious. If I needed to run Window for my own purposes, I'd use Win10. No OS is ever going to be without bugs and annoyances. But, it makes sense to use the latest version from a vendor, especially if that's the version the vendor is spending money and resources on. If someone has a Win7/8 install they like, that's great.  But the clock is ticking.

Windows has always had the problem of selling to two very different markets.  First, the enterprise market, which is essential to Microsoft.  Then, there's the home market, which is defined by the pre-installed consumer machines. Those are focused on content consumption, gaming, etc. It seems to me many features in Win10 are intended to appeal to what Microsoft's corporate mind thinks is the consumer market.  I can see, though, that many of those features would be irrelevancies or turnoffs for enterprise users.

Font rendering quality is important to me, and it (can be) better on Linux than on Win10.  Windows users multiple rendering engines to cope with older, legacy code, while Linux has one approach. Linux users can make changes in font rendering across the system, while Windows users can't, and the methods that are available, like Cleartype, have limited capabilities and do not affect the entire system.

### Post by buzzingrobot on 2015-12-29
There is, indeed, an issue in Linux that can render new software incompatible with old distribution releases, and vice versa. However, the same issue impacts any OS that is actively developed. It certainly happens in OS X. It happens in Windows as well, although Microsoft has a record of going to extreme lengths to maintain that kind of compatibility (and paid the price). Meanwhile, look at all the Win10 updates pushed out since the July release. At least Linux users can see what's being installed on their machines.

Software is made stable by persuading as many people as possible to use, collecting their feedback, fixing the bugs they uncover, fixing the bugs and introducing only the new code needed to fix those bugs. 

Change increase the probability of bugs.

All that simply means that users and customers typically place two conflicting demands on software makers:  1.  Give us stable, bug-free software. 2.  Give us the latest software packed with all the latest features.

I've never seen any convincing data supporting the notion that paid software is better the open source software. It's usually defended with anecdotal reports, which mean much at all. (The assertion also forgets that there is a fair amount of open source software you do pay for.) In addition, we'd need a definition of "better".
