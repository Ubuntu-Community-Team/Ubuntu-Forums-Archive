### Post by tgalati4 on 2014-06-01
There is a lot of code running on modern devices, so the number of possible states that the device could be in is enormous.  Rebooting often brings the device to a single, known state, one that presumably works.  Anyone with a Windows computer knows that over time, and after installing a lot of stuff, the computer will suffer random lockups that require frequent reboots.

The only way I know of "curing" this problem is to make a simpler device--simpler hardware, simpler kernel, and simpler applications--all things that reduce the total number of software and hardware states and therefore reduce the number of dead-end states.  I never have to reboot a Jitterbug phone that my in-laws use.  I do have a problem with them remembering to either turn it on, or keep it charged.
