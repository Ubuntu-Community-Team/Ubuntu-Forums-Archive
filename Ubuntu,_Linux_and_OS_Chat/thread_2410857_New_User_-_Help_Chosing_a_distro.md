### Post by 1clue on 2019-01-22
Every distro (Including Windows or Mac) has security issues. And then a patch comes out, and it's fixed.

Every distro (Including Windows or Mac) has things you can and should do to make it more secure. That's up to the user.

In my experience, a user interface which is almost the same as Windows is harder for people to figure out than something which is completely different. A decade or two ago, the entire focus of "recruitment" to Linux was aimed at making everything look like Windows, and nobody believed it who was not already a fan. Then something interesting happened:  The iPod came out. I had lots of Windows friends get one, and then from there many of them got a mac. They never had trouble figuring it out. People are smart and they figure things out. Different is the same as new, and new is a welcome thing. Or there wouldn't be so many people who buy a different car every 2-4 years, when the car will last much longer than that.

Ubuntu is popular. Meaning, the main one. If you use the Windows key in Windows 10, then you'll be able to do the same thing in Ubuntu. If you want the control panel, you hit the windows key and type 'control panel' (fewer actually, before it shows up) and it's there. You want a word processor? Windows key and type 'word' and Libre Office shows up, if you have it installed.

Linux is not developed in a vacuum, and neither is Windows or Mac OS or anything else out there. There is a huge amount of cross-talk in all directions. It's my firm belief that frequent system updates originated with Linux, and the other operating systems had to adapt or die. Clearly the standard edit keys started with Microsoft and now even Linux uses them. So it's not like Linux will be entirely alien to you. Many of the apps work both places.

Think about it like you're buying a car. Pick a look you like, and take it for a test drive. You're really testing the forum, the window manager and the desktop. The same apps are available on pretty much any Linux.

### Post by 1clue on 2019-01-25
Every distro has some sort of statement about what that distro thinks is important. Some are community-driven, some are built, tested and managed by a commercial for-profit entity. Others are something else.

They're all using the same software. The Linux kernel is common to every Linux distribution. The kernel itself is what makes the OS be Linux. Almost everything else can run on multiple platforms.

There is usually some sort of governing entity (a council, board, blah blah blah) that makes decisions for how to do things. This includes what platforms to support, what the release cycle looks like, what sort of special focus is emphasized. That's vague, but each distro focuses on different things. Gentoo is a source-based distro which focuses on giving as much control to the user as possible, in terms of what software is installed and how it's compiled. Ubuntu is trying to be easy to use, and stable so that the controlling entity (Canonical, which is a for-profit company selling software and services) has an operating system for its customers to run their commercial products on. Of course you don't have to run non-free software.

There is also a community to consider. Ubuntu, while not being community owned, has a community in the forum. It's important to check out the community surrounding a distro and make sure you like it.

### Post by 1clue on 2019-01-28
I'll grant that the forums here are the biggest collection of Ubuntu-related expertise you're likely to find.

I don't see that the best answer would be that the OP uses Ubuntu. Again, not trying to start or continue any sort of angst.

I see a "what distro?" type of post every so often on probably every forum, and while the most frequent answer is something supported by that forum, it's not always the case.  As well, the "correct" answer is frequently not the distro served by the forum. 

Examples:
[LIST]
[*]What's a good pure firewall/security device distro?
[*]What's a good source-based distro?
[*]What's a good bleeding edge, rolling release distro?
[*]What's a good multimedia center distro?
[*]What's a good community-driven distro?
[/LIST]


While Ubuntu can do some of that, there are better answers.

There are distributions which focus specifically on security/firewalll/IDS/IPS service. Surely one of those would be easier to get rolling than Ubuntu, and if they're looking for something that works on their home wireless router then Ubuntu is not going to work anyway.

Ubuntu  is not source-based at all, so anyone answering Ubuntu on that one is  just being misleading. Yes, you can get source and compile packages yourself  with Ubuntu, but the distro is not set up to be managed that way.

Ubuntu has a rolling release branch but it's not all that bleeding edge.

Ubuntu can run everything that a multimedia center needs, but it's handier when the stuff you need is in your face when you install it. Especially if you have a small device to use as the media center.

It's a mistake to assume that, coming to the forum for distribution A, you'll only find expertise about distribution A. It's also a mistake to think that you won't be razzed for asking support questions about distribution B on the forum for distro A. 

All that said, Ubuntu can do most of what any mainstream distro can do, so Ubuntu can be the right answer most of the time.
