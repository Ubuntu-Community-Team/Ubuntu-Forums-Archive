### Post by ian-weisser on 2015-12-23
> **sergiokucera said:**
> **Be Multilanguage **- When the user visits ubuntu.com the website should automatically change to user language, if possible.

Discussed many times at many summits. Possible, but simply not worthwhile.
In the past, there were internationalized versions. The myriad local language communities were willing to translate the software, but had little interest in maintaining the website.

Ubuntu is not about advertising a product to you, and advertising Ubuntu generates no revenue.
People arrive at the Ubuntu website already expecting information on how to install a computer operating system.


> **sergiokucera said:**
> **Have an accessible front page** - The first page of the site should be the Download page for desktop, with screenshots and welcome videos. Of course, the other versions and possibilities of Ubuntu would stay on top menu.

Also discussed many times at many summits. Accessible to you may not mean accessible to others.
The purpose of the main page is to make new, unskilled users  comfortable and to start the process of answering their questions. Most new users have valid questions and concerns.
Most have never partitioned a hard drive before. Most have never installed an OS before. 
Most skilled users already know the shortcut to the Download page.


> **sergiokucera said:**
> **Be easier **- How to install Ubuntu? If user is accessing the website from Windows, there could be and easy step-by-step like:

 
Already implemented. Example: [http://www.ubuntu.com/download/desktop/create-a-usb-stick-on-windows](http://www.ubuntu.com/download/desktop/create-a-usb-stick-on-windows)

Many users are downloading on a different machine than they will be installing on. Detecting Windows or Mac is not reliable...and we do not expect new, unskilled users to know the difference. So we ask.
