### Post by buzzingrobot on 2014-04-06
> **josh-photography said:**
> Why do I have to do that command apt sudo...

Using "sudo" is equivalent to being prompted for a password in a GUI app in Ubuntu or Windows or OS X. It's the operating system's way of determining if the user is authorized to do whatever he or she is trying to do.  Typically, doing something that will change the system, like installing software.

> ...it is required to do commands like sudo/apt get-@*#&(@*#&@(*#&

It is not required. It's the approach most often used to give advice here because it is almost impossible to tell someone how to click-click-click their way through a GUI solution. Blogs and other sites also typically provide command-line instructions they expect people to blindly copy and paste and execute. 

All software included in the Ubuntu repositories can be installed via the Software Center application or any of several alternative GUI's.

> I cant even install windows apps using WINE....

If you need Windows apps, you should use Windows.  Would you install Windows and then wonder why you cannot easily run Linux or OS X applications?  WINE is a tool that allows some versions of some Windows apps to run on Linux. It is not intended to allow any and all Windows apps to run on Linux.

Take some time to *learn* about Ubuntu and do not expect it to directly mimic Windows.

Bottom line:  Bad software installation habits nurtured in the Windows  world should be left behind in the Linux world.  They are unnecessary  and risky.

Linux software is distributed in two primary packaging formats.  One, the "deb" format, is used in Debian, Ubuntu, and many other distributions.  The other, "rpm", is used in Red Hat, Fedora, OpenSuse, and many other distributions.  If packages of either format are clicked on in a distribution using that format, their contents will be installed. This is the same thing that happens when a user clicks on a Windows installation package. 

Archives using a "tar.gz" or similar file extension or *not* intended for automatic installation by users and typically contain source code files, not compiled binaries.

Many Windows users seem to form bad habits of downloading and installing software from random unvetted and untested sources. This is an obvious security threat and has a lot to do with the risk posed to anyone using Windows.

Ubuntu, and the other major distributions, on the other had, maintain tens of thousands of audited software packages in repositories accessible to their users. Microsoft has never done this. 

PPA's can be useful, if wisely selected.  But, just as is the case for any independent source, PPA software is not supported by Canonical.(Nor is the vast majority of Windows software supported by Microsoft.)   PPA software can, and often does, replace, or conflict with, core components of Ubuntu, which often creates seemingly unexplained breakage later when the users tries to do an update.  PPA's should be removed and all software installed from PPA's should be removed and reverted to any original packages it may have replaced before an in-place upgrade to a new release is attempted.  Many users fail to do this and inappropriately blame the result on Ubuntu.

The command line is useful, but not at all necessary for successful day-to-day use of Ubuntu. If the newest versions of software are not available in the Ubuntu repositories, they will not be displayed by the Software Center app and other similar tools. The user who wants that newest software has two choices:  One, wait for it to be packaged and made available for Ubuntu. Or, two, find it on an unvetted and perhaps untrustworthy site, download, attempt to install it, and assume all reponsbility for fixing anything that breaks.
