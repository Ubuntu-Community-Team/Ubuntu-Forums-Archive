### Post by Jason_Hunter on 2015-09-25
According to the desktop spec, which we all are supposed to follow, apps should put their configs inside ~/.config

Some apps do and some don't, even when installed from official repository. 

Aren't package maintainers required to follow this spec and change it when they package it if the app doesn't already do it?

---

### Post by ian-weisser on 2015-09-25
'Required' assumes both a distro spending man-hours checking for compliance, and a distro's willingness to reject software for (rather minor) noncompliance.

Distros have limited man-hours available (both paid and volunteer), so rely upon upstream developer to implement standards without coercion.

Most distros focus on compatibility, not standards, and are unlikely to reject software for noncompliance if it does not cause crashes nor conflicts.

Debian, for example, considers non-compliance with ~/.config as a non-critical bug, to be upstreamed to the developers, and will usually accept a 'won't fix' reply from those developers.
