### Post by grahammechanical on 2015-01-12
If we only have a Microsoft operating system, then we must use a program written to run on the Windows operating system. 

But you are not taking into account that people have freedom to write applications and programs. When a programmer thinks that they can do a better job then they start to write code. This is how we came to have Linux. It is also why we have a variety of Linux distributions. Each group of developers choose what applications they want in their distribution.

The comparable performance of different applications is something we often find out about through using an application. I have nothing to say about different Windows applications and their performance in burning ISO images to USB memory stick. I do not have Microsoft Windows and I have never burned an ISO image from Windows. I have burned many ISO images of Ubuntu and its flavours from Ubuntu but never a derivative such as Kali Linux.

Logically, I would not think there can be any difference. An ISO image is just data. I cannot understand why one application that will correctly burn a Ubuntu ISO image then not work for burning the ISO image of another Linux distribution. It is beyond me.
