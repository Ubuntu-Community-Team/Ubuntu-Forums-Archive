### Post by grahammechanical on 2016-03-18
Define "bug."

If someone says that a feature is missing from some application or from Ubuntu someone else will say "file a bug." Personally, I do not see feature requests as being "bugs." But I do accept that filing them as a bug report is a way for a developer to be informed of what users want from his application.

Developers themselves use bug reports to create To Do lists of work to be done which can then be checked off when the work is complete. It is also a way of communicating the progress of a project to all developers of the project. Here is an example. - Translations of 16.04

[https://translations.launchpad.net/ubuntu/xenial/+translations](https://translations.launchpad.net/ubuntu/xenial/+translations)

I do not call these "bugs" but they are tracked on a bug tracker.

[https://translations.launchpad.net/ubuntu/xenial/+imports](https://translations.launchpad.net/ubuntu/xenial/+imports)

Can I report a recurring thread as a bug" :)

Oh, by the way, does any body know what the first reported Ubuntu bug was & who reported it? Bet you cannot guess.
