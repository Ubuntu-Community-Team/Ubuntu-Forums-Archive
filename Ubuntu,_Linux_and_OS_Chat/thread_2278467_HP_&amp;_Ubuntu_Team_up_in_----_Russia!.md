### Post by d-cosner on 2015-05-17
I see other countries like Russia, Germany, Australia, China etc looking at Ubuntu and other open source projects much more seriously these days. I would love to see this same interest in the US. I am realistic though that Microsoft is not going anywhere. Steve Jobs had the right idea on this subject, we have to get past the idea that Microsoft has to loose for us to win. So many of us think that Microsoft has to loose and that kind of thinking is not getting us anywhere.

Apple is a good example because by changing their thinking they took a company that was on the brink of failure and turned it around. Steve jobs said that Apple needs to concentrate on what Apple does best and focus on the future. Ubuntu and other projects need to apply this same train of thought to be successful. Think about it, most Linux distributions are trying to match Microsoft rather than come up with new innovation and ideas.

Apple concentrated on the user experience and elegantly designed products. Apple also found new ways for users to interact with existing technology. Ubuntu is showing signs of doing the same thing, convergence, Ubuntu phones, and Snappy. If these things were to mature they might just put Ubuntu in the main stream in the US. In the US most every PC comes with Windows installed, manufacturers pay a fraction of the cost for licenses so cost is really not a factor to manufacturers.

Usability, compatibility play a huge role in the US market. Windows has been around 10 years before Linux came into existence, software companies developing for the Windows platform are long established too. The factory I used to work for still had machinery that used DOS, it worked and there was no reason for them to change it. Even new machines they purchased with newer versions of Windows ran the same applications to run the equipment. Backward compatibility is a huge consideration for big corporations.

Linus Travolds said in a speech that the thing that kept Linux off the desktop was that everything had to be rebuilt for each new release. How long can a Windows user use any particular release? Even when a release reaches end of support how much of the users software that they are used to still run in the new version? This is where Windows succeeds, backwards compatibility is a good selling point. This is not true of every piece of software but more often than not it does work.

For Linux to gain popularity in the main stream it needs to have it's own selling points for the consumer. Matching features that Windows has is not a selling point, being free as in cost does not work either because the consumer does not see the cost savings when they buy a computer. Security is a selling point to a degree but that would change if Linux was in more wide spread use, we see that now with Android phones.

For Linux to enjoy main stream success it has to offer something to the consumer they can not find elsewhere. The general pubic tends to think of Linux as a free version of Windows, they think like this because we spend too much time trying to persuade them that it is a drop in replacement for Windows. New and better ways of doing things would put Linux in the public eye in my opinion.

### Post by Old_Grey_Wolf on 2015-05-17
> **VTPoet said:**
> I'm betting on licensing deals signed by OEMS and MS in the US.

I think that does have something to do with; however, not for the reason people would think. 

In the past, with Microsoft OEM licenses your customers also get support directly from Microsoft. With Linux distros that burden falls upon the sellers of the computers; such as, HP, DELL, etc. Computer sellers are reluctant to take on that challenge and expense. Customers using Ubuntu, Red Hat Enterprise Linux, and SUSE Enterprise Linux can pay extra for their own support license; however, people in the USA don't want to pay for that, and they expect the support to come with the computer they purchase. With Linux distros, the customer has to be tech savvy enough to use a Linux forum to provide their own support, or pay extra for support.
