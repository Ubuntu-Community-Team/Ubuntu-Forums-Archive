### Post by TheFu on 2018-07-22
Unix is Unix about 90%.   Linux is Linux about 95%.
The main ways that distros differ are in administration tasks and file locations.  Different "families" of distros are closer to acting the same than others.

There are 4 major distro lines
* Redhat
* Slackware
* Debian
* Arch
Each does package management differently.  That's the main differences, but some use different configuration setups.  Networking changes have happening every few years that changes within a distro.  For me, 10.04 had the best network setup. It was the same as it had been for 20 yrs prior.  Then they (you know "them"), decided to change things to make things easier for grandmas and Windows-noobs.  They've been screwing networking ever since.  Now if you ask a networking question, you need to say exactly which release you are running for anyone to help.  17.10 and later swapped out the major networking configuration subsystem completely. Some things broke.  If it is stable, in 10 yrs, I might like it. Today, I hate it because it broke things very important to my setup.

The kernel isn't the command line. Only  kernel module and kernel setup commands actually interact directly with the kernel.  99% of what we use are user commands which have versioning for every command.  ls is ls, but different versions can have different options and different behaviors.  ls from 10 yrs ago is basically the same.  sort has changed.   du and df have new options the last 5 yrs.   Evolution is constant.   Different distros evolve at different paces.

Manjaro is a fringe distro. I've never use it.  I think you'll have more luck asking that question on the Manjaro support forums.

### Post by VMC on 2018-07-22
One thing to watch out for are 'aliases'. If you see different results between manjaro/ubuntu. Type 'alias' on command line and see if command in question shows up. For example here's a snap of mine:
```
alias l='ls -CF'
alias la='ls -A'
alias ll='ls -alF'
alias ls='ls --color=auto'
...
..*etc*..
```Some *distros* change/alter aliases inside ".bashrc" . Or they may have other installed programs that interact with bash.
I always take control of my aliases by using ".bash_aliases". The last few lines of ".bashrc" should explain this.
