### Post by monkeybrain20122 on 2015-07-03
Unity if you have more than 2 G of ram and a graphic card that supports compositing. It is modern, efficient and overall awesome. If you have weak/older hardware with less than 2 G of ram or graphic card doesn't support compositing then Xfce (xubuntu), if it is really old say with < 1 G of ram then Lxde (lubuntu) 

As for workflow, Unity uses compiz and its desktop manager and it is very flexible and can be configured to suit your workflow (multiple workspaces, hot corners, keyboard short cuts) I can do more with Unity than xfce (without compiz), there is no conflict between beauty and efficiency, one have both in Unity. :) 


I echo another poster's point earlier. Please don't steer people to "anything but Unity", Canonical puts a lot of time into it and it has gotten really good. New users should give it a try.

Yes, it's quite easy to have a situation as the one outlined. Still, a user of keyboard shortcuts has quite a few options: 
Superkey+letter or number
Ctrl+Alt+letter or number
Shift+Alt+letter or number

And if the device is a personal one, having "odd" (but logical at a personal level) keyboard shortcuts isn't much of an issue. Here's how I deal with "several applications that start with the same letter".
Super+G > google-chrome
Ctrl+Shift+G > google-chrome incog
Ctrl+Shift+M > google-chrome another profile
Ctrl+Shift+N > google-chrome yet another profile
Super+2 > gcolor**2**
Super+y > gean**y**

As for PCManFM, Super+H opens my home folder, Super+K opens my des**k**top, Super+M opens my **M**usic folder, Super+V opens my **V**ideos.

*An obvious precaution is to ensure that a "new" keyboard shortcut doesn't have side-effects.* For example, I once set Ctrl+Shift+C to something but then, later on, when I tried to copy stuff from a terminal, I felt the side-effect!

No doubt, its possible to point out flaws in any one system which is why I try not to evangelise any one DE or to rubbish someone's efforts at developing a DE.

### Post by oldrocker99 on 2015-07-07
I've been a big fan of MATE since it was introduced in Mint, a few releases ago. I find Mint to be annoying, especially when trying to reinstall Ubuntu Software Center purchases. I'd been using Lubuntu and installing MATE from the PPA. When Ubuntu MATE was announced, I went into my Linux Happy Dance.

Having started with Ubuntu 8.04, and having fallen in love with Gnome 2.x, and bitterly disappointed when it disappeared in favor of Unity, which certainly is a very advanced DE. Nonetheless, I just don't like it, especially after it replaced Gnome. Gnome 3 tried to fix what wasn't broken ("We're making Gnome better!" No, they didn't), and, while it has greatly improved (similar to KDE 4, which started a mess, and became what it should have been (not entirely the KDE devs' fault) after several releases), I still didn't like a desktop that required a raft of plugins to make it usable.

Because MATE is pretty damn lightweight, and retains the configurability I loved in Gnome 2, I've been using Ubuntu MATE on my **old** Lenovo 3000 N100, and it  runs like a top. It also runs on my desktop, and flies. 

It is not the very lightest DE:[https://flexion.org/posts/2014-03-memory-consumption-of-linux-desktop-environments.html](https://flexion.org/posts/2014-03-memory-consumption-of-linux-desktop-environments.html), but it's more useful to me than the similarly-light LXDE or XCFE.  I also appreciate the renamed Gnome-like apps; Caja is very much like the old Nautilus, Engrampa is practically identical to File Roller, Eye of MATE just like Eye of GNOME, etc.  I consider Caja to be my go-to file manager, because I just like it better.

MATE has been developed from 1.2 to 1.82, and it has been greatly improved and hasn't lost features, the way certain other DEs have (mad props to the new XCFE, I will admit, and Razor QT is pretty nice). It does boil down to which DE you like the best, and I believe I've already made my opinion known.
