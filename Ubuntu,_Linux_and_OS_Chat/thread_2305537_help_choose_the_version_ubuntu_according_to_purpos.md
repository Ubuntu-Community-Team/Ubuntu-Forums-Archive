If you want to work with servers, I would suggest starting with Ubuntu Server if you are familiar with using the CLI to enter commands. Otherwise, use a version of Ubuntu with a GUI. If needed you can install multiple versions on the same computer to see which one suits you.

Configuring the server is mostly CLI commands, and as a result, a GUI is not useful, though some may like a GUI to type in.

Use the 14.04 LTS edition, the LTS release is supported until April 2019, while Ubuntu 15.10 reaches end-of-life July 2016 next year.

### Post by Bucky Ball on 2015-12-06
*Thread moved to **Ubuntu, Linux and OS Chat**.*

There are many options, as has been suggested above. All the *buntus can be tried from the install media without changing anything on your hard drive, so perhaps that is the best route for the moment. Download the ISO, create boot/install media, boot from it and choose 'Try *buntu'. 

There is no 'live' option for Ubuntu server. As sandyd mentioned, that may be best for you. To get you started you could install server then a lightweight desktop environment. When you are feeling comfortable with things, you could start booting to a straight CLI. 

In the end, what's best for your needs only you can decide. But we can give some ideas. Good luck. :)

### Post by mastablasta on 2015-12-07
might be good if you use Ubuntu as desktop and then install virtual box to it and add the server version in virtual box. it will be like having an additional server computer at home. then you can learn how to control it, set it up, secure it, monitor it etc. the reason I suggest this is because server needs very little ram assigned to it. so you can safely assign it say 512 MB (you can go even lower)  in virtualbox and it will run just fine. best part will be you can connect to it form browser, see how it  displays the web page etc. and you can try to administer it via command line and then check what effect the changes had in browser. anyway it's really like having an extra server at home and as it doesn't take much system resources it is a safer faster way of learning the OS and trying new things. best thing about it - you can set up snapshot and mess it up completely then you can restore it within a few seconds from the old snapshot.

if you want a premade images then check Bitnami stack or  Turnkey Linux (Debian).
