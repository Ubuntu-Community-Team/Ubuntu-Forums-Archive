Gnome 2 and Gnome 3 are entirely different animals.  Gnome 3's elimination of many configuration options present in Gnome 2 was, and still is, a major source of complaint.

Gnome 2 was the dominant Linux interface for several years.  Many had tweaked it to meet their specific needs. When  it went away in 2011, they found they could not bend Gnome 3 to do what they expected to do. (It didn't help that some distros blithely replaced Gnome 2 with Gnome 3 during routine updates without making any warning noises about it.)

I think it's apparent that Gnome 2, along with most other interfaces of that era, was crafted by Linux devs for that subset of computer users called "Linux users".  Gnome 3 is very obviously intended to appeal to computer users across the board, whether they have an interest in FOSS, Linux, or not.  Ditto Unity.

### Post by ssam on 2014-11-11
MATE is a continuation of the GNOME2 desktop. The first version of MATE was just the last GNOME2 release with the name changed (this had to be done so that it could be installed along side GNOME3 without clashes). Until a few years ago GNOME2 was the default desktop for Ubuntu and many other distros. It has been used by millions of people for many years, so its reliable and has few bugs even in rare corner cases and workflows.

XFCE, was always the light weight alternative to GNOME, lighter on features and memory usage. LXDE is super lightweight. If you have less than 1GB of RAM you probably should look at them. (Though I run MATE on a raspberryPi with 512MB, but mostly as a client for ssh sessions).

If you have than 1GB of RAM you probably wont get much performance gain from XFCE or LXDE, so if you want the most featureful, well tested, traditional styled desktop give MATE a try.

### Post by stevecook on 2014-11-12
> **vasa1 said:**
> @stevecook, could you please expand LTSP? Thanks!
LTSP stand for Linux Terminal Server Project. It is a system whereby you can use one machine as a server and then connect, via ethernet (or, in my case, via powerline adapters) to a number of thin clients. A thin client is typically a machine without the need for a hard drive. It can also be pretty old spec as well because all of the work is done on the server. You boot up a thin client over the network and it loads a fully functional OS. However, since all of the work is done by the server, the thin client amounts to no more than an input/output device.  I was able ti implement this without too much trouble in Ubuntu withthe gnome shell. However, try as i might over many month. Xubuntu would never play ball. As soon as i heard about ubuntu Mate, i installed it and immediately tried to install LTSP. It installed with no problems whatsoever. If you fancy having a go with it, a good instructional is here:

[https://www.thefanclub.co.za/how-to/how-create-ubuntu-1104-x64-ltsp-server-32bit-thin-clients](https://www.thefanclub.co.za/how-to/how-create-ubuntu-1104-x64-ltsp-server-32bit-thin-clients)

Although it is listed as an instructional for Ubuntu 12.04 64 bit, it works fine as an instructional for Ubuntu Mate 32-bit as I can testify.

I remember my days at university back in the early 90s when this was the type of setup they had. However, as computer hardware dropped in price, fat clients became the norm. There is a lot to be said for terminal server networks, though, since all upgrading is done on just the one machine with all of the clients immediately benefitting from any upgrade. It also means you can keep old hardware going for far longer than is the case for fat clients.

I use LTSP in my home where I have a terminal in the bedrooms and also in the kitchen. I tend to use them for the odd bit of internet browsing plus streaming radio/tv/news etc. The fact is, though, they are fully fledged OS environments on each of the clients. User areas exist on the server and so that is where all user files are stored. You add a user account on ther server in the same way you would if it was a standalone machine. This user account is them accessible from any of the clients.

Basically, I just have LTSP cos it's cool....:)

This is an erroneous inference to make since he did not run the same applications in Lubuntu, which is the desktop environment he is comparing Mate to. Only then can one be certain that any difference in performance is due to the desktop environment. Having said all of the above, Mate is bound to run slower on an Acer One netbook as compared to Lubuntu.
