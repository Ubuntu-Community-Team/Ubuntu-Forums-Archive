### Post by grahammechanical on 2014-09-15
I also wanted to give something back for the gift of Ubuntu. I found have two ways to do that. I try to give advice on this forum and I run the development branch of the next Ubuntu release and report bugs.

You think that you do not know much but you can

1) give advice based upon your own experience
2) note the advice that others give and share that advice
3) become familiar with the commands that others suggest running and investigate the printouts of those commands to work out what all that stuff means
4) search for community documentation on the subject. I start my searches with "wiki ubuntu" followed by what I hope will be relevant other search labels
5) ask the person posting about a problem for more information. Be specific about the information they need to provide
6) be accurate about the information you are providing 

The only person who can fix a problem is the person using the machine. But we can give them a nudge in what might be the correct direction.

As for running the development branch, we have a section of this forum where we share our experiences and for the more formal approach there is this

[https://wiki.ubuntu.com/QATeam](https://wiki.ubuntu.com/QATeam)

This blog post shows that there is a need for testers.

[http://blog.luke.wf/2011/05/release-sucks.html](http://blog.luke.wf/2011/05/release-sucks.html)

This is where we get the daily builds of the ISO image

[http://iso.qa.ubuntu.com/qatracker/milestones/315/builds](http://iso.qa.ubuntu.com/qatracker/milestones/315/builds)

These are some of the test cases we can run

[http://iso.qa.ubuntu.com/qatracker/milestones/315/builds/79117/testcases](http://iso.qa.ubuntu.com/qatracker/milestones/315/builds/79117/testcases)

You can do as little or as much as you are able to.

Regards.
