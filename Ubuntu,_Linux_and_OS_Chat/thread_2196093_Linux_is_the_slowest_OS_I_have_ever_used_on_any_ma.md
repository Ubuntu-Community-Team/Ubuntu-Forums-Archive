### Post by Werne on 2013-12-28
Well, as someone who used Win7 and Win8 along with different Linux distros, there is this one little thing that a lot of people tend to forget - Linux is not Windows.

With that said, Linux boots differently from Windows as well. When comparing Windows 7 to Linux distributions, Linux will boot faster in certain conditions, Win7 will boot faster otherwise. Storage speed plays a role in boot time, but from my findings, that role is not that significant under Linux but appears very significant in Windows.

Prior to getting an SSD, I was booting from a SATA II 1TB Hitachi Deskstar hard drive. Windows 7 took 43 seconds to get to desktop (if we disregard the fact that it's still initializing services in the background) while Debian Wheezy was up and running in 32 seconds. That seems about right.

But then I got an SSD and I transferred Win7 along with Wheezy on my 120GB Kingston SSDNow V300+ and the boot time was 21 second on an AMD Phenom II X4 965 BE I used prior to an upgrade (I got a part of my system and needed a CPU), while Debian booted in 28 seconds. Not much of a difference there on Debian, only 4 seconds switching from a HDD to SSD. But, once the power states are disabled in BIOS, Windows 7 booted in 19 seconds while it took Debian 19 seconds as well.

Why such a difference? Win7 uses a single-threaded boot sequence, everything it needs to initialize is run in series on a single core. I presume cause switching from a single-core Pentium to a C2D din't affect a darn thing. On the other hand, Debian uses parallel boot where it utilizes all 4 cores, AMD has the worst pstates I've ever seen (especially Phenom II) and disabling them gives a massive performance boost. Changing I/O scheduler from cfq to deadline shaved another 4 seconds on Debian, bringing it down to 15 seconds boot time.

Once I got around to buy a new CPU (AMD FX 8320) the boot times changed further. Win7 was still booting with it's 19 seconds but Debian dropped down to 10 seconds (due to parallel boot, 8 cores boot faster than 4 cores), and that's with pstates on. Once pstates are disabled, Win7 gets down a second but Debian boot time gets halved to 5 seconds cause the CPU isn't switching between power states and runs at full speed at all times. I later modified the boot a bit so Debian boots with "performance" governor to reduce boot time then switch to "ondemand" once it's done booting, boots in 7 seconds instead of 5 but it's still pretty darn good.

Then I got a Windows 8 disc from a friend who bought a new laptop, he hated it and replaced it with Win7, so I got to play around with that thing as well. Now, the difference between Win7 and Win8 is the way it handles boot. Win8 is capable of parallel boot, albeit on only 4 cores, and it has another feature - Fast Boot. Fast Boot option is basically cheating, it use hybrid boot which is hibernation coupled with a normal boot sequence. While on a FX 8320 the parallel boot means little in the way of speed, that CPU performs marvelous once you utilize all 8 cores but not that great when you don't, Win8's hibernation does make for speedy boot though - 7 seconds. Still not as fast as Debian, but that's because it doesn't use the entire CPU. Once Fast Boot is disabled, it's only 4 seconds faster than Win7, coming at about 15 seconds.

Windows seems to boot faster from Intel processors due to their per-core performance and the fact that it can't fully utilize an octa-core. My old platform had an i5 3570K inside (before the mobo died and fried everything) and while the actual boot time wasn't affected that much, Win7 was more responsive because it couldn't utilize more than two cores for itself once it's running, so Intel's per-core performance made it faster once it actually boots.

Linux thrives on multi-core systems because Linux can take full advantage of the processor, while Windows still can't. My octa-core gets used only 50% on Win8 and 13% on Win7, only 3rd party multi-threaded software can use all of it's cores. Switching Windows between an SSD and HDD gives large performance boosts since Windows relies more on storage speed than CPU power, while Linux works the other way around.

As for OS X, that's another story entirely. Windows and Linux are made to work on a wide variety of different hardware combinations, which is why they implement a lot of firmware and drivers in order to even boot. OS X doesn't, Apple's hardware ecosystem is closed, they use a narrow range of hardware and they only need to include the drivers and firmware for their closed hardware combos. That's why Apple's products are so fast, you only need to make them work on a certain set of hardware and once optimized to run fast on it, you can't match it. Nothing can come close to OS X performance on it's native system, unless you're willing to spend hours re-compiling the Linux kernel yourself for the specific hardware combination. Comparing OS X to Linux and Windows is comparing apples to oranges (no pun intended, but fits nicely) because one works on a very narrow hardware selection while other two need to cover damn near everything, a bunch of "just in case" drivers will definitely slow a system down.

And that's why I hate when people go "My Mac boots faster than your PC". You don't say! :roll:

Anyway, those are just my personal observations, since I was actually crazy enough to measure boot times on all those in different conditions. So if you're too lazy to read all that, here's a short version - Windows gains more from an SSD than from a new CPU, Linux gains from both equally, Linux can be optimized to use an SSD and multi-core processors to speed up boot, OS X is faster than both Windows and Linux cause of little hardware it needs to support.

I think I mentioned everything. And I hope there are no typos there, I hate those.

There is a considerable amount of content on the web about fine-tuning Linux boot times for those who worry about it. (Perhaps, in your hundreds of hours studying Linux, you failed to look for it?)

I ran Macs and Linux on separate machines for years, selling my last Mac a few months ago.  I never noticed a significant difference in boot time in the two operating systems.

I know a good many people apparently seem to fetishize boot time.  But, I don't see the point unless you reboot many times each day.  Unless an update requires a reboot, my machine stays up for days on end.

### Post by Bucky Ball on 2013-12-29
There's one other issue a poster's comment reminded me of. Some people just bung Windows on a 500Gb partition and then wonder why it gets slow, or is slow. Well, Win spits data everywhere so when you boot it has to put all that back together. Because of how Linux arranges data, this is not the case with Ubuntu.

Coming from an A/V world where these things matter, I've only ever put OSs on partitions just big enough to hold them and their apps. Even in Linux I never use a partition bigger than 15Gb for the OS. Personal data stored elsewhere, of course.

So, if you have your Windows plopped in an ocean of a 500Gb partition, swap to a 40Gb partition (or smaller) so it doesn't need to throw such a wide net everytime you boot. Cuts down defragging time, too (which you should do every week ... I did). ;)

Just (another) thought ...
