### Post by Jonathan Precise on 2013-12-17
You probably have UEFI, and need to turn off fast-startup and need the 64-bit xubuntu DVD. At the grub menu, press 'e' to see a ton of text. Scroll down with the keyboard up and down buttons until you see something like:

```
 **...** ro quiet splash 
```

Change it to:

```
 **...** ro nomodeset 
```

Press F10, and the screen will blank, and fill with text. Wait. Then you will be logged in to Xubuntu. Install, then reboot. You will see the same grub menu, so do the same procedure.

To make permanent, install grub-customizer (run the following commands:

```
sudo add-apt-repository ppa:danielrichter2007/grub-customizer
sudo apt-get update
sudo apt-get install grub-customizer
```).

Run it. Enter your password. Go to the "General settings" tab. In the "kernel parameters" section, change what it says in the text box to:

```
nomodeset
```

Hit "Save". You're done! Anything happens in the way, just tell us. We'll be glad to help.

### Post by wagerrard2 on 2013-12-19
A quick Google shows some reports of similar problems on Ubuntu and Ubuntu-derivatives such as Mint and Lubuntu with the Radeon video in the OP's machine. 

Although I don't think we know which version of Fedora, Ubuntu, etc., are in play here, it is possible that the video driver used by the Ubuntu release the OP is trying to install doesn't support the 8240 (or is buggy), while Fedora does. Seeing a newer driver in Fedora wouldn't be surprising.

Using the "modeset" incantation will often allow a machine to boot in a generic video mode.  It's a sub-optimal mode, so for permanent use it isn't really satisfactory.  However, if the machine boots successfully using it, then a more appropriate driver for the 8240 can be located and installed. On Ubuntu, the "Additional Drivers" panel will display a list of available drivers.  The user can select and install one from there. ("Additional Drivers" tends to be conservative about its recommendations.  In this case, selecting the newest driver offered for the 8240 might be the better choice.)

On 12.04, I believe "Additional Drivers" is found in the "Settings" display.  It's in "Software and Updates" in later releases.

(FYI:  If "modeset" is inserted by a user during the boot process, it isn't permanent.  It needs to be used at each subsequent boot until the video problem is resolved.)

### Post by lykwydchykyn on 2013-12-19
> **bgast7851 said:**
> 
Next question...are all Linux distros pretty much the same? I guess what I mean is Linux is Linux?

They are the same in some ways, and different in others.  Linux is technically just an OS kernel, which is more or less a bundle of hardware drivers and a low-level interface for programs to talk to.  Anything based on that kernel can be called "Linux", though mostly when people say "Linux" it implies a stack of software that includes the GNU software libraries and utilities, an X11 server, and a few other common whatnots that constitute a complete OS experience.

Where they differ is mainly:

- software management systems
- some miscellaneous subsystems, like the init system, e.g.
- release schedule & development standards (e.g. stable vs. cutting-edge)
- default configuration (desktop environment, applications, themes, etc)
- target audience (business, avg users, hackers, developers, etc)

For all intents and purposes, most of the mainline distributions will allow you to run the same basic pool of software.  Fedora is generally regarded as a more "bleeding edge" distro (it's RedHat's testbed); lots of new tech gets its debut on Fedora, which is probably why it works with your new hardware.

### Post by buzzingrobot on 2013-12-19
Elementary is based on Ubuntu. Mint is based on Ubuntu. To my eyes, all your reported issues with each of these distributions point to the Live CD/DVD's failure to use an appropriate video driver.  You say they don't boot, but Elementary, in fact, booted successfully and gave you a prompt.  What it did not do is launch the X graphical display system used in Linux, which, in turn, requires a working video driver.

As outlined, use of the nomodeset option can be used to temporarily work around video driver issues.

Difficult to know why Fedora doesn't see empty space on your drives. However, Linux will not recognize empty space inside Windows partitions, which, I believe, Windows sometimes reports as distinct empty space.  What Linux, or any other OS, wants to see is space that has not been partitioned.

Pretty sure Win 7 and Win 8 don't provide admin access by default, and that the insistent password demands have been greatly reduced.

Unix was designed as a multi-user system, allowing multiple users to simultaneously use a single machine.  That's what drove the creation of users, rights, permissions, etc., that we see in Linux today, even though very few Linux machines operate in multi-user environments. Because an individual user only has rights to his or her files and directories, an account with the rights to everything on the machine was needed so an administrator could, well, admin the machine. (Remember, in multi-user mode, users typically do not have admin rights and not be able to log in as root.)

Some years after the creation of Unix, Windows was created, first as a DOS extension and then as a standalone, as a single-user OS.  No need there for a system of rights, permissions, etc., to protect one user's data from another.
