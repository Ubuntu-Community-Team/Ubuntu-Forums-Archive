### Post by montag dp on 2017-07-25
Rolling releases are kind of a two-edged sword. You always get the newest features and you never have to do a full system upgrade, but the system is always changing. That results in, at least, frequent disruptions to workflow and tools, if not bugs or major breakage. (Granted, major breakage is uncommon on rolling distros that do thorough testing before releasing updates to the public.) It can be fun for hobbyist use, but I prefer a stable release model for regular usage.
