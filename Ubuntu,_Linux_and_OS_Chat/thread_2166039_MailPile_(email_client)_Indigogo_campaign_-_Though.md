### Post by ian-weisser on 2013-08-07
> **ypBym3P said:**
> Mozilla no longer actively supporting [Thunderbird]

Mozilla is not paying developers to implement new features (on a mature platform), but continues to support Thunderbird with maintenance and security updates.

Mailpile's stated goal is to "create some cool new software and advance the state of the art of free and open source software." I simply don't see how yet another open-source e-mail client will do that. The big new feature seems to be "security", yet encrypted e-mail has been around for a very long time already.

Improved UI design is always nice, of course.

### Post by ian-weisser on 2013-08-08
Fair enough, Mozilla promised support only through early 2013. The FUD comment is cheerfully withdrawn.

Nevertheless, looking through the mozilla bug tracker and the mozilla.dev.apps.thunderbird mailing list: I see bugs getting fixed, I see plenty of  (volunteer) developer discussion, I see Mozilla employees responding to questions, reviewing code, and mentoring volunteers. That's real support.  That's maintenance. It's happening pretty closely to the way Mozilla wanted it to...and resembles the way many packages are maintained in the Ubuntuverse.

So I still disagree strongly that Thunderbird is unsupported or moribund.
