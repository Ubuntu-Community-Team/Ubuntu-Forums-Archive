### Post by vtpoet2 on 2014-12-25
I've been using Firefox for years, but it keeps the CPU running hot and burns through laptop batteries like candy. One can do a search on Firefox CPU and 'High Usage' and see that [this issue is ongoing]("https://support.mozilla.org/en-US/questions/1003966?page=1") and that Mozilla has done squat to solve it. Mozilla's kneejerk response is to blame extensions, FLASH, hardware acceleration, etc... without obtaining reliable results. I've given up on them.

Just as an FYI, I made a concerted erfort to use the latest Opera since it's release on Linux, and there's no comparison. When I minimize Opera, CPU usage invariably returns to single digits. This *never* happened with Firefox after _sustained_ usage -- FF regularly used anywhere between 65% to 95% CPU _when minimized_ -- even killing the plugin container did little to help. I can reproduce this at will on any of my systems and desktops. I've switched to Opera as my primary browser. Until Mozilla someday decides to stop sailing down the river *denial*, I highly recommend Opera to anyone else whose FF install is running hot.

I haven't tested Chrome.
