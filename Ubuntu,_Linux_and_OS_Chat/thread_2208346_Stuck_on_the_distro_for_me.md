For gaming, focus on the graphics support.  I'm not a Radeon user, but I understand that currently the open source and proprietary drivers are more or less equally capable.  I suspect it's a try it and see thing.  

> I know how to setup partitions and I have once managed to setup a working arch base install, it was not very good but I got X working. Good to understand.  Remember, though, that  the same capabilities and toolsets are available in all Linux flavors.  The user-friendly bits are implemented as layers on top of all that. 

> Something visually pleasing, I like Unity, but its not lightweight, I have often opted for XFCE, but I would like to know anyone elses suggestions on MATE or something else, I dont like GNOME.
I want something easy to use, Like Ubuntu, although I hear theres stuff like spyware installed? from amazon etc, I am unsure personally, but would Ubuntu Minimal be a valid choice?

"Visually pleasing" is entirely subjective, so go with what you like.  I find all panel-based interfaces -- XFCE, Mate, KDE, etc., much the same in appearance and functionality. 

As for the so-called spyware in Unity (it's not in the derivaties like Xubuntu, Kubuntu, etc):  Ever read a blog review of a product and clicked on a link to that product at Amazon?  Very often, if you buy that product at Amazon, the blog writer gets a tiny kickback from Amazon. That's a long-standing program called Amazon Associates.  Search is the core to finding applications, etc., on Unity.  Out of the box, when you run a local search, Unity feeds that search to an online database that returns hits on Amazon products, which are then displayed to you. (That's the same mechanism used in all the other online searches available in Unity: Wikipedia, weather, etc.)  Canonical collects these data and provides them to Amazon. If you buy one of those products, Amazon sends a bit of money to Canonical. (Other data, including crash reports, are also sent to Canoncial. 

All of that can be disabled either in System Settings or by  deleting the relevant packages following guidance widely available on the web.

Whether or not you think that is "spyware" is up to you. I don't think it is, but it is a feature I don't need and I don't like the visual clutter it produces so I eliminate it. (In any case, sites have always known and recorded in server logs which sites visitors were on when they clicked through on a link.)


  > that does not kill battery life, I can always research on how to improve battery life, as I expect it to out perform Windows 8 in all aspects.

That may be iffy. Linux does not have a reputation as delivering better battery life than Windows.

As suggested earlier, boot and check out the live images of distributions that interest you. (Better on a USB stick; CD/DVD's are an annoyingly slowway to run live images). That won't get at the battery and Radeon issues, since they will be set up for a generic approach, but it should help with the look-and-feel things.
