### Post by TheFu on 2020-04-23
> **jebi said:**
> Hi,
 
Just downloaded 20.04, anyone tell me any reason to not use it yet?  Is it stable? Is it secure?

thx

Security cannot be proclaimed. Only time can prove whether any software is secure or not. There are thousands of different project teams all writing software and only a few of them have the resources to follow secure coding best practices AND to test every possible way the code might be used. It just isn't practical.

Stability is a very personal thing. It depends on your normal use, the hardware, the programs, and workflows you need.  Nobody else has quite the same way of working or the same hardware you do.

Try it. See how it goes for a few days.  Perhaps load it into a virtual machine to limit risks. You might love it and want to commit.  If you are more risk adverse like me, I'd wait until the first round of bugs get fixed, probably until mid-June.  So thousands and thousands of other users will help to find the big bugs before we get involved.

If you are really risk adverse, wait until 20.04.1 is released.  That is what many businesses do. They wait. Of course, if you are running 19.10 today, you'll have to move to 20.04 before July to stay on a supported release. 19.10 support ends July.

People running servers will need to look at the software their servers run. For example, my communications server probably won't be supported on 20.04 for about a year as the company behind it isn't ever in any hurry to support a new LTS.  Servers running normal Ubuntu supported stuff should be reasonably ok now and a little better in June, then a little better every month the next 18 months or so, before 22.04 development kicks off and 20.04 becomes nearly forgotten by the dev teams.  See, they want the new hotness, which is fine. People like me are change adverse, since change means risks.  Heck, most of my systems still run 16.04. It has only been the last month that a few 18.04 boxes have been spun up on VMs.  I'm not ready to commit to 18.04 on a VM server. 

Be we each have to find our on level of risk.
