### Post by yancek on 2018-11-23
There are some significant changes in the most recent long term release of Ubuntu (18.04) so it would be a good idea to indicate which release of Ubuntu you are using.  Generally, one installs software on a Linux system from the repositories on servers created and maintained by the developers of the operating system rather than downloading willy nilly from unknown sites on the internet.  You should have an icon showing the Software Manager, or if you are using 18.04, click on the Activities tab and start typing and you should see an icon for whatever you are looking for.  Click on it.  You don't really need to know 'where' it is installed if you are just a general user.

You need to be more specific about what you mean by opening files on an external device.  What kind of external device?  What's on it"  What filesystem are you using on it?
With regard to the Python problem, you need to specify what you did and what the warning/error message was.

If you install a program through your Software Manager or from the Ubuntu repositories, you can find out where it is by running either of the commands below in a terminal.

```
which nameofprogram
whereis nameofprogram
```

Enter the actual program name of course.

Running Windows hasn't prepared you for any other OS.  Linux is not Windows and expecting things to work the same will only lead to frustration.  People used to other OSes who move to Windows face a similar level of frustration, trust me.

At the core, Linux is Unix-like and that means it was designed for concurrent, multiple users to share a system.  This is why Linux systems have file and directory permissions that matter much more than they do on Windows.

If you are trying to install programs on Linux in the same way you installed programs on Windows, that is all wrong.

So ... the first steps you might want to take is an overview of Linux from a Windows perspective.  [http://www.dedoimedo.com/computers/ultimate-linux-guide-for-windows-users.html](http://www.dedoimedo.com/computers/ultimate-linux-guide-for-windows-users.html) is a good start.

Then gaining an understanding for how the F/LOSS movement works, compared to having a monolithic, commercial, company selling software, would be useful next: [http://linux.oneandoneis2.org/LNW.htm](http://linux.oneandoneis2.org/LNW.htm)

But I won't lie. The learning curve for Linux is steep to get a real understanding of the OS. However, for a typical end-user, it is possible to point-n-click your way for years.
