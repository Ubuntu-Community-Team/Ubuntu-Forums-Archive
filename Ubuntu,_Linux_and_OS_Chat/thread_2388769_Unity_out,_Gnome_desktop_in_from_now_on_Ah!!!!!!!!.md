### Post by vanadium on 2018-04-07
Unfortunatelly, search for files does not work in Gnome. Stock Gnome uses the resource hogging "tracker" daemon to implement full text search of your files. The initial index understandingly takes a long time, but even when the indexing is complete, you will face several minutes of fan sound as your four cores are employed by tracker to to an update. This will not affect the speed of the system, because the process is "niced" not to bog down your interaction with the OS.

Even then, a simple name search would not work. I have a file with the same name in MS Word format and in Libreoffice format. Gnome shell would only find one of these files, the word version.

Ubuntu wisely does not ship with tracker. As such, finding files through the shell essentially is not operational. It occasionally finds something, but it is entirely unproductive. For example, right now, I am searching a folder one level below Documents. It for the heck does not find that folder, yet presents some other folder two levels deep while the first characters are typed. ???. Fortunatelly, nautilus search nowadays is fast and quite good, so that does make up for it. Besides, I am using the quick launcher albert, and name based file search there works like champ. The indexing takes 0.2 seconds and it needs a mere 150 MB of RAM. 

That said, I am a gnome user. Nevertheless, I cannot understand why some aspects, which should be basic in a modern desktop, keep sucking so much in a dot plus twenty version! (searching, window switching and keyboard support in the overview, etc).

At first I have been positive about the change, I started using 17.10 for some time. It looked good and user friendly. But now I reconsider, I believe Gnome has been growing too complex and complexity in software has well known disadvantages, like:
[LIST]
[*]more source code lines and more complex constructs, resulting unavoidably in more bugs.
[*]the memory usage has almost been doubled (420MB now 910GB).
[*]the boot time has almost been doubled (32 secs now 56 secs).
[/LIST]
I have been using Ubuntu since 2008, first in dual boot with Windows Vista and after retirement as sole OS. I only have been disloyal during the first release of Unity using Xubuntu, not because I disliked Unity, but because the video HW of my 2011 PC did not work with the first Unity release.

Now I will abandon Ubuntu, because I believe, it is getting too big and too complex. Both memory usage and boot time are comparable to Windows. Besides I want to run 2-3 virtual machines and the zfs file system in 8GB memory, so memory size is really critical for me too. 
I will switch to an user friendly Xubuntu supplemented with plank dock (30 secs and 280MB), but I will still keep Ubuntu 16.04 as my banking VM/OS.
