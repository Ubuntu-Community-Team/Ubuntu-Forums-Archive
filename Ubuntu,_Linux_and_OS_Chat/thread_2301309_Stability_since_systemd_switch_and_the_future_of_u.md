### Post by grahammechanical on 2015-11-01
When Ubuntu switched to Unity there were accusations of fracturing Linux development. When Debian switched from using Ubuntu's Upstart as an init system to using Redhat's systemd a decision was quickly made that Ubuntu should switch also to systemd. I have yet to see anyone come on this forum saying "well done, Canonical for not fracturing Linux development even further." 

I am at the moment running the 16.04 development branch every day. I did the same for 15.10, 15.04, 14.10, 14.04, 13.10, 13.04. I do not see poorly maintained packages. I see development code that is boringly stable. Individual developers who put their own code into the Ubuntu archive have the responsibility for maintaining their own code. 

Every Ubuntu product, desktop, server, core, phone, snappy core is built from an archive of debian packages. The code relevant to Ubuntu is maintained by Ubuntu developers and the code in the archive relevant to the other Ubuntu family members is also maintained by the particular developers concerned.

You complain that there is instability and you also complain that there is a lack of "cutting edge." As with Gnome switching to Gnome 3 and now Debian switching to systemd Ubuntu developers are having to keep up with developments in other software projects as well as pursuing Canonical's own Commercial vision. I think that they are doing a very good job.

Regards.

### Post by Mike_Walsh on 2015-11-03
My Linux knowledge, with regard to Ubuntu especially, only extends back to the original release of 'Trusty' (14.04 LTS). This much I do know; since the point releases have appeared.....first 14.04.1, then 2 and now 3, I have discovered successive problems solved one after another.

This, to the point where I am now able to run Xubuntu 14.04.3 on a very elderly Dell laptop with a truly awful Intel graphics adapter (from the days when Intel hadn't really sussed out graphics architecture, and were still feeling their way). To begin with, I couldn't even get a display at all. Then it progressed to a small display squashed into the top left corner. And now, with 14.04.3, the system will run completely normally from the point of first install.....without the need to employ any 'workarounds' at all.

I have no idea whether systemd has had any part to play in this, or whether this is simply the natural progression of a build-up of known problems slowly & steadily being addressed. Personally, I don't care if my OS uses init or systemd, so long as it allows me to extend the lifetime of, and to continue to enjoy using, hardware that is, to all intents & purposes, still fully functional.

Whatever the answer to this is, I myself say 'Well done' to all involved, at whatever level. Thanks!
