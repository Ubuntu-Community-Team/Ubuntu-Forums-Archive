### Post by grahammechanical on 2022-09-04
Have you asked this question on an OpenSuse user forum? What did they decide?

All Linux distributions are similar. They use the Linux kernel and it is the kernel that handles hardware compatibility. This is why it is always best to use the latest version of a Linux distribution with the latest, most modern hardware. Even then, it will take a short while for the Linux developers to make the kernel compatible with the most recent hardware.

As for stability and performance these conditions are more affected by the state of the hardware in the machine than the Linux distribution. Old hardware with a low powered video adapter and a small amount of system memory will not run the latest versions of any Linux distribution very well. A spinning disc hard drive is much slower than a solid state drive and the new Non Volatile Memory express (NVMe) cards are faster by degrees than the spinning disc hard drives. The OS will boot seconds faster.

Finally, every Linux distribution is a mixture of different software projects. Those same software projects can be used by all Linux distributions. So, at a very basic level each Linux distribution is similar to every other Linux distribution.

We have choice when we choose Free and Open Source Software. So, differences of opinion are a matter of differences of choice. They are not something for users of FOSS to argue over. Please do not promote arguments by asking this sort of question.

As a user of Debian, Ubuntu, OpenSuSE, and Fedora.... I personally think they're all ~equal.

To be honest, my *favorite* is actually Debian, but Ubuntu and Debian are ~equal and so I probably treat them as the same. For servers I usually opt for Debian, but for Desktop use I think Ubuntu wins out; but I'll decide based on what I'm going to use the box for, having boxes with both & using both every day.

OpenSuSE has less software availability than Debian/Ubuntu, but I've always found it a *polished, reliable & clean* OS from when I started using it at 9.*something* through to whatever version my last Leap system was (*I now only use tumbleweed**, no longer being a leap user*). I used openSuSE before I'd ever tried Ubuntu, it being my 2nd distribution before I ever considered Ubuntu (*Debian was & always has been my first*).

- less software generally for openSuSE  (*but in some business cases it has some advantages, but more losses than wins*)

- openSuSE *stable* (Leap) has longer lives than non-LTS Ubuntu releases, but shorter than Ubuntu's LTS releases. There are *pros* and *cons* with the support life, but to me it's a pretty important/critical point in making a decision (*including the consideration of your use-case & software you'll be using, but you gave no specifics*)

- openSuSE has a smaller community for help than Ubuntu does, but do you need support? or is it time-critical, as both have good support in my opinion.

- in all other regards; they're pretty identical, as they're both GNU/Linux systems, with the packages coming from the same upstream sources, with just some *timing* differences between them, with some entering openSuSE faster, but more packages make their way into Ubuntu because of upstream Debian.

Probably as important in this opinion; my favorite ice-cream *flavor* is chocolate.. Choosing a GNU/Linux distribution with as little information as your provided is about as meaningful question as asking which is the best ice-cream *flavor* ([I]given you gave no specifics that I'd use to decide if I'd install Debian, Ubuntu, Fedora, or OpenSuSE (let alone the varies choices of each; namely Leap/tumbleweed/.. for openSuSE)
