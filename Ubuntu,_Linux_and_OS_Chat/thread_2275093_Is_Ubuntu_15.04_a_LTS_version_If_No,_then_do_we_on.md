### Post by sudodus on 2015-04-24
The versions released in April even years have been and are long time releases (and I think will be), so 8.04, 10.04, 12.04, 14.04, (16.04). So according to this schedule 15.04 is not a long time release.

So far there are two point releases of 14.04 LTS and I expect there will be a point release 14.04.3 with the kernel and hardware enablement stack from Vivid. It will probably be released after some months of debugging of 15.04. So improvements that are well tested will appear in new iso files of the long time release. This is the idea of the point releases.

On the other hand, if you run 14.04 LTS in your computer, and it works well, I recommend that you continue running it as the production system. Upgrading may cause problems. If you want to try the new version, do it with live, booted from a DVD or USB drive, or make a dual boot system.

### Post by grahammechanical on 2015-04-24
> Why can't they make it LTS,

Cost. All Ubuntu releases have to be maintained and that has a cost in the sense that someone has to do the work, whether paid or unpaid. In the past Ubuntu LTS releases got 5 years support if it was a server edition and 3 years support if it was a desktop edition. And the Interim (Standard) releases got 18 months support.

Now, Ubuntu LTS release get 5 years support for both server and desktop editions. Which is a good deal. But to balance things out a bit the standard releases are now limited to 9 months support. which is not so bad. It means we have at least 3 months to upgrade to the next standard release.

So, our choices are a) to install an LTS and upgrade in 2 years at the earliest or 5 years at the latest. b) Install a standard release and upgrade every 6 - 9 months.

This is a good deal considering we get all this free of charge.
