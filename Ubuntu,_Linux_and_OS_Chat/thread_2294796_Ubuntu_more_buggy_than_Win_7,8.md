### Post by buzzingrobot on 2015-09-14
It's impossible to say with any accuracy if Windows is more or less buggy than Ubuntu. Too many variables are at play.  You'd need to run each on identical machines on an identical network performing identical functions with identical monitoring software.

Subjectively, though, I doubt you'd notice a difference. Don't let that concern drive your decision.

 Obvious bugs that affect users across the board are usually patched rather quickly in Windows and Ubuntu and OS X.  (That's not to say fast enough to keep people happy.)  Bugs that only affect a few people with certain hardware/software combinations take longer to patch. Since Linux has a much smaller number of users, collecting accurate user data about those bugs may be more difficult.

All software has bugs, and many of them will never be noticed by most users and will never be fixed.  Operating systems like Windows and Ubuntu are incredibly complex, with tens of millions of lines of code.

### Post by Geoffrey_Arndt on 2015-09-14
In my view, Ubuntu OS and it's main supporting pieces are not especially buggy, but you really can't compare the OS's that way (and be accurate).   

Each OS has many bugs,  BUT, each OS also has inherent engineering and design benefits/flaws. Consider windows historical lack of default file permissions,and althought each OS can (one way or the other) manage authentication and access control, which offers more security and flexibility?  Which OS requires less middleware and 3rd party control protocols?   If one follows all the official rules and procedures . . . which OS is actually more stable?    

All of the above presumes each OS is installed and running in hardware designed and/or configured for it.  And correctly installed.
