### Post by TheFu on 2017-08-14
IMHO.

Adobe Flash is about a 1 out of 10 for security.

HTML5 with all the extensions is a 2 or 3 out of 10 for security - mainly because they added some things that undermine privacy in the name of "features."

Popular browsers are about a 4  - my favorite browser, lynx, is only a 7 for security, but it doesn't support many things that website makes incorrectly assume everyone has.

So ... in the end, HTML5 is significantly more secure (not as easily hacked) when compared to the total security failure of Flash.  Adobe inherited most of the security issues for Flash from the prior owner.

BTW, Adobe Acrobat has huge security issues too.  My stance is that if you don't make a living with Adobe tools, don't use them. They are simply too dangerous.  I made my living with PDF document creation, indexing, and publishing for over 3 yrs, BTW.  I haven't touched any Adobe stuff since around 2002.  It is much easier to avoid now, thanks to iOS refushing flash support.

### Post by TheFu on 2017-08-14
Both Foxit and SumatraPDF have had serious security problems - probably because they tried to implement all the PDF standards, which are poorly designed and never should have been allowed in the PDF specifications.  

It is best to use PDF readers that don't support those non-secure features and stick with simple HTML for all publications unless PDF (i.e. page layout), really is mandatory.  It is usually a false idea that page layout is necessary, BTW.  It is only needed if the pages will be placed onto physical media - paper, billboards, etc.   It isn't needed for anything electronic.

Avoid creating PDF files.
