### Post by DuckHook on 2013-11-24
Your question is a bit like asking why the sky is blue. Just because the question can be simply phrased doesn't mean that a commensurately simple answer exists. I'll try to summarize what I think are the main points.

Firstly, the phrase "operating system" is inherently vague, and one cannot understand how difficult it is to define this phrase without keeping this shortcoming in mind. That said, *any* operating system is roughly a collection of software routines and utilities that mediate between the user and the hardware by translating the hardware's impossibly dense and obscure machine language into much friendlier commands and output. It does this by grouping lengthy sets of hardware processes into commands and utilities and then assigning these commands and utilities short friendly names. By typing in the name of what looks like a simple command, we start a cascade of hardware routines that perform a mind-boggling number of functions under the hood. However, "friendly" is a relative term, and though the resulting commands are far far easier to use than the underlying machine code, they are still much too obscure and low-level for the general user.

This is where the term "operating system" gets really vague. Most computer professionals consider the "operating system" to be complete at this point, and anything beyond it, like the graphical desktop, or even command-line applications like text editors, to not be a part of the "operating system". But general users think of an "operating system" as inclusive of all the pretty desktops, mouse actions and even applications like web browsers, because stopping at a stark command-line environment is every bit as useless to them as having no operating system at all.

This vagueness is worsened by the fact that even the tribal elders of the computing world cannot agree among themselves on just where to draw the line. Many consider the Linux kernel to be the "operating system". Others insist on including all of the utilities, libraries and applications developed by the GNU organization. This actually leads to some very heated feelings within the Linux community. If the demigods of the community cannot even agree, it's no wonder that we mere mortals would find the phrase confusing.

The best way to think of an operating system, then, is as an inherently vague concept: something like a "book". Books range from things that are clearly books, like "Treasure Island" to things that can only be questionably classified under that label, like "comic books"; but the phrase is not so vague as to render itself meaningless. A loaf of bread is clearly not a "book". Likewise, a computer game is clearly not part of the "operating system".

### Post by lykwydchykyn on 2013-11-26
> **jw4165 said:**
> 
From what I understand a technical user feels the the operating system is complete when a computer user can utilise computer hardware using commands on a pc for a given task, and a modern flashy desktop with applications simply exists to be user-friendly.


I'd put it more like this:  it's technically an operating system if the system can operate in some way, regardless of whether a user can interface with commands, etc.  In the non-technical sense, it's an operating system when it looks more or less like one of the products that have been sold as an "operating system".

In other words, there's the actual thing, and there's the product based around the thing.  

For example, if you say you want to buy "a computer", that can technically be fulfilled by 

- an abacus, 
- a pocket calculator, 
- a smartphone, 
- a mainframe, or
- any number of kitchen appliances that feature a microprocessor circuit.  

But colloquially when we talk about "a computer" we usually want some kind of smallish device with a screen, keyboard, and mouse, with an operating system of some kind that provides us with a modern desktop interface.  That's the sort of packaged *product* we're used to calling "computer".
