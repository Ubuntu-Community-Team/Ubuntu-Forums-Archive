### Post by mp6453 on 2014-07-18
Ok, a quick background here, and pull up a chair if you have time to read this.  I've been building my own (Windows) computers for years, I have an electronics degree, and consider myself to be tech savvy.  I understand computer hardware and software.  I'm also a self-proclaimed geek, and I have no problem researching problems for hours on end, which I have done several times in trying to get Ubuntu installed and (properly) running.

Yes, I see the hundreds of places to get information on various problems.  My gripe though, is there doesn't seem to be many sources for *complete* newbies to the world of linux that both answers the problems I'm having and in language I can understand.  Problems with the installation/operation of linux abound, and too many assumptions are made in the community about the installer's knowledge.  Case in point, five minutes into the install process I get a popup telling me I didn't create a swap partition.  What the heck is that?  Why didn't the installer just do it?  I then had to spend a while googling it.  Ok, I'll give it 4 gigs or so.  Now I'm asked where to mount it.  Huh?  Back to google.  Other times for other problems I've read "Just sudo it".  What the heck is sudo?  Ok, so I finally figure out the correct syntax for sudo, then it asks me for a password.  "User not in the sudo list"  Huh?  How do I add my user to that list?  Search for yourself, you'll find countless ways to do this, none of them made sense to this newbie and none of them worked for me.


So I finally get Ubuntu installed, and this thing is SLOW SLOW SLOW.  Slow as in 3 seconds for each transition when opening/closing windows and such.  I can hardly use the machine in this state.  Note, this is a dual boot system with windows, and for the hardware I'm using it's pretty snappy with windows xp (this box has dual Xenon processors).  The community says to load the proprietary video drivers...  ok, done.  Now I boot into a black screen after the login prompt.  Back to google.  Hours spent getting back to the generic drivers, then trying various "fixes" suggested in different message boards.  Found one that appeared to be knowledgeable, followed the steps.  This created a new problem, now the thing won't even boot to the login prompt, despite going into recovery mode and trying a few things first.  Hours more googling/troubleshooting.  Now I'm reloading software again from scratch.


Later I read that Unity is very resource intensive and maybe that's why it's so slow (despite not having propriety drivers yet), and that Gnome is a good alternative.  Off to the software manager, found Gnome, installed, piece of cake.  Nothing looks different.  How do you switch?  Back to Google.  Ahhh, change at the login prompt.  Gnome classic froze upon logging in. Reboot.  Chose a different Gnome.  Screen went black and stayed that way.  Reboot the machine, nothing but a black screen.  Second reboot, now I can boot into Unity again.  Still SLOW.  Collectively, I've probably spent around a dozen hours trying to get this box working.  Not mentioned here was the time spent trying to boot from a usb drive for installation with my hardware that doesn't support it.


I REALLY want to learn Linux, but I'm about to give up completely.  Yes, I'm learning things with each step (that's how I learned dos and windows over the years), but sooner or later things just need to work at a basic functional level.  So I'm reaching out to the community to ask if there's a linux guru out there that wouldn't mind lending a hand with their knowledge.  An email tech support person of sorts.  I'll ask specific (and hopefully intelligent) questions, and this person wouldn't mind typing out a thoughtful response.  There may be several of these exchanges.  In return, I'll help build the positive linux karma out there and assist others once I have the ability to do so.  I know I'm not entitled to anything that isn't available to everyone else such as the message boards, but I know there's a lot of really helpful people out there, and maybe one of them has the time to answer a few messages.  After all, it's the community here that I'm talking now that keeps Linux alive.


Thanks for reading this dissertation.


Mike
