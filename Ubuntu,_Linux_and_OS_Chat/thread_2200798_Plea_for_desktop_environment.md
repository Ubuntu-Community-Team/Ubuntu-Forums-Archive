Right.  Mate is the Gnome 2 code that was the basis of Ubuntu 10.04, with some bug fixes, some name changes, and some accommodations to allow for runnng contemporary software.  Anyone who wants to stick with that interface is well advised to  use it.  

Mate installs and runs well on Ubuntu.  Best bet is to follow the install instructions at the Mate site:  mate-desktop.org.

Another very viable candidate is the Mate version of Linux Mint.  The current release -- 16 -- is very nice.

Gnome is, and always has  been, an independent project.  Canonical adopted Gnome 2 as the primary interfce for Ubuntu, as did many other distributions.  The Gnome team stopped working on Gnome 2 in 2011, leaving Ubuntu and all other distributions using Gnome 2 in search of a replacement. 

Given the nature of open source -- no real way for users to control or constrain developer choices --  these things will happen.

### Post by buzzingrobot on 2014-01-21
> **emma3 said:**
> 
Response to BuzzingRobot - "Given the nature of open source -- no real way for users to control or constrain developer choices --  these things will happen." - that seems a serious downside, and my feelings are that whilst it is an independent organization, there's no reason for a lack of professional behavior.

Open source and free software was created by and for developers.  The situation in closed source, commercial software is much the same, although customers can exert an influence because the vendors are in it for the money. Still, vendors drop commercial software all the time, for all kinds of reasons.  Those reasons can include technical issues, e.g., the software's inability to support new hardware, work with new libraries, etc.  And, too, developer boredom and lack of interest. Gnome 2 was an aging software collection that's its developers didn't feel was an easy fit in this decade's software environment. So, they walked away from it and began something new. The Mate project picked up the Gnome 2 code base and is currently working on adapting it to the current environment while keeping the same interface.  It is not a trifling piece of work and their success probably depends on the volunteer resources available to them, like almost all Linux projects.

A lot of people, including me, like Gnome 2. But, FOSS developers get nothing from users except an ego boost.  There's no real exchange going on, professional or otherwise, to obligate either community to the other. 


> ... putting something on the market for free...


There is *no* market here.  A market requires merchants selling and customers buying.  The "free" you refer to -- no cost -- is secondary here.  The "free" that's the foundation of open source and free software is the freedom of developers, and anyone else for that matter, to share the source code for their work.  The licenses that govern the release of FOSS, the GPL and others, mandate that source code be available to anyone who acquires the binaries produced from the source.  Nothing in the license prohibits the sale, at any price, of FOSS products.  The fact that it rarely happens is down to culture and expectations, not FOSS principles or legal requirements. While it's nice to get stuff at no cost, FOSS wasn't created and doesn't exist for that purpose.
