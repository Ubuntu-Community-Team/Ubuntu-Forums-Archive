### Post by buzzingrobot on 2014-03-18
The source for RHEL is freely available on their servers. What the subscription fee gets you is access to the binaries, updates and fixes, and support from RH. 

CentOS takes the RHEL source, removes the RH trademarks and other bits identifying it as RHEL, rebuilds  and releases. A few other minor differences exist that are outined at the CentOS site.

CentOS is apparently fine as a freebie learning tool for RHEL.  However, the updates come from CentOS, so you can't gain any experience using Red Hat's updating and patch service.

The current version of RHEL is 6.5. Ditto CentOS. (From the desktop point of view, it's Gnome 2.28 from 2011, an old python and an old GTK2, etc.) 

A beta of RHEL 7 is out there, built on a Fedora 19 base and using Gnome 3 as its desktop. I don't believe CentOS plans to do a CentOS 7 beta.

Fedora is sponsored by RH and, among other things, serves as something of a platform for developing new tools for RHEL. Roughly, where RHEL is deliberately very conservative, Fedora is very innovative. Currently, RHEL 6.5 and Fedora 20 are very different operating systems that share a set of basic tools.

Red Hat and, hence Fedora, are adamantly open source.

### Post by SeijiSensei on 2014-03-18
All my servers and the machines I build for clients run CentOS. Updates are available pretty much as soon as RedHat releases the source RPMs.  I don't know what buzzingrobot means about RedHat's "update and patch service," but updating CentOS just relies on the normal yum client that all RedHat-flavored distros use.  When I was using branded RedHat, I updated servers in the same way.  I've used RedHat or its derivatives on servers since the mid 1990s when it first appeared.  The introduction of RPM, RedHat's package management system, was a strong incentive to drop Slackware and move to RH.

I've also tested another RedHat respin, Scientific Linux.  I didn't find it markedly superior to CentOS, so I've stuck with that.

If you want a "pretty" distribution to run on a desktop machine, CentOS out-of-the-box probably isn't the best choice.  For that I'd probably choose Fedora.  But if you want to learn about how to manage servers and server applications, I'd go with CentOS.
