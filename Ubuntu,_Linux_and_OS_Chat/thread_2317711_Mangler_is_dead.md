### Post by YAOMTC on 2016-03-18
Mangler is a third-party unofficial [Ventrilo](https://en.wikipedia.org/wiki/Ventrilo) client for Linux.

[list=*]
[*]The website, [mangler.org](http://www.mangler.org/), no longer exists and WHOIS gives no result, meaning there is no longer any "upstream". Here's the last time the Internet Archive crawled the page: [September 7th](https://web.archive.org/web/20150907045404/http://www.mangler.org/) (over six months ago). I contacted their most recent hosting provider, RootBSD, but they haven't responded.
[*]The last update, 1.2.5, [was in 2013](https://web.archive.org/web/20150908081620/http://www.mangler.org/2013/01/mangler-1-2-5-is-now-available/).
[*][There has been a confirmed bug causing crashes since 2012](https://bugs.launchpad.net/ubuntu/+source/mangler/+bug/1059950) with no known resolution. I do have a crash every day I use it, though it takes a few hours for me.
[*][There has been a confirmed bug making the server list unusable since October](https://bugs.launchpad.net/ubuntu/+source/mangler/+bug/1507232) (five months ago), and though someone created a patch for it at the end of last year... Well, you just have to patch it yourself, since upstream doesn't exist anymore.
[*][The Android app](https://play.google.com/store/apps/details?id=org.mangler.android) hasn't been updated since 2011.
[/list]
Does anyone have any interest in taking over this project? The original developers don't care about it anymore. But then, nor do the developers of Ventrilo, apparently -- they haven't had an update since 2008. They do maintain their website though, at least...

There is a [Github mirror]("https://github.com/econnell/mangler") (where the Android souce code can be found too), and there is a mirror of the last source tarball available on the Debian servers here:

[http://ftp.debian.org/debian/pool/main/m/mangler/mangler_1.2.5.orig.tar.gz](http://ftp.debian.org/debian/pool/main/m/mangler/mangler_1.2.5.orig.tar.gz)

If anyone wants to keep the project from being dead forever, it really needs a place to serve as the new "upstream". Should the aforementioned Github be it?
