### Post by howefield on 2016-08-08
What version of Opera do you have, the current version is .. ?

```
Version information
Version:	39.0.2256.48 - Opera is up to date
Update stream:	Stable
System:	Ubuntu 16.04.1 LTS (x86_64; Unity)
```

To fix the Opera error, this might be enough..

```
sudo apt-key adv --keyserver keyserver.ubuntu.com --recv-keys F6D61D45
```

You should also disable the precise partners repository..

```
sudo apt edit-sources
```

select option 2 (nano) and scroll down to the third from bottom line, the one that includes "*deb [http://archive.canonical.com/ubuntu](http://archive.canonical.com/ubuntu) precise partner*" and preface it with a # sign.

Save and close out with.. Ctrl +o, enter and then Ctrl +x. Try the sudo apt update again - does it still give you errors ?

```
uname -a
```

### Post by howefield on 2016-08-09
Ok, glad you got it sorted.

Seems that the repository/authentication key changed for the opera repository since you used it and putting the current key in did the trick. Although it won't do any harm by being there, you can also remove the repository from your sources by using the Software & Updates application, Other Software tab, highlight the opera line and click on remove, likewise with the Authentication tab for the Opera repository keys (you may now have 2 of them).

As I say it won't do any harm, but for me, it seems of little benefit to query repositories that you aren't going to use and are trivial to add again if needed, I'd say the same for any other unused repositories that you have. Just some basic housekeeping :)

Having disabled the Precise Partners repository, you might want to enable the Xenial Partners repository using the same Software & Updates application. Worth also pointing out that you can completely remove the Precise line, putting a # sign comments the line out as if it weren't there so effectively does the same thing.

Thanks for the output of the uname command, just wanted to double check that you were using 64 bit, was initially confused by the reference to 32 bit in the sources.list. However, doesn't appear to be a problem.
