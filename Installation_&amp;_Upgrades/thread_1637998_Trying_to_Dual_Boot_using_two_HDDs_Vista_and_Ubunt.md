### Post by nevertooloud on 2010-12-05
I got this (ubuntu on one HDD, Windows 7 on the other) to work, but had some tricks to pull.

Trick 1) When I installed Windows 7, I got messages saying "Could not write to boot partition" and the install would fails. This was a totally empty HDD, and the install was letting me create partitions on the HDD, but kept generating this error.  I eventually found that if I physically disconnected (unplugged it) the HDD with Ubuntu installed, then the Windows 7 install worked. 

At this point if I plugged in the ubuntu drive ubuntu would boot and if I just had the windows HDD plugged in windows would boot. Next I needed to set up grub to select which OS to boot into. I plugged in t 

I then booted into ubuntu and ran
```
sudo grub-mkconfig
```This generated a /boot/grub/grub.cfg file grub2 uses to dual boot.
I reboot the PC and got the grub boot menu, and it had Windows 7 on it. 
But when I selected the Windows 7 option, I got an error indicating the UUID was invalid. 

Trick 2) I found a command that lists the UUIDs for all partitions,
```
sudo blkid
```This lists all the partitions and their UUID.
I found the one fro the Windows 7 boot area, LABEL="System Reserved" TYPE="ntfs", grabed that UUID and manually edit the /boot/grub/grub.cfg file to use this UUID. 

I know that grub.cfg file is 'automatically generated, but grub-mkconfig was putting an invalid UUID for windows 7. The UUID it was using was not even in the output of blkid.

This took a couple hours of my life last night to figure out. I tell my wife I am working on crossword puzzles.
