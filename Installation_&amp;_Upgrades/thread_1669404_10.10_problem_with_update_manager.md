### Post by plucky on 2011-01-18
> E: Problem parsing dependency Depends
E: Error occurred while processing libcupsdriver1 (NewVersion2)
E: Problem with MergeList /var/lib/apt/lists/us.archive.ubuntu.com_ubuntu_dists_maverick_main_b inary-i386_Packages
E: The package lists or status file could not be parsed or opened.

You could try opening **Synaptic Package Manager** and look for the "Broken Packages" and fix libcupsdriver1,which is the package that seems to be broken.

Or 

From a terminal ```
sudo apt-get clean
sudo apt-get install -f
```

The first command will clear out all the downloaded files in /var/cache/apt/archives so,if you don't want to lose those files don't run the command.It is so that apt-get will download a fresh copy of the libcupsdriver1 .deb file incase the first one is corrupt.

The second command should attempt to fix the broken package.

Good Luck

### Post by uyulala on 2011-02-03
You may want to look into the file update-manager doesn't like:  

/var/lib/apt/lists/us.archive.ubuntu.com_ubuntu_dists_maverick_main_b inary-i386_Packages

Does it look like a lot of information about packages, or something completely different?

This error happend to me because update-manager tried to download the list while I was connected to a wireless hotspot which presents a login page to users, like the type you often find in hotels and airports. The "list" stored in /var/lib/apt was in fact a html login page.

It doesnt hurt to try to replace thee broken list: Remove the list of /var/lib/apt/lists and get a new copy, 

$ sudo rm -f /var/lib/apt/lists/us.archive.ubuntu.com_ubuntu_dists_maverick_main_b inary-i386_Packages
$ sudo apt-get update


Regards,
