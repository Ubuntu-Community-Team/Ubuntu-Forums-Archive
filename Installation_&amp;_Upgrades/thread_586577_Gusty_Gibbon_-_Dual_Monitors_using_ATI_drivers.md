### Post by one0 on 2007-10-22
OK, so here is what worked for me. I got irritated with Xinerama's cursor behavior problems as as I posted elsewhere, so I decided to try and get the native ATI dual mode behavior I had set up in Fiesty, and after some tweaking, it worked. 

First a note on my hardware:
I have a FireGL x3300 with dual DVI ports
I have two identical Samsung 204t monitors, each capable of 1600x1200@60Hz

I did a fresh install of Gusty - there were some flickering issues with the live cd that were resolved by changing the default horizontal sync in the screen resolution control application, then I installed the proprietary ATI driver through the restricted drivers manager which fixed the flickering. 

From there, I used my old xorg.conf file from feisty and modified it to look like this:

************* FILE ABOVE


You will need to adjust this file to match your monitors, resolutions, possibly PCI locations, etc. 

You know you likely have it right when X starts and brings you to the login and both monitors are presenting a spanned screen that your mouse can move between (though the one on the right is just a solid color). 

When you log in, you will be reverted to a mirrored display. Go to your screen resolutions application (under preferences) and you should now have an option to increase the resolution (in my case 3200x1200). Restart X if you need to by logging out/in. 

You should then have an expanded desktop.

I already went through the process to install the graphic effects and am using xserver-xgl, which doesn't keep the monitors as separate as I like (panels expand across the whole screen, but enables all the visual goodies)... I ended up not liking this, and uninstalled xgl which has me back to the behavior of feisty. 


Your mileage may vary on this technique.
