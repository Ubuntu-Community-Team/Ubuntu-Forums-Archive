### Post by oldfred on 2015-04-26
Do not turn UEFI on. 
Your Windows install is BIOS only. Typical 100MB boot and main install. 
Windows in UEFI mode on gpt drive has an efi partition, a system reserved partition and main install.

Windows only boots from MBR(msdos) partitioned drives with BIOS boot mode. If you convert to UEFI you also convert drive to gpt and then your current Windows is erased.
Windows only boots from gpt partitioned drives with UEFI.

I am glad to see that Boot-Repair even though booted in UEFI mode still is suggesting a BIOS grub reinstall to MBR as default fix.  I thought it might try to convert to UEFI which would be wrong.
But to be on safe side I would always boot installer or Boot-Repair in same mode as your install. Or better to be in BIOS boot mode for live installer.

You can run autofix, but install in MBR and rest of install looks ok. But if you already are getting grub menu and can boot Windows from grub menu, grub is working and you are past what Boot-Repair can fix.

What brand, model system and what video card/chip?
Can you boot recovery mode or second Ubuntu entry in menu. May be in sub-menu.
