### Post by caljohnsmith on 2009-02-19
I think it would help to first get a clearer picture of your setup related to booting, so how about downloading the **Boot Info Script** to the Live CD desktop:

[https://sourceforge.net/projects/bootinfoscript/](https://sourceforge.net/projects/bootinfoscript/)

Then open a terminal (Applications > Accessories > Terminal) and do:
```
sudo bash ~/Desktop/boot_info_script*.sh
```
That will create a "RESULTS.txt" file in the same directory from where the script is run, namely your desktop; please copy/paste the contents of the RESULTS.txt file to your next post, highlight the copied text, and click the pound/hash sign "#" graphic button in the Ubuntu forum message box so that the text will get "code" tags put around it. The results of that script will help clarify your setup and hopefully what the solution to your booting problem might be.

### Post by caljohnsmith on 2009-02-19
It looks like you might have problems with your BIOS recognizing the Ubuntu sdb drive, because a Grub error 21 is "selected disk does not exist"; even though Grub is installed to the MBR of your sda drive and points to the 2nd boot drive, you shouldn't get that Grub error since you only have two drives. How about trying this:
```

sudo apt-get install lilo
sudo lilo -M /dev/sda mbr
sudo grub
grub> root (hd1,0)
grub> makeactive
grub> setup (hd1)
grub> quit

```
Then reboot, set your BIOS to boot the sdb drive, and let me know exactly what happens. We can work from there.
