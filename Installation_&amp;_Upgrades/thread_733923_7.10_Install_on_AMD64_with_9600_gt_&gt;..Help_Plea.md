### Post by alconn on 2008-03-26
Dont Panic! I have the answer! Same thing happens with my 8800gts.

Simple solution. At cd boot options press F6 to get boot options. Delete the bit that says "quiet nosplash" and then boot. Simple!

You wont get a graphical boot loader though and you will have to change menu.lst in /boot/grub/ as well (delete quiet nosplash again) once you have installed.

Easiest way to avoid all these problems is to use the 32 bit version though!

Hope this helps.

### Post by Neo0351 on 2008-03-26
I have a 9600gt and I booted the liveCD in graphics safe mode.  It might take a few tries, if not use the alt. install CD.  After you are done installing, install the updates, then you will need to install the 171.06 drivers from here [ftp://download.nvidia.com/XFree86/Linux-x86_64/171.06](ftp://download.nvidia.com/XFree86/Linux-x86_64/171.06)

then make sure you have all these replacing 2.6.22 with your kernel

Code:

uname -r
sudo apt-get install linux-headers-'uname -r'
sudo apt-get install linux-restricted-modules-'uname -r'
sudo apt-get install linux-source-2.6.22 build-essential

Then ctrl+alt+F1 and log in as root
Code:

killall gdm
chmod a+x /location/to/driver
sh /location/to/driver

After installed

Code:

nano /etc/default/linux-restricted-modules-common

Last line should look like

Code:

DISABLED_MODULES="nv"

Reboot and if all goes well you will be up and running.  I also have a 1440x900 and it's working great.  Hope this helps.
