### Post by caljohnsmith on 2009-01-30
It looks like you might have run an older Boot Info Script that did not give complete results for some reason. How about downloading the latest Boot Info Script from [here]("https://sourceforge.net/projects/bootinfoscript/") to your Ubuntu Live CD desktop, open a terminal (Applications > Accessories > Terminal) and do:
```
sudo bash ~/Desktop/boot_info_script*.sh
```
That will create a "RESULTS.txt" file in the same directory from where the script is run, namely your desktop; please copy/paste the contents of that file to your next post, highlight the copied text, and click the pound sign # graphic in the Ubuntu forum message box so that the text will get "code" tags put around it. The results of that script will help clarify your setup and hopefully what the best solution to your booting problem might be.

When you specify "/dev/sda", that is actually the same as (hd0), it's just different notation; Linux uses /dev/sda, whereas Grub uses (hd0). The reason why I suggested that is because having the installer fail at 94% with a Grub error is a known bug that can sometimes be circumvented by simply choosing "/dev/sda" rather than (hd0), even though they are the same drive. But let's try one last thing before we capitulate and do the manual approach: first make sure your other drive is disconnected again, also make sure to choose /dev/sda under the advanced settings, but this time format your Ubuntu partition to ext2 instead of ext3. That is another method of getting around a 94% fatal install error that I know of. How about giving that a shot, and if it doesn't work, we can install Grub manually.
