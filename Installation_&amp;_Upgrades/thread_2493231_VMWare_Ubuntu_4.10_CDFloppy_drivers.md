I'm sure there must be a story behind the requirement right? And it's not going to be exposed to the outside world (connected to the Internet) right? 

I think this is more a question about virtualization, and how to set up a VM guest VM Environment for a target OS... Specifically VMware vSphere, ESXI. Workstation, Player, Fusion...

Hint: in 20 October 2004, when Ubuntu Warty Warthog released, think of what hardware was available (BIOS Legacy Boot), and replicate that hardware... CPU (Intel Pentium 4, AMD Athlon FX60, RAM at round 2GB, Types of controllers (IDE, SCSI), etc.

The types of storage controller choices for VMware vSphere VMs are SCSI, IDE, AHCI SATA, and NVM Express (NVMe). The choices within SCSI are  BusLogic Parallel, LSI Logic Parallel, LSI Logic SAS, VMware Paravirtual SCSI... Bus Logic Parallel is considered as a legacy controller. IDE is considered the oldest, and choice for older VM OS'es...

For each controller you use in vSphere, it has to have an OS driver there for it in the OS, so you choose a type that was supported by the Guest OS.
