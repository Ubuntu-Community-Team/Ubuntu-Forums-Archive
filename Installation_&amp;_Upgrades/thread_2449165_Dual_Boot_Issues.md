### Post by oldfred on 2020-08-21
You have newer UEFI hardware (Z390?) and Ubuntu installed in UEFI boot mode.

But you have Windows installed in the now 35 year old BIOS boot mode on MBR partitioned drive.

UEFI & BIOS are not compatible.
Once you start booting in one mode from UEFI boot menu you cannot switch. Or grub can only boot other installs in same boot mode.

How you boot install media UEFI or BIOS, is then how it installs for both Windows & Ubuntu.

Windows only boots in UEFI mode from gpt and only in BIOS mode from MBR. And conversion of drive during install will erase it. There are tools to convert MBR to gpt, but really should only be used on a data drive.

BIOS & UEFI Windows partitions, note system has totally different format  & meaning between BIOS & UEFI
[https://msdn.microsoft.com/en-us/library/windows/hardware/dn898504%28v=vs.85%29.aspx](https://msdn.microsoft.com/en-us/library/windows/hardware/dn898504%28v=vs.85%29.aspx) & 
[https://docs.microsoft.com/en-us/windows-hardware/manufacture/desktop/configure-uefigpt-based-hard-drive-partitions#RecommendedPartitionConfigurations](https://docs.microsoft.com/en-us/windows-hardware/manufacture/desktop/configure-uefigpt-based-hard-drive-partitions#RecommendedPartitionConfigurations)
[https://msdn.microsoft.com/en-us/library/windows/hardware/dn898510%28v=vs.85%29.aspx#RecommendedPartitionConfigurations](https://msdn.microsoft.com/en-us/library/windows/hardware/dn898510%28v=vs.85%29.aspx#RecommendedPartitionConfigurations)

You can always directly boot from UEFI boot menu, (f8, f10 or f12?) with your motherboard's UEFI menu. Some prefer to do that rather than use grub's menu.

Without total reinstall, you can add a tiny 1 or 2MB unformatted partition with bios_grub flag anywhere within the first 2TiB of the gpt drive and install the BIOS version of grub and uninstall the UEFI version of grub. That only replaces grub not all of Ubuntu.

But you should always have good backups of your system, so when drive fails or user makes an error, you can easily reinstall Ubuntu and restore configuration from your backups.

Since new hardware that is UEFI, you should consider re-install of Windows in UEFI boot mode.

### Post by oldfred on 2020-08-21
It looks like you have an UEFI boot of Windows in sde and Ubuntu installed its UEFI boot files into that ESP?
Not really sure how you did that, but Ubiquity installer seems to have a mind of its own. It normally installs to first ESP found, but gives errors and will not install to an ESP on my sdb even when normally told to. There are work arounds, but one is just to use Boot-Repair to manually install grub to correct ESP.

I would change FAT32 partition on sdd2 to ESP by changing boot flags to boot/esp with gparted.
And Boot Ubuntu live installer in UEFI mode, add Boot-Repair and install grub in UEFI mode using install in sdd3 and ESP in sdd2. You need to use Boot-Repair's advanced mode as auto fix with multiple drives often does not offer correct solution.
Boot-Repair Advanced mode menus
[https://sourceforge.net/p/boot-repair/home/Home/](https://sourceforge.net/p/boot-repair/home/Home/)

You are missing the entry of Ubuntu in motherboard's UEFI. Normal UEFI install of grub adds that using efibootmgr.
You can separately see UEFI boot entries with
sudo efibootmgr -v

You may be able to use efibootmgr to add entry using Ubuntu .efi boot files in sde4, but probably better to have Ubuntu ESP on same drive as Ubuntu install.
I like to have an ESP on every gpt drive, even my larger flash drives primarily used for data backup, but often have a smaller full Ubuntu install, just for emergency boot or testing alternatives.

### Post by oldfred on 2020-08-21
If you open up gparted and right click on sdd2, it should show you various boot flag options.
You need to do this first, for Boot-Repair to then offer it as an ESP install option.

Then in Boot-Repair's advanced mode choose install and choose drive/partition to install grub2's UEFI boot files into. Be sure that you have booted live installer in UEFI mode. You can first run gparted then add Boot-Repair.

Shows installer with screen shots. Both BIOS purple accessibility screen & UEFI black grub menu screen
 [https://help.ubuntu.com/community/UEFI](https://help.ubuntu.com/community/UEFI)
