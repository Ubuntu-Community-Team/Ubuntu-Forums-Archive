### Post by SeijiSensei on 2014-04-19
Let's try rebuilding your KDE configuration first and see if that helps.

If you cannot log in at all, try this approach.

1)  At boot, place the cursor on the Kubuntu entry and hit enter.  Now choose the "recovery mode" entry for the most recent kernel; it should be second in the list.

2)  You'll see lots of information appear on the screen followed by a menu.  Choose "root shell."

3)  The hard drive is marked read-only in this mode, so you'll need to run
```
mount -o remount,rw /
```
Note there is no space after the comma.

4)  Now we'll move your old KDE installation out of the way:
```

cd /home/username
mv -f .kde .kde.old

```

5)  Now type "exit" at the prompt.  You'll be returned to the original menu.  Choose "resume".  You should now get the default KDE desktop.
