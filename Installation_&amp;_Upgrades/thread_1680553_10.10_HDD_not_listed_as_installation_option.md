### Post by Quackers on 2011-02-02
Ok thanks. I must say I'm a little concerned about the raid status.
If Windows will only boot when raid is set in the bios it suggests that raid is being used, or was at the time of Windows installation.
When raid is used, certain raid metadata can be written to the disc and this data can stop some programs from recognising the hard disc, which is what the installer is doing. Normally, this metadata can be deleted with a dmraid command, which fixes the problem. However, if you use that command when raid is actually being used, you will lose any operating system on the disc (or at least with raid0).
The fact that Windows will not boot without selecting raid in the bios is suspicious, to me.

### Post by ronparent on 2011-02-04
This may be a quagmire. One disk does not make a valid raid array. With the raid turned on meta data has been written to the drive and is persistent whether of not the raid is turned on! Normally I would advise turning the raid off in bios and erasing the meta data. This would restore normalcy to the boot on Ubuntu live cd and permit a normal install. I don't know what effect this would have on a win install however. If a win raid driver is already installed that would be a problem - especially if it is for a raid S or 1, either of which could function as a broken array, I don't know how to advise you to clear the raid to keep the windows operable.

In regards to the logical disk issue, you already have three primary partitions on the drive. You need at least two free partitions minimum to install Ubuntu (one each for the / and a swap). Since Ubuntu works fine within an extended partition, if you create an extended from the unallocated space Ubuntu could create as many logical partition as needed there and install without problem.
