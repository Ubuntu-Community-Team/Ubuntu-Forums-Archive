### Post by srs5694 on 2011-05-22
It's unclear if you're asking how to convert /dev/sda2 into a logical partition or if you think the Ubuntu installer already this with with another partition (now /dev/sda5) and you want to convert it back to a primary partition.

I've never heard of the Ubuntu installer converting a partition from primary to logical, so if the latter, and if your belief is correct, this is a new issue. I'm skeptical, though; Windows *can* install most of itself to a logical partition *if* it's got a suitable primary partition for some critical files. It appears from your output that this is the sort of configuration you've got, although I'm not an expert on Windows installations, so I can't be positive of that.

If I'm right, you need to modify your /etc/grub.d/40_custom file and re-run update-grub. Here's what I think you'd need, based on what you've posted:

```

menuentry "Windows 7 (loader)" --class windows --class os {
    insmod part_msdos
    insmod ntfs
    set root='(/dev/sda,msdos2)'
    search --no-floppy --fs-uuid --set=root 01CC031CB9B8B7C0
    chainloader +1
}

```

If I'm wrong and the Ubuntu installer really did convert a primary NTFS partition into a logical partition, you can reverse this with my [FixParts](http://www.rodsbooks.com/fixparts/) program.

Note that the Windows partitioning tool is unreliable in reporting partitions as primary vs. logical; it tends to report logical partitions as being primary. Thus, if you believed Windows was in a primary partition based on the Windows disk utility, that information counts for nothing.
