### Post by PmDematagoda on 2008-11-02
I am not sure if installing fglrx on your kernel would work, the fglrx driver has a kernel module, and it has to be built upon the normal Intrepid kernel in order to be useful, so the chances are that you would need to build the fglrx driver yourself. 

But here comes another potential problem, the current fglrx driver supports only as high as Xorg 7.3 and not Xorg 7.4 which Ubuntu 8.10 has, now Ubuntu has solved this by obtaining a beta fglrx driver from ATi which is not available publicly, you could build the source(if there is any at all) and see if it would work. But if you don't want to take the chance of breaking X, then your best bet would be to stick with either the ati or radeonhd driver, both of which have partial 3D support and will work properly with X.

### Post by kyklops on 2008-11-02
I had a previous installation (8.04) that I wanted to overwrite, where graphic drivers were installed correctly.
But booting from live cd (8.10) the issue occurred: so i have installed fglrx and copied the old xorg.conf from installed Kubuntu 8.04.

```
apt-get install xorg-driver-fglrx

sudo mkdir /media/abcd
sudo mount /dev/sda3 /media/abcd
sudo cp /media/abcd/etc/X11/xorg.conf /etc/x11/xorg.conf
startx
```

where my old Kubuntu 8.04 was installed on sda3 partition

options snd-hda-intel model=acer-aspire
