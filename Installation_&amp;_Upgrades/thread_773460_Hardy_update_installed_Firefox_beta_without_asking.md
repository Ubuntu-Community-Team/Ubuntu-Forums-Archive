### Post by HyperHacker on 2008-04-28
It seems partway through the upgrade, Firefox got updated to Firefox 3 Beta 5. Now automatic upgrading is typically a good thing but what's with installing *beta* software without asking permission? I'd much rather have waited for the final release, especially since this version is incompatible with almost every extension and theme I have installed. Now I either have to downgrade it manually or be stuck without extensions and with the ugly default theme for who knows how long.

This should not happen. Firefox is not part of the OS and thus should not be automatically updated during an OS upgrade, and especially things should not be automatically updated to beta versions. Nor should any updates be installed behind my back, without asking.

Even as a more adventurous, Firefox nightly user, I have to agree with you.  I find it odd that a piece of beta software has been distributed as the default browser.

The only reason I can think for this decision is so that the default browser is appropriately updated to Firefox 3.0 when the final version is released.  Were 2.0 kept as the default browser in Hardy, 3.0 could not be offered as an upgrade.  2.0 and 3.0 will likely be offered as separate packages.

This is just my speculation, of course.
