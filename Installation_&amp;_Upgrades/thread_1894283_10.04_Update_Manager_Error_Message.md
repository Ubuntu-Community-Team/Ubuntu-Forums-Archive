### Post by BC59 on 2011-12-12
First, open your Update Manager--Settings--Software Sources--Other Software and uncheck the CD Rom option. 

Then run from a terminal:

```
gpg --keyserver keyserver.ubuntu.com --recv 83FBA1751378B444
gpg --export --armor 83FBA1751378B444 | sudo apt-key add -
```

```
gpg --keyserver keyserver.ubuntu.com --recv 60D11217247D1CFF
gpg --export --armor 60D11217247D1CFF | sudo apt-key add -
```

Then 

```
sudo apt-get update
```

and post the results

### Post by Old_Grey_Wolf on 2011-12-15
LibreOffice was not in the Ubuntu repository of 10.04. The version of OpenOffice in 10.04 was 3.2. You must have installed from the PPA to get them or the latest development release of them. Depending on how you installed them then Synaptic may not know about them being on your system. 

LibreOffice is now at 3.4.4; therefore, the development package for 3.3.2 is no longer offered. I think they dropped the 3.3.x PPA package when they moved from 3.3.x to 3.4.x.

So no, you are no longer checking the LibreOffice or OpenOffice repositories for updates. If you are using 10.04 you will have to find out what the current PPA package is for those products and change your sources.list accordingly.

You have already disabled the old PPA entries in the sources. You could enter these command to see if it fixes your problem ```
sudo add-apt-repository ppa:libreoffice/ppa 
sudo apt-get update
sudo apt-get install libreoffice
```
