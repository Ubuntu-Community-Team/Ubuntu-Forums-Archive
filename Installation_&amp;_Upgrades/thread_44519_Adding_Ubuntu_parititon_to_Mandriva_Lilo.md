### Post by FM1 on 2005-06-27
I got it, thanks. Still don't know what I did the first time that made Ubuntu lock up, but this was all I had to put in to make it work:

image=/mnt/ubuntu/vmlinuz
	label="ubuntu"
	root=/dev/hda6
	initrd=/mnt/ubuntu/initrd.img
	append="acpi=ht resume=/dev/hda7 splash=silent"
	vga=788
	read-only

### Post by mingus on 2005-06-27
[QUOTE=FM1]Okay, but it works now--why would I want to change it?[/QUOTE]

We need to see it, not to change it, but to see if there are any kernel arguments in the working stanzas missing from the non-working Ubuntu stanza, or if there is a global that may interface with the Ubuntu stanza.  

As it turns out, the problem *is* in the stanza syntax.  The image and initrd commands will not work.  

My LILO is a bit rusty, try these two changes:
image=/boot/vmlinuz-2.6.10-5-386
initrd=/boot/initrd.img-2.6.10-5-386

This assumes you are using the Hoary stock kernel.

Notice that the Mandrake stanzas only use vmlinuz.  That's because Mandrake uses a symbolic link from vmlinuz to vmlinuz-xxxxxx (the actual kernel filename).  Ubuntu doesn't do that.  And, the "/mnt/ubuntu/" is strange; the kernel is always under /boot.

And btw, unless hda4 is too small, you don't need the hda7 swap.

Finally, be sure whenever you change LILO's conf file that you reinstall the loader; not needed with changes to GRUB's menu.lst file.

Should you want to go back to Ubuntu controlling the boot, you can do:
$sudo update-grub
this will regen grub's menu.lst file and, if you're lucky, it will find Mandrake again with any necessary stanza changes; if it checks out, then to install grub you do:
$sudo grub-install /dev/xxx (where xxx is where you want the loader installed)
