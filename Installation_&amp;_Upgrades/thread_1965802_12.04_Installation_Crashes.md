### Post by Senior_Buckethead on 2012-04-27
Hope this helps, 

You can do a md5hash check on your .iso image. In terminal Just navigate to the folder that has the original .iso image and type:
```

cd ~/
cd Downloads
md5hash ubuntu-12.04-alternate-i386.iso

```Then go to this web page:
[https://help.ubuntu.com/community/UbuntuHashes](https://help.ubuntu.com/community/UbuntuHashes)

and find the appropriate hash, in this case it would be 

```

 bcee4c03b704a9b62988505b7d8f3069

```See this webpage for how to md5hash:
[https://help.ubuntu.com/community/HowToMD5SUM](https://help.ubuntu.com/community/HowToMD5SUM)

What is a md5hash?
[http://en.wikipedia.org/wiki/MD5](http://en.wikipedia.org/wiki/MD5)

Then I copied and pasted the md5hashes from the web and the result in your terminal window into gedit, or any other editor, and compare the two. They should match.

When you boot the alternate CD, it comes up with a menu. Select 'Check CD' and run that. That should be fine. If both are good, then your CD is a valid CD and should work.

I know personally that I have had worlds of grief with 64-Bit alternate installs, 32 bit seems to work well with my processor. Its an AMD Athlon, so should run 64 bit, and does, but installing it, I have found, is a whole world of pain.

If you cant download the Alternate CD, you need to obtain it from elsewhere. Perhaps there is a Linux group where you live, or perhaps you know someone with an Internet connection you can download from?
