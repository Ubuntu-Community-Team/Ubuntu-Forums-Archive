### Post by oldfred on 2014-02-25
I do not want to change partition table size, that is the default of 128 partitions. In fact I knew you could change that but did not know how.

I have not used gdisk or sgdisk in advanced mode, but you just want to shrink the last partition by 1 or so sectors, so you do not have overlap.  I use gparted but with a partition table error it will not load.

Does just writing a new partition table from gdisk correct the error. Just loading gdisk & then reviewing the list of partitions with l and w for write often fixes minor errors.

If the w does not fix it, then you have to manually edit the size of the last partition. Just shrink by one or a couple of sectors.

### Post by josh39 on 2014-02-27
Resolved!

You were exactly right. Using gdisk, I wound up keeping track of the start and end sectors for the last partition (/dev/sda5). I then used "d" to delete that partition, then re-created it with the same sectors, minus 4 for the end. wrote this out, rebooted, and it then let me install.

Of course, I don't know if I've messed up the recovery partition, however, I can confirm that I was able to install into a new partition, and can still dual-boot into Windows 8 or ubuntu via GRUB. 

Thanks for all your help!
