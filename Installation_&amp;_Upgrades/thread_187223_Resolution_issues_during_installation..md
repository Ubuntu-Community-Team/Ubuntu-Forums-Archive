### Post by BaffledMollusc on 2006-06-05
Yes, I had this issue too. It's very annoying and I don't have a perfect solution for it, but here's something that will help.

It is possible to drag windows around the screen by holding down the ALT key when you click and drag with the left mouse button. Using this technique you can move the install windows around, even in 600x480 resolution, so you can see the whole window (just not all of it at once).

This will get you through the install.

The only way I managed to fix the resolution when I had finished the install was to install the ATI proprietory driver. To do this use Synaptic to install the fglrx driver, and then type "sudo aticonfig --initial" in a terminal to fix your xorg.conf file. (At least, I think the option is --initial; you might want to type "aticonfig --help" to make sure.)

This worked for me because I have an ATI video card; if you don't have an ATI card don't try this fix.

The other problem was that installing the proprietory ATI driver means that I now have another bug - I can't shut down Ubuntu. As soon as I choose "reboot" or "shut down" the PC hangs. I have to manually turn the power off to shut the PC down. But at least I can have any resolution I want :)

So, be warned, this might happen to you as well (for the record I'm using the 64-bit version of Ubuntu 6.06 and an ATI X800 Pro video card).

### Post by anibalrojas on 2006-06-06
I had the same problem with the higher resolution being 640 x 480, and that preventing me from running the installer. I just ran:

sudo dpkg-reconfigure xserver-xorg 

There is a screen where you can unmark the screen resolutions that you don't want to use, so I unchecked anything but 1024 x 768 and let it write down the configuration files.

After that just the hitted Control - Alt - Backspace to restart X And I went into a 1024 x 768 resolution, were I was able to run the installer.
