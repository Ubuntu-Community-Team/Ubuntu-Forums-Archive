### Post by Ti-Paul on 2005-04-15
Just did a fresh install of Hoary yesterday and found two strange behaviour:

(1)
When i logged for the first time, i found out that every "sudo commands" (like package-manager, root-terminal, etc..etc...) wasn't working at all (and returned no feedback message).

When trying these commands in the terminal, i found out that my username wasn't added to the /etc/sudoers file!!!???... (I remind you that it's a fresh install from the release version of Hoary!)

I added my user to sudoers and now it's working "normally"... Haven't got this problem with Warthy before...

(2)
I'm used to Mozilla Thunderbird, so when i got into synaptic to do a search for mozilla-thunderbird to install it, i found that the french locale (locale-fr) of thunderbird was already installed, just had to install thunderbird...

That's not anoying for me, but for people that select french language and aren't using mozilla-thunderbird... I assumed that they will get this locale installed by default... So maybe there's more locales installed vs applications?


I'm not "COMPLAINING"... I'm an Ubuntu devoted user since the start (and since Slackware stop support for Gnome)... Just found 2 new strange behavior that wasn't in my previous Warty experience...
