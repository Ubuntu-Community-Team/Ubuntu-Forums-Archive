### Post by dogon on 2006-10-29
I replaced my debian partition with ubuntu edgy eft and wanted to keep data drives intact. So I went with manual partitioning. Everything went fine and I figured out how to assign drives to mount points. I had selected a root partition but the installer would not let me proceed for a "No root selected" bug.

The solution is to go back to the partition manager and remove the partition, afterwards recreate it and proceed. Root will then be automatically marked for you and you can proceed in the installation.

The system itself works well for me, ati drivers went so easy it made my jaw drop. 

Eclipse did not auto install the jdk and I could not find that in the add/remove panel. So  I  installed it the old fashioned way with apt-get. 

Date/Time adjust crashes when I right click on the desktop clock and select "adjust date & time". (oldfashioned command line "date -s" still works)

I am also having some harddisk problems, on boot the drives get read/write errors, mounting them after a full boot makes them work like a charm. The physical drive has however caused boot problems before. So not sure if this is Ubuntu related.

I never used ubuntu before. And I hear this is a buggy release. But so far its ease of use is still an improvement for me.
