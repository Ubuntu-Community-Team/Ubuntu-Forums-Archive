### Post by coffeecat on 2009-12-09
You've been hit by this bug:

[https://bugs.launchpad.net/debian/+source/msttcorefonts/+bug/431217](https://bugs.launchpad.net/debian/+source/msttcorefonts/+bug/431217)

It's a weird one. Most people are OK with the ttf-mscorefonts-installer but a few have been unlucky. The ttf-mscorefonts-installer has to download the font packages (in .exe format) from sourceforge for licensing reasons, but the bug causes DNS timeouts for some reason.

Looking at your code posting, it seems that a few of the fonts were installed despite the timeouts, but most are not. So - a few questions:

Did you select the ttf-mscorefonts-installer yourself, or did it get selected as a dependency of another package? For example, the package ubuntu-restricted-extras brings in a load of multimedia stuff, but also includes the ms fonts.

The first thing to do is to unmark ttf-mscorefonts for installation, otherwise you'll run into errors every time you try to install something. Instead of 'Software Centre' (that's what it's called in English), go to System > Administration > Synaptic Package Manager, and look for ttf-mscorefonts-installer in the middle pane. Is the box either green or is there a yellow tick in it? If it's green don't do anything but post back. If there's a yellow tick, right-click and select 'unmark'. Now click on the 'Apply' button to install anything else that was marked and couldn't be installed because of this problem.

Unfortunately, this still means that you won't be able to complete the installation of the MS fonts. If you want the ms fonts and have access to a Windows installation, I can tell you how you can install them another way simply by copying the appropriate *.ttf files from Windows to Ubuntu. Or, if you are feeling adventurous, have a look in post #21 of the Launchpad thread. If you need help with that, just post back.

### Post by coffeecat on 2009-12-09
You need to make a hidden folder in your home directory called '.fonts' . Note the leading point - files and folders with a leading point in the name are hidden. Open you home folder (Places > Home Folder) and either press ctrl-H, or View > Show hidden files. You'll see many of them. Now simply create a new folder called .fonts . Then, copy the *.ttf files for whatever fonts you want into the .fonts folder. If you want the mscorefonts you can find them in C:\Windows\Fonts in your Windows installation. Since you're running XP you should be able to access them OK. Vista and W7 don't let you and you have to resort to some Linux subterfuge.

Just so you know, the MS corefonts include:

> Andale Mono
  Arial Black
  Arial (Bold, Italic, Bold Italic)
  Comic Sans MS (Bold)
  Courier New (Bold, Italic, Bold Italic)
  Georgia (Bold, Italic, Bold Italic)
  Impact
  Times New Roman (Bold, Italic, Bold Italic)
  Trebuchet (Bold, Italic, Bold Italic)
  Verdana (Bold, Italic, Bold Italic)
  Webdings(From the ttf-mscorefonts-installer package description.)

This will install them for your account only. If you want directions for installing them system-wide, post back and I'll post that as well.

As far as ttf-mscorefonts-installer showing green in Synaptic, this means that Synaptic thinks it is installed. Are you still getting error messages whenever you try to install something or do an update? If so, it might be advisable to mark ttf-mscorefonts-installer for removal in Synaptic. But then, be careful whenever you try to install something - the mscorefonts installer might be marked as a dependency and you'll get those errors all over again.

I mentioned ubuntu-restricted-extras earlier. If you haven't done so already, it's worth installing as it gives you various restricted multimedia goodies, flash and java. Trouble is, it gives you the ttf-mscorefonts-installer as well. So, if you want the multimedia stuff, open Synaptic, mark ubuntu-restricted-extras for installation, and then unmark ttf-mscorefonts-installer. Then 'Apply'.

Let us know how you get along.
