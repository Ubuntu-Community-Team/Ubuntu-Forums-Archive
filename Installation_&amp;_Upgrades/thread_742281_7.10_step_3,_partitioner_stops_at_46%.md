This has happened to me on some Samsung HDD drives. I have worked around it by creating the partitions manually in a virtual terminal (Ctrl+Alt+F1) before the install process.

If you want instructions on how to create the partitions manually, please tell me your target installation idea: Single boot/Dual boot, what is currently installed, how much space is used etc. A simple way to get this information is to use the command ```
sudo fdisk -l
``` in a terminal from the live CD.
