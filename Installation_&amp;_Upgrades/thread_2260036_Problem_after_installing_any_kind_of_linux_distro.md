### Post by sudodus on 2015-01-09
Now we know it is an UEFI installation with only linux (no Windows is reported by the boot-info script). And the drive is a gpt (GUID partition table) which fits UEFI. Is it correct that you have already tried Boot-Repair (as you wrote in the opening post)? Was it the same version or an older one? If you have the current (newest) version, you might try again to let Boot Repair do its job.

But I must admit, that I'm no expert on UEFI. I know a little about it, but not much enough. Also it differs a lot between computers, because many computer manufacturers and vendors use non-standard UEFI versions (we can guess in order to make it difficult to run other operating systems than Windows). I think *oldfred* is the best expert here at the Ubuntu Forums. See his tutorial
[URL="http://ubuntuforums.org/showthread.php?t=2147295"]
UEFI Installing - Tips[/URL]

See also this link and (other) links from it

[https://help.ubuntu.com/community/Installation/FromUSBStick#UEFI](https://help.ubuntu.com/community/Installation/FromUSBStick#UEFI)

-o-

You have no Windows. Is it by intention or did you overwrite Windows by mistake?

- If you want Windows 8, and want to reinstall it, you need UEFI.

- If you are happy with 'only' Ubuntu, you can change the partition table to the old standard MSDOS (with ***gparted*** when booted from an Ubuntu desktop install drive). And then install Ubuntu again. It will be more straight-forward.

*Edit:*

Please tell us more about your computer. It helps giving more specific advice.

We know the brand name and model, but please also describe

- the graphics chip/card
- the wifi chip/card

Basically, if the Ubuntu works live from a USB pendrive, it should also work when installed. And obviously it works live, otherwise you would not be able to run Boot Repair.

In most computers it will be easier to install Ubuntu in BIOS mode with an MSDOS partition table. But I don't know, some new computers might refuse to run properly in BIOS alias CSM mode (opposite to UEFI).
