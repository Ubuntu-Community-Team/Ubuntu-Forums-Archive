### Post by Kevbert on 2008-11-02
I have an old dinosaur of a computer which I'm trying to install Intrepid 32 bit on. The basic specs are Athlon 900MHz, 41Gb IDE hard disk, 512Mb RAM, Voodoo Banshee 3 graphics. On the machine I have Win98 and Xubuntu 8.0.4.1 which run fine.
When I try to install the machine gets to 94% and then I get the message Executing 'grub-install (hd0,0)' failed. This is a fatal error.
I've read elsewhere that it could be due to a large swap file which I've reduced to 990Mb, but still get the same problem. Bios boot protection is disabled. I've removed all unused Xubuntu kernels.
Advice is appreciated.

---

### Post by Kevbert on 2008-11-02
Resolved the problem. The boot partition was generated by Win98 and by installing Intrepid over the whole disk has resolved the problem. It looks like Intrepid doesn't like old FAT32 partitions.
