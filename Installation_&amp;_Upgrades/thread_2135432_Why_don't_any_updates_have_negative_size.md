### Post by Mark Phelps on 2013-04-14
> **J-E-N-O-V-A said:**
> Surely if the code is getting better then its size is going to be reducing?

There's absolutely no reason to associate "better" with "smaller".  Code consolidation (to reduce the overall code footprint) is time consuming and costly -- and when the stuff is given away for free, comes with NO financial gain for the Developer.  Also, "better" nearly always means "more features" -- which is generally going to require extra code, not less.
