### Post by yancek on 2024-10-22
The most likely source of your problem was deleting the EFI partition.  Having an EFI on each drive is not or at least should not be a problem.  Try running the command below to get pertinent information.  The information you should look for (both drives attached)  are a line Partition table which should show gpt and a partition with  EFI system partition under the name column.  

>  sudo parted -l


If you have that, boot Kubuntu and run the command below to see what you have on your EFI partition:

```
 sudo ls -l /boot/efi/EFI
```

There should be a Microsoft directory there with windows EFI boot files and if it is not available, you will have to install those files from windows. Someone here who is familiar with windows may be able to assist you with that otherwise, go to a windows forum.   If you have EFI partitions on both drives, you will need to mount the 2nd EFI partition and again look in that directory for a Microsoft directory.  If you have that directory on either EFI partition, post the contents for help.

>   i was dual booting both ubuntu  and win10 on SDB.  Ditched ubuntu, but it left an EFI partition, which I  have since deleted

There was no need to do that and as indicated above, is the likely source of your problem.  You could have deleted the ubuntu directory on the EFI partition or you could have just left it and when installing Kubuntu over the previous Ubuntu install, Kubuntu would have overwritten the Ubuntu EFI files.

Looking at the parted output you posted, you have 2 physical drives each of size 1TB and both gpt drives.  You have both windows and Ubuntu on the drive listed as /dev/sda.  sdb has 3 windows partitions but no EFI partition so is that the windows 10 drive? 

> I understand why your saying that each physical disk need and EFY partition (right?). 

No, didn't say that.  I said computers 'can' have an EFI partition on each hard drive.  You can have EFI installs on multiple drives and an EFI partition on only one but that isn't a concern in your case.  I asked in my last post that you post the output of the command below run from a terminal when booted into Kubuntu and you did not do that.  You need a Microsoft directory there:

```
sudo ls -l /boot/efi/EFI
``` 

You could try to test boot windows by copying (use sudo) the menuentry below into the /boot/grub/grub.cfg file.  You need to change "90FE-41A5" in the entry to whatever the UUID is for partition sda7, your EFI partition which should also be an 8 character entry similar to the one below.  If you do this, just save the change and do NOT update grub but reboot to test and if it works, post back for instructions.  If you do NOT have the bootmgfw.efi file in the Microsoft directory of your EFI partition, don't bother as this entry will not work.  You need to reinstall windows EFI files from some windows system if the files are not there.

> menuentry 'Windows Boot Manager'  {
    insmod part_gpt
    insmod fat
    search --no-floppy --fs-uuid --set=root 90FE-41A5
    chainloader /EFI/Microsoft/Boot/bootmgfw.efi
}
