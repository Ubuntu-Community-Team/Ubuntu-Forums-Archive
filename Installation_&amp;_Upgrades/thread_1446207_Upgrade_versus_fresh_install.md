### Post by Alan James on 2010-04-03
I personally like doing a fresh install of my computer every six months. It keeps my Windows 7 fast and allows me to experience the new Ubuntu without worrying about incompatibilities during the upgrade process. I have a small Samba server that I store all of my files on, and thus never worry about erasing files. I have written shell scripts to reinstall all of my applications and configure parts of the system the way I like it. This cuts my totally “out of commission” time to only about two hours for the entire thing.
 

 However...
 

 My roommate likes to upgrade his Ubuntu because he says it is faster and easier. He stores his files on his computer and thus doesn't like to format his drive. He has never had any problems with the upgrade process and says I am crazy to spend two hours reconfiguring my computer when he only takes ten seconds to click his mouse a few times.

### Post by merlinblack on 2010-04-08
> **g003y said:**
> But is it possible to fresh install without erasing the files in the home directory and keep the same programs that are currently installed? Or do I have to backup the home and manually keep a record of chosen software packages?
Yes you can leave the home directory untouched, provided you have it on a separate partition or even on a disk to itself.  However with a fresh install you will need to install any programs that you  want that do not get installed by default for the version you are installing.  For instance, in 10.04, GIMP is no longer installed by default, so I had to install it.

Some times settings in dot files/directories, will not work properly with the newer versions, so YMMV.  That said I personally had no trouble with going from Karmic to Lucid.  My email settings etc came across fine.

I always backup my home partition just in case.  Then if I have to, I can format completely and copy the parts I want back.
