### Post by tlcstat on 2011-03-12
Greetings,
You can update you installation iso current to Feb 17, 2011 with the zsync file on the Lucid download site. If you have a currently updated Lucid system you can copy the apt cache over the the new installation and that should supply most if not all of the upgrades. I do this all the time. I actually backup my apt cache to a backup hd. It is currently around a Gig. Your apt cache is located at /var/cache/apt/archives. You only need to copy the archives dir without the hidden system files.
Don't know if this is the best answer but its what I do. 
tlcstat

### Post by tlcstat on 2011-03-13
Greetings,
Actually I have mis-stated the case, after re-reading your post. If the system can't log-in to the Ubuntu channel and update its file list then it won't know what to do with the archive that is copied over. I miss the point that your installation wouldn't have internet access. I would say at this point to use the zsync and just make sure that you are using the most current release. A large part of the security in Linux relates to the fact that updates come through an authorized channel. If there is another answer to this it will have to come from someone with more experience than me. 
Good luck.

You probably know this already, but you need to make a persistent LiveUSB. This can be done using the Startup Disk Creator or the most recent version of UNetBootin. Use an ISO from the most current Ubuntu release (or version of your choice).

I would recommend complete removal of any unnecessary packages to reduce drive storage requirements and improve the efficiency of the update / install process. If you have a small flash drive, select a portion of the packages to update (keep related stuff together), then do a cleanup, then update another portion, cleanup, etc. There are smaller distros (specifically, Puppy) that would probably be easier to update / install.
