### Post by oldfred on 2018-01-24
17.04 is obsolete. Use 16.04 LTS or 17.10.

If Windows is UEFI, then better to have Ubuntu in UEFI boot mode.
But HP does not make it easy. They violate UEFI spec and we have to do various work arounds.

       May be best to see details, you can run from your Ubuntu live installer or any working install, use ppa version not older Boot-Repair ISO:
Post the link to the Create BootInfo summary report. Is part of Boot-Repair:
[https://help.ubuntu.com/community/Boot-Info](https://help.ubuntu.com/community/Boot-Info) and:
[https://sourceforge.net/p/boot-repair/home/Home/](https://sourceforge.net/p/boot-repair/home/Home/)

If Wi-Fi does not work with live installer, use hardwired connection. That will make it easier to download correct drivers, updates and run the above report.

Maybe.

Ubuntu uses repository and you just download directly from repository into Ubuntu.
But you then have the now old issue of dependencies. Years ago before repository and tools to manage packages, you downloaded an application and then got an error that you needed another dependency. And on and on until everything was correct. 
Now with package management & repository, that issue is long gone.
