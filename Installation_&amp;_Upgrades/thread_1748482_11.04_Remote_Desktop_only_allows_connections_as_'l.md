### Post by parish on 2011-05-03
Under 10.10 the settings dialogue for Remote Desktop said that other machines can connect to this one as kubuntu.local or 192.168.1.85 but after u/g to 11.04 instead of hostname or IP it now just says 'localhost' - which won't work for obvious reasons.

Any ideas in how to fix this? I use RD a lot.

Also, despite 'Always display an icon' being checked no icon is displayed in the panel. I tried checking 'Never display an icon' closing the dialogue, re-opening it and re-selecting 'Always display' it still doesn't appear.

---

### Post by parish on 2011-05-04
Solved it. Seems that the u/g to 11.04 had replaced /etc/gdm/Init/Default which removed the vino server startup line which i'd added:

```
/usr/lib/vino/vino-server --sm-disable &
```

Added this back in and it's working properly again :D
