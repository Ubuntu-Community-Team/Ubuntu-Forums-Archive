I'm writing this post because I'm disappointed with ubuntu's upgrade behavior: I've just upgraded the kernel and I got my grub.conf file overwritten by the upgrading system. I think that ubuntu needs a less invasive upgrading system. I'm even thinking about something like gentoo etc-update system that allow you to decide what to change....

I also want to ask a thing: Is there a public testing team? I mean, I saw that lots of people are experiencing problems with the kernel update and I think that it should exist a group of users (like for beta releases) that can test software updates before the package manager asks everybody to upgrade...
