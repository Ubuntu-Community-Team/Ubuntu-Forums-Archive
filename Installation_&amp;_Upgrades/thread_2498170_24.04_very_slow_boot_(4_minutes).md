### Post by oldfred on 2024-06-02
Back when computer was slower I prefered to see the boot process rather than some splash screen.
I liked to know something was happening. And if issue you could often see command that was problem, not always last posted.
Its same info that goes into log file.

I change some settings in /etc/default/grub. This is my grub with all the # lines removed (they are still there, just not posted)
```
fred@z170-noble:~$ cat /etc/default/grub
GRUB_DEFAULT=0
GRUB_TIMEOUT_STYLE=menu
GRUB_TIMEOUT=3
GRUB_CMDLINE_LINUX_DEFAULT='noplymouth'
GRUB_CMDLINE_LINUX=""
GRUB_DISABLE_OS_PROBER=true
GRUB_DISTRIBUTOR=noble
```

I have multiple installs so want different names, for distributor. I put all boot stanza into 40_custom as I do not want os-prober looking for the other installs, some are obsolete. I always want grub menu. And change to noplymouth to see boot process which also seems to make it faster, not sure why.

Saves manually removing # lines:
[FONT=monospace][COLOR=#000000]grep -Ev ^'(#|$)' /etc/default/grub[/COLOR]



[/FONT]

### Post by alainhenry on 2024-06-21
Sorry it took so long. 
I tried different diagnosis tools, and it seemed that the nvidia driver might be the cause (viewing the startup messages and where it halted).  I changed from the default X.org driver to the Nvidia driver metapackage from nvidia-driver-335 (proprietary, tested) (I have a GEForce MX250 GP108BM 2GB graphic card), and the boot is now back to reasonable time, largely under a minute.
