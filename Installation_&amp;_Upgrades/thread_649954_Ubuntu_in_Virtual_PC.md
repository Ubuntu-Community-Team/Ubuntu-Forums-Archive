### Post by beatme101 on 2007-12-25
Hello everyone. I've got a copy of Ubuntu running in Virtual PC. Quite a while (months) ago after some update, the mouse stopped working. After some investigation, I discovered that PS/2 mouse support was removed with that upgrade. Since performing that upgrade I am no longer able to use the virtual operating system. Is there a way to downgrade to the last version that had PS/2 mouse support? I know the developers really really didn't want people running it in Virtual PC (why else would they choose 24 bit colour depth by default, God that was hard to navigate, install, and then finally be able to change, and it has to be the least compatible depth of all), but this is a really easy test area that doesn't require me to use a whole computer.

I think I'll also note that surely some computers still have PS/2 mice, why would they remove support for those? Isn't that incredibly shortsighted? USB mice are still being made and sold today that come with PS/2 adapters. Why? Maybe for computers with no free USB ports, or faulty USB ports..

As far as I can tell, Ubuntu is completely impossible to use beyond logging in unless you have a USB mouse, as there are don't seem to be any keyboard shortcuts at all.
