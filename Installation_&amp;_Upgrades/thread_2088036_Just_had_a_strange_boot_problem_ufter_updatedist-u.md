### Post by 7bit on 2012-11-25
I just did
```

sudo apt-get update
sudo apt-get dist-upgrade

```like I have done it thousands of times before already and everything seemed to go smoothly, It installed a new kernel (I previously had 3.2.0-32 and it installed 3.2.0-33). It was a rather huge upgrade but no error at all, everything went fine.

Before I upgraded I removed all my old kernels except the one I was currently running (3.2.0-32). Then I ran the upgrade. Attached below is the log from /var/log/apt/term.log. As can be seen there was no error. It actually *ran* update-grub and it found my 2 kernels. After this I rebooted. 

And then I was greeted by an empty grub 1.99 prompt (no boot menu) and unfortunately my DVD drive is broken and no bootable USB drive around currently too. I began sweating blood at this moment.

I managed to make it boot again by doing the following on the grub prompt (I used my old kernel intentionally, not the newly installed one, but it seems the new one would have done it too)
```

linux /boot/vmlinuz-3.2.0-32-generic root=/dev/sda1
boot

```and then after successful boot I did this in a terminal:
```

sudo update-grub2

```And I did nothing else, this was the only thing I did, then immediately rebooted again and everything was ok again, grub2 boot menu like I was used to and both of my kernels booting again without problems. 

What could have caused this? As it is visible from the apt log it actually *did* the update-grub after installing the new kernel and there were *no* errors. What could have happened that would be able to cause such symptoms without actually producing any error message during grub-update?
