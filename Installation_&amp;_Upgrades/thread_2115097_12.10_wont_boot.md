### Post by ahallubuntu on 2013-02-11
I guess by "newer model" I didn't mean one that is running MS Vista.   I assume it's at least a few years old?  I meant "newer" as in "recent, maybe too new to have good Ubuntu support."

If there are two drives, one PATA (IDE) and one SATA, which one did you install the Grub boot loader on?  Are you choosing which drive to boot from with a BIOS boot menu?  Otherwise, how do you know which drive is set to boot first?  (Does Vista still boot?)

You might try disconnecting the Vista drive entirely until you get this working.  You could boot the 12.10 live CD again (with no Vista drive attached) and try boot repair.  There are various ways to do it.  I prefer the "chroot" method to re-install Grub, but it is kind of a manual method.  On the other hand, it almost always works for me:

[https://help.ubuntu.com/community/Grub2/Installing#via_ChRoot](https://help.ubuntu.com/community/Grub2/Installing#via_ChRoot)
