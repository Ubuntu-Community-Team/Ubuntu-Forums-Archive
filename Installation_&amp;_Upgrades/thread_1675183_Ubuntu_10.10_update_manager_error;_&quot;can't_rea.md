AmanjWell it looks like your /etc/modules file is missing. Check this in a terminal:
```
ls -l /etc/modules

```and if it really isn't there or doesn't have -rw-r--r-- permissions and owner root just type:
```
sudo touch /etc/modules
sudo chown root.root /etc/modules
sudo chmod 644 /etc/modules
```which will create an empty /etc/modules file with the right owner and access permissions.

AmanjThere's actually a manual page for this file:
>                                                          MODULES(5)

NAME
       /etc/modules - kernel modules to load at boot time

DESCRIPTION
       The  /etc/modules file contains the names of kernel modules that are to
       be loaded at boot time, one per line. Arguments can  be  given  in  the
       same line as the module name. Lines beginning with a '#' are ignored.

SEE ALSO
       depmod(8) modprobe(8) modprobe.conf(5)

Debian GNU/Linux                  Version 1.2                       MODULES(5)In practise it often contains nothing, everything is now found in files in the modprobe.d directory. That's why an empty file is all you need.
