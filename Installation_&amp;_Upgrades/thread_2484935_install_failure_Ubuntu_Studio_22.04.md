### Post by guiverc on 2023-03-14
Without knowing what the error messages are, I can't provide any suggestion beyond copy/paste the messages so we can read them & provide input.

If they are *squashfs* or media errors - the 22.04 media you were sent is invalid; the person who downloaded/wrote it should correct their work.

If the issue relates to the drive you're installing to, we need the message(s) to provide any advice.

If the error messages are *timeout* related, its your use of *optical* media that maybe an issue; use thumb-drive (*the intended media on any install in releases of 20.10 & later*).

Yes *flash* media is preferred for Ubuntu 20.10 & later (*which includes flavors like Ubuntu Studio*), but *optical* media was proved to work, but if your drive is getting old, or you're using a slow/old CPU the *timeout* errors can cause error messages that are consequence of bad reads of the *optical* media that you'll only detect if you read system logs. Newer hardware using *optical* media tended to operate flawlessly.

As for common problem; No, but I don't know what error messages you're asking about.
