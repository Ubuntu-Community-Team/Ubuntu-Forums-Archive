### Post by iconz255 on 2009-06-04
[FONT=Verdana][SIZE=2]Hello all, my post is about the problems I have had in *buntu and how I managed to solved them, today, just some minutes ago. Hopefully my testimony will help others having troubles with their installation procedures. 

Long story cut short - 

my specs: 
motherboard: asus p5x SE
cpu: intel core duo 2 ghz
gpu: ATI Radeon 1550XT
ram: 1gb
hdisks: 1 ide 40gb [2 equal partitions C: and D:] + 1 80gb [external usb   ]
disk drives: 1 ide cdrw asus + 1 sata dvdrw asus

try ubuntu 8.04 1st time - failed
try kubuntu 9.04 2nd time - failed
try kubuntu 9.04 3rd time - success!

Long story - 

My problem was that each time I tried to install *buntu, I got to the selection menu of the live cd and could not get to the live desktop. Instead, each time I choose the option to install *buntu without change to the desktop, I got to the process bar and then the system would stop and give me an error or a message and I got the sandbox message where I could enter bash commands. As I have no *nix knowledge, I could get nowhere from here and let alone find the error log to understand what was really the problem. When installing kubuntu 9.04 today, I got the same problem and was greeted with the same sandbox message although the error wordings differed. I was going to give up the idea of using linux but then I decided to just give linux another ride one last time. 

I had a dvd of fedora 9 which I had never used before and inserted it in my dvd drive. My dvd drive was not the default disk drive in my BIOS. I changed it and put the asus dvd drive as the primary cd drive and chose it to be the first in the boot sequence. I inserted the disk and all went right! I came to the installation menu and then to the installation options where I could actually install fedora 9. I then cancelled the installation, rebooted and inserted my kubuntu 9.04 cd in the dvd drive and it worked and booted me to the live desktop which I could use! 

Before this I used to set my asus cdrw ide drive as primary disk reader and inserted my *buntu disks in them. When I got to the live kde desktop today, kubuntu recognised all my hard drives, internal, external and both disk drives. So if you have the same problem and with nearly or same configuration, guess you could give my solution a try to see if it solves your installation issues. 

Anyway, I don't know how this managed to work or why *buntu got to install on the sata dvd drive without any issues. Maybe that the installer cannot play nice with an unsual or diverse config but this is just a guess. Anyone could share some ideas or shed some light on this issue.
