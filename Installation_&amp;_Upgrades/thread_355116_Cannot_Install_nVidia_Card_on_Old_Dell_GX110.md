### Post by p!=f on 2007-02-06
Because
> 
0000:01:0b.0 VGA compatible controller: nVidia Corporation NV34 [GeForce FX 5500] (rev a1)

but your BusID in xorg.conf says 1:0:0. It just cannot find the device. Try to reconfigure xserver by
```

sudo dpkg-reconfigure xserver-xorg

```
or try to comment BusID line
```

Section "Device"
        Identifier      "GeForce FX 5500"
        Driver          "nvidia"
        #BusID           "PCI:1:0:0"
EndSection

```
and restart GDM/KDM
```

sudo /etc/init.d/gdm restart
sudo /etc/init.d/kdm restart

```
or specify the correct value
```

Section "Device"
        Identifier      "GeForce FX 5500"
        Driver          "nvidia"
        BusID           "PCI:1:b:0" *or "PCI:1:0:b" (I'm not sure which is the correct one)*:)
EndSection

```
I also recommend to turn 3D acceleration on by adding
```

Section "Device"
        Identifier      "GeForce FX 5500"
        Driver          "nvidia"
        Option          "RenderAccel"                   "true"
        #BusID           "PCI:1:0:0"
EndSection

```
or by running
```

sudo nvidia-xconfig --render-accel

```

### Post by RandomJoe on 2007-02-07
Yeah, some BIOSes will do that.  The Dells (at least, this generation) only have "Auto" which means the PCI card gets used as primary if installed, and "On" which means the onboard is always first.

The funny bit is that when set to Auto, the system boots off the PCI card but X doesn't see that, does its own detection, and uses the first card it finds (unless otherwise told in xorg.conf) which will be the onboard.  But even if xorg.conf is using the right driver for the onboard card, it still won't work!  It displays garbage (at best) or nothing.

So to use both the onboard and addin cards in X, you have to set the BIOS "On" so it boots using the onboard video, and then both are accessible.

But to just use the PCI card, as long as xorg.conf is set to the right BusID it works fine in Auto...

Knowledge gleaned only after some time of edit/reboot/set BIOS/reboot/lather/rinse/repeat... :rolleyes:
