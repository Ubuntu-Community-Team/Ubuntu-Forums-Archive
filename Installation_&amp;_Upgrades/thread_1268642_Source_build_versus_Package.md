### Post by Robsteranium on 2009-09-17
I've often wondered whether package installations (i.e. via synaptic) would conflict with those built from source...  They do.

**If you're building a program from source, first uninstall any copies of the program that you installed with a package manager.**

The conflict isn't obvious from the error messages you'll receive (I guess the libraries overlap - they'll not work even if you've met all the dependencies).  It's therefore quite difficult to diagnose problems caused by such conflicts.

This may appear obvious to many of you!  I assumed that it wouldn't be a problem.  I couldn't find anyone saying otherwise so I'm posting the advice here.



Feels good to start a thread with a solution rather than a problem :guitar:
