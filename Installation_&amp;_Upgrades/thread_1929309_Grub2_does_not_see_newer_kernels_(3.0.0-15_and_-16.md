### Post by efflandt on 2012-02-21
I was going to say if you installed Mint last, you would need to run **sudo update-grub** from Mint to update grub, since grub in the mbr is now likely using /boot/grub/grub.cfg on sda4 now instead of on sda1. Or you could do **sudo grub-install /dev/sda** from Ubuntu to tell grub to use the Ubuntu grub.cfg.

But that likely will not work because you are apparently using **extlinux** (which I had never seen before).  So you likely need to manually update /boot/extlinux/extlinux.conf for any kernel updates to any OS judging from [http://www.vyvy.org/main/node/181](http://www.vyvy.org/main/node/181)

Is there a specific reason you are using extlinux instead of grub?

### Post by oldfred on 2012-02-22
You can easily reinstall grub2 to the MBR from which ever install you want. Just boot into that install and run this:

sudo grub-install /dev/sda

But all your other installs remember that sda(by hard drive name) is the place to reinstall to. Some updates only update grub menu, but a major update to grub2 may reinstall grub2 to the MBR.

#to get grub2 to remember where to reinstall on updates, if you choose nothing (unchoose sda drive) it should not store a reinstall location:
sudo dpkg-reconfigure grub-pc
#Enter thru first pages,spacebar to choose/unchoose drive, enter to accept, do not choose partitions

You can confirm the reinstall location or check before and see that it changed:
#To see what drive grub2 uses see this line - grub-pc/install_devices:
sudo debconf-show grub-pc
