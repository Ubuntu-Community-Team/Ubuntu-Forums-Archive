### Post by oldfred on 2010-08-31
The test at this point is to just install a windows boot loader to the MBR and see if it boots. If not repair it and then reinstall grub to the MBR.

How to restore the Ubuntu/XP/Vista/7 bootloader (Updated for Ubuntu 9.10)
[http://ubuntuforums.org/showthread.php?t=1014708](http://ubuntuforums.org/showthread.php?t=1014708)

If you do not have windows CD to run fixMBR.
Restore basic windows boot loader - universe enabled from Ubuntu or liveCD
sudo apt-get install lilo
sudo lilo -M /dev/sda mbr
May show error messages about the rest of lilo missing, ignore, we just want MBR

Then when you want grub2 back from liveCd:

Install MBR from LiveCD, Ubuntu install on sda4 and want grub2 in drive sda's MBR:
Confirm linux partition, change sda4 if not correct, and/or even sda if sdb wanted:
sudo fdisk -l
sudo mount /dev/sda4 /mnt
sudo grub-install --root-directory=/mnt /dev/sda
If that returns any errors run:
sudo grub-install --recheck --root-directory=/mnt /dev/sda
