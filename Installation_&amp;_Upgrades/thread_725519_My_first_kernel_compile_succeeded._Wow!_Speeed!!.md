### Post by keratos on 2008-03-15
Thought you good folk might be interested ...

If you are new to linux, have old hardware, and want a fast distro with lots of packages then xubuntu might be your choice, however you can speed this up to get to the likes of arch and vector. I liked vector linux as it was soooo fast however the lack of packages was a hinder so I returned to xubuntu. But, before doing so, thought I'd dabble with a kernel recompile.Never done this before so I didnt expect it to work, let alone speed my system up and not use any swap... READ ON.

System:
AMD 64 bit LE-1100 (2GHZ pentium 4 equiv)
512 DDR2 RAM
Asus motherboard with K8M890 chipsets (IDE, VGA etc)
5yr old DiamondMax 160GB HDD
Xubuntu 7.10

By default the kernel is bloated and there is lots of things going on taking CPU cycles and memory, that you definitely dont need. This is setup because the kernel needs to account for any system. So, what you do is take out what you dont need.

The process is simple but time consuming.

I'll edit this post with specific details - there is another thread on the forum "Master Kernel" thread, so details are there

But I strongly recommend you spend a day to sit down and do this, if like me you have old hardware but want to stick with Xubuntu and want more speed.

My swap space is never used now!!! Apps open fast and the system is much more responsive. This may be because I used low latency kernel options, not set by default!!
