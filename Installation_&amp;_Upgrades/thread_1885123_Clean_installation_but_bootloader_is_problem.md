### Post by darkod on 2011-11-22
1. That menu is the EFI menu because you are using the EFI boot, not BIOS boot.
2. Also, your HDD has GPT partition table. Personally I would change it with MSDOS, especially for single OS install. You don't need many partitions.

GPT is still new and can create issues sometimes, and you have no benefits in your case of using it.

What I would do:
1. Go into BIOS and turn off EFI boot, in other words configure to use BIOS boot.
2. Boot once with the ubuntu cd into live mode (Try Ubuntu option), open Gparted and write a new blank msdos partition table. This will delete all data on the disk but since this is a new install you probably have no data on it.
3. When you are in Gparted you can write new msdos table in Device - Create Partition table. msdos is usually selected by default but if not make sure it's msdos type.
4. Execute all the changes in Gparted if needed (the green tick mark icon button in the toolbar).
5. When all that is done, reboot again with the cd and do the install again. It should be fine.

Note that grub boot menu doesn't show when you have only ubuntu.

### Post by darkod on 2011-11-22
If by wipe out you mean format, that doesn't change the partition table type. It only removes the partitions.
I prefer manual partitioning during the install, it always gives you more control. Plus if you configure separate /home partition in future you can do clean installs (instead of upgrades) and still keep your data and settings in /home. You only format the / partition and you never format /home.
To create it you need manual partitioning.

### Post by darkod on 2011-11-22
Did you change in BIOS not to use EFI? I believe if you have EFI enabled it creates a small partition for the bootloader files and that bootloader automatically.

PS. You can clearly see in your results file that /dev/sda1 is EFI. Also in the part of the results where /etc/fstab is displayed it says /dev/sda1 was /boot/efi at install.
Sounds like you definitely used EFI the first time.

The EFI boot is not related to the GPT. The change to msdos was only my suggestion. Even with msdos table you will still have that small menu if using EFI.

### Post by darkod on 2011-11-22
No boot loader is installed in the MBR of /dev/sda.

For some reason it is not installing the bootloader. Also, related to that, it doesn't seem to install grub/grub.img file.

From live mode you can try entering as chroot the latest installation and recreate the grub files and install it to the MBR. Because you also specified separate /boot partition, the commands would be:

```
sudo mount /dev/sda1 /mnt
sudo mount /dev/sda2 /mnt/boot
sudo mount --bind /dev /mnt/dev
sudo mount --bind /sys /mnt/sys
sudo mount --bind /proc /mnt/proc
sudo chroot /mnt

grub-mkconfig
grub-install /dev/sda
update-grub

exit
sudo umount /mnt/proc
sudo umount /mnt/sys
sudo umount /mnt/dev
sudo umount /mnt/boot
sudo umount /mnt
```

Reboot and keep your fingers crossed. :)
