### Post by Rubi1200 on 2011-06-04
Hi and welcome to the forums :-)

Please do the following so we can see what is going on:

Boot the Ubuntu Live CD/USB. Choose the option "Try Ubuntu without any changes." Once the desktop loads come back here and do the following:

1. Download the boot info script which is in a zipped file. There is a link in my signature.
2. Once downloaded, move the boot info script by either copying or dragging and dropping the zipped folder onto the desktop and unzip the contents by using right-click Extract here.
3. Open the folder and copy the script to the desktop (you can also drag and drop if you like)
4. Open a terminal and run the following command

```
sudo bash ~/Desktop/boot_info_script.sh
```

This will create a RESULTS.txt file on the desktop. Paste the entire contents of that file back here in a new post. Once pasted highlight all text and click the # sign on the toolbar to place code tags around the text.

### Post by Quackers on 2011-06-04
You appear to have installed grub to the sda2 partition. Grub is normally installed to the mbr of the drive rather than to a partition.
Before you do the following, you should make sure that you have either a Windows XP repair disc or an installation disc, as the Windows bootloader will be over-written.

Boot from the live cd/usb and open up a terminal then copy/paste the following commands in one at a time, pressing enter after each line.
```
sudo mount /dev/sda2 /mnt
sudo grub-install --root-directory=/mnt /dev/sda
```
This should report "finished. No errors".
If it does you can reboot, taking out the cd/usb, and Ubuntu should boot directly.
Open up a terminal again and then run ```
sudo update-grub
``` and you should see the Windows Loader picked up as grub.cfg is run.
If you see this you can then reboot and you should then see a grub menu with the choice of which operating system to boot. Try both, to make sure things work.
