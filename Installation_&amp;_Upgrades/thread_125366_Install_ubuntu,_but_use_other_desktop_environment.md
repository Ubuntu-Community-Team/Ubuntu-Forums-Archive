Anyway, thanks for your suggestions![/QUOTE] I'll be perfectly honest with you: there is no sufficient substitute for iTunes in Linux. Not everyone on this forum will agree with me, but I've tried Noatun, AmaroK, JuK, Beep, Banshee, XMMS, Rhythmbox, Muine, and just about everything that people could suggest as a replacement for iTunes.

They just don't match up.

The closest I've found is Rhythmbox for Gnome or JuK for KDE.

AmaroK is very fully featured, but it's sluggish on my system, for some strange reason.


**AmaroK pros:**
smart playlists
iPod integration
lyrics fetching
global keyboard shortcuts

**AmaroK cons:**
no general library--you have to make a playlist for all songs
library refreshing iffy
some tags are not editable
playcount is by the start of the song, not the end

**JuK pros:**
fast--very responsive
library rescans reliable
global keyboard shortcuts
one general library with playlists, including smart playlists

**JuK cons**:
no playcounts
no iPod integration
some tags not editable
generally lacking in features

**Rhythmbox pros:**
responds to multimedia key keyboard shortcuts
looks sort of like iTunes
does playcounts by the end of the song
supports smart playlists
iPod integration... ?

**Rhythmbox cons:**
refers to some weird ID3 tag that's not v.1 or v.2.
can't edit tags inline
