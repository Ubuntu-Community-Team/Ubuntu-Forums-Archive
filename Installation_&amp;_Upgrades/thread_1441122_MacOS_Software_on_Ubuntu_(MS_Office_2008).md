### Post by srs5694 on 2010-03-28
AFAIK, the only way to run Mac OS X software under Linux is to run machine virtualization software with a full version of OS X running in the virtualized environment. This is tricky from a technical perspective and it's of questionable legality, especially on non-Apple hardware. Like other virtualization solutions, it also consumes massive amounts of resources, particularly if you just want to run one program. (You need to devote RAM and disk space to the virtualized OS, not just to the program you're running.)

In theory, something akin to WINE could be written to enable running OS X software under Linux. AFAIK, though, nobody's written such a program. If you're looking to start a massive Linux software project, this would be an interesting one, but if you just want to get MS Office working, you're far better off sticking with the Windows version under WINE.
