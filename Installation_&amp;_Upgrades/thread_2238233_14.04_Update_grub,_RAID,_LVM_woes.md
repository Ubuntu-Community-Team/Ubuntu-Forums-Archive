### Post by dvarrazzo on 2014-08-07
I've managed to finish the upgrade without problems.

The 'core.img' file (/boot/grub/i386-pc/core.img) is just more than 31k; the "embedding area" seems to be the /dev/hda space before the /dev/hda1 partition, which is about the size required by the image, probably a few bits short.

What I did was to delete the /dev/sda1 partition (a 40Mb Dell recovery partition) and run update-grub and grub-install /dev/sda.

The upgrade procedure went on no problem; it didn't automatically reboot at the end: I would have thought so. However after a manual reboot the machine started no problem (some error message at grub-time maybe, but it restarted nonetheless).
