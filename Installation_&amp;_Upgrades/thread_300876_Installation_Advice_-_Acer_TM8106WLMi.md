### Post by budgie9 on 2006-12-15
I tried loading the Ubuntu on my Acer, a different model. I found I could get Ubuntu to work IF i selected another of the startup option. However I don't know if this will work for you, but worth a shot.
I see no reason why you should not be able to install on to a partition. I do on my main machine, which has 16 partitions, on 1 HD. With XP on C:  Ubuntu and Suse on partitions 12 and 13 with a joint swap being 15  All you need to do is to set a partition  for root / and perhaps a 2 Gb  swap file partition. If you have another partition free I would strongly suggest, make it the /home. That will save you wiping out the /home directory each time you update or what ever. Hope this helps.
