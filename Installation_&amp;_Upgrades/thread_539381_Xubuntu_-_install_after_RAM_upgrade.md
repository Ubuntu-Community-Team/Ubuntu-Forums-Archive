### Post by SyncMaster on 2007-09-26
I can NOT confirm that installing RAM in a System is NO problem.

I am running 7.04 Ubuntu/Gnome (not Xubuntu), all Updates done, on a Centrino Laptop.
I wanted to - and did - replace the single 512 MB DRAM with a 1 GByte DRAM, but then HAL did not work anymore.

The system was running *very* slow, it even took too long for the fan to start after the third boot the temperature sensor shut the system off.. that was a bit of a shock.

The interesting thing is, that it even came into gnome desktop booting. The CPU meter on the top bar would show 100% cpu load, and it would take roughly ten times longer to boot.

The reason that I think it has something to do with HAL is a popup when starting Gnome saying something about HAL no working or HAL not loaded. 

As I have a XP / Ubuntu dual boot, I could verify that the memory is in fact ok (XP booted normaly)

So I put the old 512 back and it runs now as before - and unless someone has an idea of what I need to do, I will wait for 7.10 and do a fresh install...
