### Post by the_lamp on 2006-08-24
How exactly is ubuntu for "everyone" when the package installer and upgrader sucks so much.  Sure if everything is *perfect* it works fine.  But I just tried to upgrade my friends computer to dapper and the damn thing won't run any applications...the apps menu won't open and god knows what else won't work.  IF I have to read through  a dozen different web pages to figure this out then what was the point in using ubuntu?  I might as well have used another distro that actually gives me good performance.  Ubuntu is *** slow.  Gimp actually runs faster in windows XP than it does in ubuntu.

Besides the problems upgrading distro versions with it, if you choose to install a new package you can't uninstall it without configuring it.  I once, on a whim, chose to install gforge.  It wanted to be configured and I didn't have time to mess with that, I just wanted to see what it was.  HOWEVER, apt wouldn't let me uninstall it without first configuring it. I tried to configure it but it said I had the wrong version of postgresql or something. So I had to unarchive the package and blank out the configuration scripts then repackage it and put it in my package repository...only THEN could I uninstall it. That's retarded.  at least with rpm you can delete things without going to so much trouble.  apt is a mess, if ubuntu wants to actually succeed on a large scale it needs a simple package management system that everyone can use.  I can figure this stuff out if I work through it but there is NO WAY that non-technical people could.  It's okay to have a complex package manager IF IT WORKS, but right now I have a non-functioning PC that I'm going to have to either spend a half a day fixing or reformat it and do a fresh install.
