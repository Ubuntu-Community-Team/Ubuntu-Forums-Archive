### Post by mikeym on 2007-09-23
OK,

I've found where I managed to mess it up. I was following the advice on [this post]("http://ubuntuforums.org/showpost.php?p=483971&postcount=10") and I created a *chroot* so that I could link the 32 bit libraries to */usr/lib32*. As per the instructions I moved the original */usr/lib32* to */usr/lib32.local* and added "include /usr/lib32.local" to the file */etc/ldconfig.so* and ran *ldconfig*. I wan't sure about this so I added the same line to* /etc/ld.so.conf* and ran it again.

Now I have found that if I return the */usr/lib32.local* to */usr/lib32* I can get it to start compiling.

I have also found that if I set it up with both directories and run the command with explicit links to the */usr/lib32.local* directoryI can get it to run. 

```

CC="gcc-4.1 -m32" LDFLAGS="-L/lib32 -L/usr/lib32.local -L/usr/lib32 -Wl,-rpath,/lib32 -Wl,-rpath,/usr/lib32i -Wl,-rpath,/usr/lib32.local" linux32 ./configure --verbose

```

Shouldn't it be doing this automatically? Is there somewhere else I should be referencing this new directory?
