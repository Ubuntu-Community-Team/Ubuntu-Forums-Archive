### Post by Mark Phelps on 2010-06-01
Your continued hacking is not making things any better -- because you've not seen the basic problem ...

Somewhere in your thrashing around, you managed to replace the Win7 boot partition with a Linux boot partition, effectively, wiping out the Win7 boot loader -- clearly, indicating you had no idea regarding what you were doing.

Since you were going to be hacking around with partitions, you made sure to create and burn a Win7 Repair CD for your friend's machine?  Right??

I'm guessing you did not ... so, go to the link below, download and burn the ISO image of the appropriate Win7 Repair CD:

[http://neosmart.net/blog/2009/windows-7-system-repair-discs/]("http://neosmart.net/blog/2009/windows-7-system-repair-discs/")

IF you're really lucky, you SHOULD be able to repair your friend's machine by doing the following:
1) Insert the new Win7 Repair CD you made
2) Boot from that CD
3) Run Startup Repair three times -- yes, three times.  It takes a total of three passes to completely repair a Win7 boot problem.

IF, and only IF, this works, you will then be able to reboot Win7 on your friend's machine.

Once you get that working, come back and find out how PROPERLY to configure dual-boot on a machine with an existing install of Win7.

And, oh yeah, show some patience!  Thrashing around nearly always makes things worse, not better.

### Post by darkod on 2010-06-01
Just to add, if you use this automatic method to repair win7, that will overwrite your grub2 on the MBR. So if the repair succeeds the computer will start booting win7 directly like ubuntu is not there.
