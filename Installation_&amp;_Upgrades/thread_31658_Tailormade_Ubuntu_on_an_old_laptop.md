### Post by eivind on 2005-05-04
Hi, all

I am about to install linux on an elderly laptop (quite good, actually, but these days it's a little oldish): A Dell CPi celeron 400/256. I thought about loading it with one of the many "lightweight" distributions - Luit Linux seemed fine (xfce with desktop icons), damn small linux is another alternative. Both of these are Debian based. After a bit of browsing, I came to the obvious conclusion that there are so many distributions for so many purposes that in my case, I have two possible ways of doing it:

1. Find a distribution which exactly fills my needs, which are: Good hardware support, i686 support (at least a prepackaged i686 kernel should be available), small footprint, few services running out of the box, xfce or lightweight window managers, hal (I know how to mount a floppy, but it shouldn't be neccessary to do it. I want to access it by clicking on an icon), icons on the desktop background and debian based (apt-gettable). There are numerous distros which fill these criteria, I'm sure. I'm just not aware of all of them yet.

2. Take an existing distribution and trim it to purpose.
I have Ubuntu on my main laptop, and as it is working so well, I thought about installing the "server profile" on the new but old laptop, and add x and xfce/others according to my taste. 

In any case I would like my - what should we say - distro-specific wishes to be fulfilled.

Just before embarking on this joyful and exciting journey: Which way would you choose? 1 or 2? Suggestions and tips (what to do and what not to do) are most welcome!


Eivind, 
Norway

### Post by Xerampelinae on 2005-05-04
If you're just playing around with different distributions, then it's all fine and good.  But in my experience it is nice to have the same distribution(s) installed on any machines that I actually want to use for long periods of time.

Keeping track of the differences gets tiresome.
