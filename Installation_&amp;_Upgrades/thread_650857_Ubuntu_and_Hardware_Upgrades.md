### Post by phibit on 2007-12-26
This post is to discuss how Ubuntu (specifically Ubuntu 7.10 AMD64) handles hardware changes in a system. I'll start by describing my situation a bit:

My current setup is as follows:
- AMD Athlon 64 3500+ (socket 939)
- Asus Motherboard w/ VIA chipset
- 1 GB DDR RAM (2x 512 MB)
- Nvidia GeForce 4200Ti
- 200 GB SATA 1 Hard Drive  <-- UBUNTU is install here.
- 500 GB SATA 1 Hard Drive  <-- Windows XP installed here (rarely used)

I have recently purchase what I'd consider a major upgrade. I will be swapping out the motherboard, CPU, RAM and video cards, to be replaced with the following:
- Intel Core Duo E6300 (socket 775)
- XFX nForce 680i LT Motherboard
- 2 GB DDR2 RAM (2x 1024 MB)
- 2 x Nvidia GeForce 8400 GS (in SLI)

My goal is to keep my current Ubuntu install, and simply pop the 200 GB Hard Drive in with my new config. 
The way I see it, both CPUs are 64 bit, so running a 64 bit operating system won't change that much. This view is likely naive, but oh well.

I know that for Windows XP, sometimes you can get lucky with a hardware change, and get away with just doing a Windows Repair, so that all the new drivers are properly installed.

My main question is, will this be the case with my Ubuntu installation? Will I be able to boot off of my old hard drive and conserve all my data/settings/programs ?

I'd be interested to hear about people's hardware upgrade experience, even if you don't have a direct answer to my question.

Thanks.
