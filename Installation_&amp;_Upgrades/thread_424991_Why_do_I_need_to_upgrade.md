### Post by pellgarlic on 2007-04-27
they say "if it ain't broke, don't fix it" :) (whoever "they" are...)

seriously though, if your setup works for you, and you don't feel like you're missing any functionality, leave it alone - there are always potentially going to be problems when upgrading any systme, be it linux, windows or mac, and if you are happy with what you've got, there's no point in risking it.

some users like to have the most cutting-edge 3d wobbly window stuff, and all that. some like things just to be simple and stable. that's the reason i was drawn to linux in the first place - the sheer diversity of possibilities. if it does what you want, don't mess with it, that's what i say.

eventually, you might find that some newer multimedia files might not work until you upgrade your codecs or whatever, but it'd probably take ages, and it doesn't seem often that this requires a distro upgrade just to achieve it.

personally, i *will* be upgrading to 7.04 (once i'm finished backing up the ~200gb of files i have spread about over 8 partitions...) but that's because i want to, not because i need to.

[edit:]

let me add a caveat: i have upgraded from 5.10 > 6.06 > 6.10 and each time, the only real problems i had were with "proprietary" stuff, i.e. my nvidia graphics card drivers, and my win-modem drivers, both of which i had compiled against the currently-in-use kernel. there is nothing intrinsically risky about upgrading - it more often than not goes about 99% perfectly, and what problems there are can usually be resolved with the help of some of the more experienced members on this forum and other linux forums, and and a new distro release often brings some really new cool stuff, but my point was that if you don't feel like you're missing anything, it's probably not worth doing it. if you start to feel like the release you're using isn't cutting the mustard any more, then it'd be time to think about upgrading.

sorry for rambling on ....... i'll stop now. :)
