### Post by Catharsis on 2010-05-27
Try booting into Recovery Mode. At the grub menu, select the recovery mode kernel and select "boot normally". If that doesn't do anything try "failsafeX".

Also try editing the kernel lines like before (in the default kernel, not the recovery kernel), but just remove "quiet splash" and boot. If that doesn't work try adding "nomodeset" like before. (Trying to see if it's a problem with the splash screen, aka plymouth. This is why removing "splash" is important too.)

If "nomodeset" doesn't work, try "xforcevesa".
