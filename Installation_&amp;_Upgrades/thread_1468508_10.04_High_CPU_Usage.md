### Post by CryptoPaul on 2010-05-02
Have you ran checkbox (System - System Testing)?

There's a known bug [https://bugs.launchpad.net/ubuntu/+source/checkbox/+bug/556636](https://bugs.launchpad.net/ubuntu/+source/checkbox/+bug/556636) that causes high CPU usage after checkbox has ran.


You should be able to see the running process...

In system monitor have you checked 'View - All Processes'

Start 'Top' in a terminal window, what is that showing.

### Post by Zoharc on 2010-07-22
Had the same Issue. 

On closer inspection it seemed as if Xorg was taking about 40-50% of the CPU. However, I noticed that removing metacity greatly affected CPU usage. Without a window manager I was down to 2-4% CPU usage. Afterwards, and installed openbox.  CPU is now On-demand, at 42%, and utilizing 4-9% average for 2 displays and a swiftfox process.
