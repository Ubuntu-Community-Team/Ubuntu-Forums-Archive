1. if your source package uses GNU autoconf and automake, then it usually has/generates configure script there too
2. make is for GNU make utility, see *man make*
3. install you reference here is for different purpose, see *man install*.

when you invoke 'make install' you actually run the install *target* on Makefile
that was generated by configure.

You should read about GNU make, autoconf and automake to understand this process.
There's also easy-to-read wiki articles like [http://en.wikipedia.org/wiki/Make](http://en.wikipedia.org/wiki/Make)
