### Post by moeknows on 2007-11-20
I'm trying to boot the Ubuntu 7.10 live cd.  I have a pci nvidia fx5200 256mb video card.  I have a 19-inch lcd attached to the card via the 25pin dsub.  Ubuntu freezes during the startup/install process.  This is a hard lockup where ctrl-alt-del doesn't work, and the num lock light doesn't turn on when num lock is pressed.  To reboot I have to hold the power in a full 5-10 seconds.  Selecting the "Start Ubuntu in safe graphics mode" option does not help.  Nor does using noapic nolapic or pci=noacpi in the kernel options.  If i switch over to the built-in video card ubuntu boots up (with a fairly weird resolution that doesn't fit my monitor, but at least the live cd does boot with the built-in card.)

I've seen a few people post on the forums with similar problems, but haven't seen a working solution.

Any suggestions?

### Post by gsiliceo on 2008-04-29
Try adding this line
> blacklist intel_agp
to the end this file
> /etc/modprobe.d/blacklist
You need to open your text editor with sudo ie:
> sudo gedit /etc/modprobe.d/blacklist
