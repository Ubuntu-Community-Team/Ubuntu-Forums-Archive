```

Section "Device"

Here's some EDID Instructions I wrote in my graphics sticky in Post              #[**275**]("http://ubuntuforums.org/showpost.php?p=10907238&postcount=275").

Look where your Mode declarations are in the Monitor Section above?  What are highlighted is your "mode names."  These are just ID or variable names and must be matched in spelling and case when you reference them...  Keep this in mind.

Now look in the Monitor Section above and use the color coding to reference the respective modes. You have an "@" character for each instead of "_".  What those characters should be to correct this is a space character... When I do it, I use a space (" ") there, so it matches up and is consistent with mode names that would be generated by a box that is connected to a monitor that stores EDID data.  Look at this example:
```
[FONT=monospace]Section "ServerLayout"
  Identifier "layout1"
  Screen "screen1"
EndSection

[/FONT][FONT=monospace]Section "Device"
  Identifier "device1"
  VendorName "nVidia Corporation"
  BoardName "NVIDIA GeForce 6100 and later"
  Driver "nvidia"
  Option "DPMS" "false"
  Option "DynamicTwinView" "false"
  Option "AddARGBGLXVisuals"
EndSection

[/FONT][FONT=monospace]Section "Monitor"
  Identifier "monitor1"
  VendorName "PnP laptop"
  ModelName "2036"
  HorizSync 30-83
  VertRefresh 55-75
 [COLOR=Green] Modeline "1280x1024 75.00"  138.54  1280 1368 1504 1728  1024 1025 1028 1069  -HSync +Vsync
  Option "PreferredMode" "1280x1024 75.00"
[/COLOR]EndSection

Section "Screen"
   Identifier "screen1"
  Device "device1"
  Monitor "monitor1"
  DefaultColorDepth 24
  Subsection "Display"
    Depth 24
[COLOR=Green]     Modes "1280x1024" "1024x768" "800x600" "640x480"
[/COLOR]  EndSubsection
EndSection

[/FONT][FONT=monospace]Section "ServerFlags"
  [/FONT][FONT=monospace]Option "DontZap" "1"

[/FONT]

### Post by MAFoElffen on 2011-09-27
> **jdneate said:**
> Right - I loaded the NVIDIA v96 driver via Administration>Hardware Drivers as part of the experimentation; I'm sure that the system did not run after restart.

Not knowing very much about Ubuntu/Linux .... [COLOR=Teal]Would the v96 driver generate it's own xorg.conf file[/COLOR] and modify the GRUB loader to remove/add NOMODESET when it is installed ? 

Having got an acceptable system I am reluctant to modify it unless I can understand what the implications are and how to get back to the working system should it not work !
Yes it will. During the install it will generate a new xorg.conf (ergo the need to backup your old one.)  If it doesn't run
```

sudo nvidia-xconfig

```***It may find the montitor modes or not- what your original assumption of the Monitor not storing the EDID data may have been correct (which we all assume was right).  If so. it will fail until you add the modelines and changes from the old xorg.conf to the new one that was generated...  Thats one of the reasons I asked you to backup the old one...  You can cut and paste the changes in.  Without those modelines and other changes, it will try "nvidia-auto-select +0+0" to probe the monitor for suitable modes from the EDID in the monitor // which doesn't work for your's, because there is not stored EDID data to use for that to work...

The other reason I had you backup the old one, was that if you don't like the new driver, you can overwrite the new xorg.conf with the backup xorg.conf and you will be right back to where you were...  What tells X-Windows what driver to use and what settings to use for that driver... is the xorg.conf file.  

If there is no xorg.conf file then Xorg goes into an "auto" mode, where it probes hardware, tries to find a suitable driver and uses default settings for it. It accepts other input from KMS to try to help with that.  This doesn't always work and surely didn't for you in all resolutions.

*** By the by-- once that driver is installed and working, there is a way to confirm if your monitor has EDID data stored or not.  In the nvidia-settings applet, there is an option to retrieve the EDID data of a monitor and an option you can use in the xorg.conf file to use an external custom edid file.  If there was EDID data stored, then that file would contain hex data.  If that file did come back as good, then you could use that data to use the full capabilities of that monitor... When that happens like that, sometimes (rarely) there was a problem in how it tried retrieving the data.  When monitors and TV's first started storing that data, it wasn't universal and standardized.

If that file comes back as blank or 0 bytes, then it confirms that there was no stored data.

*** Installing the driver does not change any of the grub files.

The xorg.conf file is

spe@jdneate:~$ cat /etc/X11/xorg.conf

Section "Monitor"
	Identifier	"Configured Monitor"
