### Post by Mark Phelps on 2022-08-23
Recently have gone back to using Linux distros on a daily basis and notice NOW, since upgrading my PC to UEFI and a combination of SATA SSDs and NVMe SSDs, the Windows boot stanzas are no longer correctly generated -- making it impossible for me to select some Windows installs from the grub menu.

I have five SSDs in this PC, one for Linux distros, one (NVMe) for shared data, and three Windows installs: one to a SATA SSDs and two to NVMe SSDs.

The drives are as follows:
/dev/sda (ssd) Windows 11 Beta -- ESP is on sda1
/dev/sdb (ssd) Linux - ESP is on sdb3
/dev/sdc (nvme) Windows 11 Pro - ESP is on sdc1
/dev/nvme0n1 (nvme) DataVol (shared storage) -- no ESP
/dev/nvme1n1 (nvme) Windows 10 Pro - ESP is on nvme1n1p1

The problem is that the grub.cfg file generated has the same UUIDs for the Windows 11 Beta drive ESP as it does for the Window 10 Pro drive ESP.
