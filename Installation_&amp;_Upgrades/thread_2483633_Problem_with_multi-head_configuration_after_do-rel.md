### Post by SagaciousKJB on 2023-02-05
Hi all, I am attempting to upgrade my install of Xubuntu 18.04.6 to 20.04 in light of the former's EOL coming up soon. I am using the 'do-release-upgrade' command line tool, and it is going okay, except for one very important aspect.

On my 18.04.6 installation as it is, I am using NVIDIA's proprietary 390 driver (I need this driver specifically for use with NVIDIA's hevc encoder) installed via the 'Additional Driver's tool, and with it I use a standard monitor and a TV for a dual-screened, multi-head setup. This setup has 'xinerama' installed, so that if I open an application on the monitor's screen, I can simply grab it and move it over to the TV. Likewise, if I maximize an application's window, it only maximizes it for one screen. I didn't do anything special to achieve this configuration, other than simply install the driver through the 'Additional Driver' GUI and let it set everything up on its own.

The problem is that after the upgrade, this behavior changes. Instead of the past behavior, 'xinerama' will now make each application span the width of both screens. This means my XFCE panels span both screens, and if I try to maximize a window it will make it maximize across both screens. This is very undesirable for me, but as I understand it, is the default behavior of xinerama, so I have been searching, for but unable to find, any kind of configuration option to achieve the previous behavior. I think it's important I mention that this behavior happens with the very same /etc/X11/xorg.conf file.

What is more interesting, is that if I do a clean installation of Xubuntu 20.04 instead of upgrading, and then install the NVIDIA 390 driver, when I restart it has the same behavior I previously had with 18.04. I forgot to check if it generated the same /etc/X11/xorg.conf file, however, so I will go back and check that after making this post, but at this time I'm currently kind of burnt-out attempting upgrade/installs and then restoring the backup.

I wonder if anyone is aware of any specific xinerama configuration option that controls this, or why it might be behaving this way with a 'do-release-upgrade' versus a clean install. I really do not want to have to do a clean installation, as I have a number of things configured on this system that I do not exactly remember how I set up.

### Post by SagaciousKJB on 2023-02-06
well, it turns out it was a good idea to go back and look at what xorg.conf was being generated as with the fresh install. It turns out, it was not generating a xorg.conf at all. So I upgraded, and then deleted the xorg.conf file, and I noticed that the TV monitor was being mirrored. I looked around in the XFCE Display Settings (which were previously disabled) and was able to tell it to *not* mirror the display, and instead it started treating the TV display as a second display and not spanning applications across them as my past behavior had been.

So, it's kind of confusing, because it seems that my past installation was ignoring the xorg.conf file in favor of some other type of configuration, but I could swear I remember the XFCE Display Settings being disabled so that's why I didn't think of this.

But yeah, the basic steps I took were as follows:

1. Use "Additional Drivers" to enable the default open-source nouveau drivers
2. Upgrade the system
3. Use "Additional Drivers" to enable the NVIDIA proprietary driver
4. Use XFCE Display Settings instead of the NVIDIA Settings program to control the display
