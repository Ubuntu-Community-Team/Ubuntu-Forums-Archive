### Post by yancek on 2016-05-04
Dual booting windows/Ubuntu with UEFI is explained at the official Ubuntu site at the link below.

[https://help.ubuntu.com/community/UEFI](https://help.ubuntu.com/community/UEFI)

To determine if you are using UEFI, go to the link above and run the command just below the section "identifying if the computer boots the HDD in UEFI mode"
Just copy/paste the command in a terminal.  If you have GPT, then you are almost surely using UEFI as that is needed for windows.
From a terminal, running the command:  sudo fdisk -l(Lower Case Letter L in the command) will show drive/partition info.  You can post it here for someone to explain.

### Post by oldfred on 2016-05-10
This is a list of legacy drivers from nVidia.
 Legacy drivers by GPU model 
[http://www.nvidia.com/object/IO_32667.html](http://www.nvidia.com/object/IO_32667.html) 
And it is the same as the recommended driver shown above by Ubuntu.

As posted by Yancek, you often need nomodeset to get video working in lower quality mode. And then can install the proprietary drivers. If running full Ubuntu, not Lubuntu with that old of a video card you may need the proprietary driver and may still not have great performance. Unity requires newer systems, but if rest of system is newer & you can run Windows 10 ok then it should also be ok.

You normally install drivers with System Settings, Software & Updates icon, and additional drivers tab. Or you can use command line.
Do not install nVidia drivers from nVidia, and if you have a nVidia driver installed be sure to completely purge before installing another or you get conflicts.

### Post by oldfred on 2016-05-19
The default mount is read/write and will not mount hibernated nor chkdsk needed NTFS partitions.

But you can manually mount in read only (ro) mode. If you have multiple NTFS partitions you can create separate mount points for each. 

       mkdir /media/win
sudo ntfs-3g -o ro /dev/sda1 /media/win
