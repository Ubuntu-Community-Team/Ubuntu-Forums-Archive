### Post by darkod on 2012-02-20
Out of whatever reason, grub2 didn't install in full. You can try to add it from live mode. You will need to "enter" into your ubuntu install which is on sda5, and run commands as if ubuntu was booted (the live mode running from cd doesn't count as the hdd installation).

First do the necessary mount commands and chroot into it:
```
sudo mount /dev/sda5 /mnt
sudo mount --bind /proc /mnt/proc
sudo mount --bind /dev /mnt/dev
sudo mount --bind /sys /mnt/sys
sudo chroot /mnt
```

Just in case, remove fully grub2 and install it again:
```
apt-get remove --purge grub-pc grub-common
apt-get install grub-pc grub-common
grub-mkconfig
update-grub
grub-install /dev/sda
```

Hopefully it is fully installed after that including the part of grub2 that goes on the MBR. Exit the chroot and unmount everything:
```
exit
sudo umount /mnt/sys
sudo umount /mnt/dev
sudo umount /mnt/proc
sudo umount /mnt
```

Restart and you should see the grub2 boot menu.

### Post by darkod on 2012-02-20
You selected correct. When it has a number (like 5) it means partition number 5. When it doesn't, it means the MBR (Master Boot Record) of the HDD. That's where bootloaders go, not on partitions.

Only in some specific situations you would want a bootloader installed onto a partition instead of the MBR.
