### Post by mohapi on 2006-01-22
I'm working with an older laptop that has a BIOS limit on hard drive size (I think it's 8GB for this particular machine, but I'm not 100 percent sure). If the drive is larger than that limit, the machine sits and hovers at startup. 

Ordinarily, with Windows, I would rely on a dynamic drive overlay (probably Maxtor's Maxblast, but that's only because I've used it before) to allow the system to boot off a drive that's bigger than that.

Does Ubuntu require a DDO to use "oversized" drives like that? I'd like to put a bigger hard drive in it, but naturally I don't want to spend the money if it's not possible, or more trouble than it's worth. This would be a Ubuntu-only setup, not a dual boot.

Thanks in advance.

### Post by mohapi on 2006-01-28
I'll go ahead and answer my own question, since I managed to troubleshoot my way through it. 

I installed a 20GB 5400rpm Hitachi drive in the same machine, and had no troubles installing Xubuntu to it. The machine boots only a little bit faster, although much of that is the fault of the the connecting hardware, and not the drive itself.

I used no DDO, and the hard drive reports over 17Gb free. 

Ubuntu trumps Windows again. ... :D
