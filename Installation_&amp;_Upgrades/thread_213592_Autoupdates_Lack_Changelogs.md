### Post by mattruby on 2006-07-11
The automatic package updater on my Ubuntu Desktop is now offering me 38 updates, mostly OpenOffice.org, cups and kernel-2.6.15 packages. Only nvidia-glx, linux-restricted-modules-2.6.15-26-386 and linux-image-2.6.15-26-386 include a "Changes" log. 10 of the updates, mainly cups, have been offered for more than a day without changelogs.

I have noticed that more updates are offered without changelogs lately. The frequency seems to increase with each release. Changelogs are the absolute minimum documentation - actual release notes addressing upgrade benefits should be the rule, not the exception.

What's wrong with the update publication system? How can we ensure that changelogs, which responsible users require to determine whether to upgrade, are always delivered with the software?
