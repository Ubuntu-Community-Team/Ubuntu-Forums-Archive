Its been a few months since you first posted this thread but I would like to know if you managed to solve the problem.  I have the same motherboad with similar video card (Nvidia 7800 GS AGP) and had similar problems when I tried Ubuntu 7.10 back then when it was released.

It seemed act up after watching videos for a while.  I found out that the issue was with Nvidia drivers at the time (cant recall version) as other people had same problem.  I tried other driver versions from Envy but same problems.  I tried PCLinuxOS and same problems with the default driver.  However, i was able to install an older driver that worked just fine.  No more scrabling.  I can't recall which driver version it was but I am hoping that Nvidia would have fixed the problem by the release of Ubuntu 8.04.

Now with regards to the 3D desktop effects the problem may be with VIA chipset and AGP compatiblity.  My system would freeze as well when using 3d effects or testing glxgears for more than 10 minutes or so. 
All 3D graphix usuage was unstable.  I couldn't solve this problem so I decide to go back to XP.  Incendently by going back to XP I may be closer to a solution to this problem.  You see... 3D effects (ie. games) was also unstable in XP.  After tearing out half my hair I was able to fix it by reducing the AGP speed to 4X using Rivatuner. Unfortunately, our motherboard does not allow AGP speed change in the BIOS.  But since doing it at the driver level through Rivatuner my system has been solid like a rock playing all sorts of windows games.

As for Ubuntu, it be the same situation where the problem can be solved by reducting the AGP speed.  However, I searced the forums and was unable to find out how to do it........only solutions the Ubuntu community gave was to do it through the BIOS which our cheesy motherboard does not provide.  Hope you are still using Ubuntu and found solutions to these problems.  If you have could you let me know what you did or how we can change AGP speed.
