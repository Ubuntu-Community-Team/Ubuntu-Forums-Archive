### Post by jorge-salk on 2015-03-16
Another user with the same issue but fixed it without needing to reinstall by running the following after a single-user boot up:

update-initramfs -u

Info behind that is here: [http://askubuntu.com/questions/209702/why-is-my-raid-dev-md1-showing-up-as-dev-md126-is-mdadm-conf-being-ignored](http://askubuntu.com/questions/209702/why-is-my-raid-dev-md1-showing-up-as-dev-md126-is-mdadm-conf-being-ignored)

Seems at least for me the drive device names changed, not sure about the blkid's as I did not check that.
