### Post by earth_walker on 2006-03-24
I thought I'd post this in case anyone else has seen this behaviour.

I just tried to install Ubuntu Breezy on an old computer here (High end PIII with tons of memory, plan to turn it into a server). I selected the base install, repartitioned the drive, and let it rip. Everything went smoothly until the creation of the first user account - it fell into an endless loop asking for password and then asking for the user. I escaped out and continued with the installation, hoping that the user and password had been saved. The prompt came up again later in the install, again falling into an endless loop.

The rest of the installation went ok, the only difference being that it asked me lots more questions than usual - I assume that's because I had escaped into advanced mode.

On first boot, gdm came up normally, but didn't take my password. So I rebooted to see if it would prompt me again or otherwise resolve the problem.

The installer came up again, this time to download packages. I let it do its thing, thinking perhaps an update would fix the bug, and when gdm came up again, it was Edubuntu instead of Ubuntu! and still no user set up.

I put the install cd in and rebooted, repartitioned, and everything is installing fine as I write (including setting the user account)!
