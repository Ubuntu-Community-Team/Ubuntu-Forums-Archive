### Post by psusi on 2011-06-21
A few points:

1)  You need to actually delete the raid array with the bios utility or dmraid -E.  Simply turning off the raid support in the bios leaves the raid signatures on the disk, which Ubuntu will continue to recognize.

2)  mdadm raid is better supported than fakeraid, but for your purposes, the fakeraid should work just fine.  If you want to install Ubuntu to the fakeraid, then go ahead and install it there.

3)  You only need one partition for the root ( / ), and optionally one for swap.  If you have plenty of ram and don't intend to hibernate, then you can go without swap if you want.

4)  It sounds like you got bitten by the unfortunate installer user interface.  After choosing to install beside windows, you are presented with a screen showing the disk with a divider between the windows partition and the new linux partition so you can adjust the size.  You probably hit the button underneath telling it to go ahead and use the entire partition instead of dividing it in two.  Those buttons really need to be removed because they just confuse people.
