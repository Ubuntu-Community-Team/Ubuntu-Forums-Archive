### Post by rockney on 2009-11-02
After spending all weekend preparing to upgrade, then downloading and upgrading from a clean 9.04 server install to 9.10 Karmic the server functionality died.

The box: An appliance-style Intel Atom 1.6GHz, 2GB DDR2, 32 GB SSD with fresh 9.04 Jaunty server + XWindows + DTC and web-related components. Ran well, but with the new shiny 9.10 we thought we have an easy time dropping a clean 9.10 onto the 9.04 box,

After the upgrade --

 1) Performance is absolutely glacial, 14 minutes to boot to XWindows with no other heavy servcies, 9 minutes to login and see the desktop, over 4 minutes to open Firefox.  Very weird.

 2) Packages messed up, removed, and otherwise won't start. The web sites and configuration work seem to be a loss at this point.

 3) XWindows always starts, even if the box was configured to not start it at certain runlevels.

So now this new server is in an unknown state. Some things might be fixable ... but the underlying slowness seems to go deep into the system and does not seem to be am easy fix.
