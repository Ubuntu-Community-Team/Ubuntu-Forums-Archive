### Post by Onesimus on 2007-09-23
As a new user as of the release of Feisty Fawn in April, I think that the Ubuntu support could be improved.  For example, it is likely that new users are migrating from a version of Microsoft Windows and would prefer to dual boot their PCs.  However, there appears to be no support documentation on how to achieve this - the new user has to apparently trawl through a large number of threads on the forum (sometimes giving conflicting advice).

What would be useful would be to have dedicated web pages with clear instructions on how to dual boot their machines.  If such web pages exist then they must be well hidden for if a user of 6 months is unable to locate them - what then the new user ?

In addition, a web page with clear instructions (and screenshots) on how to upgrade to Gutsy Gibbon might be helpful.  For example, do I need to download the CD image, or will the repos automatically inform me that a new version is available, or something else ?

### Post by southernman on 2007-09-23
It is not my intention to be sarcastic in this reply... but it's likely to sound that way! ;)

What one considered to be clear instructions, may not be clear instructions to another. The same can be said for Operating Systems... What works for one, may not work for another.

My guess is the ole saying holds true here, "No need to reinvent the wheel." Doing a "really quick and dirty" search on google for [howto dual boot windows ubuntu]("http://www.google.com/search?q=howto+dual+boot+windows+ubuntu&ie=utf-8&oe=utf-8&aq=t&rls=com.ubuntu:en-US:official&client=firefox-a") returns 1.85 million hits. Apparently they all work for someone... but perhaps only a handful of them may be written precise enough to work for an individual. If one was to create a howto so that anyone/everyone could apply it, then it would far to long for most people to care to finish it. The different "technical ability" examples could go on for pages/folds.

I do admit, that a lot of documentation available is outdated to some extent, some more than others. Realizing though that all of said documentation is written by volunteers that do their best at contribution. Although not stated in most docs, a lot of things are left for the readers / end users, as an exercise to take the docs at hand and tweak it (further research, trial and error, and just plain ole fashioned experience over time) as need be. This is regardless of OS of choice, be it Windows, GNU/Linux, MacOS, BSD, and all others. For instance, a brand spanking new end user of Windows doesn't just sit down at a computer and auto magically know everything they need to know in order to use it. Just to add, Windows docs (if you can freaking find them) are the most end user unfriendly I've ran across. (pot > kettle > black) :p

Bottom line is this. Even if someone from Canonical wrote a well laid out howto for dual booting, people would, without fail, complain it's incomprehensible (meaning to them it is).
