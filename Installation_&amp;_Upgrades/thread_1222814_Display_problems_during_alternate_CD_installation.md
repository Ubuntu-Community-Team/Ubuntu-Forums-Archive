### Post by lranr on 2009-07-25
I tried installing Kubuntu 9.04 using the alternate CD and encountered problems with the display.  (I marked this thread as "all variants" because I assume that the various alternate CD installation applications are basically the same.)

This is a new machine with a Gigabyte EG45M-UD2H motherboard.  The monitor is an HP LP1965 1280 x 1024 LCD monitor with DVI (only) input.  Starting up with the alternate CD seems okay: the monitor goes into 640 x 480 mode and the graphical startup screen appears.  Once I choose to install, though, the screen gets shifted downward almost halfway and rapidly flickers like crazy.  You could still manage to navigate through the installation, I guess, but it's painful to look at.

I tried using a VGA-to-DVI cable.  The monitor stays in 1280 x 1024 mode and there are no problems with the installer.  I also tried a VGA connection with an older CRT display and it's fine there as well.  The regular versions of the Ubuntu and Kubuntu installation CDs look normal using DVI-to-DVI.

So this seems to be a problem with the DVI connection.  Could this be a problem with (selective) DVI output from my motherboard?  Are there known DVI-related problems with the text-based installer?  Thanks.
