### Post by oldfred on 2020-05-08
You seem to have it mounting in fstab, but it is not mounting?

You can try fsck.vfat on the ESP. It needs to be unmounted, not sure since not mounted if it is ok. Normally run from live installer and check that any partition is not mounted before running any fsck.
man dosfsck
sudo fsck.vfat -t -a /dev/nvme0n1p1

Since NVMe drives better to use noatime parameter. From mine:
```
# / was on /dev/nvme0n1p2 during installation
UUID=b913e1bc-6f08-4984-b653-9a604d95470a /    ext4    noatime,errors=remount-ro 0    1

```

### Post by oldfred on 2020-05-08
They changed default mount. I think for security reasons. I do change mine back to defaults. Boot-Repair changes to defaults, but that may open system to writes from a rouge process. I may change back to 0077 but like to see & sometimes edit entries in ESP. 

The 0077 prevents access.

14.04 fstab entry defaults
UUID=FD76-E33D  /boot/efi       vfat    defaults        0       1
16.04 fstab entry umask=0077
UUID=68CD-3368  /boot/efi       vfat    umask=0077      0       1
