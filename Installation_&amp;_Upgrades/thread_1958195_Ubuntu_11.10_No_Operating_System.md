### Post by darkod on 2012-04-14
The script clearly says on the top that grub2 is installed on /dev/sda, not the partition /dev/sda1. So, that is fine.

Also, there is no problem using ubuntu with only root partition, you don't need to create swap. There is no entry for swap in /etc/fstab, so that looks fine too.

Some computers can have a problem if there is no boot flag set on any partition, because they are used expecting a boot flag which windows always uses. But you do have the boot flag on /dev/sda1 so that is ruled out.

Do you have maybe EFI bios? Does it expect EFI boot?

Because your system looks set up for standard BIOS boot, not for EFI boot (for that you need to have EFI system partition which doesn't exist). If somehow it does expect to boot with EFI, the message might be because of that.

I can't think of anything else right now.
