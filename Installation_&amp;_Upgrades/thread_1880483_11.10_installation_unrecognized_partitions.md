### Post by darkod on 2011-11-18
Well, after trying most options, I think you need to consider drastic measures.

1. Do you have any data in your ubuntu that you need to copy? If yes, it doesn't seem possible to access it. If no, no problem there if ubuntu is deleted.
2. Boot win7 because it still boots fine, and open the Backup & Recovery tool. Make the rescue disc which can boot the computer and restore a backup. After making that disc, make a full backup of your win7 partition on external HDD. Note down the size of your windows partition(s).

When that is done:
3. Boot ubuntu live mode, open Gparted, select the HDD and go in Device - Create Partition table. By default it offers to create a new blank partition table of msdos type. Select that. NOTE: THIS WILL DELETE ALL PARTITIONS ON THE HDD!!!
4. After the new partition table has been written, still in Gparted create one (or more depending how much you have now) ntfs partitions for windows, with the same size(s) as you noted them down earlier.
5. Then boot the computer with the windows rescue disc and restore the backup of your windows.
6. Boot few times to make sure win7 is booting fine.
7. Proceed installing ubuntu into the unallocated space left on the HDD using manual partitioning (or some of the auto options if it can use the unallocated free space).

That should be it. Unfortunately I don't see another way to overcome this partition table problem.
