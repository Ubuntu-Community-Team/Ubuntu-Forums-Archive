### Post by 403forbidden on 2009-10-09
It was working until yesterday.

The archive webserver seems to be mis-configured. While the standard for archives expects the URL to [http://server-ip/ubuntu](http://server-ip/ubuntu), in.archive.ubuntu.com seems to starts with /.

Changing the sources.list URL to something like this and it will work fine.

  deb [http://in.archive.ubuntu.com/](http://in.archive.ubuntu.com/) jaunty-updates multiverse

How do we contact the archive maintainers to report this issue? Which mailing list do they work on?
