### Post by dnedzel on 2008-07-19
I am a new user here, just thought I would post my experience with the latest upgrade.

I ran the upgrade process and managed to get the machine into a state where it won't boot.  I was on 7.10, going to the latest version via the upgrade button on the system updates window.  It got to a point where the terminal window said something about setting the bluetooth services, then it stopped responding.  I tried starting/stopping the bluetooth services, but the upgrade process remained locked.  There was no option to cancel/rollback the update, or ignore the error and complete the upgrade.  Eventually there was nothing else to do but boot the machine.  Now the machine gets past the prompt for userid/password, then locks before painting the GUI, so it is non-functional.  Luckily I don't have much on this machine so I haven't lost much, but the upgrade process was disappointing.  I think it also should suggest a backup and give you an out to do that before the upgrade starts.  Again, not a problem for me on this machine, but could be for less experienced users.  A cancel/rollback feature in the upgrade is a must I believe - you just can't get the user to a point where the only option is to boot the machine during an upgrade - there has to be an out.

The hardware is: Winbook laptop model TL30/Z35Fm, 2GB ram, ~2GHz processor X86. 

My next step is to download the ISO image for 8.x and re-install the O/S.
