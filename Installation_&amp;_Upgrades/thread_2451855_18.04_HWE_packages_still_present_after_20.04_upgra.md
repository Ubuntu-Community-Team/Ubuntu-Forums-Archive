### Post by Impavidus on 2020-10-12
The linux-generic-hwe-18.04 package is to pull in kernel 5.4 on Ubuntu 18.04. This is already the default kernel on 20.04. The package exists on 20.04 too, to make sure the upgrade happens correctly, but it's described as a transitional dummy package. It only serves to pull in linux-generic, which pulls in the default kernel for Ubuntu 20.04. So make sure that linux-generic is marked as manually installed (so it won't be autoremoved), then it's safe to remove linux-generic-hwe-18.04.

### Post by TheFu on 2020-10-12
You can also clean up the ^rc packages, if you like.
```
sudo apt-get --purge autoremove linux-image-4.*
```
If that works, you can do the remaining ones.  apt-get supports package globbing, unlike the other popular APT tools.
