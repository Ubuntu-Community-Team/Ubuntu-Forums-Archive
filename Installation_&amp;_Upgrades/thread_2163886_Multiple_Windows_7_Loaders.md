### Post by oldfred on 2013-07-20
Many users do not know that the 100MB hidden recovery partition is really the boot partition. So Boot-Repair backs up the bootmgr to your main install. Then grub2's os-prober finds both bootmgrs and creates boot entries for both.
The one in sda1 has recovery but the boot from grub to sda1 is so quick that few can get f8 to work, so just best to have a Windows repairCD or flash drive.

You can turn os-prober off and manually add just one of those entries to 40_custom

Copy them to and edit to have only entries you want:
gksudo gedit /etc/grub.d/40_custom

sudo update-grub

gksudo gedit /etc/default/grub

or turn off executeable bit
sudo chmod a-x /etc/grub.d/30_os-prober
sudo update-grub
