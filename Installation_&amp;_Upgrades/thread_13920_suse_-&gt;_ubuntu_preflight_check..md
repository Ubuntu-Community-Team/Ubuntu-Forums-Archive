### Post by kassetra on 2005-02-03
I have a hybrid SuSE 9.0-9.1-9.2-ULB-Ximian system (yeah, I know #-o ) which I am more than ready to absolutely dump and begin my journey into Ubuntu-happy-land.

As much as I hate my current system (my cpu is STILL at 100%?  I only have a terminal window open!), I know better than to just dump what I have setup now and go head first into a new system (what do you mean what were my previous XF86Config settings?  I wiped my partitions clean!)  Also, since this is my main production / work machine I don't want to do something like this without a preflight check... and the more eyeballs that see it, the better.

That being said, here is what I have done to make sure my new system will convert over smoothly:

1. Copy down partition table information (wiping out home would be bad)
2. Make a copy of my X.org config and my XF86Config
3. Write down my hardware config (memory, cpu, hdd, cdrom, usbs)
4. Check for drivers for my nVidia card
5. Check for drivers for my ethernet cards (b44 & tulip)
6. Check my sound card info (on board, ac97)
7. Copy my vmware vmx & config files into my home partition
8. Copy any preferences that are system-wide into my home (menus, themes, fonts, etc.)
9. Write down all of my current applications
10. move all of my . files and directories into a regular directory in my home so that if I have anything funky, it won't hinder my new install
11. Test drive the Ubuntu live cd (everything worked flawlessly - including my sound!)

.... have I missed anything?
