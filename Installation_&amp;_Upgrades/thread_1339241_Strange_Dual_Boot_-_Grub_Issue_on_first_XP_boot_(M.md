### Post by darkod on 2009-11-27
You say you can use grub2 and ubuntu fine until you boot XP. If XP is doing the corruption of grub2 you should be looking elsewhere, not in grub.cfg.
I have no idea what is going on, and whether XP was somehow corrupted during the shrinking to create space for ubuntu. I always recommend to do the shrink as first step, then see if windows is working fine, and only then install ubuntu into the free space.
Not as one step with the ubuntu installer. Especially XP can be problematic if the partition was resized.

### Post by oldfred on 2009-11-27
Do you even have 2 drives, I do not think the link is related to your problem.

We have seem with HP and some Dell's where software in Windows that writes into the MBR for security purposes, in an a small area not used by the windows boot loader. The new Grub2 is larger than old grub and the windows boot loader and uses more of the MBR which is very tiny in the first place. Are you running any special security software in windows? It may be writing something into the MBR and corrupting the boot.
