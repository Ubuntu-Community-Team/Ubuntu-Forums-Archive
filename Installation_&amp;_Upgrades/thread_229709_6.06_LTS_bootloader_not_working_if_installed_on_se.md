### Post by RedBoot on 2006-08-05
The way the current version of grub works, you must boot from the IDE drive on a system with mixed drive types. In fact, even if you install to another drive, grub installs itself in the IDE MBR.

Set your BIOS to boot from the IDE and chances are good it's ready to go.

Here's a link where this is discussed in some detail.
[http://ubuntuforums.org/showthread.php?t=179908]("http://ubuntuforums.org/showthread.php?t=179908")

### Post by plouj on 2006-08-05
Well, I made the Ubuntu system finally boot by doing this in the LiveCD:

```

# grub
grub> root (hd3,0)
root (hd3,0)
 Filesystem type is ext2fs, partition type 0x83
grub> setup (hd3)
setup (hd3)
 Checking if "/boot/grub/stage1" exists... yes
 Checking if "/boot/grub/stage2" exists... yes
 Checking if "/boot/grub/e2fs_stage1_5" exists... yes
 Running "embed /boot/grub/e2fs_stage1_5 (hd3)"...  15 sectors are embedded.
succeeded
 Running "install /boot/grub/stage1 (hd3) (hd3)1+15 p (hd3,0)/boot/grub/stage2 /boot/grub/menu.lst"... succeeded
Done.
grub> quit 
```

Then I set my BIOS to boot to the disk with Ubuntu
When Ubuntu grub tried to boot the kernel it didn't find it because the changed disk boot order threw grub off.
I had to change the boot command to use:
```
root(hd0,0)
```
since this disk is the first that BIOS boots to
Now everything works fine!
