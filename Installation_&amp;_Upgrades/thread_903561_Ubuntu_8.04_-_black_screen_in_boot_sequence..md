### Post by cooke77 on 2008-08-29
APCI=off....means just that.
APCI is related....to your CPU's capabilities.
I have the same problem...with mine, when installing Linux (Ubuntu 8.04/Kubuntu 8.04).
I have a Dual-Core AMD Athlon 64 X2 CPU. (And,in BIOS, it's "APCI" control setting is Enabled...but, 'greyed-out'.)

I install mine, via the "Install within Windows" option, onto my 2nd hard-drive (Bought for this very purpose)...it is a 250 Gig H/drive.

You have to keep an eye out............for the LINE "Press 'ESC, to enter Menu."
When you enter Menu.......SELECT the one with (If you have APCI problems.)
This will be the one-n-only time you will have to use this 'option'.

The Installation will proceed. (And, take Time, in doing so. So, be patient.)
I formatted it 'NTFS' (beforehand).
Reason?
It is much easier to Uninstall Linux.
(All you have to do, is to go into Window/XP/Vista....and, use the "Add and Remove"....to uninstall Linux. Then, checking out the hard-drive (where you put Linux, to start with).....simply DEL the "Ubuntu Back-up" Folder you find there.
I have done this plenty of times, to know this does work well.
And, yes, there is no trace of Linux left on the hard-drive, done this way.
Windows just kicks straight in...no 'dual-boot' Grub in sight,either.
