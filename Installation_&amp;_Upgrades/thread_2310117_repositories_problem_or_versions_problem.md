### Post by vasa1 on 2016-01-16
Could you provide the output of ```
echo -e "Session: $DESKTOP_SESSION
Desktop: $XDG_CURRENT_DESKTOP 
Release Version: $(lsb_release -a)" 

```using code tags?

That'll tell people if your system is supported or is EOL.

Most of the lines mention "quantal" and no updates for quantal (12.10) will be available. You need to use a supported version to get updates: [https://wiki.ubuntu.com/Releases#End_of_Life_.28EOL.29](https://wiki.ubuntu.com/Releases#End_of_Life_.28EOL.29)

### Post by deadflowr on 2016-01-17
Solution #1: Grab a fresh image of a supported release (12.04, 14.04, 15.10, 1604-beta-ish)
Backup your data and do a clean install.

Solution #2: [https://help.ubuntu.com/community/EOLUpgrades](https://help.ubuntu.com/community/EOLUpgrades)

Solution #1 is the highly preferred method.

As far as EOL-upgrades go, I think I remember they were going to make it so users of 12.10 would be able to upgrade directly to 14.04, but I do not know if that ever made it into fruition.
(They have made strives since, as users of 14.04 who want to move on to the next release currently will now skip 14.10 and see 15.04 as the next release)

Otherwise you would have to jump from dead-release to dead-release to dead-release.
That being successful is probably the most apt comparison to winning the Powerball jackpot, imo.
