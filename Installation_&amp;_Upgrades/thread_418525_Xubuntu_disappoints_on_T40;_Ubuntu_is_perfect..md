### Post by cogitordi on 2007-04-22
I have two Linux computers to upgrade to Feisty Fawn: a T40 IBM notebook and a small form-factor box. I thought I'd try Xubuntu. I've been an Ubuntu user since the Hedgehog and there are some minor aspects of Gnome that I find a little irksome. (Only a little.) XFCE is attractive too.

Xubuntu installed on the SFF box without a problem. But I replaced it with Ubuntu because I didn't see any out-of-the-box way to browse my network shares in Ubuntu. I like how Gnome handles this.

Xubuntu was not a success on the T40. First, Xubuntu had a silly problem that prevented me from formatting the disk. The disk held my previous Ubuntu Drake installation. The installation would report a write error to the EXT3 filesystem. After some frustration trying to use GPartEd to wipe the data, I succeeded by quickly unmounting the disk with a mouse-click just as the Xubuntu formatting routine began. This would have been a show-stopper for many users. I find it hard to believe that I alone would have encountered this on a T40. 

Once installed on the T40, Xubuntu looked great. However, I had no wireless capability. I've suffered with Ubuntu before in trying to make wireless work and I had replaced the original Cisco card in my notebook with an Intel 2100 to cure the pain. I struggled for a while to get Xubuntu to do something useful. Commands dmesg and lshw proved that the card was recognized but I had no wireless manager. The situation didn't improve after I wired my notebook to the network. I don't know what the Xubuntu folks expected me to do to solve "my" problem, but I'm spoiled by Ubuntu and won't waste time compensating for a poor installation mechanism.

Ubuntu installed on both of these computers flawlessly. Wireless works out-of-the-box, as many others have said. The Fawn is an important step forward for Linux and Ubuntu. We're fortunate to have access to this Linux distribution for free. Thanks, Mr Shuttleworth!
