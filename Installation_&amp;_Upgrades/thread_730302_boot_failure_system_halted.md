### Post by danman on 2008-03-21
I figured it out and now I'm very surprised that this was not posted in the FAQ's some where. Ubuntu does not change the bios settings after loading like XP does, so if you take the disk out like instructed after loading the new OS of course it will have a boot failure. (if it is supposed to it does not)

The bios still is thinking to boot from the cd drive. Since on some mother boards you can only enable or disable a specific drive instead of changing the boot sequence buntu theres the problem. :twisted: I should have known by the  error instantly that that was the problem.
