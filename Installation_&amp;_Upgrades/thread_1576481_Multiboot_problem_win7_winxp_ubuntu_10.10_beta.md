```
### Post by wilee-nilee on 2010-09-17
Grub 2 is installed in the MBR of /dev/sda and looks on the same drive in
partition #6 for (,msdos6)/boot/grub.

As of now grub is looking for partition 6 in sda there isn't one. Grub generally is best when placed in the MBR of the HD that Ubuntu or Linux is in.

From a live Lucid, Karmic, or Maverick cd. Here is the commands link.
[https://help.ubuntu.com/community/Grub2#Reinstalling%20from%20LiveCD](https://help.ubuntu.com/community/Grub2#Reinstalling%20from%20LiveCD)
```

sudo mount /dev/sdb6 /mnt
```
then
```
sudo grub-install --root-directory=/mnt/ /dev/sdb
```

So this installs grub to sdb so put that HD first to be read in bios. If you want to reload the MS bootloader to sda so that if you wanted you could just boot that HD with a key prompt or having it first in bios, let us know

I am assuming this will all work I see GRLDR reference. This is a standard way of loading grub and mounting the right stuff. If you can take a look at the link it defaults to the commands.

Edit; Just run  
```
sudo update-grub
``` when you boot back into Ubuntu after the commands.

m_zeid, welcome to the Ubuntu forums!

Ubuntu is running, correct? And XP as well, except it's misidentified in the menu?

I think that the boot info script is a bit misleading in identifying sda and sdb. Everything *except* the bootinfo script header identifies Ubuntu on sdb. If Ubuntu is booting, then G2 is installed on the same drive as the Ubuntu install. In the results, the script shows Ubuntu mounted on sdb6. 

I don't know what might cause the script to reverse the header data of the RESULTS.txt. Personally, I'd leave things alone for now as Ubuntu is working normally.

If you want to explore, and possibly break things, you might look at /boot/grub/devicemap (if it exists) to see if hd0 and hd1 are identified correctly. If there isn't a devicemap, you can create one with 
```
sudo grub-mkdevicemap
sudo update-grub

```
and then run the script again while running a normal install.

The boot issue appears to be a Windows issue. I'm not familiar enough with Windows to know exactly how XP and W7 interact, so I can't offer much about how to get them both to live peacefully together. As *wilee-nilee* observed, there is only one set of Windows boot files. I don't know if a second set is missing or if the two just aren't melded correctly.
```
