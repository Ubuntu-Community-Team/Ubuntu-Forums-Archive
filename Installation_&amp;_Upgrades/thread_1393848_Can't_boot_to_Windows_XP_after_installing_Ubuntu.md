### Post by drs305 on 2010-01-30
Windows likes to have the 'rights' to the MBR. Grub knows where Windows is and transfers control, but then Windows doesn't like what it sees.

Restore the MBR to Window's liking. You can use "mbr" or "lilo" to do this. I prefer lilo. You use it to write to the MBR but don't actually configure it. (You will see a message about needing to configure lilo - don't).
```

sudo apt-get install lilo
sudo lilo -M /dev/sda mbr

```
That should make Windows happy.

Reinstall Grub 2. It shouldn't mess with what Windows sees:
```

sudo grub-install /dev/sda

```

Once everything is working you can remove lilo if you wish:
```
sudo apt-get purge lilo
```

BURG support loading of ntldr directly, so after /boot/burg/burg.cfg is generated, modify the Windows XP entry as (you need to chmod first since burg.cfg is read-only):

}

Here are view   more things to try:


**1)  Run testdisk "Rebuild BS": **

Boot into Ubuntu.

  ```
sudo apt-get install testdisk
sudo testdisk /dev/sda
```

Choose "proceed" on the first screen, then
"intel"
"advanced",
Select the Windows partition (although it should be selected already) and choose
"boot"
"RebuildBS"
"Write"  

then press "q" a few times to quit  testdisk, reboot and see whether you can boot into Windows.
If it did  not give you an option to "write", it means that the original and rebuild boot sector are identical, and  testdisk RebuildBS won't be able to solve your problem.


[B]2) Erase your mounted devices registry:
[/B]

Boot from your XP CD and enter the repair console. Type
```
 regedit 
```
to open the Registry Editor.

Select "HKey_Local_Maschine" in the left panel.
Select  "Load Hive" in the "File" menu.
In the Pop Up Box, Select "My Computer". 
Navigate to   \Windows\System32\config folder in your XP  Partition.
Select the file "system" and click open. 
In the new box type "MyHive" and hit enter.
Click on the "+"  sign next to "HKey_Local_Maschine"". 
You should now have an entry  named "MyHive in the left panel.
Click on the "+" sign next to "MyHive".
You should now have an entry  named "Mounted_Devices in the left panel.
RightClick "Mounted_Devices" and choose "Delete"
Select "MyHive" and choose "Unload Hive" in the "File" menu. 
Click on "yes" in the PopUP menu.
Choose "exit" in the "File" menu.

XP will automatically recreate the mounted device registry during boot up. But  I don't really think this will help, since you don't seem to get far enough in the boot process for this to make a difference. Also this  is usually only necessary if you moved the start point of the Windows partition.

 
**3)Bios Hidden Area **

Some  hard drives  have an area which is completely hidden from the operating system by the Bios.  This area can sometimes lead to booting problems after installing Ubuntu. (But I don't see any indication that this is case here)


**4) Raid settings in the Bios**

Check whether you have a setting in the bios for "ACHI"/"RAID" and see whether changing those makes any difference. (make sure to write down the original setting before you chance anything in the bios)
