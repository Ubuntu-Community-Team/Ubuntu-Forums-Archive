After what seemed like an eternity (a couple of hours or so), the update manager declared:

  Job for rpcbind.service canceled
  invoke-rc.d initscript rpcbind, action "stop" failed

and then continued with the installation process. When this thing completes I will mark this as solved.

### Post by haihovu on 2016-04-30
Ended up just killed the update-manager session and rebooted (systemctl will hang for any thing). Luckily the box rebooted into a console and I was able to restart the upgrade process with "dpkg --configure -a", "apt-get install -f", and "apt-get upgrade", ... Had to run the update-manager a couple more times (there was a partial upgrade in there some where), and eventually the system thinks the upgrade was successful. So out of the four boxes that went through the upgrade, two went without hitch, and two had varying problems, but all eventually recovered.
