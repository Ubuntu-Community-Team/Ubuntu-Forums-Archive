### Post by dmeehl on 2009-06-03
I have an Averatec 3225 laptop. The cd drive does not work, but I can boot from USB. 

I had Yoper installed on it for years with no problems, but I finally decided it was time to update. First, I tried to install kubuntu 9.04. It failed to start the installer (presumably because of xorg problems). I tried kubuntu 8.04 with the same results. Both dropped me to the (initramfs) promt directly after boot. 

Next I tried the 8.04 alternate disc, which failed saying that it could not find the cdrom. I tried the method from here: [https://help.ubuntu.com/community/Installation/FromUSBStick](https://help.ubuntu.com/community/Installation/FromUSBStick)
I could mount the usb but the installer just kept saying that it could not copy files from the cdrom, even though the usb was mounted to /cdrom.

Finally, i tried the 9.04 alternate disc which automatically did a network install. Not what I initially set out to do, but who's counting. The installation reported success and prompted to reboot. Upon booting up for the 1st time, I get dropped at the command line login. Xorg failed to load. It correctly identifies my laptop, chipset, etc. Default resolution is correct, but i get the warning "viaPanelGetIndex: Mode not supported by panel". Then it goes through trying a whole bunch of resolutions and settings all failing eventually saying "Mode pool is empty" and "No valid modes found"

I've tried every configuration I can think of to force the resolution and mode, but nothing works. I also found this: [http://wiki.openchrome.org/tikiwiki/tiki-view_forum_thread.php?forumId=1&comments_parentId=6376&topics_offset=2](http://wiki.openchrome.org/tikiwiki/tiki-view_forum_thread.php?forumId=1&comments_parentId=6376&topics_offset=2)
Which seems to indicate that openchrome doesnt like my chipset, but offers no work around. 
When I ran Yoper, XF86 (yes that old) used the via driver. That is not installed in kubuntu 9.04 and I dont know how to install it, but if I had to use that driver, I wouldnt complain. I'm not sure what if any benefits openchrome offers anyway. Anyone that can offer any help/suggestions is much appreciated.

### Post by bytefish on 2009-09-28
Hi,

1)
You can see if compiling the openchrome driver by following this guide: [https://help.ubuntu.com/community/OpenChrome](https://help.ubuntu.com/community/OpenChrome) helps.

2)
If it does not work yet add to your xorg.conf:

```
Section "Device"
    Identifier "Generic Video Card"
    Driver "openchrome"
    Option    "VBEModes"    "true"
    [...]
EndSection

```and restart X.

3)
If none of above works try the unichrome driver:

```

> git-clone git://people.freedesktop.org/~libv/xf86-video-unichrome/
> cd xf86-video-unichrome
> ./autogen.sh --prefix=/usr
> make && sudo make install

```and set the device driver to "unichrome" in your xorg.conf.

Cheers, Philipp.
