### Post by SeijiSensei on 2023-04-12
More recent versions of sshd like the one shipped with 22.04 no longer support older encryption formats like rsa-dsa. You can tell your client to try these older formats, or you can upgrade the client to 22.04. There are helpful pages on the web about how to specify the right options on the command line or in /etc/ssh/ssh_config.

I had some servers with the older version of ssh. Once I could no longer log in, I built a copy of sshd from source on those machines rather than upgrade the entire system. 22.04 clients have no problem connecting to a server running the newest version of sshd.

### Post by SeijiSensei on 2023-04-12
Yes, certain encryption methods have been deprecated because they were found to be vulnerable. You have a couple of solutions:

1) Upgrade the server and any clients to 22.04 or upgrade ssh and sshd by compiling from [source]("https://github.com/openssh/openssh-portable").

2) Use the "HostKeyAlgorithms" option at the command prompt with -oHostKeyAlgorithms=+ssh-rsa
or by adding the HostKeyAlgorithms directive  to /etc/ssh/ssh_config on the client machine.

You'll probably need to generate another key after this using ssh-keygen with a more recent algorithm. I use ed25519.

```
ssh-keygen -t ed25519
```

Run the command "man ssh_config" and find the HostKeyAlgorithms section.

In a nut shell, The RSA SHA-1 hash algorithm is being quickly deprecated across operating systems and SSH clients because of various security vulnerabilities, with many of these technologies now outright denying the use of this algorithm.
