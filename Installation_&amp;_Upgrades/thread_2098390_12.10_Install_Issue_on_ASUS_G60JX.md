### Post by WierdKid on 2012-12-26
I'll try to be as thorough describing the issue and what I have already tried as possible.

I am attempting to get Ubuntu 12.10 onto my stock Asus G60JX. The first time I tried to install from the Windows utility I ran into the nouveou issue. I downloaded and burned a CD next knowing that I would have more options doing it that way, but ran into nouveou again. After some deep forum searching found that the issue could be resolved by booting with "nomodeset". With that in the bootline I had no issue with installation. 

After install complete it told me to reboot and it was back to the nouveou. More forum search said I needed to get into GRUB to make "nomodeset" part of the normal boot, but I cannot seem to get GRUB to come up. The other solution I have seen is to install the NVidia drivers so that "nomodeset" is not needed. Being unable to get GRUB to come up I put the CD back in and booted from the CD, installed "NVidia binary X.Org driver ('current' driver). Reboot without the CD, still cannot get GRUB to put in "nomodeset" but the nouveou issue is still present.

I'm not sure if there is something I'm not doing right, or if this issue is something different than what I've been finding on the forum, but it is quite frustrating being able to boot off the CD and see how awesome this OS is, but not being able to run it normally. I would greatly appreciate some advice on where to go from here.

PC Specs:
[http://www.cnet.com/laptops/asus-g60jx-rbbx05-16/4507-3121_7-34024579.html]("http://www.cnet.com/laptops/asus-g60jx-rbbx05-16/4507-3121_7-34024579.html")


Edit: Not sure if it is relevent or not, but this is with the 64bit version of 12.10.

### Post by WierdKid on 2012-12-26
I have solved the problem. Running off the CD I used the boot-repair utility to set "nomodeset" as a kernel option in GRUB. After this on reboot I was able to open GRUB, I let it load the OS and it came up just fine.

[https://help.ubuntu.com/community/Boot-Repair](https://help.ubuntu.com/community/Boot-Repair)

This one shows you how to install and run Boot-Repair

[https://help.ubuntu.com/community/BootOptions](https://help.ubuntu.com/community/BootOptions)

This shows you how to use Boot-Repair to set "nomodeset"
