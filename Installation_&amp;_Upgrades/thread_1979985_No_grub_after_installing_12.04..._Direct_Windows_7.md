### Post by darkod on 2012-05-14
Looks like grub2 didn't install correctly at all. From live mode you will have to enter your root with chroot and install it:
```
sudo mount /dev/sda5 /mnt
sudo mount --bind /proc /mnt/proc
sudo mount --bind /dev /mnt/dev
sudo mount --bind /sys /mnt/sys
sudo chroot /mnt
```

Purge and reinstall grub2, make new config files:
```
apt-get remove --purge grub-pc grub-common
apt-get install grub-pc
grub-mkconfig
update-grub
grub-install /dev/sda
```

Exit chroot and unmount everything:
```
exit
sudo umount /mnt/sys
sudo umount /mnt/dev
sudo umount /mnt/proc
sudo umount /mnt
```

Restart and see if you get a grub2 menu now.

### Post by oldfred on 2012-05-14
The Windows boot loader uses the boot flag, grub does not, so moving boot flag will not fix booting other than preventing Windows from working.

Darko's instructions should get your grub installed correctly.

With a very large drive and a large / (root) partition you may just have another issue. There was (is?) a bug in grub that very large root partitions, grub just seemed to get lost. About half of those I have suggested smaller / (root) partitions and separate /home have solved grub boot issues. I normally suggest / (root) of 25GB and the rest for /home or data partitions. With Windows I also do not suggest writing much into the Window partition, but create another shared NTFS partition to use read/write for both systems. Windows sometimes has issues with too much data written directly into its system or c: partition.
