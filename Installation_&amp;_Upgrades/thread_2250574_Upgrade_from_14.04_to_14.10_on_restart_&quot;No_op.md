### Post by oldfred on 2014-11-02
Did you have any ppa enabled when updating? That almost always causes issues on update not completing.

But specs for your system say it was Windows 7. But you now show booting in UEFI boot mode.
Were you able to directly boot ubuntu entry in UEFI menu with out work arounds. 
Usually the few Windows 7 systems with UEFI did not have the issues that secure boot and vendor internal modifications to UEFI to only boot Windows 8 have.

Did you also update UEFI/BIOS?
Many vendors have modified UEFI to only boot the entry that says "Windows" in the entry. Your updates may have added that 'feature'. 

       Video on getting into UEFI - 1 Min Acer but all Windows 8
[http://www.youtube.com/watch?v=QGiG1oljjZI](http://www.youtube.com/watch?v=QGiG1oljjZI)

 Aspire E1-522 InsydeH20 Bios unlock -  7 min video
[https://www.youtube.com/watch?v=7SkBFkzOW0A](https://www.youtube.com/watch?v=7SkBFkzOW0A)

It does look like your UEFI boot order is not finding the ubuntu entry and then trying to boot from network or PXE boot. Is ubuntu in UEFI boot menu?

Post this:

 # from liveDVD or flash booted in UEFI mode and use efibootmgr
modprobe efivars
sudo efibootmgr -v

If it is now the description issue you may be able to create /efi/Boot folder and add grub or shim into that and rename to bootx64.efi. Or create a /efi/Windows/Boot folder and copy grub or shim into that and rename to bootmgfw.efi.

Summary of the various work arounds, not sure which may be easiest for best for your case.
      
 [http://ubuntuforums.org/showthread.php?t=2234019](http://ubuntuforums.org/showthread.php?t=2234019)
