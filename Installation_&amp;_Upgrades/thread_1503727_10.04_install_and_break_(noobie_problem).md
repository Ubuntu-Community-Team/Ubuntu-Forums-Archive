### Post by mikewhatever on 2010-06-07
You need to download the boot_info_script.sh file to your desktop (Live cd) before running it. Follow instructions from this thread:
[http://ubuntuforums.org/showthread.php?t=1291280](http://ubuntuforums.org/showthread.php?t=1291280)

That said, it's pretty clear from what you posted, that the partitioning part didn't go as planned, for there is no hdd with the partitioning layout you described. Somehow, the linux partitions got scattered among several hdds.

### Post by darkod on 2010-06-08
A rushed decision. :(

It was too late in my time zone and I had to go to sleep. Too bad no one else jumped in with little more explanation.

As you probably figured out by now, Linux will NEVER install onto existing partition unless you specifically tell it to! Otherwise, maybe it will overwrite something important in that partition.

Since /dev/sda5 was existing and you wanted to install there, you needed to use the Manual Partitioning option. It will show you a list of your partitions. Then you click on /dev/sda5, and the button Change at the bottom, specify the filesystem as ext4, tick the format box, and select mount point /.
It should automatically pick up /dev/sda7 as swap partition but usually I double check. Again by clicking on /dev/sda7 and the Change button. It should be specified as swap area (swap doesn't have format and mount point options enabled).

You had a very nice situation, but no advice. :( Now the partitions were resized by the guided method, and only way out is deleting all ubuntu partitions (except /dev/sda6) and installing again in that unalocated space, either with the guided method or with manual partitioning.
