### Post by lewisdeane on 2010-03-05
Try downloading Super Grub Disk via Win7 (if you can't get into win7 do a fixmbr with your disk - don't worry, you can fix it later), both SGD (legacy) and SGD2 (grub2), burn them both to separate disks and try booting from them. The SGD2 should immediately recognise your ubuntu install but, if not, you can try some command line stuff, and, if that fails, SGD legacy has a wealth of tools and help to boot any and all OS's. I actually have both SGDs installed on a small partition for just such an emergency. Good luck. Btw, this upgrade has had some tricky bugs.

### Post by lewisdeane on 2010-03-05
Btw, if you do get in to your ubuntu it might be that the new kernel image is the problem - there's a way of un-installing and re-installing the upgrade and/or reverting to the old kernel.And if you know the details of the old kernel ( something .19) try calling that up via command-line, cause it's still there. Something like:

title		Ubuntu 9.10, kernel 2.6.31-19-generic
uuid		291d56ae-06fb-442d-a302-df148da9ea88
kernel		/boot/vmlinuz-2.6.31-19-generic root=UUID=291d56ae-06fb-442d-a302-df148da9ea88 ro quiet splash 
initrd		/boot/initrd.img-2.6.31-19-generic
quiet

Though change the uuid to yours or to root (hd...). Above all, don't panic, and completely re-install until you've exhausted all possibilities!

### Post by RHTopics on 2010-03-08
granric,

If you are still interested, I believe I see why Grub was having a problem and giving you a frozen blank screen.

This line in your sda5/boot/grub/grub.cfg file:

```
  # set gfxmode=1280x1024

```
As you can see it is commented out so grub did not know what to resolution to set for the screen.  In a original, vanilla setup this line in grub.cfg would be:

```
    set gfxmode=640x480

```
Did you change your resolution in /etc/default/grub by adjusting this line to 1280x1024?

```
    #GRUB_GFXMODE=640x480

```
If you did and that worked before upgrading the kernel, I am wondering if the kernel upgrade had something to do with commenting out the line in your /boot/grub/grub.cfg file.
