### Post by raccoonstrait on 2017-04-17
Working with System76 support, they noticed this bug and solution:

[COLOR=#574F4A][FONT=&amp]https://bugs.launchpad.net/ubuntu/+source/network-manager/+bug/1638842
[https://bugs.launchpad.net/ubuntu/+source/network-manager/+bug/1658921](https://bugs.launchpad.net/ubuntu/+source/network-manager/+bug/1658921)

The current fix is to run this command:

```

sudo touch /etc/NetworkManager/conf.d/10-globally-managed-devices.conf
[/FONT][/COLOR]
```

That worked. See message 6 and 7 in the thread for [COLOR=#574F4A][FONT=&amp]1658921

Raccoon[/FONT][/COLOR]

### Post by laidbackwebsage on 2017-05-26
The solution above did not work for me.

However, the following *did* work for me:

```
sudo vi /etc/NetworkManager/NetworkManager.conf
```

Here is what my default file looked like:
```

[main]
plugins=ifupdown,keyfile,ofono
dns=dnsmasq

[ifupdown]
managed=false

```

Under ```
[ifupdown]
```,  I changed ```
managed=false
``` to ```
managed=true
```

Then, ```
sudo service network-manager restart
```

Enjoy!
