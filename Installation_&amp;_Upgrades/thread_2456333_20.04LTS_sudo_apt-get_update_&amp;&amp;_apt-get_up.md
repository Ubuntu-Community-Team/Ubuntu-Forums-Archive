### Post by TheFu on 2021-01-10
In general, it is not a good idea to get drivers from the vendor, especially GPU drivers.  Use **ubuntu-drivers** command instead.

Probably want to use 
```
sudo apt-get update && apt-get full-upgrade
``` 
to get new kernel line updates too.

As for whether the system is locked up or it is just the GUI, have you checked different ttys?  <cntl><alt>-F2 thru F8. Each is a different tty.  My need to hit <enter> a few times to wake it up.  Then you can login and do stuff from a terminal shell.

You can prevent nvidia drivers from being loaded. Use the F/LOSS nouveu drivers instead. [https://nouveau.freedesktop.org/InstallNouveau.html](https://nouveau.freedesktop.org/InstallNouveau.html) It should already be installed, just disabled.

If the ttys don't work, I'd strongly suspect a hardware issue.  Could be all sorts of things from a failing HDD to bad PSU. Check the system log files.  Google "ubuntu logs" for those.  Look for errors and warnings. If they aren't saved between boots, you'll want to change some settings to keep perhaps 20MB of logs.  
journald.conf is where those settings are located.
```
[Journal]
Storage=persistent
SystemMaxFileSize=50M
SystemMaxFiles=10

```
is what mine says, but there are lots of others and a manpage for journald.conf with explanations for each option.
