### Post by naion on 2024-08-22
Hi All,

I, also, have a Dell XPS 13 9340, and having the same issues as you. EVen with the newest kernel: [B]linux-image-6.8.0-41-generic.
[/B]
The last known good kernel was the -36, so I looked at the changes between 36 and 38 (the first with the problems) and I noticed a supposed fix / workaround for this bug:
[https://bugs.launchpad.net/ubuntu/+source/linux-oem-6.5/+bug/2046315](https://bugs.launchpad.net/ubuntu/+source/linux-oem-6.5/+bug/2046315)

After following the link in that bug report ([https://gitlab.freedesktop.org/drm/i915/kernel/-/issues/9739](https://gitlab.freedesktop.org/drm/i915/kernel/-/issues/9739)) it seems when PSR2 is enabled this triggers the glitches...

There they proposed a couple for kernel command line flags and disabling PSR2 is the one that also worked for me: "i915.psr_safest_params=1"

I think, for some reason, the workaround hasn't included our graphics card to disable PSR2 and that from 38 onward PSR2 is enabled...

So, with this my glitches are gone and the laptop now also wakes fast from sleep...

Hi nardifra, I was just able to follow the following steps from chatgpt using the same command line naion above mentioned (thanks naion!) and it worked for me (using Dell XPS 16 9640 running Ubuntu 24.04.2 Kernel 6.8.0-45-generic). I used the code in **bold** below. hope this helps you too!
____

To disable PSR2 (Panel Self Refresh) on a Dell XPS 16 running Ubuntu 24.04, you typically need to modify the kernel boot parameters. Here’s how to do it:


### Step 1: Edit GRUB Configuration

1. **Open Terminal**: Press `Ctrl + Alt + T` to open the terminal.

2. **Open GRUB Configuration**: **sudo nano /etc/default/grub**

3. **Modify the GRUB_CMDLINE_LINUX Line**:
   - Look for the line that starts with `GRUB_CMDLINE_LINUX_DEFAULT`.
   - Add **i915.psr_safest_params=1** to the list of parameters within the quotes. For example: **GRUB_CMDLINE_LINUX_DEFAULT="quiet splash i915.psr_safest_params=1"**

4. **Save Changes**: 
   - Press `Ctrl + O` to save the file, then `Enter` to confirm.
   - Press `Ctrl + X` to exit the editor.

### Step 2: Update GRUB

Run the following command to update the GRUB configuration: **sudo update-grub**

### Step 3: Reboot Your System

Restart your computer for the changes to take effect by running: **sudo reboot**

### Step 4: Verify the Setting

After your system has restarted, you can check if PSR2 is disabled by running: **cat /sys/module/drm/i915/parameters/psr_safest_params**

This command should reflect the changes you've made.

### Troubleshooting

- If you encounter any issues, ensure you entered the parameters correctly.
- If the command doesn't work, it might be worth checking if the module is loaded with: lsmod | grep i915
