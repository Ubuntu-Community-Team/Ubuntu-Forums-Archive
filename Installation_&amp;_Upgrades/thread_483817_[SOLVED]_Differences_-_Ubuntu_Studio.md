### Post by tgalati4 on 2007-07-20
The low-latency kernel devotes 95% of the CPU cycles to your primary artistic application.  The system gets by on 5%.  This works when you are using one application to draw, record, edit video, etc.  It is not suitable for desktop use.

Try running too many things under low-latency and your load will shoot up and things will seem draggy.

That is why you dual boot:  normal mode for desktop use and low-latency when you have a deadline.
