### Post by d3v1150m471c on 2010-01-14
Equivs makes it so that it circumvents your dependencies. I'm sure some people use it for a reason, but to me it just sounds like an extremely bad idea. There are obvious reasons why the ubuntu system installs packages with the other packages it needs to make things run. I unfortunately cannot offer anymore advice other than saving your important files to a cd or a usb drive and doing a reinstallation. Hopefully, someone else has a better idea, but that's what i would do. In the future, always backup your system before you make significant changes to your operating system because things like this do happen.

If you're connected to the internet you could try crtl+alt+F1 and doing a "sudo aptitude remove equivs" and then do "sudo apt-get xubuntu-desktop", then trying a reboot and see if you get your xubuntu back. If not, watch the forum, research, and/or see above. Sorry, mate.
