### Post by leo_rockway on 2007-09-27
it is usually a good idea to leave a previous kernel (besides the one you're using) in case you find any problem after updating the kernel.
Same thing for the recovery modes.
Of course, they are not really "necessary". you could edit the list (but other than an aesthetic reason, i don't see any benefit)

### Post by leo_rockway on 2007-09-27
yes, this is definately the usual outcome.
I have listed the following kernels in my grub menu: 2.6.22-12, 2.6.22-11, 2.6.22-10, 2.6.22-9, 2.6.20-16 and 2.6.20-15. each of them has its own recovery mode.

i leave them there because they don't bother me. like i said, it is usually good to leave the recovery of the kernel you're using and a previous kernel (with its recovery) in case you get any problem w/ the kernel you're using.

if i were you i wouldn't delete the entries from the menu list, i'd just comment them (#).
