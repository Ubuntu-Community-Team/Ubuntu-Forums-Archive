### Post by Hedgehog1 on 2011-04-17
sneaker12345,

Well - first the good news: because the install was confused by the HDDRECOVERY partition on your computer, it didn't actually change anything on your disk drive, so it did not damage.

Now - the bad(ish) news: because the install was confused by the HDDRECOVERY partition on your computer, it didn't actually change anything on your disk drive, so it did not install Ubuntu, either.

For your PC, you will need to manually partition your hard drive for your Dual-Boot Ubuntu Install.

This is done in three steps:

(1) Using the Windows Disk Manager, you will shrink your /dev/sda2 partition (seen as your 'C:' drive in windows) to make room for Ubuntu.  Please make at least 30 gigs of room (you can make more space if you want).

(2) Boot from your Ubuntu LiveCD/LiveUSB, use gparted to change your disk drive from this:
```
/dev/sda1 NTFS SYSTEM
/dev/sda2 NTFS Windows 7
/dev/sda3 NTFS HDDRECOVERY
```
To this:
```
/dev/sda1 NTFS SYSTEM
/dev/sda2 NTFS Windows 7
/dev/sda4 Extended Partition
__/dev/sda5 ext4 '/' (root) Ubuntu
__/dev/sda6 Swap (make the the size of your RAM plus 10%)
/dev/sda3 NTFS HDDRECOVERY
```

You will then install 'manually' to control the install.
