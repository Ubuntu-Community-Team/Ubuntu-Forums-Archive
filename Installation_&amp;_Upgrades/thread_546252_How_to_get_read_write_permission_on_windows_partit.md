
```

sudo ntfs-config

```

Tick the appropriate boxes and restart.

### Post by merlinus on 2007-09-08
By c drive you mean sda1?  Also, try removing the boot flag from sdb1 (if that is not your windows install) using gparted.

Restart, and if you still do not have write privilege, open nautilus as root, navigate to the mountpoint, and change the permissions.

```

gksudo nautilus

```

-merlin

### Post by cherry316316 on 2007-11-28
Thanks, there was problem with fstab.
I changed the line in fstab from
```

/dev/sda6    /media/e     vfat     defaults,utf8,umask=007,gid=46     0     0

```
to
```

/dev/sda6    /media/e     vfat     auto,rw,users,uid=1000,gid=1000,utf8,umask=000,exec,sync    0    0

```

now, its working fine :)
