Using VirtualBox 3.2.4 r62467 on WinXPSP2 host with Ubuntu 10.04 Server  as guest, I found that the **problem occurs only when the server is  installed in "Minimal Virtual" mode**. Here's my experience with it:

[LIST]
[*]Host: Windows XP Pro SP2 (32bit)
[*]VirtualBox: Version 3.2.4 r6246
[*]Guest: Ubuntu 10.04 Server (i386)
[/LIST]
 **_Results_:**

[LIST=1]
[*]First, I installed the server on VBox with all default options** _[COLOR=DarkRed][Controller-[COLOR=Black]SATA[/COLOR], PAE-[COLOR=Black]Disabled[/COLOR], Installation-[COLOR=Navy][COLOR=Black]Normal  mode[/COLOR][/COLOR]][/COLOR]_**. Installed smoothly, boots smoothly. [COLOR=DarkGreen]**No problems!**[/COLOR]
[*]Then on the same machine, I reinstalled server in "Minimal  Virtual" mode _[COLOR=DarkRed]**[Controller-[COLOR=Black]SATA[/COLOR], PAE-[COLOR=Black]Disabled[/COLOR], Installation-"[COLOR=Navy][COLOR=Blue]Minimal  Virtual[/COLOR][/COLOR]" mode]**[/COLOR]_. Installed smoothly, but failed to start on reboot. [COLOR=Red]**Startup aborted!**[/COLOR]
[*]As suggested above, I changed the HDD controller from SATA to IDE.  No improvement. **[COLOR=Red]Startup aborted[/COLOR]** again.
[*]I also tried changing to SCSI/SAS controllers. Nothing helped.
[*]Then I **enabled PAE**  (settings>system>processor>Enable PAE/NX) and changed back to  IDE controller** _[COLOR=DarkRed][Controller-[COLOR=Blue]IDE[/COLOR], PAE-[COLOR=Blue]Enabled[/COLOR], Installation-"[COLOR=Blue]Minimal[/COLOR]  [COLOR=Blue]Virtual[/COLOR]" mode][/COLOR]_**. Booted successfully.** [COLOR=DarkGreen]No  problems![/COLOR]**
[*]With PAE enabled, changing to SCSI/SAS controllers also worked  successfully.
[*]I changed back to SATA controller (PAE enabled this time) _[COLOR=DarkRed]**[Controller-[COLOR=Black]SATA[/COLOR],  PAE-[COLOR=Blue]Enabled[/COLOR], Installation-"[COLOR=Blue]Minimal Virtual[/COLOR]" mode]**[/COLOR]_. [COLOR=Red]**Machine started, but boot failed**[/COLOR] (just as  stated in OP's first post)!
[/LIST]
  **_Conclusions_:**

[LIST]
[*]With **everything at default** (machine settings, installation  options), there is **no problem.**
[*]If installed in **"Minimal Virtual" mode**, then** PAE must be  enabled** and **any of the IDE/SCSI/SAS controllers** can be used  for hard-disk.
[*]**SATA** controller **WILL NOT WORK on "Minimum Virtual"  installation**.
[/LIST]
 
> **bind00 said:**
> How do you change from SATA to IDE? I don't seem  to have the option. If I click add controller IDE is greyed out (on XP).  Is that because my real harddrives are all SATA? The CD/DVD is IDE but  that doesn't help much. 
I've updated to 3.2.4 with no difference.
You can not add another IDE controller since you already have one (the one hosting CD/DVD drive)
To change the HDD from SATA to IDE, simply remove it from SATA tree & add it to the IDE tree. (you may also remove the SATA controller altogether since minimal install of server is unable to use it, making it useless)

I'd like to add one more thing (It's only my assumption though):
compatibility or availability of virtual adapters (graphic, sound, Ethernet, drive controllers, etc.) in a virtual machine DOES NOT DEPEND upon their actual type available on the actual machine as long as the host OS can detect & use them (the actual ones).
All of them, in virtual machine, are emulated ones & thus only software dependent.

It is the responsibility of the virtualization platform (VirtualBox in our case here) to act as a translator/communicator between drivers of virtual hardware & those of actual hardware to get the job done.
