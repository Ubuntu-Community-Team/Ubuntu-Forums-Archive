### Post by Don Barzini on 2010-06-22
> **IAmNotAUser said:**
> What should I do to make sure that reboot works successfully? I have no other ideas as update-grub has set the drive correctly and yet it still does not work.

Can you boot into the **Recovery Mode**?

If so, boot into the Recovery Mode of the latest kernel and go to a root session.

Find your bootloader location by running...

```
grub-probe -t device /boot/grub
```

Once there re-install GRUB2 to your MBR by issuing the following command....

```
grub-install /dev/sda
```

(Note:.... The above **/dev/sda** assumes this is the first drive on a SATA disk machine and the result of **grub-probe** returned a device starting with **/dev/sda?**)

Then proceed to run **update-grub** again

```
update-grub
```

See if that works out.

### Post by oldfred on 2010-06-22
If you moved/deleted swap you may have changed its UUID, but your fstab will still have the old UUID. 

If you want to see the UUIDs

sudo blkid

If you want to see the entire boot configuration:

Boot Info Script courtesy of forum member meierfra
Page with instructions and download:
[http://bootinfoscript.sourceforge.net/](http://bootinfoscript.sourceforge.net/)
Paste results.txt, then highlight entire file and click on # in edit panel(code tags) to make it easier to read.
