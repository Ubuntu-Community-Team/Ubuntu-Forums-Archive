### Post by oldfred on 2016-07-11
Nothing obviously wrong.
But it says it cannot find Windows in sda2, but sda2 is shown a NTFS partition.

Most often related to Windows issues and you probably need chkdsk. Which you can only run from Windows.
If you have Windows repair flash drive then run chkdsk.

Grub only boots working Windows and if it needs chkdsk you cannot boot it. And Boot-Repair can only do minor repairs.
Boot-Repair may try to set chkdsk flag so on boot Windows runs chkdsk if you can get that far. 

Otherwise you may need to temporarily reinstall a Windows boot loader and boot Windows using f8 to get into internal repair console that really is in sda1, which we hope is not also corrupted.

Best to use Windows to reinstall Windows boot loader to MBR, but grub can install a Windows type boot loader using advanced mode.
Fix Windows and then use Boot-Repair to reinstall grub to MBR.

### Post by oldfred on 2016-07-12
Windows normally installs to a 100MB boot partition and another large NTFS main or c: drive partition.
The boot partition has boot flag and two of the three normal boot files. But it often gives you the change to boot into a repair console with f8 when booting, so you can fix main partition.
