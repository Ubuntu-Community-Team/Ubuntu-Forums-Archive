Here is a more complete version of the write-up.

I'd make a backup copy of your xorg.conf file and try:

sudo dpkg-reconfigure xserver-xorg

to recreate xorg.conf file.


Here is a more complete version of the procedure.

To get to a console with a system with a problem with the X11 server, you can do either of the following:

a) If you do not see the grub stuff at boot time, you will need to press esc within 3 second of poweron so to see the grub menu. Now, cursor down to Ubuntu, kernel  2.6 ... (recovery mode) and press enter.

b) Boot up your system. When nothing seems to be happening, press:
control-alt-f1.
You may have to press control-alt-f1 more than once. Note: To get to the console at any time, you can press control-alt-f1 even with X11 running fine, and to get back press alt-f7.


Both of these will bring up a console. Logon. You may have to press control-alt-f1 more than once.
```

cd /etc/X11

sudo cp -i xorg.conf xorg.conf.old

# enter your logon password

sudo nano xorg.conf

# to write out
control-o
# & answer questions

# to end
control-x

# to recreate xorg.conf file.
sudo dpkg-reconfigure xserver-xorg



```

I do not know if reconfigure restarts X11 or not.  To restart X11 do:

sudo init 1

a while later do:
sudo init 2
