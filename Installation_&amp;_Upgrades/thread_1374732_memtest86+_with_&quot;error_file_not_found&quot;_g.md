### Post by John Little on 2010-01-07
[FONT="Times New Roman"]I tracked this down, eventually:(, but I would have appreciated if the forums had given me guidance, so I'm adding the post I would have liked to have found.

The Karmic [/FONT]/boot/grub/grub.cfg [FONT="Times New Roman"]entries for memtest86+ generated by[/FONT] /etc/grub.d/20_memtest86+ [FONT="Times New Roman"]have a problem if /boot is on a different file system than /usr.  This is reported as[/FONT] [https://bugs.launchpad.net/ubuntu/+source/memtest86+/+bug/450351](https://bugs.launchpad.net/ubuntu/+source/memtest86+/+bug/450351) [FONT="Times New Roman"]and is reported as fixed in Lucid, but seems is not being fixed in Karmic.

If you just want to run it once, you can use the grub e command on an Ubuntu menu entry and note the "set root=" entry.  Escape out of that and use e on the memtest86+ entry to add the same set root command on a line before the one that's there, so that it looks like, for example:[/FONT]

[INDENT]set root=(hd0,2)
linux16	/boot/memtest86+.bin
[/INDENT]

[FONT="Times New Roman"]and run it with ctrl-x.[/FONT]

[FONT="Times New Roman"]You can fix it in[/FONT] /boot/grub/grub.cfg [FONT="Times New Roman"] but that will be overwritten the next time it is generated, or you can fix[/FONT] /etc/grub.d/20_memtest86+ [FONT="Times New Roman"]to make it call [/FONT]prepare_grub_to_access_device[FONT="Times New Roman"] in the way the other scripts do.[/FONT]
