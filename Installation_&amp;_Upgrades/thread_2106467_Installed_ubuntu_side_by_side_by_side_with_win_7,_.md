### Post by darkod on 2013-01-19
The disk seems to be with gpt table. If that is true, windows was installed in UEFI mode since it can only work with gpt in uefi.

On the other hand, you installed ubuntu in legacy mode, not uefi mode like you should have. So, there is no dual boot.

But not only that there is no dual boot, what gets me worried is that there doesn't seem to be a win7 partition at all any more. So it looks like ubuntu overwrote win7, which is not how it usually installs when you use the "side by side" option.

If you have important data in win7 that you didn't back up, I suggest you stop booting ubuntu from the hdd and use the cd in live mode. You can try restoring the previous partitions with testdisk.
[www.cgsecurity.org/wiki/TestDisk_Step_By_Step](www.cgsecurity.org/wiki/TestDisk_Step_By_Step)
