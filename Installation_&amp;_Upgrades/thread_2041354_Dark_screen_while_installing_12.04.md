### Post by Jay MC on 2012-08-12
Hi Vamsi,

I had this problem too, when I installed 11.10 via Wubi.  My solution was just to leave it and let it do its thing, plugged into ethernet (I could just about see, when I angled the screen right, that "stuff was happening", even though the backlight was off).

Eventually, I returned to the laptop to find that it had finished the install and rebooted itself to the Unity logon prompt, with the monitor working fine.

Because I had it plugged into a wired connection, I don't know whether it grabbed any extra drivers from somewhere during the install.

Also, I can't remember how much random hitting of keys I did while the screen was dark (e.g., I may have flukily hit "return" when a dialogue box was open, allowing the install to proceed).  But I've installed Ubuntu twice this way (i.e., "blind"), using the Wubi installer, and once installation is complete, I never have any problems with the monitor backlight.
