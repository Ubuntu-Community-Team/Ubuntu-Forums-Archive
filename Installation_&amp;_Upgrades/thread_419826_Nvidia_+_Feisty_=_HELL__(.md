### Post by zeldeamipro on 2007-04-25
The problem is that if you put the propietary drivers ubuntu continue trying to load the "nv" kernel module. The solution is simple:

sudo vi /etc/default/linux-restricted-modules-common

Add nv to DISABLED_MODULES:

DISABLED_MODULES="nv"

[Other nvidia in ubuntu feisty issues.]("http://mundobip.com/viewtopic.php?f=11&t=373")

### Post by fred5 on 2007-07-15
Here is my solution.

I think I had the same problem with Nvidia 8600GT and Feisty Fawn.

I installed the absolute latest video card drivers directly from the nvidia website instead of the Ubuntu package manager.  It fixed the problem.

Downloaded absolute latest Nvidia drivers from nvidia.com

Installed required libc package:
sudo apt-get build-essential

Exited into terminal mode using CTRL-ALT-F3

Quit the X-Windows system:
sudo /etc/init.d/gdm stop

Ran the Nvidia installation command:
sudo sh NVIDIA-Linux-x86-100.14.11-pkg1.run

Rebooted.

Everything is working great.  After rebooting I used the Nvidia GUI settings manager to set the resolution etc.

### Post by buckrodgers on 2007-08-10
Ah, that is ok, your old drivers are still aprtialy there 

see if you can remove your old drivers by doing that:
```

sudo apt-get --purge remove nvidia-glx nvidia-settings nvidia-kernel-common linux-restricted-modules*
sudo rm /etc/init.d/nvidia-*
sudo rm -f /lib/linux-restricted-modules/.nvidia_new_installed

```

it is well explained on this site and on the nvidia forum:

[http://www.robdian.co.uk/content/view/56/](http://www.robdian.co.uk/content/view/56/)
