### Post by grahammechanical on 2021-10-17
What exactly is going wrong. I have had experiences on Ubuntu where Startup Disk Creator would not recognise a Debian ISO as an image to be burned to USB memory stick. So, are these ISO burning applications not recognising the Ubuntu ISO image? Or, is the ISO image being burned but the live session does not load?

As a separate point I think it is a bit early to install Ubuntu 21.10. Wait a few days. There still may be issues to be corrected. Then download the ISO image again. It may have been recreated with bug fixes. 

We do know that the Ubuntu developers want to use a new installer program. It is not finished as far as I know. The old installer is still present in the ISO image but the user may be offered a choice of which installer to use. If the issue is down to the installer failing then choose the Ubiquity installer. That is the long time installer. And the Kubuntu developers may have decided to continue using the old installer. That may explain why you say Kubuntu works.
