### Post by ferrocene on 2007-10-21
In Feisty, my mouse speed was perfect.  more importantly, the Control Panel worked - select a different speed, and the mouse changes speed appropriately.

I upgraded to Gutsy and the mouse defaulted to being way too fast.  So I opened up the Control Panel and lowered the acceleration.  After I did that, the mouse is now super slow, and the Control Panel doesn't save any changes to acceleration.  It saves changes to sensitivity.  If I drag the setting to the right, close the CP and reopen it, it's back to the left again.

I'm running @ 1920x1200 and to drag the mouse from one side of the screen to the other I have to lift off of the mouse pad four times.

I've searched the forums and I don't see any issues related to the Control Panel not working.  I'm disappointed; I have been preaching Ubuntu for months and then the mouse breaks, a seemingly basic feature.  I'm frustrated. :(

Tech specs:  Ubuntu upgrade from Feisty to Gutsy, 64bit.  Replaced all xorg.conf files during upgrade.  I noticed that it changed my device from /dev/psaux to /dev/input/mice.  I changed it back to /dev/psaux but it didn't make a difference.

I'm running a generic 2 button USB mouse, no special drivers.

Also my keyboard wouldn't auto-repeat any characters, but I was able to check and uncheck the autorepeat boxes and that fixed the keyboard.

I installed envy and that wrote a new xorg.conf file, but it didn't fix the mouse problem.

Well I found the fix, from this thread:

[http://ubuntuforums.org/showthread.php?t=200496](http://ubuntuforums.org/showthread.php?t=200496)

under motion_acceleration, it was set to .2000000002328572

How the hell did it get set to that?  Whatever, case closed.  :)
