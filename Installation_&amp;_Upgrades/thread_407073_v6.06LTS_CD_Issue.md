### Post by cafeen on 2007-04-11
I'm trying to install ubuntu 6.06LTS on an IBM xSeries 205 and am able to boot from the CD to the initial menu. When I attempt to begin the install by selecting that option from the list it asks me to specify my locale, etc. However when the install is about to begin I get an error message indicating that the CD cannot be accessed. Yet it has already been accessing the CD. What's up?

I have seen this issue on this hardware with Fedora 7. I had to enter a command like "linux ide=nodma" or something like that before the install could access the CD. Do I need to do something similar?

---

### Post by cafeen on 2007-04-13
Well, sure enough I should have followed my own suggestion to begin with. I merely needed to append "linux ide=nodma" to the end of the boot parameters and it worked like a charm. 

THIS ISSUE IS RESOLVED!
