### Post by howefield on 2017-02-03
Although the upgrader usually disables PPAs prior to the upgrade taking place it doesn't revert the software back to the default Ubuntu versions, hence the dependency issues that you are seeing.

I'd suggest using ppa-purge to uninstall the PPAs and revert the software packages back to those from the Ubuntu repositories. It's a fair bit of work and I'd probably just do a fresh install.

### Post by ian-weisser on 2017-02-03
PPAs are not supported software, use them at your own risk...and this is one reason why. Some PPAs break upgrades.

Since you have numerous package conflicts to the point where apt no longer works, then you must uninstall all the non-Ubuntu software (including PPA software) using dpkg. Remember to clean out your local package cache to prevent PPA software from getting reinstalled.

Looking over your error messages, it would should take you around three or four hours to remove amarok, amarok-utils, qt-5, and the rest using dpkg. Do it from a TTY, not a terminal window - your GUI desktop is likely to break along the way.  Then reinstall kubuntu-desktop to restore your functioning system. Then it will be ready for an upgrade

Alternately, install fresh. It will be faster and easier for you. Completely uninstall all PPA software, and disable PPA sources for that software before upgrading next time.

Either way, preserve all your data first with a backup! Both paths risk data loss if you make a typo.
