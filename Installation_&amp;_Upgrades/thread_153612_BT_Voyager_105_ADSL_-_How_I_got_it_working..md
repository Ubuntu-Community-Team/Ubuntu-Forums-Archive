### Post by nerderello on 2006-04-01
How I got the BT Voyager 105 USB ADSL Modem working with Ubuntu 5.10.

I'm writing this as the method in the unofficial FAQs doesn't work (at all). This is how I got it working -

1) Downloaded file eciadsl-usermode_0.11-1_i386.deb from [http://eciadsl.flashtux.org?lang=en](http://eciadsl.flashtux.org?lang=en)

2) Downloaded the file eciadsl-synch_bin.tar.bz2 from [http://eciadsl.flashtux.org?lang=en](http://eciadsl.flashtux.org?lang=en)

3) Downloaded files pppoe_3.5-4_i386.deb and pppoeconf_1.7_all.deb from [http://www.debian.org/distrib/packages](http://www.debian.org/distrib/packages)   (although I'm not convinced that I really needed the second of those pppoe files)

4) Installed the pppoe files by :-
```

sudo dpkg -i pppoe_3.5-4_i386.deb

sudo dpkg -i pppoeconf_1.7_all.deb

```

5) Installed the eciadsl files by :-
```

sudo dpkg -i eciadsl-usermode_0.11-1_i386.deb

sudo tar -xjvf eciadsl-synch_bin.tar.bz2 -C /etc/eciadsl/ 

```

6) Configured (needed root privileges for the following config file steps, so I opened up a text terminal and entered **sudo gedit** followed by the root password) /etc/eciadsl/eciadsl.conf  like so :-
```

VID1=1690
PID1=0215
VID2=1690
PID2=0215
MODE=VCM_RFC2364
VCI=38
VPI=0
FIRMWARE=/etc/eciadsl/firmware00.bin
SYNCH=/etc/eciadsl/eciadsl-synch_bin/gs7470_synch03.bin
PPPD_USER=My.Username@btbroadband.com
PPPD_PASSWD=
USE_DHCP=no
USE_STATICIP=no
STATICIP=
GATEWAY=
MODEM=BT Voyager 105
MODEM_CHIPSET=GS7470
SYNCH_ALTIFACE=0
PPPOECI_ALTIFACE=0
PROVIDER=UK..British Telecom
DNS1=194.74.65.69
DNS2=194.72.9.38

```

Here the **PPPD_USER=My.Username@btbroadband.com** is the username that I set up in BT Broadband (the one shown is not my real one). The **SYNCH=/etc/eciadsl/eciadsl-synch_bin/gs7470_synch03.bin** (extracted from the eciadsl-synch_bin.tar.bz2 file) is needed as the default synch file (/etc/eciadsl/synch01.bin) doesn't work with the BT Voyager 105.

7) Configured /etc/ppp/peers/adsl file like so:-

```

# this file has been generated by the configuration tool of the ECIADSL driver

debug
kdebug 1
noipdefault
defaultroute
pty "/usr/bin/eciadsl-pppoeci -vpi 0 -vci 38 -vendor 0x1690 -product 0x0215 -mode VCM_RFC2364"
sync
user "My.Username@btbroadband.com"

```
There is a load of lines after the above in this file, but it's only these that needed any attention. The VCI and VPI figures have to match what is used and those in the eciadsl.conf file (above). The user is the same as the eciadsl.conf file.

8 ) Configured /etc/ppp/pppoe-server-options file like so :-

```

# PPP options for the PPPoE server
# LIC: GPL
require-pap
login
lcp-echo-interval 10
lcp-echo-failure 2

```

9) Configured /etc/ppp/pppoe.conf file like so :-

```

# Ethernet card connected to DSL modem
ETH='tap0'

# PPPoE user name.
USER='My.Username@btbroadband.com'

# Bring link up on demand?
DEMAND=no

# Make the PPPoE connection your default route.  Set to
# DEFAULTROUTE=no if you don't want this.
DEFAULTROUTE=yes

```
There is a load of other lines, but I think that these are the only ones that I paid any attention to. Again, the USER is the same as in the earlier config files.


10) Configure the password files. 

/etc/ppp/chap-secrets

```

"My.Username@btbroadband.com"	*	"my-password"	*

```
Here again we have the same username as has been used throughout (not my real one) and a password that I setup on BT Broadband (no, it's not my real password).

/etc/ppp/pap-secrets

```

"My.Username@btbroadband.com"	*	"my-password"	*

```
And the same here.

Must be honest in that I'm not convinced that I really needed to do both files.

11) Now start the link. 

i) Plug the BT Voyager 105 into the telephone line using one of the dongle/adaptors supplied (yes, even the ADSL modem needs to use one of these).
ii) Plug the USB cable from it into your PC (the green power light will come on, as the modem gets its power from your PC's USB socket).
iii) Open a text terminal and switch to super user by entering **'su -'** followed by the root password (the su - command is without the quotes but with the hyphen - ).
iv) Enter **eciadsl-start**

The green DSL light will start to flash, and stay on permanently once the ADSL connection has been made (about 15-25 seconds). This will coincide with the 'Synchronization successful' message.

The next set of messages (LCP then the CHAP ones) slowly appear (total of about 10-15 seconds), then once the 'CHAP authentication succeeded' appears, the rest zip onto the screen, and the connection is up and running.

I have (paranoia) commented out the exact IP addresses that I get given with "nnn" , you will get real numbers (between 1 and 254). Also, the exact channel may be different, and the various magic numbers and CHAP challenge numbers will be different.

_Problems_
Sometimes it doesn''t get past the LCP stage. Don't know why, I just try again later and it works.

Sometimes I have to unplug the USB cable and reconnect it, to get the connection to work. Again, I don't know why, it just seems to need resetting.	

When I first used this connection, I had a number of telephone extensions leads connected end to end so that I could reach the telephone socket (downstairs). This caused some flaky connectivity, so I replaced them with a single cable and all is well.
