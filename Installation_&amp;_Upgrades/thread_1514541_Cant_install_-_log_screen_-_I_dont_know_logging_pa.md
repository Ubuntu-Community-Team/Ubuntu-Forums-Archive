### Post by ajgreeny on 2010-06-21
There should be no request for a password if you have either run the live CD, where you should end up at the gnome desktop, or have gone straight to the install option, which would take you through the installation, with partitioning etc etc.

If you downloaded the .iso file it may be corrupted or the burn to CD may be faulty.

Start again and when you boot to the CD press any key on the keyboard as it boots.  A menu will appear and you can check the integrity of the CD you are using.  If it fails, burn or download again.  You can check the .iso file itself either in windows, though I do not know how in that, or in the Ubuntu live CD if it will boot, with md5sum name-of.iso and check the output with the list in [https://help.ubuntu.com/community/UbuntuHashes](https://help.ubuntu.com/community/UbuntuHashes)
