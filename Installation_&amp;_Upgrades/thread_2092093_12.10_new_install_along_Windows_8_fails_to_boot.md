### Post by oldfred on 2012-12-06
Clickable link.
[http://paste.ubuntu.com/1415667/](http://paste.ubuntu.com/1415667/)

You have installed grub to the PBR - partition boot sector of Windows. The NTFS PBR has to have Windows code not grub in it. NTFS keeps a backup and testdisk can restore that.

> sda1: ________________________________________________

    File system:       ntfs
    Boot sector type:  [COLOR=Red]Grub2 (v1.99-2.00)[/COLOR]
    [COLOR=Red]Boot sector info:  Grub2 (v2.00) is installed in the boot sector of sda1 
                       and looks at sector 100048688 of the same hard drive 
                       for core.img. core.img is at this location and looks 
                       in partition 1 for (,msdos3)/boot/grub. No errors 
                       found in the Boot Parameter Block.[/COLOR]
    Operating System:  
    Boot files:        /bootmgr /Boot/BCD

Fix for most, a few have other issues, better than windows fix in many cases as it also fixes other parameters:
If win7 use small 'system reserved' NTFS partition instead of the partition where windows was installed for win7
This has instructions on using testdisk to repair the install of grub to the boot sector for windows from Ubuntu or Linux LiveCD.
[http://sourceforge.net/apps/mediawiki/bootinfoscript/index.php?title=Boot_Problems:Boot_Sector](http://sourceforge.net/apps/mediawiki/bootinfoscript/index.php?title=Boot_Problems:Boot_Sector)
You want to get to this screen:
[http://www.cgsecurity.org/wiki/TestDisk_Step_By_Step#NTFS_Boot_sector_recovery](http://www.cgsecurity.org/wiki/TestDisk_Step_By_Step#NTFS_Boot_sector_recovery)

OR:
