### Post by audiomick on 2009-12-14
If you are using 9.10, it should be grub2.

Ok, I just checked. At the start you said  fresh install, so it will be grub2.

How to find a file? It is in the name. At the pinnacle of your file system (if you imagine it as a pyramid. If you picture it as a tree, at the base ) is /
Inside that, there are lots of things, like /etc, /root, /home and so on.
Inside /etc is (amongst others) /etc/X11
Inside /etc/X11 should be the text file /etc/X11/xorg.conf
If you click on it, it will open in a text editor, probably gedit. You wont be able to edit it, as it belongs to root.
To edit it, do
```
gksu gedit /etc/X11/xorg.conf
```
in a terminal.
This will open the file in gedit with root privileges.

If you want to try editing it, save a backup first. If things go wrong, you can put the original back.

If you can't find xorg.conf in /etc/X11, then the x server is running entirely on default settings. I have seen a few posts from people who say they don't have one. If you get to the point where you want to try editing it, and it doesn't exist yet, as far as I know, you can create it, include the bits you want to influence, and it will get used at the next start of the GUI. There is a chance I may be wrong on this.

You can find the file in the file manager by clicking on places/filesystem. (or maybe it is called computer) Look around a bit, and you will find it.

You can navigate the file system in the terminal with cd and ls.
Open a terminal, at this point you are in your home folder.
Do
```
ls
```
and it will list the contents.
you can enter a folder in the folder you are in with
```
cd foldername
```

you can go up a step with
```
cd ..
```

you can jump to somewhere else using the full path
```
cd /etc/X11
```

this is case sensitive.
You can do ls anywhere, and it will show you what is in the folder you are in.

Pay attention to how the terminal cursor changes as you move around. The path is shown there.
