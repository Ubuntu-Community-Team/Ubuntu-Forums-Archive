### Post by Yoann_Maheo on 2020-07-16
I found that the issue was from new Grub 2.04, because I succeed to boot on my system with a Grub 2.02 on a live key.


And after, I revert the grub from 2.04 to 2.02


I found the package here : [https://pkgs.org/search/?q=grub](https://pkgs.org/search/?q=grub)


wget [http://archive.ubuntu.com/ubuntu/pool/main/g/grub2/grub-common_2.02-2ubuntu8.15_amd64.deb](http://archive.ubuntu.com/ubuntu/pool/main/g/grub2/grub-common_2.02-2ubuntu8.15_amd64.deb)
wget [http://archive.ubuntu.com/ubuntu/pool/main/g/grub2/grub-efi-amd64-bin_2.02-2ubuntu8.15_amd64.deb](http://archive.ubuntu.com/ubuntu/pool/main/g/grub2/grub-efi-amd64-bin_2.02-2ubuntu8.15_amd64.deb)
wget [http://archive.ubuntu.com/ubuntu/pool/main/g/grub2/grub-pc-bin_2.02-2ubuntu8.15_amd64.deb](http://archive.ubuntu.com/ubuntu/pool/main/g/grub2/grub-pc-bin_2.02-2ubuntu8.15_amd64.deb)
wget [http://archive.ubuntu.com/ubuntu/pool/main/g/grub2/grub-efi-amd64_2.02-2ubuntu8.15_amd64.deb](http://archive.ubuntu.com/ubuntu/pool/main/g/grub2/grub-efi-amd64_2.02-2ubuntu8.15_amd64.deb)
wget [http://archive.ubuntu.com/ubuntu/pool/main/g/grub2/grub-pc_2.02-2ubuntu8.15_amd64.deb](http://archive.ubuntu.com/ubuntu/pool/main/g/grub2/grub-pc_2.02-2ubuntu8.15_amd64.deb)
wget [http://archive.ubuntu.com/ubuntu/pool/main/g/grub2/grub2-common_2.02-2ubuntu8.15_amd64.deb](http://archive.ubuntu.com/ubuntu/pool/main/g/grub2/grub2-common_2.02-2ubuntu8.15_amd64.deb)

# remove current Grub 2.04
sudo apt-get purge grub-gfxpayload-lists grub-pc-bin grub2-common grub-pc grub-efi-amd64-signed shim-signed
# install Grub 2.02
sudo dpkg -i *.deb


then 


grub-install /dev/sda


update-grub


My system boot in with the "fail safe" option
