### Post by darkod on 2012-08-06
If the disks have been used in raid, there is usually meta data remains and the installer is ignoring them.

First go into bios and set the SATA mode to AHCI. No, you don't need to use raid if you don't want to.

After making sure the sata mode is AHCI, boot the ubuntu cd in live mode (Try Ubuntu), open terminal and execute:
```
sudo dmraid -E -r /dev/sda
sudo dmraid -E -r /dev/sdb
```

That will remove the meta data and the installer should work fine after that.

NOTE: DO NOT run the above commands if you ARE using raid because they will break the array. You install on raid with a different procedure.

### Post by darkod on 2012-08-09
It's very rare that ubuntu can't recognize the controller and subsequently doesn't see the disks, but it can happen.

Also, take a look at the motherboard manual, sometimes one SATA ports are part of one controller, and other of other. See if they can be recognized on another port.

If you boot into the live desktop, do the fdisk and parted command mentioned earlier see the disks or not? There is a small difference between these commands and the installer, so double check whether the commands see the disks or not.

### Post by darkod on 2012-08-10
If you boot in live mode it's not that easy to edit the grub files of the hdd installation. Remember that when in live mode it will show the /etc/default/grub but it's not the same.

If you boot only ubuntu, after the POST start hitting Shift and it will show the grub boot menu.

Once you add the parameter and you are in the hdd installation, edit the /etc/default/grub adding the parameter after the 'quiet splash' in the line saying something like GRUB_CMD...etc

After that run:
sudo update-grub

 to create the new config file that will save that change. From that point on it's permanent.
