### Post by Aearenda on 2009-05-03
It's an Intel card so no 3rd party drivers are needed. It seems that the horizontal sync reported by the monitor is weird - and a bad cable could indeed do that, as elliotn says. If that's not it, please post the contents of /etc/X11/xorg.conf. 

If you can, it would be useful to verify the monitor is working at higher resolutions using a different computer and/or a different operating system (or live CD).

EDIT: I've just come across your parallel thread, with more hardware details, and I suggest you stick with that thread for now. You can force the Intel driver to load by putting 'intel' instead of 'vesa' in /etc/X11/xorg.conf. Also, you can override the horizontal sync setting in there, as documented by entering 'man xorg.conf' in a terminal. However, it may well be that the driver has regressed for your system as suggested in the other thread. There have been threads about using leading edge intel drivers using a PPA, and also about reverting to older drivers using a different PPA (see [this wiki article]("https://wiki.ubuntu.com/ReinhardTartler/X/RevertingIntelDriverTo2.4")).
