### Post by darkod on 2012-04-04
It seems it can't understand your windows partition. Maybe something got corrupted during the shrink. Did you reboot win7 few times after the shrink? And which tool did you use?

Try doing the Repair This Computer option when you boot with the win7 dvd. If it works, it will probably overwrite grub2 on the MBR with the windows bootloader. Simply reinstall grub2 back to the MBR, you don't need to reinstall the whole ubuntu again.

### Post by Mark Phelps on 2012-04-05
From the script results, I'd have to guess that sda1 USED to be your original Win7 boot partition -- which is created by defaule on pre-installed Win7 PCs.  It typically has the Win7 boot loader directory and files stored there.

The script reports what it finds in each partition, and it can't recognize the partition type.  So, it looks like you corrupted sda1 somehow.

Also, the script reports the presence of MS Windows by looking for a winload.exe file -- which in your case, it did not find.

And finally, if sda is your only drive, there is no NTFS partition -- which is where Win7 would have been located.

So ... it looks like you blew away Win7 in the process of installing Ubuntu.

Since the script didn't report a Recovery partition, if your PC came preinstalled with Win7, you won't be able to restore it.  In that case, you should contact the vendor of the PC to see if they will sell you full-restore optical media.  I've seen prices around $10 USD for these -- which is a LOT less than Win7 retail price.

### Post by darkod on 2012-04-05
Mark, unless something really weird happened, sda1 can't be the win7 boot partition. It takes almost all of the hdd, not just 100MB or 200MB.

On the disk of 160GB, sda1 is approx 125GB and sda2 30GB.

It looks like something corrupted sda1 and maybe simply chkdsk will solve it. Especially since the OP didn't use the ubuntu installer to shrink it, it was done before the ubuntu install. It only depends with what tool and how it went. And whether win7 was confirmed as working after that.
