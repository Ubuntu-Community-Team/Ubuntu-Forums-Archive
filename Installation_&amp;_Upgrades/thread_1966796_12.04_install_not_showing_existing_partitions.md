### Post by oldfred on 2012-04-27
If file system is corrupted then the partitioner has issues trying to see it.

```
root@ubuntu:~# fdisk /dev/sda

Command (m for help): p

Disk /dev/sda: 250.1 GB, 250059350016 bytes
255 heads, 63 sectors/track, 30401 cylinders, total [COLOR=Red]488397168[/COLOR] sectors
Units = sectors of 1 * 512 = 512 bytes
Sector size (logical/physical): 512 bytes / 512 bytes
I/O size (minimum/optimal): 512 bytes / 512 bytes
Disk identifier: 0xd50fcf61

   Device Boot      Start         End      Blocks   Id  System
/dev/sda1   *        2048    22530047    11264000    c  W95 FAT32 (LBA)
/dev/sda2        22530048   190302200    83886076+   7  HPFS/NTFS/exFAT
/dev/sda3       190304256   256194559    32945152   83  Linux
/dev/sda4       256194603   [COLOR=Red]488408129[/COLOR]   116106763+   f  W95 Ext'd (LBA)
/dev/sda5       256196608   266727407     5265400   82  Linux swap / Solaris
/dev/sda6       266729472   488396799   110833664    7  HPFS/NTFS/exFAT
```

The end of your extended partition is beyond the end of the drive.

First backup partition table, use your drive for sdX or sda, sdb etc.
sudo sfdisk -d /dev/sdX > parts.txt


Fixparts - Repair broken partition tables (not overlapping issues) & delete Stray gpt data from MBR drives
[http://ubuntuforums.org/showthread.php?t=1705325](http://ubuntuforums.org/showthread.php?t=1705325) 
[http://www.rodsbooks.com/fixparts/](http://www.rodsbooks.com/fixparts/)

### Post by oldfred on 2012-04-27
Never any guarantees on partition edits, and good backups are always important before any system changes.

But many users have reported good results with fixparts. And all you really are doing is editing a number in the partition table. 

You can manually do it also with sfdisk by changing the text file you save and rereading it back in. So I think it is a lower risk change.
