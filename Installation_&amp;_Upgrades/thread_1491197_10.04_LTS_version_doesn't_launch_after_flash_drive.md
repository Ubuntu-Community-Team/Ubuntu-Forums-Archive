### Post by darkod on 2010-05-25
Yes, the fdisk listing looks regular. Don't worry about sda4. The thing is that linux reserves sda1-sda4 for primary partitions, and logical partition (inside an extended partition) start from sda5.

So it seems your extended partition is sda2 and it's like a container holding the logical partitions sda5 and sda6.

That's why it looks like sda4 is missing, but in fact it's not. It never existed. Just sda1-sda3 and sda5-sda6 are inside sda2.

But by handling the problem this way, I had problems with xp booting (because xp was installed in ahci mode, when you switch to compatible mode, the device is not recognize and the only solution is to reinstall xp in compatible mode...).
djmoore85 found the solution and gave it to me by PM. Here's the link that solved my boot problem :
[https://wiki.ubuntu.com/HardwareSupport/Machines/Netbooks/ToshibaNB205](https://wiki.ubuntu.com/HardwareSupport/Machines/Netbooks/ToshibaNB205) (Section Kernel Boot fix)

On this wiki page, they say to add in the "/etc/grub.d/10_linux" file at the end of this line :
```
 linux   ${rel_dirname}/${basename} root=${linux_root_device_thisversion} ro $2
```
This :
```
nohz=off
```
So that you have this :
```
 linux   ${rel_dirname}/${basename} root=${linux_root_device_thisversion} ro $2 nohz=off
```
But for me the line looks that way :
```
linux   ${rel_dirname}/${basename} root=${linux_root_device_thisversion} ro ${args}
```
So the modified line should looks that way:
 ```
linux   ${rel_dirname}/${basename} root=${linux_root_device_thisversion} ro ${args} nohz=off
```
You don't have to omit to update grub after you have changed the file :
```
sudo update-grub
```

This solved my problem without switching to compatible mode. Ubuntu boots know in less than 30 seconds \\:D/
