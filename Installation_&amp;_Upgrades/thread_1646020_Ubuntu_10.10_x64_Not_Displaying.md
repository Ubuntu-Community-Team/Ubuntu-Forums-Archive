Welcome to the forums :-)

You mean it goes to a blank screen when you select Ubuntu from the Grub menu? If yes, highlighting the first entry in Grub menu, press 'e' to edit it. Navigate to words "quiet splash", delete them and type "nomodeset" in their place (without quotes). Press Ctrl + X to continue boot and see if it takes you to the desktop this time.

If not, we need to know which graphics card is there.

And also, it'd be even better to boot an Ubuntu Live CD/USB and post the output of bootinfoscript as per instructions here.

[http://bootinfoscript.sourceforge.net](http://bootinfoscript.sourceforge.net)

It might reveal some ongoing problems with Grub or so.

While posting, click the # icon in post menu and paste your text in between the generated code tags.

### Post by Hippytaff on 2010-12-15
Perhaps if we can see the results.txt generated by the script from the following link it will give a better indication of what is going on. Boot from livecd to run it

[http://bootinfoscript.sourceforge.net/](http://bootinfoscript.sourceforge.net/)

:-)

Also with Nvidia graphics card you might need to do the following if you have trouble with the display.

When you get to the bit of the boot process where you choose OS's, highlight ubuntu and press e. Then change 'quiet splash' to 'nomodeset' and press CTRL+X to boot. Once your in do
```

sudo apt-get update && sudo apt-get upgrade

``` in a terminal. Applications -> accesories -> terminal.

Then go to System -> Administration -> Alternative Drivers and install the recommended Nvidia graphics driver.
