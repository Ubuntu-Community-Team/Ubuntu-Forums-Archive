### Post by chadlongstaff on 2008-05-26
Regarding the ipaq not holding a charge, this can be a sign that the kernel needs training on how to interpret the info from the battery. You might find this page useful - [http://www.handhelds.org/moin/moin.cgi/battery_20learning_20_28hp55xx_29](http://www.handhelds.org/moin/moin.cgi/battery_20learning_20_28hp55xx_29) - to get enough charge to boot for learning I had to revert to a windows image, but it's working ok, now. Hope this helps ;)
