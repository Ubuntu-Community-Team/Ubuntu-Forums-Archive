### Post by oldfred on 2011-10-24
You installed the grub2 boot loader to sda5 a partition, not the MBR or sda. If you are mostly a Windows user you could use EasyBCD but since you have two drives I would just install grub2 to the MBR of sda.

#Comments are anything after the #, enter commands in terminal session
#Install MBR from LiveCD, Ubuntu install on sda5 and want grub2's bootloader in drive sda's MBR:
#Find linux partition, change sda5 if not correct:
sudo fdisk -l
#confirm that linux is sda5
sudo mount /dev/sda5 /mnt
#If grub 1.99 with Natty uses boot not root.
sudo grub-install --boot-directory=/mnt/boot /dev/sda

# If no errors on previous commands reboot into working system and run this:
sudo update-grub

Note that your Windows install is in sda2, but boots from the hidden 100MB boot/reovery partition in sdb1. If you install grub to sda's MBR you can still directly boot windows by changing BIOS to boot sdb.

I prefer to have each system installed on separate drives with all boot files & MBR set to boot that drive. Then if one drive fails you can independently boot the other. Your Windows is partially on sdb and mostly on sda with Ubuntu.
