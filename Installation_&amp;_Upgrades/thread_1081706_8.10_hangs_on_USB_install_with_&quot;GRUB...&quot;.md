### Post by caljohnsmith on 2009-02-26
It sounds like it may be an issue with how your USB drive is configured in your BIOS, but to get a better idea if that might be the case, how about downloading the **Boot Info Script** to your Live CD Ubuntu desktop:

[https://sourceforge.net/projects/bootinfoscript/](https://sourceforge.net/projects/bootinfoscript/)

Make sure your Ubuntu USB drive is connected, then open a terminal (Applications > Accessories > Terminal) and do:
```
sudo bash ~/Desktop/boot_info_script*.sh
```
That will create a "RESULTS.txt" file in the same directory from where the script is run, namely your desktop; please copy/paste the contents of the RESULTS.txt file to your next post, highlight the copied text, and click the pound/hash sign "#" graphic button in the Ubuntu forum message box so that the text will get "code" tags put around it. The results of that script will help clarify your setup and hopefully what the solution to your booting problem might be.

### Post by caljohnsmith on 2009-02-27
Just run the script from a Live CD on whichever other computer you used to install Ubuntu to the USB drive, and make sure the Ubuntu USB drive is connected when you run the script. That should give us the information about your Ubuntu USB drive to know whether it is a problem with how Ubuntu/Grub is installed, or if your have maybe an issue with the BIOS on the computer where booting the Ubuntu USB drive causes it to hang.

### Post by caljohnsmith on 2009-02-27
Grub in the MBR (Master Boot Record) of your 2 GB USB drive is definitely not configured correctly, and it looks like part of the reason may be because the sdc1 FAT partition starts at the sector immediately following the MBR; usually the first partition on a drive starts no sooner than sector 63, since that is the beginning of the next track on the HDD. I would recommend reformatting your sdc USB drive so that the first partition does not start sooner than sector 63, and then try reinstalling Ubuntu to that drive. So from the Live CD, if you first open gparted (System > Admin > Partition Editor), select the sdc USB drive, go to Device > Create Partition Table, click OK, then create a new primary FAT32 partition on the drive using gparted. Once you've done that, try installing Ubuntu to the USB drive again, and let me know how that goes.
