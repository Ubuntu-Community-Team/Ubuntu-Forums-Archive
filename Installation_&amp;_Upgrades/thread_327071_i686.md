### Post by insane_alien on 2006-12-28
the generic kernel handles the optimizations for the i686 arcitecture. the i386 and i686 (and i think k7) kernel packages are superfluous now as the generic kernel can handle all of them with the same performance as the specific kernels now. i think they're actually getting scrapped in fiesty but don't take my word for it.

i use the generic kernel on my i686 it runs fine and there isn't any difference if i load up the i686 kernel.
