### Post by westie457 on 2011-10-11
> **propercoil said:**
> The last thing i need is my ubuntu ruined after implementing an advice from a "windows guru".

Hello, welcome to the Forum.

Unfortunately you are going to temporarily lose the use of ubuntu.

Two things are needed. The first is a Windows install or repair disc (not a restore disc), the second is a LiveCD/USB (the same version as already installed) eg Natty 11.04

Repair Windows which will remove Grub then use the LiveCD to repair Grub.

Try the Windows automatic repair first then drop to a command prompt if the repairs do not work. In the command prompt type in and run these commands one at a time

```
bootrec.exe /fixmbr

bootrec.exe /fixboot

bootrec.exe /rebuildbcd
```

Note there is a space between the .exe and the /

Check Windows now boots okay.

Next reboot into the LiveCD in TRY mode and open a terminal and run ```
sudo update-grub
```

Everything should now work properly.

Hope this helps.

### Post by Quackers on 2011-10-11
It would appear that at some time you have attempted to create a 5th primary partition. The maximum number of primary partitions on a MBR partitioned disc is 4. This has resulted in your Windows partitions being changed from basic to "dynamic disks".

Although a few people have had some luck converting the dynamic disks back to basic partitions I would be inclined towards re-installing both Windows then Ubuntu - this time not exceeding the 4 primary partition maximum.

Here is a recent thread regarding converting the partitions
[http://ubuntuforums.org/showthread.php?t=1779529](http://ubuntuforums.org/showthread.php?t=1779529)

### Post by oldfred on 2011-10-11
The partitions are labeled SFS type 42 which is common for dynamic partitions.

You can run the Windows repairs from the terminal line and just not run the fixMBR, Then grub should not be overwritten. If you run the auto repair you usually have to run it 3 times and then it will overwrite grub. Many times chkdsk is also required.

Some with dynamic partitions have reported that windows repairs just do not work. Then seem to work fine on basic partitions.  IF dynamic partitions are still matching the physical partitions and you only have 3 or 4 partitions,  converting back should work. But any partition changes require good backups as who knows what may happen.
