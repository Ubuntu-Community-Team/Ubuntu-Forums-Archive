### Post by ajgreeny on 2016-05-23
You will have to disable the [http://ppa.launchpad.net/ffmulticonverter/stable/ubuntu](http://ppa.launchpad.net/ffmulticonverter/stable/ubuntu) repositories as there is no **xenial** version available; it stopped with **utopic**.

The google-chrome error is probably related to the loss of 32 bit chrome builds for Linux, and I think you can overcome that by following the info at [http://askubuntu.com/questions/724776/disable-google-chrome-warning-about-32-bit-on-ubuntu](http://askubuntu.com/questions/724776/disable-google-chrome-warning-about-32-bit-on-ubuntu)

I think that if you are using a 64 bit system the best way is to remove the current repository for chrome and then download and install the most recent google-chrome which will add the repository again to your software-sources but this time with changes to remove that error.  I don't use chrome, only chromium, so this is not from my personal experience.

It appears that you also need to run command ```
sudo dpkg --configure -a
``` to make sure that all installed packages are properly configured.
