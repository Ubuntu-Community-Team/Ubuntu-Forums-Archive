### Post by YesWeCan on 2011-06-25
Hi there.
I can see a problem with your fstab file. It references root on sda1 instead of sdb1. So when you booted without the USB it could not find root until you put the USB in.

It is best in fstab to use UUIDs to identify partitions to avoid mis-numbering issues like this. I don't know why the installer didn't do this, it did for your swap partition.

Make a backup copy of fstab:
[COLOR="DarkGreen"]sudo cp /etc/fstab /etc/fstab.bak[/COLOR]
Then edit your fstab and replace the line
```
/dev/sda1  /   ext4    errors=remount-ro  0 1
```
with
```
UUID=28e84dc4-5cc9-484c-a98e-7f5ce9b8c47a  /   ext4    errors=remount-ro 0 1
```

BTW the command [COLOR="DarkGreen"]sudo blkid[/COLOR] lists the UUIDs for all your partitions
I think the dmraid is a red herring.

> **jarryslink said:**
> recently i installed ubuntu 10.04 from a cd to my pendrive. it worked fine but i can boot only with my pen drive inserted. i visited two more threads in this forum with same topic but no answers worked for me(i have not tried any third party mbr repair softwares). i have to go through grub to access my recovery drive which is a part of my hdd.( i do not have any command prompt options in my windows recovery console which is in my hdd. i want to be able to use grub and linux but i dont want to insert my pen drive for every boot.
[edit]Hang on. What is it you cannot boot without the pen drive installed?
It may be because when you installed to the pen it put the Grub MBR on your HD, thus making a dependency between your HD boot and the pen drive.

Yes, this is probably the cause. So You need to put Grub on the pen drive and restore the standard MBR on the HD. Boot Ubuntu on the pen and run
[COLOR="Navy"]sudo fdisk -l[/COLOR]
and identify the drive name for the pen, eg: sdf. Then do
[COLOR="Navy"]sudo grub-install /dev/sdf[/COLOR]
now identify the name of the Windows disk, eg: sda
[COLOR="Navy"]sudo apt-get install lilo
sudo lilo -M /dev/sda mbr[/COLOR]
(ignore warnings)
