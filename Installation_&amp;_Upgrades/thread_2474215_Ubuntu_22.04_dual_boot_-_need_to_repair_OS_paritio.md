### Post by cookieninja22 on 2022-04-24
Hey there, has anyone else had this issue and know a permanent solution?

I have a desktop system which has both Ubuntu and Windows installed on it.

Every time I shut down Windows and then boot into Ubuntu 22.04 it fails to boot successfully.
The "fix" I use is always to boot a Live CD and do the following:
* Open the Disk application
* Check the Ubuntu OS partition to confirm it is the expected issue
* Repair the Ubuntu OS partition
* Check the Ubuntu OS partition to confirm it reports the partition is now okay
* Reboot into Ubuntu 22.04

If I am just shutting down Ubuntu 22.04 and booting back into it again, either right away via reboot or later on, then it is always fine.
This issue is only ever after I spend a bit of time using Windows 10, and then try to return to using Ubuntu 22.04

In case it is important, I upgraded Ubuntu directly from the previous LTS release and did back in Feb/March when it was still beta.
This problem only appeared after the upgrade, I have never had an experience like this before, and I am sure my disk is fine!
