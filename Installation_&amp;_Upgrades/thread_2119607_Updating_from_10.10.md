> You download and install PPA packages at your own risk. Ubuntu, Launchpad and Canonical do not endorse these packages. You must be certain that you trust the PPA owner before you install their software.

[https://help.ubuntu.com/10.04/add-applications/C/adding-repos.html]("https://help.ubuntu.com/10.04/add-applications/C/adding-repos.html")

It is nothing to panic over. When installing software from a PPA and doing it through the terminal we need to authenicate the action.

When trying to upgrade from one version of Ubuntu to another we must remove those PPAs from Software Sources. How do you do this in 10.10? That is a difficult thing to answer because I cannot now even remember what 10.10 looked like. I think that you open Synaptic Package Manager>Settings or Repositories or something like that.

The upgrade will work better without those PPAs there to trip it up. Also you should bring the desktop back to a default state as possible. Upgrades are designed to work from one default desktop to another. Failure happens when we heavily modify the desktop environment.

It should still be possible to update from one out of life version to another as that would be the upgrade path to the very latest version.

```

## EOL upgrade sources.list
# Required
deb http://old-releases.ubuntu.com/ubuntu/ CODENAME main restricted universe multiverse
deb http://old-releases.ubuntu.com/ubuntu/ CODENAME-updates main restricted universe multiverse
deb http://old-releases.ubuntu.com/ubuntu/ CODENAME-security main restricted universe multiverse

# Optional
#deb http://old-releases.ubuntu.com/ubuntu/ CODENAME-backports main restricted universe multiverse

```

You should replace CODENAME with maverick for 10.10 or natty for 11.04. 

It would be better to fully update 10.10 before attempting a version upgrade.
