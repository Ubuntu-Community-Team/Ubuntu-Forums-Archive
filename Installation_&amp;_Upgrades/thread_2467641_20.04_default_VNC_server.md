### Post by TheFu on 2021-10-03
VNC isn't a default.  It is an add on package.  There are multiple, better, options than using VNC, BTW.  Think the last time I used VNC was around 2012. It really is a very bad choice for the problem of running remote GUI programs.  Ok, that isn't true.  VNC is part of libvirt and connecting by default to local virtual machines. Still, there are better answers - like SPICE, x2go, or straight remote X11 through an ssh tunnel, which is the easiest by far.

```
$ ssh user@remote-system "command"
```

That's what I use nearly all the time. So easy. So fast.
