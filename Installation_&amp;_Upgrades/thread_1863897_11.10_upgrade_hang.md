### Post by OldWizard on 2011-10-18
I'm not sure how the startup sequence is connected,but I was running 11.04 and did the upgrade to 11.10. On startup, the cursor was in the upper left hand corner and stopped. I had researched this issue before and ended up turning off acpi to get it to boot. On bootup the disk drive devices changed (I have 4 disk drives and linux was on /dev/sdd1) I was now getting errors about fsck failing, but it was checking /dev/sdc1.

I change the fstab to fix the drive I was mounting (root is now at /dev/sdc1) and I was able to boot up.

I then went back to my original configuration w/grub taking out the acpi=off and it worked. I have as part of the boot parameters clocksource=acpi_pm which makes the acpi work.

My question is for future releases. How can upgrades be handled to not change the sequence of disk drives? This took a few hours to figure out and I don't want to have to do this with each upgrade.
