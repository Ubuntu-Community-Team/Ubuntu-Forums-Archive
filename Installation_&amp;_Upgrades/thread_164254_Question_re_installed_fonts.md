### Post by teasum on 2006-04-22
Noobie here... (Everything in this post concerns Breezy.)  Has anyone else noticed the fonts that Ubuntu installs by default?  I myself need to input Thai as well as English, so I always go ahead and install additional fonts and language tools after installation.  But I also noticed that Ubuntu installs by default a whole set of Indic fonts, and Korean fonts as well, for example.  I'm talking Tamil, Telugu, Malayalam, Oriya, Kannada, etc.  

One of the things I love about Ubuntu, especially over other linux distros, is the commitment to support for multiple languages.  However, why are these fonts installed by default?  Shouldn't the installer ask for secondary languages, or just install the default language (with the expectation that you'll install any other languages you need later)?  At first I thought it just installed all international fonts by default, but I still had to install Thai fonts, so I don't think that's it.

[EDIT]I noticed this when I did a server install on an old 200mHz machine the other day... when I tried to install a desktop (I think I tried UbuntuLite) the font configuration took up a big chunk of the install time, as well as a good deal of HD space (that box only has 4GB).  The same fonts are included in Ubuntu and XUbuntu, to my knowledge.  So, to rephrase, why does the installation include all these font packages, even on 'lite' or 'low-ram' installations? 

Just curious.
