### Post by Porpoise on 2008-01-25
Well, having been running Ubuntu for some time now on my main PC (and falling in love with it), I decided to bite the bullet and install it (dual-boot with XP-PRO) on my Acer laptop.
 
Everything installed fine and it was working great over the wired network/internet connection. Then managed to get it working fine over WI-FI (via Belkin 54g card with driver setup gleaned from here somewhere).
 
Then, having installed Ubuntu Studio onto the PC, and liking it, I decided to install it on the laptop also. Then the problems started! It hosed my WI-FI card setup. After going round in circles for a couple of hours trying to get it working again, I came to the conclusion that Ubuntu Studio and the Broadcom driver for the Belkin card just don't want to live together, so I've reverted back to the non-Studio installation.
 
Just thought I'd mention it here, in case I can save someone else the heartache, or someone comes up with a fix.
 
The next job on the agenda is to build my new AMD64 x2 system this weekend with Vista 64 Ultimate/Ubuntu 64 dual-boot with RAID - should be fun! (I only got VISTA 'cause it was at a give-away price with the motherboard/processor and realised that I wouldn't be able to re-utilise my existing W2K on the the 64bit system - and W2K is end-of-life now anyway).
 
I'll let you know how it goes (if anyone is interested!).
