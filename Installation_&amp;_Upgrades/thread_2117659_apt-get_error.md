### Post by varunendra on 2013-02-19
> **farhan mazhar said:**
> We are trying to set up private cloud in our institute using ubuntu enterprise cloud 10.04.
First of all, if you are setting up something new, you should consider a distro which would be supported long enough. 10.04 is about to reach its [End of Life]("https://wiki.ubuntu.com/Releases") this April - meaning you won't get any active updates for it afterwards.

As such, I would recommend 12.04 - 64 bit.


Next -
> W: Failed to fetch http://**in.archive**.ubuntu.com/ubuntu/dists/lucid/main/source/Sources.gz  403  Forbidden [IP : 91.189.92.202 80]
..
..
W: Failed to fetch http://**[COLOR="Red"]nz.archive[/COLOR]**.ubuntu.com/ubuntu/dists/lucid-backports/restricted/binary-i386/Packages.**gz[COLOR="Red"].gz[/COLOR]**  403  Forbidden [IP : 91.189.92.202 80]
Looks like you are using multiple servers in your sources.list file.

I don't know how you did it, but from what I understand about it *(and not very confident though)*, it is not a good idea as it may cause dependency errors even if can be successfully updated.

Also, as highlighted above, some of the entries are certainly wrong (ending with double '.gz'). There are no such files on the location the entries are pointing to.

And last, to me it seems that you are not able to access the internet at all. Can you manually get a file, or reach its page by clicking on its link? For example, try these-
[http://in.archive.ubuntu.com/ubuntu/dists/lucid/main/source/Sources.gz](http://in.archive.ubuntu.com/ubuntu/dists/lucid/main/source/Sources.gz) (for list file)
[http://in.archive.ubuntu.com/ubuntu/dists/lucid/main/source](http://in.archive.ubuntu.com/ubuntu/dists/lucid/main/source) (to visit its page)

First make sure you can access internet, then we may need to look at your **/etc/apt/sources.list** file to avoid any further possible errors.

But above all, please consider using a newer version instead of 10.04.
