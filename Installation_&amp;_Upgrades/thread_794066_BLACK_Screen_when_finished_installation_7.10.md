### Post by dstew on 2008-05-15
Pick recovery mode. You will get a text-based interface. You might have to log in, but I can't remember.

Anyway, on the command line enter```
sudo dpkg-reconfigure xserver-xorg
```That will get you a text-based program to reconfigure the software that runs the desktop graphical interface. Answer the questions the best you can, usually the default answers are correct.

When you get to the part about the display, choose the **vesa** driver. For the highest resolution, choose the one you usually use.

When you get to the end of the reconfiguration program, enter startx on the command line. That should get you a usable display. At that point you can see if there is a restricted driver available for your display. Check in the System --> Administration menu, Restricted Drivers Manager.

### Post by dstew on 2008-05-15
I have an aversion to directly editing the xorg.conf file, but here I think I will make an exception. First, make a backup of the file```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.old
```Then, use the nano text editor and edit the file:```
sudo nano /etc/X11/xorg.conf
```Change this line in the ServerLayout section:```
Screen      0  "aticonfig-Screen[0]" 0 0
```to```
Screen      0  "Default Screen" 0 0
```After editing hit ctrl-X (^X in the nano menu) to save the file and quit. Then try startx again.
