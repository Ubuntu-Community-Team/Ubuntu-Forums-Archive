I do have an external USB dvdrw drive, and, even though my BIOS detects and supports booting from USB HDDs, it apparently does not work for USB CD/DVD drives (or maybe it is HP's fault?).
I am trying to install to the external HD - it is clear of data and I can format/partition it any way I want. I would like to avoid having to buy another drive (e.g. USB thumb) just for this one purpose.
The closest I have gotten to doing this is with this tutorial I found for loading the Kubuntu Live CD from a USB pen drive: [http://www.pendrivelinux.com/usb-kubuntu-810-persistent-install-using-windows/](http://www.pendrivelinux.com/usb-kubuntu-810-persistent-install-using-windows/)

The difference is that I am using a regular HDD instead of a pen drive, but as far as I can tell, the only difference is that I did not have to use the HP USB format tool. Here is what I did:

1. Used Acronis Disk Director to create a small (5 GB) primary FAT32 partition on the external USB drive (from WinXP), leaving the rest of the drive unallocated.
2. Downloaded the Kubuntu810P.exe file from the site above, moved the ISO file as per instructions there, and ran the .bat file.
3. The batch file opened a command prompt in WinXP, prompted me for the drive letter of my target drive, then basically extracted the ISO into that drive along with some additional files, one of which was a Windows executable that makes the drive bootable. Since I am not using a USB flash drive, I had to run this executable manually with the -f option to override the check.
4. Restarted, went into BIOS and manually selected to boot from USB HDD.

Amazingly (as I had tried a million other things), the Kubuntu menu appeared, and I went ahead to install. However the installer only let me choose my primary HD (the one with WinXP - I need that one!) to install to, when I wanted to install it to the unformatted partition on the external HD.

I do not think it is due to the partition not being formatted as a filesystem, because for the primary HD, Dell already split this into 3 partitions (one FAT16 for their bootable disk utility software, one NTFS for WinXP, and one empty) and the Kubuntu installer had no problem recognizing all 3 of them.
