### Post by abcStu on 2007-05-21
I expect others have been frustrated by the desktop or alternate CDs' install menu.  I have been for several tries now.  
My problem is, that there is generally no information available at all about the differences between the installation menu items.  Granted, "memory test" and "CD test" are obvious, but that's where the obvious ends.  I even mounted the CD in my existing Linux installation and tried to find a text file somewhere with this information, or shell script which would decipher it.
Another installation 'gotcha': If you don't set the screen resolution quite high, some of the text menus go off screen, so you have no idea what your choice is.  With many language based choices, this is a disaster, as the installation breaks from that point.

Also, could not the user be asked if he wants to install lilo or grub?  That is, ask which or neither?
He may wish, as I, to boot using an existing lilo on another drive and installed on its mbr.

It is nice that the installer figures out the video and sound devices and gets them going-- when it does.  It would be nicer still if it fell back to a text mode and got it done, with the hope that additional drivers could fix the issue.  I had that problem with 6.10 alternate for athlon 64, my desktop system.  Couldn't get around it.  It has a nvidia chipset with onboard ge6100 graphics.

Overall: nice, could be nicer.
