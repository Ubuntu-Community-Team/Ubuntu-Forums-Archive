### Post by Sef on 2009-06-26
1) Have you md5summed the download?  If not, maybe the download was bad.

2) Have you burned the iso as an iso?

3) Have you burned the iso at 4x or less?  Faster can sometimes create a bad image.

4) Have you checked 'Check disk for errors' or something similar?   It is about 5th down the list of the option menu.

5) Bad dvd is another possibility.

### Post by adam_kimber on 2009-06-26
I had some problems with install before. Some CD burners do not like some disks, and whilst it burns ok, the data can be corrupted. The disk check and to a limited extent the md5sum can help get around that. 

I would run the live cd and check the disk. That way you should know if the contents are ok. Once that is done, try booting a live session and play for a bit to see if all is ok. Then install. 

IF you want to dual boot please search for guides to do that. The installer might want to use entire disk for install. Do not proceed with that until you have backed up all data that you require as this will format the hard drive completely.

### Post by lisati on 2009-06-26
If the "Try Ubuntu" option on the CD's menu isn't working, then it would be a good idea to use the "Check CD" option **before** you even think of trying the "install" option. 

Another possibility to check is to see if the CD works in another CD drive. I have one DVD which reports errors if I try to use it in the built-in DVD reader in my laptop, but works perfectly when I use an external DVD reader that plugs in via USB.
