### Post by kerry_s on 2008-03-27
give it at least 1gig.

keeping it below that is the hard part. you'll need to fine tune it so things like logs, doc's, man's, don't pig up all the space.

i would recommend debian instead, it's smaller, faster, and more stable. you also don't have to keep up with each release.
[http://cdimage.debian.org/debian-cd/4.0_r3/i386/iso-cd/debian-40r3-i386-businesscard.iso](http://cdimage.debian.org/debian-cd/4.0_r3/i386/iso-cd/debian-40r3-i386-businesscard.iso)

to get a base install, when it gets to package selection, uncheck them all.

then just log in as root and apt-get the programs you want to use for your server.

by the way you do know a server install is a no gui thing?

you can always add gui->
apt-get install xorg mc (pick a small wm)

example: i use jwm, really small full function wm, mine usually starts like this->
apt-get install xorg synaptic jwm mc rox-filer leafpad iceweasel

then "exit" to get out of root
log in as my user
startx
open synaptic and build on.

i'm at 543mb, so if you skip the gui you can get pretty small. mines etch/lenny, i added the lenny repos and the multimedia.
