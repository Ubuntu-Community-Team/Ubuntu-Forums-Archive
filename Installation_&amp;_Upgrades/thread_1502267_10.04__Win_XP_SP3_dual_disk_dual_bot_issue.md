### Post by oldfred on 2010-06-05
Your windows is now on sdb or drive1. Your boot.ini still says drive 0. Not sure if you should just change boot.ini or the drivemap command. Do not use gedit to edit windows files as the line endings may be corrupted.
If editing windows files like boot.ini
(Using nano instead of gedit, it's better for dos-style linebreaks)
Edit Boot.ini
[http://support.microsoft.com/kb/289022](http://support.microsoft.com/kb/289022)

[boot loader] 
timeout=30 
default=multi(0)disk(0)[COLOR=DarkRed]rdisk(0)[/COLOR]partition(1)\WINDOWS 
[operating systems] 
multi(0)disk(0)[COLOR=DarkRed]rdisk(0)[/COLOR]partition(1)\WINDOWS="Microsoft Windows XP Professional" /fastdetect /NoExecute=OptIn 

To change this copy to 40_custom and edit.
I thought that the drivemap command in grub made windows think it was drive 0 but I think this should be (hd1) ${root}  or (hd1) (hd0)
drivemap -s (hd0) ${root}

Copy the windows entries from this:
gedit /boot/grub/grub.cfg
Copy them to and edit :
gksudo gedit /etc/grub.d/40_custom
