### Post by oldfred on 2023-07-26
You cannot run fsck on drive, but run it on ext4 formatted partitions.
You have to use Windows tools to repair a NTFS formatted partition.
Be sure to use gpt partitioning, not very old MBR, which many partition tools still default to.

So you do have to create partition(s) first and then format them.
NTFS is ok for sharing between Windows & Linux, but you must make sure Windows fast start up is off or it sets hibernation flag & then Linux NTFS driver will not write into that partition. 
And NTFS does not support Linux ownership & permissions, so default mount is the default set of permissions.

### Post by MAFoElffen on 2023-07-26
What is your plan for disk /dev/sda?

If you want to create just one partition on /dev/sda, formatted as ext4, then do
```

sudo su -
DISK=/dev/sda
sgdisk --zap-all $DISK
sgdisk -n1:0:0 -t1:8200 $DISK
sgdisk -p $DISK
mkfs -t ext4 ${DISK}1

```
That will clear anything that is on it, add a new GPT partition table, then add a new partition of full size, then format that partition as ext4...
