### Post by bgracian on 2009-08-02
Hi,
Has anybody had this experience:

While running a standard update (not upgrade) of Ubuntu 8.04 LTS, the Update Manager asked that an Ubuntu 9.04 installation CD be placed in the CD drive. Not doing so aborts the update.

Can anyone explain why this is happening and give a possible workaround? I'm concerned about mixing 9.04 code with the 8.04 installation. 

I'm running the AMD 64-bit version of Ubuntu 8.04 on an Athlon 64 X2 4400+ cpu in a commodity PC.

As added background, awhile back I installed Ubuntu 9.04 on a different hard drive that is no longer physically present in this machine.

Thanks for any help.

---

### Post by merlinus on 2009-08-02
System/Adminstration/Software Sources (or something like that) and untick cdrom.

---

### Post by bgracian on 2009-08-02
Thanks! That was spot on. I've unticked the cdrom as source and will retry the update.

---

### Post by bgracian on 2009-08-02
As a closing note, after unticking the cdrom as source, the update proceeded normally and completed successfully. Thanks again for your help, merlinus.
