### Post by guiverc on 2022-01-07
Lubuntu 22.04 by it's very name tells you it's the 2022-April release of Ubuntu/Lubuntu.

Ubuntu uses the *year.month* format, and always had, so if it was a 2021 release it would have been [Lubuntu 21.04]("https://lubuntu.me/hirsute-released/") or [Lubuntu 21.10]("https://lubuntu.me/impish-released/") both of which were released on schedule.

For Lubuntu *jammy*  (ie. the *developement*) release that will change name to 22.04 when it reaches RC (*release candidate*) stage, the schedule can be found [here]("https://discourse.ubuntu.com/t/jammy-jellyfish-release-schedule/23906"), where you'll note the RC state isn't expected before 14-April-2022 with release scheduled on 21-April-2022 just as the 22.04 signifies (*releases are a Thursday, 3rd or later Thursday of the month*)

*The last release change I'm aware of was [20.04.4 was pushed back a week]("https://wiki.ubuntu.com/FocalFossa/ReleaseSchedule") as mentioned [here]("https://lists.ubuntu.com/archives/ubuntu-devel/2021-December/041738.html").  FYI:  Yes releases can be delayed; eg. the first release in 2006 didn't occur till June, thus it was Ubuntu 6.06 LTS and not 6.04, but that's the only exception; I mentioned 3rd Thursday which allows push back at least one week.*
