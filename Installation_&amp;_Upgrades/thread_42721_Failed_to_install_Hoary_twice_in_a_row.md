### Post by ioliver on 2005-06-19
I have some pretty standard hardware here (KT266 with Athlon 2000) and have previously installed a fair few distributions, including an Array (probably 4) Hoary.  

I've also installed the release Hoary on a couple of machines at work.

So, I was expecting that putting Hoary on another partition of my home machine (to start tracking Breezy) would be easy and smooth.

Wrong! The 1st time I got an error saying that a file couldn't be read from the CD (shipit Hoary final), so I grabbed another CD from the pile, and tried again.

This time it got further, but failed to install "linux-386". Console 3 just said it had failed and didn't tell me anything that hinted why.  I then choose Install base System from the install menu, and it failed again at the same point, but this time gave me a choice of three things to install. The 1st was "linux-386", the 2nd was "Linux-image-386" and the 3rd was the same but with a full kernel version. I chose the 2nd option and this time it worked.

A search of this forum has shown a few people having what seems to be the same problem, but I haven't seen any solutions given.

I decided to get the full wording of the error messages (I accept my woffle above is vague) so did another install completely from scratch with my digital camera in hand. And it installed without a single complaint!

Ideas?

I have a stack of Hoary CDs ready to hand out at work, but am now considering waiting for Breezy in the hope that the install is improved.
