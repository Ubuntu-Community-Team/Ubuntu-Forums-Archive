### Post by trash1464 on 2016-11-03
I have been a happy Ubuntu user for about 10 years. Release 16.04 is another matter. An upgrade from 14.04 did not go well so a clean install, including formatting the system drive, was performed from DVD media. The system has yet to perform up to expectations.

Duplicity, SAMBA and CUPS were not installed by default
System "Internal Errors" occur - intermittent
Total system lockups occur - intermittent
Total system lockups when launching a Windows 8 VM in Virtual box - solid failure
WINE app runs with corrupted fonts - solid failure
Software updater no longer operates - solid failure. Updates must be manually checked for in system settings and the Software center.
Cairo-Dock will not auto start at boot
Virtual desktops vanished and had to be re configured
Thunderbird got corrupted, removal and reinstall did not help
SAMBA client now prompts for a password when connecting to Free NAS guest account where no password is required
SAMBA client does not honor delete keystroke issued against files in the Free NAS CIFS share, rather a right mouse click and selecting delete from the flyout menu

This system ran great under 14.04 without any of the above issues. Am I the only one with a raft of issues with 16.04? Should I nuke this machine again and do another clean install using a newer image? I would rather not go back to 14.04 but the thought has crossed my mind. What would you do?
