### Post by Herman on 2008-01-27
Oh, I see, so that's what he's on about.
koldphlame, terminal's right, we don't have programs in Linux that are 'resource hogs' so there would be nothing for a 'memory cleaner' program to do.

In Windows, many programs are all trying to appear to be the fastest, so they all try to partly load themselves in your RAM, so they will appear to start up quicker, so you think they are faster programs.
In Ubuntu, you will notice that if you open, say, Open Office.org spreadsheet, it takes a minutes or so to get going. That's because it wasn't occupying any space in your RAM. Most programs take a few seconds longer to get started the first time you start them in Ubuntu, but once they are open they work a lot better.
I used to be a trucker and I have a big road map that's so big it used to freeze up if I even opened it in Windows XP, and panning around on it at all made Windows XP thrash and struggle to redraw the map.
In the same computer in Ubuntu I could pan around my huge map as fast as I wanted no problem at all, Ubuntu handles large images with ease. 
I only open my spreadsheets once or twice a week anyway, and I usually leave them open for a while once they are open, so a few seconds of extra time for that doesn't really matter.
Well, now I have a more powerful computer than I used in those days as well.

Is that what you wanted to know about?
