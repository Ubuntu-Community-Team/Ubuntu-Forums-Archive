### Post by YesWeCan on 2011-06-27
Well I installed 11.04 to a 4GB USB stick just last night and it boots fine. So there isn't a universal problem.

Assuming your bios actually tried to boot the USB stick (note some bios menus have entries that are confusing and can look like you have selected USB but don't actually try to boot the stick. Double-check the hard disk boot priority menu and make sure it refers to your specific USB device) then it is unclear whether Grub is broken or Ubuntu got stuck during the boot.

The bootinfoscript output looks ok to me, with the minor niggle that your fstab does not use UUID for /. I am not confident that this is the problem here, but you should replace
/dev/sda1 with UUID=7469c10c-6ca2-4760-b8b8-d5f5805e33ac

I would reinstall Grub again - just for completeness
[COLOR=Navy]sudo mount /dev/sda1 /mnt
sudo grub-install --root-directory=/mnt /dev/sda[/COLOR]


The other niggle is that you are using a 4GB stick with a 1GB swap, leaving only 3GB for 11.04. This is a very tight squeeze. I'd reinstall and forgo the swap altogether unless you are short of RAM. This is what I did.

### Post by YesWeCan on 2011-06-30
Well, it may be due to the way the bios names drives, preventing the first stage of Grub finding the second stage.

Want to try an experiment? Install grub to the Ubuntu partition and boot it from a standard MBR sector. If this works you can re-install Grub to the MBR from within the working Ubuntu. Keep using 10.10 to reduce the possibility of other problems.

Boot from 10.10 live CD.
sudo mount /dev/sda1 /mnt
sudo grub-install --force --root-directory=/mnt /dev/sda1
(it will winge, ignore it)
sudo apt-get install lilo
sudo lilo -M /dev/sda mbr
sudo sfdisk /dev/sda -A1     (to make sure boot flag is set to sda1)

Then try booting the USB stick.

If this works, reinstall Grub while booted
Check the drive name is still sda.
sudo fdisk -l
sudo grub-install /dev/sda  (assuming it is still sda)
