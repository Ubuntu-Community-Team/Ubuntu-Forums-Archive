### Post by oldfred on 2011-10-13
Windows will only UEFI boot with drive formated in gpt not MBR(msdos). You have msdos format.

Ubuntu will boot from gpt or MBR without any issue if you have the correct partition structure. If you were booting in UEFI mode the first partition would be an efi partition. Windows then requires another hidden boot partition.

It looks like you have not gotten grub2's boot loader installed. I always suggest installing it to the MBR of the same drive as the install, so each drive is stand along or can boot if all the other drives fail.

I have not changed to AHCI mode yet but boot from any of my 4 dirves and do not see that the AHCI selection changes that.

#Comments are anything after the #, enter commands in terminal session
#Install MBR from LiveCD, Ubuntu install on sdc5 and want grub2's bootloader in drive sdc's MBR:
#Find linux partition, change sdc5 if not correct:
sudo fdisk -l
#confirm that linux is sdc5
sudo mount /dev/sdc5 /mnt
#If grub 1.99 uses boot not root.
sudo grub-install --boot-directory=/mnt/boot /dev/sdc

Switch BIOS to boot 320GB drive &
# If no errors on previous commands reboot into working system and run this:
sudo update-grub

### Post by Tweak42 on 2011-10-13
I had a similar multi drive setup that I used [Boot-Repair]("https://help.ubuntu.com/community/Boot-Repair") to fix when testing installing 11.10.

Default setup boot order:

1. 7200rpm harddisk - Win7 -> can boot standalone
2. SSD - Ubuntu 11.4 -> Grub2 loader here
3. old ide data drive

Normally my boot order is first to the SSD which can chainload to Win7.  I use ahci and have never had or heard of problems with changing drive boot order.  

For the test I was using a sata drive in a esata dock and booting off install from a ubuntu live USB flash drive.  I set them to 1 and 2 boot positions in the bios:

1. USB flash live 11.10 install
2. external esata drive
3. ...
4. ...

Full install to the esata disk finished ok, but I don't know where the grub loader was installed as the esata drive did not boot.  The grub loader on my SSD was intact, along with Win7 drive.

After reordering and rebooting several times I ended up back in live usb environment and installed [Boot-Repair]("https://help.ubuntu.com/community/Boot-Repair").  Applied grub to the esata drive - rebooted and it worked.  It also detected and added grub entries for Win7 and my 11.04 SSD install.

Note: could have fixed everything via command line but Boot-Repair came up on a search and I want to try it out.
