### Post by Pumalite on 2007-06-26
Let me make this clear; I dont like Vista. I think is the worst yet. But, you want Vista; OK. I would just install to the second IDE. Prepare the drive with Gparted. Make one large partition of the whole drive, formatted ext3. Then, install. Choose 'Guided>Use entire disk'. ( make sure you use the right disk! ) During the installation ( step 5 or 6, right after partitioning ) you get to choose where to install GRUB  ( choose hd0, 0 ). And you are all set. Vista is more of a problem when you want to put Linux ( or another OS ) in the same drive. Then, you have to partition FROM Vista. Don't worry, go ahead; once you get used to Linux, you'll never look back.

### Post by confused57 on 2007-06-26
> **Alarictric said:**
> alright, so i was trying the method where i disconnect my windows HDD then install Ubuntu

The CD boots up, but no matter what option i select (was mainly trying the default option), Ubuntu will not start up.  My computer hangs, the CD spins for a little bit, but my moniter goes into power save mode and my keyboard lights flash.  no HDD activity can be heard
Did you connect the drive you're installing Ubuntu on as master, with the Vista drive disconnected?

If you did and it still didn't work, you could install Ubuntu with both drives connected(Vista master/Ubuntu slave)...if you can set bios to boot first from your slave drive, which "should" be designated as hdb or sdb in the partitioner.  What you can do is set your slave drive as the first hard booted in bios, before you install Ubuntu...install Ubuntu and when you get to the section to install grub, click on the "Advanced" button, then you can enter /dev/sdb as the location to install grub.  With the Vista drive connected during Ubuntu install, an entry should automatically be added to your boot menu to boot Vista.

If you bios isn't capable of booting first to the slave drive, as my Dell Dimension can't, you could probably connect your prospective Ubuntu drive as master(designated as hda or sda) and Vista drive as slave...then you would follow the same procedure as above, but designate /dev/sda as the location to install grub.
