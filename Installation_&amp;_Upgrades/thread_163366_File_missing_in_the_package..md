### Post by mrhelpman on 2006-05-10
Sorry for the bad form by replying to my own post but the solution came to me is a flash :o.

I did an updatedb and then located the libcdda.so file and found that it was not in the /usr/lib/gnome-vfs-2.0/modules as expected. It is in /usr/lib/vfs/modules/. So a quick symbolic link from /usr/lib/gnome-vfs-2.0/modules to /usr/lib/vfs/modules/ fixed the problem. Ok here is what I typed.

$ sudo updatedb
$ locate libcdda.so
/usr/lib/vfs/modules/libcdda.so

$ cd /usr/lib/gnome-vfs-2.0/modules
$ sudo ln -s /usr/lib/vfs/modules/libcdda.so libcdda.so

gedit now runs without error.


Enjoy

JT
