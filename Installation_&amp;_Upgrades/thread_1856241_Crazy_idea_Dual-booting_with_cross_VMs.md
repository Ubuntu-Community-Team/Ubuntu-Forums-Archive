### Post by daltonlaffs on 2011-10-07
So, after a period of time of single-booting Windows 7 x64, I'm thinking about getting things switched back to a dual-boot setup between Ubuntu and Windows.

However, the thing that always annoyed me the most about a dual-boot is the fact that you have to reboot into the other OS completely for even the slightest thing -- if I want to run a very basic Windows program that just happens to not work in Wine, I'm wasting five minutes minimum on the reboot, then another five to switch back. Most of the time, I don't NEED anything complicated from the other OS.

Then, I had a thought: What if I configured both OSes so that they could be run inside VMs from each other?

I could, for example, boot into Windows, but run Ubuntu in a VM without exiting Windows, and vice-versa, running Windows in an Ubuntu VM. Wouldn't this be completely possible? I know there are a lot of VM programs, surely one of them is cross-platform and capable of doing something like this? If I could pull this off, it would solve my biggest annoyance with switching over to using Ubuntu most of the time.

Looking forward to any ideas. :)
