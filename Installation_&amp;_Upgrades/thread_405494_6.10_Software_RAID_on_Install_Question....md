### Post by exodar on 2007-04-09
I have attempted to Install Ubuntu 6.10 Server several times while trying to setup a Software RAID-1 configuration during the "Partition disks" step of the Ubuntu installer and just keep running into the same roadblock.  I have even resorted to trying it in both VMWare Workstation on my Dell running Windows XP, and also in Parallels on my iMac with the same EXACT results!

Here is what I do once I get to the "Partition disks" step:

[LIST=1]
[*]I choose "Manually edit partition table"
[*]I click on "IDE1 master (hda)" and select "Yes" to create a new empty partition table on this device.
[*]I click on the newly created "pri/log 8.4 GB FREE SPACE" under IDE1.
[*]I then enter 8.4 GB as the size and click Continue.
[*]I select "Primary" as the partition type.
[*]I change the "Use as" option to "physical volume for RAID".
[*]I set the "Bootable flag" to on and click on "Done setting up the partition"
[*]I click on "IDE2 master (hdc)" and select "Yes" to create a new empty partition table on this device.
[*]I click on the newly created "pri/log 8.4 GB FREE SPACE" under IDE2.
[*]I then enter 8.4 GB as the size and click Continue.
[*]I select "Primary" as the partition type.
[*]I change the "Use as" option to "physical volume for RAID".
[*]I set the "Bootable flag" to on and click on "Done setting up the partition"
[/LIST]

At this point I have the following in my Partition disks list:
[FONT="Courier New"]
IDE1 master (hda) - 8.4 GB Virtual HDD [0]
[INDENT]#1 primary    8.4 GB   K raid[/INDENT]
IDE2 master (hdc) - 8.4 GB Virtual HDD [2]
[INDENT]#1 primary    8.4 GB   K raid[/INDENT]
[/FONT]

Now I am ready to configure my software RAID so I select "Configure software RAID" from the "Partition disks" screen and proceed:

[LIST=2]
[*]I select "Yes" when it prompts me to write the changes to the storage devices and configure RAID.
[*]I select "Create MD device".
[*]I select RAID1.
[*]I enter "2" for the number of active devices for the RAID1 array.
[*]I enter "0" for the number of spare devices for the RAID1 array.
[*]I select /dev/hda1 and /dev/hdc1 as the Active devices for the RAID1 multidisk device.
[*]I select Finish.
[/LIST]

Now I have the following in my Partition disks list:
[FONT="Courier New"]
IDE1 master (hda) - 8.4 GB Virtual HDD [0]
[INDENT]#1 primary    8.4 GB   K raid[/INDENT]
IDE2 master (hdc) - 8.4 GB Virtual HDD [2]
[INDENT]#1 primary    8.4 GB   K raid[/INDENT]
RAID1 device #0 - 8.4 GB Software RAID device
[INDENT]#1   8.4 GB     ext3[/INDENT]
[/FONT]

Now I am ready to setup a SWAP partition because I am under the impression that I should have one.  So I select "#1 8.4 GB ext3" under my RAID1 device and select "Delete the partition".  Then I select the "8.4 GB FREE SPACE" under my RAID1 device and select "Automatically partition the free space" and I get my error:

[COLOR="Red"]Failed to partition the selected disk
This probably happened because there are too many (primary) partitions in the partition table.[/COLOR]

When I am returned to the Partition disks screen I see the following:
[FONT="Courier New"]
IDE1 master (hda) - 8.4 GB Virtual HDD [0]
[INDENT]#1 primary    8.4 GB   K raid[/INDENT]
IDE2 master (hdc) - 8.4 GB Virtual HDD [2]
[INDENT]#1 primary    8.4 GB   K raid[/INDENT]
RAID1 device #0 - 8.4 GB Software RAID device
[INDENT]#1   8.0 GB   f ext3       /
380.4 MB     unusable[/INDENT]
[/FONT]

I try to change the 380.4MB unusable space to SWAP and it just asks me "How to use this free space:" with "Show Cylinder/Head/Sector information" as an option.  I have also tried starting completely over and setting up the IDE1 and IDE2 partitions as logical instead of primary and I get the same exact results.

I am at my wits end with this.  If someone knows the correct process to follow to setup a RAID1 Software RAID setup while Installing Ubuntu Server 6.10 please tell me exactly how you did it.  I contemplated just not having a SWAP partition, but that just seems bad to me.
