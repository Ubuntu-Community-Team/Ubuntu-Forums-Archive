### Post by Deathmoon on 2008-03-21
I have a dual boot system with 3 drives.  One drive is /home, second drive is /swap and / and the third drive is Windows.

If I go into the BIOS and disable the Windows drive, all is fine.  However, once it is enabled and I try to boot into Ubuntu I get the following message.

bad magic number in super-block....
fsck died with exit status 8
states that this happens on the following device:  /dev/sdb1/
if I type "exit" at the terminal it then will go to the Ubuntu login screen, but when I try to login, it says it cannot find /<username>/home.  (I think I know why see below)...

When I disable the drive (Windows) in BIOS Ubuntu works fine.  In Ubuntu I see that /dev/sdb1 is my /home drive therefore that's the cause of the error above.  It seems that my Windows drive is being placed in sdb1 slot.

How do I go about making it where I can turn on the Windows drive in the BIOS and not have the above issues?

I've done this before but just reformatted my machine and now I'm running into this issue.

Thanks in advance.

### Post by Deathmoon on 2008-03-21
Solved.
This wasn't that bad, just took a lot more digging.

First things first...
backup you fstab file

sudo /etc/fstab /etc/fstab.backup

sudo gedit /etc/fstab

Here's what mine looked like...
```
# /etc/fstab: static file system information.
#
#  -- This file has been automaticly generated by ntfs-config -- 
#
# <file system> <mount point>   <type>  <options>       <dump>  <pass>

proc /proc proc defaults 0 0
# Entry for /dev/sda1 :
UUID=42c61a68-e970-4571-84e0-cff1fa631b5d / ext3 defaults,errors=remo
I unt-ro 0 1
# Entry for /dev/sda5 :
UUID=4323bcf5-ea88-4959-bb1b-f3f2fde589c7 none swap sw 0 0
/dev/scd0 /media/cdrom0 udf,iso9660 user,noauto,exec 0 0
# Entry for /dev/sdb1 :
/dev/sdb1 /home ext3 nodev,nosuid 0 2
```

I then ran blkid, which returned:

```
/dev/sda1: UUID="42c61a68-e970-4571-84e0-cff1fa631b5d" SEC_TYPE="ext2" TYPE="ext3" 
/dev/sda5: TYPE="swap" UUID="4323bcf5-ea88-4959-bb1b-f3f2fde589c7" 
/dev/sdb1: UUID="62BC042DBC03FA75" TYPE="ntfs" 
/dev/sdc1: LABEL="home" UUID="82f44f95-9956-4b7d-88e0-4241b18ce964" SEC_TYPE="ext2" TYPE="ext3" 
/dev/sdh1: UUID="A030DDF930DDD5FE" LABEL="NTFS" TYPE="ntfs" 

```

I took the /dev/sdb1 UUID and placed it into my fstab, which now looks like (just showing last 3 lines here):

```
# Entry for /dev/sdb1 :
# /dev/sdb1 /home ext3 nodev,nosuid 0 2
UUID=82f44f95-9956-4b7d-88e0-4241b18ce964 /home ext3 nodev,nosuid 0 2
```

rebooted, everything worked.  rebooted turned on Windows drive in BIOS.  I'm able to access the Windows drive in Ubuntu and boot to Ubuntu or Windows (via changes to my menu.lst file).
