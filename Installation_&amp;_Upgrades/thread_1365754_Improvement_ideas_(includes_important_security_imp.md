### Post by mcgiwer on 2009-12-27
Here are my ideas:
[LIST=1]
[*] Seperate run/instalation modes
[*][SECURITY] Minimal services
[*][SECURITY] Minimal users & groups
[*][SECURITY] Attack prevention
[/LIST]

ad. 1:

Add following run/install modes:

[LIST]
[*]Minimal - minimal system (without any additional software, etc.) with GUI
[*] Normal - actual running/install mode
[*] Virtualization - GUI (without any additional software) + enviroment (kernel and library's) for running virtual machines
[/LIST]

ad. 2:

_**SECURITY IMPROVEMENT**_

By running or installing Ubuntu there should be run/installed only services with are _absolute required_ for running of Ubuntu. None else services should be run/installed, because they are creating an security risk

ad. 3:

_**SECURITY IMPROVEMENT**_

In running/installed Ubuntu should be only _absolute nessecary_ users and groups with are required for proper Ubuntu work. Any additionally users/groups are creating an security risk.

ad. 4:

_**SECURITY IMPROVEMENT**_

Ubuntu _**should have installed and running**_ an firewall and IDS. Additionally there should be an possibility to manage (add, remove, etc.) ports and protocols.

Note: *any unused and/or risky ports and protocols should be default closed and blocked*
