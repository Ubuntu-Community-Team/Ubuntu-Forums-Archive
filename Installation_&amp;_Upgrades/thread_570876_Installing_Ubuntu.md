### Post by Shazaam on 2007-10-08
What I would do is install some kind of virtualization app to Windows then install any flavor of linux to the program. Play around, tweak it, trash it, and get comfortable with it. THEN take the plunge, wipe the Windows drive, install linux and be done. Linux isn't hard it's just different. Some people find after a while that linux isn't for them and try to go back to Windows causing all kinds of havok. So if you do virtualization all you have to do is delete the linux files with no harm to Windows.

Some things to remember if you are going to run a virtual linux environment..
1. Some hardware that runs while virtualized won't work in a real installation.
2.3d support SUCKS in ALL virtual apps so no REAL gaming or fancy desktop effects like Beryl.
3.The only change you do to your REAL hard drives is add FILES to them. NO (zero) changes to your partitions (except take up space).
As far as a real installation you can do what you want. Dual (or multi in my case) boot, stand alone install, whatever. Ubuntu has an excelent installer, just pay attention when it comes time to install, and if you don't understand you can always abort before you commit (before partitioning).
