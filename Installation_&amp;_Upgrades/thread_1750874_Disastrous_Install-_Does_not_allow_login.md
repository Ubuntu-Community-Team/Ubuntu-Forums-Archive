### Post by Linux BASHer on 2011-05-06
It's been a disaster upgrading xubuntu. I was running 10.10 on my Asus 1001P-MU17 netbook and tried to upgrade to 11.04 using the simple "upgrade" button in the update manager. I should've known it couldn't be that simple. After giving it tons of time to upgrade, the final result was the computer looping back to the login screen after logging in. I logged in via a console. It recognized I was logged in, but still no dice. I'm a geek, but not one with enough time to start troubleshooting. I have made some modifications to fix some issues on my system (brightness, hotkeys, etc.). I figure these were fairly minor and would probably just be overwritten anyway.

No problem... I just wiped the newly defective upgrade and installed a new version via SD card. This, of course, took quite a while in itself, as the only way I've figured of making a bootable card on my Mac is by downloading the disc image, installing it on my virtual machine and making the bootable card from the startup disc creator there. Now I finally have a clean, virgin install of xubuntu on my netbook. I try to connect to my wireless network. GUI crash. I try to get a console screen. No response. <sigh> So I reboot. And now I'm having the same problem as before.. I try to log in, but after the login process initiates, I briefly see a black verbose text screen and it goes back to the login screen, wash, rinse, repeat.

I gotta say, ever since the beginning I've had some troubles on my machine. It looks like the Pinetrail machines were never 100% supported out-of-the-box, which is not a huge deal- But these post virgin-install issues (which I've never had before) are really irking me. I had trouble precisely with the wireless drivers before, which eventually a driver wrapper and then a backport fixed, but it never crashed on me, nor did it prevent me from normally booting in via the GUI.
