### Post by meierfra on 2007-12-22
The order "fdisk -l" lists the drive can be different from the order grub sees the drives at boot up.
At boot up, grub sees the drives in the same order as they appear in the bios.

Since you are able to boot into ubuntu, and ubuntu is listed as "root (hd0,1)" in menu.lst, we know for sure that (hd0) is the  sdb.

This means that sda is either (hd1) or (hd2).  Since (hd2,0) did not work,  I suggest to try:

title Microsoft Windows XP Professional
root (hd1,0)
savedefault
makeactive
map (hd0) (hd1)
map (hd1) (hd0)
chainloader +1

If this did not work you can also try

title Microsoft Windows XP Professional
root (hd2,0)
savedefault
makeactive
map (hd0) (hd2)
map (hd2) (hd0)
chainloader +1
but for this   to work you need to change the boot order in the bios: ubuntu drive first, 250 GB second, windows drive third.
