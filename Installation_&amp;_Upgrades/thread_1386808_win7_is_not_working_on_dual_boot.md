You see the filesystem of your windows partitions is SFS, not NTFS. It seems it's some kind of encrypted filesystem. I don't have any experience with it, just what I read on google briefly. I guess the resize that ubuntu installer did somehow messed up the boot files, hence the blue screen.
You should usually resize windows partition with the windows disk management if possible. If it was ntfs, it would probably work like this too, but on SFS who knows.

Also, you installed ubuntu only with root partition and without swap. While this is not a great problem, usually you would also have swap partition too. I guess you creted it manually but you should have read more information before starting, and create swap partition too.

See if you can restore your win7 to factory state by a restore partition or restore discs. And get more informed about this SFS filesystem.

### Post by phillw on 2010-01-21
+1

For resizing Win partitions, head over here --> [Resize a Partition using Vista / Win7]("http://www.howtogeek.com/howto/windows-vista/resize-a-partition-for-free-in-windows-vista/")

Also noted, you have used your last primary partition up to install the root of Ubuntu - If you use the instructions on the above link, leave the space freed up (10 GB is a fair minimum) as 'free' space - i.e. do not format it in any way. Using the LiveCD select 'use free space' and you should get your root & swap installed for you. (Typical swap = 1GB, or equal to RAM, whichever is greater).
