> Just getting going with installing Ubuntu Core and wanted to ask if I should use DHCP or a static IP?
Servers should always have static IPs. Always. I haven't any clue how to configure it on Ubuntu Core, but on regular Ubuntu 20.04, the config would be in /etc/netplan/.  While it is best to have static IPs configured ON-THE-DEVICE, for some devices like portable or streaming sticks, using DHCP Reserved IPs is a viable option. There are some security risks in using DHCP which were posted last month by security researchers, but if you are otherwise cautious with your home router and keep it properly maintained/patched, those risks can be minimized.  The key with DHCP reservations is to place all reserved IPs OUTSIDE the DHCP range for casual devices.  If your DHCP range is .100-.200, then make your reserved DHCP devices from .20 - .99 and place manually, locally, configured static devices (anything that isn't going to move) from .2 - .19. Those are just examples. 

I prefer to restrict my DHCP range to be 3x the number of people visiting.  Our phones and laptops use DHCP reservations on the guest network (different from the wired network or server network).

Run your network however you like.  If you plan to allow Nextcloud to be available over the internet, please, please, please, only allow access after you have a VPN or ssh-SOCKS5 proxy connection. Wireguard [https://www.linuxbabe.com/ubuntu/wireguard-vpn-server-ubuntu](https://www.linuxbabe.com/ubuntu/wireguard-vpn-server-ubuntu) is pretty easy to setup and runs on all the OSes.  Don't put NC naked on the internet.  HTTPS alone isn't any real security.  [https://blog.jdpfu.com/2021/10/01/tls-in-2021](https://blog.jdpfu.com/2021/10/01/tls-in-2021) links to a YT talk from a security conference.

in the format of xxx.xxx.xxx.xxx/xx... Let my try to make a simplified explanation with examples... And compare it to what TheFu tried to explain...

Old IPv4 address formats, for a private address, used to be in the format like this example:
```

Address: 168.192.0.5
Subnet Mask: 255.255.0.0
Gateway: 168.192.0.1

```
In this example, the NetworkID is 168.192.0.0, the NetMask translates to the newer mask notation of /16, so would be noted as 168.192.0.5/16... meaning the first 16 bits are the network address, and the remaining bits are addressable clients. I doubt that I need to get inot explaining subnets and subnet masks, or you would be asking questions 'differently."

Next example is another common reserved private LAN Address  
```

Address: 10.0.0.5
Subnet Mask: 255.0.0.0
Gateway: 10.0.0.1

```
Where the address is translated to the newer bit notation as 10.0.0.5/8... With a NetworkID of 10.0.0.0/8
