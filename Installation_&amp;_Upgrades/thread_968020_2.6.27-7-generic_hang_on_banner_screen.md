The following work around seems to work.
After rebooting, I press Esc when grub starts to load.  This displays a menu of kernel versions.  Then I highlight 2.6.24.21-generic(the third item on the menu).  The system seems to run fine with 2.6.24-21-generic.

I'm looking for tips on how to find out why the system appears to hang while starting up 2.6.27-7-generic.

### Post by taurus on 2008-11-02
You need to see the output when it boots to see what causes it to hang.  While in GRUB menu (after pressing ESC), highlight the first kernel, 2.6.27-7-generic, and press e to edit it.  Move to the end and remove the last two options, quiet & splash.  Then, boot it with b and see which process causes it to hang.
