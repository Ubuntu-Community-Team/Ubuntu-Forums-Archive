### Post by yancek on 2016-02-12
Post 13 in your other post shows images of 3 hard drives.  sdc is an 8GB flash drive so I expect that is not what you are referring to.  sda is a 186GB drive and your root (/) filesystem partition is 185GB+ and you have a 1GB swap.  If this is the one you are trying to enlarge the swap on, that won't work because swap is in the Extended partition and you have already used all the space in the Extended.  You need to use GParted from a Live CD/flash drive to first turn swap off, then delete it and the Extended partition and resize (shrink) the root filesystem in order to create more space for swap.

sdb also has the swap partition in the Extended partition and is using all the space in the Extended although this appears to be empty as no part is shown as "Used".

You can use GParted from an installed system as long as you do not try to make any changes to the "/" root filesystem.  Usually a lot simpler to use GParted from a bootable medium and it is on the Ubuntu installation disk or flash drive.   Your partitioning scheme is a little weird, usually a 25GB partition for / is more than enough and you can then use the rest for data.

### Post by Bucky Ball on 2016-02-12
> **yancek said:**
> There are no set standards.

+1. 20Gb is example only, but if you have a separate /home, then a place to start. If /home is next to /, then if you have a 20Gb / and your install only comes to 8Gb, you could boot to a live session, shrink / by 5Gb say, then expand /home into the free space. 

* Important note: ALWAYS leave headroom. i.e. if your install is 8Gb, don't shrink the partition to 8Gb. That will cause chaos. Always leave 4 or 5Gb headroom (if not a little more).

I use 15Gb for / generally and the install rarely gets over 10Gb (I've just done a new install on my new desktop of xubuntu-core, added most the apps I will be using on that box, and the install swings in under 5Gb).
