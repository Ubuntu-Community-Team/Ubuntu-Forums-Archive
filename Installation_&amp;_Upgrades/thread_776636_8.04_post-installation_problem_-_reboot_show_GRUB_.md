### Post by Herman on 2008-05-02
I'm not sure exactly what's wrong yet, or how long it will take to solve this problem.

If you want to be able to boot while we're working on it, please download a copy of [Super Grub Disk]("http://www.supergrubdisk.org/") and burn it to a CD, or make a floppy disk out of it or a USB, depending on what file you decide to download.

Another boot CD that's very good, especially for Windows is the one you can get from the following web page, [How to fix: NTLDR is missing, press any key to restart]("http://tinyempire.com/notes/ntldrismissing.htm").[U]

[/U]According to the [GNU Grub Manual]("http://www.gnu.org/software/grub/manual/grub.html"), Error 5 means:
```
5 : Partition table invalid or corruptThis error is returned if the sanity checks on the integrity of the partition table fail. This is a bad sign.       
```However, I can't see anything wrong with your partition table in your fdisk output there.

You could try adding a boot flag to any partition in your second hard drive, you can do that with GRUB's 'makeactive' command, from [GRUB's Command Line Interface]("http://users.bigpond.net.au/hermanzone/p15.htm#cli"), or any partition editor. I doubt if that's the problem though.[U]


[/U]
