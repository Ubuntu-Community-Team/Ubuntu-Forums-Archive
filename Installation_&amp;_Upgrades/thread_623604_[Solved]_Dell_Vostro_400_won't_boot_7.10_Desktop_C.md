### Post by treblemaker on 2007-11-26
I recently purchased a Dell Vostro 400.  Whenever I tried to boot the Ubuntu Desktop CD it could not find and mount the root file system.   The "Alternative" CD booted, but then I did not have the tools I needed (gparted) to resize the ntfs partition and add the new disk partitions for Ubuntu.  Very frustrating!

Simple solution:  Update the BIOS!  Now 7.10 Desktop boots just fine!

The system shipped with BIOS version 1.03.  The latest version is 1.08.   Unfortunately I could not get the BIOS update to run under DOS, so I had to pay the Microsoft tax (boot into windows and accept the license agreement) in order to update the BIOS.

The BIOS update can be downloaded from support.dell.com.

I hope this helps someone else to avoid the frustration I've had for the past week!
