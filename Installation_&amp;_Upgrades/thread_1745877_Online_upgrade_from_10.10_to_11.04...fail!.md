### Post by Hedgehog1 on 2011-05-01
Thanks for posting the script output!

Using the **_Natty/11.04_** LiveCD/LiveUSB:

```
sudo mount /dev/sda1 /mnt
```

```
gksudo gedit /mnt/etc/fstab
```

```
# /etc/fstab: static file system information.
#
# Use 'blkid -o value -s UUID' to print the universally unique identifier
# for a device; this may be used with UUID= as a more robust way to name
# devices that works even if disks are added and removed. See fstab(5).
#
# <file system> <mount point> <type> <options> <dump> <pass>
proc /proc proc nodev,noexec,nosuid 0 0
[COLOR="Red"][B]#/dev/sda1 / ext4 errors=remount-ro 0 1
UUID=c9b1d699-dfd2-4053-9b07-e98403c6b973 / ext4 errors=remount-ro 0 1
UUID=9210413c-c510-4fc4-8c4d-d9bfd181bdcd /home ext4 defaults 0 2[/B][/COLOR]
# swap was on /dev/sda5 during installation
UUID=f7f45244-48da-4c6f-81e6-0c0e2afcfad6 none swap sw 0 0
```

If you have not already copied your data from the '/home' partiton in /dev/sda1, do this:

```
gksudo nautilus
```

Select the 'show hidden files' and then copy /dev/sda1/home to /dev/sda4/home.

Clear all data from the old /dev/sda1/home (empty /home directory)

Lastly:

```
sudo mount /dev/sda1 /mnt
```

```
sudo grub-install --root-directory=/mnt /dev/sda
```

Then reboot.  If everything works, great!

If not, you will need to do a clean install, select the 'something else' option, and mark /dev/sda1 as the '/' with a check box to format /dev/sda1, and mark /dev/sda4 as the '/home' AND DO NOT CHECK MARK THE FORMAT BOX for /dev/sda4.
