### Post by PrinceVince on 2017-04-20
I'm trying to run the Xubuntu **17.04** LiveCD from my USB Stick but am faced with the following error:
```
[5.271295] BERT: Can't request iomem region <00000000ba8dec98-00000000ba8decab>
```
The screen also runs in a very low-res mode and shows graphical artifacts, so I can't see anything beyond this first line. I md5sum'd the iso before writing it onto the stick, though I cannot run the self-test from the LiveCD as it too halts with above problem.

I'm currently running **16.10** on this system without any problem, so I'm wondering if it's some kernel regression. My hardware is a fairly regular Desktop with Intel Core i7, Nvidia GTX970.

Of course, I cannot really know whether the shown message is actually involved in the boot coming to a halt, it might just be the thing printed near the moment of the error. Any suggestions?

**Update:** I think my problem is unrelated to the message. I added the **bert_disable** kernel parameter and the message disappeared to no change. **nomodeset** allowed me to boot (with issues), so it's probably graphics card related.
