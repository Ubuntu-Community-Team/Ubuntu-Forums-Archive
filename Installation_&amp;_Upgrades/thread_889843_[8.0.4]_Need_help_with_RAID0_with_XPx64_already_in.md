### Post by ebalter on 2008-08-14
[FONT="Arial"]Following the instructions found on [http://advosys.ca/viewpoints/2007/04/setting-up-software-raid-in-ubuntu-server/](http://advosys.ca/viewpoints/2007/04/setting-up-software-raid-in-ubuntu-server/) , (creating a RAID0 instead of RAID1). I have created a software RAID0 on Ubuntu Server 8.0.4 to mount **/home** only (I first tried to create more than one MD device, but it didn't work; instead of looking for the failure cause, I decide to create only one MD device to store /home). Following are the changed directions (as I performed):

* To install a fresh Ubuntu System with RAID, boot from the CD-ROM as usual. Follow the prompts until you get at the “Partition Disks” dialog.

* From the “Partitions Disks” dialog box, select “Manually edit the partition table”.
* Select the first disk (”sda”)
* Say yes to “Create a new empty partition table on this device?”
* Use the dialog boxes to create one primary partition for /boot and logical partitions for / (root), /tmp, swap, and the one you will use for RAID (in my case, sda7)
* For “How to use this partition” select “physical volume for RAID“, not the default “Ext3 journaling file system” for sda7
* Repeat the above steps to create one identical "sdb7" partition and one identical swap partition (sdb3, in my case) on the second drive; I have created identical partitions sdb1, sdb2, sdb4, sdb5 and sdb6 but marked them with as "do not use" or to mount on special mountpoints (for future use). 

* Once the partitions are configured, at the top of the “Partition Disks” main dialog select “Configure Software RAID”
* When asked “Write the changes to the storage devices and configure RAID” select “Yes”.
* For “Multidisk configuration actions” select “Create MD device”
* For “Multidisk device type” select "RAID0"
* For “Number of active devices for the RAID0 array” enter "2"
* When asked to select “Active devices for the RAID1 multidisk device” select both /dev/sda7 and /dev/sdb7
* From the next dialog select “create MD device”
* Finally, from the dialog “Mulidisk configuration actions” select “Finish”

It looks like dmraid is not recognizing the array and activating it.  Post the output of dmraid -ay -vvvv -dddd.

As for why sdb has no partition table, that is because it doesn't; the first sector on sdb contains the first sector of the second half of the first stripe of the raid set.
