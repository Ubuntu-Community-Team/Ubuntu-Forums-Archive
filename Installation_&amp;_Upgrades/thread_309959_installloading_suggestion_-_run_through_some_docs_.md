### Post by djlosch on 2006-11-30
We were discussing the sudo vs classic su models on the gentoo forums after a gentoo user's brother nerfed his ubuntu install and went back to windows.  Basically, the kid didn't know what was going on with sudo and destroyed his own system.  It's a well known fact of life that almost no one reads documentation for anything... furniture assembly, power tools, and linux is no exception.

Older versions of redhat and suse used to display various informative blurbs about OO.o, samba, and other projects while the install progress bar went across the bottom.  As an avid ubuntu user of almost 2 years now, I think it would be advantageous to the community to have a blurb informing people about the sudo model during the install.  

Further, World of Warcraft has the tips system that shows on every loading screen.  So, build up a cache of 1 sentence tips and have them display on the splash screens for ubuntu boot, g/kdm login, and gnome/kde loads.  The cache could possibly be wiki driven (require x number of posts on ubuntuforums to contribute, so that it cuts down on spam)... maybe even make a distinction for ubuntu tips in general, and then those that would be specific to gubuntu, kubuntu, edubuntu, etc.

The tips would just be displayed, rather than requiring interaction, so that new users could learn features better without slowing down advanced users.  A lot of tips systems have some popup when the program loads, and that's just too invasive.    It slows down advanced users, and everyone disables them because they're so invasive.  Ultimately, the tips would be able to be disabled in some options menu, but because they're not invasive, most people would leave them on.

Just an idea.
