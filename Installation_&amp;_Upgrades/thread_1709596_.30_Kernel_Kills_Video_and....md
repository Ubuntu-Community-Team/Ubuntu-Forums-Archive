### Post by JKyleOKC on 2011-03-18
The nVidia drivers require kernel drivers that match the version of the kernel image; you should get video back by getting to the grub menu at boot time (press the left shift key immediately after turning on the power and keep pressing it until the menu appears; it may take several tries), and selecting the third line rather than the first.

Then open the Synaptic package manager from the System menu, and search for "linux-headers" which will get you a list of all the header packages. Those which are installed will have their checkboxes filled in green, while the others will have empty checkboxes. Locate the one that matches your .30 kernel and check its box. This gets you a popup menu, select "mark for installation." You should also select the one that simply says "linux-headers-generic" which will assure that you get the headers automatically on future updates that need them.

While in Synaptic, search for "dkms" and if it is not already installed, select it for installation also. This is the Dynamic Kernel Module Support package that will take care of creating the necessary kernel modules automagically.

Finally, click the Apply button and follow the instructions. This will download all necessary files, install them, and DKMS will then create and install the needed modules for you. You should then find the GUI working as before when you reboot.

I don't know whether this will help the netbook or not, though. If you have enough disk space, it's worth a try -- and you can gain some space by removing older kernels via Synaptic. Always keep the most recent two, however, so that you can fall back if something like this happens again!

### Post by JKyleOKC on 2011-03-20
You can try adding "nomodeset" to the boot options and see whether that makes any difference. Recent kernels have taken over some of the video driver capabilities, and that's not been fully compatible with all of the proprietary drivers. The nomodeset option disables those kernel changes at boot time.

To test the change, get to the grub menu, highlight the kernel you want to use, and press "e" instead of Enter. This will display the full entry in an editor. Scroll down to the line that begins "linux" and move to the end of the line, where the words "quiet" and "splash" appear. Add "nomodeset" between the two words, separated by spaces on either side, and press CTRL-X to resume the boot action.

If this clears things up, you can then edit the /etc/default/grub file (with root privlege) to make the change permanent. After saving the modified file, run "sudo update-grub" to modify the grub.cfg file with your changes.

This may make even .29 and .30 work properly again. However I'm not confident right now that .30 is fully ready for prime time, as I'm having some network card problems with it and am currently on .29 myself (where the cards are working properly)...

### Post by JKyleOKC on 2011-03-21
Nope, the lines are right there: ```
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"
GRUB_CMDLINE_LINUX=" vga=792 splash"

```
Make them look like this:```
GRUB_CMDLINE_LINUX_DEFAULT="quiet [COLOR="Red"]nomodeset[/COLOR] splash"
GRUB_CMDLINE_LINUX=" vga=792 [COLOR="Red"]nomodeset[/COLOR] splash"

```

Then save the file, run "sudo update-grub" and you should be home free.

Note that you'll have to do the editing with "gksudo gedit" to be able to save your changes.
