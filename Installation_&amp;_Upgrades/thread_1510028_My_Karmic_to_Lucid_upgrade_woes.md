### Post by JimmyJimJames on 2010-06-15
I'm an experienced Linux user who decided to upgrade one of my stable dual-boot XP/Karmic machines to XP/Lucid.  Let's just say that I'm not too thrilled with 10.04 at this point and am holding off updating my other Ubuntu machines.

- My computer will not properly power off 95% + of the time from the GUI.  If I select "Shut Down...." it instead just logs me off and pops me back to the graphical login screen.  Only two or three times since the Lucid upgrade has it actually shut the machine down.  ACPI stuff seems to be fine, and the machine shuts down properly when I issue "shutdown -h now" from a su command line.  I read some of the other threads on here, but couldn't find one that seemed to match my behavior that had a solution which applied to me.

- I can no longer mount my windows hard drive partition from the GUI "places" menu as I'm "Not Authorized".  9.10 used to just prompt me for a password.  I assume I'll probably have a problem with flash drives too.

- My SATA DVD-R drive still doesn't work properly with Ubuntu.  Putting  in an audio CD continues to crash something, but this problem existed  with 9.10 too.  I've been following the bug tracking on this problem,  but no solution yet.

- My Nvidia FX5200 <-DVI-> Acer AL2216W combination still won't detect and operate at the correct resolution.  This forces me to manually craft a xorg.conf file with my monitor's modes, which works, but causes a pop-up error on the right top of the screen at startup saying something about unsupported mode (or something like that). At least the resolution is correct and everything works well with my hand crafted xorg.conf file.

- Oddly, control-Alt F1-F4 no longer seem to work properly.  It swaps me away from my graphical mode session, but my monitor goes blank displaying "Input not supported".  I don't recall messing with anything that would cause that, and don't see anything odd with my grub setup, but who knows....

### Post by cariboo on 2010-06-16
It sounds like you upgrade didn't finish properly. Try:

```
sudo apt-get -f install
```

To make sure all dependencies are installed then:

```
sudo aptitude update && sudo aptitude safe-upgrade
```

To make sure everything is up-to-date.

### Post by dino99 on 2010-06-16
i've had quite the same issues few times ago with my system, now its still not perfect but more better:

into bios: select ahci mode and take care of devices boot order

look how hardware is plugged on the board (hdd(s), dvd order)

about mounting: install mountmanager to deal with devices and partitions rights

you might clean the system with gconf-cleaner (yes to all) and bleachbit

actual kernel(s) directly deals with X, so remove xorg.conf:

sudo rm -f /etc/X11/xorg.conf

be sure that everything about grub (grub-legacy) and menu.lst is removed (only grub-pc and grub-common installed)

add this ppa to synaptic repo:

[https://launchpad.net/~ubuntu-x-swat/+archive/x-updates](https://launchpad.net/~ubuntu-x-swat/+archive/x-updates)

update, upgrade and reboot, then check that nvidia-current is activated (system admin hardware driver)

sudo apt-get install -f
sudo dpkg --configure -a
