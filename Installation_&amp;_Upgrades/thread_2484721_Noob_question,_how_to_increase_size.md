### Post by TheFu on 2023-03-09
> **nalitnyk said:**
> So I've installed Ubuntu to 45GB new partition. But now it says that I have small amount of size, what to do? I want to use full size (maybe in the root dir)

Please don't post huge images, especially when they can't be displayed in a browser.
45G is a good size for the OS partition.  You should add another partition, format it with ext4 file system, mount it temporarily to /mnt/home/ and migrate all the data in /home/ over to that new partition, being careful to maintain owner, group, permissions, ACLs and xattrs in that migration.  rsync can do that, but so can your backup/restore tool.  If you move the files, you'll get everything off the OS partition.  If you copy the files, you'll need to remember to remove them later. Modify the /etc/fstab to mount the new partition+file system onto /home/.

You are almost done.

Now if you copied all the data, it is time to remove it from the OS partition.  There are lots of ways to do that. Perhaps the safest is to boot from the Ubuntu Installation flash drive (or you can boot into recovery mode from the Advanced Grub screen).  If the updates to the /etc/fstab are correct, you can run **mount -a** in recovery mode which will mount the new partition so you can verify all is good.

Do a last reboot and all the data from the OS that was under /home before will now be in a new partition.

As for sizing for the new partition, if this storage device is an SSD, I'd leave 20% completely unused to increase the lifespan of the SSD (more unused blocks means they can be logically swapped in as needed by the SSD controller).  Besides that, if you are doing simple partitioning, use the rest.  Say you have a 500GB SSD, then 80% of that is 400GB, so if you are already using 45G for the OS, that leave 350G for use by the new /home/ partition+file system.  If you want to be really kewl, you could create a 4.1GB swap partition too. Ask if you are interested in that. New Ubuntu installs setup a swap file in /, which is a fairly new method on Linux. Not everything that a swap partition can do is supported, but for 99% of users, it shouldn't matter.  

If you'll run these commands and post the results, maybe we and provide more detailed advice.
```
lsblk -e 7 -o name,size,type,fstype,label,mountpoint
df -hT -x squashfs -x tmpfs -x devtmpfs
```

I'd like to show you how this all looks after you are done, but I don't use simple partitions.
