### Post by revol on 2008-02-22
I found the workaround!! It works fine!!:-D
Little guide to install ati fglrx 8.2 with real time kernel in gutsy:
Using a mix between [[COLOR="DarkOrange"]this guide[/COLOR]]("http://wiki.cchtml.com/index.php/Ubuntu_Gutsy_Installation_Guide") (method 2) and [[COLOR="#ff8c00"]this post[/COLOR]]("http://64studio.com/node/150#comment-1148") on Studio64 forum...I'll simply copy&paste:
> [edit]Method 2: Install the Catalyst 8.2 Driver Manually
This is just an alternative installation method for the section above. It might help if you still get 'DRI missing' errors. 
Note: The newest 8.455.2 does not enable DRI using gutsy. It needs a more recent X.org for this, but 2D does work. "(II) fglrx(0): driver needs X.org 7.1.x.y with x.y >= 0.0" 
Note: If you are running the -rt kernel, you will fail to compile the kernel module with "FATAL: modpost: GPL-incompatible module fglrx.ko uses GPL-only symbol '__rcu_read_lock'". (Follow these instructions for a workaround: [http://64studio.com/node/150#comment-1148](http://64studio.com/node/150#comment-1148)) 
Download the ATI driver installer: ati-driver-installer-8-02-x86.x86_64.run (this installer is for 32bit and 64bit systems) 
Change to the download directory. Make sure that you have the universe and multiverse repositories enabled in /etc/apt/sources.list before doing these steps. 
There is a detailed manual with screenshots at Ubuntu Wiki. 
By default, Ubuntu did not enable the Universe and Multiverse repositories, but now in Gutsy, both Universe and Multiverse are activated by default. 
Install necessary tools: 
sudo apt-get update
sudo apt-get install build-essential fakeroot dh-make debhelper debconf libstdc++5 dkms

Uninstall previous fglrx: Using Synaptic, completely remove any packages containing "fglrx" in their name.

Then the second post:
> I had to modify two lines of code in the driver source to set the license to GPL rather than proprietary. I don't know the legal status of this, you're on your own but I guess as long as you don't redistribute you'll be alright. Don't expect help or sympathy from 64 Studio support or anybody else for that matter. This is a get-it-working solution.
I unpacked the drivers with:
./ati-driver-installer-8.42.3-x86.x86_64.run --extract fglrx
Then in the fglrx/common/lib/modules/fglrx/build_mod directory there are a few files, you need to modify two of them:
drm_compat.h
 firegl_public.c
In drm_compat.h you need to change:
#define MODULE_LICENSE(x)
 to
 #define MODULE_LICENSE("GPL")
and in firegl_public.c you need to change:
MODULE_LICENSE("long license string I can't remember the exact detals of")
 to
 MODULE_LICENSE("GPL");
Then run:
./packages/Ubuntu/ati-packager.sh --buildpkg gutsy
from the top level fglrx directory. Install the generated .deb files which end up in the directory where you first extracted the drivers, with
sudo dpkg -i fglrx*deb

Finally run:
 sudo aticonfig --initial and reboot. You can then try different options to aticonfig to customise your display but this should be enough to get it working with full 3d acceleration.

et voilà!!:guitar:
