### Post by Aya Dream on 2007-03-07
Before you do your sudo grub, root (hd1,2), setup (hd1) trick you should check your fdisk -l results. If you've changed your BIOS to see hd1 as the first disk then maybe your old hd1 is now hd0, I'd have a better idea if you posted your fdisk -l results here as well as your device.map & fstab files

ps: error 22 is No such partition
                     This error is returned if a partition is requested in the device part of a device- or full file name which isn't on the selected disk.

### Post by dstew on 2007-03-07
Do you get the grub menu, then the error #22 after you select Ubuntu, or do you get the error before you see the menu? If you get the error after you select Ubuntu, you probably need to edit your grub menu.lst to change the device name where your kernel is located. If you changed the bios order of your disks, the kernel might now be on a different device.

### Post by bulldog on 2007-03-07
If you boot and select your ubuntu,hit the 'e' key and see were it's pointing to.
You changed your boot order so hd0 is hd1 now,and I'm pretty sure your ubuntu points to hd1,2)
Change this to (hd0,2) and hit the 'b' to boot.
If this works you'll have to change it permanently in your menu.lst.

Looking into your fstab to see if everything is listed as it should is the next step you'll have to take.
