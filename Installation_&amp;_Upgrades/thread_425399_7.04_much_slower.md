### Post by hal8000 on 2007-04-27
This is interesting, my PATA (UDMA100 Maxtor IDE ) hard drive was recognises as /dev/sda when booting with the original Feisty live disk.

However I suufered from the /bin/sh cannnot access tty.

After I modified /etc/initramfs-tools/modules and included the piix module and rebooted, my hard drive is again recognised as /dev/hda with the new Feisty 2.6.20 kernel

Once again i am able to load hdpam from services.
I wonder if you can create a new kernel and initramfs (included with piix modules) and boot from the new kernel. This is the thread I followed:

[http://ubuntuforums.org/showthread.php?t=421588](http://ubuntuforums.org/showthread.php?t=421588)

anc@orac:~$ df | grep hda
/dev/hda11             6297208   2614120   3683088  42% /
/dev/hda12             2104376     50416   2053960   3% /home
