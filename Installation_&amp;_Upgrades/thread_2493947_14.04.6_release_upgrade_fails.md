It is long past time to move to a supported LTS release.  Today, you should install 22.04 which will have support until April 2027, assuming it is a server. Most desktop versions only get 3 yrs of support, so there's about 1.5 yrs remaining on that of you have a desktop.

### Post by guiverc on 2023-12-30
I'll just add an alternative I'd likely use (*note: I'm a desktop user!*).

I discovered earlier this year I still had a 16.04 or *xenial* box running Ubuntu Desktop (ie. Unity 7 desktop). 

The way I wanted to *bring that box* into the modern world, was via *non-destructive* re-install; but there is where I got stuck, as I had to decide what I'd use. Ubuntu Desktop switched from Unity 7 (*used up to 17.04*) to GNOME 3 (*17.10 & later*), and I was sure I didn't want to be using GNOME 3 so I had to decide which [*flavor*]("https://ubuntu.com/download/flavours") I'd use to replace it..

I actually ended up going the *release-upgrade* way, only so I could answer a support question using it as example (*that upgrade was still possible*), but that involved many *release-upgrades* and took >12 hours, where as the *non-destructive* re-install would have been completed in less than 30 minutes!

Your hassle with re-install is you need to determine what packages you have that have been removed etc (*problems that will be detected during release-upgrade process*), as 2014-April to 2022-April is eight years of changes (*or whatever upgrade level you plan stopping at**; 20.04 is the oldest in standard support thus its six years to there*)... but that's planning I'd make in a normal *release-upgrade* which only advances you two years (for LTS).

I'll provide a link [here]("https://askubuntu.com/questions/446102/how-to-reinstall-ubuntu-in-the-easiest-way/1451533#1451533") where I wrote about it.  It's not a perfect solution, but its very fast (*when compared to release-upgrades*), can skip upgrades!, and can be used as a backup strategy if you have problems anyway.
