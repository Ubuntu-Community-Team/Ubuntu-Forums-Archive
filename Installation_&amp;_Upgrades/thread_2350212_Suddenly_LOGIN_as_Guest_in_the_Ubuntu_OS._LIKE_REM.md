### Post by Impavidus on 2017-01-22
The existence of a guest login is a standard feature of Ubuntu. When you login as a guest, the usual user profile is not accessible and you can't permanently store anything on your system. So that explains the later symptoms. The question is why you are forced to use the guest login. Did you get any warning or error messages during boot? When at the login screen, can you get to the console (ctrl-alt-F1, use ctrl-alt-F7 to get back) and login there? Maybe you get a useful error message.

When someone else has gained access to your computer, which is implausible but not impossible on Linux, the only really sure way to get a clean system again is reinstalling. But I'm not convinced that happened to you. Using a live disk you should be able to access your files.

Do you have an encrypted system or an encrypted home directory?
