### Post by beniwtv on 2007-05-14
Hi

Are you using an HP laptop? :) 

I do, and had the same problems with my HP.

First, to avoid fsck locking up, try this:

On the boot menu, scroll down to Ubuntu recovery mode and press 'b'.

Scroll down to the kernel line, and hit 'e'. Remove the "ro single", and put in instead "rw quiet init=/bin/bash".

Then hit enter to go back and type 'b' to boot ubuntu.

When the console comes up, type in:

nano /etc/init.d/checkfs.sh

comment out the line start() in the start: section.

Also, do a fschk on your main partition as follows:

```
umount /
fschk -fy /
```

Reboot by pressing <ctrl-alt-del> (there's no other way at this point), and your Ubuntu should boot, without checking partitions.
