### Post by bcbc on 2011-07-05
Wubi is a good way to try out Ubuntu without 'committing' to the whole partitioning business. It's better to do a proper install, but there's no rush... Wubi will give you a very close experience to the real thing.

Since you don't have a dvd/cd drive you can create an Ubuntu USB, following instructions here: [http://www.ubuntu.com/download/ubuntu/download](http://www.ubuntu.com/download/ubuntu/download) (Step 2, click on USB, then click on Show me how).

To partition - since you are using Windows7 - you can shrink the main windows partition using the "Disk Management" tool. Note that this only works if you haven't already used 4 primary partitions. If Windows pompts you to create Dynamic partitions, do not allow it (unsupported by ubuntu).

Then you can either let the installer install into the free space you created, or manually create partitions for Ubuntu (I prefer this).

It's easier to uninstall wubi, but not complex to uninstall a regular install:
Wubi - go to "Add or remove programs" in control panel.
Regular - boot from windows repair USB and replace bootloader (from a repair prompt, run "bootrec.exe /fixmbr"); then delete ubuntu partition from Windows disk management and reuse space. It's a bit of a pain creating a windows repair USB - windows only allows you to create a repair CD - but there are guides on the internet on how to do this (I've done it before).

Yes, you can access any ntfs partitions from Ubuntu. From the windows side, there are some tools that you can use to view files on an ext4 partition e.g. [http://ext2read.blogspot.com/](http://ext2read.blogspot.com/) (this can also view the virtual disk (root.disk) Wubi uses). But my recommendation would be to have a separate data ntfs partition to store data if you want to access it from both Ubuntu and Windows.

I don't know how much space you should take for Ubuntu - it really depends on how you will be using it. But for my usage 20-30GB is sufficient. I keep my large multimedia files on ntfs.

Some people recommend a separate / (root) partition and /home. I don't bother - that's up to you. If you keep regular backups then it doesn't really matter too much which method you choose.

### Post by bcbc on 2011-07-06
windows has an option to create a restore image... Plug in a USB stick that is big enough - 8 or 16GB - you might need a bigger one if you've made lots of changes.
Click Start, enter "restore", select "Backup and restore" from above, click on "Create a system image". 

EDIT: some manufacturers might supply a tool to create a USB factory restore. e.g. Toshiba has a "Recovery media creator".

I've heard of some clone software like clonezilla. The only clone I've used is "dd" (linux tool to do a block level partition copy). But this uses more space. The important thing is to be able to restore the factory installed OS (plus drivers) you paid for when you bought the computer. The data/programs can always be backed up and reinstalled separately - so personally I didn't bother cloning windows.
