### Post by Roberto Rosselli on 2006-12-01
Hi guys,
I am an Ubuntu user since Breezy, and have standardized all my computers on Ubuntu. I recently tried to install Ubuntu 6.10 on my main box (a fairly vanilla system: AMD Sempron 3100, 1024GB RAM, ATI Radeon 9600 graphics adapter) but in spite of repeated attempts I couldn't bring the installation to an end:

* I first tried with the Desktop (Live) CD: black screen and no way to understand what went wrong; I tried with the safe graphics mode: couldn't progress beyond a very garbled and unusable image of the Ubuntu desktop; tried with more kernel commands: no good either

* at this point I tried with the alternate CD: I just met the old "couldn't install initramfs" bug I thought defeated with Dapper, tried several times with no luck

* got back to Dapper, and installed it using the Desktop CD, *BUT* the installer forced me to format the /boot partition where the Mandriva install files lived (I used Mandriva, which installed with no problem, to download the alternate CD and to do other stuff waiting for 6.10 to install): why was that necessary? I know it's not the case, but it reminded me of the good old message from the Windows 3.x (or was that 95?) installer "OS/2 partition found, formatting..."

Note that I checked the md5sum of all downloaded images, burned them at low speed and then checked the CDs for defects: all were fine. All in all I'm quite disappointed with Ubuntu, I suppose it spoiled me working fine *after* install, but surely the installation process needs more work. If I were a newbie I'd be scared away from Ubuntu, unfortunately. I also wonder why there's no release notes with a FAQ on the more common installation problems on the Ubuntu site, at least I couldn't find it.

RR
