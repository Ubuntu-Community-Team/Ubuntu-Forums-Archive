### Post by wegorz on 2011-10-14
hello, 

1] move all /var/run -> /run and /var/lock -> /run/lock
2] delete /var/run and /var/lock
3] link /run -> /var/run and /run/lock -> /var/lock
4] delete /run/dbus/*
5] restart

Should work. gl.

This, written as commands by someone without root privileges would be e.g.: 
```

sudo mv /var/run/* /run/
sudo mv /var/lock/* /run/lock/
sudo rm -r /var/run
sudo rm -r /var/lock
sudo ln -s /run /var/run
sudo ln -s /run/lock /var/lock
sudo rm /run/dbus/*

```

More info about the problem can be found here: [Bug 811441](https://bugs.launchpad.net/ubuntu/+source/dbus/+bug/811441). Solution seems to come from post #24 there. Grateful to wegorz who posted it here though.

### Post by mixoff on 2011-10-16
"Works like a charm", except you get weird errors elsewhere. Try ```
sudo update-grub
``` and see if it works for you. I got nice error messages [COLOR=Red]ERROR: mkdir /var/lock/dmraid[/COLOR] because **/var/lock** does not exist anymore by this "fix".

Seems that symlinks are wrong and you should create them other way round:
```

sudo ln -s /var/run/ /run/
sudo ln -s /var/lock/ /run/lock/

```
The workaround helps to boot the machine, but the problem is bigger and still exist as reported [COLOR=Blue]**[here]("https://bugs.launchpad.net/ubuntu/+source/lightdm/+bug/856810")**[/COLOR].

Be careful and seek for more info that really works... like a charm.
Cheers!
