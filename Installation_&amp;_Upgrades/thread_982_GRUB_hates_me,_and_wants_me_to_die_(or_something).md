### Post by mark on 2004-10-17
Don't know about the RAID stuff, but I know it did take a bit of fiddling for me to get *grub* to acknowledge the existence of 2 Linux installations on my drive.  For what it's worth, here's my  *grub.conf/menu.lst* file:

```
# grub.conf generated by anaconda
#
# Note that you do not have to rerun grub after making changes to this file
# NOTICE&#58;  You have a /boot partition.  This means that
#          all kernel and initrd paths are relative to /boot/, eg.
#          root &#40;hd0,0&#41;
#          kernel /vmlinuz-version ro root=/dev/hda3
#          initrd /initrd-version.img
#boot=/dev/hda
default=3
timeout=10
splashimage=&#40;hd0,0&#41;/grub/splash.xpm.gz
title Fedora Core &#40;2.6.7-1.494.2.2smp&#41;
	root &#40;hd0,0&#41;
	kernel /vmlinuz-2.6.7-1.494.2.2smp ro root=/dev/hda3 rhgb quiet
	initrd /initrd-2.6.7-1.494.2.2smp.img
title Fedora Core &#40;2.6.7-1.494.2.2&#41;
	root &#40;hd0,0&#41;
	kernel /vmlinuz-2.6.7-1.494.2.2 ro root=/dev/hda3 rhgb quiet
	initrd /initrd-2.6.7-1.494.2.2.img
title Ubuntu Linux &#40;2.6.8.1-3-386&#41;
	root &#40;hd0,5&#41;
	kernel /vmlinuz-2.6.8.1-3-386 ro root=/dev/hda7 quiet
	initrd /initrd.img-2.6.8.1-3-386
title Ubuntu Linux &#40;2.6.8.1-3-686-smp&#41;
	root &#40;hd0,5&#41;
	kernel /vmlinuz-2.6.8.1-3-686-smp ro root=/dev/hda7 quiet
	initrd /initrd.img-2.6.8.1-3-686-smp

``` My original install was Fedora Core 2 - rather than let Ubuntu overwrite it, I edited the original.  Note that while my FC2 &amp; Ubuntu boot devices are */dev/hda1* and */dev/hda6*, respectively, *grub* treats them as *(hd0,0)* and *(hd0,5)*, respectively.

You might be able to boot the Ubuntu CD with "linux rescue" as the start command - I'm not sure, as I haven't tried this as yet.  The "rescue" mode (with most distros) lets you mount necessary devices &amp; attempt to recover from configuration *oopses* such as this.

Good luck!

Mark
