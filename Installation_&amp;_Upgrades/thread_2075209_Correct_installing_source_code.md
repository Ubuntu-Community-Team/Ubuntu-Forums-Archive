### Post by drmrgd on 2012-10-23
The first thing to check is the website from which you downloaded the package.  Good package maintainers will list the dependencies there with instructions for how to set the package up.  

The second place to check is the README and / or INSTALL text file(s) in the package you downloaded.  There are usually explicit instructions in these files that tell you exactly how to install a package and what (if any) other packages are required for a successful build.

Finally, if you can't find the information in either of the above, feel free to contact the person from which you've gotten the package as they'll know better that we would, what dependencies are required.  You're welcome to post some info about the package here, as well as the error you messages you're getting during your build.  However, we'll probably just do the same things and check the README / INSTALL files and the website from which you've downloaded the package :P
