### Post by mac.ryan on 2007-04-13
I would guess, that this is due to the fact the liveCD is designed for maximum compatibility with all systems, rather than for performance, thus it comes with "basic" configuration.

What you might wish to do is to arrange your system for a persistent mode (i.e. a mode in which you will still boot from CD but you will have your specific setting loaded from another media (i.e. USB key). [Here]("http://www.cyberciti.biz/tips/ubuntu-linux-live-cd-save-data-desktop-information-on-usb-device.html") is an how-to but I never tried myself. So no warranty it will work! :)

The configuration of your graphical system are in a file called /etc/X11/xorg.conf while you might wish to install proprietary drivers of your video card vendors if you need serious 3D acceleration.

Overall, if your computer is usb-bootable I would however recommend to install the whole of ubuntu on a USB removable media. It will be much faster, and much better configurable. [Here]("https://wiki.ubuntu.com/LiveUsbPendrivePersistent") the how-to.

### Post by xxdd on 2007-04-13
If you try opening up the terminal and running:
sudo dpkg-reconfigure -phigh xserver-xorg
Then choosing the video driver you want, probably nv for nvidia, i'm afraid i don't know for ati, after this it will let you select the resolutions, go down to any resolutions you want available with the arrow keys and hit spacebar to select them, then press return.
After this you'll need to kill your current x session, do this by logging out and hitting control-alt-backspace.

Hope this helps.

It seems to have something to do with the modes of the monitor not the modes of the actual graphics card.  I changed the monitor section to match what was in the xorg.conf from suse & that still wasn't enough I also had to change the screen section to use the new monitor, that's the monitor line in the screen section. ```

```
Section "Monitor"
  DisplaySize  340 270
  HorizSync    31-64
  Identifier   "Monitor[0]"
  ModelName    "1280X1024@60HZ"
  Option       "DPMS"
  VendorName   "--> VESA"
  VertRefresh  50-60
  UseModes     "Modes[0]"
EndSection


Section "Modes"
  Identifier   "Modes[0]"
  Modeline 	"1280x1024" 108.88 1280 1360 1496 1712 1024 1025 1028 1060
  Modeline 	"1280x1024" 108.0 1280 1328 1440 1688 1024 1025 1028 1066 +hsync +vsync
EndSection

Section "Screen"
	Identifier	"Default Screen"
	Device		"3Dfx Interactive, Inc. Voodoo 3"
	Monitor		"Monitor[0]"
	DefaultDepth	16

```

Anyways, It seems to be working now.

### Post by Geva Zeichner on 2007-05-11
Solved it! Followed [this]("https://bugs.launchpad.net/ubuntu/+source/xorg/+bug/3731") workaround.

I just needed to modify the monitor section of xorg.conf from this:
```
Section "Monitor"
	Identifier	"MAG 786FD"
	Option		"DPMS"
EndSection
```
to this:
```
Section "Monitor"
	Identifier	"MAG 786FD"
	Option		"DPMS"
	HorizSync	30-86
	VertRefresh	50-160
EndSection
```

* You need to find out the right values for your specific monitor.
