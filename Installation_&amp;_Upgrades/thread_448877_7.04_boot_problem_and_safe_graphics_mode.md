### Post by thewave99 on 2007-05-19
I have been running ver 6.10 in a dual boot configuration with WinXP on an Averatec 3250 (512 MB) with no problem.  I upgraded to ver 7.04 via the upgrade manager.  All seemed to go fine until I started up.  The boot process gets me into the Gnome login screen and then the process just slows to a crawl.  (The mouse barely works at this point)  If I log in and wait long enough (20 minutes) Gnome will give me an error message that some application is not responding.

I made a ver 7.04 liveCD and found the same problem booting from the CD in the normal mode.  HOWEVER (!), the safe graphics mode worked just fine.

My question is, what is the difference between safe graphics mode and normal mode?  What changes can I make to my installation that would solve my problem or at least to allow me to boot into safe graphics mode?

Thanks,

David

---

### Post by merlinus on 2007-05-19
Seems like it might be your graphics driver.  Look at /etc/X11/xorg.conf.

Maybe changing the driver to vesa will help.

Good luck!

-merlin

### Post by thewave99 on 2007-05-19

Your suggestion has done the trick.  I'm up and running.
