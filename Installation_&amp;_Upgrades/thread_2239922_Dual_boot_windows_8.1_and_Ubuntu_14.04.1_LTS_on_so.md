### Post by oldfred on 2014-08-16
Windows uses the bootmgfw.efi in the efi partition. There are several recovery partitions for Windows and for the vendor repair/recovery which may also have that file, but only are used it in repair/recovery mode.

Generally better not to use the rename of bootmgfw.efi. That also is Boot-Repairs fix, but then you have to boot Windows using a grub menu entry of the renamed file. If Windows does an update then it replaces the bootmgfw.efi and then you cannot boot grub again and have to do the rename. Because of all those issue you need both a Windows repair flash drive and Ubuntu live installer always available so you can make fixes.

These are all the alternatives I have accumulated so far. Some work better than others. But I now prefer renaming bootx64.efi or use rEFInd if that will work for your system.

### Post by fmherr on 2014-08-18
Thanks a lot. It worked.

I followed the instructions in the first link (post #6) on your option A. The only difference was in step 5. I did not find any entry like that on my files, so I googled a little bit and added this similar entry.

```
menuentry "Windows 8" {
    set root='(hd0,gpt3)'
    chainloader /EFI/microsoft/BOOT/bootmgfw.efi.bkp
}

```

Now it loads grub, with "Ubuntu" as my first option and "windows 8" as my last. Between them there are "Windows UEFI" and "System setup" options. If I choose any of them, it just return to grub menu. Nevertheless, it's fine.

With the rename of the Windows boot file, some Windows updates will revert the bootmgfw.efi file back to Windows instead of being a rename grub. Then you will only boot Windows again and have to redo the rename. So keep Ubuntu live install handy after any Windows updates.
