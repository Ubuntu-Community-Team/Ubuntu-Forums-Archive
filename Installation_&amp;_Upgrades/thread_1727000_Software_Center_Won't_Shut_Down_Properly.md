Software-Center **'is'** very buggy; but it's come a long way since Hardy-Heron.  

Software-Center, Synaptic Package Manager, and APT, are all now *'somewhat'* integrated.  They never used to be...  Now, when I add an application through Synaptic or APT, it will often show-up in Software-Center as well.  This is a *good-thing.*  These needed to be integrated...  They just need to be integrated *more.*

Unfortunately the heuristics used to keep track of application dependencies is very simple, and 'not' well integrated between the three ***package-mangement*** systems mentioned above.  Also, there's almost no feedback from the system, when application dependencies are *not* met.  There needs to be descriptive and informative feedback from the system in this event, so that the user can make an informed decision to either keep his present configuration, and abandon the installation, or to remove the offending package and proceed with the installation.  This ***'feedback'*** needs to be provided with consistency and uniformity in APT, Synaptic, and Software-Center.
