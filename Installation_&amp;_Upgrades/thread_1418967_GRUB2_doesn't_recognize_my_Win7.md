### Post by darkod on 2010-03-01
The windows boot files were on the partition you deleted/overwrote. Because grub is actually searching for boot files, there are none to find. You can't boot your win7 as it is even if you had windows bootloader on the MBR of the hdd.

Use your win7 install dvd and read here about repairing the boot files:
[http://ubuntuforums.org/showthread.php?t=1014708](http://ubuntuforums.org/showthread.php?t=1014708)

After you repair your win7 boot process and can boot it successfully, in the same article there are instructions how to restore grub2 for 9.10 on the MBR, because the windows repair process will overwrite it.

No need to reinstall ubuntu fully, just reinstall grub2.

As Darko said windows combines its boot files into the partition with the boot flag (* in fdisk). Move the boot flag to the windows partition, it may help windows repair in finding the win install. You can set the boot flag with gparted from the live CD. From the windows disk are you running the automatic repair or the command line version?
