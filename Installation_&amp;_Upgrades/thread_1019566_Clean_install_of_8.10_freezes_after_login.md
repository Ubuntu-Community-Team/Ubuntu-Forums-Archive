Hello Folks.
Clean install of 8.10 from download iso file ( hash values checked OK ),
installed as dual boot with WinXP.
System has Intel 82845G graphics controller, 512 Mb Ram, 2.4Ghz, 37Gb
During install, froze at "Installing System - 32% - copying files "
I rebooted via reset switch, and installation proceeded to completion.
But .. now after login screen of user name + password (entered correctly),
it freezes at blank, beige screen - not desktop with ugly coffee stain (?),
but totally blank beige. Will not proceed any further.
When booting into " recovery Mode " I tried all the options - no improvement.
At root shell prompt, when I attempted to edit menu.lst I got following message: " Gtk-warning 'cannot open display ' "

(Previous to installation I attempted running from CD - then froze at coffee stain desktop.)

I've used Ubuntu 5.10 + 6.10 happily for years.
Any ideas on this problem? Or is 8.10 to be avoided in favour of more stable edition.WinXP does not show linux partition on HDD, but usage of HDD

### Post by snick on 2008-12-23
Solved. Uninstalled Ubuntu via uninstaller in Ubuntu directory on Windows C:
 drive ( didn't expect to find it there!) after booting to WinXP.
Booted from 8.10 CD and ran live version. Made it to desktop this time;
 installed via "install" option on Ubuntu desktop of live version running from cd.
Now all I have to do is get my Netgear PCI WG311v2 802.11g wireless working in Ubu. 8.10. That's another post. Cheers.
