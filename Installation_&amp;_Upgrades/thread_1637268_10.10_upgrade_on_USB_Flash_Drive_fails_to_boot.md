### Post by C.S.Cameron on 2010-12-04
If this is a persistent install that uses a casper-rw file an update is likely to fill it up and it will not boot,
One solution is to delete the casper-rw file and install a new one, loosing everything that has been saved,
Another solution is to mount the casper-rw file and delete stuff from it to regain space.

### Post by C.S.Cameron on 2010-12-04
Booting from the Live CD or desktop Ubuntu:

Make a directory to mount casper-rw in:
```

sudo mkdir /media/casper

```
and then mount it:
```

sudo mount -o loop casper-rw /media/casper/
```

A good source of stuff to delete is in var/cache/apt/archives

Use gksu nautilus to get there.

You might also try using Computer Janitor.
