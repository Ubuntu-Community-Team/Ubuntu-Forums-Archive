Whichever drive is set to boot first in bios when you install Ubuntu is where grub should be installed(mbr of hda or sda), as long as this drive remains the first boot hard drive in bios.

What you could try is make sure your bios is set to boot first to the SATA drive, then try reinstalling grub to the SATA mbr(/dev/sda).  You can reinstall grub, using the live cd*:
[http://www.ubuntuforums.org/showthread.php?t=224351](http://www.ubuntuforums.org/showthread.php?t=224351)

If you're able to boot Ubuntu this way, you can add an entry to your /boot/grub/menu.lst to boot Windows...something similar to the link I gave in my other reply.

Have you tried to set your bios to boot first to the IDE drive to determine if the Windows drive will boot?  If it does, then it means grub wasn't installed to hda; however, I believe it might have been.  You might sometime in the future, if you want, repair the Windows mbr:
[http://www.users.bigpond.net.au/hermanzone/p18.htm](http://www.users.bigpond.net.au/hermanzone/p18.htm)
This link is titled "Uninstall Ubuntu", but you just need to read the info on how to restore Windows mbr.

*Installing grub with the live cd.
Open a terminal, enter:
```
sudo grub
```
this will give you a grub prompt, at the prompt, type:

```
find /boot/grub/stage1
```
this will find grub stage1 on your root partition, something similar to this:
```
root (hd0,0)
```
at the grub prompt, enter:
```
root (hd0,0)
```

```
setup (hd0)
```
this will install grub to the mbr of the SATA drive

to exit grub:
```
quit
```

Then reboot your computer, with SATA as the first boot hard drive, this should give you a grub menu to boot Ubuntu.  If  find /boot/grub/stage1 results in (hd1,0), you could still try installing to root (hd0)...but this would require booting first to the IDE drive.

I hope I've given you some ideas, maybe didn't explain it very well, but hopefully there's some things you can try.

Yes, it's worth trying...set your bios to make sure it boots first to your IDE drive, if you get a grub menu and Ubuntu still doesn't boot, you can reboot your computer, highlight the Ubuntu entry in the grub menu, press "e" to edit...change the root to (hd1,0) and in the kernel line, make sure the root points to sda1(root=/dev/sda1), then try to boot.  If the edit option works, it's only temporary, you can make it permanent in your /boot/grub/menu.lst.  You'd need to make changes in your Ubuntu recovery mode and Windows entry, if the edit works.

I'd definitely try this option, before reinstalling grub or Ubuntu...
