### Post by Antarctica on 2008-03-18
Because you have two Ubuntu options, I'm assuming she went through a complete install.  She installed Ubuntu onto a different partition (which was automatically generated by gparted, the GNOME Partition Editor).  You have two choices, one hard and one easy.  You can either remove the other partition and remove any references to the second partition in /boot/grub/menu.lst, or you can simply save all your data and reinstall Ubuntu Studio.  By the way, what attracted you to Ubuntu Studio instead of the regular Ubuntu?

### Post by unutbu on 2008-03-18
I think you are in luck. No new partitions have been added. 

You can clean up your grub boot menu this way:

```

gksudo gedit /boot/grub/menu.lst

```

Just comment out (i.e. put pound signs (#) in front of) the two stanzas below:
```

title Ubuntu 7.10, kernel 2.6.22-14-rt
root (hd0,2)
kernel /boot/vmlinuz-2.6.22-14-rt root=UUID=6f91c970-cb39-4e06-9ec4-923
771f7da8b ro quiet splash
initrd /boot/initrd.img-2.6.22-14-rt
quiet

title Ubuntu 7.10, kernel 2.6.22-14-rt (recovery mode)
root (hd0,2)
kernel /boot/vmlinuz-2.6.22-14-rt root=UUID=6f91c970-cb39-4e06-9ec4-923
771f7da8b ro single
initrd /boot/initrd.img-2.6.22-14-rt

# Put a pound sign (#) to comment-out these lines
#title Ubuntu 7.10, kernel 2.6.22-14-generic
#root (hd0,2)
#kernel /boot/vmlinuz-2.6.22-14-generic root=UUID=6f91c970-cb39-4e06-9ec4-923771f7da8b ro quiet splash
#initrd /boot/initrd.img-2.6.22-14-generic
#quiet

#title Ubuntu 7.10, kernel 2.6.22-14-generic (recovery mode)
#root (hd0,2)
#kernel /boot/vmlinuz-2.6.22-14-generic root=UUID=6f91c970-cb39-4e06-9ec4-923771f7da8b ro single
#initrd /boot/initrd.img-2.6.22-14-generic

title Ubuntu 7.10, memtest86+
root (hd0,2)
kernel /boot/memtest86+.bin
quiet

```

You can also (prepare to) remove the added kernel:
```

mkdir ~/tobedeleted
sudo mv /boot/initrd.img-2.6.22-14-generic ~/tobedeleted

```

Reboot.
If that works, which it should, then
```

sudo rm ~/tobedeleted/initrd.img-2.6.22-14-generic
rmdir ~/tobedeleted

```

You can also clean up the /boot/grub/menu.lst by deleting the two commented-out stanzas if you wish, but no harm will come of you just leaving them there.

### Post by unutbu on 2008-03-19
Ah: '11' should be '22', and '~' should be '~/'
```

sudo mv /boot/initrd.img-2.6.11-14-generic ~tobedeleted

``` 

Actually, you can remove a few more files:

```

sudo mv /boot/config-2.6.22-14-generic ~/tobedeleted
sudo mv /boot/initrd.img-2.6.22-14-generic ~/tobedeleted
sudo mv /boot/System.map-2.6.22-14-generic ~/tobedeleted
sudo mv /boot/vmlinuz-2.6.22-14-generic ~/tobedeleted

```

After you reboot, if everything appears in order, then you can issue the following command:

```

sudo rm -ri ~/tobedeleted

```
That will remove the directory and all its contents.
