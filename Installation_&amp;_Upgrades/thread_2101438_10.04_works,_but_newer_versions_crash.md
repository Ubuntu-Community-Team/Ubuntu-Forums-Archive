### Post by houseworkshy on 2013-01-04
This is really just another bump.
We have very similar machines  though you have a little more ram.  The only thing which stands out for  me in that report is that you don't seem to have a swap file, which is  odd if you did an automatic install.  Though I don't have a clue what,  if anything, that may have to do with it.  
A distro ( windows installation disks are also distro's ) should be self contained looking at nothing but hardware.
You  could perhaps do the same.  Entering "man badblocks" will give you  details of one command which may be helpful.  Just found this online  which may help if you are not used to man pages.  
[http://linuxpoison.blogspot.co.uk/2008/01/howto-check-disk-drive-for-errors-and.html](http://linuxpoison.blogspot.co.uk/2008/01/howto-check-disk-drive-for-errors-and.html)
I'm not familliar with this web site, looks ok bar the scary name to this novice, but double check with the manual just in case.
**I  strongly advise against doing anything other than diagnostics with disk  utility programs**like badblocks and fsck**  until you absolutely need to and are certain about precisely what to type**  they can be real system/data  killers if you get them wrong.   I'm only guessing at a dodgy drive as one outside possibility, it's most likely not that but worth looking at.
There is  also the posibility that you've got a damaged installation disk for win  xp.  
As to 12.04, in general clean reinstalls behave much better than version  upgrades.  So perhaps download a fresh iso of 12.04, which would also  give you the chance to test with a live cd/dvd session.  If you do that slap it on write once only media, problems often happen with re-writables,  and burn it as slow as you can. 
There is also Ubuntu secure remix
 [https://help.ubuntu.com/community/UbuntuSecureRemix](https://help.ubuntu.com/community/UbuntuSecureRemix)
which  includes boot repair so may be useful if some booting problem is  involved, which I also doubt to the extent I'm near sure it's not that. Though it would certainly be helpful if you wanted to duel boot  Ubuntu with xp and encountered any problems.
It's could be that  things are getting hot and tripping out.  If so installing "lm sensors"  would be a way of testing temperature, just enter  "sensors" in the  terminal to give you an instant temperature readout.  If it's that it  may simply be a matter of cleaning the insullating dust out of the box.
Verbose but still only wild guesses and a bump.

### Post by kurt18947 on 2013-01-06
That seems like a fairly current machine.  I've had trouble with the Ubuntu installer on a 10 yr. old laptop.  Lubuntu wouldn't install, Xubuntu would.  You could try a non-*buntu/non-Debian live install such as PCLinuxOS, OpenSUSE or Fedora.   If one of those load and run, there may be an issue with the download or Ubuntu installer and your hardware.  If those won't load either, I'd suspect a hardware issue.

I also found on that older machine that Lubuntu would load - if it were plugged into a **wired** network connection.  The installer when run on that particular machine looked for a network connection and if it didn't find one would lock up.  If I prodded further, I'd get 'I can't read the CD' messages scrolling by.
