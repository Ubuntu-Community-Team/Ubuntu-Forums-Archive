It's a bug that crept in too late to be fixed before final release, and was only noticed at the very last moment. The workaround is to open the text editor while the installer is running, type in your mountpoint, copy it to the clipboard, right-click on the mountpoint field and paste.

I've used the live desktop ISO several times to install Natty and it has not yet failed to install the bootloader. Suggest you check the md5sum of the ISO you are using and if you are using a CD rather than USB, go to the text menu of the live session at startup and do a disc integrity check.

That's very interesting. I presume that's with the issue where it's asking you to insert the CD that's already there.

Your earlier mention of the installer failing to load the bootloader reminds me of a problem I encountered back in Dapper/6.06 days when I was running a machine with two IDE/PATA hard drives. The installer failed at the bootloader installation stage. Long story short: the hard drives were correctly hooked up to the two connectors on the IDE ribbon cable, but I had the jumpers on the two drives set to cable select. When I reset them to master and slave respectively, everything worked as it should. Ever since I've always avoided the cable select  jumper setting on IDE drives, whether hard drives or optical drives. I've often seen it said that cable select is unreliable. This may not be the issue here, but it might be worth checking, because I can boot up the live CD on my machine that has an IDE/PATA optical drive.

If you want to now that you've found a workaround, that is. :)
