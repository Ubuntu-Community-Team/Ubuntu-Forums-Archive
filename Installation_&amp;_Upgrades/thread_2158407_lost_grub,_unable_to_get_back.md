### Post by papibe on 2013-06-28
Hi furbreath. Welcome to the forum ):P

The recovery process probably reinstall the Windows boot loader, effectively wiping out grub.

You can reinstall grub using the Ubuntu installation CD as a LIve CD, and then using Boot-Repair.  [Here]("https://help.ubuntu.com/community/Boot-Repair")'s a tutorial.

Hope it helps. Let us know how it goes.
Regards.

### Post by oldfred on 2013-06-28
Some of the systems seem to rewrite the partition table as soon as you start recovery. And since it does not expect Ubuntu overwrites the partition table entry. 
If you do not have any ext3 or ext4 partitions then you may need to use testdisk.

Post this.

sudo parted -l
