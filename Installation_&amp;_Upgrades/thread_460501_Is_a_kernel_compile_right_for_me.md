### Post by mlind on 2007-05-31
Hiya,

I used to compile vanilla kernel's in the past while I was fedora user and beginning Ubuntu user. The biggest reasons I did this was that it looked exiting and I had few gadgets that needed kernel patching to work. One of the reasons I'm using "binary distro" like Ubuntu/Debian compared to Gentoo is that I don't have to compile stuff manually that much and I've found Ubuntu kernel very stable for my needs.

I'm eager to test out some recent scheduler patches though, so I'm going to build a test kernel soon. But I'd say you don't need a vanilla kernel, unless you want to experiment something or you have a piece of exotic hardware which won't work unless. Stability-wise, Ubuntu kernel is very good.
