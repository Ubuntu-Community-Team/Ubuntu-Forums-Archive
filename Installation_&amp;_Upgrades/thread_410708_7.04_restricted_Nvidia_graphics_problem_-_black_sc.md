### Post by bytesmythe on 2007-04-16
ctrl-alt-F1 only works if you're in an XWindows environment. Your X isn't loading, so you would just use alt-F1 to get to the console.

We're having the same problem over here... There's no way Feisty is going to be ready for release on time at this rate!

Try this to temporarily work around your graphics issue:

[LIST=1]
[*]Get to a terminal

[*]sudo vim /etc/X11/xorg.conf

[*]Find the place in the file where it says:

[INDENT]Driver "nvidia"[/INDENT]

and change it to

[INDENT]Driver "nv"[/INDENT]

[*]Reboot
[/LIST]
This will hopefully at least let you get into your system.

### Post by Moloko on 2007-04-22
I have X up and running again, in my case the solution was add this line to the **Device** section in the **/etc/X11/xorg** file :

```

Option  "ConnectedMonitor" "CRT"
```

```

Section "Device"
    Identifier     "nVidia Corporation GeForce 6200 A-LE"
    Driver         "nvidia"
    Option         "ConnectedMonitor" "CRT"
EndSection

```

I supose that other way to do the same is this:

```

Option   "IgnoreDisplayDevices" "DFP,TV"

```

In my case using the first option was enought.  Is like if the nVidia driver doesn't know what ouput use.
