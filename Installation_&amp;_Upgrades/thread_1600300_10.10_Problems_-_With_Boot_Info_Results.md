### Post by efflandt on 2010-10-19
It is probably confused because something messed up the order of your partitions.  How did sda7 end up physically "before" sda5 (look at start and end points in fdisk output)?  Maybe it would work better if you shuffled the partition numbers to the correct physical order.

sda7 should be sda5
sda5 should be sda6
sda6 should be sda7

### Post by ronparent on 2010-10-19
I have this exact situation. The partitions are numbered in the order that they are created not in their physical order in the disk. That in itself doesn't create any problems. A simple grub reinstall will fix it as Rubi1200 suggested: [https://help.ubuntu.com/community/Grub2#Reinstalling](https://help.ubuntu.com/community/Grub2#Reinstalling) GRUB 2
