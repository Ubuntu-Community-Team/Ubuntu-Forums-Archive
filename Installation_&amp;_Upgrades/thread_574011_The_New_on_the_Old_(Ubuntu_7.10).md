### Post by irv on 2007-10-12
[CENTER]**The New on the Old (Ubuntu 7.10)**[/CENTER]
	With the release of Ubuntu 7.10 this month, I wondered how it would run on the older machines. Well, I am here to tell you in one word. “GREAT!” I have been only working with the beta version and have installed it on 9 machines. But before telling you how things went, let me first lay a little ground work.

	I have been working on a big project to ship a 40' steel shipping container to Nigeria and we are at the tail end of the container and needed something to fill it so we could get it shipped this month. Well, my son gave me a tip about the Minnesota Housing office here in town that just replaced there old computers with new ones. After talking with the people in charge of disposing the old PC's I asked if they would donate them to this project. Now instead of paying to get rid of them they can use them as a tax write-off. It was a win win situation. 

	The nine machines that were donated to this project were all Compact DeskPro's but the configuration was different on each of them. Some had only 128MB of RAM while others had 256MB. Two had 384MB. Most of them were 550Mhz machines but I had one 600Mhz one 700Mhz and one server that had a 667Mhz.  The Hard Drives were a mix also. Some 10gig 13gig and 20gig. The server was also a Compact but it had a combo Network/Controller/Video card that Ubuntu didn't like. The video was to low. I installed a new video card in it and that solved some problems but the only way I could get 7.10 installed was by doing a install of an older version and then working my way up to the 7.10. I started with 5.10 because it was not a graphic install. It now has 7.10 on it and it is running OK. 

	Now the install was not smooth on all of them and I found that the ones with the lower memory would hang after doing about 90 to 94% of the install. When I saw this happening I took memory from one machine and used it in the one that had only 128MB. This not only let the install finish but it made it run faster.

	The slow part of the install was the booting to the Live CD. This was probably due to the controller speed in these older machines. I did notice that the top panel did not show up at all on some of the PC's but the install icon did come up on all of them. It took about 20 to 25 minutes to load the Live CD on the 128MB machines but only 15 minutes on those that had more memory. 
	The amazing thing was after the install was complete, it only took about 2 minutes to boot to the login screen and maybe another minute to start using the PC. I was very impressed by the fact that they ran better then I expected. 

	When using software like “Write” in “Open Office” it does take more time than on my laptop, but over all it was usable.

 	There was one thing that happen on all these PC's that was the same. They all had Windows 98 installed on them, except for the server which had Netware 5. All of the PC's would not load the Live CD from the get go. I had to start an install of an older version of Ubuntu (5.10), because it did a non graphic install. I did not have to do a complete install just the first part and this just wiped the Hard Drive and setup for the install. After doing this it let me boot the Live CD of version 7.10 beta. 

	My plan is to setup a network for the server and all the workstations. I would have liked to installed EdUbuntu and do a “thin client/thin client server” install, but this server does not have enough memory. The documentation said to allocate 150Mg per node. 

	I also have ordered a 16 port switch from ebay to setup a network for all these PC's once they get to the Haven of Hope in Jos Nigeria. My plan is to make a trip over there once the container gets there and help them get everything setup. I am not sure if they have the Internet in the area where they are going so I hope to have them all running the full version release and not the beta when they ship.

	One more thing before I close, the update manager ran very well on all the PC's once they were up and running. The downloads were fast because of my 5MB Internet  connection, but the install of the software update also ran faster on these older PC's then I expected. In case you are wondering why I didn't put Xubuntu on these older boxes, I did at first, but for kicks I thought I would try to load the Ubuntu and after doing so, I felt that they ran well enough to leave it on. Like I said it was a slow install but once it's installed it runs fast enough to be able to use it.

	Just in case you are wondering what the system requirements are for installing Ubuntu 7.10 the requirements on version 6.10 was, technically fine (though slowly) on a dusty 25 Mhz i386 machine. However, I would not advise doing this. You shouldn't bother trying to install and run it on a system with a processor that runs slower than 166Mhz or which has less than 96MB of memory. Really, the minimum hardware requirements are as following.
[LIST]
[*]700 Mhz or better processor
[*]256MB of memory
[*]CD-ROM drive
[*]Ethernet interface
[*]SVGA graphic interface
[*]3GB of available disk space
[/LIST]

If you are like me, and push the lower limits you will soon get tired of waiting for things to load.

Irv
