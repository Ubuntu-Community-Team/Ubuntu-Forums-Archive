I have trouble getting HDMI video output from my Gigabyte GA-MA780GM-S2H motherboard using the integrated ATI HD3200 graphics card. During boot it works fine, but after login (when fglrx is activated) I get no signal anymore.

I even did a fresh install of Ubuntu 8.10, using xorg-driver-fglrx 2:8.543-0ubuntu4, but still no luck. Before that I tried to use the latest 8.11-driver. It works if I disable fglrx and it seems work fine if I use the DVI-port (using a DVI-to-HDMI cable between my computer and projector). 

I used to use the DVI-port before since my old computer didn't have a HDMI-port, but I bought a new longer cable and opted for a HDMI-cable with HDMI-to-HDMI connectors instead.

I have tried several different settings in my xorg.conf without any results (using aticonfig). Right now I have a barebones xorg.conf since I have done a fresh reinstall.

It looks as if the driver don't recognise the projector when using the HDMI-output. I've seen something about being able to use an option called ForceMonitor and use tmds<N>, I tried a litte bit but no luck. I have attached my xorg.conf and Xorg.0.log files.
