### Post by J77 on 2008-02-07
OK, I'm usually pretty rubbish at putting up when stuff works but...

Upgraded from LTS yesterday and everything went wrong! Well, the kernel had a complete hissy fit :)

Finally solved this by continuing the upgrade to 7.10 and deleting a load of junk kernels from my boot part.

Problem then was with TwinView.

Now, this worked fine under 6.06 and I read that nvidia-settings should work under 7.10 -- however, not...

After a day in which I got no work done whatsoever (I'm a scientist relying heavily on my computers...), I started afresh this morning.

A first google search brought me this:

[http://www.ublug.org/ubuntu/twinview/twinview-howto-breezy.html](http://www.ublug.org/ubuntu/twinview/twinview-howto-breezy.html)

Which worked! \\:D/

I changed the xorg.conf in a line by line manner, but the main problem seemed to be under device...

It starts of as nv but TwinView wants nvidia.

This seemed to be solved by getting nvidia-glx from synaptic instead of nvidia-settings -- plus additional restricted modules.

The above linked really helped because it explains what each line in the xorg.conf file does.

Amen and all that.

8-)

(btw. I have a nvidia quadro grpahics card, or something like that :))
