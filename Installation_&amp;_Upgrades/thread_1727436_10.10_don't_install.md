### Post by Hedgehog1 on 2011-04-13
mina123,

_First thing_ (and I am sorry I did not notice it sooner) is that you do not have enough RAM to run Ubuntu. You have **504 meg**, and Ubuntu needs **512 meg** minimum with a preferred level of **1 gig** for good performance.

If you were to load the lighter version **Lbuntu** that might work on your PC.

_Second Thing_ Your partition setup is a little odd.  You may be force to manually create the partitions for the **Lbuntu** install.

You current partition layout is:

[B]/dev/sda1 NTFS Windows XP 'C:' drive
/dev/sda2 extended partition where /dev/sda5 lives
__dev/sda5 NTFS Windows 'D:' drive[/B]

To prepare the disk for **Lbuntu**, please do this (after you have tested the 'try' option in **Lbuntu** really works):

1) Please defrag the 'D:' drive twice using XP (D: = /dev/sda5)
2) Boot from the Live CD, and resize the /dev/sda5 drive to be 15 gigs smaller using gparted
3) Using gparted, create a /dev/sda6 logical partition that is 14.4 gigs, file type: ext4.
4) Using gparted, create a /dev/sda7 logical partition that is the rest of the space (about 600 megs), partition type: 'swap'

The new layout would look like this:

[B]/dev/sda1 NTFS Windows XP 'C:' drive
/dev/sda2 extended partition where /dev/sda5 lives
__dev/sda5 NTFS Windows 'D:' drive
__dev/sda6 ext4 Ubuntu partition
__dev/sda7 Swap[/B]

The select the 'manual partition' option during the **Lbuntu** install, and make '**dev/sda6**' your '/' (root) partition.  Please install the boot loader on '**/dev/sda**' (not sda1 or sda6, just **/dev/sda**)
