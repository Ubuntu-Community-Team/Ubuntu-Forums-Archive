### Post by CatKiller on 2020-02-15
> **bwanaaa said:**
> Nvidia 1050ti with Linux kernel 4.15.0-50... I swapped out to a Nvidia 2070 super. 

New cards need new drivers. You need at least the 430 branch to have support for the Turing Supers, ideally newer than that. You being on such an old kernel doesn't suggest that you were using the newest drivers before you swapped cards.

You can boot from a live USB and chroot to your install to do package management, since you can't boot normally. Getting on the Hardware Enablement Stack, adding the graphics drivers PPA, purging your current Nvidia driver, and installing the 440 branch would be my recommendation. And remove the nomodeset option.

### Post by CatKiller on 2020-02-15
Good advice from Bashing- om. 

> **bwanaaa said:**
> the latest 440 branch driver is not available from the ppa. when i download it from nvidia and try to install it thusly  (after doing sudo telinit 3 and opening a root terminal w ctrl-alt-f1)

```
$ sudo bash NVIDIA-Linux-x86_64-430.50.run
```

gives me the following error

```
Unable to load: nvidia-installer ncurses v6 user interface
```

Three things:

The latest 440 branch driver, 440.59, is definitely available from the [graphics drivers PPA](https://launchpad.net/~graphics-drivers/+archive/ubuntu/ppa).

You've downloaded some random thing off some random website and tried to run it. Don't do that. The reason people do that on Windows is because they don't have anything better; having a package manager is probably my favourite thing about using Ubuntu. The file that you downloaded isn't particularly for end users, it's for package maintainers so that they can package it up for their distro, and that distro's users can then install that package through their package manager.

The driver you downloaded wasn't even for the 440 branch, it was the same 430.50 that's available in the standard repositories, without even having to go to the PPA for newer.


If things are all working for you now that you've reinstalled the older kernel, that's great.

If things stop working, though, and you need to tidy up, the way to do it would be to purge all of the different versions of the Nvidia driver and install just one - the latest one, for preference. The existence of other versions can prevent the installation of newer ones.

There's also a shim between the GPL kernel and proprietary driver that needs to be built for each kernel version - I suspect that it was reinstalling the kernel that caused the shim to be rebuilt, which then led to one of the driver versions working for you, in this case - but it's sensitive to the specifics of the build environment. You may find at some point in the future that the shim for a driver version can't be built unless you're on a HWE kernel; the maintainers try to not break that but, since it's a small team with finite testing time, it does sometimes happen, and is more likely as releases get older. Getting onto the HWE stack (which became the default for new installs from 18.04.2 onwards) is something to consider.
