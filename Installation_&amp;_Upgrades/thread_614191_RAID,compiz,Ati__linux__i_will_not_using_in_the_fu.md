### Post by BigBishop on 2007-11-15
I feel sad that you gave up.  The real problem is that Linux is too powerful and does what you tell it to.  Your problem with RAID is the fact that almost all on board RAID for desktops is what they call fake RAID.  Without installing some packages it was seeing your RAIDed drives seperately so when you install data is actually being writen to only one drive but when you are booting the bootloader is sees them as being RAIDed so it is trying to read off both drives so the boot fails.  I also have the same graphics card as you do and with some fiddling, and the latest updates, i got it working just fine but i have not tried Compiz yet.  If you want to try again here is the link to the post that will fix you RAID problem but be carefull Linux will do what you tell it to even if your not suposed to do it.  
[https://help.ubuntu.com/community/FakeRaidHowto](https://help.ubuntu.com/community/FakeRaidHowto)

Also i did some testing using my new PC to see how much gain i got from RAID 0 and it was minimal unless i was doing alot of I/O operations for an extended amount of time which if you have 2+ GB or RAM only happens when doing large file transfers.

If you dont try again enjoy Vista and not owning your computer.

BTW there are games for Linux and companies are now starting to develop more and more major title games for Linux.
