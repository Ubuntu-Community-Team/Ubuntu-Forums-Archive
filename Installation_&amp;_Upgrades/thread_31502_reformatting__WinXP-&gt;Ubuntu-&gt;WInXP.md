### Post by philipacamaniac on 2005-05-03
Firstly, the verb "format" means to "initialize a storage medium (hard disk, CD-RW, etc.) to hold data". To avoid confusion, make sure you use that term correctly.

Secondly, installing an operating system can be a very technical task, despite how easy we try to make it. If you aren't familiar enough with how to operate a computer, it probably wasn't the best thing to install Ubuntu over Windows. There is the option of dual booting, where you can choose to boot into Windows or Linux. That would have been a better option for you, it sounds like.

Thirdly, if your XP cd is not booting properly, it has NOTHING to do with what is installed on your hard disk, be it Windows, Linux or whatever. That being said, you haven't quite posted this question in the right forum, let alone the right website.

However, since this is an urgent request, I can try my best to help you out (since I am an A+ cert and am pretty familiar with Windows problems). First, go into your System BIOS (CMOS) and make sure the CDROM is first in the boot sequence. If it is, and the Windows CD still won't boot, then there is something wrong with the actual disc. Borrow a friends XP Professional CD and see if that works. If you don't have access to another Windows CD, and it is determined that your own CD is malfunctioning, the next step is to try and get familiar with Ubuntu, so you can get those finals done!

What do you need to get done? You have access to the Internet, and you have a word processor.

### Post by philipacamaniac on 2005-05-06
1. For smooth DVD playback, make sure DMA is enabled.

In a terminal, type **sudo nano /etc/hdparm.conf**
Then add the following lines at the end:

/dev/dvd {
dma = on
}


I don't know about transcode and dvd::rip; I'm not using them. Sorry. Maybe someone else can chime in?

2. mplayer and totem/xine are about the same. Is there a specific reason you want to switch to mplayer? The package dependency problems your experiencing are due to the fact that the mplayer package is in Multiverse, and apparently is linked to newer libraries than what is on your system. I'm looking into why that is right now. One thought: did you place the nerim.net repo at the end of sources.list? Because that is where it needs to be. Where a repo is placed in the file affects priority.

3. For multimedia keys, see [http://www.ubuntuforums.org/showthread.php?t=27039](http://www.ubuntuforums.org/showthread.php?t=27039). It also has a link to a Wiki page on the subject. You should be able to get that running fairly smoothly.
