### Post by Bluedog260 on 2007-01-10
I managed to find a solution :D :D 

Seems to be some issues with the nVidia drivers, so you do this:

1. Hit f6 on the menu, delete quiet splash from the boot parameter line, and add single. Single stops it loading x and a load more crap.

2. Once booted, type pico /etc/X11/xorg.conf. Find your video card, and change driver "nv" to "vesa".

3. Save (ctrl - o), exit pico (ctrl - x), then type startx. This starts x using the VESA drivers for video.

== Only do the rest if you want to load nvidia drivers for accelerated graphics / better performance (may as well).

4. Once ubuntu loads up, goto programs -> accessories -> terminal. Type /usr/sbin/synaptic into the terminal to start synaptic package manager.

5. Now following these instructions:

[https://help.ubuntu.com/community/BinaryDriverHowto/Nvidia?highlight=%28nvidia%29](https://help.ubuntu.com/community/BinaryDriverHowto/Nvidia?highlight=%28nvidia%29)

(I'm doing this from memory, but you will get the gist of what I mean).

In repository prefs, make sure all boxes are ticked (ie restricted packages are available). Close, click reload, wait.

6. Back in the terminal, type uname -r to see your kernel version, it's 2.6.17-10-generic.

7. Click search in synaptic, enter: linux-restricted-module. Select the correct one for the above set of numbers, you might have to right click on it and press upgrade/reinstall, but make sure you do it.

8. Now search for nVidia. Find nVidia glx and select install.

9. Press apply in synaptic.

10. Terminal again. Type nvidia-xconfig.

11. Press ctrl-alt-backspace to kill x, enter startx to restart x, and bam :)

### Post by aberry5555 on 2007-02-01
If it freezes midway through the splash I'd hazard a guess it's something as simple as the wrong graphics drivers :S I had this problem with my computer too. On start-up it will give you some boot options, I can't remember which F key it is, I think F6, will take you to boot options. Edit the boot options line, taking out "spash - quiet" and adding in "break=bottom" now go back and boot up the system. The splash screen won't load and you'll get a console line half way through the installation. when it comes up type in "chroot /root nano etc/X11/xorg.conf. This will open up a text editor so you can edit the graphics settings. Scroll down to the "device" heading. Under this it should say Driver on the left and which driver Ubuntu is attempting to use on the right, which is probably "ATI" Change this to "radeon" as these are the drivers it is supposed to assign (although Ive never tried them so, if it doesn't work, you might have to do it again but use a driver called "vesa" instead). Once you've changed it press ctrl+O then ctrl+X. This will bring you back to the command line. Type "exit" into the command line and, fingers crossed, ubuntu should boot!

### Post by aberry5555 on 2007-02-07
There is a command that automatically configures your xorg.conf file, so instead of doing the "chroot /root nano etc/X11/xorg.conf" idea boot up as normal (the non-working way) and press ctrl+alt+F1, then try this command:

```
sudo dpkg-reconfigure -phigh xserver-xorg
```

This will ask you to choose the make of graphics card, for which you should choose "nv".
Once This has run through and you get back to a blank command type "startx". If that doesn't work then Try the following:

to get the the accelerated graphics drivers would be something like "sudo apt-get install nv-glx" or something, but I don't know the exact package name for nvidia drivers myself, as I've never used them :S . See if you can use the standard or downloaded nvidia drivers otherwise I'm not really sure why it isn't working :S

***EDIT***

I found the right name for accelerated drivers: 

**sudo apt-get install nvidia-glx**

after you've installed it then running the configuration script should, hopefully, fill in the blanks on your xorg.conf file. The configuration file is this:

**sudo nvidia-glx-config enable**

> Anyone having this particular problem (black screen after splash, cannot ctr+alt+F1 to prompt) If you have an AGP card please try this.  Its what worked for me.  Change your AGP from 8x to 4x or lower in the BIOS.  I changed mine to 4x and Live CD booted with no problem able to switch back and forth from prompt to GUI.  I dont have time to finish installing tonight so I just shutdown and rebooted again just to make sure and it worked again.
