### Post by grahammechanical on 2015-01-18
Do you have hybrid graphics, such as Nvidia Optimus Technology? Or have you fitted an Nvidia adapter since you purchased this machine? With the monitor connected to the Nvidia adpater, you may need an Nvidia proprietary video driver which may have not been installed during installation of Ubuntu.

The Ubuntu install session uses an open source video driver, which is very good in normal situations but when there are difficulties with video adapters a version of the open source driver called Llvmpipe is used and it is not very effective. Its main purpose is for machines that do not have a video adapter capable of doing hardware accelerated 3D graphics. Which I do not think is your situation.

As an experiment try running the Ubuntu Installer with the monitor connected to the motherboard and then select Try Ubuntu. What user experience do you have? In this test you should be using the Intel graphic adapter and there is a driver for that built into the Linux kernel for Intel integrated graphics.

Regards.

Since you have Windows 8, did you install it or was it pre-installed. Or really is it UEFI or BIOS boot.
And then is UBuntu in UEFI or BIOS boot mode. It should be in same mode as Windows to easily dual boot.

Both nVidia and Intel need help to boot. Often nomodeset works for nVidia until you install the proprietary driver, which is in system settings, software, drivers tab.
And if using Intel you often need a boot parameter to configure correct X by Y or screen size, see several options in ubfan1's post below.

 At grub menu you can use e for edit, scroll to linux line and replace quiet splash with nomodeset.
