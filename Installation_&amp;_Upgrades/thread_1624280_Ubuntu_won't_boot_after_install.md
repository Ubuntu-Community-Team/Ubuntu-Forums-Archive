### Post by Quackers on 2010-11-17
Boot from the live cd and choose "try ubuntu" then when the desktop appears open up a terminal and type
```
sudo mount /dev/sdb1 /mnt
sudo grub-install --root-directory=/mnt /dev/sda
```
pressing enter after each line.
If no errors are reported run
```
sudo update-grub
```
Then reboot and see if the grub menu appears.

### Post by mikewhatever on 2010-11-17
I think this is the clue:
> => Grub 2 is installed in the MBR of /dev/sda and looks on the same drive in 
    partition #1 for (,msdos1)/boot/grub.
 => No boot loader is installed in the MBR of /dev/sdb


The boot loader should have been installed to the MBR of the laptop hdd.

Edit: Quackers is correct, the solution is reinstalling grub, but I think the second command should end on '/dev/sdb':

```
sudo mount /dev/sdb1 /mnt
sudo grub-install --root-directory=/mnt /dev/sdb
```

### Post by drs305 on 2010-11-17
> **imasynper said:**
> 
Where should I install grub now?

Grub2  is not installed on your current drive.

From the LiveCD:

```
sudo mount /dev/sda1 /mnt
sudo grub-install --root-directory=/mnt /dev/sda
```

Do not include the partition number in the second command.

Once you reboot to your normal install, run "sudo update-grub".

That should do it.
