### Post by kingshill on 2006-06-07
Firstly I have two machines a Laptop and a Desktop. I have installed Dapper with minimal problems in the Laptop and am using it happily. Sadly not the same for the Desktop. It is a fairly modern machine Athlon 2800 with a couple of diskdrives (several partitions) with loads of space. A Gig of RAM as well as a seperate CD writer and DVD writer. I initially tried the live CD which stalled at the mount root file system point. I tried also the server CD.

I then sought out Breezy which installed like a dream (oh that Dapper had the same installation) and I then changed the sources and did the upate and dist update bit to update to Dapper. This appeared to work, but when I re-booted it stalled at the mount root filing system. 

I then re-used the live CD and managed to force it into rescue mode and got a GRUB menu offering the Dapper Kernel and the Breezy Kernel, in both ordinary and recovery mode. The only thing I could get to work was the recovery mode of the Breezy Kernel, which evenually booted into life. Now when I reboot I get the GRUB menu with the four options (plus a Windows boot) but can still only boot into the Breezy Kernel which claims to be Dapper but is some sort of hybrid with some weird sound and video behaviour.

I now down loaded the alternate CD and tried to use it to re-install. Now it freezes at something like uncompressing kernel. If I use some of the options I can get it to freeze somewhere around the USB bits.

So I don't know what I have, I can't re-install, the live CD isn't. I think this installation is seriously flawed and has been reported as such for 2 months, by a significant number of people. Breezy is very easy so why have we moved so far backwards for Dapper?

I think Breezy should still be offered as the latest really stable release for people who have problems. This is not easy to figure out. In comparison Breezy occupied about 10 minutes of thought to figure out ias opposed to about 5 days of serious effort reading all the forums and trying all reasonable things. I am not prepared to trash my working XP installation for the sake of an upgrade to Dapper, by mucking around too much with hardware.

In summary Breezy works easily, Dapper works very well if you are lucky. It is a dire installation if you are not. The developers do not appear to have paid any attention to this problem. The only thing I can think of is to ignore Dapper and reload Breezy as the slightly bizzare installation I have now does not do sound and video is jerky.
