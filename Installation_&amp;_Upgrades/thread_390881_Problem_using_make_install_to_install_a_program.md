### Post by lloyd_b on 2007-03-22
>  Thanks for the reply Lloyd_B, but still no luck ...
Well, you *are* making progress.  The build system looks okay now.  You're just missing some required packages.
```
checking for PACKAGE... configure: error: Package requirements (gtk+-2.0 >= 2.0.0 libglade-2.0) were not met:

No package 'gtk+-2.0' found
No package 'libglade-2.0' found
```
This is pretty much telling you what it's looking for, though I suspect you'll need several "supporting" packages as well:
```
sudo apt-get install libgtk2.0-0
sudo apt-get install libgtk2.0-bin
sudo apt-get install libgtk2.0-dev
sudo apt-get install libglade2-0
sudo apt-get install libglade2-dev
```

First note: several of these may already be installed, so don't worry about it if you get a warning to this effect.

Second note: this may or may not be the end of the search.  Once you've met these dependencies, other missing dependencies may turn up.  That's why the whole packaging concept was developed - it's called "dependency hell".  Unfortunately, if you want/need to build from source, you have no choice. 
> The program says it requires pyGTK 2.8x or later, I have a package called Python-GTK2 and it's version 2.10.3-0ubuntu3, could this be the problem? All the packages I need appear to be installed
Make sure that you install the corresponding "-dev" package ("python-gtk2-dev").  Whenever you're installing things that you want to compile against, if there's a "-dev" package available, be sure to install it.

Lloyd B.
