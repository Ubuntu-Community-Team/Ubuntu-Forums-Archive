### Post by kidders on 2006-09-06
That kind of installing would lead to a tremendous amount of duplication and make it impossible to develop reliable (ie secure) ways of doing common things like HTTP, SSL, etc, etc.

There's no particular reason why dependencies should cause you much hassle though ... Ubuntu's package manager will happily tell you what they are in any given case. Of course, over time, as the number of packages on your system increases, you'll start to find that most common package dependencies will already be installed.

Try to bear with it ... the interdependent package structure is there for good reasons! :-)
