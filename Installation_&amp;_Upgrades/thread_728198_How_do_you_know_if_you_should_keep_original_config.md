### Post by bsmith1051 on 2008-03-18
When you upgrade your system it seems inevitable that the upgrade will ask you about keeping or replacing some of the configuration files.  How do you decide which to keep and which to replace?  

First of all, I am assuming that the system knows which ones have been modified from their original/default states and automatically replaces those with the new default version.

For the rest, my experience has been that most of them were modified by the system -- not manually by me -- and so I have no idea what's the best thing to do.

### Post by tgalati4 on 2008-03-18
If you don't remember modifying the config files yourself, then let the system replace them.  Only for certain installations and personal tweaking do you need to worry about it.

Whenever I make a manual change to a configuration file, I make a backup copy of the old file first.  Then I make a copy of the new, edited file to my home directory.  This way, if an update does mess up a configuration, I have the settings that I need from the previous file in my home directory.

So, as a general rule, replace them, but make personal copies of any files that you have personally edited.
