### Post by jrasmussen0 on 2007-10-19
You may still have a previous configuration saved at /etc/X11/xorg.conf.20071019 or something.  Run 'ls -l' and figure out what configuration you want to use and run 'sudo cp /etc/X11/xorg.conf{.20071019,}' to copy over the xorg.conf file.

### Post by muecker on 2007-10-24
Ok, it definitely is not my config that was the problem (other than the Files section is wrong).  The short answer is that I fixed it by upgrading to the 8.42.3 version of the fglrx drivers that were issued yesterday.  The long answer is much more interesting ;)

After trying almost everything I could think of I began to think the issue was the fglrx driver.  I tried to upgrade to the 8.40.4 driver, which was the current driver.  That was a huge mistake!  After that I could not get X to come up with the fglrx drivers even after downgrading to the 8.37.1 drivers installed by Gutsy.  I spent hours trying to get it to work and finally had to use the vesa driver just to get my machine to come up in anything except failsafe mode.

In the process I learned quite a bit about the new install.  Failsafe mode overwrites the /var/log/Xorg.0.log and backup files, however, there are logs in /var/log/gdm that have enough versions that I found logs from the attempt to start with my xorg.conf, which didn't help a whole lot.

I also discovered that the failsafe mode is controlled by files in /etc/gdm.  Specifically failsafeDexconf is used to generate xorg.conf.failsafe.  I switched the mode line to include resolutions I knew my monitor would support and could get failsafe mode at 1400x1050, which helped a lot.

The real breakthrough came yesterday when ATI released the 8.42.3 driver with support for Gutsy's version of xorg.  I installed the driver per the wiki and was up and running immediately.  I got an error about libGL.so.1 missing so I symlinked to libGL.so.1.2.  Unfortunately that is the Mesa GL driver and I have not yet found the ATI version.  So I'm once again using the above profile with dual monitor support but it isn't yet up to full speed.  I can use xrandr to switch resolutions on the fly and ATI's control panel works.
