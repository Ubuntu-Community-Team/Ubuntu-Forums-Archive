### Post by 504harry on 2005-12-27
I'm no expert but FWIW I understand the files on the live cd are as brief as possible - to  make it work reasonably quickly (given that CD's are relatively slow and you aren't providing any HDD space to augment RAM.
/
Whilst it might be nice to be able to say "yes please" and it installs "As Is" - you would find that some of the OS was missing, for example parts that install, or partition HDDs - you would forever be working with an "averaged-down" system......although I must say I tried GNOME as a Live CD and thought it's better than the Gnome that's mixed into Ububtu Distro. It may have had better instructions since the scope was less ambitious. Also you'd be burdened with Live-CD files that are no use.
/
Since there is no price to pay for the full distro (only time waiting for the disc to arrive) - it's not really a problem.....is it?
NOTE: Ububtu supplies a double-disk set: One for Live working and one for Full install.
By getting the "latest Issue" you are reasonably sure all fixes are included.
/////
As I have remarked (here) previously I would prefer Ubuntu to supply the second disc as **instruction**/helpful hints etc, so folk can save earlier settings (to floppy, say) and make better use of the Partition tools which are quite difficult to use.  [[I guess it's because it operates with only minimal software  (Like basic DOS)....so no graphical interface with the HDD shown as a Pie-Chart,    - A "live-style" CD could be used for this task (replacing the Live Option since most folk have made the choice to go "Full" ) - there could be a graphical interface to guide the installation onto the HDD ]] - it's not easy without a second CD-drive, but the clever folk at Ubuntu may be able, in time.
