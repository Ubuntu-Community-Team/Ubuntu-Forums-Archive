### Post by MWAAAHAAA on 2006-11-04

### Post by damien_vancouver on 2006-11-08
I awoke yesterday and updated, then later rebooted and found my ubuntu stuck in 640x480 mode.  When I went into preferences->Screen Resolution I could only select 640x480 with 60Hz.

Looking a little deeper, I found my /var/log/Xorg.log file was complaining in several places about not finding my nvidia card, and "dmesg | grep nvidia" showed a line about the nvidia module tainting the kernel.

After much fiddling, I got it to work by:

1) making sure "nvidia-glx" was installed in synaptic (somehow it had gotten uninstalled)

2) running "sudo nvidia-xconfig".  
Note: This should be all you have to do to re-enable the nvidia driver.  But for me, it broke X altogether and I ended up in text mode after a reboot.

I rebooted and was in text mode.  Doh!

3) running "sudo dpkg-reconfigure -phigh xserver-xorg"
this was suggested at the top of /etc/X11/xorg.conf.  When I ran this it asked me about which resolutions I wanted again, I selected all the ones I wanted (including 1600x1200 which is what I am really after).  

4) testing with:  "startx".  Sure enough X fired up.  It was still in 640x480 but now inside Preferences->Screen Resolution I was able to turn it back up to 1600x1200.

5) quitting X with "Log Out", which dumps you back to text mode, and rebooting with "sudo reboot"

6) after my reboot X came up in 1600x1200 just like before :)

I didn't see any forum threads mentioning my problems exactly so I thought I'd share what worked for me.
