### Post by Carlos Sevcik on 2009-03-04
After upgrading from Ubuntu 8.10, kernel 2.6.27-12-generic to Ubuntu 8.10, kernel 2.6.27-13-generic my HP dv8276LA laptop cannot boot. Booting with the old xxxxx-12-generic kernel solves the problem. 
Actually the boot process starts normally with the Ubuntu logo and the orange bar slowly growing from left to right through the screen, but after the screen goes blank as usually it it remains blank forever. There is no disk activity and it is not possible to exit to command line using ctrl-alt-f2. The machine has an n-Vidia graphic chip, don't know if there is a problem with graphic driver update for the new kernel but that might be the case.
Otherwise 13 is an unlucky number ;-).
