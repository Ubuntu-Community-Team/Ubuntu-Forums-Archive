### Post by taurus on 2007-05-03
And what filesystem is /dev/hdc6?  What's the output of this command from a terminal?

```
sudo fdisk -l
```
Otherwise, copy it to your own $HOME directory like

```
cp "/media/hdc6/linux software/magic-7.4.34.tgz"  ~/
tar xvzf magic-7.4.34.tgz
cd magic-7.4.34
sudo aptitude update
sudo aptitude install build-essential checkinstall
./configure
make
sudo checkinstall
```
If you get an error message after you enter one of the commands from above, you don't need to run the rest of the commands.  Instead, say which command and the whole error message.

### Post by psusi on 2007-05-03
I'm guessing you are trying to extract this on an ntfs partition, which does not support file ownership and permission information, or symbolic links.  Extract it to a linux filesystem, not a windows one.

### Post by taurus on 2007-05-03
Looks like you need both tcl8.4-dev and tk8.4-dev.

```
sudo aptitude update
sudo aptitude install tcl8.4 tct8.4-dev tk8.4 tk8.4-dev
```

Then, rebuild magic again to enable Tcl/Tk for it.
