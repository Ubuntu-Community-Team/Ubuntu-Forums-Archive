### Post by nerdtron on 2015-02-22
Let's try to narrow some options. (troubleshooting tips for the weekend project :D)
I don't think you have an exotic system that will not run Linux. 
So first make sure that the USB is bootable, a good confirmation is try it on another computer. If it boots linux from the flash drive, then your USB flash drive is good.

Next, be sure that the computer, BIOS is set to enable USB and boot sequence should use USB device first. Some motherboards treat USB drive as hard drives, so you may need to make the first boot device as USB-HDD. Other motherboards like mine have the option like pressing F12 that will make me choose which device on the computer I want to boot.

Next, if you were able to boot to USB but Linux doesn't load or is stuck, try to minimize peripherals/devices connected on the computer, say remove the CD drive and internal hard drive, just mouse and keyboard, and monitor connected to the computer.
 Just boot from USB and try LIVE mode. If you were able to get to the linux desktop, then reboot and start connecting the internal devices and other peripherals one at a time.

### Post by MAFoElffen on 2015-02-23
At the grub screen, press the <down arrow> once to see the instructions and so it doesn't go away (timer). Press the <up Arrow> key to return the the first menu option. Press what ever key the instruction say, to get into an edit mode. May be <E> or <Tab> depending...

When you get into the edit mode, go down to the line that starts with "linux". That is the kernel boot line. Go the the end of that line and append "nomodeset". Press <cntl><X> to boot...
