### Post by sir-marky on 2014-10-24
14.04 boots fine on my laptop with secure boot enabled.  Never had an issue with it.
14.10 will not.  The laptop complains there is no boot disc available.

I have tried updating 14.04 to 14.10 without sucess.
I have formatted the SSD and recreated the EFI partition (512MB), Swap (4096MB) and the root partition (200GB), formatted as normal to BTRFS.

A 14.10 USB installation stick will boot but but the SSD will not.

The BIOS settings are unchanged between versions.

I have now put 14.04 back on and all is well, but want to upgrade.

Have the secureboot certificates changed on this release?  Is BTRFS an issue, given the way Ubuntu creates the default subvols?

Can anyone help?

UPDATE - After trying many things I decided to check the information on the disc with Boot Repair.  The results are here, [http://paste.ubuntu.com/8675538/](http://paste.ubuntu.com/8675538/)
It seems 14.10 does not write a MBR to the disc to allow it to boot.  Sadly I can't get Boot-Repair to do this from the Live CD, and it won't boot on it's own for the same reason that 14.10 doesn't.  Does anyone have an easy way to repair the MBR?

### Post by sir-marky on 2014-10-29
This problem has now been fixed.
There is a raised bug at, [https://bugs.launchpad.net/ubuntu/+source/efibootmgr/+bug/1363719](https://bugs.launchpad.net/ubuntu/+source/efibootmgr/+bug/1363719) which highlighted the cause and solutions.
