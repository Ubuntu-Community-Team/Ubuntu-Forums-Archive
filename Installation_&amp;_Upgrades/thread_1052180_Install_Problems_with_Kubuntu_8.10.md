### Post by mikeah on 2009-01-27
The other day I made the momentous decision to begin the transition from Linspire, as that distro is now quite officially dead to the latest and greatest Kubuntu.  I really wanted to see the new KDE 4.1.

I've never ever have had an issue installing any version of Ubuntu on this home built machine.  SUSE, Xandros also work flawlessly on it.  It occasionally took a bit to get Linspire on it (that distro required a scrubbed HDD), although Freespire always worked perfectly.  It isn't a hardware issue. This machine was built for Linux.

I downloaded the Kubuntu 8.10 AMD64 Kubuntu from the OSL facility as well as the 32 bit version.  Also the text based installer for each of these. It would always go to a point in the install where the screen would screw up and the installation died. So I tried the text based version of the installer and on both the AMD64 and 32 bit CD and in both instances it came to a point where it asked for another CD. Of course it didn't release the CD tray and I don't know what extra CD it's asking for.

So, I gave up on 8.10 and downloaded Kubuntu 8.04 AMD64 version and it installed flawlessly, as expected.    

Note, I wrote all CD's with the same K3B on the same computer at the lowest write speed (8X) and verified the data.  On the 8.10 that let me get to that screen (preinstall) it checked the CD fro errors and found none.  It ain't the CD's.  There is something about Kubuntu 8.10 that is very different. 

Note that this system has swappable HDD's and I've always used it for evaluating OS's.  It's very rare that I have an issue with any form of Linux, but multiple burns of Kubuntu 8.10 from multiple sites could not be coaxed onto this computer.  Note that every version of Ubuntu/Kubuntu/Xbuntu before 8.10 has installed flawlessly before.

What's up with 8.10?
