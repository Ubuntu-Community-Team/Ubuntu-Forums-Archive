### Post by oldfred on 2015-06-17
You show grub in both the gpt protective MBR for BIOS boot and in the efi partition for UEFI boot. It looks like you now are configured for UEFI boot as fstab has an entry for mounting the efi partition.

Have you gone into UEFI boot tab and moved ubuntu entry to first in boot order?

What brand/model system.
Some vendors modify UEFI to boot by description and only valid description is "Windows". That is not per UEFI standard.
But we have work arounds. Many alternates, if only booting Ubuntu not Windows we can in UEFI change description from ubuntu to Windows, so it thinks it is booting Windows. Details are in link in my signature.

### Post by oldfred on 2015-06-18
HP's typical require work arounds. They have modified UEFI to boot only "Windows" description.

 HP to get into UEFI/BIOS menu - escape then f10 as soon as it starts.
[http://h10025.www1.hp.com/ewfrf/wc/document?docname=c01443329&lc=en&cc=us&dlc=en&product=5171079](http://h10025.www1.hp.com/ewfrf/wc/document?docname=c01443329&lc=en&cc=us&dlc=en&product=5171079)
It seems hp firmware do not allow you to boot anything other than windows. Hence no ubuntu option in the UEFI. To work around it
[http://ubuntuforums.org/showthread.php?t=2227889](http://ubuntuforums.org/showthread.php?t=2227889)
1) press esc key while booting to access start up menu 2) press F9 for boot devices menu.
