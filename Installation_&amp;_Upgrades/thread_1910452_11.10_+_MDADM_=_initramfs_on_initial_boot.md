### Post by bomski on 2012-01-17

My OS had been playing up so I decided to do a clean install; booted Ubuntu CD; reinstalled OS (with updates)...

Upon booting for the first time with the fresh install; the RAID array wasn't detected; so installed MDADM, rebooted...

Was presented with the 'loading operating system' message (I assume from GRUB), then nothing... just a blank screen... Hit Ctrl+Alt+Del to force reboot; was then presented with the GRUB loader (has the memtest options etc)... Selected the standard boot option; then recieved the initramfs CLI; typed 'exit' as i'd never seen that before and was worried about damaging the data on the RAID volumes... then proceeded to boot into the OS and i could see and access my RAID array...

After shutting the machine down, and powering back up this morning; the same thing occured... I've taken the liberty to leave the system booted and am in the process of running a repair check on the RAID array (6TB volume so it's gonna be 24 hours from now)...

Has anyone got any suggestions as to what I can do as further steps if the check and repair process does not resolve this?
