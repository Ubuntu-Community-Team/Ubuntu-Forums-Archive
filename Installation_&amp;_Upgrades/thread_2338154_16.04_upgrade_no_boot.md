### Post by gennarof on 2016-10-17
Actually I realized that with the kernel 3.19.0-xx-generic upstart  everything works fine (except for a very slow log out).
At boot, you should have for 1 or 2 seconds a screen with the list of  kernel you can use, to stop it, enough to press for example the down  button.
If you do not have the kernel list, I think there is a key  (maybe F5) or a combination of keys.

  The explanation, if I well understood, is that there are now 2 boot  tecnologies, the old "upstart", the new "systemd", the kernels following  3.19 (4.4...) use systemd only, the 3.19 and before make you choose.
In the upgrade from 14.04 to 16.04 program there is an error and you can  boot only with 3.19... upstart.
If you absolutely need/want to use the  latest kernel, for the moment, I found around as solution only reinstall  : (
