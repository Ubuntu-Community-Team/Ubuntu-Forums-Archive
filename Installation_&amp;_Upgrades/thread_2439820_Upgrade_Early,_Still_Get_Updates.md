### Post by guiverc on 2020-04-02
The 'daily' builds (of which there are many, eg. 18.04.5 is one daily, *focal* is another ... continue as normal. The *focal* release is the development build (it'll get renamed 20.04 at release time)

One daily gets named 'beta' and exists as 411 on the iso.qa.ubuntu.com site.

I haven't downloaded any beta, but I am recorded as as doing a number of tests on it, because I just continue with 408/*focal*-daily and the system records my normal 'daily' testing as 'beta' because for awhile they're identical. If problems are detected and a new 'daily' is spun, they no longer are the same but the beta remains the same.  If you look at link I provided ([http://iso.qa.ubuntu.com/qatracker/milestones/411/builds](http://iso.qa.ubuntu.com/qatracker/milestones/411/builds)) you'll note the version field on the right, which is the date of the ISO. You can see some respins in the Ubuntu Server image (where the version ends in a .1)

To me I see difference between daily (there are many dailies, eg. 18.04.5 which isn't a development release any more), but yes a development (*focal*) build gets given the name "*beta*". Sorry if this is unclear.
