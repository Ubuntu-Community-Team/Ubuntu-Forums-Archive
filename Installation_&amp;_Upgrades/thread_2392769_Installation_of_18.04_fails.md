### Post by ubfan1 on 2018-05-25
Any desktop release will come with a GUI.  Of course, you could start with a server, add X, add a GUI, add ..., but why not just start with the desktop release.  I think of a server as a box without a monitor, without a keyboard, and providing something like web or database on a network port.  If you have a really limited machine incapable of installing a desktop release, I suppose the server might still fit, and be used for things like command line complies, etc.  You would access the server via another machine running ssh.  What are you trying to do?  The desktop release fits most people's needs.
