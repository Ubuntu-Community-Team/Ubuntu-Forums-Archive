### Post by 69cruiser on 2011-07-06
Hello,

I am running Windows XP Pro SP3 and it is currently setup on a system running 2 SCSI drives with an adaptec 2940 UW (non RAID) PCI card  both are 18.4 GB. Windows XP is installed on the first one and I am trying to install Ubuntu 11.04 onto the second, but the installation doesn't recognize that windows is installed on the first drive and only gives me that drive as an option to install... Gparted recognizes both, one as an NTFS file system (sdb) and the second as a unpartitioned drive (sdc) and for some reason is placing the usb drive as sda which i'm assuming might cause some issues with grub boot loader later on after the drive is removed. any ideas on what might be causing this issue and how i would go about fixing it... should i just go to 10.04 and upgrade to 11.04 after?

### Post by oldfred on 2011-07-06
Were the drives ever in RAID. RAID setting puts meta-data on drive that can interfere with install.

If USB is seen as sda, you will have to do manual install to make sure that you can choose which drive to install grub2's boot loader to. It does otherwise default to sda.

Presence1960 on remove old raid setting from HD
[http://ubuntuforums.org/showthread.php?t=1325650](http://ubuntuforums.org/showthread.php?t=1325650)
sudo dmraid -E -r /dev/sda
sudo dmraid -E -r /dev/sdb
Also check BIOS for raid settings
More discusion:
[http://wwww.ubuntuforums.org/showthread.php?p=9274738#post9274738](http://wwww.ubuntuforums.org/showthread.php?p=9274738#post9274738)

### Post by YesWeCan on 2011-07-06
Hi there,
Using separate disks for Ubuntu and Windows is a good idea. Make sure the Grub boot code is put on the Ubuntu disk.

What oldfred says plus:
You will find the installer on the "alternate" ISO more sophisticated than the regular one. Worth trying it before you revert to 10.10 (you need 10.10 to go to 11.04).

Alternatively, disconnect the Windows disk while you do the Ubuntu install. After installing Ubuntu, reconnect the Windows disk and boot Ubuntu and run 'sudo update-grub' to add Windows to the boot menu.

### Post by 0DarkKing0 on 2012-06-17
I experienced the same problem on windows XP SP 2.

In my problem:
I checked the disk managment and had 4 partition:
-primary 100 GB
-secondary 100 GB
-Free 5 GB
-Free 3 GB

I was suspicious of the free partition so I deleted and made it into:
-primary 100 GB
-secondary 100 GB
-unallocated 8 GB

I reboot and install ubuntu, it finally detect my windows XP SP 2
