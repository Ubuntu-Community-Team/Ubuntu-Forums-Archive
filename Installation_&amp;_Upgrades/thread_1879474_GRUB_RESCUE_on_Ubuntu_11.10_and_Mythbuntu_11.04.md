### Post by darkod on 2011-11-11
From live mode did you try fsck on both root partitions (/dev/sda2 and /dev/sda4)?

You should not mount them first, just run:
sudo fsck /dev/sda2

Another thing that might be the problem, /dev/sda4 is reported as BIOS Boot partition which it obviously is not. I wonder if you need to use testdisk to change the partition type to linux partition, not bios boot.

### Post by darkod on 2011-11-11
Run the bootinfoscript again and check whether sda4 is still reported as BIOS Boot partition.

I guess as long as it is, it will be ignored as a root partition which it should be.

You can try and see what a testdisk scan says about it. Usually it should pick up that it is ext4 partition and offer to change its type.
[http://www.cgsecurity.org/wiki/TestDisk](http://www.cgsecurity.org/wiki/TestDisk)

PS. Be careful with testdisk, it's a very powerful tool.
