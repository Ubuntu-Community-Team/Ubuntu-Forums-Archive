Alright then, I'm satisfied that something weird is going on there and it probably will be simpler to just set up chainloading.

Here's how to set up your chainloader entries,

1. The following command should create for you an empty file which we will copy and paste some commands into in a minute,
```
gksudo gedit /etc/grub.d/06_custom
```2. Here are the commands you can copy and paste into the empty file,
```
#! /bin/sh -e
echo "Adding custom boot entry(s)" >&2
 cat << EOF
menuentry "Chainload MBR second hard disk" {
    set root=(hd1)
    chainloader +1    
}
menuentry "Chainload MBR third hard disk" {
    set root=(hd2)
    chainloader +1    
}
EOF

```3. You will need to make the new file executable, ```
sudo chmod 755 /etc/grub.d/06_custom
```4. Now we will update your GRUB menu,
```
sudo grub-mkconfig -o /boot/grub/grub.cfg
```Now you can reboot and see if it works. If you have Mandriva's GRUB installed in your second MBR, choosing to chainload the second MBR should boot it, otherwise try booting the third MBR.

### Post by Herman on 2009-12-26
```
#!/bin/bash
#  script for setting up GRUB to chainload any other hard disks 

sudo echo "#! /bin/sh -e
echo "'"Adding custom boot entry(s)"'" >&2
 cat << EOF
menuentry "'"Chainload MBR second hard disk"'" {
    set root=(hd1)
    chainloader +1
}
menuentry "'"Chainload MBR third hard disk"'" {
    set root=(hd2)
    chainloader +1
}
menuentry "'"Chainload MBR fourth hard disk"'" {
    set root=(hd3)
    chainloader +1
}
menuentry "'"Chainload MBR fifth hard disk"'" {
    set root=(hd4)
    chainloader +1
}
EOF" > 06_custom
sudo cp -av 06_custom /etc/grub.d/
sudo rm 06_custom
sudo chmod 755 /etc/grub.d/06_custom
sudo grub-mkconfig -o /boot/grub/grub.cfg

```Alright, so now we have a simple script that we can give people to run if they want to have entries added in their GRUB 2 for booting a few extra hard disks, (providing of course they have a boot loader installed to MBR in the other disks), and some people might also find it useful if they want too an external USB drive. It should work equally well for that.
 
In case anyone doesn't know how to run a script yet, all you do is copy and paste the text from the above text box to a blank text file, and save it to your hard disk.
Then run 'sudo chmod 755 chainloader_setup.sh', (that's to give it the file permissions to make it executable),
```
sudo chmod 755 chainloader_setup.sh
```Where: you named the file 'chainloader_setup.sh', you can call it anything you want, but that's what I call mine.

Now you should be able to double-click or right-click on the file and select either 'run in terminal', or 'display' if you only want to look at it or if you want to make some changes to it to adapt it to your own particular needs.

Alternatively, you could run the script by opening a terminal and typing,
```
./chainloader_setup.sh
```

### Post by Herman on 2009-12-26
Oh, and if somebody wants to remove the MBR chainloader entires again at some later date, all they need to do is remove the 06_custom script and re-run grub-mkconfig,
```
sudo rm /etc/grub.d/06_custom
``````
sudo grub-mkconfig -o /boot/grub/grub.cfg
```That will make a new grub.cfg without the extra chainloader entries.
