### Post by scottvance on 2012-08-14
I was having the same problem recently, and tried everything to fix it.  I changed sources, servers, settings, cleaned out cached files ... all manner of things to no avail.  After searching and experimenting for several weeks I finally found the answer.  There is a bug that leads to a missing directory in the installation of perl.  The directory /usr/lib/perl/5.10.1/asm is missing.  See this bug report:

[https://bugs.launchpad.net/ubuntu/+source/perl/+bug/777903](https://bugs.launchpad.net/ubuntu/+source/perl/+bug/777903)

I don't know if this problem impacts all uses of apt, but it impacts apt-cacher and makes it unable to update.  The simple solution is to restore the missing directory.  A kind user posted a zip file with the directory contents to this thread:

[http://ubuntuforums.org/showthread.php?t=1811040](http://ubuntuforums.org/showthread.php?t=1811040)

As soon as I extracted the contents and restored the missing asm directory, all updates and upgrades began functioning again.

This is a pretty old question and the answer is out there if you can find it, but since I see dozens of people searching for an answer, and some very recently, I figured I would post what worked for me.
