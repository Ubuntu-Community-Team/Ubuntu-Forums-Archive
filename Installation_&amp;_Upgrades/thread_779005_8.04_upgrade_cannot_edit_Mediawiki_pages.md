### Post by BryanKeith on 2008-05-02
After upgrading to Hardy Heron 8.04, I am unable to edit Mediawiki pages.  Here is the message I get when I try to edit a page:


Fatal error: Allowed memory size of 16777216 bytes exhausted (tried to allocate 7680 bytes) in /var/www/wiki/includes/SpecialPage.php on line 596

Sometimes the line number is different depending on which page I try to edit.

Maybe it's a php problem?  Any ideas?

Bryan

---

### Post by BryanKeith on 2008-05-02
Solved.  I had to up the memory_limit in php.ini from 16MB to 32MB.  Then I had to restart apache.

grep -n memory /etc/php5/apache2/php.ini
270:memory_limit = 16M      ; Maximum amount of memory a script may consume (16MB)

I doubled that to 32MB.  To restart the server:

sudo /etc/init.d/apache2 restart

Now things are working again.  Hope that helps someone.

Bryan
