### Post by GuiGuy on 2009-06-05
48 hours ago my mythbuntu box wouldn't reboot. I was getting ata:Softreset failed (device not ready) errors, [something akin to this bug]("https://bugs.launchpad.net/ubuntu/+source/linux/+bug/285392"). The startup then bombed with a "gave up waiting on boot device" errors. I decided to do a clean install. 

ata:Softreset failed (device not ready) errors. No cursor when it halts. So, during the install or boot process one cannot get past this point in the install or on a reboot. 

Faulty hardware I thought. Mucked about, googled, learnt a bit more, made sure I had the latest BIOS (ASUS M3A), replaced the boot drive with one containing WinXP and tried again. WinXP boots, no problems. 

Tried the Mythbuntu install again on a new HDD; ata1:Softreset failed (device not ready) 

Tried a kubuntu install; ata1:Softreset failed (device not ready) 

Tried a vanilla ubuntu install; ata1:Softreset failed (device not ready) 

Downloaded MythDora. Installed it, no problems. 

So time to move on, I think. Unless someone knows a workaround?

**NB: I just installed mythbuntu 8.04 AMD64. No problems and SATA performance is perfect!! I have to conclude that jaunty is a step backward.**
