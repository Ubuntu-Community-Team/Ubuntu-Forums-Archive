### Post by KellanStec on 2010-05-01
Greetings,

I am having some issues with an upgrade I performed.

I was running a hardware RAID 1 mirroring on two 1 TB drives on my ASUS M2N motherboard in version 9.10. All was working fine.

I upgraded to 10.04 today, and suddenly the RAIDed device that was presented to the operating system is no longer available, but I do see both devices listed as mountable, and all of my data is on them. They both have the same uuid.

I noticed on boot after grub does its thing, I see an error on the screen for less than a second:

```
nForce2_smbus 0000:00:01.1: Error probing SMB2
```

I see the following in /var/log/dmesg:

```

[   33.866002] i2c i2c-0: nForce2 SMBus adapter at 0x1c00
[   33.866006] ACPI: resource nForce2_smbus [0x1c40-0x1c7f] conflicts with ACPI region SM00 [0x001c40-0x001c45]
[   33.866008] ACPI: If an ACPI driver is available for this device, you should use it instead of the native driver
[   33.866011] nForce2_smbus 0000:00:01.1: Error probing SMB2.
```

I am wondering if these are related somehow, but am unsure where to start looking.

I did verify that RAID is still configured in my BIOS, and it seems to be working just as it was before the upgrade. For some reason, Ubuntu is not seeing the RAIDed device presented to it by the hardware, and is instead seeing the devices individually.

### Post by KellanStec on 2010-05-02
I was able to get this one knocked out on my own. 

The kernel included in 10.04 does not include the dm_raid45 module by default.

I installed it by issuing the following command as root:

```
aptitude install dmraid
```

After a reboot, it worked just as it did before the upgrade.
