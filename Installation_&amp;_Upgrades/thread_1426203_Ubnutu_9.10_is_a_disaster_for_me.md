...

In the end, I had to format hard drives and re-install 9.04.


This "disaster" might have been avoidable if you'd taken the extra time to boot from a LiveCD to see what then happened.  I'm not saying it would have revealed all the problems you later encountered, but it certainly would have revealed some of them.

Another approach (one that I and several others use) is to install the latest Ubuntu version into a different partition such that, in the event of such "disasters", it's a simple matter to wipe the new partition and go back to the previous version.  That way, you don't run the risk of losing anything.

Some things to think about when you go to trying 10.04 the end of April ...

### Post by presence1960 on 2010-03-10
+1 for mark phelps.

I have not had any problems with 9.10 on my machines nor those machines I installed it on for clients.

It is also my observation that a lot, but not all of those who bash each release as it comes out are usually the ones who do not troubleshoot or file bug reports but rely on complaining and whining.

Now to be fair I have to say that some who have trouble with 9.10 definitely do not fall into that category.

But ubuntu is like any other OS- it works for most but has it's problems with certain hardware or configurations.

I always do what mark suggests. I install the "new" release to it's own partition and "test" it. If I need to trouble shoot there is no rush as I still have a working "prior" release on a different partition to boot into if need be.

A little thought and planning can go a long way in preventing the need to panic if the new install does not work properly.
