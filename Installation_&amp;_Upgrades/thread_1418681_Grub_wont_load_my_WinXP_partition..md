### Post by buntudawg on 2010-03-01
There is a possible solution in the troubleshooting section of Ubuntu user magazine:

Boot from the live CD and open the terminal.

You already know Ubuntu is mounted  at /dev/sda5 so mount it:

[FONT=Courier New]sudo mount /dev/sda5 /mnt
sudo mount --bind /dev /mnt/dev
sudo mount --bind /proc /mnt/proc[/FONT]

[FONT=Verdana]Change the active root directory from the sda5 drive with chroot:

[FONT=Courier New]sudo chroot /mnt[/FONT]

Then you should be able to repair grub (you may need to use sudo but probably not):

[FONT=Courier New]grub-install /dev/sda[/FONT]

If you get errors run the following,[FONT=Courier New]
grub-install --recheck /dev/sda[/FONT]
else move on to the next step
[FONT=Courier New]exit
sudo umount /mnt/dev
sudo umount /mnt/proc
sudo umount /mnt
sudo reboot

[FONT=Verdana]after you reboot into Ubuntu re run update grub2:
[FONT=Courier New]sudo update-grub2

[FONT=Verdana]Dunno if this will work (never tried it) but it might be worth a try if no one else has a better idea. Good Luck.[/FONT]

### Post by meierfra. on 2010-03-01
> OK I think if you can boot into Ubuntu those results are somehow fault
No, they are not.   The windows MBR loads the boot sector of the  partition with the boot flag.  The boot flag is on /dev/sda1. Grub is installed in the boot sector of /dev/sda1  and so Grub takes over the booting.

I don't have time right now for  detailed instructions. But this is what needs to be done:

1) Install Grub to the MBR of /dev/sda

2)  Fix the boot sector of the /dev/sda1 (either with "fixboot" or testdisk)

3) Put the boot flag on /dev/sda3 and run "startup repair" from the Window7 DVD( the boot files for Win 7 are missing)

### Post by meierfra. on 2010-03-01
Here are the details:


Boot into Ubuntu.

[B]Step 1  Install Grub  to the MBR of /dev/sda:
[/B]

```
sudo  grub-install --recheck /dev/sda
```

** Step 2 Put the boot flag onto /dev/sda3**

```
sudo sfdisk -A3 /dev/sda
```


[B]Step 3  Fix the boot sector of your XP partition
[/B]

```
sudo apt-get install testdisk
```
(if "apt-get" does not find  "testdisk", you need to enable the "universe" repository in "System->Administration->Software Source" and then run "sudo apt-get update" to load the new packages)


Choose "proceed" on the first screen, then
"intel"
"advanced",
Select the XP partition (although it should be selected already) and choose
"boot"
"BackupBS"
Type "Y" to confirm
  
then press "q" a few times to quit  testdisk, reboot and see whether you can boot into XP.

Once you successfully booted into XP you can also try to 

**Step 4 Fix Window 7.**

Boot from the Window 7 DVD (If you do not have one, you can download a  Windows 7 CD from [here]("http://neosmart.net/blog/2009/windows-7-system-repair-discs/"))

At the first screen select your favorite language.
At the second screen choose "Repair your Computer".

If a pop up  appears, offering  to repair a problem with the "Startup options",   decline.

On  the  next screen select "Use recovery tools ..."  and click on "Next".

Click on "Command Prompt"

and type

```
bootrec /fixboot
exit
```

Click on "Startup Repair"


"Startup Repair" tends to fix one problem at the time. So you might have to run "Startup Repair" several times.

After running startup repair, boot into Ubuntu and run

```
sudo update-grub
```

reboot and see whether you can boot into Windows 7.


If you run into problems anywhere along the way,  report all error messages you got, run the Boot Info Script, and post the new RESULTS.txt (it might be called RESULTS1.txt)
