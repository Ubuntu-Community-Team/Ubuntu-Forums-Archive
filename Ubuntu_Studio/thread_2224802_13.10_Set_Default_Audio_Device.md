To find out cards number use
```
cat /proc/asound/cards
```
.asoundrc is a configuration file in $HOME directory. You need to make it.
Inside, put the code I've supplied.
This configuration is user based, it is valid for all applications started in that user session.
I don't now what kind of computer you use, so, regarding of disabling one of the sound card you can physically remove it, or unload kernel module for it. The second one can be tricky, if both of them uses the same driver.

Linux doesn't need/uses extensions, if it does - it is just for fun.

### Post by CraigPid on 2014-05-20
I know you probably wouldn't want to try this but if anyone else reads this I've found that the most reliable way to fix this would be to compile a new kernel and build the driver for the default card into the kernel and the other card as a module.That way the one built into the kernel always loads first and takes slot 0. If they're both built as modules then you never know which will load first.
There was another method I used. There's a config file named alsabase.conf I believe it's in /etc/modprobe.d. If you can find out the names of the modules that the cards use you can tell them which slot to use.Look at the Alsa ArchWiki.About halfway down there's a section on setting the default card and it tells exactly how to do it.
[https://wiki.archlinux.org/index.php/Advanced_Linux_Sound_Architecture](https://wiki.archlinux.org/index.php/Advanced_Linux_Sound_Architecture)

In some distros it causes strange behavior so that's what led me to compiling the default card driver into the kernel. If I recall though, I'm pretty sure the alsabase.conf method works in Ubuntu.
