On the same hand, if all you want to do is complain or tell other people to do things, then you can't always expect those people to do what you want.


[SIZE="3"]Upgrade bugs are ***very*** common and ***very*** useful.[/SIZE]  In fact, before any package can be upgraded directly in Ubuntu, an upgrade bug needs to be filed (then a patch is accepted).  The only other way packages can be upgraded in Ubuntu is during the upstream imports from Debian.

So please, choose one or more of the following options:
[LIST]
[*]tell us what software didn't work (hopefully with a bug report)
[*]tell us what software is out of date (hopefully with a bug report)
[*]understand that developers aren't machines, and be happy with the most up-to-date version they've been able to package for you
[*]quit creating pointless noise on these forums (where nearly no developers read due to the volume of noise)
[*]file a request for deletion bug on the packages that are SEVERELY out of date and dead upstream
[/LIST]

I really don't mean to have a rude tone to this post, and I'm sorry if it comes off that way.  Hopefully we can help you find a usable video editor in the repositories (have you tried PiTiVi, Kino, or Blender?), or maybe we could find out which packages you're even requesting to be deleted (all three of those video editors were updated for Karmic's release).

Developers really do value feedback from users, but it needs to be done in at least a partly constructive manner.  Negative rants as feedback only tire developers out and make them walk away from open-source projects.  Negative criticism is another story, this can be the best type of feedback as it helps us fix serious problems, but please be cautious with crossing the line into rant territory where statements become outlandish for the sake of theatrics and the central point of the criticism is lost in an excess of blanket statements.

### Post by Madspyman on 2009-11-11
Video Editors in the repo's for Ubuntu.

Blender - Best editor Ubuntu has to offer, very HD friendly, nice export options, but not listed in The repo's as a video editor.

Pitivi - Works, easy to use, HD friendly, plenty of export options (gstreamer), but has no  available transitions/effects. Great effort, but  unless you're Micheal Bay, without transitions/effects this is worthless.

Kdenlive - HD friendly, but very slow preview playback. Must work better for KDE users, cause it's very bloated in Gnome, very annoying to test.

Open Movie Editor - Randomly crashes, no real project setting options, preview playback doesn't seem to work.

Kino - Good for import from an outdated camera, not very HD friendly, poor export options, more of a trimmer than editor, home video from the early 90's worthy, not for serious filmmakers.

Lives - Radomly Crashes, seems to have sound issues (caused by pulse), very confusing interface, seems HD is supported, but it's not very HD friendly.

Avidemux - Not yuv color friendly, has h264 issue's, sometimes video is imported upside down, not much of an editor more of a trimmer, good export options.

Project X - Import errors, couldn't test it.

VideoCut - Randomly crashes, couldn't get it to work.

This is based on my experience, but Blender seems like the only video editing app for Ubuntu that works, but the version in the repo's is older and stripped. For ffmpeg support, I had to download the latest version for Ubuntu, from Blender's website.

Editing and exporting Video properly in Ubuntu is possible, just not with any of the programs in the repositories, it's embarrassing.
