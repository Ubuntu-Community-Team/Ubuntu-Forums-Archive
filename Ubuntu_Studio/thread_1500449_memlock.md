### Post by Pablo_F on 2010-06-06
I am late here but:

> So if lucid is using this second file (/etc/security/limits.d/audio.conf), what good are the Ubuntu Studio controls? is this a bug that should be reported?

See:
[https://bugs.launchpad.net/ubuntustudio/+bug/571688](https://bugs.launchpad.net/ubuntustudio/+bug/571688)

On the other hand, Jackd post-installation script writes memlock unlimited whereas jack messages say this is dangerous and recommends some amount of your RAM in kB (75% if I am not wrong). This should not be changed through ubuntustudio-controls which writes to the deprecated limits.conf file. 

In any case, how to check the actual limits for the user? Use "ulimit":

$ ulimit -r (prints rtprio)
$ ulimit -l (prints memlock)

Cheers, Pablo
