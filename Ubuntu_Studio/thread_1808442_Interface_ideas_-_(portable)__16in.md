### Post by jeebustrain on 2011-07-20
I'm looking to build a portable rig for live mixing/recording my drumkit/band. In the windows/mac world it'd be easy - get a decent 8in interface with built-in preamps and add a 2nd 8 channel preamp via spdif. Blammo - 16 preamped inputs. The linux world is a little more complicated. I know if I went the desktop route, I could use a pair of delta 1010s (with separate preamps), but I'm looking to run this off a laptop via firewire (or USB if it can handle it). I've been checking out interfaces on the FFADO website and I haven't found anything in the "fully supported" category that seems very promising. 

In an ideal world, I'd like to use a Focusrite Saffire 40 w/ a 2nd set of preamps, but I've read mixed results on that particular interface (though some of the older ones apparently work - but I can't find any of those on Ebay or anywhere else). I'm a linux user to the core, but I have a feeling I might end up having to run Windows on my laptop in order to make this happen reliably.

Has anyone been able to accomplish this feat?

### Post by sgx on 2011-07-22
While not a specific hardware choice, these 41 pages might be central to your end result:

[http://www.harrisonconsoles.com/mixbus/Mixbus_Install_QuickStart.pdf](http://www.harrisonconsoles.com/mixbus/Mixbus_Install_QuickStart.pdf)

I would not recommend a laptop, as you can fasten a handle to a desktop case, and bracket an lcd to one side of the case, and use the 1010s, for which there is documented success.

Check the used gear search from Guitar Center, all their stores have their many trade-ins listed. You might find some firewire/usb devices.

### Post by trulan on 2011-07-23
I use a laptop and two PreSonus Firepods (daisychained via firewire) for exactly this type of setup.  While they are not officially supported by Ffado, the chipset type they use was among the first to work in Linux, back in the FreeBoB days, before Ffado even existed, and they work perfectly.

Linux, when properly configured, blows Windows out of the water for this kind of thing, at least in my experience.  The only downside is, you need to generally avoid the latest-and-greatest hardware, as it generally takes a year or two for a device to be supported in Linux.

### Post by trulan on 2011-07-25
I'm tracking to the internal drive, it is a 7200rpm SATA II drive.  My old laptop had a 5400rpm SATA I, and that worked too.  I've seldom recorded all 16 tracks simultaneously, usually 11 or 12 are enough for what I do.   I've never tried overdubbing that many tracks at a time either, overdubs for me are always only one or two tracks.  The only time I've seen the hard disk unable to keep up was on the old laptop, running on battery power.  In hindsight some ACPI tweaking probably could have fixed that too.
