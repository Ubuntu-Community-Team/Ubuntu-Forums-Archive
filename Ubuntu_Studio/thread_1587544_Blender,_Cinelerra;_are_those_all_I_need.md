Hello,

After investing several months to overcome the linux learning curve both as a OS and in what concerns audio, I can say that I'm very comfortable now. Now I know that proffessional sounding music can be done in linux, and audio manipulation as well.

Now I want to move to the second phase of my "plan" and that's video. I've been playing around with avidemux, kino, openshot... but I've never spended much time on them. As I said before I'm just beggining the research on linux video, but I think I'll be spending lots of time in blender's forums, and learning about this software. The point of this post is to have some opinions from anyone that's on the same position as I am now, or advice from people who has already overcome this issue.

Thank you

### Post by linuxpyro on 2010-10-22
Glad to see Linux pro-audio's working out for you!  I know what you mean about Ardour and Jack, they are great applications.  

Cinelerra is very powerful but be warned, it can also be unstable.  In my experience it is not too uncommon for it to crash if you try to load a video file that is not supported.  However, if you pay attention to what causes this it's not that hard to use it for getting things done.  

Blender is neat as well, but if you're looking for another program to go with it, you can check out [Ramen]("http://ramenhdr.sourceforge.net/"), an open source compositor.  (Blender has one built in, but I don't think it does quite as much.)  I'm keeping an eye on it; I've used Apple's Shake (closed source) before on Linux, and having a free software alternative is nice.  Not sure what you're looking to do with video, but you might want to play with it.
