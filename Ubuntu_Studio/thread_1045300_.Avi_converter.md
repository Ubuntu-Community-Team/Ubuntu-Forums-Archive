### Post by javierrivera on 2009-01-20
I usually use Avidemux to convert videos. It's probably too complex for my needs but I'm used to it.

Here is a quick how-to:

1) Open the video with Avidemux
2) On the left, there is a selection box under a the label "Video", it should have "Copy" selected. Select the format that you want (MPEG-4 should work).
3) A button labeled "Filters" under the box should be activated. Click it. A filter list should appear, choose one with the word resize. Select some resolution that is appropriate for your iPod.
4) Hit save.

I remember that before I stumble into Avidemux I used some other simpler utilities, but I finally settled for it (a custom mencoder script was my older bet ;). It works quite nicely.

### Post by andrew.46 on 2009-01-24
Hi aadipotter,

> **aadipotter said:**
> is there a converter to convert .rmvb files to .vob(dvd format) for ubuntu 8.10?

MEncoder does not create vobs as such but it can create a a valid mpeg2 stream from a rmvb file which you then use to create a dvd with the dvd authoring software of your choice. The encoding details for MEncoder are pretty much laid out [here]("http://www.mplayerhq.hu/DOCS/HTML/en/menc-feat-vcd-dvd.html"). I have not done this before so I downloaded [a sample file]("http://samples.mplayerhq.hu/real/AC-cook/cook_5.1/hotel_california_ra5.1_640x480_30s.rmvb") which is made up of:

```

Input #0, rm, from 'hotel_california_ra5.1_640x480_30s.rmvb':
Duration: 00:00:30.18, start: 0.000000, bitrate: 1240 kb/s
Stream #0.0: Video: rv40, yuv420p, 640x480, 1054 kb/s, 12.00 tb(r)
Stream #0.1: Audio: cook, 44100 Hz, 5.1, s16, 183 kb/s
Stream #0.2: Data: 0x0000

```

and converted it to a format suitable for dvd as per manual (**PAL DVD**):

```

mencoder -oac lavc -ovc lavc -of mpeg -mpegopts format=dvd:tsaf \
-vf scale=720:576,harddup -srate 48000 -af lavcresample=48000 \
-lavcopts vcodec=mpeg2video:vrc_buf_size=1835:vrc_maxrate=9800:vbitrate=5000:\
keyint=15:vstrict=0:acodec=ac3:abitrate=192:aspect=16/9 -ofps 25 \
-o movie.mpg hotel_california_ra5.1_640x480_30s.rmvb

```

and it seemed to work well enough and produced the following:

```

Input #0, mpeg, from 'movie.mpg':
Duration: 00:00:29.92, start: 0.240000, bitrate: 2681 kb/s
Stream #0.0[0x1e0]: Video: mpeg2video, yuv420p, 720x576 [PAR 64:45 DAR 16:9], 9800 kb/s, 25.00 tb(r)
Stream #0.1[0x80]: Audio: ac3, 48000 Hz, 5.1, s16, 192 kb/s

```

Next step is to find some dvd authoring software which will complete the conversion. Will a program like DeVeDe take over from here? It is a front for the commandline dvdauthor which itself could manipulate the file produced here.

I noted with some interest that ffplay choked on the rmvb file which is the first time I have seen it fail :-).

All the best,
