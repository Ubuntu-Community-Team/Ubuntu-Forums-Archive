### Post by Madspyman on 2009-11-10
For native Linux photo editing Gimp is really your best option, some of the alternatives mentioned really aren't as capable, it's the main reason that Gimp keeps getting compared to Photoshop, because it's as close as open source gets to an alternative. 

With that being said Gimp is certainly not Photoshop, and people who use Photoshop extensively can't stand the differences. Back in 2002 when Disney switched over to Linux as there main OS Gimp wouldn't do the trick, they paid the developers over at Wine $15,000 to make Photoshop (version 7 at the time) work in Linux, as a result  every version of Photoshop since version 5 will run in Linux using Wine. [http://www.eweek.com/c/a/Linux-and-Open-Source/Penguin-Moves-to-Disney/]("http://www.eweek.com/c/a/Linux-and-Open-Source/Penguin-Moves-to-Disney/")

I can't recommend an alternative. If you have any version of Photoshop I recommend installing that using Wine. 5 - CS2 install with no issues, Cs4 will require some tweeks to get the installer working. [http://thesmilingpenguin.com/post/105517877/running-photoshop-cs4-on-linux-using-wine]("http://thesmilingpenguin.com/post/105517877/running-photoshop-cs4-on-linux-using-wine")

I'm currently using CS2 in Linux and have a demo of CS4 installed, and am planning to upgrade to the full version soon. Using Linux doesn't mean giving up Photoshop.
