### Post by Scott L on 2011-02-18
> **Cthulhu_Dreams said:**
> I've been using Ubuntu Studio since Gusty Gibbon and I feel as though there has been a loss of steam in recent years.  I don't know anything about the development side of it, so I was wondering if there has been some changes behind the scenes which have affected its development?

Hi Cthulhu_Dreams,

I am the project lead for Ubuntu Studio and I hope to answer your questions satisfactorily.

I feel development for Ubuntu Studio began to lose momentum after Hardy.  The team suffered from lack of continuous, active leadership and general attrition during many releases following that time.  A stagnation in proactive development is probably evident for a period afterwards.

I believe it was during Lucid I became involved with Ubuntu Studio development and assumed the responsibilities of project lead during Maverick.  I did this because I asked the same question as you and I felt that Ubuntu Studio was too important to me to let it actually happen.

Since then development has increased at a slow pace but the momentum is certainly building because others too have felt the same way and have added their efforts to help development.  I would like to point out that neither I nor any recent additions to the team have had significant development or coding experience before.

I am not sure I can state the following emphatically enough:**  Everyone can help with Ubuntu Studio development!**

Keep these items in mind:

[LIST]
[*]Coding is _not_ required to help development
[*]Testing ISO images or if a patch fixes a bug is helping development
[*]Creating artwork or themes is helping development
[*]Giving feedback about new applications is helping developing
[*]Giving suggestions for new features is helping development
[*]Updating or creating new documentation is helping development
[*]Answering mail in the mailing list/posts in the forums/questions in IRC is helping
[/LIST]
There are many ways for anyone and everyone to help Ubuntu Studio and this list certainly isn't comprehensive!

And remember, prior development or coding experience isn't required.  As I recently blogged, "[tenacity, inquisitiveness, and initiative]("http://dullass.blogspot.com/2011/02/amicable-antidisestablishmentarianism.html")" are qualities enough.

Looking forward, we have several improvements in process for Natty.  You can see them here:
[https://wiki.ubuntu.com/UbuntuStudio/ReleasePlanning](https://wiki.ubuntu.com/UbuntuStudio/ReleasePlanning)

So, to answer your questions directly: No!  Ubuntu Studio is **not** dead.

The numbers in the IRC channel have been the highest I've seen in years and I've seen very, very robust discussion pertaining to develop.  Visit us there and say, "Hi."  You can find us at #ubuntustudio-devel on Freenode.

Recent discussions on IRC have centered around the viability of having a graphical installer or possibly a liveDVD.  What we currently lack is not the desire, but the committed manpower to implement and maintain these things!

If IRC isn't your thing, you can also use the mailing lists as another method to contact us:
_ubuntustudio-users mailing list_ - [https://lists.ubuntu.com/mailman/listinfo/ubuntu-studio-users](https://lists.ubuntu.com/mailman/listinfo/ubuntu-studio-users)
_ubuntustudio-devel mailing list_ - [https://lists.ubuntu.com/mailman/listinfo/ubuntu-studio-devel](https://lists.ubuntu.com/mailman/listinfo/ubuntu-studio-devel)

If you have concerns about Ubuntu Studio then please contact us and help us help Ubuntu Studio.

ScottL

Hi mango42,

My understanding is that Network Manager was not originally included because it resulted in performance degradation.  Which means that Network Manager periodically places an appreciable load on the CPU which resulted in additional and *indiscriminately timed* xruns during testing.

I was unaware of a conflict with -rt kernels however.  I will ask some of the elder Ubuntu Studio devs if they remember this.  If this is true it still may not be an issued going forward as we are moving towards having a -lowlatency kernel in the repositories and shipping on the image.  **NB**: the -lowlatency is neither in the repos at this time nor shipping on the image currently; however we are moving towards that, perhaps in time for Natty.

Notwithstanding, due to continue poor performance with gnome-network-admin we decided to move to Network Manager for Natty.  And this decision came after modifications to gnome-network-admin we generously made, but testing showed that making connections for wireless were still prohibitively difficult in comparison.

Therefore Network Manager is currently included in the Alpha and daily images.  If you would like to test the image or see how network manager performs, you can find the daily images at:
[http://cdimage.ubuntu.com/ubuntustudio/daily/current/](http://cdimage.ubuntu.com/ubuntustudio/daily/current/)

Any other questions, please let me know.

Cheers,
ScottL
