### Post by dawiba on 2011-03-17
My thoughts...

I kind of sympathise with your position of wanting an all-in-one solution, as this is what users are used to on other OS's. If you're after similar functionality for both audio and midi, try QTractor. If you just want audio, try Ardour, instead of just going back to Mac. Both programs are in the repos.

I use regular Ubuntu with some packages added (Ardour for audio and Muse for midi) from the repos and, speaking just for myself, I quite like the fact that the way of working is slightly different in Linux (I also use Win7). Overall, I think it's a good idea for Ubuntu to have a different identity/methodology to Windows or Mac. Some would say it's essential. I guess the trick is in getting the balance right so that Ubuntu is different, but still seems familiar enough for users of other platforms, including any programs it offers.

To my mind, there is already a strong convergence between the platforms and it could be argued that for Ubuntu at least, this is necessary if it is to grow. Linux market share, for home use (I don't run a supercomputer, mainframe or server :)), seems to be perpetually hovering around about 1%. If it's achieved, what that growth would mean for the identity of Ubuntu is another question, but my guess is we'll see more programs that run in a way that Mac/Windows users will expect (much to the chagrin of some).

Try to stick with it and hopefully you'll be working as quickly and as smoothly as you'd like before too long.

If you're having problems, try asking a bit more often for help in the forums. You never know, someone might just have the solution you're looking for and that way you could keep using Ubuntu :)

### Post by dawiba on 2011-03-19
[QUOTE="sgx"]How do you like Mixbus?[/QUOTE]

I like it fine and find it easy to use. It's quite good to have everything to hand, so to speak, without having to open another window to adjust a compressor or an eq for example. Because it's built onto Ardour, I just open up Mixbus now instead of Ardour as all of Ardour's features are available as well as the Mixbus features, so all the bases are covered. Mixbus got [a favourable review in SOS]("http://www.soundonsound.com/sos/may10/articles/mixbus.htm") as well. There's also been a healthy discussion about it in the Music Recording Technology forum over there.

I'm only an amateur messing around at home, so I'm not qualified to comment on it's 'sound', but everything Mixbus does sounds fine to me (my playing is another matter :redface:). I find making adjustments with the eq's and compressors is really easy to do and to hear, which is what's wanted really.

So a thumbs up from me :)

### Post by Pablo_F on 2011-06-11
Hi,

I wouldn't use Audacity to record from rakarrack. jack support in Audacity is suboptimal. It has been so for many years. It is a good editor though. That said, it seems to me that, if Audacity records from rackarrack via jack 32 bit floating point digital audio, you can't blame audacity for the recorded sound. 

Suggestion: To do simple audio recordings I use jack time machine or jack_capture (the latter is cool but it is not in the software center with the default repos enabled, unfortunately). To do multitrack recording, I use ardour.

Cheers, Pablo
