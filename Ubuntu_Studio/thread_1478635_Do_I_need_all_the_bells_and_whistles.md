### Post by Mr A Mouse on 2010-05-09
I've done some audio work in Windows, and want to see if I can translate the workflow into Linux. I have some examples on the web ([1]("http://filkarchive.scrumpy.org/cgi-bin/song.cgi?FileID=112"), [2]("http://filkarchive.scrumpy.org/cgi-bin/song.cgi?FileID=115"), [3]("http://filkarchive.scrumpy.org/cgi-bin/song.cgi?FileID=113"), login required but it's free, and I've released my stuff under CC)--fundamentally speaking, I want to see if the same workflow will work on Linux.

My usual Windows workflow is as follows:
1. Create or adapt a MIDI file for the music. (Noteworthy or similar was my weapon of choice.)
2. Run the MIDI file through a synth, record the output. (I've used everything from QuickTime Pro to Acid.)
3. Record the voice track, add sound effects as needed. (Acid, Audacity)
4. Profit! (Oops--wrong punchline, as this is strictly a hobby.)

Now, with Ubuntu, I'm having problems with some of the tools ([can't get Jack to run in realtime]("http://ubuntuforums.org/showthread.php?t=1478620")), but that started me wondering. I know with Windows, I had a lot more software capability than I was actually using. The programs I ran were the moral equivalent of driving five blocks to the grocery store with an RV. If all I'm doing is the above workflow, what tools do I actually need?
