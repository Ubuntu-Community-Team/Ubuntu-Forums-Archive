### Post by smoorby on 2010-03-22
Thanks, the info above proved very helpful.  The module 'snd-mtpav' is part of Karmic.  In order to load it I needed to free up other modules using the parallel port :-

sudo modprobe -r ppdev
sudo modprobe -r lp
sudo modprobe -r parport_pc
sudo modprobe -r parport

After this I could install the module with 
sudo modprobe snd-mtpav

which gives 
[   99.233840] Motu MidiTimePiece on parallel port irq: 7 ioport: 0x378

cat /proc/interrupts gives :-
 7:          1          0          0          0   IO-APIC-edge      MOTU MTPAV

showing the module owns the IRQ

Alas, that is the end of my success story.  Despite having successfully loaded the driver I don't get any MIDI communications. I have the device in "Direct" mode in which I believe channel 1 should be routed to the PC.  I can see activity on the channel 1 LED and also on the green PC LED but no MIDI activity in any sound application.
I've checked in the BIOS that my parallel port is 0x378 and IRQ 7.  It's also set to SPP although I've tried ECP+EPP too.
