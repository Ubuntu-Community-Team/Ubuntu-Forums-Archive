Let me say that he is a drummer (retired since his last stroke), but he also writes songs and plays guitar. He has always wanted to record his songs and learn a bit about editing the recordings. 

He is on a fixed income and could not afford expensive equipment, so his new computer is fairly low resource (by today's standards), but it's considerably better than his old WinXP rig. It cost a little over $300, but I was careful to pick hardware I knew would work with Ubuntu.

We tried Ubuntu Studio, but it just seemed like over-kill for his purposes. So we just installed Maverick and spent a few days reading about various music software deciding which programs might be useful to him. I showed him how to find and install a few, so he could add more on his own later, if he wanted.   

We had problems getting his microphone to work (argh). So he just uses a [Zoom recorder]("http://www.musiciansfriend.com/pro-audio/zoom-h1-handy-recorder/h70244000000000?src=3WFRWXX&ZYXSEM=0&CAWELAID=711185146") to record his songs (he has the inexpensive model), and connects it, via USB, to his computer to move the sound files to it.

I'm sure there are those who will laugh at a couple of old folks and their make-shift home recording adventures, but it's a start and we both learned a lot...and everyone has to start somewhere, right?

In the end the best way to succeed is to start simple, and build from there. If you run into problems, don't give up, there's almost always a work-around or solution.

In the end, we didn't use Ubuntu Studio, but we learned a LOT from their web site and documentation.   

Whatever you do, just don't give up. :)

### Post by Pablo_F on 2011-07-17
Hi,

> what is going on??? 

This:

"the playback device "hw:0" is already in use. Please stop the application using it and run JACK again"

jack needs to take hold of your audio device (hw:0 in this case) but there is another application using it so jack can't grab it.

Now, normally you have to just close the applications that are using your audio card. If you are not sure which it is, try and retry. 

If still it is not clear and you don't know what flipping app is using your card and how to close it, type in a terminal:

**lsof | grep pcm**

and you will see something opening /dev/snd/pcmC0D0p

Then you can kill the process with 

**kill PID** 

where PID is the process ID number.

This is an example:

```
$ lsof | grep pcm
plugin-co 5397      pablo  mem       CHR      116,5             6206 /dev/snd/pcmC0D0p
plugin-co 5397      pablo   26u      CHR      116,5      0t0    6206 /dev/snd/pcmC0D0p
```

In this particular case, the following command releases the sound card:

```
kill 5397
```

Cheers, Pablo

### Post by Flaggmann on 2011-07-17
where jack is concerned, jackd is the daemon that shows in the list of processes running when it is launched. If you open a terminal and do a ps -A|grep jack it will show if it is indeed up and running.  Some have reported that it is also user sensitive, that is if you have it set up to start up the daemon jackd by using the /etc/init.d on the boot sequence it will be running as root, and any tasks that are running under a different user might not see that process and thus give an unable to connect error. 

To avoid that, I opened the qjackctl window and went to setup options, put a check mark in launch application on startup (this launches a jackd specific to the user and that instance of qjackctl) the every application that you run as that user should see the jackd server. You may have to go into the app preferences for each app and make sure the options are set so that it will recognize jack audio. Also in the setup take the check mark out of single instance only box. 

To get it to start automatically as soon as you log in, add a link to qjackctl in the START-->SYSTEM-->PREFERENCES-->Startup Applications list.

Also some programs won't show up in the jack connections/Patch bay windows until you actually click the play button. Some will show up as soon as you launch the app before you click play, others not. Audacious is one that does show. 

The reason you got the could not connect message is because proably the daemon, jackd, was not running and hence qjackctl could not connect  to it. By using the automatic start procedure on login, it should be sure jackd is launched, and should connect for you.
