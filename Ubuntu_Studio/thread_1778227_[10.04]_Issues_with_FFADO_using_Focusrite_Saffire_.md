### Post by dawiba on 2011-06-09
Even though I use a Saffire LE, I can't help you very much as I've had a trouble free run with it (I'm using 11.04 and the new Firewire stack), so most of the code you've copied means nothing to me.

A couple of suggestions though...

I don't think you need Ubuntu Studio Controls. Have you tried running your setup with it disabled or removed?

Also, if you type...

```
sudo dpkg-reconfigure -p high jackd2
```

into a terminal, it'll reconfigure Jack for you. Make sure you select the option to enable realtime priority.

Following the advice [on this page]("http://jackaudio.org/linux_rt_config"), I created a realtime group and made myself a member...

```
groupadd realtime
usermod -a -G realtime dawiba
```

of course, you would change 'dawiba' for your own username ;)

I'm sure somebody who understands all the code will offer more help.

### Post by trulan on 2011-06-10
IIRC, Ardour reports one-half of the round-trip latency of the JACK server.  Then you add a little latency on the input and output A/D and D/A converters on you soundcard, so your actual latency is probably more like 10ms.

If you want to know exactly what you latency is, there's a neat little tool called jack_delay available.  (one variant is called jdelay).  It's a JACK signal generator, you patch its output to a soundcard output, plug a cable from your soundcards output to an input, and patch that to the input of jack_delay, ad it will give you a very precise reading of your actual round-trip latency.
