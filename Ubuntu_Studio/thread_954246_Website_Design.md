### Post by paulmerchant on 2008-11-04
There's some great information in this thread.

First, I agree that using a CMS is the way to go. With a good CMS, you'll have the WYSIWYG power of Dreamweaver for your content and almost anyone will be able to dynamically update your site. Joomla is bloated and convoluted (in my mind), but many people like it. You might want to go to opensourcecms.com and pick a CMS that directly fits your needs. There's some wonderful lightweight CMS software that you can test on that site. I like modx. Also, check what your hosting provider can install by default. Often software like Wordpress and Joomla can be installed with the click of a button by your hosting provider.

Once you start needing to customize templates or tweak code, you will have to dive deeper into the HTML, CSS, PHP, etc. Bluefish can help. I found that using the Eclipse IDE with Aptana provided me with a lot more power and assistance. Using Aptana, I get exceptional code completion, comprehensive integrated help, and powerful error checking for my XHTML, CSS, PHP, Javascript and more. You can even configure Aptana to work directly with a local LAMP server (XAMPP is easy to install on both Linux and Windows). Once you have Aptana integrated with your CMS running on a local server, you'll have an amazingly powerful coding and testing environment.

If you're new to all of this, take baby steps. Use a CMS that your hosting provider can install with the click of a button. Follow tutorials on how to tweak the templates. Don't worry about installing a local server like XAMPP or a full IDE until you start needing those kind of tools.

And, of course, for the actual design elements beyond HTML and CSS, Gimp and Inkscape are awesome.
