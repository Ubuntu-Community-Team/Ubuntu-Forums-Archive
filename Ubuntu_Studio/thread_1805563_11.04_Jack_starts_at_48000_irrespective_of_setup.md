### Post by Pablo_F on 2011-07-16
You can always "force" the sample rate in jack by using the plughw device. With it, alsa performs a sample rate conversion and it is useful when the sample rate you need is not natively supported (by the driver or by the hardware itself).

If you think this is a driver problem, you should report a bug in the alsa bug tracker or ask in the alsa users mailing lists.

Cheers! Pablo

### Post by Pablo_F on 2011-07-16
Hi, 

Always run jack in realtime mode. Although I am not sure if this has something to do with the sample rate issue and the fact that jackd doesn't start now with the plug device, running jackd in non-realtime mode is a bad idea.

Just make sure that your audio group has rtprio and memlock priviliges by running in a terminal:

sudo dpkg-reconfigure -p high jackd2

Select YES.

Then add your user to the audio group:

sudo adduser your_user_name audio

And reboot. 

Now, try again from the beginning, I suggest.
