### Post by RaumTrug on 2011-05-30
ubuntu does "feature freeze" by intention, meaning at the point the "freeze" happens (some time before official release), version numbers of software won't change anymore. all modifications done afterwares (updates) are on the frozen version, to fix bugs & sec holes, only. this is not only blender, but everything in the official repos.

if you need newer versions, seek a ppa, compile yourself or whatever. in case of blender, you can even just download the releases from blender.org, no need to really "install" it.

of course you can also wait for the next ubuntu release, which has a newer version of blender "frozen" in itself, which will then soon be outdated as well ;)

many distros do it that way (to prevent all kinds of problems), choosing the versions of software they include carefully (not always the newest at freeze time, but the "most stable" etc.). if you always need latest, there's distros juggling with new versions of everything, i.e. arch linux. this is called "rolling release", and rolls around the latest "stable" fuzz available.
