### Post by arefem on 2008-09-22
Hi, not sure if this is the right place, but here goes.

I am a presenter on a local (unpaid) community radio station.

Traditionally we have used an ageing Windows 98 based PC for playing jingles using either [Web Jockey ]("http://www.11software.com/products_jockeypro.htm")or [Combiwave]("http://www.combiwave.com/").

Does anyone know of any suitable equivalent packages for Linux?

We require the software to be able to load more than one audio file (MP3, Wav, WMA etc), and preferably 15 or more, allow the volume levels to be set individually, allow chaining of jingles, allow pre-cuing of next jingle to be played after current jingle, jingle action (eg. play, pause, stop, stop all) to be controlled from the keyboard.  Oh yeah, and preferably free or very cheap:)

There's plenty of packages that will play audio, but none that allows the flexibility we need.

Of the two we use now, my personal favourite is [Combiwave]("http://www.combiwave.com/")

I'm not asking anyone to sit down and write a package, although if you want to I'm not going to stop you either.

### Post by Stochastic on 2008-09-22
It's not really geared towards what you're looking for, but I think some of the DJ apps around might get the job done - you be the judge.

[Mixxx]("http://www.mixxx.org") - a very stable two-channel mixing app.
[Terminatorx]("http://terminatorx.org") - this is the only one I know of that will allow more than two turntables, but it's interface isn't the most intuitive.
[DJPlay]("http://djplay.sourceforge.net/") - another two-channel mixing app.

Others have attempted to solve this same problem in the past on these forums, so a little digging (it might be in the archives now) could turn some results.
