### Post by Manolicious on 2010-06-16
I have a starling netbook running ubuntu 9.04, and I had just plugged in an external keyboard (dell model sk-8115) into one of the usb ports and was doing fine working off that keyboard.  However, when I now type on the starling's keyboard, some of the keys have been reassigned, such that when I logout I can't log back in because I can't type my password.  This occurs whether the dell keyboard is plugged in or not, and so long as I keep the dell plugged in I can type just as normal on it, I just can't use the starlings keyboard.

However, if I just restarted the computer and I'm logging in for the first time, the starling keyboard completely works, either at the login screen or on a virtual port.  In fact I can login through a virtual port and still type as normal.  However as soon as I login to the gui (ctrl f7) the starling keyboard goes back to being crap, even if I had previously previously signed in on a virtual terminal it instantly goes back to typing crap.

MUST I reinstall ubuntu with the 76 driver?  I've looked for options in the gui but ubuntu seems sparse in offering ANY means to "safely eject" an external keyboard, or restore the starling keyboard to its proper form.

Anyway, for anyone using a starling netbook right now, I recommend to NOT plug in an external keyboard.

### Post by isantop on 2010-06-17
Make sure Numlock is not on. Some keyboards turn it on, or set it to turn on, when they are plugged in. The indicator light is near the left hinge, and it's the third one from the left. If it is, you can press Fn+F11 to turn it off.

I just noticed that too last night, that the number side of the keyboard seemed permanently on, but I didn't know that it was simply the numlock key that was responsible.

It's fixed, thanks!
