### Post by nicf on 2011-05-08
I just attempted to upgrade to 10.04 LTS from 9.04 (or 9.10; not positive I remember right) on my Starling. In the middle of the upgrade it informed me that the linux-headers package wasn't installed correctly, which sounded bad, but it finished anyway. When the upgrade finished (I hadn't even restarted yet) the wireless network stopped working, and when I tried to connect to a wired network after restarting, I found that that wouldn't work either. I tried adding eth0 to /etc/network/interfaces, following instructions from somewhere else on the Internet, but that seems to have done nothing but make the Network Manager applet thingy not show up anymore. What should I do about this?

---

### Post by isantop on 2011-05-10
First, you'll want to remove the eth0 interface information from /etc/network/interfaces. This file is controlled directly by network manager, and shouldn't need to edit it at all. You will want to leave the lo interface present.

I would attempt a fresh installation of 10.04. The upgrade path from 9.x to 10.04 was littered with issues, and I think you may have run into one. Installing a fresh copy of Ubuntu will fix this up.
