### Post by isantop on 2010-05-04
Our official opinion is to wait for an update. It shouldn't be long since that is a pretty invasive bug.

For those who really can't wait, if you have an Nvidia card installed, you can follow the following fix:

Open a terminal (Applications > Accessories > Terminal) and run the following command: ```
sudo gedit /etc/default/grub
``` You should get a text editor window. Add the following lines to the bottom of the file: ```
GRUB_GFXMODE=XXXXxYYYx32
GRUB_GFXPAYLOAD_LINUX=keep
```
where XXXXxYYY is your screen resolution, for example, 1280x800. leave the x32 on the end.

Next, save and close the file. From the terminal, run ```
sudo update-grub2
``` Once that command finishes, you should be set. Try rebooting to check.

### Post by isantop on 2010-05-04
It is actually a bug in plymouth. It happens across both proprietary drivers (Nvidia and ATI), plus if Mode Setting didn't work properly, plymouth should fall back to a text-mode boot. And, plymouth works fine with the open-source versions of the graphics drivers (on both ATI and Nvidia)
