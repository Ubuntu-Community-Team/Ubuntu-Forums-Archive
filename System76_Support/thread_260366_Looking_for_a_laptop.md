### Post by lord trousers on 2006-09-21
Xgl WILL NOT work very well with an Intel card. I've got a Pangolin Value (Intel 945GM), and I tried that - it was very, very slow.

AIGLX, on the other hand, works like a dream. I'm using it right now. :D I followed the instructions here:

[https://help.ubuntu.com/community/CompositeManager/AIGLX](https://help.ubuntu.com/community/CompositeManager/AIGLX)
[https://help.ubuntu.com/community/CompositeManager/InstallingCompiz](https://help.ubuntu.com/community/CompositeManager/InstallingCompiz)

Do them in that order. No snags whatsoever for me.

The only problem I've had is that I can't use Compiz as my window manager *all the time*. It doesn't work very well with certain OpenGL programs (even if you check "Unredirect Fullscreen Windows" in the Compiz Settings Manager), and the logout button locks up the display. (It's a known bug - the logout window grabs the X display entirely for itself, which keeps Compiz from rendering, so the window is invisible. The Ubuntu people know about it, but it doesn't appear to be a priority.)

### Post by lord trousers on 2006-09-21
Forgot to say: I've put two custom application launchers on my panel. Ones does: "compiz-start", the other does "metacity --replace". I switch back to Metacity when I want to use certain OpenGL programs or log out.

One more thing: as far as I can tell (I haven't really looked into it), if you want your videos composited with everything else, you have to set up your video players to NOT use the XV extension - use XSHM instead.
