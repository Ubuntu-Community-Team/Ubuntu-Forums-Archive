Hi
I started using Linux while studying towards my M.D. Although i am a big advocate of linux and OSS, there are limitations to use in medical schools.
[LIST=1]
[*]Most textbook and other interactive software written for medical students runs on Windows. I haven't had too much success running them in wine
[*]Readers for various ebook formats like pdf, djvu, chm are available in linux. There are issues with some propriety formats though, for which support is not available. These constitute a very small fraction of what's out there.
[*]Ubuntu, as mentioned above, is useful for making presentations and taking notes. It is also useful for watching medical videos and flash animations. However, as mentioned in my first point, animations and videos that run in propriety wrappers will have issues in WINE.
[/LIST]

My suggestion, buy a low cost PC/netbook/laptop with at least 1 GB RAM (2 GB then very good) and install Ubuntu or Ubuntu netbook remix. Then when Windows 7 is out in September you can buy a license through your University, it'll be cheaper that way.

To make a long story short, you really cannot do without windows alltogether, unless you are ready to pass all those wonderful medical teaching software out there, or are ready to use an alternate windows system when required.

Think about it !

### Post by glacialfury on 2009-04-28
I **am** a medical student currently, so I get the last word!  Ah ha ha!

Anyway, I chose to go with Ubuntu when I started medical school this past fall.  I am the only person in my class using Linux; most use Windows, maybe 20-30 use macs.  It is true that nearly everything will be "Windows only", and school policy probably supports Windows and *maybe* macs.  

The biggest thing I'd note is that both macs and linux have compatibility problems, but I have had *much* more success adapting to those than the mac users.  I'll list some examples below.

True: most of the software that comes in the textbooks runs poorly on wine.
Caveat: most of the software that comes with your textbooks is crap, amounting to little more than a collection of images in various formats.
Workaround: There are a ton of image extraction/conversion tools for linux.  It is fairly simple to take any one of those CDs and rip out/convert all the images in them.  You use access to the viewing software, but there's plenty of that available too.  Ultimately, you'll use those CDs a lot less than you think you will.

True: Office compatibility is hardly ever an issue.  Every now and then you'll get some powerpoint slides or a Word document that just doesn't open quite right, but you can still read it (the format just looks funny).  99% of the files convert perfectly with no difference.

As for watching streaming lectures etc, most universities do simple *.wmv streaming using the mms protocol.  Watching online is not an issue, and if you want to download them, there is mimms, or I think even RealPlayer can record them directly.

After the first few months, as you acclimate to linux, you will need wine less and less.  I haven't used it in ages now.

There are several free viewers for medical/radiology images available for linux (Aeskulap and Amide).

Ultimately, there is really little difference.  You'll spend the vast majority of your school-related time either in an office suite or a web browser, and those are probably the two *most* compatible things between all the operating systems.  I'm happy I chose Ubuntu, mostly because I can do things to files that terrify the school and earn the gratification of my classmates.  There's nothing quite like taking a shoddily put together PDF file, ripping all the images out, and making it useful for people :-)

### Post by surfdoc on 2009-05-02
I'm a 2nd year med student in the UK. I think it depends on your knowledge of linux - you don't want to be messing around with the OS when you've got deadlines looming. 

I recently bought a new laptop and I run ubuntu on it. My experience is that most universities only support windows. This means that anyone running OSX or linux may equally run into problems - although often linux fairs better because it is more flexible than OSX and evolves quicker. So my advice would be to buy a Windows machine and then install ubuntu with a dual boot (or Virtualbox if you can). You can use ubuntu and resort to windows if you need to. In reality I haven't had to use windows at all, although I have been rather determined not to. 

One problem I have had is with office files. I hate to admit it, but OpenOffice doesn't always format MS Office files well. Also the new .docx .xlsx files can cause problem. I can open .docx usally -and I'm sure OO will support them better soon - but I have had to request my department to not use them (anyone with OSX or an older MS Office suite was equally affected).

There are a few media issues. Shockwave can just about be installed on ubuntu, but not brilliantly. Some resources such as 3d anatomy software use shockwave, but it hasn't been a problem for me. Other systems require installation of more obscure plugins/programs. These are unlikely to be supported in linux. Again hasn't been a problem for me.

The flash plugin for linux is poorly written and can produce poor video quality on perfectly fast machines. 

Some websites don't work too well in firefox. You can install ie6 on linux if you need to. Fortunately firefox has a sufficiently broad user base that any website not supporting it will get plenty of complaints from MS OSX and linux users alike.

Hope this helps

PS. My best advice is what ever OS you use, write your assignments on something like Google Docs. That way you can never (unless google messes up) loose your work. The suite isn't anywhere near as advanced as OO or MSO but for draft work it is fine. It even keeps a revision history for you, so you can re-load any previous save that it has done (it autosaves every 5 mins i think). Then you have no messing around with disks or dongles or emails, when working between machines.
