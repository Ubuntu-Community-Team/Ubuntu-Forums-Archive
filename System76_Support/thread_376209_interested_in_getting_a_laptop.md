3) A system builder will always create a system design that suits a wide range of users which is to say a generic configuration. When a specific configuration is needed by a user with custom settings and hardware configurations, then they should either build their own or pay some one to do it the way they want it, because a system builder would not try one custom system configuration.

4) I would blame a support group, no matter which manufacturer, if they failed to support what they sold me. It makes no sense in taking them to task for something they did not sell me, because of the simple reason that I did not pay them for it.



If the hardware configurations are the same, I would buy the lesser priced unit, even if I had to sit and figure out things on my own later.


NOTE : This is my personal belief. I do not expect others to agree or concur with it. Like I said, it is all individual perspective.

                           In a nutshell, most people depend on a system builder, no matter how big or small, because 
1) It is easier to have someone else build a computer for you
2) It is easier if it comes to your doorstep with an opertaing system installed in it
3) When something goes wrong with what was sold, it is easier to have it fixed at one place. When you buy a build system, you simply call the system builder and he fixes it)

                          Linux is a different operating system. They way it works is different from the way Windows does. For the simple fact that most hardware and software is designed to run on a Windows Box, compatibility and support is mostly for Windows. Linux is slowly but surely catching up as more users find it productive ( I do, that's why we are here).  Many people, incuding me are new to the way things work in Linux and it is more of an individual perspective as to what they choose.

Personally, as long as the technolgy is up to date and and the prices for the same are right, you could get pretty much any type of system today, to match your needs and your pocket. Now, expecting things that where not on the box when it was built, to work is something else.

### Post by rumblered on 2007-03-05
A few things to keep in mind:
[LIST=1]
[*]The System 76 configuration you posted has a T7400 processor on it.  The Dell has a T7200.  System 76 offers the T7200 for $140 cheaper than the T7400.
[*]The System 76 system has nVidia graphics, and the Dell has ATI graphics.  I'm not sure of the relative performance between those two particular chips, but as far as Linux drivers go, you're much better off with the nVidia chip.
[*]The System 76 system includes an Intel 3945 wireless chip, and the Dell has a "Dell 1390."  It appears that there are no open-source drivers for this chip, so you have to use a hack for loading Windows drivers.  That's a potential source of trouble, and Dell charges $29 more to switch to an Intel 3945.
[/LIST]
