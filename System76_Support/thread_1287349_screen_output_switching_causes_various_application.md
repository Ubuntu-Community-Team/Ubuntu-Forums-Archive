### Post by Temposs on 2009-10-10
The screen output mode switching with Fn+F7 works great on my Darter Ultra when I connect an external monitor. However, doing this causes various programs to crash. Gnome-Do w/ Docky crashes. Pidgin and Gwibber and Gedit all seem to disappear/crash. Everything else I use seems fine.

I have taken to preempting this by making sure none of these programs are running when I change screen output mode, but it's annoying if I forget or am too lazy to close these apps.

Is this a known issue and is there a more permanent workaround that I can use?

### Post by thomasaaron on 2009-10-12
There are a ton of bug reports on Gnome-do, and not much surprises me with it.

Try switching displays using System > Preferences > Display instead of the function key buttons. You may also want to see if it happens if you turn off desktop effects (System > Prefs > Appearance > Visual Effects > None). Could be related to compiz.
