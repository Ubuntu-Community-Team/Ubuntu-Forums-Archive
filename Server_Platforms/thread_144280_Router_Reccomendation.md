### Post by SundaySushi on 2006-03-14
I currently have a Linksys wireless router BEFW11S4, firmware 1.2. The IP address on the WAN side of my router changes every so often, but I use the DynDNS service, so the domain name stays the same. Additionally, I am running Ubuntu 5.10 with an Apache web server 2.0 on one of my LAN side laptops. I have no problem accessing the server locally (i.e. 192.168.1.105), however I have had no success accessing it from inside the LAN or outside the LAN via the domain name (blah.blogdns.com). I have fiddled with every possible port forwarding combination to no avail. I took my laptop to my friends house. He uses an Apple AirPort Extreme. To my surprise, I was now able to access my server via domain name both inside and outside of the LAN. As such, I have decided that there is something wrong with the router (or me). I am going to purchase a new wireless router to fix the situation. Can some people tell me their success stories, and more specifically, what specific routers you have had success with.

### Post by Zelut on 2006-03-14
I am using a Netgear WGR614v4 and it works great!  Its been powering my webhosting for months now (email, ftp, http, gnump3d, smtp, torrents, ssh, vnc, etc).  I can access my hosted sites from inside or outside the network easily.  I can vnc to any of my internal machines whenever I need & I regularly maintain my network with ssh with no problems.

I've got four wired machines & three wireless, all handled very reliably.  I've got MAC filtering setup, also use dyndns.org for a site as well.  I'd recommend this router to anyone for regular home, small network use.

So far its been a great router, doing everything I need it to do & it cost me $20 after rebate :)
