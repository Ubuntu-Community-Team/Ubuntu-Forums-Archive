### Post by tturrisi on 2006-11-25
That makes things much easier!
You can use iptables (it's already installed!) to grab the data you want.
See this:
[http://www.linux.com/article.pl?sid=05/12/15/177232](http://www.linux.com/article.pl?sid=05/12/15/177232)

I'm not certain, but if the lan hosts use dhcp addresses then you may be able to use a wildcard in the rule(s).  192.168.1.*  If not, then config the router dhcp to use a limited range of ips for dhcp, such as 192.168.1.2-30 (if have 29 hosts or less), then make a rule for each ip whether it is used currently or not.  This saves having to make a rule for all 254 possible dhcp addresses.

You could even use something like this to put the grabbed data into a db & view the log via a browser:
[http://www.iptablesrocks.org/guide/analyze.php](http://www.iptablesrocks.org/guide/analyze.php)
