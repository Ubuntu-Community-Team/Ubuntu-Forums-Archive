### Post by TheFu on 2017-10-10
A headless server doesn't need a GPU if you use the correct desktop.  That would be LXDE or XFCE or something lite.
VNC isn't exactly the best solution for remote management.  ssh is.
If you need a Lite-GUI, I'd strongly recommend dumping VNC for x2go.  x2go is very stable and feels 2-3x faster than VNC.  It uses ssh credentials, so your ssh-keys already setup will just work. So will the ~/.ssh/config file settings.

x2go removes all the old-school X/setup stuff too.  Clients for the main 3 desktops exist.  I've used it from 5 different continents.  x2go will install whatever dependencies it needs. None of the existing X/server stuff is needed.  The only trick is to setup ssh BEFORE installing x2go.  Use the x2go ppa from the x2go.org website. [https://wiki.x2go.org/doku.php/wiki:repositories:ubuntu](https://wiki.x2go.org/doku.php/wiki:repositories:ubuntu)

People say it is like night and day after switching.

If you need VNC for some other reason, like trying to use android as a client, then x2go really isn't an option. I haven't used VNC for about 6 yrs, but it didn't take much to get it working.  Never ran xstart - always just started vnc-server with the quickie settings, then connected from a VNC client through either a VPN or ssh tunnel.  To make certain other users cannot connect via VNC in a non-secure way, use the --localhost option onthe VNC server, to prevent any connections that don't originate on 127.0.0.1.  Much better for security.  VNC security is basically a joke.

BTW, none of my x2go systems have a GPU installed.

If you are on the same LAN for both systems, use *ssh -X userid@server application* (as SeijiSensei says below). Over the internet, x2go is really the best answer I know.  There are other options for KVM VM clients.

If the system is running inside a virtual machine managed by libvirt, then you can use the SPICE protocol.  I've only used it on KVM VM hosts and only on the same LAN. I've had some keyboard mapping issues with SPICE, but it had fairly good performance - a little better than x2go.  My VM hosts are still on 14.04, so some of the great improvements with spice and libvirt aren't available to us, but I have seen some pretty impressive 16.04+ SPICE connections - to make migrating when I have time a serious consideration.

### Post by SeijiSensei on 2017-10-11
Are you running the proprietary NVIDIA driver?  If not, run "sudo apt-get install nvidia-current".  If you have the driver installed, did you generate a new xorg.conf with the command "sudo nvidia-xconfig" after the update?  Give that a try as well.
