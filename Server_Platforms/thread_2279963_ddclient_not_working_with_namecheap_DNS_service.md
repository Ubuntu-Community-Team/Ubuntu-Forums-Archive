### Post by Silman on 2015-05-27
I have just set up my first ever home server. I created root, made the IP static on my local network, installed openssh, and can now ssh into the computer as long as i am connected to my homes router. Now i want to be able to ssh in over the internet.

I have bought a domain (lets call it example.com) and want to configure NameCheap's dynamic DNS service for the server. And i have set up the host record as[ in this image.]("http://i.imgur.com/PdttzCC.png?2")

I have also enabled the dynamic dns namecheap offers as [this picture shows.]("http://i.imgur.com/05XnXIU.png?2")

Now for the server side

I did

    ```
apt-get install ddclient

```

and accidentally pressed some buttons on its walk-through configuration installer. woops.

So i did ```
apt-get remove ddclient
``` and then reinstalled with ```
apt-get install ddclient
``` hoping i would get the walk-through config again, alas i did not.

No biggie, i said.

I went into ```
vi /etc/ddclient.conf
``` and modified the file to look like this

    ```
root@servername:/home/username# cat /etc/ddclient.conf 
    # Configuration file for ddclient generated by debconf
    #
    # /etc/ddclient.conf
    
    
    use=web, web=dynamicdns.park-your-domain.com/getip
    protocol=namecheap
    server=dynamicdns.park-your-domain.com
    login=example.com
    password=reallylongpassword
    @

```

as per the guide [here]("https://www.namecheap.com/support/knowledgebase/article.aspx/583/11/how-do-i-configure-ddclient").


Then i do ```
/etc/init.d/ddclient restart

```

and it tells me

     ```
* To run ddclient as a daemon, please set run_daemon to 'true' in /etc/default/ddclient 
       ...done.
```

which makes me thing this isnt set up to run on startup, but im not sure what daemon mode is?

Here is my ```
cat /etc/default/ddclient
```
```

    # Configuration for ddclient scripts 
    # generated from debconf on Tue May 26 20:36:36 PDT 2015
    #
    # /etc/default/ddclient

    
    # Set to "true" if ddclient should be run every time a new ppp connection is 
    # established. This might be useful, if you are using dial-on-demand.
    run_ipup="true"
    
    
    # Set to "true" if ddclient should run in daemon mode
    # If this is changed to true, run_ipup must be set to false.
    run_daemon="false"
    
    
    # Set the time interval between the updates of the dynamic DNS name in seconds.
    # This option only takes effect if the ddclient runs in daemon mode.
    daemon_interval="300"

```

Is this the way a server should have it set up, or should daemon-mode be on?

Anyway, i waited 300 seconds like in the default file and my namecheap IP hasn't updated it is still listed as 127.0.0.1

when i run ```
service ddclient status
``` the result is 

    ```
Status of Dynamic DNS service update utility: ddclient is not running.
```

So i tried running ```
service ddclient start
``` and it results in
```

     * To run ddclient as a daemon, please set run_daemon to 'true' in /etc/default/ddclient 
       ...done.

```

the `...done` makes me feel as if it is running, but i checked again and it isnt.

### Post by darkod on 2015-05-27
I haven't used ddclient yet but according to the messages you are getting you need to modify your /etc/default/ddclient to say:
run_ipup="false"
run_daemon="true"

Then try to start the service and check if it will adjust the ip after 5mins.
