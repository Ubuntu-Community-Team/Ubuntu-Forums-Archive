### Post by MakOwner on 2014-05-18
I had a server running with the OS on local SATA attached disk, and a RAID6 array of 16 2TB volumes attached via SAS from an external enclosure in a single ext4 filesystem.
I had some failing sectors in one of the SATA drives, so took the system down and replaced the drives and reloaded ubuntu server.  

The filesystem passes a forced fsck check, but hangs when I read certain files (which I assume are out past the 16TB portion of the filesystem).
 
What setting or option did I forget to set to allow the system to handle the large filesystem?

---

### Post by sandyd on 2014-05-18
Can you post the output of
```

tune2fs -l /path/to/filesystem/block/device
```

It may be that your ext4 doesnt have 64bit flag set. In that case you can't go over 16TB
To fix it, the filesystem will have to be recreated with -O 64bit
