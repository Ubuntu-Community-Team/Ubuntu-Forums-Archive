### Post by anystupidname1 on 2011-01-03
From "man usermod"
>        -G, --groups GROUP1[,GROUP2,...[,GROUPN]]]
           A list of supplementary groups which the user is also a member of. Each group is separated from the next by a comma, with no intervening
           whitespace. The groups are subject to the same restrictions as the group given with the -g option.

           If the user is currently a member of a group which is not listed, the user will be removed from the group. This behaviour can be changed via
           the -a option, which appends the user to the current supplementary group list.


So, yes, I think you removed your user from whatever groups you didn't include in the command.

Can you not simply:
```
sudo su -
```
to get root and re-add your user to the appropriate groups?

### Post by CharlesA on 2011-01-03
If they removed themselves from admin group, so they can't use sudo.

You can use su to switch to someone else who can use sudo, but if there is only one user and the root account is locked/disabled, you are pretty much out of luck for doing anything remotely.

Best thing to do would be to drop to a root shell (boot into recovery mode by holding down shift to get to the GRUB menu) when you get home and add yourself back to the groups you removed yourself from.
