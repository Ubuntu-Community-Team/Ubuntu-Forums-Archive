### Post by SeijiSensei on 2012-06-30
I take it you enabled packet forwarding in sysctl.conf on the office machine, yes?

You need a routing table entry on the client that tells it to send traffic for the remote LAN network over the tunnel.  The easiest solution is a script in /etc/openvpn that is launched with the "up" directive in openvpn.conf.

Suppose the office has network 192.168.1.0/24, and the tunnel has address 10.10.10.10 on the client and 10.10.10.1 on the office server.  On the client, create the file /etc/openvpn/add_route:

```

#!/bin/bash
/sbin/ip route add 192.168.1.0/24 via 10.10.10.1
```

and mark the file executable by root.  In /etc/openvpn.conf, add the line 

```
up ./add_route
```
