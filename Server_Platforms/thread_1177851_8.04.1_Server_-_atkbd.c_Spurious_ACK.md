### Post by GumbyNoTalent on 2009-06-04
[http://lxr.linux.no/linux+v2.6.29/drivers/input/keyboard/atkbd.c#L417](http://lxr.linux.no/linux+v2.6.29/drivers/input/keyboard/atkbd.c#L417)
```

case ATKBD_RET_NAK:
	if (printk_ratelimit())
		printk(KERN_WARNING "atkbd.c: Spurious %s on %s. "
 		"Some program might be trying access hardware directly.\n",
		data == ATKBD_RET_ACK ? "ACK" : "NAK", serio->phys);
	goto out;

```

so it is a valid message caused by the KVM...
