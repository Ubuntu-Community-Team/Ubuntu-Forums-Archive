### Post by flyingsuperhigh on 2011-06-07
Whenever I try to execute a python program stored in the cgi-bin directory, I get a forbidden error saying: you don't have permission to access /~/public_html/cgi-bin/test.py on this server. For the cgi-bin directory itself, I have chmoded it to 711. For the python file, I have chmoded that to 700. I am able to view html pages stored in the public_html directory, but whenever I submit data through html forms to the python program I get the forbidden error. Any ideas what is wrong?

### Post by ruffEdgz on 2011-06-08
I'm guessing its because you are able to execute everything else leading up to the python program because of 1 in the 'other' section. Change your 'test.py' file to the following mode:
```
 chmod 701 ./test.py
```
I have two questions:
[list]
[*] who owns(user/group) the file test.py
[*] What user/group runs apache? (you can find this in your /etc/apache2/apache2.conf looking for 'User')
[/list]
Thanks!

Just want to make sure that:
[list]

[*]If you so the following command 'grep ScriptAlias /etc/apache2/sites-enabled/*' what do you see?

It does seem like its a permission/mode issue but will need to know more about your current setup since the default cgi-bin directory is usually stored in /usr/lib/cgi-bin/ and have the default mod value of 755 (for anyone to cd into a directory, you need both read and execute).
