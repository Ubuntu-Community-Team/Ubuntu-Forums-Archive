### Post by darkod on 2017-04-06
When the server boot it usually shows the grub menu but only for about 3sec. You have to be fast and using the arrows select Advanced Ubuntu options (or something along those lines) instead of the default ubuntu top entry in the menu.

Then in the advanced options it will give you the possibility to load previous kernel. Select the previous to the most recent, and see if it boots like that.

Also if it still doesn't boot they can at least tell you of any error messages on screen...

### Post by darkod on 2017-04-06
You can, especially if you still have the person on-site to help you. You can try removing the latest kernel and do apt-get dist-upgrade again (most people use dist-upgrade instead of only upgrade). Then reboot and check if it boots. If it doesn't, let him boot the previous kernel and advise all not to reboot server until further notice (when another new kernel comes out so you can check if it works with that one).

### Post by darkod on 2017-04-06
If I'm not mistaken the exact package name is:
```
sudo apt-get purge linux-image-4.4.0-72-generic
```

Using purge will remove it completely.

You can get the full package name with:
```
dpkg --list | grep 'linux-image'
```
That will list all kernels installed.
