### Post by Titan8990 on 2008-07-16
Squirrelmail is not a mail server. It is webmail that goes over top of you mail server. It requires a fully functioning webserver such as Posfix of Exim4.

I first set up Exim4 with squirrelmail as webmail and it was a headache. Once you fight through that headache it is a very rewarding experience. 

Now, about Squirrelmail config files. You don't and shouldn't directly edit Squirrelmail config files. That is what the plugins are for. To configure squirelmail:

$sudo /usr/share/squirrelmail/config/conf.pl

That is the configuration page. Did you even read the documentation?

Have you considered getting a book dedicated to say Posfix or Exim4?

Also, I agree with not using a GUI as a crutch. You learn config files and CLI well then when you find yourself in a GUI using situation you will find it both easy and inadequate due to limiting functionality. GUI users will be "newbs" to you.

Personally, I would rather not even use a web interface.

Also, setting up new users:

I'm not sure how it is on other Email servers but I assume that it is the same. On Exim4 the users and their Email address are the local users on server. Kind of like how Exchange does it with Active Directory. 

Aliases can be created in the /etc/alias file. Here you can email going to [email]root@YOURFQDN.com[/email] goes to all your Email administrators and etc.

Edit: Reading this and some of your other topics I think you should actually pick up a book on the basic fundamentals of Linux. This is the book that got me started: [http://www.amazon.com/Practical-Guide-Ubuntu-Linux-R/dp/013236039X/ref=pd_bbs_sr_2?ie=UTF8&s=books&qid=1216236874&sr=8-2](http://www.amazon.com/Practical-Guide-Ubuntu-Linux-R/dp/013236039X/ref=pd_bbs_sr_2?ie=UTF8&s=books&qid=1216236874&sr=8-2). It even includes a short chapter on setting up Exim4 (although I bought a separate book on Exim4).

### Post by Titan8990 on 2008-07-16
> i want web mail not network mail...

In order to have webmail you need a Email server. There are three things you need to send mail and have it viewed at it's destination: a MTA (Mail Transfer Agent), a MDA (Mail Delivery Agent), and a MUA (Mail User Agent). 

A webmail client is nothing more than a MUA.
