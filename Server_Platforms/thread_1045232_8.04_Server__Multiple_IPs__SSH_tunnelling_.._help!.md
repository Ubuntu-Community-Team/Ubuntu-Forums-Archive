### Post by AndiTails on 2009-01-20
Hi all,

I have a dedicated server hosted in France, and have an additional IP which is UK-based in order for websites to be seen as UK sites, etc (so their DNS points to the IP-Alias).

Below is my config for the networking adapters (have removed the actual IPs)

```

# This file describes the network interfaces available on your system
# and how to activate them. For more information, see interfaces(5).

# The loopback network interface
auto lo
iface lo inet loopback
pre-up iptables-restore < /etc/iptables.up.rules

auto eth0
iface eth0 inet static
        address 94.------
        netmask 255.255.255.0
        network 94.-------
        broadcast 94.--------
        gateway 94.--------

auto eth0:0
iface eth0:0 inet static
address 87.----
netmask 255.255.255.255

```

So my box has eth0 - the real IP for the box within the datacentre, and also the virtual adapter eth0:0 which means sites who's DNS points to the UK 87. IP work perfectly and are seen to be UK sites. Great.

Now, I also use the box as an SSH tunnel through work. (I am allowed to - I develop websites and need full access). Under CentOS which I ran before (and its networking setup was pretty similar) I could choose the IP/adapter a SSH login was bound to - so then I could use an SSH login with my tunnel software which when breaking out on to the net would be seen as a UK person, defaulting to google UK, etc.

I've tried binding SSHd to the 87. ip address, and couldn't connect to the box at all. SSHd is now binded to 'all adapters' and can be reached via both IPs.

How can I make an account which I can SSH into the box with and with a tunnel it will be a UK IP on eth0:0?
Is it possible?

Am happy to change the config, as long as it all works ;)

Thanks for reading.
