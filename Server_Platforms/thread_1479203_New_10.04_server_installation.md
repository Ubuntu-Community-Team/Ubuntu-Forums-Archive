What makes you think it doesn't have a hosts file?  Windows machines have hosts files too...

So it seems to be a DNS issue.  There's two options - your DNS server holds the relation for your server.  Your client **has** to be using a local DNS server that has that relation, not your ISP's or your router's.

The other option is to use the hosts file on the client PCs.  This avoids having to use DNS, but if you have a lot of machines that need to access this server, DNS is the easiest way to achieve that.

### Post by cdenley on 2010-05-10
You must have bonjour installed on your windows system. avahi-daemon broadcasts your hostname so it can be resolved by bonjour with "hostname.local".
[http://packages.ubuntu.com/lucid/avahi-daemon](http://packages.ubuntu.com/lucid/avahi-daemon)

A default server configuration of course isn't going to broadcast your hostname. Not with netbios (windows, samba) or Multicast DNS (mac, bonjour, avahi).

### Post by arrrghhh on 2010-05-10
Glad to hear you got it sorted.

The ONLY reason I recommend using aptitude instead of apt-get is when you install a big meta-package like ubuntu-desktop, you can easily remove all the stuff that it installed with it using aptitude.  Unless something's changed, last I checked the --purge option of apt-get wasn't so good at doing that.  Then you wouldn't have to completely reinstall the server :D

mituw16 - my solutions listed earlier stand.  Either setup DNS on a server and point your clients to that DNS server, or configure the hosts file on every machine you need to have access that server from the familiar name.  DNS is eaiser to add clients to (no need to edit files on each client once it's working), but requires more setup initially.
