### Post by KingBahamut on 2005-09-14
Heres my I-Cant-Live-Without-These-Tools-Top-Ten List. 

CLI List

1. Iptraf
[http://iptraf.seul.org/](http://iptraf.seul.org/)
2. Nmap
[http://insecure.org](http://insecure.org)
3. Tcpdump
[http://www.tcpdump.org/](http://www.tcpdump.org/)
4. Slurm
[http://www.llnl.gov/linux/slurm/](http://www.llnl.gov/linux/slurm/)
5. Ngrep
[http://ngrep.sourceforge.net/](http://ngrep.sourceforge.net/)
6. Tcpflow
[http://www.circlemud.org/~jelson/software/tcpflow/](http://www.circlemud.org/~jelson/software/tcpflow/)
7. Arping
[http://www.habets.pp.se/synscan/programs.php?prog=arping](http://www.habets.pp.se/synscan/programs.php?prog=arping)
8. PipeBench
[http://www.habets.pp.se/synscan/programs.php?prog=pipebench](http://www.habets.pp.se/synscan/programs.php?prog=pipebench)
9. uuturn
[http://www.habets.pp.se/synscan/programs.php?prog=uuturn](http://www.habets.pp.se/synscan/programs.php?prog=uuturn)
10. whowatch
[http://wizard.ae.krakow.pl/~mike/#whowatch](http://wizard.ae.krakow.pl/~mike/#whowatch)

Using these tools properly, you should be able to route out any network problem without fail and using minimum resources to do it. Bear in mind that the use of these tools requires intimate knowledge of following a physical packets through all the OSI layers (Specifically Tcpdump, TcpFlow, and Ngrep). 

What do you guys prefer?
