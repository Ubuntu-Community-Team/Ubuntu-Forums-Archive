### Post by Steve3 on 2007-09-26
Hi all,

I've recently set up a server with Ubuntu 7.04, Apache 2.2.3 and Webmin 1.350 for our 3 web developers to work on.  So far, it's working well, as they've made and are hosting about 12 development subdomains on it.  These were all set up via webmin, with .conf files in /etc/apache2/sites-available.

I now need to move over an internal webserver to the Ubuntu server.  The content and DNS have moved just fine, but now I need to get SSL working on it- I have a signed certificate from Comodo, not a self-signed cert.

Here's the steps I've taken to get it working:
-moved the .ca-bundle and .cert files to etc/ssl/certs
-edited the .conf file to add the following:> 
<VirtualHost *:443>
DocumentRoot /usr/share/squirrelmail/

SSLEngine On
SSLOptions +FakeBasicAuth +ExportCertDAta  +StrictRequire
SSLCertificateFile /etc/ssl/certs/<certificate>crt
SSLCertificateKeyFile /etc/ssl/certs/<certificate>.key
SSLCertificateChainFile /etc/ssl/certs/<certificate>.ca-bundle

ServerName squirrelmail.<domain>.com
<Directory "/etc/squirrelmail">
allow from all
Options +Indexes
</Directory>
LogLevel debug
</VirtualHost>
-And because I thought that Apache only allowed one SSL site per IP, I went to webmin and edited the port of each virtualhost to be 80, with the one SSL site to be 443.

Obviously, this isn't right, because starting apache tells me> usr/sbin/apache2 -t
[Wed Sep 26 14:48:08 2007] [warn] module rewrite_module is already loaded, skipping
[Wed Sep 26 14:48:08 2007] [error] VirtualHost *:443 -- mixing * ports and non-* ports with a NameVirtualHost address is not supported, proceeding with undefined results
[Wed Sep 26 14:48:08 2007] [error] VirtualHost *:80 -- mixing * ports and non-* ports with a NameVirtualHost address is not supported, proceeding with undefined results

(with a repeating VirtualHost *:80 for each other virtualhost)

So, it's unhappy about the * ports and declared (non-*) ports.  According to webmin, the only server on "any" port is the default server.  But webmin won't let me change that like the others.
So I went to /etc/apache2/sites-available, and edited default.conf.  It now starts with > NameVirtualHost *
<VirtualHost *:80>
        ServerAdmin webmaster@localhost
But that's not right either.

So, when using virtualhosts and SSL, how should my addresses and ports be set up?  Bonus points for anyone who show me how to force a user to SSL- so they can ask for [http://site.domain.com](http://site.domain.com) and transparently be pushed to [https://site.domain.com](https://site.domain.com)

Thanks.

### Post by dantrevino on 2007-10-04
It sounds like these vhosts are all on the same IP, so.....

default only should have:

NameVirtualHost *:80

all non-ssl domains (including default) should have:
<VirtualHost *:80>
...

ssl site should have:

NameVirtualHost *:443
<VirtualHost *:443>

Only 1 ssl site per IP.
