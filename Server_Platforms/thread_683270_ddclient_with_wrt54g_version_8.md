A how to configure ddclient on ubuntu server while sitting behind a linksys WRT54G version 8 router

reasons for using both ddclient and the dynamic dns update service built into the wrt54g version 8 .....  uhhh well I'll let you figure that one out!   

My hardware configuration  Dell Dimension XPS 733R
cpu: Pentium III (coppermine), 733 MHZ 384MB  ECC/ nonECC dram in two sims
3 ware 8006-2LP SATA RAID controller two 500 gig maxtor sata 5400 rpm raid 1 yeah, what a waist back ups are key!
yamaha crw2100E  
Intel 82557/8/9 [Ethernet pro 100]
nVidia RIVA TNT 2 Model 64/Model 64 Pro

Ubuntu server Gutsy 7.10
kernal ver 2.6.22-14-server 

okay so first off install your ubuntu server don't care how you do it how you did it or how your going to do it JUST DO IT!

then go to one of many free dynamic dns service providers such as [[COLOR="Blue"]http://www.dyndns.org[/COLOR]]("http://www.dyndns.org")
or if your hosting a web page on your server and need multiple domain names hosted such as..  ###.com  [www.###.com](www.###.com)  ftp.###.com go to 
[[COLOR="Blue"]http://www.zoneedit.com[/COLOR]]("http://www.zoneedit.com")

I'm sure there are many others please choose one that is compatible with ddclient !!!!  

then comes the next fun 

username = your user name that you set up when you set up your server

localhost = your computers name that it will broadcast out to the local network if you see fit.

1.[COLOR="Red"]* username@localhost:~$ sudo apt-get install ddclien*[/COLOR]t
2.configure your ddclient for your DDNS service
3. [COLOR="Red"]*username@localhost:~$ sudo nano /etc/ddclient.conf*[/COLOR]
this will give you the following screen assuming you used zoneedit

[COLOR="SeaGreen"]# Configuration file for ddclient generated by debconf
#
# /etc/ddclient.conf

pid=/var/run/ddclient.pid
protocol=zoneedit1
use=if, if=www.zoneedit.com
server=www.zoneedit.com
login=johndoe
password='1234abcd'
[www.###.com](www.###.com), ###.com, ftp.###.com[/COLOR]

4.  change your file so it looks as follows

[COLOR="SeaGreen"]#Configuration file for ddclient generated by debconf
#
#/etc/ddclinet.conf[/COLOR]

[COLOR="SeaGreen"][COLOR="SeaGreen"]
pid=/var/run/ddclinet.pid[/COLOR]
protocol=zoneedit1[/COLOR]
[COLOR="Purple"]use=linksys-wrt854g, fw=192.168.1.1:80
fw-login=blank, fw-password=yourpassword[/COLOR]
[COLOR="SeaGreen"]server=www.zoneedit.com
login=johndoe
password='1234abcd'
###.com, [www.###.com](www.###.com), ftp.###.com[/COLOR]

5. okay up to now everything for pretty much every router is the same the only major difference is the use= line.  please note that for some reason the username is an issue, 
you can't seem to change it it used to change with the wireless network name, not any more,  just enter your networks name in this spot  it'll work, promise!


5a. [COLOR="Red"]*username@localhost:~$ ddclient -help*[/COLOR]    this is very useful check it out!

6. [COLOR="Red"]*username@localhost:~$ sudo nano /usr/sbin/ddclient*[/COLOR]
go down until you find the following 

 [COLOR="Green"]    'linksys-wrt854g'        => {
                                 'name' => 'Linksys WRT854G FW',
                                 'url'  => '/Status_Router.asp',
                                 'skip' => 'IP Address:',
                               },[/COLOR]

7. edit it to match the below

[COLOR="Green"]     'linksys-wrt854g'        => {
                                 'name' => 'Linksys WRT854G FW',
                                 'url'  =>[/COLOR][COLOR="Purple"] '/StaRouter.htm',[/COLOR]
                               [COLOR="Green"]  'skip' => 'IP Address:',
                               },[/COLOR]
or just edit in a new one and call it something else doesn't matter it's just a script!  pointing it to thecorrect url and line on the page is what matters, took a bit of puttering around with my router to get it right!  I've also got a linksys WRT54G version 3 and version 5 but I borked the two of them recently so I haven't tested!  after all this is done edited and saved you should be able to do a   
[COLOR="Red"][I]
username@localhost:~$ sudo ddclient[/I][/COLOR] and your ip address should be updated to your chosen DDNS host!  now you can go on configuring the rest of your ddclient happiness will abound!  hope this helps someone out there this post would have saved me about 5 hours of pulling my hair out!
