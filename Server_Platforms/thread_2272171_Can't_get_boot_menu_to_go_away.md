### Post by MAFoElffen on 2015-04-04
On your /etc/default/grub file...
```

GRUB_DEFAULT=0
GRUB_HIDDEN_TIMEOUT=[COLOR=#ff0000]1[/COLOR]
GRUB_HIDDEN_TIMEOUT_QUIET=[COLOR=#ff0000]false[/COLOR]
GRUB_TIMEOUT=[COLOR=#FF0000]1[/COLOR]
GRUB_DISTRIBUTOR=`lsb_release -i -s 2> /dev/null || echo Debian`
GRUB_CMDLINE_LINUX_DEFAULT=""
GRUB_CMDLINE_LINUX=""

```
Your menu timeout was set to 10 seconds. 2 seconds is the default for server, 10 seconds is default for desktop...

The setting of the hidden timeout to "0" and the hidden timeout quiet to "true" is a work-around I found about 4 years ago... to cause the grub menu to force itself to show and remain shown... (will not go way until a key is selected manually), even without another OS installed or via the shift key hotkey at boot.  "0" is a non-valid timeout value for that and that is why is remains. You probably now notice an error message that go along with that, that says that the use of "0" as a timeout value is deprecated and is not a valid value when the hidden quiet value is set to true. (it still works to force the menu to force-show...).

So if you set the hidden timeout to 1 second and/or optionally set the hidden timeout quiet value to false... it will continue after the non-zero value is expired. After making those edits, re-run update-grub to pick up the changes.

When fixed how you want it to react, be sure to revisit this thread to mark it as solved.
