### Post by el_badbit on 2013-02-14
I set up an Ubuntu Server 12.04.1 with two interfaces. My plan is to use it as a router, with eth0 connected to the Internet and eth1 offering DHCP service.

My interfaces file is as follows:

```
auto lo
iface lo inet loopback

auto eth0
iface eth0 inet static
        address 10.10.10.11
        netmask 255.255.255.0
        network 10.10.10.0
        broadcast 10.10.10.255
        gateway 10.10.10.10
        # dns-* options are implemented by the resolvconf package, if installed
        dns-nameservers 8.8.8.8 8.8.4.4
        dns-search dhcp.com

auto eth1
iface eth1 inet static
        address 192.168.10.1
        netmask 255.255.255.0
        network 192.168.10.0
        broadcast 192.168.10.255
        gateway 10.10.10.10
        dns-nameservers 8.8.8.8 8.8.4.4
```Everything works fine. The server connects to the Internet and the clients receive an IP address. The problems is that the clients can't access the Internet. I know it has something to do with the routes, and I have read several forums but none of the suggested solutions have worked for me. What can I do?

Heres my dhcpd.conf file:

```
authoritative;

subnet 192.168.10.0 netmask 255.255.255.0
{
        range 192.168.10.10 192.168.10.250;
        option domain-name-servers 192.168.10.1;
        option domain-name "cepatria.edu.mx";
        option routers 192.168.10.1;
        option broadcast-address 192.168.10.255;
        default-lease-time 600;
        max-lease-time 7200;
}
```My gateway is 10.10.10.10, and my routes are:

```
default via 10.10.10.10 dev eth0  metric 100 
10.10.10.0/24 dev eth0  proto kernel  scope link  src 10.10.10.11 
192.168.10.0/24 dev eth1  proto kernel  scope link  src 192.168.10.1 
```Any ideas?

### Post by jdthood on 2013-02-14
> **el_badbit said:**
> ```
auto eth1
iface eth1 inet static
        address 192.168.10.1
        netmask 255.255.255.0
        network 192.168.10.0
        broadcast 192.168.10.255
        gateway 10.10.10.10
        dns-nameservers 8.8.8.8 8.8.4.4
```

It actually isn't correct to have that dns-nameservers option in your eth1 stanza. The dns-nameservers option lists the nameserver addresses accessible *over that interface*, but nameservers 8.8.8.8 and 8.8.4.4 are not accessible over eth1 in your setup.
