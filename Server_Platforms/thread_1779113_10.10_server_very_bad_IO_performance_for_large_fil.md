### Post by RuralHunter on 2011-06-10
I set up a new ubuntu 10.10 64bit server and found very bad I/O performance for large files. It looks that the initial IO performance is good. but when the I/O operation continues, the performance drops very quickly. I tried to cp/gzip one file(100G) and 10 files(2G each). in both cases I can see the performance of first 2-3G is very good. but after that the performance drops sharply(from 2G/several seconds to several MB/every min). The operation is on my raid0 disk(set up with sdb and sdc). I'm crazy why this happens. I have another server with very similar set up and there is no problem there at all. I ran smarttools to check the disk and it reported the disk status is healthy. Here is some statistics I can found(when running gzip -d [large file]):
