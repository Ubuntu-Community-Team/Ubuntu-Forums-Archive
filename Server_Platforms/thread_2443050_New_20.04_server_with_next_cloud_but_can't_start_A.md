### Post by LHammonds on 2020-05-11
What is the result of this command?

```
snap list
```

If NextCloud was installed as a snap package, it might have bundled Apache2 with it.  If you also installed LAMP, it might have installed a separate instance of Apache2 and thus, both Apache services cannot operate (bind) on the same default ports (80).

I am not very familiar with snap since I do not use it yet...nor have I started converting any of my 18.04 production servers to 20.04.

**EDIT:** FYI, you should not be using "service" anymore.  Use systemctl.   Example: **systemctl start apache2** or **systemctl status apache2** or **systemctl stop apache2**

### Post by SeijiSensei on 2020-05-11
If nextcloud is bound to port 80, Apache cannot also bind there.

You say you changed the port in Apache, but you need to do that in a couple of different places.  First, in /etc/apache2/ports.conf you need to change 

```
Listen 80
```

to "Listen 81" or whatever other port you've chosen.  You also need to edit /etc/apache2/sites-available/000-default.conf and change
```
<VirtualHost *:80>
``` 
to "*:81" or whatever.  Then restart Apache.
