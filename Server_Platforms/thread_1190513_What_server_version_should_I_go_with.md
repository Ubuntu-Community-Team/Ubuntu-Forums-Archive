### Post by Vunutus on 2009-06-17
I realize that it's preferred to go with LTS releases on servers for obvious reasons (distro upgrades are messy enough on non-critical home systems).

Now, I know hardy servers are supported until 2013, which is quite a while, but I'm always one to prefer more current versions of software, especially operating systems.

The network that I'm going to be setting up soon is due to launch September of this year. That puts me in a little bit of a strange position. I recall reading that the next LTS release is April 2010, which means the network is launching only a little more than half a year before the LTS release.

My goal is to switch everything to the next LTS release whenever it comes out then stick with that for its life cycle. My question is: which release I should run in the time until the next LTS is launched? Would running a more recent server release (Jaunty) ease the eventual transition, or should I run Hardy?

### Post by hictio on 2009-06-18
I'd suggest the current LTS, Hardy, so, if there is any problem and you can't switch to the newest and brightest for whatever reason, you are still running on a version that will get updated for a while (hopefully, untill you workout whatever was the problem that didn't allow you to switch in the first place ;) )

### Post by ghen on 2009-06-18
Best intentions of course would be to migrate to the latest LTS, but without knowing feature set and what packages it will break that are mission critical for you its impossible to say. I would definitely stick with hictio's recommendation and use 8.04 due to that fact that you might be stuck with the release you install for a longer time than the anticipated April-May 2010.
