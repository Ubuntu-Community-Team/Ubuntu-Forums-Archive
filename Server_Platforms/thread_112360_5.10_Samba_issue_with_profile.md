The important things are to be searched in smb.conf and that machine accounts are generated on Linux box. This can be done automatically via script, too (skript that is controlled by smb.conf)

Oh - and before I do forget: You definitively NEED a valid machine GroupID. Mine is 3000, here.

Here an abstract of smb.conf that was key part for me:
(currently I don't have time. If I am more relaxed then I will generate a better description for the case of a small PDC box. I found lot of outdated docs that try to describe the way with LDAP. A little bit oversized for a 10 client PC installation - right?)
#=Domain related stuff========== Global Settings 
   domain logons = yes 
   domain master = yes
   preferred master = yes
   local master = yes
   os level = 200

# this is necessary for joining into Domain users have to be member
# of this group
   admin users = @smbadmin

# group is named 'machines' somehow it worked not :(
#   add machine script = /usr/sbin/useradd -d /dev/null -g machines -s \
/bin/false %u

# so I provided GID
   add machine script = /usr/sbin/useradd -d /dev/null -g 3000 -s \
/bin/false -m %u
   username map = /etc/samba/smbusers

## attention! all new users need to have this
## chown -R <user>:<user> /srv/samba/home/<user>/profile/
## AFTER they logged in first time because owner is set to root. Maybe 
## a bug in Samba?
   logon home = \\%L\%U
   logon path = \\%L\%U\profile

#=Domain related stuff========== Global Settings
