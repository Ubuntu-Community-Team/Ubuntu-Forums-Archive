### Post by SeijiSensei on 2011-02-05
If you're connecting to a server from a known set of network addresses, just use iptables to enable those addresses and block everything else.  Even if you have dynamic addressing, if you know your network prefix, you can limit potential intruders to a relatively small number of hosts.

Suppose, for instance, you generally have an address in 24.24.24.1-255.  You can add iptables rules like this:

```
iptables -A INPUT -j ACCEPT -p tcp -s 24.24.24.0/24 -d 0/0 --dport 22
iptables -A INPUT -j REJECT -p tcp -d 0/0 --dport 22
```

The first rule allows SSH connections from hosts in 24.24.24.1-255; the second denies connections from anywhere else.

If your address varies more than within an "octet" (1-255), you may need to use larger subnets like 24.24.0.0/16, but even that only permits 64K hosts to connect out of the millions of machines on the Internet.
