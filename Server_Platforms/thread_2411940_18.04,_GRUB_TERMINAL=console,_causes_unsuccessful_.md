### Post by strangerthanbland on 2019-12-23
I've experienced some boot and shutdown *hang time* from Ubuntu derivatives (lately Mint), not sure if any of the following will help but at least your thread'll be a little more active with possible solutions.

Reading through one of the more recent [release notes]("https://www.linuxmint.com/rel_tina_cinnamon.php") (near the bottom) *"Solving freezes during the boot sequence"* states that setting `nomodeset` may help; digging a little deeper, according to [Dell support]("https://www.dell.com/support/article/us/en/04/sln306327/manual-nomodeset-kernel-boot-line-option-for-linux-booting?lang=en") regarding Intel states that this'll, *"disable the intel graphics features"*. Seeing that you're wanting to run headless (without a monitor), I'm thinking that disabling various graphics features may prevent the system from expecting a display.

During boot process, inconsistently, I'd find the initramfs shell popping because the root partition wasn't getting found in time; setting `rootwait` within the list of `GRUB_CMDLINE_LINUX` stuff seems to have resolved it.

Note, Grub edits *should* go into the `/etc/default/grub` file (or related directories), **not** in the `/boot/grub` path, and a `sudo update-grub` command should follow to prevent Apt related upgrades from obliterating customizations.

The other thing that seems to have helped the boot process was running `sudo update-initramfs -u` after installation to update the existing initramfs with modules and stuff.

Shutdown hang time, for me, was resolved by opening the Driver Manager and selecting the latest drivers from various manufactures.

Considering that your system is *Intel inside* it might also be a good idea to install, and keep up to date, the Intel Microcode packages; think that may look a bit like `sudo apt-get install microcode.ctl intel-microcode`

If ya get things resolved please let us all know, because I came from web-search results and others in the future likely will too.
