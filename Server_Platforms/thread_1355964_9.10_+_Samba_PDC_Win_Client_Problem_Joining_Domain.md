### Post by demonGeek on 2009-12-15
I'm just setting up my first Ubuntu 9.10 server to (hopefully) replace a W2K PDC that I'm running at home.  I understand networking principles from a Windows perspective but this is my first experience with Linux.

The OS install went flawlessly.  The only software I added at install time was SSH.  After the install I did an apt-get update/upgrade and then modified the network settings to give the machine a static IP etc.

After that I followed [the official docs]("https://help.ubuntu.com/9.10/serverguide/C/samba-dc.html#samba-pdc-smbpasswd") to set up an Ubuntu PDC with Samba.  My smb.conf looks exactly like the examples given in the docs (with the exception of a different domain name).

Now, when I try to join the domain using a Windows client (XP Pro or Win2K) I get a message telling me that the following error occurred attempting to join the domain: 'The user name could not be found'.

I have verified that the domain is visible to the Windows machine and that the user name/password is being accepted (to some degree) - if enter a non-existent user name or invalid password it reports a 'Unknown user name or bad password' instead of 'The user name could not be found'.

So it appears to be authenticating to some degree - a fact which I can confirm by viewing (from Windows) the default shares on the Ubuntu machine where I can create files without problem.

I'd guess that I've missed something in my server setup that should have been done alongside setting up Samba as a PDC but I don't have a clue what it might be.  

Any help would be greatly appreciated.

 - Adam

### Post by munky99999 on 2009-12-15
Samba 3 isnt quite capable of taking on the job as a PDC in win 2k or higher. Sure there's a hack which could get it to kick in... but it wont work so well.

Samba 3 only does windows NT domains.


Now Samba 4 which is in alpha right now is supposed to get up to win 2k3 active directory. Personally I couldnt get the backend working on karmic for samba 4 to get going.

So sadly right now. Linux just cant take on the role as active directory domain controllers.
