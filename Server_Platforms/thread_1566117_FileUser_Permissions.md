### Post by Raptor354 on 2010-09-02
I'm trying to set up an Apache webserver that allows SSH and FTP connections into it.  I got the webserver part and SSH part working satisfactorily, but no matter what I try, I can't get FTP to work.

When I connect through, say, Filezilla, I can log in and browse directories, but I can't create anything.  I'm logging in as 'developer' and I'd really like to use /var/www/ as the web root.

My question: how do I allow 'developer' full read/write access to /var/www/ and, when he logs in via FTP, is limited to only that directory

If anybody sees a better way to do it, please say so!

Edit: By the way, I'm using vsftpd.  If anybody has any other recommendations, I'm open to suggestions.

### Post by Bachstelze on 2010-09-02
Make /var/www the home directory of the user, and give it ownership of it (chown -R developer /var/www).

And activate chrooting in your ftpd.
