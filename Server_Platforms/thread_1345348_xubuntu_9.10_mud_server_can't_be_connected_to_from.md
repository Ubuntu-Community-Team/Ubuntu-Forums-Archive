### Post by riste on 2009-12-04
Running Xubuntu instead of the server edition shouldn't prevent it from working.

Can you connect to the mud software from the machine that's running it?

If not, I'd look again at the configuration for the program you're using, make sure you've followed the install steps correctly, etc.  If you can connect locally but not remotely, it sounds like it might be a network or a firewall issue.

Things that might be useful to know include what MUD software you're running, the exact error messages or program output you get when you try to connect to it, and any messages the MUD software might log when a connection attempt is made.  You might find a solution by just Googling for these error messages if you're lucky.
