### Post by SeijiSensei on 2018-05-11
On 16.04 you should be able to add the entries to /etc/resolvconf/resolv.conf.d/base and make no other changes.

```

#     DO NOT EDIT THIS FILE BY HAND -- YOUR CHANGES WILL BE OVERWRITTEN
search teknet int.xxx.dk dev.xxx.dk opasia.gkl.xxx.dk dk.eng.xxx.net
nameserver 10.10.10.10
nameserver 10.10.10.11

```
This will all appear ahead of the entry for 127.0.0.1 in the generated resolv.conf file so those servers will be consulted first.

On 18.04, DNS resolution has been changed yet again (please stop doing this, Canonical).
