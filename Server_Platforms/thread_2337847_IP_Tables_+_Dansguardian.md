### Post by SeijiSensei on 2016-09-22
First, add the keyword "transparent" to the http_port directive in squid.conf:
```
http_port 3128 transparent
```

Next, you'll need to add some iptables rules like these:
```

/sbin/iptables -t nat -A PREROUTING -p tcp -s 10.10.10.1 --dport 80 -j ACCEPT
/sbin/iptables -t nat -A PREROUTING -p tcp -s 10.10.10.2 --dport 80 -j ACCEPT
/sbin/iptables -t nat -A PREROUTING -p tcp -s 10.10.10.3 --dport 80 -j ACCEPT
/sbin/iptables -t nat -A PREROUTING -p tcp -s 10.10.10.4 --dport 80 -j ACCEPT
/sbin/iptables -t nat -A PREROUTING -p tcp -s 10.10.10.5 --dport 80 -j ACCEPT
/sbin/iptables -t nat -A PREROUTING -p tcp -s 10.10.10.0/24 --dport 80 -j REDIRECT --to-port 3128

```
Those rules would exempt 10.10.10.1-5 from the proxy while forcing all the other machines in the subnet through Squid.

How all this interacts with dansguardian is outside my experience since I've never needed it.  At the client for whom I maintain a Squid proxy, we construct our own ACL lists of banned hosts and URLs.

### Post by SeijiSensei on 2016-09-24
Squid lets you write a wide variety of rules to control access by domain name, hostname, URI, source IP, file extension, and many other things.  We also use SquidClamAV to funnel each object to clamd for virus scanning before being passed back to the requesting user.  This all runs on a pretty hefty box with dual Xeons, but it has other tasks as well like scanning all incoming mail for spam and viruses with MailScanner.

The major drawback to Squid concerns SSL connections.  While there is a way to pass https requests through Squid, it requires creating a certificate on the Squid box and installing it on all the clients. (This avoids the proxy appearing as the source of a "man-in-the-middle" attack.)  My client and I discuss doing this from time to time, but have yet to do so.  (They have 250+ client workstations that would need to be configured.)  So we bypass Squid altogether for https requests and use iptables rules instead to block access to places like Facebook.
