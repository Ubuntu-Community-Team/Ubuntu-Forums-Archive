### Post by Bill_Parrott on 2014-07-29
I have spent the past couple hours searching the forums here and elsewhere and trying everything I can find, to no avail. I have an Ubuntu 14.04 server with Apache2 installed. I can access the sites on it successfully, but if I try to use mod_rewrite or .htaccess files, they are completely ignored. Here's what I have:

.htaccess
```
Options +FollowSymLinks
RewriteEngine On
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule . /index.php [L]
```

apache sites-enabled/000-default.conf
```
<VirtualHost *:80>
    EnableSendfile Off

    ErrorLog ${APACHE_LOG_DIR}/error.log

    # Possible values include: debug, info, notice, warn, error, crit,
    # alert, emerg.
    LogLevel warn

    CustomLog ${APACHE_LOG_DIR}/access.log combined
    ServerSignature On

    ServerName www.local.dev
    ServerAlias *.vm.dev *.vm.prd *.local.dev *.local.prd *.python.dev *.python.prd *.sf.dev *.sf.prd
    DocumentRoot /var/www

    <Directory />
        Options Indexes FollowSymlinks MultiViews
        AllowOverride All
        Order allow,deny
        allow from all
    </Directory>

    <Directory /var/www/>
        Options Indexes FollowSymlinks MultiViews
        AllowOverride All
        Order allow,deny
        allow from all
    </Directory>

    DirectoryIndex index.html index.php app.php index.shtml

    RewriteEngine On

    RewriteMap lowercase int:tolower

    # *.(local|vm).(dev|prd)
    RewriteCond ${lowercase:%{SERVER_NAME}} ^[a-zA-Z0-9-]+\.(local|vm)\.(dev|prd)$
    RewriteRule ^(.+) ${lowercase:%{SERVER_NAME}}$1 [C]
    RewriteRule ^([a-zA-Z0-9-]+).(local|vm)\.(dev|prd)/(.*) /var/www/$1_$3/public_html/$4
</VirtualHost>
```

The rewrite rules in my default.conf file work correctly, so I know that mod_rewrite is working. But it looks like it is completely ignored when I call it from the .htaccess file. I tried adding another <Directory> section, but that didn't have any effect either. The .htaccess file has 777 permissions from VirtualBox, so Apache should be able to read it just fine.

I am at wit's end here. Any ideas what could be causing this?

---

### Post by Bill_Parrott on 2014-07-29
FINALLY. I figured it out! It turns out the MultiViews directive was  messing things up. I removed it, and things work correctly now.
