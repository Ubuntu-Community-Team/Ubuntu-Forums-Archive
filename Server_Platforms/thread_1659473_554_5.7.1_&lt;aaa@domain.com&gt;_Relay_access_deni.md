### Post by SeijiSensei on 2011-01-04
Let's first make sure your ISP isn't blocking outbound SMTP connections.

For the domain you used above, try this:

$ host -t mx domain.com

That will return the MX host(s) for domain.com.  Choose one (call it mx.domain.com as an example], then try

$ telnet mx.domain.com 25

You should see a banner from the remote SMTP server; if you don't and the connection hangs, you're being blocked.

### Post by Calimo on 2011-01-04
> **FloM said:**
> ```
mydestination = testserver.local, testserver, localhost.localdomain, localhost

```

With this line, postfix thinks it's not the destination of mails to your domain.com. Therefore, it'll try to forward it to the next server. But your server is configured to deny forwarding (which is good, otherwise it would be used by spammers!) So… it's blocked.

What you need to do is telling postfix to accept mails to domain.com. Change the previous line to:
```
mydestination = domain.com, testserver.local, testserver, localhost.localdomain, localhost

```
