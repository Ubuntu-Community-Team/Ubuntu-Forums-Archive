### Post by papibe on 2014-04-02
Hi  wlraider70.

My thoughts:

Yes, if you later intent to recover files. For instance, you samba config file (/etc/samba/smb.conf).

No, if you want to recover objects that have a state. For instance, you don't back up a live database. You either stop it, or create a special dump for backup.

No, if you want to create a bootable system with it. In this case, you may want to use a tool to copy partitions, like gparted, or a proper imaging tool like clonezilla.

Hope it helps.
Regards.
