### Post by oke on 2012-10-24
Looking more deeper into this issue I saw that freeradius fresh installs with service start default at position 01 in rcN.d. Hence, I changed position to almost max (100) by following commands (see [http://stackoverflow.com/questions/12080294/update-rc-d-and-init-d-dependencies]("http://stackoverflow.com/questions/12080294/update-rc-d-and-init-d-dependencies") and manpages update-rc.d):
```

cd /etc/init.d
sudo update-rc.d -f freeradius remove
sudo update-rc.d freeradius defaults 98 02

```
Freeradius is now at position 98 in rcN.d as expected. After shutdown and restart freeradius is starting up during boot without problems **after** mysql :). Note that I had mysql nas table query enabled (readclients = yes in sql.conf) for clients such that freeradius immediately contacts mysql after starting. 

@Krijn Tanis:
Maybe repeating this procedure to alter start position also works for you. Otherwise, there is a hint to put $all in the LSB init script at the "Required-Start" or "Should-Start" position in /etc/init.d/freeradius. The "virtual" facility name $all is used to start a script after all the other scripts, at the end of the boot sequence (see [http://wiki.debian.org/LSBInitScripts]("http://wiki.debian.org/LSBInitScripts")). Happy puzzling!

### Post by oke on 2013-02-23
After last "safe-upgrade" today in Ubuntu 12.04.1 (3.2.0-38-generic-pae #61-Ubuntu SMP Tue Feb 19 12:39:51 UTC 2013 i686) the issue returned after reboot. Unfortunately, the trick of changing the service start position of freeradius didn't work anymore. Also fiddling with $all in the LSB init script of freeradius didn't the trick.

This problem seems to be difficult to reproduce on a new build. See also [https://bugs.launchpad.net/ubuntu/+source/freeradius/+bug/1081509]("https://bugs.launchpad.net/ubuntu/+source/freeradius/+bug/1081509").

Pragmatic solution is similar and simple as that given in [http://ubuntuforums.org/showthread.php?t=1728094]("http://ubuntuforums.org/showthread.php?t=1728094"). Add ```
service freeradius restart
``` to "/etc/rc.local". Might introduce redundancy if you don't have the issue (yet), but works anyhow.
