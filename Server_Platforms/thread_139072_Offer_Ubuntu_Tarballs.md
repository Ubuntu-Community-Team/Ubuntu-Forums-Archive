### Post by hachre on 2006-03-03
Hi there,

I think that the Ubuntu Server branch looks quite good especially now with 6.04. However I'm missing something I'd really like to see offered.

I often have to install distros on Server to which I have no physical access. That means I cannot install Ubuntu there with a CD-Rom.
Normally those Computers are started up via a minimal networked "rescue system" which offers full access to the harddisks for partioning, the internet, and so on. Those often include the standard tools, chroot, tar, wget and so on.

Installing Ubuntu on machines like that isn't possible with a CD. I need a tarball of a very minimal ubuntu system that I can directly download to the machine. Then partition the harddrive via the rescue system and extract the tarball with the p (for permissions) parameter directly to the disk.

Afterwards I need to manually add a user and install grub so it can boot, restart and cross fingers.

When the machine comes up I have successfully installed a system and can start updateing it and installing more software via apt-get.


Normally I make those tarballs myself of course by installing the os of my choice on a local computer, grabbing a tarball, and using that for the next machine.

However I really thing it would be a fancy thing if Ubuntu offered tarballs for download.

Also I could imagine a little setup utility that can help you create the first user and install grub (since the tarball needs that done to be fully working)


This would offer people like me to be able to download the latest minimal server tarball from ubuntu, extract it onto a harddisk over a rescue system, start an application like ubuntu-setup, reboot and be done!

Would be really great in my opinion. What do you guys think?
