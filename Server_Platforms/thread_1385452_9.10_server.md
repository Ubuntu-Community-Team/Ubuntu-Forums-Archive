Looks like you have Apache installed, did you connect using port 10000? e.g., [https://192.168.1.2:10000](https://192.168.1.2:10000)

Anyway, here's the script I use to install webmin:

```
#!/bin/bash

WEBMIN=webmin_1.500_all.deb

#Install Webmin dependencies
echo "Installing Webmin dependencies..." && sleep 2
apt-get install perl libnet-ssleay-perl openssl libauthen-pam-perl libpam-runtime libio-pty-perl libmd5-perl --assume-yes >/dev/null && echo -e '\E[30;32m'"Completed."; tput sgr0 && sleep 5
#Download Webmin and install
if [ -f $WEBMIN ];
then
  echo "$WEBMIN already exists. Installing..." && sleep 2 && dpkg -i webmin_1.500_all.deb && echo -e '\E[30;32m'"Webmin installed successfully."; tput sgr0 && sleep 5
else
  wget http://prdownloads.sourceforge.net/webadmin/webmin_1.500_all.deb && dpkg -i webmin_1.500_all.deb && echo -e '\E[30;32m'"Webmin installed successfully."; tput sgr0 && sleep 5
fi
```

### Post by myth1914 on 2010-01-22
I think you're attempting to GET webmin on the server.  

Go to the terminal on the server and use:

wget [http://prdownloads.sourceforge.net/webadmin/webmin_1.500_all.deb](http://prdownloads.sourceforge.net/webadmin/webmin_1.500_all.deb)

That will download the file.  If you don't know the path it downloaded to (should be /home/<user_name>) run this:

sudo updatedb && locate webmin_1.5*

which will output the file location.  THEN install with:

dpkg -i <path_to_webmin>

### Post by CharlesA on 2010-01-22
You shouldn't need to. You can use wget to get webmin and then install the dependencies and whatnot.

```
#!/bin/bash

#Install Webmin dependencies
apt-get install perl libnet-ssleay-perl openssl libauthen-pam-perl libpam-runtime libio-pty-perl libmd5-perl
#Download Webmin and install
wget http://prdownloads.sourceforge.net/webadmin/webmin_1.500_all.deb && dpkg -i webmin_1.500_all.deb
```

You can then access webmin from [https://your.ip.goes.here:10000](https://your.ip.goes.here:10000)
