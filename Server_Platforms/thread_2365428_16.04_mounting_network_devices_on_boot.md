### Post by mcowart123 on 2017-07-06
Is this completely broken? I cannot get either NFS or iSCSI volumes to mount on boot (the _netdev option in /etc/fstab does nothing). After boot with the network fully up everything works fine, but I need them to mount on boot in the proper order (one of my servers is serving NFS off an iSCSI volume, which when it tries to start up it hangs forever (there is no timeout on the NFS server attempting to start)).

I've added the 'x-systemd.automount,nofail,noauto' workaround, but that is flaky at best, and does not solve the NFS server issue. This wasn't a problem at all in 14.04.

EDIT: I've managed to 'solve' this in a pretty convoluted way. I had to create a systemd service that calls a script to mount the drives and start NFS, but even after adding all the dependencies I still had to add a 10 second wait to get it to actually work.

Script:
```

#!/bin/bash


#do these here since systemd can't handle a simple network mount on boot
sleep 10
mount /var/sanstorage
mount /var/nasstorage


systemctl start nfs-server

```

Service:
```
[Unit]Description=Post Boot (iSCSI mounts)
After=network.target iscsi.service iscsid.service multipathd.service multipath-tools-boot.service


[Service]
Type=oneshot
ExecStart=<PATH TO SCRIPT HERE>


[Install]
WantedBy=multi-user.target


```
