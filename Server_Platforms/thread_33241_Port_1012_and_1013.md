> There could be messed with netstat, a so called rootkit, but that's very exceptional on home computers. I wouldn't worry too much about it...**No, wrong**.  This is the absoutely wrong kind of thinking to have.
 
 If something odd that shouldn't be happeningcomes up, you just don't ignore it.  You figure out why and fix it.
 
First off, we need to know something.  Where did you run the nmap scan from?  If it was from localhost, you may be getting an odd false positive, but it still worries me the ports aren't shown in netstat.

The OP really needs to install losf and run:```
sudo lsof -i TCP:1012
``` and see if any process reporst being open on those ports.

If they don't, but nmap from a remote machine still reports those ports being open, you should immediately disconnect the machine from the network, by ripping it's cable out; boot off a known good rescue/live cd; and run a rootkit check program like chkrootkit or rkhunter.

You fail to understand this. If nmap reports open ports and the system says their not, there are only a few possiblities. One is that he scanning a router that has other mapped ports, which is why I asked how he was running nmap. Another is the box itself is doing NAT, but he would have to set that up himself and would be presumably aware of it. The other is that **his box has been rooted**.    

Since it's a very real possibility, you don't ignore it. This is as dumb as, and akin to saying , "Maybe if ignore the fact I came home to my door standing open nothing will be taken from house". It's just not logical.

He needs to boot a rescue cd and at the very least, do at rootkit scan. This is presuming of course, he ran the scan correctly in the first place. If he didn't, then he needs to run a proper scan to verify that it was indeed an incorrect scan.
