### Post by papibe on 2016-12-23
Hi Edgar_Oliveira.

A few thoughts...

It is not too long, however when you put too much commands in a single line it is difficult to debug it when you have errors. Also, adding functionality could be a pain.

I would put all the commands on a script:
```
0 5 24 * *  /path/to/script.sh
```
Make sure that the script starts with the line:
```
#!/bin/bash
```
and that it has executions permissions: 
```
chmod a+x  /path/to/script.sh
```

As for the actual code: you're using pipes to execute several commands, but you are not piping data through them. I recommend putting one command per line:
```
#!/bin/bash
date >> /path/scanclam/log.txt
clamscan -r / >>/path/scanclam/log.txt
date >> /path/scanclam/log.txt
echo "-----">>/path/scanclam/log.txt
echo "-----">>/path/scanclam/log.txt
```
Hope it helps. Let us know how it goes.
Regards.

### Post by deadflowr on 2016-12-24
Installing clamav-daemon and running it once will produce the clamd.conf configuration file.
from there you can set various options, like no verbose (to output only the summary in the logfiles.
As well as disable any log rotation, which will prevent logfiles from regenerating a new every time.
Then you can use the in-built --log=path-to-logging-file.

Basically run you command, but change the clamscan line to
```
clamscan -r / --log=path/to/file.log
```

the clamd.conf file will appear in /etc/clamav

you can get a idea of what you can do by reading it's manpage [man clamd.conf]("http://manpages.ubuntu.com/manpages/precise/man5/clamd.conf.5.html"),

Hope it leads you in a direction

### Post by SeijiSensei on 2017-01-01
From the command prompt you should run clamscan as root with sudo.

I usually filter the results through grep like this:
```
sudo clamscan / | grep -E -v 'OK|link'
```
This removes all lines with "OK" or "Symbolic link" from the output.  On a clean machine, you'll get only the summary results at the end.  Infected files will be reported since they won't return an "OK."
