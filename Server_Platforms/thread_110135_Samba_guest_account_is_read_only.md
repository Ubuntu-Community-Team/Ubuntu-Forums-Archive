### Post by TopDog on 2006-01-01
If you don't want to bother with access control at all, then this is my setup (*I got some very good help from LinuxCult for this*):

```
[global]
workgroup = WORKGROUP
netbios name = MYSERVER
server string = SAMBA server
security = share
local master = yes
os level = 35

[Share]
comment = Shared Folder
path = /var/shared
public = yes
browsable = yes
writable = yes
guest ok = yes
```

Then I ran a **chmod 777** on the **/var/shared** folder. Restart Samba at it worked.

### Post by guy_uk on 2006-01-26
if you want to say have some shares that are restricted to users, and also some shares that are completely open so to speak, you can also use

only guest = yes 

on the shares you want open

or 

only guest = yes
write list = @users

for example if you want anybody to read, but only a group of people to write

i don't know if this is the best way, but it allowed me to have problem free sharing and yet still keep control of my personal shares
