### Post by volkswagner on 2011-09-30
First let me ask do you really need shared hosting?

Do you need other site admins access to just there site?

Can you get away with just using A[pache2 and NameVirtualHosts]("https://help.ubuntu.com/10.04/serverguide/C/httpd.html")?

Hopefully yes to the latter, as it is fairly easy to setup and will not require additional software running eating precious RAM and disk Space.

You can even setup sites so they are pointing to /home/user/www and give the user FTP access and jail them to that directory.

You can easily just use your providers DNS control panel and add a domain name.  You friend can also add a domain name or you can create a subdomain from yours, ie: buddy1.mydomain.com and use the same DNS control panel to point that "A record" to your servers ip.  Apache will take care of the sorting.

Before I go into more detail lets us hear what will work best for you.  If you must have some sort of "shared hosting" you can look into ispconfig.

Here is a [list]("http://controlpanelblog.com/general/10-free-web-hosting-control-panels.html") of other alternatives, none of which I have tried.

### Post by Dangertux on 2011-09-30
Ok -- sorry if I agitated you I'm sure that response seemed a little short (I typed it from my phone, thank you new ati drivers for turning my laptop in to junk) 

However as far as shared hosting goes you can do it rather simply.  I will outline the topics, I'm not going to google a ton of tutorials I'm sure you can do that once you're on the right track.

SSH - you know how to do this, you can install it from repos or from source whichever floats your boat better. 

```

sudo apt-get install openssh-server

```*config file will be in /etc/ssh/sshd_config , keys will be in /home/username/.ssh


LAMP - Again, from repos or source.

```

sudo apt-get install lamp-server^

```*config will be in /etc/apache2/ for apache, /etc/mysql for mysql and /etc/php5/apache2/conf.d : default web root is /var/www , sql db's are stored by default in /var/lib/mysql 

Now how to accomplish your goal : I suggest virtual hosts, there is no reason you can't do this with one apache instance (unless of course these blogs get huge traffic which they apparently don't)

Basically you can set up vhosts in apache in terms of "sites" not just subdirectory based vhosts so if you were so inclined you could set up a site file that is for [www.brohoof-blog.com]("http://www.brohoof-blog.com") and the webroot would be /home/brohoof etc..

A simple example config would be something like this :

```
 
<VirtualHost *:80>
           DocumentRoot /home/brohoof
      ServerName www.brohoof-blog.com
      
      # Other directives here
      
         </VirtualHost>

```Then another would be for say ihearthp-ux.com

```

<VirtualHost *:80>
DocumentRoot /home/hpux
ServerName www.ihearthp-ux.com

# Other directives again...
</VirtualHost>

```So any incoming http requests bound for brohoof-blog.com would be directed to /home/brohoof and [www.ihearthp-ux.com]("http://www.ihearthp-ux.com") would go to /home/hpux (obviously more directives are needed in this example but you get the point, I'm sure. Keep in mind if you're doing this for SSL you need the proper number of certificates. 

You're on the right track as far as FTP jail users call it a day, also make sure to jail them in ssh as well, also dont't give them a shell unless they need it, shared hosting shouldn't.

As far as DNS records go you would basically just have the A record for each point at the same IP. CName's as appropriate if you need. 

It's pretty straight forward all around.
