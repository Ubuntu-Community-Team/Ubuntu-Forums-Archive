### Post by Bachstelze on 2010-05-19
> **fred2028 said:**
> What about any scalability issues? The server is for an internal website, and because it has very recent specs (2 Xeon quads, 12 GB RAM, etc.) the GUI wouldn't be a problem.

The main difference between the server and dektop versions is the version of the kernels that comes with it by default (the -server kernel is geared towards server-class hardware and the -generic kernel is for consumer-grade hadware). With that kind of hardware, you would probably be better off using the -server kernel. This does not mean you have to install Ubuntu with the Server CD,t hough, you can very well do a Desktop installation and install the -server kernel afterwards.

Long story short, a Desktop installation can be turned into a Server one with very little effort, and vice versa, so don't worry too much about it and just do what's simpler for you.

### Post by The Real Dave on 2010-05-19
The guys above have already explained the real differences, but I think it's where they're the same is important. Most things that you'll want your server to do, LAMP, Samba whatever, will be edited from the command line. 

Either way, you'll most likely use the same CLI tools, to set up the same things in the same way. So I don't see the point myself in using a GUI on a server. CLI is not difficult at all to learn, it just looks scary. I mean like, I'm 17, had never made a server before or worked with CLI and I got it going. A year on, and I'm flying. 

CLI is daunting, but for no good reason :) Read some tutorials, try out some things. Get yourself a VM or a spare machine, and experiment, just see what you can do. You don't need to remember all the commands, I sure dont. I generally have lists, or scripts to remember commands. 

If you really don't want to mess with CLI, use Webmin and PHPMyAdmin. Then all you need CLI for is to give a first time connection to the net and to install the programs.

There's a MUCH better method.

```
sudo apt-cache search ubuntu-desktop|grep Depends:|sed s/Depends:\ // |sed s/,//|xargs sudo apt-get remove --purge
``` This way all of the dependencies of ubuntu-desktop are taken care of, no matter what version of Ubuntu you're running.  If you change the two "depends" to "recommends", you'll get the remaining packages.  Or, just do an apt-get autoremove.  Should grab all the remaining recommended packages.
