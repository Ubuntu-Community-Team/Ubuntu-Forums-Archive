you could install with with usb instead of with the cd, i once did it like this in debian, same basic principal would apply to ubuntu i think..
just google arounf dor the specifics
 To download the installer we do as follows

```
$ wget http://ftp.nl.debian.org/debian/dists/etch/main/installer-amd64/current/images/hd-media/boot.img.gz

and we will also need the same iso file as when making an installation cd

$ wget http://cdimage.debian.org/debian-cd/4.0_r2/amd64/iso-cd/debian-40r2-amd64-netinst.iso

The exact mountpoint of the usb can be found with

$ fdisk -l

Assuming your usb is mounted as /dev/sda we can extract the boot.img.gz to the usb stick.

$ zcat boot.img.gz > /dev/sda

Then mount the usb.

# mount /dev/sda /mnt

when this is done simply copy the .iso to the same usb disk

# cp debian-40r2-amd64-netinst.iso /mnt

and then unmount the usb stick

# umount /mnt
```

### Post by cdtech on 2008-05-11
> Like I said I need to know If I can upgrade with a pci usb 2.0 card easily as I will also be using usb drives. 

Personally I would stay away from the Pent 111's due to the BIOS limitations. The Celeron looks promising.

> Can I take the cd drives / floppies out after install and use the extra cable for another 2 hard disks?

You sure can, I run 4 drives in my server (storage, file sharing etc..). The limitations are dependent on the power supply and the BIOS.

> Is a 10/100 card standard nowadays or should I put a quicker one in? 

Thats all I run for my home network, along with a Linksys wireless router.

> When I mentioned how does a router work, I meant is it like a hub or like a switch because I know that switches are faster because of the way they perform i.e. a hub is kind of like having all the ethernet ports in p2p so what is a router like? switch or router? 

The term layer 3 switch often is used interchangeably with router, but switch is really a general term without a rigorous technical definition.

> I will access it remotely for admin etc but as I'm still a noob I will install server edition and do apt-get kubuntu desktop for a gui.

Really no need to install gui on a server. Once you have it setup you can just slogin or use putty from a windows desktop. (learn the command line). I've not once put a monitor on my server, everything is through remote access.
