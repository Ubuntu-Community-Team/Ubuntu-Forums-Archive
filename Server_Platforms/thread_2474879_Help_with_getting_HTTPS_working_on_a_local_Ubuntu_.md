### Post by CharlesA on 2022-05-11
Hi,

Chrome (and probably other browsers too) look at the Subject Alternative Name (SAN) record in the SSL certificate instead of the Common Name (CN) value to determine what domain the certificate is for. This might not be included by default.

If this is a self-signed certificate, you can force SAN when generating the certificate via the openssl command. More info here: [https://geekflare.com/san-ssl-certificate/](https://geekflare.com/san-ssl-certificate/)

If you want to use a signed certificate, you can get one from LetsEncrypt via any number of ACME clients (I believe certbot is the recommended one).

Since you mentioned Apache, this might be helpful: [https://www.digitalocean.com/community/tutorials/how-to-secure-apache-with-let-s-encrypt-on-ubuntu-20-04](https://www.digitalocean.com/community/tutorials/how-to-secure-apache-with-let-s-encrypt-on-ubuntu-20-04)

### Post by SeijiSensei on 2022-05-11
I'm using acme.sh these days but invoking the LetsEncrypt server which is much more reliable than the default one at zerossl.com.

```
acme.sh --issue -d main.domain.name -d alternate.domain.name --server letsencrypt --apache
```

This will issue a single certificate authenticating both main.domain.name and alternate.domain.name.

For this to work, those names must be associated with a valid DNS domain. If not, you might be able to use the "--webroot" method. See "acme.sh --help" for details.

### Post by CharlesA on 2022-05-11
> **Commandrkeen said:**
> I would like to use a signed certificate. Can I use this article to create a signed certificate that is for local development of sites? I don't have anything live yet.

Yes, that's the whole point of letsencrypt. The certs are only good for 90 days, but they support automated renewal, so you can script the renewal and not have to worry about paying for the certificate.

If you don't want to go that route, another option might be to put cloudflare in front of your live site because they can do SSL with a cloudflare signed certificate and a self-signed or signed certificate on the target server (I use a lets encrypt cert myself).
