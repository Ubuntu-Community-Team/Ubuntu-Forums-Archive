### Post by darkod on 2016-03-16
In the reverse zone the IN NS usually would not be localhost (even though the same machine is DNS server). Put the hostname instead (the short, not the FQDN hostname). You must have given the machine some name during install. Check the FQDN with:
```
hostname -f
```

Use the first part of the FQDN in the reverse zone as:
```
@   IN   NS   <host>.
```

Don't forget the trailing dot at the end.

As for the rest of the FQDN, even if you don't have proper domain set up, you probably have one as part of the hostname -f, so you can simply use that if it doesn't matter for you. In general you should enter any basic local domain during installation even if not running full domain. A mycompany.local is enough.

More info here: [https://help.ubuntu.com/lts/serverguide/dns-configuration.html](https://help.ubuntu.com/lts/serverguide/dns-configuration.html)

### Post by SeijiSensei on 2016-03-16
You need the name of the reverse domain in the SOA record, not "localhost".  My reverse file for the 192.168.100.0/24 subnet looks like this:
```

@       IN      SOA     100.168.192.in-addr.arpa. root.localhost.  (
                        2016012801 ; Serial
                        28800      ; Refresh
                        14400      ; Retry
                        3600000    ; Expire
                        86400 )    ; Minimum
                                      
        IN      NS      ns.example.com.

1       IN      PTR     gateway.example.com.
[etc.]

```

### Post by SeijiSensei on 2016-03-16
If you are using isc-dhcp-server to distribute addresses, there are ways to have it [communicate with BIND]("https://www.google.com/search?client=ubuntu&channel=fs&q=isc+dhcp+bind&ie=utf-8&oe=utf-8"). I don't manage any large networks, so my reverse zone is managed by hand.

Before that feature was available, I wrote a bash script to scan the network and compile both the forward and reverse zones.  Here it is for your amusement.  I wrote this for a RedHat server where the zone files are stored in /var/named/chroot/var/named.  You can modify the variables at the top to fit your situation.  It writes a log in HTML which was intended to be viewed from a web browser.

```

#!/bin/bash

DOMAIN="example.com"
PREFIX="172.25"
SUB="151"

#RANGE=1-254            # if empty, range is entire /24 subnet
RANGE=

# where i live
ROOTDIR=/usr/local/sbin

# where the zone files live
ZONEDIR="/var/named/chroot/var/named"
#ZONEDIR="/var/named/test"

# user name that BIND runs as
BINDUSER="named"

### END OF USER-SERVICEABLE PARTS ###

# create lockfile to guarantee only one process running
if [ -s /tmp/census.lock ]
then
    echo "$0 is locked!  Please delete /tmp/census.lock to continue."
    exit
fi    
touch /tmp/census.lock

# forward domain zone file
A="$ZONEDIR/census.$DOMAIN"

# reverse IP zone file prefix
PTR_PREFIX="$ZONEDIR/census.ptr"

# header for html output of this script
HDR="$ROOTDIR/nmbscan.header.html"

# override subnet list at command line
[ "$1" != "" ] && SUB=$1

BACKUP_DIR="$ZONEDIR/census-backup"
NOW=`date +%y%m%d`

# delete backups older than 8 days
STALE=`date +%y%m%d --date='8 days ago'`
rm -rf $BACKUP_DIR/$STALE

BACKUP=$BACKUP_DIR/$NOW
mkdir -p $BACKUP

#### BEGIN ####

# backing up yesterday's files
rm -f $BACKUP/*
cp -f "$A" $BACKUP
A_NEW="$A.new"

echo "; generated by network-census.sh" > $A_NEW
echo -n "; on " >> $A_NEW
echo `date` >> $A_NEW

echo "<html>

<head>
<title>Network Census</title>
</head>

<body>
<h2>Network Census for $DOMAIN</h2>

";

for s in $SUB; do

    if [ "$RANGE" = "" ]
    then 
        SUBNET="$PREFIX.$s.0/24"
    else 
        SUBNET="$PREFIX.$s.$RANGE"
    fi
    
    # display subnet header in html log file
    echo -n "<h3>Scanning $SUBNET beginning at "
    echo -n `date`
    echo " </h3>"
    echo "<pre>"

    # create reverse domain file for this subnet
    PTR="$PTR_PREFIX.$PREFIX.$s"
    
    # save yesterday's as backup
    cp -f $PTR $BACKUP
    # write to new file
    PTR_NEW="$PTR.new"
    
    echo "; generated by network_census.sh" > $PTR_NEW
    echo -n "; on " >> $PTR_NEW
    echo `date` >> $PTR_NEW
    
    # use nmap ping scan to find active hosts
    echo "Using nmap to scan subnet $SUBNET"
    HOSTS=`nmap -n -sP $SUBNET | grep 'up\.' | sed s/\(//g | sed s/\)//g | awk '{print $2}'`
    
    
    # scan the subnet with nmblookup
    for ADDR in $HOSTS
    do    
       # get the final octet
       IP=`echo $ADDR | awk -F. '{print $4 }'`

       # find the NMB hostname
       LINE=`nmblookup -A $ADDR | grep "<00>" | grep -v "~" | grep -v "GROUP"`
       
       # trim the result to include just host's name
       HOST=`echo $LINE | sed s/".<.*"//g | sed s/\ /\-/g | tr A-Z a-z | sed s/_/-/g`
       
       if [ "$HOST" != "" ] 
       then 
           # display host and create the forward and reverse records
           echo "$ADDR <--> $HOST"
           echo "$HOST          IN      A       $ADDR"          >> $A_NEW
           echo "$IP            IN      PTR     $HOST.$DOMAIN." >> $PTR_NEW
       fi
       # end of host address
       
    done
    
    
    echo "</pre>"
    
    
    # write summaries    
    echo -n "<h4>"
    echo -n "`grep "IN" $PTR_NEW | wc -l`"
    echo " hosts found in $SUBNET</h4>"
    
    echo -n "<h3>Finished scanning $SUBNET at "
    echo -n `date`
    echo " </h3>"
    echo "<br><hr><br>"
    
    mv -f $PTR_NEW $PTR
    
# end of subnet
done

# update domain zone
mv -f $A_NEW $A

cd $ZONEDIR
# update serial numbers
SERIAL="$(date +%Y%m%d%H)"
sed s/SERIAL_HERE/$SERIAL/g < $DOMAIN.template      > $DOMAIN
sed s/SERIAL_HERE/$SERIAL/g < $PREFIX.$SUB.template > $PREFIX.$SUB.rev

chown $BINDUSER:$BINDUSER $ZONEDIR/*

# restart nameserver to use these new files
# actual zone files have $INCLUDE statements that
# link to the files created by this script

echo "<h3>Restarting nameserver</h3>"
echo "<pre>"
/etc/rc.d/init.d/named restart
echo "</pre>"

echo "</body>"
echo "</html>"

# unlock this process
rm -f /tmp/census.lock

exit 0

```
This requires the nmap package and nmblookup from the Samba suite.  You may or may not have these installed already.

I ran the script at midday with cron when most of the client machines would be in use.  You could run it more frequently depending on how much change there is in your network.  

You need to create two "template" files with all the zone information except for the A and PTR records.  Replace the serial number with the text "SERIAL_HERE", e.g.,

```

@               IN     SOA      example.com. support.example.com. (
                                SERIAL_HERE ; serial

```
Each template also needs an [$INCLUDE](http://www.zytrax.com/books/dns/ch8/include.html) directive that points to the forward and reverse zones.  You can also use BIND's [custom "include" statement]("http://www.zytrax.com/books/dns/ch7/include.html") instead.
