I had some progress with it. I found articles about how to do it.
Since i already had installed from the repositories the php5.3 i downloaded the php 5.2, extracted it, configured it with --prefix /usr/local/php52 and the rest of the options that i wanted and then make && make install.

After that i went into my apache.conf for virtual hosts and i added:

```

<VirtualHost xxx.xxx.xxx.xxx>
    ScriptAlias /cgi-bin-php/ "/usr/local/php52/bin/"
    <Directory "/usr/local/php52/bin/">
    AllowOverride All
    Options None
    Order allow,deny
    Allow from all
    </Directory>
    SetEnv PHP_INI_SCAN_DIR "/usr/local/php52/etc/"
    AddHandler php-script .php
    Action php-script /cgi-bin-php/php-cgi

    ServerName domainaddress.com
    DocumentRoot /var/www/development
    ServerAdmin urwebmaster@domainaddress.com
    ErrorLog /var/log/apache2/development-errorlog.log
    CustomLog /var/log/apache2/development-commonlog.log common
        php_flag display_errors off
        php_flag log_errors on
        php_value error_log /var/www/development/PHP_errors.log
 </VirtualHost>

```

The problem is that it doesnt work. Apache can be restarted without errors but when i create a file in my /var/www/development directory which has the phpinfo(); and i go on my browser and type the domainaddress.com i get the 5.3 version of php.

Note: i have virtualhosts on the same ip but for different servername that look on different documentroot

UPDATE:
i managed to make it work for .php5 files
The website works with a core file index.php. So i decided as a temp solution to rename the file to index.php5 but leave all the links to point to index.php and have a redirect rule to change the index.php to index.php5.
This way search engines can keep both of the links php and php5 and still work fine.
Once i will decide to upgrade the website for 5.3 i will use the reverse rule so that old links that point to index.php5 will be automatically be renamed to index.php

>         RewriteEngine on
        RewriteRule ^(.*)index\.php$ $1index.php5 [R=302,L]
        ScriptAlias /cgi-bin/ /usr/lib/cgi-bin/
        AddHandler php-script .php5
        AddType text/html .php5
        Action php-script /cgi-bin/php52
        DirectoryIndex index.php5

### Post by phyerlight on 2010-10-14
I think I figured out a way to get around the redirects while still being able to run versions (some forum posts were satisfied by disabling php 5.3 when they wanted to test 5.2).

Add this snippet to the apache virtual host to force the cgi script to handle files with php extension:
```

<FilesMatch "\.php">
    SetHandler php-script
</FilesMatch>

```

I got this idea from the /etc/apache2/mods-enabled/php5.conf file :)
