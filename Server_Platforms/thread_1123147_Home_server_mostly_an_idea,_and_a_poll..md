### Post by balloooza on 2009-04-11
Before I go anywhere, I need to tell you what my terms are (by reading this post you accept them)
A HOME SERVER is somthing ABSOLUTELY NOT for a REAL network admin, it is not a server for running a mail server, and a big company.

In this case a home server is a server with media capabilities, a nice web interface, and also usable as a normal desktop computer.

WHY, (if you are asking that you still have the wrong idea)
Immagine a world where if you wanted to use your alarm clock, and to set it, you had to issue some commands over an ssh to set the time and the alarm time, and become sudo o activate the snooze function. That seems unnecessary, well, a home server should have this to keep it as easy as installing normal ubuntu (witch I think is very easy, and straight forward)

- Fool proof install, meaning, it is a ubuntu server distro (meaning it is like kubuntu to ubuntu) as an install, a GUI will guide you through the install, identically to the ubuntu setup, with a small portion to setup things such as the initial rolls, (media, file share, calender, etc.) witch would be addons once the web ui is loaded at first boot

- The static ip would be generated by getting a dhcp address, then truncating the last 8 bits (the last .x) and replacing it with some large number, then pinging that, to make sure the address is not used.

-no need to even know what an ip is, "just go to [https://192.168.0.222/](https://192.168.0.222/) to access your server" would be proudly proclaimed, if the user selected the express setup (last option)

Features, or more requirements to call it a "Home Server"

-Web interface, sporting the ubuntu theme, as the main page (after successfull login as the instaler user (what was setup at install) the main page would display roles, including:
Home file server
Itunes media server (also compatible with rythmbox [http://www.fireflymediaserver.org/](http://www.fireflymediaserver.org/))
Website server (or web server, lamp)

after these check boxes, would be options, like
Add file shares | domain name | Adjust user access, and privileges (for files)
change media collection location | 
Change website | Choose web types (blog, content management system (like Google sites, concrete5) and standered HTML)

-Also users, configure users and groups, and set them as:
limited account, or child (maby add some sort of web filtering)
system user
administrator

-manage dynamicip services like no-ip.org and make an easy interface.

- No DNS, DHCP, VPN, this is a home server, there are allready plenty of tools to get that stuff up and running.

I like the idea, it is where some are headed, the catagory of 
I can use a computer, and I am smart enough to know that linux is good because it is a humane community developed os, but I don't want to mess with cable, and acronyms (esspecialy recursive ones) And this person also says
I want a central place to store backup and house my media, and save money by doing it open source.

Please know that this platform would not be for everyone, and the votes might be biased, because the forum users are, most likly the ones that are past this catogory, and into getting more out of hardware, and configuring these things themselves. But I know there would be a large catagory o people that would be thrilled to boot up a server, and check a box, and instantly listen to all there music, streaming it around there house at a party, and they would just be baffeled by loging into the same account all over there house, and would see it as perfectly rational to not wast a computer as a server, and use it as a desktop station as well.

### Post by ugm6hr on 2009-04-12
Home Server software options are already available.

Windows Home Server does most of what you suggest, although doesn't have generic desktop capabilities.

I think Amahi does exactly this already - check it out.  They are looking for someone to head development of an Ubuntu version.  It is Fedora-based, but includes file-server, calendar, wiki, media streaming and network backup features.

Unfortunately, the Ubuntu Home Server project appears to have gone dormant.

FreeNAS does most of this (file server, dynamic DNS updates, media streaming, torrents, backups, web server with PHP).  It *could* benefit from auto-acquisition of static IP for those beginner users.  It is easy enough to add a php blog like pivotlog, although I haven't managed to install a wiki yet, but there are some PHP options that do not need a database:
[http://www.9bis.net/pumawiki/](http://www.9bis.net/pumawiki/)
[http://www.dokuwiki.org/dokuwiki](http://www.dokuwiki.org/dokuwiki)
[http://lionwiki.0o.cz/](http://lionwiki.0o.cz/)
Not so sure about calendars on FreeNAS... Not had a proper look yet - maybe something like this: [http://www.kubelabs.com/phpcalendar/](http://www.kubelabs.com/phpcalendar/)

I think the major benefit of Windows Home Server is its availability with custom server hardware, which some companies are replicating with OSS.  Excito do a low-power Debian-based device.  I think there is a definite market for this kind of hardware.

Combining a full desktop with a server is perhaps not the best idea, since:
[LIST]
even if it is on a lot of the time, it is power inefficient unless the desktop is actually being used
a desktop user could potentially crash the server, annoying other users within the home network
if access is open from the web (e.g. with no-ip /dyndns service for dynamic IP), security may be an issue
[/LIST]

Good luck if you want to pursue your idea; I have recently been looking at options for Home Servers, and hence had a few opinions to share.  Hope that's helpful.  I opted for FreeNAS for simplicity.
