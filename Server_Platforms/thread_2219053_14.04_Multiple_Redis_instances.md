### Post by mtrivs on 2014-04-22
Hello there.  I am no expert in the matter, but I know that some packages by default do not include an init script (which is stored in /etc/init.d).  I found that there is a redis-server init script which should work to get the server started.  You must download the script to the /etc/init.d directory and name it redis-server (the wget statement below).  Then to correct the permissions of the file, you enter the chmod statement below.  This will allow your system to execute the script, which will start the server.  Next, you must inform the system of the new service (so to speak) and run the update-rc.d statement.  This will install the redis-server init script as a service and set it to run on boot.

As root [or add sudo to each command]
```

[COLOR=#333333][FONT=Helvetica]wget -O /etc/init.d/redis-server [/FONT][/COLOR][https://raw.github.com/gist/286018/d48b84d89289df39eaddc53f1e9a918f776b3074/redis-server-for-init.d-startup](https://raw.github.com/gist/286018/d48b84d89289df39eaddc53f1e9a918f776b3074/redis-server-for-init.d-startup)
[COLOR=#333333][FONT=Helvetica]chmod 755 /etc/init.d/redis-server[/FONT][/COLOR]
[COLOR=#333333][FONT=Helvetica]update-rc.d redis-server defaults
[/FONT][/COLOR]
```

[http://www.debuntu.org/how-to-managing-services-with-update-rc-d/](http://www.debuntu.org/how-to-managing-services-with-update-rc-d/)   <-- More on managing services 
[http://www.linux.com/learn/tutorials/442412-managing-linux-daemons-with-init-scripts](http://www.linux.com/learn/tutorials/442412-managing-linux-daemons-with-init-scripts)  <-- More on init scripts
