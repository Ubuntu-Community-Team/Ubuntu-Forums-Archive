### Post by MJN on 2007-04-24
Given it's a 'home' server it's unlikely to be under quite the level of stress/load that perhaps a server in the conventional sense would be (e.g. enterprise mail server, busy e-commerce website, DNS root server etc etc).
 
My home server is a PIII 1GHz, 512MB RAM and runs Dapper. It plays host to a few 'personal' websites of family and friends in addition to providing mail hosting for around a dozen users. In addition to this it acts as a nightly over-the-Internet backup server for two others and during the day provides home security of motion-detection/recording of 3 IP cams. It also houses my entire music collection for LAN-wide streaming throughout the house.
 
All in all it does a fair bit and, yes, I use it for my daily PC a lot of the time hence it has X (and KDE) running constantly.
 
This has been up for around 3 years now and has not suffered any ill effects from this 'dual use' - indeed it still has plenty of capacity in reserve.
 
The specs you've quoted clearly knock spots off my machine yet this could be good and bad. Good in that you could get away with doing more than me (not that there's anything I haven't been able to do e.g. watching DVDs etc) but bad in that such a machine on 24/7 will likely be using more power than mine. Not only does this cost to run but the additional heat may be a relevent factor. My machine runs at around 80W when ticking along and as such doesn't require much in the way of cooling hence I've been able to fit sound-deadening linings and some temperature controlled low-noise fans which make it all but completely inaudible (and that's with 3 hard disks inside, but two of those are spun down when not in use for backups/storage etc).
 
Happy to provide further info if required. The bottom line is - how busy will your server likely need to be?
 
Mathew
 
P.S. Something to bare in mind is 'experimenting' - I don't mess around with large new isntalls - e.g. MythTV with a capture card - on this 'live' server as it's all too easy to break something. Hence, I have got an identical 'spare' machine which all my real testing is done on. I'm not suggesting your buy 2 machines (!) but just pointing out that for large-scale experimentation I wouldn't recommend doing it on your live system given you really need 24/7 uptime if others are relying on it.
