### Post by objr.jtree on 2017-03-13
Make sure CUPS is running. I was having trouble using the webinterface and I realized the cups.service wasn't actually running. You can use the following commands to check and restart cups.service:

[FONT=system][FONT=comic sans ms]systemctl status cups.service

systemctl restart cups.service
[/FONT][/FONT]
The cups.service should start on startup, but I have found sometimes it doesn't. I haven't had time to look into that yet though. I don't think you need to run those with sudo, but I could be wrong. Good luck!
