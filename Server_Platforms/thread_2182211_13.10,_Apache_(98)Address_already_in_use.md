### Post by sigdrifa2 on 2013-10-20
Hi all,
I just upgraded my desktop to 13.10, and now apache2 won't start. It returns the following:

```
 * Starting web server apache2                                                                                                        
AH00558: apache2: Could not reliably determine the server's fully qualified domain name, using ::1. Set the 'ServerName' directive globally to suppress this message
(98)Address already in use: AH00072: make_sock: could not bind to address [::]:443
 * 
 * The apache2 instance did not start within 20 seconds. Please read the log files to discover problems
```

The latest entry in /var/log/apache2/error.log is from before I started the upgrade.

First thing I tried was to reboot the system again, no change.

netstat -tnpl does not return anything listening on port 443 (or 80, for that matter.)

I tried changing the SSL port in /etc/apache2/ports.conf, but the only effect that had was that I got the same error message with the new port number (which also wasn't in use otherwise.)

Suggestions, anyone? Help would be very much appreciated.

### Post by sigdrifa2 on 2013-10-20
Yay! Solved! In case anyone else runs into this problem, here's where I found the solution:

[http://www.who.is.free.fr/wiki/doku.php?id=apache#make_sockcould_not_bind_to_address_443](http://www.who.is.free.fr/wiki/doku.php?id=apache#make_sockcould_not_bind_to_address_443)

I changed the ports.conf file according to this (the section labled "success"), and now it works.
