### Post by robert_pectol on 2010-10-06
You flushed the rules but your modified default policies remained in effect.  That's why you got locked out.  To completely flush the ruleset AND restore the default policies back to their defaults, you need to do the following:

/sbin/iptables --policy INPUT ACCEPT
/sbin/iptables --policy OUTPUT ACCEPT
/sbin/iptables --policy FORWARD ACCEPT
/sbin/iptables -F

That **should** bring you back to a default Ubuntu configuration with everything allowed.  Good luck.

### Post by SeijiSensei on 2010-10-06
If you make the default INPUT policy ACCEPT, then you must specifically block unwanted packets in your rules.  Add the following to the end of your ruleset:

```
-A INPUT -j LOG
-A INPUT -j REJECT --reject-with-icmp-host-prohibited
```

This will log all unwanted traffic to syslog and block it.

I'd bet you're running Webmin which listens on port 10000.  I don't know about 8009/8081, though they seem bound only to the machine's IPv6 address.  If you're not intending to run the other services, you need to disable their servers.  Otherwise you're accepting FTP, SSH, SMTP, and HTTP/HTTPS.  Is that what you expected?

```
 tcp        0      0 127.0.0.1:3306          0.0.0.0:*               LISTEN
```
You also have a MySQL server listening on port 3306, but only on the "localhost" interface so it's not open to the outside.

Run "sudo iptables -L -nv" which will show the complete set of rules in operation.  Are they identical with what you think you've created via the script?
