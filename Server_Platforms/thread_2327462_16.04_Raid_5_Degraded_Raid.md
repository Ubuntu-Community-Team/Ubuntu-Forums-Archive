### Post by darkod on 2016-06-11
Do you have separate /boot partition or not? For raid5 I think you need to use separate /boot partition because raid5 splits the files over the disks and boot files do not like being split.

What you can do is create small partitions on each disk (like 2GB or something) and create 4-disk raid1 from them to use as /boot. With the rest of the disks do one big partition if that's what you want and do the raid5 from them. Try it like that...

Also don't forget to install grub on all disks...

### Post by troep on 2016-06-21
I have the same problem but with a simple RAID 1 setup for root (16.04)

I cannot get the system to boot properly in degraded mode. I have tried:

**sudo dpkg-reconfigure mdadm **    

Does not ask to enable degraded boot (I think it used to ask in older versions)

**/etc/initramfs-tools/conf.d/mdadm**

This file no longer exists and does not get created by dpkg-reconfigure mdadm, so I created it manually as follows:

echo 'BOOT_DEGRADED=true' > /etc/initramfs-tools/conf.d/mdadm && update-initramfs -k all -u

After creating the file, the system still drops to initramfs shell on boot.

**bootdegraded=true kernel argument**

Still drops to initramfs shell on boot with this argument active.
