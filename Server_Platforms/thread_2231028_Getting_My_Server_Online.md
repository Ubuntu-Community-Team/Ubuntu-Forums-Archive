I use google's default and then my ISP's auto DNS get added.  So you can run DHCP to find out what your local provider's DNS addresses are then add them, or just use google's they seem to work fine and are fast.

### Post by kosmokramer314 on 2014-06-23
> **nerdtron said:**
> Also you need to have your own domain name like [www.myname.com]("http://www.myname.com") which you'll need to buy from domain registration sites like [www.godaddy.com](www.godaddy.com)

do NOT use godaddy..they are garbage.  Use namecheap.com as they provide free dynamic DNS that you will need to use if you don't have a static PUBLIC IP...which you probably don't.  You can contact your internet service provider to purchase a static public IP, but you'll probably be told no.  SO in short

[LIST=1]
[*]buy a domain name from namecheap.com
[*]setup ddclient on home server
[*]configure namecheap DNS to work with ddclient
[/LIST]


You'll also need to setup port forwarding of port 80 on your router to the static IP of your server.  Don't do this until your site is up and functional because as soon as you open the port on your router, you will be exposing your server to the world.  That was quick and dirty, so if you have additional questions, just post them :)
