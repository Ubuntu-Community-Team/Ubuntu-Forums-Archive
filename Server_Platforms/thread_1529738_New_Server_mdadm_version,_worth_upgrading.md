### Post by arrrghhh on 2010-07-12
For the most part, Ubuntu tries to stay up-to-date with what can be tested.  Usually the repo's lag behind a couple of versions so they can be properly tested and put into the repo's.

Typically I recommend people stick with the repo's - unless you **need** something from a newer version, it's best to run stuff from the repo's.  That way you get updates automatically, if you compile or use a DEB then you have to update manually.
