Samba acts as a domain controller and/or file server, so all of the machines that you are wanting to migrate to a new domain will have to Join the Samba Domain. From what I understand there is no limitation to user count. You could have 300 seperate usernames all authenticating to the one Samba server, If that samba server is also acting as a file server you can limit the number of connections to individual Network Shares.

Windows 7 is fully compatible with Samba, just make sure you use version 3.4 or later (samba). 

The following link will give you some information regarding windows 7 compatibility with samba. There are a few steps you need to do on 7 before they can be added to the Samba domain controller.  BUT, if you already have a windows server running, I would set a logon script on all of the windows 7 machines that will make the required registry change on all the windows 7 machines, that will make the domain migration much easier.

[http://wiki.samba.org/index.php/Windows7](http://wiki.samba.org/index.php/Windows7)

Here is some other information for samba:
[http://wiki.samba.org/index.php/Samba4/HOWTO#Configure_a_Windows_Client_to_join_a_Samba_4_Active_Directory](http://wiki.samba.org/index.php/Samba4/HOWTO#Configure_a_Windows_Client_to_join_a_Samba_4_Active_Directory)
 going on the information in the link above, what would make it easier, instead of manually configuring DNS settings. make a test environment, seperate of your real one (so you dont get multiple DHCP Servers and desktops start dropping off your network lol) with Ubuntu server installed, install Samba 3.4 or later, install DNS and DHCP, create a username and see if the client desktop will join the domain after making the changes. You have many desktops and having a checklist or plan premade will make the migration much much easier. 

Ive done a few domain migrations and having a checklist for yourself in the migration takes a lot of heartache out of the process. ):P


My question is: Are you currently running a domain controller, and if so why are you migrating?

### Post by domgross on 2012-03-23
That amount of users / computers should not be a problem.
If you are concerned about handling of the user database you might want to consider LDAP as backend for Samba.

From my point of view the main issue is the traffic generated by the windows clients. When a user logs on / off the whole profile (which included the users files) are synced with the server. 
So if lots of users log in/off at the same time you will need enough bandwith to handle this.

To avoid syncing everything at log on / off take a look at folder redirection so that most of the data is served directly by the samba server and the profiles remain small.

### Post by collisionystm on 2012-03-26
Regardless, here is the .reg file I use to join a Win7 computer to a samba domain.

> Windows Registry Editor Version 5.00
       
       ; Win7_Samba3DomainMember
       [HKEY_LOCAL_MACHINE\System\CurrentControlSet\Services\LanManWorkstation\Parameters]
       "DNSNameResolutionRequired"=dword:00000000
       "DomainCompatibilityMode"=dword:00000001
       
       ; Speedup settings
       [HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows\System]
       "SlowLinkDetectEnabled"=dword:00000000
       "DeleteRoamingCache"=dword:00000001
       "WaitForNetwork"=dword:00000000
       "CompatibleRUPSecurity"=dword:00000001
       
       ; Can drive you nuts
       [HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System]
       "EnableLUA"=dword:00000000' | tee Win7_Samba3DomainMember_jelledj.reg
       
       unix2dos Win7_Samba3DomainMember_jelledj.reg
