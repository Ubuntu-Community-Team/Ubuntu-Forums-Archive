### Post by volkswagner on 2012-05-26
You can add it to your vhost file.  If you're using only the default vhost file, edit:

```
sudo nano /etc/apache2/sites-available/default
```

Add the following for the directory in question.

```
 
        <Directory /var/www/folder2>
                IndexIgnore *.jpg
        </Directory>


```

Of course this only removes it from the list when navigating to that directory.  If someone enters the full pathname, the file will be displayed.

### Post by Habitual on 2012-05-27
Roasted:

I have always used/utilized an .htaccess file. 
One method is to modify /etc/apache2/sites-available/default.
Hiding them from the Index and Denying Apache to "serve it" are two different mechanisms.

"IndexIgnore *.jpg" in either /etc/apache2/sites-available/default or edit an /var/www/folder2/.htaccess file with that directive in it will HIDE the JPGs but will NOT deny them to any one who knows their location on the site.

Similarly, 
```
<Files ~ "^.*\.([Jj][Pp][Gg])">
 Order allow,deny
 Deny from all
 Satisfy All
</Files>
```in a /var/www/folder2/.htaccess file or  /etc/apache2/sites-available/default should deny access to all JPG/jpg files.

Make copies before edits! ;)

HTH.

### Post by volkswagner on 2012-05-27

[http://httpd.apache.org/docs/2.0/mod/mod_autoindex.html](http://httpd.apache.org/docs/2.0/mod/mod_autoindex.html)

You can simply add switches as part of the url to do sorting. 

like [http://domain.com/dir/?C=S](http://domain.com/dir/?C=S) will sort the items by size then name.

You can also click on the column headings to resort the order by clicking size, name, date, etc.  Take notice how the url is rewritten.
