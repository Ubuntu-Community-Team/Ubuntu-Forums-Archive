Munin has several modules to monitor apache:

> 
apache_accesses
apache_processes
apache_volume


and the following for BIND9:

> 
bind9
bind9_rndc


The apache plugins require that the server-status directive in apache2.conf be enabled.

```

<IfModule mod_status.c>
    #
    # Allow server status reports generated by mod_status,
    # with the URL of http://servername/server-status
    # Change the ".example.com" to match your domain to enable.
    
    <Location /server-status>  
        SetHandler server-status
        Order deny,allow
        Deny from all
        Allow from .example.com
    </Location>
</IfModule>

```

If the munin plugins are not already installed in the plugins directory, you can
symlink them from (I think, please check as I'm not sure about this on ubuntu
just yet) /usr/share/munin/plugins/<plugin name>. 

You can also look inside each of those plugin files, which has additional
commented information at the top for setting them up.

I'm not sure about the bind9 requirements so you should investigate that
yourself (i.e. like above, look inside the plugin file for additional information).
