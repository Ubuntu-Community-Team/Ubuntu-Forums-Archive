### Post by arrrghhh on 2010-08-13
It's only really good if you want to crunch a TON of numbers.  For example, seti@home.

It doesn't sound like it would do you any good.  A cluster is good for redundancy, so if one box fails the entire system doesn't.  Mostly enterprises, large organizations, governments, universities... these are the kinds of places that run clusters.

Google uses a cluster so they can easily add hardware and add capacity easily.  Just another use, again it sounds like something you would not need.
