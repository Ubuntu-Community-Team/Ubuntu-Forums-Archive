Setup:

Windows server share listed in /etc/fstab mounted via cifs on 7.04 server.
Regular backup to disk listed in cron:

```
1 19 * * * /usr/bin/rdiff-backup /home/user/backup/dir /backup/dir
```

This works once and then fails when run from cron - contents of rdiff-backup-data:

```
backup.log

mirror_metadata.2007-10-25T19:01:02+01:00.snapshot.gz
session_statistics.2007-10-25T17:01:01+01:00.data
```

So it looks like the backup has worked once (at 2007-10-25T17:01:01) when run from an earlier cron but the next has failed (2007-10-25T19:01:02).

rdiff-backup works fine when run manually but shows ```

``` after the cron job has run and failed.

This is using rdiff-backup 1.1.5.

Any ideas?
