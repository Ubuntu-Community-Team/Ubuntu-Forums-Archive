### Post by nedwed on 2010-02-18
Hi,
I've decided to upgrade my simple SMB File Server to a more advanced level and installed Ubuntu Server 9.04 64-bit.

The Samba file server works perfect...as always:]
But I have some problems with dns/iptables.
My goal is to chache/accelerate http through squid.
Currently squid runs with this options:
http_port my_IP:3128 transparent
http_port my_IP:80 accel

Now I want to forward DNS through my new server.
I installed BIND9 and made the following changes in the named.conf.options file:
        
        forward only;
        forwarders {
                192.168.1.99; //ISP DNS
        };

        auth-nxdomain yes;    # conform to RFC1035
        listen-on-v6 { none; };
        listen-on { 192.168.0.110; };
        query-source address * port 53;
        transfer-source * port 53;
        notify-source * port 53;
        version "Microsoft DNS Server v5.1";

My iptables config script contains the following lines:
 
        echo 1 > /proc/sys/net/ipv4/ip_forward
       
        echo 0 > /proc/sys/net/ipv4/conf/all/accept_redirects
        echo 0 > /proc/sys/net/ipv4/conf/default/accept_redirects
        echo 0 > /proc/sys/net/ipv4/conf/all/send_redirects
        echo 0 > /proc/sys/net/ipv4/conf/default/send_redirects
        echo 0 > /proc/sys/net/ipv6/conf/all/accept_redirects
        echo 0 > /proc/sys/net/ipv6/conf/default/accept_redirects
       
        echo 1 > /proc/sys/net/ipv4/icmp_echo_ignore_broadcasts
        echo 1 > /proc/sys/net/ipv4/icmp_ignore_bogus_error_responses
        echo 0 > /proc/sys/net/ipv4/icmp_echo_ignore_all
       
        echo 0 > /proc/sys/net/ipv4/conf/all/log_martians
        echo 0 > /proc/sys/net/ipv4/conf/default/log_martians

        echo 0 > /proc/sys/net/ipv4/conf/all/accept_source_route

        echo 1 > /proc/sys/net/ipv4/conf/all/rp_filter

        echo 1 > /proc/sys/net/ipv4/tcp_syncookies

$IPTABLES -F
$IPTABLES -t nat -F

$IPTABLES -P INPUT DROP
$IPTABLES -P OUTPUT ACCEPT
$IPTABLES -P FORWARD DROP

$IPTABLES -t nat -A PREROUTING -i $LAN -s $LAN_IP -p tcp --dport 80 -j DNAT --to-destination $PROXY_IP:$PROXY_PORT
$IPTABLES -t nat -A PREROUTING -i $LAN -s $LAN_IP -p udp --dport 80 -j DNAT --to-destination $PROXY_IP:$PROXY_PORT
$IPTABLES -t nat -A POSTROUTING -o $WAN -j MASQUERADE

$IPTABLES -A INPUT -i lo -j ACCEPT
$IPTABLES -A OUTPUT -o lo -j ACCEPT
$IPTABLES -A FORWARD -o lo -j ACCEPT

$IPTABLES -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
$IPTABLES -A OUTPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
$IPTABLES -A FORWARD -m state --state ESTABLISHED,RELATED -j ACCEPT

$IPTABLES -A INPUT -s $LAN_IP -j ACCEPT
$IPTABLES -A FORWARD -s $LAN_IP -j ACCEPT

These are my basic rules, that should work ok for now. The main problems I have now are:
1. Slow dns response.
2. Slow/none Internet traffic.
I've experimented with various rule settings but with no luck.
Maybe someone has an idea where I made mistakes or had similar problems?

### Post by nedwed on 2010-02-19
I deleted bind and modified my iptables.

$IPTABLES -F
$IPTABLES -t nat -F

$IPTABLES -P INPUT DROP
$IPTABLES -P OUTPUT ACCEPT
$IPTABLES -P FORWARD DROP

#forward DNS requests to ISP DNS server
$IPTABLES -t nat -A PREROUTING -i $LAN -s $LAN_IP -p tcp --dport 53 -j DNAT --to-destination 192.168.1.99:53
$IPTABLES -t nat -A PREROUTING -i $LAN -s $LAN_IP -p udp --dport 53 -j DNAT --to-destination 192.168.1.99:53

$IPTABLES -t nat -A PREROUTING -i $LAN -s $LAN_IP -p tcp --dport 80 -j DNAT --to-destination $PROXY_IP
$IPTABLES -t nat -A PREROUTING -i $LAN -s $LAN_IP -p udp --dport 80 -j DNAT --to-destination $PROXY_IP

$IPTABLES -t nat -A POSTROUTING -o $WAN -j MASQUERADE

$IPTABLES -A INPUT -i lo -j ACCEPT
$IPTABLES -A OUTPUT -o lo -j ACCEPT
$IPTABLES -A FORWARD -o lo -j ACCEPT

$IPTABLES -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
$IPTABLES -A OUTPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
$IPTABLES -A FORWARD -m state --state ESTABLISHED,RELATED -j ACCEPT

$IPTABLES -A INPUT -s $LAN_IP -j ACCEPT
$IPTABLES -A FORWARD -s $LAN_IP -j ACCEPT

This configuration works very nice.
