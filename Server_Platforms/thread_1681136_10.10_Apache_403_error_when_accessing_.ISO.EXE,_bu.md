### Post by BilboBaggins11 on 2011-02-03
Ubuntu 10.10, freshly installed. Got my own PHP script written and everything. Im going to be using this laptop as a simple file server for a LAN party. It simply lists files, and links to a download for them. 

Installers for software, ISOs and stuff are being provided to people. The only problem is:
When i try to access certain files, i get
**Forbidden**

 You don't have permission to access /files/game.iso on this server.
  Apache/2.2.16 (Ubuntu) Server at localhost Port 80

Yet, when i try to access my ZIP files, they immediately start to download.

What permissions do i have to change to make it so i can download all file types?

---

### Post by BilboBaggins11 on 2011-02-03
SOLVED


I tinkered arount a bit and found the cause.

It was caused by file permissions. By default, zips and stuff had full permissions and others had none.

Right-click on the file (in your folder) that you want to be downloaded and change it from "-" to "Read" or "Read and Write"
