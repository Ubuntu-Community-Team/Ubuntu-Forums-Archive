### Post by lampyridae on 2014-06-05
I'm using Ubuntu Server 12.04. To be more precise, I'm running a [Bitnami LAMP Stack  5.4.29-0  (64-bit)]("https://bitnami.com/stack/lamp/virtual-machine") on VirtualBox 4.3.12 on OSX.

I've edited /etc/network/interfaces to use a static IP:

```

# This file describes the network interfaces available on your system
# and how to activate them. For more information, see interfaces(5).

# The loopback network interface
auto lo
iface lo inet loopback

# The primary network interface
auto eth0
iface eth0 inet static
   address 192.168.3.46
   network 192.168.3.0
   netmask 255.255.255.0
   broadcast 192.168.3.255
   gateway 192.163.3.1
   dns-search XXX.YYY
   dns-nameservers AAA.BBB.CCC.DDD AAA.BBB.CCC.DDD

```

After a reboot, Apache services are indeed reachable through 192.168.3.46 inside the local network, but the VM itself is unable to resolve domain names ("apt-get update", "ping google.com" fails).

"cat /etc/resolv.conf" reveals the following:
```

# Dynamic resolv.conf(5) file for glibc resolver(3) generated by resolvconf(8)
#     DO NOT EDIT THIS FILE BY HAND -- YOUR CHANGES WILL BE OVERWRITTEN

```

From what I understand, it should contain the two nameserver and the one search entries I've put in "/etc/network/interfaces" but it doesn't.

If I revert to DHCP and reboot, "/etc/resolv.conf" is automatically filled correctly and the VM is able to resolve domain names.

What am I doing wrong? What else could I do to track down the issue?

Thanks!
