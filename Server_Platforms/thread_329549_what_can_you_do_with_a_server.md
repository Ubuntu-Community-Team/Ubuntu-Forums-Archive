### Post by tturrisi on 2007-01-01
www sites
testing code (php, html, css, perl, cgi-bin apps)
databases
web applications
email
serve files
serve media (audio, video)
network shares
groupware
networked office applications

Just turn that old system into a server and start doing your web development, then add some php scripting, then do php-mysql-html forms, etc.  The rest of what can be done w/ a server will come naturally as you progress.

### Post by az on 2007-01-02
> **zedd2006 said:**
> would anyone know how much it would cost to set up my own server and list it with dns so it could be a www? or how does this work?

Some ISPs do not allow you to run a webserver from your home - see your contract or ask the sales reporesentative.  Sometimes, they just block port 80.

In Canada, it costs about 15 dollars to register a DNS and another 15 for an ISP to host your DNS for you.  If you are not running a mission-critical site, just use the free dynamicDNS services offered - they are usually 99 per cent reliable.

[https://help.ubuntu.com/community/DynamicDNS](https://help.ubuntu.com/community/DynamicDNS)
