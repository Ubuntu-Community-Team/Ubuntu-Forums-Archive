### Post by doogy2004 on 2009-07-15
> **melgish said:**
> Well, I had no luck finding out how to add by-id to the fstab.

There should be an entry in /dev/disk/by-id that identifies your usb drive, and another entry typically with a "part1" at the end.  The "part1" or "part2" and so on are partitions on the usb disk.  Simply replace the UUID with the path e.g. /dev/disk/by-id/usb-somedisk-part1 such as:

> **melgish said:**
> ```
UUID=db7f927d-cf0b-4a1e-affe-6737e0396fdf /media/usb1 ext3 rw,defaults 0 1
```
```
/dev/disk/by-id/usb-Maxtor_OneTouch_II_B622182H-0\:0-part1  /media/usb1 ext3 rw,defaults 0 1
```

You could also try adding a errors=remount to the fstab line, such as:
```
UUID=db7f927d-cf0b-4a1e-affe-6737e0396fdf /media/usb1 ext3 rw,defaults,errors=remount-rw 0 1
```

Adding errors=remount-rw should make the system attempt to remount the disk.  During the time between then the first mount fails and the system attempts to mount the disk again the usb drive MAY have time to get recognized by the system.

Try the remount option first, then try the by-id method, if those don't work try a combination of both.  Let us know if that works.
