### Post by jchappy65 on 2013-04-25
OS: Ubuntu 12.04 64-bit Server

I setup and installed this new server and everything worked 100%. but after I setup a static IP on the server, I cannot "apt-get" any file at all, yet I can still access and sftp into the server remotely.

I am somewhat familiar with ubuntu, but not servers. I followed a guide on setting up the Static IP offsite and it told me to remove DHCP from my server so it could only see the static IP.

How can I fix this so I can retrieve files from apt again and maintain my static IP?

---

### Post by darkod on 2013-04-25
After changing to static IP you have to set the dns nameservers manually. Look for your ISP/provider nameservers and you can use them. Or simply use the google nameservers which are 8.8.8.8 and 8.8.4.4.

To your /etc/network/interfaces file, in the interface section add:
dns-nameservers x.x.x.x y.y.y.y

After that restart the networking or the whole server. It sounds like a dns issue and that should sort it out.
