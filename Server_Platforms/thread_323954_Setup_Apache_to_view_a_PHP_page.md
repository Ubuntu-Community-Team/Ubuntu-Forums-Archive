### Post by kidders on 2006-12-23
Hey again,

That all seems pretty good. PHP 4 & 5 aren't necessarily intercompatible, so if you've decided that 4 is the one to go for, stay with it :-) There are two things to check:

[LIST]
[*]That you've installed the PHP module for the right version of Apache (ie libapache-mod-php4 vs libapache2-mod-php4)
[*]That the PHP module is referenced in /etc/apache2/mods-enabled (or /etc/apache/mods-enabled, if you've opted for an older version of Apache, as well as PHP). For instance, here's mine:
[/LIST]
```
$ ls -l /etc/apache2/mods-enabled/
lrwxrwxrwx 1 root root 36 2006-12-19 03:05 cgi.load -> /etc/apache2/mods-available/cgi.load
lrwxrwxrwx 1 root root 37 2006-12-19 03:05 php5.conf -> /etc/apache2/mods-available/php5.conf
lrwxrwxrwx 1 root root 37 2006-12-19 03:05 php5.load -> /etc/apache2/mods-available/php5.load
lrwxrwxrwx 1 root root 40 2006-12-19 03:05 rewrite.load -> /etc/apache2/mods-available/rewrite.load
lrwxrwxrwx 1 root root 40 2006-12-19 03:05 userdir.conf -> /etc/apache2/mods-available/userdir.conf
lrwxrwxrwx 1 root root 40 2006-12-19 03:05 userdir.load -> /etc/apache2/mods-available/userdir.load
```

### Post by kidders on 2006-12-23
Apache2 is SSL capable out of the box ... imo you should use that, really.

As far as I can make out, the problem is that your PHP module isn't loading. An additional LoadModule or AddModule directive should fix it, but I'm not completely certain what it should look like. If you have a /usr/lib/apache/1.3/mod_php4.so, you could try **LoadModule php4_module /usr/lib/apache/1.3/mod_php4.so** ... or perhaps it should be **AddModule mod_php4.c**. The other thing you should check for is some reference to php4.conf in apache's main config file. It might look something like **include /etc/apache/*.conf**.

Sorry I can't be more specific :-(
