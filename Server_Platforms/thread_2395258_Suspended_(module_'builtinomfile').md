Did you check the URL in the message?

> 
                     **What does it mean?**

 This is a generic error message that unfortunately can happen in a number of cases.
 In practice, it is often associated with suspension of actions. Then it comes with a text like[INDENT]action "action 17&#8243; suspended
[/INDENT]
The number behind action changes.
 **How to solve it?**

 A frequent case for this error message on Debian-based distributions  (like raspbian) is that rsyslog.conf contains the instruction to write  to the xconsole pipe, but this pipe is never read. If so, you can simply  delete these lines to remove the error message. These lines are usually  found at the end of rsyslog.conf.
 For other error message, it probably is a good idea to check [rsyslog&#8217;s issue tracker at github]("https://github.com/rsyslog/rsyslog/issues") and file a new issue if you can&#8217;t find a related case.
 Note: we try to keep this page update if we see other frequent causes of this error.
                     


There's also some discussion on U&L: [Repeated &#8220;rsyslogd-2007: action 'action 18' suspended&#8221;.]("https://unix.stackexchange.com/questions/404070/repeated-rsyslogd-2007-action-action-18-suspended")

Note that (at least in the case of 16.04), the relevant conf file appears to be /etc/rsyslog.d/50-default.conf rather than /etc/rsyslog.conf itself
