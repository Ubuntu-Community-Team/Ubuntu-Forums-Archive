### Post by sdpagent on 2013-07-18
It would be interesting to know if you have the same issue on a real physical/dedicated machine at home with the same or less memory. With OpenVZ vps especially, there are a zillion 'beancounter' limits that the provider can set, so two providers can be miles apart even if they both say '1 cpu, 512 ram, 10gig storage' which is so vague. With openvz your provider can 'oversell'  your vps, so he may say you have 1.5 gigs of ram, but during heavy usage (other people on the same node', you actually have less.) This helps make OpenVZ vms so much cheaper than Xen. Personally, I'm an OpenVZ fanboy, but only for my private VPSs that I have full control over and know exactly what I'm getting.
Feel free to [refer to this]("http://openvz.org/Man/vzctl.8") for loads of config options (look under User Beancounter limits heading).
