Maybe deleting /boot/grub/grub.cfg, then running the update command again. It's obviously detecting it like normal, but somehow it's not getting to your configuration. Also, you could try using grub-emu. It's a grub emulator. So, you update the configuration, run grub-emu, and it shows you the boot screen without needing to restart. Great for quick debugging without restarting 10zillion times.

Edit: Grub-emu works on my desktop, it shows in a window just like my other apps, but I don't know how it'll fair on a server. It may need/install a graphical (Gnome/KDE) system. If so, it's probably best to not bother installing it. But, you could still run update-grub, then cat /boot/grub/grub.cfg and see if what you need is in there.

### Post by drs305 on 2010-10-21
This is certainly a head scratcher.

When update-grub searches for other OS's using os-prober, it will look for other grub.cfg and menu.lst files, as well as kernels.

Does Centos use either a grub.cfg or menu.lst file? If so, and if Centos uses a different way of writing the file, perhaps G2 can't sort between them and just omits making an entry. This is pretty far-fetched, but you could test it by renaming any grub.cfg or menu.lst file within the Centos boot folder and see what happens.

### Post by oldfred on 2010-10-22
This is my best guess on your grub2 version of a boot stanza to add to 40_custom. When you have separate /boot partitions the set root is to the /boot but the kernal is to the / (root) partition. In your kernel line you are using the label which is /. Grub2 changes partition numbers so hd0,0 becomes hd0,1
I think the search line is optional. It is to fixup systems where the set root line may not always be correct, like connecting drive to different systems and then using UUID to override the set root line.

menuentry "CentOS release 5.5 (Final) (on /dev/sdb3)" {
insmod ext2
set root='(hd0,1)'
search --no-floppy --fs-uuid --set 31e0a955-45eb-4710-835c-2256a53281e5
linux /vmlinuz-2.6.18-128.1.10.el5 ro root=LABEL=/ noapic
initrd initrd /initrd-2.6.18-128.1.10.el5.img
}

Copy stanza to 40_custom:
gksudo gedit /etc/grub.d/40_custom
sudo update-grub

For more info on customizing:
includes line to limit display to two
Grub 2 Title Tweaks Thread -drs305
[http://ubuntuforums.org/showpost.php?p=8082954&postcount=1](http://ubuntuforums.org/showpost.php?p=8082954&postcount=1)

If you put your menu entry in /etc/grub.d/40_custom it will not be over written and will be at the end of your menu. Or you can copy it to 09_custom and make it executable and it will be at the top of your menu.
