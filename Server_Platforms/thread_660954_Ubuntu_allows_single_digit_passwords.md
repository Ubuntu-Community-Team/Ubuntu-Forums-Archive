### Post by rickyjones on 2008-01-07

> Its against the security features of linux .

Allowing the use of a null or single digit password is, in my opinion, not against the security features of Linux. The fact that you are able to customize this scrutiny is what makes Linux what it is.

Assigning null passwords is usually a large security no no. However, what about average joe that has a single PC in his house that is not connected to the internet? Does he really need to remember a password if it is just him?

Final thought: One thing that I like about Windows XP personally is that you can have user accounts without passwords. But, in order to connect to the computer remotely the user account must be assigned a password for security purposes. Good idea I think.

Maybe Ubuntu could incorporate something similar. Passwords are not required unless remote access is in the equation.

Just my two cents.

I am not sure why this should be. Do you want to enforce a certain password policy on your users at home? Or what is the purpose here for such a requirement? Actually XP does not enforce this either for individual installations.
