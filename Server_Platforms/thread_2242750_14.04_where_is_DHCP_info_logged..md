### Post by volkswagner on 2014-09-05
Sorry I don't know specifically, but I'd first guess /var/log/messages.

You can search the entire /var/log directory and subdir's using the following command.
```
grep -nr "dhcp" /var/log > results.txt
```

The above will write results to results.txt file in your current directory for easy searching.
You can drop "> results.txt" portion if you just want it to output to the console.

If you get too many results, you can change search string to something more specific, like "DHCPREQUEST".

### Post by SeijiSensei on 2014-09-05
> **PeterK2003 said:**
> actually nothing is being logged to syslog...

Are you sure the rsyslog service is running?

```
sudo service rsyslog restart
```

/var/log/messages is the RedHat equivalent to /var/log/syslog on Debian flavors like Ubuntu.  My CentOS systems log to /var/log/messages; my Ubuntu ones use /var/log/syslog.
