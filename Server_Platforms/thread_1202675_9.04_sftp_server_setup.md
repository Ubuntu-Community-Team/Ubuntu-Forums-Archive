### Post by ainsworth_t on 2009-07-02
> How do I make sure everything's up to date. I'm use to MS and I know from a clean install you have to download and apply all the service packs and updates, is it the same for Ubuntu?

There are no Service Packs for Ubuntu because the distro releases are on a 6 months release schedule, meaning that major updates are already included in the base distro. However, periodic security and application updates are released. To make sure that your system is up to date, just run the following command:

```
sudo apt-get update && sudo apt-get upgrade
```

This command will first update the stored package list available from the repositories and then will perform upgrades to whatever packages, if any exist.

There are many differences between KDE and Gnome, too many to list here. KDE has a more Windows-like feel and would be good for a first time Linux user. Gnome takes a more simplified and clean approach to it's desktop.

> How do I make the folders and how do I know where their being made at?

You were right about the *mkdir* command being the command to make folders. Wherever you run the *mkdir* command is where those folders will be created, unless you specify another location. To see your current location from the command line, use the command *pwd*. If you want to make your folders in a different location, you can either a) navigate to the location where you want to make them with the *cd* command (like in Windows) or b) you can specify the full path in the *mkdir* command, for example:

```
sudo mkdir /home/newfolders/1
```

### Post by ainsworth_t on 2009-07-08
VI has a pretty steep learning curve and isn't as straight forward as some other editors. Personally, and especially for a beginner, I like nano. So to edit the sources list just run the command

```
sudo nano /etc/apt/sources.lst
```

By locked down, I assume you mean so that the user who is connected via SFTP cannot change configuration files and things of that nature? If that's the case, Linux is already locked down as long as you're not connected as the root user. The root user is the only user that can change system and application configuration files (or any user in the sudoers file with appropriate privileges). 

Also by default, SSH on Ubuntu is configured to separate privileges, meaning that the SSH process will spawn another process with the same privileges of the authenticated user. Also, Ubuntu Server installs AppArmor by default which restricts capabilities of programs. So as long as you're not logging into the SFTP server via the root user, you should be fine.

However, if that is not enough to satisfy your locked down requirements, you can try creating a new, unprivileged user (within Gnome System >> Administration >> Users and Groups), set their home directory to the location of the folders you created, and use that account to test the server.
