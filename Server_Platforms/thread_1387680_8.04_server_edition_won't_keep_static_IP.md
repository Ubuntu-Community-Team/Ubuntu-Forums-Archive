### Post by markseddon on 2010-01-22

/etc/network/interfaces file below and ran

sudo /etc/init.d/networking restart

and all seemed well. I just got in this morning and the IP address had changed
from 192.168.5.27 to 192.168.5.19 which meant the hostname didn't resolve
correctly. Have I missed something?


less of /etc/network/interfaces

# This file describes the network interfaces available on your system
# and how to activate them. For more information, see interfaces(5).

# The loopback network interface
auto lo
iface lo inet loopback

# The primary network interface
auto eth0
iface eth0 inet static
address 192.168.5.27
netmask 255.255.255.0
network 192.168.5.0
broadcast 192.168.5.255
gateway 192.168.5.254

### Post by kamaji792 on 2010-01-22
Hi markseddon,

Just for the record my interfaces file is:

```

# The loopback network interface
auto lo
iface lo inet loopback

# The primary network interface
auto eth0
iface eth0 inet static
address 192.168.0.127
netmask 255.255.255.0
gateway 192.168.0.1

```

so I can only suggest you remove the **network** and **broadcast** addresses.

FYI the first part of the output from the **ifconfig** command is:
```

eth0      Link encap:Ethernet  HWaddr 00:05:5d:01:99:2a  
          inet addr:192.168.0.127  Bcast:192.168.0.255  Mask:255.255.255.0

```
