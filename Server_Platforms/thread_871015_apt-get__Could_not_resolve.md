### Post by nochids on 2009-08-30
I know this is an old post, but I had the same problem with Ubuntu 8.04 Server and wanted to share what fixed it for me.  I could not ping or update, and the ssh to the server seemed to take a LONG time before I would be prompted for my password.  I had recently switched ISPs from comcast to qwest and as it turns out, I had comcast in the /etc/resolv.conf file.  I deleted all lines and save the file.  I could immediately ping, and I was able to then sudo apt-get update without errors.  I checked the ssh and got the password prompt immediately.  I am SOOOO happy!!  I dont know if having an empty resolv.conf file is a bad thing, but it did fix the problems I was having. 

Cheers!

### Post by svsig on 2010-07-28
Thanks a lot! I also had this problem, and thanks to this post I managed to solve it, but I did it in a slightly different way. I have just got a new ADSL modem from my ISP, and it has a different IP address than my old modem. I had only one line in my resolv.conf file:
```
nameserver 10.0.0.254
```
This was the internal IP address for my old modem. I changed it to the IP address for my new modem and it worked again. I could have changed the IP address in the new modem to match the resolv.conf file instead, but I chose to change the contents of the file.

The message about not being able to determine the server's fully qualified domain name is normal.  Don't worry about this for now.  As forthe network configuration saying [OK], that's also good, but it doesn't necessarily mean you have connectivity; it just means your network hardware was detected properly.

In order to get an actual IP address so that you can connect to the Internet, try typing:
```

sudo dhclient eth0
```

This will request a dynamic IP address via dhcp.  For this to work, your VMware networking (and possibly your physical network) need to be configured properly to allow dhcp.

If the dhclient command times out without giving you an IP address, you may need to use a static address instead, which you can set by editing the file /etc/network/interfaces (the command "man interfaces" provides information about how that file works).

If you can't figure out how to get an address, please let us know how your network settings for VMware are configured.  You should be able to find this information from the VMware graphical interface.
