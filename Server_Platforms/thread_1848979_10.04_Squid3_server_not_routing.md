### Post by SeijiSensei on 2011-09-26
> ```
-A PREROUTING -i eth1 -p tcp -m tcp --dport 80 -j DNAT -to-destination 192.168.0.1:3128
-A PREROUTING -i eth1 -p tcp -m tcp --dport 80 -j REDIRECT -to-ports 3128
```

You need two hyphens before "to-destination" and "to-ports", not one, e.g., "--to-ports 3128".  Many of the options to iptables use this syntax, like --dport which you entered correctly.

### Post by SeijiSensei on 2011-10-12
After looking at this again, the problem may be that both NICs are in the same address space.  That causes all sorts of routing problems.  Put the externally-pointing NIC and the router on a different subnet and see if that helps.
