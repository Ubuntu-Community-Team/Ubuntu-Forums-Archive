### Post by elvar on 2010-05-19
Hello, I upgraded one of my ubuntu servers last night from 8.04 to 10.04 and found that I could not use the 'pg_upgradecluster' utility to upgrade postgresql 8.3 to 8.4 because 8.3 is replaced by 8.4. This seems like a huge problem that surely is affecting many other users. I realize I could manually dump and reimport all my databases but that is a huge pain. Can anyone else offer any better solutions?


Regards,

### Post by meldon on 2010-05-20
Hi elvar,

I'm in a similar situation. I just found this post:
[http://ubuntuforums.org/showthread.php?t=1473404#3](http://ubuntuforums.org/showthread.php?t=1473404#3)

That will get 8.3 installed. Then I ran this command:
```

pg_dropcluster 8.4 main  #this will wipe out the new databases from the new install
pg_upgradecluster 8.3 main  #this to upgrade the 8.3 data to 8.4

```
