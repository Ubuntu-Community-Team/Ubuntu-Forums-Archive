### Post by TheFu on 2012-01-23
I run lots and lots of different OSes with Win7 x64 as the hostOS and VirtualBox as the hypervisor. I suppose you can use VMware Player if you prefer proprietary software.

Visit the Ubuntu Server Download page [http://www.ubuntu.com/download/server/download](http://www.ubuntu.com/download/server/download) . LTS server releases are on even years, in April - 8.04, 10.04, and in a few months 12.04.  Download the 32-bit ISO to your PC.  At this point, 10.04 is the default LTS and should probably be used until about June 2012 as the next LTS version gains stability. Just my opinion. I wait a month after an LTS release before I grab it for testing.

Here's a link [http://blog.jdpfu.com/2010/06/22/virtualbox-performance-improved](http://blog.jdpfu.com/2010/06/22/virtualbox-performance-improved) to some easy tips to make the VM as fast as possible.  
* Use virtio drivers whenever possible. I don't recall whether Windows supports this. KVM does automatically.  
* Preallocate all the virtual HDD storage - DO NOT LET IT GROW as needed. I've been running my main desktop with 10GB of virtual HDD for over 3 yrs. For a specialty install, 2-4GB would be plenty.
* If you can't use virtio for HDD and network virtual devices, use
** SATA HDD controler
** Intel PRO/1000 network adapter
It doesn't matter what your real hardware is, this is the virtual HW that the client VM will see.

Setup a VM with the RAM, Networking, HDD setting that is recommended for your particular application. Point the CD or DVD device towards the ISO you downloaded above and boot.

I'm not a fan of downloading pre-built VMDKs. I like to know what's been installed in VMs running on my network. Basically, you could end up with anything inside those prebuilt images - rootkits, back doors, anything - even a perfectly configured solution.

If you are new to Linux, you may want to start off with a desktop install. Linux/UNIX servers don't have a GUI at all. It can be overwhelming for someone new to a *nix environment to not have a GUI to help them figure things out.

Regardless, good luck.

### Post by pctx on 2012-01-23
I'm running my 10.04.3 LTS server on Workstation 7.1 and I use it on my main Windows 7 machine w/ 12GB RAM and my 3TB+ of storage. :)

Honestly it's easy and I've found that VM (function and stability wise) is a lot more stable than Virtualbox.  

If you're running Workstation or even Player, you can setup your headless server and get access it just like you would any other system on your network using "Bridged" mode.

I use mine for "real world" SSL over SSH for website testing from work and connectivity issues for web browsing and ping tests.  It is very doable and very easy to setup.

Best of luck.
