### Post by Yoooder on 2007-06-26
"Server" is kind of a fuzzy term.  In general it is a machine intended to serve functionality to other machines, and not to be used directly by the user as a desktop/workstation.

Now, a desktop can run individual servers (like ftp, http, sql, and game servers) and still be a desktop, a server can also have a desktop (a gui) installed to it, and be used like a desktop while still acting primarily as a server.

Hope that helps confuse you :-P

### Post by Alterax on 2007-06-27
Server can mean either a machine dedicated to giving services, or the program itself that provides those services (which is why I refer to a server program as a "service"--it may not be correct per se but you usually know what I mean).

A server is different from a workstation or a standard computer because of its function.  Workstations, PCs, and laptops are there for the user.  Servers on the other hand are there to provide services FOR the workstations.  A file server is there for mass storage of files, a web server is there to make webpages, and so on and so forth.  Say you have two users that have a lot of files to store.  They can each get massive hard drives, or they can get one and put it in a server, and then they don't have to risk losing everything if their PC crashes.

Servers (especially fileservers) also make backups easy.  Anything important goes there, and then you back up the server regularly.  Your home directory can be set to automatically back up from your PC to your server, so then if you have to reinstall, you won't lose as much.

Servers can also do other things.  databases can run on those and put less overhead on the user's computer.  You can even run applications on the server, but actually deal with the program on your workstation, PC, or laptop.  So you can either get a mediocre server and 20 really good PCs, or just ONE really good server and get better performance out of 20 mediocre PCs.  Major cost savings.

--Alterax
