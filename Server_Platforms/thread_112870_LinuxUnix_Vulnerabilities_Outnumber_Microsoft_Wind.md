### Post by LordHunter317 on 2006-01-07
[QUOTE=azz]From a security standpoint, which one is easier to manage overall?  Which one is cheaper to manage?[/QUOTE]That depends on your situation, application, existing infrastructure, staff experience, etc.

It's really dependent on the situation: from a security POV, both are equal for just about any deployment I can think of.  They both can be made to be equally as secure for all situations you would deploy them.  In some situations, Windows will be easier to deploy securely; in some others, Linux.  But the differences aren't major enough for me to often make a decision based on security and security alone.

Typically, I decide which one is best based on the problem I'm solving, then come up with a security solution.  Other factors are almost always far more important both in terms of time spent and total cost over security considerations.

For example, if someone wants to deploy an ASP.NET-based web solution, I'll likely suggest Windows Server 2003.  

If the wanted a Java-based solution, then I'd be likely to recomment Apache and Tomcat on Linux, unless they were an entirely Windows shop.  Then I might recommend Windows simply to keep the platform heterogenous.

You'll note in my reasoning, security never entered the picture: I know I can secure both just as well when it comes time to do so.
