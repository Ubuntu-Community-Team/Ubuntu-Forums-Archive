### Post by nerdtron on 2014-10-23
I am puzzled by your network interfaces file. Can you explain it and what are you trying to achieve?

What is the output of
```

route -n
ifconfig
cat /etc/resolv.conf

```

If you want your server to NAT the inside network, say all of the eth0, and then send out to eth1, then you should have at least 2 different IP address block on both interfaces.

Something like how you configure a router, 1 port as WAN and 1 port as the LAN DHCP server.

### Post by Jason_Bean on 2014-10-23
Alright, I fixed it. The reason my interfaces file is that way is because I was not able to resolve hostnames on my own network from the gateway. I went through a bunch of stuff to try and figure it out. Your last suggestion, looking at the resolve.conf file, made me notice that the old IP address for my primary DNS server was in there. I had originally modified the dhclient.conf to prepend that address. I guess it hadn't been a problem until I had to modify my subnet so that it would not clash with the networks I was attaching from through OpenVPN. I'd forgotten I had made the change. Now everything appears to be good. Thank you.
