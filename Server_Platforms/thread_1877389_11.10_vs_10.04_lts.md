### Post by matt_symes on 2011-11-08
Hi

I would also suggest 10.04, but on a production server i would highly recommend not using a GUI.

There are far less things to go wrong without a GUI and you wanted stability. It will also free up more resources for your server.

The server edition, that does no have a GUI, is optimised for server usage in the schedulers it uses. Don't install the desktop version.

Just enable security updates.

So to administer your server without a GUI and remotely, i would suggest webmin - if you feel the need for some kind of user interface - or ssh using keys.

That is what i would do if i was you. Learn the command line. You will not regret it.

If not having a GUI is a worry to you then remember this is a great place to ask questions and get advice on all things related to servers headless or otherwise and with or without a GUI.

Kind regards

### Post by Linux&amp;Gsus on 2011-11-08
I also would say that 10.04 would be the better choice, you can get the next LTS going after it has been out for a while and is tried and tested and stable.
Ubuntu is based on Debian. Unlike chrisjsmith I never really had trouble with either of them. However, I have only used Debian as server I depend on. Also, I use the servers mainly (only 1 exception) at home to mess around, share a handful of files between computers, temp storage, and I had 1 machine running for my emails (which happened to be Debian, as well, simply because back in the day when I got that going Ubuntu wasn't really there for servers yet, and I simply never changed it).

So, in other words, if you can manage a Ubuntu server, you can manage a Debian server, and vice verca. I personally (for non professional tinkering at home), I can't even figure out the differences between the 2. Others might be able to pick the details.

Also, if you want to run the machine as server, I'd say you are better of running the server version. Meaning not with a desktop environment. It might be helpful to learn a few useful and/or often used commands.
For graphical administration [webmin]("http://www.webmin.com/") ([webmin @ wikipedia]("https://secure.wikimedia.org/wikipedia/en/wiki/Webmin")) is often recommended, I use it as well. It's browser based and works well.
Not being an expert in that area at all, but I actually don't know how a desktop environment, be it LXDE, GNOME, KDE, or what ever else, will be helpful e.g. for administering a database. Most help for servers you will get on the forums would be either via command line or guided through a tool like webmin.

Webmin works for both, Debian and Ubuntu. Which one you prefer at the end is kinda your personal choice, I'd say.

### Post by matt_symes on 2011-11-08
Hi
> **greatkalu said:**
> One more question regarding Ubuntu Server. I'm going to run it on a 64-bit machine. I've heard that there may be some issues installing a 64-bit OS ... is this true or should I just avoid them and install the 64-bit 10.04 LTS Server?

There used to be issues along time ago with the 64-bit libraries but not any more.

If you want to run 32-bit applications as well then you will need the 32-bit libraries. Not a problem.

If you have more than 4GB of memory in this server then you will need to run 64-bit Ubuntu to access all the memory unless you use a PAE kernel.

64-bit Ubuntu will also take advantage of the 64-bit registers natively.

Me ? I would run 64-bit Ubuntu on a 64-bit machine.
