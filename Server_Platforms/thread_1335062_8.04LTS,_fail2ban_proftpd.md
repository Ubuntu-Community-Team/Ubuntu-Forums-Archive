### Post by Gorlist on 2009-11-24
Appears though its correct, the fail2ban-regex isn't always reliable for testing. Working on its own accord.

However im having to manually include my server name before <HOST>, before it matches any entries:

```
myserver.com \(\S+\[<HOST>\]\) - USER \S+ \(Login failed\): Incorrect password.
```

Or even better:

```
\S*\s*\(\S+\[<HOST>\]\) - USER \S+ \(Login failed\): Incorrect password.
```

You may need to include **\S*\s*** before all of the regex lines.

So much rubbish or dated information around :(
