### Post by skymt on 2006-09-14
Do you have a lot of users? Are there services accessible to the Internet? Is there much chance of a malicious insider? If the answer to any of these questions is no, don't bother. Password aging is useless for homes or small organizations.

### Post by daxelrod on 2006-09-17
If you do decide that password aging is worth it, give some thought to your aging policies.

When users are forced to change their passwords too often, they start writing them down, putting a number at the end of their password, or other things to not have to remember a new password all the time.

Also make sure that **every** account's password is aged (or at least, everyone over a certain level of access). I've seen systems where root was the only user that didn't have an aged password. That completely defeated the point.
