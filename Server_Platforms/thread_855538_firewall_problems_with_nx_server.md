I am running a proxy server using squid and dansguardian, there is an iptables firewall that drops all non-proxied connections.
I am able to ssh into the box, but whenever i try to connect using nx client i get a message saying: the remote proxy closed the connection while negotiating the session. if i clear all the iptables settings using putty i am able to connect fine, but as soon as i rerun the iptables script i am unable to connect, either from the internal network or the external network.
Here is the firewall script:
#!/bin/sh
# Firewall for Mexico network.
# By: James Ranson <XXXXXX>

# External interface (connected to internet).
EXTIF=eth0
# Internal interface (connected to the LAN).
LANIF=eth2

# Set to REJECT for debugging and DROP when stable.
BLOCK_ACTION=REJECT

# Erase all existing rules.
iptables -t filter -F
iptables -t nat -F
# Zero statistics
iptables -t filter -Z
iptables -t nat -Z

iptables -t filter -A FORWARD -p icmp -j ACCEPT
iptables -t filter -A FORWARD -p tcp --destination-port ssh -j ACCEPT
iptables -t filter -A FORWARD -p udp --destination-port ssh -j ACCEPT

iptables -t filter -A FORWARD -i $LANIF -o $EXTIF -j $BLOCK_ACTION

# Accept incoming packets by default.
iptables -t filter -P INPUT ACCEPT

# Block all incoming connections from the internet except ssh.
iptables -t filter -A INPUT -p tcp --destination-port ssh -j ACCEPT
iptables -t filter -A INPUT -p tcp --destination-port udp -j ACCEPT
iptables -t filter -A INPUT -p tcp --syn -i $EXTIF -j $BLOCK_ACTION

# All traffic generated by the router itself is allowed.
iptables -t filter -P OUTPUT ACCEPT
iptables -t nat -P OUTPUT ACCEPT

#
# Configure NAT.
#
iptables -t nat -P PREROUTING ACCEPT
iptables -t nat -P POSTROUTING ACCEPT
# Redirect http requests to local proxy.
iptables -t nat -A PREROUTING -i $LANIF -p tcp --destination-port http -j REDIRECT --to-ports 8080
iptables -t nat -A PREROUTING -i $LANIF -p tcp --destination-port https -j REDIRECT --to-ports 8080
iptables -t nat -A PREROUTING -i $LANIF -p tcp --destination-port 3128 -j REDIRECT --to-ports 3128
# Allow intentional access of the local proxy.
iptables -t nat -A PREROUTING -p tcp --destination-port 8080 -j RETURN
iptables -t nat -A PREROUTING -p tcp --destination-port 3128 -j RETURN
iptables -t nat -A PREROUTING -p tcp --destination-port 8090 -j RETURN
# Allow LAN to access DNS on the internet.
iptables -t nat -A PREROUTING -i $LANIF -p udp --destination-port domain -j RETURN
iptables -t nat -A POSTROUTING -p udp --destination-port domain -j MASQUERADE
# Allow LAN to send/receive e-mail from the internet
# (21 April 2006)
iptables -t nat -A PREROUTING -i $LANIF -p tcp --destination-port 465 -j RETURN
iptables -t nat -A POSTROUTING -p tcp --destination-port 465 -j MASQUERADE
iptables -t nat -A PREROUTING -i $LANIF -p tcp --destination-port 995 -j RETURN
iptables -t nat -A POSTROUTING -p tcp --destination-port 995 -j MASQUERADE
iptables -t nat -A PREROUTING -i $LANIF -p tcp --destination-port smtp -j RETURN
iptables -t nat -A POSTROUTING -p tcp --destination-port smtp -j MASQUERADE
iptables -t nat -A PREROUTING -i $LANIF -p tcp --destination-port pop3 -j RETURN
iptables -t nat -A POSTROUTING -p tcp --destination-port pop3 -j MASQUERADE

iptables -t nat -A PREROUTING -i $LANIF -p udp --destination-port 465 -j RETURN

iptables -t nat -A PREROUTING -i $EXTIF -p tcp --destination-port ssh -j RETURN

### Post by k_grdn on 2008-07-10
Hi,

Run tcpdump on the incoming interface for the connecting host, find out which port nx connects to, also check if there's a daemon listening on that socket.

Then open the required ports in your firefall.

iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT 

iptables -A INPUT -i eth$ -p tcp -s $source_ip -d $dest_ip --dport $dest_port -m state --state NEW -j ACCEPT 


Regards,

k_grdn
