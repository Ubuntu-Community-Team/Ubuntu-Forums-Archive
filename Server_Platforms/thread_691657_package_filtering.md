### Post by gaten on 2008-02-08
While its quite hard to figure this out without the code, here are some common pitfalls:

1. Using ports below 1025. Binding anything to ports 1 - 1024 is restricted to root users, so if your listen  port is in that range, you'll have to run the program as root or change the port.

2. Are you trying to listen and talk on the same port? For instance, if your server listens on port 2000, and you client tries to send data out on port 2000 as well, you will get a 'connection refused' error.

And after re-reading your post, it could be possible that there is no server listening at all on the remote computer to respond to your client (most likely). Try scanning the machine with nmap ala
```
nmap -vv -sT -p **port** -P0 **server**
```
and see if it's open. You could also try connecting to it with telnet or netcat.

If you post at lest some of the code, you have a much better chance at getting this solved. Also, you should probably move this to the "Programming" forum, as it's setup for this purpose.

### Post by The Cog on 2008-02-09
The easiest way to see if your program is even listening is to use the command
**netstat -ltn**

And to see if any firewall rules are active:
**sudo iptables -L -v -n**
For an inactive firewall, you should get three heading lines with no entries under any of them.

### Post by HermanAB on 2008-02-09
Hmm, hosts.allow/deny is a feature of tcpwrappers.  It only affects programs that has libwrap explicitly compiled in.

Check to see whether you have an iptables firewall running with:
$ sudo iptables -L

If so, and you are on a LAN, behind a real firewall, disable yours while you are playing.

### Post by The Cog on 2008-02-09
Additionally, I believe that hosts.deny causes connections to be accepted and then immediately dropped, rather that refused immediately.
