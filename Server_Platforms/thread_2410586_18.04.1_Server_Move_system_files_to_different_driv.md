### Post by darkod on 2019-01-17
Don't do anything!!! :)

According to what you said, it is set up the way you want it. The "Found Ubuntu..." message is for secondary OS (OS detected on any of the disks in the system). The main OS never says for itself Found Ubuntu...

Look closely at the lsblk output. It does show you root partition on /dev/torus-vg/root on the 2TB disk. And it shows as encrypted I believe:
```
sdb                         8:16   0   1.8T  0 disk  
&#9500;&#9472;sdb1                      8:17   0   731M  0 part  /boot
&#9500;&#9472;sdb2                      8:18   0     1K  0 part  
&#9492;&#9472;sdb5                      8:21   0   1.8T  0 part  
  &#9492;&#9472;sdb5_crypt            253:0    0   1.8T  0 crypt 
    &#9500;&#9472;torus--vg-root      253:3    0   1.8T  0 lvm   /
    &#9492;&#9472;torus--vg-swap_1    253:4    0   976M  0 lvm   [SWAP]
```

Look closely at the Found Ubuntu message, it says that seconday OS is on /dev/torus-old-vg/root and mounted at /old. Obviously you still keep it around.

### Post by darkod on 2019-01-17
You don't have to bother with renaming files. The lsblk output is quite clear and it shows you where your root partition is. That is definitive proof. You can also complement the test with output from:
```
df -h
```

If that shows /dev/torus-vg/root is mounted as / then that is all you need. Because /dev/torus-vg/root is on the 2TB disk.
