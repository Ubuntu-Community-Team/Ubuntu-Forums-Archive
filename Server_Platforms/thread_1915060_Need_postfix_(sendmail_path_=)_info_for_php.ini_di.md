### Post by SeijiSensei on 2012-01-25
Postfix is designed to be a drop-in replacement for sendmail.  It accepts sendmail's command-line parameters so the default entry in php.ini, "/usr/sbin/sendmail -t -i", should work for Postfix as well.

More pertinently, is Postfix running on the machine in daemon mode and listening on port 25?  If not, and you send a message via PHP mail(), what SMTP server is supposed to handle it?

[COLOR=#444444][FONT=Arial][FONT=Times New Roman]Connection refused[/FONT][/FONT][/COLOR]
[COLOR=#222222][FONT=Arial][FONT=Times New Roman]Assuming your postfix is running and listening on localhost, another possible problem is that postfix is not[/FONT][/FONT][/COLOR]
[FONT=Times New Roman][COLOR=#222222][FONT=Arial]configured to run in IPv6 mode, but your [/FONT][/COLOR][COLOR=black][FONT=Arial]/etc/hosts [/FONT][/COLOR][COLOR=#222222][FONT=Arial]file specifies ::1 as localhost. In that case mailman tries[/FONT][/COLOR][/FONT]
[COLOR=#222222][FONT=Arial][FONT=Times New Roman]to send mails to ::1 which has no postfix listening, thus resulting in a (111, 'connection refused') error.[/FONT][/FONT][/COLOR]

[COLOR=black][FONT=Arial][FONT=Times New Roman]SPF and multiple external ip addresses[/FONT][/FONT][/COLOR]
[COLOR=black][FONT=Arial][FONT=Times New Roman]Explanation[/FONT][/FONT][/COLOR]
[FONT=Times New Roman][COLOR=#222222][FONT=Arial]I have some systems that are networked on an internal private ip address subnet ([/FONT][/COLOR][COLOR=black][FONT=Arial]192.168.0.0/16[/FONT][/COLOR][COLOR=#222222][FONT=Arial]). For a[/FONT][/COLOR][/FONT]
[FONT=Times New Roman][COLOR=#222222][FONT=Arial]few reasons I email reports and such to [/FONT][/COLOR][COLOR=black][FONT=Arial]<user>@mail.internal [/FONT][/COLOR][COLOR=#222222][FONT=Arial]where [/FONT][/COLOR][COLOR=black][FONT=Arial]user [/FONT][/COLOR][COLOR=#222222][FONT=Arial]is an address that is not valid[/FONT][/COLOR][/FONT]
[COLOR=#222222][FONT=Arial][FONT=Times New Roman]for receiving mail via the external interfaces. These systems also share a public ip address subnet so they[/FONT][/FONT][/COLOR]
[COLOR=#222222][FONT=Arial][FONT=Times New Roman]could email each other that way, but I'd prefer they didn't for local addresses. I have published SPF records for[/FONT][/FONT][/COLOR]
[COLOR=#222222][FONT=Arial][FONT=Times New Roman]the public mail servers because all of our mail routes through those servers so if others care to check they can[/FONT][/FONT][/COLOR]
[COLOR=#222222][FONT=Arial][FONT=Times New Roman]ignore email claiming to be from us but being delivered from other servers as per our SPF record.[/FONT][/FONT][/COLOR]
[COLOR=#222222][FONT=Arial][FONT=Times New Roman]Recently I have expanded the ip addresses these systems are using externally to support multiple instances of[/FONT][/FONT][/COLOR]
[COLOR=#222222][FONT=Arial][FONT=Times New Roman]port-based services like https (adding :oddport doesn't impress the customers.) I could have expanded or[/FONT][/FONT][/COLOR]
[COLOR=#222222][FONT=Arial][FONT=Times New Roman]added more liberal SPF record values, or added more forward and reverse DNS records but I wanted to stick[/FONT][/FONT][/COLOR]
[COLOR=#222222][FONT=Arial][FONT=Times New Roman]with less ip addresses.[/FONT][/FONT][/COLOR]
[COLOR=#222222][FONT=Arial][FONT=Times New Roman]So to recap my system has:[/FONT][/FONT][/COLOR]
[FONT=Times New Roman][COLOR=#222222][FONT=Arial]_ [/FONT][/COLOR][COLOR=#222222][FONT=Arial]eth1 <public ip with spf published>[/FONT][/COLOR][/FONT]
[FONT=Times New Roman][COLOR=#222222][FONT=Arial]_ [/FONT][/COLOR][COLOR=#222222][FONT=Arial]eth1:1 <public ip for extra port-based services>[/FONT][/COLOR][/FONT]
[FONT=Times New Roman][COLOR=#222222][FONT=Arial]_ [/FONT][/COLOR][COLOR=#222222][FONT=Arial]eth0 <private ip on>[/FONT][/COLOR][/FONT]
[FONT=Times New Roman][COLOR=#222222][FONT=Arial]By using the settings in [/FONT][/COLOR][COLOR=black][FONT=Arial]/etc/postfix/master.cf[/FONT][/COLOR][COLOR=#222222][FONT=Arial], [/FONT][/COLOR][COLOR=black][FONT=Arial]/etc/postfix/main.cf[/FONT][/COLOR][/FONT]
[FONT=Times New Roman][COLOR=#222222][FONT=Arial]and [/FONT][/COLOR][COLOR=black][FONT=Arial]/etc/postfix/transport [/FONT][/COLOR][COLOR=#222222][FONT=Arial]as outlined above I was able to get my outgoing smtp traffic to use my SPF[/FONT][/COLOR][/FONT]
