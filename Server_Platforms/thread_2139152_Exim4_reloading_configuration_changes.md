### Post by al_dente on 2013-04-26
Hello folks

I'm administering a server setup by someone else, so learning how Exim works as I go along :)

We're running Exim 4.71-3ubuntu1.4 on our 10.04 LTS server and have noticed some issues with SPF rejecting for IPv6 hosts when it shouldn't. There's a known bug I found, which I think is our problem.
[http://bugs.exim.org/show_bug.cgi?id=860](http://bugs.exim.org/show_bug.cgi?id=860)

Anyway, it's not practical for us to upgrade to a newer version, so I thought I'd disable SPF. Looking at the logs, we don't block many emails via SPF checking anyway. 

So, I have edited /etc/exim4/conf.d/main/01_spamblock

and changed 
CHECK_RCPT_SPF = true 
to 
CHECK_RCPT_SPF = false

Then ran 
update-exim4.conf
service exim4 reload

Checking the mainlog file this morning, I can still see a few 'SPF check failed' entries, so this hasn't disabled SPF checking. Have I missed something? Is there something else I need to run to reload the config?

I've checked in /var/lib/exim4/config.autogenerated and my change shows in there.

---

### Post by al_dente on 2013-04-30
This is fixed.

Needed to remove the 'CHECK_RCPT_SPF =' line completely. 

The SPF checking script looks to see if that variable is defined, so it doesn't care whether it's true or false - just that it's something or nothing.
