### Post by Ryan Dwyer on 2010-07-12
You probably don't have mod_rewrite enabled.

sudo a2enmod rewrite
sudo /etc/init.d/apache2 restart

### Post by jhollinger on 2010-07-12
Although these are essentially the same command, try these:

```
sudo aptitude purge squid
```

```
sudo dpkg purge squid
```

(FYI, apt-get and aptitude and just fancy front-ends for the dpkg command.)

If they don't work, you've probably run into the extremely rare occasion where a package is broken beyond repair. Something similar happened to me once when my system froze while installing apache. Eventually I had to remove all the package files manually. Afterwords I was able to reinstall it.

This will list all the files and directories in your squid package:
```
dpkg -L squid
```

I don't have squid installed, but as an example, here's what I saw for apache:

```

/usr
/usr/share
/usr/share/bug
/usr/share/bug/apache2
/usr/share/bug/apache2/control
/usr/share/doc
/usr/share/bug/apache2/script
/usr/share/doc/apache2

```

It's showing us the entire directory tree, but we're just concerned with the directories that **only** contain apache2 (in your case squid) files. If you remove say, /usr, you'd completely hose your system. So in this case I  ran:

```
sudo rm -r /usr/share/bug/apache2 /usr/share/doc/apache2
```

Then I was able to reinstall apache with apt-get. Hopefully this will work for you, too.
