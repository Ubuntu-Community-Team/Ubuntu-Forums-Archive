### Post by hawkmage on 2011-05-11
What do you have in your /etc/nsswitch.conf file?  There should be a line that starts with "hosts:" and has something at least "files dns".  Mine has "files mdns4_minimal [NOTFOUND=return] dns mdns4"

### Post by hawkmage on 2011-05-12
If you don't already have one create an ssh id key.

Create a text file with all of the host names or IP addresses in your home directory.  Lets assume that the file is called client_names

Run the command:
```
for S in `cat ~/client_names`;do ssh-keyscan ${S} >> ~/.ssh/known_hosts;done
```
This will get the ssh host keys into the known_hosts file so you are not prompted to add each one.

Next you will need to get your ssh id pub key into the authorized_keys file on the clients.  This is the tedious part.  Run the command:
```
for S in `cat ~/client_names`;do cat ~/.ssh/id_rsa.pub | ssh ${S} "cat >> ~/.ssh/authorized_keys";done
```
You will have to provide your password for each client.  But once you do this you will be able to ssh to them with out a password.

Next to update the nsswitch.conf file:
```
for S in `cat ~/client_names`;do ssh ${S} "sudo sh -c \"cp /etc/nsswitch.conf /etc/nsswitch.conf.save;cat /etc/nsswitch.conf.save | sed 's/^hosts:.*/hosts:\t files dns/'>/etc/nsswitch.conf\"";done
```
I know this command is a bit complex.  It will make a copy of the /etc/nsswitch.conf to /etc/nsswitch.conf.save then uses sed to update the line starting with "hosts:".  If sudo on these machines requires a password you will have to supply it for each system.
