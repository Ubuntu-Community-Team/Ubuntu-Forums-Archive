I installed 10.04 LTS using RAID1 with two arrays: /dev/md0 as swap and /dev/md1 the rest. Initial bootup was fine, but after a shutdown, it fails to boot and ends up in grub rescue instead with the message "error: no such disk". Doing ls shows:
(hd0) (hd0,2) (hd0,1) (hd1) (hd1,2) hd(1,1) (fd0) (md0)

Notice that (md1) is not listed, even though /dev/md1 is where grub2 loader is installed.

I can get back in by using the server install CD to reinstall grub on /dev/sda and /dev/sdb. But once I am back in, if I do a reboot, the system boots fine. But if I do a shutdown, then do a manual boot, I end up in grub rescue again.

How can I resolve this? Why does the system act differently when rebooting versus just a shutdown and booting up again? Any insights would be greatly appreciated....
