### Post by tinny on 2008-07-27
Sounds like your firewall is blocking the DNS and DHCP protocols.

Have you tried unblocking UDP/TCP Port 53? (port used for DNS requests), I think...

The configuration for DHCP is a little more complicated.
[http://en.wikipedia.org/wiki/Dhcp#DHCP_and_firewalls](http://en.wikipedia.org/wiki/Dhcp#DHCP_and_firewalls)


BTW: UFW is just a tool for configuring IPTables.
[https://help.ubuntu.com/8.04/serverguide/C/firewall.html](https://help.ubuntu.com/8.04/serverguide/C/firewall.html)

This will list your current rules. (can you post the output)

> **crashdogy said:**
> Well just setup DHCP server ubuntu 11.10 and cant get the ufw setup to let my clintes surf please help .

It was better to open a new thread than to post in a thread 4yrs old. :)

Are you trying to use the ubuntu server for routing too, or only dhcp?

If you want it to do the routing too and you have ufw installed, you need to:

1. open /etc/default/ufw and change DEFAULT_FORWARD_POLICY to "ACCEPT"
2. open /etc/ufw/sysctl.conf and uncomment (activate) the /net/ipv4/ip_forward=1 line
