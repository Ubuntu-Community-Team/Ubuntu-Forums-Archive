```it goes back
### Post by steeldriver on 2013-01-14
If your system is running resolvconf as a service (which it sounds like it is - that's the default for 12.04 Server) then the right way to do it afaik is to specify the nameserver in your interfaces file and let resolvconf handle it e.g.

```
# The primary network interface
auto eth0
iface eth0 inet dhcp
dns-nameservers 192.168.1.1

```then restart resolvconf and check that /etc/resolv.conf has picked up the change

```
$ sudo service resolvconf restart
resolvconf stop/waiting
resolvconf start/running
$
$ cat /etc/resolv.conf 
# Dynamic resolv.conf(5) file for glibc resolver(3) generated by resolvconf(8)
#     DO NOT EDIT THIS FILE BY HAND -- YOUR CHANGES WILL BE OVERWRITTEN
nameserver 192.168.1.1

```

### Post by jdthood on 2013-01-14
> **steeldriver said:**
> If your system is running resolvconf as a service (which it sounds like it is - that's the default for 12.04 Server) then the right way to do it afaik is to specify the nameserver in your interfaces file and let resolvconf handle it e.g.

```
# The primary network interface
auto eth0
iface eth0 inet dhcp
dns-nameservers 192.168.1.1

```

then restart resolvconf and check that /etc/resolv.conf has picked up the change


Yes, on a resolvconf system you specify the nameserver address on a dns-nameservers line in /etc/network/interfaces if you are using ifup to configure interfaces. If you are using NetworkManager then you specify the nameserver address in the IPv4 Settings | "Additional DNS servers" field in the Connection Editor.

After editing /etc/network/interfaces it does *not* suffice to restart resolvconf to activate any changes you have made. You have to ifdown and ifup the interface in question; or reboot.
```

### Post by SeijiSensei on 2013-01-25
You really don't want to use DHCP to assign server addresses unless you use a MAC-based reservation on the DHCP server.  Servers need static addresses so you can reliably connect to them by domain name.  So arrange with whomever runs your network to agree on the server's IP, then use static addressing in /etc/network/interfaces to assign the address and gateway and assign the DNS resolvers with the "dns-nameservers" directive as shown. Read the section called "Static IP Address Assignment" in [this document](https://help.ubuntu.com/12.04/serverguide/network-configuration.html) for details.

### Post by jdthood on 2013-01-26
> **cariboo907 said:**
> You can't, if you are using NetworkManager. As the others have told you, /etc/resolv.conf is controlled by NetworkManager,and any changes you make to /etc/resolv.conf will be overwritten when you reboot.

If resolvconf is installed and /etc/resolv.conf is a symbolic link to ../run/resolvconf/resolv.conf then strictly speaking it's resolvconf that updates resolv.conf dynamically. But if you are using NetworkManager then normally it is the only source of nameserver information for resolvconf and thus NetworkManager effectively controls the file.

When resolvconf isn't installed, various other utilities (including some versions of NetworkManager) overwrite /etc/resolv.conf directly in various conflicting and broken ways. Resolvconf was introduced in order to put an end to this chaos.

> **CharlesA said:**
> This is how you would set the correct dns servers in /etc/resolv.conf: [...]

Note that if you edit dns-* options in /etc/network/interfaces it does *not* suffice to restart resolvconf to activate the changes. You have to ifdown the interface and then ifup it again. (If you don't want to ifdown the interface, e.g., because you are administering the machine over ssh via that interface, then there is another way, but it's less straightforward.) Or reboot.
