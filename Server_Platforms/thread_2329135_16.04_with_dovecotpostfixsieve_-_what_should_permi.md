### Post by theodore2 on 2016-07-16
The failure might be caused by [COLOR=#008000]dovecot's lda[/COLOR] agent to which [COLOR=#008000]postfix[/COLOR] passes messages to deliver applying sieve filter to them. I also ran in such problem. I did not studied this problem thoroughly but did the following:
[COLOR=#0000ff][FONT=lucida console]
chgrp mail '/usr/lib/dovecot/dovecot-lda'
chmod s+g  '/usr/lib/dovecot/dovecot-lda'

[/FONT][/COLOR]Now messages got delivered except the [COLOR=#ff0000]root[/COLOR] user. Maybe I'll need to change [COLOR=#0000ff][FONT=lucida console]dovecot-lda[/FONT][/COLOR] group to [COLOR=#ff0000]root[/COLOR]? So far I'm not ready to do this morally... You may try to dare to do that.
