### Post by volkswagner on 2012-12-14
Did you add users to system and to samba?

Try using smbclient at the server to test usernames an password:

```
smbclient //servernameOrIPaddress/tes5 -U sambausername
```

You should be prompted for the password, then you will have a command prompt like:

```
smb: \> 

```

Where you can run file commands like ls, cd

Then quit to exit.

Also check the logs for any info.

Yes when using SAMBA3, SAMBA users also need to have a system account.  These users can have a disabled shell if you like, but the user must exist.  I have never used Webmin for SAMBA, but the limited time I have tried Webmin, I was frustrated with the way it handled config files.  Webmin is not supported by Ubuntu for some time now.

One other thing you may add to the smb.conf is the Netbios name, for easy browsing of the server on the LAN.

```
netbios name = fileserver
```
