### Post by terazen on 2010-06-17
If your server works locally then all you need is dns (maybe dynamic dns?) and port forwarding more than likely.  I'm assuming your server is at home behind a router like a linksys or something.

I'd definately read up on security as well if I were you.  Specifically protecting against brute force ssh attacks and ufw basics to start.  There is more like securing vsftp too, but securing ssh is the first thing to always do.
