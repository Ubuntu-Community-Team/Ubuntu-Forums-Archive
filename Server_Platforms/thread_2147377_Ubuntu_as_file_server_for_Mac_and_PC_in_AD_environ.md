### Post by Sphyrna on 2013-05-21
[FONT=Helvetica]I wanted to create a storage set up for user data for a environment where Macs and PCs are mixed in an Active Directory structure. OS X doesn't yet have a very good implementation of SMB (samba) and performance is down quite a bit there. So I wanted a solution that would provide both APF and SMB for the same data (users might want to access their data from a Mac one day and a PC the next). There are solutions that do this, for example FreeNAS, but I wanted to give a "real" server a shot. [/FONT]
[FONT=Helvetica]I have found several HowTo-s on how to set up either SMB/Samba, connect to Active Directory or setting up Netatalk/AFP (most of the latter aims at setting up a Time Machine server). But none that combines them, at least not for recent versions.[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]Along the way I took notes on how to do it (so I could replicate things) - and why not publish that as a how-to. (And perhaps get some feedback as well).[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]My set up was done in a test bed on a Windows Server 2012 Hyper-V VM with dynamic memory allocation starting at 512Mb and modest CPU resources. [/FONT]
[FONT=Helvetica]When done - the windows clients get about the same file transfer performance as to a Windows can, but the Mac's went from a throughput of about 120 mbps to Windows (SMB) shares to about 800 mbps to the Netatalk (AFP) shares (on a Gbit LAN). A pretty good performance increase. And everything is accessed with Active Directory SSO.[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]Now, I don't have any long time data on stability, no in-depth security evaluation or any experience on how the solution scales. That's where feedback would be interesting…[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]Enough said. Here is the how-to on how to set up an Ubuntu 13.04 Server as a file server for SMB and AFP using AD SSO. It is more or less cut-n-paste and once the server is installed the procedure should not take more than 30 mins. [/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]---[/FONT]
[FONT=Helvetica]**Active directory integration**[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]1. Install Ubuntu 13.04. Standard installation, EXT4 file system (not LVM) and SAMBA file server as the only package when given choices for LAMP etc.[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]2. Set up networking. I prefer to use a static IP (it might work with DHCP as well, but that is not best practice for a server).[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Courier New]$> nano /etc/network/interfaces[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Courier New]#Loopback[/FONT]
[FONT=Courier New]/etc/network/interfaces like this:[/FONT]
[FONT=Courier New]auto lo[/FONT]
[FONT=Courier New]iface lo inet loopback[/FONT]
[FONT=Courier New]
[/FONT]
[FONT=Courier New]#Primary NIC[/FONT]
[FONT=Courier New]auto eth0[/FONT]
[FONT=Courier New]iface eth0 inet static[/FONT]
[FONT=Courier New]address 10.0.1.55[/FONT]
[FONT=Courier New]netmask 255.255.255.0[/FONT]
[FONT=Courier New]gateway 10.0.1.1[/FONT]
[FONT=Courier New]network 10.0.1.0[/FONT]
[FONT=Courier New]broadcast 10.0.1.255[/FONT]
[FONT=Courier New]dns-nameservers 10.0.1.2 10.0.1.3[/FONT]
[FONT=Courier New]
[/FONT]
[FONT=Courier New]#Search domain equals AD domain long name[/FONT]
[FONT=Courier New]dns-search demo.org[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]Restart the network service.[/FONT]
[FONT=Courier New]$> service networking restart[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]And as a precaution, for some reason Windows does not always handle a linux server going from DHCP to static well - so I prefer to manually remove the DHCP lease from the DHCP server in order not to confuse the DNS. (Supposing windows handles DHCP and since this is for AD integration you will have a windows DNS)[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]3. To make administration easier - install SSH-server[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Courier New]$> sudo apt-get install openssh-server[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]4. To make life easier still - since you will be installing and configuring and need root access, make yourself root for the session:[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Courier New]$> sudo su[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]5. Now it is time to join the domain. Use Beyond trust's  PowerBroker Identity services (*pbis*, a newer version of likewise open) this can be found and downloaded as open source at [www.beyondtrust.com]("http://www.beyondtrust.com"). (You will have to provide your email, but it is free except for that.)[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]I find it easiest to do this via a GUI-browser and save the file to a network share that is accessible from the Ubuntu server. Since I only have windows shares, I use the following to transfer the file to Ubuntu:[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]Create directory to mount in:[/FONT]
[FONT=Courier New]$> mkdir /mnt/win_share[/FONT]
[FONT=Courier New]
[/FONT]
[FONT=Helvetica]Mount the windows share[/FONT]
[FONT=Courier New]$> mount.cifs //[windows server]/[share name] /mnt/win_share -o user=[AD user with permissions to the share][/FONT]
[FONT=Courier New]
[/FONT]
[FONT=Helvetica]Create directory for the pbis files[/FONT]
[FONT=Courier New]$> mkdir /usr/pbis[/FONT]
[FONT=Courier New]
[/FONT]
[FONT=Helvetica]And copy the installation file:[/FONT]
[FONT=Courier New]$> cp /mnt/win_share/pbis-open-… /usr/pbis[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]6. Install phis according to the documentation:[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]pbis conflicts with *winbind*, that is installed by default - so we'll have to un-install that:[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Courier New]$> apt-get remove winbind[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]Make the install script executable[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Courier New]$> chmod a+x pbis-open-…[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]and run the script[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Courier New]$> ./pbis-open-…[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]7. Now you are ready to join the domain:[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Courier New]$> /opt/pbis/bin/domainjoin-cli join [domainName] [account with permission to join][/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]And reboot the Ubuntu server[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]Then validate by logging in with [domain]\\[user][/FONT]
[FONT=Helvetica](You will have to escape the backslash separator with another backslash, so for the domain DEMO with user john.doe on the server Skywalker, an ssh login will be [FONT=Courier New]ssh DEMO\\john.doe@skywalker[/FONT])[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]8. You are now connected, but a few things make life much easier.[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]Start by loging out and back in with your "native" linux user to get sudo rights.[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]By default obis does not use *bash* shell. In order to configure that:[/FONT]
[FONT=Helvetica]
[/FONT]
[COLOR=#323333][FONT=Courier New][COLOR=#000000]$> [/COLOR]/opt/pbis/bin/config LoginShellTemplate /bin/bash[/FONT][/COLOR]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]And if you are in a single domain env. set default domain to not have to provide the domain name all the time[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Courier New]$> /opt/pbis/bin/config AssumeDefaultDomain 1[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]Then to get sudo access for your domain users:[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]Edit the sudoers with [/FONT]
[FONT=Courier New]$> sudo visudo[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]To give all domain users sudo rights enter these line:[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Courier New]#For all domain users with DOMAIN-name provided[/FONT]
[FONT=Courier New]%[Your domain]\\domain^users ALL=(ALL) ALL[/FONT]
[FONT=Courier New]
[/FONT]
[FONT=Courier New]#For all domain users with default domain login[/FONT]
[FONT=Courier New]%domain^users ALL=(ALL) ALL[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica](For a single use, omit the % and change domain^users to your user name.[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica][FONT=Courier New]domain^users[/FONT] is the AD group and can be any group. You can view available groups by[/FONT]
[FONT=Courier New]$> /opt/pbis/bin/enum-groups[FONT=Helvetica])[/FONT][/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]Save and restart sudo[/FONT]
[FONT=Courier New]$> /etc/init.d/sudo restart[/FONT]

[FONT=Helvetica]Now you can log in as domain user without providing [FONT=Courier New][DOMAIN]\\[/FONT], will be granted sudo rights and will have *bash* as shell.[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]Time to get the file servers running:[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]**SAMBA for starters**[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica](Be sure to make yourself root or use sudo for each step)[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]For an AD-domain named DEMO.ORG and the ubuntu server Skywalker with a directory [FONT=Courier New]./share[/FONT] that you want to share with all domain users, do this:[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]Create the dir:[/FONT]
[FONT=Courier New]$> mkdir /share[/FONT]
[FONT=Courier New]$> chmod a+rx /share[/FONT]
[FONT=Courier New]$> chown [For example a domain administrator] /share (i.e. chown administrator /share)[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]1. Delete the default configuration file[/FONT]
[FONT=Courier New]$> rm /etc/samba/smb.conf[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]And make a new one:[/FONT]
[FONT=Courier New]$> nano /etc/samba/smb.conf[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]That looks like this [/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Courier New][global][/FONT]
[FONT=Courier New]security = ADS[/FONT]
[FONT=Courier New]workgroup = DEMO[/FONT]
[FONT=Courier New]realm = SKYWALKER.DEMO.ORG[/FONT]
[FONT=Courier New]
[/FONT]
[FONT=Courier New]#Required for pbis:[/FONT]
[FONT=Courier New]machine password timeout = 0[/FONT]
[FONT=Courier New]
[/FONT]
[FONT=Courier New][ubuntu_share][/FONT]
[FONT=Courier New]comment = My share[/FONT]
[FONT=Courier New]path = /share[/FONT]
[FONT=Courier New]read only = no[/FONT]
[FONT=Courier New]valid users = @DEMO\domain^users[/FONT]
[FONT=Courier New]create mote = 0777[/FONT]
[FONT=Courier New]directory mode = 0777[/FONT]
[FONT=Courier New]writeable = yes[/FONT]
[FONT=Courier New]guest ok = no[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]Don't forget the *machine password setting* that is required by pbis-samba integration!!![/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]2. Then you will have to run the obis interoperability installer to fix things up with pbis/samba integration[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Courier New]$> /opt/pbis/bin/samba-interop-install --install[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]3. And restart the samba service[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Courier New]$> service smbd restart[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]Now you should be able to access the samba share from a windows or mac computer in the domain.[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]Google for samba documentation in order to do other stuff with your SMB shares...[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]**And now for AFP**[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica](Be sure to make yourself root or use sudo for each step)[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]At the time of writing - ubuntu does not have apt packages for netatalk version 3. - much preferable to version 2 - so we will have to compile for ourselves:[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]1. First install git[/FONT]
[FONT=Courier New]$> apt-get install git[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]2. Create a dir for the source files[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Courier New]$> mkdir /usr/source[/FONT]
[FONT=Courier New]$> cd /usr/source[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]3. And download the files with git:[/FONT]
[FONT=Courier New]$> git clone git://git.code.sf.net/p/netatalk/code netatalk-code [/FONT]
[FONT=Helvetica](This takes some time)[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]4. Now you need a few more packages to compile[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Courier New]$> apt-get install autoconf libtool automake pkg-config make build-essential[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]5. And the following packages are required for all options (except ACL and Kerberos - even though AD is a Kerberos implementation, pbis makes PAM use sufficient) with netatalk. This requires a bit more disk than is absolutely necessary, but better safe than sorry if you want to change your setup later on:[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Courier New]$> apt-get install libssl-dev libgcrypt11-dev libdb-dev libpam0g-dev libwrap0-dev libavahi-client-dev libldap2-dev libcrack2-dev systemtap-sdt-dev libdbus-1-dev libdbus-glib-1-dev libglib2.0-dev libevent-dev[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]6. We will also want the server to be announced via Bonjour (avahi) so:[/FONT]
[FONT=Courier New]
[/FONT]
[FONT=Courier New]$> apt-get install avahi-daemon[/FONT]
[FONT=Courier New]
[/FONT]
[FONT=Helvetica]7. And we are ready to make and install[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]Go to the netatalk-code dir[/FONT]
[FONT=Courier New]
[/FONT]
[FONT=Courier New]$> cd /usr/source/netatalk-code/[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]8. Prepare scripts by[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Courier New]$> ./bootstrap[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]9. And configure the build[/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Courier New]$> ./configure \[/FONT]
[FONT=Courier New]    --with-init-style=debian \[/FONT]
[FONT=Courier New]    --with-cracklib \[/FONT]
[FONT=Courier New]    --without-libevent \[/FONT]
[FONT=Courier New]    --with-pam-confdir=/etc/pam.d \[/FONT]
[FONT=Courier New]    --with-dbus-sysconf-dir=/etc/dbus-1/system.d [/FONT]
[FONT=Helvetica]
[/FONT]
[FONT=Helvetica]Successful completion gives something like this:[/FONT]

[FONT=Helvetica]
