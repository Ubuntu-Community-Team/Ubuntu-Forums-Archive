### Post by Morbius1 on 2018-05-05
We can do this the hard way or the easy way.

Let's do the easy way first:

On your ubuntu server run this command to find the exact host name:
```
hostname
```
On your Win10 machine go to explorer and enter that same host name like this - and don't forget the ",local' at the end:
```
\\ubuntu-host-name.local
```
For this to work you need to have avahi-daemon running on Lnux. To see if it's running:
```
sudo service avahi-daemon status
```
Of course you coule also use it's ip address:
```
\\192.168.0.100
```

If you want to do this the hard way I will need more information. Please post the output of the following commands:
```

```


### Post by Morbius1 on 2018-05-05
> Processing section "[General]"
Did you forget to post that part of the output? Because right now you have no shares for anyone to access.
> hostname - [server-ubuntu]
I'm not 100% sure on this but I do not believe you can have a netbios name with a [ or a ] in it.

What I would do is edit /etc/samba/smb.conf and:

[1] Add a share definition at the bottom of the file.

[2] Then right under the **workgroup = WORKGROUP** line add these two:
```
netbios name = ubuntu-server
name resolve order = bcast host lmhosts wins
```
[COLOR=#0000cd]*The netbios name does not have to be ubuntu-server but it does have to be 15 characters or less in length.*[/COLOR]

Save smb.conf then restart samba in this order:
```
sudo service smbd restart
```
```
sudo service nmbd restart
```

Now take the dog for a walk. The old nmbd mechanism is somewhat primitive in today's world and restarting it makes the clients to this machine have to rescan the network. Maybe 5 minutes or so.
