auto lo
iface lo inet loopback
 
# The primary network interface
auto eth0
iface eth0 inet static
 
          address 192.168.2.106
          netmask 255.255.255.0
          gateway 192.168.2.1
          network 192.168.2.0
          broadcast 192.168.2.255
 
             dns-nameservers 192.168.2.105
             [COLOR=red]dns-search google.com[/COLOR]
```To This:```
auto lo
iface lo inet loopback
 
# The primary network interface
auto eth0
iface eth0 inet static
 
          address 192.168.2.106
          netmask 255.255.255.0
          gateway 192.168.2.1
          network 192.168.2.0
          broadcast 192.168.2.255
 
             dns-nameservers 192.168.2.105
             [COLOR=red]dns-search linux.local[/COLOR]

### Post by jdthood on 2013-02-16
A few comments.

1. Having
```

    dns-search google.com

```

in /etc/network/interfaces is unusual. Why would you want to be able to resolve short names (i.e., non-FQDNs) in the google.com domain in particular?

2. Using dns-domain
```

    dns-domain linux-master

```
is deprecated. Use dns-search instead.

3. ranger12 is right: mDNS uses the '.local' TLD. I suggest: save yourself trouble in the future and just adopt some other private TLD such as '.private'.

4. On the machine running named, configure named to listen on 127.0.0.1 and set "RESOLVCONF=yes" in /etc/default/bind9. That way, programs running on the server use the local nameserver instead of the upstream nameserver.

To make sure named listens on 127.0.0.1 and an external IP address x.x.x.x, add a line
```

    listen-on { 127.0.0.0/8; x.x.x.x; };

```
to the options{} statement in named.conf.options.
