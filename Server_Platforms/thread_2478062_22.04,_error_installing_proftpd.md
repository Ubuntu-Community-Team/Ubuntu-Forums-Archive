Have you tried to install a fresh copy of Ubuntu 22.04 and fresh install of proftpd to see if it "can" work?  If you can get the default into a working state, then start modifying it to fit your use a little at a time and insuring it continues to work after every change.

I'd recommend doing this test install inside a virtual environment (such as VirtualBox on a Windows PC).  Once you have a test version working with your modified config/data, you can then figure out how to migrate.  I always prefer installing the next OS version from scratch and configure the new application based on current defaults and migrate the specific config changes to ensure I do not have a stale config with depreciated settings that don't actually work.  Then migrate the data and once I verify everything works and mimics production, I flip a switch (by swapping IPs) and make the new box the production system.   If I only have 1 physical server, I would use a virtual machine to temporarily host production so the services remain online while I blow out the physical box and re-build it.  Of course, by this point, you should already have proven documentation on how to get it working on the current version and all data / config on the VM should be able to migrate over to the freshly reloaded box without issue.

Another suggestion is to purge the configs.  When you do "apt remove proftpd-core" it will only remove the application.  Configs tend to remain behind and if there is something wrong with them, a re-install will do nothing to change the result.  You will need to also run "apt purge proftpd-core" to remove the default config files.  If you added more config files, you will likely need to manually remove them too.  But make sure you keep a copy somewhere first so you can see what is different from the default install and make only the necessary changes to the default install by copying needed changes in your old config.
