Hello

I need to configure Ubuntu Server to work with vhosts.

I think that problem is connected with my local DNS server which i configured first time in my life :)

In my company we have Windows server with main DNS server, and my Ubuntu Server, Windows administrators don't know anything about configuring DNS Server on linux so i need to do it by myselft (i'm programmer and i'm using linux for last 7 year in home and offices as main system, so i'm not newby like them).

I will tell few words about our infrastructure:

- main server has IP 192.168.6.103, this is Windows Server with DNS server
- my Ubuntu server has IP 192.168.6.111 and Ubuntu is installed in virtual machine on Windows Server

Main domain in my office is corp.office.pl, my Ubuntu Server DNS has domain corp2.office.pl, all requests from from corp2.office.pl domain are redirected to my Ubuntu DNS ns1.corp2.office.pl

I have lamp server installed, and when i'm opening on my local pc: [http://ubuntu.corp2.office.pl](http://ubuntu.corp2.office.pl) it correctly opens my main apache page.

I have created vhost using webmin, and i know that my vhost is correctly visible because when i'm running on Ubuntu Server command: "wget myvhost.corp2.office.pl" i get my vhost index page - on server it works great but when i'm trying to open the same vhost url from my local pc it don't works :(

Maybe it's connected with some wrong configuration of my DNS server because when i'm running "dig myvhost.corp2.office.pl" on local pc (with linux) i'm getting info that url is redirected on ns1.corp2.office.pl DNS (this is DNS server on my Ubuntu Server).

Here are my DNS config files:

/etc/bind/db.corp2.office.pl

```
; BIND reverse data file for empty rfc1918 zone
;
; DO NOT EDIT THIS FILE - it is used for multiple zones.
; Instead, copy it, edit named.conf, and use that copy.
;
$TTL    86400
@    IN    SOA    ns1.corp2.office.pl. admin.corp2.office.pl. (
                  1        ; Serial
             604800        ; Refresh
              86400        ; Retry
            2419200        ; Expire
              86400 )    ; Negative Cache TTL
;
@    IN    NS    ns1.corp2.office.pl.

ns1    IN    A    192.168.6.111

plume    IN    A    192.168.6.111

*.corp2.office.pl    IN    A    192.168.6.111

*.plume.corp2.office.pl    IN    A    192.168.6.111


```


/etc/bind/named.conf

```

include "/etc/bind/named.conf.options"; // standard file without changes
include "/etc/bind/named.conf.local"; // standard file without changes
include "/etc/bind/named.conf.default-zones"; // standard file without changes


zone "corp2.office.pl" {
        type master;
        file "/etc/bind/db.corp2.office.pl";
};

```
