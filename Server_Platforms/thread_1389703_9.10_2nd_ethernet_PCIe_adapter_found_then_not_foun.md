### Post by DrJohn999 on 2010-01-24
During install of 9.10 server, both ethernet adapters were found. One is built-in to the motherboard, the second is on a PCIe slot. Selected the on-board adapter for eth0 during install, set fixed IP on the web, connected, updated, etc. w/ no problems. The second adapter is for the LAN.

BUT, on first and subsequent reboots after installation complete the second adapter is not detected at all, as if it were not even plugged into the PCI or the driver not loaded. This happens on either of two slots, with two different adapters, Intel or Netgear, both pretty recent, and both of which are detected and work in a different box. 'Ethtool eth1' says device not found.

This is a fairly recent motherboard (ASUS M3N78EM). Since the adapter was found during installation I'm assuming that the problem is not with not the board. I've built several server systems with two or three ethernet adapters and have never run into this situation.

### Post by DrJohn999 on 2010-01-24
I confirmed this is not a hardware problem. This system was in use as a 9.10 desktop system. I had replaced its drive with a pair of new ones as part of starting to convert it to a server, so I had the older drive intact on hand. The only h/w difference between the last boot on the original drive and now is the addition of the second ethernet adapter.

Booting off of the original drive with 9.10 desktop found the new card immediately and autoconfigured it to DHCP.

So, there is something about the new 9.10 server install that's causing the card to be missed. I notice now that the OS has not been updated post-installation, so that's next...
