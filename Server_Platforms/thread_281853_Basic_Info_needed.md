### Post by elst on 2006-10-21
The RUTE book gives you a lot of information about the command-line and system administration. If you install the package "rutebook" you will have a copy on your system, and can then read it by putting this in the address bar of your Web browser:

file:///usr/share/doc/rutebook/html/index.html

I always keep this in my bookmarks! It's a little dated, but is one of the most comprehensive texts around.

For a gentle introduction to scripting and the command-line:

[http://www.linuxcommand.org/](http://www.linuxcommand.org/)

The Ubuntu Server Guide is also helpful.

### Post by elst on 2006-10-21
Advanced users tend to stick to the command-line, because it's much faster and you can automate all your routine tasks with scripts. The linuxcommand tutorial tells you pretty everything that you need to know about shell scripting - it takes about an hour to go through, and if you have only used DOS and Windows you may be amazed at what you can do.

There are graphical tools around for various functions, but many users don't feel a pressing need for them, so this area  is underdeveloped compared to other operating systems.
