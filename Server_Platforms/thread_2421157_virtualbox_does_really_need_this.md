### Post by TheFu on 2019-06-17
virtualbox is for desktop-on-desktop virtualization.  It is designed that way.

If you want true server virtualization, then look to KVM+qemu or Xen or a paid/commercial solution. KVM can run desktop OSes.  I run my main desktop in a VM and a Win7 media center in another. Use remote access methods to access those systems. Use virt-manager from a local or remote system to create/install new VMs. It can be used to access the VMs, but I find it cumbersome. Prefer using remote X11 or remote SPICE protocols to access the clients. Vinagre is a connection manager tool if you don't want to control access from a shell.

Lots of choices. Extremely flexible.  Lots of storage and networking capabilities, unlike other dumbed-down tools. Virt-manager makes using KVM as easy as vbox or VMware player/Workstation, but with access to the advanced capabilities still.

### Post by dmnur on 2019-06-19
If you really need VirtualBox, you can add the **--no-install-recommends** option and thus skip the Qt frontend packages:
```
apt-get install --no-install-recommends virtualbox
```
Some X11 components are still required, but not that much.
