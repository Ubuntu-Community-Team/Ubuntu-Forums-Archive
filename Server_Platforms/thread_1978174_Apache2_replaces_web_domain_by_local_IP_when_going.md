Hi again, I have installed Ubuntu Server 12.04.

I have set up an Apache installation with MySQL, PHP, etc. Everything works great within my network. 

When I try to connect to /home/user/www/index.html from a browser on a remote computer typing something like

[http://mydomain.com/index.html](http://mydomain.com/index.html)

I can see the index.html page with no problem. Now, if I create a subdirectory and a web page within, like /home/user/www/subdir/index.html by typing

[http://mydomain.com/subdir/index.html](http://mydomain.com/subdir/index.html)

Apache will redirect to

[http://192.168.1.XXX/subdir/index.html](http://192.168.1.XXX/subdir/index.html)

which of course works OK within my network but not on a remote network where 192.168.1.XXX is obviously not the IP address of my server but of some local machine that might not even exist.

Anybody has an idea of what I might be doing wrong here?

```

$wgserver = "http://192.168.1.xxx/";
```

in LocalSettings.php and every request of a page in the wiki was being redirected to that IP address.
