### Post by darkod on 2017-01-06
On the external hdd, does the fstab contain UUID for the root partition or name like /dev/sda1? If it's /dev/sda1 it might be confusing it with the internal HDD. If it's the UUID it should boot the root partition from the ext HDD correctly. And in such case it should not be read-only.

Also grub from the ext HDD should point to UUID, not sda1 because again that might confuse it with the internal disk. Check out /etc/fstab and /boot/grub/grub.conf on the ext HDD.

### Post by darkod on 2017-01-06
The whole content of grub.conf is important, not only where it is generated from. Is that the whole content? Those are only the beginning comments...

Why does the ext HDD fstab include cryptswap? Did you have encryption on the server? Or you put that line there? If you did, remove it. If you have encryption on the laptop, no need to add cryptswap while booting the ext HDD. In fact it can work without swap too, so you can decide to try boot it using the swap on the ext HDD, or not using swap at all. But don't add encrypted swap if you had no encryption set up on the server. And if you did have encryption, I think you will need to boot it another way, not like this.

The /etc/default/grub is not so relevant, no need to post it. The grub.conf is much more important.

### Post by darkod on 2017-01-06
If you look in the grub.cfg, in the BEGIN /etc/grub.d/10_linux section, look closely in the menuentry entries.

They say:
set root='(hd0,msdos1)'

It is true that also a search statement exists with the correct UUID for the server root partition, but the above set root would lead it to the internal HDD because that's hd0 (first HDD). Not sure if the search statement overrules that or not... You can try setting the set root to (hd1,msdos1) because that would be second HDD, first partition. And the ext HDD is the second HDD for your laptop.

Also if it reports errors on sdb1 you might need to run fsck on it unmounted...

So boot your laptop OS and then try something like:
```
sudo fsck /dev/sdb1
```

You can search for more syntax about fsck in the man page or on google.
