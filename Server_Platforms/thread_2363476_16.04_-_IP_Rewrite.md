### Post by sleepingpanda on 2017-06-10
So i'm trying to make my local IP on my VPS redirect to the domain name so you can only access the site through the registered and encrypted domains.


I'm altering .htaccess trying to figure how the code should be blocked.


this is what I have so far, the first block works fine, it's the second that I need assistance with.

```
RewriteEngine on
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]
#
#
#
RewriteEngine On
RewriteBase \
RewriteCond %{HTTP_HOST} ^my\.ip$
RewriteRule ^(.*)$ http://www.mywebsite./$1 [L,R=301]


```

The script was messy. By deleting the following lines and just adding extra conditions and rules I got the redirect to work.


```


RewriteEngine on
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]
RewriteCond %{HTTP_HOST} ^ip address$
RewriteRule ^(.*)$ http://www.urlofchoice.org./$1 [L,R=301]


```
