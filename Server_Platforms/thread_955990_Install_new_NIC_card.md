### Post by Krupski on 2008-10-22

> I had a server crash... I purchased a new motherboard (on board NIC) and attached the harddrive.  Everything works great except the NIC.  Can someone point me to a resource on how to setup the new NIC?

Thanks mucho

Your server still has the MAC address of the old NIC associated with the "eth0" network device.

So, do this:

```

sudo nano /etc/udev/rules.d/70-persistent-net.rules

```

It should look something like this:

```

# This file was automatically generated by the /lib/udev/write_net_rules
# program run by the persistent-net-generator.rules rules file.
#
# You can modify it, as long as you keep each rule on a single line.

[COLOR="Red"][B]# PCI device 0x8086:0x294c (e1000e)
SUBSYSTEM=="net", ACTION=="add", DRIVERS=="?*", ATTR{address}=="00:1c:ff:ff:ff:ff", ATTR{type}=="1", KERNEL=="eth*", NAME="eth0"[/B][/COLOR]

```

Note the part in red - that's the info from your old NIC. DELETE that part.

In other words, the file should be empty except for the comment header at the top.

Then, save the file (ctrl-o to write the file, ctrl-x to exit the editor), then reboot.

Your new NIC should now be working.

Good luck!
