Just like websites are built from text pages so is Apache2.  The configuration files are just that.

If you want to learn about Apache2 there is an enormous amount of info on the net.  Such as [links]("https://help.ubuntu.com/10.04/serverguide/C/httpd.html") on this forum.  [The Apache Cook Book]("http://oreilly.com/catalog/9780596001919").

If you want to be able to manage Apache2 with a GUI, there are web interfaces such as Webmin.  But that does not teach you Apache2, it just aids the setup of websites and such.

In my opinion the best way to learn is jump in and ask specific questions when you need help or have a problem.

Try it with just editing the text files first.  It is easier than you think.
