### Post by boudies on 2010-01-19
A good resource for RAID is [http://www.linuxhomenetworking.com/wiki/index.php/Quick_HOWTO_:_Ch26_:_Linux_Software_RAID](http://www.linuxhomenetworking.com/wiki/index.php/Quick_HOWTO_:_Ch26_:_Linux_Software_RAID)

The full details of my RAID devices can be found as follows:
```
# mdadm --detail --scan --verbose
ARRAY /dev/md1 level=raid1 num-devices=2 metadata=00.90 UUID=73489ef2:98e077ef:14da86fa:9ad3ba1d
   devices=/dev/sda3,/dev/sdc3
ARRAY /dev/md2 level=raid1 num-devices=2 metadata=00.90 UUID=c6c659d5:fd95a340:e06c55c3:bc566ef8
   devices=/dev/sda5,/dev/sdc5
ARRAY /dev/md3 level=raid1 num-devices=2 metadata=00.90 UUID=5092d4ab:d6a47555:e7b1a5c9:e748c450
   devices=/dev/sda6,/dev/sdc6
ARRAY /dev/md4 level=raid1 num-devices=2 metadata=00.90 UUID=3d3fe3f0:5a5bd675:921009a9:066146e5
   devices=/dev/sda7,/dev/sdc7
ARRAY /dev/md0 level=raid1 num-devices=2 metadata=00.90 UUID=0c8cda94:8e708953:74dbe486:d2cb712b
   devices=/dev/sda1,/dev/sdc1
```Which reveals that the second entry in mdadm.conf is the active one.

I have removed this entry to get a clean mdadm.conf. Issue solved.
