### Post by SeijiSensei on 2011-01-30
> **living2000 said:**
> I have been the webmaster of a webpage for a few years and I have always installed Apache and PHP using the source code. The reason is that I wanted to use a recent version of them and I couldn't find it in the repositories.

I stopped building them by hand years ago now.  I compiled my own binaries for Apache 2.0 and, especially, PHP 4 when the standard binaries were at 1.3 and 3.x respectively.  Now the standard releases offer Apache 2.2 and PHP 5.x.  Other than security updates, I don't see much reason for trying to stay on the cutting edge.  All my pages function as I want them to, and I don't find myself wondering if there's some exciting new feature of PHP I must have.

I run CentOS 5.5 as a webhosting platform with Apache 2.2.3 and PHP 5.1.6.  I update from the repositories when there are security fixes; other than that I'm happy with those.
