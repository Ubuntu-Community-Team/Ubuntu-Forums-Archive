### Post by drfox on 2013-12-19
The fix is pretty easy. Log into your server with SSH. Change to sudo user and navigate to the root directory. Then mkdir /tmp
Next time you log in with webmin, the error will be gone.

### Post by SeijiSensei on 2013-12-20
The directory may be there already but with the wrong permissions.  /tmp is world-writable with the "[sticky bit]("http://en.wikipedia.org/wiki/Sticky_bit")" enabled.  First, type "ls /" to make sure you already have a /tmp directory.  If not, create it like this:

```
sudo mkdir /tmp
```

Next you have to set its permissions like this:

```
sudo chmod 1777 /tmp
```

Because everyone can write to /tmp, it would be possible for one user to delete another's files.  The sticky bit prevents that; only a file's owner may modify or delete it.
