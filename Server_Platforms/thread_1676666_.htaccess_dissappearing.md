### Post by gmargo on 2011-01-27
Files or directories that start with a dot are not truly "hidden".  It's just a convention that programs like "ls" respect.  The .htaccess file will be found by the apache web server just fine.

[http://www.linfo.org/hidden_file.html](http://www.linfo.org/hidden_file.html)
Or google for "unix hidden file"

### Post by SeijiSensei on 2011-01-27
> How can I make it visible in ftp applications?

Usually .htaccess doesn't change very often.  If you have shell access you can create a symbolic link to htaccess.txt like this:

```

cd /path/to/your/website
mv .htaccess htaccess.txt
ln -s htaccess.txt .htaccess

```

Obviously you need the appropriate permissions to make these changes.

Now if you change htaccess.txt the server can still reference the file as .htaccess.

On the other hand, there's a reason why .htaccess is "hidden."  It contains important configuration information that you don't want to allow random browsers to display.  If you take the path I just suggested, and someone knew that there's also an htaccess.txt file, they could GET that file with an HTTP request.  Requests for .htaccess are routinely blocked by the Apache server. (Look in the configuration files in /etc/apache2 for details.)

My advice is to upload the file when you're ready, then name it .htaccess and leave it alone.
