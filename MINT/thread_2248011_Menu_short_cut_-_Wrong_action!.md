Solved - after another issue, (M17 locks all controls after log in) 

A 'IT' friend helped with the "Controls locked" issue and this fixed the menu shortcuts "crosslinks" somehow. As a result both isses were resolved. Unknown why or how this occured? If you have this problem then one way to solve could be to create an admin login on the login page via terminal, step 2--> change the current (hidden folders) .local and .config to--> .localold and .configold - step 3 create new new folders in the "Old Login" which has the problem and then try login again.

However... in Evolution (Evo) email,  it required that the emails all had to be copied from the .localold and .configold locations... to fully retrieve them, other wise (Evo) requested a new setup by default, Goodnews Thunderbird did not and all emails were saved!
