### Post by Laibcoms on 2008-01-28
Its a fun question, and let's play the "What If?" game here.

So What If that happened?

The most likely scenario I can see is this:

The MS Windows code will be studied and copied to be implemented in GNU/Linux and OSX platforms, to the point that MS Windows will become a 3rd Ranked OS but never forgotten, and with GNU/Linux and OSX being the 'new' stiff competitors.

The reason for this view is that, we all know all the advantages of GNU/Linux platform already.  Why fix the flaws of MS Windows when you already solved it with Linux?  Example, Viruses and Disk Fragmentation to mention just two.  Not that Linux is "virus-immune" or "fragmentation-immune", but because GNU/Linux is a community project, every possible new way of doing things have been put into it that made it almost virus-free and almost fragmentation-free, where MS Windows utterly failed.

Will there be people who will "fix" things with MS Windows?  Yes there will be LOTS of them, but I doubt these "lots" are enough to be at par with the support of GNU/Linux and OSX.

Besides, GNOME, KDE, XFCE will also use the codes of MS Windows to improve their respective desktop environments.

And then, people will also see that with competition, there is improvement.  GNOME, KDE, and XFCE (there are lots more) are competitors.  But under the MS Windows platform, it has no competitor.

I doubt someone will create a competitor desktop environment for MS Win platform, why create under it when you can do better stuff under GNU/Linux?

So as I've said earlier, MS Windows will become a 3rd ranked platform, but it will never be forgotten.
