File sharing *does not automatically qualify* as piracy.

Piracy is basically infringement on one's intellectual property rights (including copyright). People can share files without infringing on other people's rights *if the rights holders allow their files to be shared*.

**Case in point:** The Ubuntu OS can be shared through BitTorrent. This is not piracy because the makers of Ubuntu, who own the rights to it, allow people to share it.

### Post by Samhain13 on 2009-01-30
Ang mga binanggit mong file types ay gumagamit ng XML (Google: "ODF vs OOXML"). Ang XML, depende sa gamit, ay maaari din tawaging source code tulad sa gamit ng XHTML, MathML at SVG. May halo silang data, may halo din silang instructions tulad ng embedded Javavascript at CSS (hindi lang data ang mga ito). Ultimo ang "markup" ay hindi lamang data (ang data ay ang mga binabalutan ng tags o nilalaman ng attributes); ang markup ay may mahalagang ibig-sabihin sa mga user-agent o host applications. Kaya nga sa iyong browser, ang sabi ay "View **Source**" at hindi "*View Data*".

Ganito din ang .blend files. At kung tutuusin, mas komplikado pa ang mga ito kaysa sa mga popular na XML formats. Ang Blender ay hindi lamang "drawing software" na gumagawa ng mga .gif, .jpg, atbp. data containers. Maraming interaction ka na puwedeng gawin sa Blender gamit ang "GameEngine" o Crystal Space, na tumatanggap ng C++ at/o Python instructions na na sa loob ng .blend files. Ang .blend files ay naglalaman din ng mga instruction para sa Compositing at Animation (hindi lang ako sigurado kung ito'y Python o C++ o halo) kung gagamitin mo ang Blender sa paggawa ng mga palabas gamit ang Video Sequence Editor.
