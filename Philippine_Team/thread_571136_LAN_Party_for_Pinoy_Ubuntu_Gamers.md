This is a multiplayer roleplaying adventure game with a system based on TSR`s 3rd Edition Advanced Dungeons & Dragons(R) rules. Set in a vast land of challenges and opportunities, you can change the destiny of the cities of Neverwinter and Luskan in over 25 single-player missions that will keep you playing for hours.

System requirements (edited)
2.2.x or better Linux kernel with glibc-2.x and X-Windows
733 MHz CPU
128 MB RAM
2.1 GB free disc space
Hardware Accelerated 3D Graphics card with 32 MB video memory
OSS Compatible sound cardcr

Installation instructions
These instructions are included on the ISO's (the emphasised part is mine):

Make sure you have the latest Graphics driver installed for your card. I can't say for sure whether or not this game will run with a generic X driver.

If you choose to not "burn" this image to disk and would prefer to just keep the .ISO images around, you can mount it to the directory of your choice (eg. /mnt/cdrom) with the following command:

1) mount -t iso9660 /path/to/NeverwinterNights1-linux.iso /mnt/cdrom/ -o loop

Obviously, you'll need to make sure 'Loopback device support' is enabled in your kernel if you choose to go this route. Statically compiled or as a loaded module is fine. If you have 'burnt' this image to disk, and it is no longer an ISO, mount the CD as you normally would.

2) cd /usr/local/games

3) tar -xjvf /mnt/cdrom/files-1.tar.bz2

Unmount the first image and then mount the second one

4) tar -xjvf /mnt/cdrom/files-2.tar.bz2

5) ln -s /usr/local/games/NeverwinterNights/nwn /usr/local/bin/nwn

Now you can start the game by typing nwn, IF your working directory is /usr/local/games/NeverwinterNights.


Note - Obviously, the CD key won't work for Multiplayer. You'll have to get one on your own if Single Player doesn't interest you.
-=-=-=-=-=-=-

If you are having problems saving games and settings, run the fixinstall script, and make sure there are no warnings. By default, the archive extracts with UID (User ID) 500. On current distributions however, the first user has UID 1000. This causes the file ownerships to effectively be mapped to root, so any given user account does not get write permission on them. To change the UID of a given file, use chown (as root). The command takes a username and a file or directory. (Like chown voidz0r readme.txt.) To apply recursively on whole directory's use -R (chmod -R voidz0r exampledir). For details, see man chown.
