### Post by Laibcoms on 2009-04-03
> **dodimar said:**
> 32 bit... sa desktop ko .. siguro sa mga experimental virtual machine ko mag install din ako ng 64 bit... pero totoo bang napapabilis ng 64bit OS ang system kahit less than 4gb ram?

For me, I can feel the difference when playing (online) games.  For example, World of Warcraft and Guild Wars under Ubuntu 64-bit is much faster and real-time compared to under Ubuntu 32-bit and Windows XP Pro 32-bit.

I say, Jump to 64-bit for Jaunty.

The question is this: Which applications are you using that doesn't work for 64-bit pre-Jaunty?

For me, almost all of the applications I use works under 64-bit (games not included).

I switched to 64-bit since Gutsy.  I can say that Jaunty is the most stable and much more "out-of-the-box" of all the releases I've tried (I started with Fiesty; but I first touched an Ubuntu with Breezy or Dapper via a friend trying to convince me to go back to Linux :p )

Go for it, don't waste the power of your 64-bit processor.  Probably there's only 1 or 2 applications that will require recompiling - but so far, by experience, the last time I have to recompile is back with Gutsy, and a few for Hardy.

Besides, 64-bit is not the future anymore, it is today's playground.  But personally, based on my own research, if an application was not built to take advantage of the power of 64-bit, there's little gain.  The major change is with the way the memory blocks are divided and read/write (someone else can explain it in more details).

But again, at least for gaming, I experienced a difference.  Maybe, and I am not sure, WoW and GW clients auto-switches to 64-bit mode when it detects a 64-bit OS.

Again, install 64-bit Jaunty, it'll be fine.  If you don't use extensive apps like graphics editing and video editing, you won't notice you're using a 64-bit OS :p  at the end, you're not wasting your money ;)
