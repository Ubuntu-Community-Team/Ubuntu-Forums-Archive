### Post by djsroknrol on 2006-04-15
Welcome all;

I've been working on and off on Virtual Desktops, or as they're called "Big Desktops" since I installed Ubuntu on my main 'puter. I'd like to share this as they're isn't much on ATI, and the OSS radeon driver.

I have a dual head setup on a Radeon 9200SE going to two monitors, one 17" (my main) and a 15"(my secondary).

The only difference in my setup is that my second monitor sits on top of my computer hutch shelf. If there was a way to go side to side, why not up and down? Well, it so happens that if you use the Radeon driver, you don't need Xinerama, as the driver because it runs it's own "sudeo-Xinerama". and the Radeon driver supports 2D and 3D acceleration. The Radeon driver supports the following cards:
R100         Radeon 7200
RV100       Radeon 7000(VE), M6
RS100       Radeon IGP320(M)
RV200       Radeon 7500, M7, FireGL 7800
RS200       Radeon IGP330(M)/IGP340(M)
RS250       Radeon Mobility 7000 IGP
 R200        Radeon 8500, 9100, FireGL 8800/8700
RV250       Radeon 9000PRO/9000, M9
RS300       Radeon 9100 IGP
RS350       Radeon 9200 IGP
RS400       Radeon XPRESS 200/200M IGP
RV280       Radeon 9200PRO/9200/9200SE, M9+
R300        Radeon 9700PRO/9700/9500PRO/9500/9600TX, FireGL  X1/Z1  (2D only)
R350        Radeon 9800PRO/9800SE/9800, FireGL X2 (2D only)
R360        Radeon 9800XT (2d only)
RV350       Radeon 9600PRO/9600SE/9600, M10/M11, FireGL T2 (2D only)
RV360       Radeon 9600XT (2d only)

Before you start, remember to back your xorg.conf in case you want to go back..

This is how I did it:

```
sudo dpkg-reconfigure xserver-xorg
```

I took the defaults, and let the routine configure a clean xorg.conf file for me. 

```
sudo apt-get install xserver-xorg-driver-ati
```

and then:
```
sudo gedit /etc/X11/xorg.conf
```

Then I changed the Section "Device" to look like this. Replace my identifier with  your card identified from the list above with the rev number 

```
Section "Device"
	Identifier	"ATI Technologies, Inc. Radeon 9200 SE (RV280)"
	Driver		"radeon"
	BusID		"PCI:1:0:0"
	Option "RenderAccel" "True"
	Option "SubPixelOrder" "None"
	Option "ColorTiling" "False"
	Option "MergedFB" "True"
	Option "MetaModes" "1024x768-1024x768 1024x768-1024x768"
	Option "MergedDPI" "100 100"
	Option "CRT2Position" "Below"
EndSection
```

There are 4 values for CTR2Position, namely "RightOf" "LeftOf" "Above" and "Below" (Note the CRT2Position option on mine..below makes the top monitor slide windows to the monitor "below") Seeing as I had similar monitors, I thought I'd try similar settings and "MetaModes" saves alot of key pounding..:)

Next, we add the second monitor, and define the two monitors and both screens:

```
Section "Device"
	Identifier	"ATI Technologies, Inc. Radeon 9200 SE (RV280)secondary"
	Driver		"radeon"
	BusID		"PCI:1:0:1"
	
EndSection

Section "Monitor"
	Identifier	"DELL D1025TM"
	Option		"DPMS"
EndSection

Section "Monitor"
	Identifier	"PRINCETON ULTRA 51"
	Option		"DPMS"
EndSection

Section "Screen"
	Identifier	"Screen0"
	Device		"ATI Technologies, Inc. Radeon 9200 SE (RV280)"
	Monitor		"DELL D1025TM"
	DefaultDepth	24
EndSection

Section "Screen"
	Identifier	"Screen1"
	Device		"ATI Technologies, Inc. Radeon 9200 SE (RV280)secondary"
	Monitor		"PRINCETON ULTRA 51"
	DefaultDepth	24
	
EndSection

```

After that, I changed the "ServerLayout" section to look like this:

```
Section "ServerLayout"
	Identifier	"Dual Head Layout"
	Screen  	"Screen0" 0 0
	InputDevice	"Generic Keyboard"
	InputDevice	"Configured Mouse"
```

Note here that there is only one reference to a screen. That was all...and it works so good, that I went a step further....

```
Section "Extensions"
	Option "Composite" "Enable"
EndSection

```

This allowed me to keep my Xcompmgr, 3Ddesk, and my xdesktopwaves which is a quick converter to show someone who's only ever known M$...
