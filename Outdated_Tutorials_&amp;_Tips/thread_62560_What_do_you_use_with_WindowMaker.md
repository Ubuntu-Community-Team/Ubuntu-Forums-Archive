### Post by Lord Eclipse on 2005-09-06
I'm not sure I understand your question, and what works for me may be a "Your Mileage May Very" scenario. However, as long as you have all the underlying libraries in place for Gnome and KDE, WM integrates nicely with both and runs all programs just fine. It just looks like a NeXT app instead of a Gnome / KDE app for the most part. Some fonts and hard coded things may look a little out of place, but I have never had a program refuse to run under WM. 

I've run YaST, Evolution, Firefox, Koffice, GIMP, VLC, etc., all with WM as my Window Manager and had no problems. So you should be able to run everything you are able to run under Gnome or KDE, however if you're having problems getting, say KDE, running under Ubuntu then WM won't be of any help anyway. It needs for the libraries to already be installed and ready to run. So Gnome apps under Ubuntu and KDE apps under Kubuntu should be absolutely no problem at all. Getting both sets of apps running on either system may be a little work, however.

I use xubuntu (or Xfce4) with WindowMaker.  Just install xubuntu-desktop, put "xfce4-session &" in $HOME/GNUstep/Library/WindowMaker/autostart (make sure the file is executable), and tweak just about everything for WindowMaker with WPrefs.  I disabled the dock and clip since I use the Xfce4 panel.  It gives me a ubuntu-like desktop, with the fine granular control that Window Maker provides.  Desktop, panel, device management, etc, is all there.  I enjoy it, pretty much the perfect desktop for me.  Oh, and I had to add gnome-screensaver using xfce4-autostart-editor.
