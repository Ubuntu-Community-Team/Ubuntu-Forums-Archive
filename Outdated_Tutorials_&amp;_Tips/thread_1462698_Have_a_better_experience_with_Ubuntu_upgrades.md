### Post by estebarb on 2010-04-26
When we install Ubuntu, it automatically sets the default mirror to the nearest. In my case and I'm sure that many others have the same problem the mirror in my country (Costa Rica) is slow and is shared with many other countries. So, in a distro upgrade, many people downloading from the same server can reduce it quality and the installation gets broken.

To solve this, is as easy as change the mirror. Go to (Sorry, I have it in spanish, I'm translating menus) System>Administration>Software Origins. Next in download from yo check "others" and in the new windows press something like "seek best server". Close that and if it works then you are ok. If not you should select another server (near the later, but different). The active servers are here: [https://launchpad.net/ubuntu/+archivemirrors]("https://launchpad.net/ubuntu/+archivemirrors")

Why I post it? The last distro upgrade the automatic process (from auto updates) didn't work. It halts because can't reach the server. I think it will help people that don't have lots of near mirrors.
