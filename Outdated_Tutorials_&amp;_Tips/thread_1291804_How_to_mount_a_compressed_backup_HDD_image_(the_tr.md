### Post by tonifiddle on 2009-10-15
Few days ago I tried to restore a single file from a compressed partition image file generated from Partimage. Unfortunately I had a really bad luck .Once I checked the FAQs on the official website  I found out that this kind of backups are almost useless. What I mean by saying useless is that you can only backup and restore the entire partition of your hard drive but you cannot decompress, browse or mount that archive and so you cannot restore a single file/folder or group of files/folders. That is really pity because most of the people who perform that kind of backup typically want to be able to restore specific files later on.
 
 So... here is the solution a have found out and I hope that it might  help to someone there who want to mount his backups , browse and restore single files seamlessly.  
 
 Make sure you have installed  DD and  squashfs  
 ```
sudo apt-get install coreutils 
sudo apt-get install squashfs-tools
```The backup file generated by DD is RAW data and it will have the same size as the source partition. If your source partition is 20 GB than you need as a minimum 20 GB of free space on the destination drive where you want to place your backup. Before start the process, you might want to fill the partition with zeros. That way you will save a lot of space on the final squashed backup file.
 
Source partition have to be **mounted** in order to generate zeroed file :  
 To simplify the sudo commands ,you may use ```
sudo -s
``` That will switch Terminal to super user session so you don't need to type sudo every time

 ```
dd if=/dev/zero of=/media/yourpartition/big_file_full_of_zeros bs=1024
```once it is done we have to delete it:
```
rm /media/yourpartition/big_file_full_of_zeros
``` If we need a full hard disk backup (including all of the partitions)than we have to repeat the same procedure for each partition.Once it is done we have to unmount all source partitions and we are good to go.
 If we need a full hard disk backup (including all of the partitions)than we have to do same procedure for each partition
 Very often we need to backup system or home partition which cannot be unmounted . In this case boot from a CD-ROM and perform the backup  
 
Identify your source partition and unmount it
```
fdisk -l
umount dev/sdx 
```Start backup process
```
dd if=/dev/sdx of=/media/your-destination-drive/DDbackp
```When done we have to squash the file.
 ```
mksquashfs DDbackup /media/yourbackupdrive/squashedDD
```when ready we have to mount the squashedDD and DDbackup
 
first create mount points
 ```
mkdir /mnt/squash
mkdir /mnt/dd
``````
mount -t squashfs /media/drive/pathtofile/squashedDD /mnt/squash -o loop
``` then we need to examine the DDbackup file in /mnt/squash in order to obtain necessary information to determine the superblock
 ```
fdisk -lu /mnt/squash/DDbackup
 
```[IMG]http://img527.imageshack.us/img527/5906/shoty.png[/IMG]  
offset = 512 x start block value (in my case 63 ) = 32256
 sudo mount -o loop,offset=32256 -t auto /mnt/squash /mnt/dd
 Now open up your favorite file manager and browse and copy whatever you need
