### Post by Lednar on 2009-12-22
This is an article with an idea I am forwarding 
about what I think it should be done.

In my opinion now that Google has come out with the idea
of an operating system with Google Chrome
(browser) as the user interface, why not take
this and apply it to a browser that would worth the effort.

The idea is to create from an existing distro, 
another distro with Mozilla Firefox as user interface.

Today you can do many things with a
website interface: configure a router, manage
printers, manage package dependencies (sushi huh
 [http://sushi-huh.sourceforge.net/](http://sushi-huh.sourceforge.net/) ) and so on.

In my opinion a interface / program should be developed
that can be accessed through Firefox
to update / install applications within the
same operating system. (gnu / linux)

For example when accessing that interface 
Firefox requests the username and password 
and if these belong to a user from the "sudo" 
group of users (for example), then to be allowed 
to perform modifications.

It is not necessary to develop a new form of packaging
applications, firefox might communicate with
aptitude/yum through a plugin "mozilla-aptitude / mozilla-yum".
It could be used in deb / rpm packages updating
and app installation.

Moreover application developers 
should work with Mozilla to create plugins 
that make apps run within the 
browser interface.

There are many examples already being implemented:
vlc-plugin, mplayer-plugin, totem-plugin etc.

Developing plugins for other applications
should not be a barrier. Integrate gedit, abiword,
open office within Firefox. Make Firefox
behave thanks to a plugin or web application as
a half file browser (as Konqueror) but
also with ftp function as Total Commander or Gspace.

This is not what Google intends to do, is much better
because it would allow the user to use PC offline.

I think all developers should join to
achieve this, not just mozilla, but also
os developers as Ubuntu/Debian Community, 
Fedora Community, Sun and many others that 
wish to join.
