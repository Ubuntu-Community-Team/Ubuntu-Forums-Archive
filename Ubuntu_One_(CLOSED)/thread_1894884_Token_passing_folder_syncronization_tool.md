### Post by Denver Dave on 2011-12-13
Do you know of a sequential single-user, token passing folder syncronization tool?

I've been experimenting with SpiderOaks backup, which I gather is somewhat similar to ubuntu one and using it to sync KeepNote notes across PCs and operating systems.

This is asking a lot.  I've simplified so that I only am running SpiderOak and KeepNote on one PC at a time, but still, it is asking a lot to be changing notes as files are being synced with the network.  While the sync sort of works, there are issues.

So, I'm thinking, since I'm only one person, what is needed is some sort of token passing store to the server application.

One PC gets the token and stores changes back when they want and then releases the token when done.

A second PC would be notice that the first PC has the token and asked if wish to force release.  If the token was changed after last pull, pull again.

I'm just making this up, but seems like since there is only one user that would be willing <sigh> to only use the KeepNote application on one PC at a time, there ought to be a way to make this work.

Is anyone using ubuntu one with Keepnote and multiple PCs and Operating systems?

ReminderFox FTP network syncronization, while on a smaller scale, seems to work fairly well as long as I don't have two ReminderFox applications open and making changes at the same time.

Comments?  Suggestions?  Experience with similar?
