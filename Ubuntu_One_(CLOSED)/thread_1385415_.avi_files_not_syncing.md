There are many bugs in U1 client and they need to be resolved before doing anything else: [https://bugs.launchpad.net/ubuntuone-client/+bug/455544](https://bugs.launchpad.net/ubuntuone-client/+bug/455544)

This problem should be solved. You may need to stop the Ubuntu One client and delete the following config file: ~/.config/ubuntuone/syncdaemon.conf  Once, you've done that, try starting Ubuntu One again and it should work properly.

> **suniyo said:**
> I have two basic question only:
1. What file types are supported on ubuntu one? .avi, .ovg and .ogg are supported or not


We'll sync any type of file, without any concern for the file type. If you're referring to the web UI, we try to determine the file type and serve it up properly based on the mime/type. If we can't determine the file type, then the file should be offered for download instead of displaying directly in the web browser.

> **suniyo said:**
> 2. What is the file size limit for ubuntu one? 100mb? 500mb ??

The single file limit is 5 GB, as this is the limit of Amazon's S3 service. As Amazon changes that limit, our limit will change.

Thank you,

Joshua
