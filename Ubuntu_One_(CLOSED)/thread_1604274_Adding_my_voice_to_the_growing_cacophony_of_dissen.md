### Post by Carl Hamlin on 2010-10-23
So far, my experience with Ubuntu One has been pretty negative.

I installed 10.10 a couple of days ago, and got to work getting an Ubuntu One account established. Then, I put my contacts up, which worked like a charm, and still does.

However, file syncing doesn't work for me, and hasn't, ever. At first, I had the option in Nautilus, but now I don't even have that. When I tried to sync up in the beginning, it made noises like it was doing it, but never actually uploaded anything.

Since then, I have gone through a number of troubleshooting steps as outlined in the fora here:

Smoked the 'ubuntuone' directory in .config
Got rid of the cached passphrase in Passwords and Encryption Keys
Diddled about with u1sdtool --stop, --connect, etc...

Sync still doesn't work, the 'Connect' button is *still* greyed out, and now I'm to a point where anything I do with u1sdtool results in the following error:

```

Oops, an error ocurred:
Traceback (most recent call last):
Failure: dbus.exceptions.DBusException: org.freedesktop.DBus.Error.NoReply: Did not receive a reply. Possible causes include: the remote application did not send a reply, the message bus security policy blocked the reply, the reply timeout expired, or the network connection was broken.

```

I'm discouraged by the number of similar stories I'm reading, as well. Has anyone gotten this thing to work properly and consistently yet?
