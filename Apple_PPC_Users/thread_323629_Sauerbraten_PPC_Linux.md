### Post by Kubunteando on 2007-01-21
I have tried quickly your configuration and I don't have found anything.
I don't get the error: "libGL warning: 3D driver claims to not support visual 0x4b"

So in your case I would try the following:

. have you installed "driconf" from the repositories? I would execute "driconf" and enable HyperZ if not yet done.

- do you have S3TC enabled,  If not you can check:
[http://homepage.hispeed.ch/rscheidegger/dri_experimental/s3tc_index.html](http://homepage.hispeed.ch/rscheidegger/dri_experimental/s3tc_index.html)

- in Shell I would try to enable the MESA debug with "export MESA_DEBUG=true" and then execute "glxinfo", Check if you get any error messages in the output of "glxinfo".

Another possibility (what I did) is compile the DRI myself from the source:
[http://dri.freedesktop.org/wiki/Building#head-18dc229f35ef85098e2f2d76427eca910221ebcf](http://dri.freedesktop.org/wiki/Building#head-18dc229f35ef85098e2f2d76427eca910221ebcf)

I am using edgy but I compiled MESA 6.5.2 and it works fine.

Those are some of my ideas so far.
