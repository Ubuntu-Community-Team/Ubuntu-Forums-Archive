### Post by pxwpxw on 2007-05-21
**rccharles**

Your script says /dev/hda11 is the bootblock.
The partition table should show this as Apple_Bootstrap type for a NewWorld bootblock, and
the size needed is only order of 1 Megabyte, certainly not 2.0G, so something has gone wrong,  it would be reasonable to have hda12 (1.4M) as the bootblock. 
 
Like this:

/dev/hda12        Apple_Bootstrap        2835 @ 80019765 (  1.4M)    NewWorld bootblock

The Apple_Bootstrap type for the bootblock is normally set by the install partitioner, and before the yaboot install step, so a reinstall should fix everything. 

But if thats not an option, it could be set up using mac-fdisk from a rescue cd or live cd, or by running the partitioner again, without formatting the other partitions ( hazardous).

The bootblock would then need to be formatted hfs and configured, using yabootconfig.
