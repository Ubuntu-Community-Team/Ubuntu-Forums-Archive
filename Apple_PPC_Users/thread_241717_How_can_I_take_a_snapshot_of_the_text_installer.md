### Post by Herman on 2006-08-23
Hello, kulath,
The way I do things is very simple, but a little time consuming. I have two  computers beside each other and I perform and install (or other process to be illustrated), in one computer and make my images in the other computer beside it to resemble as closely as I can, the images shown in the real install.

I used to use Microsoft 'Paint' at first, to make bitmap images and then convert them to .gif format with GIMP in Ubuntu. A .gif image is around 1/20 th the number of kB for the same image, so they save a lot of disk space on my ISP's sever. I'm allowed 10240KB total, of which I'm currently using about 64%. I have 16 web pages. If the webpages are lighter, they load quicker in people's browsers too. At first I wasn't fast enough with GIMP to make them all 'from scratch' with GIMP, and also because for some reason they turned out to be the a smaller filesize (in Kb.) that way, (when made in 'Paint' first), for some mysterious reason.  

When I learned about NVU, I switched over to that website editing software in Ubuntu for making all my web pages. I don't use Windows anymore for anything now, (except maybe as a guinea pig for showing how to dual boot with it). With NVU I was able to use .png format for images. .png format is excellent quality and even lighter in kB than .gif images. Ubuntu screen caps are .png format by default.  For a while I made them all with GIMP,  and made all except the animated images in .png format, the animated ones are .gif format. Most of my current web pages now have mostly .png images, and just a few .gifs.

My latest method that I am trying and still learning more about, is not to make images at all, but make 'tables' with NVU instead. Those, I think, are even lighter than image files and they are a lot quicker and easier to make too. The  Xserver page and the new Super Grub Disk Page have just 'tables' and no real images at all. If you look at them, you can see the table's borders. I do know now how to get rid of those, and I will update those webpages soon with invisible table borders.
Another big advantage of using tables is where people might want to copy code off the website, say for example from the menu.lst file. It is possible for people with their Windows entry missing from their menu.lst, for example, to copy the entry of mine from the website and paste it into their own file and modify it to suit their needs. That wouldn't be possible if I was still using image files.

So there's no magic about it at all really, it's just a little work. Well, I enjoy it though, it's a rewarding hobby, especially if and when it helps people.

All the best, kulath, good luck with your new website,
Regards, Herman :D
