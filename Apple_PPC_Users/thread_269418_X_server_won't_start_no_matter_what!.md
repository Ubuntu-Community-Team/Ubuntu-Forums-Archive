### Post by k49 on 2006-10-03
Here's what I did:

I wanted to boot Ubuntu by default instead of Mac OS X (I have both installed, and for some reason Mac OS X was default), so I looked at /etc/yaboot.conf. I couldn't figure out how to make Linux the default, but I saw a line that said append="quiet splash video=ofonly." I remembered that in order to see anything when booting from the Live CD, I had to type "live video=ofonly" at the prompt, and I knew this had somethng to do with video. So I tried commenting out this line with a #.

I ran "sudo ybin" to do its thing (I don't understand how it works) to the yaboot.conf file, and restarted. This time, for no apparent reason, Linux started by default, and the desktop appeared. (Actually, there was no video, so I had to go back and fix /etc/X11/xorg.conf again, but after that it was fine.)

I don't know how I fixed it, but it doesn't matter so long as it stays fixed. Thanks for the suggestions.
