### Post by stmiller on 2007-08-02
PowerPC Linux **IS** being 'kept up' as well as x86 Linux. The same packages, same updates, same everything that the x86 and 64bit Ubuntu users download are available in PowerPC form.

There is also PowerPC Linux from the other major Linux distros:

Suse
Fedora
Gentoo
Debian

If you search any kernel changelog, you'd see that quite a lot of work being done by kernel maintainers is work on PowerPC. 

Search for 'powerpc' with your browser on this page, for instance:

[http://git.kernel.org/?p=linux/kernel/git/torvalds/linux-2.6.git;a=log;h=v2.6.23-rc1](http://git.kernel.org/?p=linux/kernel/git/torvalds/linux-2.6.git;a=log;h=v2.6.23-rc1)

Most people aren't aware of this.


As far as skype and similar things: it's a chicken and egg thing. So the more users we have demanding skype, then it will come. The company has stated this on their forum.

Xbox 360, Wii, PS3, Cell processor, machines from IBM, etc. all are derivatives of the POWER arch. The POWER arch is not going away, but rather growing. The architecture does not live/die by Apple. IBM just announced new POWER6 cpus a week ago. You can browse the IBM website and see the Power machines they offer serving companies and clients worldwide. If anything the architecture needs Linux more and more...

### Post by calinb on 2007-08-11
I have Ubuntu 7.04 running on my PPC Mac mini.  I also have Ubuntu, Fedora, and Gentoo running on my x86 PCs.  My PPC Mac works just as well as the x86 PCs for non-CPU intensive use and it makes a lot less noise and uses a lot less electrical power than the others.   I recently set up the PPC Mac as my 24/7 household server because I was tired of the high power consumption of my previous x86  server.   For additional storage,  I setup an external USB enclosure with four drives and RAID-5.  I only  connect it and start the RAID when I need access to the files (more noise and power).  

Thanks to these forums, I have my wireless Apple keyboard and mouse and Airport wireless working.  I run the Shorewall firewall and my Mac provides an effective bridge and router between my wireless internet connection and my wired home ethernet.  The only things that do not work on my Mac are G-band wireless and the internal modem.  (Blame the hardware vendor).  My cable modem internet connection doesn't support more than B-band speeds anyway so I don't care about G-band.

I also have an OS X Tiger image on my disk and I occasionally run it virtually using Mac on Linux (MOL).  MOL is a bit tougher to setup than Ubuntu, but it's easy to boot Mac OS from an external firewire drive too, but I rarely have any desire to do so.
