### Post by frog_pilot on 2008-01-30
thats what I found:
[http://penguinppc.org/bootloaders/yaboot/doc/yaboot-howto.shtml/ch6.en.shtml#s6.2]("http://penguinppc.org/bootloaders/yaboot/doc/yaboot-howto.shtml/ch6.en.shtml#s6.2")

A utility for finding the OpenFirmware device path corresponding to a given Linux device path is provided: ofpath. ybin uses ofpath internally to convert Linux device names you use in yaboot.conf to OpenFirmware equivalents. Example: ofpath /dev/hda (should return hd:). You can also figure out OpenFirmware device paths yourself, see Recovering From Misconfiguration below.

e.g. type in a terminal under linux```
ofpath /dev/sda2
``` according to your setting, where sda2 is the newworld bootblock and verify the output with the of path in your yaboot.conf

### Post by frog_pilot on 2008-02-01
its a good choice. iwondered if you have scsi-drives because of the sda-things in your mac-fdisc  -l posting. 

please tell something about your hardware configuration, especially about the hds.

the solution in the linked thread was to mount the newworld boot partition in the live system and to get the ofpath with the command 

```
cat /mnt/ofboot.b
``` (if the partition is mounted to /mnt/)

After getting the new ofpath, you would have to chroot to the installed system from the live-system, to edit yaboot.conf and to execute sudo ybin to update the newword boot partition.
