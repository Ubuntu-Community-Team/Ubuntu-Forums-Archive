### Post by fkdev on 2007-03-29
If I understand right I believe that the current situation is as such:
Rage 128 VR crash with GL apps even with ForcePCIMode        
Rage 128 Pro Ultra and other cards seem to work with ForcePCIMode.
So: If your dri doesn't work, make the following edits to /etc/X11/xorg.conf
- Add this to the device section
```
 Option "ForcePCIMode" "true" 
```
- And if your /var/log/Xorg.0.log has something like: 
```
 static buffer allocation failed. 
```
then modify this in xorg.conf, in the Display section, such that this line:
```
 DefaultDepth 24 
```
becomes (or is added if you don't have it) 
```
 DefaultDepth 16 
```
- In addition, make sure this line is NOT in your xorg.conf
```
 Option "EnablePageFlip" "true" 
```


If GL apps freeze, and your lspci says something about a Rage 128 VR, then you'll have to wait for a fix.
If someone has a Rage 128 VR and it works, then I would be highly interested.
