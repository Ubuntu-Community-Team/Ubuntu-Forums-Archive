### Post by HandyAndyShortStack on 2007-10-27
Last week I got a Powerbook G3 Lombard 333 for free.  At first, I thought it could be turned into really fun little xubuntu machine for homework and web browsing.  Unfortunately, xubuntu does not want to install on this system without a fight.   
First of all, live CDs are out of the question: though I was sucessful in getting into Xfce a couple of times, it ran very slowly and crashed soon.
Each time I try the alternate installer, the screen turns red near the end of the "install base system" step and I am faced with with the ominous message "unable to install selected kernel."  This happens to me using the Dapper, Edgy, and Feisty (Gutsy doesn't seem to want to boot).  It doesn't care which kernel I use or how I partition the drive.  
After some googling, I have come to suspect that this has something to do with my CDROM drive and the installer not getting along.  I suppose the next thing I should try is figuring out how to do a network install with the help of my desktop, but this may be more work than I have time for.  
Is there any way I can boot the installer from the CDROM drive, but install the kernel from a different source?  Could a new CDROM drive clear up this problem?  A new hard drive?  Has anyone had a similar problem?

### Post by HandyAndyShortStack on 2007-11-01
Well, I feel like an idiot.  For anyone experiencing similar difficulties, BURN AT 1X!!!  4X doesn't cut it with these old drives.  I burnt at 1X and it solved all my problems.  Now I have the Xubuntu laptop I wanted.  *ride into sunset*
