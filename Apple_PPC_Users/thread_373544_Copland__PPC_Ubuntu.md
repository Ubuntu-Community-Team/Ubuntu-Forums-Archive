### Post by Dave / Mosaic on 2007-03-01
Picking up a new thread on a related topic...:

> **3rdalbum said:**
> As an aside, there is a Mac-specific Linux distribution that is currently being developed called Copland - yes, this is my own project :-)   When it's released I'll post the announcement to this forum, so stay tuned! I'm sure Debian will be a better choice at the moment, due to stability, but before long I'm hoping that Copland will become a good choice due to better features, and concentration on what Mac users want and need.

You know what _I'd_ like to see...

A Linux desktop environment like MacOS has.  By which I mean, that _really_works_...

I haven't actually used OSX, so I don't know what that is like; I run a couple of older generation Macs here, with OS9.x...  The Macintosh OS and desktop experience up through the OS9's were really so elegant and functional and beautiful.  (I suppose OSX must be the same way.)  Especially in the very old historical Macs.  It didn't need processors running at multiple gigahertz in order to be snappy and responsive...  It was easy to understand even for normal real people...  You could keep things organized in nice folders, and they didn't get all messed up all the time...  Installing software or system extensions was easy and simple to understand, with the OS keeping accurate track of the nuts and bolts behind the scenes...

It seems to me, that for that to work in Linux, you need to have the whole OS based around a common data repository like the MacOS desktop.  Being new to Linux, I'm not sure what equivalence there might be with anything in modern Linux...  Also, the file structure with data and resource forks, that helped a lot of stuff to work really nicely, too...

Windows 98, XP, etc., with the registry and all, tried to duplicate the experience; the result is okay in ways, and certainly usable by lots of people, but it's also a huge nasty mess internally, with the sharp edges of the mess always poking up through the facade and annoying you...

I've just been using Linux now for a couple of weeks, after being away from Unix world for over ten years...  From what I see of Linux desktops so far, the "desktop" experience is still a pretty poor approximation of what it could be.  There are things that _look_ like folders on the "desktop", and you can click on them to browse them, or run things, but it isn't the same...  Anybody who hasn't actually used a Mac, and only knows Windows, would not understand...  Now there are things like Beryl, with all fancy 3D graphics, etc., and hundreds of different Themes to choose from to control how your scrollbars work and such, but still it's based on the same underlying inadequate structure, and it doesn't really _work_...

Anyway, just a tirade here...

Still I like Ubuntu...

### Post by ssam on 2007-03-02
if you want mac os classic style folders, then you might want to turn on spatial mode in nautilus (the gnome file browser).

open up a folder. go to Edit -> Preferences. on the behaviour tab untick 'always open in browser windows'. now close that folder. next time you open a folder it will be in spatial mode.

for installing new applications
Applications -> add/remove
is about as simple as it can get. the package management system is inherited from debian, and is very robust. not all the applications are perfect (some are definatly not very polished), but for most tasks there is a good one.

perfect hardware detection is very difficult, especially as the ubuntu developers do not even have access to every mac ever. also without cooperation from all the hardware manufacturers it is hard to write drivers.
