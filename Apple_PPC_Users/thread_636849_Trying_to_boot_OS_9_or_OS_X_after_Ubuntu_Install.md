### Post by guidop on 2007-12-11
Assuming your hardware is working and you have a "New World" Mac (roughly 1998 or later):

Boot into Ubuntu and insert the CD or DVD you want to boot from.

Shut down with the disk still in the drive.

Start up while holding down the "option" key 
  - Open firmware should scan for all attached bootable disks and partitions.
  - Wait for the scan process to finish, and select the CD/DVD you want to boot from.



If the above doesn't work, check your hardware:

- Can you boot into Ubuntu and play or mount/read a CD, either a music CD or perhaps the Ubuntu CD you installed from?
 -- If you are burning a new install CD, burn it at the slowest speed possible.


- If you are trying to boot from a DVD, can you mount or play a DVD?

### Post by guidop on 2007-12-15
As far as I know, your /etc/yaboot.conf file looks correct, and matches your partitioning scheme, 
assuming you want only linux on this hard drive, 

EXCEPT the second of these two lines confuses me:

boot=/dev/hda2  # This looks correct, as it points to the apple boot partition
boot=/dev/hda12 # This points to a non-existent partition !? ? Delete this line ?

What does running:  sudo ybin -v output?


Still, it seems really strange that you can't boot into open firmware and then boot a CD.

If you're feeling brave, and once you verify that a verbose ybin does not give errors, and did not cure your CD boot problem, you could try resetting your PRAM, by holding down the four keys:  command-option-p-r   at power on.  I THINK this will still have you default to normal booting from partition 2, but may clear some strange configuration that's preventing you from booting from a CD.  I worry, however, that if something really unusual is wrong, it may render the machine completely unbootable.
