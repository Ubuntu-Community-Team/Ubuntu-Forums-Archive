I really do not think that people should think of Ubuntu in isolation. There are many other distros out there, and that is a great thing. Ubuntu is a mighty beast at the moment, but that can be problematic, for example some people here have complained about Yellow Dog not being the most up-to-date or having a small community. There is one reason for this: Fedora supports PowerPC. While Fedora was x86-only there was a lot of support for Yellow Dog because it was a Redhat/Fedora-based distro which worked on PowerPC (IIRC Fedora's YUM is the Yellowdog Update Manager? That shows the level of support for Yellowdog, since it created a better package manager than up2date, and Redhat eventually switched to using it.) Then, of course, Fedora started supporting PowerPC and a large number of Yellow Dog users went to that.

This is similar to the anti-M$ argument of vapourware, where people will wait for a Microsoft version because they see it as better, the market appears to dry up because everyone is waiting and not buying, then M$ decide not to release their product at all. I don't think this is the way forward for GNU/Linux distros, so maybe instead of people's complaints about Ubuntu's problems on PowerPC going unheard there should be a realisation that a PowerPC OS would HAVE to listen, as Yellow Dog had to listen to any PPC complaints from it's users.

Since the technology is freely redistributable then there are no worries about lack of support. At the end of the day, Apple PPC hardware has stagnated, and it was never Ubuntu's goal to bail out people who bought into proprietary hardware and got cut off by their manufacturer (maybe this should be seen as a lesson? Sorry, this isn't meant to sound spiteful, I like PPC a hell of a lot better than x86, but I hate Apple hardware which uses it). However, it IS Debian's goal to be a "universal operating system" (hence it supports so many architectures, and has experimented with HURD and some BSD kernels) and it would be a good thing if they got a large influx of PPC users to test and improve their packages (once again, people should be distro-independant, so Debian or Ubuntu doesn't really matter as the technology works for both)

I htink there is some confusion over PPC architectures in this thread though. Apple hardware doesn't really show any similarity with the PS3, and as I have commented in the Wiki I don't think Ubuntu should enter the PS3 arena because any smaller distros with a narrower focus (and therefore better support for their users, like Yellow Dog used to have) would get crushed because most people would ujust use Ubuntu. Once again, distro-dependancy should not ruin the diversity of distros that are out there, since there is no one-size-fits-all distro, so Ubuntu certainly is not one. I think getting GNU/Linux, and generally Free Software, receptive hardware into people's homes like the PS3 is great, and it's CELL/PowerPC base makes it even better since the completely ignorant arguments against FLOSS like "but all of my hardware works on Windows" is shown as it really is, that PC manufacturers make Windows-compliant drivers for their hardware, but Microsoft couldn't care less about hardware support since they still only use x86.

Basically my message is this: Stay distro-independant, then you won't care if Ubuntu supports PowerPC or not, since you can always switch to a distro which does a much better job. For example, I disagree with Ubuntu's default use of proprietary graphics card drivers when there are Free alternatives available, but rather than complain about it and point out all of the technical issues which have obviously been thought through by the developers I am saying one thing "OK, that is fine because it is your choice. I disagree with it, but that is OK since I can go back to Debian."

I think one of the things that has not been discussed in the latest part of this thread is the original raison d'être of Ubuntu - to push Debian to create a unified distribution (across the Debian supported platforms) that was updated on a regular basis and was easy for new users, while contributing to the entire community.

There has always been a certain amount of doubt among the Linux community regarding Mark Shuttleworth's motivation for starting Ubuntu and whether or not Ubuntu would be a) vaporware or b) a Trojan horse. Ubuntu has not however, improved the plight of PPC users that have always been step children at best within the Linux community - even before Apple's announced switch to Intel, despite high expectations from the Mac community.

The internal discussion within Ubuntu with regards to whether or not to continue to support the PPC platform looks to me more and more to be a cut and run maneuver where Canonical and Ubuntu run away with their tail between their legs at the first sign of difficulty. Yes, there have been hurdles to overcome with regards to the PPC platform, but all distros that support the PPC platform have had to overcome the same hurdles.

It does not bode well for the Ubuntu distro in general if rather than creating a PPC version that is on an equal par with the x86 version, the Ubuntu development team throws in the towel at the first sign of adversity. Is Ubuntu incapable of completing the mission that they originally set out to accomplish? Are there more serious difficulties at Ubuntu that the proposal of dropping the PPC platform reflect?

### Post by Decade on 2006-11-21
Don't stop using Ubuntu because Canonical might decide to stop supporting PPC. It's currently supported, and that's what matters in the short term. Canonical might decide to keep PPC. It's also in the Dapper LTS, so we will have a probably-better-than-Debian stable for a while.

In the medium term, the community won't simply disappear. If I remember correctly, Fedora PPC came about in the first place because of volunteers. A dedicated group of volunteers would probably emerge to support PPC for a while after Canonical leaves.

Now, about Gentoo. Its unique property is the Portage system. It's basically a large number of scripts to download, compile, and install packages. It can also install binary packages, but they're typically not available except for the set on the install CD. The issue is that in ordinary distributions such as Ubuntu, the maintainer makes decisions about which options are enabled, and makes separately named packages for different options. In Gentoo, the maintainer discovers what options are available and practical, and exposes them to the user, and the user decides whether to stray from the default. Then "emerge" builds a package according to the preferences, optionally making a binary package. The package's name has no hints as to the architecture or options.

In a binary distribution, the deb mirror holds a compressed list of available packages and their details, which the client uses to decide which binary packages to download. In a source distribution, the mirror and the client synchronize their install scripts, which take a lot more space than the available package list, and then the client downloads the source, which takes somewhat more bandwidth than the binary package, and so on. I'd guess compiling takes roughly 10 to 1000 times as long as binary packages, usually, depending on Internet connection and processor speed.

The Gentoo installer is one of the hardest I've seen, but that's in the sense of a high learning curve rather than braindead design. The Gentoo installer is more a set of instructions for the user than a specific computer program. A large amount of the knowledge you can gain from the Gentoo installer applies to any other Linux distribution, and if you know Linux fairly deeply then Gentoo is easy. The instructions are detailed, but in some places they require thinking, and sometimes they're slightly inaccurate. In addition, the Portage tree is not guaranteed to be in working condition at any time, but it breaks most often in the process of major updates, like point upgrades of KDE or GNOME, and it usually works. The install CD should almost always work.

In summary, Gentoo is now not a good substitute for Ubuntu.

> 09:49   jbailey I see we're close to the end, so probably best to dive in to PPC in general.
09:49   jbailey Earlier on:
09:49   jbailey <sid> QUESTION: Is PowerPC no longer and official build for Ubuntu?
09:49   jbailey <ma1kel> QUESTION: Ubuntu and PS3?
09:49   jbailey At this time, ppc is still an official build for Ubuntu.
09:50   jbailey A spec was raised at the Ubuntu Development summit in Mountain View, California with the proposal to drop support for the PowerPC architecture.
09:50   jbailey That is to say, move it to being a community supported architecture.
09:50   jbailey This is based on a few things:
09:50   jbailey 1) Low community participation in powerpc.
09:50   jbailey - Few install reports during the development cycle.
09:51   jbailey - Few people jumping up to look at bugs on the port, such as the R300 crashes.
09:51   jbailey 2) Low download rates.
09:51   jbailey - I don't have the numbers handy, but as a total percentage of people downloading it, ppc has fallen to half what it was a year ago.
09:51   jbailey 3) Difficulty in obtaining new hardware.
09:52   jbailey - PowerPC is certainly a strong and viable architecture, but with Apple not producing desktop boxes anymore, there will be fewer systems out there in developers hands.
09:53   jbailey - New embedded systems tend to be mips and arm, rather than ppc.  Unless you happen to be porting to a Cisco Router. =)
09:53   jbailey The spec isn't finalised, and is sort of in a limbo stage right now.
09:53   jbailey I don't know off hand which way the decision will go.
09:53   jbailey But the Sony Play Station 3 uses a variant of the Ppc chip known as the Cell processor.
09:53   jbailey For those who don't know, the cell processor is amazing pile of crack just waiting to be smoked.
09:54   jbailey You have a ppc main chip, and I think 8 DSPs on board.
09:54   jbailey So the main chip can copy things into private address space for the DSPs to deal with and return values.
09:54   jbailey It's an interesting hack in that the DSPs don't speak powerpc assmebly.
09:54   jbailey And the DSPs also don't have access to main ram, or the cache.

### Post by Torrance on 2006-12-05
> Sorry but I have to admit I am a little confused by this. Every linux distro, whether community-based or business run, has some commercial aspect to it. In most cases the "profitability" aspect comes from professional support and I don't see Ubuntu or Debian any different in this regard.

Well yes and no. Yes, in both cases any profit derived from these distros is from support and services, not the software itself. The difference is, however, that Canonical is both the developer of Ubuntu and the primary support for it. Their development is motivated by (eventually) achieving profitability through providing support for their free product. Now, if PPC desktop is not going to be big enough to generate income through support and services, then there is no drive for them to develop the distro itself.

OTOH, Debian (and Gentoo) are entirely separate from those that provide services for these distros. Whether or not services and support are profitable for these distros is not their concern. This is the pragmatic advantage of community-based distros - that they can cater for non-profitable areas - as well as the ethical component of free software made freely by volunteers (mutual aid, free communism, etc.).

With this in mind, from a shrewd business perspective (is business ever not shrewd??) it makes sense for Canonical to dump PPC desktop as that is not an area that they will likely be making money simply from services. The PPC server market however, which traditionally are more likely to pay for servcies anyhow, makes more sense to continue as there is ongoing hardware production (ie. IBM, etc.).

If you aren't interested in editing text files in nano to initially get a system set up, then it might not be for you. For instance, you have to create an /etc/fstab, /etc/make.conf and other files, though there are excellent ppc examples provided that you can simply alter (which is what I did). Xorg.conf however, is configured automagically with a gentoo configurator. 
 And there is a wonderful online install guide to hold your hand through the entire process (which I strongly needed, and had to constantly go back to read through). There is no 'installer.' You are the installer. heh! 

But it is quite rewarding, as you have a system that you configured. And fast. There are no odd processes running which eat your memory which you do not need. 
(For example, the stock Ubuntu kernel loads PCMCIA and RAID modules for everyone, among others, if you need them or not! Eating your memory. Of course you can disable these if you know how.)

And once everything is configured in Gentoo, there is a GUI program (Kuroo) to handle all installation of packages, if you do not care for the command line. 

The most critical thing for Gentoo is setting the USE flag correctly in /etc/make.conf. I emerged VLC, but found it was a command line only version. There was no VLC gui. It seems I didn't list the required Wxwidget USE flags when emerging VLC. So I had to emerge it again with the correct USE flags. ...and so forth... So there is a bit to learn about setting the USE flags, but there is lots of help out there. And once you get a good set of Use flags made up, you can basically leave them unaltered.


Debian is also no walk in the park to set up though, and is probably similar in some ways to setting up Gentoo. I always thought of Debian as the most advanced Linux distro out there. Others may disagree though. I have not used Debian, so I can only comment from my friend (who has set up Debian on many servers, and says it's the most rock solid version of Linux out there). Yes, Wine basically duplicates the minimum necessary functions of Windows to allow a Windows app to run in another OS on an x86 processor without having to run the whole Windows OS. It's never run on any PPC processor, and couldn't. The entirely unrelated Virtual PC for Mac, and similar efforts, were required to run Windows on a PPC, and did so even at best rather poorly, since Windows is of course entirely designed to run on x86.
