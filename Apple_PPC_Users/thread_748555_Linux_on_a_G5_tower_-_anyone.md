```
boot=/dev/sda2
device=/ht@0,f2000000/pci@9/k2-sata-root@c/k2-sata@0/disk@0
partition=3

timeout=50

image=/boot/vmlinux
               label=Linux
               read-only
               initrd=/boot/initrd.img
               append="quiet splash"

image=/boot/vmlinux.old
               label=old
               read-only
               initrd=/boot/initrd.img.old
               append="quiet splash"
```

### Post by slacka-vt on 2008-04-08
Sorry for the delay. I've been away for a few days.
I installed Ubuntu Hardy on my PowerMac G5 Dual 2.7Ghz tower over a month ago.
It runs beautifully !
In fact, I don't even boot into MacOS anymore.
I tried installing YDL, Debian, Slackintosh. None of them worked.
here's my yaboot.conf

```
## yaboot.conf generated by the Ubuntu installer
##
## run: "man yaboot.conf" for details. Do not make changes until you have!!
## see also: /usr/share/doc/yaboot/examples for example configurations.
##
## For a dual-boot menu, add one or more of:
## bsd=/dev/hdaX, macos=/dev/hdaY, macosx=/dev/hdaZ

boot=/dev/sda2
device=/ht@0,f2000000/pci@7/k2-sata-root@c/k2-sata@0/disk@0:
partition=4
root=/dev/sda4
timeout=100
install=/usr/lib/yaboot/yaboot
magicboot=/usr/lib/yaboot/ofboot
enablecdboot
macosx=/dev/sda3

image=/boot/vmlinux
        label=Linux
        read-only
        initrd=/boot/initrd.img
        append="quiet splash"

image=/boot/vmlinux.old
        label=old
        read-only
        initrd=/boot/initrd.img.old
        append="quiet splash"


```

~s

### Post by stream303 on 2008-07-17
One thing you might try is to see if there are any runaway processes hogging the cpu with top, or better yet, download htop from your repo and see what it reports.

Check this out:  you can also use Intel's PowerTop utility, also downloadable from most normal distro repos - it works on PPC with some limitations, but might expose something else, like runaway interrupts and things you normally might not see when running the standard top.  Having that utility from Intel work to a limited extent on PPC blew my mind.

It may not solve your problem, but could point you in a direction to follow if you like to compile your own kernels, or are active in kernel testing.

I guess the catch-22 is that the system needs to stay on long enough for Powertop to actually catch the data - takes about 5-10 seconds.. :)
