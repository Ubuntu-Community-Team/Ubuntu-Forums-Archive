### Post by nattugglan on 2007-04-29
I found the solution to this problem a while back.
The only thing that needs to be done to get the correct screen resolution in Ubuntu 6.10 is to edit xorg.conf.

```
$ sudo gedit /etc/X11/xorg.conf
```

Find the *Monitor* section in xorg.conf.
```
Section "Monitor"
	Identifier	"Q17 Digital"
	Option		"DPMS"
	HorizSync	30-65
	VertRefresh	50-75
EndSection
```
	
Change HorizSync to 28-70
Change VertRefresh to 43-60

Find the *Screen* section in xorg.conf (just below the *Monitor* section).
```
Section "Screen"
	Identifier	"Default Screen"
	Device		"ATI Technologies, Inc. RV280 [Radeon 9200]"
	Monitor		"Q17 Digital"
	DefaultDepth	24
	SubSection "Display"
		Depth		1
		Modes		"1280x1024" "1024x768" "800x600" "640x480"
	EndSubSection
	SubSection "Display"
		Depth		4
		Modes		"1280x1024" "1024x768" "800x600" "640x480"
	EndSubSection
	SubSection "Display"
		Depth		8
		Modes		"1280x1024" "1024x768" "800x600" "640x480"
	EndSubSection
	SubSection "Display"
		Depth		15
		Modes		"1280x1024" "1024x768" "800x600" "640x480"
	EndSubSection
	SubSection "Display"
		Depth		16
		Modes		"1280x1024" "1024x768" "800x600" "640x480"
	EndSubSection
	SubSection "Display"
		Depth		24
		Modes		"1280x1024" "1024x768" "800x600" "640x480"
	EndSubSection
EndSection
```
Make sure that the screen resolution(s) you want to use (i.e. the native screen resolution of the LCD-screen) is on the *Modes* line in each *Display* subsection as shown above. In my example (a 17" LCD) the screen resolution I want to use is the native 1280x1024 screen resolution of the LCD. If the screen resolution you want to use is missing, simply add it to each *Modes* line.

X needs to be restarted for the changes to take effect. To do this you can i.e. log out, and log back in.
The new screen resolution(s) will now be available in *System: Preferences: Screen Resolution*.

I have not tested Ubuntu 7.04 Feisty Fawn on a Mac mini G4 yet, but on all i386 systems I've tried so far the screen resolution has been correct out-of-the-box with Ubuntu 7.04.

nattugglan
