### Post by kellogs on 2006-06-06
if you've got an ATI card 9200 or less, then you are lucky, opensource drivers (xorg-drivers-ati) work for 3d and 2d. if you have something a bit higher (as in my case , I have a 9600) then ATI does not make fglrx drivers for ppc and 3d acceleration is not yet working for the >9200 cards.

there is a project with DRI/DRM at X.org to enable those (r300) working, but it haven't been included yet in latest OS Xorg drivers. I have seen some people in other distros made them work for pcc using CVS versions of very recent x drivers, but that is above my skill.

### Post by stmiller on 2006-08-14
You can comment out all of these lines:

```
Section "InputDevice"
  Driver        "wacom"
  Identifier    "stylus"
  Option        "Device"        "/dev/wacom"          # Change to 
                                                      # /dev/input/event
                                                      # for USB
  Option        "Type"          "stylus"
  Option        "ForceDevice"   "ISDV4"               # Tablet PC ONLY
EndSection

Section "InputDevice"
  Driver        "wacom"
  Identifier    "eraser"
  Option        "Device"        "/dev/wacom"          # Change to 
                                                      # /dev/input/event
                                                      # for USB
  Option        "Type"          "eraser"
  Option        "ForceDevice"   "ISDV4"               # Tablet PC ONLY
EndSection

Section "InputDevice"
  Driver        "wacom"
  Identifier    "cursor"
  Option        "Device"        "/dev/wacom"          # Change to 
                                                      # /dev/input/event
                                                      # for USB
  Option        "Type"          "cursor"
  Option        "ForceDevice"   "ISDV4"               # Tablet PC ONLY
EndSection
```



```
InputDevice     "stylus" "SendCoreEvents"
	InputDevice     "cursor" "SendCoreEvents"
	InputDevice     "eraser" "SendCoreEvents"
```

And your computer will be much happier. 
Do:

$ sudo bash

$ less /var/log/X(something)

to see your xorg log. It will most likely be trying to load all of those non-existant things every time. I'm not sure why ubuntu put all of those in.
