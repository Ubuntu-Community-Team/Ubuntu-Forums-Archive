### Post by didg on 2007-01-10
I've looked at it a little more and in my understanting it's when you're using video=ofonly
cf. /etc/yaboot.conf or /var/log/syslog for  open firmware init. 

with ofonly the framebuffer device driver doesn't map mmio and returns 0 ==> SIGSEV.

Workaround 1: 
in /etc/X11/xorg.conf 
set
Option          "UseFBDev"              "true"
to false, note in this case switching VT is broken and locks the box.

Workaround 2: 
use 
Driver          "fbdev"
rather than "ati" 
If you really need ofonly
You may have to set
 Option "ShadowFB" "false"
When it is set to true the screen is mangled for me (a pb when the stride is not the same as the line length, 1024 for 800 in my case).
and remove all 
        SubSection "Display"
in "Screen"
and  DefaultDepth    
if any.

Here ofonly uses 800x600 256 colors only.

If the workaround 1 works for you then you don't need ofonly, remove it from /etc/yaboot.conf an rerun ybin or mount the bootstrap partition and change yaboot.conf there.

### Post by didg on 2007-02-19
For info what's in you /etc/yaboot.conf and does VT switching (CTRL ALT+F1) works for you?

For the iMac you'll get the black screen...

The problem is that Xorg can't get the monitor info with DDC and don't know about the open firmware tree so it uses xorg.conf resolution which is set to 1024x768 at 60Hz,  an invalid one for iMacs. 

Xorg.conf resolution is wrong because the liveCD calls dpkg-reconfigure in a chroot without the /dev/fb0 entry and xresprobe fails. 

So you need to rerun  dpkg-reconfigure later when the right /dev is mounted.
