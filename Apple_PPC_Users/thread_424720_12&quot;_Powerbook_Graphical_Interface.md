### Post by ssam on 2007-04-27
sometimes reconfiguring xorg can resolve problems like this. this may save you from manually having to edit the xorg.conf file.

try going to the terminal (press ctrl+alt+F1 if you are not already there)

log in

run

```
sudo dpkg-reconfigure -phigh xserver-xorg
```

(the sudo it to run the command with admin privileges, dpkg-reconfigure is the comand to reconfigure a package, -phigh make it only ask high priority questions, xserver-xorg is the package name)

it may ask you a few questions, if you are not sure how to answer them ask here.

once this is done you can either reboot

```
sudo reboot
```

or restart the login screen

```
sudo /etc/init.d/gdm restart
```

### Post by APU on 2007-04-27
Hi!

All 12" PowerBooks have Nvidia Graphics and therefore use the "nv" driver.

I had similar problems very recently when I installed Debian "etch" on my PowerBook 12" and fiddled around for quite a while to get everything working. But fear not, here is my xorg.conf and you can most likely use it without many problems. Just replace your /etc/X11/xorg.conf file after you have backed up the original one and restart your system.

I also used "dpkg-reconfigure xserver-xorg" to obtain the base configuration from where I made the current one. Normally it is sufficient to always choose the default configuration when you walk through the dialogs (always pressing Return on your way)

```

# This file was generated by dexconf, the Debian X Configuration tool, using
# values from the debconf database.
#
# Edit this file with caution, and see the /etc/X11/xorg.conf manual page.
# (Type "man /etc/X11/xorg.conf" at the shell prompt.)
#
# This file is automatically updated on xserver-xorg package upgrades *only*
# if it has not been modified since the last upgrade of the xserver-xorg
# package.
#
# If you have edited this file but would like it to be automatically updated
# again, run the following command:
#   sudo dpkg-reconfigure -phigh xserver-xorg

Section "Files"
        FontPath        "/usr/share/fonts/X11/misc"
        FontPath        "/usr/X11R6/lib/X11/fonts/misc"
        FontPath        "/usr/share/fonts/X11/cyrillic"
        FontPath        "/usr/X11R6/lib/X11/fonts/cyrillic"
        FontPath        "/usr/share/fonts/X11/100dpi/:unscaled"
```

An additional hint. I have just a very basic trackpad config in my xorg.conf file and use qsynaptics to finetune most of the settings. Install the "qsynaptics" package by using "apt-get install qsynaptics" from the terminal (or any installer gui frontend of your choice).
Then run "qsynaptics", configue your trackpad, tapping, scrolling and everything.

The only thing is that you have to run "qsynaptics -r" again every time after you log into your system but you can easily put this into your startup items too.
