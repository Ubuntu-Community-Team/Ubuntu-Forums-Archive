### Post by xurios on 2006-06-07
> ...but i do not understand a thing about nvram or booting ram in macs.

Oh, I didn't realize. When you mentioned the OF path of the drive, I assumed you had messed with it a bit and had some experience with it. No worries. Anyway, OF stores a lot of info about your system in the Non Volatile RAM (NVRAM), regarding basic things about your system and what it should do during boot, etc (like BIOS on x86). One of the settings, boot-device, tells it what partition to boot from. You can list it in termnal (OS X) by typing 
```
nvram -p
```
You can also list it from ubuntu, but I don't remember the command. It's not hard to find it using google or a forum search.

yaboot writes to nvram based on what's in yaboot.conf. So you can piece together what should be in boot-device by looking at the yaboot.conf file and manually setting it from OS X. Pressing option during boot on my machine resets boot-device to the apple boot partition, and then it no longer boots into yaboot (which means no choice between linux and OS X). To get yaboot back, you have to set boot-device to the yaboot partition, which is why I wanted to know what yours was set to. I am using a powerbook g4, which uses ata and not sata, so I wanted to see your boot-device in nvram and compare to mine, in the hope that we could piece together what should be in there on swartzfeger's machine. But of course we can only trust swartzfeger's yaboot.conf if we know that it has successfully booted ubuntu at least once.
