### Post by joninkrakow on 2007-07-12
Here's what I did/do on my Pismo, which gives me the same problem. 

I learned the following from another post on this forum. I printed out the instructions and filed them in a folder with other useful tidbits for Ubuntu. :-) I highly recommend printing out the instructions.

When you get to the desktop, either launch a terminal, or hit <shift>-<alt>-F2. This should bring up a full-screen terminal. Into that terminal window (You may have to login, but if you are on the CD, a return at the password should do it) type "sudo dpkg-reconfigure xserver-xorg" without the quotation marks.

You can just keep hitting return through all the options, until you get to the screen resolution section (numbers like 1024x768 in a long column). On this screen, make sure you native resolution (1024x768?) has an asterisk next to it, and (use the cursor keys to select the space next to it, and space bar to toggle the setting on if it's off). Finish up to the very end. Hit <shift>-<alt>-F7 to get back to the normal desktop if you called up a full-screen terminal, and then <control>-<alt>-Delete to reboot X, and you should have full screen, and should now be able to see all of the install window to install. :-) 

BTW, my first install, I did it with the small screen, and it took several tries to find out how to blind-type my way through all the options. :-)

-Jon

If you have more than one computer on a network, a handy thing to do in this case is to do a server install on the Mac and/or make sure you install the ssh server. That way if the video is totally messed up you can ssh in from another computer and use dpkg-reconfigure xserver-xorg from there until you get the video working right.

On older Macs (the iBook G3 may or may not be affected) you have to pass a kernel parameter on bootup to get the video to be detected correctly the first time. But if you can get the xserver-xorg configuration working that's probably easier.

I never could configure the xorg.conf for my Powerbook Pismo correctly either.  Finally, I  believe it was talkeetnik that gave me his xorg.conf, which worked masterfully.

backup your xorg.conf
sudo  cp /etc/X11/xorg.conf /etc/X11/xorg.conf.screweddisplay

remove xorg.conf
sudo rm -R /etc/X11/xorg.conf

sudo nano /etc/X11/xorg.conf
