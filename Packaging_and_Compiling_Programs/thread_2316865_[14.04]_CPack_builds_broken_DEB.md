### Post by philip-lamb on 2017-05-29
We ran into the same problem and found a solution. Building with cpack produced a corrupted deb when we included doxygen-generated documentation. Actually, the cause is apparently that since CMake 3.4, it uses the 'pax' archiver, which can't handle long file names. The CMake bug tracking for the issue is here: [https://gitlab.kitware.com/cmake/cmake/issues/14332](https://gitlab.kitware.com/cmake/cmake/issues/14332)

The fix is to force CMake/CPack to use the 'gnutar' archiver. This option is available on CMake 3.7 and later via a build setting.

set(CPACK_DEBIAN_ARCHIVE_TYPE "gnutar")

See [https://cmake.org/cmake/help/v3.7/module/CPackDeb.html#variable:CPACK_DEBIAN_ARCHIVE_TYPE](https://cmake.org/cmake/help/v3.7/module/CPackDeb.html#variable:CPACK_DEBIAN_ARCHIVE_TYPE)
