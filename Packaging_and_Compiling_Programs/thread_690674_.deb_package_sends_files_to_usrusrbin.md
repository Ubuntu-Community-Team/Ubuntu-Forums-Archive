### Post by geraldm on 2008-02-08
If you configure with --prefix=/usr and make install using DESTDIR=/usr it would be possible to get equally wrong results.
debian uses debian/tmp directory, but the results must still be checked when making a package.

### Post by shirilover on 2008-02-09
> **Major_Kong said:**
> ? 
I've installed the package, it does create the files in /usr/usr.

Here's the "rules" :

Your problem lies in the following lines:
```

# Add here commands to install the package into debian/rubyripper.
$(MAKE) prefix=$(CURDIR)/debian/rubyripper/usr install

```

Change them to be(taken from the debian/rules file I used to build 0.4.4):
```

# Add here commands to install the package into debian/rubyripper.
$(MAKE) install DESTDIR=$(CURDIR)/debian/rubyripper

```
