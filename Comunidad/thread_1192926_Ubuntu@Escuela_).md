**[CENTER][SIZE="5"][SIZE="5"][COLOR="Sienna"]PCs para Escuela: instalando Ubuntu+Customizacion[/COLOR][/SIZE][/SIZE][/CENTER]**

Esta es un relato de lo que estoy haciendo actualmente,
es en una Escuela local muy grande, desde Jardin a Polimodal, yo curse alli,
sucede que el gobierno habia entregado 20 PC HP Minitower nuevos, 
muy lindos equipos, muy completos, venian creo sin SO o con un FreeDOS, 
para instalarlos varias personas han pasado por ellos, 
*pseudo*Tecnicos, el *chePibe* que piensa que por tener un Desatendido ya sabe todo,
haciendo desastres, mucha plata invertida y luego perdida,
pero a la actualidad, estaban desde hace casi 1 año los equipos parados,
algunos no booteaban *(ni se que les habran querido instalar)*, 
otros con un Mandriva que no terminaba de llegar al escritorio que se reiniciaba solo,
otros sin Sistema Operativo o la particion arruinada *(no lo se, no investigue mucho tampoco)*,
otros con Desatentido y la salsa de siempre de BSOD, Virus, falta el archivo de booteo pepe.sys, etc

Habia hardware necesario para tener una red, pero no habia servidor, y la redes solas no andan.
*Y si no hay una red operativa que sentido tiene unirse a Internet no...?*

Para solucionar el tema, se reviso los equipos y la instalacion del lugar,
se comenzo con diseñar como seria la PC Desktop para el alumno, 
y tambien como seria un Servidor que haria las veces de PC para el Profesor,
usando todas las herramientas Libres disponibles, y borrando los truchos,
se instalo Ubuntu 9.04, 
particion EXT4 de sistema, particion SWAP para paginacion, 
particion EXT3 para Recovery conteniendo imagen Clonezilla de la PC como *0Km*.,
se instalo updates y todos los programas en la categoria *"Educacion" *del servidor de Ubuntu.

El servidor tiene DHCP, DNS, Ruteo basico, Interfaz de control Web,
Proxy de Navegacion Segura para los alumnos, 
Sistema de control de las PC de los alumnos en tiempo real,
Sistema de Educacion a Distancia mediante una Web.
*Eventualmente preparado para futuras expansiones...*


*bueno basta de palabrerio, vamos a lo que a la gente le gusta...*

La libertad, en un Sistema Operativo, pasa por: tener acceso al código fuente, tener derecho a copiar y modificar, y por lo tanto, personalizar y adaptar el sistema a los gustos, necesidades, estrategias, etc., de cada usuario o institución que emplee dicho sistema.

Esto incluye, correr software privativo en el sistema en cuestión. No sería libre un sistema que prohíbe que determinado software, por ser privativo no pueda ser empleado en él.
