The problem is that the current strategy of Canonical is to ship UfA on new devices already installed (by cooperating wth OEMs in the device development stage). They wouldn't like people hacking this onto their devices like the chrooted Ubuntus.

I believe this is not a very good strategy because it points to a direction of Apple's walled garden approach and is completely in dischord with Linux and free software principles (which is: "take it and use it the way you want").

It seems that Canonical is working very actively on UfA for mobile phones which have very similar hw to the Transformer Prime and other ARM tablets and netbooks.

I hope they change direction and release UfA to the public as soon as possible. This would allow hackers to create install methods for the Prime as well. The Prime is a fairly well selling ARM device.

### Post by sola on 2012-05-21
Hi,

There are two ways:

1) Installing Ubuntu as a standalone boot. Currently, this will result in a half-working Ubuntu (no wifie...etc) on the Prime, so I don't recommend it.

2) Installing Ubuntu as a chrooted OS on the Prime. This way you keep Android and Ubuntu will run in parallel with Android. You access Ubuntu with a VNC viewer program which runs on Ubuntu. This is acceptable for certain software (e.g. using LibreOffice) but graphical interaction is not very good (it goes through VNC). For this: Search for Ubuntu or Linux on Google Play.

The third would be Ubuntu for Android but that is not available yet. This would be mostly 2) but with much-much better performance and a lot of smooth integration between Android and Ubuntu.
