### Post by darmok on 2012-01-10
Background
-------------------------------------------

I've been having trouble booting 64-bit Linux distributions on my Asus M4A87TD/USB3. Linux distributions using 32-bit kernels, including Ubuntu 11.10 and Fedora 14 Security Lab Spinn, boot without incident. I've been successfully dual-booting Fedora 14 64-bit and Windows 7 Professional on separate hard disks for the past year, so something has happened within the last month (approx. 5 December) to break 64-bit support entirely.

The Asus M4A87TD/USB3 is supporting an AMD Phenom II 1075T processor with 16GB DDR3 RAM. Harddrives and the optical drive are connected to a SATA bus.


The Problem
-------------------------------------------

Using both new and old LiveCDs, I've determined that both old and new 64-bit kernels are having difficulty booting. When I try to boot a 64-bit distro, I get past the Grub screen only to find a screen full of kernel errors. I then need to reboot. This happens when trying to boot from Fedora 14 installed on one of the hard disks, from LiveCDs mounted in the optical drive, and Live and install images booted from USB sticks.

Investigation of this problem started with this thread: [http://ubuntuforums.org/showthread.php?t=1907010](http://ubuntuforums.org/showthread.php?t=1907010)


Questions to the Community
-------------------------------------------

* Has anyone else had an Asus motherboard suddenly stop supporting 64-bit Linux kernels on this or some other Asus motherboard?
* Does anyone know of a possible solution to my being able to boot only 32-bit Linux kernels?


Thanks, everyone.
