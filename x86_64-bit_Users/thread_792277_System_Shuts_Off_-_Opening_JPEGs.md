### Post by HalloweenB on 2008-05-12
Just built a new system running Hardy 64-bit.  Most of the bugs are worked out, except this one:  opening and working with JPEGs in F-Spot and gThumb causes the system to shut off.  "Working" means basic opening, closing, and previewing pictures.

Since no message comes from Hardy and it's an immediate shut-off, I assume it's a bug in the processor or mobo.

Concerning an overheating issue, it is a small form factor case.  However, it reboots fine and the temperature shown in the BIOS setup (assuming it's accurate) hovers in the 30-32 C range.

Other apps run fine including running WoW using the latest Wine version.  Perhaps I can download a program to run the processor under heavy load to see what happens?

Any troubleshooting hints would be appreciated.

AMD 64X2 4800+ processor
Thermaltake Silent 939 K8 processor fan
Antec Formula 5 silver thermal compound
Abit NF-M2SV mobo
2 GB 800mhz PC6400 Crucial RAM

### Post by HalloweenB on 2008-05-14
This is probably not the forum to be asking this type of question, but it turned out resetting the BIOS and setting the correct RAM frequency (800) solved the issue.
