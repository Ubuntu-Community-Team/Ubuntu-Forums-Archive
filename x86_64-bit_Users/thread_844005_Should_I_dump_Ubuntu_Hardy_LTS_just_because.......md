First, I assume you are moving files with the GUI (Nautilus). Nautilus is part of Gnome, so Ubuntu doesn't have any direct control over it.

Second, I have Hard 64-bit and I can move 25 GB of files all at once without a problem. It takes a while, but that is due to hard disk speed. Furthermore, if this were a problem for all users the forums would be alive with yelling. I conclude that the problem is unique to your system. What went wrong and how to fix it is another matter.

In fact, you can copy/move files from the command line, so that is another thing to try in troubleshooting.

After you have tried a few alternatives, post back with the results and we'll go forward with additional suggestions and ideas.

### Post by ian dobson on 2008-07-12
Hi,

Are you using S-ATA drives? I've had problems with WD and Samsung drives on x64. One solution I've found is to disable NCQ by adding:-

echo 1 > /sys/block/sdX/device/queue_depth

to the end of rc.local for each drive.
