### Post by 3Miro on 2009-05-27
The reasons a lot, the importance varies from person to person. Here are some: 

- Google Free Software and GNU project as well as search you tube for Richard Stallman, you will learn the philosophy of Linux.

- Security, there is a reason why Linux is dominating the large servers, it is in any respect more secure than windows. Linux is not perfect, but whenever a problem in Linux is found, people fix the problems. Whenever a problem in windows is found, you have to
use an anti-virus program, note the problem is still there, just masked.

- About half of the video games out there and occasionally some other pieces of windows software do not run under Linux and have no Linux counterpart. Other than that, everything you can do in windows you can do in Linux (I am playing and modding windows games under Linux). There is a learning curve involved and yes Linux is very different and you are just frustrated because you have not figured out yet.

- I am doing a lot of Mathematics on my computers and trust me, there is a reason why 90% of the supercomputers are running and Linux and virtually none are using windows. Windows simply cannot handle applications that require a lot of computing power.

To the above you can add better hardware support, faster update cycles and generally the size and power of the Linux community.

### Post by 3rdalbum on 2009-05-31
If you want to use an operating system that is VERY much like Windows and will run Windows software, there's an obvious answer.

If you want:

1. Flexibility
2. True security (not just from viruses!)
3. Free major upgrades
4. An amazing amount of free uncrippled software that will never nag you for money
5. A choice of excellent desktop environments
6. A system that will truly work out-of-the-box on a large number of hardware configurations, including non-x86 processors
7. Great community support and documentation
8. The ability to write your own programs and deploy them anywhere
9. A high degree of stability
10. The ability to look at (and modify) the code that the software uses
11. The ability to make a real difference to the software you use in bug reports, feature requests and contributions
12. High quality contracted support
13. The knowledge that your software can be trusted as a result of the peer review process
14. True plug 'n' play from devices (no driver installation)
15. Freedom from anyone telling you how to use your own computer
16. A different operating system with a different way of doing things

Then Ubuntu is for you. If you don't want any of those things, or you do but you are not willing to learn Linux, then you would be better off putting up with a different operating system.

Asking why anyone would use Linux when they can use Windows, is missing the point really. Remember, the percentage of computers running Linux is constantly increasing. The percentage of computers running Windows is constantly decreasing. The point is more "Why would anyone use Windows when they can use Linux?"

EDIT: I once submitted a feature request for some open-source software a while ago, and the developers implemented the idea. My coding skills weren't up to scratch to actually code it myself, but I could still contribute in this way.
