### Post by nss0000 on 2008-09-20
Gents:

To conclude my updating experience from x64-Dapper_Drake to HH_8.04.1 .....
from the POV of a long-time, but casual *nix lusr: 

NB: my hardware is well-performing, but legacy AMD kit ( 5400+ ), a desktop unit running only UBUNTU with a single 19" LCD monitor and DSL support. 

1) DLoad, install and configuration vis' WWW was simply automagic. No issues no glitches .. and quick ( 2-h start-to-finish ). Only ng. reader PAN failed to carry-over configs. Note: Casual lusrs do NOT separate apps from OS, so finger_pointing blame means nothing.

2) Post install details included CLI tasks using KILZ nswrapper script to enable browser_FLASH, and well-documented removal of a directory to enable SYNAPTIC / UPDATE MANAGER. A bit of typing, yes ... both worked the 1st time. No browser JAVA plugin, of-course. 

3) I did-NOT enable the NV_drivers as the display looks fine to me.

4) Sadly ... still no automagic configuration of my LAN. This would have been the STARS in my UBUNTU update. Guess it will never happen; it's surely a show_stopper for many potential UBUNTU users.  

All things considered ... the experienced UBUNTU lusr with modest expectations has nothing to fear from x64_HH. It's quick and gorgeous and  provides both sun & moon. However, the "stars" that might drive new users to UBUNTU are surely lacking.
