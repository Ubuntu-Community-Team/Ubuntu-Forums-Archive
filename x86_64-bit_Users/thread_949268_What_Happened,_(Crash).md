### Post by Milleman on 2008-10-16
Same happened to me. The latest update contained a new kernel and new Nvidia drivers. It seems that the nvidia wasn't applied to the latest kernel update. After logging in, you have to download and install the latest nvidia driver again, from the Synaptic Package Manager. Serch for "nvidia" and click the checkbox for:

linux-restricted-modules-2.6.24-21-generic

[IMG]http://www.breilind.com/pics/synaptic.png[/IMG]


And then press Apply.
Then you have to go to 'System' - 'Administration' and start 'Hardware Drivers'. Then enable the nvidia graphics card driver.

Reboot!

It's most likely that after reboot, your screen still only use 800 x 600. You can change the Desktop resolution in 'NVIDIA X Server Settings' and get the resolution you want. But after a new reboot, everything is back to 800 x 600 again.

The cure for this, is to start the 'NVIDIA X Server Settings' from a terminal as root:

```

sudo nvidia-settings

```

Then you set the desired resolution. Before you exit, you have to click the button "Save to X Configuration File".

[IMG]http://www.breilind.com/pics/nvidia-settings.png[/IMG]

Now you can quit. Next time you start your computer, the desktop will remain as you selected last time. Enjoy! :guitar:

If you want to get rid of the NVIDIA Logo during boot, you have to edit the "xorg.conf" file and add one line:

```
sudo gedit /etc/X11/xorg.conf
```

Under the "Device" section, add [Option "NoLogo" "true"] to the last line:

```
Section "Device"
    Identifier     "Videocard0"
    Driver         "nvidia"
    ... (etc)
    Option         "NoLogo" "true"
EndSection
```


I don't know why this happened this time. In the past, everything used to run fine after kernel updates.

### Post by Thelasko on 2008-10-16
The Nvidia graphics driver has a module that interfaces with the [kernel]("http://en.wikipedia.org/wiki/Operating_system_kernel"). (kernel module)  Your system restarted because you just updated your kernel. (the only reason a Linux machine should restart)  When the kernel was updated, the Nvidia driver had to be updated as well.

This is quite normal.
