### Post by mess110 on 2009-08-22
My 2cents:

I also had the same problem. Lets analyse:

First from a technical point of view: If you have more than 3GB of RAM and really want to use it all than you should install 64 bit OS. But I doubt people really need it. (unless they do some really intense rendering. for example gfx artists)

From a end user point of view: Sometimes you don't always find 64 bit software (although nowadays its really unlikely) so you might have to do some tweaks to run the software you want. And if you don't like tweaking or this sort of stuff it might not be worth it.

I read a lot of pros and cons performance wise. For instance a lot of people said their computer works way better with 32 bits than 64 bits. I for one didn't mind making software work on 64 bits (compiling or sth like that) but it got to a point when it just got a little frustrating. 32 bits would just make my life easyer (for arduino and some BlueZ stuff. and emulation with J2MM) I am not saying it didn't work but it would have been a lot easyer if it wouldn't have happened.

I think the best person to decide is you. If you really want to go for what is optimum for you, than just install both and do a performance test (if you can spare the time). You won't see a huge improvement though. Ubuntu rocks either way!

Support for 32 bit won't seize to exist but if you do want to help the comunity why not upgrade?
