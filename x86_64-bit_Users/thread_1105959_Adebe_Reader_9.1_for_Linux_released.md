And that's only if you **NEED** version 9.1 (which I'm sure most people don't).

The phobia is not confined to Linux; what upsets me is that the "4 GB of RAM or more" mantra persists when recommending a 64-bit operating system (any 64-bit operating system).  Over the past four months personally (and over the month prior on a test system), I'd run 64-bit versions of Windows and Linux on sub-2 GB (in fact, mostly sub-1 GB) configurations.  While in only a few cases did the 64-bit version or applications thereof run better than the 32-bit version, in not one case did any operating system or application run worse (64-bit vs. 32-bit), and in every case, the 64-bit operating system crashed less (in all except one case, the 64-bit operating system didn't crash at all; in the one case where the 64-bit operating system DID crash, it was because it had been pushed far beyond the recommended tolerances).  I recommend 64-bit primarily for increased stability, not for increased speed; and that is especially true if you have less than 4 GB of RAM.
