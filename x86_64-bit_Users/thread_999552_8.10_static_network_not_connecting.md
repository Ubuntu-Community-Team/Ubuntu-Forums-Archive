### Post by felker2 on 2008-12-02
```
DEVICE=eth0
BOOTPROTO=none
BROADCAST=210.219.61.255
HWADDR=00:0C:76: D0:09:EA
IPADDR=210.219.61.110
IPV6_AUTOCONF=yes
NETMASK=255.255.255.0
NETWORK=210.219.61.0
ONBOOT=yes
GATEWAY=210.219.61.1
TYPE=Ethernet 
```

Yes, these IP addrs, netmask and gateway do match: IPADDR=210.219.61.110 and GATEWAY=210.219.61.1 are OK with the NETMASK=255.255.255.0. As 210.219.61.110 is a public IP address, this system is directly connected to Internet.

However, your first post used 192.168... as IP-address. Then you can't use the gateway 210.... . 
BTW: 192.168.... is a private IP address, so this system must be behind some NAT device. Is that correct?


But, once again: why aren't you using DHCP on your LAN behind your NAT device? That will take care of the IP addressing automagically.

HTH

### Post by automaton26 on 2008-12-02
192.168... is a local IP address range, which would only be valid if the machine is behind a NAT device/router. If that is the case, try setting both the gateway and DNS server to the IP address of that NAT device/router (note that the NAT device/router may need to be configured to allow the IP address and network card MAC address).

Otherwise the IP address must be 210.219.61.XXX (as in the working example you gave), which must be allocated for you by the administrator of that subnet range.
