### Post by ivan.bertona on 2009-06-05
Hello. I recently bought a brand new Acer Aspire 5930G notebook (there are many variants, mine has part number LX.AQ40X.132). Some relevant specs are: Core 2 Duo P8400, 2.26GHz - 4GB DDR2 - GeForce 9600M GT, 512MB dedicated VRAM, up to 1791MB with TurboCache. I have installed Ubuntu 9.04 64bit and I am really happy with it. Fast installation as usual, hardware working out of the box, everything is ok but one detail: according to System Monitor and "free -m" a full gigabyte of RAM is missing. I checked and two 2GB modules are physically installed in the appropriate bay.

I googled the problem but most information is about 32bit processors and how to enable PAE, and many pages clearly state that theoretical limit with a 64bit kernel is indeed 16TB. I think it may be the video card sucking memory but the "NVIDIA X Server Settings" application says "Memory: 512MB". I found no information on methods to display or edit the amount of RAM shared with the video card. I use NVIDIA proprietary drivers version 180 (the one provided by Ubuntu). I'd like to rescue my 4th gigabyte of RAM, somebody help? Thank you in advance!

Here is some information I think may be useful:

edit: i forgot to mention that BIOS detects correctly 4GB of RAM and does not have any option regarding shared video memory or extended memory

> **dabl said:**
> What does this mean?  Does it mean the video chip can grab 1.2G out of memory and reserve it?  If so, there is your answer.

From the wikipedia [http://en.wikipedia.org/wiki/TurboCache](http://en.wikipedia.org/wiki/TurboCache)

> NVIDIA's TurboCache technology is a method of allowing video cards more available framebuffer memory by using both on-board video memory and main system memory. Main memory is accessed using the high-bandwidth PCI-Express bus.

<snip>

While TC can improve 3D quality in games using a great deal of video memory for texture rendering, performance of games utilizing a large percent of system RAM (such as EA Games' Battlefield series and their mods that typically consume 512MB-1GB RAM) can suffer dramatically on systems with 1GB RAM or below. Google searches show many users of cards with TC desperately trying to disable TC. Such an option is not supported by NVIDIA's official drivers but can be accomplished using a registry modification for Windows XP users or a configuration file modification for X.Org Server users.

So indeed, if the TurboCache is 1.5 GB, it's indeed eating the RAM from Linux.

### Post by sanderj on 2009-06-06
> **lvc said:**
> I have a similar problem on my 3.0 GHz Pentium D. I have 4 GB of DDR2 installed (and the bios recognises all of it)


@lvc:
you're running a 64-bit OS, so that's not the problem.
The problem is that your BIOS is not offering memory above the 0100000000 barrier, so you will not see more than 3.2 GB.

Solution:
1) Upgrade your BIOS. 
If that does not work:
2) search your BIOS for "memory hoisting" and "memory hole remapping": that feature should be on.

The older your hardware/mobo/BIOS, the smaller the chance it can handle/offer more than 3.2 GB RAM. I think hardware before 2007 is a serious problem.
