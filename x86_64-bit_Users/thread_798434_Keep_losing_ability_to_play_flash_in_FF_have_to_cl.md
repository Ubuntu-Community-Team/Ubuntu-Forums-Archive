### Post by John Jason Jordan on 2008-05-19

This is a very common complaint with the beta of FF3. But having said that, it is the only problem I have had with it since it was installed with the upgrade to Hardy.

At first I questioned the logic of Ubuntu including beta software in Hardy, which is a "long term support" (LTS) release. But then it was pointed out in another thread that including the beta as the default browser means that Update Manager will always automatically offer upgrades for it. If it had not been included in Hardy then the only way people would ever get 3.0 would be by manually installing it.

If you spend all day on Youtube, then maybe you should just go back to 2.0, or install Seamonkey or something else instead. In my case, the upgrade to Hardy left FF 2.0 installed, so I can use it if I want to. However, I'm using the 3.0 beta and if once in a while I have to restart it to get flash working again, that is a small price to pay for the convenience of getting automatic upgrades for it.

### Post by NilsE on 2008-05-21
If you use flashplugin-nonfree it should work OK but some users had to remove the libflashsupport that gets installed by default in the early version.  I removed gnash and reinstalled flashplugin-nonfree and removed the libflashsupport and have not had a problem since.  

However, some people have experienced losing sound in flash and had to reinstall libflashsupport.

So, it is worth a try.  Remove anything that is flash related, reinstall (or install) flashplugin-nonfree and if libflashsupport is installed remove it.  If it works OK but no sound then reinstall libflashsupport
