### Post by phlembob on 2008-11-07
Synopsis
I have installed 3 Ubuntu 64 bit Operating Systems: Ubuntu 8.04 LTS 64 bit, Ubuntu 8.04.1 LTS 64 bit, and Ubuntu 8.10 64 bit BETA.

Installations of 8.04 LTS 64 bit and 8.10 64 bit BETA had some minor difficulties, but Ubuntu 8.04.1 LTS 64 bit was smooth.  This version is down-loadable now or 11-6-2008.  

When using multiple OS-s on one computer Grub is placed for selectable Operating System.  In all instances &#8220;Grub&#8221; has not been consistently perfect.  

At this time, I have installed 8.04.1 64 bit LTS once.  I must ask myself if that was a fluke with my experiences with the other 2 OS-s.  I'm not going to redo it to find out because it is installed on a separate SATA HDD.

Proprietary NVIDIA video driver for the 8.04 and 8.04.1 work with NVIDIA 6100 and 6600 video cards with a VGA CRT monitor and/or LCD DVI and VGA plugged monitors.  It is possible but it is not expected, people with different monitor manufacturers with these video cards to have problems.

Proprietary video card drivers with the 8.10 has blanked (black) monitors.  This has happened with both ATI and NVIDIA with both the 8.10 BETA and the 8.10 release.  HEY --- Without video there isn't any human interface!  Some experienced people have overcome this problem.

Hope this helps some of you.

Here is some other information that should help answer that question.

1. The open source development method - Open source projects like Ubuntu follow this plan. 
  A) Create Distribution, get it reasonably working well.
  B) Release it so the community can find the problems.

2. Both Hardy Heron release 8.04 and Intrepid Ibex have followed point #1. Hardy Heron release 8.04.1 is a fixed version after the bugs were found and fixed.

New users who come from say Windows expect a release, or even a beta to work perfectly. That's because for the most part that's how it works in the Windows world. But that's not how it is in free software.
New users who rush to install the latest release will suffer most of the bugs. Granted, some will have good installs and not have problems. But that may be because they dont use the area of the OS that has the bug.
This is important to understand if you are a new user who wants stability. Dont download the latest thing. So no, your experience of having minor issues with a first release is not strange.
