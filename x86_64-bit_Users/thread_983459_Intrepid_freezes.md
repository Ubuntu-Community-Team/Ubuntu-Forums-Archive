### Post by utrading on 2009-08-12
I too have been struggling with hard freeze (both Jaunty 9.04 and Intrepid 8.10), for about 2 months

Here is what I did since last week: 
1. Download the lastest nvidia driver 185.18.31 to your home directory
2. Install fresh copy of Intrepid 8.10
3. After first reboot, don't ever activate bundled nvidia driver!!! 
4. Switch to console
5. kill gdm by running "[FONT=Courier New][SIZE=1]sudo killall gdm[/SIZE][/FONT]"
6. Install the downloaded nvidia driver (answer Yes when asked to update /etc/X11/xorg.conf)
7. launch X by running: "[FONT=Courier New][SIZE=1]sudo gdm start[/SIZE][/FONT]"

I frequently run virtualbox or vmware, and I found virtualbox tends to trigger some kernel bugs due to hardware issue (mostly related to irq conflicts of nvidia cards with usb devices), more so than vmware. If you see the dmesg log and found something similar to 
[FONT=Courier New][SIZE=1]irq 18: nobody cared (try booting with the "irqpoll" option)
[/SIZE][/FONT]  
then you can try the following:
1. In BIOS: disable IDE compatible mode, i.e. enable AHCI mode only
2. At boot pass extra parameters to kernel (what was what I did):
[FONT=Courier New][SIZE=1]acpi=off irqpoll
[/SIZE][/FONT]
Now I run vmware instead of virtualbox. It's been two days and so far so good. Crossing fingers...

My hardware: Shuttle XPC + Nvidia 9400GT (PCIe) + Linksys Wireless
