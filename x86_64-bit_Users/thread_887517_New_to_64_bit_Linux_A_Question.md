### Post by Thelasko on 2008-08-12
> 64bit is not in its infancy with Ubuntu.
Correct, 64-bit in Ubuntu is as advanced as 32-bit.

Issues only arise with proprietary software that isn't as advanced (closed source 32-bit only software).  Thanks to people like Kilz, there are workarounds to many closed source applications.  In Hardy, they are executed seamlessly, and you won't even notice.  Don't forget to take a look at the [Advantages and Disadvantages of 64-bit page.]("http://ubuntuforums.org/showthread.php?t=765428")

The most common problems with 64-bit right now involve, Java (being resolved), and printer drivers (Lexmark being the worst offender).  I advise you to look around the forums and see if your printer is supported in 64-bit (or at all by linux).  Another excellent resource is [Linuxprinting.org]("http://openprinting.org/printer_list.cgi")  

If you require either of these, I recommend 32-bit.  (Java may sound like a deal breaker, but I've discovered I rarely use it, and most websites offer an alternative.)  If you don't, try 64-bit, because there is no reason not to.

### Post by Thelasko on 2008-08-20
For most people, navigating to a flash website will guide you through the process of installing flash, and you won't need [the guide.]("http://ubuntuforums.org/showthread.php?t=772490") (it's the same as in 32-bit Ubuntu)  If that doesn't work, many people have success with:
```
sudo apt-get install flashplugin-nonfree
```
Apt should take care of the dependencies for you.  If it doesn't, [the guide]("http://ubuntuforums.org/showthread.php?t=772490") will walk you through the steps to install them manually.

If everything has failed, you need to [file a bug report.]("https://launchpad.net/ubuntu/+filebug")

Java can be a bit more difficult.  The simplest solution is to install OpenJDK.  There is a prompt when you visit a Java website, similar to the one you see for Flash, or you can use the terminal.
```
sudo apt-get install openjdk-6-jre openjdk-6-jre-headless openjdk-6-jre-lib icedtea-gcjwebplugin
```

That should get most Java applets working.  If that doesn't work you need to install [32-bit Java.]("http://ubuntuforums.org/showthread.php?t=202537")
