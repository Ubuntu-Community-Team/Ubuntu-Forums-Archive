### Post by grahammechanical on 2014-11-03
A couple of development cycles ago, soon after the start of the cycle there was a kernel update that broke the OS and it took three further kernel updates but for the matter was fixed. These things happen.

Ubuntu Desktop Next is being built on the Vivid Vervet code base, which itself is built on the 14.10 code base. Think of Desktop Next as a temporary parallel development branch where convergence of Ubuntu for Devices with Ubuntu Desktop is worked out.

Desktop Next and Ubuntu Desktop have the same repositories. So, Desktop Next gets what comes into Vivid Vervet (15.04) but Desktop Next has stuff that Vivid Vervet does not have and will not have because 15.04 will be stll using the Xserver, LightDM, Compiz and Unity 7 just like 14.04 and 14.10.

Where it starts to get very interesting, in my opinion, is when we get to the development cycle of 15.10. Then we might see the Desktop Next 15.04 code become the basis for the development of 15.10. If it can be done we might see 15.10 using MIr and Unity 8 with added phone functions. It is certainly the intention to have all this in place by the time of the release of 16.04.

So, by tracking Desktop Next we are right at the very bleeding edge of Ubuntu development. And sometimes we get cut.
