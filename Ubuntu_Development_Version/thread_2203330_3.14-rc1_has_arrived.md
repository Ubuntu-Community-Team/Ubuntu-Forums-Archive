### Post by VinDSL on 2014-02-04
BTW, here's what I did...

I'm running 304 Experimental (for no particular reason). The hack should work with other versions.

```
vindsl@Zuul:~$ apt-cache policy nvidia-304-updates
nvidia-304-updates:
  Installed: 304.117-0ubuntu1
  Candidate: 304.117-0ubuntu1
  Version table:
 *** 304.117-0ubuntu1 0
        500 http://archive.ubuntu.com/ubuntu/ trusty/restricted i386 Packages
        100 /var/lib/dpkg/status
vindsl@Zuul:~$ 
```

The file I hacked was 'nv-linux.h'

```
vindsl@Zuul:~$ whereis nv-linux.h
nv-linux: /usr/src/nvidia-304-updates-304.117/nv-linux.h

```
Using GEdit

```
vindsl@Zuul:~$ sudo gedit /usr/src/nvidia-304-updates-304.117/nv-linux.h
```

Near the top, around line 40, I added:

```
#include <linux/version.h>
#include <linux/utsname.h>

[B][COLOR="#FF0000"]#if LINUX_VERSION_CODE >= KERNEL_VERSION(3, 14, 0)
#include <linux/acpi.h>
#endif[/COLOR][/B]

#if LINUX_VERSION_CODE < KERNEL_VERSION(2, 4, 7)
#  error This driver does not support 2.4 kernels older than 2.4.7!
#elif LINUX_VERSION_CODE < KERNEL_VERSION(2, 5, 0)
#  define KERNEL_2_4
#elif LINUX_VERSION_CODE < KERNEL_VERSION(2, 6, 0)
#  error This driver does not support 2.5 kernels!
#elif LINUX_VERSION_CODE < KERNEL_VERSION(2, 7, 0)
#  define KERNEL_2_6
#elif LINUX_VERSION_CODE >= KERNEL_VERSION(3, 0, 0)
#  define KERNEL_3
#else
#  error This driver does not support development kernels!
#endif
```

That's it...

Save.  Install 3.14-rc1.  Reboot.  ;)
