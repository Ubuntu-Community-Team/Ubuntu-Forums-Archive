As I understand it  the devs will be adding Gtk support and this will allow  a program like nautilus to be installed as a .deb app and  be able to work with USB drives. As it stands now you can install nautilus through a container and it will work (to some degree) both in the container and  as an app on the unity8 desktop. So the idea is to get these programs working with mir and have unity8 graphically functioning as a viable desktop.  The reason why it is going so slow is that developers would render that snap apps would be more viable to install across the board in both containers and the unity8 desktop (most likely because of mir) so this will be the main move to the unity8 desktop and secure containers. Convergence then is second on the list during this cycle and so  I think what we will see is a viable and highly modified version of a unity8 theme that will work with xapps. So basically we will be able to use phone and tablet like gestures with a desktop mouse on older desktops. To draw another comparision .. Ubuntu will succeed in presenting  a working  type kaelidoscopic metro like desktop on many legacy systems where microsoft has failed. I think the caveat here will  be that unity8 will then get lots of attention as unity8 will become the default desktop with unity7 as a fall back option. I think the side panel that we now see is a technical preview of some of the mir capabilites. it is the next new thing ! :)
