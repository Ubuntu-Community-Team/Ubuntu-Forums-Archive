### Post by dino99 on 2015-02-15
there are some acpi errors around line 800 and later, like:
ACPI Exception: AE_AML_PACKAGE_LIMIT, Index (0x000000008) is beyond end of object (length 0x1) (20141107/exoparg2-420)

so be sure about the bios/efi settings used.
Is that system a fresh installation ? myself boot with systemd-sysv (package installed, grub not modified, not needed now) because ubuntu is slowly moving to it , and results are better now than with upstart which get less devs attention. 
To get errors highlighted, open a terminal and run "sudo journalctl"
If your system(s) have been only dist-upgraded since a while, then maybe you got some weird configs registered inside the hidden files: .gconf, .config, .gnome2, .local ; you can save them by adding .old to their names for example, then reboot (they will be automatically recreated, but without your custom settings of course).
That way you can narrow down the issue. Also downgrade to genuine package(s) in case some ppa's package(s) are used, and possibly disturb the system.
