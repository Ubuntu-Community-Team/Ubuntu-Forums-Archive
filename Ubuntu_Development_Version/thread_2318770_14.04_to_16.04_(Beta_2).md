### Post by prusswan on 2016-03-29
Just wanted to report that the upgrade path is still very rough atm. On the two vms I tried, both were unable to complete dist-upgrade cleanly, with the first reboot into 4.x kernel resulting in kernel panic, which requires a switch to 3.x kernel followed by completing the rest of the upgrade process via ```
sudo apt-get update
``` and ```
 sudo apt-get upgrade
``` (after fixing whatever packages that are broken, initially it cannot even start the window manager, just showing the terminal login prompt)

As for fixing broken packages, at some point ```
sudo apt-get install -f
``` and ```
 sudo dpkg --configure -a
``` did not work due to broken packages and dependencies (some error similar to E:Error, pkgProblemResolver::Resolve generated breaks, this may be caused by held packages.) and needed to be resolved manually by identifying, removing, and hard-installing individual packages with ```
apt download
``` and ```
dpkg -i
```

It did not help that aptitude had been uninstalled and removed as a default package, which made recovery harder.
