### Post by chorca on 2016-04-06
I've got sort of an odd setup, here it is:

Running a storage server with an array of drives on ZFS, entire server running Ubuntu 16.04. Latest updates.
Several LXC containers on the server, serving various purposes, some require access to the disk array. Have given them this via either privileged containers or via ACLs and user ID mappings.
Recently the mapping/ACLs just don't seem to be working right, and after some investigation, I found that after a reboot, the ACLs on ZFS don't seem to be taking effect until the directory is 'ls -l'd on the host, then the container's user is able to see it.

I have an example here, since this is a sorta complex concept.. the directory is mapped into the same directory structure inside the container so it's consistent:

(Inside the container, unable to cd into a directory it should have access to)
```

root@datawork:# cd Thein/
bash: cd: Thein/: Invalid argument

```

(From outside the container, on the host, doing a quick ls)
```

(Back inside the container, notice now the + is visible beside the permissions showing ACLs are working)
```

This only works for the directory and stuff immediately beneath it, anything further or outside that directory still is stuck without ACLs until it's 'read'.

I'm struggling to figure out what's going on here, and which part I should be looking at.. ZFS, ACLs, LXC/LXD, or some interoperation between the three.. It seems as if the ACLs don't initialize properly, so i'm stuck writing a script to traverse the entire ZFS filesystem each time the machine boots to make sure the ACLs 'load'.

Anyone have any ideas on where to look?

### Post by chorca on 2016-04-09
> **brand2 said:**
> Could be related to this: [https://github.com/zfsonlinux/zfs/issues/4177](https://github.com/zfsonlinux/zfs/issues/4177)

It appears you're correct... this seems to be related.

I contacted [stgraber]("https://github.com/stgraber") about this asking if it may be related, and he seems to think it is.
He mentioned that he did not recommend utilizing ACLs in unprivileged containers until the bug is fixed, and I have destroyed and recreated the container as a privileged one, and disabled ACLs across my ZFS filesystems.
There appears to be risk of corruption on the filesystem side, and until the issue is worked out, ACLs in containers shouldn't be used. (at least on ZFS FSs)
