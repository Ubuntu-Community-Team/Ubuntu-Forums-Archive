### Post by ooddiittyy on 2013-09-28
I've been fighting with hibernate configuration on a Sony Vaio Pro (2013) laptop for a few weeks now; initially, fixes listed on Launchpad - for example, [here]("https://bugs.launchpad.net/ubuntu/+source/gnome-settings-daemon/+bug/1194328") - worked, but as of a couple of weeks ago that fix is no longer functional. Based upon recommendations from another Launchpad user, I have successfully gotten a script to run through the acpid which will initiate hibernate successfully, but upon resume the screen simply goes blank.

In /etc/acpi/events, I have a file called "lid" which contains: 

```
event=button/lid[
action=/etc/acpi/lid.sh
```

This calls /etc/acpi/lid.sh, which does the following:

```
#!/bin/sh
/bin/grep -q open /proc/acpi/button/lid/LID/ state && exit 0
/usr/sbin/pm-hibernate
```

To get this to initiate hibernation, I had to modify the ownership of lid.sh to root; otherwise, running it through my user didn't do anything.

As far as I can tell, the running of this script should be equivalent to running pm-hibernate in the terminal; what's odd is that running either "sudo pm-hibernate" or "pm-hibernate" as root works fine and the system resumes correctly, but using the script above via closing the lid does not correctly resume. Any thoughts would be appreciated.
