### Post by candlerb on 2014-03-19
I have been trying snapshot 20140319 of trusty server amd64. I uses Ubuntu as a virtualisation host.

In Precise, there was /etc/default/qemu-kvm which contained settings:

KSM_ENABLED=1
SLEEP_MILLISECS=20

and the script /etc/init.d/qemu-kvm wrote these to /sys/kernel/mm/ksm/{run,sleep_millisecs}

This config file has now gone, and /etc/init.d/qemu-system-x86 no longer uses those locations. As a result, ksmd is disabled:

$ cat /sys/kernel/mm/ksm/run
0
$ cat /sys/kernel/mm/ksm/sleep_millisecs
20

Is this feature deprecated? If we want it, is /etc/rc.local the place to configure it? (Unlike /proc/sys settings, which can be adjusted using /etc/sysctl.conf or /etc/sysctl.d/*, I don't think /sys/kernel/mm can be set in that way)

Thanks,

Brian.

### Post by candlerb on 2014-07-17
This feature appears to be have been brought back in trusty release.

```

$ grep -1 SLEEP /etc/init/qemu-kvm.conf
		if [ -w /sys/kernel/mm/ksm/sleep_millisecs ]; then
			if [ -n "$SLEEP_MILLISECS" ]; then
				echo "$SLEEP_MILLISECS" > /sys/kernel/mm/ksm/sleep_millisecs || true
			fi

```
