### Post by oldfred on 2013-10-31
There always is a potential of conflicts if multi-booting. Software is designed to upgrade or update to new version, but if changes are significant then old version may not work or give strange results.

I have many installs and keep /home inside / (root), but have separte /mnt/data partition that is just all the data without any settings.

       Data can be shared without the possible conflicts of user settings being different in different versions. I only copy some settings from one install to the next, normally. But I have to separately back up /home and the /data partition. Also saves the error of reformating a /home partition accidentally. I never reformat my /data and just configure / for install.

### Post by grahammechanical on 2013-11-01
You will also get newer versions of some of the default applications. For example, you will have two versions of Libreoffice accessing and writing to the same user folder in /home. How that will work out is any body's guess. Trusty Tahr at present has Libreoffice 4.1.2.3 and may well get Libreoffice 4.2 by the time 14.04 is released.

Keep 14.04 with its own /home folder and use the file manager to open the 12.04 partition and then point Libreoffice to your documents in the 12.04 partition. It will remember where the documents are but that 12.04 partition will need to be opened or mounted otherwise Libreoffice will give a cannot find error message. We can do the same with applications like Rhythmbox.

This is what I do and I have all my documents on a separate data partition.
