### Post by Roasted on 2014-04-11
I like how the webapps invoke the Ubuntu Web Browser instead of a regular Firefox instance. That way it feels like a slimmed down centralized little 'app' window for that specific function. Thing is, these things are still broken in some ways. Recent updates tipped the scale a bit with that. For example, if I pin Gmail and Google Calendar to the Unity bar, Gmail will work, yet Google Calendar will not. If I unpin from the Unity bar and launch *the same darn thing* from the dash, BOTH work fine. Seems to be a lack of consistency there. On one hand, yes, it's beta. On the other, we're so close to release date that it's a bit concerning.

Likewise, as mentioned in another active thread I posted here, the Unity pips are too small now as a side effect of converting them to SVG for HiDPI support. Highlighting icons that need attention has also been reworked, which makes it darn near impossible to decipher if, say, Skype or Quassel really need attention or if they are simply running. The difference between running vs needs attention is so minimal that it makes the highlighting feature... do I dare say... useless.

As a result I halted on upgrading most of my systems to 14.04 as this is quite a headache to deal with on a system you use all day for work. Bugs reported and fingers crossed, though.
