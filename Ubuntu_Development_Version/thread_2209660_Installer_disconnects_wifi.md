### Post by monkeybrain20122 on 2014-03-06
Ok, get it!

The culprit is the package bcmwl-kernel-source
If during installation check the box install third party software then it will be installed and the installer switches to wl0. Not checking the box and installation goes through with the free driver which works.

### Post by varunendra on 2014-03-06
> **monkeybrain20122 said:**
> I tried to remove wl0

```
sudo modprobe -r wl0
```
and it says module doesn't exist??!

Yes it doesn't. What you have been blacklisting and playing with is just an alias for it. The correct module name is "wl", and just blacklisting it is not a sufficient fix. Yes you can manually remove the blacklist file it created, but a much better and recommended fix (with BCM4313 card) is to simply purge it -
```
sudo apt-get purge bcmwl-kernel-source
```

The purge routine automatically removes the driver as well as all the configuration files (including the custom blacklist file).

This driver (also known as the "sta" driver) is automatically installed IF you have selected the option to "Install Updates" during installation of Ubuntu, OR if you choose to install the STA driver from "Additional Drivers" option later. It is supposed to work with BCM4313, and it even does, but quite often not decently and sometimes causes more trouble than solving anything.

The native "brcmsmac" on the other hand works quite well with BCM4313, and so one should avoid installing the proprietary STA driver if they have this card.

Let me know if purging the wl driver doesn't fix your problem.
