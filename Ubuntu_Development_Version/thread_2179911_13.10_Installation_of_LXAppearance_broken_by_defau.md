### Post by syntaxerror74 on 2013-10-10
I solved this problem myself, but I thought it may help others explain why things don't work as expected in Saucy.
It seems the installation of LXAppearance is not working by default here. First of all, there is **no menu entry** actually created by lxappearance, "*Customise Look and Feel*".

- I cannot start 'lxappearance' from console: it always says "The program 'lxappearance' is currently not installed. You can install it by typing: ... "

This is nonsense. It gets installed, but probably not where the system expects it to be. First, I tried to *apt-get remove* the old package (in case any was there) and reinstall it. After apt-get attempted to install a 0.5.3-bzr-something version, I still couldn't start lxappearance. So I removed the bzr version again, and installed the regular

lxappearance_0.5.2-1ubuntu2_i386.deb

via *dpkg -i* manually.
VOILA!
Finally I could use LXAppearance. I haven't found any bug report on Launchpad about it, but maybe I should file a report in case someone else can confirm this on their system. (Lubuntu 13.10 or higher preferred)
