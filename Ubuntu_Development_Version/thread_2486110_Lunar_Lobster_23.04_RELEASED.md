Rather than where you download the ISO (*which is accessible via many means thus what you see varies on how you accessed the page*), I think the assumption is you go first to an announcement page which will have many links (*including download, release notes etc*).

An example of an announcement page (*please note this is only one page, there are many*) is [https://fridge.ubuntu.com/2023/04/20/ubuntu-23-04-lunar-lobster-released/](https://fridge.ubuntu.com/2023/04/20/ubuntu-23-04-lunar-lobster-released/)

Evidently the installer can make a mess out of the self  partitioner without the user even knowing something is wrong. Hence a  actual release dated ISO isnt available.


As for ISO date; there is no single ISO date..  The released image is the last RC (*Release Candidate*) image for that product/architecture. To see the various ISO dates for the various 23.04 products you can view the Ubuntu 23.04 lunar ISO tracker site - [http://iso.qa.ubuntu.com/qatracker/milestones/445/builds](http://iso.qa.ubuntu.com/qatracker/milestones/445/builds) which has the ISO date for each architecture/build/product listed... where you'll note its not a single ISO, but varies on product/architecture

FYI:  As I'm involved with QA; this is a page I frequent ~often, thus am familiar with it. Its likely a page most end-users don't see, but can be very useful for a number of reasons (*not all evident*).

Also fyi:  For ISOs which have more than one build in a day; they have a slightly different format, eg. you'll note one has a 20230414.1 as date meaning it was the 2nd build on the 14th of April.
