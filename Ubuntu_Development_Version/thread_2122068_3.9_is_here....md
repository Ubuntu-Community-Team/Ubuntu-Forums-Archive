### Post by zika on 2013-03-05
OK, so You want to use mainline kernel...
1. find kernel You want to use (go to [http://kernel.ubuntu.com/~kernel-ppa/mainline/](http://kernel.ubuntu.com/~kernel-ppa/mainline/) and pick one, like with file on PC)
2. find 4 that are OK with Your architecture (3 for Your architecture and one with &#8222;all&#8220; in its name)
3. DL them to Your machine
4. ```
sudo dpkg -i *names_of_files*.deb
```
Next time You reboot pick one that You've installed...

It would be usual, as that is simply a build of whatever is in the upstream git repo, warts and all.

During the merge window for the next kernel, that will include whatever changes were pulled in the previous day.

As happened this time that could mean build failures, until the build scripts/configs are updated for the new code/version. i.e. it was trying to build a half baked 3.9 using config/scripts meant for 3.8.

Download the latest kernel

```
tar xvzJf linux-3.9-rc7.tar.xz
cd linux-3.9-rc7
make menuconfig
##--select what options you need available in the kernel, I don't know your hardware so can't help--##
make && make modules_install

cp arch/x86_64/boot/bzImage /boot/kernel-3.9-rc7
grub2-mkconfig -o /boot/grub2/grub.cfg
mkinitcpio -p linux
^.this will make sure that grub will make an entry for 3.9.

```

Than I generally just run an update after reboot and it's good to go, I personally find the 3.9-raring.deb kernels buggy... so I compile them myself.

It's pretty easy, just make sure you know your hardware!  Your compiled kernels will run a lot better than the generic kernels for very obvious reasons, your only selecting what you need, the binaries probably have EVERYTHING ticked.

The way I do things is the way I've always done with linux from scratch or gentoo, I prefer source based distros to binary but I prefer apt-get to pacman (prefer arch being more vanilla but like some stability) so I generally compile a custom kernel and I generally only do it once a new stable kernel is released.

> I tried to use earlier kernel. No help. I did some 13.10 kubuntu updates earlier today which reflected initramfs (what ever that really is?)

The Initial ram filing system (initramfs) is a compressed file that contains a bunch of kernel modules and scripts required to mount your root partition and start the 'init' process (amongst other things).

> 
What does this mean, dmesg 

[   10.124116] EXT4-fs (sda5): re-mounted. Opts: errors=remount-ro
[   10.801965] EXT4-fs (sda1): mounted filesystem with ordered data mode. Opts: (null)
[   11.512209] EXT4-fs (sda6): mounted filesystem with ordered data mode. Opts: (null)

These are the message about your filing systems being mounted. 

The top one (sda5) is your root partition. 

It gets mounted read-only at the start so fsck can be run if required. It then gets re-mounted read-write. If any errors are encountered then sda5 will be re-mounted read-only again to stop any more filing system damage.

The other two (sda1, sda6) are supplementary filing systems that it is mounting.

> I added some new space to my linux system today, created new partion which I seem not to have write access at all. What's the easiest way to give access?

Change the mount point so that you are the owner. Give that a try after it's mounted.

```
sudo chown $USER:$USER <mount_point>
```
