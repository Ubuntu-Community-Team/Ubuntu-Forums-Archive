### Post by Doug S on 2023-05-30
> **jagsdesai said:**
> I'm getting no sound with kernel 6.4 **RC4**. Though sound is working just fine with **6.4 RC3**.

Now I don't know if the issue is related to the following or not but I did come across this while searching for reports with similar issues:

"[GIT PULL] sound fixes for 6.4-rc4" (May 25, 2023): [https://lkml.org/lkml/2023/5/25/192](https://lkml.org/lkml/2023/5/25/192)

If you comfortable using git and compiling the kernel, then you would need to bisect the kernel to isolate guilty commit. Since you already found a likely candidate group of commits, you could try a before and after test:

```
doug@s19:~/kernel/linux$ git log --oneline | grep -A 2 -B 2 029c77f89a15
3bf8c6307bad cpufreq: amd-pstate: Update policy->cur in amd_pstate_adjust_perf()
eb03e3181354 Merge tag 'for-v6.4-rc' of git://git.kernel.org/pub/scm/linux/kernel/git/sre/linux-power-supply
[COLOR="#FF0000"]029c77f89a15[/COLOR] Merge tag 'sound-6.4-rc4' of git://git.kernel.org/pub/scm/linux/kernel/git/tiwai/sound
ecea3ba2fba2 Merge tag 'platform-drivers-x86-v6.4-3' of git://git.kernel.org/pub/scm/linux/kernel/git/pdx86/platform-drivers-x86
5566051fa673 Merge tag 'm68k-for-v6.4-tag2' of git://git.kernel.org/pub/scm/linux/kernel/git/geert/linux-m68k
```

by compiling a kernel as of ecea3ba2fba2 to verify that sound works, and then compile a kernel as of 029c77f89a15 to verify that sound doesn't work. You would still have to bisect from there is isolate the exact commit from that group, but it would only be a few iterations. You could then search for additional information using the commit number, as often it turns out to be a known issue that someone else has already reported. And/or submit an upstream email to the commit author copying the appropriate kernel email list.

If you are not comfortable using git and compiling the kernel, then just try the next RC's in the series to see if the issue gets fixed.
