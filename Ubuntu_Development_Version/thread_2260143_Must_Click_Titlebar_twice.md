I wanted to ask here before I post a bug report on this, to see if any other KDE user is running into the behavior:

I've noticed that when running the plasma desktop under vivid, I have to click the titlebar of a window twice in order to activate it.  Every once in a while, single clicking works, but it exhibits this two-click behavior quite frequently.  I may be noticing the issue more than others, because I've set a color-scheme that makes it really apparent when the window becomes actively.

Oddly, this is *never* a problem when switching between windows if I click the client-area instead.  Clicking the client-area always switches between them in a single click.

My Window Behavior | Titlebar Actions is set to "Activate and Raise" on Left Button for Inactive windows, so I believe single clicking should work fine.  I switched the behavior to to other things (like just Raise), and find those actions require two clicks to actually activate as well.

I've done some googling, but I don't see any reports of this bug in launchpad or the KDE bugbase.  Of course, the relevant keywords are all pretty generic.

Anyway, if anyone else is seeing the same issue - I'd appreciate some confirmation that it's not just a local issue.
