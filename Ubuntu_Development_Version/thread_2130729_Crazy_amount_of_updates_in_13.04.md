### Post by grahammechanical on 2013-03-30
When Raring development began in November it had the kernel from 12.10 (Linux kernel 3.5.0). Since then I have had 10 - 12 updates to the 3.7.0 and I have just had my 15th update to the Linux kernel 3.8.0.

Each release is built upon the code base of the previous release. Sometimes the differences between one release and the next are small. But there are big differences between 12.10 and 13.04 and even more to come. There was an intention to bring in Unity 7 and 100 scopes into 13.04. That would have meant a massive download of small packages. It is not going to happen now. But just wait until 13.10 starts development. In the next year we will get Unity 6 changing to Unity 7 and then to Unity Next. There will be 100 scopes or more and then the display server is going to be changed as Ubuntu converges desktop, phone, tablet and TV into one user interface.

It is a time of change. Ubuntu 14.04 will be as different to 12.04 as 12.04 was different to 10.04. Think of the development branch as "bleeding edge."

### Post by IanW on 2013-03-30
Updates always speed up in the last few weeks of a development phase, as more & more "showstopper" bugs are closed before the final release date.

To minimise breakage, either don't use the development version of Ubuntu, or at least don't use the "proposed" repo (as buggy software in proposed is fixed, it gets shunted to main).
