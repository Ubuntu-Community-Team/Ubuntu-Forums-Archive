### Post by buzzingrobot on 2015-09-25
I'll try hard not to make this a rant...

Taking a foray into Kubuntu final beta, I see that KWallet *still* demands a password every time I launch Kmail, and every time it needs to (re)connect with an IMAP server,  That last request seems to pop up beneath anything else on the screen, so while you think mail is being checked regularly, it won't be until you happen to see that popup. (And if the point of KWallet is to remember passwords, it clearly isn't in this instance.)

Bewilderingly, I have not created a wallet.  But, one seems be there by default.  Why?  Does KDE even work anymore without a wallet?

"Old" KDE was infamous for Kwallet popping up at maddening frequency demanding a password. Most sane users just disabled the thing or avoided KDE altogether. For about 5 minutes, it seems, that was fixed. Now, it looks to be back.

If  users want that kind of annoying password manager, let them install it.  Don't force that annoyance on everyone. 

Gnome's had this right for umpteen years.  Why not KDE?  If Window 10 did this, the tech press would be salivating.
