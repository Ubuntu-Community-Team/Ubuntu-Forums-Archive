### Post by dino99 on 2016-02-15
what i've understood about your system:
- you have several partitions with a few OSs installed , each one with its own grub bootloader
- in such a case, the first installed grub play the role of 'master', the other grub(s) is/are slave(s)
- so when the 'slave partition(s) get some new kernel(s), their own grub is rebuilt
- meaning that the 'master' grub is not aware of the changes on the other upgraded partitions

So to get an updated grub menu (which is on the 'master' partition), you need to boot on that one, and from its terminal, run the command:
> sudo update-grub or > sudo apt-get install --reinstall grub-pc

Then you should get all the other partition(s) with an updated grub menu.
