### Post by grahammechanical on 2014-05-16
It makes sense to me. Sometime in the next 12 - 18 months Ubuntu will get not just a different but a completely new display server/compositor/UI combination. And when that change happens it will

a) take place without anybody noticing anything.
b) take place but give a sub-standard user experience
c) take place and break the desktop and "down will come baby, cradle and all."

The development version is supposed to be stable enough to allow other developers to use it as a development platform. It does not make sense to risk that stability. Nor does it make sense to create another development archive so this new code for the desktop can be tested. So, it seems to me that ISO images is the way to get this code out to testers.

When i think of these developers of the flavours who have already decided against using MIR, then I realise that whatever compositor they choose will have to be tested and so they also risk breaking their development releases. Major breakage in the development release particularly when it is new code means bad publicity.

### Post by ventrical on 2014-05-17
Installing DEs from the repos does break 'Ubuntu' to some extent. ie; if you install kubuntu-desktop over an ubuntu (unity) install there will always be overlay or dups of similar programs. For testers it is not really that bothersome but for noobs and the curious they may find breakage at any time.

  In some ways it is more of a problem than a bug. It is the way the different DEs install themselves and then jockey for video drivers or servers. What's good for the goose is not always good for the gander! If these linchpins can be set correctly and oiled a little bit it would make for a much more pleasant experience.

From my testing with unity8 in the last cycle I notice that it has some similar attributes to Unity 2D! That was a great little  DE for lower end vintage and legacy machines. It was a unity standard that could be relied upon. But maintaining it  would mean weaving it into the newer projects or at least this is what we are led to believe.

 But I'm not complaining because I have access to more than enough cutting edge hardware. Installing and testing is a breeze these days but it is always challenging to get older machines working with newer releases. With the prospect of breakage and llvmpipe rearing it's ugly head it sort of takes the wind out of my sails when working with lower end machines.

regards..
