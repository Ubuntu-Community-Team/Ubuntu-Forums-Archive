### Post by zika on 2014-12-16
You have 3.16 kernel installed due to linux-image and linux-headers packages that pull these particular kernel and headers and keep them up-to-date.
Each time new edition of kernel/headers package come old ones can be purged with &#8222;autoremove&#8220; as it is clearly suggested by apt-get during upgrade.
Once these parent packages are upgraded to 3.18 and child-packages are ready You'll get automatically installed 3.18 kernel/headers and read first two line again with 16 substituted with 18.
3.18-rc kernels You've installed manually. 3.18-final You've installed manually. So, You can purge them manually. If You're sure You want to purge them here is how:
First try this with dry-run switch (-s)
```
sudo apt-get purge linux-headers-3.18.* linux-image-3.18.* -s
```and if You're sure/satisfied/ready remove &#8222;-s&#8220; and... 
No regrets, no compunction, no guarantee from me... ;)
You can not purge 3.16.0-28 if You want to have automatical upgrade of linux-image and linux-headers. Well, You could but it is not worth explaining. It should not bother You, leave it there until 3.18 comes from official repositories. Then You will purge it with &#8222;autoremove&#8220;...
