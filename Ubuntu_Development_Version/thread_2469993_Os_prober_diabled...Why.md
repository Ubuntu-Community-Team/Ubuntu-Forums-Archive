### Post by norobro on 2021-12-16
From a Debian update today:> Date: Thu, 16 Dec 2021 12:33:09 -0600X-UID: 97
Status: O

grub2 (2.06-1) UNRELEASED; urgency=medium

  * Boot menu entries for other operating systems are no longer generated by
    default.  To re-enable this, set GRUB_DISABLE_OS_PROBER=false in
    /etc/default/grub.

 -- Colin Watson <cjwatson@debian.org>  Wed, 18 Aug 2021 13:03:23 +0100


As to why:
[FONT=arial]$ [COLOR=#000000]info grub -n 'Simple configuration'[/COLOR]
 > [COLOR=#000000]'GRUB_DISABLE_OS_PROBER'[/COLOR]
     The 'grub-mkconfig' has a feature to use the external 'os-prober'
     program to discover other operating systems installed on the same
     machine and generate appropriate menu entries for them.  It is
     disabled by default since automatic and silent execution of
     'os-prober', and creating boot entries based on that data, is a
     potential attack vector.  Set this option to 'false' to enable this
     feature in the 'grub-mkconfig' command.
[/FONT]

### Post by oldfred on 2021-12-16
I always turn it off and copy boot stanza's I want into 40_custom.
I have multiple old installs that I do not want in boot menu. 
And I want to add some other settings anyway into 40_custom.

If you put your menu entry in /etc/grub.d/40_custom it will not be over written and will be at the end of your menu. Or you can copy it to 09_custom and make it executable and it will be at the top of your menu.

### Post by corradoventu on 2021-12-17
After recent updates on grub-common os-prober has been disabled for security reasons [https://bugs.launchpad.net/ubuntu/+source/grub2/+bug/1955109](https://bugs.launchpad.net/ubuntu/+source/grub2/+bug/1955109)
so boot menu in multiboot configurations does not appear.
 To re-enable this, set GRUB_DISABLE_OS_PROBER=false in     /etc/default/grub.
