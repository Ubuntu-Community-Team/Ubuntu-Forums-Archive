### Post by guiverc on 2021-09-20
I'll add some thoughts.

I use the *development* branch as my daily system, and problems are **extremely** rare, but issues can occur thus the warnings. I can't remember the last problem, but 2-4 problems are *possible* during a *cycle* (*or the six months between releases*) but don't usually impact all users (*maybe hardware specific*) and I cannot remember the last time I was forced to use my *dual boot* alternative (*20.04 for me*) or if I ever had to use it - but I have it just in case as @Bashing-om suggests :)

(I boot into my dual-boot about once a fortnight primarily to update it; so it's always reasonably ready; my last real *issue* was on that *stable* system though (when it *bumped* from *bionic* to *focal*), not this *development* one)

My install was done in the *artful* (17.10) cycle and is now *impish *(*will be *21.10) and I've not yet had to re-install, but as we generally all have different hardware, use different packages, our own experiences can differ.

Updates occur far more frequently; though as we're already in the *freeze* era of the cycle, they are **greatly **reduced to earlier in the cycle.

Another thing I'll suggest is some support sites don't allow #ubuntu+1 or *development* release questions (eg. askubuntu), but you're already here on UF where there is an area for it :)
