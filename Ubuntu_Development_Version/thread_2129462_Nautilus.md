### Post by grahammechanical on 2013-03-26
Nautilus is developed by the Gnome organization and there have been changes which some users have objected to. I do not have a problem putting in a new folder, whether by right-clicking or from the Nautilus cog menu. I have just tested this to see if I was getting the same result.

> [COLOR=#000000]Also for my planning how do I eventually replace 12.10 with 13.04 once released?[/COLOR]

Software Sources should be set to give you a notification of the next Ubuntu release. At the time of the release the Update Manager when open will show an upgrade button. Click on it to upgrade. Or download the release 13.04 ISO image and install over 12.10. Back up your data first.

We are expecting some changes in May. So, keep watching this section of the forum. There are two ways to run the development release.
 
1) Install it in another partition, as you have done, and do daily updates and see how it changes over the weeks. You will find that you can use it for daily work but always have your data on another partition and always have a stand-by Ubuntu installed (like you do now) which you can use if the development branch breaks. Some time it does.

2) have a second install of the development branch in which you can do that aggressive testing by installing PPAs to bring in experimental code. If this OS breaks and you get too frustrated trying to fix it, as I often do, then simply re-install.

These link will give you some useful commands

[http://ubuntuforums.org/showthread.php?t=1970289](http://ubuntuforums.org/showthread.php?t=1970289)

[https://wiki.ubuntu.com/U+1/tester-wiki](https://wiki.ubuntu.com/U+1/tester-wiki)
