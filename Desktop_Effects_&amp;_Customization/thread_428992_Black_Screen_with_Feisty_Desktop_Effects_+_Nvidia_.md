### Post by rolf-c on 2007-05-01
There is an identified black screen bug with beryl + nvidia.

From [http://mepislovers.org/forums/archive/index.php/t-6649.html](http://mepislovers.org/forums/archive/index.php/t-6649.html) :
==========
There is a known bug (Black Window Bug) that afflicts NVidia video cards under Beryl. When you run out of video memory, the next screen opened will be black. Usually, this occurs when several windows are open at the same time. It seems that you have a severe case of this bug if it happens right away. How much memory does your card have?

Some folks have tweaked the Beryl settings, adjusting rendering to Copy, forcing AIGLX, etc. and have seen some improvement. You should go to the Beryl web site and search on "black window bug" and you'll see some recommended settings. Strangely, the bug wasn't too bad when I ran the MEPIS Live CD. It got much worse when I installed MEPIS 6.5 on the hard disk.

NVidia are aware of the issue, but haven't promised to fix it. It's possible that NVidia could come out with new drivers that do fix it, or maybe the Beryl team can find a workaround. But for now, we're out of luck. Keep in mind, however, that Beryl is still in the early beta stage of development, so there may be improvements to the performance down the road. But for now, those with NVidia cards should consider Beryl as a "work in progress."
=======
