### Post by PaulFXH on 2007-06-09
We have three desktop computers, two are set up to dual-boot Feisty with WinXP and the other has Vista with Feisty (Ubuntustudio) running as a VM in VMware Server.
On the first (older) computer, Feisty works very well with Beryl -- no problems whatsoever.

I then tried to install Beryl on Feisty (Ubuntustudio as VM) on the newest computer that runs with Vista.  The Beryl install really messed things up such that I couldn't even start Ubuntustudio.  I was finally able to get Feisty (u/studio) running again by disabling the Beryl autostart.

The third computer WAS operating also with Ubuntustudio (but as a dual boot with XP). However, the Beryl install also messed things up here in that the cube became completely white with no icons or anything else visible. The cube however was rotatable, although somewhat slowly, allowing me to see the Skydome picture (note that I always leave the Home partition unformatted to avoid losing my stuff).

So, I overwrote the Ubuntustudio root partition with a fresh install of Feisty. Then I re-installed Beryl which gave the exact problems I had with Ubuntustudio/Beryl on the same computer.
To get out of the problem I had on this computer, I had to re-install Feisty and resist trying to install Beryl.

I'd be grateful for any comments as to what might be going wrong here.
