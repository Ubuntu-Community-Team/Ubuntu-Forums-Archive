### Post by Zorael on 2008-04-16
Are you using the correct video drivers or did they get changed somewhere alone the road?

Please tell us what videocard you're using. Also, paste the contents of your **/etc/X11/xorg.conf** file, and the **terminal output** of typing the following command in a terminal.
```
$ compiz --replace &
```

Don't forget the ampersand (&) or things will go crazy if/when you close the terminal window.


You can also consider reinstalling Compiz. Upgrading is always a tenuous business. The graphical way of doing it would be to open Synaptic, find everything related to Compiz (just search for Compiz), rightclicking and picking Complete Removal. Then install.

The terminal way would be this.
```
$ sudo apt-get autoremove --purge compiz* libcompiz* libdeco* emerald* libemerald*
```

Then reinstall.

### Post by probinson_27 on 2008-04-17
I can also confirm that removing xserver-xgl from my system improved the performance of compiz.

I had to also update /etc/X11/xorg.conf to enable composite as follows

FROM

Section "Extensions"
        Option          "Composite"     "0"
EndSection

TO

Section "Extensions"
        Option          "Composite"     "1"
EndSection

Heya
I think that when you upgraded to hardy, some of the features are not installed on the new Hardy, therefore compiz cannot start because it wants to use some plugins that do not exist. You have a saved file with all your configurations in the file (remember to enable 'show hidden files')
/home/YOURUSERNAME/.config/compiz/compizconfig/Default.ini

you could either, go into ccsm and disable all the plugin options and then enable them back one by one to see which works or you could edit this Default.ini file. I think if you delete and/or edit this file to nothing, then all your settings will be reset :D
