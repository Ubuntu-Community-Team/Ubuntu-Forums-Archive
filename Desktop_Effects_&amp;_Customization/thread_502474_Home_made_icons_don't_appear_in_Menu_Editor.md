### Post by cogadh on 2007-07-16
I'm sure this has been posted before, but the board is giving me database errors when I try to search.

I've taken a few icons for Windows games and edited/resized them for use on my Ubuntu shortcuts, but I have encountered an odd problem. When I create a desktop shortcut or when I create an entry in the menu editor, none of the icons I have created are visible in the "Browse Icons" dialog. I have already browsed to the folder in my home directory where I store the icons, but the folder appears empty, even if I have hidden files shown (yet they are clearly there when I use the normal file browser). I have created them in both xpm and png format, matched the size to other default Ubuntu icons and tried adding my icons to the /usr/share/pixmaps directory, it doesn't seem to make a difference. 

Here's the really odd part; I created a desktop shortcut and couldn't change the icon, so I left it as the default. After I exited out of the Create Launcher dialog, I right-clicked on the newly created shortcut, selected Properties and tried to change the icon to one of my edited ones. This time I was able to see the icons I had edited and was able to use it. Just in case it was a fluke, I created a different desktop shortcut the same way and I was only able to change the icon if I created the shortcut first, then went back afterwards and changed it through the Properties dialog. 

Here's the problem though, what I really want to do is change the icons used in my Applications menu, and I can't find any way to do that. Any suggestions would be greatly appreciated.

---

### Post by cogadh on 2007-07-16
Nevermind, search just started working again and I found the answer. For those that may want to know, double-clicking in the file browser to open a directory prevents the icons from being displayed and selectable. Instead, use the Open button on the bottom of browse dialog, the icons that were always there will "magically" appear.
