    HorizSync       30.0 - 80.0
    VertRefresh     56.0 - 76.0
and you may want to try and remove the references to other resolutions other then 1280x1024, I think that worked for me when I had this issue.

### Post by heimo on 2007-05-19
May not make any difference, but put these to your xorg.conf (replace values on existing lines):
```
    HorizSync 30-83
    VertRefresh 50-76
```Specification for your monitor:
[http://h10010.www1.hp.com/wwpc/hk/en/ho/WF06b/1090681-1124275-1124303-1124303-1124303-12117476-25194623.html](http://h10010.www1.hp.com/wwpc/hk/en/ho/WF06b/1090681-1124275-1124303-1124303-1124303-12117476-25194623.html)

If that doesn't work, check your /var/log/Xorg.0.log file for any errors (EE) and warnings (WW). You could also try i810 driver instead of intel, you would have to install i810 driver, which will uninstall intel driver, so be sure to make backup of your xorg.conf so that in case your Xorg / GUI doesn't start, you can go back to old configuration. 
```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf_backup
sudo aptitude install xserver-xorg-video-i180
```

More instructions here:
[http://ubuntuforums.org/showpost.php?p=454217&postcount=1](http://ubuntuforums.org/showpost.php?p=454217&postcount=1)
