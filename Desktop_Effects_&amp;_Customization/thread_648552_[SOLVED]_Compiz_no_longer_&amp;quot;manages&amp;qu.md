### Post by mathenge on 2007-12-23
Niall, there's way too many things that can go wrong that can make compiz stop working properly. I had an issue with an upgraded device driver and you can see the thread of responses here: [http://ubuntuforums.org/showthread.php?t=648250](http://ubuntuforums.org/showthread.php?t=648250)

Have you tried opening a command prompt (terminal) and typing the following:

              compiz --replace

See if that helps to wake up compiz and force it to reset itself without closing anything.

If that doesn't work, can you tell me a little bit more about the type of computer this is? Better still, post the contents of the following two files:

/var/log/Xorg.0.log
/etc/X11/xorg.conf

Andrew.

### Post by niallporter on 2007-12-24
RESOLVED!

I went into Synaptic thinking maybe reinstalling all the compiz stuff might help.  I did a search for "compiz" and out of all the things in there the only compiz package that was installed was "compiz-core", I guess this is not enough to get compiz running properly.  I marked "compiz" itself for installation and let it install it's dependencies and now it works again.

I can't believe for the life of me that my parents would have managed to uninstall this so my best guess would be that some update must have broken compiz.  Thanks for the responses everyone, if anyone else is having this problem then my suggestion for the first thing to check would be to look in Synaptic, search for "compiz" and make sure compiz itself is installed.
