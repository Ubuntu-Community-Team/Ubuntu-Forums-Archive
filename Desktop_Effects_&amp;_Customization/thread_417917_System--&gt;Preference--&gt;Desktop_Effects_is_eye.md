### Post by AusIV4 on 2007-04-22
> **BoneKracker said:**
> I think until somebody comes up with a way to make them actually contribute to the usability of the interface, they are both a waste of resources.

I don't care if people love them, they just better not become mandatory until they are useful.

I'm a big fan of beryl, and it seems to be a common misconception that it's a waste of resources. Granted, beryl does take a little bit of memory and CPU, but being a composite window manager, it offloads window rendering to the GPU, so the effects really aren't very detrimental to your system's performance, unless you have your graphics drivers improperly configured.

As far as contributing to the usability of the interface, there are some great beryl plugins that I can barely go without now that I'm used to them.

1. The Desktop cube - while it may seem completely arbitrary, the cube helps me visualize virtual desktops. I never used virtual desktops until I had a physical representation to help me organize my windows.

2. Scale - Like OSX Expose, scale is nice if you have a lot going on at once. You can find the window you need quickly.

3. Put - I believe this is disabled by default, which is a shame in my opinion. Put lets you move your windows around using the Super key + the number pad. I can put all my windows where I want them and my hands never leave the keyboard.

4. Group and tab windows - I'm a student and I use tomboy to take notes. Frequently, I'll have a lot of notes that I don't want to close, but I don't need them cluttering my desktop. This plugin (from beryl-plugins-unsupported), lets me group my windows together and tab through them. I can easily get to the windows I use frequently, and the ones I need on occasion are just a few key presses away.

5. WIndow Switcher - Functionally, alt tab works exactly the same as it did before, but with beryl it becomes quite clear which window is being selected. I find that I get to the window I'm looking for a lot faster than I did before.

I also rather enjoy the less useful effects - it seems more pleasant when a window seems to have an origin, rather than just appearing all at once. These strictly eye candy features are admittedly useless, but as I've said, there are a lot of practical features in beryl as well. I agree that desktop effects should never be mandatory (Linux is all about choice, right?), but I do like that it's becoming very easy to use. I wouldn't mind seeing these effects as part of the default installation, but I think they should always be disable-able.
