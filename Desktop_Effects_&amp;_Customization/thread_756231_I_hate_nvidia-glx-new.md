### Post by Zorael on 2008-04-15
Make a habit of backing up your xorg.conf before doing such updates. :>

Have your friend set the dual monitors up again - I wouldn't know where to begin - and then we can help you troubleshoot what went wrong with the desktop effects. Likely the new driver itself isn't at fault, but rather the nvidia-xconfig configurator that parsed your xorg.conf file made some unwanted change someplace.


Ahem.

Install the **compizconfig-settings-manager** package, either via Synaptic, Adept Manager or in a terminal. It's always good to have.
```
$ sudo apt-get install compizconfig-settings-manager
```

Now, run the following command and paste the output error message here.
```
$ compiz --replace --loose-bindings &
```

### Post by erginemr on 2008-04-16
I don't know much about dual screen configuration as I am not using dual monitors, but from the looks of your current xorg.conf file, it seems you don't have GLX enabled, which is required for desktop effects to work.

1. First and always, back up your currect file with:
```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.twinview_working
```
You may need this file as a reference, if you lose your dual screen settings down the road later on.

2. To check if the 3-D GLX module is active or not, you can try the following command from the terminal:
```
glxinfo | grep -i direct
```
The answer should be "yes".

3. Please refer to this post: [http://ubuntuforums.org/showpost.php?p=4695502&postcount=5](http://ubuntuforums.org/showpost.php?p=4695502&postcount=5)

and add the lines in red to the corresponding sections (i.e. to Section "Device" and Section "Module"). 

However, it seems there is no Section "Module" at all. So, please add the following lines to the end of the file in addition to the two lines to be added to "Device":

```
Section "Module"
        Load "glx"
EndSection
```

Restart your X server and re-check the outcome of the above command in item #2.

4. If the above does not solve your Compiz problem, then give a try to ENVY at:
[http://www.albertomilone.com/nvidia_scripts1.html](http://www.albertomilone.com/nvidia_scripts1.html)

which will find and install the correct NVidia driver from the net, so you won't have to bother with nvidia-glx-* any more.

5. You can have a look at "nvidia-settings" any time with:
```
Alt+F2 -> gksudo nvidia-settings
```

which should have settings to adjust dual monitors with twinview. However, you should always back up your current working settings before any radical change with the command in item #1.
