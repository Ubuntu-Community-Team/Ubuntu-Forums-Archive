### Post by airtonix on 2007-06-15
**Firstly**

sexy. your getting there just few more refinements.....but the rest is in the realms of custom beryl plugins....and more extensiblity within the provisioning of certain aspectes between the windows managers and the plugins.

anyway...some tips: 
 make the left&right window borders virtually non-existant.
 use genie effect for minimise & maximise window animations.
 turn of wobbly windows.
 toggle 3d desktop to planar slide instead of 3d cube rotate.
 modify windowsKey+D to be showDesktop-plugin, tie this plugin to a corner of your screen.
 modify printscreen button to save screenshots as PNG's inside a folder of your choice, desktop is good since you will see them as they are created.
 add tile-windows-plugin to a screen-corner
 add scale-windows-plugin to a screen corner
 change the lat-tabe binding to the carousel application switching preview method (front row style)

for another aspect to window & workspace management try this : 
 install the beryl-wall-plugin

replace the window-scale-plugin (expose) with the wall-expo plugin on both keys and screencorners.
 you can zoom out and see all your desktops on a wall....it is possible to even use the programs on these workspaces.

now the question for real inovation....is : 
 what kind of animation methods can you suggest?
 should it be possible to specify a animation for every window type that beryl handles?
 would it even be possible to develop some sort of 3d transform language that reads xml and compiles binary transform plugins ? 


**Secondly**
gnome-panel theme is controlled by the gtk libs.....thus if you load up : 

system -> preferences -> theme 

and then click "customize".

the first pane is where you can choose the theme for all gtk ui control widgets. but since you are using the emereald window manager, the third tab you are proly used to seeing when using metacity window manager wont be there.

and yes i agree the name should reflect what it actually looks like....which is definitly not windows of any variant. its infact as you have stated often here, a reprodcution of yourMacLove.generateTangent( self.fragment(Math.mooPoly(System.LOLLERCOPTER)) )....
