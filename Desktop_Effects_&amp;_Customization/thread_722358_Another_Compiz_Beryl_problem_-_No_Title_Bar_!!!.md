### Post by Zorael on 2008-03-12
What version of Ubuntu are you running? Beryl doesn't work very well in Gutsy.

If you've updated from Feisty to Gutsy, you may just have added foreign repositories to your sources.list file (like Treviños'; something with tr3vin0s or such in the address) which leaves files that mess things up something fierce when you install Compiz from the Gutsy repositories.

So, in that case, first remove those entries from your Software Sources, or by manually editing /etc/apt/sources.list.

Then **completely remove/purge** your Compiz and Beryl installation. I'm not sure what the package names are, but I think you'll get them all if you do this in a console.
```
$ sudo apt-get remove --purge beryl* libberyl* compiz* libcompiz* emerald* libemerald* libdeco*
```

Then reinstall Compiz and Emerald - and again, make sure you remove/comment out any foreign repositories that might interfere.
```
$ sudo apt-get clean
$ sudo apt-get update
$ sudo apt-get install compiz compiz-fusion* compizconfig-settings-manager emerald*
```


Also, you could check [this thread](http://forum.compiz-fusion.org/showthread.php?t=6794) to see if there's something you missed when setting up your ATI driver. I have an Nvidia card myself, so I don't know much about them.

### Post by Zorael on 2008-03-13
[You can find Envy here.](http://albertomilone.com/nvidia_scripts1.html) It's a script written by a user, so it's not supported by Canonical per se, but it just might solve your problems. It can completely remove your installed driver too, if you at any point wish to revert to the driver from the official repositories, so it's reversible. Do note though that you will need to run the script and reinstall the driver again if you upgrade the kernel, but that's easy enough. Can be done from inside Gnome/KDE or from a terminal.

Clarification on "completely removing"/"purging"; it does not remove your personal settings stored in your home directory (in the .compiz directory underneath), but some other remnants stored in your system from earlier installations.

Whenever I install Compiz (*NOT* Beryl, it's old and icky), I...
[LIST=1]
[*]Install drivers, either from official repositories or with Envy. With Envy, I'm sure not to get any Nvidia kernel mismatch errors. If you want to guard yourself against it without Envy, just do **sudo apt-get remove _--purge_ nvidia-glx*** first, then install
[*]Make common changes to xorg.conf, in my case for Nvidia cards. Search the forums for these, or the Ubuntu wiki. Google is also your friend
[*]Remove *any* eyecandy or similar repositories from your Software Sources (or **/etc/apt/sources.list**, or files under **/etc/apt/sources.list.d/**). They'll just make you install non-supported versions of Compiz, which may or may not work, but I'd suggest you install the official Compiz first, then try other ones if you wish
[*]*Completely remove* your current installation of Compiz or Beryl, it's called Purge in Adept Manager if you run KDE. Or **sudo apt-get remove _--purge_ compiz* libcompiz* libdeco* emerald* libemerald* beryl* libberyl***
[*]Clean apt cache and update from sources, should be an option to Clean in Synaptic, not sure. Else just do **sudo apt-get clean** then **sudo apt-get update** in a terminal. You *must* have removed foreign repositories by this point
[*]Install Compiz *from the official repositories* (since it's tested to work), with all the plugins you want
[*]*Install Emerald* (package name **emerald**). You can always try to get gtk-window-decorator running afterwards, but it's good to have Emerald to fall back to. I use Kubuntu, and kde-window-decorator is buggy as heck. **Window borders will not show without a proper window decorator, and Emerald is the default one, so unless you modify the wrapper script or launch another manually after evoking Compiz you will not get any**. Also, Nvidia users might not see any without enabling AllowGLXWithComposite and AddARGBGLXVisuals in your xorg.conf, I'm not sure. I always do this when I set it up (earlier step).
[*]Reboot unless you already did after installing drivers, since it needs to reload the kernel
[*]Run **compiz --replace** in a terminal, so you can see what error messages it produces instead of just "Desktop effects could not be enabled", which doesn't tell us much about what went wrong
[*]Search these forums for the error message if you got any, to see if there's any known solutions for it
[*]Google the error message if you got any, to see if there's any known solutions for it
[*]Ask for help on forums if it did not work
[/LIST]

My suggestion is to install Envy even if you don't want to install drivers through it, because having the ability to completely remove the installed driver (of any kind) is handy, if you don't remember the **sudo apt-get remove _--purge_ nvidia-glx*** procedure. It's easier to get a script to do it.

Having a messy xorg.conf file (**/etc/X11/xorg.conf**) can also lead to some strange errors, but your mess may be needed for it to work with, say, multiple monitors. Heck if I know, but you could try resetting it anyway.

To reset it (and *be sure* to have made a backup of it first), enter **sudo dpkg-reconfigure -phigh xorg.conf** in a terminal. Then set it up again, either manually (by changing the Driver entry under Generic Video Device or whatever it's called), by entering **nvidia-xconfig** in a terminal (for Nvidia cards, don' know the ATI command), or by using Envy.
