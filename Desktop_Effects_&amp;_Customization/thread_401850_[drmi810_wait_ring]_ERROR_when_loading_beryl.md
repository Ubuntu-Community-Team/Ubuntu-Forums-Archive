### Post by clsgis on 2007-07-24
I suspect you're looking in the wrong place.  I am getting this i810_wait_ring error on a motherboard I'm using for a server.  gdm happens to be up, but I'm logged in through secure shell on another machine.  It happens when I run disk tests.  Badblocks -s.  Remember the i810 is not just a video controller, it's got the ATA disk controllers in it too.  Could be it happens when you start Beryl because launching Beryl is the most disk intensive thing you've run since booting.  This is a cheap little motherboard from a cheap little consumer desktop made a long time ago.  It was probably never tested with a 160 GB drive and this could be a motherboard hardware issue.
