### Post by 23meg on 2007-04-15
You may get various kinds of crashes. If only Beryl itself crashes, you'll have to restart it, and you'll  just lose the existing state of your windows. If the X server goes down because of a Beryl bug or an interference with another app (commonly 3D and video apps), you'll lose all unsaved work in all open applications. If your whole system hangs (unlikely but possible), you'll need to power down and restart, and again, you'll lose all unsaved work.

Also, if you misconfigure X when setting up Beryl, you may not get a graphical interface at all and be left with a command prompt.

### Post by maxamillion on 2007-04-15
Another side note: I have run beryl and compiz both rather extensively (i don't use either on a daily basis because the whole "3d user space" thing kinda annoys me) but have done so only for testing purposes and I have found that compiz is generally more stable but beryl's configuration manager is a very nice feature, so if you like tweaking things the easy way ... beryl is for you, but if you just like the default and want extra stability ... compiz might be more your flavor. Both projects apparently are supposed to be converging code soon and possibly merging back together so there won't be these issues of choosing because the result will (hopefully) have the best of both worlds.

Also, if you don't use gnome or kde (ie- xfce) i recommend you go the beryl route because compiz on xfce seems to be somewhat of a hack job and makes life a little harder.

.... just my two cents, take them for what they are worth. :)
