You're still talking about reinstalling the system....I think I've got a better solution:
Uncomment lines with "Extensions" section, and set "Composite" to "1".
In other words, replace
```
# Section "Extensions"
#        Option  "Composite" "0"
# EndSection
```
with
```
 Section "Extensions"
        Option  "Composite" "1"
 EndSection
```

And type
```
sudo apt-get install xserver-xgl
```
to install XGL you;re probably missing.
It worked for me, hope it will work for you:D

### Post by Arand on 2008-01-26
I was having the same problem, with ATI radeon 200M on an Acer TravelMate 2450.

What I did to solve it was:
(I am now starting from the point of a successfully installed 8.01 driver)

Reinstall the whole Compiz-Fusion as described in the 'Install Compiz Fusion'-section here: [https://help.ubuntu.com/community/CompositeManager/CompizFusionATI#head-958a1bad18942c1f683aba018f2d970c3f91d188](https://help.ubuntu.com/community/CompositeManager/CompizFusionATI#head-958a1bad18942c1f683aba018f2d970c3f91d188)
```
sudo apt-get install compizconfig-settings-manager compiz compiz-core compiz-fusion-plugins-main compiz-fusion-plugins-extra compiz-gnome compiz-plugins libcompizconfig-backend-gconf libcompizconfig0
```

Whitelist the fglrx driver in Compiz as described in the cchtml guide: [http://wiki.cchtml.com/index.php/Ubuntu_Gutsy_Installation_Guide#3D_desktop_effects](http://wiki.cchtml.com/index.php/Ubuntu_Gutsy_Installation_Guide#3D_desktop_effects)

I did not have to edit the PCIID:s as described there, but a X300 card might have to...

Then I further edited /usr/bin/compiz according to *smcintyre* in this thread: [http://ubuntuforums.org/showthread.php?t=585252](http://ubuntuforums.org/showthread.php?t=585252)

-Done-

You might want to try that.

PS: I would recommend against reinstalling the Xserver, since all that did was disable my internet connection, for some obscure reason...

And with the new AIGLX-supporting drivers you shouldn't need Xgl, right...?

- Arand
