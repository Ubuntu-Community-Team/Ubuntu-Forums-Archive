### Post by public_void on 2007-12-26
I had a similar problem with the Intel graphics on my laptop. This is going to require some editing of an important file, /etc/X11/xorg.conf. This has the configuration information for many of your devices, including your monitors. xorg.conf is important, if it contains errors your'll be left without a graphical display, just a command line. Therefore before you make any changes always backup the original file. If your unfamiliar with anything search the forum and google, so your prepared, better to be safe than sorry. 

I added a second  Device, Monitor and Screen section, and added the second screen to the ServerLayout section. Note the commented out line in the ServerLayout, that is the key line to place the second monitor. Its commented out because I'm currently only using my laptop screen. Also note the HorizSync and VertRefresh are set in the second Monitor section, this fixed the problem of the second monitor only displaying in 640x400 (I think this is an issue that effects Intel cards).

For specific options check the manual page, in the terminal
```

man xorg.conf

```or [http://www.x.org/archive/X11R6.8.0/doc/xorg.conf.5.html](http://www.x.org/archive/X11R6.8.0/doc/xorg.conf.5.html)

Hope this gets you started.

### Post by public_void on 2007-12-27
I noticed you have copied much of my xorg.conf. However when writing, I was lazy and copied the first Device section and appended a 2 to the identifier. The two screens then refer to the different  Device identifiers. In your xorg.conf both Screens refer to the same Device identifier. I'd suggest you change the identifier in the Device sections so they aren't similar, to prevent the mistake. Also you need to add the xinerama option to the ServerLayout. This allows your two monitors to act as one to allow, for example moving a windows from one to the other. Make sure you have xinerama installed, via synaptic or with the command apt-get install xinerama.

Each Device (and most other sections) have a unique identifier that is refer to by other sections.
```
Section "Device"
    Identifier    "Intel Graphics Controller One"
    Driver        "intel"
    BusID        "PCI:0:2:0"
    Screen 0
    Option          "MonitorLayout" "CRT,LFP"
    Option          "DevicePresence" "true"
    Option          "AllowGLXWithComposite" "1"
EndSection

Section "Device"
    Identifier    "Intel Graphics Controller Two"
    Driver        "intel"
    BusID        "PCI:0:2:0"
    Screen 1
    Option          "DevicePresence" "true"
    Option          "AllowGLXWithComposite" "1"
EndSection
```The two screens refer to the identifiers in the Device and Monitor sections.
```
Section "Screen"
        Identifier      "Default Screen"
        Device          "Intel Graphics Controller One"
        Monitor         "Generic Monitor"
        DefaultDepth    24

...

Section "Screen"
    Identifier    "Extra Screen"
    Device        "Intel Graphics Controller Two"
    Monitor        "Extra Monitor"
    DefaultDepth    24

```Add this option to the ServerLayout section to enable Xinerama.
```
        Option          "xinerama" "true"
```

Edit: Noticed the VertRefresh and HorizSync values are almost the same at mine. Make sure there correct for your monitor.
