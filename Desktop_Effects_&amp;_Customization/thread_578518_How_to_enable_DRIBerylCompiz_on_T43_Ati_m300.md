### Post by jimerickso on 2007-10-17
you need to have the xgl session run when you log into the x session. just logging into xgl will never work as i tried this several thousand times before figuring it out. place a copy of the xgl binary in /usr/share/xsessions. it should then run (as root in the background) when you log into your x session. if i remember correctly that is what i did when running feisty (your mileage may vary) but i have now switched to gutsy and all this is taken care of automatically with a wrapper when you install xgl. so you may want to switch to gutsy to avoid the hassle. keep in mind that the xgl session will run as root but there is no way around this as xgl will only run as root. i think. or just wait for the new ati fglrx driver.

### Post by jimerickso on 2007-10-18
did you install xgl on gutsy?

sudo apt-get install xserver-xgl

that should give you the xgl binary and wrapper script so it will run on the current x session. then just enable the the fglrx driver in the system > administration > restricted drivers manager. this will give you fglrx 8.37.6 which should run your card as the latest ati fglrx driver will not work with mobility cards. hope that helps!
