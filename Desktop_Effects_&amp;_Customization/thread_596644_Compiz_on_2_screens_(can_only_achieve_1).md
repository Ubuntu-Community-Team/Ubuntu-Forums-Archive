### Post by tokinux on 2007-10-29
I'm running Gutsy with fglrx drivers and a dual screen setup. However I can only enable the second screen with this startup script and adding an Xgl option to my login sessions:

```

#!/bin/sh
DISPLAY=:0.0
Xgl :0 -fullscreen -ac -accel xv:pbuffer -accel glx:pbuffer &

DISPLAY=:0.1
Xgl :1 -fullscreen -ac -accel xv:pbuffer -accel glx:pbuffer &

#for hibernate / restart menu
cookie="$(xauth -i nextract - :0 | cut -d ' ' -f 9)"
xauth -i add :1 . "$cookie"
xauth -i add :2 . "$cookie"

export DISPLAY=:0
exec dbus-launch --exit-with-session gnome-session &

export DISPLAY=:1
exec dbus-launch --exit-with-session gnome-session

```

If I login using my normal session, the second monitor remains blank. The mouse is able to travel to it yet it just appears as an "X".

If I login with my Xgl script, I have 2 separate X sessions running but I can only enable compiz on the primary monitor. If I run "compiz" command in a terminal on the second monitor, it states that xgl is not present.

Anyone with some input on this?
