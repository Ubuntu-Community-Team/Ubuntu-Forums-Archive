### Post by GFree678 on 2007-08-23
I have found the Avant Window Navigator, being my first experience with a dock application, to be a very nice task manager/launcher but with some usability issues.

My solitary issue with it is how it obscures the top of any maximized window I have open. A lot of the time this doesn't annoy me too much, but sometimes it does. The problem though is that there aren't many solutions available which have an elegant solution to the problem:

(a) I can set AWN to auto-hide, but then I have to move the mouse down to the bottom of the screen to bring it back, and I like having the ability to instantly check on the status of all running apps like a regular taskbar provides. Plus if I want to switch to a particular app, I'd prefer having AWN stay up for a while before hiding, but auto-hide doesn't seem to have much flexibility that I know of.

(b) I can set AWN to prevent maximized windows from being extended under it, but this is kinda ugly. AWN only takes the center portion of the screen, and so having the rest of the desktop showing and windows not utilizing available real estate is kinda dumb. Taskbars are normally thin enough not to be a problem, but AWN is huge in comparison.

My solution has been to write a script which toggles the running of the AWN application. I have it set to a button in my top panel so that when I click it, AWN disappears permanently until I click the button again, at which point it returns and stays there until the next time I need to remove it. Allows me to keep AWN alive when I want easy visual access to all the apps, while allowing me to remove it when needed for an extended time.

I still feel I'm missing something though - it's not elegant. Should I change my usage habits or something? How do other people use AWN effectively without it annoying them?
