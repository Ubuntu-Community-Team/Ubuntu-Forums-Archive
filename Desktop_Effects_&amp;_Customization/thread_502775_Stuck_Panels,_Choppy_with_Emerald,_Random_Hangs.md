Alright, so I'm pretty new to composite window managing, and I've racked up a number of questions, some specific to my situation and others more general, with my experiences.  I apologise if they've been addressed in previous threads, but I wasn't able to find any solutions to my problems.

With compiz fusion, however, only the top one is accessible--the other two are stuck hidden.  I can reset them with gconf-editor, but they go back to stuck pretty quickly.

The second issue I'm having is with choppiness.  I have a Compal HEL80 Notebook, with an Intel T7400 Processor, 2024 GB of DDR RAM, and an Nvidia GeForce Go 7600 with 256MB.  Without Emerald, everything is very smooth and pretty.  Once I install Emerald, however, even simple actions, like minimizing a window, are noticeably choppy.

The third issue, and perhaps the most annoying, is that once I enable compiz fusion, my system hangs about every five minutes.  This never happens when compiz fusion is not running.  Sometimes it resumes on its own, but usually I have to do a hard reboot.  Sometimes it lets me stop the X window manager with CTRL+ALT+BACKSPACE.  Sometimes it even lets me move the mouse.

One last side note: when I first installed compiz fusion, I had lots of problems getting the window borders to appear.  While I was reading about potential solutions to this problem, I read a lot about adding the "AddARGBGLXVisuals" line, as well as some others that escape me right now.  What finally worked was running:

```
sudo dpkg-reconfigure -phigh xserver-xorg
sudo nvidia-xconfig --add-argb-glx-visuals
```

This is confusing, however, as I read somewhere that this is only for the proprietary nVidia drivers.  I have nvidia-glx installed, and this fixed my problem.  Is there some sort of documentation on what this and other options in xorg.conf do?  Why don't the install scripts for compiz-fusion automatically include them if they're necessary for window borders to appear?
