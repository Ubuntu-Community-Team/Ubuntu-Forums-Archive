_Environment:_ 
Ubuntu 7.10, Gnome, Avant Window Manager, twinview (left monitor = primary; right monitor = secondary), nvidia, 4 virtual desktops

_Problem:_ 
When I click in the AWN on the icon of an open (already started) application (e.g. firefox), that is running on another (non-current) virtual desktop on the right monitor the following happens:
- the virtual desktop changes /swithes to the location of the running application (in this example firefox)
- the application (e.g.) firefox is **automatically moved from the right to the left monitor** (it does not happen the other way around: from left to right). In other words, the position of the window of the running application is changed from the right to the left monitor. This does not happen when I switch the virtual desktops by using keyboard or mouse (not AWN). If I click on the running application icon on AWN and the application is located on the current virtual desktop the window of the running application is not moved from right to left. It only happens when the running application is located on a non-current virtual desktop. 

_Question:_ 
How can it be avoided that the window is moved from the right to the left monitor when clicking on an open application icon on AWN?
