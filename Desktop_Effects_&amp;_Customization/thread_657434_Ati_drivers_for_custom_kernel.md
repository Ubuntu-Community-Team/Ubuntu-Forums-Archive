### Post by boban on 2008-01-04
I used to have custom kernel  with ati radeon 9550 and I didn't have any problems with fglrx (now I have nvidia). I'm not aware of any technical limitation, that would  forbid usage of fglrx  with custom kernel.

Do you know why fglrx driver is not working? Do you have any error output? Perhaps fglrx module isn't loaded? try lsmod | grep -i fglrx .  Or maybe its a problem with X.org configuration? If so try sudo dpkg-reconfigure xserver-xorg. 

And if this all will fail you can always try fglrx drivers directly from ATI website.

### Post by boban on 2008-01-04
OK... I think that you need to add information about refresh rates to monitor section. You should find them in manual. The syntax of this (this is for my monitor):

```


Section "Monitor"
	Identifier	"SyncMaster"
	Option		"DPMS"
	HorizSync	30-81
	VertRefresh	56-75
EndSection


```

EDIT:
And if it won't help, then post whole xorg.conf...
