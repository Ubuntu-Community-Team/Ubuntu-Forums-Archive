### Post by Espreon on 2007-07-30
Method 1 (try this 1st):

Do this in a terminal:

```
sudo apt-get install beryl beryl-core beryl-manager beryl-plugins beryl-plugins-data beryl-plugins-unsupported-data beryl-plugins-unsupported beryl settings beryl-vidcap emerald emerald-themes seom libseom libberyldecoration0 libberylsettings0-gconf libemeraldengine0 heliodor
```

After it is done installing goto: Applications>System Tools>Beryl Manager, if this worked the title bars on your windows should be a transparent red. Also try moving around a window, if it is wobbly and fluid it worked

If you have no window bars hit the ruby icon and select the window decorator to emerald (if you like to use metacity themes more change it to Heliodor).

If not and it shot back to Metacity, then hit the ruby icon and goto advance beryl settings and change everything with the options aiglx and indirect to aigx and indirect. Now hit the ruby icon again and select the wm to be Beryl.

If it is still not working then try method 2.

If you got the whiteness described in the post then try this:

```
sudo apt-get remove xserver-xorg-video-ati
```

(WHATEVER YOU DO DON'T RESTART YOUR COMP UNLESS NO GUI DOES NOT SCARE YOU!).

then enter ```
sudo apt-get install xserver-xorg-video-ati
```

Then hit CTRL-ALT-Backspace, log back in and then try running Beryl Manager again if it work congrats but if still no dice try method 2.

If Method 1 worked enter this in a terminal: ```
sudo apt-get remove closure-svn xserver-xgl
```
If you want an eyecandyish way of shutting off your computer then enter this instead:

```
sudo apt-get remove xserver-xgl
```

Method 2:

1st enter this in a terminal:

```
sudo apt-get install xorg-driver-fglrx
```

After installation hit: CTRL-ALT-Backspace, if you still have a GUI than congrats but if you get some crappy BSOD-like error message about X then your pretty much screwed (meaning no Beryl 4 you).

The Below is if you have no GUI after installing fglrx, if you still have a GUI goto the section below the word END.

If you got the error message shut-off your computer, and enter recovery mode (they second entry in the GRUB menu). When it stops starting up do this:

nano /etc/X11/xorg.conf

sudo is not needed since recovery mode is root.

Then look for fglrx and change it to ati, then hit CTRL-X, hit y and hit enter.

Restart your comp and enter this in a terminal:

```
wget http://albertomilone.com/ubuntu/nvidia/scripts/envy_0.9.7-0ubuntu3_all.deb
```

Enter this in a terminal: ```
sudo apt-get remove xorg-driver-fglrx
```

Then goto your home folder and double-click envy_0.97-0ubuntu3_all.deb then install it. Goto Applications>System Tools>Envy 

And chose the Option: Install ATI Driver. When it prompts you to configure your xorg.conf hit yes. After Envy is done hit: CTRL-ALT-Backspace if you have a GUI awesome! But if no GUI you are doomed to a life of a boring effectless desktop. There is some effects like snow and fire that come with Enlightenment (a wm that acts as a DE, look 4 it in Synaptic if you are doomed, but 1st we need 2 get your GUI back).

Now if you failed go back in recovery mode and enter: nano /etc/X11/xorg.conf and change fglrx to ati hit CTRL-X, hit Y and hit enter twice and reboot your comp and go for Enlightment (if you wana have some effects).

END

Now do this in a terminal:

```
gksudo gedit /usr/bin/startxgl.sh
```

Put this in:

```

#!/bin/sh
Xgl -fullscreen :1 -ac -br -accel glx:pbuffer -accel xv:pbuffer &
sleep 4  
export DISPLAY=:1 
cookie="$(xauth -i nextract - :0 | cut -d ' ' -f 9)"
xauth -i add :1 . "$cookie"
exec gnome-session

```

Then save and enter this in a terminal:

```
sudo chmod a+x /usr/local/bin/startxgl.sh
```

Then enter this in a terminal: ```
sudo mkdir -p /etc/X11/sessions
```
then this: ```
sudo gedit /etc/X11/sessions/xgl.desktop
```

Put this in the blank file:

```

[Desktop Entry]
Encoding=UTF-8
Name=Beryl+GNOME
Exec=/usr/local/bin/startxgl.sh
Icon=
Type=Application

```

Then save.

Now goto: System>Administration>Sessions

click add and for the name type Beryl Manager
for the command enter beryl-manager
hit ok.

Now log out and hit Sessions and hit Beryl+GNOME, log in, make it your default session if you wish, if done correctly the screen should open with a black color (not with a powerpoint-like slide transition) or it should look like steel wool. If it is garbled let me know. But if not congrats. But if there is a crappy GTK/Icon Theme download my GTK/XGL Decrapfier Script. (it is in my post)

Put it in your homefolder and goto: System>Administration>Sessions and hit add call it GTK/XGL Decrapifier and for the command enter(change yourhome to the name of your home folder: /home/yourhome/gtkxgldecrapifier.sh 
hit ok. log out and back in.

Now test Beryl out to see if it works congrats. If you get blown down by the big bad white screen look at this:

[http://64.233.169.104/search?q=cache:56KVWX0FmlwJ:wiki.beryl-project.org/wiki/Install_Beryl_on_Ubuntu_Feisty_with_XGL+ugly+gtk+themes+xgl&hl=en&ct=clnk&cd=1&gl=us&client=firefox-a](http://64.233.169.104/search?q=cache:56KVWX0FmlwJ:wiki.beryl-project.org/wiki/Install_Beryl_on_Ubuntu_Feisty_with_XGL+ugly+gtk+themes+xgl&hl=en&ct=clnk&cd=1&gl=us&client=firefox-a)

and look for White Screen.

### Post by dougfractal on 2007-07-30
XGL is old way

Ubuntu installation with AIGLX

The ati driver is already shipped with Ubuntu, so you won't have to download it. This driver is completely compatible with AIGLX, so if you want an accelerated desktop, you don't need to install XGL.

see
[https://help.ubuntu.com/community/RadeonDriver#head-93fc0a86162afbf71a5b84eff7da6c6a338656b1](https://help.ubuntu.com/community/RadeonDriver#head-93fc0a86162afbf71a5b84eff7da6c6a338656b1)
Accelerated 3D support (r300 series)

All these cards and derivatives have good 3D acceleration support

9500 / R300 based cards.
9600 / rv350 or rv360 based cards.
9700 / R300 based cards.
9800 / R350 or R360 based cards.
X300 / rv370 based cards.
X600 / rv380 based cards.
X700 / rv410 based cards.
X800 / R420 or R423 or R430 or R480 based cards.
X850 / R480 or R481 based cards.
X1050 / rv370 based cards.

RadeonDriver
The ATI Radeon open-source driver HowTo

This guide will show you how to use the Free, Open Source driver for many ATI graphics cards called "radeon". It will provide 2D and 3D acceleration in your video hardware. This driver might not be as fast as the fglrx driver for a few cards, but it is compatible with AIGLX and has better dual-head support.
