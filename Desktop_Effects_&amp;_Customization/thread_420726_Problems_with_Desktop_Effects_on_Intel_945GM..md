### Post by DizzyTech on 2007-06-06
Yes, **I've fixed the bug!**
[noob-friendly]

Edit your xorg.conf by running this in either a Terminal or by pressing Alt+F2:
```
gksudo gedit /etc/X11/xorg.conf
```

Now, look for a section named Device. It'll look like this:
```
Section "Device"
```

Right before the part where it says:
```
EndSection
```

Cut-and-paste this on a new line *before* EndSection:
```
        Option          "UseFBDev"              "true"
```

Click on the close button in the right-hand corner of the text editor window, and MAKE SURE YOU SAVE!!!  You can close your terminal window.

Now restart what is called the X-Server.  You can do this by pressing the key combination Ctrl+Alt+Backspace.  (The X-Server is the de-facto Linux standard for interfacing with input hardware and output hardware).

After restarting that X-Server, you will be taken back to your login window.  Go ahead and log back in.  Once you get to your desktop.  Start up a terminal, or press Alt+F2 to bring up a run application window.  Type in:
```
glxgears
```

Alright, so here's the moment of truth.  Resize the window, drag it around, have fun!  *It works!*

[/noob-friendly]

On the subject of the resolution, make sure you've purged the 915resolution package via Synaptic.  (It's "Completely Remove").  Make sure the i810 driver (not the intel) is installed.  Then install the 915resolution package.  If your xorg.conf is already setup for your resolution, simply restart X and it should work.  Then reinstall the intel driver (removing the i810).
