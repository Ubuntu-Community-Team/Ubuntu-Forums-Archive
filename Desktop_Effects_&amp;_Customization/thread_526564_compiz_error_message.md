```

mkdir ~/Xinfo

echo "$ubuntuname" > "$ubuntuname"X.info;

cat /proc/cpuinfo | egrep "model name|MHz"  >> "$ubuntuname"X.info

glxinfo | head -4 >> "$ubuntuname"X.info
xdpyinfo | egrep "version:|dimensions|depth of" >> "$ubuntuname"X.info
script -c " glxgears &  sleep 11 ;  pkill -15 glxgears"   
cat typescript | grep FPS >> "$ubuntuname"X.info
rm typescript
cp /etc/X11/xorg.conf ./   # graphics file
cp /var/log/Xorg.0.log ./
cp /etc/modprobe.d/blacklist ./
cp /etc/apt/sources.list ./
aptitude search ~imesa ~ixorg ~invidia ~iflgrx ~idbus ~icompiz -F "%20p %10v %10V %20d" > installed.txt
tar -czf "$ubuntuname"xinfo.tar.gz "$ubuntuname"X.info xorg.conf Xorg.0.log installed.txt blacklist sources.list
echo -en "\nCreated $PWD/"; echo -n "$ubuntuname"; echo "xinfo.tar.gz" ' | tee xinfo.sh
sed -i '1i#!/bin/bash' xinfo.sh

```

### Post by dougfractal on 2007-08-30
use firefox (gdebi) to install the package.

[http://ppa.dogfood.launchpad.net/amaranth/ubuntu/pool/main/c/compiz/compiz-core_0.5.2-0ubuntu3~ppa4_i386.deb](http://ppa.dogfood.launchpad.net/amaranth/ubuntu/pool/main/c/compiz/compiz-core_0.5.2-0ubuntu3~ppa4_i386.deb)

or
search "compiz" with
> sudo synaptic
and see if it gives any clues (properites)  
you could **force** an earlier version, or **hold** present version but you would have to remember to manually update it in a few weeks time.


I've just noticed that you are still using i810 instead of intel.  some people have problems with intel, but it's the newer driver and may be better for your board
1)activate sudo
> sudo echo "activate sudo"

2)copy & paste.
> sudo apt-get remove xserver-xorg-video-i810 nvidia-kernel-common
sudo apt-get install xserver-xorg-video-intel
sudo apt-get install --reinstall libgl1-mesa-dri libgl1-mesa-glx
sudo sed  -i 's/i810/intel/'   /etc/X11/xorg.conf   # change i810 to intel
**sudo sed  -i 's/Option "XAANoOffscreenPixmaps"/# Option "XAANoOffscreenPixmaps"/' ** /etc/X11/xorg.conf   # don't need XAANoOffscreenPixmaps


4) check
> grep i810 /etc/X11/xorg.conf     #   should have no result
grep XAANoOffscreenPixmaps /etc/X11/xorg.conf   #  should have a "#" at beginning


4)restart X
> sudo /etc/init.d/gdm restart


NOTE i had a missing  ' on last sed command
