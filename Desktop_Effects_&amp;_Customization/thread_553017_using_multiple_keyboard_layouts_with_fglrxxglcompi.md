### Post by Forlong on 2007-09-18
Hi,

that's a great post you made there. Very descriptive.

Alright let's focus on the problems:

1. Xgl is dreadful.
You should consider switching to the open radeon driver. That's the best thing you can do if you are not a gamer. I did it myself when Feisty came out. In it's kernel is a better driver included and I haven't used fglrx/Xgl since. And I'm happy (using a Radeon 9600 XT).

2. Xgl messes with the keyboard settings.
That's a known and annoying fact. Here's how I used to solve this:
(_disclaimer:_ I am not using Xgl since Edgy and I only used Beryl then, so don't expect any miracles but this *may* be of help)
This is the startscript (/usr/local/bin/startxgl.sh) I used:
```
#!/bin/sh
Xgl :1 -nolisten tcp -fullscreen -ac -br -accel xv:pbuffer -accel glx:pbuffer &
sleep 2
export DISPLAY=:1
cookie="$(xauth -i nextract - :0 | cut -d ' ' -f 9)"
xauth -i add :1 . "$cookie"
exec dbus-launch --exit-with-session gnome-session
```
(to everybody else reading this: do not use this, if you don't have any specific problems with Xgl)

3. Compiz Fusion needs too much resources on Xgl.
Try starting Compiz this way:
```
LD_PRELOAD=/usr/lib/fglrx/libGL.so.1.2.xlibmesa compiz --replace
```

I hope this helps.


P.S. You don't need to run GWD separately in a startscript, that's already taken care of by the compiz-wrapper of Amaranth's repository.
But I recommend doing the "First Steps" in my [How-To](http://forlong.blogage.de/article/2007/8/26/The-best-way-to-install-Compiz-Fusion-on-Ubuntu-Feisty) (not the first steps of the guide but the section later on).
And set up the cube as described [here](http://forlong.blogage.de/article/2007/8/29/How-to-set-up-Compiz-Fusion).
If you still can't use the cube afterwards with the keyboard, try middle-clicking on your desktop and move the cube. If this works for you, there's still a problem with your keyboard settings.

I figured out a huge part of the problem...there was a security update to the xorg-fglrx package in the repos a while back that didn't play well with my setup.

Well, I poked around a bit, because OOo worked fine for me before I had played with Envy to upgrade fglrx, uninstalled again using Envy, reinstalled via the repos (and upgraded with the security patch without realizing it), etc. Anyway, rolling back to the original fglrx version in the repos lets me use xgl/compiz with multiple keyboard layouts/languages, and OOo works fine.
