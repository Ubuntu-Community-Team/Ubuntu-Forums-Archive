### Post by Ub1476 on 2007-05-29
You need to run in a XGL session!:p

Also XGL only works with beryl version 0.2.0

```
beryl --version
```

Did you follow at guide to install beryl, or did you download from add/remove?

If you downloaded from add/remove XGL isn't installed, and you got beryl version 0.2.1..

So if you ask me: the cleanest way is to remove it then reinstall:

Here's guide to install XGL and beryl version 0.2.0

[Linky]("http://ubuntuguide.org/wiki/Ubuntu_Feisty#How_to_install_Beryl_.28ATI.29")

Scroll down to: Alternate method: Using closed source FGLRX drivers from ATI. and follow guide form there.

Should you have happened to install XGL and beryl version 0.2.0, you can choose to login to XGL from the log in screen.

Good luck:D
