### Post by live4fun on 2007-04-01
I turned out the problem is the false detection of my LCD Acer AL1916W. 

Adding the following option helped.
```

Option         "ModeValidation" "NoDFPNativeResolutionCheck"

```
Into the screen section of my xorg.conf resolved it. It allows also resolutions which couldn't be veryfied by a native resolution test.

Looks like the reason for this problem are LCD monitors returning wrong information.
[BugReport]("https://bugs.launchpad.net/ubuntu/+source/xorg/+bug/67369")

Others claim it is about the right combination of graphics card and monitor understanding each other.

I wounder why my pervious xorg.conf without this option worked and now after updating I suddenly need it.

No matter why, if this option can help anybody else as well I am happy and if anybody comes along and explains the technical issue in detail we can learn a lot.
