### Post by freebird54 on 2007-04-24
Here are some 'chunks' from my xorg.conf, working with a similar (?) ATI card.  I'll highlight some of the things I notice with **bold** type:

```
Section "Device"
	Identifier	"ATI Technologies Inc RV350 AS [Radeon 9550]"
	Driver		"ati"
[B]	Option		"EnablePageFlip" "true"
	Option		"RenderAccel" "true"
	Option		"ColorTiling" "on"
	Option		"AccelMethod" "XAA"
	Option		"XAANoOffscreenPixmaps" "on"
	#Option		"AGPFastWrite" "on"
	#Option		"AGPMode" "4"
	Option		"AGPSize" "32"[/B]
	BusID		"PCI:1:0:0"
EndSection

```

Most of those were auto-generated on install of Feisty.

```
Section "Monitor"
	Identifier	"Samsung 900IFT"
	Option		"DPMS"
[B]	HorizSync	28-100
	VertRefresh	50-160[/B]
EndSection
```

The Sync and Refresh rates must match YOUR monitor if you put them in!!  I couldn't get the correct resolutions till I specified the monitor correctly.


```
Section "ServerLayout"
**	Option          "AIGLX" "true"**
	Identifier	"Default Layout"
	Screen		"Default Screen"
	InputDevice	"Generic Keyboard"
	InputDevice	"Configured Mouse"
	InputDevice     "stylus"	"SendCoreEvents"
	InputDevice     "cursor"	"SendCoreEvents"
	InputDevice     "eraser"	"SendCoreEvents"
EndSection
```

The Desktop Effects that ship with Feisty are set up to run with AIGLX - can't hurt to turn it on!

```
[B]Section "Extensions"
	Option "Composite" "Enable"
EndSection[/B]
```

With the 'ati' driver, this can be enabled.  If you change to fglrx drivers, this must be DISabled.

Hope some of these help!

BTW - before you start making changes :

```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.backup_070424
```

to make a backup first!
