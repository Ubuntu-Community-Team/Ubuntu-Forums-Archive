### Post by luceerose on 2008-03-27
Step 1) Open your xorg.conf file for editing.
Do this by opening a terminal and typing;
sudo gedit /etc/X11/xorg.conf
(that is case sensitve, by the way)

Step 2) Once the xorg.conf file is open,
Find the line that says 'Virtual'
and change those two numbers to whatever resolution you typically use.

Step 3) Save & close file & exit from terminal

Step 4) Either reboot or restart the xserver.

### Post by luceerose on 2008-03-28
Sometimes there isn't already a 'Virtual' option in xorg.conf
You''ll have to put the section in.
It goes in the 'Display' subsection under Screen.
Here is an example, I,ve highlighted the appropriate section Green & the virtual option you need to put in is in Red;


Section "Screen"
	Identifier	"Default Screen"
	Device		"Configured Video Device"
	Monitor		"monitor1"
	Defaultdepth	24
[COLOR="SeaGreen"]	SubSection "Display"
		Depth	24
		[COLOR="Red"]Virtual	1400	1050[/COLOR]
		Modes		"1024x768@85"	"1024x768@75"	"832x624@75"	"1024x768@70"	"800x600@60"	"1024x768@60"	"800x600@85"	"1024x768@43"	"800x600@75"	"1152x864@75"	"800x600@72"	"1280x960@60"	"800x600@56"	"1280x1024@60"	"640x480@85"	"1400x1050@60"	"640x480@75"	"640x480@72"	"640x480@60"
	EndSubSection[/COLOR]
EndSection


My example is not a Widescreen mode, so don't put "1400  1050" 
You'll have to look up your resolution or know what you want it set to.
You should already have the appropriate subsection in there so, make sure you insert that RED LINE ONLY, don't change anything else.

EDIT: If you want, copy & paste your xorg.conf file here so we can look at it for you.

### Post by liquid circuit on 2008-03-28
Yea, it would be a big help if you posted your xorg.conf file so we can see what you have now.

When you start playing around with xorg.conf, its best if you know how to get yourself back to where you started, otherwise you may find yourself lost.

**First thing you want to do before you start changing stuff is back up your current xorg.conf in case anything goes wrong.**
```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.bak
```
*This copies your current xorg.conf file to a file named xorg.conf.bak, which will now be your backup configuration.*


**If something were to go horribly wrong** (which it probably will, so write this down on some paper... trust me), **to get your original configuration back** once you get back to the terminal (press CTRL-ALT-F1 if you get the fail safe warning), **you can run**
```
sudo cp /etc/X11/xorg.conf.bak /etc/X11/xorg.conf
```
*This copies your backup configuration file to the actual configuration file that is used, so you are back where you started.*


then reboot...
```
sudo shutdown -r 0
```
...or... restart the X server..
```
sudo /etc/init.d/gdm restart
```



So now that the insurance plan is out of the way, if you want to post your xorg.conf to the forum, type the following in a terminal...
```
gksudo gedit /etc/X11/xorg.conf
```
and copy and paste everything from that window into a post (CTRL-A to select everything in that window, CTRL-C to copy everything selected, then CTRL-V to paste everything that was copied).  This will help us figure out how you are currently configured, and what might be wrong.
