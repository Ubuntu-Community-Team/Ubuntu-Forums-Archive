### Post by Faud on 2007-10-04
[https://help.ubuntu.com/community/FixVideoResolutionHowto?highlight=%28resolution%29](https://help.ubuntu.com/community/FixVideoResolutionHowto?highlight=%28resolution%29)

That page has a howto with your video card and higher resoultions. Make sure that you have the latest drivers

Intel Graphics driver (i810) won't use high screen resolutions

There is a common issue with Intel graphics resulting in screen resolutions not being available even after adding them to the xorg.conf file. This happens with the i810 video driver, but the newer intel driver does not have this problem.
Install newer modesetting Intel video driver

Unfortunately the newer driver is only available from 6.10 onwards (Edgy, Feisty etc), but not in 6.06.1 (Dapper). If you have Edgy or above then you can install the newer intel video driver with:-

sudo apt-get install xserver-xorg-video-intel 

Then edit your /etc/X11/xorg.conf and change the

Driver "i810" 

to

Driver "intel" 

You can edit the xorg.conf from the command line using

sudo nano /etc/X11/xorg.conf 

Alternatively from withing the GUI using

ALT+F2 
gksudo gedit /etc/X11/xorg.conf

### Post by roscoetuff on 2007-10-06
Solution:

Okay... the fancy stuff didn't work. Guess what did?

First, turn-off the Desktop Effects
1) System
2) Preferences
3) Appearance
4) Visual Effects - set to "None"

Do the following:

1) System
2) Admin
3) Screens + Graphics
4) Select Model + Screen 
           Mine was mis-specified. Corrected that (Dell E172FP).
5) Then Check the Resolution
           Mine was set to 1600 x 1200. Selected 1280 x 1024

Immediate resolution. Vahalla has never been so happily welcomed!

Oh, and then I turned effects back on.... got a slight flicker of an attempt to shift back to the over-wide screen problem... big sigh of Uh-Oh... but then splashed back to the normal size specified here. Real relief.... without Rolaids. Yah... and that dates me, f'sure.

Give it a whirl. So the problem is incorrect detection.... and doesn't need the hard solution... just a menu-driven approach. Probably others assumed that this had already been tried? Imagine so.
