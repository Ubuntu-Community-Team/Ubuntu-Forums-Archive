### Post by freebird54 on 2007-04-16
OK - couple of things.  Beryl will only go if you are in an Xgl session.  When you ran fglrxinfo it appeared that you were not.  Do you start it up specifically each time you want to run Beryl, or is it supposed to do it automatically?

Also - there are a couple of items in your xorg that are not optimum.  I will post them here: (note the **bold** bits - they should be added)

```
[B]Section "ServerFlags"
        Option      "AIGLX" "off"
EndSection[/B]
```

```
Section "Device"
        Identifier  "aticonfig-Device[0]"
        Driver      "fglrx"
       ** Option      "UseFBDev" "true"**
        Option      "VideoOverlay" "on"
        Option      "OpenGLOverlay" "off"
EndSection
```

And the composite disable section is unnecessarily repeated in your xorg.conf - though that probably doesn't matter.

Another thing that can be tried is to change the Beryl options from right-clicking the red diamond.  One of them is 'copy' mode.  This is not optimum, but might get you going if you can't otherwise.  Make that change, Make sure that Beryl is set as windows manager, and reload manager (all from the right click menus).

Let me know how it goes with these!
