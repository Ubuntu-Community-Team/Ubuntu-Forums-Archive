### Post by RomeReactor on 2007-10-26
Do you use your PC with a second monitor, or a TV?

At this point I would suggest reconfiguring your xorg.conf, but if you *do* use a second monitor, I don't know if you'll have trouble getting the second one to work after that; I'm not very familiar with Intel cards. I recommend you wait for someone who actually has an Intel card in Gutsy to provide you with more information, before reconfiguring the display.

However, in case you do want to continue with the display reconfiguration, press CTRL+ALT+F1, log in if asked to, and enter:
```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.backup
```
to make a backup of your xorg.conf file. Then:
```
sudo /etc/init.d/gdm stop
```
and:
```
sudo dpkg-reconfigure xserver-xorg
```
accept all the defaults, except when asked which video driver to use, select **intel**; after that finishes, if your display doesn't start automatically:
```
sudo /etc/init.d/gdm start
```

If for any reason you want to restore the xorg backup, you can do so by entering:
```
sudo cp /etc/X11/xorg.conf.backup /etc/X11/xorg.conf
```
and restarting your display (CTRL+ALT+BACKSPACE), or reboot:
```
sudo reboot
```

After reading that article, that was written in Sep.  We are in Oct.  and mine still didn't work.  Here is the work around for that.  Press alt-F2 and in the box that comes up type:
```
SKIP_CHECKS=yes compiz
```  Then hit return.  Compiz should start up.  If it hangs or crashes, then you will need to hit alt+ctrl+backspace  and that should log you out.  If all works fine, then you can put it into a script like this:

```

#!/bin/sh
SKIP_CHECKS=yes compiz

```
Save it as: .startcompiz or something like that.  In a terminal in your home directory make the script executable:
```

chmod +x .startcompiz

```
Now in the menu **System -> Preference -> Sessions**  Add a new one, you will need to use the entire name:  /home/**username**/.startcompiz  You can call it what you want.  I know there are other ways of starting it all the time, however this way gives you an easy out if it ever hangs, you can just switch to another tty and delete or rename your .startcompiz file and it won't load at startup.  That is my workaround.

Shane
