### Post by slooksterpsv on 2015-09-19
If you're wanting to gear more towards Ubuntu Touch, you should learn QML, Qt, and/or C++. If you want to dabble around with learning languages and constructing programs, it depends on the language you want to use.

Start simple and build either upon it or learn from it and build something else.

A few great beginner console applications would be:
Ask the user for input of a number and check if the number is prime. --Seems very basic, but important
Read in two files and sort/merge them together. --Again simple, but the principles are important

GUI wise:
Build a text editor that outputs the file in a format you'd like and be sure it can read it back in that way. Example:
[quote=MyFile.myf]
Author: user
Date: today
Last Modified: xyz
Content:
All content for the file
[/quote]

Make a calculator. Start working with those simple items and move forward into bigger projects.

I have yet to meet a project that does not welcome new contributors.
Everyone was a beginner once...and will be again on a different project.

Find ANY upstream project you think is interesting, and join their mailing list (or whatever they use).
A good place to start is triaging and patching simple bugs for that project. It's a way to start contributing quickly and meet other project members, it familiarizes them with your skills and interests, and it helps you learn the codebase for the application.
From that starting point, after you have learned the codebase, the people, and the workflow, then you have many options to progress to more advanced and interesting contributions.

Some people discover, after trying it, that coding just isn't for them. They enjoy bugfixing, writing documentation, broadcasting release announcements, liason and packaging with downstream distributions, organizing get-togethers, and other non-coding tasks more. Those are all essential contributions, too.
