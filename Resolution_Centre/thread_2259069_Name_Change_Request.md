### Post by coffeecat on 2015-01-01
You may very well have typed the username KitchM in your Ubuntu One account, but you then you confounded the forum account creation process by unticking your name in your Ubuntu One Personal Data Request page. The system then had no choice but to use your email address as described here:

[http://ubuntuforums.org/showthread.php?t=2230004](http://ubuntuforums.org/showthread.php?t=2230004)

> If you un-tick your name in your Ubuntu One Personal Data Request page, the system uses the local part of the email to create a forum username.

Which sticky is linked from the link on the orange bar at the top of the screen helpfully labelled "Please read before SSO login".

I've changed your username to KitchM.

I do, however, see that something did go wrong. The system should have used only the local part of the email address, not most of it as it seems to have done with you. Which is very odd. I've not seen this before. There must be a bug. I'll bring this to the attention of those that can do something about SSO login procedures, which does not include the forum admins.
