### Post by aysiu on 2008-12-09
Today at work, I had someone ask me for help with a problem on her computer.

She said she had lost iTunes from her Start Menu and didn't know how to get it back. I double-checked with her that it was installed, and she said it was.

So I told her to click on My Computer and go to Local Disk C:\ and then click on Program Files and then iTunes.

Then I told her to drag the iTunes.exe file to the Start Menu. Before I could finish telling her to do that, she dragged it on top of the Start Menu button and then let go of the mouse.

I then told her to drag it again to the Start Menu and wait for the menu to expand, then keep holding on to the mouse button over All Programs and wait for that to expand and then let go once it's in the Start Menu where she wants it.

She was grateful to have it back.

I often see Windows power users here attempting to speak for "the average user" and proposing that the average user has no problems in Windows but would be baffled by Linux. I don't see that at all. My co-worker is not an idiot. I run into people like her all the time in various workplaces or through acquaintanceships or kinship. She is the average user, and computers to average users of all kinds are a mystical thing. None of it is intuitive to them.

I as a power user think it's "obvious" that getting something back into the Start Menu would be finding the location of the executable file and then dragging (without letting go of the mouse button) to the Start Menu, but why is that intuitive? Think about it. How would an average user know to do this?

Among the average users I know (not the power users who pretend to be average users and try to use reverse elitism to distance themselves from Linux "geeks"), there is rarely if ever any kind of basic understanding of how things work on the surface. I'm not talking about learning programming or bits and bytes. I'm talking about on the surface here. To my co-worker in the above example, everything is a step. Do this step. Do the next step. She doesn't think, "Oh, the executable file lives here. If I drag it to this other place, a shortcut icon will be created that will link back to the executable."

I see this a lot with people writing down instructions. Every time I've had to write down instructions for how to do something on a computer, it's because I had absolutely no concept of what was happening. Every time I watched someone do something and then was able to replicate the actions without writing down notes, it's because I understood the basic approach.

For example, when I first learned how to do a mail merge, I didn't have to take any notes, because I understand the basic concept: this Word document has the letter, and that Excel document has all the stuff that gets merged in (name, address, gender of recipient).

But when I learned pivot tables in Excel, I had to take furious notes, and even after I took notes, I still couldn't replicate what I saw in the lesson, because I had no concept of how the tables related to the data. It was like magic to me. I saw the end result. That's all I saw.

So let's please stop all the bull excrement about how Windows is intuitive and easy for "average users." What you as a Windows power user think of as intuitive is not intuitive at all to the average users you try to speak on behalf of.

### Post by Giant Speck on 2008-12-09
And then she switched to Ubuntu and all was magical...

Anyway, no operating system is intuitive enough for the average user.  You described how Windows isn't intuitive or user-friendly by demonstrating a simple problem with a difficult solution.

The same problem has a just as difficult solution in Ubuntu, for example.  Say that, for some reason, a program is missing from the Applications menu on the top panel.  You right-click the Applications menu and select "Edit Menus."  Then you go to menu that the program would most likely be under.  For convention, let's just say that the program is iTunes, despite the fact that iTunes doesn't have a Linux version.  Sometimes, if you are lucky, the program will be in that menu, but will be unchecked.  All you have to do is put a check in the box next to it and it will appear in the Applications for now on.

However, if it is not there, you need to click "New Item," type in the name of the program, and its command.  If you don't know the name of the program, you can click Browse to find it.  [COLOR="DarkRed"]Up until this point, Ubuntu is much more intuitive than Windows.  However, with Windows, program executables are located in Program Files.  In Ubuntu, however, they are found in /usr/bin. That really doesn't scream "this folder has programs in it!" to an average user.[/COLOR]

My point is, that if you present the same problem in both operating systems, one may seem more intuitive than the other but then backfires on itself by presenting a process that is less intuitive than it made itself out to be.

Now, if the problem was executing the program, Ubuntu would have been much more intuitive than Windows, because all the user has to do is press Alt-F2 and then type in the name of the programs.  They would have typed "iTunes" and it would have shown up instantly.  That isn't so easy in Windows.

### Post by tsali on 2008-12-13
I don't know...I think the line between USER and POWER USER can be fuzzy, but for the most part, tasks like adding iTunes to the Start menu are wading into that blurry pool.

A basic user can be counted on to learn basics like mouse navigation and clicking icons to get the result they want.

Any task that takes them into changing preferences or making customizations outside of a "wizard" dialog puts them across that fuzzy line of "power user". Because Linux users appreciate the ability of Linux to be easily customized, they tend to view these types of administrative tasks as living in user space as well...but they aren't really.

The intuitive elements of Windows happen to be shared with Macs and most DEs for linux...the most basic of which is the concept that pointing a mouse and clicking some object with have some type of effect.

Having introduced Ubuntu to several senior customers, the concept of windows was immediately apparent to them, however, the SKILL of clicking, double-clicking and click-drag took them a long time to master. Once they got these basic skills down, they are ready to explore many more options.

Contrary to Apple's model, my seniors had no trouble mastering multi-button mice with click wheels.

Just my opinion.
