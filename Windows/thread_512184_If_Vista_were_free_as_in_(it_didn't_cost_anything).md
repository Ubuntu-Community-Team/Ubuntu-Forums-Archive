### Post by cmat on 2007-08-17
About super fetch. Ubuntu does cache in RAM, however it's not at startup. 

[QUOTE=http://ubuntu.wordpress.com/2005/10/07/memory-swap-management/]

On a linux system, the memory is used to &#8220;cache&#8221; data that is used by the CPU. The idea behind caching is that it takes longer for your CPU to access data on the hard drive than it does to access data that is present in the main memory. So caching using the main memory effectively speeds up the system. On a windows system, there is no such optimization, so free memory is wasted as it does not get used.[/QUOTE]

The article is a bit out of date, since Vista now caches more effectively. But it's fundamentally the same principle, using unused RAm to reduce access on the drive. My computer is using 33% of it's RAM for applications and the rest is cache right now. However you need to run an application first, after that it's cached. On Vista it's at start-up. What ever is better I guess it's up to the user's preference.
