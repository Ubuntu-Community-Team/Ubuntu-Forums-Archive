### Post by Dr. C on 2006-11-20
and Novell then distributes the resulting code under the LGPL. If there is Microsoft patented code being distributed by Novell then there are two possibilities.

1) Microsoft gave Novell permission to distribute this code under the LGPL
2) Microsoft did not give Novell permission to distribute this code under the LGPL

IANAL, but I fail to see the legal risk here. In the first case how can Microsoft sue for patent infringement anyone who distributes the code under the LGPL, if they agreed to the code distribution, under the LGPL in the first place. The second case is more interesting they are aware that Novell is infringing and do nothing about it. This leads to the defense that Microsoft has abandoned the patents in question by turning a blind eye to the infringement by Novell, effectively placing the Microsoft patents in the public domain. Furthermore all the FUD spewed out by Balmer could be considered evidence that Microsoft is aware of the patent infringement and doing nothing about it. 

The choice in my opinion is between a GPL/LGPL frying pan for Microsoft Patents (case 1), or a Public Domain fire for Microsoft Patents (case 2).
