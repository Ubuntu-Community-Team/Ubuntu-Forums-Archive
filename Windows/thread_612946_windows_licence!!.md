### Post by igknighted on 2007-11-15
> **zakeen said:**
> I want to install XP with MVware. I have a recovery disk with my XP on it and Im thinking there is no chance in getting that to work. So what Im thinking is, how is it possible for me to install XP with someone elses XP CD and use my licence. Is this legal?

Be aware that technically speaking this is a EULA violation.  The license you have with an OEM install is only for that hardware, and running in a VM is technically "virtualized hardware", not your physical hardware, so it doesn't count.

That said, no one will likely ever say anything about it, so I wouldn't worry about it if you did decide to do that.  Just pointing out that *technically* its illegal.

For XP, this is debatable.  I don't know that it has been tested in court, until it is your view expressed here is just an opinion.

There is a good argument that the XP software has been licensed for use on one piece of physical hardware. The XP license agreement does not recognise the possibility of virtual hardware - so has not specified disallowed a virtualisation layer between the (licensed) physical hardware and itself. 

Multiple copies, use on other machines - these may well be breaches of the license agreement, but I'm not going to agree that "virtualised hardware" is a breach of the license agreement. 

I have stated this to an MS employee and been given a new activation code - this person asked "am I using this on the original machine?": "yes" and "how many machines would I use the software on?":  "one" 

I'm happy that I am completely within the bounds of the license agreement.
