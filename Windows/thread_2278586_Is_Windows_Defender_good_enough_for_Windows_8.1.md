### Post by buzzingrobot on 2015-06-24
No definitive answer can be given to that "good enough?" question.  It depends on user behavior. Anecdotal experience from other people is useless because we all behave differently.
Let's assume that all anti-virus products provide identical signatures and update at the same time.  For the sake of argument. Then, what about malware attacks, phishing, blocking known malevolent URL's, etc.? Exposure to those risks typically begins with the user taking some unwise action. E.g., software can try to stop a malware download triggered by clicking on a link in a phishing exploit, but we should not depend on it.  If we aren't naïve enough to click on those links, we won't have those problems.

Scores in test suites are useful if we know what exactly they test and measure.  Few of us know those things. If a product uses techniques that a test does not measure, then that test will give it a score that does not reflect its actual usefulness.

Running any kind of anti-virus/anti-malware software seems several orders of magnitude better than not running any at all, especially for naïve or unsophisticated users who can't, or won't, think twice when they're at the keyboard.

That's especially true if they're someone who does things like disable updates, keep outdated Flash around, enable Java browser plugins, and turn off the firewall. (Or, open mail that's tagged as spam to 'see what's there'.  Why do people do that?)

I've been running a Win10 test machine using only Defender.  No problems there. I intend to use only Defender after the final release if/until it fails. OTOH, my habits are reasonably secure and I tend to confine my browsing to a small number of the same sites.

### Post by buzzingrobot on 2015-06-24
> **The_Forgotten_King said:**
> If you are browsing safely, yes.
If you are going to getmeavirus.com every day then no.


To be honest, I really don't think we can make generalizations like that. Way too many variables.  To accurately determine if Product X will defend against Threat A better than Product Z, we'd need to run identical tests on identical hardware and software, against a specific threat.  The businesses that say they test AV products say they do all that.  Even if they do, few of us have the skills needed to translate their raw statistics into something that we can use when we make buying decisions. 

I tend to think AV technology isn't magic and that any current popular product will probably use much the same techniques to deliver much the same protection. The things to look at would be speed and frequency of signature updates and speed of reaction to zero-day threats.  Products can be tweaked to score high (or low) in tests, too.
