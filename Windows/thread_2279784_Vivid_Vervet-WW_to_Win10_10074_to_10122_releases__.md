I when testing less then a week Windows 10:  A heavy weight O/S (3.6GB installer).  I created an iso image of Win Ten 10074 release under Vivid Vervet using mkusb.  I was not able to install it (not seen properly in the BIOS or in boot sequence) and force to create a DVD that had the exact amount of files.  The installation took me about 40 minutes.  I rarely seen an installer with a visual rendering so high. 


Lost visual rendering quality:  Win 10 is default to a lower resolution 1024 x 768 while my HD monitor (1920 x 1080) is set default under Ubuntu (no possibility to downgrade the resolution since Lucid Lynx).  When I set it back to it's default setting, the visual rendering on 1080p web sites was less then under Ubuntu.  When to Intel, install **Intel Driver Update Utility** and was able to update my graphic card.  Running the 10074 drivers, I was mot able to update my Chip-set.  After this operation, I gain a lot on visual rendering but a notch under Vivid.


Win 10 settings were hard to find, nothing looks the same.  When I finally set them my own way, I discover rapidly that I was not allow to do so.  On shutdown, the next morning, my settings were reset to default on a light blue screen saying Preparing Windows.  I reboot twice to recover the task-bar settings that I created.  Face many unstable issues that make me say that Vivid cycle was a piece of cake...  


Also, many usefulness applications, my HDD was noisy, an on-bias sub-woofer compare to Ubuntu noise free must of the time.  On the 10122 upgrade (it was a bit too much), Microsoft was scanning my HDD about every 30 minutes and a empty terminal was shown each time for about 3 seconds.  I came back to Ubuntu very easily the day after.
