### Post by 3rdalbum on 2007-05-30
#10 - There's a reason why you should unmount filesystems before fixing damage to them. It was possible to repair mounted drives on Mac OS 9, and this hastened directory corruption. Bad, bad idea Microsoft.

#7 - Will either not dramatically help with this, or will slow the system down to a crawl.

#5 - Address Space Mode Randomisation. Good idea. Pity it doesn't seem to work in Vista.

#4 - Maybe I don't quite understand what's being talked about here, but it seems like Windows is the last operating system to get this.

Many of the others were a bit "meh". But that number 10; automatically fixing filesystems - it's typical of the Windows Attitude To Security: Limit threats once they start running on your system. Rather than automatically fix filesystem damage, why not make a filesystem that is reliable enough to stay up for years?
