### Post by FoolsGold_MKII on 2007-06-29
I noticed it recently as I was prepared to convert back to Linux. I needed to create a partition to install Ubuntu, but I didn't have any partition modification tools available, and I wasn't certain if gparted on the LiveCD could resize an NTFS partition.

However, it seems that in Vista the disk management application has two new functions: Expand/Shrink Partition. This made it easy to reduce the size of a partition which was very large but not particularly full, allowing Ubuntu to create some new Linux partitions in the freed space. I don't recall see this partition size manipulation capability in XP.

So there you go.

### Post by kamaboko on 2007-07-19
> **saulgoode said:**
> Even if RAM is available at no cost, more memory usage means slower overall processing. A program that is double the footprint of another will not only take twice as long to load, but will leave less memory available for caching disk activity.

Huh?  Vista caches all or most of the RAM.  It recognizes a pattern to my daily activities and caches the RAM for those programs, therefore it runs fast as hell.  It's certainly faster than my Ubuntu install.  For instance, Office 2007 Word, PP, and Excel (all of which I use often) open w/in one second.  No joke.  Lightning fast. The same goes for Nero and a host of other programs I use.  

Here, this is a better explanation:

SuperFetch works like this: Vista will load anything it needs to load in memory first, and any applications you manually run will also be loaded into memory. Any memory left over is fair game to be used by SuperFetch as a cache of frequently used pages. Vista keeps track of what memory pages are frequently requested and what files they are tied to, and based on that data SuperFetch will populate as much free memory as it can with pages it believes you will need in the future. This data contains both frequency and temporal history, so not only how often but when you run these applications will influence what SuperFetch does at any given time.
