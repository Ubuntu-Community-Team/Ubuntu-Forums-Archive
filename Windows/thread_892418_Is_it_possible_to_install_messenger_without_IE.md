### Post by thumper13 on 2008-08-26
Now I'm a little rusty on my MSN skills (I worked for Verizon's MSN team back about a year ago), but I do recall that MSN only looks for the .dll files (I think) from Internet Explorer. I am certain that Windows Live Messenger requires the IE7 files, not IE6, any less will not do. 

To fix the issue, there are 2 paths a padawon can choose:

a) Install IE7. It is available from MS's site (I was there today, and it's there, in the flesh). The cons to this are that a) IE is not good at all. and b) Windows Genuine checks. If you're running a legal legitimate copy of windows, this might be the best option, and install Firefox, Opera, etc. along side it. Since you're legit, there's no problem with the Genuine checks either.

b) Find the missing .dll's IE7 installs and copy them to the correct directories. This method used to work, I'm sure of that, but as I said, I'm a little rusty, and they may have fixed this. Also, without some form of IE, you will fail the Genuine checks.
There may be ways around it, I'm not sure, and I'm sure it's quite illegal. Normally, on the phone, if someone failed the Genuine checks we told them to call MS, so my support ended.

In this case, I'd also advise not to run the System Restore.. Seems like there may be more files missing than just IE files if it fails to reinstall.. It's like removing someone's heart, and then replacing it when you realize the person needs it to live.. IE is quite an important part to Windows, and theres no sense beating a dead horse, she still won't run.. So it's off to the glue factory with this install.

Hope this helps you a bit.
