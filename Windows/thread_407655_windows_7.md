### Post by Yossarian on 2007-04-12
There are actually two distinct operating systems sold as Windows for IBM compatible PCs:

DOS-based: Windows 1, 2, 3, 9X/ME (aka Windows 4)
NT: Windows NT 3, 4, 2000/XP (aka Windows 5), Vista (aka Windows 6)

NT started with version 3 as it came out at the same time as the DOS-based Windows 3.

### Post by Rhapsody on 2007-04-13
**_Microsoft Windows History Lesson_**

First you have Windows 1.0, 2.0, and 3.0. No one really cares about them, so let's skip over the lot.

Then there's Windows 3.1, a kind of 'multimedia bug-fix' to give Windows 3.0 what qualified as 'multimedia' back in the early 90s. Then came Windows 3.11 as a collection of minor bug-fix to that (which is where I actually started, with Windows for Workgroups 3.11).

Next was Microsoft's proper breakthrough to market dominance, Windows 95. This was commonly known as Windows 4.0 before Microsoft decided to use years for versioning, so that's what we can call it.

Microsoft then came out with a series of small advances, these being Windows 98 (Windows 4.1), Windows 98 Second Edition (Windows 4.11), and Windows Me (Windows 4.2). After this, the ageing Windows/386 architecture was killed off in favour of the newer Windows NT kernel, and not before its time either.

This only left Windows NT, which had advanced through versions 3.1 (it started at the same version as the newest 'regular' Windows of the time), 3.5, 3.51, 4.0, and 2000 (previously known as 'NT 5.0') by this time. Since it was now the only Windows kernel left, it could now be called just 'Windows', which Microsoft started doing.

So Microsoft advanced 'new' Windows slightly with Windows XP, which I sometimes referred to as 'Windows NT 5.1', or just 'Windows 5.1' now. They also split off the server version of Windows (which were previously released alongside their respective versions of Windows NT) and gave them different names than the 'regular' versions. This resulted in Windows Server 2003, which we can call 'Windows 5.2'.

Then there's Vista. Given the rather small advances, I was calling this 'Windows 5.3', but Microsoft is determined to salvage something from the protracted development and convince people that they didn't screw up *that* badly, so they're calling it 'Windows 6' instead.

So finally, that's why we're at 'Windows 7' with the current development version. This is actually rather curious, since Microsoft has always referred to the next version of Windows by its codename (Chicago for Windows 95, Whistler for Windows XP, Longhorn for Windows Vista) but this one's a number. Is it not the same as Vienna? Are Microsoft going back to version numbers? Or is Microsoft just trying to do things a bit differently this time? No one can be sure for now.

That concludes my rather long-winded explanation of why we're at Windows 7 right now.
