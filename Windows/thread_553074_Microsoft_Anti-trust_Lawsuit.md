This will allow other vendors the oppertunity to create products that work seamlessly with windows stuff. It would be something in the lines of SAMBA. Instead of reverse engineering the protocols used by windows for networking/file systems MS would have to provide the specifications on how to access these services.

I don't see this as then having to supply code but merely publish the standards documentation. Kinda similair to the documentation AMD/ATI is now making available to the open source community. Like MIPS said, they are not obliged to open their code at all. They are obliged to open their APIs and standards, so that people can use WMA, DOC, XLS seamlessly with competing products. 

They are also obliged to unbundle all the crap that they say is impossible to do (IE, WMP) so that people can choose what they like.

This means nothing for US Windows users, since they are not obligated to do anything here.

For Linux and BSD, the only real upside is that you won't get those weird formatting issues with Microsoft standarded documents. AFAIK, they are not being forced to show the code.
