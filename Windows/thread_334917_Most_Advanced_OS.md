### Post by Sunnz on 2007-01-14
I find "add/remove program" easiest and quickest to use. I almost never use Synaptic, since if I find specific programs in "add/rem", I just use aptitude.

Anyway, I don't think it is probable to copy how programs are installed in OSX... I don't think it is very practical neither. There are so many flavours of Linux that copying the Mac method is just going to cause problems. The current distro-organised repo are the best for Linux IMHO.

On the topic of getting people to switch, I actually had my own idea - bridging the gap between Windows and FOSS.

The biggest hesitation I have experienced from switching to Linux so far isn't the concern if one can use the OS - Gnome/KDE/XFCE is doing a great job, they are different but it is not like they are completely foreign to your average Windows user - the X button close down the window and the SQUARE button maximises/minimises the window.

The thing is, people are worrying about the software that runs on top of the OS. There are really good replacements on Linux, like Openoffice to MS Office. But unless the switcher isn't doing anything critical at the moment, it would be a nightmare for them to just switch each Windows-only to Linux equivalent, each app needs to be unlearn and relearn.

So I think the solution should be, to provide a CD that contains heaps of FOSS software that runs on Windows, but specialised to teach someone to be familiar with *nix. I know of FOSS CDs that merely contains installed of say firefox, OOo, etc. But I think that wouldn't teach the user much about *nix. A 'bridging the gap' CD shall have one giant installer that installs Cygwin, a *nix style CLI on Windows, and have FOSS apps installed in respective *nix hierarchy under Cygwin, like /usr/bin/firefox or something, so the user can start using bash and understand where programs are installed under a nix system.

This way the user may still use what they are working on at the moment, maybe they were writhing an article in MS Word, they can finish that article and start their new one under Cygwin using OOo, and perhaps get used to it. And when they are finally using FOSS only apps on Windows, then maybe they are ready to actually switch.

[URL="http://www.fsf.org/licensing/licenses/index_html#GPLIncompatibleLicenses"]The **Original BSD license** is a GPL-Incompatible, but nevertheless, *Free Software* License.
[/URL]
I think there is a huge misperception about GPL and BSD Licenses.

The GPL license by itself, is actually more restrictive than BSD - it does not grant the licensee more freedom than BSD, it *forces* the licensee to give the same freedom they got when they re-distrubute GPL software. That's the whole point of GPL, to force free software to everyone; whereas BSD License allows you to keep the source to yourself - it is a bit more permissive in the hand of its licensee.

Note that I am not saying that GPL is bad, but just stating how it works.
