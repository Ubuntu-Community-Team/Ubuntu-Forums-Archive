### Post by saracen on 2006-09-30
Microsoft is release a beta version of a piece of software they call [Windows Mobile Device Center]("http://tech.cybernetnews.com/2006/09/29/windows-mobile-device-center-beta-available-soon/").  Basically it's ActiveSync redone and is meant to be a one-stop-shop for all your mobile device needs.  This is one area where I believe Ubuntu needs to focus a lot of attention on because if they truly want to gain a larger market share, then there needs to be seamless integration with mobile devices.  Everyone has a at least one mobile device today - be it a cell phone, a PDA, or a Blackberry.  And these devices will only become more ubiquitous in the future.

I'm hoping that the devs will put some love into this for the next development cycle so that Ubuntu can make even more headway into the market.

### Post by BoneKracker on 2006-09-30
I would love to have that too.  But I think this should be a 3rd-pary app, not ubuntu-specific code.

It's middleware, and by definition of a modular nature.  It should be "many-to-many" (with API's for all major mobile device categories, but also portable to many OS platforms).

So this is something to find out there in FOSS-space and integrate into the distribution, not something for the ubuntu developers to code themselves.

Right now, both Ubuntu and Kubuntu include PIM suites that include mobile device synchronization (albeit less complete or robust than what you want).  

I think the need is for "upstream" work.  Or maybe there's a network-centric answer (like something out of Google Labs).

### Post by BoneKracker on 2006-10-01
I didn't mean we have to "wait" idly for upstream to just randomly happen upon the improvements our community desires.  

What I meant was that we should seize upon a promising-looking external product and build (or get other to build) on it.  It's open source.  They would almost certainly incorporate any well-engineered improvements.

The important point I was trying to make is that a primary requirement for the success of any middleware product is adoption.  It's driven by network-effect economics.  The number of products that will produce interfaces via a given middleware's API is driven by the number of products on the other side of that middleware.

So I'm with you as long as whatever app is developed uses API's / interfaces / middleware that's intended for a broader user base than just Ubuntu.
