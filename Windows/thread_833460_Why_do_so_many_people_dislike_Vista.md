### Post by BlueSkyNIS on 2008-06-23
**< Very high system requirements >**
	* ***Basic services (processes) consume just too much memory (comparing to XP).*** In my opinion, an OS should consume as much as low memory possible and leave the rest of RAM for user needs. Vista uses about 1GB of RAM right after fresh install without any application started. Windows XP will run quite happily with only 512MB. I am aware of Vista's aggressive cache memory management, but with only 512MB or 1GB of memory it is pretty useless and even unproductive feature. The consequence: no free memory for user apps and much HD trashing. 
	* ***More services (processes) are running automatically by default, therefore eating more RAM.*** The number of automatically started services has increased by 2 (Home) &#8211; 3 (Ultimate) times comparing to XP. Many of them are unneeded for a great deal of users. It would be nice to start some of them on demand, not automatically on every boot. Unfortunately, you can't turn off many services, because they are needed by system, 
	* ***Vista's performance is lower than XP's. ***
	
Recent, most obvious example: 
> [FONT="Courier New"]
1. Win XP: 	
Click &#8220;My Computer&#8221; => shows up after a second or two
Click again &#8220;My Computer&#8221; => shows up instantly, because it was cached 
2. Win Vista:	
Click &#8220;Computer&#8221; => shows up after a second or two
Click again &#8220;Computer&#8221; => does not shows instantly, instead, the window frame is shown first and then after a split of second the actual contents of windows showing hard drives and other stuff.[/FONT] 
When turning on Aero, this behavior is pretty nicely masked: you can't see this slowness glitch because &#8220;Computer&#8221; window is being drawn to the screen gradually.
	* ***As I sad more services (processes) are being run, therefore eating CPU cycles.*** This is self explanatory. In typical Win XP setup there are running between 20-40 processes or 300 to 400 threads, Vista on other hand is running 40+ processes on fresh install, and if you choose Ultimate version you will end up with 60+ processes and more than 500+ threads running!

**< Very expensive for very little benefit >**
	* ***UAC is very, very good thing if you are using a limited user account for daily work.*** Unfortunately, while installing Vista, nowhere is mentioned this. Many people are turning it (UAC) off, instead of using more secure limited account, they use their administrator account, and all Vista security falls down the toilet.

**< EULA and activation >**
	You don't own the software, you can't modify it, reactivation on every hardware change...

**< Useless by itself >**
	* ***There are better security products then Windows Defender.** *Fortunately, you can disable it. 
Recent example:>  My friend's Vista computer used for only a week was infected with viruses without he even knowing it. There is a possibility that he got it from an USB flash memory, because he was not using internet (IE8 beta constantly crashed and you can't uninstall it so he gave up for the time being).
	* ***As it's predecessor (Win XP) Vista's look also isn't very customizable as it should/could be.*** You are given only several  themes and in Aero theme you can only change the color of windows and level of transparency. Nothing more! You can't  install new or custom made themes, you can't change or add icon themes, although you can change a few icons manually one by one.

In some aspects Vista is an improvement over XP, but there are also many drawbacks.

NOTE: I am comparing Vista to previous version of windows, Windows XP, not Ubuntu.
