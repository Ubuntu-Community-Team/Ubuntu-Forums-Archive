### Post by asmoore82 on 2007-10-08
The first and most obvious reason is that the Linux networking code is just plain better. I have achieved sustained transfer speeds from Linux to Linux of over 11 MB/s on 100 Mbit switches; but replace either Linux with a *faster* Windows machine and never even get close to the same performance.

Also, the windows system is very, very likely to contain unwanted baddies that are borrowing some bandwith. And of course there are some *wanted* baddies that also borrow bandwith without asking...
being the hodgepodge of proprietary software that is the Windows desktop with no centralized control, many programs that you would want to have are individually fighting to update themselves;
such as antivirus software, antispyware software, adobe flash, sun java, adobe reader, heck even the microsoft "OS" itself likes to silently download updates if you haven't taken charge of it and turned it off.
