### Post by cl333r on 2009-01-27
Yes, I've also been thinking about the no-or-all browsers issue. Indeed, how is one supposed to install a browser or surf the net if there's no one at all.
We all remember the smart move MS took when they were forced to also sell windows without Windows Media Player (WMP) (they obeyed to sell both versions of windows, with and without WMP, but they charged the same price for both versions and of course people chose to buy only the version that came with WMP, thus the settlement over that case proved useless).

I'm pretty sure when we are talking about bundling the browser into the OS we are rather talking about MS in/directly forcing companies like Dell, HP through pricing policies to never give up on their IE, we know the cases when MS has been found guilty doing so, including the browser wars and the resulting trial "US against M$".
Thus the core issue is rather "How do we protect the manufacturers (OEMs) from being forced in a hidden or visible way by Microsoft to bundle their IE". This is the real challenge the judge(s) must face and solve. Ironically this is perhaps impossible to achieve, and thus whatever the judge decides MS will find a hole in the (judicial) system to pressure the OEM vendors to bundle its browser.

I have an idea, which I find hard for MS to sabotage or pervert:
The OS must come bundle with at least 2 browsers, this way one of them won't be IE for sure. Moreover, these browsers must be the top ones that meet these priorities: best web standards compliance (IE will die off) - most important priority, and 2nd: security, features and other technical characteristics - of less importance. Note that the accent is on web standards compliance, thus Opera, Safari, Firefox, Chrome are the ones that are eligible for inclusion, because of their merits not because of what MS or the OEM (pressured by MS) decides. For this idea to live up to its promise there must be (created) some kind of committee that decides according to the above priorities - which niche every browse occupies and thus whether it has a right to be bundled with the OS.
Here's an example.
Say the committee decides the browsers according to their merits form the following hierarchy top to bottom best to worst (which must be up-to-date and published on the web):
1)Opera v10.0
2)Firefox v3.1
3)Chrome v2.0
4)Safari v4.1
5)Firefox 3.0
5)IE 7.0
6)IE 6.0
thus, if the OEM wants to include only 1 browser, it must be Opera, if 3, these must be Opera, Firefox and Chrome correspondingly. I find it fair and very good for the internet and its users. This of course won't work if MS bribes the committee, but one can easily spot that because IE meets very poorly the first and most important priority: web standards compliance.
The committee could be built off the w3c consortium because it has a very good reputation IMHO and good experience in web standards.

### Post by Frak on 2009-01-27
It wasn't whether Microsoft should be allowed to include a browser, it was whether Microsoft was allowed to include a proprietary browser, default with the OS, based on proprietary standards. In this way, they could EEE the competition.

Microsoft still could take the alternate route of opening up IE's code and/or force it to comply with standards. If they did that, none of this would be happening. Though, they decided to take the weasel route and just embed IE into Windows, thus saying that it cannot be removed without leaving the system in a crippled manner (many applications use the Trident rendering engine).

This is why Apple not only used the open source webkit (heavily modified KHTML from Konqueror), but allowed it to strictly follow standards. Ubuntu doesn't need to worry since Firefox is not only open source, but it somewhat follows standards (not completely, but much more than the Trident engine does).
