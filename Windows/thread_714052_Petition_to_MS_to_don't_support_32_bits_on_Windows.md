### Post by livio_ribeiro on 2008-03-03
Hi, I'm here to show to the community this petition to MS to don't support 32 bits in Windows 7

The reasons are:

1.Windows is a heavy OS and needs heavy computers to run it

2. Keeping support to 32 bits won't encourage software developers and hardware manufacturers to embrace the new platform.

3. Using 32 bits OS in a 64 bits processor won't take the maximum advantage that the processor could offer.

and many others

This petition will benefit all the GNU/Linux communities, giving more option of 64 bits compatible hardware and forcing developers to port their softwares to this platform.

[Here's the petition]("http://www.petitiononline.com/win764b/petition.html")

### Post by roaldz on 2008-03-04
I find reason 1 the most stupid reason for having heavy hardware. I agree on the point that windows is a heavy OS, especially Vista. But an os demanding SO MUCH of your system for SO LITTLE from the os, it´s out of proportions. An OS should be lightweight and should **support** applications you´re running. It needs to make it easy for the user to use the applications. These applications can belong to the OS, but the should not be necessary to run the OS.
Applications need speed and power. The os should not take this away for itself.
it should be able to idle at low CPU consumption, something Vista isn´t capable of.
To switch to 64bit just because Microsoft developers can´t code something decent is not the right reason.

Microsoft should release their new versions of windows for 64bit CPU´s, but it must be backward compatible with 32bit apps, as it´s a proprietary OS, you can´t recompile your apps for 64 bit.

I agree on the point that it will stimulate developers to create 64bit apps, which will be good for the Linux community, in a way that we don´t have to do nasty stuff to run that proprietary software (32bit chroots, library dependance, nspluginwrapper).

I´m _not_ supporting it to block out 32 bit, I´m supporting it to support 64bit, because I´m supporting 64bit in general.
