### Post by #&amp;thj^% on 2022-03-12
2

If you didn't do this before you messed up your system, you can boot into a live-cd or another install, and use chroot to get at this info.

```
sudo chroot /path/to/old/system /bin/bash
```
Example:
```
cd /run/media/me/Data/

```
lets say that's my broken OS
I would then run:
```
sudo chroot /run/media/me/Data/ /bin/bash ###do not copy my path
```

Then do the dpkg --get-selections dance, and you can use the resulting file to setup your new system.
export your current package list on your old system:

```
sudo dpkg --get-selections > mypackages.txt

```
Then select this list as the packages to install on your new system:

```
sudo dpkg --set-selections < mypackages.txt
```
Don't forget your Browser stuff in  a hidden directory ".config "

### Post by colson69 on 2022-03-14
I had the same 134 error as OP after updating Pop OS 18.04. Long story short, I traced it to a "double free" error in fc-cache (fontconfig 2.12) in /usr/share/initramfs-tools/hooks/plymouth. This bug:
[https://bugs.debian.org/cgi-bin/bugreport.cgi?bug=927153](https://bugs.debian.org/cgi-bin/bugreport.cgi?bug=927153)

...possibly related to having multiple languages installed? (English and Swedish for me) The error is apparently fixed in fontconfig 2.13, but trying to upgrade fontconfig manually cascaded into a bigger headache.

So, upgrading to Pop OS 21.10 fixed it for me. Had to reinstall applications, but the upgrade otherwise preserved all my user data and app settings. Maybe there is a way to kick fc-cache into a better state though without upgrading.
