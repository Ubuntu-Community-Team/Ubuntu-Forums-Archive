### Post by grahammechanical on 2016-01-23
I am posting this from nelum-16.04-64 on a live USB stick session. A little bit of early confusion. The website said: "user is live. no password." This means when the live session asks for a user name type live and when it asks for a password press enter. And expect a black screen before you get to a desktop.

And it is Ubuntu 16.04. It has got the 4.3.0-5 kernel and lsb_release -a reads

Distributor ID: Ubuntu
Description: Ubuntu Xenial Xerus (development branch)
Release: 16.04
Code name: xenial

It must be piggy-backing the ubuntu xenial repositories. Cheeky! And I do not see any archives in sources.list other than Ubuntu archives. such as

> deb [http://security.ubuntu.com/ubuntu/](http://security.ubuntu.com/ubuntu/) xenial-security main restricted
deb [http://pl.archive.ubuntu.com/ubuntu/](http://pl.archive.ubuntu.com/ubuntu/) xenial multiverse
