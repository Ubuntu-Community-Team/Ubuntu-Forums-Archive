### Post by ld114 on 2012-04-09
Debian is certainly robust once you get it set up - you will need to find and apply your own drivers, and newer PCs/Laptops are sometimes difficult to get Debian up and running on. However, part of the fun of Linux can be learning how to do all that if you have the time and inclination. I agree that you will not get versions of software that are totally up to date - they are maybe two years behind the versions you get on Ubuntu itself. And that can be a challenge too.

Centos and Oracle are based on Red Hat and are also very solid distros, but will tend to have the same issues as Debian with regard to new hardware and up to date software. An Ubuntu LTS version (10.04 will be supported for another year; 12.04 is released later this month) is always a good place to start, as you will tend to get more up to date software and support for newer hardware.

So if you are on new hardware, I would start with Ubuntu which is pretty sound once a release gets bedded in. If you have issues installing, go with the alternate install iso as this is more likely to get you up and running. If your hardware is a few years old, then you can explore with Debian or the other "stable" (meaning they don't change much) distros. I have explored Debian and Centos, and others, but tend to stick with Ubuntu for my main machine.

### Post by buzzingrobot on 2012-04-19
One reason a release like Debian Stable is stable is that is made up of well tested and debugged software.  The flip side is that the newest code in Stable dates from well before its release date, so it is never the place to go for the latest. Sometime we need the latest because the old stuff can't do what we need it to do. Also, bug fixes don't do anything for software that is basically inadequate in the first place.

 I've used Slackware a great deal.  It's lean and solid.  However, it does not provide repositories and dependency resolving software.  That means if you install ABC from source and it complains about missing library XYZ, it is up to you to find and install that library. If you do give it a try, I recommend counting the first couple of installs as a learning experience. If you want to learn the fundamentals of Linux/Unix without the intricacies of distribution-specific addons and code getting in the way, it's your best bet.

Centos and Scientific, both based on RHEL -- they compile Red Hat's source -- are probably at least as stable as Debian Stable. For that matter, you can still buy RHEL for the desktop, if you want, at Red Hat's online store.
