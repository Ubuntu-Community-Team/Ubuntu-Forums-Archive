### Post by zhogan85 on 2011-10-05
I recently installed Crunchbang (#!) and Arch, and I can vouch for both working quickly on older hardware. I looked into Archbang, but eventually decided against it because it is essentially a preconfigured Arch and the reason I wanted Arch, was to use for learning about the innards of the os, and setting up a system from the ground up. That being said, if you want a quicker install of Arch, with openbox, Archbang is the way to go, and then you will still have the latest software as it is rolling release (since its essentially Arch + Openbox).

I've not used Sabayon, but I think its Gentoo based? I think Gentoo allows for even more custimization than Arch, but I don't know about Sabayon.

### Post by mips on 2011-10-05
Go with Arch (or Archbang) & Crunchbang.

Arch will give you the latest versions of packages and AUR is also excellent.

Crunchbang is based on Debian Squeeze (Stable) so packages are not as new as some people would like.

Either option would be a great choice though, just depends on what you want.
