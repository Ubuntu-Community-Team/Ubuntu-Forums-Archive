### Post by qyot27 on 2011-05-29
I've been occasionally testing it in VirtualBox for a couple years.  If one looks at the last three or four releases, there is visible improvement.  Even in a VM, I was able to satisfactorily navigate the system, play Spider Solitaire, and do a little browsing of the repository manager - all things that were uncomfortable to do a year or two ago, even on an Intel iMac as the host computer.  Said repository system is one place where they've shown willingness to diverge; another one is that I'm 90-95% sure that you can install ReactOS to an ext2 partition, and ostensibly, eventually other filesystems as well.  At any rate, it does ship with an ext2 driver without needing to go to a third party to get it.

Reaching maturity is a long way off, but it's a worthy project regardless.  Getting wider support is something that will likely start to happen once it hits beta; again, something I wouldn't bet on happening any time soon.  In 5 years, maybe.  After all, the Wikipedia article states that ReactOS' active dev team is about 40 people; Windows 7's dev pool was about 25x larger than that, as there were 25 teams of around 40.  So for having so few people working on it, I think what they do have is pretty impressive.  And at least they've proven themselves to be persistent.


Anyway, for the actual topic, the users that will likely be most drawn to it are threefold:
A) FOSS supporters that want an actual Windows-like system instead of having to change to a totally different system paradigm like *nix.  This is actually the reason given by the developers for ReactOS' existence.

B) XP users that want an up-to-date* OS without changing systems, similar to point A, and also do not want to buy Vista or Win7 (or Win8, 9, 10, or so on).  In 5 years, if ReactOS is finally at beta, then it will likely get a boost from the fact that XP's support cycle will have ended.

A subpoint of this is legacy program support.  Wine and ReactOS have a somewhat symbiotic relationship, and both of them give (or have the design goal of giving) better legacy support than Windows itself does.

*At least once the ball has started rolling for development pace, and like I said, I think the beta will be a watershed for that.  Once at beta, it'll get more users and draw more attention from those it can plausibly get funding and resources from, which should help speed up development.

C) Those who just like it for the kick or perverse thrill of it.  Sort of like using [Wine DLLs to](http://wiki.winehq.org/WineOnWindows) [fix Windows problems](http://wiki.winehq.org/WineD3DOnWindows) (I've actually done the latter one before; works great, and no performance hit from having to translate DirectX calls to OpenGL).  Or those who simply like it for research purposes.  ReactOS will give a comprehensive blueprint of the NT kernel and system structure where Microsoft doesn't and probably never will.  There are tons of OSes that use the *nix blueprint, perhaps because of how well-exposed and well-documented it is (aside from security and all the other usual reasons); variety is the spice of life.


I only wish it was able to be properly loopmounted with GRUB2 so I wouldn't have to burn any discs to test it out on real hardware.  There is a topic or two about that on their forum, so maybe it'll be possible in an upcoming release.
