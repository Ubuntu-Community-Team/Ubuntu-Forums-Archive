### Post by novafluxx on 2012-10-23
If it's not booting to the optical drive, then the disc is not bootable, your optical drive has an issue, or the system is not set to boot from the optical drive at all.

If it just simply skips over the optical drive and goes straight into Ubuntu, then the disc in the optical drive is NOT bootable.

It's got nothing to do with Ubuntu. Try redownloading the ISO image for Mint and reburning it at a slower speed. If you are using a re-writable disc, try an R disc, RW's degrade over time.

### Post by jwbrase on 2012-10-24
> **irishjay1 said:**
> When I installed ubuntu, windows never did this.

Windows has nothing to do with it. No operating system is yet loaded at the time that the computer decides whether to boot from the CD drive or the hard disk, and so neither Windows nor Ubuntu nor any other OS can cause the computer to boot from one or the other. Either you changed the BIOS settings to boot from the hard disk first without remembering it, or your CD is bad, or, possibly, your CD drive is broken.
