> **Dry Lips said:**

Some of us want a desktop ready solution, something which vanilla Arch isn't.
True but to get to a desktop solution is simple but then again getting there would probably waste a lot of the ops internet.   Arch gives you a command line install by default and you have to add xorg, video driver, and de/wm which could use a lot of bandwith. Arch setup goes like this.  Partition, mount partitions, download and install base system, chroot into new system, edit a few config files, setup your network, install bootloader, reboot into new system, then download even more stuff that is nescessary for a working desktop.   I don't know if that adds up to more or less than a 700mb cd or not  but for someone trying to conserve bandwith it's probably not the right distro. You can of course tell certain packages not to update in pacman.conf so if you just update the small stuff you could be fine.

### Post by mamamia88 on 2013-02-04
> **offgridguy said:**
> This whole subject has me wondering if it might be better to try building
something  on ubuntu mini remix, but that's  probably better addressed in a new thread.

same concept different exectution.  apt will pull in more stuff while pacman will pull in only the absolute nescesseties and then give you optional stuff you can install and tell you why you might want it at the end.  Then you just install the stuff you do want with pacman.   Also arch requires a little more manual configuration.   For example you want a display manager and desktop envrionment install it and reboot in ubuntu.  You have to edit a config file and enable the display manager upon startup for that in arch.  Once again it's tradeoffs.  With arch you end up with nothing more and nothing less than what you want but you have to work harder to get it.  In ubuntu mini you will end up with more stuff installed on your system but most if not all of the configuration is done for you. Also upon updating your system in arch you may get a message telling you of .pacnew files meaning that you might have to merge an old config file with a new config file.   Ubuntu though will use less of the OP's bandwith in the long run if that is a concern.  Real world performance differences between the two?  Modern hardware probably not much.  Personally I used arch for awhile and then felt like i missed ubuntu and switched back and then i felt like i missed arch and switched back again.  Grass is always greener on the other side lol.  If you learn arch though it will really help you with troubleshooting your problems no matter what distro you may switch to in the future
