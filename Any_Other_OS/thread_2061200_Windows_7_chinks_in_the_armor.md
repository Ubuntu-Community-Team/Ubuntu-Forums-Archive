### Post by Welly Wu on 2012-09-21
Since June 2012, Microsoft Windows 7 32 and 64 bit in all of its versions experienced several 0 day vulnerabilities which were quickly fixed with a combination of Microsoft Fix-It and Microsoft security updates including one out of band update today. XML, Microsoft Gadgets, and Microsoft Windows Internet Explorer 6 - 9 all had 0 day vulnerabilities in the past three and a half consecutive months which are now fixed if you applied the updates on your PCs by now.

My point is that Windows 7 is showing its age and there are more chinks in its armor. It is realistic to assume that Windows 7 will continue to gain more market share among all of the previous versions of Microsoft Windows and other operating systems worldwide in the foreseeable future so this constitutes a majority of users and their precious data. If you installed Adobe Flash and Oracle Java JDK or JRE, then your risk profile is elevated as Oracle recently pushed out an out of band patch for Java 7 Update 7 for JDK and JRE recently. Microsoft is staying on top of Windows 7 and they should be commended for their quick responses to close security flaws, but the fact of the matter is that it is buckling under the heavy targeted attacks against Windows 7. At this point, Windows 7 is usurped by Windows 8 RTM in terms of speed and security. Windows 8 especially the Pro 64 bit version is much more secure than Windows 7 and it is safe to use in production environments for now.

As time passes, Windows 7 will continue to wither new attack vectors aimed and specific 0 day vulnerabilities and exploits and there will be more security breaches. My best recommendation for Ubuntu users here is to install Microsoft Windows 7 or 8 in a guest virtual machine and you should use VM Ware Workstation 9 or Fusion 5. This will give you added security of a virtual guest machine contained within a sandbox and you can turn off your guest virtual machines for added safety. At this point, Windows 7 is still fairly secure and robust in terms of its security features and the amount of regular updates that it continues to receive, but this will not be the case once Microsoft Corporation focuses its efforts on Windows 8 deployment worldwide in the future. Time will tell when that tipping point will occur when Microsoft advises its consumers to upgrade to Windows 8, but that will be a few years from today.

I still consider Windows 7 to be the best version of Microsoft Windows in the company's history and the sales record is proof. However, I am beginning to form the opinion that Windows 7 is starting to age past its prime years in terms of security and I would advise Microsoft customers to upgrade to Windows 8 before January 31st, 2013 especially if you are a home consumer or a family. Windows 7 is a good operating system, but you should expect to see more security breaches if you continue to use it over the course of the next three years and they will become more critical and frequent.

Finally, Ubuntu users should consider avoiding Microsoft products and services if conditions permit. If you don't have to use Microsoft Windows or Office on your PC at work or more reasonably at home, then that will be the best security practice. After I migrated to Ubuntu on January 4th, 2012, I have found that my privacy, safety, and security have been seriously bolstered and I have more peace of mind knowing that my precious and confidential user data is far better protected. This is not to say that Ubuntu is impervious to future attacks, but it is a much wiser alternative choice to using Microsoft products including Windows and Office from the standpoint of security. A moderately hardened Ubuntu is exceedingly difficult to attack and compromise compared to a fully hardened and latest version of Windows. Ubuntu has a much more robust security architecture and model that allows for finer granularity of access control and superior security tools and mechanisms that are more open, flexible, and highly customizable which are available to all Ubuntu users for free. The latest versions of Ubuntu can be easily installed on older and legacy PC hardware while consuming far fewer PC resources and using Ubuntu prolongs the useful service life of a PC by several additional years. Canonical and the general GNU/Linux community pushes out patches for security vulnerabilities within a few hours whereas it takes Microsoft Corporation a few days or an entire month until next Patch Tuesday to fix issues.

I think that Windows 7 will eventually wind up heading in a somewhat similar direction as Windows XP in the next several years, but I do think that Windows 7 will have a shorter useful service life than Windows XP and Microsoft will advise customers to upgrade to the latest version of Windows much sooner in the future. Windows 7 is by no means insecure as of today, but I would be hesitant to use it on a daily basis as my primary desktop.

Windows 8 makes is far more tantalizing from a security perspective to use on a daily basis as a desktop, but I would not install it bare metal on my System76 Lemur Ulra Thin (lemu4) notebook PC. Too little is known at this point and there are considerable uncertainties surrounding it. It may wind up becoming the next Windows Vista.

### Post by Welly Wu on 2012-09-22
Windows Vista and 7 users are encouraged by Microsoft to install EMET. EMET stands for the Enhanced Mitigation Experience Toolkit. It is free of charge. You should install it and configure it for maximum security and reboot your PC. If you want to configure DEP and ASLR for specific software applications, then Microsoft provides a knowledge base article on how to do that in detail. It should be a TechNet article for IT professionals. Go to the Microsoft Support website and do a search for EMET.

EMET reduces the risk of successfully exploiting 0 day vulnerabilities with Vista and 7. You can install EMET on Windows 8 for enhanced mitigation for security vulnerabilities and exploits too.

It's not perfect and EMET may break application compatibility, but it is highly recommended to install if you use Windows Vista or 7 or 8.

If you can avoid using Microsoft products, then that would be better.
