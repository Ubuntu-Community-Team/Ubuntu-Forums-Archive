### Post by exploder on 2011-05-13
I was at the point of giving up on Linux after many years of use because I was so fed up with nvidea drivers, plymouth, mplayer and the list goes on of things that I constantly have to fix.

I have been a Linux user for about 10 years or so now and have always had the mind set that Linux would make it into the mainstream desktop market. I know, that's beating a dead horse....

Well, I am off work on medical leave so I could not afford to go and buy a copy of Windows 7 and I still needed an OS that would hold up to updates and have good hardware support. I tried several distributions that I thought might be up to the task, then I tried PCLinuxOS KDE mini.

I have been a long time fan of Texstar's work because he builds packages from source and uses source code from many distributions to resolve bugs and issues. Texstar knows how to build a decent kernel too and the hardware support has always been very top of the line.

I chose the KDE mini because I wanted control over all of the application selection. I built the system in Virtualbox and thanks to the task scripts I was able to put together exactly what I wanted in a few hours time. I used the "mylivecd" application to build my iso and burned it to disk.

I tested out the Live CD I made, it looked good and I went ahead and installed the new system. Plymouth worked, yeah there is some text showing but that is perfectly acceptable to me because there was no kernel errors of any kind. My nvidea drivers worked well, MPlayer did not double buffer or crash out of full screen, all multimedia was playing and the system was very quick and responsive. I did turn off the blur effect and set the animation speed to fast.

Everyone knows PCLinuxOS is a rolling release and this was a big plus for me. I want the latest Firefox, Thunderbird, etc, doesn't everyone? I really do not want to re-install my operating system every 6 months to have current applications and have to deal with regressions and trying to fix the same issues I had in the last release. I wanted real long term support.

I like how PCLinuxOS has one package manager. I have always liked the one application per task ideology because duplicated applications are just a waste of hard drive space to me.

I like the way the PCLinuxOS team tests updated packages, they go to testing and the pass server before going into the main repository. This generates income to keep PCLinuxOS going strong and it ensures the packages you get have been tested.

The "mylivecd" application that comes stock with PCLinuxOS provides an easy way to back up my perfectly configured system. If by chance something should happen I know I can be back up and running in about 10 minutes or so and I can use my remastered disk to give family and friends the same good experience I have.

I took the time to write all of this for those that might be at the point of giving up like I was. I see a small group of developers doing very fine work and making a difference. I see so many new releases that are released with so many bugs and regressions and they just will not work for the average user. I understand that new technology needs to be introduced and tested but not put out as a final release for the masses.I also know that software is never perfect.

If a small group of volunteers can build an operating system with with decent hardware support, provide updated applications, provide a means of backup and provide bug fixes, why can't the major distributions do this?
