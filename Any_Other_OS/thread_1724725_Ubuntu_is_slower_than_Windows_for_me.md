### Post by NightwishFan on 2011-04-08
At times on certain hardware Ubuntu can be pretty slow. At least GUI wise.. I never have any problems getting video encodings done quickly. This percieved sluggishness is a real problem though.

Ubuntu on 1gb of RAM seems like it crawls with a few things running. Not to mention video card weirdness. My suggestion to you is to slowly rule out causes. I will give an example of my results for each one:

Video Card: (for 2d) Install gtkperf and run the benchmark. Does it score between 5-15 seconds? If so, probably not video card making it feel slow.
[QUOTE="My Result"]My Gtkperf on 100 had a Total time: 29.60. This means I have some 2d drawing issues. This is both because I am running mutter (which has strict animations) and a bug in drawing circles on my intel card. (GtkDrawingArea - Circles took up this much of the time: 21.92. Without it my result is only around 7 seconds. So I can rule out 2d drawing safely even though it has a corner case performance problem on my laptop.[/QUOTE]

Video Card: (for 3d) Can you run Compiz smoothly? Use the benchmark plugin (might have to install extra plugins and compiz config to access it). If you get a solid 60fps or more you have decent 3d acceleration.
[QUOTE="My Result"]Compiz and mutter perform solidly on my machine, as well as various 3d games. So I can rule this issue out.[/QUOTE]


RAM: Run free -m. Look at the line that says buffers/cache, do you have a large amount of ram free? Look at the line that says swap: Is the swap usage 0? If so, it is probably not memory making it slow.
[QUOTE="My Result"]```
-/+ buffers/cache:        773       2215
Swap:         3336          0       3336
```
I have well over 2gb of ram free and am using 0 of my swap. Ram is not the issue.
[/QUOTE]

CPU: Run top, and look along the top for an entry that shows the idle cpu (example: 92.8%id). Is your cpu over 70% idle most of the time when not doing anything? If so, then its not a cpu issue.
[QUOTE="My Result"]My cpu is over 90% idle while I am typing this post.[/QUOTE]

Disk: Run the READ ONLY benchmark in Gnome Disk Utility. Do you get higher than 50mb per second? Is so it is probably not a disk problem. Though on a 5400rpm disk, things can seem slow. Do programs take a while to start? And the big one, flash based devices REALLY choke on write speed. So when Linux reaches the write-back point when there is a lot of IO things might lock up for a while.
[QUOTE="My Result"]My disk is fast (for a 5400rpm) and programs open quickly.[/QUOTE]

Things along that line can tell us more about the problem.  "Windows is faster" only says to me that Windows is more responsive at launching programs. Which from my experience, it usually is. Probably due to different kinds of preloading and library link strategies. I think Linux uses cache better for programs that are already running. A good way to test this is: Start Firefox, it may take up to 10 seconds to start. Close and start it again, it should be near instantanious. If not, then you are probably low on memory and may need a lighter environment like XFCE or LXDE.

### Post by seierson on 2011-04-08
I saw a lot of good advice on here.  If part of your hard drive is bad and that portion also happens to be where Ubuntu is installed then it can slow down considerably by being forced to re-read areas multiple times before returning data from that sector.  You could run SpinRite on the drive to figure that out (DOS-based and commercial, however there is nothing better or even comparable).  

I don't know about your Ubuntu variant, but to install proprietary video drivers (if there are any and if you do have integrated Intel graphics then there isn't anything) click System --> Administration --> Hardware Drivers.  If it shows any drivers you can activate then do so.  

If you want to know what graphics card you have run windows, press Windows + R, type devmgmt.msc, and expand the display adapters tree.  What does it say you have for a display adapter?  

Also try: System --> Preferences --> Appearance, Visual Effects = None

Your slow experience may be caused by generic drivers.  Linux doesn't always have well optimized drivers (in fairness nor does Windows), but that could be the difference.  

Good luck,
Christian Blackburn
