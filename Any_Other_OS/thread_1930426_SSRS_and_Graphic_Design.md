### Post by samalex on 2012-02-24
Funny seeing a question about SSRS on here, I'm an MS SQL DBA and my day is spent exclusively working with SSMS and by proxy SSRS, SSAS, SSIS, etc :)

Honestly though I don't see the skills of a graphic designer being of much use when designing reports other than possibly helping to layout data as organized as possible.  Most reports will have a logo or heading on the top that's reflective of the company, but in many cases that's already preset by the marketing department or even just a small logo or text as to save real state.  Also some companies have set templates and design layouts they use, so there's not much left for the imagination.

If you've never worked with SSRS I'd suggest downloading [MS Report Builder 3]("http://www.microsoft.com/download/en/details.aspx?id=6116") which is amazing!  It will connect to a wide variety of data sources so you don't need SSRS or even an instance of SQL Server per say, but it's a good way to see what you'd be using.  Some businesses also use BIDS to maintain SSRS reports since you can create a Report Project in BIDS that can easily go into source control, just depends on the standards of the shop you'll work in.

If you want to work with BIDS, SSRS, or SQL Server I'd suggest either [spending $49 for SQL Server Developer edition 2008 R2]("http://www.microsoftstore.com/store/msstore/en_US/pd/productID.221634300") which includes everything SQL Server has to offer including SSRS and BIDS which you can run locally on your system.  Also CodePlex has all the demo databases you can use to write reports against.  Also if you're feeling REALLY daring you can [download SQL Server 2012 RC0 for free]("http://www.microsoft.com/sqlserver/en/us/editions/developer.aspx").  REport Builder 3 will work with SSRS 2012 or it will come with it's own report builder which is launchabled directly from the SSRS website (when you have it running locally).  I'm VERY impressed with SQL Server 2012 and all the improvements with SSRS like shared data sources and data sets.

Please post back or email me if you want to discuss this more, and I don't mind sending you more info via email if you'd like. 

Take care ;)
