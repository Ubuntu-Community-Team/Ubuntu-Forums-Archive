### Post by krapp on 2011-05-11
Arch can cure cancer when set up properly.

In all seriousness, I don't see much difference between Ubuntu and Arch once you subtract their ideological components, aside from the later being one of the so-called rolling distributions (which is totally useless if you break your install, as you'd have to re-install). I doubt there would be much difference between a minimal Ubuntu install and an Arch install using the same kernel and drivers. Both have large repositories, Ubuntu because it has Canonical and Debian in the background, Arch because its users are very busy porting the latest software. I think it just boils down to Arch being geared toward tinkerers, whereas Ubuntu increasingly discourages tinkering. Archfans love professing their KISS ideology, which has always struck me as bizarre considering how much time they spend customizing their installs, sometimes in the most abstruse and esoteric manners. Which amounts to a fine learning experience, to be sure, but not necessarily a more efficient or productive computing environment than that of a more stable or user-friendly distribution.

tl;dr: Use Arch if you want to post in the Arch forums, and be part of a very active tinkering community. I'm sure you'd learn a lot about Linux.

### Post by jeffathehutt on 2011-05-11
Make sure you read and understand [The Arch Way]("https://wiki.archlinux.org/index.php/The_Arch_Way").  If it still sounds appealing after reading that, next read the [Beginner's Guide]("https://wiki.archlinux.org/index.php/Beginners%27_Guide").  Make sure everything in the guide sounds like something you can do, and if you are confused then you should ask for help before you install.

Arch is a great operating system for those who like it.  If you want everything done for you, then arch is probably not what you want.  On the other hand, if you want more control over you system, and more up-to-date software, then maybe arch will work out.

The "endless tweaking" rumour is just that.  I installed arch three years ago, and I have done very little tweaking since then.  Usually, installing a program is as simple as 'pacman -S *programname*' and most programs do not require any tweaking to get them to work.  For those that do, the arch wiki is excellent and almost always has detailed instructions on how to install and configure each program.

Arch really is what you make of it.  Some hate it, some like it.  You will have to decide for yourself if it is the right distribution for you. ;)

### Post by snowpine on 2011-05-12
> **krapp said:**
> Sounds like a misnomer to me. How exactly is, say, a minimal Debian install less simple than Arch?

Debian way: Test a package for up to 2 years as it passes through Experimental, Unstable, Testing branches. Patch to satisfy "free" ideology (example: Firefox->Iceweasel). Release as Stable. Support outdated software with bug fixes and security patches for ~3 years. Create a backports repository to satisfy users who want updated applications. 

Arch way: Release vanilla packages on a rolling release schedule shortly after they are released as "stable" by the upstream developers.

A gross oversimplification of course, and it all depends on what "simple" means to you. :)
