### Post by BrokenKingpin on 2011-08-18
Android runs the Linux kernel, but that is where the similarities end between Desktop Linux and Android.

A typical Linux Distro has the Kernel, X, and the DE. Android has the Kernel and then just a Java Virtual Machine that sits on top if it where everything runs out of. Because of this, most applications do not port well to Android.

I always found Maemo a far more interesting mobile OS than Android, mainly because it follows a more tradition Linux setup. Meego as well, but from what I have seen it looks to bloated for a phone. I suppose it doesn't really matter though, I don't see either of these OSs having a bright future, which is a damn shame.

### Post by akand074 on 2011-08-18
Yeah, Android is based on the Linux kernel. It's actually a fork of the Linux kernel and customized (because apparently some of Google's additions to the Linux kernel was rejected so they had to fork it and maintain their own to put their changes in). Other than that, there is nothing the same to the desktop OS. A lot of people seem to think that anything using the Linux kernel are a lot alike. This isn't really true, there is a huge amount of things using the Linux kernel. It's just a kernel, all it deals with is hardware, that's it. So basically it just lets the software access/read/understand the hardware. Everything else is just what the developer decides they want to do and can be very similar to completely different than anything else using Linux.
