### Post by oldfred on 2011-08-30
You cannot install os-prober in Fedora as it still uses grub legacy. Then version of Fedora just released as alpha is changing to grub2.

You installed grub legacy to the MBR and should be able to boot Fedora. But you will have to add an Ubuntu boot stanza. Change x to one less than the partition number as grub legacy counts from 0. (Gub2 counts partitions from 1)

title Ubuntu 11.04
root (hd0,x)
kernel /grub/core.img
boot

### Post by oldfred on 2011-09-07
You also have a separate /boot partition for Ubuntu. Now that is not normally required for desktops. Fedora has it because they use LVM as the default for the system partition. Some suggest when installing Fedora to create partitions in advance to avoid the LVM setup.

Only one system is in charge of MBR. If you have Fedora as your main system, it should be in the MBR and you edit its grub legacy to also boot Ubuntu.

But is using Ubuntu primarily, then you should install grub2 to the MBR and use it to boot. Then you may have to edit 40_custom, but usually grub2's os-prober finds other installs and adds them automatically. 

Some also prefer chain loading which you can also do since you also have grub legacy installed to the Fedora boot partition. The advantage of chain loading is if the kernel is updated in Fedora it updates Fedora's grub, but you have to reboot into Ubuntu to rerun the grub-update (os-prober) to get the new kernel in the menu. With chain loading all you do  is link to the grub legacy which then has the new kernel on updates. But a chain entry has to be manually added to 40_custom.

Fedora does not use sudo. And unless you created a Bootscript folder your path is not correct. In Ubuntu you normally save to Desktop or Downloads. Then just cd to that folder, su to be root and bash /boot_info_script.sh.
 > 
sudo bash ~/Desktop/BootScript/boot_info_script.sh

### Post by oldfred on 2011-09-08
Site is not bad, but I do not agree with separate /boot partitions except for real old computers that can only boot from the first 137GB, servers, RAID & LVM (maybe a few other cases). Since Fedora uses LVM you need the separate boot, but a few posters here have suggested to just create a partition and install to that without the LVM. I do not know how to do that since I do not have Fedora.

I normally suggest a separate /home when you just have one install as that makes it easier to upgrade. But if sharing with Windows you should create a shared NTFS partition that all systems can read. Then you can mount it in every system and have the same data. If most of you data is in the shared partition you have much less need for a separate /home. My /home is inside / (root) since I have all data in other partitions and it is only 1GB with 3/4s of that .wine for Picasa.

Bottom line is that there is no wrong way to install as long as it works. But whichever system you want to boot the most should be the one booting from the MBR.
