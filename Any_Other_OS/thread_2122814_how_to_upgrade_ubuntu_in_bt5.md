### Post by Cheesemill on 2013-03-06
Although BackTrack is originally based on Ubuntu 10.04 it has been highly customized and uses its own repositories.

It uses a custom kernel as well as custom versions of other core applications. Adding Ubuntu repositories to a BackTrack installation to try and upgrade it is a quick way to seriously break your installation.

Unless you do pen-testing for a living there is no reason that you actually have to use BackTrack, it would be easier to install the few applications that you need into a standard Ubuntu installation as the majority of them are already available in the Ubuntu repositories. Also the BackTrack devs are moving away from using Ubuntu as a base, the next distribution that they are currently working on (Kali Linux) is based on Debian instead of Ubuntu.
