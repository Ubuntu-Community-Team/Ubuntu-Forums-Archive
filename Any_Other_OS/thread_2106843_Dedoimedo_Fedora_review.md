### Post by buzzingrobot on 2013-01-20
My take on Dedoimedo reviews is that whatever he says, my experience is the opposite.

That's the case with Fedora 18.  I installed it several times during the alpha and beta. The new installer was still under development at that point and was buggy, as one would expect it to be.  But, it never prevented me from doing an install.

When the Release Candidates were available, I burned an net install ISO, installed it with zero problems, and have not looked back. 

On my hardware, F18 is solid, fast, and reliable. If someone wants the real Gnome experience, this is better than anything Ubuntu has on offer (I know, I've used them.)

Dedoimedo does make one valid criticism:  The installer does not initially label drives with the traditional "/dev/sda/, /dev/sdb,..." nomenclature.  That can be confusing.  It uses the traditional nomenclature when you map partitions to physical drives. Unless you use the default LVM, of course, in which case none of that matters.

When you read these quick and dirty installation reviews, you need to pay attention to what is actually being reviewed, the hardware in use, and whether the reviewer followed the recommended install procedures.  Dedoimedo focuses a lot of space on live cd performance. That is irrelevant to me, as I expect LiveCD performance to be awful.  He installs on a multi-boot laptop.  Mixing operating systems is always a fragile proposition.  I install one OS only on a multi-disk desktop tower after reading that distro's release file and learning about the most common bugs. I install for real, not in a virtual machine.

In other words, don't trust any single opinion, including this one, unless you try things for yourself on your hardware. Tales of success or dismal failure are not useful or informative unless you also are given the specific hardware details.  You almost never see that in these online reviews.
