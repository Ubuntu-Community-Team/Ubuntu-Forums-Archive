### Post by hhh on 2011-07-12
I just had to do a fresh install and I went with Debian squeeze Xfce from the Live images (linked earlier in this thread, but here...)
32 bit via Transmission or another bitorrent client...
[http://cdimage.debian.org/debian-cd/6.0.1-live/i386/bt-hybrid/](http://cdimage.debian.org/debian-cd/6.0.1-live/i386/bt-hybrid/)

64 bit...
[http://cdimage.debian.org/debian-cd/6.0.1-live/amd64/bt-hybrid/](http://cdimage.debian.org/debian-cd/6.0.1-live/amd64/bt-hybrid/)

My 32 bit system is currently using 234MB RAM running Iceweasel 5 (113MB w/ 6 tabs open, Iceweasel is Firefox rebranded for Debian to bypass a copyright issue), Nautilus Elementary (not installed by default, Thunar is and it's much lighter than Nautilus), a Last FM client to stream music, and the Xfce Task Manager, so it should run fine on your system. The LXDE version is even lighter, but Xfce is easier to configure IMO. I did disable a couple of services, like the Power Manager/screensaver, print cue applet and kernelloops applet, but none of those are very heavy. I know for a fact that both Crunchbang Xfce and Crunchbang Openbox have similar RAM usage (they are both respins of straight Debian squeeze with some extra scripts thrown in). I'd imagine the Linux Mint Debian Edition Xfce version would also have similar RAM usage as it is Debian testing based with some customizations.

Package management is the same in Debian and Ubuntu, you can use apt-get, aptitude, Synaptic, etc... Debian supposedly has the largest package selection of any distro.

If you install from the Debian Live images, there are a few oddities to be aware of. One is that a feature is enabled by default that makes it so that when you do a restart Grub is bypassed entirely which makes rebooting faster. If you don't want that, just remove the package kexec-tools. I had to do a hard shutdown (hold down the power button till the machine turns off) on my first reboot after install, that's probably something hardware-specific to my system. Finally, if you plan to install from USB, use dd to transfer the iso (Unetbootin failed for me) and you'll have to edit the USB entry out of /etc/fstab when the install is finished.

Hope that helps. Sorry this post is long and feel free to PM me if you decide to go this route but run into problems.

1. Get the newest hardware you can get, and use GNU/Linux, on that. Not your old stuff. 

2. Now that Ubuntu has devolved, you might as well go with Debian. Complete first timers, start with Linux Mint 11 (right now). Once you learn how to fix your system, consider Debian. 

IHMO, don't sweat it. Use Debian stable, on **separate server** boxes. Use Debian **un**stable, for the latest greatest. Just don't dist-upgrade when things (you care about) are broken. Use apt-listbugs. Consider smxi, to manage sid, add most of the things you like, similar to Ubuntu. Check forums, for status warnings.

I've found that sid affords the latest version(s), of the software such as photography, that I want to test. 

You can build just what you need, per the hardware you have (it scales better). You can use the standard Debian package system. You can install any desktop environment (just use the default Gnome). You can set up your system, similar to Ubuntu's tweaks, if you want to.

You can standardize all your boxes, on Debian. It's rolling release. No more upgrade madness, but for many (just do clean installs, with Linux Mint). Do not (IMHO) mix the Debian tiers! That's why I said just use unstable/sid, and not testing (once you throughly understand what you're getting into). Not novices, and I say that without any snobbery.  

If you feel like sid, is too breakable, or takes too long to fix-up (EX: You're just getting started, with all this stuff), use Linux Mint 11, as your primary production system, and use Debian **separately**.

Tip: New users (who wish to boot Windows, and Linux separately) should know, you might have to delete the stupid "Recovery" partition. Unless your computer is less than 3 months old; where that MIGHT still be **somewhat** relevant, and you think you might need it; so, that you can, with the Linux Mint 11 installer, then easily (delete Recovery) and resize Windows; leaving a hole, that the installer (pointed to that "hole"), will then fully, and automatically, take care of the rest of it's install. 20 minutes later; viola; peanut butter sandwiches, you're dual (or it's a duel), "booting".

Alternatively, you could just use a secondary hard drive, but older PATA drives, need jumpers and cables, done right. Again, don't use you old, peeve poor, hard drive, for "Linux". Use you fastest one. New ones are cheap.

Of course, the smartest thing to do (after backing up what you can't live without) is just overwrite Windows altogether. If you start with Mint 11, you'll thank me. That's the easiest, and then you have only one desktop to keep up with. 

Just for kicks, I also installed Plymouth, and the sunrise Debian theme (on sid), at start, and even sun**set**, on shutdown. I also customized Debian Gnome, to look rather Windows-like. Download the win2-7 script, and even the converted XP icons, if you dare. Such madness! Your momma will even like it. :)
