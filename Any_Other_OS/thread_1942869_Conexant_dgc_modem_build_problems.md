### Post by kidoruigenso on 2012-09-07
There is maybe a better solution, see [http://ubuntuforums.org/showthread.php?t=1939787&page=5](http://ubuntuforums.org/showthread.php?t=1939787&page=5) and the prior pages.

This is thanks to "Computator" of the UbuntuForums (apparently Derek Neely) who, in the Linux/Ubuntu spirit, creating a patch that allows the Linuxant USB modem driver to work with the latest kernels like you find in Ubuntu 12.04.

Now you can keep the 12.04 kernel.

On my system, with a fresh install of Ubuntu 12.04 and a ZOOM Model 3095 external USB mini external modem, the following worked.

1) Get the generic source tar package from [http://www.linuxant.com/drivers/dgc/downloads.php](http://www.linuxant.com/drivers/dgc/downloads.php) and put it in the user's home directory. Linuxant calls the file "dgcmodem-1.13.tar.gz".

2) Get the patch of Computator found at [http://derekthewriter.com/ubuntu/fixes/dgcmodem-1.13.patch](http://derekthewriter.com/ubuntu/fixes/dgcmodem-1.13.patch).  I copied the text, pasted into a simple blank text editor file (e.g. with gedit), and saved this as a file with the name "dgcmodem-1.13.patch".
    
3) Extract the tar package to your home directory.  You get a new directory called "dgcmodem-1.13".

4) Put the patch file (dgcmodem-1.13.patch) into the same directory.

5) From a terminal window, change to this directory, probably using the command
    cd ~/dgcmodem-1.13
and enter
    sudo patch -p1 --verbose < dgcmodem-1.13.patch
in this terminal window.
Make sure you are in the correct directory and have properly identified the name of the patch file in the command.

The output from this was for me...
    Hmm...  Looks like a unified diff to me...
    The text leading up to this was:
    --------------------------
    |diff -ur dgcmodem-1.13-old/modules/mod_dgcusbdcp.c dgcmodem-1.13-new/modules/mod_dgcusbdcp.c
    |--- dgcmodem-1.13-old/modules/mod_dgcusbdcp.c    2010-03-25 10:31:18.000000000 -0700
    |+++ dgcmodem-1.13-new/modules/mod_dgcusbdcp.c    2012-07-25 22:30:08.366214038 -0700
    --------------------------
    Patching file modules/mod_dgcusbdcp.c using Plan A...
    Hunk #1 succeeded at 260.
    Hmm...  The next patch looks like a unified diff to me...
    The text leading up to this was:
    --------------------------
    |diff -ur dgcmodem-1.13-old/modules/osdcp.c dgcmodem-1.13-new/modules/osdcp.c
    |--- dgcmodem-1.13-old/modules/osdcp.c    2007-05-07 13:27:35.000000000 -0700
    |+++ dgcmodem-1.13-new/modules/osdcp.c    2012-07-25 22:30:07.126284673 -0700
    --------------------------
    Patching file modules/osdcp.c using Plan A...
    Hunk #1 succeeded at 28.
    done

6) Then plug the USB modem into a USB port and enter in the same terminal window
    sudo make install
and then follow the instructions, i.e. next enter
    sudo dgcconfig
in the terminal window.
When you see the prompt:
    Where is the linux source build directory that matches your running kernel?
go ahead and select the default choice (i.e. hit "Return"), it is probably ok.

7) For the ZOOM USB Mini External modem, that uses the address ttyACM0 when correctly installed, I could then verify the install was completed with the terminal commands
    ls -l /dev/ttyACM0
and
    ls -l /dev/modem
and look for the link indication between the two in the terminal output.  My output looked like
    user@user:~$ ls -l /dev/ttyACM0
    crw-rw---- 1 root dialout 166, 0 Sep  7 20:51 /dev/ttyACM0
    user@user:~$ ls -l /dev/modem
    lrwxrwxrwx 1 root root 7 Sep  7 11:08 /dev/modem -> ttyACM0
    user@user:~$ 
showing the link /dev/modem to ttyACM0.

### Post by Mr_Leo on 2013-04-24
I fixed the problem with a solution I found here:
[http://forums.linuxmint.com/viewtopic.php?f=49&t=105696]("http://forums.linuxmint.com/viewtopic.php?f=49&t=105696")

had to modify untared source files..... and modify a line. on 2 files found in ../dcgmodem-1.13/modules/


mod_dgcusbdcp.c

Code: Select all
    /* static spinlock_t dgcusbdcp_lock = SPIN_LOCK_UNLOCKED; */
    static DEFINE_SPINLOCK(dgcusbdcp_lock);


just comment out line like above and add line below.

osdcp.c

Code: Select all
    /* static spinlock_t dcp_lock = SPIN_LOCK_UNLOCKED;*/
    static DEFINE_SPINLOCK(dcp_lock);


then do a

sudo make uninstall
sudo make clean

and then go through the steps for a normal install; make install, dcgconfig, ...
...and it worked!
