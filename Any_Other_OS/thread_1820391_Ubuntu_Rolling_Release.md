### Post by magical2hobo on 2011-08-07
Usually rolling release and user-friendly don't mix. Rolling releases always have the risk of breakages with any given update and that usually doesn't lend toward a user-friendly distro. But if u must some alternatives would be run straight debian testing, aptosid, and im not sure but i think pclinuxos is a rolling release too. but if u wanna just dive in and do everything yourself arch is a viable option too.

### Post by trinux_bc on 2011-08-07
PCLinuxOS is rolling release, and is pretty user friendly. It's not like Ubuntu at all (rpm's, doesn't use sudo, etc) but it is an easy rolling release distro. You've just got to keep an eye on their forums for announcements and things you have to do to make sure your system won't get borked by the updates.

### Post by BrokenKingpin on 2011-08-08
I will second PCLinuxOS. The only downside is there is no 64-bit version, but they are apparently working on it.

LMDE is OK, but after only using it for 3 weeks an update killed my system. I also found it to be overly bloated... a normal Ubuntu 10.10 seemed a lot quicker than LMDE did.

OpenSUSE can be turned into a rolling release distro by enabling the Tumbleweed repositories. I have not tried it myself, but it sounds pretty cool. I did read that there may be an issue with this if you use proprietary drivers (NVidia for example), so you will have to read up on it a bit to see if it is right for you.

Arch is a cool distro, but it is not user friendly.

I would love it if Ubuntu switched to a yearly release cycle, and throughout the year updated the user packages to keep things more up to date without breaking the system (half rolling release). A 6 month release cycle just feels too short, and a pain to re-install/upgrade that often.
