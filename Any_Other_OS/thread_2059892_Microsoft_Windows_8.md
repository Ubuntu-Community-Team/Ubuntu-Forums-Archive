### Post by Welly Wu on 2012-09-19
Is it worth upgrading to Windows 8 Pro 64 bit or not?

My biggest concern is the fact that I run Windows 7 64 bit inside a VM Ware Workstation 9.x 64 bit guest virtual machine. I am not sure if I qualify for the $39.99 USD upgrade price on October 26th, 2012 or not. I got the System Builders OEM version of Windows 7 64 bit Home Premium Service Pack 1 and I used my Windows Anytime Upgrade product keys for Professional and Ultimate 64 bit.

That's my real concern.

I have looked at some of the benchmarks published between Windows 7 64 bit Professional and Windows 8 Pro 64 bit. It's a mixed result. Windows 8 Pro 64 bit is clearly faster for cold boots, reboots, and shut down timed performance and it is better for DirectX 11.1 3D accelerated graphics and it is more energy efficient which means that there is more battery life, but Windows 7 still holds the edge in desktop performance on a few benchmarks. See the Maximum PC review of Windows 8 Pro 64 bit.

The problem is that the dual user interfaces affect the user which in turn affects real world performance. You're juggling between the Windows 8 User Interface and the Windows 8 Desktop mode. You also have two versions of Microsoft Windows Internet Explorer 10 for both interfaces. This adds to confusion. There is a lot more keyboard and mouse navigation which require more key stroke combinations and mouse clicks to navigate through both user interfaces.

One clear advantage for Windows 8 Pro 64 bit is Super Speed USB 3.0 performance: it is clearly faster and transfer times are more accurate. You can pause and resume transfers.

The final thing to keep in mind about Windows 8 is the fact that it has a long road ahead to overtake Windows 7. That is one of the true benchmarks of success for Windows 8. Can it replace Windows 7 in the next 3 years or not? If it can, then Windows 8 will be a runaway success for Microsoft Corporation and it will finally break their history of producing a winner followed by a loser. So far, the technology journalists have given it mediocre reviews for the most part. It will be interesting to see what real world Windows 8 users have to say about it.

### Post by Welly Wu on 2012-09-21
One of the disadvantages about Microsoft Windows 8 especially the Pro, Ultimate, and Enterprise 32 and 64 bit Upgrade and System Builders OEM versions is the fact that it has a very limited license which practically guarantees vendor lock in. With these specific versions of Windows 8, you will not be able to decommission an older PC and re-install Windows 8 on a new PC without having to re-purchase Windows 8 again. Microsoft is closing the loophole that allows you to transfer your old product key, activation, and license from one PC to another PC and they are applying this same rule to virtualized environments that don't use Hyper-V technology such as VM Ware or Oracle VM Virtualbox. This means that if you run Windows 8 Pro 64 bit in a guest virtual machine, then it will be treated as a single PC entity according to the license and you will be stuck with it in the future unless you purchase another genuine copy of Windows 8 for a new PC or a different non-Microsoft virtualization software product. You can't call the toll free Microsoft Activation center and transfer Windows 8 to another PC or virtual machine with Windows 8. You will be locked in permanently.

Microsoft is doing away with the boxed retail versions of Windows 8 and they are going with the System Builders OEM version at retail if the new information from PC World is correct. This means that you will be locked in once you purchase a new PC, enter the product key, and you activate Windows 8 on that new PC and you agree to the license agreement.

The other issue to be concerned with is that you will experience difficulty in dual-boot environments with older versions of Microsoft Windows XP, Vista, or 7 on a new PC that is certified with Windows 8 pre-installed. UEFI and Secure Boot restrictions will make it more difficult to purchase and install Windows 7 in a dual-boot configuration on the same Windows 8 PC. If you dual-boot with GNU/Linux, then you may wind up corrupting your GNU/Linux partition with Windows 8 once you hibernate the PC after you shut down Windows 8.

I want the increased speed and performance along with the new security features of Windows 8, but I have some reservations about upgrading even though I will go ahead and upgrade on October 26th, 2012. If I run into a problem with Windows 8, then it will require a full re-installation of Windows 8 from the blank DVD-R installation disc that I can create upon initial upgrade from scratch as my old Windows 7 64 bit license will be invalidated by upgrading to Windows 8 Pro 64 bit. This means that I will be more heavily reliant upon Windows 8 to be reliable and stable over the course of the next three years to avoid this scenario altogether. Given my past history with Microsoft products especially Windows, this may not be the case in the future if I run into a serious problem such as corrupted system folders or files.

These are risk factors worth considering for others before they make the decision to purchase or upgrade to Windows 8.

### Post by Welly Wu on 2012-09-23
Unfortunately, my System76 Lemur Ultra Thin (lemu4) does not have UEFI or Secure Boot hardware. This means that I am not going to be able to take advantage of Windows 8 or Ubuntu 12.10 or 12.04.2 64 bit secure boot feature at all which means that I should be concerned with traditional boot sector attacks like the Evil Maid attack. One thing that I could do is to put the /boot partition on a USB thumb drive, but this will require that I re-install Ubuntu using the alternative installation disc for the best results. I don't plan to re-install any operating system from scratch in the near future. It's just too much hard work and it takes too much time.

If you take away UEFI and Secure Boot, then Ubuntu 12.04.x 64 bit LTS has far superior safety, privacy, and security features than Windows 8 Pro 64 bit. Ubuntu users rarely receive the attention of attackers worldwide. We are small niche group of users that represent a low risk of attack. This makes Ubuntu much safer to use than any version of Windows. The new privacy features built into Ubuntu 12.04.x are quite comprehensive and they allow for some coarse granularity to specify which types of software applications and data and directories should not be monitored or recorded. Windows 8 does not have this feature. You still need to rely on a third-part software utility to clean up and erase your sessions and usage history on Windows 8. In terms of security, Windows 8 finally incorporates anti-virus and anti-rootkit scanning starting with Secure Boot and throughout the entire boot to shut down phase. It also comes with Windows Firewall which is quite good as it is pretty finely granulated. It also comes with BitLocker with a couple of new features and the ability to connect to Home Groups, corporate networks, and VPNs seamlessly. This is where Windows 8 takes the lead over Ubuntu in making it easy to configure the Windows Firewall and to connect to corporate networks and VPNs or to create a home network with shared drives, folders, and files. Ubuntu still requires more work in setting up SAMBA, UFW, WinBind, and VPNs although it is fairly easy to do so with the right tools. Where Ubuntu has a clear advantage is that these technologies are continually being upgraded to incorporate new features and capabilities for free. Windows 8 will eventually be replaced with a newer version of the Windows operating system in the future and that will require more future purchases to get those new features.

So, Ubuntu 12.04.x 32 and 64 bit LTS is the clear leader for safety and privacy features in particular and I would rate the security features of both Ubuntu and Windows 8 to be similar in general especially if you plan to purchase a new desktop or notebook PC with UEFI and Secure Boot hardware technologies built in. It comes down to your priorities and budget. I give Ubuntu the decisive victory due to its free cost of ownership and its greater safety and privacy features. Ubuntu has not seen a 0 day vulnerability or exploit being actively targeted against Ubuntu users in the wild in years due to hourly updates including security updates. There are very few viruses, polymorphic viruses, root kits, worms, spyware, adware, ransom ware, crime ware, key loggers, trojan horses, and bot nets targeting Ubuntu users worldwide that can successfully target, penetrate, and compromise individual Ubuntu users or larger groups. Most hackers and attackers go after Windows or Apple users because there are more of them in this world than GNU/Linux users and Ubuntu users in particular.

A moderately paranoid Ubuntu user and hardened operating system that is being kept up to date is exceedingly difficult to create a security breach because of the dearth of attention directed toward Ubuntu users by hackers and attackers worldwide. A targeted advanced persistent threat that exploits 0 day vulnerabilities is exceedingly dangerous and it is difficult to accomplish against an individual Ubuntu user especially if he has a notebook PC with Ubuntu pre-installed and he is a mobile user that connects to multiple networks in a region or if he travels worldwide. If he has access to a trustworthy and secure Virtual Private Network, then this becomes exceedingly difficult to target him and attack him.

Windows 8 users face a similar scenario, but the number of attackers targeting Windows users is enormous and the number of new attacks that are discovered before they are fixed occurs on a weekly basis while Microsoft Corporation or the software vendor in question pushes out updates and patches on a monthly or quarterly basis. They only push out patches and updates out of band for extremely critical software flaws and it usually takes a few days depending upon the criticality of the bug and the number of users affected by it.

I wish that I could stop using Microsoft products, but the reality is such that my alma mater and future university require me to use Windows and Office to connect to their on campus network to begin with. The other problem is that we live in a Microsoft dominated IT industry worldwide and it is very difficult to purchase and use hardware devices and software applications that cater to GNU/Linux and Ubuntu users specifically. Windows 8 will continue to be the gold standard by which most hardware and software vendors will develop products and services to work with over the next three years and this fact alone justifies the low cost to upgrade for me. I know that I am going to need to use Windows 8 or Office 2013 in the future. I just choose to use it inside a VM Ware Workstation 9.x 64 bit guest virtual machine for added safety and security. It won't destroy my primary default Ubuntu host operating system and there are no known VM Escape vulnerabilities or exploits for Windows 7 or 8 to compromise Ubuntu 12.04.x while running inside a guest virtual machine.

This is why System76 recommends that its users install and run Windows inside a guest virtual machine. It just makes sense to do so.
