### Post by ninjaaron on 2011-07-31
I deleted it after I got the DE setup.  It was just an learning experience.

> **Barrucadu said:**
> So… you can follow the Beginner's Guide / Xorg wiki page? :P

Mostly what I learned about was configuring network stuff on the command line. xorg was easy.  I installed it on a computer with no Ethernet port, so I had to configure the wifi, and I was getting weird messages... and now I know how wifi works... kinda... and the mirrors were acting weird...  So that's what I learned about... the internet... right after I invented it.

@geogur

Yeah Arch is a distro.  The core installation just leaves you with a terminal and a few cli apps including a package manager.  From there, you pull in whatever parts you want to create your own system.  Arch is famous for violently adhering to principles of simplicity in design.  It uses a rolling release system, so your packages are always bleeding-edge.  However, it still tends to be very stable cause all the official packages are vanilla without any distro-specific patches, working like they were designed to work.

I might use it for real one day, once I figure out how to set up a home file server with remote access, so I'm not addicted to UbuntuOne.

But it's just Linux, at the end of the day.  Same programs and UIs as every other distro, just that they are always the latest version so they basically have no development cycle and no releases for the distro as a whole.  Neat idea, but it's not a game-changer or anything.  Linux is Linux.
