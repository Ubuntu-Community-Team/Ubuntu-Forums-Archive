### Post by qyot27 on 2012-01-16
**Music Player / Media Player**
mplayer, or even mplayer2 (and since this is Windows, I don't mean mplayer2.exe, aka Windows Media Player 6.4 that isn't even usable on Vista and Win7, [I mean the fork of mplayer](http://www.mplayer2.org/)).  If you need a GUI, there's UMPlayer, which was forked away from SMPlayer and heavily revamped.  It works well with mplayer2 if you prefer that one.

If system-integrated (in other words, stuff that uses DirectShow) solutions are more appealing, Media Player Classic Homecinema.  It has internal filters but you can also use a combination of LAV Filters and/or ffdshow-tryouts to support a wider range of files.

**PDF viewer**
[Sumatra PDF](http://blog.kowalczyk.info/software/sumatrapdf/free-pdf-reader.html).  It even has the option to replace Adobe Reader as the system default.  I've been using it for years as my default reader since it requires so little resources.

**Ripping CDs to .ogg format**
[fre:ac](http://wiki.hydrogenaudio.org/index.php?title=Fre:ac)

Normally I don't rip to compressed formats (and actually use ImgBurn or EAC for dumping to WAV/CUE), but this seems to fit the bill.

**Burning CDs/DVDs with support for images.**
Well, if cdrecord works as well under Windows as it does on the various *nixen, then cdrecord.  And you can pack the data up into images with mkisofs first.  Maybe there are GUIs for those to make it easier, if that's needed.

(Although quite honestly, I'd just go with ImgBurn.)

**Screen Recording**
[CamStudio](http://en.wikipedia.org/wiki/CamStudio)

Not really in a personal position to vouch for it, since my computer is way too old to reliably use it.  However, I have played with it some, and as long as the machine is recent enough it shouldn't have much of an issue.

**Video editor**
[AviSynth](http://en.wikipedia.org/wiki/AviSynth).  Incredibly masochistic to get it to do non-linear editing, but if you were looking for an advanced NLE that's open source and available on Windows, it's really your only option (even though what you'll be using AviSynth for most of the time isn't editing at all, but pre- and post- work, or in tandem with industry software like Premiere or Vegas).  Prosumer-level video editing software is still sorely lacking in FOSS.

If you need to see the result of the scripts, you can open them in mplayer or [VirtualDub](http://www.virtualdub.org/) (VDub can also encode using VFW codecs, or you could use FFmpeg or x264 or whatever encoder you want directly).
