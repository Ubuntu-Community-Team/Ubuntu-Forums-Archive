### Post by QIII on 2012-03-01
Not objective.

Furthermore, a comparison would require that the same number of nearly identical packages were being processed.

For an objective assessment you would at least need:

The bandwidth capabilities of the server used, the number of users downloading and the saturation of the server's bandwidth.
A controlled number of downloads/installs each.
The same packages in the population.
A recorded measure of processing time.

Humans can be fooled easily.

For instance:
Lines at a single grocery store cashier may make the shopper feel the process is going slowly because each individual cashier line moves slowly.

A single line feeding into multiple cashiers may make the shopper feel the process is going more quickly simply because the line is moving more quickly.  The checkout process takes the same amount of time.

My *impression* is that yum goes faster than apt, but I have not analyzed it objectively.

I'm not knocking the question, really -- but it would be more valid to ask "Is apt slower than X?"

EDIT:  Actually, it might be worth it to see if someone like phoronix has done an objective test.
