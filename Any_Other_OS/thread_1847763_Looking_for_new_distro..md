### Post by oscarivera9 on 2011-09-22
You're not alone in this.  I also don't quite like this Unity thing.  I also understand that Gnome 3 is somewhat similar to Unity and KDE is a totally different ballgame, but if you like it there are lots of distros out there that use it.
As for what you heard about Linux Mint, here's the scoop:  Linux Mint is based on Ubuntu, which in turn is based on Debian, and it's always been like that.  It's like Debian is the grandpa, Ubuntu is the dad, and Linux Mint is the adult grandson that just had a new baby.  This new baby is called Linux Mint Debian Edition and it is exactly what it says.  It is more Debian than Ubuntu, which is what the original Linux Mint is supposed to be.  However, Linux Mint is not going away or changing or anything like that.  There is simply yet another version of Linux Mint that coexists with the original.  I believe all in all there are 5 different versions of Linux Mint.
  I know it all sounds kind of confusing, it was for me also; but now I think I get it.
The short answer is this: Keep using Linux Mint if that's what you like.  It is not going away.  Just make sure that you keep using the same edition that you like.  If you were using the KDE version then keep using that one, but remember that the main version uses Gnome and I belive they've already switched to Gnome 3.

### Post by kazuya on 2011-09-23
I like all suggestions. I am a distrohopper and have dabbled into almost all the popular ones and the not so well known ones. It is great for wolfen to grace us. I still remember wolvix the slack-based distro from a while ago that was simply amazing. 
 
Pclinuxos and Mepis as Xub uRoxMySox mentioned are some of the best kde-based distros in my opinion. Fedora is very nice, but a bit slow for me (subjective) compared to even the Debian-based distros. Opensuse is good, but not as responsive as I would like. Pclinuxos is one of the speedier rpm-based distros I have ever used. It holds up well against the Debian counterparts.
 
_**Chakra linux**_ is certainly deserving to be mentioned here when it comes to kde-based distros. It is the fastest of the whole bunch. (essentially based on arch) - drawback is that it is strictly kde-based distro.
 
Arch with kde is near unparallelled. But not quite for beginners or those that really require a graphical package manager identical to synaptic. There are some front-ends in archlinux for pacman, but I cannot attest to their state. I somehow got so used to pacman, packer, and yaourt via command line. It is faster in many regards than apt-get or synaptic for installing and uninstalling.
 
pacman -S kde
pacman -R kde
 
No surprise **_Archbang_** is always my first recommendation. The main reason I am not mentioning Vectorlinux is due the lack of a 64bit version.
