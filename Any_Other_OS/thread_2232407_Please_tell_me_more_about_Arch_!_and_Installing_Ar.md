### Post by Toz on 2014-07-01
A general description of what Arch is can be found [here]("https://www.archlinux.org/about/"). To summarize:
> Arch Linux is a versatile, and simple distribution designed to fit the needs of the competent Linux® user. It is both powerful and easy to manage, making it an ideal distro for servers and workstations. Take it in any direction you like. If you share this vision of what a GNU/Linux distribution should be, then you are welcomed and encouraged to use it freely, get involved, and contribute to the community. Welcome to Arch! 
Might also be useful to read through [The Arch Way]("https://wiki.archlinux.org/index.php/The_Arch_Way").

Arch has a very comprehensive [wiki]("https://wiki.archlinux.org/") - probably one of the best sources of linux information available (though naturally specific to that distro).

As for installing Arch, the [beginner's guide]("https://wiki.archlinux.org/index.php/beginners%27_guide") is where to get started. Prepare yourself for a bit of a learning curve. The control is in your hands.

### Post by malspa on 2014-07-02
Keep an eye on Arch's main page and the Arch forums -- helps quite a bit with avoiding any major problems that might be caused by package upgrades.

Don't go too long before bringing in updates. I try to do it at least once a week.

There are some nice Arch derivatives out there. They'll help you to get a bit of a feel for Arch, and are quicker and easier to install than Arch is, but they won't teach you as much about Arch as you'll learn once you actually install Arch itself. I tried out Bridge Linux, Chakra, and ArchBang before installing Arch. Those gave me some practice with the pacman package manager, and after seeing how nicely things went with those distros, I was convinced to try Arch.

I did a couple of practice installation on a spare computer because I wasn't confident about being able to install Arch. And took good notes. That kinda helped for when I went ahead with the "real" installation.

When you do the installation, just carefully follow the steps and read the documentation and you should be fine. I used the Beginner's Guide ([https://wiki.archlinux.org/index.php/beginners%27_guide](https://wiki.archlinux.org/index.php/beginners%27_guide)) as well as Installation Guide ([https://wiki.archlinux.org/index.php/Installation_Guide](https://wiki.archlinux.org/index.php/Installation_Guide)).

As has already been mentioned, the wiki is great. There's also tons of great info at the forums, if you take the time to search. I haven't needed to post questions at the Arch forums, but I did get a little help elsewhere (mainly at the Bruno's All Things Linux section of the Scot's Newsletter Forums). But web searches and the wiki have been the best sources of info for me.

Arch is awesome. Right now, I'd have to say that Debian and Arch are my favorite distros, but for different reasons, mostly. I like pacman better than any package manager I've used from the command line.

### Post by grier-devon on 2014-07-09
If you want to install Arch without the mess then just use this....

[http://sourceforge.net/projects/evolutionlinux/](http://sourceforge.net/projects/evolutionlinux/)

This will install the base Arch, desktop of your choice and some select software but is still pure Arch.

First thing to do is setup yaourt after your install of Arch.
[https://www.digitalocean.com/community/tutorials/how-to-use-yaourt-to-easily-download-arch-linux-community-packages](https://www.digitalocean.com/community/tutorials/how-to-use-yaourt-to-easily-download-arch-linux-community-packages)

Arch is as awesome as you decide to make it, just remember not everything works out of the box like Ubuntu, but usually can be fixed with a simple dependency. Example being I had some work to do to get hplip installed and working. Many games in steam to work required me to get additional packages installed. You also may sometime have to start services before you can use some applications like Bittorrent Sync and Teamviewer.

BIGGEST KEY: When reading wiki, guides and blogs on how to do something never skip steps and read everything first and then go back and attempt.
