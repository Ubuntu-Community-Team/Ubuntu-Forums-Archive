### Post by mendhak on 2011-09-14
I've been trying it out for a few hours now and it basically wants to be a tablet OS and a desktop OS at the same time.  

There are a few nice things like the improved task manager and Visual Studio 11.  

[Task Manager 1]("http://ubuntuforums.org/attachment.php?attachmentid=202191&stc=1&d=1316034889")
[Task Manager 2]("http://ubuntuforums.org/attachment.php?attachmentid=202190&stc=1&d=1316034889")



[Visual Studio 11]("http://ubuntuforums.org/attachment.php?attachmentid=202192&stc=1&d=1316034835")



There's a user-friendly [control panel]("http://ubuntuforums.org/attachment.php?attachmentid=202189&stc=1&d=1316034889") for basic operations.  

There are a few annoying things, the first one being that the Start button takes you to the [metro home screen]("http://ubuntuforums.org/attachment.php?attachmentid=202188&stc=1&d=1316034835").  I also noticed that metro apps can't be closed, they simply get suspended when you move away from them.  The apps that you'd normally install aren't easy to get to now.  I'm still working on that one.  The metro home screen does come with a list of apps that it thinks you would like to access.  Suppose I install Winamp and months later (when it's not on the metro screen), I want to get to it.  I can't see how I'd do that, it must be something BLINDINGLY obvious that I'm missing.

That said, I'm wondering, with the desktop apps running on an OS meant for tablets too, whether there will be some sort of a segregation or cutoff so that it doesn't get overloaded and die quickly.  

BTW, to get the Win7 style start menu back, change HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\RPEnabled to 0
