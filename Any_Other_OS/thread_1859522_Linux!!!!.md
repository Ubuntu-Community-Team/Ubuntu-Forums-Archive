### Post by gradinaruvasile on 2011-10-14
Well those issues are not related to specific distros. Those are releted to specific applications.

Yahoo protocol - there isnt a free client (and i dont know any paid one either) that has voice capabilities in the Yahoo network (AFAIK the Jitsi devs are working or planning to work on it, but it isnt on the very urgent todo list yet).

Best Yahoo protocol compatibility goes to Gyachi though - it has suppport even for webcams (but not one-to-one voice), photo sharing, file transfer and stuff like themes to certain extent and many config options.

Working voice apps:

Skype - great sound quality and the program is stable (with the exceptions when decides to abort after a few seconds - something that happens on Both Linux and Windows). Skype has clients for Linux, Windows and Mac and they work well in any combination.
But Microsoft acquired it so non-Windows Skype users are at their vim. Google seems more open minded and platform-agnostic.

Jitsi - Multi protocol messenger - supports Yahoo (text/file transfer), Google talk (text/file transfer/voice/video), SIP (voice/video/text - even direct IP-to-IP calls), Jabber (voice/video/text/file transfer), Fabebook (text) and many others.
It has good sound quality and low bandwidth if Jitsi is used on both ends with Jabber/Google talk accounts - make sure you select the silk 24000 codec (the one used by Skype or very close to it) on the top of the list. It will work with most other google talk apps out there anyway, but it probably will use a lower quality codec like PCMU.
It also has supplementary srtp/zrtp on the fly end-to end encryption (again, if Jitsi is used by both parties), meaning that even if the data flows through a server it is encrypted in a way only the 2 or more talking parties can decrypt it.

Gmail web plugin - this is a browser plugin (works in all browsers i think) that can be installed separately (very easy, the link is given by the gmail site) and it permits voice and video calls between google mail accounts - it is also interoperable with Pidgin/Jitsi/Empathy (most google talk client programs).
The installation is very simple and you need only a browser (and a gmail account of course).

PS. Most issues related to voice over ip applications i have seen are in fact related to microphone misconfiguration (Both on Linux and Windows), not the applications themselves.
Also, certain voip protocols work better than others when used over the internet - SIP for example does not fare as well as Jabber/Google Talk/Skype.

Before slamming the Ubuntu/Kubuntu/whatever developers, keep in mind how the development process works:

First, the protocol behind the messenger needs to be available; without that, the developer would have to reverse-engineer a chat program, which can be legally problematic. 

Second, the Ubuntu developers ([www.canonical.com](www.canonical.com)) don't have much power over a lot of the software. The programs used in Ubuntu are "upstream" programs. If you want to push for better support in a certain area, then get involved with the developers of those individual projects. 

The IM situation in Linux is a little bit bad, yes. But it also depends on what sort of people you talk to. For me, all I need is Epiphany and Skype. Good luck :)
