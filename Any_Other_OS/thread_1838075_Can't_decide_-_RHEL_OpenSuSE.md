### Post by NightwishFan on 2011-09-03
I recommend Debian Stable as well. I am using it to post this right now.

When shopping for an Ubuntu replacement my criteria for picking a distro was:

[LIST]
[*]Preferably Debian based as I know the core/administration of very well. No better pick than Debian itself.
[*]Time based freeze release cycle. Debian is now on a 2 year cycle, with 3 years of support. This is equal to Ubuntu LTS on the desktop and there is nothing stopping a savvy administrator using it longer.
[*]Must be able to upgrade to a new version. An apt-get dist-upgrade might be a bit daunting to newbies (even if its mostly just pressing enter to a few questions). However it has never failed me. Debian is designed to be upgraded.
[*]Released Stable. Rather than rolling up to a point a distro must try to have only release ready version of packages. A good period for a freeze where bugs are fixed.
[*]Must run on 64-bit, though having more Archs is a very awesome bonus point. Debian runs on 10 or so arches and 3 different kernels.
[*]There is more but those are the major ones.
[/LIST]

Basically you can see I was quite picky. Ubuntu LTS works for most of this, except I much prefer to not use Ubuntu any longer. I am very conservative and Ubuntu is not.

CentOS/Scientific Linux/RHEL was another choice but it did not seem to be geared toward random programmer/multimedia/web users as much. I would use it more for workplace desktops and servers. Debian can be used in both situations quite well.

There are other Debian based systems such as Mephis (Debian Stable with upgraded KDE, very very nice). Zenix (lightweight and elegant and again fully based on Debian Stable.

Mint is just Ubuntu, sorry but it is. Mint Debian (please change the name to something else like Mint rolling) is a rolling release which is fine if you like rolling dice if your system works or if package A is available next week. No thanks. Debian Testing/Sid works surprisingly well but I do not like the rolling model.

Arch. I hear very good things but 1)I do not like to have to customize every darn aspect of my system, just slap down a binary general purpose Gnome for me! 2)It is a rolling release. :(

OpenSUSE is a very good distro but I find it a bit buggy. Just a bit. It was my first distro I really enjoyed back at version 10.3. Way ahead of Ubuntu at the time.

Fedora is just too 'new' for me. Great features and attention to detail. I also like how it feels very 'enterprisey' and yet modern. Again, just a bit too new but I give it a thumbs up.

So I am sure there are more distros but these are the ones I like. Out of all of them Debian was the best fit. Very light, very conservative, incredibly universal(works on many different types of computers and is not geared toward a single purpose) and best of all reliable. A nice difference is Ubuntu's main supported packages selection is very trim. Debian's 'main' of supported packages is HUGE.

Been using it full time about a year now and I love it. Not going back. The only cons are it takes a bit to get used to and is slightly more manual than Ubuntu, but not as much as people say. It is kind of a 'once you set it up, you are set' kind of deal. No non-free software by default but it is easily available. A Debian Live install out of the box plays pretty much every video file except super new WMV or Realplayer. A bit of command line knowledge (running a few commands) is pretty much essential if you want proprietary drivers. It is normally copy/paste a few commands.

It has the update manager and software centre from Ubuntu so it has stuff you are used to.

I say Debian. :)

### Post by kaldor on 2011-09-03
For a long-term OS, I highly recommend Scientific Linux 6.1 over Debian. 

Scientific Linux is a RHEL-based distro with some extras (such as Firmware and Flash) included by default. It's easier to set up and maintain than Debian is from my experience, and it works with all the RHEL 6.x software and repositories. Don't let the name put you off; it's a very pleasant and stable desktop OS for most people. 

More info: [http://scientificlinuxforum.org/index.php?showtopic=924](http://scientificlinuxforum.org/index.php?showtopic=924)

Please note that I am not trying to put down Debian... I just disagree somewhat with Debian's way of doing things and lack of certain firmware/drivers that I need which require additional tinkering.

Edit: I just noticed you looking at OpenSUSE. From my experience, it's extremely bittersweet. It's not a very stable or welcoming experience and tends to break a lot; it's the only distro that really gave me true headaches. If you want a stable system, don't bother with SUSE.

Edit 2: Some good SL resources here: [http://scientificlinuxforum.org/index.php?showforum=6](http://scientificlinuxforum.org/index.php?showforum=6)

Edit 3: Should also add that their community IRC Channel (#slforum at Freenode) has some of the most helpful and respectful people I have ever found in a Linux support community.
