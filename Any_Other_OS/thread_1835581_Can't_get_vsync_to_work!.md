### Post by BicyclerBoy on 2011-08-30
If you are doing video decode/playback using VDPAU then there are **no** settings for vsync.
VDPAU is always vsync & guaranteed not to tear.

The problem sounds like compiz & unity.
Compiz traditionally caused problems with nVidia & twinview.

Have you tried using ccsm ? and setting the refresh rate manually ?

Compiz (nvidia 266) runs fine on 10.04 & separate X screens but this is not to everyone's taste. You need the later driver..

If you don't mind the classic desktop..you could remove (uninstall) unity & compiz; install metacity.

The gt520 is a good cheap low power HTPC card but a bit weak for VA de-interlace.
This card is not an openGL powerhouse.

### Post by BicyclerBoy on 2011-09-03
Not saying I understand this but AFAIK&U.
Twinview uses a meta-mode to create 2 screens in 1. It also uses a bogus refresh rate to get a unique identifier.
The twinview refresh rate reporting to composite managers seems to cause the problems.
The composite manager gets the bogus refresh rate not the real one.
A similar problem occurs with apps getting the wrong screen info & not working full-screen.

Note that twinview requires both monitors to have the exact same refresh rate.

Check out this thread from 1 day ago..
[http://ubuntuforums.org/showthread.php?t=1683875](http://ubuntuforums.org/showthread.php?t=1683875)
