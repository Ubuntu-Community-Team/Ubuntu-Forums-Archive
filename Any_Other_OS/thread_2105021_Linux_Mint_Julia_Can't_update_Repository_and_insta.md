### Post by xenopeek on 2013-01-16
To help us help you, next time please just copy the text from the terminal to a support request. That way we can copy bits of it (like the URLs) and have a look or Google with it easily, without having to type it from your screenshot. You can copy text to/from the terminal, see its Edit menu.

But yes, Linux Mint 10 is based on Ubuntu 10.10 and both have been obsoleted and reached end of life since April 2012. Recommend you upgrade to a newer release, if you want to install software.

If you were using the main edition, with GNOME 2, you will probably like the MATE edition of Linux Mint. Either Linux Mint 13 MATE, the current long-term support release (supported till April 2017), or Linux Mint 14 MATE, the latest release with the newest software (supported till April 2014).

### Post by Erik1984 on 2013-01-16
In Ubuntu it is possible to point the sources to the old releases repository so you can still apt-get install software that was part of that particular version including all updates up till the EOL date. [https://help.ubuntu.com/community/EOLUpgrades](https://help.ubuntu.com/community/EOLUpgrades) Maybe this can be applied to Linux Mint as well? I have tried this with Xubuntu 7.10 in the past and it worked great. It is still better to reinstall a newer, **supported**, version.
