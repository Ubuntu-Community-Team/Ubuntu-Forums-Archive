### Post by manzdagratiano on 2011-05-02
Debian unstable (Sid) is most updated. It is theoretically also the most unstable, though I do not think that is true any more, since in the last year or so they have added another layer - Experimental - above Sid. So Sid is probably not as bad as it used to be. Testing is almost as good as Stable - because passing to Stable means being able to build on all 13 architectures supported by Debian, and since x86 is usually the first one to be supported, it means that most of Testing should be as good as stable for x86. It does not break as bad as Sid when it breaks, but when it does break, things take longer to fix in Testing than they do in Sid, because while the new patch will quickly go to Sid, to come down to Testing it may depend on other libraries which are not in Testing.

I used to run Debian Testing, but I have been running Debian Sid for a year now and it has never broken on me. The key is to install 'apt-listbugs', which tells you the serious issues some packages which are to be upgraded might have. That information and some common sense will tell you whether or not to upgrade. If an upgrade looks bad, just don't do it for a couple of days and it is usually fixed.

I used to once also run Debian Stable, and it is great and rock solid by itself, but it is now a problem since I run Ubuntu/Arch/Slackware with it and these ones are usually much more bleeding edge. Hence software compatibility with old is not always a given - for instance LyX - 1.6.7 refuses to open what I made in 2.0.0rc3 etc.

Just explore the Debian web page, and they give you an insane amount of information.
