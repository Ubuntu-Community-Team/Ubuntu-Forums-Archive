### Post by chrisdeckard on 2008-03-01
The great thing about Linux is that it runs great with less memory, but it runs better with more.  Like the previous post says, it depends what you want to do.  If you're planning to have many applications open, do development (programming), graphics work, or video or other multimedia, you will quickly want more.  In the last year I've run with 1GB, 2GB, and now 4GB.  Everything is fine with 1GB, but the more you give it, the more you'll be glad you did.

If you're just doing web browsing, 1GB is more than enough.

-Chris

### Post by kaens on 2008-03-01
Indeed, I've found that linux systems scale pretty well to available memory - although you're going to have to do some learning if you want to use it on a low-memory system.

I do a lot of development on a thinkpad R61e with a gig of ram in it, running kubuntu, dual-booting with windows XP pro, and kubuntu runs *significantly* faster than windows with default settings for both.

I can tweak XP to run as fast as the kubuntu defaults, but it takes some esoteric changes. I can't tweak XP to run as fast as a tweaked kubuntu.
