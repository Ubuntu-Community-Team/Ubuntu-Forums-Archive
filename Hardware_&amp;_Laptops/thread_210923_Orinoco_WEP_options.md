### Post by Bonkabonka on 2006-07-07
Hi!

I've got a pokey old Toshiba Libretto L5 with the integrated Orinoco-based wireless (or at least that's the driver that gets loaded).  It works fine until I enable WEP but then it refuses to connect to my LAN.  Windows also has this problem most of the time but I was able to bash it into working once and was afraid to touch it after that.  :)

That said, my experience with Ubuntu has been astonishing.

It.  Just.  Works.

I am utterly blown away by how painless it's been.  The L5 is an import and came only with a Windows XP cd (which is amusing considering that there isn't a CD drive in the machine and attaching one requires a specific model external drive with a proprietary PCMCIA controller).  Nothing else was provided (discounting the couple sheets of documentation in Japanese - which I cannot read).  I haven't been able to find official installers for the various windows utilities (hotkey support, acpi bits and transmeta CPU tools) but now I don't need 'em.  PXE boot to the installer, wait a while for the stuff to get streamed to the harddrive and voila.

Wacky screen resolution (1280x600)?  Detected out of the box and configured properly.

Oddball ALi southbridge?  No problem - not even with the sound controller or IDE interface (this used to trip up folks back when Fedora was a pup).

So, other than the WEP thing - which also gives Windows gas, I'm golden.  Incredible work!  This whole Linux on the desktop might not be so far-fetched after all.  :D
