### Post by mwinokur on 2007-06-28
For me the solution was to roll back the i810 driver.  In my case I use Debian testing so I needed to uninstall these 3 packages: 
dpkg -r 
xserver-xorg-video-intel 
xserver-xorg-video-all
xserver-xorg-video-i810

I then downloaded the xserver-xorg-video-i810 in the stable branch and installed it:
dpgk -i xserver-xorg-video-i810_1.7.2-4_i386.deb

After this my original xorg.conf (dual head) worked again.

Note these errors remain in the /var/log/kdm.log file:
(EE) intel(1): detecting sil164
(EE) intel(1): Unable to read from DVOI2C_E Slave 112.
(EE) intel(1): Unable to read from DVOI2C_E Slave 236.

(EE) I810(0): [dri] DRIScreenInit failed. Disabling DRI.
(EE) AIGLX: Screen 0 is not DRI capable
