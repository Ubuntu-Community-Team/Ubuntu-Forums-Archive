### Post by daller on 2006-09-03
Well, there's only ONE mouse configured in your Xorg.conf!

If that one is not plugged, no mouse is configured!

X needs at least 1 mouse to start!

I expect that your laptop has a touchpad!

...that touchpad should be configured individually from the generic mouse!

I have a part like this in my xorg.conf: (below the generic mouse!)

```

Section "InputDevice"
        Identifier      "Synaptics Touchpad"
        Driver          "synaptics"
        Option          "SendCoreEvents"        "true"
        Option          "Device"                "/dev/psaux"
        Option          "Protocol"              "auto-dev"
        Option          "HorizScrollDelta"      "0"
EndSection

```

And in Serverlayout:

```

        InputDevice     "Synaptics Touchpad"

```

### Post by mscherzer on 2006-09-13
G'Day,
 I finally resolved the issue of the x-server refusing to start / crashing without connecting a USB mouse (when using the evdev driver.) :)  
 
  Hopefully this will also apply to the issue you describe, here is what I did:
  
 Apparently the x-server requires a “CorePointer” in order to start, so I configured the “Synaptic Touchpad” as the “CorePointer” by adding the line:
  *Option         "CorePointer"*
  to the Synaptics Touchpad Section. To prevent a conflict I also removed the line
  *Option         "SendCoreEvents" "true"*
  as the x-server would not start with this line present in the “Touchpad Section”.
 
 To continue using my usb mouse (logitech MX510) when connected I had to set the mouse to send core events, thid I did by adding the line:
  *Option         "SendCoreEvents" "true"*
 to the “Configured Mouse Section”. Since (as far as I know) only one input device can be configured as the CorePointer I also remove this option from the “Configured Mouse Section”. 
 
  The working device sections look like this:
 
 ```
Section "InputDevice"
      Identifier     "Configured Mouse"
      Driver         "evdev"
  #   Option         "CorePointer"
      Option         "SendCoreEvents" "true"
      Option         "Device" "/dev/input/event9"
      Option         "Buttons" "10"
      Option         "Resolution" "800"
  EndSection 
 
  Section "InputDevice"
      Identifier     "Synaptics Touchpad"
      Driver         "synaptics"
      Option         "CorePointer"
  #   Option         "SendCoreEvents" "true"
      Option         "Device" "/dev/psaux"
      Option         "Protocol" "auto-dev"
      Option         "HorizScrollDelta" "0"
  EndSection
  
```hope this works for you.
