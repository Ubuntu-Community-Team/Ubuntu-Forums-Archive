### Post by Clunsford on 2007-03-05
I know there is often complain about how Ubuntu performs poorly with the suspend and hibernate features on notebooks.

On an IBM T30, Ubuntu 6.10; 2.6.17.10 generic, I experience some issues with the notebooks autosuspend, where screen blanking will occur, and will not bring up video on keyclick/mouseclick.

a ghetto workaround to this i found works by, when blanked out like that, pressing Function (Fn, the blue command key) and F4; this gives the machine a hardware initiated call to sleep, which Ubuntu happly complies with. After it has fallen to sleep, pressing the same Fn F4 (or a tap on the powerbutton) will wake the device and you will be greeted with a password prompt.

I just figured I would give back this particular method back to the community (I know its not the propper linuxish fix where I give you infromation how to recompile some library or kernal or what not to ACtually fix the problem :) )  Also, if I should post this info to the buglists, how should I go about it?
