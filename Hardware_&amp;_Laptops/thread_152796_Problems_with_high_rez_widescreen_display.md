### Post by bluevoter on 2006-04-08
I was able to get the X server working by changing the device from "i810" for the Intel 945GM chipset to "vesa" and adding a line to use 12M of video RAM. Unfortunately, the max resolution for vesa is 1280x1024, which creates an incorrect aspect ratio on the display, making it impossible to work with images and skewing web pages.  

I didn't want to use the 915resolution hack, since it requires applying it every time you boot.

So I am still looking for a straightforward solution (no kernel recompilation).  I've seen some discussions around this problem for the Dapper release (6.04), but it doesn't appear to be resolved yet, so I haven't tried the Flight 6 beta.  

On a related topic, has anyone had success in getting full and correct 1650x1080 resolution with a different distro?

### Post by ghakko on 2006-04-08
To avoid having to run "915resolution" manually on startup, you need to do just two things:
[LIST]
[*] Edit "/etc/default/915resolution" and set your target resolution/bit depth.
[*] Run "update-rc.d 915resolution defaults 12" to create startup links for the 915resolution boot script.
[/LIST]

---

### Post by ghakko on 2006-04-08
You'll also need 1.4.1.3-0ubuntu2 (or later) of the "xserver-xorg-driver-i810" package. Previous versions did not include the PCI ID of this video chipset, which would have prevented the X server from even starting with the "i810" driver.
