### Post by errenay on 2007-08-04
The following how-to was prepared by FokkerCharlie & me, based on solutions posted in this thread by Deinstein, FokkerCharlie, Neptunus, me & other Acer owners.

**Problem:** In case of Acer Aspire 5920, Ubuntu recognizes two touchpads (first is the "real touchpad" and second are the "multimedia keys"). Because these touchpads are mismatched, many problems arises (no "right tap", multimedia keys do not work, etc.). This error could be solved manually.

**Solution:**
[CENTER]***[SIZE="4"]Follow this guide at your own risk. I strongly recommend that you only proceed if you are familiar with recovery mode, and will be able to undo anything that you do here.[/SIZE]***

Additionally, if anyone tries this, please provide feedback on if/how well it worked, so that the howto can be tweaked if required.[/CENTER]

1. First, backup your current xorg.conf file.  Open a terminal and type:
```
sudo cp /etc/X11/xorg.conf /etc/X11/xorgbackup.conf
```

2. Then, open up your xorg.conf in favourite editor. For example in case of Ubuntu, use
```
sudo gedit /etc/X11/xorg.conf
```
In case of Kubuntu, use
```
sudo kedit /etc/X11/xorg.conf
```
Look for any "InputDevice" sections that reference 'synaptics' 'touchpad' or 'acermediakeys'.  You should find two, one for the touchpad proper, one for the media keys.  Delete these sections, or, if you prefer, comment out (place a # symbol at the start of each line) in the section, including the Section and EndSection lines.

3. Then, find the "ServerLayout" section, normally at the end of your xorg.conf.  Delete (or comment out) the lines corresponding to the touchpad/mediakeys sections that you just deleted.

4. Insert this into your xorg.conf:
```
Section "InputDevice"
   Identifier       "TouchPad"
   Driver           "synaptics"
   Option           "CorePointer"
   Option           "Protocol" "event"
   Option          "Device"   "/dev/input/eventtouchpad"
   Option           "MaxTapTime"  "100"
   Option           "MaxTapMove"  "220"
   Option           "AccelFactor"	        "0.05"
   Option           "SHMConfig" "true"
EndSection

Section		"InputDevice"
	Identifier	"AcerMediaKeys"
	Driver		"evdev
	Option        "Device" "/dev/input/eventkeys"
        Option        "SendCoreEvents" "true"
EndSection
```

You **MUST** insert the above text BEFORE any section referring to another mouse.

5. Find section which refers to your mouse (with identifier "Configured mouse" or something similar). Delete the line:
```
   	Option          "CorePointer"
```
And put in the same place the line:
```
	Option		"SendCoreEvents"	"true"
```

6. In the "ServerLayout" section, insert the following lines:
```

	InputDevice	"TouchPad"
	InputDevice	"AcerMediaKeys"
```

7. Now, save and close your xorg.conf. This is example of mine (your file may be different!):

8. Now ensure that there are no other scripts in the startup folder that may be dealing with the touchpad (for instance one that you may have created during a previous attempt!).  You can do this with nautilus (type sudo nautilus from the terminal to get root privileges first), or use this code:
```
sudo rm /etc/udev/*acer*
```

9. We now need to create a small script to run on startup, so back in the terminal (in Ubuntu):
```
sudo gedit /etc/udev/rules.d/10-acersynaptics.rules
```

In Kubuntu:
```
sudo kedit /etc/udev/rules.d/10-acersynaptics.rules
```

Insert this text:
```
SUBSYSTEMS=="input", ATTRS{phys}=="isa0060/serio4/input0", KERNEL=="event*", NAME="input/eventtouchpad"
SUBSYSTEMS=="input", ATTRS{phys}=="isa0060/serio3/input0", KERNEL=="event*", NAME="input/eventkeys"
```

10. Save and close the file.

11. OK, we're nearly there.  If you would like, you may install the Touchpad settings manager, so go into synaptic package manager, and search for 'gsynaptics'.  Install this program.

12. Reboot.  Your touchpad should now be working.  You may need to configure it to work as you wish by going to System > Preferences > Touchpad.

13. Install xbindkeys. In console type:
```
sudo apt-get install xbindkeys
```

14. Then set xbindkeys to start automatically. Because I'm using Kubuntu, I have placed link to the xbindkeys in Autostart
```
ln -s /usr/bin/xbindkeys ~/.kde/Autostart/xbindkeys
```
In case of other distros, for example you may add:
```
/usr/bin/xbindkeys
```
in your .bashrc file.

15. Create .xbindkeysrc file (in your home directory) with your favourite editor. Place the text below in this file
```
"amarok -t"
   b:17

"amarok -s"
   b:18

"amarok -r"
   b:19

"amarok -f"
   b:20

```

15a. Alternate method (that does not start Amarok whenever a media key is pressed) - place these lines in .xbindkeysrc:
```
#Map 'e' key to launch amarok
"amarok %U"
    c:159

#Mutlimedia Play/pause:
"xvkbd  -text "\[Control_L]\[Alt_L]\[p]""
    m:0x0 + b:17
#Stop: 
"xvkbd -text "\[Control_L]\[Alt_L]\[s]""
   m:0x0 + b:18
#Multimedia fwd/back:
"xvkbd -text "\[Control_L]\[Alt_L]\[z]""
    m:0x0 + b:19
"xvkbd -text "\[Control_L]\[Alt_L]\[x]"" 
    m:0x0 + b:20
```
If you're implementing this, you need to go to Amarok->Configure Global Shortcuts, select the shortcut in question, choose 'Advanced', click the alternate shortcut (hence leaving the original setting unchanged), and press the media button.

15b. The other alternate method. You may use GUI for creating the shortcuts. Just install additional package:
```
sudo apt-get install xbindkeys-config 
```
and then use 'xbindkeys-config' to make your own shortcuts.

15c. In case of Rhythmbox you may use these lines in your .xbindkeysrc:

```

"rhythmbox-client --play-pause"
   b:17

"rhythmbox-client --pause"
   b:18

"rhythmbox-client --previous"
   b:19

"rhythmbox-client --next"
   b:20

```

16. Restart your session. Now Amarok or your other favourite player should work!

### Post by errenay on 2007-09-20
As I have previously mentioned, there are two touchpads in /proc/bus/input/devices. The first device (isa0060/serio3/input0) is connected with multimedia keys (play/pause/stop, etc.). The second one (isa0060/serio4/input0) is real touchpad. I have prepared crude script and the rules for udev to manage the real touchpad.
This is file "acersynaptics.rules":
```

SUBSYSTEMS=="input", ATTRS{phys}=="isa0060/serio4/input0", KERNEL=="event3", RUN+="/usr/sbin/eventtouchpad 3"
SUBSYSTEMS=="input", ATTRS{phys}=="isa0060/serio4/input0", KERNEL=="event4", RUN+="/usr/sbin/eventtouchpad 4"
SUBSYSTEMS=="input", ATTRS{phys}=="isa0060/serio3/input0", KERNEL=="event2", RUN+="/usr/sbin/eventkeys 2"
SUBSYSTEMS=="input", ATTRS{phys}=="isa0060/serio3/input0", KERNEL=="event3", RUN+="/usr/sbin/eventkeys 3"

```
(there is link "30-acersynaptics.rules" in "/etc/udev/rules.d" to this file).
The script "eventtouchpad"
```

#!/bin/bash

rm /dev/input/eventtouchpad
ln -s /dev/input/event"$1" /dev/input/eventtouchpad

```
and "eventkeys"
```

#!/bin/bash

rm /dev/input/eventkeys
ln -s /dev/input/event"$1" /dev/input/eventkeys

```
(both in "/usr/sbin").
And the code in xorg.conf for the touchpad:
```

Section "InputDevice"
   Identifier       "TouchPad"
   Driver           "synaptics"
   Option           "CorePointer"
   Option           "Protocol" "event"
   Option          "Device"   "/dev/input/eventtouchpad"

   Option           "MaxTapMove"  "220"
   Option           "AccelFactor"	        "0.05"

```

Ad 1 & 2.This code is for file "acersynaptics.rule" which is used by udev mechanism to create the links "/dev/input/eventtouchpad" and "/dev/input/eventkeys". You should create such file, put it somewhere (for example in "/etc/udev") and then create link entitled "30-acersynaptics.rule" in "/etc/udev/rules". To create the link you have to use code
```
ln -s /etc/udev/acersynaptics.rule /etc/udev/rules/30-acersynaptics.rule
```
Of course you need to have root privileges (use "sudo") to create this file and link. If everything is OK, use commands
```
udevcontrol reload_rules
udevtrigger
``` 
(also as root) and check if there are new links (eventtouchpad & eventkeys) in "/dev/input".
Ad 3. If you have done all previous steps, check your xorg.conf and change appropriate lines for the touchpad. Probably you have already had the section for touchpad, so change only necessary lines.

Your error means that (probably) you did not create rules for udev mechanisms or they do not work.

More here:
[http://www.linux.com/feature/118108](http://www.linux.com/feature/118108)
[http://reactivated.net/writing_udev_rules.html#builtin](http://reactivated.net/writing_udev_rules.html#builtin)

### Post by B00R4dL3y on 2008-05-09
Here's what I have for my /etc/udev/acersynaptics.rules file after installing/upgrading to Ubuntu 8.04.  Basically the event numbers changed.  They're probably the same for you if you just installed 8.04; otherwise you'll have to track down which events you're touchpad is connected to.  I noticed that now the media keys play/pause works as scroll-up and stop now works as scroll-down on my keyboard.

```
SUBSYSTEMS=="input", ATTRS{phys}=="isa0060/serio4/input0", KERNEL=="event9", RUN+="/usr/sbin/eventtouchpad 9"
SUBSYSTEMS=="input", ATTRS{phys}=="isa0060/serio4/input0", KERNEL=="event4", RUN+="/usr/sbin/eventtouchpad 4"
SUBSYSTEMS=="input", ATTRS{phys}=="isa0060/serio3/input0", KERNEL=="event8", RUN+="/usr/sbin/eventkeys 8"
SUBSYSTEMS=="input", ATTRS{phys}=="isa0060/serio3/input0", KERNEL=="event9", RUN+="/usr/sbin/eventkeys 9"

```

If you have a mouse plugged in when you boot-up it may changed the event numbers for the touchpad.  So don't have any other mouse-like devices plugged in when booting up.

Hope this helps you out.

First opend a terminal make the eventtouchpad and eventkeys scripts.

Create eventtouchpad script:
```
sudo gedit /usr/sbin/eventtouchpad
```

(copy the text below into the text editor and save/quit)
```
#!/bin/bash

rm /dev/input/eventtouchpad
ln -s /dev/input/event"$1" /dev/input/eventtouchpad
```

Create eventkeys script:
```
sudo gedit /usr/sbin/eventkeys
```

(copy the text below into the text editor and save/quit)
```
#!/bin/bash

rm /dev/input/eventkeys
ln -s /dev/input/event"$1" /dev/input/eventkeys
```

Next make them executable with the commands below:
```
sudo chmod +x /usr/sbin/eventtouchpad
sudo chmod +x /usr/sbin/eventkeys
```


Now we need to make the acersynaptics.rules file.  This is what works for me after installing Hardy.  The event numbers have changed from 7.10
```
sudo gedit /etc/udev/acersynaptics.rules
```

(copy the text below into the text editor and save/quit)
```
SUBSYSTEMS=="input", ATTRS{phys}=="isa0060/serio4/input0", KERNEL=="event9", RUN+="/usr/sbin/eventtouchpad 9"
SUBSYSTEMS=="input", ATTRS{phys}=="isa0060/serio4/input0", KERNEL=="event4", RUN+="/usr/sbin/eventtouchpad 4"
SUBSYSTEMS=="input", ATTRS{phys}=="isa0060/serio3/input0", KERNEL=="event8", RUN+="/usr/sbin/eventkeys 8"
SUBSYSTEMS=="input", ATTRS{phys}=="isa0060/serio3/input0", KERNEL=="event9", RUN+="/usr/sbin/eventkeys 9"
```

Create link entitled "30-acersynaptics.rule" in "/etc/udev/rules"
```
ln -s /etc/udev/acersynaptics.rule /etc/udev/rules/30-acersynaptics.rule
```

Next you need to modify your /etc/X11/xorg.conf file.  Make sure you have a backup of your xorg.conf incase something goes wrong.
```
sudo gedit /etc/X11/xorg.conf
```

### Post by FokkerCharlie on 2008-06-26
OK, by popular demand, here's a howto, distilled from the information in the thread, particularly from Deinstein and Errenay.

[CENTER]***[SIZE="4"]Follow this guide at your own risk. I strongly recommend that you only proceed if you are familiar with recovery mode, and will be able to undo anything that you do here.[/SIZE]***

Additionally, if anyone tries this, please provide feedback on if/how well it worked, so that the howto can be tweaked if required.[/CENTER]

First, backup your current xorg.conf file.  Open a terminal and type:

```
sudo cp /etc/X11/xorg.conf /etc/X11/xorgbackup.conf
```

Then, open up your xorg.conf in gedit:
```
sudo gedit /etc/X11/xorg.conf
```
Look for any "InputDevice" sections that reference 'synaptics' 'touchpad' or 'acermediakeys'.  You should find two, one for the touchpad proper, one for the media keys.  Delete the sections, or, if you prefer, comment out (place a # symbol at the start of each line) in the section, including the Section and EndSection lines.

Then, find the "ServerLayout" section, normally at the end of your xorg.conf.  Delete (or comment out) the lines corresponding to the touchpad/mediakeys sections that you just deleted.

Insert this into your xorg.conf:

```
Section "InputDevice"
   Identifier       "TouchPad"
   Driver           "synaptics"
   Option           "CorePointer"
   Option           "Protocol" "event"
   Option          "Device"   "/dev/input/eventtouchpad"
   Option           "MaxTapTime"  "100"
   Option           "MaxTapMove"  "220"
   Option           "AccelFactor"	        "0.05"
   Option           "SHMConfig" "true"
EndSection

Section		"InputDevice"
	Identifier	"AcerMediaKeys"
	Driver		"evdev
	Option        "Device" "/dev/input/eventkeys"
        Option        "SendCoreEvents" "true"
EndSection
```

You **MUST** insert the above text BEFORE any section referring to another mouse.

In the "ServerLayout" section, insert the following lines:
```

	InputDevice	"TouchPad"
	InputDevice	"AcerMediaKeys"
```

Now, save and close your xorg.conf

Now ensure that there are no other scripts in the startup folder that may be dealing with the touchpad (for instance one that you may have created during a previous attempt!).  You can do this with nautilus (type sudo nautilus from the terminal to get root privileges first), or use this code:

```
sudo rm /etc/udev/*acer*
```

We now need to create a small script to run on startup, so back in the terminal:

```
sudo gedit /etc/udev/rules.d/10-acersynaptics.rules
```

Insert this text:

```
SUBSYSTEMS=="input", ATTRS{phys}=="isa0060/serio4/input0", KERNEL=="event*", NAME="input/eventtouchpad"
SUBSYSTEMS=="input", ATTRS{phys}=="isa0060/serio3/input0", KERNEL=="event*", NAME="input/eventkeys"
```

Save and close the file.

OK, we're nearly there.  Now it's time to install the Touchpad settings manager, so go into synaptic package manager, and search for 'gsynaptics'.  Install this program.

Reboot.  Your touchpad should now be working.  You may need to configure it to work as you wish by going to System > Preferences > Touchpad.  You should find that your media keys work, and are mapped as mouse keys 17-20.

Enjoy your fully-functional touchpad and media keys!

Once again, please post feedback to this thread, I will update this post accordingly.

### Post by errenay on 2008-06-27
Small addition to FokkerCharlie's how-to, based on Deinstein's solution.

**Media keys for Amarok**

1. Install xbindkeys:
```
sudo apt-get install xbindkeys
```

2. Set xbindkeys to start automatically:
Because I'm using Kubuntu, I have placed link to the xbindkeys in Autostart
```
ln -s /usr/bin/xbindkeys ~/.kde/Autostart/xbindkeys
```

3. Create .xbindkeysrc file (in your home directory) with your favourite editor. Place the text below in this file
```
"amarok -t"
   b:17

"amarok -s"
   b:18

"amarok -r"
   b:19

"amarok -f"
   b:20

```

4. Restart your session. Now Amarok should work!
