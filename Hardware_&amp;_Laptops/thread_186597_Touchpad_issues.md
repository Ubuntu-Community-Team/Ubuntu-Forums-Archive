### Post by joske on 2006-06-02
For some reason dapper installs inputdevices for wacom tablets, but not for the synaptics touchpad. Strange since this touchpad is in almost all laptops. Fix it by adding a section like this:

```
Section "InputDevice"
        Identifier      "touchpad"
        Driver          "synaptics"
EndSection 

```

and then add the touchpad line to the serverlayout:
```
Section "ServerLayout"
        Identifier      "Default Layout"
        Screen          "Default Screen"
        InputDevice     "Generic Keyboard"
        InputDevice     "Configured Mouse"
        InputDevice     "touchpad" "AlwaysCore"
EndSection

```

hope this helps
jos

```

       Driver "synaptics"

       Option "HorizScrollDelta" "0"
EndSection
```

and in the server layout section:

```
InputDevice	"Synaptics Touchpad"
```

### Post by kurbacik on 2006-06-02
I had the same problem yesterday, and it took me not a little amount of time to solve. First of all, check the log file /var/log/Xorg.0.log. In my case I noticed that the driver synaptics_drv.o just simply wasn't being loaded, even though it had been installed and the xorg.conf file had been modified with the appropriate lines. I've installed the driver through compilation of the source code, which it an easy process to go through (the source code is at [http://web.telia.com/~u89404340/touchpad/)](http://web.telia.com/~u89404340/touchpad/)). My simple workaround was to create a symbolic link in /usr/lib/xorg/modules/input to synaptics_drv.o which was in /usr/X11R6/lib64/modules/input/synaptics_drv.o. This is my solution, and it works for me. I didn't use the ubuntu synaptics driver because it just gave the cursor a crazy behavior even though vertical scrolling was working (everything else was just plain awful). Hope this helps.

### Post by HankB on 2006-06-03
I covered installation of the custom built driver here: [http://ubuntuforums.org/showthread.php?t=184167&page=2](http://ubuntuforums.org/showthread.php?t=184167&page=2) (last message on that page.)

Basically you need to rename the existing driver (synaptics_drv.so) so it doesn't load and put the new driver (synaptics_drv.o) in that directory. The command 

   locate synaptics_drv.so

is the best way to find the directory. (Of course you need to restart your server before the new driver loads. You can kill the existing server with <ctrl><alt><backspace> and a new one should start automatically.

If you wish to use helper programs like qsynaptics, you will also need to likewise install the syndaemon and synclient programs that get built with the new driver.
