### Post by oldsoundguy on 2008-03-07
if the printer is in cups:
System> administration> Printing> configure printer> new printer and follow the yellow brick road.

Now that is IF it is in cups (the usual printer library)

AND here is a really handy page to keep bookmarked:

[http://ubuntuguide.org/wiki/Ubuntu:Gutsy](http://ubuntuguide.org/wiki/Ubuntu:Gutsy)

shows you how to install all kinds of programs that are already available in the system!
You will have to use terminal and root (sudo most of the time) and learn to cut and paste with the mouse as control c and control v do not work.

### Post by oldsoundguy on 2008-03-08
and that row of items in the upper toolbar is how you get around and FIND things in Ubuntu.  When I mentioned SYSTEM .. that means click on SYSTEM and so on.  IF the driver is already in the library, following that pathway will let you install it.

That link I posted is general help and a read of it will show you the basics of how things are done in Linux.  Unlike WinDOZE, Linux assumes (maybe wrongly in some cases) that the user knows how to read and follow directions and that the user wants to control their computer vs the MS way of controlling the user.

In some cases, that link is very specific as it will give you each step for installing things that are not in the basic repositories (ON LINE LIBRARIES of programs and updates) or that came on the disk. items such as Adobe programs (the free ones).

The first thing that has to be done is to install the repositories so you can install programs and keep updated.

In most cases you don't have to go searching all over the web for a specific program. IF it is available in Linux, it will be in a repository somewhere.  If it is NOT, then to install it becomes problematic.  ESPECIALLY when it comes to hardware as a lot of hardware makers don't even acknowledge the existence of Linux in any way, shape or form.  And without their help, creating drivers for their hardware is a project of reverse engineering.  MS has a strangle hold in the way of threats to remove certification  for MS if the maker co-operates with Linux. Only the super large makers of hardware that are already cross platform with Mac can tell MS to take a hike.

You should also learn to search the Ubuntu site for specific wiki's (how to pages in a way) of specific problems .. the site is loaded with them as Ubuntu seems to be the most user conscious of them all.
