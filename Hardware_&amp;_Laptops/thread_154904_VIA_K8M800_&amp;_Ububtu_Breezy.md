### Post by philipgm on 2006-04-08
Hi Prism,

1. You should follow the instructions that are given on the page.

2. I never did get dri to work in Breezy, but with this I was able to get my TV card working.

3. Yes you can go back to vesa. The simple way to do this is to create a copy of the xorg.conf file in /etc/X11, then edit xorg.conf (not the copy) to change vesa to via. If it doesn't work you would be able to login and copy the backup of the xorg.conf file over your modified version.

In detail:

create your backup
1. sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.bak

2. sudo gedit /etc/X11/xorg.conf

change vesa to via and exit.

3. Restart X - I usually just reboot the machine and have a cup of coffee

4. If it works... leave it alone

5. If it doesn't: login, you'll probably only have the command prompt, then

sudo cp /etc/X11/xorg.conf.bak /etc/X11/xorg.conf

will put back your old vesa thing without having to learn to do stuff in vi.

Regards

Phil

### Post by petehall on 2006-04-22
I was having difficulty getting these drivers to work on my setup (Ubuntu Breezy on a K8MM-V motherboard). Everytime I changed the driver from vesa to via, Xorg wouldn't start. I discovered that this was because of the presence of the line:

```
Option     "UseFBDev"     "true"
```

I have no idea what UseFBDev means, but commenting it out meant that via worked.

I can now watch movies full screen without unbearable jerkiness. I'm very happy. Thanks to all at openchrome and these forums.
