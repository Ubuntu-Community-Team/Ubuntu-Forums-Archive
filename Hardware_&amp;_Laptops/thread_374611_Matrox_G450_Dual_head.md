### Post by deathshadow on 2007-03-02
Really you shouldn't need to mess around with any of that - as everything to set up dual display SHOULD be done from your xorg.conf assuming a baseline x.org ubuntu install. The default drivers on a raw ubuntu install should work dual display with xinerama.

The big file you want is at 
/etc/X11/xorg.conf

and you might need info from doing a:
sudo lspci | grep VGA

The process is generally as follows:

Make a second 'device' section in your x.org similar to your existing one (assuming you have single head working) with a different indentifier line (I like to just add the word 'secondary' to it). Some cards (ATI) have their second 'head' at a different PCI bus location, so you get that from the lspci output. You then need to make a 'display' section for the second monitor, then a 'screen' section tying the second device to the second display... then in the 'ServerFlags' section turn Xinerama on so it's treated as one wide-screen instead of two separate desktops.

This page (written for ATI, but all much the same) should cover the basics.
[http://www.ubuntuforums.org/showthread.php?p=1773624](http://www.ubuntuforums.org/showthread.php?p=1773624)

If you post up your existing xorg.conf and a copy of the LSPCI output, I or someone else here could probably toss together a xorg.conf that will work for you. Getting the monitor timings right on both displays CAN be a little tricky - I've found the best route is to connect only one display, boot from the live CD and pull the monitor info from the xorg.conf created by the auto-config, then shutdown, disconnect the first display, connect the second one, boot from the live CD and pull it's info too. 

It's a LOT more involved than RoW (Rest of World) on this... no magic bullet 'extend my desktop to this display' here.

### Post by deathshadow on 2007-03-03
Well, the problem with the second one you 'got from someone else' is apparant - laundry list of issues, I'm not surprised it threw errors. It's pointing at the wrong PCI bus ID, pointing both at 1:0:0 when both should be 2:0:0 - that's gonna bomb right out of the gate. Second the "monitor" line for screen2 points at a monitor that's never declared, the screen declarations in the serverlayout section try to position display 1 relational to display 2, before display 2 is even DECLARED (?!?)... and while TECHNICALLY declaring an option should default it to 'true', it's just sloppy to do just a Option "Xinerama"

That should be all it takes. You want to reverse which monitor is on which side, just change 'RightOf' to 'LeftOf' - you can also use pixel offsets there (x,y) which if they are different size displays running at different resolutions lets you line them up properly.

For example:

Screen 0 "First Screen" 0 0
Screen 1 "Second Screen" 1024,200

puts second screen 1024 pixels to the right and 200 pixels down from the first display. (usually it's best to make the x distance equal to your display resolution)

You might have to tinker with the 'Modes' lines for your second screen, and/or possibly tinker with actual modelines in the monitors section (depends on what your second display is)
