### Post by linuxNewb on 2006-06-30
Just my 2 cents...

I used to run Windows (like everyone) on my Presario 900 laptop, which is now over 2 years old. A little over a year ago on a whim I installed Ubuntu (warty) over my Windows install -- with no Windows CD to recover it. By the way, I had no linux experience or knowledge. Ubuntu worked right out of the box. Even my laptop's touchpad worked. Ubuntu's documentation is so thorough along with this forum, that even a total newb like me can adjust with about a day of learning curve. I later installed a Windows partition so that I could run Macromedia Flash, but I only used it the one time. When I upgraded to Dapper (with a fresh install) I removed the Windows partition because I never used it.

I'm not trying to convince you to switch, because only you know what's best for you. I just want to convey that I was scared to switch to linux because of the stories and my lack of knowledge, but Ubuntu really was very easy. It really does change the entire computer experience in too many ways to mention here (unless you want me to).

The installer is so easy that even a Windows user will breeze through it. Hope this helps.

### Post by sanchrts on 2006-06-30
[QUOTE=saxsux]Excuse me if this is a bit n00by, but are there any specific chipsets I should look out for?[/QUOTE]

saxsux - I have a fairly old Dell Latitude X300 and as with some of the other posters in this thread nearly everything worked 'out of the box' except the wireless card.  

The reason why discovering the chipsets is of some relevance is that some chipset manufacturers do not release linux versions of their drivers and you therefore have to do some workarounds to get the Windows drivers working in linux. 

As an example, I have the Broadcom 4309 chipset in my wireless card and I had to use the "ndiswrapper" method to get them to work (do a search on these forums for "broadcom" and "ndiswrapper").

Nevertheless, after a couple of hours of tinkering I got my wireless to work (with WPA) without too many hassles and I am NOT experienced at command line input!  The great contributors on these forums will be all you need!

### Post by zneastman on 2006-07-05
That Chipset information is useful, but the other thing I can really recommend is going with  mojor brand laptop, even if it ends up being a touch more expensive.   The first reason for this is that brand laptops generally have better support -- there's always someome trying to get every last little thing to work on the three-million sold Toshiba Satellite, but not so much on the one three-thousand sold LanoBook 300.   I've got a Dell B130, and everything works out of the box, including the funky keyboard layout and custom keys.  I would be surprised to see that kind of support on a less common machine.

The second reason is that there's all kinds of information available on brand laptops that just isn't out there for other kinds.  If you need to use ndiswrapper for an Acer aspire, you can probably google to find the right parameters and the right drivers and find ten other people whove solved whatever problems you're likely to face.  With a less popular machine you're on your own.

2. Adding the following lines to my /etc/network/interfaces file (apologies for similarities to other posts):
> auto ra0
iface ra0 inet dhcp
        pre-up ifconfig ra0 down
        pre-up iwconfig ra0 essid "<my network SSID>"
        pre-up iwconfig ra0 mode Managed
        pre-up iwpriv ra0 set AuthMode=WPAPSK
        pre-up iwpriv ra0 set EncrypType=TKIP
        pre-up iwpriv ra0 set WPAPSK="<my 63-char WPA pre-shared key>"
        pre-up ifconfig ra0 up

### Post by jml on 2006-07-11
I've used Linux on several laptops successfully. I've had good resulte with IBM ThinkPad "R series" laptops.  If you plan to buy a new laptop from Lenovo be sure that it is a ThinkPad and not one of their less expensive lines. They are more Linux friendly.  Refurbished IBM ThinkPads are also available and work quite well.  Another laptop that works reasonably well with Linux is the HP Compaq Presario NX 6110 (0r any of the 6000 series) laptop.  It can be purchased for  a reasonable price and works very well.  The only problem is that some models come with a wireless card that uses a Broadcom Chipset that is not very Linux Friendly.  What I did was to purchase one without a built in wireless cars and add a PCMCIA wireless card that was more Linux friendly.  Hope this helps.
