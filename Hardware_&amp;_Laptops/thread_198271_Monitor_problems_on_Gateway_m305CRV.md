### Post by lsyn on 2006-07-19
I have exact same problem!  Ubuntu refuses to work in anything higher than 800X600 on my GTW-M305CRV, but I can get Windows to display 1024X768.  I have tried a few things, as suggested by the website [http://www.geocities.com/stomljen/](http://www.geocities.com/stomljen/), but I still can't get it working.  I'll tell you what I've done, so maybe we'll have some ideas; 

1.  First, I reconfigured my /etc/X11/xorg.conf file (yours looks very familiar for the most part.)  Here are the modified sections,

#/etc/X11/xorg.conf
Section "Monitor"
        Identifier      "Generic Monitor"
        Option          "DPMS"
        HorizSync       28-51
        VertRefresh     43-60
        ModelName       "1024X768@60HZ"
        UseModes        "Modes[0]"
EndSection

Section "Modes"
        Identifier      "Modes[0]"

        # 1024x768 @ 60.00 Hz (GTF) hsync: 47.70 kHz; pclk: 64.11 MHz
        Modeline "1024x768_60.00"  64.11  1024 1080 1184 1344  768 769 772 795  -HSync +Vsync

EndSection
#END

The rest is the same as yours, I believe.  The modeline was generated with gtf.  Next,

2.  I used the **915resolution** tool from [http://www.geocities.com/stomljen/](http://www.geocities.com/stomljen/) to change the VBIOS resolution.  I followed the readme file coupled with instructions from the links at the bottom of the page (which are a bunch of 'success stories' about using this command.)

3.  In /etc/init.d/bootmisc.sh, I added the line "/usr/sbin/915resolution 52 1024 768" right before the ": exit 0", so the 915resolution tool would be run at boot time, but this didn't work either - the resolution stayed at 800x600.

4.  After all of this frustration, I thought, "maybe the driver is the problem."  So, I went to the Intel website and tried to download the so-called 'final driver' for this chipset, but when I opened the package it was an entirely different version of X11, which bombed on install.  Ugh....

So, I'm very frustrated and running out of ideas.  Have you had any luck with this issue?  ](*,) 

Best,
DJ
