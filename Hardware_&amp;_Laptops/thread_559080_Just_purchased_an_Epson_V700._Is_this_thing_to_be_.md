### Post by ddrichardson on 2007-09-26
I'll be honest with you because I'm not an evangelist - if Linux is more hassle time wise and you have access to the licensed resources under Windows then that's probably the way to go.

I view it as tools for a particular job, my servers all run Linux and so do a number of my desktops, but there a a couple of machines where its not worth the effort - such as my tablet PC, where configuring the touch screen, fingerprint reader and so on is simply easier under the installed Windows.

Linux does thing better in many cases, in particular security - but if manufacturers choose to make support obtuse to obtain then I don't buy thier products, but that's true of products that don't work well under Windows or tie me into ridiculous solutions (TV cards that don't use BDA drivers for example).

I'm not sure in any case what geek talk is, but then again I have only ever experienced a reluctance to use the correct engineering technology in computing - in aviation, where I work, everyone from pilot to groundcrew uses the same terminology because it's the standard and aids communication.

I hope you give Ubuntu a chance - because it is very good at what it does, provides a relatively user friendly access to one of the securest and most robust systems out there, but if the hardware works out of the box under Windows and not in Linux and there is a time investment to be made to get it to work that you don't have then the solution to me would seem to be to go with Windows,

Best regards
