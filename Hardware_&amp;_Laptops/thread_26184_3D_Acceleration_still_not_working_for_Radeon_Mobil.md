### Post by zoso on 2005-04-12
I guess I might as well jump right into things. First let me say that I have a Toshiba A75-226 with a Radeon Mobility 9000, running Ubuntu Hoary. Everything is great except I have been unable to get 3D acceleration to work. This card is IGP, so the drivers provided by ATI will not work since they don't support IGP yet. However, the driver that is distributed with Xorg should work. On my system it does not. I've neen troubleshooting this for about three days with no real success, but I do have some information to share. Hopefully this will enable some of the willing experts around here to give me some advice. Huge thanks to Albatorsk in #ati at irc.freenode.net - yesterday he spent about 2 hours going over this with me. The information I'm going to provide is available entirely due to his efforts.

So, on to the problem. I'll largely be recalling the conversation I had in IRC yesterday to walk everyone through the problem.

After initially attempting to install the fglrx driver package provided by the Ubuntu repository (following the guide found at the [Ubuntu Wiki](http://ubuntulinux.org/wiki/BinaryDriverHowto). This seemed to work as fglrxinfo gave the expected return, but running fgl_glxgears caused the system to lock up. After asking about it in #ati I found out that this is an IGP card and the ATI drivers don't support it. So I went back to the ATI driver included with Xorg, which should still support hardware 3D acceleration.

After doing so everything appeared to be ok at first, but 'glxinfo | grep 'Direct Rendering' still returned 'no'. Next thing we tried to load the 'radeon' module (after killing the X server) since it did not appear to be loaded according to the output of lsmod and dmesg. Doing so returned an error, and we then noted that it appeared that the vesafb module was loaded, which could be preventing the radeon module from loading. I attempted to 'modprobe radeon' and still got an error. Unfortunately I don't have the exact one any longer.

At this point we decided to try a different kernel, so I installed the 686 version available on the Ubuntu repository. Doing so also breaks sound by the way, but that's a topic for another post. In any case we killed the X server again and attempted to load the radeon module, and were met with success this time. At least it didn't return any errors.

Direct rendering was still not enabled according to the xorg.log, so we poked around with the debug output of libgl and some other things. The final conclusion was that there must be some sort of bug with my distribution.

I'm at my wits end with getting 3D acceleration. Any insight that anyone might be able to provide would be greatly appreciated.
