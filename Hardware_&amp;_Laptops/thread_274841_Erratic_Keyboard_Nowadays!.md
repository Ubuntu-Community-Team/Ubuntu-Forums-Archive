### Post by rykel on 2006-10-10
Hi,

These days, using Ubuntu is a pain because my keyboard seems to have become randomly unresponsive.

This happens especially when I create a new folder in GNOME, let's say, the Desktop, and then when I try to name/rename the folder, the keyboard just won't work.

Additionally, sometimes, after a while of using the computer, I would NOT be able to input anything on the net, such as this very UbuntuForums. The keyboard looks like it is "dead", but I know that is not the case, since I can still Alt-Tab, Alt-Control-Backspace and so on. Just the alphabets cannot be typed anymore.

The way to solve this problem is to log out and log in again, but this problem is recurring.


The last program I installed was SCIM (through the Ubuntu System/Administration/Language Support), and the Firefox I am using is the original version from Mozilla, and I suspect that it could be a GCC3 vs GCC4 problem.

Will be uninstalling SCIM soon, to see if my system can be regained.

Hope you can help!

### Post by hrp2171 on 2006-10-23
Bad news and good news!

The bad news: I could not figure out why the keyboard would not work with the GX280.

The good news: I replaced the GX280 with another computer I had laying around: a GX270.  It's a bit older than the 280 but Kubuntu works excellent

rykel, I found out from linuxquestions-ubuntu forum that someone went into their BIOS and disabled Legacy USB support, rebooted and the keyboard worked fine after that.  You could try that and see if it works out for you.  It didn't work for me because the GX280 didn't have that option in the BIOS.

Best of luck!
