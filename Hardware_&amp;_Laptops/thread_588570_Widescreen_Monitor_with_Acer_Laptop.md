### Post by MaximusTG on 2007-10-25
I have a Acer 3680, and I use it in combination with an external 19" widescreen monitor. Since Gutsy, it is possible to use the latest Randr.

With this it is possible to use the xrandr program to dynamically switch between monitors. I have it setupped so that when booted with the external attached, xorg puts both monitors in the correct resolution, and then I call my switch script after login to put it exactly right (switch of internal panel when external is connected). The switch script is also bound to a function key, so when I want to take my laptop and unplug the monitor, I press that key, and it detects the monitor is unplugged and re-enables the laptop panel.

However, I'm not entirely sure whether this works the same with an ATI card. I have an Intel integrated card. 
I would advise you to borrow an old CRT monitor and test it out with Gutsy first. If you want, I can post my xorg.conf and the switchscript.

That's my xorg.conf, you can see I have specified 2 monitors, this wasn't really necessary, since I adjust the monitorsettings after boot with the script, but this way the external monitor is already at 1440x900 when logging in, so it looks nicer.

Then the script gets run:

```

#!/bin/bash
sleep 10
if xrandr -q | grep -q  "VGA connected"; then
  xrandr --output LVDS --off
  xrandr --output VGA  --off
  xrandr --output VGA  --mode 1440x900
else
  xrandr --output VGA --off
  xrandr --output LVDS --off
  xrandr --output LVDS --mode 1280x800
fi

```

The sleep 10 is because I also use avant-window-navigator and compiz-fusion, and it seems to work better when I do the final resolution/monitor switch when everything has 'settled down'. 
The script I have bound to a function key to do a dynamic switch is the same, but has the sleep 10 removed.

I believe for ATI cards you have to change --output VGA to --output VGA-0 

Good luck!
