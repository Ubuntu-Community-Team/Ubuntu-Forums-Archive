### Post by az on 2005-11-02
[http://dri.freedesktop.org/wiki/Trident?action=highlight&value=CategoryHardware](http://dri.freedesktop.org/wiki/Trident?action=highlight&value=CategoryHardware)

dri in trident chipsets is not far along.  Perhaps disabling dri in your configuration would make it work?

You can do
sudo dpkg-reconfigure xserver-xorg 

from a terminal   (you may have to do this from recovery mode or kill x first and do it from a virtual console since hardware autodetection is not straightforward when X is running).  It asks you about mid-way through if you want to enable dri and defaults to "yes".  Make it say no.

If you do it from recovery mode run
init 2
to easily boot into your desktop runlevel.
