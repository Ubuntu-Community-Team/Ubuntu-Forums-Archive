### Post by locutus42 on 2005-07-09
Just wanted to give a heads-up that Ubuntu will work on the Compaq R4000 laptop.

 I have a Compaq R4000 also and have figured out how to fix the 2x clock problem and the IDE DMA problem( disabled ). It requires patching 3 kernel source files and rebuilding the kernel so unfortunately, it's not something for a newbie. The installation locks up with the default ati driver so I had to add [ Option "noaccel" ] to /etc/xorg.conf file right after the [ Driver "ati" ] line. Use ctl-alt-f1 console as soon as you see the round cursor when the Xserver display starts. After adding that line, run "sudo /etc/init.d/gdm restart" and you'll have a working installation. I can post my xorg.conf file if anybody wants it( otherwise, use fglrxconfig to generate your own).

Here's a listing of the diffs for the stock 2.6.10-5 Ubuntu 5.04 kernel. Note, my filenames indicate path AND filename.

Here's a diff of the io_apic.c file:
/R4000/How-To/Patches/Kernel-timerhack$ diff arch_i386_kernel-io_apic.c arch_i386_kernel-io_apic.cORG

< static int timer_hack = 0;
2169,2171c2168
< /*if (timer_irq_works()) {
< */
< if ((!timer_hack) && timer_irq_works()) {
---
> if (timer_irq_works()) {
2246,2251d2242
< static int __init timerhack(char *str)
< {
< timer_hack = 1;
< return 1;
< }
< __setup("timerhack",timerhack);

you'll then want to add "timerhack" to the #kopt line in /boot/grub/menu.lst and run:
sudo update-grub

The IDE DMA problem requires changes to 2 files: atiixp.c and pci_ids.h. Here are the diffs:

or I could just email you the kernel.deb and headers.deb files generated when I rebuilt the kernel(stock 2.6.10-5 except for above patches and CPU=K7 ).

As far as getting ATI's proprietary driver working goes, I just had to change pci_find_class to pci_get_class in the /lib/modules/fglrx/build_mod/agpgart_be.c file and update the GCC line in the build_mod/make.sh script. Oh, after using "alien" to gen a .deb from the .rpm and using dpkg --force-overwrite -i fglrx-xxxxxxx.deb to install it.

Cheers.

PS regarding the AMD64 kernel:
Getting Ubunutu-AMD64 running was pretty much the same except I had to apply 2 patches to the ATI driver and instead of hand patching the timer_hack, I used someone elses patch( for 2.6.11 kernel ) on the stock 2.6.10 K8 kernel source.
