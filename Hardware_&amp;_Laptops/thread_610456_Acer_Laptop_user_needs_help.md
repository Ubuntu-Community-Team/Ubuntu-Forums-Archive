I have the same notebook. I don't use any bluetooth devices and don't know much about it. I did find this HowTo which may be a little outdated but seems informative: [http://ubuntuforums.org/showthread.php?t=371009](http://ubuntuforums.org/showthread.php?t=371009)

As for Compiz-Fusion: you didn't mention the Ubuntu version your using.  With the ATI proprietary driver (fglrx) Compiz-Fusion needs to run under XGL, the driver doesn't support running desktop effects on its own. 

I don't know if there's a Compiz-Fusion package available for Dapper. There may not be.


[COLOR="DarkRed"]**Using Feisty (7.04)**[/COLOR] you need to install XGL and create a session to run it, I used a combination of these HowTos:
[http://ubuntuforums.org/showthread.php?t=488385](http://ubuntuforums.org/showthread.php?t=488385)
[http://ubuntuforums.org/showthread.php?t=482773](http://ubuntuforums.org/showthread.php?t=482773)
... and created a script to delay Compiz-Fusion's startup by a few seconds as given here: [http://ubuntuforums.org/showthread.php?t=501925](http://ubuntuforums.org/showthread.php?t=501925) 

Note: For Feisty - the current(?) repostitories that need to be added to /etc/apt/sources.list are 
deb [http://ppa.launchpad.net/amaranth/ubuntu](http://ppa.launchpad.net/amaranth/ubuntu) feisty main
deb-src [http://ppa.launchpad.net/amaranth/ubuntu](http://ppa.launchpad.net/amaranth/ubuntu) feisty main
(once you have Compiz-Fusion installed you should comment the repositories back out by putting a # in front of them. There was a bug with them that would notify that there was an update when there actually wasn't one.)


[COLOR="DarkRed"]**Gutsy (7.10)**[/COLOR] will automatically prompt to install XGL (and enable it at startup) when you enable desktop effects (System > Preferences > Appearance > Visual Effects).
-Note: It isn't always preferable to run XGL (and the desktop effects) since it can be CPU intensive. You can disable it by creating the directory "~/.config/~/.config/xserver-xgl" and do:
```

touch ~/.config/xserver-xgl/disable  

```
and retstart X (Ctrl+Alt+Backspace)
- to re-enable it just go to your /home/<username> directory, hit Ctrl+H to show hidden folders and delete the ~/.config/xserver-xgl/disable file that was created.

I don't recall for sure but I may have had other problems when enabling the Visual Effects in Gutsy and may have edited my /etc/X11/xorg.conf file going by what I learned messing with it using Feisty. (If you edit it back it up first! It helps to know how to use the command prompt if you need to restore it.)
