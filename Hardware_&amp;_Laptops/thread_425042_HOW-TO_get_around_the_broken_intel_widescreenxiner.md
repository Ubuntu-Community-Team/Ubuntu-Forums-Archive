As the topic already describes: The problem I faced is a little bit hard to explain, but reading through the forum I saw that quite a few people suffer from it.

I wanted to have widescreen support & xinerama in feisty with my intel 945 graphic card which was a pain because the old i810 driver didn't support widescreen and the new intel driver does not support xinerama any longer.
I finally got it solved, mainly by reading [http://burtonini.com/blog/computers/randr-2007-02-06-17-50](http://burtonini.com/blog/computers/randr-2007-02-06-17-50)

The solution includes some BETA xserver packages which are not in the ubuntu repositories. I also have no idea if the combination is stable as I run it for 30 min only.

But if you are as desperate as I was this shouldn't block you from experimenting  ;-)

What you basically need to do is update the graphic driver, update the xserver and update xrandr which will be much more powerfull then the current version installed with feisty.

1) Install the xserver-xorg-video-intel driver instead of the xserver-xorg-video-i810 driver as the intel driver is newer and the i810 is discontinued.

2) Create a new xorg.conf without all the references to two displays and devices. You can do this by either running 

```
sudo dpkg-reconfigure xserver-xorg
```

or copying my xorg.conf

3) If you have created the xorg.conf yourself, remove the MODE entries within the DISPLAY section. Instead insert VIRTUAL entries. That's the amount of framebuffer(?) the xserver will allocate during start. It can not be changed dynamically, so choose the combined values of your both displays. As I have the monitors (1920x1200 and 1280x1024) next to each other, I had to add up the width:

```

Virtual 3200 1200

```

4) Read [http://burtonini.com/blog/computers/randr-2007-02-06-17-50http://burtonini.com/blog/computers/randr-2007-02-06-17-50]("http://burtonini.com/blog/computers/randr-2007-02-06-17-50"). I'll explain the most important steps below:

As root add a new line to /etc/apt/sources.list:

```

deb http://burtonini.com/debian/ feisty/

```

and add the corresponding gpg key:

```

sudo apt-key advanced --keyserver subkeys.pgp.net --recv-keys 0x510E0293

```

Now you can install a new version of xrandr, xserver and the intel driver by doing a 

```

sudo apt-get update
sudo apt-get upgrade

```

5) Restart your computer or only the xserver by hitting CTRL+ALT+Back.

6) Hopefully your xserver is back again in a new version together with the 1.2 version of the xrandr utility. You should now be able by switch on/off the monitors via xrandr and you should be able to change resolutions as well as rearranging the monitors. Some sample commands are below. VGA is my external monitor and LVDS is the notebook screen. If you have your display attached via DVI the name could be different. Just call xrandr to see the names of the displays that are attached.

```

 xrandr --output VGA --left-of LVDS  
 xrandr --output LVDS --mode 1920x1200    
 xrandr --output VGA --off       

```


I hope that helps to get you up to speed. If you find any other tricks please post.
