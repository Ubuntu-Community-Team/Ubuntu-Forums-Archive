### Post by exile on 2005-11-08
I think it's always good to try such things "for the hell of it" really. But this assumes that you have the time and desire to give it a go. I have run ubuntu on a Celeron 450 with 256Mb RAM and 10gb disk.

It runs slow as a desktop, it is fine as a firewall/printserver etc. I think in these cases the amount of RAM you have is more important than the processor. If you want to use 64Mb of RAM, I'd go for a leaner window manager - start with XFCE and then step to WM or FVWM (etc) if XFCE is still too heavy.

A laptop without a window manager but with some decent disk could make a useful protable networked fileserver or a LAN day network shaper and so on.

Good luck.

### Post by jml on 2005-11-08
I agree with what has been said.  If you can add more ram, then the laptop should be quite capable of running linux.  You might need to load a window manager that uses less resources than Gnome, (like enlightenment, or flux box.)  If you can't add more ram, one other option to try before turning the laptop into a doorstop would be to download one of the mini distros.  Damn Small Linux takes up about 50mb of disc space.  Its a live CD that can then be installed to HD.  Download and burn a copy to CD and see how it runs on your laptop.  Just remember that the CD-Rom drive is much slower than a hard drive.  DSL will run much faster once its installed to a HD.  Also, keep your expectations realistic.  DSL is no Ubuntu.  Its fully functional but it does not have many bells and whistles.

Joe
