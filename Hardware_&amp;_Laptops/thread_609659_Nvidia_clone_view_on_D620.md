### Post by Chadarius on 2007-12-04
I had some trouble getting dual monitors running as well. I created two different xorg.conf files and just change them back and forth. The following is the relevant parts of my dual monitor xorg.conf file

```
Section "Monitor"
    Identifier     "Generic Monitor"
    HorizSync       28.0 - 72.0
    VertRefresh     43.0 - 60.0
    Option         "DPMS"
EndSection

Section "Device"
    Identifier     "nVidia Corporation G72M [Quadro NVS 110M/GeForce Go 7300]"
    Driver         "nvidia"
    Option	    "TwinView"
    Option	    "RenderAccel"	"true"
    Option	    "TripleBugger"	"true"
    Option	    "CoolBits"		"1"
EndSection

Section "Screen"
    Identifier     "Default Screen"
    Device         "nVidia Corporation G72M [Quadro NVS 110M/GeForce Go 7300]"
    Monitor        "Generic Monitor"
    DefaultDepth    24
    Option 	    "TwinView" "1"
    Option         "metamodes" "CRT: nvidia-auto-select +1440+0, DFP: nvidia-auto-select +0+62"
    Subsection	"Display"
	Depth	24
	Modes	"nvidia-auto-select"
    EndSubSection
    Option         "AddARGBVisuals" "True"
    Option         "NoLogo" "True"
    Option         "AddARGBGLXVisuals" "True"
EndSection

```

This lines up my laptop screen as the left hand screen and my external 1280x1024 LCD monitor on the right with the primary desktop on it. This works for me with KDE. I don't use Gnome so I don't know how Gnome would handle this. 

I think I used the Nvidia X Server Settings utility to configure things that way I wanted it and told it to generate an xorg.conf file, which I hacked up a little to get what I really wanted.
