### Post by DrMega on 2008-02-07
Sorry, I do buy that. MS knew how Vista differs from XP, and could have included a compatibility layer, allowing certain apps to running in a context that mimics XP. Security wouldn't need to be compromised in any way, because you could require that an admin user grants the program the right to run as though it was XP, and the OS could limit visibility of memory and file system elements that the XP prog had direct access to.

MS chose not to do this.

Oddly enough, when they released XP they did use a similar concept to increase compatibilti for old DOS programs.

Before the domination of MS, I remember having a few Amiga's. I remember having to hold down both  mouse buttons when switching on my A1200 to raise a menu that allowed me to temporarily switch off certain CPU and graphics chip features so I could run pre AGA software.

Even earlier than that when the Commodore C128 came out, it had a boot option to make it mimic a C64.

At the end of the day, MS got lazy and greedy when they released Vista without making sure compatibility issues were kept to an absolute minimum. I have no sympathy for them.

As for Linux drivers, we must remember that the hardware manufacturers are out to make a profit. They can't do that if they pay developers lots of money to develop for an OS that their hardware is unlikely to be bought for. As Linux gains ground (as it is doing now in leaps and bounds) more and more manufacturers will recognise an emerging market, and will plough resources into tapping that market. We are already beginning to see that ATI and nVidia making a real effort to support Linux, and a few others supporting other types of hardware.

### Post by aikiwolfie on 2008-02-07
Any OS developer should know that major changes to the OS can break any type of software. Including drivers. No OS is immune from this side effect of making major changes. Not even Linux.

Reading the article posted it would seem Microsoft have at best fumbled the process of working with partners to make sure drivers work. At worst Microsoft simply haven't bothered to aknowledge the problem because it would then have to admit Vista SP1 is effectivley a new OS. Which ironically wouldn't be such a bad thing for Microsoft to do.

Some of the blame will lie squarely with hardware vendors who have either written bad drivers or just haven't bothered to update their drivers at all yet. Of course that would be an argument for open source everything. No need to depend on the hardware vendor.

The fact remains however Microsoft are the OS vendor. It's up to Microsoft to manage this change over. If a kernel update in Linux broke something, most Linux users would be complaining to the kernel developers or the distrobution vendor. Not the hardware vendor.
