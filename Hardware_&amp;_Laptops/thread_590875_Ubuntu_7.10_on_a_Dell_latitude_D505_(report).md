### Post by vdbergh on 2007-10-25
Hi I justed wanted to report for the benefit of others.

I installed Ubuntu 7.10 on a Dell Latitude D505. Everything worked
***except*** that closing the lid gave a hard lockup. 

Changing the intel driver to i810 in /etc/X11/xorg.conf fixed this. 
Now everything works it seems, including suspend/resume.

Oh I forget. I had to install 915resolution to get the the full display
resolution (1400x1050).

### Post by YannBuntu on 2008-01-31
Markyp, the original poster (vdbergh) kindly gave me the following tip:
you need the following section in your xorg.conf. In the standard install the Driver line says "intel". Replace it by "i810" (as shown below).

Section "Device"
Identifier "Intel Corporation 82852/855GM Integrated Graphics Device"
Boardname "i810"
Busid "PCI:0:2:0"
Driver "i810"
Screen 0
EndSection

I'll try this asap and keep you informed. :)

### Post by Bumbler on 2008-03-03
anneb, I'm going to guess you have the latest BIOS update from Dell. A11 and A10 cause problems with the Linux kernel. I had to downgrade to A09 to get any version of Linux to work at all.

To others, please note there are two basic models of the D505. One has the higher resolution 14" screen and better wifi chipset. The other has a smaller 12" screen, and is limited to 1024x768, plus a crappy Broadcom wifi requiring you to use NDISwrapper. Last time I checked, Ubuntu buglist shows no fix for bfwcutter, because the developers seem to have dropped support for the particular version of Broadcom chips (BCM4306 rev. 03) in this slightly cheaper version of the D505.
