My main problem now is that my system won't boot. I've rigged together a GRUB boot floppy that lets me get access to Linux, but it absolutely refuses to boot from either hard drive. If I try booting from the internal drive with Linux on, it fails to boot with the message "No Operating System Found". If I try booting from the external drive with Windows on, it just jams on a black screen - I've let it sit for five or ten minutes with no result whatsoever. Both partitions are correctly flagged as /boot by GPartEd.

The first thing I tried was tinkering with the BIOS settings, especially the boot order - I couldn't find anything that would allow successful booting into Linux. Then I tried reinstalling GRUB on the internal hard drive, using the following sequences of commands alternately:

```
sudo grub
find /boot/grup/stage1
[That command returns the output (hd0,2)]
root (hd0,2)
setup (hd0,2)
quit
```

And:

```
sudo grub
find /boot/grup/stage1
[That command returns the output (hd0,2)]
root (hd0,2)
setup (hd0)
quit
```

Both failed repeatedly - whether I was trying it from a LiveCD or from my actual installation booted from the floppy. (There were no error messages - it just didn't make the system bootable.) I've also tried using "sudo grub-install /dev/sda" and "sudo grub-install /dev/sda3" from my actual installation, both of which resulted in the error message "/dev/sda does not have any corresponding BIOS drive." (Replace /dev/sda for /dev/sda3 for the latter command.) That sounds like it might be a key clue to the real problem, but unfortunately I have no clue what it means. Now I'm out of ideas. I suppose I could try installing LILO, but it looks like quite a complicated process, I have a horrible feeling that it might end up rendering my boot floppy useless, and I just really don't want to do it unless I'm sure I have no choice. Any ideas?
