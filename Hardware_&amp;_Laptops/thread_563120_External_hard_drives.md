### Post by Xenaco on 2007-09-29
I recently had occasion to mount an external hard drive to make a backup copy of a server running Gentoo Linux.  I decided that the safest way to backup the system would be to use dd command to make a copy of the system.  I booted Xubuntu Live CD and copied the Gentoo system without any problems.

I used this setup:

[Rosewill RX-358-S BLK (Black) 3.5" USB & eSATA Ext. Enclosure w/Int.80mm fan]("http://www.newegg.com/Product/Product.aspx?Item=N82E16817173042")

with this hard drive:

[Seagate Barracuda 7200.10 ST3320620AS (Perpendicular Recording Technology) 320GB 7200 RPM 16MB Cache SATA 3.0Gb/s Hard Drive ]("http://www.newegg.com/Product/Product.aspx?Item=N82E16822148140")

The installation of the disk was very easy and I like the built-in ultra quiet 80mm fan which has a separate on/off switch.

You can use a larger or smaller SATA drive depending upon your needs, preferences or wallet.

The external case will use USB 2.0 or eSATA (external SATA) if you have it available.  You can use gParted to partition and format the external drive which is recognized automatically by Ubuntu as a new /dev/sdx device (using USB or eSATA).
