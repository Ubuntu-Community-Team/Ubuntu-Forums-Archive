### Post by besada on 2007-11-04
Next I am describing the solution to the suspend/resume (and also hibernate/resume) problem for an Nvidia card, in a 8510w laptop, with all those nice features we like. It is a Quadro FX 570M, but I hope it will work for other cards, as there are may questions in the forums. Please let me know it the solution solves your problem.

First of all you must enable the restricted driver (using the restricted drivers manager in System -> Administration menu), and reboot.

Then I enabled compiz and played with the Advanced Desktop Setings Configuration program. Everything worked very well. To learn how to install/enable it, please do a forum search.

After these, there were several problems in the support of this card. I was not able to suspend every time I wanted, but sometimes it suspended. But it was not capable of resuming, as when it resumed I had a blank screen and no keyboard support.

So, the solution to those problems have several aspects, and I have to thank the people in this forum as they gave some of the solutions, and additionally gave me the ideas for what I think is the definitive solution (in my case).

I will provide you with the following:

- My /etc/default/acpi-support, and /etc/X11/xorg.conf files.
- A pair of modules to be blacklisted
- Some instructions on disabling some problematic features of Xv and of compiz.

So, let us start:

My /ect/defaults/acpi-support
```


# Comment the next line to disable ACPI suspend to RAM
ACPI_SLEEP=true

# Comment the next line to disable suspend to disk
ACPI_HIBERNATE=true

# Change the following to "standby" to use ACPI S1 sleep, rather than S3.
# This will save less power, but may work on more machines
ACPI_SLEEP_MODE=mem

# Add modules to this list to have them removed before suspend and reloaded
# on resume. An example would be MODULES="em8300 yenta_socket"
#
# Note that network cards and USB controllers will automatically be unloaded 
# unless they're listed in MODULES_WHITELIST
MODULES=""

# Add modules to this list to leave them in the kernel over suspend/resume
MODULES_WHITELIST=""

# Should we save and restore state using the VESA BIOS Extensions?
SAVE_VBE_STATE=false

# The file that we use to save the vbestate
VBESTATE=/var/lib/acpi-support/vbestate

# Should we attempt to warm-boot the video hardware on resume?
POST_VIDEO=false

# Save and restore video state?
#SAVE_VIDEO_PCI_STATE=true

# Should we switch the screen off with DPMS on suspend?
USE_DPMS=true

# Use Radeontool to switch the screen off? Seems to be needed on some machines
# RADEON_LIGHT=true

# Uncomment the next line to switch away from X and back again after resume.
# This is needed for some hardware, but should be unnecessary on most.
# DOUBLE_CONSOLE_SWITCH=true

# Set the following to "platform" if you want to use ACPI to shut down
# your machine on hibernation
HIBERNATE_MODE=shutdown

# Comment this out to disable screen locking on resume
LOCK_SCREEN=true

# Uncomment this line to have DMA disabled before suspend and reenabled
# afterwards
# DISABLE_DMA=true

# Uncomment this line to attempt to reset the drive on resume. This seems
# to be needed for some Sonys
# RESET_DRIVE=true

# Add services to this list to stop them before suspend and restart them in 
# the resume process.
STOP_SERVICES=""

# Restart Infra Red services on resume - off by default as it crashes some
# machines
RESTART_IRDA=false

# Switch to laptop-mode on battery power - off by default as it causes odd
# hangs on some machines
ENABLE_LAPTOP_MODE=true

# Spindown time on battery
SPINDOWN_TIME=60

```(note some values have to do with the solution of the disk life problem I found in the forums, with work perfectly, by the way)

```I also blacklisted two modules, by insertind in /etc/modprobe.d/blacklist file, the following:

```

blacklist intel_agp
blacklist agpgart

```Do not use the new X applets in ubuntu. use nvidia-setting program instead (I have even changed the links in the menu).

Within this program, disable sync to VBlank both for OpenGL and Xserver Xvideo setings.

Additionally, disable in Advanced Desktop Setings Configuration program the following aspects:
- sync for VBlank. It can be found in General Options -> Display Settings.
- Video Playback plugin (in utility list).

Until I did not disable video playback I was able to resume if I had not seen any video but not if I had seen any .... This is a problem with youtube, zatoo, sopcast, ... around, isn't it ... ;)

Now it works ...:guitar:

### Post by besada on 2007-12-13
In my case it only worked if I had vsync dissabled in nvidia-settings, when invoked  both as root (using sudo or gksudo), and as the current user.

I think there is some kind of problem with suspend + video playback using xv with vsync. Maybe leaving the posibility of vsync both for opengl and compiz default settings you may have good results for compiz, but in my case I had to do everything to make everything work.


My howto was compiled using information from other posts, and after tweaking with/without opengl, with/without compiz, with/without video playback. Some tweaks solved some problems in the simpler configurations, but in order to eliminate all problems I had to do these all changes.
