### Post by biodiesel-bri on 2008-02-26
I have an HP dv9610us running 7.10 amd64.  I was previously using 7.10 but I reinstalled using the alternate CD to get full disk encryption.

I added both the nvidia graphics and broadcom 43xx restricted drivers.

After the reinstall I found that ksoftirqd was pegging the CPU at 100%.  Investigating further I found the logs cluttered with errors /var/log/messages like this:
kernel: [   61.746166] bcm43xx: Unsupported 80211 core revision 13
kernel: [   61.747967] bcm43xx: Invalid PHY Revision 9

I blacklisted the broadcom and CPU is now normal.  Of course, I don't have wireless.  :(

Either deinstall the Broadcom restricted driver OR add the following:
```
blacklist bcm43xx
```
to /etc/modprobe.d/blacklist

Reboot and you're back in business.  I hope this is fixed in Hardy, it would be nice to use my wireless card.
