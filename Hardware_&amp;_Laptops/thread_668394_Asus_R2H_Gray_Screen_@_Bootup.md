### Post by Matthew Green on 2008-02-04
Hi - just spotted this one when i was browsing for other help with the Asus R2H.

You'll need to do some editing to fix this, so I hope you're OK with vi?

* Press Escape when the "grub loading" message appears soon after boot.
* choose the second boot option with the word "recovery" on the far right
* when presented with the next red and blue menu, choose "root terminal" or similar
* enter the following command

vi /etc/X11/xorg.conf

In order for the internal screen to boot with the correct settings, you need to find the following three sections (Device, Monitor, and Screen) and make sure they look like the following:

Section "Device"
    Identifier    "Intel"
    Driver        "i810"
    BusID        "PCI:0:2:0"
EndSection

Section "Monitor"
    Identifier    "Generic Monitor"
    Option        "DPMS"
EndSection

Section "Screen"
    Identifier    "Default Screen"
    Device        "Intel"
    Monitor        "Generic Monitor"
    DefaultDepth    24
    SubSection "Display"
        Depth        24
        Modes        "800x480"
    EndSubSection
EndSection


After this, save and exit vi, and type "exit" at the command prompt - you should see the three option menu, choose "continue/boot normal/or similar" and you should boot with a working screen.
