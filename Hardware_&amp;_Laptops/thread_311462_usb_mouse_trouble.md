### Post by Frem on 2006-12-03
Yes! Someone else who has the bug!

I posted about this a while back, but didn't get any responses: [http://ubuntuforums.org/showthread.php?t=227412](http://ubuntuforums.org/showthread.php?t=227412)

My laptop is a Toshiba M55-S139.

It doesn't freeze when switching to console for me. All the USB ports stop responding when the mouse dies, though.

From experimentation, I can tell you that isn't just an Ubuntu problem. Pretty much every distro on the planet has it. Farther experimentation shows that it's not just one model or brand of mouse. I've tried two or three and all of them have the problem.

Just today, I found this Debian bugs mailing list thread which seems to indicate that the problem is a bug which has to be corrected in the kernel.
[http://bugs.debian.org/cgi-bin/bugreport.cgi?bug=314954](http://bugs.debian.org/cgi-bin/bugreport.cgi?bug=314954)
They provide a patch, but I've never compiled a kernel before, much less patched source code. It would seriously pwn if this fix was included in Feisty Fawn. (Devs? Please? [-o< )

Finally, here is my list of things I've been trying to get working in Ubuntu. Might help you with configuring some things, might not. [http://frem.pbwiki.com/UbuntuChecklist](http://frem.pbwiki.com/UbuntuChecklist)


Edit: I've searched around and this seems to be related to the 'fglrx' drivers. But you're using the 'ati' ones, aren't you? There goes that theory. There's probably some sort of relation to the card, though.

I've started compiling a wiki page documenting my findings. Of course, I made a new wiki to go with it. ;-)
[http://toshibaubuntu.pbwiki.com/MouseFreezing](http://toshibaubuntu.pbwiki.com/MouseFreezing)

### Post by Frem on 2006-12-03
Hey! I think I *may* have found a solution for you here: [http://ubuntuforums.org/showthread.php?t=212438&page=2](http://ubuntuforums.org/showthread.php?t=212438&page=2)

In grub, hit *e* to edit the boot line. Then move the selection over the kernel line and hit e again. Move the cursor to the end of the line and add the following:
```
noapic irqpoll pci=routeirq
``` Then hit *enter* to get our of line editing mode and *b* to boot the modified kernel line.

This will set you up until you power down or reboot. Edit /boot/grub/menu.list to make it permanent.

I kinda hosed my Ubuntu install (in a completely unrelated incident :p), but this seems to work in my Mandriva setup on a temporary partition. It might mess up hibernation or something, I haven't had a chance to fully test if everything is still working properly. 

Let me know how it goes! :D
