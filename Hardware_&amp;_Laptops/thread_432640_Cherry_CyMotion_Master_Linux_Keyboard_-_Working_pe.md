### Post by darkrose on 2007-05-04
Well the first bug just showed up, keyman wouldn't start due to an error in /etc/init.d/keyman

Now fixed, grab the new package:
[http://www.ltmnet.com/downloads/keyman_1.0.0-0ubuntu2_i386.deb](http://www.ltmnet.com/downloads/keyman_1.0.0-0ubuntu2_i386.deb)

Alternatively, you can fix the old one by editing /etc/init.d/keyman and changing the first line to:
#!/bin/bash
then run:
sudo /etc/init.d/keyan start
sudo nohup keymanserver &

If you get a window popping up to say some keys are in use, just click ok, this won't affect the workings of the keyboard. But do let me know if you do/don't get this, as i can then make changes if everyone gets it.

On another note, if you use kubuntu you will need to install the following packages before installing keyman:
libbonobo2-common
libbonobomm2-common
libcairomm-1.0-1
libglademm-2.4.1c2a
libglibmm-2.4-1c2a
libgnome-keyring0
libgnome2-0
libgnomeui-0
libgnomecanvass2-0
libgnomevfs2-0
libgtkmm-2.4-1c2a
libwnck18
libxerces27

I haven't tried a fresh install on Ubuntu (using Gnome) yet, but would expect a similar list but with KDE libs needed.

### Post by darkrose on 2007-05-10
you should be able to add it by going to:
system --> administration --> services

or by linking /etc/init.d/keyman to /etc/rc2(3, 4, 5).d/S99keyman:
sudo ln -s /etc/init.d/keyman /etc/rc2.d/S99keyman
sudo ln -s /etc/init.d/keyman /etc/rc3.d/S99keyman
sudo ln -s /etc/init.d/keyman /etc/rc4.d/S99keyman
sudo ln -s /etc/init.d/keyman /etc/rc5.d/S99keyman

Incidently it starts on boot for me, so it's either a 64bit problem (unlikely) or it is a problem stemming from the previous one, so uninstalling and reinstalling could solve the problem.
