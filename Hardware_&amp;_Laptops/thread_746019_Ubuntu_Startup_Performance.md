Hi!

I'm new in the forum, this is not a classic support question, but more a discussion about present a future features regarding the performance of this wonderful Linux distribution - congrats to all for the incredible work, BTW. Please feel free to redirect me to the correct forum, if this is not appropriate.

I really think Linux & Ubuntu are great when comes to performance and responsiveness, important qualities that affect user experience and about which I really care. AFAIK, though, it still lacks a general approach to the 'cold cache' performance slowdown - the slowdown you clearly observe when you launch an app for the first time after a boot, for example. 

There already are some solutions to the problem, i.e. readahead and preload, which are used and available in Ubuntu. Even though they are great apps, I think to solve the problem a more general approach is needed: somethink inside the OS should observe IO activity and predict it, in order to silently fill the caches so that they are 'warm' when you need it. Somethink like Vista SuperFetch, for example. 

I've read that the prefetch project aims to solve this issue, but too bad it has been removed from Hardy. 

So the question is: will this problem be addressed in future Ubuntu releases? Will prefetch be used or there are plans for other solutions? What is the Ubuntu's team opinion about that?

Thanks in advance!!! Keep up with the wonderful work.
