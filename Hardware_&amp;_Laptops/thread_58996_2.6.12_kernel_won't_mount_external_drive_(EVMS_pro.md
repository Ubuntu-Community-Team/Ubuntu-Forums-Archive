### Post by 23meg on 2005-08-22
booting with the 2.6.12 kernel i get several instances of the error "device mapper : dm-linear : device lookup failed" while Enterprise Volume Management System is initializing, and my external USB volumes are not mounted.

 i googled and found that this is due to a conflict that occurs between the volume mounting mechanism of some 2.6 kernels and EVMS (see here: [http://evms.sourceforge.net/install/kernel.html#bdclaim](http://evms.sourceforge.net/install/kernel.html#bdclaim) ). i did what was suggested as a solution on some forums: told EVMS to ignore my external volumes. here's the relevant part of my /etc/evms.conf file:

```


# Use this section to tell EVMS where to look for devices and which devices to
# include or exclude on a system with sysfs (i.e., 2.5 and later kernels).  In
# order for this to work, sysfs must be mounted somewhere on your system.

sysfs_devices {

	# "include" are the block devices found in the [sysfs_mount_dir]/block/
	# directory that you want EVMS to use as disks.
	#
	# Block device names can be specified using "*", "?", and "[...]"
	# notations.

	include = [ * ]

	# "exclude" are the block devices found in the [sysfs_mount_dir]/block/
	# directory that you don't want EVMS to use as disks.  Entries here
	# will override any possible matches from the "include" setting.
	#
	# Block device names can be specified using "*", "?", and "[...]"
	# notations.

	exclude = [sda5]


```

anyone else having this problem? i've also tried putting"/dev/sda5" and "sda5 , sda6" (i have two external volumes) inside the bracket..

### Post by Laterix on 2005-09-04
I have a similar kind of problem. Difference is that I can't mount my normal harddrives. I really didn't understand the solution here so if someone could help me I would be a very happy man.

**I got ansewer in other thread**
Solution is simply to update /etc/fstab -file

For example if you have /dev/hdb1 you change it to /dev/evms/hdb1
