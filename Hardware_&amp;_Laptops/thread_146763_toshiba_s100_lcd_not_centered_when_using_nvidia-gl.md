### Post by probablydrew on 2006-03-18
the problem I have is that when I use the standard nv video driver the display works correctly, when I change to the nvidia-glx 8178 closed driver so that I can use 3d acceleration the display gets shifted down and to the right.  Leaving a black bar along the top and left of the display, and the bottom-right corner of the desktop is pushed off the lcd.

one thing of note is that the native resolution of this lcd is 1024x768@60
using this display mode, and nearly every other display mode I run into the same problem of the desktop being shifted down-right, the only exception I have run across is if I run at 1280x1024.  This causes the desktop to begin in the top-left corner as it should, and the black bars are gone.  The obvious problem with this is a fair amount of the desktop is still hanging off the bottom-right corner of the screen and not visible.


things I have tried:
I have played around with the modeline generators listed in the following thread to no avail:
[http://ubuntuforums.org/showthread.php?t=83973](http://ubuntuforums.org/showthread.php?t=83973)

I have also attempted to move the screen around and generate modelines using xvidtune but the screen shift functions, and the modelines it generates, seem to have no effect at all with respect to shifting the image into place.

and the stretch/no-stretch options in the bios, as well as the lcd/crt display switching function keys also have no effect


this is a problem I have seen documented a couple other places with laptops in this family so I don't believe it is specific to my configuration, what I am wondering is:
is this a problem that lies within the nvidia binary and I can do nothing about it?
is this a problem that lies within the nvidia binary but there may be a workaround?
is there something elsewhere that is incorrectly reporting the native resolution of the monitor as 1280x1024 when it should be 1024x768?

other places I have seen the issue mentioned briefly which make me think it is not specific to my config:

### Post by probablydrew on 2006-03-20
my previous question can be disregarded, this is apparently a bug with the nvidia drivers that will be corrected in the next version, and a workaround was found in this thread using the option "ExactModeTimingsDVI"
[http://www.nvnews.net/vbulletin/showthread.php?p=843472](http://www.nvnews.net/vbulletin/showthread.php?p=843472)
