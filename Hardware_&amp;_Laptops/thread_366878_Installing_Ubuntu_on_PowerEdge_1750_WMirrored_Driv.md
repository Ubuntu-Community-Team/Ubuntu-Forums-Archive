### Post by Dean Bradley on 2007-02-28
Before you start reading this I just wanted everybody to know that I love Ubuntu. I ran Suse 10.0 for over a year and just didn’t like the program manager “RPM” and YAST and the Novell/Microsoft deal really was upsetting. I’ve always hated Microsoft, I’ve been developing micro-based software systems for over 20 years and cussed every time Microsoft dropped a language or wanted to charge for something that use to be free. 

I discovered Ubuntu 6.10 last November and immediately feel in love. I liked the fact that it was thin, didn’t pre-install a lot of unwanted software, and seemed to have the all the tools I needed to run a web server. A couple of weeks ago I had a chance to buy a server at a great price and decided to upgrade. I purchased a Poweredge 1750 server /w RAID5 and started to investigate what I needed to do to remove the Windows 2003 and install Ubuntu (After all, Ubuntu is Linux and Linux is used on 70% of the worlds web servers ). 

Well I now have a broken heart. I found a lot of how-to guides (listed below) and after carefully reading found no solution for support for RAID5 / FakeRAID (RAID on the Mother Board). You would think that Debian based Linux would have a solution, but they ready don’t, and I can’t take that chance. I’m not going to “mickey mouse” my server in order to run Ubuntu. Suse 10.2., RedHat and a couple of smaller distros have solutions, but they’re not Debian based. I’m hoping that the new Ubuntu 7.4 code name "Feisty Fawn" will have the support, in the mean time I guess I’m stuck with Windows (which I hate dearly). I hope this helps someone – thanks.
