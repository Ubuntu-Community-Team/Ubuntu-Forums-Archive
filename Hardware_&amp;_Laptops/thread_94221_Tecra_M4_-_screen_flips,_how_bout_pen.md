### Post by 23meg on 2005-11-23
Enter
```
xsetwacom set stylus rotate X
```
```
xsetwacom set cursor rotate X
```
```
xsetwacom set eraser rotate X
```
where X is one of "cw", "ccw" and "none". Ignore any possible errors.

### Post by 23meg on 2005-11-23
Happy to know you got it sorted.

Unfortunately there's no command for inverted landscape at the moment. The latest "vanilla" version of the linuxwacom driver still hasn't integrated the patch (from Bj&#246;rn Bringert) that enables on the fly tablet orientation rotation, but we have the patch applied in the version that Ubuntu ships. However, the patch hasn't implemented inverted landscape for some reason; maybe it was just overlooked. I'm investigating the code to see if this is the case but my C knowledge is nowhere near those of the linuxwacom developers. I'll ask for some help from a coder friend in the weekend and see if we can come up with anything; that's on my todo list. 

The xsetwacom userspace tool was also recompiled to allow on the fly rotation just in time for the Breezy release, thanks to Matthew Garrett's attention to my report on the bugzilla. It would otherwise have been a pain to repatch and recompile (I and a few others have failed gloriously), and would have straight away destroyed all practical end user tablet pc usability with Breezy. I hope I can figure out how to recompile it this time, since functionality in the driver means nothing when you don't have the tool to activate it.
