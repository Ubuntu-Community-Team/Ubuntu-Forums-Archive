### Post by monktbd on 2006-10-18
i have a NEC chipset on mine.
> 
0000:00:0f.0 USB Controller: NEC Corporation USB (rev 43)
0000:00:0f.1 USB Controller: NEC Corporation USB (rev 43)
0000:00:0f.2 USB Controller: NEC Corporation USB 2.0 (rev 04)
not that that should change a lot though.

lsmod showed me the following modules loaded for usb:
> usbhid                 39904  0 
ehci_hcd               34184  0 
ohci_hcd               21892  0 
usbcore               130820  5 
usb_storage

and yes removing the ehci should disable usb2.0, ohci is usb 1.0.

you can try to remove these modules with
> sudo modprobe -r modulname
and loading them again with
> sudo modprobe  modulname
and look what dmesg or /var/log/messages or /var/log/syslog say, although the latter probably wont produce any usb related output.

besides that i dont have many more clues, maybe switching the ide slot of the adaptor, but since it worked with breezy it is only a wild guess into the blue here.
