### Post by zenzero-2001 on 2006-11-30
I also have problems with my touchpad. I am using kubuntu 6.10 with an Acer Travelmate 8000. There is sometimes a problem with web browser pages magically going back, but this can be solved with ksynaptics (see below), which I think is just a different front end for qsynaptics. My other problem is like yours. The pad will work fine for an hour or so but then it will act all screwy and won't go where I want it. For example, the pointer will be over the right hand side of the screen and I want to move it to the left. I pass my finger over the touchpad and the pointer goes to the center but as I stop my finger or raise my finger the pointer shoots back to the right! Aaaarghh! ](*,)  The only way to fix it is to reboot, logging out and back in again does not help.

Ksynaptics can be used to cure problems in a web browser. After installing ksynaptics from the repository, log out and log back in again to activate the icon in the systray. Then try to configure it by right clicking, you will probably receive an error about shared memory and you will have to edit some config file (can't remember the name off the top of my head) and add "SHMConfig" "on" to the touchpad section (else all the options will be greyed out). Next reboot. Then configure the touchpad with ksynaptics by  deselecting all the tapping and scrolling options. Then OK.

I will be giving Debian 4.0 stable a shot when it comes out next month. Hopefully it won't have the same problem as (k)ubuntu. May also give openSUSE 10.2 a go, if it significantly quicker than 10.1 was.

Good luck!

### Post by saintchuck on 2006-12-10
The Synaptics touchpad is mis-identified and usually double identified as a Wacom Touch Tablet.  If you do not have/use a Wacom Touch Tablet then this will fix your problem.

Comment out:

```
Section "InputDevice"
  Driver        "wacom"
  Identifier    "stylus"
  Option        "Device"        "/dev/wacom"          # Change to 
                                                      # /dev/input/event
                                                      # for USB
  Option        "Type"          "stylus"
  Option        "ForceDevice"   "ISDV4"               # Tablet PC ONLY
EndSection

Section "InputDevice"
  Driver        "wacom"
  Identifier    "eraser"
  Option        "Device"        "/dev/wacom"          # Change to 
                                                      # /dev/input/event
                                                      # for USB
  Option        "Type"          "eraser"
  Option        "ForceDevice"   "ISDV4"               # Tablet PC ONLY
EndSection

Section "InputDevice"
  Driver        "wacom"
  Identifier    "cursor"
  Option        "Device"        "/dev/wacom"          # Change to 
                                                      # /dev/input/event
                                                      # for USB
  Option        "Type"          "cursor"
  Option        "ForceDevice"   "ISDV4"               # Tablet PC ONLY
EndSection
```

As well as the 3 highlighted lines here:

```
Section "ServerLayout"
	Identifier	"Default Layout"
	Screen		"Default Screen"
	InputDevice	"Generic Keyboard"
	InputDevice	"Configured Mouse"
	[COLOR="Red"]InputDevice     "stylus" "SendCoreEvents"
	InputDevice     "cursor" "SendCoreEvents"
	InputDevice     "eraser" "SendCoreEvents"[/COLOR]
	InputDevice	"Synaptics Touchpad"
EndSection
```
