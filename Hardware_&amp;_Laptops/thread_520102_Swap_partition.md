> 
   According to this it looks like it has no swap. Any way to make it recognize it easily? 
If you know what your swap partition is you can turn in on with:```
 sudo swapon <partition>
```You can check your swap  partition by issuing: ```
sudo fdisk  -l
```'swapon -a'  will  enable  all  swap  paritions  listed  in  your  /etc/fstab  unless they have  'noauto'  as  an  option and is run at boot. If you do have the swap parition listed in your fstab, I'd double check that the right UUID is being used for it

```
ls -l /dev/disks/by-uuid 
```and edit your fstab if needed.

### Post by 5-HT on 2007-08-07
Looks like either you do not have an entry in your /etc/fstab for your swap partition, or the entry is incorrect. The partition is there, and working with a swapon, but to get it to mount at boot you'll need to edit your /etc/fstab.

First thing is to take a look at the file (gedit /etc/fstab ; if you're using Ubuntu, otherwise just replace 'gedit' with your preferred text editor). Then look to see if if you have a swap parition listed there. It may not show the device as /dev/sda6, but instead use a UUID. You can either replace the UUID with /dev/sda6, or determine the proper UUID (by issuing the command I mentioned earlier) and make sure the UUID used in your fstab reflects your swap partition's UUID (to save the file you will need root privileges, and can get them easily by just using:
sudo gedit /etc/fstab

If you'd like, I can show you what needs to be changed if you post 1) /etc/fstab and 2) the output of 'ls -l /dev/disks/by-uuid'

Should be :) The UUID was incorrect (happened to me a few times on install).

Please change the following from your original fstab:
```
 #/dev/sda6
UUID=153f9b52-3663-4b22-a8a7-a72f7e2e1c75 none            swap    sw              0       0
```To: ```
#/dev/sda6

```You can do so with a 'sudo gedit /etc/fstab'.
