### Post by virgule on 2005-06-20
You mention the 'GFT trick' but I dont see the **Modeline** line in your xorg.conf file :?:
Try to change the Monitor section so it look just like that:
```

Section "Monitor"
	Identifier	"Gateway EV500"
	Option		"DPMS"
	HorizSync	30-69
	VertRefresh	50-160
	# 1152x864 @ 75.00 Hz (GTF) hsync: 67.65 kHz; pclk: 104.99 MHz
	Modeline "1152x864_75.00"  104.99  1152 1224 1352 1552  864 865 868 902  -HSync +Vsync
EndSection

```
Similar to you situation, my Xorg would not let me use anything above 1024x768. So, after several hit and miss experiments, pasting the gtf output in there then manually insert "1152x864" on all Subsection "Display" Modes solved it right away.

What is important is to stay within the monitor's specs. Its paramount to NOT exceed the HorizSync and VertRefresh hardware limits otherwise Xorg will bypass you custom settings for a safe setup (?) Notice hsync: 67.65 kHz is within 30-69.

This is the values I fed gtf with to produce the Modeline above:
```

gtf 1152 864 75

```
 Let me know if this worked for you. Im confident it will :8):
