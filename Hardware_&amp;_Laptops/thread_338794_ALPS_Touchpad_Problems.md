### Post by SugarDaddy on 2007-01-14
I am running Edgy and I am having a strange problem getting my ALPS touchpad settings to load every time following a reboot. I am wondering if this is a bug because I have searched other posts for a solution and many have cited the same problem but the responses never suggest a fix. So here is my attempt to elicit a solution.

1. My ALPS touchpad settings in xorg.conf won't load unless I change the Option "Protocol" to "event" vs "auto-dev". From here I make sure to map the Option "Device" to the right event handler e.g. "/dev/input/event3" as listed under "$cat /proc/bus/input/devices" vs the default Option "Device" setting "/dev/psaux"

2. Now here is the funny part. When I reboot my laptop, Ubuntu, the Linux kernel, not sure which, assigns the touchpad to another event like event2 and my xorg.conf settings for the touchpad don't load. So I have to go find which event it is assigned to and change the event number in my xorg.conf, then restart x. Rather annoying to say the least. It is  like Linux is randomly assigning the event handler for the touchpad.

Is there any way to get Ubuntu to assign the touchpad to the same event handler each time or am I going about this wrong? Below is my xorg.conf file. Thanks for the help!
