This is a bug mentioned in other places on the net. Ubuntu unmounts external drives for some reason after a certain amount of time. The solution I've used is to remount the drive at the command line. ```

```

### Post by moore.bryan on 2007-07-10
[i]okay, humor me.  [LIST=1]
[*]check to make sure you have a folder in /media named sda1.  if you don't, make one. ```
sudo mkdir /media/sda1
```.
[*]then, run this command in the terminal:```
ls -l /dev/disk/by-uuid
``` with the usb drive connected.  it will tell you what the uuid is of the external drive, which is handy when switching-up drives.
[*]find the /dev/sda1 entry and change your fstab for the external drive from ```
/dev/sda1 /media/sda1 vfat iocharset=utf8,umask=000 0 0
``` to ```
UUID=###########    /media/sda1     auto     defaults     0     0
```, where you put the uuid numbers from the ls -l /dev/disk/by-uuid command in place.
[*]save fstab and reboot.[/i][/LIST]

*that is interesting.  try unplugging any/all usb devices and attaching your external drive directly to the computer (not the hub).  reboot.  when you're back in, run the ```
ls -l /dev/disk/by-uuid
``` again and find the uuid for the external drive; that way, it doesn't really matter what /dev the usb is assigned, fstab will always know what it's looking for...*
