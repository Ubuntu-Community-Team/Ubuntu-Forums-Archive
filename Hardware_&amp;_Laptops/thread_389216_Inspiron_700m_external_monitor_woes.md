### Post by moschops on 2007-03-20
I've been struggling to get my Inspiron 700m to display at 1280x1024 on an external LCD monitor.  I'm on a Fiesty 7.04 code base but I don't think that has anything to do with my problems - I never was able to get it working with 6.10 either (but I didn't try quite as hard).  

The Inspiron 700m has an Intel 855m onboard graphics adapater with a 1280x800 local flat panel (LFP).  I have the 915resolution module loaded too which is able to tweak the resolution correctly to 1280x800 for the LFP.  But when I do Fn-F8 to switch to the external monitor plugged into the VGA output my LCD barfs at the wide screen resolution - so I switch to 1024x768 using the screen resolution applet and it scales okay but I'm loosing full resolution. 

I'm happy to live with having to do a manual switch of resolution when I work on the external monitor - I don't really need a dual head system (both monitors displaying something different) - but so far I can't even get Gnome to list 1280x1024 as a valid resolution - whenever I add it to xorg.conf my Xserver log says its ignoring that mode because the scan rate is too high. So all I get is 1280x800, 1024x768, 800x600 and 600x640. 

Some people have suggested in other threads and places that you can switch to the external monitor before Ubuntu boots (at Grub loader time).  When I try this it just means I get 1024x768 on the external monitor.  Thinking I really need to have two Monitor and Screen sections in my xorg.conf I tried some dual-head configurations.  The best I have gotten is a 640x480 spanning display which is kind of worse because now neither display is giving me optimal resolution. 

I'm pretty sure this something to do with the 915resolution module which I think is proably incompatible with running an external monitor.  I think if I removed it from the picture and just configured my machine to only work with an external monitor I could probably get it running at 1280x1024 however I really, really need that builtin flat panel to be at 1280x800 which as far as I know can only be done by running the 915resolution hack at start up.

Is there someone out there that has a working xorg.conf for the 700m or a similar laptop with Intel 915/855/815 or related adapter and wide screen LFP that also works with an external monitor (either one at a time or dual head).

Next time I boot to Ubuntu I'll add my current xorg.conf file and my dual head attempt...
