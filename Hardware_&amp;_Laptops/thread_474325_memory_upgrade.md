### Post by jotagab on 2007-06-15
The answer depends on how much memory you currently have (which you didn't say...) and how memory-hungry your applications are (also you didn't say what applications you normally use...).
In theory more memory will require more energy, so it will reduce your battery autonomy. However if you don't have enough memory for your regular notebook usage you'll need to use disk for swapping, thus also reducing your battery autonomy. The sweet spot will be to buy the exact amount of memory you need, not more...
My advice? Get one stick of 1GB. You probably have 2 slots in your notebook, so leave the other one free for a future upgrade of another GB. By the time you need more than 2GB or memory you'll probably need/want to buy a new notebook!
Regards.

### Post by WiFi Ed on 2007-06-19
Since you have one gigabit now and have a dual channel memory configuration you most likely have two 512Mb modules. If you have two additional unused slots get 2 more 512Mb modules for a total of two Gb, or two additional 1Gb modules (if your wallet allows) for a total of 3 Gb. If you have only two memory slots total I'd spring for two 1Gb modules.

You are most likely running the 32bit version of Ubuntu which will only use about 3 Gb max anyway. One noticeable improvement would be to use the fastest memory your laptop supports. My E520 desktop came with 533MHz memory but supports 667MHz (some have reported success with 800MHz) and became a bit "snappier" with 667MHz memory installed.

I bought a total of 4 Gb before I found out about the 3 Gb limit for 32bit operating systems:(

it will run 4 gigs but you have to disable swap to do so, as the swap is subtracted from the 4 gig limit.

or 2 gigs ram + 2 gigs swap
or 4 gigs ram + no swap
