### Post by super carrot on 2007-07-01
Hi, I have a Toshiba Tecra M7 and I have got it working pretty nice, I have been meaning to share some of the things that I have got right.

Auto rotation
Stylus, with right mouse button working
Bluetooth

I am running Ubuntu Studio ( Feisty Fawn )

This is not going to be exhaustive, but I'm sure that it will help someone. I have not created any of these scripts myself, I have pieced them together from articles and things that I have found in forums and on in other places in the Internet.

First, you will need to install some programs to get bluetooth working, and I don't know what else it might help get working.

sudo apt-get install toshset

Then we need to get it to start every time we log on. so add it to 

/etc/rc.local

you can use

sudo gedit /etc/rc.local

Put this just before the 'exit 0' line

toshset -bluetooth on

save this, every time that you start, the adapter will be picked up, now you need to install 

sudo apt-get install bluetooth-propertiesm gnome-obex-server


Thats the bluetooth adapter sorted, now for the stylus.
Well the good news is that the pen works straight away, well sort of, here's how we get it working properly
First you need to make sure that wacom-tools are installed, I don't know if it is installed to begin with, if it is not then run sudo apt-get install wacom-tools.

Then you need to install your propriety drivers for your nvidia card, well you don't need to really, but if you use my xorg.conf file you will need to use that driver, so start by going to System ->Administration->Restricted-Drivers-Manager and enable the NVidia driver, if you don't want to use the propriety drivers and want to use the xorg.conf file that I will paste, then you will need to change the

	Driver		"nvidia"

line to

	Driver		"nv"

The line

	Option		"SHMConfig"	"on"

Is added so if you install gsynaptics ( sudo apt-get install gsynaptics ) you can change the operation of your touch pad.

You will not be able to use 3D functionality if you do, or desktops effects, not that they work properly anyway!

next lets sort out the auto rotation

I use this file, I got it from somewhere on the Internet again, don't know where, I have altered it slightly to meet my needs.

Because of the size and shape of my laptop I don't want it to go into portrait mode when I rotate the screen, I want it to go into inverted mode, so the changes that I have made do this, if you want it to go into portrait mode you need to change where it says 'inverted' to 'right' and where it says 'Rotate 3' change it to 'Rotate 1'
you can use gedit rotate.sh then copy and paste the file below into it. Once it has been written and saved, follow the instructions below the script.

#------ Copy from below this line -------


#!/bin/sh

#Author: Patrick Coke & Tim Pope

#modified for ToshibaTecraM7

# by Alexandru Romanescu
# Modified again by Justin Emmanuel

while xset q >/dev/null 2>&1; do

sleep 2 # Polling Interval, 2 seconds

lid="`cat /proc/acpi/button/lid/LID/state|awk '{print $2}'`"

# Looks to see if what orientation the screen is in (Normal or Left)

# and puts the orientation into the $orientation variable

orientation="`xrandr --query | /bin/grep 'Current rotation' | /usr/bin/awk '{print $4}'`"

dpms="`xset q|grep 'Monitor is'|awk '{print $3}'`"

if [ "$orientation" = "normal" -a "$lid" = "closed" ]; then

xrandr -o inverted
xsetwacom set "stylus" Rotate 3
xsetwacom set "cursor" Rotate 3
xsetwacom set "eraser" Rotate 3
orientation="inverted"

elif [ "$orientation" = "inverted" -a "$lid" = "open" ]; then

xrandr -o normal
xsetwacom set "stylus" Rotate NONE
xsetwacom set "cursor" Rotate NONE
xsetwacom set "eraser" Rotate NONE
orientation="normal"

fi

done

#-------------- This is the end, you don't need to copy this line ----------------

Once you have saved this file run the command on it chmod +x rotate.sh
Then I placed this file into /usr/bin/ so that it can be run from $PATH using the command sudo cp rotate.sh /usr/bin
Then go to  System->Preferences->Sessions->new add the name Rotate and the command rotate.sh.

When you log back in the auto rotate should work and the button on the stylus should give you right mouse button functionality.

As for stylus aware applications, there is not much out there for Linux at the moment, I use an application called Xournal which I like for drawing notes. Gimp can be made stylus aware with pressure sensitivity, among a few others. But this is just to get you up and running.

### Post by olejorgen on 2008-01-14
If your volume controller doesn't work you can use this little script.
volume-buttond
```

#!/usr/bin/python

import sys
import os
import dbus
import gobject
from dbus.mainloop.glib import DBusGMainLoop

STEP = 3
CONTROLLER = "Front"
alsain = os.popen("amixer --stdin -q", "w")

def adjust_volume(gain):
	if gain < 0:
		cmd = "set %s %sdB-\n" % (CONTROLLER, abs(gain))
	else:
		cmd = "set %s %sdB+\n" % (CONTROLLER, gain)
	alsain.write(cmd)
	alsain.flush()
def toggle_mute():
	alsain.write("set %s toggle\n" % CONTROLLER)
	alsain.flush()
 
def control(*args):
 	command = str(args[1])
	command = command.strip()
	if command == "volume-up":
		adjust_volume(STEP) # or whatever you want to do
	elif command == "volume-down":
		adjust_volume(-STEP) # or whatever you want to do
	elif command == "mute":
		toggle_mute() # or whatever you want to do
	
DBusGMainLoop(set_as_default=True)
loop = gobject.MainLoop()
bus = dbus.SystemBus()
bus.add_signal_receiver(control, None, 'org.freedesktop.Hal.Device', None,
'/org/freedesktop/Hal/devices/platform_i8042_i8042_KBD_port_logicaldev_input')
loop.run()

```
Add it to your startup
