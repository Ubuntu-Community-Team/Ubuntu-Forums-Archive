### Post by evtedeschi3 on 2008-01-25
OK, so I have a laptop with an X3100 using the intel driver, and an external display. When Ubuntu starts, the LCD and the external monitor are always cloned. Right now, the way I switch between the two is manually using xrandr.

The problem is that sometimes, maybe 40% of the time, when I start Ubuntu and it gets to the login screen, the external monitor displays nothing, even though the laptop's LCD is clearly set at the resolution of the monitor (and even though my external monitor displayed the BIOS screen and the Ubuntu startup load meter). If this happens, then even if I login and run xrandr, the monitor just displays black. If I then blindly use xrandr to go back to LCD-only mode, the laptop LCD displays fine. I then have to restart X, which again doesn't always work.

Is there someway I can just force Ubuntu to go into External-only mode automatically and reliably at startup?
