### Post by ShareBuntu on 2007-03-11
Hey folks,

I decided to give the proprietary ATI driver a shot today. I've got an ATI Mobility Radeon 9000 IGP card. First I added these lines to my xorg.conf file:

Section "Extensions"
        Option  "Composite" "Disable"
EndSection

Section "ServerFlags"
        Option  "AIGLX" "off"
EndSection

Then I installed fglrx using:

sudo apt-get update
sudo apt-get install linux-restricted-modules-$(uname -r)
sudo apt-get install xorg-driver-fglrx

Finally, I ran:

sudo aticonfig --initial
sudo aticonfig --overlay-type=Xv
sudo shutdown -r now

Unfortunately my system didn't boot into X so my assumption is that my card isn't supported. I restored the backed up and xorg.conf file and everything is back to normal. However, do you think it is necessary to keep fglrx installed just in case somehow ATI decides to support my card?

### Post by sysdrum on 2007-03-11
you have to self mod the xorg.conf to get that card to work the sudo aticonfig --initial screws up the xorg file
Waht I did was took my base xorg.conf 
Copied it then added in the parts i need to to run the fglrx driver
You will have to play with it but it will work on your card

```
Section "Module"
	Load	"Glcore"
	Load	"i2c"
	Load	"bitmap"
	Load	"ddc"
	Load	"dri"
	Load	"extmod"
	Load	"freetype"
	Load	"glx"
	Load	"int10"
	Load	"type1"
	Load	"vbe"
EndSection
```

```
 
	Option	    "VideoOverlay" "on"
	Option	    "OpenGLOverlay" "off"
        Option          "AGPMode" "4"
        Option          "AGPFastWrite" "yes"
        Option          "EnablePageFlip" "on"
        Option          "RenderAccel" "on"
```
