### Post by graigsmith on 2007-01-12
search synaptic for usplash. uninstall it.  it will uninstall ubuntu desktop. (that's ok, you can install it later.)

usplash is the thing that block all the error messages on boot.

when you reboot, let us know if you see any error messages.   

if you see one that says usb takeover failed, (bios/smm) bug or something like that. it means your bios has a bug. sometimes you can fix this by disabling "usb Legacy" support.   if not try upgrading your bios.

### Post by graigsmith on 2007-01-12
also, i found that mine didn't work with the extra commented out things.  And you can only use the wacom device like below, if you install the wacom-tools package.

```
Section "InputDevice"
	Identifier "cursor"
	Driver "wacom"
	Option "Device" "/dev/input/wacom"
	Option "Type" "cursor"
	Option "USB" "on"
	Option "Mode" "Relative"
EndSection

Section "InputDevice"
	Identifier "stylus"
	Driver "wacom"
	Option "Device" "/dev/input/wacom"
	Option "Type" "stylus"
	Option "USB" "on"
	Option "PressCurve" "50,0,100,50"
	Option "Mode" "Absolute"
EndSection

Section "InputDevice"
	Identifier "eraser"
	Driver "wacom"
	Option "Device" "/dev/input/wacom"
	Option "Type" "eraser"
	Option "USB" "on"
	Option "Mode" "Absolute"
EndSection

```

### Post by calvmari on 2007-01-13
This is just a shot in the dark, but I have heard that Wacom tablets have very sensitive connections. You must have the Wacom tablet plugged when X starts, or it doesn't work at all. If you unplug it while running X, and plug it back in it won't work. If your Wacom gets bumped and it looses connection for like half a second, it won't work anymore. Could this be your case?

Next time your tablet cuts out, try Ctrl + Alt + Backspace (this reboots X), and see if it starts working. If that's the case, it's a connectivity issue.
