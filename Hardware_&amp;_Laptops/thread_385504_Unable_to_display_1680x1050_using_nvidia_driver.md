### Post by hardyn on 2007-03-21
the driver is able to set the screen resolutions automatically... it will butcher your xorg.conf file, but if you make a backup, then allow the driver to modify it, then merge the two manually, you should be good to go.

gksudo nvidia-settings

make your changes, then save the file, either as xorg.conf or something else.

Ok, that is a flat panel LCD.  On my laptop there is only ONE resolution for the screen 1280x800 in the xorg.conf.  Delete all the other resolutions and have a line like this:

SubSection "Display"
		Depth		24
		Modes		"1680x1050"
EndSubSection

Do that for all the color depths as to give it only one choice.  Running an LCD at any other res only makes the PQ look bad.  If you are not a gamer then running at the native res all the time is not an impact.  For games, running them at 1680x1050 requires a healthy video card and a lot of games don't support WS gaming.

### Post by futz on 2007-03-24
I run a Samsung SyncMaster 205BW at 1680x1050 with  NVidia 7950 GT KO and the latest NVidia drivers. It was easy to get 1680x1050 to work. 

_The critical part is HorizSync/VertRefresh._ You **MUST** get this correct for your monitor or the higher resolution won't be available. I googled for your monitor and found that they only publish the maximums. For the 204BW, max HorizSync is 81 and max VertRefresh is 75. So change your HorizSync and  VertRefresh to
```
    HorizSync       30.0 - 81.0
    VertRefresh     56.0 - 75.0
```

Don't bother using modelines. They're confusing and almost always unnecessary.

Here's the pertinent part of my xorg.conf. Yours would have the difference above, as well as the video card different,  and if you want you can add other resolutions below 1680x1050 to the list:
```
Section "Monitor"
    Identifier     "SyncMaster 205BW"
    HorizSync       30.0 - 65.0
    VertRefresh     50.0 - 75.0
    Option         "DPMS"
EndSection

Section "Device"
    Identifier     "NVIDIA 7950 GT KO"
    Driver         "nvidia"
EndSection

```
Section "Monitor"

    HorizSync       30.0 - 81.0
    VertRefresh     56.0 - 75.0
    Option         "DPMS"
    Option         "ModeValidation" "NoMaxPClkCheck"
EndSection
