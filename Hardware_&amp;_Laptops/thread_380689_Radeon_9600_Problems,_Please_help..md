UPDATE:

Okay, after hours of meticulous searching (Seriously, it's 3AM now) I've come up with nothing. XFree86-DRI seems to want to carry on being horrible and be missing on display 0.0. Fine.

However, I've come up with some good stuff that may be of benefit to all with this issue.

The first, and most fundamental, is to check out your /var/log/Xorg.0.log *every single time you make a change to your xorg.conf*. It's full of useful information. If you do it in a desktop environment, provided you can get in, then It's System > Administration > System Log and apply the filters (WW) and (EE) to find the pertinent problems. 

If you're in command line, i.e. going to the recovery modes provided (very usefully) by GRUB, then you'll wanna do something like:

```
grep -i "(ee)" /var/log/Xorg.0.log
grep -i "(ww)" /var/log/Xorg.0.log
```

With luck, it'll give something like 
```
(WW) Open ACPI failed (/var/run/acpid.socket) (No such file or directory)
(WW) fglrx: No matching Device section for instance (BusID PCI:1:0:1) found
(WW) fglrx(0): board is an unknown third party board, chipset is supported
(WW) fglrx(0): Direct rendering has been disabled.
(WW) fglrx(0): ***********************************************
(WW) fglrx(0): * DRI initialization failed!                  *
(WW) fglrx(0): * (maybe driver kernel module missing or bad) *
(WW) fglrx(0): * 2D acceleraton available (MMIO)             *
(WW) fglrx(0): * no 3D acceleration available                *
(WW) fglrx(0): ********************************************* *

(EE) fglrx(0): DRIScreenInit failed!

```
from desktop.

From command line, I was able to find out some interesting things; this meant I could boot up the xorg.conf (i.e. with my Section "Extensions" | Option "Composite" "disable" | EndSection in place), wait for the blighter to crash and then reboot in recovery mode. Bish bash bosh, grep the logs for warnings and errors and I came across *nothing* which mention DRIScreenInit failing. Closer inspection of the log revealed that DRI had initialised, which was strange to thing considering that all I had had was a black screen in front of me.

Whoop-e-doo, DRI enabled. But where the blazes was it? I'm thinking that the problem **must be the X.org server.** Probably.

One warning at a time, then:

**1. (WW) Open ACPI failed (/var/run/acpid.socket) (No such file or directory)**

Okay, this is plain odd. I've checked on Nautilus and, lo and behold, this file doesn't exist. Not sure what it does, but it has something to do with acpi_listen which according to man (man's greatest friend)

```
        acpid  is  the system-wide ACPI event catcher.  acpi_listen is a simple
       shell-friendly tool which connects to acpid  and  listens  for  events.
       When an event occurs, acpi_listen will print it on stdout.
```

Well, that sounds important and possibly something I might need. Anyone with a working fglrx xorg server get this warning?

**2. (WW) fglrx: No matching Device section for instance (BusID PCI:1:0:1) found**

Again, sounds quite crucial. However, I tried swapping over the BusID for the default Device and also copying it to a new Device section. De nada. Does anyone with working fglrx get this error?

**3. (WW) fglrx(0): board is an unknown third party board, chipset is supported**

If I'm honest with you, I'm not concerned about this in the slightest, since it does state very explicitly that the chipset is supported. The chipset, specific to this thread about Radeon 9600s, is RV350.

**4. (WW) fglrx(0): Direct rendering has been disabled**

(In desktop mode) Great...well, WHY?! I have followed the advice - disabled Composite, added in the sneaky XFree86-DRI disabler, turned around, touched the ground and bagsied to have an fglrx driver working...So why is it being disabled? Obviously this isn't necessarily applicable to this thread, more just a vent for frustration, but is there a way to gather additional information? A way of making fglrxinfo verbose?

(by the by, there were other warnings there too...I dealt with them whilst I could and have now forgotten entirely what they were. One was a font issue, though, others involving some options I'd enabled as a workaround which didn't work, and I also commented out the Wacom drivers in my Xorg.conf. Don't follow my bad example :) )

I appreciate that this edit has been hugely random, but it was also necessary to keep y'all posted. The following got me into my GNOME desktop:

```

EndSection
```

So it can be done...that just doesn't give you the 3D bits 'n' bobs I want, really.
