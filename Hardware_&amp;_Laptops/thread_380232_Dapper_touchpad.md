### Post by vayde on 2007-03-09
No worries. Below is the touchpad section of my xorg.conf.  You will need to paste it into the matching section in /etc/X11/xorg.conf .  Thats the X server config file.

```

Section "InputDevice"
    Identifier     "Synaptics Touchpad"
    Driver         "synaptics"
    Option         "SendCoreEvents" "true"
    Option         "Device" "/dev/psaux"
    Option         "Protocol" "auto-dev"

	Option 		"LeftEdge" 		"130"
	Option 		"RightEdge" 		"840"
	Option 		"TopEdge" 		"130"
	Option 		"BottomEdge" 		"640"
	#when finger pressure drops before this value it counts as a release
	Option 		"FingerLow" 		"7"
	#when finger pressure drops below this value it counts as a release
	Option 		"FingerHigh" 		"8"
	#max. time (in miliseconds) for detecting a tap
	Option 		"MaxTapTime" 		"180"
	#max. movement of tthe finger for detecting a tap
	Option 		"MaxTapMove" 		"110"
	#duration of the mouse click generated by tapping
	Option		"ClickTime"		"130"
	
	Option 		"EmulateMidButtonTime" 	"75"
	#move distance of the finger for a scroll event
	Option 		"VertScrollDelta" 	"20"
	Option 		"HorizScrollDelta" 	"20"
	#min./max. speet factor
	Option 		"MinSpeed" 		"0.60"
	Option 		"MaxSpeed" 		"1.10"
	#acceleration factor
	Option 		"AccelFactor" 		"0.030"
	#slowest setting for edge motion speed
	Option 		"EdgeMotionMinSpeed" 	"200"
	#fastest setting for edge motion speed
	Option 		"EdgeMotionMaxSpeed" 	"200"
	# finger pressure at which minimum edge motion speed is set
	Option		"EdgeMotionMinZ"   	"30"
	# finger pressure at which maximum edge motion speed is set
	Option		"EdgeMotionMaxZ"   	"100"
	# If on, the up/down buttons generate button 4/5 events.
	# If off, the up button generates a double click and
	# the down button generates a button 2 event.
	Option 		"UpDownScrolling" 	"1"
	#switch on/off shared memory for configuration
	Option 		"SHMConfig" 		"on"
	Option 		"Emulate3Buttons" 	"on"

EndSection


```

The first 6 or so lines are probably already there.  You just need to paste in the others.

If you don't know how to do that, run the following:

first back up your current config
```

cd /etc/X11

sudo cp xorg.conf xorg.conf.backup

```

then edit the file
```

sudo gedit xorg.conf

```

then you can reboot or just restart the X server and see what happens.

If the X display won't start up, you need to copy the backup to the original with the following:

```

sudo cp /etc/X11/xorg.conf.backup /etc/X11/xorg.conf

```

and you will be back where you started.

Let me know if you need more info.  

Unfortunately you are playing with X server configuration, which means if something is wrong, you won't have a graphical display to fix the problem.

Good luck, and don't hesitate to holler if you need to

You can also reach a bunch of help by coming to #ubuntuforums-beginners on freenode irc

### Post by vayde on 2007-03-10
Oh No!  I apologize.

It's well worth your time to check out #ubuntuforums-beginners on freenode irc.  I'm often there, and a bunch of others who can get you up to speed.

You should know that all the various parts of a linux installation are modular.  The can function independantly of each other, and they can be broken and fixed independant of each other.  Almost certainly, it was just the graphical server that broke.  It could have been fixed without the need for a re-install.

There are some basic skills that you will need to pick up in order to fix something like an X server.  It's not too bad though.

If you crash out your X server again, log in on the command line and run the following command:
```

sudo dpkg-reconfigure -phigh xserver-xorg

```

that will rewrite the file you altered from scratch and everythign will be fine again when you reboot.  You don't really even need to reboot, but rebooting is probably the easiest way for you right now.

Don't give up.  The learning curve levels out pretty quick.  There's just a couple of basic skills you need to pick up.

Contact me here, or directly on Yahoo, or better yet on freenode irc.  We'll walk you through the process.
