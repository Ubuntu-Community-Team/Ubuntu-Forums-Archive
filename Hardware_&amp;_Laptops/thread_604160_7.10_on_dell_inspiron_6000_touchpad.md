### Post by RogerTa on 2007-11-05
Hi all,

I am running ubuntu 7.10 on a dell i6000.  Out-of-the-box, the touchpad is way too sensitive (as many people have reported), but through lots of reading of these forums, I have found the following makes the experience *much* better for me:

FingerLow=3
FingerHigh=35
MaxTapTime=150
MaxDoubleTapTime=150
MaxTapMove=180

(The idea here is to decrease sensitivity by increasing the spread between low and high, as well as decreasing the maximum time and movement of what is considered a "click")

However, this is still not as good an experience as windows on this same machine.  Does anyone know if there exists a tool for windows like synclient?  I'd like to see if I can gather the parameters used by windows and apply them to 7.10, to see if I can get an even better experience.

Thanks.

### Post by Egalus on 2007-11-06
I also use an inspiron 6000, my touchpad config in xorg.conf looks like this:
```

Section "InputDevice"
  Driver "synaptics" 
  Identifier "Synaptics Touchpad"
  Option "Device"               "/dev/input/event6"
#  Option "Device"               "/dev/psaux"
#  Option "CorePointer"
  Option "SendCoreEvents"       "true"
  Option "Protocol"             "event"
  Option "LeftEdge"             "130"  
  Option "RightEdge"            "840"  
  Option "TopEdge"              "130"  
  Option "BottomEdge"           "640"  
  Option "FingerLow"            "15"   
  Option "FingerHigh"           "20"   
  Option "MaxTapTime"           "180"  
  Option "MaxTapMove"           "110"  
  Option "EmulateMidButtonTime" "75"   
  Option "VertScrollDelta"      "20"   
  Option "HorizScrollDelta"     "20" 
  Option "MinSpeed"             "0.60"
  Option "MaxSpeed"             "1.10"
  Option "AccelFactor"          "0.030"
  Option "EdgeMotionMinSpeed"   "200"  
  Option "EdgeMotionMaxSpeed"   "200"  
  Option "UpDownScrolling"      "1"    
  Option "CircularScrolling"    "1"    
  Option "CircScrollDelta"      "0.1"  
  Option "CircScrollTrigger"    "2"    
  Option "SHMConfig"            "on"   
  Option "Emulate3Buttons"      "on"   
  Option "TapButton2"           "2"    
  Option "RBCornerButton"       "2"    
  Option "LBCornerButton"       "1"  
  #Option "Repeater"            "/dev/input/event2"
  Option "SHMConfig"            "true"
EndSection

```

Warning, /dev/intput/eventx probably needs to be changed for your kernel as I am running a 2.6.23.1 kernel with linux-phc and tuxonice as hibernate/suspend with stock ubuntu kernel (and stock ubuntu) simply sucks.
You can find the proper /dev/input/event when checking the name checken the event names in /sys/class/input.
Example:
```

root@schleppy:/sys/class/input# cat input6/name 
AlpsPS/2 ALPS GlidePoint

```
