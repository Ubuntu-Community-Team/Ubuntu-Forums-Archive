### Post by louis_nichols on 2007-02-16
My guess would be that you don't have the videro driver for the curent kernel... It sometimes happens after the kernel is updated that the video driver takes a while to make it into the repos. Also, it can be that you don't have the linux-restricted-modules metapackage installed, so this means that when you update your kernel, the video drivers don't get updated automaically.

So I would recommend you do this:

sudo apt-get install linux-restricted-modules-`uname -r`

Then reboot and tell us the outcome.

### Post by louis_nichols on 2007-02-16
OK. so that explains it. You really don't have the nvidia driver installed.

Here's the thing. They seem to not be packaging nvidia drivers for the geeneric line of kernels. I got the exact same problem some weeks ago. The solution is simple: just switch to the 386 line of kernels. So what you need to do:

sudo apt-get install linux-image-386 nvidia-glx linux-restricted-modules-386

Then reboot and you should be alright from now on on every update. Your xorg.conf is ok ;)

### Post by louis_nichols on 2007-02-16
actually, that doesn't really matter. nv is the open source driver for nvidia cards and there is no need to list it as disabled, once you install the proprietary driver. the idea is that the free driver gets loaded if the one from nvidia isn't available but if it IS available, then the free one is just ignored.

did you install all the packages I listed? If you did and still have this problem, then install also the package nvidia-xconfig and then execute in terminal the command *nvidia-xconfig*. then reboot and it should work. The list of disabled modules doesn't have to be changed at all.
