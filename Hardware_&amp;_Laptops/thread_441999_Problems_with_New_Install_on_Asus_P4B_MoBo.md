### Post by jbrice on 2007-05-13
As a relative newcomer to Linux, I thought I would try out Ubuntu 7.04 as a dual-boot on an older WinXP system that uses an Asus P4B266 motherboard.  
Ubuntu is generally working OK, but there are some problems that seem to me to be related to compatibility with the mobo. hardware:
- boot takes nearly 5 minutes, with most of the being taken with "waiting for root filesystem",
- the hard disk drive indicator is on all of the time (tho. the sounds of disk activity follow the sort of pattern you would normally expect),
- on shutting down, the system runs to a halt, but doesn't switch off (again with hard disk LED on all of the time).

Sysinfo identifies the Intel chipset components OK, but also includes an "Asustek Inc unknown device 8028" under "IDE interface", and a similar "Asustek Inc unknown device 8070" under "Host Bridge".

After all the good things I have heard about Ubuntu I'm really disappointed to be having problems like this with what I thought was a pretty standard motherboard. :-(  So any suggestions for fixes would be appreciated.
