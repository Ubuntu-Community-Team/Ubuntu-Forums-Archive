### Post by charles.figura on 2006-05-23
I've got an problem that seems very repeatable.  I've got a Dell D620 with Dapper that I've been working on for a couple of days.  In the past day or so there's been a new development - when I open the lid after closing it (even for a moment) the keyboard goes (almost) completely dead - only the Function combination to increase/decrease screen brightness function.

It's important to note that the lid switch is set up to do *nothing*.  It doesn't sleep, doesn't hibernate - neither of these things work right anyway.  When I open up the lid, the screen comes back on, the touchpad and pointy stick and buttons function, they system is responsive - the keyboard is just *dead*.  Capslock, numberlock don't respond - they don't flash the indicator lights.

Using the mouse I can tell the computer to shutdown (rightclick on the desktop).  It starts to shutdown, but hacks before the X server quits - I end up with a blank screen (back light on), x cursor visible and responsive, but no action, nothing.  I have to hold down the power button to reboot.

When it reboots, the system is fine.  Keyboard works just like normal.

It didn't start out like this.  Anyone have any ideas here?  I can't figure out what happened.

### Post by unknown555525 on 2006-05-24
It could be your OS, it could be a worn ribbon, I've had both problems.

In most cases you should just reinstall your OS, or check the ribbon connecting the keyboard to the mainboard.

I had a Compaq EVOn610c with the exact same issue. I just reinstalled my copy of XP pro everything worked fine, I had a lot a viruses on it before I reinstalled xp, it could have been that.
