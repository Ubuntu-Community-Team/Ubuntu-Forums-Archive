### Post by zgornel on 2007-01-14
Right. I was thinking about a P4 but 915/945 are only for Pentium M derivates :D . So that's what you have to do:
1. Get the archive, extract the files somewhere safe. 
2. make a backup copy of your /usr/lib/xorg/modules/drivers/i810_drv.so and overwrite it with that from the archive.
3. make a backup copy of your /usr/lib/libI810XvMC.so.1.0.0 and overwrite it with that from the archive. 
Your X server might crash and restart when overwriting the i810_drv.so, not to worry.
Google earth also moves better ;)
Please give some feedback if it worked so other can use the libs. Again, these are the latest xorg 2D drivers from intel, not 3D related (those are Mesa).

P.S I have compiled the Mesa 6.5.2 3D driver but there is absolutely no performance improvement. I can post those too with the install instructions.

I am having the same badalloc problem under feisty fawn with an intel i915 video card when using xv as the output video driver.

I solved the problem adding the following line in the Device section of xorg.conf:

```
Option "LinearAlloc" "8192"
```
