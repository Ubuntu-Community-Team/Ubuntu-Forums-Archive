### Post by gazj on 2006-04-14
ok, the could be in a file called xorg.conf, this is where all your graphical options are stored one of them being the maximum screen resolution allowed

to change this we need to open what is called a terminal

press alt+F2 together

then enter 

```
gnome-terminal
``` followed by enter

then when the terminal comes up type in it

```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.backup
```
this backs up the file incase anything goes wrong

```
sudo gedit /etc/X11/xorg.conf
```
the file should be opened in what is similar to windows notepad

now you have to find the piece in the code where your available screen resolutions are set and add the ones you want

here is an example of my file

notice the bit under the section headed screen

once you have edited your xorg.conf file, click save and close it

now you have to restart you xserver (this is the name of the service that draws all graphics to the screen), press ctrl+alt+backspace to do this, and hopefully your problem will be solved

now you might want to write this bit down on paper

if for some reason the xserver doesn't restart you will be left at a black and white command prompt screen and you will need to reuse your backup to do this type the commands

```
cd /etc/X11
rm xorg.conf
cp xorg.conf.backup xorg.conf
startx
``` 

sounds like hell if your new to linux, but its not as hard as it sounds

and it may not be the problem but its the first place to ususaly look

### Post by Lars Skovbo on 2006-04-24
I'm having (have had) the same problem with my Acer TravelMate 230. Even tho i configured the xserver, edited the xorg.conf and verified the resolution to be 1024x768, the resolution in Gnome would still be 640x480. Changing the driver (from i810 - the inbuilt intel graphix driver) to vesa (universal) just messed thinges more up. - I tried this because it works on my other system with a ATI X1900XT.
What realy got me laughing was gazj's post about changing the refreshrate manually.
Just to make thinges clear for you (johnc9451(??) :) i'm a noob like you ;)) you could do this:
Log into Gnome (with your awsomely low resolution)
Start the terminal and type in the following:
sudo gedit /etc/X11/xorg.conf 
*(this opens a text editor where you can alter the xorg.conf)*
Now go to the monitor section (in my case line 78 ) and make it look like this (copy/paste): 

Section "Monitor"
	Identifier	"Generic Monitor"
	Option		"DPMS"
	HorizSync	31.5-48.5
	VertRefresh	59.0-75.0
EndSection

Hurray. Now you should see things in a mutch broader perspective, after a ctrl+alt+backspase restart, that is.
