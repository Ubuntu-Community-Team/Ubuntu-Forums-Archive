### Post by solar george on 2007-11-29
Make sure that th¡s section of your xorg.conf is uncommented:
```
        Driver          "wacom"
        Identifier      "stylus"
        Option          "Device"        "/dev/input/wacom"
        Option          "Type"  "stylus"
        Option          "ForceDevice"   "ISDV4"# Tablet PC ONLY
EndSection

Section "InputDevice"
        Driver          "wacom"
        Identifier      "eraser"
        Option          "Device"        "/dev/input/wacom"
        Option          "Type"  "eraser"
        Option          "ForceDevice"   "ISDV4"# Tablet PC ONLY
EndSection

Section "InputDevice"
        Driver          "wacom"
        Identifier      "cursor"
        Option          "Device"        "/dev/input/wacom"
        Option          "Type"  "cursor"
        Option          "ForceDevice"   "ISDV4"# Tablet PC ONLY
EndSection

```

and

```
        # Uncomment if you have a wacom tablet
        Inputdevice     "stylus"        "SendCoreEvents"
        Inputdevice     "cursor"        "SendCoreEvents"
        Inputdevice     "eraser"        "SendCoreEvents"

```

(I´m writing this us¡ng handwrit¡ng recognition on a Portege M200 tablet pc)

### Post by solar george on 2007-12-20
> get an error indicating that I don't have enough privileges to make changes
That is because even though you are running as an admin user you do not have permisions to change the file - only root the super user can do this.
You will have to prefix commands with sudo to run them as root.
e.g.
```
sudo gedit /etc/X11/xorg.conf
```
rather than
```
gedit /etc/X11/xorg.conf
```

To uncomment a line remove the # symbol at the start,
```
# Commented line
Uncommented line
```

### Post by caliston on 2007-12-20
I don't think the open source 'nv' driver supports rotation, while last time I checked the closed-source 'nvidia' driver doesn't do hibernation (suspend-to-disc) correctly.

Today I've been playing with the synaptics touchpad driver.  By default the pad emulates a mouse, but you can fully configure it.  The M200 doesn't actually have a Synaptics touchpad, but it has a pad made by ALPS which seems to work in a similar way.  Since they're called the same I refer to the hardware as 'Synaptics' and the driver as 'synaptics'.  Even more confusingly Ubuntu also has the 'synaptic' package manager, a GUI version of 'apt', which is nothing to do with touchpads.

The synaptics driver lives at:
[http://web.telia.com/~u89404340/touchpad/](http://web.telia.com/~u89404340/touchpad/)
but it was already installed in my Ubuntu 7.04.  You might want to install the 'xserver-xorg-input-synaptics' package if it isn't.

In my /etc/X11/xorg.conf I have (mostly pinched from README.alps in the synaptics source tarball):

```

Section "Module"
...
        Load    "synaptics"
EndSection

Section "InputDevice"
  Driver        "synaptics"
  Identifier    "Synaptics Touchpad"
  Option        "Device"                "/dev/psaux"
  Option        "Protocol"              "auto-dev"
  Option                "SendCoreEvents"        "true"
  Option        "LeftEdge"              "120"
  Option        "RightEdge"             "830"
  Option        "TopEdge"               "120"
  Option        "BottomEdge"            "650"
  Option        "FingerLow"             "14"
  Option        "FingerHigh"            "15"
  Option        "MaxTapTime"            "180"
  Option        "MaxTapMove"            "110"
  Option        "EmulateMidButtonTime"  "75"
  Option        "VertScrollDelta"       "20"
  Option        "HorizScrollDelta"      "20"
  Option        "MinSpeed"              "0.3"
  Option        "MaxSpeed"              "0.75"
  Option        "AccelFactor"           "0.015"
  Option        "EdgeMotionMinSpeed"    "200"
  Option        "EdgeMotionMaxSpeed"    "200"
  Option        "UpDownScrolling"       "1"
  Option        "CircularScrolling"     "1"
  Option        "CircScrollDelta"       "0.1"
  Option        "CircScrollTrigger"     "2"
  Option        "SHMConfig"     "on"
EndSection

Section "ServerLayout"
...  
        InputDevice     "Synaptics Touchpad"
EndSection

```



Note that the original Synaptics hardware has a different resolution - using the default synaptics options gives very slow pointer movement and no scrolling.

The line:
```
  Option        "SHMConfig"     "on"
```
allows the 'synclient' program to configure the driver - see the man page for synaptics(5) for all the options.  Textual configuration is not very easy because you want to keep trying out different options, though it's OK for just tweaking a few things.

There are some GUI frontends.  I've installed the Ubuntu packages gsynaptics, ksynaptics and qsynaptics.

gsynaptics is a GNOME app, but it doesn't configure very much
ksynaptics is a KDE configuration plugin - I don't run KDE so don't know too much about it
qsynaptics is a standalone QT-based app - configures a few more things than gsynaptics

I have to say, though, that these aren't much use with the ALPS touchpad because their default options are for the Synaptics touchpad and break the ALPS.

So, for example, I've turned a touchpad single-tap into a middle mouse click with:
```
$ synclient TapButton1=2
```
or an entry in the synaptics InputDevice section of xorg.conf:
```
  Option        "TapButton1"    "2"
```
TapButton2 is a two-finger-tap and TapButton3 a three-finger-tap, but I can't get them to work.
