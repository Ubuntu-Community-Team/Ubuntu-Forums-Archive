### Post by DrMega on 2008-02-26
Here's how I fixed it on mine.

1. Go to the directory that contains menu.lst

```

cd /boot/grub/

```

2. Backup menu.lst in case we bust it good and proper by mistake:

```

sudo cp menu.lst menu.lst.backup

```

3. Open it up in your favourite text editor:

```

sudo gedit menu.lst

```

4. Find a line that looks something like this:

```

kernel		/boot/vmlinuz-2.6.22-14-generic root=UUID=d0c2bf1f-a099-40d0-9ff2-7194dcacda45 ro quiet splash

```

5. See where it says 'splash' at the end of the line? Simply remove the word splash.
6. Save your changes.
7. Reboot.

What happens then (in my case at least), is your machine then boots up fine with the TV s the only monitor, remembering all the settings that you configured while the monitor was connected (remember to make the TV the primary display).

The only downside is that you lose the nice graphical boot sequence, and have to make do with loads of text whizzing up your screen during boot.

In my setup I also setup my account as the default login and set it so that I don't have to put my username and password in. This is easy enough to do from the GUI, but I'm sure any number of people will pile on to tell you why you shouldn't really do that.

Good luck.

### Post by DrMega on 2008-02-26

Ok. Is your monitor connected or are you still wrestling with the TV-out? I would have it on the monitor while sorting this if I were you (if you aren't already).

If you have the monitor connected, I would install the nvidia driver and the config tools mentioned earlier, and then configure it so that it clones your monitor with your TV. **However** if you are experiencing the problems you describe with the monitor connected (and not just the TV alone), then we need to take some drastic action to get you back on track. Continue reading if this is the case.

I would reboot, and when you get to the grub menu, choose to boot into safe mode. This is text based and so immune to the problems you are having. Once booted, take the following steps:

1. Take a backup of your xorg.cong (not that much point as it is bust anyway, but it is good practice)

```

cd /etc/X11
sudo cp xorg.conf xorg.conf.backup

```

2. Edit xorg.conf to change the driver to the safe but feature poor MESA driver.

```

sudo pico xorg.conf

```

Find the line that looks something like:

```

Driver = nv

```

and change nv to MESA

3. Reboot

```

sudo shutdown now -r

```

This should take you to a rather basic 800x600 or 1024x768 desktop just on your monitor (no TV-Out) and no acceleration. From here I would (re)install the the proprietary nVidia driver (either using the restricted drivers manager or Envy), and configure your TV out using the nVidia utilities.

You've already edited your menu.lst file so from hereon, you should be home and dry.

One last thing I would say though, each time I've done a rebuild I've had to reconfigure my TV-out (because I forgot to backup my xorg.conf file), and each time it takes a bit of perseverance.
