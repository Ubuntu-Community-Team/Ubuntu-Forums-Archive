### Post by jjordan on 2007-06-10
OK,  I address this from an Ubuntu perspective since it is actually a bit off topic.  I have owned 10-20 diffrent PocketPC devices that is to say devices running the Microsoft Windows CE / Windows Mobile operating system.  To date I have never successfully syncronized any of them with Linux.  I have been trying on and off for a couple of years.  ( I also never really got them to sync with Windows reliably to tell the truth.)   The current version of Windows Mobile is "5"  I have an iMate Jamin (HTC Prophet) that I have not been able to get synce (the Linux syncronization program) to recognize, it uses WM5.  I also have an iMate Jam (HTC Wizard) which uses Windows Mobile 2003 and it at least is recognized by synce.  To make matters more interesting WM6 is coming out.
[http://www.synce.org/index.php/SynCE-Wiki](http://www.synce.org/index.php/SynCE-Wiki)


I believe the sync situation with Palm devices is much better but Palm is moving to Windows Mobile.

Within the next year the FIC NEO1973 should be released to the general public.  (I will be standing in line to buy one).  This is a Linux based OPEN SOURCE PDA Phone.  I am sure reliable Linux sync will be available shortly after it is released.
[http://www.openmoko.org/](http://www.openmoko.org/)

Since I cannot wait that long I am currently in the market for an HTC 3600 because I want to use it as a UMTS modem.  That (modem) should work with Ubuntu without too much difficulty.
[http://www.htc.com/product/03-product_p3600.htm](http://www.htc.com/product/03-product_p3600.htm)

[B]
Here is what I am doing now to sync things:[/B]
Email:  I have a gmail account which grabs email from my other accounts: everything in the gmail account is forwarded to my AIM account (gmail does not offer IMAP but AIM does which allows me to see all mail on both my PDA and my home/labtop system).  I tried 1and1 hosted Exchange Server but it seems they have intentionally set up their servers to prevent using a PDA or Linux (they want you to do it all through a browser).
[http://www.gmail.com](http://www.gmail.com)
[http://www.aim.com/](http://www.aim.com/)

Calendar: There is a free external program to sync Google Calendar to Windows Mobile.  Kmail can read but not write to a Google Calendar.   There is a daemon available to handle reading and writing but I have not set it up (it is a Java program which means it is going to have a massive memory overhead [bad thing in a daemon]) When I schedule something at work using OutLook I simply invite my gmail address and that way it shows up in my gmail calendar.  (My work does not even allow PDAs to be brought into the building so direct sync is out of the question.)
[http://www.freewarepocketpc.net/ppc-download-activegcsync.html](http://www.freewarepocketpc.net/ppc-download-activegcsync.html)

Files: I use a 4GB SD card with a bunch of movies and other stuff I either use the card reader slot on my laptop or Card Export on the PDA with a usb cable (Card Export works fine with Ubuntu).  Every morning I run a script that uses wget downloads NBC News to the SD card so I can watch it on the way to work. [http://www.softick.com/pocket-pc/cardexport2/](http://www.softick.com/pocket-pc/cardexport2/)

Most modern PDA-Phones have WiFi you might also want to look for GPS.  I have been fairly happy with HTC products but they do crash (this is Windows after all) so keep your sense of humor.  There is a great hacker community for the HTC devices.
[http://www.xda-developers.com/](http://www.xda-developers.com/) 

Sorry, Too Much Information

-j
