### Post by scott_g on 2007-05-28
You could also use a tool called gtf.  It allows you to set any screen resolution and refresh rate.  It is used like this:

in terminal, type:

gtf <1st dimension> <2nd dimension> <refresh rate>

eg: 
gtf 1280 1024 60 

produces

  # 1280x1024 @ 60.00 Hz (GTF) hsync: 63.60 kHz; pclk: 108.88 MHz
  Modeline "1280x1024_60.00"  108.88  1280 1360 1496 1712  1024 1025 1028 1060  -HSync +Vsync

Copy and past this line in your monitor section.

Be sure of two things:
a) you have a corresponding mode in your modes section
b) you hsync value is the same as the one given in the commented line (this one stumped me for a while; couldnt get a resolution over 1024x768 because of my hsync value).

Hope if helps.
Scott

### Post by ben.s on 2007-05-29
I managed to fix this by explicitly setting the VertRefresh and HorizSync values to outrageous values (1-1000 for both).  Then magically all the modes started to work.  Of course, there is the off chance that doing this could blow up your monitor.  YMMV.

### Post by ben.s on 2007-05-29
Slight correction -- that only worked with an old version of the nVidia driver installed.  However, I did manage to fix the problem by adding the following option in the Device and Monitor section of xorg.conf:

```
Option "UseEDID" "False"
```

Again, YMMV.
