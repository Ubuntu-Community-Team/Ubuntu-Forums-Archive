### Post by aaronc222 on 2005-12-04
Well, I did some more searching on my own and it was a fairly easy fix.

Just add "acpi=off pnpbios=off" to the end of the "kernel" line in grub for the 2.6.14.2 entry. I didn't need this for 2.6.12 kernels, just the 2.6.14.
```
title		Ubuntu, kernel 2.6.14.2 
root		(hd0,0)
kernel		/boot/vmlinuz-2.6.14.2 root=/dev/hda1 ro quiet splash acpi=off pnpbios=off
initrd		/boot/initrd.img-2.6.14.2
savedefault
boot

title		Ubuntu, kernel 2.6.12-10-686 
root		(hd0,0)
kernel		/boot/vmlinuz-2.6.12-10-686 root=/dev/hda1 ro quiet splash
initrd		/boot/initrd.img-2.6.12-10-686
savedefault
boot
```

Sound now works in 2.6.12-9, 2.6.12-10, and 2.6.14.2. I know I've got too many kernels, but I still don't want to get rid of the old ones 'just in case'.

Hope this helps anyone else whose sound worked in 2.6.12 but tried 2.6.14 and didn't have sound. I found it from someone whose network didn't work in 2.6.14 and thought I'd try it.
