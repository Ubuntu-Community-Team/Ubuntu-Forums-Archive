### Post by timredfern on 2006-12-14
I have just installed ubuntu, 'edgy eft x64' on my new laptop, a dell d620 (nvidia quadro version).

It all seems to be working except for one really annoying problem,

When logged into ubuntu the trackpad erratically seems to lose focus in a weird way.
The pointer can still be moved, but the buttons become non-operational. Sometimes the cursor changes to a closed fist (I think).
When in this state the mouse is useless, but activating a menu through a keyboard command (i.e. alt-tabbing through applications when there are more than one open, or pressing alt-F or whatever the file menu shortcut is) will restore the trackpad buttons operation. This will stay working for maybe 60 seconds, it problem seems to happen no matter what I am doing.

I have looked through the thread 'Anyone tried Ubuntu on Dell Latitude D620' and also 'alps track pad problem - dell 9300'

Whch mentions a problem with an alps trackpad which was reported solved:
'I think upgrading to kernel 2.6.15 686 version fixed this - the alps track pad is working perfectly now'

However apparently 'edgy eft' has kernel version 2.6.17

Any ideas?

Tim Redfern

### Post by timredfern on 2007-01-03
well it turns out, installing the x86 release 100% fixed it.
