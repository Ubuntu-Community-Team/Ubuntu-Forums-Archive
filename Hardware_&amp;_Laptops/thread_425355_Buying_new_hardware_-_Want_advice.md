### Post by jespdj on 2007-04-27
The nVidia 8xxx video cards are so brand new (the 8600 was only released last week or so!) that there aren't yet very good drivers available. I've seen a lot of people complain about the lack of good drivers for them, not only for Linux but also for Windows XP and Vista. The drivers will get better but it will take some weeks or months.
> **skizero said:**
> Every once in a while I think I read a post that talks about lack of 64-bit drivers/programs. Is that a concern? Ways to overcome them?
With 64-bit Windows the lack of 64-bit drivers is a serious problem, because almost all Windows software is proprietary, so you're in the hands of vendors with regard to drivers. If a hardware manufacturer decides that it's not worth it to make a driver for Windows x64 then you're screwed.

I think this is less of a problem with Linux because most Linux software is open source, and someone will have built a 64-bit version of whatever driver you need. Unfortunately the two big video card manufacturers (ATI and nVidia) both only have proprietary video drivers for Linux, but they both have 32-bit as well as 64-bit versions available. As far as I know ATI's drivers are much worse than nVidia's, especially when you want accelerated 3D graphics (OpenGL). I'm thinking of tossing out my ATI X1600 and getting an nVidia 8600GTS myself but I'll wait some time until the drivers get better.

32-bit programs also run on 64-bit Linux, so most software isn't a problem.

I'm running 64-bit Ubuntu 7.04.
