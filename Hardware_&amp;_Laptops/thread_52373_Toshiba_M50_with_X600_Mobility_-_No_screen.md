### Post by varunus on 2005-07-27
Well, one thing to try:  Open up your xorg.conf file and change the driver to "vesa" (lower case) instead of fglrx.  Then try getting into a GUI.  Does this work?  It should let you get into a GUI, but you won't have any acceleration at all (so performance will be bad).

Can you post your /etc/X11/xorg.conf file here?  Also your /var/log/Xorg.0.log file.

        Option          "MonitorLayout" "LVDS,STV"

in the Screen section.  I tried "AUTO, AUTO" and other combinations but this is the one that worked.  I haven't tried it with an external monitor.

### Post by Kanhorta on 2005-08-19
I've got good news!
I finally got it working at a 1280x768 resolution ^_^
Still no 3D acceleration but with a bit more investigation it should work too.

It's kind of a hack, but, who cares, it works :P

First, I did it with the last version of ati-drivers (8.16.2) so download them from their website.
Install them following its instructions and run fglrxconfig to generate a fully ati-configured xorg.conf.

Now comes the hack:
We are gonna fool Xorg and ATi drivers and make them think we have a "supported" ATi graphics card instead of Toshiba's ATi X600SE.
To do this trick we need this little program: [http://members.lycos.co.uk/wedge_antilles/fglrx_binary_edit.tar.bz2](http://members.lycos.co.uk/wedge_antilles/fglrx_binary_edit.tar.bz2).
Uncompress and compile it by executing "make". You'll need pciutils, so it might be necessary to "apt-get install pciutils".
Once compiled do this:
```
sudo cp fglrx_binary_edit /lib/modules/fglrx/build_mod/
cd /lib/modules/fglrx/build_mod
fglrx_binary_edit 3152 5462

``` 
This will swap the MOBILITY RADEON X300 (M22 3152) (with id 0x3152) (which is supported) with our X600SE (id 0x5462) (which isn't supported).
I don't know if all Toshiba M50-137 laptops have the same graphics adapter so you better check your id with "lspci". 

And finally, we're gonna fool Xorg by adding this line in /etc/X11/xorg.conf:
```
Section "Device"
    Identifier                          "ATI Technologies, Inc. Radeon X600 (RV380)"
    Driver                              "fglrx"
...
**ChipID 0x3152**

```

Make sure you configured a widescreen resolution too:
```
Section "Screen"
    Identifier  "Default Screen"
    Device	"ATI Technologies, Inc. Radeon X600 (RV380)"
    Monitor     "Monitor genèric"
    DefaultDepth 24
    Option	"backingstore"

    Subsection "Display"
        Depth       24
        Modes       **"1280x768"** "1152x864" "1024x768" "800x600" "640x480"
        ViewPort    0 0  # initial origin if mode is smaller than desktop
#        Virtual     1280 1024
    EndSubsection
EndSection
```

I've also merged my previous fonts section of xorg.conf with the new one generated by fglrxconfig because the font anti-aliasing wasn't properly configured and fonts were ugly...

Reset Xorg (Ctrl+Alt+Backspace) and *voilà*.

Give thanks to [http://odin.prohosting.com/wedge01/gentoo-radeon-faq.html](http://odin.prohosting.com/wedge01/gentoo-radeon-faq.html)
