### Post by EntropicTundra on 2007-04-08
Not so long ago I bought an Acer Aspire 5051 AWXMi laptop with the intention of loading Ubuntu onto it (beats Vista Home Basic any day). So far everything's working well (including the wireless - yay!), with one exception: The graphics. Both the fglrx and ati drivers fail to load properly, and the vesa driver is the only thing that works. If I try the ati driver and check the Xorg log, my chipset is in its giant list and picked up on (Xpress 200M, claimed to be an Xpress 1100), but both fail with the same error. The system's also slighly likely to hang with a messed up screen if I switch between the desktop and a tty too much, but this is a secondary issue for now. Essentially, I'd just like to get one of the drivers working (I don't really need 3D acceleration) so I can have my nice, lovely 1280x800 resolution.

Relevant specs:
Graphics chipset: ATi Radeon Xpress 200M (labelled as Xpress 1100, a connection confirmed by ATi's specifications page), 128MB(?) shared memory
Processor: AMD Turion64 MK-36 (2GHz)
512MB RAM
