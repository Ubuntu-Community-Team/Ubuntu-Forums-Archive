### Post by elver on 2007-10-25
I got myself a brand new 22 inch Fujitsu-Siemens ScaleoView L22W-7 TFT display. It's quite excellent, except there's a problem whereby certain hues are overlaid with flickering vertical green stripes. The interesting thing is that the hues where this happens can be "chosen" by changing the "black level" setting in the monitor configuration.

Another interesting aspect is that the Ubuntu 7.04 login screen makes for the perfect test image for this. It features a radial gradient around the word "ubuntu" and when I change the black level from 0 to 100, the area of flickering vertical green stripes starts out as a tiny thick oval around the text, expands further out as I up the value, expands out of the screen area, and starts again from the center -- this time as a large island that covers the whole logo.

Here's the important info, condensed:

Graphics card: nVidia 6600GT, DVI connector. Latest stable drivers.

Monitor: 22 inch Fujitsu-Siemens ScaleoView L22W-7 TFT display.

Operating system: exact same problem exhibits itself under both Windows XP SP2 and Ubuntu Linux 7.04.

Symptoms:
1. Certain hues of color are overlaid with flickering vertical green stripes.
2. Changing the "black level" of the monitor changes the hue on which the stripes appear.

Additional information:
1. According to Fujitsu-Siemens' website, there are no drivers for this monitor for any operating system. It's a plug-n-play monitor.
2. Because the monitor came packed with only the analog VGA cable, I purchased a separate cheap Bandridge DVI cable and I'm using that.
3. I have not tested to see whether the same problem appears using an analog cable. Planning to do that tomorrow unless someone has a solution to offer, but all in all, I would prefer to use DVI.

Has anyone had similar problems with their displays?

How do I solve it? Watching movies is quite unpleasant with the green flicker going on in certain hues.
