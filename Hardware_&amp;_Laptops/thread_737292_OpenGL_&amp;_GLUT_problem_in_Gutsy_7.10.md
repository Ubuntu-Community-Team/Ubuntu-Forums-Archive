### Post by tyrell_turing on 2008-03-27
Hi to all,

I have just upgraded to Gutsy (7.10). The upgrade went fine, and I couldn't detect any major problems. However, I went to work on an OpenGL/GLUT application of mine (in C++), and I got some really weird behavior. The windows wouldn't disappear when I closed the program, they left traces all around the screen when I moved them, and other windows would erase them permanently when they were moved over them. I figured it was my program, and spent a lot of time mucking about. However, eventually I started to suspect that it was something else, especially because my program worked just fine before the upgrade.

So, I went and ran glxgears. The animation looked fine (though the FPS was a bit low, e.g. ~700-800 FPS). But (!), the exact same behavior as my own program occurred. That is, when I drag the glxgears window it left traces, other windows wiped it out, and it remained when I closed the program. A screen shot showing the junk I get is attached.

I have also repeated this problem with another very simple GLUT program that worked before. All it does is draw a triangle, here it is:

```

/*
 * triangle.c -- A simple example of OpenGL and GLUT.
 *
 * compile with the following command
 *
 * cc -lGL -lglut -o triangle triangle.c
 */

#include <stdlib.h>
#include <GL/glut.h>



void displayCB(void)    /* function called whenever redisplay needed */
{
  glClear(GL_COLOR_BUFFER_BIT);   /* clear the display */
  glColor3f(1.0, 1.0, 1.0);   /* set current color to white */
  glBegin(GL_POLYGON);      /* draw filled triangle */
  glVertex2i(200,125);      /* specify each vertex of triangle */
  glVertex2i(100,375);
  glVertex2i(300,375);
  glEnd();        /* OpenGL draws the filled triangle */
  glFlush();        /* Complete any pending operations */
}

void keyCB(unsigned char key, int x, int y) /* called on key press */
{
  if( key == 'q' ) exit(0);
}


int main(int argc, char *argv[])
{
  int win;

  glutInit(&argc, argv);    /* initialize GLUT system */
  glutInitDisplayMode(GLUT_RGB);
  glutInitWindowSize(400,500);    /* width=400pixels height=500pixels */
  win = glutCreateWindow("Triangle"); /* create window */

  /* from this point on the current window is win */

  glClearColor(0.0,0.0,0.0,0.0);  /* set background to black */
  gluOrtho2D(0,400,0,500);    /* how object is mapped to window */
  glutDisplayFunc(displayCB);   /* set window's display callback */
  glutKeyboardFunc(keyCB);    /* set window's key callback */

  glutMainLoop();     /* start processing events... */

  /* execution never reaches this point */

  return 0;
}

```

Thus, it seems to be something about OpenGL/GLUT in Gutsy.

I have done a lot of searching in the forums, and I haven't found anyone with my exact problem. I have found other people with the same graphics card as me (Intel 945) who report problems with resolution or direct rendering when they upgraded, but this doesn't seem to be my problem because a) my resolution is fine, b) the output from glxinfo says direct rendering is okay.

Indeed, everywhere I look in my system, everything seems to be fine. I have xserver-xorg-video-intel installed, and Ubuntu automatically set everything up with no complaints. Also, from what I can tell (I'm not terribly knowledgeable) my /etc/X11/xorg.conf file is fine (it is posted below).

Any thoughts? I'm completely stuck... :confused: I would really value any help. :)
