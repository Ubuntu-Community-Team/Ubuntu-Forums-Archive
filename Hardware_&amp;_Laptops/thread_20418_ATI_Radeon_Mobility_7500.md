### Post by HeavyAl on 2005-03-16
Ok, from searching around the forums I have seen this addressed in several fashions but none of them seem to point to a magical catch-all fix. Here is the problem.

I am running Ubuntu (HH) on an IBM T40 with an ATI Radeon Mobility 7500 AGP 4x video card. It is very responsive, good looking, functional, and I seriously want to use it as my main os but I cannot get video acceleration to work which is a rather large problem because I'm an avid gamer and want to be able to play at least NeverWinterNights and UT2004, both of which require acceleration.

I have been over and over the xorg.conf file and there are no magic settings in it which seem to make any real difference. No matter what I've tried, the most I can get out of the x server is 285fps (using glxgears). This is frustrating because if I use my Knoppix Live CD it auto-detects the video card and enables the acceleration automatically - and running glxgears on Knoppix gives me frame rates in excess of 1000fps! But obviously I don't want to use Knoppix - I want to use UBUNTU! 

The only discernable difference I see between the Knoppix set up and the Ubuntu set up is that Knoppix is using XFree86 and Ubuntu (HH at least) defaults to Xorg. I've been away from the Linux scene for quite some time so I don't know when people started switching to Xorg from Xfree (or even what the differences are) but Xorg seems fine other than not working with the acceleration. And again, I'm not even convinced it's Xorg that is the problem.  

To add to this already lengthy message, here are some things I've tried with the xorg.conf:

* changed Driver from "ati" to "radeon"
* added Option "AGPMode" "4"
* added Option "RenderAccel" "on"
* added several other 'Option' tags under the video device section
* added GLCore to the modules loaded when I noticed it wasnt there (incidentally, this actually seems to slow things down
* rebuilt the whole config file from scratch using the dpkg xorg-configure thingie
* a few dozen other things i cant recall at the moment

**So**, can anyone enlighten me as to what might be going on? 

Thanks in advance for the assistance,

HeavyAL


------------------------

UPDATE:

Ok. After downloading about two dozen updates for today, completely ripping out FGLRX, and then rebooting, somehow I have acceleration - my FPS is still only 590 or so which is about half of what I have been able to get on the Knoppix live cd but it's definately better than the frame rates I was getting before and makes Ubuntu a much more useful system to me.

Incidentally, if there are any other experiences with this particular card that users have had in a similar vein, how about posting them here for comparison? I'd be interested in knowing how many people have had this issue.

### Post by HeavyAl on 2006-06-14
I'm not exactly sure how driconf actually fits in to the whole graphic card driver thing but heres what I've done that has brought my FPS up to almost 1800 FPS using glxgears (-iacknowledgethatthistoolisnotabenchmark .. who the heck thought up that option?!?!?)

installed driconf:

$ sudo apt-get install driconf

ran it

$ driconf

checked the 'user hyperz to boost performance

saved and exited because the rest of it didnt mean a lot to me.

went to my xorg.conf and noticed that the section for my video card was using the ati driver so i swapped it out for the radeon driver ala:

Driver "radeon"

then added:

Option "RenderAccel" "true"
Option "AGPMode" "4"
Option "AGPFastWrite" "true"
Option "EnablePageFlip" "true"

Saved, rebooted to make sure everything started fresh and BAM! 1789 FPS!
