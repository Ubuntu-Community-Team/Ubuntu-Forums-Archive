				Hi everyone,
those are the times when I rememeber why I prefer linux: because it surprises you.

I've a centrino duo (core duo) laptop from Packard Bell (V7908) in which the brightness control never worked (through the command line I always got <not_supported>), and I could never shutdown properly (see launchpad bug).

But yesterday I tried anyway the hibernate function, and even if the laptop cannot actually shutdown after saving all the stuff to the disk, it resumes correctly after the manual poweron/off cycle (I can poweroff keeping the powerbutton pressed for 5 or so seconds). And then was my surprise: some penguin god whispered me to give the brightness controls a try, and so I did. The thing worked, after it never (I mean: NEVER) worked. Without any divine suggestion, I gave a try to the poweroff function too: it worked.

So my question is: how can I spot the difference between a normal ubuntu loading process and a resume from hibernation process? Are there any logs, modules list, options hidden in some etcvarlogusrbin folder?

Thanks for reading, sorry for the messy language: I'm italian and a bit excited about the miracle.
