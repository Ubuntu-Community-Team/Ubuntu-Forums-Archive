### Post by vanadium on 2007-09-08
An ntfs partition can only be read under Linux. If you try to mount a type ntfs with rw, obviously the option is invalid.

Today, it has become possible to mount an ntfs volume with write permissions. However, you need to install the ntfs-3g driver. It is now stable to the extent that it can be recommended.

If it is an external drive, it should not be in /etc/fstab. I see an entry mounted  /media/sdb1 and /media/sdb2 on your system. I guess these are invalid, especially as your /dev/sdb1 is an ntfs volume while the mount point /mount/sdb1 is expected to mount a FAT volume. Comment these out (= put a # in front of the line). To be on the safe side here, I recommend you to turn your computer of, remove the external hard drive, boot up again and then plug the USB back in. It should mount automatically, *read only* because it is an ntfs. Install ntfs-3g if you want write support on this volume. There is a HOWTO on this forum.
