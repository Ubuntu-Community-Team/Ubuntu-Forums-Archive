### Post by Max Luebbe on 2007-08-15
I'll favor quality over quantity.

On both my desktop and laptop that run linux I've got a gig in each, and thats never really left me unsatisfied in anyway with regards to performance. 

I'm assuming from the problem statement, that you have gotten some more ram for your box and are debating whether or not to keep the old ram thats slower in the system?

I'd just use the faster ram and get the performance boost from that - watch your system monitor once in a while when you think youve got the system loaded pretty well and check your resource utilization. If you're using up all the ram and using a decent amount of virtual memory, then stick that extra slower 256 back in.
This will slow your system down, but not as much as using the hard disk for vm.

If that happens, id start putting aside some cash for some more fast ram while using that interim solution.

My .02

### Post by vinnn on 2007-08-15
Personally I usually go for the best quality & speed over quantity.
I recently built my new PC for running Fedora, decided to go with 1GB DDR3 PC3-12800 rather than 4GB DDR2 PC6400.
You just can't beat snappy lagless system response even when your system is under load.

On the other hand if you're a tinkerer and like running multiple virtual machines under Xen, VirtualBox, VMWare etc. you'll want as much RAM as possible.
