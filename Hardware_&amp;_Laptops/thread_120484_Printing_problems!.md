### Post by danko on 2006-01-30
Memory overflow on IIIp is driver's fault. This is a known problem (You can check this on linuxprinting.org site). The only way to resolve it I'm aware of is to add additional memory modules to the printer. I bought memory modules for my printer very cheap on eBay. Now it works great.

I have two printers (the second one is HP5652 usb printer) coexisting happily on my server (Running Debian Sarge, but there should be no difference compared with Ubuntu).
