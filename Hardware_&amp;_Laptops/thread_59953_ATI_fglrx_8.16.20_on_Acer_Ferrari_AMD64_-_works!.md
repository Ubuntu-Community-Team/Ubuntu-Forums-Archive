### Post by Archimago on 2005-08-25
Hey guys, finally got the Ferrari ATI X700 3D accel to work.  CHeck out the excellent page for general info on this laptop installation:
[http://pages.cpsc.ucalgary.ca/~weckerl/ferrari_ubuntu_64.html](http://pages.cpsc.ucalgary.ca/~weckerl/ferrari_ubuntu_64.html)

I recompiled linux-2.6.12.5 as suggested.  Got the APIC speed problem fixed, and DMA working.  Did not want to use the Breezy repositories since ran into trouble and didn't want to upgrade libraries and such for fglrx.  Also tried the Breezy Colony 3 update but Linux lost track of my drives and refused to boot!

So, using essentially just an up-to-date Hoary 5.04, this is what I did to get the newest ATI drivers working...  glxgears now runs at 3700+ fps on full resolution 1680x1050 screen.  I'm a newbie so I'm sure there are more efficient ways to get this running!  Seems to work fine so far.  Good luck...

Archimago

--- instructions (mostly from memory with help from the web) ----
- Go to Synaptic and remove unnecessary packages: 
	xorg-driver-fglrx
	linux-restricted-modules (unless has Atheros wifi for Madwifi?)

- Download from ATI web site: ati-driver-installer-8.16.20-x86_64.run
- Now get out of X:
	sudo /etc/init.d/gdm stop
- cd into the folder where ati-driver-installer-8.16.20-x86_64.run is located
- chmod +x ati-driver-installer-8.16.20-x86_64.run
- ./ati-driver-installer-8.16.20-x86_64.run
- Select custom package and make for Ubuntu 5.10 (Breezy which supports kernel 2.6.12)
- This fails, so in terminal, do:
	perl -pi -e 's|asm(/ioctl32.h)|linux\1|' /lib/modules/fglrx/build_mod/firegl_public.c
- Now:
	cd /lib/modules/fglrx/build_mod
	./make.sh
	cd ..
	ls
- Should now see fglrx.ko
- copy fglrx.ko into modules directory... This should do...
	./make_install.sh
- Should see fglrx.ko module in directory /lib/modules/`uname -r`/misc (if not can probably also cp fglrx.ko /lib/modules/`uname -r`/misc)

- Make sure to change /etc/X11/xorg.conf, here's mine:

- sudo modprobe -r fglrx - see if it loads (should!)
