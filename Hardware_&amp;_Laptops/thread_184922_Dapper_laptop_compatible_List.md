### Post by benoit_breizh on 2006-06-01
There is a video problem with "Acer Travelmate 4100", the screen is black when X starts.
It seems that X try to use the auxiliary VGA port.
To solve it, you have to add two lines in the xorg.conf file :

Section "Device"
	[INDENT]Identifier  "ATI Technologies, Inc. Radeon Mobility X700 (RV410 PCIE)"
	Driver      "ati"
	[B]Option	    "CTR2Position" "clone"
	Option	    "MonitorLayout" "LVDS,CTR"[/B]
	BusID       "PCI:1:0:0"[/INDENT]
EndSection

### Post by HankB on 2006-06-02
I have to add my HP Limited Edition L2000 to the list. It is an AMD64 system and I just upgraded to the 64 bit version of Dapper from Breezy.

I did have to do the following:

1) download the 0.14.4 version of the synaptics driver because 0.14.3 has bugs for 64 bit systems. (See [http://ubuntuforums.org/showthread.php?t=186597](http://ubuntuforums.org/showthread.php?t=186597) for more details.)

2) download and install ndiswrapper 1.16 and install it. Actually I had previously used version 1.5 with Breezy since that seemed to be the first one that worked with the on board Broadcom chip (Broadcom Corporation BCM4318 [AirForce One 54g]). I wasn't sure which one came with Dapper so I just built and installed as I had before. It is a minor inconvenience as I will probably have to re-install the module each time the kernel is upgraded. (That's what I had to do with Breezy since each kernel upgrade would install an older non-working version of ndiswrapper kernel module.)

I did try the fwcutter and  bcm43xx module, but it just caused my system to hang on boot until I deleted the files that fwcutter produced.

The memory card reader now seems to work fully with SD cards! Yay! :p 

I haven't tried the modem or IEEE 1394 ports. [edit]I'm capturing from a CV camcorder using ieee 1394 right now and it seems to work.[/edit]

Nor have I bothered with the ATI drivers. Yet.

### Post by bcherry on 2006-06-07
Dell Inspiron 700m, works beautiful.  Dapper works fully out of the box (not really a box i guess, more out of the wonderful iso download), far better than any other distro I've tried (including next-best Suse 10.1).  Even the multimedia fn-keys work as you'd expect!  The only non-functional hardware is the SD card slot, which should work with linux kernel 2.6.17 rc1, as well as the final 2.6.17 whenever it comes out.  I'm hoping that will be default in Edgy, so here's crossing my fingers.  That problem is probably the most widespread hardware problem in laptops these days across most distrobutions, but within a short time it should be good to go.  
To get dapper working on a 700m w/ XGL and all codecs, etc:
1) Download or ShipIt the latest dapper cd
2) Install it (duh)
3) Download [EasyUbuntu]("http://easyubuntu.freecontrib.org/")
4) Run it (duh), don't check ati or nvidia graphics, but modify repo list
5) Fix video resolution (requires adding "universe" so might as well wait for easyubuntu to change the repos): "$> sudo apt-get install 915resolution"
 (remarkably the newest version of that requires no additional editing, just reboot!)
6) follow [this guide]("http://www.ubuntuforums.org/showthread.php?t=145068") to install XGL
7) Bask in the glory of a 700m running Dapper drake and go nuts
8) ?
9) Profit

That should have you good to go, add other repos, install other apps to you heart's content.

I'd also like to add that I have (briefly) tried the live cd on a Gateway M360, with good success.  Seems everything should work about as well as on the 700m.  Actually if you followed the same guide i just wrote, you should be set (uses intel graphics in the same family).

and in closing, Ubuntu rocks!
