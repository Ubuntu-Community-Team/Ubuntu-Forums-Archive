If we want this problem with the mouse not being recognised after login without unplugging it and plugging it back in fixing, then we need to be posting here:

[https://bugs.launchpad.net/ubuntu/+source/linux-source-2.6.17/+bug/74069](https://bugs.launchpad.net/ubuntu/+source/linux-source-2.6.17/+bug/74069)

The current stance on it is 

> 
Seems not to affect a lot of people and not disturbing too much the users (fixable by unpluging/pluging it again) --> priority low


The more people that report the bug, the more likely it is to be fixed sooner rather than later.

You need to include:

1. Please include the output of the command "uname -a" in your next response. It should be one, long line of text which includes the exact kernel version you're running, as well as the CPU architecture.
2. Please run the command "dmesg > dmesg.log" and attach the resulting file "dmesg.log" to this bug report.
3. Please run the command "lspci -vvnn > lspci-vvnn.log" and attach the resulting file "lspci-vvnn.log" to this bug report.
