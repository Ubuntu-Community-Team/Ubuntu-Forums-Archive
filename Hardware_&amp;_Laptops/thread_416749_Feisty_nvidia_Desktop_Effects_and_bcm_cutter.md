### Post by deeptingler on 2007-04-21
Hi All, not sure if this is a prob at the moment but posting it for you guys to test out.

I did a clean install of feisty this morning and as I use an internal broadcom based PCI wireless adapter on the main PC, I then downloaded BCM43-cutter via synaptic.  This, in turn updated feisty ready to connect via wifi connection.  I setup the network name and password for logging in and all ok (even after a reboot).  Loaded firefox and browsed the internet for a while.

The change came when I updated to the nvidia restricted drivers and enabled the desktop effects.  I found that whenever I tried to connect via wireless, it took ages to connect (watching the icons) and would no longer pull anything off the 'net.  Even updates were failing.

I disabled desktop effects and also disabled the nvidia restricted driver and did a reboot.  Tried to connect to wifi network after this and connected within seconds and working at full speed.

I would love to say this was just a glitch but managed to repeat the process so not sure if any of you guys (and girls) want to test this out incase you can replicate it also.

Sorry, but I have not fed this back as a possible bug as not sure how to do this yet, but any feedback on this, either to myself or to the developers would be appreciated, especially if you manage to confirm what I describe on your machine also.
