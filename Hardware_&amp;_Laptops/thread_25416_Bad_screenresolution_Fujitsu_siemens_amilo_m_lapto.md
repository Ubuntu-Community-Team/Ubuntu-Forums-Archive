### Post by heimo on 2005-04-10
Couple tricks.

if xresprobe is not installed, install it first:
```
sudo apt-get install xresprobe
```
then you can run in X:
```
sudo xresprobe siliconmotion
```
Check the frequency limits and put them to your xorg.conf. I have 19" CRT and in my xorg.conf:
```
Section "Monitor"
        Identifier      "Generic Monitor"
        Option          "DPMS"
        HorizSync       31-101
        VertRefresh     50-160
EndSection

```

OR reconfigure xorg using:

```
sudo dpkg-reconfigure xserver-xorg
```

Problem seems to be that your HorizSync upper limit is too low. If you can't get those limits autodetected (my xresprobe is not working for some reason), try increasing it little by little and manually try to run X by typing 'startx' in console.

### Post by KurtReidar on 2007-04-15
Simply change the monitor section of your xorg.conf file
add an extra "1280x800" on the last sectionline. It works for me. Have an amilo 1425. (with widescreen)

Section "Screen"
	Identifier	"Default Screen"
	Device		"ATI Technologies Inc RV350 [Mobility Radeon 9600 M10]"
	Monitor		"Generic Monitor"
	DefaultDepth	24
	SubSection "Display"
		Depth		1
		Modes		"1024x768" "800x600" "640x480"
	EndSubSection
	SubSection "Display"
		Depth		4
		Modes		"1024x768" "800x600" "640x480"
	EndSubSection
	SubSection "Display"
		Depth		8
		Modes		"1024x768" "800x600" "640x480"
	EndSubSection
	SubSection "Display"
		Depth		15
		Modes		"1024x768" "800x600" "640x480"
	EndSubSection
	SubSection "Display"
		Depth		16
		Modes		"1024x768" "800x600" "640x480"
	EndSubSection
	SubSection "Display"
		Depth		24
		Modes		"1280x800" "1024x768" "800x600" "640x480"
	EndSubSection
EndSection
