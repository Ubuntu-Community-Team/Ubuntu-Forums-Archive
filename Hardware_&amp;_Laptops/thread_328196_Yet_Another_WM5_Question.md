It's the semi-noob once again trying to bring myself closer to a long time want to be Windows free.  I'm not a hater just not a fan of dealing with... But I digress

I have a T-Mo MDA, HTC Wizard, and am attempting to sync it with Evolution.  I've tried using the many SyncE How-To's available here and on the web and I keep getting an error that states:

```
pstatus: Unable to initialize RAPI: An unspecified failure has occurred
```

According to the SyncE site it may be part of a kernel bug. There FAQ is listed here at [http://synce.sourceforge.net/synce/qa.php](http://synce.sourceforge.net/synce/qa.php) I tried to use their debug instructions but did not see it.  I have the all.log file they say to create saved if any are interested, I may have overlooked it.  Also the other suggestions of :

    * you are not using an external USB hub
    * you have installed the patch for the "we're stuffed" bug
    * you supplied a password to dccm if your device is password-protected
    * dccm was started before you ran synce-serial-start
    * that the PPP connection is successful (look in your system logs)
    * no firewall configuration prevents the PDA from connecting to dccm
    * you run dccm and the tools as the same user

I am using 6.10 with the latest stable kernel for those that are curious of that.
