### Post by GSF1200S on 2007-05-16
I think I may have finally figured this one out...

What causes a flash? I guarantee all of you had the flash where the black flash appeared to be left to right.

Vertical refresh rate!

In order to fix both the flashing and the freezing, try the following.

First off, this is for NVIDIA users.. ATI users should be able to make this work too, assuming they know there monitors refresh rate.

If you use NVIDIA drivers, make sure to download the latest 9755 driver (or legacy if you have an older card). Despite the fact the 9755 has been out a while, they just released a new version May 7th.. You can only tell by looking at the release date; the version stays the same. This is more than likely why darkmaster stopped having the freezing problem: he used Envy to install the NEWEST drivers, while weve been using ones that we dowloaded 2 months ago, thinking it was up to date, when in fact there have been updates to this driver since then.

Now that the driver is installed, lets see if this helps with the black flashes. I noticed yesterday that if I changed the refresh rate in the Nvidia control panel to 60hz (which is my comps PARTICULAR refresh rate, yours may be different), the flashing would stop. After doing this, I tried to save the changes to the Xorg.conf. For some reason the changes wouldnt stay. So, I changed the refresh rate to 60Hz, opted to save to the xorg.conf, and checked the "show preview" button. It then popped up with a example of the Xorg, with my refresh rate listed at 61Hz. I then typed:

sudo kate /etc/X11/xorg.conf

to open the xorg manually, and manually changed the refresh rate to 61Hz, then saved it. I restarted X, and I havent had a flash YET... Ill report back in a few days if I havent had a problem, and if I do have one, Ill report back with that.

Please let this thread know if this solution helps... Ive tirelessly investigated this trying to find an answer, because Ive gotten so tired of flashes/freezing... If this doesnt fix it for you, Ill keep looking!

Good Luck! :)

**EDIT** If you use Gnome, the command to manually open xorg.conf will be:
sudo gedit /etc/X11/xorg.conf

Also, make sure you restart X just to make sure these changes have taken effect. Of course, you could press "Apply" on the Nvidia panel if you dont want to restart right away.

### Post by BoardDWorld on 2007-12-01
The following fixed the screen flashes and system freezes for 30 straight days without problems until I installed the current 169.07 driver using ENVY.
The problem, for those who don't know, is caused by the nVidia PowerMizer (in the nVidia Driver) changing the clock speeds from 2D to 3D. People have been blaming Compiz but Compiz just brings out this flaw in the driver because of the frequent change between the 2D and 3D clocks. The following fix locks your video card to the single 3D clock speed.


The fix below is for the nVidia driver from Ubuntu only(Synaptic or clicking the Restricted Drivers check box etc). 

Open nvidia-kernel-nkc using Terminal with the following command:

```
sudo gedit /etc/modprobe.d/nvidia-kernel-nkc
```

Copy & Paste the following at the bottom in a new line:

```
options nvidia NVreg_RegistryDwords="PerfLevelSrc=0x2222"
```

Save the file and restart your PC.


Source:
[http://www.nvnews.net/vbulletin/showthread.php?t=96673&page=2](http://www.nvnews.net/vbulletin/showthread.php?t=96673&page=2)

For those using ENVY to install their drivers or are downloading them from nVidia and installing them manually (there's no point doing it manually, use ENVY) go to the next page.

Notes: As said this fix locks the video card to the 3D clock, for some this could cause heat problems and greatly reduce battery life. For myself I don't have heat problems at all. It sits at 58degrees in both the 2D without this fix and at the constant 3D clock speed with this fix. Battery life, when I use it on battery to that extent, is reduced by 10-20 minutes which still gives me over 1 hour. This makes sense as even though it's at the 3D clock it really only starts causing heat/using power when it gets used/stressed, eg playing games. No negative reports have been made on the nvniews forum and I keep pretty frequent tabs on it.

### Post by lamegaptop on 2008-02-08
Hello all

I was able to fix this with the advice located here:

[html]http://www.nvnews.net/vbulletin/showthread.php?t=96673&page=3[/html]Basically the flashing is caused by the dynamic clock changes of the GPU. Turning that off cures the flashing but disables the dynamic control which increases heat, reduces battery life, etc.

Edit: Setting the option -

Option   "Coolbits"   "1"

in the screen section of my /etc/X11/xorg.conf 

turns on the overclocking screen in nvidia-settings. One can use this screen to throttle the settings of the GPU and memory clocks to conserve power whilst on battery. No reboot or restart of X required.

**WARNING:** be careful not to use this to exceed the hardware default settings of your laptop video card!! You will damage components.

Moving the sliders to the left throttles the clock settings. Use the "reset to hardware defaults" button to return the clocks to full whilst on AC or for gaming, etc.

Life is good.

### Post by lamegaptop on 2008-02-09
Adding this line -

```
options nvidia_new NVreg_Mobile=1 NVreg_RegistryDwords="PerfLevelSrc=0x2222"
```to /etc/modprobe.d/nvidia-kernel-nkc

Locks the clocks on high then the overclocking section  in nvidia-settings can be used to throttle back, if you wish,  to conserve battery or whatever.

Have fun
