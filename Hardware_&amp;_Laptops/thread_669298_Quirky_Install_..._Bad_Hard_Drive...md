### Post by Quilzas on 2008-01-16
I have an old Dell Inspiron laptop that I got for free.  I want to say 1100, but I'm no where near it right now.  

I dropped Ubuntu 7.10 on it.  But I had a lot of issues where the install would either crash at certain random points (near the end) or when I would go to boot for the first past post install, it wouldn't.

I ran a utility disk to check the harddrive - no problems.

I tried installing several times - Ubuntu and Xubuntu.  Always the same problem.  A Live disk would fail quickly.  An Alt disk would usually get through most of the install.

Eventually, I found a solution.  If, when making the partitions (manually) I set them to start at the end of the hard drive (rather than the beginning) the system installed perfectly and the system has been running fine without a hitch since (this was a few months ago).

Just occurred to me to post about it.  Could the harddrive indeed be bad even though a disk checked found nothing?  It's not worth it for me to replace the harddrive.  I just got a new laptop, and this one may be given to a friend, so I want to warn them about this experience of mine.
