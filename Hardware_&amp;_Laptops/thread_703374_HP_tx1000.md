### Post by hojthojt on 2008-03-04
**How I installed Ubuntu Gutsy 7.10 64-bit on HP Pavilion tx1151ea**

This installation story was collected when I installed my machine 2008-02-29. Most of the info in here was collected by reading the tx1000 thread in the Ubuntu forum, but since it now contains 600+ posts, it took me several hours just to read it through. This post is intended to help people to get an up-to-date install guide on the tx1000 series.

I have the F.1D BIOS installed (aka sp38257.exe by HP, flashed from Vista)

The first thing you need to consider is whether to use the 32-bit or 64 bit version
of Ubuntu. I decided to go for 64-bit, since the tx1151ea is a 64-bit machine, it
just feels better :-). There might be applications not available in 64-bit version,
but then I simply have to use another one that is.

**Boot**
Download Ubuntu 7.10 LiveCD for AMD64

[http://www.ubuntu.com/getubuntu/download]("http://www.ubuntu.com/getubuntu/download")

Burn CD

Boot with CD

During bootup select F6 and enter the following extra kernel options: 

```
noapic irqpoll noirqdebug
```

**Install**
When booted, run the install by double click on the install icon on the desktop.

Answered some questions:
System language: xxxx
Location: xxxx
Keyboard: xxxx
Guided partitioning of SCSI1 (0,0,0) (sda)
Part #1 as ext3, part #5 as swap
user: xxxx
computer name: xxxx

Once the install is finished, the machine needs to be rebooted (remember to remove the install CD). I would recommend to have a wired Internet connection plugged in during this reboot (will enable Ubuntu to update itself over the Internet).

Note that you need to halt the boot this time as well and add the special kernal options:
```
noapic irqpoll noirqdebug
```

When the system is up again after the reboot, the Synaptics update manager will inform you that updates are available (if you have a wired internet connection that is). Accept the updates.

Update the kernel boot options so you do not need to do it manually at each restart. 
This is done by editing /boot/grub/menu.lst (as root):
```
sudo gedit /boot/grub/menu.lst 
```

Add the kernal options "noapic irqpoll noirqdebug" (somewhere in the end of the file where the GRUB boot menu options are listed). The line to edit should look something like this after the edit:

```
kernel /boot/vmlinuz-2.6.22-14-generic root=UUID=<some id> ro noapic irqpoll noirqdebug splash
```

**Wireless**
My machine is equipped with the Broadcom BCM4328 802.11a/b/g/n WLAN chip.

I followed the following instruction:
[URL="http://ubuntuforums.org/showpost.php?p=3491929&postcount=257"]
http://ubuntuforums.org/showpost.php?p=3491929&postcount=257[/URL]

It worked well except for one thing. First I tried to use the latest driver for the tx1151ea 
Broadcom Wireless chip downloaded from [www.hp.se](www.hp.se) (called sp37746.exe and unpacked with "wine"), but that one did not work. Instead I used another driver that I found referenced in some post somewhere:

```
wget http://ftp.us.dell.com/network/R151517.EXE
unzip -a R151517.EXE
```

After unzipping, you can find the bcmwl5.inf file in the DRIVER directory that is created by the unzip.

**Sound**
I followed this instruction
[http://ubuntuforums.org/showpost.php?p=3475485&postcount=235]("http://ubuntuforums.org/showpost.php?p=3475485&postcount=235")

Tip:
You might need to use alsamixer (start it by double click on the volume control icon)
to make sure no devices are muted.

It worked with the following exceptions:
-The mute button located just above the left hand side of the keybord works,
but the color of the led does not change color as it does in Vista.
-The built in microphone (just above the screen) does not work. I'm suspecting I have
the wrong settings in alsamixer, but I have not found any settings that make the front mic record any sound.

Note: The auto sensing of the two headphone jacks works fine as well.

Tip from mikedep333: 
I got the following comment from mikedep333:
"There is a much easier way to get sound working. Install linux-backports-modules (which I think is under main, not the backports repo.) It then just works, and you do not have to fiddle around and reinstall stuff from time to time."

I have not tried it myself (don't fix it if it ain't broken), but for anyone  making a fresh install, it could be worth a try. 

**Video**
2D Graphics works out of the box, but to get 3D graphics, you need to install NVIDIAS drivers.

For installing the display drivers use the menu: "System / Administration / Restricted Drivers Manager". Click on the Nvidia accelerated graphics driver check box. The drivers should be installed automatically. I got NVIDIA driver version: 100.14.19

Note: You might want to backup your xorg.conf before installing NVIDIA drivers, they have been known to corrupt xorg.conf, but it worked out fine for me.

****************************
Tip:
I have not tried this myself yet, but if you have trouble to get the external VGA port to work using the standard System -> Administration -> Screens and Graphics, 
then try  nvidia-settings.
```
sudo apt-get install nvidia-settings
```

After installing you can launch the application by typing "nvidia-settings" in the shell.
Note that you can see the temperature of the GPU in this tool. My GPU temp is above 80 C even though I'm just sitting writing in a text editor.....

**Touchscreen**
I used this instruction:
[http://ubuntuforums.org/showpost.php?p=4197376&postcount=542]("http://ubuntuforums.org/showpost.php?p=4197376&postcount=542")

With the 64-bit driver and calibration tool for the 2.6 kernel from:
[http://210.64.17.162/web20/TouckDriver/linuxDriver.htm]("http://210.64.17.162/web20/TouckDriver/linuxDriver.htm")

Note: When I edited the xorg.conf file according to the instruction, I found entries
for a WACOM tablet in there. Even though they should not need to be there I left them
since they seem to be disabled anyway.

After the installation of the touch screen driver described above, you need to calibrate it.

Start the calibration with:```

cd <the directory where you unpacked the TouchKit package>
sudo ./TouchKit

```
Choose the "Tool" tab and then click the "25Ptz Linz" button and calibrate away. I get best
results with the 25Ptz calibration (the touch screen actually works much better in Ubuntu,
then it ever did in Vista).

The only problem is the screen rotation, more on this later.

*********************************************
Tip B:
xvkbd: It is an on-screen keyboard that can be usefull when using the tx1000 in tablet mode.
```
sudo apt-get install xvkbd
xvkbd
```

Tip C:
You could also try (I have not):
[http://www.risujin.org/cellwriter/]("http://www.risujin.org/cellwriter/")

Tip D:
Taking handwritten notes in table mode? Try Xournal.
```
sudo apt-get install xournal
xournal
```

**Rotating screen**
I used this instruction:

[http://ubuntuforums.org/showpost.php?p=4241341&postcount=579]("http://ubuntuforums.org/showpost.php?p=4241341&postcount=579")

Note that you need to reboot the computer or restart X (with Ctrl+Alt+Backspace) before you test the rotation (or if you know of another way of getting the system to re-read xorg.conf I would be glad to know how).

Unfortunately, it does not rotate the touch screen coordinates, which makes it pretty useless.

I have an idea to solve this, but havn't solved it fully yet. What I did was to calibrate 4 different touchscreen calibration files, one for each screen rotation. Between each calibration I copied the calibration data into separate files:
```
<calibrate>
sudo cp /etc/egalax.cal /etc/egalax_normal.cal
<rotate screen with rotate.sh>
<calibrate>
sudo cp /etc/egalax.cal /etc/egalax_left.cal
<rotate screen with rotate.sh>
<calibrate>
sudo cp /etc/egalax.cal /etc/egalax_inverted.cal
<rotate screen with rotate.sh>
<calibrate>
sudo cp /etc/egalax.cal /etc/egalax_right.cal
```

Then the idea was to change calibration file used together with the screen rotation script. The problem is to get the touch screen driver to re-read the calibration file.

The calibration is normally saved in /etc/egalax.cal and is referenced from /etc/X11/xorg.conf 
To change the calibration settings when the screen is rotated, I need a way to get the xorg.conf file to be re-read or to restart the Xserver when I rotate the screen. Or even better, just get the driver to use another calibration file. Any ideas are welcome.

My workaround sofar is to have four different xorg.conf files which references different egalax.cal files, then I change the xorg.conf that match my screen rotation and restart X with <Ctrl+Alt+Backspace>.

**Webcam**
Works ootb except that the built-in microphone doesn't work. See the section about sound.

I tested the webcam with the tool called "cheese" that lets you record video and photos 
with the webcam.

**Express slot TV card (DVB-T)**
Works ootb using kaffeine application, but of course you need tune it and have a good antenna signal.

**Suspend and Hibernate**
It feels a bit scary, but I seems to work most of the time. The scary part is that I get colored vertical lines on the screen when waking it up after suspend or hibernate. I hope it doesn't do any damage on the LCD. You also need to be patient, sometimes it takes a very long time before it restarts a and regain control over the screen output. When I first encountered this I thought that the machine had hang and turned it of hard, but now I have learnt that I just need to wait.

Now I have actually dared to changed my power management settings  (System/Preferences/Power Management) to suspend when I close the lid and to hibernate when the battery is critically low.

**Media card reader**
Worked ootb, I have tested xD and SD cards so far.

**Mini Remote Control**
The remote works ootb by triggering the media buttons on your tx1000 cover.

If you assign the physical buttons to the apps, the remote should activate them accordingly as if you actually pressed those buttons on your laptop.

************************
Tip:
Try the remote out with the keyboard shortcuts application.

**tx1000 Special buttons**
Note: Most buttons seems to be mapped correctly ootb. Exceptions: Screen rotate and cog-wheel buttons, these do not seem to generate any keycode/event at all.

The fn-buttons on the keyboard (print, www, suspend, change screen brightness, media control) and the media buttons on the side of the screen works. Lid switch works as well. The DVD button starts the music player.

One oddity is that when I turn on the synaptics touch pad after having it turned off, the Ubuntu Help Center is started (i.e. the same as pressing fn+F1).

The button mapping can be checked and changed by "System/Preferences/Keyboard Shortcuts".

**Bluetooth**
First I tried to connect to my Sony Ericsson K610i with bluetooth. The tx1000 finds it when browsing
for BlueTooth devices, but when connecting to it I got the following error message:

"OBEX [mac address] is not a valid location"

Googled and found a tip to install the following:

```
sudo apt-get install gnome-vfs-obexftp
```

Now it works to connect and browse files on the phone :-)
