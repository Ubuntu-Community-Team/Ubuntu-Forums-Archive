### Post by ricrac on 2007-05-16
I see you fixed it, but did you see why it was broken?  

The X errors were very explicit in this case:

Excerpt from previous post...
I get the following in logs if i omit the BusID for ati device.

(II) ATI: Candidate "Device" section "ATI Technologies Inc Radeon RV100 QY [Radeon 7000/VE]".
(WW) RADEON: No matching Device section for instance (BusID PCI:0:8:0) found...

Excerpt from X config from previous post... 
Section "Device"
   Identifier    "ATI Technologies Inc Radeon RV100 QY [Radeon 7000/VE], screen 1"
   Driver        "ati"
   BusID        "PCI:0:8:0"
   Screen        1
EndSection...


The two lines to compare:
(II) ATI: Candidate "Device" section "ATI Technologies Inc Radeon RV100 QY [Radeon 7000/VE]".
   Identifier    "ATI Technologies Inc Radeon RV100 QY [Radeon 7000/VE], screen 1"

The lines and error, parsed to read easier:
"ATI Technologies Inc Radeon RV100 QY [Radeon 7000/VE]"
"ATI Technologies Inc Radeon RV100 QY [Radeon 7000/VE], screen 1"
RADEON: No matching Device section found



The problem was/is the comma in the "Device" section "Identifier" field. 
It appears to truncate on one side of the compare only, a fairly common programming error. 
The parser should reject the identifier and output some type of  "Invalid string input error" well before the compare is attempted.  


Although the error did state "No matching Device section found", I wish the general error at least gave notice that this was a typo error and (usually) all else is irrelevant until the config file is typographically "correct".  I don't want the error dumbed down, just get to the point quicker, any "No matching ****** section" errors are typo/bad/corrupt conf files. 
Maybe something like starting the error log entry with..

"Bad X Conf File Error - Check for matching quotes, matching Identifier spellings, and correct syntax."
