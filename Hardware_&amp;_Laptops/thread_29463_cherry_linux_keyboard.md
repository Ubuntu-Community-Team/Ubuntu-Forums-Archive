i've often read of compile errors concerning the cherry linux keyboard driver. cherry offers the source and only delivers a rpm for suse. after a little work i could compile a debian package on ubuntu which works fine. requires kde. [http://www.liedler.at/dl/keyman-bin.deb.tar.gz](http://www.liedler.at/dl/keyman-bin.deb.tar.gz)

The deb file is no longer supported. Look here for a simple way to activate the keyboard:
[http://web.liedler.at/index.php?option=com_content&task=view&id=157&Itemid=49](http://web.liedler.at/index.php?option=com_content&task=view&id=157&Itemid=49)
