### Post by growse on 2007-07-07
I'm currently using Ubuntu Feisty on my Thinkpad T42 (running Beryl, rather nicely) and have one or two annoyances relating to the VGA out port. I run windows within VMWare Workstation 6 for my office use, and occasionally that calls on me needing to plug my laptop into a projector.

My X config happily mirrors my laptop display out to the VGA port, but only if something was plugged into that port when X starts. This means that if I boot up my laptop with nothing plugged in, and I'm working within windows, if I want to plug into a projector I have to:

1) Suspend windows in VMWare
2) Close VMWare and log out of my X session
3) Plug in projector
4) Restart X
5) Load VMWare and resume windows.

I'm not entirely sure why this is the case, but it's certainly very annoying. Ideally, I'd like to always be able to plug the projector in and have it just work.

I have a second, not-so-annoying-but-curiously-related issue, which is that if I ever turn the laptop on whilst it is plugged into a docking station with a monitor attached, it assumes that that monitor is the primary and doesn't display anything on the laptop panel. I presume this is more of a hardware thing, but the fact that X seems to use the plugged in monitor rather than the laptop monitor as the primary is a bit weird.

If anyone knows of any way to overcome either of these problems, that'd be great. :)
