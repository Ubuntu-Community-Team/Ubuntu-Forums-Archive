### Post by neurosion on 2005-07-12
I was having a persistent problem where X would start up to a hard-locked black screen with no available terminals when I cold-booted using the NVIDIA binary driver.  Strangely, everything worked fine if I started up any other operating system and performed a soft reboot.

It turns out that the problem could be traced to the cutting-edge BIOS on my Asus A8V.  I was using BIOS 1013 until I found out that several people, even ATI users, solved this same problem by backing out to BIOS 1009.  Since I've seen complaints about this problem all over the place, it wouldn't surprise me if other Asus motherboards had a similar problem.

If you've got the problem I describe with an Asus motherboard and a fairly recent BIOS, you might want to try backing out to an earlier rev.  Of course, YMMV; you may want to Google around a bit before doing anything drastic.
