### Post by anujseth on 2007-04-16
i 've been using ubuntu since dapper 6.06 ( i switched from fedora core, and i'm glad i did). all the while i have been using a Dell Latitude C600 laptop machine. this old trusted p3 machine of mine is still super awesome. anyway i never had any hardware detection problems in ubuntu 6.06 or 6.10. however when i installed the feisty fawn testing it set my display resolution to 800x600 and refused to give me an option to make it better.  the problem was solved by editing the xorg.conf file. below are the details. 

the xorg.conf is located in /etc/X11 i.e /etc/X11/xorg.conf

anyway you must backup the file before you start hacking away. as another option there are a few threads that make you re-configure the gdm setup etc., personally i feel that this gives most people more than they bargained for and just stay away if you're not sure about what you're doing. the other option is to manually edit your xorg.conf file. here's how i did it for the dell c600. do note that you will need to accordingly put in the resolution details scan frequencies etc. for your monitors.

```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf-backup
```
```
sudo gedit /etc/X11/xorg.conf
```

you need to edit the section which says something like this - Section "Monitor" ... scroll down you'll find it. i have pasted below the old( problem giving 800x600 ) and the new( corrected 1024x768 ) files. the changes are highlighted in [COLOR="SeaGreen"]green[/COLOR] (sea green actually :)  )

you need to make 2 changes basically specify the resolution and the refresh / scan rate etc. please be sure of the values you enter. you can check these details from your manufacturers page or a manual etc. after you have made the changes ... restart ... you should see the changes immediately take effect on the log in screen. to change your resolution go to System -> Preferences -> Screen Resolution and you should see the 1024x768 option there.

for most people this will solve the problem. a few may however get into trouble. the commonest one might be a gdm error or something when you restart. simply ignore all reconfiguring options(if you are given any) and go to the character based shell(no GUI X Server) by pressing ctrl + alt + F1. login with your normal password etc. and simply type the following 
```
sudo cp /etc/X11/xorg.conf-backup /etc/X11/xorg.conf
```
this will give you back your old faulty resolution configuration(atleast it works) ... restart ... you're now ready to edit your file again ... with different settings ofcourse :).

### Post by Mumra on 2007-04-25
I solved it - I kick bottom.

What I didn't realise is that even on a live CD, you can play around with the xorg.conf file. So, if you boot up and get a fuzzy display running at 800x600 and can't run the live CD installer (because the install box is too large to fit on an 800x600 screen!) ...

 - Go to a terminal by pressing ctrl+alt+F1
 - type: sudo su
 - type: nano /etc/X11/xorg.conf

 ... and play around with it. You can insert the modeline for 1024x768 given earlier in this thread.

After I got the beast installed, a simple:

 - type: sudo dpkg-reconfigure -phigh xserver-xorg

 ... got me rocking in the full 1400x1050. On my C600, this is the default, and beautiful, resolution.

Hope that helps somebody!
