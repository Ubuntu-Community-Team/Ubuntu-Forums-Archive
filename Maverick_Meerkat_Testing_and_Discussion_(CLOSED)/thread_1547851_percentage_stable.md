### Post by 23meg on 2010-08-07
> **DouglasAdams said:**
> would it be possible to "convert" alpha, beta, rc into, for example, a percentage stable figure, say, for a fairly basic user?

No. Milestone releases are simply slightly stabilized (thanks to a few days of ISO testing) builds of certain sets of packages from the package archive at predetermined dates, which means that it's impossible for you to make a judgment on the relative "safety" (which is actually not a word that goes with pre-release software) of a milestone release without following up closely with the development process. In the "does not crash and burn" sense of "stable", there's no way of ranking milestone releases; Alpha 3 can make you lose data under the exact same setting where Alpha 1 *seemed to* work perfectly.

> **DouglasAdams said:**
> if, for example, i knew that a forthcoming release was currently about 90% as stable as the finished product would be for most users who browsed, twittered, utubed and facebooked then i would be quite tempted to load it onto one of my kids laptops or maybe my fallback machine.

It's not advisable to use pre-release Ubuntu under either of those settings. It keeps changing substantially in ways you cannot predict without close involvement, it doesn't guarantee timely security updates, and there's a far greater likelihood of serious bugs.

### Post by andrewabc on 2010-08-07
Couple reasons to use development versions:

1. Stable versions do not work great for your hardware, or for some reason too much outdated software for your needs.

2. Use dev version to make sure everything works fine on your hardware. This way no surprises on release day when stuff doesn't work. And this happens all the time for every release. So better off figuring out the tweaks/fixes before release day and having to wade through the thousands of posts in tech support when the dev forum has less 'noob' posts.

That said, dev versions can break, explode etc at any time, so if it happens do not complain, just file bug report etc.

As long as you don't store important information on computer that uses dev version and it doesn't matter if you have to format it at anytime, then feel free to use dev version.

If you want 'stable' then use stable release.

There is no proper or safe time to start using dev version. It is different for each person and their needs. For me I am using 9.10 on main desktop because 10.04 didn't have enough features to warrant the >=6 hours to install/tweak. Will upgrade (format/install) 10.10 on RC or release day. The one thing I fail to understand is why quite a few people want a single /home directory for various installs. Most distros have different version of programs who's configuration files are stored in the home directory. 

For instance if you use vlc, the version in use for maverick is far different from the version in use in hardy, this could lead to it not working the way you expect it to if you are using a later version.

If hard drive space is a concern, just create a fairly small /home and then use a single storage partition for storing downloaded programs that is accessible from all the operating systems you have installed.
