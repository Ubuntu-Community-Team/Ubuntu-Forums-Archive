### Post by 23meg on 2010-05-31
In case some of you missed it, Matt Zimmermann (of the Ubuntu Technical Board, and Ubuntu CTO at Canonical) [posted today]("http://mdzlog.alcor.net/2010/05/31/summary-of-development-plans-for-ubuntu-10-10/") about post-UDS summaries written up by leads / representatives of four key teams (Desktop, Server, Foundations, Kernel).


[QUOTE=Matt Zimmermann][list][*] The [Desktop Team plan]("http://theravingrick.blogspot.com/2010/05/track-desktop-and-une-in-maverick.html"), via Rick Spencer, includes both Desktop Edition and Netbook Edition, and covers Software Center, GNOME, application and browser selection (including Chromium), X11 and social networking


[*] The [Server team plan]("http://ubuntuserver.wordpress.com/2010/05/26/ubuntu-server-update-for-maverick-meerkat/"), via Jos Boumans, covers migrating services to Upstart, improvements to the mail stack, cloud and cluster storage systems, cloud deployment and load balancing, Ubuntu Enterprise Cloud, and Java


[*]The [Foundations team plan]("http://oubiwann.blogspot.com/2010/05/ubuntu-foundations-and-maverick-meerkat.html"), via Duncan McGreggor, covers system initialization, btrfs, cleaning house, improvements to the installers, Software Center (also) and Upstart (itself), along with a batch of miscellaneous projects, including compiling Ubuntu for the i686 instruction set


[*]The [Kernel team plan]("https://lists.ubuntu.com/archives/ubuntu-devel/2010-May/030764.html"), via Leann Ogasawara, covers the choice of kernel 2.6.35, the current Ubuntu patch set, the configuration of our various kernel flavours, bug management, and the availability of backports of newer kernels to Ubuntu LTS releases [/list][/QUOTE]

Keep in mind that these summaries are non-exhaustive; there are many other teams and individuals working on [lots of other things]("https://blueprints.launchpad.net/ubuntu/maverick").

### Post by ranch hand on 2010-06-05
They are community developed variations and official members of the Ubuntu "family".

They must meet certain standards to do so and this is only right.  They are free to use the Ubuntu name, logo and repos.

Lubuntu, for instance, has permission to use the name "Lubuntu" but is not, yet, a full member of the "family".  Hopefully this will be changing soon.  It is a small core of folks working their butts off to become another option in the Ubuntu circle.

I think this is a really great thing.  I am not a KDE fan at all, but a lot of folks are so we have Kubuntu, I kind of like Xubuntu, it has my 3rd favorite file manager.  My son uses Ubuntu Studio.

Soon, I hope, we will have Lubuntu with my number 2 file manager (pcmanfm2 is even better than pcmanfm).

All the apps for all these variations, right there in the Ubuntu repo so that you can build your own "flavor" to just suit you.  Is this great or what?

It is up to the devs of those variations to get the OS in shape for distribution and ISO building.  Both Kubuntu and Xubuntu A1 ISOs were tested by the ISO test team.  I only do Ubuntu (Alt, Desktop and Live Session) and I did not look to see how many folks did test them.

If you are interested in these OSs join the ISO testing team, test the buggers.  We are the reason that the ISOs work as well as they do.  More hardware = better ISO.

I am on the wrong OS to give you the links for that.  It is in the Ubutu wiki however.

As I use Ubuntu and Lubuntu, I am not sure where you get the dailies for Kubuntu or Xubuntu.  This may help and may not but everyone should have this link;
