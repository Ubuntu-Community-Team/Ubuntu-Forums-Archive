### Post by Stochastic on 2007-10-08
Well I'd say that the king of all LIVE performance software in Linux is ChucK.  Simply put, this is a programming language that lets you create music and or visuals by coding live in a strongly timed environment.  You've got to be pretty good at coding to use it live though.
Other similar languages exist, I'd check out [http://www.toplap.org](http://www.toplap.org) if you're good at coding.

There's also PD/GEM, which is Max/MSP's little FOSS brother.  It's essentially also a programming language that you can do audio and video with, but this is more designed as as: Build your system, then take it out and perform with it.  You can do some pretty amazing things with this. (everything from signal processing, to midi controlling, to generation of graphics, to audio synthesis, to creating programs that work on cell phones, to large scale interactive art installations, to....) [http://puredata.hurleur.com/index.php](http://puredata.hurleur.com/index.php)

If you're looking for something a little more pre-structured for audio there's 
FreeWheeling (a loop-based manipulation program)
XWAX (a way to control pre-recorded music with turntables) - just like FinalScratch
Hydrogen (a very nice drum machine)
ZynAddSubFX (very powerful midi-controlled virtual synth)
JackRack (a versatile signal processor)
Wired
Jokosher
Aldrin

Those last three I don't use much myself, but they look much more like Reason and Garage Band style apps.
You can see lots at [http://linux-sound.org](http://linux-sound.org)

As for Video I really don't know what to suggest other than PD/GEM for live performance.  There's a new FOSS programming language based on Java (so runs across all platforms) called Processing [http://processing.org](http://processing.org) that is directed specifically towards visuals.  That website has some examples and is worth a browse to see what people have done with it.  But I'm not too sure how useful it would be in a live setting.

Here's a couple other live visual programs I stumbled across:
Fluxus [http://www.pawfal.org/Software/fluxus/](http://www.pawfal.org/Software/fluxus/)
Thingee [http://deprogramming.us/perfs/#thingee](http://deprogramming.us/perfs/#thingee)
all of which were links from the [http://www.toplap.org](http://www.toplap.org) website, so will be fairly coding-heavy apps.

Does this help, or are you looking for something much more specific or obvious that I'm not mentioning?
