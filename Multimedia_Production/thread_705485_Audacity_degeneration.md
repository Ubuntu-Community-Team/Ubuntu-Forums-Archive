### Post by Stochastic on 2008-02-23
Honestly one of the best features of Ubuntu IMHO is the ability of choice.  There are multiple different apps to accomplish the same task - and all are free to use.  If Serpentine suddenly dies (which seems quite odd, but odd things happen) then open up GnomeBaker or K3b and keep on truckin'  If Audacity won't record, Rezound, Ardour, or even GnomeSoundRecorder usually will (unless the issue lies in your sound config).  That being said there's no logical reason for Audacity to do this (though bugs do exist in the software world), so let's try to see what's wrong...

First we need a bit more info from you:  
What audio configuration are you running Audacity through, ALSA, JACK, etc (can be found in Audacity's Preferences section - Ctrl+P)?  
What version of Audacity are you running (can be found in Audacity's Help menu -> About Audacity)?
Have you attempted a reboot (though you shouldn't need to reboot in Linux, sh*t happens)?


I would guess that your issues with movie playback, mp3 'corruption', and rar files are codec issues - which set of codecs are you running?
