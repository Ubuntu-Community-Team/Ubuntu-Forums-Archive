### Post by Iehova on 2011-04-09
Hi all,

So I see that the launcher behaviour is changing to reveal when the mouse hits the left screen edge. This seems eminently sensible to me and I'm really happy that the devs are making these changes. However I think that there is another element of the launcher behaviour which makes it vastly more difficult to use, [described in bug 734946](https://bugs.launchpad.net/unity/+bug/734946), which I'd like to draw your attention to in the hope of generating some progress.

IMHO the current behaviour makes no sense. As soon as my mouse if over the launcher the entire thing expands, pushing half of the icons off the screen. If I want an icon that happens to be in the top half of the launcher then fine, it's right there, but if I want something right at the bottom it's not quite so simple. Both scrolling and click-dragging are a needless waste of time and quite awkward and jarring, especially if you're not using a traditional mouse. Even autoscroll would be unnecessarily slow and painful.

It seems to me that icons should never be pushed off the screen. Unity has a very good concertina effect which was particularly designed so that icons would be viewable even if there were large numbers of them. The sensible behaviour would therefore be, IMHO, if the launcher's default state were that all icons were folded by the same amount. This means that unless you had hundreds of the things, all of them would be recognisable.

On mouseover, the icon that the mouse is over (and perhaps surrounding icons) could unfold with the folding radiating nicely outwards so that at no point do any icons leave the screen. This would be very visually appealing, IMHO, like flipping through a pack of cards, and yet it would also be much more sensible than the current behaviour.

I know it's probably a bit late in the dev cycle for Natty, but does anyone have any thoughts?
