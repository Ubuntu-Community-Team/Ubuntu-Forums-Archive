### Post by mc4man on 2011-03-21
> **Starks said:**
> for the moment at least, cube and wobbly need to be sacrificed to make a stable, usable unity.
Here's the orig. bug concerning the requirement of 'wall'

It was more a matter of protecting a user from losing focus on some windows if they disabled wall, or the default set of unity compiz plugins got unset
(and the user didn't know to re-enable wall

The 'focus' deal is provided by both wall and rotate, but I gather there is no easy way to write a requirement as 'wall or rotate'

An alternative would be to make the requirement 'largedesktop' - this would translate to either wall or cube. In that case those that choose wall would maintain focus, those that choose 'cube' would not unless they also enabled 'rotate'

i'm in favor of the largedesktop require - most people that want cube would also enable rotate.

I guess though that largedesktop doesn't quite account for users who don't know about 'rotate'

[https://bugs.launchpad.net/ubuntu/+source/unity/+bug/683211](https://bugs.launchpad.net/ubuntu/+source/unity/+bug/683211)
