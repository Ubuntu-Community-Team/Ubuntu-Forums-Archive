After re-reading this and the first response a couple of times, I realize where the confusion came from.

The 'places' menu you're talking about has nothing to do with unity.  Unity, and that menu are completely separate, and unless it dropped today, there is not working places for unity yet.

What you're seeing up there is simply the file menu for nautilus, which shows up as a global menu in Unity.  All apps 'should' have their menus show up there, and most of the gtk apps already do.

The reason it only shows up sometimes, is because it only shows up when the current active window is a nautilus window, like the the applications folder that pops up when you click on the icon that will eventually provide the real menus.  It shows up sometimes when you click or right-click the desktop because the desktop is drawn by nautilus.


The fact that you only noticed this after upgrading to the latest unity is purely coincidental.  It's been there as long as there have been global menus instead of per-window menus.
