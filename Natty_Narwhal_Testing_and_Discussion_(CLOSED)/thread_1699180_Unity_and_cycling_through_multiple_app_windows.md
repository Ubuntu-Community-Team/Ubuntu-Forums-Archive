### Post by Framli on 2011-03-03
I believe it works like this :

First click on launcher : focus most recently focused window of this app (or go to second click if it already is).
Second click : minimize windows on this workspace and expo mode of all windows of this app.
Third click : exit expo.

It's good if you *want* to minimize all the windows, otherwise you would select the window you want, correct?

At least that's how it works for me. :P

> This requires too many clicks with mouse and in my opinion that hinders the usability and not forgetting the previous paragraph where exiting the expo minimizes all windows.

How do you define too many?

If one of the applications windows has focus it takes 2 clicks to switch to any of it's other windows.

If one of it's windows doesn't have focus it's 1 or 3.

One click to raise all that application's windows above all the others.
If the window you want is not the one showing in the foreground....
A second to display the expo view of windows belonging to that application.
A third to select the window you want.

There may be corner cases where it's not perfect, but generally speaking it seems pretty good to me.
