### Post by MightyMouse on 2011-04-13
I think that we can already see some common issues crystallizing from these posts.

CCSM: 
Should be default, it could help with the initial 'transition' with something as simple as allowing to switch of the launch auto-hide which can be frustrating for users that are not familiar with this type of launchers. 

Dash: 
Not everybody is convinced that the Dash is an improvement. The categories are not to everyone’s satisfaction and the search function could be improved. I have seen a mock-up here ( [http://www.omgubuntu.co.uk/2011/03/a-gnome-menu-style-unity-dash/](http://www.omgubuntu.co.uk/2011/03/a-gnome-menu-style-unity-dash/)) and think that it would be great to have a layout like this as an option to choose for users (along with the possibility to completely tweak the Dash to personal preferences).
The search function could probably be better, but if it would be no longer required to find apps but merely for documents maybe it would suffice?

Launcher: 
Drag and drop function to place objects in the launcher sounds like a good idea (not just applications but also files one might want to have available at all times).
The Expo-type display of multiple instances of the same program (several documents or web-pages) is not always sufficient, an option to have a list with file-names instead (for those that have a lot of open documents, web-pages that cause the thumbnails to become so small that they are of no use any longer) would be nice.
I also agree that it should be possible to open several instances of the same program from the launcher, after all not everyone likes to work with multiple tabs.

Minimized apps: 
Totally agree that there is a problem with minimized apps. I for one have simply added the old gnome-panels to the bottom of the screen, just because I kept 'loosing' applications or documents. This is part of the Expo-issue I think. The little arrows in the launch bar are not really the best solution. And lots of people have gotten used to being able to simply choose a file from the bottom gnome-panel. One of the gnome-panel function that is missed by more users.

Documentation: 
I totally agree that a lot of frustration would be minimized if the documentation had been available already. I had to hunt down a lot of info not in the forum (where I would normally expect to find answers) but by googeling. Seems that a lot of Ubuntu users were as clueless as me! This is getting better by the day but still....
I also miss something as simple as info on right-click. If you did that on a gnome-panel you at least knew what that thing was called. Click on the top bar in Unity and you get zilch info. The same for the launcher. This makes it difficult to even report what bothers you because you don't have the right terminology!


All in all it pretty much comes down to that there are not enough options to customize the system to personal preferences. And then I am not just talking about the 'looks' but also about functionality. An extra mouse-click required to do the same I could before with one less is not an improvement of the work-flow even if some people try to convince others of that. Wanting some features that we were used to have for years and were practical to have should not be turned down as 'old fashioned' and 'standing in the way' of progress. 

I hope that someone who can make a difference actually reads this thread, I for one don't know how to make a Wiki-page or Papercut out of this?  Maybe Lucazade has a suggestion?
