### Post by jennybrew on 2011-02-21
Is anyone else is a little uneasy about the progress, so far, on user interface ?

Unity is going to struggle in this area, anyhow, as all atempts at a desktop metaphor have been abandoned.
All the more reason that a lot of attention to GUI design is embedded in the system from the outset.

However:

The left part of the screen is becoming increasingly busy and I do wonder if its going to end up unusable if the race to crowd more functionality in that area of the screen continues apace. Is there a logic to this departure from best practice ? (it should be attractive, uncluttered and functional: or so we were taught at college!)

A good GUI should provide feedback to the user. This has formerly been one of Gnome's strengths and hopefully Unity will provide good user feedback by the time the final release arrives. It does help, especially on a test system, to know if something is actually happening or if the system is frozen!! This, coincidentally, is a problem shared by the boot screens when one is confronted with a blank purple coloured screen with no indication if anything is happening under the hood.

Hopefully design considerations will follow on quickly behind development progress
