### Post by rg4w on 2011-03-10
Good article - thanks for posting that.  I admire many things in Gnome Shell, and while I'll probably stick with Unity on my main laptop I'll likely put Gnome Shell on one of my other systems so I can keep up with all the good work they're doing with it.

> **KrazyPenguin said:**
> And the bar at the left side can not be moved or customized in any way.
Maybe this will change in the future , because that bar just takes up space.
My understanding is that by Beta 1 the Launcher will be able to be moved to either side (with a goal of being able to also use the bottom if preferred in the 11.10 release), and there will be a control available to adjust the icon size.  Other refinements will likely be in place as well (I hope having the menu bar persistent by default will be among them; it seems a lost opportunity to conceal an app's features from the user as the default behavior).

In Alpha 2 the auto-hide behavior for the Launcher seems pretty good, one of the cleaner aspects of it thus far.   If you don't see it moving out of the way as you'd like you might consider filing a bug report; here that behavior seems well done, striking a reasonably useful balance between being able to access the Launcher easily while still giving space to apps as needed.

> And I don't really see the improvision with this bar, isn't it like a MAC but move to the side instead of the bottom??It's Mac-like only coincidentally.  While not aiming to be Mac-like specifically, the solution serves similar goals to what the Mac team is addressing, so it's not surprising that in some superficial respects the two may appear similar.  

While Gnome panels are wonderfully flexible, they only become really useful when you modify them, adding launchers and such as you've done.

Ted Gould gave a [talk at SCaLE]("http://www.socallinuxexpo.org/scale9x/presentations/unity-why-does-it-matter") on some of the rationale behind Unity, and he described it as being targeted more at the average demographic than the Linux or Ubuntu enthusiasts who currently enjoy the flexibility of Gnome panels.

He described one of the goals of Unity as making apps more readily discoverable and accessible, and in that regard I would agree with him that the Launcher provides better support for those goals than panels.

Fortunately, while Unity aims for a broader demographic (many of us hope that over the next three years the number of new users will be much larger than the sum of all users who've adopted Ubuntu to date), the install will still include the Gnome 2 we enjoy today, so those who prefer it can continue to use it.

I think it's all a matter of what you are used to, and if you are willing to learn a new way of doing things. Most of the complaints I've seen here, are about how the user used to do things, and now that way doesn't work as expected.

Also I think many people expect Unity to be fully functional, like the two-panel interface that has been around for many years, I can tell you that in the beginning Gnome 2 was so bad, as far as functionality went, that it was almost unusable, I switched to KDE, and used it up until I installed Dapper, just to see if gnome had gotten any better, I'm happy to say, I've been a gnome user ever since, and will continue to be one.

Personally I like Unity, I use it much the same way I use Windows 7, using the search to find an application, as the standard Win 7 menu is just a mess, when it comes to opening applications. You can pin your favorite apps to the task bar in Windows, and the panel in Unity. I like the way they both get out of the way when you don't need them, although you have to set the win 7 task bar to hide, as it doesn't do it by default.

I can't honestly say if Unity has more/the same functionality as the two panel interface, but from what I see here, most of the complaints are more about customizability, than the user interface itself.

In the spirit of being a little picky, I don't take any issue with the thread title, but I agree in comparing apples to apples, namely Unity and Gnome Panel. I am unable to *separate* out the customizability of a given interface as something *apart* from the interface; it is an *organic* feature that is either present or not present, and thus a valid criterion for comparing shell A to shell B (on top of the Gnome desktop environment).

Moreover, it is not as simple as merely learning "a new way" of doing everything. There is currently no path for "learning a new way" to have a system monitor readout on the Unity panel, for example (even porting the applet using the indicator API creates something more static). Sure, I can open a System Monitor application, but I can also do that with the Gnome Panel shell.

If Unity is simply lacking customizability because it will essentially be in beta for another year or so, that is understandable. Then it will be more appropriate to compare the two shells once Unity is mature. If it is never intended to be customizable, then it is lacking a feature, which is a valid criterion for criticism.  Newbies aside, there ought to be "simple" and "advanced" modes of pretty much anything on a desktop computer, especially the graphical shell. (Given that Gnome Panel is historically maligned by KDE folks as lacking customizability, that's not a good base to start with and then subtract even more!)

I do like the window controls, window title, and global menu aspects of Unity. I'll upgrade to Natty and continue to use Gnome Panel instead, however, until Unity becomes more mature (if you follow the Ayatana mailing list, even the Springboard launcher panel has the possibility of significant change). Nonetheless, I am looking forward to using Unity when it is more complete, I have no interest in the new Gnome shell (interestingly, Shuttleworth accuses it of beginning to look more like Unity after Unity was made public).
