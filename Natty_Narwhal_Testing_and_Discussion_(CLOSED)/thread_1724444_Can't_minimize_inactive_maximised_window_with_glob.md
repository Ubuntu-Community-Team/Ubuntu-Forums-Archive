### Post by Eestlane on 2011-04-08
Assume you have a browser maximised.
Now you open XChat which is not maximised.
Then I would like to minimize my browser to see behind it, but I can't see it's window controls without first clicking on the browser.

Wish there was some nice solution for that.
Show Desktop button (there [isn't one]("http://askubuntu.com/questions/29969/is-there-a-unity-equivalent-of-the-gnome-panel-show-desktop-applet") anymore) wouldn't help if I just wanted to see behind the window not just desktop.

Comments? Do you also find it to be slowing you down a bit?

My solution would be to use global menu only for maximised windows. So that when another window is on top of it the panel is still reserved for the maximised window.

Or show both on the panel with focus for the active window, however include the window controls for the inactive maximised window with only small part of it's title, like:
X_O Chrom... | X_O XChat
So that only the part after | can be hovered and the global menu appears only for the active window.
