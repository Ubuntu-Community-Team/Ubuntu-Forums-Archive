### Post by coffeecat on 2010-12-08
A minor set of usability issues in Unity concerning trash, rubbish bin, wastebasket, or whatever it's called in your neck of the woods.

1 - Drag a file with the mouse from the desktop into the trash in the launcher and it elegantly and very stubbornly slides back to where it was. Try the same from a Nautilus window and nothing much happens. In fact, nothing happens. Some people are used to using the mouse to delete.

2 - Delete a file with the delete key and it certainly goes to the trash, but the trash icon does not change - no scrunched up paper. You don't know that there's anything in trash.

3 - You can only empty trash by left-clicking on it to open a trash Nautilus. (Or is that a Nautilus trash? :?) Right-click gives you no context menu. Again, some people may miss this functionality.

I couldn't find anything on Launchpad and I don't know whether the devs are aware of any of this or, indeed, whether the dev team consider it worthy of attention.

Apologies if someone has brought this up before and I've missed it.
