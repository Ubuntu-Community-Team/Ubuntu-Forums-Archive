### Post by ranch hand on 2011-07-24
> **ronacc said:**
> question RH , what is the difference between the sid and unstable repos ? I thought sid was the unstable branch but I notice there are repos both for sid and unstable .
There is a Huge difference.  Well, a little.  Kind of. Not really any at all.

If you have your sources list set to Sid or unstable it will always get the same packages no matter what happens.

The code name for Debian unstable is always Sid.

This is not the case with stable (currently squeeze) and testing (currently wheezy).  With those, if you have the code name in your sources.list it will always stick with that code name.  When I installed the OS I am on now (90% of the time) it created the sources.list with "squeeze" in it back in Nov. last year.  It was the testing version then.  If I had left it as "squeeze" it would now be the stable Squeeze.

I liked it so well as the testing release that I changed my sources.list to "testing" form "squeeze".  The day Squeeze went stable I had a huge update/upgrade as testing was moving on.  Dropped OOo for LibreOffice and a few other things that migrated to testing from Sid on that day.

If I were to change my sources.list to wheezy now it would become the stable version when wheezy becomes stable.  I will be leaving it as "testing".

This is the main thing, I think, that Ubuntu has done to Debian.  Created a repo system that is somewhat comprehensible.  I know that my first install of Debian (Lenny), right after installing Ubuntu 8.04, was a nightmare.

The configuration was not really that hard to figure out although Ubuntu made it easier.  Just finding the non-free repos was really tough.  The ones I found did not even work right.  Found the "post your sources.list" thread over on LQ and that got me straightened out.  The fact that there is such a thread and that it is a sticky in their Debian section pretty much tells the story as far as the ease of use and understanding the Debian repo system.

It is, really, a very sensible, straight forward repo system.  At least if you started using Debian at the very first release.

The system now, with Squeeze, is better than it was with Lenny.  The multi-media repo is still not on the Debian servers.  At least the contrib and non-free ones are.

Ubuntu really makes it a lot more sensible with medibuntu picking up all the non-free stuff.

My sources.list for this install known as Squeezy;
> 
## This is the Main OS, Squeezy, be nice to it.

deb [http://http.us.debian.org/debian/](http://http.us.debian.org/debian/) testing main contrib non-free
deb [http://security.debian.org/](http://security.debian.org/) testing/updates main contrib non-free

## debian multimedia
## deb [http://www.debian-multimedia.org](http://www.debian-multimedia.org) testing main
deb [http://www.debian-multimedia.org](http://www.debian-multimedia.org) testing main non-free

## Liquorix
# /etc/apt/sources.list.d/liquorix.list
deb [http://liquorix.net/debian](http://liquorix.net/debian) sid main
## deb [http://liquorix.net/debian](http://liquorix.net/debian) sid main past

### Public Key

### apt-get install debian-keyring debian-archive-keyring
### apt-key update

### Or

### If you get errors about missing keys, run these commands (replace KEY with key number)

### gpg --keyserver subkeys.pgp.net --recv KEY
### gpg --export --armor KEY | apt-key add -

Even if you get rid of everything starting with the Liquorix stuff on down it is way more complex than with a base install of Ubuntu with medibuntu added.

In Lenny there was another section for the contrib and non-free stuff with a completely different address.

This is what can really scare a noob to death.  After you get that straight the configuration really doesn't seem to bad at all.

### Post by Reiger on 2011-08-09
> **BigSilly said:**
> I get annoyed when using other desktops that there is no expose effect to reveal all my running apps.

I see what you mean, but it is there and has been for ages. The effect is provided by kwin, the Plasma thing does allow you to trigger them at Window edges but you can access them all through simple keyboard shortcuts -- so you could also set up your favourite DE with all of them by using kwin as your window manager instead:

With kwin (a Window manager which you could use for any DE) by default these exposé actions are bound to Ctrl F9 (all windows on current screen/desktop) and Ctrl F10 (all windows).

With Ctrl F8 you get a grid like overview of all virtual desktops, Ctrl F11 does the cube thing if that floats your boat, and Ctrl F12 will show a &#8220;Dashboard&#8221;. The Dashboard is useful if you like to have widgets on your desktop and want to access them without minimising windows first. Or, if you are like me you can use it to call on another virtual desktop layer instead (I've edited my configs so the Dashboard does a completely different type of thing from what's on my normal desktop -- a fullscreen widget/plugin called SAL which is a shorthand for Search And Launch.)

Switching between virtual desktop is Ctrl, F1... Ctrl F4 for me with 4 virtual desktops.

The shortcut Meta, F6 is also very useful: it positions your mouse cursor in the middle. Meta, + magnifies your virtual desktop/screen then you can navigate it with Meta, <Arrow Keys> to position it, to zoom out again you can use Meta, -, and to reset zoom to 1:1 you can use Meta, 0.

Then if you have Invert effect enabled, Ctrl, Meta + U, and Ctrl, Meta + I can be used to invert colours of the currently selected window and everything respectively. This is useful when working in e.g. a badly lit room.
