actually you can file a bug for this.

Feature requests / wishlist items are usually for things that are of low priority.

A bug is for things you think have higher priority and affect many users.

Clearly bad upgrades affect a vast number of users, so that is considered a bug.

is all about how you word things or present your case.
[I]
Upgrading fails frequently; upgrades are not fail-safe; **no way to revert back from bad upgrades**, etc.[/I]

In this case you would file a bug on ubuntu itself and/or its upgrade method. So we're not going to file a feature request, we just giving 1 suggestion on the bug report for a fail-safe upgrade, which is pretty simple to implement compared to other options (new install and migrate versus something like a sandbox). They may fix it any way they feel is better, but they need to fix it, else they have to keep it open.

**edit:** you can also submit to brainstorm.ubuntu if you want the community to help you with the brainstorming and/or other suggestions.

Either way is a problem/bug
