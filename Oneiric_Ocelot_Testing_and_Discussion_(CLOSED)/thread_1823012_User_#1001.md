### Post by fjgaude on 2011-08-11
When a partition is mounted on the boot box, x32, not the boot partition, the ownership of that partition is user #1001. No writing is permitted even using root, adm permissions.

Anyone see this situation? How to fix it? Will it be fixed before 13 October?

### Post by fjgaude on 2011-08-12
I believe the issue was I was mounting a complete partition that could be booted. The files therein got as owner #1001. The adm is usually #1000.

So I took the data files off that partition and put them onto one by themselves. Now no issue with writing to them.

This is still likely a bug in 11.10, so we will see at Beta 1 time.
