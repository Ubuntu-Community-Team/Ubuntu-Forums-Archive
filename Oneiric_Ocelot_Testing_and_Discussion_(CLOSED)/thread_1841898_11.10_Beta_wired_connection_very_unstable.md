### Post by effenberg0x0 on 2011-09-10
There's mention to a very similar bug in Kernel 2.6.38. Kernel was improperly setting up /sys/class/net/eth0/carrier to '0' for some cards. If it's set to zero, Network Manager will disconnect (because kernel is telling the system that there's no carrier. If it's set to 1, NM will try to connect). bugzilla.kernel.org seems to be off right now for me, otherwise I could try to give you more detail. But I would say that look very similar to what you are referring to.

I'm not sure, but I think 2.6.38 was what Natty shipped with by default? (I  jumped from Maverick to Oneiric - and I compiled my own Kernel 3.0.4 from kernel.org). Oneiric default is 3.0.0-something.
Do a uname -a and get back with your kernel version.

When you loose connection, do a cat /sys/class/net/eth0/carrier
If it's 0, do a sudo ifdown eth0 && sleep 5 && sudo ifup eth0
See if cat /sys/class/net/eth0/carrier is now set to 1

If this is really the problem you have there, or if you are in fact using kernel 2.6.38, using or not using Network Manager wouldn't make a difference. I would try a more recent kernel.

Regards,
Effenberg
