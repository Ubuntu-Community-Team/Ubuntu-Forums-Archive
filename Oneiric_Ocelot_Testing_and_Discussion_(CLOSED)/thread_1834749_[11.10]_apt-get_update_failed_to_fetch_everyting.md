### Post by hyperdude111 on 2011-08-28
As someone who regularly uses pre-releases I can tell you that this problem is not rare. (I have it at the moment).

I assume you add ppa's via the terminal and when that happens your computer identifies itself as "oneiric" to the ppa so it can download the appropriate packages. However _most_ (not all) software ppa's update their ppa to the latest official release ie. "Natty" and so when you try to download the oneiric packages you recieve a 404 error because those packages don't yer exist.

You choices are
1. Wait until the final release and all the ppa's you use are updated for oneiric.

2. Manually go through the sources files with a text editor and replace the word "oneiric" with "natty". (However this will cause problems when the final is released).

3. Using the software sources app that you can find hidden in software center go around editing the ppa's that don't work again buy replacing the word "oneiric" with "natty". (Be careful this will also cause problems  when the final is released).

4. Manually go through each ppa through your web browser make sure the series you chose is natty and download-install each .deb that corresponds (not dbg).

I understand some of these are complicated, If I were you I would wait until the final release. Unless you are aware that some software you tried to install can not be found because of these sources the effect is minimal. It looks annoying when you update in terminal but most of the time it isn't a real problem. 

Finally based on your output a couple of your sources are quite dated anyway such as [http://ppa.launchpad.net/danielrichter2007/](http://ppa.launchpad.net/danielrichter2007/) I doubt that these will update anyway it seems the projects have died.

### Post by JRV on 2011-08-28
This is a common problem when dealing with the development version. This sticky post from the top of this thread explains the problem:

[http://ubuntuforums.org/showthread.php?t=1751299](http://ubuntuforums.org/showthread.php?t=1751299)

It is a good to stay away from the development version unless you can put it on an machine where you don't care if it crashes. Because there **will** be problems.

When Update Manager offers a partial upgrade it is usually best not to do it.  If there is an update to something you are testing you can try :
```
sudo aptitude safe-upgrade
```

This will usually do the upgrade without removing something you need. (I say usually because I have seen the safe-upgrade mess things up on rare occasions.)
