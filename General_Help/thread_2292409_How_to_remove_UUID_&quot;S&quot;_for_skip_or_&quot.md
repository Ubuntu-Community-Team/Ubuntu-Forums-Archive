```
sudo blkid

**Found:**
```
UUID="732116d3
UUID="830729c6
```

[B]Asking for:

UUID-51529e40

The one it is asking for doesn't exist and I want to get rid of the warning.

### Post by yancek on 2015-08-27
If you had an external drive attached and now do not, that could be the reason as well as new swap UUID.  So open the /etc/fstab file as suggested (use sudo) and see if there is a line there with the UUID in question "51529e40..." and comment it out by putting a hash mark (#) at the beginning of the line.  Reboot and if the error messages is gone you're safe to delete that line.

### Post by yancek on 2015-08-28
Deadflwr was correct.  Look at the output you posted from the /etc/fstab file and you will see this entry for swap which is the UUID you indicate you are getting the message about:

> swap was on /dev/sda6 during installation UUID=51529e40-96d7-4ecd-8586-ec99a376daf5 none            swap    sw              0       0

See my post above on how to use a text editor as root user to comment out this entry so it is now read on boot.
