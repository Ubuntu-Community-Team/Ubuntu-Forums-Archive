### Post by oldfred on 2017-05-18
With a copy like that you left all the UUID references the same. So trying to boot the backup directly still has all the old UUID of original partition.
Often easier to reinstall & copy /home data & settings over with rsync.

You need to reinstall grub using sdc1 and edit fstab in sdc1 to have correct UUIDs.

May be easier to reinstall grub using Boot-Repair and in advanced mode choose sdc install and sdc drive if BIOS installs.
        [https://help.ubuntu.com/community/Boot-Repair](https://help.ubuntu.com/community/Boot-Repair)

If you have UEFI installs sdc1 should be  an ESP, but grub will only install to ESP - efi system partition on sda1. You have to also copy that and use efibootmgr.
