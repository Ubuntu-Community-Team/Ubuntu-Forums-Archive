Running fglrx-config only generates valid xfree86 configurations. Just copying/renaming the resulting configuration to xorg.conf results in a unusable X (as you have seen).

For your keyboard error, change the line
```
Option "XkbRules" "xfree86"
``` 
to
```
Option "XkbRules" "xorg"
``` 

If this doesn't work please take a look at my xorg.conf
[http://www.ubuntuforums.org/showthread.php?t=8993&page=2&pp=10](http://www.ubuntuforums.org/showthread.php?t=8993&page=2&pp=10).

### Post by jensyt on 2005-01-25
This advice is probably useless, but I had a similar problem just two days ago. I had my drivers working fine, and then they stopped working with a message similar to this. I got them to work again by uninstalling the ATi drivers and control panel, and then reinstalling them. After a not-so-quick reboot, everything was back to normal.

If that doesn't work either, try adding a section:
```
Section "Extensions"
   Option "Composite" "Enable"
EndSection
```
This won't help your problem, but should show you an error when typing "fgrlxinfo" in the console. If this error does show up, the drivers **should** work fine when removed. I realize this is a seemingly worthless thing to do, but to me it indicated that the drivers were back to the functional state they were in, permitted I removed those lines.
