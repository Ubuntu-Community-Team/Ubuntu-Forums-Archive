### Post by kingtaurus on 2008-04-28
Hi,

Can you post /var/log/Xorg.0.log?
Could you also give a more complete set of specs for your system (amount of RAM, Motherboard, CPU, etc.)?

The restricted driver manager is now actually called: Hardware Drivers
Which is under System->Administration->Hardware Drivers

Your xorg.conf shows that it should be trying to use the fglrx driver (if the module is installed on your system). Have you tried installing the ATi driver from the ubuntu repositories?(code below should install it)

```

$ sudo apt-get update
$ sudo apt-get install xorg-driver-fglrx

```

### Post by yaknowwat on 2008-04-29
I had this problem upgrading from 7.10 to 8.04

This is the topic :
[http://ubuntuforums.org/showthread.php?t=762320](http://ubuntuforums.org/showthread.php?t=762320)
```

 Google Earth + Latest ATI driver = Crash
I noticed that the ATI driver wasn't set as a restricted driver anymore in this release.

But the issue i have is that the newer driver will cause X to restart if Google Earth is loaded.

What ATI driver version was in 7.10 i am currently using 8.04 so i can't just unistall the driver.

~Solved~
Needed to reconfigure Xorg Server so that Restricted Drivers would work.

to reconfigure Xorg Server use this command

Code:

$ sudo dpkg-reconfigure xserver-xorg

And I am sure : http://ubuntuforums.org/showthread.php?t=758576
(linked to it by Tim sharitt)
helped a lot too.

After that i played a 3d game and I was asked if i wanted to enable restricted drivers (it wouldn't do this the other 5 times i tried the game).

```
I edited what i did in the thread starter post along with the commands needed.

I started off thinking it was something different and found out how to solve it.

turned out to be a simple need for an xorg-xserver reconfigure and to play an OpenGL in order to be asked about restricted drivers.
