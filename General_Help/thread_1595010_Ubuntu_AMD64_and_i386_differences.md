### Post by formaldehyde_spoon on 2010-10-12
They are the 32 and 64 bit versions of Ubuntu.

  Your hardware (and 99% of all new hardware) is 64 bit, so it is capable of running either 64 or 32 bit software.  Older (32 bit) hardware can only run 32 bit software. 

You have 64 bit hardware, so you should run 64 bit software(Ubuntu).  If you use 32 bit Ubuntu you are restricting the maximum performance possible from your machine - for example one thing you've already noticed is 32 bit Ubuntu currently only recognizing 3.3 GB of your RAM.

When running 64 bit Ubuntu you should use 64 bit versions of programs/apps where available (although 32 bit apps will also work).  For 64 bit flash click on the link in my signature, extract the .so file from the archive and place it in ~/.mozilla/plugins

If you have any trouble just post here! :)

### Post by 3Miro on 2010-10-12
Depending on what applications you are running, 64-bit it between a bit and a lot faster (general very CPU intensive tasks like video encoding). 64-bit occasionally runs into trouble with some drivers (like some printers don't have 64-bit drivers). Other than that flash and some rare codec may require an extra effort to get running.

Basically, there is pretty much no difference. Look around the "recurring discussions" sub-forum for more details on this (it is the same forum where this thread will be moved soon).
