### Post by tpapastylianou on 2010-12-01
Presumably you've found out already, but might as well answer this for anybody ending up here with the same question.

[LIST]
[*]Right click on the file or folder you want to open
[*]Select the 'Open With Other Application' option from the menu
[*]Choose your app from the list. If it's not there and you know the name of the binary executable for the package, then put that in the 'Use a custom command' field (the binary is likely to be /usr/bin/PACKAGE_NAME, where PACKAGE_NAME is the name given to the package in synaptic e.g. totem for Movie player, evince for Document Viewer, etc.
[*]If you leave the "remember this app" tickbox ticked, then the application you just chose will appear in the choices for 'Open with other application' whenever you right-click a file of that type from now on.
(NOTE: I have a bug on lucid lynx, for some reason this last bit doesn't happen with folders. don't know if people have come across the same problem)
[*]If you want to permanently associate this program as the default file-handler for this type of file, then right-click, go to 'Properties'. There should be an 'Open With' tab, click it, and select your app from there. (again, in lucid, no 'Open With' tab appears in folders anymore, whereas it did until Karmic. I found this very useful to open folders with totem. Does anybody else have this?)
[/LIST]
