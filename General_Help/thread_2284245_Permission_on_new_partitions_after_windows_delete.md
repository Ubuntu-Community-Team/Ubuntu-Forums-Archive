### Post by oldfred on 2015-06-28
You do not need sda3 the unformatted Microsoft reserved partition. With gpt there is no space after MBR. Grub used to write into that space and Windows would write into that space. When using BIOS boot with grub you have to have a unformatted bios_grub partition which is used just like the space after the MBR with MBR(msdos) partitioning. And Windows has to have its unformatted reserved partition for the same reason. It is small but not now needed.

You have to remove /EFI/Microsoft folder from the ESP - efi system partition. And if you do not remove that, grub will keep finding the Windows boot files in the efi partition and assume you have Windows to boot.

You also have to delete the UEFI entry as it has its own NVRAM to store boot info. It may even re-add the Windows entry if you delete the UEFI entry, but still have an entry in the ESP partition.

       sudo efibootmgr -v
The "-v" option displays all the entries so you can confirm you're deleting the right one, and then you use the combination of "-b ####" (to specify the entry) and "-B" (to delete it). Examples #5 is delete:, with Ubuntu you need sudo, others must be at root.

If you have Windows and delete the entry you can re-add it:
If you have Windows to restore Windows boot entry:
sudo efibootmgr -c -L "Windows Boot Manager" -l "\EFI\Microsoft\Boot\bootmgfw.efi"

Efibootmgr Examples:
[http://linux.die.net/man/8/efibootmgr](http://linux.die.net/man/8/efibootmgr)
