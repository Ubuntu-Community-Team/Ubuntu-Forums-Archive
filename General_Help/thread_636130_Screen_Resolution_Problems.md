### Post by stuart.mchattie on 2007-12-09
A friend of mine has installed Ubuntu on her Toshiba laptop and there's one strange thing we can't seem to sort out.  The screen is widescreen with a native resolution of 1280x800 but Ubuntu only offers 640x480, 800x600 and 1024x768.  I have tried to specify a generic monitor with a resolution of 1280x800 but the list of resolution available are still only the three normal ratio ones.  The other weird thing is that when 1024x768 is selected, the screen is actually displaying 1280x800 with the top left corner showing the desktop at 1024x768.  The taskbar and the main menu bar only fit into this 1024x768 and any maximised application only takes up the 1024x768 area.  However icons on the desktop can be placed in the area outside this region and application windows can be resized to fit outside the restricted area.  It's like the area exists, but Ubuntu refuses to believe it is on the actual screen.  Is there anything I can do to make it recognise that the area that contains the desktop is actually 1280x800?  Thanks in advance.

Stuart

### Post by oodledoodle on 2007-12-09
have you upgraded to gutsy? if not, try that as the 'screens and graphics' tool is pretty nifty. 

if not, have you tried manually editing xorg to include the resolution? 

```
sudo gedit etc/X11/xorg.conf
```

scroll down to the list of resolutions and add yours. save it, reboot, should work. be careful though, if you change anything else in there you might end up losing your graphics altogether. but you should see where you need to add it, its easy enough.
