### Post by TheFu on 2018-12-29
For a 2nd disk, you can just encrypt the partition. This will make a block device (like a partition). Then create a file system onto that block device and treat it like any other file system.

Use **cryptsetup** with the _create_ and _open_ options to create the encrypted block device.  Then use **mkfs** to create the file system.  There are many guides for the use of cryptsetup.

Doing full-system encryption manually is hard. [https://help.ubuntu.com/community/ManualFullSystemEncryption](https://help.ubuntu.com/community/ManualFullSystemEncryption)

Doing dm-crypt+LUKS on a non-OS partition is pretty easy: [https://www.cyberciti.biz/hardware/howto-linux-hard-disk-encryption-with-luks-cryptsetup-command/](https://www.cyberciti.biz/hardware/howto-linux-hard-disk-encryption-with-luks-cryptsetup-command/)  Ignore the veracrypt and encfs/encryptfs stuff.  That link uses yum commands which don't apply to Ubuntu.  Translate those into apt-get.  Also, some of the cryptsetup options have changed, so you need to read the manpage for it.  It is pretty clear.  If you have trouble doing that, perhaps encryption isn't the best choice?

Lastly, 100% know-you-can-restore backups are mandatory for encrypted storage. If anything, anything at all bad happens, only a backup can help.

You will need to manually "open" the LUKS volume before it can be used.  This can be scripted or manual. 100% up to you.  Some file managers understand LUKS encryption and will prompt for passphrases as needed to provide access.  I know that Caja does this, but don't know any other file managers.
