### Post by vanadium on 2007-08-29
@expatCM: This thread was split off from another thread (probably right so ;)) and that is why the issue is not clear. The issue I raised was that Grip lacks a logging feature: you cannot know after the rip if any error occurred. You therefore cannot be confident in the rip.

@MetalMusicAddict: I do consider this as a big bother (as youu already know ;) ). Grip has a very nice interface, is highly customizable and you can automate the rip such that with one click all your tracks are properly encoded, tagged and stored in correctly named directories. Yet, after the rip, you need to listen to all tracks to make sure there are no glitches, because you can not see after the facts whether there was a problem or not during extraction.

There is light in the tunnel, though. Grip is, as a graphical shell, so customizable. I now see that you can specify in fact any ripper, and there is an option to call cdparanoia from the shell (rather than "build in"). It might therefore be possible to modify the command line to obtain output from cdparanoia and redirected that to a file.
