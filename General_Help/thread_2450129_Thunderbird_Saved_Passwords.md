### Post by TheFu on 2020-09-08
I've had the same issue with Thunderbird and calendar credentials about 16 months.  Sometimes, closing the program and restarting it will "find" the credentials.  

The big hassle for me is that the calendar setup screens don't support editing the credentials, so it is a delete and re-enter to get it working again for a few weeks, before the credentials are lost.  I use multiple calendars, connecting using different credentials and some external calendars with no credentials.

Because submitting a bug to Mozilla requires setting up an account, I haven't bothered.

I run thunderbird on 2 different systems, but connections to the same accounts, same calendars.  The failures happen on both, but not usually the same week.  I think it is a memory bug that trashes a file. I was a cross-platform C++ developer for about a decade.  We considered any "wild pointers" has personal failures where I worked and used tools to closely watch all pointers built into our source code to prevent them being "wild".  I've learned by working a number of places that our techniques for pointer safety are seldom used elsewhere.
