### Post by Therion on 2009-02-20
I'd suggest trying 8.04 version of Xubuntu.

8.04 is significantly faster and more responsive on my system than 8.10 all the way around.

The Xfce DE should keep overhead to a minimum, you get the forum access you want, and a snappy, stable release in 8.04 vs. 8.10.

Barring that, maybe try Sidux.  Very stable and very, very fast on my system.  It's a far cry from Ubuntu, but maybe that's what you want:  [http://sidux.com/](http://sidux.com/)

It's hard to be more suggestive because you don't really indicate your priorities.  Do you value speed over stability?  Do you want a "rolling-release" distro like Arch, or incremental releases like Ubuntu? Do you want/have to have Fluxbox or Xfce or some other particular DE over another?  

A distro choice is a very personal decision.

### Post by mb_webguy on 2009-02-20
I know you said you didn't want to hear "Ubuntu", but... Ubuntu.  Actually Xubuntu, possibly with a window manager like Openbox instead of Xfce.

I've tried several different distributions over the years, and I haven't really noticed much difference in speed except for that incurred by various applications.  Lighter desktop environments were obviously faster than larger ones, etc.  I'm running Ubuntu with an Openbox session as an alternative to the standard Gnome, and it's quite fast.

If you want speed and aren't terribly concerned about convenience, go with Xubuntu, but replace Xfce with a pure window manager like Openbox.  Xfce is already pretty lean, but a window manager will almost always use fewer resources than a desktop environment.  The Xfce applications are smaller and faster than their Gnome or KDE counterparts, though generally not as full-featured.

If you're a true speed freak, [compiling your own kernel]("http://ubuntuforums.org/showthread.php?t=311158") will probably give you a bit of extra speed.
