### Post by kikkegek on 2008-04-28
I ran the update manager last night,
updated all the available packages (198 MBytes)

Enabled the "Ati restricted driver" again!

And Xorg is now down from 99,9% too about 1 or 2% when idle
and to 40-50% when running XBMC in a window.

I have not checked how much it is using when running fullscreen, but did notice everything was less fluent.

So, I dont know what exactly was causing the high cpu load by Xorg, but is seems a lot better now!
