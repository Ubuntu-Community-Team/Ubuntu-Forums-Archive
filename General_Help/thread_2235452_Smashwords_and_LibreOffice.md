### Post by Gordonbp531 on 2014-07-22
Of course it can be done. Libre Office has always been able to save a document as .doc. The problem has been in the past that the Word type documents created by Libre Office have sometimes not been 100% compatible with Microsoft Word.
That has almost dissappeared now with .doc format, but there can still be issues with the .docx format.
If you really want some expert advice, sign up to the Libre Office Users mailing list here:
[http://www.libreoffice.org/get-help/mailing-lists/](http://www.libreoffice.org/get-help/mailing-lists/)

### Post by Buntu Bunny on 2014-08-01
Here's an update for anyone who might be interested. I downloaded the Smashwords .doc template, opened it in LO, and have been copying and pasting from Leafpad. I tweak the formatting, saving all the way (in.doc). When I reopen the file, line spacing is doubled and tripled. I have to open and save several times before it "sticks." I don't know if anyone else has had that experience, but if you're looking for info about LO and Smashwords, there's my experience.

The solution is to save in LO's native format, .odt. while working on it. I'll convert to .doc once it's done.

### Post by Buntu Bunny on 2014-08-13
I wanted to follow-up and report success. For anyone else searching the forum for information on how to do this, my advice is to follow the Smashword's Style Guide by using the same functions in LibreOffice. The  eBook I mentioned, [Making Word.doc Files On Linux]("http://www.smashwords.com/books/view/387381"); was helpful but if you're familiar enough with LO you may not need it. LO seems to be more streamlined than Word, so the process is actually not as fussy with LO. 

I had best results by staring in LO's .odt format, then saving a final copy in .doc before uploading to Smashwords. I had quite a few aborted attempts until I figured out I needed to close the document on my computer before uploading. First go-round there was only one error, easily fixed. 

With that, I'll mark this thread solved.
