### Post by Don Barzini on 2011-02-02
Wow...

I have no idea why those files have gotten so out of control. Maybe logrotate has failed to do it's job.

For now... try clearing the log files by doing the following in a termial window...one at a time...

```
sudo cat /dev/null > /var/log/syslog

sudo cat /dev/null > /var/log/messages


sudo cat /dev/null > /var/log/kern.log
```

It might take a bit.... Then reboot your machine.

Then you can look into possible logrotate problem if that works.

---

### Post by Don Barzini on 2011-02-02
Oh yeah... I forgot..

Just delete the ".1" files. E.g....

```
sudo rm /var/log/messages.1

sudo rm /var/log/kern.log.1
```

### Post by hawthornso23 on 2011-02-02
I agree it would be ideal to extract a piece of those files to diagnose the problem. But it depends on how urgent the disk space problem is. If he's really on the point of running out of disk space then he has to fix that right away, otherwise his system could lock up completely.  

Also trying to edit an 11Gb file ... I hope he is using a really well written editor explicitly designed to handle huge files by opening them a piece at a time. If the editor tries to load the whole 11Gb into memory at once ...!

If it were me I'd just get rid of them. As deleting the logs won't fix the underlying problem the log files are going to start growing again immediately anyway. He can always give us samples of the new logs, which will be a lot easier to work with after the logs have been cut down to size.
