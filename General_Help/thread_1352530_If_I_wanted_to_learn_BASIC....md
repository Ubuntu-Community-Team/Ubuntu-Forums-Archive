### Post by akashiraffee on 2009-12-11
I used to write things in BASIC too, on the early Amigas, like the VIC-20.

Bash is useful, but it has a lot more limitations than even BASIC does.  Also, I think it is awkward to learn because of its whitespace sensitivities, which are unusual.

If I were you, I'd try and learn a modern "interpreted scripting" language like python, perl, or ruby.  Ruby is very straightforward; it's  taught to kids and recommended as a "first language".

I'm not so familiar with python but it is very popular.  Perl is totally great, but sort of arcane.

You can use any of those to do just about anything, eventually, including making GUI applications and graphics.  That is not feasible with shell scripting -- bash is intended for shell related things, sys admin and some string parsing.

There is also something called realBASIC around, I don't know how successful the linux port is.  BASIC and BASIC spin-offs are more things used on microsoft platforms.  If you want to get more into linux, go with python, perl, or ruby.  Honest!
