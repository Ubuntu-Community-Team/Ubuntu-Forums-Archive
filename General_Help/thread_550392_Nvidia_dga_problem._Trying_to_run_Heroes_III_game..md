### Post by thedarkwinter on 2007-09-14
Hi

You generally often cant start graphics programs using the sudo command. I think this is due to the x-server session running as your user.

try running the command

```

cd ~
sudo chown -R username:username ./gamedirectory

```

where username is your username, and gamediretory is the location of heroes3. This will make sure that your user has permission to access all of the files in the directory.
(Sometimes when you install using sudo, it will create files owned by root.)

And then try running the game again without sudo.

Cheers,
tdw

### Post by ad_vlad on 2008-01-16
I know it's late. but the solution is simple.

When you run it for the first time as root, it will create a directory in /home/<your_user>/.loki/heroes3
The program will fail because of the X and so on.

Afterwards when you try to run it as a normal user, h3 complains about the free space because the .loki dir was created by the root and you can't access it. Either sudo and remove the .loki then execute as normal user, or chown -r the .loki dir.
