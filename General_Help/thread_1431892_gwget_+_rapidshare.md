### Post by t0p on 2010-03-17
I don't know about gwget specifically; but I frequently use **wget** a command-line relative of gwget, and as far as I have been able to ascertain, rapidshare won't work with this kind of utility.

I'm not really sure why you want to use gwget with rapidshare anyway.  Rapidshare does not allow downloads to be paused and resumed, which is the main feature of wget that I particularly like.  When I want to download something from rapidshare, I just go through the waiting palaver, then click on the "Download" button - a little notification appears, asking what I want Ubuntu to do with the file I want to download: do I want to open the file with an application or do I just want to do a straight download?  I click on "Download", and the file downloads to my Desktop.  No problems.

Some other of these file hosting sites (Megaupload for instance) *do* seem to allow pausing of downloads in a small way; so if the ability to pause downloads is important, maybe you ought to stick to Megaupload and its ilk.  If you want to be able to stop a download some of the way into it, then resume it later, I don't think any of the file hosting services do that.
