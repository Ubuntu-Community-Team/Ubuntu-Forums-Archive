### Post by wilee-nilee on 2012-05-22
Yeah I would say a purge will help the boot repair will give you the  commands to run. sda1 needs the boot flag as well so if the purge gets  you into ubuntu again open gparted there right click the sda1 partition and then flags then boot, close gparted, then run in the terminal.

```
sudo update-grub
```

I have had the grub bootloader read my W7 partition on my setup without a boot flag,  but as of now it is on a sda3 it is better being on the sda1.

Make sure the sda HD is also read first in the bios I think it is set as being read before the sdb just check that if you have any boot problems after the purge.

### Post by wilee-nilee on 2012-05-22
So if this has not reset grub so you get a grub menu showing ubuntu at the boot, lets do a chroot purge and reinstall. To be honest not sure what the errors are you will need to say when and how the were generated clearly. So run these commands from the booted live ubuntu cd.

  	 	 	 	 	 	   [FONT=Verdana, sans-serif][SIZE=1]```
sudo mount /dev/sda3 /mnt
``` [/SIZE][/FONT] 
 [FONT=Verdana, sans-serif][SIZE=1]```
for i in /dev /dev/pts /proc /sys; do sudo mount -B $i /mnt$i; done
``` [/SIZE][/FONT] 
 [FONT=Verdana, sans-serif][SIZE=1]```
sudo chroot /mnt
``` [/SIZE][/FONT] 
 [FONT=Verdana, sans-serif][SIZE=1]```
apt-get purge grub-pc grub-common
``` 
[/SIZE][/FONT]
[FONT=Verdana, sans-serif][SIZE=1]```
apt-get install grub-pc grub-common
```[/SIZE][/FONT]
[FONT=Verdana, sans-serif][SIZE=1]Here you will be asked where you want grub choose sda only use the space key to tick it then.
[/SIZE][/FONT] 
 [FONT=Verdana, sans-serif][SIZE=1]```
grub-install --recheck /dev/sda
``` [/SIZE][/FONT] 
 [FONT=Verdana, sans-serif][SIZE=1]```
update-grub
``` [/SIZE][/FONT] 
 [FONT=Verdana, sans-serif][SIZE=1]```
Exit chroot: CTRL-D on keyboard
``` [/SIZE][/FONT] 
 [FONT=Verdana, sans-serif][SIZE=1]```
sudo reboot
``` [/SIZE][/FONT]
