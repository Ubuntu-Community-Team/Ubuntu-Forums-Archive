### Post by Mark Phelps on 2009-11-15
The other problem you may run into is that screen redraw rates (fps) in 3D games is critically dependent on hardware acceleration being enabled for the video card/chip -- and AMD/ATI dropped support for all but the latest HD 3x/4x/5x cards back in May.

So, if you're running one of the unsupported cards (or one of the same chips), you won't be able to get good framerates in any Ubuntu version newer than 8.10 due to the unavailability of hardware drivers.

### Post by 3rdalbum on 2009-11-15
> **Mark Phelps said:**
> and AMD/ATI dropped support for all but the latest HD 2x/3x/4x/5x cards back in May.

Fixed for you.

To reiterate: Linux doesn't run Windows programs, and Windows doesn't run Linux programs. There is a reverse-engineered compatibility layer called Wine that has some degree of success in running some Windows programs and games (note the repeated use of the word "some"). You can try that and see what works. I think a couple of the games you mentioned are on the Play On Linux supported list (PoL is a program that can set up Wine for you, for specific games).

Your best option is to dual-boot: Keep Windows, and also install Ubuntu in a side-by-side configuration. This requires resizing the existing Windows partition to make space for Ubuntu. The Ubuntu installer can do this for you, just make sure you tell it how much space you want. When you have installed Ubuntu in this way, you will see a menu whenever you boot up that will ask you what operating system you want to run.
