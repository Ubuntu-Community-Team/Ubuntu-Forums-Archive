### Post by Vaphell on 2011-04-04
more cores means that system doesn't grind to a halt when some process takes 100% cpu. Majority of apps doesn't benefit from multiple cores per se, but they are evenly scattered around so they don't have to compete as much with other apps for computing time. Also some classes of math heavy apps take advantage of multiple cores - math software, scientific simulations, video decoding/encoding, etc.
Quad core processors from AMD are cheap, offer great bang for the buck and are good enough for anything but the most demanding tasks. My quad sits there idling 99% of the time and i can't say a bad word about its performance when required.
If you are not a gamer, modern quad core with plenty of fast ram should do just fine for many years to come (pick option with 1333MHz or 1600MHz RAM, faster RAM=better loading times of everything and better performance in tasks requiring tons of data bandwidth)

### Post by 1clue on 2011-04-05
Are you going to be running virtual machines or server-type services with a high load?  If so, then more cores makes sense.  Or if you do a lot of really heavy-duty compiling, which I doubt most Ubuntu users do.

I've spent quite awhile monitoring what goes on in my i7.  If I'm not running a VM and not compiling, then chances are I won't use significantly more than 2 cores.  This includes a relatively huge flight simulator, x-plane.  It turns out that the current version can't heavily load more than one core.

That said, I know that x-plane and several other vendors are trying to offload more work to other cores.  There is currently an architecture vacuum to use all those cores manufacturers are adding, but I don't expect it to be too long before they can use more if you have them.

I would not even remotely consider buying a system with a single core anymore.  The advantages of a dual core system are amazing.  However, if you're just an average user then I would say a screaming fast dual core system is better than a quad core.

### Post by lithopsian on 2011-04-05
Currently four cores is about two more than most people will use.  Ubuntu certainly offers great support for four or more cores but most users simply don't do things that need more than 1 most of the time, and a spare one so the machine is still nicely responsive when one core is busy with an application, X, and maybe some background stuff.

Google Chrome is possibly now the most common app where more than two cores could be useful.  It throws most tabs into a separate process and so you can potentially end up utilising several cores although there would have to be pretty heavy duty stuff going on like flash videos.

Firefox doesn't do this yet except for farming out Flash onto a separate process. but in the next release (or maybe the one after) it will be split into several processes too.  If you build it (multiple cores) they will come (multi-process apps) :)
