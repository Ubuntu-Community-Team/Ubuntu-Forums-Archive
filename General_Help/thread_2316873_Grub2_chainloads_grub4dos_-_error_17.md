### Post by oldfred on 2016-03-11
Root(0,0) is only valid with grub legacy and grub4dos is based on grub legacy.
Partition numbering changed with grub2, and now in grub2 first drive and partition is (hd0,1)

Why do you need grub4dos? Grub2 can do everything you want if you boot it.

### Post by oldfred on 2016-03-11
Windows is not designed to be booted other than directly by BIOS or UEFI.

But I was able to use grub to boot a Windows repair flash drive in BIOS mode.
I installed Windows repair to flash drive. Then installed grub to same flash drive. What is/was important is that Windows boot code is still in PBR or partition boot sector and that grub use of a /boot folder and Windows /boot folder are the same. Often with Linux you get /Boot or vice-versa? But with NTFS case is not important, but in Linux it is. So you break Windows if grub installs to a different /boot. Just combine them, so you only have one.
Then since I had grub and Windows repair was small, I shrink partition, added another partition and as many ISO as would fit. Then grub loopmounted those and I had a repair tool for both Windows & Linux.

### Post by oldfred on 2016-03-12
You cannot directly boot the Windows ISO as far as I know from grub.
But you can extract ISO into NTFS and install grub to that. It would have to be first partition on drive.
Windows has more boot code in the PBR or partition boot sector. And that is normally what grub chain loads to, or MBR loads.

How BIOS Windows (and BIOS systems in general) work.
 Pictures here worth 1000+ words - Vista but all Windows with BIOS/MBR
[http://www.multibooters.co.uk/multiboot.html](http://www.multibooters.co.uk/multiboot.html)
[http://www.multibooters.co.uk/articles/windows_seven.html](http://www.multibooters.co.uk/articles/windows_seven.html) 

But grub does not use PBR, syslinux & Lilo are just like Windows boot loader in that code in MBR loads code from primary partition with boot flag and more code in PBR.
