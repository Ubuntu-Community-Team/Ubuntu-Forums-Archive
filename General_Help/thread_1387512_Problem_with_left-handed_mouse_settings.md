### Post by pi/roman on 2010-02-19
check the button map for a mouse with the word razer in it
```
xinput list --short | grep -i razer | grep -o '"[^"]*"' | xargs xinput get-button-map
```reverse the first and third buttons for a mouse with the word razer in it
```
xinput list --short | grep -i razer | grep -o '"[^"]*"' | xargs -I mb xinput set-button-map mb 3 2 1 4 5 6 7
```

### Post by pi/roman on 2010-02-21
Ok, I didn't realize you also had a razer keyboard too. But you listed all the necessary info so this is fairly simple.

```
#to check the buttons
xinput get-button-map "Razer Razer Lachesis"
#to change the buttons; and you can use the first 3 or add as many as are available as long as they are not doubled or missed
xinput set-button-map "Razer Razer Lachesis" 3 2 1
```You can put that in startup programs to make it persistent. You could use the id number instead of the name but it will change so to make it persistent you need to use the name in single or double quotes.

You can also reverse the first and 3rd buttons on any pointer attached to your computer by the following... open up .Xmodmap in your home directory:
```
gksudo gedit .Xmodmap
```Then paste the following into it:
> pointer = 3 2 1 4 5 6This setting will be persistent each time until you remove it from .Xmodmap

Let me know how that works and if you would like to reverse just the buttons on the razer mouse I could probably put together a custom command in order to do that. The reason you need a customized command is because although you can use the device id to change the settings, for instance your mouse in that example was id=10 so the commands would be:
```
xinput get-button-map 10
xinput set-button-map 10 3 2 1 4 5 6 7 8 9
```the next time you reboot the id may change. So in order to change the settings automatically the command will also have to search for the current id to use.
