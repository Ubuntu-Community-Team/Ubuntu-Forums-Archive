### Post by efflandt on 2014-02-27
Some computer makers (HP in my case in 2004) had cylinders on the drive set to 240 for some reason and more recent OS's try to change cyl to 255 (and change heads to match) often without warning. I had that issue years ago when I installed 64-bit WinXP Pro beta. 64-bit Win changed my cyl from 240 to 255 even though I already had a partition for it and neither it nor 32-bit Win XP Home could boot. Fortunately I had run fdisk once (although, I did not save the output), so I did the math and was able to play with cyl and head numbers from a Linux repair CD until I got them able to boot. Then when I went to install some SuSE linux version at that time on a different existing partition, SuSE install tried to do the same thing, but I noticed the different numbers, stopped it in its tracks, and told it to use the partition as is. I still have that old computer with obsolete 64-bit WinXP beta and SuSE removed and Ubuntu 10.04 needing to be updated or replaced. But WinXP Home still works on it without ever having been reinstalled.

I have never used boot repair, so I don't know if it can automatically fix the possibly changed cylinders and heads. It did flag that yours possibly should be 240 heads instead of 255. But I cannot tell you how to proceed, just that it does sometimes happen.
