### Post by 3rdalbum on 2009-02-13
On Linux, there are certain types of programs that are actually written in text files. HTML files (web pages) are also types of text files.

If you tell it to Display (rather than "run" or "run in terminal"), then Ubuntu will open the file as it would normally, rather than try and run it as a program.

It's nothing to worry about, it's just a feature of Linux and of Ubuntu in specific.

### Post by Zimmer on 2009-02-13
As the post #3 says, it is down to permissions for text type files. You can change the behaviour globally with System>preferences>File Management   and select your preference on the 'Behaviour' tab. The default for executable text files is 'Ask each time' , which is why you get asked, when you double click a text file, what you want to do with it.. 

If you change the behaviour in this way to 'view' then any executable text script file will NOT run when double clicked... (you will still be able to run it with a Terminal command, of course.
