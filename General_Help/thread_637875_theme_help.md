### Post by p_quarles on 2007-12-11
"A" GTK theme engine is installed by default in Ubuntu, but it looks like you're trying to install Aurora, which is not a default application.

As for the error, this looks like a simple case of not having the correct tools for compiling the software. Most likely, this will fix it:
```
sudo apt-get install build-essential
```
Or, alternately, you can use Synaptic and search for "build-essential."

### Post by p_quarles on 2007-12-12
This is pretty standard when you're compiling something from source code -- you usually find that you need development versions of libraries you already have installed. In this case, installing the following package should clear things up:
```
sudo apt-get install libgtk2.0-dev
```
I did a quick search on this, and it looks like that will be the only dependency that you need to install [[link]("http://phorolinux.com/how-to-install-aurora-gtk-engine-13-on-ubuntu-710-gutsy-gibbon.html")].
