### Post by FAJALOU on 2008-10-05
Aha ok thank you;).  It looks like you are running a card with the chipset called Atheros AR242x.  To get this card to work, all you should have to do is install linux-restricted-modules.  It would be *nice* if you could hook up the computer to  an ethernet connection, and then once connected, type in the command:

```
sudo apt-get install linux-restricted-modules
```

If you can get connected to an ethernet port, however, try to just go into System>Administration>Hardware Drivers and see if your card is found there, but if not try the code above.

If you cannot get to the internet at all (ie it's not a laptop ;) ), you can get linux-restricted-modules from here: [http://packages.ubuntu.com/hardy/misc/linux-restricted-modules-2.6.24-19-generic](http://packages.ubuntu.com/hardy/misc/linux-restricted-modules-2.6.24-19-generic)

That is if you are using the 2.6.24-19-generic kernel.  To figure out what kernel you are using go to terminal and type ```
uname -r
```
You are probably using this kernel, but just to so you know ;).  Hopefully this helps, respond with any issues/ problems :D :guitar:

### Post by FAJALOU on 2008-10-06
Hey:  well you have gotten somewhere ;).  First try running sudo apt-get update     from terminal.  If it gives you that error again, run   sudp apt-get update again.  Notice that SUDO.  It's needed for that command, even if it says otherwise.  If the first method works, great, continue installing via zipperback's url. it looks stable.  if the above doesn't work... try changing your servers, this can be done in System>Administration>Software Sources  and look for "Download from:"  And change that to universal.  Then click okay at the bottom, it will ask you to reload everything and THEN try your commands.  If you are still having problems, make sure that you can get on the internet from Firefox with the ethernet connection.  If you cannot, then this is a much larger issue than just wifi ;).  Hopefully this works.  As always, report back.

PS.  the reason why 'apt-get update' wasn't working was because it was not as sudo.  apt-get update ALWAYS needs sudo, and if you ever get a permissions error like that, try usin sudo to get rid of it.
