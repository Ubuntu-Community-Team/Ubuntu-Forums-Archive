For some reason when you installed ubuntu it considered your hard drive to be /dev/sdb, not /dev/sda. And now it's showing it as /dev/sda. I've seen this on one of my computers when I boot from USB. But for a wubi install this is strange, unless you had a USB/SD device plugged in during the install and your bios considers this before the hard drive. 

You've already edited the grub.cfg to point root at /dev/sda1, so if that works to boot Ubuntu, then just drop to a terminal and run "sudo update-grub" to permanently modify the grub.cfg.

If you can figure out in which cases your drive appears as /dev/sdb and prevent this from happening it should work fine. Maybe there's a bios setting?

### Post by bcbc on 2011-03-10
The CD won't affect /dev/sdX designations. Card readers, USB devices etc. might.

PS to install wubi you don't need to unpack the ISO (except to retrieve the wubi.exe - although the download site has a separate copy of wubi.exe available). Just place the ISO and wubi.exe in the same folder and run wubi from there. (I doubt that the method you used would affect the drive ordering - just wanted to mention that).
