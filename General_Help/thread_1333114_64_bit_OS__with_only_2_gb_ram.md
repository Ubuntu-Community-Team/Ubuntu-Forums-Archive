### Post by DeMus on 2009-11-21
Choosing 32 or 64 bit is not determined by the amount of ram. Okay, using 32 bits limits the amount of ram to 3GB, while 64 bits does not do that.
What is important is what you want to do with your computer. When you have CPU-intensive programs running then it is a good idea to choose the 64 bits while, as the name says it already, can calculate with 64 bits numbers, instead of doing 2 calculations with 32 bits numbers. It makes it faster.
When you "only do the normal computer stuff" stick with 32 bits since it seems to be easier on installing certain things, like drivers, flash, etc.
Get the hang of Ubuntu and when you are confident enough you can always make the change when a new version comes out for example.

Have fun.

### Post by nem75 on 2010-04-07
At least you won't have any major drawbacks nowadays when going with the 64 bit version.

It won't make your computer magically faster though, perceptions of that kind tend to be subjective and anecdotal. Fact is that you will only have performance increases if

- the CPU is 64 bit
- the OS is 64 bit
- the application in question is 64 bit
- and the data processed can and is actually represented in > 32 bit integers (which is only the case in video encoding, encryption and some database scenarios)

So the main criteria for 32 or 64 bit OS, especially if you don't do much / any video encoding, is still actually the size of your RAM.

And even with more than 4 GB you might still want to use a 32 bit OS, because Linux is PAE capable and therefore can use up to (I think) 32 GB of RAM with a 32 bit memory system, provided you don't have processes that need more than 4 GB assigned to them.
