### Post by loomsen on 2008-10-31
Write it down in your xorg.conf.

Your Screensection should look sth like this:
```

Section "Screen"
# Fernseher
    Identifier		"Screen1"
    Device			"Videocard1"
    Monitor		"Monitor1"
    	DefaultDepth		24
    Option			"TwinView"	"0"
    Option			"TVStandard"	"PAL-B"
#	Option 			"PixmapCacheSize" "2500000"
#	Option 			"AllowSHMPixmaps" "0"
    Option			"AllowGLXWithComposite"	"true"
    Option			"AddARGBGLXVisuals"		"true"
#    Option			"metamodes" "CRT: 1360x768 +0+0,1024x768 +0+0"
#    Option			"DPI"		"100 x 75"
    SubSection		"Display"
	Depth			24
#	Modes		
[color=red]	Modes		  "1360x768" "1024x768" "nvidia-autoselect"[/color]
    EndSubSection
EndSection

```

The red line specifies the resolutions, it will first try and set 1360x768, if it doesn't work 1024x768, and if that fails too it will use nvidia-autoselect option...

### Post by ruff on 2008-11-02
I had the same problem myself when changing my settings for dual monitor setup. Turns out that the setting were not being saved. I had to rename the xorg.conf file prior to saving for it to work.

Open a terminal window and code: 
gksudo nvidia-settings

Change your settings as desired but do not save yet. keep the application open.

open another terminal window and rename the xorg.conf file

code: 
cd /etc/X11
sudo mv xorg.conf xorg_conf_backup.txt

Then save your nvidia-settings pointing to the /etc/X11 directory and calling the file xorg.conf

It will complain that there is no xorg.conf to merge with however creates the new one as required. 

I hope this clarifies it enough for you.

Cheers.

### Post by loomsen on 2008-12-13
[color=blue]*edit*
[Mode validation by the nvidia driver](http://us.download.nvidia.com/XFree86/Linux-x86/1.0-9755/README/appendix-j.html)
*edit*
[/color]


This means that this mode could not be validated by your nvidia-driver. To find out which calculations are done exactly to validate modes, browse through nvidias homepage, I've recently stumbled upon when searching for custom DPI settings I think.

However, your actual and potential resolution, as well as the refresh rates, are determined by different variables, like screen size, DPI, max pixel clock, etc.

So, in your case, your video device isn't able to draw your screen with a vert refresh rate of 85. (Even if it may be capable, nvidias mode validation blocks it from doing so.)

You can disable the mode validation, but I'd not recommend doing so. 

Actually, your vert refresh rate should be 60 anyway, so just let nvidia-settings take care of it and find the refresh rate for you.

```

Section "Screen"

# Removed Option "metamodes" "1024x768_85 +0+0"
    Identifier     "Screen0"
    Device         "Device0"
    Monitor        "Monitor0"
    DefaultDepth    24
    Option         "TwinView" "0"
    Option         "TwinViewXineramaInfoOrder" "CRT-0"
    Option         "metamodes" "CRT-0: 1024x768 +0+0"
    SubSection     "Display"
        Depth       24
        Modes      "1024x768"
    EndSubSection
EndSection
```
Something like this should work just fine.


BTW, there isn't a Category created or such, the line inserted is just an information for you for debugging purposes. It tells you, that your custom metamode is not valid so you might as well remove it.
Lines with # prepended are comments and will be ignored by your X server. This is useful if you specify a lot of custom options to make notes for yourself.
