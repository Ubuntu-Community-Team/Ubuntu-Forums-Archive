### Post by alexfish on 2009-10-21
try this

sudo aptitude -o Acquire::http::No-Cache=True -o Acquire::BrokenProxy=true update


then test with

sudo apt-get update

### Post by naphtha on 2010-08-09
Had the same problem.
I my case it was something fishy in the old apt lists.

Solved by deleting and reloading:

```

sudo rm -r /var/lib/apt/lists
sudo mkdir -p /var/lib/apt/lists/partial
sudo aptitude update

```

### Post by nardis_Miles1 on 2011-06-09
We just resolved this problem. While the remedies listed didn't work, it turned out the the mirror had an inconsistent set of resources. Specifically the anl ubuntu mirror has most items under [http://mirror.anl.gov/pub/ubuntu/](http://mirror.anl.gov/pub/ubuntu/), the lucid-updates is under [http://mirror.anl.gov/ubuntu/](http://mirror.anl.gov/ubuntu/), even thought there was another, older lucid-updates in [http://mirror.anl.gov/pub/ubuntu/](http://mirror.anl.gov/pub/ubuntu/)

So, when all else fails, look really carefully at sources.list and at the mirror.
