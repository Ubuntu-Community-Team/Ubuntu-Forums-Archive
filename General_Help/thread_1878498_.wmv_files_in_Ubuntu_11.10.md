Can we see the output of 

```
mplayer /path/to/file.wmv
```

Remember that WMV is just a "container" format; what matters is how the content inside the container is encoded.

When I use that command to play the Windows sample WMV file called Wildlife.wmv, I get this:

```

Opening video decoder: [ffmpeg] FFmpeg's libavcodec codec family
Asking decoder to use 2 threads if supported.
Selected video codec: [ffvc1] vfm: ffmpeg (FFmpeg WVC1)

```

That means it's using mplayer's internal codec ffvc1 (part of the ffmpeg suite) to decode the video.
