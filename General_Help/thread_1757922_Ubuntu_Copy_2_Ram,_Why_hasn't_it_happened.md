AFAIK "toram" is a function for Live CD/USB boots to load the whole CD ISO image to RAM so it does need to access that slow media again.

This has no use whatsoever in normal installs as the kernel is already loaded to RAM and it would be pointless in loading every other possible library to RAM just in case it *might* be used.

Ubuntu - or more accurately, the Linux kernel - already manages available RAM quite effectively.
