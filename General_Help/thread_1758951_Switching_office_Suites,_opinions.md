### Post by sanderd17 on 2011-05-15
Some history:

Oracle does not like to give anything away for free

-> developers of OpenOffice.org leave Oracle and found the Open Document Foundation

-> since OpenOffice.org is open, they can take the code with them. But they can't use the name, since the name is property of Oracle. So they choose a new name: Libreoffice.

-> Major companies like Google, Canonical and Novel start to support Libreoffice instead of OpenOffice.org.

-> Oracle sees that those clients are running away and "gives OpenOffice.org back to the community". Than means that they are not working on OpenOffice.org anymore. But since Oracle doesn't give the name away, the statement "giving back to the community" is worthless. So the devs stay with Libreoffice and OpenOffice.org gets abandoned.

For the moment, the differences between OpenOffice.org and Libreoffice are minimal. If you made a study for OpenOffice.org, you will get the same results for Libreoffice. If you have files or templates prepared for OpenOffice.org, they will work on Libreoffice too.

If you want professional support for Libreoffice, Novel is giving that. 

So if you want any kind of support or updates, Libreoffice is the best choice.
