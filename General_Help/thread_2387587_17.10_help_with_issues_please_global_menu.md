### Post by monkeybrain20122 on 2018-03-21
You can easily get unity back in 17.10 and 18.04
```

sudo apt install unity-session
```

Then when you reboot, you will find an option to log into unity desktop if you press the ubuntu button. 

gnome shell uses lininput for touchpad and if you need tab for click in unity you need to install one extra package
```

sudo apt install xserver-xorg-input-synaptics
```

The community is maintaining unity and there will be a unity respin for 18.04 [https://community.ubuntu.com/c/desktop/ubuntu-unity-dev](https://community.ubuntu.com/c/desktop/ubuntu-unity-dev)

screenshot taken from ubuntu 18.04 unity desktop, global menu works flawlessly here.

I think you are kind of missing OP's point. Sure you can hide the top panel but who cares if you have a top panel or not if nothing is running. But if you open an application like Firefox there is a big fat ugly empty slab on the top part of the window. You can make it smaller with some css hack but you can only shrink it that much, whereas the menu and tool bar in Unity blends in with the top panel so that it is both efficient and aesthetically  pleasing. **Edited:** there is an global menu like addon but if IIRC it is in  conflict with hide top bar, and often the menu simply don't show

And speaking of mouse move the entire design is awkward. Workspaces are exposed on the right edge, but you have to move he mouse to the top left hot corner to expose it (which is disabled by default in Ubuntu, though enabled in Fedora). Virtual desktop movement is restricted to vertical (whereas in Unity you can move horizontally and vertically if you configure the desktop wall options in ccsm) There are some extensions that change these insane design to some degree, but they are not particularly good (like there was a thing that let you expose virtual desktop when you move the mouse to the edge and also change the vertical layout to a grid like unity, but that doesn't work very well because the desktops are sometimes exposed randomly even if you don't want them to) 

The panel layout of gnome abs (nautilus, evince etc) are inconsistent with everything else and oddly design. e,g In Nautilus why do you have "new window" on  the left top corner but "new tab" on the opposite side in a button along side with change to icon view/list view??

The launcher bar may look like unity's but lacks the functions. In Unity you can stack as many applications in it because of the accordion design, gnome's  is pretty much useless, with a few icons and suddenly the size shrinks drastically (you can make it somewhat less abrupt with some dash to dock setting but only to a degree, it can't remedy the inherent flexibility of gnome's design). Notification icons are kind of broken, there are a few extensions trying to fix it but they are in conflict with each other and hide top bar....

Gnome's dash search is night and day comparing to unity's and it still can't search files. ..

Lastly the laggy animation. There is an addon called "impatient" I think that is supposed to speed it up, it does, but the animation becomes very jerky.

I have dualboot with Fedora for a few years so I am quite familiar with gnome shell. I didn't have a prejudice against it and really wanted to give it a chance, I like the way it looks and I don't have a preference for the "desktop metaphor" (How can I ? I am a big fan of unity), but I just can't adjust to its anti-user design philosophy, the arrogance of the devs (I have been following the bug tracker and filed a few bugs)and the awkward workflow. No wonder Redhat doesn't use gnome-shell as RHE's default interface. I tried gnome shell on 18.04 for a few weeks before I nuked it and installed the unity respin iso. To its credit I think the Ubuntu team does try very hard to make gnome shell more pleasant by including some extensions like dash to dock. But to get the real gnome experience you have to go to Fedora, I was told. Well I have been there and don't miss it a bit.

Gnome addons are basically a bunch of third party hacks attempting to overcome GS's design flaws and work around its inflexibility, so they are not addons in the sense of Firfox's and their existence speak against GS's customizability rather than for it like GS supporters often cliam. You don't need a dozen of random, often poorly maintained third party hacks to change KDE's settings, you can find those settings in KDE itself.
