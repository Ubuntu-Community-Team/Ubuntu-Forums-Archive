### Post by Volt9000 on 2009-05-18
Experience, that's pretty much it. I doubt anyone sits down, studies those websites and memorizes every command. ;)

The best thing to do, if you don't know something, is first try researching yourself. If you come across a series of commands that seem to work for you, then research each of those commands, and figure out exactly what's going on at each step. Linux has very rich documentation, so you can find out details about any command by typing

```

man commandname

```

in a terminal window.

I find the best way to learn this sort of stuff (for me anyways) is to try and figure it out myself. Understanding things on my own instead of someone dictating it to me will not only help me understand it better, but also feels more rewarding in the end when I can apply it myself.
