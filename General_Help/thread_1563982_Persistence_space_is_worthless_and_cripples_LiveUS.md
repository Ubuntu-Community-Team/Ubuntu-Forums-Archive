### Post by Starks on 2010-08-29
It only takes a few uses of a LiveUSB drive with a persistence file to fully saturate and generate "0 bytes remaining" messages.

When this happens, you can no longer use the drive to install Ubuntu until casper-rw is resized or deleted or the USB device is reformatted.

I think persistence shouldn't even be an option and that storing files temporarily in RAM is a better solution.

### Post by sgosnell on 2010-08-29
What I consider a better solution is doing a full install to the flash drive.  The only thing I use the liveCD version for is an initial install.  For drives I intend to use portably, I do a full install, and the entire drive is available for data files and added packages.  Installing to a 16GB USB drive or SDHC card is a very viable solution.
