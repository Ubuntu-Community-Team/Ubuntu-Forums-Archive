### Post by fragos on 2010-06-16
When a window greys out it would normally mean that the program running that window isn't responding within a time limit. If the application responds after the window greys out it will regain it's colors. I've not experienced this problem for a while with any application. If you start a desktop application from a terminal window the terminal stays open and reports error or warning messages that aren't visible with the GUI. It's the 1st thing I try when an application acts up. You mentioned folders which I assume you were viewing with Nautilus. Folders with a large number of files, say /usr/bin, can take a long time to come up. The first time you view a file that has a thumbnail that thumbnail has to be built which can also slow down response.
