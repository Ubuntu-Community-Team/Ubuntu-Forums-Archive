### Post by munkymac on 2013-08-13
I find that I often (but not always) have difficulty signing in to WIFI networks via the web browser in cafés, libraries and the like when the network is open but the login must be validated with a username and password (or simply a login phrase) in a web browser. I've tried disabling my pop-up blocker and adblocker, but that doesn't necessarily change the problem. Here's how it usually works (which is to say, how it doesn't work):

1. Connect to the "open" WIFI network
2. Open my browser
3. Sometimes the login page loads, but often it doesn't, or won't until I perform multiple refreshes.
4. When the login page does finally load and I enter in the info, I get the spinning grey circle until a "can't load" page comes up
5. Sometimes, the login page will just reload, blank. Sometimes, I can't reload it at all. 
6. Sometimes (rarely) it connects, and I have no clue why. 

That being said, I've noticed that if I try to use Epiphany to load the page, that is absolutely guaranteed not to work. It will only sometimes work when using Chromium.

My setup:
Ubuntu Gnome 12.10
Chromium Browser
Samsung NC10 netbook.
