### Post by Holger_Gehrke on 2022-08-02
The official place to get Balena Etcher is [https://www.balena.io/etcher](https://www.balena.io/etcher) . They offer an AppImage which is very easy to install. Download it, put it in a place where the system will find executables ($HOME/.local/bin/ is a good place, you don't need root to access that; careful, the dot at the beginning of the name is the Unix way of saying 'hidden file or directory', if you're using the GUI then you might need to enable showing hidden files (ctrl-h in the file manager)), make it executable (in Unix-like operating systems executability does not depend on the extension, instead it's part of the permissions of a file; 'chmod u+x balenaEtcher*Appimage' in a shell in the directory where you put it should do it; or you could just use the file properties dialog in your file manager) and then call it from the shell. If you want to get fancy you can create a .desktop file for it - also called a launcher - a simple text file that tells the GUI where to find the Program, what it's called and what icon to show for it and put it where the system looks for desktop files  ($HOME/.local/share/applications/ is one such place). Alternatively they offer a repository for installing the program from with apt and explain quite clearly how to set that up.

For cleaning up the mess I'd remove both previous attempts and the configuration left behind by these failed attempts. Getting rid of the old config is easy: delete the directory $HOME/.config/balena-etcher-electron/ with your file management tool of choice. The way to get rid of the installed program(s) is similar, but since you put these in system directories you need to act as the system administrator - called 'root' on Linux and all other Unix-like OS. In your file manager open the location bar (ctrl-L) and type 'admin:///opt'. You will be asked for your passwort (on my system I was asked twice; might happen to you, too). You should now be able to remove the directories you created.
