### Post by psychofish on 2007-04-12
Hi everyone... A few days ago something started happening to my Ubuntu installation which made me very sad. :( It now is in a continuous reboot loop. Here's what happens:

- I turn on my computer, wait for POST to finish, etc.
- GRUB screen, default OS is my latest kernel, I boot with that
- Ubuntu startup screen with progress bar
- NVIDIA splash screen
- Gnome Login screen. I login with my user name and password
- Gnome startup splash, with the startup programs
- Sometimes, I see my wallpaper for a split second. 

And then, all of a sudden, it goes BLACK! There's a blinking cursor on the upper left of the screen. I wait for what seems like thirty seconds and the entire process starts over from the NVIDIA splash screen, almost like Gnome or X-Windows is rebooting itself (but not the system).

I can't boot into "Failsafe" Gnome, it does the same thing. I can boot into a terminal window okay.

I don't know what I did to make this happen. On the last successful boot the only change I did to my system was the standard synaptic upgrades. I haven't installed anything new since then. It's possible something in the upgrade screwed up my system? But how do I even begin to look for the problem? What's the process to investigate? Just a side note, I've been using the system with no apparent problems for almost two months now.

Please help!! Thanks!

### Post by rai4shu2 on 2007-04-12
Most likely a Xorg problem. The way you discover the problem is by looking at the contents of your "/etc/X11/xorg.conf" file and your "/var/log/Xorg.0.log" file. If you post those here, someone can help determine what's going wrong.
