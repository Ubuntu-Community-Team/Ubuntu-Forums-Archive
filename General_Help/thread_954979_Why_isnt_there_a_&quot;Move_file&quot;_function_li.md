### Post by spupy on 2008-10-22
> **bab1 said:**
> So it's like this: If you are drag and drop in the same partition the default is to move and the default drag and drop to another partition to copy it.

That does sound logical. Notice that moving to the same partition is instantaneous - the file is not really written to another place on the physical hard disk. You can just move it back if you change your mind and want to copy instead. Copy to the same partition on the other hand requires more time to write the data (copy it) to the new location. 
Now, move to another partition requires the data to be copied to the new place on the hard disk and deleted from the old one. Copy to another partition requires the same, minus deleting from the old place. So default is copy, if you change your mind and want to move instead, wait for the copy operation to stop and delete the source files.
Any ideas what Windows' defaults are?
