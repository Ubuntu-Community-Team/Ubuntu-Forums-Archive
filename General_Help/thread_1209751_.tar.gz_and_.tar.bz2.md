You need to be in the right directory when you extract the archive, ie the directory with the .tar.gz archive in it (otherwise the system won't know where it is, that's what active directories are for). To print the active directory to the screen, enter pwd.

When you've extracted it to a folder, cd to that folder and run ./configure. When you ran configure, there was no file called configure cause you hadn't extracted the archive.

PS. If one step fails, don't just blindly go on to the next like that! It will muck up your system if you do that. For example, say you're moving lots of files from a directory and deleting the rest. You do a typo in the move command and they don't get moved. If you then do rm * (remove all files in current directory), it would also delete the files you want (permanently, there is no 'undelete' in Linux not like NTFS, AFAIK).

PPS. Only ever run a command if you know exactly what it does. If you run it without knowing what it does, your inviting trolls to tell you to 'sudo rm -rf /' (never EVER run that command btw). It would delete all files and folders in the system, on every single writable media including things such as external harddrives, provided it's mounted.
