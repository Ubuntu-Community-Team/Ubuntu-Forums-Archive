### Post by Mark Phelps on 2012-09-14
Basically, apart from the default open-source drivers, which are installed by default, there are no ATI drivers for that card.  ATI dropped Linux support for that card years ago.

This means that you will be using the Radeon driver now and you will not be able to install or use Catalyst Control Center.

### Post by QIII on 2012-09-14
Again, no on the Catalyst driver.  As far as ATI support goes, your card is ONLY supported by the ATI legacy driver, which does not work with any version of X Server post 2008.  You will need to use the default open source Radeon driver.  There is simply no other solution.

I have the exact same setup in my Linux install, I have a Thinkpad T60p with a Core2Duo T5550 and the ATIx1400 wtih 128MB of VRAM, the external monitor should work, I had mine connected to a projector on the VGA, and I also have a docking station on my desk connected to a 24 inch LCD display.  In order to use the proper resolution on the secondary display, you have to make sure to disable "mirror display" on the displays control under system settings, that will allow your to specify a separate resolution for the secondary display, otherwise your lower resolution on the connected displays (your laptop's internal LCD included) will be the upper limit for the display.  
BTW, secondary display works much better with a docking station, I have one on my desktop I bought for $16 (the docking station for thinkpads are so cheap used), that allows me to use my 24 inch display as the primary display using the DVI connector on the docking station, color quality is perfect. 
The Ubuntu 12.04 built-in ATI card driver is less optimized than the previous proprietary ones from ATI, but it works.
