### Post by waynefoutz on 2011-02-24
Debian and Ubuntu have completely different release cycles. Actually, both are based on Debian testing. The version number on Ubuntu signifies a date. 10.04 is April 2010, while 10.10 is October 2010. The release number means that version most closely resembles Debian testing as it was at that date. Debian testing is a rolling release, which means it's constantly updated. Debian stable is not. Like Ubuntu, it's a snapshot of testing, frozen in time. That's not to say Ubuntu is a version of Debian, it's not. It just shares some basic core architecture. In fact, it's drifting further from it's Debian roots with every release. 

To answer your question, yes, 10.04 is more stable than 10.10. 10.04 is a Long Term Release, which means it will be supported for three years instead of 10.10's 18 month life-span. Every two years, Ubuntu release a more stable, longer supported version. 10.04 is one of them. The next one will be 12.04. If you want to be on the cutting edge, then you can install the releases in between, like 10.10. If you just want to install a version that will last a few years and not be bothered, then an LTS version like 10.04 will probably be more to your liking. It's up to you.
