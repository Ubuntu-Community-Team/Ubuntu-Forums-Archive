### Post by dougfractal on 2007-07-19
```
sudo apt-get install nvidia-glx-new
```

Ubuntu had to create a new nvidia package because some popular cards (440MX) became legacy but were better than the other legacy.

see [http://ubuntuforums.org/showthread.php?t=452879](http://ubuntuforums.org/showthread.php?t=452879) has some more info


> nVidia 8800GTS and nVidia 97.55 driver
The biggest issue is the video card. Installing the nvidia-glx-new (97.55) driver included with Feisty does not install the modules properly, resulting in X not working. A few simple steps are required to get it working again:

Code:

sudo apt-get install nvidia-glx-new

What follows is a shameless rip from this [http://ubuntuforums.org/showthread.php?t=449105&highlight=libwfb.so&page=3](http://ubuntuforums.org/showthread.php?t=449105&highlight=libwfb.so&page=3). Get the driver package from nvidia and extract it:
Code:

wget [http://us.download.nvidia.com/XFree86/Linux-x86/1.0-9755/NVIDIA-Linux-x86-1.0-9755-pkg1.run](http://us.download.nvidia.com/XFree86/Linux-x86/1.0-9755/NVIDIA-Linux-x86-1.0-9755-pkg1.run)
sh NVIDIA-Linux-x86-1.0-9755-pkg1 -x

Copy the missing module and link to it:
Code:

cp -f NVIDIA-Linux-x86-1.0-9755-pkg1/usr/X11R6/lib/modules/libnvidia-wfb.so.1.0.9755 /usr/lib/xorg/modules
cd /usr/lib/xorg/modules
sudo ln -s libnvidia-wfb.so.1.0.9755 libwfb.so

Finally, activate the driver
Code:

sudo nvidia-glx-config enable

Beryl will likely be missing window borders, the following should fix this problem.:
Code:

sudo nvidia-xconfig --add-argb-glx-visuals

After a reboot, X should be fine.

### Post by Astro96 on 2007-07-19
This is happening because you have a CRT and and LCD.  The NVIDIA card automatically makes the CRT the default screen.  For a while there wasn't much you could do but NVIDIA finally came out with a fix.

You should add the following to the device section of your xorg.conf.  Note that you might not have "DFP-0" and "CRT-1" (ie the numbers could be switched or different).

```
Option "TwinViewXineramaInfoOrder" "DFP-0, CRT-1"
```

Take a look here: [http://http.download.nvidia.com/XFree86/Linux-x86/1.0-9625/README/appendix-d.html](http://http.download.nvidia.com/XFree86/Linux-x86/1.0-9625/README/appendix-d.html)

and here:
[http://www.nvnews.net/vbulletin/showthread.php?t=54638&page=12&highlight=crt+default+display](http://www.nvnews.net/vbulletin/showthread.php?t=54638&page=12&highlight=crt+default+display)

Hope that works for you,
Andy

### Post by jsgarvin on 2007-10-21
I was trying to figure the same thing out for my Vostro 1700 Laptop.  Adding this line to the Device section for the video card was all that was necessary for me.  


```
Option         "TwinViewXineramaInfoOrder" "DFP, CRT"
```

Now all the important items (panels, menus, panel applets) are in the laptop's monitor so I can unplug the external monitor and take the laptop off to another room without having to restart X every time.

I also had one instance a few weeks ago where I booted the laptop without the external monitor plugged in (was at a LUG meeting downtown) and the login screen came up on the external monitor that wasn't there. Restarting X corrected it, but still, it took me a minute to figure out what was going on.  I don't expect that will happen anymore with the above setting.
