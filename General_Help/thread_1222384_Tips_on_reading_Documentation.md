### Post by kk0sse54 on 2009-07-24
It depends on what it is. Generally you don't need to read documentation cover through cover however they should always be your first point for reference. For example if you ever need help with a shell command make sure you've read it's man page first or if you're looking to do something like a minimal Ubuntu install make sure you've searched through help.ubuntu.com. However if your using something like FreeBSD you'd better have read through the FreeBSD handbook. That being said I don't search and read documentation on setting up things like Raid just for the sake of reading documentation. 

Again it totally depends on what you're trying to do, just remember that the documentation is there to help you and when done right is a fantastic tool. That being said I wouldn't fret about it too much :)

P.S. I only use the Python documentation for reference purposes
