### Post by monkeybrain20122 on 2018-03-28
From your output
> build.py: error: cannot find source directory (/home/scott/Documents/dupeguru/sphinx-build) 
If you install python3-sphinix from repo that error will go away, make install will complete but your app won't run because the makefile installs the required python packages in a virtual env, once you make install they are no longer picked up. So, if you want to make install you have to install the packages in requirements.txt system wide with pip3, in that case you don't need to install them in  the virtualenv, you can comment out line 51 in the makefile
> ./env/bin/python -m pip install -r requirements.txt
(I think you can build without the virtualenv thing altogether, but I am not going to spend time hacking the makefile)

Tested on Ubuntu 18.04 beta with python3.6, see screenshots  (I changed the prefix to /home/monkeybrain/opt in the makefile instead of /usr/local, so make install just installs in my home/bin and /share instead of the system, that way I can easily delete it after this exercise)



In any case you don't need to make install,  if you want a launcher icon you can just write a script like```

#!bin/bash
cd  /path/to/dupeguru-src  &&
make run

```

Call it dupeguru, make it executable, put in /home/user_name/bin (if not exists create it) then make a .desktop file to invoke it.
