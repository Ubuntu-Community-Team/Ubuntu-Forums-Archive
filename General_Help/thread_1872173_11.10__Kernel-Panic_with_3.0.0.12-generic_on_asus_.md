### Post by makitso on 2011-10-30
> Ubuntu has made me upgrade

The offer to upgrade was optional and you chose to do that.  For users such as yourself, I would recommend waiting for at least 30 days or more into the cycle before doing the upgrade or clean install.  That way, the first round of release bugs can be dealt with.

Kernal panics are difficult to deal with.  It probably is a hardware / driver issue.  Video cards or power support can be sensitive.  Since all the *butu's use the same base the problem may exist there as well.  

Suggestions:

Try running in 2d mode 
Same problem if you boot up on the liveCD?
Any messages in the logs, e.g. dmsg?
Is there a sequence to what you do that causes the panic?

### Post by cwatkins on 2011-11-06
After a lot of research, frustration and hacking,I have made a minor breakthrough.
The key problem with Ubuntu 11.10 on my eeepc1000h appears to be the rt2800 WiFi driver.
After following the instructions here:

[http://michael-peeters.blogspot.com/2011/06/fixing-rt2860-wifi-chipset-under-ubuntu.html](http://michael-peeters.blogspot.com/2011/06/fixing-rt2860-wifi-chipset-under-ubuntu.html)

I have managed to change the driver to the rt2860 driver. The instructions didn't work exactly as described (they may be for a different version - 11.4?) and I had to blacklist all the rt2800 drivers, but eventually I managed to get the rt2860 drivers working.
Whilst using the original rt2800 drivers I would get a kernal panic with an hour of use, sometimes with five minutes. 
I've used the rt2860 for about three days now and I have not had a single kernal panic or any other WiFi problem.
:D
