### Post by drs305 on 2010-09-22
You can modify scripts but that takes a bit (or more than a bit) of work. The easiest way is to add the menuentry to the /etc/grub.d/40_custom file.

The custom file already exists. You just leave the existing lines intact and add the menuentry. It will be incorporated into the Grub menu the next time you run "sudo update-grub". The menu item will appear at the bottom of your Grub menu. If you want it at the top, rename 40_custom to 09_custom, as the scripts in this folder are run sequentially.

Don't forget the last line in a menuentry is:
> 
}

It's often left off when copy/pasting

grub.cnf (or grub.cfg in Grub2 at least in Ubuntu), normally isn't edited as it is modified every time grub is updated. The grub2 developers went so far as to make it un-editable even as root unless you change the write bit.

My scripting skills are pretty marginal, so I only write scripts that I can test on my own machine (so I can keep altering them until they work).

If you need Grub2 to automatically change the device, you would alter the /etc/grub.d/30_os-prober file. You would add a conditional that states something to the effect of "if this title" or "if this partition" or some other variable unique to the specific entry, then "make the device sdXY".

> @drs305, I am trying to edit etc/grub.d/40_custom but its just a 4 line script without any provision to edit sda6, how to go about correcting the sda#.

Open your /boot/grub/grub.cfg. Find the same menuentry that you have to manually edit from the Grub menu. Copy the entire contents of that menuentry to 40_custom, below the existing lines.

The first line of what you paste should start with "menuentry ...." and the last line should be "}" on a line by itself.

Edit the entry if necessary. If you have been editing the entry on the Grub 2 menu during boot, make the same changes here (i.e. change the sdXY value, etc).

Save the file and run "sudo update-grub".
