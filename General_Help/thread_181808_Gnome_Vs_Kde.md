### Post by adamkane on 2006-05-24
The question is more like: Do you like Nautilus or Konqueror? 

There are many more KDE service menus than there are Nautilus scripts available, so I'm tempted to switch to KDE. Instead I've been rewriting the service menus for Nautilus, and this has helped me develop some meager programming skills.

KDE is buggy on breezy, so that's probably the main reason to use Nautilus for now.

amaroK on dapper is a good reason to use KDE though.

It's a personal decision, and you won't find any agreement on the forums.

### Post by elamericano on 2006-05-24
Why not run amarok on gnome? I'm trying out kubuntu now. It's more configurable, but I *had* to configure it to look nearly as good as gnome - still some ugly dialogs (e.g. logoff). Still everything works, and it has some utilities that I will probably continue to use even if I go back to gnome. Yes, it is buggy, but I'll take people's word for now that Dapper will be better.

On the other hand, the gnome file browser was too slow for me. Where's my directory bar with tab completion? They should make it easy to show hidden files and change permissions too. I'll probably try rox if I go back, but apps still use the standard save dialog, which is what I hear most people complain about. Neither one is perfect. It's worth trying both - maybe xubuntu too.

Oh, another consideration is that the Ubuntu community is also much bigger. That usually means faster/better response on the forums, more howtos, etc.
