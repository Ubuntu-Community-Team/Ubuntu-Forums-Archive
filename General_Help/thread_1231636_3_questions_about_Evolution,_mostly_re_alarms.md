> What about HTML emails? It seems like you can view the source on an email you've received, but not one you're composing. and you can add an <HR /> and some basic formatting but that's about it.


HTML e-mails are a general mess because there is no standard of what HTML e-mail clients should support, and they should be avoided if you care about the receiver actually being able to read what you send.

I am personally sick of receiving HTML e-mails designed for people reading them inside web browsers, so they don't render correctly in all e-mail clients.

Evolution will allow you to send HTML e-mails when you create them externally (File-Compose in External Editor).
