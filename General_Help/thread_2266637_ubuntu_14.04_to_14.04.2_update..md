### Post by tgalati4 on 2015-02-24
That is correct.  I'm not sure if there are security improvements in the newer kernel.  That you will have to research in the change logs.  In general, if your system is working the way you want, there is no need, and some risk in upgrading the kernel.  Kernel upgrades are designed to allow newer motherboards to work with an older Long Term Release--something called [Kernel Enablement Stack]("https://wiki.ubuntu.com/Kernel/LTSEnablementStack").

### Post by grahammechanical on 2015-02-24
As kernel 3.13 is supported until 2019 it will get security fixes until then. So, stay where you are. As you have noticed we are automatically moved on to the next point release but we do not automatically get the newer hardware enablement stack with a newer kernel. And that is because the newer kernels are only supported until a few months after 16.04 is released. We, the user have to make the decision.

Regards.
