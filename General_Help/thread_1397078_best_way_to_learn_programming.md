### Post by t4thfavor on 2010-02-02
Find a computer problem that you really hate, or something you do everyday that needs automation. Then find a language you like, and start writing.

Really, the only way to really understand a language is use it to do something with it you already understand.

Look over books on whatever language you choose, and pick one that is engaging, and understandable to someone on your level.

If you find that you are bored while programming, you will not do well, it has to be something you like to do.

### Post by lovinglinux on 2010-02-02
I don't think there is a "best way to learn programming", but if you already have something in mind that you want to do, then will be easier IMO. 

I like to get my hands dirty since the beginning, so I learn by solving small problems and by trial and error.  The main problem with this approach is that, at some point, you you probably find better ways of doing things you have already done and will feel the need to re-write your entire code or completely switch to another language. 

For example, I started with bash scripts because I wanted to automate several tasks on my machine. So whenever I wanted to do something new, I would Google for how doing such thing using bash. Soon I started to combine small snippets of code into more complexe scripts until I had a decent collection of my own scripts. Then I realized that I needed a program with graphical interface to control all those scripts from a single place. For instance, I wanted to manage TV recordings from my browser. So I started to learn xul and javascript, to create a Firefox extension. In the beginning it was basically an interface for all those bash scripts. Since I started to share my extension with other Firefox users, I realized would be nice to provide the same extension for Windows. Additionally, using bash scripts wasn't the best (safer) way of doing things from Firefox. So I started to learn how to do the same things I did with bash, using javascript and re-wrote the entire code.

After a year reading a lot of stuff and re-writing a lot of code, [this is the result ]("http://foxmediacenter.lovinglinux.megabyet.net");)

Anyway, whatever method you choose, the are some things that will make your life easier in the long run.

Best place to start and ask programming questions is the [Programming Talk](http://ubuntuforums.org/forumdisplay.php?f=39) forum. Make sure you read the stickies there, since they have lots of useful information.

After reading lots of documentation, build a development environment, with the tools you need. 

Setup an [svn repository](http://en.wikipedia.org/wiki/Subversion_%28software%29) using [subversion](apt:subversion), so you can easily revert changes during development. You probably re-write your code several times. I recommend using [esvn](apt:esvn) or [kdesvn](apt:kdesvn) (Gnome and KDE respectively). I also recommend [kate](apt:kate) for writing your code.

It would also be advisable to use a [Virtual Machine](http://en.wikipedia.org/wiki/Virtual_Machine) to test your programs, so if anything goes wrong it doesn't affect your system. You can install [virtualbox](apt:virtualbox) for that.

Documentation is also very important. Make sure you bookmark all guides you can find on the web, [add comments to your code](http://en.wikipedia.org/wiki/Comment_%28computer_programming%29), so you don't get lost when it grows, and save every [code snippet](http://en.wikipedia.org/wiki/Snippet_%28programming%29) you can find.
