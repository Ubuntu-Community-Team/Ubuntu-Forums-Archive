### Post by vanadium on 2008-07-29
To clean up your /media directory, unmount first then delete the directory. As Nikitas350 said, directories will be automatically recreated when reconnecting removable devices.

e.g.
```

sudo umount "/media/E Partition_"
sudo rm "/media/E Partition_"

```

Only partitions that you permanently want to be available should be included in /etc/fstab. External drives should never be included in /etc/fstab.

If you want the partition to be represented by an icon on the desktop, mount it under the /media directory. If you do not want an icon on the desktop, mount it under /mnt.

Your /etc/fstab has been badly tampered with. Either the UUID= lines or the /dev/sda5 and /dev/sda6 lines have to go. It is preferred to keep the UUID= lines if these are still valid. To check that, compare the UUIDs given by the command

```

sudo blkid

```

with the ones in your /etc/fstab. If they match, then remove the lines 

```

/dev/sdc5 /media/sdc5 ext3 defaults 0 0
/dev/sdc6 /media/sdc6 swap defaults 0 0

```

from your /etc/fstab.

### Post by drs305 on 2008-07-31
> **vanadium said:**
> 
psydm indeed can help to change fstab without the need to editing it by hand, but it is not foolproof and I just had a thread where it had caused inconsistencies in /etc/fstab.


As you correctly pointed out, removable drives should not normally (if ever) be listed in fstab. 

The problem is not really with pysdm (other than it can't insert UUIDs/labels) - it is with using a static identifier (sdc1, sdf2, etc) instead of using a uuid or label. The problem would be the same whether you used pysdm or manually entered a static designation in fstab.

Again, I'll reassert your main point: don't use fstab to mount removable drives. ;-)
