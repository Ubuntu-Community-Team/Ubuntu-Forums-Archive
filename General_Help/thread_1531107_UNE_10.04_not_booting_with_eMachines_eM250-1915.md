### Post by cariboo on 2010-07-14
The first screenshot tells you what the problem is, the UUID of your boot partition has changed. 

Boot from the Live CD/USB device and once at the desktop open a terminal and type the following:

```
sudo fdisk -l
```

note which partition is the boot partition, it will have a ***** in the boot column. Once you know what the boot partition is, in the same terminal type:

[list=1]
[*] sudo mount /dev/sda1
[*] sudo mount -o bind /dev /mnt/dev
[*] sudo mount -o bind /proc /mnt/proc
[*] sudo mount -o bind /sys /mnt/sys
[*] sudo chroot /mnt
[/list]

press enter after each command. Once you are at the prompt type:

[list=1]
[*] grub-install /dev/sda
[*] update-grub
[/list]

press enter after each command. Once you have finished, press Ctrl-d to exit, then reboot.

**Note:** substitute your boot partition for the one in the example above.
