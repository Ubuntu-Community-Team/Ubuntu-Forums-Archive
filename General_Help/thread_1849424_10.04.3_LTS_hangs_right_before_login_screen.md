### Post by stinkeye on 2011-09-25
I was having the same problem with hanging at the plymouth screen.
I was told it was probably a "race condition" at boot.
The only solution that worked for me was to do a fsck every boot.
Don't know why it works but it appears to slow down the boot process
so everything loads properly

This command will set the disk check at every boot.
Adds about 10 secs to the boot time but at least , for me, booting never failed.

```
sudo tune2fs -c [COLOR="Magenta"]1[/COLOR] /dev/[COLOR="DarkOrange"]sda1[/COLOR]
```

[COLOR="Magenta"]1[/COLOR] makes it scan at every boot.
[COLOR="DarkOrange"]sda1[/COLOR] **Your** Ubuntu partition.

To set back to a scan every 30 boots use
```
sudo tune2fs -c [COLOR="Magenta"]30[/COLOR] /dev/[COLOR="DarkOrange"]sda1[/COLOR]
```
