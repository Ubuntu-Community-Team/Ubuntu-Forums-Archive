### Post by mb_webguy on 2009-04-26
I see a lot of questions about different distros, and they usually concern different applications. This confuses me a bit, because you can install any open-source application (which is almost all Linux apps) on any distribution.

Your choice of distribution shouldn't concern applications, but support.  How well maintained and comprehensive are the repositories?  Does the distribution use a package management system with which you're familiar?  Does it use a rolling release schedule or a stepped release schedule (and which would you prefer)?  How large, knowledgeable, and helpful is the user base?

These are the questions that you should be asking when choosing a distribution.  If you're concerned about applications, then ask about applications.  Linux is Linux, and any Linux application will run on any distribution.

### Post by Titan8990 on 2009-04-26
My breakdown of major Linux distros:

Ubuntu - Oriented towards ease of use and ease of transition from Windows. Typically sacrifices stability for up-to-date software

Debian - Similar to Ubuntu but with better documentation (but smaller support group). Typically sacrifices up-to-date software with bleeding edge

SUSE - Oriented towards workstations in business environments. SUSE locks in updates to versions of the OS including kernels.

Gentoo - Everything compiled from source (with the help of a package manager). The best choice for learning how things work within Linux and for people who like the highest level of customization. 

Arch - Similar to Gentoo only with binary packages instead of sources.


Most Linux enthusiasts will end up trying most or all of the major distros. The above is actually the order in which I tried distros. Gentoo is the distro I stuck with.
