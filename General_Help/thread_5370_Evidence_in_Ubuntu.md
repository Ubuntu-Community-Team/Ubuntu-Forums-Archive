### Post by ashley_v on 2004-11-18
If you want nice Evidence than compile it from cvs.  I like evidence but it's more for looks IMO.  It's functional as a file manager but it depends on how you build it.  For instance if you build with no kde or ark support then you will more than likely have no option when you right click to extract archives or what not.  There are many other options to use when building it but the important thing is to build just the stuff you need.  So if I only run kde then why would I want gnome support?

So what it comes down to is if someone built a evidence package for Ubuntu and it made it in the repo then it may not be built with the things you have.  Be it as it may it will still run but prebuilt evidence packages I found were lacking archive and image support as well as some other stuff.

Evidence is great to use with E but there are other file managers out there that are 10 times more useful.

As for DR-17 well they got a lot of folks hopes up with that but last I checked it was broke and always have been broke in cvs so you can't even run a pre-version of it.  I doubt there will even be a dr-17 the E team may change the schema altogether.
