### Post by colonelpanic2409 on 2008-08-08
EDIT: Thought I would let the world know that my problem is solved. The Acer Aspire 5520 has a 'feature' that allows you to de-activate the trackpad by pressing F7, which I obviously did as described in the text below. Simply pressing it again in Vista turned the trackpad back on! A bit of a funny feature for the firmware to have, surely it could only ever cause confusion like this, but anyway - problem solved. *phew*

Hello Ubuntu community,

After installing Ubuntu onto an Acer Aspire 5520 and using Wubi, I have reached a stage of wanting to uninstall it. From the off, the WiFi didn't work. I managed to get that to work using someone's instructions and madwifi, but then the resolution wouldn't go above 800x600, so I got half way through somebody's instructions to fix that, when I had to restart part way through their instructions since I was stuck in the terminal (after pressing ctrl-alt-F1, and ctrl-alt-F7 wouldn't take me back to X).

On rebooting, the trackpad didn't work. The trackpad no longer works in Vista either.

This is a terrible mess, in order just to have my trackpad back, and a computer that I can rely on not to break (it had been fine for months before this little adventure), I want to uninstall Ubuntu and get just Vista back.

The 'Uninstall-ubuntu' program in the ubuntu folder on my HD under Vista did not work however. I clicked 'Allow' to run it, but nothing appeared. I therefore followed the manual uninstall instructions on the wiki, and Ubuntu appears to be gone completely, which as far as I am concerned now is good.

So, can anyone tell me how I can have a working trackpad?

Thank you for your time and any help you can give.

### Post by FXEF on 2008-08-08
Sorry to hear you are having problems with Ubuntu.

In Vista:
 Click on the Start button.
 Go to Control Panel.
 You will find Programs there.  Click Programs.
 Click Program and Features.
 Select wubi and then click the Uninstall/Change button.
You will need to be running as Administrator.
This should uninstall Ubuntu and the Dual boot option. I have never used this method because never wanted to remove Ubuntu from my system, however have talked to others who have used this method and it worked for them.
