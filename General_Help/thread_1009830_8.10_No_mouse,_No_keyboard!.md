### Post by PmDematagoda on 2008-12-15
Allright, then try this:-
1) Kill the X-Server with:-
```
sudo /etc/init.d/gdm stop && killall Xorg
```

2) Make a new X configuration file with:-
```
Xorg -configure
```

3) Copy the new configuration file over with:-
```
cp /root/xorg.conf.new /etc/X11/xorg.conf
```

4) Restart GDM with:-
```
sudo /etc/init.d/gdm start
```
and see if the mouse and keyboard starts to work properly now.

Lots of trouble there! :)

I chose the easy solution and switched back to 8.04. That's not trouble free either, but far less than 8.10 (Which has some nice features and some things work better than in 8.04, but some important programs were just "impossible" to get in working order).

My own setup is keyboard and mouse with a bluetooth usb pin, and it took me some time to figure out how to make it start up without the hassle of replugging the pin to be able to log in.

After the first start I do this from a terminal:

```
sudo apt-get purge bluez-audio bluez-cups bluez-gnome bluez-utils
```

and it starts nicely every time, just by moving the mouse and pressing any button on the keyboard. You could disable (but not remove!) the bluetooth manager applet as well.

My history with Ubuntu is that I tried 7.10 but it wouldn't install. However 8.04 installed nicely. Guess my computer was too new for 7.10! :)
