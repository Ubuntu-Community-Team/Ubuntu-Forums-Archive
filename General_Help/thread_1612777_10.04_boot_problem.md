### Post by sikander3786 on 2010-11-03
Hi.

Dell most probably has got an Intel Graphics Chip, right?

Press the Shift key as soon as the computer starts. Once you see the Grub menu, highlight the 1st entry, press 'e' to edit it. Navigate to the text "quiet & splash", delete it and type "i915.modeset=0 xforcevesa" instead. Press Ctrl + X to continue boot.

If it works, you can later make the changes permanent by editing /etc/default/grub but first make sure it works.

If "i915.modeset=0 xforcevesa" doesn't work, try "nomodeset" and let us know the results.

### Post by bwestover on 2010-11-04
Yeah, that could be doing it.

Found a few references to that error

[here]("http://www.spinics.net/lists/xorg/msg49727.html") and [here]("http://lists.debian.org/debian-kernel/2010/03/msg00107.html")

Which kernel are you using?

you can run 'uname -r' to get at that quickly.

Do you still have the version prior installed? Perhaps try selecting that one from the Grub menu instead and see if its more stable.

Also, you can search a ways back in your messages log.. in /var/log there should messages.1, message.2 etc... going further and further back.

You could try searching each of these to see how far back that error goes

```

cat messages.1 | grep "registered panic notifier"

```
