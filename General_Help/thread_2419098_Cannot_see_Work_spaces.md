### Post by Dennis N on 2019-05-16
In default Ubuntu, workspaces are accessible in from the Activities overview by moving cursor to the screen's right edge where they are revealed; you can click on the empty one to use it, or drag an existing window onto it to make it active. Also you are can use this to switch between workspaces in use.

I think an easier way is to install a gnome shell entension to switch workspaces from the panel - like what you are used to in other distros. There are several available. I use one called "Workspace indicator" by fmuellner. However, none of these show a preview of what's on each workspace. I also use one called "All windows" which lists all open windows in all workspaces and allows switch between them. This one names each window.

That said, out of old habits, I usually rely on Alt-Tab to switch between open windows  There is a gnome shell extension to make Alt-Tab include windows on all workspaces.

Experiment with the different ways, and find one that works for you.

### Post by dragonfly41 on 2019-05-16
If docky is in 18.04 this has a Docklet option .. Workspace Switcher (they appear as a grid at bottom of page).
That is what I'm using right now, looking for ways to automate workspace switching by commands.
So far I found that devilspie does this and also   CompizConfig Settings Manager

But the easiest way is the manual method of mapping any open Window to a Virtual Workspace
 is to right click on the top bar of the application Window, to see this ..


Always on Top
Always on Visible Workspace
Only on This Workspace
Move to Workspace Right
Move to Workspace Down
Move to Another Workspace >
