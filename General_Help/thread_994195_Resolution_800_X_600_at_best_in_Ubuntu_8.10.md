```I think this is what you need, tell me if I'm wrong.
### Post by Chunky Dunk on 2008-11-26
I had to manually edit the "Screen" section in my /etc/X11/xorg.conf to get the higher resolutions.
The relevant part:
```
Section "Screen"
	Identifier	"Default Screen"
	Monitor		"Configured Monitor"
	Device		"Configured Video Device"
	DefaultDepth	24
    SubSection     "Display"
        Depth       1
        Modes      "1280X1024" "1024X768" "800X600" "640X480"
    EndSubSection
    SubSection     "Display"
        Depth       4
        Modes      "1280X1024" "1024X768" "800X600" "640X480"
    EndSubSection

    EndSubSection

### Post by JKyleOKC on 2009-01-25
Try changing the DefaultDepth value in xorg.conf from its current value of 24 to 15. If that doesn't work, try a value of 8. In my similar situation, the value of 15 worked to allow 1024x768 resolution, which was all I wanted.

These values are the number of bits per pixel, and thus determine the number of possible different colors that the system can display. The value of 24 allows 16 million colors while 15 permits only 32,767 colors and 8 allows only 256. The 32K value is almost indistinguishable visually from what you see with 16 million, but 256 is so few that you find definite "banding" effects so it's not at all useful except as an emergency fix.

The xorg system underwent some significant changes between 2007 and 2008, and it's my conclusion that some of the changes impacted its detection of video card attributes in such a way that it decides there's not enough video memory present to show all the colors that XP allows...
