First off on the hardware. If you get that mobo, might I recommend getting 800MHz RAM. Also: 1033MHz ram does exist too. You might try swapping out a mobo that supports this. This is assuming you don't have other reasons for choosing slower ram. 

Second, are you on a specific budget. I'd definitely suggest that a quad core processor would find use in your specific line of work of audio processing. There are several nicely priced AMD quad cores [and some not so ... ]

Third, I would definitely recommend more RAM starting off, even if its going to be Linux. I think 4GB would be a good amount for a 64bit processor [if you are going to put a 64 bit os on it]. Even still, for audio processing, more ram may be of benefit. 

If you are under some kind of budget cap here, take the advice here with a grain of salt. At the very least however, I'd recommend swapping out an alternate mobo and some faster ram. That shouldn't be a deal breaker. If you can possibly get a quad core, that would be nice too, if it can be had for a song.

That said, I typically only play games on Windows if I can't get them easily working on Linux. I have to say that you should cover your self. Set up a dual boot, throw in a shared hard drive for data [ntfs or something like that]. And use whatever operating system you have to. 

Getting a working audio system on Linux will be challenging, but definitely rewarding when you do succeed as it will be more stable. Just to be on the safeside, have a dual boot of Windows to run those things that are absolutely necessary.
