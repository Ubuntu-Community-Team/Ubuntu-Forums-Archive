### Post by P4man on 2010-05-01
you already removed the incorrect entries it seems.

Can you still not mount the drives in nautilus by clicking them?

As for the one(s) you cant mount, possibly its a windows partition and windows is hibernated, or the file system is "dirty" if windows didnt shut down properly. Ubuntu will not mount such volumes to avoid corrupting them. Boot windows, run chkdsk, and do a proper shut down.

If you have other problems, ill need some more specifics.

### Post by badperson on 2010-05-01
I was able to add all of them to /etc/fstab using that second tutorial. 

> # mounting photos partition
/dev/sdb1  /media/Photos ntfs-3g defaults,locale=en_US.UTF-8 0 0
# mount files partition
/dev/sdc2  /media/files ntfs-3g defaults,locale=en_US.UTF-8 0 0
# mount music partition
/dev/sdd1  /media/music ntfs-3g defaults,locale=en_US.UTF-8 0 0
