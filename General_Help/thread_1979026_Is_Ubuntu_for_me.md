Unity probably is a little resource intensive for an 8 year old laptop.  You can install the latest Ubuntu though, and use the gnome classic session to avoid using Unity. If you want to get a new computer anyway, I endorse Lenovo whole heartedly, though if you get one of the ideapad laptops, research compatibility.  Generally it's fine, but sometimes they have problems with Ubuntu.  I think the thinkpad line is generally safe.

### Post by Curious00 on 2012-05-13
[QUOTE=startac]does Ubuntu use a journaling system like Mac HFS or SGI XFS [/quote] 

Ext3 and Ext4 file systems are both journaled. Ext2 is not.

[QUOTE=startac]does it properly recognize and utilize RAM, cores, GPUs, etc- all natively or with customization? [/quote]


Ubuntu should automatically recognize and utilize all of your RAM, and all of your processor cores.

You can open /proc/cpuinfo with a text editor to see a list of all the CPU cores that Ubuntu recognizes, and statistics about them (The first is number 0, the second is number 1, and so forth). You can type "free" at the terminal command prompt to see how much memory the system is recognizing.

Does that laptop of yours have more than one video GPU? I haven't used a computer with more than one, so I couldn't answer your question from experience. However, my experience with Ubuntu tells me that you ought to be covered, here. The default video drivers ought to be pretty good. You can get an even better boost of performance if you've installed a special video card made by ATI or Nvidia, by going to the "System Settings" and seeing whether there are any "additional drivers" recommended for you.

[QUOTE=startac]Is there a utility program like Activity Monitor available?[/quote]

It's called "System Monitor" and can be found with the dash, or you can type at the terminal prompt: gnome-system-monitor

### Post by kurt18947 on 2012-05-14
Historically Thinkpads have been excellent Ubuntu machines; things just work.  I have an R61 & and X61.  All the 'special' keys like sound and wifi work as advertised.  One problem is laptops have 'hybrid' graphics where a laptop will have 2 graphics chips.  One is typically Intel which is less capable of high end graphics but less power hungry and a second is Nvidia or AMD chip for 3D graphics and gaming.  This on-the-fly switching between graphics systems is not supported in Linux by manufacturers.  There are community efforts underway and some success but it's a work in progress AFAIK.  I have no first hand experience so I'm just repeating what I've read.
