### Post by MrAlex on 2012-01-12
Hi guys,

I've recently decided to give Linux (Ubuntu 11.10) a chance for my home computer (switching from Windows 7) and I need to come up with alternatives for most things I normally use in windows. So I installed  and got cracking... for the first time in 6 years there's a linux on my PC agan :)

Well, I need to come up with a replacement for yahoo messenger. It's quite important for me. Now I ran into 2 options and both of them are unsatisfactory... first one is the thing that installed by default (empathy I think) and then there's pidgin (I used that on windows for a while, so I'm sticking to that for the moment).

Both of these however have the following shortcomings:
- there's no way to access the online log (yahoo saves all chats on the web so I can for example see on my home PC the log of a discussion I had from my work PC). This is rather major...
- most avatars are not showing... they do show up when the user comes online (but it's not shown if the user was online before me). Not a problem if I can get all 50 or so contacts I have to quickly quit messenger then log back in when they see me coming online :)
- no "photo share". While file transfer is supported there is no photo share feature. Quite annoying really.

Now what would a seasoned linux user do in this situation? Surely reporting these as bugs to pidgin and empathy is not going to matter much (they are obvious bugs, I can't imagine they are not aware of them). Second option would be for me to try and fix that myself, which is probably beyond my skills.

The third option I can see sounds a bit complicated... I was thinking of trying to run yahoo messenger on wine (or something similar if it exists). However that sounds a bit daunting (searching for wine in the software centers showed 4 "wines" + a few frontends). Also I think the colorful yahoo messenger would look quite nasty in the rather spartan looking ubuntu...

What do you guys think I should do? Try the wine approach? Do I have another option?
