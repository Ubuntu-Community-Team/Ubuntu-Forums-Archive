If the arrow keys don't work, try plugging in a spare keyboard, and select generic pae kernel. If that doesn't work, try it in recovery mode. Or boot of live USB and backup files & reinstall ubuntu.

Did you change any system files by any chance? An incorrectly edited system file might prevent the linux kernel from booting.
Either that, or there is something wrong with grub itself. You might have a broken BIOS, so if none of that works, you can check your manufacturers website on how to flash your BIOS.
It could be a hardware problem, in which case, you might have to replace the motherboard.

I can't really tell for sure, just try what I told you and report back to us.

### Post by ajgreeny on 2012-09-17
Was the problem after an update which gave you the new 3.2.0-30 kernel?
If you have a USB keyboard make sure **legacy USB** is enabled in the BIOS.
If it is a wireless keyboard try using a cable attached keyboard instead.

Try using the "Previous Linux version" line in grub which should then take you to another menu with previous kernels listed.  Perhaps there is a problem on your machine with the 3.2.0-30 kernel.
