### Post by H4rm0ny on 2006-12-26
Just to say that I've now managed to solve this issue and to post how I did it to anyone that wanders into this thread by googling for the problem.

I don't know the ins and outs of it, but it seems that I was locked out from using the display size that I wanted because I wasn't using the frequencies that my monitor (a Samsun 913N 19") supports. I went to the manufacturers website, looked up the model and noted down the horizontal and vertical refresh rates. I then manually typed these into /etc/xorg.conf and restarted X using Ctrl + Alt + Backspace. Came straight back up in 1280x1024. Also got the refresh rate up to 75Hz and the screen is now so sharp it hurts.

I believe it's not wise to meddle with these settings unless you need to as you can damage your monitor. Hope this helps someone and thanks for the nudge that got be started on working this out.

Cheers,

-H.

### Post by robw003 on 2007-02-06
Hello all,

I think I have found the solution.

The changes you made will not be usable/visible in the controls for screen resolution until you change your session type.

It appears that Edgy Eft defaults to "last session" when logging in.

To change this, select Options from the bottom left of the login screen
Click on Select Session
Change to a desktop type -- I picked Gnome
This made the new resolution available.
I have actually changed my default to be Gnome rather than the last session

Oh, by the way this also works if you are using Ubuntu under Parallels on the Mac.

Note for parallel users: Make sure you have a custom resolution defined that matches your desired screen resolution or you may get a Xserver start failure since there is not a matching resolution available.
