### Post by oldfred on 2014-09-19
Moving the start of a partition causes issues with both Windows and Linux.

With Windows the partition boot sector has to have the same start and size info for the NTFS partition as the partition table. You have to run chkdsk to update that info if it changes and may have to do that from your separate Windows repairCD or flash drive.

With Linux you may have to reinstall grub, but if UUID does not change (it should not) then it may still work.

But moving partitions around like that is higher risk. Any powerfailure or interruption will totally corrupt system, so good backups are essential.
