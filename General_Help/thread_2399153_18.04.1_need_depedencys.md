### Post by dpCools on 2018-08-22
Found Fix need to add universe to sources.list

Update your installation -- even a fresh installation will have a few out-of-date packages

sudo apt-get update && sudo apt-get upgrade
Reboot
sudo reboot
Log-in to your pihole
Back up your sources.list file
sudo cp /etc/apt/sources.list /etc/apt/sources.list.bak

To add the repository, edit the file using nano (or your favorite editor)
sudo nano /etc/apt/sources.list
Find the line
deb [http://archive.ubuntu.com/ubuntu](http://archive.ubuntu.com/ubuntu) bionic main
Append universe -- this instructs apt-get to first look for packages in the main repository then the universe repository

deb [http://archive.ubuntu.com/ubuntu](http://archive.ubuntu.com/ubuntu) bionic main universe
Save the file
CTRL + O then ENTER when prompted for the file name (autofilled /etc/apt/sources.list)
Exit nano
CTRL + X
Update your sources with the new repositories
sudo apt-get update

### Post by deadflowr on 2018-08-22
You ran in to this:
[https://bugs.launchpad.net/subiquity/+bug/1783129]("https://bugs.launchpad.net/subiquity/+bug/1783129")

fwiw, you can add repositories with add-apt-repository command like
```
sudo add-apt-repository universe
```
Last I checked it only applies to a single repo at a time, so if you need to enable more than one such as adding universe and multiverse then you run it for each individually.
then run apt update, as usual.

You can also, when needed, run apt's builtin edit command
```
sudo apt edit-sources
```
to edit the source list file.
It'll use whatever the default editor you have set.
(The default being nano (usually), but if you set vim or another editor, it'll use that)
