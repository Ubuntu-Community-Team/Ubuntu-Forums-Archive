### Post by matsuzine on 2010-10-31
Hi,

Gnome Partition Manager is gparted, a graphical partition editor that uses the libparted library and gtk for a graphical user interface.  

kde Partition Manager is also based on libparted, but uses the KDE graphical components ( which are based on the Qt gui library, not gtk ).  

The reason there are two is because generally users who are on gnome or kde don't necessarily want to download all of the libraries for the other desktop just to run an app. Sometimes the application looks very out of place, as well. 

On linux, you have many choices of desktops environments and window managers.  There are some that are very lean and lightweight ( fluxbox, openbox, jwm ), some that are intermediate ( xfce, lxde ), and some that are more robust ( gnome, kde ) and lots that I didn't list.  They are all pretty different, and have varying degrees of ease of use.
