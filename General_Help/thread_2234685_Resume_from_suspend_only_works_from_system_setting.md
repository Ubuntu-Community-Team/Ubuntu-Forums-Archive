Graphics: Quadro FX 3700M/PCIe/SSE2. 
Monitors: Internal LGD as DFP-0 and external Samsung Syncmaster monitor as CRT-0. (MetaMode 3520x1200)

The symptoms are easy to reproduce.  If I suspend by clicking the gearwheel icon and then select 'suspend', the system suspends correctly and I can then close the lid.  When I later reopen the lid, after a moment I'm back where I was - perfect.

BUT if I just close the lid, the system seems to suspend correctly, but then on lid open, it powers up ok, but the internal LGD is disabled, all my virtual screens are on the external monitor, some with odd window decoration. There is also a popup saying 'Could not switch the monitor configuration - could not set the configuration for CRTC 632'.

If I then use the Nvidia X Server Settings tool/X Server Display Configuration page and select DFP-0, I find the Resolution is set to 'Off'. If I reset it to 'Auto' we are pretty much back to where we were before standby and the window decorations are normal again but not an ideal scenario.

My question is - is there any way of changing the lid close script (?) to act like the 'Suspend' button instead ?
