Failed attempts prior to the above success included editing /etc/X11/xorg.conf to have an InputDevice entry with Option "MapButton*" "disable-mouse", Option "StartMouseEnabled" "no", && a ServerLayout InputDevice "Joystick0" entry without "SendCoreEvents", which left my machine unbootable. =( Required a boot-disk to repair. ### Post by Tripmonkey_uk on 2011-03-22
The pads that I'm using are [Dual USB Gamepads which use a single USB port]("http://panda-usb.com/en/panda-usb-shop/2-usb-accessories/5-input-devices/39-dual-usb-gamepads.html") (playstation style with rumble support).

Had the same problem on Ubuntu 10.10 and decided to try deleting the **50-joystick.conf** file first as pipstuart suggested.

Opened up a terminal and entered..

> sudo cp /usr/share/X11/xorg.conf.d/50-joystick.conf /usr/share/X11/xorg.conf.d/50-joystick.conf.old
This backed up the old file first (just in case) and changed the name to 50-joystick.conf**.old** so it wouldn't conflict.

Then I did..
> sudo rm /usr/share/X11/xorg.conf.d/50-joystick.conf
Which deleted the file.

A quick restart and the pads worked without a problem.

Cheers to everyone for posting your results on here, you made it a lot easier to get this sorted out :D
