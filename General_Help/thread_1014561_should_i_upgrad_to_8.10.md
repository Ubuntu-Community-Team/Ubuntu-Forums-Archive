### Post by jerome1232 on 2008-12-18
This has a whole lot of personal prefrence written on it.

uprading to 8.10
pros: Newer versions of software, so possibly new features (and maybe new bugs of course)

cons: More bleeding edge and therefore less apt to be stable

sticking with 8.04
pros: It's LTS so you will be getting major bug fixes and security updates for quite a while yet to come (I forget how long LTS is supported)

It's been around for more than 6 months now and has the cinks worked out, much more stable.

cons: Well if some software has new features that you need you can only hope it get's backported or manually install it (either by adding a repo or manually downloading a .deb or compilling it yourself)

That's my summary, I have a desktop running 8.10, a laptop running 8.04 and a server running 8.04. I have to say I love many of the improvments in 8.10. Like for instance in 8.04 gedit would crash if I tried to save to a mounted sshfs share. In 8.10 gedit handles this scenario nicely.
