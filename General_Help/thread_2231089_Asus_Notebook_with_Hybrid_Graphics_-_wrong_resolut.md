### Post by evanrevans on 2014-08-06
It appears to me to be a kernel problem.

 If I go into Advanced Options; 
with 3.13.0-32-generic, the problem exists
with 3.13.0-30-generic and below, the problem does not exist.

The lspci -k output in -32 shows both Intel and Nvidia drivers in use.
The lspci -k output in -30 shows only the Intel driver in use.

So it appears to be something with the way the current kernel handles the hybrid graphics.
