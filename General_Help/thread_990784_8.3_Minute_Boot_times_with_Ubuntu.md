### Post by ciscosurfer on 2008-11-23
The timing sequence seems drastically off, at least it appears that way at first glance.  And I agree that initramfs is hanging for a very long time. This could, however, just be an oddity related to using a core 2 quad setup.  Here is a link to Intel on [Core 2 Quad architecture]("http://www.intel.com/products/processor/core2quad/index.htm").

I've highlited in red where the jumps occur.  On any other system (and perhaps yours as well) this is usually an indication of a hardware/kernel/timer problem or mismatch.  I think this problem is directly related to your cpu setup.

I will continue to search on this issue and in the meantime, it would be a good idea to also check on Launchpad for similar issues related to: tsc (time stamp counter), timer, core 2 quad cpus, hpet, jiffies, acpi_pm, boot, initramfs.  It might be possible to fix the boot issue by changing the clocksource parameter.  You can see what your current clocksource is by running the following in a terminal```
  cat /sys/devices/system/clocksource/clocksource0/current_clocksource

```And these are the options available to you```
  cat /sys/devices/system/clocksource/clocksource0/available_clocksource
