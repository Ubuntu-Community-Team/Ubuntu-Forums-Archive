### Post by kay-man on 2009-05-25
I don't think ISO image is read-only, but the drive /media/disk, which obviously means it can't be written.

I remember seeing a few threads in the past weeks with people reporting similar problems. You could try a quick search and see if that brings you anything helpful.

Is the flash drive always read-only, or can you write to it first? Try:

```
cat /etc/mtab
```

after you insert it, and post the results so we can see the options it was mounted with.

### Post by kay-man on 2009-05-26
None that I'm aware of. I guess you need hal and udev, but they'd be running anyway, so nothing special there.

There seem to be many people with these kinds of problems, and this imho is one of the things that is still needlessly complex and way too unreliable in linux.

I use udev rules and autofs to automount a number of phones and usb sticks always to the same locations, and it works really well. It's way more powerful than windows, but it just takes to much fiddling.

I don't see my mother doing that, so in that sense there's still a way to go I suppose.
