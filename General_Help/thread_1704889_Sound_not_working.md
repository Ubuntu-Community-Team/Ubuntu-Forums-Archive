### Post by smurphy_it on 2011-03-11
Sounds like it was muted.

Try right clicking on the sound icon, and select Mute (should already be selected).

If that doesn't do it, try alsamixer from a terminal.

Otherwise, you sound driver might of been removed.

First step, would be determining the audio hardware you have.

```
lspci | grep [Aa]udio
```

### Post by smurphy_it on 2011-03-11
OK found a previous problem of mine ages ago, different laptop and stuff, but some of the commands should still be relevant.

Followed this recipe found in
[http://www.ubuntuforums.org/showthre...ight=intel-hda](http://www.ubuntuforums.org/showthre...ight=intel-hda)

"How to fix:

1) You need the latest ALSA driver. Do this:

1. Get build-essential and linux-headers packages.
2. Get alsa-driver package from alsa-project.org
3. Unpack it.
4. Run "./configure --with-cards=hda-intel"
5. Run "sudo make"
6. Run "sudo make install"
7. Edit /etc/modules and add "snd-hda-intel"
8. Reboot."

It was that simple!! After that make sure your channels are not muted.

### Post by smurphy_it on 2011-03-11
Similar to what you done before... You download the alsa source packages, and then in a terminal you type ./configure --with-cards=hda_intel
followed by sudo make and sudo make install

Do that for each of the three packages.  If all goes well, you'll have sound by the end of it... otherwise, you have more issues.

As you loaded a kernel, you could always just not use that problematic kernel, and use your old one (modify /etc/default/grub ) then run ```
update-grub
```

If for example, on your grub menu you have for argument sake, 5 entries...

0)  defective kernel with sound not working
1)  defective kernel with sound not working [recovery]
2)  older kernel
3)  older kenel [recovery]
4)  windows 7

Right now, the default grub entry is 0, you can edit the /etc/default/grub file and change the default 0 to default 2

run update-grub, and on next reboot it should automatically log you back into the workable kernel.
