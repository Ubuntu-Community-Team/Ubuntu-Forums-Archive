### Post by solitaire on 2009-10-21
[https://wiki.ubuntu.com/LTS](https://wiki.ubuntu.com/LTS)

Basically LTS versions are supported for 3 years for desktop and 5 years for servers
Normal releases are only supported for 18 months

Next LTS is 10.04 Lucid Lynx (LTS is released every 2 years)

LTS are mainly aimed at OEM makers and large organisations that need to have a standard across the board. (the change every 6 months is not good for large orgs or pc makers as they keep playing catchup and if you've ever worked in a large org it takes months to get an upgrade tested and deployed and if you did it every 6 months you'd never finish). So they have a known working stable OS to work from.

### Post by Mighty_Joe on 2009-10-21
> **eival said:**
> im basically trying to avoid major bugs/glitches like when i first started using Ubuntu..

Because of the practically infinite combinations of hardware and software, there are no guarantees that a particular release will be stable on your hardware.  
For example, I had a major problem upgrading to 8.04 LTS.  I had skipped 7.10 and an Xwindows change introduced in that version rendered my hardware inoperable. I couldn't roll the upgrade back, so I was stuck with a computer that would hang within a minute of booting.
Your best bet is to keep updated backups and thoroughly test a new version before committing to using it.  Personally, I plan to install 9.10 to a flash drive and run like that for a week or so to make sure I cover all my bases.  Running a Live CD or installing to a spare hard drive are other options.
