### Post by QIII on 2010-03-26
I have installed the last several versions of Ubuntu in less that 30  minutes without fuss, and install applications from the repos with the  click of a mouse.  

While there are many third party applications that I have installed  which took somewhat more effort, that is due to the nature of the  application provided by the software developer.  Many of them do not  provide a great deal of Linux support.

As a certified MS solution developer, I have to jump through hoops to get the vaunted pat on the back from MS.  I necessarily create applications that work with Windows-compatible installers because, frankly, that is where the money is.

MS does not create applications that work with MS except their own products.  Nor does MS create drivers for OEM hardware that work with Windows.

That is the responsibility of the software developers and OEMs.  If no OEM driver is available for Linux, it is because the OEM has not prepared one.

If a user is required by his work environment or by his personal tastes to use a product that is only compatible with Windows, then so be it.  That is no fault of the Linux community, nor is it any indictment of the Linux community.

I, and millions of others, use Linux as my primary OS on my desktops very happily.  Most casual users of a desktop can also do so very easily.  Most users can happily go through life without ever even knowing the CLI exists.

And there are things that may need to be done in Windows that can only be done in the command line.

There are Windows help forums aplenty.  All is not lollipops and sunshine with Windows either.

I don't bang Windows.  It is a capable OS that has a large user base, and it enjoys the attention of software purveyors and OEMs who go where the money is -- as a business should.

But to say that Linux is not for the casual user is simply not true.
