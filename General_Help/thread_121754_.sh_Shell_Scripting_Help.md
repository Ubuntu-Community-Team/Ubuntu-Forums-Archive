### Post by jrib on 2006-01-25
$1 signifies the first argument, $2 the second, etc...

so inside your script should be:
```
btdownloadheadless.bittornado $1
```

### Post by Iandefor on 2006-01-25
That would entail adding a commandline argument (which is actually what you're describing... it never hurts to know the correct terminology). To refer to an argument passed on the command line, you would refer to it by a dollar sign and the number of the argument. For example, to refer to the first command line argument passed, use $1. So, to use the example you gave, to use a URL passed at the command line, you would use

```

btdownloadheadless.bittornado $1
``` Does this help?

_jason: from the ABS: 
> $0 is the name of the script itself, $1 is the first argument, $2 the second, $3 the third, and so forth. After $9, the arguments must be enclosed in brackets, for example, ${10}, ${11}, ${12}. Section is 4.4.
