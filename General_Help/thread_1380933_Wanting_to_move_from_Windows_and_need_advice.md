### Post by Mark Phelps on 2010-01-14
Being a long-time user of different flavors of MS Windows and Linux, and doing daily support for a variety of clients using both, please allow me to pass along some observations.

First of all: "Linux is not Windows".  This is really important to understand.  While Linux can do nearly everything that MS Windows does (some will even claim that it can do more), is does not do everything the same way that MS Windows does.

This means that any migration over to Linux is going to require folks change the way they use their computers on a day-to-day basis.

The more your daily work is browser-based, especially if it's NOT Internet Explorer, the more you will find the Linux world nearly identical in functionality to the MS Windows world.

Conversely, the more your daily work is MS Windows-software based (e.g., Internet Explorer, MS Outlook, MS Word, etc), the more difficulty you will have learning to do the same stuff but using Linux apps instead of MS Windows apps. 

And, don't get me started on Wine ... I've had nothing but problems with it and really don't want to "go there".

An additional complication in migrating to Linux from MS Windows is that of hardware drivers.  Few manufacturers provide Liunux-specific drivers, so it really is a guess regarding whether or not a particular machine will have all the hardware up an running after an Ubuntu install.  That's one major reason that booting and running from a LiveCD of the distro you want to use will provide you valuable insight into what problems (if any) you will have using Ubuntu on a specific machine.

MS Windows typically does not require much intervention when installing new apps or hardware.  You stick in a CD, it autoruns, you click OK to a few things, and ta da, a few minutes later your app and/or hardware is installed and working.

Ubuntu provides much the same experience with apps, especially now that Ubuntu 9.10 has gone to the sofware center ... but hardware is an entirely different story.  Anything that doesn't work by default could be anywhere from trivial to impossible to fix.

This means, that unless an Ubuntu install works perfectly afterwards, there could be considerable hacking (meaning on-site support) required to get it working.  If you're not willing or able to provide that support, unless the LiveCD shows no problems at all, I would suggest you NOT force an Ubuntu install onto a machine.

As to the Windows viruses, there are a variety of free downloadable antivirus products available that can be used to clean Windows machines.  Avira, Kasperky both provide free ISO downloads; Symantec provides free trials of Norton 2010.  

You should really try cleaning your MS Windows machines before forcing Ubuntu on folks -- especially if you're not prepared to help them over the initial hurdles.
