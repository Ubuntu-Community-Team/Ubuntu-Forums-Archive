### Post by eriqjaffe on 2008-09-23
Xubuntu is the official light-weight fork of Ubuntu, but it's not really all that light.  That being said, I've been running Ubuntu (regular Gnome desktop) on an older laptop with 512MB of RAM and the base idle system uses up about 180ish megs, so there's a fairly sizeable amount of spare RAM.

If you **really **want to maximize your RAM usage, you should go with a command-line install and a window manager like Openbox, Fluxbox,  or IceWM.  My desktop (using Openbox, LXPanel and Cairo-Dock) only uses about 85mb at idle.  The downside is that you don't get all the control panels and GUI config tools.  Check out psychocat's tutorial for a bare-bones install:

[http://www.psychocats.net/ubuntu/minimal](http://www.psychocats.net/ubuntu/minimal)

### Post by snowpine on 2008-09-23
512mb is plenty for Ubuntu (with desktop effects disabled) or Xubuntu. If you are using Ubuntu on your desktop, I think there is something to be said for using Ubuntu or Xubuntu on the laptop, so that all of your skills transfer over.

I think Fluxbuntu is overkill, or rather, *under*kill, for that machine. Crunchbang is in the same spirit, but more up-to-date (it's based on 8.04 instead of 7.10 like Fluxbuntu) and has more familiar applications (Firefox, Openoffice, etc.). If you do some research, you'll realize there are many alternatives lighter than Ubuntu, however, the lighter you get, the more you'll have to learn. SliTaz, for example, is a fully modern Linux that uses about 25mb at idle. However, it has a much steeper learning curve and lacks many of the features you're used to (wireless and printing don't work out of the box, Openoffice isn't in the repositories, etc). So basically, it is a trade off between familiarity and performance. I think something like Ubuntu + Openbox (which is basically what Crunchbang is, or you can "roll your own" like Eriqjaffe suggests) hits the "sweet spot" and is a perfect compromise for my needs.
