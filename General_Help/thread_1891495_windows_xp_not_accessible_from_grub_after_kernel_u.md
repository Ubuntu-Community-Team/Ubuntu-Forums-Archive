### Post by oldfred on 2011-12-05
You have a strange way of booting. Normally you install grub2's boot loader to the MBR. You have installed grub2 to the PBR or partition boot sector of Ubuntu and are using the Windows boot loader to chainload into grub2. The install of grub2 to a partition is not recommended as it converts to blocklists and is less reliable. You may have to reinstall on grub update.

Your Windows is missing in grub since grub does not see all the boot files. You now are missing boot.ini? Do you have a backup or you can manually create a boot.ini as it is just a text file.

From a Windows XP disk:
BOOTCFG  /rebuild  # rebuilds boot.ini

A discussion about the Bootcfg command and its uses fix boot.ini
[http://support.microsoft.com/kb/291980](http://support.microsoft.com/kb/291980)
[http://pcsupport.about.com/od/fixtheproblem/ht/repairbootini.htm](http://pcsupport.about.com/od/fixtheproblem/ht/repairbootini.htm)

If editing windows files like boot.ini in Linux:
(Use nano instead of gedit, it's better for dos-style linebreaks)
Linux, of course, uses only LF as newline and DOS is expecting CR/LF so text files look wrong in DOS.
New versions of gedit have an combo box under save as to cchoose windows format.


How to edit the Boot.ini file in Windows XP
[http://support.microsoft.com/kb/289022/](http://support.microsoft.com/kb/289022/)

Every boot script posted with a dual boot of Windows XP has a copy of boot.ini in it.

### Post by oldfred on 2011-12-05
I do not think you can boot into Windows without a boot.ini. Glad you got it solved. 

It is very easy to reinstall a boot loader to the MBR, so if you want to remove Ubuntu you just reinstall the Windows MBR. You can also install lilo's boot loader to the MBR from Ubuntu as it also chainboots to the boot flagged partition just like Windows. If grub is updated in your Ubuntu install and the file is moved, then you will have to use the liveCD to reinstall grub2 to the PBR.

How to restore the Ubuntu/XP/Vista/7 bootloader (Updated for Ubuntu 9.10 - grub2) - talsemgeest
[http://ubuntuforums.org/showthread.php?t=1014708](http://ubuntuforums.org/showthread.php?t=1014708)


Restore basic windows boot loader - universe enabled
Simply open Synaptic and Settings > Repositories and tick the box against the Universe repo in the Ubuntu Software tab. Close that window and click on reload before installing lilo with Synaptic or command line.
sudo apt-get install lilo
sudo lilo -M /dev/sda mbr
May show error messages about the rest of lilo missing, ignore, we just want MBR with bootloader.
