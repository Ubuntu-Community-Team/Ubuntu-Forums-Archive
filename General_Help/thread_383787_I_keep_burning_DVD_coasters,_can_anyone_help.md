### Post by mac.ryan on 2007-03-13
Difficult to say what's wrong (at least for me). Spare suggestions (you might have tried them already though...):

1. Are you burning DVDs or CDs? I would suggest you to burn a CD. There is really pretty much everything you need in it, it is smaller file to download and burning CDs is an operation less prone to errors than writing DVDs (although these should be the exception, not the rule!).

2. Once you have downloaded the ISO file, just to be sure, make a MD5 [checksum]("http://en.wikipedia.org/wiki/Checksum") (it's a procedure to verify the integrity of the download). You normally find the MD5 checksum signature needed for the procedure into the same directory you downloaded the ISO from. For your comfort, this is one possible URI: [ftp://ubuntu.univ-nantes.fr/ubuntu-cd/edgy/](ftp://ubuntu.univ-nantes.fr/ubuntu-cd/edgy/)

3. If you pass the test, burn the CD with the "verify" option on, so that the programme will verify the file you downloaded and the CD you burnt, match. (Of course the CD has to be burnt in the "burn ISO image" mode... you must not simply burn a CD with the ISO file on it....)

At this point you can try to reboot the system from the CD. If your misfortune will keep on, it is reasonable to assume there is a problem with your hardware (something crashing the installation process, or even errors during the CD reading).

Best luck!
