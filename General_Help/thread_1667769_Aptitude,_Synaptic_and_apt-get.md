### Post by Nytram on 2011-01-15
Apt-get is a CLI version of Synaptic, some people prefer using the terminal. Aptitude is an alternative to Apt-get, going back a few years Aptitude was able to handle dependencies better, but I believe they're pretty much on par now.

### Post by ZebaSz on 2011-01-15
Aptitude and Synaptic are front-ends for apt-get, which allow you to be more selective with installs, updates, and removals, and do them simultaneously. Synaptic uses GUI, while aptitude uses the command-line.

### Post by karthick87 on 2011-01-15
[COLOR=Red]**apt-get**[/COLOR] is a command-line tool. It's handy if  you know the exact package name of what you want to install and don't  want to spend time clicking through a GUI to get it.
  
[COLOR=Red]**aptitude**[/COLOR] is very similar to apt-get, and I've heard  that it deals better with crazy dependency situations. Which one is  really better is debatable.

[COLOR=Red]**synaptic**[/COLOR] is a low-level GUI. This is a good choice  if you are a fairly advanced user but are not comfortable with  command-line utilities.

Have a look at this [COLOR=Red][**link**]("http://askubuntu.com/questions/76/whats-the-difference-between-package-managers")[/COLOR] for more informations.
