### Post by fluffman86 on 2009-11-22
See my post at the top of this page.  You do *NOT* want to fool around with UNR.  I mean, it's *great* for small screens.  Really, I use it everyday on my Dell Mini 9 with a 9 inch screen.  But I would *HATE* trying to use it on my 21" desktop monitor, or even on my old 15" laptop.

You want to get the normal desktop version.  It's called Desktop as in "Desktop (for normal users) vs. Server (for power users)" not Desktop as in "Desktop vs. Laptop."  In other words, the desktop version works on laptops.

Check out:
[http://www.psychocats.net/ubuntu/installing](http://www.psychocats.net/ubuntu/installing)
For detailed instructions on how to properly install Ubuntu.

I would recommend reading over that, then check out:
[http://www.psychocats.net/ubuntu/installseparatehome](http://www.psychocats.net/ubuntu/installseparatehome)

For installing Ubuntu with manual partitions.  The first one will be easier *now*, but the second one will save you time and trouble in the future if something goes wrong.  

*IMPORTANT*: I only recommend the second option if you have at least a 30GB hard drive in the D610.  I know some of them came with 20GB hard drives, in which case I would still recommend the first link I posted.  But if you have a 30 or 40 GB hard drive, then go with the second link, with a few changes.

1.  In the second link, I recommend creating at least an 8 GB / (or root) partition if you have a 30GB drive, and if you have 40 GB or more, then make it a 10 or 12 GB root partition.  You'll also want a 1 GB swap partition, with the remainder for /home.

2.  Also in the second link, they recommend ext3, but ext4 is newer and faster.  When choosing the partition type, select ext4 instead of ext3.  

If you have any more questions, feel free to post them here. :)

### Post by fluffman86 on 2009-11-23
Your CD is bad.  If you're downloading it on Windows, then download it again, and when it's done get winmd5sum from

[http://www.nullriver.com/downloads/Install-winMd5Sum.exe](http://www.nullriver.com/downloads/Install-winMd5Sum.exe)

and scan the file.  It will give you a long string of characters.  Compare them to:
8790491bfa9d00f283ed9dd2d77b3906

That's the one for the 9.10-desktop-i386.iso.  More md5sums can be found at:
[http://releases.ubuntu.com/releases/9.10/MD5SUMS](http://releases.ubuntu.com/releases/9.10/MD5SUMS)

 If the numbers don't match, you should redownload, or use bittorrent.  Bittorrent checks the files automatically.

Edit: concerning Feisty Fawn: You *really* need to back that up and reinstall with the latest version.  There are some pretty serious security bugs in that version (particularly in Firefox) that are no longer being fixed because that version is no longer supported and no longer gets updates.

### Post by sgosnell on 2009-11-23
I agree that you have a bad CD.  It could be either because of a bad .iso download, or from a bad burn.  Either is possible, and probably equally likely.  Download the desktop .iso file again, and check the md5sum against what the website shows it should be.  If it checks, meaning you got a good download, then burn the .iso to CD as slowly as you can stand, faster burn means greater chance of errors.  Once you get it burned, boot from the CD and before you try to install select "Check disk for errors".  If it checks ok, then try the install, otherwise do another burn.  Nero can be problematic sometimes, so you may want to install ImgBurn, which has worked better for me in the past, although I haven't used Windows in some time.
