### Post by 3rdalbum on 2009-08-21
You cannot distribute the Adobe Flash plugin without Adobe's permission.

And you can't distribute a Linux kernel with a pre-linked proprietary driver. Linking GPL code with non-GPL code is generally against the license, but the Linux kernel developers generally look the other way for personal use. But if you link GPL with non-GPL and distribute the result, the FSF will come after you.

**Basically, you need to sell the computers to the shop with plain Ubuntu installed, and then go there and install the necessary Flash plugin, graphics drivers etc.**

### Post by paulderol on 2009-08-26
Dell already pays various other vendors for the right to distribute various drivers and other software goodies like flash. Having already paid a bulk fee as a manufacturer, they install the software according to the agreement they signed. the difference is contractual agreement and scale. you would never be able to negotiate the price that Dell can for each install, and you don't have to, since you can go and do it on-site as a service in addition to providing the saleable machine/OS combinations. For you, it's almost optimal to not distribute those bits of software.
