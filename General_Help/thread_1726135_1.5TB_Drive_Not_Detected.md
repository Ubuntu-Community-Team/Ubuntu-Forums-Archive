```

*-ide UNCLAIMED         
       description: IDE interface
       product: Marvell Technology Group Ltd.
       vendor: Marvell Technology Group Ltd.

```

Maybe you will need to find a specific driver for that device.

### Post by srs5694 on 2011-04-11
It's possible that physically moving the cable for that drive from one motherboard connector to another one will fix the problem. Some motherboards, and particularly those that support more than four hard disks, use a primary hard disk controller that's build into the chipset and a secondary controller than handles the extra disks. Sometimes Linux drivers exist for one controller but not the other, and when this is the case, moving the connections around should fix the problem. Consult your motherboard manual for details about what's what, or just try some random swaps.
