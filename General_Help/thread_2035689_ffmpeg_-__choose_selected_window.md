### Post by Vaphell on 2012-07-31
should be doable if you don't plan to move the window around

according to this:
[http://ubuntuguide.org/wiki/Screencasts](http://ubuntuguide.org/wiki/Screencasts)
```
-s <SIZE> -i :0.0+<OFFSET>
```, eg
```
ffmpeg -f alsa -ac 2 -ab 128k -i pulse -f x11grab **-s 320x240** -r 30 **-i :0.0+10,20** -acodec libmp3lame -vcodec mpeg4 -vtag xvid /home/user/capturedvideo.avi
```

to fill it you need to determine the size and location of the desired window.
**wmctrl -lG** shows list of windows with geometry data but with grep it can be narrowed down to the desired one, eg
```
wmctrl -lG | grep Firefox
```
id's are unique and would be best fit to extract the proper line. There are few other tools that can access window data, like xprop or xwininfo, eg look at
```
xprop -id <hex_id>
xwininfo -id <hex_id>
```

if you read that extracted line into variables you can use them to generate proper ffmpeg parameters
```
read -r x y w h <<< $( wmctrl -lG | grep Firefox | awk '{ print $3, $4, $5, $6 }' )
ffmpeg ... -s ${w}x{$h} -i :0.0+${x},${y} ...
```


to make it easier to select proper window you can create a menu from the wmctrl output.
```
windows=()
while read -r l
do
  windows+=( "$l" )
done < <( wmctrl -lG )

select win in "${windows[@]}"
do
  read -r x y w h <<< $( echo $win | cut -d' ' -f3-6 )
  echo "${w}x{$h} -i :0.0+${x},${y}"
  break
done

ffmpeg ... -s ${w}x{$h} -i :0.0+${x},${y} ...

```
also xwininfo and xprop allow you to select the window with mouse, in case you want to guify the script/not have terminal open.

### Post by Vaphell on 2012-09-04
well, info about existing windows is not served in a pretty form
xprop output is not too useful, but xwininfo and wmcontrol are quite nice. Either way you need to get your hands dirty to extract the coordinates and size from one of these.

```
0x052000b8  0 852  723  958  1023 ubuntu Google - Mozilla Firefox
  win_id    ?  x    y    w    h    host    win_name
```
you need x,y,w,h for -s ... -i ..., right?

the snippets i posted in #2 using wmctrl, with some commentary
```
[COLOR="SandyBrown"]# create array and populate it with data about known windows[/COLOR]
windows=()
while read -r l
do
  windows+=( "$l" )
done < <( wmctrl -lG )

[COLOR="SandyBrown"]# use windows array to create menu
# type in number to select, selected row is copied to win variable[/COLOR]
select win in "${windows[@]}"
do
  [COLOR="SandyBrown"]# extract x,y,w,h from selected item (cols 3-6) stored in $win and break the loop to continue[/COLOR]
  read -r x y w h <<< $( echo $win | cut -d' ' -f3-6 )
  echo "${w}x${h} -i :0.0+${x},${y}"
  break
done

[COLOR="SandyBrown"]# put x, y, w, h to actual use here
# ffmpeg ... -s ${w}x${h} -i :0.0+${x},${y} ...[/COLOR]
```

this uses pretty much the same mechanism, x, y, w, h variables are filled with proper columns of wmctrl output, here for the Firefox window
```
read -r x y w h <<< $( wmctrl -lG | grep Firefox | awk '{ print $3, $4, $5, $6 }' )
echo -s ${w}x${h} -i :0.0+${x},${y}
[COLOR="SandyBrown"]# ffmpeg ... -s ${w}x${h} -i :0.0+${x},${y} ...[/COLOR]
```

these two are copy-paste ready, you can highlight select them and middle-click paste in terminal to get them to run and have it show required values. The only thing left would be to add the complete ffmpeg command to use those values and maybe wrap it up in a nice script.

example how to get info from xwininfo:
```
xwininfo -id <some existing win_id> | awk '/Absolute.*X/{x=$NF;}; /Absolute.*Y/{y=$NF;} /Width/{w=$2;} /Height/{h=$2;} END{ print w"x"h, ":0.0+"x","y; }'
1920x1171 :0.0+0,29
```
