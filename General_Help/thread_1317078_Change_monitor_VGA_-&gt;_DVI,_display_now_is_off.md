### Post by UranUtan on 2009-11-06
Hi,

Using Ubuntu 9.10 x32, video ATI Radeon 9200. Compiz off, Resolution 1680 x 1050, using VGA cable. Everything is working OK.

Then I replace the monitor which native resolution is 1920 x 1080 and it only has a DVI cable.

When booting with new monitor, the login screen is not displayed, the screen remains black. If I boot with Live CD then the screen is recognized at 1920 x 1080.

What is the issue and is there a way to fix it?

Thanks very much in advance for any help.

### Post by UranUtan on 2009-11-06
Strange: the problem solved itself.

Using another computer, I was browsing the Net searching for the solution. The computer where I had changed the monitor was still having a blank (completely black) login screen. But may be Ubuntu was reconfiguring itself in the background.

Then I hit Ctrl-Alt-Del and the login screen appeared with the correct resolution. Not sure what was going on but at least now the issue is fixed.

EDIT: Update #2
The issue has nothing to do with Ubuntu. It was just the video cable. The monitor accept analogue input signals. But I used a DVI-D single link cable (not sure why Samsung gives this cable !?!). Using an analogue cable, the problem is solved.
