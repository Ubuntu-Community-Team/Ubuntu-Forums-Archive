### Post by drs305 on 2008-07-11
> **phildaman46 said:**
> Bump again....

Sorry you aren't getting an answer. 

Although external USB drives should mount without an fstab entry, here is how I mount my **external usb ntfs drive**:

First, I labeled it - *ntfslabel <label> <device>* (the ntfs partition on this drive initially identified as sdg2 - note later the computer identifies it as sdc2):
```
ntfslabel eagle.ntfs /dev/sdg2
```

Then I created a mount point:
```
sudo mkdir /media/ntfs.external
sudo chown *username* /media/ntfs.external

```

Next I added an fstab entry:
```

LABEL=eagle.ntfs /media/ntfs.external ntfs-3g defaults,uid=1000,gid=46,umask=022 0 0

```

It is mounted on boot. The owner of the mount point is *username* and the group is *plugdev*.

When I run the mount command, this is what it shows:
```

/dev/sdc2 on /media/ntfs.external type fuseblk (rw,nosuid,nodev,noatime,allow_other,default_permissions,blksize=4096)
```

[B]
For an internal ntfs partition:
[/B]

Create a mount point as above. I use the */media/ntfs* folder; note the standard convention to mount an internal drive is to put it in /mnt .

My fstab entry looks like this:
```
UUID=56B07FD96 /media/ntfs ntfs-3g defaults,uid=1000,gid=46,umask=022 0 0

```

When booted, the mount command gives this entry for the internal ntfs partition:
```

/dev/sda6 on /media/ntfs type fuseblk (rw,nosuid,nodev,noatime,allow_other,default_permissions,blksize=4096)

```
