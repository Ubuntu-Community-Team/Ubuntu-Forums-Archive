### Post by Morbius1 on 2010-01-12
I'm not sure I understand the question but the line you had in the original fstab for sdb1 will still work in this one. The reason the entire fstab from the original won't work in the new setup is because all the UUID numbers have changed. I'm guessing because you reformatted the partitions.

/dev/sdb1 /media/1TBsamsung ntfs defaults 0 0

Just so you know what this is going to do, it will make root the owner but in the default is an implied umask=000 which will give everyone read/write access.

Are you saying that this doesn't work?

[I]BTW, that's a rather curious HowTo you referenced. He has you chmod'ing a mount point for an NTFS partition. Which is fine it won'thut anything but it will be completely overriden by the mount command in fstab. The other thing he has you do is create a samba user called "family" but all the shares allow guests which don't require a username or password. There's a lot of curious things in there.



[/I]

### Post by Morbius1 on 2010-01-14
Please note that the following is only speculation on my part. The inner working of when and how services are started are not my area.

I'm wondering if samba is starting before the network is up. Perhaps a script added to /etc/network/if-up.d that would execute only after the network is up would resolve this matter. For example:

Open an editor as gksu and create a little script:
```
#!/bin/sh
/etc/init.d/samba restart
```Save the file to say: /etc/network/if-up.d/sambaup
Make the file executable: sudo chmod +x /etc/network/if-up.d/sambaup 

Like I said, I don't know the cause nor do I know if this will fix it so consider this a very verbose bump ;)
