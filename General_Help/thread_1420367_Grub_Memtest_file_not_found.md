I recently pulled out some old Crucial Ballistix ram to put into a machine that could use it. This ram used to give me problems (because of the motherboard, voltages, I don't know), but it seems to be performing well for now.

I want to run memtest to make sure, but when launching either memtest entry in Grub, I get:
```
error: file not found
Press any key to continue...
```

The Windows 7 RC also recently expired, so on the same machine I deleted and the 7 and XP partitions. Upon doing a fresh XP install, I was having problems reinstalling grub. I assume this to be related. In grub, issuing root (hdx,y) and setup (hdx) appeared to work (i.e. found the stage files and completed setup), but upon reboot I was only greeted to the grub command line. I fixed this by reinstalling from the live cd while not formating the partitions (but I think this really only did a complete reinstall while keeping home intact)

Anyway, I have a separate partitions for /, /home/, and /boot and am using grub-pc (1.97~beta4-1ubuntu5). When trying to reinstall grub, I found that the only stage1 file was contained in '/boot/grub/boot/grub' perhaps causing my problem. I tried copying this boot folder to the root folder creating '/boot/boot/grub'

I understand that I could just run a memtest86+ livecd, but I'd rather sort out the apparent problem I'm having with my seperate boot partition.

Contents of /etc/grub.d/20_memtest86+
```
#!/bin/sh
set -e

prefix=/usr
exec_prefix=${prefix}
libdir=${exec_prefix}/lib

# older versions of grub2 do not have this yet (LP: #459080)
if [ ! -e ${libdir}/grub/grub-mkconfig_lib ]; then
    echo "no grub-mkconfig_lib, exiting"
    exit 0
fi

. ${libdir}/grub/grub-mkconfig_lib

# We can't cope with loop-mounted devices here.
case ${GRUB_DEVICE_BOOT} in
  /dev/loop/*|/dev/loop[0-9])
    exit 0
  ;;
esac

if test -e /boot/memtest86+.bin ; then
  MEMTESTPATH=$( make_system_path_relative_to_its_root "/boot/memtest86+.bin" )
  echo "Found memtest86+ image: $MEMTESTPATH" >&2
  cat << EOF
menuentry "Memory test (memtest86+)" {
	linux16	$MEMTESTPATH
}
menuentry "Memory test (memtest86+, serial console 115200)" {
	linux16	$MEMTESTPATH console=ttyS0,115200n8
}
EOF
fi
```
