### Post by geoffmacartney on 2008-06-12
When you start up synaptic, click the Search button, and 
type "build-essential" (without the quotation marks) into the
text box marked Search, then press the Search button. This should bring back a list of packages, with build-essential itself among them. Careful with the spelling, it is "build" not "built", and no "s" at the end of "essential".

Click on the checkbox against build-essential in the list of packages and hit "Apply".

### Post by ibuclaw on 2008-06-13
Hmm.. OK,

Try this:
```
echo '#include <stdio.h>
/* confdefs.h.  */
#define PACKAGE_NAME ""
#define PACKAGE_TARNAME ""
#define PACKAGE_VERSION ""
#define PACKAGE_STRING ""
#define PACKAGE_BUGREPORT ""
#define PACKAGE "Xdialog"
#define VERSION "2.3.1"
#define PACKAGE "Xdialog"
#define VERSION "2.3.1"
/* end confdefs.h.  */

int main ()
{
  printf("hello world\n");
  return 0;
}
' > /tmp/test.c
echo -n ""

```
Followed by
```
gcc -Wall -o /tmp/test /tmp/test.c
```
and
```
/tmp/test
```
It should compile just fine and produce the output "hello world"

If not, run this by copying the lines and pasting it in the terminal (**Ctrl+Shift+V**):
```
sudo sed -i '/cdrom/s/deb/#deb/' /etc/apt/sources.list
sudo apt-get update
sudo apt-get dist-upgrade
sudo apt-get install dpkg-dev g++ gcc libc6-dev make gdb
sudo apt-get install build-essential
echo -n ""

```
Then try:
```
gcc -Wall -o /tmp/test /tmp/test.c && /tmp/test
```
Again. And now it should compile.
