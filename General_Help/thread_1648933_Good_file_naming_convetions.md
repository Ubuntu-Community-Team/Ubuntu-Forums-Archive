### Post by echo314 on 2010-12-19
I have a lot of files that I store for my school program. Generally, the hierarchy goes Quarter - Course - Module - SubModule, and within each SubModule (ex. Learning Objectives) are questions and answers or other notes.  

Lets say I have a document that contains my notes for submodule Learning Objectives for Module 1.0, Course 110, Quarter 1. I'm unsure if I should name the file something like "1-110-1-LearningObjectives" or if I should just give it the last-most hierarchy name, calling it "Learning Objectives" but differentiating between it and all the other modules Learning Objectives by maintaining a folder hierarchy, so there are multiple files called "Learning Objectives" but to get different ones, I would go to, say, the Module 2 folder instead of Module 1.

### Post by edhplus2 on 2010-12-19
Personally, I would use the second method, using the directories and subdirectories in the file structure as my filing system.
I do a lot of genealogy and follow a similar method. Surname, Given Name, then folders for censuses or sources, etc., for each person.
I think the hyphenation system would be hard to remember and maintain after a while.
