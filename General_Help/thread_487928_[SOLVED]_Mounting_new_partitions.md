### Post by dreadlord_chris on 2007-06-29
First you need to make a couple folders to mount the new partitions in:
```

sudo mkdir /media/**NAME1**
sudo mkdir /media/**NAME2**

```
You'll probably want to use more descriptive names then **NAME2** & **NAME2**
Next we'll find out what the UUIDs for the new partitions are:
```

ls -l /dev/disk/by-uuid/ | grep sdb

```
This should spit out 2 lines - copy the long string that comes before the ->
i.e.
```

lrwxrwxrwx 1 root root 10 2007-06-16 15:49 efe89fd9-58ab-456c-9881-528a9149e3c7 -> ../../sdb1

```
you would copy efe89fd9-58ab-456c-9881-528a9149e3c7. Now edit fstab as root
```

gksudo gedit /etc/fstab

```
At the bottom add the following lines - replacing **<<UUID>>** with the UUIDs copied earlier, and **NAME*** with the mount points you picked:
```

# /dev/sdb1
UUID=**<<UUID>>** /media/**NAME1** ext3 nouser,defaults,noatime,auto,rw,dev,exec,suid 0 2
# /dev/sdb2
UUID=**<<UUID>>** /media/**NAME2** ntfs-3g defaults,locale=en_GB.UTF-8 0 1

```
Make sure that you get the right UUID matched to the correct /dev/sdb* then save the file. Now back in the terminal:
```

sudo mount -a

```
if you get any errors post back....
Otherwise you should be good to go...
