Hardware:

 Additional PCIe Ati/AMD Radeon HD 2400 Pro Graphics Card.

ConsoleKit:
I have to use the original /etc/ConsoleKit/displays.d/Local.display.
Else I get an error in Xorg.x.log: 
```

(EE) Screen(s) found, but none have a usable configuration.
```
The Ati screen remains blank (with Ubuntus upstart Loading splash)

I can even copy a working Local.display to SeatX.display... no chance. (Yes, I edited the corresponding "DisplayTemplate="-line in the session file)
So now I use Local.display for all seats:
```
[Display]
Type=X11

[X11]
Exec=/usr/bin/Xorg $display -layout $layout $novtswitch $sharevts -br -verbose -auth $auth -nolisten tcp $vt
```

This way I can start all those different Seats with one display-Template:
/etc/ConsoleKit/sessions.d/Seat1.session :
```
[Session Entry]
Name=Local
Type=LoginWindow
Description=Local Login Screen
DisplayTemplate=Local

[Local]
display=:1
vt=vt8
layout=atiLayout
sharevts=
novtswitch=
```
/etc/ConsoleKit/sessions.d/Seat2.session :
```
[Session Entry]
Name=Local
Type=LoginWindow
Description=Local Login Screen
DisplayTemplate=Local

[Local]
display=:0
vt=vt7
layout=viaLayout
sharevts=-sharevts
novtswitch=-novtswitch
```

As you can see I use sharevts/novtswitch for the via screen. Without this the via screen doesn't come up. Adding it to the ati screen lets crash this one...

And because both seats are next to each other and both seats share one sound card so audio for both is played simultaneous:
I added ```
load-module module-alsa-sink
``` to /etc/pulse/default.pa 

Side effekt: I had stutter with Pulsaudio and TeamSpeak3 Microphone. So I had to kill Pulseaudio to use ts3. Now I can use alsa dmix from ts3 while Pulseaudio is online. :-)

The other things are the same as here:
[http://multiseatonlinux.blogspot.com/2010/06/part-1-setting-up-base.html](http://multiseatonlinux.blogspot.com/2010/06/part-1-setting-up-base.html)
```
