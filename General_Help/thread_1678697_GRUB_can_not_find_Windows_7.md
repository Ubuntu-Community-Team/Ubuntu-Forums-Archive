### Post by drs305 on 2011-01-31
There is nothing in the os-prober section, indicating that the script which searches for other operating systems may not be working:
> ### BEGIN /etc/grub.d/30_os-prober ###
### END /etc/grub.d/30_os-prober ###


First, see if this command works:
```
sudo os-prober
```

If it appears to run, make sure the grub script is executable:
```
sudo chmod +x /etc/grub.d/30_os-prober
```

If os-prober is not installed, try installing it with:
```
sudo apt-get install os-prober
```
and then try updating Grub2 again:
```
sudo update-grub
```

### Post by Mark Phelps on 2011-02-01
> ... does this (Loader) load my windows 7 as is or does it load the start up for it... This i am checking because the boot up text print out i did earlier showed that windows 7 was in sda 2 not 1 as well as that i know there is a partition on my computer in case windows is infected that came with the laptop rather then a recovery cd.

Windows 7 installs by default into two partitions, on your box, sda1 sda2.  The sda1 partition contains the boot loader files; the sda2 partition contains the Win7 OS files.  Because there are "pieces" of Win7 in both partitions, os_prober thinks they are both OSs.

The correct boot selection for Win7 is sda1.

### Post by cdaringe on 2011-08-27
hi all, just FYI, i had a very similar problem.  I ran the scripts posted above, and found that the script outputted *3 drives* and i only had two physical drives.

*sda
*sdb
*nvidia*(#&$@(&%

i had installed via the alternate CD, was having issues, and someone recommended that i toggle a RAID config option to bypass the error i was experiencing.

well, after reading the info on grub2 [here]("https://help.ubuntu.com/community/Grub2") i found that os-prober (which hunts down all OS's to put into your grub menu) does not search devices flagged with a raid.  So, i found [here]("http://ubuntuforums.org/showthread.php?t=1322108") the command ```
dmraid -rE
``` purges all RAID flags/stuff.  I never had my twin drives setup as RAID, ever, so i felt comfortable to do this.  reran os-prober & update-grub2...bazinga!  my other OS's decided to show up!  happy day.  haven't restarted yet, but i thought it was worth sharing with the crew.

so, if you are having this issue, just do a quick check to see if you have something similar. run ```
sudo dmraid -r
```, and perhaps you can save the headache i went thru. :)
