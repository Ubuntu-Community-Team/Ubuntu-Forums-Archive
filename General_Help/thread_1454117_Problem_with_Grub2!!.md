### Post by drs305 on 2010-04-14
Your updates appear to be changing menu.lst instead of grub.cfg. You first need to confirm which Grub you are using. You can look at the top of the Grub menu during boot (Version 1.96 or later is Grub 2) or you can run the following:
```
grub-install -v
```
Since odd things appear to be happening on your system, I'd trust the version displayed on the menu.

You can change the default OS by manually editing /boot/grub/menu.lst and changing the default from 0 to the appropriate value (count the number of "title" entries in the bottom section, starting the count with 0).

Perhaps a better solution would be to uninstall all grub versions and reinstall grub 2. If you do this make sure you have a good internet connection and steady power source since you will momentarily not have a bootloader.

If you want to do this, here is a link. See post 12:
[http://ubuntuforums.org/showpost.php?p=9107370&postcount=12]("http://ubuntuforums.org/showpost.php?p=9107370&postcount=12")

The procedure is also detailed in the guide linked below as "GRUB2".

If you still need guidance, please post the results of Meierfra's boot info script:
[http://bootinfoscript.sourceforge.net/]("http://bootinfoscript.sourceforge.net/")
