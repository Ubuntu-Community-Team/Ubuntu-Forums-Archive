### Post by meldroc on 2007-02-12
I'm looking for something like Microsoft Project - where I can create tasks, assign them resources, and have the system do automagic scheduling, with nice Gantt charts so I can create sane estimates for my boss at work.

So far, I've looked at KPlato and Project, and those pieces of software are missing features.  Specifically, they're not smart enough to not double-book a resource.  In other words, if I create a bunch of tasks, assign them all to a resource, which I'm assuming can't be doing more than one thing at once, and recalculate the Gantt chart, it schedules everything in parallel, which is great if I can clone myself.  Unfortunately, I haven't mastered that ability, so I would like the project management software to be smart enough to schedule two tasks so their times don't overlap.  The only way to get KPlato or Project to do that is to link tasks, which is a pain, and doesn't really fit my mental model.

So where can I find project management software that isn't totally brain dead?

### Post by TheWizzard on 2007-02-12
KDE has TaskJuggler ([http://www.taskjuggler.org/screenshots_ide.php)](http://www.taskjuggler.org/screenshots_ide.php)). Use it together with KArm.

KArm tracks time spent on various tasks.  It is useful for tracking billable hours and can report the hours logged by task and day.  This time history can be exported to a comma-delimited text file for import into other billing and/or project management tools. 
KArm detects when your keyboard and mouse are idle and can associate different tasks with different desktops, two tools that can help keep the timer running on the correct task.
