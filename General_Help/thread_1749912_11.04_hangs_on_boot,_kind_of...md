### Post by vk1ac on 2011-05-05
Playing around with this problem on a HP dv6000 with the nVidia chip that dual boots with Windows Vista, I discovered if you run the recovery selection in GRUB and then chose to boot with a file system check, I could actually boot Ubuntu 11.04 normally, with full Unity.

Having a Toshiba Satellite laptop with the same dual boot and the same upgrade from 10.10, I found that machine had no problem whatsoever booting Ubuntu 11.04..  No PSoD.

What could it be?  Well...  seeing I could actually boot my HP/Ubuntu if I ran fsck first, I decided to take a page from elsewhere in this forum..

in terminal (no matter how you get to it)..

su -
cd /
touch /forcefsck

reboot

This worked for me.. Yes, it does a fsck on each and every boot, but it does boot and I don't get the PSoD.  It seems to get around the bit that keeps hanging on the purple screen and might work for others....  

Cheers,

for the record, the file /forcefsck is automatically deleted after the filesystem is checked on boot; even setting the file as read-only for root will not save it from being deleted.

To force a check on everyboot (and hence to be able to use my ubuntu 11.04 installation) the following worked for me:

[B]tune2fs -c 1 /dev/whatever

[/B]Since I did this on my SSD drive, the fsck adds only about 2 or 3 seconds to the boot process - something I can live with until this nasty Ubuntu bug gets fixed.

replace the hash mark with your drive number...  works EVERY time...  ;)
