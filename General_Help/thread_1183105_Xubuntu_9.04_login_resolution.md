### Post by mbessjs3 on 2009-06-09
I'm having issues with screen resolution in Xubuntu 9.04.  I have seen many posts discussing the same problem: that the login screen resolution is too high.  I have looked at a variety of methods but still haven't got a full solution.

Current situation:
My xorg settings were configured automatically (as first installed, and as set by dpkg-reconfigure) Login screen resolution is too high (1600x1200) despite laptop screen resolution being 1280x800.  Once logged in, I added

```
xrandr --output default --mode 1280x800
```

to my Autostarted programs.  This sets the correct resolution once I am logged on.  All is good, except for the login screen.  My xorg.conf looks like:

```

# xorg.conf (X.Org X Window System server configuration file)
#
# This file was generated by dexconf, the Debian X Configuration tool, using
# values from the debconf database.
#
# Edit this file with caution, and see the xorg.conf manual page.
# (Type "man xorg.conf" at the shell prompt.)
#
# This file is automatically updated on xserver-xorg package upgrades *only*
# if it has not been modified since the last upgrade of the xserver-xorg
# package.
#
# Note that some configuration settings that could be done previously
# in this file, now are automatically configured by the server and settings
# here are ignored.
#
# If you have edited this file but would like it to be automatically updated
# again, run the following command:
#   sudo dpkg-reconfigure -phigh xserver-xorg

Section "Device"
	Identifier	"Configured Video Device"
EndSection

Section "Monitor"
	Identifier	"Configured Monitor"
EndSection

Section "Screen"
	Identifier	"Default Screen"
	Monitor		"Configured Monitor"
	Device		"Configured Video Device"
EndSection

```

I can't get the new settings to stick.  I've tried running xrandr as root, and as a command in /etc/rc6.d but with no luck.

Alternative solution: 
Many postings suggest changing the xorg.conf.  I looked up settings for the laptop (Packard Bell Easynote R1935) and changed it to the following:

```

# xorg.conf (X.Org X Window System server configuration file)
#
# This file was generated by dexconf, the Debian X Configuration tool, using

#
# Edit this file with caution, and see the xorg.conf manual page.

    Identifier "screen1"
    Device "device1"
    Monitor "monitor1"
    DefaultColorDepth 16

        Depth 8

    EndSubsection
    
    Subsection "Display"
        Depth 15
        Modes           "1280x800"
    EndSubsection
    
    Subsection "Display"
        Depth 16
        Modes           "1280x800"
    EndSubsection
    
    Subsection "Display"
        Depth 24
        Modes           "1280x800"
    EndSubsection
EndSection

```

On first impressions, this has worked and the login and desktop have the correct resolution, but when I watch video files or DVDs the display is just a green screen.

So my question is:  how do I (a) get xrandr to set the resolution for the login screen? or (b) configure xorg.conf so that it doesn't mess up video playback?
