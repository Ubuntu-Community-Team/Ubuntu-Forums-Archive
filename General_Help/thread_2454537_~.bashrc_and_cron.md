### Post by TheFu on 2020-12-01
There are interactive logins and there are batch logins.  cron is a batch login.  I use this at the top of my ~/.bashrc to prevent batch processes from getting screwed with non-batch stuff.  This is the first lines:
```

#If not running interactively, don't do anything
[ -z "$PS1" ] && return
```

After having aliases or functions break other scripts, you'll probably decide it is just easier to keep any needed environment variables inside the scripts directly.  That's what I've learned over 25+ yrs.
