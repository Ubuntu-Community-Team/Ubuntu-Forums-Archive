I'm not sure what else to suggest. I don't use any NTFS partitions myself (or have any to test with), and I'm certainly no expert on the subject, but I"m sure I've mounted NTFS paritions with these options in the past without issue.

To the best of my knowledge (and with a cursory glance at some documentation) these options should be supported by the ntfs-3g driver and unless I've missed something obvious you're using them correctly. So I'm tempted to say that this is a bug and needs to be filed in launchpad. If nothing else you may get a more informed answer there.

Also, you may want to raise the issue over at the [ntfs-3g forums]("http://forum.ntfs-3g.org/") since there are some known premissions-related bugs in the 2.6.24 kernel series (used by Hardy) - I had a quick look myself but none seemed to match your problem exactly.
