### Post by Firetech on 2005-12-05
[QUOTE=ewschone]I did search the forums but didnt find anything related to this... FF 1.5 has been officially released now, so I expected it to be in the repositories, that's all.[/QUOTE]

There are three main reasons that this won't happen in a near future:
1. Firefox is so well integrated in ubuntu that backporting it would result in a lot of other broken apps/packages. Mostly only for gnome, but anyway... this needs to be fixed and examined before backporting.
2. Backporting won't start until FF 1.5 is in dapper, which might take a few weeks.
3. Breezy is stable. That means that the only new versions that enter the official repositories (not including the backports) are security fixes, or very important other updates (like when kopete stopped working with MSN some months ago.)

New KDE releases end up in separate repositories and are not ffected by the state of the kubuntu releases.
