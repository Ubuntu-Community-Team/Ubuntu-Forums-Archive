### Post by bcbc on 2010-08-23
My post wasn't entirely complete. You need to mount your windows partition containing the root.disk under /win

e.g. 
sudo mkdir /media/win
sudo mount /dev/sdaX /media/win
sudo fsck /media/win/ubuntu/disks/root.disk

You need to find replace /dev/sdaX with the partition containing the root.disk.

The thing is, it sounds like the root.disk is OK but it's the attempt to mount the ntfs volume (as the /host ) is what is failing. So I would think it's more a chkdsk fix.
