What is your system config; especially RAM?

In any case, you can speed up open office startup by disabling Java within OpenOffice; most people don't usually need it, and it can be re-enabled easily when required.

Open any OpenOffice document (blank document is also OK), then click on Tools-Options-OpenOffice.org-Java and _uncheck_ use "Use a Java Runtime Environment." then click OK

Gives a great speed boost. There is also a memory management tip, but that will have to wait until you post your system/RAM specs.

### Post by N.N. on 2008-05-19
The recommended minimum requirements in terms of RAM are 384 MB. Sluggish performance, therefore, is to be expected when you run a default install of Ubuntu on a system with only 122.9 MB of RAM.

You can achieve snappier performance, however, by replacing some of the default applications with lightweight alternatives.

Thus, instead of using a full-fledged desktop environment such as Gnome or KDE, consider using just a window manager such as IceWM to control the appearance of windows and so on. Forum member aysiu hosts an illustrated guide on how to install IceWM in Ubuntu on his Ubuntu Linux Resources website: [http://www.psychocats.net/ubuntu/icewm](http://www.psychocats.net/ubuntu/icewm).

Also, instead of using an office-suite such as OpenOffice.org for word processing, consider using a lightweight alternative such as Abiword, as strabes already recommended.

By switching to applications that require less system resources to run, you should be able to make your system perform acceptably. Also, if you really want to optimise your system for speed, forum member K.Mandla has made an extensive guide on how to do just that: [http://kmandla.wordpress.com/2008/05/04/howto-set-up-hardy-for-speed/](http://kmandla.wordpress.com/2008/05/04/howto-set-up-hardy-for-speed/).
