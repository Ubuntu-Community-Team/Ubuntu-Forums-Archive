### Post by logos34 on 2008-06-30
Follow what VMC said.

Open a terminal and start gparted:

gksudo gparted

-right-click on vista partition and 'unmount'

-delete it

Clean up menu.lst and fstab:

gksudo gedit /boot/grub/menu.lst

-remove windows entry at bottom

gksudo gedit /etc/fstab

-remove line for windows ntfs partition.

Now to resize/grow ubuntu to take up empty space you'll have to boot the ubuntu live cd and use gparted there (because / cannot be mounted)

Edit: and backup (home folder), like VMC says.  But grub on the mbr should be fine.  It's pointing to the ubuntu root partition (i.e. menu.lst), not to vista.  As long as the partition # doesn't change, everything should be ol. (abd even if something goes wrong wiht boot, you can always reinstall grub from the live cd.)
