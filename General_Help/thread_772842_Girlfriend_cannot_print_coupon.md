### Post by jeff80534 on 2010-08-20
Ok, I have a strange solution I implemented for this. I have a script to create and start a headless Virtualbox.

a=$1
cp xp.vdi.orig xp$a.vdi
VBoxManage createvm --name "xp$a" --ostype WindowsXP --register
VBoxManage modifyvm "xp$a" --memory 2048 --acpi on --nic1 nat
VBoxManage storagectl "xp$a" --name "IDE Controller" --add ide --controller PIIX4
VBoxManage internalcommands setvdiuuid xp$a.vdi
VBoxManage storageattach "xp$a" --storagectl "IDE Controller" --port 0 --device 0 --type hdd --medium "/VB/Windows XP/xp$a.vdi"
VBoxHeadless -s xp$a -p 3389,3390
VBoxManage storagectl "xp$a" --name "IDE Controller"  --remove
VBoxManage unregistervm "xp$a" --delete
rm "/VB/Windows XP/xp$a.vdi"

Then the virtual box automatically starts a batch file that looks for another batch file on my samba share and runs it.

Then I use the java robot class to control firefox inside of the virtual box and print out the coupon to a network printer. (The coupon printer doesn't like java running, so I break the program into 2 pieces and stop it after hitting print.)

Here is a good hint:

couponprinter /s /v/qn   ---  installs the coupon.com printer silently!!

When it's all put together, it automatically boots up, prints out 2 coupons, shuts down.  Put that in a for loop, and before you know it, you have a stack of great coupons.

### Post by DodgeV83 on 2010-10-26
First of all, I believe Ripfox deserves a congratulations on the recent (within the last two years at least) marriage :)

Now to the matter at hand...I was able to get the original link to print in VMware, after applying the following changes to the .vmx file:

[http://www.woodmann.com/forum/showthread.php?13168-Tips-for-thwarting-VM-detection](http://www.woodmann.com/forum/showthread.php?13168-Tips-for-thwarting-VM-detection)

```
isolation.tools.getPtrLocation.disable = "TRUE"
isolation.tools.setPtrLocation.disable = "TRUE"
isolation.tools.setVersion.disable = "TRUE"
isolation.tools.getVersion.disable = "TRUE"
monitor_control.disable_directexec = "TRUE"
monitor_control.disable_chksimd = "TRUE"
monitor_control.disable_ntreloc = "TRUE"
monitor_control.disable_selfmod = "TRUE"
monitor_control.disable_reloc = "TRUE"
monitor_control.disable_btinout = "TRUE"
monitor_control.disable_btmemspace = "TRUE"
monitor_control.disable_btpriv = "TRUE"
monitor_control.disable_btseg = "TRUE"
```

With the free VMware Player: 

[https://www.vmware.com/tryvmware/?p=player&lp=1&source=web&cd=1&ved=0CCEQqwMoADAA&url=http://www.vmware.com/go/downloadplayer/&rct=j&q=vmware%20player&ei=m6bGTImVI8G78gaUsPDHDw&usg=AFQjCNH1Wx2_2jFhdO7dowNA8gCkOwMaiw&sig2=xVc_gBoTNNdb4kskQ13Dlg](https://www.vmware.com/tryvmware/?p=player&lp=1&source=web&cd=1&ved=0CCEQqwMoADAA&url=http://www.vmware.com/go/downloadplayer/&rct=j&q=vmware%20player&ei=m6bGTImVI8G78gaUsPDHDw&usg=AFQjCNH1Wx2_2jFhdO7dowNA8gCkOwMaiw&sig2=xVc_gBoTNNdb4kskQ13Dlg)


Test 1.  Try to print with the changes, successful

Test 2.  Try to print *without* the changes, on a normal VMware installation, UNsuccessful

Test 3.  Try to print again after reapplying the changes, successful

Test 4.  Try to print in Virtualbox...the website froze Virtualbox, twice.  On the third try Internet Explorer was smart enough to shutdown gracefully with: "A problem with this webpage has caused Internet Explorer to close and reopen this tab"

So yes, it IS possible to print the coupons using VMware, you just need to change a few settings.  Hope this helps!
