### Post by srs5694 on 2010-09-28
> **WorMzy said:**
> ```
sudo lshw -C cpu
```Should give you something like this:

```
       **_width: 64 bits_**

```If the width is 64, I /think/ that means you're capable of running 64-bit OS'. I can't confirm that it works like that though.

I can confirm this; I just did it on a 32-bit installation on a 64-bit AMD Sempron LE-1250 CPU, and it reported "width: 64 bits". FWIW, I didn't need to run it as root (via sudo), although the program complained that I "should" run it that way. Chances are it needs root privileges for some types of hardware detection, but it doesn't need it to determine the CPU's bit width.

> If you can run 64-bit, you can run 32-bit.

This is true of the x86-64 line, which is the most common type of CPU end users will find in laptops today. There are some relatively exotic CPUs for which this isn't true, though. This fact isn't important for Megaptera, but I think it should be mentioned in case somebody else who *does* have such a system reads this thread.
