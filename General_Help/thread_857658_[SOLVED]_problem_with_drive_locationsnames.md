### Post by drs305 on 2008-07-12
It's possible that your partitions changed and thus are incorrectly set in fstab.

Here are a couple of commands to see if the proper partitions are being mounted to the correct location. Post the results if you need help:
```

sudo fdisk -l
cat /etc/fstab

```

If you use UUIDs in fstab:
```

sudo blkid

```
Remounting won't help *if* the problem is in fstab as it will just use the same instructions.

### Post by drs305 on 2008-07-12
I'm wading through this and I see a few mistakes. I'll add to the post as I go, so check the edit time. I'll mark it done at the end when I've gotten as far as my limited abilities will take me ;-)

First, your comments for the linux entries are not correct. Before continuing, save a copy of fstab and then open it for editing:
```

sudo cp /etc/fstab /etc/fstab-backup
gksu gedit /etc/fstab

```
Change the entries in bold. These two edits clarify things and make the references valid but don't actually change anything system-wise:
```

# Entry for **/dev/sdc5**:
UUID=78abc143-3c0b-4657-b19a-1f61fbd4c1da / ext3 relatime,errors=remount-ro 0 1
# Entry for **/dev/sdc6**:
UUID=05da5fed-f792-46c3-bcf6-a96a43fca420 none swap sw 0 0

```


Next, the only NTFS paritition you have in fstab is:
```

/dev/sdb1 /media/Music ntfs-3g defaults,locale=en_AU.UTF-8 0 0

```

It appears your Music partition is now sda1 so this should be:
```

UUID=B0F0B209F0B1D636 /media/Music ntfs-3g defaults,locale=en_AU.UTF-8 0 0

```

I'm using UUIDs for now but labels may be even better once things are sorted out.

You can try unmounting and remounting now and see what is happening. You will get some messages as all the partitions won't unmount:
```

sudo umount -a
sudo mount -a

```


Try these changes and let us know what works, what doesn't, and what you would still like to change.

### Post by drs305 on 2008-07-12
Try unmounting the partitions (right click in nautilus or other file manager and select unmount), rename the existing folders to something else, and then run "sudo mount -a" and see where they mount and what messages you get in the terminal. Don't forget you will get 'busy messages' if you unmount with "sudo umount -a".

Added: When you unmount them, do the folders disappear?

If the above doesn't work, rename one of the empty folders to /media/Music and add this to fstab:
```

LABEL=Music /media/Music ntfs-3g uid=1000,gid=100,umask=022 0 0

```

then "sudo umount -a" and "sudo mount -a"
