### Post by meng on 2006-10-25
Try this:
locate javac
this command should give you the location of your javac (java compiler); it is clearly not in your PATH, because otherwise javac would be a valid command (and your error message indicates it is not).

Let's say that the full path to your javac is:
/usr/lib/j2sdk1.5-sun/bin/javac
(that's where mine is)

Now my suggestion is to put a link to this in your /usr/bin, like so:
sudo ln -s /usr/lib/j2sdk1.5-sun/bin/javac /usr/bin/javac

So that now there is a javac (or a symlink to javac anyways) in your PATH.

Then try using jEdit to compile your program again.

EDIT: above solution also works, by altering your PATH.

### Post by meng on 2006-10-26
cliff notes eh?

Jedit is itself a java app, and requires a java runtime environment (which you had). It also requires a java development kit (which you didn't have) in order to compile the programs you wrote. It tried to invoke the javac (Sun java compiler) but ran into trouble because the compiler did not exist (remember eclipse uses a different java compiler).

So you need to install Sun's JDK in order to use Jedit. Only you hadn't enabled the repositories required to install JDK from repositories.
