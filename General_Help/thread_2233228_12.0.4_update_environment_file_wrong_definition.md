### Post by steeldriver on 2014-07-07
OK, so your original post says usr/environment, which got me confused

**TBH I can't see why you would want that copy command in your /etc/environment (i.e. executed every time a user logs in) - it looks like something that should have been run one time when you installed the (xim?) software**

Regardless of whether you want to correct it or remove it, you will need to use elevated permissions either

```

sudo nano /etc/environment

```
for a CLI editor or (assuming your regular GUI editor is gedit)
```

gksudo gedit /etc/environment

```

The correct name for the Compose file is likely

```

/usr/share/X11/locale/en[COLOR=#ff0000]**_**[/COLOR]US[COLOR=#ff0000]**.**[/COLOR]UTF-8/Compose

```
