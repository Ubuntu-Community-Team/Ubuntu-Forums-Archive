### Post by RMOP on 2008-11-20
Is there anyway that an application expecting a USB device can be transparently redirected to a Bluetooth device instead?

Mobile Broadband devices within Network Manager expect a USB device. When my Nokia phone is cabled to my eeePC, it works fine as a Mobile Broadband modem. Uncabled, it also works fine as a Bluetooth modem, but not from Network Manager.

Might there be a way set up a "virtual" USB port which is actually redirected to a Bluetooth device?

This idea is hardly original. The old OS/2 VModem created a virtual serial port for legacy software expecting an RS-232 COM port, but in fact communicated over an ethernet connection.
