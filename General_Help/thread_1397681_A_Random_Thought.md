### Post by Bachstelze on 2010-02-03
> **Ceiber Boy said:**
> As Macs are Unix based, would that mean programs that run on macs will run on Linux?

No. A program compiled for one OS cannot normally run on another. There are binary compatibiliy layers (e.g. WINE) that can achieve that, but there is no such thing that can make OS X programs run on Linux. The fact that both are UNIX-like OSes is unimportant.

You mean Linus? Hardly just a "guy" Mac is now BSD based, and therefore not binary compatible with Linux. Unix is just a specification, Mac OS whatever was just given full Unix certification. Linux is still just Unix-like.

Some Linux applications will compile on one distro, and run on another. Depending on alot of things (libraries, architecture, etc).

You can't compile something on BSD, and run it on linux. However you can get the source, and if it's not using Mac specific calls(and vise versa) you could recompile it, and most of the time run it.

This works with windows to a point as well. 

Hope that has helped a bit.
