### Post by inameiname on 2010-04-30
I found this when I was in search of a resolution to the same problem. And it seems to work:

                                 go to System > Preferences > Startup Applications

in the startup tab, look for 'Volume Control' and check it if its unchecked.

If its not there, 'Add' it using the following parameters:

Name: Volume Control
Command: gnome-volume-control-applet
Comment: Show desktop volume control

...and then restart....

### Post by Calash on 2010-04-30
To fix

Right Click on the panel
Select "Add to Panel"
Add Indicator Applet

Apparently during an upgrade Gnome likes to hang on to some setting that tells it you do not want the new volume indicator to show.  By adding it again it refreshes and it "should" show up

The same process will fix if you are missing the -me menu.  Just add Indicator Applet Session,


If this does not work check that you have the following packages installed.

indicator-me
indicator-sound
indicator-session
indicator-applet-session
indicator-applet

(SOLUTION)

Good morning you all guys!

I installed Lucid Lynx a couple of days ago, and had some problems installing my NVidia drivers and making them work properly. After some google and some work after stopping the gdm I managed to install the latest version and have a smooth system.

But in the first reboot right after installing those drivers I noticed something different in the top panel, although couldn't tell what exactly.

Well, I found the solution for this. Couldn't tell why, (I'm kind of a n00b), I hope some of you explain it!

Just open terminal and type:

```
python /usr/share/gnome-panel/add-indicator-applet.py
```

Both volume and message icons show up instantly, and you don't need to do anything else (don't add it to the startup apps). Reboot and see how they load properly.


Cheeers!!

For those of you who uninstalled pulseaudio, that's one possible reason why the applet is missing.
