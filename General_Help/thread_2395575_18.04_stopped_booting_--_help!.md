### Post by oldfred on 2018-07-03
How you boot live installer UEFI or BIOS is then how it wants to repair.
It looks like you have a gpt partitioned drive, but are booting with BIOS and have the required bios_grub partition.
So you do not need an ESP - efi system partition unless you want to convert to UEFI boot.

Boot live installer in BIOS mode, and run Boot-Repair. In advanced mode is the full uninstall/reinstall of grub and updated the kernel check box. Run those fixes.

### Post by oldfred on 2018-07-04
Did you upgrade or reset UEFI. You probably should anyway, but if reset, then any settings you change in UEFI revert to defaults and you have to change them again. I have multiple settings I change, so I keep a list.
One setting is typically all full USB or something similar. That may be related to keyboard & mouse issue.

Commands always have to be entered one at a time.
Grub says it installed correctly, except for sdb which is your ISO and would not be up-dateable.

Not sure with UEFI hardware in BIOS boot mode, if it is escape key or shift key. Normally with UEFI you use escape key to get grub menu and with BIOS use shift key. But if you have fast boot on in UEFI that may not give you time to press any keys. 
Again check settings in UEFI.

### Post by oldfred on 2018-07-04
Are are in effect, two versions of 16.04. You have the one with the original kernel, but updated 16.04.1 and it will stay at original kernel for the full 5 years.

You then have the enablement stack version. It will go from 16.04.2 thru 16.04.5 getting newer kernels to support newer hardware. Each kernel is only supported for a short time and then upgrade needed. And final kernel will be the rest of the 5 years from original 16.04.
       [https://wiki.ubuntu.com/Kernel/RollingLTSEnablementStack](https://wiki.ubuntu.com/Kernel/RollingLTSEnablementStack)
