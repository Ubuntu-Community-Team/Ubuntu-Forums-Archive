### Post by drs305 on 2010-12-08
I've looked at the RESULTS.txt and, aside from all the # symbols, can't find anything wrong with the script. I've taken the 30_os-prober section of the menu and placed in my own grub.cfg and it displays the Windows titles. That means it's not a formatting problem of the menuentry.

In cases like this, I usually suspect a different grub.cfg is being used. I edit the current grub.cfg and change one of the titles to "test" to see if that is how it displays when I reboot. You save the file but don't run update-grub. In your case however, I don't see any other Ubuntu partitions from which the grub.cfg file would come. (But being inquisitive, I'd probably try it anyway.)

The other thing I would try would be to boot Windows from the grub prompt (press "c" when the menu displays). Enter the Windows menuentry lines manually - skip the 'search' line, and press CTRL-x when done and see if it boots. If it does, you can add the entry in 40_custom.

You could also just add the menuentry into /etc/grub.d/40_custom, make sure the file is executable, and update-grub (don't combine this with the 'typo' test or you will erase the modified grub.cfg). See if it appears in the menu on reboot.

I vaguely seem to recall coming across this issue before but don't remember if/how it was resolved.

Let us know how the above works out and if I can come up with something else I'll post back.
