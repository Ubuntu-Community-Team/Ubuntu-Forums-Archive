### Post by scythe_machine on 2011-02-26
Hello, I'm relatively new to Linux.  I put a line of code in my .bashrc file:
"sudo chmod 666 /dev/windrvr6"
which allows me to use a driver for a cable connected to my computer, but every time I open a terminal, I have to type in my password.  Is there any way to put my password into my .bashrc file so that I don't have to type it in every time?

Much thanks for any help you can give me.

---

### Post by blueridgedog on 2011-02-26
You could put the command in in /etc/rc.local, without the sudo, so that it is run as root when the system starts.

### Post by sisco311 on 2011-02-26
Write an udev rule. Something like,
/etc/udev/rules.d/windrvr6.rules:
```
KERNEL=="windrvr6", MODE="0666"
```should do the trick. 

For details, check out:
[http://www.reactivated.net/writing_udev_rules.html](http://www.reactivated.net/writing_udev_rules.html)
