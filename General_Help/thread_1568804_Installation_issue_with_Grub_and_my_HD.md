### Post by oldfred on 2010-09-06
If your computer is that old you may still have BIOS limits.

Herman discusses the issue with old grub here but it applies for all boot loaders.

[http://members.iinet.net.au/~herman546/p15.html#18](http://members.iinet.net.au/~herman546/p15.html#18)

You do not necessarily have to have a separate /boot partition as that is the most common recommendation. If you just make your current / (root) smaller - about 20GB to 25GB so it is within the limits. You then can use the rest of the space as /home or a /data partition and mount those partitions into your smaller root. I use several small roots with separate /data just so I can have the same data in multiple installs.

Partitioning basics with some info on /data older but still valid by bodhi.zazen
[http://ubuntuforums.org/showthread.php?t=724817&highlight=partition](http://ubuntuforums.org/showthread.php?t=724817&highlight=partition)
GParted partitioning software - Full tutorial 
[http://www.dedoimedo.com/computers/gparted.html](http://www.dedoimedo.com/computers/gparted.html)

To move /home uses rsync  
[https://help.ubuntu.com/community/Partitioning/Home/Moving](https://help.ubuntu.com/community/Partitioning/Home/Moving)
Uses cp -ax
[http://www.ivankuznetsov.com/2008/04/moving-home-to-its-own-partition.html](http://www.ivankuznetsov.com/2008/04/moving-home-to-its-own-partition.html)
