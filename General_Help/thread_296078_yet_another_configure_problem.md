### Post by an.echte.trilingue on 2006-11-10
Well, the ubuntu laptop is away with my wife for a couple days, so this is how this works in debian etch, but this *should* work the same in ubuntu...

Make sure you have all your repositories, including the deb-src ones, enabled.

gnomeconf.sh is in libgnome-dev, so:

```
#apt-get install gnome-core-devel build-essential libgnome-dev checkinstall fakeroot

#./configure --prefix=/usr

#checkinstall
```

Be sure to select "d" when it asks you what kind of package to build. There is a bug in the version number that prevents it from building a package.  When checkinstall asks you to make any changes, change number 3 to be 0.3, like this:
