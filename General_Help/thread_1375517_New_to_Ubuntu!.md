### Post by HiImTye on 2010-01-08
I would give the live CD a try. you don't need to install it to try it and you can see if the 'feel' is right for you. if you like how it's set up/laid out you can install it.

keep in mind you can change pretty much anything but it gives you a sort of idea of how things are going to be

Windows commands and Linux commands are drastically different. also switching from the 'Windows mindset' to the 'Linux mindset' takes a little bit of patience, time and understanding.

currently there is no 64 bit version of wine that is reliable, however 32 bit wine works fine on 64 bit Linux. there is a bunch of information on [WineHQ]("http://winehq.org") including how to run 32 bit on 64 bit and the app database (what apps work well; how to fix problems with programs; etc)

there are also a slew of Linux alternatives to Windows programs so even if something doesn't work well under WINE, you can likely find something similar (and often, better). there are exceptions to this rule, which is why there is WINE.

if you need access to any specific piece of software I would check around the forums and google to see if there are any issues, or simply ask here :)

Welcome to Ubuntu!!  Glad you decided to give it a try.

If you're completely new to Linux, I'd definitely suggest trying it out for a while.  It's definitely different in a lot of fundamental ways from Windows.  That's not to say it's hard, you just need to re-learn a lot of things you've probably learned through years of experience with MS products.

First and foremost, to get a taste for the OS, try it out on a live CD.  Download the Ubuntu ISO image, burn it to a disk, and boot your computer with it.  If your computer does not automatically boot from the CD first (ie, just goes straight to the HDD and Windows) you need to change you BIOS settings.  Post back if you're having that problem and we'll be glad to help you through it.  Once the live cd gets to the main menu, make sure to select the option to try Ubuntu without making changes.  I was a little anxious about doing this myself at first, but I assure you it's completely safe (and will become very handy for you in the future).  That said though, it never hurts to backup your data, so now would be a great time to do it!

If you like what you see based on the live CD, then I'd suggest either installing Ubuntu to a virtual machine (like you mentioned) or dual booting with Windows and Ubuntu for a while as you get more familiar and comfortable with your new OS.

Hope this helps, and once again, welcome to the community!

### Post by chessnerd on 2010-01-08
First, welcome to the world of Linux and Ubuntu.

If you want some practical tips about Linux, here's what I've learned after using it for a year:

1. Put it on a test-machine. It is fun to mess around under the hood, and there is no better way to learn about Linux, but it can become very unstable if you start digging around in config files too much. So, if you have a second computer that you don't use much, I'd put it on there first, and backup all important files.

2. When you do break something--and you will--the community is your friend. These guys have brought me out of many a jam and can likely help you too. Don't always expect a response: I've got a thread about a webcam sitting un-responded after three days. But usually they are very helpful and nice. If any of them flame you for this post I'll be shocked.

3. Have fun. Honestly, if you don't enjoy using Linux there isn't a reason to use it. A DIY attitude and some positive thinking are important, but being able to laugh at your failures is better. When X crashes, it's better to laugh than to pull your hair out.

4. There's no need to rush. You can make the change as fast or slow as you want. I've only recently gone to over 50% Linux usage and I've been using Ubuntu since last February.

If you stick it out a few months, I expect you'll be using Linux for a long time. I hope to see you around the forums. :)
