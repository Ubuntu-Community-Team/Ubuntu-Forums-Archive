### Post by molotov00 on 2008-06-12
There's so much going on here I just want to take a moment to explain exactly what's going on:

Linux sees partitions as 'files'. Hard drives are /dev/something. So you have 2 drives, it looks like, /dev/sda and /dev/sdb, with a few partitions on each.

To mount the partitions you need to know some information, such as the partition name and filesystem, which is why people asked you to run fdisk -l.

The 'mount' command will take a partition, like /dev/sdb5, and make it available in a folder, such as /media/windows or any other folder you want.

fstab (filesystem table) is simply an automated way to mount drives, which is run when you boot.

### Post by drs305 on 2008-06-12
While waiting for the UUIDs, here is what your new fstab entries should look like for sda1 and sdb1. Put the applicable UUIDs in place of the XXXs.

Type in **bold** indicate you must change to your specifics.
Make the desired mountpoints for the partitions:
sudo mkdir /media/**mount1**
sudo mkdir /media/**mount2**

Make yourself the owner of the mountpoints if desired:
sudo chown **username:username** /media/**mount1**
sudo chown **username:username** /media/**mount2**

Make a backup of fstab and open for editing:
```
sudo cp /etc/fstab /etc/fstab-`date +F%`
sudo gedit /etc/fstab
```


Add these lines:

```
# /dev/sda1
UUID=**XXX** /media/**mount1** ntfs defaults,umask=007,gid=46 0 1
# /dev/sdb1
UUID=**XXX** /media/**mount2** ntfs defaults,umask=007,gid=46 0 1

```
