However, if you compile programs, or perform memory heavy tasks, then the advantages of using a 64bit OS are greater. While some may argue some applications don't work properly under 64bit, the number of applications is rapidly diminishing and most applications will work fine.

### Post by mrmorris on 2008-08-26
> **mrsteveman1 said:**
> don't 64bit apps tend to actually use MORE memory because of register size (or something to that effect)?

Close, they require more memory because every pointer (address) now takes 64bit rather than 32bit. However with 64bit registers many operations which used to take 2 or 4 clock cycles now take only 1 or 2. Thus in practice, there really isn't much of an issue (slight memory overhead vs. slight performance overhead).

With 2GB RAM I would probably stay with 32bit. However 64bit is the future and if you do relatively heavy processing or plan to spend $70 on 2GB more, I'd definitely go with 64bit.

### Post by viciouslime on 2008-08-26
If you ever do any sort of video encoding work, audio encoding work or other cpu intensive stuff the 64bit version will make a massive difference regardless of how much ram you have. Applications on 64bit ubuntu are not only compiled as 64bit, but with certain other performance tweaks, like sse2 enabled as all 64bit processors are modern enough to be certain to support such extensions.

However, if all you ever do is browse the web, read/send e-mail and type the odd document, you'll probably be best off with 32bit until adobe finally release a native 64bit flash player.
