### Post by deoanam2002 on 2011-01-28
Running Ubuntu 10.4 32bit (only) on my system with ati graphics card.
I just ran a software update, and I believe a new kernel was installed.
When I rebooted, I saw the "Ubuntu" loading screen, but then the system froze up and the screen went completely blank.

Thinking it was a graphics issue, I tried to press escape to get into the grub bootloader. However, when I pressed escape at the proper time, the system just rebooted.

Thinking that grub was broken, I used a live cd to purge grub from my system and re-install it.

When I rebooted, I got the same initial problem: I get the "Ubuntu" loading screen, but then the screen goes black and the system appears to be frozen.
Again, if I press escape to try the grub bootloader, the system just reboots.

Any ideas what the problem could be?

---

### Post by deoanam2002 on 2011-01-28
Not sure why I insisted on pressing ESC to access grub menu (read it online, actually). However, pressing the left shift key got me into the grub menu and I was able to run Ubuntu in low graphics mode. So my initial assessment of the problem being related to a graphics driver issue was correct.
