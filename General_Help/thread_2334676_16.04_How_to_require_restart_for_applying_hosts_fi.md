### Post by TheFu on 2016-08-22
You can script it, but this is an unusual request. Guess there is a specific reason it is necessary? Perhaps some other software reads the /etc/hosts and configures itself based on that?  I would put the new hosts file in during shutdown, not at boot, but I don't know your software.

For every Unix system I've managed the last 25+ yrs, altering the /etc/hosts file immediately didn't have any unexpected impacts.  After all, DNS changes happen all the time and are usually outside our control.  Of course, I can think of a few ways in tightly coupled systems where that would cause an issue.

Just to clarify - the /etc/hosts file is all there is. There isn't a DB or something else that is translated somewhere else. Any process that chooses can directly read that file and make decisions based on it.  If it cannot happen until reboot, then don't write the changes until just before shutdown or immediately at boot (before networking starts).
