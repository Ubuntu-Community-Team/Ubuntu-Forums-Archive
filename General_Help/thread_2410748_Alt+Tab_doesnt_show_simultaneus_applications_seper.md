### Post by iamtheeggman2 on 2019-01-20
Hi,

In the last week I've noticed when it go to use alt+tab to switch between two different windows of the same program, they are shown as grouped together. So for example where I see a list of applications as icons across the screen, the two windows belonging to the same program are one icon, when you tab to that icon, two more icons open up beneath it, showing miniature pictures of what is going on in those two windows . Rather stupidly, this disqualifies the use of the alt+tab feature because you cannot select a specific windows to jump to using the alt and tab keys. you are apparently forced to then use the mouse cursor to select or highlight the one you want to open and then it opens after you release the tab key - rendering the use of this keyboard shortcut redundant, how can I get to use the old style alt+tab function please?

### Post by vanadium on 2019-01-20
The old-style alt+tab style can be obtained by installing gnome-shell-extensions, if not already installed. This package contains a few official gnome extensions. Then also install Tweaks (gnome-tweaks), which will allow you to activate the "Alternatetab" extension.

With respect to the current switcher, you eventually need to combine it with Alt+Key_above_tab. The latter one switches between windows of the same application. Combining these two keys allows you to switch using only the keyboard. Admitted, it is all more convoluted, but for users working with many windows and applications simultaneously, the approach may work better than the old style Alt+tab function.

Also important is that the Alt+tab switcher works across all your desktops. The combination can be quite powerful to enable fluent navigation in a more complex work flow. The approach is actually quite different from how desktops worked traditionally, and requires some re-muscling of old habbits to start appreciating it.
