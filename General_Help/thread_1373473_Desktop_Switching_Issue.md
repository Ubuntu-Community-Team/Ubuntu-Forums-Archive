### Post by errigalmarten on 2010-01-05
I've recently done a fresh install of Ubuntu on my computer. I was last using 9.04, and I decided to go with 9.10 this time around. Everything seemed to be going smoothly until I got to the GUI and tried to switch desktops by rotating the scroll-wheel on my mouse. Nothing happened. Normally this action switches me to the next desktop, or rotates the cube if I've got Compiz installed (didn't work in Compiz, either). It's a minor nuisance, but one I'd be happier without.

My question is this: Is there any known way to resolve this issue? This wasn't an issue with any previous iteration of Ubuntu I've used, and I haven't physically done anything different to my PC. The scroll-wheel works in any other capacity save switching desktops.

Any input would be appreciated.

---

### Post by pme 72 on 2010-01-05
I Googled the issue and found a link recommending I go to System/Preferences/CompizConfig Settings Manager/Viewport Switcher/Desktop-based Viewport Switching and enable Button5 for Move Next and Button4 for Move Prev. That enabled fast desktop switching with the mouse for me in 9.10.
