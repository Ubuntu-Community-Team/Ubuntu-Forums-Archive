### Post by FaizanKazi on 2009-04-06
firstly, setting up windows xp/vista/7 as a virtual machine under virtualbox/vmware/qemu is great if youve got a fast dual core system with at least 2GB ram and a decent graphics chip/card, but if youre hoping to play 3d games like counterstrike or call of duty, no way dude! not unless youve got some really high end hardware...

now back to the original problem... your grub "menu.lst" says windows XP is on (hd0,0) BUT it seems to be that your windows xp is on (hd0,1) [the 25gb ntfs partition]. if you change that in your menu.lst you should be fine!

and if you select windows xp in grub at boot time, and it gives you something like "NTLDR missing" it means that windows xp's original boot loader is corrupted. you need to get a copy of vista boot pro or Easy BDC (this ones free) to repair your windows xp boot loader, then you'll have to boot off an ubuntu live disk and repair grub ("sudo grub" at the terminal > "root (hd0,0)" [thats your linux partition], "setup (hd0)" and everything should be fine then!!
