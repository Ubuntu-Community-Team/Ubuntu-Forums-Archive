### Post by Vunutus on 2009-05-17
Given that it's mostly seen as a desktop OS, how does Ubuntu server edition stack up when compared to other distros?

I'm very likely to get a job soon at a small family business as their only IT guy to set up and maintain all technology from the ground up. I'm excited about this, but as a programmer, networking and such isn't really my specialty, so I'm doing my research ahead of time (although I have taken numerous classes and have a handful of networking certifications).

What I know so far is that the proprietary software they have to use is Windows only, so all workstations will be running Windows (most likely XP since I can't see any possible reason they will need 7 and I'm sure as hell not putting Vista on them). This software needs a server for database services (what kind of database is unknown to me at the moment) as well as standard file/print services (Samba obviously). The network will start out very small (about 5 workstations), but will most likely grow to a pretty decent sized network over a few years.

Obviously the Ubuntu forums are going to favor Ubuntu, but I'm wanting to know if Ubuntu is the right server OS for the job.

### Post by mb_webguy on 2009-05-17
Pretty much all of the reasons you might choose to use Ubuntu for your desktop system applies to using it as a server as well.  Ubuntu has comprehensive and well-maintained repositories, a large, knowledgeable, and helpful user base, and the stepped release schedule helps ensure stable systems.  

If I were going to set up a Linux server that I was to maintain, and had the option of which distribution to use, I'd almost certainly go with Ubuntu.
