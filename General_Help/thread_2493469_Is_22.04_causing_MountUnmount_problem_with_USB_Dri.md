### Post by TheFu on 2023-12-19
> **ishmael3 said:**
>   After reading your warning I searched for an authoritative Linux/Ubuntu source that might explain the "clear, technical reasons" for avoiding FATs and couldn't locate one. Not questioning your expertise, but if you could cite a source or give a brief explanation I'd count myself as educated.  

FAT doesn't provide journaling.  That's a huge negative. It means data loss is much more likely and unrecoverable.  Always use a journaled file system on HDDs and SSDs when possible.

If dealing with cheap flash storage, like a USB thumb-drive or microSD cards, the preferences become more complex.  I've written about this a number of times in these forums, but I suspect you don't follow **every** post from me (and I can't blame you!).  

[B]Concerning cheap flash storage, usually with 1000 - 10000 write cycles per cell, 

[/B]If there is a specific hardware device that only supports FAT32, then there's no discussion. Use it. Live with the limitations. None of the others below have limitations anywhere near what anyone today would run into.  Well, except MS-Windows file systems have many character limitations for directories and filenames that don't exist on other OSes.

If **exFAT** is supported and you need cross-platform access, exFAT is a better file system, fixing many of FAT32 issues, while also limiting the number of writes.
If it is Linux-only storage and you can load software onto all the systems you need to, **f2fs** is a native Linux, "flash friendly" storage file system.  This is the best choice since it supports POSIX standards and Unix permissions. The down-side is that the f2fs-tools have to be installed almost always.  Some android devices have support for f2fs built in, which makes it behave like internal storage on those devices, but that only happens with {ooops, forgot to finish this thought ...}  .... select models from a few vendors.  Sadly, the day that all android devices support f2fs isn't here.  f2fs has been released as 100% F/LOSS, no patents. It also performs very well according to comparison file system benchmarks.  [https://www.phoronix.com/news/Linux-5.14-File-Systems](https://www.phoronix.com/news/Linux-5.14-File-Systems) is a comparison between ext4, btrfs, xfs, and f2fs for different sorts of workloads.  f2fs almost always beats btrfs and usually ties xfs and ext4, while still being "flash friendly".  That website has a number of other posts comparing different versions of f2fs.

ExFAT and f2fs are the most reasonable choices.  Neither are journaled, which reduces writes and loses recovery options if there's any interruption before all writes complete.

Sometimes, it does make sense to use ext2, ext3 or ext4 on flash storage.  ext2 isn't journaled, so that would have lower write counts.  ext3 is an extension of ext2, so the journal is added, but can be removed to downgrade to ext2 again.

Permissions are a problem only if the storage is non-native Linux. On Linux systems with a native file system, like f2fs, ext2/3/4 or any of the 20 other native Linux file system, the permissions are retained.  Of course, if you need to connect the storage to some other, paid, proprietary OS, you are S-o-L and have limited choices. For flash storage, exFAT is the non-journaled standard and has been for about 5 yrs.  NTFS is the journaled standard for cross-platform storage, unfortunately.  NTFS is a fine file system for MS-Windows and is proven to be very safe on that platform.

### Post by TheFu on 2023-12-19
The wording of the question to anybody, including AI, matters greatly.
Ask about exFAT and f2fs.  Also, ask different AI ... iask.ai is another tool.

Here's the question I asked:
> Which is the best file system for microsd to be shared by Linux and Windows?
The answer summary (with 8 paragraphs above it):
> In conclusion, the **exFAT** file system is the best choice for sharing a MicroSD card between Linux and Windows systems due to its compatibility, large file support, efficient data transfer, ease of use, and lack of restrictions on file system features.

See how I left it open for the solution, but limited parameters -  only specific flash memory and 2 OSes.
If I'd asked about SSD and HDD physical storage instead of a microSD, I bet NTFS would be the answer for cross platform. It is journaled. Anyone who's had to wait 12+ hrs for an fsck to complete because a non-journaled file system was used knows how important using journaled file systems is.

Be careful how you ask the questions. I don't have any problem bursting bubbles.  It was my job for over a decade to bring bad news to project teams when they didn't follow enterprise standards.  Of course, if their solution WAS the best answer, regardless of the standards, I was their advocate before the specific standard board to get their use approved.  I also sat on a few of these boards, trying to look out for the enterprise in specific areas of IT expertise.
