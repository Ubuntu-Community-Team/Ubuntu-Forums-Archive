### Post by JohnElway on 2010-10-15
I'm currently running good ol' Ubuntu with GNOME, but I'm interested in using a more lightweight system(something like Ubuntu minimal + Openbox or Arch + Openbox). To prepare for this, I'm trying to get more proficient with the terminal by using CLI alternatives to my commonly used programs(cd/ls/pwd/mv/cp over nautilus, vim over gedit, finch over empathy/xchat, rtorrent over transmission, htop over system monitor, etc.). 

But right now I'm using all these apps through GNOME-Terminal, which I've heard is not very lightweight. I know the obvious alternative is xterm, but it seems much uglier and harder to configure. Is there a terminal lighter than GNOME-Terminal that still allows easy configuration of fonts/colors and has simple startup options for executing a command (--command), starting a terminal with a certain size (--geometry), and starting a terminal from a certain directory (--working-directory)? I don't need features like tabs, profiles, transparency, etc.

It's possible that this might not even make a difference. On a system with 256MB-512MB of ram, would multiple open GNOME terminals (5 or so) make much of a difference on system performance? Does GNOME-Terminal install significantly more dependencies that other terminals?
