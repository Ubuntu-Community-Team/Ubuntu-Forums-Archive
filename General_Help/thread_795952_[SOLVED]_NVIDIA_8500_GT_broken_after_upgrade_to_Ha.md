### Post by fyo on 2008-05-17
My 8500GT works just fine with Hardy, so I would second the previous suggestion: Get the latest NVIDIA driver and let it configure your xorg.conf file.

I'm using the following with my 8500GT on Ubuntu Hardy 64bit:

[http://us.download.nvidia.com/XFree86/Linux-x86_64/169.12/NVIDIA-Linux-x86_64-169.12-pkg2.run](http://us.download.nvidia.com/XFree86/Linux-x86_64/169.12/NVIDIA-Linux-x86_64-169.12-pkg2.run)

For 32bit, you need to grab this one instead:

[http://us.download.nvidia.com/XFree86/Linux-x86/169.12/NVIDIA-Linux-x86-169.12-pkg1.run](http://us.download.nvidia.com/XFree86/Linux-x86/169.12/NVIDIA-Linux-x86-169.12-pkg1.run)

You can find full instructions (and downloads) from NVIDIA's site here:

[http://www.nvidia.com/Download/index.aspx?lang=en-us](http://www.nvidia.com/Download/index.aspx?lang=en-us)

by entering your graphics card, language and OS.

Basically, you type "sudo sh NVIDIA-Linux-x86-169.12-pkg1.run" to install the driver. It will only run if X is NOT running, so either boot into a non-X mode or kill X from the command line ("ps -A | grep gdm" then kill the gdm process with "kill -INT").
