### Post by ronmarley1 on 2006-11-23
Hi Trance,
Here's a solution that will work (but you may not like it) because I have the same graphics card.  
1.  Forget XGL on the Intel card.  I could never get it to work, going back to compiz.  AIGLX and Beryl work well together for me.  AIGLX is built into Edgy.  
2.  Most of your problems come from that fact that you have installed drivers for a video card you do not have.  You don't need to mess with the ATI or fglrx drivers (as you have an Intel card) which is why you're getting the Xorg BSD.  The Intel 810 diver that comes with Edgy will work for you.  If you want, I can send you a copy of my xorg.conf -OR- search these forums for a post from me concerning Beryl.  You'll find my xorg.conf file there.
Now, for the solution:
1.  Backup all your files that you want to keep (digital pics, documents, etc...).
2.  Do a clean install of Edgy (many threads mention that a fresh install will be much better than the upgrade you did).  
3.  Install Beryl using this guide: 
[http://wiki.beryl-project.org/index.php/Install/Ubuntu/Edgy/AiGLX](http://wiki.beryl-project.org/index.php/Install/Ubuntu/Edgy/AiGLX).  After a clean install of Edgy, the guide had me running Beryl in less that 10 minutes, and you don't need to touch your xorg.conf file.
*  When using the guide, don't add Trevino's repo.  That will give you the SVN (daily-ish updates).  Sometimes that breaks more that it fixes.  Use the "ubuntu2" repo you mention above.  That will give you version 0.1.2.

To verify that this will work for you, boot the live cd, install Beryl from the link above, and run "beryl-manager" in a terminal window and all will work (albeit slowly, since you'll be running from the live CD).
