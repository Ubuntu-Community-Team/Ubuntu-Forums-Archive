### Post by drs305 on 2008-08-29
Backup and edit fstab:
```
sudo cp /etc/fstab /etc/fstab.old
gksu gedit /etc/fstab
```

Replace entire contents with:
```

# /etc/fstab: static file system information.
#
# -- This file has been automaticly generated by ntfs-config --
#
# <file system> <mount point> <type> <options> <dump> <pass>

proc /proc proc defaults 0 0

# Entry for /dev/sda2 :
UUID=37873d21-0b45-4a66-837a-f28bedd2cc5b / ext3 defaults,errors=remount-ro 0 1
# Entry for /dev/sda3 :
UUID=fff6dd52-1487-481a-ac39-1c9f1b6943f0 /boot ext3 defaults 0 2
# Entry for /dev/sda1 :
UUID=07D6-0302 /media/sda1 vfat defaults,noauto,utf8,umask=007,gid=46 0 0
# Entry for /dev/sda5 :
UUID=69781bac-7140-4cfa-8171-d4661bd2cb0f none swap sw 0 0

/dev/scd0 /media/cdrom0 udf,iso9660 user,noauto 0 0

```


One observation - If your /boot partition really is 100MB: As new kernels are introduced it is possible your /boot partition will become full. When this happens, you will get a message saying that a kernel install failed. At that point (or earlier), you can remove older kernels via synaptic to free up space. You will want to keep at least one older (working) kernel for backup. Using synaptic to remove an older kernel (linux-image-2.6.24-XXX) will free up space and remove older kernel options from your boot menu. (Should you decide to reformat your disk at some point in the future my recommendation would be to eliminate the separate /boot partition.)

**Added:**
I noticed your swap partition UUID changed. It is possible you could have hibernation issues should the stored swap UUID is incorrect. Run the following command to see if the correct swap UUID is recorded:
```

cat /etc/initramfs-tools/conf.d/resume 

```
If they don't match:
```

sudo dpkg-reconfigure  initramfs-tools

```

### Post by drs305 on 2008-08-29
> **thefacelesswarrior said:**
> if which one's dont match? when i type that first command, it shows me:

RESUME=UUID=38269d54-743b-4f8d-88a7-20c0d9142726

which is the same as the one in my original post...why are they different?

The ones you compare are the current UUID from the command "sudo blkid" and the first command, which displays the swap UUID stored in the system.

Yes, the swap and resume uuid's don't match, so just run the second command and it will update the resume image so the swap is correctly identified. 

The image isn't automatically updated if the swap partition is reformatted, and reformatting changes the UUID.

Also note that the comment # symbol is *_not_* a comment in menu.lst  These lines will be read and acted upon.

### Post by ben2talk on 2008-08-31
I managed to clean mine up. I took the following steps - having XP installed, first I booted XP and renamed the XP partition, and my ntfs STORAGE partition (on my first disk HD0, named it as STORAGE1-1).

Next, boot Ubuntu and open the Partition Editor.
In partition editor, I was able to easily see the layout of partions disk by disk.

Here is the place to see the mountpoints and understand the layout.
I also installed an old ubuntu program called 'Disk Manager'.

In Disk Manager, I unmounted all drives except for root. Then opened Nautilus and deleted every empty folder in /media/
Now starting in Disk Manager, rename each mount point to correspond to the partition.

When I now mouse over the disk mount applet, instead of having to click to open '30.0GB' to see '/media/XP', it shows as 'XP' in the applet tooltip.

I think that some tools like this can make life a lot easier for anyone who doesn't think it's necessary to always ask for help from the staff.

Ubuntu takes away many real problems associated with other operating systems. It gives a nice mix of options - many people want to set it up to be like a mac (silly idea if you ask me, but I do love 'scale windows') and others want it to be like XP or Vista (Surely Vista scored a huge hit by building into its kernel the integrated search and launcher!
