### Post by Gizenshya on 2009-02-01
I've wondered this myself, about the fragmenting.  Hopefully the coders had enough sense to allocate space per file before transfer... but I don't know if such is actually the case.

I do A->B and C->D often... and I've done your situation sometimes as well.  But, only with 2 external USB drives to one SATAII drive.  Because that pretty much takes up the 'bandwidth' (?) of all drives 100%.  At least, there isn't overhead loss.

Except for that, though, I don't see a reason why you would need to do A and C to B at the same time.  If the drives all have approximately the same transfer bitrate, that is.  You'll actually lose speed from overhead.
