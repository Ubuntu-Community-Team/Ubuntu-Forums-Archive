### Post by papibe on 2011-09-30
Thanks for posting those results.

I'm assuming that 194.168.4.100 is your router, and you are getting that configured as default.

It seems that your router is NOT working properly as as a local DNS (translating local names to LAN IPs). Either is is not properly configured, or it actually does not do any work as DNS (it just relay all request to an external server).

That's the reason why 'myhost' is being translated to an external IP (81.200.64.50). It should be translated to something like  194.168.4.xx

I would start checking your router configuration. For starters, its address is very odd to me. Have you change its settings?

In the meantime you can use the zero configuration (the avahi services). That way you can access your machines using the .local domain.
```
$ ssh myhost.local
```

I hope this helps,
Regards.

### Post by papibe on 2011-09-30
Here's a simple [tutorial]("http://ubuntuforums.org/showthread.php?t=88206"). You edit one file, and install the service. Pretty simple. Note that in your case, the edit would look more like this:
```
hosts:          files mdns4_minimal [NOTFOUND=return] **wins** dns mdns4
```
Regards.

EDIT: I have to add that this setup may slightly add a little bit time to translate addresses when browsing the Internet (since every address will be check locally first). However, for most people is not noticeable.
