### Post by drs305 on 2011-09-21
Since you only want Ubuntu and only have one version, perhaps the easiest way is to just disable the script that searches for other operating systems.

There are two ways to do this:

1. Make the script unexecutable:```

sudo chmod -x /etc/grub.d/30_osprober
```

2. Disable it in the grub configuration settings:
```
gksu gedit /etc/default/grub
```
Add this line (for Grub 1.99 in Natty):
> 
GRUB_DISABLE_OS_PROBER='true' 
Save the file.

FOR EITHER 1 or 2:
Run the following to update the menu:
```
sudo update-grub
```

Note that is does not remove anything from the partitions, it only alters the output to the menu.

ls -a /mnt

You should generally avoid mounting directly to /mnt, as it may contain other mounted filesystems (though you would have to mount them manually).

[quote=http://www.control-escape.com/linux/lx-filesys.html]
Linux will allow you to mount a file system on a directory that is not empty. Any files in that directory will become invisible in the virtual file system. They will still exist on disk, but you will be unable to access them until the file system that obscures them is unmounted. There are rare cases where this might be desirable, but in general it can create unexpected behavior and is not advisable. For these reasons, only the administrator may authorize the mounting of file systems.[/quote]

I generally create a mount point in /tmp for one time mounts:
```
mkdir /tmp/mount
sudo mount /dev/sdb1 /tmp/mount
```

That way they disappear on reboot ;)
