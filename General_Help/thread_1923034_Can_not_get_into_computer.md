### Post by oldfred on 2012-02-09
Welcome to the forums.

Did the update complete, partial updates will often create major problems. 

Did you force a shutdown without closing it correctly as that can cause file corruption.

proper shutdown:
Holding down Alt and SysRq (which is the Print Screen key) while slowly typing REISUB
R-E-I-S-U-B to force shutdown
A good way to remember it is.
Raising Skinny Elephants Is Utterly Boring ...or
Reboot System Even If Ultimately Broken ...LOL.
[http://kember.net/articles/reisub-the-gentle-linux-restart/](http://kember.net/articles/reisub-the-gentle-linux-restart/)

If it is file corruption you may be able to use a liveCD and run e2fsck to run file checks on the Linux partitions:

#From liveCD so everything is unmounted,swap off if necessary, change sdb1 to your partition(s)
#e2fsck is used to check the ext2/ext3/ext4 family of file systems.
sudo e2fsck -C0 -p -f -v /dev/sdb1
#if errors:
sudo e2fsck -f -y -v /dev/sdb1

### Post by oldfred on 2012-02-10
the F2 key takes you into BIOS. Some computers require you to turn on USB keyboard and mouse in BIOS to get them to work in grub. Both Windows & Ubuntu seem to add their own drivers and bypass the BIOS setting, but grub uses the BIOS setting.

Then f12 or another f key should give you boot choices like hard drive, CD, or USB if a newer computer.

Everytime you shutdown and are not sure if you have started to boot or not use REISUB, just in case.
