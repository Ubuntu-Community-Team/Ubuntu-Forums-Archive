### Post by earthpigg on 2009-11-24
if you stick to the repos to get the software, no you do not need an antivirus. thats one of the nice things about repos: all of the 20,000 packages found in ubuntu's synaptic and/or software center and/or add or remove programs... have been vetted by Ubuntu as safe.

FrostWire is not in the repos... hence, my warning. i dont know that the program itself does have any malware, but thats how a Linux virus would spread if they did exist: popular software not in the Ubuntu repositories that you must download from random websites.

if you plan to download a bunch of stuff in frostwire, understand that the stuff found there (legal and illegal) is ***quite*** infested with Windows viruses.

which will not affect you... but it will affect your windows using buddies if (for example) you let them copy your mp3 collection gathered on frost/limewire.


assuming the FrostWire .deb itself is not infested with anything (and i very seriously doubt it is)....

do not download any actual software from frost/limewire. ever. regardless of OS.

and, on Linux, do not access/run/open anything you got from lime/frostwire as root. ie: you should never enter your password while doing anything involving anything you got from lime/frostwire.



if you have the most expensive antivirus in the world that costs you $500,000 per month, it still will do absolutely nothing to protect you from a day-zero attack using lime/frostwire as the attack vector.

[http://en.wikipedia.org/wiki/Zero_day_attack](http://en.wikipedia.org/wiki/Zero_day_attack)

and the frost/limewire platform is the #1 most ideal vector for a bad guy to release his malware on. could you think of a better place to spread your malware, if you where a bad guy?


if you must use crap like frostwire, be aware that you are playing with fire.

it is much much safer to play with fire as a Linux user than it is as a Windows user, but i will not vouch for anything pertaining to it. even the .deb itself that i linked above.

when you stop using the ubuntu official repos and start downloading random .deb files from random websites you find on google (guess how i found the link in my first post?)... well, you may as well be using windows and downloading random setup.exe files from random websites you find on google. 

when it comes to that, the user is 100% to blame. Linux is no more fool-proof than Windows is in that regard. if you specifically ***tell*** Ubuntu to install malware and then enter your 'sudo'/administrator/root password to confirm your command of "Yes Ubuntu, please install this malware-infested .deb i have given you", then it will do exactly as you say (even if you didn't know it was malware... that's your own fault.).
