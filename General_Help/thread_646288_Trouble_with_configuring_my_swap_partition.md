### Post by bingoUV on 2007-12-21
inzania,
    Don't reduce the size of your swap partition if you can avoid it. Because you want to hibernate in the swap partition, small swap will make hibernate fail if you have big applications open. 4GB swap is good price to pay for a reliable hibernate.

### Post by jeffus_il on 2007-12-21
I have never used a "swap file" in linux, but you should be aware that you have created a swap file in an existing partition, see your fstab line :
/media/Documents/swap.swap  none  swap  sw  0 0

in a partition mounted at /media/Documents.

I would check to see if hibernate knows how to use a swap file or if it really needs a partition to swap to.
