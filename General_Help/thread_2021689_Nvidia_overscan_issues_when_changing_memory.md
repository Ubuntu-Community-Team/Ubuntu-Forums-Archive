### Post by mogators on 2012-07-09
Short version:  Nvidia overscan option disabled with 1GB ram, works with 2GB or 4GB.

Long version:  I discovered an interesting problem with my Ubuntu 12.04 64bit system I have connected to a 40" TV via HDMI this weekend.  Using the Nvidia "additional drivers" current version, I had an option in the Nvidia settings for correcting overscan that worked perfectly with 4GB's of RAM installed.  I was swapping that memory out to upgrade my son's computer and tried running 1GB of RAM in the Ubuntu system.  Before shutting down, I applied a bunch of updates to the system and then swapped out the memory.  After rebooting, my overscan problem returned and the option in the Nvidia drivers to fix it was dimmed out.  Not thinking the memory had anything to do with it, I assumed it was from one of the updates.  After trying many things to fix it, I decided to just start from scratch.  After getting everything working again (with the 4GB RAM installed), I swapped the memory again.  Once again I lost the overscan setting.  I then swapped the memory with 2GB's of RAM and got the overscan setting back.  I was using the integrated graphics and had the graphics memory set to use 256MB in all cases.  Does anyone have any idea why this option gets disabled with 1GB of RAM?
