### Post by oldfred on 2012-09-23
Not sure why you have two swaps? You must have installed twice.

Your Ubuntu install has to be sda6, but you install grub to the MBR not the partition.

You installed the lilo boot loader, which is a Windows work alike. It boots to the partition with the boot flag (*) just like Windows.

To reinstall grub2's boot loader to the MBR.

Boot Repair -Also handles LVM, GPT, separate /boot and UEFI dual boot.:
[https://help.ubuntu.com/community/Boot-Repair](https://help.ubuntu.com/community/Boot-Repair)
You can repair many boot issues with this or 'Create BootInfo' report (Other Options) & post the link it creates, so we can see your exact configuration and diagnose advanced problems.

### Post by oldfred on 2012-09-24
You need the 64 bit version. 

You should be able to use the liveCD or USB that you used to install, and just add Boot-Repair to that. You do have to re add each time you reboot with the lIveCD as it does not save anything you download.

You then can reinstall grub2's boot loader to sda.

But if issue continues, you need to check BIOS settings. Is system in old IDE mode or newer AHCI or SATA or Large mode?

If still issues then it seems grub has a bug where it cannot find its install partition on large drives. Some systems seem to work without issue and others only work with a /boot partition or the full / (root) inside the first 100GB of a drive.
