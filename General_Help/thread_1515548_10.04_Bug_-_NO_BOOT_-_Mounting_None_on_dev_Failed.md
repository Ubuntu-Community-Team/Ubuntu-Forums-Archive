### Post by dabl on 2010-06-23
If you want to try the chroot approach, you're going to have to be mostly on your own -- I'm not the expert.  But basically, you will:

1. In your Live CD session, make a mount point and mount the hard drive partition where your problem OS is located.

2. chroot to the "/" of the mounted filesystem

3. Run your apt-get commands to ensure that whatever OS is actually there (I'm wondering how much of it really updated) is fully dist-upgraded

4. Run your update-grub command to get it to read the other partitions and write the correct boot menu

5. Exit the chroot and umount the partition

6. Say a prayer to the little monkey-god and reboot the system.

[http://en.wikipedia.org/wiki/Chroot](http://en.wikipedia.org/wiki/Chroot)

[http://www.gentoo.org/proj/en/base/x86/chroot.xml](http://www.gentoo.org/proj/en/base/x86/chroot.xml)

[http://slackworld.berlios.de/2007/chroot_howto.html](http://slackworld.berlios.de/2007/chroot_howto.html)

[http://www.linuxforums.org/forum/gentoo-linux-help/41167-daft-question-how-chroot.html](http://www.linuxforums.org/forum/gentoo-linux-help/41167-daft-question-how-chroot.html)

Because I hate having a system with "mystery configuration" -- questions about whether all the files that are supposed to be there really are, whether the package manager was updated with all the right packages, etc. etc.  In other words, my life is too short to spend time wondering about a system that I cannot rely on.  ;)
