### Post by drs305 on 2010-11-28
I don't think the problem is with Grub specifically unless it's the instructions you are passing in the custom menu. Note I am not a Windows person. I don't have anything against it, I just don't use it so my knowledge of how it operates is pretty limited.

Try this as your custom menu entry:
> 
menuentry "Windows 7 Enterpise" {
insmod part_msdos
insmod ntfs
set root='(hd0,msdos1)'
search --no-floppy --fs-uuid --set 786053F06053B39A
chainloader +1
}

Save the file, update grub and then reboot and try Windows again.

If that doesn't work, the next thing to do would be to make sure Windows is working properly. I know you said it booted without problems, but since you have done many things to your bootloader it's time to make sure it still does.

We can do it all from a LiveCD. We will tell the MBR to boot Windows again. We can do this by partially installing *lilo*. Install it and disregard the messages it gives you about not being fully installed. Once you run the commands, reboot and make sure Windows boots normally. 

```
sudo apt-get install lilo
sudo lilo -M /dev/sda mbr
```

If your Windows system is in fact broken, use the W7 repair disk to fix it and then we can deal with Linux. If you need Linux back first, boot the LiveCD, mount /dev/sda5 and reinstall G2. Do NOT include the partition number in the second command.

```
sudo mount /dev/sda5 /mnt
sudo grub-install --root-directory=/mnt /dev/sda

```

### Post by drs305 on 2010-11-28

> I'm trying to make the custom menu entry and it won't show up. I've done update grub, and edited 40_custom...

Here is a copy that gets incorporated into my system when I run update-grub. I've added a line near the top so it gives you a message as update-grub is being executed:


> 
#!/bin/sh
echo "Adding 40_custom." >&2
exec tail -n +3 $0
# This file provides an easy way to add custom menu entries.  Simply type the
# menu entries you want to add after this comment.  Be careful not to change
# the 'exec tail' line above.

menuentry "Windows 7 Enterpise" {
insmod part_msdos
insmod ntfs
set root='(hd0,msdos1)'
search --no-floppy --fs-uuid --set 786053F06053B39A
chainloader +1
}


Make sure the file is executable and that you have the last **}** on a line by itself (it's sometimes overlooked). 

Watch for the "Adding 40_custom" message as you run update-grub, then check /boot/grub/grub.cfg for the entry in the ### BEGIN /etc/grub.d/40_custom section.

I can't guarantee this menuentry will work, but it should at least be shown in the Grub menu on boot.
