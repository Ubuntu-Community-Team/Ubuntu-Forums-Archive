### Post by ajgreeny on 2013-02-02
What hardware have you got?  You say it is 32 bit, so is obviously not very new, but what cpu does it have, and is the cpu sse2 enabled?

Run ```
cat /proc/cpuinfo | grep sse
```to see if it's just sse, or both sse and sse2 that show in the flags line.  For all the cpuinfo just omit the "| grep sse" part of the command.

If it is only sse, not sse2 follow the info at [Flash 11.1.102-63]("http://ubuntuforums.org/showthread.php?t=1953796") to get a slightly older version of flash, but note the security risks and make sure you run **no-script** and/or **flashblock** if you have to use the older flash version, as I do on my main desktop machine.

Downgrading solve the problem, as you mentioned does it look like to problem is caused by the missing flag "sse2".

Solution:
Download the flash version [Flash 11.1.102-63]("http://ubuntuforums.org/showthread.php?t=1953796") (option 2, downgrade flash), if you haven't already installed flashplugin-installer, install it. When that is done, copy the libflashplayer.so file from the downloaded file (in  the tar.gz file) to the folder '/usr/lib/flashplugin-installer/', this should overwrite the latest version of flash (the one causing the problem)

I'm guessing that any update of flashplugin-installer, would cause the problem to remerge since the update would replace the libflashplayer.so .

 As ajgreeny mentioned there is a security vunerbility associated with flash version 11.1.102.63 . More info can be found here: [http://www.adobe.com/support/security/bulletins/apsb12-05.html](http://www.adobe.com/support/security/bulletins/apsb12-05.html)
