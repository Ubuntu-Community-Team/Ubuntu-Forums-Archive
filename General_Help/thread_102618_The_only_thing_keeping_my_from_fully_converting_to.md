No. The reason is because "Linux" is a kernel, not a company. MSFT and Mac have incentives and tools for developers to make games on their platform. Wine (and the commercial versions such as cedega and crossover) have been able to take the windows API and make many programs work in Linux especially on the x86 platform since it doesn't have to emulate the processor (thus the "isn't an emulator" part of the meaning of the acronym "wine").

A few games have been made to run natively in Linux most notably titles by ID Software. Gaming on the PC is almost dead anyway especially now that the X360 is out.
