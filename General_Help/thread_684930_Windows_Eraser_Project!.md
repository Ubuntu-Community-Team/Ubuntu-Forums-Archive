### Post by pointone on 2008-02-01
First of all, I think you should read this:

[http://linux.oneandoneis2.org/LNW.htm](http://linux.oneandoneis2.org/LNW.htm)

Switching to Linux requires some adjustment. Not all Windows programs will run in Linux; you'll need to make substitutions. I find a lot of Linux applications are superior to Windows applications, though. After I made the full switch to Linux, I kept wishing for certain Linux programs were available in Windows; not the other way around.

You can run a lot through Wine, though; and there's VirtualBox or VMware for the stuff that doesn't. Visit [http://appdb.winehq.org/](http://appdb.winehq.org/) for a list of apps that work through Wine, and to what degree of success.

What usually doesn't work is the absolute latest games.

Source games work fine, though; and DOSBox is available in Linux already.

> STEAM -ANY GAME MUST WORK THAT I DECIDE TO BUY FROM THERE

You may want to consider just dual-booting.

### Post by SpiderGorilla on 2008-02-02
I'd say, to be honest, just stick with Windows. So much of what you want to do just either isn't defaultly supported on Linux or would require a ton of effort.

You might want to ask yourself why you're converting to Linux. Gamers generally don't find a happy life on Linux right now, because so much of what they enjoy just isn't on this system natively yet. Wine can run almost any Windows application without a lot of issue, so I'd recommend that. But you're going to suffer a performance hit because it doesn't do the API 100% perfectly.

I'd say, ask yourself how much of your time on your machine is spent actually gaming. If it's more than 25%, just stick with Windows. If it's less than 25%, you might consider dual-booting and going into Windows when you're needing to game.
