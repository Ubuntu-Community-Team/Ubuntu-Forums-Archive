### Post by telecastle on 2013-12-28
I've enabled Desktop Sharing in Ubuntu LTS 12.04 (running as a VM on a server) and am accessing the Ubuntu via OS X Screen Sharing application running on my MacBook Pro. Everything is working fine except that I see two mouse pointers in Ubuntu when I access it via OS X Screen Sharing. The black mouse pointer is the one that belongs to the OS X Screen Sharing application, and the white mouse pointer (located right underneath the black mouse pointer) is the mouse pointer that belongs to the Ubuntu itself. When I move the black mouse pointer in OS X Screen Sharing, the Ubuntu's white mouse pointer lags behind a fraction of a second. When I rest the OS X Screen Sharing black mouse pointer, the Ubuntu's white mouse pointer slides right underneath the black mouse pointer. So, as long as I am not moving my mouse, the two mouse pointers are completely synchronized with the black mouse pointer sitting on top of the white mouse pointer. But as soon as move the mouse, the white mouse pointer lags behind. The lag is minimal, but still this does not feel like working with Ubuntu natively. 

The Ubuntu LTS 12.04 is running as a VM on a server. I have another instance of OS X running as a VM on that same server, and when I access that instance of OS X with the Screen Sharing application running on my MacBook Pro, I have only one mouse pointer, and the impression is that I'm working with that instance of OS X natively - the same experience as when I navigate the screen on the MacBook Pro itself. 

Is there any way to synchronize the two mouse pointers completely to get a "native" desktop experience?
