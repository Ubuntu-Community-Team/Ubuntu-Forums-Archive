### Post by oldfred on 2019-02-18
At beginning of gdisk it says MBR, so you do not have gpt partitioning.
UEFI normally uses gpt and if Windows you have to have gpt for UEFI boot.
But Windows in BIOS boot mode requires the very old MBR partitioning.

You can convert MBR to gpt, but may have to reinstall boot loaders and make other fixes. Just make sure your backups are current.
       Converting to or from GPT
[http://www.rodsbooks.com/gdisk/mbr2gpt.html](http://www.rodsbooks.com/gdisk/mbr2gpt.html)

### Post by oldfred on 2019-02-18
The normal.mod error is usually where you have both BIOS boot  in MBR and UEFI boot, but only one actually works. So trying to boot in the other mode causes the error report.

You should have full backups whether changing or not. Hard drives fail, users make errors and systems just do something wrong sometimes. If you have a good backup, then those issues are not the end of the world.
