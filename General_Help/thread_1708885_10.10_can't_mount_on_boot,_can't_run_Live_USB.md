### Post by oldfred on 2011-03-18
Boot script has to be able to mount partitions to review them.

Several users had similar issues but were able to download Slitaz and run the e2fsck from that. It does not see something and then the filecheck works.

[http://www.slitaz.org/en/get/](http://www.slitaz.org/en/get/)

From liveCD so everything is unmounted,swap off if necessary, change sdb1 to your partition(s)
e2fsck is used to check the ext2/ext3/ext4 family of file systems.
sudo e2fsck -C0 -p -f -v /dev/sdb1
if errors:
sudo e2fsck -f -y -v /dev/sdb1

### Post by oldfred on 2011-03-21
Not sure what the problem is then. If you do not have data you have to recover then a clean install should be the quickest & easiest way to get it working again.

I prefer smaller system partitions. With Ubuntu I have several 20-25GB system partitions with different versions. Since my data is elsewhere I only have used about 6-7GB in each. 

For the Total space you want for Ubuntu:
Ubuntu's standard install is just / (root) & swap, but it is better to add another partition for /home:
1. 10-20 GB Mountpoint / primary or logical beginning ext4(or ext3)
2. all but 2 GB Mountpoint /home logical beginning ext3(or ext4)
3. 2 GB Mountpoint swap logical

Depending on how much memory you have you may not absolutely need swap but having some is still recommended. I do not hibernate (boots fast enough for me) but if hibernating then you need swap equal to RAM. And if dual booting with windows a shared NTFS partition is also recommended. But you usually cannot create that as part of the install, just leave some space. Or partition in advance (recommended).
One advantage of partitioning in advance is that the installer will use the swap space to speed up the install. Thanks Herman for the tip.
