### Post by paulmerchant on 2008-09-23
1. You may have a disk that didn't burn well. In that case, try using a slower burn speed.

2. and 3. You might not have very much RAM on the machine (live Ubuntu distributions run best when there's 512MB or more). Or you might have incompatible hardware that is going to require some tweaking. In either of those cases, you could simply try another live Linux distribution. GParted Live and RIPLinux are both live distributions that are made to help recover computers that have failed. They use less memory and boot a lot easier (because they aren't made to be full workstation environments).
