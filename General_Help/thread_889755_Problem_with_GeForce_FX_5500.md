### Post by WRDN on 2008-08-14
> **GoranI said:**
> I have UBUNTU 8.04 and graphic card nVidia GeForce FX 5500. After instaling ubuntu i can run only with max resolution of 800x600.I enabled restricted drivers for nVidia but after this resolution degraded on 640x... How can I solve this ???

Download the official driver from the nVidia website (it will be called [driver].run). Then:

- Press Ctrl, Alt and F1
- Login and then type:

```
sudo /etc/init.d/gdm stop
```

- Now, change to the directory of the nvidia driver you downloaded, and type:

```
sudo sh [driver].run
```

- Let it install and create a new xorg.conf file when it requests to.
- Now, type:

```
sudo /etc/init.d/gdm start
```

This will restart the Gnome Display Manager, and you should now be able to use a much higher resolution.

- go to Display configuration
- configure the screen the way you like it
- hit "Save to X configuration file"

### Post by stoneage on 2008-08-16
I had the same problem yesterday. I (eventually) resolved it by doing without the nvidia-xserver config. and installed nvidia-glx-new with synaptic, then copied sections of the original xorg.conf to the rewritten version to ensure my monitor and keyboard were correctly detected. Check to see that you have only one card driver installed. As I understand it the installed driver is autodetected at boot, but if two drivers are installed the system cannot use them both and will default to failsafe mode.
It also changed my keyboard layout from en to us.

You may also want to add 
	'Option 		"DynamicTwinView" "false"'

to 
        'Section "Device"'
in xorg.conf to ensure your monitor's refresh rate is correctly reported. It is almost certainly running at the appropriate rate but due to an incompatibility it is wrongly reported in System => Preferences => Screen Resolution
"DynamicTwinView" is enabled by default and commenting it out cures the mismatch.

Good luck.
	
Gainward 8800 GS on Hardy 64 bit.
