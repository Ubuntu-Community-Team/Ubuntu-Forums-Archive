### Post by cm0z on 2008-07-15
Hi 

I don't know if you found a solution.. but here's how I fixed the issue on my setup. 

I stumbled upon this thread, while searching for a solution to using TwinView and still having a Xinerama-like separation of the two screens, i.e. panels on separate screens and maximizing/fullscreen only on the screen which application is actually located.

If this is already been listed in a FAQ or sticky thread, I apologize.


I created a xorg.conf with the nvidia-settings cp, so both screens were listed in the conf. This added metamode settings on both screen0 and screen1. Twinview was set and enabled on screen0(XPS m1330 laptop screen), but not on secondary screen (screen1 - 46" Panasonic connected via HDMI - model  TH-46PZ85)

Then I edited /etc/X11/xorg.conf manually.

What seemed to fix my config, was settings this for both screens:

```

Option         "TwinView" "1"
Option         "metamodes" "nvidia-auto-select, nvidia-auto-select"

```

In short Twinview "1" on both screens and nvidia-auto-select instead of specific metamodes.
