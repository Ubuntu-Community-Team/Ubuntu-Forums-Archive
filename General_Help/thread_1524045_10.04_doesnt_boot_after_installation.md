### Post by oldfred on 2010-07-05
If you have multiple drives it is less important what you install first especially if you put windows on one drive and Ubuntu on another. It often is best but not required to have windows on sda and grub/Ubuntu on another drive. It also is slightly better to have grub in the MBR of the same drive as the Ubuntu install and then in BIOS set to boot that drive.

You can only have one boot flag per drive, but can have one on every drive. It just is that the windows boot loader will boot the active partition (boot flag) of the drive you boot from. Grub does not use a boot flag, but some BIOS do want a boot flag on a primary partition.

To install grub2 to sdd, and then set BIOS to boot sdd - 80GB drive in BIOS.

reinstall from working (not liveCD) system - first find Ubuntu drive:
sudo fdisk -l
if it's "/dev/sdd"  then just run:
sudo grub-install /dev/sdd
If that returns any errors run:
sudo grub-install --recheck /dev/sdd
Then:
sudo update-grub

So that on updates it reinstall grub to sdd:
sudo dpkg-reconfigure grub-pc
spacebar to choose drive sdd, enter to accept, do not choose partitions

When you want to install windows set BIOS to boot sda and then install windows to sda. It will put its boot loader into sda but when in BIOS you change back to sdd you will boot grub.
Once in Ubuntu run this and it will find your windows install and add it to your grub menu.
sudo update-grub
