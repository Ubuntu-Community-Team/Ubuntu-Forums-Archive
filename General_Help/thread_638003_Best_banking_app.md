### Post by oygle on 2007-12-23
> **Raja Mukherjee said:**
> 
If I want to switch over from Microsoft Money to kmymoney or some other linux banking package, is there a way I can import the MS Money data? Thanks.

If you have been using M$ Money, I'd suggest you go for KMyMoney, as GnuCash or Grisbi don't have the same functionality that you would be used to.  I changed over from Money to KMyMoney , after trialling both GnuCash and Grisbi. Neither of those 2 products could do what I had been used to using, with M$ Money.

So, ........... KMymoney is the best accounting system, IMO.

For importing, all you need to do is ...

1.  Export the transaction to a QIF file from M$ Money (one file per account).

2.  Import that same QIF file into  KMyMoney.

HTH

### Post by oygle on 2008-01-06
> **aonegodman said:**
> Well I took a look at KmyMoney and it looked good. I did an QIF import of my bank statement and it had problems with it. For one thing it insists on a date format of yyyy/dd/mm, which is common I understand in European countries. I had to go through every imported transaction and let it change the date.  Anyway . .  

You can actually set your own date format for the import. See [Why does the QIF import choke on a wrong date format?]("http://kmymoney2.sourceforge.net/faq.html#question4")

So, KMyMoney doesn't actually force a specific date format, you can set it to whatever your import file format has. This is in the QIF import profiles in KMM.

> **aonegodman said:**
> 
The biggest problem I had with it was that it would not allow me to just delete the checking account data and just start over. I wound up just removing the whole program.


As it is an initial import/setup, it would have been easier to simply delete the KMyMoney file and start again.

Before doing that, do an export and a backup, you can use the export data (a QIF file) to setup the category (account) structure again, very easily.

HTH
