### Post by florin on 2015-07-02
I have CouchDB 1.6.1 installed on Ubuntu 14.04. Recently I hit a limit and I need to pass ERL_MAX_PORTS as an environment variable to the couchdb daemon when it's starting, to fix the crash.

Looking at the couchdb package, I've noticed it has a /etc/init/couchdb.conf file and a /etc/init.d/couchdb script. What is the best place to put an environment variable so that the daemon can pick it up when it's being launched?

---

### Post by TheFu on 2015-07-02
I don't know anyting about couchdb, but /etc/init.d/couchdb-local is where I'd do it. Basically export the env variable, then call /etc/init.d/couchdb and pass all parameters.  You'll need to remove the couchdb startup using update-rc.d and add in your new command.

Why go to this effort?  Every time couchdb gets updated, it is possible for the init script to be replaced. By having your own startup/shutdown script, you've handled that.

Make sense?
