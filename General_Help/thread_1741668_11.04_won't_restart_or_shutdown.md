### Post by sikander3786 on 2011-04-29
We need to know where it gets stuck during the shutdown process and for that, we need to edit the /etc/default/grub file.

```
gksudo gedit /etc/default/grub
```

Find this line.

```
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"
```

Delete the words quiet splash so the line looks like

```
GRUB_CMDLINE_LINUX_DEFAULT=""
```

Save and close and run this command in Terminal.

```
sudo update-grub
```

Reboot. You'll still see the Ubuntu Logo Screen. Hard Reboot as it will get stuck most likely and once you see the desktop after reboot, shut it down once more. You'll not see the Logo Screen this time, instead it would be text. We need to see that text please.

### Post by BigErn on 2011-05-05
I had this problem, and it was due to the fact that I'm on a laptop with wifi networking but mounted some network shares through fstab.  Basically, when I try to shutdown/restart it's trying to close these cifs or nfs connections but since the window manager has been closed at that point the wifi connection is gone, so it just hangs.  Sometimes it would eventually finish shutting down after 10 minutes.  Anyway, this link describes the problem, and there are a number of good ways to work around it:

[http://ubuntuforums.org/showthread.php?p=10730790](http://ubuntuforums.org/showthread.php?p=10730790)

FYI, the solution I ended up using was to stop mounting network shares through fstab and use gigolo instead.  Now after thinking about it, I think mounting through fstab is in principle not such a great idea with laptops with very dynamic and transient network connectivity.

Try it out, I hope this helps you.

### Post by fabio.hipolito on 2011-05-25
Hi,

I'm running Ubuntu 11.04 amd64 on a Dell Optiplex 990, which had the same problem and none of the solutions above worked for me. I found a solution in bug report in [launchpad]("https://bugs.launchpad.net/ubuntu/+source/linux/+bug/768039")

Just add the option "reboot=pci" to GRUB_CMDLINE_LINUX in /etc/default/grub
to do this
```
sudo gedit /etc/default/grub
```
look for

GRUB_CMDLINE_LINUX=""

and change it to

GRUB_CMDLINE_LINUX="reboot=pci"

save, exit and update grub
```
sudo update-grub
```

Sorry for being bothering a solved thread.
Best regards.

> **fabio.hipolito said:**
> Hi,

I'm running Ubuntu 11.04 amd64 on a Dell Optiplex 990, which had the same problem and none of the solutions above worked for me. I found a solution in bug report in [launchpad]("https://bugs.launchpad.net/ubuntu/+source/linux/+bug/768039")

Just add the option "reboot=pci" to GRUB_CMDLINE_LINUX in /etc/default/grub
to do this
```
sudo gedit /etc/default/grub
```
look for

GRUB_CMDLINE_LINUX=""

and change it to

GRUB_CMDLINE_LINUX="reboot=pci"

save, exit and update grub
```
sudo update-grub
```

Sorry for being bothering a solved thread.
Best regards.
