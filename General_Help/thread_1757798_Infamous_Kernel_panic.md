### Post by Rubi1200 on 2011-05-14
Hi and welcome to the forums :-)

I don't see anything immediately wrong with the script results. And you are unable to access any of the kernels from the GRUB menu?

Let's try this from a LiveCD with Internet connection:

```
sudo mkdir /mnt/temp
sudo mount /dev/sda1 /mnt/temp
for i in /dev /dev/pts /proc /sys; do sudo mount -B $i /mnt/temp$i;  done
sudo cp /etc/resolv.conf /mnt/temp/etc/resolv.conf
sudo chroot /mnt/temp
```
Assuming no errors, the last command will give you a root prompt.

Then, run the following commands to try and do some housecleaning:
```
apt-get autoclean
apt-get clean
apt-get update
apt-get upgrade
apt-get dist-upgrade
apt-get install -f
dpkg --configure -a
```
If no errors are reported, then exit and unmount the filesystems:
```
exit
for i in /dev/pts /dev /proc /sys; do sudo umount /mnt/temp$i ; done
```
Reboot, taking out the CD, and let me know if you are able to get back into the Ubuntu install.
