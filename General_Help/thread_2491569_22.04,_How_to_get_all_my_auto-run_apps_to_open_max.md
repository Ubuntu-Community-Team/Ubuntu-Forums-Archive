### Post by MAFoElffen on 2023-10-13
Well it's not a simple question... 

On startup in Gnome, for most applications, there are no startup options to start as maximized. 

There is a way to hack that using xdotool or dev2spie2, but those are going to take some playing around with to get to work.

The hot-key combination in Gnome to maximize is <Super><Up-Arrow> and to minimize is <Super><Down-Arrow>. The maximize is written as "Super_L+Up".

So if I did
```

xterm & xdotool search --sync --onlyvisible --class "xterm" windowactivate key "Super_L+Up"

``` 
It started xterm (waited for it to start) then did a keypress of <Super><Up> to maximise it. The trick is to figure out how xdotools see's the window to ID it in the class option for what you are running.

To use it, you would have to install xdotool, then edit the autostart execute command for each application that you want to start as maximized.

I looked at devilspie2, and it works, but, would be a very steep learning curve for you to use. Would require you to manually write lua scripts that would be i it's user config folder... Once learned, it can do quite a lot.

EDIT:
This also works...
```

firefox & xdotool search --sync --onlyvisible --class "Firefox" windowactivate key "Super_L+Up"

```

### Post by dragonfly41 on 2023-10-14
Combining the two ideas above of Stacer and xdotool my revised thought is to user Stacer as your startup app manager ..  and then instead of pointing to the usual launch location, insert a combined script such as given earlier

firefox & xdotool search --sync --onlyvisible --class "Firefox" windowactivate key "Super_L+Up"

This way you have all your apps in one Stacer panel .. either direct to app or via xdotool addon. All can be toggled on or off.

P.S. I do not use Wayland but understand that there are issues of xdotool on Wayland.

[https://superuser.com/questions/1032270/wayland-alternative-for-xorgs-xdotool](https://superuser.com/questions/1032270/wayland-alternative-for-xorgs-xdotool)

An alternative tool to use is the old Actiona tool in the repo (I use Actiona extensively for desktop automation) and in there you can create a list of commands which launch startup apps and setup the status at launch.
