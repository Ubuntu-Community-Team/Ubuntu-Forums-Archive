### Post by Daniao on 2007-04-21
Hi all,

I just upgraded from dapper to feisty to try the so famous beryl, but unfortunately it is not working. Here is my story:

I have a thinkpad t43, that uses an ATI graphic card. Therefore the drivers for my graphic card are not enabled in Feisty by default, by if I run "restricted-manager" I can enable them. So far so good. If I enable the proprietary drivers then the 3D acceleration works, proof of it is:

#glxinfo | grep rendering
direct rendering: Yes

And I can run Google Earth perfect.

Now, I want to try the desktop effects, so if I go under System->Preferences-> Desktop Effects, it tells me that Composite Extensions is not enabled. Then I go to my /etc/X11/xorg.conf and I Enable the Composite extensions, by doing "Composite" "Enable" in the correspondent section. What happens then is:
1) Google Earth stops working
2)Now I get the dialog to enable Desktop Effects, but when I try to enable it tells me it says it can not

I also tried with beryl, I followed the usual guides to install, and it installed without problems. I select the beryl-manager but when I want to load beryl, it crashes, well basically nothing happens and it falls back to Metacity.

Any ideas that could help?

Thanks!


Daniel
