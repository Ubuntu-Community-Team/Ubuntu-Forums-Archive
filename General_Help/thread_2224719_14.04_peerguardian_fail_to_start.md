### Post by jre on 2014-05-20
The advice you saw is for temporarily unavailable blocklists. But the atma blocklist was removed/receives no more updates. So just remove the atma entry in /etc/pgl/blocklists.list. I've done so already for the next release of pgl.


EDIT: I contacted iblocklist and the atma list is now available again. So there is no need to remove it any more if you choose to use this list. Still continue with the following:


After that issue a ```
sudo dpkg-reconfigure --force pglcmd
```. you may need to do a "sudo pglcmd stop" first, in case the daemon status is in a broken state now.

Besides that, I'm a bit puzzled by some output you posted (probably it is not relevant to your current problem, but I'd like to understand it):
First there is output that pgl will be started, then there is output that pgl is configured not to start. Both things are fine, but only one of them should be shown. Therefore can you please post the output of ```
pglcmd show_config|grep INIT
```.
What is the init system on your machine. I guess it is "upstart"? (I'm on debian, never used Ubuntu for myself so I'm not totally familiar with it)
