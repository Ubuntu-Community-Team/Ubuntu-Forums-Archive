### Post by Wim Sturkenboom on 2012-09-30
One of your problems will be that cvt tells you that the 'required' horizontal sync is 65.29kHz. If you check your xorg.conf, horizontal sync is limited to 55kHz max.

So that's never going to work ;)

You refer to Windows, but is that running on the same system? If so, it's safe to change the high value of the horizontal sync in xorg.conf to around 70kHz.

In the device section of the xorg.conf, you can force the system not to use the edid information

```

Section "Device"
    Identifier     "Device0"
    Driver         "nvidia"
    VendorName     "NVIDIA Corporation"
    BoardName      "GeForce GTX 670"
*[COLOR="Red"]    Option         "UseEDID"   "FALSE"[/COLOR]*
EndSection

```

Next you can add a modeline to the monitor section (note that this is just an example, use your own values)

```

Section "Monitor"
    Identifier     "Monitor0"
    VendorName     "Unknown"
    ModelName      "CRT-0"
    HorizSync       28.0 - 55.0
    VertRefresh     43.0 - 72.0
    Option         "DPMS"
*[COLOR="Red"]    Modeline    "1920x1080_60.00"  173.00  1920 2048 2248 2576  1080 1083 1088 1120 -hsync +vsync[/COLOR]*
EndSection

```

And next add the mode to the screen section (based on above example, so adjust)

```

Section "Screen"
    Identifier     "Screen0"
    Device         "Device0"
    Monitor        "Monitor0"
    DefaultDepth    24
    Option         "Stereo" "0"
    Option         "nvidiaXineramaInfoOrder" "CRT-0"
    Option         "metamodes" "1152x864 +0+0; nvidia-auto-select +0+0"
    SubSection     "Display"
        Depth       24
*[COLOR="Red"]        Modes      "1920x1080_60"[/COLOR]*
    EndSubSection
EndSection

```

I can not help with the xrandr error.
