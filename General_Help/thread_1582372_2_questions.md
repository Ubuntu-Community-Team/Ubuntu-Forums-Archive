### Post by Lateralis on 2010-09-26
> **Delsos said:**
> 1. I want to make some partitions to my 750gb HDD....i want a partition for my windows 7, parition for ubuntu and parititon for alll my DATA....
so how big should i make each partition? 


Ubuntu installations can be sub partitioned if you fancy doing something fancy.  Otherwise, how big each partition needs to be depends on what you are going to be using each OS for and how much data storage you actually need.  What I would do though is make a small Windows partition (40 gigs or so), a 40 gigs Ubuntu partition, a swap area equal in size to at least your RAM (so a swap of 4 gigs if you have 4 gigs of RAM) and the rest as a shared space.  You can install any games under Windows to that shared partition as well, so 40 gigs for Windows will be enough for office software and all the other stuff you might need.  40 gigs for Ubuntu again will probably be enough.  

If you want to do something fancy with your Ubuntu installation then report back. =)  

[quote=Delsos]
2. Should i use 32 or 64 bit ubuntu? are the rules the same as for windows? where 64bit is faster and if your processor supports 64 bit u should get it?
 
thanks[/QUOTE]

In my experience, you don't notice a speed increase from 32 to 64 bit.  The one thing to consider is that Adobe do not make a 64-bit Flash plugin, so Flash support on 64-bit Linux is a little ropey.  There are ways and means around this, but it is something to bear in mind. 

So to be honest, I don't think there's too much between them.  If in doubt, use the 32-bit - the speed increase between the two will hardly be noticeable.
