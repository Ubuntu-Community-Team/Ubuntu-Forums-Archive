### Post by Chronon on 2009-12-18
Hi.  You know a lot more about potential problems with this software than I do, I'm sure.  I certainly don't want to lead you astray and don't know anything about the stability of this software.  However, assuming that the software works, it appears to me that this procedure should work to give you a launcher that will work:
```
sudo gedit /usr/local/bin/jbidwatcher
```
Then paste this code, changing the "/path/to" as appropriate, into the document, save and quit:
```
#!/bin/bash
java -jar -Xmx512m /path/to/JBidwatcher-2.1pre4.jar

```
Make it executable,
```
sudo chmod +x /usr/local/bin/jbidwatcher
```.

Now you should be able to run it from the terminal just by typing "jbidwatcher" at the prompt.

I use KDE, so I'm not as familiar with .desktop launchers, but I believe that saving
```
[Desktop Entry]
Comment=eBay bidding, sniping & monitoring
Name=jBidWatcher
Exec=jbidwatcher
Terminal=false
Encoding=UTF-8
Type=Application
Icon=/usr/local/share/java/jbidwatch.jpg
Categories=Network
```
as jbidwatcher.desktop to your desktop should give you a launcher icon.  You will need to make sure that the "Icon=" line contains the correct path to the image that you want to use.  You should be able to create a menu launcher too.  I do have Gnome on my netbook (which isn't here right now), so I can give you more explicit instructions later if necessary.

This just provides an automated way to run the command that you already tried.  I don't think there's any way for this to mess up your java installation, but I don't know anything about the stability of the software itself.

### Post by Chronon on 2009-12-18
It's mainly just organization.  All that's happening in my previous post is that you're creating a launcher script in /usr/local/bin (the standard place for user-created exectubles).  Then this gets called by a .desktop launcher.  Since  "java -jar -Xmx512m /path/to/JBidwatcher-2.1pre4.jar" launches the program from its current location, it clearly doesn't invoke relative paths that would get messed up by placing it in a different directory.  You can really put this anywhere you deem convenient.  

By the way, you could put the image for the icon absolutely anywhere.  Don't let a difference of /usr/local/share to /usr/share disturb you too much.  You could just stick the image in your home folder somewhere if you want and put the right path into the .desktop file.

If "java -jar -Xmx512m /path/to/JBidwatcher-2.1pre4.jar" would cause damage to your system it would have done it already.  Creating a launcher script as described above won't cause any additional damage.

### Post by bobbiescap on 2009-12-19
Hi, as an expatriate sandgroper I may be able to put this in a more sandgroper-understandable language. This is how I set up JBidwatcher, perhaps it is not quite correct protocol but it works fine. Particularly if you use a separate home partition so you don't lose your stuff during upgrades etc.

I am one of those people who really enjoyed the old MacIntosh computers with the nice GUI and was really pleased when Microsoft moved from DOS to Windows. I avoid the command line when I can, it is very unforgiving and I can't be bothered learning the lingo well enough to be adept, therefore I am not going to give directions for every step, if you like the command line you will use that, if you prefer the GUI you will use that.

First of all make a folder in your home directory called ~/bin and place your copy of JBidwatcher in there. Go to the JBidwatcher website and download the icon image (jbidwatch.jpg) to the newly created bin folder.

For Gnome it is easier to put the icon in the default folder, the dreaded command line is probably easier for this one:

```
sudo cp ~/bin/jbidwatch.jpg /usr/share/icons/hicolor/scalable/apps/jbidwatch.jpg
```

Next open a text editor as a normal user and type or paste):

```
#bin bash
java -jar ~/bin/JBidwatcher-2.1pre4.jar
```

Save it as JBidwatcher.sh to your bin folder and make it executable, either by altering the file properties in a file manager, or at the command line.

Next RH click on the menu icon and open the edit menu screen. Single click on the "Internet" icon and then select "new item".

Call it jBidwatcher, set the command by browsing to the jBidwatcher.sh file in the bin folder, or type the path in the "command' field. In the description put something like "eBay bidding and sniping", then click on the icon with the spring at the top left, you should be able to scroll down the list and find the jbidwatch.jpg icon there, even though it is not a vector graphic, select it and save the new item.

You should now have an entry in the Internet section of the menu that will launch the application. Please note that JBidwatcher is not installed as such but runs on the JRE environment. You can follow this procedure for any Java application like Buddi.

Mate I am pretty tired writing this so please let me know if there are any problems with the details and I will troubleshoot.

All the best and c'mon the Dockers!
