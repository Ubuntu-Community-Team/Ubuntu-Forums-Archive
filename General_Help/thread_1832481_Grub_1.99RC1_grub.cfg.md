### Post by ossgrouch on 2011-08-24
[FONT="Arial Black"][SIZE="2"]**Grub 1.99RC1 grub.cfg**[/SIZE][/FONT]

I have been playing around with the Grub boot screen for a while (lots of waisted time indeed). I upgraded, rather started with a clean slate, to Kubuntu Natty (11-04) 64-bit, which came with Grub 1.99RC1. And I lost my favorite customized Grub log in screen of course (background, resolution, menu order and text color).

Keep in mind that if everything goes awry, you can start with your install disk and reinstall Grub only.

The GNU Grub 1.99 Manual can be found at: [http://www.gnu.org/software/grub/manual/grub.pdf]("http://www.gnu.org/software/grub/manual/grub.pdf")

It contains lots of information on the commands, valid colors etc., but not necessarily on where in the grub.cfg file everything needs to be entered.

[FONT="Arial Black"][SIZE="2"]**First, Grub 1.99 Background image:**[/SIZE][/FONT]

That was easy, drs305 had information on Ubuntu Forums:  [http://ubuntuforums.org/showthread.php?t=1739412&highlight=Grub+Drop-In+Backgrounds+Font+Selection]("http://ubuntuforums.org/showthread.php?t=1739412&highlight=Grub+Drop-In+Backgrounds+Font+Selection")

Procedure:

1. Copy any image (jpg, png or tga) to the /boot/grub folder. It seems it only works if you make sure that the image name has the correct three letter extension, .jpg, .png or .tga

2. Run "sudo update-grub".

3. Done. How easy was that?

In the example below of my grub.cfg myimage.png entry was added on the 6th line of the ### BEGIN /etc/grub.d/05_debian_theme ### section.

[FONT="Arial Black"][SIZE="2"]**Second,screen resolution:**[/SIZE][/FONT]

You can change the file /etc/default/grub by uncomment the "GRUB_GFXMODE=" line and enter your desired screen format, but when you run "sudo update-grub" it makes a new copy of /boot/grub/grub.cfg which removes any changes already done. So why not just enter it into grub.cfg. In the section ### BEGIN /etc/grub.d/00_header ### about 36 lines down is the statement "set gfxmode=auto", I changed that to my desired 800x600. Make sure it is a value supported by your monitor and graphics driver. (Open System Settings, in the Hardware section open Display and Monitor, and in the Size & Orientation tab you can find your supported screen resolutions).

[FONT="Arial Black"][SIZE="2"]**Third, menu order:**[/SIZE][/FONT]

Menu order is given ### BEGIN /etc/grub.d/00_header ### section on line 5 by the statement "set default="0"". The application StartUp-Manager didn't get it right, so I counted the menu entries in grub.cfg in the sections beginning with ### BEGIN /etc/grub.d/10_linux ### , counting the sub menus as one entry, and my Windows menu was number 6. Grub starts counting with 0, so by changing set default="0" to set default="5" the grub menu would now automatically go to the Windows entry.

**[FONT="Arial Black"][SIZE="2"]Fourth, text color:[/SIZE][/FONT]**

Neither StartUp-Manager nor Grub Customizer would do that, so I added my favorite colors my self in the section ### BEGIN /etc/grub.d/05_debian_theme ### right after the background image line, see example. Keep in mind that black is transparent on top of an image.

# DO NOT EDIT THIS FILE
#
# It is automatically generated by grub-mkconfig using templates
# from /etc/grub.d and settings from /etc/default/grub
#
### BEGIN /etc/grub.d/00_header ###
if [ -s $prefix/grubenv ]; then
set have_grubenv=true
load_env
fi

*set default="0"*
if [ "${prev_saved_entry}" ]; then
set saved_entry="${prev_saved_entry}"
save_env saved_entry
set prev_saved_entry=
save_env prev_saved_entry
set boot_once=true
fi

function savedefault {
if [ -z "${boot_once}" ]; then
saved_entry="${chosen}"
save_env saved_entry
fi
}
function recordfail {
set recordfail=1
if [ -n "${have_grubenv}" ]; then if [ -z "${boot_once}" ]; then save_env recordfail; fi; fi
}

function load_video {
insmod vbe
insmod vga
insmod video_bochs
insmod video_cirrus
}
insmod part_msdos
insmod ext2
set root='(/dev/sdb,msdos2)'
search --no-floppy --fs-uuid --set=root ee984ebd-8682-4ff1-b5f3-30f73cca2f90
if loadfont /usr/share/grub/unicode.pf2 ; then
*set gfxmode=auto*
load_video
insmod gfxterm
fi

terminal_output gfxterm
insmod part_msdos
insmod ext2
set root='(/dev/sdb,msdos2)'
search --no-floppy --fs-uuid --set=root ee984ebd-8682-4ff1-b5f3-30f73cca2f90
set locale_dir=($root)/boot/grub/locale
set lang=en_US
insmod gettext
if [ "${recordfail}" = 1 ]; then
set timeout=-1
else
set timeout=10
fi

### END /etc/grub.d/00_header ###
