### Post by srs5694 on 2011-09-05
If the original Windows system booted with UEFI and if it's still booting in UEFI mode, I'm rather surprised that your computer boots at all. Windows won't install to MBR disks in UEFI mode, and without an EFI System Partition (ESP; the 100 MiB partition you mentioned but that you've clearly deleted), a UEFI implementation probably shouldn't boot from the hard disk. My best guess is that your UEFI implementation has some sort of fallback to boot from BIOS-style boot code in the partitions it finds, and that you've somehow got enough there for it to work. That's a wild guess, though; it seems improbable, but I haven't thought of a better explanation.

If I'm right in these guesses, then your configuration is extremely delicate and non-standard. Therefore, fixing it could be very difficult when it breaks -- and it's almost certain to break at some point as you fiddle around with trying to install Linux, or in the future when you install Linux or even Windows software updates. Therefore, my recommendation is to start again from scratch. You have two basic options:


[list]
[*]Reconfigure your firmware to boot in BIOS mode and do a standard BIOS-style installation of Windows followed by Ubuntu on an MBR disk. This has the advantage of being a well-travelled path, but if you replace your boot disk with an over-2TiB model in the future, it'll be harder to get it to work, since you'll need to switch to UEFI booting and GPT at that time.
[*]Keep your system configured to boot in UEFI mode (or reconfigure it to do so, if I'm wrong and it's set up to boot in BIOS mode now) and do UEFI installations of Windows and Ubuntu. This *is* possible, but because Ubuntu (including the 11.10 beta 1) has rather flaky UEFI support, there are significant hoops to jump through. The easiest way, in broad outline, is probably to install Ubuntu in BIOS mode to a GPT disk, which goes much more easily if you create a [BIOS Boot Partition](http://en.wikipedia.org/wiki/BIOS_Boot_partition) in addition to the ESP and other Linux and Windows partitions. Once this is done, you can install a UEFI-capable Linux boot loader. Personally, I recommend [ELILO](http://elilo.sourceforge.net) for this, since it's more reliable than GRUB 2 in my experience. Unfortunately, ELILO can't chainload to the Windows EFI boot loader, so you'll need to either rely on your firmware to select which OS to boot or install another EFI boot loader, such as [rEFIt,](http://refit.sourceforge.net/) which has as a sole purpose the selection of other EFI boot loaders. If you set rEFIt as the default boot loader, you'll be able to use it to choose between Linux and Windows. When you select Linux, ELILO will launch and start up Linux.
[/list]


Overall, the BIOS compatibility approach is likely to be the easiest to make work in the short term. UEFI mode may offer advantages in the future, but right now it's pretty much "bleeding edge," owing largely to the fact that the Ubuntu developers haven't yet polished the numerous rough edges off of their UEFI support. (Linux tools to do it right exist; it's just that the Ubuntu installer doesn't use them correctly.)

### Post by oldfred on 2011-09-05
That now looks like a standard BIOS/MBR(msdos) install. If you install grub2's boot loader to the MBR you should be able to boot Ubuntu & windows.

Make a windows repair CD so you can reinstall Windows MBR if necessary.

Make your own Windows recoveryCD/repair:
[http://windows.microsoft.com/en-GB/windows7/Create-a-system-repair-disc](http://windows.microsoft.com/en-GB/windows7/Create-a-system-repair-disc)
[http://forums.techarena.in/guides-tutorials/1114725.htm](http://forums.techarena.in/guides-tutorials/1114725.htm)

Windows 7 repair USB, Also Vista if service pack installed
[http://www.intowindows.com/how-to-repair-windows-7-from-usb-flash-drive-repair-without-installation-dvd-disc/](http://www.intowindows.com/how-to-repair-windows-7-from-usb-flash-drive-repair-without-installation-dvd-disc/)
[http://www.webupd8.org/2010/10/create-bootable-windows-7-usb-drive.html](http://www.webupd8.org/2010/10/create-bootable-windows-7-usb-drive.html)

Then use your Ubuntu liveCD to install grub2's boot loader to the MBR.
Note you are using Oneiric and grub 1.99 which is still beta and subject to breakage.

#Comments are anything after the #, enter commands in terminal session
#Install MBR from LiveCD, Ubuntu install on sda5 and want grub2's bootloader in drive sda's MBR:
#Find linux partition, change sda5 if not correct:
sudo fdisk -l
#confirm that linux is sda5
sudo mount /dev/sda5 /mnt
#If grub 1.99 with Natty uses boot not root.
sudo grub-install --boot-directory=/mnt/boot /dev/sda

# If no errors on previous commands reboot into working system and run this:
sudo update-grub
