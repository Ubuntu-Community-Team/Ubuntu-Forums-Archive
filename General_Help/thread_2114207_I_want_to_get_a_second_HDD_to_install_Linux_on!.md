### Post by oldfred on 2013-02-10
Yes, but we seem to find that too much writing into Windows can cause issues.

The issue may be more that some hibernate and that will always cause an issue if you attempt to write. I think now the NTFS driver will check for hibernation and not let you write into the system partition. 
Also the Linux NTFS driver shows everything. Like in Windows unhiding all the system files & folders. But then it is easier to accidentally move or delete a system file. 

We suggest setting Windows 7 system partition as read only and mount with fstab. Auto mounting with Nautilus will make it read write.
And then create another NTFS shared data partition for any data you want to read and write from both systems. I use that shared partition for my Firefox & Thunderbird profiles for the last 6 years with XP, so I had same email & bookmarks in all my systems.

       Ownership and permissions of vfat / ntfs are set at the time of mounting. This is often a source of confusion.

UUID=xxxxxxxxxxxxxxx /WinC ntfs defaults,noauto,ro,umask=227 0 0
