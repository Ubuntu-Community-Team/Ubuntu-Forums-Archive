### Post by 3rdalbum on 2010-08-03
I briefly tried PC-BSD a while back (less than a year ago). The installation went fine, and I got a usable desktop. Program selection was a bit weird - the system includes an icon editor and several text editors, which is a bit of a throwback to pre-Ubuntu days. But the biggest problem was software availability; there are very few PBI packages available for PC-BSD and you're basically expected to activate Ports (FreeBSD's source package system) in order to get anything useful. Not quite what I was after, so I removed it.

### Post by Dustin2128 on 2010-09-23
well the BSD family of OSes, while still free in the same way as ubuntu (and more permissive in many ways), is totally different and quite a bit older than linux. A good explanation I've read on these forums:
> Linux is basically when a bunch of PC hackers got together and wrote a unix-like OS for the PC

BSD is when a bunch of unix hackers got together and wrote a unix-like OS for the PC
Aside from that, BSD usually lags a bit in hardware support and has few pre-built software packages available, if any (though it is largely binary compatible with linux). It does have a ports collection upon which gentoo is based, though it's not the same thing as a repository.
[http://en.wikipedia.org/wiki/Ports_collection](http://en.wikipedia.org/wiki/Ports_collection)

If you would like to make the swap, I'd recommend an intermediate OS like slackware or gentoo, to help out the learning curve a bit (e.g. retraining you to compile instead of apt-get). But in all honesty, I have yet to see an advantage over linux.
