### Post by drs305 on 2009-04-12
> sudo mount -a /media/500g drive

Actually, the command should be:
```
sudo mount -a
```
The "-a" switch tells the mount command to mount *all* - everything listed in fstab.

Looking at your fstab entries, does the mount point /media/500g exist? You could also add "uid=1000" and "users" to the entry so that you are the owner of the mounted folder and non-root users could mount it. The entry would look like this:
> 
/dev/sda1 /media/500g ntfs-3g defaults,users,uid=1000,locale=en_US.UTF8 0 0 

This assumes you are user 1000. To check, run "id" in a terminal.

### Post by drs305 on 2009-04-12
The unclean shutdown is recorded in a log file. Instead of the force option, you can clear it with an app called "ntfsfix", which is part of the ntfsprogs app.

```

sudo apt-get install ntfsprogs 
sudo ntfsfix /dev/sda5
sudo mount -a

```

Since you didn't get any error message about sda1, which is the partition I think you were first concerned with, it should be mounted on /media/500g.

Apparently it never got to sda1 to report that it also was the subject of an unclean shutdown. Run the same ntfsfix command on sda1.

Now, it appears that there is no mount point "/media/files". ntfs-config should have created the mount points when you ran it. Since it didn't or you subsequently changed things, we can make the mount point after correcting sda1's unclean shutdown.

```

sudo ntfsfix /dev/sda1

sudo chown -R konan: /media/files  # not required for ntfs but will make you the owner anyway

```
