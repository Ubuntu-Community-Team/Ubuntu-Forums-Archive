### Post by kpm on 2005-12-10
Hello,

On a Windows's machine running Firefox, a web page that displays a series of embedded Quicktime H.264 files that have no controls showing, just the instructions to 'click the image to play the video', the page loads and when one clicks the image (the freeze frame of the video I suppose), the video plays.  On my recent Ubuntu Linux install, if I visit this web page, (a blog that has numerous Quicktime files on its main page ([www.onegoodmove.org/1gm/)](www.onegoodmove.org/1gm/))) all of the videos start up automatically and chaos reigns!  The Mplayer plugin displays  controls, so I can scroll down and click 'pause' on each embedded video, but when 10 or so of these go off at once, resources are dwindled (old machine) and ... well, it is a pain, so I don't go to that web site unless I am infront of a windows machine... does anyone know if there is anything I can do to Mplayer's configuration to prevent this??

Thanks

### Post by mustang on 2005-12-10
There is a possible solution:

Download this extension for firefox:
[https://addons.mozilla.org/extensions/moreinfo.php?id=446](https://addons.mozilla.org/extensions/moreinfo.php?id=446)

Now whenever there is embedded video, there will be a link for you click to play it. You can specify in the extension's settings what player you want used for what type of file (in your case, make mplayer handle quicktime)
