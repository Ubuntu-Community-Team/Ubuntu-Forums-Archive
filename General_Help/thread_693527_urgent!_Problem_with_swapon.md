There it's advised to do what follows: 
```

1. Found out where my swap file was mounted:
swapon -s
2. Turned off swap and unmounted it
swapoff /dev/sda5
3. Cleared out any corrupt resume image in the swap file as follows:
mkswap -c /dev/sda5 # NOTE, I'm assuming mkswap will clear any corrupt resume image!
4. Turned swap back on, mounting the newly created swap file:
swapon UUID=/dev/disk/by-uuid/... # used old UUID, as new links not yet updated
5. Check that swap is turned on and swap file mounted okay:
swapon -s
6. Updated both /etc/fstab and /etc/initramfs-tools/conf.d/resume to use the new swap
file's UUID, as generated by mkswap, above (step 2).
7. Regenerated the initramfs image:
update-initramfs -u
8. Rebooted.
```
