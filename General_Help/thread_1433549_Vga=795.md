### Post by prodigy_ on 2010-03-21
You see, **/boot/grub/grub.cfg** is assembled from other files (you can found them in **/etc/grub.d**) by "update-grub" command. There's little purpose in editing grub.cfg directly, because with every kernel update you'll be back to square one.

Instead, you should to check **/etc/default/grub** file. Specifically, "GRUB_CMDLINE_LINUX" and "GRUB_CMDLINE_LINUX_DEFAULT" variables. Remove any wrong options from them and run update-grub.

### Post by oscarmeyer51 on 2010-10-24
FYI

On my system, the problem was that /etc/default/grub stil had the vga=795 line in it. 

Don't know if that was there prior to 10.04 or not. Here was my fix.

From a terminal window, do the following:

1) make a backup copy of /etc/default/grub (sudo cp -p /etc/default/grub /etc/default/grub.bkup)
2) manually edit the /etc/default/grub file (sudo gedit /etc/default/grub)
3) find the line that has the 'vga=795' and replace that part of the line with the new set command as below:

Old line:
GRUB_CMDLINE_LINUX="splash quiet vga=795"

New line:
GRUB_CMDLINE_LINUX="splash quiet set gfxpayload=1280x1024x24,1280x1024"

4) update grub with the command 'sudo update-grub'
5) restart your computer

PS. Can a system person get this noted as to be included in some future update so /etc/default/grub is built properly from the repositories? 

Thanks!
