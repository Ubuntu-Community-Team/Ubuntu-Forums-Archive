### Post by 3Miro on 2009-04-14

```

       capabilities: pm agp agp-2.0 bus_master cap_list
       configuration: latency=32 maxlatency=1 mingnt=5

```

You have GeForce4 MX, which is somewhat old card and it might be part of the problem. The driver for those is version 96.* (1.77 is for the new nVidia cards).

We can try something, lets hope it works:

Go to Terminal and do the following:
```

sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf_mybackup
gksudo gedit /etc/X11/xorg.conf

```

This creates a backup copy of xorg.conf and then open it for editing with gedit. Now be careful, mistakes in this file can prevent X server form starting.

Change your file's Device section to look like mine:
```

Section "Device"
        Identifier      "Configured Video Device"
EndSection

```
to look like:
```

Section "Device"
        Identifier      "Configured Video Device"
        Driver  "nvidia"
EndSection

```
Pay attention to tabs and alignment. Then save and reboot. (Read the rest first)

If the driver is already installed and there was an error setting the xorg.conf, this should fix it. 

Try changing the resolution after reboot. If it still doesn't work, you can try changing the Screen section to add DefaultDepth just like in my file. Create another backup first, but name it mybackup2 or something.

The bad scenario is if the X server doesn't boot at all. It may try coupld of times and fail or keep on trying to startx and fail. Press Ctr + Alt + F1 to go to simple terminal. You can log in and now restore the old X server settings:
```

sudo cp /etc/X11/xorg.conf_mybackup /etc/X11/xorg.conf

```
Then reboot again by typing:
```

sudo shutdown -r now

```
Go back to the graphical enviroment and try again. It may be that you edited the file improperly.

Hopefully you will not have to do the bad scenario, but just in case.
