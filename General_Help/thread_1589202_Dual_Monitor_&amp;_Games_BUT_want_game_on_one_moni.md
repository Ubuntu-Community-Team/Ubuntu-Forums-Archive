### Post by Grenage on 2010-10-06
I had this, long ago (when I had two screens).  I actually put it down to applications rather than the OS, since it didn't happen with 'everything'

What I ended up doing was adding another metamode to disable the second monitor, and toggling it with *xrandr -s etc*.  It was all I ever got to work.

I'm not sure how I managed to solve it, or exactly what the problem was. But I'll post here the xorg.conf file before and after.


But first,

this is how I got it working the way I wanted.


1: Launch NVida X Server Settings

Choose X Server Display Configuration

Under the "Display Tab" choose Configure

Choose Seperate X Screen

Apply and save the new settings and restart GDM

(from terminal, sudo restart gdm)

2: Launch Nvida X Server settings again when you log back in

CHoose X Server Display COnfiguration

Turn on Xinerma

Apply and save

Restart X Again

3: Launch NVida X Server settings once more

Back to X Server Display COnfiguration

Choose Configure again

Turn back on TwinView

Apply and save those settings

Restart GDM once more


Now, I dont know why this worked (maybe it filled in some settings automaticly for me somewhere between one of the steps) but now its working as intended.
