### Post by 3Miro on 2012-07-09
All modern systems can run both 64-bit and 32-bit. Your i5 CPU can run either. The AMD64 is just the name for the 64-bit standard, AMD64 is the 64-bit version for any CPU.

64-bit gives you boost in speed vs slightly increased RAM usage. Unless you are hitting a rare driver bug, if you have at least 3GB of RAM, then go for 64-bit.

### Post by 3Miro on 2012-07-09
> **misterplayb said:**
> i have 4gb of ram but the system monitor tells me i only "have/can use" 2,7gb

If you have a 32-bit system, then you will be limited in the RAm that you can use (depending on how much video RAM you have). You can use the 32-bit PAE kernel, which is a hack that lets you use more memory on a 32-bit system, but you do take a performance hit.

Some of the old 64-bit systems were also limited on the amounts of RAM that can be used, unless you enable "memory remapping" in the BIOS. However, your CPU is new enough, so it shouldn't be a problem.

### Post by 3Miro on 2012-07-09
> **misterplayb said:**
> will i have compatibility isues with the software in the ubuntu software center using 64 bit os?

Ubuntu has two copies for most software in the Software Center. One copy for 32-bit and one copy for 64-bit. On Windows most of the software is still 32-bit even if Windows is 64-bit, this used to cause issues in the past. By now, most of those have been resolved, however, you if you are using 32-bit software on a 64-bit machine, you are not getting the full speed boost. Ubuntu comes with 64-bit software for almost all of its software.
