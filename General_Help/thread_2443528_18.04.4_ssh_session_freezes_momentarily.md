I have an Ubuntu 18.04.4 system that I connect to with `ssh` frequently from my Windows 10 client.

I leave the `ssh` session open lots of times. Often, when I go to the `ssh` window after not having been using it for awhile (maybe 15 minutes) and type a command, the session doesn't respond at all. Keystrokes are not even echoed back. But if I wait about 15 seconds, suddenly it responds. If I'm then actively working in the shell session, it all works fine. If I leave it alone for awhile, then come back to it, it's temporarily frozen again and doesn't respond for about 10-15 seconds after I type.

I've done a lot of searching and can't find this problem described anywhere else. Any ideas on what might be causing this?
