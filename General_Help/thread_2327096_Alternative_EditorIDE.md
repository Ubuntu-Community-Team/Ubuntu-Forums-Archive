### Post by TheFu on 2016-06-07
Welcome to the forums.

Did you consider just passing in a different config file to geany for the different purposes? Should work.

I use the OS as an IDE, not any specific program.
Open 3 terminals.
1) editor
2) make/build 
3) debugger

slide the mouse between these to use each.  This has been the way Unix-types do development since X/Window became available.  Prior to that, they'd use something like emacs and change "buffers" between the different needs.

The issue is you are asking for _exactly_ the same for asm. Doesn't exist outside some unknown geany ASM extensions.
