### Post by jesushero on 2010-10-25
When I installed Ubuntu, I decided to enable the root account at install time. I rarely use it, but on some occasions it can be useful, so usually I enable it later on, on an already installed system. 

Well, enabling it at install has made it so whenever I try to run an administrative task on GNOME it asks for the root password instead of the sudo (user) password. Is there any way I can change this behavior so it can ask for the sudo password instead?

### Post by sisco311 on 2010-10-25
> **jesushero said:**
> When I installed Ubuntu, I decided to enable the root account at install time. I rarely use it, but on some occasions it can be useful, so usually I enable it later on, on an already installed system. 

Well, enabling it at install has made it so whenever I try to run an administrative task on GNOME it asks for the root password instead of the sudo (user) password. Is there any way I can change this behavior so it can ask for the sudo password instead?

Check the /etc/polkit-1/localauthority.conf.d/ directory. You should see two files: 50-localauthority.conf and 51-ubuntu-admin.conf.

50-localauthority.conf:
```
# Configuration file for the PolicyKit Local Authority.
#
# DO NOT EDIT THIS FILE, it will be overwritten on update.
#
# See the pklocalauthority(8) man page for more information
# about configuring the Local Authority.
#

[Configuration]
AdminIdentities=unix-user:0

```

51-ubuntu-admin.conf:
```
[Configuration]
AdminIdentities=unix-group:admin

```

If the latter does not exist, create it. 

See:
[http://hal.freedesktop.org/docs/polkit/](http://hal.freedesktop.org/docs/polkit/)
and
```
man polkit
```

EDIT: oh, and make sure that the *Authentication mode* in:
```
gksu-properties
```
is set to *sudo*.

### Post by gmargo on 2010-10-25
> **jesushero said:**
> ... change this behavior so it can ask for the sudo password instead?

I used **gconf-editor** to change /apps/gksu/sudo-mode to checked, which is the same thing that **gksu-properties** will do.  That's it; no policy-kit changes.
