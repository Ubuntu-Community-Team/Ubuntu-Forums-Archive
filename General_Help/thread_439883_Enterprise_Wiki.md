### Post by az on 2007-05-22
I have used Drupal for enterprise sites.  If you like the idea of a wiki, but hate wiki markup, Drupal can give you the benefit of both worlds.

In drupal, you create a content type and then set the access to it to everybody (or whatever groups you want to have access to it).  You tell it to create a revision for each edit by default.

Voila!  You have a website which has user-contributed content in the exact same way as a wiki, but is a lot easier to use.  Especially if you install a WYSIWYG html editor module.
