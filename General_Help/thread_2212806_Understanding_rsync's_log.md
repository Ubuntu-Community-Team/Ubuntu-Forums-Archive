### Post by Dennis N on 2014-03-23
> Can someone please explain ".d..t......", ">f.st......", etc? I looked at man rsync and man rsyncd.conf but with no joy.

A line starting with **[FONT=Courier New]>f+++++++++[/FONT]** indicates a new file was created on the destination, followed by the path.
A line starting with **[FONT=Courier New]cd+++++++++[/FONT]** indicates a new directory (folder) was created on the destination, followed by the path.
A line starting with **[FONT=Courier New]>f.st......[/FONT]** indicates a file was updated because of size and time stamp difference, followed by the path.
A line starting with **[FONT=Courier New].d..t......[/FONT]** indicates a directory (folder) was updated because of time stamp difference, followed by the path.
**[FONT=Courier New]*deleting   [/FONT]** indicates the file (or folder) was deleted from the destination, followed by the path.

Note: You can get these messages from rsync in the real-time terminal output if you include the **i** option with your command.
