### Post by Leoazul on 2011-03-26
Hi, Sammiev,

I still don't know what we did wrong, but I had to fix my boot too, like you, and had the same problem. I've found the solution [here]("http://http://ubuntuforums.org/showthread.php?t=1662142"). 

Follow this:

Open Synaptic (Menu System > Administration > Synaptic Package Manager). Go to the packages *grub-pc* and *grub-common* and mark the squares choosing the option *mark for complete removal*. This will remove grub from our system, including configuration files. We had a problem with them (the configuration files), but I don't know exactly where. Click on Apply.

After uninstalling, remark them for installation and click Apply again. Follow the instructions choosing the partition where you would like to install the boot. I didn't put anything in "Command line...". You will have a brand new grub installed and the default configuration files. No more messages.

It worked very well to me. Tell me your result.

Thanks.

Leo
