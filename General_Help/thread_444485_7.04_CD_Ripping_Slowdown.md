### Post by MoLE on 2007-05-23
This may be a result of losing DMA access under the new libata drivers in the Feisty kernel.  There have been reports of some chipsets not being able to access IDE devices at full ATA speed.  Unfortunately the current kernel libata is not yet compatible with the hdparm utility used for this.
I'm not sure if downgrading the kernel is an option to solve this - maybe someone else can help with this one.
