### Post by dabl on 2010-06-21
Looks OK -- do you have compositing and GLX?  If not, you might try (a) backup the present xorg.conf file, and (b)Ctrl-Alt-F1 out of the desktop to the tty console, then

```
sudo service gdm stop
```

```
sudo nvidia-xconfig
```

```
sudo service gdm start
```

That will write a new xorg.conf file that should take advantage of most or all of the capabilities of your card, except overclocking.

Here's more:  [http://kubuntuforums.net/forums/index.php?topic=3107406.0](http://kubuntuforums.net/forums/index.php?topic=3107406.0)

### Post by dabl on 2010-06-22
There are reports that not only nouveau needs to be blacklisted, but also nvidiafb for some cards.  I only have the nouveau module blacklisted.

Couple of other things, FYI:

- Nvidia driver ver. 256.35 (Beta) is available (working good on my GTX 260): [http://www.nvnews.net/vbulletin/showthread.php?t=122606](http://www.nvnews.net/vbulletin/showthread.php?t=122606)

- I personally use the downloaded driver installer rather than the jockey thing -- here's the how-to:  [http://kubuntuforums.net/forums/index.php?topic=3107406.0](http://kubuntuforums.net/forums/index.php?topic=3107406.0)

It's written for Kubuntu, but just substitute gdm for kdm, and gedit for kate, and the rest is the same.
