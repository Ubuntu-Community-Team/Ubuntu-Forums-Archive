### Post by harris4got on 2010-12-07
Hello All,

I have an Ubuntu 10.4 installation (dualboot with windows XP) with grub2.

After I resized some of my partitions using a gparted live cd, the system goes directly to grub rescue mode every time I boot.

Then I follow the instructions which are given in grub2 [wiki]("https://help.ubuntu.com/community/Grub2#Rescue%20Mode") site to boot.

The grub boot directory is now located in sda6 (hd0,6) in my system as found out using 'ls' command.

But in the grub rescue mode when I enter the 'set' command (one of the  very few commands available), to display current prefix and root, it  gives "(hd0,7)/boot/grub" as 'prefix' and "hd0,7" as 'root'.

After entering the following commands, I'm able to boot.

```
set prefix=(hd0,6)/boot/grub
set root=(hd0,6)
insmod linux
insmod normal
linux /vmlinuz root=/dev/sda6 ro
initrd /initrd.img 
boot
```Boot is successful and I can log in to my system and GUI and all

Then I update grub using update-grub command. Then I check the grub.cfg file to make sure the entries are right.

And the /boot/grub/grub.cfg seems to be ok and it has (hd0,6) as root.

```
...insmod ext2
set root='(hd0,6)'
search --no-floppy ..
```(hd0,7) doesn't appear anywhere in this file.

Then again at the next boot, there is only a grub rescue prompt :shock: and prefix & root has (hd0,7) values

So I can boot into Ubuntu after entering all these commands and still have not figured a way to get into Windows installation.

There are other users at my home not familiar with ubuntu. windows is  their OS of choice. So I don't want to remove windows installation :cry:.

I'm grateful to anyone who can shed some light on this or may know whats wrong here.

One solution that I can think of is creating a grub rescue CD using grub-mkrescue, then using windows cd to fix mbr (which will overwrite grub? or the pointer to grub?)  and then using the grub rescue cd to boot into Ubuntu but I'm not sure.

### Post by surfer on 2010-12-07
you should not edit /boot/grub/grub.cfg!

edit /etc/default/grub instead and run update-grub.
