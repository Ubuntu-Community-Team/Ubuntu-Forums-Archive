### Post by reckless2k2 on 2007-06-09
hhmmmmm...these are not difficult questions to answer but rather long in some cases:

1 - yes ubuntu includes everything debian includes but are precompiled specific to ubuntu. youcan install debian packages in ubuntu but that's a different issue. in some cases it could cause issues.

2 - ubuntu and debian differences are more in the many variants of debian rather than the stock ubuntu. the refresh cycle of ubuntu is much faster than debian proper which would be the "main" debian. ubuntu has more current stable packages than debian stable/proper. there is debian testing and unstable which get into other areas of confusion. ubuntu is more current with package versions and kernels than debian stable. the main differences come from comparing the release cycles and installed packages of ubuntu and debian stable. 

3 - i'm sure a google search might help you out with that one.

really, depending on your experience with linux, ubuntu is very capable especially if you're young to linux. debian typically requires a little more work to get things going and or moving along butthe similarities are great especially since ubuntu is a debian variant. debian is good to get your feet wet in but you'll probably find yourself coming back to ubuntu for a more fresh product along with the vast support community.

### Post by steveneddy on 2007-06-09
We had this discussion in our local LUG meeting today. 

The consensus is that Debian is the basic model and Ubuntu is a more polished version than Debian proper.

Although Ubuntu is based on Debian, Ubuntu may be a little more stable due to the large community and developer base. Overall, those in my LUG that have installed Ubuntu have been pleasantly impressed with the completeness of the distro, but lacking is some things that you may fine in a 'Professional" distro like Fedora, RedHat and Suse.

Ubuntu is a great tool for your needs, though. Even though there are Linux distros that are more suited to heavy calculations or graphical optimizations, Ubuntu is a very robust software package that could, and is, used in the professional fields and business applications.

Just my two cents.

### Post by reckless2k2 on 2007-06-13
Again it works back to the top-level differences between debian and ubuntu. debian core design is in stability for a server environment. packages and kernels tend to be more dated in debian but the stability of the product is next to none. ubuntu core design is in desktop usability for the masses. this tends to support more current packages and kernels addressing issues related to desktop/laptop usability. ubuntu is based off of debian testing/unstable. debian stable has older more conservative packages. Being a variant means that they take debian "branded" packages and tweak them for their distro and or built on the core base of the debian distro. directory tree, package management system, supported packages, etc. I am trying to express this as simple as I can. Hopefully, you get the idea. 

This site shows you a list of distros based off of debian and explains some or why the variation:
