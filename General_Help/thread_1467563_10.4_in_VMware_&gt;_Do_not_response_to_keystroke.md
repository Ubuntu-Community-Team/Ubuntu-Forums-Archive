I had the exact same problem.  I'm Using vmplayer 3.0.1 though.  This is strange though, because just a few days ago, I installed the rc from the iso without any problems.  After getting this problem, I deleted the rc and re-installed it.  Now, the same problem.  I thought the problem may be with some upgraded packages.  So this is what I did to solve it.  I don't use WMWare 7.1 but I hope it's similar to VMPlayer.

1.  Create a new Virtual Machine
2.  From the VM Wizard select the option to **[COLOR=black]install the OS later[/COLOR]**.
3.  Select **Linux** from the **Guest OS** list and **Ubuntu** from the Version combobox
4.  Name the VM accordindly.
5.  Select the disk size.
6.  On the last page of the wizard select **Customize Hardware**.
7. Disable the **Network Adapter** by selecting it from the device list and unchecking the **Connect at power** on option.
8.  Click on **save** and then on **Finish**.
9.  Click on **Edit Virtual Machine settings **from the Home Menu.
10.  Select the **Cd/DVD** from the device list and click on **Use Iso**.  Select the iso file and click on **save**.
11.  Play the virtual machine and install 10.04.

The password box now responded for me.  I'm in the process of updating the packages hoping that the problem does not re-appear after the updates.

Hope this works for you.
