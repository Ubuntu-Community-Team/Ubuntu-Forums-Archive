### Post by sanderj on 2011-07-25
If you try to view [http://static.die.net/earth/mercator/1600.jpg](http://static.die.net/earth/mercator/1600.jpg) in Firefox/Chrome, you don't get that URL, do you? You are redirected to [http://www.die.net/earth/](http://www.die.net/earth/), containing the picture somewhere in it.

So it's not a wget thing. 

It's my guess that that site doesn't want you to use direct links / deep links to pictures. Probably to avoid that others use links in their own sites directly to pictures on die.net

So nothing wrong with that picture or wget. Just a site protecting it's content.

### Post by Habitual on 2011-07-25
```
wget --quiet -U Mozilla/5.0 -r -O /home/jj/Pictures/daylight.jpg http://static.die.net/earth/mercator/1600.jpg
```

change /home/jj/Pictures/daylight.jpg accordingly.

sanderj is correct, so you must use the -U Mozilla/5.0 option (it is the UserAgent switch for wget) else, you'll only get a white bg and that's it.

HTH.
