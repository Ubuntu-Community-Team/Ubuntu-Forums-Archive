### Post by oldfred on 2017-04-01
Partition numbers should not have changed.
But even if partition numbers change, years ago Ubuntu/grub changed to use UUID which is never supposed to change unless you reformat partition or manually force a change. So system should always boot using UUID.

You show the full chroot.
And the simple just mount / and install grub to MBR. If separate /boot must mount that also.

But if separate /boot (or with UEFI, the ESP) you have to mount those partitions for chroot. And the few users that have other system folders in separate partitions may have to mount those.

Most now find Boot-Repair works easier. It can do the simple restore grub and the full chroot. But it only knows/asks about /boot not other system folder as partitions.

 How to restore the Ubuntu/XP/Vista/7/8/10 BIOS bootloader 
[https://help.ubuntu.com/community/RestoreUbuntu/XP/Vista/7Bootloader](https://help.ubuntu.com/community/RestoreUbuntu/XP/Vista/7Bootloader)
[https://help.ubuntu.com/community/Grub2/Installing#Fixing_a_Broken_System](https://help.ubuntu.com/community/Grub2/Installing#Fixing_a_Broken_System)
