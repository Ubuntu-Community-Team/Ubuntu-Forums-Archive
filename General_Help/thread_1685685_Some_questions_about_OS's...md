### Post by Wtower on 2011-02-11
I will try to answer in short. The following are merely not all about it, but others please excuse my attempt to be simple in my answer.

Ubuntu, Backtrack, Fedora etc are actually different Linux distributions. In short, a Linux distribution (distro) is a collection of several (not necessarily different) software libraries, applications and settings based on the Linux kernel. There are so many of them simply because it is not very difficult for anyone to compile his own collection. So personally for me the choice of a non-specialised distro is a matter of the community that supports it at the first place. Of course there are many-many differences for which we could write for hours. 

Ubuntu is based on the Debian distribution which features the Debian package-based application installation, which for me is to no match for any other distribution, plus there is a huge community behind it. Probably Ubuntu is the most popular choice.

BackTrack is a distro aimed to test the security of a server or network (for good or bad). It is a specialised purpose far from the typical home use.

Fedora is a popular distribution based on the Red Hat distro, with large history behind it. The purpose of it is similar to Ubuntu, and there are red hat packages as well, but I find the lack of a Software Center like the Ubuntu one disturbing.

The FreeBSD project is based on BSD Unix and not Linux and I would refrain from using it for anything apart a server installation or specialised business uses as tejendra suggests.

Bottom line and personal recommendation: if you have the time to experiment, by all means do so and try everything, the experience is quite educating. Otherwise stick with Ubuntu desktop.

### Post by MiKOTRON on 2011-02-12
I think that once you learn to use the basic/advanced commands that you type into a terminal you'll find that any Linux flavor is just about the same once Linux is installed and you're using it.  The real differences between the distros is the package management software and the pre-configured config files.  For example Ubuntu usually works right out of the box while other distros like maybe Gentoo, requires you to edit every single config file to make it work.  The advantage to editing and writing your own config files is that there isn't as much bloat in the files.  On older computer that would be a real advantage. Now you could always go over your Ubuntu config files and delete all the REM statements and really slim down your system if you like.  Oh yeah the other thing is Ubuntu right out of the box might install a bunch of software that you don't really need.  With gentoo for example you build your system your self, compiling every bit of code your self. So you only compile the software that you need, making the computer faster, if you set up your configurations properly that is.  So in conclusion I love the debian package manager that Ubuntu uses.  I love the fact that Ubuntu works right out of the box. I love the community behind Ubuntu.  Now I've been using Linux since 1998 and the thing that I don't like about Ubuntu is that the config files are too bloated.  Don't get me wrong the config files are well documented using REM statements, and that's totally awesome for new comers to Linux.  Also I don't like how many unnecessary software packages are installed and running on my computer.  Now I love gentoo because of the reasons that I don't like Ubuntu.  What I don't like about gentoo is the portage package manager.  I've found that Debian's package manager is the best in the Linux world.
