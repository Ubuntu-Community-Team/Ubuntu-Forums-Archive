### Post by empthollow on 2007-12-28
you can make a file executable by right click -> properties and go to the permissions tab and check the allow executing of file box.  as for double clicking to run, some of them you can and some you can't.  it depends on how the script was written.  a .run or .sh file is more like a batch file (.BAT).  it in my opinion sh is much more powerful than windows batch files but that is aside the point.  the point  i would draw from that is that i have never been able to double click on a batch file and i have on ocasion been able to double click on an sh file.  you seem to have run into people who wrote sh files to install programs which some companies do because they don't want to take the time to create binaries for all of the different linux distros.  the doubble click to install in windows i believe you are used to is the .exe file which in ubuntu is equivelant to a .deb file which in ubuntu you can double click on.  the problem comes in where there are hundreds of different distributions of linux that all have different packaging requirements and only about 5 versions of windows which have pretty much the same packaging requirements.  i too have wondered why is linux not more user friendly and the answer i came up with is that some distros like ubuntu are but linux doesn't have the universal commercial support that windows has.  i personally have gotten used to all the programs in the repositories and find it annoying when i have to google to find a program i want to install.  hope this answers your question.
