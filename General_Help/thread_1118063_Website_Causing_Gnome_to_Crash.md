### Post by LoneWolfJack on 2009-04-07
I'm having the same problem, but it's actually not crashing gnome but hiding all gnome panels. panels are restored once the browser window containing the website is closed, very odd, but not necessarily serious in terms of security vulnerabilities.

the problem appears to be a line break within the title tag. very strange. I wouldn't believe it myself if I wasn't able to reproduce the problem by creating a local html file with this html code:


```
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">
<meta http-equiv="Content-Type" content="text/html; charset=UTF-8" />
<title>   Nine Words You Might Think Came from Science but Which Are Really from Science Fiction&#8232;&#8232;  : OUPblog</title>
</head>
<body>
</body>
</html>
```

so it's a firefox rendering issue. probably some problem with gnome api calls by firefox.
