### Post by Mark Phelps on 2011-07-05
First of all, do NOT, repeat NOT, update your grub.cfg file.

When you open that you will see, in plain text, wording telling you NOT to edit it.

While that can work in an emergency, (it's OK to hand-patch a change to see if it works) it's really a BAD long-term solution.  Why? Because the next time you do an update, or do "update-grub", the cfg file gets completely rewritten -- effectively, erasing ALL the changes you made to it.

For persistent changes, you edit the files in /etc/grub.d
