The only real list is Launchpad. That's what Launchpad is for. You can search for bug reports and subscribe to them, or, if you can't find an existing report,  you can use the report bug feature (I often find that the report bug feature's search is a good way to find bugs).

### Post by mssever on 2008-10-19
> **editrix said:**
> Which I've done, but perhaps I've been particularly unlucky in that none of them seem to have been fixed?Not all bugs get fixed, because the developers simply don't have time to address all of them. The more work you can do to make the developers' lives easier, the greater the chances of the bugs getting fixed. Some bugs I've filed have gotten fixed (sometimes a year or two later). Others haven't.

> I guess my general question also includes, What do other people do? Is there a strategy--other than patience? Also, is it safe to assume that, if something has been fixed, it will show up as an update?Well, here's what I do. I file bugs on Launchpad. Sometimes I file bugs upstream, as well, and link them with the Launchpad bug. After being bitten hard by Gutsy's bugs, I've learned to evaluate each release with a live CD before upgrading. I should have skipped Gutsy. I'm now running Hardy and have no major problems.

When bugs are fixed, the fix by default does into the current *development* version. Depending on the severity of the bug, the fix might get pushed to released versions as an update.
