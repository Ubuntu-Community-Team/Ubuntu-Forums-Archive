This issue isn't actually solved, as Juicer still doesn't recognize CD1.  But I'll mark this thread [SOLVED] anyway as it appears to be a piffling problem.  It's also correctly identified Fun Lovin' Criminals' "Loco" album.

There are 3 main CD databases I've run across- musucbrainz, freeDB and CDDB (now gracenote). 

[http://musicbrainz.org/](http://musicbrainz.org/)

[http://www.freedb.org/](http://www.freedb.org/)

[http://www.gracenote.com/](http://www.gracenote.com/)

Personally, I've foudn that freeDB and CDDB are more likely to get the CD/tracks right. I wont touch CDDB since 2008 (sony bought it out) and prefered freeDB for a long time before that (licensing crap). 

As for how it got CD1 wrong but CD2 right, well, it happens. Acoustic fingerprinting, the way that musicbrainz IDs tracks, isnt perfect. 

Not that it was a major issue, but you could always install a ripper that uses a different database if you get errors. Rubyripper (freeDB) would be my choice ;)
