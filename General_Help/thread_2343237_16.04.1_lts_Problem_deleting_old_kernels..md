### Post by Impavidus on 2016-11-15
You don't have the packages linux-image-4.4.0-*-generic (probably uninstalled), but you do have the packages linux-image-extra-4.4.0-*-generic, which depend on the former. That's why you have broken dependencies. Your fancy command```
dpkg -l | tail -n +6 | grep -E 'linux-image-[0-9]+' | grep -Fv $(uname -r)
```lists the old linux-image-4.4.0-*-generic packages, but not the linux-image-extra-4.4.0-*-generic packages. That's what's going wrong.

Furthermore,> **usernameheru said:**
> ```
ii  linux-image-4.4.0-21-generic       4.4.0-21.37                       amd64        Li
```

I know ii means its eligible for removalii means the target state of the package is to be installed (the first i), the package is currently installed (the second i) and the package is in good condition (no third letter).

To get an overview of the current status of your packages, run these two commands and post the output:```
uname -r

dpkg --list | egrep linux-image\|linux-headers
```
