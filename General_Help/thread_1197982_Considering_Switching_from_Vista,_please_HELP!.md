### Post by t0p on 2009-06-27
> **philcamlin said:**
> ubuntu is for newer pc's 

for your vista pc use ubuntu because its decent

kubuntu is for P3's and people who dont quite like the gnome desktop and want themes 


What the heck!?  OP, don't take any notice of the above!  Ubuntu and Kubuntu work on the same hardware equally well.  One is not objectively "better" than the other.  The difference between them is they use different desktop interfaces/window managers.  Ubuntu uses Gnome, Kubuntu uses KDE.

The way to decide which is better *for you* is to try both.  If you [download the ubuntu desktop cd]("http://www.ubuntu.com/getubuntu/download"), it will have the Gnome desktop environment. Then, when you want to try KDE, you can install a software package called "kubuntu-desktop".

> **MRife said:**
>  This may be a stupid question, but will I be able run the software that came with my camera for photos?  I really like it. [QUOTE]

The Windows software *might* work, through a pseudo-emulator called "Wine".  But don't count on it.

[QUOTE=MRife;7523745]
Will I have to reload all of my photos on my hard drive or will they still be there if I add the new operating system (Ubuntu)?


If you are going to dual-boot (install Ubuntu next to Vista on the hard drive) or use wubi ("install" Ubuntu inside Vista like a regular Windows program), your photos might be safe.  But back-up anyway.  Things can go wrong.

If you are dumping Vista, and plan to install Ubuntu over it, your photos will be4 deleted in the process.  So you'll need to back them up to a usb stick or something before you start the installation.

Either way, you should back them up before the installation.  But you've already got them backed-up.  Haven't you?

### Post by Gizenshya on 2009-06-27
I strongly agree with those who mentioned trying it as a LiveCD first.  That way you don't change anything, and you'll know fairly well if it will work.

One thing to keep in mind is to shut windows down properly!  it sets a "$logfile" in drives it accesses, and unless it changes that $logfile on shutdown, Ubuntu will not boot from that drive, or mount it (without using the force option).  It took me a while to figure this out when I first started, and it is incredibly annoying.

I only use Windows for games now.  Every program I liked in Windows that was keeping me there a while back has a better substitute in Ubuntu.

If you are proficient with Google, and have the ability to follow instructions *carefully*, without making assumptions (or at least, willing to learn those things), you'll do just fine.  That is pretty much where I stand now, and I have not yet found aproblem I cannot address.  I went in blind with no knowledge whatsoever with command prompts and the like ("no, I'm not in the mood for music, I just want to move to another folder!").  Whereas I've filed with other linux distros (Redhat, Fedora Core, and a few others), I'm using Ubuntu just fine now and I love it!

Another thing is to always be on the lookout for keywords.  There are lots of them, and the more you know, the faster you can find solutions (in search engines), or ask for help with a given issue.

Being familiar with (or at least aware of) words/terms like "mount," "sudo," "terminal," "man," "xorg.conf," "swap," "synaptic," "gnome," "grub," "root," "fstab," and many others has helped me greatly.
