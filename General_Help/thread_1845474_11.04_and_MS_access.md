As far as i know as long as the MS Database supports ODBC which by default all access DB should so then you can use Openoffice or Libreoffice Base application.

So it used to be you could use Openoffice base or Libreoffice to connect to the ODBC data source but i havent done this in a while.

Openoffice and Libreoffice are great packages but dont deal perfectly well with MS Access but then nothing does apart from MS Access.

So your choices are probably use a Virtual Machine to carry on running native MS Access or set up ODBC.
