### Post by Sonsum on 2012-06-21
Whenever I log into Ubuntu (either Unity or Gnome-Shell) my resolution changes from 1366 x 768 (the correct resolution) to 1024 x 768. Changing the resolution back does not stay across persistent reboots.

I believe the problem lies with the script "/etc/gdm/xsession", for running it while logged in will change the resolution to the improper 1024 x 768. However, Google searches involving fixing this problem with this script have turned up nothing. 

Does anybody know how I could go about fixing this?

### Post by bobppower on 2013-01-16
I don't know if you still have the problem, but, as I had and didn't find an easy solution, I'm gonna post it.

And, actually, I use gdm only on Fedora, so, I've tested there only. Therefore, if it doesn't work in Ubuntu, tell me and I'll install it here.

First, type this command in a terminal:

```
xrandr -q
```The output will be something like this:

```

Screen 0: minimum 320 x 200, current 1280 x 720, maximum 4096 x 4096
VGA1 connected 1280x720+0+0 (normal left inverted right x axis y axis) 340mm x 270mm
   1280x720       59.9* 
   1024x576       60.0  
   832x624        74.6  
   800x600        72.2     75.0     60.3     56.2  
   848x480        60.0  
   640x480        72.8     75.0     66.7     60.0  
   720x400        70.1  
```By this output you can see that, in my case, the screen identifier is **VGA1**. Use the screen identifier that appears in that place in your output.

Then, open```
/etc/gdm/Init/Default
``` in a text editor (nano, gedit) as root an type the following line **right before **```
exit 0
``` (remember to change the VGA1 with your screen identifier - and your screen resolution too :) - from the last output):

```
xrandr --output VGA1 --mode 1280x720
```Save the file and restart. Hope it works!
