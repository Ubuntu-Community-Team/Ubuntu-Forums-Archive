### Post by Per Olav on 2011-10-20
The same happened to me. I am running the 2.6.32-34-pae kernel. 
I didn't have the time to look more into it, so I just rolled back to the previous kernel, 2.6.32-33-pae, which works with the Nvidia driver. 

Quick howto for this _workaround_:

1. Look into /boot/grub/grub.cfg
2. Find which menu entry that corresponds to the kernel you want to use.
3. Set this as default in grub config file, /etc/default/grub. (Probably "GRUB DEFAULT=4")
4. Run update-grub
5. Reboot
