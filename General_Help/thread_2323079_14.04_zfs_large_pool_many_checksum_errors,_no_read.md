### Post by lukeiamyourfather on 2016-05-11
The read and write errors literally mean it was not able to read or write data. The checksum error is for when a block of data is returned but doesn't match the checksum for the block which is stored away from the block. I would check the system memory for errors after it's been up for a long time (edac-util) or run a memory checking utility, check all physical controller connections, check for controller firmware updates, and check the system for bus errors (like "Bad DLLP" which indicates corrupt PCI Express packets). It's unlikely that the send and receive introduced the errors.

### Post by eric122 on 2016-06-19
I figured out my issue, it had to do with mismatched mpt2sas firmware on my controller/kernel.  When I set up the system, I flashed my card to the latest mpt2sas available (ver 20) while the version in my kernel was 16.  I flashed my card to ver 16 to match the kernel and haven't had any problems since.
