### Post by L a r r y on 2011-07-30
[SIZE="3"][COLOR="Red"]**I**[/COLOR][/SIZE] am in an application with a very small edit window relative to the length of the document I want to edit.  I want to highlight a larger chunk of text than will fit within the scrollable edit box or edit screen, and I need to use the mouse to drag from the highlight start point to the finish point.

**To cite an example,** try writing a big list of your favorite books, music, about yourself, etc in a YouTube user's channel in Ubuntu Firefox, in a form that is 3 lines high and about half the width of the tags field under the message edit box on this forum.  The edit box does not permit use of the arrow keys in Ubuntu, but it does permit normal use of these keys in MS Windows.  (Firefox 5 both places)  

Then try to edit a good-sized portion of that information using click-and-drag to do the highlighting.

What happens is the scroll mechanism takes off at high speed and there is 3 or 4 times too much text higlighted.  Back it up and try to stop it at the end of the desired text to highlight, and you overshoot, and have a bunch of text highlighted, right up to the start of what was supposed to be highlighted. 

[INDENT]*A work-around here is to open the edit box on YouTube, select-all and copy.  Open the Text Editor and paste.  Edit it then select-all and copy, and then paste into the still-highlighted YouTube edit box, replacing the existing content.*
[/INDENT]

A note here:  I just tried highlighting text beyond the boundaries of this message's edit box, but because this box is 3x as wide as the one offered in editing a user's YouTube channel, with 5x as many lines of visible text, its scroll speed is easy to manage.

Another note: When posting comments under a video, there is more room to edit your message than is provided on the YouTube user's  Channel, so that also is easier to manage.



**To cite another example using OpenOffice,** I am in OpenOffice Writer, and I want to highlight text that crosses a page boundary.  I was trying to highlight about 1-1/2 pages of text, and once it took off, it was very hard to stop where I wanted it


[COLOR="DarkGreen"]**There must be a setting somewhere that I can adjust to put the brakes on the scrolling during the highlighting of text?**[/COLOR]

My past experience in Windows would start out slowly, and if I position the mouse further beyond the edge of the scrolling window, the faster it would scroll, to cover large ground, but by moving the mouse nearer to the scrolling window it would slow down to a crawl.
