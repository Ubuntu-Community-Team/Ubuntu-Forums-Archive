### Post by ryan858 on 2010-05-13
Open a terminal and type this command:

**$ sudo gedit /boot/grub/grub.cfg**

Then select all, copy, and paste it in a post here.

I'll check it out and post the revised version if I make any changes... It would be better that you do it yourself for the experience... but it's actually very complicated... it's basically just a shell script, not like the old Grub Legacy grub.conf file. And it's really the only method to edit Grub2 until they improve it a lot more in later releases, or someone makes a decent Grub2 editor.

You can still compare the two versions to see what I did though... you might be able to figure a few things out.

 There are other methods that are supposedly "easier" like the  /etc/grub.d/10_linux file or 40_custom or /etc/default/grub, where you  edit those files and run **grub-update** which parses those files and  compiles them into the /boot/grub/grub.cfg file... but it's pretty  buggy still. it's (imo) easier to just edit grub.cfg, which the  developers highly recommend you don't do, cause you can mess it up  easily without grub-update to check the contents for errors. Problem is  grub-update itself produces errors when writing grub.cfg, especially  with the os-prober. So until all the bugs are worked out, I think you  might as well just learn to edit grub.cfg.

Yeah, he's right, you just have different kernel versions... I thought it you meant it had duplicates. It's hard to see the differences in the versions, the 1 instead of 2 in 2.6.32-21 or 2.6.32-22.

Well, if you are sure you won't have to ever use the old kernel ever again, then you can comment out the menu entries for the old kernel, and memtest too if you want.

This would be an unsafe thing to do though, because of what plucky mentioned. If it does break, then you have no other kernel and no way to get it back, unless you use a Live CD to edit grub.cfg and uncomment the old kernel.

I personally have a Live CD and can restore the old kernel if something bad happens, so I removed the extra menu entries (old kernels and memtest)... but it's up to you what to do with your computer.

Below is the grub.cfg with the entries commented out. If you want to use it, then backup your old grub.cfg, just make a copy and name it grub.cfg.bak or something, in terminal you'd type:
[B]$ cd /boot/grub
$ cp grub.cfg grub.cfg.bak[/B]
 Then you need to give it write permission, open it in gedit and paste the new stuff and save, then remove write permission:
**$ chmod +w /boot/grub/grub.cfg && gedit /boot/grub/grub.cfg && chmod -w /boot/grub/grub.cfg**

### Post by scouser73 on 2010-05-16
Rather than tell you to paste command to remove the previous linux kernels, go to [[COLOR="Red"]**Ubuntu Tweak**[/COLOR]]("http://ubuntu-tweak.com/") and install the software (if you haven't already got it).

Once installed, you will find Ubuntu Tweak in System Tools, click on it and the program will start, next click on **Package Cleaner** and click on **Unlock**.  Enter your password, and click **Authenticate**, now click on **Clean Kernels** (which will remove only the previous versions) then tick **Select All** & finally click on **Cleanup**.

The next time you start the pc, you will only see the current linux kernel.

"linux-image-generic" doesn't contain a kernel. The other two packages do. You can verify this by looking at the "installed-size" in Synaptic.

Linux-image-generic actually has the latest kernel as a dependency. It's just the way for the system to be able to install a new kernel without it replacing the old kernel.
