### Post by oldfred on 2010-10-22
I think you would do better to boot with Ubuntu as its version of grub2 is a lot newer. Grub2 counts partitions different from grub1 and there is no hd0,0 it has to be hd0,1 and you do not need the hd0 ,0  entries on each line


Copy this into 40_custom in your backtrack install:
gksudo gedit /etc/grub.d/40_custom
sudo update-grub

menuentry "Ubuntu" {
    set root=(hd0,1)
    linux /boot/vmlinuz-2.6.32-21-generic root=UUID=b2d3d32b-6ea3-4e0d-9508-452877c7dd8e ro
    initrd /boot/initrd.img-2.6.32-21-generic
}
