### Post by johngreth on 2011-06-02
The workaround is to use xscreensaver instead of the default.
I have always used xscreensaver because it has more screen savers to choose from. I figure if xscreensaver works for you then there is no real reason to push the issue.

However, I did notice this problem right after I upgraded as well. I think that you can only have the power manager or the screen saver (not both) decide how long to wait before turning off the screen. Otherwise they get in each others way. I didn't do any testing to verify this because I was going to switch to xscreensaver anyway...
