### Post by ankspo71 on 2010-10-01
Hi,
Ubuntu is already like this by default, except the 'full administrative' (Root) user login is inaccessible for security reasons (and is not even necessary IMOP). It only appears that you have one user account when you install Ubuntu, but technically you already have 2 (Root + jodlajodla). Some distributions aren't as secure though, and have the root user login accessible at login, or worse, use root user as the primary user account... and this is where it would be crucial to create another user account.

Unless they were referring to letting friends and family use your computer, then I would agree. Creating additional user accounts (or guest accounts) for  friends and family to use  would be safer for the system and your data. 

If you would like to keep using multiple users as a single user computer, then I think Ubuntu has a logout option called "Switch User" too. In Kubuntu, "Switch User's" description says "Start a parallel session as a different user". I never needed to use that though as I am the only user.

You can also modify your user(s) permissions like the above poster mentioned too.

### Post by Iowan on 2010-10-01
The account you set up when installing Ubuntu has "pseudo" administrative privileges. From the command line, instructions that require **root** privileges use **sudo**. [This]("https://help.ubuntu.com/community/RootSudo") help page might explain it better.
