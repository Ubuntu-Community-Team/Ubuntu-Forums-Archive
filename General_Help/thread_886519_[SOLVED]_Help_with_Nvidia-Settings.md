The default NVIDIA driver (NVIDIA-new) that loads in Synaptic, apt-get, or via the Hardware Driver under Systems Admin has never worked for me.  I always got the same message you do.

Uninstall that driver via hardware drivers in System > Admin.  Go to the NVIDIA site and download their latest driver according to your specifications.  Load the driver from a terminal as "root" using sh NV*.  Follow the steps as they come up.  You take defaults on all questions except the first one (Accept the license agreement) and the last one (which asks if you want to write a new xorg.conf file.

Note - you will need libc6-dev installed.  Use apt-get or Synaptic.

You also need to do this via a terminal with the X-server off.  You can turn off the X-server via System >Administration>Services.  Unlock Services and deselect the GUI checkbox. Remember to save you file to X11 in the settings manager.  If it doesn't let you save its because you don't have writer permissions to X11.

### Post by jwferk on 2008-08-12
So was I for quite some time; I'm getting more familiar with things.  This explanation looks long but really isn't; just tried to be complete. I suggest reading the entire thing first if you want to try it. 

I'M ASSUMING YOU ARE USING GNOME. 

1 - go to the NVIDIA site and download their most recent driver for your system.  The easiest place to save it is your Desktop.

2 - Go to System > Administration > Hardware Drivers (I'm at lunch at work on a Microsoft PC; can't guarantee that name is exact but it is close enough).

3 - deselect the NVIDIA driver if this is checked.  This is the default driver that Ubuntu has.  I have never gotten it to work for NVIDIA dual monitors.  We want to remove it.  If things absolutely go awry, just follow the process above in reverse to re-install it.

Select OK.  The driver will uninstall automatically.  You may get some warning telling you what you will lose if you continue, e.g. 3D graphics etc.  Don't worry about them.

YOU WILL NOW TURN OFF YOUR GRAPHICAL USER INTERFACE.  NVIDIA WILL NOT INSTALL IF IT IS ON.

4 - Go to Applications > Accessories > Terminal

5 - type (without the quotes) "sudo /etc/init.d/gdm stop"

6 - type in your password at the prompt, hit ENTER.


YOU ARE GOING TO SEE YOUR USER INTERFACE DISAPPEAR AND YOU WILL BE LEFT IN A TERMINAL SESSION.  THAT IS WHERE YOU WANT TO BE.

(IF YOU WANT TO GO BACK TO THE GRAPHICAL INTERFACE AT ANY TIME USE THE REVERSE COMMAND:  sudo /etc/init.d/gdm start  

7 - Change to the directory where your NVIDIA driver file was downloaded too.  If it was your Desktop the command would be

"cd /home/your username/Desktop"

If it is somewhere else, replace the directory names as needed.

8 - install the driver.  You need to be root to do so.  At the prompt type

sudo sh NV*

THE "*' IS JUST A WILDCARD SO THAT YOU DON'T HAVE TO TYPE IN THE ENTIRE NAME OF THE DRIVER WHICH IS A BIT LONG.

9 - enter your password at the prompt and hit ENTER

NVIDIA SHOULD NOW START ITS INSTALL SEQUENCE.  TAKE THE DEFAULT ANSWER ON EVERYTHING BUT THE FIRST AND LAST QUESTIONS (ACCEPT THE LICENSE AGREEMENT & WRITE TO XORG.CONF).

10 - If everything went well the driver is now installed and you are back at the terminal prompt.

IF YOU GET AN ERROR MESSAGE IT IS USUALLY BECAUSE YOU DO NOT HAVE THE FILE libc6-dev INSTALLED ON YOU MACHINE.  THE EASIEST WAY TO DO THIS IS TO USE THE SYNAPTIC PACKAGE MANAGER.  USE THE  /etc/init.d/gdm start COMMAND TO RESTART YOUR GRAPHIC INTERFACE.  GO TO SYSTEM > ADMINISTRATION > SYNAPTIC.  WHEN PROMPTED ENTER YOUR PASSWORD TO START SYNAPTIC.  SELECT "SEARCH" FROM THE MIDDLE OF THE TOOLBAR.  TYPE IN "libc6" (without the quotes)WHEN PROMPTED.  HIT ENTER.  

SYNAPTIC WILL PRESENT A LIST OF PROGRAMS THAT ARE INSTALLED (BOXES ARE MARKED) OR ARE AVAILABLE FOR INSTALLATION.  "libc6" is probably already installed by default.  YOU NEED "libc6-dev".  CHECK THE BOX AND HIT THE "APPLY" BUTTON (BIG CHECKMARK ICON AT THE TOP OF SYNAPTIC).  NOW YOU HAVE libc6-dev installed.  

Go back to Step 4 and install the driver again.  If the driver installed the first time around, proceed to step 11.

11 - type (without the quotes) "sudo /etc/init.d/gdm start"

12 - enter your password if prompted to do so and hit ENTER.

YOUR GRAPHICAL INTERFACE SHOULD NOW START.  THE NVIDIA LOGO SHOULD APPEAR DURING THIS PROCESS. YOU WILL STILL HAVE ONLY ONE SCREEN ACTIVE.

13 - Log in to GNOME as you normally would.

14 - Go to Applications > System (if I recall correctly) and locate the NVIDIA utility which is now a new program in your Applications menu.

15 - make the appropriate settings on NVIDIA Manager.  Basically, you need to activate the second monitor and use configure button to select "Separate X Screen" or "Twinview" mode.  I suggest you start with "Twinview" as "Separate X Screen" requires a system re-start to take effect.  It occasionally glitches.  Get familiar with Twinview first.

16 - Once you have selected the items you want, select "Apply".  Your second monitor should now light up in "Twinview" mode.

17 - Select "Save File". (if you don't and you shutdown or reboot, you will need to reconfigure the NVIDIA Manager again!!!!)

YOU MAY GET A MESSAGE THAT NVIDIA CAN'T WRITE THE "xorg.conf" FILE TO X11.  THIS MEANS THAT YOU DO NOT HAVE PERMISSION TO WRITE TO THAT DIRECTORY.  YOU WILL HAVE TO CHANGE THAT USING A TERMINAL.

18 - open the terminal again (Applications > Accessories > Terminal.

We can accomplish the change in a single string.  Without the quotes type

"sudo cd /etc; chown yourusername.root X11; chmod u+w X11; cd X11; chown yourusername.root xorg.conf; chmod xorg.conf u+w"

Enter your password at the prompt and hit ENTER.

[What's happening - the file "xorg.conf" is the file that NVIDIA writes to to control the monitors.  xorg.conf is located in the directory "X11" which is a sub-directory of "etc".  These are initially "owned" by "root" and you cannot modify them.

The first command (cd /etc) moves you into the directory "etc". "chown" means "change owner" "chown yourusername.root X11" now gives you partial ownership of the directory X11 (previously it would have been listed as "root.root" "chmod" is "change mode." "u+w" means "user+write".  "chmod u+w X11" is simply "change the mode of X11 to allow the user to write to this directory."  Finally, you move into the directory X11 (cd X11), change the ownership of the file xorg.conf (chown yourusername.root xorg.conf) and give you permission to re-write the file xorg.conf (u+w xorg.conf).

NOW YOU CAN SAVE THE NEW xorg.conf FILE WRITTEN BY NVIDIA TO THE X11 DIRECTORY USING THE NVIDIA MANAGER. NVIDIA WILL AUTOMATICALLY MAKE A BACKUP FILE OF THE ORIGINAL xorg.conf FILE AND THIS WILL BE STORED IN THE X11 DIRECTORY UNDER THE NAME xorg.conf.backup

19 - Close out of the NVIDIA Manager.  You should be set to go.

### Post by jwferk on 2008-08-12
I forgot to add something else.  Sometimes I have had the NVIDIA driver crash on me for no apparent reason.  The only thing I have been able to trace it to is that a file named

xorg.conf.failsafe.bk

is written at boot to the X11 directory.  X11 then contains three files

xorg.conf
xorg.conf.backup
and 
xorg.conf.failsafe.bk

When that happens the system appears to boot into Failsafe mode and a low resolution single monitor graphical interface.

Unfortunately, I have had to re-install Ubuntu using my system backup files, or have needed to do a clean install when this happens as I have no figured out a work around.  Removing the "xorg.conf.failsafe.bk" file does not help as it is re-written at each boot.  

I have a post on forums here about the problem but no one has been able to come up with suggestions as of today.  Hope you don't encounter the issue.
