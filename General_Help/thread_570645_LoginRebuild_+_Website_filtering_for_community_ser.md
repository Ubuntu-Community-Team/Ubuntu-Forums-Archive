### Post by jhmaddox on 2007-10-08
Greetings, 

I'm on a team working on a community service project to install solar powered computers with satellite internet to rural schools in Mexico. We are looking to install our first system in January. We have several sponsors including VIA, who is donating low power consumption components. It is really turning out great so far, but I was hoping I could get some advice.

We will be using edubuntu. Right now, we are planning on doing remote administration via ssh.

We have been brainstorming ways to handle logins. The absolute ideal situation would be some kind of advanced login manager. There won't be any onsite computer administrators so it needs to be as autonomous as possible. Ideally, a user could sign in once with their username/pw and it would create an account for them with 100 megabytes of space in their home directory. Users would have to be automatically deleted after a certain interval of inactivity, or perhaps if the machine is running out of memory.

Somehow I doubt such a system exists. A less ideal situation would be just "student" "password" with automatic system rebuilding/deepfreeze. I've seen some stuff on how to do this, but like I said, it isn't the most ideal situation.

Many of these people have never used computers before, so I don't know if we can rely on them to logout or reset the machine. 

Another issue is that most people using the machines will be students. The culture is also very conservative, so we think there has to be some kind of internet filtering. Has anyone ever had any success in this area? I could really use some advice. 

Any ideas or recommendations would be greatly appreciated. Thanks!
