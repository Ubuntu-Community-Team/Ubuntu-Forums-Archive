> **iissmart said:**

Also, shouldn't it *not* ask for the root password when using "sudo su"? For my server, "sudo su" only prompts for the normal users password to get root access.

It helps to remember what the term "su" really means.  The term "su" means _switch user_. When you switch to a different user the confimation that you are allowed to do that is using that users password.

When you are using your desktop do you have a root password?  If you don't then you can't confirm the switch to root.

The term "sudo" means *run the following command as root*.  As you are a member of the sudoers it asks for YOUR password.

***passwd -l username*** adds a '!' at the beginning of the password
```

username1:**!**:14383::::::
username2:**!**$1$DAkzHTp9$NLULWUToLeHEHExtpaS$OgME1:14383::::::
```

In the above example both username1 and username2 are locked accounts. 

As a side note, you can manage the account settings with the **usermod** and **passwd** commands. For editing the shadow file manually, in a safe manner, use the **vipw** command.


After rethinking the problem with a fresh brain, I think that su is not configured in the way you want it.

Take a look at the /etc/pam.d/su file. Here is mine for reference (the relevant part is **bold**):

```

# The PAM configuration file for the Shadow `su' service
#

[B]# This allows root to su without passwords (normal operation)
auth       sufficient pam_rootok.so
[/B]
# Uncomment this to force users to be a member of group root
# before they can use `su'. You can also add "group=foo"
# to the end of this line if you want to use a group other
# than the default "root" (but this may have side effect of
# denying "root" user, unless she's a member of "foo" or explicitly
# permitted earlier by e.g. "sufficient pam_rootok.so").
# (Replaces the `SU_WHEEL_ONLY' option from login.defs)
# auth       required   pam_wheel.so

# Uncomment this if you want wheel members to be able to
# su without a password.
# auth       sufficient pam_wheel.so trust

# Uncomment this if you want members of a specific group to not
# be allowed to use su at all.
# auth       required   pam_wheel.so deny group=nosu

# Uncomment and edit /etc/security/time.conf if you need to set

# (Replaces the `PORTTIME_CHECKS_ENAB' option from login.defs
```
