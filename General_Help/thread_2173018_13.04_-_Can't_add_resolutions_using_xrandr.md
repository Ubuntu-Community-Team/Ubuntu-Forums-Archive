### Post by carlwsnyder on 2013-09-08
In many of your posted listings, you show an inconsistent naming of the resolution you are attempting to use.  
[list]
[*] First, you don't need to use the '_60' at the end of the name unless you are using multiple vertical scan frequencies and you need to keep them separate. 
[*] Second, You must use the exact same name from the --addmode name of the resolution.  You can call the name Custom, 1280x800, lappy, or whatever, but it must be consistent. 
[*] Third, if you include anything except numbers and an 'x' in the name of your resolution, you must use quote marks around the name in the --addmode line, at least, and if you are using something besides 1280x800, I would recommend using quote marks surrounding the name in the --newmode and --mode lines for xrandr as well. 
[/list]
Perhaps it is just in your posts that you have been careless, but xrandr **does not like inconsistent names**.  xrandr does not read the numbers, it simply uses the label you have given the resolution.

### Post by carlwsnyder on 2013-09-08
I think you are misunderstanding the use of /etc/X11/xorg.conf and xrandr.  You either 
[list]
[*] add the resolution in /etc/X11/xorg.conf and use arandr (the GUI for xrandr) to select the resolution (menu >> Settings >> Display in Xfce), 
[*] use the nVidia software to set the resolution added in /etc/X11/xorg.conf for the display, or 
[*] add and set your resolution in xrandr. 
[/list]
  That is one reason you keep getting errors.  Don't mix methods.

I'm not sure which method will work, or if any will work, but using both methods of adding 'new' resolutions will definitely give you errors when you use the second method.
