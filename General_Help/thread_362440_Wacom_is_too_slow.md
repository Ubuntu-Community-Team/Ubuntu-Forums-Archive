### Post by graigsmith on 2007-02-16
the howto you went through, was totally unnecessary.  Wacom drivers come with ubuntu.

basically all you have to do is set up your xorg.conf file. and install 'wacom-tools'

follow this howto.  some of the steps are unnecessary.  Figuring out which event your tablet is on. is not unnecessary if you install the wacom-tools package.  then instead of device/input/event0  or something like that, you'd use device/input/wacom

[http://ubuntuforums.org/showthread.php?t=25151&highlight=wacom](http://ubuntuforums.org/showthread.php?t=25151&highlight=wacom)

If you get stuck, remember the commandline command nano.  and this command should allow you to edit it from the command line.
```
sudo nano /etc/X11/xorg.conf
```

in your mouse config you have to change the mouse driver to something like mouse0.  The howto details how you can figure which mouse your mouse is on.  most likely it's on 0.

make sure the input devices are listed under the server layout like mine.

### Post by Curtisc on 2007-02-20
I've had the same problems.  Prior to Dapper, I was able to change the speed of the wacom mouse by adding to xorg.conf under the cursor section:
```
Option        "Speed"        "2"
```
In Dapper + however, this just makes the mouse uncontrollably fast, no matter what the number is (1.0000000001 is infinitely faster than 1.0).

As to the xy mapping, the easiest hack is to create a file called .xmodmaprc containing:
```
xsetpointer "cursor"
xmodmap -e "pointer = 1 2 3 5 4"
```
If you put this in your home directory and make it executible, it should be loaded automatically when you log in.
