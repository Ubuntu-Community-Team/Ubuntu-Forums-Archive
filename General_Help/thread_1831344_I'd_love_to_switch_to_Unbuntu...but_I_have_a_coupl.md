### Post by Idefix82 on 2011-08-23
For transferring files, I suggest defragmenting your hard drive a couple of times, then while in Windows, resizing the partition on which Windows is installed, creating a separate partition just for data and moving the files there, and leaving enough unallocated space (20 GB should be plenty) for a Ubuntu install. Remember that you can only have four primary partitions on a drive, and I would guess that Windows has already created two. So just make the rest logical partitions.

I second realzippy's suggestion to install Ubuntu on a partition of its own, instead of inside Windows. Officially, they should be the same. But if for example your Windows becomes unbootable one day, you could then still boot into Ubuntu and rescue everything that needs rescuing, or wipe Windows off and either replace of comfortably continue working with Ubuntu. If Ubuntu is installed inside Windows and Windows dies, then it takes Ubuntu with it.

### Post by grahammechanical on 2011-08-23
The Wubi install option has the purpose of giving Windows users a taste of Ubuntu without making changes to the partitions of the hard disk. It is not designed to be a permanent way of using Ubuntu. If you do not want to continue using Ubuntu you can easy remove it. If you want to use Ubuntu long term then you should dual boot. 

Please read about it before you choose the Wubi option.

[https://wiki.ubuntu.com/WubiGuide]("https://wiki.ubuntu.com/WubiGuide")

Some install Ubuntu using the entire disk. They choose to remove Windows and then change their minds and ask on this forum for help in putting Windows back on their machine. They are asking the impossible.

So, do you have Windows install disks? Do you understand the difference between a Windows install disk and a Windows recovery disk and what you can and cannot do with a Windows recovery disk?
