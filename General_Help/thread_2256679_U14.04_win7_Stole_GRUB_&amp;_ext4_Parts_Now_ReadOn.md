### Post by oldfred on 2014-12-18
You can only have one extended partition that is the container for all logical partitions. And then all logical partitions must be next to each other.
Windows only boots from primary partitions.
Partitions numbered 1 thru 4 are always primary and 5 and up are always logical.
So your sda1 was the Windows boot partition. It must be primary and have boot flag.
Your next NTFS was sda2, so it is also primary.
You sda3 is the primary that is the extended partition and acts as a container for all the logical partitions. It is not shown in testdisk, but will  be the container for Linux & swap, and maybe the Factory Image.
The Linux & swap are logical.
Not sure about the last NTFS  or Factory image. That would have been originally primary, but you probably can make it logical. Your first summary report showed it as logical. 

Many users burn a couple copies of the factory image using its tools to create the recovery DVD set and just delete that partition. Since when hard drive fails and you have to restore image, you cannot restore from a dead drive.

I would hesitate on using testdisk to recover SFS or dynamic partitions. Only if you have good backups of all data would I try that.
Many users have used the tools I posted in Post #3 with success.
But there are two threads where users did use testdisk.

### Post by oldfred on 2014-12-19
The * should be on sda1 as that is Windows boot files. You may not be able to boot from sda2 unless you had run Boot-Repair before. Many users do not know how essential the hidden in Windows 100MB boot partition is and delete it. So Boot-Repair copies boot files from sda1 to sda2 so either partition can boot. But sda1 also has a recovery mode. But you should create a separate Windows repairCD or flash drive.

The only reason of the recovery partition or set of DVDs is to restore system to as purchased. Often best to do if selling system as then it has none of your data. And selling a system with erased drive or with Linux is not as valuable.
Swap is not essential to recover. Ubuntu will boot without it, but since entry is in fstab and then it cannot mount it, it will complain. And you can create a new swap, but it will have new UUID, so you have to edit fstab with new UUID to get swap working again.
I think deeper search is to let you copy files to another drive as backup in case you did not have backups. Of course all users have current backups.

I am not sure why it would say structure bad. That usually is overlapping partitions which you cannot have, or having logical partitions with a primary in between as you cannot have two extended partitions.

I have never use Clonezilla, but many have recommended it.

### Post by oldfred on 2014-12-20
Restore those partitions.

Then use gparted on live installer to recreate a new swap in the same space that your old swap was. It should be unallocated.
If not in extended partition first expand extended partition to include it, so swap becomes a logical partition as you cannot create it as primary, if you have used all 4 primary partition.

System should let you boot, but with an error on missing swap. Just say continue.
      
 To find UUID of new swap.
 sudo blkid -c /dev/null -o list

sudo cp /etc/fstab /etc/fstab.backup
Copy UUID from terminal list of UUIDs and paste over old UUID:
gksudo gedit /etc/fstab

 # Anytime you edit fstab always do this before rebooting. If no errors it just remounts everything, but if errors you have to fix before rebooting or you may not be able to, Make sure you have partition unmounted if previously mounted when creating new mounts:
sudo mount -a
