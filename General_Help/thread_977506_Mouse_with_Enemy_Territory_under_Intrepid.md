### Post by gerryAU on 2008-12-13
G`day after much trial and error, trolling different sources & pages with similar problems, I have managed to revert to my 8.04 xorg cfg with some slight changes, I have all my mouse buttons back :)

I had to add ```
Option "AutoAddDevices" "off"
```to the ServerLayout part of xorg.conf which stops input hotplugging (hal)

I also had to completely remove the section
```
Section "Files"
    RgbPath         "/usr/X11R6/lib/X11/rgb"
EndSection
```

Now my new xorg.conf looks like this; (nvidia card with deathadder or mx510/518 mouse)

```

# xorg.conf (X.Org X Window System server configuration file)
#
# This file was generated by dexconf, the Debian X Configuration tool, using
# values from the debconf database.
#
# Edit this file with caution, and see the xorg.conf manual page.
# (Type "man xorg.conf" at the shell prompt.)
#
Section "ServerLayout"
    Identifier     "Layout0"
    Screen      0  "Screen0" 0 0
    InputDevice    "Keyboard0" "CoreKeyboard"
    InputDevice    "Mouse0" "CorePointer"
    Option "AutoAddDevices" "off"
EndSection


Section "Module"
    Load           "dbe"
    Load           "extmod"
    Load           "type1"
    Load           "freetype"
    Load           "glx"
EndSection

Section "InputDevice"

    # generated from default
    Identifier     "Mouse0"
Option       "Buttons" "10"
    Driver         "mouse"
    Option       "Protocol" "explorerps/2"
    Option         "Device" "/dev/psaux"
    Option         "Emulate3Buttons" "no"
    Option         "ZAxisMapping" "4 5"
Option "ButtonMapping" "1 2 3 6 7"
EndSection

Section "InputDevice"

    # generated from default

### Post by gerryAU on 2009-04-19
After many months of looking for a way to use all my mouse buttons with hal input hotplugging in xorg, without the use of a fdi file. I have found using xmodmap with the unmodified xorg.cfg that ubuntu creates after the glx video card driver is installed works & also retains all the new notifications in 9.04 (multimedia keyboard shortcuts don`t work unless you use the original xorg.conf or a program like keytouch with a custom xorg.cfg as in my previous post)
This xmodmap works with my mx518 & my deathadder mouse
```

xmodmap -e "pointer = 1 2 3 4 5 8 9 6 7"

```

Add it to startup System-Preferences-Startup Applications (gnome)

Or add it to ~/.Xmodmap
```

pointer = 1 2 3 4 5 8 9 6 7

```
Then an "xmodmap .Xmodmap" to apply it.

Also you could make a script & add it to autostart in kde.
```

#!/bin/sh
# add to /home/(user)/.kde/Autostart
# mouse button mapping mx510/deathadder with new input hotplugging (hal) xorg
xmodmap -e "pointer = 1 2 3 4 5 8 9 6 7"

```

hope this helps
