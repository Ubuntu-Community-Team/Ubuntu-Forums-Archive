### Post by shawnrgr on 2008-10-21
can you post your xorg.conf? also...

drop to a terminal (alt+f3) and use this to see your drives..

```
sudo fdisk -l
```

do you see that UUID? if not check your fstab

```
cat /etc/fstab
```

```
sudo vol_id -u /dev/sda1
```

make sure to change the /dev/sda1 to whatever drive you want to check.

you may need to set the proper UUID in your fstab that corresponds to your fdisk -l output

### Post by Elfy on 2008-10-25
OK - the UUID's have changed and fstab no longer shows the correct numbers which is why you get.

Start the livecd, do not try and mount the partitions from nautilus.

Run these commands to mount the linux partition

```
sudo mkdir /mnt/tmp
sudo mount -t ext3 /dev/sda2 /mnt/tmp
```

Now we'll open the file to edit it

```
gksudo gedit /mnt/tmp/etc/fstab
```

Change the lines to the new numbers

```
# /etc/fstab: static file system information.
#
#  -- This file has been automaticly generated by ntfs-config --
#
# <file system> <mount point>   <type>  <options>       <dump>  <pass>

proc /proc proc defaults 0 0
# Entry for /dev/[COLOR="Red"]sda2[/COLOR] :
UUID=[COLOR="Red"]83b09cce-3108-42d2-b1e6-d9437a5e5efd[/COLOR] / ext3 defaults,errors=remount-ro,relatime 0 1
# Entry for /dev/[COLOR="Red"]sda3[/COLOR] :
UUID=[COLOR="Red"]f3900651-550a-428f-8558-7b29bda2763f[/COLOR] none swap sw 0 0
/dev/hdb /media/cdrom0 udf,iso9660 user,noauto,exec 0 0
/dev/sda1 /media/Windows ntfs-3g defaults,locale=en_US.UTF-8 0 0
/dev/sdb1 /media/share vfat defaults,umask=000 0 0
```

Save and close, try to reboot now.

Incidentally, looking at the old fstab which references sdb2 and sdb5 that you had other partitions on the second drive.

In future if you boot and get a fsck exit error 8 and then Ctrl+D to reboot it has meant (for me in the past) that fstab can't find a drive.
