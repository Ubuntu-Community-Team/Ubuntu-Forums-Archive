### Post by srs5694 on 2011-02-27
I should stay out of this, but I can't. Choice of programming language has to do with a lot of things, including the type of problem being tackled, the available resources, the knowledge of the person doing the programming, and the preferences of the person doing the programming. Don't overlook the importance of that last item. People can get downright irrational about their language choices, just like their choices of OS, car, clothing, etc.

IMHO, absolute statements ("Language X is terrible! You should never use it!" or "Language Y is the best one ever invented! You should use it for everything!") reflect the speaker's prejudices more than anything else. For this reason, I ignore them. In reality, every popular programming language has its advantages and disadvantages. Some are better or worse general-purpose languages, but each has its place.

Also, the fact that Programmer A likes or dislikes a particular language is unimportant. Chances are you don't think exactly like Programmer A, so no matter how accomplished that individual is, that person's preferences should not sway you one way or another. (Of course, if you read an extended essay or interview or have a conversation with Programmer A, so as to learn specific reasons for the like or dislike, you can better judge whether the specific reasons might be important to you.)

Nobody here but AmanSin knows enough about the project at hand to make a judgement about what languages are well-suited to handle it. If advice on that specific project is needed, more information must be posted. I don't think that's really the point of the question, though. To address that point, my advice would be to learn as many languages as possible. If you're going into a career in programming, then of course learning the popular ones (C, C++, Perl, Python, Java, probably a few others) makes sense. If you're just a hobbyist, then learn whatever seems interesting to you, or whatever's used by the open source projects to which you want to contribute. Once you know a few languages, you'll be able to choose a suitable one for whatever your current project is. In the meantime, ignore the language snobs and use whatever you know well enough to handle the current project. (Keep in mind, though, that to learn a language, you need to push your boundaries, so you may want to use something you don't yet know all that well in order to learn it better.)

#define LENTH 21

int main()
{

	printf("Reversed String is %s",str);

void reverse(char str[])
{	int i,j;
	char c;
	for(i=0,j=(strlen(str)-1);i<(strlen(str)-1)/2;++i,--j)

}
