### Post by unutbu on 2008-08-28
This is just a wild guess, but perhaps check that you are the owner of ~/.esd_auth (as opposed to root). Make sure you have read/write permission too.

My wild guess is if you run sudo (instead of gksu) on a graphical app maybe root overwrote your ~/.esd_auth and it became root's. Then when you as a normal user run the same app and it tries to read ~/.esd_auth you are left without proper ownership...?
