### Post by ian-weisser on 2015-04-29
No such thing as a 'repair' install.
All install/reinstall activities risk data loss. Backups are wise.
A hardware issue is unlikely to be fixed by a reinstall. What would be different?

If it's a missing-driver issue, then simply locate and install the correct driver.
If the correct driver is already installed but inactive, then simply adjust the blacklist so the kernel assigns it to the hardware.
