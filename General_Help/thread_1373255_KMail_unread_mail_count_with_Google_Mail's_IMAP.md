### Post by StiltonSandwich on 2010-01-05
I recently upgraded from Kubuntu 8.04 to 9.10.

In KMail on 8.04, when I unchecked the option to "Act on new/unread mail in this folder" for a folder, not only did the folder not count towards the mail count in the system tray but it was not shown as containing any new mail until it was actually opened.

In 9.10, the same option only prevents the folder from contributing to the total unread mail count; the folder itself is still marked as containing new mail.

I access Google Mail via IMAP. For anyone who is not familiar with Google Mail's IMAP functionality, the folders are actually more akin to 'tags', and the same message can appear in more than one folder. For example, a new message will turn up in both the "All Mail" and "Inbox" folders.

On 8.04, this was not a problem, because I could simply set the "All Mail" folder to not show whether it contained new messages. However, I now have to view every message twice in order to get rid of all the new mail indications.

I can also no longer hide the number of new spam messages in the same manner.

Is this a bug? Is there another option somewhere that I have forgotten about since I configured KMail in KDE 3.5?

I considered applying a filter to incoming mail in the "All Mail" folder to mark it all as read, but that would not be a satisfactory solution because it would mark the mail in Inbox as read too (via Google's server), with unpredictable results.

Has anyone else encountered this problem or found a way to solve it?
