### Post by Zorael on 2008-04-26
To answer your question, **kwin --replace &** replaces compiz as the running window manager with KDE's standard one (kwin).


To troubleshoot: do you get to the login window, or does it never enter the graphical interface?


If so (never get to login, else skip to below), Compiz isn't at fault but instead your video driver. Please enter the following commands, when it's broken, from your terminal.
```
$ cp /var/log/Xorg.0.log ~
$ sudo dpkg-reconfigure -phigh xserver-xorg
$ sudo killall kdm
$ sudo kdm
```
Then post the contents of the Xorg.0.log file in your home directory (~).


If it goes black after having logged in, Compiz is likely at fault, yes. Try doing the following from a terminal, *when* the graphical interface is running.
```
$ DISPLAY=0:0 kwin --replace &
```
That should give you control over your windows again. Then open up Konsole and enter the following.
```
$ compiz --replace --ignore-desktop-hints & sleep 5; kwin --replace &
```
That will launch compiz, wait for 5 seconds, and then return to the standard window manager. Then please post whatever Compiz output.
