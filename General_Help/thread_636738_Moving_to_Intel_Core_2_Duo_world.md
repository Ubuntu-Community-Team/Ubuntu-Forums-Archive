### Post by 3rdalbum on 2007-12-10
What he means is that, if you use the 32-bit version, you don't get any compatibility hassles; so it's fun to use, rather than a pain.

Unless your laptop has more than 4 gigabytes of RAM in it, you don't need to be running a 64-bit operating system. The 32-bit version will run perfectly on a 64-bit processor - in fact, it was a good six months after I installed Ubuntu on this machine that I realised the computer is actually 64-bit :-)

### Post by chewearn on 2007-12-10
I have been running strictly 32bit until recently, when I discovered that running folding@home 64bit smp gives you way more points (could be about 10 time more).

I then spent a few days reading up about pros and cons of 32bit vs 64bit (see the sticky threads in the x86_64 section).  The conclusion I got was that 64bit ubuntu is now pretty much ready, and as good as 32bit.

So, I am now happily running 64 bit on two systems, one a AMD64 X2 desktop, and one a Core2Duo laptop.

No issue so far; only tricky part is installing 32bit firefox as a second browser to run java (so i can access online banking).  And setting up/compiling the kernel module for a wizardpen tablet.

You mileage may  vary, depending on you PC usage.
