### Post by Hagar Delest on 2011-02-09
There are some reports about bugs in LibO, some of them having workarounds already.

Personally, I still use OOo at work. LibO has some show stopper "features" for me so I won't use it until I can change those things.

The big question is now about the quality tests: will users massively switch to LibO and then the OOo build may suffer less complete tests? Will the many contributions be robust enough to integrate all together?

As for the support, since the forums is the best way IMHO, let's see how it grows. LibO team preferred to run their own forum instead of using the already established ones about OOo (user.services.openoffice.org and oooforum for the main ones). Difficult to say how it will evolve!

### Post by Hagar Delest on 2011-02-10
OOXML support will be indeed a major difference as long as Oracle doesn't want to implement it. Personally, I agree with Oracle, the less it is supported, the less this format may develop. This file format should be avoided like the plague.

On the other hand, it means that it is difficult for users to have an alternative to MS Office. But I think that the benefits of not supporting OOXML (i.e. making the users aware of the risks using it) overwhelm the risks of frustration with a format that is impossible to implement correctly.
