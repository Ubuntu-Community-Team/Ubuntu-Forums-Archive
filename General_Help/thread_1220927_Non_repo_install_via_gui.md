If the program is in .deb package you can just click it to install.

For everything else you pretty much just have to check the instructions that came with the program. That's all that can be said as if it's not from repositories and it's not a native .deb package it can be pretty much anything.

There's no standard beyond the standard of using repositories/.deb. Pretty much the same situation if you had a zip archive of windows program that doesn't include a .exe installer..

That being said most of the time the programs that aren't distributed as native binary archives (.deb/.rpm) the archive simply contains the source code. And there isn't any GUI method to compiling and installing from source.
