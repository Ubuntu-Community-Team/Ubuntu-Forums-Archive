### Post by az on 2005-03-27
You get a stable release now, and another stable release in six months.

A longer release cycle promotes security and stability.  A shorter one promotes newer software.  You will find that most of the stuff in a newly released Ubuntu is pretty much cutting edge.  It is for me, the best of both worlds.  

You can always use Debian Sid, if you want new stuff without stability.  If you want stability without the newest features, use Stable (Debian or Ubuntu)

If slackware offers something you need, use it, by all means.



Slackware is that up to date?  I thought it too had a longish release cycle.  I also thought that the package management was really a pain in Slackware.  How do you handle new stuff in such a system without breaking it and not having to konw it intimately?
