### Post by Zill on 2007-05-10
Some organisations, such as banks, allow downloading of html files with an .xls extension presumably in the assumption that these files will usually be opened by MS Excel.

OpenOffice 2.0.2 correctly opens these files by default in Calc when running Kubuntu 6.06 LTS with KDE.  Konqueror file properties show Contents as "HTML document" and File Type as "application/msexcel".

However, when running Ubuntu 6.06 LTS with Gnome, the identical version of OpenOffice opens the same file by default in Writer/Web.  Nautilus file properties show Type as "HTML document".  The "Open With" tab shows a list of html applications with Firefox as the default.  Adding and selecting OOo Calc to this list still does not open the file in Calc.

It is possible to force OOo to open html/xls files in Gnome OOo Calc by opening the file manually by setting the OOo filetype to "HTML Document (OpenOffice.org Calc), but this is not the default option.

Any ideas how to change the Gnome OOo functionality to open these html/xls files in Calc as per KDE?
