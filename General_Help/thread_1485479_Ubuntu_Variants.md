### Post by Oasu4g on 2010-05-16
Usually there will be a big difference between what programs come with each variant. Say, with Gnome, you get the DE with several other applications developed under the Gnome project, like say, Rhythmbox, or the network manager, for instance. I believe KDE uses an audio manager other than Pulse audio, also. So each variant will have big differences between them, but they all run basically the same way, and "under the hood" they're all Ubuntu.

### Post by XubuRoxMySox on 2010-05-17
"Under the hood" they're all Ubuntu, and Ubuntu is *"GTK-biased."* Others have called it *"Gnomecentric."* What we mean by that, is that the base is built for a GTK environment. Gnome, Xfce, and LXDE are all GTK environments, but KDE is QT-based. KDE applications work just fine in any GTK environment, but it becomes necessary to add on abuncha QT *libraries* to support the QT environment and/or QT applications. If you have plenty of hard drive space, it's not a problem. Kubuntu is *a GTK-based distribution with QT added*. 

On my aging hand-me-down desktop, Kubuntu was very slow and sluggish (as well as bewildering, despite the Fischer-Price looking desktop). Gnome was much faster and less confusing. Lubuntu (and LXDE in general, on my hardware) was maddening with persistent bugs, but it's plenty fast. Xubuntu strikes an ideal balance for my old hardware. Full-featured like Gnome (only faster and much simpler) but light and responsive like LXDE.

Try 'em all! That's really the only way to know what the differences will mean to *you* and how they perform on *your* machine(s).
