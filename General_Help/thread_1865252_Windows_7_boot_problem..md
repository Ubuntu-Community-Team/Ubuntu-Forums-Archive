### Post by oldfred on 2011-10-19
Windows NTFS partitions also have to have a valid NTFS signature in the PBR partition boot sector. Partition table says it is NTFS but PBR is unknown.

Testdisk may be able to recover the backup that NTFS keeps.

You need to move boot flag to your sda1 recovery partition and run this as you have installed grub to that PBR. Then move boot flag back to sda2 and run same repair to see if that will fix sda2. 

Fix for most, a few have other issues, better than windows fix in many cases as it also fixes other parameters:
If win7 use small 'system reserved' NTFS partition instead of the partition where windows was installed for win7
This has instructions on using testdisk to repair the install of grub to the boot sector for windows from Ubuntu or Linux LiveCD.
[http://sourceforge.net/apps/mediawiki/bootinfoscript/index.php?title=Boot_Problems:Boot_Sector](http://sourceforge.net/apps/mediawiki/bootinfoscript/index.php?title=Boot_Problems:Boot_Sector)

You can move boot flag with gparted, click on partition and right click manage flags or use Disk Utility.

Part of testdisk which is in the repositories or most recovery CDs. 
[http://www.cgsecurity.org/wiki/Advanced_NTFS_Boot_and_MFT_Repair](http://www.cgsecurity.org/wiki/Advanced_NTFS_Boot_and_MFT_Repair)
As described, it has an option to "Recover NTFS boot sector from its backup"
If Backup BS isn't available, choose RebuildBS. 


Instructions
[http://www.cgsecurity.org/wiki/TestDisk_Step_By_Step](http://www.cgsecurity.org/wiki/TestDisk_Step_By_Step)
[http://www.cgsecurity.org/wiki/Menu_Analyse](http://www.cgsecurity.org/wiki/Menu_Analyse)
[http://www.cgsecurity.org/wiki/TestDisk:_undelete_file_for_NTFS](http://www.cgsecurity.org/wiki/TestDisk:_undelete_file_for_NTFS)

### Post by oldfred on 2011-10-20
A standard Windows 7 install has a hidden 100MB boot/recovery partition that has the boot flag. That would be the one grub would use to boot from. But you still have grub in the NTFS boot sector.

Windows boots from the MBR and in effect just chain loads to the boot partition just as grub chains to it. But the NTFS partition has the NTFS file structure (Somewhat duplicating the partition table info) and a reference on what file to boot with ntldr for XP and bootmgr for Vista/7. Hirens just about has to skip all that and just use bootmgr to boot. With Vista/7 you also have to have the BCD which tells which partitions to boot from(like XP's boot.ini but in a hive like Windows registry).

You have to move boot flag to sda1 and run the test disk repair on that. It has to show a NTFS partition.

```
sda1: _________________________

    File system:       ntfs
    Boot sector type:  [COLOR=Red]Grub2 (v1.99)[/COLOR]
    Boot sector info:   [COLOR=Red]Grub2 (v1.99) is installed in the boot sector of sda1 
                       and looks at sector 981149936 of the same hard drive

                       for ?? on this drive. No errors found in the Boot [/COLOR]
                       Parameter Block.
    Operating System:  
    Boot files:        /bootmgr /Boot/BCD
```

Both of the red entries must be NTFS.

### Post by Quackers on 2011-10-21
It is unlikely that you need to re-install Ubuntu. Just re-installing grub to the mbr of the drive should be enough.
From a live cd desktop open up a terminal and if your Ubuntu installation is on /dev/sda5 run these two commands one at a time, pressing enter after each line ```
sudo mount /dev/sda5 /mnt
sudo grub-install --boot-directory=/mnt /dev/sda
``` whcih should report no errors. If so, reboot and Ubuntu should boot directly. If it does, open a terminal and run ```
sudo update-grub
``` when you should see the Windows Loader recognised. If so, reboot and you should see a grub menu giving you the choice of which OS to boot.
Try both options to make sure they work.

It may be an idea to move the boot flag back to where it was originally before carrying out the above.
```
