Ok, so in 2007 people complained about console-kit-daemon spawning about 60 threads and using a lot of cpu in certain cases.

In 2008, same thing.

In 2009, same thing.

It's almost 2010, Karmic is (almost) ready and the problem is still there.

The problem is, despite the fact that a patch has been posted, developers seem to prefer to spend their time trying to convince us that this behaviour is "normal" when it's not. See for example the comments in one of the bug reports: [https://bugs.launchpad.net/consolekit/+bug/148454](https://bugs.launchpad.net/consolekit/+bug/148454)

A patch to fix this BUG has been released a long time ago ([http://patchwork.kernel.org/patch/33044/](http://patchwork.kernel.org/patch/33044/)), so I wonder why nobody is merging it in the new Linux releases? ](*,)
