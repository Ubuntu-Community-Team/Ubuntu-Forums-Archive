### Post by Impavidus on 2020-01-09
Your second screenshot also list a 2GB loop device. May I conclude you run a live session ("Try Ubuntu before installing")? That doesn't use the internal harddrive unless told so.

If your intention is to install Ubuntu, let the installer do its job. I think it mounts the target partition at /target. Otherwise, explain.

Also, read up on file permissions: [https://help.ubuntu.com/community/FilePermissions](https://help.ubuntu.com/community/FilePermissions)
File permissions are at the core of security in Linux. Use it, don't work against it.
