### Post by bpalone on 2009-07-14
Here are couple:

Running Linux by Dalheimer & Welsh, published by O'Reilly

Linux In A Nutshell, by Siever, Webe, Figgins, Love & Robbins, published by O'Reilly

The first one is distribution neutral and covers most all topics. It and the internet will answer most all of your questions.  The second one, is great when you are wanting to do something and can't remember the command, but think it is xxxx.  Most of the book lists Linux commands alphabetically which makes for a good reference book to have around.  There are others out there, but these would be the two I would recommend.  In fact, the second one was this years pick in Linux Journal.

Welcome to the new world and remember to have fun.

### Post by Revolutionary101 on 2009-07-14
On the PC you installed linux on open Synaptic and find the packages you want to install and mark them for installation then in the File menu, select "Generate Package Download Script" then put it on a flash drive and open it on either a mac or another linux computer. It will not work on a Windows computer. It will download the .deb packages for you.

I agree with bpalone's recommendations... the nutshell series are usually good and they provide a short and concise presentation, which is useful for rapid learning if you are an experienced user of any operating system. ### Post by gldearman on 2009-07-14
> **dhughes said:**
> Not a book, well it is sort of, Linux From Scratch (LFS) is pretty good. [http://www.linuxfromscratch.org/](http://www.linuxfromscratch.org/)

Yeah, I agree with the *Linux from Scratch* recommendation.  If you actually use the book and build a Linux installation from scratch, you will know everything there is to know about Linux.

I also suggest two others.

The [Slackbook]("http://www.slackbook.org/") is specifically about the Slackware distro, but it is very user-friendly and you can learn a lot about Linux in general from it.

I also recommend [*Rute User's Tutorial and Exposition*]("http://rute.2038bug.com/index.html.gz").  But that comes with the caveat that it is very old (2001).  Some of the info in there is seriously outdated.  But a lot of it is not, and it is very in-depth and thorough.

Oh, yeah, and welcome to Linux (where the real geeks hang out)!  Have fun!
