### Post by aeronutt on 2021-12-26
There seems to be a problem will nearly all or all Dell laptops with 11th generation (Tigerlake) CPUs.
Randomly, lagging and inaccurate touchpad. It's quite annoying by the way, making the touchpad/pointer nearly unusable for 1-5 minutes several times an hour. There are a handful of threads on the issue:

[https://gitlab.freedesktop.org/libinput/libinput/-/issues/618](https://gitlab.freedesktop.org/libinput/libinput/-/issues/618)
[https://www.dell.com/community/XPS/XPS-15-9510-Touchpad-Intermittent-touchpad-lag/td-p/8012567](https://www.dell.com/community/XPS/XPS-15-9510-Touchpad-Intermittent-touchpad-lag/td-p/8012567)
[https://bugzilla.kernel.org/show_bug.cgi?id=214597](https://bugzilla.kernel.org/show_bug.cgi?id=214597)

Linux distribution doesn't seem to matter, Ubuntu, Fedora, Manjaro. Desktop choice doesn't matter, gnome, xfce, etc.
Making changes to touchpad parameters using xinput while the pointer is erratic, didn't change anything.
Gnome restart, doesn't change anything.
I'm running out of things to try for a temporary fix.  Even if I could find a command to run from a hot-key to reset the problem, I'd deal with it.

Any insight, ideas, or clues for a fix or workaround would be appreciated.
