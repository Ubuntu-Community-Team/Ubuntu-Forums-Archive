### Post by deadflowr on 2015-05-20
Moved to **General Help**.
Since booting into text-mode is the antithesis of running a desktop environment...

Basically, though, you are setting it up incorrectly.
Better to edit the file
/etc/default/grub
than to edit the /boot/grub/grub.cfg file.

The /boot/grub/grub.cfg file only holds as it is until an update comes to grub, then the file gets overwritten.
The grub.cfg file is generated by using what is set in the /etc/default/grub file as well as the scripts in the folder /etc/grub.d.

So...
better to set the configuration file instead.

I believe to set text mode you can open  a text editor as root.
Something like
```
sudo nano /etc/default/grub
```
and add a # to the line
```
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"
```
like so
```
#GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"
```
 and change the line for 
```
GRUB_CMDLINE_LINUX=""
```
to
```
GRUB_CMDLINE_LINUX="text"
```
then save
(ctrl + x to save and exit in nano; press y to accept the modifications and then enter)
then run 
```
sudo update-grub
```

There might be a better method, but I know this one works.
And it'll stay this way forever, unlike an edit to the grub.cfg file, which won't.

Hope it helps.

Edit: here's a nice reference on grub
[https://help.ubuntu.com/community/Grub2/Setup](https://help.ubuntu.com/community/Grub2/Setup)

No. ONLY tty1 shows such chars.

in /etc/init/tty1.conf, i write: exec /bin/login -f myuser </dev/tty1 > /dev/tty1 2>\&1. and in ~/.bashrc i write: if [ $(tty) == "/dev/tty1" ]; then exec startx fi. and in /usr/bin/startx i write: xinit "$client" vt1 -- "$server" $display $serverargs.

in such way I start both X11 server and XFCE on /dev/tty1. this is what i do to boot a text mode and then XFCE. I bypass LightDM because it's more heavy than text mode.
