well saying it is "absolutelly horrible" doesn't provide much description on what is the issue. 
 
It could be that upgrade simply failed, it could be you had some proprietary drivers or PPA's before but he forgot to disable them before upgrade. a clean install should solve these issues.
 
As i know latest version should work faster than previous ones. if howveer you are sitll having a problme wiht speed you can try using the KDE-netbook desktop (perhaps that is what you used before?!), then there is the new low-fat desktop that consumes less resources. but it all depends on what the problem is...
 
if things don't work on it it could be that it has some specific components that were made for windows only. and for these to work in linux, the  drivers had to be reverse engineered or some other solutions found which is not always a good idea. in that case perhaps the best is to simply use windows. it is much less troublesome.
 
as for lighter, less system resurces consuming distributions you could try:
from Ubuntu family:
Xubuntu (lighter than Kubuntu and the regular Ubuntu)
Lubuntu (very light, but looks a bit like old Windows :), however it's a modern operating system that uses very little resources to run)
 
Another good option seems to be Chrunchbang linux based on Debian Stable , but a bit more user friendly than Debian itself.
