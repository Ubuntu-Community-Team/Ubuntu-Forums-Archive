+1. Have you thought of this; have a second install for play. Keep your current install stable/production and try things out on the play install. This is sometimes really helpful if you want to try out some software before installing it on the main system. You can also use your exsisting /home partition (if you have one) and /swap for the second and consequent installs; don't need to create new ones.

This the way to do it.

I still have 10.04, with separate /home partition, as my main system on this desktop box working brilliantly.  On a second hard disk in the machine I also have 12.04 versions of Ubuntu, Xubuntu, Lubuntu and for something rather different, Bodhi.  They all have data files linked to the /home of my 10.04, but have their own homes in the root partitions.

This is a great way to look at new OS or version possibilities and not throw away the old working system until you have had a chance to test everything as much as you want.
