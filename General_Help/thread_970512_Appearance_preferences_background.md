### Post by loomsen on 2008-11-04
> **Convert said:**
> Thanks, loomsen, I found the preference file.  It's not in /usr/share/backgrounds/*somexml*, it is in /home/<username>/.gnome2 and the file is called "backgrounds.xml".

Thats the one for your $USER account only.

The global file is located under /usr/share...
```


me@tiffany:~$ locate xml | grep background | grep -v home
/usr/share/gnome/help/control-center/C/config-background.xml
/usr/share/gnome-background-properties/debian.xml
/usr/share/gnome-background-properties/gnome-abstract.xml
/usr/share/gnome-background-properties/gnome-nature.xml
/usr/share/gnome-background-properties/ubuntu-wallpapers.xml
me@tiffany:~$ 

```

Global files are usually located there, so if you as admin want to make a theme available to all users, like an icon theme for example, place it at /usr/share/icons/<themename>

You may as well define whether users shall be able to override your settings or if they shall be mandatory for all users.

Modular, means exactly that... multiple path' are 'known' to the system.

Your shell, your defined path' etc then build your environment. So your system lives basically. Thats why you won't be able to have any binary installed twice.  Grab some cheatsheet...
