### Post by HiImTye on 2012-11-01
even with the added overhead of the larger addressing, 64bit has a larger memory pipe, which results in a net speed benefit, not a speed loss. this is without even considering added benefits of the 64bit architecture in areas like instructions and features.

if all of the programs you use work in a 64 bit environment (and with multiarch, pretty much everything does), there is no reason to stick with 32bit, especially if you want to take advantage of the extra security features of 64bit instructions. using the PAE kernel is slower than even the regular 32bit kernel
