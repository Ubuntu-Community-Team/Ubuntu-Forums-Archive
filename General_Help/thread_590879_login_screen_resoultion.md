### Post by TeaSwigger on 2007-12-21
Hello, 

Where are you at in this? Still the same?

I had some kind of problem like this (with Feisty) on my 2nd PC. The screen size at the login screen was so far off it was nearly unusable. I'm sorry to relate I don't remember how it was fixed, but it was. Some of the things I do remember doing:

- Installed the right friggin' drivers for the built-in Intel graphics. I needed the 915resolution fix and the new(er) intel drivers. Set it to use restricted drivers and specified the monitor (had to set the right driver in xorg.conf).

- Used Xorg-Edit. It's a tool you can find somewhere in these forums. It specified previously missing monitor stuff like displaysize etc, entered it in the xorg file and let me test it first. Nice, since I was overwhelmed at the time.

- Specified a vga=# in the kernel line in /etc/X11/xorg.conf. In this case it had to be vga=0x305 (8 bits, for goodness sake, oh and something I hadn't seen mentioned, it had to be in hex, the decimal version - vag=773 - was a no-go). Edit: note this only sets bootup res; when X loads at the login screen it runs at 24bit depth from there on. Which is slow sometimes; the cheap built-in graphics card likes to run at 16bit better.

- ran sudo update-initramfs -u -k `uname -r` as I'd changed the usplash resolution. 

Something in all that fixed things. I finally have usplash, login and desktop res all working perfectly. I only hope there's something useful to you in all that. Luck!
