### Post by igknighted on 2007-02-03
Well, a lot of it is opinion.  I have never been satisfied with the way kubuntu is put together.  Ive always felt that it was an afterthought, like "oh, lets just throw KDE on top of this".  I say this as a user who loves what Ubuntu has done, it is a tremendous OS, I just find the KDE offering lacking.

Mepis uses the same repositories (and hence mostly the same package base) as Dapper.  The stability has never been an issue, but they do use their own repo's, Dapper's, and also Debian's.  If you can deal with the older packages, and dont mind changing the way it looks to something that you like (the default desktop is VERY bland and well, not very pretty) then give it a shot.  It comes via a DVD unfortunately, so its a big download, but it will run live so you could play around with it before you installed.  Overall, I think it is more stable, because the whole distro is built around providing good packages for a KDE distro, while Ubuntu is made for gnome and repackaged with KDE.

As for KDE4, no distro will have KDE4 preinstalled until a while after it is out (needs time to get into a distro in the alpha stage then go through testing and such).  I assume the Mepis release after the first Ubuntu release with KDE4 will be the one with KDE4.  As with any distro, you could always just install KDE4 onto whatever distro you are using when it comes out if you dont want to wait.

### Post by harley_frog on 2007-02-03
As a current MEPIS user, I have to say it is an excellant distro.  I have yet to try Kubuntu, but I'm sure whichever distro is the first to come out with KDE 4.0 will be the one I switch to.  Both distros use the Ubuntu repositories, but MEPIS adds a couple additional ones.  I don't see much of an advantage except, perhaps, for Opera (which is in the MEPIS repo).  Either way, I would still need to download a butt load of packages that neither distro comes packaged with (GIMP, GnuCash, XGalaga, emacs, etc.).  Thankfully, I have two computers that are more or less clones (although I may need a new hard drive for the other one), so I can install Kubuntu, do the updates, get lists of installed packages for both systems, run diff and output a list of packages to install on Kubuntu.   Gads, I love Linux! ;)

The code to list all installed packages (for those interested):

```
apt-cache --installed dumpavail | grep 'Package: ' | sed 's/Package: //' > ~/installed_pkgs.txt
```
