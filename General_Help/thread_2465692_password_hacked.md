### Post by HermanAB on 2021-08-09
Note that if you change language, your keyboard layout may also change, which may make your password difficult to type in correctly.  I get around this conundrum by setting a password that is the same on both Qwerty and Azerty keyboards.

### Post by HermanAB on 2021-08-09
BTW, provided that you use a looong password >14 characters (mine is about 25) then you are good against password attacks. 

If you do not run VNC, FTP or Telnet servers then you should be good against remote login attempts.

If you need remote login, use SSH with either a very long password (e.g. a modified nursery rhyme) or a key (which is merely a very long computer generated password).
