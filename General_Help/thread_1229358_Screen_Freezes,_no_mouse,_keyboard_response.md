### Post by sydbat on 2009-08-06
Have you tried this:

hold down Alt+SysReq and type R E I S U B

It *should* safely reboot a non-responsive Linux box.

If it does not, and you have to hard reboot, you may have a more serious problem...most likely a hardware issue. If this is the case, run the memory check from the GRUB recovery menu or insert the Live CD and check it that way. Sometimes RAM can go flaky.

If that all goes well, the next thing to do is shut off the box, unplug it, open it up and check every connection, making sure everything is seated correctly and cleaning out any dust buildup (I use a vacuum).

After all that, if you still have problems, it might be the power supply is going and causing random voltage changes (not a good thing) or the motherboard has a problem (like a solder line has a microscopic crack) or something else that you cannot diagnose without professional help.
