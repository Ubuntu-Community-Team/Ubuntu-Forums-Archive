### Post by josephellengar2 on 2013-10-19
Hi! I'm using 13.10 with Cinnamon and have a problem.

I have some specific settings that I like in my synclient and I have a script in my directory /home/user/.scripts/synclient.sh

This is the script:

```

#!/bin/bash
sleep 2
synclient HorizTwoFingerScroll=1 &
synclient Palmdetect=1 &
synclient TapButton3=2 &

```

Simple right? I've tested it independently and I know it works.

I execute this script at boot with the following startup applications line:
```

/home/ross/.scripts/synclient.sh

```

This works too.

Then, I also want this script to execute at resume from sleep, because the settings don't persist.

For this, I have the following script in the directory /etc/pm/sleep.d

This script is titled 99_synclient.sh:
```

#!/bin/sh

case "$1" in
    resume)
        /home/ross/.scripts/synclient.sh
esac

```

This doesn't work. I don't know why. According to everything I have read, that script should execute as root when the system resumes. It is executable. I checked. Any advice?

Thanks.

### Post by Toz on 2013-10-20
synclient requires an active X session to work. When resuming, the scripts in /etc/pm/sleep.d are run as root and it doesn't have access to your X session. Try something like this:

1. First rename the file to 0000_synclient.sh so that it is run last on resume to avoid the possibility that something else might overwrite these changes.
2. Try this version of the file that sets up the correct display and runs the command as your user:
```
#!/bin/sh

case "$1" in
    resume)
        DISPLAY=:0.0 su ross -c '/home/ross/.scripts/synclient.sh' ;;
esac
```

You might also want to check the **/var/log/pm-suspend.log** file for errors if it still doesn't work. (I'm thinking you may also need to export your Xauthority to get this to work).


Another option would be to set this up from the start by configuring Xorg with these settings. To do this, with root privileges, edit the file /usr/share/X11/xorg.conf.d/50-synaptics.conf (from a terminal window):
```
sudo -i gedit /usr/share/X11/xorg.conf.d/50-synaptics.conf
```
...enter your password when prompted and paste the following at the end of the file:
```
Section "InputClass"
    Identifier "touchpad"
    Driver "synaptics"
    MatchIsTouchpad "on"
        Option "HorizTwoFingerScroll" "on"
        Option "TapButton3" "2"
        Option "PalmDetect" "1"
EndSection
```
Save the file. You will need to log out and back in again to test. By setting it up like this, you may not need to run anything in your application startup or in /etc/pm/sleep.d to get it to work the way you like. It will happen automatically when X starts up.

### Post by Toz on 2013-10-20
Glad to hear it worked. I was wondering if you wouldn't mind doing one more thing for me (like a test). At the top of /usr/share/X11/xorg.conf.d/50-synaptics.conf, you will see:
> # DO NOT EDIT THIS FILE, your distribution will likely overwrite
# it when updating. Copy (and rename) this file into
# /etc/X11/xorg.conf.d first.

The problem is that the file we wrote to might get over-written during an update and you will lose your changes.

So to that end, can you:
1. Delete the stuff that you added to /usr/share/X11/xorg.conf.d/50-synaptics.conf
2. Create the directory /etc/X11/xorg.conf.d:
```
sudo mkdir /etc/X11/xorg.conf.d
```
3. Create the file /etc/X11/xorg.conf.d/50-synaptics.conf with the following contents:
```
Section "InputClass"
    Identifier "touchpad"
    Driver "synaptics"
    MatchIsTouchpad "on"
        Option "HorizTwoFingerScroll" "on"
        Option "TapButton3" "2"
        Option "PalmDetect" "1"
EndSection
```
4. Save the file and log out and back in again.
5. Check to see if it still works. 

I'm curious to see whether it will because Ubuntu doesn't create the /etc/X11/xorg.conf.d directory by default and I'd like to know if Ubuntu supports the use of this directory as per the xorg documentation. If so, it might save you a headache down the road.
