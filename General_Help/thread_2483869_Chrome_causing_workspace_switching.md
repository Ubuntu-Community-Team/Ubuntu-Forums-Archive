### Post by gusgf on 2023-02-11
I am constantly running multiple applications on multiple workspaces. I want to remain on the same workspace, and only switch workspaces when I explicitly ask for it. Currently when I click on the Chrome icon in the app bar Chrome pops up asking me to pick the required profile and if I happen to pick a profile which is open in another workspace I'm switched to that workspace, this is so annoying, I don't want this behaviour. If I open Chrome and select a different profile it will remain on the current virtual desktop that I'm on. How do I get this behaviour for all profiles i.e. staying on the current virtual desktop/workspace no matter what profile I select.


To try and circumvent the above issue I've tried right clicking on the Chrome icon in the app bar and selecting 'New Window'. When Chrome opens it prompts me to pick a profile and If I dare pick a profile that's already open elsewhere in another desktop the mouse icon changes to a spinning circle for half a minute and then disappears and no new window opens?!!


This behaviour doesn't happen with Firefox, or with Chrome on Windows. At the moment the only workaround is to open a new Chrome tab and drag it to the workspace where I'm working. Anyone else having this problem?

### Post by gusgf on 2023-02-11
Since I've been unable to figure out how to post the screenshot directly into this post I will have to describe how I solved this. If you click on the profile pic in your Chrome browser and select from the drop down the cog icon i.e. 'other profiles' it will bring up another Chrome window with all your active profiles. Untick the option in the bottom right corner 'Show on startup'. This now means you can open a window for a profile already open in another virtual desktop/workspace and remain in your current workspace.
