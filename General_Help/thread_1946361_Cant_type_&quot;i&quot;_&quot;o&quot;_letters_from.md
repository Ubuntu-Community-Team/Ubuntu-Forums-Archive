### Post by Dreamer Fithp Apprentice on 2012-03-24
Did you just install the OS or is this a system that was working and for no apparent reason started doing this? If you just installed it I wonder if perhaps in installation you picked a keyboard setup (I think the word may be "map" but don't hold me to it) that didn't match your physical keyboard. I'm not sure why that would affect the onboard unless maybe it wasn't written with sufficient attention to how it might interact with less common keyboard set ups. The other thing that occurs to me is that there is a file that stores how the signals from the keyboard are to be interpreted and maybe it got corrupted. Possibly the tty doesn't use it but uses the keyboard signals raw so to speak. Somebody who knew might could tell you where that file is and how to check for this and for the other possibility as well. But in the meantime, reinstalling doesn't take that long and if you pay careful attention to the stuff at the beginning about picking your keyboard layout being sure at that time to try out the keys you are having trouble with in the testing field that should fix either or both of those two possibilities if one or both was indeed the source of the problem and you might get if fixed before anyone provides you with a more sophisticated answer.

Now if you DIDN'T just install it and this is a system that was working normally until recently, the first thing I'd wonder is what changes have you made recently that might account for it.  Have you just installed something or changed some setting or config file somewhere? I'd be especially suspicious of any software that fiddles with the keyboard somehow such as a key remapper. 

Sorry if I'm pointing out kindergarden stuff but you didn't say anything about what you had already tried or what possibilities you had excluded. Good luck with it.
