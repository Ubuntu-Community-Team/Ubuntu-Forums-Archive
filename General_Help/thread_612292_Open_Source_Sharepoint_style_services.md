### Post by scorp123 on 2007-11-13
[http://www.opengroupware.org/](http://www.opengroupware.org/)

I until recently worked for HP and we used this in our department (around 40 people). Our reason for using this: We were a UNIX-centric group and we were totally unhappy with the heavily Microsoft-centric SharePoint services provided by corporate IT. As server we used an old Vectra VL-400 PC with only 512 MB RAM and a Pentium-3 CPU (800 MHz). So yes, OpenGroupWare is up to the task and it does its job pretty well.

### Post by whitemice on 2007-11-26
We use OpenGroupware.  In fact we've built a very nice CRM application and several Intranet application around it.  One advantage it has is that there is a very good XML-RPC API so you can use it from your own local applications regardless of platform of choice (PHP, Python, C#, Java, etc...) as anything can make an XML-RPC call.

Also the schedular is full featured with support for conflict detection, participant roles and status, and resources.  Many other Open Source "groupware" servers feel allot more like Personal Information Managers with some sharing sprinkled on top,  OpenGroupware is a *group*-ware solution to it's very roots.
