### Post by ajgreeny on 2009-08-27
Try a manual edit of xorg.conf with ```
gksudo gedit /etc/X11/xorg.conf
``` to look like this:-
```
Section "Device"
    Identifier    "Configured Video Device"
EndSection

Section "Monitor"
    Identifier    "Configured Monitor"
            HorizSync    30-80
            VertRefresh    50-75
EndSection

Section "Screen"
    Identifier    "Default Screen"
    Monitor        "Configured Monitor"
    Device        "Configured Video Device"
    DefaultDepth    24
    SubSection "Display"
        Modes        "1366x768" "832x624" "800x600" "720x400" "640x480"
    EndSubSection
EndSection
```
Add your own figures if you want other resolution as well as the 1366x768, these are just my own from a previous xorg, and change the refresh rates if necessary, or omit them all together.

I hope it helps you.

### Post by crowell22735 on 2009-09-10
I recently purchased a Dell 1366x768 monitor and I can tell you what worked for me.  I'm running Hardy Heron.  When I plugged the monitor, it would only run at 1024 x whatever.  I tried editing the xorg.conf file like you did and got the same result.

However When you open the /etc/X11/xorg.conf file, there is a reference to using a "real" utility such as:

sudo dpkg-reconfigure -phigh xserver-xorg

I copy/pasted this into a terminal window and ran it.  It overwrote the xorg.conf file (saving the original in backup with an extension containing the current date).  I restarted the computer and the resolution was set to 1366x768.

Make sure you check your xorg.conf file for the command and copy/paste it from there in case you are running something different.  I found out the hard way that editing the xorg.conf file manually gets messy fast.
