How do I suppress the intermediate purple screen in the hand-off between plymouth and gdm?

### Post by again? on 2018-01-31
You need to install gnome-session and use it's gdm3 theme.
```
sudo apt install gnome-session
```

Choose gnome-shell.css...
```
sudo update-alternatives --config gdm3.css
```

This will change the login screen to grey and the highlight color from orange to blue.
To get rid of the purple flash before the desktop loads you may have to use the Gnome session as your default session
and enable the ubuntu dock extension with gnome-tweak-tool.

Details [HERE]("https://didrocks.fr/2017/09/11/ubuntu-gnome-shell-in-artful-day-11/").

### Post by TJSL on 2018-02-01
The gray login screen of gnome-session is definitely better. However, it  appears it is still necessary to change the GDM login screen  background. 

Using the following, the color of the login screen  can be changed, but noise-texture is still visible during the transition  between the login screen and display of the desktop wallpaper.

xhost +local: && sudo nano /etc/alternatives/gdm3.css

### Change from:
# lockDialogGroup {
#  background: #2c001e url(resource:///org/gnome/shell/theme/noise-texture.png);
#  background-repeat: repeat; }
#######################################

### Change to:
# lockDialogGroup {
#  background: #000000 url(file:///usr/share/backgrounds/solid_black_1920_1080.png);
#  background-repeat: repeat; }

Changing  purple (#2c001e) to black (#000000) changes login screen to black.  However, gdm does not respect file:///usr/share/backgrounds/solid_black_1920_1080.png. As a result, resource:///org/gnome/shell/theme/noise-texture.png  is displayed during the transition between the login screen and display  of desktop wallpaper. Since gdm uses gresouce (binary), it appears it  is necessary to extract gresource, edit, and repackage the gresource to  change noise-texture.png.

[https://bbs.archlinux.org/viewtopic.php?id=197036](https://bbs.archlinux.org/viewtopic.php?id=197036)

### Post by again? on 2018-02-01
Perhaps install lightdm with lightdm-gtk-greeter.
This is what I'm using now in xubuntu 17.10.
Much easier to configure using lightdm-gtk-greeter-settings.
```
sudo apt install lightdm lightdm-gtk-greeter lightdm-gtk-greeter-settings
```

If you don't get the package configuration dialog during install run...
```
sudo dpkg-reconfigure lightdm
```
