### Post by oldfred on 2010-07-08
Did you update Vista or just install 7 into the Vista partition. Windows combines its boot files into the windows partition that is the active partition (boot flag). The other windows partition is not separately bootable as the files are missing. If you installed 7 over Vista you may have deleted the XP boot files.

To get each MS to have its own boot loader make a partition and set its boot flag on, then install the 2nd product in it. Multibooters, Pictures here worth 1000+ words

### Post by oldfred on 2010-07-08
We really need the boot info script. And with that many systems & drives you should also learn about the script. It will tell you what boot loader is in each drive and how it boots. It will show files  essential for booting and after learning you can see what is missing.

If you had a windows installed then the second windows will combine boots. Grub will then fail in trying to boot the second one as its boot files are not there. Typically only one windows boots and then windows boot.ini (XP) or BCD (7 or Vista) have a menu showing the systems you can load.

### Post by oldfred on 2010-07-10
If you are committed to windows wubi is a way to test Ubuntu. It installs into the NTFS partition of windows and uses the windows boot loader. It is like a program and can be uninstalled easily.

If you plan on using Ubuntu then a full install to a partition gives more flexibility. It is also more reliable as it has its own files system. With Wubi it is inside the NTFS even though it creates a file inside windows.

I do not know wubi as I have always installed to a partition.
[https://wiki.ubuntu.com/WubiGuide](https://wiki.ubuntu.com/WubiGuide)
[https://help.ubuntu.com/community/Wubi](https://help.ubuntu.com/community/Wubi)

[http://howsoftwareisbuilt.com/2009/03/12/interview-with-agostino-russo-wubi-ubuntu/](http://howsoftwareisbuilt.com/2009/03/12/interview-with-agostino-russo-wubi-ubuntu/)
Agostino: Wubi actually wasn’t designed to do long-term installations. The main aim was really to let people try out Ubuntu with confidence. Normally, users that start with Wubi tend to upgrade to a full installation to a dedicated partition at the next release cycle.

If you do a full install it will overwrite the windows boot loader in the MBR but then give you a choice to boot Ubuntu or windows. Only one windows will work but that copy of windows will give you a choice (the same choices of windows you now have).

### Post by oldfred on 2010-07-11
If you have separate drives that is even better. Leave the windows boot loader in the windows drive. When you install Ubuntu to the 160Gb drive use the advanced button to install grub to the 160GB drive. Then in BIOS set the 160GB drive as the boot drive. It will let you boot Ubuntu or windows. If you want or the 160GB stops working you still can directly boot the windows drive thru changes in BIOS.
