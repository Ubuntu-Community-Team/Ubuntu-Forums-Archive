Have Ubuntu developers ever heard of regression testing?

Every time I get an update it is always an exciting adventure to see what doesn't work after the update. 

The latest victim of an update? The num lock key. I now can't use my number pad on my keyboard.

My speakers (laptop) work sometimes but not others (although they have been non functional for quite a few weeks now). My dock headphone jack works sometimes and not others too (it has been non-functional for quite a while now too).

The little green man that lets me shutdown works about 25% of the time the other 75% of the time clicking him locks up the taskbar. The only way to shutdown is to use gnome do to get a terminal and type "sudo shutdown -h now". 

If I do fresh install from my 8.04 disk everything works great (including speakers and headphone jack) as soon as I get some updates it breaks.

Can't upgrade to 8.10 because of the X-Server bug dealing with Java on some Intel chipsets.

All in all my Ubuntu experience is pretty poor. The only reason I still use it is because even with all of its faults it is still a more productive experience than trying to use Windows for development.
