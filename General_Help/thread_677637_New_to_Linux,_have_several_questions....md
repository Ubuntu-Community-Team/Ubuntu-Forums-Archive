### Post by matthewcraig on 2008-01-25
Hey there, glad you are considering Ubuntu for your next computer.  First, do you realize there are vendors who sell their computers with Ubuntu already installed?  Then, when you call for technical support on your purchase, they will support your Ubuntu software as an operating system.  This is worth paying extra, in my opinion.  There are a good number of vendors here in the USA who sell Ubuntu computers, but I am cannot make any recommendations without knowing where you live.

As for your questions, Xubuntu is a lightweight user interface, but the core programs are mostly the same as Ubuntu.  Your new computer will be more than capable of running the more demanding user interface, but it is every possible to install the Xubuntu interface right on your Ubuntu installation.  You don't have to choose one or the other.

Games?  You'll find lots of Free Software games built into Ubuntu.  More are available in the official Ubuntu software repositories, so you can just click on one to download and install it.   If you want the commercial titles, then there are many companies that sell titles that run on Ubuntu.  But, Ubuntu is not Windows, and it will not immediately run programs that are written for versions of the Windows operating system.  There are some options you have with WINE and DOSBox that may or may not help you run a Windows program.

The VT chipset features do work with Linux, especially since Linux is used so often in data centers to run virtual machines.

I run a 64-bit version of Ubuntu, and I find all the Free Software programs work very well and every 32-bit Ubuntu program works.  You will notice a couple proprietary programs failing to support 64-bit, Adobe's Flash Player being the most obvious.  There are work arounds that you can read about on these forums.  I think most people will tell you to go with the 32-bit version, unless you are getting more than 3 GB of memory.  For me, I like using the 64-bit features that I purchased.

Doing some research on Google will give you an idea whether your hardware will be supported or not.  Most hardware devices, especially ones a few years mature, will be supported.  Some have never had drivers written for them, but the Linux kernel supports more hardware devices than any other operating system.  Odds are good your hardware will be supported, and they will not require the manufactures' drivers.

You might try to buy a computer without an operating system, but why not buy a computer with the operating system you plan to use?  You'll know all the hardware you purchase will work with the operating system, and, more importantly, they will support their hardware if it ever fails to function properly with Ubuntu.

So, yes, you can recompile your kernel, but (ducks from people who will throw stones at me for saying it) you really shouldn't do so with Ubuntu.  The best part of Ubuntu is that it is all configured in a way where you are not required nor expected to do such technical tasks.  You can always try a different Linux operating system later, on your computer, which is more suited for tinkering.  Ubuntu is unique in that it does not require nor even recommends your login with system administrator access.  This will be different from your Windows experiences where you stay logged in with that access by default.

Sounds like you are really interested in computing, so I suspect that you will really enjoy the Ubuntu Linux experience.  I hope you will give Linux a chance to grow on you.  Remember, it took you years to understand Windows, so acknowledge you will need several months to really understand the Linux way of things.  Don't get me wrong, you'll be using basic functions in minutes.  In fact, download and burn an Ubuntu CD-ROM and you can boot to it on your existing system and start to use it.  You can run Ubuntu right on the CD-ROM without making changes to your computer.  Have fun with Ubuntu!

I don't know why the poster said that. Obviously, that poster doesn't play games on the computer. I am not a computer gamer myself either. There are some games for Linux, some Windows games work in Wine, and some will work in Windows in a VM. BUT Linux != Windows. Don't expect to run native Linux programs in Windows and don't expect to run native Windows programs in Linux. As a note, Linux software typically runs in Windows, because it is open source, and Windows programs can work in several emulators in Linux. You wouldn't expect a PS3 game to work in an XBox would you?

Good news, and bad news. It is likely that Ubuntu will be by default good enough, and only require a few extra steps to get the software you want, and, you rarely have to reboot, even when installing updates and new hardware, unlike Windows. However, running Windows programs that use a hidden API called "DirectX" in a software emulator means certain trade offs. One is, you lose the function of advanced features. Linux usually uses OpenGL for 3d, and it is very good, as good as DirectX. Remember: Linux != Windows. 

It isn't that bad. If you have the video capabilities and RAM, running Windows applications in Linux ranges from perfect to sketchy. Unfortunetly, I do not have much experiene in this. Running other OS's in a VM usually gets you good performance, if you have enough RAM.

Performance wise, you can get everything from near native to low. I have 2.5 GB of RAM, and a dual core 64 bit CPU, and running live disks, VM's and other often memory intensive tasks in Ubuntu gets me good performance. It ranges according to what you are doing and what you have hardware wise.

### Post by matthewcraig on 2008-01-25
Hey, sorry that this will sound negative, but figured I would save you some frustration.

You seem to be going into this process with the concept that if Ubuntu doesn't do everything that your Windows will do, then it's not for you.  You have to understand that Ubuntu is not Windows.  It's a different operating system, like Mac.  Not everything that runs on Windows runs on Mac.  If you started using a Mac thinking it should be exactly like Windows, then of course you will be disappointed and discouraged.

You're talking about integrating Ubuntu with proprietary, closed-source, restrictive software and hardware.  This is definitely not the Linux's strong points.  It is an operating system that trying to bring Free Software solutions to people, instead.

Start with what it does do and move up from there.  The big things are that it is Free Software, open source software, and no-cost software.  Plan in your mind that you can do many of the basic computing tasks and everything else is a bonus.  You'll be pleasantly surprised at how many things Ubuntu does that you won't expect.

If you start off by setting the bar at: Will this do everything my Windows can do and more?, then I can assure you that you will be disappointed.  Take a look at the Testimonials & Experiences forum here.  It's full of disillusioned people who thought they were getting a no-cost Windows operating system with Ubuntu.

You'll need to get past the fact that Ubuntu is not Windows.  Plan for what all the great things Ubuntu can do... rather than reluctantly accepting what it can't do.
