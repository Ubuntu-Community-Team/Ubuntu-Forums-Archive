### Post by bigbrovar on 2008-10-14
NU nano 2.0.7 Fil: /etc/fstab
ok what we can do is reinstall ntfs-config but before we do lets take out the the ntfs line from ur fstab
before we do anything lets create a back of the fstab as it is so if the heavens fall u can always revert back 
```
sudo cp /etc/fstab /etc/fstab.bak
```
now ```
 gksu gedit /etc/fstab
```
 comment out the last line by adding # infront of it to make it look like this 

```
#/dev/sda1 /media/ntfs ntfs-3g defaults,locale=en_US.utf8 0 0
```
when you are done reinstall ntfsconfig and give your mount point a name.give it read write and lets see what happens 



# /etc/fstab: static file system information.
#
# -- This file has been automaticly generated by ntfs-config --
#
# <file system> <mount point> <type> <options> <dump> <pass>

proc /proc proc defaults 0 0
# Entry for /dev/sdb1 :
UUID=1032646e-50c3-426a-8dff-e8ad2dca4d97 / ext3 relatime,errors=remount-ro 0 1
# Entry for /dev/sdb5 :
UUID=bf57a4c6-ab45-46d5-af65-8547ad857c87 none swap sw 0 0
/dev/scd0 /media/cdrom0 udf,iso9660 user,noauto,exec,utf8 0 0
/dev/sdc1 /media/fat32 vfat defaults,user,dmask=027,fmask=137 0 0
/dev/sda1 /media/ntfs ntfs-3g defaults,locale=en_US.utf8 0 0
