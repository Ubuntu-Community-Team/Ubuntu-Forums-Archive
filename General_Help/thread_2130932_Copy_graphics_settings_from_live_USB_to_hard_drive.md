### Post by sudodus on 2013-03-31
> **TheGuyWithTheFace said:**
> I've looked at those threads and the links they link too, and every time I try to use xrandr to change the size to 1024x768, it complains that it cannot find the mode, whatever that is. I tried the addmode and newmode commands, but both of those fail. I think that ties in somehow to the display settings having no resolution options other than 640x480, but how can I add a bigger resolution?

This is the crucial code
```
cvt 1680 1050
[COLOR=#0000ff] xrandr --newmode "1680x1050_60.00"  146.25  1680 1784 1960 2240  1050 1053 1059 1089 -hsync +vsync[/COLOR]
xrandr --addmode VGA1 "1680x1050_60.00"
xrandr --output VGA1 --mode  "1680x1050_60.00" 

```
And I think you can use your output from the live session to make an xrandr --newmode command line, which was made via cvt by [I]                                                                                     richardjl.
[/I]```
[COLOR=#ff0000]"1024x768" 65.00 1024 1048 1184 1344 768 771 777 806[/COLOR]
```
so maybe something like this would work for you
```

[COLOR=#0000ff]xrandr --newmode [/COLOR][COLOR=#ff0000]"1024x768[/COLOR][COLOR=#ff0000]_65.00" 65.00 1024 1048 1184 1344 768 771 777 806[/COLOR] [COLOR=#0000ff]-hsync +vsync[/COLOR]
xrandr --addmode VGA1 "1024x768_65.00"
xrandr --output VGA1 --mode "1024x768_65.00"

```

An alternative is to run ```
cvt 1024 768
``` to get the string for 'newmode'.

If this won't work, I think different graphics drivers are used in the live and installed systems, or at least some different settings, and I don't know how to change it in the installed system.

### Post by sudodus on 2013-03-31
Yes, let us use the output of cvt, since the line from the live session was not accepted. (It is actually the same as I get in my computer.)

```

cvt 1024 768
# 1024x768 59.92Hz (CVT 0.79M3) hsync: 47.82 kHz; pclk: 63.50 MHz
[COLOR=#0000ff] Modeline "1024x768_60.00"   63.50 1024 1072 1176 1328   768 771 775 798 -hsync + vsync[/COLOR]

```
So try the following three lines
```

xrandr --newmode "1024x768_60.00"   63.50 1024 1072 1176 1328   768 771 775 798 -hsync + vsync
xrandr --addmode VGA1 "1024x768_60.00"
xrandr --output VGA1 --mode "1024x768_60.00":

```
