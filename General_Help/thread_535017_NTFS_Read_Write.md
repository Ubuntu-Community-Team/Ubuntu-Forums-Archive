### Post by merlinus on 2007-08-26
You can use ext3, and install fs-driver so windows can read and write to it.

[http://www.fs-driver.org/](http://www.fs-driver.org/)

Installing the linux driver for ntfs is very easy, if you know how to open a terminal (Applications/Accessories) and type a few words.  Even better, simply copy-and-paste.

```

sudo apt-get install ntfs-config

```

then

```

sudo ntfs-config

```

Tick the appropriate boxes, and restart.

-merlin

If it is a non-vista formatted NTFS partition, than it can be repaired from linux using the ntfs-fix command.

If it is a Vista formatted NTFS drive it can't be fixed this way because Microsoft has made minor variations to the NTFS filesystem spec to squeeze more performance out of the FS and to make it difficult for non-approved projects (anyone who hasn't licsensed the full specs and signed the NDA for them) to interact with the FS on this level.

Just create the partition using Gparted from inside linux and you'll be okay.

### Post by somedamfool on 2007-09-05
Try a simpler statement without the uuid:
/dev/hda1	/media/hda1          ntfs-3g	silent,umask=0002,utf8=true
This worked for me both in Fedora and Kubuntu. This won't give you write permission, but you can get it (and a lot of other privileges) by adding yourself to group root. Also try mounting them in /mnt/ instead of media. media is controlled by Hal, and he's very particular about who he lets in.
