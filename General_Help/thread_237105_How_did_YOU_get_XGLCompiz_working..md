### Post by peabody on 2006-08-16
Radeon 9000 64MB  on an Athlon Thunderbird 1Ghz with 512 MB PC2100 DDR RAM with a funky Asus MB running the ALi Magick chipset.

I'm running the fglrx drivers version 8.27.10

I'm also using the quinn packages for compiz and company.

[Here's](http://www.ubuntuforums.org/showpost.php?p=1350423&postcount=368) a post I've made previously.

The one thing I truly needed to get it working was a previous version of the Xgl server, not the latest from quinn and company (the CVS merge).  Compiz wouldn't launch and stick unless I tried the startcompiz script again, and again and again and again until finally it worked.  This older version of the packages fixed that.

I also needed to use the LD_PRELOAD hack with the mesa OpenGL libraries.

I'm still have problems I'd like fixed.  Running any type of OpenGL anything from within Xgl seems to crash it (which is apparently a problem with the fglrx drivers with Xgl).  This means I can't get any accelerated video within Xgl, though I plan to use [this](http://www.compiz.net/viewtopic.php?pid=11827#p11827) guide to get around that.

Xgl/compiz is cool, but the setup right now is so incredibly painful.  I hope things become smoother in the future.

Notice I had originally got the "Extensions" bit at the end, but Flash doesn't like this (kills Firefox) so I've taken it out.


My gdm.conf-custom:

```
[daemon]

[security]

[xdmcp]

[gui]

[greeter]
Browser=true

[chooser]

[debug]

[servers]
0=Xgl 

[server-Xgl]
name=Xgl server 
#command=/usr/bin/Xgl :0 -fullscreen -ac -accel glx:pbuffer -accel xv:fbo 
#flexible=true

command=/usr/bin/Xgl :0 -fullscreen -ac -accel glx:pbuffer -accel xv:pbuffer 
flexible=true

```

And finally, I start it all on login witha script I've butchered together from various places, originally based upon "thefuture" (anyone who's read the XGL How-to will understand this):

```
#!/bin/bash
killall cgwd
killall compiz
killall gnome-window-decorator
killall gnome-panel
wait
cgwd --replace &  
compiz --replace gconf decoration wobbly fade minimize cube rotate zoom scale move resize place switcher &
xmodmap /usr/share/xmodmap/xmodmap.uk -e "keycode 22 = BackSpace"

```

Hope that helps someone.

### Post by Dinerty on 2006-08-23
Tried about 4 guides and none of them seem to work for me, so I found these 3 VERY HELPFUL guides below and did them in this order:

1. [https://help.ubuntu.com/community/CompositeManager/Xgl](https://help.ubuntu.com/community/CompositeManager/Xgl)

2. [https://help.ubuntu.com/community/CompositeManager/InstallingCompiz](https://help.ubuntu.com/community/CompositeManager/InstallingCompiz)

3. [http://www.ubuntuforums.org/showthread.php?t=222034](http://www.ubuntuforums.org/showthread.php?t=222034) because I had this error 



  [I]"You need to type sudo gedit /etc/X11/xorg.conf in a terminal and look for your "Screen Section" which should look something like this:

Code:

Section "Screen" Identifier "Screen1" Device "Device1" Monitor "Monitor1" DefaultDepth 24 SubSection "Display" Depth 24 Modes "1600x1200" "1280x1024" "1024x768" "800x600" "640x480" EndSubSection EndSection


You need make sure that DefaultDepth is set to 24
Also make sure that the SubSection "Display" Depth is not set to 32 (it may have several depths from 1-24, so long as the only Depth set is not higher than 24 you can leave this SubSection unedited.)

Save the file and restart Xgl by doing ctrl+alt+backspace and logging back in. Make sure you recheck the Compiz Themer to confirm that you have selected a theme and set the default bar layout in the "General" tab.

If you are still unable to see the window decorations, please come to the #Xgl irc channel on irc.freenode.net and ask for assistance."[/I]
