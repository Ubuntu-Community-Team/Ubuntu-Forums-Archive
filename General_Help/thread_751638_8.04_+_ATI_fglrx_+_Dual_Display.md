### Post by mag911 on 2008-05-01
In answer to the question, yes, I now have 8.04 + fglrx + dual monitors working on my T42p Thinkpad ... took a little bit of doing though :)

From my not so good memory:

1) Fresh install of Ubuntu 8.04
2) Install libstdc++5 using Synaptic (needed for ATI Catalyst Control Center.
3) Enable Restricted Driver + reboot.
4) Install ATI driver 8.44 downloaded from AMD website + reboot.
5) Enable Restricted Driver (the driver name should now read ATI accelerated graphics driver) + reboot.
6) Enjoy the White screen of death. Reboot using the recovery mode, choosing to fix the X server, then continue booting.
7) Enable Restricted Driver (again) + reboot.
8. Finally enjoy Ubuntu 8.04 + ATI driver! (whew)
9) Launch glxgears before attempting to launch the ATI Catalyst Control Center. Don't ask me why, it's just something I stumbled upon in my testing. Must be a bug?
10) The ATI Catalyst Control Centre confirms the correct driver version.
11) Launch aticonfig from the command line, using the following command: sudo aticonfig --dtop=horizontal
12) Reboot
8) Enjoy Ubuntu 8.04 + ATI + Dual monitors.

For the record, my command line was sudo aticonfig --dtop=horizontal,reverse
This is because my second monitor is to the left of my laptop.

### Post by amedornbirn on 2008-05-09
My system is:
newly installed Ubuntu 8.04 + ATI Radeon card + 2 monitors (1 for digital connector and 1 for analog connector)

Here is a summary what I've done (quite a newbie description cause I always forget all this menu names and simple stuff :)).

1) installed via Aplications/Add/Remove... ATI binary X.Org driver and ATI Catalyst Control Center

2) installed CompizConfig Settings Manager using 
sudo apt-get install compizconfig-settings-manager 

3) reboot

4) opened System/Preferences/Advanced Desktop Effects Settings
selected General Options, then Desktop Size tab and set Horizontal Virtual Size slider to 4.

5) opened Aplications/Accessories/Terminal and typed:
sudo aticonfig --dtop=horizontal,reverse
(reverse means that screen 1 should be to the left of the screen 0, use --dtop=horizontal if you want screen 1 to the right of screen 0)

6) opened System/Preferences/Screen Resolution and selected 2560x1024 instead of 1280x1024.
(If you have different default resoltuion, you may select another desktop resolution, just the wide should by two times larger. The 2560x1024 resolution appeared 'automatically', that is, it wasn't available from the beginning and appeared there due to some setup step, don't know which one :( )

7) reboot, and that's it

You may need also to reboot between steps 4-5, if it didn't work in the way that I described.

You may want to use some additional options, you can get pretty much info about them using:
aticonfig --help

Thanks to ppl in this thread for help!

### Post by aouie on 2008-07-08
I have an acceptable dual head setup good enough to watch movies (except for no Vsync as yet (resulting in tearing), TexturedVideoSync isn't doing what it should??), play / develop OpenGL apps on Screen 0. I tried numerous things to get a dual head setup to work and hence I am not sure if there are somethings that I am missing in the below steps.

At some point I had installed libstdc++5.
Installing the ATI drivers using the automated default install.

Ubuntu 8.04 + dual-head + catalyst 8.6 drivers + compiz = white screen bug.

Remove compiz

Ubuntu 8.04 + dual-head + catalyst 8.6 = hang on reboot.
Also, anytime I ran fglrxinfo or glxinfo the programs wouldnot exit normally (I had to use CTRL+C). Also, some terminated apps using opengl would be left eating up (lots of) CPU cycles (as seen in the system monitor).

I had to go over to Catalyst 8.5 drivers. With 8.5 you can't have any Opengl apps on the second screen. Also, there are still some screen access issues that will amongst other things make Totem and Ubuntu crash (but you can CTRL+ALT+F1 to a shell and restart the computer (or maybe simply restart gdm??)). But, things are good enough. I have a dual head setup, with the CRT being the primary and am able to run OpenGL apps (played a lot Nexuiz) on Screen 0, and use VLC on all screens (with X11 video output)). Able to logout, reboot, shutdown, suspend and hibernate without issues.

To do so what you probably need to do is :
Maybe install libstdc++5 (don't know if this step was necessary).
Uninstall all compiz components (since compiz will try to use Opengl on both screens).
Install ATI catalyst 8.5 using automatic install options.
Backup /etc/X11/xorg.conf and then run aticonfig
```
sudo aticonfig --initial=dual-head -f --screen-layout=left --sync-vsync=on --vrefresh=0,85-85 --vrefresh=1,85-85 --resolution=1,1280x1024 --resolution=0,1280x1024 --overlay-type=Xv --overlay-on=1 --max-gart-size=196 --sync-video=on
``` --swap-screens=on
Add the following to xorg.conf (see attached xorg.conf)
	Option      "TexturedVideo" "on"
	Option	    "TexturedVideoSync" "on"
Restart the computer.

Enjoy your dual-head setup (Hopefully AMD/ATI will fix related issues in the next driver). Also, it would be a good idea to keep an eye on [RadeonHD]("http://www.x.org/wiki/radeonhd").
