### Post by kerryhall on 2011-08-16
It is necessary or helpful to keep MD5 hashes of (generally static) files and verify them every so often? 

I would imagine that most file systems verify my data in one way or another, but it seems like a good idea to verify them myself, or is that just not needed?

---

### Post by cprofitt on 2011-08-16
I think this is an occasion which begs for a 'balance' to be arrived at.

If you have a high risk system and some high value files -- it sounds like good practice. Several security programs use the practice.

If this is your desktop and has limited exposure to the 'outside' world and your files are you pictures... then the effort requires is likely not worth it.

---

### Post by AlphaLexman on 2011-08-16
IMO, I only check the md5sum of files I download to test the integrity of the download process. 
Local files may change from editing or updates, so the md5sum would also change. No need to build a database of changes to local md5sums!
