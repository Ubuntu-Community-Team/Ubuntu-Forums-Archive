### Post by HalPomeranz on 2008-06-18
Huh, that's weird.  I never noticed that behavior of the file command on my 64bit machine, but that's definitely a bug that should be filed against the OS.

Anyway, you can use gdb to find out which program dumped the core file:

```

# **gdb -c core**

(gdb) 

```
