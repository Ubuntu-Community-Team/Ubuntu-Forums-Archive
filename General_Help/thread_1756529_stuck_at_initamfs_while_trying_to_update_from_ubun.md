### Post by coffeecat on 2011-05-13
Hi, pgonzamailcn.

I agree with Rubi1200. Looking at your boot script output, I notice that there is a blank by "Operating System:" in the sda1 information. It would normally say something like this (from my system):

```
sda6: _________________________________________________________________________

    File system:       ext4
    Boot sector type:  -
    Boot sector info:  
    Operating System:  Ubuntu 11.04
    Boot files/dirs:   /boot/grub/grub.cfg /etc/fstab /boot/grub/core.img
```Also, there is no mention of the /etc/fstab file in that part of the output (whereas you can see there is in mine) and no contents of /etc/fstab either which should  appear.

Finally, the only kernels the boot script could find are the old 10.10 ones. Unfortunately, your upgrade to 11.04 (it's 11.04 and not 11.01, by the way) has gone badly wrong.

The only option I can see is to backup your personal files using a live CD or live USB and then do a fresh install of either 10.10 or 11.04. If you want to install 11.04, I suggest you choose "try Ubuntu" from the live CD to make sure your hardware runs 11.04 before you attempt the install.

One warning about backing up data using the live CD. Because the UID of the live "ubuntu" user is different from the UID of your account on the hard drive, you will probably be denied access to your own home folder when using the live ubuntu account. You can get round this by using a root nautilus browser. Post back if you need any help with this.

### Post by coffeecat on 2011-05-13
> **pgonzamailcn said:**
>  I would appreciate if you could give me some instructions on how to proceed with this. Thanks so much!

You need a live CD, and an external USB drive to copy your files to.

Boot up the live CD and choose "try Ubuntu". Once you get to the live desktop, open a Nautilus window and find your sda1 partition in the left pane. Since the partition does not have a label it will probably say something like "<size in GB> filesystem". Click on that and you will see the root of your sda1 filesystem in the main pane. Open the home folder and then your account folder and see if you can access your files. If you can, plug in your USB drive and copy your files. If you get a permission denied, close the first window and open a terminal. Now:

```
gksudo nautilus
```

A nautilus window will open with root privileges showing the contents of the live session /root folder. Click on "File System" in the left pane. Open the /media folder and you should see a folder named "14ab1e25-845e-4d64-9a8d-305d163ce489". Open that and you will then see the filesystem of your damaged hard drive installation. Open the home folder and now your account folder and now you should be able to copy everything you need.

Good luck with that.
