### Post by newbie-user on 2011-10-12
Assuming your server is on the same subnet as the rest of the computers on the local network, it should be visible to the other computers.  

To check if the server is visible on the lan:
Type ifconfig on the server and get the server's ip address.  Take that ip address to another computer and type it in that other computer's browser.  If apache was installed correctly, then you should see apache's default webpage.  You could also ping or ssh to that ip address from another computer.

To access the server via hostname:
If you want to get to the server using a hostname, such as mysite.com, then you'd have to either edit the /etc/hosts file on the other computers to point to your server, or add your server to your local dns.

It seems like perhaps you're not familiar with IP addresses.  Apologies and no offense if you've got that down.  But if you are indeed unclear on the process, this site may help you understand a bit:
[http://computer.howstuffworks.com/router7.htm](http://computer.howstuffworks.com/router7.htm)

You have your IP address which is what your computer uses to tell the router how to get stuff to and from it.  You also have your logical IP addresses, which are the physical locations of the stuff on your network.  I'm new to the whole process myself, but it sounds to me like you might be confusing the two.

BTW, I would strongly recommend avoiding sharing your actual MAC and IP addresses with anyone.  Someone who knows what they're doing can do all sorts of nasty stuff with that info.
