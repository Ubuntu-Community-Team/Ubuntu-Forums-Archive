### Post by wilee-nilee on 2011-04-25
grub gui memory removal
sudo chmod -x /etc/grub.d/20_memtest86+

That should remove both memtest entries. You can always bring them back by running the first command again with +x instead of the -x, and running update-grub again. 

I assume that the bring back command will work, use at your own risk.;)

I assume where it is pointed is important as well.

You can test your memory from a live cd or several 3rd party bootable programs like gparted, many install or live cd's that are linux will have a memeory test I believe.
