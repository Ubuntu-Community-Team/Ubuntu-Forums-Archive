### Post by andrewthomas on 2011-03-11
> Grub 2 is installed in the MBR of /dev/sda and looks on the same drive in 
    partition #1 for /boot/grub.But Grub 2 should look in partition #1 of /dev/sdb for /boot/grub

You need to reinstall Grub2.
Go to 
[https://help.ubuntu.com/community/Grub2#Reinstalling%20GRUB%202]("https://help.ubuntu.com/community/Grub2#Reinstalling%20GRUB%202")
and follow the instructions for 
**SIMPLEST - Copy GRUB 2 Files from the LiveCD**

From a LiveCD:
After using 
```
sudo fdisk -l
```to make sure sda is windows and sdb is linux
```
sudo mkdir /mnt
sudo mount /dev/sdb1 /mnt
sudo grub-install --root-directory=/mnt /dev/sda
```then reboot into ubuntu and make sure to 
```
sudo update-grub
```

By the way, get rid of some of those old kernels.  You really only need one back-up to the current.

### Post by oldfred on 2011-03-13
Boot script also had the problem. And then I am sure the os-prober had the problem so it could not add your windows to the grub menu.

> Mounting failed:

Ubuntu often does not mount a NTFS partition if chkdsk is needed, but it is strange that both partitions are not mounting.

I would try chkdsk from a windows repair CD. Do you have files backed up. If you can see your data from the liveCD then I would make sure you have good backups.

chkdsk c: /r
You can use the following options:
/p Does an exhaustive check of the drive and corrects any errors.
/r Locates bad sectors and recovers readable information.
Note If you specify the /r option, the /p option is implied. When you specify the chkdsk command without arguments, the command checks the current drive with no options in effect. 
[http://www.microsoft.com/resources/documentation/windows/xp/all/proddocs/en-us/chkdsk.mspx?mfr=true](http://www.microsoft.com/resources/documentation/windows/xp/all/proddocs/en-us/chkdsk.mspx?mfr=true)
