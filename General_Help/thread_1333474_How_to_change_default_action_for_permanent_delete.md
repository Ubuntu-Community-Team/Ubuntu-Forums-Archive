Hello everyone.

Today I've stumbled to a very annoying problem when I was renaming many files from nautilus. I was typing a bit fast and replacing small letters by capital ones in filenames when I found myself pressing [Delete] to just delete a character while the [Shift] key was still pressed, so nautilus decided that I was not typing text but issuing the permanent delete command. As I was typing fast, the next key I pressed was [Return] and ooops! file deleted forever! It happened so fast that I don't even remember seeing the confirmation dialog!

Well, after the mandatory 5 seconds of astonishment, I decided that it was a very bad idea that the "Delete" action was the default one in the permanent delete dialog.

So _my question_ is: is there any way to make the "Cancel" action the default one for the permanent delete dialog box, so this doesn't happen again? 
Or do you think that detecting the [Shift]+[Delete] key combination while renaming a file should be considered a bug and be reported?

Answers, ideas and comments are welcome.

Thank you!
