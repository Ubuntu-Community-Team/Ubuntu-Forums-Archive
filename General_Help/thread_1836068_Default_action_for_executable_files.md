### Post by nerdy117 on 2011-08-30
I recently switched from Windows to Ubuntu, and I installed LAMP and was going to use Scribes for PHP development. After moving over all of my files, I changed Scribes to be the default editor for PHP files, but every time I try to open a file it tells me that it's an executable file and asks me what I'd like to do. Is there a way to make "display" a default option? Thanks

---

### Post by Enigmapond on 2011-08-30
You can change that by opening up Nautilus/Edit/Preferences & the Behaviour tab.

---

### Post by mcduck on 2011-08-30
Sure, you can configure that in the file manager's preferences.

Edit -> Preferences, and on the Behavior-tab set the "Executable Text Files"-option to "View executable text files when they are opened".

(on the other hand you shouldn't need the execute permission on any PHP file, so simply removing that would solve the problem as well)
