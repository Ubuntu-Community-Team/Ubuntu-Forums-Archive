### Post by doas777 on 2010-05-04
well, first thing to keep in mind is the differance between a runtime and an dev kit. in windows, you only use an SDK to write code. you use a JRE (java runtime environment) to actually execute code. there is also a web browser plugin that enables access to the jre.

yes, sunjava and icedtea can coexist, but the system will be configured to use only one of them (using 'update-alternatives') as default.  

check your browser plugins to make sure you have one for java, and i would point the alternatives to use the sun version.

### Post by doas777 on 2010-05-04
> **shuknjive said:**
> Can you detail how to configure Firefox to use sun-java as the default?  You mentioned 'update-alternatives', but unfortunately, I'm not clear on what that means or how to use it.

oh sorry. I was under the impression you had started here:
[https://help.ubuntu.com/community/Java](https://help.ubuntu.com/community/Java)

anyway, lets take it from the top.
```

sudo apt-get install sun-java6-bin sun-java6-jre sun-java6-jdk  sun-java6-plugin 

sudo update-java-alternatives -s java-6-sun 

```
this will install the sun jre, and casue ubuntu to use sun java as the default. after that you can run 
```
java -version
``` 

```
karmic:~$ java -version
java version "1.6.0_15"
Java(TM) SE Runtime Environment (build 1.6.0_15-b03)
Java HotSpot(TM) Client VM (build 14.1-b02, mixed mode, sharing)
```
to confirm that your system is pointing to the sun java.

hopefully that will do ya
