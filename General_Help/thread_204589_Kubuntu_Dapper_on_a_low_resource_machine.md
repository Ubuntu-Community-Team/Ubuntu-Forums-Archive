### Post by ozdevelop on 2006-06-27
Hi, I installed Kubuntu 6.06 (which I already use on my uber-desktop 3Ghz - 1GB RAM)  on an old 600Mhz laptop PC with only 128mb RAM as a last resort, as xubuntu was too  buggy and not really ready for the desktop any time soon.

What I found was a system that (considering the meagre memory spec) was snappy, responsive and simply a joy to use - provided I didn't become a uber-task running king. 

Only running with say, Kdevelop3 or even eclipse (with a small project loaded), combined with a terminal or two for command-line development tools - or one openoffice2 application running - the swap file was only using 20% with disk thrash eminently acceptable. System maintenance performance (usual KDE utilities) was good, in fact the whole user experience is pleasing.

The last time I tried a ubuntu variant on such a machine (using ubuntu 5.10) - the performance was horrible. I always thought KDE used more system resources than Gnome - so I am amazed that this system runs so well with kubuntu.

Has the code for kubuntu 6.06 been optimised for better memory - or am I just lucky?? Either way - dapper really does seem to be the BEST ubuntu version yet!!

:cool:
