### Post by chaosx9 on 2010-07-03
A good one that i use is called conky, but it takes a little bit of learning to get it to your liking. You can install it via synaptic, or apt-get. (The package name is conky)

If you want, I can e-mail you my config file to use.

EDIT: I should learn to read better. Install conky, then do the following: (i'm kinda a newb too)

Step 1: Open a terminal (Applications>Accessories>Terminal)

Don't be scared, embrace the terminal

Step 2: Use the following command to make a new file, and open it:

```
gedit .conkyrc
```

Step 3: In that file, copy and paste the following (which is my config file):

Now, you have conky setup, but if you login, you'll notice the transparency might mess up. So there's a simple way to solve it.

Go into preferences>startup applications
Look for an entry called "conky", and if there isn't one, then click "Add" then name it Conky. If the entry is there, click on it, then click edit

Change the "command" to the following:

```
conky --pause=20
```

This'll make it pause for 20 seconds while starting up. Make sure the entry is checked, and it'll startup well.

### Post by rollin on 2010-07-04
> **Cpierce said:**
> I do have one issue, with the above configuration, conky stays above the windows. So if I open a browser, conky is covering part of the webpage.

You could try changing: 

```
# Window
own_window yes
own_window_class Conky
own_window_transparent yes
own_window_type _override_

```
To

```
# Window
own_window yes
own_window_class Conky
own_window_transparent yes
own_window_type _desktop_ 

```

That has worked for me a few times! Post back if you get any problems and good luck with it, I'm sure you'll soon be doing loads of awesome stuff with conky, it's great!!

Failing that you could try "normal" instead of "desktop"...
