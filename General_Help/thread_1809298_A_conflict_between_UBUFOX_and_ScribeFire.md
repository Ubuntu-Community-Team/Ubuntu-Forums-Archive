This is my very first post after a very very long lay off. I mad some changes to my blogg the template and the name here thought the old one was bit too long and not that catchy. Any way I found some thing very funny today I'm using 2 Linux os in my desktop and in my netbook one is Ubuntu the other is LinuxMint (love this distro.) in both of them I use FireFox4(FF) as my default browser with my usual FF extensions.

        I may be not the biggest blogger in the world but I use ScribeFire because I just love it and very handy when it comes to quick post and it was rock solid in stability unlike most add-ons out there. But I started to face some issues with the newest release of the FireFox family the version 4 ok initially this was little late to get upgraded with compatibility but after the fix was released I was facing a very strange issue the funny thing was this issue was not to be found in my LinuxMint netbook along with the same FF version installed.

       Then I wanted some answers so I dialed my best friend Mr.Google but he was unable to say a ting to me so I embarked to find the problem my self and tried everything I knew to find the issue. Nothing came up and I was getting frustrated when I saw the Ubuntu default extension that get installed automatically with the system this is known as Ubuntu FireFox Modifications just to see what will happen I disabled this and boom ScribFire was functioning the way it should be.

         This extension was installed by two packages that get installed when we install the system,

          1. ubufox

          2. xul-ext-ubufox

You could keep the extension disabled or you can remove it by removing theres no harm do to the FF or to the usability of the software. To remove the 2 packages clos FF and go to System -> Administration -> Synaptic Package Manager and search for those two packages and mark them to be completely removed thats it no more issues with your favorite blog editor.

NOTE: Just want to inform you guys about this.
