### Post by Archimedes0212 on 2009-03-28
Do you have any obscurely named folders on your external hard drive.  I had a problem similar to this some time back.  My boot sequence was set up so that it would load from my external first.  It just so happened that I had a couple of folders on my external named exactly like Windows system folders (ie: WINDOWS).  Once I renamed these folders my problem went away.

### Post by Bucky Ball on 2009-03-28
Have a look in the BIOS and make sure the external drive is not set as any boot device, just your internal and the cd.

You say only Windows won't boot with the external plugged in? The common one is having a dongle plugged in while booting Ubuntu, tends to confuse it.

I have the same problem with my external drive on my desktop and have always had it even before I dual-booted this machine with Ubuntu. I only switch on that drive when I need to access it anyway so it is in no way an annoyance for me. Do you need to have it on all the time and waste that energy? Is it for data backup or you are using programs from it? You could save yourself a few bucks and leave it off until you need it.

An easy solution would be to have it off at boot then flick it on once your up. You don't need to unplug. :)
