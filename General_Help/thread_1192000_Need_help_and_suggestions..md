### Post by t0p on 2009-06-19
Just because your family can use Windows okay, doesn't mean they can use Ubuntu okay.  Ubuntu and Windows are very different operating systems.  There *are* similarities, but there are also quite a few differences.

There *is* a way to try Ubuntu before you install.  Download the installation cd image, burn it to a disk.  Now alter your BIOS options (if necessary) so your computer will boot from the cd drive.  Insert the disk, and you will be given the option to "try ubuntu without changing your machine".  This called a "live session".  The disk is known as a "live cd".  This means you can try ubuntu without changing partitions or installing to disk.  The system works slower when in a live session, but you'll get a good idea of what ubuntu is like.

### Post by t0p on 2009-06-22
Re: drivers

Some hardware manufacturers release drivers - nvidia is an example of a commonly used driver release - but "most" hardware has compatibility written into the kernel.  This means that "most" hardware will work "out of the box".

TBH, newer hardware is less likely to Just Work.  But if you have gear that's not bleeding-edge brand-new, and perhaps made by a "name" manufacturer, there's a good chance Linux will work fine, no drivers necessary.

Of course, some brand-new bleeding-edge kit will work fine, as will some no-name equipment.  I'm generalizing.  Which I'm told I'm always doing too much...
