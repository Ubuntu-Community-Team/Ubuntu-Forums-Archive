### Post by taurus on 2007-03-23
> **spankymasterc said:**
> i have the same problems... heres my results if u could help me 2

my /dev/hdb5 doesnt show its a 30 gig partiton made ext3 for extra space for linux but it doesnt show up and i look in media and i see Lost and Found with a red X

Okay, you're first.

Edit /etc/fstab

Applications -> Accessories -> Terminal
```
gksudo gedit /etc/fstab
```
and add this line to the end of it.

```
/dev/hdb5   /media/hdb5   ext3   defaults   0   1
```
Save the change.  Now, create a new mount point, mount it, and change the ownership from root to you, **tweeter**, so you can write to it.

```
sudo mkdir /media/hdb5
sudo mount -a
sudo chown -R **tweeter**:**tweeter** /media/hdb5
df -h
ls -la /media
```

### Post by taurus on 2007-03-23
> **kirtan said:**
> This is my results of  two commmands

Okay, you're next.

According to the output, your /dev/sda3, /dev/sda4, & /dev/sda6 are all vfat!  I guess you need to reformat /dev/sda3 to ext3 again from a terminal.

```
sudo umount /dev/sda3
sudo mke2fs -j /dev/sda3
```
Now, you need to modify your /etc/fstab since /dev/sda3 is not vfat anymore.

```
gksudo gedit /etc/fstab
```
Replace this line

```
UUID=4602-76A7 /media/sda3 vfat defaults,utf8,umask=007,gid=46 0 1
```
with this one.

```
/dev/sda3   /media/sda3   ext3   defaults   0   1
```
Save it and reboot.  Then, have a look at this output from a terminal.

```
df -h
```
