### Post by smokey_58au on 2009-10-23
I have an issue that is causing me some grief and I can't make any logical sense of it.  I am running Karmic 64bit on my HP laptop and it performs very well - it fixed a lot of issues from Jaunty and I am much happier with Karmic.  I use Evolution for emails (in the hope that one day they will make a connector for RPC over HTTP) and have 3 different mail accounts that I access via 3 different POP servers.  I use the same SMTP server for all 3.

I have been playing around with network connections a bit recently and now have an issue where I can't receive or send emails from Evolution (or any other mail client) from my laptop through my home router.  I can connect to the internet/Skype etc through it but no emails.  However (and here's where it gets confusing) I can send/receive using these same accounts with mail clients (including Evolution) on my home PC or wife's laptop through the same router (1 wireless connection, 1 ethernet, all auto dhcp).  So I 'think' I can rule out the router as the cause.

So if that thinking is right then my thinking is that it should be a setting on my PC and when I disconnect my wireless connection from the router and connect my mobile broadband modem, mail comes good in all clients including Evolution.  And when I take my laptop to work it connects to the wireless router there and emails work fine.  So I 'think' I can rule out Evolution or the other mail clients as the cause.

I have checked my wireless connection settings on my PC and they are the same as my wife's machine that works correctly, and as I said using this connection I have all other local and external connectivity.  I have deleted and re-added the connection just in case but still no good and if it was DNS related on my router then it would fail for any machine connecting through it surely?

This all was working up until a few days ago and I'm not sure what local settings I could have changed that would cause this.

_Here is my hosts file:_

127.0.1.1    robd-laptop

# The following lines are desirable for IPv6 capable hosts

fe00::0 ip6-localnet

ff02::1 ip6-allnodes
ff02::2 ip6-allrouters
ff02::3 ip6-allhosts

_And my hostname file:_

robd-laptop

_And my resolv.conf file:_

# Generated by NetworkManager
nameserver 192.168.1.1

And my dhclient.conf file is 'out of the box' - I haven't made any edits to it.

Can anyone tell me what might cause the connection on my local PC to fail for all my email clients yet work for all other networking?

### Post by jbrown96 on 2009-10-23
Maybe it's a firewall issue. You can check the status with ```
sudo ufw status
``` I don't think it is enabled by default, but perhaps something you did activated it. You can disable it with ```
sudo ufw disable
```
The only other thing that I can think of is a proxy setting, but that wouldn't make sense because internet browsing is working. 

You may want to try purging your mail programs and reinstalling them. Make sure you backup your config files. They are located in your home folder (they are hidden). Thunderbird would be in .mozilla/thunderbird. I would imagine that evolution either has it own .evolution or it might be buried in .gnome or something similar. You can just copy these folders to the desktop then purge them in Synaptic. I would try to recreate the accont settings to see if it is a config error before restoring your backupped settings. 

You may also want to take a look at network manager. If you right click on the network icon and go to edit connections. Take a look at the ipv4 setting (especially routes) and that ipv6 is set to ignore. Compare these settings to your mobile broadband settings.

Hello Rob;

a) No, if the other programs already had cached DNS entries before the problem arose, then they could potentially still work while the entries for the mail comms had already expired. Eventually everything would stop one the relevant DNS entry's time ran out.

b) Network Manager rewrites entries in files each time it establishes a connection (when you log in), it should delete old entries but sometimes it does not (I hate it and have reverted back to the "old" **gnome-network-admin** package). Check the IP ranges on the Ethernet connection of your router versus the Wireless side, then check your network with the following terminal commands:

```
netstat -rn
ifconfig
iwconfig
```

You can post the output here if you cannot see an obvious problem.
