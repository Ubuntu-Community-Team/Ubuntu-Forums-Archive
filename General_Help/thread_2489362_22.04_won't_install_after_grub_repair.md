### Post by oldfred on 2023-07-27
When Boot-Repair suggest to post link, please follow those instructions.
Easier to go back an forth from one tab to another than have to scroll all over place to review your report.

But first house clean all the duplicate UEFI entries. That may be part of issue.
Report says you have UEFI, but not Secure Boot which is unusual. Or is this a very early UEFI system from about 2011?
What brand/model system? What video card/chip?
Some early UEFI systems with too many UEFI boot entries would totally crash. They could only be repaired by sending back to vendor.
Orginally they blamed Linux but then Windows users had same issue. UEFI was only using 50% of available space and crashed.
So remove duplicate/repeated entries. They are in report or run this again to confirm numbers.
sudo efibootmgr -v
sudo efibootmgr -b XXXX -B # where XXXX is the entry you want to remove.


You also are using the old MBR(msdos) partitioning. Since 2012, gpt has been required by Microsoft for UEFI boot of Windows. Ubuntu has not required it, but should. But conversion from MBR to gpt totally erases a drive, so I think they did not want installer converting a BIOS install when user tried (incorrectly) to do a UEFI install on same drive.
Long term plan on converting to gpt. I started converting to gpt in 2010, so you are a year or two behind. :)
