### Post by Jimysbil on 2016-08-28
I don't know if this is helping you with your problem but according to [this]("http://www.dedoimedo.com/computers/linux-iphone-6.html") post by using the command
```
fusermount -u /media/iPhone/
```
should unmount your iphone.

Also the command
```
idevicepair unpair
```
is a separated command and it should unpair your device.

So in my option the syntax of this command
```
alias umip='fusermount -u /media/iPhone/'idevicepair unpair'
```
is incorrect because they are separated commands.
[COLOR="#FF0000"]* The command should work like this:[/COLOR]
```
alias umip='{something} | {something else}'
```
or
```
alias umip='{something} && {something else}'
```

[COLOR="#FF0000"]**NOT** like this:[/COLOR]
```
alias umip='{something}'{something else}'
```

So I think you should try:
```
alias umip='fusermount -u /media/iPhone/ && idevicepair unpair'
```

### Post by Keith_Helms on 2016-08-28
```

alias umip='fusermount -u /media/iPhone/'idevicepair unpair[COLOR=#ff0000]_**'**_[/COLOR]
# Add an "alert" alias for long running commands.  Use like so:
   sleep 10; alert


# Alias definitions.
# You may want to put all your additions into a separate file like
# ~/.bash_aliases, instead of adding them here directly.
# See /usr/share/doc/bash-doc/examples in the bash-doc package.

if [ -f ~/.bash_aliases ]; then
    . ~/.bash_aliases
fi

# enable programmable completion features (you don[COLOR=#ff0000]_**'**_[/COLOR]t need to enable
```

Your original problem is just what happens when an unmatched quote causes part of the script to "go off the rails".  The extra quote in the middle of 
```
alias umip='fusermount -u /media/iPhone/'idevicepair unpair'
```
ended that block of text so that the one at the end started a new block of text which ran until it hit the one in the middle of a comment
```
# enable programmable completion features (you don't need to enable
```
At that point, bash thought the words need, to, and enable were commands and it couldn't find tham.

### Post by cmcanulty on 2016-08-28
OK I solved this and now get non root access. What was needed was ifuse, so for others who need this here are the commands that work in 16.04. I have tried it several times. Thanks for the previous tips. Beforehand I edited /etc/fuse.conf with this 

```
# Allow non-root users to specify the allow_other or allow_root mount op$
user_allow_other be done first
```

then to pair iphone:

```
idevicepair pair
```

for mounting as non root:

```
ifuse /media/iPhone
```

for unpairing and unmounting:

```
'fusermount -u /media/iPhone/ && idevicepair unpair'
```

the aliases I set up in my /home/.bashrc file for the above commands are:

alias ipp='idevicepair pair'
alias mip='echo m |sudo -S ifuse /media/iPhone'
alias umip='fusermount -u /media/iPhone/ && idevicepair unpair'
