The problem is that when printing from evince (pdf viewer) there is no top margin. It keeps on shifting the page right up to the top.
This only happens on the odd pages, though even pages seem to have the problem to a lesser extent - and it may be worst on the first page, or just that the first (often title) line starts lower and so it gets tucked up more.
Printing webpages from firefox does this too I believe.

Trying alternative pdf viewers didn't help: Atril document viewer; printing pdf from firefox; and also qpdfview, where even setting the top margin in it's print properties did nothing.

Non of the print settings help:
- shrink to fit
- scaling down reduces the size, but still tucks the page to the top.
- unchecking   Select page size from document properties.
- checking/unchecking  Auto rotate and centre
- even setting up a custom page with a generous top margin.
- others I can't remember (sorry hard to keep track of all of them)

Printing a test page from system settings does print a good page though. It does use almost all of the page though, so there's only a little room to shift up, but there's a definite small gap. (Btw the bottom thick border is absent if it's meant to be there.)


Using Ubuntu 16.04, and the printer is Cannon MG5600.
