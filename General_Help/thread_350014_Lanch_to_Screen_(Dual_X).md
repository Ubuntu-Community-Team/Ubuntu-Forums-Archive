### Post by RAOF on 2007-01-31
> **russell.h said:**
> ...
Is there a way to launch a program to one monitor or the other? I'm sure I read something about this somewhere, something like
```
<command> :<screen number>
```
but that doesn't seem to be working.
...

You'd be thinking of```
DISPLAY=:*<screen number>* *<command>*
```

The reason being that apps read which display they need to use from an environment variable, DISPLAY.  That code sets DISPLAY to the number of your choice (just for this command), and then runs *command*.

Secondly, as far as I'm aware, you **can** have multiple monitors, each at a different resolution, with Xinerama or MergedFB or TwinView or whatever.  Can you drag apps from one screen to the other?  Because you should be able to.

### Post by russell.h on 2007-02-02
I think the reason I had to use a bash script was that the session manager thing wants a program to run, as opposed to a command to execute. For example if you press Alt+F2 and type in "DISPLAY=:0.1 gdesklets" it will give you an error about how it couldn't find that. To get around that all I did was create this script:
```
#!/bin/bash
sleep 3
DISPLAY=:0.1 gdesklets
```
and told the session manager to run it when I log in. I'm not sure if the sleep is really necessary, but I wasn't sure what would happen if it tried to launch the desklets before the desktop was even up. I don't really think that will solve your problem though, my desklets automatically load when I start gdesklets. I have version .35.3 for whatever that is worth.

Good luck getting it working, I'll be happy to help any way I can.

Edit: Also, it seems that KDE will let you set two different backgrounds (although its support for this particular setup seemed buggy at best), but thus far I have been unable to get gnome to let me do so.
