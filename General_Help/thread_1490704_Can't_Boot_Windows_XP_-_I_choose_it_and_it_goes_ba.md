### Post by angry_johnnie on 2010-05-22
:o

if you installed grub ON the windows partition, i think you killed windows. :(

you could use a windows cd to fix the broken install.
it will, in turn, kill grub.
you would then need to reinstall ubuntu (or just [reinstall grub]("http://www.ubuntu-inside.me/2009/06/howto-recover-grub2-after-windows.html")), and let it install grub to mbr.

if you don't want to install grub on mbr, you'd need to install it to it's own (ubuntu's) partition, and then use another tool to add ubuntu to the windows boot loader.

or, you could install via [wubi]("http://wubi-installer.org/").

### Post by louieb on 2010-05-22
What you did with Super GRUB was try to fix the MBR. Not what you want - don't confuse the MBR with the boot sector.  A hdd has one MBR and each partition has a boot sector.  

You need to fix the boot sector of sda1.  

Did you try booting to the recovery console of the XP install CD? Thats what you want.

the run the command 

```
fixboot c: 
```

Another option to fix is [TestDisk - CGSecurity]("http://www.cgsecurity.org/wiki/TestDisk") 

There wiki has a detailed how to use it [http://www.cgsecurity.org/wiki/TestDisk_Step_By_Step#NTFS_Boot_sector_recovery](http://www.cgsecurity.org/wiki/TestDisk_Step_By_Step#NTFS_Boot_sector_recovery)

### Post by grnorris on 2010-05-24
A password protected hard drive shouldn't stop you from doing anything on the computer the hard drive was originally installed in.

Now, I read a lot of the post prior to mine and noticed a lot of the same recommendations that almost killed my computer.  It may be to late but, you can try (From another site, didn't mark the source so I don't remember exactly where I got this):

sudo apt-get install testdisk
sudo testdisk /dev/sda

proceed
Intel
select windows partition /dev/sda1
choose boot
BackupBS
Y to confirm

quit

These instructions might not be perfect, I seem to remember having to a few more steps than were listed.  Also For me it was sda1 which is the Win7/Vista bootloader partition otherwise known as system reserved (Which you never want to delete).  XP would probably be installed to sda1 as well.  If you're unsure then while in Ubuntu find a your windows drive and take note of it's name.
