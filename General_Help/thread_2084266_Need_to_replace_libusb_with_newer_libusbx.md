### Post by within on 2012-12-08
To install libusx PROPERLY and without touching libusb, you need to do:

```
$ sudo su
$ ./configure && make && make install && sudo ldconfig
```

works like a charm. libusbx will be installed here: /usr/local

Then you can run FlashTool:

```
$ sudo su
$ ./FlashTool
```

ALL these are NOT from me, but from internet search, leading to these 2 helpful pages:
[http://duopetalflower.blogspot.fr/2012/08/flashtool-in-ubuntu-linux.html](http://duopetalflower.blogspot.fr/2012/08/flashtool-in-ubuntu-linux.html)
[http://stackoverflow.com/questions/4853389/how-to-install-libusb-in-ubuntu](http://stackoverflow.com/questions/4853389/how-to-install-libusb-in-ubuntu) (see Renaud's post down the page)

and it simply works.

Try to go again in libusbx-1.0.14 dir, and run the command again with su rights:
```
./configure && make && make install && sudo ldconfig
```

If it works fine, you should have no errors, if there are errors, it means something might be missing, such as gcc or make, or something else. watch out the message after running the above command to be sure it goes smooth until the very end!

I know it is not directly related to libusbx, but did you add the persistent rules as mentioned in post:
[http://duopetalflower.blogspot.fr/2012/08/flashtool-in-ubuntu-linux.html](http://duopetalflower.blogspot.fr/2012/08/flashtool-in-ubuntu-linux.html)

mine look like these (in file: etc/udev/rules.d/80-persistent-usb.rules):
```
# This file was automatically generated by the /lib/udev/write_net_rules
# program, run by the persistent-net-generator.rules rules file.
#
# You can modify it, as long as you keep each rule on a single
# line, and change only the value of the NAME= key.

# PCI device 0x14e4:/sys/devices/pci0000:00/0000:00:1c.5/0000:06:00.0 (tg3)
SUBSYSTEM=="net", ACTION=="add", DRIVERS=="?*", ATTR{address}=="f0:4d:a2:54:e8:26", ATTR{dev_id}=="0x0", ATTR{type}=="1", KERNEL=="eth*", NAME="eth0"

# PCI device 0x8086:/sys/devices/pci0000:00/0000:00:1c.1/0000:04:00.0 (iwlwifi)
SUBSYSTEM=="net", ACTION=="add", DRIVERS=="?*", ATTR{address}=="00:21:6a:c8:a0:9e", ATTR{dev_id}=="0x0", ATTR{type}=="1", KERNEL=="wlan*", NAME="wlan0"

# Xperia USB access for flashtool
SUBSYSTEM=="usb", ACTION=="add", SYSFS{idVendor}=="0fce", SYSFS{idProduct}=="*", MODE="0777"
```
 
As mentioned in the link above, I added the line below my comment:
# Xperia USB access for flashtool
