### Post by lovinglinux on 2010-04-18
Chrome/Chromium seems to have a pretty good method of keeping things stable, by running each tab as a separate process, but that hasn't prevented it from freezing with some flash sites here. Nevertheless, the browser itself doesn't crash in this situation and it provides a way to terminate the problematic content. 

Chrome/Chromium wasn't working for me until Lucid. Whenever I tried to load a page it simply froze and I had to kill it.

Firefox is pretty stable for me. Sometimes it crashes, but because of an extension that I don't want to get rid of. Nevertheless, I have 51 extensions installed and Firefox works like a charm.

Firefox will introduce a new feature soon that prevents the browser from crashing when loading problematic plugin content. It works like a charm and it's awesome (see [Lorentz](http://ubuntuforums.org/showthread.php?t=1450678))

Ironically, several users have been experiencing flash crashing issues recently with Firefox 3.6.4pre (Namoroka), due to this new Lorentz technology. Nevertheless, this seems to be fixed now and was only affecting this particular development version.

I have Opera installed, but never used it enough to make an opinion on stability. It isn't good for my taste.

If you are not satisfied with Firefox performance, see [Firefox optimization and troubleshooting thread](http://ubuntuforums.org/showthread.php?t=1193567)

### Post by Serpher on 2010-04-18
Install Lynx, press Ctrl + Alt + F2 (Ctrl + Alt + F7 to return to GUI), login, and type $lynx <url>

It's a text based browser, but doesn't process flash. If you don't really need flash and just need to be logged in, i'd just use that. Very stable.
