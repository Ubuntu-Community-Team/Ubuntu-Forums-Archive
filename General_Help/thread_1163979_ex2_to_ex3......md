### Post by jkarcz on 2009-05-19
The process is usually:

umount /dev/sdaX
tune2fs -j /dev/sdaX 

(where sdaX is the device in question)

Basically ext3 is ext2 with journaling turned on.

However, if you installed Ubuntu with ext2, you will either have to reinstall or possibly use a live CD and use the above commands in a terminal or run gparted.  The / partition is hard to unmount on a system while using it.

### Post by geirha on 2009-05-19
Ok, so /dev/sda2 is ext2 and mounted as / (the root of the filesystem). That means you shouldn't do this from your installed Ubuntu system. The safest is to boot up the Ubuntu CD. Once in the live session on the Ubuntu CD, Open up a terminal, and run ```
sudo fdisk -l
```
Double-check that /dev/sda2 is listed with "Linux" in the last column. Then you can add a journal to your ext2 filesystem, effectively making it ext3
```
sudo tune2fs -j /dev/sda2
```

That's it. Just one more thing you should do before you boot, is to edit your system's /etc/fstab to mount / as ext3 instead of ext2. (Note that ext3 is ext2 with journaling, and mounting an ext3 filesystem as ext2 is perfectly safe ... the journaling feature just won't be used)

Mount your Ubuntu system's root partition to /mnt, then edit /mnt/etc/fstab:
```
sudo mount /dev/sda2 /mnt
sudo gedit /mnt/etc/fstab

```

Change "ext2" to "ext3" on that one line, save the file and reboot into your system.
```
UUID=0aa80814-e2bb-4662-be3f-b107382f94a4 / [COLOR="Blue"]ext3[/COLOR] relatime,errors=remount-ro 0 1
```
