### Post by mb_webguy on 2009-04-29
> **creess1 said:**
> Wine needs to be heavily funded and working perfectly.
Linux isn't a free version of Windows.  Use Linux-native software alternatives.  Linux-native alternatives exist for almost every type of software you could possibly want, and they're free.

And hey, at least you can use some Windows software on Linux.  Try that on a Mac.  Or try running Mac software on Windows.

> **creess1 said:**
> Basic codecs were missing in the installation of the OS, a lot of streaming content wouldnt work OOtB. 
Those codecs you're talking about are most likely the proprietary ones that can't be included in Ubuntu for legal reasons.  It takes all of 2 minutes to add the Medibuntu repository and install the non-free-codecs package.  And once you're done, you can play any of the media you can play on Windows, unless it has DRM.  Compare that to the installation of numerous individual codecs or messy codec packs on Windows.  

> **creess1 said:**
> Drivers feel like their non existant.  While granted almost everything "just worked", its the little things that really make the difference.  I get these audio glitches in streaming audio playback on certain web radio sites.  Getting my mic to work in ventrilo was a nightmare, never could get it to work.   And then when i started looking for hardware that would be compatible with linux, i just felt so limited.  Nothing in socket 1366 is on the compatability hardware list yet.
And this is different from Windows *how*, exactly?  I've had *lots* of hardware that wouldn't work correctly with Windows over the years, despite it being the most widely used commercial OS.  And I find your claim that the "nothing in socket 1366" is supported by Linux a bit hard to believe, since the Core i7 is an x86_64 processor.  The i386 and AMD64 versions of Ubuntu should both run just fine on that processor.  I'm not saying you're wrong,  I just don't see why Linux (and therefore Ubuntu) wouldn't run on a particular x86_64 processor, when it runs just fine on every other x86_64 processor.

I'm sorry you didn't feel Ubuntu was for you, but considering your complaints, I think you might have had the wrong expectations going into it.  Linux isn't Windows.  It isn't trying to be Windows.  It doesn't *want* to be Windows.  It wants to be better than Windows.  I personally think it succeeds in quite a few areas -- security, ease of use (for new and experienced users, but not necessarily Windows converts full of misconceptions and bad habits), customization, and cost being a few that quickly come to mind.  It's impossible for something to be exactly like something else and bettwe than it at the same time.

Linux isn't Windows.  And if you're expecting it to be, then you'll be disappointed.  But Linux also isn't for everyone.  If Windows makes you happy, then use it and be happy.
