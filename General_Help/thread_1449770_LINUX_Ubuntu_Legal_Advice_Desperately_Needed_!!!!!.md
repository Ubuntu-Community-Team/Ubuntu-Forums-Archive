It's absolutely legal to sell and redistribute software with GPL license. All you need to do is to give credit to those who created the program, and provide the GPL license and the information that it's free software.  

You only need to worry about things if you modify the programs in any way (in which case selling and redistribution is still allowed, you just need to make the source code with your modifications available..)

### Post by 3rdalbum on 2010-04-08
This is perfectly fine to do. Just make sure you don't preinstall anything from the "ubuntu-restricted-extras" package, and don't preinstall anything from the Medibuntu repository; those items are "restricted distribution" in many parts of the world, possibly including yours.

It's even legal to sell GPL software; but note that, under the terms of the GPL, if you provide a copy of the software then you must also provide the source code if the end-user requests it. Some people would say that the responsibility falls on you, other people would say that you can just tell the user where to get the source code (in this case, using *apt-get source <packagename>* is a good way).

I think telling the user how to use *apt-get source* would put you into full compliance, but I Am Not A Lawyer and some people would disagree with me. If I was in your situation, I'd wait and see if anyone asked for the source; if they asked, then I'd tell them how to use *apt-get source*, and if that wasn't good enough for them I'd do *apt-get source* on my own computer and e-mail the result to them.

### Post by hawthornso23 on 2010-04-08
Only a lawyer can offer legal advice (by law in some countries). I am not a lawyer so I am not permitted to offer legal advice. This therefore isn't legal advice. However my OPINION and UNDERSTANDING is that there is absolutely no problem in doing what you propose. 

Don't worry about SELLING ubuntu. Go ahead and sell it. Proprietary software people care about that sort of thing. The ubuntu community does not. The license even permits it. In fact you can download ubuntu, burn a bunch of DVDs and sell them commercially and nobody will mind a bit. In fact we'll all be very glad for you if you can make a profit doing it.  

There is a lot of scare stuff about the GPL (promulgated mostly by you know who). But obeying the GPL is quite easy. Don't strip off licenses. Make sure the person who receives the computer sees the GPL. Include the source code. That is it. And it is simple to do all that.  Just do a full ubuntu installation (which includes licenses and source code), or include a DVD with source code on it with the computer when you sell it. That takes care of the GPL.

The only other issues are trademark issues. I'm not so up on trademarks but I don't think there is too much of an issue there as long as you don't pretend to be Canonical or Mozilla or whatever. Canonical makes its money offering support. They would probably appreciate it if you would make the person you are selling the computer to aware of this. 

That is my OPINION and UNDERSTANDING. Of course if you want legal advice, you need to talk to a lawyer.

It's very unlikely that you'll get any such requests if this is only a limited run of machines, but just be aware that you'll need to abide by the GPL when redistributing Ubuntu or any other GPL software.[/QUOTE]
