### Post by m_duck on 2014-04-04
Sorry, to post the content run the following in the terminal and copy/paste here:```
cat /etc/apt/sources.list.d/SOGo.list
```That should work as your regular user so no need for sudo there.

I'll assume the installation of the samba related packages worked. So the next bit is that vim is not installed. Vim is a text editor, so the line **vim /etc/fstab** means, edit the file at /etc/fstab using vim. You *will* need to be root to edit that file so will need to prepend **sudo** to the command.

That said, if you aren't familiar with vim, you probably want to use an alternative text editor. Nano is a simpler CLI editor and gedit should be installed by default as a graphical alternative which will be easiest, so instead of the vim line use:```
gksudo gedit /etc/fstab
```
This will achieve the same as the above, but will open **/etc/fstab** in **gedit** as root, using **gksudo** (gksudo is a command to run graphical programs as root). [More info on sudo vs gksudo at Psychocats](http://www.psychocats.net/ubuntu/graphicalsudo).

### Post by m_duck on 2014-04-12
OK, so then the summary of what you have already run is (don't run these again):
```
sudo su
echo -e "deb http://inverse.ca/ubuntu-nightly precise precise\ndeb-src http://inverse.ca/ubuntu-nightly precise precise" > /etc/apt/sources.list.d/SOGo.list
apt-key adv --keyserver keys.gnupg.net --recv-key 0x810273C4
apt-get update
apt-get -y install acl attr samba4 winbind4 krb5-user
```

Working in reverse order, this should undo everything (run these, one by one):
```
sudo apt-get remove acl attr samba4 winbind4 krb5-user
sudo mv /etc/apt/sources.list.d/SOGo.list /etc/apt/sources.list.d/SOGo.list.bak
sudo apt-get update
```

That will remove any packages that the final command you ran installed (if any), then remove the extra package source (the one with errors for SOGo) and then update your package sources. If you want to update your system afterwards, run the following.
```
sudo apt-get upgrade
```
Hopefully this should get you back to normal.

Let me know if you have any other problems. If any of the above don't complete properly, post the errors here and I'll have a look.

### Post by m_duck on 2014-04-14
Sorry Dave, I thought I'd answered this post. A fresh install of Ubuntu (with "add third party whatnots" ticked during installation) has the following enabled (Software Centre: Edit -> Software sources):


[LIST]
[*]Ubuntu Software tab
[LIST]
[*]main - enabled
[*]universe - enabled
[*]restricted - enabled
[*]multiverse - enabled
[*]source code - bar (some source repos selected, see next tab)
[*]Cdrom - disabled
[/LIST]

[*]Other Software tab
[LIST]
[*]Canonical Partners - disabled
[*]Canonical Parters (source) - disabled
[*]Independent (extras.ubuntu.com/ubuntu saucy main) - enabled
[*]Independent (source) - enabled
[/LIST]

[*]Updates
[LIST]
[*]saucy-security - enabled
[*]saucy-updates - enabled
[*]saucy-proposed - disabled
[*]saucy-backports - enabled
[/LIST]
[/LIST]
 
(change mentions of "saucy" above to whatever your release is)

You might also have subsequently enabled the partner repos for e.g. Skype. Finally, if you've added any PPAs or third-party repos, they will be listed in "Other Software" too.

Hope this helps!
