### Post by TBABill on 2011-01-31
In my signature is a link to Ubuntu certified machines. However, many other machines are equally as great with Linux but just have not been officially certified. Dell has the largest number of certified machines, but for laptops they often use Broadcom wireless devices that require a proprietary driver. Not a big deal if you have access to an ethernet port to download updates and install the driver, but for those without a nearby access point other than wireless this can be very difficult to overcome. Intel wireless and Realtek wireless are both supported out of the box, but Intel seems to be much easier if you look at recent posts by owners of Realtek devices.
 
I agree a certified Linux machine is bound to be best, but if cost becomes an issue (they are usually higher in cost, probably due to lower volume sales than HP, Dell, ACER, etc.) you could check out the link. The problem with certification is they don't seem to become certified quickly so the list shows machines that are not the latest technologies all the time.

### Post by SeijiSensei on 2011-01-31
I'm with realzippy.  I've installed various flavors of Linux on a wide array of desktop machines and can't think of the last time I had problems.  In most cases these machines have used wired ethernet connections, so I haven't had to deal with dicey wifi drivers.  The only exception is the machine I'm writing on now when kernel support for Ralink chips went through some contortions a year or so ago making my Linksys USB wifi adapter unstable.  I ended up installing 10.10 to get access to the newer kernel, and things have been fine since.

Compatibility is usually more of an issue with outboard devices like webcams, scanners, etc. I've found Logitech generally makes excellent Linux-compatible devices.  This machine has a wireless keyboard and mouse and a webcam, all from Logitech, and all working flawlessly.  

If you don't care about preserving an existing Windows installation on the hard drive, then you can just repartition at installation and let Ubuntu have the entire disk.  If not, the installer provides some good tools to repartition on-the-fly and set up dual booting.

### Post by perspectoff on 2011-01-31
> **piquat said:**
> HP isn't the one shipping with 4 primary partitions already set up is it?  I remember reading a thread on here about it but I can't find it now.  I'm only half sure it was an HP.  Something else to watch out for if you're considering dual booting.

Yes, the new HP computers I've seen have 4 partitions, and the discussions did relate to the HP. However, it has been noted that HP generally puts 4 primary partitions instead of the more desirable 3 primary partitions and one extended partition.

So one primary partition ought to be converted to an extended partition (which can entail moving any data that HP has thrown onto the fourth partition, first). That's not a particularly onerous task, though. Having the drive already divided into 4 partitions is worthwhile, IMO.

While in general I am a fan of HP, and it is an American company (and if you are in the US you may have a desire to buy from an American company), I have never had a problem with eMachines, Gateway, Dell computers, nor with the Taiwanese Acer machines (which are excellent computers).

As the previous post mentioned, the sticky wicket is the graphics card, and that depends on your intended use. For certain 3-D games and intensive graphics applications (movie editing, etc.) you ought to pay attention to the graphics (and that requires a lot of advice, not a little). For routine usage, though, it is not that important. 

There have been problems alternately with Intel, nVidia, and ATI graphics over the years, but each one seems to be solved within a few months. Any advice you get in these forums ought to be viewed suspiciously, because people often give their advice based on some experience in the remote past, and that isn't generally the right thing to do. Only trust advice about graphics card from someone who has gone through the installation experience recently, as graphics driver options change rapidly.

For example, last year I had a devil of a time with nVidia graphics, but since then I have had no problems whatsoever with nVidia. 2 years ago I had a problem with ATI, but that got sorted out quickly. My longest standing problems were with Intel integrated graphics, which were so bad that for two computers I went out and bought nVidia cards. I have not had problems recently with newer computers with integrated Intel graphics, though.

Besides, most of the graphics problems recently haven't necessarily been with the graphics drivers but with the Plymouth splashscreen, which is the most consistent and persistent problem in Ubuntu I have ever seen. It still does not work on any (that's right, not one) of my computers.(Although initially this caused my computers to refuse to boot, the current symptom is merely a blank screen for 5 seconds while Plymouth runs, which is a minor annoyance but not crippling). 

I have tried oodles of workarounds for Plymouth -- nothing seems to work. (I mostly have nVidia and Intel graphics these days.)

All I can recommend is that if you are using a recent version of (K)Ubuntu, make sure you download a recent release (instead of borrowing a friend's older burned copy), since graphics (and Plymouth) updates are more likely to be included.

People like to say that Windows (or Mac) is so much easier and doesn't have these problems, but that it isn't at all true. Two of my kids use Windows Vista and Windows 7 (Netflix and one or two games like Spore keep them in Windows), and they have endless configuration problems and installation problems that I have to end up solving for them. Networking on Windows is still a nightmare, and seems to be worse now than in the XP days. No OS is free of the need for tweaks.

### Post by Topsiho on 2011-02-01
I have read through all replies so far, and though they are mostly quite helpful, as far as I can tell no one suggested that before you buy your new computer you should try the live CD on it. If that works OK, then chances are quite good that the installed version works OK too.

That said, I think you should only worry about graphics (I prefer NVidia, but I understand ATI works too, and somewhere I have read that they made the specs public), and wifi. If wifi doesn't work, and it's difficult to get it working, many or most of the older USB g-adapters (which cost only a little) work quite well.

I recently aquired a new system, with an NVIDIA graphics card (	
240 nVidia Asus ENGT240 Silent VGA/DVI/HDMI/GDDR3 1024MB) which works quite well, both with the open source driver, and the proprietary driver, which may be installed via hardware drivers in the menu. The latter makes possible to use the quite useless but funny desktop effects :). The not silent version is quite cheaper, but has a fan for cooling.

This computer was built for me, and was delivered without an OS, with only plain vanilla parts. Laptops, and desktops from the big brands often have customized parts which may cause problems. That said, I almost never have any problems installing Ubuntu on computers that come my way. Only once, when I repartitioned a Windows computer when installing Ubuntu: Windows did not boot anymore as the system had changed without Windows knowing it. From this experience, when installing a dual boot system, I first make room for Ubuntu using the partitioner that is in Windows itself, before proceeding.
