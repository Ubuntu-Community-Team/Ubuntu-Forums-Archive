### Post by Brandon Williams on 2009-10-31
The Bugs page for a component has a 'Report Bug' link. In some cases, most gnome components for example, it will take you to a page that provides links for reporting upstream bugs and reporting bugs against the ubuntu package. If you want to report a bug against the karmic package, you simply click the appropriate link to go to the ubuntu bug reporting page.

If this isn't what you're seeing, please provide a url so that we can figure out how to navigate the bug reporting page for the specific component you're having trouble with.

### Post by Brandon Williams on 2009-10-31
What I ordinarily do is search for the project from the main page. For example, if I want to enter a bug against karmic, I search for "karmic bugs", and then click the "Bugs in The Karmic Koala ..." link from the resulting list. In your case, searching for "window picker bugs" from the main launchpad page brings up a list with, among other things, "Bugs in Window Picker Applet". On the selected bugs.launchpad.net page, you will find a "Report a bug" link.

From the karmic bugs page, the "Report a bug" link redirects to a wiki document about reporting bugs. That page indicates that if you want to report a bug against ubuntu but you don't know the package, you should use [https://bugs.launchpad.net/ubuntu/+filebug/?no-redirect](https://bugs.launchpad.net/ubuntu/+filebug/?no-redirect).
