### Post by grsing on 2006-06-08
[QUOTE=MAXDDARK]you've missed my point

The problem is (well, part of it), those old games were made to run on comparatively very slow hardware, which means that a modern computer has little trouble emulating it, even though it takes considerably more power than it would by running it natively (for example, and these numbers are completely made up, say an old game ran on a 50 mHz processor on the hardware it was designed for; a modern computer with 2000 mHz or whatever can easily soak up the performance penalty, which means the game requires 100 mHz or whatever).  However, since Linux and Windows run on machines of the same power, it's much harder to do an actual emulation of a Windows environment without severe performance penalties.  That's why WINE isn't actually an emulator, but also why it can't run everything (when things that programs need aren't available in Linux; someone please correct me if I'm wrong, but I think I got the general gist of it).  There's no technical reason that Windows programs can't be made to run on Linux, but emulating Windows is an entirely different beast.
