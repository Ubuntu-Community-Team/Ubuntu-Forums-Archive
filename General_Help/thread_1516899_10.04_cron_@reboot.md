### Post by scifan on 2010-06-24
I use RTG to monitor bandwidth use on some of our network gear...

Prior to upgrading to 10.04, I launched RTG using an @reboot entry in the root crontab... (I haven't got any clue how to setup an init.d file for it... and this worked for me)

However, this tag doesn't seem to work after upgrading to 10.04.

How should I go about ensuring this application get's launched at startup?

(is there a better way I SHOULD be doing this?)

The basic command line used to launch it is this: 

/usr/local/rtg/bin/rtgpoll -v -t /usr/local/rtg/etc/targets.cfg -c /usr/local/rtg/etc/rtg.conf 

Was @reboot a Vixie-Cron feature that isn't supported in Anacron?

### Post by ajgreeny on 2010-06-24
I think you should add the contents of your script to the existing file, /etc/rc.local.

You will not need sudo, if the script normally needs root permissions, as being in that file it is already root enabled.

There may be other ways to do this as well, but this should work as far as I can see.
