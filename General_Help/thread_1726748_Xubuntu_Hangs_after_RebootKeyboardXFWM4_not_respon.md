### Post by 42dorian on 2011-04-11
Okay, here's the deal.  I currently use Xubuntu 10.10, Intel model 530 3.00Ghz Hyperthreading processor, 4GB of DDR400 RAM, several hard drives, separate drive for /home, and a few others for storage.  ATI All-In-Wonder 9600XT Video card.  I have the Microsoft Wireless Optical Desktop 2.0 (Has the Microsoft Wireless Natural Multimedia Keyboard and a Wireless Intellimouse Explorer 2.0 in one set.)  I use Keytouch and xbindkeys to map the special keys.

For the past couple installs, I've been running into a problem.  Namely when I boot up, from time to time, the Keyboard stops working, and I only have the left mouse button and mouse tracking left in my control.  I can minimise and maximise windows, but I can't move them.  I can open GnoMenu, but none of the programs will run, nor will shutdown/reboot.

And at this point in time, I'm running the LiveCD off a USB drive in order to access the computer at all.  The install is... to say the least... impossible to get into for longer than a couple minutes.

What should I be looking into?  Suggestions?  What's up here?  Anybody?

*EDIT*  Found it.  It's Keytouch crashing.  The solution is to run a system monitor and kill it so it'll restart.  Luckily, this can be done without the keyboard.
