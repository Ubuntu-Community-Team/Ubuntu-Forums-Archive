### Post by Ramses de Norre on 2007-03-07
Firefox can't render php files, it can only render html files. So you need a webserver to interprete the php code and turn it in html which is then rendered by firefox.
Every site you visit that uses php works like this, they send you html that is generated by their webserver from a php file.

### Post by tocleora on 2007-03-07
> **zeroblitzt said:**
> Yup, you don't see PHP, you see the HTML that comes as a result of the PHP. Took me a while to understand that concept when I was first learning how to code in PHP :)

It's simple really...

1. Page resides on Server with PHP code in it.
2. Client browser requests document from server.
3. Apache receives document request, grabs php, parses the php code and submits parsed php code (converted to html) to browser.
4. Browser receives document in HTML format.

I might not be 100% accurate on #3 and I've simplified it, but that should give a general idea of how it works. :)  PHP is known as a "Server Side Script" meaning it's code is processed on the server side before submitting to the browser.  Javascript for example would be "Client Side Script" which means it's sent as-is to the browser and the browser itself parses it.  That's why you can view|source and see javascript but you can't view|source and see php.
