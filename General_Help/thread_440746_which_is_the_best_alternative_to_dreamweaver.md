### Post by cleentrax on 2007-05-12
What I mean is that the current best practice for designing with css and html doesn't work well in a WYSIWYG environment, especially considering that WYS depends largely on your browser and OS.


If you use it in "Design" mode, Dreamweaver creates bloated, ugly code. If you use it in "Code" mode, Dreamweaver is a much bigger and more complicated app than you need. 

Create your HTML so that it makes sense, using div tags to set off sections, and using tags semantically rather than for positioning. Then, start creating CSS to style and position your div sections in your HTML. If you work this way, you will end up with a cleaner, faster site.

### Post by Jay_in_TN on 2007-05-13
Dreamweaver has the option to preview your page in any browser you choose. You edit the menu and then preview in any browser. Besides--of the browsers you mentioned--the vast majority of users are still using Internet Explorer or Firefox. These account for like 95-99% of all browsers in use. 

A wise developer uses Dreamweaver in "split" view with Design Mode sharing space with Code mode. The code is not that bloated or ugly in Dreamweaver 8 and there is a code verifying tool built in. Additionally you can verify your code through the WC3 consortium and get a double verify.

I simply can't agree that Dreamweaver is outmoded. In fact, it is my opinion that it is the premier development tool available. And I use CSS almost exclusively in Dreamweaver and I have no problems whatsoever. 

I would also like to see a web development tool for opensource/Linux that rivals Dreamweaver. It is really surprising that a program has not kept up. OpenOffice is a total replacement for MS Office and The Gimp is a suitable (not perfect) replacement for Photoshop. But web development seems to be lagging waaaaaaaay behind.

### Post by Peter Mount on 2007-05-13
I'm working on my own Web Development business and I tend to use Kate myself. I use CSSED for CSS and FireFTP in Firefox for ftp.

I learned to use Dreamweaver in my IT studies. At the time (a few years ago) we were told to always back up our settings in Dreamweaver because it had a reputation for crashing with the need for re-installation.

It is possible to do Standards Compliant code in Dreamweaver and to have it validate properly. If you have a lot of money to burn it might be fun to check out an evaluation copy to see what it's like before buying it. A lot of people just use open source software simply because it's free and Dreamweaver is really expensive.

A lot of Mac people seem to like BBEdit. I haven't tried it myself as it's only available for the Mac.

I've tried Eclipse with plugins but that does need a reasonably fast machine to run well. Plus I had a problem with one plugin causing it to crash and this made me wary of using software that relies on third party plugins.

EDIT: There's a comparison of different PHP editors (including Dreamweaver) at [http://www.php-editors.com/review/](http://www.php-editors.com/review/)
