### Post by guiliker on 2009-11-02
I have found a temporary workaround!!!

firstly, modify "**/etc/X11/xorg.conf**" with:
```
Section "ServerFlags"
        Option "RandR" "on"
EndSection
```

then create an executable file entitled: "**~/.xprofile**" with the following contents: 
```
xrandr --newmode "1024x768_60" 65.0 1024 1048 1184 1344 768 771 777 806 -vsync -hsync
xrandr --addmode VGA1 "1024x768_60"
xrandr --newmode "1280x960_60" 102.1 1280 1360 1496 1712 960 961 964 994 -hsync +vsync
xrandr --addmode VGA1 "1280x960_60"
xrandr --newmode "1280x1024_60" 108.0 1280 1328 1440 1688 1024 1025 1028 1066 +hsync +vsync
xrandr --addmode VGA1 "1280x1024_60"
xrandr --output VGA1 --mode "1024x768_60"
```

then go to Menu>System>Preferences>Startup Applications and add the file.

Desired screen resolution however some icons on the panels are duplicated or randomly moved. Currently the best I can do as any xorg.conf file configuration makes no difference whatsoever or renders Ubuntu useless needing a "**sudo dpkg-reconfigure xserver-xorg**" needing to be run from the CLI, which flickers unless it is in "failsafe" mode.

Also to note I have made a bug report using the command "ubuntu-bug xorg"
