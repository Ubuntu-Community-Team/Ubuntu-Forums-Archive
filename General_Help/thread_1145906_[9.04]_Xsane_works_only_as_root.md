### Post by ilpiero on 2009-05-03
Thanks a lot CatKiller, I solved the problem, I'll try to summarise what I've done in case someone else steps in the same trouble.

1) Apparently jaunty does not create the scan group for the user, (maybe the scanner should be powered on during the install??), so we can add it by entering:

```
sudo addgroup scanner
sudo adduser <username> scanner

```

2) For some reasons the SCSI scanner epson GT5500 is accessible only as root, by typing in the console 

```
sudo chmod 777 /dev/sg4
```

where sg4 is my device 

xsane will work for the session, but will eventually give you the error
like (I don't remember the exact phrase) "unable to access file" when closed, to solve this I gave the permission to all the users in the group scanner to modify the files in the folder /home/<username>/.sane/

3) To grant access to the peripheral an udev rule must be inserted, so i added a file 40-permissions.rules from my old ubuntu installation in the /etc/udev/rules.d/ folder.
I don't know exactly how the udev rules work, but maybe the part to be added is (from the 40-permissions.rules file):

```
# SCSI devices
SUBSYSTEMS=="scsi", GOTO="scsi_start"
GOTO="scsi_end"
LABEL="scsi_start"
ATTRS{type}=="0",			GROUP="disk"
ATTRS{type}=="1",			GROUP="tape"
ATTRS{type}=="4",			GROUP="cdrom"
ATTRS{type}=="5",			GROUP="cdrom"
ATTRS{type}=="6",			GROUP="scanner"
ATTRS{type}=="8",			GROUP="tape"
ATTRS{type}=="3", ATTRS{vendor}=="HP",	GROUP="scanner"
ATTRS{type}=="3", ATTRS{vendor}=="Epson", GROUP="scanner"
ATTRS{type}=="3", ATTRS{vendor}=="EPSON", GROUP="scanner"
LABEL="scsi_end"
```


regards.
