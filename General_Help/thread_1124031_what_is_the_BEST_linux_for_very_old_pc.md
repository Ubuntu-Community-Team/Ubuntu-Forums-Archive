### Post by Sorivenul on 2009-04-12
With that PC, I would suggest upgrading your RAM (if possible - RAM is cheap these days) to at least 256Mb.

If you can, you could easily run SliTaz (loram), or Tiny Core. SliTaz, last I knew was installable and has a base ISO of about 30Mb and can do most basic tasks out of the box. TinyCore doesn't need to be installed as its 10Mb ISO runs entirely from RAM after loading from CD or USB, using a folder/directory on the hard drive as persistent storage for files and packages.

I see no need for an entirely new computer, and see no need for something with as much bloat as Ubuntu or any other "major" distribution.

**EDIT:**
For those saying the OP needs and entirely new computer, maybe a new view of how to utilise software may be a good course of action. There are many possibilities out there that can *easily* run on a machine with the stated specifications and provide the tools any common user would need. Slackware, SliTaz, Tiny Core (loram), DeLi, or any BSD can easily handle the job. Just because Ubuntu (or any of its derivatives) won't run on that machine does not make it "too old" and certainly does not require the purchase of an entirely new machine - a bit more RAM would be a nice addition, but not necessary.
