### Post by ashmael on 2019-04-10
Ubuntu 18.04

Just a few months ago, I could right click a .img file in Nautilus, select 'Mount Disk Image' and it would mount the image, as writable, and show the mounts in the left tab of Nautilus.

Today, I right click a .img file in Nautilus, select 'Mount Disk Image', and it mounts the image as read-only, and it doesn't create links the mount in the left tab of Nautilus. I have to navigate to '+ Other Locations' to see the mounts, and they are now read only.

I want to mount as writable, just like I was before. It looks like 'gnome-disk-image-mounter' says that by default it mounts only as readable. But that wasn't the behavior just a few months ago, when I had to mount and edit dozens of images.

Once again I have to mount and edit dozens of images, and the default behavior of this OS has changed from something useful and user friendly to something that Apple would come up - looks nice and removes any configuration ability from the users.
