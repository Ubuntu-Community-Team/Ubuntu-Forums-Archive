### Post by bcbc on 2010-10-26
> **harrylucas1 said:**
> and its running fine.....

You're still running off a virtual disk, not a separate partition. (LVPM has two functions - migrate to a partition, and resize the root.disk. You must have done the latter.)

You only have one ntfs partition, the other is the loop mounted root.disk. Your fstab refers to ext4 whereas the loop mounted file system is ext3. You probably have grub-legacy installed so your grub menu will never be updated automatically (when you install new kernels etc).

To fix it, update your fstab so that the root.disk is ext3. Reinstall grub-pc (taking care to NOT install the bootloader to /dev/sda or /dev/loop0 or /dev/sda1 i.e. don't install it anywhere). 

You'll still be running wubi after that.

I've never heard of the c:\ubuntu\disks folder disappearing. I recommend running that bootinfoscript I linked to, but you'll need an ubuntu cd to boot from (select "try without installing").

You should have a c:\wubildr and c:\wubildr.mbr file, and then a c:\ubuntu directory (or d:\ubuntu or whatever depending on where you installed wubi), and within that you'll find an \ubuntu\winboot\ directory containing wubildr.cfg, wubildr, etc. and \ubuntu\disks\ directory with swap.disk and root.disk. The root.disk is the most important as it contains all your data, and the other files are required to loop mount and boot the ubuntu on the root.disk.

If you aren't able to boot a live cd use windows search to look for the root.disk file.
