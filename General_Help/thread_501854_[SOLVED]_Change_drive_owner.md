### Post by bodhi.zazen on 2007-07-16
The way to set permissions with FAT and NTFS are different from linux native file systems, so 

chown ...
chmod ...

Will not work.

Two options :

1. Install [ntfs-config](http://doc.gwos.org/index.php/Ntfs-config), This will allow you to do it graphically.

2. Command line

With the terminal, 

[list][*]First make a mount point, I will call it data, you can call it what you will ... ```
sudo mkdir /media/data
```

[*]For ntfs, install ntfs-3g : ```
sudo apt-get install ntfs-3g
```

[*]Now for both fat and ntfs you set permissions at the time of mount.

[list=number][*]From the command line :```
sudo mount /dev/<device> /media/data -o uid=<user>,gid=<group>,umask=000
```[list][*]<device> = partition to mount [*]<user> = 1000 (or user name) [*]<group> = 100 (or users)[/list]

[*]Now to mount at boot you must update fstab.```
gksu gedit /etc/fstab
```
Add a line(s) like this : > /dev/<device> /media/data vfat users,uid=1000,gid=100,umask=007 0 0
/dev/<device> /media/data ntfs-3g users,uid=1000,gid=100,umask=007 0 0[indent]The vfat line is for fat partitions and the ntfs line is for ntfs partitions[/indent][/list][/list]

_Windows:_ [Psychocats Mount windows ](http://www.psychocats.net/ubuntu/mountwindows)

_Linux_: [Psychocats Mount Linux](http://www.psychocats.net/ubuntu/mountlinux)

[http://doc.gwos.org/index.php/Understanding_fstab](http://doc.gwos.org/index.php/Understanding_fstab)

HTH

### Post by bodhi.zazen on 2007-07-17
Awesome :)

I understand your being a little gun shy, but you are doing just fine.

OK a few pointers if you will.

1. Mount point. You can make a mount point where ever you like. The "ubuntu way" is to use /media so I suggest you make a mount point of /media/Windows

2. Nothing will happen to the other mount points. They exist as empty directories.We can clean them up if you like :

```
sudo umount /media/BackUP
sudo umount /Windows
sudo rm -r /media/BackUP
sudo rm -r /Windows
```


3. Any device or partition you mount in /media will appear in Places and as an icon on your desktop. If you do ot want it there use /mnt ( ie /mnt/Windows )


4. You can use other mount points, it will not hurt anything if you use /Windows or /home/user_name/Windows.

What you can do is a link.

```
ln -s /media/Windows ~/Windows
```

This creates a link, or shortcut in your home directory directly to /media/Windows.

Cool ?

5. What was 5 ?

Oh, I would use this in fstab :> /dev/sdb1 /media/Windows ntfs-3g defaults,locale=en_CA.UTF-8 0 2


That should do it. Hope you learned a little something along the way. Feel free to ask if your would like to know more or do ot understand something.

### Post by bodhi.zazen on 2007-07-17
The columns on fstab are :

1. Device
2. mount point
3. file system identification/driver
4. options
5. dump
6. Check (at boot)

See this link : [http://www.tuxfiles.org/linuxhelp/fstab.html](http://www.tuxfiles.org/linuxhelp/fstab.html)

And the long version : [http://ubuntuforums.org/showthread.php?t=283131](http://ubuntuforums.org/showthread.php?t=283131)

To edit fstab :

Well, I do this from a terminal as well ;)

```
gksu gedit /etc/fstab
```

### Post by bodhi.zazen on 2007-07-18
1. That error message is a well know bug (it has been reported). It has been given a low priority to fix because, as you can see, it does not affect functionality.  So you can ignore it for now and perhaps some day they will fix it ;)

2. The options in the 4th column are comma delineated, so you need to change >  rw user,exec,defaults,locale=en_CA.UTF-8 to ```
rw,user,exec,defaults,locale=en_CA.UTF-8
```

But in all likely hood the following will work just fine :

```
user,locale=en_CA.UTF-8
```

You *might need to add uid=1000,gid=100,umask=007, but I doubt it.

For the benefit of anyone who has been following this saga, I am pleased to report the problem has finally been resolved, though I am not sure exactly how.

After repeatedly following the instructions kindly provided by Bodhi.zazen without being able to get the d**ned drive to play properly, I concluded that there MUST be something wrong with the way the drive is formatted.
