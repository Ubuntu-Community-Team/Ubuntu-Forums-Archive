### Post by Morbius1 on 2010-11-29
I truly and honestly do not know why so many Ubuntu users spend so much time deviating so far from the default smb.conf.

Anyway, looking at your smb.conf for "dad" you need 3 things to make that work:

(1) you need to change "encrypt passwords = No" to "encrypt passwords = Yes"

(2) You are going to have to create a samba password for everyone except "nobody" in this line: "valid users = dad, nobody, smbguest"

sudo smbpasswd -a dad
sudo smbpasswd -a smbguest

[COLOR=Blue]* Since you have nobody and smbguest in there I'm not entirely sure why you have a "valid users" line at all since it seems you want to allow guest access.*[/COLOR]

(3) As dandnsmith mentioned above you need to make sure your linux file permissions are in sync with your Samba authentication requirements.

By default /home/dad will have owner:group = dad:dad and permissions will be set to 755. dad will be able to read and write and everyone else ( and that includes nobody and smbguest ) will only be able to read.
