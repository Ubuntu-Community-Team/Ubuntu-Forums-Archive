### Post by yancek on 2017-11-06
You have installed Ubuntu in UEFI mode while your windows is installed in Legacy mode thus the conflict.  You can boot one system but not the other.  Take a look at the Ubuntu documentation at the link below.  Scroll down to the section Converting Ubuntu into Legacy mode.

You can tell this is the case because you have windows code in the MBR and trying to boot Ubuntu from a windows bootloader is not going to be that easy.  You also have an EFI partition which you would not have with a Legacy install and the only files on that partition are Ubuntu/Linux files.

[https://help.ubuntu.com/community/UEFI](https://help.ubuntu.com/community/UEFI)

Since you just installed Ubuntu, you might try reinsalling it in Legacy mode to match your windows install.  How to tell if you are booting the Ubuntu install media in Legacy mode is shown at the link above.  Probably be a good idea to read over the entire page at the link above to familiarize yourself.

Is this a Mac?
Some with older Mac installed Windows in BIOS mode and Mac is UEFI (actually earlier EFI). And then Ubuntu was normally installed in UEFI mode.

But then you had to use hybrid partitioning which is not particularly reliable.
       [http://www.rodsbooks.com/gdisk/hybrid.html](http://www.rodsbooks.com/gdisk/hybrid.html) 

Not sure with a hybrid configuration, but on a PC grub only boots other installs in same boot mode. So if Ubuntu is UEFI, it will only offer to boot other UEFI installs.

Many with Mac use rEFInd which I believe can reboot and then boot other systems in different modes.
      
 [http://www.rodsbooks.com/refind/](http://www.rodsbooks.com/refind/)

Windows 7 can be installed in UEFI boot mode, but not from default DVD. You have to copy to flash drive and move files around to make it UEFI bootable.
