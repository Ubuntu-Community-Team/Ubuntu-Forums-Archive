### Post by Bashing-om on 2016-03-22
alex519; Strange;

But .. as you can get to a terminal, we can work from there.

How about we do not start the GUI by booting from grub to the TTY1 terminal ? From TTY1 we can look around and see what we can find that is missing. Maybe

Reboot the box, and as soon as the bios screen clears ( legacy hardware that is ) depress and hold a shift key -> grub menu.
At the grub menu 'e' key for edit mode -> boot parameters screen;
Arrow down to the line starting with linux and across to "quiet splash" replace these terms with the term "text" - without the quotes.
Key combo ctl+x to continue the boot process to TTY1.
Log in here with user name and pass word - there will be no response to the screen when pass word is entered, Enter pass word blindly and hit the enter key.

At this terminal what results:
```

sudo apt update
sudo apt upgrade
sudo apt-get -f install
sudo dpkg -C

```
We here make sure the package manager is consistent and then we look at the system.
