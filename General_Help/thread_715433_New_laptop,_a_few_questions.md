### Post by gpilkay on 2008-03-04
To be honest, I wouldn't try it until you DID ask the company directly, as different people have different ideas of what constitutes a warranty violation.  Sometimes re-installing the OS with a different one, even if you technically didn't overwrite it, still can count against you.

I myself had no end of trouble with it on my Dell, I wound up running it off a Live-CD when I needed it for a while, then I wound up working around it, by installing VirtualBox.  I now can hit cntrl-F and I have a virtual Ubutu Laptop running with XP hidden in the background.  Not ideal, of course, but it bypassed all the hardware troubles by fooling Ubuntu into thinking it had everything it wanted.

So, I'd definitely check the warranty first.  After that, we're happy to help however we can!

Also, 64 bit Ubuntu...I'm not sure about how it will work with your computer, but I tend to keep it at 32 bits if the Windows version was the same.  Not to mention 32 bit Ubuntu has a much better compatibility range.  Try a few Live-CD's and see what you think!  They're the easiest way to see if something will work.  I've got a stack of them.

### Post by pointone on 2008-03-04
Dual core does not necessarily mean 64 bit capability. 32 bit Linux can make use of four or more processors. 

If it is an Intel Core 2 CPU, then yes, 64 bit is supported.
