:KS  My two cents.......(about .05 in our current economy ;) )

1.The Open Source backbone of Ubuntu fosters more effective communication regarding bugs and/or holes in the system.  The community can react more quickly that the equivalent corporate entity to vulnerabilities.  The list of Linux malware out in the world is only a fraction of that for Windows/VB architectures.

2. Yes, as the average user will be installing an OS which comes complete with at least 90% of the software that he/she will need on a daily basis.  Fewer outages due to installation/device conflicts and to the average user, less technical knowledge required.  I personally consider the Ubuntu GUI very intuitive and less technically "cluttered" than XP Home.

3. Many of the big ones are beginning to see the importance of Linux support, such as printer makers.  The community will continue to evolve as the OEMs do.  As with anything like this, as the community and demand grow...so will their response.  Some good info:

[http://www.linux.com/print.pl?sid=06/11/02/1515243](http://www.linux.com/print.pl?sid=06/11/02/1515243)

[http://ubuntu.wordpress.com/2006/04/24/emulating-linuxs-hardware-support-in-windows/](http://ubuntu.wordpress.com/2006/04/24/emulating-linuxs-hardware-support-in-windows/)

[http://www.cyberciti.biz/tips/why-linux-dont-support-mp3-and-selected-wifi-cards-out-of-the-box.html](http://www.cyberciti.biz/tips/why-linux-dont-support-mp3-and-selected-wifi-cards-out-of-the-box.html)

4. Haven't needed it, as most problems are quite fixable.  The more "open" nature of the system allows better access to critical code for problem correction.  The forums offer a great resource for me, making formal support unnecessary thus far.

5. A session, as in Windows, could be application-based (such as a browser), or OS-based.  It really depends on the code, which data it's built to retain "per session", etc.  Personally, I'd consider a Ubuntu-level session to be from login to shutdown, as "switch user" functions retain some data.

6. I'm not sure how the Geek Squad would react to it these days :cool:  but one could take the hardware on which Ubuntu rests to any commercial shop.  If the user is lucky, there will be a Linux person there.  A great many software issues can be resolved via the Forums.

7. Ubuntu disadvantages: Less support for "off the rack" applications, but strength in the diversity of applications found throughout the community.  Windows disadvantages: Security and considerably tapping one's bank account when upgrade time comes around (not to mention the compatibility fun that will come with Vista)...strength for those who want to pick up their apps at Best Buy :mrgreen: 

8. Probably a good Ubuntu Server question.  I haven't run it yet in a production environment, but I'm betting that it's solid........like when I first removed Microsoft FrontPage Extensions from a production box :)

All in all, I feel that Ubuntu helps to return computing to its "essence".......true innovation, evolution and collaboration.  All of the good stuff from the old days (keeping in mind that I am a 40 y.o. user who began on a TRS-80 and Timex Sinclair ;) ).  I haven't missed Windows XP.

Best of luck, and a very Happy New Year to you as well.

### Post by sloggerkhan on 2006-12-29
Using Ubuntu somewhere from 5-7 months....

1.Would you say that Ubuntu has better security than Windows XP Home Edition? If so, why?

Yes. Because it doesn't call Microsoft and spy on you. (I have heard ubuntu can be forkbombed, but I haven't tested that, and Debian is supposed to be immune, so maybe just heresy. If nobody can log in besides you anyhow, it doesn't matter.)
Also, you can verify the integrity of open source apps yourself if you have the inclination.

2.Do you think that a user who knows nothing about computers could learn how to use Ubuntu?

Yes. My experience is that if you pre-install Ubuntu or even Xubuntu, the less users know about windows or other OSes, the easier it is for them, and people who have constant trouble with windows seem to like Ubuntu the best. In many ways Ubuntu is much simpler than other OSs widely used. You can just put check marks next to the applications you want. You don't have to enter tons of registration codes and activate products. Things just work.


3.Do you know of any manufacturers of hand that have or are starting to support Linux? If so, what version of Linux is it?

I've heard HP "supports" debian, but I think it's mostly a symbolic thing. Dell does linux for businesses. IBM also supports linux. 

There are also a number of vendors that sell linux computers.
[http://www.linux.org/vendor/system/index.html](http://www.linux.org/vendor/system/index.html)
[http://www.linuxhq.com/vendors/systems.html](http://www.linuxhq.com/vendors/systems.html)
There are more extensive lists elsewhere.


4.Have you ever paid for Ubuntu commercial support? If so, was it good and useful? Would you recommend it?

Never paid for it. Forums are too good. ;)

5.What is a Session in Ubuntu terms?
I have no idea what you mean by this question. I'd hazzard that I consider a session a running login.

6.Do you know of many computer shops where you can take a Ubuntu system in to get worked on?

I know they exist, but I actually couldn't imagine needing to.

7.What are the advantages and disadvantages of Ubuntu over Windows XP Home Edition?

Advantage of XP home edition: Game Compatibility, Guaranteed Wireless Drivers
Otherwise, all advantages are Ubuntu:
Easy and Quick Install, Browse Net while you install, install includes basic Apps, you can install apps by clicking a box, more customizable desktop, can have better than Aero effects on computers that couldn't run Vista, more stable, doesn't really need anti-virus, etc. 

As a side note, Ubuntu on a computer without fast internet could be annoying, possible more annoying than on windows.


8.How good is Ubuntu's Fault Tolerance?

Not sure what you mean...


And one final note: Cost

On windows it is very hard to have a 0 cost machine. You can do it, I've done it, but it requires a great deal of searching to find and download all the free apps for windows.

daemon tools, (which you'd never need on linux)
avast
gimp/gimpshop
open office
a good text editor
blender
etc

ALL have to be downloaded and installed seperately on windows. On Ubuntu, they're preinstalled or I go to add-remove and click a few check boxes.
