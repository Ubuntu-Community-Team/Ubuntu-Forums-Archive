### Post by Zeljka Lin on 2011-05-04
JEEEESS I've solved it!!!!

So for of you that have Ubuntu and Windows 7 installed on separate hard drives and you want to be able to choose OS at startup here it is:
There are several problems to solve so I'll try to be as precise as I can:

1) Boot into your Ubuntu, go to the terminal and type:

> sudo os-prober
if you get something like this:
```

/dev/sda1:Windows 7 (loader):Windows:chain

```you are OK and can go to part 2)

If you get the following error (you are not OK):
```

ls: cannot access /var/lib/os-prober/mount/boot
Boot: No such file or directory

```
So, to solve this error first:
> sudo fdisk -l
to see where your Windows is installed (in my case it is on sdb1)
than type
> sudo mount /dev/sdb1 /mnt
and go to ->  /mnt folder (Home -> and then press the up arrow till you can find /mnt folder) and there  **delete /boot folder. WARNING: DO NOT delete the /Boot folder since it contains files needed by Windows to boot properly. you will see two folders with same name /Boot and /boot.**

when you delete it, type
> sudo umount /mnt
to unmount your drive

OK, now try again 
> sudo os-prober 
to see if the error is still there (hopefully it is not)

When this is solved we can continue:

2) Since the windows bootloader is on an entirely different hard-drive than grub, we must trick windows into believing that it is in fact the first hard-drive.

So type in terminal:
> sudo gedit /etc/grub.d/40_custom
and there, on the end of this file add the following lines:

```

# (2) Windows 7 
menuentry "Windows 7" {
set root=(hd0,1)    #this is the location of the GRUB file
drivemap -s hd0 hd1    #my Ubuntu is on hd0, and windows in on hd1, you must adjust this lines accordingly)
chainloader (hd0,1)+1
}

```than save and exit
type in terminal:
> sudo grub-mkconfig -o /boot/grub/grub.cfg
> sudo update-grub

than restart and you should see the boot menu with your win7 on it!!

If you are confused with all those hd-s, sd-s ad other stuff you may download Boot Info Script and run it, it will tell you all you need to know. Here is the link to download it: [http://sourceforge.net/projects/bootinfoscript/](http://sourceforge.net/projects/bootinfoscript/)
instructions on how to run it are here: [http://bootinfoscript.sourceforge.net/Hope](http://bootinfoscript.sourceforge.net/Hope) this helps!

### Post by idoitprone on 2011-05-06
Sorry I did not follow up. I hope that link helped :). A few things, I have to correct you. If your os-prober worked, then you would not need to add an 40_custom to your grub menu. I believe ```
sudo grub-mkconfig -o /boot/grub/grub.cfg
``` is an uneeded step as grub was making the proper config file with update grub.

You should check the permission on 40_custom and 30_os-prober. If they both work, then you should two entries on your grub list not one.

One more thing, if you invoke ```
update-grub
