### Post by oldfred on 2010-01-31
You add custom menu entries into 40_custom and run update grub. You can turn off the prober with this to avoid duplicate entries:
GRUB_DISABLE_OS_PROBER=true
or you can turn off the execute flag on the prober also.

Only if you updated system would you need to turn prober back on.

Another alternative if DSL uses old grub is to install old grub to the DSL partition and add a simple chainboot entry to link to the DSL grub. (Similar to chainboot to windows)

menuentry "Chainload Other System on sda1" {
set root=(hd0,1)
chainloader +1
}

Grub2 with chainboot examples
[http://kubuntuforums.net/forums/index.php?action=printpage;topic=3106368.0](http://kubuntuforums.net/forums/index.php?action=printpage;topic=3106368.0)
