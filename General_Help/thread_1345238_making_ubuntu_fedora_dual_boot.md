### Post by jbrown96 on 2009-12-04
grub.cfg should always be writable by the owner. you want to do the first chmod command that lawaladekunle said, but don't undo it with chmod u-w grub.cfg. The system re-creates grub.cfg on every time the computer is booted. Look at the beginning of it > # DO NOT EDIT THIS FILE
#
# It is automatically generated by /usr/sbin/grub-mkconfig using templates
# from /etc/grub.d and settings from /etc/default/grub
 Changing this file won't help because any chages will be over-written. Furthermore, if the system doesn't have write access to the file, then grub may fail really hard, and then you will be in much more trouble because Ubuntu might not work.

/usr/sbin/grub-mkconfig -o /boot/grub/grub.cfg is the command you want to run. > sudo update-grub is the same command, just easier to type. 

What happened when you ran update-grub?
