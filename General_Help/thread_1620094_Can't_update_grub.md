### Post by drs305 on 2010-11-12
plasticookies,

Welcome to the Ubuntu forums.  :-)

If Grub2 doesn't find /etc/default/grub it can use default values and run without errors. The fact that you received a number with the error may indicate usage of Grub legacy even though that file isn't used by Grub legacy.

Please run the boot info script and post the results here. The script can be downloaded from:

### Post by drs305 on 2010-11-12
[Added: are you trying to update with the correct command: "sudo update-grub" ?]

If the file /etc/default/grub is missing (and you didn't delete, move, or rename it), you can restore it with the following command - assuming you have booted into your Ubuntu installation normally (not via the LiveCD):

```
sudo apt-get install --reinstall grub-pc
```

If that doesn't work, you may have to purge and reinstall Grub2 completely or if that is the only missing file we can post the contents. 

Before you do a complete G2 reinstall (it's easy, but I tend to support minimal changes if possible), however, check to see if the /etc/default/grub file exists with the following command. If it does exist, please post the contents:
```
cat /etc/default/grub
*or *
gksu gedit /etc/default/grub

```

### Post by drs305 on 2010-11-12
I think we can clear this up by renaming the post-installation script for grub-pc. We'll rename it and then you can try running the reinstall command again:

```
sudo mv /var/lib/dpkg/info/grub-pc.postinst   /var/lib/dpkg/info/grub-pc.postinst.bad
sudo apt-get update
sudo apt-get install --reinstall grub-pc
```

### Post by drs305 on 2010-11-13
> After completing the stated steps and reinstalling, I still end up with the same error as mentioned in the first post. I guess this means I have to completely purge and reinstall grub? 

Try these commands first, as the system may not like purging the way I describe in my chroot guide.
```

sudo -i
dpkg --purge --force-depends grub-pc
apt-get install -f
apt-get clean
apt-get -d install grub-pc
dpkg --force-overwrite --install grub-pc
dpkg --configure -a
exit

```

If that doesn't work, then a purge/install would be the next step.
