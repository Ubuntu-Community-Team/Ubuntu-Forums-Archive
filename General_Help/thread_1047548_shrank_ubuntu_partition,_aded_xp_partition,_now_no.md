### Post by drs305 on 2009-01-22
Use this to make sure your UUIDs are correct:
```

sudo blkid -c /dev/null

```

For your flash drive, the easiest thing would be to make a label for it and then it would always mount by the label name. How to label it depends on the format.  ntfslabel for ntfs (from ntfsprogs), tune2fs for ext3, or gparted can do both if it is a recent version.

### Post by drs305 on 2009-01-22
I just put this into my fstab and it worked fine - same as yours except mine had ntfs-3g, which isn't a factor unless you are running an older version of ubuntu:
```

/dev/sdd5 /media/ntfs ntfs-3g defaults,force 0 0

```

Note you said you were using ntfsprogs. That is an ntfs utility program but you are using ntfs-3g if you are using Hardy/Intrepid. One of the utilities is ntfsfix - you can read about it in the man pages. There are mixed opinions on whether it should be run on a troublesome partition if you can't clear an unclean mount with a windows remount. I have used it without problems but others warn against it.

Normally this would have been asked much earlier in the thread. If you have a problem with the partition table this might help find it:
```

sudo fdisk -l

```
