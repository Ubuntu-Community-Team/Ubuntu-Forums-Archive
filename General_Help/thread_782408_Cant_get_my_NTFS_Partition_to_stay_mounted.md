### Post by Rocket2DMn on 2008-05-04
I'm leaving right now for dinner, but your fstab entry will look like this:
```
/dev/hda5 /media/<your_mount_point> ntfs-3g defaults,locale=en_US.utf8 0 0
```
Make sure you create the mount point 
```
sudo mkdir /media/<your_mount_point>
```
Then you can mount
```
sudo mount -a
```
<your_mount_point> can be whatever you want to call it, just don't use a name with spaces in it.

I prefer to use UUIDs, but it's your call.  Just make sure it really is hda5 using the fdisk command (sometimes fstab says hda when it's really sda).

### Post by Rocket2DMn on 2008-05-05
Those commands show that hda5 is a fat32 partition, and hda2 is your windows ntfs partition.  If the ntfs partition is the one you want to mount, make the fstab entry (for hda2)
```
UUID=3E30A0A030A0611F /media/<your_mount_point> ntfs-3g defaults,locale=en_US.utf8 0 0
```
If it's the fat32 partition you want, add the entry (for hda5)
```
UUID=64FF-39A0 /media/<some_other_mount_point> vfat defaults,user,rw 0 0
```
