### Post by Kirk Schnable on 2012-11-20
I am personally a web programming guy, and I recommend learning PHP.  It is heavily used in a lot of popular websites, from Facebook to YouTube, and very versatile. 

You can write simple command-line scripts in PHP that do not have to run on a web server, as well. 

PHP is also very easy to use with databases like MySQL, which makes it an excellent tool for making dynamic content.

It's got some of the best documentation I've ever seen available for free on its official website: 
[http://www.php.net/manual/en/](http://www.php.net/manual/en/)

It's a nice beginner language, because it doesn't worry as much about data types, it's less picky about syntax, and it generates very helpful errors along the way when you make mistakes.

Cheers
Kirk

### Post by drmrgd on 2012-11-21
It may also be helpful to correlate your first programming language with your interests.  For example, if you're leaning toward or very interested in Web development, I'm not sure learning C would be as beneficial as learning Java, PHP, and / or Perl (for example).  If you're interested in science (for example work as a bioinformatisist), I think that Perl is an absolute must.  

Once you get a good grasp on a language (and personally I'd recommend sticking with one until you're fairly comfortable with it instead of trying to learn 2 at the same time), it's not all that difficult to transition to a new language.  It's simply a matter of using the basic constructs that you learned (loops, tests, etc.) with a new syntax.  

I'd also recommend learning shell scripting as you'll find it very helpful to working with the terminal as you're developing programs and practicing your coding skills.  Plus bash scripting is fairly straightforward and simple, with a minimal amount of confusing syntax and complex expressions (at least to start!).

### Post by ex-oficio on 2012-11-21
Hi,
 
some points that others may have missed:
 
I would suggest that whatever you go for, choose a language that is well-supported by an integrated development environment: many dinosaurs will tell you to do everything in vi or emacs and compile step by stem at the terminal. I say if you want the worst intro possible and if you want to get stressed out and give up, take their advice.
 
I assume you're on linus, in which case.....
 
If you want to do C++ (which I would strongly recommend as the best "fully compiled" language out there from namy points of view) install netbeans with the C++ plugin. It will make your life so much easier, and allow you to focus on learning instead of figuiring out the tools.
 
if you want to get started quickly, and learn a language that is object oriented, generic and quite easy, Try either Java or C# (mono on linux). I will be slated or saying this by purists but they are both basically the same, both conceptually and in terms of syntax and look and feel. They are good choices if you want to create programs quickly, if you want to make user interfaces, or if you want to get into a career in programming as there is a lot of demand for skills in these languages.
 
Java is nicely supported in the eclipse development environment, while C# is best supported by the monodevelop environment.
 
Again, I will probably be jumped upon for being oput of fashion, but if you want to write something that crunches numbers FAST, and you know you will never need a GUI, give fortran95 a try. I learnt my trade on fortran, and I honestly feel I couldn't have had a better introduction to the programming world.
 
In the end, unless you want to go down to the level oif assembler, the C programming language, is, and always will be the ultimate. C is what is ultimately used to make all the other languages possible.
 
Finally, as others have suggested, if you are interested in web/browser based programming, then you need to get into html, php and stylesheets. work your way through downloading and installing a free content management website kit like joomla or wordpress on apache, then look into how its all working, and you'll be well on your way....
 
Why listen to me? you dont have to, but I'm a professional and I write everything from compute intensive physics models to graphical user interfaces for defence clients. I've been at it for close to 10 years, and I'm considered good at it.
