### Post by mephisto56 on 2009-09-22
If I ./configure a program to install in a custom directory with --prefix, does it install everything in there or are some files (eg configuration files) still installed somewhere else (eg home directory)? Is there a convention that says what will be installed in the custom directory and what still can be placed somewhere else with ./configure?

---

### Post by bmorency on 2009-09-22
Where are you planning on installing your program? Usually the default is /usr/local. If you keep that path it will install your config files under /usr/local/etc. your libs under /usr/local/lib and so on. There is also a bin folder under /usr/local where it will put the executable. It usually tries to keep everything under /usr/local. Sometimes if you look at the help for ./configure you can also set where you want all the files. I have never really tried something else but it might make the required folders under your custom install path.
