### Post by Scott_Lindner on 2018-09-28
I used Ukuu to upgrade my kernel to see if I could solve a screen flickering issue. That didn't work and it broke VirtualBox support so I reverted back to the Ubuntu kernel 4.15.0-34.37. After doing this, I cannot log in. I authenticate, it pauses for a short moment and goes right back to the login screen. If I boot into recovery mode for kernel 4.15.0-34.37 I can login and if I boot into normal mode for kernel 4.15.0.33.36 I can login. I created another user and it happens to that user as well. If I ssh into the machine I authenticate fine and the shell doesn't report any errors. I'm using zsh if that matters.

Any thoughts what's going on and how to fix it? Thanks!

---

### Post by Scott_Lindner on 2018-09-28
I figured it out. I had my display driver mismatched or something like that. I booted into recovery mode for the latest kernel. Then opened Software & Updates / Additional Drivers and changed the driver and changed it back to the driver I wanted. Restarted and boom. All worky worky again.
