### Post by Objekt on 2022-03-16
I have Ubuntu 20.04.3 LTS installed on an Acer Swift 3 laptop (model SF314-42-R6T7).

Initially, suspend did not work right.  The system would go into suspend by pressing the power button, but took an excessive amount of time - around a minute or so - to wake from suspend after pressing the power button.

The first couple of times this happened, I thought the laptop was broken or had run its battery completely dead.

I upgraded the kernel from the existing 5.13.0-35 to 5.16.13-051613.  Suspend started working as expected, but now I get a torrent of messages like this:
```
[114.585803] bpfilter read fail: 0
```for a minute or more  before the system actually restarts or shuts down.  

To be clear, I do not get the same message over and over - it's more like, the number on the left (which seems related to time since startup) increments, while the message remains the same.

This happens for a minute and a half or so before the system finally shuts down or restarts.

So, restarts have become super annoying.

FWIW, the behavior is very consistent.  using the GRUB menu, if I reboot into an older kernel (5.13.0-30 or 5.13.0-35), I get broken-ish Suspend behavior, but no nearly-endless "bpfilter" messages on restart or shutdown.  Restart with 5.1.13-051613?  Suspend works, but the bpfilter messages are back.

Is there a fix for this?  Or will i just have to put up with it until the next LTS release, 22.04 scheduled for late April?

Fwiw, it seems that on kernels that replace iptables with nftables
programs calling into iptables will cause this bpfilter dmesg logging.
Source: [https://lore.kernel.org/lkml/20200727142114.uzxr4kl7xdj2ildi@wittgenstein/](https://lore.kernel.org/lkml/20200727142114.uzxr4kl7xdj2ildi@wittgenstein/)

```
sudo dmesg | grep bpf

```
Well very few then:
```
sudo iptables -S
-P INPUT ACCEPT
-P FORWARD ACCEPT
-P OUTPUT ACCEPT
-A INPUT -s 37.19.212.147/32 -i wlan0 -j ACCEPT
-A INPUT -s 37.19.212.147/32 -i eth2 -j ACCEPT
-A INPUT -s 37.19.212.147/32 -i enp5s0f4u1u1 -j ACCEPT
-A INPUT -s 37.19.212.147/32 -i eno1 -j ACCEPT
-A INPUT -i wlan0 -j DROP
-A INPUT -i eth2 -j DROP
-A INPUT -i enp5s0f4u1u1 -j DROP
-A INPUT -i eno1 -j DROP
-A OUTPUT -d 37.19.212.147/32 -o wlan0 -j ACCEPT
-A OUTPUT -d 37.19.212.147/32 -o eth2 -j ACCEPT
-A OUTPUT -d 37.19.212.147/32 -o enp5s0f4u1u1 -j ACCEPT
