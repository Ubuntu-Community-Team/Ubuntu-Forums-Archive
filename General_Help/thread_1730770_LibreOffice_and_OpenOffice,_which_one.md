### Post by lykwydchykyn on 2011-04-16
Well, previously the version of OpenOffice in Ubuntu (and most other Linux distros) was not OpenOffice.org from Sun/Oracle, but a patched/modified version from the go-oo project.

When LibreOffice forked from OpenOffice, all the go-oo patches were folded in and most of the developers working on go-oo joined LibreOffice.

So, from a code perspective, you were more or less running LibreOffice before there was a LibreOffice.  It just has its own name and artwork now.

Nobody knows what's going to happen to OpenOffice at this point, since Oracle just announced they are giving it "back to the community", whatever that means.

### Post by rg4w on 2011-04-16
The LibreOffice project was started in response to concerns that Oracle might stop or at least hamper development of OpenOffice.

But now that [Oracle has given OpenOffice to the community]("http://ubuntuforums.org/showthread.php?t=1730112"), LibreOffice would seem to be struggling to find a reason to maintain the expense of a fork.

Last I heard the decision to switch from OO to LO in 11.04 was still up in the air, and with these latest developments from Oracle it would seem even more cumbersome to switch to LO just to go back to OO if LO is no longer needed.

As the other posters here have noted, at this point OO and LO are pretty much the same code base with minor cosmetic differences; the file formats and key features are the same, so it probably doesn't matter which one you use.

What hasn't been noted as often in these forums is why the concerns about Oracle's intentions with OpenOffice don't also apply to MySQL, VirtualBox, or any of the other FOSS projects they acquired with Sun.

If these concerns were significant enough to warrant such an expensive undertaking as forking OO, why isn't there a rush to also fork VirtualBox or MySQL?

### Post by lykwydchykyn on 2011-04-16
> **rg4w said:**
> 
As the other posters here have noted, at this point OO and LO are pretty much the same code base with minor cosmetic differences; the file formats and key features are the same, so it probably doesn't matter which one you use.



The differences are mostly cosmetic between the LibreOffice and the OpenOffice shipped by Ubuntu (well, apart from the version difference - 3.2 vs 3.3); they are NOT merely cosmetic between LibreOffice and the OpenOffice shipped by Oracle.

There are a lot of extra features and fixes in go-oo/LibreOffice over Sun/Oracle OpenOffice.  People never realized they were using a different version because prior to forming LibreOffice the name and branding was not changed, even though the code was.

If you want to see the tangible differences between the two, take a look at the asterisked items on this page: [http://www.libreoffice.org/download/new-features-and-fixes/](http://www.libreoffice.org/download/new-features-and-fixes/)
