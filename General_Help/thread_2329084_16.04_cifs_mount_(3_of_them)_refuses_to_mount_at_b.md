### Post by jcollum-4 on 2016-06-27

```
//192.168.x.x/music /media/nas-music cifs username=media,password=xxxx,iocharset=utf8,sec=ntlm,gid=1000,uid=1000,file_mode=0777,dir_mode=0777 0 0

Everytime I reboot the system I have to SSH in and re-mount these drives. It's annoying! I don't recall having to do this with previous versions of Ubuntu but I wasn't using a NAS then. It's a Synology. Could that be the culprit? I'm out of ideas.

### Post by bab1 on 2016-06-28
It's a timing issue.  The network is not up when the initial mount command (using fstab) is invoked.  This is systemd not Upstart.

### Post by vanadium on 2016-06-28
To work around, you could add the "noauto" option. This just "declares" the mount in /etc/fstab. Then, you can automatically perform the actual mount adding a "simplified" command in /etc/rc.local, which is executed a bit later:
```

mount /media/nas-music

```

This probably will be fine, but if you need to delay any longer, you can work with "sleep":

```

sleep 1 && mount /media/nas-music &

```
