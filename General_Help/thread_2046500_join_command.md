### Post by drmrgd on 2012-08-23
I don't know what the elegant solution for this is, and I'm interested to see the better solution.  The problem you're having is that with a normal sort, 950 > 1000.  So, just running normally, you'll get an error when running join:

```
join: file2.txt:51: is not sorted: 1000    -162.65

```

In order to overcome this - and again, this is a little clunky - I found that if you sort both files first with a sort command (note: don't use the -n option to sort the files numerically!), and then pass them to join it'll work.  You can then just re-sort the joined file to get it back the way you'd want it:

```
sort file1.txt > file1a.txt; sort file2.txt >file2a.txt; join file1a.txt file2a.txt | sort -n > sorted_file.txt
```

There must be a better way to handle numeric fields in join, though.  I did try the --ignore-case option, but that doesn't seem to do it.

### Post by erind on 2012-08-23
With awk you can try something like this:
 
```
awk 'NR==FNR { a[$1]=$2; next } 
     $1 in a { print $0, a[$1] } ' [COLOR=Blue]file2 file1[/COLOR]
```Note that the order of files above is important:  **file2.txt **then [B]file1.txt.

### Post by yirgacheffe on 2012-08-25
Thanks everyone for the suggestion, it ended up being done with awk. Join would have required adding leading zeroes for to all 3 digit numbers in the first column.

Awk solution was ( as erind's suggestion): 


```
awk 'NR==FNR{a[$1]=$2;next}(a[$1]){print $0,a[$1]}' file2 file1 > result
```
