### Post by eggdeng on 2010-06-13
I take it you are trying to connect via wifi (wlan0). Try
```
sudo ifdown wlan0
```
then
```
sudo ifup wlan0
```
to restart the interface.

To see if you can get the network manager running, try first
```
killall gnome-panel
```
then
```
nm-applet
```
 
Is your network using DHCP?

### Post by eggdeng on 2010-06-14
The output from ifconfig suggests that the wireless interface is configured (but is not getting an IP) and you really need to be able to use the Network Connections dialogue to do anything about it :-[ Is it still unresponsive?
Try killing the network manager applet
```
killall nm-applet
```
to see if that's what's holding things up. You can restart it with
```
nm-applet
```
You might try a forum search for your wireless card to see if other people have had problems. To identify the card, do
```
lspci | grep Wireless
```
if you get no result, just do
```
lspci
```
and browse the output.
Sorry not to be of more help.

### Post by 3rdalbum on 2010-06-16
I think it may be a problem with DBus:

> ** (nm-applet:22583): WARNING **: <WARN>  constructor(): Couldn't initialize the D-Bus manager.

This can stop Network Manager from working. Maybe try this command"

```
dbus-daemon --system --fork
