### Post by QIII on 2015-10-08
Hello!

There are so many combinations it would be impossible to give you a list or what's best.  Then there is the question we always ask:  Best for what?

If the default open source Radeon driver is not able to sufficiently provide 3D support for running Unity (which it should...), there is still another, probably cheaper, answer:  Get an inexpensive graphics adapter currently supported with a proprietary Linux driver from the manufacturer.  If you want to stick with AMD, that would be any HD 5000 series card or later.  

An HD 5450, which is passively cooled, would be the very lowest end card that would suit the bill.  You wouldn't be able to game, but normal video performance would be sufficient for most general tasks.  The HD 5450 usually goes for < $50(US).  I found one [here]("http://www.newegg.com/Product/Product.aspx?Item=N82E16814102882") for $40.

Some folks may come along and say that AMD dumps Linux support for its cards all the time.  Not all the time.  That has happened twice.  Once while it was ATI (before it was bought by AMD) and once when AMD was clearing out support for the chipsets that had been designed by ATI and were carried into their new stock.  Could AMD drop support?  Sure.  But at $50 it might be worth the risk.

To my mind, an AM3+ board that will handle a 125W FX processor is definitely not old enough to put out to pasture.  One of my machines runs a Core 2 Quad in a socket 775 board with 4GB of DDR2 memory and an AMD HD 5450.  It runs like a champ.
