### Post by Rubi1200 on 2011-01-02
Hi,
I am not sure what you have tried so far, but you cannot have GRUB files on sda1. This is the XP partition.

You need to first restore the Windows bootloader and then reinstall GRUB.

Restore Windows bootloader:
[http://ubuntuforums.org/showthread.php?t=1014708](http://ubuntuforums.org/showthread.php?t=1014708)
Reinstall GRUB:
[https://help.ubuntu.com/community/Grub2#Reinstalling](https://help.ubuntu.com/community/Grub2#Reinstalling) from LiveCD

Mount the Ubuntu partition on sda5 and install GRUB to the MBR of **sda**

Don't forget to update GRUB once you are booting Ubuntu again.

### Post by drs305 on 2011-01-02
Your Grub2 files look like they are in the correct location (other than an isoated grub.cfg file in sdc1.

If you can boot a LiveCD, run these commands:
```
sudo mount /dev/sda5 /mnt
sudo grub-install --root-directory=/mnt /dev/sda
```
That should ensure G2 has the MBR pointing to the grub.cfg in sda5. Reboot. 

If it still doesn't boot to Ubuntu, tell us what happens during and after it attempts to boot.

Also try holding down the SHIFT key if you don't see the Grub2 menu. If you get to the menu, press "c" to get to the grub prompt, and run:
```
ls (hd0,5)/boot/grub
```
If you get nothing, try (hd1,5) or (hd2,5).

Do you see a lot of *.mod files? If not, reboot, enter your BIOS and find the screen that displays your disk size. If you find it says the drive is 137GB or less, it means your BIOS can't see the G2 files in their current location. You will either have to enable the 'large disk' feature of your BIOS or update it to a newer version which can see today's larger drives.

### Post by drs305 on 2011-01-04
I've gone over your posts several times and my best *guess* is that your BIOS is probably only capable of seeing the first 137GB of the drive. I think this because you say your original disk was 100GB, you could boot on an 80GB drive, and you get an "out of disk" error when looking for partitions at the grub prompt.

Options:

1. Since you have only used 2 of the 4 allowed primary partitions (Windows and the extended partition) you could shrink Windows by 200MB or so and make it a /boot partition for Ubuntu. It would probably take several hours to move your Windows partition to the 'right' to create a new partition at the start of the disk. And there is always the slight possibility something could go wrong and you could lose your data. 

If you decide to do this, resize and move Windows with Windows software.


2. Make a boot partition on one of your USB drives and install G2 to the flash drive. It means you will have to always have the USB inserted when you boot.

3. You mentioned a 100GB and an 80GB drive. If they are available, use one of them for Ubuntu.
