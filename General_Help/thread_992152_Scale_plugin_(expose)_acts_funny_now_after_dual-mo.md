### Post by loomsen on 2008-11-24
Jeez!

Did you screw this xorg that way? :)

So, you've got the driver installed? If so, then remove your actual xorg.conf, if it's the one posted above. Rarely seen such a messed up one :)

You've got, for instance, so many nice custom Modelines in it, just, you don't use this section.

Look, let's get this sorted somehow.(I omit Input devices for simplicity)

Your xorg basically specifies which video to use in conjunction with which monitor, to have a Screen.

So, you ServerLayout defines what to use. Basically which Sections should work together to provide the Server.

Alright, let's specify a video device then first.

```

[color=blue]## THIS IS A COMMENT ## BY PREPENDING YOU CAN ADD COMMENTS
## Now we wanna specify a Video Device to use, right? So we start a
[/color]
Section "Device"
    Identifier     "Device0"    [color=blue]# This value is used in the ServerLayout l8r on[/color]
    Driver         "nvidia"    [color=blue]# You got a nVidia card, so lets use this driver[/color]
    VendorName     "NVIDIA Corporation" [color=blue] # Some more info[/color]
    BoardName      "GeForce 7800 GT"    [color=blue] # Some more info[/color]
EndSection

```

OK, done, you may specify even more, available Options for this Sections:
```
man xorg.conf
```
If you feel like you have to, but this should be enough ftm.

Now, let's get a Monitor where our Device can draw to:
```

Section "Monitor"
    Identifier     "Monitor0"
    VendorName     "Unknown"
    ModelName      "Envision H193Wk"
    HorizSync       30.0 - 83.0
    VertRefresh     55.0 - 75.0
EndSection

```
Pretty self explainatory, else --> **man xorg.conf**

OK, got a Device and a Monitor, let's get ourself a Screen set up
```

Section "Screen"
    Identifier     "Screen0"
    Device         "Device0"  [color=blue]## The one from our first section above[/color]
    Monitor        "Monitor0"[color=blue]## The one from our second section above[/color]
    DefaultDepth    24
    Option         "TwinView" "1"
    Option         "TwinViewXineramaInfoOrder" "CRT-0"
    Option         "metamodes" "CRT-0: nvidia-auto-select +0+0, CRT-1:1440x900 +0+0"
EndSection

```

Now, if you take another look, you want to use two Monitors, but we have only one Section Monitor so far. So specify another one.

Adjust your resolutions, or just set the 2nd one to nvidia-auto-select too.(which is default anyway if no valid mode is found)

Alright, basically thats it. Now, how should our X-Server look?
Don't get misled by the word Server, what he actually provides is an Interface for our different INPUTs and OUTPUTs. 
This makes our environment more than BASIC 
(Until your X starts up, you have ONE Input Device and ONE Output device, keyboard and monitor in a shell)

NOTE:
BIOS=BASIC INPUT OUTPUT SYSTEM

```

Section "ServerLayout"
    Identifier     "Default Layout"
    Screen      0  "Screen0" 0 0
EndSection

```

And you're done with your basic setup. Anything else are additional Options/Modules/Extensions you may or may not specify.

But actually I hope you undertstood how X is defined.

The manual will give you further options.

man xorg.conf
