### Post by squaregoldfish on 2008-09-27
> **Cyanide Cloud said:**
> WHY???  Thats just...  Thats...  Stupid as hell if it doesn't  :shock::shock::shock:

You'd have to ask the authors, but the most common answer is simple supply and demand, and a lack of time. This is the problem that faces all open-source software.

POV-Ray started out life as a command-line tool on all platforms. After many years, they decided to add a nice GUI front end. Since there's only a few developers who aren't being paid for their time, this detracts from work on the ray-tracer itself. While it was deemed worthwhile to spend the time on this for Windows users, who probably make up the vast majority of their user base, it's simply not worth putting so much development time into similar features for the few Linux users that there are.

Ultimately, it's not stupid as hell, it's just pragmatic. Since you're not paying for the software, and the developers aren't being paid to make it, you have to accept that things like this will happen. You can request that features are added, but if no-one has the time to do it, it won't get done, unless someone new volunteers to do it.

Don't despair though. Several people have felt the same as you, and developed tools to make using POV in Linux a little easier. They're listed on the POV website here: [Useful tools for POV-Ray for Unix]("http://www.povray.org/download/unix-tools.php")

Steve.
