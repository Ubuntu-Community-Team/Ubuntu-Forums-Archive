### Post by jibber_sc on 2007-06-07
I did some google searching and came back and now there are two replies.
Well, I found my problem, but thanx for the help anyway.

My problem was that my xorg.conf file did not have the horizontal synchronization or vertical refresh rate.
This is what the monitor section looked like:
```
Section "Monitor"
	Identifier	"MX75"
	Option		"DPMS"
EndSection
```

I ran 'dpkg-reconfigure xserver-xorg' again, making sure to keep a backup.
As my first post said, doing this messes up my 3D Driver.
So, I copied the HorizSync and VertRefresh from the new xorg.conf and moved it to the backup, and changed the backup back to xorg.conf.
Basically, I ran 'dpkg-reconfigure xserver-xorg' just so I could get the values for HorizSync and VertRefresh.
So my new monitor section in xorg.conf looked like this:
```
Section "Monitor"
	Identifier	"MX75"
	Option		"DPMS"
	HorizSync	28-49
	VertRefresh	43-72
EndSection
```
And now I can change my resolution to 1024*768.

Supposedly the problem is that higher resolutions need higher HorizSync and VertRefresh.

### Post by apetresc on 2007-08-08
> **jibber_sc said:**
> Supposedly the problem is that higher resolutions need higher HorizSync and VertRefresh.

It's actually the other way around :) The higher your resolution, the harder it is for monitors to refresh it as quickly (since there's more to refresh, but less time to do it in). What was probably happening is that X was choosing some default values for HorizSync and VertRefresh which were too high for any resolution beyond 800x600. What you set is almost certainly LOWER than the defaults that X.org was choosing by default, which is good, because it allowed the monitor to also support higher resolutions :)

Glad your problem was solved (you should add "[SOLVED]" to the topic title ;))
