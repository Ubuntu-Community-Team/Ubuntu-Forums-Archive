### Post by ulrich on 2004-11-03
though i use linux for over 3 years as main-os now, i never tried debian (ouch:)
but i think most stuff you learn about debian can be transferred to ubuntu. 

as for linux is just the kernel, the main os and its behavior is about the distro you use. so i think if ubuntu is debian-based, it should behave mostly as debian does and therefore should use the same 'base'.

which leads to your next question about redhat, suse...
suse/redhat are rpm-based and deb/ubuntu uses deb for package management.
the biggest difference for me as an average user is the fact, that all distros using different schemes to store/organize the system.
so mostly the stuff thats in /etc. 
how and where are config-files, start- and stop-scripts....

as said: im just a **user**, no network/shell-guru, etc... 

hth, and have fun with oss. its the beauty of the computer-industry.
