### Post by jsnlevi on 2010-06-13
Lots of gnome-related (I think) wonkiness here. This is a fresh install of 10.04 x64 on known good hardware in a custom laptop built off of an MSI 163k barebones chassis/mobo. 

First issue (not a huge deal): the notification area of the gnome panel keeps rearranging. Upon login, I never know what order my username/logout button/time and date/network manager/volume manager are going to be in. Sometimes the logout button is at the far right edge, sometimes it's the date, sometimes it's the network manager. It seems totally up to chance, but is only mildly frustrating.

Second issue (very annoying) the Firefox window will intermittently be selected by mousing over it. This becomes exceptionally frustrating when working with multiple windows in the same workspace as Firefox and I can't select any other windows because they get thrown to the back and deselected as soon as I pass over the browser. As far as I can tell, only Firefox is behaving this way, and only intermittently. The temporary workaround has been to keep Firefox in a separate workspace, which is not ideal.

Third issue: every couple of boots, my screen resolution is wrong and has to be reset to native. The really funny thing is that some applications, like the screenshot tool, will assume that my ratio is the native 16:10 while the desktop is running in an incorrect 4:3, which leads to windows stretching off-screen and screenshots having stretches of blank space on the sides. I am working to fix this issue now, but wanted to address it directly in this thread.
