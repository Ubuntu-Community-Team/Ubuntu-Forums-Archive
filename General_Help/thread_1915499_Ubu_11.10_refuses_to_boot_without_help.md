### Post by efflandt on 2012-07-04
I suspect your problem has to do with using one grub in a single mbr from 2 different Ubuntu versions.  So if one or the other updates the entire grub package, which version ends up in the mbr, and in which Ubuntu does grub look for the rest of itself and grub.cfg?

I don't know if there is a way to disable or uninstall grub in one of the Ubuntu versions.

Otherwise maybe from 11.10 make sure that its grub is in the mbr, then from 10.10 try to put its grub on sda3 (where it will not actually be used).

From 11.10 (to mbr of sda):[FONT=monospace]
[/FONT]```
sudo grub-install /dev/sda
sudo update-grub
```From 10.10[FONT=monospace] (note sda3):
[/FONT]```
sudo grub-install --force /dev/sda3
sudo update-grub
```On my computer I have:
sda (std Windows mbr)
sda1 (Dell Utilities)
sda2 (Recovery)
sda3 (64-bit Win7)
sda4 [was 64-bit 10.10 (now 12.04) and its grub]

SSD
sdb (grub of 11.10 boots everything)
sdb1 (64-bit 11.10)

no swap (8 GB RAM)

### Post by YannBuntu on 2012-07-04
**@efflandt:** no need to install in the PBR (sda3 for ex). And DON'T remove GRUB from any Ubuntu!
If you have 2 Ubuntu systems, each one has some files (sort of "stage2) in its system partition. The one you will see at boot is the one you install in the MBR (eg if you do sudo grub-install /dev/sda from Ubuntu1, you will see the GRUB of Ubuntu1 at boot, and it will allow you to boot also Ubuntu2).

### Post by YannBuntu on 2012-07-09
ok.
Here is what i would try:
1) boot on 12.04, open a terminal (Ctrl+Alt+T), and type:
```
gksudo gedit /etc/default/grub
```

2) In the file that opens, uncomment (remove # at the start of the line) the GRUB_TERMINAL line. 
3) Save the file.
4) In the terminal, type:
```
sudo update-grub
```
5) Reboot and indicate what you observe

6) If still not good, redo this process (1 -> 5), but this time replace this line:
```
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"
```
by:
```
GRUB_CMDLINE_LINUX_DEFAULT=""
```
