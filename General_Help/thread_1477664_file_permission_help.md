### Post by chappajar on 2010-05-09
For a FAT/NTFS disk you can use gid= or uid= to set the owner/group of the disk, and/or use umask= to set a permissions mask for the disk in /etc/fstab 

For an ext* disk just add the 'user' option to it in /etc/fstab.  Note that this also sets 'noexec,nosuid,nodev', so you may also want to add 'exec,suid,dev'

### Post by chappajar on 2010-05-10
So if I am understanding you correctly, you are having trouble with 01-my-data and 02-my-data, nothing else right?

Why are you editing mtab? Don't change mtab, change fstab.

I suggest changing the two problem partition entries in /etc/fstab to this:

# Entry for /dev/sda1 :
UUID=b7035b65-1e3f-45c8-a531-d19d7dd662f4 /home/user8/01-my-data ext3 relatime,user,rw,suid,dev,exec,auto,async 0 0
# Entry for /dev/sdd2 :
UUID=fb0ffa6c-040b-4e28-88c4-dcdf48a1857f /home/user8/02-my-data ext3 relatime,user,rw,suid,dev,exec,auto,async 0 0

Restart (don't just logout/in) after changing /etc/fstab.

If this still doesn't work then try replacing(or adding) the 'user' option with 'owner',
or,
replace 'user' with 'users' and make sure user8 is in the users group.

I don't know what effect your edits of mtab will have, but if nothing is working after trying all of the above it may be because of what you have done to mtab.  You can try to restore mtab with this:
sudo [FONT=monospace][/FONT]restorecon '/etc/mtab'
but I've never done this myself before, so I don't know what it will do exactly. Use as a last resort!
