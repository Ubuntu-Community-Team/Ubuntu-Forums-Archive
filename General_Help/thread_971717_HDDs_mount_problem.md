### Post by taurus on 2008-11-05
Instead of using /dev/sda1, /dev/sdb1, and /dev/sdc1 in /etc/fstab, try using the UUIDs for the drives.  You can get the them from

```
ls -la /dev/disk/by-uuid
```

### Post by geirha on 2008-11-05
They switch around everytime you boot? That sounds a bit odd. Though, instead of using the device names /dev/sdb1 and /dev/sdc1, use their respective UUIDs. The command ```
sudo blkid
``` will list the UUIDs for all filesystems.
```
UUID=1234abcd... /media/disc1 ntfs-3g ...
```

### Post by geirha on 2008-11-05
If it's not listed in /etc/fstab, it will not be mounted automatically on boot, it will show up in nautilus though, and once you click it, it will be mounted by gnome-volume-manager.

I'd recommend adding it to fstab with a line ending in "0 2" so that the filesystem will be checked during boot once in a while.
