### Post by drs305 on 2011-10-31
I'm pretty sure this isn't a Grub issue but one occurring after control is passed from Grub to the system. Nevertheless there are a couple of things you can check since RESULTS.txt can't pinpoint the exact location of the files.

At the Grub menu, press 'c' to get to the Grub prompt. If you don't see the menu, hold down the SHIFT key during boot until it appears. 

Make sure Grub is able to find all the boot files. The text after the # symbol is what you should be included in a return:
```
ls (hd0,1)/  # vmlinuz, initrd.img
ls (hd0,1)/boot # vmlinuz-2.6.38-12-generic-pae  initrd.img-2.6.38-12-generic-pae
ls (hd0,1)/boot/grub  # Lots of *.mod files

```
Assuming it found all the files, I would try to load up the kernel options disable as much as possible. Press ESC to get back to the menu, then press 'e' to edit the main menu. 

Cursor to the end of the 'linux' line and remove "quiet splash vt.handoff=7" and add the following: "nomodeset noapic nolapic acpi=off"
Press CTRL-x to boot. If it boots, try removing these options one at a time until the boot fails.

If one or a combination of the above works, we can make it permanent later.

Here is a link to other kernel options:
[http://www.kernel.org/doc/Documentation/kernel-parameters.txt]("http://www.kernel.org/doc/Documentation/kernel-parameters.txt")

If none of these work, are you able to boot to the Recovery mode?
