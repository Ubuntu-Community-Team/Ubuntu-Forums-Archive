### Post by oldfred on 2022-02-22
For my Samsung NVMe drive, I downloaded a bootable ISO with new firmware. It looked like an old DOS screen and was only for my model and newer firmware.

To see firmware version:
sudo apt install nvme-cli
nvme - the NVMe storage command line interface utility (nvme-cli)
nvme --help
sudo nvme list

How to update Samsung:
[https://wiki.archlinux.org/index.php/Solid_state_drive](https://wiki.archlinux.org/index.php/Solid_state_drive)
[https://www.samsung.com/semiconductor/minisite/ssd/download/tools/](https://www.samsung.com/semiconductor/minisite/ssd/download/tools/)
Look at firmware, not magician. Download ISO and create a bootable flash drive from it. 
I was able to loopmount it from grub like I do for all my ISO now, so did not use a flash drive.Surprised loopmmount worked.

### Post by TheFu on 2022-02-22
After updating the firmware, run a SMART test, give it some time to complete, then run a SMART report on the device.
I've seen a Samsung SSD fail that was 30 days old. The guy got it replaced and hasn't had any issues the last 18 months.  I do like Samsung SSDs, have 2 here myself.

**smartclt** is the command. It is part of a smartutils package, I think.
