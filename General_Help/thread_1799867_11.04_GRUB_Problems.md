### Post by oldfred on 2011-07-08
Did you do a full purge & reinstall of grub2. You seem to be missing init line and parameters. You probably have to manually add the parameters to gksudo gedit /etc/default/grub. Are you perhaps updating an install in sda and your install is in sdb or vice-versa?

I would try a full purge & reinstall, if you can boot you do not have to chroot:

chroot & grub uninstall & reinstall -drs305
[http://ubuntuforums.org/showthread.php?t=1581099](http://ubuntuforums.org/showthread.php?t=1581099)


If you want to have a backup way to boot you can add an entry to 40_custom, it uses link in root to boot.

gksudo gedit /etc/grub.d/40_custom
Then do:
sudo update-grub

```
menuentry "Boot on sdb1" {
    set root=(hd1,1)
        linux /vmlinuz root=/dev/sdb1 ro quiet splash
        initrd /initrd.img
}

```
