### Post by taurus on 2006-12-21
Boot into recovery mode from GRUB menu and edit your /etc/X11/xorg.conf.  Replace "nvidia" back to "nv" and reboot...

```
nano -w /etc/X11/xorg.conf
```

old:
```
Driver               "nvidia"
```
new:
```
Driver               "nv"
```
reboot:
```
shutdown -r now
```

### Post by Valinski on 2006-12-21
I also had that problem. It stumped me for ages then i found a post that told me to remove some "Restricted modules" in the package manager. its Linux-restricted-modules and in it you will find this;
Non-free Linux 2.6.17 modules on 386
This package provides restricted modules for Linux version 2.6.17 on
386.
Currently the following modules are included:
 - madwifi (Atheros)
 - fglrx (ATI)
 - nvidia
 - fcdsl, fcdsl2, fcdslsl, fcdslslusb, fcdslusb, fcdslusb2, fcdslusba,
   fcpci, fcusb, fxusb (AVM ISDN)
 - ltmodem (Winmodem)

These modules are "restricted" because they are not available under a
completely Free licence.

As you can see the nvidia driver is restricted. I uninstall this and it solved the problems. X server booted on a reboot.

### Post by taurus on 2006-12-21
Okay, you are using a generic driver for your video card, vesa, so you need to change that to "i810" for your graphic card then.  From a terminal, run

```
gksudo gedit /etc/X11/xorg.conf
```
Then, scroll down to the line that looks like

```
Driver       "vesa"
```
and change that to 

```
Driver       "i810"
```
Save it and either reboot or restart X with <Ctrl><Alt>Backspace...
