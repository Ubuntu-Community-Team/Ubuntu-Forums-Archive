OK, let's start by removing the program you downloaded
```
sudo apt-get remove xserver-xgl
```

### Post by Rocket2DMn on 2008-02-10
OK, I think you might be out of luck for using the intel driver.  It seems that your chipset just doesn't have full support in Ubuntu right now.  Intel has release drivers for your setup, but they are not available directly through Ubuntu yet.  You can compile them yourself (which will also require recompiling the kernel if you're up to it) - but we can't really provide support for it which means you're essentially on your own if you choose to.
[http://intellinuxgraphics.org/](http://intellinuxgraphics.org/)
[http://intellinuxgraphics.org/download.html](http://intellinuxgraphics.org/download.html)
[http://intellinuxgraphics.org/install.html](http://intellinuxgraphics.org/install.html)
It is not for the timid or faint of heart. 

You may just want to try the vesa driver - you can just edit xorg.conf manually for that
```
gksudo gedit /etc/X11/xorg.conf
```
And replace "intel" with "vesa" under 
```
Section "Device"
Identifier "Intel Corporation Mobile GM965/GL960 Integrated Graphics Controller"
Driver "intel"
BusID "PCI:0:2:0"
Option "UseFBDev" "true"
EndSection
```
Then restart X with CTRL+ALT+BACKSPACE.  If that fails, do the reconfiguration and select "vesa" from there.

Here is a link from the Development (Hardy) forum about your chipset - [http://ubuntuforums.org/showthread.php?t=686638](http://ubuntuforums.org/showthread.php?t=686638)
The developer says it might not be available until the release following Hardy (so another 9 months or so at least).  With any luck they will make the driver available through the backports repository in Hardy after awhile.
