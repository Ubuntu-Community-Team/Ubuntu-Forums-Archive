### Post by snowpine on 2009-10-01
Hi Monolith, a couple of things:

1. Memory usage in Linux is totally irrelevant as long as it's less than 100% and swap is not being used. 489 of 1010 MB is wonderful, no problem there. Unused ram is wasted ram.
2. You can get much lighter than "standard" Xubuntu by starting with a minimal install (using the mini.iso) and then installing basic packages like xfce4 instead of xubuntu-desktop.
3. Check out different distros like Debian or Sidux for a truly fast and lightweight Xfce implementation.

Good luck!

### Post by JC Cheloven on 2009-10-01
I had some time to spare this summer, and tried to build "my ideal system" starting from the ubuntu minimal c.l.i. install. 

My ideal system has a clean, minimalist-looking desktop, but with all the functionality of some light, well selected applications at my fingertips. I almost did it. The desktop manager of choice was openbox. As my little project grew, I found myself building something very similar to crunchbang, only without panels or "start button" (I like it this way). 
It happens to be that I don't like very much crunchbang itself (I had tried before, and wasn't as fast and responsive as I'd expect). So I gave up my project.

As said before, xfce is nowadays just an alternative, with about the same weight and functionality than gnome. My advice would be that if you want something lean, you'd better to look outside ubuntu. If you're a knowlegdeable person, Arch could be for you. If you see yourself as a regular user, then Puppy is fantastic (but in the client side, not in the server side).

BTW, my selected browser was epiphany, much ligther than firefox. That single thing can make the difference for a low end pc.

Hope to have been of any help.
