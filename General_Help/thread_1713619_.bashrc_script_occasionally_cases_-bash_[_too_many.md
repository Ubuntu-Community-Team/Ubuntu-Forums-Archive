### Post by l0uismustdie on 2011-03-24
I have a .bashrc script to shorten the length of my shell path and occasionally I get the error:
```

-bash: [: too many arguments

```I've tracked it down to this line
```

elif [ $HOME ==  ${PWD:0:${#HOME}} ]

```But can't seem to find anything wrong with it.

Anyone have any ideas?

---

### Post by DaithiF on 2011-03-24
most likely paths with spaces .. e..g
if [ /home/dave == /its/stupid/to use spaces/in/paths ]; then ...

would be an error because bash expects [ A == B ], not [ A == B C D ] 

enclose in quotes to have bash do the right thing.  
if [ "$HOME" == "${whatever...}" ]; then

that, and don't use spaces :)
