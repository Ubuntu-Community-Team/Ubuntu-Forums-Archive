### Post by AxelMan0 on 2010-12-16
I know it's been a long time since I started this topic, but I figured I should finish it off. After trying every distro imaginable: tinycore, dsl, deli, Lubuntu (wayy too big), arch linux, slackware, vectorlinux light, and several others, the ones that worked best were Puppy Linux 5.0 and ConnochaetOS (0.8.9b2) (a recent fork of DeLi Linux). All distros were run on a hard drive install, so that the computer could be used to it's fullest potential. A 1x CDROM drive does not make a good LiveCD experience, 1 USB port does not leave any to spare, and my goal was to have as many 'features' offered by the laptop available as possible. 

           Most (tinycore, dsl, DeLi, Lubuntu, Arch) failed to boot. VectorLinux Light could boot but frankly, it wasn't light enough. Slackware was a bit too advanced for me when I tried it, but it's a year later and I've got a CLI-only arch linux setup on my main box now that is loads of fun :D. A huge setback was the fact that this laptop cannot connect to the internet. The whole distro had to be relatively self contained as adding any extra packages meant I had to download them from another computer, put them on a flash drive, mount it, unpack the packages, try to install them, realize I forgot about dependencies, and then do it all over again.

            I attempted to put Arch on the old laptop with no GUI, which would clearly be the slimmest option, but for some  reason it hung on boot despite my greatest efforts. I tried compiling  LFS on it too but that did not work (yet). I tried cross compiling LFS  from my 64-bit lappy and I got oooohhhh so close, but couldn't complete  it (got stuck on the third-pass of Glibc on the target laptop).

           Puppy Linux 5.0 was a fun run, but it's quite an odd distro, and I wanted something I could start to learn the guts of a 'classic' linux setup with. I am unfamiliar with the entire '.pet' philosophy. Apparently they are marked with a certain ease of installation, but Puppy Linux seems to have quite a heavy reliance on having access to many .pets. This was a huge mark down for this distro since installing new packages, no matter how nifty the packaging system, is a pain in the butt without an internet connection (not to mention situations in which a .pet of the desired program is nowhere to be found). The use of single rather than double clicking always caught me off guard for the first few minutes of use whenever I turned it on as well.

           ConnochaetOS worked the best. It was extremely snappy (snappier than the 4 Ghz P4 windows XP my family has out in the living room). It's memory footprint was about 57 Mb out of the 64 while playing a CD with a large cache size (due to the 1x cd drive not being fast enough) on one VT, 3 open xterms on the GUI as well as the file manager, and at least 2 other VTs with something on the screen. about 30 Mg of that memory was Cache memory. I found ICEwm to be pretty neat, much better than Puppy's (JWM). It actually moved all of the window graphics rather than a border like Puppy, and felt more responsive at the same time. If the ICEwm menus and taskbars were not easier to configure than JWM's (and I think they were), it's configuration files were extremely well documented. The 'help' button in the main menu of ICEwm takes you straight to a detailed and well organized help file. Granted this help file is not complete yet, you have to give the ConnochaetOS team a break--this is not even a 1.0 release. When working in VTs I could move around the system easily and did not feel as if I was working 'out of bounds'. This is partly because most of the configuration can be done with text files (ala KISS) so you are not *forced* to use the GUI for anything. Puppy Linux defaulted to only 4 VTs, which for me feels a bit cramped.

           When using Puppy Linux (which was on there for a good few months) the laptop found it's use as a glorified cd-playing eReader. I put all my digital books on it and listened to a cd occasionally, and this was all the use I really had for it. When using ConnochaetOS I could play CDs and read books, but I was also able to experiment with (clearer) text configuration files, learn all about a new desktop environment and window manager thanks to the ConnOS documentation project, and even install some of the cross compiled packages I had made to entertain myself with some simple C programming. (Note: ConnOS did not actually come with any development programs, but the atmosphere it provided I feel was more friendly towards that sort of thing).

           This rant may not be the most scientific presentation, but I have tried to organize it as well as my scattered brain can. I am not so well informed about the Puppy way of doing things and I hope I didn't offend any Puppy die-hards out there, please correct me if I misrepresented the OS :). I hope this hodge-podge of personal preference and comparisons helps someone out there who wants to breathe life into a crappy laptop. At the very least we now have a nice list of potential distros for the Senile Silicon of the world. I know what my go-to OS is if I am ever asked to put Linux on the slowest of machines. ConnochaetOS gave me a modern feeling experience out of a 19 year old laptop. Kudos.
