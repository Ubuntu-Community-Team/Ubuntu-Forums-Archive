### Post by JBA2337 on 2010-01-10
I formatted my Linux Ext3 partition before installing Karmic Koala, version 9.10, so that I did a clean install of version 9.10. I have used the last 3 different versions of Ubuntu, 8.04, 8.10, and 9.04, with hardly any problems, but after installing version 9.10, I have had nothing but problems. The new version 9.10 definitely has problems that I never experienced with the other versions of Ubuntu. So I switched back to version, 9.04. 

Here is a list of the most bothersome problems that I have had with version 9.10.

(1) I had to burn the 9.10 iso file to a CD disk 3 times to finally get one that had no errors. (With version 9.04, I only had to burn one disk, and it still works perfectly). I used the same software (Nero) and the same computer to burn all of the CD disks, so I don't see how the bad 9.10 disks could be a hardware problem.

(2) When I tried to install version 9.10, the process failed at least 4 times before I finally got a successful installation. The computer simply locked up, and quit loading the program from the CD installation disk. On the exact same computer, this NEVER has happened with version 9.04.

(3) With version 9.10 there is no sound at all; I can't play CD's or mp3 files.

(4) In version 9.10 the new "Software Center" is, in my opinion, definitely not an improvement over the previous version. In version 9.04 the "Add/Remove Applications" program is much easier to use, and if I want to I can install or uninstall 15 or 20 software packages all at the same time. The new Software Center only allows me to install or uninstall ONE software package at a time. The new 9.10 "Software Center" is not an improvement, rather it is a step backwards. My only solution to this problem would be to only use Synaptic Package Manager to install or unistall software.

(5) Also, In version 9.10, when I re-boot into recovery mode, there is no File System Check (fsck) listed on the Recovery Menu. Why was  "File System Check" left out of the Recovery Menu for version 9.10? Leaving this feature out is an unbelievably bad idea. Once in a while I may have difficulty getting Ubuntu to boot-up properly. If I can't boot-up the ONLY way I can fix the problem is to boot into recovery mode and run the File System Check. Please put the File System Check back on the Recovery Menu! Not having this feature is a serious defect in version 9.10.

I definitely am not going to use version 9.10 for now. Maybe some of these problems can be fixed when 9.10 is updated. Most of the problems mentioned above have not been separately posted in the Ubuntu Forum. Rather than spend hours of time posting every problem and trying to fix them one-by-one, it was easier for me just to go back to 9.04. I know that 9.04 works well on both my computers. (I have a home-built generic desktop computer and a Toshiba laptop.)


JBA2337
