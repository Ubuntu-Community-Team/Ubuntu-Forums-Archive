Hello all

I'm running Lubuntu 13.10 using pcmanfm (1.1.2) and lxterminal (0.1.11). I have an issue opening a terminal from pcmanfm using the "f4" shortcut.
It seems like it does not work when there is a space in the directory name so for example if hit "f4" when in the directory: '/home/rob/example directory' lxterminal wont open.
It has been my one gripe with Lubuntu for a while now and I'm hoping there is a simple solution.

Apart from this I've had a great experience using Lubuntu, been at it since 12.04

### Post by Bashing-om on 2013-12-11
Bobalas; Hi ! Welcome to the forum.

In linux, a space is seen as a delimiter. Such that, I think, the file name   '/home/rob/example directory' is most likely seen as two files - example and directory - and the system does not know how to handle that situation.

The simple solution is to AVOID any spaces in file names. Those situations can be trying to deal with.
