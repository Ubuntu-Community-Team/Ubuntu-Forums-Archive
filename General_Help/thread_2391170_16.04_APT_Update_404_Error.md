### Post by hellblazer13212 on 2018-05-06
@Cruzer had nothing to do with my sources list... sources definitions end at the "Main/Restricted/etc". However, you pointed me in the correct direction to aim "the google". Turns out the architecture is defined in dpkg and somehow arm got added in there. For anyone else having this problem, I was able to find the resolution [here]("https://askubuntu.com/questions/705895/how-to-fix-a-failed-to-fetch-binary-armhf-packages-error-during-apt-get-update"). Also pasting the fix below:

[COLOR=#111111][FONT=Ubuntu]You might install the armhf cross tools which add armhf as foreign architecture (your architecture maybe i386 or amd64). You can launch this command to verify:[/FONT][/COLOR]
```
dpkg --print-foreign-architectures
```
[COLOR=#111111][FONT=Ubuntu]If the output includes armhf, you can simply remove it by running the command:[/FONT][/COLOR]
```
sudo dpkg --remove-architecture armhf
```
[COLOR=#111111][FONT=Ubuntu]Finally, launch your sudo apt-get update again.[/FONT][/COLOR]
