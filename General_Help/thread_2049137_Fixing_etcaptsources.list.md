### Post by drmrgd on 2012-08-28
If you launch Muon Package Manager, and under settings choose "Configure Software Sources", can you add in the missing ones from there?  I'm not entirely sure what autogenerates the default list upon installation or upgrade.  However, it should be that easy to add in what you're missing.  You can, of course, manually edit /etc/apt/sources.list.  Here is mine (note that I've added backports and extras:

```
## Major bug fix updates produced after the final release of the
## distribution.
deb http://archive.ubuntu.com/ubuntu precise-updates main restricted
deb-src http://archive.ubuntu.com/ubuntu precise-updates main restricted

## N.B. software from this repository is ENTIRELY UNSUPPORTED by the Ubuntu
## team. Also, please note that software in universe WILL NOT receive any
## review or updates from the Ubuntu security team.
deb http://archive.ubuntu.com/ubuntu precise universe
deb-src http://archive.ubuntu.com/ubuntu precise universe
deb http://archive.ubuntu.com/ubuntu precise-updates universe
deb-src http://archive.ubuntu.com/ubuntu precise-updates universe

## N.B. software from this repository is ENTIRELY UNSUPPORTED by the Ubuntu 
## team, and may not be under a free licence. Please satisfy yourself as to 
## your rights to use the software. Also, please note that software in 
## multiverse WILL NOT receive any review or updates from the Ubuntu
## security team.
deb http://archive.ubuntu.com/ubuntu precise multiverse
deb-src http://archive.ubuntu.com/ubuntu precise multiverse
deb http://archive.ubuntu.com/ubuntu precise-updates multiverse
deb-src http://archive.ubuntu.com/ubuntu precise-updates multiverse

## N.B. software from this repository may not have been tested as
## extensively as that contained in the main release, although it includes
## newer versions of some applications which may provide useful features.
## Also, please note that software in backports WILL NOT receive any review
## or updates from the Ubuntu security team.

deb http://archive.ubuntu.com/ubuntu precise-security main restricted
```

### Post by drmrgd on 2012-08-29
I think I figured it out.  While working on something unrelated, I realized that after the last install I had added the Ubuntu backports repo, but didn't add the Kubuntu backports repo.  Turns out that there is a newer version of KDE (v4.9) in backports, which contains Dolphin 2.1.  

From a terminal run:

```
sudo add-apt-repository ppa:kubuntu-ppa/backports
```

Then update and upgrade as normal.  You should then be at KDE4.9 with Dolphin2.1.

Hope that helps!

However, I can confirm on my system that the old repos are stored in that file.

### Post by drmrgd on 2012-08-31
I'm not sure about that as I've never used the tool.  So, I don't know how the file(s) are generated by it.  If you want to try, to be on the safe side, I would back up what you have first by just (for example) running:

```
sudo cp sources.list sources.list.bak; sudo cp -r sources.list.d/ sources.list.bak
```

That way you can easily restore them in case of failure.  This level of repository management and such is at the edge of my knowledge base, though.  I did have a quick look at what was generated, and it looks like everything is lumped into the one file rather than being split between the two.  Plus, I've seen this tool recommended a few times now, so it might be safe.  Not sure.

To be honest, though - and now that I'm back on my home computer where I can play around a little, I think the easiest thing to do is to add what you need using Muon.  If you launch the Muon Package manager and go to Settings > Configure Software Sources, you should be able to add the default ones in.  To test, I moved my default list:

```
sudo mv /etc/apt/sources.list /etc/apt/sources.bak
```

and then ran an update.  I didn't see anything for the default repos.  I then went to the Configure Software Sources utility, noticed all the check boxes were empty.  I selected all the usual suspects, and closed the tool.  Then when I looked in /etc/apt/, I could see that a new sources list was generated, and had back all of the old repos.  To me, I think this is a better way to go than a 3rd party tool.  Both would probably work, though.
