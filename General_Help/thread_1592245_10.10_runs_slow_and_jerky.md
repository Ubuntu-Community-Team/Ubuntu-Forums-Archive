### Post by efflandt on 2010-10-10
If you have nvidia check which version drivers you are running.  Earlier 10.10 beta used 256.53 which some people with GTS cards had (slow) trouble with.  Sometime during RC 260.19.06 became the default.  I got those during RC by deactivating nvidia in Hardware Drivers, rebooting, activating, then rebooting.  But my GT 220 had no problems with either version.  Although, glxgears in latter version was 50% faster.

If you upgraded and had regular repo drivers, you might try the deactivating, activating thing in Hardware Drivers.  If you installed video drivers some other way, you are on your own.

I have not downloaded the 10.10 release yet, so not sure which version that uses.

### Post by ahanssen on 2010-10-12
Well I have to say... it took me a while but after installing the 2.6.36-rc7 kernel 10.10 is amazing. It really was a great turn around. The process was a little intense because it was my first time but here is the process I followed. Because the terminal in the GUI is really laggy I did all of this from the command line in tty1 by doing Ctrl+Alt+F1

```

sudo su -
cd / 
wget http://kernel.ubuntu.com/~kernel-ppa/mainline/v2.6.36-rc7-maverick/linux-headers-2.6.36-020636rc7_2.6.36-020636rc7.201010070908_all.deb

```
For x64 
```

wget http://kernel.ubuntu.com/~kernel-ppa/mainline/v2.6.36-rc7-maverick/linux-headers-2.6.36-020636rc7-generic_2.6.36-020636rc7.201010070908_amd64.deb
wget http://kernel.ubuntu.com/~kernel-ppa/mainline/v2.6.36-rc7-maverick/linux-image-2.6.36-020636rc7-generic_2.6.36-020636rc7.201010070908_amd64.deb

```
For x86
```

wget http://kernel.ubuntu.com/~kernel-ppa/mainline/v2.6.36-rc7-maverick/linux-headers-2.6.36-020636rc7-generic_2.6.36-020636rc7.201010070908_i386.deb
wget http://kernel.ubuntu.com/~kernel-ppa/mainline/v2.6.36-rc7-maverick/linux-image-2.6.36-020636rc7-generic_2.6.36-020636rc7.201010070908_i386.deb
dpkg -i linux-headers-2.6.36-020636rc7_2.6.36-020636rc7.201010070908_all.deb

```
Here you will do the dpkg on whichever version you downloaded
```
 
dpkg -i linux-headers-2.6.36-020636rc7-generic_2.6.36-020636rc7.201010070908_amd64.deb
dpkg -i linux-image-2.6.36-020636rc7-generic_2.6.36-020636rc7.201010070908_i386.deb

```

This should update your grub and then you can reboot into this kernel.

This solved my problem and you can always update to the final 2.6.36 later. Hope this helps!

Glad I could help!  I should probably provide a bit more information.

Before moving to Mint I never used Ubuntu's update utility or Ubuntu Software Center.  I only used Synaptic, so I don't know the features of the official Ubuntu software updating/installing programs.

Mint's update utility, however, is pretty awesome.  Anyway, the specific packages it labels as "dangerous" are:

linux-headers-2.6.35-22.35
linux-headers-2.6.35-22.35-generic
linux-image-2.6.35-22.35-generic
linux-libc-dev 2.6.35-1022.35
udev 162-2.1

But these are not the names by which apt recognizes them (that is, with complete version numbers).  Pinning the version below each of these packages in Synaptic is easy enough, but I do not know how Ubuntu's other utilities work; I can only assume they use apt.  On top of that, if you use apt on the command line it will still attempt to install the updates, even though they are pinned in Synaptic.

If you would like apt to hold a particular version of a package you have to do it from the command line.  First of all, these are the version numbers recognized by Mint as "safe":

linux-headers-2.6.35-22.33
linux-headers-2.6.35-22.33-generic
linux-image-2.6.35-22.33-generic
linux-libc-dev 2.6.35-1022.33
udev 162-2

If you choose to use these versions it's easy enough to do through Synaptic Package Manager.  Once you have the package versions you want you'll need to run these commands in a terminal:
```

sudo apt hold linux-headers-2.6.35-22
sudo apt hold linux-headers-2.6.35-22-generic
sudo apt hold linux-image-2.6.35-22-generic
sudo apt hold linux-libc-dev
sudo apt hold udev

```You have to run each command separately, apparently, or add " && " without quotes after each cmd.  To confirm that the packages are held use this cmd to list all held packages:
```

apt held

```The downside is that this will eventually be fixed and you will want to upgrade your kernel.  When this happens the cmd
```

sudo apt unhold (package)

```will once again allow you to upgrade your held packages.

I'm no expert, but this has worked for me so far.  It should be noted that the kernel is super important and having an outdated kernel with everything else up to date probably isn't the best idea (seriously, I really don't know the ins-and-outs!).  So if this solves the problem for you remember it's just temporary!  I assume apt will only update packages up to the point of being compatible with your current kernel, but I don't really know how all that works.

As for graphics drivers and such, everything else on my system is up to date.  I'm still using Nvidia current and so on.  The only things I've held are the above mentioned kernel related packages.  Hope this helps, and if I've made a mistake please let me know!

> Well I have to say... it took me a while but after installing the 2.6.36-rc7 kernel 10.10 is amazing. It really was a great turn around. ```

sudo su -

```For x64 
```

wget http://kernel.ubuntu.com/~kernel-ppa/mainline/v2.6.36-rc7-maverick/linux-headers-2.6.36-020636rc7-generic_2.6.36-020636rc7.201010070908_amd64.deb

```For x86
```

wget http://kernel.ubuntu.com/~kernel-ppa/mainline/v2.6.36-rc7-maverick/linux-headers-2.6.36-020636rc7-generic_2.6.36-020636rc7.201010070908_i386.deb

```
