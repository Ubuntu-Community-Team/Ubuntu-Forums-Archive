### Post by mb_webguy on 2009-05-03
ASP and PHP can do the same things, but they're fundamentally different.

"ASP" can refer to two completely different technologies.  Classic ASP is an outdated Microsoft-proprietary scripting engine that allows the creation of interactive web sites using any of several scripting languages, including VBScript (a Microsoft-proprietary language based on Visual Basic) and JScript (Microsoft's implementation of JavaScript).  ASP.NET, on the other hand, is a Microsoft-proprietary framework that allows you to use any of the otherwise compiled .NET languages like Visual Basic.NET for web scripting.  To my knowledge, either version of ASP can only be written and implemented on Windows machines.

PHP is a scripting language.  It's most commonly used for server-side web scripting, but can be used in other ways as well.  It's possible to use PHP to write scripts for use in the terminal (using the PHP-CLI) and even desktop applications (with PHP-GTK).  Since it's open-source, it can be implemented on any system, and web sites and applications built using PHP are much more portable than those built using ASP.

Because ASP is a proprietary technology that requires commercial products to develop, and can only be implemented on Windows platforms, PHP is without a doubt the better choice, in my opinion.
