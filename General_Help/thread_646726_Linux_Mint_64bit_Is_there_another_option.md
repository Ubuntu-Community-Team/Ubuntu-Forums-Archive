### Post by zettberlin on 2007-12-22
Ubuntu Studio does not mention the Multimedia/Mpeg/DVD-issue because Ubuntu tries to avoid any discussion about software-patents (such as MPEG) and/or so-called "copy-protection" and Linux.

You may wonder, why the hell you can download and install and run all this "copyright/patent-infringing" software from universe? And even a library that handles a distribution-control-mechanism on commercial DVDs from a french project?

Because in the EU most of the patents in questions are still null and void and a software like lame that implements a industy-standard (MP3) with a library written from scratch is considered being the work of those, who have actually written that software. 


It is sad, that Ubuntu does not adress this issues officially, the situation in the US is indeed bizarre enough that even the big beast from Redmont is not perfectly happy with it. So it would be great, if Ubuntu/Canonical would state loudly that there could be full Multimedia support in the distro in the EU but not in the US.

They do not, actually.

Still it is no big issue to add that support to Ubuntu. After installation I use to install Synaptic select some 400 packages with it and have a cup of coffee while everything installs automagically. Afterwards I only need to tell Gdebi to install a single deb-package from that french site and everything works OK - much better then on MacOS that is ;-)

To switch to another Distro like Mint appears not worth the effort to me - I can still upgrade Ubuntu with 3 simple operations...
