### Post by Endolith on 2009-02-04
Has anyone else ever noticed this?

If you search through a user's chat logs in Pidgin, the first result is always at the *end* of the conversation, making it seem like there was only one match.  But then if you press the Search button again it will find another match *earlier *in the conversation.  It searches *up* the conversation, in reverse order compared to the search feature in every other program on the planet.

Obviously it should find the most recent *conversation *first, but then *within* that conversation you want to search down through the conversation chronologically, no?

I always assumed this was an obvious bug that would be fixed as soon as someone noticed it, but it's been several releases since I noticed it and it's still happening.  Does it work this way for you, too?
