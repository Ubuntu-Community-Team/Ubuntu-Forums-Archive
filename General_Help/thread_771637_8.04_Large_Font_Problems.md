### Post by chewearn on 2008-04-28
Ok, don't know if this will work in Hardy; try it out and feedback the success or failure.

1. Measure your display horizontal and vertical dimensions in millimeter.

2. Add the following in xorg.conf "Monitor" section:

```
DisplaySize <horz> <vert>
```where <horz> and <vert> are the dimensions measure in 1.

### Post by decapo on 2008-04-28
I measured the screen.  It was around 700 mm x 395 mm.  I opened my xorg.conf file and typed

```
DisplaySize     700 395
```

Then restarted X (i.e.logged out and then logged back in).  I'd like to note that before did all this I installed some KDE applications (Korganizer and Taskjuggler) and they all had enormous fonts too.  After I restarted X, I opened one of the KDE apps and the font was NORMAL SIZE!!! BUT, when I opened firefox the fonts were still big.

I read that this is a bug in firefox and X.  Here is the webpage

[http://bugs.launchpad.net/ubuntu/+source/firefox-3.0/+bug/201487]("https://bugs.launchpad.net/ubuntu/+source/firefox-3.0/+bug/201487")

I found another page with a fix here

[http:////blog.wolffmyren.com/2008/04/26/firefox-3-ubuntu-804-large-fonts-fix/]("http:////blog.wolffmyren.com/2008/04/26/firefox-3-ubuntu-804-large-fonts-fix/")

I tried this fix before and it did not work, but for some reason it worked this time.  After I did that I restarted X again (I don't think it was necessary).  When I logged back in The fonts in firefox were ok but the fonts in the KDE apps that I just installed were big again.

So I'm not actually sure if the xorg.conf edit worked or the firefox fix worked or both.  But i guess I'm happy for now.
