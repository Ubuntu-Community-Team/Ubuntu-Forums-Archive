Not what you asked, but there really are no headaches with 64-bit.
At this point, most developers use 64-bit themselves as they can compile and test both targets on the same readily available hardware.

Java may be an issue, icedtea may work. Me - I don't use java browser plugin.

Flash - you won't want to install the 32-bit package that ubuntu offers (that pulls in a bunch of 32-but deps), you want the "alpha" plugin from Adobe that works extremely well.

W32 codecs - I don't know about, I purchased fluendo gstreamer plugins 64-bit and they work extremely well for mpeg2, h.264, wma. Better than open source solutions on 32-bit.

google earth is 32-bit, but synaptic grabbed all dependencies and it runs flawlessly. Adobe Reader is 32-but but I just use evince. However, on CentOS I do use the 32-bit Adobe Reader and it works flawlessly.

The days when 64-bit Linux was a pain are over.
