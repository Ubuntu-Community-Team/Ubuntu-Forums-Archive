### Post by PapaGoose on 2012-03-26
Hi, I've just set up a folder on my desktop for syncing via FTP to my iPad- the idea is that I can dump files in there to read and then delete them from the device (and have them removed from my desktop). 

This all works fine, but what I can easily see myself doing is dragging and dropping a file onto the folder, moving it instead of copying. In which case, deleting it from the device will remove the original from the desktop instead of copying. 

There are obviously many workarounds to the problem- but as one piece of the puzzle, I'm wondering if there is any way to change the default drag-drop action in Nautilus to copy instead of move just for that folder? In the same way that drag and drop to a different device e.g. USB stick copies by default
