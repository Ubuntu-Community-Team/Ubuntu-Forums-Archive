### Post by minhmeoke on 2008-07-31
> You could use a VM
More interesting: andlinux, cygwin, mysis, gnuwin32 (mostly command line) 

These are all pretty good, but there is a slight performance hit for virtualization or emulation, so it is best to use native windows programs or ports if they are available. Opendisk has a pretty good collection: [http://www.theopendisc.com/](http://www.theopendisc.com/)

You could also compile linux programs from source to run on windows using MinGW. [http://en.wikipedia.org/wiki/Mingw](http://en.wikipedia.org/wiki/Mingw)
