### Post by c2tarun on 2013-02-04
Hi friends, I switched to KDE. Previously I was using Thunderbird as my default email client. Now I tried KMAIL. The one thing I really liked about KMail is that is sends mail quickly. It might be cheating by sending mails slowly, but it disappears the compose window and show notification after sometime that mail is send.

Now, in thunderbird when I send a mail I get a annoying sending message progress window. I google and was able to turn off the progress window. But then compose window will only close after sending the mail completely. Is there anyway I can make thunderbird give the feel that mail is sent quickly?

---

### Post by c2tarun on 2013-02-06
I found a solution to this problem. In config editor there is a property of, send mail in background. Setting it true resolved this issue.
