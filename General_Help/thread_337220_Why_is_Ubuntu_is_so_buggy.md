### Post by Wim Sturkenboom on 2007-01-13
To all:
Jasin posted some questions regarding his/her problems.

> **jasin said:**
> Actually that is not true, developers can and often do develop drivers for linux, developers that are not affiliated with any specific hardware manufacture.

Let me give you a few examples:

Synaptics TouchPad driver for X.Org/XFree86 server
Synaptics TouchPad driver for X.Org server
Linux driver (tpcopnfig) for the Synaptics Touchpad
Intel® PRO/Wireless 2200BG Driver for Linux
Pcmcia-cs Package
Alsa-driverIt highly depends on the hardware if a developer can do the job easily. If the HW documentation is freely available, yes:
Open up the PC, check the hardware connected to i.e the touchpad, look up the documentation for the hardware that you've found and you can easily write a driver.

If the documentation is not freely available (i.e. nVidia/Ati graphics), no:
Open up the PC, check the hardware on the graphics card, look up the documentation for the hardware that you've found and if you can not find it, monitor all data going to the hardware and coming back from it in i.e Windows, try to understand why this is all happening and write the driver (with the risk that you miss something as your setup does not use a specific function while anybody elses PC does use it). This is reverse engineering and very time consuming.

So, as long as HW manufacturers do not provide documentation on their hardware, Linux will not be able to work out of the box without proprietary drivers.
