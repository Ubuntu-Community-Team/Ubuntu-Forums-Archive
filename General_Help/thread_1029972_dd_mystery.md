### Post by meierfra. on 2009-01-04
This seems to be a bug in the Ubuntu 8.10 version of "dd". 
I run the same commands from Puppy Linux   and both commands produced the correct output (the first one from above).
 Repeated it from Knoppix and again "dd" worked  just fine.
 Repeated it one more time from my Ubuntu 8.10 Live USB and "dd" (applied to the partition) malfunctioned  just as from my regular Ubuntu.

dd is part of the coreutils package, which also   includes cat,  chmod, chown, chroot,  cp, mv, ls, rm ...

Kind of scary to find  a bug in such a fundamental package.

Ubuntu 8.10 uses coreutils 6.10. The debian stable version is 5.97 (and that's what Knoppix is using)

I filed a bug report at launchpad.
