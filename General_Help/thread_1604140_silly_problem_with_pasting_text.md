### Post by Muster Mark on 2010-10-23
Hi all,

So, for some reason I can no longer paste anything any more by typing Ctrl-v.  I can copy with Ctrl-c just fine, and obviously the 'v' key is working.  This is so for every application I have tried to do this in, except the Terminal, in which the command for paste is Ctrl-Shitf-v.  I have gone through all the key bindings settings pages I can find (System>Preferences>Keyboard Shortcuts as well as Compiz settings), but haven't managed to find what this key combination is being used for.  Any ideas?

Thanks!

---

### Post by Muster Mark on 2010-10-27
I finally figured out how to resolve this.  It was the viewpoint switcher in compizConfig settings manager that was taking up ctl-V.  When switching between graphics modes, a lot of my settings were changed, and apparently that was one of them.
