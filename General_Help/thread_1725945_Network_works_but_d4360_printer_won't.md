### Post by jemvyc on 2011-04-10
I have 2 laptops running Ubuntu 10.10.  They are part of a windows XP network.  All the network functions seem to work OK except for the HP d4360 printer.  Even the HP c4280 photosmart printer works from any computer in the network.

The printers are installed via USB on a computer in the network running XP.  As I said above, the c4280 works but the d4360 does not.

When a print job is sent to the d4360 the print manager on the XP computer registers one print job on the printer, then the printer clicks and the power light blinks but no printing.  A little later the printer clicks again and much later it clicks again.  The printer will not print documents even from the XP network until I cancel all documents on the XP computer. Then printer que clears and the XP network can print on that printer.

These symptoms made me think that the problem is the printer driver on the Ubuntu computers.  I have searched and searched for an alternative and I can't find one.  

I appreciate any suggestions about where to look for help with the problem.
