### Post by StuartN on 2010-05-31
> **mindlessbrain said:**
> the file that is right on the desktop it tells me no such file.

At the risk of stating the obvious, filenames are case-sensitive and spcial characters (like space) must be escaped or the filename quoted. If you are using the command line, then type the first character or two and press tab for autocompletion. Double- or right-clicking in the GUI file manager should not be able to give a file-not-found error.

What is the full path to the file that fails to open?
