### Post by mackinax on 2006-03-08
[QUOTE=Mr Wrath]Do you recommend it, question its stability, or just stay away from it. [/QUOTE]
I highly recommend it.

I don't see how "stability" would be an issue. "Dual Boot" does not mean that you boot two systems at once. You have two *(or more)* options to boot up with. Either one OS will be running or the other. So it's not like they can interfere with each other or anything.

My preferred "dual boot" configuration is to install different OSs onto completely separate hard drives, not just different partitions on the same drive. 
Advantages: Not having to mess with the MBR on the windows drive. And if one drive fails, the other system will still work.

### Post by Ubuntist on 2006-03-08
[QUOTE=Mr Wrath]I like Linux (actually Ubuntu), but my wife uses XP.[/QUOTE]
That's my situation too.  Dual booting scared me, because I'm such a noob, but in practice it's worked well.

I did once have a small problem with dual booting, however.  I ended an Ubuntu session by hibernation rather than a full shutdown.  Later, having completely forgotten the hibernated Ubuntu session, I booted up in XP (or maybe it was a hibernated XP session followed by an Ubuntu boot-up, I can't quite remember).  I then found that my Firefox bookmarks had been trashed.  I presume this is related to the fact that I use the same bookmarks.html file for both Ubuntu and XP.

It was easy to fix -- I just copied the back-up bookmarks file to bookmarks.html and didn't lose much.  The moral of the story is just remember not to boot up in one OS when you have a hibernated session in the other OS if Ubuntu and XP share a partition.
