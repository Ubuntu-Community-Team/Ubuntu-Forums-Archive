### Post by Billsey on 2011-03-19
I haven't seen the list of changes for the most recent kernel update to Lucid, and I've only been on it with this machine for a few minutes, so I might be premature on this.

For the last couple of versions, my P4 install (running graphics at 2048x1536) had an annoying problem in that on boot, as soon as the system shifted into my chosen screen resolution, my mouse pointer would disappear. I was able to work around it by launching gFTP and going through the motions of trying to access my website's FTP server. That would bring my mouse pointer back. It was, however, still an annoyance. Tonight I just downloaded and applied the latest kernel update for Lucid and went outside and had a conversation with a neighbor and a member of the complex management team (very pleasant, by the way). When I got back inside, the system was waiting patiently for my password to log-in and start-up again. After putting in my password, I was so amazed at the events that I almost didn't know what to do: my mouse pointer ***did not*** disappear when the chosen resolution was engaged. It was still there! Like I said, this might be premature. But on the chance that it is not, I'd like to issue an "Atta boy" to those responsible for the update.
