### Post by #&amp;thj^% on 2016-12-21
Use this for held Packages.
```
sudo apt dist-upgrade
```
that code will not take you to a new release...it just updates the System Fully.
And don't forget to clean the /boot to keep you healthy "space wise" with:
```
sudo apt autoremove
```

### Post by #&amp;thj^% on 2016-12-21
Cool! Glad it worked.
Now I will try to explain the other questions as best I can.
First, make sure you have the update-manager-core package installed:

   ```
 sudo apt-get install update-manager-core

```

This is the officially recommended upgrade path for server upgrades which must be performed over a remote connection.

Start by running do-release-upgrade with no options:

   ```
 sudo do-release-upgrade
```

 to use the development releases:( (For People who know what they are getting into and can fix their systems).

  ```
  sudo do-release-upgrade -d
```
The "-d" switch is for development releases.
Hope that helps.
