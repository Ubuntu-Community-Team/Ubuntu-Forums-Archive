### Post by ilrudie on 2008-08-19
First:  If you have a question just post the question to the forums.  As much as it makes you feel better to bash on Linux it is unlikely you will get answers to your problem.  I'd suggest reposting your question with more detail, what version of Ubuntu you are trying to use, any error messages that show, tutorials or other posts you already tried and leave out the all the stuff about leaving Linux for Windows and how the command line is the worst.   If you don't want to use the command line just state it clearly before your question and most people will try to tell you how to do it with menus or explain commands more clearly if they are necessary.  Now regarding some of your comments:

Microsoft has been trying as hard as they can to make Windows incompatible with other operating systems.  They have had to pay record fines in Europe for doing this over and over again.  They do not like competition because they have an (IMO)inferior product and development model so they create incompatibility and then people with lesser market share look like they have shoddy products because Windows does not work with them.  If you want to blame someone for <insert application or OS here> not working "out of the box" with Windows the blame has to rest first with Microsoft.  If the entire MS stack was properly and openly documented things would be smoother.

Regarding your position that the command line is bad and MS got rid of it in 1995 you could not be more wrong.  The command line is the most powerful tool in Linux (and just about every non MS OS out there).  Its fast and reliable.   Some people who got used to windows prefer to search through gui menus and screens to find the programs and setting they want.  To me that sounds like you working for the computer.  With the command line I just type what I want to do and the computer does the work of finding where the program lives ( it can be different on different Linux and Unix platforms) and where the config files are, not me.  Microsoft with Server 2008 just now realized that running a mail server or a web server with a GUI is just a waste.  Its a security flaw and its not efficient.  The command line still has a place in computing.  Microsoft just didn't realize it.  There a plenty more good arguments for why having a real command line is better than having a gui and a dos emulator.  Too many for one post.

Regarding the command line's use in the Forums its because its much easier than screen shotting and posting a bunch of pictures about how to do it in the gui.  Most everything in Ubuntu can be done from the gui but is easier to explain from the command line.  You can cut and paste the command and you are off and running.  No searching through menus.   Once again its the computer doing the work for you.  It knows where this stuff lives so why should you have to search for it.

Lastly it seems like you are looking for exactly Windows but somehow better.  This is a common occurrence for lots of people.  Someone tells them <insert favorite linux disto here> is the best.  That it is the future.  This some how translates into "Linux is finally just windows that doesn't crash all the time".  I hope this never becomes the case.  Linux is striving every day to be truely better than Windows... at everything.  This means that it is going to be different.  I don't mean to put you off of Linux but if you don't want to learn something new and all you want is Windows there is already a solution for you out there.  If you don't want to put in the effort to learn (which is understandable, people are busy) then stick with what you know.  If you are looking for a better computing experience and you are willing to learn and put in some effort then Linux may be right for you and there will be plenty of people offering free support, advice, teaching and sometimes a little preaching.

Sorry for the long post.
