### Post by wpshooter on 2007-01-14
I think you are saying that you are still using M/S Windows on this computer, is that correct ?

If so, is your sound working O.K. in Windows ?

If it is working O.K. in Windows and NOT in Ubuntu then you know it has to be a hardware detection problem with Ubuntu.  I think you are saying that you are not getting sound when you are running Ubuntu off of the DESKTOP (live) CD.

If you are having this problem while using the live CD, you can give installing from the live CD to your hard drive a shot and see if the sound problem is cured during the hard drive installation BUT my guess would be that you would have a better chance of the sound working correctly after installation if you used the ALTERNATE CD instead of the DESKTOP.

In any case, if it does not work after installation you may be able to change the settings in the sound section and get it to work.

### Post by MontanaMax on 2007-01-14
The "missing" space on your hard drive is most likely due to Linux separating it into several partitions - roughly speaking, one for the root system, one for memory swapping, and one for user data. The one you're seeing the 100 gigs available is probably the user data partition, and the other space is reserved for the other uses.

This thread discusses partitions more and can show you how to see the sizes on your system.

[http://ubuntuforums.org/showthread.php?t=332976&highlight=partition](http://ubuntuforums.org/showthread.php?t=332976&highlight=partition)
