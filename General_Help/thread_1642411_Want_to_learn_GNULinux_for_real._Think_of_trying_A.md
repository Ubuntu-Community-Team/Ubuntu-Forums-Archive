I've been using Arch exclusively for three or four years now and would encourage you to familiarize yourself with the [Arch wiki]("https://wiki.archlinux.org/index.php/Main_Page"). You might find the [beginner's guide]("https://wiki.archlinux.org/index.php/Beginners'_Guide") especially useful. You'll note that "beginner" does not mean beginner to Linux, but beginner to Arch. 

You won't have to learn vi if you're comfortable with nano. Nano is one of the editing tools on the installation CD. You will spend a lot of time at the command prompt before you have your GUI desktop up and running. Installing Arch can be a good learning experience if you take the time to read through the configuration files. 

Arch is what you make it, be that a learning tool, a web server, or a desktop media station.

Good luck in your adventure with Arch.

### Post by Axolotl9250 on 2010-12-10
I currently use Arch and Ubuntu and I'd definately say it was worth it. It allows you to learn about linux, but more it makes you 'appreciate' exactly how much has already been done for you in Ubuntu, Mac, and Windows, and all other distros that give you a good graphical installer an desktop. Although My arch is currently set up almost exactly like Ubuntu, with some differences, it made me appreciate both OSes a lot more. In addition, whilst building up a system yourself, because of it's package manager and AUR, it is quicker than doing the same with Slackware of Gentoo. In fact, many people on Arch migrated from Gentoo or Slack, quoting that they still end up using and learning the same knowlege and skill to use it, but it's just faster - no compiling or building directories to make packages etc, the AUR and yaourt builds things from source for you, so it's faster, but you do still learn the process.
