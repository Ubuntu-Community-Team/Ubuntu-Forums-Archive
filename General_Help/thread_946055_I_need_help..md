### Post by lykwydchykyn on 2008-10-12
Ubuntu is certainly a good choice to start off with.  I would go ahead and install the current version (8.04) because it is a "long term stable" version-- that is, it will be supported for much longer than a normal release.  It's also fairly stable at this point and you will be happier with it than if you grabbed the newest version on day 1.

Ubuntu has a windows emulator available called Wine, and there are also commercial emulators available.  Even so, I would recommend not trying to install Windows software on Ubuntu until you are more comfortable with the environment.  It's better off in the long run to find free, open source alternatives that support Linux natively.

Best way to get started is to download a liveCD iso, burn it to CD, and boot to it.  You'll be able to test all your hardware and check out the feel of the system.

Ubuntu can run Windows program using an API layer called Wine. If you spend most of your time with Windows-only applications, you probably don't want to use this. In many cases, Wine is less stable than a native Windows environment, and doesn't offer any advantages to speak of apart from the fact that it's open source and doesn't charge licensing fees. 

If you tell us which Windows applications you use, we might be able to suggest replacements or alternatives that are available for Linux. In many cases, of course, developers release versions of these applications for each platform simultaneously.

### Post by darksoul7 on 2008-10-13
I can definitely feel you there. I was a Windows user since Win 95 and now consider myself a Linux user. This was all thanks to Ubuntu. 

Since my switch, I have gone and studied Linux and will be taking the CompTIA Linux+ certification exam before the end of the year.

There are a few things you need to know about Linux. 

1. It will run a vast amount of Windows programs, but they are run through emulation (WINE) and not all of them will run as smoothly, or as consistently as you once experienced. For the best information on this, please look to [www.winehq.org](www.winehq.org). You can then look up the programs you use and see how well they run.

2. The file system will take some getting used to. The move from your standard Win/DOS c:\ to the Linux "/", or root folder, as well as figuring out where everything is placed will take time. You just have to play with it and learn it. 

3. Your hardware MAY prove difficult to configure. This, of course, depends on how old/new your hardware is and if drivers are available. If drivers are available, it's pretty simple to install them and get your hardware running. Keep in mind, Linux does not like ATI cards. This is mainly due to poor support from ATI in making their driver source code available to the public.

4. Installing native Linux programs is entirely different. If the program is in the repositories, then it's just a matter of typing it in either Add/Remove applications or the Synaptic Package Manager. If they are not in the repositories, you will need to learn to make an installer from the RPM packages and figure out how that all works. To this day, I still have some issues with this. Windows programs installed from .exe or CD's will still bring up the Windows installation wizard and should install just the same. Please look back to point #2 for this. 

5. Linux is freedom. Once you learn the system, the way it works, the way it thinks, you can do literally anything with it. You can change the look, the feel, the way it works. Literally ANYTHING can be modified. 

I think Ubuntu is the best Linux flavour in existence right now. It's simple. It's easy. It's human. I've since dealt with Fedora, DreamLinux, and a couple of other flavours, but I will not get rid of Ubuntu. I've even spread the Ubuntu virus as I like to call it to friends and co-workers. Many of them are now using Ubuntu more than Windows and with good reason.

The main thing is this: Don't be afraid to try new things. You may just like it.
