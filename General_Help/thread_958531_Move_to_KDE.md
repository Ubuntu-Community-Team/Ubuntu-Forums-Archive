### Post by CloudFX on 2008-10-25
You can install Kubuntu within an Ubuntu installation.

```
sudo apt-get install kubuntu-desktop
```

When you log in, go to Options in the bottom left, select Session, and choose KDE.  You can do the same thing to go back to Gnome.

The same thing works for Xubuntu.

```
sudo apt-get install xubuntu-desktop
```

### Post by bryonak on 2008-10-25
To express it in a rather moderate way, KDE vs GNOME has been (and in some places, still is) the reason for a lot of zealotry and fanboism.

KDE is generally considered more polished and 'stylish', whereas GNOME focuses on simplicity and moving as much of the 'unnecessary' stuff as possible into menus and extended option tabs.
GNOME wants to allow the user to concentrate on the important matters, improving productivity. KDE tries present the user as many options as possible at once, in order to improve productivity...

To phrase it more incitefully, KDE is often decried as bloated, 'too much of the bleeding edge' and GNOME is picked on the claim that it's users are simple, dim folks.

I think you'd like the look and feel of KDE, since it isn't that far away from OSX. I myself prefer GNOME, but KDE's nice too.
Which one you run by default doesn't really matter, since they are perfectly interoperable... it's just that your install grows by one or two hundreds of megabytes if you want to have both, and the start up time of KDE applications is noticably longer if you install them in GNOME.

Besides, the Compiz project (together with Emerald) makes the eye-candy of both desktop environments superfluous.
