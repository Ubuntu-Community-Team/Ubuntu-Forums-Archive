### Post by snowpine on 2010-01-06
Each Ubuntu release is 6 months apart, therefore the kernel, libraries, and other important system packages are six months newer with each release. So, developers (like Miro) need to make sure their application is compatible with each Ubuntu release separately. For example, the Hardy version (April 2008) can't depend on a feature from 2009. Does that clear things up?
