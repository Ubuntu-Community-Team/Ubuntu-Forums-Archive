### Post by 3rdalbum on 2013-01-13
Ubuntu 12.04 may be a little less glitch-prone in general, but it's important to remember that there's not any extra "bug fixing" time allocated to LTS versions. It's just that LTS versions are a little less bleeding-edge than non-LTS versions. It's also possible that you might run into a bug in 12.04 that has been fixed in 12.10.

The biggest advantage of LTS is that you can run the same system for 5 years without necessarily needing to upgrade. This is also a downside, too; in five years time when you are forced to upgrade, Ubuntu will have ten versions of changes and probably be nearly unrecognisable! You'll then have to adapt to all changes at once; whereas with the six-monthly releases you'll only get a few changes in each version, so it's easier to adapt each time.

### Post by Mark Phelps on 2013-01-13
> **jim_deadlock said:**
> It's so easy to upgrade Ubuntu from one release to the next that this really isn't an issue...

While it is easy, I would take exception to it not being an issue ...

Upgrading IS an issue -- for the following reasons:
1) No rollback -- after all these years, Canonical still has not seen fit to provide a simple rollback mechanism such that, if folks have problems, they have to reinstall from scratch -- erasing all the customization they spent hours and hours doing.
2) Upgrades break stuff -- upgrading from 12.04 to 12.10, if you're using an AMD chipset older than the HD5x stuff, will result in major problems because the restricted drivers for those older cards will NOT work with 12.10.

So, it is never a good idea to rush into an upgrade.
