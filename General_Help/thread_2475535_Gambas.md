### Post by Impavidus on 2022-06-01
What is your goal, what is your comfort zone and what is your willingness to leave it?

I've never used Visual Basic, Gambas or any other IDE. My experience with BASIC (of which Gambas is a dialect) was mostly on a TI-83 calculator. I usually program in a text editor plus some command line tools, mostly in C. The compiler and required header files can be installed with a few mouseclicks. I rarely build a GUI for my applications and on the few occasions where I did, it was either from scratch in openGL or a web interface. But if you want to build simple tools with a GUI, don't do it like that. When an application has scriptable elements, I put them in a stack-oriented programming language with postfix notation. That allows for a very simple interpreter.

So yes, there are alternatives, but are they in your comfort zone?

Bash, Python 3 and probably some others are already installed on your computer, as parts of the operating system depend on them. The interpreters and compilers of many other languages can be installed from the repositories with just a few mouseclicks. Most programming languages can be written any any text editor you like, but if you prefer an IDE, some are available.
