### Post by TeaSwigger on 2007-08-07
Hi :) I write fiction for a hobby and have recently converted to Linux and ubuntu. Previously on win2k pro, I was using the late, great KeyNote ( [http://www.tranglos.com/free/keynote.html](http://www.tranglos.com/free/keynote.html) ) for all my writing. It's a perfect app for it, and served me well (as well as being responsible for raising my awareness to the merits of open source software). It runs passably under WINE, although of course it is a bit long in the tooth in some respects and isn't as lightening fast via WINE as it was on its native OS. Sometimes it gets sticky. Oh and the spell check isn't supported. And...

The main problem, however, is that the KeyNote project has been dead for years now, and with its unfortunate demise my writing is stuck on KeyNotes' own obscure format (.knt). I realize my writing isn't "important" but it means a lot to me, so I'd like to keep it accessible. There is apparently a program that will take the multi-tree knt and convert it to one big single-tree pdf - a Windows(tm) only, proprietary and closed-$ource prog. I'm through with that game if you know what I mean. But to transfer the writings manually note-by-note would be a nightmare and take an impractical amount of time. So my "wants" are twofold:

1) Converting the .knt materials to a format with a healthy prospect for future support.

2) Finding a clean, light, native Linux writing app which composes the given format in an interface that equals the basic functional outlay of KeyNote.

To which there would seem to be two courses: 

1) Methodically transfer and rearrange all the documents somehow to a bunch of much smaller bits in a format used by the best available linux app.

or

2) Resurrect KeyNote for Linux.

Well, knowing nothing of programming, I've tried an impressive parade of native Linux note-taking apps. There are a lot! LyX, Tea, Kjots, notecase, Tomboy, TuxCards, kdissert, BasKet Notes, even kile, and probably a couple I can't recall. None equate. Many are part way there, having (single) tree structures and everything but tabs (which enable you to flip handily between separate trees), or tabs but no tree, or they have both but are intended for programming work (kile, etc) and therefore are either ill-suited to composing a text book doc.

A .knt file appears to be based on .rtf, perhaps simply a single .rtf file with special markup which the app parses to present the content in tree nodes and tabs. I can't speculate further, but the code for KeyNote and its file format is freely available. That being the situation, I wonder if I should ask around if anyone else might be interested and try to make a project of either an open-source converter or a KeyNote app for Linux? I really don't know what all is involved.

So, for any who made it this far... Should I try moving forward in any direction or let it go? Advice? Thoughts? Suggestions? Volunteers? ;)
