### Post by dje on 2008-10-07

### Post by fballem on 2008-10-07
As I understand it, if a specific release is identified as an LTS (Long Term Support), then that release will continue to be supported for 3 years on the Desktop, 5 years on the Server. If a specific release is not identified as an LTS, then that release will continue to be supported for 1-1/2 years on Desktop, 3 years on the Server.

The next release, Intrepid Ibex, will be identified as release 8.10, and is not an LTS.

By the way, the numbering system is year.month. For example, 8.10 will be released in October (10th month) in 200_8_. The last release was 8.04, and was released in April (4th month) in 200_8_.

Hope this helps,

I've only been at this a while (8.04 is my first foray into Linux), but I gather that there can be significant differences between versions. In addition to fixing bugs, new programs and new versions of existing programs are often included, as well as updates to the Linux kernal (the core operating system).

The currently scheduled release date for 8.10 is October 30, 2008. Ubuntu is on a 'time-driven' release schedule. A new version is released every 6 months. From time-to-time, a release will be identified as an LTS. I believe that the last LTS prior to 8.04 release was 6.06.
