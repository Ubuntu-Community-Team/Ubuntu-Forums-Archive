NTFS drives that are just pulled from windows without removing properly will be locked to ubuntu, and either need to be force mounted, or correctly removed from windows first.

### Post by c1rcu1t on 2009-05-30
i read another forums posts about a similar issue and i think i might have an answer for you.
you'll have to install ntfs-3g first by typing in your terminal 

```
sudo apt-get install ntfs-3g
```

then once that is completed you can try to force mount it with this code, only replace the path of /dev/sdb1 with whatever your path is, more than likely it will be the same path, but just to be sure check your error message when trying to mount and make sure. 

```
sudo mount -t ntfs-3g /dev/sdb1 /media/hd-ntfs -o force
```

again, i've never actually had to do this so it's only in theory, but give it a try. the link to the page i was referring to can be found here.

[https://answers.launchpad.net/ubuntu/+question/16872](https://answers.launchpad.net/ubuntu/+question/16872)

if this works, i would follow the advice of the user **marcobra** in the thread and do the following after successfully mounting the drive. 

modify your /etc/fstab to force mount
open a terminal and type:


```
sudo gedit /etc/fstab
```
 

and add this row
 

*/dev/sdb1 /media/hd-ntfs ntfs-3g defaults,force 0 0*
 

Save and exit
