### Post by waynefoutz on 2010-03-29
> **gadolinio said:**

The repositories will be moved to another server, so Synaptic or apt in terminal will no function after April 30th. What you are going to have to do to get it back working is this:

```
sudo gedit /etc/apt/sources.list
```

Then make the following changes:

These lines:
```
deb http://us.archive.ubuntu.com/ubuntu/ intrepid universe
deb-src http://us.archive.ubuntu.com/ubuntu/ intrepid universe
deb http://us.archive.ubuntu.com/ubuntu/ intrepid-updates universe
deb-src http://us.archive.ubuntu.com/ubuntu/ intrepid-updates universe

```

And these lines:
```
deb http://us.archive.ubuntu.com/ubuntu/ intrepid multiverse
deb-src http://us.archive.ubuntu.com/ubuntu/ intrepid multiverse
deb http://us.archive.ubuntu.com/ubuntu/ intrepid-updates multiverse
deb-src http://us.archive.ubuntu.com/ubuntu/ intrepid-updates multiverse
```

need to be replaced with this:
```
deb http://old-releases.ubuntu.com/ubuntu/ intrepid main restricted universe multiverse
deb http://old-releases.ubuntu.com/ubuntu/ intrepid-updates main restricted universe multiverse
deb http://old-releases.ubuntu.com/ubuntu/ intrepid-security main restricted universe multiverse
deb http://old-releases.ubuntu.com/ubuntu/ intrepid-backports main restricted universe multiverse
```

Also the ones near the end of the file that have "security' in them need to be deleted or commented out with a # because security updates will no longer be offered. I'm not sure if any third party sources will be affected.  This should get Synaptic functioning again. You'll know when it's time to do this when you get error messages when you hit the reload button in Synaptic.

Security is never a binary safe-unsafe situation.  The bad guys are always one step ahead, even when you're getting all the security updates your system needs.  If you stop getting the updates, the bad guys will be two, three, many steps ahead.

This doesn't necessarily mean you'll end up in trouble.  Some people lead charmed lives and never suffer a single blow.  But having an unsupported system will increase your risk.  Not ideal.

But you don't need 3D graphics to do money transactions.  So you could keep a live CD of an up-to-date distro to do your money stuff with.  Or you could dual-boot.  It doesn't have to be an either-or situation.

I found no intrepid repositories under [http://old-releases.ubuntu.com/ubuntu/](http://old-releases.ubuntu.com/ubuntu/), but I did find them in [http://archive.ubuntu.com/ubuntu/dists](http://archive.ubuntu.com/ubuntu/dists).

```

gksudo gedit /etc/apt/sources.list
```

```
deb http://archive.ubuntu.com/ubuntu/ intrepid universe
deb http://archive.ubuntu.com/ubuntu/ intrepid restricted
deb http://archive.ubuntu.com/ubuntu/ intrepid multiverse
deb http://archive.ubuntu.com/ubuntu/ intrepid-backports main
deb http://archive.ubuntu.com/ubuntu/ intrepid-backports universe
```
