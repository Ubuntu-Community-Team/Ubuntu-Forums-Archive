### Post by Sylos on 2010-10-22
Ok - Sorry my advice up to now has been lame BUT I finally got time to have a proper look at the documentation for this.

Your original post does not show that you have included the "Twinview" option in the right place. Without this none of the Twinview options will be used. So you need this:

```
Option "TwinView"
```
This goes in the Device section - Not the Screen Section.

You also need to specify the orientation of the two monitors in relation to each other using "TwinViewOrientation". eg

```
Option "TwinViewOrientation"      "RightOf"
```

I think you also need these options with the relevant values for your hardware:

```
Option "SecondMonitorHorizSync" 
Option "SecondMonitorVertRefresh"
```

You can also force the settings for your device if it isnt detected correctly by using:

Option "ConnectedMonitor"

Check out this page for the info

[ftp://download.nvidia.com/XFree86/Linux-x86/1.0-8774/README/appendix-g.html](ftp://download.nvidia.com/XFree86/Linux-x86/1.0-8774/README/appendix-g.html)

You could also run two instances of X for this but it is needlessly complex I think as you have to set up the server layout section to tell the card to use both monitors and how to orientate them.

Here is an example of how the section should look.

```
Section "ServerLayout"
        ...
        Screen         0 "Screen0" 
        Screen         1 "Screen1" leftOf "Screen0"
        ...
    EndSection
``` 

I also think you need to create a new Device section for the second monitor as well as the new Monitor and Screen sections.

Here is a link to the Documentation for this 

[ftp://download.nvidia.com/XFree86/Linux-x86/1.0-8774/README/appendix-p.html](ftp://download.nvidia.com/XFree86/Linux-x86/1.0-8774/README/appendix-p.html)

I suggest trying the Twinview options first and only embark on the use of 2 X instances if all else fails.
Post back with your results and let me know if it works.

Good luck
