### Post by Rallg on 2009-04-19
I looked at this a few months ago, and found nothing satisfactory. The problem, it seems, is that a program needs phoneme models (a large database) These are difficult to develop, and the already developed models (from commercial suppliers) are proprietary.

If you are licensed for Windows, MS has some sort of basic speech to text recognition that is included with Vista and optional with XP (find it on the MS site via search engine). Since it needs .NET structure to run, it may or may not work with WINE. I don't know how good it is.

### Post by rvk on 2009-08-21
What is needed are good acoustic models for different languages and to make those a lot of speech is required. This speech needs to be uncompressed (so no MP3 for example) and needs to correspond to a text. This text should be copyright free.

If you're interested in speech recognition (open source) you should really visit VoxForge regularly and submit some speech (preferably a couple of hours, though every minute counts). Use the web app on the VoxForge website or record another copyright free text. You could for instance help with recording an audiobook at LibriVox and submit your speech to VoxForge as well.

Use a good microphone and a good soundcard to get the best results (both when submitting speech and when using speech recognition this is important).
