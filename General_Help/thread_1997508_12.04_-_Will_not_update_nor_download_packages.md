### Post by Kr0nZ on 2012-06-06
This is a routing issue I think.
I had his exact same problem yesterday.

I could SSH in, resolve dns, but could not update or access internet

type 'route' in a terminal
if your missing a line that says default, such as
```

Destination     Gateway         Genmask         Flags Metric Ref    Use Iface
default         DD-WRT          0.0.0.0         UG    0      0        0 wlan0
```
then routing is messed up.

I fixed it on mine by typing
```
sudo route add default gw 192.168.1.1 netmask 0.0.0.0
```
192.168.1.1 should be your routers IP

but this is likely to reset when you restart the PC
