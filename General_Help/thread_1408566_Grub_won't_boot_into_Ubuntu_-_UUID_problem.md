### Post by ratcheer on 2010-02-16
If you can log in, you need to "sudo grub-install /dev/sdX", where X is the drive you have Ubuntu installed on, without the partition number. E.g., /dev/sda, not /dev/sda2. It should discover and set up to boot to that drive.

Tim

"sudo update-grub" should do the same thing, but without re-writing the boot sector, which is clearly intact since grub is loading

To boot into a your system for that, edit the boot line from grub, and replace UUID=344ef89b-4054-4639-9f2c-8fa94d8e71c7 with /dev/sda9
