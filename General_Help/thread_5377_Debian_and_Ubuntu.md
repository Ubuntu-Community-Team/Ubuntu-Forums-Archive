### Post by ynef on 2004-11-18
The main differences are listed on the official website. To me, Ubuntu is a really good distro, because it includes all you need to be productive in an easy to set up manner with very little bloat (you don't NEED 30 different text editors installed by default -- you can add the ones you want later). Ubuntu is also geared towards a more uniform desktop experience for its users -- Gnome is default and that may not be for you, if your avatar choice is an indication of your favourite desktop environment... ;)

It's easier to use in a way, since it manages to do a lot of the tedious configuration on its own. I haven't used debian in years though, so I don't know how far debian has come, but even complete newbies can (and have) start with Ubuntu.

What do you do in debian? Not sure what you mean by the question.

Define power. You have a stable kernel that is up to date and maintained by the folks behind Ubuntu, which is good -- in debian, unless I'm mistaken, the official kernel (in "stable") is really old and you don't have any official support if you use the other versions ("unstable" and "testing").

You can run debian packages, but you should favour the ones that are from Ubuntu, since they are ment to work with the Ubuntu system layout -- the same cannot be said for debian, since it's not the same distro. Installing non-Ubuntu packages may of course break your system, even if I have yet to see this with my own two eyes.

I hope I've given you the answers you were looking for. Welcome to Ubuntu!

### Post by calvinpriest on 2004-11-22
I've used Debian Unstable for the last couple of years and I think Ubuntu is a godsend.  It has the same breadth of packages available (in universe) with the stability and security (pretty much) of Debian's Stable distribution.  Debian Unstable is a different distribution pretty much everyday -- packages are constantly updated and added.  This was very exciting to me for a while, but the downside was that little things (and occasionally big things) were always breaking.  At first I appreciated the learning experience aspect of this, but eventually I just wanted things that were working to continue working.

It's also awfully nice when installing a distribution to have good package choices made, necessary configuration done, and to just have a nice useable environment without a lot of manual configuration. 

I'd say if you want to maximize learning with Ubuntu, and simulate a Gentoo-like distribution, just make the choice to do your configuration on the command line.  Avoid the graphical configuration tools.  That will be a pretty close approximation.  That and whenever you find a package you want that isn't in Ubuntu, plunge right in and compile from source.  Then you can use your Linux and eat it too.
