### Post by Friqenstein on 2009-08-07
Kdar,

That's sort of an odd way of putting it. You shouldn't really try to get into a language because you simply want to use it. If you have to ask what can you do with it before you even start with it, then you probably don't need it.
Now don't get me wrong, I'm all for people learning more, especially languages, and especially Perl, but you really should have a goal in mind.

But I will try to help anyway with your question.
First off, try to think of something you want to do.. in respect to an application or script you want to write. Take a look at some small programs you may already have, or on the web and see if you can try to duplicate it in a very simple manner using Perl.
Or, you could get int Perl/Tk. That is what did it for me. Perl/Tk allows you to create your own GUI interface for your Perl code so it isn't always a terminal based program. While this is a bit more advance in Perl, they do go hand in hand, and I found it fairly easy to learn them as one rather than adapting Tk after learning basic Perl. I just dove right into it because at the time I had a project I was working on and I decided it was small enough for Perl (which is uber flexible BTW) but the project would eventually need a GUI... this I could see from the beginning. So away I went into the world of Perl/Tk.

As far as where/how to start... internet examples. Also try Devshed forums. Great for help with code but be advised that they are fairly advanced there. While they don't mind helping, they don't expect people to show up saying "Hi, show me this" They are of the mentality of "we'll help bump you in the right direction and possibly rework your code an rare occations" while this may seem a bit harsh or standoffish, it really gets the up & coming coders to learn the very basic art of... Learn to Learn!

At any rate, you should also try the O'Reilly site (spelling?) they have great books online, but you will need a subscription.

If you do persue Perl, then I welcome you to a world in which Perl will make your life Hell and Heaven both at the same time. But that is true of all programming languages. Once you get down the basics, you just have to use your imagination.

Ok, enough of my ranting.
Good luck!

L8rz
