### Post by greenstar on 2010-04-15
Note: There are some assumptions made in this post, primarily that you're running 9.10 Karmic, however this fix also worked on past releases with the difference being that the xorg.conf file was used in past releases and accordingly wasn't blank.

----------------------------------------------------------------------------------------------

When I can't get the Ubuntu login screen to display properly (it's squished or improperly sized), I use a custom xorg.conf file that I created for just this purpose a year or two ago. 

I read somewhere that they're doing away with xorg.conf completely. Until then, this should do the trick.

To access this file, do in terminal:
```
sudo gedit /etc/X11/xorg.conf
```This file is likely blank or non-existent on your system. If it's not empty when it opens, do the following:

1. Save it as xorg.conf.original in case you need to go back. 
2. Then close the text editor and 
3. back at your terminal, hit the up arrow & enter which should re-open the previous file and return us back to where we were.

Continuing along, if there's anything in the file at this point, hit Ctrl+A to select all and delete to clear it.

Next you'll paste the contents of the following box into your empty xorg.conf:

```
# xorg.conf (X.Org X Window System server configuration file)
#
# This file was generated by dexconf, the Debian X Configuration tool, using
# values from the debconf database.
#
# Edit this file with caution, and see the xorg.conf manual page.
# (Type "man xorg.conf" at the shell prompt.)
#
# This file is automatically updated on xserver-xorg package upgrades *only*
# if it has not been modified since the last upgrade of the xserver-xorg
# package.
#
# If you have edited this file but would like it to be automatically updated
# again, run the following command:
#   sudo dpkg-reconfigure -phigh xserver-xorg

Section "InputDevice"
    Identifier    "Generic Keyboard"
    Driver        "kbd"
    Option        "XkbRules"    "xorg"
    Option        "XkbModel"    "pc105"
    Option        "XkbLayout"    "us"
EndSection

Section "InputDevice"
    Identifier    "Configured Mouse"
    Driver        "mouse"
    Option        "CorePointer"
EndSection

Section "Device"
    Identifier    "Configured Video Device"
EndSection

Section "Monitor"
    Identifier    "Configured Monitor"
EndSection

Section "Screen"
    Identifier    "Default Screen"
    Monitor        "Configured Monitor"
    Device        "Configured Video Device"
    Defaultdepth    16
    SubSection "Display"
        Depth    16
        Virtual    1024    768
        Modes        "1152x864@75"    "640x480@60"    "640x480@72"    "640x480@75"    "640x480@85"    "800x600@56"    "800x600@72"    "800x600@75"    "800x600@85"    "800x600@60"    "832x624@75"    "1024x768@85"    "1024x768@75"    "1024x768@70"    "1024x768@60"    "1024x768@43"    "1152x864@75"    "1280x960@60"    "1280x1024@60"    "1400x1050@60"
    EndSubSection
EndSection

Section "ServerLayout"
    Identifier    "Default Layout"
    Screen        "Default Screen"
EndSection
```You can alter the line near the bottom that looks like this:
        Virtual    1024    768
to suit your monitor's native resolution. That resolution is the one the login screen will use if it's a resolution the system supports. Start with the 1024x768 and increase or decrease from there as your needs require.

Now hit Ctrl+S to save & Ctrl+Q to exit the text editor.

Restart your PC and you should be good to go.
