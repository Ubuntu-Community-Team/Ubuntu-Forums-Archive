### Post by talhaguy on 2010-03-05
Hello everyone,

I currently have Ubuntu 9.10 and Windows XP installed on my computer. I installed Ubuntu 9.10 on another partition again. Then I deleted that partition and gave the space back to the original Ubuntu.

Now its seems that grub does not know which file system to look for or something because when it is supposed to give me the OS selection screen, I instead get:

GRUB loading.
error: unknown filesystem
grub rescue> _

So I did some searching on the web and found a temporary solution to this:

grub rescue> set prefix=(hd0,x)/boot/grub
grub rescue> insmod (hd0,x)/boot/grub/normal.mod
rescue:grub> normal

I replace the 'x' with 5 just because I think I remember seeing that /dev/sda5 or something like that in grub previously when it was working. But anyway, using this method I can get into another grub screen and pick Ubuntu and go about my business.

But now every time I restart the computer, I have to go through this process. Is there some way to set the path or whatever to what it should be?

Thanks!

### Post by darolu on 2010-03-05
When you manage to log in to Ubuntu, open a terminal and run:
```
$ sudo update-grub
```
It should fix your grub problem. Notice this will work if you have GRUB2 only, if you made a fresh install of 9.10 you will have it, but if you upgraded from 9.04 you have GRUB 'Legacy' (grub1) and you'll have to configure your menu.lst file manually.

Also check the size of your Ubuntu partition with 'df' or 'fdisk -l' as your file system may be still configured to your previous partition scheme (smaller than your partition really is).
