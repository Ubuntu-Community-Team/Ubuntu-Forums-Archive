### Post by varunendra on 2012-11-05
> **anon_private said:**
> If I put the programme on a pendrive, does the pendrive have to empty, or can it have directories and applications that are not bootable?

will lubuntu (12.04 - alternate) burn onto a CD or will it need a DVD?
The pendrive should have **FAT32** filesystem, which is aleady the default filesystem for almost all such storage devices, so you shouldn't need to format it again.

It doesn't have to be empty - all you need is some empty space on it slightly larger than the size of the ISO (approx 210MB for Slax, or 39MB for Slitaz). You can use the rest of the space normally as storage. Just make sure you don't accidentally delete any of the OS folders or files (not a big deal if you do, you'll just have to copy it again from the iso).

Lubuntu ISOs are less than 700MB in size, so you can burn them to CD.

In general, if an ISO is larger than 700MB (precisely 703MB), then you'll have to burn it to a DVD. If it is less than that, burn it to a cd.

Running a proper installation will have at least two clear advantages -
1)  Programs will be directly read from hard disk - as opposed to a live  session where they are first 'extracted' to memory, then run from there -  thus occupying extra space in memory.
2) It will have the advantage  of using swap which can significantly improve performance by swapping  less used stuff in memory to the swap area (it's a virtual RAM on disk).  Although this can be done from a live cd as well. If you wish to try,  here is how -
* Using gparted in live session, create a 1GB swap partition on the hard disk. BE [COLOR=DarkRed]*CAREFUL  ! Partition manipulation tools should be used with extreme care if you  have important data on the target hard disk. Ask us a hundred questions  before proceeding if you don't know what you are doing.*[/COLOR]

* Once the swap partition is created, right-click on it and select  "swapon" in the drop-down list. It should now a 'key' icon beside the  partition name in the list below, means the swap is now in use.
You can now verify the usage of swap by ***free -m*** command.

**Extra info -**
*  (This step is not necessary ) Before closing gparted, make a note of  the swap partition's address that gparted shows (for example, let us  assume it is /dev/sda2). Knowing this address would be helpful when you  reboot and want to re-use it.

* Next time you boot with any  *buntu live cd, you can make use of this swap partition by running the  following command in terminal (assuming the swap partition is still at  /dev/sda2) -
```
sudo swapon -v /dev/sda2
```
Run free -m to verify that it is in use now.
