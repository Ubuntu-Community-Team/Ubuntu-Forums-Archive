### Post by oldfred on 2011-10-27
You have installed part of grub to sda1 and are missing boot.ini.

> sda1: ________________________

    File system:       ntfs
    Boot sector type:  Windows XP
    Boot sector info:   No errors found in the Boot Parameter Block.
    Operating System:  Windows XP
    Boot files:        /ntldr /NTDETECT.COM [COLOR=Red]/boot/grub/core.img[/COLOR]
Delete the /boot/grub folder. Or back it up. 

You need to create a boot.ini with Windows line endings. It needs to be one for sda1. You can manual copy another one or use Windows to repair it. Many boot scripts show an XP install and its boot.ini on this forum.

If you have your Windows XP install disk this will repair it (# is comment):
Description of the Windows XP Recovery Console for advanced users list of commands
[http://support.microsoft.com/kb/314058/](http://support.microsoft.com/kb/314058/)

BOOTCFG  /rebuild  # rebuilds boot.ini

A discussion about the Bootcfg command and its uses fix boot.ini
[http://support.microsoft.com/kb/291980](http://support.microsoft.com/kb/291980)
[http://pcsupport.about.com/od/fixtheproblem/ht/repairbootini.htm](http://pcsupport.about.com/od/fixtheproblem/ht/repairbootini.htm)

If editing windows files like boot.ini
(Use nano instead of gedit, it's better for dos-style linebreaks)
Linux, of course, uses only LF as newline and DOS is expecting CR/LF so text files look wrong in DOS.
New versions of gedit have an combo box under save as to choose windows format.
