### Post by mwi on 2007-01-15
Ok,

I figured it out.  A look at /tmp/env.properties.20314 (generated by the installer) showed:
lax.nl.env.ps1=\[\033[0;31m\]___\n\u@\h in \w \n > \[\033[1;37m\]
lax.nl.env.PS1=\[\033[0;31m\]___\n\u@\h in \w \n > \[\033[1;37m\]
and this is the source of the 'Malformed \uxxxx encoding' error.  Changing my PS1 parameter to ">" resolved the issue.  Just for the record, here is what I did to install maple (see also [http://ubuntuforums.org/showthread.php?p=1708055):](http://ubuntuforums.org/showthread.php?p=1708055):)
```

# as root do:
mount /media/cdrom
mkdir ~/tmpmaple
cp -R /media/cdrom/* ~/tmpmaple/
cd ~/tmpmaple/Linux/Disk1/InstData/VM
mv LinuxInstaller.bin LinuxInstaller.bin.pak
cat LinuxInstaller.bin.pak | \
    sed "s/export LD_ASSUME_KERNEL/#xport LD_ASSUME_KERNEL/" > \
    LinuxInstaller.bin
cd ~/tmpmaple
export PS1=">";
./installMapleLinuxSU

# to update to maple 10.0.2 (possible for the version I have):
cd ~/tmpmaple/
cp Maple1002LinuxUpgrade.bin Maple1002LinuxUpgrade.bin.pak
cat Maple1002LinuxUpgrade.bin.pak | \
    sed "s/export LD_ASSUME_KERNEL/#xport LD_ASSUME_KERNEL/" > \
    Maple1002LinuxUpgrade.bin
export PS1=">";
./Maple1002LinuxUpgrade.bin
umount /mnt/cdrom/
# after that you can remove ~/tmpmaple/ if you want to
# rm ~/tmpmaple/

# copy license file to <maple-dir>/license

```

And then execute <maple-dir>/bin/xmaple as non-root.

I hope that helps.

Martin
