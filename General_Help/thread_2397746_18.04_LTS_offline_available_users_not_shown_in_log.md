### Post by lmig on 2018-08-02
I have an Ubuntu 18.04 PC joint to windows ActiveDirectory using samba, with offline login active.

When the PC is connected to network I can login with any domain user, and a kerberos ticket for that user is stored in /tmp/

When I reboot the PC, and connect it to the network, it will show all previously logged users in the login screen. It doesn't matter if I'm connect or not to the domain network - it simply allows me to login in "online" or "offline" mode.

But, if I don't connect it to the network, only local users are shown in the login screen. If I try to login as other user, access is denied even if the user has an offline kerberos ticket available.
Oddly enough, just 90 seconds after login screen showing up, the offline users are finally added to the login screen, and offline login starts working.

My question is why do I have to wait 90 seconds for it to work? :?
I wanted to be able to do offline login immediately...

In samba configuration file /etc/samba/smb.conf I have the following winbind related entries:

winbind use default domain = true
**winbind offline logon = yes**
winbind nss info = rfc2307
winbind enum users = yes
winbind enum groups = yes
winbind nested groups = yes
winbind cache time=300

In /etc/nsswitch.conf I have the following related entries:

passwd:         compat systemd winbind
group:          compat systemd winbind
shadow:         compat winbind
gshadow:        files

What could I be doing wrong?

Thanks!
