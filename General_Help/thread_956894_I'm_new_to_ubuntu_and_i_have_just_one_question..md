### Post by MaxIBoy on 2008-10-23
Linux (Ubuntu is Linux with a certain set of pre-bundled programs and stuff) can run *some* Windows programs with the aid of a program called [W.I.N.E.](http://winehq.org) (Wine Is Not An Emulator.) Not all of them work though. In Linux, though, you will probably never need to defragment, because the swapping is on another partition and because the file-system is a better design.

[Linux is not Windows!](http://linux.oneandoneis2.org/LNW.htm) It is its own operating system. Its usually better to find a native Linux program instead of using a Windows program.

### Post by Pro-reason on 2008-10-23
That raises a whole load of questions and answers other than the one you meant to raise.

Why and what are you wanting to defragment?  Linux filesystems (ext3, reiserfs...) have no real need for defragmentation.

There are utilities available such as fidefrag which can run on Ubuntu and defragment any filesystem.

You should not start out by thinking of running a program from one operating system on another.  You should be thinking of tasks you need to complete, and then seek a program that will help you in that.

If you really, really need to run a Windows program on Ubuntu, there is an experimental program called Wine that can make that possible sometimes.  Don't rely on it.

I wouldn't run a defrag program through Wine.  If it does low-level operations, it probably will not work properly, and there's a risk of corruption.  If it doesn't do low-level operations, then it's really easy to find a native Linux program that will do the job.
