### Post by dino99 on 2010-05-02
sudo gedit /etc/default/grub

modify this line

GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"

to

GRUB_CMDLINE_LINUX_DEFAULT="quiet splash acpi=force"

### Post by tc7 on 2011-01-22
I've had the shutdown from indicator applet issue for some time - running Ubuntu 10.10 64 [2.6.35-25-generic] with a Gigabyte GA-P35-DS3P motherboard.

To resolve problem I made the following change in Power Management settings in the BIOS.

existing:
  ACPI Suspend Type: S3 (STR)

changed to:
  ACPI Suspend Type: S1 (POS)
Using this setting resolved the indicator applet reboot issue immediately.


According to the manual on the Gigabyte support site:

S1 (POS):
Enables the system to enter the ACPI S1 (Power on Suspend) sleep state.
In S1 sleep state, the system appears suspended and stays in a low power
mode. The system can be resumed at any time.

S3 (STR):
Enables the system to enter the ACPI S3 (Suspend to RAM) sleep state
(default). In S3 sleep state, the system appears to be off and consumes less
power than in the S1 state. When signaled by a wake-up device or event,
the system resumes to its working state exactly where it was left off.



I will try removing acpi=force from grub and grub-update to see if this still works (I suspect it will).
