### Post by PABlanche on 2013-08-28
Hi,

After un-installing re-installing Ubuntu 13.04, my computer was not getting out of suspend mode properly:
 The GUI was frozen in the login window, mouse was running extra slow, could not enter my pswrd.

I tried the old fix I used to have for my 10.10 system but it did not work:
```
create files : /etc/pm/config.d/00sleep_module and /etc/pm/config.d/unload_module
add line to files : SUSPEND_MODULES="xhci-hcd"
```

I tried the fix people indicated for the 12.04 system, did not work either:
[http://ubuntuforums.org/showthread.php?t=1978290](http://ubuntuforums.org/showthread.php?t=1978290)

By looking at the terminal <ctrl>+<alt>+<f1> when resuming from suspend, I noticed some error:
nouveau E[ ... pgraph ... page-not-present.

So apparently the problem was from the nouveau display driver. When looking at the configuration the graphic card was not the one I am running: a nvidia NVS 5100M/pcie/SSE2.

I went to the software&updates -> Additional drivers, the Xorg drivers - nouveau display was selected. I selected the nvidia 310.44, then restart. Now the suspend function is working properly.

With the nvidia driver the startup is not as clean though, I see a bunch of error signals flashing like: "kvm disable by bios", and some colored lines that disappears too fast for me to take note.
The font for "Ubuntu 13.04" during start-up is messy. But everything seems to work properly once logged-in.

Hope this helps
