### Post by r m h on 2010-05-13
You may also want to purchase and read some of the O'Reilly books on Linux and Unix.  [http://oreilly.com](http://oreilly.com) is a place to start.  Particular interest may be [http://oreilly.com/pub/topic/linux](http://oreilly.com/pub/topic/linux)

Once you start to get to know the philosophy behind the way Unix was created, you can start applying that to Linux and see that it has a similar philosophy.  

Many things in Unix are the same in Linux.  How's that for an overly broad generalization?  

Let's not get into the BSD vs. SYSV or CentOS vs Debian arguments.  Let's keep things at a higher level for now, such as simple commands that you can pipe together to mash up an end result that you're looking for, and super powerful command line commands that allow you to process TBs of data without endlessly mousing and clicking until you'd rather poke your eyes out with a rusty spoon.  Think sed, awk, grep, and xargs to name but a few...

### Post by sedawk on 2010-05-14
If you want to know about all the fancy command line
tools GNU/Linux has, I would recommend to read the man pages
of all commands from a to z.
E.g. man pages of section 1 (see "man 1 intro") contain
user commands and section 8 (see "man 8 intro") contains
commands for system administrators.
All commands of section 1 are located in a directory like
/usr/share/man/man1.
(A command reference book would be handy, but do they exist
 anymore? I remember a two volume set from Solaris or
 Ultrix that I used for that purpose ...).

Another way is to work task-oriented (Learning-by-doing,
Learing-by-example,  etc.). Give yourself a (simple) task and
then try to find out how to do it properly. But I guess that 
ould be hard if you start with the wrong tasks ;-)
Get one of those Unix books for beginners. Check your
local library. If english isn't your native tongue try to
get a good book in your language first.
Especially if you want to learn the command line tools you
should also look at older books if available.
