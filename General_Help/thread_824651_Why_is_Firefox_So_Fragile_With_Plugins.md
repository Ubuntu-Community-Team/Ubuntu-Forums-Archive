### Post by Cerin on 2008-06-10
I reported this bug a while ago, but since no one seems to care, I'd like to get some feedback. Why is Firefox so fragile when it comes to embedded plugins like Quicktime/Mplayer/Flash/etc? If these ever crash, they take Firefox along with them. Fortunately, Firefox's session saver helps the recovery process tremendously, but still, why can't Firefox just let the plugin die and carry on with its business?

I understand that the Firefox crew can't be responsible for crap third-party software, especially from Adobe, but I can't imagine it'd be terribly difficult to add some protection around the plugin system so that one tiny bug in a flaky plugin doesn't destroy your browser.
