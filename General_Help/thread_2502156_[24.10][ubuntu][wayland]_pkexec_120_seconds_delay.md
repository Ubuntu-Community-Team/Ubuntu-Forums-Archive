### Post by maniacx2 on 2024-11-03
I am author of Gnome extension [B]Battery health charging
[/B]The extension uses pkexec with combination of polkit rules to change the charging threshold in sysfs node or  package such as libsmbios / dell command center / framework_tool / ectool which all require privileged access.

User using ubuntu reported this, so I installed ubuntu 24.10 and started debugging.
I found that script/command  executed using pkexec takes approximately 120 seconds delay to execute.
And delay only happens in wayland. In Xorg the pkexec command/script executes instantly.
I have experienced this issue in **Ubuntu 24.10 running Gnome47 / polkit 124** and **debian trixie running Gnome 46 (Now also updated to 47) /  polkit 125,** but the problem appears only in wayland and only appear when command with pkexec is executed through gnome-shell.
I have **Fedora 40 running Gnome 46 with polkit 124** and **Fedora 41 Gnome 47 with polkit 125 **do not have this problem and the command executes instantly in Wayland and in Xorg.

I thought, it could be something to do with my extension, so I created a simple test gnome- extension that do not use any polkit rules/policy and when enabled, uses Gio.Subprocess runs simple command such as **pkexec uname -r** / or **pkexec echo "hello". **
The polkit gnome agent password dialog pops up instantly but the command take roughly 120 seconds to execute.
I have been fighting this for past month and still haven't found out what is the issue. Comparing logs, environment between ubuntu Xorg and Wayland, and Fedora Wayland, I haven't found anything strange.

I hope someone here can figure out this issue, as it only affects ubuntu 24.10 wayland / debian trixie. Fedora do not have this issue.

**To summarize.**

* **Ubuntu / Debian Wayland:** Executing command using pkexec from **gnome-shell-extension** cause [B]delay 120 seconds.
[/B]* **Ubuntu / Debian Xorg**: Executing command using pkexec from **gnome-shell-extension.** There is **no delay.**
* **Ubuntu / Debian Wayland/Xorg**: Executing command using pkexec from  **shell (gnome-terminal)**. There is **no delay.**
* **Ubuntu / Debian Wayland/Xorg**: Executing command using pkexec from in gjs script (**gjs-console**). There is **no delay.**
* **Fedora Wayland/Xorg**: Executing command using pkexec from **gnome-shell-extension.** There is [B]no delay.
