### Post by wlk on 2010-01-08
I had a similar problem where I would run the grub update, see everything in the grub.cfg, but then those changes would not show up on the menu.  After I had installed karmic, I installed another distro (64studio) which uses grub 1, and didn't ask what I wanted to do with bootability.  Sucks that this is not forward compatible.  I have a separate partition for /boot/.  In trying to fix my problem, I copied my /boot/ partition contents into the /boot/ directory on my root partition.  Eventually got karmic working again by reinstalling grub2 using my install disk, but then I ran into the menu issues.  Finally (tonight :) it dawned on me that it could be getting the menu from the /boot/ info on my root partition, so I unmounted /boot/, copied the grub.cfg and grub env from the boot partition to my root partition, recycled, and voila.  I guess now I have to figure out where the mbr is pointing, and I'm not smart enough for that.
