### Post by the_nite_owl on 2007-10-12
I have not worked with Canon scanners but I have recently setup my HP all-in-one printer under Feisty using the HPLIP drivers provided by the HP Linux Imaging and Printing project.
[http://hplip.sourceforge.net/](http://hplip.sourceforge.net/)

They have drivers and instructions for a large number of HP products that have no native linux drivers from the vendor.  If you find an HP product that you might consider then you could look there and see if it is supported.

My HP OfficeJet 3300 printer has a scanner and slid/negative scanning capability.  I have not tried the negative scanning ability under Linux but I have used the scanner for scanning pictures and documents into Linux.

Also, though this is not a solution as much as a work around, you might consider installing Virtual PC software and installing Windows as a virtual.
Then if there is any hardware or software you cannot get working under Linux you can set it up in the Windows virtual OS.
