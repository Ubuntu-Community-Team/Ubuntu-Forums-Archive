### Post by smcleish on 2009-06-02
Hi,

After making sure that the live CD of jaunty worked fine on my machine, I carried out an upgrade from intrepid. I've tried various different ways to get X windows / gdm to work, using the vesa drive and/or the free ati driver (the proprietary driver no longer supports my video card). With the free ati driver, the Xorg process hogs 99% of CPU all the time, and with the vesa driver, X windows crashes about once an hour in normal use (for me, which includes a lot of audio editing in audacity). Neither of these options seem to output anything to any useful log file that I can see. I can also get a blank screen, which is what I get from the xorg.conf file installed by the upgrade using hal to find things, rather than the previously working xorg.conf. I also get this error:

Xlib:  extension "Generic Event Extension" missing on display ":0.0".

repeated 5 times if I try starting a graphical application from the command line.

I've tried regressing X windows to hardy, the last version where things worked properly - I was doing this in intrepid, and it's one reason I was keen to upgrade. I used the instructions in [http://ubuntuforums.org/showthread.php?t=1027418&highlight=xorg+intrepid+hardy](http://ubuntuforums.org/showthread.php?t=1027418&highlight=xorg+intrepid+hardy) but this seems to make no difference. I also tried moving to the alpha drivers, as described at [http://ubuntuforums.org/showthread.php?t=1164013&highlight=jaunty+upgrade+%2Fdev%2Fcdrom](http://ubuntuforums.org/showthread.php?t=1164013&highlight=jaunty+upgrade+%2Fdev%2Fcdrom)

I'm hoping that someone will have some useful suggestions to try next. I've thought about backing up /home and installing from scratch, and even of replacing the video card. But I'm not a hardware expert, and so I'm not sure how big a job this would be or which would be a good card to get.
