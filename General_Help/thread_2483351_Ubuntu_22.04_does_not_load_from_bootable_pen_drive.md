### Post by C.S.Cameron on 2023-01-27
**Full install to USB**
You might prefer running a **Full install** of Ubuntu from the USB stick.
This would be identical to an installation to an internal drive except reading and writing to the USB stick would be a little slower.
Ubuntu will run in RAM if there is enough memory.
See: [https://askubuntu.com/a/1403793/43926](https://askubuntu.com/a/1403793/43926)

**Full install of Ubuntu vs Persistent install**
For the difference see: [https://askubuntu.com/a/1367263/43926](https://askubuntu.com/a/1367263/43926)

### Post by sudodus on 2023-01-28
You may find it relatively easy to get an **installed Ubuntu system in a USB drive via a compressed image according to** [**this link**](https://ubuntuforums.org/showthread.php?t=2474692).

I would recommend [the version 'Jammy' alias 22.04.x LTS](http://cdimage.ubuntu.com/ubuntu-server/jammy/daily-preinstalled/current/).

-o-

It will bring Ubuntu Server (without a graphical desktop environment), but it is easy to install the desired desktop environment (for example Ubuntu Desktop, Kubuntu, Lubuntu ... Xubuntu) after you have booted into the text user interface (and reboot into a graphical desktop).

```

sudo apt install ubuntu-desktop
#or
sudo apt install kubuntu-desktop
#or
sudo apt install lubuntu-desktop
#or
sudo apt install xubuntu-desktop

```

### Post by yancek on 2023-01-28
The image you posted shows the USB being formatted as ntfs.  FAT32 is the standard and some software to create bootable Linux USB's work better with FAT32.  I'm not familiar with Rufus so I don't know if this is the problem.  Have you tested this USB on another computer to see if it boots?  Have you tried different USB ports to see if it boots?  I've had computers with multiple USB ports that will only boot from one port.

If you are able to boot and use the USB created with Rufus, it will be only a 'live' system which is read-only and you will lose any data you create on each reboot.  Is that what you want?  Otherwise, a full install suggested above would be A better alternative.
