The blanket do-it-all fix for upgraders is to tag all compiz-related packages in Synaptic to be **completely removed**, not just removed. Then reinstall compiz.

If you keep the xserver-xgl package installed, Compiz will use software rendering, so you may want to remove it.

### Post by Zorael on 2008-04-27
Okay. For an Nvidia upgraders, this is everything that *should* be necessary. This will destroy your current xorg.conf and your compiz profiles.

Perform at your own risk. I would otherwise not recommend this to *anyone* but obviously something keeps going wrong.
```
*(remove extra repositories from software sources, /etc/apt/sources.list or files in /etc/apt/sources.list.d/)*

$ rm -rf ~/.compiz
$ sudo su
# cp /etc/X11/xorg.conf /etc/X11/xorg.conf.backup0804
# apt-get autoremove --purge nvidia* compiz* libcompiz* beryl* libberyl* libdeco* emerald* libemerald* xserver-xgl
# apt-get update
# apt-get clean
# apt-get install [COLOR="Cyan"]nvidia-glx-new[/COLOR] compiz compizconfig-settings-manager emerald
       *(make this [COLOR="Cyan"]*nvidia-glx-legacy*[/COLOR] if your card is oldish. see http://www.nvidia.com/object/IO_32667.html)*
# wget http://mirrors.kernel.org/ubuntu/pool/universe/e/emerald-themes/emerald-themes_0.2.1-0ubuntu1_all.deb /tmp/
# dpkg -i /tmp/emerald-themes*
# rm /tmp/emerald-themes*
# dpkg-reconfigure xserver-xorg
# nvidia-xconfig -d 24 --allow-glx-with-composite --add-argb-glx-visuals
*(add --no-composite if you installed the legacy driver package. see https://help.ubuntu.com/community/BinaryDriverHowto/Nvidia#head-c9d4ef0c1d4b240f14d85a42581e1307fee964a0)*

# reboot
```
This should be possible to do from the graphical interface, else you could print it and do it from the console. (such as Ctrl+Alt+F1.)


**envyNG basically does all this.** The parts related to your video driver, not the compiz/emerald parts. So if envy didn't work for you, shrug. If you're running the legacy drivers and everything else seems to work but Compiz, try disabling that Composite extension, supposedly helps.
```
$ sudo nvidia-xconfig --no-composite
```


To restore your system to default settings if things went down the drain:
```
$ sudo dpkg-reconfigure xserver-xorg
```
If you get to choose video driver, pick nv to get the non-accelerated and relatively safe one.

Even if your computer does not boot into the graphical interface, that doesn't mean you have to reformat and reinstall, just pick 'fix X' or 'safe mode' in the menu that pops up instead of the login window. Or enter recovery mode and perform the above dpkg-reconfigure command.
