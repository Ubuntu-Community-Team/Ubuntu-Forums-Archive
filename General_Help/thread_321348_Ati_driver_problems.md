### Post by madcow72 on 2006-12-18
Hey!  Have you looked into which driver you need for your particular card?  I have a Radeon 9250 and have realized that it *needs* the 8.28.8 driver which I downloaded from [here.]("http://ati.de/support/drivers/linux/linux-radeon-prer200.html")  (I cannot use the 8.26 driver, and am so far unsure of 8.32)

Regardless, I had the exact same problem installing in Edgy, and found the solution to my problems using Method 2 from [this link]("http://wiki.cchtml.com/index.php/Ubuntu_Edgy_Installation_Guide"), but using the 8.28 driver for my card.  

So I would, 1) make sure you are using the correct driver, and 2) use the directions from the site above.  

The magic happened with this part of the code:
```
sudo ln -sf bash /bin/sh
bash ati-driver-installer-8.28.8-x86.x86_64.run --buildpkg Ubuntu/edgy
sudo ln -sf dash /bin/sh
```  which I think will fix your problems.

Good luck!

### Post by madcow72 on 2006-12-18
For some reason, I often have to re-install the .debs and recompile the package before I get the video card working.  Although, usually this is because I tried to get dual monitors working at the same time - which seems to be a bad thing to do.  So first, don't try to do anything other than the:  sudo aticonfigure --initial  and  sudo aticonfig --overlay-type=Xv.   Another thing to try may be their suggestion of NOT running the aticonfigure, but instead do:
```
sudo nano /etc/X11/xorg.conf
``` and then replace the mesa driver with "fglrx".

By the way, are you sure you're using the correct driver for your card?  What does:
```
dmesg | grep fglrx
``` give you?
