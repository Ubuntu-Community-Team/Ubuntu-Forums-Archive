### Post by joiningtheherd on 2007-05-06
Sounds like you modelines for your monitor may be a bit off.  Try looking over this site to see if you can find an answer that will suit you.
```

http://wiki.x.org/wiki/FAQVideoModes

```
Alternately, if it's an LCD you can just force the resolution using lines like this from /etc/X11/xorg.conf (which is the configuration data for X11 -- the graphical desktop).  Use any text editor (try ```
$  sudo nano /etc/X11/xorg.conf
``` for editing at the commandline) to edit this file. This is in the section titled **screen**. 
```

        SubSection "Display"
                Depth   24
                Modes           "1280x1024"   "1280x800"   "1024x768"   "800x600"   "640x480"

```

This file is *the* one stop shop for the configuration values of the most important functions of your graphical interface.  Edit wisely, and make backups.  I cannot stress this enough!  I hosed many an X11 back in the late nineties due to seat of the pants x11.conf editing.

Good Luck!  I'll keep an eye on this thread.  If this doesn't work we'll break it down a few other ways, but this would be the most common.  If you do get it working don't forget to run ```
$ fglrxinfo
``` to make sure that the drivers are loaded correctly by X and acceleration is running.
Correct output should echo this:
```

me@linux-laptop:~$ fglrxinfo
display: :0.0  screen: 0
OpenGL vendor string: ATI Technologies Inc.
OpenGL renderer string: ATI Mobility Radeon X1300
OpenGL version string: 2.0.6334 (8.34.8)

```
With your adapter in place of my X1300.

Shibby.

### Post by mlind on 2007-05-06
What's the output of
```

glxinfo | grep direct

```


In case you're using fglrx and you see drm module in the output of
```

lsmod | grep drm

```

then try this on on a separate tty (Ctrl+Alt+F1)
```

sudo invoke-rc.d gdm stop
sudo modprobe -r drm
sudo modprobe -r fglrx
sudo modprobe fglrx
sudo invoke-rc.d gdm start

```
