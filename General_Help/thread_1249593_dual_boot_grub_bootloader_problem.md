### Post by gali98 on 2009-08-28
Sorry. I had written a post up and apparently it didn't get submitted. >:(
Anyways... um..

The easiest way would be to run update-grub from centos. If it works right, that's all you'll have to do. (Test it then reboot and see if you can boot ubuntu from the centos drive.)
If not, then boot the centos drive. From there, mount the ubuntu drive and open the menu.lst from the ubuntu drive. Copy all ubuntu related boot info to the centos menu.lst. That should be it. If you update the ubuntu kernel you would have to do this again (Unless of course the update-grub method works.)

I noticed you used the chainloader +1 entry. You shouldn't have to use this since grub can boot off other drives. (it would save time without that entry.)

Hope that helps. Sorry for the delay.
Kory
