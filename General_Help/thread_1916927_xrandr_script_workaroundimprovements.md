### Post by Shinon Yoshida on 2012-01-29
Hi all.
I've got a monitor that has problems with any computer not auto detecting it's supported resolution modes. I used xrandr to manually add the resolutions and this has worked, it took me awhile but I finally figured out a way to make a script to automatically start the needed commands on startup, however it's not fully satisfying as it has some kinks in it.

Here is the script I set up:
```
#!/bin/bash

xrandr --newmode "1368x768"   85.25  1368 1440 1576 1784  768 771 781 798 -hsync +vsync
xrandr --addmode VGA1 1368x768
xrandr --newmode "1280x720"   74.50  1280 1344 1472 1664  720 723 728 748 -hsync +vsync
xrandr --addmode VGA1 1280x720
xrandr --newmode "1280x768"   79.50  1280 1344 1472 1664  768 771 781 798 -hsync +vsync
xrandr --addmode VGA1 1280x768
xrandr --output VGA1 --mode 1280x768
sleep 10
nautilus -q
nautilus
```
I used this script on startup, unfortunately I have to press enter on a error on startup (since the modes aren't automatically detected and the last mode I used at startup was the one generated by xrandr, Ubuntu is trying to load something it thinks doesn't exist). Then my script starts loading. I had to do nautilus -q because the nautilus theme turned into something really close to Windows Classic, I'm assuming it's a gnome classic thing of some sort.

Anyways, I'd like to know if there is a better approach to this. I'm on Ubuntu 11.10 desktop.
