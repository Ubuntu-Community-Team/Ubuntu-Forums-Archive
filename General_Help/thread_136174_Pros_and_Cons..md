### Post by Quirky on 2006-02-25
At work I use Suse 10 (recently "upgraded" from 9.1 with a new install), at home I use Ubuntu 5.10. I wouldn't use Suse at home since I prefer Ubuntu's more open philosphy, but they are both good distros. Of the two, Ubuntu feels a lot tighter - it installs the best stuff for what you need and doesn't mess around. Suse tends to throw in the lot by default and let you guess which is worth your time.

**Suse Advantages:**
* Graphical Install
* Flashier boot screen/shutdown screen
* Automatically chooses the best kernel for your hardware (i686, smp, etc)
* Beagle beta installed by default (Ubuntu's kernel is not new enough to use it efficiently, perhaps in Dapper?). V. helpful at work this one.
* "Enterprise" edition of Suse came with Java, Flash, etc all set up. MP3 too perhaps, I haven't looked.
* Yast configuration makes life easy for most things...

**Suse Disadvantages:**
* Duplicated programs: Mozilla AND Firefox AND Epiphany by default!? Konsole and Gnome-terminal even if you choose Gnome-only. 
* The Suse Menu is horrible to navigate compared to default Gnome one. duplicate entries, things in odd categories, etc
* If Yast can't configure something, then you may have a hard time tracking down which file you need to edit since it could well have a different location to the "standard" configuration file setup.
* Documentation is a bit sparse at times, but this is probably as bad in Ubuntu just that I use other programs at work and haven't checked at home.
* Language set up is not as good in Suse as in Ubuntu - at home I can have a Spanish desktop for one user, another in English, or spell check in one language or the other in most programs. It was very easy to set up in Ubuntu. This has not been possible so far in Suse - you can add a secondary language, but it doesn't seem as well integrated as in Ubuntu. I doubt many people would care too much about this option as long as the primary language is set up ok.
