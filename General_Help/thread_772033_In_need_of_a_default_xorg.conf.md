Edit: since you want one, here is mine. (see forrestcupp's post first.)

```
# xorg.conf (X.Org X Window System server configuration file)
#
# This file was generated by dexconf, the Debian X Configuration tool, using
# values from the debconf database.
#
# Edit this file with caution, and see the xorg.conf manual page.
# (Type "man xorg.conf" at the shell prompt.)
#
# This file is automatically updated on xserver-xorg package upgrades *only*
# if it has not been modified since the last upgrade of the xserver-xorg

#
# If you have edited this file but would like it to be automatically updated
# again, run the following command:
#   sudo dpkg-reconfigure -phigh xserver-xorg

Section "InputDevice"
	Identifier	"Generic Keyboard"
	Driver		"kbd"
	Option		"XkbRules"	"xorg"
	Option		"XkbModel"	"pc105"
	Option		"XkbLayout"	"us"
EndSection

Section "InputDevice"
	Identifier	"Configured Mouse"
	Driver		"mouse"
	Option		"CorePointer"
EndSection

Section "Device"
	Identifier	"Configured Video Device"
	Driver		"nvidia"
	Option		"NoLogo"	"True"
EndSection

	Identifier	"Configured Monitor"
EndSection

Section "Screen"
	Identifier	"Default Screen"

	Device		"Configured Video Device"
	Defaultdepth	24
EndSection

Section "ServerLayout"
	Identifier	"Default Layout"
  screen "Default Screen"
EndSection
Section "Module"
	Load		"glx"
EndSection

```

It doesn't work like you're thinking.  There is no universal default xorg.conf.  When you install Ubuntu, it creates one that is specific for your machine.  There's no getting around having to reinstall the restricted drivers.

If you're using Hardy, one way you could get back to what your original was is to boot to recovery mode.  It will give you a menu where you can choose to fix X.  That will reset it for you.  Then you can choose to resume the normal boot process and you will boot into Ubuntu with xorg.conf set like it was the first time you booted, which means you will have to set up your restricted drivers again.

It's not that no one wants to share with you; it's just that it doesn't work that way.

Edit:
Read what I said, and don't use some other guy's xorg.conf unless you want trouble.
