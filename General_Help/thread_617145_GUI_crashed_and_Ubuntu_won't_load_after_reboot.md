### Post by Thyme on 2007-11-19
When Ubuntu is finished booting and you're seeing "Running local boot scripts (/etc/rc.local) OK", then try and press <cntrl><alt>F6 to log into a virtual terminal i.e. the control key + alt key + F6 key all together. If F6 doen't work then try all the other F keys until you see a virtual terminal. 

There is a problem with Gusty though where you can't access any of the virtual terminals.

You'll then need to login as root, mount  you flash drive or windows partition, copy the xorg.conf and Xorg.0.log files over and post it with Windows!!

Let me know if you're need help :)

### Post by Thyme on 2007-11-19
```

(++) Using config file: "/etc/X11/xorg.conf.failsafe"

```

Your system shouldn't be trying to load this. Remember that you need these files from the partition with Ubuntu already installed on it. The Live CD has it's own "xorg.conf" and "Xorg.0.log" files.

In your xorg.conf file, change the following:

```

Section "Device"
	Identifier	     "Intel Corporation Mobile GM965/GL960 Integrated Graphics Controller"
	Boardname	     "intel"
	Busid		     "PCI:0:2:0"
	Driver		     "i810"
	Screen	0
EndSection

```

Don't know if this will fix it but it's worth a shot!

to:

```

Section "Device"
	Identifier	      "Intel Corporation Mobile GM965/GL960 Integrated Graphics Controller"
	Boardname	      "intel"
	Driver		      "nv"
EndSection

```
