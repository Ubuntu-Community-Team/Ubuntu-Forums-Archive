### Post by TheFu on 2020-02-20
Using testdisk wasn't the directory where to recover files required?  All the files recovered would go there.  Move the file you want to keep to other storage and delete that area.

Would really be helpful to know which file systems are involved.  Boot from a "Try Ubuntu" flash drive, mount all the partition on the 1.5T dsk then run** df -hT -x squashfs -x tmpfs -x devtmpfs** .  So we can get a feel for what's happening.

I would start by using 'find' to seek out huge files - say over 2GB in size next.
```
find . -type f -size +2G -exec ls -lh {} \;
```
The first '.' says to search from the PWD down.  If you run it from / as the PWD, it will look through all mounts, all files, and will take a while.
Change the size to be smaller and smaller, but don't forget the leading '+' ... +1G then +500M then +250M ....
Those are the most-bang-for-the-effort files to remove.
