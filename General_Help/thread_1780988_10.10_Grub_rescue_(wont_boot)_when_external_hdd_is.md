### Post by 23dornot23d on 2011-06-13
Not really but its depends on when you put new OS's in and when you update them on each drive,

 ..... I use 4 USB drives each with a grub on each them and have no problems  ...... and use multiple operating systems on each. 
2 x 1 terrabyt drives .... 1 x 500 Gig ..... 1 x 320 Gig ...... 13 Operating systems in all on them .....

So I can take them and plug them into other computers and have my full system and all the files available ready to work .....

You have to decide how you want to organize the USB setup ....... 
but there is no other way that I know where you can plug a USB in without a boot option
and expect the BIOS to boot from it by putting in the boot list first  .....
(  With no boot option on it .... it should fail ..... the BIOS expects some boot information )

Unless the BIOS supports a way to time it out after the error booting the drive in question .......

Ubuntu grub is not being used in the problem you have here ....... 

Its the fact you are asking the BIOS of your computer to boot from a boot option that is not there ......
 the fact it has not got a boot option on that drive should cause problems for the BIOS .... if its plugged in
before you boot it up - so here it will usually give you a error saying cannot boot ........ from usb xx

That happens before you even get to the UBUNTU GRUB .....

But if you had a boot option present then the BIOS would quite happily use it and your problem would be solved ..... it would see the boot record on the drive and then use it.

So with multiple disks ..... 

I have a Grub on each of the USBs the menu on each relative to the operating systems on it ........
( but obviously it will pick any other operating systems on other disks too and list them.)

My main Drive sda ....... has a grub and also lists other operating systems - I can do a update-grub
with a USB plugged in ...... ( but this does not mean that I cannot unplug the USB )
as the sda grub keeps the first options relative to this drive first in the Grub menu.

It just means if you do have a USB plugged in when you update-grub using your main OS on sda that there will be entries in the main sda grub that no longer work for sdb if it is unplugged
but the system will still work.

USBs can be plugged in once you are running your desktop and the System will also pick them up that way too ....... just unmount them if you are going to unplug them while still using the desktop though.

Its a little out of date but should give an idea of what can be done with USB's



[B]I would put at least one operatng system on the USB though ..... 
( mainly there for updating the grub on it ...... sdb )

Your main sda ..... you can update it when you run a new O/S from it  ...... and it keeps itself updated ..... this was as each time a new system is added - it runs update-grub on the installation.

Then this way the drives remain independent of each other .....
