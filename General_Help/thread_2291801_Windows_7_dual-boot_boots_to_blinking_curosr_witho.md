### Post by SagaciousKJB on 2015-08-22
I'm really confused as to why this is happening, because it would seem that my GRUB entries are correct.  Basically, any time I have a CD in the drive which is bootable, I can load up Windows 7 through GRUB but if I don't have a CD in the drive during boot then it just leads to a blinking cursor.  Whatever is in the grub entry doesn't seem to matter because I've gotten it to boot with just

```
root=(hd1,1)
chainloader +1
```

...with SuperGRUB detecting any OS.  However if there is no bootable Cd in the drive, then it just simply leads to a blinking cursor.

### Post by yancek on 2015-08-22
Since you are posting at the Ubuntu forums, can we safely assume you also have some version (which one) of Ubuntu installed?
What bootable CD are you using?
The Grub entry you have should chainload the system on the second drive, first partition if you are using Grub2. 
Do you have Grub installed in the master boot record of the drive?   If not, that would explain this situation.

### Post by oldfred on 2015-08-24
I would use Boot-Repair or your Windows repairCD or flash drive to put a Windows boot loader in the Windows drive. Windows often works better when it is the first drive. Drive order can create issues.

Grub only boots working Windows, so you need a way to directly boot Windows or have a Windows repairCD. With a Windows boot loader in the Windows drive you should be able to boot from BIOS or one time boot key like f10 or f12.

With Multiple drives I do not like using Boot-Repair's auto fix. It just puts grub in every drive, so no matter what drive is set to boot from BIOS should work. 
But better just to have grub in Ubuntu drive and set BIOS to boot that drive.
