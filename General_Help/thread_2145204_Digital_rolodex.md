### Post by TheFu on 2013-05-14
There are 500 different solutions for this. The easiest is probably a personal wiki program. You can put a different recipe on each page and either manually link them by type or just use a search facility.

BasKet
Tomboy
TiddlyWiki
MediaWiki (probably more effort than useful)
come to mind, but there must be 50 other wiki programs in Synaptic for you to try.

Another option is to create text files - 1 per recipe - and use CLI tools (awk, grep, perl, python, etc...)  or a good search tool like **Recoll** to locate the drinks.  This would have less overhead, but Recoll is a heavy program - still, it is fantastic for locating things on a HDD. Basically, Recoll is like google for your entire PC.

Or you could create HTML pages and put each recipe into a different page. Then just use a web search tool like e-swish++ or htdig to index all the pages. This is method isn't too hard for someone with an admin background, but might be too much for a normal desktop user to accomplish.

I would avoid using any non-text files.  Also, be extremely dilligent about backups to different media. Murphy will wait patiently until you get everything created and entered, then cause some failure that will wipe out everything.

Sorting files happens automatically, so if you name them correctly (highly recommend you avoid spaces and funny characters inside filenames), then nothing more will be needed for organization.

Definitely avoid using Office-like tools. The extra overhead will be a waste and slow down searches a little and make you a slave to a specific program.

Anyway, those are a few thoughts.
