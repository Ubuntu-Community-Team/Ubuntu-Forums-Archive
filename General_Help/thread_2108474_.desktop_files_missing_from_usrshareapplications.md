### Post by steeldriver on 2013-01-24
Well I guess you could do something like this:

1. make a list of all the packages that place files in /usr/share/applications

I can think of a couple of ways of doing that - either using the contents of someone else's /usr/share/applications (or one from a running live CD/USB) or simply EVERY package that COULD have put files there. Both use apt-file (which is not installed by default) to find what package(s) the file(s) belong to:

```

while read -r -d $'\0' file ; do apt-file search "$file" | cut -d':' -f1 ; done < <(find /usr/share/applications -type f -print0) | sort -u > pkglist

```OR
```


apt-file search -x '/usr/share/applications/.*' | cut -d':' -f1 | sort -u > pkglist

```(the first gives about 80 packages on my system; the second gives about 2500 but takes less time)

2. read the package list from (1) and reinstall any package if it is currently marked as installed on your system - something like (the 'echo' is just there for testing - also you'd obviously need to do this with 'sudo')

```
while read -r pkg ; do dpkg-query -l "$pkg" 2>/dev/null | grep -E '^ii' && echo apt-get install --reinstall -y "$pkg"; done < pkglist
ii  activity-log-manager-control-center    0.9.4-0ubuntu3.2                        blacklist configuration for Zeitgeist (control center integration)
apt-get install --reinstall -y activity-log-manager-control-center
ii  aisleriot                              1:3.2.3.2-0ubuntu1                      Solitaire card games
apt-get install --reinstall -y aisleriot
ii  apport-gtk                             2.0.1-0ubuntu17.1                       GTK+ frontend for the apport crash report system
apt-get install --reinstall -y apport-gtk
.
.
.

```

**DISCLAIMER: I have never tried this and there may well be better ways to do it - please back up your data first in any case**!

It depends on how you reinstall a package. I can't speak for synaptic, but I tried apt-get on the command line for a package, and it recreated the .desktop file.

```
sudo apt-get install --reinstall *package_name*
```
