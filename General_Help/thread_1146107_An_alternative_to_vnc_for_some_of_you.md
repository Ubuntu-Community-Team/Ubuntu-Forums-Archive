### Post by panmanphil on 2009-05-02
I just spent a couple of days trying to figure out a way to get vnc working somewhat as well as remote desktop works between either a windows client and server or a linux client and server. My goal, control both my laptop and ubuntu studio desktop from my laptop keyboard and mouse. My physcial desktop has two monitors and just the laptop keyboard and mouse. 

Long story short, every combination of vnc clients I tried with the default vino server was really slow to keep the screen and mouse updated. Not really usable. I also tried the xrdp server, but it seems the project has died. 

Then I remembered something my son had done awhile back. [Synergy]("http://synergy2.sourceforge.net/") allows sharing of the keyboard and mouse between separate systems. You just move your most past the edge of the screen and it is on the other operating system. My second monitor is dedicated to the ubuntu install in this scenario.  It works like any multiheaded monitor setup except the computers are separate. It is totally seemless. Supposedly has clipboard sharing. Hmnnn....
ardour 2.7.1
(built from revision 4296)

Works!

I would say for anybody that has mutiple monitors and computers at their desktop this is a great alternative to VNC. 

PS I have a keyboard/mouse hardware switch, but since the new computer and old use different types of ports and because I want both computers live at the same time, it was not helpful.
