### Post by schragge on 2013-03-14
The only thing there that has something to do with locales is [localepurge](http://manpages.ubuntu.com/manpages/precise/en/man8/localepurge.8.html). The following will show if *locales* and *localepurge* are properly confured.

Results of *dpkg-reconfigure locales* can be seen with ```
grep '^[^#]' /etc/locale.gen
```
Results of *dpkg-reconfigure localepurge* can be seen with ```
grep '^[^#]' /etc/locale.nopurge
```

### Post by schragge on 2013-03-14
The only thing there that has something to do with locales is [localepurge]("http://manpages.ubuntu.com/manpages/precise/en/man8/localepurge.8.html"). The following will show if *locales* and *localepurge* are properly confured.

Results of *dpkg-reconfigure locales* can be seen with ```
grep '^[^#]' /etc/locale.gen
```
Results of *dpkg-reconfigure localepurge* can be seen with ```
grep '^[^#]' /etc/locale.nopurge
```

Note that if you accidentally removed the Italian locale with *localepurge* then the only way to regain it back is the full re-installation of _all_ packages that contain Italian localization. This could be achieved with
```
sudo sh /usr/share/doc/localepurge/reinstall_debs.sh
```

sudo dpkg -i /var/cache/apt/archives/*.deb
