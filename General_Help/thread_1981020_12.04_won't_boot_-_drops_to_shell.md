### Post by porchrat on 2012-05-16
> **kc1di said:**
> this page may be of help in understanding it. 
it's a little dated but think it's still relevent.

[http://linuxers.org/howto/how-configure-grub2-ubuntu-910]("http://linuxers.org/howto/how-configure-grub2-ubuntu-910")

Utilising the information in this link, my own grub files and a few other links that I had picked up along the way I edited the 40_custom file found in /etc/grub.d in the following way:

```
menuentry "Ubuntu 12.04 LTS (on sdc5 (use this one the sdb5 one is busted))" {
	insmod ext2
	set root='(hd2,5)'
	search --no-floppy --fs-uuid --set be2cf821-fb25-4f09-bd43-41ba05e57f92
	linux /boot/vmlinuz-3.2.0-23-generic root=UUID=be2cf821-fb25-4f09-bd43-41ba05e57f92
	initrd /boot/initrd.img-3.2.0-23-generic
}
```
This worked and 12.04 booted with no problem. So grub2 can indeed be told to hand over UUIDs in place of device names.

For those watching wishing to replicate this please note that you would need to replace the value for UUID with the UUID of your own partition which you would get using the command 'blkid'.


Here is the weird part: Once I got 12.04 booting properly I decided that it was time to update it. 12.04 proceeded to pull down, among a myriad of other updates, the new kernel update. Now my 2 (1 for each kernel) automatically generated grub entries for 12.04 all point to the correct partition. I can't believe it. All that effort just to have it sort itself out. Well at least I now know how to work with grub2. :P
