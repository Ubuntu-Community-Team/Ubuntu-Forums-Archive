### Post by jimbo99 on 2008-04-08
One common mistake people make is in trying to get Linux to be Windows (either in looks or in function).  One thing, that usually takes several retries (over time--installs, going back to windows, trying Linux again, going back to windows, trying linux again), is to finally conclude that Linux isn't Windows and never was intended to be Windows.

Linux isn't trying to clone Windows.  Linux isn't trying to do it the Windows way.  The idea is that there are better ways to get things done and everyone gets to help make that happen, whereas in the windows world only microsoft actually says what can be done and how.

If you go to [www.gnome-look.org](www.gnome-look.org) and you look at the screen shots you'll be quite impressed at how functional the Linux desktop is and how impressively configurable it is as well.  Everyone has managed to pick up their own little twist on how they want their computer to look.  It's brilliant.

Lindows was one of those distros that sought to look and act like Windows and they did a pretty good job, but Lindows is no more.  They are now Linspire.  I'm not sure how much effort they put into mimicing the Windows desktop look.

I'd say, learn the lesson now, and early.  I'd say that you should simply adopt the Linux flexibility and go with your own way of making it look.  If that way is still to look like Windows then so be it, but most people don't spend their time making their Linux boxes look like Windows, except maybe in the "vista" title bar effect which you see a lot of emerald window decorators that mimic the look of Vista.
