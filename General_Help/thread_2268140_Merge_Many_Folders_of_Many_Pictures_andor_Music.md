### Post by TheFu on 2015-03-06
Photo organization is a lifestyle.  Having too many photos in a single directory is a bad, bad, bad, bad idea. One tiny corruption in the diectory - poof - all that data is lost (but not gone). It is also a performance issue.  A directory with 200 files is much faster than one with 1000 or 10000 files.  Best to organize them using some method that has multiple directories. 

What I would do is:
Copy all the photos to a single HDD structure. 

If it is easy to do, I'd start with YYYY/ at the top level. The more metadata you can put into the directory structure, the better. Date tends to be the most important for a home photo collection.  Using other metadata, you can create virtual folders in different photo software.  It really depends on the file types and source of the files - jpg scans, PnS camera files, raw files each have different (on zero) metadata.  Even if the filename has the timestamp - split them up by YYYY/ at least for now.

If you don't have that metadata, I'd put the top level as either the person's computer name, then YYYY/ under that.
We are trying to get all the photos onto the same file system, so we can run a file duplicate finder. There are a bunch of these that work fairly well.   [http://en.wikipedia.org/wiki/List_of_duplicate_file_finders](http://en.wikipedia.org/wiki/List_of_duplicate_file_finders) has a list of duplicate finders. I've written one as a small scripting exercise - it was fun. Most people use one from that list. fdupes is popular.

I have a standard naming convention for directories and photos. Here's how I process them: [http://blog.jdpfu.com/2011/01/05/tips-for-digital-photo-organiz-storage-and-archival](http://blog.jdpfu.com/2011/01/05/tips-for-digital-photo-organiz-storage-and-archival) - of course, when all those files are over on a single disk, be certain to get a backup method working. Losing all that work would really suck.  Backups are critical, as you know. 

So - get the files in 1 place, do some minimal organization, remove duplicate files, start following a file name convention that works for you and place files into a directory structure you like.  All this directory work removes a dependency on a program that organizes files "virtually" and might change or disappear.  Adobe is found of doing this and storing the metadata in proprietary Exif fields or data formats.
