### Post by grasyop on 2016-07-22
Hello ! :KS

After an upgrade to *Lubuntu 16.04*, my mail notifier *Gnubiff* was upgraded to version 2.2.16 which happens to encounter [this bug]("http://osdir.com/ml/general/2016-02/msg31703.html"), so I want to upgrade it to 2.2.17, since [this]("http://gnubiff.sourceforge.net/changelog.php") says: "Changelog - Version 2.2.17: Fixed reading/writing config file".

So I downloaded *gnubiff-2.2.17.tar.gz*, unzipped it, tried to do ./configure, and there, I didn't have all what's needed to, **since I'm not used to do hand installations**. A few internet searches told me that I had to install gcc, g++, build-essential, libpopt0, libpopt-dev, pkg-config, gamin, and here I am:

When ./configure, I get: > No package 'gamin' found
But when sudo apt-get install gamin, I get: > gamin is already the newest version (0.1.10-5).

More precisely ./configure tells:

```
checking pkg-config is at least version 0.9.0... yes
checking for main in -lfam... no
checking for FAM... no
configure: error: Package requirements (gamin >= 0.1.0) were not met:

No package 'gamin' found

Consider adjusting the PKG_CONFIG_PATH environment variable if you
installed software in a non-standard prefix.

Alternatively, you may set the environment variables FAM_CFLAGS
and FAM_LIBS to avoid the need to call pkg-config.
See the pkg-config man page for more details.
```

I don't know what a prefix is and how to adjust an environment variable.
How do I set Gamin and other stuff needed in order to install Gnubiff 2.2.17 ?

** *** Edit *****

I got a bit further with:

sudo apt-get install libgamin-dev libgamin0
sudo apt-get install libgtk-3-dev
sudo apt-get install intltool
sudo apt-get install libpanel-applet-dev

** Where was I supposed to normally find out the exact name of all these dependencies I had to install?**

So, now :
./configure
make
sudo make install

...and it looks like it works! :)
