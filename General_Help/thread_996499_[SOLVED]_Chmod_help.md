### Post by Dr Small on 2008-11-28
I generally don't have these type of questions/problems, but the situation seems strange, so I thought I would ask.

I have a directory called:
```
/home/drsmall/public_html/testing/feedmailer/tmp
```

That has files generated by the webserver, and are owned by www-data:www-data. The directory is owned by drsmall:www-data, please see:
```
$ ls -l | grep tmp
drwxrwx--- 2 drsmall www-data 4096 2008-11-28 22:08 tmp

$ cd tmp/
$ ls -l
-rw-r--r-- 1 www-data www-data  632 2008-11-28 17:25 rss_17872e63844dd6b0547d1a5fd0bd800a.218d0b4b407df4843c6ee6010e352ea6
-rw-r--r-- 1 www-data www-data   69 2008-11-26 17:51 rss_17872e63844dd6b0547d1a5fd0bd800a.2738d92aeddfeb36df6fdbe1452b487f
```

I own all the directories from /home/drsmall to tmp; Yet when I go to chmod the files in tmp/, I am not allowed to.
```
$ chmod 606 *
chmod: changing permissions of `rss_17872e63844dd6b0547d1a5fd0bd800a.218d0b4b407df4843c6ee6010e352ea6': Operation not permitted
chmod: changing permissions of `rss_17872e63844dd6b0547d1a5fd0bd800a.2738d92aeddfeb36df6fdbe1452b487f': Operation not permitted
```

Simple question... Why?
I own tmp/ and I own all the other directories. Shouldn't I be able to change the ownership and permissions of these files that were generated by the webserver?

### Post by bab1 on 2008-11-29
Nope, you don't have the permissions needed to change the files mode or ownership.  Only the owner of the file has that right (or root).  Ownership of the directory des not give you owners rights to the files in it.

You might look into sticky bits on the directory.
