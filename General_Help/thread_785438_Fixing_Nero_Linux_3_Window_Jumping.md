### Post by brewstah on 2008-05-07
I am running Hardy Heron x86_64 with Gnome+Compiz and don't anticipate wanting to switch to anything else.  This problem didn't occur when I was using Dapper Drake x86 with Gnome+Metacity.  What I present here is a simple and effective workaround for those who've encountered it.

Here's what happens:

When running Nero Linux 3.5.0.1 while its window is maximized, the window will jump up or down about half the height of the title bar.  This makes it almost impossible to select files or open directories in the file browser, or change settings.  No Compiz setting seems to affect this irritating behavior.

Here's the fix:

While Nero Linux is running right-click on its taskbar icon and select **Unmaximize**.  Now move the left and right window borders to suit.  Easy!

(When I first saw Unmaximize in the taskbar icon context menu I actually laughed.  *Unmaximize?!?*  That was the first time I saw that.)

I would be using k3b instead.  The latest version seems to want to do all that I need.  Unfortunately it has a known bug that, until it's fixed, makes it useless to me.
