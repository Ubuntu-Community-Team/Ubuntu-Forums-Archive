### Post by nlong on 2008-03-07
It could be many different issues with hardware.  Since RAM is so inexpensive,  it would probably be best to start troubleshooting there.  I don't know of any one comprehensive linux system diag tool.  You might try Memtest86 for testing the ram. [http://www.memtest86.com/](http://www.memtest86.com/)
  I've gotten several brand new bad RAM modules in the past.  Otherwise, Having 768mb,  you have more than one chip installed.  I would start by pulling one of the chips and letting it run.  If it happens again, swap the removed module for another one.  However any piece of hardware could cause this problem.
