### Post by Calash on 2010-04-02
> **msy07 said:**
> Whoa, thanks guys for the help, I appreciate those links. Maybe I'm just making things hard for my self lol. :)

Alot of new Ubuntu users do.  They search for a way to install applications and tend to stumble upon source code instead of the easier methods.


Your first visit for any software install should be the Ubuntu Software Center under your applications menu.  This provides a user friendly method of installing from the software repositories (Online libraries of applications).

As you get more advanced you can move to Synaptic Package Manager.  It is a less friendly but arguably more powerful software installation.

The command line versions of this type of install comes in two flavours.  apt-get and aptitiude.  With many instructions you will see people reference these as they will work no matter what flavour of Ubuntu you are using (GUI instructions depends on your desktop shell).

If that fails you should then look for a .deb file to install from.  They do not update automatic but are installers designed for this environment and usually only require a click to get them setup.


Only after all these options are exausted, or if you really want to learn how to do it, should you consider source code compiling (andything where the install instructions include the commands make and/or make install)


On a side note, never be afraid of the terminal.  It is not harder, just different.  Learning it will allow you to get the most out of your Ubuntu install, or any Linux install.  While it is not necessary I highly recommend taking the time to become familiar.
