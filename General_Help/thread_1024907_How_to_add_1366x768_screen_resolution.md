### Post by loomsen on 2009-01-01
You can use cvt or gtf to create custom timings for your needs.

Usage:

cvt --help
gtf --help

Most likely you want to use cvt, gtf will create modelines which arent VESA compatible, but will also work.

To create an interlaced, reduced blanking modeline for instance you could do:
```

cvt -i -r 1366 768 

```

> 
[color=red]docter[~][/color] cvt -i -r 1366 768 
# 1368x768 119.79 Hz (CVT) hsync: 48.76 kHz; pclk: 74.50 MHz
Modeline "1368x768R"   74.50  1368 1416 1448 1528  768 771 781 814 interlace +hsync -vsync
[color=red]docter[~][/color] 


Oh, and then add this Option to your Monitor Section (the one for your TV, obv)

This is what mine looks like:
```

Section "Monitor"
    Identifier     "Monitor1"
    VendorName     "Unknown"
    ModelName      "CRT-0"
    HorizSync       28.0 - 55.0
    VertRefresh     43.0 - 120.0
# Default 60Hz
# 1024x768 59.92 Hz (CVT 0.79M3) hsync: 47.82 kHz; pclk: 63.50 MHz
    Modeline "1024x768_60"   63.50  1024 1072 1176 1328  768 771 775 798 -hsync +vsync
# Interlaced 60Hz
# 1024x768 119.68 Hz (CVT 0.79M3) hsync: 49.67 kHz; pclk: 66.75 MHz
    Modeline "1024x768_60i"   66.75  1024 1080 1184 1344  768 771 775 830 interlace -hsync +vsync
    Option         "DPMS"
EndSection
	##########################
Section "Screen"
    Identifier     "Screen1"
    Device         "Videocard1"
    Monitor        "Monitor1"
    DefaultDepth    24
    Option         "TwinView" "0"
    Option         "TVStandard" "PAL-B"
    Option         "Coolbits" "1"
    Option	   "metamodes" "CRT-0: 1024x768_60i; 1024x768_60"
    Option         "NoLogo" "true"
    SubSection     "Display"
        Depth       24
        Modes      "1024x768_60i" "1024x768_60"
    EndSubSection
EndSection

```

### Post by loomsen on 2009-01-03
You can delete all modes u dont need.

Further, check your xorg log for information which of the modes were validated and which were removed.

As I said, just run cvt in a terminal, copy and paste that created modeline to your monitor section, and check if its valid in your xorg logfile. 

If so, you can set your desired resolution in your Screen section.

Mine:
```

Section "Screen"
    Identifier     "Screen0"
    Device         "Device0"
    Monitor        "Monitor0"
    DefaultDepth    24
    SubSection     "Display"
        Depth       24
[color=red]        Modes      "1440x900_60"[/color]
    EndSubSection
EndSection

```

This Mode will be set then, if valid.
Just make sure your created modeline and the mode you set have a unique name, in this case 1440x900_60

*edit*
You dont create the modelines for your TV but for your video device, just fyi.

Drop into a terminal, stop your display manager and reconfigure your xorg, you could do this:
Hit Ctrl+Alt+F3 to drop into a terminal.
Login, then:
```

sudo /etc/init.d/gdm stop
sudo dpkg-reconfigure xserver-xorg
sudo nvidia-xconfig
sudo /etc/init.d/gdm start

```

Now you should have a cleanly configured xorg.conf, where most options are set to default.

Try and understand this file, using:
Check the documentation on Xorgs homepage, the nvidia documentation on xorg config is pretty good as well.


### Post by Saturisation on 2009-04-10
It took me three days of trail and error configurating, but finally my dual-monitor setups works at 2 x 1366x768.

Install the NVidia restricted drivers, look for version 173 (it's recommended, the latest (version 180) doesn't really work).

The restricted driver dialog pops up when you try to enable visual effects (System > Preferences > Appearance or System > Administration > Restricted Drivers). You need to reboot the pc to activate the driver. 

Get the nvidia-settings package, if it's not there (sudo apt-get install nvidia-settings). With this tool you can easily enable TwinView! (sure beats editing xorg.conf). Launch this tool by typing
"sudo nvidia-settings" in the terminal. You need to do the sudo if you want to be able to write your configurations to xorg.conf. 

Check the X Server Display Configurations tab. Down below is a button titled "Save to X Configuration File". Don't press it yet.

You can set appropriate resolutions for each monitor. If the "1366x768" is not available, select the something else. You can press "apply" to test the configuratoin, if it's working, hit the "Save to X Configuration File".

Logout and log back in. Open a terminal, type:

sudo gedit /etc/X11/xorg.conf

A part of my xorg.conf (which allows me to use the 1366x768 resolution):
```

Section "Device"
    Identifier     "Device0"
    Driver         "nvidia"
    VendorName     "NVIDIA Corporation"
    BoardName      "GeForce 7600 GT"
    Option         "NoLogo" "True"
    BusID          "PCI:1:0:0"
    #Add the line below to your "device" section.
    Option	"ModeValidation" "NoWidthAlignmentCheck, NoDFPNativeResolutionCheck"
EndSection

Section "Screen"
    Identifier     "Screen0"
    Device         "Device0"
    Monitor        "Monitor0"
    DefaultDepth    24
    Option         "TwinView" "1"
    Option         "TwinViewXineramaInfoOrder" "CRT-1"
    #If you have can find this line in your xorg.conf, alter all
    #resolutions to 1366x768!
    Option         "metamodes" "CRT-0: 1366x768 +0+0, CRT-1: 1366x768 +1366+0"


    SubSection     "Display"
        Depth       24
        #And last but not least, add Modes "1366x768"!
	Modes	    "1366x768"
    EndSubSection
EndSection

```
That should be it, good luck
