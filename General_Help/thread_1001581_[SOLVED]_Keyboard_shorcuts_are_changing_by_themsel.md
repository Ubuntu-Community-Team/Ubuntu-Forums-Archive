### Post by emil_donca on 2008-12-04
Hello, 

I have been using Ubuntu 8.04 for 6 months, at my workplace, so I am a beginner.

It is the second occasion when something like the following happens:
An action of a program is assigned a very trivial shortcut, i.e. a single letter, which interferes with normal work.

Occasion #1: Application gnome-terminal, action 'Open Terminal' is assigned shortcut 'N'. Meaning, every time I hit 'N' in the system, I open a terminal! I was able to reset the shortcut using System > Preferences > Keyboard Shortcuts, never thought about it again.

Occasion #2: Application Evince (v2.22.2), action 'Open...' is assigned 'A'. Now every time I do Ctrl + F to search something in a document, if the searched string contains 'a', the 'Open Document' dialog pops up!


I have no idea what I did to trigger such a change in shortcuts. I can't explain this to myself, other than by thinking that there exist shortcuts to change shortcuts??? which I must have accidentally pressed? So, sorry for the lack of info. Any help would be appreciated, thanks.

Emil.

---

### Post by emil_donca on 2008-12-04
Wow.... I just figured it out...

I found that my System > Preferences > Appearance > Interface > Editable Menu Shortcut Keys box was CHECKED.

So probably I was accidentally opening a menu and typing while hovering the menu item, thereby assigning a shortcut!

Perhaps someone runs into this like I did, if not, sorry for the post!

Emil
