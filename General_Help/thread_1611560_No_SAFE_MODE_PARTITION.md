### Post by wilee-nilee on 2010-11-02
So it looks as though you have overwritten Vista, **if you only have the one HD sda listed in the script.** Do you have a Vista install disc, and do you need to back anything up or are you backed up in any manner. Backups of the OS or whatever. If there is stuff on the computer you need and you can boot a live cd you can pull it off to a external thumb or HD.

The problem here is that you could if you get Ubuntu running, shrink it from the front of the disc to the end, left to right in gparted, but this takes a long time and will most likely need grub reloaded to boot before you reinstall Vista. 

Looks like a full reinstall for both OS to be honest, for the best results, although I don't know your situation fully.

It is difficult to really tell if Ubuntu is actually booting, and if your missing the second kernel line in the grub menu. The script shows the Ubuntu Recovery in the OS-Prober. Have you run in Ubuntu
```
sudo update-grub
```

The other problem is that sometimes a recovery can be done but I think with a overwrite and if you have run Ubuntu enough the recovery of anything Vista is unlikely.

Recovery is best run from a system that has not been overwritten or run again, from what I understand.
