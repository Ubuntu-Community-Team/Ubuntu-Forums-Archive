### Post by thndrbck2 on 2013-12-11
I ran into a problem with unetbootin that is serious.

I was using unetbootin to burn a few iso images to usb thumb drives.
My iso files were in the Downloads folder of my home folder
While clicking my way from the root to the Downloads I deleted two folders.
I deleted the Downloads folder, and while looking for it, deleted another folder as well.
I cannot locate those folders anywhere in the file system.
The deletion happened while I double clicked on them.
I thought perhaps I had accidentally moved them, but that doesn't seem to be the case.

After deleting the folders, I tried to shut down the computer using the shutdown button, but that didn't respond.
I had to use the terminal and the shutdown command to shut the computer down.

I have since tried using a usb thumb drive version to restore the folders but extundelete cannot bring them back

I urge extreme caution while using unetbootin during the file selection process.
Perhaps not using it at all on an important file system.

This is a potentially disasterous flaw.
