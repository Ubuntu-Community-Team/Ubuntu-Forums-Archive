### Post by tvtech on 2008-04-11
Gnome and KDE are windows managers.  they are the gui interface that your computer runs on top of X  which is the default linux display manager i.e. it controls how graphics make it to your screen.  there are others such as XFCE and fluxbox  as well. each one has a slightly different philosophy on graphical displays and configurability.  they are all much like "windows"  which was originally a gui display manager for DOS.  and was created to compete with other simple third party graphical windows managers such as mac os  but more specifically shell which was a third party product that ran on dos and gave you a gui interface to access dos computers.

if your really interested in the matter check out[ http://www.x.org/wiki/]("http://www.x.org/wiki/")

[http://www.fluxbox.org/]("http://www.fluxbox.org/")  this is the most significantly different windows manager from... well windows and even from mac   mac uses a special display manager called aqua because the way that X  implements itself.  it was designed to be fairly simple there is lots of information on it 

here[http://www.x.org]("http://www.x.org")

this website gives you some more display manager info and some more advanced features closer to aqua  [http://www.compiz-fusion.org/]("http://www.compiz-fusion.org/")  there is some talk about dumping X  in the future for better graphical displays of course this absorbs more hardware and Xorg is designed to support EVERYTHING  aqua by mac supports well the hardware they sell with it, and that's it.
