### Post by pauljwells on 2005-10-25
Try starting a terminal, type sudo su <return> password <return> 
this gets you to a root terminal. then type fireglcontrol. This way you get root permissions which the program needs in order to write the changes to xorg.conf.

Needless to say, it's a good idea to copy /etc/X11/xorg.conf to your home directory so you can copy it back if you lose X and end up with a screen full of garbage at reboot... 

I found that after running fglrx-config and then trawling through xorg.conf to pick out the bits that looked like they were doingwhat I wanted got me an xorg.conf without having to use the panel.

The most important thing in the file is the PCI address, make sure you use the one in your file if it differs from mine.

Yes the settings will stick if you are logged in as root. Also here's my working xorg for dual monitors with ati's 9800 card.



# /etc/X11/xorg.conf (xorg X Window System server configuration file)

#   sudo dpkg-reconfigure -phigh xserver-xorg

	Screen         "Screen 0" RightOf "Screen 1"
	InputDevice    "Generic Keyboard"
	InputDevice    "Configured Mouse"

	InputDevice    "cursor" "SendCoreEvents"
	InputDevice    "eraser" "SendCoreEvents"
