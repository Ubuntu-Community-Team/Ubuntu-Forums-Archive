### Post by qajaq on 2019-09-27
I've recently installed Lubuntu 18.04 (after having used Kubuntu for eight years) and under certain circumstances, clicking to open a folder opens in instance of Kaffeine rather than displaying a list of the folder's contents.

One such circumstance is when I click the "Directory Menu" icon in a desktop panel. The Directory Menu is configured to point to a root-level folder named /libdata, in which I have numerous sub-folders. When I click on the Directory Menu icon, I see an option for "Open" and one for "Open in terminal" followed by a list of the many sub-folders within /libdata. If I click on any of those sub-folders, I am then presented with a listing of their content, as expected. However, if I click on the "Open" option, a Kaffeine window opens. That is totally inappropriate and useless. (The "Open in terminal" option behaves as expected.)

Another circumstance is similar. I have an icon for pCloud in my System Tray. If I left-click on it (which I would expect to open either the pCloudDrive virtual drive on my HDD, or take me to my online pDrive account) or if I right-click and select "Open Drive" (which I would expect to open the local file on which the virtual drive is mounted), I again get a Kaffeine window. All the other options that are available from the right-click menu work as expected.

The only commonality I see is that both the /libdata and /home/username/pCloudDrive files are mount-points, one for an actual disk partition and the other for a virtual drive. But I cannot understand why clicking on an icon that is designed to point to either of these files/mount-points should open the Kaffeine software.
