### Post by jcolyn on 2011-01-05
Did you do a clean install after wiping the drive or did you choose to install over the entire disk/partition during install??

I have found that on some machines installing over an old install sometimes does not delete all the old partitions which means grub may be trying to read the old partition.

My preferred method is to first use super fdisk [http://www.ptdd.com/manual2.htm](http://www.ptdd.com/manual2.htm)
to delete the mbr. Be sure to download the bootable CD version. This will delete all partitions including any hidden partitions. Then wipe using a wipe program such as diskwipe [http://www.diskwipe.org/](http://www.diskwipe.org/) or kill disk [http://www.killdisk.com/](http://www.killdisk.com/) there are others available by doing a google search  disk wipe.

Once done you should be able to do a clean install and get it working..

### Post by Bucky Ball on 2011-01-06
10.04. Manually partition, use all others drives for data.

You can go back (cancel partitioning) and spend an hour or two playing around with the partitioning to make sure you get it right when you get to that part of the install so no panic. ;)

Disconnecting is only necessary if you don't feel confident in identifying the drives when you get to partitioning. At least you don't have to worry so much then about backing up all your valuable data. Not plugged in you can't erase it and we're only human; occasionally we hit the wrong button! Put it this way: they will show up as NTFS or FAT if you have them formatted that way. Just don't touch 'em in the partitioning process. The free space will be visible so aim for that.

DBan I have had some strange results with. DBanned a disk once when setting up Xubuntu for a friend and the disk was then not recognised at all by anything! DBan removes *everything* from the disk.
