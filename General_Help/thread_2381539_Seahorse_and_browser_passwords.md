### Post by again? on 2018-01-02
google-chromium already uses seahorse by default to save passwords.
In my opinion you are better off using something like keepassX to create an easily transferable local password data base and back that up somewhere.
On a new OS install, I copy in passwords from keepassX to chrome as needed where chrome then prompts to save in seahorse
for future use.

Chrome passwords stored in seahorse aren't easily transferable because the passwords are associated to a unique application number
as seen by double clicking a saved password in seahorse and viewing the "Details" tab.
