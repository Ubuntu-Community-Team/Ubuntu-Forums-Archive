### Post by QIII on 2018-09-29
Again, Ubuntu isn't "ours".  We are volunteers.  I don't work for Canonical (which distributes Ubuntu).  I own my own software company.

I can't say what will and will not work -- only that you will need to use an ARM port, not a normal x86 (PC) verison.  Usually each SBC manufacturer has their own port so that things work with their particular hardware setups.  The individual hardware manufacturers are responsible for their ported versions, not Canonical.  If something doesn't work, the manufacturer is the only one that can fix it.

You can find out more about Canonical's Ubuntu ARM port [here]("https://wiki.ubuntu.com/ARM/RaspberryPi").

You might try Raspberry Pi's Ubuntu MATE port [here]("https://ubuntu-mate.org/raspberry-pi/").  I don't know if it will work, but it is at least an ARM port.
