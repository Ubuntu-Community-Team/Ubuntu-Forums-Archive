### Post by 2CV67 on 2014-12-08
I have been using Rapid Photo Downloader for about a year, mainly with a Panasonic Lumix TZ18 & Ubuntu XFCE.

Recently, I changed to a new PC (64-bit), upgraded to Ubuntu 14.04.1, started using Unity & switched to a TZ35.

At some stage along the line (I think with the TZ35 but not 100% certain) RPD has started a new quirk when trying to mark multiple photos for downloading.

RPD finds all my photos OK.
I can select a block of photos I want to download OK.
But when I then check one of those photos, only alternate photos get a check mark!
If I go ahead & download then only the alternate photos are downloaded.
If I try to check one of the non-checked photos while the alternate ones are checked, nothing happens.
If I re-check one of the checked ones, the checked ones become unchecked & the unchecked ones become checked!
So I have the choice of downloading my photos in 2 lots, or manually checking each one I want to download (that works OK if I have not selected a block of photos first).

Not the end of the world, but weird.

Same result when trying to download directly from the camera (my preference) or directly from the card (RPD recommendation, I think).

I am using RPD 0.4.10-1, which I think is the current version?

Have I missed something, or is this a bug?
