### Post by tommcd on 2009-11-26
If you are **absolutely sure** you know which partition is your swap partition, you can activate it like this:
```

sudo mkswap /dev/sdXY
sudo swapon /dev/sdXY

```
Where X is the hard drive and Y is the partition number of the swap partition.
Again be sure you choose the correct partition. Choosing the wrong partition will wipe out any data that was on it.
Then run **free -m** and it should list your swap partition along with your memory usage.
To load swap automatically add it to your fstab. The entry will look something like:
```
UUID  none swap  sw  0  0
```
Run **sudo blkid** to get the UUID of the swap partition.

### Post by tommcd on 2009-11-26
These are the options in fstab for the swap partition on both my desktop and my laptop:
```
UUID none swap sw 0 0
```
The only thing that is different is the UUID.
Your entry in the fstab you posted in your last post:
```
UUID="e18d697e-577e-4aa5-b5b0-d60f679cf914" TYPE="swap"

```
does not make sense. If the UUID is correct (run **sudo blkid** to check), then just add these options after the UUID with at least 1 space in between each:
```

 none            swap    sw              0       0

```
It should work.

### Post by anaconda on 2009-11-26
heh..
You were supposed to add your swap partitions UUID after the UUID on the line in fstab.

But I would highly recommend using the "old" format 
/dev/sda5 none swap sw 0 0 
instead of the 
UUID=489372-3894werjfsdf-342348 none swap sw 0 0  
format, because the UUID will change every time you reformat your swap, or run mkswap on your swap. And the /dev/sda5 will always point to the right partition, (unless you eg. add a new hard drive or something..)
