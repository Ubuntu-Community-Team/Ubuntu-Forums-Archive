### Post by neo2001 on 2014-04-24
From time to time I'm using Handbrake to encode some videos. Naturally this comes with a very high CPU load on all cores.

While using 13.10 (Saucy) this wasn't an issue at all because the system was still fully responsive and there was no negative impact from the running encoding job at all - at least it wasn't really noticeable (as it should be on a multitasking capable OS).

Now, using 14.10 (Trusty), it's a real pain to work while encoding a video. Even typing text and opening menus is a little bit delayed. The whole GUI is sluggish and stuttering. As soon as the CPU load drops, everything feels more or less normal again - though I have to say that it still seems a litte bit less fluid compared to 13.10.

I did a clean install (no upgrade) - but I used my existing /home partition from 13.10.

CPU is a FX 8350 (8 cores, 4 GHz) and a Radeon 5850 (AMD/ATi Cypress PRO).
As on 13.10 I'm using the "radeon" open source driver (it is the recommended default anyway - and I never changed it because it worked fine).

Any ideas? Someone else having the same problem? :)
