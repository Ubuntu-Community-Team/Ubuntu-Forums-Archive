### Post by heindsight on 2009-01-23
the line you put in your .bash_aliases file should read:
```
alias rm='rm -i'
```

Also the changes you make to .bashrc don't take effect immediately, because .bashrc is only read when the shell is started - so these changes will take effect in any shell instance you start after editing the file. For any shell instance that was already running before you edited .bashrc, you can run ```
. ~/.bashrc
``` to have the changes take effect in that instance too.

### Post by ibuclaw on 2009-01-23
I prefer functions over aliases personally, myself...

[EDIT] updated this to avoid infinite recursion.
```
function rm {
command rm -i "$@"
}

```
Much easier to handle, AND you can put them anywhere you like in a bash command (aliases are restricted to being just the first command).
