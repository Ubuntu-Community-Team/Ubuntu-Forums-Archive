### Post by grahammechanical on 2021-12-08
I would suggest that you have an second install of Kubuntu/Ubuntu installed and experiment in the second installation before installing in your main installation. Either that or be prepared to reinstall after the OS has been messed up. This often happens.

Of the two choices LibreWolf has the most friendly installation method because it comes as either an Appimage or a Flatpack. Both Appimage and Flatpack versions require some command line skill as there is not a Ubuntu repository for LibreWolf.

[https://librewolf.net/installation/linux/](https://librewolf.net/installation/linux/)

If you are tempted by the claim that LibreWolf can be installed on Debian and therefore think that it must also work on Ubuntu, please note that the repository that has to be installed is an Opensuse repository and there is this warning:

> **Note** This is for Debian Unstable only - do not try to install this package on any other branch of Debian or Ubuntu/Mint, or you will have errors about being unable to resolve the dependencies. Instead, try the [instructions for other linux distributions]("https://librewolf.net/installation/linux").

Regards and Good luck.
