### Post by 3rdalbum on 2010-11-04
The Netbook Edition is the exact same as the normal Desktop edition, except that the interface has been modified to work better on the small screens that netbooks have. Netbook Edition requires 3D acceleration out-of-the-box, which basically means Intel graphics.

Your notebook, assuming it has Intel graphics, will run the netbook edition of Ubuntu but you might as well use the ordinary Desktop edition.

Incidentally, netbooks can also run Ubuntu Desktop.

EDIT: Essentially, use Netbook Edition or Desktop if you are on a netbook. If it's not a netbook, then use Desktop.

### Post by janpol on 2010-11-04
> **Carl Hamlin said:**
> Unless I'm severely mistaken, the amount of memory on a system is not the determining factor so much as whether or not the equipped processor(s) are operating 32 bit registers (eax, ebx, ecx, edx) as opposed to 64 bit (rax, rbx, rcx, rdx).

32 bit will run on 64 bit architecture, however the reverse is not true.
With 32 bits you can address up to 4Gb of Ram only (and its like 3.6Gb actually), that's why it is an issue. But Ubuntu uses PAE (a little trick in order to address more than 4Gb of Ram) when it detects 4Gb of Ram or more so, in Ubuntu (and Linux using PAE), it's not an issue.

Also, it's true that 64bit editions can run 32bit software, but sometimes some compatibility issues do appear (that's why Ubuntu recommends using 32bit edition for the desktop).

I am using Ubuntu 10.10 64bit on my main PC, and its great so far, never had a single problem. On the other hand, I don't have any advantages either. In theory, it should be faster, but you only feel the difference on apps that are optimized for 64bits (like video coding, rendering, etc..). Also, 32bit apps run slower on 64bits (because they need to do a little emulation first).

For general use, install whatever you want, most likely is that you won't feel the difference.
