### Post by freerick on 2007-03-20
I have three monitors and was able to get them to work with my Nvidia 6200 without much trouble, but I did have to play around with the drivers and xorg.conf for a while.
In order for you to extend your monitor across more than one desktop, you should use Xinerama. Add the following to your "ServerFlags" section:
```

Section "ServerFlags"
    Option         "Xinerama" "1"
EndSection

```


That should do the trick; if you continue to have problems, post your modified xorg.conf as well as /var/log/xorg.log.
You can also find a very good manual by nvidia regarding how to properly install and configure your card on linux at the following link: [http://http.download.nvidia.com/XFree86/Linux-x86/1.0-8178/README/appendix-p.html](http://http.download.nvidia.com/XFree86/Linux-x86/1.0-8178/README/appendix-p.html). (this is the specific section about multiple screens)


Good luck.

### Post by freerick on 2007-03-20
> **WickedAwsome said:**
> I've got an ATI card, isnt xinerama Nvidia only?

No, TwinView is an Nvidia-only technology, but Xinerama is a feature of the X server, so it's available in any linux distro that has a GUI.
Read the Nvidia documentation above, it explains the differences between Xinerama and TwinView. In my experience, you should be ok with either; I prefer Xinerama because that way I'm not limited to Nvidia's driver and I'm not limited to two screens. :)

### Post by freerick on 2007-03-22
If I had known that you're trying to use beryl I could have told you that it doesn't work with Xinerama :confused: 

The reason is that beryl depends on the RANDR extension to work. This is a built-in extension to the  X server, but it's automatically disabled when Xinerama kicks in. The reason is that RANDR is used to dynamically adjust the resolution/orientation of your screen and that is not yet implemented with multiple screens. You can confirm this by calling beryl from the command line.


This issue is known to the beryl developers and they are planning to make it no longer dependent on RANDR in a future release. Meanwhile, you'll be able to do it with TwinView if you have an nvidia card or can get your hands on one. Refer to nvidia's linux driver documentation on how to enable this. Conceptually, the setup is very similar to that of Xinerama, but if you don't want to mess with Xorg.conf, use the GUI driver configuration utility that the nvidia driver installs. Let me know what happens.
