### Post by escape on 2007-02-14
Many of you are familiar with the Glossy P theme for GTK: [http://art.gnome.org/themes/gtk2/571](http://art.gnome.org/themes/gtk2/571). 
This is certainly one of the better themes for GTK, and a must for anyone trying to make their desktop resemble OS X. It came out in December 2003, with a lot of workarounds for bugs that existed in pixbuf theming for gtk at the time. Since then a lot of those bugs have been fixed, and Glossy P now shows up a little funny.

The maintainer of the theme has a habit of saying he'll update the theme, but never gets around to it. If anyone has knowledege of pixbuf type themes in GTK, and would like to fix bugs in Glossy P, I know many people here would appreciate it. If no takers arise I may consider putting up a bounty (maybe other people would like to contribute to this too?). 

Below are a list of the bugs that I've noticed. If people are aware of any others, please post them here. Although I've desaturated the theme for a graphite look, the bugs are still the same:

[[IMG]http://img441.imageshack.us/img441/479/screenshotls3.th.png[/IMG]](http://img441.imageshack.us/my.php?image=screenshotls3.png)

1. Text entry fields in open office aren't rounded (or have any border). This is similar to #2
2. In rhythmbox the text entry manages to be rounded on one side but appears to be cut off.
3. When the scroller on vertical scrollbars is at the top-most or bottom-most position (or both simultaneously) the pixbuf engine can't render the scrollbar handle (the arrows on either end). This is actually quite annoying.  Note how on rhythmbox you see the top handle has defaulted to the basic image while the bottom one is kept. This happens in all gtk apps afaik except openoffice or themed firefox.
4. What the heck is this? It's on rhythmbox too, but more so. 
5. A default gnome-panel image would be nice. I can provide this one in a variety of resolutions if needed.
