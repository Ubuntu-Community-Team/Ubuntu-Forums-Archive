Have two systems with 12.04 and Libreoffice 4.x installed.  Fairly stock, other than using LO 4 PPA.

Upon updating today to LO 4.0.3.3, the Global menus are now empty.  They drop down like there's text in them, but they have no actual text.

Couldn't find anything useful via search.

(FYI "sudo apt-get autoremove libreoffice-gtk" will solve the problem by giving LO standard menus, but it does this by removing the gnome theming from LO, so it looks a bit ugly and loses some appearance adjustability.  It's a work around until the actual problem is fixed.)

### Post by cochisebt on 2013-05-20
Hi
I confirmed I have the same issue here.

To make Libreoffice working properly again, I uninstalled the following packages:

appmenu-qt
appmenu-gtk
appmenu-gtk3
indicator-appmenu

The result is that the global menubar is disabled. However, it makes Libreoffice works properly again. Uninstalling libreoffice-gtk and libreoffice-gnome packages makes libreoffice work again, but it is ugly, and unable to write characters such as "à" "è" "ç" "û", which are essential to write in French.

It seems that we have a problem of integration of libreoffice in Unity.
