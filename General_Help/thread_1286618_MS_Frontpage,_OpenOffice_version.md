### Post by Lars Noodén on 2009-10-09
> **Ottawajoe said:**

As techstop wrote, thankfully, no. FrontPage is (in)famously bad.  Back when there were software reviews one review said basically if you care about other people being able to view the page you stay away from it. 

As good as OpenOffice.org is for other things, don't use the HTML editor from OpenOffice.org, the HTML editor part is not good.  Any other editor is 'compatible' with OOo anyway. ;)

### Post by Lars Noodén on 2010-01-05
> **L4U said:**

OOoWeb has the closet look and feel to Frontpage than any of the few Linux WYSIWYG html editors out there.

I'm curious if any uses, or has used OOoWeb and how well it actually works for a beginner web building app?

OOo has some very good components, but OOoWeb is not one of them.  If you are serious about using a web editor to manage the whole site, then look at [bluefish](http://bluefish.openoffice.nl/), [quanta](http://quanta.kdewebdev.org/), or [kompozer](http://kompozer.net/).  

Myself, I find it much faster and easier to work directly with XHTML and CSS with the help of tidy and emacs.  However, I liked Quanta best when I tested the three, but I found that the business students at a junior college who tried the three favored Bluefish.  YMMV.
