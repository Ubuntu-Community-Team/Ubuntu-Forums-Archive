### Post by powerslave86 on 2009-01-12
I'm very new to Linux, and I am currently attempting to install Ubuntu 7.10 so that I can dual-boot my PC with Windows XP.

However, when I tried to install the 64-bit version, (in graphical safe mode and normally) my screen would go blank, and stay blank (for over an hour), even though there was hard disk activity indicated.  This also happened when I was checking the disc for errors.

Now when I try to install the 32-bit version, (in both modes) I get to the screen with the orange progress bar, but then my computer starts beeping continuously, and does not stop.

I looked for help online, and it has been suggested that this is a hardware problem (that something is failing or about to fail).  However, I do not face this problem when booting into Windows XP.

Could anyone please advise me on what to do, to fix the problems with installing either version?  I am completely lost, but I need to have some version of Linux running on my PC.  I have an AMD 5200+ processor (64 X2 Dual, 2.70 GHz), an NVidia GeForce 9600GT, and my motherboard is an ASUS M3N-H/HDMI.  Thank you very much in advance!

### Post by mssever on 2009-01-12
It does sound like some sort of hardware incompatibility. The fact that Windows works fine means nothing. You can try the alternate install CD, though even if you manage to install that way, there's a good change you'll just have more problems. Perhaps a better choice would be to try out another distro or two. The various distros aren't identical on the hardware front, and you might get lucky and find one that can handle your hardware.
