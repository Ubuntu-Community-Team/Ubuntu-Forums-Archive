### Post by MaxIBoy on 2009-06-14
You can pretty much pick one.

There are a few different text editors which work as terminal programs. 

[LIST]
[*]nano - pretty basic. It's almost 100% guaranteed to already be installed on your Linux computer, just type the command "nano name_of_file." If the file doesn't already exist, it will be created. A nice thing about nano, especially for beginners, is that there's a list of the most common key combos at the bottom of the screen. Just remember that "^" means "control."
[*]emacs - exact opposite of "nano." The key combos are very different than is normal, but they're easy enough to learn. (In fact, they're the same in nano, they're just not plainly labeled.) Emacs is big and bloated, but it's very full-featured. You could probably use your computer for days without leaving it-- it has multiple-file support, split-screen support, an email client, I think somewhere you can download a 3D renderer for it, and you can drop into another command line within emacs if you need to.
[*]vi (or vim) - Yeah, this is pretty much a sick joke, invented by Communists to stop Americans from getting stuff done. (Just kidding. If you take the time to learn it, I hear it's very nice.)
[/LIST]
I'd recommend just using nano to start with.
