### Post by lukeiamyourfather on 2013-08-26
I guess I'm not really sure what you're asking for then. If all you have are shortcuts and the original drive has been wiped/reinstalled then those files are gone. There might be partial data left on the drive but it won't be easy to recover, assuming there's even something left to recover. You could try this command.

```
strings /dev/sda1 > stringdump
```

Where sda1 would be the partition you're interested in and stringdump would be the output file where all the strings would get dumped. Basically you get a huge text file with every text string on the partition but nothing else (like file names or directories). If the blocks on the disk have been overwritten the data is truly gone forever. To avoid overwriting even more blocks don't boot from the disk or write anything to it.

Oh that sucks so badly. I seriously feel your pain.

One thing you might want to consider about how Windows copy works for certain file extensions: when you drag and drop files files from one location to another, Windows will sometimes create shortcuts to the original files instead of making copies of them. This is especially true for files having the ".exe" extension. The way around this is to copy and paste by either selecting the files and then choosing either copy or paste from the menu you get by right clicking your mouse button, or using copy [Ctrl+C] and paste [Ctrl+V] from the keyboard. _Always_ check that the files created in your destination are really copies of the files and not just links back to the originals, and NEVER rely on "drag and drop".
