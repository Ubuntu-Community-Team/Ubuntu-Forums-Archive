### Post by yjwong on 2008-09-02
You have multiple "device", "screen" and "monitor" sections, this may confuse the X Server. In your second "device" section, the X Server is configured to use the "ati" (open source) driver, whereas in the first "device" section, the X Server is configured to the the "fglrx" (proprietary) driver. This is inconsistent and thus may have contributed to the problem you are having.

What you can try is to remove the second "device" section.

### Post by pytheas22 on 2008-09-02
Yes, the multiple device sections may have caused problems.  How did they get in there in the first--did you add them when you were trying to figure this out?

You may want to run this command, which will replace your current xorg.conf with the default one generated when the system was installed (backup your current xorg.conf first if you want it for reference):
```

sudo dpkg-reconfigure -phigh xserver-xorg
```

Another thing you might try is starting compiz from the command line, which you would do by opening a terminal and typing:
```

compiz --replace
```

If the screen turns white, press control-C and compiz should die, reverting you back to metacity (the non-accelerated window manager).  But the output from compiz will remain in the terminal--see if it says anything interesting about why the screen turned white.  It might not say anything, but it's worth a try.
