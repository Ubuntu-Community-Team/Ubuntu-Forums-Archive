### Post by JoelOl75 on 2010-04-02
Possibly try pci=routeirq as a kernel boot option.

In GRUB2 if you are not dual booting you need to hold down the SHIFT key to see the menu, then edit the boot parameter removing quiet splash from the end and type pci=routeirq then CTRL-X to boot.

If this helps it's a kernel bug... Sounds like a IRQ conflict to me?  If your MB has BIOS options and you have built in graphics on the motherboard, try shutting them off?  Try setting options having to do with the video cards try changing them 1 at a time and see if it helps?

Try other kernel options as well like nolapic noapic acpi=off

Sometimes its as simple as just removing the splash screen which caused some problems for me with nvidia cards in the past.  Try just booting with quiet and splash removed from the boot line.

If you find a combo that works it can be made permanent by modifying the /etc/default/grub section and running sudo update-grub2 from the command line.

DO NOT EVER modify /boot/grub/grub.cfg as any update will revert its settings.

Hope this gives you some direction.
