### Post by Scunizi on 2007-10-04
This isn't a direct solution to your problem but something to think about.  I have a headless LAMP server (Dapper) I installed and then installed xubuntu-desktop after the install.  The server portion comes online on reboot regardless of my not logging in.  The services are running.  If I want to log in to a terminal I ssh into it and enter my user name and pass.  Then I can move around in the file system from the console.  If I want a gui (remote) after logging in via ssh I issue "sudo /etc/init.d/xfce4 start and the gui starts (with a couple of quirks).

If you're using VNC, the server portion of the service should start up on reboot regardless of a monitor or not.. 

I hope that help.
