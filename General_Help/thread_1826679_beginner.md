### Post by Zill on 2011-08-17
> **kumaranphoenix said:**
> I'm a beginner with linux and want to become a pro in two months...
I have been using Linux systems for around seven years now and still regard myself as a "beginner". ;-)  There is *always* more to learn!

My suggestion is to keep one PC as a stable working system for your day-to-day "real" work and then use another "test" system to experiment with (an older, lower-spec PC should be perfectly adequate for this).

Stick as many Linux distros as you like on to your "test" system and then just try things out.  You will steadily learn as you go and, if you manage to break it, you will learn even more when you fix it. :-)

In the meantime, read all the stickies on these forums and use the web to find whatever Linux documentation is applicable to your requirement.

Good luck.

### Post by IWantFroyo on 2011-08-17
For programming, learning Bash and Python are probably your biggest priorities. You might also want to learn some XML (manually editing XFCE and Openbox menus).

Bash tutorial- [http://tldp.org/LDP/abs/html/](http://tldp.org/LDP/abs/html/)

Python- Covered above.

XML- Just look at the HTML tutorials at w3schools.com. You really just need to be familiar with what elements are to edit XML files (you can just copy and paste, the files are explanatory enough that you don't need to know anything advanced to create menu entries).

As for server stuff, I'm not a server expert, but you're going to want to learn how to get around the command line.
First, check this out: [http://mally.stanford.edu/~sr/computing/basic-unix.html](http://mally.stanford.edu/~sr/computing/basic-unix.html)

Then, just try to do everyday tasks in the command line. You'll get it soon enough. To open the command line, hit the 'Terminal' app.
