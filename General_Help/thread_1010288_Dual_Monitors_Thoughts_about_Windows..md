### Post by falkaholic on 2008-12-13
I've used Mac, Windows, and for the past year, Linux as my home desktop.

Being playing with *nix a servers for years, switch it to the desktop. Continued to use Windows at work.


One thing that really grinds my gears to getting around the interface with dual monitors. I end up having too many windows and its hard to get things done at the speed of thought. No easy way of flinging windows around. Using multiple workspaces doesn't have much point; I see two all the time.

I use Gnome, KDE3 was ok, KDE4 is defiantly not for me. Xfce is too basic. The Linux nVidia drivers only seem to offer DualView or dual desktops.

I found working this with Windows  was preferable because:

1. The Nvidia drivers with allowed great customization. I could have a keyboard the would move a window to my other monitor. Great. Resizing grid, all sorts of neat stuff.

2. There are tools like WinSplit Revolution ([http://www.winsplit-revolution.com/](http://www.winsplit-revolution.com/)), that use keystrokes to tile my windows anywhere on my screen. Really nifty app.

3. With Gnome, the Keyboard Shortcuts panel has some issues. Worst of all the Super (aka Win) key isn't usable. That key is crazy helpful for common system-wide commands. 

4. Configuring Compiz is getting better, but is is hard to work with more then a couple plug ins without keystrokes or effects overlapping. The isn't an equivalent in Windows, but I'm in a complaining mood.

5. Other found other mangers like Spaces ([http://www.spatialresearch.com/spaces/](http://www.spatialresearch.com/spaces/)) for Windows that offer a more tactile window management. The "taskbar" type doesn't work for me. Can't find anything like that for Linux. 

6. No, having my desktops on a rotating 3D cube doesn't help.



I've looked into tiling window managers, I like that idea very much. Haven't found one that worked, was still being developed, wasn't configurable via scripting, or seems to explicitly support dual monitors.
