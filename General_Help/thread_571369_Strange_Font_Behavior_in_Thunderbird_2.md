### Post by hasinasi on 2007-10-09
Weird thing in TB 2.0.0.6 / Kubuntu Feisty using the Font "linux libertine": 

To reproduce just apt-get the package linux-libertine (universe/X11), select it as font in TB. Then compose a new message, type the letters "TM" or "No". What I see is that the "TM" in converted into superscript "TM", as if I wanted to say "Trademark" (which I don't). Similarly, in "No", the "o" is positioned above the regular text line and gets underlined, as if I wanted to abbreviate "number" (unfortunately, I simply wanted to write the word "Now", which now looks quite odd). It also happens in the display of received mail. 

Obviously, there is some sense behind this "intelligent" feature; the mechanism may also be used for fancy font features, such as ligatures. But like this, it is out of control. There is now way of switching of or un-doing it. The only way I found to write "ATM" without looking like "A"-(trademark) is if I select slightly different colors for the "T" and the "M". Even in MS products, where "intelligent" features like this annoy the hell out of me, they can at least be worked around (e.g. "backspace" after the auto-replacement occured).

I had already posted this on the TB forum. I am not sure though, whether this is a TB-only thing. [http://forums.mozillazine.org/viewtopic.php?p=3090000#3090000]("http://forums.mozillazine.org/viewtopic.php?p=3090000#3090000")
I have, however, found that the problem does not occur in OpenOffice 2.2.

Any idea what is behind this or where to start? Is it some font rendering engine? TB-related or not? All thoughts welcome. 
Thank.
--hasi.
