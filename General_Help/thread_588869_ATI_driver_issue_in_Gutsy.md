### Post by Jive Turkey on 2007-10-23
After expirimenting with Vista on one of my other partitions, I decided to fix grub by just installing Gutsy. Vista Millenium Edition hates my card even more, and after 2 days of updating and installing programs its unusable probably due to a resource hungry worm or something.  Anyway, the latest release of ATI's driver (8.43.2) solved this issue for me. 
Here is what I did (not exactly convenient to reproduce, sorry):
1. Installed Feisty to put Grub back after Vista broke it
2. Upgraded to Gutsy using Software Update
3. Installed (note I had the open "ati" driver at this point) the [proprietary driver from ATI]("http://ati.amd.com/support/drivers/linux/linux-radeon.html") using this guide: [http://wiki.cchtml.com/index.php/Ubuntu_Gutsy_Installation_Guide]("http://wiki.cchtml.com/index.php/Ubuntu_Gutsy_Installation_Guide")
4. (disable compiz now if you're following this at home unless you have total resolution lower that 2048 pixels in either direction by typing "metacity --replace" in a run dialog or terminal) You won't believe this, I enabled the second monitor and set big desktop mode using the "ATI Catalyst Control Center" from the accessories menu.  It works now!

Note:  You will have screen corruption problems running compiz if your total resolution exceeds 2048x2048 which is usually true if you are running big desktop.  I can't recall but that may have been the source of my pain all along. ### Post by rbmorse on 2007-11-25
For what it's worth, on my system the fglrx 7.11 package on 32-bit Ubuntu 7.10 (generic kernel) works flawlessly...Desktop effects (woobly windows et.al.) video playback in a window, the works.  The not-a-benchmark glxgears turn around 11,000 fps. My video card has an R580 GPU (X1950XTX). 

I can't get fglrx to work worth rotten fruit on 64-bit Ubuntu. 

Not sure this will work for everyone, but what I did:

```
sudo apt-get remove restricted-manager
```  Restricted-manager will mount a tmpfs called voltile under the kerner modules directory. This "volatile" contains an older fglrx which will get loaded instead of the new ATI drivers. If you need restricted-manager for some other component, the work-around is to find the line "DISABLED_MODULES= in the file 

/etc/default/linux-restricted-modules-common

and add the string fglrx between the quotes.  If other items are listed, use a space character between strings (i.e., "atheros fglrx")

Reboot now.     This is important. 

Download the ATI driver package from ATI:

[http://ati.amd.com/support/drivers/linux/linux-radeon.html](http://ati.amd.com/support/drivers/linux/linux-radeon.html)

Open a terminal window, cd to the location where the ati package is stored (I use Desktop)

```
chmod a+x ati*
sudo ./ati*
```

The installer will run, accept the defaults.
