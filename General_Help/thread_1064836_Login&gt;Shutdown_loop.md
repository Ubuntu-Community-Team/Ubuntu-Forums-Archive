### Post by ecowan on 2009-02-09
Yesterday I changed the Power Manager option 'When Laptop Lid Closed' for my administrator user to 'Shutdown'. Last night I closed the lid.
This morning when I boot and login, the KDE desktop shows for a few seconds, then KDE shutdown kicks in.
My other, non-admin users are fine.
Any ideas, anyone?
Desperately, Erny

p.s. I just found a crude workaround. If I first start a non-admin user and then do a switch to my admin user, the admin user will start, even though it flashes a notice that the lid was closed and it is going to shutdown, even though the lid was not closed since the last reboot.
This allowed me to set the 'lid closed' option to none. But if I set it to anything else, that's the action I get whenever I reboot, whether the lid was closed or not.
