### Post by sisco311 on 2009-11-01
check the permissions of the file:
```
ls -al /dev/video*
```

it should look like this:
```
crw-**rw**----+ 1 root **video** 81, 0 2009-10-31 23:52 /dev/video0
```

make sure that your user is in the video group:
```
sudo gpasswd -a **username** video
```
replace **username** with your login name;
log out and log back in.

### Post by trpnblies7 on 2009-11-10
I actually solved this problem, at least for me, over the weekend. My issue dealt with the file /etc/ld.so.preload. See here: [http://ubuntuforums.org/showthread.php?t=1316498](http://ubuntuforums.org/showthread.php?t=1316498).

In order to fix it, I edited /etc/rc.local to create the file on boot, and then I placed a script in /etc/rc0.d to delete the file when I shutdown my computer. Now everything works just as it did in Jaunty for me.
