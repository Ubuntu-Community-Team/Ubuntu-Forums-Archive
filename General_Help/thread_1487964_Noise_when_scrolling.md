### Post by lovinglinux on 2010-05-20
Check if is not the disk being accessed. 

I'm not sure if scrolling Firefox would affect this, but it could be due to Firefox database issues. Firefox includes the page position into the history so you can go back to the exact same spot on the page. So I guess when you scroll, this information is updated on-the-fly. If your bookmarks database is bloated with left-overs, it would require a lot of reading to update that information, thus generating the disk noise. 

I'm just speculating, but is worth trying to optimize your bookmarks database. See [this](http://firefox-tutorials.blogspot.com/2010/05/database-optimization.html).

BTW, for it Windows that generates a lot of disk noise.
