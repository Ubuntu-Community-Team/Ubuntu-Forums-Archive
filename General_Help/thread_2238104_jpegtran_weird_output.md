### Post by sudodus on 2014-08-06
jpegtran wants somewhere to read input and somewhere else to write output. When you write only jpegtran without an input file, it waits for input from standard input, in this case the keyboard. This is not what you want, so don't just type "jpegtran"!

```
jpegtran
abcde
Not a JPEG file: starts with 0x61 0x62
```

If I interrupt the input with ctrl d, it reads abcde, and finds 'a' alias 0x61 and 'b' alias 0x62, which is enough to know it is not reading from a jpeg file. So it quits with an error message.
You can also interrupt with ctrl c. Then it just quits.
