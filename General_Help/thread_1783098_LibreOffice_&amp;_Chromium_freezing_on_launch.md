### Post by dugh on 2011-06-15
I am using 11.04 classic desktop.  I have a shortcut to an .odt document on the top toolbar.  If I click the icon to launch LibreOffice, the entire desktop completely freezes for about 15 seconds before it launches.  This doesn't happen if I start LibreOffice from the command line, and it doesn't happen for other applications (but see related issues with Chromium below).

I've seen this other thread about freezes with LibreOffice, and the fixes didn't work (turning off compiz, which should be off already): [http://ubuntuforums.org/showthread.php?p=10941936](http://ubuntuforums.org/showthread.php?p=10941936)

In Chromium, when I launch (any method), it hangs (but not the whole desktop) for about 10 seconds on launch and on quit.  It does freeze up everything when I try to add a bookmark, or do anything that pops up a new window.  I've tried disabling flash and java, and disabling extensions, no change.  I noticed if I enable google sync, the freezing happens much more frequently.

So it seems to be some conflict with Gnome and/or perhaps networking.  I remember a similar freeze with OpenOffice long ago, and to fix it you had to turn off something that was making it do DNS lookups and internet connections on launch, but I don't remember the setting anymore.  Firefox doesn't freeze at all, but I stopped using it over a year ago because it is just generally slower and hangs for other reasons sometimes.
