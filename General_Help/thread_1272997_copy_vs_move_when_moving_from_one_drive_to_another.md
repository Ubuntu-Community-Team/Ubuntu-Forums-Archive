When I attempt to move a file from one drive to another in nautilus I instead end up copying the file.  Is there some way that I can change this behavior in the mount options.  

I.e. say I have a file.mp3 on drive A, and I want to move file.mp3 to drive B graphically using nautilus.  I drag and drop to drive B.  The file is then on drive B, but it's still on drive A.  I would like the outcome to be file is on drive B and no longer on drive A.

### Post by shazbut on 2009-09-22
By default dragging a file between different drives will copy. When the source & destination drive are the same it will default to move.
There's couple ways to tell it what to do:

Left-click-drag while holding :-
shift to move (notice the cursor changes to an arrow)
ctrl to copy (cursor changes to +)
alt to pop up a context menu on release (cursor is ?)

middle-click-drag will do the same as left-drag+alt.
