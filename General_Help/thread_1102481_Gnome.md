### Post by mb_webguy on 2009-03-22
This is one of those issues that can be confusing to former Windows users.

An operating system does low-level work like memory allocation and file access.  A user does not directly interact with the operating system, but uses some sort of user interface, which on modern computers is a graphical desktop.  A desktop environment is essentially just a program that runs on the operating system that makes it easy to interact with the computer.

In Windows, however, if you use the Windows operating system, you're also stuck using the Windows desktop environment.  Microsoft doesn't give you a choice in the matter, because the OS and the desktop environment are integrated.

In Linux, on the other hand, there is a clear delineation between the operating system (e.g. the Linux kernel) and the user interface, and there are many such user interfaces from which to choose.  The simplest user interface is the shell.  This is what you see when you open the terminal, or what you'd see on a text-only Linux server.  There are also graphical desktop environments like the one you're currently using.  If you're using Ubuntu, then you're using the Gnome desktop environment.  It's what gives you the panels at the top and bottom of the desktop, the desktop itself with icons and wallpaper, and windows in which to run programs.

Several other desktop environments are available for Linux, the most popular of which are KDE (used by Kubuntu) and Xfce (used by Xubuntu).  You can try these desktop environments for yourself by installing the kde and xfce4 packages in Synaptic Package Manager.  Then, at the login screen, you can go to Options->Select Session to choose which desktop environment you want to use when you login.
