### Post by Xian on 2005-11-24
[QUOTE=Kanephan]**Where is the linux source build directory that matches your running kernel?**

It's not a folder, but a symlink that most people make to point to the installed kernel source. After you install the kernel source package and untar it to /usr/src, you would just make a symlink named 'linux' for that directory. The names might be different, but something like:

$ cd /usr/src
$ sudo ln -s linux-source-2.6.12 linux

### Post by tokyovigilante on 2005-11-24
It's looking for binary kernel packages, but you haven't built them yet ;) Ok, let's build you a new kernel. I'll explain the steps as I go.

```
sudo -i
```
Enables super-user mode.
```
apt-get install kernel-package gcc gcc-3.4 libncurses5 libncurses5-dev
```
Makes sure you have all the requisite packages needed to build a kernel
```
cd /usr/src/linux
make menuconfig
```
Enters the kernel configuration program. Just exit without saving, and answer yes to write the configuration.
```
export CC=gcc-3.4
```
Switches to the 3.4 version of the C compiler
```
sudo make-kpkg clean
```
Preps the source tree for build
```
sudo make-kpkg --initrd --append-to-version=-custom1 kernel_image
```
Builds the kernel and generates a .deb with the kernel image. You shouldn't need to install this as the installer only requires that the modules be present in your source tree. This will take ~1/2 an hour depending on your processor speed, so have some dessert. ;)

Now give that driver another go.
