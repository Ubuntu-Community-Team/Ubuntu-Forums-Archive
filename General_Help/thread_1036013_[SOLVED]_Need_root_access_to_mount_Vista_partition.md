### Post by mgol on 2009-01-10
If You want to be able to mount ntfs partitions not being root, You should use 'user' option in fstab, like here:
```
# /dev/sdb5
UUID=4953-6095  /media/fat32  vfat  defaults,user,noauto,exec,uid=mgol,umask=022  0  0

```

I looked into my fstab, and here are lines responsible for my ntfs partitions (however, I use XP, not Vista):
```
# /dev/sda3
UUID=24ACAB6EACAB3966  /media/win   ntfs  defaults,umask=027,uid=mgol,gid=ntfs  0  0
# /dev/sda10
UUID=B2081C04081BC5EF  /media/ntfs  ntfs  defaults,umask=027,uid=mgol,gid=ntfs  0  0

```
As You can see, I mount it used ntfs, not ntfs-3g option. I don't know, but I looked into mount man page and there is nothing said about ntfs-3g, but there is a lot about ntfs.

It's also strange that You have some parts of fstab set using UUIDs and some not...
If you want to standarize it and use UUIDs everywhere, You can look for the proper names using the command:
```
ls -l /dev/disk/by-uuid
```

### Post by mgol on 2009-01-10
I don't know if its doable. The site mentioned says that one should have ntfs-3g binary is at least version 1.2506, whereas my Hardy has an older one:
```

$ ntfs-3g --help

ntfs-3g 1.2216 external FUSE 27 - Third Generation NTFS Driver
(...)

```

You can try a version from hardy-backports:
[http://packages.ubuntu.com/hardy-backports/ntfs-3g](http://packages.ubuntu.com/hardy-backports/ntfs-3g)
It is 1.2506, so there's a chance it will be ok.

To turn on hardy-backports repository, do the following:
1) go to System -> Administration -> Software Sources
2) go to the tab Updates and turn on a hardy-backports option
3) it should propose to reload apt info, say ok or just put in the terminal:
```
sudo apt-get update
```
3) update ntfs-3g package:
```
sudo apt-get install ntfs-3g
```
4) if everythings updated properly, You can disable hardy-backports repo now.

You can also install libntfs-3g28:
[http://packages.ubuntu.com/hardy-backports/libntfs-3g28](http://packages.ubuntu.com/hardy-backports/libntfs-3g28)
and ntfs-3g:
[http://packages.ubuntu.com/hardy-backports/ntfs-3g](http://packages.ubuntu.com/hardy-backports/ntfs-3g)
manually (in this order), uninstalling the previous packages BEFORE:
```
sudo apt-get remove ntfs-3g libntfs-3g23
```

Afterwards, You can follow instructions from ntfs-3g site, as below:
```

sudo chown root $(which ntfs-3g)
sudo chmod 4755 $(which ntfs-3g)

```

'users' option simply won't work on ntfs-mounted systems, so You should remove it. See:
```
man mount
```
section OPTIONS, option -o. There You can see which sections can be used with each system. Options dedicated to a particular partition type are described in sectuions named like "Mount options for ...". I proposed 'user' option before because I was unaware it's not working on ntfs filesystems.

You should try what has been linked already, that is this solution:
[http://ntfs-3g.org/support.html#useroption](http://ntfs-3g.org/support.html#useroption)

I think it's something from the past, when ntfs partitions had to be mount as read-only and ntfs-3g was not stable enough. This is not the case now. To say even more, I looked at:
```
$ ls -l /sbin | grep mount.ntfs
lrwxrwxrwx 1 root root       12 2008-12-28 06:26 mount.ntfs -> /bin/ntfs-3g
lrwxrwxrwx 1 root root       12 2008-12-28 06:26 mount.ntfs-3g -> /bin/ntfs-3g
```
As You can see, mount.ntfs and mount.ntfs-3g direct to the same executable, so there is no difference between them...

Some sites I visited (even this official one) also suggest that making this symbolic link (/sbin/mount.ntfs -> /bin/ntfs-3g) can solve some problems with some apps, sometimes... I don't suppose it to change anything (especially in this case); nevertheless, just for the future usage, You can make this link by:
```
sudo ln -s /bin/ntfs-3g /sbin/mount.ntfs
```
