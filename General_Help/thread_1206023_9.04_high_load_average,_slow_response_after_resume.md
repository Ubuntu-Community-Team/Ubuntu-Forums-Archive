### Post by GameboyRMH on 2009-11-14
Sounds like it could be this problem:

[https://bugs.launchpad.net/ubuntu/+source/compiz/+bug/225858](https://bugs.launchpad.net/ubuntu/+source/compiz/+bug/225858)

I'm having the same issue (Toshiba TE2100, GeForce 420 Go on Nvidia drivers, running Karmic). Disabling all desktop effects seems to be an effective workaround.

You could also try running compiz --replace from an alternate terminal - this is supposed to fix the problem. I haven't been able to test it though since the problem occurs randomly on my laptop. Having the command run automatically on resume could be a better workaround.
