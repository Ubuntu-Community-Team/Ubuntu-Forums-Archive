### Post by nabl on 2009-02-09
You should be able to do this with a simple .sh file. Just add one line to the beginning as shown below (using your example). 
```
#!/bin/bash

cd Desktop
cd Foldername
javac .*java
```Just save it as filename.sh, and you should be able to double click that, and it will run. In addition, you can use the command [FONT=Courier New]chmod +x filename.sh[/FONT] in order to run it from the CLI with [FONT=Courier New]./filename.sh[/FONT].

### Post by jenkinbr on 2009-02-09
> **OutOfReach said:**
> Create a .sh script, and type that into it. But remember to put this at the top:
```

#!/bin/bash

```
That, enter your code, and then save it to a file that has no spaces in the title.  You will probably need to change permissions to give it execution privilages.  If you save your file as 'myscript', you can change the permissions by typing ```
chmod +x ./myscript
```

to run it, you can type ```
./myscript
```

Note that the './' is important for security reasons.

If you like, you can create a ~/bin directory and store scripts there.  After creation, the next time you login, that directory will be added to your execution path, and you will not need to type the path to the script.

### Post by mb_webguy on 2009-02-09
Btw, in reference to the original issue...  Linux generally doesn't care about file extensions, and uses MIME types to identify files.  You can remove the .sh from the filename if you want it to look more like a command.  As long as you have the shell specification line at the beginning of the file (the line that begins with "#!") and the file is executable for your user account or group, then it will work just fine.  If you create a bin directory under your user's home directory and place the file there, you can use the command/script from anywhere in the terminal without specifying the path.
