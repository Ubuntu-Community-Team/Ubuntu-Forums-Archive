### Post by Impavidus on 2014-11-12
14.04.

Unless you plan on reconfiguring your entire system every six months when moving from one interim release to the next, you can best use 14.04 LTS. LTS (Long Term Support) releases get 5 years of support. I always recommend installing LTS, unless the next LTS is less than half a year away. 14.10 has only few improvements compared to 14.04, and less stability.

### Post by Impavidus on 2014-11-12
I think there can be three reasons to use the interim releases instead of the LTS releases. First, when it's less than six months until the next LTS release. When the new LTS has been released, most people want to move to it in a few months, so using the interim release doesn't cause an extra upgrade for you and allows you to use more recent software in the first few months after installing. But as the next LTS release is not the first release from now, this is not applicable. Second, if you want bleeding edge software. But be careful, the bleeding edge is sharp and you could hurt yourself. Third, if you have very recent hardware it doesn't always word with the LTS release, as drivers haven't been build into the LTS release's kernel. So first try 14.04 LTS from a live disk. If you encounter obvious problems caused by buggy drivers, try 14.10 from a live disk to check wether the bugs have gone.

Edit: In those specs I didn't immrdiately see hardware for which I expect problems. Graphics and wifi sometimes give problems, but with Intel graphics you're usually fine.
