### Post by efflandt on 2010-06-22
I have not used Unetbootin.  But if you use System > Administration > Startup Disk Creator to create bootable USB iso, it has a slider to choose the size of persistent data (casper-rw file).  However, since the file system is FAT or FAT32, the maximum size of any file (including persistent data) is 4 GB.  But you can also still store files on remaining USB space (FAT32 or what Linux calls vfat).

Or with 8 GB stick you could do a regular install with Linux filesystem.  However, you have to make sure that you pay attention after setting up your user and getting to the summary screen to use the **Advanced** button to put grub on the mbr of the USB stick (instead of default to mbr of your main drive).

### Post by C.S.Cameron on 2010-06-23
UNetbootin copies the Live CD to USB and makes it bootable.
This takes up about 750 MB
It does not provide any persistence, you can not save changes.
The remaining disc space should still be usable unless you have created extra partitions.
If you have, Windows will not be able to see them.

An UNetbootin install is mainly just an alternative to the Live CD. It is faster than the Live CD at installing Ubuntu and saves CD's.

Follow efflandt's advice if you want a portable O/S.

When you are running Ubuntu in "Live" mode (from a CD or a USB you have created with Unetbootin), you are running entirely in RAM. The 900mb is not actually physical disk space, but how much RAM the system has set aside as a virtual "disk."

If you want to save any documents that you create in Live mode, you'll need to mount a physical drive (from the Places) menu and copy the documents there. Otherwise they will disappear when you power down.

It sounds like you want a "persistent" USB install, which you can create with Ubuntu's built-in USB utility (not Unetbootin).
