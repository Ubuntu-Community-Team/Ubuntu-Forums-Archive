### Post by sefs on 2009-09-03
I am trying to add multiple network blocks to /etc/wpa_supplicant/wpa_supplicant.conf

However if i add a second block wpa_supplicant fails to start

my blocks look like this:

/etc/wpa_supplicant/wpa_supplicant.conf
```

ctrl_interface=/var/run/wpa_supplicant
network={
	ssid="athome"
	scan_ssid=0
	mode=0
	proto=RSN
	key_mgmt=WPA-PSK
	group=CCMP
	pairwise=CCMP
	psk=asdjfsfdyusfdsifydsifydsufydsiyfdyfsigfdgfdfd
}
network={
	ssid="atwork"
	scan_ssid=0
	mode=0
	proto=RSN
	key_mgmt=WPA-PSK
	group=CCMP
	pairwise=CCMP
	psk=iueyreyruwyerywiueyirueywuyreiwyiey
}

```

/etc/network/interfaces
```

auto ath0
iface ath0 inet dhcp
	wpa-driver wext
	wpa-conf /etc/wpa_supplicant/wpa_supplicant.conf

```

It works for one network block but wpa daemon fails to start if i add a second like above.


Update: Never mind.  It was the psk part.  wpa_supplicant seems to need a key of a specific lenghth.  the ones generated by the wpa_passphrase command.
