### Post by oldfred on 2022-09-30
The grub in the ESP is a configfile type entry to the grub in your install.
It normally does not need updating as it should not be changing, unless you redo install with new /boot partition or folder with different UUID.
Not sure with LVM and no separate /boot partition, if you also need extra grub .mod files to load LVM to see /boot? I do not use LVM, but most that I have seen have separate /boot partitions & grub in ESP just refers to that, just like my desktop refers/configfiles to /boot folder in my / (root).

What does your grub.cfg in ESP look like?

The update grub only updates grub menu in install.
I think a full reinstall of grub updates UEFI & grub in ESP.

### Post by yancek on 2022-09-30
The grub.cfg file in the EFI partition has 3 lines and reinstalling it will only likely change the UUID pointing to the / system file.  Running sudo update-grub will only update the grub.cfg file in /boot/grub.  What exactly are you trying to accomplish?  If you want to add entries to boot iso files, put the menuentries in /etc/grub.d/40_custom and run sudo update-grub. If you edit the grub.cfg file in /boot/grub by adding entries, they should be there on reboot unless you run sudo update-grub.  If you do that all your manual changes will be lost.

The standard grub.cfg file on the EFI partition is not updated and I expect if you reinstall grub it won't change anything as you will have the same / (system) partition with the same UUID.  What exactly is your end goal.
