### Post by Bucky Ball on 2017-03-30
Are you running a UEFI system? If not, no need to reinstall grub or fiddle with fstab (unless you are, for some off reason, automounting the Win partition at boot using fstab). Just boot to Ubuntu, make sure you have grub on sda with this:

```
sudo grub-install /dev/sda
```

Once that's done, open Gparted or install from the repositories and then open, find the Win partition, right click to make sure it is unmounted and kill it (right click> Delete).

Once that's all done, run this:

```
sudo update-grub
```

Reboot and Win should be gone from your boot list options. I did this myself about two weeks ago so 'from the horse's mouth'. 

Take note: This method is ONLY for a non-UEFI boot system. I have no idea how you would do this with an EFI boot, but my suspicion would be that you simply leave the EFI partition alone and that would leave Ubuntu bootable, but don't take my word for this. Do further research. I take no responsibility for anything you might try re. removing Win from a UEFI Win machine. :)

Good luck and hope that helps.

### Post by yancek on 2017-03-30
Deleting primary partitions will not change the numbering.  Deleting a lower numbered logical partition will change sda7 and sda8.  If you deleted sda6, it would change sda7 and sda8 dropping each down a number.  Deleting the boot partition (sda1) if you no longer want windows should work with the same process you used for sda2.

Keep in mind that when you install the 18.04 LTS release (is that what you mean by 18 LTE?) that will take over grub. You can change it back again, of course, by booting into the 16 install and running the install and update grub commands given previously.

Just a thought: You don't say which release of Ubuntu you are currently using.
