### Post by erjoalgo on 2010-02-23
I was working on a .tex file on Ubuntu, stored in a folder in my Windows 7 partition. I had to rename the folder, which was originally created in Windows, because the folder contained a space. However, after booting into Windows, the folder still had its old name, and none of the new files or folders added through Ubuntu were present. 
After booting into Ubuntu, those files were also gone.
Mysteriously, after booting into Windows a second time, the folder was correctly renamed, but the files were still not there.
Does anyone know about this issue or has experienced this before?

This is terribly frustrating as the files were an assignment due for class.

### Post by erjoalgo on 2010-02-23
Update:
The files were recovered after windows insisted on a checkdisk routine at startup. They were placed at found.000 folders at the windows partition.
