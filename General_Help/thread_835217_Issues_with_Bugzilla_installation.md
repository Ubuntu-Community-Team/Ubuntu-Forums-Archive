### Post by rogdawg on 2008-06-20
I am using Ubuntu 7.10 Deskstop and Apache2. 

I wanted to get the latest version of Bugzilla, so I downloaded directly from the Bugzilla site, instead of using the Synaptic Package Manager, which, I believe, installs an earlier version. 

I used the installation instructions found here: [http://www.ibm.com/developerworks/linux/library/l-bugzilla.html#listing2](http://www.ibm.com/developerworks/linux/library/l-bugzilla.html#listing2)

When installing Perl, I used the ./checksetup.pl command, and it produced a list of commands that I needed to run to install various packages. I ran all those commands, with no errors, except for this one:
 /usr/bin/perl -MCPAN -e 'install "Template"'

I get lots of errors that seem to be generated by header files in this directory: /usr/lib/perl/5.8/CORE/

I have verified that the header files exist in the path shown above, so I assume that the "make" functionality is expecting a certain format/version of those files that it is not finding. 

So, the install doesn't work for "Template". I continued on with the instructions, but when I try to navigate to Bugzilla on my machine([http://localhost/bugzilla/index.cgi](http://localhost/bugzilla/index.cgi)), I get an error page: 403 Forbidden - You don't have permission to access /bugzilla/index.cgi on this server.

I think this problem may be unrelated to the problems installing "Template".

But, I am stuck.

Any suggestions that might help me get past either of these issues would really be appreciated.

Thanks

### Post by rogdawg on 2008-06-20
This has been resolved. When trying to install Templates, I was always choosing to install 'XSpash' (I think that was the name), and other option packages. I tried the installation once without selecting those additional packages, and the installation worked.

After I worked through a few more issues (MySQL password, etc.) I was able to get Bugzilla up and running. WOOHOO!
