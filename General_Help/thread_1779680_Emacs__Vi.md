### Post by psusi on 2011-06-10
This is often the subject of religious wars.  Both have a steep learning curve, though I find vi to be far less user friendly at first, I am sure it is a fine editor.  On the other hand, vi users like to say with derision that emacs is a good operating system.  Myself, I like the fact that it does everything under the sun.  I like to be able to compile and debug programs, handle version control, manage directory trees, transparently access remote and compressed files, run a shell or two, get on IRC, and be able to attach to the session remotely via ssh.  If you just want a light weight text editor, then vi might be more your style.
