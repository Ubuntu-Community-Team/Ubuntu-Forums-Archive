### Post by dbott67 on 2006-11-21
The list comes from 'www.getautomatix.com'  --- it started off as a script written by 'Arnieboy' to help users easily install very commonly used apps (that were not necessarily available in the repositories).

As for a list of software available, there are over 18,000 packages in the Ubuntu repos.  The package manager (**apt**) is accessible by command line using **apt-get**, via text gui using **aptitude**, or via full-blown GUI using **Synaptic**.  Synaptic is the easiest to use for searching and installing for the newcomer.

By default, only **Main** & **Universe** software is available.  In order to access the full gamut, you need to enable '**restricted**' and '**multiverse**'.

Ubuntu's philosophy is that software should be free of restrictions and free (as in 'no cost').  Things like codecs for playing MP3s & DVDs are restricted formats, and are not included because of the 'restricted' format (the philosophy encourages usage of open formats like .ogg).

As such, the repositories are divided into 2 categories:

**Main** and **Restricted**.  

Software on the 'main' side is part of the FLOSS philosophy (free/libre open source software).

Software on the restricted side is proprietary/closed (such as the MP3 format, ATI binary video drivers, Flash, Java, etc.).

The distinction is setup so that people wishing to only use FLOSS software can do so, without ever worrying about licensing or proprietary software.

There are also 2 other sections to the repositories: **Universe** and **Multiverse**.

Universe contains all of the 'officially' community-supported software packages, whereas Multiverse contains 'unofficial' packages.

Lastly, there are **Backports**.  Backports are newer versions of software packages that have been re-written to work with older versions of Ubuntu.  For example, the latest package of XYZ may have been written to work with 6.10 Edgy, but you're only running 6.06 Dapper.  Someone would backport the XYZ package to work with Dapper.

There are also hundreds of unofficial and unsupported repositories that you can add to your list, but you need to take extreme care when installing from unknown/untrusted sources.

Hope this helps.

-Dave

### Post by Cynical on 2006-11-22
I dont believe there are any in the wild. 


Anyway,


**CD/DVD Burning** - Brasero (used to be known as bonfire) and Gnomebaker are good choices, you shouldn't have any problems with them. You could also consider Graveman (error checking is possible with these programs)

**Download Manager** - Gwget, excellent looking app that can do exactly what you need. (pause/resume/split downloads) For firefox integration you can use the [Flashgot](https://addons.mozilla.org/firefox/220/) extension. It will detect gwget if it is installed.

**Peer Guardian** - Could you describe what p2p software you use? If its a bit-torrent application, Azureus has its own peer guardian plugin you can install. If not there was a post above with a link on how to install it.

**iTunes** - I prefer using one app per job. For cd ripping you can use Sound Juicer, (default app in Ubuntu) abcde, or grip. (there are more of course, I just don't know them all) For media playback you can use winamp style clients like [bmp](http://upload.wikimedia.org/wikipedia/de/5/59/BeepMediaPlayer.png) and [xmms](http://gnrfan.org/pix/blog/xmms-human-ubuntu.png), or you could try [rhythmbox](http://oxlife.free.fr/rhythmbox.png), [exaile](http://www.exaile.org/screenshots/exaile.jpg), [quod libet](http://img305.imageshack.us/img305/6166/quodlibet14uu.jpg), [banshee](http://www.foresightlinux.com/sitemedia/screenshots/banshee.png), [muine](http://muine-player.org/images/3/31/BrandonMainPageScreenshot.png), and [listen player](http://costa.kofeina.net/uploads/soft/listen-screen01.png). I'm going to warn you now though, dont base your opinion soley on the images, you can customize these players to look drastically different from their defaults.

**Backup Program** - You can use a simple frontend like sbackup to do regular backups of your Ubuntu partition. To be honest though it seems rather pointless. It's not easy to disable a linux machine unless you do something stupid, and even then it doesn't matter if you keep your home partition separate from your root partition, because you can keep all of your settings intact while reinstalling.

**Antivirus** - Pointless

**Firewall** - Also pointless.

**Temperature Monitor** - I've always used the one that accompanied my BIOS so I cant give any recommendations here.

**Winrar** - Fileroller is an application that already comes with Ubuntu. To enable rar support you have to use apt-get to install the unrar-nonfree package.

> 2. Don't you need a firewall at al in linux? I mean in windows you want a software firewall even though you got a hardware one. Both for incoming and outgoing traffic. Is is really impossible to attack linux (I know anything is possbile but you get the point)?

By default, Ubuntu does not open service ports unless required.  If you inadvertently install a service that "listens" then there is the potential for attack.

For example, if you install Samba for file-sharing with Windows, then it needs to open ports 137-139 & 445.  If you don't use it, those ports are closed.

I suppose the real reason for using a firewall would be for 'stealth'.  A "closed" port tells an attacker that there is a computer on that IP and it would allow them to scan for other open ports or vulnerabilities.  A firewall would drop 'unsolicited' packets and not return any message --- to the attacker, it would appear as if a computer was not at that address (or off).

For HDD SMART check out **smartmontools** - [http://smartmontools.sourceforge.net/](http://smartmontools.sourceforge.net/) (Interestingly, the author uses it on a *600*-disk cluster!)

It consists of two parts - smartctl which can be used on an adhoc basis for checking discs, retrieving SMART info (like the temps you mentioned) and setting self-tests etc. The other, smartd, runs as a daemon and keeps an eye on your disks alerting you by e-mail (amongst more local notifications) if any SMART data indicates impending failure.

Highly configurable and extremely powerful - like many Linux apps it's not configured with a GUI as this can often limit the configurability. It's not hard to use though - just read the excellent man pages and configure exactly as you require. I can always share my straightforward config (3 disks monitored 24/7, short tests performed every morning with a long test at the weekend - all stats monitored with an e-mail alert to predict future failure).

I'm afraid I can't help you on the CPU temps etc as I've got a Dell motherboard which are infamous for not supporting anything like lmsensors etc.

### Post by JohnPhys on 2006-11-25
If your usb drive won't be handling large files ( > 4Gb ), then you can put a fat32 ( or vfat ) file system on it and use it in both windows and linux.  I do this regularly on my SanDisk Micro.  In fact, it came with that file system anyway, so I didn't need to do anything differently.

If you're using the drive to transport large files, you'll need to use another fs like ext3 since vfat/fat32 can't handle files in excess of 4Gb.  I've heard that you can mount and write to ext3 within WinXP, but not natively, so I'm not sure what program needs to be installed.  Alternatively, you could possibly put ntfs on the usb drive and set up read/write to ntfs in ubuntu.  I'm not sure how safe writing to ntfs is, so you'll have to read up on it.  I read stuff from my WinXP partition all the time though, and haven't ever had an issue.



As far as the partitioning, and you putting ubuntu on a separate drive, or just another partition?  I told the breezy installer to use it's automatic partitioning on my empty drive, and everything went smoothly.

Hope this helps.
