### Post by Quiane on 2012-08-25
This doesn't happen every time i sleep, but it is consistent.  I have my power options set to put the computer to sleep after 10 min of inactivity.  This works well the first 2 times after a power cycle..but the third the computer will either 1) not go into sleep mode and instead become unresponsive (purple screen, hdd lights going, tower powered on, but no gui..no amount of ctl +alt+ fkey does anything, need to hard reboot) or 2) go into sleep mode, but when the power button is pressed it is unresponsive (as described above).  

I was going to include a log file, but i can't remember which one would be helpful...

Any suggestions on finding out where to start with this issue would be helpful.

I just did a clean install last week...

---

### Post by 2F4U on 2012-08-25
The log file dedicated to suspend would be /var/log/pm-suspend.log. It may also be worth looking into /var/log/syslog, as this is the general log file (also accessible through the menu and a spedial editor).
