### Post by ninjaaron on 2010-12-09
Kubuntu and Arch are almost the opposites of each other.

Anyway, there are good utilities for programming in most distros of Linux, and you can certainly revert to console mode in any of them. Arch is for if you are something of a Linux vet, and you want a completely customised. Ubuntu is for if you don't want to mess with all that.

For the record, Linus Torvalds, the creator of Linux and an expert programmer by all accounts, hates Arch because it's so much work to set up. I believe he likes Kubuntu and Fedora, if I'm not mistaken.

But excellent tools for programming are available in all distros. GNU/Linux is, after really just an experiment that the hackers are trying out to see what they can do. It will always cater first and foremost to that crowd. We're just along for the ride.

Edit: by the way, to use kubuntu, you don't have to delete anything. It runs together with Ubuntu just fine. Just install the kubuntu-desktop package (you'll have to use synaptic or apt-get, since it doesn't appear in the software centre). When you boot, there will be an option in a menu near the bottom of the screen to boot your session either to the KDE or the Gnome desktop. Good Luck!
