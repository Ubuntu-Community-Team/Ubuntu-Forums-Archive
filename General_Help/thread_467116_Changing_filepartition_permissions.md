You can use the terminal to do this. 

To view the permissions of a file, type the following [/code]ls -l filename[/code]
You will see something like this in front of the file:
```
rwxr-xr-x
```
That means the owner has read, write, and execute permissions (the first group of three letters, "rwx"); the group has read and execute permissions (the second group of three letter "r-x"); all other users have read and execute permissions (the third group of three letters "r-x").

To change the permissions of a file so that the user has read and write access, the group has read and write access, and all other users have read only access, type the following in the terminal:
```
sudo chmod 664 filename
```
Check now, and the permissions will be```
rw-rw-r--[code]

To change the owner and group of a file, type the following:
[code]
sudo chown username:groupname filename

```

Read more by typing the following:
```

man chmod

```
and
```

man chown

```

### Post by Thomar on 2007-06-12
Heh.  Fixed it myself.

Okay, /etc/fstab looked like this:

```
# /etc/fstab: static file system information.
#
#  -- This file has been automaticly generated by ntfs-config -- 
#
# <file system> <mount point>   <type>  <options>       <dump>  <pass>

proc /proc proc defaults 0 0
# Entry for /dev/sda2 :
UUID=8488df67-254d-4d28-9f8a-379997efcdff / ext3 defaults,errors=remount-ro 0 1
# Entry for /dev/sda1 :
UUID=84606AA6606A9EA4 /media/sda1 ntfs-3g defaults,locale=en_US.UTF-8 0 1
# Entry for /dev/sda5 :
UUID=15b381df-4c4b-edb4-4831-63fe53b4245c none swap sw 0 0
/dev/hdc /media/cdrom0 udf,iso9660 user,noauto 0 0
```

Changing it to this (bolded part added) makes it invisible to anyone but the root user.  I had to reboot to get this to work, so apparently it can only be set when the drive is mounted.

```
# /etc/fstab: static file system information.
#
#  -- This file has been automaticly generated by ntfs-config -- 
#
# <file system> <mount point>   <type>  <options>       <dump>  <pass>

proc /proc proc defaults 0 0
# Entry for /dev/sda2 :
UUID=8488df67-254d-4d28-9f8a-379997efcdff / ext3 defaults,errors=remount-ro 0 1
# Entry for /dev/sda1 :
UUID=84606AA6606A9EA4 /media/sda1 ntfs-3g defaults,**umask=007,**locale=en_US.UTF-8 0 1
# Entry for /dev/sda5 :
UUID=15b381df-4c4b-edb4-4831-63fe53b4245c none swap sw 0 0
/dev/hdc /media/cdrom0 udf,iso9660 user,noauto 0 0
```

### Post by vanadium on 2007-08-20
(sorry I saw this a little late) ntfs-3g partitions are mounted with root ownership and all permissions open. In fact, Linux permissions are not supported on ntfs. I am afraid that there is no way to have an ntfs read-write for one user and read-only for others. If you want to use a drive with full Linux permissions, use a native Linux file system like ext3, and not a proprietary one like ntfs.
