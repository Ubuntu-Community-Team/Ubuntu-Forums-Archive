### Post by kidders on 2007-06-06
Hi there,

Afaik Linux doesn't support the concept of a "hidden" file. The idea of not displaying dotfiles is really only a notational convention ... which is not the same thing. I'd say many Linux users would probably find the idea that you could arbitrarily cloak any file at will a little absurd. Having said that, I can sympathise with how irritating it must be to have to look at all those irrelevant folder config files & silly thumbnail caches. At least Apple starts all *theirs* with a dot!

Unfortunately, I'm not sure that Linux has anything that a "hidden" flag can be mapped to. The same would apply if you tried to find some way of reflecting the MSDOS "archive" or "system" file flags in Linux. :-(
