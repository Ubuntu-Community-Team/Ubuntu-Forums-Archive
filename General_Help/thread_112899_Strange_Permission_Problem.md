### Post by sarah_b on 2006-01-05
I have some partimage CD's that I used for restoring a partition and it was successful. But I had some unrelated issues, so I reformatted the partition. But when I used the same CD's the second time, the permission changed from read & write to read only, which I thought was no big deal, I'll just change it back to read and write. When I put the check mark on write, it told me that it was a read only permission and could not change it to read and write.

How can this be ? I had ownership and it was read & write the day before, how could it have ever got changed to just read all by itself ? And seeing I am the owner, why could I not change it back to read & write ?

 I do not understand this, it just does not make sense to me. Can someone please help me with this ? All  opinions, suggestions, and comments on this bizarre problem are welcome.

sarah

### Post by sarah_b on 2006-01-05
I found some information on this doing a search in a defferent forum. It appeared that this problem was created when using a CD-RW for an image. So I burned a new image using a CD-RW also, this time I just copied it to my desk top, which in effect is simular to restoring the file to a partition, and yes in deed, it remove the execute option. In my post I said it removed write, which was not the case, it removed execute. 

My advice would be....never use a CD-RW if you ever want to use it twice to restore an image to a partition
