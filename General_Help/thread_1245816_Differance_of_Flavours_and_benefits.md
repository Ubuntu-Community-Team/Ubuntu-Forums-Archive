### Post by fluffman86 on 2009-08-21
I have to disagree with j.bell730 on this one.  Instead, I would recommend trying out the different flavors using their respective live cds.  If you install the packages per j.bell's instructions, you end up with lots of extra programs that clutter the menus, and that bothers me.

Here's a basic rundown:
**_Ubuntu_**
This is what I use.  I switched to Gnome because most of the programs I use on a daily basis are based on Gnome or GTK, like GIMP, Firefox, and Pidgin.  Gnome offers lots of sane defaults and comes mostly preconfigured.  Gnome programs don't include random K's in the name somewhere just to indicate what it's built with.  Overall, Gnome is easier to use, and I switched so I could be familiar with the same sets of programs and menus as the people I was installing Ubuntu for.

The downside is that KDE and Qt apps don't really look right on Gnome.  I honestly had trouble switching at first simply because I missed AmaroK and some of the little widgets, but the new Banshee is really awesome.  

**_Kubuntu_**
This is what I started with.  I'm pretty techie and like lots of customization, and KDE offers that--you can change pretty much anything.  Kubuntu runs GTK apps great, while Ubuntu looks funny running Qt apps.  KDE is also very pretty--there's lots of widgets and cool things to do with your taskbar.  I liked (after switching from windows) the large K-Menu for a more Windows-like feel, as well as the centralized control panel and single, double-height taskbar.  Now I think I prefer the double taskbars at the top and bottom like Gnome.

On the other hand, Konqueror and Kopete didn't really cut it as a web browser and Instant Messaging program, respectively.  Kate, Kmail, and Konsole were pretty much identical in my usage to Gedit, Evolution, and Gnome-Terminal (with perhaps a slight edge given to Evolution over Kmail).  And what's with K's being in every app name?    Also, the extensive options provided by KDE can be confusing for new users.  

For example, K3B is the default CD burner.  It's got more options than Nero (like md5sum checking), but they aren't as well layed out.  This is GREAT if you need specialty stuff, but 99% of the time, Nautilus or Brasero are going do what you want without any fuss.  Same goes for Ktorrent--it's got options and features out the wazoo, kinda like the old Azureus.  I guess it was kinda neat to sync up my .torrent files with Azureus in Windows while I was dual booting, but was that necessary?  Transmission lets me download .torrent files, and it does a great job.

**_Xubuntu_**
This runs the XFCE desktop environment.  It's pretty similar to Gnome as far as layout and appearance, but supposedly runs well on older hardware.  While this was true 2+ years ago, personally, I don't find it much faster than Ubuntu now.  One great thing about XFCE is that it handles Qt and KDE apps really well.  So while Mousepad, the default text editor sucks, you can then install Kate *OR* Gedit and either will look and run just fine.  You can't run Kate in Ubuntu without it looking stupid and breaking all kinds of conventional ideas.

The downside is that it's not very pretty, and you have to replace a good bit of stuff--the default terminal doesn't offer tab-completion, and Mousepad does not offer syntax highlighting or line numbering.

**_Ubuntu Studio_**
This is Gnome with a new theme and a new Linux Kernel that lets sound run better when you're doing audio processing.  It also includes some sound and video editing programs by default.  All of this is available to be installed into regular Ubuntu.

**_Edubuntu_**
Vanilla Ubuntu + educational software on a separate disk.  

-----------

Hope that helps.  Basically, if you only need to run GTK apps, and if you aren't very tech savvy, stick with Vanilla Ubuntu.  If you are using Kubuntu, you might want to switch to Ubuntu just so you can provide tech support for lots of computer illiterate people.  Both Kubuntu and Xubuntu run Qt and GTK programs just fine, so really either of them would work better in theory than vanilla Ubuntu.
