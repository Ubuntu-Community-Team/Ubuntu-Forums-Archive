### Post by mozetti on 2008-02-25
> **ele_mugv said:**
> 2. I would like to know if there is a way to run both synaptic package manager and the terminal simultaneously such that each one downloads and installs individual files without giving the error "...LOCK COULD NOT BE OPENED..IS ANOTHER APPLICATION USING IT...?"

I assume you mean running Synaptic and either apt-get or aptitude in the terminal. The answer is, no, both can't be run at the same time. Synaptic is basically a GUI front-end for apt-get/aptitude, with a bunch of extra features. Due to the nature of these programs, installing/updating software and keeping track of what's installed, they can't be run simultaneously.

### Post by mrsteveman1 on 2008-02-25
Ubuntu really does need a way to restore broken system files easily, not necessarily a system restore but just a way to authenticate and check previously installed files against the package they came from and reinstall the packages if needed.

Windows system restore was intended to make up for the fact that 3rd party software, drivers, and malware might change something that Windows otherwise didn't have a clue about, but Linux shouldn't have this problem since almost everything is in the repos, its all signed, and almost every file on the system can be traced back to a package, authenticated and reinstalled if necessary.
