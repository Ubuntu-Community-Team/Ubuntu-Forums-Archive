### Post by frodon on 2005-12-19
Ok, if you are using the default totem (which use the gstreammer backend) i advice you to install the "totem-xine" package, so your totem player will use the xine backend which works without any problems for most of us.

### Post by az on 2005-12-19
[QUOTE=frodon]Ok, if you are using the default totem (which use the gstreammer backend) i advice you to install the "totem-xine" package, so your totem player will use the xine backend which works without any problems for most of us.[/QUOTE]

For xine to work, you need to install the w32codecs package which is not in the archive.

You can use the gstreamer codecs by installing them:


[http://packages.ubuntu.com/cgi-bin/search_packages.pl?keywords=gstreamer&searchon=names&subword=1&version=breezy&release=all&page=1&number=50](http://packages.ubuntu.com/cgi-bin/search_packages.pl?keywords=gstreamer&searchon=names&subword=1&version=breezy&release=all&page=1&number=50)

[http://packages.ubuntu.com/breezy/libs/gstreamer0.8-plugins](http://packages.ubuntu.com/breezy/libs/gstreamer0.8-plugins)

See this page for the easy instructions:
[https://wiki.ubuntu.com/RestrictedFormats](https://wiki.ubuntu.com/RestrictedFormats)

"If you live in a country where it is legal to play MP3s and other non-free media formats without a license, enable the universe and multiverse repositories, and install the support packages by typing in a terminal: 

sudo apt-get install gstreamer0.8-plugins gstreamer0.8-plugins-multiverse gstreamer0.8-ffmpeg 
"
