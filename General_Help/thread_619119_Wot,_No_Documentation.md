### Post by Onesimus on 2007-11-21
After many weeks of careful consideration I feel that I have to write this post.  At the outset I have to say that I am very impressed with Ubuntu as an operating system.  However, my complaint is the total lack of documentation that is provided with the operating system and the applications that can be installed from the repositories.   

The user community is great and thriving with many posts being answered within a few days.  But it does beg the question whether the forums would be slightly quieter if the user was not forced to come to look for a solution at the forums but that they could find a solution in documentation that is provided with the application.

I wrote a post ([URL="http://ubuntuforums.org/showthread.php?t=589933"]
http://ubuntuforums.org/showthread.php?t=589933[/URL]) approximately three weeks ago requesting for documentation on setting up compiz fusion.  Whilst, 85 people read the message no one replied to my request.

Today, I had a notification window indicating that my laptop had not been able to hibernate correctly and that I should read the help page.  However, no link or indication of which help page that I should read was given.

It would appear that I am not alone in believing that documentation would be a good thing.  Carla Schroder writes much the same thing with regard to the Ubuntu Server edition: "*Whatever anyone may think of Ubuntu, you can't deny they're busy little critters, stuffing all manner of new things into every release. Which is a splendid thing, but what would make it even better is if they documented all of these wonderful new things, and also the old things." "It's the worst of the major Linux distributions for documentation*."  (For a full article see the weekly Newsletter letter #65.

Please, developers could you spend some time writing documentation to the great applications that you are developing.

### Post by hikaricore on 2007-11-21
Ubuntu devs have nothing to do with the documentation for nearly any of the software you're refering to.

All software is developed and has it's documention written by it's authors.

You can always reach the general help section from the help icon on the top bar of Gnome.
Also too read the doumentation for any installed software, from a terminal type:

```
man application
```

Where application is the software you want more info on.

Example:  *man sudo*

Will tell you everything you need to know about using sudo.
