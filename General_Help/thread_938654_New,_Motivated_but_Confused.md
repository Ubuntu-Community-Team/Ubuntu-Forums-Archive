### Post by Trebaruna on 2008-10-05
ratmandall is trying to tell you to use Gentoo or FreeBSD, or perhaps merely suggest them as alternatives to Ubuntu. It really quite depends on what your aims are: for instance, I use Ubuntu because I feel it gives me a good balance between usability and support on one hand, and tweaking-abilities on the other. [EDIT: tweaking meaning I can poke around and see how things work. Not simply trying to get it to run faster. Often it will do quite the opposite :P]

If you want to write your own programs, you'd do well to learn a programming language. The linux kernel is written in C, but there are numerous other languages out there, each with advantages and disadvantages. My C++ teacher would, however, say that "C++ is the most beautiful <expletive> language in the world" ;)
Try and find some good documentation online. I recommend the C++ Annotations by my teacher ( [http://www.icce.rug.nl/documents/cpp.shtml](http://www.icce.rug.nl/documents/cpp.shtml) ) but it does assume knowledge of C, so you'll need another work alongside it to get you up to speed on that. The book is licensed under the GNU GPL.

### Post by Calmatory on 2008-10-05
Slackware is considered good distro for learning how things work. It isn't as harsh as gentoo, thus it is easier for newer users to learn.

I've been using Ubuntu for several years, though, quite on/off, but few weeks ago when I set up my Arch server I learnt more about linux in two days with Arch than what I've learnt so far with Ubuntu. 2 days outplays 2 years when it comes to Arch against Ubuntu in terms of learning the system. ;)

If you want to have minimalistic, but usable approach, I'd suggest Arch Linux. In Arch you decide what you install, the base system only comes with tools to install packages, which then build the whole system. No need to sit nights compiling packages and thinking about dependencies, besides, pacman is the best package management system I've used. Very neat distro. ;)
