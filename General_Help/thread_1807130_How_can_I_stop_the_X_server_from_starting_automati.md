### Post by PyrexKidd on 2011-07-18
Use update-rc.d or if you're a redhat user, you can install chkconfig and use that.  The Debian/Ubuntu way is update-rc.d though.  Youll need to stop the respective service, gdm, kdm or xdm (in this case xdm if you installed x, gdm for gnome, kdm for kde.)

```

sudo update-rc.d <service name> disable
sudo update-rc.d xdm disable

```a word or warning, I **do not** recommend removing the scripts using update-rc.d <service> remove, this is typically reserved for a package's post removal script.

from man update-rc.d:
```

update-rc.d is usually called from a package's post-removal script when  that  script  is
given  the  purge  argument.  Any files in the /etc/rcrunlevel.d directories that are not
symbolic links to the script /etc/init.d/name will be left untouched.

```

### Post by SeijiSensei on 2011-07-18
> **sirspazzolot said:**
> I have Ubuntu and Ubuntu Server installed. I want the Server installation to boot with the text parameter and the normal one to boot normally.

Just edit /boot/grub/grub.cfg manually and add the "text" parameter to the appropriate kernel there.  You'll need to make the file writable before editing and reset it to read-only after you're done.  Something like:

```
linux   /vmlinuz-2.6.35-28-generic root=UUID=cf42c8af-a1ed-4704-8d71-4b33dfd3eaea ro quiet splash **text**
```
