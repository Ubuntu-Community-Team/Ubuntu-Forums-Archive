### Post by Zorael on 2008-04-26
Seems like it, yes.

Make a backup of your xorg.conf, then modify your video device section in it to add the Driver line illustrated below.
```
Section "Device"
Identifier "Configured Video Device"
**Driver "sis"**
EndSection
```

Restart X with Ctrl+Alt+Backspace and see if it works. If it doesn't, we want to have a look at your **/var/log/Xorg.0.log** file.

### Post by Zorael on 2008-04-26
Groan and moan.

Okay, use this tool to generate a modeline, and add it to your **Monitor** section.
[http://xtiming.sourceforge.net/cgi-bin/xtiming.pl](http://xtiming.sourceforge.net/cgi-bin/xtiming.pl)

Pick 1280x800 and 70hz if that's the resolution you want. The Xorg.0.log suggests the max clock speed is 340mhz.
```
(II) SIS(0): Clock range: 10.00 to 340.00 MHz
```

I think you will end up with this.
```
Modeline "1280x800@70" 101.92 1280 1312 1696 1728 800 816 825 841
```


There *is* a chance of your monitor taking damage from an incorrect setting if left for a long time. So if it's flashing something fierce, you want to undo the changes. If you can't use the graphical interface at all, hop to a terminal (Ctrl+Alt+F1) and reset xorg.conf again.
```
$ sudo dpkg-reconfigure xserver-xorg
```
Alternatively, open up the file with the **nano** text editor and remove the modeline we added.
```
$ sudo nano /etc/X11/xorg.conf
```
