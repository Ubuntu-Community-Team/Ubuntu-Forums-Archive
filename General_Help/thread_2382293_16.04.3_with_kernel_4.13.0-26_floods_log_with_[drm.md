### Post by Max303 on 2018-01-11
[FONT=Verdana]The solution is to remove "[/FONT]**drm.debug=0xe**[FONT=Verdana]" from your grub config and reboot.

[/FONT][FONT=Verdana]The problem originates from a kernel boot parameter in /boot/grub/grub.cfg:[/FONT]
```
[FONT=Verdana]linux   /vmlinuz-x.x.x-xx-generic root=UUID=<UUID> ro text plymouth:debug **drm.debug=0xe** init=/sbin/upstart[/FONT]
```
[FONT=Verdana]To stop the logging without reboot you can do:[/FONT]
[FONT=Verdana]```
# echo 0 >/sys/module/drm/parameters/debug
```
It is also possible that you have an entry similar to this one in /etc/default/grub:
[/FONT]```
[FONT=Verdana]GRUB_CMDLINE_LINUX_DEFAULT="plymouth:debug **drm.debug=0xe**"[/FONT]
```
In that case you have to remove **drm.debug=0xe** from that file and run **update-grub**
