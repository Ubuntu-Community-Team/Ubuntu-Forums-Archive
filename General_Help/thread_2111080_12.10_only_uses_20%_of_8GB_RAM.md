### Post by uaebuntu on 2013-02-01
Well you could use the extra RAM as something else, for instance create a RAMDISK for temporary files to speed things up, and it will use up a few MB of that unused 8GB.


Edit your fstab file, but first take a copy as a backup.

```
sudo gedit /etc/fstab
```


then mount /tmp (temporary files) as tmpfs (temporary file system):

```
tmpfs /tmp tmpfs defaults,noatime,mode=1777 0 0
```


Reboot for the changes to take effect. Running df, you should see a new line with /tmp mounted on tmpfs:

tmpfs 513472 30320 483152 6% /tmp


Then if you are a big Firefox user move its cache from your home partition to this /tmp directory and speed up Firefox.

Open about:config in Firefox. 
Right click in an open area and create a new string value called browser.cache.disk.parent_directory. Set the value to /tmp.

Has benefits as things run faster and your disk gets less exercise, but things can get lost if there is a power out, however that's why they were temp files .....

### Post by tgalati4 on 2013-02-01
OSX is based on a modified version of BSD (Unix).  So both linux and OSX use cache to speed up operations.  If you don't reboot, only put your machine to sleep, then over time all of your daily applications will be in RAM anyway.  There are methods to preload them, but it's not necessary.  If you are dual-booting, then you lose that cache advantage because you will be rebooting often.

As soon as you start creating virtual machines (VM's) or running wine--(for some Windows applications), RAM will get used quickly.

Also understand that some machines run faster at 2GB and 4GB than at 8GB, because of declocking of the RAM bus and I/O handling of the Southbridge chip.  You will have to perform benchmarks and research on your motherboard to confirm this.

### Post by tgalati4 on 2013-02-02
To answer the OSX question, yes, put it to sleep.  You only need to reboot if you run out of power or you have a kernel panic (and that will be obvious).  There might be a "preload" in OSX, but you would have to search for it.  Both linux and unix(bsd) use RAM efficiently, so any performance degradation is usually caused by using swap (having too little RAM), or a slow hard disk (perhaps too full), or a rogue program that is eating up CPU cycles--use "force close".
