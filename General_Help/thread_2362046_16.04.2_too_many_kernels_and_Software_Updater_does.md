### Post by Dennis N on 2017-05-23
I use Synaptic Package Manager to remove older kernel(s).

1) search for kernel by series (i.e., 4.4.0)
2) sort by installed version
3) select files with exact same version number
4) mark this group for complete removal and apply.

### Post by oldfred on 2017-05-23
[https://ubuntuforums.org/showthread.php?t=2361761](https://ubuntuforums.org/showthread.php?t=2361761)

If you run these does it show old kernels and suggest running autoremove?
       sudo apt update
sudo apt upgrade 
sudo apt autoremove

Running the upgrade I got this:
```
The following packages were automatically installed and are no longer required:
  linux-headers-4.4.0-75 linux-headers-4.4.0-75-generic
  linux-image-4.4.0-75-generic linux-image-extra-4.4.0-75-generic

```

And then running autoremove, it removed that kernel & updated grub.

But your older kernels 4.4 may not be in dpkg if an upgrade. Then you probably have to use rm to remove all the bits & pieces.
I used to use synaptic and search for linux-image to houseclean old kernels.
      
 Determine your current kernel:
uname -a
-s is similate so you can review first:
sudo apt-get -s autoremove
sudo apt-get install synaptic
In synaptic search for linux-image to choose to delete old ones
Using synaptic
[http://ubuntuforums.org/showthread.php?t=1283521](http://ubuntuforums.org/showthread.php?t=1283521)
cd /boot
ls -l /boot/vmlinuz*
sudo apt-get purge  linux-image-[version]-generic linux-image-[version]-generic
Multiples, just be sure not to delete your current kernel:
sudo apt-get purge linux-image-3.13.0-{XX,XX,XX,XX,XX,XX}-generic

### Post by deadflowr on 2017-05-23
To remove the 4.4.0-78 kernel you need to remove the linux-image-generic and linux-headers-generic packages, as those packages are dependent on the latest kernels for that series.
once you remove those packages the 4.4.0-78 kernel can be remove through autoremove, if need be.
(It's quite possible the kernel will be remove when you remove those packages, but if they don't autoremove should remove them)

The 4.8 kernels will keep updating as long as you still have the linux-image|headers-hwe-16.04 installed.

Hope it helps

### Post by deadflowr on 2017-05-23
Yeah, as long as you remove those pesky linux-image|headers-generic packages it'll prevent it from running a cat and mouse thing, since that package would've cause the updates to install the next 4.4 kernel when it comes.
so even if you remove the specific 4.4 kernels, a new one would be installed and you'd have had to uninstall that one, in a viscous cycle.
> Things got confused when I upgraded to 16.04.2 . So when 16.04.3 comes along I should be ready as I will copy all this to my notes.
I think that for this cycle if you have the -hwe-16.04 package installed it is suppose to automatically update to the 17.04 kernel as a normal update.
and then the 17.10 when it reaches 16.04.4, and so.
I think they learned that having set lts-release-name kernel branches for the LTS left too many users on unsupported kernels,
(like someone who installed 14.04.2 would have been on the 14.10 kernel (whatever that kernel series was; I think 3.16.X) and may have been on it for months or even years after Ubuntu ended support for that kernel series)
The new way should just roll into the next kernel and not require the hoops it did for the way it did before. 
As if that makes sense.

I think that's the plan.
