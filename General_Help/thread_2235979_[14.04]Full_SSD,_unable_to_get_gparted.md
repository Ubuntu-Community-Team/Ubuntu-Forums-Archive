### Post by oldfred on 2014-07-24
You cannot run gparted from your working system anyway. You cannot modify mounted partitions.

You need to use Ubuntu live installer DVD or flash drive or a gparted liveCD. You may have to also unmount swap as it usually is automounted to speed up live installer. You have to click on swap and right click swap off. Little key symbols show mounted partitions.

       GParted partitioning software - Full tutorial 
[http://www.dedoimedo.com/computers/gparted.html](http://www.dedoimedo.com/computers/gparted.html)
Screenshots of using gparted
[http://www.howtoforge.com/partitioning_with_gparted](http://www.howtoforge.com/partitioning_with_gparted)

It seems like you made partitions a bit small. Typically I suggest 10 to 20GB for / (root), swap at 2 or 3GB and /home the rest of the space you want to allocate to Ubuntu. Often if dual booting another NTFS shared data partition makes sharing data easier & safer.

If course with any major system change like moving partitions around, good backups must be made.

### Post by oldfred on 2014-07-24
Best then to see details. Just run BootInfo report. So not run auto repairs with multiple drives, that just installs one boot loader to every drive, and most users want or need different boot loaders in every drive.

 Boot Repair -Also handles LVM, GPT, separate /boot and UEFI dual boot.:
[https://help.ubuntu.com/community/Boot-Repair](https://help.ubuntu.com/community/Boot-Repair)

New hardware is UEFI, but has a BIOS mode. But you need to be consistent on how you install systems. Either all UEFI or all BIOS. If not the same then the only way you can dual boot is from UEFI/BIOS menu. And some systems require you to manually turn on/off UEFI or BIOS Mode to boot a different system in different boot mode.
