### Post by devil103 on 2011-12-15
Currently, I have a headless file server running on Ubuntu 11.10, kernel 3.00.14 (originally 11.04 netinstall) which has two main harddrives, one of which is completely storage and the other contains the Ubuntu installation

This disk is split into two partitions:
/boot ext4

The rest of the disk is an lvm group divided into
lvm-var ext4
lvm-root ext4
lvm-home ext4
lvm-swap

This has all been working great and the choice was made for LVM to be able to dynamically add or remove capacity from the different partitions.

However, booting has been extremely slow! Over two minutes in dmesg!

I've recreated my setup in a VirtualBox and have encountered the same problem.

Here are the useful excerpts:

dmesg
> [    4.937977] EXT4-fs (dm-1): mounted filesystem with ordered data mode. Opts: (null)
[   67.451970] udevd[290]: starting version 173
[   67.780143] Adding 975868k swap on /dev/sdb1.  Priority:-1 extents:1 across:975868k

fstab generated by Ubuntu installation
> 
/dev/mapper/homeserver-lv--root / ext4 errors=remount-ro 0       1
/dev/mapper/homeserver-lv--var / ext4 defaults 0       2
/dev/mapper/homeserver-lv--home/ ext4 defaults 0       2
UUID=c257565b-ced4-4cbe-a3d6-d013dbf12e12 /boot ext4 relatime        0    2


Since I've been able to recreate it, is this a known bug with a solution? If not, what are possible solutions to try?

### Post by bryan.ross on 2012-01-23
Thanks cweiske.  The following workaround worked for me.

                 Added the --noudevsync parameter to the vgchange command in /lib/udev/rules.d/85-lvm2.rules. Regenerated the initramfs with update-initramfs -u
