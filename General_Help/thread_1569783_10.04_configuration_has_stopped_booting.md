### Post by an0dos on 2010-09-07
The script you tried probably created a bad entry on your  /etc/fstab. I think you can boot from a live cd, navigate to your hard drive, and remove the problematic entry (it will probably be the one that contains 'ntfs-3g'). Be sure to back up you fstab before editing it.

Read more about the fstab here: 
[https://help.ubuntu.com/community/Fstab](https://help.ubuntu.com/community/Fstab)
[https://help.ubuntu.com/community/AutomaticallyMountPartitions](https://help.ubuntu.com/community/AutomaticallyMountPartitions)
[https://help.ubuntu.com/community/Mount](https://help.ubuntu.com/community/Mount)

### Post by coffeecat on 2010-09-12
> **j3k2008 said:**
> time to find an alternate way to mount them.

Kindly advice: don't use ntfs-config from the Ubuntu repository. I was involved in another thread where it too made a system unbootable. :-s

an0dos has posted some good links. I know it's not as friendly as using a GUI tool, but there's nothing better than editing /etc/fstab manually. Especially since GUI tools for this tend to make systems unbootable!

If, after reading through those links, you need some help with fstab, post the terminal output of these three and someone will help you:

```
sudo fdisk -l
sudo blkid
cat /etc/fstab
```Please enclose the output in code tags (easy way is to click on the # icon just above the message field) to preserve formatting and keep the whole thing readable.

By the way, if you are wanting to automount your Windows C: drive, that's a bad idea. OK to read from it, but writing to a Windows 7 or Vista C: drive can seriously upset Windows. Better to use a separate NTFS partition for your personal data which can be accessed from both Ubuntu and Windows.

### Post by Lateralis on 2010-09-12
Yeah, your fstab is wrong.  I've highlighted the offending lines below:

```

# /etc/fstab: static file system information.
#
# <file system> <mount point>   <type>  <options>       <dump>  <pass>

proc    /proc    proc    nodev,noexec,nosuid    0    0
#Entry for /dev/sda6 :
UUID=EAFC17D9FC179F3F    /media/Mony    ntfs-3g    defaults,nosuid,nodev,locale=en_US.utf8    0    0
#Entry for /dev/sda1 :
UUID=F8A870DBA87099B8    /media/sda1    ntfs-3g    defaults,locale=en_US.utf8    0    0
/host/ubuntu/disks/root.disk    /    ext4    loop,errors=remount-ro    0    1
/host/ubuntu/disks/swap.disk    none    swap    loop,sw    0    0
[B]UUID=Mohsen    /media/sda5    ntfs-3g    defaults    0    0
UUID=Mony    /media/sda6    ntfs-3g    defaults    0    0[/B]

```

You need to change these two lines.  I believe changing the "UUID=" to "LABEL=" will work, although I have never done that before.  So my suggestion would be to change the highlighted lines to:

```

/dev/sda5    /media/sda5    ntfs-3g    defaults    0    0
/dev/sda6    /media/sda6    ntfs-3g    defaults    0    0

```

or if you fancy:

```

UUID=20DC1154DC112616  /media/sda5    ntfs-3g    defaults    0    0
UUID=EAFC17D9FC179F3F  /media/sda6    ntfs-3g    defaults    0    0

```

There's also another problem. Near the beginning of the file is another one for sda6, mounting to another mountpoint - /media/Mony.

@mina mohsen, change these two lines:

```
#Entry for /dev/sda6 :
UUID=EAFC17D9FC179F3F    /media/Mony    ntfs-3g    defaults,nosuid,nodev,locale=en_US.utf8    0    0
```... so they look like:

```
#Entry for /dev/sda6 :
#UUID=EAFC17D9FC179F3F    /media/Mony    ntfs-3g    defaults,nosuid,nodev,locale=en_US.utf8    0    0
```The # at the beginning of the line comments it out making it inoperative. As far as the options are concerned, "defaults" for ntfs-3g is fine. You don't need all those extra ones in the now commented line.
