### Post by wilee-nilee on 2011-01-21
Has windows ever shown in the grub menu? Install gparted a partitioner in Ubuntu.
sudo apt-get install gparted

Open gparted and look for any flags on the Windows partitions, if so right click the partition then information. If you resized W7 with the Ubuntu installer partitioner you may just need to run a chkdsk /f/r on windows.

You may also be having problems here.
sda5: _________________________________________________________________________

    File system:       ext4
    Boot sector type:  -
    Boot sector info:  
    Operating System:  Ubuntu 10.10
    Boot files/dirs:   **/boot/grub/menu.lst** **/boot/grub/grub.cfg** /etc/fstab 
                       /boot/grub/core.img

You have grub-legacy here and grub2, the script shows it is using grub2. Did you do a grub-legacy upgrade or a OS upgrade that didn't finish the grub2 install.

### Post by wilee-nilee on 2011-01-22
> **Castlez93 said:**
> you have to understand i have absolutely no experience with computers at the exception of a little of this and that. in laymens terms please =)

Courtesy of drs305, from the link.
In the installed Ubuntu terminal copy and paste these commands, 

```
 
[COLOR="DarkRed"]sudo apt-get purge grub grub-pc grub-common[/COLOR]

``` 


**[COLOR="Navy"]4. Reinstall the grub packages.[/COLOR]** Here is what will happen:
[LIST=a]
[*]You will be given the opportunity to add extra kernel options to the kernel line. If you don't know, you probably don't need them ;  TAB to highlight "<OK>" and press ENTER. 
[*]Read the installation notes. TAB to "<OK>" to continue.
[*]When presented with the device option, use the UP/DN keys to select the correct *drive* (**[COLOR="DarkRed"]sdX[/COLOR]**). 
[LIST]
[*]Make sure the installation *drive* **[COLOR="DarkRed"][noparse]  [*]     /dev/sdX[/noparse][/COLOR]** has an asterisk next to it ( example: [noparse]  [*]  /dev/sda  [/noparse]). If it doesn't, highlight it and press the SPACE bar to select it. 
[*]Do not select a partition ( example: [noparse]  [ ]   /dev/sda5  [/noparse] , etc).
[/LIST]
[*]TAB to "<OK>" and press ENTER. When it has finishing the installation, you should have Grub 2 installed.
[/LIST]

```
[COLOR="DarkRed"]sudo apt-get install grub-common grub-pc[/COLOR]
```
then
```
sudo grub-install /dev/sda
``` 


**[COLOR="Navy"]5. Update the Grub 2 files .[/COLOR]** This command shouldn't be necessary, but it won't hurt to update Grub once more before exiting.  
```
 
[COLOR="DarkRed"]update-grub [/COLOR]

```
