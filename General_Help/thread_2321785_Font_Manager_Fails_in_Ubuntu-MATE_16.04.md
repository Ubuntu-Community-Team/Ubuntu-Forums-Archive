### Post by wagb278 on 2016-04-24
The Font Manager application fails to initialize in Ubuntu-MATE 16.04.  I installed this package using the Ubuntu Software Center, getting version 0.5.7-4.  When I run Font Manager it starts to initialize saying "Querying installed files..." but quickly hangs.  An Exit needs to use the Force Quit and then I get the "Not responding" message.  So there is something wrong with this version of Font Manager or its integrating with Ubuntu-MATE 16.04. I run 64-bit version of Ubuntu-MATE.

My objective in using this tool is to disable the non-English fonts so my list of available fonts in LibreOffice includes only the fonts I want to use.  If there is another tool to accomplish this I welcome advice; but I have been using this Font Manager tool for many earlier Linux distributions and have never before had a problem disabling fonts. 

**EDIT**: I removed Font-Manager version 0.5.7 (the one in the Ubuntu Repository) and installed version 0.7.3 using the PPA method and that version does run on Ubuntu-MATE 16.04.  So I was at least able to disable all the fonts I don't use - the GUI in the new version took some getting used to.
