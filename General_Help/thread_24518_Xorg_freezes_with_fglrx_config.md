### Post by maqi on 2005-04-07
Which version of the drivers did you install -  official ubuntu or the rpm from the ATI website?

Are you using an official ubuntu kernel?

I had this problem when I used fglrxconfig to generate my xorg.conf file. I suspect that your problem is there.

I fixed it by logging in at the console - do <Ctrl>+<Alt>+<F1> when the xserver freezes. Did you back up your original xorg.conf file before you ran fglrxconfig? If so then it's easy - just do:

```
$ sudo cp -f /path/to/xorg.conf-backup /etc/X11/xorg.conf
$ sudo reboot
```

Hopefully that stops the xserver from freezing and you can then fix your xorg.conf file from a your GUI environment **without using fglrxconfig**. 

Otherwise:

```
$ sudo sh -c 'md5sum /etc/X11/xorg.conf >/var/lib/xfree86/xorg.conf.md5sum'
$ sudo dpkg-reconfigure xserver-xorg
```

And answer the questions - defaults should be fine to start with - but don't take my word for that ;)

After that try rebooting. Hopefully your xserver will work again.

Here is my xorg.conf. I got the necessary changes from [here](http://xoomer.virgilio.it/flavio.stanchina/debian/fglrx-installer.html#install). Check it out and have a read. My machine seems similar to yours - although I have a radeon9600pro. So the device identifier should be different - xorg should pick it up fine with the 'dpkg-reconfigure xserver-xorg' command. ### Post by cdhotfire on 2005-04-07
can you access computer if you choose ati?
if so, make sure u have installed fglrx drivers, and do this:
```

sudo /etc/init.d/gdm stop

```
that should take you off gnome now type this:
```

sudo dpkg-reconfigure xserver-xorg

```
choose fglrx not ati and continue the setup until finished, then do
```

startx

```
or you may do
```

sudo /etc/init.d/gdm start

```

Hope this works. :)
