### Post by Scoobin on 2010-08-02
I hadn't done any major updates for about a couple weeks, and then I downloaded a whole bunch - 100+ mb worth. One of these updates was the new kernel 2.6.32-23 (I'm now on the 2.6.32-24 kernel with no improvement). After doing these updates my computer froze at the point where the 5 white dots (10.04) became red upon booting. Since then it generally crashes about every 2nd boot. Also, the times when it does boot, it takes a bit longer to boot (maybe 5-10 secs). It's no longer snappy like the 10.04 it was just prior to doing the updates. 

Today it seemed to freeze during login with the prompt still showing once I'd put in my details. When I say it freezes, nothing is happening but I can still do CTRL+ALT+DELETE to reboot the computer, so it's not a full freeze.

I've been asked does the old kernel crash it and having tried it once or twice I think it still crashed once. So perhaps it's not the kernel. 

Does anyone know any other recent updates from July that might be causing this, or have any other ideas?

### Post by Scoobin on 2010-08-03
I've fixed it. Uninstalling and reinstalling the Nvidia drivers did the trick. I recall one of said updates being an Nvidia update, which must have conflicted with something when I installed it. It still doesn't boot as quick as it used to but at least it's working.
