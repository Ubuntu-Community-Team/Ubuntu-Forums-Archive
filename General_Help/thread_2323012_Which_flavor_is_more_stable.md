### Post by grahammechanical on 2016-05-02
All computer operating systems require maintenance in the form of updates. How often we update is up to us. We can set the frequency of when to check for updates. We can set to be notified of any security updates immediately and even to have them installed automatically.

This is the same for all flavours of Ubuntu because all flavours of Ubuntu use the Linux kernel. So, when there is a security fix to the Linux kernel all flavours of Ubuntu will get the upgraded Linux kernel because all flavours of Ubuntu use the same software repositories. It is the same with any other kind of security fix

Are you thinking of defraging the hard disk as a form of maintenance? Not necessary in Linux. Anyway, all flavours of Ubuntu use the same file system format of EXT4 as the default file system format.

In many ways the only distinguishing features of the different Ubuntu flavours are the choice of desktop environment (DE) and user interface (UI). Our choice of DE is dependent on (a) the hardware being capable of running the DE. (b) Our taste in UI.

Every Linux distribution has as many problems as it has users. Everything happens for a reason. And the reason most users have problems with the OS is because of something that the user has done. 

Problems installing? If there are going to be problems installing then it makes no difference what flavour of Ubuntu we use. The problems are going to happen just the same. And we need to do the same pre-installation research for all flavours. Besides, each flavour uses the same installer program. 

Building a Linux distribution on Ubuntu has certain advantages. One of which is to have the same stability and low maintenance costs to the user that Ubuntu has.

Regards.

### Post by HermanAB on 2016-05-08
Linux is as much, or as little trouble as you want it to be.

If you want something 'different', then install regular Ubuntu. 

If you want something that 'looks like Windows', then install Xubuntu.  

If you want zero maintenance, then turn automatic updates off and leave the machine alone.  It will work perfectly fine for years and years.  Usually, the hardware fails after a few years, before anything goes wrong with Linux.  

If you want low maintenance, then instead of regular updates, you can periodically install a newer version of Ubuntu - say every year at Christmas.

...and if you want a fresh load of trouble every few weeks, like everyone else on these forums, then leave automatic updates enabled.
