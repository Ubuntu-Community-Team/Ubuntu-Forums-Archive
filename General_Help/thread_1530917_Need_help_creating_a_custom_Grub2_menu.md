### Post by drs305 on 2010-07-14
freddiespagheti,

Added: Please read the next post first.

Thanks to you I learned a bit more about Grub 2 today. I had not used a custom configuration file before, so I had to do a bit of studying.

First, I would keep the menuentry for the custom configuration file as brief as possible to avoid errors. The entry below is all that should be necessary in your 40_custom file for the proper transfer to the custom cfg file. If you want to leave the class info such as "--class Ubuntu" you can but it doesn't seem to be required. 

Note I removed the lines that may have caused you problems but added the device designation in the path to the custom config file. On my system, it was (hd0,1), but make it point to whatever device contains your /boot/grub folder.

> 
menuentry 'Configfile' {
	configfile (hd0,3)/boot/grub/custom_config.cfg
}


That should pass control to your custom config file (make the change if it's somewhere other than sda3).

Since we haven't seen your custom config file, just remember it must be Grub2 compatible. Since I didn't have a custom cfg file, I used "sudo grub-mkconfig -o /boot/grub/custom_config.cfg" to create one that I knew would be acceptable to Grub 2. Then I edited it to suit my needs.

Once I generated a new custom cfg file, I opened it, left the first portion intact and just removed the menuentries I didn't want in the #10_linux, #20_memtest86+, #30_os-prober and #40_custom sections.

Then I ran update-grub to incorporate the changes (for the changes I made to 40_custom).
Added: A note about updating. "update-grub" is a stub for "grub-mkconfig -o /boot/grub/grub.cfg". It will update the contents of the main configuration file (/boot/grub/grub.cfg) but will not effect /boot/grub/custom_config.cfg unless the full command/path are used (at which point it would overwrite the contents of the custom config file).

I assume you followed a similar procedure. Hopefully just changing the config file address was all you needed. If your custom config menu isn't working, I suppose it would be time to post the contents and possibly run the [http://bootinfoscript.net/]("http://bootinfoscript.net/").

Let us know either way.

### Post by drs305 on 2010-07-14
Reviewing your posts, I realize that I responded based on your "configfile" entry. However, a custom config is not required to use a custom menu. The custom configfile is used to transfer control to a completely new cfg file. 

A custom menu merely adds entries to your existing menu. Usually a 40_custom entry is easier. It is incorporated into the standard grub.cfg file merely by running "sudo update-grub", provided your custom menu file is executable and located in the /etc/grub.d folder.

### Post by freddiespagheti on 2010-07-14
Aha! I figured it out. First I tried as you suggested and used "sudo grub-mkconfig" to be sure the menu I was using was compatible with grub2. It still didn't work though.

Instead, the problem was really that I have a separate boot partition, so instead of

```
menuentry 'Configfile' {
configfile (hd0,3)/boot/grub/custom_config.cfg
}
```

I needed to omit the 'boot' directory like so:

```
menuentry 'Configfile' {
configfile (hd0,3)/grub/custom_config.cfg
}
```

and now it works like a charm.


As for your last note, I wanted to create a custom config because my default menu is very cluttered from having 3 OS's. One of them doesn't boot from the default entry created by grub so I had to create a custom one. On top of that, there's a Windows recovery partition that I want to hide. My new menu is just three entries plus one more that I called 'Advanced' to take me to the original grub.cfg .
