### Post by Dennis N on 2016-03-09
> Error: You are missing the following 32-bit libraries, and Steam may not run:
libGL.so.1

So is there anyway to fix this without a clean install? 


First, enable installation of 32-bit packages:

**sudo dpkg --add-architecture i386**

When I wanted to add 32-bit capability for games, I followed the suggestions for Ubuntu 13.04 and later in the following link, and installed all their suggested packages:
[https://support.humblebundle.com/hc/en-us/articles/202759400-Installing-32-bit-libs-on-a-64-bit-Linux-system](https://support.humblebundle.com/hc/en-us/articles/202759400-Installing-32-bit-libs-on-a-64-bit-Linux-system)

As I remember it, when you install the suggested package **libglu1-mesa:i386** you also install some dependencies, including a package that contains the file you mention. I installed mine with Synaptic, because it is easy to filter for 32-bit packages with the "architecture" button on the left. All of them end with :i386
