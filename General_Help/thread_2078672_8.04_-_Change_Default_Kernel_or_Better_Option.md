### Post by snowpine on 2012-10-31
Welcome to the forums!

The 8.04 release is end-of-life, obsolete, done.
If there is a bug in its kernel, it will never ever be fixed.
I recommend you install the current 12.10 release or the 12.04 long term support release from: [http://ubuntu.com/download](http://ubuntu.com/download)

That being said, the temporary band-aid solution in your case is to remove the offending .32 kernel using apt-get (terminal) or synaptic package manager (GUI). 

Here is a kernel removal how-to (be sure to replace their kernel number with your actual kernel number): [http://askubuntu.com/questions/106031/how-can-i-remove-new-kernel-versions-after-downgrading](http://askubuntu.com/questions/106031/how-can-i-remove-new-kernel-versions-after-downgrading)
