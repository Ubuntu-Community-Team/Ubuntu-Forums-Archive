### Post by jmmcd on 2006-10-20
Hi nmsb,

I've also been struggling with xmodmap recently, but I think I've cracked it now. As well as using 'keycode x = y' commands (which change the keysym generated by a keycode) you also need to think about the modifier map, which says which keycodes cause which modifiers (shift, lock, control, and mod[1-5]). 

Use xmodmap -pm to see what the current modifier map is, and check that there is no overlap between modifiers: eg check that ISO_Level3_Shift doesn't cause more than one modifier.

Btw (you probably already know this) you can try out individual xmodmap commands using xmodmap -e, rather than editing a save file - this might be quicker for experimentation.

Good luck,
James
