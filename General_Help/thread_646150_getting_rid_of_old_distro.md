### Post by logos34 on 2007-12-22
> **MEGAMANULTIMATE said:**
> Unable to delete /dev/sda5!
Please unmount any partitions having a number higher than 5.

I get that message a lot--you have to umount the other logical partitions inside an extended.  But you may have some apps cached in the swap, so the safer thing to do is to reboot to the live cd and use gparted to work on your unmounted partitions.  The easier approach is to simply delete the extended partition and create a new primary for your swap.  Format the old / partition sda1 as ext3 and turn it into /home by copying over your home dir on new / partition ([this is the method]("http://www.psychocats.net/ubuntu/separatehome") I'm familiar with.).

Try this:

1.  Edit fstab first:

gksudo gedit /etc/fstab

change swap line to this (w/o UUID):
> 
/dev/sda2 none swap sw 0 0

2. Insert ubuntu live cd and reboot.  The Bios should be set to boot from cdrom first.

Open a terminal on the desktop and type:

gksudo gparted

-format sda1 as ext3.  This wipes it clean.

-delete sda5 and sda6, then delete sda2, the extended.  Create a new partition the same size as sda6 at the end of the disk. Format as linux swap.  Expand sda3 (root) to incorporate the intervening empty space. 

Now, your disk should look like this (ignore scale):

|------------sda1---------|--------------sda3 (root)---------------|sda2(swap)| 

Now you have three primary partitions, with swap last.

Reboot into ubuntu.  Shouldn't have a problem since we took out the UUID for swap and used the '/dev/' format.

Now you can proceed with making a separate /home.
