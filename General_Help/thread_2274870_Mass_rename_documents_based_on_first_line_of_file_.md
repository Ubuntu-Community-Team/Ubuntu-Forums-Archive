### Post by Impavidus on 2015-04-23
Simply using **head** will just get the magic numbers. Not very useful indeed. The tool **docx2txt** (available from the repositories in a package of the same name) is supposed to convert docx files to plain text. Pipe the result into **head** to get the first line. Use this to rename the file. I never used docx2txt myself and don't have it installed, so I can't comment on the details. First run some tests in a safe place.

Note that photorec may have recovered many different versions of the same documents. These may have the same first line. To prevent old versions of the file from overwriting a new version, keep the filenames generated by photorec and prepend the first lines:```
f8349839.docx >> First line A-f8349839.docx
f2932938.docx >> First line B, which has a duplicate-f2932938.docx
f2398483.docx >> First line B, which has a duplicate-f2398483.docx
```
**pdftotext** (available from the repositories in the package poppler-utils; installed by default) can do the same for pdf files.

### Post by Holger_Gehrke on 2015-04-23
> **Roasted said:**
> 

I was looking at something like: catdoc filename.doc | head -n1

That displays the top line. BUT how can I take that top line and rename the file "as" the top line *and* retain the .doc extension? If I can figure that out that'd be significantly closer.

You use a command line substitution like so:```

mv -n filename.doc "$(catdoc filename.doc|head -n1)".doc
```
The "$( ... )" puts the output of the command inside it into the command around it. The double quotes around the substitution are important, the output of 'catdoc' will probably have spaces in it which would split the new name with disastrous results ... The '-n' option to 'mv' prohibits overwriting - a sensible precaution, I think.

And now an example how to do this with a lot of files all at once, using wvSummary to get the title from the metadata:
```

 for i in *doc ; do a=$(wvSummary "$i" | grep 'Title = ') ; a=${a%*\"} ; a=${a#*\"} ; if [ -n "$a" ] ; then echo \"$i\": \"$a\" ; fi; done

```
This loops over all doc files, gets the Title line from the metadata, removes everything outside the double quotes around the title from it and outputs the old filename followed by the new one, leaving out any files which don't have a title. Replace the 'echo \"$i\": \"$a\"' with 'mv -n "$i" "$a".doc' to actually rename the files.

Did you try wvSummary on a few files ? If the files are okay, there **always** is some metadata, at least the dates of file creation, last modification and last printout. That second command depends on the title being set in the metadata and leaves files without a title alone (the if ... ; then .. fi part of the command executes only if "$a" (the variable containing the title) is not empty.
> **Roasted said:**
> 
I'm curious on how else I can structure a scripted way to do it. Any ideas on how I could substitute the filenames in the actual command to cycle through and take care of every .doc in the directory, assuming no metadata is there to come back full circle like the command in your 2nd example?

You'd do a 'for' loop similar to my second command but without all the string manipulations.```

for i in *doc ; do mv -n "$i" "$(catdoc "$i"|head -n1|tr -d '\000-\037[]{}()/\?*')".doc; done
```
The "for" takes the list of all doc file names, assign them one at a time to the variable i. For each value of i it does an mv with the command substitution. I've added a 'tr' command to the substitution to filter out illegal or inconvenient characters. Do make backup copies of the files before you run this.
