### Post by gradinaruvasile on 2010-01-18
There isnt a big difference between 8.04 and 9.10 speed and hardware requirement wise - 9.10 actually is faster (and uses the same amount of memory roughly) AND its compatible with newer/more hardware (has updated drivers included) - and all recent applications are easier to find for this version, the 8.04 despite the LTS status is not supported by many new application versions. 

So, i recommend 9.10.

Its not the same comparison as in case of Windows versions - Linux isnt growing in hardware specs by hogging more bytes of memory and hertz of your CPU... 
It just uses common sense - and its not really loved for this - what would the hardware vendors do if there wasnt a Vista/Windows7 that require you to buy a new, faster computer?

The difference between Xfce and Gnome is quite visible - but as always there is a tradeoff - Gnome has built-in support for more stuff than Xfce, its more capable out of the box. 
But Xfce is faster because it doesnt have to load that many modules in memory - and the windows draw/move faster than on Gnome too. Xfce doesnt have hardware accelerated screen compositing included by default. The engine is based on GTK toolkit just as Gnome - the apps are perfectly compatible. 

Xfce handles shares differently from Gnome - you have to use a specialised program to mount them (gigolo) and another one to share them (dunno its name). In Gnome all this is included in Nautilus - you can share easily with right-click a la Windows and access shares directly from the address bar (smb://computername). Gnome it a more tightly integrated environment.

But seeing your specs you can run either Gnome or Xfce without problems. If you are a beginner i would say Gnome (its the default Ubuntu).
