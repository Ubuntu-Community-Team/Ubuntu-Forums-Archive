Hi all,

I have recently migrated my home storage server from centos to ubuntu 8.04.3 LTS.

One of the things I did have working previously was a bash script which would be invoked by the cron daemon every five minutes. This script would check for the existence of the /storage/shutdown directory (/storage is a directory being shared by samba). Once the directory was detected it would be deleted and the server shutdown. 

I am trying to recreate the same behaviour with ubuntu but to no avail.

Below the cron job
```
billy@kanga:/etc/cron.d$ cat /etc/cron.d/shutdown-check 
#
# Regular cron job for shutdown check
#
# Every 5 minutes.
*/5 * * * *	root	/usr/local/bin/shutdown.sh 2>&1 >> /var/log/shutdown.log

```

Below the contents of /usr/local/bin/shutdown.sh
```

billy@kanga:/etc/cron.d$ cat /usr/local/bin/shutdown.sh 
#!/bin/bash
if [ -d /storage/shutdown ]; then
	rm -Rf /storage/shutdown
	rm -Rf /storage/temp/*
	poweroff &
fi

```

Any ideas?

Cheers ws

### Post by geirha on 2009-08-22
> **websavages said:**
> Hi,

In BASH '2>&1' will redirects stderror to stdout. In other words any output generated by the command will be written to the file. 


That's not entirely correct. Firstly, cron uses sh, not bash. And 2>&1 will redirect stderr to the same file as stdout is currently redirected. And the redirection operators are processed left to right, so with 2>&1 >> file, stdout is going to file, but stderr is not. You probably intended it to be >> file 2>&1 which redirects both stderr and stdout to file.

```
$ { echo stdout; echo stderr >&2; } >>/dev/null 2>&1 # (no output)
$ { echo stdout; echo stderr >&2; } 2>&1 >>/dev/null
stderr

```
