### Post by john newbuntu on 2009-12-19
Somebody correct me if I am wrong.  The GRUB_HIDDEN_TIMEOUT, from what I saw in the 30_osprober script, gets used only if the osprober finds no operating systems. If at least one OS is found, the menu will be shown.

When no OS is found, the GRUB_HIDDEN_TIMEOUT is used for either overriding the GRUB_TIMEOUT value or for stopping at menu with the shift key.

So, if you really need to skip the menu, then set GRUB_TIMEOUT to 0.

### Post by Leppie on 2009-12-20
this is a known issue, see [launchpad bug]("https://bugs.launchpad.net/ubuntu/+source/grub2/+bug/444495").

as a workaround, try adding the following to the end of /etc/grub.d/00_header:
```
### BEGIN Hidden Menu Test ###
cat << EOF
if [ ${timeout} != -1 ]; then
if sleep --verbose --interruptible ${GRUB_HIDDEN_TIMEOUT} ; then
set timeout=${GRUB_HIDDEN_TIMEOUT}
fi
fi
EOF
### END Hidden Menu Test ###
```
