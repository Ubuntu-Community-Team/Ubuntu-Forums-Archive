### Post by renzokuken on 2007-02-23
if you have installed the fglrx driver then you are not using it, you are actually using the old open source "ati" driver, as shown here

```
Section "Device"
	Identifier	"ATI Technologies, Inc. 3D Rage PRO AGP"
	Driver		"[color=red]ati[/color]"
	BusID		"PCI:1:0:0"
EndSection
```

firstly back up your xorg.conf

```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf_backup
```

then open the xorg.conf to edit it

```
sudo gedit /etc/X11/xorg.conf
```

you need to change that section so it looks like this........

```
Section "Device"
	Identifier	"ATI Technologies, Inc. 3D Rage PRO AGP"
	Driver		"[color=blue]fglrx[/color]"
	BusID		"PCI:1:0:0"
EndSection
```

### Post by dsimpson on 2007-02-24
Hello MurasakeNemo, 

I don't know if this applies to your card or not, but my card is an Ati 128 Rage and I had the same problems as you or so it seems. I was told there is a bug that affects VIA Ati video cards and the fix that I used was to edit my /etc/X11/xorg.conf file and change the driver from ati to vesa. I don't know exactly why that works but it did for me. I tried the fslgx drivers and they did the same thing you mentioned. I tried setting back to ati and my screen went black and I was unable to log in. Finally I admitted that the only way I could run my system with my video card was to set it at "vesa". Go to your /etc/X11/xorg.conf file and where it says Section , "device", driver ati, change to driver vesa. You may have to log out and back in but I think that might do the job for you. I set my resolution to 1024 X 768, and the refresh rate at 60, you might try it higher if you feel you need to. Hope this helps.:)
