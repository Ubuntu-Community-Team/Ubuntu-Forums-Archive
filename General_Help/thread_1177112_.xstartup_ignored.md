### Post by Ace_NoOne on 2009-06-03
My Intrepid machine uses the following *.xstartup* config: 
```
# use Windows key as modifier
#xmodmap -e "remove mod4 = F13"
xmodmap -e "keycode 115 = Super_L"
xmodmap -e "add mod4 = Super_L"

# disable Caps Lock
xmodmap -e "remove Lock = Caps_Lock"
xmodmap -e "keysym Caps_Lock = Escape"
```
While remapping the Windows key works fine, remapping Caps Lock does not.
However, if I manually source that file, it both works.

Is it possible that *.xstartup* is ignored?
I know I could use *.Xmodmap* instead - but where would I source that from? Neither *.bashrc* nor *.gnomerc* seem quite right for that...

---

### Post by Brandon Williams on 2009-06-03
I think that .Xmodmap is handled automatically if it exists. However, if that doesn't work, then you should use either .xsessionrc or .xprofile, both of which are sourced during session startup if they exist.
