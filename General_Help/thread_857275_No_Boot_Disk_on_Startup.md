### Post by sclcommish on 2008-07-12
I'm relatively new to xubuntu, but about a month ago, I couldn't boot my computer. It would just tell me no book disk found, run setup. I'd go into the BIOS and my hard drive wouldn't be recognized.

I chalked it up to a bad hard drive. But when I took the hard drive out and connected it to another machine, it seemed fine.

So I switched out the IDE cable on a lark, and I was able to boot. I thought it was just the cable.

But this morning I tried to boot and got the same error. I'm 95% sure this is probably a hardware error - is there anything I can/should do to confirm this? Any suggestions?

It just seems odd that it booted for a while and now it doesn't, so is there any chance this is a software issue?

### Post by coffeecat on 2008-07-12
A long shot this, but it could be a failing power supply. I've had two fail but when they failed there was still some power getting through - fans came on - so it took me some time to realise what was going on. When the first one failed the hard drive was never detected. With the second one (different machine with 2 HDDs) sometimes the HDs were detected and I could boot up and sometimes not. But I put up with this only over a few hours. The previously near-silent PSU had begun to get noisy so I swapped in another PSU and haven't had a problem since.

But in your case this first manifested a month ago. I really don't know whether a failing PSU would last that long. Nevertheless, if you can get hold of one it might be worth fitting a different PSU and seeing if that makes any difference.

Yes, and recheck the IDE cable. Swap it for another one. Make sure it's seated properly.
