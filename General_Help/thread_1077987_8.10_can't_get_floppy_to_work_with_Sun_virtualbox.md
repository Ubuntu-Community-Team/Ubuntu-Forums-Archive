### Post by guiliker on 2009-02-22
I'm posting here as I feel it may be a problem with Ubuntu rather than Sun VirtualBox

Host OS: Ubuntu Intrepid Ibex
Kernel: 2.6.27-11-generic
Virtualisation: Sun xVM Virtual Box 2.14 (from the .deb file on the website)
Guest OS: WinXP Spk3 with Guest Additions installed

Firstly I realised just when I needed it I couldn't get a floppy to work so after googling, reading the forums and launchpad I managed to get it to work by following the these instructions:

adding floppy to /etc/modules using sudo gedit /etc/modules
then sudo gedit /etc/fstab to add the line: 
/dev/fd0 /media/floppy0 rw,user 0 0 0

sudo mkdir /media/floppy0
sudo chgrp plugdev /media/floppy0
sudo chmod g+rwx /media/floppy0
sudo chmod o-rx /media/floppy0
sudo mount -o uid=$USER,gid=floppy /dev/fd0 /media/floppy0

floppy seems to work although I've not rebooted yet...
I open VirtualBox and add floppy to the settings yet when I came to start the guest box I get the following error:

```
Cannot open host device '/dev/fd0' for read/write access. Check the permissions of that device ('/bin/ls -l /dev/fd0'): Most probably you need to be member of the device group. Make sure that you logout/login after changing the group settings of the current user (VERR_ACCESS_DENIED).
Unknown error creating VM (VERR_ACCESS_DENIED).

Result Code: 
NS_ERROR_FAILURE (0x80004005)
Component: 
Console

IConsole {e3c6d4a1-a935-47ca-b16d-f9e9c496e53e}
```

so naturally I go:
user@ubuntu:~$ /bin/ls -l /dev/fd0

What do I do to get it to work?

I also am confused with the amount of mozilla/firefox results coming up when I google the error failure code as it is fine as soon as I remove the floppy drive from the settings.

### Post by cariboo on 2009-02-22
Make sure you have the floppy module loaded. In a terminal type:

```
sudo modprobe floppy
```

By default the floppy module isn't loaded in Intrepid.

Jim
