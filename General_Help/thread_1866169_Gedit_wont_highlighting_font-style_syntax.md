### Post by MorrisseyJ on 2011-10-21
Hi, i am not sure if this is the best place to be asking this question, but people on here seem to know about these things.

I am teaching myself html, using gedit. When i try and build an external CSS file and set the font-style, gedit doesn't highlight some of my font syntax.

So some fonts will be highlighted, such as monospace and sans-serif, but others such as Garamond, arial and Helvetica won't be. I have all these fonts downloaded into my .fonts folder and can use them in other programmes such as LibreOffice. 

Also strange is that when i set one of these fonts as my font-style, my page opens displaying in the chosen font. The problem appears to be simply that that gedit is not highlighting the 'font' syntax. That said the highlighting is clearly working as it picks up some of the fonts, namely monospace and sans-serif.

This clearly isn't a debilitating problem in that the fonts are recognised and displayed when the html file is opened in a browser, but i was wondering if someone could tell me why the syntax highlighting doesn't work and/or how i could get it to work.

Thanks.

### Post by crdlb on 2011-10-21
Gedit is just highlighting those names as keywords. It has nothing to do with the syntax. 'monospace' and 'sans-serif' are not font names, they are generic descriptions.
