### Post by Patb on 2005-11-28
A minor annoyance that I have found in Ubuntu relates to the way the modifier key must be used for most menus and submenus, if one chooses to avoid the mouse.  In Gnome Gedit for example, to quit a file without saving, I need to press Alt-F, then Q, then Alt-W.  IMO, it would be simpler and more intuitive to hold the Alt key down while typing all keys for the menu commands, in other words to type Alt-F Alt-Q Alt-W.  After all, on the "File" menu, the Q of "Quit" is underlined - so why does it not respond to Alt-Q (as well as just Q)?  

This 'annoyance' is not present in Openoffice Writer running on Ubuntu nor on Firefox, but the inconsistency between programs is itself not good.  I find I have to stop and think which program I am in to undertake essentially the same step.

Have I missed something?  Is there a way to get all program menus to accept Alt-key combinations where it is obvious what the user intends?  Comments appreciated, Pat. (New to Ubuntu).
