### Post by schmolch on 2010-07-21
Copy your existing xorg.conf to xorg.conf.backup (or something like that) so you have a backup.
Then you can edit it as root (sudo gedit /etc/X11/xorg.conf).

### Post by schmolch on 2010-07-21
Also, if your edited xorg.conf does not work, you will find the reason in /var/log/Xorg.0.log

"cat /var/log/Xorg.0.log" will display the whole file.
If you do "grep EE /var/log/Xorg.0.log" it will only show the lines with EE (meaning error) in it.

AFAIK such cards dont exist.
There are a few nvidia quadro cards but they have just 2 GPUs on the same board so technically you still have two seperate cards and need to use xinerama.
I think there are some new radeon cards with many outputs but i doubt the linux radeon driver can handle them yet.

The only good solution i know about if you already have 3 monitors is using the matrox triple-head-2-go which is a external box that can split the video-signal to 2 or 3 monitors but it costs a few hundred bucks. You can checkout the nvidia linux forum where a few people have experience with it.
