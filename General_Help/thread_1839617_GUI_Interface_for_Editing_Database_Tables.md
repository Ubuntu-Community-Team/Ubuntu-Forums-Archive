### Post by oldfred on 2011-09-06
If you have created the database then TeoBigusGeekus's suggestion of LibreOffice or if using an older version OpenOffice. 

Some other choices.
Kexi but the database has to be created with Kexi.
[http://www.kexi-project.org/](http://www.kexi-project.org/)

Collections - music, books, etc: both in repositories
Tellico - qt4 & python xml for data
[http://tellico-project.org/](http://tellico-project.org/)
Gcstar - perl & gtk xml for data
[http://www.gcstar.org/](http://www.gcstar.org/)

I am learning Python and you can with Python and gtk or qt realtively quickly create a displayable grid. Many examples in Google.
