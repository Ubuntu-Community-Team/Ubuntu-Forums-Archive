### Post by sudodus on 2018-05-15
The slow boot could be caused by some other problem too.

[hr][/hr]
This is one example, that made my Lubuntu boot slowly (until the problem was identified (by another person). It can happen when you upgrade Lubuntu from 16.04 LTS to 18.04 LTS.

If you have the file **/etc/initramfs-tools/conf.d/resume** (pointing to a swap partition or file), you can try to remove the file, or if you are cautious, change its content in order to make it 'do nothing'. See the following example,

```

#RESUME=UUID=03bed9e9-e39b-4107-8cd6-c2612af9a6aa
RESUME=NONE

```

After that you should make the change effective by running the following command

```
sudo update-initramfs -u
```

### Post by sudodus on 2018-05-23
Use encryption only if you need it, because it adds a layer of complication.

- More work must be done by the processor
- It is more difficult (maybe impossible) to recover data in case of a damaged file system
- You are lost, if you forget the passphrase

I think you selected 'LVM with encryption', when you were installing. You can re-install and select another alternative. The easy and simple standard alternative is to let the installer use the whole drive (but without LVM and without encryption).
