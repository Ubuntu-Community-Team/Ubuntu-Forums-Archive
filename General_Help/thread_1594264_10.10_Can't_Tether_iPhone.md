I've just followed this steps:


**UNR 10.10 - Update on Tethering to iPhone**
Like a few others, upgrading to [Maverick Meerkat]("http://www.ubuntu.com/netbook/get-ubuntu") killed iPhone tethering. Actually I found that this occurred on most kernel upgrades. Luckily, the fix is simple. [Recall]("http://frankhjung.blogspot.com/2010/09/ubuntu-netbook-remix.html"), that normal installation is simply:
Set repositories:
[INDENT][COLOR=navy][FONT=courier]sudo add-apt-repository ppa:pmcenery/ppa
sudo apt-get update[/FONT][/COLOR][/INDENT]Then install the drivers:
[INDENT][COLOR=navy][FONT=courier]sudo apt-get install gvfs ipheth-dkms ipheth-utils[/FONT][/COLOR][/INDENT]Now, on an upgrade these are already installed, and even a re-installation failed:
[INDENT][COLOR=navy][FONT=courier]sudo apt-get install --reinstall gvfs ipheth-dkms ipheth-utils[/FONT][/COLOR][/INDENT]Though, this normally works on a kernel upgrade.
Instead, un-install then re-install. If the kernel isn't updated on the re-installation the chances are it hasn't worked.
[INDENT][COLOR=navy][FONT=courier]sudo apt-get remove gvfs ipheth-dkms ipheth-utils
sudo apt-get install gvfs ipheth-dkms ipheth-utils[/FONT][/COLOR][/INDENT]

### Post by jpfiset on 2010-12-02
If you can not login, it is probably because the gnome-session was removed in the process, above.

You can reinstall gnome-session by typing: "sudo apt-get install gnome-session"

More details are available here: [http://www.bitsbythepound.com/re-install-gnome-session-in-ubuntu-361.html]("http://www.bitsbythepound.com/re-install-gnome-session-in-ubuntu-361.html")

### Post by johnbyrne on 2011-01-20
I survived this nightmare....twice! The solution is as follows:

You MUST be connected to a hardwired ethernet internet connection for this to work. In other words, you MUST have a reliable, cabled connection.

At the login screen, press Ctl-Alt-F1 to get a command line prompt.
Login with your username and password.
Type the command:
  sudo apt-get install ubuntu-desktop
Enter your password to allow the command to complete.
Type the command:
  sudo reboot
Enter the password, if necessary.

Voila! You should have your desktop back. Note, you will lose your customisations, but maybe someone else on the list knows a way to prevent that.

At least you didn't have to re-install from scratch!

Good luck

John
