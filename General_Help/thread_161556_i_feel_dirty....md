### Post by towsonu2003 on 2006-04-17
[QUOTE=frizzl]i was going to try it out my sis's computer, told it runs well on older systems... but then my DSL disc finished and i thought i'd check that out first.[/QUOTE]
dsl is very light. here are some info from inside a vmplayer dsl livecd, used as my sandbox (though it runs faster in qemu):

Swap Usage: 0 / 235MB total
RAM Usage: 24.7MB / 503MB total
kernel: 2.4.26

very flexible via "myDSL" and can also use apt-get (with tweaks??). and if run from livecd, will backup to an ext2 formatted partition if you tell it to do so. 

I usually copy all my .dsl files to an extra partition (in vmplayer), remove the dsl files I put to /home/dsl and tell it to backup to this extra partition. when rebooted, I "open" dsl's I want activated and the backup thing is already done automatically. 

a little info about backup I found useful: /home/dsl/.filelist.lst will have names of directories you want backed up; /home/dsl/.xfilelist.lst will have those that will be excluded from back up.
