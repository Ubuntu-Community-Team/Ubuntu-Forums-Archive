### Post by capscrew on 2010-04-08
I just thought of something that might help.

If you are NOT running Gnome (no X) then gvfs is not there ( or you can delete it.  It is dynamic and should be replaced when x and Gnome is running again.

First I would try a non-gnome console session.  Alt-f3 I believe is the command.  If that fails then boot the CLI and run quotacheck.

Edit:  it looks like this is the way to do it.  See:[_http://how2forge.org/comment/reply/4904/22982_]("http://how2forge.org/comment/reply/4904/22982").

I spotted this:> 2. In step 12 when installing Journaled Quota It kept giving me this error:

quotacheck: Scanning /dev/??? [/] quotacheck: lstat Cannot stat `//home/?????/.gvfs': Permission denied
Guess you'd better run fsck first !
exiting...

It made it so I could not install quota properly which meant ispconfig also wouldnt install. If I rebooted I was in a world of hurt. Answer was not easy to find either cause others in the forum just ignored it like it was silly or something. Thank goodness for Melask:

Just log off from your graphical environment and switch to e.g. tty1 window with the ctrl+alt+F1 keys. Run all the commands there (after u login ofc) and u are ok.

Switch back to kde/gnome with ctrl+alt+F7
