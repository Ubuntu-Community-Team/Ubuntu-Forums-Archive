### Post by CatKiller on 2022-01-13
The desktop files are just text files. The command that gets executed is specified by the Exec line.

If you want to know the path of an executable of a given name, if the full path isn't given in your desktop file, say, you can use **which** to show it.

### Post by GhX6GZMB on 2022-01-13
The command:
```
echo "$PATH"
```

Will tell you where to look for your executables (or symbolic links to them).

### Post by TheFu on 2022-01-13
To find any files on a Unix/Linux computer, use these answers:

Answer A: 
The '**find**' command:
[Https://www.howtogeek.com/771399/how-to-use-the-find-command-in-linux/](Https://www.howtogeek.com/771399/how-to-use-the-find-command-in-linux/)

Answer B:
The '**locate**' command:
[Https://ubuntuforums.org/showthread.php?t=2470757&p=14074744#post14074744](Https://ubuntuforums.org/showthread.php?t=2470757&p=14074744#post14074744)

Basic skills for any Unix user.

The term "shortcut" isn't normally used in Linux/Unix.  It isn't precise.  There are .desktop files which are similar to the old Windows 3.0 .pif files. Then there are symbolic link files and hard link files.  Those are part of the file system.  I think MSFT added support for them recently in NTFS, but I've never seen them used on Windows.  They are commonly used on all Unix-like OSes and have been for 40+ yrs.  Wikipedia has articles that explain *symbolic links* and *hard links*, if you'd like more details.

If a program is in the PATH, then we can search for the real location of is using '**which**'. For example:
```
$ which ls
/usr/bin/ls
```

### Post by schragge on 2022-01-14
This is how I find executables installed by a package
```
[COLOR=green]$[/COLOR] cat ~/bin/dlb[COLOR=green]
#!/bin/sh
# list executables installed by a package
dpkg-query -L $@ | xargs -r stat -c'%A%n' 2>/dev/null |
    sed -nr '/^-.{2,8}[xs]/s/^.{10}//p' | sort | column[/COLOR]
```
