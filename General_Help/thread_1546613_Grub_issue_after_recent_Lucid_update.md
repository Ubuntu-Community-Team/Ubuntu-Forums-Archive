### Post by oldfred on 2010-08-05
i do not really see anything wrong.

  47.3GB: boot/grub/core.img
 150.9GB: boot/grub/grub.cfg
  47.4GB: boot/initrd.img-2.6.32-21-generic

That grub.cfg which is in the same directory is so far different is a little strange but that may be normal as you have a large root partition. I have 20-25GB roots with separate /data or separate /home but that is because I have multiiple root partitions - previous install, current , and future/beta.

This should be the same as the update-grub but some claim it works better. In Karmic when I checked update-grub was just a link to this:

sudo grub-mkconfig -o /mnt/boot/grub.cfg

This also runs some of the same commands but it more like a new install:
to get grub to remember where to reinstall on updates:
sudo dpkg-reconfigure grub-pc
spacebar to choose drive, enter to accept, do not choose partitions

Is this what you ran to reinstall grub or just sudo update-grub
sudo grub-install /dev/sda
If that returns any errors run:
sudo grub-install --recheck /dev/sda
Then:
sudo update-grub
