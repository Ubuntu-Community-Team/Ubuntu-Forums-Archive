### Post by flabdablet on 2010-01-21
Personally, I'd put /etc/fstab back to the way it was originally, because mounting your NTFS partition during startup is exactly what that line is for.

I don't think this is actually an Ubuntu configuration problem at all. I've seen similar behaviour on a customer machine, and the cause turned out to be an unclean shutdown of XP. Ubuntu will refuse to automount an NTFS file system that hasn't been marked as cleanly shut down. As they say in so many Microsoft knowledge base articles, "this behavior is by design."

So if your computer is running XP and suffers a power failure, or somebody hits its Reset button and then boots straight into Ubuntu, your Windows partition will become inaccessible. The safest fix is to boot XP, shut it down properly, then reboot Ubuntu.

System Restore is all fun and games until somebody loses an eye...

Don't ever do a System Restore unless you absolutely have to. Things will break. Also, System Restore doesn't affect the integrity of the file system proper, which is the only thing Ubuntu cares about, so it will *not* resolve this issue. No way, no how.

You should be able to force your HP_PAVILION filesystem to reappear in your Places menu by simply [dragging it back into the shortcut panel in Thunar.]("http://ubuntuforums.org/showthread.php?t=957775") It may well be that it got removed from there via an inadvertent drag and drop in the first place.
