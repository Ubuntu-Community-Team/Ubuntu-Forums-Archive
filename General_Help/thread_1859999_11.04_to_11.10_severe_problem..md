### Post by effenberg0x0 on 2011-10-14
Guys, if you're stuck at the battery message, all it means is that the system couldn't run a Graphical Session. Generally that happens when you don't have VGA drivers installed, or they are not properly installed.

When you're at that screen, you can switch to another VT using Ctrl+Alt+F1 to F6, therefore you can install the proper drivers for your card, Once that is done, all you have to do is sudo service lightdm restart to start the GUI session and your setup is OK.

So, all you have to do now is check which VGA card you have (Intel, Nvidia, ATI) and install it.

Ubuntu documentation has detailed procedures on how to install drivers for this 3 platforms. The forums have them too. If you look at my posts on the (now archived) Oneiric testing forum, you'll find tutorials for nvidia and ATI (proprietary) drivers. In a few commands you can solve this issue.

Regards,
Effenberg

### Post by dgundling on 2011-10-14
As a possible temporary solution and assuming you have the disk space, why not partition your hard drive so that you can keep the current working version of Ubuntu on one partition and have another for use when updating to a newer version? If your data is on still another partition (total of 3 minimum.) you can continue to work with the older version until you can get the newer version to settle down. Then switch to the newer version and, when the next update version comes along, have it update your older version.
 
BTW, I had the same problem with 11.10. Black screen and can't do anything about it. Because I partitioned as above, I can use the 11.04 version while I wipe the partion with the 11.10 version and try again. My update with a wireless network connection took almost 24 hours.
