### Post by TheFu on 2019-07-12
Home directory encryption is deprecated.  There are both security problems AND really poor performance issues.
Rather than saying "it is slow", can you please gather some facts related to reading and writing to the disk outside and inside the encrypted HOME?  **bonnie++** is the tool I'd use.

I'd be curious about ext4 vs ext3 too.  [https://www.phoronix.com/scan.php?page=article&item=ubuntu_netbook_fs&num=1](https://www.phoronix.com/scan.php?page=article&item=ubuntu_netbook_fs&num=1) is from 2010, but shows ext4 to be much faster than ext3 in most of the benchmarks.

And ... cough ... emacs isn't helping. ;)

### Post by TheFu on 2019-07-12
LUKS isn't deprecated.  I'd assumed you were using the old encfs/encryptfs that was built-into Ubuntu.

LUKS performance should be within 3% of non-LUKS assuming the CPU has AES support.  This assumes you use AES. luksDump will confirm the cipher name and mode.

Any plans to run the 2 bonnie++ tests?
