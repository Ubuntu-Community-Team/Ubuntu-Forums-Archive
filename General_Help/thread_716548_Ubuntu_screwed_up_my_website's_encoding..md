### Post by Korea Beat on 2008-03-06
Two days ago I started using an old laptop which my brother in law had set up with Ubuntu 7.10. I run a Wordpress blog which is dedicated to translating Korean news articles so naturally I set things up so that I could type in Korean. With that done I logged in to the blog.

Then I found that every single post -- that is well over 600 -- and every single comment -- that is well, well over 1,000 -- has had its encoding destroyed. Anything which was written in Korean has been totally munged. I'm in the process of using an XP system to manually fix things but I simply don't have the time and a lot of it is just going to be deleted. It's not a disaster, exactly, but it is a gigantic waste of time and it makes Ubuntu as useful as a hole in the head for me. I could try to fix things but the only way to know if the fix worked would be log into the blog again and risk screwing up every post and comment all over again. There is no way I will do that. The encoding of other sites appears to work fine. Creating new posts in Ubuntu also works -- they show up fine in XP -- but I just can't risk messing up old posts.

So this is a big report I guess. I followed the instructions at this post which I found through Google.

[http://ubuntuforums.org/archive/index.php/t-618761.html](http://ubuntuforums.org/archive/index.php/t-618761.html)
