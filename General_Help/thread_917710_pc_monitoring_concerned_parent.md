### Post by mb_webguy on 2008-09-12
> **dsr4866 said:**
> these are mostly blocking and filtering systems which don't really work. I am looking for something so the kids know I have my eye on them. something that will list the sites they go to.
I think there's a saying...  something about an ounce of prevention being better than a pound of cure...  *shrug*

Personally, if I were trying to make a kid-friendly system, I'd...[LIST]
[*]Make a user account for the kids, and take it out of the admin group so that they can't install or uninstall software or make potentially disastrous changes to the system
[*]Install the [ProCon Latte]("https://addons.mozilla.org/en-US/firefox/addon/1803") extension for Firefox
[*]Install the [Menu Editor]("https://addons.mozilla.org/en-US/firefox/addon/710") extension for Firefox and use it to remove the Add-ons menu item so that they can't easily disable ProCon Latte, and the Preferences and Clear Private Data menu items so they can't delete their browsing history
[/LIST]

This setup isn't perfect, of course, since if they're moderately smart they can go into the .mozilla folder and delete the extension files, and then reinstall the extensions when they're done.  Of course, you'll be able to know if they've done this because either their browsing history will show the naughty sites, or if they deleted it, will be conspicuously short.  And of course you'll have to log in using your account if they want to install any software, which means you'll actually have to be somewhat involved in their computer usage.

But really, I think PrivateVoid has the right idea.  Participating in a child's activities is the best way for a parent to monitor the child's activities.

### Post by FluffyElmo on 2008-09-24
You can install a proxy server such as Squid. Then just configure Firefox to route all traffic through the proxy (these settings can be locked). You can also configure Ubuntu to use a proxy under Preferences->Network Proxy in case they go outside Firefox. As all traffic is going through the proxy you can just enable logging and run a log file reporting tool against the log.

A quick Google search turned up the following. The instructions also include filtering, but you can simply ignore that package and just follow the basic proxy set-up.
