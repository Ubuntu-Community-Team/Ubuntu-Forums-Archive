In a guest session, Shutdown will shutdown the guest session and not the machine. What is wrong with that? Why would I allow a guest to shutdown my working session with applications open and perhaps mess up the system by not doing an orderly shutdown?

In switch users, Reset switches from the present session session to a login screen to change users and Shutdown only shuts down the present user's session and not the original user's session. This is sensible in my opinion.

The alternative way of working is to close down all applications and log out and allow another user to log in. I have not tested if this allows the   new user to shutdown the machine. It may do.
