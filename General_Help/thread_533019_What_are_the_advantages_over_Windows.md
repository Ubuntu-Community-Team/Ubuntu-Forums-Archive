### Post by buntunub on 2007-08-23
What about.. How to list them all?

1) Linux requires you to learn about your hardware and your OS. This will be daunting at first, and require s you to ~use your brain?~ :shock:  Yes, I know, its quite different from Windows where your brain is never used at all but somehow your wallet keeps growing smaller. That is its biggest advantage imo. That it gives YOU the power and the control over everything at the price of having to learn things. 

2.) Linux is far more secure and comes with a full security suite out of the box (iptables). Nope, no virii here and no worries about worms either. :popcorn:

3.) Linux is modular, and therefor, does not require a system reboot each time you sneeze at it or install something. In fact, you should never really have to reboot ever, unless you upgrade the kernel (which is usually never for most). I am not 100% sure on the longest running linux machine, but its many years now without ever rebooting.

4.) All Linux distro's come preloaded with the GNU toolsets. This includes the compilers, and developer tools. Quite handy if your into programming in all the coolest of languages.

5.) ssh and vnc preconfiguration. If you know what these are, then enough said. All distro's (that im aware of) come preconfigured for this. In Ubuntu, you must load the server daemon if you want to be a host, however. Samba is also usually preconfigured or is quite easy to do. Either way, Linux=networking, and so it fits naturally into any network.

6.) Open Source/FOSS. This is seperate from Linux itself, but because right now GNU uses the Linux Kernel rather than HURD (and now probably always will), they are closely related. Since all Open Source software falls under the GPL (including the Linux kernel), it all abides by the rule of source code sharing and availability. The "million eyeballs" looking at the code, identifying issues, and fixing it immediately as well as improving upon it. The proof of that simple concept can be easily seen in the software, which, in most cases, is far superior to its closed source alternative. This is the philosophy that Microsoft and others ~should~ adopt, but sheer avarice and greed prevent. Apparently, the $60 billion in their bank account is just not enough!

There is more of course. Much more. But thats a good start.

### Post by tszanon on 2007-08-23
You're dealing with Free Open Source Software. Most of those guys that develop such software do it for free, in their spare time. They do it because they like to do it. They do the best they can to make sure that their code is clean, fast, reliable. Some even try to make their code as easy as possible to understand! :) They don't have any obligation to keep their projects running. When they don't want to continue, they stop, and the code is kept as good as before.

When you are obligated to code something you just don't want to, for any reason, the code you write can still be clean, fast and reliable. But the code you write will probably be something that is meant to work. You probably won't care if it's clean, fast, reliable, easy to understand, and all those things that you should care about when developing software. All you want to do is write some code that gets to the desired results.

And that's only about the development per se. Let's now think about testing the software. In closed source software, testing is made internally and, in some cases, there's some few outside testers too (in beta testing of games, for example). Then, when it seems they've tested enough, the software is released. And then comes Service Pack 1. :p

When it's FOSS, anyone can download the software and help to test it. When it's released - usually version 1.0 - it's very stable. Wine itself is very stable, and it's still 0.9 (in fact, if you pay attention, a lot of programs that you install are not 1.0 yet, and they work flawlessly).

I'm not saying that closed source software is not good. I'm just saying that FOSS has better chances of being high quality software than closed source. And since it also includes the OS kernel...

Just my 2 and a half cents.
