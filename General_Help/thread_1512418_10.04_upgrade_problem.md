### Post by oldfred on 2010-06-21
Wubi relies on windows. You only see grub to make it look the same as a normal install. By installing grub to the MBR you prevent windows from booting and you also installed grub2 to the windows partitions PBR or boot sector, which doubly prevents it from booting as windows has part of its boot in the partition boot sector.

You need to run this on sda1 and sda2, possibly on sda3 but it may only be data where the wubi install is. this should fix the PBR or partition boot sectors.
Fix for most, a few have other issues, better than windows fix in many cases as it also fixes other parameters:
This has instructions on using testdisk to repair the install of grub to the boot sector for windows from Ubuntu or Linux LiveCD.
[http://sourceforge.net/apps/mediawiki/bootinfoscript/index.php?title=Boot_Problems:Boot_Sector](http://sourceforge.net/apps/mediawiki/bootinfoscript/index.php?title=Boot_Problems:Boot_Sector)

If you have Ubuntu liveCD you can install a generic windows boot loader to the MBR.

Restore basic windows boot loader - universe enabled
sudo apt-get install lilo
sudo lilo -M /dev/sda mbr
May show error messages about the rest of lilo missing, ignore, we just want MBR

If you have a windows repair CD and above does not work you can run fixMBR to repair MBR and fixboot to repair boot sector. But because you have win7 with the hidden 100mb boot/repair partition and a separate working partiton, it often has trouble repairing itself. If nothing else works you have to do all the fixes to the main partition and in effect obsolete the boot partition.
