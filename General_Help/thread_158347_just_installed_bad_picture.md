### Post by greenstar on 2006-04-11
Sounds like you need to check your monitor settings, color depth & screen resolution.

Check System->Preferences->Screen Resolution & see what's there.

Sounds like you're looking at 256 colors, you need to raise that a couple of notches to at least 16-bit.

If you have an old monitor, you might have to dig up its sync & refresh rates (specs from manufacturer/manual) and enter these manually in /etc/X11/xorg.conf
You'll need to know the horizontal sync & vertical refresh rates as well as maximum resolution & maximum bit depth.

Example: The box I'm currently logged onto is an old K6-2 500 Mhz system with 384 MB RAM & Voodoo Banshee video card with an even older 15" Packard Bell monitor that maxes out at 16 bit @ 800x600 screen resolution. 

So, I looked on the back of the monitor, found the model number and Googled it.
After some searching & browsing, I had the monitor specs in hand. 

Then I opened a terminal and entered:
```
sudo gedit /etc/X11/xorg.conf
``` 

Be sure to save a backup of your xorg.conf file before editing so it will be easier to recover if this doesn't correct your problem or makes it worse.

**Refer to my xorg.conf below. The parts which are important here will be highlighted in [COLOR=Red]red[/COLOR]****.**

**Find Section "Screen"** and enter the appropriate bit-depth for DefaultDepth. I'd try 16-bit first, and go from there. Make sure the Device & Monitor fields match what you find in the 2 sections above "Screen". Set Identifier to [COLOR=Red][COLOR=Black]"Default Screen"[/COLOR][/COLOR]

**Then, in each SubSection "Display"** (below) remove all resolutions your monitor can't handle, as X will use the first value listed under the SubSection "Display" with the same color depth as is listed as DefaultDepth under Section "Screen".

So if DefaultDepth is set to 16, then whatever the 1st resolution listed under SubSection "Display" with Depth of 16 is the resolution that X will use when loading the desktop.

**If your monitor picture is looking like scribbles going back and forth across your screen, try this:**  (I'd leave this one alone unless you absolutely need to change it. You could wind up having to correct these values while in text mode.)
Find Section "Monitor" and enter the appropriate values for HorizSync & VertRefresh (If your monitor is old like mine, try my values for these.)

**My xorg.conf looks like this:**
```

### Post by Ramses de Norre on 2006-04-11
To find the values you'll need to use another computer or your windows install if you've got that. Once you've got those boot into ubuntu and when you're on the log in screen press ctrl+alt+F1 to get into a shell.
Do ```
cd /etc/X11/
sudo cp xorg.conf xorg.conf.bak
sudo nano xorg.conf
```
And applie the changes (look at the example given before).
Then save and do ```
sudo /etc/init.d/gdm restart
``` and press ctrl+alt+F7 if you don't get the login screen automaticaly.

OK, here's what you have to do.

1. Tell us the make & model of your monitor. You should be able to find this info on the back of the monitor.

2. Now we can Google that info to get the specs for your monitor. The manufacturer generally publishes that info on their website.

3. Then we plug those values into your xorg.conf just like I showed above. It's really not hard, don't let this stuff whip you. You can do this. :)

Using my HorizSync & VertRefresh values would probably be a bad idea unless your monitor is as old and small as mine.

Here's the xorg.conf from my main box. It's much newer hardware and the monitor doesn't need specific HorizSync & VertRefresh rates to be set, it's plug & play. I'll highlight the sections we're looking at with [COLOR=Red]red[/COLOR].

Note that there are more resolutions available for each bit depth in this xorg.conf. That is because the monitor on this machine can handle resolutions up to 1280x1024. This monitor's specs & capabilities are pretty standard for a fairly recent model 17" monitor . Also note, highest resolutions are listed first.
