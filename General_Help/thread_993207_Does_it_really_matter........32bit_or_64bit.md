### Post by kanikilu on 2008-11-25
The obvious benefit to a 64-bit OS is the ability to use 4GB (or more) of RAM. If you have less, there isn't really much of a reason (IMHO) to bother with it.

In fact, because there are some flakey (e.g. Flash -- although this is supposed to be getting better) or unavailable (e.g. Amazon MP3 downloader) software, I reverted back to 32-bit even though I have 4GB of RAM. Ubuntu is more than happy with 3.5GB of RAM ;-)

There are some workarounds and possible fixes or alternatives out there for the various problems you might encounter, but for me, the hassle wasn't worth the relatively little extra RAM I'd get out of it.

### Post by kanikilu on 2008-11-25
I think mostly yes, but I suppose it depends on what you consider to be a "main" application. Instead of guessing, I would suggest you make a list of software you can't live without. Then go search the packages page to see if a 64-bit version exists:

[http://packages.ubuntu.com/intrepid/](http://packages.ubuntu.com/intrepid/)

If the software is from another source, check out their site beforehand. For instance, I couldn't do my job without [VirtualBox]("http://www.virtualbox.org/wiki/Linux_Downloads"), so that would be a deal-breaker for me if they didn't have a 64-bit version (although they do).

### Post by teddks on 2008-11-25
Every free software program (read: nearly all things in Ubuntu) will work on 32 and 64 bit, because typically the code is portable, and there is very little porting required to go from 32 to 64 bit.

Using 64 bit will allow you to address more RAM, and can also give you a boost in security, because 64-bit systems use non-executable memory pages by default.
