### Post by aysiu on 2005-10-14
As I understand it, Ubuntu is all about consistent release schedules--every six months without fail. Maybe Debian's more to your taste? Their release schedule can be on the order of years, sometimes.

Personally, I like it. If I'm the kind of person who doesn't like to update stuff that often (except security, which is supported for 18 months after release), then I'll stick with my old stuff. If I want more cutting edge (not bleeding edge, necessarily), I'll keep up with the release cycle.

Bleeding edge, unfortunately, the backports don't even keep up with that well sometimes. For that (which is really the opposite of what your post addresses), I think Gentoo is the best bet, as I understand it.

### Post by chanders on 2005-10-14
Because a new version of Ubuntu is released every six months, doesn't mean that everyone running the previous version has to upgrade. Upgrade is a matter of choice and preference. 

If you think the new releases have some features you like, then by all means go ahead and upgrade. If you have a currently working system that you like, then you should keep it that way if you dont want the hassle of reinstalling.

Granted software development will continue, and we cannot expect developers to use 'old' packages and libraries to do their coding when newer and maybe easier methods, are only available with newer releases of the operating system.

The choice is ultimately yours......

(Fresh installs usually work the best ;) )
