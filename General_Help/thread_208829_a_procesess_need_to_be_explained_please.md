### Post by joft on 2006-07-04
Hi,

the users are system users, users with which you cannot login and which are just there to separate the system processes which run with the priviledges of these different users.

atd, I think is the "at" daemon. It's a kind of another cron - if cron means something to you. (Or "task scheduler" in m$ windowz).

dbus is a "software bus" system to distribute system messages from the kernel for example to various other system processes. This way an USB memory stick eventually can appear on your desktop automatically.
