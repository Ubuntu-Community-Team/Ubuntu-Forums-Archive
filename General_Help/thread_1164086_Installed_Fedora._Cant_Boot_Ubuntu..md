That's because Fedora uses ext4 by default, but grub doesn't support ext4 yet.  So you have to have a small /boot partition in ext3 to make it happy.  You also could have formatted / as ext3 rather than ext4, but ext4 is so much better that it's worth it.  Hopefully grub will be updated for ext4 soon.

 As for your problem, it should be perfectly possible for the two OSs to coexist.  Unfortunately, I don't know much about grub, but someone should be able to help you as many people here are quite good with that stuff.

### Post by sedawk on 2009-05-19
One solution is to add the "boot configurations" of Ubuntu's
/boot/grub/menu.lst to the ones of Fedora's /boot/grub/menu.lst
(or vice versa, depending on the which system was installed last).
Then when booting Fedora one of the options in the boot menu is
to start Ubuntu (or vice versa).

Keep in mind that you have to update this file (e.g. Fedora's
menu.lst) manually, if the latest update included newer kernels.

I think there is also a (better) way to include the second menu.lst into
the active one with some kind of "include" command (never used it).
I'm not sure how to do it, e.g. if you have to do some
cross-mounting first or if you can access menu.lst
even from unmounted partitions.

Another problem might be that a (shared?) partition from /etc/fstab
doesn't mount anymore. If I remember right installing Fedora
can change the UUID of a partition you want to use with
Fedora and Ubuntu. Because the UUID has changed Ubuntu cannot
mount it anymore and you have to update the UUID in /etc/fstab
or switch it to a non-UUID mount.

### Post by sedawk on 2009-05-20
File not found error?

Oops - I think I forgot something:
Fedora doesn't find the Ubuntu kernel and initrd.img at Fedora's /boot
Try to create a subdirectory /boot/ubuntu in Fedora's /boot directory.
Then copy Ubuntu's kernel(s) and initrd(s) to /boot/ubuntu/
Then update menu.lst and change the Ubuntu boot configuration
to look for the kernel and initrd at "/ubuntu/kernel..." and 
"/ubuntu/initrd..."

(You don't need that subdirectory, but if you copy Ubuntu's 
 files directly to /boot you might mess up stuff - depends
 on the naming conventions for kernel and initrd files ...)
