### Post by TheFu on 2014-02-22
I haven't used B-i-T in years.  I thought that link was more about showing the differences between 5 backup tools. With a visual impairment, avoiding GUI tools would be important, correct?  BackInTime has a GUI front-end, so that is how it is configured. It is not suitable for system-level backups unless you are fairly experienced with file permissions and ownership. Did I recommend BackinTime for your situation?

If you want to control where a partition is mounted, then you'll either manually mount it using the **mount** command or setup a mount using **/etc/fstab** to a pre-existing directory for both.  If the /backups directory does not already exist, both of those methods will fail. Also, the permissions on the mount location need to be changed to non-root if you want any other userid to have write access.

Sorry I'm terrible at teaching. I'm best used for ideas on which tools should be used, but not for a detailed how to use them instructions.
If you want a CLI backup tool and you are storing the backups on a local USB drive with a Linux-friendly file system, then either rsnapshot or rdiff-backup would be my recommendation.  I use rdiff-backup.  These are not perfect and 1 of my machines here is not liking rdiff-backup anymore. Nothing has changed on the client or server, so I'm at a loss.

Either of those tools can be installed from the normal repositories and there are examples of usage on their websites for simple needs like yours.  Run them automatically from the crontab.
