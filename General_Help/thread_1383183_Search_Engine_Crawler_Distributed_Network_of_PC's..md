### Post by unclesamslair on 2010-01-16
I'm no computer newbie (or Linux newbie for that matter), but I'm no programmer either. I only have an odd but somewhat specific question; is there an application out there (Widows, Linux etc.) that allows personal computers to share a search engine function over a distributed network with an optionally encrypted torrent-like function where each PC is also tasked with the duty of acting as servers and crawlers in this network? If not, then where can I go to suggest this as a project for some volunteer developers (I'm just starting to learn to program, and I understand that this might sound as a selfish request but it's not.) I feel that Google might have long ago fallen the way of the rest of the corporate search engines (Yahoo!, Microsoft, and Ask) towards mainstream censorship (especially in places like China) and all the smaller search engines are likely spam factories. I feel that this is both bad for the community and bad for the spirit of the free internet. Now, there's already a browser-plugin that runs on all platforms and allows you to view a censorship-free open source version of the WWW that you can pretty much surf the same way as the real WWW. This program, called FREENET, is also hosted through various encryption protocols on a darnet of personal computers on which the FREENET browser plug-in is installed. It also has it's own search engine. And due to it's distributed nature it cannot be easily taken down. Then why bother with the separate app for the same thing? One word people: performance. The FREENET project's network is so overtaxed with the load of hosting so many websites, it's search engine is slow and almost impossible to use. A search-engine-only application that returns search results of the real WWW along with accompanying URL's, on the other hand, could be more feasible as it's already done on most p2p networks and this would only require at the very least a text interface, making the bandwidth requirements more manageable. The search engine results though, are not provided by the user, but by open source algorithms written both for the crawler and server functions of this application. By making the application open source we also make sure no poisonous code is injected which would either serve to censor, give unfair search results ratings,  or infringe on user's privacy.
