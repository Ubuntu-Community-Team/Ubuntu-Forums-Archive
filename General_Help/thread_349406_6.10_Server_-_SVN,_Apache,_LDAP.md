### Post by Phykus on 2007-01-30
Hello,

I have an Ubuntu 6.10 server that I use as a SVN server, running the following services:
Apache/2.0.55 (Ubuntu) DAV/2 SVN/1.3.2 mod_ssl/2.0.55 OpenSSL/0.9.8b Server 

The authentication is done through LDAP with a Microsoft AD. The configs have the following form:
```
<Location /svn/>
  ...
  AuthLDAPURL ldap://host/dc=ex,dc=local?sAMAccountName??(&(objectClass=person)(!(objectClass=computer)))
  AuthLDAPBindDN cn=user,ou=users,dc=ex,dc=local
  AuthLDAPBindPassword pass
  AuthLDAPAuthoritative on
  ...
  Require group CN=Everybody,OU=users,dc=ex,dc=local
</Location>
```

The problem is that the authentication is very slow. If I try to access a SVN repository through Firefox, TortoiseSVN or Eclipse it takes about 2 minutes to access the data after entering the user name and password. This happens once per session but it's very annoying. Does anyone know what the problem might be?

PS. Using ldapsearch, the query took around 1 second to return the 200 or so results.

Thanks.
