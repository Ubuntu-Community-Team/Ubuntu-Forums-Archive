### Post by Vaphell on 2012-04-12
> if there would be performance issues considering wine is an emulator

Wine
Is
**Not** an
Emulator

it provides support for windows calls and when you run a program, it runs natively without virtualizing anything.

performance - it depends. DX->OpenGl conversion is a considerable overhead. WoW having OpenGl support right off the bat runs similarly to windows, Starcraft 2 having no OpenGl support in windows version (mac one has it) is approx -15 framerate difference from what i've read. Quality of gfx drivers is another issue, windows ones coming from nvidia and ati are finetuned, while linux ones are the same code + wrapper.
for current status of programs:
[http://appdb.winehq.org/](http://appdb.winehq.org/)

If you can't be bothered to solve issues with games in wine stay with windows for gaming purposes.

### Post by 3Miro on 2012-04-12
Dual boot is easy to set, but then you need to reboot your machine every time you want to play a game. I play Starcraft II, Dragon's Age (all 3 of them), Oblivion and Skyrim, Heroes of Might and Magiv V, Civilization IV and V, Stalker SOC (two of those are Steam games). All of those games, I play under Linux and I even made a huge Civilization IV mod under Linux.

Generally speaking, games are a hit and miss. Some games run just fine, some don't. Popular games and little bit older games usually run. Games usually take an effort to setup, but then it is so much easier than to reboot every time. 

Make sure to get a good Nvidia card, AMD and Intel are not good for wine games. There is a performance hit, but if your machine is good enough, it is hardly noticeable.
