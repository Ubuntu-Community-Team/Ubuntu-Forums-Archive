### Post by d99 on 2010-05-01
This seems to be a real and critical bug in ubuntu 10.04.  It affects anyone with  an intel 82852 or 855GM graphics chip

See these comments:
[http://linsux.org/forum/index.php?/topic/6874-critical-bug-in-ubuntu-10-04-release/](http://linsux.org/forum/index.php?/topic/6874-critical-bug-in-ubuntu-10-04-release/)
[https://bugs.launchpad.net/ubuntu/+source/xserver-xorg-video-intel/+bug/568779](https://bugs.launchpad.net/ubuntu/+source/xserver-xorg-video-intel/+bug/568779)


Here is a temporary solution which worked for me:
- Get into the grub menu by holding down escape when the computer is loading.  (I think you can also do this by holding down shift).
- Select recovery mode
- Choose "Run ubuntu in low graphics / failsafe mode"
- Hopefully this allows you to log in.
- If it did work, then go Applications -> Accessories -> Terminal, and type:
> cd /etc/X11
> sudo cp xorg.conf.failsafe xorg.conf

This should make your computer run with the same graphics settings as in failsafe mode.  You won't be able to change the display resolution etc.


Hopefully some someone smart will fix this bug soon.
