### Post by #&amp;thj^% on 2017-04-30
And just to add I've seen this also, and this worked for me.
To disable the screen blackout:

```
gsettings set org.gnome.desktop.session idle-delay <seconds> (0 to disable)

```
To disable the screen lock:

```
gsettings set org.gnome.desktop.screensaver lock-enabled false

```
There is also caffeine that handles this nicely. [https://www.devmanuals.net/install/ubuntu/ubuntu-17-04-lts-Zesty-Zapus/installing-caffeine-on-ubuntu17.04.html](https://www.devmanuals.net/install/ubuntu/ubuntu-17-04-lts-Zesty-Zapus/installing-caffeine-on-ubuntu17.04.html)

### Post by Dennis N on 2017-04-30
I would suspect the DPMS system if time to blank was 10 min., which was a problem a few years ago, when power manager failed to control this properly. Newer xfce4-power-manager fixed this problem and now correctly commands X to blank screen at specific time, or turn off blanking; but you have xfce4-power manager _not_ handling display power management, so what is? maybe xscreensaver?  xscreensaver when running used to prevent the screen blanking, so was a counter to X shutting down the display. Maybe not now? Lastly, X itself can blank the screen if not prevented by another agent.

There is command **xset** to directly control X preferences. Some suggest using it to shut down any blanking by X. See its man page for details.

(xfce4-power-manager handles things correctly on Lubuntu 17.04. I have it set to "handle display power managment".)
