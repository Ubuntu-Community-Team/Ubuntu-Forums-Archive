### Post by ajcham on 2008-12-24
The most common ways of installing software in Ubuntu.

[LIST=1]
[*]Installing from repositories: Using the package management tools such as Synaptic and and Apt you can install any of a massive number of apps with a single command or a few clicks - significantly easier than installing apps in Windows.
[*]Installing Deb packages: These also use the package managment tools, and it is usually a simple matter of double clicking on the package and letting it do its work - pretty much equivalent to an installation .exe in Windows.
[*]Installation scripts: Some programs are distributed as binaries which self-install by running an accompanying script - this too is similar to running an installation .exe, but may be more difficult to uninstall than programs installed through the package manager and you may have to track down some dependencies.
[*]Compiling from source: This is the more difficult method of installation. In it's simplest form it is a three-step process of Configure, Make and Make Install commands, but it will differ depending on the program (there should be a README). The process is usually painless, and goes without a hitch as long as the instructions are followed properly, but again you may need to find dependencies.
[/LIST]

In my experience, the vast majority of software is installable by either of the first two methods, and most others by the third method.  Method 4 is very rarely necessary - the only times I've *had* to use it are for very obscure or fledgling programs, but usually installing this way is a matter of choice rather than absolute necessity.
