### Post by landotter on 2006-01-12
> 
IMO:
KDE is better than Gnome anyday! I challenge anyone to run KDE for a week and see how much more (usable and better looking) it is than Gnome. Pretty sad Gnome is focusing on performance when it should be focusing on usability.

KDE's "opt-in" philosophy goes against the grain of "usability", and Gnome has been focusing on usability at the expense of performance. This is pretty much accepted dogma.

Can KDE be the right option for folks that need a certain level of fine grained configuration? Absolutely!

Can Gnome be overly dumbed down at times? Sure--but not horribly so.

I'd rather fire up a traditional two pane file manager for those times when Nautilus falls short, than deal with the fourteen headed beast which is Konqueror. It's impressive the same way that those really fancy Swiss army knives are impressive, lots of features, but not that easy to use. Some folks take to the integration, but some, like myself--prefer seperate file, web, and ftp programs.

If I was forced to run KDE, I'd choose SUSE over Ubuntu, simply because the system tools are better integrated with KDE/Yast. That said, Ubuntu package management is far superior to SUSE's--but, you CAN run Synaptic/apt under SUSE and Fedora. 

All DE have their limitations and personally I'm an opt-out kind of guy. That's why if I want a fat environment, gimme Gnome--but with only a single panel, or if I'm really feeling serious--it's Openbox with FBpanel.
