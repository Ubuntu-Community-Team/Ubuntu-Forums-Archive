### Post by AgenT on 2005-06-29
[QUOTE=ZephyrXero]oops....sorry, didn't see it.

Well...after reading the guide you linked to, maybe I put it in an appropriate place. One of the requirements for a backport is that it be a package currently in the official Ubuntu repositories....their just doing newer versions it looks like...[/QUOTE]

Good catch! But there have been programs backported that were not in any current Ubuntu repositories. As the guide says:

 >  When unsure, feel free to post the request anyway. We don't bite. 

If I were you, I would post your request in the backports forum anyway. You are much more likely to have the program in backports than in an official repository.

Also, if your program is in Debian (any branch), that would make the backports admins much more willing to backport your program because it is much easier for them to do so. Check [www.debian.org]("http://www.debian.org") for your package in the different branches (stable, unstable, testing). If available, be sure to mention it (with a link!).
