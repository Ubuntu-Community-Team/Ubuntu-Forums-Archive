### Post by swoody on 2008-11-28
I have a 64-bit machine as well, it also has 4GB RAM. I chose to run 64bit Ubuntu, but at this level of memory it is mostly a preferance. Anything over 4GBs I would reccomend 64 bit; anything less than 4GB - 32bit. If you run a lot of memory-heavy programs (editing audio/video, CAD programs or solid modelling, extensive gaming, etc) you would probably find it useful to use 64 bit. If you're not using demanding programs (internet, email, word processing, low-end games, etc) you'll probably be happier with the ease of finding compatible programs for 32-bit.

Edit: Good post on memory Vor! That is basically what I was trying to get across. That running 32 bit won't be a terrible loss if you're not really using and needing all 4GB, but if you need all your memory for heavy applications, your best bet is 64bit :D

### Post by sdennie on 2008-11-28
> **jespdj said:**
> The 64-bit version is not more buggy or less stable than the 32-bit version. Please do not spread FUD.

No, the 64-bit version is not inherently buggy or less stable but it is less widely adopted and, as such, for the desktop, probably less well tested (fewer users) or supported (fewer vendors (or even compilers) understanding how to write 64-bit things).  Having said that, the vast majority of users will have no problems with 64-bit or 32-bit.

I suspect that when Windows 7 comes out and all machines start shipping with 4GB of RAM to support it, 64-bit adoption will become more widespread.  ;)
