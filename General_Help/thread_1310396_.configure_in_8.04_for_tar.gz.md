### Post by falconindy on 2009-11-01
[http://packages.ubuntu.com/karmic/bibus](http://packages.ubuntu.com/karmic/bibus)

Bibus 1.4.3 is available in the repos. You're much better off installing it from there via apt-get or synaptic.

FYI, configure is a script that's very commonly included with source files in order to ensure that the user environment is sufficient to build said source (e.g. checking dependencies). It's also used to customize parameters for the build. That said, it's not **always** included, but it should be.

The article you linked is heavily generalized, but has the right idea that the core of any typical build is:
```
./configure
make
make install
```

### Post by Giblet5 on 2009-11-01
You're building source. It's not a "click here" kind of process. Yet.

Open a terminal window.

Change to the directory where the configure script is.

That's the "cd" command. Yes, it's case sensitive.

Then enter ```
./configure --prefix=/usr
```

Then open the README file and INSTALL file and read them. Follow the instructions.

### Post by geirha on 2009-11-01
./configure && make && make install is a commonly used way of building software, but it only works if that type of build system is provided with the source. In this case it obviously doesn't (since there's no configure script provided). And no matter what software it is, the best way to figure out how to build the software is to read the software's own documentation. They'll usually provide some text files like README and/or INSTALL along with the code, that describes what to do to build it.

A quick google on bibus shows that they have quite comprehensive documentation on how to build and install it on their website. [http://bibus-biblio.sourceforge.net/wiki/index.php/Installation](http://bibus-biblio.sourceforge.net/wiki/index.php/Installation)

Unless you really need to build the source yourself, I recommend you try their ubuntu repository first.
