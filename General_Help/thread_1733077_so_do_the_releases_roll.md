No, there is no feature updates between releases so if you use LTS you will be stuck with 2-3 year old softwares. "Bug fixes" basically just mean something security related. Otherwise you can have broken packages throughout the lifespan of a release. This is the worst thing about Ubuntu IMO.But you can circumvent that by installing from ppas tp keep your softwares up to date. I install most softwares I use with ppas instead of the official repos.

This looks like a good one for someone using 10.04.
[https://launchpad.net/~guido-iodice/+archive/guiodiclucid]("https://launchpad.net/%7Eguido-iodice/+archive/guiodiclucid")

I have a dual boot between Ubuntu10.04 and LMDE in one machine. I actually prefer Ubuntu+ppa to LMDE. It has all the good things that Ubuntu has to offer and more up to date software than LMDE (LMDE is based on Debian testing, whereas a fresh release of Ubuntu is based on a snapshot of Debian unstable, which is newer than testing)

### Post by TBABill on 2011-04-18
No, that's not it at all. You WILL get updates. Frequently. But they're usually minor version updates, not major moves (like from Firefox 3 to 4). You can still get those via PPA's (or backports??) along with bug and kernel fixes. It's not "stagnant", but neither is it rolling. You won't go from kernel version to kernel version using a single Ubuntu release, but you will get kernels with patches applied that may have been introduced in newer kernels.
