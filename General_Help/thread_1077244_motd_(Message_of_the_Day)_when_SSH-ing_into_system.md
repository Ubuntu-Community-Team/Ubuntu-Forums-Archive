### Post by dwhitney67 on 2009-02-22
I just recently upgraded one of my notebook PCs to using Ubuntu 8.10; when I ssh into this PC, the following message, which is contained in /etc/motd, is displayed.
```

Linux bluediamond 2.6.27-11-generic #1 SMP Thu Jan 29 19:24:39 UTC 2009 i686

  System information as of Sun Feb 22 07:30:01 EST 2009

  System load:    0.12               Swap usage:  0%     Users logged in: 1
  Usage of /home: 56.4% of 56.41GB   Temperature: 46 C
  Memory usage:   20%                Processes:   136

  Graph this data and manage this system at https://landscape.canonical.com/

```
Which application, or startup-script, is writing this message into /etc/motd?  I would like to disable it.

----------------------------------------------------------------------------------------------------------

UPDATE:

I believe I figured out how the /etc/motd file was being updated.  An entry was made into the cron-table using the information in /etc/cron.d/update-motd.  I edited and commented out the entry in this file, and also edited and removed the contents from /etc/motd.  This resolved the issue.

Now, the more important question is what would compel Canonical to insert nonsense like this into their OS?  If a particular individual needs to see their system stats, then have that individual insert their own script into the cron table (or whatever); but don't make it policy to provide all users the same.

### Post by oraerk on 2009-09-30
Please post contents of **/etc/motd** and** /etc/motd.tail** files.
The message of the day (motd) is regenerated by the cronjob from modt.tail file. If there is smth in ** /etc/motd.tail **which is pointing to this message, you can do the following to remove it (just an example):
```

cp /etc/motd.tail /etc/motd.tail.bak
echo "" | sudo tee /etc/motd.tail
sudo update-motd --force
```By the way, **man update-motd** and **man motd** is not a very bad idea at all.

### Post by rhysforge on 2010-06-04
Actually it is a lot easier to fix.      

/etc/motd   is actually a symbolic link to /var/run/motd   just blow away the symbolic link   

```
sudo  rm /etc/motd
```then 

```
sudo vi /etc/motd 
```
Then type what ever you want.      

if you want to keep your /etc/motd dynamic   then you will have to fiddle with the bash scripts under /etc/update-motd.d      

```
 sudo ls -l /etc/update-motd.d 
```
