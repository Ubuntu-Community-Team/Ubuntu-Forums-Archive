### Post by papibe on 2016-11-23
Hi Kris_M.

168.x.x.x is not your router. Although 192.168.x.x might as well be.

There's really not much value running namebench several times in a row, or on different OSes. First run should be enough. Subsequent runs would not reflect get the fastest reponses, because of all the caching the services did the first time.

Ubuntu uses a special plug-in to reroute DNS over itself. That's why you see 127.x.x.x, which is the machine itself (local loop). To see the actual DNS you system is using, run this:
```
nmcli -f IP4.DNS device show
```

Hope it helps. Let us know how it goes.
Regards.
