### Post by brentoboy on 2005-10-23
Ubuntu has only 1 CD to download, openSuse has 5 (and yes, you need them all, it isnt like debian where the other 12+ cds are optional)

When you add and remove packages, Ubuntu doesnt require cd swapping, because it allows you to go with repositories without needing the CD after the initial install. OpenSuse doesnt offer that option, and packages are scattered around on cds, so adding a single program with its depenencies will likely require 3 or 4 cd swaps.

package resolution: ubuntu, as you select a package, it tells you about dependecies that it needs, and just selects them on the fly. OpenSuse's package manager waits until you are ready to apply the new packages, and then it presents you with a rather criptic dependecy resolution dialog with entries for all the packages that need supporting packages.  Instead of the "ok" "cancel" button, you have to read and select between two choices (same end function, more effort).

OpenSuse does way better with samba - you can enable the samba services without ever touching the config file by hand, Ubuntu looks like it will, but doesnt deliver.

Both run recent kernels, so they have both have pretty good hardware support.

Suse's installer is graphical, but not any easier.

OpenSuse only includes open source packages. Ubuntu has free and "restricted" packages in its repositories.  So, there is stuff that isnt available in opensuse that is avaiable in ubuntu. That is the main difference between Opensuse and regular suse.  If this is a "bad" thing for you, you can get the suse eval of the full version instead of opensuse, I hear it doenst ever expire -  but never tried.

Suse has a more lively color scheme - mostly green.
