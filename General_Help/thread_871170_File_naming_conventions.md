Forum,
Can someone explain to me the file naming convention in Linux/Unix? I have read in the past that file names should contain no blank spaces. Are there any special characters which can be used, or should not be used? Just a little lost on (1) naming convention I see with day-to-day Linux files, and (2) OSI distributions and files relating to OSI distributions.

### Post by ajgreeny on 2008-07-26
Linux can deal with spaces in file or folder names but not as well as windows (one of the few cases where windows in best).  Linux file names are best with underscores _, dash or hyphen -, in place of spaces.  If you have file names with spaces and you need to navigate to them, no problem in nautilus, it's just like windows explorer, but in the terminal you will find it easiest to put the whole path in "path/to file" style.  You can do it with extra slashes in the name in place of a space, but I find that more difficult to remember so always use the quotes marks.

When you save files use the underscore by default, renaming attachments others send you if necessary, and you will find it much easier to use the terminal when you need to.
