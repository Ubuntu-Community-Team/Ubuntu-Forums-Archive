### Post by audiomick on 2010-02-05
Hi.
If Flash is your biggest issue, look here
[http://ubuntuforums.org/showthread.php?t=1358591](http://ubuntuforums.org/showthread.php?t=1358591)

I can't give you a definitive answer. I have been using 64 bit for a couple of years now, and haven't actually noticed any problems that I could say were directly attributable to the system being 64bit.

I have seen a fair bit of discussion on the subject, and general opinion seems to be, if you can run 64bit, you should. Reasons for not doing so seem to include things like memory usage or what have you, and less that anything doesn't work.

For me, at least, the hassle of re-installing to change would be more than it is worth.

If you really want to know, make a new partition and put a 32bit install on it. Then you can compare.

### Post by crazy_fox on 2010-02-05
A 64bit installation will future proof your installation. There is services and user applications already benefiting from it. I personally prefer my operating system to be able to use any hardware features where it can.

The only possible downside is that 64 bit binaries tend to be larger, therefore stress your memory more (including hdd).

### Post by Yellow Pasque on 2010-02-05
64-bit can provide significant gains in some areas, especially on Ubuntu/Debian, where 32-bit userland isn't i686-optimized (i.e no SSE/2).

Run 64-bit and if you need a 32-bit environment for a few troublesome apps, play with VirtualBox, or a chroot. Heck, you can even run a 32-bit Firefox and Flash pretty easily if you want to.
