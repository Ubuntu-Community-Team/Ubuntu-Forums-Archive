### Post by bodhi.zazen on 2011-04-03
Linux is not a drop in replacement for windows and using wine can become complicated fast.

If you have a problem running an application in wine, you should look to the wine web site for advice:

[http://appdb.winehq.org/](http://appdb.winehq.org/)

Configuring wine often requires intimate knowledge of BOTH windows and Linux.

Alternates are:

1. Dual boot to play games - probably most popular.

2. Use virtualbox - may be limited , especially with games and graphics.

3. Give up windows games and play Linux games - may not be viable.

### Post by 3Miro on 2011-04-03
Wine needs to be setup as game per game basis. Some games work great, some work OK, some don't work at all. Check with winehq for specific information about the specific game.

If you installed wine, it will automatically pickup any 32-bit drivers that it may need.
