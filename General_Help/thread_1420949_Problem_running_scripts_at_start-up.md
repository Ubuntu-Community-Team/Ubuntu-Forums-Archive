### Post by gsmanners on 2010-03-03
I can't help you with your wireless problem, but to have a drive mounted at boot only takes a simple modification of your /etc/fstab:

For example:

```
UUID=A341244AD8276C3B /mnt/win2 ntfs     defaults,nls=utf8,umask=007,gid=46       0       0
```

You do not need to specify "ntfs-3g" as the format in Karmic. To find the UUID of your drive do the following:

ls -l /dev/disk/by-uuid/

Then see which UUID -> to your device /dev/sda7.

### Post by johnmic on 2010-04-14
Finally found the solution! Turns out I wasn't meant to be using bcmwl-kernel-source after all, and by using it, it was automatically blacklisting the module I needed. The solution was as follows:
```

sudo aptitude update
sudo aptitude remove bcmwl-kernel-source
sudo rm /etc/modprobe.d/blacklist-bcm43.conf
sudo aptitude install b43-fwcutter

```
Don't think you need the penultimate command - I didn't - but I put it in there for completeness' sake!
