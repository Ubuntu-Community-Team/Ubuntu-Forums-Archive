### Post by mcgeek on 2010-01-31
I was going through my old posts and found this one. I did manage to find a solution so I thought I would post it here in the hope it will help others.

I moved to a tty by pressing Ctrl+Alt+F1 and logged into the system with my regular log on.

I stopped Gnome with:

```
sudo /etc/init.d/gdm stop
```

then I ran:

```
sudo X -configure
```

I copied the new xorg.conf file over the previous one in /etc/X11/xorg.conf.

I restarted gnome with:

```
sudo /etc/init.d/gdm start
```

Then I started nvidia settings:

```
sudo nvidia-settings
```

I then set up the proper resolution and refresh rate and saved them to the current xorg.conf file by clicking "Save the X configuration file".

The important point is to run nvidia-settings as root, otherwise you will not be able to modify the xorg.conf file. 

Below is my new xorg.conf file:

```

Section "ServerLayout"
    Identifier     "X.org Configured"
    Screen      0  "Screen0" 0 0
    InputDevice    "Mouse0" "CorePointer"
    InputDevice    "Keyboard0" "CoreKeyboard"
EndSection

Section "Files"
    ModulePath      "/usr/lib/xorg/modules"
    FontPath        "/usr/share/fonts/X11/misc"
    FontPath        "/usr/share/fonts/X11/cyrillic"
    FontPath        "/usr/share/fonts/X11/100dpi/:unscaled"
    FontPath        "/usr/share/fonts/X11/75dpi/:unscaled"
    FontPath        "/usr/share/fonts/X11/Type1"
    FontPath        "/usr/share/fonts/X11/100dpi"
    FontPath        "/usr/share/fonts/X11/75dpi"
    FontPath        "/var/lib/defoma/x-ttcidfont-conf.d/dirs/TrueType"
    FontPath        "built-ins"

Section "Module"

        Depth       24

```

Everything is working great.

### Post by mcgeek on 2010-05-15
Lucid Lynx update.

Last night I upgraded this machine to 10.04 by way of distribution upgrade in update manager.

Guess what...resolution got messed up again.

I upgraded without a problem and the monitor sync was fine but XBMC wouldn't load due to an error. Even though LL showed the Nvidia driver was loaded XBMC wouldn't load because it couldn't detect 3D acceleration.

Solution.

I removed the Nvidia driver and reloaed through Hardware Drivers. Restarted. No video--same old problem. I logged in through remote desktop and launched Nvidia Settings and set my resolution and saved them to the X configuration file. As before you will need to start Nvidia Setting at the terminal with sudo so you can overwrite the old X config file. 

As a side note I did have to reactivate the repo for XBMC and update before I could use it.
