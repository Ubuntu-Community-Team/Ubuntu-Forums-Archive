### Post by btindie on 2015-08-18
Have you tried googling that error message that's displayed?? 'chromium ssl_error_weak_server_ephemeral_dh_key'

Read [https://support.mozilla.org/en-US/questions/1067995](https://support.mozilla.org/en-US/questions/1067995)

And [https://productforums.google.com/forum/#!topic/chrome/o3vZD-Mg2Ic](https://productforums.google.com/forum/#!topic/chrome/o3vZD-Mg2Ic)

> 
[FONT=Roboto]If a secure website gets the error ERR_SSL_WEAK_SERVER_EPHEMERAL_DH_KEY, it means the website is trying to set up a secure connection, but it is actually insecure because the SSL/TLS uses a Diffie-Hellman group size smaller than 1024-bit.[/FONT]

[FONT=Roboto]This is the problem in the Logjam vulnerability, which affects both _browsers_ and _servers_:   [https://weakdh.org]("https://weakdh.org/") [/FONT]
[FONT=Roboto]
In this case, the website/webserver needs to be fixed.  Google Chrome won't use insecure connections in order to protect your privacy.
[/FONT]
[FONT=Roboto]Please send a support request to the website owner/admin to tell them to fix the TLS security on their website and give them this info:[/FONT]
[FONT=Roboto][https://www.chromium.org/administrators/err_ssl_weak_server_ephemeral_dh_key](https://www.chromium.org/administrators/err_ssl_weak_server_ephemeral_dh_key)[/FONT]
