### Post by Tadaen_Sylvermane on 2014-08-29
Backup your data and reinstall. It can be fixed but... wow what a mess. You can try reinstalling every one of those packages manually I suppose but I wouldn't bother, something will inevitably not work right.

*EDIT* For future reference. Don't play with the command line unless you read what its going to do or have the experience to know, even then you should read what it's doing before hitting yes. If you are learning / testing using the command line do it in virtualbox so this doesn't happen lol.

Another thing. If they would overlap then it would have replaced python 2.7 on it's own. You shouldn't have had to remove it manually. Let the package manager do what it does best. Don't go randomly removing things, especially programming language packages.

### Post by sisco311 on 2014-08-29
You don't have to reinstall every package manually. Reinstalling the *ubuntu-desktop *meta-package will install  all of the default packages in the Ubuntu desktop system. After that you will have to reinstall all the extra packages you installed, like virtualbox, steam, playonlinux, ubuntu-tweak, the printer driver...
