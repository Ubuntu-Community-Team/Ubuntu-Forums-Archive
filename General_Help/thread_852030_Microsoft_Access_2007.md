### Post by cyblance on 2011-12-05
I'm replying here because I asked the same thing a few months ago. 

Microsoft Access is a great intro to database building but it is just that - some kind of expensive tutorial.
Split your database into a backend (Where the Tables are) and a Front End (where the Forms Queries and Reports sit.)

MySql is fantastic as a backend (there are others) and Base limps along nicely as a front end (I started with Knoda).

Soon you will begin using queries and creating new forms and realise that most of what you are looking for lies within this talent.

Not saying its easy, but it is a solution.
