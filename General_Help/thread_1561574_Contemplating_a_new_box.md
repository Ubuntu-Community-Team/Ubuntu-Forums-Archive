My machine is starting to show its age.  It is an NVIDIA-based HP box with an AMD 2400+ processor and a GeForce2 mobo base.  32-bit, of course.

I have looked through the (now closed) 64-bit discussions and I gather that this is now a non-issue and that most current machines are, in fact, 64-bit.  I find this commendable from an operating systems point of view, but wonder, since the program I run most of the time is a 32-bit Windows executable that I run with wine, if I should go to a 64-bit architecture.

I can see from the bandwidth alone that there is a serious advantage to 64-bit architecture if it truly is.  The hardware implications are what one would expect.  Better page tables, bigger virtual addresses, and so on, and of course wider registers so bigger integers, and better floating point.  The question that lingers in my mind is "Is the software work completed with respect to the use of all this nice hardware?".

If I go 64-bit, what of all the 32-bit stuff I have?  Does the software recognize a 32-bit program text, and act accordingly?  Some of the programs I have were written in the stone age (around 2000), and there is no help for them but to run as is, with a little paging boost, and that's all you can really do.  One of the programs is a game that uses DirectX 9.0c, manfully handled by wine and enwrangled into openGL.  Will this still be true?  I know wine has a 64-bit version and I guess I should really ask them about that aspect.

Any comments on this from anyone will be appreciated.

And one more thought.  One of the last things I worked with was an AIX box that had 128-bit processors.  How far are we from the 128-bit PC?
