### Post by Jive Turkey on 2010-04-30
I don't know if this will help but here is the relevant portion of my /boot/grub/grub.cfg file that pertains to booting windows7:
```
### BEGIN /etc/grub.d/30_os-prober ###
menuentry "Windows 7 (loader) (on /dev/sda1)" {
	insmod ntfs
	set root='(hd0,1)'
	search --no-floppy --fs-uuid --set 528068c08068abe3
	chainloader +1
}
### END /etc/grub.d/30_os-prober ###
```

You don't edit this file yourself it is written when you run the command ```
sudo update-grub
``` or ```
grub-mkconfig -o /boot/grub/grub.cfg
```

You could try running one of those commands in your ubuntu install to see if it does a better job of detecting your windows OS than the installer did.  It sounds like it is currently mapped to your OEM restore partition and not you windows partition.

### Post by 98cwitr on 2010-04-30
you should be able to use the Windows OS disk to run a fixboot from the recovery console. That will give you access to your Windows OS (hopefully). Then you will need to run a partition editor to blow away your Ubuntu install, CORRECTLY repartition the drive (NTFS and then a RAW for the unallocated space). Then, you should install Ubuntu on the *separate* configured partition:

Outline:

1. Run fixboot from Windows OS Recovery Console. This will rewrite the MBR

2. Run Partition Magic or other partition editor to erase your Ubuntu install, then create a new partition that you want to install Ubuntu on.

3. Reinstall Ubuntu on separate partition

Lesson of the story would be not to use the LiveCD to setup a new partition b/c grub2 really doesn't like it.
