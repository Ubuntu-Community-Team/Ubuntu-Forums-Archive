### Post by 3rdalbum on 2010-11-20
If your CPU supports virtualisation extensions, then you can use a 64-bit guest operating system. So if your CPU doesn't have virtualisation extensions, you'll have to run Ubuntu 32-bit as the guest and Windows 64-bit as the host (because you won't be able to use your Windows 64-bit as the guest).

Which operating system will need access to the hardware directly? Have you got a specific peripheral that only works with Windows and would require Windows to be the host? Or would you be running 3D programs in Ubuntu and not in Windows, that would necessitate Ubuntu being the host OS?

The "Ubuntu download that installs Ubuntu alongside Windows" is just another method of dual-booting, NOT virtualisation, so it's not what you say you want.
