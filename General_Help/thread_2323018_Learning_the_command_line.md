### Post by montag dp on 2016-05-02
In addition to the resources people have already mentioned, the best advice I can give is to just practice.  Force yourself to navigate the file system and do basic tasks with the command line.  For that, all you need are a few commands, like cd, mv, cp, rm, and ls (and don't overlook tab completion!).  After that, you will start to learn how powerful it can be with commands like grep and find and special characters like | and *, and with additional options to the commands you already know.  And then you can start playing with scripts for automation...

The terminal may be scary at first, but it is extremely powerful and well worth learning.

### Post by him610 on 2016-05-03
FWIW, there are a lot of commands one can run from the CLI. You are probably saying to yourself, "Where do I begin?"
Begin in small increments. Try to learn five new commands each day. For instance, the first day try these five...
```
pwd
ls
ifconfig
free
df
```
You might want to first try *man command* to get an idea how to use the options and what to expect.
The next day, repeat those five commands plus five new ones.

You might consider checking this website...
[http://linuxcommand.org/]("http://linuxcommand.org/")

And as Vasa1 said, "It would help you familiarize yourself with this forum first"
You can learn a lot when you use the diagnostic commands the responders recommend in the different problem solving threads.
