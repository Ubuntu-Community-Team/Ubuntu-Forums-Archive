### Post by Mud.Knee.Havoc on 2007-09-03
The problem is most likely that the bin/cue combo was created in Windows, where it's not case-sensitive.  

You can see in his error message that his cue file contains:

/home/flyingroof/Desktop/KAL-DW7A.BIN

and I'd be willing to bet money that his file is named kal-dw7a.bin, which unix/linux sees as completely different than KAL-DW7A.BIN.

All you need to do is open the .cue file and put in the correct (in terms of directory and upper/lower case) file's location.  Go for it, open the .cue up in a text editor... it's not scary at all. :D
