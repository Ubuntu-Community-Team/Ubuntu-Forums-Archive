### Post by phidia on 2008-09-16
From the [sane link]("http://www.sane-project.org/man/sane-plustek.5.html") I previously provided:
> CONFIGURATION

       To use your scanner with this backend, you need at least two entries in
       the configuration file /etc/sane.d/plustek.conf
              [usb] vendor-id product-id
              device /dev/usbscanner

       [usb]  tells the backend, that the following devicename (here /dev/usb-
       scanner) has to be interpreted as USB scanner device.  If  vendor-  and
       product-id  has not been specified, the backend tries to detect this by
       its own. If device is set to auto then  the  next  matching  device  is
       used.
       The  following  options can be used for a default setup of your device.
       Most of them are also available through the frontend.

       The Options:

       option warmup t
              t specifies the warmup period in seconds,  if  set  to  -1,  the
              automatic warmup function will be used

       option lampOff t
              t  is the time in seconds for switching off the lamps in standby
              mode

       option lOffonEnd b
              b specifies the behaviour when closing the backend, 1 --> switch
              lamps off, 0 --> do not change lamp status

You may need to install sane-extras and look at your file system to see if the above referenced file exists. Use sudo gedit or nano (from a terminal) to edit the file.

### Post by Robin T Cox on 2008-09-17
I have a CanoScan LiDE25 that now works fine with my set-up. Though I have also had similar problems. I have sane and sane-utils installed, and assume you do too. This scanner doesn't need any extra software, it works out of the box, but there is a bug in Ubuntu that seems to interfere with the scanner permissions settings, so that the scanner doesn't get 'seen' by the system if the user is other than root.

Can you get your scanner to work if you enter (Alt/F2) the command:
```
sudo xsane
```

If so, it will warn you of the security risk. But go ahead this once, and see if xsane works.

To avoid getting this message in future, and to get the scanner to work properly, you need to include your username in the 'scanner' group in your permissions settings. After you reboot, you should find that xsane will work OK.
