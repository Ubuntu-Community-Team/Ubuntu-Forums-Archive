### Post by 2rB on 2009-11-03
We are making a information display system for use in this building only, and will probably end up with 2 or 3 screens.
Stability is not the most important requirement for us as there always will be someone around the screens to let us know if one of them stops.


I think we will end up with [Xibo]("http://xibo.org.uk/") running windows at the front end and probably a virtual version of Linux at the back end.
Been testing it for a few hours now and it seem to offer the features we want.

It does not seem to be scalable (resolution) at the front end, making some problems with making web-pages readable. However this probably can be solved by adjusting the desktop resolution on the client, or flip the screen so that we have a height-format with 1080 pixels width.
