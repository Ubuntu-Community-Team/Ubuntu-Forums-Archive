Hi

I wish to thank LovingLinux and his posting of the Firefox optimization and troubleshooting thread. The result, that I achieved, was not as hoped but the information gained, was very useful.

The link to speednet [http://www.speedtest.net/]("http://www.speedtest.net/") was useful in comparing the connection speed between 2 PC's on a network one running windows another running Linux.Was slower on the Linux system.

The link to Peacekeeper [service.futuremark.com/peacekeeper]("service.futuremark.com/peacekeeper") was equally used to compare the browers on the same two systems. Same Browser but different OS.

As I said the results were not, as, I would have liked but confirmed a problem. The windows PC with a processor, half the speed of the Linux PC gave a rating of 6 times  faster. Some of the problem seemed to be related to IPV6. By using the information, in one of the links supplied, I was able to some increase the response time on the linux by disabling IPV6.

Unfortunately, it still does not match the Windows speed. What I am seeing is the CPU usage going to 100% for a couple of minutes. Currently I am not sure what the problem is, so any suggestions would be gratefully received. I wondered if it was the video driver, I noticed that moving the System Monitor (System ==> Administration ==> System Monitor) window caused the CPU load to go to 100% while moving the window, but quickly returned to a lower value. Equally I have noticed that sometimes the forum pages load quickly and other times  have this long wait with High CPU load, Is this a slow DNS link or wrongly setup network? I have recently changed the main router to wireless/4 wired port router from a 4port wired router.

If I am not able to solve problem easily, I will buy a used more modern system running XP Pro and will convert to (X)Ubuntu, in case, the problem is related to hardware.

So Thanks again to LovingLinux, a very useful posting.
