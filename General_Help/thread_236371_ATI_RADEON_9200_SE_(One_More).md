### Post by virtual_void on 2006-08-17
Your ServerLayout is using "aticonfig-Screen[0]" as Screen which uses 
"aticonfig-Device[0]" as Device (not "ATI Technologies, Inc. RV280 [Radeon 9200 SE]"). So test to change the Driver of the other Device (the one that currently uses the "fglrx" driver).

The "ati" driver might work, but you should really use the "radeon" driver or the "fglrx" (if it works on the Radon 9200, I'm not sure).

I had the same problem when installing Kubuntu 6.06, the driver used for my Radeon 9800 Pro was the "ati" driver which caused system lockup when X started. Changing the driver to "radeon" made it possible to start X, but it was not very stable. I'm now using the "fglrx" and haven't seen any lookups so far.
