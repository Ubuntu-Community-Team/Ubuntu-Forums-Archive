### Post by oldfred on 2017-06-09
If creating an installer, it can be gpt or MBR. A few systems like one or the other but most just work.
Partition must be FAT32 formatted with Boot flag. 
Normal installers erase drive, repartition drive, format a FAT32 partition with boot flag, extract ISO and install a BIOS boot loader.

Many installers use the hybrid capability of the ISO can just dd the image to a flash drive. Then it is neither MBR nor gpt, but a hybrid of DVD and flash drive, but bootable.
This uses dd under the hood, but offers some more protection from errors than just using dd.
 [https://help.ubuntu.com/community/mkusb](https://help.ubuntu.com/community/mkusb) 


       If UEFI only, then can just extract ISO with 7-zip to FAT32 formatted drive with boot flag on FAT32 partition.
Will not boot in BIOS mode as syslinux is not installed.
UEFI only USB key, just extract ISO ( 7 zip or similar) to FAT32 formatted flash & set boot flag.
[http://askubuntu.com/questions/395879/how-to-create-uefi-only-bootable-usb-live-media](http://askubuntu.com/questions/395879/how-to-create-uefi-only-bootable-usb-live-media)
UEFI only bootable flash drive
[http://ubuntuforums.org/showthread.php?t=2299040](http://ubuntuforums.org/showthread.php?t=2299040)

Bit more complicated, you can directly install grub to flash drive and direct boot an ISO with loopmount commands in grub.
[https://help.ubuntu.com/community/Grub2/ISOBoot](https://help.ubuntu.com/community/Grub2/ISOBoot) 
Then you can have more than one installer on on somewhat larger flash drive.
