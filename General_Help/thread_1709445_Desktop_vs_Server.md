### Post by seawolf167 on 2011-03-18
I have not setup the Google package/project you [m]("http://ubuntuforums.org/showthread.php?t=801404")ention, but essentially the main difference between the server & desktop versions of Ubuntu is that the server version does not have a GUI, its is command line only.

Other minor differences include different installed packages/applications, etc., and you can certainly install a GUI on the server version, but servers generally don't have GUIs because you don't want it eating up your CPU.

For a first time user, I'd probably suggest getting started with the desktop version as you may find it easier.

[Here ]("http://ubuntuforums.org/showthread.php?t=801404")is a good starting place for basic/general questions.

### Post by AllGamer on 2011-04-26
***@raycoulter***

Just think of it this way... 

Using the windows terms, Ubuntu Server = clean lean version of Linux, and Ubuntu Desktop = your typical broaded windows machine ;) :p

The above statement is part joke, part true.

If you want to experience the full capability of Ubuntu, then by all means install Ubuntu Desktop Edition.

but if you are looking for a clean mean fast machine, install Ubuntu Server Edition; which you can easily install the additional items to make it an Ubuntu Desktop, without all the extras that comes with Ubuntu Desktop default install from CD.

So, once you have played around a bit with Ubuntu Desktop Live CD, and you know what you want or don't want installed.

Then you can do a Server install, and systematically choose only the Desktop packages that you want to actually use.

running ```
sudo apt-get install "package name"
``` you will easily get any of the apps that you really want in your server/desktop ;)
