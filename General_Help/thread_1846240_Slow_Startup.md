### Post by aprochaska on 2011-09-18
Ubuntu startup is slow compared to windows.  I have a dual-boot configuration with ubuntu 10.04 and windows XP.  XP starts up quickly (0:32), ubuntu does not (2:43).  I found a thread that recommended changing file systems in /etc/fstab from uuid=XXX to /dev/XXX.  I did that, but no change in startup time.  

Another note in the same thread instructed to change references in /boot/grub.menu.lst the same way.  I don't find that file in my system, and I think that's because I'm on grub2 now instead of grub (legacy).

In /boot/grub/grub.cfg, I notice that the ubuntu partitions are specified uuid=XXX, but the  windows partitions are specified /dev/XXX.  Also at the top of grub.cfg is "DO NOT EDIT THIS FILE.  It is automatically generated by /usr/sbin/grub-mkconfig using templates from /etc/grub.d and settings from /etc/default/grub"

Ok, so I'm trapsing through the files in /boot/grub/*.lst and /etc/defaults/grub trying to figure out what to change (legally) so grub.cfg will find the ubuntu partitions by /dev/XXX instead of uuid.

Suggestions?  Several data items are included below.
Thanks for any help.

Alan Prochaska
