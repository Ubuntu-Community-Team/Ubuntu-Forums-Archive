Welcome to the forums.

Used a WD TV Live HD box about 5 yrs ago.  It was a CIFS server, so chmod doesn't work on that storage.  CIFS doesn't have permissions that chmod can touch.  Generally, permissions are applied only though the mount options.  It didn't have any security ... everyone/anyone could see and modify the storage connected to it.  I never knew how to stream from a computer through the WD box to the monitor/TV.  "Play to" from Windows sorta worked, but only with mpeg2 content.

No photos (or links to photos [better]) came through.

Perhaps if you listed the exact, correct, WD thingy folks with could help?

Linux has multiple ways to "mount" storage.  There is the old-school, "mount", way and there is the easier, gnome, GUI, way that isn't really a mount. That method uses something called gvfs and puts the remote storage under /media/{userid}/{label} ... it doesn't show up when you type "mount" inside a terminal either. Plus gvfs performance sorta ...er ... sucks when compared to other mount methods.

Samba is used to service other systems with storage. Usually those other system are running Windows, since Unix/Linux/OSX are Unix-based, NFS would be the preferred, faster, better integrated, way to share storage on a LAN. 90% of Unix people would agree with that.  Samba provides CIFS compatibility - this is what Windows shared storage uses.

IMHO.
