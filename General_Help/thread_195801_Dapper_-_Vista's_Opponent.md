Dapper doesn't require 50,000,000 lines of proprietary code to work effectively for most *common* PC apps.

From what I can tell, there are some primary deal-breaker barriers for Linux OS in gaining more widespread acceptance and use by the public. Win has most of us GUI-dependent in the extreme, while all Linux OS depend to some extent on some command line knowledge. Same issue for simple mouse-click / .exe apps installations, pull-down menus, etc.  Hardware and peripheral installations in Win OS run mostly on auto-pilot, while most Linux OS often require serious time and harsh-language investment to get even a wireless card working.  And most of the PC-buying public is production-focused, and not motivated - even by price - to learn command line protocols.

The partnering of M$ SuSE makes this mess more complicated with Gates co-opting (in his mind at least) Linux code into the M$ proprietary code pig pen.  If I recall correctly - in the aftermath of the M$-SuSE hook-up - Gates went so far as to say that anybody using Linux code *might* owe money to M$.

If the latter is factually true - or if Gates can force the issue on legal turf to his advantage - it would make the whole subject kinda academic.  In other words, if that turns out to be true by whatever means, Gates will have achieved his objective - a genuinely global monopoly on the bulk of the world's OS and OS-dependent hardware and peripheral products.

If any of that turns out to be true, Ubuntu's competitive business model will circle the drain like most other Linux OS.  

If none of that turns out to be true, then Ubuntu's Linux-niche market might continue to grow, but Ubuntu's business model is still a frail weapon with which to fend off M$ when it comes to GUIs, plug-n-plays, auto-installs and all the other production focused OS functions.   In that respect, Ubuntu is no competitive threat to M$, especially with M$ war chest in the billions.  

I don't know that there's any way to bridge that gap, unless commercial Ubuntu gets all GUI-slicked-up, functions without command line, stays affordable / low-priced, but still stays lean-n-mean in code volume and speed.

But I don't know Ubuntu's biz plans or strategies.  If Ubuntu handles those subjects like they did their mass volunteer promotion campaigns in many countries recently, they may be in for alotta group-grope delays and other perpetual Open Source community problems. 

Time will tell.

Personally, I never want to be on a naval surface vessel again when the M$ battle platform goes tits up.  But I don't particularly like to spend 20% of my computing time doing debugs and fixes for every freekin Linux OS and Linux app install either... unless that's what I get paid to do with 20% of my time.
