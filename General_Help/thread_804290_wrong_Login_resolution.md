### Post by HoTMetaL on 2008-05-28
The solution I found for this problem was to edit /etc/X11/xorg.conf and make changes (using sudo) to the section of the file named 'Virtual'. I changed whatever resolution was there to: 1024 768 and saved the file. Restart and it should force a 1024x768 login screen resolution.

```
sudo gedit /etc/X11/xorg.conf
```

### Post by Zorael on 2008-05-28
Your issue could either be because you have a virtual resolution set up in [FONT="Courier New"]/etc/X11/xorg.conf[/FONT] (as earlier posters mentioned), or that X picks the highest resolution available and you want a lower one.

If the login window is shown at too high a resolution (*true* resolution), then you want to modify the Screen section of your xorg.conf to define what resolution is to be the default. See [this post](http://ubuntuforums.org/showpost.php?p=5041727&postcount=4).

If the login window "isn't centered", then you want to find and nuke the Virtual line in xorg.conf. It should also be in the Screen section. Comment it out by prepending a hashmark (#) to it, or by just deleting it altogether.

Save, logout, Ctrl+Alt+Backspace, see it if improved.

### Post by Zorael on 2008-05-28
Right. I've no idea if this'll work, but I suggest you try to make your xorg.conf as completely explicit as possible. Backup first. No guarantees.


[list][*]Specify driver in the videocard device Section.
```
Section "Device"
	Identifier	"Configured Video Device"
	**Driver		"nvidia"** # or intel, or nv, or ati, or fglrx, or whatever you're using
EndSection
```

[*]Specify available modelines in the monitor Section. You can calculate these using the **gtf** program in a terminal. Here's an example for my monitor.
```
familjen@mediaburken:~$ gtf --help

usage: **gtf x y refresh** [-v|--verbose] [-f|--fbmode] [-x|--xorgmode]

            x : the desired horizontal resolution (required)
            y : the desired vertical resolution (required)
      refresh : the desired refresh rate (required)
*(truncated)*

familjen@mediaburken:~$ gtf 1280 1024 60

  # 1280x1024 @ 60.00 Hz (GTF) hsync: 63.60 kHz; pclk: 108.88 MHz
  Modeline "1280x1024_60.00"  108.88  1280 1360 1496 1712  1024 1025 1028 1060  -HSync +Vsync

familjen@mediaburken:~$ gtf 1024 768 60

  # 1024x768 @ 60.00 Hz (GTF) hsync: 47.70 kHz; pclk: 64.11 MHz
  Modeline "1024x768_60.00"  64.11  1024 1080 1184 1344  768 769 772 795  -HSync +Vsync

familjen@mediaburken:~$ gtf 800 600 60

  # 800x600 @ 60.00 Hz (GTF) hsync: 37.32 kHz; pclk: 38.22 MHz
  Modeline "800x600_60.00"  38.22  800 832 912 1024  600 601 604 622  -HSync +Vsync

```

[*]Add them to your xorg.conf like so:
```
Section "Monitor"
	Identifier	"Configured Monitor"

	# 1280x1024 @ 60.00 Hz (GTF) hsync: 63.60 kHz; pclk: 108.88 MHz
	**Modeline "1280x1024_60.00"  108.88  1280 1360 1496 1712  1024 1025 1028 1060  -HSync +Vsync**
	# 1024x768 @ 60.00 Hz (GTF) hsync: 47.70 kHz; pclk: 64.11 MHz
	**Modeline "1024x768_60.00"  64.11  1024 1080 1184 1344  768 769 772 795  -HSync +Vsync**
	# 800x600 @ 60.00 Hz (GTF) hsync: 37.32 kHz; pclk: 38.22 MHz
	**Modeline "800x600_60.00"  38.22  800 832 912 1024  600 601 604 622  -HSync +Vsync**
EndSection
```
Obviously your numbers might differ so don't copy paste what I posted *here*, but rather your terminal output from gtf.


[*]Define *all* available resolutions, but put lower priority on the ones higher than you want. Much like you have it now, I guess. I don't think you need the Viewport line, though.
```
Section "Screen"
	Identifier	"Default Screen"
	Monitor		"Configured Monitor"
	Device		"Configured Video Device"
	DefaultDepth	24
	SubSection "Display"
		Depth	24
		Modes	"1024x768" "800x600" "1280x1024"
	EndSubSection
EndSection
```

[*]Save, log out, restart X (Ctrl+Alt+Backspace), see if it worked.[/list]

### Post by Zorael on 2008-05-29
The login screen is basically a greeter that runs under X. Your resolution is "correct" once you've logged in, since then it will have read your user settings and applied your preferred res saved in your personal configuration files. Up until that point it doesn't know which user you are, and as such, defers to the X default, which is the highest autodetected resolution available.

If you create a new user, upon logging him in you should get the login screen resolution at the desktop, too.

### Post by parsim on 2008-05-31
I fixed this by:
(a) Listing my desired resolution first under "Modes"; and
(b) Setting that same resolution as "Virtual"

Until I did this, I could only see a small part of the login screen -- my monitor was displaying 800x600 but the greeter was apparently much larger, meaning the login box was mostly off the screen.

The appropriate part of my /etc/X11/xorg.conf now looks like this:
```
Section "Screen"
        Identifier      "Default Screen"
        Device          "Configured Video Device"
        Monitor         "Configured Monitor"
        Defaultdepth    24
        SubSection "Display"
                Depth   24
                Virtual 1280 960
                Modes           "1280x960@85"   "800x600@85"    "800x600@60"    "800x600@75"    "832x624@75"    "800x600@72"    "1024x768@85"   "800x600@56"    "1024x768@75"   "640x480@85"    "1024x768@70"   "640x480@75"    "1024x768@60"   "640x480@72"    "1024x768@43"   "640x480@60"    "1152x864@75"   "1280x1024@75"  "1280x960@60"   "1280x960@85"   "1280x1024@85"  "1280x1024@60"  "1280x960@75"   "1400x1050@60"  "1400x1050@75"  "1600x1200@65"  "1600x1200@60"  "1600x1200@75"  "1600x1200@70"  "1600x1200@85"  "1792x1344@75"  "1792x1344@60"  "1856x1392@60"  "1920x1440@60"  "2048x1536@60"
        EndSubSection
EndSection
```
