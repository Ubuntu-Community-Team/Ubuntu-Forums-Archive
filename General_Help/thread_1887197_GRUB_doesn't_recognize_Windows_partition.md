### Post by oldfred on 2011-11-26
What did sudo update-grub give you? Files look correct.

The boot flag is not used by grub or by grub in booting Windows, but if you want to boot or repair windows the boot flag would have to be on the 100MB (hidden in Windows) boot/repair partition sda2. Your sda1 looks like the computer vendor recovery partition which is just an image of your hard drive.

With grub legacy we almost always had to add Vista/Windows 7 boot stanzas manually. It would only find XP automatically.

If the update does not work, then the windows Partition(s) may need chkdsk.

### Post by oldfred on 2011-12-26
Grub1 will not boot Windows any better than grub2. Both just chainload to the Windows PBR and from the PBR Windows does its own booting. 

Grub1 usually only found XP boot stanza's and not always correctly. We made many fixes to grub's menu.lst with Windows chainload entries.

Do you know anyone else with Windows same 32 or 64 bit version? Then you can make a repair CD.

Make your own Windows repairCD (not vendor recovery):
[http://windows.microsoft.com/en-GB/windows7/Create-a-system-repair-disc](http://windows.microsoft.com/en-GB/windows7/Create-a-system-repair-disc)
[http://forums.techarena.in/guides-tutorials/1114725.htm](http://forums.techarena.in/guides-tutorials/1114725.htm)

Windows 7 repair USB, Also Vista if service pack installed
[http://www.intowindows.com/how-to-repair-windows-7-from-usb-flash-drive-repair-without-installation-dvd-disc/](http://www.intowindows.com/how-to-repair-windows-7-from-usb-flash-drive-repair-without-installation-dvd-disc/)
[http://www.webupd8.org/2010/10/create-bootable-windows-7-usb-drive.html](http://www.webupd8.org/2010/10/create-bootable-windows-7-usb-drive.html)
