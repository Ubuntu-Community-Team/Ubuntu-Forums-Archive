### Post by ProjectKITT on 2015-03-28
Hi all,

I'm new to the forums, but not new to Linux. Back in the summer of 2012, I was looking for a replacement OS for my HP laptop and I tried about every flavor of Ubuntu, as well as Linux Mint and a few versions of Puppy Linux. I settled on Kubuntu 12.04 LTS, which I used for a few years, then I switched to Ubuntu 14.04 LTS, which I liked quite a lot. Unfortunately it doesn't run quite as well on my new-to-me 1.6GHz/1GB Acer netbook as it did on my 2.5GHz/4GB HP. I replaced Ubuntu 14.04 with Xubuntu 14.10 on the netbook, and it's running much smoother now (I'm also glad to say that I like 14.10 more than I did the version of Xubuntu I tried in 2012). I've been spending the last few days working out the bugs (or working around them, I should say), and I noticed something curious and thought I'd post it here.

My computer had the "standby when lid is closed, even when set not to" problem. After a lot of Googling, I found out that it had to do with the Light Locker settings (not a surpise to some of you here, I'm sure), but the interesting part is that if the 'Lock on suspend' option is turned on, even if 'enable light-locker' is off, the computer will still standby when the lid is closed. 'Lock on suspend' needs to be turned off, regardless of the Light Locker setting, to prevent unwanted standby. I can duplicate this effect reliably, which would be a great way to annoy anyone who borrows my computer :p 

So I just turned the lock on suspend off, which is a bit of a shame since it works so flawlessy in Ubuntu 14.04, but it gets rid of the problem. Now I'm back to troubleshooting a few other things, mostly power-related as luck would have it...

Anyway, thanks for this awesome forum. I've been lurking here quite a lot, looking for answers to things, and I figured it was time to participate a bit ;)

Take care,
