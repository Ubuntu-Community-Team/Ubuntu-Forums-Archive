### Post by drs305 on 2009-12-26
I have the exact same /etc/default/grub settings and my menu is hidden on boot unless I hold down the shift key. 

The "hidden menu" feature has been a bit problematic. During Karmic testing there were bug reports on the "hidden menu" feature. One workaround I found was to add "GRUB_DISABLE_OS_PROBER=true" and then update-grub. The menu was then hidden, but of course you either lost any OS menuentries on other partitions or had to place them in a custom menu.

Here is the original bug report. Workarounds (not fixes) are presented near the end.
[https://bugs.launchpad.net/ubuntu/+source/grub2/+bug/444495]("https://bugs.launchpad.net/ubuntu/+source/grub2/+bug/444495")

The previous post (#7) doesn't quite tell the entire story. The menu is hidden upon installation if no other OS is found. The developers were trying to speed up the boot process and made the determination that if there was not other OS detected there was no need to display the menu on boot. The user could intervene to display the menu by holding down the SHIFT key if the menu display was desired. If another OS was detected, the menu was designed to be displayed. 

These settings were supposed to apply until the user changed them. I have hidden menus on one Ubuntu only machine and another with both Ubuntu and Windows installed.

Why the hidden menu doesn't work for all users remains a mystery.

Here are two options:

1. Retain Grub2, edit 40_custom to put in the kernel you want to boot, then remove the executable bit from the 10_, 20_, 30_ files in the /etc/grub.d/ folder. This should leave you with only what you have in 40_custom when you run update-grub. If you only have one kernel listed and you say it hides that way, this is a way to keep G2 installed until you find a better solution.

2. Remove G2 following the instructions in the community doc:
[https://help.ubuntu.com/community/Grub2#Uninstalling GRUB 2]("https://help.ubuntu.com/community/Grub2#Uninstalling%20GRUB%202")

### Post by rcorbish on 2009-12-26
Thanks for the response.  I played around with the templates and found this - hope it helps someone ...

I did manage to get the menu with a 10 second timeout ...

At the end of /etc/grub.d/00_header change the -1 timeout to 10


[B]cat << EOF
if [ \${recordfail} = 1 ]; then
  set timeout=10   # used to be -1 in original 
else
  set timeout=${GRUB_TIMEOUT}
fi
EOF[/B]

Then sudo update-grub to set the /boot/grub/grub.cfg with the new timeouts

Now I just have to be careful with updates...
