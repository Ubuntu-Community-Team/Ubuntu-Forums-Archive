### Post by mgol on 2009-01-25
NO! You **should** read warnings. They're often **very** important. And running fsck without parameters run it on Your main partition, so You could destroy the whole system... Fortunately, You didn't run it as root, so You were not allowed to do anything potentially harmful. **Don't ever do fsck on a mounted partition, especially the one that is running the whole system!**

A few questions. Does the external harddrive have more than one partition? Are You unable to get to all of them or to just one?

Post, as I said before, the output of:
```
mount
```
command. Post also the output of:
```
sudo fdisk -l
```
so that I know the structure of Your HDD. From that output You'd know which partition You have to check. **Remember to unmount it first, using:**
```
sudo umount /dev/sdXY
```
where X is a letter (probably 'b') and Y is a number of Your partition.
