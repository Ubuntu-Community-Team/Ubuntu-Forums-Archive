### Post by coffeecat on 2011-01-18
@runrun395, sorry for the delay getting back. Timezones.

Apart from the boot flag (which won't do any harn on sda1 for now) I can't see any problem in your bootscript output. With grub2 in the mbr looking in sda1 it *should* work. The instructions in the link you posted are correct but overly complicated. Try this. Boot up with the live CD and open a terminal. Now:

```
sudo mount /dev/sda1 /mnt
sudo grub-install --root-directory=/mnt/ /dev/sda
```

Now see what happens when you try to reboot. If you can reboot into Ubuntu, to get Windows into the grub menu, you simply open a terminal and:

```
sudo update-grub
```

To be honest, it looks as though you have already done the equivalent of the grub-install command above, but it's worth trying again anyway in the form I've posted. Let us know what happens and in the meantime I'll have another look at your boot script output and see if I've missed anything.

By the way, about the boot flag and Windows. I believe - but I'm not 100% sure - that the boot flag is only needed by Windows if you are booting with Windows boot code in the mbr. If you boot into Windows using grub, it doesn't matter about the boot flag. Whatever - I'm happy to be corrected on this point and it's a minor one. We can attend to that later once you're able to boot into Ubuntu.

### Post by coffeecat on 2011-01-19
Editing the Windows registry like that may affect Windows booting but it will have absolutely no effect on grub or Ubuntu. It's very perplexing that the normal fix did not work. I think you'll need to try something more complex. Have a look at this link:

[https://help.ubuntu.com/community/Grub2#Reinstalling%20from%20LiveCD](https://help.ubuntu.com/community/Grub2#Reinstalling%20from%20LiveCD)

Scroll down a bit to the heading 'METHOD 3 - CHROOT'. Follow that method for repairing grub. Where it says 'sdXX' substitute sda1, and for 'sdX' substitute sda. Don't bother with step 5; you don't have a boot partition.

Good luck with that.
