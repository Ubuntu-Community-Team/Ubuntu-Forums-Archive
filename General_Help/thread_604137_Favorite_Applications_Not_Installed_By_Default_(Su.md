### Post by sloggerkhan on 2007-11-07
The trick to media playback on linux is to use the right video overlay or output method. No matter which media player, xine or vlc, you can get good playback of must stuff scaled if you use teh xv output.  I used to hate using VLC too, but after switching playback to an xv overlay, it works pretty well. I just prefer xine usually because the UI and such is a little more to my taste. You will get bad playback if your CPU is picking up the processing instead of a video card, which is what I think several of the media players default to.

### Post by brundlelinux on 2008-03-24
I've been using Ubuntu for about a year.  Right now, I'm helping two of my very closest friends make the switch from dumping Windows to Ubuntu.  I completely understand why the official "out of the box" Ubuntu doesn't come with some support.  So, speaking from my own personal experience and from what my friends are going through being 'nix newbies, it would be great to have the following in a "batch install" package:

Yakuake
Cairo-dock
The Ubuntu-restricted-extras / Medibuntu / w32codecs / gstreamer gauntlet of packages
Frostwire
Checkinstall
Deborphan
build-essential
Amarok (face it, it's a LOT more functional than rhythmbox)
Gpodder (Again, because since Democracy became Miro, it's been mighty unstable on Linux)

Those are things that I think EVERY ubuntu user should immediately install after a fresh OS install (except cairo and Frostwire, which are personal taste).  It's easy to do so, yeah, but for a linux noob, having a one-stop-shop package would be great.

Also, let's not forget that, for better or for worse, Ipods are the current rage.  Having Ipod interface support in a package like this would go a LONG way for the non-coder types who are just getting their feet wet in real "under the hood" computing.

Just my 2 cents
