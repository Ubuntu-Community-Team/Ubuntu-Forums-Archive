### Post by sisco311 on 2008-11-18
mount the partitions by uuid.

change the last two lines in fstab to:
> UUID=3EE87E06E87DBD29 /media/HP_PAVILION ntfs defaults,umask=007,gid=46 0 0
UUID=3A585457585413CB  /media/Windows\040XP ntfs defaults,umask=007,gid=46 0 0


[https://help.ubuntu.com/community/UsingUUID]("https://help.ubuntu.com/community/UsingUUID")
> Linux now prefers to use UUID (Universally Unique Identifier), LABEL, or symlinks to identify media storage devices on a system. Directly using /dev/hd*# or /dev/sd*# is no longer preferred since these device assignments can change between system boots.

### Post by HousieMousie2 on 2010-08-11
Update: In my case, using e2label is easier and has virtually the same results, except that the labels are maintained even when drives are shuffled around.
Using the UUIDs, requires altering fstab every time the drives are shuffled (removed, replaced, excluded, etc.)
Just make sure the partition labels are not duplicated.
In the event that a machine is part of a network, the UUIDs would be preferable, ensuring that no two partitions have the same 'name.'
