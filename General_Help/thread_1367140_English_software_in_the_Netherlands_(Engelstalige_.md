### Post by Pander on 2009-12-29
Many internationally oriented users or organisations, which are physically located in the Netherlands, communicate mainly in the English language and therefore use software in English. However, due to the geographic location, it can be desirable for data to be represented according to the local Dutch notation. This locale is called en_NL and provides such a configuration. It is based on en_US and altered with parts from nl_NL.

See the attachment for details. Please remove the .txt extension before using the file. Any improvements are welcome in via this thread.
