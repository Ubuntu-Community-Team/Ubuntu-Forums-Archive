Computers have "Virtual Memory", this is a combination of fast memory (RAM) and slow memory (Swap space).

A computer needs a certain amount Virtual Memory to run its programs, the mix of fast and slow memory in the overall available pool will determine how quickly things can run - obviously if everything that needs to run can use the fast memory then things will run better than if they had to use the slower option.

Modern computers have a lot more fast memory (RAM) than in the "old days" when this was (far) more expensive, so the need to use slower Swap Space is generally diminished compared to the ancient days of computing (that means 10 years ago.....)

Some software tries to use Swap Space deliberately in the misguided (and now obsolete) belief that it knows better than the Operating System when it comes to managing memory, if an O/S has a need to use Swap Space then it will, otherwise it will use the available fast memory for what it decides is the best use of this resource.

If you have enough RAM for your applications, your may never (or rarely) use Swap Space, the "rules" about 1.5 times or 2 times RAM are nothing more than obsolete myths that are still propagated by those that don't understand the hardware of today.
