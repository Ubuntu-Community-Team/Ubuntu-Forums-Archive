### Post by earthpigg on 2009-07-12
if you can handle a gentoo setup, but are still perhaps attracted to ubuntu's large 3rd party support (opera provies an ubuntu-specific .deb download, for example), and/or ubuntu's community (you are posting on this forum, so you probably are).... you sound like an ubuntu minimal command line install kind of guy.

[https://help.ubuntu.com/community/Installation/MinimalCD](https://help.ubuntu.com/community/Installation/MinimalCD)

and select the option for command line only install. avoid huge meta-packages to stay slim and fast.

the standard ubuntu desktop install includes everything and the kitchen sink with it - which you probably do not want.

or, maybe you later decide you do:

```
sudo apt-get install ubuntu-desktop
```

You will not miss Windows at all, just switch; if you are CS:Source addict just keep a small partition with Windows and label it "Video Games Console"; or if you have a powerful enough computer you can try virtualizing, would be interesting to see if CS:Source works on a virtual machine :p

About what GNU/Linux distro should you use; well whatever you like. I'm on the "the less time you spend configuring the more time you'll get to work/do what you like" side, so that's why I'm here, using Ubuntu or Debian, you just install and in about 30 mins you are good to go, and can concentrate on your work instead of compiling and configuring/customizing everything; even Linus Torvalds prefers distros like Fedora or Ubuntu ([read it!]("http://news.oreilly.com/2008/07/linux-torvalds-on-linux-distri.html")); so my advice would be to not fight with 100% customized distros just for the sake of it (a 100% custom gentoo install [for example] does run a lot faster though), if you want to learn about installation processes, then install Gentoo from the minimal CD and READ all the [Gentoo documentation]("http://www.gentoo.org/doc/en/"), is in my opinion the best documentation out there, I learned a lot about Linux with that documentation.

So my recommendation would be Ubuntu, because you get what you need fast and easy, so you can star doing what you want in minutes, not days.
If you want to try a middle-point distro, I recommend Slackware, it is sweet, very fast and quite easy to use, the bad side is it has no native-package for GNOME (I love gnome) it comes with XFCE, KDE and others desktop environments, it is really good though; you can also try ArchLinux, is like the next step after Gentoo minimal ("difficulty"-wise).
