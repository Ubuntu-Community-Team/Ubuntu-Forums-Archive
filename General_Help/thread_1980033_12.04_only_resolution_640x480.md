### Post by papibe on 2012-05-14
Hi ToothlessFerret.

I would first backup your current graphic configuration:
```
sudo cp /etc/X11/xorg.conf  /etc/X11/xorg.conf.old
```
If you are not using a configuration file the previous command may gave you an error. Just ignore it.

Then to recreate and specific Nvidia configuration:
```
sudo nvidia-xconfig
```
I hope that helps, and tell us how it goes.
Regards.

### Post by bogan on 2012-05-14
Hi!,** toothlessferret**,

You need to be in root [su] to be able to save the Xserver settings.

 gksudo nvidia-settings

### Post by mickey13 on 2012-08-15
If running

```
sudo nvidia-xconfig
```

fails, it could be that nvidia-xconfig is in a directory that's not in your $PATH.  Try

```
sudo find / -name nvidia-xconfig -print
```

Then run

```
sudo <a_result_from_the_previous_command>
```

Something else to note for Dell machines; if you have a newer Dell machine that comes with Windows 7, there is some "technology" called Optimus (I think) that needs to be disabled in your BIOS for the nVidia driver to run properly (at least that's my understanding from my experience).  Optimus is only applicable to Windows 7 32-bit/64-bit.
