### Post by dixonstalbert on 2010-09-16
Hello

First thing I would do is run fsck from live CD.

I noticed you tried ```
fsck /dev/sda 
```

and got a badblock error. I think fsck needs you to specify the partition; eg

```
fsck /dev/sda2
```

My memory is you boot from ubuntu LiveCD, and open a terminal then type

sudo fdisk -l to get your partition list.

sudo swapoff -a to unmount the swap partition on the harddrive that the LiveCD automatically mounted

sudo e2fsck -f /dev/sda2  (and all other linux partitions listed by fdisk -l)


this will clean up your linux partitions


If things still do not work I would try downloading and burning "Super Grub Disk" - this is a utility that automates rebuilding Grub. I like it because I rarely have to fix Grub and have to re-learn the process every time I need to fix it.  

If things still will not boot you have to fix the mystery partition on sda1- probably a damaged msdos partition. The best way to do this is running Microsoft Scandisk/chkdsk from a Windows Install disk. there is a dos fsck for linux but the last time I tried it was quite limited.

good luck!
