Edit; SOLVED! after digging in launchpad, I found that:

In GRUB menu you press "e"

Add this line to the end of the boot options (the ones that end with ro quiet splash)
[INDENT]all_generic_ide=1[/INDENT]

Boot and it will all be fixed

Note: this works in most configurations with this problem, yet there are still some that won't be fixed by this

### Post by mc4man on 2008-06-06
Your dvd drive is dev/scd1, linked at /dev/dvd2, cdrom2, ect. Your cdrw is dev/scd0, linked at /dev/cdrom (not hdb, hdc, /dev/dvd1, ect.)
You basically have 3 choices,  -  1. you could do a fresh install.
2. Go into the set up for your dvd apps and change the default device to /dev/dvd2
3. Edit your fstab and your 70-persistent-cd rules file to set things right
If you want to do 3 post from /etc/udev/rules.d - the contents of 70-persistent-cd rules.
While yours is slightly more complicated read here to get idea
[http://ubuntuforums.org/showthread.php?t=801771](http://ubuntuforums.org/showthread.php?t=801771)

### Post by mc4man on 2008-06-06
We can fix things up without much problem, I'll post the way your ...rules should look. It would be good if you could first describe your set up.(as best you can)
1. which _optical drive _can you boot off of?, ie. if you were to boot to a live cd which drive would you put the disk in? 
2. you have 2 hard drives and 2 optical drives - are they all on ide cables (flat ribbon) or do you _have any sata drives_ and if so which one(s)
3. if you could describe the position of the 2 optical drives on the cables that would be great, if not that's ok. 
I'm only asking because there's some inconsistencies in the info from posts 6 and 12 and i'm somewhat torn about which device to give the /dev/cdrom link.

_On 2nd thought_!  - if you happen to have a _Hardy_ boot disk then we don't need any of the above info. Simply boot to the live cd, choose 'try ubuntu without any change to your computer', after the Os loads navigate to /etc/udev/rules.d/70-persistent-cd rules. If you can post the contents of that we're good to go.
If you can do the above (with boot disk) also run sudo lshw -C disk and post that too. ( doing all of this from the live cd will provide perfect info that hasn't been skewed by the update process and a borked ..rules file)
i know this might sound confusing but the whole deal is very simple - just want to give you the best setup.(if you read the thread I linked you'll see how easy it is)

### Post by Takmadeus on 2008-08-16
SOLVED! after digging in launchpad, I found that:

In GRUB menu you press "e"

Add this line to the end of the boot options (the ones that end with ro quiet splash)
[INDENT]all_generic_ide=1[/INDENT]

Boot and it will all be fixed

Note: this works in most configurations with this problem, yet there are still some that won't be fixed by this
