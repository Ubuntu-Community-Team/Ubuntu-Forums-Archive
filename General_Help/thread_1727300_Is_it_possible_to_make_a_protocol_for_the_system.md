### Post by xic816 on 2011-04-12
Hi, I was wandering if it's possible to create an protocol in the system. With that I mean to make the computer do curtain things when curtain events happen, for example; 

You loose your Internet connection for some reason, the protocol 
go into network manager to disconnect and reconnect


or

You are using a music program with commercials after a song, you just make
a protocol stopping the song at 99.99% and make it start a new one automatically.

Is any of this possible, thanks for answers!

### Post by Mark Phelps on 2011-04-12
> **xic816 said:**
> ...
You loose your Internet connection for some reason, the protocol 
go into network management and disconnect and reconnect

This might be possible but you'd need a program running in the background monitoring the internet connection and launching a script to reset the network.  Best place to ask this kind of question would be in the Networking forums or the Programming forum.

> You are using a music program with commercials after a song, you just make a protocol stopping the song at 99.99% and make it start a new one automatically.

My guess on this one would be NO.  Why? Because the event handler to do this would have to be able to distinguish audio present in a commercial from audio present in a tune -- and I don't think that is possible in pure audio streams.  There have been rumours about making something similar (detecting and skipping over commercials) in DVRs, but the entertainment industry is hard afoot making sure that does NOT happen.
