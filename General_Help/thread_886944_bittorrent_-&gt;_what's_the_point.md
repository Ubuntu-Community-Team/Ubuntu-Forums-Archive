### Post by issih on 2008-08-11
Ok fair enough, there is a point that p2p software is harder to trace and makes it far less clear who to prosecute as the "uploader" as far as illegal file sharing goes, but to imply that is the only point of bittorrent is unfair. Lots of the media streaming applications you can install use torrent style behaviour to download their content, this is used because it minimises the bandwidth cost to the original producer.

You upload a few times, and then all the seeds start sending to each other in a storm. If you have enough seeds a torrent tends to be far faster than a normal streamed download, and the bandwidth cost to the person sending it is much lower because you utilise the upload bandwidth of all the downloading clients for free.

Its a clever idea, and has its legitimate place on the internet, it has plenty of less legitimate uses, but so does almost every technology on the planet.

### Post by kernelhaxor on 2008-08-11
Sharing illegal media/files is not the purpose of Bittorent ..

Bittorent is a very powerful and inexpensive way of catering to a mass audience .. lets say I want to deliver a file to a large number of people but don't have the resources to have a high bandwidth and speedy central server from which everyone can download from.  I can rely on bittorent .. it pushes small amount of burden to each peer and thus gives me a very practical and inexpensive solution ..
the traditional client-server architecture doesnt scale infinitely but bittorent does ..

to answer ur question abt speed, speed depends on the number of peers in the network that have the file and their uploading speeds ..
