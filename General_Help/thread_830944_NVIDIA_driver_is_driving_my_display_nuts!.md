### Post by danwood76 on 2008-06-16
Well your xorg isnot configured right.

open it up in gedit with root access:
```

sudo gedit /etc/X11/xorg.conf

```

And change the first device sectiuon to be:
```

Section "Device"
	Identifier	"Configured Video Device"
	Boardname	"Quadro 540"
	Busid		"PCI:1:0:0"
	Driver		"nvidia"
	Screen	0
EndSection

```
Save it and reboot, this will then load up the nvidia driver instead of the free 'nv' one.
Of course if you havent installed the nvidia driver correctly you will end up in low graphics mode.

What was wrong with the drivers available in the repositories?

### Post by cocopuffz on 2008-06-16
I use this as a partial reference from [**here**]("http://amazingrando.wordpress.com/2007/04/10/compiling-and-using-newest-nvidia-drivers-in-ubuntu-its-easier-than-you-think/")
--------


"...Compilation dependencies

The Nvidia drivers have a shell script to install and compile, but they&#8217;ll need some dependencies, which can done by typing the following in the terminal.

sudo apt-get install linux-headers-`uname -r` build-essential gcc gcc-3.4 xserver-xorg-dev

Step 3 - Clean out the old Nvidia drivers

First, you&#8217;ll need to purge the nvidia-glx driver

sudo apt-get remove --purge nvidia-glx

Second, you should make sure that the linux-restricted-modules for nvidia are disabled (blacklisted).

sudo gedit /etc/default/linux-restricted-modules-common

Note that it may be linux-restricted-modules on pre-Feisty systems.  You&#8217;ll know if it&#8217;s not the right one if the file you open is blank.

There, put in the following code, and save.

DISABLED_MODULES="nv"

Step 4 - Reboot

This isn&#8217;t probably necessary, but good to make sure the purged stuff isn&#8217;t still loaded

Step 5 - Stop the X server (i.e. in Gnome go "cntr + alt + f1) and go to a command line interface. Note that this is/was a bug in the Feisty beta that may prevent you from doing this command.

sudo /etc/init.d/gdm stop

Step 6 - Run the installation script(make sure you've moved to the dir (cd /home/you/Desktop as and example...) it's in and "chmod +x" to be able to run it)

sudo sh NVIDIA-Linux-whatever-your-version-is.run

Follow prompts to accept the license, and look for a pre-compiled kernel. Eventually it will ask you to compile a custom one, which is what you want.

After it finishes it will ask you to modify the xorg.conf file. Choose yes.
-----end of reference page..

once that's done I do this.

type while still in commandline land.. "gksudo displayconfig-gtk" and that should bring up a gui to edit your xorg file and auto detect your monitor. If you find your brand and model.. good for you. if not, just use the generic "plug and play" one but with the correct resolution. select the widescreen radio tab if you have a wide screen...

--------

You should reboot. I usually go:

sudo shutdown -r now

You'll notice it still looks like crap. The option is now available to you in your display settings and in the Nvidia x server settings to change res etc. Good luck.

### Post by danwood76 on 2008-06-18
Are you sure you installed the correct version of the nvidia driver?

If you want a good res you might try using the free nvidia driver, in your xorg.conf change the driver line in 'Configured Video Device' to be nv. This should then give you good resolutions.

The other thing you could check is that there is no xorg errors when loading up.
To do this run the following commands in the terminal and post the results:
```

cat /var/log/Xorg.0.log | grep -i '(EE)' # Check for errors
cat /var/log/Xorg.0.log | grep -i '(WW)' # Check for warnings

```
