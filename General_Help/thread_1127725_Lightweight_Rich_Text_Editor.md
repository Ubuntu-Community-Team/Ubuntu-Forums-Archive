### Post by ilembitov on 2009-04-16
Hi, all. I was looking for a good lightweight rich text editor for Linux for pretty a while.
Basically, I want a lightweight GTK app without GNOME dependencies to use in XFCE. It should support some basic supporting (paragraphs, fonts, styles - nothing more than WordPad in Windows) and at least one popular format (rtf, doc, odt - anything).
So far I have found only these lots:
-AbiWord. Obvious, but not really the best. First, it was a little buggy when I used it. Second, it had some weird plugin system. And overall the experience wasn't too pleasant.
-Ted. It's not GTK, it doesn't support Unicode (and I write Russian mostly) and doesn't handle fonts properly. A new version is under development, which is suprising, but currently it's not useable.
-WordGrinder. It is CLI and it doesn't support any format like RTF or DOC.
-PyRoom. Great experience (although it still lacks some functions of, say, Q10), but doesn't allow formating and saving in formats other than plain text - that kind of defeats the whole purpose of the project.

Can you advice something to me?
