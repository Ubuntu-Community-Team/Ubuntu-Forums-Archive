### Post by fridaythe14th on 2009-05-31
I'm currently using RSSOwl which has all the features I want but is too slow/unresponsive and too much of a memory hog.

Any advices for a lightweight reader? Preferrably with somewhat advanced filtering features, but if not it must at least have some kind of filtering.

It doesn't have to be in the repos.

I'm using RSSOwl today because all the other RSS readers I've tried have the traditional [field] [contains/does not contain/etc] [key phrase] filtering where I'd have to enter each key phrase manually. RSSOwl does too, but you can write a list of key phrases where it will match any of them, which is great for me as I'm generating my filter from my last.fm library and updating it every once in a while.

However if the RSS reader stores it's filter as a plain text file I could mess around with that instead of the gui. I have found this to work partially in Thunderbird, but I have to go through all my feeds and manually check the filters against them after I've updated to get the old news filtered with the new filters.
