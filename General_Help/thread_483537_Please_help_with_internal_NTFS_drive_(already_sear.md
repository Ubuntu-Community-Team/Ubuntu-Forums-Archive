```
OK.  sdb1 is your ntfs partition, but as you can see from /etc/fstab, it is not mounted.

You can try mounting it manually:

```

sudo mkdir media/sdb1
sudo mount -t ntfs /dev/sdb1 /media/sdb1

```
If this works, then we can add a line to /etc/fstab so it mounts on start-up, and go from there.
```

### Post by mckryptyk on 2007-06-25
Hi this is what you need to do to enable NTFS read/write ability:

(Assuming your using feisty, if not I can still help)

1) Enable "Universe" and "Multiverse" Repositories (check forums for this one if your stuck)

2) This will install ntfs-3g
sudo apt-get install ntfs-3g

3) This will tell you what partition is NTFS 
sudo fdisk -l | grep NTFS

4) This will backup fstab before you edit it 
sudo cp /etc/fstab /etc/fstab.bak

5) This will edit fstab 
gksu gedit /etc/fstab

6) Write this into your fstab below the other lines 
/dev/<your partition>     /media/<mount point>     ntfs-3g     defaults,locale=en_US.utf8   0    0

7) This will make the folder for it to mount 
sudo mkdir /media/<the name you want>

8) This will unmount and remount all partitions from fstab 
sudo umount -a && sudo mount -a

All Done :) 
If there are any problems post here.
