A LiveCD is an operating system (usually containing other software as well) stored on a bootable CD-ROM or DVD-ROM that can be executed from it, without installation into permanent memory, such as a hard drive. The system returns to its previous OS when the LiveCD is ejected and the computer is rebooted. It does this by placing the files which typically would be stored on a hard drive, into temporary memory, such as a ram disk. This however does cut down on the RAM available to applications, reducing performance somewhat.

ISO image
From Wikipedia, the free encyclopedia.
Jump to: navigation, search
It has been suggested that this article or section be merged into disk image. (Discuss)

An ISO image (.iso) is an informal term for a disk image of an ISO 9660 file system. More loosely, it refers to any optical disk image, even a UDF image.

As is typical for disk images, in addition to the data files that are contained in the ISO image, it also contains all the filesystem metadata (boot code, structures, and attributes). All of this information is contained in a single file. These properties make it an attractive alternative to physical media for the distribution of software which requires this additional information as it is simple to retrieve over the net.

Some of the common uses include the distribution of operating systems, such as Linux or BSD systems, and LiveCDs.

Most CD/DVD authoring utilities can deal with ISO images: Producing them either by copying the data from existing media or generating new ones from existing files, or using them to create a copy on physical media.

Most operating systems (including Mac OS, BSD, Linux and Windows, with third-party tools) allow these images to be mounted as if they were physical discs, making them somewhat useful as a universal archive format.

Console emulators, such as ePSXe, and many other emulators that read from CD/DVD, are able to run ISO/BIN (and other similar formats) instead of running directly from the CD drive. Better performance is achieved by running an ISO since there is no waiting for the drive to be ready and the hard drive I/O speed is many times faster than the CD/DVD drive

### Post by steve.horsley on 2006-01-25
Two things:

You must burn the CD in the correect manner. You have to burn the .ISO file as an **image** of a CD. That is to say, you don't just burn the file to the CD, and you don't unpack the file and write the unpacked files to the CD. In the program nero, there is a menu option to burn a CD image, and if you choose that, it then opens a file-open dialog and asks for the .iso file you want to burn. I can't say about other windows CD burning packages. If you do not burn the CD properly, it will not be bootable.

You must boot from the CD instead of from the hard disk that contains windows. On my laptop, F12 gets me a boot menu that includes options like boot from CD or USB-disk. Choose to boot from CD of course. If it fails to boot from the CD, then probably the CD hasn't been burned properly.

### Post by cotcot on 2006-01-25
What operating system do you run on your laptop now ? Suppose it is windows. What is the CD burning program you use ? Let us suppose it is Nero. If you burn an image in Nero you have to do it with the tab "ISO" otherwise the CD will not boot. In NERO Smart Start there is an entry "burn image to disk". If you use other burning software you have to look to specifically burn an ISOimage. In k3b there is also possibility to select burn image.

Check the BIOS. Are you familiar with BIOS ? You need to check if your CD rom is enabled (obviously it is because you have seen data on the CD you burnt) and if the your CDROM is first in the boot sequence.
Is it your intention to get a dual boot of your actual operating system with Ubuntu ? 

(I apologise if this seems too basic for you). Hope this sets you on the way.
