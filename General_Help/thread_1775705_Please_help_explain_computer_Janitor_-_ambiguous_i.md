### Post by hawthornso23 on 2011-06-05
Thought I'd have a poke around with computer janitor since my system has been through many upgrades and there is no doubt some redundant stuff on it. I am generally aware of what the application is supposed to do but have never used it before.

When I open it up a list of applications appears. Each is preselected with a tick box. When I click on an application I get an explanation as to why it might not be needed on my system any more. At the bottom of the screen is a button which says "perform selected tasks". 

"That seems clear", I think. "When I click on the button then all of the selected applications will be removed." However after looking through the list there are some I want to keep. So I go to unselect them in the tick box list. At that point I am asked to authenticate with the explanation that root permission is needed to remove applications. That was unexpected, and now I'm confused. 

I would not expect to be asked to authenticate to remove something from a list of things to be removed. Indeed the wording of the authentication notices makes it seem like unselecting an application will remove it.


So are the selected applications the ones which are to be removed or the ones which are to be retained. What exactly will happen when I click the mysterious and completely ambiguous "perform selected tasks" button. I realize I have no idea how this application works and what it will do. Fearful of having it do the exact opposite of what I intend if I make the wrong choice I choose to do nothing and simply close it. 

So can someone who knows how this thing works please tell me, is it the selected applications that are removed or the deselected ones. I really think a bug should be filed against this application as it is completely ambiguous to the extent that someone trying to use it has a 50% chance of having it do the exact opposite to what they intend.
