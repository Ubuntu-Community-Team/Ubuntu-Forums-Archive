### Post by Oinos on 2016-11-23
**Issues:**
Mainly graphical bugs:
-Terminal shows as pitch-black rectangle, have to resort to back-up consoles(ctrl+alt+F1 to 6).
-Black rectangles and areas show elsewhere in the GUI - mainly instead of shadows around window borders, on top of tooltips when hovering cursor on links, etc.
-"Mousewheel" built in the touchpad deactivated or no longer working - not sure if directly related.

**Backstory:**
A program I use released a new version, which requires OpenGL 2.1 at minimum to function. I have **Intel® Core&#8482;2 Duo CPU U7700 @ 1.33GHz × 2  w/ Graphics Intel® 945GM x86/MMX/SSE2**, on which the maximum supported OGL version is 1.4. However, I got 2.1 support and the program functioning via mesa - [this]("thttp://feedback.wildfiregames.com/report/opengl/device/Intel(R)%20945GM") and [this]("https://people.freedesktop.org/%7Eimirkin/glxinfo/#p=compat&b=version&g=Intel%C2%AE%20GM45%20Express") show that mesa 10.1.0 supports OGL 2.1 on my config, so I got that. Immediately after everything was normal, but after the next system start I got all the issues described above.
I made use of [step 2.5 here]("http://www.mesa3d.org/faq.html") while installing mesa.
