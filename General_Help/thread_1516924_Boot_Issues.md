### Post by drs305 on 2010-06-24
Do you have a floppy drive? It could be the BIOS is trying to find a floppy that doesn't exist. You could enter your BIOS at system boot and disable the floppy drive option if it is enabled. If you have one, it may not be working with the current BIOS options.

Similarly, the second error is most likely being generated by fstab, which is referencing a drive that isn't being found. As a temporary measure, until you can figure out the floppy problem, you could disable the line in /etc/fstab which references this device. Just put a comment symbol at the start of the line which references *ISO 9660*. 
You can edit fstab with this command:
```
gksu gedit /etc/fstab
```

With either of these, make sure you are not disabling something that is required for you to boot into an OS (although it shouldn't be a problem, since it doesn't seem to be working regardless).

It looks like you lost a "space" in the fstab address for you cdrom. As a minimum, it should probably be
> /dev/scd0  /media/cdrom iso 9660,udf user,noauto,exec,utf 8 0 0

You might also check to ensure the folder */media/cdrom* exists as a mount point. If it isn't and you have a "/media/cdrom0", you may want to change the entry to match the mount point.
