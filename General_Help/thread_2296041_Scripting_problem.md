### Post by geohei on 2015-09-23
Hi.

Sounds easy but I didn't manage though ...

I'd like to prefix the output of a command (returning several lines) with the date. To simplify here, I use the ls as an example.

The **ls** says:
```
root@gan:~/test# ls -l
total 0
-rw-r--r-- 1 root root 0 Sep 23 08:54 file_a
-rw-r--r-- 1 root root 0 Sep 23 08:54 file_b
-rw-r--r-- 1 root root 0 Sep 23 08:54 file_c
```
I'd like to have this as result (date generated by **date +"%Y%m%d-%H%M%S"**):
```
20150923-085527 -rw-r--r-- 1 root root 0 Sep 23 08:54 file_a
20150923-085527 -rw-r--r-- 1 root root 0 Sep 23 08:54 file_b
20150923-085527 -rw-r--r-- 1 root root 0 Sep 23 08:54 file_c
```
I tried everything with a pipe to awk/print, but didn't get it since I didn't manage to include a command to the print statement inside awk.
(Something like ... **ls -l | awk '{ print (date +"%Y%m%d-%H%M%S") $0 }'** )

Any ideas?

Many thanks!

... later ...

I discovered that following works, but the date is not updated!
```
root@gan:~/test# ls -l | awk -v var=$(date +"%Y%m%d-%H%M%S") '{ print var " " $0 }'
20150923-102310 total 0
20150923-102310 -rw-r--r-- 1 root root 0 Sep 23 08:54 file_a
20150923-102310 -rw-r--r-- 1 root root 0 Sep 23 08:54 file_b
20150923-102310 -rw-r--r-- 1 root root 0 Sep 23 08:54 file_c
```
As I said before, the ls is quick, but I my case, I need a date which is updated for every line since the command might run for minutes. As it is now, the date is the one when the command is executed, hence not updated per line ls spits out.

Any ideas?

### Post by Lars Noodén on 2015-09-23
You could use [command substitution](http://wiki.bash-hackers.org/syntax/expansion/cmdsubst) with the help of $( ... ) inside sed:

```

ls -l | sed "s/^/$(date +%Y%m%d-%H%M%S) /"

```

or

```

ls -l | sed "s/^/$(date +%F-%T) /"

```

[sed](http://manpages.ubuntu.com/manpages/trusty/en/man1/sed.1.html) is a bit lighter than awk and gets the job done in this case.

### Post by Lars Noodén on 2015-09-23
Ah.  I misunderstood.  

Probably [stat](http://manpages.ubuntu.com/manpages/trusty/en/man1/stat.1.html) would work better:

```

stat -c "%A %h %U %G %s %y %n" *

```

The time is in local time not UTC, so the time zone is tacked on there.  It gives the time that the files were last modified.

### Post by steeldriver on 2015-09-23
You could pipe the output of your command into a while loop, and evaluate the date each time a new line is read

```

command | while read line; do printf '%s %s\n' "$(date +"%Y%m%d-%H%M%S")" "$line"; done

```

### Post by geohei on 2015-09-23
Thanks for all the answers!

This did the job exactly as desired ...
```
ls -l | awk -v cmd="date +%Y%m%d-%H%M%S" '{ cmd | getline d; print d,$0; close(cmd) }'
```
