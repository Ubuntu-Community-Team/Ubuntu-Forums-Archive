### Post by gpilkay on 2007-11-25
Not really.  You can use WINE to try to run some Windows applications, but the success is hit-or-miss.  The best advice is just to try it and see, as pathetic as it sounds.

Some people, like myself, use a duel-hard drive setup so as not to have to deal with partitions.  We just put Windows as SLAVE and a blank as MASTER, and load Ubuntu onto that, so no change to the Windows drive occurs.

Partitioning is a lot more intricate and I've never tried it, so I wouldn't even attempt to give advice on it, if you wanted to duel-boot your system for your Windows games.

If this is an Xbox or Xbox 360 game, you're likely out of luck, they only run on their designated platform, far as I know.

### Post by matthewcraig on 2007-11-26
Try shifting your thinking toward native Ubuntu games.  When switching from Windows to Ubuntu, it is tempting to ask yourself how you can get XYZ program running in Ubuntu.  Instead, ask yourself what functionality XYZ program gives you and find a native Ubuntu program that has the same functionality.  This is especially true for games.  You may have a first-person-shooter in Windows that you love, which has no Ubuntu support.  Well, it turns out there are many first-person-shooters available in Ubuntu, too!  Some are good, some are not so good, some are no-cost, and some are commercial.  Stay focused on the Ubuntu supported software, and you will help build the community for Ubuntu supported software.  Supporting Windows software on Ubuntu will only help promote developers who ignore Ubuntu support, especially when you pay for those games.  Got some extra money burning a hole in your pocket?  Take a look at [these games]("http://www.linuxgamingworld.com/games-catalog/") released by commercial developers who actively support Ubuntu gaming.  By paying developers who support Ubuntu, you will be encouraging them to spend more time developing new games in the future.
