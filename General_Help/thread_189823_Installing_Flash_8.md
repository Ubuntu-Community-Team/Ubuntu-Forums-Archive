### Post by link141 on 2006-06-05
[QUOTE=MasterChief1234]Or you can just use firefox under wine and install the flash plugin...[/QUOTE]
I just have to say how well that solution worked for me a couple weeks back. 
The problem of needing Flash 8.0 (not available for linux) to veiw- not optional, but required!- most websites that I like to visit (nintendo.com being one of them) almost had me right a big nasty gram directly to Adobe's top managers I was so furious!  Luckily I had a clever idea to use the windows version of Flash on the windows Firefox with the latest WINE.  To my surprise, it worked beautifully.  Infact, the easiest and best running windows firefox plugins I got working on Kubuntu.  

Of course, the Linux flash player 9 will be better than this method, but if you choose to use it for the time being, I would reccomend using it with firefox.  I tried getting internet explorer  working, but I found that I would have to go on a wild-goose chase for the packages necessary, and then fittle around with them for hours to get them to work.  I like things to be lazy-man-accessible, and I also don't have an experienced knowledge with Linux.  Even if you get Internet explorer usable with Ubuntu, firefox would probably run better (tighter code) under the cumbersome emulator code that WINE must use.

One more thing, when you tell Flashplayer 8 to install (windows .exe) and it messes up at the language, it will still work after you restart Firefox.  

Unlike I presumed, this was actually a reasonable solution, that was easy, fast to set-up, ran surprisingly fast, and worked with all Flash Player 8 sights I used it on.  So I would reccomend this as a practical solution for using Flash player 8 on Linux.
 Infact, it worked so well, that I was going to post it as a thread on this forum, but this will work just as well for Flash-desparate Ubuntu users.  Tell me if this post was useful :).
