### Post by TheFu on 2016-01-28
Good first post.

Didn't read the entire thing carefully. A quick scan didn't show any issues.

Did you try telling rbenv that ruby 2.16 was the ruby you wanted to use the run **ruby -v**? Nothing seems wrong.

Perhaps this will help? [http://blog.jdpfu.com/2012/07/18/ruby-on-rails-environment-on-ubuntu-12-04](http://blog.jdpfu.com/2012/07/18/ruby-on-rails-environment-on-ubuntu-12-04)  Running configure just sets up the build environment. Normally a "make" is needed after configure runs to actually build the software, followed by a make install to put the newly built software into the installed place(s).  I'd assume that rbenv handled that for you, unless no ruby was built and installed under the rbenv area.  Did you setup rbenv inside your shell environment?  The way to tell is when you run **which ruby** - it doesn't point to the system ruby in /usr/bin/ - instead it points to a ruby under your HOME somewhere.

My local ruby group has a small team with the sole purpose to help folks get Ruby installed on their development machine. It is a non-trivial task - at least it was.  They also teach beginning RoR classes, but changed from local installs to cloud-based development. That made life easier for the Windows people who showed up without any background and weren't prepared to use the OS as a development environment - like Unix is meant to be used.

Good using rbenv. I used rvm. Similar idea. Haven't done anything with Ruby in over a year, so I'm out of date.  That's an issue with Ruby and Rails - they change one of them completely every year - so if ruby is stable, then rails has been completely reworked. The amount of chnage in the dev tools and dev knowledge is just too much for me to keep up with. However, RoR is a great place to be if you are dev and want higher pay than the python/php guys. If I were starting out all over again, I'd do Ruby and Rails or Sinatra for a happy, well-paid career.

I'll stick with Perl.  Heck, I need to learn Perl6 now that it has been released after 15 yrs of development. ;)  That's a rate of change I can keep up with. Even C/C++ has changed faster than Perl 5-->6 did. ;)

Thanks for the code-tags. Very helpful.  Hope all of this makes sense.  BTW, ubuntu has installers just for developers.  I haven't used them, but they seem to be created for devs by devs.  Not talking about [https://help.ubuntu.com/lts/serverguide/ruby-on-rails.html](https://help.ubuntu.com/lts/serverguide/ruby-on-rails.html) or [https://help.ubuntu.com/community/RubyOnRails](https://help.ubuntu.com/community/RubyOnRails) - we both know never to use system versions of ruby for our custom webapp deployments.  We don't want to be stuck with only the versions provided by some packaging team.  That's why rbenv/rvm are so important to use.  [https://gorails.com/setup/ubuntu/14.04](https://gorails.com/setup/ubuntu/14.04) might be what you need.

Oh - and if there isn't a C compiler, your install won't work. Need to install the **build-essentials** package. I think that's the name. My link has the list of required packages before you attempt anything with this install. Those packages may have changed slightly since 12.04 - look for a gist on github for the rbenv and ruby version(s) you want.  Heard that github was under a DoS attack yesterday. Hopefully, that is ended.

BTW, kali should only be used for that purpose, not as a server or as a daily-use desktop. Many reasons for this.  Didn't they just have a fresh release - like 2 weeks ago? They switched to rolling release model, which means they probably switched from Ubuntu as the base OS.

Good luck!
