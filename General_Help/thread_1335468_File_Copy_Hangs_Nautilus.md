### Post by jcobban on 2009-11-23
I am trying to copy the contents of a folder on my Ubuntu system over to a Windows Vista system as a backup.

I opened two Nautilus windows on my Ubuntu system: one with the local folder, and one with the corresponding folder on the Windows Vista system.  The copy calculated that there were about 6,000 files to copy totaling about 3GB (mostly JPGs).  After copying 589MB the operation froze;  none of the three windows involved (the two folder displays and the "File Operations" window) responds to anything I do.

The only way I found to break out of this is to kill the Nautilus process and resume the copy using smaller subsets of the data.

I am running Ubuntu 9.10 - Karmic Koala with Nautilus 2.28.1.
