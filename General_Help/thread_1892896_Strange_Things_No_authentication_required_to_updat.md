### Post by airspoon on 2011-12-09
Hello, I'm running 11.10 64-bit w/ Gnome Shell on a C2D (E6600) with 4g RAM. I'm having some strange things happen with my Ubuntu machine, including the update manager, which no longer requires authentication before upgrading packages.

I'm not sure if its related or not, but another problem I'm having, is that I need to reboot about 2 times a day because Gnome is taking up too much resources (according to my conky readout). When I try to do alt+F2 > r to restart Gnome, it restarts, though lacking the top taskbar and all window taskbars (with the min, max and close buttons). This basically means that I have to literally log out (ctrl+alt+del) and then back in each and every time my computer starts to freeze, which is about twice a day on average.

Another issue, which I'm again not sure if it's related, is that whenever I type real fast, such as when I'm writing a paper for school, I get little pop-ups or other things, just as if I'm hitting keyboard shortcuts (which I'm not, or don't want to do). In fact, it's even happening as I type this.

So far, I'm having more issues with Ubuntu than I had with Vista, which is saying a lot because the multitude of issues and bloat in Vista is what lead me to completely abandon Windows.

However, the biggest issue to me, is the lack of authentication being needed in the update manager. I'm wondering; since I'm not required to authenticate in the update manager, then what else can be done willy-nilly, without authentication. Should I be worried about security? Maybe I have already been compromised?

Security is my biggest concern, so I'm really worried about it. The paranoia in me is making me think that my machine has been compromised, leading to the aforementioned problems. Maybe someone else has already authenticated, thus update manager is not requiring me to do so again. That's pretty scary... 

Thanks for any and all help!

### Post by Larkspur on 2011-12-09
Don't worry.  Since 11.10, the Update Manager no longer needs a password in admin accounts. It's not a sign that your install is compromised.
