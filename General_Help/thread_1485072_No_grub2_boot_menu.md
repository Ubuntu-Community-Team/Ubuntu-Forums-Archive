### Post by drs305 on 2010-05-17
The keystatus/SHIFT check is currently contained only in a conditional in the 30_os-prober script. So if the conditional isn't met, the SHIFT function isn't available.

As currently written in 30_os-prober, for the keystatus check, Grub 2 must think there is no other OS on the system. There are several additional conditional steps before the keystatus check portion of the script. If it finds another OS, the keystatus check won't be made. I've brought this up with the devs a few times but that is the way it remains.

You can force Grub 2 to *always* perform this check by disabling the conditional within which it resides or adding it to another script. Here is the keystatus section of the file:
>   if keystatus; then
    if keystatus --shift; then
      set timeout=-1
    else
      set timeout=0
    fi
  else
    if sleep$verbose --interruptible 3 ; then
      set timeout=0
    fi
  fi

Personally, if I had problems with the SHIFT key not working, I would insert this at the end of the /etc/grub.d/00_header file. You would add "CAT << EOF" before the first quoted line above, and "EOF" as the last line. Then run "sudo update-grub".

### Post by drs305 on 2010-05-18
> **Cariboo1938 said:**

As long as it doesn't have a tendency to drop. The files aren't especially large by program standards. And should something unexpected happen GRUB can be reinstalled via the LiveCD.

Open a terminal. Removing grub-common will also remove grub-pc (Grub2). Installing grub-pc will also install grub-common:

```
sudo apt-get purge grub-common
sudo apt-get install grub-pc

```

As you remove grub-common, you will be warned that you are removing the bootloader and unless you install something your system could be left unbootable. That is OK.

As it installs, it will show command lines which will either be blank or possibly contain "quiet splash". This is where special boot instructions would be added. Normally, accept the defaults by pressing TAB to highlight "OK" and then press ENTER.

When you get to the partitions, use the UP/DN keys to highlight the **drive** Ubuntu is installed on. Press the space bar to select (*) the drive. Do not specify any partitions, just the drive. TAB to "OK" once you have an asterisk next to the proper drive and press ENTER.

It should install and run update-grub. Hopefully the new install will solve your problems. If not, please rerun and repost a new copy of the boot info script RESULTS.txt.
