### Post by chessnerd on 2010-02-13
You can change your login manager if you want. Some people on here recommend using SLiM as the login manager, which you can get from Synaptic (or aptitude install slim) and it will replace GDM when it installs if you tell it to.

As for a command-line boot, I think you could just remove GDM and that would force you to boot from the command line, but I'm not 100% sure...

### Post by flippo on 2010-02-13
There are a great many scripts that are run at system boot, they are all in special folders in /etc/.  If you remove one, or just make it not executable, then it wont be run at boot time.  Therefore, to stop GDM from being run all you should have to do is ```
sudo chmod -x /etc/init.d/gdm
```Although, uninstalling it *should* have the same effect.
