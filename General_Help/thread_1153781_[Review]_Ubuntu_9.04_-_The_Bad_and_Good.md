### Post by TDK800 on 2009-05-09
Been using Windows for close to 20 years and been using 9.04 for a week now. Funny thing, when I was testing Ubuntu from time to time on Windows machine I always thought things looked better under Windows, so took screenshots of Windows before leaving for Ubuntu, now after a week of Ubuntu when I compared the Windows screenshots to Ubuntu again, things looked like crap on Windows, so it's all a matter of what you're used to.

Overall, after customizing and getting used to Ubuntu, I can honestly say it is better than Windows XP/Vista in every way! Here's my list of bad and good for both Desktop and Server:

The Bad of Desktop
==================
* Full Disk Encryption Error - on around 50% of first bootups unable to enter the disk decryption password and have to restart, sometimes even have to restart multiple times, other times the problem doesn't appear for multiple restarts

* Screen On/Off flickering - When I close the notebook lid and re-open it, the screen starts flickering on and off, sometimes closing the lid and re-opening it fixes it other times hitting the key combination to minimize all windows and show desktop fixes it (Dell Vostro 1510 with Nvidia drivers installed). Probably problem with the Nvidia drivers?

* Gnome Commander problems - not sending files to trash on delete, it's on GC's to do list of fixes, but not much development there lately. Sometimes not able to go to root mode and edit files anymore with F4 after some usage. Restarting computer fixes.

* Editing menus - unable to select multiple menu items for drag and drop or deletion, unable to move a whole sub-menu with it's items, unable to delete places and system menus.


The Bad of Server
=================
* Unable to run Root terminal - it's also been reported as bug long time ago already, still not fixed for some reason.

* Firefox auto-offline mode - when booting up with cable connected and then unplugging cable to develop website on localhost firefox always starts in offline mode and have to uncheck it everytime. It's also alreday reported as bug long long time ago, still no solution though.


The Good of Desktop
===================
* Absolutely all the complaints I had with the desktop in 8.10 have been fixed in 9.04. I was like "WOW! They've fixed EVERYTHING, even THAT." when I first tried it. :)

* Add/Remove - coming from Windows world, it's a strange sensation when You want some app and just click on add/remove, search, and install it in 5 seconds, for free.

The Good of Server
==================
* Apache source install was super smooth, much easier than on RHEL 5.2
* PHP source install also much smoother than RHEL 5.2
* MySQL also installed perfectly, all in all a lot less problems with LAMP than in Redhat Linux which claimed to be the #1 server Linux.
* Binding IP addresses easier than RHEL 5.2 - all in one file instead of separate files, is much better.
