### Post by djeikyb on 2011-01-24
**!!! Please, for the spite of Eris, backup any important non-system data you have first. If something goes wrong, like your hard drive dying all of a sudden, at least you'll have your home folder.**

For task of this magnitude, I like to use dd. Now, this is a simple, powerful tool. Misused, you could lose it all. But what it will do is copy everything, from the master boot record to the end of your partitions, to another hard disk. Because it copies at the block level, you don't need to worry about special files that might resist file copy programs like cp or rsync. Search and read a few guides on using dd to clone a hard disk first. Make sure you type your command precisely. Here are a couple links to get you started:

[http://www.howtogeek.com/howto/19141/clone-a-hard-drive-using-an-ubuntu-live-cd/](http://www.howtogeek.com/howto/19141/clone-a-hard-drive-using-an-ubuntu-live-cd/)
[http://serverfault.com/questions/4906/using-dd-for-disk-cloning](http://serverfault.com/questions/4906/using-dd-for-disk-cloning)


There are a couple other options that I've never used, but may be nicer for those disinclined to the command line:
**Gparted:** [http://ubuntuforums.org/archive/index.php/t-302717.html](http://ubuntuforums.org/archive/index.php/t-302717.html)
**Clonezilla:** [http://www.tuxradar.com/content/how-clone-hard-drives-clonezilla](http://www.tuxradar.com/content/how-clone-hard-drives-clonezilla)

### Post by djeikyb on 2011-01-24
Yes. 
Small -> Big == GOOD.
Big -> Small == FAIL.
Same -> Same == Should work (and I've had success), but what if your original disk is 100% full and the destination disk isn't quite to spec? What if it's actually slightly smaller by manufacturing error or by post-factory defect? I prefer in this case to at least go from Small -> Slightly bigger. However, I usually don't worry about this because I rarely have a original disk that is 100% full. So..don't worry too much about this.
