If it is not picking up the splash image, it is probably not finding the correct file. If you are using Grub 1.97 beta, you must carefully look at the "for i in" ... line.

Make sure the image is in one of the referenced folders, that there is a period after the image name, and do not include the extension (although it must be of the type designated with the {} ). A properly designated file in the 05_debian_theme file should look like this (with emphasis) and you should make sure the size matches your resolution:
>   for i in {/boot/grub,/usr/share/images/desktop-base}/moreblue-orbit-grub[SIZE="6"][COLOR="Red"].[/COLOR][/SIZE][COLOR="Red"]{png,tga}[/COLOR] ; do

It's a bit simpler in Grub 1.97 and later, where the line starts with "WALLPAPER" and is much easier to understand.

If you have problems, post your /etc/grub.d/05_debian_theme file.

There are two ways to "turn off" 30_os-prober:

1. ```
sudo chmod -x /etc/grub.d/30_os-prober
```
This removes the executable bit from the file, so it won't run when update-grub is executed. It can be reversed by changing *-x* to *+x*

2. You can add the following entry to /etc/default/grub (note is it not spelled exactly the same as the file):> 
GRUB_DISABLE_OS_PROBER="true"

In either case, run update-grub after making the changes.
