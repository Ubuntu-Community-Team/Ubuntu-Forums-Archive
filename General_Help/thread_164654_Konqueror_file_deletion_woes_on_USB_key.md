### Post by v4169sgr on 2006-04-23
Could someone tell me please why:

1. .Trash is being stored locally on the USB key so that the right hand mouse click delete option only moves the affected files to .Trash, not clearing the space on the key?

2. Performing the same operation on a directory [folder, if you will] fails with error that folder deletion is not supported by protocol trash?

This makes managing files and diretories on removable USB media much more of a pain than it need to.

Many thanks!

---

### Post by jeremy on 2006-04-23
If you open konqueror, then go to 'Settings', then 'Configure Konqueror...', under 'Behaviour' you will find the option; 'Show 'delete' context menu entries which bypass the wastebin'. Select it and Apply it, then you will be able to right click and delete your files, as opposed to sending them to .trash.
