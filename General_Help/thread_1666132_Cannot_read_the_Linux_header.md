The error you get
```
error: cannot read the Linux header.
```seems to indicate that the file "/boot/vmlinuz-2.6.32-27-generic" is either missing or corrupted.  If you have recently updated your system, a broken update could have caused this.

The message
 ```
 Failed to boot both default and fallback entries.
```indicates that grub tried to load both "Ubuntu, with Linux 2.6.32-27-generic" and "Ubuntu, with Linux 2.6.32-27-generic (recovery mode)" and failed.

I would try to boot into an older kernel and see if that works.  Boot the PC and on the grub menu, select an older kernel like "Ubuntu, with Linux 2.6.32**-26**-generic".  If you don't see a Grub menu while booting, reboot and hold down the shift key while the PC is booting up.
