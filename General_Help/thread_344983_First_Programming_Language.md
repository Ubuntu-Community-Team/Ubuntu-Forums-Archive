### Post by matthewstory on 2007-01-23
Python would be a bad start.  It is my favorite language, but it won't help you with many other languages, and it's pretty syntatically unique.  Pascal is dead, and sytactically wierd as well, so really really don't start there.  I'd start with C++ only because this is the mother of all object oriented languages (like python for example) and if you know C++ it's easy to learn Java, Javascript, Python, PHP, etc, etc, etc.  Plus C++ is what I learned on. Maybe start with javascript, because it's easier than C++ and very similar syntactically and conceptually, this might ease you into objects and the like.

### Post by kebes on 2007-01-23
This question often comes up and isn't really easy to answer. I mostly agree with the things matthewstory said.

I would say if you're planning on taking a class of some sort, start with C++ (or Java), since this will force you to learn all the "proper" techniques of high-level languages, how to write object-oriented code, etc. After learning something like C++ you will be able to quickly transition to whatever other language you want (Python, PHP, C, etc.). This is certainly a good way to go.

However if you're really planning on learning on your own, in your spare time... then my answer is slight differnt. When learning on your own, the most important thing (in my opinion) is sticking with it. In that case C++ might be a frustrating way to start, since you won't be able to create anything "cool" or useful right off the bat. It will take a long time before you can code anything that actually helps you get work done.

By comparison, if you learn something like Python, PHP, or even bash scripting, you'll be able to create neat, useful programs almost immediately. This can be very satisfying and will keep you interested, so you will learn faster. And when you want to transition to other languages, you'll be able to with some effort.


So my advice is: find something that you want to "make work" and use this as a focus while learning your first language. The language you pick will of course depend on the task you want to tackle. (System automation? Learn bash. Creating neat web reports? Learn PHP. Like 3D graphics? Learn povray. etc.).

If you're really willing to put in the time and put up with the frustration, C++ is a good choice.


Hope that helps. Good luck! Programming is tons of fun!

### Post by matthewstory on 2007-01-24
I was just rereading this post, and i reread the first sentence:

I want to learn linux and i want to learn programming.

Just as kind of an aside, these are really two completely different things, unless you're learning a scripting language like BASH or PERL or Python.  I'm a linux administrator who codes a bit, and i work with a bunch of programmers who are very good at writing code, but who are utterly lost when it comes to actually using linux.  These skills are completely seperate, but often are confused as being somehow related.  So I would propose that if you want to learn linux, learn BASH scripting, and pick up a book specific to Ubuntu (or Debian, as they  are basically the same).  And for the coding side, again I would have to say C++, though kebes makes a good point.  All programming languages end up doing certain things well, and so the choice of a language for a programmer is made based on what the program will have to do.  So what you can do with a language is largely limited, and for learning, you'll want to go with something that turns you on.  So if you want to do cool web applications learn PHP or Javascript, and if you want to write automating scripts learn BASH or PERL, and if you want to write regular programs and stuff learn C++.  Anyway, that's 2 more bits from me.
