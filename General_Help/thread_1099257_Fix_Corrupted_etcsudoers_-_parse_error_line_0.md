### Post by fieroboom on 2009-03-18
You need to reboot, and watch for the screen to say "Press ESC to enter GRUB menu" (or whatever the heck it says... ;) )
Anyway, press Escape, and at the menu, select recovery mode and boot into it. It will drop you into a root prompt, and you can then edit /etc/sudoers.
However, and please don't take this the wrong way... If you messed up that file, then chances are good that there are other critical files messed up, unless it's the result of you purposely editing the sudoers file.

Did you recently run chown or chmod on anything, or did you just edit the sudoers file?
Once you drop to the root prompt, you won't have the benefit of internet research (unless you install & learn to use lynx/elinks for text browsing), so be sure you have all your data in order before doing so.
Without seeing your sudoers file, it's impossible for me to say exactly what's wrong, however, if you'd like more precise guidance, you can:
- reboot,
- go into recovery mode,
- run this command: ```
cp /etc/sudoers /home/<your_username>/sudoers.txt
```
(the above command copies the info from /etc/sudoers to sudoers.txt in your home directory)
- then run this command: ```
chmod 666 !$
```
(the !$ means "last argument", so the above command will place global read/write permissions on the copied file so you can view it later without sudo)
- reboot again, this time normally
- open /home/<your_username>/sudoers.txt and copy/paste it here

That's a lot of stuff to do, but by allowing us to have a peek at the contents of your /etc/sudoers file, we can precisely guide you on what to fix so you don't feel lost at the root prompt in recovery mode (it can overwhelm you quickly... ;) )
:D
-Paul

### Post by geirha on 2009-03-18
The name in /etc/hostname **must** match the entry for 127.0.1.1 (yes that's .1.1 at the end) in /etc/hosts. Boot into recovery mode and and ensure that those matches.

```
$ awk '/127\.0\.1\.1/{print $2}' /etc/hosts ; cat /etc/hostname
```
