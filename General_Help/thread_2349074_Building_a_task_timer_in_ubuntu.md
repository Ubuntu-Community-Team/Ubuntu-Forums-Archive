### Post by TheFu on 2017-01-11
I used a spreadsheet to track project-based billing.  Liked it because it does in/out time math.
Mine had 3 parts.
a - Weekly simple in/out for arrivals to do total daily time at work.
b - Weekly project codes for billable hrs
c - Monthly summaries (by pay date), so I could validate the payments matched.  Since the client paid net+90, it was a long time coming.

A and B help validate each other. If the math doesn't add up, there is a mistake.

Realize this isn't what you hoped to find and the spreadsheet is manually tweaked for almost every week. Zero macros. Just sharing another option. I had coded some C++ that attempted to do the same thing. Only got the time in/out working before I abandoned it.
