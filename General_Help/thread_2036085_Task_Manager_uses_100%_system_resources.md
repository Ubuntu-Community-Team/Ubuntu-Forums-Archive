Hi all,

This is a weird one....

I tested an old usb video camera before sending it to a family member for them to use with skype. It worked fine in a brief test with cheese, I made a short video. When I tried to use it for a google talk session, I heard a loud pulsing high pitched audio tone during the entire chat. A few seconds after hanging up, the tone stopped by itself.

However, the computer ran real slow and I discovered the task manager was using 100 percent of the processor!! I attached a screen capture.

I unplugged the camera, no change. I closed firefox, no change. I uninstalled cheese (I had installed it just minutes before in order to test the camera). I restarted the computer, no change. I logged out of my account and logged in to the super users account and found that the task manager was still using all the system resources.

So, I downloaded system monitor using synaptic to make sure I had all the dependencies included. System monitor showed the same type of behavior.....with only system monitor running, it was also using 100 percent of the system resources.

I'm running Xubuntu 12.04 in an old P4 desktop system (MSI motherboard, 1.5 GB of ram).

I have been getting crash reports, which just started when 12.04 was installed. I am prompted to click on the icon in the top panel for more details, but nothing happens when I click there. I've been using this computer for quite awhile, and it doesn't always shut down by itself. I can hear the harddrive park, but it doesn't power down.  I have to just touch the power button, and it shuts off instantly tho.ugh. I've always regarded that problem (which is intermittent) to be an annoyance though.

I'm going to mark this one as solved.

Problem turned out to have nothing to do with the new software, the video camera test or the odd behavior of the video cam after it was hooked up.

After lots of searching, a reinstallation of the task manager fixed the problem.
