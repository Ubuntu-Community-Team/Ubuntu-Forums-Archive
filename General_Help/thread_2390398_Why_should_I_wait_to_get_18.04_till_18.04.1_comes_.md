### Post by TheFu on 2018-04-28
There are issues that are spelled out in the release notes. But ... testing is never as complete as what 2M users will accomplish in a few months of daily use.

For example, 18.04 has issues with upgrading with LVM according to the release notes. NOT using LVM is a non-starter for me. It is simply too important for my needs.   There are other issues.

If you have a system that is stable and working for you, there isn't any harm in waiting.   If you want to peak at this new release, try it inside a virtual machine.

Everyone is a little different. I will be waiting until June before loading 18.04 on a single test system.  Perhaps I'm more cautious than most?  I didn't upgrade my 14.04 servers to 16.04 until last Feb, so not being in a hurry is something that works for me. Even waiting almost 2 yrs, a few of the systems did have upgrade problems.

### Post by TheFu on 2018-04-28
> **pretty_whistle said:**
> That makes me wonder why they offer it on their page if it's not ready, and it sounds like it's not really ready. Weird.

The release month was fixed 5+ yrs ago. We know that 18.10 will be released in Oct and 19.04 will be released in April next year.  Software developers are optimistic people and routinely underestimate how long things take.  I don't work for Canonical, but I was a professional software dev for 15+ yrs. It was only the last 5 yrs where I learned to triple all my estimates. Getting it "working" can be hard, but making it safe for end users is much harder, especially when it is something that developer has never done before.

If you were paying $120 for each release ... call customer support.

So ... June is my plan for 18.04 on 1 test system.

As for GUI interface stuff, different interfaces have different customization.  fvwm lets you set the size, width, color for almost everything you can see. I have no idea what gnome supports, but there should be a document somewhere.
