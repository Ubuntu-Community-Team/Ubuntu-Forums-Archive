### Post by Mark Woodward on 2008-04-21
Hi Sn3ipen,

You'll possibly ignore this, but I would say learn Vim or Emacs. Either one. It really doesn't matter. Both are extremely powerful and will probably do everything you'll ever need a text editor to do. 

I've never used it, but I think I can confidently say either will out perform notepad++. Yes you could learn notepad++/UltraEdit/NoteTab Pro etc in a fraction of the time. But with just a little bit more effort you'll have an editor that:

 (Vim bias as that's what I use)
  - runs on Windows/*nix and Mac.
  - can be used for PHP as well as hundreds of other filetypes.
  - Can utilise *nix utility programs with a simple :! (probably means nothing to you ATM, but it will)
  - Many, many plugins. (Extensions that users have created because they saw the need).
  - scriptable using vimScript/Ruby/Python/Perl (Vim) or eLisp (Emacs) - that's a BIG advantage   
    over notepad++/UltraEdit/NoteTab Pro etc
  - can edit remote files
  - etc, etc, etc (honestly, could spend next week describing the benefits)

Yes, they're harder to learn. Is it worth it in the long run? Absolutely!


cheers,

-- 
Mark
