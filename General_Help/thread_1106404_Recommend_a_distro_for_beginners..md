### Post by Topsiho on 2009-03-26
I choose (of course) Ubuntu, though I'm not sure that it now still is the right choice. Before I hit on Ubuntu 4.10 (the first release, the very first one that connected me to the internet without any fuss), I played with Mandrake (now Mandriva) (which was crippleware if you only used the internet version, but was very easy to work with generally, and very nice). When I discovered the crippleware aspect I changed to Fedora, much less beautiful, but still nice to work with. Only: the updates were very slow coming in, and it did not work sometimes when there were more than a few of them. Just didn't work ....
I also tried SuSE, very beautiful, but the dependencies were treated very badly. I remember that only too often installing a package was not possible due to dependencies, installing those dependencies were not possible due to their missing dependencies, and so on.
When I discovered Ubuntu I was quite amazed: the internet worked, no crippleware in it, the updates coming in real fast, and even an upgrade, with hundreds of updates in it work very smoothly.
But as Ubuntu developed from release 4.10 (Warty Warthog), I guess Mandriva and Fedora have developed over all these years too.

Only thing in Ubuntu which drives me nuts (being a translator) is that the translations are often meddled with by some idiots or by a defective system. Sometimes very obsolete translations pop up suddenly (and now quite expectedly).

I recently discovered Debian 5.0 (Lenny), and saw that translations are not being tampered with. Very very nice. I am not sure yet, however, of this. But it might be that I, for this very reason, will change to Debian.
When one is used to Ubuntu, one can easily use Debian!

Only thing with Debian: it uses the normal root, with a root password, just like all other distributions except Ubuntu and it's derivatives. Though I first hated the sudo-ing in Ubuntu, I have realized it is perfect. The realization is now already long standing, but was enhanced after I discovered that in Debian I su-ed to root, and forgot for quite a while to exit again to a normal user. That's of course very very bad.

Topsiho

### Post by Simian Man on 2009-03-26
> **Topsiho said:**
> I am sure you have to agree that using sudo or using su are quite different things, as you yourself confirm that you have to do additional things to su to make it work like a sudo.
When using sudo you are only root for this one command.

Not to derail this thread (and I hope I'm not coming across like a jerk) but if I want to execute 1 command as root, I either use 'sudo' or 'su -c' depending on if I'm on Ubuntu or another distro.  These both execute only 1 command as root.

If I want to execute several commands I either use 'sudo su' or 'su -' depending on if I'm on Ubuntu or another distro.  These both allow several commands as root until you logout.

My point is that the fact that, by default, sudo only executes one command isn't really a reason it's better than su.  You can do both ways with either command.

The real argument is convenience and what's more of a risk: having a root account or having regular users have administrative rights.
