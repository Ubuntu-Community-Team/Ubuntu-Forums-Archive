### Post by jamieb22 on 2010-01-08
Hi

I've been using Ubuntu for a about a year now and find it a frustrating experience to say the least. There are so many niggly issues that are a major frustration and a serious dampener on productivity. 

For instance, whenever I click on a file to download and install in Firefox, something goes wrong. I tried to download and install Google Chrome deb package, Firefox reports "unknown error". I had to navigate to the /tmp directory and click on the file to install the package. Why the heck won't it open from the Firefox download manager? Surely, the use of Firefox is a very major aspect of Ubuntu use. Why then does the download functionality not work out of the box? I have to go tearing through reams of technical pages to figure out how to resolve this problem. This is symptomatic of the whole Linux experience. We are told to move over to Linux, but as a user we spend our day mucking around with stuff that should just work.

I've done by part to submit many bugs to the One Hundred Paper Cuts project, but progress on fixing the bugs in there seems painfully slow.
I guess Ubuntu is free software, so we should not expect too much from it.

### Post by mcduck on 2010-01-08
Just download the file to your computer and click it (or install using CLI as suggested above). The downlaod functionality in Firefox works just fine, opening files directly from Firefox doesn't if Firefox doesn't know what program it should use for each type of file.

The same happens on all operating systems. It's just a web browser. And downloading programs from internet using your web browser really isn't the normal way for installing application in Linux... :)

So, no need to "tearing through technical manuals". Just right-click -> Save Link As and your problem is solved.
