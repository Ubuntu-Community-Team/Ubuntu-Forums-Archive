### Post by delerious2 on 2016-11-06
I booted up my Windows computer using an Ubuntu 16.04 LTS Live CD.  Then I plugged in a brand new hard drive that I had formatted as NTFS into one of the USB 3.0 ports.  I tried copying a 300 MB file from the internal hard drive onto the external hard drive, and it worked fine.  But when I tried copying a 4 GB file to the external hard drive, I got a "Error splicing file: Input/output error".  I tried a few more times and kept getting that error.

I searched on the internet about this error and found one guy who said he got the error with a USB 3.0 port but not with a USB 2.0 port.  So I unmounted the drive and then plugged it into my computer's USB 2.0 port.  Then copying the 4 GB file to the external drive worked fine.  Then I deleted the file from the external drive, unmounted it, and plugged it back into the USB 3.0 port.  I tried copying the 4 GB file over again, and suddenly it worked.  I deleted the file and tried copying it again, and it worked again.

Any idea why I kept getting the "Error splicing file: Input/output error" when I first connected the external hard drive to the USB 3.0 port, but then I never got the error the second time I connected it to the USB 3.0 port?
