```where i is an index that changes with the for loop.
### Post by Vaphell on 2012-02-06
```
#!/bin/bash

i=0
for f in *.txt
do
  (( i++ ))
  echo == "$f" ==
  num=$( grep '^SOL' "$f" | sed -r 's/.*#([0-9]+)#/\1/' )
  if [ -z "$num" ]; then echo no num found; continue; fi
  newnum=$(( num-22 ))
  echo num: $num '->' $newnum
  sed 's/SOL #'$num'#/[COLOR="DarkGreen"]SOL #'$newnum'#\nNA+22\n#ifdef\n#some crap (i='$i')\n#endif[/COLOR]/' "$f"
done

```
### Post by Vaphell on 2012-02-07
you can skip printing to terminal and use **sed -i ...** or **sed ... > "new_$f"**
and if you really want to print to terminal and to file at once, you can use [B]tee
[/B]

```
*command* | tee *file(s)*
```
*tee* takes stdout of <command> and prints it to all listed files AND to standard output

edit: i'll check what's the problem

i assumed there are #'s in that SOL line  and because of that num is not properly extracted in num=$( ... ). I'll modify it to be more robust
strip all non-digit chars
```
num=$( grep '^SOL' "$f" | sed -r **'s/[^0-9]//g'** )
```
removed #'s and added start and end-of-line markers (^$), just in case
```
sed 's/^SOL **'$num'$**/SOL **'$newnum'**\nNA+22\n ..... /' "$f" | tee "new_$f"
```
as mentioned earlier, tee will write to screen and file simultaneously
```
