### Post by Mark Phelps on 2010-10-30
Going against the flow here ...

Being new to Linux, you're asking for a lot more trouble than you will be able to handle.

Consider some specifics:
1) Support -- you said they don't have any themselves, and you admit to being new yourself, so you're putting these folks into the position where, basically, you are having to rely on this forum to support them -- bad idea.  Why? because a problem that can be solved by an expert here having hands-on support to a machine in a few minutes is likely going to take HOURS to support remotely via a forum.  That's no reflection on Ubuntu or on your setup -- just reality.
2) MS Office -- NO, it is NOT a simple matter to simply install Wine and start using MS Office.  You specifically mention Access --  which does NOT run in Wine.  Period.  So, whatever is relying on Access and VB is NOT going to work anymore once you switch over.
3) NFIRS -- Did a search on the WineHQ site and this is not even listed, meaning, most likely, it too is NOT going to work in Wine.  So, that's a second thing your folks need that they are not going to be able to use once you switch.

Look, these folks are trying to do THEIR jobs, you're trying to force an experiment on them.

If you really want this to work, then do the following:
1) Pick one machine -- the one that is LEAST critical to their work -- and setup it as a trial machine to test out the conversion.  
2) Install everything you need on that test machine.
3) See how well things work -- the critical apps first, other things second.

If the test works out OK, then set up each of the other machines as a dual-boot setup -- being sure to leave the MS Windows install intact and working.  This way, you can resolve any problems on each of the other machines in getting Ubuntu working, and the folks can still use these machines in XP to do their work. Once each machine is working OK, you can change the dual-boot setup to boot into Ubuntu by default.

As to VM, that takes even more resources than running native.  So, if the machines are really slow running XP native, they're going to be all but useless in VM.  And besides, using VM requires that you do a complet OS and App reinstall on each machine -- did you known that?  So, you're basically going to have to reinstall each machine from scratch -- the OS, the apps, all settings, all files --just to use VM.

claven123, I'm sensing some frustration in your post. I can't see how Ubuntu wouldn't be a great thing to try. Just take it slowly and do as an earlier poster advised, use the Live CD or USB approach to see how it works with the hardware. Then try one machine, set it up for them and let them try it out. 

When their Windows machines become slow, or infected, or problematic in some way, while the Linux system always stays clean and stable and fast, they'll soon see the differences.  

In the meantime, suggest they try Firefox, and OpenOffice (or LibreOffice when it's ready) on their Windows machines, I've installed OO on lots of Windows machines right alongside MSOffice, with no problems. After folks have gotten used to programs that work on both Windows and Linux, the operating system they choose becomes less important (and that's as it should be, really).  

The funny thing about getting people to try new things is that you really can't predict how they will handle it. The ones you think will complain the most, might surprise you and take to the changes with no problems.  The ones you think will be easy to teach might end up being horribly annoying...there's no way to know for sure until you try...

And I believe it's definitely worth a try. 

So, don't let anyone discourage your efforts.  Just be patient and take things one step at a time. And don't forget to let us know how it goes.
