### Post by the_camera_man on 2010-11-26
This is my first post in this forum, so thank you for taking the time to read this and possibly throwing some advice my way.

I have recently installed Ubuntu 10.10 64-bit on a custom built desktop.  I am using MythTV with the HD-PVR, along with ZoneMinder for security cameras, and CUPS for interfacing with Windows printing.  Thus far, things have been very smooth and I am enjoying the transition from XP and BeyondTV.

I have hit a little snag with lirc.  I have decided not to try to use the Hauppauge remote with my setup, rather I purchased a Pinnacle MCE transceiver to perform remote functions and IR blasting.

I have followed a number of rabbit trails by doing anything from compiling lirc from source, to installing from the repositories (apt-get), to simply trying to use inputlirc.  None of these options appear to be getting me where I needed to be.

My current status is:
1) The device is recognized by viewing lsusb, lsinput, etc. as /dev/input/event3
2) When I run "sudo evtest /dev/input/event3" and start pushing buttons, I get output on the screen that corresponds with the button I press, so the remote itself appears to be functioning.
3) After installing lirc from the repository (apt-get install lirc) and without the lirc-modules-source installed, I start up irw and very few buttons respond.  Things like volume and power actually perform system events, while things like Up,Down,left,right produce output like "]^]D" and "]^]C" ... One thing to note is that upon installing lirc, I choose "Linux Input Layer" for the remote, but the lircd service will not start until I modify the hardware.conf file and put /dev/input/event3 as the remote_device, and then restart the service of course.  Is that normal?
4) I have tried by using the lircd.conf file that is specified by default, and I have also tried the lircd.conf file from sourceForge.  They seem to be the same file, except one specifies the key values in hex and the other in decimal.  I have heard however, that the lircd.conf file is usually the reason you get garbage from irw.
5) Finally, irrecord does not function for me.  I try "sudo irrecord -H dev/input -d /dev/input/event3 ~/lircd.new" and I receive "driver 'dev/input' not support -- supported drivers: default' ...

I have spent a few hours trying to shake this booger loose, but I am running out of fresh ideas.  Has anyone hit this before?  If I need to provide any of my config files or any other pertinent information, I will be more than happy to.  Thanks for your time!

### Post by liamgh on 2011-01-16
Hi,

I ran into a problem where my MCE remote stopped working after an upgrade to 10.10. After lots of possible solutions I found this:
[http://ubuntuforums.org/showthread.php?p=9984844#post9984844](http://ubuntuforums.org/showthread.php?p=9984844#post9984844)

Option 2 seemed to work for me, I did this:
Added a new line to the end of **/etc/modprobe.d/blacklist.conf**:
```
blacklist mceusb
```

Installed the **lirc-modules-source** package then rebooted. 

After the reboot ran:
**sudo dpkg-reconfigure lirc** and selected "Windows Media Center Transceivers/Remotes (all)" from *Remote Control* configuration. 

The remote seems to now work when I test it with the "irw" command. 

Hope this helps, it was a frustrating problem!
