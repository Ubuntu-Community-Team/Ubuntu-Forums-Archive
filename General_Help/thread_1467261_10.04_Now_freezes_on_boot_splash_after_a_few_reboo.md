### Post by GSF1200S on 2010-04-30
Ok, so lets kick this off:

Mount /dev/sda2 in /mnt:
```
sudo mount /dev/sda2 /mnt
```
chroot into the partition mounted at /dev/sda2:
```
sudo mount --bind /dev /mnt/dev
sudo mount --bind /proc /mnt/proc
sudo mount --bind /sys /mnt/sys
sudo cp /etc/resolv.conf /mnt/etc/resolv.conf
```

Then, lets remove the offending Nvidia drivers:
```
sudo apt-get update
sudo apt-get remove nvidia-190 nvidia-190-modaliases
```

If you were to boot know, the kernel would load and would use the vesa driver (if not automatically, it would allow you to start low-graphics mode where you could at least get back to the desktop and reenable the 195 series).

You can try the below, but I havent tried installing Nvidia drivers via a chroot jail, so Im not sure if this part will work:
```

sudo apt-get install nvidia-current nvidia-current-modaliases
sudo nvidia-xconfig
sudo update-initramfs -u
sudo update-grub2
```

Then unmount the partition (for good measure):
```
sudo umount /dev/sda2
```
and restart. With any luck, it should work. If it doesnt load X with the nvidia drivers, you should at least get a low graphics mode and be able to go back to 195 for now..


and navigate to that location. Then, navigate to (within the mounted directory):

and change:
```
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"
```

(or whatever it has in it) to:
```
GRUB_CMDLINE_LINUX_DEFAULT="nomodeset"
```
