### Post by grahammechanical on 2012-02-23
My Asus motherboard came with some Windows utilities but Asus did not provide Linux versions of this software. It is the same with you and Lenovo. 

If Lenovo does not provide Linux utilities on a CD or on its website then you will not be able have use of those features that you want. The responsibility to provide these things rests on Lenovo.

For example, Asus has provided a utility that will let me upgrade the BIOS from within Windows. But it has not provided a utility to let me do that through Linux/Ubuntu.

I have a Nvidia graphic card and in Ubuntu I run a utility called Additional Drivers which loads and installs the latest Nvidia video driver for me. It comes with a Nvidia settings utility.

I am sure that you could do the same with your ATI Radeon graphics card. Review some to the other posts on this forum. You will see people talking about their ATI cards and their Catalyst drivers.

Please be specific about what these Lenovo software utilities do. You may find that these things are already taken care of in Linux/Ubuntu.

For example, I do not need the Asus audio utility because that is taken care of in Linux as standard. I also use a utility to measure motherboard fan speeds and temperatures.

Regards.

### Post by greenpeace on 2012-02-24
I have a dual boot Lenovo E420s (Windows7 / Ubuntu 11.10) and I would suggest that is the way to go for you.

All the fn keys work well, as did the keyboard light and wireless (check these yourself with a live disk)


The only real issue I have had is with the discrete graphics card (AMD Radeon HD6630).  

I cannot use this in Ubuntu at all, there is pretty patchy support for discrete/switchable cards across linux.  

This isn't actually a big deal for me, as the CPU runs the Unity 2D desktop with no issues, but if you were looking for fancy graphics/desktop effects or to run games you might be disappointed.

that said, this guy seems to have had some joy with it: [http://askubuntu.com/questions/96063/ubuntu-doesnt-detect-my-ati-vga-card-only-intel-using-lspci-grep-vga-s](http://askubuntu.com/questions/96063/ubuntu-doesnt-detect-my-ati-vga-card-only-intel-using-lspci-grep-vga-s)
