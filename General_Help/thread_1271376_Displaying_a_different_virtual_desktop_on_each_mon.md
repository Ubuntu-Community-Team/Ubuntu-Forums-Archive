### Post by King_Critter on 2009-09-20
I just bought a second monitor, and half been trying for the past few hours to set them up the way I want.

Here's the deal: I've got a CRT and an LCD, using VGA and DVI, respectively, on a GeForce 6800. I'm using the propriety drivers. The options I see are: 
[LIST=1]
[*]Separate X Screens
[*]Separate X Screens with Xinerama
[*]Twinview
[/LIST]

The problem is that none of these work for me. Oh, they do what they're supposed to -- I just don't like it.

What I want is to display a different virtual desktop on each monitor.

Example usage scenario: I have four virtual desktops. Monitor 1 is displaying the first desktop, while monitor 2 is displaying the second. A chat window pops up in virtual desktop 1 -- and, thus, monitor 1. I drag it to the right, onto virtual desktop 2 -- and, thus, monitor two. So far, this can be easily accomplished with twinview. But here's where it gets tricky: I want a separate taskbar/window list for each desktop, so when I move the chat window over it disappears from the window list on the first desktop. I also want to be able to switch to another virtual desktop independently. So I could switch to virtual desktop 3 on my first monitor, while still displaying desktop 2 on my second. Still with me?

So, the problems with the current options: twinview/xinerama is good because it lets me drag windows from one to the other. But it's bad because it's still on the same desktop -- meaning it takes up room in the window list, and when I switch desktops moth monitors switch. One workaround, at least for the second problem, is setting the windows on the second monitor to sticky mode, meaning they appear on all desktops. This is a bit better, but that still means you can't switch the second monitor to a different desktop. 

With separate X screens, it's good because you can switch desktops independently, and they each have their own window list. Bad, because you can't drag windows from one to the other, and there's a few little random bugs. 

Well, anyway... thanks for reading that wall of text, and I hope someone can help me out.
