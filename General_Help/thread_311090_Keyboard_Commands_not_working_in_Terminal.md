### Post by Scottla on 2006-12-02
Hi,

Hoping this is a simple one.

I am getting accustomed to the Ubuntu environment. And, like in all Linux distros, the command line is pretty essential, as is good syntax.

When I come across recommended command lines, etc. I often "cut and paste" instead of re-key.

The problem is that the CTRL-C and CTRL-V commands do not work within the Terminal window.  So, for example, I can copy...

*sudo apt-get install build-essential*

...from a post here and put it in this window, but if I try to paste it into the command line of the terminal, it does not work.  Instead, I have to use the GUI interface menus to accomplish the paste.  (i.e. mouse to EDIT > PASTE).

Any idea why the keyboard commands are not working in Terminal, or is this standard?

Thanks for any help!

### Post by LLRNR on 2006-12-02
Copy the text you need and paste it in the terminal by pressing the middle mouse button.

OR right click in the terminal and select the "Paste" option.

LLRNR

Any idea why the keyboard commands are not working in Terminal, or is this standard?

It's standard, ctrl-c ends running program in linux CLI..

Instead you can use middle-click to paste, or shift-ctrl-c and shift-ctrl-v.
