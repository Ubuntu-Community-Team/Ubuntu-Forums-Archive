### Post by CantankRus on 2014-11-12
Apparently it's disabled because of security issues.
This is from the file **/etc/sysctl.d/10-magic-sysrq.conf** where you can re-enable.
> # In general, the magic SysRq key is not needed for the average Ubuntu
# system, and having it enabled by default can lead to security issues on
# the console such as being able to dump memory or to kill arbitrary
# processes including the running screen lock.
```
gksudo gedit /etc/sysctl.d/10-magic-sysrq.conf
```
and change the last line to...
```
kernel.sysrq = [COLOR="#FF0000"]1[/COLOR]
```
Reboot.

You can see if working by switching to a tty ctrl+alt+F1 and using reisub
Security issue is your decision.

PS Many people think you need to hold down all three keys...ctrl+alt+SysReq and enter reisub
You only need to hold down ctrl+alt, press and release SysReq then enter reisub while still holding down ctrl+alt
