### Post by flippo on 2010-03-05
I had the same problem myself, read a book and got all excited, tried to find some good C++ projects I could do, but my experience wasn't enough.  Try looking up some good algorithms (not sure how in depth your book was) but stuff like [Dijkstra's Algorithm]("http://en.wikipedia.org/wiki/Dijkstra%27s_algorithm") always seems to inspire programming ideas in me.  If that is too much for now its also good practice to try and re-implement some standard STL classes (vector, list, map, hash-map, string, etc.).

### Post by Agent ME on 2010-03-06
Learn how to use a source control tool (I recommend mercurial and git).

Take a look at open source programs to see how they work. Look at the commit history in their repository, and look at how specific bugs are fixed and how features have been added. Make your own tweaks to the code, and contribute anything you think is worth contributing.
