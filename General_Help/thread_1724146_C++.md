### Post by amauk on 2011-04-07
What you've written is source code
You need to compile the source to a binary executable

Install the build-essential package```
sudo apt-get install build-essential
```(or search for it in the GUI package manager)

Then, assuming your program's source is saved in a file called myprog.cpp
Open up a terminal, navigate to the directory containing your source file, and do```
g++ -o myprog myprog.cpp
```

To run the program, do
```
./myprog
```

### Post by amauk on 2011-04-08
Just a few points (I realise you're just starting out)

The cin.get(); is unnecessary
This will hang the program until it receives a key press
In the majority of cases, this is undesirable

Also, you want to specify an return value
The main function returns an integer```
[COLOR="Red"]int[/COLOR] main()
```
So add a return value at the end of your program

Common convention is to return zero for success (other return values can indicate some sort of error), so```
#include <iostream>

using namespace std;

int main()
{
  cout<<"HEY, you, I'm alive! Oh, and Hello World!\n";
  [COLOR="Red"]return 0;[/COLOR]
}
```

### Post by amauk on 2011-04-08
> **vgulavni said:**
> This is really good example for the C++. Thank you.
Well, there's a few other points I would make

I realise some of this won't make any sense (especially if you're following a tutorial and haven't got to certain concepts yet), but still

I dislike the "using namespace..." thing
This pulls an entire external namespace into your program's namespace, and is (in my opinion) extremely bad practise

If there's a function **foo** in the C++ standard library, using "using namespace std;" will pull that function (and all other C++ std functions) into your local namespace

This means you cannot have a function called **foo** in your program as it'll conflict with the std::foo

I know "using namespace std;" is used in most of he tutorials out there, but like the cin.get(), it's bad and can have undesirable consequences

You are calling something that is external to your program's native namespace, so it's my opinion that such calls should be explicit, using the
[COLOR="Lime"]namespace[/COLOR][COLOR="Black"]**::**[/COLOR][COLOR="Red"]object[/COLOR]
format

Secondly, instead of using \n (newline character) to terminate strings, use use the C++ standard library variable std::endl

You may be aware that Unix-like systems use \n (single newline character) for newlines, whereas Windows uses \r\n (carriage return + newline characters)

If you use std::endl, upon compilation std::endl will be translated into the system's native "newline" character sequence, making programs more compatible with different systems

So, in my opinion, the best way to write the above hello world program would be
```
#include <iostream>

int main()
{
  std::cout << "HEY, you, I'm alive! Oh, and Hello World!" << std::endl;
  return 0;
}
```

Welcome to the wonderful world of debugging
Where the program is doing exactly what you asked it to do, it's just not doing what you meant ;)

Try and step through the program, and see exactly what's going on at each stage

a = 6

Added to:4
b = 4

So far:10
e = a + b
e = 6 + 4
e = 10

// While loop start

// Loop 1
another3
d = 3

So far:13
c = e + d

c = 13

// Loop 2
another3
d = 3

So far:13
c = e + d
c = 10 + 3
c = 13

// Loop 3
another3
d = 3

So far:13
c = e + d
c = 10 + 3
c = 13
