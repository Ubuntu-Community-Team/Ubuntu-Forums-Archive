### Post by geo909 on 2008-05-31
Look, if you can get more ram without paying too much (sometimes old rams for laptops are expensive), this is always the better solution.

 If it's expensive, you can always install xubuntu. I have a laptop with similar specs and xubuntu ran very decently.

 How familiar are you with linux/ubuntu? If you are pretty familiar and have some time to spend, you can always do a a barebones installation with a lighter window manager like fluxbox, openbox, JWM, ICEWM etc. Or you can install Xubuntu and install a different window manager afterwards. Using light window managers may turn your old machine to a really fast one!

> Is there any problem with having older version?? Security threats or something???
No, I really don't think there are such issues.

> What's the useful of getting my laptop upgraded???

Well, that's your call, really! If you use it a lot, maybe that's the right choice.
If you have another newer laptop and you want your old one for experiments, then you better
don't spend money on this.

When memory is <512MB, upgrading the memory gives your machine a REAL boost. More than if
you had more CPU power.

What I would advise is to try Xubuntu first. If you want to use heavy apps, maybe
you should make a quite big swap partition. if you like Xubuntu, then keep it!

There are many other light distros. Some are VERY light, like Puppy Linux (I love it),
DSL linux and many others. If you want a lighter *ubuntu-based* distro, that is fluxbuntu. It is even lighter than Xubuntu, 
but it uses fluxbox which you may not like if you are new to linux.

### Post by geo909 on 2008-05-31
LTS stands for "Long Term Support" (I think!).
When you have an LTS version, it's guaranteed that you will
have security and other updates for a long time. Hardy Heron is also LTS and I think its support ends somewhere in 2011.

When 7.10 will no longer have support, the LTS 6.06 will still have for a couple of months!

Look [here]("http://en.wikipedia.org/wiki/Ubuntu#Release_History") for all the releases and end dates.
