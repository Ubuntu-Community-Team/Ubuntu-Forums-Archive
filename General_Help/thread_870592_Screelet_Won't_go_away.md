### Post by Saint Angeles on 2008-07-26
this has happened to me... which screenlet is it?

i found that if i checked the options, made sure "autorun on startup" (or something like that) is unchecked... then manually quit screenlets and start it back up.

sometimes if gnome closes on a session with screenlets running, config options arent saved. but if you save the options, close them all, and restart them... they seem to save better.
