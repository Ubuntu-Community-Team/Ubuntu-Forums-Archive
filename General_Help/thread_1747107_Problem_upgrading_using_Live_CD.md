### Post by rejinarudo on 2011-05-02
Thanks for asking.

Well, as far as I know, both "desktop-i386" and "alternative-i386" images are designed for the Intel x86 systems or what we know as the 32-bit systems.

You said that you were using an AMD processor, well, in your case, you might want to use the "desktop-amd32" or "alternative-amd32" if you're using a 32-bit AMD processor; "desktop-amd64" or "alternative-amd64" if 64-bit.

I recommend you to do a fresh install using that "amd" image to replace the Ubuntu 10.10 "i386" install.

### Post by Slim Odds on 2011-05-02
This has confused people for ages.

The i386 builds are for ANY x86 processor (32 or 64, Intel or AMD).

The amd64 builds are for ANY 64 bit processor, Intel or AMD.

AMD created the original 64 bit architecture and the name has stuck since then.

I'm not completely sure about the current release, but in the past the Live CD could NOT be used for an upgrade. The alternative had to be used and it's a text only interface.
