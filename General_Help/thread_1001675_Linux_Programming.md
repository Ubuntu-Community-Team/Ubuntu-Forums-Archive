I take it to mean knowing how to get things done

I suggest you set yourself goals
Decide what do you want to do
Then do it

Be that,
- Writing a shell script to parse a news site and output a ticker type feed to a box on the desktop

- Setting up a personal mail server so you can access your email from anywhere (desktop, phone, web cafe, anywhere else)

Whatever
pick something you want to do
and do it

### Post by cmay on 2008-12-04
if you want to be an true _expert _then i would say you need to know about shell scripting both bash perl and python and know a lot about how to set up servers and system administration. and then maybe if you want to be a really hardcore user start to make your own distributions and start coding c and write device drivers and start hacing the linuxkernel or something like that.

 but as an expert it means maybe just a expert in a  part of the linux system that you become a "expert" user in. that could be shell scripting or that could be server administration or sysadmin tasks. i would not pursue this goal however. i would be mighty satisfyied with knowing how to fix the things and problems that comes along when they do and  just be happy as common desktop linux user. why do you want to be an expert ? . if it is jobs then focus on the server side of linux. or take a certification course. red hat or novell offers them at prises i cant afford even i would think it was great to try out.

linux could be a life long job to become an _expert_ in but it sure would be fun. 
just learn what ever it is you want and then move on to the next challenge. 
good luck :)

Advanced Bash-Scripting Guide [quite recent and very good]
version:  	5.5
author: 	Mendel Cooper, <thegrendel(at)theriver.com>
last update: 	Nov 2008

----------------------------------------

The Linux System Administrators' Guide
version: 	0.9
authors: Lars Wirzenius, Joanna Oja, Stephen Stafford, and Alex Weeks
last update:  	July 2005

----------------------------------------

The Linux Network Administrators' Guide
version: 	1.0
author: 	Olaf Kirch
last update: 	March 1996

---------------------------------

The Linux Kernel Module Programming Guide
available versions: 	2.4 and 2.6
authors: 	Peter Jay Salzman, Michael Burian, Ori Pomerantz
last update: 	May 2007 (kernel version 2.6)

----------------------------------------

[http://seifried.org/security/index.php/Linux_Security](http://seifried.org/security/index.php/Linux_Security)
Linux Administrator's Security Guide (free book) 

-----------------------------------

A Great book, but not readily available for free, is
"Mastering Regular Expressions: Understand Your Data and Be More 
Productive", Third Edition by Jeffrey Friedl

----------------------------------

Try reading the descriptions, especially for Advanced Bash-Scripting Guide.
Writing scripts is a deep geeky topic and this, I think its 1100 pages,
goes from beginner to expert. Using regular expressions within
scripts is at their core, so another book, MRE, helps and augments ABS.
