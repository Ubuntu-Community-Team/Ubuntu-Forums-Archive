### Post by papibe on 2012-04-06
Hi dmeadows013.

The cron environment is very limited so I would start by adding full paths to all commands that are called assuming a path:
```
repo
lunch
make
```
(although 'make' should be OK because it is in /usr/bin/)

If that does not work, some of the scripts need the environment variables of a regular interactive shell. The simplest way I know is to 'source' the system and user's profile at the beginning of the script:
```
#!/bin/bash
.  /etc/profile
.  /home/youruser/.profile
...
```
Hope that helps, and tell us how it goes.
Regards.
