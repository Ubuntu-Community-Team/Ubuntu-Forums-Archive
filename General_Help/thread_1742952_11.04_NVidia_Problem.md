### Post by scampbell70 on 2011-04-29
I have 11.04 running on my Sony Laptop and have for a few weeks. It was a nightmare getting my nvidia drivers to work but heres what I did and it worked like a charm


 1) Blacklist the modules. Open the blacklist.conf file.

    $ sudo gedit /etc/modprobe.d/blacklist.conf

add the following modules to the end of the file.

    blacklist amd76x_edac #this might not be required for x86 32 bit users.
    blacklist vga16fb
    blacklist nouveau
    blacklist rivafb
    blacklist nvidiafb
    blacklist rivatv

Save the file and exit.

2) Remove all the nvidia* packages

    $ sudo apt-get remove --purge nvidia*

1. Go to [NVIDIA website]("http://www.nvidia.com/") and download the compatible driver for your graphic card series. Make sure its in your home folder (not downloads)
2. Reboot your Ubuntu.
3. Choose the &#8216;Recovery Mode&#8217; on GRUB.
4. Go to the bottom of menu list, choose &#8216;root console&#8217;.
5. Disable the Kernel Nouveau.
 #echo options nouveau modeset=0 | sudo tee -a /etc/modprobe.d/nouveau-kms.conf #update-initramfs -u
Reboot to root console again
 6. Change to init 3 
       #init 3 
7. Install the NVIDIA driver
       #sh NVIDIA-XXXXXX.run (easiest way is to type sh NV then hit tab) make sure you say yes to writing a new xorg and updating the system
8. Reboot your Ubuntu.
[LEFT][COLOR=#000000]
[/COLOR][/LEFT]

Also Nouveau drivers do work for me BUT only after I uninstall the Nvidia drivers and reboot. I had to go to additional drivers and remove nvidia then Nouveau worked. I have tried both and both worked great but I prefer the actual nvidia drivers for now.


Hope this helps. The steps come from a few different websites

### Post by heisters on 2011-04-30
Thanks for the tips, @ scampbell70. I tried something dumber and simpler, but along the same lines, and it seemed to work:

[LIST=1]
[*]At the grub menu, select recovery mode
[*]At the recovery menu, boot into FailsafeX mode
[*]You'll get some warning about no graphics support, Unity won't boot, you'll end up in classic mode
[*]Go to Administration > Additional Drivers
[*]Remove the "version current" Nvidia driver. It should already be deactivated, be sure to remove it.
[*]Re-enable the "version current" Nvidia driver. You shouldn't need to restart or anything before this step.
[*]It will download and re-install the Nvidia driver, presumably reinstalling the kernel modules and whatnot
[*]Restart
[*]Boot normally into Unity
[*]Enjoy
[/LIST]

Worked for me, at least.

### Post by cwwilson721 on 2012-01-26
> ***Zugzwang* said:**
> ... I have been searching for a while and found many complicated "solutions" to this problem (and a lot more simple non-solutions, such as simply booting into the older kernel), but your simple two commands went directly to the problem and fixed it. Wonderful.

Thanks!

So..some are asking "Why does this solve the Nvidia 'driver' issue?"

Let me try to explain in a 'simple' manner (I know this isn't complete, or exact, but will do for the purposes of this thread). Note: I'll just say "Nvidia" when I mean the proprietary Nvidia Driver:

In the 'old' version of Ubuntu that you had installed there were "kernel header files" that MANY programs use to access the kernel itself (Nvidia being one of them). Nvidia compiles itself against these, and 'inserts' a module (driver) into the kernel. When you boot the OS, it reads these modules, and to "save itself from tampering", the modules check if the header files it was installed with is still valid. Normally, the answer is "Yes", and the boot process continues. Eventually, the x server starts, and has a module in it that looks for the Nvidia module in the kernel (the xserver and the kernel modules are NOT the same). If it finds it, it goes on about it's business of booting into your GUI Desktop.

So far, so good. But what is causing this 'issue'?

When you 'upgraded' the Operating System, it installed a different kernel than what you had in your 'old' OS. However, Nvidia does not know this. When the boot process gets to the Nvidia module in the kernel, it finds that the kernel header files it was installed with are not the same header files that are currently "working", and as such, doesn't load. So when the x server gets to the point that the Nvidia module is supposed to load, it cannot, because there is no kernel module "active', and calamity results...

The "fix" described in the post mentioned resolves the issue by installing the correct headers and having Nvidia re-compile itself against them, thus 'validating' the kernel module, and in turn, allowing the x server module to work. As the person quoted said "...simply booting into the older kernel.." works, because the module thinks everything is 'normal'.

As it turns out, you can also have the same issue if you install a kernel update thru Update Manager. Usually, it goes without any issues, but this method will fix this if it does not boot to the GUI.

Now, for the second issue in this thread, that is worth mentioning:

"My Additional Drivers box says that the Nvidia Driver is activated, but not in use!"

Ignore this. The best way to check if your Nvidia driver is actually installed and working is to open a terminal, and type "nvidia-settings"

If the Nvidia settings box opens, and says you have Nvidia drivers, you're all set.
