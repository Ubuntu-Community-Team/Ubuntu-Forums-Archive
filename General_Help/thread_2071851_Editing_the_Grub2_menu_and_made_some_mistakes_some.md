### Post by Wim Sturkenboom on 2012-10-16
Put your backups of the files that you modified back and run _sudo update-grub_ again. Or the backup of _/boot/grub/grub.cfg_. That would be the easiest :twisted:

### Post by Wim Sturkenboom on 2012-10-16
You can edit grub.cfg and simply remove a complete entry; I've marked one entry below so you can recognize it. Don't make mistakes because you might end with a system that can't boot (which can be fixed with a liveCD).

```

### BEGIN /etc/grub.d/30_os-prober ###
menuentry "Windows Recovery Environment (loader) (on /dev/sda1)" --class windows --class os {
    insmod part_msdos
    insmod ntfs
    set root='(hd0,msdos1)'
    search --no-floppy --fs-uuid --set=root 9C6A07C66A079BE0
    drivemap -s (hd0) ${root}
    chainloader +1
}
[COLOR="Red"]menuentry "Windows 7 (loader) (on /dev/sda2)" --class windows --class os {
    insmod part_msdos
    insmod ntfs
    set root='(hd0,msdos2)'
    search --no-floppy --fs-uuid --set=root 64E8CE09E8CDDA06
    chainloader +1
}
[/COLOR]### END /etc/grub.d/30_os-prober ###

```

Note:
This is a temporary modification; if there is a kernel update, you probably have to go through the exercise again.

PS
My crystal ball told me that you don't have backups.
