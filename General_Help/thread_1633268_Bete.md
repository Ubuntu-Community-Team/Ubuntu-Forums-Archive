### Post by grahammechanical on 2011-05-06
Update manager should be telling that there is a distribution upgrade available. It should be 10.10 as you have 10.04 installed. There should be a button to click. As part of the distribution upgrade files made obsolete will be removed. You do not have to remove 10.04 before upgrading to 10.10. The installation process with remove it and all unnecessary kernels leaving just 2. 

The second kernel will be a 10.04 kernel. You may need to boot into that if you have problems with the new upgrade. This is why one 10.04 kernel will be left behind. It is wise to always have another kernel to boot into.

Once you have 10.10 installed, then Update manager will give you notice that a distribution upgrade is available for 11.04. It is your choice if you wish to upgrade further. You do not have to. You can wait. But when you do upgrade you will still have the latest 10.10 kernel. You can use Update manager to remove it. You should not try to remove the kernel that you are using. Perhaps this is why Update manager did not let you remove the 10.04 kernel, because you did not have a 10.10 kernel that you were using.

Regards.
