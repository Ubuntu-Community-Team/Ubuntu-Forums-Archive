### Post by TheFu on 2023-04-03
There are many integrations that don't happen when we install a desktop onto a server system.  This is one.
If it bothers you, install a desktop system from the ISO.

 [https://wiki.ubuntu.com/DebuggingPolicykit](https://wiki.ubuntu.com/DebuggingPolicykit)  Running almost any GUI program with elevated privileges is dangerous. There are just a few exceptions - gparted is the only one I can think of right now, but I use the shell and CLI for most admin tasks.

But if you really need to do it, use 
```
sudo -H /path/to/program
```

This will prevent root from owning directories and files in the user's HOME, which can have terrible results, like a login-loop that never works.

### Post by TheFu on 2023-04-04
> **digioso said:**
>  
I insalled the GUI because there are some applications (like Filezilla) that require a GUI. Yes, I could use CLI FTP, but... Filezilla is pretty comfortable. :)

Filezilla?  You don't need a full desktop just to run filezilla.  And any Linux file manager can do what filezilla does.  BTW, there are better transfer tools.  wget, rsync, curl.  If the client and server have ssh-key exchanged, then rsync will work over ssh-key authentication connections. Everyone should know how to use rsync and scp.  Before a filezilla GUI is displayed, you could already have completed the transfers ... or started them and gone for coffee, if they are long transfers.

Plain FTP should have been killed off in 2002. There's little use for it today. For sharing files with the world, HTTP/HTTPS are good when you don't need any credentials.  If limited sharing is desired, use sftp, which is provided by installing openssh (metapackage "ssh" on Ubuntu).  If the client machine has the "ssh" package installed, then whatever file manager it has will support SFTP - drag-n-drop.  The URL can change, I use Caja and it supports sftp:// URLs, but some others use ssh:// and others use fish://.  Just depends on the file manager.  Plus, with ssh authentication, remote tab completion for directories and filenames work.  These things are hard to type out, but I see so many new users not using tab-completion to get the correct paths into their commands and prevent dumb misspellings that it hurts to watch.

Perhaps you have a different use-case?  There's little need for a full DE on any remote system that nobody sits behind.
