### Post by coolethan on 2012-06-20
So a while ago I was trying to create a bootable flash drive of Ubuntu on my Mac. I went through all the documentation on the Ubuntu website and I could not get it to work. I did some research and discovered that the commands listed on the website were not in the proper order. Upon just rearranging a few small things within the command line it worked perfectly. 

Now, just a few moments ago I was installing a program through the Terminal with apt-get and some of the dependencies were not met. It then told me to run "apt-get -f install" which did nothing to solve any problems. I looked through the documentation on the Ubuntu website and it told me to run "apt-get install -f" and it worked beautifully. Yet another instance where just rearranging a single letter was all it took. 

But why? why would the terminal tell me to run a command that didn't work? Why did the Ubuntu documentation list a command that didn't work on my Mac? And why does the official Ubuntu documentation list commands that do and don't work? What's up with the inconsistency?
