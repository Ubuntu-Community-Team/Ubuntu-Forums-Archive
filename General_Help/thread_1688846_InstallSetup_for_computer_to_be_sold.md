### Post by ken_23434 on 2011-02-15
I have toyed with the idea of building some systems with Ubuntu to sell locally.  Most of the systems I see advertised in the local papers seem WAY over priced, so I thought I could make a few bucks by selling competitive systems.

However, I would rather send them out with Ubuntu on them.  Maybe the $100 savings over the other OS might get a few more people trying out Ubuntu.

However, I would rather have the system installed, but have it go through some of the questions from the normal install to set up the user accounts, computer name etc...  I would not want systems going with with usernames and computer names that I selected.

I have done some searching, and have found "how to's" for making your own install disk (custom packages already part of it, back grounds etc...) but that is not really what I want.

Plus, an install like this could then be an easy way for local computer shops to sell computers with Ubuntu.

Thanks for any help.

Ken

### Post by clgy15 on 2011-02-15
What about the using the Ubuntu Alternate Disk and selecting the OEM Option. It sets up everything during the install except the actual user accounts, and it lets you install anything on them before the user gets them!

### Post by Bucky Ball on 2011-02-15
Yep, this is definitely possible and maybe you should do a little more searching as this has been asked and solved on these forums a couple of times already.

From memory, there is an option for OEM install when you boot from the installation disk. Have a look at the options there. It installs the system but next time it is booted from the hard drive it asks all the questions you are requiring; user name, password, location, etc. The usual stuff. Then boots into Ubuntu, voila. There are machines already available with Ubuntu only and that would be how it is done. ;)
