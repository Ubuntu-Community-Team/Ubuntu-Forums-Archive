### Post by Abdul_Hakim on 2016-08-14
Hello everyone,

I recently upgraded to 16.04 and I'm having some issues with Conky widgets.

The first issue I'm having is most of them keep appearing in their own windows, rather than appearing as a desktop skin/widget like they're supposed to.

Example here:
[http://askubuntu.com/questions/812151/conky-widgets-are-opening-in-their-own-windows](http://askubuntu.com/questions/812151/conky-widgets-are-opening-in-their-own-windows)

I was able to fix this with the LSD widget by adding the following lines:

```
[COLOR=#111111][FONT=Ubuntu]own_window_class conky
own_window yes
own_window_type desktop

```

[SIZE=2][FONT=arial]However when I make sure these lines are on the Seamod widget, the widget disappears as soon as I click on the Desktop or launcher. I found these threads that address this, but then this causes the same issue as before where Seamod appears in it's own window instead of appearing like a desktop widget.

[https://ubuntuforums.org/showthread.php?t=1717351](https://ubuntuforums.org/showthread.php?t=1717351)

### Post by Abdul_Hakim on 2016-08-14
Wait I think I figured it out

```

own_window_hints undecorated,sticky,skip_taskbar,skip_pager,below

```

it said "undecorate" instead of "undecorated"
