I have a very simple question about Postfix (I wasn't really sure how to search for this problem)

Ubuntu 9.04 server, I only installed Postfix so I could use sendmail in Bash scripts.. no other reason

Now I notice that any output that is generated by cron jobs gets saved in a file like:
```
/var/mail/root
```

how do I turn off this functionality of Postfix? If I want a cron job to send email I will decide when and where it will send them :)

sorry for the lack of terminology

### Post by jobsonandrew on 2009-09-21
Not to worry, I found it in the end:
```
sudo vi /etc/postfix/master.cf
```
comment out the 'local' line with a hash:
```
#local     unix  -       n       n       -       -       local
```

solved!
