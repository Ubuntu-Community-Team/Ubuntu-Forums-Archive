### Post by omrsafetyo on 2007-11-09
is the drive SATA or PATA?

And then, which channel is the drive on?  e.g.  if it is IDE, is it on slave channel, or primary channel?  etc.

if you go to /dev

```
cd /dev
```
and 
say 
```
ls -l | grep brw | grep -v ram
```
(ram is regarded is a location in unix... the brw will show most drives, including cd, floppy, ram, and HDDs)

you should get a list with like 

fda1
hda1
hda2
cdrom1

and etc.

Your hard drives will show up as  sd[abc..][123...] or hd[abc....][123...]

it looks like your primary hard drive is serial (sda1, sda2, sa3)

your other harddrive, if serial will be sdb or sdb1 (or both), etc.

if it is parallel, it will be hdb or hdb1 (or both).

Once you have located it, you need to mount it.

Once you find its name.. repost.

### Post by perixx on 2008-01-03
Hi guys!

Have you tried the command ```
blkid
``` by chance?

Perhaps you'll get a more accurate output of what's connected to you system this way.
If you get sth. like UUID.... with your HDD (possibly '.../dev/sdb' if it's a SATA drive, secondary master) - 
you can use this info to write it into your /etc/fstab (remember, you'll need to use 'gksudo mousepad /etc/fstab', or whatever texteditor you use).
E.g., my mounted ntfs-partition entry looks like that:
> # UUID=3ED4DBB9D4DB721F 		  /media/hda1   ntfs    defaults,umask=007,gid=46  0       1
I'm not perfectly sure about the mounting options for ntfs-type partitions, but that might help you on your 'quest'.

Furthermore, you'll need to create a directory and mount the drive in question to itby hand, using a command like 
```
sudo mkdir /media/sdb1
sudo mount -t ntfs /dev/sdb1 /media/sdb1
``` 
and so on, assuming that 'sdb' IS your 2nd ntfs-partition.

Please note, that I used commands like for mounting linux-native filesystems (like ext3) not being the OS-partition (but that should be allright, if what you mount is only a data disk) and that you'll possibly need to add some parameters for NTFS....

I hope I could help a little with this :]

perixx
