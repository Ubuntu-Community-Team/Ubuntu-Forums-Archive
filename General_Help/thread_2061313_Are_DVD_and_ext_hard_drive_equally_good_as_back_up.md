### Post by VCoolio on 2012-09-22
Depends on what you need to backup and how important it is. An usb-stick (which is the same idea as ext hard drive) can be ok to backup an os. If you have data, a dvd can be useful and portable (unless in case of a notebook without drive), but in time it seems to deteriorate. So then an ext hard disk comes in, but it can crash and you need to use a portable file system. If you have imporant data, backup twice, like in a NAS or at least a RAID-setup. But multiple backups in one place are vulnerable e.g. in the case of a fire. So if you have VERY important stuff, you'll need to have copies at different places or trust the cloud with it (dropbox, github, cloud server, depending on size). 

I don't see why you should backup an os. It's soon outdated and reinstalling is simple. Just keep a sane backup of configs and data. Then with a live usb or live dvd available you can always be productive in case of a software breakdown.

### Post by oldfred on 2012-09-22
I do both, but my system is just a home desktop not a business system.

I regularly backup using rsync to another drive my /home and some other data partitions.

But I also backup to DVDs as a historical image. That way if I delete something or damage a file that gets backed up to my rsync backup, I have an older copy on DVD. I consider the DVDs vital for photos and personal data and make duplicates which I have off site.

With our business servers we ran RAID but that was more for speed & reliability as failing drives were replaced while running. But we did tape backups every night with a daily, weekly, monthly, yearly archived tape rotation. And some backups had to be stored off site.
