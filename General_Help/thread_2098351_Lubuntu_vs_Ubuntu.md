No, not really. Underneath Lubuntu is *exactly* the same operating system as all the other Ubuntu variants are, the only difference is the desktop envrironment and the applications included by default.

However you didn't mention what's causing your errors (kernel panics?). LXDE itsef, the desktop environment used in LUbuntu, isn't nearly as mature project as Gnome, KDE and XFCE are, so it could be less stable. Although I definitely haven't seen such issues myself. Also you are comparing two very different Ubuntu versions (9,10 and 12.04) so there could also be some issues related to your specific hardware.

### Post by efflandt on 2012-12-26
No doubt any 12.04 version of Ubuntu (including Lubuntu) is going to be more bloated than 9.10, especially a live system running on 512 MB.  And if video RAM is shared, that can eat into available RAM.

Even with a dual core laptop from 2006 w/2.5 GB RAM, for some reason Ubuntu 12.04 install would hang indefinitely while installing programs with no errors, but never completing. I had to use the alternate iso to install it (works fine after installation).

I have both 64-bit Ubuntu and Lubuntu 12.04 installed on SD card with common /home partition for booting a 2 GB 1 GHz dual core tablet PC, and the only issue I had is that sound did not work in Lubuntu (no pcm device in alsamixer) until I installed pulseaudio.
