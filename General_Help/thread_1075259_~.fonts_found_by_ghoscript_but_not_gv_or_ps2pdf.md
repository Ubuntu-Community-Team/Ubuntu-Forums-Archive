### Post by fresta on 2009-02-20
Hello,

when I put fonts in my local ~/.fonts folder, they are found by ghostscript but not by gv or ps2pdf. Other apps like inkscape can also find the fonts. Can anyone confirm this?

Here's some steps to reproduce:

1. Put font file in ~/.fonts, for example the free type1 [Teleprinter]("http://www.zanzig.com/download") font from Mark Zanzig.

2. View a simple postscript file with the font, for example

> 
/Teleprinter findfont
48 scalefont
setfont
newpath
100 500 moveto
(Test font) show


gv does not display the font.

3. Pass the font through ghostscript and convert to paths by running 
> 
gs -sDEVICE=pswrite -sOutputFile=test.ps prfont.ps

and typing
> 
/Teleprinter DoFont

at the gs prompt, hit return a few times and then typing quit. The file test.ps displays correctly in gv since the font has been converted to paths by gs.

If the font is put in, e.g., /usr/share/fonts, gv does find it so it should be a path issue with gv. But I thought it would use the same path as gs? ps2pdf doesn't find the font either.
