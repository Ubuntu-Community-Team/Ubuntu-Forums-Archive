### Post by vasa1 on 2014-11-24
I've read that some distros offer hot corners to run certain commands when the mouse pointer is in a particular corner.

Am I correct in thinking that for hot corners to work there needs to be a process running which constantly monitors where the mouse pointer is? If that is true, wouldn't it be simpler to just run the particular command when needed by assigning a keyboard shortcut?

---

### Post by deadflowr on 2014-11-25
It only needs to know when the mouse pointer is within the hot corners threshold.
It is no different than when you hover the pointer on something like a link and a tooltip shows.
It only needs to know that the pointer is in it's area, and has no need to know where it is otherwise.

But, yeah, some systems that can use hot corners also can set keyboard shortcuts to do the same thing.
Some even have default keyboard shortcuts that do what the hot corners would do.
Like in unity, there are keyboard shortcuts for scale and expo(super +w, and super +s, respectively) but you can also add mouse(hover in hot corner) or mouse(click in hot corner) for those if you want.
