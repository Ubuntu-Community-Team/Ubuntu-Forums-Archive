### Post by TheFu on 2013-09-04
If you boot into a liveCD and go into recovery mode, can the machine mount the main Ubuntu partition?  If so, I don't think that drivers are the issue - IDE and SATA stuff is pretty standard. 

If it won't mount, do you have a funny RAID card or are you trying to use a RAID setup from the motherboard?  The boot info doesn't suggest either, but I have to ask.

Any encryption?  I can't tell.

Did you use LVM or did you go with the straight partition+ext4 install?  Looks like the later, so it should work easily.

If none of this makes sense, have you tried "boot repair?"  My signature has links.

Nothing looks funny to me. Using 12.04.3 is smart, IMHO.  This post is really for lurkers ... 

If nothing worked at this point, then it is time to check the chipset and swap the SATA cable.

### Post by oldfred on 2013-09-04
Is BIOS set for AHCI not IDE? Sometimes BIOS settings can make a difference.

A few BIOS also have issues with very large / (root) partitions. If that is the case a smaller 10 to 25GB / and the rest of the drive as /home then may work. 
You can quickly test just by shrinking current / to less than 100GB. Sometimes that works and others it does not.
