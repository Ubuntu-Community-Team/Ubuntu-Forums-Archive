### Post by mb_webguy on 2009-05-09
I believe people have been having some problems with the Intel integrated video.  I've had both ATI and nVidia, and while I haven't run into the problems with the ATI card that others have reported, I'd still go with nVidia if I were to buy another computer right now.

I don't know about multi-touch touchpads.  Mine's a Synaptic, but it's not multi-touch.

As far as processors, it really doesn't really matter.  I'd recommend a 64-bit CPU like an AMD64 or a dual- or mult-core Intel.  Only fairly low-end processors are 32-bit anymore, anyway.

And if you go with a 64-bit processor, you should definitely use the 64-bit version of Ubuntu.  Most people I've seen telling others not to use 64-bit and that it's not ready or that it's poorly supported don't actually use it, and are parroting information they were told by others, who were repeating information people told *them*, etc.  The fact is that 64-bit Linux is well-supported both in terms of drivers and applications.  I've been running 64-bit Ubuntu for over a year, with no problems whatsoever.  The very few apps that aren't available in 64-bit packages can be installed by forcing the architecture on the 32-bit deb, or by compiling it from source.  I've only encountered one or two applications  (out of maybe a hundred or more I've tried) that aren't available for 64-bit.  And running 64-bit will allow you to use more than 4GB of memory if you want, and resource-intensive tasks like transcoding video will get a noticeable speed boost.
