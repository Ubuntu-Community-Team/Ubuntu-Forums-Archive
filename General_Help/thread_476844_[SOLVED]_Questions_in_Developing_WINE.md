### Post by machoo02 on 2007-06-18
Well, to actually *develop* a program _requires_ programming skills...no ifs/ands/buts about it.  Otherwise, how would you be able to actually fix all of those fixmes or function stubs?  However, you can assist the Wine developers (and developers of any open source program) by testing out their programs and filing detailed bug reports explaining exactly what the bug is and how to reproduce it.  That's probably the easiest way to help with development of a program without actually doing any coding.
