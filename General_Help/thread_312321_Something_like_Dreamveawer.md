### Post by gunksta on 2006-12-04
I don't know if there is a true, drop in replacement for Dreamweaver for Linux. As is typical in the Linux Community, there isn't as much hand-holding. If enough people respond to this thread I promise, someone will tell you to just use Gedit, since it has syntax highlighting for HTML, CSS, Javascript, etc. Personally, I think that person (who-ever they may be) is nuts.

That being said there are some nice programs out there.
[Bluefish]("http://bluefish.openoffice.nl/index.html") is a nice HTML editor, but it's not Dreamweaver. It is designed to help you write your page, and offers some tools to help you. There is also [Screem]("http://www.screem.org/"), which I prefer. Again, this isn't a Dreamweaver knock-off, it's an IDE for web development. I really like how it integrates it's tools.

Closer to the Dreamweaver style there is [NVU]("http://www.nvu.com/index.php"). This is a pretty good application, sponsored by Linspire. It's really an expansion of the Mozilla browser's old HTML authoring tool. But, they've done quite a bit with it, and it's much better than the old mozilla tool. I thought it was abandonware since I hadn't seen an update in the last couple of Ubuntu upgrades, but I just checked the web page, and the author has a blog that looks pretty active.

A tool I have been playing with lately is [Amaya]("http://www.w3.org/Amaya/"). It's a web-browser / composer published by the W3C. If you are running Dapper, don't waste your time. Dapper comes with version 8 of Amaya, which looks horrible and is hard to use. Did I mention it looks horrible? If you are sporting Edgy, then install Amaya. It's one of the more interesting tools I have seen in a while, and you know it is standards compliant.  :p

--andy
