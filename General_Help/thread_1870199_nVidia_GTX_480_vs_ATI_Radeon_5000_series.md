### Post by chrisrodriguez on 2011-10-26
I'm building my new computer from scratch, and I'd like to add an add-on graphics card. I'm stuck between the nVidia GTX 450-480 and the ATI/AMD Radeon 5000 series. PCI-e ports are not the issue, nor is memory (6 PCI-e ports and 8Gb system RAM). I dabble in different areas such as network analysis to scripting to RAID 10 arrays to embedded server(s). I also contribute to several BOINC projects, which is what steered me towards the nVidia and Radeon debate. nVidia has CUDA, which can be used to bruteforce hashes (MD5, SHA1, SHA256 and the like), but Radeon has OpenCL, which can do the same. CUDA works only with some utilities, OpenCL only with others. BOINC could care less, it does it's own thing. I'm almost wondering if I could buy a low-cost one of each, and between the two, they would cost the same and perform the same as one of either (but that seems nasty to deal with, so that's an option if all else fails). Any input and/or feedback would be GREATLY appreciated, this is going to bug me for a while...

### Post by lakerssuperman on 2011-10-26
I run an NVIDIA 560ti on my main desktop.  I also have a machine with an older AMD 4870 and one machine with a newer, if low end, Radeon 5570.  The hardware is largely a wash on Linux.  The biggest difference is driver quality.  NVIDIA is much better in this aspect.  AMD has improved, but they aren't on the level of NVIDIA's driver quality yet.  If you want a no hassles solution, go with NVIDIA.  AMD will generally work, but there may be an extra tweak here or there to get the same finished product as you get with NVIDIA. 

NVIDIA also releases drivers faster and are generally ahead of the curve on supporting new features and software such as the X Server or the new Gnome Shell (which still hasn't received proper support from the AMD official drivers).  Often, you'll have to wait longer for AMD drivers to support new versions of Ubuntu and the new version of X it ships with.

Do Not Use internal ATI Video Graphics cards.  Avoid them like the plague. Nothing but endless headaches.  Use something from this list instead:

[http://www.nvidia.co.uk/object/linux-display-ia32-270.41.19-driver-uk.html](http://www.nvidia.co.uk/object/linux-display-ia32-270.41.19-driver-uk.html)

[http://support.amd.com/us/gpudownload/linux/Legacy/Pages/radeon_linux.aspx](http://support.amd.com/us/gpudownload/linux/Legacy/Pages/radeon_linux.aspx)
