### Post by scragar on 2009-09-01
The problem is simple, debian modified the partition somehow, now ubuntu doesn't like it.
```
sudo gedit /etc/fstab
```
comment out:
```
[color=red]# [/color]UUID=2e1d6002-fb23-4064-94fd-767b9195c7a9 none swap sw 0 0
```
Add under it:
```
/dev/sdb5 none swap sw 0 0
```

Or you could run:
```
ls -l /dev/disk/by-uuid | grep sdb5
```To find the new UUID and correct it if you want.
