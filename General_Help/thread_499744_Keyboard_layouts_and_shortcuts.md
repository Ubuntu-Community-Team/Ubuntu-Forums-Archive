From time to time I use a Cyrillic keyboard layout for typing in Russian. (It's actually the rus variant of us, but my question should apply to any alternate layouts.) A particularly inconvenient thing about this is that none of my keyboard shortcuts work as I expect them to when I use the alternate layout. The reason is pretty clear: Ctrl-Q is not the same as Ctrl-&#1071;, even though I'm pressing the same keys. For a few important shortcuts (like "Switch to next keyboard layout"), I mapped the alternate shortcut to its Cyrillic version so I can use it in either layout.

My question is, is there any way I can have this sort of thing apply to all my keyboard shortcuts without me having to set them all manually? (The current solution also means that I can't have a real "alternate" shortcut, because the Cyrillic one is taking up that spot. If there's a way to get more than two combinations per shortcut in KDE, that would also be useful information.)

An example of behavior I like is in Mac OS X. There, if you press Cmd-Q, it will trigger a Cmd-Q regardless of what keyboard layout is active; it depends (AFAICT) on the default layout, which for me is QWERTY.

I'm running KDE 3.5.6 on Feisty.
