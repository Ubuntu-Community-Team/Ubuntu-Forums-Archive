### Post by quickk on 2007-02-16
Hi everyone, 

   I'm trying to get used to using Linux, but in my opinion, one of the main problems is that the clipboard isn't very functional.  My main gripes are:

1.  You can't paste objects from one program into another program.   For example, say I'm writing a document in a text editor, and I want to past a graph I made with a spreadsheet application.  In windows I just cut the graph from the spreasheet and paste it in directly into my text editor with the CTRL-C, CTRL-V combo.  Even better, I can edit the pasted graph from the text editor by double clicking on it.  I can't find any way to do this in Linux.

2.  Same problem as 1.  I just wan't to stress how much of a problem this is.  Say I want to paste an image from the web into a document.  In windows I can just cut and paste directly from firefox into my document.  In Linux, I have to save the image to a file, and the open it in the text editor I was using, which involves a lot more steps.  A simple cut-and-paste would be extremely nice.

3.  The clipboard seems to lose it's data once the application that you pasted from is closed.  An example where this is a problem:  I write a long email, I copy it using CTRL-C (as a backup), the browser crashes when I send the message, my message is lost :( .  

Am I just missing something?

Thanks for any suggestions!

### Post by strabes on 2007-02-16
There's a gnome app that you can install called "glipper" that works as a clipboard manager. It lives in your notification area. ```
sudo apt-get install glipper
```
