### Post by geraldm on 2008-02-09
This issue comes up regularly.  After a keyboard type has been selected, the user still has to learn it.  Problems crop up when a user changes keyboard, say from en_US to one of the US international ones. 
One keyboard does not provide the variety of types needed for other languages/areas.
So there are special characters introduces into a two-keypress input to access other codes that were unavailable without the lead keypress. the single and double quote are used in a two-keypress sequence in some keyboard types. 
Also, there are keyboard change sequences for a user that needs two different keyboards for two different languages.  The sequence is a level shift key sequence that activates a different tier for unshifted and shifted characters.
There are also "Compose" keys.

For the two-key sequence problems search for "deadkeys' 
It would probably be useful to provide a keyboard introduction to explore the available options.

Gerald
