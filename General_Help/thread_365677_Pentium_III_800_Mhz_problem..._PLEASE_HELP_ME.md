If the processor was the only change, then possibly it is not compatible with the motherboard.

You should boot up and select the "**memtest**" option, and then let your PC run for a couple of hours doing intensive memory testing - that will prove if your hardware is capable of running under stress.

You should also have reset your BIOS to default after putting in a different CPU, as some timing settings may not be compatible for the new processor.

There also may be some Voltage/Frequency jumper settings on your motherboard that have to be manually altered when changing such ancient CPUs - check the manual.

### Post by trashmatic on 2007-02-21
Celerons of that vintage are mostly 66mhz FSB clock, if I remember correctly,  The P3-800 could be either 100mhz or 133mhz FSB.  So if your memory only supports 66mhz, and the P3 tries to set the FSB to 100 or even 133mhz , bad things will happen like you are seeing.  Lots of data corruption.

Sorry, but I think you probably need new memory to run in that configuration.  What you can do is go into the BIOS and see if you can set the FSB to 66mhz manually... essentially "underclock" the P3 to match the memory and see if that works.

But if you have important data on that computer, *do not continue to operate it* in its current condition.  If data is being corrupted in RAM, it can seriously mess up the data on your hard drive when it is written to!
