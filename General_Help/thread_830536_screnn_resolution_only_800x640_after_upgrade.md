### Post by nkri on 2008-06-16

```

```
and see if you can get back to 1024x768 from the first window that pops up.  Also, make sure you have the correct graphics driver installed and in use.  If you are not sure about the driver, post the output of
```
lspci
```

### Post by alterpinguin on 2008-06-17
no switching screens with CTRL+ALT+Fn:
check:
```

man xorg.conf

```
there is explained the disable of this feature,
Option "DontVTSwitch" "boolean-value"
default should be "off", so without such an entry to "on"
it should work -
- but you mentioned not to be able to use a modified xorg.conf file.
This could be you are not using this file (you did modify) - your x11-server
is still using a different one. There are different places for an xorg.conf and at last a special option to start the server (-config ? or environment-settings)

- next, the xorg.conf file you did show, that file has NO entries for a special screen-resolution - this means: the graphic-card-driver has to detect a possible setting and this means, if you switch to a new version, this could have been changed(for example from nvidia to support newer cards)
there is no such section in your xorg.conf like this example:
```

Section "Screen"
    Identifier     "Default Screen"
    Device         "Configured Video Device"
    Monitor        "Configured Monitor"
    DefaultDepth    24
    Option         "NoLogo" "True"
    SubSection     "Display"
        Depth       24
        Modes      "1280x1024" "1024x768" "832x624" "800x600" "720x400" "640x480"
    EndSubSection
EndSection

```
this is with screen-sizes for ranges from 1280x1024 down to 640x480,
but even if you have such a section, it may be you need one for the sync-ranges your monitor and graka-card can do (incl. VTimings for this mode-setting).
