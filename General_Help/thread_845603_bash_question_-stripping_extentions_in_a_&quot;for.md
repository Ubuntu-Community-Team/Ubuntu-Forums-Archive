### Post by rliegh on 2008-06-30
Hi

A lot of times I'll have files of the same extention that I need to do something with, that will require a for loop. 

For example, let's say I want to rip the strings from a bunch of doc files (1.doc 2.doc 3.doc) to convert the output into a text file. usually I would use a 'for' loop; something like:
```
for x in *doc;do strings $x >$x.txt;done
```
What I'd end up with would be
1.doc.txt
2.doc.txt
3.doc.txt

What I'm after is a way that I can make it so that I can remove the old extention ("doc"); I'm imagining I could probably use sed, but sed confuses the **** out of me.

Thanks for your time. :guitar:

---

### Post by sdennie on 2008-06-30
Maybe try:

```

for x in *.doc;do strings $x > `basename $x .doc`.txt;done

```

### Post by ghostdog74 on 2008-06-30
dont' need to call external command. the shell can do simple substitution
```

for x in *doc 
do 
 newfile=${x/.doc/} 
 strings $x > "${newfile}.txt"
done

```

### Post by ad_267 on 2008-06-30
Have a look at the basename command. If you specify an extension it will remove it.

Eg:

```
basename /home/me/file_name.odt .odt

```
