### Post by 3rdalbum on 2009-10-09
Ubuntu is close to perfect for new users. What you might not realise is that Windows is a very messy operating system, quite difficult to use unless you're used to it, and sometimes things in Windows don't work when there's no reason why they shouldn't.

I have:

1x almost middle-aged lady using Ubuntu to copy DVDs, conduct her eBay business, download movie torrents and manage her family photos. She's virtually computer-illiterate (i.e. she once asked me "How many megabytes has my computer got?"). She doesn't install software unassisted, but the system doesn't put any hurdles in her way for day-to-day work.

1x 60-year-old man who's had a stroke, using Ubuntu to check email, watch TV shows on ABC iView, convert videos for his iPhone, edit videos and photos. He's used Windows and Mac OS for many years, and he has installed software on Ubuntu by himself.

1x 25-year-old accountant who's not really that good with computers, using Ubuntu in a dual-boot with Windows. I'm not sure what he uses Ubuntu for; I know he rips DVDs and watches Ice Hockey online. I do know that he rarely boots into Windows. Once when he did, he found that Vista wasn't working, so he reinstalled it which stopped GRUB from working. I found a HOWTO for him to get it back, and he managed to follow it (and then loved the feeling of power that it gave him).

Ubuntu does slow down a tiny bit over time, but that only occurs when you go to browse your home directory and you've got 208 items directly in it; or your desktop takes a little longer to load because there's 32 icons on it and you've been neglecting cleaning it! The system itself doesn't go slowly, but your use of it may cause it load more data when that data is called for.

### Post by BT1 on 2009-10-20
> **Meow27 said:**
> the comment about swap was so ignorant i had ot correct you..

in windows you can use a paging file which acts as swap in the C:\ drive (typically) 

the only advantage linux has on this is that it only needs one swap partition in a system as opposed to windows, which it would need consistent paging files.

True. Upon Installation (even in the drive manager after the system is installed) you can create a partition for your page file if you want (somewhat redundant if you make that partition on your primary hard drive since that's where the pagefile automatically goes). With Windows pagefile you can:

1.) Make multiple page files across multiple hard drives which can speed up the page file read/write.
2.) Make the page file it's own partition on any drive and point the system there with a few clicks.
3.) Make one page file and put it on an alternate hard drive, and not on your main drive (this also speeds up page file access).

As far as I have seen, Ubuntu's swap and Window's page file are nearly identical. But then again I'm a Ubuntu nub.
