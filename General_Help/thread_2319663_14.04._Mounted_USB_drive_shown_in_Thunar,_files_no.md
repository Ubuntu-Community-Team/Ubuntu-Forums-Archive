### Post by TheFu on 2016-04-06
Welcome to the forums.

The way that file managers mount drives isn't a "real mount".  They use a FUSE technique and gvfs. It is often convenient for drives that aren't always connected, but there are issues (permissions/performance/access) with this.  Using the /etc/fstab is the normal way to make permanent storage available on a Unix PC, but that isn't always as convenient.  Also, there is a tool called autofs which can mount storage on-demand, but it is more complicated to setup. I use autofs for all USB storage.  Both fstab and autofs mounts are "real" and provide complete control over the mount options, permissions, and give the fastest performance available.

The issue could be any of a few different issues.  Just guessing ... 
* wrong driver for the storage is being used
* missing dependencies for the file manager to access the storage
* missing file system support for the type of file system on the USB drive.

Before we get too far ... lets try something simple .... open a terminal, inside there run **thunar**. Are any error messages shown?  Please post those back here.

If that didn't work, lets see what we can see about this disk.  These will all be terminal commands. Please copy/paste the command AND the output back here, so we can see what is happening. Also, please use "**code tags**" so the output lines up - we prefer reading it that way.

First, unplug the USB from the system, wait 10 sec, then plug it back in, wait 10 sec and run **dmesg | tail -n 20**. Post those results here.  We want to see something like **sdb ... device added**  Could be sdc, sdd, sde, sdf ... depends on the number of storage devices already connected.

2nd, if the device is recognized, run **sudo parted -l** - so we can see the partitions on the disks and the type of file system. Post those results here.

If that showed partitions on the USB device, great!  Now we know the file system type and can load the correct dependencies so it can be accessed. Don't know which packages to install until we see the dependencies and you tell us which file manager is being used.  I'll await your reply.

Ref: [https://en.wikipedia.org/wiki/GVfs](https://en.wikipedia.org/wiki/GVfs)

### Post by Cavsfan on 2016-08-16
I also finally figured out a way to mount my windows 10 drive (C) in Ubuntu as you cannot just click on it because it says it's in hibernation mode, etc.

First off you enter this: 
```
sudo mkdir /media/windows


```

Then instead of having to enter the whole command to mount it each time I created an alias in ~/.bashrc:

```
#Mount C: drive
alias mountc='sudo mount -t ntfs-3g -o ro /dev/sda1 /media/windows'

```

Then it shows as C: in Thunar and has to have a password to un-mount it.

I like the ability to mount it as read only too as you can mess windows up by modifying any files from Linux.
Been that and done that once.

I got the info from this page: [http://askubuntu.com/questions/145902/unable-to-mount-windows-ntfs-filesystem-due-to-hibernation](http://askubuntu.com/questions/145902/unable-to-mount-windows-ntfs-filesystem-due-to-hibernation)

If the drive is hibernated in Windows and you force mount it in Ubuntu you could do great damage to the filesystem on the disk; if the disk is the Windows disk with the Windows OS on it you may kill your Windows installation completely.

Make sure your Windows 10 installation has the fast-startup (I think that's what it is called) disabled and you should find that any disks or partitions mount properly when you "shutdown" Windows;  fast-startup enabled means that Windows does not actually shutdown but goes into hibernation.
