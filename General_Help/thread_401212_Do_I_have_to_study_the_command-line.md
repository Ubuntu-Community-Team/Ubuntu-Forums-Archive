### Post by kidders on 2007-04-04
Hi there,

Most people prefer to use the most effective interface available, for the task at hand. For example, I often go for quite some time without touching my mouse. That's not to say I can't do everything I want with a mouse (even use a word processor) ... it's simply a question of the way efficiency and personal preference combine at the particular moment I make the choice.

For many users, the command line interface allows one to do in less than a second, what might otherwise take 4 or 5. In addition, of course, the degree of precision required to operate it, and the unique exposure to the operating system it gives you, make it ideal for diagnosing problems, and guiding other users through something in a 100% unambiguous way (which is why you see so much of it here).

### Post by aysiu on 2007-04-04
I don't know where you got the idea there's any pressure to "study" the command-line. If you want to use the GUI, use the GUI. People may give you commands to copy and paste into the terminal, but no one says you have to study the command-line. Use what works for you.

> It is also possible that, one day, people will not have to use the command-line or the codes that one has memorized are obsolete. I find it sort of a waste of time to memorize a lot of things that I could not be using one day. Also, I do not think that a code is universal. Maybe, different distros require different codes. By the way, you've got it a little  backwards here. Commands are rarely if ever obsolete. Linux is using the same commands Unix has had for at least thirty years. Mac OS X uses those same commands. It doesn't matter what GUI interface you're using (Aqua, KDE, XFCE, Gnome), the commands all work the same way. Codes are universal; GUIs are not. Different distros, same codes. Different distros, different GUIs. Sometimes, even on the same distro, different GUIs.

That's another reason people give codes. If you tell someone > Paste this code in ```
sudo apt-get update && sudo apt-get install *packagename*
``` it'll work whether the person is using Ubuntu or Kubuntu. But if you say > Go to System > Administration > Synaptic Package Manager That won't work for Xubuntu or Kubuntu--only for Ubuntu.
