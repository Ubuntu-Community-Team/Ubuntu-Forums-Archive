### Post by grahammechanical on 2016-08-21
Personally, I think that it is better to install an alternative distribution built on Ubuntu than it is to install an alternative desktop environment onto Ubuntu.

The different flavours are built on Ubuntu. They use the same software repositories as Ubuntu. They have the same kernel and driver modules as Ubuntu. When the Ubuntu kernel team pushes an update to the kernel into the update channel it will go out to Ubuntu users and also to the users of all the flavours built on Ubuntu.

The flavours have smaller development teams. So, they limit their support to 3 years for their LTS releases. But by the time one LTS has lost support there is another LTS release for that flavour. So, it is matter of upgrading to the new LTS release every 2 - 3 years.

Ubuntu LTS users can also upgrade every 2 - 3 years. To get the full 5 year support for the kernel Ubuntu LTS users must remain on the kernel that came with the initial release of the LTS or upgrade to each new hardware enablement stack every time it is released.
