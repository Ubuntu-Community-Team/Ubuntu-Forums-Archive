### Post by btindie on 2010-10-03
I get the same result as *SeijiSensei *when connecting to **cp5556.cz.cc** from the UK, so it seems that you've now got it working.
 
Note though it appears that you are on a dynamic IP address
```
host 81.156.35.38
38.35.156.81.in-addr.arpa domain name pointer host81-156-35-38.range81-156.btcentralplus.com.
```
which means it is not unlikely that at some point your IP address may change and in which case your website will stop working. You'll then need to update your A record to point it to your new IP address.
 
Do you know if nic.cz.cc has some kind of Dynamic DNS update service?
 
There are [various sites]("http://en.wikipedia.org/wiki/Category:Dynamic_DNS") that offer free [Dynamic DNS]("http://en.wikipedia.org/wiki/Dynamic_DNS") update services such as [no-ip.com]("http://www.no-ip.com/"). You run a client on your computer and it keeps your IP address updated with their DNS. The domain you get is a sub-domain of one of their's - i.e. cp5556.no-ip.org.
 
Within DNS you also have a [CNAME]("http://en.wikipedia.org/wiki/CNAME_record") record which is basicly an alias for another record. So if you setup the above you could change your current A record to a CNAME record and set it to cp5556.no-ip.org. Providing you run the update client, if your IP address changes then so will the IP address that cp5556.cz.cc points to via the cp5556.no-ip.org record.
```
cp5556.cz.cc.      CNAME    cp5556.no-ip.org.
cp5556.no-ip.org.  A        81.156.35.38
```

### Post by btindie on 2010-10-03
Try what? You editing your */etc/hosts* file will have no effect on the rest of the internet. It's still the same as before for me.
 
You could test it yourself to see how it's seen on the internet by using a [proxy]("http://www.google.co.uk/#&q=free+proxy+list").
 
To setup [www.cp5556.cz.cc]("http://www.cp5556.cz.cc") you need to create a new record in the DNS. This can either be another **A** name record pointing to your IP **public** address or a **CNAME** record pointing to *"cp5556.cz.cc."* (Make sure you have the final DOT after cc). Using the CNAME record means you don't have to change the IP address of both records.
 
You then need to modify your Apache configuration to read
```
ServerName cp5556.cz.cc
ServerAlias [www.cp5556.cz.cc]("http://www.cp5556.cz.cc")
```
That is if you want them to both be the same site. You can also configure cp5556.cz.cc to redirect to [www.cp5556.cz.cc]("http://www.cp5556.cz.cc") if you desire, but leave that till you get the basics working first!
