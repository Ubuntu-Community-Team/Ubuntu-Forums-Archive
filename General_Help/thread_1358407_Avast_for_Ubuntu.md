Oh, I'd heard of Bliss - but it was nothing more than a proof-of-concept, and the vulnerability was quickly patched. I mean, bliss even had a "--bliss-uninfect-files-please" command line switch that did just that, and it kept a nice neat log of what it had done!

If you were talking about that screensaver deb issue, that wasn't a virus KeLa. Search here on ubuntuforums and [http://www.h-online.com/](http://www.h-online.com/) for screensaver deb. And while we are at it, perhaps google too for definitions to trojans, worms, viruses, rootkits and likes perhaps? Then there is your Linux file permissions thingy, superuser privileges. As a derivative of UNIX, the OS is just by design to be more secure than some others unlike some others where it is but an afterthought. 

Don't get me wrong, there are still threats e.g. buggy code opening up opportunities for things like remote and local exploits. But herein lies one of the greatest strengths of Linux which is most of the software comes with source code. Potential exploits are readily identified by a very active, technically astute community of users from four corners of the world. Just compare the track records of Linux Vs MS Vs Mac as to the relative speed to which vulnerabilities are identified and workarounds, patches, fixes are developed (e.g. that screensaver deb). Anyway, it ain't easy to get away with hiding malicious bits within source code with so many prying eyes around.

I do however concede that there can be no substitute for intelligence. An OS is only as safe as its user wants it to be. Very few will be sympathetic to those who chooses to install stuff from untrusted sources especially those that do not provide for source code and/or are digitally signed.
