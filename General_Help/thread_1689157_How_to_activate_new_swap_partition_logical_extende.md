### Post by dubref on 2011-02-16
Having problems activating a new swap partition.

Using instructions per [https://help.ubuntu.com/community/SwapFaq](https://help.ubuntu.com/community/SwapFaq)

```

~$sudo fdisk -l
   Device Boot      Start         End      Blocks   Id  System
/dev/sda1               1        2550    20482843+  83  Linux
/dev/sda2   *        2551       10199    61440000    7  HPFS/NTFS
/dev/sda3           10200       26517   131074335   83  Linux
/dev/sda4           26518       60801   275386230    5  Extended
/dev/sda5           26518       29067    20482843+  83  Linux
/dev/sda6           60280       60801     4192933+  82  Linux swap / Solaris
/dev/sda7           29068       60279   250710358+   7  HPFS/NTFS

Partition table entries are not in disk order
```

```
~$ sudo swapoff -a 
~$ sudo mkswap /dev/sda6
Setting up swapspace version 1, size = 4192928 KiB
no label, UUID=bea42673-885b-4866-8cfa-14bb5675d933
~$ sudo swapon -a
swapon: cannot find the device for UUID=5c0fe46c-f6cc-4bee-9e3d-f455e9ff0b7c
```

Searching, I came upon instruction to correct UUID in resume file. [http://ubuntuforums.org/archive/index.php/t-915659.html](http://ubuntuforums.org/archive/index.php/t-915659.html)

I corrected the UUID then updated with 
```

~$ gksudo gedit /etc/initramfs-tools/conf.d/resume 
~$ sudo update-initramfs -u
```

```
~$ cat /etc/initramfs-tools/conf.d/resume 
RESUME=UUID=32e6bcfe-9d25-4a5a-8a0e-6984546791d6
```

Now, I have three different UUIDs, bea42673***, which generated current mkswap command, 32e6bcfe***, which was generated by previous mkswap attempt(obviously I should correct /resume to current UUID), and finally 5c0fe46c***, which was UUID for original swap file in primary partition, and which I changed in resume file . Still it must be kept somewhere else?

```
~$ sudo mkswap /dev/sda6
``` command seems to generate a new UUID each time of use, then I have to correct it in /resume file and somewhere else. 

Question is, where else do I have to enter the new UUID for swap partition ?


=================
Background: Used to have 4 primary partitions ext4 (/),NTFS(win7),reiserfs(/home), 200GB of unused space(don't ask...)  and finally swap.
I disabled swap with ```
sudo swapoff -a
```, used GParted to delete swap, then created new extended partitition in unused space, creating 3 new logical partitions: ext3, NTFS and finally 4GB swap partition at the end.

### Post by dubref on 2011-02-16
Upon further reading of man swapon, I decided to try to activate swap with -U flag specifying UUID
```
~$ sudo swapon -U bea42673-885b-4866-8cfa-14bb5675d933
```

Which seems to have solved the issue.

However, that still leaves the issue, that somewhere in the system the old swap UUID is lingering.
