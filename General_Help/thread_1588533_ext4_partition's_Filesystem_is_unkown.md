### Post by srs5694 on 2010-10-05
Edit: Oops... I didn't see your last post, eliwis. I'm glad that fsck worked for you. I'm leaving my original post, below, since it explains what caused the problem and so could help you avoid running into the same problem again in the future. Also, creating a BIOS Boot Partition is a good idea. If all your partitions are in use, the best way to do this is either to shrink one partition by a small amount (1MiB is more than enough) or delete the swap partition, create the BIOS Boot Partition and a new swap partition in its place, and reconfigure /etc/fstab for the new swap's UUID. After you create a new BIOS Boot Partition, re-install GRUB 2 and it will use the new BIOS Boot Partition as part of its boot process.

> **eliwis said:**
> so i started playing around with GParted, and while doing that i also put a bios_grub flag on my main ubuntu install partition, thinking that it would force grub to load from that partition. But that just gave me the GRUB Rescue prompt when trying to boot.

You've wiped out your Linux installation. What GParted calls the "bios_grub flag" is actually a separate partition type code for the BIOS Boot Partition. That type code should *only* be used on a partition that GRUB 2 will access to store its stage 2 boot loader. Therefore, when you re-installed GRUB with that type code set on your main Linux partition, GRUB 2 overwrote the ext4 filesystem data with its boot loader code.

There's a slim chance that you can recover your system by using fsck or e2fsck, perhaps in conjunction with the -b option to read an alternate superblock. Unless you're familiar with such operations, though, I recommend you just re-install Ubuntu from scratch. Since you've got GPT, I suggest the following:


[LIST=1]
[*]Delete /dev/sda1 (this appears to be the damaged Linux installation partition).
[*]Create two new partitions: A small (1MiB) BIOS Boot Partition and a larger (~1MiB smaller than the current /dev/sda1) partition for the Linux root (/) filesystem. The BIOS Boot Partition can and should be created with no filesystem, but you should use whatever filesystem you want for the Linux root (/) partition (ext3fs, ext4fs, XFS, etc.).
[*]Re-install Ubuntu, mounting /dev/sda3 and /dev/sda4 at whatever locations are appropriate, but of course be sure not to erase them. Note that it's possible their partition numbers will change when you create two partitions in place of the current /dev/sda1.
[/LIST]


Good luck!
