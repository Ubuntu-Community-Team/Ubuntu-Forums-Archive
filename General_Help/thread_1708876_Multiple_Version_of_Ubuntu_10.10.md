### Post by lithopsian on 2011-03-17
You would normally want to boot into the newest version because it will contain small security or bug fixes.  If that is stable for you then you can delete some or all of the older versions.  A "kernel package" consists basically of the linux-image package and headers package with the same number.  Always keep at least one stable kernel image so you have something to boot to when you install a dud or break something.
