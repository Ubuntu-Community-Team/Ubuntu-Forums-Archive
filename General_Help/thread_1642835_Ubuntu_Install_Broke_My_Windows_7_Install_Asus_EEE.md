### Post by Rubi1200 on 2010-12-12
Not quite sure what is going on here, but there are definitely some things out of sync.

GRUB thinks the Windows partition is on sda2:
> "Windows Vista (loader) (on /dev/sda2)"This is clearly incorrect as sda2 is the extended partition.

As sikander3786 pointed out you are missing the boot files for sda1 and there is no operating system defined there.

If you look at the script posted by Jose Catre-Vandis you will see that you are missing the following files:
> /Windows/System32/winload.exeIf recovery mode does not work, you may want to try Testdisk as already suggested, but I fear the Windows installation may be lost.
[http://www.cgsecurity.org/wiki/TestDisk_Step_By_Step](http://www.cgsecurity.org/wiki/TestDisk_Step_By_Step)

I assume you do not have a Windows recovery medium?

If yes, try using that first to restore the installation.

All that said, you may want to wait for some other opinions too.

In any event, good luck.

EDIT: using GParted from the LiveCD/USB you could try moving the boot flag (indicated by an asterisk) from sda1 to sda3 and try and get into the Recovery Partition that way. But, I do not know if that will actually work.

GParted > right-click and there should be a setting for managing flags. Uncheck for sda1 and then right-click on sda3 and put the boot flag there. Accept changes and attempt to boot into the partition from the GRUB menu.
