Hi all,

What I want is to have several keyboard layouts available with only two switched with Alt-Shift (or whatever).

Eg I have Us, Ru, Fr, and Ua layouts. Us and Ru are two main ones which I use 90% of time. So I don't want to press alt-shift four times. But when I need Ua, I press Ctrl+Alt-4 or click something in system tray and here I am.

What I described is exactly what KKBSwitch allows to do. It is just another QT-based layout indicator written for KDE3, which had very clumsy layout switcher. I love KKBSwitch and use on all my machines running Ubuntu, Debian, Slackware or CentOS. Debian-based distros have KKBSwitch included in repositories, others require to compile it myself.

But the latest release of this nice tool is dated 2005. It uses QT3 and so on. I am really afraid it will get thrown out in some next release or two.

So is there an alternative more modern way of doing so? I tried to tune XKB parameters, but did not succeed much. I also checked modern Gnome and KDE indicators. Seems like nothing can do so. 

Please tell me I am not right :) I don't believe nobody needed it. In counties like Swiss or Belgium where people speak too many languages this should be extremely useful.
