### Post by rbmorse on 2007-06-24
Sometimes it takes more than 4 hours.

The error is generated by the X server and indicates a device is configured but not accessible/present/turned on.  

This is a very common error because X configures a Wacom tablet by default.  If you don't have a Wacom tablet, the error message is  harmless and you can ignore it. 

However, you can clean up xorg.conf to deal with this if you want: 

First, make a backup of the file /etc/X11/xorg.conf. The easiest way is to open a terminal window:

cd /etc/X11
sudo cp xorg.conf xorg.conf.backup   (provide your user password when prompted)


then, still working from the terminal, open the file in the text editor with root level permissions

sudo gedit xorg.conf

As you scroll down the file you will find three sections dealing with the driver "wacom": 

```

# Section "InputDevice"
#  Driver        "wacom"
#  Identifier    "stylus"
#  Option        "Device"        "/dev/wacom"          # Change to
#                                                      # /dev/input/event
#                                                      # for USB
#  Option        "Type"          "stylus"
#  Option        "ForceDevice"   "ISDV4"               # Tablet PC ONLY
#EndSection

#Section "InputDevice"
#  Driver        "wacom"
#  Identifier    "eraser"
#  Option        "Device"        "/dev/wacom"          # Change to
#                                                      # /dev/input/event
#                                                      # for USB
#  Option        "Type"          "eraser"
#  Option        "ForceDevice"   "ISDV4"               # Tablet PC ONLY
#EndSection

#Section "InputDevice"
#  Driver        "wacom"
#  Identifier    "cursor"
#  Option        "Device"        "/dev/wacom"          # Change to
#                                                     # /dev/input/event
#  .                                                   # for USB
#  Option        "Type"          "cursor"
#  Option        "ForceDevice"   "ISDV4"               # Tablet PC ONLY
#EndSection

```
You need to put a # character at the beginning of every line in each of the three sections, as shown. The # character at the start of a line will keep the system from processing that line...i.e., it turns the line into a comment. 

Then also a bit further down in the file you need to look for these lines and fix them, too: 
```

Section "ServerLayout"
        Identifier      "Default Layout"
        Screen          "Default Screen"
        InputDevice     "Generic Keyboard"
        InputDevice     "Configured Mouse"
#     InputDevice     "stylus" "SendCoreEvents"
#     InputDevice     "cursor" "SendCoreEvents"
#     InputDevice     "eraser" "SendCoreEvents"
EndSection

```
**WARNING** If you fail to fix these three lines X will *NOT* start the next time you try to  restart the X server.

Once you fix those things out save the file, close all open windows  and restart the X server (<ctrl><alt><backspace>).  Once X is back up and running again you shouldn't see those errors anymore. 

I really don't think it is related to your printer at all.
