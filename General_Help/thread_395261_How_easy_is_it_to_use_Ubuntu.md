### Post by jem7v on 2007-03-27
Setting up and installing the OS sometimes takes some configuring, but not much, and most of the time you can just copy and paste the text from tutorials.  Installing most software is pretty painless, thanks to the spectacular Add/Remove program....

Basically, you scroll through a list of programs and check off the ones you like, then click Apply. And it does the work for you.

Ubuntu is pretty intuitive to use, because program and utility menus are sorted by function rather than the company that wrote the software for it (as they tend to be organized in the Windows start menu).  I had practically no usability issues moving from Windows to Ubuntu. I actually think it's easier to figure things out... I get so lost in XP.

### Post by Martogh on 2007-03-27
I'm quite a noob with ubuntu myself, but personally i found it fairly easy to adjust to from windows. There were a couple of things i had problems with, and a couple i still do from time to time.

Once its set up its extremely easy to use, my wife has very little computer experience and uses it all the time. To just be a user you dont need any syntax knowledge.

Setting up things is more difficult, some syntax is very helpful in setting things up. Getting your head around things being a bit different is the main thing. Things that i took a bit to adjust to are:

the new file system of course. There are no drives any more, and everything is just a branch of the root / folder.

editing fstab to tell ubuntu to mount a drive. Windows always does this automatically (different partitions and media devices like cd-rom are given a drive letter associated with it automatically). In ubuntu i had to edit a file called the fstab to add some of my new partitions, and to update things when i put a new hard drive in, although i think it automatically set up everythign that was there when i installed it.

everything is much more secure, but you gotta remember that lots of things can only be done now as a 'root user'. Windows by default basically has no security, and most users can move, alter, delete anything. But ubuntu is by default much more secure so you have to type passwords and be logged in with right user, etc. 

networking was a little more complex, i had to edit some samba files to establish networks rather than just have windows do it automatically. Having said that there are very helpful guides to help you do this and it didnt take long (and once its done it runs smoothly without hiccups.) Actually the biggest problem i had with this is that i didnt have a password set on my windows machine and so security problems werent letting my ubuntu machine connect to it. Once i worked that out i just needed to put a password on and it worked fine.

There'll be some other thigns you'll have to learn too, but all in all i had my machine working fine in a couple of days and i had no previous linux experience.

### Post by 3rdalbum on 2007-03-28
> **Lucifiel said:**
> To Martogh:

Thank you for the information. :) Oh dear, that means: no drives? Darn... given that I get confused easily, I wonder how fast I'll be able to adapt to it. I look forward to not wrecking havoc on my system. 


I'm  a bit confused myself, actually. Internal drives do mount, you've just got to tell Ubuntu where they are and what they are. Apparantly it's some sort of security feature, because most other distros mount internal drives automatically.

Having said that, you'll find that the Linux way of working with drives is different to the Windows way. The Windows way is to assign each drive a "letter" - like A, B, C etc. With Linux, you get "mount points" - so all your storage devices are mounted in folders in the filesystem. This is difficult to explain, but it provides a lot of fantastic advantages. Essentially, it means that you can access your storage devices by going to the "/media" folder.

> Editing tons of files to get things to work? That doesn't bode well for me. =( 

Plenty of passwords? Uh oh... >>;; 


I only remember editing 3 files to get things to work - the graphical configuration file, a disk access settings file, and the automatic drive mounting file. These days, there is a program called Envy which should set up the graphical configuration, you don't need to touch the disk access settings, and there are plenty of howtos about the automatic drive mounting.

Depending on your hardware, you may need to edit one or two more; but remember that the plaintext configuration files end off being a tweaker's paradise, and there are always documents online telling you exactly how to edit those files. Once you've got everything set up, you don't need to touch the files again.

"Plenty of passwords" isn't quite true. You only need to know one password - your user account password. The reason why you get asked it a lot is due to security. Your user account runs as a normal, limited user by default, and you only get asked for your password again if a program needs to run as an administration user. This, along with some other security features, is the reason why viruses are nonexistant in the Linux world. Be thankful.

I, too, am a relatively recent convert from windows XP.
I am running Feisty (upgrade from edgy) on my laptop. I ran into two problems whilst getting things running: First, I am using a widescreen monitor, which the edgy install did NOT like. Now that Ive got everything running, my linux install runs cooler than my windows install (Ive got the drive split up into two, so I can fall back to windows for emergencies), and is more reliable and useful. I highly reccomend either waiting a month and going with the Feisty release, or just grabbing the Feisty beta now. Youll have a much easier time configuring your video card, and manging all of your hardware (Feisty has done a miraculous job detecting just about everything in my computer on its own).

### Post by jem7v on 2007-03-28
Actually, Ubuntu includes a lot of drivers by default so you won't have to install many.

The issue is usually with things like NVidia video cards, where there is a default open-source, community maintained driver included for them but it doesn't function as fully as NVidia's proprietary driver, or proprietary media codecs like mp3 or wmv.  Those drivers are not included by Default for legal/political reasons - hence the need to install some drivers and codecs.  Codecs are almost always painless to install, and video drivers can be frustrating sometimes but they tend to have excellent HowTo instructions written for them.
