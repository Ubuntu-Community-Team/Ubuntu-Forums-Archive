### Post by lopzided on 2006-03-31
i've been having a bad problem with a total freeze-up at the login screen...after searching and seeing plenty of others have the same problem with no solution, i thought i'd come back and tell how i fixed this problem quickly and easily...i hope the same works for you!

turns out my BIOS was set to use PCI as the primary video thingy....i don't know how it got switched to this (my computer was working quite fine before this)..but i have an AGP card.  i was thinking that my graphics card was fried, so i was gonna switch to my onboard card and see if that worked...when i noticed it was set to PCI.  i was like WTF?  so i changed it to AGP, it booted up, and blammo...no problem whatsoever.

too bad i had decided that it was my primary HD...and i pulled it out, nuked 30 gigs of mp3's that were backed up on my only other drive, and installed...just to find that the problem still remained...and the answer was right there the whole time...

anyway, i hope this helps someone out...see ya!

***EDIT***
actually, this only solved the problem temporarily...it only gave me enough time to post that message!

however, i did eventually fix this problem...don't know why i didn't think of it before...but what was really wrong was my sound card had wiggled a little bit loose.  the reason it was crashing at login, i'm guessing, is because that's the first instance when ubuntu plays a sound.

so if you're having this same issue, you might want to unplug your sound card and plug it back in, making sure it's seated firmly...this fixed the problem for me altogether!
