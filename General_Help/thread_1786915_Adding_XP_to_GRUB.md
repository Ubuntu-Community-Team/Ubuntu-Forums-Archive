### Post by oldfred on 2011-06-20
XP should work from sda2. I have seen it work from a logical partition but only using Linux tools as Windows does not support a first boot from a logical.

I suspect either your partition is too small. Windows/NTFS really wants 30% free to allow it to fragment the way it wants. At 10% free it just about stops working. 
The other issue may be a BIOS setting. XP needs IDE. I updated my motherboard and changed BIOS to AHCI and XP stopped booting. It needs special drivers that are easier to add during install, so I still am in IDE mode.

Comments from others on AHCI and BIOS issues:
> SATA mode should stay in AHCI unless you are getting nasty issues (which shouldn't happen with linux or vista/win7). The main reason why people change SATA mode to 'compatibility' is because winXP wont work with AHCI without loading the drivers..and you need a floppy drive to load the drivers, which most newer computers don't have (and lots of people are lazy anyway).
> Oldfred was right on, it was the SATA Native IDE setting that hosed Grub. I had to set the OnChip SATA Type to AHCI in order to boot using Grub. As for the OnChip SATA Port 4/5 Type setting it doesn't have any affect on Grub, it can be set to IDE or as SATA Type. My MB BIOS doesn't have a Compatibility or LBA setting so I have to find the AHCI XP drivers if I want to Multi-Boot XP and Ubuntu.

### Post by oldfred on 2011-06-20
I think if Windows senses it is a USB drive it does not allow install. IDE and SATA are internal drives which is all Windows allows. It cannot be used with multiple systems per the license.

Are you sure the size you are look at is not for a USB installer?  As that is what is most typically posted.

I usually suggest using Windows tools to edit windows (primarily shrink windows Vista or 7) and Linux tools to add partitions and format Linux partitions. Windows XP does not have partition tools you do use gparted for XP.

While most Linux tools will work with most windows, there have been issues. Windows is proprietary and does not share, so everything has to be reverse engineered. Windows also does not "see" Linux partition and has corrupted partition tables or creates its own proprietary SFS or dynamic partitions which do not work at all with Linux.
