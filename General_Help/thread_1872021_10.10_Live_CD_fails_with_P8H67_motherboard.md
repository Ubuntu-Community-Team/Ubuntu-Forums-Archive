Try using a LIVE USB stick of Ubuntu 11.10 and booting from that.. Could be a driver issue... Maybe the latest Kernel will help in this situation.. 

[https://debianhelp.wordpress.com/2011/09/12/to-do-list-after-installing-ubuntu-11-10-aka-oneiric-ocelot/](https://debianhelp.wordpress.com/2011/09/12/to-do-list-after-installing-ubuntu-11-10-aka-oneiric-ocelot/)

If you can prove that it is an updated Kernel that is needed, and you really want 10.10.. you can create a customized version of your own Ubuntu with the 3.0 kernel in it, and use Mainline, and use something like Relinux to make a live disc with it. But you will need another computer that will install 10.10 to use to make the remastered Ubuntu 10.10 you need to create with Relinux.  Don't do it virtualized 11.10..   Only trouble that way.

[https://debianhelp.wordpress.com/2011/10/12/relinux-a-new-fork-of-remastersys/](https://debianhelp.wordpress.com/2011/10/12/relinux-a-new-fork-of-remastersys/)

That way you will have 10.10 installation disc with the latest Kernel on it.
