### Post by Cheesemill on 2012-08-01
You need to use sudo, this is one of the tasks it's designed for:
```
gksudo -u colin.pidgin pidgin
```If you need to be able to run this command without entering a password then you need to edit /etc/sudoers, however, do not edit this file directly. You should always edit it by doing:
```
EDITOR=nano visudo
```Add the following line to the bottom of the file:
```
<username> ALL = (colin.pidgin) NOPASSWD: /usr/bin/pidgin
```Replacing <username> with your actual user name (colin I presume).
 
Your user can now run pidgin as the colin.pidgin user without having to type a password
