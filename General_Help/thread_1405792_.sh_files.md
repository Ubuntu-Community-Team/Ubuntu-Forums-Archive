You should keep your executable scripts in a specific directory such as ~/bin (e.g /home/*yourlogin*/bin.  Then you need to add this directory to the $PATH variable.  This is done in the .bashrc file.  See [**_[COLOR="Navy"]here [/COLOR]_**]("http://forum.vectorlinux.com/index.php?action=printpage;topic=10999.0")for an explanation.

### Post by squareff255 on 2010-02-13
Hey.

You can use your terminal to navigate to the Desktop:
```

$ cd ~/Desktop

```
and then you can use the sh command to run it. For example:
```

$ sh whatever_your_script_is_called.sh

```
That's all it takes. :)

Another example: I just wrote a script today that creates a folder, then creates 200 empty text files inside that folder as well as another folder. It then navigates inside that subfolder and repeats the process 3 times. Kinda stupid, but I find joy in the simple things. :) So, if I have the script in my Documents folder, but my terminal is currently parked on my Desktop or in my Home folder, I can run this command:
```

$ sh /home/administrator/Documents/asdf.sh

```
and although the script is in the Documents foder, it will still create the first folder in whatever place my terminal is parked at (like my Home folder) and then create all the text files in there and so forth. If I navigate to the Documents folder:
```

cd ~/Documents

```
then I would only need to use:
```

$ sh asdf.sh

```
to run the script and it would do all of it's business in the Documents folder cuz that's where my terminal is parked.

GET IT? :)

Happy Scripting

### Post by SPr on 2010-02-13
> **fwin said:**
> 
Is there any way to run this just by double clicking the file. In other words i just want to double click on it and then see the message hello world in the terminal (like in windows).

Go to the desktop and double click the icon for the script. In Gnome a prompt opens asking what you want to do with it. Choose "Run in terminal". I don't know what happens with KDE or XFCE.

The terminal will disappear as soon as the script finishes so add the following line to the end of the script.
```

read -p "Press enter to exit. "

```

Running the scripts from the terminal is much easier than double clicking icons. The command "./my_script.sh" only needs typing once as using the up-arrow key will scroll through previous commands.
