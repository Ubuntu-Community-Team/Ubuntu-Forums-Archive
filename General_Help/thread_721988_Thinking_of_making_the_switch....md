### Post by Zorael on 2008-03-12
Warning: tin-foil hat conspiracy ahead. Here be dragons.


First of all, credit to you for being open-minded enough to consider it!


There's plenty of applications to suit your needs; from several basic notepad-like programs to developer editors such as eclipse. Remember; linux *is* desktop-ready, by this point.

The main issue is lack of corporate support, and bullying from Microsoft. There aren't any (official) native Flash editors, much like there aren't any native Photoshop executables. To an extent, you can install open source alternatives to these, but they generally enjoy less features. Enter wine; with it, you can run many/most popular proprietary Windows programs, such as Photoshop, and some versions of Internet Explorer. See [the app database over at winehq.org](http://appdb.winehq.org/).

There aren't many official drivers, either because the manufacturers believe linux is a fad, that the userbase isn't large enough to warrant the effort, or - again - because of contract-bound Microsoft-bullying. Most drivers are written by reverse-engineering the devices. That said, you may experience issues with some hardware (wireless is a touchy point, for instance, as are any very new hardware like the latest Nvidia cards); but unlike Windows, most of your hardware will already have their drivers as soon as the operating system has finished installing. Meaning, no hounding the internet to get that videocard to display more than 800x600; it already does. linux *has* the best driver support, after all.


As for shell commands, Ubuntu is a linux distribution like any other, so naturally there are terminal applications for you to toy around with the linux shell. Distributions are just a bundle of select packages + specific art; meaning, you can install Ubuntu and then go install any linux program not included in that distro. They're not distro-specific, per se.


Lastly - and this I **emphasize** - do try the other flavours before defaulting to Ubuntu. I installed Ubuntu on my main system, then experimented and installed Kubuntu on my laptop. Afterwards enlightened, I tried Xubuntu again on my main system to try it out, then realized I'd fallen for Kubuntu. Which I now have installed on all my computers. All these share the same package repositories, so you can install any Ubuntu apps on your Kubuntu installation, and vice versa. There may be some performance loss as they're written in different - uh - window languages (Qt/Gtk), but unless you're very short on internal memory you generally needn't worry.

See [this page on KDE/Gnome controversy](http://www.cbbrowne.com/info/qtcontroversy.html).

Again, the only difference is the bundled packages; installing Kubuntu alongside Ubuntu is as easy as typing a command in a console. Key word: alongside. So you can essentially install Ubuntu, then migrate to Kubuntu. Etc etc. It's very, very modular. And Microsoft are doing their best to ridicule it, making people think it's immature and bad.

So, grab three cds, download (at least) Ubuntu, Kubuntu and Xubuntu, then run each and try them out as LiveCDs before deciding. No basic blue-background Windows installer here. :>
