### Post by delcypher on 2009-05-03
I don't know much about qemu so I don't know what the IP address of the physical host is from the virtual machines perspective.

Your qemu command is wrong. You have not specified a share under the -smb option. You need to create an smb share (e.g. /home/username/mysambashare) and then your qemu command should read.```
qemu -boot c -fda /dev/fd0 -cdrom /dev/cdrom -smb /home/username/mysambashare -hda /home/labinnsw/qemu/hd.img -m 1024 -localtime -net nic -net user -usb -usbdevice host:05dc:a400
```


Anyway I looked at your smb.conf file and I can't see any shares in there, all I see is the printer. e.g. ```
[share_name]
path = /media/disk-1
browseable =yes
read only = yes
guest only = yes
guest ok = yes

```

You need to check that samba is working properly in your system 1st.

Try ```
smbclient -U guest -L localhost

``` from within your physical machine (Ubuntu), when it asks for a password just press enter.

You should get a list of services that you are hosting, mine looks like this. ```
Domain=[DX2] OS=[Unix] Server=[Samba 3.2.3]

	Sharename       Type      Comment
	---------       ----      -------
	IPC$            IPC       IPC Service (dan-desktop server (Samba, Ubuntu))
	external_drv    Disk      
	windows         Disk      
	print$          Disk      Printer Drivers
Domain=[DX2] OS=[Unix] Server=[Samba 3.2.3]

	Server               Comment
	---------            -------

	Workgroup            Master
	---------            -------
	WORKGROUP            DX2

```

If you don't get a list then your samba server isn't configured properly.

So to summarise, get your samba server working properly on ubuntu, then worry about getting qemu to use your share.
