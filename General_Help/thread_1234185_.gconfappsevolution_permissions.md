I use two user accounts on my Ubuntu installation that I switch between, and I thought I could give the second account access to the first account's evolution files, so I can just access the email when logged in as the second user.


(Both users belong to group admin).

However, the second user, who also belongs to group admin, still cannot access the directory and subdirectories. Why?

However, the second user, who also belongs to group admin, still cannot access the directory and subdirectories. Why?

Probably because the .gconf folder is a system folder and is specifically set up **not** to allow other users access for quite specific (and valid) security reasons.

You may be able to set up foreign user access to the mail files, you are **not** allowed access to the credentials/settings.
