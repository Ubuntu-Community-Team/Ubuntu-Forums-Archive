### Post by OffAxis on 2008-04-04
I'm not certain how you killed the tap so there's a few possible solutions.

First of all, before you start(ed) messing with anything, you should back up the configuration file which is [B]/etc/X11/xorg.conf
[/B]
```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.backup
```


If you were messing with the synclient command and killed it then a restart of the xserver will reset it it default (which is working).

The settings for the tap (for the whole touchpad) are kept in the /etc/X11/xorg.conf file.
Look for the section labeled "Synaptics" (this is the name of the driver).

There's a few things that might be screwed up:
TouchpadOff option should be set to 0, not 1 or 2.
MaxTapTime should be 100, not 0.

So the format would look like:
```

Option "TouchpadOff" "0"
Option "MaxTapTime" "100"
```

I'm not sure what these options should be set to, but they may be the problem:
MaxDoubleTapTime
ClickTime
FastTaps
MaxTapMove
TapButton1

If all the options aren't there don't worry about it.
