### Post by mcduck on 2009-03-31
Synaptic lists all packages available in all the repositories you have added to it's list, plus all the extra packages you have installed.

"Mark for Removal" removes  package but leaves it's configuration files. "Mark for Complete Removal" removes package and it's configuration files. (I'm now talking about system-wide configuration files, user's personal configuration files are never removed by the package manager).

synaptic package manager displays the packages that are available to you based on which repositories you have enabled.  i think the only difference between mark for removal and mark for complete removal is that mark for complete removal will remove the package itself and any configuration files that go along with the package whereas mark for removal will remove the package only.  either way you will still see the package in the synaptic package manager.
