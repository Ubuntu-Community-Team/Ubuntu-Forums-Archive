### Post by Impavidus on 2017-06-21
If you want to see your RAM used, take your web browser, open 40 tabs and load 40 heavy pages. Or run a video editor, a couple of virtual machines, a flight simulator or do some scientific data processing. I once ran out of memory using grep, scanning for some 2000 patterns. But ordinary office applications don't need more than a few tens of megabytes. I just opened a 570kB gnumeric spreadsheet with 20 years of weather data: 96MB ram, or 1.2% of what I've got installed.

The CPU operates on data, RAM stores it. The only trade off between them is that you can choose between using RAM to cache results or the CPU to recalculate them when you need those results again. When dealing with amounts of data large enough to care about this, most application do pretty well (and most choose to cache).

If a program is not designed to use multiple cores, it will use a single one. Many programs are nowadays designed to run on multiple cores, but even then it may do some things with a single thread because not everything can be parallelised easily. So your spreadsheet application has a lot of work to do when saving your spreadsheet and does so in a single thread. The operating system switches that thread sometimes from one core to another, but it cannot be on multiple cores at once. Once finished with this serial job, the work is distributed over multiple cores in parallel. As long as the application is busy, the window manager thinks the application is unresponsive and makes the window grey. Other applications should still be responsive. Are they?

In any case, it seems that saving your spreadsheet is a very CPU intensive task. The question is whether this is to be expected or something is wrong and if so, what. There must be something about this spreadsheet that's different from other spreadsheets. Are there many formulas in it?

I noticed since a few versions back that Firefox sometimes saturates one core for ten seconds or so and becomes a bit less responsive during that time. It doesn't bother me too much, just slightly annoying, but maybe that's the same you experience in browsers.

> 
How would I "choose between using RAM to cache results or the CPU to recalculate them"?

I didn't really mean you specifically, but more in general. That's a design choice made by the programmer.
