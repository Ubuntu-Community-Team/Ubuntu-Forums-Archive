### Post by Ayuthia on 2010-10-22
You can try:
To turn it off
```

xinput set-prop 'N-Trig MultiTouch' 'Device Enabled' 0

```
To turn it on 
```

xinput set-prop 'N-Trig MultiTouch' 'Device Enabled' 1

```

When you have the N-Trig MultiTouch device, the N-Trig Touchscreen does not report anything.  The device uses the N-Trig Touchscreen for single-touch reporting when the multitouch firmware is not present.

I hope that helps.

### Post by utilitytrack on 2010-10-22
> **gohanssjn said:**
> Did not work

Try this for to make this settings persistent:

to disabple a touchscreen
```
sudo echo "xinput --set-prop 13 'Device Enabled' 0" >> /etc/gdm/Xsession

```
to disable, for example, a Pen stylus 
```
sudo echo "xinput --set-prop 12 'Device Enabled' 0" >> /etc/gdm/Xsession
```

### Post by Ayuthia on 2010-10-22
This one is not as pretty, but if that one does not work you can try this method:
```

cd
gedit notouch

```
then add:
```

#!/bin/bash
xinput set 'N-Trig MultiTouch' 'Device Enabled' 0

```
Then save it.  Next:
```

chmod 744 notouch

```
Then go into System->Preferences->Startup Applications->Add
Then add the following:
```

Name: Touch Off
Command: /home/username/notouch
Comment: Turns off touch

```
But replace username with your username and save it.

When you log out and log back in (or any time you restart), touch should be turned off.
