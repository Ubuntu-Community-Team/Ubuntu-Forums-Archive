### Post by mtippett on 2011-05-16
What are you trying to achieve?  What sort of tests do you want to run?

A good way of reducing the number of tests is working out what you are using the old or new machine for?  There are usually some tests that map into those usages.

The other option is to look at what annoyed you about the old system that triggered the upgrade?  Was it slow compiling (there's a suite for that), was video decoding painful (there's a suite for that).
