### Post by drs305 on 2011-11-12
Mathivh,

Welcome to the Ubuntu forums!

First, it's best to start your own post rather than use one created by another user. Even though the problems may appear similar, often they are not and it just confuses things.

But since you have posted the script...

Grub 2 was installed in the Windows partition and that is what is causing the problems. Boot the Live CD and then open a terminal and run the following commands. They will mount your real Ubuntu partitions and install Grub 2 to the correct location.
```

sudo mount /dev/sda7 /mnt
sudo mount /dev/sda5 /mnt/boot
sudo grub-install --boot-directory=/mnt/boot /dev/sda
sudo umount /dev/sda5 /dev/sda7
```

Reboot after removing the CD. If you don't see a Windows entry, run:
```
sudo update-grub
```

### Post by oldfred on 2011-11-12
Partition table says sda1 is 07 or NTFS but boot sector is blank. All NTFS partitions have to have a NTFS PBR or partition boot sector.

NTFS keeps a backup which may be ok. If so you can use testdisk to restore it. Otherwise you will need a Windows 7 repair CD to fix Windows. Testdisk has a rebuild NTFS boot signature and that may help, but you still need a Windows 7 repair disk, since the one created by testdisk is a basic or possibly XP type. Did you make the Windows repair CD while Windows worked?

Fix for most, a few have other issues, better than windows fix in many cases as it also fixes other parameters:
If win7 use small 'system reserved' NTFS partition instead of the partition where windows was installed for win7
This has instructions on using testdisk to repair the install of grub to the boot sector for windows from Ubuntu or Linux LiveCD.
[http://sourceforge.net/apps/mediawiki/bootinfoscript/index.php?title=Boot_Problems:Boot_Sector](http://sourceforge.net/apps/mediawiki/bootinfoscript/index.php?title=Boot_Problems:Boot_Sector)
Also check for /boot/grub in addition to /Boot

You will also need to rerun testdisk for sda2 as it is also a NTFS partition but has grub in the PBR.

### Post by oldfred on 2011-11-29
You can run the fixMBR command, and later using Ubuntu liveCD reinstall grub2's boot loader after you fix Windows.

But just installing the MBR for Windows will not fix it. You need to run the full set of commands on the bootable partition. Windows repairs only work on the partition with the boot flag as that is the Windows active or bootable partition. It may just need chkdsk to clean up the mounting or you may have moved/deleted the windows partitions which would not be repairable.

### Post by oldfred on 2012-01-16
So Windows boots normally with the Windows boot loader in the MBR? 

The fixMBR command just puts the Windows boot loader in the MBR. 

The Windows boot loader looks for the active partition (boot flag) and jumps to the rest of the boot code in the PBR - partition boot record/sector. When grub boots to Windows it just  jumps to the PBR exactly like Windows does.
