### Post by zemeron on 2009-03-24
I'm primarily a windows system admin but I'm giving ubuntu another try and I'm having a problem with attempting to access FQDNs (fully qualified domain names) from ubuntu. For example if I _*ping internaldomain.local*_ I recieve _*ping: unknown host internaldomain.local*_

If I do _*nslookup internaldomain.local*_ I receive the IPs of the Domain Controllers. 

My /etc/hosts file:
-----------------------------------------------------
127.0.0.1       localhost
127.0.1.1       nandLNX01

# The following lines are desirable for IPv6 capable hosts
::1     ip6-localhost ip6-loopback
fe00::0 ip6-localnet
ff00::0 ip6-mcastprefix
ff02::1 ip6-allnodes
ff02::2 ip6-allrouters
ff02::3 ip6-allhosts
-----------------------------------------------------


My /etc/resolv.conf
-----------------------------------------------------
# Generated by NetworkManager
domain internaldomain.local
search internaldomain.local
nameserver 172.16.0.50
-----------------------------------------------------


FQDNs work properly if I remove internaldomain.local from resolv.conf, but then naturally I always have to specify FQDNs which I would prefer not to do. In windows I can choose to use the short name and have it append my domain or use the FQDN.

So I guess my question is does Ubuntu handle FQDN lookups differently than windows or is there something wrong with my config?
