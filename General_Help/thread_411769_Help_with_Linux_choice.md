### Post by kidders on 2007-04-17
Hi there and welcome,

Fundamentally, there isn't a huge difference between the various Linux distributions, so in that sense, your choice isn't critically important. The major differences tend to include...[LIST]
[*] Distro-specific modifications to the generic Linux kernel (core operating system) that a particular community considers desirable.
[*] Superficial changes to the way certain graphical environments look.
[*] Significant differences in how software management is done.
[*] The regularity of system updates (bleeding edge versus scheduled upgrades, with an emphasis on stability).
[*]Suitability (or lack thereof) for certain hardware ... CPU architecture, for instance.
[*]Etc, etc...[/LIST]So essentially, if you choose one and turn out not to like it, most of the skills & knowledge you pick up are readily transferable to other distros.

If you opt for Ubuntu, you would be getting a relatively stable, friendly operating system that only distributes free (a word which has special meaning to Linux users) software, and is specifically intended to be suitable for recent Windows converts. In my experience, its user support is excellent, but can falter a little, if you start doing unusual or advanced things, and get stuck. I can't say that it would be any more (or less) suitable for multimedia work than any other distro though.

The best advice would be to try one or two Linuxes ... give each a few days ... and see what you think.

### Post by kidders on 2007-04-17
I don't think that there is, really ... although I'm open to correction. Your choice of distro doesn't normally limit your software choices, for instance.

In my opinion, people interested in rendering, video editing, etc. *might* want two things:

[U][B]I - Bleeding edge drivers & software
[/B][/U]You may find that Linux is the wrong operating system, when it comes to issues like these. Because hardware manufacturers (Nvidia, ATI & Canon are good examples from the media world) can sometimes be quite protective of their APIs, Linux is often the last platform to acquire support for new hardware. In software terms, many would argue that MacOS (which is quite similar to Linux) has the best multimedia development apps.

Ubuntu keeps pace with hardware developments quite well, but isn't particularly suitable for users that like to run development & pre-release versions of applications.

_**II - Extreme configurability**_
Linux is certainly great for customisability. You might, for instance, want to rewire your OS so it is extremely tightly optimised for your hardware, and do the same for all the applications you run. If you have the expertise required to configure specialised build environments & compile software (for rendering, perhaps), then Ubuntu is _definitely_ the wrong distro to be using.


If you're not interested in making seeeeerious demands like these, that Ubuntu simply isn't interested in catering for, then it might well be perfect for you. :-)
