> **Omegatron said:**

Also, this line in your /etc/fstab doesn't make any sense, and needs to be removed...```
/dev/ /media/floppy0 auto rw,user,noauto 0 0
```

I wonder if your problems are being caused by these issues. It seems unlikely to me, but it's worth checking anyway. Since Linux is not designed to handle block devices the way Windows users like it done, the mechanisms Ubuntu uses to do so are not very straightforward. Before going there, you might as well check everything else first!

Also, glad you sorted your Kubuntu out, vitalik!

### Post by kidders on 2007-04-28
Hey again,

> **Omegatron said:**
> I have no idea what it is.  Where would a CDROM drive show up?That depends on where it's plugged in. For instance, you'd expect a serial optical drive to show up at someplace like /dev/scd0 ... then again, an IDE one would turn up at /dev/hd[a-z], depending on the controller its attached to. (For example, /dev/hdc would correspond to the primary device on the secondary IDE controller.)

> **Omegatron said:**
> Why does this not make any sense?"/dev/" is a meaningless device node. That line should either be removed or corrected asap. It's pretty important that _everything_ in /etc/fstab be syntactically correct, and accurately reflect your system ... if not, you will encounter problems (eg odd behaviour or difficulty booting). You should really make sure you understand everything in it, imo.

Anyhow, here's a very brief summary of what should be happening when you plug removable devices into your PC. I've used my own box & a USB stick for the sake of an example...

[U][B]dmesg
[/B][/U]Something like the following should be appended to your dmesg almost immediately, when you plug something new in.

```
scsi 7:0:0:0: Direct-Access     USB 2.0  MobileDisk N4HY  1.00 PQ: 0 ANSI: 2
SCSI device sdd: 253952 512-byte hdwr sectors (130 MB)
sdd: Write Protect is off
sdd: Mode Sense: 03 00 00 00
sdd: assuming drive cache: write through
sdd: sdd1 sdd2
sd 7:0:0:0: Attached scsi removable disk sdd
sd 7:0:0:0: Attached scsi generic sg4 type 0
```That tells me that a new device, connected to my system at /dev/sdd has appeared, with two partitions (/dev/sdd1, /dev/sdd2).

_**fdisk**_
Just for the sake of it, you might like to use fdisk to see what's there.```
$ sudo fdisk -l /dev/sdd
   Device Boot      Start         End      Blocks   Id  System
/dev/sdd1               1          16      122880    b  W95 FAT32
/dev/sdd2              16          16        4032   af  HFS
```_**mount**_
You can check to see whether Ubuntu has automagically mounted anything, and find out where.

```
$ mount | grep sdd
/dev/sdd1 on /media/KIDDERS type vfat (rw,nosuid,nodev,shortname=mixed,uid=1000,utf8,umask=077)
/dev/sdd2 on /media/UID type hfsplus (rw,noexec,nosuid,nodev)
```If, for some reason, Ubuntu didn't mount anything, I could always have done so manually ... keeping an eye out for error messages that might explain what went wrong.

So, I guess the first question is (once your /etc/fstab is sorted out), how much of this does *your* Linux box do?
