### Post by blackbird3216 on 2011-07-25
One amazing difference that I actually learned yesterday is that linux, thanks to the X window system, has two forms of copy and paste. 

First, you have the generic "right click" popup menus, wheere you can pick copy/paste/cut just as usual. This is the same as ctrl c,p,x. So that means whatever I copy with the context menu, I can paste with ctrl p. 

However, I just learned that there is an amazing feature on the X-window system. The best way to try it is in Firefox/Chrome. Simply highlight the text you want to be copied, and without pressing any additional buttons, move the cursor over to the location where you want to paste it (e.g. search bar). Then, press the middle mouse button and it is copied. Amazing! The best part is that it doesn't interfere with the other two copy and paste methods (mouse+keyboard).

### Post by coffeecat on 2011-07-25

> Amazing!

Another nice trick involving a mouse button is if you want to type a very long path including the filename at the end in a terminal command. Simply open a Nautilus file-browser and navigate to the file in question and then drag and drop it into the terminal. Useful if you're editing a system file -  type "gksu gedit" into the terminal, and the drag and drop the file. The line is filled in for you.

You can do the same with a MacOS terminal, but just try with a Windows command prompt! :-s
