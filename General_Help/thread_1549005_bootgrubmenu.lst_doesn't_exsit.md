### Post by Vaphell on 2010-08-09
/boot/grub/grub.cfg is not supposed to be manually edited
disclaimer at the beginning of the file:
```

#
# DO NOT EDIT THIS FILE
#
# It is automatically generated by /usr/sbin/grub-mkconfig using templates
# from /etc/grub.d and settings from /etc/default/grub
#

```

it is created based on mentioned files. That way you don't risk getting broken config, if the settings are bad for some reason, grub.cfg file won't be overwritten and your old working configuration will remain.

### Post by oldfred on 2010-08-09
But Grub 2 also lets you use the title. Suppose your Windows title is "Windows Vista (on /dev/sda1)". Then you can use in /etc/default/grub, and you won't have to edit Grub after kernel updates.
to copy entry from:
gedit /boot/grub/grub.cfg
To paste entry into ( I have to have both windows open for the copy/paste to work):
or
cat /boot/grub/grub.cfg | grep Windows
gksudo gedit /etc/default/grub
Change GRUB_DEFAULT=0 to your windows entry like this as example
GRUB_DEFAULT="Windows Vista (on /dev/sda1)"

But many prefer a graphical way. Download this:
Startup manager in synaptic will allow you to set boot default and the timeout even in lucid.

### Post by oldfred on 2010-08-09
The grub default is just which line to boot. The new grub2 lets you use names in place of number. You do not need 40_custom for any of the standard entries which should work for most users. 

For myself with many installs and those who want to customize the menu entries, using 40_custom is a way to customize some or most entries. I like to add a spacer line just to make menu easier to read, but once set the grub default to the spacer line and had real fun trying to figure out why my grub all of a sudden stopped working.

The grub sites and drs305 have all the info but it is a lot to absorb. I did a little at a time as I was converting from grub legacy. Grub2 offers many more features but then is more complex.
