### Post by Chronon on 2009-06-18
Many applications are coded using gnome or KDE libraries.  This makes them run lighter on those desktops since their dependencies are already loaded.  While you can run KDE applications in Gnome and vice-versa, you'll probably find that memory usage and such is a bit lighter if you run KDE apps in a KDE desktop, etc.

I have both KDE and Gnome installed on my desktop machine.  I mostly try to run the Gnome apps while in the Gnome desktop (and KDE apps when running the KDE desktop).  As the previous poster mentioned, you can choose which desktop you wish to load when you login.

### Post by Chronon on 2009-06-18
That sounds about right.  It's all the same kernel.  As far as I understand, it's just a matter of which interface you load on top of it to make use of it.  A desktop suite basically consists of a set of applications that allow you to interact with the kernel and share dependencies on a certain toolkit (libraries).  Of course you can mix and match these in any way as long as you're willing and able to load all of the libraries that are needed by these applications.  Obviously, there's aesthetic as well as practical appeal to choosing a suite of programs that share dependencies.

### Post by DeMus on 2009-06-18
The big difference between the various Ubuntu flavors (and also other Linux distros) and Windows is this:

In Linux you have a kernel, the basis which is really the OS. On top of that to make it easier to operate for most people there are graphical display managers like Gnome, KDE, XFCE and others. The GDM's are separate from the basis, from the kernel. You can also control the machine without a GDM, just by using the Command Line Interface. (looks like DOS in the eighties).
Each GDM has it's own set of programs it uses to do the normal every day tasks.

Windows has everything integrated. The kernel, which is still DOS although MS doesn't want to call it DOS, and the graphical part has been merged together. Hence, there is no choice, there is one system and that's what you gonna get when you buy "the right to use it".
The first Windows was more like Linux today, Windows 3.1 was nothing more than a graphical shell on top of DOS. But they changed it with Windows '95 and went down hill from there. (must be a deep valley since the bottom is not in sight yet).
