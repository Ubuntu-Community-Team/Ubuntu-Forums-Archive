### Post by gtdaqua on 2008-06-19
You wont see any remarkable difference in 64-bit desktop editions. They could be a pain when you try to install apps! You have to convert them or rely on a patch or a workaround. Its never straight.

32 bit processors, with the right cpu/motherboard, can access up to 64 Gb RAM (the 4Gb limit on a 32 bit processor is simply not always true).[/i]

> **Physical Address Extension** (**PAE**) refers to a feature of [x86]("http://en.wikipedia.org/wiki/X86") and [x86-64]("http://en.wikipedia.org/wiki/X86-64") processors that allows more than 4 [gigabytes]("http://en.wikipedia.org/wiki/Gigabyte") ([GB]("http://en.wikipedia.org/wiki/GB")) of [physical memory]("http://en.wikipedia.org/wiki/Physical_memory") **to be used in [32-bit]("http://en.wikipedia.org/wiki/32-bit") systems**, given appropriate [operating system]("http://en.wikipedia.org/wiki/Operating_system") support.[http://en.wikipedia.org/wiki/Physical_Address_Extension](http://en.wikipedia.org/wiki/Physical_Address_Extension)

PAE is "built into" the openVZ and server kernels.

Again, quote:> Linux The [Linux kernel]("http://en.wikipedia.org/wiki/Linux_kernel") includes full PAE support starting with version 2.6.[[7]]("http://en.wikipedia.org/wiki/Physical_Address_Extension#cite_note-6") The 2.6 version of the kernel fully supports PAE, enabling access of up to 64 GB of memory on 32-bit machines. A PAE-enabled Linux-kernel requires that the CPU also support PAE.Second, you should install the version appropriate to you processor ie if you have a 64 bit processor -> use 64 bit Ubuntu.
