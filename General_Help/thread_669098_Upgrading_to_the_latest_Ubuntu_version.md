### Post by scizzo on 2008-01-16
1) Upgrading to 8.04 is not a good idea if you do not have experience with dpkg and apt-get. The system can break and it is nothing uncommon for development systems. Testing it is ok however keeping a stable system as the main system is the best. So if it works good on 7.10 then I suggest you stick to that.

2) When the release is done the manager will tell you that there is a new version out. (if you are using the update manager). It will then fix the packages however the config files can be a bit dodgy _IF_ the programs have changed in anyway. But most programs nowadays look at the old configs and ask if you want to revert them into the new version.
