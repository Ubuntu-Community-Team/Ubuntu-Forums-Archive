### Post by TheFu on 2022-07-18
I recall that Canonical used to have training for Ubuntu Certs, but that seems to have been dropped for the beginner level.
LPI has both training and tests for users and admins of Linux systems.  [http://www.lpi.org/our-certifications/summary-of-certifications](http://www.lpi.org/our-certifications/summary-of-certifications)

There are also free learning materials, but I can recommend the "LPI Linux Certification In A Nutshell" book.  Just start from the beginning and work your way through, chapter by chapter. That book is for admins.

If you aren't at least a power-user yet, start with "The Linux Command Line" [http://linuxcommand.org/tlcl.php](http://linuxcommand.org/tlcl.php)  This book can be bought in many languages from almost any bookstore or it is a free, non-hassle, PDF download from the author's website. 

I used both these books for my classes - end-users or admins.  To see if the admins were ready, on the first day, I'd ask them to create a tgz backup of /etc/ that created a date-stamped file.  That's 1 line. 1 command, perhaps 3, but only 3 in 25 could do it.  Linux admin is about creating little scripts like that.

The key to all self-learning is to make a workable plan, then to work it. Consistency matters.  Do 1 hr a day, perhaps with 2 days off each week, no more.

At the server level, 95% of linux is the same. Just the package manager and init files are different between the different releases, so when you master LVM on Ubuntu, you've mastered it for RHEL, SuSE, and all other distros too.  Those small differences are really small. Don't get caught up with "Ubuntu" as the only option.  Learn them all.  The ideas are all the same, just a few commands are slightly different and the release cadence of the different distros are different.
