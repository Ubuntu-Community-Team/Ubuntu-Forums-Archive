### Post by cariboo on 2009-10-29
Start in recovery mode, and when you get to the prompt, select drop to root prompt with networking.

Create a backup of /etc/X11/xorg.conf:

```
mv /etc/X11/xorg.conf /etc/X11/xorg.conf.back
```

Then at the prompt type **exit**, select resume from the menu, and you should be taken to your desktop. Depending on the graphics adapter, Karmic doesn't use an /etc/X11/xorg.cnf file any more.

Once at the desktop you will be prompted if there restricted drivers available for your graphics card.

### Post by tim.hamels on 2009-10-30
Hello,

I've solved the problem

I've added "i915.modeset=0" before splash in /boot/grub/menu.lst

[INDENT]*title           Ubuntu 9.10, kernel 2.6.31-14-generic*
*uuid            c9f6e5c3-90db-4869-a00f-11de1823796a*
*kernel          /boot/vmlinuz-2.6.31-14-generic root=UUID=c9f6e5c3-90db-4869-a00f-11de1823796a ro **i915.modeset=0** quiet splash*
*initrd          /boot/initrd.img-2.6.31-14-generic*
*quiet*
[/INDENT]
