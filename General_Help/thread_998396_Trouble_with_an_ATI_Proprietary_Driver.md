### Post by markbuntu on 2008-12-12
The driver is misinstalled, most probably because old driver kernel modules are present from before the upgrade. You must remove ALL the old flgrx drivers on your system and then try hardware manager again. If you installed any with synaptic or apt-get then remove them the same way. If you installed any using the ati automatic installer or some manual method then you need to open a terminal and go to /usr/share/ati and run the command

sudo sh ./fglrx-uninstall.sh

If this does not work you can try

sudo apt-get purge xorg-driver-fglrx

you should aso check in /var/lib/dkms/fglrx for any old drivers and remove them if you find any there.

### Post by quadproc on 2010-03-28
> **AFarris01 said:**
> Ever since upgrading to ubuntu 8.10 on one of my computers, i've been having an issue with the graphics on it.  It's using a ATI Radeon HD 2600XT, with the ATI proprietary driver installed with EnvyNG.  However, the driver does not appear to be being used.

The OP's problem has been solved so I won't speak to that issue, but I will say that I have found that the System -> Administration -> Hardware Drivers utility (actually named jockey-gtk) does **not** necessarily report the correct status.

On this system, jockey-gtk correctly finds and reports the fglrx driver but then **says** it is not in use, even though it is in use and is working correctly.

Two other comments: ATI is no longer supporting drivers for their older graphics cards, while Ubuntu after 8.10 requires a newer ATI driver; and if you have more than one graphics device in a system running 8.10 the X server can't figure out which device is primary and gives up - the workaround is to furnish a BusID in the xorg.conf file's Device section for the primary graphics card.
