### Post by drs305 on 2010-11-16
> **XEtedBear said:**
> 
If I understand it correctly, the 'LABEL=' information is not used in fstab - right?

I think the thread has covered the UUID issues. 

For some reason, we don't seem to discuss LABELS in fstab very often in the Ubuntu forums, but in fact they *can* be used in fstab and are a much much more user-friendly way of identifying partitions.

One reason labels probably weren't mentioned in the past was that it wasn't always easy to label certain formats. Commands had to be used in terminal, and sometimes additional apps had to be downloaded to provide the capability.

You can now easily label most partition filesystems (ext2/3/4, NTFS, vfat, etc) through Disk Utility (System, Administration, Disk Utility: Edit Filesystem Label); or with Gparted (Partition, Label).

The fstab format simply is:
> LABEL=mydata 	/mnt/data 	ext4 	users,exec 	0 2

### Post by endotherm on 2010-11-16
one advantage to labels, is that UUIDs change with each format of a drive, so if you modify a partition, and then attempt to remount, the process will fail because the UUID listed in fstab is invalid, whereas if you bind by Label, you can set your new HDDs label to be the same as the old one, and remount without editing the fstab. 

it's the same benefit that UUIDs hold over /dev names. if the /dev name of a drive changes, then the fstab needs revised, but with UUIDs, the /dev names can be dynamically reassigned with no affect on the fstab.

### Post by bwayson on 2010-12-17
In response to [XEtedBear]("http://ubuntuforums.org/member.php?u=641391"), you can find all the different "names" for disk devices and partitions under the folders under /dev/disk/.  See [http://ubuntuforums.org/showthread.php?t=283131](http://ubuntuforums.org/showthread.php?t=283131) for a good fstab tutorial.  While I understand some of the advantages of using UUID in fstab, it really does fail when a partition is reformatted after Ubuntu is installed (e.g. after a gparted live CD is used to resize partitions or a second Linux distro is installed after Unbuntu).  One method, besides using labels, is to use the disk's or partition's ID.  These are found under /dev/disk/by-id/.  And "ls -l" to that directory from a terminal will show the IDs and where they are linked to.  The ID survives reformats.  I believe the only thing that would change it is changing the number of partitions, but in this case, no naming scheme will save you.  The ID is the concatenation of the text "ata-" (or "scsi-"), the drive's model as read from the drive, a hyphen, the drive's serial number, and, in the case of partitions, the text "-part#" where '#' is the partition number starting with 1.  In fstab, the device is given as something similar to  (without the quotes) "/dev/disk/by-id/ata-WDC_WD400BB-60DGA0_WD-WMADK1424593-part1".  The rest of the fstab line -- the mount point, options,etc. -- remain the same.  openSUSE has used disk and partition ids in its fstab for several versions now.  I've converted three Ubuntu boxes to this method and have not experienced an issues (yet...) due to the change.
