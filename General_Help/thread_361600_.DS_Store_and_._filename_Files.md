### Post by dannyboy79 on 2007-02-15
well I cam sorry but i have to be real here, if you read the entire man page than you would have read that you can easily delete at the end of the find command. there are examples at the end of the man pages that specifically show you. so what you can try is to export what is found into a file, this way you can see what files you will be deleting. this is what you can try

sudo find / -name '_.' > files_to_delete

what this will do is look for any files that contain am underscore just prior to a dot and the put the results within a file called files_to_delete which will be saved into the directory you perform the search in. after you review the file and makme sure that you don't accidentally delete anything you don't want to, you can do this command

sudo find / -name '_.' | xargs /bin/rm -f

this will delete everything that was put into that file. let me know how it goes

### Post by dannyboy79 on 2007-02-15
actually I found something else out, I was thinking what would happen if when you output the find command to a file and then when you quickly browse it and realize that it found a file that you don't want it to delete, well then you would be screwed because that find command is gonna delete everything it finds. so  i did some more research for ya and found that you can have the delete function actually read a file as input of what to delete. So, what you will do now is write your find command and have it put it's results into a file like I showed you before. you can change what it is looking for by simply changing whatever is inside the quotes but you need to leave them so the find command doesn't think they are part of the command options themselves. you could change it to '/' or '\' and anything with that exact character in it will be found and put inside the file. now that you have done that, then you will want to open the file and just quickly look it over, you'll be able to tell right away if something you don't  want to delete was found because of it's location, the file will contain the entire path of the file it found, for example, when I did a search using '*screensaver', this is what was inside the file:
/home/daniel/.gconf/apps/gnome-screensaver
/etc/pam.d/gnome-screensaver
/usr/bin/gnome-screensaver
/usr/lib/gnome-screensaver
/usr/lib/gnome-screensaver/gnome-screensaver
/usr/lib/xscreensaver
/usr/share/doc/gnome-screensaver
/usr/share/doc/rss-glx/README.xscreensaver
/usr/share/doc/kde/HTML/en/kcontrol/screensaver
/usr/share/gnome-screensaver
/usr/share/xscreensaver

and for example I can quickly see that I only want it to delete things in the /usr/share/ sub-folder, so I would simply remove the other entries from the file that the find command outputted, i would SAVE IT, close it and then repoen it, it would now be this:

/usr/share/doc/gnome-screensaver
/usr/share/doc/rss-glx/README.xscreensaver
/usr/share/doc/kde/HTML/en/kcontrol/screensaver
/usr/share/gnome-screensaver
/usr/share/xscreensaver

then I would use this command to delete all the files that are within the file that was outputed from when I did the find, for example, I named the file, delete_me and since I performed the find when I was in my /home/username directory.

sudo rm -f < delete_me

and then it will actually use the delete_me file, read each line and remove (completely delete from your system) it and your done!!!

note: if you notice that it ends up finding directories that you brought over from the mac, then they won't get delteted even if their path is in your delete-me file because the remove command has to have an additional option given to it to remove directories and their contents recursively. if that were the case, than you would want to use this command:

sudo rm -rf < delete_me

and within your delete_me file, the line that would be for a directory would not look any different so this is where you'll want to be careful. let me know how it goes and it you need anymore help. bye for now
