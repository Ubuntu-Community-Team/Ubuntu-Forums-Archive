Running Ubuntu 11.04 on a bit of a Frankenstein machine; all I know for sure is it has an AMD64 Athlon, though the 11.04 install is 32 bit.

When first installing 11.04 the CD was laggy and required several tries to go through the process, but once installed the OS worked fine with no problems, with the main drive on /dev/sda. It's been running for several weeks.

Today a family member hit the "sleep" button on the keyboard, and when the screen went to black they moved the mouse around and pressed an unknown number of key combinations before finally forcing a hard reboot. (The joys of shared computers!) The main install now won't even get to GRUB. The BIOS screen displays, then goes to black with a blinking cursor. Leave it long enough and the cursor eventually displays "Read Error" and nothing else.

I have an 11.04 and a 10.04.2 live CD and have been trying both of them so that I can get in and run a fsck. Neither of them will run if I try to use the default live CD settings; they just get to the loading screen and cycle forever, with either silence or lots of spinning and clicking from the CD. The BIOS also seems to be spastic about deciding whether to boot from CD or not; sometimes it will, and other times it'll bypass the CD entirely to go to the "Read Error" message.

I tried running both CDs with edd=on, noapic, and acpi=off. That got me to the live CD desktop, but in both instances I have no use of the optical mouse, which hinders me quite a bit.

Once in 11.04 I swapped out of X into the tty and ran fsck. The first time I did this I got a stream of error messages that eventually wound up looping:

```

```

Gparted shows no devices. I used the [Bootinfo script]("http://bootinfoscript.sourceforge.net/") and got this:
