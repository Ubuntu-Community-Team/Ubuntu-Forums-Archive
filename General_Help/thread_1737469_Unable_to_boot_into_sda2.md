Your problem is that you've installed grub 2 to the boot sector of sda2. Vista will not boot now because it has a Grub 2 type Boot Sector. It should have a Windows type boot sector.
At what point did you install grub 2 to sda2?

### Post by Quackers on 2011-04-23
Boot from a Vista recovery disc (or installation disc) and enter the repair console. Ignore the automatic repair and choose instead the command prompt option. In the command prompt window enter ```
bootrec.exe /fixboot
``` and if no error is reported reboot.
In Ubuntu open up a terminal and run ```
sudo update-grub
``` and watch to see if the Windows Loader for sda2 is found. If so reboot and try to boot it.

### Post by -kg- on 2011-04-25
Quite confusing.  Here's my take...

In the beginning, you had access to Ubuntu, but not Windows.  You found that GRUB had been installed into sda2 (your main windows partition) and not sda5 (where it belongs).

You repaired the Windows bootloader (I'm assuming the one resident in sda2) and now it won't boot to anything.

Did you change the bootloader in the MBR?  If you repaired the bootloader in sda2, but did nothing with the code resident in the MBR, then that would still be the GRUB bootloader (Stage 1) in the MBR.  You would not be able to reset the code except by booting back into Ubuntu and doing a "sudo update-grub" command in the terminal.

Unfortunately, you couldn't boot back in, because if it's still the GRUB bootloader, it's still pointing to sda2 for subsequent GRUB stages.

The bootloader in the MBR is a very small segment of code that looks for further boot information from subsequent stages loaded in /boot, which is normally loaded on the "/" (root) partition.  Since you mistakenly loaded it to sda2, then repaired the bootloader in sda2, those stages would no longer exist anywhere, since you didn't install it to sda5, either.

Normally, you would get a GRUB error.  Since you don't, I could be completely off base here.  However, since you have fixed the bootloader in sda2, you might try re-installing Ubuntu and make ***sure*** GRUB is installed in the right partition.

The thing that really baffles me is how GRUB got installed in sda2 rather than in sda5.  The bootloader, yes...you can change the installation location during install time.  It obviously got installed in the MBR, or you wouldn't have been able to boot into anything except Windows (as you did before the installation).  The only way I could see it happening is if you did a manual partitioning and marked sda2 as the "/boot" mount point.
