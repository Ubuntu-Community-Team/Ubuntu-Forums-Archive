### Post by lordL on 2010-05-17
I've been using Gnome Do for a long time, and never really encountered any problems. A few days ago however, it stopped responding to the keyboard shortcut (Super+Space) that normally launches it. Using the tray icon I could still launch the app, and it worked in every way I tested it - except for the keyboard shortcut.

I found a solution (or maybe more of a workaround), and I thought I'd share it here in case anyone else encounteres the same problem: **It's as simple as changing the keyboard shortcut.** I set it to the "menu"-key next to the right Ctrl-key, which works like a charm. And it's not like I ever used that button for anything anyway.

As for the cause for the Super+Space-combination suddenly not working, I'm not sure. I did recently do some changes in CompizConfig Settings Manager, but I'm quite sure I didn't set up any new key-bindings that should conflict withe Do's shortcut.

If anyone's got a theory on what may be causing Super+Space to stop working, please do share it. Otherwise I'm also happy to stick with using the "menu"-key :)
