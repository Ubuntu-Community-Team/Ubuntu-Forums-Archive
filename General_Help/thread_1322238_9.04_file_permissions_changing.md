### Post by Citizen Bleys on 2009-11-10
Lately I've had problems with some of the files on my system randomly having their permissions changed.

The first one that I noticed is that whenever I reboot, I lose direct rendering:

```

marcus@wintermute:~$ glxinfo | grep rendering
NVIDIA: could not open the device file /dev/nvidiactl (Permission denied).
direct rendering: No (If you want to find out why, try setting LIBGL_DEBUG=verbose)
```

I resolved this by putting "chmod go+rw /dev/nvidia*" in my rc.local script.

Later, however, it happened again; Now whenever I launch a game (World of Warcraft), all of the Owner permissions for the folder it's in vanish.  Since it's a 2-step launch, the launcher.exe locks me out of the actual game unless I have my ~/.wine/drive_c/Program Files folder open in nautilus and *manually restore the permissions every time*.  This is unacceptable

Any ideas on what could be causing it and how to stop it?
