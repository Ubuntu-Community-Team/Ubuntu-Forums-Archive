### Post by spiderbatdad on 2010-01-05
The claim that another distro lets you do more advanced things than the Ubuntu distribution is completely bogus, a joke even. Ubuntu is considered a beginner's distro, primarily, due to the absolute simplicity of installation, in that the kernel detects and loads modules for most hardware. And there is normally nothing to configure to get a basic working desktop right from the installation cd. With this simplicity comes a price though, it is called bloat.

I disagree.  Whenever I do a fresh install I remove various default packages and install others, and this results in *ubuntu-desktop* being uninstalled.  But I haven't experienced any of this "serious trouble" that you fear.  And I have been customizing Ubuntu in this way since the time of Feisty.

I know that in Synaptic it warns "[Ubuntu-desktop] is also used to help ensure proper upgrades, so it is recommended that it not be removed."  So I guess that's where trouble might lie: in *upgrades*, not updates.  But this is easily solved.  Remove whichever packages you like; then, when you want to upgrade to the next version of Ubuntu, reinstall the packages so *ubuntu-desktop* is installed again, then do your upgrade.  Once that's done, you can remove stuff again.

Ubuntu has got its "beginner's distro" label because it's unlikely a beginner will have any problems when installing it.  But that certainly doesn't mean Ubuntu is *just* for beginners.  I'm no guru, but I've been using Ubuntu for a few years now, and I've never felt restricted by it.  Ubuntu *does* have a bit of a bloat problem, but that's easily remedied by removing some of the stuff included by default.  Ubuntu is a great distro for anyone who doesn't feel the need to compile stuff from source and build an individual system from scratch.  You can get things done without any fuss; and if you *want* to customize your system, you can.

### Post by lykwydchykyn on 2010-01-05
For Ubuntu, or any other distro, it's never about what you CAN'T do.  It's about what you can't do EASILY.

Ubuntu is designed to be easy-to-use for the "average user".  This often requires a more complex design under the hood.  If you want to dig around under the hood, it's a little more complicated with Ubuntu because of all the automation and moving parts.  Prime examples: GRUB2 vs GRUB1, Upstart vs. init scripts.  Even .deb package management vs. compiling things yourself.  These things aren't bad (in fact, from a use perspective they're awesome), but they add complexity to the design of the system.
