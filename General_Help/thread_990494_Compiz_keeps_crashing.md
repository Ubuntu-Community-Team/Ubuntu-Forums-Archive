### Post by loomsen on 2008-11-23
Check if you have the proper driver installed for your card, and also configured in your xorg.conf

> 
Checking for Xgl: not present.

A proper driver should fix this issue I think.

Just make sure you have it installed:
```

sudo apt-get install xserver-xorg-video-intel
```

And then make sure you have your xorg set up to use it:
```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf_backup && gksu gedit /etc/X11/xorg.conf
```

There's a section Device, make sure it has a line:
```


Driver    "intel"

```

Should look sth like

>  Section "Device"
         Identifier "Intel GM965/GL960" [color=red]NOTE: you may make up a name here, just make sure it's the same used in your serverlayout section[/color]
         Driver "intel"
         ...
       EndSection


*edit*

More information about options you may, or may not specify with your card, type:
```
man intel
```

### Post by loomsen on 2008-11-23
Actually my post was related to the Threadstarter, but however :)

As you yourself can see, you're already using the nvidia driver, in case you're using Intrepid as well you'll have to follow the advices in my sig and add yourself to the group video.

This is new in Intrepid I guess, which is why so many have the same problem.
Look:
[[img]http://www.ubuntu-pics.de/thumb/6277/screenshot_03_998Wms.png[/img]](http://www.ubuntu-pics.de/bild/6277/screenshot_03_998Wms.png)

Without being in the group video, and granting permissions to that group in your xorg, you won't be able to use hardware acceleration of your card. That's what causes the segmentation fault.

You can check your groups with:
```
groups
```

[[img]http://www.ubuntu-pics.de/thumb/6278/screenshot_04_60QwCE.png[/img]](http://www.ubuntu-pics.de/bild/6278/screenshot_04_60QwCE.png)

You can check for other users, if you have some on your system, by giving their names after groups.
If you wanna add a user to a group, do this: I first created the user foo, then added him to the group video, to enable 3D acceleration for this account:
[[img]http://www.ubuntu-pics.de/thumb/6279/screenshot_07_hSA89F.png[/img]](http://www.ubuntu-pics.de/bild/6279/screenshot_07_hSA89F.png)
Log out and back in. Check the link in my sig, explain a little bit more there.

*done*

### Post by loomsen on 2008-11-24
> **shadowdude1794 said:**
> Out of curiousity... what is the point of this? I can't think of any reason why any user shouldn't be allowed permissions for 3d acceleration ect.

Well, if you think of linux, or UNIX more likely, which linux derived from, as a multiuser system, it shouldn't be to hard to make up a reason.

Ressource allocation, just for instance.

Further, 3D accel is what comes out in the end. But actually it's about writing permissions on the nvidia-ctl file, which is an interface to control and communicate with your device. Now make up you're a sys admin of a system with about 100 users. Granting wrong permissions to users could ruin your whole network.

[Check this thread for a comprehensive approach to X and what it does](http://ubuntuforums.org/showpost.php?p=5939543&postcount=6)



Ideas:
Post your Xorg log, or more likely the parts of interest.
Do this:
```

cat /var/log/Xorg.0.log | grep EE

```
This will show any X related errors.

Further:
```

cat /var/log/Xorg.0.log | grep WW

```
will show warnings.

Post the commands output.
