### Post by russo.mic on 2009-05-06
Your going to have to provide more information.  Are you talking about a CD containing a program for windows?  Or just a data CD written for windows?

A data CD will read just fine on any OS.  A windows program can't be run natively in Linux.  Wine is an option, Virtualbox is another.

### Post by acimmarusti on 2009-05-06
Also, you don't need to rename your ms office documents (.doc, .xls, .ppt) to their openoffice counterparts (.odt, .ods, .odp) in order to "read" them in Linux. As long as you have openoffice installed (which is default in most linux distros), you can open ms office documents, and not only read, but write and edit them and keep their format. Even so with the newer ms office 2007 file formats

### Post by Cybie257 on 2009-05-06
> **Camilia said:**

Windows CDs can be read in Linux, while Linux CD's can be read in Windows. They follow the ISO standards. No need to reboot to read the data. As for installing software, different story. Some, and I quote "some" Windows software will install on Linux using WINE, which you need to install via your package manager. But, nothing Linux will install in windows. At least that I have seen to date.
