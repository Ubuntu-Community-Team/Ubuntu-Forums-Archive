### Post by Raptor_85 on 2010-05-09
This won't fix anything, but might help diagnosing the problem. When you get to the grub menu you can press e to edit an entry's boot parameters for a single boot. Edit your default entry and remove the "quiet splash" from the boot parameters (ctrl+x boots once you edit I think, the instructions are on screen). 

This will give you all the commands that are going on during the blinking cursor. Make note of what happens right after it thinks for a good amount of time. If you need to access that text once you boot, use the dmesg command in a terminal to output all the boot text.

### Post by Raptor_85 on 2010-05-16
Sorry, I meant to reply more quickly. The ATA 1: soft resetting link is the DVD drive being set to a slower mode. You can see it does this several times testing each speed. Make sure you DVD Drive jumper isn't set to slave if its the only one on the cable. Also if you have another IDE cable try swapping it out. That's what fixed my problem finally. 

Other than that you might be able to set your bios IDE configuration settings to a slower speed for that drive. It will have an effect on the transfer rate of your DVD Drive but I have no idea if its noticeable or not. It looks like the final mode it is setting is PIO4 and UDMA Mode 1.

I'm new to this forum, but not ubuntu - I've been using 6.06, then 8.04 and yesterday I decided to buy a new HDD for my laptop and put 10.04 on it. The laptop is a Fujitsu Siemens Amilo Xi2428, couple of years old already.

I enabled the proprietary NVIDIA drivers, this solved the first problem I encountered, namely out-of-control fan behavior (sounded like the laptop was preparing for take off). It also solved another problem, which manifested itself during downloads - the download would stall until mouse was moved or a key pressed.

Back on topic, I experienced the slow boot problem described here and in other threads, but it is by no means 'solved'. I've disabled the quiet splash options, enabled nomodeset etc etc - but the system stalls for about 60s at about 4.9s, last thing displayed is 'running /scripts/init-bottom' 'Done' 

I found a workaround for my system, installing bootchart. The first two boots were unsuccesful (crash, high fan speed). Third boot was fast, 30s to login. Then I uninstalled bootchart and behavior was back to slowness (although some error messages were shown about bootchart returning some code, which is strange, since I uninstalled it). Reinstalled and the system boots within 30s, **however** there is still a problem which I also had with 8.04: sometimes (totally random) the boot process stalls and I have to hit a key for it to continue. This is also very strange. 

It works now, but it's still the kind of magick I'd accept from Windows, not from a linux distro. Something, somewhere is still very much wrong, and I hope it will be solved. Note that the previous versions (8.04, 6.06) had no such problems on the same hardware.

Hope this helps,

Regards.
