### Post by Rocket2DMn on 2008-07-07
When you install a program from the package manager, it is precompiled and will automatically download all dependencies.  If for some reason it can't, it will tell you so, then you can come here and ask for help.  You don't normally need to use wget, or extract gzipped tarballs, or compile from source, or anything like that.

There are plenty of pages out there that can help you learn how to use the terminal.  If you google around, you will get a billion hits.  Since we are using Ubuntu, here is a link from the community documentation - [uhelp]community/UsingTheTerminal[/uhelp]

The Ubuntu Community Documentation is a great place to get help with different programs or otherwise setting up linux.  I find it easy to just search google for "site:help.ubuntu.com <type query here>".

If people give you commands to run, DEFINITELY feel free to ask them exactly what that command is doing, what the different parts of the command are, etc.  Most people will be happy to explain.

It was mentioned earlier that terminal is very powerful and easy to share.  It is easy for those of us helping out to post a terminal command rather than explain in text how to navigate a GUI, but most stuff can be done from the GUI as well as the terminal.  I know the terminal can be a bit scary, but it's really powerful and you should take some time to learn the basics as you go along.

### Post by cariboo on 2008-07-07
You can run Ubuntu completely by point and click. There really isn't a need for a newbie to use a terminal for anything. I suggest getting comfortable with Ubuntu before venturing in to the land of terminal. For those of us that have been using linux for several years, the terminal is a lot quicker than mouseing through a bunch of windows to get something accomplished.

Just to get you up to speed a little quicker check out Applications-->Add/Remove Programs there should be enough in there to keep you busy for a day or two. There are over 12,000 packages listed in the repositories, not all of them are programs, some are libraries that the applications need. There Are Games, Video and Audio applications and tools, and more. If you look through the different sections it will give you and idea of what is available.

### Post by kuja on 2008-07-07
> **xx.canes.rites said:**
> Yo thanks for welcoming me!

Also I just wanted to ask before I go take a shower and wait for some more responses. Where do I go to find out which companies support linux kernel based OSes? My logitech cam doesn't work because they don't issue drivers for Linux 2.26.16 kernel or w/e version Ubuntu Hardy is using, so I won't be buying from them anymore.

I want to build a gaming computer too specifically designed for linux so is there also a website or something that will show me which companies support linux or which computer parts are compatible or w/e.

You can use whatever processor you want, it'll be supported without issue.

When you're looking for a motherboard, google for something like "linux problems <NB chipset name>" to see if there are any problems with that particular chipset under linux.

Sound card .... Check the alsa website to see whether it will be supported under linux. [http://www.alsa-project.org/](http://www.alsa-project.org/) -- Note: avoid Creative X-Fi for now.

The very new ATI cards supposedly have better Linux support than any of there previous offerings, but I can't verify that seeing as I don't have one. If you go with an NVidia offering, stay a generation behind because the driver support will be much better (not to mention how much cheaper it is).

Games .... some can be run under linux and many cannot be. Check [http://appsdb.winehq.org](http://appsdb.winehq.org) and look up the things you'll be intending to play. On another note, idSoftware releases linux binaries for their games. Unreal Tournament also runs under Linux natively. Some other companies have tried to make Linux support better/easier ... you'll have to look around based on what you're interested in.
