### Post by WizRd on 2010-07-13
It sounds like your menu.lst file in /boot/grub/ is incorrect.  Does the grub menu even appear?  If so edit the default menu option with the 'e' key and change the line
 
```

root            (hd0,0)

```
 
You will need to figure out what the correct numbers are, which someone else might be able to give you a quick and easy way to do this.  The only way I can think to do this is with the live cd and running "sudo fdisk -l" from a terminal window.
 
Once you have the correct disk number and partition number you can then fix it in /boot/grub/menu.lst to make the change perminant.

### Post by prodigy_ on 2010-07-15
> **michaelespinosa said:**
> 
```
set root='(rockdev-root)'
search --no-floppy --fs-uuid --set 219c1638-4985-4334-8edf-f11e4780b7d4
```

These two lines in your grub.cfg look suspicious because there's no partition with UUID=219c1638-4985-4334-8edf-f11e4780b7d4. I'm not sure what /dev/mapper/rockdev-root is because it's not listed among your partitions either.

Anyway. Chroot into your installation and edit your boot config: ```
gksu gedit /boot/grub/grub.cfg
``` Try to comment these two lines out. Then change ```
linux /vmlinuz-2.6.32-21-generic-pae root=/dev/mapper/rockdev-root ro quiet
``` to ```
linux /vmlinuz-2.6.32-21-generic-pae root=/dev/sda1 ro quiet
``` save changes and reboot.

If this trick helps, you'll need to reinstall Grub2 with proper settings because editing grub.cfg directly is only a temporary solution.
