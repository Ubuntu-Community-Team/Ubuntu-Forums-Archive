### Post by BLTicklemonster on 2006-09-28
I'd like to test Edgy out before I install it, so I decided to take advantage of VMWare server by following (roughly) this guide: [http://www.ubuntuforums.org/showthread.php?t=183209&highlight=iso](http://www.ubuntuforums.org/showthread.php?t=183209&highlight=iso)

I have an iso of Edgy on a hard drive and set up vmware to install from it > "Edit Virtual Machine settings."

Click on the CD-Rom Drive and pick Use ISO.
Below that, type in the path to your Edgy ISO file, remember at this time it has to be done downloading. and it comes up eventually to the live cd desktop, but it the hard drive thrashes about miserably and the entire thing is unresponsive as heck, then I get an error telling me that the sound icon can't be placed because a sound card wasn't identified, then it locks up.

I know that you can change settings to include a sound card on an existing installation, but because I'm on a live disk, when I try, my options for all settings are grayed out, which means I can't make any changes. And because there's not an os installed, vmware tools is pretty much not going to install.

Has anyone else gotten edgy to install in vmware, and how did they overcome this (if they had a similar problem), or does anyone know how to correct this?


*edit:

Ah yes, I do have xp installed in vmware, so I opened the vmx file, and use the sound setting from it in the ubuntu.vmx file, and now I can get to the desktop in live cd complete with sound. (upped the ram to 267, too)


I'm in Edgy in VMWare server on Dapper right now posting this. Appears sluggish and "laggy" for lack of a better term. I need the vmware tools, of course, and there's an update pending. Looks pretty sharp.

Hope this helps if anyone has problems.
