### Post by andso on 2008-08-04
try this
edit the file /etc/hosts and modify the first line :
127.0.0.1 localhost **name-machine**
127.0.1.1 name-machine
You should notice a crazy performance boost.
->Reasons
This is a fix for the new system that was started back in edgy where the hostname was split off to 127.0.1.1, the problem is that some applications still look for the host name @ 127.0.0.1, so to keep those applications happy and running smoothly you simply need to add the host name where those applications expect it to be.

You need to use the command:

```
cp -r /home/j/Desktop /media/BACKUP-DATA/Ubuntu-backup
```

You can't copy a folder by using "cp [folder] [location]" because, cp is used for files. With the addition of "-r" it recursively copies the folder and file(s).

Also, by default you cannot login as root in Ubuntu. Instead, use "sudo" before the commands, then enter your own password. sudo means super user do, and allows for temporary administrative privileges. If there are multiple commands that need sudo access, then type "sudo -i".
