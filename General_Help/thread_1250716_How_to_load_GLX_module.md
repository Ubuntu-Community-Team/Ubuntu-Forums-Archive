### Post by bchurchill on 2009-08-27
try 

sudo modprobe glx

if that works, then you can add that line to your

/etc/rc.local 

for a quick work around.  If not you're missing the glx module... like me on Arch Linux.

EDIT:: oops, I see you've already tried that.  Exact same issue on Arch.  Removed nVidia drivers (using some open source ones instead), very similar glxinfo (I also get a segfault at the end and several lines saying Xlib:  extension "GLX" missing on display ":0.0".)

FURTHER EDIT::
 One failed recomendation was to put

direct rendering: Yes
Load "glx"

into /etc/X11/XF86config but this hasn't helped.

I'm on an x86_64.  Are you as well?
