### Post by coffeecat on 2011-10-16
Welcome to the forum!

You have a wubi install, which uses a virtual disk. This is a file within the Windows filesystem which acts as a pseudo-partition. My guess is that it is only about 16GB in size - that's a common default, perhaps only 8 - and that is why you are getting the "not enough space" error message. More on wubi in these two links:

[https://wiki.ubuntu.com/WubiGuide](https://wiki.ubuntu.com/WubiGuide)

[https://help.ubuntu.com/community/Wubi](https://help.ubuntu.com/community/Wubi)

Since Windows is damaged, I would suggest your first priority should be to backup your personal files to external media. If the Windows filesystem gets more damaged than it is at the moment, not only will you lose Windows, but you will lose Ubuntu.

Having done that, and having read those links, I suggest you consider whether you want to try to repair Windows and/or make a fresh installation of Ubuntu, but this time a conventional one to its own native partition. It will work better that way. Post back if you need any help.

### Post by coffeecat on 2011-10-16
First, you need to download the ISO of the Ubuntu desktop CD from here:

[http://www.ubuntu.com/download/ubuntu/download](http://www.ubuntu.com/download/ubuntu/download)

If you have a 64-bit processor and 3GB of RAM or more I would suggest you download the 64-bit version. It's a source of continuing puzzlement to many on the forum why the Ubuntu site still recommends the 32-bit version as the default, but there you go!

You then need to burn the ISO to CD. Guide here:

[https://help.ubuntu.com/community/BurningIsoHowto](https://help.ubuntu.com/community/BurningIsoHowto)

Then you need to set the BIOS on your machine to boot from CD, boot up the Ubuntu CD and install from that. Useful guide here:

[https://help.ubuntu.com/community/GraphicalInstall](https://help.ubuntu.com/community/GraphicalInstall)

And more information here:

[https://help.ubuntu.com/community/Installation](https://help.ubuntu.com/community/Installation)

If you are absolutely sure you want to wipe out Windows (if that was my machine I wouldn't), then you need to choose the "erase disk" option in the installer. However, this will delete not only your Windows partition, but also the recovery partition, which gives you no way of restoring Windows should you wish to do so. If you want to do something more selective with partitioning, you need to choose the "something else" option in the installer, and set up your partitions manually. If you need to know about partitioning as it relates to Ubuntu, there's a good guide here:

[https://help.ubuntu.com/community/HowtoPartition](https://help.ubuntu.com/community/HowtoPartition)

Unfortunately I'm getting a server error with that particular page at the moment. I hope it's a temporary glitch because that page was fine just a day or so ago.

Do please post back if you have any more questions, but if you decide to go with a manually partitioned install and want some help with that, you might want to start a new thread with a different title.

Good luck with your Ubuntu-ing!
