### Post by kanikilu on 2009-10-06
I haven't tried this myself, but from this page:

[http://www.freebsd.org/doc/en/books/handbook/network-ntp.html#AEN39246](http://www.freebsd.org/doc/en/books/handbook/network-ntp.html#AEN39246)

It looks like ```
restrict default ignore
``` in the ntp.conf should work. It does say, however, that by itself, it would prevent connecting to other servers, and to add them manually. This page has some more information on how to do that:

[http://support.ntp.org/bin/view/Support/AccessRestrictions#Section_6.5.1.1.2](http://support.ntp.org/bin/view/Support/AccessRestrictions#Section_6.5.1.1.2)

and

[http://support.ntp.org/bin/view/Support/AccessRestrictions#Section_6.5.1.2.1.](http://support.ntp.org/bin/view/Support/AccessRestrictions#Section_6.5.1.2.1.)

Like I said, I haven't done this yet, but I'm going to try it soon. My infosec dept at work is rather stringent about running "unnecessary services". How keeping time synced across the network is "unnecessary" is beyond me, and besides, I'm running a newer version of NTPD than their "official" servers, but that's neither here nor there :P
