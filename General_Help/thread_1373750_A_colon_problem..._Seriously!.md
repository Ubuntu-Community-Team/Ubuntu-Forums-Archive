### Post by Shibblet on 2010-01-06
I just found out that if you put a colon : in a file name, and copy it to an NTFS formatted drive, that file will be corrupted.

I tried to copy a file named "24 - Resurrection Ship: Part 1.mkv" to my laptop, and it just wouldn't copy.  I tried EVERYTHING next to re-encoding the file.  

It opened in SMPlayer just fine, I just couldn't copy the darn thing.  Finally I realized a few other files hadn't copied either, and they all had colon's in the file name.  Rename the file, and it copies perfectly.

I normally don't slam on Windows or Microsoft in general, but this one got on my last nerve.  And I posted this in General Help, because it might actually help someone to know this.

Way to think ahead Microsoft.

### Post by John Bean on 2010-01-06
Most OS have their special characters that shouldn't be used in filenames and Windows is no exception, although the use of a colon as a device name terminator predates Windows by some considerable margin and has been used in many OS.

I can't imagine anyone who has ever used a Microsoft OS - or one of the DR or DEC OS that contributed to this convention - will be unaware of the reserved use of the colon.
