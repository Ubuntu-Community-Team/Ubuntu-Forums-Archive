### Post by mb_webguy on 2009-01-23
I think the main problem is that there is quite a bit of inconsistency across applications when it comes to Linux.  Some applications have an "Ok" button (and occasionally a "Close" or "Cancel" button as well), and settings are applied when the "Ok" button is clicked.  Others only have a "Close" button, and settings are applied as soon as they are changed.  I have seen very few, if any, Linux applications that use the "Apply", "Ok", "Cancel" scheme that is common in the Windows world.

I don't think that an "Apply" button is absolutely necessary, but I do prefer that settings not be applied immediately.  This prevents crashing your computer by inadvertently applying settings that your computer can't handle (as mentioned by the OP).  I also think it's more intuitive that changes be committed rather than automatically applied.  And while I don't think that an "Apply" button is strictly necessary, it can be convenient to apply settings which you're unsure about without closing the settings dialog.

However, if all applications conformed to a particular scheme, and users knew what to expect from a properties or settings dialog, then it would certainly make things easier, regardless of the specific scheme used.  If you're expecting settings to be applied immediately, you'll probably be more careful about adjusting those settings.  If you're used to something like the Windows scheme, it can be very frustrating using a scheme in which settings are instantly applied.

Unfortunately, given the way in which Linux applications are developed, getting everyone to agree on such a scheme is not an insignificant feat, and getting everyone to conform to it even more unlikely.
