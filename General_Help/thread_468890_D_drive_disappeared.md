### Post by tpg on 2007-06-10
The permissions seem fine. If your other ntfs partition works fine, then it could possibly be that the filesystem in /dev/sda5 got corrupted somehow. Boot into Windows, and try the CHKNTFS command (see [http://www.ss64.com/nt/chkntfs.html]("http://www.ss64.com/nt/chkntfs.html")).

If all else fails, I'd suggest backing up your data, and reformatting that partition (which will obviously destroy anything that was on the partition). You could try FAT32, as there are few (if any) problems using it with linux, and windows can read/write to it as well.
