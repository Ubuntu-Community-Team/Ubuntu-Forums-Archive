### Post by Morbius1 on 2010-05-20
> **apurbapaul0 said:**
> [FONT=Lucida Console]
/dev/sda3    /media/OS    ntfs    rw, suid, dev, exec, auto, user, async    0    0
/dev/sda5    /media/disk    ntfs    rw, suid, dev, exec, auto, user, async    0   0
[/FONT]
I don't know if those lines are typos or if you copied it manually but they will never work because of the spaces after the commas. If you did a "sudo mount -a" after you added those changes it would have told you that.

BTW, I would change those entries to something more traditional like:
```
/dev/sda3    /media/OS ntfs    default        0    0
/dev/sda5    /media/disk ntfs    default        0    0
```Or if you want to do it the way Ubuntu would add them during an install ( using the "specify partitions manually" option ) they would look like this:
```
/dev/sda3 /media/OS  ntfs    defaults,nls=utf8,umask=007,gid=46 0       0
/dev/sda5 /media/disk  ntfs    defaults,nls=utf8,umask=007,gid=46 0       0
```The **umask=007** will give owner (root) and group read/write access and no access to anyone else.
The **gid=46** represents group=plugdev. All local login users are members of plugdev.
So what you get is a mount point that allows read/write access to all local login users.
