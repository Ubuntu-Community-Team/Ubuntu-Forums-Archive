I'm going to see what happens with the other suggestions.  In the meantime, any other thoughts anybody?

the "upgrade" term is just used in the same context as with apt-get, which uses "update" when meaning that it's refreshing the information about available packages, while "upgrade" and "dist-upgrade" refer to actually downloading and installing the new package versions.

...and "partial upgrade" simply means that the package manager has information about new versions of some packages, but for some reason or other some of them aren't actually available or can't be installed at the moment. Usually that happens when the update manager checks for updates while new packages are still being added to the repository server, in which case simply waiting a while and refreshing the package information solves the problem.

At least in your screenshot there's nothing that would point to a release upgrade.

edit: Of course if you want to be sure, you can always open a terminal and run "sudo apt-get update && sudo apt-get upgrade" and post the output here.

### Post by mcduck on 2012-09-16
Based on that output, it really is just soem normal updates. I actually had the same issue with only aprtial update being possible, and it was because the Java updates required adding a couple of new packages and removing one existing one, which the Upadate Manager isn't allowed to do.

So yes, you should install the updates, but instead of doing the partial upgrade with the Update Managager you should either use Synaptic Package Manager, or run "sudo apt-get dist-upgrade" in a terminal.

### Post by Paddy Landau on 2012-09-16
The wording is misleading. It is not a partial upgrade, but what is known as a "distribution upgrade" (although that wording, too, is misleading).

Go ahead with the "partial upgrade". It is absolutely fine.
