### Post by FuturePilot on 2007-08-29
You shouldn't be running k3b with sudo. Now it won't work without sudo because it changed all the permissions on the files in your home directory. I think the things you were seeing in your first post are normal. Lots of apps output stuff in the terminal. I don't think it's anything to worry about if it's working.

Try renaming the .kde folder to .kde_old (it's  a hidden file) and try running k3b *without* sudo

### Post by forestpixie on 2007-08-29
And if you do want to get rid of the

```
 X Error: BadDevice, invalid or uninitialized input device 169
```

you can comment out these bits using # in front of each line as you'll be using them in future ( I just deleted)

```
Section "InputDevice"
Identifier "Configured Mouse"
Driver "mouse"
Option "CorePointer"
Option "Device" "/dev/input/mice"
Option "Protocol" "ImPS/2"
Option "ZAxisMapping" "4 5"
Option "Emulate3Buttons" "true"
EndSection

Section "InputDevice"
Driver "wacom"
Identifier "stylus"
Option "Device" "/dev/input/wacom"
Option "Type" "stylus"
Option "ForceDevice" "ISDV4" # Tablet PC ONLY
EndSection

Section "InputDevice"
Driver "wacom"
Identifier "eraser"
Option "Device" "/dev/input/wacom"
Option "Type" "eraser"
Option "ForceDevice" "ISDV4" # Tablet PC ONLY
EndSection
```

also you need to comment out the appropriate lines in this section

```
Section "ServerLayout"
Identifier "Default Layout"
Screen "Default Screen"
InputDevice "Generic Keyboard"
InputDevice "Configured Mouse"
[COLOR="Red"]#[/COLOR]InputDevice "stylus" "SendCoreEvents"
[COLOR="Red"]#[/COLOR]InputDevice "cursor" "SendCoreEvents"
[COLOR="Red"]#[/COLOR]InputDevice "eraser" "SendCoreEvents"
EndSection
```

make sure you backup first

```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.bak.2908
```

then to edit the file 

```
gksudo gedit /etc/X11/xorg.conf
```
