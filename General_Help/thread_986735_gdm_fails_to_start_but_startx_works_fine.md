Hi,

I have a Clevo M66SE with a VIA VN896 graphics card. After a bit of a wrestle it worked fine running Ubuntu 8.10 with the openchrome drivers until my disk died last week.

After install, booting in normal mode, I just get a blank screen. To start with, it looked like it was still booting with "xforcevesa" in the grub config, so I removed it, and it still boots with a blank screen.

Attached are the logs and process lists.

I've booted in recovery mode and manually written an xorg.conf based on the default (which was also missing the touchpad entry) and I can run startx and it works fine, although I don't get a particularly nice resolution, and when I quit (with menu quit or ctrl-alt-backspace) it doesn't return the prompt and gives a blank screen again which can't be fixed without a hard reboot.

It looks like for some reason isn't using the correct xorg.conf but the messages shows the kernel arguments correctly had xforcevesa removed. From the Xorg0.log files it looks like X starts OK but then the "unload" lines suggest someone's killed it for some reason (gdm? Why? How do I know?)
