### Post by texaswriter on 2011-09-23
> **beew said:**
> Is that a fact or just your opinion? If you run everything on WINE why not just use WIndows?.

You are basically stating that if you run applications that were made to run on Windows that you should just use Windows. This is a  [***non*** ***sequitur***]("http://www.google.com/search?hl=en&client=ubuntu&hs=KQH&channel=fs&sa=X&ei=BSl9TpuRFqqosQK1uJAa&ved=0CCAQvwUoAQ&q=non+sequitur&spell=1"), which is to say, that statement does not follow. If you can run an application on whatever operating system you want, this is a good thing for that person. 

Wine/Crossover/Cedega/etc allow one the choice of running applications on the operating system of their choice. 

The statement "why not just use Windows" would only benefit Microsoft, as this is what they want you to do. They also do not want you to have the choice to run software on the operating system of your choice. 

So, I'm not sure what your original intent of this was, but I hope this shows that allowing people the choice of operating systems, whether it is Windows, Mac, Linux, BSD, etc is a grand idea. 

When I am trying to understand people, especially ones whom I do not agree with, I try to avoid asking loaded questions, ad hominem statements, non sequitur statements, and red herring statements. 

A loaded question is one that assumes something that the other person will likely not take for granted, thereby giving them a damned if they do damned if they don't choice as a response. The correct answer is to say that your conclusion is invalid because it is based on a faulty reasoning(s) or wrong assumption(s). 

An ad hominem is where you attack or denigrate the other person instead of responding to their argument. Attack the argument not the person. This is also called metadiscussion, where instead of responding to the ideas, you attack the discussion yourself. (I.e. talking about the talk and not about the ideas presented in the talk)

A non sequitur statement is saying something that logically does not follow. For example, there is not implicit or explicit in the universe that suggests that programs must run on Windows or Linux or Mac, therefore to state that an application that was made to run on Windows should ONLY run on Windows (which is implied from your statement) is logically fallacious. Ultimately, ALL programs (even HTML) inevitably run on bytecode and binary (at some level); this binary is essentially the same regardless of platform and is used regardless of platform. 

A red herring statement is where you respond to something other than the original post (OP) in a way that does not abide by the spirit or intent  of the original poster. For example, if somebody posts a logically invalid question that leads, begs the question, or makes one of the above logical fallacies, then it is *somewhat more* appropriate to respond away from the direct aim of the question, as long as it forwards some valid point or forwards the actual intent of the OP (however mis-guided). 

As such, the original post asked about professional software that runs natively on Linux. 

Most directly this would include software targeted to be ran on Linux. 

But, indirectly, because Wine = WINE IS NOT AN EMULATOR, programs that you run in WINE/Codeweavers (and some others as well) actually translate everything to native libraries or run these other libraries natively (as opposed to a virtual machine, which is an actual emulator). This is my experience and opinion speaking, but Windows programs that install and run on Linux through wine or crossover typically run better (faster, more stable, etc) than what I experienced when those programs ran on Windows. This is IMO, and YMMV (in my opinion / your mileage may vary). 

I hope this clarifies the situation immensely. I like to see everybody helping people, especially those looking for help, and I do believe everybody's intent was to help each other here... Just thought perhaps a tiny lesson in typical logical fallacies was in order. 

HTH, 

Good Luck!!
