### Post by adam.tropics on 2006-07-17
triage is a word more commonly used in the medical profession to describe the process of assessing the severity of injuries so they can be treated in an appropriate order, and given a sensible degree of attention. So no, it doesn't solve a bug, but it does correctly place a bug to recieve the most appropriate level of attention. As an example, if the kernel is completely broken, and the desktop wallpaper is displaying incorrectly, triage (and common sense!) is what stops all of the developers from trying to fix the wallpaper, leaving nobody fixing the kernek, which is far more important.

### Post by az on 2006-07-17
> **mvaniersel said:**
> Does that simply mean to solve a bug?


Actually it's just the opposite!

If you are not skilled enough to be able to help out by fixing bugs, you can learn a few basic things and then be able to help out by sorting out the various bugs.

There are thousands of bugs, far too many for the developers to be able to see and address each one.  To triage means to do simple organization to eliminate duplicates, ask for clarifications, check to see if the problem has already been fixed, etc...

Basically, it's to help out with the clerical work so that the time can be spent more efficiently by the people who can fix the bugs.

For example, some bugs are simply technical support questions.  Some bugs are filed for the wrong package.  Some bugs are too vague.

You don't have to be very skilled to spot these things and ask the bug reporter to add information, to reassign the bug to the proper package (so that it gets seem by the right person) or to answer a question.
