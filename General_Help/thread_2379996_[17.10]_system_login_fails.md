### Post by gemmathetechie on 2018-01-01
This could be a problem with the GUI, I have had this problem with Zorin OS (Using GNOME).

Have you tried logging into the system using the console?

To do this, you need to put in the key command Ctrl + Alt + F1

It should show a login script, input the username that you put in that was completely lower case.

Then input the user password.

If you login correctly, attempt to create a new user by following the steps below.

```
user@PLATINUM:~$ **[COLOR=#008000]sudo adduser _foo_[/COLOR]**
[sudo] password for user: **(your password goes here)**
Adding user `foo' ...
Adding new group `foo' (1002) ...
Adding new user `foo' (1002) with group `foo' ...
Creating home directory `/home/foo' ...
Copying files from `/etc/skel' ...
Enter new UNIX password: **(The new user's password goes here)**
Retype new UNIX password: **(The new user's password goes here)**
passwd: password updated successfully
Changing the user information for foo
Enter the new value, or press ENTER for the default
    Full Name []: **[COLOR=#008000]User[/COLOR]**
    Room Number []: **[COLOR=#008000]#[/COLOR]**
    Work Phone []: **[COLOR=#008000]#[/COLOR]**
    Home Phone []: **[COLOR=#008000]#[/COLOR]**
    Other []: **[COLOR=#008000]#[/COLOR]**
Is the information correct? [Y/n] **[COLOR=#008000]Y[/COLOR]**
user@PLATINUM:~$
```

The bold green text is what you type into the command line.
The bold green underlined text is what you use to name the user.

Now type [FONT=courier new]exit[/FONT] into the console and use the same key combination to go back to the GUI replacing F1 with F7.

Now you should see the new user in the login screen, attempt to log in as the new user you have created.
