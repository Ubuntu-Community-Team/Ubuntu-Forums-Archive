### Post by organica on 2006-11-13
ok, from what I can tell:

backup your current xorg.conf

> sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.backup

Add this to the Section "ServerLayout"
```
 Option "AIGLX" "true" 
```
(this may or may not do anything to help the situation)


add this to you Section "Module"
```
        Load    "ddc"
        Load    "dbe"

```

and add this to the end of the file:

```
Section "DRI"
        Mode    0666
EndSection

```

give X a restart and then run beryl-manager

also check with System>Preferences>Sessions then Startup Programs.  You should have beryl-manager and emerald running at startup.

### Post by organica on 2006-11-13
if you can get a metacity desktop back up make you startup session writable:

sudo chmod a+w /home/[USERNAME]/.config/autostart/

change your [USERNAME] to your username of course.

Then add beryl-manager and emerald to the session> startup commands.

For some reason Edgy Eft did not make the startup session writable to a user AND emerald starts TWICE if you don't have beryl-manager and emerald in the startup sessions.  If emerald is running two instances, then you desktop will not work properly.  

If this doesn't do it, then my magic bag is empty.
