### Post by TheFu on 2022-09-09
I have a centralized enterprise calendar (Zimbra), so the clients which connect into it is a mandatory feature.  Thunderbird has moved the external "Lightning" addon to be included with Thunderbird. That's what I use.  However, I still find that connecting to the Zimbra webapp is needed to really get the events all handled as far as Zimbra is concerned, but with Thunderbird's Calendar interface, "Dismissing" a reminder and seeing all, future, or some subset of events works.

I have a Home and Work calendars, but also have live calendars provided by groups pulled into thunderbird over the web.  For example, when my college football team is playing, who, when, where, and how I can watch the game are in that calendar. For example, I know my team will be on broadcast TV tomorrow at noon. Usually, I have to stream an AM radio station to hear the game.  It has been a while since I setup that calendar, but I think it is an .ics file.  Have some for meetup groups and holidays around the world, which helps me keep in contact with friends in a few different countries.

Zimbra is a pain.  It is like MS-Exchange, on steroids.  It does email, calendaring, address books, pulls in multiple other email accounts, bookmarks, shared calendards, delegation, private calendars, and has an addon capability for SSO and many other uses.  It also has smart linking, but that only works through the webapp.   It uses standards, so any standards-based clients can be used to access any of these things - be it email, calendar, addresses, documents, SSO, and there must be 50 other things it can do.  I looked at replacing it with Nextcloud's calendar app, but NC just wasn't integrated where I felt it needed to be.  For a few years, I used Zimbra's LDAP server as the SSO for many little companies - other webapps and Linux logins were all tied to Zimbra's LDAP. Had to add the POSIX schema to LDAP and that became a pain during Zimbra upgrades.  Breaking logins for 10 other applications for a few hours whenever there was a Zimbra upgrade was a bad idea.
