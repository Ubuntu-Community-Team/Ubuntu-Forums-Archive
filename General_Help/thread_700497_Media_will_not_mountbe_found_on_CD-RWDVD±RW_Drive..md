### Post by legzelda on 2008-02-18
You might want to check out your fstab file configuration.  I don't see anything listed for an optical drive.  I'll show you mine for comparison:

```
# /etc/fstab: static file system information.
#
#  -- This file has been automaticly generated by ntfs-config -- 
#
# <file system> <mount point>   <type>  <options>       <dump>  <pass>

proc /proc proc defaults 0 0
# Entry for /dev/hda1 :
UUID=5a92659e-2288-4cb2-a486-b21b4a4ee89d / ext3 defaults,errors=remount-ro 0 1
# Entry for /dev/hda5 :
UUID=e033a1e8-07be-4f79-83fc-3273cd75cc62 none swap sw 0 0
/dev/hdc /media/cdrom0 udf,iso9660 user,noauto 0 0
/dev/hdd /media/cdrom1 udf,iso9660 user,noauto 0 0
#/dev/fd0 /media/floppy0 auto rw,user,noauto 0 0
/dev/hdb1 /media/archive ext3 defaults,errors=remount-ro 0 0
```

As you can see, I have two entries for my two optical drives.  I'm not sure of the exact settings for your particular system, but this might be a good starting point.  I'd suggest altering your /etc/fstab file and remounting all your drives and see if you have more luck.
