### Post by s.mulleady1122 on 2010-05-13
Ok so I found the solution. Looks like on 10.04 you need to copy some script files before installing a kernel, no idea why. It is documented here:

[https://help.ubuntu.com/community/Kernel/Compile](https://help.ubuntu.com/community/Kernel/Compile)

```
sudo cp /usr/share/doc/kernel-package/examples/etc/kernel/postinst.d/initramfs /etc/kernel/postinst.d/initramfs
sudo cp /usr/share/doc/kernel-package/examples/etc/kernel/postrm.d/initramfs /etc/kernel/postrm.d/initramfs
```

### Post by mrrusof on 2010-05-24
My experience fiddling with kernel 2.6.34 ( [http://kernel.ubuntu.com/~kernel-ppa/mainline/v2.6.34-rc1/](http://kernel.ubuntu.com/~kernel-ppa/mainline/v2.6.34-rc1/) ): 

Summary: copying the files made the package install fail; after undoing the copy of initramfs, install succeeded.

1. I had the .deb packages for image and headers installed.
2. I copied the files you mention.
3. Tried reinstalling without removing or purging. Install failed.
5. Uninstalled packages.
6. Tried reinstall. Install failed.
7. Removed the files I copied in 2.
8. Install worked and I was able to boot. I noticed that the corresponding entry in the updated /boot/grub/grub.cfg used the drive uuid instead of /dev/sda1 in the [linux] line.
