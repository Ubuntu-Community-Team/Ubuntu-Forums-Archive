### Post by 2F4U on 2013-05-15
Well, it depends. The 2.6 kernel series is still in use in some distributions, mainly enterprise Linux (very stable) such as RHEL, CentOS, Scientific Linux, so it is actually not out-dated. On the other side, kernel development happens pretty fast and I doubt that there are any books already covery the latest kernels (which are, by the way, hosted here [https://www.kernel.org/](https://www.kernel.org/)). A nice graphical overview about what is happening in kernel land can be found here: [http://en.wikipedia.org/wiki/Linux_kernel#Development](http://en.wikipedia.org/wiki/Linux_kernel#Development).
I don't think that the basic concepts are changing that fast, so I think it is still a good idea to start with one of these books, even if they are based on an older kernel line.
