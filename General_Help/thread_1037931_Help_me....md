### Post by HunterK on 2009-01-12
Those programs were written and designed to work only in a Microsoft Windows environment.  Linux is a totally different type of operating system. (There is no such thing as ".exe" in linux!)  To get around this, you have a few options.  I wont get into too much detail on HOW to go about doing these things, but here they are: (relevant google searches provided)

1) Attempt to use WINE to run your windows applications in linux.  WINE kind-of "tricks" your windows programs into thinking they are really running in windows.  They dont know that they are actually running in linux!  Not all applications run as they were designed, but WINE has come a long way and may do the trick for you.

[http://www.google.com/search?hl=en&q=WINE+linux&btnG=Search&aq=f&oq=](http://www.google.com/search?hl=en&q=WINE+linux&btnG=Search&aq=f&oq=)


2) Install Windows into a virtual machine within Linux.  Its almost like virtually booting a PC inside linux.  You boot up your "virtual" PC in Linux and install Windows as if it were a plain old computer.

[http://www.google.com/search?hl=en&q=virtualbox+linux&btnG=Search&aq=f&oq=](http://www.google.com/search?hl=en&q=virtualbox+linux&btnG=Search&aq=f&oq=)

3) Dual boot windows and linux.  You can setup things to a point where you turn your computer on, you have the choice to start windows or linux.  You basically will have 2 operating systems installed directly on your computer.

[http://www.google.com/search?hl=en&q=dual+boot+ubuntu+windows&btnG=Search&aq=f&oq=](http://www.google.com/search?hl=en&q=dual+boot+ubuntu+windows&btnG=Search&aq=f&oq=)

### Post by jespdj on 2009-01-12
Because Ubuntu (and Linux in general) [is not Windows](http://linux.oneandoneis2.org/LNW.htm). It is not meant to be a free clone of Windows, it's a totally different operating system. Programs written for one operating system (Windows) do not run on other operating systems (Ubuntu, Mac OS X or anything else).
