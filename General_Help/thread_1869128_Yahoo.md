### Post by eMJayy on 2011-10-26
I just encountered this exact problem a few minutes ago. I use Firefox, Opera and Chrome on Ubuntu 11.10 and Ubuntu 10.04. Firefox behaved normally, but Opera and Chrome were suddenly being forced by Yahoo to go to the mobile version for some reason. 

Anyway, I've found a solution. I was able to mask Opera to make it appear to be Firefox (that's a built-in feature of Opera) and that seems to have forced the mobile Yahoo site to offer me the option of switching to the regular desktop version (the Desktop version link suddenly appeared right at the bottom of the page after masking). Anyway, after selecting the desktop version and loading the page, I saw that the URL in the Opera address bar changed to this:


[http://www.yahoo.com/?m=1](http://www.yahoo.com/?m=1)

I copied the link above to Chrome and it's now going straight to the desktop site as well. I think the link above resets the Yahoo cookies to force the desktop version to load instead of the mobile one. Hope this helps.
