### Post by Cheesemill on 2012-11-19
Not all .04 releases are LTS.

Ubuntu release numbering is based on the release date of the OS in the format YEAR.MONTH, for example 12.10 was released in 2012 in October (the 10th month).

Currently there is an LTS release every 2 years, for example 8.04, 10.04 and 12.04 are the LTS releases that are still supported. From 12.04 the LTS releases are supported for 5 years on both the Desktop and Server editions, previously to this the LTS releases were supported for 3 years for the Desktop version and 5 years for the Server version.

When a version of Ubuntu is released it doesn't ever get any major application updates, only bug fixes and security updates are released. For example 12.04 was released with GIMP version 2.6 and will never get GIMP version 2.8 through the normal update channels, only security patches and bug fixes to the 2.6 branch will be released.

In the end you have to weigh up the pros and cons of newer software vs. length of support. In a business environment you would obviously want to stick with LTS releases as it means you only have to upgrade the OS every 4-5 years instead of every 18 months, but you are stuck with the versions of software that were available when the release happened, this could mean running software that is a few years old. Home users usually tend to want the latest versions of software so the non-LTS releases may be more appropriate.

> ...with supposed very minor incremental improvement(s)...
This really isn't true. If there has been a major update in an application that you use since the last LTS release then the improvements could be anything *but* minor.
