### Post by ajy0852 on 2009-12-05
right-click on one of the files, and choose "properties".
Go to the "Permissions" tab and un-check the box that says "Allow executing file as program".

It should now open normally in whichever program you want, without that prompt.

You can select multiple files to change permissions on all of them at once.

### Post by ad_267 on 2009-12-06
What file system are they on? If they're on FAT or possibly NTFS too then you won't be able to make them non-executable because those Windows file systems don't support marking files as executable. The file system will be mounted with all files as executable by default. I think you can set the mount options to make the drive non-executable in /etc/fstab if you want.

Edit: Yup, you can use "gksu gedit /etc/fstab" to edit your fstab file, then look for the line corresponding to your windows drive and change "exec" to "noexec".

### Post by ad_267 on 2009-12-06
No it will just change the options used when mounting the drive in Ubuntu. The worst that could happen is if you mess up that file Ubuntu won't be able to load, so it would probably be best to make a backup first then you can fix stuff from a live CD. From a terminal you can run "sudo cp /etc/fstab /etc/fstab.backup" to make a backup copy.

It might also be tricky to work out which disk is your Windows drive. The fstab file uses UUIDs to identify partitions. You can open a terminal and run "sudo blkid" to get the UUIDs for all your partitions.

There are also graphical tools for editing fstab, pysdm is one I've heard of that's available in the Ubuntu repositories. But editing the file manually is probably a lot more straightforward.
