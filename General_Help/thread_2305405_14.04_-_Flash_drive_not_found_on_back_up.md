### Post by sandyd on 2015-12-05
If you want to have a try...

Deja-Dup is believed to use UUIDs/Labels to recognize a device. Deja-dup won't backup to this USB drive because the partition has a different UUID/Label

Possibly, all you would need to do is to dd one drive to another, and resize the partition.
The partition can be resized with GParted after the dd is done.

Something like
```

sudo dd if=/dev/*originaldrive* of=/dev/*newdrive*
```

Install gparted, run it from the menu, and resize the fat32 partition on your new usb drive.

Note that *originaldrive* and *newdrive* have to be replaced with the propper values.

It should be something like "sdb" or "sdc" or similar (without the numeral at the end)

You can get a list of all disks on your system with either of the following commands
```

sudo fdisk -l
sudo parted -l
```

Be careful about what drives you are running dd from, if you type the wrong values, you could be copying your usb drive directly onto your hard disk and wipe out your system.

### Post by sudodus on 2015-12-06
a. Assume that you have one internal drive, which is recognized as /dev/sda.

b. Then connect your original backup drive. It will be recognized as /dev/sdb.

c. Finally connect your new and bigger backup drive. It will be recognized as /dev/sdc.

But double-check that you are right about it - in order to avoid overwriting valuable data by mistake. Now you can use the method described by *sandyd*, or if you want to wrap security around the cloning process, you can use [mkusb](https://help.ubuntu.com/community/mkusb).

In order to read from a block device, you need to run mkusb from the command line.

```
sudo -H mkusb /dev/sdb  # assuming the original backup drive is /dev/sdb
```

Anyway, mkusb will provide a method the find, identify and select the target drive - and help you avoid mistakes.

After cloning you can expand the partition and file system on the target drive to use the whole drive space. Expand alias 'grow' the first partition according to [GrowIt.pdf]("http://phillw.net/isos/linux-tools/9w/GrowIt.pdf"). It is probably enough to look at the pictures (you need not read the text, some of it is not relevant for your particular case).

Good luck :-)
