### Post by Bladeforger on 2009-02-13
Issue: I generate, and need to print, maybe 100-300 pages a day of pdf files.

I began with Adobe Reader, which everyone says is bloated.  It took up to a minute per page.

On another thread about favorite pdf viewers, I read that Evince was pretty slow, too, but that epdfviewer was more lightweight and fast.  I installed it and it works, most times, printing about 15 seconds per page.

Then I read in Scott Grammeman's neat little book, the Linux Phrasebook, about using the lpr command from the terminal to print pdf files (and text files and postscript files).  

Given a mere 9 page document (a tax return with all the form info), the lpr command only printed 3 pages.  Then it was done.  No warning that it was quitting midstream, no pending print documents, just the first 3 pages like it was all done.:lolflag: Because it was as done as it was going to be.

And, occasionally, epdfviewer does the same thing--when I have a lot of programs working at the same time.

So I'm back to epdfviewer and making sure that I count pages and reprint as necessary and save print jobs until I can close some other programs--pain in the rear but do-able.

Any thoughts?

### Post by Bladeforger on 2009-03-06
I have an update.  It appears that the problem can be solved with an upgrade to printer memory.  My printer is a Brother HL-5240.  It comes with a mere 16mb on board.   For cheap, I have ordered a 256mb DIMM... cost about 35.00 including shipping.  When  it arrives, and it's installed, I'll report back.  For anyone else that is having this same problem, the solution may be a cheap, easy upgrade.
