### Post by darkmire on 2010-02-11
> **savvavas said:**
> Due to an unexpected error, OpenOffice.org has crashed. All the files you were working on will now be saved.

I get this error every time when I get to a specific point in my powerpoint presentation.
What should I do to fix this problem?

First of all, there is a known bug with Impress which hasn't yet been sorted as far as I know.   What happens is that when you are editing large presentations with many embedded files, Impress will crash on cut and paste.  I'm not sure how much of a bug it is as almost anything will crash if your memory runs out.

The best solution I've found is to keep images down to the smallest size possible without losing clarity for the size I want to project it to.  This keeps things more compact.  Similarly, if I'm using media files, I use the most compact formats if I can as this seems to persuade Impress to run better.

If you are crashing at a specific point in just one presentation, then the problem is with the presentation - again it is usually an embedded file that's corrupted.   Try removing and embedding it again, or even changing it to a different format and then embedding it.
