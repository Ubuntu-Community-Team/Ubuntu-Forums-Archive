### Post by Paddy Landau on 2012-01-03
You aren't installing according to [the installation instructions]("http://support.crashplan.com/doku.php/getting_started/installing_crashplan#linux"). It says the command is:
```
sudo ./install.sh
```

### Post by Error403 on 2012-01-03
Thanks, I tried that yesterday but this morning with a fresh coffee I set the install.sh permissions to executable and tried again. Now it works flawlessly!  Sometimes the easiest fixes are right in front of us ;)
