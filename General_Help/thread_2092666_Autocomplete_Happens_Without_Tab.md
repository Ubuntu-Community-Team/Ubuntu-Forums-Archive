### Post by BookofJarom on 2012-12-08
I'm running Mythbuntu 11.04 with xfce 4.8.0 desktop environment.
 
The problem: When I type a path to open a file in some applications (xfwrite and leafpad come to mind), the path autocompletes automatically, without me pressing <Tab>.
 
So, for example, if I'm trying to go to /var/lib/mythtv/recordings/, here's what happens as I type:
"/va" ... suddenly the location autocompletes to show "/var/"
"li" ... suddenly it autocompletes to "/var/lib/"
"myt" ...suddenly it autocompletes to "/var/lib/myth"
"t" ...suddenly it autocompletes to "/var/lib/mythtv"
 
Please note that I'm not complaining about the box simply showing the rest of the autocompletion in shaded text that I can still type out or use <tab> to autocomplete--that's the behavior I'd like. Rather, I'm complaining that it actually autocompletes automatically and puts the cursor at the end of the autocompletion.
 
So, if I type "/var", I get "/var/" as soon as I type just "/va". Then, the system thinks the remaining "r" is to be considered after the autocompleted "/var/", giving me "/var/run".
 
How can I fix it so it lets me type what I want and waits for me to press <tab> to actually autocomplete? It's so annoying I'd disable autocomplete altogether if necessary, but there must be a better way.
