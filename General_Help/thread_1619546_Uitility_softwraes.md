### Post by endotherm on 2010-11-11
always try to use the repositories for software first. software-center, apt-get, and synaptic all use the repositories. software center is closest to "Add/Remove"

if the software is in the repositories but not of the newest version,  and you have a compelling reason to need the upgraded version, then PPAs are a great way to get individual pieces of software based on the developers timetable, not cannonicals. or you can install the backport repositories, to recieve software that has not yet been fully tested with ubuntu. both the PPAs and the backports can be risky, but the PPAs more so, since no one has approved much of anything pertaining to that code, so only download if many others appear to trust that PPA. 

if that fails, you often have to go to their website, and download a deb, or if all else fails, source. if it's a deb, just double click it, and if it is source, they shoudl provide instructions for compiling it. finding and installing the dependencies however can be a pain.

doing it in that order, will usually minimize your hassle.
