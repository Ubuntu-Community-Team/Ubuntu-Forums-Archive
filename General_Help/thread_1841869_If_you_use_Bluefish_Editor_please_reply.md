### Post by grahammechanical on 2012-04-23
Now, imagine me visiting your web site but I do not like what it looks like because I do not see the page as you see the page. I do not have the same fonts installed on my PC as you have on your PC. So, my web browser will substitute a font on my machine for the one that your have chosen in the web page.

It is best to chose fonts that are available to most people and then offer alternative fonts if the first choice is not available.

This is what CSS (Cascading Style Sheets) are for. CSS is a page that the web browser reads to find out what fonts and what styles you want the text and stuff to be rendered in. It is where you set the alternatives.

For example:

>   {
  font-family: FreeSans, frutiger linotype, arial; font-size: 100%;
  }

In a cascading style sheet will tell the browser to first use the FreeSans font and if that is not available then to use frutiger linotype and if that is not available to use arial.

That is how it is done.

Books are written on this subject.

If you can code, you can code...and your possibilities are basically endless.

If you use WYSIWYG (What You See Is What You Get...a graphical editor) or some other type of "drag-and-drop" editor, like the one you get at Google Sites, then you'll only learn how to make websites using that specific application or service. This is perfectly fine, if you just want to create a simple blog for your organisation or something like that...but if are planning on becoming a web developer, then you need to learn how to code. 

If you use Bluefish (or my personal favorite: [Aptana Studio]("http://www.aptana.com/")), then you are still writing your code yourself. 

Many people also like to use some type of framework or CMS (Content Management System) to get complex things done faster and easier...but that's all a matter of personal preference. Anyway, you should know the basics first.

IMHO, studying a little HTML, CSS, PHP and Javascript, should give you a pretty solid ground to stand on.
