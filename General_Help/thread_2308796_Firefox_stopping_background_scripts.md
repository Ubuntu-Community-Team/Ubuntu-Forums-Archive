### Post by drhex on 2016-01-05
When the computer feels slow and unresponsive and fans are working hard, "top" usually shows that Firefox is the culprit, even though it does not seem to be doing anything in particular.
I typically have 10-20 tabs open, and some of them are probably running scripts that use a lot of cpu time.
My usual remedy is to close tabs at random until firefox calms down, but that feels clumsy and I'd like to keep some of those tabs.

Is there a way (configuration setting, extention) to have firefox only allow running scripts for the tabpage that is **currently displayed**?  Then I could locate the problem much more easily.

### Post by ajgreeny on 2016-01-05
The FF add-on noscript will stop all scripts on all tabs unless you allow them, so might have a configuration setting that allows fine-tuning that will allow you to do what you want, but I'm afraid I gave up on noscript as I found it to be too much of a nuisance in ordinary day-to-day use.

Others swear by it, so it may be worth investigating.
