### Post by wjstarck on 2011-11-17
OK, fixed.

The solution was response #40 here:

[https://bugs.launchpad.net/ubuntu/+source/lightdm/+bug/856810](https://bugs.launchpad.net/ubuntu/+source/lightdm/+bug/856810)

What I had to do was

1)Boot into recovery mode
2)Select fsck from the menu. When this is done, a menu pops up. Selected 'root: Drop to root shell prompt', which drops me to a root shell with write access after fsck is done
3)Run the following commands and reboot:

sudo mv /var/run/* /run/
 sudo mv /var/lock/* /run/lock/
 sudo rm -r /var/run
 sudo rm -r /var/lock
 sudo ln -s /run /var/run
 sudo ln -s /run/lock /var/lock
 sudo rm /run/dbus/*

This is a nasty bug. I hope it gets addressed soon
