### Post by Rubi1200 on 2011-04-02
Hi,

please do the following:

Boot the Ubuntu Live CD/USB. Choose the option "Try Ubuntu without any changes." Once the desktop loads come back here and do the following:

1. Download the boot info script. There is a link in my signature.
2. Once downloaded, move the boot info script to the desktop.
3. Open a terminal and run the command

```
sudo bash ~/Desktop/boot_info_script*.sh
```

This will create a RESULTS.txt file on the desktop. Paste the entire contents of that file back here in a new post. Once pasted highlight all text and click the # sign on the toolbar to place code tags around the text.

### Post by Rubi1200 on 2011-04-02
Are you still unable to boot either operating system?

When did this start?

How new is the computer?

The reason I ask is because an initial look at the results does not suggest any major problem.

There are 2 things you can take a look at and try:

1. check your BIOS to see whether it "sees" the whole disk or only the first 137GB. If there is an option to enable "large disk" or something similar, do so.

2. when the computer starts and you see the GRUB menu, when the entry for Ubuntu is highlighted press "e" to edit.

Use the arrow keys and navigate to the line that ends with quiet splash

Add rootdelay=90 so it looks like this>  quiet splash rootdelay=90
Then "Ctrl" + "x" to boot.

### Post by Rubi1200 on 2011-04-02
If you try booting the older kernel version do you still have the same problem?

The only other thing worth trying is a file-system check from the LiveCD:

```
sudo e2fsck -C0 -p -f -v /dev/sda5
```If this tells you there are errors, then run this command:

```
sudo e2fsck -f -y -v /dev/sda5
```Reboot.
