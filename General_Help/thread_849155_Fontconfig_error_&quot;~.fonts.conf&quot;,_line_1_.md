### Post by txcrackers on 2008-07-04
Sounds like the file itself has been corrupted. Try re-setting the font rendering options via system settings. If that doesn't work, delete the file, then try re-setting.

 Here's what mine looks like (but it may not be appropriate for your system/monitor):
```
<?xml version="1.0"?>
<!DOCTYPE fontconfig SYSTEM "fonts.dtd">
<fontconfig>
 <dir>~/.fonts</dir>
 <match target="font" >
  <edit mode="assign" name="hinting" >
   <bool>true</bool>
  </edit>
 </match>
 <match target="font" >
  <edit mode="assign" name="hintstyle" >
   <const>hintmedium</const>
  </edit>
 </match>
 <match target="font" >
  <edit mode="assign" name="rgba" >
   <const>rgb</const>
  </edit>
 </match>
 <match target="font" >
  <edit mode="assign" name="antialias" >
   <bool>true</bool>
  </edit>
 </match>
</fontconfig>

```

### Post by jespdj on 2008-10-21
Edit the file **.fonts.conf** in your home directory, for example with gedit:
```
gedit ~/.fonts.conf
```
Make sure that the first line looks exactly like this:

**<?xml version="1.0"?>**
