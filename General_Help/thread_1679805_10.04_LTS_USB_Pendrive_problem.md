### Post by chaosmaster on 2011-05-12
In case someone else gets directed here by google:

It seems like the stick doesn't behave correctly when the partition-table is read first.

A temporary fix is:

```

sudo blockdev --rereadpt /dev/sdX

```A more permanent solution I came up with is the following udev-rule:

```

SUBSYSTEM=="block", ACTION=="add", DRIVERS=="?*", ATTRS{product}=="Padlock v2", KERNEL=="sd*", RUN="/sbin/blockdev --rereadpt /dev/%k"

```
