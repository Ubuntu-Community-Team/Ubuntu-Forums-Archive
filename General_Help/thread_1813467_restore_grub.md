### Post by oldfred on 2011-07-27
Do you get a grub menu if you hold the shift key down?

It looks like you may have a invalid entry in fstab. I would remove or comment this line out of fstab. It generally is better to use UUIDs for mounting / & swap. but the /dev/sda5 shuld work.
[COLOR=DarkRed]proc            /proc           proc    nodev,noexec,nosuid 0       0[/COLOR]
/dev/sda5       /               ext4    errors=remount-ro 0       1

Your windows sda3 has the boot flag, so it looks like the boot partition but it has both /Boot/BCD which is essential for windows and /boot folder with grub files. Since windows is not case sensitive, like Linux, it gets confused as it cannot have two identical folders. Delete the /boot with the grub files. But be careful not to delete the /Boot with the BCD.
Boot files:        /bootmgr /Boot/BCD /boot/grub/core.img

Then if you can boot into your Ubuntu install, run this and it should find windows and add it to the menu.

sudo update-grub
