### Post by audiomick on 2010-01-05
For my part, I use, amongst other things, an acoustic analysis program on my laptop. In fact I don't have an Ubuntu on it at the moment.
What stops me from trying a virtualisation for that is that I cannot imagine that the constant latency that the program needs could be possible if there is a host system in between that the client system running the program doesn't know about.

Similarly, the programs for Yamaha audio equipment use a network driver that is sulky at the best ot times. I simply can't be bothered trying to make that work through a virtualisation.

From what I have read here, particularly gamers have a better experience in a "real" windows than in a virtualisation. In that instance, the issue is the, albeit small, loss in performance that  a virtualisation will always have compared to running native on the same hardware.
