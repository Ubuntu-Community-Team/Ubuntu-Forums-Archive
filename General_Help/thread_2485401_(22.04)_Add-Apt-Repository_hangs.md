### Post by MAFoElffen on 2023-03-29
Further from their PPA page:
> 
Third-Party Python Modules
==========================

 [COLOR=#008000]Python modules in the official Ubuntu repositories are packaged to  work with the Python interpreters from the official repositories.[/COLOR]  [COLOR=#ff0000]Accordingly, they generally won't work with the Python interpreters from  this PPA.[/COLOR] As an exception, pure-Python modules for Python 3 will work,  [COLOR=#ff0000]but any compiled extension modules won't.[/COLOR]

 To install 3rd-party Python modules, you should use the common Python  packaging tools.  For an introduction into the Python packaging  ecosystem and its tools, refer to the Python Packaging User Guide:
[https://packaging.python.org/installing/](https://packaging.python.org/installing/)

 
It would work inside Python virtualenv, if constrained within.

I used to run into this with multiple installs of Java versions and targeting code to a certain version, or updating old code to work with current versions.

With Python code, I try to stay within what is current, looking ahead to the next version for just trying not to back myself into a corner.

**[COLOR=#ff0000]Do not reset what is set in update-alternatives for the interpreter. It will break apt and you will be @#$%'ed.[/COLOR]** 

You really have to stay aware and be careful at what $PYTHONPATH is set to. I use Eclipse PyDev, which at least you easily set which interpreter you are setup to use, and try to keep track of that in the preferences.

I don't see why you are getting errors with that.

Adding that PPA should just add these two lines to your sources.d file:
```

deb https://ppa.launchpadcontent.net/deadsnakes/ppa/ubuntu jammy main 
deb-src https://ppa.launchpadcontent.net/deadsnakes/ppa/ubuntu jammy main 

```
I can resolv that repo, so I know it is there. It may be that launchpad just did a revamp to proxy all content to https to 'ppa.launchpadcontent.net/username/<whatever>'

For instance for the [UbuntuForums 'system-info' Script PPA]("https://launchpad.net/~mafoelffen/+archive/ubuntu/system-info") Repo resolves to: [https://ppa.launchpadcontent.net/mafoelffen/system-info/ubuntu/](https://ppa.launchpadcontent.net/mafoelffen/system-info/ubuntu/) , which the deadsnakes repo has been around for awhile, and the resolv is somehow different for that PPA... IDK.

That might be an issue to report to the "deadsnakes" Team at: [https://github.com/deadsnakes/issues](https://github.com/deadsnakes/issues)
