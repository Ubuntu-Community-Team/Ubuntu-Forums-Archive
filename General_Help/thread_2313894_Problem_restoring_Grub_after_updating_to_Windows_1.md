### Post by oldfred on 2016-02-17
You cannot easily update your system.
 Vivid Vervet (15.04) reached End of Life on February 4, 2016

 Old Releases past support
 [http://old-releases.ubuntu.com/](http://old-releases.ubuntu.com/)
[https://help.ubuntu.com/community/EOLUpgrades](https://help.ubuntu.com/community/EOLUpgrades)

Better to update to supported version of Ubuntu. 
I typically use a LTS as main working install, but often install newer version in another / (root) partition for test.

It also looks like you left Windows fast start up on. That is always on hibernation. Linux will not mount hibernated Windows normally so it cannot see the NTFS partition's data or add it to grub menu.

You may need to temporarily reinstall a Windows boot loader to MBR to directly boot Windows. You can use your Windows repair disk/installer or Boot-Repair can add a Windows type boot loader. 

To reinstall grub Boot-Repair tries to download from repository, but that is now closed. Otherwise Boot-Repair probably would be able to fix system.

Supergrub is still available. I like to have Boot-Repair, Supergrub, Knoppix, Parted Magic, gparted and whatever else will fit on flash drive as ISO and directly boot with grub loopmount.

 [http://www.supergrubdisk.org/](http://www.supergrubdisk.org/)
[http://www.supergrubdisk.org/wiki/Main_Page](http://www.supergrubdisk.org/wiki/Main_Page)
