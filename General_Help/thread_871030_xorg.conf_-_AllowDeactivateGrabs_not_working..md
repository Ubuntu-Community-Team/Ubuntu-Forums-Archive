### Post by hpesoj on 2008-07-26
I have an application which uses an input system (OIS to be specific) which locks the mouse and keyboard for its own use.  I can alt-tab out when the app is running, but when using the gdb debugger, not even OIS is processing input, and I'm left with no option but to hard reboot.

I have heard the best solution is to add this to /etc/X11/xorg.conf:

```
Section "ServerFlags"
	Option "AllowDeactivateGrabs" "on"
EndSection
```

Supposedly this is meant to allow you to unlock the input using CTRL-ALT-KEYPAD_DIVIDE, but it appears to do nothing for me.

I have tried both "on" and "true" as I have seen both used, but neither work.  Here is my xorg.cfg, and Xorf.0.log:
