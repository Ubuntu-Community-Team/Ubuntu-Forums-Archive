### Post by aitkinst on 2017-10-03
I've been trying to setup an Active Directory Domain Controller on my home LAN using Samba and am stuck wondering what is wrong.  Help!

I'm using Ubuntu Server 16.04.3 LTS which is a VM on an unraid server inside the LAN.  My LAN is behind a router running DD-WRT which acts as a DHCP, and DNS forwarder.  The DHCP is assigning the hostname 'adc1' and IP '192.168.1.5' based on the MAC Address of the VM when it boots (I tried using a static IP from within /etc/network/interfaces but I think I was doing something wrong because I could never get resolver to push the nameservers or search lines to /etc/resolv.conf).  This was it essentially boots with a static IP and DNS seems to work both inside (DNSMasq on the router), and it resolves outside the LAN.

My Hosts file:
> todd@adc1:~$ cat /etc/hosts
127.0.0.1       localhost
127.0.1.1 adc1


# The following lines are desirable for IPv6 capable hosts
::1     localhost ip6-localhost ip6-loopback
ff02::1 ip6-allnodes
ff02::2 ip6-allrouters

My /etc/network/interfaces file:
> todd@adc1:~$ cat /etc/network/interfaces
# This file describes the network interfaces available on your system
# and how to activate them. For more information, see interfaces(5).


source /etc/network/interfaces.d/*


# The loopback network interface
auto lo
iface lo inet loopback


# The primary network interface
auto enp2s1
iface enp2s1 inet dhcp
        address 192.168.1.5
        netmask 255.255.255.0
        gateway 192.168.1.1
        dns-nameservers 127.0.0.1 192.168.1.1
        dns-search mydomain.lan

My /etc/resolv.conf file:
> todd@adc1:~$ cat /etc/resolv.conf
# Dynamic resolv.conf(5) file for glibc resolver(3) generated by resolvconf(8)
#     DO NOT EDIT THIS FILE BY HAND -- YOUR CHANGES WILL BE OVERWRITTEN
nameserver 192.168.1.1
nameserver 127.0.0.1
search mydomain.lan

So steps I took:
```
sudo apt-get install samba krb5-user krb5-config winbind libpam-winbind libnss-winbind                  #Downloaded and Installed packages
```
During the install I set:
Default Kerberos version 5 realm = **MYDOMAIN.LAN**
Kerberos servers for your realm = **mydomain.lan**
Administrative server for your Kerberos realm = **mydomain.lan**

```
sudo systemctl stop samba-ad-dc.service smbd.service nmbd.service winbind.service                      #Stopped existing services
sudo systemctl disable samba-ad-dc.service smbd.service nmbd.service winbind.service                  #Disabled services
```

```
sudo mv /etc/samba/smb.conf /etc/samba/smb.conf.initial                                                            #Prevent existing smb.conf to conflict with new setup
```

```
sudo samba-tool domain provision --use-rfc2307 --interactive                                                        #Provision the new domain
```

```
sudo mv /etc/krb5.conf /etc/krb5.conf.initial                                                                                 #Save default/previous conf
sudo ln –s /var/lib/samba/private/krb5.conf /etc/                                                                           #Link new krb5.conf
```

```
sudo systemctl start samba-ad-dc.servicesudo systemctl enable samba-ad-dc.service
```

When I test out the system I get good pings for mydomain.lan adc1.mydomain.lan and adc1 (both locally or from any other machine on the LAN).  I also get good references for:
```
host -t A mydomain.lan
host -t A adc1.mydomain.lan
```

But when I check for SRV I get nothing on:
```
host -t SRV _kerberos._udp.mydomain.lan
host -t SRV _ldap._tcp.mydomain.lan

todd@adc1:~$ kinit
kinit: Cannot find KDC for realm "MYDOMAIN.LAN" while getting initial credentials

```
