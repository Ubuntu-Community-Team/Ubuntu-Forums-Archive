I don't know if this is the reason, but one of the most unfriendly parts of Windows is the way it allows you - maybe even encourages you - to make your desktop messy with program shortcuts and folders and files.

The choice was made by Gnome developers, not Unity developers. Since Gnome 3's default user interface, Gnome Shell, doesn't use desktop icons they had no reason to include any option to create one. And Unity is just a shell running on top of the Gnome 3 desktop environment.

Anyway, if you want my explanation to why most modern desktops (like those used on OSX and Windows 7, as well as Linux) try to encourage users to place program launchers in the panels instead of desktop, the desktop icons would be hidden by any windows, forcing the user to minimize/maximize or move windows around to access the launchers and then return back to whatever tasks they were doing. Panels can instead easily be hidden and shown when the mouse cursor is taken to a screen edge, thus making panel launchers more easily accessible less interruptive to the user's workflow.
