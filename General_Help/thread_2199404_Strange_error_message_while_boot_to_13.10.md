### Post by efflandt on 2014-01-13
You may get one or two DRM errors like that in 13.10 during boot, even after you install nvidia drivers. And I occasionally get those pop ups too. ### Post by buzzingrobot on 2014-01-14
1.  Try opening a terminal and running "sudo nvidia-xconfig", then reboot.  That ensures you have the xorg.conf file needed to run the Nvidia proprietary driver.  Typically, you will get a black screen with Nvidia's driver if xorg.conf isn't there. (Nvidia-xconfig should execute during the driver install process, but it often does not, on every distribution i've used.)

2.  The "System Program Problem..." popup is generated by a program called "apport". In very many cases, if that Apport did not produce that popup, you would not have known something happened. It often reports known, and unpatched, bugs. A number of distributions based on Ubuntu disable Apport.  The same bugs likely still happen but users don't notice them.  Google "disable apport" if you want to disable it on your system.

### Post by buzzingrobot on 2014-01-14
> **danbuz said:**
> When I go to "additional drivers" it is empty and there is no additional drivers to isntall.

"Additional Drivers" won't display any proprietary drivers if it doesn't detect any Nvidia card.  The most likely reason it doesn't detect any Nvidia card is that the card is disabled in BIOS.  You mention Optimus so I will assume you are using a laptop. That also suggests the machine has onboard Intel video as well as the Nvidia card.  

Bumblebee is an open source Linux effort to provide capability similar to that of Nvidia's Optimus on Windows.  The Optimus package is not available for Linux. Nvidia doesn't make Optimus for Linux and, therefore, it is not included in the proprietary drivers it does make available. Hence, those drivers will not work on a machine set to use Optimus. Someone else will need to offer suggestions for installing and using Bumblebee. I don't use it.

The BIOS in many machines allows a user to select either the onboard Intel video or the discrete Nvideo card.  And some offer a third Optimus option.  If you select Intel video in the BIOS, you do not need to install any drivers because the Intel drivers are included in the Linux kernel.  If you select the Nvidia option in the BIOS, Additional Drivers should detect it and offer a selection of appropriate drivers.

In any case, the preferred way to install Nvidia proprietary drivers is via Additional Drivers. Once an incorrrect Nvidia driver is 
installed, or the correct driver installed incorrectly, it's best to remove and back all that out before trying to install via another method.
