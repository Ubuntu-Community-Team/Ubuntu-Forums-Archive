### Post by hansdown on 2010-06-09
In case I may be wrong, could you download the boot infi script, and run it?

[http://bootinfoscript.sourceforge.net/](http://bootinfoscript.sourceforge.net/)

If you know for sure, where the boot flag should be, run the live ubuntu cd, and run it "without changes to the computer".

Click system> administration> gparted.

Right click the partition> flags, then you can put the flag there. If it works when you boot back into your system, then it would be safe to remove the other flag.
