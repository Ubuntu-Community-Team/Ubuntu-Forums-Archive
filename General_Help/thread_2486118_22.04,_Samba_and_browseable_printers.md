### Post by strungoutfan78 on 2023-04-20
I'm currently trying to build a cups server at work and for some reason the printers share is not browsable, despite my samba config:

Samba config:
```
[printers]
   comment = All Printers
[B]   browseable = yes
[/B]   path = /var/spool/samba
   printable = yes
   guest ok = no
   read only = no
   create mask = 0700
   write list = root @adm @wheel cups_user
```

testparm
```
[printers]
[B]        browseable = No
[/B]        comment = All Printers
        create mask = 0700
        path = /var/spool/samba
        printable = Yes
        read only = No
        write list = root @adm @wheel cups_user
```

Does anyone know what gives with this?  Is it a bug in the current version?  My Google-fu is failing me on this one.  Thanks!

### Post by #&amp;thj^% on 2023-04-20
In the [printers] section, change the guest ok option to ‘yes’:
```

browsable = yes
guest ok = yes
```
and just a reminder, After editing smb.conf, restart Samba:
```

sudo systemctl restart smbd.service nmbd.service

```
Also take a peek here (global) [https://linuxconfig.org/how-to-configure-samba-server-share-on-ubuntu-22-04-jammy-jellyfish-linux](https://linuxconfig.org/how-to-configure-samba-server-share-on-ubuntu-22-04-jammy-jellyfish-linux)
