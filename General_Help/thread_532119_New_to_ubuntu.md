### Post by schallstrom on 2007-08-22
Okay, to prevent any misunderstandings: When you put the Ubuntu Live CD into your drive and boot from it, a fully functional operating system is booting up, but **nothing** gets installed onto the hard disk. That's why it's called a "Live CD". It's running only in memory, the hard disk remains totally untouched.

I assume that is the status you are talking about (I deduce this from your 2nd question).

So, to answer your questions:

1) The Ubuntu Installer does not format your hard disk without asking you if it should do so. During the installation process you get the opportunity to format / partition your hard disk in various ways, or just let the installer apply some default scheme.

2) If you want to install Ubuntu to a hard disk (it's running much faster than when started from a Live CD) there must be partitions on that hard disk with a file system supported by Linux. The installer helps you to prepare your hard disk. You may install Ubuntu besides Windows on the same hard disk (although 10 GByte might be a bit small to run Windows and Ubuntu in parallel) or you just tell the installer to use the whole disk and wipe out Windows, if you don't care about the Windows installation anymore. If you have trouble doing so, feel free to ask a more specific question.

3) You can't really 'skip' this step. The installer needs to know where to install the Ubuntu system. But the easiest would be to instruct the installer to use the whole disk and do the partitioning automatically. That's perfectly acceptable if it's just a test system as you wrote.

Hope that helps.
