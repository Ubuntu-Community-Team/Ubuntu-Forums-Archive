### Post by jloveless on 2008-07-03
First, my PC setup and then my question(s)
SETUP: I have a 1 year old HP PC with a 250GB (sda) and a 750GB drive (sdb). 

The 750 G drive has 4 partitions:
sdb1 - an early version of 8.04 U
sdb3 - /home - shared among 2 installs of 8.04 U 
sdb4 - a current version of 8.04 U
sdb5 - swap - used by both Ubuntu installs.

The 250 GB drive has one partition
sda1 - a single partition with Windows Vista Basic

sdb1 has the boot MBR - as Grub - it offers to Boot Vista and either the 1st install of Ubuntu on sdb1 or the 2nd version of 8.04 U through Chainload +1. If I select Vista, Windows boots from the MBR on sda1. If I select the current install of 8.04 U it chains to Grub on sdb4 where I have yet another boot menu which includes Recovery and Memtest for the second version.

The 8.04 U on sdb1 is pretty messed up and I want to clean it off. Obviously I don't need 2 similar installs of Ubuntu, so I thought I would use that first partition as a test bed for other distros.

Here's my question(s):
1 - how can I clear that partition without disturbing the grub boot record?
2 - can I boot directly from sdb4?
3 - do I have any other options that would make the 8.04 on sdb4 my primary boot and whatever is on sdb1 and Vista as options?

I am afraid if I try to install, say Kubuntu, on sdb1 that I would wreck my boot record and not be able to get to the primary Ubuntu install or Windows.

Sorry for being long winded but it is not a simple setup to describe. Many thanks for any thoughts and comments.

Jon

### Post by VMC on 2008-07-03
You did a good job explaining the situation, but I would like to see how fdisk views your setup. Could you output that?

sudo fdisk -l

Also, this is the first time I've heard of someone chainloading ubuntu. I'm not quite sure what to make of it. I'm not clear on the picture, but it sounds like you have two grub menu's.

Could you output the "/boot/brub/menu.lst" as well

As for as installing another Linux, it shouldn't matter if you over write any partition with a new install, as long as you don't over write the mbr. Just skip that section on the install.
