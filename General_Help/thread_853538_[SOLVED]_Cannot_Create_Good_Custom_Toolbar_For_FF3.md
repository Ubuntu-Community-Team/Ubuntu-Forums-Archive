### Post by Tim_Olaguna on 2008-07-08
I rely heavily on a custom toolbar which contains quick icon links to services and features I need. I've used such toolbars for many years and several generations of Firefox. The one I use with Firefox 3 on Vista and our iMac is just perfect.

But I'm having trouble creating and keeping a good one while using FF3 and Ubuntu Linux 8.04.1. FF3 will:
- Create a new toolbar (or so it says).
- And I am able to drag icons to it.
- And the icons sometimes stay in place.

But it won't:
- Label the toolbar with any name I create.
- All of the icons are added to the beginning of the Bookmarks Toolbar[1] after I click the "Done" button (which makes for an unacceptable display mess - nothing like I get on the Vista side or on our iMac version of FF3).
- Sometimes all of the carefully chosen and arranged icons simply disappear from the toolbar immediately after clicking "Done".
- And now the application simply doesn't do anything if I click on the option to add a new toolbar and type in a name for it. It takes the name, then the box in which the name was typed then simply disappears. The typed-in name shows up nowhere. Just as in the case when I tried to create the first custom toolbar.

Note [1]: The icons which display themselves in front of the items contained in the Bookmarks Toolbar are NOT present when you go to Bookmarks > Organize Bookmarks > Bookmarks Toolbar . But you do see them when you go back to viewing the standard browser window.

Comments and suggestions are appreciated. Though suggestions to create a new profile will be less so. I've put a lot of work into customizing my current configuration and "view" of FF3 on Ubuntu and don't relish the prospect of having to do it all over again just to see if maybe it works.

### Post by Tim_Olaguna on 2008-07-08
Solution

It appears the issues I described are theme related.  It turns out I can add custom toolbars as expected as long as I am using the default Firefox Ubuntu theme.  I have the problems and issues I described above trying to add new custom toolbars when I use the W3v8, Nautipolis, or other themes.  

Once I have established a new custom toolbar and added at least one icon to it, however, I can then switch from the Ubuntu theme back to one of my preferred favorites.  The new toolbars stay in place and I can add new icon items to them (items which stay even after a FF close and reopen) as usual.

This is acceptable to me.  I add and delete icons on a custom toolbar far more frequently than I add or delete toolbars themselves.
