### Post by pbpersson on 2009-03-20
When you say "he checked the disk for errors" are you referring to the downloaded CD or the hard drive?

If he does not care about any of the data on the hard drive I would boot from a live CD and then go to System-->Administration-->Partition Editor, delete all the partitions, and then try installing again.

If you still have a problem you might have a bad hard drive.

I think the live CD also has a memtest you can run although that takes a VERY long time as I recall because it does....a VERY thorough test on the RAM.

I am thinking in typing this there might be a hardware problem on that machine which is why XP was doing the BSOD thing....but that is just my gut feeling.
