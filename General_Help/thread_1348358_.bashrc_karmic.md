### Post by 3togo on 2009-12-09
I don't know the exact reason but I guess some ssh server 
still following classical standards which looking for
.bash_profile first, then .bash_login and finally .profile

therefore u need to add a .profile file to call your .bashrc 

jc@jc-laptop:~$ cat /etc/skel/.profile
# ~/.profile: executed by the command interpreter for login shells.
# This file is not read by bash(1), if ~/.bash_profile or ~/.bash_login
# exists.
