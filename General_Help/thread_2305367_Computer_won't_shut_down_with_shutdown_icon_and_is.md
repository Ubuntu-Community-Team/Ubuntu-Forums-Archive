### Post by Vladlenin5000 on 2015-12-05
The commands [here]("http://ubuntuforums.org/showthread.php?t=2305035") are fine and have nothing to do with anything else.
You're still not understanding a few details. Let's try some clarification:

1. You don't upgrade a 32-bit to a 64-bit system.
2. The whole point of the instructions given in the aforementioned thread is to assure you have a good **backup** of your personal file and settings so you can **install from scratch** the 64-bit OS you want (Xubuntu??).
3. Considering #2 there's absolutely NO point in troubleshooting your current system.
4. > I would like to install an operating system on my computer that didn't constantly have to be updated and be done with it
**There's no such thing!  **the closest you can have is a major Linux distro and among them Ubuntu and derivatives. Unlike Windows where you're supposed to be constantly and independently updating the OS, drivers, anti-virus, anti-malware, etc. etc. etc. etc., in Ubuntu you can benefit from a centralized updates manager that takes care of everything from a single easy-to-use tool.
**Updates are crucial to protect YOU from online threads regardless of the OS you're **using.

Perhaps you should have invested more in learning the basics - and it's soooo easy, a 6 years old child can do it (really) - when you had the time instead of relying solely in "some guy". Now you either find another "guy" or you learn the basics for yourself. We can definitely help with the latter, not so much with the former.
