### Post by mb_webguy on 2008-09-20
Personally, I don't trust WYSIWYG editors, and my experience with them has proven that distrust well-founded -- from a web developer's perspective.  Sure, they may let you create a pretty page quickly and easily, but you just don't really know what's going into your code.  You usually get a lot of useless garbage, some of which is redundant, other extraneous, and all of it there in anticipation of something you'll probably never do to the page.

I want a page to look the way I want it to, and not by some accident of fate, but through my applied intellect and creativity.  So, I also use gedit to write the code, and use various web browsers to view  my work in progress.

However, you're not a web developer.  A WYSIWYG editor might be for you.  In that case, look into Kompozer.  It's in the repositories.

But people learn faster by doing, so if you really want to know about web design, I'd suggest getting your hands dirty every once in a while and actually write the code.
