### Post by cesoid on 2007-03-12
After upgrading to Edgy Eft, I found that on the Workspace Switcher, when I drag one of the little windows to move it from one desktop to another, they suddenly become two or three times bigger. Previously, with Dapper, they would just stay really small, which meant it was easier to tell which desktop I was holding them over (now they overlap several of them). The other annoying part about it is that, since they're so large and I'm holding them by the top, they extend off the bottom of screen and I can't see the icon on them or the shape of them, which would help indicate whether I got the right one.

This is minor, but it seems like it should have a simple solution. There aren't any settings for "window size while dragging" in the preferences, and I haven't been able to find anyone else complaining about the same problem.

Well, you wouldn't find me complaining about it since I didn't know the problem existed. Now I do. :)

Basically, you have to:

1.  hold the window icon in such a way that it is mostly off-screen as you move it from left to right or right to left. By doing so, you can see the virtual desktop color change as it becomes active. At least in this scenario, you know which virtual desktop the window will end up on.

2. if you need to know which window you are dragging, simply slide it upwards to see what it is before doing Item #1 above. If you have grabbed the wrong window icon, rinse and repeat.

It'll be interesting to see if someone knows of settings that will revert this to the previous behavior. It may be that people previously complained that the icons were too small and now the developers have responded by making them too big.

The main reason I hadn't noticed this before is I rarely used that technique to move windows. I have XFWM installed as the windows manager on top of Gnome, and you can drag the windows full-size from virtual desktop to virtual desktop.
