### Post by mcduck on 2007-08-23
That would hardly help anything, as you would still need to load the program into the RAM disk before you can run it. Just like you normally load a program into RAM to run it (and normally after the program is once started it will stay in RAM anyway, at least as long as you don't run out of memory).

If you want to speed up program startup check the 'preload' package, it learns what programs you use often and then starts loading them into RAM during the boot.
