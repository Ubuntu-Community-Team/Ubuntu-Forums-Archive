### Post by Derspankster on 2009-01-29
This morning (January 29, 2009) I installed some updates that included a kernal update on 2 computers running Hardy.  Both machines are set up to show certain items on their desktops, including as Trash icon. After the update, neither desktop trash icon functions. Moving a file into trash doesn't change the icon to show anything in the trash and if the trash is opened it will show the file in there but it cannot be deleted. Adding the trash icon icon to a panel will show the file and functions normally. 

All the other desktop icons function normally. I added the desktop icons using the configuration editor. Both machines and their desktop trash icons worked fine BEFORE this mornings update.
