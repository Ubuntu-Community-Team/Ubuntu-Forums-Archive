### Post by bodhi.zazen on 2007-10-04
I am not sure if I fully understand the question/concern.

One can not look at the source code for windows, yet there are thousands of worms/viruses for that operating system. That alone disproves your hypothesis.

Yes, there are crackers out there. There are no linux viruses in the wild *because the code is open source*. Because of this, potential holes in the code are patched much more rapidly then is possible with closed code.

For example you might know windows is vulnerable to evil virus xyz, but knowing this how would you patch the code? You can not, you have to wait for someone else to do it for you.

If *nix is found to be vulnerable to evil virus xyz, well you could write the patch yourself and share the solution, problem solved.

Is it perfect, no. But the *nix community is very proactive in terms of security.

I think your assumption that crackers ignore *nix is flawed. *nix has been around for a long time ans is a very mature OS. In addition there is no lack of highly valuable *nix servers and certainly there is ample motivation to crack *nix.

Take a look at this link : [Install evilmalware](http://www.gnu.org/fun/jokes/evilmalware.html)

### Post by notwen on 2007-10-04
> My concern isn't so much with virius because I am fairly sure that I will not be getting one. Rather my concern is that my operating system or any of its applications being capable of being exploited because the code is readily available.

This works both ways, of course someone could write code to open up a Linux distribution, but it's just as easily secured by modifying the same source code.  Downloading the more popular distributions generally guarantees a less-likely chance of having a unsecured Linux distribution.  Once a hole is found, it is patched very quickly because a user has found it and... once again the source code is readily available to be modified and the fix is included in the next release and also patches are generally available for insecurities as well.  Hope this helps you understand a little bit better. =]
