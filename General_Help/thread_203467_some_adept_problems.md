### Post by Lil_Eagle on 2006-06-30
It's not KDE.  I have both kubutnu and ubuntu (so gnome and KDE).  The problem lies with sudo and the security.  If you open a console and do something with sudo, that prevents kdesu or gksu from working.

It is very annoying.  I suppose it is a lesson that we either need to not use the command line, or only use the command line.

What is more interesting is that it works to enter sudo and the command from a terminal.

For example.  I have an icon on my panel that is ```
kdesu /usr/bin/konqueror --profile filemanagement
```It sometimes works but usually not.  But, if I open a terminal and type:
```
sudo /usr/bin/konqueror --profile filemanagement
```
after several warning messages about the display, konqueror opens as root.  That is what I want the command to do.  What I don't understand is why it has to work that way.  Something is broken.  

According to all the documentation you're not supposed to call GUI apps with sudo, that's why there is a kdesu and gksu.
