### Post by UnusedUserNameHere on 2020-05-12
I followed the guide in the wiki for full disk encryption when installing Ubuntu: [https://help.ubuntu.com/community/Full_Disk_Encryption_Howto_2019](https://help.ubuntu.com/community/Full_Disk_Encryption_Howto_2019)

I run into the issue where sometimes I incorrectly type the password, and it will drop to what I believe is called the Grub rescue shell. I can "reboot" to try again, but it's not ideal since I have a RAID card that takes about a minute to initialize each reboot.

I've seen instructions on how to get Grub to "retry" without rebooting, similar to: [https://blog.stigok.com/2017/12/30/decrypt-and-mount-luks-disk-from-grub-rescue-mode.html](https://blog.stigok.com/2017/12/30/decrypt-and-mount-luks-disk-from-grub-rescue-mode.html)

I am able to decrypt the correct partition, for me: (hd3,gpt1). I will get the "Slot 0 opened" message.

Then when I type "insmod normal" and "normal" neither command gives any output to indicate any error. However, it just stays in the rescue shell and nothing further happens.

Is there anything else that I can try from within the rescue shell to reattempt the boot process without restarting the PC?

Thanks!

**Edit / Solution:**
I had to add a couple of steps to make it work.
```

> cryptomount (hd3,gpt1)
[I]Your specific partition may vary.
Once you enter your password, you should get a "Slot X opened" message, pay attention to the number[/I]
> set root=(crypto**0**)
> set prefix=(crypto**0**)/grub
*You may need to change the **0** to whatever "slot" was opened*
> insmod normal
> normal

```
