### Post by mcduck on 2007-05-04
Try this:

Open the fstab for editing by running 'gksudo gedit /etc/fstab', and then change the last number on your NTFS & FAT drive entries from '1' to '0'.
```

# /etc/fstab: static file system information.
#
# -- This file has been automaticly generated by ntfs-config --
#
# <file system> <mount point> <type> <options> <dump> <pass>

proc /proc proc defaults 0 0
# Entry for /dev/sda3 :
UUID=a077f582-614a-40a9-91a5-0a0e59e0ac73 / ext3 defaults,errors=remount-ro 0 1
# Entry for /dev/sda2 :
UUID=7420256F20253A0E /media/Windows ntfs-3g defaults,locale=en_US.UTF-8 0 0
# Entry for /dev/sda1 :
UUID=07D7-0110 /media/sda1 vfat defaults,utf8,umask=007,gid=46 0 0
# Entry for /dev/sda4 :
UUID=f32c73cf-226d-4e6f-8c14-c551ac3ec470 none swap sw 0 0
/dev/scd0 /media/cdrom0 udf,iso9660 user,noauto 0 0
none /proc/bus/usb usbfs devgid=1002,devmode=664 0 0
```

Then reboot, and your problem should be history. :)

The number tells if the partition should be checked on boot, and in which order. '0' means that fsck should not check the partition, '1' should only be used for / partition, and for other partitions you wish to check use '2'. I don't think fsck can actually do anything useful for windows partitions so I recommend disabling it completely, but in no case should it be '1' like you have now.
