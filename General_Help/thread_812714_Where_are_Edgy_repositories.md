Support usually means development. Not all of us want to take on a part time volunteer job as system administrators for our home computers. Some of us just want to stick with whatever works without having to wrestle with new bugs and quirks every few years just to maintain the same functionality. Maintaining stable backwards compatible software versions is a long time unix tradition. That's why your Ubuntu is chocked full of 25 year old applications, some of which haven't been updated for the last 20.

If it ain't broke then don't fix it. My Edgy install works for me and I don't need Hardy to force me to rehack various drivers, loaders, etc just to keep my TV out functionality from breaking (for example).

If I wanted my system to stop working every few years then I would run Windows, but I run Linux so I don't have to upgrade unless I want new features (and Ubuntu, in particular, so I don't have to reinvent the wheel on the command line whenever I decide I do want to reinstall and/or upgrade).

Compiling from source is aside the point. The point is to maintain a stable snapshot of software (whether binary or source) known and tested to be compatible with well documented bugs and fixes so that I don't have to go hunting the internet for information on version compatibility just to install an application and when I do have to hunt down a fix I only have to do it once (not once every three years).

Nevertheless, it seems my expectations are not so unrealistic after all because the official breezy and edgy repositories seem to still be available here:
[http://old-releases.ubuntu.com/ubuntu/dists/](http://old-releases.ubuntu.com/ubuntu/dists/)

BTW, OpenBSD has CDs for sale containing their full official repository as far as 20 releases back:
[http://www.openbsd.org/orders.html](http://www.openbsd.org/orders.html)
OpenBSD 2.8 released May 19, 1998 is in stock
They also have online repositories as far back as 10 releases (5 years)

Granted, the ubuntu official ubuntu repositories are much larger, but I'd gladly pay for a DVD set once the Edgy repositories go offline.
