### Post by SpectrumDT on 2023-11-09
Dear forum,

I have a .NET application that can generate and validate certificates. This validation logic behaves differently on different Linux versions. 

I have a full working example here (C# code): [https://github.com/CAP-3Shape/CertificateValidatorSample](https://github.com/CAP-3Shape/CertificateValidatorSample)

In the code above repository, I load one fixed certificate from a resource (i.e., it is always the same certificate) and try to validate it. I expect to get two status flags. But on some Linux machines I get a 3rd flag, *PartialChain*. The error message looks like this:

> Unhandled exception. System.Exception: Illegal status flag found: PartialChain (unable to get local issuer certificate)

The problem does not happen on every Linux machine. The code works correctly on this Docker image: 

* mcr.microsoft.com/dotnet/aspnet:6.0-focal (Ubuntu)

I can reproduce the error on the following Docker images: 

* mcr.microsoft.com/dotnet/aspnet:6.0-jammy (Ubuntu)
* mcr.microsoft.com/dotnet/aspnet:6.0-alpine (other Linux; not Ubuntu)

I have 3 Dockerfiles in the repository above which reproduce the behaviour I describe.

I suspect it has to do with the software installed on each of those images. I've compared the two images using *docker scout compare*, and I notice that the following cryptography-related packages have different versions between the Focal and Jammy images:

* ca-certificates
* libcrypt1
* libgcrypt20
* libk5crypto3
* libxcrypt

I tried to install different versions of them, but that didn't work. Apparently the versions used by one Ubuntu version aren't available for the other Ubuntu version.

Does anyone have advice for how to debug this? How can I figure out which packages I might need to install or uninstall to get this code to pass or fail?

Thanks in advance!

### Post by SpectrumDT on 2023-11-20
I found a workaround: My leaf certificate contained **X509KeyUsageFlags.None**. Apparently some newer cryptography libraries consider this an error. When we changed our certificates to use another flag, they started working on all Linux versions.
