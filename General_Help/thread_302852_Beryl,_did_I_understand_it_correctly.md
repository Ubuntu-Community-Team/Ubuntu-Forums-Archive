### Post by JohnUK89 on 2006-11-19
Beryl is simply a window manager, not a desktop environment. There needs to be some form of environment installed (eg GNOME) before Beryl will work. So yes, the two can (and must, actually) be installed at the same time

Beryl is more visually appealing, yes, it uses 3d acceleration to creat more eye candy on the desktop, including a cube for virtual desktops, wobbly windows, etc. That said, it does use more resources than Metacity (GNOME's window manager)

There are a couple of programs that display visual flaws when Beryl is installed (eg XaoS becomes translucent) but there should be no programs that stop working altogether. That said, Beryl is still a beta, and YMMV. Because it is in Beta, I would consider if you REALLY want it before installing, it may break something.

You do not need to exit GNOME to install Beryl, although to activate it you will need to close GNOME as per the how-to (There are several at [http://doc.gwos.org/index.php/EyeCandyDapper](http://doc.gwos.org/index.php/EyeCandyDapper))

It is reasonably easy to switch between Beryl and Metacity(GNOME's window manager). on your notification area there should be an icon similar to a ruby, if you right click that and go to Select Window Manager you can choose between the ones you have installed.
