### Post by unutbu on 2009-10-07
When placed in a conkyrc, 

```

"${execi 30 tail -n3 | awk '{print " ",$5,$6,$7,$8,$9,$10}' | fold -w50}"
```

tells conky to run the command
```

tail -n3 | awk '{print " ",$5,$6,$7,$8,$9,$10}' | fold -w50
```

every 30 seconds. The problem is, the above command is malformed. It has no output.
The "tail" command returns the last 3 lines of its input.
The command provides no input to tail, so you get no output. The rest of the command also does nothing since it is not being fed any input.

### Post by unutbu on 2009-10-07
"tail" reads the file /var/log/messages and outputs the last 3 lines.
This output is piped to awk. awk processes each line, one at a time.
It splits the line into space-separated fields. For each line, it outputs the 5th through 10th fields. 

So if /var/log/messages has a line like
```

Oct  7 09:52:23 localhost kernel: [ 657.089647] Inbound IN=eth1 OUT=
```

then 
```

tail -n3 /var/log/messages | awk '{print " ",$5,$6,$7,$8,$9,$10}'

```
prints
```

kernel: [ 657.089647] Inbound IN=eth1 OUT=
```

The output of awk is piped to the command "fold -w50". fold formats the line to be no more than 50 characters wide. 

I don't really know what the output means. All I know is this message was generated by the kernel, about 657 seconds after boot. It seems to have something to do with networking, and in particular the eth1 network device. 

For more details on what the commands mean, run
```

man tail
man awk
man fold
```

### Post by unutbu on 2009-10-08
Perhaps a more robust way to get at the information is to use a little script. If you save this in ~/bin/print_network_messages.py

[PHP]#!/usr/bin/env python
import sys
for line in open(sys.argv[1]):
    if 'Inbound IN=eth1' in line:
        fields=line.split()
        info=' '.join([field for field in fields
                       if field.startswith('DPT') or field.startswith('SRC') ])
        print ' '.join(fields[:3]), info[/PHP]

and make it executable:
```

chmod +x ~/bin/print_network_messages.py
```

then you could change your conkyrc to 
```

${execi 30 print_network_messages.py /var/log/messages}
```

This script will ignore all lines in /var/log/messages unless it contains the string "Inbound IN=eth1". Doing this should cut down on the bogus lines that 
```

tail -n4 /var/log/messages | awk '{print " ",$1,$2,$3,$21,$11}'
```

is probably printing.

### Post by unutbu on 2009-10-09
Here is a "souped-up" version of the little script.
[PHP]
#!/usr/bin/env python
import optparse
import sys
from dateutil.parser import parse

usage = 'usage: %prog [Options]'
parser = optparse.OptionParser(usage)
parser.add_option('-n', '--num', dest='num', 
                  default=0,
                  type='int',
                  help='set number of lines', 
                  metavar='NUM')
parser.add_option('-s', '--sort', dest='sort',
                  default=[],
                  action='append',
                  help='set sort type', 
                  metavar='TYPE (src,port, or datetime)')
parser.add_option('-f', '--filter', dest='filter',
                  default='',
                  help="set filter (e.g. 'Inbound IN=eth1')", 
                  metavar='STRING')
(opt, args) = parser.parse_args()

results=[]
for line in open(args[0]):
    if opt.filter in line:        
        fields=line.split()
        data={}
        timestamp=' '.join(fields[:3])
        data['datetime']=parse(timestamp)
        for field in fields:
            if field.startswith('DPT'):
                data['port']=int(field.replace('DPT=',''))
            if field.startswith('SRC'):
                data['src']=field.replace('SRC=','')
        results.append(data)
if opt.sort:
    results.sort(key=lambda x: tuple([x.get(sort_type) for sort_type in opt.sort]))
for data in results[-opt.num:]:
    print('%s SRC=%s DPT=%s'%(data['datetime'].strftime('%b %d %H:%M:%S'),
                              data.get('src'),data.get('port')))

[/PHP]
To print all the hits in /var/log/messages:
```

print_network_messages.py -f 'Inbound' /var/log/messages
```

"-f 'Inbound'" tells print_network_messages.py to filter out all lines unless the line contains the string 'Inbound'. 

To print just the last 4 hits in /var/log/messages:```


print_network_messages.py -n4 -f 'Inbound' /var/log/messages
```

To save hits to a file called output:
```

print_network_messages.py -f 'Inbound' /var/log/messages > output

```
To print hits in /var/log/messages sorted by src ip address:
```

print_network_messages.py -f 'Inbound' -s src /var/log/messages 
```

To print hits in output sorted by src ip address:
```

print_network_messages.py -s src output 
```

Notice the "-f 'Inbound'" is dropped because lines in output need not be filtered.

To print hits in output sorted by port:
```

print_network_messages.py -s port output 
```

To print hits in output sorted by port first, then src:
```

print_network_messages.py -s port -s src output 
```

To print hits in output sorted by src first, then datetime:
```

print_network_messages.py -s src -s datetime output 
```

And so on. You can use "-s src", "-s datetime", "-s port" to sort by source ip adddress, datetime stamp, port, respectively, and you can chain them together to sort by one field and break ties with the next field.

```

print_network_messages.py -h
```
prints a help message with all the options.
