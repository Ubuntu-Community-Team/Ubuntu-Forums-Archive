### Post by Zorael on 2008-04-29
The bootloader (grub) lists all the installed kernels, so likely you installed a kernel upgrade and now you have the new one and the old one available to boot. You can uninstall that package if you note down the version number.
```
$ sudo aptitude purge linux-image-2.6.xx-xx-xx
```

As for the screen resolution, this just might be due to a setting in the configuration file for X (the graphical interface). Try entering this command and see if it outputs anything.
```
$ grep -i virtual /etc/X11/xorg.conf
```
If it says something like 'Option VirtualSize *<resolution higher than the one you should have*', open /etc/X11/xorg.conf up in an editor with superuser rights, find that line and comment it by prepending a hashmark to it (#).
```
$ gksu gedit /etc/X11/xorg.conf         # Gnome
$ kdesu "kate /etc/X11/xorg.conf"       # KDE
$ gksu mousepad /etc/X11/xorg.conf      # Xfce
$ sudo nano /etc/X11/xorg.conf          # terminal
```

### Post by Zorael on 2008-04-29
As with most upgraders, your xorg is a mess. :>

Anyway, let's try rebuilding it. (prepended # means performed as root)
```
$ sudo su
# mv /etc/X11/xorg.conf /etc/X11/xorg.backup0804
# dpkg-reconfigure xserver-xorg
# nvidia-xconfig -d 24 --allow-glx-with-composite --add-argb-glx-visuals --randr-rotation
# gedit /etc/X11/xorg.conf &
```
Gedit will open your new xorg.conf

Add your mouse settings from the old one to the "Configured Mouse" InputDevice section. It may have a different title than Configured Mouse, but don't change it. Just paste the settings in there.
```

	Option		"ZAxisMapping" "4 5"
	Option		"Emulate3Buttons" "true"
	Option		"Buttons" "7"
	Option		"ButtonMapping" "1 2 3 6 7"
```
Lastly, do the same with your refresh rate ranges in your "Monitor" Section.
```
	Horizsync 30-81
	Vertrefresh 56-75
```
Just add them if they're not there.

Restart X with Ctrl+Alt+Backspace, log in, and see if it works. If it doesn't, we'll want to see your new xorg.conf and your /var/log/Xorg.0.log files.

### Post by Zorael on 2008-04-29
Almost forgot. Parsing the directory listing of your /boot, you do have two kernels builds; the xxx-xxx-xxx aren't the same.
```
vmlinuz-2.6.24-16-generic
initrd.img-2.6.24-16-generic (initrd.img-2.6.24-16-generic.bak)
abi-2.6.24-16-generic
config-2.6.24-16-generic
System.map-2.6.24-16-generic

vmlinuz-2.6.22-14-generic
initrd.img-2.6.22-14-generic (initrd.img-2.6.22-14-generic.bak)
abi-2.6.22-14-generic
config-2.6.22-14-generic
System.map-2.6.22-14-generic
```
See? 2.6.24-16 vs 2.6.22-14.

So to get rid of the older one, do this.
```
$ sudo aptitude purge linux-image-**2.6.22-14**
```
Or just remove the package in Synaptic. Should work.
