### Post by nc_jed on 2010-02-06
Ok, I read a post a while back about a guy that had an older machine and wanted to get some information about installing an older version of the OS (I think it was Ubuntu, but maybe not).  The reply was along the lines of "Why would you want to install a downlevel/older OS?"  His reasoning (as is mine) is that that the newer versions of the OS require more system resources than the older ones.  I agree this is true to a degree, but how much of this is the Windows mentality (newer releases are primarily meant to drive HW upgrade sales</tinfoil>) and how much is not?  I think of Ubuntu and Linux as providing new functions and features by default, but providing a graceful degradation if unwanted.

In short, all things (*cough* packages) being equal, how different will the Ubuntu 9.04 experience be on a 1 GHz machine versus a Ubuntu 7.04 install on the same machine?  I realize about compositing, etc (I don't even use that on my good machine) but what about for the basic vanilla install?

I guess that depends on what you want from your Linux experience. The official recommended specs for Ubuntu 9.10 is a 700MHz processor with 384MB of RAM. If you put Ubuntu on a machine like that and plan to run modern apps your going to be in for a painful experience.

Linux can be installed on older hardware. But you have to stick with lighter distros (Slackware, for instance), lighter DE/WM's,  and be careful what packages you install. Because many packages will load dependencies up that will slow down your whole computing experience.
