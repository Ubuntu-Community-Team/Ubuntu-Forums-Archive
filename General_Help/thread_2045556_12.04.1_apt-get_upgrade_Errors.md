### Post by mikechant on 2012-08-22
Please post the output of the command

```
fdisk -l
```

(Note that the character after the - is a lower case L not a number one!).

This will show what else is on /dev/sdc and how the disc is arranged so we can see if /boot can be extended easily.


Also, note that you could free up space on /boot by deleting old unused linux kernels and related files with one of the package manager tools. You typically only need to keep the current and previous kernels, anything older can go.

### Post by mikechant on 2012-08-22
If you still want to expand the boot partition, you need to boot from a live CD/USB, then use Gparted to:

a) Shrink /dev/sdc5 and move it to the right so there is free space before it (and no free space after it).
b) Move the start of the extended partition (/dev/sdc2) to the right to match the start of /dev/sdc5.
c) Expand the boot partition /dev/sdc1 into the space you created.

Make sure you back up anything important first.

500Mb should be plenty.

However, if you keep the number of kernels installed small you should be fine with the current /boot size.
