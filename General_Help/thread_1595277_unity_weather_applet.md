### Post by morsedl on 2011-12-21

I wish they'd stop redoing everything and just fix/improve what has been in existence for years already.

Yeah, I often feel the same way.  I'm constantly finding little things in Unity where functionality has been scaled back to barebone basics, and it can be very annoying and frustrating.

That said, I think Unity is worth the investment.  They're re-inventing the core of the user interface (UI) experience to ultimately make it all much, much better, in particular to make the experience much more consistent, no matter what program / app / applet is running.  Do this, they have to "break" a lot of existing software.  This is because they are re-implementing the core UI API with a better, richer, and more future-proof one (in this context, API = what programmers use to interface with the operating system).  They are borrowing a lot from good UI design in other platforms, most notably iOS (i.e., iPhone, iPad, etc.).

It will take a little time (e.g., a year or two) for app developers to catch up with what the Unity developers are doing, but the longer-term benefits should be quite significant.  Managing apps, indicators, notifications, and user and system settings should all start to become much more intuitive and consistent (Mac OSX is a great example of this).  Integration across apps should become much easier to implement and use (iOS is a great example of this).  Creating and using themes and skins will be a lot easier and "just work", and ultimately changing around most of the UI experience on a whim should become nearly trivial.  Yet, at the heart of things, it will still be the same Unix/Linux we value, and advanced users and experts will be able to get under the hood as always, ideally finding better, cleaner, simpler ways of dealing with things when it comes to the UI components.

Anyway, just my $0.02. :)
