### Post by karissa on 2006-03-09
Hello,

I have been trying (for hours now!) to install Ubuntu onto a Dell Precision 670 that currently has Windows XP Pro-64 installed.  I've downloaded the .iso onto a CD, checked the MD5 sums, changed my BIOS settings to check the CD-ROM drive first when booting up, but XP still boots up every time.  I've also burned Ubuntu onto a second CD, just in case the disk was the problem.

I've looked through the forum a bit, and it seems like some people with a similar problem were able to solve it by using a floppy disk, but unfortunately I don't have a floppy drive, so that option is out.

Is there any chance that it has something to do with the brand of disks I'm using?  And, apologies for my naivete here, but my processor is: 
"64-bit Intel  Xeon  processors: Dual-core 2.80 GHz 800MHz FSB with 2x 2MB L2 cache"
--I'm assuming that the AMD64 download is the appropriate one?

Thanks in advance for any input at all!

### Post by bluevoodoo1 on 2006-03-09
[QUOTE=karissa] I've downloaded the .iso onto a CD[/QUOTE]

this might be a silly question but... when you burned the .iso file did you burn it as a CD image file? What cd burning software are you using? You'll have to look around for an option to burn a "CD image" or an option to make it bootable. Burning it as just a regular data CD won't work. 

Once that's worked out [http://users.bigpond.net.au/hermanzone/](http://users.bigpond.net.au/hermanzone/) is a great link for dual booting.

### Post by karissa on 2006-03-09
Thank you both!

bluevoodoo1 was in fact correct (and I'm enough of a newbie that there's little danger of silly questions!)  I was using Roxio to burn the CD, and although "type" was listed as "CD Imaging File,"  the problem was solved when I downloaded ISO Recorder and recorded the file that way.  Ubuntu is installing at this very moment. :)
