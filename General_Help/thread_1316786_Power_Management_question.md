### Post by Trabb's Boy on 2009-11-06
Hi Everyone,

I'm running Ubuntu 9.10 x64 on my recently upgraded system.  It is an AMD Phenom II 720 CPU in an AsRock K10N78M motherboard.  Ubuntu detects and sets up everything on this board correctly, but I am having a problem with the Power Management functionality in Gnome.  In Preferences-->Power Management, I set the "put display to sleep" option to, say, 10 minutes, and that works fine.  But when I set "put computer to sleep" to any amount of time other than "Never", I have a problem.  The computer goes to sleep alright - The Big Sleep!  It shuts off completely.  What's more, when I power it back on, it doesn't even boot up.  The fans and drives spin up, but no boot.  I have to power it back off, then re-power on and this time it will boot up normally.  I get this behavior whehter I select the "spin down hard disk when possible" option in Power Management or not.  I do have the Cool and Quiet option enabled in the BIOS.  This is a dual-boot system with Windows 7 x64.  All the sleep stuff works fine in Windows, but I spend most of my time in Ubuntu, so I would like to get it working there.  This was a default install from the Live CD.  I see, in Synaptic, that I do have the "powermgmt-utils" package installed.  But is there maybe some additional package(s) that I may need?  Or some BIOS settings that need to be tweaked.  If anybody has any experience with this problem, I would love to hear your thoughts.  I am new to Ubuntu, so I don't know if this was something that might have worked in previous versions.  Apart from this issue, I am loving Ubuntu so far.  I was a SuSE guy for years, but I may have found a new distro in Ubuntu.  Thanks in advance for your help.
