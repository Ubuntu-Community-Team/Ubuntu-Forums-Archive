### Post by rac9876 on 2008-04-27
I'm using Xubuntu 8.04 (Hardy), upgraded from 7.10 (Gutsy) over the internet. I had the proprietary NVIDIA driver working fine in 7.10 for my nvidia GeForce 2MX (it's an old PC) but when I upgraded to 8.04 the login screen was so large it wouldn't fit on the monitor.

I tried fiddling with various things but with no luck. When I ran *dpkg-reconfigure xserver.xorg *(which worked under 7.10) I found it errored out after asking about keyboard and mouse, and never even got to the display settings. I have seen others saying that this command is broken under Hardy.

I ended up stuck with Vesa graphics and 800x600.

I then tried displayconfig-gtk, which made even more of a mess of everything.

Here's what worked:


[LIST=1]
[*]Open Synaptic and installed **envyng-gtk**. (I chose the gtk version because this is Xubuntu.)
[*]Run **Applications > System > EnvyNG**.
[*]Choose NVIDIA and "Install NVIDIA driver (Automatic Hardware Detection)".
[*]Let EnvyNG install and configure the driver.
[*]Go to **Applications > System > NVIDIA X Server Settings**. (This should have been installed by EnvyNG in the last step.)
[*]Under "X Server Display Configuration" you can choose your resolution and refresh rate - it should be OK to use the Basic settings though you can click Advanced for more control. (If you have more than one monitor attached you should be able to configure that here too, but I haven't tried that.)
[*]Click Apply to change resolution on the fly and check your settings.
[*]To persist the settings, first click "Save to X Configuration File"
[*]Deselect "Merge with existing file".
[*]I saved the settings to a new **xorg.conf** file in my Home folder since under my user account I didn't have permission to overwrite the one in /etc/X11.
[*]I opened up a Terminal window and typed **cd && sudo cp xorg.conf /etc/X11** to overwrite my old xorg.conf. You might want to back  up the old one first!
[/LIST]
At this point my login window was looking the right size but the resolution used by my monitor was higher than I would have liked - it was always selecting the "best" resolution available for the given colour depth. (The monitor is a CRT that goes up to some crazy resolution but becomes unpleasant at more than 1280x1024@85Hz.) So I went into xorg.conf and added the line in red below:

[FONT=Courier New]Section "Screen"
    Identifier     "Screen0"
    Device         "Videocard0"
    Monitor        "Monitor0"
    DefaultDepth    24
    Option         "TwinView" "0"
    Option         "TwinViewXineramaInfoOrder" "CRT-0"
    Option         "metamodes" "1280x1024 +0+0; 1280x1024@85 +0+0"
    SubSection     "Display"
        Depth       24
       [COLOR=Red]Modes      "1280x1024, 1024x768, 800x600"[/COLOR]
    EndSubSection
EndSection[/FONT]

I now have a working login screen and a desktop at 1280x1024, 85Hz.

I'm relatively new to this and can't claim to have a thorough grasp of xorg.conf, so I won't attempt to actually explain the above. It just had the right effect.
