### Post by Kirk Schnable on 2017-12-04
You could create the file /etc/default/locale, then it would exist and no longer encounter this error.

On my system, that file contains this text:
```
[COLOR=#000000]#  File generated by update-locale
[/COLOR][COLOR=#000000]LANG="en_US.UTF-8"[/COLOR]
```
