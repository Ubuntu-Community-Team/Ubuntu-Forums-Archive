### Post by Ayuthia on 2008-12-09
It all depends on why you want to recompile a kernel.  Some people like to compile a kernel so that they can speed up their system.  It might speed up the boot process, but for most applications, there is not too much of a speed gain.  The other downside to this is that if you have an Nvidia graphics card (or some other proprietary graphics card), you will have to compile the driver manually so that it will work with your kernel.

There are also people who like to compile kernels so that they can understand their system a little better.  You have to know a little bit about your hardware to compile a kernel.  If you don't, you might miss a module or even create a kernel that will not boot.

Also, there are people who compile kernels because they have newer hardware and would like to use the newer kernel that contains a new/updated module that will make their hardware work.

For the most part, a normal user does not really need to compile their own kernel.  I would think that you really don't need to do this unless you want to learn more about how your system works.
