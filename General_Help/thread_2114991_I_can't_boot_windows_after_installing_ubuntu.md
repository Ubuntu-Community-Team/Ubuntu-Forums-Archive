### Post by darkod on 2013-02-11
You somehow installed grub1 (not even grub2) on the partition where the win7 boot files are, sda1:
```
sda1: __________________________________________________________________________

    File system:       
    Boot sector type:  Grub Legacy
    Boot sector info:  Grub Legacy (v0.97) is installed in the boot sector 
                       of sda1 and looks at sector 1878595584 of the same 
                       hard drive for the stage2 file.  A stage2 file is at 
                       this location on /dev/sda.  Stage2 looks on partition 
                       #5 for /boot/grub/menu.lst.
    Mounting failed:   mount: unknown filesystem type ''
```

Use this procedure either from the installed ubuntu or from live mode to restore the backup boot sector of sda1:
[http://sourceforge.net/apps/mediawiki/bootinfoscript/index.php?title=Boot_Problems:Boot_Sector](http://sourceforge.net/apps/mediawiki/bootinfoscript/index.php?title=Boot_Problems:Boot_Sector)

PS. Yes, you can also use the win7 repair process if you have the dvd or a rescue cd. But note that that will delete grub2 from the MBR also, so you will have to restore that too.

And related to earlier command mentioned grub-install /dev/sda1, NEVER, ever install grub2 to a partition (and /dev/sda1 is a partition). You need it on a partition only in special cases, very rare. The boot process starts on the hdd MBR and that's where a bootloader needs to be, not on a partition.
