### Post by drs305 on 2010-10-10
It looks like your W7 partition is missing some of the files needed to boot. Grub probably isn't picking up W7 because it isn't finding the necessary boot files where it expects them. 

Compare the RESULTS.txt entry for sda2 with this entry from a working W7 menuentry:
> 
sda1: _________________________________________________________________________

    File system:       ntfs
    Boot sector type:  Windows Vista/7
    Boot sector info:  No errors found in the Boot Parameter Block.
    Operating System:  Windows 7
    Boot files/dirs:   **/bootmgr /Boot/BCD** /Windows/System32/winload.exe


I'll leave it to the Windows guys to tell you how to repair it, but this link may help out:
[How to restore the Ubuntu/XP/Vista/7 bootloader]("http://ubuntuforums.org/showthread.php?t=1014708")

### Post by Mark Phelps on 2010-10-10
You'll need a Win7 Repair CD.  Did you bother to create one when you were running Win7 by lauching the Backup feature and using the option to create and burn a Win7 Repair CD?

Don't bother to answer that ... no-one does that.

So, you will have to go to the link below, download and burn the appropriate Win7 Repair CD:

[http://neosmart.net/blog/2009/windows-7-system-repair-discs/]("http://neosmart.net/blog/2009/windows-7-system-repair-discs/")

Then, you will need to boot from it and run  Startup Repair three times -- that's three times!  That should rewrite the Win7 boot loader files to the proper partition.

That will, of course, overwrite GRUB, so afterward, you will have to boot into an Ubuntu LiveCD and rewrite GRUB to the MBR.
