Wondering if I can get some suggestions of Usenet readers to try.

Read some of the other threads but my preferences/requirements are different than what other people seem to be looking for.

I'm trying to find a good text-only (console) newsreader. There seem to be quite a few, but I'm not clear on the relative merits of each. I'm looking for something that I can use in an SSH session, remotely. I don't care about binaries. (At all. I only read text groups, so I don't care about the ability to save files, view graphics, batch download, or anything else like that. I just want a good text reader / poster.)

I'm also trying to find something that doesn't download the full text of every article in a group when I open it. I've played around with tin, and it seems to do this, and I can't figure out any way to turn it off. (Example: when I go into a high-traffic group that I haven't been to in a while, so there are a few hundred or thousand unread messages, tin downloads ALL the unread messages to my machine, which takes a long time and uses up a lot of disk space.

I don't want this ... I'd really prefer it only download the headers, because those don't count towards my ISPs quota, and then grab individual posts when I go to read them. (I usually skim the headers and only read 10-20% of the 'unread' articles anyway.) Aside from that, which is a real deal-breaker, I'd probably be happy with tin.

Can anyone familiar with slrn, nn, rn, or the other console-based readers weigh in on how the other programs compare?

Thanks very much!
