### Post by Thomeduk on 2009-02-19
I had ver 7.4 happily co-existing with WINXP on my machine which I updated (Gigabyte GA-EP45-DS3+4 SATA HDD=SATA DVD+IDE CD-RW+4GB RAM).  First problem was while WINXP 'exported' happily to the upgraded machine UBUNTU ver 7.4 did not.  I got the 'Busy Box, initramsfs problem. I eventually found that by disconnecting the IDE CD-RW drive and substituting another SATA DVD drive I could overcome the 'Busy box' problem and I upgraded to UBUNTU V8.10.  Bad mistake as it turned out. 

While version 8.10 installed I could not get it to allow printing from either my HP Business Inkjet 2280 (hard wired connection via Ethernet) or my HP Deskjet 6980 (wireless connection).  Both printers have been installed every which way, that is via 'Cups' via the HPLID (I think that is what it is) utility and simply via the 'Printing' utility (excuse the terminology as I am by no means a UBUNTU expert).  In each case the printers appeared to have installed (I used both the ppd files included in the distro and the ppd downloaded from HP).  Neither printer will print.  The HP 6980 (wireless) simply does nothing and yes I have verified the IP address from a direct print out from the printer.  The HP 2280 will print but prints a bunch of 'rubbish', not what is sent to the printer from the machine.

I have now spent two days trying to get UBUNTU V8.10 running, that is to recognize the printers but have given up.  WINXP is a breeze to configure and use compared to this!

As an additional piece of info that may be of use to others.  I tried to boot my machine from a bootable CD with GPARTED V0.3.4-10.  This CD booted my pre upgraded machine without problems.  However, with my upgraded machine it wouldn't and I ended up with the dreaded 'Busy box'.  I Downloaded and burned a new CD using the latest GPARTED iso (ver 0.4) and this booted my new machine without incident. 

I've had enough.  UBUNTU is supposed to be easy to install and configure and I found that to be the case with ver 7.4 but it certainly isn't for v 8.10.  I suspect the problem may relate to the machine, that is the hardware is to recent.  It is also possible that the printer 'problem' can be overcome if I connected the printers directly to the machine rather than via Ethernet and wireless but that is not the configuration I need.
