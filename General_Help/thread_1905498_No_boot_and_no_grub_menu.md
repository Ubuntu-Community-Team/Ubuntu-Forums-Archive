### Post by oldfred on 2012-01-07
Boot script looks normal to me.

Are you holding shift down from BIOS until menu appears?

Often cursor in left corner is the start of booting, have you let it wait for a  while to see if it does error out or even boot.

If forcing a shutdown, do this.

Holding down Alt and SysRq (which is the Print Screen key) while slowly typing REISUB
R-E-I-S-U-B to force shutdown
A good way to remember it is.
Raising Skinny Elephants Is Utterly Boring ...or
Reboot System Even If Ultimately Broken ...LOL.
[http://kember.net/articles/reisub-the-gentle-linux-restart/](http://kember.net/articles/reisub-the-gentle-linux-restart/)

Did you uninstall any software, or have an update that changed things, since last working reboot?

### Post by rsvancara on 2012-01-07
If you are interested in repairing the system please attempt following.

1.  Is this a 64 bit or 32 bit system?

2.  Once you have identified which system it is, please download the appropriate Ubuntu live CD release, x86 for 32 bit and x86_64 for 64 bit.  

3.  Place the CD into your CD ROM drive and boot off the live CD image.  

4.  Once the live CD is up and you have networking, open a terminal and type:

```
sudo add-apt-repository ppa:yannubuntu/boot-repair && sudo apt-get update
sudo apt-get install -y boot-repair && boot-repair
```

5.  Now follow the instructions on this page

[https://help.ubuntu.com/community/Boot-Repair]("https://help.ubuntu.com/community/Boot-Repair")

This should be fairly straight forward if your disk is in good shape and all you have done is broke your GRUB installation.  

If you are stilling having errors, 

Copy and paste the output of

dmesg |grep sd to the forum so we can see what is going on with your disks.
