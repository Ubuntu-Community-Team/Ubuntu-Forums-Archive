### Post by az on 2006-01-30
A tgz  may sometimes just be a tarball and sometimes it is a slackware package.  You would have to give more information.

If it is a slackware package, you have to realise that it is built on a system that has things organised differently than a debian (ubuntu) system - installing it will probably not work.  You should try to install it from the source to have a better chance of getting it to work.

Whether you get the cd by downloading it or by shippit, they are identical - provided the one you burned yourself is not corrupt.  You can make the cd check itself by booting it and scrolling down to the bottom of hte menu and selecting "check cd integrity"
