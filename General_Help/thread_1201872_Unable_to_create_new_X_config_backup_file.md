### Post by merlinus on 2009-07-03
Quite possibly.  What may help is to edit the grub booting line and remove quiet and splash.  That way the loading processes will scroll by on your monitor.

Press e at the grub menu, which should take you to the bootup line.  Press e again, delete quiet and splash from the end, and press b to boot.  This is not a permanent change, only for this one instance.

### Post by merlinus on 2009-07-03
You can edit menu.lst

```
gksudo gedit /boot/grub/menu.lst
```and either change the timeout

## timeout sec
# Set a timeout, in SEC seconds, before automatically booting the default entry
# (normally the first entry defined).
timeout        10

or delete quiet and splash from the end of the bootup line that will look something like this:

kernel        /boot/vmlinuz-2.6.28-13-generic root=UUID=8268d051-ae3a-4d8b-9c21-ddf82d782734 ro quiet splash

### Post by giancast on 2010-02-14
