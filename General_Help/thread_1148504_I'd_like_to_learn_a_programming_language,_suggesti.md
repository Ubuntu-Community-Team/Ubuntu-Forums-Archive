### Post by codeseer on 2009-05-04
C# with Mono.

You don't have to deal with the quirks of C/C++ nor garbage collection.

I was a major C++ supporter until about 5-6 years ago; when I switched it felt like the difference between sandpaper and silk. I haven't really looked back since. The only thing you should be aware of is that a decent bit of the 'older generation' is still holding onto C/C++ and it's used a great deal in open source projects; however, if you learn C#, you will be able to make out what is being done in C++ for the main part.

It is a small learning curve. If you're willing, just learn both and use C/C++ where you have to for getting along with the older generation and use C# where you want insane speed of development without having to worry about where that memory leak is coming from in your 500,000 lines of code. That's another thing I've come to enjoy; the amount of code needed to perform everyday tasks is reduced in C#.
