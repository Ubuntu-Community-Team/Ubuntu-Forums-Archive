### Post by plentyTypos on 2010-06-10
Hi all. My first post, if I do it wrong please forgive (but tell me).

Here is an odd thing I ran into when installing Lucid 64 bit on a Sony Vaio VPCF116FX; a dumb error I admit, but significant insofar as there was no obvious feedback telling me that things weren't working:

I had a working connection between the computer and the DSL modem (and the little icon in the Ubuntu menu bar showed that), but of course without having entered settings required by the ISP there was actually no working internet connection. Meaning that in spite of the fact that there seemed to be constant download/install activity, in fact what was happening was that the installer was just timing out with each download try, and that wasn't obvious until much later.

If you first "test drive" Ubuntu and then directly from there start the installation, after entering necessary ISP settings and confirming that you actually have an internet connection, then it should all go as expected. Pretty obvious, but it sort of had me confused for a while.

By way of background I should explain that I'm switching not from Windows (which I've literally never used and so didn't know anything about either) but from Mac OSX. Perhaps only Mac users can identify with this problem; we have long years of habits that say that when you want to install an OS you begin by restarting the computer, booting from the install CD, then doing the install; later you do "Software Update", only after the OS has been installed.
