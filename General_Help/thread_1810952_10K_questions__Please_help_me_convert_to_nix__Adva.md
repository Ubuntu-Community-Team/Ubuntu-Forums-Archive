### Post by -kg- on 2011-07-24
Wow!  A lot of questions!  I can tell you work in IT.  :P

I have no idea on a number of them, so I will suggest a couple of first steps.  I would download or otherwise obtain a good .iso of Ubuntu/Kubuntu/Xubuntu...whichever is your cup of tea.  If you want the latest and greatest, make that 11.04...if you want stability and longevity, go for the 10.04 LTS (Long Term Support) release.  Support for the LTS goes around a year longer than 11.04.

I will assume that your computer(s) will boot to a USB device...if so, burn the iso to a flash drive and boot to it.  If your computer won't boot to a USB device (2 months old?  Probably won't be an issue), then burn the iso to a CD-R and boot to that.

While booted to the Flash drive/CD-R, check all the functionality with your computer.  Make sure it will run everything and works well with your hardware.  If it works acceptably with your hardware, you can install it and run Windows in a VM (I have VirtualBox, but there are others).  If you want Windows as your base installation, then follow these further instructions.

You're having trouble deleting your Windows partition(s)?  While booted to the Ubuntu Live CD, launch GParted.  You should be able to delete all the desired partitions from there, and you should be good to go for a re-installation.  I don't know why your Windows CD won't do all that for you, but GParted should have no problem.

I don't know where GParted can be found in 11.04 Ubuntu...in Kubuntu it can be found under the "Launch (similar to the Start button in Windows)/Applications/System/Partition Editor" and under 10.04 (the standard 2.x Gnome Desktop) it can be found under "System/Administration/Gparted".  Perhaps someone will come along and be able to tell you where to find GParted under 11.04, if you choose to go that route.

Most of the other things you asked about I'm really not that strong on.  Windows games will either have to be run under Wine or under Windows, whether through a VM under Ubuntu, if you choose to install Ubuntu rather than Windows as your primary installation.

Of course, you can also dual boot Windows and Ubuntu side by side, with both of them installed in separate partitions.  Then you can have Windows for your games and anything you just ***must*** do in Windows, and have Ubuntu for surfing the web and anything you can do in Linux instead of Windows.

You're like me...I've been into computers nearly 30 years, before Windows existed.  The learning curve is large, but not insurmountable.  It took me quite a while to change from "Windows-think" to "Linux-think", but now I find it increasingly hard (and greatly undesirable) to revert back.

You have an advantage (and a disadvantage, at times)...you're computer literate.  I seriously don't know what books to suggest.  I learned most of what I know by reading on the 'net and cruising the Ubuntu Forum, coupled with what I already knew.  But right now we need to get your computers up.  The rest will come with time.

About that "critical update" that toasted your Windows installation...have you found out whether it was another typical MS screw-up (I've had a few of those...nothing that critical, though) or was it perhaps a corrupted download?

```And then check in Applications --> System Tools --> System Profiler and Benchmark. It's the closest thing to a Device Manager in Linux.
### Post by coldraven on 2011-07-25
If you are going to use VirtualBox then download the latest version  (4.1) from their web-site. After the install add yourself to the  "vboxusers" group in System>Admin>Users and groups. Reboot or  log-out and back in. (USB won't work if you don't do this)
Download the VirtualBox extension pack and double-click it to install.
To avoid the XP activation nightmare read carefully the following info.
I have copied an old post of mine below to save you hours of head-scratching!

My HP laptop had XP preloaded and came with a restore disc, that's the OEM sort which does not have a serial number on it.
 The problem you get is that when it installs it looks for some data in the BIOS which generate the activation key.
 If you just install in a fresh VirtualBox the activation will fail because VBox presents a fake BIOS to the guest OS.
 However, you can get the info from your BIOS and configure VBox to present this instead.
 Read this [http://www.virtualbox.org/manual/ch09.html#changedmi](http://www.virtualbox.org/manual/ch09.html#changedmi)
 Then with the info you got from dmidecode edit this script. My VM is  called xp01, edit to suit yours. Save the script as "vmbios" and make it  executable by right click, select "Properties" and change the  Permissions.
    
     ```
#! /bin/bash
VM_NAME="xp01" # Name of your Virtual Machine 
VSETED="VBoxManage setextradata 
$VM_NAME" CFG_PATH="VBoxInternal/Devices/pcbios/0/Config" 
$VSETED $CFG_PATH/DmiBIOSVendor       "Hewlett-Packard" 
$VSETED $CFG_PATH/DmiBIOSVersion      "F.0C" 
$VSETED $CFG_PATH/DmiBIOSReleaseDate  "06/05/2008" 
$VSETED $CFG_PATH/DmiBIOSReleaseMajor  "15" 
$VSETED $CFG_PATH/DmiBIOSReleaseMinor  "12" 
$VSETED $CFG_PATH/DmiBIOSFirmwareMajor "113" 
$VSETED $CFG_PATH/DmiBIOSFirmwareMinor "45" 
$VSETED $CFG_PATH/DmiSystemVendor     "Hewlett-Packard" 
$VSETED $CFG_PATH/DmiSystemProduct    "HP Compaq 6715b" 
$VSETED $CFG_PATH/DmiSystemVersion    "<RK154AV>" 
$VSETED $CFG_PATH/DmiSystemSerial     "XXXXXXXXXX" 
$VSETED $CFG_PATH/DmiSystemUuid       "XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX" 
$VSETED $CFG_PATH/DmiSystemFamily     "103C_5336AN" 
```**[COLOR=Red]Run this script after making your virtual disc but before you  install XP.[/COLOR]** This script was on the VBox forum (named VMbios), I've forgotten the  poster's name but thanks anyway
 This works for XP, I have no idea about Win7.
When your virtual XP is working make a backup by exporting a copy somewhere safe.
If you update VirtualBox you will probably have to run the script again before you try to start your XP VM.
