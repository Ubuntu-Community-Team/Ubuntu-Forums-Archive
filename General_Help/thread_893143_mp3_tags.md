This is not really a problem per se, but has anyone ever noticed that in the comments tag of mp3 files in your collection... after you have used your library for a while, develop strange text strings like:

000004B1 000004D8 00001139 00001167 00027145 00027145 0000631B 000066EC 000222E0 000222E0

Where do these come from? I have noticed it in many files. It occurs in Ubuntu, Win XP, Win Vista, etc. using programs like Music Match, Windows Media Player, Media Monkey and Amarok so it is not related to player or OS. I have deleted the text strings and they always seem to reappear. I have a huge collection (20k songs+) and sometimes don't get around to playing the same thing for a while, but they always appear regardless of platform or player.

I always leave the comment tag blank on my files so its not me putting them in there. They reappear even after I delete them. I have wondered about this for a long time. It does not affect the music in any way, I am just curious about it.

### Post by mcduck on 2008-08-18
At least iTunes add data like that to the comment field if you enable the SoundCheck (the tool that makes sure all the files wil play at the same volume. The piece of code tells what the volume level of that file is).

Of course any other library-based player could do the same thing if you use any auto-gain features.
