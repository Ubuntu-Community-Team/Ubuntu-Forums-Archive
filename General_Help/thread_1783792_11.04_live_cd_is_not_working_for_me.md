### Post by mastablasta on 2011-06-16
ah ok no alternate install CD needed then.
 
well on boot up there is a menu and on bottom there is one key that opens a line where you can add commands that are executed at boot. used to be F6... and then you just type in nomodeset.
 
th eissue is probabyl because in liveCd the X-windows system or graphics card is not porpperly configured. while upon install everything is configured propperly. have seen it happen on my system wiht sound but ion reverse way (sound worked live but not on install). when you boot live system loads opensource drivers first. it may also be that they don't support your card. these additonal commands can set it for example to basic vesa mode (like repair mode in windows). so you can later install proprietary drivers via graphics interface. 
 
and i know that in OpenSuse (though i think this shoudl be only their bug) ATI cards cant' boot without nomodeset command. [http://wiki.sugarlabs.org/go/Nomodeset](http://wiki.sugarlabs.org/go/Nomodeset)
