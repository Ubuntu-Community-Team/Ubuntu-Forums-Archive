### Post by nevius on 2010-10-22
If he uses outlook for his email client, then his emails and contacts are stored in a .pst file. On a default install it is in a hidden directory located in "C:\Documents and Settings\User Profile\Local Settings\Application Data\Microsoft\Outlook\". To be on the safe side, you may want to copy the entire contents of his user profile.

You should consider the following:
1) **Take your time to do this right.** The more research you do before hand, the smoother the transition will be. Don't try to start from scratch and do it overnight. Bad things will most likely happen if you do that.

2) **Talk to him about what programs he uses.** *Write them down.* Make sure that you research issues regarding switching from those applications to equivalent ones under linux. Also research compatibility of the files in his old windows application and the linux applications.

3) **Back up his files.** Preferences and files are strewn all over the Windows XP file system. It is hard to make sure that you get everything. Go through the list you made in (2) and make sure that you get everything. See the above example with Microsoft outlook. You should also consider imaging his hard drive (if you have room on an external drive). If you image it then you will always be able to get his OS back if he decides he doesn't like Linux.

4) **Configure Ubuntu for him**. Test all of the programs and hardware to make sure that they work. Set Ubuntu to automatically install security updates. If you have a label maker - label his computer with his password so that he won't have to ask you repeatedly for it.

These are just some thoughts based on things I've had go wrong in migrating people to linux.

### Post by kalos on 2010-10-23
Hold on!

Some people mentioned cloning the drive and that's a very good idea. Especially if the machine is used for business. If you have an empty external drive that is big enough to hold everything on the computer's hard drive, then I'd recommend copying everything. The thing you need to back up the most is always the one you miss.

You can find a [tutorial for clonezilla]("http://linuxgravity.com/creating-and-restoring-an-image-of-hard-disk-with-clonzilla"). You'll want to clone your windows drive to an image on the external drive. In ubuntu, you should be able to double click on the output .img file to mount it and browse the files. You should check this from the LiveCD before you install.

Personally, I'm crazy and paranoid, so I'd copy all the My Documents stuff somewhere _and_ make a backup image.


(And if you can't figure out Clonezilla, at the very least you can fire up the Ubuntu LiveCD and run a copy:
```
mkdir /media/external_drive_path/windowsbackup
cp -rf /media/windows_drive_path /media/external_drive_path/windowsbackup
```
Although Clonezilla is probably a more reliable method.)

### Post by kumaryu on 2010-10-23
1. As suggested by a previous post, I would strongly recommend using something like Clonzilla Live [http://clonezilla.org/](http://clonezilla.org/) to copy the entire disk to an external hard disk before you start. 

This will allow you to restore the computer to its original XP installation when (not if) things go wrong. With only data backed up, it may be very difficult to restore all the accumulated apps and utilities to the way they were before they were wiped.


2. Why Maverick? More issues will be resolved for Lucid, which is a long-term support release (3 years) than will be for Maverick in its entire lifecycle (18 months).

3. Before you start, use something like HWiNFO32 [http://www.hwinfo.com/](http://www.hwinfo.com/) (its a Windows app) and note down the "real" hardware specs of the PC you are working on. Make certain you know the work-around (if any) to get the graphics hardware to work on Ubuntu. (a frequent gotcha is the Intel 855 series GPUs on laptops from 5-6 years ago) Even if the PC worked with the "Live" disk, you may end up with a blank screen after bootup otherwise.

### Post by kumaryu on 2010-10-23
It may not be complete bu here's a summary.

1. Install HWiNFO32 [http://www.hwinfo.com/](http://www.hwinfo.com/) and print out the hardware specs of the PC. 
This will allow you to troubleshoot easily during Ubuntu installation.

2. If Outlook is used: 
a. Export Contacts as: *.csv or *.vcf files. Backup to external hard disk.
b. Export Calendar data as *.icf/.ical. Backup to external hard disk.
c. Use Thunderbird to convert the mailbox files (*.pst files) to mbox format by importing them.
(Thunderbird .mbox files are located at:
C:\Documents and Settings\<Windows user name>\Application Data\Thunderbird\Profiles\<Profile name>)
This will allow you to choose from most of the common mail clients in Ubuntu. 

3. Back up the entire contents of 
C:\Documents and Settings\<Windows user name>
folder to an externally attached hard disk.
(Although this copies a lot more than is strictly necessary,  its usually much faster than trawling though the folders.)
Ask the owner if he/she keeps any other documents elsewhere and back them up (if any).

4. Note down Outlook POP/IMAP and SMTP settings.

5. Note down Wifi WEP/WPA key info (if any).

6. Use Clonzilla Live [http://clonezilla.org/](http://clonezilla.org/) to copy the entire disk to the  external hard disk. 
This step will allow you to restore the PC to the exact state it was in prior to installation of the new OS.

7. Install the new OS of choice.

8. Restore all files backed up.

9. If it all went pear-shape, use Clonezilla Live to restore to previous state.

### Post by kumaryu on 2010-10-23
Graphics Processing Unit

Its the chip on the motherboard or a dedicated "video" card that "draws" the images on the monitor. Lots of different ones are in use on different computers and each one requires a specific driver software  to do all but the simplest of tasks. 

A modern graphical user interface (GUI), such as those on Ubuntu (X11/Gnome) require the correct driver to be installed in order to utilise the GPU. Without it, you end up with a command-line interface, ie the  terminal screen.
