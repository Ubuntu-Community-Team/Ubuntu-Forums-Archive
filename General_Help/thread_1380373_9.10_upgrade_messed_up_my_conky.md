### Post by Brandon Williams on 2010-01-14
I'm not sure why it's moving over, or why it's behaving differently now than in the previous release. However, if you want it to stay in one place and to never change its width, try adding the variables minimum_size and maximum_width to your .conkyrc. Set them both to the same value. Experiment a bit to find a good value that provides the width that you want. After you find a good width, you might also want to try setting gap_x in the forecast config to that same value, so that the weather display will never try to overlap where the other conky output is displayed.

An alternate approach is to run a single instance of conky and write the .conkyrc so that it displays everything with the appropriate parts side-by-side. I don't know whether that will work, though since you've got bigger lines in the forecast display than in the other one.
