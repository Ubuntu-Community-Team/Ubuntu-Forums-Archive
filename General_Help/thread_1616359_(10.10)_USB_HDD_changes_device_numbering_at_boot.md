### Post by cro on 2010-11-08
I have an interesting issue that has occurred since upgrading to 10.10.

My desktop (x86_64) has two physical hard-drives installed, primary as /dev/sda, secondary as /dev/sdb, configured in fstab to be mounted in a subdirectory of my home folder, as well as at a root level (so ~/Workspace and /Workspace are equivalent)

The issue occurs now when I reboot with an external hard drive plugged into a USB port - the boot process detects /dev/sda as the first drive, then enumerates the USB-connected hard drives, before finding the second internal drive.

This is quite annoying since this second drive exists in FSTAB, and rebooting with a USB-attached HDD causes the mount process to fail throwing warnings everywhere.

This has onlt started happenning since the upgrade to 10.10 - prior to this (and through every upgrade from 8.04 onwards) I've never had any issues with the driver order being messed up on boot.

Any suggestions gratefully accepted - my workaround at the moment is the obvious one - don't reboot with any USB storage devices attached...

### Post by Verbeck on 2010-11-08
try using uuid in fstab, it wont change. to check the uuid :
sudo blkid -c /dev/null

so in fstab, it looks like[FONT=monospace]:
[/FONT]UUID=1713728c-b4e1-4ab4-9c27-481f8ba3a5ca /home ext4 defaults 0 2
