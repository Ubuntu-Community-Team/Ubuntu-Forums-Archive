### Post by SimonJW on 2005-12-11
Not that it's very helpful to say this, but a lot  of the development work currently going on in GNOME seems to be in the area of speed - most notably work on the login speed, and on the speed (and memory usage) of things like panel applets and GConf. There seems to be some real progress being made (that is, noticeable speed increases).

So I guess if you find it slow, but want to try it out as an alternative to KDE, wait one or two releases (release cycle is 6 months, I think) and try it out. Dapper should probably include a more speedy GNOME!

Saying that, that's quite a long wait! I'm sure some people have tips to speed it up a bit. Personally, I've never noticed a particular difference in speed in KDE and GNOME - the only time I find GNOME slower is when running KDE apps.

### Post by Iandefor on 2005-12-11
You could try Poofyhairguy's [Enlightened GNOME howto]("http://www.ubuntuforums.org/showthread.php?t=54476&highlight=Enlightened+GNOME+EYECANDY"); Metacity is a big resource-devouring monster, as is Nautilus. Get rid of one or both of those, and you've got a much lighter desktop. 
As replacements, you could try Enlightenment to replace Metacity, and Idesk to replace Nautilus as the desktop manager; I'd still use Nautilus as the file manager.
