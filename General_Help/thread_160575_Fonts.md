### Post by trotski on 2006-04-15
It might have something to do with GTK version 1.  It doesn't support fonts as well (or fully) as QT and GTK2.  That always bugged me in Gnome.  According to the xCHM website, it uses wxGTK.  Does Ubuntu not use GTK2-enabled wxGTK?  I guess it is a question of stability.  Mplayer doesn't use GTK2 by default, either.

XMMS may never support GTK2, so it will always look "ugly" compared to other programs where fonts are concerned.  There is always Beep Media Player (BMP) which was initially an XMMS fork that uses GTK2.
