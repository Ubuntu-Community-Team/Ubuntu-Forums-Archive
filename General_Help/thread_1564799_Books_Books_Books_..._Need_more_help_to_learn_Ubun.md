### Post by srs5694 on 2010-08-31
> **sim085 said:**
> I am interested in learning ;

 - best practices when installing and using linux.

There are lots of introductory books on Linux generally and Ubuntu in particular. I recommend you go to a well-stocked book store and browse a few. If you don't have good book stores in your area, Amazon lets you browse many books' contents.

> - how to set-up a network between multiple installations.
 - how to set-up a network between linux and other machines running different type of operating systems.

My own [*Linux in a Windows World*](http://www.rodsbooks.com/linwin/) covers much of this, particularly the latter. Note that "setting up a network" is a very broad topic. Most people only need a handful of servers running on their local networks, and many of these duties (DHCP, DNS, sometimes NTP) are performed by broadband routers. What's left is likely to include file sharing servers (Samba and/or NFS), printer sharing servers (Samba and/or CUPS), possibly a Web server (Apache or something slimmer like thttpd), perhaps a time server (NTP), and maybe some other more specific items.

 > - how to set-up machine and network security.

There are Linux security books, but most individuals are well enough served by the security sections in more general Linux or networking books. Linux security books are usually written for professional system administrators. Of course, if you're interested enough, such a book could be worth a read.

 > - how to handle hardware incompatibilities.
 
This is a topic that doesn't get a lot of coverage in Linux books, although introductory books sometimes have some scattered information on it. There were once two or three titles on the topic, including my [*Linux Hardware Handbook,*](http://www.rodsbooks.com/hardware/) but these books are all old enough that they aren't really very useful any more. Apparently none sold well enough to keep the publishers interested in keeping the books updated. (It's conceivable that something newer is available, but if so I'm unaware of its existence.)

### Post by fuzzylogic25 on 2010-09-02
My favorite is Practical Guide to Ubuntu Linux, A (3rd Edition) by Mark G. Sobell. However, I think it's best to just install Ubuntu and work with it.

I too am a beginner. Here is what I have done. I looked at all the applications I use on Windows and listed them down. Some are apps I use frequently (almost 24/7) while some are apps I rarely use. List the top priority apps, the ones you know you would immediately require if you were to reinstall windows or something. Find equivalent or similar ones for Ubuntu. You might have alot of difficulty finding similar ones in rare cases, in such a situation try installing the windows app through wine.

Once you have Ubuntu installed and running with the core applications you always need, you can slowly work on installing other apps and also learning Ubuntu. It would be best if you also had a laptop. My main desktop had Windows 7, while the laptop had Ubuntu/Win XP for testing. By testing I mean, installing different apps and finding out the best ones I like. I only looked for my core apps though, the ones ill need straight away, such as a good back up utility that has versioning of files. Once I found the required apps I need for Ubuntu, I installed Ubuntu on the main computer and am now running it. If I need to use Windows for something, I refer to the laptop quickly. Such a situation is when I need to print something, I use windows under the laptop since I dont have drivers for ubuntu yet. It may seem like a hassle, but I dont print everyday so its no big deal to me. I print in batch anyway.

I think the best way to learn is to have it as your main operating system. That's most likely how you learned to use Windows. You installed it and over time learned little things here and there. Through repetition, you remember where a lot of things are. Understanding the inner workings of the linux kernal however, could be easier through a book.
