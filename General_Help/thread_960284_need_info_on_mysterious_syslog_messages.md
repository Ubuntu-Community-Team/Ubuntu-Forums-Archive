### Post by vob on 2008-10-28
Ok, issue mostly solved: Browsing the kernel source code revealed that these "PM:..." messages are generated by the kernel power management, controlled by the routines device_pm_add() and device_pm_remove() (see file drivers/base/power/main.c in the linux source tree). Each time the kernel finds a new device while booting it is added to the power manager's list. It turns out that I've added power management and kernel debugging about six months ago in order to get powertop running.

As for the weird time format: this seems to be defined in drivers/base/power/trace.c and derived from reading the realtime clock in the BIOS. Interestingly enough, the command there is 
```
printk("Time: %2d:%02d:%02d  Date: %02d/%02d/%02d\n",
                time.tm_hour, time.tm_min, time.tm_sec,
                time.tm_mon, time.tm_mday, time.tm_year);

```i.e. the format string for the year should only be 2 characters wide, although in my case there are three characters ("108"). But perhaps the kernel's print function printk (from kernel/printk.c) defines something else -- didn't check the 1139 lines of code in there...
