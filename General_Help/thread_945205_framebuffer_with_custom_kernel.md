### Post by ajmorris on 2008-10-12
> **lavinog said:**
> is that different from make xconfig? (this was what i was doing)


nope, make xconfig, make menuconfig, and make config are all the same, just different interfaces are used :)

try the following steps to configuring framebuffer support (i will use make menuconfig in my example, however, feel free to use which ever config method you want) :

Navigate through the following, and enable as shown (make sure to compile them into the kernel (denoted by *, as opposed to compiling them as modules (denoted by M):
Device Drivers ->
 Graphics support ->
  
[*]   Support for frame buffer devices
  
[*]   Enable firmware EDID
  **---   Frame buffer hardware drivers**
  ...
        Console display driver support ->
  
[*]    Video mode selection support
  <*>    Framebuffer Console support

that is really all you need for framebuffer support... if specifying a vga mode still does not function correctly, try the following:

enable:
Device Drivers ->
 <*>   Connector - unified userspace <-> kernelspace linker  --->
 ...
 Graphics support ->
  <*>   Userspace VESA VGA graphics support

and append this line to your kernel mode, instead of a vga mode:

video=uvesafb:1024x768-16@50,mtrr:3,ywrap

This means 1024x768 with 16 bit depth (the equivalent of 791 vga) and the
@50 is for 50hz screen refresh rate. Change this if your monitor requires 
a different refresh rate (this can cause the error you were experiencing 
with a blank screen)


Hope that helps,

AJ
