### Post by pytheas22 on 2008-06-23
You might get better answers if you post in the wine forum.  But I think that CrossOver has become less and less valuable as wine has become better.  At one time you needed CrossOver if you needed to run certain programs, but a lot of the applications that used to only work under CrossOver now work in wine with minimal hacking.  But there are still some programs (Office 2007 I think for example) that you need CrossOver to run.  CrossOver would also be worth it if you want stuff to "just work" instead of hacking wine--some programs require substantial effort before they'll run properly in wine (although more and more just work perfectly out of the box).

So you should take a look at CrossOver's list of supported applications and compare the ones that interest you with their entries in the [wine database]("http://appdb.winehq.org/") (pay attention to the dates and versions of wine used with each test because compatibility for different programs can change quickly), where you can find out how well they work with regular wine.  If you need something that will only work with CrossOver, use it.  Otherwise just stick with wine.  At least, that's what I think.

(on the other hand, I think that sales of CrossOver help support wine, so that's nice).

Also keep in mind that it's really easy these days to set up a virtual machine running XP to get your Windows software to run.  3D games won't work but pretty much everything else will, and most modern computers have more than enough resources to run XP (Vista probably not so much) at amazing speeds inside vmware or VirtualBox, both of which you can get for free.  That might be a better solution for you depending on what you need to do.
