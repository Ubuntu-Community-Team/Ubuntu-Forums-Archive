### Post by pjd99 on 2011-09-25
Generally speaking, if you can find an equivalent that has a linux build (i.e. native binaries), go with that first. You'll probably find that most lack some features of their windows/mac counterpart, but are good enough ~95% of the time.

Otherwise:
[http://appdb.winehq.org/](http://appdb.winehq.org/) is about as simple as it gets, unfortunately.

Games: you might have some success with playonlinux. This piece of software will install a known working version of wine for a particular game and keep it separate from other wine installs. Has profiles for other applications. Has the disadvantage of being rather obfuscated when compared to a normal wine install, so is kind of hard to customise if the default settings dont work.

Other Wine offshoots include crossover and cedega (don't know if it's still being developed or not) that have the advantage of being customised for certain pieces of software (like MS office, popular games etc).
