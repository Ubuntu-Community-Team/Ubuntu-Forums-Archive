### Post by cmp1988 on 2007-08-05
Most of that stuff you have listed is Windows specific, however, you can Dual Boot the two OS'es.  Meaning you have Windows for stuff that's Windows specific, and Ubuntu for everything else.

Photoshop is a definite no in Linux.  The alternative is GIMP, but it's absolutely no replacement for Photoshop
Not sure about Nero, Hamachi, or Daemon Tools.
The 2 games you listed are more than likely going to need Windows, as Wine (the program that trys to add a Windows compatability layer to Linux) is far from perfected.  Steam is Windows only and even mentioning Linux in the Steam forums gets you banned.  Besides, all the games on Steam are Windows anyway.  

Basically, you're best to dual boot if you like Ubuntu.  You can use the LiveCD to try it out without touching your HD install.  There are plenty of people who can help you with specific issues on this forum.  Not me very much, cause I'm still in the learning process.  Be sure to get more opinions on your situation however.

### Post by joe.turion64x2 on 2007-08-05
Of course you can always make a backup of all your stuff (in fact it is always recommended you do so).

Ubuntu can't run most of the programs you have mentioned, however there are alternatives. If you still need a particular program it is a good idea to make a dual boot setup (both Windows and Linux installed) so you can boot in Linux or Windows at will (or according your requirements). Also, if you have a large enough hard drive you can use it for both OS, again just be careful and back up your valuable data first.

If you have at least 10GB for your Ubuntu main partition and 1GB for swap then everything should be easy. Or if you can afford another drive just for Ubuntu even better.

There are several how tos in this forum, just search 'install ubuntu', also check the hardware requirements (but if you run AOE3 I can almost bet your machine is good enough).

### Post by yaustar on 2007-08-05
It is worth noting Windows games and Linux don't really go together. While it is possible to play them on Linux, it can require some fiddling. If you just want to try Linux, you may want to use a Live CD, virtual PC or dualbooting. 

Live CDs are slow but allows you to test drive Linux without installing anything onto your hard drive, it basically just runs off the CD.

Using a Virtual PC allows you to try a Linux OS inside Windows. Look at [VMware player (free)](http://www.vmware.com/products/player/) and download the a virtual Linux disk image [here](http://isv-image.ubuntu.com/vmware/). You want the desktop version.

Finally dual booting. You install Linux on your hard drive alongside windows, it allows you to choose at boot up which OS you want.
