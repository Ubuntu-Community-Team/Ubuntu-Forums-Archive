### Post by eXisor on 2006-11-09
Try changing the driver in the devices section to "radeon".

Below follows the device section of my xorg file. (The extra options are for optimising when using Beryl). DRI should work out-the-box for your card in Edgy. 

In Dapper I couldn't get my card going with DRI at all, atall.

```

Section "Device"
	Identifier	"ATI Technologies, Inc. Radeon RV100 QY [Radeon 7000/VE]"
	Driver		"radeon"
	BusID		"PCI:1:0:0"
	Option		"AGPMode"		"4"
	Option		"AGPSize"		"64"
	Option 		"ColorTiling" 		"true"
	Option 		"EnablePageFlip"	"true"
	Option 		"AccelMethod" 		"XAA"
	Option 		"XAANoOffScreenPixmaps" "true"
	Option 		"AGPFastWrite" 		"true"
	Option 		"BackingStore" 		"true"
	Option 		"SubPixelOrder" 	"NONE"
	Option 		"DynamicClocks"		"true"
	Option		"RenderAccel"		"true"
EndSection

```

### Post by lazyart on 2006-11-09
I have a 7200, which is a glorified 7000 and DRI works.

In a nutshell, remove fglrx, install the open source driver, then reinstall fglrx.  Then install driconf and tweak.

I used this guide: [http://www.ubuntuforums.org/showthread.php?t=221672&highlight=2d+performance](http://www.ubuntuforums.org/showthread.php?t=221672&highlight=2d+performance)
