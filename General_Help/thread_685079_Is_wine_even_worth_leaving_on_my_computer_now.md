### Post by Nythain on 2008-02-01
my advice... nuke it... nothing windows can do that linux cant do just as good... except for a few things, like Game, or Microsoft Office stuff like word and whatnot... and well... wine isnt going to do those things as well as just running windows anyways usually.
So, my advice, if you NEED certain windows apps and arent running a server, dual boot... if you dont need a certain few windows apps then, just run linux and use a linux alternative. The only real need/reason i had for wine, was when i only had one pc, and it was running daemons i couldnt afford to shutdown to dual boot.

### Post by arvevans on 2008-02-01
In my situation I find that occasionally one of my acquaintances will send me something that was written for windows, and wine does provide a way to run that. 

Wine does not occupy all that much space on my hard drive, and does not use memory when it is not running.  

So, for me the decision was to let it lie there dormant but available in case I do need it.

It also provides a ready put-down for my Windows running friends...
  "Sure I can run windows applications on my Linux computer...Can you run Linux applications on your windows computer!"  :)
