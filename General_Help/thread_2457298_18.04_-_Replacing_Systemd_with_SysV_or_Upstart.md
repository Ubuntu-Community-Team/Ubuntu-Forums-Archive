### Post by grahammechanical on 2021-01-29
In my uneducated opinion removing systemd will break the operating system. It was Redhat that produced systemd and it was Redhat employees working on Debian that pushed Debian to drop upstart and replace it with systemd. Debian developers who lost the argument (and apparently it was a bit of an argument) quit Debian and forked the code with something called Devuan.

The reason that certain Debian developers opposed systemd was because it was not going to remain a simple init system or startup utility but would almost take over the whole of the OS. This is why I think that removing systemd will break the OS.

[https://wiki.debian.org/Debate/initsystem/systemd](https://wiki.debian.org/Debate/initsystem/systemd)

[https://www.devuan.org/os/init-freedom](https://www.devuan.org/os/init-freedom)

Once Canonical decided to accept Debian as the upstream code of Ubuntu without replacing systemd, then Canonical would have stopped maintaining and developing Upstart.

If you are really that opposed to systemd, then install Devuan. The default desktop environment is Xfce but you can also chose Cinnamon, KDE, LXQt; Mate.
