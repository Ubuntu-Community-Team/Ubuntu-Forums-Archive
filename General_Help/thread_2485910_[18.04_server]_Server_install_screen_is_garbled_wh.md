Hi all, im using Packer to create some device images using QEMU and im having problems with 18.04 server ISO having a garbled screen during debian-installer install. This only happens when I boot into UEFI and not when I boot with regular BIOS. In both cases, the initial selection menu comes up for BIOS or GRUB menu for UEFI as expected but once it goes into the installer when using UEFI it craps out as shown in the image below.

Any ideas on boot parameters I could use to try to resolve this? I have tried `nomodeset` and `vga=788` (I think it was 788, it was in one of the other UEFI grub options) but neither helps.

### Post by ju2wheels on 2023-04-13
Ok, i figured out the issue. For whatever reason when using UEFI, qemu now needs `-vga virtio` option to not mess up the graphics although this wasnt needed when doing the same thing using BIOS.
