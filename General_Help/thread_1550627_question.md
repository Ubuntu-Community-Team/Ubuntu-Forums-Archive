### Post by 3Miro on 2010-08-11
> **TNT1 said:**
> 
A) well done for recommending the software center, and not somemore command line stuff, and 
B) remember with windows, installing and uninstalling lots of  programs resulted in a corrupt registry and pain and suffering all round. Does this still apply in Ubuntu?

A) Command line is nice for giving instructions, it is easier then "click here", "click there". However, every Ubuntu user should be familiar with the software center.

B) There is a database repository in Ubuntu that can get somewhat clogged if you install a lot of things. At one point I had really a lot of stuff since I had Gnome, KDE and XFCE installed all at the same time. There are, however, many differences between the package database and the windows registry. Here are a few:

1. In windows programs depend on the registry to run, in Ubuntu programs depend on the registry only for install/uninstall purposes. If the registry gets clogged, everything in the system slows down, in Ubuntu only the software center gets somewhat slower, everything else works just as well. Furthermore, a complete break into the windows registry breaks the entire windows system, while a break in the Ubuntu package database, results only in break of the software center and updates (which is much easier to fix).

2. In windows, it is generally very easy to install something, however, it is almost impossible to remove something completely. Every program that you ever install and later remove, usually leaves some trace. In Ubuntu things get removed completely, so even if you install a bunch of things and later remove them, it will be as if they were never installed. Multiple install/remove in windows tend to slow down the system, however, in Ubuntu they have no effect.
