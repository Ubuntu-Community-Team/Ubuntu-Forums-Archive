### Post by krt on 2010-05-23
The default location for saving things from Open Office is my home folder Documents. If I save an OO doc there, it works fine. Unfortunately, I attempted to save a document to the desktop and the page grayed out and the thing stalled. When I forced quit, of course, I lost it for good. OO offered to "recover" it, which I OK'd and got a new blank document.

Further experimentation shows that this is a problem with any OO document I make for test purposes. I have never run into this problem before with OO in Linux. I guess there is no big problem in saving it to the default Documents folder, but I have been used to saving it to the desktop in Ubuntu 9.1, and it came as a rude shock when I lost it in Ubuntu 10.04. this being the first time I have done it in Lucid.

Incidentally, I have OO set to automatically save stuff in the Word .doc format/extension, but it makes no difference what extension I use (e.g., rtf, default OO, etc.)

There is no problem saving other things to my desktop . . . for example, I can make a document in Gedit and "save as" to Desktop; the problem only occurs with OO.

I checked the properties>permissions and as owner, I have read - write permissions just as I do in Gedit.

I have installed Lucid on two different computers, and OO behaves the same way in both installations.

Anybody else run into this problem?  Any suggestions?
