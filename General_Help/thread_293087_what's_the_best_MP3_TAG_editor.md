### Post by mmcmonster on 2006-11-04
> **OneSeventeen said:**
> That's funny, I was going to mention EasyTag.  I use it for my podcast each week, and it is a lifesaver.  (surprisingly few mp3 tag editors seem to easily support embedding graphics in the MP3)

I agree.  [Easytag]("http://easytag.sourceforge.net/") is quite good to use, and will allow you to embed an image in the .mp3 file.  The only problem I see is that it wont show any tag that it doesn't expect, and it wont let you create a new tag other than the ones hard0-coded into the program. (ie: If you want to create a tag called "volume to be played at", it's not the program for you.)

[Ex falso]("http://www.sacredchao.net/quodlibet") is very good as well.  The on two negatives about it are that it can blow away an image that is embedded in the mp3 file, and that the interface is clumsy when you want to edit many fields for the same file.

I suppose [cowbell]("http://more-cowbell.org/index.php/Main_Page") is another application you can give a try.  I was never that interested in it, for some reason I can't remember anymore. :-)

### Post by tlacuache on 2007-03-15
If you're looking for something to take a little more of the work out of it for you, I recommend [MusicBraiz Picard]("http://musicbrainz.org/wd/PicardDownload"). It will analyze your mp3's, create an "audio fingerprint" of each one, and look it up in database of these fingerprints. I used it to completely re-tag my whole collection and it worked like a charm.
