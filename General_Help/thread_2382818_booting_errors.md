### Post by oldfred on 2018-01-17
Do you have AHCI on? With SSD it must be on for trim to work.
That can also be a reason for outside of disk error. The old IDE setting which then tries to be the IDE configuration may have the IDE error of not being able to boot from files beyond 137GB of drive.

When booting can you directly boot Windows from sda when sdb plugged in. Probably f12 key.

Your fstab has the Windows partition mounted by device. Best to change to UUID as device can be an issue. 
       [https://help.ubuntu.com/community/MountingWindowsPartitions](https://help.ubuntu.com/community/MountingWindowsPartitions)
[http://ubuntuforums.org/showthread.php?t=1983336](http://ubuntuforums.org/showthread.php?t=1983336)

BIOS boot order and Ubuntu's drive order are not necessarily the same. 
BIOS & grub see drives as hd0, hd1. Then only after system started are drives assigned.
I have had drive order issues with grub. The drive you boot from will always be hd0.
So if booting sdb, sdb still will be hd0 in grub, so if Windows is hd1, it may be confused. Grub used to do a map command to swap drives, so Windows would think it was still hd0 as that is how it is configured.

You can try an entry in 40_custom.


```
 menuentry "Windows 7"
{

 drivemap -s (hd1) (hd0)
set root='(hd0,msdos2)'
ntldr /bootmgr 

 }
```    

Not sure on order of hd1 & hd0 settings, you may have to experiment.

You may have to experiment and see if different entries give same errors.
