### Post by bodariarska on 2010-12-17
Flooded log entries are generated by Realtek wireless driver (rtl819xSE / r8192se_pci) and appear at least on Shuttle XPC XS35 / XS35GT (10ec:8171 / RTL8191SEvA) with BIOS v1.09.
 
Until driver is fixed you can create new file named /etc/rsyslog.d/00-filter.conf with following lines and restart syslog using "service rsyslog restart" command. You'll still end up with dmesg filled with these debug log entries, but at least they're no longer sent do logs on disk.
 
```
# Don't log debug messages from Realtek wireless driver on Shuttle XPC XS35(GT)
# http://ubuntuforums.org/showthread.php?t=1619788
# http://www.rsyslog.com/doc/rsyslog_conf_filter.html
:msg, contains, "rtl8192_SetWirelessMode(), wireless_mode" ~
:msg, contains, "=====>rtl8192_set_chan()====ch" ~
:msg, contains, "pwrdown, 0x6(BIT6)=71" ~
```
