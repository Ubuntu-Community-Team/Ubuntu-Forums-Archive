gksu gedit

### Post by drs305 on 2011-04-17
Boot the LiveCD. Run the following just to confirm sda is still your Windows drive and sdf has your Ubuntu installation:
```
sudo fdisk -l # Lowercase L
```
If the above provides different results, amend the instructions to reflect the proper drive.

Install Grub2 to the sdf drive/MBR. Do not use the partition number in the second command.
```
sudo mount /dev/sdf5 /mnt
sudo grub-install --root-directory=/mnt /dev/sdf
```

Reboot, and make sure your BIOS boots the external drive (sdf) first.

You should see menu entries for both Ubuntu and Windows. If you don't, after booting Ubuntu, run:
```
sudo update-grub
```

### Post by drs305 on 2011-04-17
> **TacticalApe said:**
> Do I reboot with or without the cd?
Also, I looked at the chroot link in your signature, and it looks too confusing for me.
I am the first to admit it looks complicated. But it is actually just cutting/pasting the commands, modifying them only for your particular drive/partition. The other option would be to try reinstalling Ubuntu.

Yes, boot without the CD for this part. (For installing lilo (later), you will boot the LiveCD.)

> 
Also, if this means anything, I get the same error messages when the EHD isn't plugged in, and that's where I installed ubuntu to.
The reason you get the same message is that Grub can't find it's files, or it finds them but at least one is corrupt. So with or without the external connected, it doesn't find what it is looking for. (You should be booting with it connected.)

> 
 When I fix this, will I still be able to acess Windows when the EHD isn't plugged in?
At the moment you probably can't because the boot script indicates that you installed Grub2 on the sda MBR. This is easily repaired though.

From the LiveCD, install lilo. This is an alternate bootloader. We aren't going to install it completely, so disregard the configuration messages. We can point the sda MBR back to Windows. If your BIOS boots sda first and your Windows files haven't been altered you will get your Windows menu.
```

sudo apt-get install lilo
sudo lilo -M /dev/sda mbr
```

In an ideal world, you would have the Windows bootloader on sda, the Ubuntu bootloader on sdf, and have the system boot sdf first. With the external connected, you would see the Ubuntu/Windows menu (Grub). If you ever had problems, you would change the BIOS to boot sda first  or simply disconnect the external so it would boot Windows.
