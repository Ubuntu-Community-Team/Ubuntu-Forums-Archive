Please remember that many things are different than the way you were used in windows, which, to be honest, is not really a bad thing, simply different.
Through the package management system you do not need to know where the programs are located in order to remove them. Just ask the utility to remove them. All will be done automatically, without you needing to search for all sort of directories and files.

I think this system is way more elegant than what Windows have. Also, you need to know that the directory structure used in linux is different to the one used in Windows. Normally, the zone accessible to regular accounts is located in /home/user_name, as well as in temporary directories. The rest need not to be touched by it.
The other directories can (and should normally) be accesses only for administrative purposes via sudo (Ubuntu uses this) or a root account.

When a program is installed, its files are distributed across different directories, depending on their type. For instance system-wide configuration files will go under /etc, libraries in /usr/lib /usr/local/lib, executables in /usr/bin /usr/local/bin /bin, and so on.

You can read more here: [http://en.wikipedia.org/wiki/Filesystem_Hierarchy_Standard](http://en.wikipedia.org/wiki/Filesystem_Hierarchy_Standard)
