PLEASE:  No "LINUX isn't Windows" comments.  Been there done that.

As a practicing computer systems engineer for decades now, I can tell you that we do NOT like being mislead.  It is as important to us to know what something "is not" as it is to know what something "is".

So, I would start out disagreeing with your No ... comments; instead, I would begin the presentation with a "Linux is Not Windows" slide, indicate the major differences between Linux and Windows -- and then follow that with slides detailing the advantages of Linux over Windows.  This will get the negative stuff out of the way early, and leave your audience with a postive impression.

### Post by Mark Phelps on 2010-07-31
> **n2stc said:**
> ...What would you consider the "major differences"?...
You'll get as many different answers to this question as folks you ask ...

But since you DID ask, here are my answers (for Engineers):
1) Linux is built on a foundation of open-source, meaning not only CAN you change the source code, but you can freely install any one of a variety of Linux variants (all known as Distros) without worrying about Activation.
2) Linux is an alternative to MS Windows, meaning that while it WILL do nearly all the things that MS Windows does, since it is not doing them using MS Windows OS or MS Windows apps, there are likely to be some visible differences in how things get done.  This means that folks WILL have to learn how to do some things differently in Linux.
3) Linux does NOT use several of the infrastructure components that have become embedded into MS Windows (e.g., UAC, Registry, IE7/8).  Thus, folks who will be providing System Admin support for Linux will have to learn how those functions are imp[emented and supported (if at all) in Linux.
4) The Linux community is not intent on replicating the latest and greatest features of MS Windows.  Thus, stuff that is built into Win7 (i.e., Homegroups, Aero Peek) is simply not available in Linux.
5) Distribution rights: Unlike with MS Windows, you have the legal right to install, or copy, an installation onto as many machines as you want.  This means no dealing with WGA/WPA, and no worrying about de-activation.
6) Driver support: While the "myth" if often said that MS Windows has a lot more drivers than Linux, primarily due to OEMs NOT writing Linux drivers, in reality, it really comes down to specific hardware items and the versions of each.  Unlike MS Windows, Linux provides the ability to try out each version of the OS, without installing it, to see how well each version supports the hardware on a specific machine.
