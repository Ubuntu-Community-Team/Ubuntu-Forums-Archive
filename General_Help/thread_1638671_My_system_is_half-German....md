### Post by Veteropinguis on 2010-12-05
A few weeks ago I changed my system language to German. Since I'm not yet proficient enough in German to handle that, I uninstalled the German language packs.

Several reboots, complete shut-downs, and battery removals later, my system is still partly German. Terminal output is almost entirely in German. If I get a 'Fortfahren [j/n]?' and hit J, it cancels the terminal action- but if I hit Y, it continues. Sometimes error messages come up in English, other times they come up in German.

If I sudo apt-get install anything, it installs in German. Openoffice runs in English, but instead of 'Word Processor' it's 'Textverarbeitung', instead of 'Spreadsheet' it's TabellenKalkulation. Banshee is 'Banshee Medien-Wiedergabeprogramm' and is fully in German.

If I open up system Help, German text welcomes me to the Ubuntu-Hilfezentrum, but when I pick an option from the Hilfethemen, my options and the pages are in English, except for the page explaining how to use the Help Center.

I installed Latin and Swedish language packs at the same times as the German. For a brief while I somehow ended up with a Latin-German hybrid system; never booted in to Swedish. The Latin all went away when I uninstalled the language pack, and nothing is in Swedish, so why is the German so persistent?

### Post by davidvandoren on 2010-12-06
There is no reason to uninstall the language pack

What you do is;
Log out click on your login name and then at the bottom you see the language options.
There you change to your language.
I'd say reinstall the German.
You must use gmd window manager though. Not the kmd from kubunto
