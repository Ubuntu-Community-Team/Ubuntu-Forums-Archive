### Post by Herman on 2007-06-02
Hello Shampyon,
You should boot your Ubuntu Live CD and mount your /dev/sda1 like this, ```
sudo mkdir /media/ubuntu
``````
sudo mount -t reiserfs /dev/sda1 /media/ubuntu
``` and then do ```
sudo gedit /media/ubuntu/etc/fstab
``` Leave that file open and go to another workspace and open a second terminal.
In your second terminal run,
```
ls /dev/disk/by-uuid/ -alh
``` Then copy and paste the filesystem UUID numbers from the output of that to correct your /etc/fstab in the first workspace and save it.
It's easier to read that output if you expand the terminal before you apply the command too, just as an extra little tip there.
If you have too much trouble with that see if you can post a copy of your /etc/fstab and the output from that command here and someone should be able to help you more.

Regards, Herman :D

### Post by Shampyon on 2007-06-02
Alright, I'm back with great news. Your advice was flawless. Everything's back to normal!

I messed up when fixing it all by somehow mounting two partitions (sda6 and sda7) to the one mount point (sda6). It came up as two drives(sda6 and sda6[2]), so that clued me in. A quick edit to change the extra sda6 in fstab and mtab back into sda7 and the whole system is runnig perfectly. Thanks again :)
