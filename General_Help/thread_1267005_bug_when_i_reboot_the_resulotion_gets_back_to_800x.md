### Post by cesium62 on 2009-09-17
Display resolution...

There are about 5 different ways to tweak this...

System: Preferences:Appearence -- if you use this, it may prompt you to install a version of an nvidia driver that is relatively smart about display settings.

System: Preferences:Screen Resolution -- this is fairly useless until you have global settings set correctly.

System:Administration:X Server Settings -- this tends to prompt you to run a command that initializes your /etc/X11/xorg.conf files.  Once other things are configured correctly, this becomes mildly useful.  And running the command to initialize xorg.conf tends to be useful.

sudo displayconfig-gtk -- This tends to be confusing as the inutuitive button clicks lead to dead ends.  But, once your xorg.conf files have been initialized, you can select generic monitors, and then screen resolutions that tend to configure xorg.conf.

The 'gtf' command is useful for generating additional 'modeline' lines to add to the Monitor section of the xorg.conf file.

Eventually, you may want to edit /boot/grub/menu.lst and set your 'vga=' parameters.  An extensive web search will help you understand what values might work.


If anyone wants to work on this as a project for Karmic...

1)  Change xorg.conf so that the horizontal and vertical pixels and the refresh rate are stored.  X can run gtf itself to figure out the overall modeline.

2)  Reduce the number of tools to 2.  We want one tool that *always* works to tell the system what kind of monitor and video card we have.   This should install all the right nvidia (or other) drivers, and leave a global description of the monitor where grub will correctly find it while updating xorg.conf.  The second tool allows an individual user to screw around with their personal settings.
