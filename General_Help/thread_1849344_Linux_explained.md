### Post by WorMzy on 2011-09-24
> **sanderd17 said:**
> First: install Arch
Second: install a Linux from scratch.

For both, the documentation is most important. Arch has precompiled packages, but you still need to install all packages yourself. LFS only has source code.

I agree with this. If you're like me, you won't learn much by just using Ubuntu.

Basically, what you get when you install Arch, is a command line, networking, and the package manager. No sound, no desktop environment, no bells and whistles, just the very bare essentials. From there, you can learn how to get everything else working. You can choose which sound architecture you want (ALSA or OSS, whether you want to install Pulse, JACK, etc), install the driver for your graphics card (either one of the open source ones, or the propriety one, where applicable), then install and configure X, then install the DE/WM you want, etc.

You'll learn more in a week setting up and using Arch, then you will in four years of using Ubuntu.
