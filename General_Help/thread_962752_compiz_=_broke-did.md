### Post by loomsen on 2008-10-30
Alright, here we go.

Got 8600M GT, so we should be able to use the same Options and such in xorg.conf.

As you log(the last two commands) show you try and load type1, which doesn't exist tho. So I'd say open up your xorg and edit it, at least remove/comment out type1.
My Module Section looks like this:
```

Section "Module"
    Load           "bitmap"
    Load           "ddc"
    Load           "extmod"
    Load           "freetype"
    Load           "glx"
    Load           "int10"
    Load           "vbe"
    Load           "dbe"
    Load		"record"
    Load		"dri"
EndSection

```

You should be able to use similar/the same settings.
Monitor Section is ok as it is, 
```

Section "Device"
    Identifier		"Videocard0"
    Driver			"nvidia"
    VendorName		"NVIDIA Corporation"
    BoardName		"GeForce 8600M GT"
    BusID          	"PCI:1:0:0"
    Option		"AllowIndirectPixmaps"	"true"
    Option		"Coolbits"	"1"
    Option 	   	"AllowGLXWithComposite"	"true"
	Option		"NoLogo"	"true"
[COLOR=BLUE]##    Screen       		0  ## You may omit this line, I'm using multiple Screens tho, so I need it.[/COLOR]
EndSection

```

Adding the BusID of your card shouldn't be a 2 bad idea, you may find out with:
lspci | grep nVidia

This is what I get:```
docter[~] lspci | grep nVidia
01:00.0 VGA compatible controller: nVidia Corporation GeForce 8600M GT (rev a1)
docter[~] 

```

Try and add the Options too. The two (Allow...) Options shouldn't be of any disadvantage.
The Coolbits Option enables an additional field in nvidia-settings where you may overclock your card if you like to.
[[img]http://img135.imageshack.us/img135/759/screenshot28oi8.th.png[/img]](http://img135.imageshack.us/img135/759/screenshot28oi8.png)

Further, this is how my Screen Section looks like:
```

Section "Screen"
# Laptop
    Identifier		"Screen0"
    Device		"Videocard0"
    Monitor		"Monitor0"
    	DefaultDepth    	24
    Option			"AllowGLXWithComposite"	"true"
    Option			"AddARGBGLXVisuals"	"true"
[color=blue]##    Option		"TwinView" "0"   ## only of interest with multiple screens
##    Option			"TwinViewXineramaInfoOrder" "DFP-0" ## multiple screens again[/color]
    SubSection		"Display"
        Depth			24
        Modes		"1440x900" "nvidia-autoselect"
    EndSubSection
EndSection

```

Compare it to yours, and make additional changes where you'd like. I commented out lines which are obsolete if you're using only one Screen.

Further, I have another Section "Extensions":
```

Section "Extensions"
	Option 	"Composite"	"Enable"
	Option 	"RENDER"	"true"
	Option 	"DAMAGE"	"true"
EndSection

```

Just add it to the bottom of your xorg.conf if you'd like. 

```

docter[~] grep Composite /var/log/Xorg.0.log
(**) Extension "Composite" is enabled
(II) NVIDIA(0): Support for GLX with the Damage and Composite X extensions is enabled
docter[~] 

```

Then your xorg.conf should be fine with using compiz.
