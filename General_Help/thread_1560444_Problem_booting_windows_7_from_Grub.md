### Post by myrmecophilous on 2010-08-24
edit: Nevermind, I figured it out. The problem was a BIOS setting.


I'm having the same problem others have had trying to boot windows 7 from grub. I installed ubuntu first on one hard drive, then windows 7 on the other. Grub sees the drive with windows and gives me an option for it in the menu, but when I try to use it I get the error:

> error: no such device 3ce661377e89ec8e
error: invalid signatureThis is the same situation as was discussed in [this thread]("http://ubuntuforums.org/showthread.php?t=1549788") and others before it, but I tried the given solution:

[quote=oldfred]
You need to add a command to grub for mixed msdos (MBR) and gpt partition schemes for different drives.

[http://ubuntuforums.org/showthread.php?t=1405650](http://ubuntuforums.org/showthread.php?t=1405650)
Grub 2 malfunctions with a mixture of GPT and MS_DOS partition tables. But there is an easy fix, add to /etc/default/grub:
GRUB_PRELOAD_MODULES="part_msdos"

[http://ubuntuforums.org/showpost.php...0&postcount=14]("http://ubuntuforums.org/showpost.php?p=8822200&postcount=14")
For GPT sdc disk:
added the GRUB_PRELOAD_MODULES="part_msdos" to /etc/default/grub and ran grub-install /dev/sdc and it worked!
now see all the partitions on my IDE drives and boot from them too!!!!!
[https://bugs.launchpad.net/ubuntu/+s...b2/+bug/604743]("https://bugs.launchpad.net/ubuntu/+source/grub2/+bug/604743")
[/quote]

but that didn't work, despite apparently working for everyone else who has ever had this problem.
