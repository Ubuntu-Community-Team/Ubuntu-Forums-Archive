### Post by tbraun on 2009-11-27
DeMus,

Your entire premise is flawed: You talk about wasted CPU cycles and time. This is just plain wrong.

When set up correctly, with compiz you use less CPU cycles than without it. Why is that? Because all the windows actually remain open all the time (on the surface of that cube, if you have that enabled) and don't have to be re-opened when you switch desktops/viewports.

If compiz is disabled and I switch between viewports I can see a noticeable CPU spike and also feel a noticeable delay as all the various application windows in the new desktop are being opened and re-drawn. With compiz this is instantaneous, because the window was still open. The CPU does not need to be bothered at all. I know that the GPU is involved instead, but only ever so lightly. For example, I don't have the cube enabled, I just set the 'desktop wall' to do an instantaneous (non-animated) switch. When I switch between viewports now there is hardly any CPU load at all, and even the graphic card does not switch to a higher frequency, since it's such a light operation for it.

Therefore, with compiz: LESS CPU cycles wasted. Without compiz: MORE CPU cycles wasted.

I don't really care how you have set up your windows and how you normally work on the computer. That seems to work for you. Whatever. You don't seem to consider the fact that other people work differently than you.

For example, for me having the ability to rapidly switch between viewports without delay and the CPU revving up every time is worth a lot, since I use this all the time (via keyboard shortcuts no less, since grabbing the mouse just takes too much time).

Also useful for me: The ability to very quickly do a smooth zoom anywhere on the desktop. Very nice to have when you visit a web-site with the font being set too small. Again, hardly any CPU load for that, since the GPU does it for you. Sure beats doing a CTL-+ on Firefox, which would use much more CPU.

Also useful: The ability to quickly make a window transparent to watch what's going on behind it while you are still working in it. Exposing all windows at the same time, and so on.

Your criticism focuses too much on the cosmetic aspects of compiz and ignores various shortcuts and features that are indeed productivity enhancing, doesn't take into consideration that other people may work differently with their computer than you and furthermore is based on wrong assumptions about supposed waste of CPU cycles, time and energy, when in reality the opposite is the case.

Yes, compiz still has bugs. It's not 100% stable for me. But the enhanced desktop experience is actually still worth putting up with it.

### Post by SmSpillaz on 2009-11-29
Just to prevent this thread from spreading into out of control chaos, here are comments from a developer.

1) Compiz uses CPU cycles just like any other program. In fact, the current design of compiz is rather inefficient since when you have a lot of plugins enabled they each have to process information on certain events. The 0.9.x series aims to  fix a lot of this
2) You'll typically see a spike in CPU usage on a non opengl-composited desktop since the CPU has to redraw windows when they are moved / resized or you change desktops
3) Some people like superfluous effects ;-) That's what the target audience of those effects are. The whole point of plugins is that you can turn them off
4) We cant show minimized windows in scale since there is no proper solution to doing it. We generally don't like hacks
5) Every program crashes, but the 0.8 series has had a lot of work put into it to prevent these. Please report crashes. We fix them.
