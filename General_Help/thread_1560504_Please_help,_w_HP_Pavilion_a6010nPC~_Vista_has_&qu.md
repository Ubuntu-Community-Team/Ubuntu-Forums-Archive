### Post by Rubi1200 on 2010-08-25
I checked online and the graphics card shouldn't be an issue so a 9.10 CD probably won't help with that.

A quick Google search also indicates that the original error
 > No physical memory is available at the location required for the windows boot manager. The system cannot continue seems to be a Vista thing.

Anyway, try booting the LiveCD again like this:

Put the LiveCD in and boot up;
 
when you see the Ubuntu logo hit Enter;
 
Choose language and leave the default Try as a live cd or whatever it says again;
 
Important: hit F6 and then Esc to see the boot line
 
**file=/cdrom/preseed/ubuntu.seed boot=casper initrd=/casper/initrd.lz quiet splash --**
 
Now, again this is important, move the cursor backwards and remove quiet splash;
 
Add the following parameter:
 
i915.modeset=1
 
In other words, the boot should look like this:
 
**file=/cdrom/preseed/ubuntu.seed boot=casper initrd=/casper/initrd.lz i915.modeset=1 --**
 
Now hit Enter and see if you get any further.

Hope this helps.

### Post by Guilden_NL on 2010-08-25
Could be a few things.  Off the top of my head in no particular order:
[LIST]
[*]BIOS corrupted (Actually the more I look at the screens, this is the most likely
[*]Hard Drive and/or controller fried
[*]Corrupted/Fried RAM memory
[/LIST]

Popping the power off and on to the power source while it's fired up is about the worst thing you can do to a PC.  Expect the worst, pray for the best and make sure that you get a small UPS to prevent it ever happening again!

You need to try to get into the BIOS by pressing F1 during the very first start up.  There are several utilities that can indicate if this is the case.  If it looks like it, then I would recommend flashing the BIOS with the latest version.

If BIOS is OK, then you can look at the drive and the controller.

BTW, this system has an internal Intel GMA 950 graphics card, but I seriously doubt that this has anything to do with the problems you're seeing.
