### Post by jaikiran on 2010-05-01
Hello everyone,

This is my first attempt at trying to install multiple OS on a single machine. I have a laptop which already has RHEL5 installed on it. Now I am planning to install Ubuntu 10.04 on it so that I can have a dual boot setup with RHEL and Ubuntu co-existing on the same machine.

I decided to install it by following this guide [https://help.ubuntu.com/community/Installation/FromLinux](https://help.ubuntu.com/community/Installation/FromLinux). I downloaded the appropriate .iso and followed the steps mentioned there. To briefly summarize what I did:

1) I am on RHEL5, so through the LVM UI, I created a separate logical view within an existing group. So I now have the following:

```
[jaikiran@jaikiran ~]# df -h
Filesystem            Size  Used Avail Use% Mounted on
/dev/mapper/Root
                       11G  9.7G  452M  96% /
/dev/sda1              99M   25M   70M  26% /boot
tmpfs                 1.5G     0  1.5G   0% /dev/shm
/dev/mapper/Ubuntu
                      5.0G  839M  3.9G  18% /Ubuntu
/dev/mapper/Home
                      6.0G  5.5G  128M  98% /home

```The /dev/mapper/Ubuntu is where I plan to copy over the .iso image in the following steps.

2) Followed the guide and ran the following commands:

```
mkdir /tmp/install_cd
mkdir /tmp/installer

mount ubuntu-10.04-desktop-i386 -o loop /tmp/install_cd
mount /dev/mapper/Ubuntu /tmp/installer

rsync -a /tmp/install_cd/ /tmp/installer

umount /tmp/install_cd
umount /tmp/installer
```3) Followed the guide to edit grub.conf to add the Ubuntu installer:
```

# grub.conf generated by anaconda
#
# Note that you do not have to rerun grub after making changes to this file
# NOTICE:  You have a /boot partition.  This means that
#          all kernel and initrd paths are relative to /boot/, eg.
#          root (hd0,0)
#          kernel /vmlinuz-version ro root=/dev/LVMGroup/Root
#          initrd /initrd-version.img
#boot=/dev/sda
default=0
timeout=5
splashimage=(hd0,0)/grub/splash.xpm.gz
hiddenmenu
title Red Hat Enterprise Linux Client (2.6.18-164.11.1.el5)
    root (hd0,0)
    kernel /vmlinuz-2.6.18-164.11.1.el5 ro root=/dev/LVMGroup/Root rhgb quiet
    initrd /initrd-2.6.18-164.11.1.el5.img
title           installer
    root            (hd0,0)
    kernel          /casper/vmlinuz boot=casper root=/dev/ram ramdisk_size=1048576 rw
    initrd          /casper/initrd.lz    

```Rebooted the system and was able to see "installer" in the GRUB menu. On selecting it, I was shown a "File does not exist" error, which kind of is obvious when I start looking at the comments in the grub.conf which say:

```
# NOTICE:  You have a /boot partition.  This means that
#          all kernel and initrd paths are relative to /boot/
```And obviously, it takes /casper/vmlinuz as relative to /boot and can't find it.

So how do I get past this? The /casper/vmlinuz doesn't reside in my /boot partition and instead is at /dev/mapper/Ubuntu which is mounted at /Ubuntu. I can't copy over that file to the /boot partition because, the boot partition is not big enough (and I wouldn't want to increase the size).

Can someone suggest a way to get past this? Ultimately, I would like to install Ubuntu from my existing RHEL.
