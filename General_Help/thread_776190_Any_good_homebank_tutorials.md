### Post by timzak on 2008-05-05
> **geokok1981 said:**
> I wish someone would give homebank the documentation it deserves sometime soon.

As for gnucash, how is it? Is it ok to use or would you describe it as overkill?

It took me awhile to get used to because it works so differently from what I was used to (MS Money).  Rather than importing my data, I felt more comfortable starting from scratch.  I started at the beginning of the new month so it was easier to set up that way (I still have MS Money on an old laptop that I can refer to for the old data).

The cool thing about Gnucash is that you don't have to use the "overkill" features if you don't want to.  You set it up how you want to.  Right now I am struggling with the mortgage druid, but in the end, if that's not set up perfect, it won't affect my bank account setups (which is what really matters most to me).  Right now my bank accounts are balanced and working correctly, but Gnucash shows an imbalanced mortgage setup.  I'm able to ignore this because the bank and credit accounts are set up properly and working accurately.  If I eventually get the mortgage set up right, it will be very cool.  Basically, when set up right, you just input your monthly mortgage and it automatically adjusts the various sub accounts (principle balance, interest paid, escrow, etc).  So you'll get a running total of how much you owe on your house as well as how much interest you've paid, escrow, etc.  Like I said, if you don't want to set this up perfectly, it will still work fine for your bank and credit accounts.

The other thing I am considering is that now that I have it set up and working properly for my bank and credit accounts, I can slowly (manually) input my old data one month at a time if I WANT to.  This is just for the sake of having the data start at the first of the year.  So I'm currently working on April's data.  If I get this done in a decent amount of time, then I'll start with March, then Feb, then finish with Jan.  I like inputting manually because then I am sure the transactions are being assigned to the right categories to begin with.  It seems too disorganized for me to to import a bunch of data, then go through and assign categories.  One thing if you try to do this:  make sure that each time you start with a month, that you FIRST go and reset your opening balances for each affected account.  So I had an opening balance as of 5/1/08.  When I started inputting all of April's transacations, I put a new transaction for 4/1/08 for opening balance of what it was for that day.  Do this for all accounts (savings, credit, etc).  Then delete the 5/1/08 opening balance transaction as it no longer applies, THEN start inputting all of April's transactions.

Hope this helps.  Gnucash is a great program once you get past its quirkiness.

Oh, one other thing:  Gnucash creates a bunch of files that default to your Home folder.  To clean up the clutter, I created a folder in Home called Gnucash, then Saved As to that folder, then deleted the original set of files in Home.

Good luck with your decision.
