Hi zenarcher,

I'm not sure if I can fix your problem, but I thought I'd throw a few ideas around for you.

1. Regarding evolution not automatically showing images, there is a reason for that. I've described this in another thread, but basically it depends if the email is a HTML messages with inline images, OR if the images are attachments. If they are inline images they will be displayed on screen as you'd expect. But if they are attachments, you have to specifically open them. This is the way things are designed to work, however I do agree that Thunderbird is very cool how it just displays attached images automatically. Perhaps some day evolution will do the same, maybe via a plugin.

2. Regarding the opening of attachments - I'm using evolution with karmic, and I don't have any problems - I just click it, and it opens as I would expect. This is just an idea, but have you tried starting GNOME instead of KDE, and seeing if the same thing happens? Reason I suggest this is because evolution (I think?) is offially a GNOME application, and hence may rely on some GNOME features for working out which program to use for which file. In KDE, I think kontact is the default? So anyway, if you have GNOME on the system too, start that, see if the problem is fixed.


Good luck!
