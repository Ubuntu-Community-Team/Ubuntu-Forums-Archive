### Post by Mighty_Joe on 2010-03-30
Python is a great language to start with.  [Python for Software Design]("http://www.greenteapress.com/thinkpython/") is a good beginner book that's free for download.
There are many [online communities]("http://www.python.org/community/lists/") for Python developers as well.
When learning how to write code, there is no substitute for writing code.  A LOT of code.  
Once you master the basics of how to get code to do things, think about what else you like to do and how to get code to help you.  For example, there's Python libraries for MP3 tags, so I wrote an app to organize and normalize my MP3 collection.  
This will do two things: It will challenge you to learn new things and it will keep you motivated because you will be using your skills to make your life better (easier, more fun, etc.).
Some people don't need formal schooling but I found taking college-level programming courses really stretched me in ways I never would have gone had I stayed on my self-taught course (i.e. we had to learn C, LISP, IBM assembler, Motorola 68000 assembler, write a compiler, interpreter, basic operating system, work on IBM mainframes, Solaris and Windows workstations).

### Post by DrMelon on 2010-03-30
First things first, as you are already aware of the differences between the two jobs, we won't need to cover that.
Secondly, you'll need to become very familiar with networks and how they work - the best way to do this is to go to a college course about it, or otherwise teach yourself network structure. You must learn all the nitty-gritty details, including packet encryption and transfer, RSA-keys, everything. In short, you need to tool up with all the knowledge there is. It is not an easy road.
