### Post by SagaciousKJB on 2016-01-20
That's usually a permissions problem. Show the results of 'ls -la .Xauthority' or just try 'chmod 777 .Xauthority' to give everyone read/write/execute permissions and see if that changes, if not then it's likely not the file itself.

It doesn't make much sense that chown said it was an invalid user. Try this...

```
chown -R $(whoami):$(whoami) ~/
```

That will insert the current logged in user and say to apply it to their home directory. Just in case you misspelled something or you are using your full name instead of the username.

If you get the same error do...

```
grep $(whoami) /etc/passwd
```

Paste the results here.
