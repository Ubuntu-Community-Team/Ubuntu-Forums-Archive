### Post by Brandon Williams on 2009-05-14
If you write code on many different *nix systems, including ones that you don't administer, then I recommend using either emacs or vi(m), since you will be able to find these on virtually every *nix system you log in to these days. vi is a little bit safer in this regard, but only a little. They are both very capable code editors, and I have not encountered any features in a newer graphical editor that made it look worth switching to. 

If you don't move around to different systems, then it really just comes down to personal preference. I'm sure that you could find someone to recommend almost every possible programmer's editor out there.
