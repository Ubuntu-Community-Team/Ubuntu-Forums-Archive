### Post by distroman on 2007-07-20
I am running NVIDIA-Linux-x86-100.14.09-pkg1.run right now. Ill try update in a bit see how it goes.

Try setting the driver to vesa first.

gksudo gedit /etc/X11/xorg.conf

In Section "Device" try changing the "Driver" to "vesa"
See if you can use <crtl><alt><f1 if not try the following.

1. sudo apt-get install linux-headers-`uname -r` build-essential gcc gcc-3.4 xserver-xorg-dev

2. reboot, (I triple boot so I get the grub menu) think you have to press "esc" on boot then you should be able to pick "recovery mode".

This is not how it's suppose to be done, so it's all on your own time, :)

3. su "username"

4. cd   

5. sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.backup

6. sudo sh NVIDIA-Linux-x86-100.14.11-pkg1.run

7. sudo nano -w /etc/default/linux-restricted-modules-common

Add the following,

8. DISABLED_MODULES="nv nvidia_new"

ctrl+x to save,

sudo /etc/init.d/gdm restart

Hope it works.

Step by step.

```
ctrl > alt > f1
```

```
sudo /etc/init.d/gdm stop
```

```
sudo apt-get install linux-headers-`uname -r` build-essential gcc xserver-xorg-dev pkg-config
```

If the nvidia-installer are placed on your desktop.

```
cd Desktop
```

```
ls
```

You should see it now.

```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.backup
``` 

```
sudo sh NVIDIA-Linux-x86-100.14.11-pkg1.run
```

Follow the instruction, let it configure your xorg.conf

```
sudo nano -w /etc/default/linux-restricted-modules-common
```

Add the following, 

```
DISABLED_MODULES="nv nvidia_new"
``` 

```
ctrl+x
``` 

```
y
```

```
enter
```

```
sudo /etc/init.d/gdm restart
```
