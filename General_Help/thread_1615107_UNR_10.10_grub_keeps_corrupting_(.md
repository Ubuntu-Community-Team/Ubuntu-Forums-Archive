### Post by drs305 on 2010-11-06
I have seen the issue mentioned by *ktoepke* in an earlier post. Ubuntu works fine (repeatedly) until Windows is booted. At that point, an app in Windows (sometimes a DELL app) will write to the MBR and corrupt Grub.

If this only happens after you run Windows, you have a few choices. 

Some available choices:
1. Remove the offending Windows application to prevent it from writing to the MBR; 
2. Install Grub2 to another drive and have BIOS point to that drive first; 
3. Install G2 to a flash drive and tell BIOS to boot the flash drive first if it's inserted. When inserted at boot, you will get your Grub menu. If not, boot to Windows. To do this, you would have to install the Windows boot loader on the hard drive.
