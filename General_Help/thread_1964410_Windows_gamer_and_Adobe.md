### Post by GameX2 on 2012-04-24
Have the same problem, but found my solution to avoid dual-boot (as much as possible).  Either way, you're still forced to use Windows, but that's an alternative solution.  Try running a Windows XP (Or 7) in a virtual machine, in VMware.  I run PhotoShop there (I use it all the time at school, so), on a 8GB of RAM laptop.  Run like a charm (accorded 2GB of RAM to the VM).  However, I can still run Photoshop on a 1.2GB PC, with  700MB of RAM for the VM.  Expect this to be slow, but *usable* if you can't use GIMP, like me.  If you don't have SWAP, expect crashes, on a slow PC, however!

Games are the same, they run great.  However, a few games will still refuse to start in a virtual machine, be aware of that.

The best about VMware, it's his Unity mode (Not referring the Unity interface).  Let's say I run PhotoShop in a windows, this mode hide the Desktop, and you really have the feeling you're running Photoshop IN Linux!  The mode also add a "Start Menu" in the corner of the screen, so you can see your Windows apps, and even drag them to your Desktop (and even Unity menu)!! ;D

When you're done gaming, suspend the virtual machine, so you won't have to wait for the VM to turn ON.
That's the solution I'm using.  Work like a champ!! :D

### Post by WorMzy on 2012-04-24
Another vote for dual booting. It's what I do (more or less, I have nine operating systems installed on this PC, two of which are Windows).

I still play some games on Linux (Sims 3 with wine, Dwarf Fortress, Minecraft, World of Goo, and Katawa Shoujou natively), but the majority of my games work best on Windows, so that's what I use.

As for your other applications; as a web developer myself, I use GIMP for designing webpages and creating images, and vim for writing the pages themselves. If you don't know how to write (x)html/css/javascript/php/etc., or you just prefer to use an IDE, there's several of those available on Linux too.

Conky might be able to replace Rainmeter for you. There's several pre-written config files out there with examples of what you can do with it; from clocks, to system stats, to displaying todo lists, etc..
