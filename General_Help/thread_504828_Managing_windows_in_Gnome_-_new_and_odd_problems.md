### Post by mhenriday on 2007-07-19
I've been running **x86_64 Feisty** with **Beryl** with **Firefox** as my browser ever since the release of the first named in April ; generally speaking I've been very satisfied with ease with which the graphics can be managed. Now, today, I suddenly find that I an unable to manage windows and tabs in the customary manner. I can no longer change the size of an opened webpage (tab) from maximised to a smaller display or close all **Firefox** tabs in a given window by clicking an icon at the upper right of my of the **Firefox** (**2.0.0.4**) menu bar  just below the upper **Ubuntu** panel ; these tasks can now only be performed by right-clicking on the minaturised window icon in the lower panel (task bar). Nor, even when a web page is no longer maximised, can I adjust its size by drawing on it when arrows appear on the edges or in the corners - these are no longer available. Moreover, the red «close» button that formerly appeared in inactive tabs when touched by the mouse pointer now appears only in the active tab (but inactive tabs can still be closed by middle-clicking)....

As mentioned above, I did not notice any difficulties in carrying out these procedures before today, which leads me to suspect that I may have inadvertently modified some important setting(s) when I installed **flash** on my **AMD 64 X2** machine last night according to the procedure outlined by **pinguin** on his [ *Flash for Firefox thread*]("http://ubuntuforums.org/showthread.php?t=341727&highlight=flash+for+firefox"). Could it be possible that some of the procedures mentioned there - or my attempt to download Miro Public Preview [Video] Platform (0.9.8 ) - which, unlike **flash**, I was unable to get to work - has somehow modified my ability to manage windows and tabs ? Thanks in advance for any ideas or suggestions on this matter - not being able to work in the customary way with opened windows or tabs is a pain ! (My ability to enter **Unicode** characters with keyboard shortcuts (Ctrl + Shift + u ; hexcode ; space) has also been affected, but that's a matter for a separate thread....)

Henri
