### Post by |{urse on 2008-05-04
Hard to say. Everything looks correct. I myself am having tons of issues with my ati radeon x1600, one thing that did help was to use the proprietary radeon driver, this may make things better for you and on the other hand this may make things worse. It's up to you whether you want to risk it.

now make a backup of your xorg.conf just in case.

> sudo nautilus /etc/X11/

> right click and copy your xorg.conf , paste it on your desktop and rename it to xorg.conf.backup
> 
WRITE THIS DOWN! if you need to restore your xorg.conf from a command line after rebooting

> sudo cp /home/yourusername/Desktop/xorg.conf.backup /etc/X11/xorg.conf

okay now go download the ati drivers.

[http://ati.amd.com/support/drivers/linux/linux-radeon.html](http://ati.amd.com/support/drivers/linux/linux-radeon.html)

open a terminal and 

> sudo apt-get update

sudo apt-get install build-essential fakeroot dh-make debhelper debconf libstdc++5 dkms linux-headers-$(uname -r)

cd Desktop

chmod +x ati-driver-installer-8-4-x86.x86_64.run

sudo sh ati-driver-installer-8-4-x86.x86_64.run --buildpkg Ubuntu/hardy


now you need to block the preinstalled fglrx

> sudo gedit /etc/default/linux-restricted-modules-common

now you need to add "fglrx" in the empty quotes

> DISABLED_MODULES="fglrx"
like that

then
> sudo dpkg -i xorg-driver-fglrx_8.476*.deb fglrx-kernel-source_8.476-0*.deb fglrx-amdcccle_8.476-0*.deb


and lastly

> sudo aticonfig --initial -f

reboot and hope for the best

WRITE THIS DOWN! if you need to restore your xorg.conf from a command line after rebooting

> sudo cp /home/yourusername/Desktop/xorg.conf.backup /etc/X11/xorg.conf
