### Post by foxy123 on 2006-04-16
I always had ATI. The last one was Radeon 9550. The most annoying thing with it was a random freeze problem when X just freezes while mouse, sound and other stuff still running. The way I solved it with Radeon was the following: I downgraded AGP from x8 to x4, disabled fast-write and increased AGP voltage. After that freezes stopped. However I still was not able to have dri and composite together and everybody was saying that nVidia has better support for Linux overall. After it started crashing on Windows (my desktop is dual-boot) and decided that I was done with ATI and bought nVidia 6200A looking forward to better Linux experience and posibility to get xgl and compiz working on my desktop. Alas it's not happened.

Now I am facing the same random freeze problem. I used the thread in HOWTOs to install the latest driver (method 2) and I put in my xorg.conf:
```
Option "NvAGP" "0"
Option "RenderAccel" "Off"
Option "IgnoreDisplayDevices" "DFP,TV"
Option "NoRenderExtension" "Off"
Option "Accel" "Off"
Option "AllowGLXWithComposite" “Off”
EndSection
```

It has not changed a thing. My desktop still freezes after several minutes of work. 

Now I am wondering if it is a card problem or maybe it is my motherboard. Is it possible that there is some incompatibility with it as both cards behave the same.

I have MSI KT6V-LSR (VIA® KT600 Chipset). If it is a motherboard, does anyone know which motherboard work fine with 6200A?
