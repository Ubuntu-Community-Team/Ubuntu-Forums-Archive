### Post by ankspo71 on 2009-09-07
"open in terminal" is practically the same as using a terminal and running/opening the file, except "open in Terminal" is limited to only running or opening the file. What I mean to say is, there are more things you can do with a file in the terminal itself, than you could by using "open in terminal". "Open in terminal" does come in handy for the files that require it, such as an installer with an interface that only runs in a terminal, or for diagnostic reasons. I don't think anything changes about the way your scripts will run or change any settings though. "open in terminal" would be good to use to check for errors etc. I believe it is also there for time saving reasons... so people don't have to open a terminal and type every time.
