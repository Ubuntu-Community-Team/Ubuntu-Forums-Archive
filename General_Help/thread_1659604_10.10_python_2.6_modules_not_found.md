### Post by Micronaut on 2011-01-04
Hello,

I am trying to import some Python modules into Ubuntu 10.10 (PySCeS and PyDSTool).  If I download the modules and extract them into a directoy in my home directory, open that directory in terminal, and then import them into python (>>>import module_x) then it loads fine.  However, if I am outside of the directories then it always returns an error saying the module is not found.  I have tried putting the modules into usr/local/lib/python2.6/site-packages or dist-packages and usr/shared/pyshared etc but it never sees any of my modules.  It also does this with modules installed from the repository.  They are always "not found" but if placed manually in the working directory they are.

Does anybody know where Ubuntu is looking for python modules, or how to get python to pick up my own installed modules?

thanks for any assistance.

### Post by Cheesehead on 2011-01-04
```

>>> import sys
>>> sys.path               # These are the directories where python looks
['', '/usr/lib/python2.6', '/usr/lib/python2.6/plat-linux2', '/usr/lib/python2.6/lib-tk', '/usr/lib/python2.6/lib-old', '/usr/lib/python2.6/lib-dynload', '/usr/local/lib/python2.6/dist-packages', '/usr/lib/python2.6/dist-packages', '/usr/lib/python2.6/dist-packages/PIL', '/usr/lib/python2.6/dist-packages/gst-0.10', '/usr/lib/pymodules/python2.6', '/usr/lib/python2.6/dist-packages/gtk-2.0', '/usr/lib/pymodules/python2.6/gtk-2.0', '/usr/lib/pymodules/python2.6/HTMLgen']

>>>sys.path.append('/path/to/new/directory/')    # Add a new directory
>>>import mymodule                              # The module is in the new directory, of course

```

### Post by Micronaut on 2011-01-07
Thanks for your help, the code was useful.

Python couldn't see the module inside a directory listed by sys.path.  It must not be able to scan recursively through the paths.

When I manually added the path, including the sub-directory it imported the module.

> **Cheesehead said:**
> ```

>>> import sys
>>> sys.path               # These are the directories where python looks
['', '/usr/lib/python2.6', '/usr/lib/python2.6/plat-linux2', '/usr/lib/python2.6/lib-tk', '/usr/lib/python2.6/lib-old', '/usr/lib/python2.6/lib-dynload', '/usr/local/lib/python2.6/dist-packages', '/usr/lib/python2.6/dist-packages', '/usr/lib/python2.6/dist-packages/PIL', '/usr/lib/python2.6/dist-packages/gst-0.10', '/usr/lib/pymodules/python2.6', '/usr/lib/python2.6/dist-packages/gtk-2.0', '/usr/lib/pymodules/python2.6/gtk-2.0', '/usr/lib/pymodules/python2.6/HTMLgen']

>>>sys.path.append('/path/to/new/directory/')    # Add a new directory
>>>import mymodule                              # The module is in the new directory, of course

```
