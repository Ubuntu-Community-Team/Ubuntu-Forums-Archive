### Post by fenomenoxp on 2007-01-03
Hi everyone
I had the same problem. I have found a workaround to solve (or at least hide) the problem.
I read that for some people the problem dissapears when they open nvidia-seetings. I tried that and it worked for me. But the next time I booted the problem was there again. So what I did was execute nvidia-settings in the background with some "stupid" argument, for example:

```
nvidia-settings --assign CursorShadowBlue=0
```

You can set up a script to execute this on startup (when logging in X, or in ~/.kde/Autostart). That will make the problem dissapear as soon as you start your session

### Post by lferree on 2008-07-12
It could be the memory.

I've always had my RAM set to default/AUTO settings in BIOS.  One day I just happened to notice it was set at 333FSB, when in fact, it should have been 400FSB.  I manually changed this thinking nothing other than cool, maybe my system will be a little faster.

Pretty much forgetting I had done that, I started noticing some strange behavior, random reboots, apps closing for no apparent reason and the red and blue pixels appearing in all videos (even when running VLC).

If you made some RAM changes in your BIOS recently check this.  If not, check it anyhow, could be some RAM going bad.  I would just try lowering the FSB by one setting.

I'll definitely be contacting Corsair, as my RAM has a lifetime warranty and it wasn't cheap at the time.  Good news is RAM's dirt cheap now, if you don't have a warranty or want to run down to the computer store and get something fast.

Hope this helps.
