### Post by skoal on 2005-04-30
1. Add 'agpgart' and 'intel-agp' to the hotplug blacklist: (ie.)

in /etc/hotplug/blacklist/
[...]
# To use Nvidia supplied agpgart
agpgart
intel-agp

2. Change this option under 'Section Device' in xorg.conf to 1:

Option          "NvAGP" "1"

* After you reboot, you can verify that you are using the Nvidia agpgart by issuing the following:

skoal@morpheus://~ $ cat /proc/driver/nvidia/agp/status 
Status:          Enabled
Driver:          NVIDIA
AGP Rate:        4x
Fast Writes:     Disabled
SBA:             Disabled

* you'll notice the 'NVIDIA' under driver.  That's what you want...

### Post by ritger on 2005-05-01
nvidia_agp actually needs agpgart.ko to be loaded, before you can use it. This is a change from the old 6xxx drivers to the latest 7xxx nvidia driver. Typical setups will use agpgart and a specific driver for your motherboard. Look up the name of this kernel module driver (in my case via-agp). Add the name of this driver to your /etc/hotplug/blacklist file, to prevent it from being loaded at boot time. 

Add both nvidia and nvidia_agp to /etc/modules, and make sure you put nvidia_agp on a line before nvidia. Now reboot and you should be using the nvidia kernel module and agp interface. You can change the NvAGP line in xorg.conf to either 1 or 2 (look in the README for the nvidia driver what they both mean), but once the nvidia_agp driver is loaded either option works.

You can check for module names by using lsmod. Hope this helps.
