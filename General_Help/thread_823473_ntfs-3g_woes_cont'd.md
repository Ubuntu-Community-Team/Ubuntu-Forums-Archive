### Post by danwood76 on 2008-06-10
I would say sda1 is your 'Music', sdc1 is your 'WIN', and sda5 is your video store.

We will edit your fstab to reflect this, also changing the settings to allow ntfs writing on these partitions.

So open the fstab for editing (root user privs):
```

sudo gedit /etc/fstab

```
Then we want to replace the last 3 lines with this:
```

#/dev/sda1
UUID=82444AB2444AA92D /media/Music ntfs-3g defaults,locale=en_CA.UTF-8 0 0
#/dev/sda5
UUID=4CFC48C2FC48A7D2 /media/VIDEOSTORE ntfs-3g defaults,locale=en_CA.UTF-8 0 0
#/dev/sdc1
UUID=AE5052F35052C22F /media/WIN ntfs-3g defaults,locale=en_CA.UTF-8 0 0

```
Save it and reboot, then your drives should load on boot with write access.

You can see from your fdisk output and your fstab that your problems are caused by drives swapping block dev IDs.
This should solve your problems.

### Post by danwood76 on 2008-06-12
I've just re read your outputs to the commands I posted and I have made a slight typo.
Your music store is a FAT16 drive and so should be mounted with vfat instead of NTFS. (you might want to convert it to NTFS in windows as FAT16 isnt a good fs type)
I also made a typo on one of the lines.
The last lines in your fstab _should_ look like this:
```

#/dev/sda1
UUID=82444AB2444AA92D /media/Music vfat defaults,locale=en_CA.UTF-8 0 0
#/dev/sdc5
UUID=4CFC48C2FC48A7D2 /media/VIDEOSTORE ntfs-3g defaults,locale=en_CA.UTF-8 0 0
#/dev/sdc1
UUID=AE5052F35052C22F /media/WIN ntfs-3g defaults,locale=en_CA.UTF-8 0 0

```
By the way the lines with a # infront are ignored by mount when loading the fstab.
