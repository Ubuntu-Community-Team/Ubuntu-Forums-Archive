### Post by Dylan1473 on 2012-08-05
**Regarding graphics:**

Avoid AMD Radeon graphics cards, their support in Linux is terrible. AMD CPUs are fine, but don't go for Radeon graphics if you want to use Linux for gaming or if you mind potentially having to work through some serious graphical issues (that's not guaranteed; I have a Radeon card and I haven't had too many problems outside of graphically demanding applications but when it comes to games and stuff *I have had some serious problems*). Otherwise I don't know too much about graphics cards; I'm a gamer but I'm not really a hardcore graphics enthusiast and I'm perfectly happy with medium or low quality.

**Regarding CPUs:**

1.5Ghz per core on a processor isn't very good. Many cores is definitely good as is more total speed, but you want a reasonably high speed per core since not all applications are capable of using multiple cores. I couldn't tell you exactly what ratio you're looking for, unfortunately (e.g. is a dual core 2.5Ghz processor better or worse than a quad core 1.5Ghz? No clue, probably depends on what you specifically want to do). Note that gaming actually does tend to make really good use of multiple cores, though I'd still say shoot for a bit higher than 1.5Ghz.

**Regarding Operating System:**

It's also worth noting that if you don't want Windows on your computer (and maybe you do) you can sometimes get cheaper  operating system free computers and install Linux on them later. Most people don't think of it since it generally just comes with your  computer, but Windows costs money and can easily add an extra $100 odd  onto the price. Of course, maybe you do want Windows as well - that's up to you. If you're planning on using Windows for gaming, for instance, you don't need to worry about the graphics card as much.

**In Summary:**

As you've probably pretty much guessed (what with it being more expensive and all) the third one is the best computer there with a nice, Linux friendly Nvidia card and a faster processor than the other two. If you were to pick between the first two, I'd say go for the second one for sure - the main thing it's missing is 2 gigs of RAM and you can probably upgrade that yourself later if you even need to. 6GB should be plenty though I'd think, I've done alright with 3GB.

**An Addendum Regarding Wireless:**

Another thing you'll want to take note of: find out the wireless cards in those computers and what drivers they use. Linux supports most wifi cards to some extent, but some may be more of a headache than others to setup and most others won't have some particularly interesting features supported. I lucked out and this laptop, the first I've ever used Linux on, had a beautifully supported Atheros based wireless card that supports master mode and basically every other wireless card feature that Linux offers.

**An Addendum Regarding Manufacturers:**

I'm really happy with my Acer laptop that I'm using now (though I have to upgrade soon for school) and would certainly recommend them as a manufacturer. The awesome-for-Linux wireless card I've got is just a stock Acer one. I don't know if they're all like that, of course.
