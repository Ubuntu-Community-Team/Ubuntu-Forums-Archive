### Post by waltd on 2017-01-29
Hi all,
    My OS is Xubuntu 16.10
    I can use the default wallpapers and choose between them without a problem.  But, when I choose a folder with my pictures in it to use as wallpaper, all the picture choices are grayed out.  It was working perfectly yesterday but I don't know what I did to break this function.   
    For example, I navigate to my photo folder by:
        Desktop Settings - Folder - Other - wallpaper (where my photos are located).  All the picture choices are grayed out and none can not be selected.  Any ideas on how to fix this?

---

### Post by gdesilva on 2017-01-29
Worthwhile checking whether you still have read access rights to the folder and the files located in it.

---

### Post by waltd on 2017-01-29
Thanks for the suggestion. Permission probably has an aspect to the problem.  But after spending some time in Google I found an easy way to make my photo's the wallpaper.  Right click on the photo and select from the context menu "select as wallpaper".  It's easy and it works.  I can use any of my photos into wallpaper.

### Post by Keith_Helms on 2017-01-30
I had the same problem under Xubuntu 16.04.  There was a thread back in November about this kind of problem and I posted my workaround:

> For some reason I had to copy images I wanted to use as wallpaper  to /usr/share/xfce4/backdrops and change the ownership of them to root  in order to get settings manager->desktop->background to open  them.  The background configuration screen doesn't freeze on me, but  when I select a folder such as /home/keith/wallpaper the jpgs there are  ghosted out and I can't select them.
