### Post by techbrainless on 2012-10-17
Thanks a lot albandy for your reply ,

1. [B]one important point to mention is that :
    * ]project-open[ is not a project scheduling software.  ]po[ does not not replace Microsoft Project or other scheduling software such as OpenProj, GanttProject etc. Instead, ]po[ integrates with this type of software, allowing project plans to be made available and published for all stakeholders inside and outside the company.
this has been mentioned in their main website  : [http://www.project-open.org/en/faq_general](http://www.project-open.org/en/faq_general)[/B]

2. I have already tried ]ProjectOpen[ , it is very nice but it does not support all the my feature requirements such as :

*Scheduling* : for detailed scheduling we need to import .gan or .xml files which means that you have to make your detailed schedule in another application like Gantt Project or Planner .

* Resource Management* : does not support it , it allows the resource(employee) to work in multiple projects without making any notifications that the actual working hours is greater then schedule working hours ,  for example :
if employee 'A' schedule working hours is only 10 hours per day , if the employee 'A' spent 6hours in project 'A' , then the employee is available for only 4hours , if this employee has been assigned to work in project 'B' for 5 hours daily , then some mark should be shown stating that this employee is overdue because he is going to spend 11 hours on project A and B and his schedule time is only 10 hours.

* Base line* : does not support this feature.
