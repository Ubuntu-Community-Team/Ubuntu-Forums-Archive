### Post by kodoku on 2007-02-03
Beryl is a fork or branch of Compiz.  Basically some developers of Compiz developed a branch that was too different than the original Compiz started by Novell, and after the Novell XGL/Compiz team refused to merge the the branch with the original, Beryl was founded.

Functionality wise, I'd say they're the same.  Compiz might be more stable than Beryl, considering the former is almost always in Beta stage development.  But I've been using Beryl for a long time without ever seeing crashes or other graphical mishaps.

I'd say stick with Beryl unless you're having major issues with it.

### Post by glabouni on 2007-02-03
compiz is openGL 3D acceleration coming to Xwindows. it's been released by novell in january of 2006.

first there was compiz-quinnstorm, a fork of compiz. When the compiz team refused to merge quinnstorm changes into compiz main branch, the decision was made to make a real differentiation and compiz-quinnstorm has been renamed beryl.

list of differences between compiz and beryl can be found in beryl FAQ:
[http://wiki.beryl-project.org/wiki/Beryl_FAQ#What_is_the_difference_between_freedesktop.org_compiz_and_beryl.3F](http://wiki.beryl-project.org/wiki/Beryl_FAQ#What_is_the_difference_between_freedesktop.org_compiz_and_beryl.3F)

on one hand compiz is kinda more stable than beryl but lacks features beryl have.
compiz is more stable only because beryl changes frequently with new features added on a daily basis. apart from that beryl is quite stable. if you stick only to milestone release and don't get updates as soon as they're released beryl would as stable as compiz.
