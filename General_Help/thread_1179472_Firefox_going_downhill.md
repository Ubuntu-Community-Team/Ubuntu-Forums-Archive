### Post by Rallg on 2009-06-05
I have never seen the profile problem described in the original post. Could it be that on the particular computer, some critical directory (such as ~/.mozilla) has been inadvertently assigned to root rather than to user, or has limited access?

FF is not the fastest browser, but it is not bad. However, I have found that a lot of commercial web pages do not define the dimensions of included content (images, Flash, content inserted by third-party servers). Then, when the numerous servers respond with the included content, the page must reflow and reflow again each time new material arrives. This is very poor web coding on the part of whoever does the pages, but it is common.

Try Midori browser. It uses WebKit (similar to Safari). I first found it for my Nokia N810 Internet tablet, which uses Maemo Linux, and microB (a Gecko-based browser, downscaled from FF) as its default. since the power of the little machine is limited, I found it annoying to deal with pages that reflow 20 times. In Midori, I set it so that it does not load any content other than formatted text by default, and does not run scripts. Pages that took a minute to load in FF take maybe 5 seconds to load in Midori, only because it is not loading tons of garbage content. If I need to see the garbage, I use FF.

Midori is also available for Ubuntu.
