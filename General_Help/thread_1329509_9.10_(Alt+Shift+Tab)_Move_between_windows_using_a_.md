Mystery solved. "Extra Visual Effects" will disable that shortcut, switching to "None" gives the shortcut back =D>

### Post by mdemuro on 2009-11-22
Found in 
[https://bugs.launchpad.net/ubuntu/karmic/+source/metacity/+bug/150702](https://bugs.launchpad.net/ubuntu/karmic/+source/metacity/+bug/150702)

Use gconf-editor to set
/apps/metacity/global_keybindings/switch_windows_backward to "<Alt><Shift>Tab"
