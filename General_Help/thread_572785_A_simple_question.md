### Post by HermanAB on 2007-10-10
Well, the more things you change, the more things will go wrong.  This is due to the first law of thermodynamics: Entropy always increases.

Checksum failures on RPM files can be because the package maintainer forgot to update the checksums, or it can be because the CD is bad - hard to tell which.  For this reason you always have to test your install CDs before you start to use them.

So, you can leave the system as is and it may fail since whatever file it was is bad, or you can upgrade it and then it fails because the upgrade is bad, or you may be lucky and all works out in the end.  I suggest you go and get some coffee and a doughnut and see what happens...

BTW, if you get fed-up with the Suse forums, but like a Suse-like system, then you should get the latest Mandriva (2008 was released yesterday, so now is the time to join the download frenzy!) and subscribe to the Mandriva forum, which is very active.  

You could post your Suse questions in the Mandriva Community Chat forum which is open to anything and since it is very similar to Mandriva, someone should be able to help you:
[http://forum.mandriva.com](http://forum.mandriva.com)

I run various systems, including Ubuntu and Mandriva, but no Suse.  If you get badly stuck, post me a private message and then I'll give you my real email address.

You can also try the 'Other OS' forum here on Ubuntu forums.

Cheers,

Herman
