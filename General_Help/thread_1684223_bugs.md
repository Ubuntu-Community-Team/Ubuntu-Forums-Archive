### Post by 3rdalbum on 2011-02-08
I think you're getting confused between bugs and viruses. A bug is an unintentional flaw in a piece of software. A virus is a malicious piece of code that spreads from computer to computer.

You don't need to worry about viruses on Linux - there aren't any.

If you are having trouble with your computer being slow and crashing, the first thing you should check is your computer's RAM. You can try the Memtest86 program from the GRUB boot menu, but this won't always point out bad RAM; I tend to recommend using the Phoronix Test Suite memory benchmark, which usually crashes when the RAM is faulty.

Or, better yet, take out a RAM stick and see if you get crashes; if you do, then try taking out a different one, etc.

If it's only one program that crashes, then that program might have a bug (unintentional coding flaw) in it, in which case you should try submitting a bug report to the developers of that program.
