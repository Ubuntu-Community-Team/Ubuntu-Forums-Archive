### Post by Mark Phelps on 2012-03-03
If you still can't boot into Win7, you may be able to fix that with Boot-Repair (details below)...

To do this, do the following:
1) Download the Boot-Repair ISO and burn that to CD
2) Disconnect the Ubuntu drive
3) Boot into Boot-Repair and use it to repair the Win7 boot
4) Remove the CD and reboot -- still with only the Win7 drive connected.  You should boot OK into Win7 now.

Boot-Repair info:  [http://ubuntuforums.org/showthread.php?t=1769482&highlight=boot-repair]("http://ubuntuforums.org/showthread.php?t=1769482&highlight=boot-repair")

### Post by oldfred on 2012-03-03
Not sure that Boot-Repair can fix the missing files. Windows 7 normally has a separate boot partition of 100MB with two boot files and the boot flag. You have the 100MB NTFS partition with the boot flag but bootmgr and BCD are missing. Often a Windows repairCD or USB is needed to replace those files. One user just copied the bootmgr & BCD to his computer and edited the BCD with BCDedit to repair his system.

Boot files:
WinXP
/boot.ini /ntldr /NTDETECT.COM
Vista/7 (with 7 the first two files are usually in a separate 100MB boot partition)
[COLOR=Red]/bootmgr /Boot/BCD[/COLOR] /Windows/System32/winload.exe 

You can use any Windows repair CD that is the same 32bit or 64bit version. So if you do not have one, find a friend and make one from their system.

Make your own Windows repairCD (not vendor recovery):
