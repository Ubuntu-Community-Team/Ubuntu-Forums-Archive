        Option          "ModeDebug"                     "True"

### Post by CatKiller on 2009-07-22
> **megatenplayah said:**
> where would it show me why it's being rejected?

Normally before this bit.

```

(II) NVIDIA(0): Validated modes:
(II) NVIDIA(0):     "1024x768+0+0"
```

There's normally a list of all the modes that your screen can possibly do, and then any invalid ones are rejected. Possibly because you've got the MetaMode specified, it hasn't bothered. I must admit, I don't know how they work.

You've also got this in there, too.

```

(WW) NVIDIA(0): Option "ModeDebug" is not used
```

which is quite strange, since the nvidia driver definitely used to support that option; I've used it myself. Perhaps the documentation says why.

Actually, the documentation (/usr/share/doc/nvidia-glx-180/README.txt.gz) says that it does still work:
> 
Option "ModeDebug" "boolean"

    This option causes the X driver to print verbose details about mode
    validation to the X log file. Note that this option is applied globally:
    setting this option to TRUE will enable verbose mode validation logging
    for all NVIDIA X screens in the X server.

Having read a bit of the documentation, commenting out all of that TwinView stuff would probably be a good idea, including the MetaMode setting.

I noticed that you're using the 173 driver rather than the 180 that I'm using. You might want to check your version of the documentation to see if there are any significant differences between yours and mine. Chapter 13 describes TwinView and MetaModes to some extent.
