### Post by GSF1200S on 2010-05-02
While I see that Grub2 is installed to the MBR (Grub2 is the bootloader that allows you to select which OS to boot, and the MBR is where the computer checks to boot the OS bootloader), I dont see a linux install anywhere.

I see a number of partitions, so Im assuming you have a seperate C and D drive on Windows, and as well perhaps a recovery partition and maybe a Windows 7/Vista install? It appears that Windows XP is installed on /dev/sda1. If you have Windows 7, the best thing to do would be to use the Windows 7 partitioner and create a partition for Ubuntu, then tell Ubuntu when you install to format that partition using the manual partitioner. If you have only Windows XP, which of these partitions are you wanting to resize/install Ubuntu on?

So, for example (this is an example- I dont want you to break anything, so ask for clarification to make sure no data gets lost :) ), if /dev/sda1 is your windows install, and you want to install Ubuntu on /dev/sdb1, you would tell Ubuntu to install on /dev/sdb1 and tell Grub to install to the MBR of /dev/sda. Im gonna post this and see if this makes any sense to you. I need to know:
1) How many hard drives do you have, which ones windows are on, and where you want Ubuntu.
2) Which partitions are you going to resize to make room for Ubuntu?
3) Which hard drive are you booting from? (Grub is installed to /dev/sda, but you say it doesnt even recognize Grub and boots straight to Windows- could this be because you are booting from a different harddrive?)

Im sorry if this seems horribly complicated, but I dont want to make you lose anything and I want it to be setup how you want it.

### Post by GSF1200S on 2010-05-02
Download the 10.04 alternateCD. Install Ubuntu on whatever partition you like. When it asks you where to install grub, tell it to install to /dev/sdc instead of /dev/sda or the drive you installed the root partition on. Make sense? Only the AlternateCD has this option according to the guys in the IRC channel.

Ive never had this issue because my windows install (which i never use) is installed on /dev/sdb along with all three of my Linux installs. Try it that way, and if you still have an issue, report back.
