### Post by scarolan on 2008-06-19
I've been using Ubuntu for a couple of years now, and work full time as a Linux system administrator.  I had really hoped that Ubuntu would be a respectable desktop system but now I'm starting to have doubts.  Until we can get flawless, consistent display and monitor configuration there will never be a mass market for Ubuntu on the desktop, especially in the business world where external monitors and projectors are a necessity.

Anyway, here is the problem.  A little over a week ago I upgraded this Thinkpad T43 system to Ubuntu 8.04, after having run Hardy on my home system for a few months.  I figured that since it was fairly stable on the home box that I might be ok with upgrading my work laptop.  Well, this turned out to be a big mistake.  After the upgrade my external monitor does not work anymore, whereas before it worked perfectly with no extra configuration on my part.  I'm sure this was not done on purpose, but it's extremely frustrating to spend an entire day fscking around with Xorg settings just to get basic functionality that Windows users have had for 15 years or more.

At this point I'd even be happy just to use the external monitor without dual-screen but it won't work at all.  The annoying part is that when I first boot up, the external monitor shows the Ubuntu logo so I know it is not a hardware problem.  The docking station works fine, and I can get dual-monitor when booted into Windows XP.  It's just when I get to the GDM login screen that the external monitor goes blank and ceases to function any further.

I have tried every permutation possible with the display resolution and displayconfig-gtk settings.  I have tried both the proprietary and free driver and nothing makes the external monitor work.

Has anyone else experienced the same thing?  If so were you able to fix this?

I'm really quite fed up with this - I thought we were past these problems, especially on Linux-friendly hardware like the Thinkpad line.  I sincerely hope these hardware compatibility issues can get prioritized ahead of new bells and whistles in the operating system.
