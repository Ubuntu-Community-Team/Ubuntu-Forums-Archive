### Post by erginemr on 2008-06-26
OK. I assume the visual tools above didn't work. 

Based on this example:
[http://bugs.debian.org/cgi-bin/bugreport.cgi?bug=428711](http://bugs.debian.org/cgi-bin/bugreport.cgi?bug=428711)
[http://packages.ubuntu.com/hardy/xserver-xorg-video-siliconmotion](http://packages.ubuntu.com/hardy/xserver-xorg-video-siliconmotion)

1. First save your current xorg.conf file with:
```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.working
```

2. Then edit the file with:
```
gksu gedit /etc/X11/xorg.conf
```

3. Add the following lines accordingly:

```

...
Section "Device"
Identifier "Configured Video Device"
[color="red"]**Driver		"siliconmotion"[/color]**
EndSection

Section "Monitor"
Identifier "Configured Monitor"
EndSection

Section "Screen"
Identifier "Default Screen"
Monitor "Configured Monitor"
Device "Configured Video Device"
[color="red"][b]DefaultDepth	24
SubSection "Display"
	Depth		24
	Modes		"1024x768" "800x600" "640x480"
EndSubSection[/color][/b]
EndSection
...

```

4. Save the file and restart your computer

5. In case you happen to find yourself in a worse situation, then restore your backup file with:
```
sudo cp /etc/X11/xorg.conf.working /etc/X11/xorg.conf
```
and restart your computer again.

Good Luck!...
