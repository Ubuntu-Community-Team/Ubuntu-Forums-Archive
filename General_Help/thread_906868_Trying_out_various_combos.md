### Post by Ethyrdude on 2008-08-31
Ever since I first installed Ubuntu a couple of weeks ago, I've been struggling with Gnome, I just am not happy with Compiz-Fusion, it does do some cool looking things but it's not worth the bother of having to do so many hard resets, so I was trying out a few different things.

I did install Kde, just to make sure that it wasn't something else causing the lock ups, and I had no problems with it but I just am not a Kde person.  Gnome runs fine with Metacity but that is kind of boring, I do like a little bling and so I decided to try enlightenment.  E17 still seems to be out in the beta frontier somewhere so I installed e16, and first tried it out on it's own but I really hate the menus, because they propagate wherever you click, if you happen to select something with a lot of choices, you tend to easily loose focus of the menu and end up clicking on the wrong selection.

I then tried it first with just the gnome panel and then by running Gnome with enlightenment for the windows manager.  This was OK too, but just didn't do it for me.  I also tried Xfce during this time but after about a half an hour I decided not to keep it, there was no spark.

So I installed Fluxbox, I've used it in the past, I like the way you can stuff applications together on the top of the desk, but then something weird happened, while I was playing around, Gnome started on top of fluxbox and although it disabled the fluxbox menus, I did get a Gnome panel started and I think I can actually live with this combo.  It's kind of a happy accident because I'm not fond of having to chase my menu around in fluxbox either and making hotkey shortcuts wasn't something I was looking forward to.

OK, so if you start Nautilus in fluxbox, Gnome thinks you want to start it and so it does.  So to make it happen automagically I uninstalled enlightenment, removed it from the session box and added fluxbox as a program I wanted to start when starting Gnome.
