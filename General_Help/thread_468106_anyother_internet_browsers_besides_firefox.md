### Post by meatpan on 2007-06-08
lynx and links are useful because they are independent of the graphics subsytem.  

Text browsers like this can be very handy when X crashes.  They require some extra app knowledge of keys and commands, and many sites are not written to be visualized in a text browser, so they probably aren't going to be your every day browser.  However, they are invaluable for small tasks and substituting while you are debugging problems with a graphical browser.

### Post by goodtimetribe on 2007-06-08
I like Opera, but there's times where it's not 100% compliant with all web pages. Namely gmail's interface to gtalk. That one really irks me. That leaves me using multiple browsers concurrently, and that's a nuisance. I like Konqueror. 

It's pretty nice, but there's times when I prefer Opera or Firefox. Opera's Speed Dial is nice, but can easily be replicated by any bonehead with an idea of how to make a 3x3 HTML table and a couple of links. I also like Opera's widgets. Goodie for it. The other good thing about opera is the scaled to text page zooming. When zooming a web page, the images are scaled in the same proportion to the text, so the page looks more "natural" when viewing it with an increased text size. Try it. It's awesome.

I like Galeon. It's nice. It's fast. It's compliant.

Konqueror's hard to beat for file manager integration.

I'm still kind of trying them out to see where I land, but it seems I'm still having to use a couple to do everything I want.

This is a good thread though. I've gotten some recommendations I haven't seen before, so I'd like to see some more suggestions.
