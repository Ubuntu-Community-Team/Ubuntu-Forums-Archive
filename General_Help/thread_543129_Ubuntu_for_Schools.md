### Post by bigboy_pdb on 2007-09-05
Microsoft stopped supporting Windows 2000 years ago. That's most likely why the school is having so many problems with their computers so yes they should change their systems. The problem is that you're going to have to know about networking and setting up a Linux server in order to replace those systems with a Linux system.

What you should do is based on a number of things. Is it an elementary school or a secondary school? What is your relationship with the school (i.e. student, teacher, principal, or director)? What do you know about Linux? Do you know anyone who knows a lot about Linux?

If the school is a secondary school then I don't think that you should use Edubuntu because a good portion of it is software for kids (and older kids and teenagers may be turned off by this). If you're working with the school and you will be there for a long time then installing Linux may be a good thing, but only if there will always be someone to work with the system who knows what he/she is doing. If you don't have experience with Linux then you'd probably need to hire someone who knows what he/she is doing. However, if you want someone with knowledge about servers and Linux systems this could be costly. If you're a business person who's trying to sell Linux as a free alternative and you haven't done anything like this before then you're going to make both Linux and yourself look bad.

### Post by bigboy_pdb on 2007-09-05
ev5unleash1, the main difference between Ubuntu, Kubuntu, and XFCE is the desktop environment along with some of the packages that are installed. For example, Ubuntu installs all/most of the software in the Open Office suite but XFCE only installs the word processor. The main difference between Edubuntu and Ubuntu is the packages that are initially installed. On the Edubuntu website (in the Edubuntu FAQ) there's emphasis on how Edubuntu installs a number of educational programs that aren't installed with Ubuntu.

If your school were to use a given flavour of Ubuntu, I think that your school should make their own installation CD which includes whatever software they'll need. They could add packages that they want to the chosen flavour of Ubuntu or remove packages that they don't want. You can find out how to do that here:
[https://help.ubuntu.com/community/InstallCDCustomization](https://help.ubuntu.com/community/InstallCDCustomization)

It might be a good idea to perform some tests when a long school break approaches. Students could save their computer contents and the school computer staff could take some images of the hard drives. They could then install and set up Linux systems. If there's a problem with their Linux system, they can restore the hard drives to their original states. Alternately, they could try to set up a secondary Linux system and have some of the students and faculty use the one system over the other system in order to get an idea of how this transformation would affect the school.

It also wouldn't be a bad idea to look into using other Linux systems or Unix/BSD systems. If company support is important, you might want to look at Red Hat Enterprise Linux as a possible option. You can find information on their academic edition here:
[http://www.redhat.com/solutions/education/](http://www.redhat.com/solutions/education/)

I think that it's important that you do an analysis of the long term costs of fixing the current system, implementing a full fledged Linux system, and implementing systems that may be a combination of Linux and Microsoft operating systems. Fixing the current system might be easier and less costly than creating a whole new system. If you do this, you could still have a secondary Linux system in order to give the students and faculty a chance to learn to use Linux so that you can later switch to using Linux/Unix/BSD systems without any problems when you can no longer use software from the Windows 2000 Software Suites.
