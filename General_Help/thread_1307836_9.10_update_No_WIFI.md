### Post by dummy910 on 2009-10-31
just fixed this same issue because the user did an "unattended"(let the machine upgrade while they went to work) upgrade and missed one of the blacklist questions during the upgrade, so here goes...

let's see if the system even fired up your network card, in a terminal do:```
lshw -C network 
```
now look for your wireless card and look for the word UNCLAIMED in the section that shows your wireless nic.

**if** it says **UNCLAIMED**, go to the next **step A** below, if not, try **step B**.

* [color=red]step A[/color] - let's say you have an [color=blue]Atheros[/color] Wireless Card, you would type into the terminal:```
cd /etc/modprobe.d
```
then ```
sudo gedit blacklist-[color=blue]ath_pci.conf[/color]
``` ([color=green]note[/color]:**if** you have a broadcom, just ls the folder in terminal and you'll see a [color=blue]bcm****.conf[/color] file to open with gedit)
once gedit opens the file, you'll see a _blacklist [color=blue]ath_pci[/color]_. Now put a **#** in front of that line, reboot and see if your wireless nic came back to life.



* [color=red]step B[/color] - let's try installing the backports package. In a terminal type ```
sudo apt-get install linux-backports-modules-karmic

```, then reboot and you *might* have to do step A if the card does not fire up, so bookmark this page.

### Post by delcypher on 2009-10-31
It is possible to connect to wireless networks via the command line. This is useful because you can keep track of errors that appear.

run iwevent in another terminal to monitor what's happening.

In these examples I've used wlan0 as my wireless interface please replace this with your wireless interface name (run iwconfig to get a list)

0. I forgot about this step when I was writing this guide and I can't be bothered to re number all my steps so this is step zero. We need to disable NetworkManager so it doesn't interfere. 
```
sudo /etc/init.d/NetworkManager stop
```

1. Bring your wireless interface up.
```
sudo ifconfig wlan0 up
```

2. Do a scan for wireless Networks and find the ESSID you want to connect to and the encryption type. Make a note of this information
```
iwlist wlan0 scan
```

3. Set the ESSID where YOUR_ESSID is the ESSID you want to use
```
iwconfig wlan0 essid "YOUR_ESSID"
```

Now if you're connecting to a WEP network do the following (WEP is NOT secure at all, why are you using it?!)

4. Set the key where YOUR_KEY
```
iwconfig wlan0 key "YOUR_KEY"
```

5. Now you need to either set your IP address and gateway manually or get the details from a DHCP server, most people do the latter if so run the following
```
sudo dhclient wlan0
```

If you didn't get any error messages during this process then you should be successfully connected.

Now if you're using WPA/WPA2 (you should be!) the process is more complicated because you need to use the wpa_supplicant tool to do the authentication. I'm going to assume you're using WPA-PSK (look at the "Authentication Suites" line from `iwlist scan' to find out what you want) if you're not please look at
```
man wpa_supplicant.conf
```

4. Go into your home directory and now we'll create the configuration file wpa_supplicant needs where YOUR_ESSID is the wireless network's ESSID and YOUR_PASSPHRASE is the passphrase for your wireless network note the quotes are necessary to stop bash interpreting special characters (e.g. * [ ] )
```
cd ~ 
wpa_passphrase "YOUR_ESSID" "YOUR_PASSPHRASE" > wpa_supplicant.conf
```

Look at the file you just made ( wpa_supplicant.conf in your home directory). It should look something like this (note your psk=... should be different from mine)
```
network={
	ssid="YOUR_ESSID"
	#psk="YOUR_PASSPHRASE"
	psk=0f1881ed177fa606896a36b1b4d0979e91b07c89f91da16c7705d40466e43569
}

```

Now Add the line ```
key_mgmt=WPA-PSK
```, you can also remove the #psk=... line as it is a comment. So now your wpa_supplicant.conf file should read
```
network={
	ssid="YOUR_ESSID"
	key_mgmt=WPA-PSK
	psk=0f1881ed177fa606896a36b1b4d0979e91b07c89f91da16c7705d40466e43569
}

```

5. Now we will use wpa_supplicant to authenticate with your wireless network. 
```
sudo wpa_supplicant -i wlan0 -D wext -c ~/wpa_supplicant.conf &
```

If this doesn't work then take a look at ```
man wpa_supplicant
```
and ```
man wpa_supplicant.conf
``` for help using wpa_supplicant.


6.Provided there were no errors we can now either set the IP address and gateway manually all get the details from a DHCP server. In the latter case run.
```
sudo dhclient wlan0
```

Hopefully now you should be connected. Please note do not close the terminal you lauched wpa_supplicant in because it needs to be kept running.


These are the basics steps to manually connecting to a wireless network. By doing these steps individually you may be able to find out where the problem lies in each of your situations.

4. Go into your home directory and now we'll create the configuration file wpa_supplicant needs where YOUR_ESSID is the wireless network's ESSID and YOUR_PASSPHRASE is the passphrase for your wireless network note the quotes are necessary to stop bash interpreting special characters (e.g. * [ ] )
```
cd ~ 
wpa_passphrase "YOUR_ESSID" "YOUR_PASSPHRASE" > wpa_supplicant.conf
```

Look at the file you just made ( wpa_supplicant.conf in your home directory). It should look something like this (note your psk=... should be different from mine)
```
network={
	ssid="YOUR_ESSID"
	#psk="YOUR_PASSPHRASE"
	psk=0f1881ed177fa606896a36b1b4d0979e91b07c89f91da16c7705d40466e43569
}

```

Now Add the line ```
key_mgmt=WPA-PSK
```, you can also remove the #psk=... line as it is a comment. So now your wpa_supplicant.conf file should read
```
network={
	ssid="YOUR_ESSID"
	key_mgmt=WPA-PSK
	psk=0f1881ed177fa606896a36b1b4d0979e91b07c89f91da16c7705d40466e43569
}

```

5. Now we will use wpa_supplicant to authenticate with your wireless network. 
```
sudo wpa_supplicant -i wlan0 -D wext -c ~/wpa_supplicant.conf &
```

If this doesn't work then take a look at ```
man wpa_supplicant
```
and ```
man wpa_supplicant.conf
``` for help using wpa_supplicant.


6.Provided there were no errors we can now either set the IP address and gateway manually all get the details from a DHCP server. In the latter case run.
```
sudo dhclient wlan0
```

Hopefully now you should be connected. Please note do not close the terminal you lauched wpa_supplicant in because it needs to be kept running.


These are the basics steps to manually connecting to a wireless network. By doing these steps individually you may be able to find out where the problem lies in each of your situations.
