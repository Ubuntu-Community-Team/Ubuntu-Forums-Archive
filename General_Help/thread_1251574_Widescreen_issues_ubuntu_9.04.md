### Post by cmost on 2009-08-27
Hello!  Use the following as your xorg.conf file.  Since you and I both have Nvidia cards and wide screen monitors, you should be able to get away with using my xorg.conf file.

Here's how:
1.  Open a gnome-terminal
2.  sudo gedit /etc/X11/xorg.conf
2.  File --> Save As -->xorg.conf-bak
3.  Copy and paste the following into the open file (delete existing text...

```
# xorg.conf (X.Org X Window System server configuration file)
#
# This file was generated by dexconf, the Debian X Configuration tool, using
# values from the debconf database.
#
# Edit this file with caution, and see the xorg.conf manual page.
# (Type "man xorg.conf" at the shell prompt.)
#
# This file is automatically updated on xserver-xorg package upgrades *only*
# if it has not been modified since the last upgrade of the xserver-xorg
# package.
#
# Note that some configuration settings that could be done previously
# in this file, now are automatically configured by the server and settings
# here are ignored.
#
# If you have edited this file but would like it to be automatically updated
# again, run the following command:
#   sudo dpkg-reconfigure -phigh xserver-xorg

Section "Device"
	Identifier	"Configured Video Device"
	Driver		"nvidia"
#	Driver		"nv"
	Option      	"Coolbits"		"1"
	Option		"NvAGP"			"0"
	Option		"AddARGBGLXVisuals"	"1"
	Option		"AllowGLXWithComposite"	"1"
	Option		"TripleBuffer"		"1"
	Option		"AccelMethod"		"EXA"
	Option		"EXAOptimizeMigration"	"true"
	Option       	"MigrationHeuristic" 	"greedy"
	Option 		"PixmapCacheSize" 	"5000000"	# needed for nvidia 180.xx
#	Option 		"AllowSHMPixmaps" 	"0" 		# needed for nvidia 180.xx
	Option 		"AllowIndirectPixmaps" 	"True"
	Option		"RenderAccel"		"True"
	Option 		"OnDemandVBlankInterrupts"	"True"
	Option 		"InitialPixmapPlacement"	"2"
        Option		"DisableGLXRootClipping"	"True"
	Option		"DamageEvents"		"1"
	Option		"BackingStore"		"1"
	Option 		"UseEvents" 		"False"
EndSection

Section "Monitor"
	Identifier	"Configured Monitor"
EndSection

Section "Screen"
	Identifier	"Default Screen"
	Device		"Configured Video Device"
	Monitor		"Configured Monitor"
	DefaultColorDepth 24
	SubSection "Display"
		Depth	8
		Modes "1280x1024" "1152x864" "1024x768" "800x600" "640x480"
	EndSubSection
	SubSection "Display"
		Depth	16
		Modes "1280x1024" "1152x864" "1024x768" "800x600" "640x480"
	EndSubSection
	SubSection "Display"
		Depth	24
		Modes "1440x900" "1280x1024" "1152x864" "1024x768" "800x600" "640x480"
		Virtual	1440 900
	EndSubSection
EndSection

Section "DRI"
	Mode 0666
EndSection

Section "Extensions"
	Option	"Composite"	"Enable"
EndSection
```


4.  File --> Save As --> xorg.conf
5.  Reboot!

Enjoy!

If you can't restart X, then simply type:
sudo mv /etc/X11/xorg.conf-bak xorg.conf
sudo reboot

This will restore your old xorg.conf

### Post by KinKiac on 2009-08-27
What you do is rename the original xorg.conf file to xorg.conf-bak or really anything that tells you that this a backup of your original xorg file.  If you updated or changed your xorg.conf file, even with just saving within the nvidia-settings app, it will have probably automatically created a backup of the original xorg file, which is what you are seeing in your /etc/X11/ folder.  It doesnt matter if one is already there, you want a new one that is simply a copy of your current working xorg.conf file so that if you mess anything up, you can always use the steps provided here to restore the file to what it was before.  

Just follow the instructions provided exactly, and it will create a new backup of your current xorg settings.  Then you are free to change your existing xorg file without having to worry about messing it up.
