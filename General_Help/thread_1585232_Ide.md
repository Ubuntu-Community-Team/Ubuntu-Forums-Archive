### Post by Simian Man on 2010-09-30
There are a lot of them with no clear "best".  There are Eclipse and NetBeans which were written for Java but also have very good support for other languages including C++.  There is Code::Blocks which is kind of like a simplified version of MSVC.  There is Anjuta which I really saw no point in.  There is also Qt Creator and Kdevelop which I have little experience with, but they are supposed to be good as well.

Personally I use Eclipse, but I think Code::Blocks is the easiest to start out with.

### Post by Locke_99GS on 2010-09-30
I'm a CS student as well. :)

Obvious choices are Eclipse CDT, Anjuta, Code::Blocks, KDevelop, and NetBeans. You could of course use your favourite text editor (gEdit, Kate, Mouse, nano, vi, eMacs, whatever) and I believe most of all of them have code highlighting which is nice but you will have to link and compile yourself.

A less obvious choice for a nice IDE is MonoDevelop. I know, I know, it's evil Mono, whatever. It works out of the box, and works with more than just C#. I happened to have this already instaled as I was using it to compile Pinta. It is similar to MS Visual Studio that you may be using at school, and will work with Visual Studio solutions.

MonoDevelop is what I'm using for c/c++ development. I'm happy with it.
