### Post by molotov00 on 2008-06-23
I've had the same problem with wine. I don't recall if 'sudo apt-get purge wine' worked correctly. 

.desktop files are 'icons'

Run:
sudo find / -iname *wine*.desktop

(Maybe doesn't have to be done as sudo, but it ensures you see all files that include *wine*.desktop)

When I run that command I get:
```
/usr/share/app-install/desktop/wine.desktop
/usr/share/app-install/desktop/winefish.desktop
/usr/share/app-install/desktop/wineconfig.desktop
/usr/share/applications/wine.desktop
/usr/share/applications/wine-winecfg.desktop
/usr/share/applications/wine-uninstaller.desktop
/usr/share/applications/wine-browsedrive.desktop
/usr/share/applications/wine-notepad.desktop
```

So those are my wine-related icones. Now... most programs should delete their icons but I have noticed wine is an exception... unfortunately. 

Check out those wine desktop files and delete 'em. (sudo rm filename)

Hope that helps.

### Post by molotov00 on 2008-06-23
Anyone else think it's funny there are three different answers? I'll clarify:

'sudo apt-get purge wine' deletes the config files and *should* completely remove wine. Uninstalling via synaptic doesn't do this as far as I know -- I don't use synaptic often. So replies 1 and 3 are identical instructions.

The wine icons are deleted with 'purge', but not always because sometimes wine forgets what icons it creates. For instance, there's a 'configure wine' icon for me.

Since you've already uninstalled, it's stupid to install then remove (with purge) again. So just do what I said, and delete the .wine folder (but that part has nothing to do with icons...)
