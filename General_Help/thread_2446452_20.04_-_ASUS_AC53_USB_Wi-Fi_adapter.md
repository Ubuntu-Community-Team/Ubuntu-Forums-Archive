From what I can tell, the driver doesn't use dkms, so it probably failed after kernel update and reboot. If that's the case, you could simply run 'make clean' and then run the build.sh script for the current kernel and reboot.

Then again, the repo you used may not be the best choice (I can't keep track of the multitude of rtl88xx github repos and what devices they support). Maybe it would be a good idea to have your thread moved to Networking an WIreless subforum where there are more knowledgeable people?
