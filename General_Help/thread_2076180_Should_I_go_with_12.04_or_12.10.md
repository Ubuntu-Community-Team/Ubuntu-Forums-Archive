### Post by Dragonbite on 2012-10-25
Here's my situation and I am debating whether to go with 12.04 LTS or 12.10 on the family computer.  

THE SYSTEM:
I have a duo-core desktop computer currently running an Ubuntu 10.04 LTS release.

This is the family's computer which is used by 2 family members more than me (*though I am using it more since my wireless router has died, but that's another story)*.  Of these family members one is more tech savvy (my son) while the other is less so and needs things to be visually consistent (my wife).

I also have 2 daughters who are less inclined to using the computer but one does not deal with changes easily and is being forced to use the computer more for schoolwork.  The other is more flexible.

I need the system to be stable and largely "install-and-forget".  I will be doing a fresh install because I have heard 10.04 doesn't upgrade very nicely, and I take these opportunities to clear out applications I've installed that I no longer use (*e.g. if I install Banshee for a while but really don't need it any longer, or Monodevelop with this idea I'm going to use it and never do*)

I know there is going to be a change from Gnome 2 to Unity but I am working through that by introducing them to it via LiveUSB.  The good thing is that they are visual people so the large icons and such are generally acceptable.

The biggest application usage includes OpenOffice (it's what came with 10.04), Shotwell, Firefox, Thunderbird and Chromium with some TuxPaint, Neverball and KPatience mixed in.

_THOUGHTS ON 12.04 LTS:_

Consistent:

I was thinking of installing 12.04 because being an LTS it will only go through point upgrades and nothing drastic for a couple of years.  

Stable:

Sometimes the in-between releases include something that may de-stabilize the system, or changes the whole layout / flow and I am looking to minimize that.

_THOUGHTS ON 12.10:_

Responsive:

My LiveUSB is currently running 12.10 and it ran nicely on the system when I tried it out.  I don't know how much of it is CD vs USB, but 12.10 seemed a little more responsive from the USB than 12.04 on a CD.  I'll probably need to run 12.04 on a USB for a more accurate comparison.

Online Accounts:

This is one of the things I liked about Gnome-shell and am glad to see it in Unity.  I, at least, spend more time using online accounts than anybody else but we all have different accounts.

My wife is heavy into Facebook, which she uses as a marketing tool for her business.

Support Time:

One downside is that it is supported for only 1.5 years and I don't think I can afford refreshing every release or every other release.  

Updated applications:

Open source applications generally update more often and sometimes significantly.  

To get the most out of some applications and upgrade or two may be beneficial, but the risk is time it take to make sure it will work and if something goes wrong.  Unless I can get 12.10 (and later) on my laptop so I can try in-between releases and determine if the benefit is worth the upgrade, my "testing" is somewhat limited.

If I follow the same path as I have with 10.04 then the next upgrade will be in 2.5 years, or spring of 2015 (which just happens to be when 12.04 drops support and the next LTS will be 1 year old)

I am leaning towards installing 12.04 LTS (LTS to LTS) at this point but wonder if anybody, preferrably with some 12.10 experience, has any thoughts on why I should go to 12.10 instead?

### Post by grahammechanical on 2012-10-25
In my opinion, you should most definitely fresh install whatever version you decide to run.

Do you have a separate /home partition? That will make things a lot easier.

Then you can create another 10 - 15GB partition and install 12.10, 13.04 etc., into it to test it out in the sense of finding out what it does and if you can configure it the way you and others want. It will help you work out if you want to upgrade.

I have been testing 12.10 for months. It has been a bumpy ride at times. But that is exactly what we expect when using the development branch.

I have 12.04 which I keep as a fall-back OS for when the development branch breaks. But, I tell you this in 12 to 18 months time 12.04 is going to seem dated.

I have just done a fresh install of 12.10 and it is stable and fast. I have set it up so that by doing daily updates it will turn into 13.04. I will put in other installs of 13.04 as it is developed.

I have more or less decided that I will keep 12.04 as my fall-back OS until April 2013 at least. Then I might switch to 13.04 as fall-back while I test 13.10.

I know that you do not need to go in this direction but you asked for advice from someone using 12.10.

Regards.
