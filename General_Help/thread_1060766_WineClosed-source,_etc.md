### Post by ajgreeny on 2009-02-05
I don't think you need to worry too much about this.  There are some programs which will run in wine and allow you to do certain things for which there is no real alternative.  For example I had a large number of old Lotus Smartsuite Wordpro97 files which were impossible to open in any linux application I could find, so I had no problems installing Wordpro from that in Wine in order to avoid booting into WinXP, just to read those files.  I could have converted all of them to .doc files in WinXP, but there were hundreds of them and it was not an economic option, so it was Wine that eventually solved the problem, though when I first tried it, Wordpro did not run and always crashed when I tried to open a file.  Later versions of Wine, from around 0.95, I think, ran it very sweetly.

Just do what you find you need to, and don't worry.  Once you get totally happy with Ubuntu, or whichever distro you choose, I suspect you will seldom boot back into windows.  You could always ask more about Linux alternatives for the application you are having to run in wine, using this forum.  There are sometimes people who have tried to do what you are trying to, and may be able to help.

### Post by phillies on 2009-02-05
From my point of view - as wine being an open source application - there is no 'harm' to the OSS movement by using wine.
When you use wine to run e.g. Microsoft Office for writing texts one could blame you a hypocrite ;) Since there are 100% compatible OSS solutions available.

On the other hand, there will never ever ehehever be a solely OSS world. So using wine e.g. for playing games or using (even open source) software not yet ported to linux instead of having a second partition or a virtual box running windows is quite a good solution.
