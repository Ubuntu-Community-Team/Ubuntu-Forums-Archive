### Post by davidvandoren on 2010-11-15
I have the same graphic card.
Nvidia G220

Here is what I did to solve the freeze ups.
 In one of my Ubuntu installs I got this in the logs very 2 minutes
 	Code:
 	NVRM: os_raise_smp_barrier(), invalid context 
 sometimes 10 in a row.

Symptoms:
Freezes
Applications snap
Drop back to login screen
Drop onto the command line.

My hardware 
NVIDIA graphics N220GT series
Motherboard MSI H55M E33

Installed Windows XP and Ubuntu 10.10 on a 1TB sata
And ubuntu 10.04 on a 30GB hard drive
Ubuntu has the proprietary Drive via Ubuntu installed 

**[SIZE=3]How I fixed this on the Ubuntu 10.10 is as follows.[/SIZE]**

Open terminal 
 	Code:
 	sudo gedit /etc/modprobe.d/nvidia.conf 
paste into the open file the following.
 	Code:
 	options nvidia NVreg_RegistryDwords="PerfLevelSrc=0x2222" 
click on save and **reboot**

Open** NVIDIA X server settings** it's under System, Administration. 
There click on** Powermizer**
There on the Preferred mode **drop-down  menu** chose  [B] Maximum Performance.


PS.
But after each restart I had to reset the Maximum Performance again. However other users confirmed that it worked for them Permanently>
[/B]
