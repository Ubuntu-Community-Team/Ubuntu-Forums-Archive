### Post by jed-reisner on 2014-02-23

Recently the OS disk completely failed and was totally unreadable.  I popped a new disk in and reinstalled ubuntu and zfs (same zfs and zpool version as before).  

But now my pool won't import. The sudo zpool import command shows this:
```
   pool: slimer     id: 10000377792241237537
  state: UNAVAIL
 status: One or more devices contains corrupted data.
 action: The pool cannot be imported due to damaged devices or data.
   see: http://zfsonlinux.org/msg/ZFS-8000-5E
 config:


        slimer      UNAVAIL  insufficient replicas
          raidz1-0  UNAVAIL  insufficient replicas
            sda     ONLINE
            sdb     UNAVAIL
            sdc     UNAVAIL

```

The problem seems to be that the drive names changed so zfs can't recognize them. However, running "sudo zpool import -d /dev/disk/by-id/ slimer" shows this: ```
cannot import 'slimer': pool may be in use from other systemuse '-f' to import anyway

```


Then I tried "sudo zpool import -d /dev/disk/by-id -f slimer"
```
cannot import 'slimer': no such pool or dataset
        Destroy and re-create the pool from

Does anyone have any ideas for how to recover this? I've been googling for the past 2 days and haven't found anything that worked. I have the important data (pics, documents) backed up, but it also had about a terabyte of media on it that wasn't backed up.
