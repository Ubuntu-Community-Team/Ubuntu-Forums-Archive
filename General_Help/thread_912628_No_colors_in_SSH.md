### Post by jerry79 on 2008-10-19
You need .bash_profile to load .bashrc during the login envirionment setup.  Here is mine (which originated on a debian system - the example directory is not on Ubuntu :icon_frown: ):

```
# ~/.bash_profile: executed by bash(1) for login shells.
# see /usr/share/doc/bash/examples/startup-files for examples.
# the files are located in the bash-doc package.

# the default umask is set in /etc/login.defs
#umask 022

# include .bashrc if it exists
if [ -f ~/.bashrc ]; then
    . ~/.bashrc
fi

# set PATH so it includes user's private bin if it exists
if [ -d ~/bin ] ; then
    PATH=~/bin:"${PATH}"
fi

```
