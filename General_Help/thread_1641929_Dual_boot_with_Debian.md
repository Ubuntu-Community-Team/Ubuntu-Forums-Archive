### Post by oldfred on 2010-12-09
I thought the osprober found other systems by looking at that systems menu.lst or grub.cfg. You do not have a Debian one and it still created a boot stanza.

The only thing I see as a possibility is Debian is still using hda and Ubuntu sda.

I might try this minor change from sda to hda as that is what Debian wants to see:

gksudo gedit /etc/grub.d/40_custom
sudo update-grub

```
menuentry "Debian GNU/Linux (5.0.7) (on /dev/sda3)" {
    insmod part_msdos
    insmod ext2
    set root='(hd0,msdos3)'
    search --no-floppy --fs-uuid --set 0433b711-956c-47a6-92e6-cf23fc6a3bbf
    linux /boot/vmlinuz-2.6.26-2-686 root=/dev/[COLOR=Red]h[/COLOR]da3
    initrd /boot/initrd.img-2.6.26-2-686
}

```

If that works then you can turn off osprober so you do not have duplicate entries.

gksudo gedit /etc/default/grub:
GRUB_DISABLE_OS_PROBER=true
or
sudo chmod a-x /etc/grub.d/30_os-prober
sudo update-grub

### Post by oldfred on 2010-12-09
Does Debian use grub or grub2? You can install Debian's grub to the sda3 partition boot sector and chainload. If grub2 it will complain, but can still be installed to the PBR.
