### Post by ajgreeny on 2016-11-01
Hi kodro and welcome to the forum.

I do not know much about sylpheed, but in your situation I would start by renaming the configuration folder **.sylpheed-2.0** as .sylpheed-2.0backup, then start sylpheed again.
I think a new configuration folder will be created automatically which you can search for and you may then be able to copy any sub-folders from the original to the new and thereby keep your configuration.

Alternatively there may be an import/export utility in sylpheed which allows you to import from a previous version, but not using sylpheed I am not sure about that.
