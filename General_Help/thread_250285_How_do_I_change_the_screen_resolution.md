### Post by DoktorSeven on 2006-09-04
xorg.conf only uses the card name as a description.  As long as it's using the same driver, there's no need to update the name, since it's not important.

Make a backup of xorg.conf:
**sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.bu**
 
so if something messes up you can restore it by 

**sudo cp /etc/X11/xorg.conf.bu /etc/X11/xorg.conf** 

later.

Edit it with **gksudo gedit /etc/X11/xorg.conf**

Where you see this:
```
Section "Monitor"
Identifier "hp mx704"
Option "DPMS"
HorizSync 30-70
VertRefresh 50-160
EndSection
```

Change the values after **VertRefresh** to the values you found above -- **50-140** (that's assuming what you pasted above: Horizontal Sync = 30 - 70 and Vertical Refresh = 50 - 140 -- is correct for your monitor), note no spaces between the numbers and the dash.

**VertRefresh 50-140**

You can then change your video modes here:
```
SubSection "Display"
Depth 24
Modes "1024x768" "800x600" "640x480"
```

Make sure you edit under the correct color depth (**Depth 24**, the highest available).

If you want 1280x1024, just add that value in quotes **before** the "1024x768", as so:

Modes "1280x1024" "1024x768" "800x600" "640x480"

The *first* value shown is the default one.

Save xorg.conf and exit gedit.

You now need to restart X -- simplest way is to reboot, but CTRL+ALT+BACKSPACE should also do it (warning, this will close X and everything you have running instantly).  If Ubuntu can't restart X and dumps you back at a command prompt, log in and restore your xorg.conf file as outlined above, then do a **sudo reboot**.
