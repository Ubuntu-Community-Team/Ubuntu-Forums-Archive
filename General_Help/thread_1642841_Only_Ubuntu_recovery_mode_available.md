### Post by tredegar on 2010-12-11
Not sure if backtrack tried to install grub legacy.

Anyhow, boot to ununtu's recovery root shell, then you need to update grub2

I *assume* your disk is /dev/sda (if not change the following commands)

```
grub-install /dev/sda  # use if BT messeded up your original grub2
update-grub            # have grub search for all bootable partitions
reboot
```

### Post by Rubi1200 on 2010-12-11
Try running these commands from within Ubuntu:

```
sudo update-grub
```
and if that doesn't pick it up then this:
```
sudo os-prober
```

Funny thing is that grub.cfg shows your Ubuntu entries on sda1 (including recovery mode), memtest, and Backtrack.

It may be necessary to purge and reinstall GRUB, but first try the other commands before anything.

Obviously, you will either see the entries when you run the command and if not let me know.
