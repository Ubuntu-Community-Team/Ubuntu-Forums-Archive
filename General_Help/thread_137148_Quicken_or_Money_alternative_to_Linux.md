### Post by buzz40000 on 2007-07-05
I've spent man-months of effort trying to find a solution for migrating away from MS Money.  I've got a 10+ year history with Money and the bloated file to prove it.  It's the only thing that kept me in the Windows O/S universe for the last few years.

The key points for me are :
1) Automatic updating of accounts via on-line connections
2) Complex investment tracking including rollovers, shorting, splits, etc..

So, I've experimented with Quicken, GNUCash, Moneydance, and a few other obscure ones.   I also took a spin with WINE and even got Money to work, except that most of the text was missing from the user interface.

The mechanics of importing data from Money into these applications via QIF has worked flawlessly in all cases.  Unfortunately, combing through hundreds of transactions to correct catagorical errors  and math mistakes would take far too long.

So, I decided, the only real option is to start over fresh from a new start date.  OK, but now on to the second round of problems.   Only Quicken and [to a certain extent] Moneydance made quick work of setting up an OFX exchange with the majority of my institutions for on-line account updating.  GNUCASH "Might" be capable of it, if I could find the correct URL's for all of the 15 or so institutions I have accounts with, and correctly identify my username and password (not always the same as my website login).   Further, GNUCash still lacks features.

In the end, moving to Quicken didn't gain me anything, and I found Moneydance to be less intuitive, and lacking some obscure features that are important to me.  SO after all this effort I was forced to admit, I was staying with MS Money.

BUT... not with Windows.  My solution, which has proven to be quite elegant, is to run a copy of the Windows XP O/S on a virtual machine (VMWare Server automatically starts when I boot my FF Ubuntu).  Going to MS Money is as easy as alt-tabbing or switching workspaces, and it actually runs faster under the Virtual Machine then it did when I booted a machine to it!  This solves the "dual boot" problem and I've noticed no performance degradation either in Ubuntu or in the virtual machine (note: make sure to install VMtools in XP after you've installed it into the virtual machine).  I even fire up a game of Civilization now and again in the Windows VM and it runs great.

Until the banks stop supporting the Money 2005 (the version I use) OFX interface, I'm good.
