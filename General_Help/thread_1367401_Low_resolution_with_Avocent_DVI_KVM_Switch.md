### Post by JohnFante on 2010-02-20
Found a much simpler and better solutions for my EDID problems. With the above solutions my monitor would go blank on low screen resolutions.

Here is what I did:

I bypassed the KVM and connected the monitor directly to the videocard.

Deleted my xorg.conf, rebooted and ran "sudo nvidia-xconfig" to get a fresh xorg.conf.

Then I started Nvidia X server settings and went down to DFP-0 and pressed "Acquired EDID". I saved the file to my home directory and added these two lines to my device section in xorg.cof:

    Option 	   "ConnectedMonitor" "DFP"
    Option 	   "CustomEDID" "DFP-0:/home/me/edid.bin"

So simple was it. Works very well :-)

Hope this helps others.

### Post by Amof on 2012-12-28
Wow thanks!

I have been struggling with this issues for a long time now. We utilizes a lot of KVMs at my place of work because we are working on up to 12 machines at a time. The ones that ran Linux would never get the resolutions right due to the KVM blocking the EDID and making a custom xorg.conf for each machine was not an appealing option. Thanks to this post I was finally able to get them working on the native resolution.

One thing I would like to add on too JohnFante's post 

> I bypassed the KVM and connected the monitor directly to the videocard.

Deleted my xorg.conf, rebooted and ran "sudo nvidia-xconfig" to get a fresh xorg.conf.

Then I started Nvidia X server settings and went down to DFP-0 and pressed "Acquired EDID". I saved the file to my home directory and added these two lines to my device section in xorg.cof:

Option "ConnectedMonitor" "DFP"
Option "CustomEDID" "DFP-0:/home/me/edid.bin"

Make sure that you change the "DFP" option to reflect what type of KVM you use. We are still using VGA so I had to change it to "CRT" and "CRT-0:/home/me/edid.bin".

Also here is the documentation on Nvidia's site.
[http://http.download.nvidia.com/XFree86/FreeBSD-x86/280.13/README/xconfigoptions.html#ConnectedMonitor]("http://http.download.nvidia.com/XFree86/FreeBSD-x86/280.13/README/xconfigoptions.html#ConnectedMonitor")

This solution still works as of 12.10 with the newest Nvidia binary drivers.

I am hoping this can help someone avoid the frustration that I went through.
