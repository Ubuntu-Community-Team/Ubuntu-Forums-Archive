### Post by cbj50 on 2008-03-08
For me the problem was the website at URL [www.cbs.com](www.cbs.com). This site has a selection menu (Shows, Video, Community, etc.) and below the menu header line a Flash animation display. When you mouse over the menu selection items on the menu bar most or all of the menu structure remains behind the Flash animation. 

First, I am not Ubuntu but Debian etch with all latest patches as on 3/7/08 running on a Dell Optiplex GX620 with an Intel Pentium 4 3.2 GHz with the Intel 945GZ chip set. My browser is Iceweasel 2.0.0.12, using Flash Player 9.0.115.0  

It appears that the functionality which allows this Java script controlled menu structure to appear on top of the Flash animation is the WMODE parameter set in source code of the web page.

Using the following link it is easy to test your Browser/Flash Player combination.

[http://www.communitymx.com/content/source/E5141/wmodetrans.htm](http://www.communitymx.com/content/source/E5141/wmodetrans.htm)

For my Browser/Flash combination the menu structures remain behind the Flash animation regardless of the absence of the WMODE parameter or whether WMODE is set to &#8220;transparent&#8221; or &#8220;opaque&#8221;.

Testing with IE and Firefox on MS Windows 2000 reveals that without WMODE set the menus remain behind the Flash animation and when WMODE is set to &#8220;transparent&#8221; or &#8220;opaque&#8221; the menu structure appears on top. The &#8220;transparent&#8221; option simply allows you to see what is below the Flash animation in the DHTML layers.

Debian support denies that this is a problem with Iceweasel.

I was told that it might be a problem with XORG Composite Extension not being enabled. My current version of xorg is 7.1.0-19

In response I added the following to the end of xorg.conf for my userid:

Section "Extensions"
Option "Composite" "Enable"
EndSection

This had no effect on the menu structure appearing behind the Flash animation.

The problem exists for both Iceweasel and Epiphany browsers on my copy of Debian etch.

The following links provide some insight into the use of WMODE and Z-INDEX for layering in the web site source but I do not think there is a problem with the source.

[http://forums.mozillazine.org/viewtopic.php?t=612985](http://forums.mozillazine.org/viewtopic.php?t=612985)
[http://forums.mozillazine.org/viewtopic.php?p=3212046&sid=5a138fd504a70b174596ce8bb9dd6ac](http://forums.mozillazine.org/viewtopic.php?p=3212046&sid=5a138fd504a70b174596ce8bb9dd6ac)

As indicated previously Adobe has said that when WMODE is implemented correctly in the browser the layering will occur properly. Personally I do not know where the problem is yet and it may well be with Adobe.

I hope there is eventually some resolve in the Linux community and in the Ubuntu community I at least found a group that cares about the problem.
