### Post by jnkerwin on 2019-07-26
**Info:** Ubuntu 18.04.2 running on a Core i3 machine with 6 Gb RAM. SpinRite seems to verify that the 500Gb HDD is healthy.

**Problem:** After an update about two weeks ago, we cannot get rid of the Ubuntu "Settings" application. It pops over the application which should be getting the focus (usually Rhythmbox, since this is my wife's "music machine"). 

**Symptoms:**

[LIST=1]
[*]Settings "pops over" the currently running app, launching itself without being requested. 
[*]Settings pigs out on system resources. It will intermittently allow the focus to be passed to Rhythmbox, but only for a few seconds. 
[*]Settings usually shows the spinning-wheel "busy" cursor. 
[*]Settings resists being closed, and, if closed successfully, it will re-launch itself. 
[*]The Settings problem seems to be independent of Rhythmbox (i.e., it will pop up and persist whether or not Rhythmbox is running). 
[*]Sometimes, after a re-boot, Settings won't appear for a period of time (5-30 minutes). 
[*]With enough patience, Software Updater has been run, and all updates and patches are kept current. 
[/LIST]

Any help on this annoyance would be appreciated!
