### Post by BertN45 on 2012-12-30
You can run the memory test normally from any live USB stick or live CD of Ubuntu, It is one of the choices you get in the initial boot menu.

Since you installed the system 10 days ago and updated the system since then and probably installed a lot of things in the mean time. Did you see any error messages during installation or updates the last couple of days? The chance that something went wrong during that process is somewhat more probable now. 

I assume you have enough free disk space or was that the reason to use the mini version?

The last two releases things have changed in the system start-up process, so it is dangerous to follow old posts. I think also Ubuntu nowadays is using lightdm in the bootprocess, so by installing lightdm, lightdm-gtk-greeter you might have also downloaded the the lightdm setup for the Ubuntu Unity configuration.

Try to boot in recovery mode and check [COLOR="Blue"]sudo nano /etc/lightdm/lightdm.conf[/COLOR]. 

[SeatDefaults]
greeter-session=unity-greeter    
user-session=ubuntu 

If you see this, you have found the problem. You can set the correct values by hand (note Lubuntu is with a capital) or even better run:

[COLOR="blue"]sudo /usr/lib/lightdm/lightdm-set-defaults -s Lubuntu[/COLOR]

That last one I would try in any case.
