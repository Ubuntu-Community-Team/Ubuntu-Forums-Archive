### Post by baffenzeller on 2008-09-16
About a week ago, I decided to replace my openSuse 10.3 installation with Xubuntu. I've noticed a nice performance increase. Since the install, vim occasionally exhibits odd behavior. Sometimes when I make a mistake typing and try to fix it a hiragana character appears. I have since discovered what causes this to happen. If you want to write the hiragana character for 'ha', you type "h (backspace) a." When the 'a' key is pressed, the hiragana character for 'ha' appears. I have also found out that it is not restricted to hiragana characters. I  would like to know more about what is causing this. Is there a way to turn that functionality off, and if so, how do you turn it back on? Also, is there a site that tells you how to create other characters? Thanks in advance for your help.

### Post by Pro-reason on 2008-09-17
I haven't looked into how this affects Vim, but if you have SCIM installed, Ctrl-Space generally toggles it.  This allows me to input Chinese (and sometimes other languages) in any application.
