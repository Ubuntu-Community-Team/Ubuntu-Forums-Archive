### Post by mb_webguy on 2009-04-09
System Restore doesn't have an equivalent in Linux to my knowledge, but it doesn't really need it, either.  

Linux doesn't have a registry, so installing and uninstalling software isn't an arcane process -- either the application files are there, or they aren't.  System configuration settings are likewise not contained in a mysterious registry, but in easily editable text files.  If you're using Gedit to edit these files directly, a backup copy is automatically created (though it shouldn't be a replacement for backing these up yourself).  And if you're using some sort of GUI to manage these settings, restoring them should be just as easy as changing them in the first place.

### Post by mocoloco on 2009-04-09
There's a preferences backup tool you can find in Add/Remove Applications.  That will take care of your personal preferences and settings if you mess them up.  If you're thinking in terms of system-wide, I used to sudo-mess-stuff-up all the time until I started documenting in a text file the changes I would make.  Then it was much easier to go back and fix things after my tinkering went awry.
