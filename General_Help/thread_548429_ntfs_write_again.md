### Post by merlinus on 2007-09-11
Post /etc/fstab.  That is the file that needs to be edited.

sdb1 and sdb5 are formatted as fat32, not ntfs.  If you so desire, you can format them as ntfs from windows without any harm to your data.

The advantage is less fragmentation, and the ability to have very large files.

But you can leave them as is, and still mount, read, and write.

### Post by merlinus on 2007-09-11
Lose these lines:

//dev/sdb1 * 1 807 6482196 0
/dev/sdb2 808 19929 153597465 0 0
/dev/sdb5 808 3357 20482843+ 0 0
/dev/sdb6 3358 19929 133114558+ 7 0

They are the output of sudo fdisk -l, which is a command, not a file.

Add these lines in place of them:

/dev/sdb1 /media/sdb1 vfat iocharset=utf8,umask=000 0 0
/dev/sdb5 /media/sdb5 vfat iocharset=utf8,umask=000 0 0
/dev/sdb6 /media/sdb6 ntfs-3g  defaults,locale=en_GB.UTF-8 0 0

just beneath the line for swap (sda5).

Make sure the mountpoints exist.  If not, you will have to create them:

```

sudo mkdir /media/sdb1
sudo mkdir /media/sdb5
sudo mkdir /media/sdb6

```Then:

```

sudo umount -a 
sudo mount -a

```
