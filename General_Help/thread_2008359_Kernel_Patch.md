What is a kernel patch exactly? I have a patch that is suppose to upgrade speed, performance etc? Is this the same as kernel compiling/rebuilding?

---

### Post by Redblade20XX on 2012-06-22
Consider the kernel as compiled program. Since it is compiled, it must have source code. A patch is a modification/extension to this source code, it is essentially more code or a modification to pre-existing code, to provide some service not included in the orginal kernel source code. To include the patch, you must recompile the modified kernel source code.

It would be beneficial to include where you got the patch but if you are a beginner I wouldn't recommend to try on something as big as performance modifications. You might get tremendous errors.
