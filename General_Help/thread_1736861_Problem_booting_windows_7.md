### Post by Quackers on 2011-04-23
sda1, which is a Windows partition is now showing as having a "Grub2" Boot Sector type. This is because you appear to have installed grub to that partition. Not a good idea! Windows will never start with that in there.
I would suggest that you fix the mbr with a Windows repair/installation disc to get Windows booting. If necessary, you can then re-install grub to the mbr from the live cd.

### Post by kiyop on 2011-04-23
Because PBR(partition boot record) of /dev/sda1 is damaged, fixboot should be done to resolve the problem.

If you can use "Recover Disc" USB memory,
bootrec.exe /fixboot

Please refer to
[http://support.microsoft.com/kb/927392](http://support.microsoft.com/kb/927392)

Another way to resolve the problem is to use grub4dos.
It can chainload directly /bootmgr in /dev/sda1.
```
grub> root (hd0,0)
grub> chainloader /bootmgr
grub> boot
```I am not sure, but you may be able to chainload to /bootmgr in /dev/sda1 by grub2 command line.
In grub menu, press c key to enter into command line.
```
grub> insmod ntfs
grub> set root="(hd0,1)"
grub> chainloader /bootmgr
grub> boot
```If you cannot use "Recovery Disc" (USB pendrive), you may be able to use "backup boot sector" of ntfs partition (/dev/sda1) located at the end of the partition to recover PBR of /dev/sda1.

### Post by Quackers on 2011-04-23
That's good news :-)
The problem was that you installed grub to the Windows partition. This should never be done. It is not normally a good idea to install grub to any partition, but certainly not to a Windows partition.
Grub is normally installed to the mbr of the disc (like /dev/sda NOT /dev/sda1 for instance - no number at the end).
