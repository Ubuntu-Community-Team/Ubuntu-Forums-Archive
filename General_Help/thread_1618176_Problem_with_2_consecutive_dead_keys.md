### Post by sebagr on 2010-11-10
This is my first post, so hi everyone :)

I've recently moved away from Windows (finally!) and I'm mostly happy with the change. There's one thing that's driving me crazy, though, and after searching everywhere (i.e. Google for about 1 hr) I couldn't find how to change.

I'm a native Spanish speaker and I've been using the USA International keyboard layout for decades. I need accented characters quite a lot and find extremely convenient to use dead keys for vowels, like *áéíóú*, but there's no point to have accented consonants, like *&#347;* or *&#7765;*. In these cases, I expect to type *'*, then *s* and get *'s* to easily type, for example, *it's*. Currently, I need to type a space after *'* to get that character in place.

Also, typing the *'* key twice gives me *´*, but in Windows behaves differently, giving me *''* (that is, the dead keys cancel out and give me two plain quotation marks). This is extremely convenient when typing strings while programming, which I do for a living - currently, I need to type the single quotation mark, then space, and repeat.

In other words, if the key following a quotation mark is not supposed to be accented, I'd like to have a single quotation mark printed before the 2nd keystroke.

So, the question is, how can I change this behavior? Can I do it at all? This is the default behavior for USA International on Windows, but I can't make it work with any keyboard distribution on Ubuntu.

I'm not trying to make Linux look like Windows, that's not the point, but rather feel more productive on Ubuntu (which I already do) by making some of my typing easier.

Thanks in advance for your help!

Sebastian.

### Post by Tamhvm on 2011-04-04
Well, I know this thread is kinda old, but however, if someone stumbles to this post, I want to give out my solution to this problem.

I basically created a series of scripts to mimic the behavior of Windows' US - International keyboard layout using Linux's US - International with some slight changes. The result is named Win US Intl 4 Linux, and it is available on [http://t.tam.x10.mx/en/win-us-intl-4-linux/](http://t.tam.x10.mx/en/win-us-intl-4-linux/) for download and review.

The project is not complete, but so far it works for most mainstream cases (like that '' situation you got there, together with some other common situations like 's, 'r and 'm), as I'm a spanish speaker programmer too and I need those God forsaken quotation marks in Linux too.
Give it a try!
