### Post by minimec on 2014-10-23
@ubuserone

We met on #ubuntu. I am back from the dog walk...

Looks like the nvidia driver got an option "metamodes" that you can use in your xorg.conf. Sorry if I led you in the wrong direction on irc. 

So one additional line in the screen section of your xorg.conf might do the trick. 

See: 
[https://bbs.archlinux.org/viewtopic.php?id=145671](https://bbs.archlinux.org/viewtopic.php?id=145671)  #3
[https://bbs.archlinux.org/viewtopic.php?id=144545](https://bbs.archlinux.org/viewtopic.php?id=144545)  #4

That new line should give you 1600x900 on DVI-I-2 without adding any mode to xrandr.

```


Section "Screen"
    Identifier     "Screen0"
    Device         "Device0"
    Monitor        "Monitor0"
    DefaultDepth    24
    Option         "metamodes" "DFP-2: nvidia-auto-select; nvidia-auto-select {ViewPortIn=1600x900}     ## <-- NEW ###
    SubSection     "Display"
        Depth       24
    EndSubSection
EndSection

```

Let's hope for the best...

Cheers

I only installed directly from nVidia many years ago and back then they did not have an uninstall. And it did not work. So never have installed directly from nVidia again.

There are 3 ways to install, from Ubuntu repository, from ppa and from nVidia. If you directly install from nVidia you also have to integrate it into kernel. When you first install it does that, but then with every new kernel you have to redo that part again.
 If from ppa or nVidia you may need to manually make new initramfs.
 man mkinitramfs


Unless you have a very new card/chip the version in the repository usually works. If not the the ppa is usually a better choice. 
In all cases if you convert from one type of install to another you must totally purge the old version or else you will have conflicts and new install will not work.
