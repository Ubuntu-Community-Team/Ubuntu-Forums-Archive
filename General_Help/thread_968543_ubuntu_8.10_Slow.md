### Post by cmnorton on 2008-11-04
If your RAM is not already maxed out, doing so may improve your performance.

Last year I started using a Thinkpad T61p with 2GB RAM, SATA 7200, and dual-core. I am running Kubuntu 8.04. with the vesa driver and no extra graphics, like compiz. 

Performance was already pretty good, but improved a little when I maxed the RAM to 4GB, but not as much as I hoped. Mine is database, not graphics work, and this laptop clearly meets my needs for development workstation. 

Given the robustness of my Thinkpad, if I were running 8.10, hopefully I would not notice a difference, but I don't know enough about 8.10 to make an expert comparision. 

I have this feeling that OS releases, not just Ubuntu, are maxing graphics needs each time there is a major release. At least, one could come to that conclusion reading these forums. It would be nice to install Ubuntu -- maybe the feature is there and I don't know about it -- and say don't give me everything graphically.

I've probably already taken that step by installing the vesa driver, but graphical performance was not the reason there. I did so, because vesa was the only driver that would with the laptop standalone and in a KVM environment.

Good luck.
