### Post by Helkaluin on 2011-04-03
In Openbox, the actions MoveToEdgeNorth / South / East / West can be customised to keybindings of choice. These actions, when invoked, move the focused window to the direction as indicated, up till the nearest 'edge', that is, borders of other windows.

Currently I bind them to <super>+the vim j/k/h/l.

I find these supremely useful for uncluttering my windows without the use of the mouse, as the movement really intelligently snaps to the 'edges' instead of going all the way to N/S/E/W. I noticed there's the 'Put window' plugin in Compiz which allows the movement of windows by such keybindings, but alas, it moves the windows all the way to N/S/E/W instead of snapping to the nearest 'edge'.

So is there a way of implementing this 'move to edge' feature in Compiz? I miss my modern compositing compared to xcompmgr, but this keyboard-driven window management is something so very useful I found in Openbox.
