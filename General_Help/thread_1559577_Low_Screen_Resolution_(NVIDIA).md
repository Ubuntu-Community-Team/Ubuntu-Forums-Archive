### Post by andthewicked on 2010-08-23
type xrandr in the command line this will bring up all available display modes to switch to one of those choices you type xrandr -s * "whatever one you want"

```
sudo nvidia-bug-report.sh
```

creates a error log file in your home directory,can you post this and your xorg.conf?

### Post by realzippy on 2010-08-24
*Unable to get display device CRT-0's EDID*

The driver cannot get the monitor EDID.This sometimes is a monitor cable issue.
Can you test another one?

If the EDID cannot be acquired,the driver constructs the resolutions offered
from HorizSync and VertRefresh values in the xorg.conf monitor section.
You can edit the file and increase the values carefully e.g.

from

HorizSync       28.0 - 55.0
VertRefresh     43.0 - 72.0

to
HorizSync       28.0 - 64.0
VertRefresh     43.0 - 88.0

use

```
gksudo gedit /etc/X11/xorg.conf
```

to edit.Then restart X (or reboot) and check the nvidia-settings if now any other resolutions are offered...

### Post by petercool on 2011-01-29
> **realzippy said:**
> *Unable to get display device CRT-0's EDID*

The driver cannot get the monitor EDID.This sometimes is a monitor cable issue.
Can you test another one?

If the EDID cannot be acquired,the driver constructs the resolutions offered
from HorizSync and VertRefresh values in the xorg.conf monitor section.
You can edit the file and increase the values carefully e.g.

from

HorizSync       28.0 - 55.0
VertRefresh     43.0 - 72.0

to
HorizSync       28.0 - 64.0
VertRefresh     43.0 - 88.0

use

```
gksudo gedit /etc/X11/xorg.conf
```

to edit.Then restart X (or reboot) and check the nvidia-settings if now any other resolutions are offered...
