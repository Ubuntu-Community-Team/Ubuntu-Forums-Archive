### Post by deadflowr on 2017-03-30
Try changing the nomodeset to be right after the quiet splash and before the vt handoff.

Better yet, do not edit that file anyway, edit the /etc/default/grub file
add nomodeset to the line
```
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"

```change it to
```
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash nomodeset"

```save the file and run 
```
sudo update-grub
```

### Post by oldfred on 2017-03-30
That looks normal other than the bbswitch.
But I think I tried with another user to remove bbswitch and it that did not help.
[https://ubuntuforums.org/archive/index.php/t-2344901.html](https://ubuntuforums.org/archive/index.php/t-2344901.html)
But you should not need it. 

bbswitch is for laptops that can auto switch between nVidia & Intel.

bbswitch is a kernel module which automatically detects the required ACPI
calls for two kinds of Optimus laptops. It has been verified to work with
"real" Optimus and "legacy" Optimus laptops (at least, that is what the
author Lekensteyn calls those).

This package uses DKMS to automatically build the bbswitch kernel module.
