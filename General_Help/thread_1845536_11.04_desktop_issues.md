Next question: do you still have the proprietary driver installed?

Anyway, to add nomodeset to your boot options: At the grub menu, highlight the first entry and press the 'e' key to edit. At the end of the linux line (it will have wrapped over), you'll see the options:

```
quiet splash vt.handoff=7
```

Simply add "nomodeset to the end, so that it looks like this:

```
quiet splash vt.handoff=7 nomodeset
```

Now press ctrl-x to boot and let us know what happens.

### Post by coffeecat on 2011-09-20
Just a thought. I've no experience of the ATI driver install script from AMD, but the proprietary driver is sometimes known to leave bits of itself behind which interferes with the open source driver. It sounds from your OP that you were having problems booting with the open source driver at the beginning anyway, but it might be worth trying a few steps to completely purge the system of any trace of the cataylst driver to see if that helps. Have a look here:

[https://wiki.ubuntu.com/X/Troubleshooting/FglrxInteferesWithRadeonDriver](https://wiki.ubuntu.com/X/Troubleshooting/FglrxInteferesWithRadeonDriver)

Go for the "Problem: Need to fully remove -fglrx and reinstall -ati from scratch" option. Run the uninstall script in the first line **and** the rest.

### Post by coffeecat on 2011-09-21
No, but re-reading your first post I am struck by the installation taking hours. I agree with walt.smith1960 - that's not right at all, nor is the boot process taking a long time. I suggest you check the md5 hash of the downloaded ISO, 

[https://help.ubuntu.com/community/HowToMD5SUM](https://help.ubuntu.com/community/HowToMD5SUM)

If the md5sum checks out, start up with the live CD but when you get the purple screen with the two small icons, tap any key to take you to the text menu. Choose the disc integrity check. If the md5sum passes and the disc integrity passes, boot up from the first choice in the text menu - try Ubuntu. Do you get to a usable desktop? If you do, let's go back to your idea about the hard drive. Perhaps you need to exclude a failing drive. Open Disk Utility and look for the SMART data. What does it say? Run the self-tests.
