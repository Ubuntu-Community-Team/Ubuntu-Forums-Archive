The ping command simply ran a DNS query to see what IP address 199.181.132.250 resolves to (i.e. what domains point to it), and the server is hosting a large number of virtual domains.

If you were to run nslookup for any domain on the list (games.soapnet.go.com), you would get the same IP address.

It's simply configured that way in the DNS server.

I could write a lengthy explanation, but if you really care to understand the details, read [http://www.philchen.com/2007/04/04/configuring-reverse-dns](http://www.philchen.com/2007/04/04/configuring-reverse-dns) ;)
