### Post by scratched on 2006-08-27
After following the [Howto install xorg-aiglx + compiz]("http://www.ubuntuforums.org/showthread.php?t=145068") thread, I can no longer start the Xorg server. I get a "BSOD" exactly like the one [some people got from the recent updates]("http://www.ubuntu.com/FixForUpgradeIssue") (that isn't my problem though) I'm managed to narrow down the problem to the file /etc/gdm/gdm.conf-custom.

This is what I added to the file that breaks Xorg:
> [servers]
0=aiglx

[server-aiglx]
name=aiglx server
command=/usr/bin/Xorg-air :0
flexible=true
I found [a post on the Howto thread that suggests a solution]("http://www.ubuntuforums.org/showpost.php?p=1098212&postcount=887") of just removing the ':0' at the end of the 'command=' line but that didn't work for me.

The short version of the errors I get when Xorg tries to start is this:
> Failed to load module "i810" (module does not exist)
Failed to load module "kbd" (module does not exist)
Failed to load module "mouse" (module does not exist)
Failed to load module "wacom" (module does not exist)
Failed to load module "synaptics" (module does not exist)
No drivers available.
Fatal server error: no screens found.
There is also a long log file that gives more details but I can't post it. I can attach it if anyone can read it and make more sense of it than I can.

My graphics card is an i855gm Intel integrated graphics card running on a laptop. The modifications made to the gdm.conf-custom file is the only thing that breaks Xorg. I managed to install xgl but it was extremely unstable and pretty slow on my computer so I figured I'd try aiglx instead since it is supposed to run better on my card but I can't get the Xserver to even start.

Could anyone tell me what I need to do to fix this or what I might have done wrong?

I apologize if a solution has been posted for this already. I tried searching and couldn't find anything. Let me know if you need any more information to help me.
