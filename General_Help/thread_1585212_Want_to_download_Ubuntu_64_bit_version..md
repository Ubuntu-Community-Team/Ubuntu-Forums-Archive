### Post by Objekt on 2010-09-30
Hmm, didn't know they were throwing that disclaimer in there.

I've been using 64-bit Ubuntu 9.10 as my "desktop" OS for nearly a year.  Apparently, all this time, I was doing it wrong.

The previous post pretty much nails it.  There can be some...issues...with 64-bit this and that which can be frustrating for the new user.  I eventually got them all straightened out for my 9.10 install, but I can see why the Ubuntu powers that be would steer new users away from the 64-bit version.

When I finally got around to installing Ubuntu 10.04, I opted for the 32-bit version.  I just didn't want to deal with the manual tweaks necessary to get everything (well, Flash in Firefox, mainly) working in the 64-bit environment.  

I don't think I'm missing anything by running 32-bit Ubuntu instead of the 64-bit flavor.  Physical address extension (PAE) allows use of my entire 4 GB of RAM.  And I don't have to hunt down an unstable, 64-bit experimental build of Firefox to have all my plugins work properly.  

Hope this helps.
