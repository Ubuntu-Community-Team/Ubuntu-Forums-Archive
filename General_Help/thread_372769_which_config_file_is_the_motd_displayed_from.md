### Post by x1a4 on 2007-02-28
Actually, by default /etc/motd is a symlink, although I can't for the life of me remember which file it pointed to--just *ls -l /etc/motd*.  If you want your custom motd get rid of the symlink and just create your own file, as the Ubuntu one will be changed to default after every update.  

To answer your question, motd is run by *login*, which isn't run in the GUI.

### Post by llamakc on 2007-02-28
/var/run/motd (where the /etc/motd symlink points) is generated by /etc/init.d/bootmisc.sh. In there, you'll find the code where it uses /etc/motd.tail to generate the /var/run/motd. Put your customizations in /etc/motd.tail or hack /etc/init.d/bootmisc.sh to change it.

### Post by x1a4 on 2007-02-28

any idea for the PATH assignation in ~/.bash_profile rather than in ~/.bashrc ?

Depends how you run *bash*.  If you want *cat /etc/motd* to run only when you run bash as a login shell, put it in .bash_profile.  If you want to run it when you run bash as either an interactive OR login shell put it in .bashrc
