### Post by qpieus on 2007-09-14
So it sounds like the 2 partitions on the external drive are mounted automatically via fstab when the system boots? Also I see that you get the error that /media/disk doesn't exist. Does it actually exist?

Couple things to try. Try manually mounting the external drive partitions to a different directory. I use /mnt/temp, for example. So create whatever directory you want or use some other existing directory. Also change the ownership of the directory to you. I suggest using a directory that is not in /media. I've tried manually mounting stuff in that directory and had some strange problems.

sudo mount /dev/sda1 /mnt/temp

The other thing to try is use pmount instead of mount. pmount is used by an ordinary user to mount hotpluggable devices . Although I see you are running as root, so I'm not sure how pmount will work if you're root. I guess it still should work. See man pmount for details.

### Post by logos34 on 2007-09-14
Your 80 gig external drive is 'sdc' so take out this part of fstab
> # Entry for WD Passport
[COLOR="Red"]/dev/sda1 /media/External ntfs ,umask=000,users 0 0
/dev/sda2 /media/disk ext3 ,umask=000,users 0 0[/COLOR]

and replace with
> # Entry for WD Passport
[B][COLOR="Blue"]/dev/sdc1 /media/sdc1 ntfs defaults,nls=utf8,umask=007,gid=46 0 0 
/dev/sdc2 /media/sdc2 ext3 defaults 0 2[/COLOR][/B]

Create the mount points:

sudo mkdir /media/sdc1  
sudo mkdir /media/sdc2

*you can name the mount anything--e.g. 'sdc1,' 'passport1', 'usb_ntfs' and 'usb_ext3', etc.  Or use a volume label.

Let sdc1 mount read-only and then use ntfs-config to enable write support, which will automatically edit the fstab entry.

ADD: if you can do that, then I'd create new mountpoints for sda1 and sdb5 just for the sake of consistency

### Post by qpieus on 2007-09-14
For your external drive to be in fstab, it must have been connected during the OS installation. During the install, the system called that disk's partitions sda1 and sda2. From your most recent post, I see that the external drive is now sdc1 and sdc2. That's not unexpected at all - usb devices often get different device designators depending on when they're plugged in to the system.

As far as fixing this, I suggest deleting the external drive entires from fstab. I just looked in my fstab and I have no entries at all for my various external drives and usb sticks. They mount automatically with no problems. Now, as I said before, they may get a different device designator from time to time, but that is not really much of a problem, for me at least. Example, sometimes my external drive is sdf, sometimes it is sdg. It still works the same.

So make a backup of your fstab first and then delete

# Entry for WD Passport
/dev/sda1 /media/External ntfs ,umask=000,users 0 0
/dev/sda2 /media/disk ext3 ,umask=000,users 0 0

from your fstab. unmount and unplug your external device and run sudo mount -a to run the new fstab. Then plug in your external device and it should auto mount to a directory in  /media, as long as the automount stuff is working OK. The system should automatically create a mount point for it.

If you manually unmount it and then later want remount it, you'll need to figure out the /dev/sd? device designator. Usually you can get this by looking at the end of the dmesg file. Type:
```
dmesg | tail
```  and you should see some reference to /dev/sdx. Then use your mount command with that device designator and include the mount point you want in your mount command like I talked about in an earlier post. You can see why I recommend not manually mounting stuff in the /media directory - because that's where the OS mounts stuff. I've had way better luck manually mounting into a different directory.

### Post by logos34 on 2007-09-14
The only reason I suggested a specified mount point/fstab entry is because the OP seemed to want a specified mount point rather than 'disk' or 'disk-1'--at least for the ntfs partition.  

Things can quickly get confusing in the case of more than one usb device, with shifting mount points and all.  So if you plan to use a flash drive and/or other usb devices at some point it might be a better idea to go with the UUID format instead:

> #/dev/sdc1 
UUID=xxxxx /media/sdc1 ntfs ...
#/dev/sdc2 
UUID=xxxxx /media/sdc2 ext3 ...

I've never had a problem with the /media dir, but you could use /mnt just as well (that was the old default location)
