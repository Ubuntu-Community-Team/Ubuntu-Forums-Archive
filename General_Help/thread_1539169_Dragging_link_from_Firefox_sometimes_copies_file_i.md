### Post by gencon on 2010-07-26
Using: Ubuntu v. Lucid, with Gnome - all up to date.

Hi,

I often find it useful to drag a link from the icon on the left of Firefox's location bar to either the desktop or to a directory in an open Nautilus window.

Most of the time this creates a link, 'Link to name of page'. Sometimes it copies the file of the web page instead (containing HTML/PHP/Ect. code), in which case I can no longer click on it to open the web page (obviously it opens the local copy instead). This can sometimes be worked around by highlighting the actual URL in the Firefox location bar and dragging that instead, again most of the time that creates a valid link but sometimes that also copies the web page instead.

I am unsure why dragging the link works most of the time but not every time. Is there any way to fix this or to workaround the problem so that when dragging a link from Firefox to the desktop or to an open Nautilus window I always get a web link and not the actual file?

Please don't suggest I just use Firefox's bookmarking capabilities. I find it useful to put URLs on the desktop to serve as a reminder and when I have any kind of project I like to have all the files grouped together in a directory, then I can quickly open whatever document I want including web pages.

Thanks all.
