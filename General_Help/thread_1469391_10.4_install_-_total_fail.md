### Post by pbr on 2010-05-21
[FONT="Arial Black"][COLOR="Lime"]Great News![/COLOR][/FONT]

So for whatever reasons my network-based 10.4 upgrade got corrupted.  Big time.

I could boot and reboot but never got X, and had a VERY minimal system running (amazingly minimal; around a dozen processes listed by 'pstree').

Here's what got me past the major hurdles, toward a working system again:

```
sudo dpkg --configure -a
```

This was very verbose and took some time to run (around 1o minutes, but it was BUSY!) 

The "hacker" in me expected that the X-window system would just come up and work after that - it didn't.  BUT - just about everything came up fine after I rebooted.

Odd things:
- random data in file "rsync" in /etc/init.d/
- binary in the files "pon" and "ufw" in /etc/bash_completion.d/
- half-dozen packages whose post-install caught "Exec format error"

I attribute all of this to a corruption of the download at some point.  10.4 is good to go!
