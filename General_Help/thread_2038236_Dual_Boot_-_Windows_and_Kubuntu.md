### Post by oldfred on 2012-08-08
It should be repairable, but then a new Ubuntu install often takes less than an hour, not sure with Windows anymore but it is a lot longer. But then when we try to repair something it can take days. 

Many suggest just physically disconnecting each drive and installing. With XP you want it to be sda or first connection on motherboard and be sure to reconnect to same connection. Ubuntu is more flexible and if it sees a different drive number it will still work.

Besure to backup anything you want to save from both systems. If you  have the /home in a separate partition you can reuse it. Just be sure to mount it as part of the install but DO NOT click on the format box.

After installing and reconnecting both drives, run this. Ubuntu will update itself to be sdb and add the Windows boot entry to the grub menu. Be sure to set BIOS to boot from the Ubuntu drive or sdb.

If you do not want to open case, you can just install Windows to sda, but to make sure Ubuntu does not install its boot loader to sda, you have to use Something Else or manual install. Then during the partitioning menu you get a combo box on which drive's MBR (or partition's PBR which you do not want) to install the grub2 boot loader into.
