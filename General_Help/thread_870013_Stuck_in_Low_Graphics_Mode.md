### Post by northern lights on 2008-07-25
Has your system worked with regular graphics before or did you fail to get out of low graphics mode from the very first installation?

If it was fine previously, you might also wanna try reconfiguring X also:

First, backup your current xorg.conf```
cp /etc/X11/xorg.conf /etc/X11/xorg.backup
```

As for X, run ```
sudo /etc/init.d/gdm stop
``` from a terminal. You will be leaving the graphical environment, so remember the next two steps!

Press Alt+F1 to log in and execute ```
sudo dpkg-reconfigure -phigh xserver-xorg
```

Then restart the interface with ```
sudo /etc/init.d/gdm start
```

Try to change resolution again.

In case something got (even more) screwed, revert back with```
cp /etc/X11/xorg.backup /etc/X11/xorg.conf
```

### Post by Ataris on 2008-07-25
This is exactly what I did in this situation, and it worked for me.

1. Download EnvyNG
2. Download the latest driver via Envy Applications --> System Tools --> EnvyNG
3. Reboot
4. Do nothing else but open a terminal and type: ```
sudo dpkg-reconfigure xserver-xorg
```
5. On the first box hit the recommended option, whatever it is, then keep pressing OK and go through the "wizard"
6. Reboot
7. This time Ubuntu should have high-graphics and the resolution back
8. DO NOT: RENABLE THE NVIDIA DRIVERS!!! Then you'll have to do everything again. It will say they're disabled, but they are really there and 3D acceleration will work fine
