### Post by SickTwist on 2005-12-24
I don't think there is anything like that.

Wine only works for x86 binaries (and even then, only when Wine is running on an x86 processor).

In the past, Apple has used different processors in their machines so the games that you have are not compiled for x86 processors.

The only possible solution that I can think of is if you were to run Linux on on hardware with the same Macintosh processor for which those games were compiled. Then you would need to find a program that translates the Macintosh API to the Linux API (like Wine translates Windows API to Linux API) but I don't know of such a program.

### Post by magnusbb on 2005-12-25
maconlinux is not an emulator. Thus, it doens't work on the x86 plattform. It is designed for those using Linux on the Mac plattform, wanting to run MacOS inside their Linux operating system.

But you can use Basilisk II for Classic Mac emulation, or PearPC for OS X emulation. The latter is rather slow, but the first one works great for pre OS9-versions. It only emulates 68k-processors, though.
