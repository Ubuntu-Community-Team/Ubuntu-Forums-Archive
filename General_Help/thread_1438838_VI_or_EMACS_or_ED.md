### Post by blur xc on 2010-03-26
> **Paddy Landau said:**
> Nope. I don't think that you'll find a CLI editor better than a good GUI one. For example, mouse support is hard (nano does support it, to an extent). Surprise me, if you can!

How is mouse support make a text editor better?  I always thought (correct me if I'm wrong) when editing text your hands should be on the keyboard.

To delete a word in a gui - take hand of keyboard, double click mouse on word, put hand back on keyboard, hit delete key.

In vim- use hjkl- w b, or go straight to the line with the offending word by hitting :{line number} (ie. :59 jumps to line 59), jump to correct word and hit dw.  It's very fast, and verry efficient.  dd to delete a line.  5dd to delete 5 lines.  5dw to delete 5 words.  5yw to copy 5 words...  so on and so on, and I'm only a beginning vim user.

There is a steep learning curve, but if you are going to be editing a lot of text files, it's way worth learning.  Learning a good cli text editor is also great in case you can't get to a gui text editor (crashed gui).

I'll agree- nothing beats a gui editor in the regards that it's easy to use and you don't have to learn anything new to use it.  A good cli one can be even easier to use, but only after learning a lot.  User friendliness is not thee same a intuitiveness.  

Here's a good read- [http://www.swaroopch.com/notes/Vim_en:Introduction](http://www.swaroopch.com/notes/Vim_en:Introduction)

No offence to emacs users- I'm sure it's great too...  It's just that back when I was trying to pick between vim and emacs, the sheer magnitude of what emacs can do overwhelmed me.  Vim is simpler...
