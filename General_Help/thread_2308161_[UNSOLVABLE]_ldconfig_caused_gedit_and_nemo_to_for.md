### Post by landstander on 2015-12-31
**_Specs:_**
**OS:** Xubuntu 15.04
**ldconfig** is a member of package libc-bin
**libc-bin** version: 2.21-0butuntu4
**gedit** version: 3.10.4
**nemo old** version: 2.2.4-2 (vivid)
**nemo new** version: 2.8.6+really2.6.7-2~webupd8~vivid1 (vivid)

**_Description of how the problem occurred:_**
On compiling a newer version of GEGL (a Gimp dependency) for the latest development branch of Gimp, output of an "./autogen.sh" command instructed me to run "ldconfig". After viewing the output from "man ldconfig" it seemed like it might be a quick fix to get GEGL to compile. So from the command line...:
```
sudo ldconfig
```
Here sudo was used due to a message about links not being created by running "ldconfig" by itself due to permission errors. For whatever reason this allowed GEGL to compile without any further errors, but after discovering problems from running ldconfig (see below) and a PPA for the latest dev.version of Gimp, compiling Gimp was ditched (as in deleted) in favour of an install via PPA.

**_Description of the problem:_**
After running "sudo ldconfig" as instructed by a message from a configuration script (most likely this was done in a directory that got deleted), both the Nemo file manager, the Gedit text editor, and possibly other applications forget their settings / preferences after the following conditions:
**1.)** After nemo or gedit (and possibly other applications) have been restarted.
**2.)** After any changes have been made to the panel (on the desktop where the icons for gedit and nemo have been placed).
**3.)** After any system restart or login / logout sequence has occurred.
**Note:** The system was finding and using whatever it needed for Nemo and Gedit to remember their settings before this problem started.
