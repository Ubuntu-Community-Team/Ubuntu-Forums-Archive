### Post by doncristobal on 2009-12-25
If you see huge amounts of rubbish in your sent emails (example below) when you send .doc attachments, try the following: 

[LIST=1]
[*] go to ~/.mozilla-thunderbird
[*] open your profile folder (usually xxxxxx.default). 
[INDENT]If you don't know where your profile folder is, you can look that up in the profiles.ini file; it says "Path=...". 
If there are several [ProfileX] entries, you'll have to find out which one you're currently using. 
Paths can be relative or absolute, which is also indicated: "IsRelative=1" means a relative path; 0 is absolute.
[/INDENT][*] make a backup copy of the file mimeTypes.rdf
[*] open the file mimeTypes.rdf
[*] look for an entry that rules the "text/richtext" mimetype and contains the "doc" file extension: 
```

  <RDF:Description RDF:about="urn:mimetype:text/richtext"
                   NC:value="text/richtext"
                   NC:editable="true"
                   NC:description="Microsoft Word-Dokument">
    <NC:fileExtensions>doc</NC:fileExtensions>
    <NC:fileExtensions>rtx</NC:fileExtensions>
    <NC:handlerProp RDF:resource="urn:mimetype:handler:text/richtext"/>
  </RDF:Description>

```
(5th line)
[*] remove the doc line
[*] save the file
[*] restart Thunderbird
[/LIST]

Now your .doc files should be sent properly. 

**Please give feedback!** Thank you.

A rather brute force alternative is to simply delete the mimeTypes.rdf file. You'll lose attachment actions that you defined if I'm not mistaken, but besides that Thunderbird will work perfectly without that file. 
In any case, keep a backup copy of it!
