### Post by IMSargon on 2010-10-24
Step 1: Slow down. There are an awful lot of typos in your post. I point this out not to be pedantic, but because if you really want to get inside Linux to understand how it works, you'll spend quite some time executing root commands. Under these circumstances, one wrong character can do a great deal of damage to your system. If you take it slow and make sure you fully understand each command before executing it, you will learn Linux even faster than you think.

Step 2: Choose a Distro
There's a lot to like and Gentoo and Ubuntu. It's fun to take Gentoo and just install it a few times. You'll learn plenty about Grub and filesystems and a bit about the Linux directory structure. Go ahead and customize every module the Kernel offers - but beware the unforeseen consequences! Do this on a spare computer, rather than a daily work computer. Just be aware that not everything you learn applies to all Distros. Ubuntu doesn't have emerge or USE flags. I like Ubuntu and Debian. apt-get gives you quick access to a vast array of software without having to worry about dependencies. The userbase is huge, so you get the benefit of active forums and broader software availability. Check out Distrowatch.org for a good list of possible distributions and their philosophies. Another way to decide is to find out what distro is being used in the field. Where would you like to work? What distro are they using and why? 

Step 3: Ask good questions
Try to do things. Make mistakes, find out how to remedy those mistakes. Experience is truly the best teacher, and you'll build your confidence and competence quickly. 


On the other hand...

Don't just use Linux because all the cool computer guys use it. The Windows command line is plenty powerful, and all of the core Unix commands have been ported to windows. What kind of programmer do you want to be? Would your time be better spent studying math than Linux? Try to find a mentor. Look for someone you respect doing what you wish you could be doing. See if you can get them to guide you. 

I could go on and on, of course, but I'll leave some time for others to chime in.

Best of luck to you!

### Post by IMSargon on 2010-10-24
At this point, either Windows or Linux will suit you just fine. If you'd like to try Linux, Ubuntu is an excellent choice. The difference for you is that under Windows you'll have to install your compilers, and under Linux, the most common ones are already included. Many programmers do use Linux, so becoming familiar with the environment is a good choice.

You know, with each new programming language you learn, learning new ones gets easier. I'd start with a nice, high-level language like Perl or Python. Perl is famous for its text-handling abilities. Python is a great general purpose language. Also, Bash scripts (or batch files under Windows) are a good way to build programming experience. Once you learn a bit of these languages, you'll find ways you can use them as tools to make your everyday tasks easier. Write programs to solve your friends' problems.

If you're the kind of person to be interested in programming, you're the kind of person who can just jump into Ubuntu and pick up how to do things as you go along. If find yourself frustrated with that approach, you can pick up a book like this one: 
[http://www.amazon.com/Beginning-Ubuntu-Linux-Fourth-Thomas/dp/1430219998/ref=sr_1_1?s=books&ie=UTF8&qid=1287972011&sr=1-1](http://www.amazon.com/Beginning-Ubuntu-Linux-Fourth-Thomas/dp/1430219998/ref=sr_1_1?s=books&ie=UTF8&qid=1287972011&sr=1-1)
I'm noticing it has a whole chapter on the shell, and a command reference.

You can start learning a language right away. Any old book will do. How's this one:
[http://oreilly.com/catalog/9780596802387/](http://oreilly.com/catalog/9780596802387/)
Looks pretty easy to me, presenting python as a very first programming language.
or this one:
[http://oreilly.com/catalog/9780596000271/](http://oreilly.com/catalog/9780596000271/)
This one is for Perl, and looks a bit more in-depth.

You can also Google around. There are great resources for free.
[http://ocw.mit.edu/courses/electrical-engineering-and-computer-science/6-00-introduction-to-computer-science-and-programming-fall-2008/](http://ocw.mit.edu/courses/electrical-engineering-and-computer-science/6-00-introduction-to-computer-science-and-programming-fall-2008/)


Ubuntu is a pretty cutting-edge operating system. Are you using the latest version, 10.10? If so, you're as up-to-date as it gets. Moving to Gentoo would offer you more customization per-program (for example, if you compile sudo with the "offensive" flag, it will insult you if you type your password incorrectly), but it won't actually give you a more modern system. If you want a newer version of a particular program, you can always download a tarball or other source repository and compile it yourself on Ubuntu. I find this is rarely useful, though, unless you want to help the developers find new bugs. What Ubuntu offers over Gentoo is that you can get started faster. The OS you install is complete and ready to handle the most common tasks you would expect from an operating system.

Have I answered your question completely?
