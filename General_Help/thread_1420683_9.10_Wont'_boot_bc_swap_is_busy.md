### Post by abrrymnvette on 2010-03-03
My 9.10 Ubuntu won't boot and sometimes an error goes across the screen saying that swap is busy. 

So, what I did is booted to a live cd and "mount /dev/sda1 /tmp" then I went into the /tmp/etc and nano fstab. I changed it from using the UIID to just using /dev/sda1 / and /dev/sda5 as swap. 

Rebooted and it came up fine. 

My question is, how to I fix this from happening again? I'm trying to create a backup image of this machine using Acronis, but it keeps telling me there are errors. The errors are in the swap partition.
