### Post by MartinG on 2005-12-13
[QUOTE=Donnut]Are there any desktop specific package makers?[/QUOTE]

Not sure I understand what you mean here, so what follows may be irrelevant ;-).   You can get alien via apt-get, and if you then run "sudo alien -r <package>.deb" it should produce "<package>.rpm".  This is in theory installable on any .rpm-based system, regardless of whether it has KDE, Gnome, Enlightenment or any other (or no!) desktop.  There's no such thing as a "KDE package", just a regular package that uses the KDE/Qt libraries and looks for them as dependencies.  

The difficulty that I can see is that different distros put things in different places (is KDE in /opt, /usr/, /usr/share, or where?), and a package generated by alien will install according to the Debian (or maybe Red Hat) schema, which may or may not suit.  This is why earobinson wants somebody to try it out.  Can you  take a .deb package from your /var/cache/apt/archives, convert it, and see if it installs trouble-free on Mandriva?  You might make a lot of people happy :-)
