Well, it's been a while since I used frames but a basic page

```

<html>
<head>
<title>Foo bar</title>
</head>
<body>
Hello world!
</body>
</html>

```Works .. 
Read this page:-
[http://www.w3schools.com/html/html_frames.asp](http://www.w3schools.com/html/html_frames.asp)

> 
**The Frame Tag**

 
[LIST]
[*]The <frame> tag defines what HTML document to put into each frame
[/LIST]
 In the example below we have a frameset with two columns. The first column is set to 25% of the width of the browser window. The second column is set to 75% of the width of the browser window. The HTML document "frame_a.htm" is put into the first column, and the HTML document "frame_b.htm" is put into the second column:
    <frameset cols="25%,75%">
   <frame src="frame_a.htm">
   <frame src="frame_b.htm">
</frameset> 




May help as I think your frameset code's duff.

```
<frame src="" frameborder="1" name="" scrolling="auto">
```src is empty, IIRC, you need to put a file name in there for it to load that file into the frame.

Screem generates good code.I think this a hammer issue..

html takes a while to learn but not years. Working with an editor is ok but even Dreamweaver, supposedly one of the best, makes WYSIWYG errors aplenty.

Its best suited to code assist mode where it can help speed up hand written code. Writing by hand is much faster and far more accurate than trying to make a web page using only WYSIWYG edit mode. WYSIWYG is more of a helper to cue things up better but its not going to write perfect pages for you, you need to tell it what to do otherwise it will make assumptions that are often incorrect especially for the first time programmer.Its code output is grossly messy and confusing as well. To a pro however, who knows how to correctly set up a WYSIWYG editor the code is more or less accurate. But most would not use it beyond a quick spacing check and an alignment. Otherwise its all back end hand written code.  

Best thing is to learn CSS basics then go back to that HTML thing. Once you understand CSS the rest is logic and easy.

Everything wants to go down - everything starts on the left side.
You can draw a grid using center but it will be wrong. better off using css margin:0 auto; - anything inside this container will begin from its inherited container and not the default. All style information cascades or becomes inherited. That way you really only need to set a style once and everything else will follow that unless you specify otherwise. Hence Cascading Style - sheet.
