I have searched the forums for this link but can't see it being posted before.

[https://bugzilla.gnome.org/show_bug.cgi?id=97545](https://bugzilla.gnome.org/show_bug.cgi?id=97545)

This bug affects gedit and any other programs which use pango such as geany. You've probably all seen it in action. The spell checker will highlight the first part of words such as "wouldn't" "shouldn't" as incorrect, even though they obviously aren't. It seems to be related to unicode and the ' character being treated as " or something similar.

I wasn't sure if this should be submitted as a papercut since something that hasn't been fixed in 8 years is obviously difficult. However gedit is likely to remain the default text editor for Ubuntu and I feel it would be nice to see this fixed. The pango devs already know about this though so I don't know if there's anyone left to inform.
