### Post by ajgreeny on 2012-02-05
Try Alt+V which may bring up the View menu, if there was one, where you may be able to re-enable the menu-bar.

In most cases, Alt + the first letter of an application's lost menu item will show that menu item on screen and may allow you to reset things as you want, eg
Alt+F shows File menu.
Alt+E shows Edit menu

I don't know xara extreme, so am not sure what menu items should exist, but try all possible letters.

### Post by Buntu Bunny on 2012-12-03
Well, I have figured out that I can access the menus with the alt key:

alt f = file menu
alt e = edit menu
alt u = utilities menu
alt r = arrange menu
alt w = windows menu
alt h = help menu

Once a menu is open I can also use the arrow keys to access the neighboring menu. I do not find this ideal, but at least I can get to the menus!

I agree it's an excellent program so I'm not going to give up on it.

### Post by Mr.Goose on 2012-12-25
Buntu Bunny! Ha! We meet again! lol :) No, I'm not stalking you - honest!

Seriously, my G/F was moaning about Xara's lack of menus. Actually, it all started because she was having issues with LibreOffice Draw, and decided to try Xara instead. Then she immediately ran across this very annoying menu issue.

As you rightly observe, it seems the Xara menubar only appears with Unity DE, and even then it is far from perfect. We're a (mostly) KDE household. Therefore, we are pretty much snookered as far as the Ubuntu repository version of Xara is concerned ;) 

Good news is there is an alternative version of Xara, available directly from the developers site that is not hobbled in this manner.

[http://www.xaraxtreme.org/download.html](http://www.xaraxtreme.org/download.html)

Sadly no *.deb. However you can install it manually from a tarball. 

They also have an Autopackage version, which seems to install pretty easily. Basically all you do is download the autopackage file, make it executable and double-click on it. If you opt to install it system-wide, then you will need your root password. However if you do install it system-wide, then it will overwrite the broken Ubuntu version, with a version that actually works properly!

I just installed it via the Autopackage method and it seems excellent. However Autopackage itself is now obsolete - so I regard this as a one-off hack to get round a problem. 

Bit off-topic, but here's more about Autopackage, if you're interested:-
[http://en.wikipedia.org/wiki/Autopackage](http://en.wikipedia.org/wiki/Autopackage)

HTH. Best wishes, G.
