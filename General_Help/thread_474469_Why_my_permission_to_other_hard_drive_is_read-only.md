[/B]
Ok, so this might be part of the problem. You should only need ONE mount point for windows, and you have 3 really.
Also, there's some sort of confusion as to which drive is your windows drive here. Out of the three, you have listings for:
/dev/sdb1
/dev/hda1
You need to figure out which one is the actual drive (I'm guessing the /dev/sdb1 because all external USB drives should be listed as SCSI and not as IDE), you can check for sure with this command:
```
sudo fdisk -l
```
Once you figure it out (again, in my example I will assume it is /dev/sdb1, if it **isn't** make sure to modify my command appropriately!), take out the three lines highlighted in bold above and replace them with this line:
```
/dev/sdb1    */media/Test*     ntfs-3g      defaults,locale=en_US.UTF-8      0     0
```

the part where it says /media/Test can be changed to any EMPTY folder on the filesystem (e.g. /localhd   /media/windows   /mnt/usbdrive    /home/$USER/my_drive), as long as that folder exists, and is once again empty.
If you mount it in /media/ or /mnt/, it should show up on the desktop by default. I'm pretty sure there is a way to make it only show up when mounted, like CD-ROM drives currently do, but you'll prolly need to search around for that.

If you still have problems unmounting the drive, we will probably need to modify it to mount with GUID/UID set to 1000 (your user), but first get your /etc/fstab sorted out, reboot and see if it works.
