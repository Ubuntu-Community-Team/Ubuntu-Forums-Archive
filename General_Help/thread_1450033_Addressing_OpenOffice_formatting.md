### Post by Hagar Delest on 2010-04-10
ODF is an open and ISO-standardized format. It means that its maintenance is linked to a group (OASIS, whose members include IBM, Novell, ...) dedicated to promote a robust format for office documents. The purpose is therefore to create a file format that would not be tied to a specific application like the MS Office file formats (.doc, .xls, .ppt).

OOo has adopted this format logically since it's completely documented. Whereas until recently, the MS Office formats were not documented. The OOo devs had to reverse engineer them to create import/export filters that are quite good. The problem is that they can't be sure that the implementation is correct. Hence such potential flaws concerning formatting.

File format are a key point now because the users have the possibility to get back the property of their own data. To read .doc, you need MS Office whereas .odt can be read by anyone who want to implements the documented specification (not only OOo can read ODF).

Formatting problems is not the stake. It is the mere safety of your data that are at stake here.
