### Post by cipherboy_loc on 2011-02-12
Do you want to work with CSS (which is just used to beautify HTML), or will you actually be working in HTML (using CSS to style it)? There are a number of WYSIWIG HTML editors, bluefish and winefish come to mind of the top of my head, but there is a third that I forgot the name of. These are good to a certain point, but in the end it is best to use something like [jkg]edit, emacs, vim, nano, and/or pico to create HTML/CSS. 
W3 Schools has a bunch of information on how to code web pages, without using a WYSIWIG editor.
[http://w3schools.com/](http://w3schools.com/)

BTW, (g/j/k)edit and the others are not WYSIWIG editors.
