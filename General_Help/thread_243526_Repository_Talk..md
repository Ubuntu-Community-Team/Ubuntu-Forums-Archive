### Post by peabody on 2006-08-26
Mmmm, network ports is a bad analogy in my opinion.  A better analogy is downloading an untrustworthy program.  The repositories are responsible for how synaptic knows where to download and install software.  If you use a repository that is untrustworthy, whoever is running that repository could put a malicious update in their repository that could then end up getting installed via the update mechanism.  That sound reasonable?

This keeps your system clean, apt-get lets your system cluttered with unused dependencies.

### Post by jimmygoon on 2006-08-26
Repositories are not like ports... having them enabled do not make you vulnerable as a port might. For example, you could have a port open for a legitimate reason but a hacker could exploit it. The only way a repository could 'hurt' you is if they had a high version number on like a corrupt piece of software so that when you ran it... it ran the bad version and hurt your computer.... so just make sure you use trusted sources (ie: ubuntu, automatix, easyubuntu, anything in an ubuntu guide should be fine too)

### Post by Irony on 2006-08-26
Repositories can be kept to a minumum of the official binaries. If you want to upgrade something you can temporarily enable say the backports.

Or you could manually download say a debian file and install it with the GDebi installer without enabling any extra repositories. For example someone on the forums had the 0.44 version of of Inkscape whereas the repository version is 0.43 so I went to the address via Firefox and downloaded the file just like I would download any file and installed it by double clicking on it. The GDebi installer informs you whether you have all dependencies satisfied or whether you need to download more stuff.

You won't get a notification to update if you disable a repository because synaptic won't know about the update.
