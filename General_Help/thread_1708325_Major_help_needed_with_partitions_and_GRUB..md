### Post by drs305 on 2011-03-16
Unless you partitioned it manually, the Ubuntu installer would have created an extended partition, and then logical partition(s) within it. You would not be able to add the free space back directly.

First you would need to delete the logical partition(s) and then the extended partition. That will leave just the Windows partition(s) and unallocated space. At that point you can add it to Windows.

The Ubuntu app you used was Gparted.

As *lithopsian* suggested, posting a screenshot will answer the question.

If you want to remove the Grub bootloader, you should be able to restore the Windows bootloader via a Windows repair CD or via the Ubuntu LiveCD. If running the LiveCD and Windows is on your sda drive:
```
sudo apt-get install lilo
sudo lilo -M /dev/sda mbr
```
Don't run any of the other lilo commands it may suggest.

### Post by drs305 on 2011-03-17
To expand your Windows partition using Gparted you will first have to shrink the extended (sda3) partition, then expand Windows (sda2):

Click on "sda3" in the lower window.
Partition, Resize, and drag the light blue left border all the way to the right.

Click on the Windows partition (sda2).
Partition, Resize, drag the right border all the way to the right.
Apply.

It will take some time, but at the end you will have a large Windows partition.

The above actions will do what you want, but many will suggest that having space in an extended partition available to make logical partitions in the future is a good thing. I'm in that group so I'd expand Windows as necessary but not use all the space unless you have to.

You didn't mention if you can now boot into Windows after applying the 'lilo' fix.

### Post by drs305 on 2011-03-17
> **DDarkusTriforce said:**
> Wait a minute, on the GRUB menu, windows is the 4th or 5th item on the list. Ubuntu is first. WIll this mean that the bootloader will load Ubuntu? DO i need to move WIndosw to the top first?

Once you ran the lilo command the Grub files won't be looked at. The command should have told the computer to look to sda1 for whatever boot files exist there. That should then transfer control to your sda2 Win7 boot files and it should boot if those files are all intact. Just make sure to remove the flash drive and that the computer will look at sda first.
