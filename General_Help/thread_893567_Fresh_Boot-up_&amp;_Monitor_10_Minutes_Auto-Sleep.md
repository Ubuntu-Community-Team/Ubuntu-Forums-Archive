Hello. My computer is having a strange Power Saving function problem. It isn't crucial, though annoying.

Since a few weeks ago every time I boot-up my computer my monitor goes to sleep after 10 minutes of inactivity. When I first installed Ubuntu on my desktop I changed the auto-sleeping power saving function to "Never". It has stayed that way every time I check my Power Savings preference. Lately, however, it seems that Ubuntu has been ignoring this preference and instead, despite being still set to "Never" (even when I check it after a fresh boot-up), puts the monitor to sleep after 10 minutes of inactivity. I can reinstate my preference of "Never" putting the monitor to sleep by going into the Power Savings preferences and sliding the bar from "Never" to something else then back to "Never" and clicking "Close". This seems to remind Ubuntu that I want my monitor to always stay on, at least until the next reboot when Ubuntu forgets this preference.

Hopefully there is a hidden nVidia control panel or obscure X config file that will solve this problem.
On a related note my last Ubuntu install of 7.10 eventually developed this behavior which I fixed by simply reinstalling, which points to the possibility of this being an inherent bug in Ubuntu.

If you need any more information about this please let me know and I'll try to provide it. Thanks.
