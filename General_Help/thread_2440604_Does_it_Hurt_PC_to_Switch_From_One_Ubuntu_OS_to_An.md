### Post by guiverc on 2020-04-13
By switch do you mean reboot & select another at reboot? or re-install a OS constantly.

If it's dual boot, I cannot see how it would do any harm at all (the system I'm replying on here has 18.04 & 20.04 installed dual-boot & I select when I turn it on).

If you mean re-install, again I cannot see how it'll harm cpu or RAM.  Yes there will be wear & tear on HDD/SSD due to writes of data when re-installing (writes wear SSDs, especially if you're using early SSDs), but that's just normal *wear and tear* usage as I see it, you're just causing more wear because of the re-install.

My 2c.

### Post by MoebusNet on 2020-04-15
> **bionic3epl said:**
> Does it hurt or harm my computer or PC CPU & RAM if I constantly switch from one Ubuntu version to another & back to another?

1) If you mean constantly install/reinstall different versions of Ubuntu, I would generally say "No". If you'll go to Distrowatch.com [https://distrowatch.com/](https://distrowatch.com/) and read the reviews of the distributions, you'll quickly see that *that* is what the reviewers are doing - using the same hardware to install/reinstall over and over. That said, it is possible to lose data or corrupt shared file systems.

2) If you mean changing desktop environments at boot-time, I would say "No". The ability to choose among different Desktop Environment (Ubuntu, Kubuntu, Xubuntu, Lubuntu, etc.) is one of the delights of Linux and Ubuntu generally.

3) As previously mentioned, it can be quicker, easier and more convenient to run various Operating Systems (Windows, Ubuntu, Mint, PopOS!, Fedora, Arch, etc., etc.) from a host Operating System with various guest Operating Systems installed to a Virtual Machine (ie, VirtualBox, OpenBox, etc.)
