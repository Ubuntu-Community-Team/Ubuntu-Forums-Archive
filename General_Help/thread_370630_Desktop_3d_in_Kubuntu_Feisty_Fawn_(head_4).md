### Post by Leeghoofd on 2007-02-26
Beryl works perfectly on my machine, which surprised me actually as i have it running on a Intel graphics card on one my office pc's.

Installation was a piece of cake and all I had to do is install beryl from the synaptic repository ( or use apt-get tool whichever you prefer ) and after installation run the beryl manager.  In order to get it to run it at startup i added it to the sessions. 

What i would like to know however is how other people are experiencing the load on their processor/memory etc. I've read from people that they have a 2/3% processor workload while running Beryl. I actually have some screenshots showing 40+ procent being used by beryl, altho i have to admit that this was during the active use of the Cube function... Still i think beryl CAN slow down a machine under certain conditions...

But hell who cares, it looks great:popcorn:
