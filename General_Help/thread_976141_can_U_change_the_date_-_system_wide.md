### Post by Pro-reason on 2008-11-09
The specific-to-general order (09-11-2008) is not Aussie!  It is used virtually everywhere.

The general-to-specific order (2008-11-09) is used in East Asia.  It is preferred in computing because it is more logical for sorting purposes.

The mixed-up order (11-09-2008) is just an abomination.

Make sure that your desktop environment (KDE, GNOME...) is set to use standard English (en-GB, en-AU... whatever... anything but en-US).  Apps which are part of that DE will then display dates properly.  If they don't, file a bug/suggestion.
