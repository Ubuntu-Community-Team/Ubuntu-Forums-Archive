### Post by Jukka76 on 2011-05-05
If you like to have refreshrate displayed correctly in other tools you need to set

    Option         "DynamicTwinView" "false"

under screen.

It's described here:

[http://us.download.nvidia.com/XFree86/FreeBSD-x86/270.41.06/README/faq.html](http://us.download.nvidia.com/XFree86/FreeBSD-x86/270.41.06/README/faq.html)

With DynamicTwinview enabled, it's possible to set more detailed refreshrates with xrandr, for example both 23.976Hz and 24Hz. It's not possible with DynamicTwinview disabled.

For example below modes are displayed in xrandr 50, 51, 52, 53, 54, 55.

So if you set xrandr -r 55, it actually sets "1920x1080@23", which is configured with modeline to be 23.976Hz.
So I can set any of the exact refresh rates, 50Hz, 60Hz, 59.94Hz, 24Hz, and 23.976Hz.

With Dynamictwinview disabled you would see and able to set only 50.0, 60.0, and 24Hz with xrandr.


```
Modes "1920x1080@50" "1920x1080@60" "1920x1080@59" "1920x1080@24" "1920x1080@23"

# Väldigt nära 23.976
Modeline "1920x1080@23" 74.23 1920 2560 2604 2752 1080 1084 1089 1125 +hsync +vsync

# 1920x1080p @ 24Hz (EIA/CEA-861B)
ModeLine "1920x1080@24" 74.250 1920 2558 2602 2750 1080 1084 1089 1125 +hsync +vsync

# 1920x1080p @ 50Hz (EIA/CEA-861B)
ModeLine "1920x1080@50" 148.500 1920 2448 2492 2640 1080 1084 1089 1125 +hsync +vsync

# 1920x1080p @ 59.94Hz (EIA/CEA-861B)
ModeLine "1920x1080@59" 148.350 1920 2008 2052 2200 1080 1084 1089 1125 +hsync +vsync

# 1920x1080p @ 60Hz (EIA/CEA-861B)
ModeLine "1920x1080@60" 148.500 1920 2008 2052 2200 1080 1084 1089 1125 +hsync +vsync

```
