### Post by gubemton on 2007-11-19
The impossibility of porting Mac Software to Linux relies not in the architecture but in the graphics management or window manager. GNOME uses Metacity, KDE kwm, xfce xfwm, etc...  there are really a few that are that nice as the MacOS' one (I think it's Cocoa)... anyway

There are libraries that are used to make that bindings and use certain types of applications... That's why not only there's no Amarok for GNOME (it uses KDE libs over gnome) but also the libraries have got different capabilities, tools, api's, system calls and all that stuff.

So it's not that simple. And so iwork is not open source...
