That's probably mostly because the advice you're getting from people is to use the command line, even though there are GUI ways of doing the things you want to do. There are very few things that I actually find I need the command line for on Ubuntu. I've even found one or two things that you can do through the GUI on Ubuntu that you can't on Windows. However, most people give directions using the CLI because, unlike Windows, Ubuntu has more than one possible GUI, so they don't know which GUI to give directions for. To some degree you can just remember that certain CLI actions correspond to certain GUI actions (apt-get, for example, corresponds to Synaptic Package manager). Otherwise, when you ask a question and somebody responds with a command line command, ask if there's any GUI way to do it in your GUI (GNOME, KDE, etc).

Which "Autoinstall thing" are you talking about? Add/remove programs? Synaptic?

### Post by hwttdz on 2009-11-21
Also, I'd much rather give cli directions than gui directions.  I mean how much easier is it to say, 

chmod +x filename

than it is to say

right click->properties->permissions->check execute box

I mean both are easy enough, but the gap seems to grow for some things. Besides, when you give someone a gui way, they invariably say "I couldn't find x" or "it didn't work", whereas when you give someone a cli way they say "that returned a "chmod: cannot change permissions: Permission denied""or somesuch.  Which makes it much easier for me to tell them how to fix the problem.
