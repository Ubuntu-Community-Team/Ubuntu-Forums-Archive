### Post by waperboy on 2012-05-19
I've been suspending and resuming without rebooting for months on end on 10.04 with no trouble.

After doing an upgrade to 12.04, resume does not work at all. Resume produces flashing scroll-lock and caps lock leds on keyboard (with a black screen) within 2-4 seconds of resuming, invariably. This is before anything has had a chance to happen, so there are no logs or anything.

Very frustrating, since I normally have a large number of open applications on several physical and virtual screens.

I think it shouldn't be video driver related, since resume will work fine if I disable ACPI in BIOS - but then only one CPU core is enabled.

Has anyone else had this experience, or know how to resolve it?

System specs:
Motherboard: Asus P6T SE
Cpu: Intel Core i7 930
Ram: 24Gb Corsair Vengeance DDR3
Disk: OCZ Vertex II 90Gb, WD Caviar Black 1Tb

### Post by Toz on 2012-05-21
Try the acpi_sleep parameters:
```
acpi_sleep=	        [HW,ACPI] Sleep options
			Format: { s3_bios, s3_mode, s3_beep, s4_nohwsig,
				  old_ordering, nonvs, sci_force_enable }
			See Documentation/power/video.txt for information on
			s3_bios and s3_mode.
			s3_beep is for debugging; it makes the PC's speaker beep
			as soon as the kernel's real-mode entry point is called.
			s4_nohwsig prevents ACPI hardware signature from being
			used during resume from hibernation.
			old_ordering causes the ACPI 1.0 ordering of the _PTS
			control method, with respect to putting devices into
			low power states, to be enforced (the ACPI 2.0 ordering
			of _PTS is used by default).
			nonvs prevents the kernel from saving/restoring the
			ACPI NVS memory during suspend/hibernation and resume.
			sci_force_enable causes the kernel to set SCI_EN directly
			on resume from S1/S3 (which is against the ACPI spec,
			but some broken systems don't work without it).
```

I would suggest trying:
- s4_nohwsig
- s3_bios
- s3_mode

EDIT: Or possibly a BIOS update?

EDIT2: You might also want to have a look at the hpet kernel parameter ([http://www.kernel.org/doc/Documentation/kernel-parameters.txt]("http://www.kernel.org/doc/Documentation/kernel-parameters.txt")), possibly **hpet=disable** (or disable hpet in the bios).
