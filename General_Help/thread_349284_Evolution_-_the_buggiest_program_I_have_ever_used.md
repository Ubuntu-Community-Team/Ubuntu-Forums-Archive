### Post by yigal.weinstein on 2007-02-01
It is really quite simple to move from evolution to another mail client.  This is for thunderbird but will work for other mail clients as well.

1. Make sure ~/.mozilla-thunderbird - the folder - has been created by running the client for the first time.  Don't worry about what you do to initially configure it.  Just fill in what you want, and leave the rest for editing later.

2. Close it down.

3. Find the folder(s) names you want to move to your new client.  These are the names when you open evolution, this includes your Inbox and what ever order you put in Evolution.  

4.  These are just mbx files located in ~/.evolution/mail/local/.  So, in a terminal or with Nautilus or Konq or however you like to do it.

```
cp ~/.evolution/mail/local/names_of_the_folder(s) ~/.mozilla-thunderbird/.mozilla-thunderbird/l850af01.default/Mail/Local\ Folders/

```

do not copy the rest i.e. the meta data the meta-meta data etc. just these files are enough.  The I850... may be named differently but there are very few folders so finding the folder should not be difficult.

5. Fire mozilla-thunderbird up again and presto there is your mail ready to go.

This works for mutt to thunderbird.

### Post by geoffbeaumont on 2007-06-29
I've been using Evolution as my PIM for years - I've had recurring problems with it, mostly the infamous (and still not fixed) 'Summary and folder mismatch'. As this has cropped up yet again and the usual fixes aren't working I decided it was time to have a play with Kontact, which I hadn't looked at for quite a while.

First impressions were good - it's fast, feature rich and, with a bit of messing around managed to import my (fairly large) collection of Evolution mail folders. Calender, contacts and tasks imported easily too, only things I had to set up manually where email accounts and filters.

Sadly, Kontact made Evolution look like a paragon of stability. As soon as I started setting up filters (which also handle spam filtering, so you really do need them), I started getting crashes and email corruption. Didn't take much searching to discover that Kontact (or more specifically the KMail component) has it's own set of long-standing unfixed bugs, including the one I hit. While I now know how to trigger the bug (select all then apply filters), an email app that destroys my emails is *not* safe to use, so needless to say I won't be continuing my Kontact experiment.

I did turn up something interesting when importing my mail, which might well shed some light on why Evolution is both slow and prone to problems with it's meta data - as the spam filters in Evolution are pretty much useless, I filter all mail from known sources into sub folders. When I check my mail I scan the inbox for any genuine mails then mark everything that remains as spam (hence moving it to the junk folder). My inbox was, therefore empty. Or so I thought... After importing my mail into Kontact, which I had to leave running overnight, I had over 19,000 emails in my inbox. Yep, Evolution hadn't actually deleted them!

I've found an expunge folder option in Evolution which I'm trying now, but even assuming this works it should really be automated so that Evolution keeps its own house in order. If it really has to be manual expunging only, then it should be far more obvious that you need to do this! If expunge doesn't work I'll edit the mbox file by hand - easy for the Inbox as I know there shouldn't be any emails in there, but not practical for my other folders.

Incidentally, the messing around I had to do to get my mail into Kontact was to make a copy of my Evolution mail folders then remove all meta data files from this, leaving just the mbox files. Kontact then imported these quite happily. Presumably Kontact uses the meta data to work out things like which of the emails have been deleted or moved but still left there by Evolution - can't really blame it for not managing to read it, given Evolution couldn't either.

All in all, I'd have to say that Evolution is the only viable option for a PIM app on Linux at the moment, but even then it's a long way from perfect. Kontact would be great if it worked, but with serious data eating bugs in it I'd argue it shouldn't even be present on the Ubuntu repositories. I'm not aware (and it seems, searching this forum, neither is anyone else) of any other full featured PIMs for Linux.

Geoff

### Post by geoffbeaumont on 2007-06-29
Brief update to the above - the expunge folder facility in Evolution appears to do absolutely bugger all. Tried it with the ridiculously large inbox mbox and with a small one with only a couple of deleted mails, and it did nothing whatsoever with either of them.

I deleted the contents of the inbox mbox file by hand, deleted the inbox meta data files then fired up Evolution - Summary and folder mismatch errors have finally gone away and Evolution is much, much faster (surprise...). However, this is only going to be a practical fix for people who use their inbox as a scratch file in the way I do. No way I could have gone through finding two years worth of genuine emails in amongst all the spam!
