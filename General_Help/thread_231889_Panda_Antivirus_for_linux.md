### Post by benjaminwr on 2006-08-08
Hi,

I would like to say that panda for windows seems to detect absolutely everithing that goes on on the computer, but only at the expense of using up absolutely every resource it finds available. It also alerts of an "intrusion" whenever you set up the network and computers are trying to talk to each other... so the first things that happens as soon as you install panda on a networked environment is that id doesn't work because whoever is using the computer (not tech savvy)was told there was an intrusion and was asked if he would like to block the connection...

I am a network administrator, i have only been in this company for two months and dealing with panda "intrusions",and have just convinced my boss to migrate from win to linux. And it was all thanks to panda's ability to make a year old computer run like a 386.

I run nod32 on my win conf and I am really happy with it. No resource use at all. And it detects everything. The computers that still run win at work. suddenly "run" with nod32. and the remaining have got ubuntu.

I would hate to see panda do for linux what it did for windows, haven't tried it though, but if the development team is the same... how the he..." did they manage to make panda 2006 so resource dependant.

As an anecdote... I detected a virus with panda. that it said it could'nt delete. I went to symantec and downloaded the "fix" to remove that virus.. as soon as it had finished running a panda popup came up telling me how panda had just removed a virus from my system.
