I have had a look at planner and (like MrProject and its Windoze equivalent, MS Project) it is a competent Gantt chart based planner.  Unfortunately this falls short of a ToC tool.

The key features of the ToC environment are:

1) It works **backwards** from a goal to identify all sub-goals that need to be in place to achieve allow you to reach the final goal (deliverable).  And then iteratively back from each sub-goal to the point where you have a list of tasks that are both necessary and sufficient to complete the task.  Typically, you would use a Pert chart to construct this but in (say) MS Project you can only work **forwards**.  This leads to a number of tasks included which are not necessary and (potentially) missing tasks that are.  i.e the task list is not sufficient.

2) Estimation is quite different under ToC.  Basically, you need two duration figures for each task... a 'challenging but achievable' figure and a 'safe but not excessive figure'... which leads to ->>>>

3) ToC uses the concept of a 'buffer'.  As such there is no critical path based on task dependencies but rather a 'critical chain' based on resource availability.
