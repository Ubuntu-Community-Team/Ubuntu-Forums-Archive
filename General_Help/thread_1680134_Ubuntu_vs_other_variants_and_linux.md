### Post by toolzen on 2011-02-07
Personally, if you want to learn "Linux," I would go with a generic distro like Slackware. This will teach you how to use the base programs and utilities for doing all of your day-to-day Linux administration. Once you got that down, then switch to an easier distro like Ubuntu.

The things you learn from a generic distro will be applicable across all the various Linux-based systems you find yourself working on, whereas many of the programs that Ubuntu/Fedora/etc. use for typical Linux tasks are actually just front-ends to a more universal utility.

For example, if you know how to use the apt package manager, it won't help you much if you find yourself working on a Fedora-based OS (such as Red Hat, which is very popular with businesses), but learning how to use gcc on the other hand will be useful regardless of what Linux distro you use.

Of course, if you're just wanting a "hit the ground running" OS that is a good replacement for Windows, then the more user-friendly distros like Ubuntu are a good place to start.

### Post by Nixarter on 2011-02-07
Well, IMO the best way to categorize them is by the goals that mastablasta was talking about.  The only thing is that there aren't really set-in-stone category names, and many in the same category may refer to themselves differently, based on personal preference and sub-goals.  Take Mint and Ubuntu Studio.  The goals of both are media-based, so you'll hit the ground running with little work.  However, Mint is more for casual users who like lots of file codecs and compatibility (restricted drivers et alia), and Ubuntu Studio has more recording/studio-related (FOSS) programs and settings.  Both are basically just Ubuntu with a group of programs pre-installed.

Many of the goals of the distros overlap as well, so it makes it even more difficult to categorize them.  But, the media-related ones are interesting and useful, depending on the situation.  For example, I usually use Sabayon or Mint LiveCD's (both what I consider media distros) when showing people Linux for the first time when away from my computer.  I can just download one, put it in the disc drive, and I have all the eye candy/media compatibility I could want.  I've used it for classroom presentations on Linux before, as I do not have to do any prep work at all (though many have a very hard time believing this, lol).

Another category to look into is Security.  Backtrack is a good example.
