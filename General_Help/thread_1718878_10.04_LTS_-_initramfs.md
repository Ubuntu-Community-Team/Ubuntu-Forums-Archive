### Post by drs305 on 2011-04-01
At the grub menu, if you press 'c' to get to the grub prompt and then type the following commands:

```

ls # You should see drives (hdX) and partitions (hdX,Y)
ls (hdX,Y)/  # Do you see 'vmlinuz' and 'initrd.img'
ls (hd/X,Y)/boot # Do you see the kernel and initrd image?  vmlinuz.2.6... initrd.img-2.6...
ls (hdX,Y)/boot/grub  # Do you see a lot of *.mod and grub.cfg ?

```
X,Y are the Ubuntu drive (0,1,2, etc) and partition (1,2,3,5)

### Post by cody.t.sterling on 2011-04-01
After more hours than I would like to admit, I finally found the solution!  I figured the problem was that my GRUB was corrupted somehow, so I followed Rubi1200's advice to hold down 'shift' while booting and then pressed 'c' to get to the command line.  I then used the following command to remove the partition table:

```
dd if=/dev/null of=/dev/sdX bs=512 count=1
```

Next I just booted to a 10.04 LTS live CD and I was in business!

Thanks for all the help!


Cody
