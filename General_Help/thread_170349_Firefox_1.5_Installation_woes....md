### Post by jazzmuzik on 2006-05-04
re: [https://wiki.ubuntu.com/FirefoxNewVersion](https://wiki.ubuntu.com/FirefoxNewVersion)

I do a similar installation as above link. However, since I'm a single user computer, I install Firefox in my user directory. Upgrading from the web is pretty easy that way. 

The one problem I encountered installing a FF binary from the mozilla.org site was -- back when I used Fedora -- I had to install the "compat-libstdc++" package to make it work. Mozilla.org is apparently compiling their FF binaries with the need for external libraries and compat-libstdc++ is just such a thing it needs.

Then when I started working with Ubuntu a couple of months ago I found I had the same difficulty: a library package was needed in order to install a mozilla.org ff binary, but Ubuntu's libraries are named differently. I cannot remember which lib I installed to make it work, but my guess it's this package:

libstdc++5

To the original poster. Install the above library and see if FF loads.


OH SHEESH. I just looked at the linked page and it mentions everything I just said:

```
sudo apt-get install libstdc++5
```

On the bottom of that link where it says, "Installing Firefox quick and dirty" is basically what I've done.
