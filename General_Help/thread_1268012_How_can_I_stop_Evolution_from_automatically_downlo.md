### Post by nbtrap on 2009-09-16
I'm using Evolution with my GMail IMAP, and every time I try to view an email that contains both text and an attachment (let's say, an image), Evolution will not display the text until it has also downloaded the image(s) for preview display. I'd rather it download only the text and the attachment file name(s) and let me download the attachments manually if I wish, which can be accomplished in Thunderbird, for instance, by unchecking the "Display Attachments Inline" option. This is very annoying with Evolution, because I have to wait, depending on how large the attachments are, upwards of minutes, for an email to load, even when I only want to view the text of the email.

I understand that Evolution has a "Download Attachments Automatically" option. However, this is misleading, for it should read "*Save* Attachments Automatically," since Evolution already automatically *downloads* the attachments, only it saves them to a temporary folder to be discarded later. Is there any way I can stop Evolution from automatically downloading the attachments, even for temporary viewing?

### Post by dranockcir on 2009-09-16
Have a look at Edit|Preferences, click Mail Preferences on left, choose the HTML Messages Tab and check "Never load images from the Internet."

Rick
