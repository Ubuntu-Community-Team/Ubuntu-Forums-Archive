### Post by eth on 2006-06-14
Hi, I had the very same problem, so here you are a quick walkthrough to get the stuff working

. first of all, upgrade your dist with **apt-get update** followed by **apt-get dist-upgrade**
. go to [www.ati.com](www.ati.com) and get the latest driver
. place them in file system folder (/)
. be sure to have "make" and "gcc" installed
. type (as root) **./ati-driver-<arch>-<version>.run**
. it will pop up a conf tool, answer all question with preselected options
. it will say "an error occurred during installation, log is in /usr/share/fglrx/fglrx-install.log but no matter
. open as root the file /etc/X11/Xorg.conf and be sure he takes fglrx as driver for your  video card
. type in console **cd /var/modules/fglrx/build_mod/** and from here type **./make.sh**
. then **cd ..** and from here **./make_install.sh**
. he will tell you he cannot load fglrx module, but no matter, he's using radeon, so just **reboot**

now it should work everything

### Post by Cataphract on 2006-06-14
I had the same problem, with a different ATI Radeon model, just yesterday (I fixed it late last night)

Go here: [http://www.ubuntuforums.org/showthread.php?t=188326&page=7]("http://www.ubuntuforums.org/showthread.php?t=188326&page=7")

Review pages 7 through 9 in that thread. The problem may be a result of your xorg.conf file (at /etc/X11/xorg.conf) being improperly set-up to have two different Screen and Device sections. Note my solution to the problem on page 9.

To be more specific, notice in your xorg.conf file that you posted that you have two different Screen sections for which you have the Identifier "aticonfig-Screen[0]"? You should probably comment out (by putting '#' at the beginning of each line) the second one that references "aticonfig-Device[0]" and keep the first one that references "ATI Technologies, Inc. Radeon Xpress 200M (RS480)". At this point, your Screen section is pointing to the correct Device...but the problem is the Device section is wrong! Comment out both existing Device sections and add these lines:

> 
Section "Device"
Identifier "ATI Technologies, Inc. Radeon Xpress 200M (RS480)"
Driver "fglrx"
BusID "PCI:1:0:0"
Option "UseFBDev" "true"
EndSection


Now, your Screen section points to the proper Device section, which in turn points to the proper Driver ("fglrx"). I hope this helps.

[QUOTE=jamesford]

I think your problem is here, the bold part. It usually means that you haven't installed the linux kernel headers properly (or wrong version):

try this:

. open a shell
. type **cd /boot** and **ls**
. there is a initrd.img-2.6.xx-xx-386 file if you are working on an Intel based machine, or you may find a -amd64- token (instead of 386) if you are running on a AMD64 based machine
. type **apt-get install linux-image-2.6.xx-xx-386** or eventually -amd64- token just like it appears in initrd.img file
. type also **apt-get install kernel-headers-2.6.xx-xx-386**
. try running the procedure again
