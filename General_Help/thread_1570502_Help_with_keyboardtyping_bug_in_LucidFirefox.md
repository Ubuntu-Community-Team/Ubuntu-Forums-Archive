### Post by BetterSense on 2010-09-08
It appears that my wife has this keyboard bug. Sorry, it's hard to explain, but debilitating, and we have not found a root cause. It appears only under her user account. My user account never has it.

Description:
When typing in a text box in Firefox (when computing, she's using Firefox 90% of the time; I'm not sure if the bug only happens in Firefox), text appears in the text box as its supposed to. Then, randomly, and with no warning, all the text that has been entered up to this point will become highlighted, as if the user pressed Shift+Home. Any subsequent keystrokes delete all the text in the box and begin at the beginning of the box again. This is extremely annoying. 

This can happen repeatedly while you are trying to type. So for example if you just went to type, without stopping, "The quick brown fox jumps over the lazy dog" you might see in the text box "The quick brow" then the bug would occur and if you kept typing you would see "n fox jumps over" then if it happened again you would see " the lazy do" then, finally, you might just be left with "g". 

Somedays this bug is not present. Sometimes it happens at almost every keystroke. Then the only way to type any text online, such as online forms, is to type, very slowly, one letter at a time. If the bug happens, all the text will become highlighted, and the user can press the right arrow key or click to place the cursor at the end of the line again. Then the user can type one more letter, wait several seconds to make sure all the text does not spontaneously highlight, and then type another letter, and so on. If the user goes too fast all the text will become highlighted and typing the next character deletes all the text leaving only the latest character. 

We can't figure out what triggers the bug, we have switched keyboards. I'm running a fresh install of Lucid Lynx, and I added my wife's account after the install.
