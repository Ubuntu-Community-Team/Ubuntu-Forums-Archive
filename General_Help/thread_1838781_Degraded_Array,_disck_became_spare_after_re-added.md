### Post by brucehohl on 2011-09-04
You did not show the add command you used to add sdb3 back into md3 but it should be like this:
% mdadm /dev/md3 --add  /dev/sdb3

If an array fails more than a few times you should power down the box and check the disks with the manufacturer diagnostic tools. Strongly suggest you replace the disk if it fails the diagnostic tests.

### Post by brucehohl on 2011-09-04
I have not used spares in my RAID1 set-ups but you might try something like this:

# mdadm /dev/md3 --fail   /dev/sdb3
# mdadm /dev/md3 --remove /dev/sdb3
# mdadm /dev/md3 --add /dev/sdb3

The idea is to fail and remove /dev/sdb3 as a spare from the array /dev/md3, then add back as part of the mirror.

# man mdadm is quite comprehensive.

### Post by brucehohl on 2011-09-04
Since you are getting unusual errors you should power down the box and check the disks with the manufacturer diagnostic tools. For example, Seagate has a free bootable tool to check Seagate disks.  Replace the disk if it fails the manufacturer's diagnostic tests. You can get unexplainable errors from bad hardware so check the disks before further trouble shooting. If you need to replace the disk and are unsure of the procedure I can post my notes for doing so with RAID1 which I have used several times.
