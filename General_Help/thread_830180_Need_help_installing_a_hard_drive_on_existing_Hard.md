### Post by drs305 on 2008-06-16
First, I agree that formatting it to ext3 has advantages to ext2. If you are going to do that, change the format before continuing. Ext3 enables journaling, which means if your computer is unexpectedly shut off there is less chance you will lose recently-changed data.

Make a mount point to mount /dev/**sda1**. You can pick any name you want. Substitute the new mount point name for sda1 where it is typed in bold. Type in YOUR username:
```
sudo mkdir /media/**sda1**
sudo chown -R **username:username** /media/**sda1**
```

Make a backup copy of /etc/fstab and open text editor (you can use another if you don't want gedit):
```

sudo cp /etc/fstab /etc/fstab-bak1
gksudo **gedit** /etc/fstab

```

Add line. Change the bold type to the mount point you created. If the drvie si still formatted as ext2, change that entry as well:
```

# Entry for /dev/sda1:
/dev/sda1 /media/**sda1** ext**3** defaults,user 0  2

```

If you would prefer to have UUID identify your sda1 partition:
```
sudo blkid
```

Note the UUID for /dev/sda1 and replace **/dev/sda1** with the entire UUID number:
```
UUID=**XXXXXXXXXXXX** /media/**sda1** ext3 defaults 0  2
```

Refresh fstab (you will get some "device is busy" messages; that's okay.
```

sudo umount -a
sudo mount -a

```
Your sda1 should now automount on boot.
