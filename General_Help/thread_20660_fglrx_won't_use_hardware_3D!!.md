### Post by emperor on 2005-03-17
Your xorg.conf file does not look like it was generated by fglrxconfig as there is a bunch of fglrx specific setting missing. I would backup your xorg.conf and then run fglrx (as root).

You'll need the info in the original xorg.conf file concerning the mouse, keyboard and etc.

In fglrxconfig select:
Yes for "external AGPGART
No to "Locked user pages

Your new file may be missing some fonts too, specificly the True Type fonts, but you can add these manually

### Post by emperor on 2005-03-23
When you type "lsmod", does it show the fglrx module as loaded?

The "fglrx" module must be loaded explictedly in the "/etc/modules" file.
The "echo" command below will add it for you instead of doing: 
"sudo vi /etc/modules".

The below snippit is from from [http://www.ubuntulinux.org/wiki/BinaryDriverHowto/](http://www.ubuntulinux.org/wiki/BinaryDriverHowto/)
[list=1]
[*]  echo fglrx | sudo tee -a /etc/modules  

[list]
[*]   This command won't have any effect until you reboot.  To load the kernel module immediately, run sudo depmod -a ; sudo modprobe fglrx, but this will only work if you have already rebooted since upgrading the kernel. If it doesn't work, perform the next step (below) and then reboot. 

[/list]   
[/list]
