### Post by nothingspecial on 2011-01-21
This will tell you what`s taking up the space
```

du -a -x / | sort -n -r | head -n 20
```

du tells you the space taken up
-a means files and directories
-x means don`t check other file systems (your 2 other drives which I am guessing you want excluding from the search)

sort - I think that`s kind of obvious
-n numerically
-r reverses it

head -n 20 shows the first 20 lines (because we have sorted the output of du to show everything in descending numerical order it will print out the largest 20 files and directories)

### Post by nothingspecial on 2011-01-21
Well, you have a load of stuff in your home directory

```
26886000	/home/chris
```

Which is mainly being taken up by a gigantic .xsession-errors file

```
25840804	/home/chris/.xsession-errors
```

So the first thing to do is to delete that, ubuntu will create a new one so don`t worry. But keep an eye on it, the size of it suggests something major is wrong. In fact have a look at it before you delete it, there will be some info to stop this happening again.

You also have a load of stuff in your apt/archives

### Post by matt_symes on 2011-01-25
Hi

> The tweak cleaner I wasn't aware off but I'll try that too. 

It has a section to remove packages or you can do it by hand.

> sudo apt-get clean
sudo apt-get autoremove
sudo apt-get autoclean


from man apt-get

> clean
           clean clears out the local repository of retrieved package files.
           It removes everything but the lock file from
           /var/cache/apt/archives/ and /var/cache/apt/archives/partial/. When
           APT is used as a dselect(1) method, clean is run automatically.
           Those who do not use dselect will likely want to run apt-get clean
           from time to time to free up disk space.

       autoclean
           Like clean, autoclean clears out the local repository of retrieved
           package files. The difference is that it only removes package files
           that can no longer be downloaded, and are largely useless. This
           allows a cache to be maintained over a long period without it
           growing out of control. The configuration option
           APT::Clean-Installed will prevent installed packages from being
           erased if it is set to off.

       autoremove
           autoremove is used to remove packages that were automatically
           installed to satisfy dependencies for some package and that are no
           more needed.

You could also uninstall packages no longer required.
