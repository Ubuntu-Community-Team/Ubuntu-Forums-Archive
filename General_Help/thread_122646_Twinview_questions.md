### Post by localzuk on 2006-01-29
I use kaffeine for viewing movies and I use the following:

I have a file in ~/.kde/share/apps/konqueror/servicemenus/

called 'displayontv.desktop'

In this file I have the following:

```

[Desktop Entry]
Actions=PlayOnTV
Encoding=UTF-8
ServiceTypes=all/all

[Desktop Action PlayOnTV]
Exec=kaffeine -f -x sdlvideosync --display :0.1 %F
Name=Play this movie on TV
Icon=kaffeine

```

This then appears on every file I right click on in the 'Actions' menu. It is possible to limit it to just the file types you want, but I forget how.

To keep it simple, I would either select a single file or create a playlist and right click and run that.

It should work for totem also.

Totem uses the --display=:0.1 format for a command line option. You can also just create a link on your desktop which runs 'totem --display=:0.1' for you instead of the right click action.

I don't use totem myself - found it too unstable so I don't really know much more than that. I would just advise taking a look at totem --help from a terminal.

### Post by mo_x on 2006-01-29
You can set the DISPLAY environment variable to start something on another display. I use a little script to start mplayer on the TV.
```
#!/bin/bash
export DISPLAY=:0.1
mplayer -fs "$1"
```

### Post by randomnote1 on 2006-02-17
In the 

Option "TwinViewOrientation" "Clone"

change "Clone" to "RightOf" or "LeftOf" if you want to utilize both screens at the same time.  The NVidia driver will "trick" X into thinking that it's displaying on one large screen, but it acts at two different monitors.  Kinda weird, but it works!
