Text editors and IDE discussions have a tendency to rely on mere personal preferences and to become holy wars.

Personally, I'm comfortable with XEmacs for my coding, GNU Makefiles for compilation, and gdb for my debugging needs.  There are, of course, deeply depraved heretics who like vi (*shudder*).   'ddd' is a GUI for gdb, but I don't recall using it more than once or twice.

Many, including quite possibly yourself, would probably prefer to use an integrated development environment.   Eclipse is one popular example; it's perhaps more commonly used for Java, but it can handle C/C++ development too, apparently. 

As a side note, it might be good to learn about version control systems, too (svn, notably).  They'll be very useful if you ever are in a group project.
