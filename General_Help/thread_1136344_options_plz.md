### Post by mb_webguy on 2009-04-26
This is actually one of the points Ubuntu shines in my opinion in comparison to some other distributions that use rolling releases.  Since software in the official repositories doesn't change except for bug fixes and security updates, but versions are officially supported for 18 months to 3 years (for LTS desktop releases), you can install an older version of Ubuntu and not have to worry about any of the software you like being automatically upgraded to a new, less desirable version.  And for the software you *do* want to upgrade, you can add PPAs, install deb packages from official project websites or GetDeb, or compile it yourself.

Synaptic will also allow you to lock a package to a specific version, so even if it a new version is released through one of your software sources, it won't be updated.

Or you can go the other way around, and install the latest version of Ubuntu to get the latest versions of most software, and then install older versions of specific applications.  I use Amarok 1.4 as my music player, but the version in Jaunty is Amarok 2, which is completely different.  So I found a PPA that provides Amarok 1.4 for Jaunty.
