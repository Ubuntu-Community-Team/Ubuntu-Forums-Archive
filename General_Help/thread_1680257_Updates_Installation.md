### Post by janpol on 2011-02-02
Hi! Lets see, usually, a new version of a piece of software aims to:

.Fix bugs
.Fix a security hole
.Add new functionality

For my PC at home, I just take a quick look at the packages that are going to be updated and run the update, I really don't worry too much about it (and I love having the latest and greatest :P).

At work is a little different, specially on servers, I usually wait for a while before updating anything (in order to see if somebody reports that an update is breaking stuff), and I also run a little test of the updated packages on a virtual machine. 

About the "If it ain't broke, why fix it?", in this case sometimes it is broken, you just haven't experienced it yet :P (specially when it comes to security bugs) or maybe there is a new cool feature for an app or a performance boost. If you are worried about stability, try using the LTS versions (10.04 is the latest one) instead of the regular ones. If you wanna know what's being updated in your system thats great, but don't worry too much about it (I assume we are talking about your personal PC here, not a server) and remember to always have backups ;) (even if you do check everything you can have a HD failure of something like that, things just break).

Regards!
