I am a Linux/Ubuntu noob, and Quickbooks was one of my issues as well. Actually, Quickbooks is overkill for me, but I inherited a hand-me-down version once upon a time and am kind of stuck, as there is (as of yet) no way to reliably and simply export QB data to another format.

As a noob, I am dual booting Feisty and Win XP while I get used to Ubuntu. My machine is a Pentium 4 3.2 Ghz with 1 GB RAM. I ended up going the VMWare route (as a main motivation for moving to Linux was to go all free). Also, as a noob, I am only running stuff I can get through add/remove programs or through Synaptic Package Manager (I am not interested in learning how to compile programs at this point).

Anyway, I installed VMWare Server through synaptic and used that to build my virtual machine. I installed Win XP on that machine and then Quickbooks 2004 (no payroll, so no need for me to upgrade). I've had no problems running QB in this way. It is not as fast as Ubuntu, but it is not any slower than a fully configured Win XP install, either. Just be sure to install VMWare Tools after the Windows install, which VMWare Server will prompt you to do.

Once I got my virtual machine up and running, I started using VMWare Player to run the machine. For your assistant who needs to take things slow, the Player is much less intimidating than the Server. I run it on a separate workspace and everything stays nice and tidy.

I've gone on for a while now, but one more thing I was able to do is share my .QBW file between the virtual machine in Ubuntu and the native XP install on my harddrive. Any changes I make in one are there when I run QB from the other!

The only concern I would have about recommending this method is if you use QB in a multi-user format. But if it is just one person on one machine at one time doing your books, and if that computer is well-equipped in terms of processor and memory, then it may work well for you.
