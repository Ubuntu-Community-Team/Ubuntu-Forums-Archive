### Post by mcduck on 2010-08-01
The desktop is one and the same for all virtual desktops, this is how it has always been in Gnome (and most desktop environments/window managers.

As far as I know KDE is the only desktop environment where you can easily get different icons on the virtual desktops.

### Post by WorMzy on 2010-08-01
By default, Nautilus renders your desktop, and it uses the contents of ~/Desktop. So anything in ~/Desktop will be rendered on all workspaces. I'm not sure if other file managers behave in the same way, but it appears Dolphin (or Krusader, or Konqueror) doesn't (as indicated by mcduck's post about KDE).

### Post by ajgreeny on 2010-08-01
Do you by chance mean that any windows in workspace 1 which show as buttons in the bottom panel window list applet are now showing in all workspaces?  If you do not like that behaviour you can change it to how you want it by right clicking on the vertical bars to the immediate left of the window list button and in the preferences choose "Show windows from current workspace"

If your problem is different to this, my apologies and ignore me, many people do!
