### Post by BarisBlaq on 2009-10-29
First off, I'm no expert, just wanted to share how I solved this annoying bug which I think others might be having now.

There is a bug in the 2.6.31.14 kernel (which I think comes from 2.6.31.08 that affects mostly laptops, and results in the network adapters (wired and/or wireless) not being recognized.

Also as a result, the 9.10 live cd does not install, since (I think) the install proces requires a live internet connection. In my case, when I initiated install, the ubuntu logo would display for about a minute, and then the system would just reboot. And then again, over and over. Running 9.10 from cd (with no install) worked, but there would be no network connection.

If you're having similar problems, to install 9.10, you need to choose "acpi=off" from the F6 menu of live cd. Try it without installing first. If that option cures the network connection issue, you are probably being affected by the same bug.

Before I forget, the original bug thread is here:
[http://bugs.launchpad.net/ubuntu/+source/linux/+bug/418933](http://bugs.launchpad.net/ubuntu/+source/linux/+bug/418933)

So, you can install after choosing "acpi=off", but you'd end up with no network connection. There are 2 ways to cure this.

First, you can edit grub to enforce acpi off. This is easier, but in my opinion, might be problematic because it could cause overeheating on the cpu, and also will probably render obsolete the sleeping function of the laptop. In my case, the laptop wouldn't even shut down, I had to physically press the power switch to shut it down. Anyways, here's how to do it:

```
sudo gedit /etc/default/grub
``````
[I][COLOR=Blue]on the line where it says[/COLOR]
[/I]GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"
[COLOR=Blue]*change it to*[/COLOR]
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash acpi=off" 
```save and exit, and then

```
sudo update-grub
```When you reboot, network should be up.

The second solution, which takes slightly longer, but (in my opinion) safer, goes like this:

From
[http://people.canonical.com/~apw/lp386468-karmic/]("http://people.canonical.com/%7Eapw/lp386468-karmic/")
download the file that ends with "all.deb" (3rd from bottom)
download the file that starts with "linux-headers" and suits your system (either i386 or amd64)
download the file that starts with "linux-image" and suits your system (either i386 or amd64)

After getting all 3 files, right click, choose "open with GDebi", and install the packages IN THE ORDER ABOVE.

This installs a modified version of kernel 2.6.31.12 with the bug corrected. Notice that this is 1 step behind the 2.6.31.14 kernel that comes with the normal 9.10 install. I wasn't able to find a debugged version of that kernel.

After installing the kernel, grub should have normally updated, but just to be safe you can
```
sudo update-grub
```it only takes a couple seconds anyho =P

And then you can reboot, and at the grub menu choose the kernel ending with .12 and not the .14 on the top. The network should be ok.

Again, I'm no expert, and I just wanted to share how I solved my case. If it works for you as well, I'd be glad. If not, I guess someone else on the forums can help you. Or if that someone else wants to help me with a better solution, I'd be even more glad.
=)

Cheers..
