### Post by Dr_Willis on 2009-04-25
Why use 64bit when theres no 64bit optimized apps, why no optimized apps because no one uses 64bit... Catch 22.  to get around the 'catch 22' of the whole thing, people just  need to start using 64bit. 

 A year ago i would of suggested sticking with 16 bit. But now  64bit has progressed so well that I promote it.


There are some 'issues' to watch out for that i have seen discussed on other sites. _Some of these old problems may no longer be an issue._

Driver Support - some drivers have not been ported to 64bit (not seen this issue lately

Java  Support  - No longer an issue from what ive seen.

Flash - Same - theres 64bit flash out, and 16 bit flash can work.


Some programs are not ported to 64bit.  This is true for at least one app that i know of 'zsnes' - because that program uses assembly programing in some parts.  But i cant think of any others.

Other issues ive heard of. (which may not be true any more)

Increased Power usage, less battery life.  More memory usage. 

Benefits of 64bitness. (exact gains may vary)

Ability to use more ram. 
Faster speeds at very math intensive tasks. Like video encoding.

Others? 


In short. Give it a try.  Ive not heard any issues of 64bit distros in some time.
