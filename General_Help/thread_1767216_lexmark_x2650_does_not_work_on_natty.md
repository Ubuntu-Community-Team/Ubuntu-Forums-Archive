### Post by retrokid on 2011-07-14
Ok, I think you guys are having the same issue as me. The key part of the error log for me was: 

'D [25/May/2011:13:08:08 +0200] [Job 50] /usr/local/lexmark/lxk08/bin/printdriver: Permission denied',

I solved this issue with this fix:
[http://islandlinux.org/howto/upgrading-ubuntu-breaks-printer-cupsys](http://islandlinux.org/howto/upgrading-ubuntu-breaks-printer-cupsys)

The lexmark uses a third-party driver which was not recognised by
AppArmor after the update. I opened /etc/apparmor.d/usr.sbin.cupsd as
root and scrolled down to the "  # third-party printer drivers"
section. Then I added a line to tell AppArmor to make the driver a security exception.

The line I added was:

 /usr/local/lexmark/lxk08/bin/printdriver Ux,

The section now looks like this:

  # third-party printer drivers; no known structure here
  /opt/** rix,

  # FIXME: no policy ATM for hplip and Brother drivers
  /usr/bin/hpijs Ux,
  /usr/Brother/** Ux,

  /usr/local/lexmark/lxk08/bin/printdriver Ux,

Be careful when editing this file - make a backup in advance. If you put in an invalid line cups will not open after the next restart. Hope this helps.
