### Post by slakkie on 2009-08-18
[7zip is available on Ubuntu](http://packages.ubuntu.com/search?suite=default&section=all&arch=any&searchon=names&keywords=7zip), but not with a GUI. From their own website:

>

The other tool I don't know about, but their sf.net page says the following:

> 
[http://sourceforge.net/projects/freedownload/](http://sourceforge.net/projects/freedownload/)
Using this free download manager and accelerator, you can download files and complete web sites up to 6 times faster than before. **Also we plan to port it to Linux and Mac OS.**

And because something is open source, doesn't mean Unix/Linux can run it. Open source is just about giving people the freedom to modify the source so you can do with it what you want as a user. It does not mean, must run on Windows, or must run on Linux, etc.

### Post by 3rdalbum on 2009-08-18
The developers of Free Download Manager probably don't think anyone would benefit from their software on Linux; after all, we all just use wget (don't we?)

Why do we need the GUI from 7-zip, when most people will use File Roller or Ark which comes preinstalled with their distribution? That's probably what the 7-zip developers are saying.

Windows is a very different operating system and some programs that are needed for Windows are simply not needed on Linux.

Some other reasons could be: They want to use GPL (not LGPL) libraries or programs in their own program, and the only thing they know about open-source is that they must open-source their software in order to legally use GPL code. Or they might know a lot about the GPL and Free Software/open-source but not actually know anything about the libraries and system calls available on Linux.
