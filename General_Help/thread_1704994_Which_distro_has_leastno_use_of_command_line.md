### Post by tordeu on 2011-03-11
The battery life of windows 7 starter an Kubuntu should not be too far apart. I remember once having an old Ubuntu an Windows XP on a laptop and Ubuntu gave me slightly less battery life (only slightly and barely noticeable without a stop watch), but that was like Ubuntu 6.X or something, if I remember correctly. 
I do not think that you really need something more "lightweight" for a netbook. I had Ubuntu running on a netbook and will be again, soon.  The RAM in most netbooks is 1GB or more, which should be enough, although I upgraded to 2GB was very happy with that.

So, I would try Kubuntu first, if you want it and switch only if you think it feels to heavy for the machine. In that case you could try Xubuntu, for example.

I also think that it is not necessary to use the command line in Ubuntu. It's just that you can achieve things very quickly if you know how to do them. And you have the opportunity to combine several commands to achieve things quickly instead of trying to find a GUI which supports something like that.
Pretty often, the GUIs are just using "the command line" anyway, so you just choose some options and then it executes the necessary command to achieve this. The problem is that there are often several GUIs. That leads to 2 problems:

1. If someone asks a question, not everybody knows that particular GUI, but they know the 1 or 2 command lines which will fix your problem; far easier than to say "click here, then select that, then scroll down there,...

2. A lot of GUIs do provide an interface to a command line program, but they to not offer all options or all possible combinations of options. And you can't use other command line utilities in combination. For example: If you use tar to make a backup and use date to retrieve the current date and time, so that you can name you backup "foo_yyyymmdd_hhmmss.tar", then a GUI which might use tar to backup some stuff, might not support this exact format, but instead only offer use 2 or 3 alternatives. In those cases you might not even find a GUI anywhere which will do what you want. (Of course, this is not a real-world example, but it illustrates the problem...a little).

Long story short: In most cases you won't need command line in Ubuntu (Ubuntu is designed to be user friendly and that includes the idea of offering easy user interfaces), but if you run into problems you might to copy&paste some commands from this forum until the problem is fixed. Also, it's still a long way to go for Ubuntu. There is still a lot of cases where (good) GUIs are missing, partly due to the fact that it is a hell of a job to create an ergonomic, user-friendly GUI for a program. In a lot of cases it might be more work to create a great GUI than to create the program itself.

Almost all problems can be solved in a multitude of ways, some CLI, some GUI (in most OS, not just Linux-based).
In a text-based help forum, such as this one, CLI is easier and quicker to *describe without risk of misinterpretation*. Nothing to do with feeling superior.
As for which distro to use - if you are using your netbook only for web-based activity, a netbook-specific distro such as JoliOS (or PepperMint) might be worth considering. I've not tried either - but both appear to have positive reports - whereas Lubuntu seems to have received a mixed response.
