### Post by dorlow on 2010-11-21
Hi, I have dual monitor.  It's my laptop and one external monitor.  It works fine except the monitor on the laptop is shorter height-wise than the external monitor, and the resolution reflects that.  When moving my cursor to the top of the laptop screen, it does stop at the edge.  You can't move it into the "void" area.  But, icons can go and get stuck there.  I can get them out by opening up the desktop folder and dragging them to somewhere where I can see it. 

Using Windows, they seem to figured that out.  You don't have that issue with Windows.  Is there a trick or a fix for this, other than changing both monitors to the same resolution?

Just to clarify everything, I'm attaching a screenshot.  You'll see the black in the upper left.  That is where icons get stuck.  You can't see it.  It's above the laptop monitor.  It's like Linux insists both monitors are the same resolution.  If they're not, it will make it that virtual resolution.

### Post by dorlow on 2010-11-24
Think I just figured out a work around.  I always delete the top toolbar from the desktop and make it similar to Windows XP.  If I leave the toolbar at the top, it stops the icons from going into that void area.

### Post by joepie91 on 2010-11-24
Something I found in OpenSuSE/GNOME (when using the default graphical settings dialog): you can actually drag the monitor to change the location compared to the other monitor. Try dragging it up so the top border aligns instead of the bottom. Maybe it works in the nVidia software as well.
