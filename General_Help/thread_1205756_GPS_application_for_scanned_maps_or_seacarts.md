### Post by sinsailor on 2009-07-06
Hi,

I'm searching for a GPS plotting application that can use scanned maps and (preferebly) even calibrate them from the real world. Any ideas will be appriciated.

I've been using a nice little $40 utility called SmartComGps on my Nokia N95 phone to handle raster seacarts. Its a simple GPS plotting application that allows me to load a map (PNG or OGF2 format) into the phone and then calibrate it by pointining out 2 or more coordinates at the map. The phone is then able to plot my position on this seacarts when travelling.

However, the screen on the phone is so small that I've decided to set up a laptop in the boat instead, running Linux and a GPS. The problem is that I CANT FIND any application in the whole open source/Linux/GNU community that does the same thing.

I've tested:

GpsDrive [http://www.gpsdrive.de/](http://www.gpsdrive.de/)
It's very hard to read the included/downloaded maps. In the documentation, it seems to be a feature for installing your own maps. However, then those maps need to be exactly 1280x1024px and there is no calibratation feature like the one at my phone.

TangoGPS [http://www.tangogps.org/](http://www.tangogps.org/)
Best so far. Very nice GUI for beeing a Linux application. However this tool lacks support for installing your own maps since its using the OpenStreetMap API to get feeded by maps from the Internet. It should be possible to creat your own OSM service (like this: [http://b.tah.openstreetmap.org/](http://b.tah.openstreetmap.org/) with TilesAtHome) but I cant figure out how to create the needed OSM format for creating the PNG files. 

Navit [http://www.navit-project.org/](http://www.navit-project.org/)
I cant understand how to configure this program. Seems vector/internet based and has no documented feature for installing scanned maps.
