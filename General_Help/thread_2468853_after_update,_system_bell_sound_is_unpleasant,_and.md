### Post by pep37619 on 2021-11-12
I'm running Ubuntu 20.04 LTS, and this morning it wanted to install some software updates, so I let it.

Before the update, the system bell was something short, minimal, and pleasant-sounding.

After the update, the system bell sound is unpleasant, like metal hitting metal.  More importantly, it is now doubled.  The sound happens initially when it is supposed to, but then a slightly muted version of the sound plays again about 2 seconds after the first one.

Under "Sound" in the "Settings" application, I tried changing the "Alert Sound", but that did not seem to have any effect on the sound that is actually played for an alert.  The "previews" that were played in the Settings application sounded correct, but even they were doubled.

So, why is my sound doubled, and why can't I change the alert sound?  It was fine before the update this morning.

### Post by grahammechanical on 2021-11-12
Sometimes an update/upgrade will set certain things back to default settings. Click the sound icon in the top panel and see if is set to maximum. Would reducing that setting make a difference? Is System Settings>Sound>System Volume set for Over-amplification?

Regards
