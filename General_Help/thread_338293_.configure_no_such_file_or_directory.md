### Post by ardchoille42 on 2007-01-14
Just to make things clear.. the "./configure" command is not an Ubuntu command, well, it is and it isn't. When you run ./configure, you're telling Ubuntu to "find the configure script within the source files and execute it" and then the system executes the configure file to generate/configure the make file. The configure item is usually a script that is included with source code and it is a file: the configure file. There are a lot of source code tarballs that don't include the configure script but use a makefile that is already configured, so you then just run "make". The best thing to do, after unpacking a source tarball, is to read the README and INSTALL files if there are present, which should give you some info on how to build the app.
