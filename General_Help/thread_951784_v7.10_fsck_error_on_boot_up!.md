error:

fsck 1.40.2

/dev/shm/root: Resize inode not valid

/dev/shm/root: unexpected inconsistency: run fsck manually

I then get dumped out into a maintenance shell asking for the root password.  It says "root filesystem has been mounted in read only mode.  However if I try to run e2fsck it says that the root filesystem is mounted.

What would be the best way to handle this problem (including command examples).  Does the installation disc have some kind of rescue mode on it (shell, through which I can umount the root file system i.e /dev/hda1

### Post by RealPSL on 2008-10-18
The best way is to probably boot from the Ubuntu Live CD. Open a terminal (Applications > Accessories > Terminal). Run the command ```
sudo fsck /dev/hba1
``` Where /dev/hba1 is your root partition. Repeat the command until the file system is clean and then reboot with ```
reboot
```
