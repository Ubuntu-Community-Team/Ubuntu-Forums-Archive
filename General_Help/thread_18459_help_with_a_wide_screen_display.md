### Post by yippy on 2005-03-09
You'll need to create the Modes section, then add a UseModes line to  your Monitor definition.

My display is 1280x800, so yours will be different, but this is what mine looks like:

```

Section "Device"
	Identifier	"NVIDIA Corporation NV17 [GeForce4 440 Mac/GeForce 440 Go 64M]"
	Driver		"nv"
	BusID		"PCI:1:0:0"
EndSection

Section "Modes"
	Identifier      "1280x800"
	Modeline	"1280x800" 84 1280 1344 1480 1680 800 801 804 828
EndSection

Section "Monitor"
	Identifier	"Generic Monitor"
	Option		"DPMS"
	HorizSync	28-60
	VertRefresh	43-60
	UseModes        "1280x800"
	DisplaySize	330 260
EndSection

```

I'm using the nv driver at the moment because the 6629 nvidia driver croaks on my display.

### Post by bdalzell on 2007-02-18
I just (Feb2007) purchased an Acer 1680 by 1050 LC display and Ubuntu 6.10 was only giving me the option of the 1280 x 1024 display. Then i tried editing the /etc/X11/xorg.cof file as has been suggested in many of the posts I tracked down on display problems. This resulted in becoming unable to start X11. However the solution for me was simply to do a cold boot and get into command line mode and run:

sudo dpkg-reconfigure -phigh xserver-xorg

which is what the analysis message that appeared when I tried to use the repair system disk (Ubuntu 6.10 Alternate) told me to do - but you have to scroll down through a lot of text to find the suggestion.

After that when I rebooted the plug and play attibutes of this Monitor combined with my graphics card (an Nvidia gForce4 Ti) resulted in Ubuntu coming up with a 1680 by 1050 display.

My system is an Athlon 2700+ CPU on an MSI motherboard and both the mother board and graphics card are over 2 years old so this is not a cutting edge set up. But the new display is working fine with it.  

I still do not understand why editing xorg.conf in pico resulted in an useable xorg.conf. Even when I undid the editing it was reported as having a typo in it. dpkg-reconfigure is your friend!
