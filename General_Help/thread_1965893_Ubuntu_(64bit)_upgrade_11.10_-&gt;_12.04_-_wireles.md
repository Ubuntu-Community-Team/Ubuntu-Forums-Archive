### Post by dminio on 2012-04-26
Created an account for this and posting real quick because I should be working...

I run the 32 bits version, but I got stuck in the same position. I opted to try krige's solution from the Ubuntu+1 forum (you can find it here: [http://ubuntuforums.org/showpost.php?p=11843035&postcount=17](http://ubuntuforums.org/showpost.php?p=11843035&postcount=17) ), but here's what I ended up doing :


[LIST=1]
[*]    boot from an external 12.04 installation USB drive;
[*]    open a terminal and run: 
```

sudo mkdir /mnt/linux 
sudo fdisk -l

```
[*]    identify your Linux partition with gparted (mine was sdb5) and run:
```

sudo mount /dev/sdb5 /mnt/linux 
sudo chroot /mnt/linux su 
mount /dev 
mount /proc 
mount /sys 
apt-get update

```
[*]    Then the process stopped with an error (didn't write it down - sorry!), but it suggested running this line:
```

sudo dpkg --configure -a

```
[*]    So I ran it without checking what it did (why not - it's a hobby PC). It took it's own time, but when it was done I just rebooted it and had a working PC.
[/LIST]

Hope this helps!

EDIT: didn't catch your edit after first read, step 4 is probably all you need - sorry!
