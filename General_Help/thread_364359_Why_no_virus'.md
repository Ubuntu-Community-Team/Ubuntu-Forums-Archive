This is not exactly true. Most of Internet servers run Linux, and also many companies, Banks and such instances are running Linux. These would be far more valuable targets for viruses and crackers than desktop machines, as they have lot more CPU power, memory and high-speed network connections (not to mention all the data they handle every day).

The big difference is in the way the system works, Linux and Unix systems have been designed to run networks and multiple users from the very start, whereas such things are 'glued on top' of Windows.

### Post by ardchoille42 on 2007-02-18
Getting a virus to run on Linux would require the user to download the virus, make it executable and then run it. And even then, without root permissions, it won't do any damage other than possibly trashing the user's home directory. And, we all make daily backups of our personal files, don't we?

With the current security methods in most Linux distros, it doesn't matter if one person used Linux or the whole planet used Linux, viruses just aren't worth writing for Linux, due to the inherent security in Linux, because viruses in Linux don't yield the maximum damage to a large number of system like the virus writers want.

I have seen normal users, without admin privileges, install and run software in Windows XP. This should not be allowed but it is. This is exactly what the virus writers want.. open systems that they can own and trash.

This is why there are a huge volume of viruses for Windows and very little for Linux. The popularity of the operating system has nothing to do with the number of viruses for that operating system.
