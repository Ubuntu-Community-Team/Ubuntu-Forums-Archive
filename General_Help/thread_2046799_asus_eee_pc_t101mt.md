### Post by Rexilion on 2012-08-26
> **sgtsmash336 said:**
> I'm getting an asus eee pc t101mt and want to put ubuntu 12.04 on it, but had sokme questions, I'm also new to linux systems.

Welcome to Ubuntuforums! Thank you for considering Linux :) .

> **sgtsmash336 said:**
> Should I do a complete wipe of the hard drive to get rid of all the bloatware and windows 7? 

If you want, it's not a requirement. You can keep Windows 7 and put Ubuntu on it as well. You do have to shrink the partition Windows 7 is residing on, however. This choice will not affect the performance of Ubuntu though.

> **sgtsmash336 said:**
> Will all the features work in tablet mode right off the bat or do I need to download something?

I usually steer clear of new hardware as Linux is always playing catch up with hardware vendors. If the device is all Intel hardware (except for the dreaded Intel GMA500!) then you are good to go. For example, intel wireless, intel sound, intel video, intel CPU all work correctly since Intel's employee's coöperate with Linux development. This is not the case for nVidia, but somewhat for Ati/AMD though.

You will only really know if you find someone else with this hardware and he/she reports his/her findings. Otherwise, there is only one way to test...

This is an argument to keep Windows 7 alive...

> **sgtsmash336 said:**
> If I do need to download anything is there somewhere that will list the steps on how to get ubuntu fully functioning in tablet mode?

This is related to the question above, you need to find out what hardware it runs. I find this hard to find out based on marketing names.

> **sgtsmash336 said:**
> Is 12.04 the best version to use

Generally, the latest version is indeed the best version to use, yes. This is recommended.

> **sgtsmash336 said:**
> sorry if anything has been asked before but I could not find any answers for 12.04.

You can always use Google as well, [_look_]("https://wiki.archlinux.org/index.php/ASUS_Eee_PC_T101MT")! and [_here_]("http://www.cnpbagwell.com/linux/asus-eee-pc-t101mt-fedora-15-install")! and even more [_here_]("http://forums.whirlpool.net.au/archive/1504082")!

Remeber, Ubuntu is based on the Linux kernel. So, generally, anything that is based on the Linux kernel will work in the same way. So you can look at manuals for Arch and Fedora as well. Except for the distro-specific part of course.

If something works in Fedora, but not in Arch. Then that means a program has been updated or incorrectly configured. _But it means that it does function in Linux!_

Good luck!
