### Post by oldfred on 2018-05-24
Is this your pastebin? The pastebin is not at ubuntu forums.
[http://paste.ubuntu.com/p/FjtwGXQzDj/](http://paste.ubuntu.com/p/FjtwGXQzDj/)

If your pastebin, you have Windows in UEFI boot mode on sdc using gpt partitioning.
But sda & sdb are MBR partitioned for BIOS boot.

Microsoft requires gpt for UEFI and MBR for BIOS. Ubuntu can boot from gpt with BIOS, but UEFI should use gpt.

UEFI and BIOS boot are not compatible. They write hardware data to drive for operating system differently.
And then once you start booting, you cannot switch. Or you can only dual boot from UEFI boot menu and may have to turn on/off UEFI/BIOS boot settings to match how you have installed. And grub only then can boot other systems installed in same boot mode.

Best to always have all systems in same boot mode. And if you have Windows follow its boot mode & partitioning for all drives & installs.
For all systems how you boot install media UEFI or BIOS, is then how it installs.

### Post by oldfred on 2018-05-24
Grub always defaults to first drive.
You have to use Something Else to have a choice on where to install grub2 boot loader. And with multiple drives you should use Something Else install option to be sure where or what partition on what drive it installs into. I prefer to partition with gparted first (and now only use gpt whether BIOS or UEFI), but you can partition during install. But a BIOS install will not give you the option to create gpt partitioning unless drive is gpt, or you are installing in UEFI boot mode.

And an UEFI install which still shows the combo box for which drive to install grub2 boot loader, does not work. It only installs to the first ESP it finds usually sda. If no ESP or a BIOS install to gpt without a bios_grub partition, grub install fails.

If you can disconnect a drive either physically or in UEFI/BIOS that often avoids some of the issues as then you can use an auto install. I prefer to add partitions and only use 25GB for / which an auto install does not do, it just uses entire drive.

For more info on UEFI see link in my signature below.
