### Post by Quackers on 2010-11-14
Your Windows installation is not being picked up by grub because the first 2 boot files for Windows 7 are missing. They would usually be in the first partition and the Windows OS would be in the second (in an OEM installation). This is not the case with your system. 
Was W7 a pre-existing OS when you installed Ubuntu? Or was Ubuntu installed first?

### Post by Quackers on 2010-11-14
Assuming, then, that you have a Windows installation disc, I would recommend that you first move the boot flag from sda1 to sda4. You can do this by using GParted from within Ubuntu (you may need to install it first) and right click on sda4 and select "manage flags" then check the "boot" box. This should move the boot flag to sda4.
Then I would boot from the Windows CD and run the startup repair in the recovery console. It may be necessary to run startup repair 3 times, with reboots in between. It may repair it on the first pass, but it may not. 
If this does not make Windows bootable please come back here, as there are other commands which will help.
