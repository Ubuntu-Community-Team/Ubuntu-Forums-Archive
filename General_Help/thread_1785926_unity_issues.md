### Post by vanadium on 2011-06-19
Different instances of one application are "maintained" under the same icon. You can tell if there is two or more instances by the double tickmark that appears to the left. If there is only one instance running, this will be an arrow.

Clicking on the icon brings all windows of the application to the front. Of these instances, the one that last had focus, will again get focus.

Clicking another time on the icon will "zoom out" on the windows, so you can click and select another one.

That is how it works. It is a novel, more application centric way of managing windows. It eliminates the cluttered task bar of the classical gnome desktop, where applications, and specific instances, were difficult to recognize and would change location all the time.

With respect to vuze, this is an issue with the notification area. Only "whitelisted" applications are allowed to use the tray. As long as vuze is running, it will also be displayed in the launcher, though, giving you quick access anyway. Not through a tray icon, but through the launcher. Some might call this a more consistent approach to application switching. The initial aim of the notification area indeed is not to minimise applications. You can get the vuze icon in your notification area, though, by adding it to the whitelist.
