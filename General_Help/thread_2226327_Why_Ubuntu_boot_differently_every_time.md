### Post by papibe on 2014-05-27
The Xorg conf file is set to use the nvidia driver:
```
    Driver         "nvidia"
```
but it fails to load:
```
[    23.960] (WW) Warning, couldn't open module nvidia
[    23.960] (II) UnloadModule: "nvidia"
[    23.960] (II) Unloading nvidia
[    23.960] (EE) Failed to load module "nvidia" (module does not exist, 0)
```
If you want to give the nvidia driver another chance, take a look at this post [here]("http://ubuntuforums.org/showthread.php?t=2223757&p=13021986#post13021986"). Since you have a custom xorg.conf I would just recover it instead of generate a new one.

The benefits of the nvidia driver are smooth 1080p playing and better performance in gaming. If that does not work, you may want to upgrade your driver to one upstream to see if that works better. Take a look at [this]("http://ubuntuforums.org/showthread.php?t=2223757&p=13028215#post13028215") upgrades procedures.

In case you want to keep the nouveau driver, comment out the nvidia line on your xorg.conf file, to make the graphic initialization process faster:
```
#    Driver         "nvidia"
```
Hope it helps. Let us know which way you decide to go, and how it goes.
Regards.

### Post by papibe on 2014-05-28
I see significant error in the log.

I would try to reset the resolutions using 'Nvidia X Server Settings' and then reboot to see if they stick.

Alternatively, you may want to update your Xorg config file using the same nvidia tool:

First, save your current file:
```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.old
```
Then, open the nvidia tool as root:
```
gksudo nvidia-settings
```
Then set the resolutions as you want, and then press 'Save to X Configuration File' to update your xorg,conf.

Reboot to test if all stick. Let us know how that works.
Regards.
