### Post by mjpatey on 2009-11-03
Hi, all-

I like to keep my system running at peak performance all the time while I'm using it, and therefore have always set the CPU Frequency Scaling Monitor to "Performance" once booted.  I've encountered one problem that's existed for as long as I can remember, and now there's a new one.

1.  The problem that's been around forever is that Ubuntu doesn't automatically remember my CPU frequency setting after a reboot.  I've learned to accept this, and just set the thing myself every time.

2.  The problem that started with Karmic is that there's no way to get the app to remember your authorization forever, i.e., I have to enter my password every time now.  In previous versions, I checked a checkbox the first time the app was run, telling it to remember my authorization forever.  This checkbox is missing from the CPU Frequency Scaling Monitor in Karmic.

Is there some way to get the CPU Frequency Scaling Monitor to remember my authorization so I don't have to type my password every time?  And for the bonus question, any way to get Ubuntu to remember the frequency setting after a reboot?  Basically, I want full CPU all the time, and to make it permanent, so I don't have to keep setting it every session.

Thanks in advance for any light you can shed!
