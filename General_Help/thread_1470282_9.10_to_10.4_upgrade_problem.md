### Post by zeus1 on 2010-05-03
Can you post the error message you are getting when you try to boot. I had a similar problem when upgrading to 9.10. My problem stemmed from relative link for root used in the menu.lst. To fix the problem I hardcoded the location of the root file system (/dev/sda7). If this is your problem go to /grub/conf/ and edit the file menu.lst. Make sure you make a backup of menu.lst before you do any changes. 

EDIT menu.lst WHERE

```
title        Ubuntu 9.10, kernel 2.6.31-19-generic
uuid        bdba4c77-f0e9-41fa-8f07-7b3e0d20b576
kernel        /boot/vmlinuz-2.6.31-19-generic** root=UUID=bdba4c77-f0e9-41fa-8f07-7b3e0d20b576** ro quiet splash quiet 
initrd        /boot/initrd.img-2.6.31-19-generic
quiet
```


you need to change where the root location from the relative address to the absolute. The absolute being the partition that contains your ubuntu root filesystem. 


EDIT menu.lst TO

```
title        Ubuntu 9.10, kernel 2.6.31-19-generic
uuid        bdba4c77-f0e9-41fa-8f07-7b3e0d20b576
kernel        /boot/vmlinuz-2.6.31-19-generic **root=/dev/sda7** ro quiet splash quiet 
initrd        /boot/initrd.img-2.6.31-19-generic
quiet
```


Where /dev/sdaX is the partition that contain ubuntu.
