### Post by Gustav on 2006-05-30
The package that is named linux-386 is just a meta-package that makes sure that all packages regarding the kernel is installed.

The packages that are named something with restricted-modules are non-free and you can easely search with synaptic for any packages that has the phrase in them and remove them.

### Post by az on 2006-05-30
[QUOTE=zenlunatic]How do I find out the true names of the package (don't know if you use vrms), the end is hidden.  Also I think one thing is the kernel itself how can i remove that?[/QUOTE]

Linux-386 depends on linux-image-386 and linux-restricted-modules-386.  Linux-image-386 always depends on the latest version of the linux-image-386 kernel - it is a meta-package, too so that you always get your uptades (the latset kernel image).

If you remove linux-restricted-modules-386, you remove linux-386, but not linux-image-286.  So you still keep the kernel and you still get updates, but you can live happily without the restricted-modules.


Simple.
