### Post by TheFu on 2020-06-01
#2 is odd.  #1 and #3 are fine.  American speaker.  

Two plural words back-to-back is almost always (maybe always) odd and screams non-native speaker.

From a "click-ability" standpoint, I wouldn't follow those TLDs, ever.  They seem highly suspicious, perhaps because they aren't used much?

[https://en.wikipedia.org/wiki/List_of_Internet_top-level_domains](https://en.wikipedia.org/wiki/List_of_Internet_top-level_domains) says that "**.systems**" is for technology sites.  Probably not language learning.
**.solutions** = business services
**.prof** 	Professors, teachers and learning

.prof seems like the expected TLD.

### Post by SeijiSensei on 2020-06-01
I'd start by searching in the .com and .net domains myself.  Like TheFu says, top-level domains like .systems raise suspicions.

There aren't many options in .com, but I did find that "tutoringsystems.net" is available.
```

$ host tutoringsystems.com
tutoringsystems.com has address 207.148.248.143
$ host tutoringsystems.net
Host tutoringsystems.net not found.

```
You can do searches like these with a WHOIS client, but I find the host command simpler to use.
