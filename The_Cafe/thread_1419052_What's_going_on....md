### Post by Lux Perpetua on 2010-03-01
> **Gumm said:**
> So I installed Ubuntu Karmic right over Windows 7 on my GF's laptop. During the install, I noticed that the machine time is set to 2 hours in the future... At first I was like "Whats going on??!" and then I realized its just the times that are changing...As Stephen King might say, *time is slipping*...

In seriousness, though, I had a similar problem recently whereby on every boot, my system clock (via date) would be 8 hours behind, although hwclock would show the correct time. I just did **sudo hwclock --hctosys** every time, but it got extremely annoying when my filesystem started reporting errors because some timestamps were in the future. It turns out that I had a typo in my rc.conf, so the system clock probably thought my hardware clock was set to UTC, when it's actually localtime (GMT-8, not coincidentally). That's for an Arch Linux system, but maybe there's a similar configuration issue for Ubuntu.
