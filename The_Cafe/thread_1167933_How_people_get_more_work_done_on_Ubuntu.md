### Post by albinootje on 2009-05-23
> **aceplayer97 said:**
> Every review I read the person always mentions how you can finish work much faster on Ubuntu.What kind of work are these people talking about?

I think those reviews are exaggerating.

But for me personally (as a Linux sysadmin), here's just an example, I'm much faster with editing text files with joe or vim 
as editor, than when I would have to mouse click my way around in some GUI text editor in MS-Windows.

And I can also quickly close down my ssh-session, or use screen to let processes run in the background, and then log out.

For MS-Windows i would need a GUI remote desktop solution, and I don't know whether it is even possible to manuall y run scripts or software upgrades in the background, and then log out of the session.

Upgrading software is also much easier and faster, I can start 5 workstations, then remotely ssh in, and start the upgrades, and do something else in the meantime.

In other words, in my opinion, the fear of the CLI that so many people have for unclear or invalid reasons, is a pity.
The CLI can save so much time.

An example, a colleague emailed me about a problem with Firefox in MS-Windows on his home laptop.
When he would have been using Linux, I could him written him :
1) open a terminal 
2) firefox -P
In MS-Windows there is this "safe mode" for Firefox as a menu entry, but try doing that in the MS-Windows CLI... 
you can't easily do that unless you put all the different program directories in the PATH.

Imho the default MS-Windows CLI (I'm not talking about Cygwin) is pretty frustrating, but the Linux CLI is wonderful.
