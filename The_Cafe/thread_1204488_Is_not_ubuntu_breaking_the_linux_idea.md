### Post by Mr. Picklesworth on 2009-07-04
Well no, it's not breaking the "Linux" idea at all. Linux, and all the open source or proprietary software you run on said kernel, can be optimized however you or others desire.

However, Ubuntu Netbook Remix (like Moblin) works quite well for the Intel processors because targeting a specific platform permits an optimal experience for those users while users of other hardware platforms can look for optimal experiences in other places. (And the Intel optimized stuff still works just fine for other hardware anyway).
It's just like how UNR targets the netbook sector, or how there is a 64-bit ISO available, except here we are targetting quirks specific to certain *brands* of processors. It's still the same Ubuntu with the same repositories and the same [source code.]("http://launchpad.net/ubuntu")

Further, Intel does an astounding job supporting Linux, so be nice. Carrying around grudges is not productive.

### Post by darolu on 2009-07-05
My two cents: the "linux" idea is wrong here, first Linux is just the kernel most our distros use; I read someone said NetBSD is "much more of a linux than Debian" that statement is wrong; NetBSD doesn't even use Linux.

Free Software is the concept that embraces and encourages the freedom to use, study, modify and distribute software; the Linux kernel is free software; the GNU operating system is free software, some distros are free software and most distros are "open source" software... there is a hot debate on what is free software and what is not and its difference with open source software, etc, etc... I don't intend to bring that on right now; my point is don't mix concepts that may lead to more confusion among FLOSS users. (Free Libre Open Source Software)

Part of this freedom, is the liberty to create modified versions of our software, what Ubuntu developers did with this netbook remix version you mention is make use of this freedom; I see it more like a "very linux idea" move, they are creating "new software" thanks to these freedoms and it will benefit a lot of users; free software doesn't stop because Ubuntu made a specialized version of GNU/Linux for Intel processors; there are plenty of distros out there for pretty much all the architectures and Ubuntu has GNU/Linux versions for most of them (at least the more popular ones) so I don't really see where this rant is coming from.

Another example of Ubuntu based distros that aim a very specific "market" is Goobuntu, which is a Google-Ubuntu distro exclusive for Google employees to use at work... I don't see rants about Ubuntu "breaking the 'linux' ideal" because of this one. I see this as a similar case, that shows the good side of free software/open source software.
