Point 1:  Windows only boots 'fast', in my opinion, before its service packed and patched to death.  Virus Shields, Anti-Spyware/popup blockers are almost required in Windows, so I'd say its silly to remove them.  All these things add time to your boot process.  As far as Windows Defender / Anti-spyware definitions in XP and Vista, I'm not interested in a vendors spyware package if it takes care of everyones spyware and malware .. except the stuff they write and put out. :-D

Being docked in the system tray just sucks.

Point 3:  Warnings about viruses and virus update procedures are a situation currently much more inherent to Windows than to Linux.  The absence of the maintenance is a breath of fresh air to most people.  I know it has been to me.

Point 4:  fragmentation happens, **period**  NTFS does not have the logic to self-heal itself in regards to fragmentation whereas linux filesystems are exceedingly better at keeping themselves lean and clean.  Installing applications causes fragmentation, deleting applications also does.  Temporary install files from MSI packages, web cookies, **windows' gross abuse of swap**.  A user doesn't have to be manually moving around a lot of data to have fragmentation.  Besides that, Who are you, or I, to say someone doesn't have to move files at all.  Its their system!  I don't know how he uses it anymore than you do, but if he needs to move around a lot of data, so be it. :)

Point 5:  Registry problems are directly related to the registry being a VERY BAD IDEA in general.  The registry was a solution in search of a problem, because there was absolutely no QC, and logo compliance came way too late. I've seen microsoft apps make an absolute mess of the registry.  Everything uses GUIDS, which ensures that nobody but a packager or a programmer is going to have the slightest idea how to track all their software components in the registry.  Its ridiculous that to this day, HKEY_CURRENT_USER of a particular remote host cannot be manipulated by a **domain**administrator across a windows network.

Point 6: I actually partially agree with you there. :)  I've made windows look and feel as great as I've made linux look and feel, but I personally think its easier to do in linux than in windows.  Especially now with Compiz-Fusion.  Windows *could* give users more native control over the GUI interface, but they choose not to.  So, if you truly want a completely customized interface, you have to use tools like Object Desktop, which is layered over the windows GUI and eats a lot of resources.
