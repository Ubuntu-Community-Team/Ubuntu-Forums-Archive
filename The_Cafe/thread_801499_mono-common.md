### Post by Methuselah on 2008-05-20
People are concerned about future issues with Microsoft.
I love how some people love to paint the concerned ones as paranoid. I don't think it's wrong to view mono with suspicion because of its relationship to microsoft technology. This is the same microsoft that loves to spew about linux patent infringements even when their claims are invalid. What will happen if they have actual grounds?

The technology is not even particularly unique. Java existed before. Java is now GPL. For those who love C# syntax, a compiler can be implemented that generates java bytecode. 

The problem as I understand it is that certain parts of the .NET stack have been put forward as an open standard by microsoft. IIRC, components such as ASP.net and Windows Forms have not been. However, this potentially infringing technology is a part of mono. It's a legal grey area that causes some discomfort.

Personally, I don't think mono is bad.
Compatibility is good.
Being able to run a .NET app on linux with a free implementation is good.
However, I wouldn't develop an open source program that relies on it.
