### Post by Marlonsm on 2009-07-19
It depends, Windows has most support from manufactures, so it's much easier to find drivers for it.
But Linux can do some really impressive stuff, like using my HP printer out of the box, while Windows needs an 80Mb+ driver that comes loaded with bloatware, or using a webcam OOTB, with no drivers installed, while in Windows, you'd need the instalation CD that also comes with bloatware.

Do you remember the last time you installed Windows? There are lots of CDs just for drivers: sound, video, network, printer, webcam... Tons of CDs, and many reboots later, I had a system where I could start installing an office suit, a pdf reader, another web browser...

While in Ubuntu/Linux Mint, after the 20 min install, I just needed to go to System > Admim > Hardware drivers and activate the proprietary nVidia driver, and for Ubuntu, adding the medibuntu repo and installing ubuntu-restricted-extras. Then I already had a completely usable computer.


Even though, there are still some unsupported hardware in Ubuntu, like some printers, some Wifi devices... that are quite hard to get working. But it's manufacturer's fault, that just because we are minority, don't care about making their hardware compatible with Linux.

### Post by bodhi.zazen on 2009-07-20
I would have to say Linux hands down. Look at all the arch Debian supports for example.

[http://www.debian.org/releases/stable/installmanual](http://www.debian.org/releases/stable/installmanual)

If you limit your question to a specific subset of hardware your results will be skewed.

new hardware
wireless
sparc
powerpc


But if you ask sheer numbers of hardware supported I think Linux has a longer list then windows, OSX, and Unix / BSD.

In my experience I have less trouble with hardware in Linux, much of it is supported out of the box, with windows, most hardware comes with installation CD because the windows. Such driver CD are showing how limited Windows support really is.

And as has been mentioned already on this thread, if you have ever had to install vanilla windows, and find the drivers yourself, rather then use the "rescue" media, which contains the drivers, you know how limited windows hardware support really is.

If you ask a similar question, what OS is most supported by hardware manufacturers, Windows hands down. But that is different from the sheer range of native hardware support built into the Linux kernel(s).

### Post by JDShu on 2009-07-20
Happily for me, the Lexmark works fine with a driver I found online.

My theory is that Linux wins in sheer number of drivers, especially for older hardware. Meanwhile Windows has much fewer drivers out of the box, but you can be sure that any hardware manufacture will provide you with drivers that work on the *current* version of Windows. Hence in most practical situations, its probably easier with Windows to get all your hardware working since people generally use relatively contemporary hardware. However as for the question, Linux has more driver support.

### Post by Deamos on 2009-07-20
Out of the box driver support:  I would say Linux wins this.  I find it 50x easier installing a printer on Linux than Windows (Especially HP printers) 

Device Diversity Support:  Unfortunately, Windows does win in this category.  Even though you may pick up some obscure device for cheap that works in Windows, you will find hell trying to get it to work in Linux.  

Long Term Driver Support:  I say Linux wins in this one.  Hardware developers drop devices once they reach "End of Life" which in some cases way too soon.  Linux continues to support this legacy hardware for much longer.
