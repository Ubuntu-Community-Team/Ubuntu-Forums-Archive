### Post by Bloodfen Razormaw on 2007-02-14
> i thought it was exactly the opposite, (well not exactly) but isnt the point of debian to spend LOADS of time testing the OS with locked versions of everything so its nice stable.... and then it ends up having older versions of most programs because they dont want the OS to become unstable because of packages updating before they check them out first
No.  That is the point of Debian STABLE.  Debian has 3 primary branches, plus an experimental one.  Stable means the packages are supported for the long term.  Testing is software that is as up-to-date as it can be without risking breaking a system.  Unstable means there is the possibility that an update might cause package conflicts or the like, but has very up-to-date software.

Ubuntu is based on Debian Unstable, but is rarely as up-to-date since it must constantly get the new development from that branch to stay up to date with Debian.  The only cases where Ubuntu is more up-to-date are in some core Ubuntu packages, e.g. GNOME is 2.14 in Debian unstable because GNOME development is of...less than stellar quality and produces buggy code upstream.  Debian fixes problems in desktop environments like KDE or GNOME, and other vital software like X, etc. before shipping it to users.  In contrast, to use GNOME as an example, Ubuntu times releases with the new GNOME releases and ships those versions without QA work done on the final release.
