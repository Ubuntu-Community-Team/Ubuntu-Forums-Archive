> **Sporkman said:**

The only difference between distros is the package manager, default settings, default apps, and artwork.

Even the base packages can be different.  For example, Debian (and hence Ubuntu) modify packages quite a bit in some cases while most other distributions (Fedora, Slackware and Arch for example) try to use vanilla packages as much as possible.

Another big difference is how up to date the packages are.  Ubuntu needs everything to be stable and work out of the box so they ship a lot of older software.  I prefer to have the latest even if I have to do a little work for it, so I am a happy Fedora user.

### Post by dspari1 on 2009-04-07
I couldn't find Sabayon, so I voted Gentoo since it's Gentoo based. 

Sabayon is great because it has a binary package manager called entropy that's completely compatible with portage. Because of this, I can get a Gentoo based system up to date in a matter of hours instead of days. After that, I can use portage to recompile all of my packages on a working system (and at my own pace) to gain all the advantages of having a Gentoo based system.

I tried Arch, and it really has a great thing going for itself, but the repository for pacman wasn't large enough to fit my needs. On the other hand, Gentoo's portage repository is very intensive, and I pretty much get anything I want. The downside of Sabayon is that it has a too many junk selected by default; however, the Redhat Anaconda Installer lets you remove all the unwanted packages. It's a pain, but it beats days of compiling just to get a working system.

One more plus to note is that like Mint, it comes with all my codecs, java, and flash already installed.

edit: I also wanted to say that Sabayon's default artwork is epic. It's simply beautiful.

### Post by yabbadabbadont on 2009-04-07
> **cardinals_fan said:**
> If only there were some magical way of measuring the speed with which apps are launched and run...

The binaries on Gentoo aren't really that much faster, even if they can be built for your specific CPU.  A lot of that is placebo effect.  However, where it does make a difference, is that it lets you strip out any unneeded and unwanted optional features through the proper USE flag configuration.  This leads to smaller binaries/libraries, less memory usage, and through that way a snappier system.  Some of it can also be attributed to building a custom kernel that only has what you need.
