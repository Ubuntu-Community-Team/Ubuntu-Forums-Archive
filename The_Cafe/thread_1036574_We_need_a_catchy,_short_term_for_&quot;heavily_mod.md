### Post by lykwydchykyn on 2009-01-10
Follow me on this:

In the Ubuntu world we've got Ubuntu, Kubuntu, Xubuntu, Fluxbuntu, edubuntu, ubuntu server, ubuntu studio, etc etc.  As most of us experienced users know, it's all just different combinations of packages from the same repository -- that even though I installed from a Xubuntu disc, I can just install the packages for Kubuntu and be running Kubuntu as much as someone who installed from the Kubuntu disc.

On top of this, you have all these derivative distros who also pull from these repos, maybe add a few custom tools and extra sources; but for the most part are just another permutation of packages from the same repos.

Now, if you have run Linux for a few years, you probably aren't running the default install.  If you're like me, you started maybe with a minimal install, or Xubuntu; then decided to add KDE, then got bored and switched to icewm with xcompmgr; then custom compiled a few things, and added some third party repos; removed a bunch of stuff to save space; changed out your login manager, customized a lot of graphics, etc.

Essentially, you've probably customized your system until -- if you decided to put it on a liveCD and have an installer -- it might easily qualify for "derivative distro" by Ubuntu family standards.

So when people ask you what you are running on your system, it feels silly to say "ubuntu", because then they think you've got GNOME and evolution and all the default stuff, when you haven't laid eyes on GNOME in six years.  Or, they go out and download Ubuntu thinking it will look and behave like your box, and it's nothing like it.

It would be convenient to have a term that basically says "I use the Ubuntu repos, but my system looks and behaves nothing like what you can get on a disc". 

What do you think?

PS -- before anyone flies off on a rant, I'm only half serious here.  Ok, not even half. It's my birthday and I have license to be pointless.:guitar:

### Post by SomeGuyDude on 2009-01-12
How about... Ubuntu?

The whole POINT of Linux is the vast amount of customization you can put into it. I have a feeling very few long-time Linux users are running a system that resembles the default installation. As long as it's all sitting on a distro's skeleton (package management, patched kernel, whatever) then it's that distro.

All those various names for Ubuntu derivatives (Kubuntu, Edubundtu, GoBuntu) are more like pet names for a given set of packages included by default. It's all Ubuntu.

After all, a whole lot of distros exist that have no default configuration, yet we don't try and come up with different names for them. It doesn't matter what your system looks like, if it's Gentoo it's Gentoo. If you're running KDEMod or Openbox, it's still Arch. You get what I'm saying?

Let's just call it all Ubuntu, and mention particulars when the situation warrants it.

### Post by SomeGuyDude on 2009-01-12
This is one big reason people have a hard time making the conversion. The vast number of different distributions is intimidating.

Think about how much Vista got made fun of because it had 5 different versions or whatever. Now try to imagine what these people would think of Ubuntu/Kubuntu/Xubuntu/MythBuntu/Studio/Edubuntu/Gobuntu/etc. It makes it sound like there's a billion different versions when really everything is the same just with a different set of packages installed.

**Piraja**, the simple answer is the basic system setup. No matter what version of Ubuntu you use, it uses the same core repos, it uses apt-get and debs, it's built on the same kernel and various patched system files. The skeleton of all Ubuntu systems are the same, as with Arch or Slack or Fedora. It doesn't matter what DE or WM you throw on there or what office suite and panel is running, it's still got the same setup as every other one.

Linux may be about diversity, but it should also be about unity. There's a difference between "diverse" and "segregated".
