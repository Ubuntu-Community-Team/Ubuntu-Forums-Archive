### Post by JazzPotato on 2015-05-22
There's no short ,or even "right", answer to this question, but here's a few pointers:
[LIST]
[*]Don't deliberate over platform choice, programming language choice, framework choice, etc... I have wasted alot of time doing this before I came to realize that programming languages are just tools that can be used to solve problems. That being said, Java is a good general-purpose language, with a large community, and it has the property of being "statically typed" which will teach you about data types properly, and it also has a strong bias towards object-oriented code, which is a modern and robust coding style.
[*]Start writing code from day one, and buy some technical books on your programming language. You should also read about the MVC pattern and unit testing. I can highly recommend books form the Pragmatic Programmers, O'Reilly Media, or Apress.
[*]Once you have become familiar with a programming language, run through some of problems on [Project Euler]("https://projecteuler.net/"), this will start teaching you to solve problems using code, which is a strange adjustment at first but soon becomes second nature (or first nature, depending on how much time you spend in front of a computer)
[*]Next you should open a GitHub account, and start contributing back to some of the projects you use or start your own. The more of other people's code you read, the better a programmer you become.
[/LIST]

Your success with programming will vary depending on how much time you invest in the subject, but the earlier you can start becoming engrossed in the programming community on forums, message boards and Q/A sites the better.

Have fun!

Here's the big secret to newbies - the language isn't actually that important.

say you want to make apps - you'll probably use Java, kernel coding C, games maybe C++.

but what actually matters at the beginning is to understand what is actually happening. It is infinitely more important to know what variables, loops, functions, classes and the like are than it is to know them in a specific language.

obviously if you know what you'll be studying in the fall choose that language.

### Post by veddox on 2015-05-22
Actually I wouldn't recommend learning a language completely online - I find proper books to be invaluable when learning a new language from scratch. They're just a lot more comprehensive than any online tutorial, and easier to flick through than a video course when you need to check up again on something.

I recommend the "Head First" series, they're really good, solid books that are easy (and enjoyable!) to read. Check your local library.

Although having said that, jim50 is right, there are good courses online too (there's a really popular one on Python over at Udacity - a MOOC with its roots in Stanford). Use both books and online resources for maximum benefit.

Never underestimate the importance of actually writing code yourself, like JazzPotato said. **There is no substitute for practice.**

And lastly, Google is your friend ;-) Your not the first person to ask this question on this forum - see [here]("http://ubuntuforums.org/showthread.php?t=1766253") for a sticky on the subject...

### Post by Old_Grey_Wolf on 2015-05-22
Learning a programming language is more than follow the examples and tutorials in a book or online tutorial. As a programmer you will be given some requirements for what the program must do. You may also be limited in what languages you can use to write the program. It is up to you as the programmer to solve the problem in an efficient way.

What worked best for me (may not work for you) was to pick a goal for a program to accomplish. It doesn't have to be original. Then start writing the program as you read the book. At some point you will get something that works; however, as you continue reading the book you will learns some programming techniques that make the program more efficient. As you learn you keep modifying the program.

I looked at the books I have on programming languages and the books published by O'Reilly Media are prevalent.

### Post by veddox on 2015-05-28
Make sure you don't accumulate languages too quickly - it will just confuse you, and slow down your overall learning progress. Learn one language well, become comfortable and gather experience with it, and then move on to another language. (I spent about a year on my first language - Java - before starting with Python.)

When you do start learning other languages, try to learn some very different languages that have different approaches to programming. Programming is a way of thinking, and learning different approaches is a good way to broaden your mind and deepen your skills. So instead of learning only C-based languages, also go for some others: Python, Lisp, Javascript...
