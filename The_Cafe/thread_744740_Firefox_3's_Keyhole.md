### Post by ChrisNiemy on 2008-04-26
In my opinion the inherent/integrated look with native GTK-Icons is a great thing and I'm happy with that. But, for new users to ubuntu, or Firefox running on Linux in general, it will look a little bit old, when there's no keyhole icon. Because the keyhole icon was somehow advertised as a new feature (hey it's a feature... ;-)) in Firefox 3. So in default there should be a keyhole icon on linux/gtk. Of course, for experienced users  the native gtk icons are an advantage, so also for unexperienced users, but for the last it will look somehow that the linux design for Firefox 3 was not as well done as on the other platform, because the icon is missing.

I found something (on that topic) very interesting here:
[What would a linux keyhole (icon) look like?](http://ventnorsblog.blogspot.com/2008/03/what-would-linux-keyhole-look-like.html)
In the comments someone mentions how about a patch for GTK+ like GtkKeyholeIcon. It think this is a great idea to make everybody happy. So you could just switch between native classic forward/back-gtk-icons and GtkKeyhole. Though I know nothing about how much or less complicated this would be.
Ah, here's the quote: > Personally, I think you should submit a patch to GTK+ adding support for GtkKeyHole and then have Firefox require GTK+ 2.16 or higher

[In the comments of this blog entry](http://ventnorsblog.blogspot.com/2008/01/hi-mum.html) there's a discussion about the linux icons, too.
