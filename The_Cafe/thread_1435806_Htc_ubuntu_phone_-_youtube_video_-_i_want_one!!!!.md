### Post by gnomeuser on 2010-03-21
curious but pointless.

It turns a phone into a tablet with a desktop ui that doesn't scale well. I also suspect that the data connection might not be hooked up yet so you are also tied to wifi now. 

That phones run Linux isn't new, seeing one run a basically off the shelf is new in the sense that it shows how far Linux has come in a very visual fashion. We always really ran on most anything, but that our full usable systems at least now just run is still good.

Still till it stops being something it is not, namely a tablet/phone OS it isn't going to be truly useful to any but the few. That would also raise questions as to what truly is Ubuntu, at this point you'd be running the same technology, but your UI much of that Ubuntu is shaped around. The experience is different, is it Ubuntu or just Powered by Ubuntu. At which point there are far a better chance of this becoming Ubuntu MeeGo instead, a product I fully expect we will see soon. We already have the Moblin releases and these will naturally become MeeGo but I think it would also be useful for Canonical to be able to provide a working full reference implementation of MeeGo for tablet and phones. Much of the work would come from upstream and it widens the range and reach of Ubuntu. 

How cool e.g. would my Nokia N900 be seeing as it runs Maemo a pretty much off the shelf Linux with nearly the same ui technology stack, if that supported Ubuntu One for those to sync my tomboy notes (Conway is supposedly a Tomboy clone in C for the device, I have not yet checked how accurate the replication of the experience is yet), my contacts, calendar appointments and give me a harbor to publish them to the world. I would pay good money for that.
