### Post by cmnorton on 2009-01-24
I would like to take the "Don't Preach..." thread -- [http://ubuntuforums.org/showthread.php?p=6468757#post6468757](http://ubuntuforums.org/showthread.php?p=6468757#post6468757) -- one step further, and ask what a single person can do to encourage more commercial packages to run on the Ubuntu distribution.

We are a Linux shop by a thread. That is if it were not for a major home-grown tax collection package runs on Linux and that our school system's administrative package ran on SCO (and a "lite" version has been converted to run on Linux, we would be a Windows shop. 

The Ubuntu community and Canonical support is excellent, but I am seeing a lot of commercial systems, applications like general ledger and voice mail gravitating towards RPM based distributions (Red Hat/CentOS). To people who do not understand Linux, it is almost as if these were different OSs rather than different distributions. 

I have had two bosses afraid of Linux because I am the only person who understands it. My contention is that we have a smart group of people, who could learn some elementary command line skils. Even our general ledger server (RH EL 4) is running X, so CUPS can be manipulated though firefox. What is so difficult about that, other than it is not Windows?

My Linux workstation is Kubuntu 8.04 with LAMP and other server components, so I can develop and test our Informix SE and web applications. My home laptop and work connectivity system is Ubuntu 8.10.

So, the bottom line is, is it good enough we have the number of Linux workstations, even though none of the production boxes are Ubuntu (server), or is there more someone can do like identifying companies for Canonical to contact.
