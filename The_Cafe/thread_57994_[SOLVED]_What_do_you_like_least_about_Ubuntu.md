### Post by dtfinch on 2005-09-12
I'd like to run modern apps on a stable base, without leaving the standard repositories. Few people want to run only apps that are as old as their operating system. A new app doesn't necessarily need to force an upgrade, but it should at least be available if the user wants to search for and install it. Otherwise, I have to uninstall the repository packages and install them manually, which sometimes forces the removal of metapackages which are often important for upgrading to the next Ubuntu release.

The hdparm defaults are a little sad. Hard drives and DVD drives need at least DMA enabled by default. PIO is so old it's no longer the stable choice. Your system must be over 10 years old to not support DMA, which the installer ought to be able to detect. A modern system should get modern defaults.

The default setup is a little difficult to work with as well. Everyone goes to ubuntuguide.org to complete their setup, or they just get frustrated. A lot of the fixes should be in Ubuntu by default, at least those that won't cause licensing issues.

Ubuntu (probably more gnome's fault) seems to be tailored for resolutions near 1600x1200. Everything looks gigantic by default if someone happens to use 1024x768. You can change font sizes, but I don't see an easy way to change the default icon sizes and the thick padding in most gtk2 themes.

The default font selection is a little on the frugal side. The default serif and sans-serif fonts in Firefox are eyesores next to arial and times new roman. The default serif font in Firefox is especially oversized, which messes up a lot of pages, like Slashdot.

The "Debian" menu should be installed by default, but with a different name. Just call it "More" or something. Right now, users very often need to use the command line to find and run GUI applications they just installed from the Ubuntu repositories. This is unacceptable for new users.

If Firefox 1.5 won't make it into Breezy, maybe another Gecko based browser should be the default. Maybe even the bulky Mozilla Suite, the latest stable of which doesn't seem to have the severe performance bugs that Firefox 1.0.x does on Linux.

The brown theme. I sink into depression whenever I see it, and the bundled alternatives aren't very clean. Red Hat's BlueCurve is a very good theme, and I have a nice blue version of the default Ubuntu wallpaper as well. Blue is easy to stare at for long periods of time. On my last install, which I blew away to install Breezy, I got the official BlueCurve installed, minus the Red Hat logo icons which I swapped out for the defaults.

### Post by Brunellus on 2005-11-10
also, nothing prevents you from downloading the individual .debs and updating one package and its dependencies at any time, or from deleting the packaged and installed version using apt, and then building from source.

The former is only marginally more complicated than the usual windows routine of uninstalling, finding the right .exe on the 'net, and reinstalling.  The latter is a very *nix way of doing things (indeed, some distributions encourage or compel you to do it this way), but is admittedly not newbie friendly in the slightest.

as to the fragmentation problem:  [Wikipedia](http://en.wikipedia.org/wiki/Ext3) notes that it is theoretically possible to defragment an ext3 filesystem by converting it back to ext2, and then running e2defrag, then converting it once again to ext3 (all operations conducted on an unmounted filesystem, of course).  

You really have to ask yourself if this is really worth all the bother.  The other question you have to ask yourself is whether or not people who are actually using their filesystems much more intensively than you have ever run into the problem.  I don't have the time at the moment to investigate this, but I suspect that the limiting factor has more to do with the mechanical life of the hard disks in question than the fragmentation of the filesystems.

As regards your system restore point, I agree.  the  default installer should put /home on a totally separate partition.  Fewer eggs, more baskets.

That's not an Ubuntu problem, it's a legal/licensing problem---SUSE and other Linux distros have similar issues.

I think you will find that if you live in the US, the Linux solutions for MP3, DVD playback, etc., are actually illegal because of patent/DRM laws.

Linux distros get round this by not including the codecs etc. in the basic install, but making it possible for users to download the packages---while warning them about legal issues.  My guess is that this places the onus for any lawbreaking on the user's head.

See
[https://wiki.ubuntu.com/RestrictedFormats](https://wiki.ubuntu.com/RestrictedFormats)
[http://en.opensuse.org/Restricted_Formats](http://en.opensuse.org/Restricted_Formats)

The latter link notes that "It is not legal to distribute unlicensed MP3 Decoders and Encoders in most countries."

It's one of the places where Ubuntu needs some serious work done. Monitor resolutions can be adjusted by typing the following in the terminal:

```
sudo dpkg-reconfigure xserver-xorg
```

Then you select your video driver and then basically just the most sane choices until a menu comes where all the resolutions are listed. All you have to do is to click space in front of the resolutions that you want and then enter-enter-enter your way out of it.
