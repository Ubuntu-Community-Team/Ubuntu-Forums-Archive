### Post by DBO on 2009-01-12
Wow, thank you guys.  The kind comments are a real breath of fresh air after developing Docky in the dark for so long.  I know things aren't going perfect for all of you, so if you could reply here with the state of your issues, I would be more than happy to help you get Docky going.

A couple things I want to mention:

First, there is a PPA going up with docky AND plugins momentarily.  When the PPA goes live, you will be able to install Docky from your software management tools just like you do everything else.  You just have to add the ppa to your sources.list.  This is the suggested way to test Docky as it will keep all the beta testers at the same version.  Watch here: [https://launchpad.net/~do-testers/+archive](https://launchpad.net/~do-testers/+archive)

Second, Docky is still improving!  Even the version being put in the PPA has "known bugs" that are fixed in my bzr version.  I am trying to steamroll out the last of the major bugs before release.  A couple issues you may want to be aware of that are already fixed in my local version.
[LIST]
[*]Docky does not group window items
[*]Docky does not get good icons for grouped applications
[*]The icons in the configuration right click menu seem like odd choices
[*]A couple minor performance issues
[/LIST]

To address some additional comments.

**Does docky have plugins?**
Yes, no, and it will, all at once.  It obviously can do anything GNOME Do can do, which can be extended by plugins.  However internally it will also support plugins directly like AWN does, and to prove this was possible to myself I made the trash icon.  Trash is more or less a plugin built into Docky.  So plugin authors, V 2.0, it will be there.

**How come the Do page says 6.1 when docky needs 0.7.95 or newer?**
Well, we haven't released it yet =).  It's not really ready, but it's getting close.  To get Docky you either need to install the PPA or compile from source.  Hardy folks can compile from source if you have the directhex ppa (google).

**Can Docky run without compositing?**
No, docky requires composite due to the very nature of how it is drawn.  Docky does however work with VERY simple compositors.  The metacity compositor, xfwm, xcompmgr have all been personally tested by me and confirmed to work with Docky.  You do not by any means have to run compiz (though I do).

**Its slow!**
Chances are you are running a nvidia card with a somewhat old driver.  The latest nvidia drivers fix this issue.  However if you have a nvidia driver of the 1.7x series, you can fix the performance issues by running "nvidia-settings -a InitialPixmapPlacement=2" and restarting GNOME Do.

**On GConf Options**
Docky uses gconf to store variables, it does not however monitor gconf actively and will not respond to changes you make in gconf while it is running.  Be warned that modifying gconf variables can result in badness and even crashing right now.  I will fix this as soon as possible, my bad!  You should try not to play with too many of the settings in there...

**Found a bug?**
Chances are Docky has quite a few bugs.  If you would be so kind, please report them to our [bug tracker]("https://bugs.launchpad.net/do/").

**How Do I...**
*Resize Docky?* Just grab the separator on the dock and drag up or down.

*Add/Remove Automatic Icons?* Drag the left or right edges of the dock.
[B][I]
Help Docky get better?[/I][/B] Give me feedback, and tell me what you think is wrong or could be better!

### Post by smbm on 2009-01-12
To compile the plugins.

```
sudo apt-get build-dep gnome-do-plugins
```
```

sudo apt-get install automake bzr mono-gmcs libmono-cairo2.0-cil ca-certificates libtool intltool libndesk-dbus-glib1.0-cil libndesk-dbus1.0-cil libgnome-vfs2.0-cil   gtk-sharp2-examples gnome-desktop-sharp2 libgconf2-dev build-essential

```
```
bzr branch lp:do-plugins
```
```

cd do-plugins
./autogen.sh
make
sudo make install
```

did it for me.
