### Post by cleverselfreferentialname on 2008-10-07
> **PetePete said:**
> 
The descriptions are also written by an expert user, for expert user. ... i mean, common, seriously what new person to linux will know what this means ."grep pattern files" what patterns valid? what files? how do i specify? its useless. And all the different types of rm commands. .. too confusing. 


It's really very simple. rm -r is recursive, it removes a directory. rm -rf forces the removal of things, it won't acknowledge read-only bits that you can set or unset yourself. rm -r will work in every location rm does, even if you aren't using it on a directory. I don't see how it's confusing, unless you're really, really uninitiated.

grep woot file1 will search in file1 and print lines containing 'woot' without case insensitvity. grep -i woot file1 will print lines containing woot with case insensitivity. grep -i woot file1 ~/stuff/* will print all lines containing woot in any case in the directory 'stuff' under your homedir.

It can be better used as a reference for people with intermediate skills. I wouldn't use it personally as I probably have all of them committed to memory (except for a few things - dig I never use).
