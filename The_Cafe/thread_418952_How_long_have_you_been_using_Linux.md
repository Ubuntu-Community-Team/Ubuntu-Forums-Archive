### Post by Ram Crammer on 2007-04-22
I remember when I first discovered Linux, about 14  years ago (1993).  
Linux had already been around for at least a year.  I  was amazed.  
Here was a powerful, multi-tasking, time-sharing,  multi-user, 32bit 
OS, open and free to any that wanted it.  Prior to  this time, one would 
have to spend hundreds of dollars to obtain a  commercial unix with 
all the functionality of upstart Linux.  

I recall the excitement of receiving my first Linux distro--Slackware with
kernel v. 0.99, which came on 12 x 1.2MB 5.25" floppies. Downloading
Linux from home would have taken days with the 12 or 24 KB/s modems
of that period.

I remember being amazed at how well it ran on my modest PC.  It was
blazingly fast, and I honestly don't recall that it ever hung, even
when my own buggy programs bombed!!  And this was when Windows 
3.0 would crash and burn 3 or 4 times in a single session, for no 
apparent reason. Linux didn't have much of a GUI in those days, but it
did include a functional X server, it could be networked, and could
access the Internet via modem.  It came with all the standard unix
commands, compilers, and the granddaddy of all programmer's editors,
GNU-emacs.  What ever Linux lacked in those early days, was
frantically being developed within the burgeoning Linux community, and
besides, the system was (and still is) so transparent, that one could
often write code to do things that weren't there, yet, such as write
an application and a device driver for a school stage lighting system.
In it's openness, and in other important ways, Linux remains unchanged
from its early days.  

Soon, I was up and running with a real unix-like OS on an i386-SX with
a whole 1MB RAM.  I was delighted, because in those days the Computer
Science department at my college was a UNIX based department, and 
with Linux I could learn how to use the college UNIX system from home. 
I would dial into their mini-computer which was running UNIX System
V, and by using TELNET, I issued commands on their system from my
Linux box.  I could write my C code, compile it, and debug it, all at
home, then upload my debugged code and re-compile it on the college
system, and run it on their mini-computer.

Using Linux and dialing in from home gave me a nice advantage
over those students without a Linux system.  My peers had to
spend long hours in the computer lab writing their programs, in order to
complete their class assignments.  I spent long hours too, but at
least I could do it at home. :-)  Of course, my advantage didn't last
long, as others discovered Linux.

Those were heady days and I learned a lot.  But, unix spoiled me for
Windows.  I've never been able to really appreciate Windows much,
despite its slick interface and impressive applications and games.
Windows always felt limiting to me, and hard to work with, from a unix
programmer's viewpoint. 

Unix has always been considered an excellent programmer's OS,
offering a simple and consistent system interface to the applications
programmer; an interface which has changed very little over the last
20 plus years!  Unix, which of course includes Linux, offers a
plethora of programming tools, editors, compilers, IDE's, command
shells, text formatters, just about anything the programmer could hope
for, most of which is in a very high state of perfection and thoroughly 
tested.  It's all there, free and open source, and on most Linux 
distributions, it's right up-to-date.

Over the years, I've maintained an interested in Linux. I have nearly
every issue of Linux Journal in hardcopy.  For awhile Linux was in the
background, as Windows overtook it for some time, as the predomanant
and (let's admit it) the superior OS (on the desktop, not on servers and 
mainframes).  But as we are seeing with Ubuntu, Linux is now a fully 
viable alternative to Windows on the desktop too, at least for people like 
me, and even for many non-technical folks.

Games don't hold much interest for me.  I prefer playing with the
Linux OS, writing shell scripts, Perl scripts, and an occasional C
program, as a way to learn the system internals, and to help me do
stuff.  While my carreer in software engineering is probably behind me
now, I hope to keep my hand in by programming in Linux, which seems
like a grandiose Erector Set for people like me.
