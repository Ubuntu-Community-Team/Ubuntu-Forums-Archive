### Post by koenn on 2008-04-21
I'd say Dbian because that's where I learned most of my Linux stuff. 

It all depends on what you want to learn (and why) but consider doing a base install (command line only, no other software except the basic tools), then 
- add stuff you're interested in
- try to change the default config of the apps you install
like, say, decide you want to have mysql keep its databases in /srv/databases and figure out what it involves regarding modyfying config files, setting permissions, ..., 
or see if you can manage a situation where things don't "Just Work", eg  set up LVM spanning multiple partitions on multiple disks, and find out what it takes to move the disks to a new machine. or whether you can recover from a failed disk or a corrupted partition or whatever,  and how.

- do al that with nothing but vim and a shell : learn some shell stuff, some vim stuff, and learn where all the files are that you need to modify, ... 

Doing this in debian (or a similar distro) is a good idea because it's well organized and doesn't use a central config file for everything (like RH and Suse did years back when I first looked at them, I don't know if they still do)

### Post by DogFoodTaster on 2008-04-21
I began my first real exploration of Linux with Gentoo and used it as a secondary OS that got about 1/3 of my use time.  About a year after I switched to Arch because I became fed up with the need to spend so much time on updates.  Overall I would say that Arch is slightly more user friendly, but its by no means a cake walk for a novice.  I would say if you can spare the time needed for Gentoo to compile and install software (were talking hours to start with plus a lot of time spent on updates later) it's probably the better bet.

The main advantage of Gentoo is really the Gentoo community and wiki.  The wealth and quality of Gentoo knowledge available online compares to Ubuntu and this is where you'll see Arch falter a little.  You will still be able to find a lot of information for Arch, but Gentoo's resources are just so much more.

A few things you'll probably be learning with either Gentoo or Arch are:
* Mounting disks and writing a correct fstab to do it automatically on boot.

* Configuring dbus, hal, and (gnome-volume-manager / ivman) to automatically mount newly connected devices like CDs and thumb drives.

* Configuring and compiling a kernel (You can do this during the install process, but if you only have a single computer you might want to wait until after you have a working install to worry about it.  It's fairly hard to get it right your first few trys.)

* setting up an xorg.conf file (although this knowledge has become almost obsolete with recent versions of xorg, which use an automatic configuration if an xorg.conf file is missing.)

* It's possible you could pick up some basic knowledge of how a compiler works because you'll be using one so much.  Although portage is mostly automated so you really won't get to far into it unless you explore beyond what is required of you.

* Setting up users, groups, and permissions.  (Along this line you can also install "sudo" and learn about how it aids the process of assigning users special permissions.)

* Configuration of ALSA for your soundcard.

* Configuration of a window manager or desktop environment.

There a ton of other things you can learn to so if you have any specific interest in a topic it might be best to search available online resources for it to see if a distro has what you think you would need if you run into trouble with it.


EDIT:
I forgot to say you'll need to setup networking.
This involves installing WPA supplicant if you need WPA or WPA 2 encryption.  The ways to go about networking are very different on Gentoo and Arch and frankly I think the normal Arch method, while alright for something stationary, is terrible for a mobile device.  If you go the Arch way I highly suggest setting up network manager (what Ubuntu uses) instead.
