> **MG&TL said:**
> I expect this has been posted before, but I couldn't find it.

From wikipedia:
> The name “GNOME” is an acronym of GNU Network Object Model Environment. It refers to GNOME’s original intention of creating a distributed object framework similar to Microsoft’s OLE.[6] This no longer reflects the core vision of the GNOME project, and the full expansion of the name is now considered obsolete

As for Unity, why not? Someone at Canonical decided to call it that I guess.

> 
BASH is 'Bourne Again Shell Script'-but why is it a 'terminal' in the first place? It seems a bit odd, considering I START from the terminal...

The name terminal is a legacy from the old mainframe days. A terminal was a fairly dumb keyboard and display that you'd use to talk to the mainframe. I guess they were called terminals because they were at the boundary of the system. We no longer have the hardware, but the software they displayed has carried on the name.

### Post by 3Miro on 2011-08-08
Unity is not an acronym, it is reference to something being United, basically it is marketing.

Red Hat may be a reference to someone's hat, but I think it is also a reference to "black hat" vs "red hat" bad vs good hackers.

KDE, GNOME and XFCE started as acronyms, both they have all lost their meaning. Gnome is much more than "Network Object Model Environment". XFCE is a reference to the forum where the devs first meet and the forum no longer exists. I think KDE is now K Desktop Environment (i.e. without the Kool) although they do use the K all over the place (Konquerer, Kaffeine ...)
