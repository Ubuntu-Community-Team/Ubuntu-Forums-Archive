### Post by AtariBaby on 2013-07-14
I realize this is an old thread but found this post by "Evilkitty" regarding Asus a54 backlight behavior:

[COLOR=#4D4D4D][FONT=helvetica]to fix backlight behavior o the new BIOS after the 3.6.11 kernel you need to add this kernel boot parameter:[/FONT][/COLOR]
[COLOR=#4D4D4D][FONT=helvetica]acpi_osi="!Windows 2012"[/FONT][/COLOR]

[COLOR=#4D4D4D][FONT=helvetica]a modern linux system will use grub2, so you will want to edit this file:[/FONT][/COLOR]
[COLOR=#4D4D4D][FONT=helvetica]/etc/default/grub[/FONT][/COLOR]
[COLOR=#4D4D4D][FONT=helvetica]you will see a line that starts with this[/FONT][/COLOR]
[COLOR=#4D4D4D][FONT=helvetica]GRUB_CMDLINE_LINUX_DEFAULT=[/FONT][/COLOR]
[COLOR=#4D4D4D][FONT=helvetica]if you have something special in here (you know who you are) you will need to edit the next line[/FONT][/COLOR]
[COLOR=#4D4D4D][FONT=helvetica]i changed that entire line to this:[/FONT][/COLOR]
[COLOR=#4D4D4D][FONT=helvetica]GRUB_CMDLINE_LINUX_DEFAULT='quite splash acpi_osi="!Windows 2012"'[/FONT][/COLOR]

[COLOR=#4D4D4D][FONT=helvetica]before:[/FONT][/COLOR]
[COLOR=#4D4D4D][FONT=helvetica]GRUB_CMDLINE_LINUX_DEFAULT="quite splash"[/FONT][/COLOR]
[COLOR=#4D4D4D][FONT=helvetica]after:[/FONT][/COLOR]
[COLOR=#4D4D4D][FONT=helvetica]GRUB_CMDLINE_LINUX_DEFAULT='quite splash acpi_osi="!Windows 2012"'[/FONT][/COLOR]

[COLOR=#4D4D4D][FONT=helvetica]then run sudo update-grub and then boot the new kernel and enjoy the back-light acting like it used to[/FONT][/COLOR]
[COLOR=#4D4D4D][FONT=helvetica]I have only tested the 3.9.0-030900rc6-generic kernel but it should work on the other affected kernels[/FONT][/COLOR]

[COLOR=#4D4D4D][FONT=helvetica]here is the bug report where i got this fix[/FONT][/COLOR]
[COLOR=#4D4D4D][FONT=helvetica]bugzilla.kernel.org/show_bug.cgi?id=55161[/FONT][/COLOR]
