### Post by 3Miro on 2011-04-23
There are two types of attacks, psychological and technological. Psychological exploit ignorance of the user to fool him/her to damage their machine. The only defense against this kind of attack is education. If Linux were more popular, then you will see more such attacks.

The second type of attack is technological, where a vulnerability of the system is used to damage a system. This is done regardless of how experienced the user may be. Linux will remain just as resistant to those types of attacks as it is now.

You also ask about being "free". Do you mean "free as in freedom" or "free as in cheap"? There are currently payed Linux distributions (Red Hat, Mandriva ...), if Linux were more popular there would be more payed distributions and there is nothing wrong with that.

What is more important, is that Linux is "free as in freedom". I don't think this can change as it would be a legal nightmare to sort out. Linux is not made by a small group of people, but by hundreds if not thousands of people (we are talking about not just the core, but the entire GNU/Linux system). Getting a significant number of those people to agree to change the license of the code that they wrote, is practically impossible.
