### Post by mcduck on 2008-06-01
I do most of the design part in Inkscape, and if needed then use Gimp to add some details and other stuff not easily done with vector graphics.

For all coding part my tool of choice is Gedit. It's amazing how much Gnome's default text editor can do, when you may feel that it's just a Linux-version of the rather useless Notepad. It's not. Just get the gedit-plugins package and check the options & enable the plugins you find useful..

And of course the Web Developer toolbar for Firefox is absolute must to have.. (did you know that it's even in Ubuntu's repositories!)

I also depend a lot on using virtual desktops while working. Usual setup is to have Firefox on first desktop (I need to see what I'm doing..), Gedit and file manager with the project directory open on second desktop, Inkscape on third. This way I don't even need to touch the mouse when coding, flipping between Firefox and Gedit with keyboard shortcuts..

Oh yeah, I almost forgot.. Learn to use Imagemagick. It's the best way for compressing large amounts of images and generating thumbnails and other such tasks often needed.. When I was still studying my teacher was boosting about how great the batch tool in Photoshop is for that kind of tasks.. I asked if he wanted to try a bit of a competition, resizing 200 images to suitable size for web, compressing them and generating thumbnails. He didn't even get the Photoshop started before I had already typed the necessary command to do the job and was already heading to get a cup of coffee while Imagemagick was doing the work for me. :D
