### Post by pissedoffdude on 2015-05-25
In all honesty, I don't think it really matters *that* much.  What's most important is knowing how to do things in linux, eg creating new users, setting permissions, knowing how to install/upgrade software (which isn't hard to learn on a new system), networking, etc.  From my experience, I've seen various clients using rhel, centos, ubuntu, debian, etc, but it all seems to narrow down to if they know how to edit their hosts file, configure file descriptors (ulimit), provision new users, cron jobs, etc.  

I'd say that after getting a good grip on basic linux administration, fine tune the area you're really interested in.  So, if you're interested in web hosting, learn more about apache, php, etc.  If you're interested in devops, learn sql, nosql, containerization, etc.
