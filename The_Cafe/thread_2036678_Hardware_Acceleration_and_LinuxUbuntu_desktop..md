### Post by Kreaninw on 2012-08-02
The most important thing that Linux/Ubuntu lacks in term of OS environment is hardware acceleration. There's no way to compared the speed of overall users experience Windows VS Linux now in these day and age. GPU driver lacks support on Linux is one long story that's not yet to be fixed. 

I'm using Ubuntu 12.04 LTS 64-bit and I would say Firefox, Chrome, Opera are really slow compared to their Windows edition. Why? Because it's not hardware accelerated. For Firefox, there's no way I can tuned it on successfully. Chrome, by overrode system configuration in chrome flags, it give me a much better result, a lot faster web experience. Opera too, it's easy to turn hardware acceleration on. Firefox, Chrome or even IE9 are hardware accelerated by default, but not on Linux. And noticed that not all GPU relate features in these browsers support on Linux platform.

More advance tools like GIMP(Photoshop), Inkscape(Illustrator) are not hardware accelerated in their canvas. It's true GIMP and Inkscape are noticeable faster on Linux, but can't be compare to Adobe speed. 

Linux is really capable of hardware acceleration, see Blender for an example. But it's just not wildly use compared to Windows. And lacking games and professional apps are another reasons entirely. I hope this situation will change. I knew it's not OS fault, but that's the thing we've to bear with it seeing it as a whole Linux experience.

Should Ubuntu define guideline for apps developers to build their app accordingly to hardware acceleration thing? I heard entirely of Windows 8 system will be hardware accelerated, and it's a lot faster than Windows 7 now, is that true?
