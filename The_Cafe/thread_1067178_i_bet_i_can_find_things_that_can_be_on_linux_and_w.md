### Post by Kopachris on 2009-02-12
(too lazy to read the whole thread here...)
This is basically what the WINE project is all about.  What happens is that each OS has its own libraries that executables are linked to when they're compiled.  So when an executable is run, it calls functions from those libraries.  If those libraries don't exist, then the program can't run.  It might be a fairly simple matter to move all Windows DLLs over to Linux and set up a compatibility layer so the can work, but that's illegal in most countries, so WINE has to reverse-engineer them the best they can without actually reverse-engineering them (disassembling/decompiling).

The best we can hope for in order is:
A company porting and distributing their software for Linux
Virtualizing/Emulating Windows in a program like VirtualBox
Use WINE and hope the program works
