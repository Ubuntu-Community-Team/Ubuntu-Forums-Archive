### Post by montag dp on 2016-08-16
You don't need to have some great idea for the "next killer app" in order to have fun programming. Just start with something small that interests you. Maybe it's a utility to make some aspect of your computing life easier, or maybe it's a niche application related to one of your other hobbies. It doesn't even have to be that useful; all that matters is that you enjoy doing working on it. Maybe eventually after working on it little-by-little, it will actually become pretty good and others will benefit from it too.

One trick that can sometimes combat the problem of getting tired of your project before finishing it is to work on one main project, but let that blossom into supporting side projects. For example, I'm working on a program (for fun) to do airfoil optimization. It's terminal-based and currently just uses an input file for all the settings. I wanted to create a better user interface but didn't really want a full-blown GUI, so I started working on an ncurses-based menu library that will eventually go into the main code and some others too. This code will also eventually use embedded python for displaying plots as it is running, which is another example of a way to diversify your project and keep yourself interested. You could also go the other way -- for example, write a GUI in python and use C/C++ extensions to handle any CPU-intensive parts.
