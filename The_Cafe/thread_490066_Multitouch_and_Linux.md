### Post by Mr. Picklesworth on 2007-07-02
I was [looking at this](http://www.instructables.com/id/EJIXKOEF3ER7VN5/), and at this point (recently bumped into a cheap projector at a thrift store, and everything else is easily come by) I am now seriously considering building a multitouch table. Seems the sort of technology that will grow a lot on the software side without too much change for hardware, so I'd best start now so I can be the first on the block with the next big thing!

However, that particular article suggests some software currently only for Windows to get it all going. Not quite what I was hoping for!
Are there any projects to create a generic multitouch library for Linux? Multitouch desktop features for Gnome? (Considering toying with that one once I get such a thing talking to Linux; I have some ideas to chase for features tying in with the device recognition framework that shall conquer Surface...)

My thought for this is that X does not urgently need multitouch support; a multitouch system could interface with a regular "mouse driver", perhaps, with the multitouch support enabled by a specialized multitouch library, allowing for advanced shape recognition when enabled by applications on the desktop.

Anyway, is there any progress anywhere beyond my dreams?
