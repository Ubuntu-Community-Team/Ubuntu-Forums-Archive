> **Paqman said:**

But they don't do it nearly as well as a client, and it's not nearly as customizable as how clients like Thunderbird can separate the different accounts and run rules on them. > **Dragonbite said:**
> I've never had a problem with my IMAP email accounts not deleting my email.  Are you sure you aren't connecting over POP?  That would download your email to multiple clients.I could almost guarantee that the OP was using POP instead of IMAP and that the client wasn't set up to delete after downloading. The beautiful thing about IMAP is that everything is always stored on your email server, so everything you do, including deleting emails, actually affects whatever is on the server. With IMAP, you can have the same email account set up in clients on an unlimited number of computers because they don't remove the emails, they just fetch whatever is on the server, which remains there. On the other hand, POP actually downloads your emails, and can be set up to delete what's on the server after the emails are downloaded. If you do that, you can only have one email client set up. And if you're using POP and the client is not set up to delete after downloading, every email you ever get will stay on the server until you actually go to the web site and delete them, even if they're deleted in the client.
