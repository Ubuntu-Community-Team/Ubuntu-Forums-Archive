### Post by earthpigg on 2011-05-01
me again, with the classic "yay netbook" session setup.

-openbox

-lxpanel on autohide

-nm-applet and gnome-power-manager run on lxpanel if told to do so, so i can have uniform netowrking and power management options set.

-lxpanel "menu settings: icon" is "/usr/share/icons/Human/scalable/places/gnome-main-menu.svg"

```
[chris: ~/.config/openbox]$ cat autostart.sh
# run system wide openbox config
. $GLOBALAUTOSTART

# autorun programs
lxpanel &
nm-applet &
chromium-browser &
gnome-power-manager &
[chris: ~/.config/openbox]$ 
```

### Post by hhh on 2011-05-03
re: Using Nautilus not drawing the desktop (in KDE or Xfce, for instance), there's no need for gconf-editor or the --no-desktop switch, though both are also easy solutions. In a terminal, just run```
gconftool-2 --set "/apps/nautilus/preferences/show_desktop" --type bool false
```

Confirm it by running```
gconftool-2 --get "/apps/nautilus/preferences/show_desktop"
```
Terminal should report back false. WFM.

-edit- You could also run this for good measure...```
gconftool-2 -s -t bool /desktop/gnome/background/draw_background false &
```

These will persist into a Gnome session though. nautilus --no-desktop is probably best if you boot into Gnome sometimes.

It's easy, you just need to do a script like this:

```
#!/bin/bash

sleep 10 && conky -c ~/.conkyrc &
sleep 11 && conky -c ~/.secondconkyrc &
```

Then add it it to you startup list.

You really need to get smooth-tasks installed from an active branch of the source,since it has persistent launchers. Laternatively you could try installing flexible tasks ([http://kde-apps.org/content/show.php/Flexible+Tasks?content=129291](http://kde-apps.org/content/show.php/Flexible+Tasks?content=129291))


to install smoothtasks from source
```
sudo apt-get install build-essential cmake git 
```

move to a directory that you want to install to, I install to  ~/Programs/Kde/
```
hg clone https://bitbucket.org/jimi312/smooth-tasks-kde-sc-4.6 
```
This will pull in a folder called smooth tasks, go into it.

```
mkdir build
cd build
cmake .. -DCMAKE_INSTALL_PREFIX=`kde4-config --prefix`
make -j 2
sudo make install
kquitapp plasma-desktop && sleep 2 && plasma-desktop
```

pm me if you get stuck. :)

The equinox themes can be fixed to look good with a global menu and nautilus elementary (if you have the latest). Assuming you have them from their ppa:
Press alt+f2 and type gksu nautilus, to open a nautilus window as root.
Go to /usr/share/themes/Equinox ????/gtk-2.0/ where you replace the questionmarks with the name of the theme (evolution, dawn, dusk, evolution light, etc). 
Open the gtkrc file there with gedit (or whatever) and scroll to the bottom. You'll see two lines

#include "apps/nautilus-elementary.rc"
.
.
.
#include "apps/globalmenu.rc"

Well remove the # from the beginning, save the file and reload your theme.
