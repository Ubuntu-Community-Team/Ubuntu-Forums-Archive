### Post by TheFu on 2013-01-06
Leaving x86 compatibility will cause some programs that you want on the ARM not to work.  I'd strongly .... STRONGLY suggest checking into a ChromeBook, but wiping the OS for something you like.

I have a netbook (Asus Eee) AND I have a 10" tablet (Acer A500) with a keyboard. I haven't been able to load a native Linux distro onto the tablet, but I have loaded Ubuntu ARM into a chroot environment running under Android, then inside the Ubuntu, started a VNC server, then in Android started a VNC client to access the VNC server running on the same machine.  Clear as mud?  Check the comments here [http://blog.jdpfu.com/2012/02/03/how-to-root-an-acer-a500-running-3-2-1](http://blog.jdpfu.com/2012/02/03/how-to-root-an-acer-a500-running-3-2-1) for more details.  At the time, the keyboard and mouse/touch screen interaction was badly broken.

Your idea about using a tablet instead of a netbook is a great idea, BTW.  There is a guy in Europe that switched to an iPad with a keyboard as his main laptop PC for day to day needs, including work as a software developer.

Myself, I've switched to a cloud-based desktop and use whatever client machine I'm need to access all my normal stuff over an NX connection.  More about that is here: [http://blog.jdpfu.com/2012/10/23/remote-desktops-rock](http://blog.jdpfu.com/2012/10/23/remote-desktops-rock) . I tested this remote desktop setup from Istanbul, Turkey a few months ago.  I'm sold.  Actually, I'm using it right now to access a Linux desktop running in a private cloud over an NX connection from a 15" laptop.  

I wish, wish, wish, wish there was an NX client for Android, then all my remote desktop problems would be solved. I will look for a way to load Ubuntu/ARM onto my A500 again. Perhaps all is working now?  I really want a dual-boot situation, but a minimal Ubuntu + NX client would be just fine for me on the tablet.

Anyway, hopefully, this will give you some new ideas.
