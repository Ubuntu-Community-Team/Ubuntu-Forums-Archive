### Post by saulgoode on 2007-04-01
> **Ozor Mox said:**
> > for example if i wrote a program and distributed version 1.0 as GPL. i could then rerelease it as a proprietry version 2.0 (even if the code was exactly the same). however someone could take the version 1.0 code and use it to start a new project and keep it gpl.

Does that apply to many of the free software licenses or only the GPL?

The *main* difference between GPL and other open source license is the first part of Ssam's quote: "**if I wrote a program**". *Only the copyright holders* of GPLed software can "proprietize" later releases (John Smith cannot change the GPL licensing of Tom Jones' software). For GPL and other open source licenses (MIT, for example), the original version (1.0) is still open source. For non-GPL open source licenses, non-copyright holders can release "proprietized" versions.

From a "do whatever you want" standpoint, other open source licenses permit more freedom; but many authors prefer that later versions of their software are released under the same conditions under which they licensed it. This is basically the main reason why 70% of open source projects are released under the GPL.

Also, copyright licenses only extend for about 70-90 years, so in about a century all GPLed code will become public domain (just like proprietary software).
