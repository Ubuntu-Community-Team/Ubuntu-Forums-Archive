### Post by jvc26 on 2008-03-09
Despite it now being 2008 I would say that both have their place. CLI for a server install or for fast execution of particular tasks which would take ages via GUI and would probably be more resource heavy. But if I'm just using stuff on a desktop install I would probably say use GUI. The other CLI advantage is the portability of settings between servers and different computers finch, screen, irssi and mutt offer a highly portable setup which can quickly be grabbed from an SVN repo and have you ready to work in moments.
