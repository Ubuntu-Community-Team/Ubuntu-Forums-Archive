### Post by TheFu on 2022-04-01
We should all remember that "computer" used to describe a human who performed complex calculations. The word "computer" wasn't created for the machines created in the 1940s, it existed for centuries, if not longer, before that time.

I have an abacus, but cannot use it as quickly as I've seen in different markets around the world.

> ps: I loved the ISPF editor on the mainframe.. If I use the command `ispf` it'll start a open-source editor that is based on the ISPF editor of IBM mainframes on this Ubuntu box.. I may not really use it, but it sure brings a smile to open a text file up with it on occasion, stare it awhile & not remember how it works, then just put fingers to keys & my fingers know what to type still. 

I liked SPF-Edit enough to buy a copy for OS/2 when it was available. Probably have the floppy disc here somewhere still.  I loved the 'hide' feature.  Today they call it folding.  Made it easy to get the flow of a module quickly.  In maintenance programming, we'd have 800 line modules with many decision points and only need to modify a few variables.  For performance reasons, all the variables were global (the stack was too slow). Plus the system was multi-threaded, real-time, and we needed to ensure that other threads weren't reading data out of turn.  Anyway - I'd hide everything in a module, then 'find' all the instances of that variable, followed by 'find'ing all the decision statements (if/else/case/switch/goto/....).  It was a way to quickly follow a few variables in a very complex module.  I've never found anything similar in other tools, but suppose that vim or neovim can do it with addons. Yep: [https://vim.fandom.com/wiki/Folding#Folding_methods](https://vim.fandom.com/wiki/Folding#Folding_methods)  
With OOP, the data is more important and it becomes easier to follow where it is modified and why, so following the use of a variable across 10 highly complex modules just doesn't happen anymore.

I haven't used "college math" in decades. Normal programming doesn't need it for the vast majority of problems and the implementation in computing tends to be for 'close enough' approximations, not the math correct answer (for non-trivial calculations).

### Post by lammert-nijhof on 2022-04-03
In 1970 I used the first computer. It was a 32/8-bits Philips P1100 main frame; 64KB  of core-memory; three 7.5 MB removable disks, paper tape and punch card  equipment. I was working in the prototype test department and I used the  computer hands-on for one hour/day to test the interface between  computer and new peripherals, in my case data communication equipment. The test programs we would insert using  the 32 binary keys on the console. At the end of the hour we would punch the binary to paper tape for the next day :)

In 1990 I bought my first PC, a 2nd hand Philips P3105. I was using computers all day at work, so at home I tried to avoid PCs for ten years or so.  My first computer was that P3105 XT clone, because I wanted Girotel, a banking service from the state owned Postbank. The Postbank did beat all commercial Dutch banks with this on-line banking. The computer was a XT clone with a 8088; 768MB of memory; 5.25" floppy disk; 20MB HDD and a 2400 bps modem for Girotel. The system did run MS-DOS, first 3.3 and later 6.0. The only useful app was of course Girotel, it allowed me to check my bank account and to transfer money for electricity, mortgage and other payments. 

In 1993, I was lost and I did build a 486DX66; 8MB and a 300MB disk and I connected the two by Ethernet, at that time a thin-wire-coax at a whopping 10 Mbps.
