### Post by jwbrase on 2009-04-11
> **zekopeko said:**
> i tried BumpTop and didn't like it. it's just bling (and rather foogly at that) and no revolutionary desktop metaphor.

There's some stuff in the video that looks fairly interesting, and I wouldn't say that it isn't a new metaphor (or at least a major extension on the existing one) but generally I tend to use menus, panel launchers, and file managers. The desktop is just a place to hang a pretty picture and put a few icons I don't use often that don't fit into the launcher space (Not to mention the place that gets covered up by open windows). 

BumpTop seems to extend the desktop metaphor in a really interesting and revolutionary way, but the desktop metaphor in the first place is something that just tends to, well, sit on my desktop, while all the real action is taking place on the panel, it's menu(s) and in Nautilus. (Or Windows explorer, in Windows). And the layout of BumpTop seems to make it harder to make that pretty picture really look good. (If it's there at all)

What's really needed is a revolution in the panel/menu/filetree metaphor. It would, for instance, be somewhat nice to have a filetree that could be browsed "sideways" as well as up and down. 

For instance, the Linux filetree is one of my pet peeves, because all the files for one program end up scattered across fifteen different directories like /usr/bin, /usr/share/doc, /home/username/.programname, etc. 

In Windows it's just C:/path/programname/, generally with the binaries in the program directory itself, documentation in a directory like ./manual or ./docs, data in a directory like ./data, and so forth. 

It would be nice to have a filetree that could be browsed both ways, so that you can either go to /usr/bin/programname/ or /applications/programname/bin to get to the directory that contains the binaries for that program, and either go to /usr/share/doc/programname or /applications/programname/doc to get to the documentation folder.

EDIT: Now if you really want to extend the desktop metaphor in a way that would really work for me, the way to do it is, instead of adding extra desktops like alot of existing DE's do, or doing some of the stuff that BumpTop does, rather, make it a matter of a single desktop, scrollable and zoomable, and more or less infinite in extent. That way, I can lay out windows next to each other without quite as much, if any, overlap (depending on exactly what I'm doing), and just scroll the desktop back and forth between them.
