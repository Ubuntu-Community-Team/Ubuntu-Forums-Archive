### Post by dsginter on 2005-04-11
Ubuntu is getting closer...

BUT...

After a decade of doing it in Windows, I miss being able to close windows simply by jamming the mouse into the upper right corner of the screen and clicking 'X' to close.  Even with the extra taskbar moved to the bottom, I still have to place the mouse directly over the X in order to close the window.  Yes - this is TREMENDOUS nit-picking but I don't even think about this anymore after doing it for so long in Windows.  It is a subconscious action at this point.

Once again, this is a show stopper for myself and most of the people that I know.  It is worth the $50 or $100 bucks for MS Windows just for this feature.  If I could get this along with some real multimedia support out of a single media player, I'd be sold.

While I am nit-picking (I truely have no problem with this), most of the n00bs that I've introduced to Ubuntu don't seem to take well to the three menus in the upper left (applications, places, system).  I believe that making them each more distinct (and possibly "button like") would help eliminate this confusion.

FWIW, I only gripe because I want to dump Windows.  This is getting closer but not quite yet for me.

Thanks Ubuntu Developers!

### Post by wayne040576 on 2005-04-12
I don't think the issues mentioned are any reason to drop a desktop. They are small problems. But I think its true that for certain users,  they see this type of thing as a bigger issue. I have been using Linux for about 2 years now (mostly slackware). I think over time as a user, you become immune to the small problems and don't see them  because you know that practically everything in Linux is configurable. But we have to remember that new users do not always know this.  For the last 6 months or so, I have had an Ubuntu box set up in the house for the rest of the family and it was through them that I saw small issues like this that really annoyed them. The biggest problems related to not being able to find things in the menus. Also today, while looking on Distrowatch I came across this link. 

[http://mpt.net.nz/](http://mpt.net.nz/)

I'm not sure if it has been posted on these forums before (I couldn't find it) and I'm sure many others here saw it on distrowatch today. It's just a list of small annoyances on the Hoary desktop. A lot of them are related to the UI (some gnome , some ubuntu). The author does go a bit overboard in places but I think there are a few fair comments. One that I can relate to is the system checking for the NTP time server on bootup even when there is no network connection. This really annoyed me a few weeks back when trying to set up a wireless connection. It seems to be on by default on a lot of Distros (excluded in slackware). I know it can be easily turned off using init scripts, but perhaps this is the type of thing that can left off by default to prevent new users having trouble,  and enabled by the user if necessary?
