### Post by Circus-Killer on 2007-10-03
to go against the norm of this thread, no i do not use the command-line to listen to music. i also dont rub sticks together to make fire.

again, the age old discussion between minimilists and....what can we call them.....evolutionists. :P

basically, i am fan of the command-line, in its time or place. for example, if i were to set up a server, i would have only a command line, nothing else, so that all the resources can go into doing the servers job.

however, on a desktop pc, i disagree with the whole minimilist approach when taken to extremes. i dont see a point in have a dual-core P4 with 2GB of RAM and you dont use anything more sophisticated than command-line tools. and when you trying to get desktop activities done, they usually take longer to do than in a gui mostly because of all the typing and doing this and doing that. essentially, minimilists try to play as the efficient type, but often the efficiency is thrown out the window on a minimilists machine. doing things require a lot more actions from the user when in command-line.

as i said, time and place. this is obviously just my humble opinion, but i just think that some people dont use enough discretion. like i know that from the command-line, individual low-level commands will process quicker, but on todays machines you also have to look at the time the user is spending just "requesting" the task.

a perfect example is gentoo. everyone raves about gentoo's efficiency. but what people forget to take not of is that the efficiency only occurs whilst the system is at a single state. but when the next lot of upgrades become available the next day, you in for at least a day of compiling, which throws the efficiency idea right out the window.

so yah, to finally get to answering the question. no i dont use a command-line music player. if i'm gonna have a desktop pc, it must have the same functionality and efficiency that i expect to get from a modern desktop. just my opinion.

### Post by sunexplodes on 2007-10-03
I think it's useful in having a balance. Especially when you have browsers like the 2.x line of Firefox which is an enormous memory hog, it's nice to have a fully-featured music jukebox that takes up ~3MB of ram. And if you don't feel like being a luddite, you can use a frontend like Sonata, which adds a really elegant GUI for another 4 or 5 megs of ram. 

Just because you have a powerful system doesn't justify using bloated software. I want an media player that handles playlists and large libraries well, and that's all i really need. so should i waste my memory on amarok? 

I'd rather have all that ram powering Compiz-Fusion, Firefox, and the Gimp.

And I understand your point about the time involved in using a command line player, but you're making the mistaken assumption that we're typing in a command every time we want to skip a song, when it's very simple to map keyboard shortcuts to those commands.

### Post by mali2297 on 2007-12-13
> **bionnaki said:**
> is there anyway to bind a player to a keyboard shortcut so that it will appear/disappear? foobar can do that on windows & it would be wonderful if a CL music player could as well.

I'd rather not use all-tray, but I'd like quick access to the player without having to keep it in a panel windowlist or enter a command into a terminal.

possible?

With mpd+ncmpc, you could map a keyboard shortcut  to
```

xterm -e ncmpc

```
This will open ncmpc in a terminal window and you can create a playlist. When you exit ncmpc, the window closes but mpd will continue to play the music.
