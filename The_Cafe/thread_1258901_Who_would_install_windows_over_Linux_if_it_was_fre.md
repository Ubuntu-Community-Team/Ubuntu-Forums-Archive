[http://technet.microsoft.com/en-us/library/cc785638%28WS.10%29.aspx](http://technet.microsoft.com/en-us/library/cc785638%28WS.10%29.aspx)

This explains it better than I could. Another very nice feature in Win Server is that NTFS is now self-healing. You know how in most other OSes, you get a big warning when you try to fsck a mounted filesystem that the operation could damage them? Well, starting with NTFS6, chkdsk can automatically "shut down" a directory to check it, and all write operations to that directory will just be put in cache until the check is done. Then it does the same with another dir, and in the end, you have your whole filesystem checked without needing to bring the whole server down. For a file server with 10+ TiB of data, I can tell you this is bliss.
