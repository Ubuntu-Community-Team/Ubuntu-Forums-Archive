> This isn't going to be in German, is it? I have a hard enough time speaking English!

Oh, good. Wow, nice!!!

Just to let you know, the GIMP uses GLIB for internationalization (i18n) and will honor the LANG environment variable to set the language preference. This allows different users on the same machine to run in different languages.

You can override your user's language preference by invoking the GIMP from a BASH shell with LANG set to your language of choice:

***LANG=de_DE gimp***

You can even open up multiple instances of the GIMP in different languages at the same time, but you will have to use the '--new-instance' option:

***LANG=fr_FR gimp --new-instance***
***LANG=en_US gimp -n***

If '--new-instance' is not specified and you already have the GIMP open, then any files specified would be opened in that already-running GIMP (this assumes you have DBUS installed on your system). 

The command 'locale -a' will list the languages available; though not all are fully supported by the GIMP.

I am really looking forward to Gimp 2.4; at last; "completed the color management functionality" :)
Well, don't get too exited about the color management functionality in GIMP 2.4. I've looked around in GIMP's source code and unfortunately the color management functionality is still very limited. You can set a monitor profile and GIMP will recognise ICC profiles in your images, but GIMP does not work correctly when your image is in any other color space than sRGB.

Unfortunately, many operations in GIMP are hard-coded for sRGB, and doing those operations on images in any other color space will simply produce wrong results, that you might not even notice immediately (because the difference between what GIMP does and what is actually correct can be subtle).

See for example [GIMP bug #499794](http://bugzilla.gnome.org/show_bug.cgi?id=499794).
