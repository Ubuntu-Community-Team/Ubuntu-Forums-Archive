### Post by xterminus on 2006-10-15
> **bonzodog said:**
> xterminus: the bash set-up is neat;
do you have all your .<configfilename> in the /dotfiles of your home directory?
If so, how do you get programs to write and read from there?

Yes.  For a few years I kept all of my home directories stored directly in a subversion repository (before that CVS).  However, maintaining different branches of wildly divergent configuration files became a chore so instead I created one repository of configuration files which I checkout right away when I sit down at a new pc.  Then I symlink .whatever to dotfiles/program/whatever.  This allows me to keep things nice and tidy across all my different workstations (and in one branch).

Think of it as another take on [Joey Kite](http://kitenet.net/~joey/blog.html)'s take on [Subverting your home directory](http://kitenet.net/~joey/svnhome.html).

### Post by testube_babies on 2006-10-17
> **m.musashi said:**
> Okay, I found the references to changing the menu icon. These are the tips given to me by renzokuken...

...Hope that helps.

In Breezy this process was simple; each icon theme used the same menu logo.  But it Dapper and Edgy, each icon can individually define this logo.  Since each iconset is structured differently, the location of the logo is always different.  Luckily, the logo is almost always called distributor-logo.  So if you search for distributor-logo you can then find the one in your icon theme that you need to change:```
sudo updatedb
locate distributor-logo
```
If the logo is an SVG, you can replace it with a custom SVG logo.  If it's a PNG, you can replace it with a custom PNG that is a square image the same size as your panel.  After you've replaced it, this should get it to show up: ```
sudo gtk-update-icon-cache
sudo killall gnome-panel
```
Note that if you are using an older icon theme that doesn't specify a distributor logo, this is the one you need to replace: /usr/share/icons/hicolor/48x48/apps/distributor-logo.png

I did try Suse 10.2b1 this weekend, and it looks pretty, but it's pretty broken here. Sound didn't work, banshee isn't in their factory repository, and gstreamer is pretty crippled, unless you want to compile everything yourself. Suse should switch to apt :P.
