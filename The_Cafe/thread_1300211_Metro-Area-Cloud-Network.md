### Post by Tristan Tonks on 2009-10-24
Hi everyone, I'm working on a Metro-Area-Network for my town (Minehead,UK). I have started by getting a few businesses to commit to serving up their websites from linux servers hosted on their own premises. I will be maintaining them and when the technique is finalised I will be rolling out the networking and cloud computing software across the servers. The Idea is that the system will offer free access via wi-fi to local browsers who will be able to browse locally for products and services while paying for access to the wider web. The cloud computing 'layer' will provided enhanced processing and data storage for local businesses and the web servers while the wifi network will allow better local communication. My main USP's are reliability over typical internet format, cost effectiveness and reduced dependency on fossil fuels thanks to local sourcing and solar-powered; server modules independently maintained and connected.
One of the key objectives of this is that the web should be free, and the internet should be reinforced by modularity, communications that piggy-back on any available hardware, and also that no hardware should be left idle for hours at a time when some other hardware is overloaded (cloud computing). In addition I am hoping in the long-term (8 years?) for the servers to provide back-up databases for supporting the incoming AR apps for future devices.
_This is what I'm I think the targets should be for this system._
[LIST=1]
[*]Wi-Fi based server networking.
[*]Cloud Computing sub-layer over network.
[/LIST]
I have posted this here so that as a community we can decide where/and how to take this forward.
...Discuss....

### Post by Tristan Tonks on 2009-10-25
That's not 'downer'....Its constructive criticism. We have a Wi-Fi network that extends about 5 miles across an area of Exmoor (a national park with residences both business and domestic) to provide internet access to paying customers. (not my venture). That network is relatively stable and works in a 'ad-hoc' way. I am anticipating the use of the latest Wi-Fi standards and equipment in all the machines which technically should be fine if suitable software is available. This is really the biggest problem, The software needs to be based on cloud computing methodology, packet management, chaos engine type packet transfer and crucially have the ability to request bandwidth and processor time from devices including devices that are passing though the system; mobiles, PDA's and other computers (carputers, media devices).
My thoughts are:
[LIST=1]
1) C++ Cloud Computing layer.
2) Python packet manager/layer bridge
3) ROOT (or other) Chaos Engine for packet transfer.
[/LIST]
