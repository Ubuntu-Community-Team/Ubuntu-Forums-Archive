### Post by carson.matthew on 2007-10-28
First OS was MS-DOS. Dad had a Tandy, and thats where I learned the basis of my computer knowledge. There was a program that was a video of how computers work and stuff. It was amazing. I was only 5-7 years old at the time.

Parents eventually upgraded to a Compaq that had Windows 98. Seems how I was the computer geek of the family, I was pretty much the only one who used it.

My first pc that was officially mine was a Windows 95 machine. It was an IBM with a 75Mhz processor, 16MB of ram, and a 1gig hard drive.

Later down the road, Parents upgraded to an HP Pavilion with Windows ME. Going from Windows 98 to Windows ME for the parents was good. ME was a great OS for the home user who didnt have an internet connection. All the security holes didn't exist without the internet. However, all it was was a facelift for 98.

Started using XP when I was a junior in high school. I remember fearing it so much with it's "gay color scheme" and incompatibility. I set up a dual boot with Windows 98 as my primary. I slowly but surely learned XP.

Once in college, I was exposed to more ideas and faster internet! A friend of mine was an avid linux user. He got me setup on Gentoo, but I was never able to get that running (this was about 4 years ago when you had the option of compiling everything, with no GUI installer. Took me a full weekend just to fail).

Once I got into the Computer Science program, we were learning Linux and I really took a joy to it. There was so much more you could do with it compared to Windows. I couldn't game with my friends, but there was enough exploration that kept me content. Then I went back to Windows to game.

Since then, I have been in a cycle: run windows for a few months, run linux for a few months. I am waiting for the day Game Developers notice the potential for creating Linux clients. It will be a grand day indeed.

BTW, used Vista with a AMD X2 4200, 3GB of RAM, ATI X1800XT. Runs good just like XP when freshly installed. After a few months, it's bloat catches up to it and slows down terribly. I am much more satisfied in Linux all the way around besides the gaming aspect. :)

[EDIT:] Wow...after reading this, I feel OLD! Back when computers were 75Mhz, had a 1 gig hard drive.....wow...

Personally, I would combine Windows 3.X and MS-DOS 5-6 as one operating system. Starting with Windows 3.0, Microsoft started using a VxD based virtual machine manager that hooked the protected mode interrupt vector table to replace certain DOS int 21h function calls (essentially the heart of the OS), so that it became hard to tell where DOS ended and Windows began. A similar 32-bit DOS-Extender technology was common in DOS games of the early to mid 90s. 

I lumped Windows 95 and 98/98SE together as Windows 9X since they're pretty much the same system, just an evolution in terms of replacing old DOS code under the hood, upgrading to IE 5.0, as well as including better USB and DVD support. Windows 98 also introduced the Windows Driver Model (WDM), common to Windows XP drivers, but driver developers continued to develop VxDs to remain compatible with Windows 95. That being said, the transition from Windows 3.X to Windows 95 was exponentially  bigger than from 95 to 98 -- at least it was for me. Windows 98 just felt like a polished, rock steady Windows 95.

Bearing in mind that many of the versions of BASIC in the late 70s and early 80s were licensed, repackaged and hacked versions of Microsoft BASIC, I estimate that over 70% of the respondents so far began using computers on a Microsoft system. The next closest competitor is (surprise) Apple at 7.3%.

> **Mizzou_Engineer said:**

It had DOS ROMs that were actually ROMs, not EEPROM Flash like we have today, so a computer pretty much ran whatever OS it was shipped with. I don't believe that I saw computers that booted from the hard drive rather than ROMs until about the time 80486s came out.

The first I'd heard of Linux was in the late 1990s when Red Hat IPO'd and we got the Microsoft Halloween E-mails.

The 80486 came out in 1989, but well before that in the 80s computers had hard drives and *Disk* Operating Systems (DOS). Even in the late 70s and early 80s -- before hard drives were common, back when cassette storage was popular and 8-bit computers came with BASIC in ROM -- if you used floppies (5.25" or 8") or a 'Winchester' drive, you probably booted with a sys formatted disk that had a DOS on it (CP/M, Apple DOS, etc). 

The first IBM PC in 1981 only had an optional 5.25" floppy disk with 160 KB capacity, but the first clone PC in 1982, the Eagle PC, had the option of a 10 MB hard drive. With the release of the XT, IBM's 1983 successor to the PC, a 10 MB hard drive became standard issue. This was facilitated by a new feature in MS-DOS 2.0: subdirectories to manage all those pesky files. The only way to get an XT to execute the ROM BASIC, as opposed to booting DOS from a disk, was to physically disconnect both the hard drive and the floppy drive.
