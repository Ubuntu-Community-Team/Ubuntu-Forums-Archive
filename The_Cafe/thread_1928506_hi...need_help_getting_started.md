### Post by TheFu on 2012-02-20
This question is very common. In short, 
* Find an itch - something you want to program
* Search for others already working on it - freshmeat.net is a place to find like-minded developers. Check Launchpad and Google for projects too.
* Download the source code from git, sourceforge, where ever
* Build the programs and start using the software.
* When you find something that is broken, submit a bug through whatever channel that specific project uses - do they have a bug tracker?
* Try to fix the bug yourself and provide a patch or "pull request" to the team.

Doing all of this will force you to learn about bug trackers, version control, working with the community, coding styles and they will probably reject your first updates.

I've attempted to answer this question in more detail here: [http://blog.jdpfu.com/2011/10/19/how-to-learn-to-program](http://blog.jdpfu.com/2011/10/19/how-to-learn-to-program)
