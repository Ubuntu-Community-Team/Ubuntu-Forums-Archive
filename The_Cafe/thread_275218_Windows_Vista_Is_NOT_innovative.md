Ahh another poor brainwashed soul. The kernel was not redesigned from the ground up. Vista is designated NT 6.0, I'll give you a guess as to why ;) Yes SOME of the stuff now runs in userspace, most notably sound drivers. 
However:
1) Nothing that requires speed runs at userlevel 
2) Just about everything that requires speed runs DMA completely bypassing the kernel, so a poorly written driver will bring your OS completely down (same goes for Linux and OS X btw). 

Reasons Linux based OS's aren't bigger:
1) They do not come preinstalled from OEM's. There are small companies like System76 and Emperor Linux that will sell you a Linux based system but no large company sells Linux preloaded desktops/laptops to the public. You can get servers with Linux and if you are getting a T60 from a Lenovo redistributor it is possible to get SLED preloaded instead of Windows but it is a highly specialized machine not meant for the J6P. 

2) Organizations use Windows on their desktop. The reason that the PC got so big was the fact that organizations (schools, governments, corporations, etc...) saw that they would be beneficial to their bottom line. As people tend to take work home quite a bit they started getting computers for home use as well. Now, with all the specialized software and telecommuting capabilities you need to have the same OS installed at home as you do at work. Anyone who tried installing VPN on Linux will understand me :) 

3) OEM's like Windows. 
  a)Windows is uniform and XP is an XP no matter if it's Pro, Home or MCE.     Aside from some features it's the same thing across the board so it is easy to support (lets face it tech support people are rarely very technically proficient). Linux has quite a number of distribution each with it's own tools and little specialties, the most obvious being DPKG vs RPM so an OEM has to basically stick to one distro and becomes as locked in as MS. 

  b)Linux is an extremely efficient OS that can run on very limited resources. Ubuntu while a very userfriendly and GUI centric OS can run just fine on a P4 2.2 w/ 512MB RAM and a 4200 RPM HDD (the system I am typing from) to make it worse Dapper is actually faster on here than Breezy was and one CD comes with a complete OS that includes and Office Suit and a professional grade graphics manipulation software (please no GIMP vs Photoshop arguments, GIMP is professional software that can be (and is) used by artists). The only thing that can raise hardware rq's for Ubuntu is Compiz/Beryl stuff that requires a decent 3D card even then it doesn't need much more than an old 64MB ATI/nVidia. As a consequence 5-6 year old hardware will work just fine with newer versions of Ubuntu so that the users wouldn't want to bother with new hardware. 

  c)Very little incentive to upgrade from one version of an OS to another. Dapper will be supported for the next 5 years, and even if it isn't it will still work fine on w/e it is currently installed on. Also given the overall security of Linux patching it is less important than it is with Windows (although with greater numbers there are greater threats but still won't be nearly as bad as Windows). With preinstalled OS's upgrades usually happen along with new hardware, if people don't see a point in upgrading the OS (and as a consequnce the hardware) OEM's won't be making as much money as they do with Windows bloat. Look at Vista's rq's there is no reason whatsoever for a J6P who just does the online/web/porn stuff to upgrade to the hardware it will require.

4)Drivers. The kernel is GPL'ed, which means that it is ILLEGAL to create closed source drivers for it. The driver has to be compiled against the kernel and is considered a derivative work so has to be released under the GPL. Some companies such as nVidia find ways around it with a GPL'ed wrapper around binary code to install their drivers. That however is more work than most OEM's are willing to put into the development of drivers. There are enough OEM's that don't want to open up their drivers for one reason or another. The current licensing prevents them from being able to make Linux drivers (not that they really want to).

5)Binary compatibility. Linux strives for source compatibility, which is great and I agree. However ISV's might be reluctant to develop software if they can't guarantee that it will be compatible with Debian, Ubuntu (at this point they are different enough), SuSE, RHEL(FC/CentOS), Mandriva, etc... If you read the Flash development blog that has been linked to a few times you will see the problems they are having. I only listed the 4 most popular distros and they all have very serious differences. 

I would like to say that all of these issues can be overcome and fairly easily if organizations start adopting Linux to replace Windows. Eventually there will be one most popular distro that is preisntalled by most OEM's and all those problems will more or less go away. Any distro that Dell and other pick will be fine as anything that works with one distro can be made to work with others by the community. If Dell starts selling RHEL based machines I'm sure that Novell, Canonical, The Debian Project, Mandriva will be able to modify their respective OS's to work with the hardware and software that works on RHEL. I do believe the kernel should be LGPL'ed same way Gnome is, to allow hardware vendors easy proprietary drivers.

### Post by prizrak on 2006-10-11
> Look at the QWERTY keyboard layout. Wasn't by all means the best one... But everybody got used to it, we have been using it for over 130 years (Remington type-writer, remember that?).
Interesting you should mention it. The QWERTY keyboard was invented specifically to stop people from typing too fast. Before the IBM ball, the typerwriters would jam very easily if you type 70+ words a minute as the mechanical parts of them couldn't keep up with human fingers. Later when computers w/ keyboards entered the market Dvorak layout was invented to make 100+ word a minute typing easy and achievable by just about anyone (now the standard is 50). However due to the QWERTY layout already being around for like 70 years before people just kept using it. 

As for the topic. No Vista is not innovative, neither was XP or even DOS for that matter (it was bought from someone else not even developed). Linux is also not all the innovative it is very heavily Unix based and both of the major DE's borrow quite a bit of concepts from both Apple and MS OS's. In fact Windows can never be innovative, it is almost impossible. MS cannot change their OS around as drastically as Apple did when they released OS X. They have to keep to a certain level of familiarity in the interface and the API. There are way too many users, ISV's and OEM's that MS would piss off. 

The main argument against switching to/developing for Linux is that there is a learning curve. If MS changed their OS too much they would create a learning curve as well at which point it is extremely likely that organizations (not home users, those don't know what this leeenoooks thing is) would start looking at alternatives such as Linux. If you have to retrain all the users and relearn how to develop software for Windows might as well go for an OS that doesn't charge an arm and a leg for licensing (organizations would likely still purchase support but it's still quite a bit cheaper). 

As far as innovative ideas go. Good ideas are usually passed along pretty quickly and for the most part have been in place for years but could not be implemented. 3D accelerated desktop ideas have been around since before OpenGL but the hardware was a constraint. Desklets have also been around for a while but with CPU rendered desktops tend to use alot of power (my gdesklets use 15-20% of CPU) so again a hardware constraint. I by no means am trying to defend Windows or say it's innovative, in reality very few things are actually new, they are normally old ideas that for some reasons couldn't be done when they were conceived. 

It also depends on the point of view, if you compare Vista to XP then it is quite innovative. If you compare it to OS X then Vista is lagging behind. If you compare both to Linux then you start wondering why does Apple and MS still bother :)
