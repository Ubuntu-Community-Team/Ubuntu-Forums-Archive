### Post by Mr. Picklesworth on 2008-02-17
My argument for running Linux on newer hardware is that this is the exact same kernel (give or take some configuration options), and in many cases the same distribution, that is used on enormous and important server setups. Google, for instance!
You see that type of argument all the time for things ranging from batteries to headphones, so why not operating systems?

Besides that, Ubuntu is doing a much better job keeping up with modern standards than the competition. I just witnessed a new digital camera not connecting out of the box with a Windows XP machine, and Windows failing to find the necessary software online. (PTP, I believe...). The driver CD for said camera was not available. Ubuntu sorted it out in seconds, since supporting new standards is not restricted to enormously spaced releases -- a problem that is becoming quite evident given XP's very, very long existence.
(Granted, Ubuntu's included software probably wouldn't start supporting a new picture transfer protocol within a single major release, but the major releases are much closer together, so it feels like no time!)
