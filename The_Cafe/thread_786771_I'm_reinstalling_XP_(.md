### Post by toupeiro on 2008-05-09
I definately agree with the OP in regards to package management.  I've long believed that GNU/Linux should strive towards a standard package management format, which could be used seamlessly in all distros that are GNU/Linux based.  However, even with the misc. package management formats within linux, let us not forget Microsoft has its share of this:

- Traditional .EXE
- Miscosoft .MSI/.MST
- Wise Installer
- InstallShield
- AdvancedInstaller
- InstallAware
- Neosoft
- NETOPSYS FEAD

The list goes on and on...

The biggest diggest difference here between Windows and Linux is that for the most part, windows packages are transparent to the package management tool because there is a universal standard to go by.  EXE is just an old legacy DOS file extension to identify executable files that never died because there is no reason for it to.  Anymore, all an installable EXE is is wrapping paper around a self-contained proprietary software package.

  As seamless as packages may be in windows, thats not to say those differences are not displayed when an installation has a problem.  Wise, for example, is exceptional at rollback and uninstall capabilities, even for patches, whereas Installshield is more prone to leave you with broken things in Add/Remove Programs for the life of your OS should you have problems.    I think both .deb and .rpm are FAR better at non-interactive processing.  To get any windows app to silent install like this, it requires a lot of pre-configuration or knowledge of how the application was compiled..  Windows package managers and linux package managers could learn alot from eachother in their respective processes.

I think though that if you *really* compared the two, you would find there is much less YOU have to do with .debs and .rpms than you do with .EXE's or .MSI's.  I hope you give linux another shot, but in the end you have to do whats right by you. 

Cheers!
