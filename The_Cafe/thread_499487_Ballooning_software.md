### Post by Erdaron on 2007-07-12
Why does software keep increasing in size with every new turn of technological development?

I've only started using Linux, but I've been using Windows for years. Yet, while it is easy to blame Microsoft for shoddy development practices, Ubuntu's hardware requirements are not too different from XP. Though my post may mostly refer to Windows-based software, the intention is more general.

Anyway, a specific case in point. Base installation of Windows 98SE was about 100MB. Base installation of Windows XP is about 1GB. Why is there a ten-fold increase? As the end-user, I've only seen a marginal difference in operability - such things as native USB support, improved support for large harddrives and greater memory, more network support, and so forth. All of which seem like they wouldn't require a huge pile of extra resources.

One explanation I can think of is increased DLL libraries which are supposed to give more functionality to programs. So you could expect programs to be reduced in size, since many of them would be able to use common libraries, but that is not the case at all - all software is growing huge. In case of games I can kind of understand - better graphics mean more data to be stored and processed. But why would something like an office suite explode in size? It does pretty much the same stuff as before.

I should admit that I am no expert in the area of software development, and there are probably key points that I am missing. I would appreciate some enlightenment on the subject.
