### Post by 23meg on 2009-08-22
Reporting an issue and not following up with it is not good practice. Bug triagers and developers will often have to ask you for details you didn't initially provide, and for that, you need to be contactable in the context of the bug. That's why you're subscribed to bug mail for every bug you report.

If you're sure you don't want any bug mail, and want to only check manually, simply unsubscribe from the bug.

If you're a dedicated tester, it's also a good idea to set up mail filters for bug mail so that's it's filed into a folder automatically (or multiple folders according to some criteria) rather than flooding your inbox.
