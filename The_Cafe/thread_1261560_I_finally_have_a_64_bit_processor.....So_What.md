### Post by WalmartSniperLX on 2009-09-09
A simple way to think of the possible advantage is to imagine the processor as a **bridge** between the bus and memory, and the binary data that passes thru is a **lane** (like a bridge connection across a body of water). Dual-Core processors have 2 bridges as single core processors have one, a quad-core has 4 bridges and so on. Well lets say that a 32-bit, single-core processor is one bridge with 32 lanes. Traffic can easily flow thru the 32 lanes on the single bridge. Of course a dual-core setup (2 bridges with 32 lanes each) is faster as long as you're running a program that utilizes all the bridges and lanes. 

Well a 64-bit single core processor is like a single bridge with 64 lanes (technically). Data threads go thru the processor in 64bits instead of 32. Of course not all programs are designed in 64 bit or made to use multi-cores. Also, you need a 64bit kernel/OS to run 64bit programs. 

And that is a general understanding. My explaination may have holes in it but I think its an easy way to get an idea on the subject.
