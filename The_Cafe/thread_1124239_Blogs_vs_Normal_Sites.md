### Post by samjh on 2009-04-13
I was just thinking... will blogs eventually replace normal websites?

I don't mean that normal websites will disappear, but I wonder if blogging platforms like Wordpress will eventually displace a large portion of conventional personal and business websites and become the dominant form of web content delivery.

Today I spent several hours setting up Wordpress on my website, and I am experimenting with the idea of replacing my business site with a blog site featuring static pages for business information (yeah I know, it will completely screw up my Google indexes).

Advantages of blogs:
[LIST]
[*]No need to use HTML to post or edit content.
[*]Easily pluggable themes and widgets.
[*]Integrated news and updates via blog posts.
[*]Static pages for "normal" web pages.
[/LIST]

So are blogs the way to go for personal and business sites in the future?

### Post by James_Lochhead on 2009-04-13
Just to get a little nit-picky here. A blog is technically a journal, whereas (since you are talking about websites in general) I think you are referring to a Content Management System (CMS). Blogging software is a type of CMS.

Back on topic, I think that CMS will become the major player in web design in the future. It is much easier to do (not that HTML is hard) and can be extended. Even web designers will use it to save time and such.

It still has a long way to go though. I will still design websites manually until a better CMS comes along. I want to design highly customized websites and I don't like wading through overly complicated badly written documentation to find what I want. I would much rather sit down and write it myself.

### Post by Paqman on 2009-04-30
> **billgoldberg said:**
> Not blogs, but CMS's like wordpress, joomla, drupal ... are what is great about web development now.

Writing the code yourself isn't really a great idea nowadays when you have free open source solutions that makes the webmaster's task that much easier.

I've just started using Joomla properly to teach myself all about it, and i've got to say I find it a real pain. It's so restrictive about what you can do. It strips out a lot of code that i'd want to include in articles.

I can see how they're useful for a collaboratively managed site, or for getting dynamic content up without learning php, but for the majority of sites I think writing the html/css yourself and tacking on any php you need later is a lot more flexible and easier. A lot of the forum scripting software will do SSI that satisfies all of what most sites are going to need for dynamic content without complicating things with a full-blown CMS.

As always in web design this depends entirely on the site, of course.
