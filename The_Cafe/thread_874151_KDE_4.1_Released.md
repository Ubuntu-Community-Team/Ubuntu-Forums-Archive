### Post by tbroderick on 2008-07-29
[From KDE.org]("http://www.kde.org/announcements/4.1/")

> 
The KDE Community today released KDE 4.1.0. This release is the second feature release of the KDE 4 series, sporting new applications and newly developed features on top of the Pillars of KDE4. KDE 4.1 is the first KDE4 release to contain the Personal Information Management suite KDE-PIM with its E-Mail client KMail, the planner KOrganizer, Akregator, the RSS feed reader, KNode, the newsgroup reader and many more components integrated into the Kontact shell. Furthermore, the new desktop shell Plasma, introduced in KDE 4.0, has matured to the point where it can replace the KDE 3 shell for most casual users. Like with our previous release much time has been devoted to improving the framework and underlying libraries on which KDE is built. 

[Kubuntu packages]("http://www.kubuntu.org/news/kde-4.1")

Congrats to the KDE and Kubuntu teams.

> The updated packages for Kubuntu 8.04 are located in the Kubuntu Member's KDE 4 Personal Package Archive (PPA) repositories. To update to KDE 4.1, please follow these instructions:
 Add deb [http://ppa.launchpad.net/kubuntu-members-kde4/ubuntu](http://ppa.launchpad.net/kubuntu-members-kde4/ubuntu) hardy main to your /etc/apt/sources.list. 
 If you already have the kubuntu-kde4-desktop packages installed, simply type sudo apt-get update && sudo apt-get dist-upgrade and answer the questions in which you are prompted. If you do not have kubuntu-kde4-desktop installed, simply type sudo apt-get update && sudo apt-get install kubuntu-kde4-desktop and answer the questions. Both of these options are to be typed at the command prompt.

### Post by Trail on 2008-07-31
Desktop effects work for me (even OpenGL, which didn't work when I custom-compiled beta1) on OpenSUSE. Which surprised me, to be frank. And no, I don't use them, as I don't use Compiz either.

Since you mention you have a slow graphic experience, are you using an nVidia card? The drivers that nVidia issues are buggy and are causing these slowdowns, which sucks. Try with the 'nv' driver and see how performance is. If you are not getting that much slowdowns, then it's the drivers; visit kde.org and search for a link on quick fixes to minimize the problem.

On another note, I set my bar vertically to the left of the screen and it works very well. Doesn't look THAT good, but it certainly is better than KDE3.5 (where I had to custom-compile the taskbar to display the icons differently to make it usable).

Overall, I am very satisfied. It just needs a bit more time to be truly awesome (and resolve the stupid nvidia performance issue).
