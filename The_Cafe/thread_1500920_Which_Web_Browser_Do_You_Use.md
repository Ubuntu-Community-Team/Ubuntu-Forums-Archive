### Post by lovinglinux on 2010-06-03
> **PsychoDevon said:**
> Honestly, I'm hoping that Chrome's add-on library can one day challenge firefox's.

I have my doubts if they will ever be capable of that, due to limitations of the extension API. If you see the difference between some popular extensions for Firefox that have been ported to Chrome you will notice limitations of functionality and interface. 

Also [these videos](http://blog.chromium.org/2010/05/learning-from-experience-of-four.html) show created to promote Chrome extensions shows that experienced extension programmers had to came up with creative hacks to allow their extensions to work as expected in Chrome.

> **PsychoDevon said:**
> However, I do notice that a good portion of firefox's addons have to do with modifying itself, i.e. hiding the menu, changing how it manages downloads, etc. Which is really unneeded in Chrome.

I don't think they are unneeded, the problem is that Chrome doesn't offer customization capabilities to the extensions or the user. This is one of the reasons I don't like Chrome. I like to customize the browser interface to my needs and Chrome doesn't offer that.

Although there are tons of Firefox extensions that modify the interface, there are also tons of extensions that add new features or improve the user experience.  For instance I use one that saves Firefox passwords in Kwallet instead of the built-in password manager. I can't live without it.

I have no experience with Chrome extensions, although I have already made some "Hello World" experiments. Anyway, I have serious doubts I would be capable of porting some of my extensions, like [FoxRunner](http://foxrunner-extension.blogspot.com) to Chrome. I haven't tried yet, but I imagine the extension wouldn't be capable of launching a terminal for example.

> **PsychoDevon said:**
> In most situations, I think people only use firefox either because they're used to it or addons. There really are no other dominant reasons among ff users.

I believe the main reasons why Firefox users like it is because of customization, which includes add-ons. This is viewed by some Opera supporters as a bad thing, but in fact is great thing in my opinion and makes a lot of sense. Instead of distributing a browser bloated with features that not every user might need like Opera, Firefox relies on a powerful extension API, which allows the users to do almost whatever they want. 

While Chrome also has extension and is a lot cleaner than Firefox, it doesn't have a fraction of Firefox customization capabilities.

I think people only use Chrome because of speed, but if [compare a vanilla Firefox with a vanilla Chrome in a real environment situation,](http://lovinglinuxblog.blogspot.com/2010/05/browser-speed-test.html) the difference is minimal.

I think most people use Opera because they are stubborn :) Just joking, don't take me seriously. I know Opera 10.6 will be the fastest browser ever, but that will never make me switch to Opera.

I presume you are talking about extensions not plugins. 

> Extensions differ slightly from plug-ins. Plug-ins usually have a narrow set of capability. For example, the original impetus behind the development of Mozilla Firefox was the pursuit of a small baseline application, leaving exotic or personalized functionality to be implemented by extensions to avoid feature creep. This is in contrast to the "kitchen sink" approach in its predecessors, the Mozilla Application Suite and Netscape 6 and 7. Therefore, after integration, extensions can be seen as part of the browser itself, tailored from a set of optional modules.

Firefox also supports plug-ins using NPAPI. When the browser encounters references to content a plug-in specializes in, the data is handed off to be processed by that plug-in. Since there is generally a clear separation between the browser and the plug-in, the results are discrete objects embedded within a webpage. The same distinction between plug-ins and extensions is in use by other web browsers, such as Microsoft Internet Explorer, where a typical extension might be a new toolbar, and a plug-in might embed a video player on the page. Since plug-ins and extensions both increase the utility of the original application, Mozilla uses the term "add-on" as an inclusive category of augmentation modules that consists of plug-ins, themes, and search engines.
