[ATTACH]219079[/ATTACH]

Also wanted to share this with you, it's a fix I'm using to help correct Firefox with dark GTK themes.

This gives you a nice black text cursor, & black on crisp white in the text entry input areas.

####  Firefox text fix goes inside the <theme name>/gtk-2.0/gtkrc (or an app  fix in the gtk-2.0/styles) This should work with all problematic dark  themes in Firefox #########

style "ff" {
text[NORMAL] = "#000000"
base[NORMAL] = "#FFFFFF"
}
class "GtkInvisible" style "ff"
widget "GtkInvisible" style "ff"

style "custom" {
GtkWidget::cursor-color = "#000000"
GtkWidget::secondary-cursor-color = "#FF0000"
}
widget_class "*" style "custom"

###### End of Firefox fix ##########

I don't know what the secondary cursor is, but you'll have a red one with these codes, naturally you can change them to whatever works.

I see a lot of good looking desktops this month. :)

I apologize up front if I drifted off topic but if this works as well for others as it does for me I'd like to make sure it spreads throughout the community.  If you did the userChrome.css fix you may need to remove or disable it for this to work properly.

* This may also be a global fix if it's inserted in the ~/.gtkrc file but I haven't tested that extensively yet.

Yes, the persistent (as in not fading in on mouseover) global menu is alive and well in Classic. You can check to see if you've already got it installed by super-alt-right-clicking on the panel, then clicking "Add to panel." It would be in your list as "Indicator Applet AppMenu."

```
sudo add-apt-repository ppa:jconti/gnome3
sudo apt-get update
sudo apt-get install indicator-applet-appmenu

### Post by roelforg on 2012-06-11
I attached my boot theme, it looks very good when you switch-out the standard lightdm unity-greeter login background out for the one used in the theme, this makes the transition very smooth (the name is kinda a pun).
This is how you install it (assuming it's saved as smooth_plymouth_theme.tar.gz in your homedir) (note: that's two dashes in front of --install on the second line):
```

sudo tar xzf smooth_plymouth_theme.tar.gz -C /lib/plymouth/themes/
sudo update-alternatives --install /lib/plymouth/themes/default.plymouth  default.plymouth /lib/plymouth/themes/smooth/smooth.plymouth 100

```

Now use (again, 2 dashes in --config):
```

sudo update-alternatives --config default.plymouth
sudo update-initramfs -u
sudo update-grub

```

Have fun.

### Post by cbowman57 on 2012-06-14
Don't know if you guys encounter this alot, but I used to when dealing with dark GTK themes, but have been meaning to share it.

Some dark themes have always been a problem with the text entry boxes in  Firefox, &/or invisible text cursors, I think this will cure it.

```
####  Firefox text fix goes inside the theme name/gtk-2.0/gtkrc (or an app  fix in the gtk-2.0/styles)  ####
####          This should work with all problematic dark  themes in Firefox #########

style "ff" {
text[NORMAL] = "#000000"
base[NORMAL] = "#FFFFFF"
}
class "GtkInvisible" style "ff"
widget "GtkInvisible" style "ff"

style "custom" {
GtkWidget::cursor-color = "#000000"
GtkWidget::secondary-cursor-color = "#FF0000"
}
widget_class "*" style "custom"

###### End of Firefox fix ##########
```I don't know what the secondary cursor is, but you'll have a red one with this. 
May also work in ~/.gtkrc but that hasn't been thoroughly tested.

> **Copper Bezel said:**
> It doesn't quite work that way - a window manager also controls focus states, listens for keyboard shortcuts, etc. Apparently it's **herbstuftwm**, though, which sounds very *close* to not having a wm. = )
Au contraire mon ami!  It is actually much more powerful at managing windows than wm's like metacity or clutter.  It is probably even more powerful at actually *managing windows* than compiz, though compiz does a lot too (just that most of what it doed has more to do with effects than managing windows; still, window rules in compiz are the best).

There are several beautiful things about herbstluftwm:

1.  It is a tiling window manager, so screen space is always used efficiently, and it is great for tasks that require mutiple working in multiple windows at once (basically everything except lesiur web browsing).

2.  Most popular tiling window managers (Awesome, dwm, xmonad, spectrwm [formerly scrotwm]) use *auto* tiling, which, while convinient, limits flexability for window layouts.  herbstluftwm requires a little more effort be put into window placement, but allows virtually endless possible layouts for whatever best suits the task.

3.  It is controlled entirely by shell commands, which means it can be scripted and made to talk to other applications.  The config file is actually just a bash script.  It's very unix-y in this way:  it does one thing well, and it uses text streams, so it interfaces well with other programs that behave in the same way.  Modularity and portability, so to speak.

4.  It was written by a guy who uses a multi-monitor setup, and it deals with that situation brilliantly.  Any workspace can be sent to any monitor independantly of the others, but it sticks them together from the perspective of the X server, so you can still mouse across them seemlessly.


However, you must read some of the man page and edit the config file before it will do much.  No fancy graphical tools (unless you count a GUI text editor :P), so it might be intimidating for someone not used to configuring their system via text files.  It's a great experience though; makes you feel very hackerish, and you're bound to learn something. I'll post some pictures of me doing actual work tomorrow, to demonstrate how the tiling can be used.

It also doesn't handle any desktop or window effects.  They can be handled by external programs (like cairo-compmgr), but I'm not a huge fan of desktop effects anyway.
