### Post by FoolsGold on 2007-04-02
> **SOULRiDER said:**
> Thats a great Conky script, could you please post it?
Sure thing:

```
# set to yes if you want Conky to be forked in the background
background no

# Create own window instead of using desktop (required in nautilus)
own_window yes
own_window_type override
own_window_transparent yes
own_window_hints undecorated,below,sticky,skip_taskbar,skip_pager

# Use double buffering (reduces flicker)
double_buffer yes

# fiddle with window
use_spacer yes

use_xft yes
xftfont Bitstream Vera Sans Mono:size=8
xftalpha 0.8 

# Update interval in seconds
update_interval 1.0

# Draw shades?
draw_shades yes

# Stippled borders?
stippled_borders 3

# border margins
border_margin 9

# border width
border_width 10

default_color white
own_window_colour brown
own_window_transparent yes

alignment top_right

# Gap between borders of screen and text
gap_x 30
gap_y 50

# stuff after 'TEXT' will be formatted on screen

TEXT
${color orange}SYSTEM ${hr 2}$color
$nodename - $sysname $kernel on $machine
${color lightgrey}Uptime:$color $uptime 

${color orange}CPU ${hr 2}$color
${color lightgrey}Case Temp: $color${i2c 9191-0290 temp 1}C ${color lightgrey}CPU Temp: $color${i2c 9191-0290 temp 2}C ${color lightgrey}CPU Fan: $color${i2c 9191-0290 fan 1}

${color lightgrey}Freq: $color${freq}MHz   ${color lightgrey}Load: $color${loadavg}
${color lightgrey}CPU Usage: $color$cpu% ${color #00aaff} ${cpubar}
${color lightblue}${cpugraph 35,0 ff0000 00ff00}

${color orange}PROCESSES ${hr 2}$color
${color lightgrey}Processes:$color $processes  ${color grey}Running:$color $running_processes

${color}Name              PID     CPU%   MEM%
${color #ddaa00} ${top name 1} ${top pid 1} ${top cpu 1} ${top mem 1}
${color lightgrey} ${top name 2} ${top pid 2} ${top cpu 2} ${top mem 2}
${color lightgrey} ${top name 3} ${top pid 3} ${top cpu 3} ${top mem 3}
${color lightgrey} ${top name 4} ${top pid 4} ${top cpu 4} ${top mem 4}

${color}Mem usage
${color #ddaa00} ${top_mem name 1} ${top_mem pid 1} ${top_mem cpu 1} ${top_mem mem 1}
${color lightgrey} ${top_mem name 2} ${top_mem pid 2} ${top_mem cpu 2} ${top_mem mem 2}
${color lightgrey} ${top_mem name 3} ${top_mem pid 3} ${top_mem cpu 3} ${top_mem mem 3}

${color orange}MEMORY ${hr 2}$color
${color lightgrey}RAM: $color$memperc%   ${membar 6}$color
${color lightgrey}RAM Usage:$color $mem / $memmax

${color lightgrey}Swap: $color$swapperc%   ${swapbar 6}$color
${color lightgrey}Swap Usage:$color $swap / $swapmax

${color orange}DISKS ${hr 2}$color
/           ${fs_used_perc /}%   ${fs_bar 6 /}$color 
Usage: $color${fs_used /} / ${fs_size /}
Free:  ${fs_free /}

/home       ${fs_used_perc /home}%   ${fs_bar 6 /home}$color
Usage: $color${fs_used /home} / ${fs_size /home}
Free:  ${fs_free /home}

/datadrive  ${fs_used_perc /datadrive}%   ${fs_bar 6 /datadrive}
Usage: $color${fs_used /datadrive} / ${fs_size /datadrive}
Free:  ${fs_free /datadrive}

${color orange}NETWORK (${addr eth0}) ${hr 2}$color
Down: $color${downspeedf eth0} kB/s ${alignr}Up: ${upspeedf eth0} kB/s
${downspeedgraph eth0 35,140 000000 ff0000} ${alignr}${upspeedgraph eth0 
35,140 000000 00ff00}$color
Total: ${totaldown eth0} ${alignr}Total: ${totalup eth0}

${color #ffff00}Port(s)${alignr}#Connections$color
Inbound: ${tcp_portmon 1 32767 count} Outbound: ${tcp_portmon 32768 
61000 count}${alignr}Total: ${tcp_portmon 1 65535 count}

${color orange}FORTUNE ${hr 2}$color
${execi 120 fortune -s| fold -w45}

you should find that most icon themes work well with dark themes (especially if the theme is all grey or all black).
rather than getting the icon theme to match the wallpaper, get it to match either the background of the theme or the selected elements of it. for example, if you have a theme which is of a white background and the selected parts are lightish blue, the tango icons may well work well. see screenshot 1. afterall, you're going to be seeing the icons set against the theme more than set against the wallpaper.
for dark themes, i think that the buuf icons tend to go well with most dark themes. try them out and see for yourself. they can actually look really nice with their overall dark shades.
for light themes where i can't match the icons to a colour element(either the background or selected items) of the theme, i tend to choose white-ish icons such as whatever2, micro,  snowish, etc.

go to your .gtk-2.0 file in your home directory. then insert the following code in (change the colour to whatever you want):

```
style "menu-item" {

 fg[NORMAL] = "#ffffff" # CHANGES THE TEXT COLOUR TO WHITE

}
class "*MenuItem*" style "menu-item"
```
then type the following on the commandline to refresh the panel and the menu:
[B]killall gnome-panel

### Post by m.musashi on 2007-04-23
Okay, I have to share this.

I just learned how to make my desktop annimated. I'm using a screensaver as my wallpaper. I doubt I'd leave this on for more than a few minutes but the fact that you can is really cool. Unfortunately a screen shot is a poor way to demonstrate so do this:
```
gconftool-2 -t bool /apps/nautilus/preferences/show_desktop -s false
nice /usr/lib/xscreensaver/lattice -root --circuits \ -n -s -e 1 -f -o 30 -x 20
```
Basically it stops nautilus from drawing the desktop and sets up xscreensaver to. In my case this didn't work until I took the "nice" part out. Of course that caused the CPU to do a lot of work. Anyway, thought it was cool. The screen shots were taken a few second apart so you can get an idea of "movement"

EDIT: Oh and to put everything back, kill it and do this
```
gconftool-2 -t bool /apps/nautilus/preferences/show_desktop -s true
```
and then hit ctrl-alt-backspace and log back in.

EDIT2: btw, that came from a new book on Ubuntu called "Hacking Ubuntu". It just came out. Pretty good so far.

### Post by Phatfiddler on 2007-04-29
> **Horizon said:**
> You mean like synaptic? You mean like /usr ? :p

No, not like synaptic.  They are called PBIs and BSD also has ports.  PBIs are similar to .deb files except they have a graphical installation like windows install files, and also include dependancies.  Ports is like apt-get except it configures and compiles the program(s) based on your system configurations and settings.  An equivalent to synaptic would be bpm (bsd port manager).

BSD uses /usr as well.  "Programs" is a link to /usr/Programs to aid in finding installed programs for new users.

### Post by kvonb on 2007-04-29
PCBSD is absolutely fantastic, I used it back in it's 1.0 days and it has just gotten better and better.

But, BSD is different to Linux, it is more like pure Unix, some things are done differently which is not a bad thing in my opinion.

I don't use it for my desktops because it is not as flexible or as modifiable as Linux, and unfortunately there are not many options when it comes to games and things like scanners/webcams/printers etc'.

I'm not bagging it, far from it, I would love to run it on ALL my systems including my server, as I find that networking is a lot easier under xBSD distros because I cut my teeth on FreeBSD back in 1997/98.

Greg Lehey has done a marvellous job with PCBSD, taking BSD from being mega complicated and awkward, to being extremely simple and a pleasure to use.
