It is a shame.

Gimp is clearly farther along, but I think this has more potential.
(Specifically for users switching from Photoshop)

Like I said, when I tried it out, I did not have to think at all, I just did what I normally do in Photoshop, and it all worked out the same.
(Minus actions, and guides)

Also, Gimp still does not support CMYK, which is 100% necessary for anyone who plans to print what they are designing.
(And it does not look like there will ever be plans to add cmyk support)

And no, I am not referring to exporting a cmyk file using an obscure plug-in.

You need to actually be able to view and edit cmyk images.

RGB is made of light.

CMYK is for ink.

Since you cannot possibly print light, and many shades and colors made with RGB cannot be duplicated using CMYK, it is impossible to guesstimate how the colors of a printed piece will turn out, if you design it in RGB.

So if you ever want to design something professionally, for the purpose of printing, Gimp is not, and never will be an option.

Aside from Pixel Studio, the only programs I have ever seen that fully support CMYK, are Adobe (Photoshop, Illustrator, Acrobat, Etc)

So without a program like Pixel, if you want to design something for printing, and you care how the colors turn out, you have no choice but Adobe.

Which is how Adobe can charge what they charge.  (CMYK Monopoly)

Not sure if the source is available, or even if tampering would be allowed.

But it would be a great Idea for someone to grab the "Pixel Studio Ball" and get it running.  (If such a thing is possible)

Otherwise, there is no professional way for anyone using linux to create designs for the purpose of printing.  (Aside from Photoshop in Wine)

Which means, linux will probably never have a place for professional graphic designers.

Shame.
