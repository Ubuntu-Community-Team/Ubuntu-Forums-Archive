### Post by Ioky on 2010-04-29
All in ONE or Do ONE thing well?

I think this question should be one of "those" question Developer should always ask themselves before they develop any Application. As an Designer myself. sometime it very hard to define the line. 

For example. Image browser use to be just browsing image. but now day's many "image browser" come with image management, and base image editing feature. Things come very handy. but often time, those side feature is too "weak" to serve your needs. and they ending up sitting there and do nothing, except slows things down. Often time. you ending up have two difference software that is doing exactly the same things, but each one of them have one or two features that doesn't exist on the other software. 

But if you have software that only doing one thing. you might ending up opening like 3 or 4 software just to got one job done. Like you want to pick a image and send it to your friends, and you need to edit it just a little bit.

If you are the developer which philosophy you are going to follow. Making god like app that does everything. or making Doing one thing well app?

It can be neither. but what do you think?

I personally think, if there is a good way to allow each Doing one thing well software to better communicate with each others. It would be best. Therefore I come up with something call the "Object Oriented Operating Environment" also known as O3E, which is basically like Object Oriented Programming. Even Object serve a function, and only do one thing. If the user want, they could just link to function together. and setting their own tool. Without any programming needed. Anyway, that is just what I think. and if any of you want. I can post the detail here. It is just a little bit too long.

But yeah.. what is your option?

### Post by antenna on 2010-04-29
I much prefer do one thing well.  

If I have an app that is primarily an image viewer, I don't want it to be able to edit images (poorly).

If I have an app for playing music I don't want it to be able to rip or burn cds.  I also don't want it to sync with anything.  I can use a file manager for that.  Not surprising then that I can't stand iTunes. 

Thankfully a majority of software on Linux tends to follow the one thing well, though perhaps a little less these days.
