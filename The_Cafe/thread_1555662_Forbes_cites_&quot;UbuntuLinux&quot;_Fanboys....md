There are few things what most people do not know:

**a)** Linux kernel is the operating system (Linux is not microkernel but *a monolithic*).

**b)** Monolithic kernel is the *first and original operating system architecture*. After a decade or so, there came new ones: Server-Client and Layered architectures. They were designed to meet new needs what Monolithic architecture could not offer in theory. But monolithic architecture is still widely used, even that most Unix or Unix-clones have started to use Server-Client architecture. 

**c)** RMS thinks that operating system is the installation media, at that time programs were distributed with the OS in magnetic tape (magtape). To use the program, you needed to change the tape. It was more like CD-R's with LiveCD's. You could not get all the programs in the one tape but to get different programs, you needed to use different tapes. Think a case where multimedia programs are at one LiveCD and office programs at another.

**d)** At the beginning (60-70's) the software what we now call OS, had multiple names. They were a: supervisor, master program, core, controller, control program, nucleos and *kernel*. In the beginning, most used term was the kernel. But the term kernel, was not anymore valid because the new architectures. The OS overthrown the kernel because the software what build the OS in first original architecture, was monolithic, a single binary. But with new architectures, the OS was multiple binaries, a tiny microkernel and servers what hosted all programs, everyone with one OS tasks, like filesystems, networking, I/O, memory management and so on. While monolithic includes all the OS functions in it. (there has, and there alwas will be a depate is monolithic a bad choise for the OS). Todays monolithic OS's can be modular as well, but the modularity is in binary level and not in the architecture level.

**e)** For RMS, the GNU utils and developlment tools are more important than the application programs itself, like email-reader, news-reader, web-browser, multimedia-player and so on.
It toke 3 years from RMS to start demanding the Linux would be renamed as GNU/Linux. The problem was that people talked about OS named Linux. But RMS saw that people referred the software system as Linux, where most softwares indeed were from GNU. But for RMS the software system = operating system, while for rest of the world the OS was just the Linux kernel.
But RMS used to depate a idealogical, not technical facts. He said that compiler, text editor, Clibrary or Shell were more important to user than anything else, like Linux and because user did not use Linux, it was not OS. Just totally skipped the fact that all the GNU softwares (excluding HURD/GRUB, in the sense of / meaning "or" and not "and") did not work *without OS*

**f)** Many does not know that HURD is not the kernel, but the operating system. HURD can use two microkernels, GNU Mach or L4. The microkernel is most important part of the operating system and hardest to code. The servers are more like a normal programs (but not easy to code either). But RMS (or GNU) does not want to talk about the microkernel what HURD use, there are ideas to talk about HURD as it would be kernel in the sense of the monolithic architecture, what HURD is not, in contrary, HURD is a Server-Client OS. 

**g)** GNU people even falsified  the UNIX standards to give them a reason to call Linux as "just a kernel" and GNU/Linux as a OS.
The uname programs purpose is to print the OS name, the CPU architecture and so on. While uname program what follows UNIX standard, prints the OS name correctly and tells that the monolithic kernel is the OS... and does not mention at all about kernels.
The GNU version of the uname have been modified and altered that it does presents real OS as a kernel by just changing the "operating system" to "kernel", and adding a non-standard -o option what prints the "operating system name", what is non-existing in standard version. This was done in silence in long period time (might been few years). Non-GNU uname programs prints the OS name correctly, but GNU version of uname does not, just to give the misinformation.
