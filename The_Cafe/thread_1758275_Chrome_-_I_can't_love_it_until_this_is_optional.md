### Post by nrundy on 2011-05-15
This dropdown window from the Omnibar can be handy for finding bookmarks. However, Chromium's implementation of it is lousy.

It works much much better in Firefox. You can set firefox to only report bookmarks in this dropdown. Type in letters or a word from the "URL name" or the "Bookmark's name" and it will be listed in the dropdown window. For example, if you create a bookmark for ubuntu.com and then type "U" in the firefox URL bar it will show the bookmark. Tap tab and hit enter and you are there = a lot easier than using the mouse to find the bookmark. It can all be done from the keyboard.

For years, I disabled the dropdown window in firefox. But once I tried it out I found it was a lot easier to find bookmarks this way. I also didn't have to take time trying to find where I put bookmarks (what folder etc). I would suggest trying this out in Firefox.

Like I said, Chromium's implementation of this feature is terrible. It's weird because you'd think Google would be good at searching. But Firefox has it down beautifully and it will simplify and make web browsing and calling up bookmarks more efficient and require less effort.

"Forgets active logins on close": make sure you don't have Chrome set to delete cookies upon closing. A more secure way to deal with this though is to use the extension "LastPass" with a strong password and set the sites you want to auto-login. This way you don't have to type in the username and password for that site. Leaving cookies on your drive can be a security hole. Cookies can be hijacked and siphoned by attackers.

It won't fix all your problems, but you might try running in Incognito by default (add the --incognito flag to your exe file). This will auto-delete history, cache etc. Make sure you delete ALL items before engaging the flag and it will also prevent auto-complete from adding text to what you type in the omnibar.
