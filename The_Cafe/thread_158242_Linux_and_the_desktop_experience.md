**Video editing and 3d rendering on linux.**
We all know successful video editing requires alot of harddrive space and alot of ram many would suggest a scsi 
for a harddrive, now this leads me onto talk about filesysytems why do we use a file system that is generic by this 
I mean on linux we have a choice and each file system handles files differently for example the SGI XFS is superb for video editing and 3d rendering 
as it is the most scable filesystem around.
It supports files up to 9 extabytes and remember we aren't even dealing with peta bytes yet which proves how advanced it is.
Linux supports such a file system and has fantastic support for raid and scsi.
SGI have moved away from their own unix to linux as they see how powerful it is when it comes to such tasks.
Two programs that have taken 3d rendering on linux to another height have been the famous opensource blender and the well known closed source application known as maya.
Maya is consider to be one of the industries standards

Networking with linux [/B]
General support for networking has always been a big thing for unix based systems such as linux, bsd..etc
It offers support that windows users can only dream about. (Mac Os has firewalling features built in thanks to BSD.)
This is both useful for home users (peer 2 peer traffic) and corporations looking to secure their network.
