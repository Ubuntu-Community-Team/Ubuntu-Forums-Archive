Not a bad idea.  For some reason I think I'd prefer to have things flipped left/right (the workflow and upload buttons in the upper-left, and the minimize/maximize/close buttons back up in the upper-right) but that's not exactly relevant.

I only skimmed the page you linked to, but is the intent to have the workflow button close the current application and open the next one?  So if you have your image open in Inkscape and you click "Open in GIMP" Inkscape would close (releasing any R/W locks it might have on the file), and then GIMP would open?  Or would you export a copy somewhere, and GIMP would work on the copy?  I think closing Inkscape makes more sense to me; you're done one step, and moving on to the next.

You could maintain some kind of a workflow stack so at any point you can go back to a previous program you've used in your current workflow.  So if you're working in GIMP you'd have a shortcut that says something like "Go back to Inkscape", which would simply to back to the last item in the workflow, as well as the standard list of options.

I'm assuming the workflow applications would somehow be hooked up to the file's MIME type, so whatever applications Nautilus lists in the standard "Open File With..." list would appear in the workflow list?

In OS X we have this in the form of something called a "proxy icon" (situated just to the left of the window title), which i think is a better (but a bit similar) concept. You can drag it to an application of your choice (yes, it's draggable), or better yet, drag it to any location to copy (or link) it there. It also provides a "breadcrumb path" (which can be opened with the file manager) to the file in question on right clicking it.
