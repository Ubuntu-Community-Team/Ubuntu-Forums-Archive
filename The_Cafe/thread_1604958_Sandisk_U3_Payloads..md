### Post by ironic.demise on 2010-10-24
The U3 Drive released by Sandisk (and long since abandoned, Windows bought them out to try and create some Windows memory stick that lets you create files, bring settings with you and generally take your user account everywhere... something Sandisk had already succeeded at and then MS dropped it aswell).

The U3 drive allows you to install and use programs without installing them on the host computer and without changing anything on the computer.

The U3 drive does this by having two partitions on the memory stick, one emulates a cd drive and the other contains your personal files.

Some people hacked the CD partition so you could add your own files and replace the U3 default "launchpad", this could be used to make your USB run like a CD.

There were several "payloads" created that would rip as much information of the host PC onto the USB and create certain settings on the host, invisibly meaning you could get the host to email you sensitive data.
AVs cannot delete the cd files as it's a read-only partition... no matter what you do in most cases.

I have one myself, I've tried things here and there, all on my own computer.

Two questions:
Does anybody know how a similar thing could be achieved on Ubuntu (Autorun.inf and a .sh file) but I was hoping for a "lunchpad" in Ubuntu.

Does anybody have access to recent payloads?
