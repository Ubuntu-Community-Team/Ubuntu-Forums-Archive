### Post by kelvin.illa on 2009-09-24
[B]What we know:

[/B]Firefox 3.5's address bar is called the "Awesome bar"
[U]
The search bar and the Awesome (address) bar in comparison with the unified Google Chrome search bar:[/U]
- Chrome takes less space; looks tidier
- Chrome's unified bar is easier to access; only one item to select
+ Firefox's awesome bar has the Google "I'm feeling lucky!" search function, Chrome doesn't (Chrome can only 'search')

_The Awesome bar alone:_
- Though it has regular search, it prioritizes the "I'm feeling lucky!" search over regular search. Type "firefox wiki" in the Awesome bar, and it can only go straight to Firefox's Wikipedia page.
- doesn't have the additional functions of the regular search e.g. "define:" 

Those above are the only... (what do you call them?)... *'pros and cons' *of a unified search bar and address bar I could think of right now.

**My take:**

I think the unified bar (by Chrome) is better overall than the individual search and address bar with one exception: it doesn't have Firefox address bar's "I'm feeling lucky!" search function. I think this could be solved by assigning a useless key-press to toggle between search and address bar functionality. 

_To illustrate:_
The default setting for the unified bar would be the Awesome bar. To toggle to search mode, the space-bar (I think this button is best since I don't think there's even one reason for anyone to search for " ") should be pressed once, creating a one-character gap between the wall of the bar and the first character of your input. The bar will now recognize input after that one 'space' even if the succeeding character is another 'space.' From here, the user can now use the "define: " function of the regular search mode. 

This is how that would look like: 
[[ define: dictionary                                         ]] <-- the unified bar; notice the space between '[' and 'd'

 To toggle back to the Awesome bar, simply remove that single 'space.'

[B]What do you think? If you're thinking that you don't understand what I'm getting to in the first place, please say so too; I'll make my words better.

[/B]BTW, I'm also for 'tabs on the title-bar when maximized'--it saves space.
