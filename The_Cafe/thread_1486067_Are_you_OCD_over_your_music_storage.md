### Post by julio_cortez on 2010-05-18
Well, I may not be this OCD over my music, but I like to keep it in order.
I use three particular conventions:

**1)** For the CDs I make (mostly house music, mixed ones) I use to simply save a mp3 file of the entire length of the mix (around 75-80 mins) and put it in a folder named **[yy-mm-dd] Place - Occasion of the mix**.

**2)** For the playlists I make (with various songs) I use to keep both the files and the playlist in my **My Music** folder. Every music file has to be *.mp3 (at least 160 kbps) and tagged with artist and title. Nothing else.

**3)** For the entire albums, I have a **Music** folder in my external storage drive. The folders and files are organized with a hierarchy like **Artist\[year] Album\Track_no - Title** and even in this case, every music file has to be *.mp3 (at least 160 kbps).
Tags are derived from the folder/file structure via mp3tag which lets me specify a tag like "*%artist%\%album%\%trackno% - %title%*" (I usually did it under Windows).

The only thing (regarding **case 1 of my list**) I need now is:
is there a software (to be installed via apt-get possibly) that can rip an audio CD in a *.mp3 file PLUS a cue sheet?
And if so, is there a music player (to be installed via apt-get, too) that can actually READ this cue sheet and treat the said *.mp3 file as a "CD with splitted tracks (but possibly without silence between them)" instead of a single file?
If so, I'd be very happy because I could store all my mixes as *.mp3 files, and then play them exactly as they were CDs..
