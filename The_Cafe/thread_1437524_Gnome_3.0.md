### Post by uljanow on 2010-03-24
hi,
these days there seems to be a controversy over window button placement, but if GNOME 3.0 will be part of Ubuntu 10.10 there will be a whole new UI paradigm and the "old way" of doing things will be deprecated by Developers of GNOME. Take for example the GNOME shell:

*Debian Package Description:*
> The GNOME Shell redefines user interactions with the GNOME desktop. In particular, it offers new paradigms for launching applications, accessing documents, and organizing open windows in GNOME.** The GNOME Shell is intended to replace functions handled by the GNOME Panel and by the window manager in previous versions of GNOME.**

*live.gnome.org:*
> Some of us believe that it would be a good thing to keep **providing the old elements for a limited time, to ease the migration.** That being said, doing that would obviously take some development resources and slow down work on what should be the future.


First of all, I do love my current GNOME 2 with it's clean and coherent interface and design, but changing decades-old UI paradigms is just not acceptable.
