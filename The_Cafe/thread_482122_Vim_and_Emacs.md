### Post by arvevans on 2007-06-25
In the bad old days we used VI and Emacs for programing, but today there a number of Integrated Development Environment (IDE) applications that have their own built-in editors.

Learning rudimentary Vi (vim) commands may still be a good idea because most of the CLI (Command Line Interface) functions work much like the Vi command set.

Emacs has a built-in program execution capability that allows it to be used as basis for develop0ment of scripted applications.  If you are doing extensive shell script based application programming, then this might be useful to know.

Do you have to know either of these to make effective use of Linux?  No, but they are sometimes handy and are interesting from a historical viewpoint.

### Post by Kelsin on 2007-06-26
> **blah blah blah said:**
> Or maybe they're isn't much advantage to them.

If you use emacs or vim like notepad of course you're not going to see the advantage.  Anytime you are working with text using a text editor (I prefer emacs) is going to help. Just the fact that these programs offer search that doesn't have to bring up a dialog makes them 100x more useful then any editor that does. The fact that everything in emacs can be included in a macro. Commands can be repeated x number of times with a simple command. Search and replace can use regular expressions and matching. You can move around a file by searching, sentences, paragraphs, lines, characters, you name it. Bookmarks, completion, integration with cvs/svn, dired mode in emacs (file management), transparent remote editing with tramp. The list goes on. 

If you ever think that there is not a large advantage to using a proper text editor when editing any type of text then something just hasn't clicked for you yet and I urge you to keep working on it until it does.

Here is a great example:
[http://derekslager.com/blog/posts/2007/02/screencast-formatting-a-css-file-with-emacs.ashx](http://derekslager.com/blog/posts/2007/02/screencast-formatting-a-css-file-with-emacs.ashx)

That is a screencast using basic emacs. He has a css-mode on for highlighting but doesn't use any of the features. He formats a horribly formatted (made for demonstration purposes) css file using just plane emacs commands. Mainly macros involving search and replaces and then repeated calls on those macros. It's incredible. Try doing that in notepad, or textmate, or even eclipse and jedit. It's just not possible. If you're argument is that some of these editors have auto-formatters for css so who cares, again you've missed the point.

To the OP: I don't use emacs for email since I run my email in gmail. I havn't spent the time to downloading the proper plugins for Firefox to use emacs as my text field editor but probably will do that soon. I do use emacs for word processing using latex. I was a CS student so using formulas and the ability to search (using grep on the command line, or in emacs) was extremely useful for me. I use emacs as my calculator (using calc mode, I love how search and macros work in calculator just like they work in every other mode) and I use it daily at my current job to remotely edit files on servers. It's a great tool for working with any type of text. I can't argue against using vim either. Both are incredible and as far as text editing goes, the best tools you can use. It's worth it to learn both slightly, and definitely worth it to try and master one.

(Edit) one more point that I would like to add: I mentioned a lot of different features of emacs up above (dired mode, cvs/svn etc). I know a lot of IDE's have those. But the power is that all of emac's text editing power works in every mode. I can use the same movement, search, macro, bookmark, custom functions in every mode and feature. I can rename groups of files using the same search and replace that I can with files. I can browse diffs between cvs revisions with the same commands I use to browse files. The incredibly simple base that emacs has (mainly a lisp engine) provides amazing power.

### Post by cdupree on 2007-07-09
Personally I'd call Emacs a necessity.  I rarely go a day without using it, unless I'm away from my computers altogether.  It's one of three pieces of software I have an emotional attachment to (not counting stuff I wrote), because it's so 

[list]
[*] stable
[*] capable
[*] fast
[*] well documented
[*] consistent
[*] designed for fast typists
[/list]

Non-Emacs fans consider Emacs hard to learn, but I didn't find it so.  There's a tutorial that will get you through the basic fifteen or twenty commands, and teach you how to look up the rest.  The Info tree is very helpful, the Apropos command finds more information than you want.  You can interact with the file system (create directories, delete, copy, and rename files, etc.) and do other command-line activities.  

I'm completely addicted to the query-replace function, which has about ten possible replies including recursive edit ("Replace this?" "Yes, yes, no, yes... Woops, hold on, I gotta do something special here. ... Okay, done, resume."), as well as keyboard macros ("Beginning of line, search for next 'XXX', back one word, forward one letter, delete the next two words, end of line, add a closing paren, down one line"; now do that sixteen more times); Emacs's ability to understand language structures, whether C++ or SGML or Perl; and lots of other stuff.

It's not for those who require nifty graphics in their interfaces; but it is for people who write lots of text and don't want to move their hands off the keyboard if at all possible.  As Neal Stephenson [says](http://artlung.com/smorgasborg/C_R_Y_P_T_O_N_O_M_I_C_O_N.shtml):

> I use emacs, which might be thought of as a thermonuclear word processor. It was created by Richard Stallman; enough said. It is written in Lisp, which is the only computer language that is beautiful. It is colossal, and yet it only edits straight ASCII text files, which is to say, no fonts, no boldface, no underlining. In other words, the engineer-hours that, in the case of Microsoft Word, were devoted to features like mail merge, and the ability to embed feature-length motion pictures in corporate memoranda, were, in the case of emacs, focused with maniacal intensity on the deceptively simple-seeming problem of editing text. If you are a professional writer--i.e., if someone else is getting paid to worry about how your words are formatted and printed--emacs outshines all other editing software in approximately the same way that the noonday sun does the stars. It is not just bigger and brighter; it simply makes everything else vanish.

Of course that was eight years ago, he might differently now.  Some of what he said then has changed: Emacs now does fonts and faces of all types.  It interfaces with chess programs, compilers and debuggers, browsers, and anything someone writes the interface for.  And I think Stallman is becoming more controversial as time passes, a controversy in which I'm not currently taking a position.

I don't know VIM well enough to make comparisons there.  But I can tell you, my feeling about Emacs hasn't changed: you'll pry Emacs from my cold, dead fingers, or their virtual equivalent.
