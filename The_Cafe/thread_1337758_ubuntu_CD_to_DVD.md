### Post by MaxIBoy on 2009-11-25
What's the point of a DVD image?

DVDs are faster; if you can, you should use them instead of CDs. But I'm not convinced that the extra space is really needed for anything. Sure, liveCD environments are nice, but PuppyLinux uses, what, 200Mb, and it works great for that. My ideal install CD is a netinstall CD like you can get for Debian or Ubuntu. You don't waste time downloading a version of a program which is already out-of-date, then downloading a new one. 

I think there are scripts to combine many CD images into a signle multiboot ISO, and that would be a very good use for a liveDVD. In addition, PuppyLinux can use up extra space on a liveDVD to save files when you shut down, a very handy feature.

LiveDVDs should be used for things like that, not for tacking on extra software which needs to be updated anyway. Live images are nice for installing (where you need a package manager, net drivers, a kernel, and that's it,) rescue CDs (where you also need the GNU utilities and some other knick-knacks,) and trialing a distro (where a premade VBox image is nicer anyway.)
