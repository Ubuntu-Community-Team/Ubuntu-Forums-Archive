### Post by Wolki on 2006-05-13
[QUOTE=Sheinar]CTRL+Tab works fine for me.[/QUOTE]

Ctrl-tab will actually work in a number of applications, I advise against using it though. Since most applications (at least on the standard ubuntu desktop) try to follow the Gnome HIG. this will not work for many applications. I reccomend using ctrl-pageup/ctrl-pagedown or, for direct tab-access, alt-number, these should work with any application (including gaim).

Here's the explanation. Gnome has a focus on accessibility, and it's important that everything that can be done, can be done without using a pointer device (ie mouse). Accelerators and Shortcuts are part of this,  as well as having a key that allows you to pass the focus between different parts of a window (like different entry fields). The key that does this is the tab key. This presents us with a problem, however: Some parts of the window must be able to accept tabs, for example to display them (think text editors/word processors), or because the widgets allow tab-completion. This means we need yet another key that allows us to pass focus to the next widget, and in gnome this key is ctrl-tab. This obviously means that applications that try to be accessible in this sense can't use this combination for saomething else.

Also, KDE uses ctrl-tab to switch between workspaces, so it likely wouldn't work there either.


[edit} nearly forgot it: about the tab size and scrolling, yes this can be quite annoying; As an Epiphany user (which works in a similar way) I have the experience that this method has some advantages though, and that you can often get used to it quite quickly. Tabs that are out of your view suck, but tabs that are so small you don't have any clue what's in there suck just as much if not more, you're just more used to it. Also, don't forget that you can use the scroll wheel in such cases, and that you can use more than one window.
