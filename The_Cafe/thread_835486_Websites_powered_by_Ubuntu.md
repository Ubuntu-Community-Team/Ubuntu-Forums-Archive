If you're hosting it under the same IP as you're viewing it on, your router may catch that and send you to [http://localhost](http://localhost) instead. It still should sent the server ID but that might be why.

It shows up as > Apache/2.2.8 (Ubuntu) PHP/5.2.6-0.dotdeb.1 with Suhosin-Patch for me, so don't worry.

If you want all the headers from your website, install [httpliveheaders]("https://addons.mozilla.org/en-US/firefox/addon/3829") (a Firefox add-on) or install 'curl':
```
sudo aptitude install curl
```
and run this command:
```
curl --head matthewboston.com
```
> $ curl --head matthewboston.com
HTTP/1.1 200 OK

Last-Modified: Thu, 12 Feb 2009 23:14:45 GMT
