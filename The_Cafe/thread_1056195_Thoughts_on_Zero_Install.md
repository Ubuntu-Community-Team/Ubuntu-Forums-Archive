### Post by Polygon on 2009-01-31
the reason we dont have a centralized system for packaging is because linux is SOURCE compatible, not BINARY compatible.

windows is binary compatible. A program can be compiled on computer A and run on any computer that runs windows (if its the same cpu architecture)

however

if i compile a program on ubuntu, and give it to a person who uses opensuse, that program *might* not run, even if our cpu architectures are the same. 

So even if there was a central packaging system, companies would have to make separate packages for each distro anyway.

its better to just let the package maintainers of each distro package the programs for their respective distro, and companies like nvidia or adobe or whatever just give the source (hardy har har) out on the internet and its all done for them.
