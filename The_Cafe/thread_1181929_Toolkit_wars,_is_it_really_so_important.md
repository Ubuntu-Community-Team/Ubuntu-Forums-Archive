### Post by gnomeuser on 2009-06-08
It is important to me for the following reasons:

1) Consistency across the desktop, apps should look the same, act the same. A KDE app can look on the surface every bit as much as my GTK theme but it will never be a GNOME app, it will never be consistent with my desktop.

2) Performance, whenever I need to load an alternative toolkit it means more ram is sucked up, load times increase and I can do less with the resources I have available.

3) Having a defined platform, if we want lots of long tail type software developed or just to attract new developers we need to define a standard for what Linux is and what APIs to program for. It doesn't mean ending diversity just have a sane default to attract people with.

I am pretty sure web applications will change the scene, but in the world we live in now it matters.

### Post by Kazade on 2009-06-09
Here's my take on the whole thing. If a company wishes to write a cross-platform application, they should treat KDE and GNOME like different operating systems, just like they would for OSX and Windows. It's not just the toolkit that matters, it's the human interface guidelines also. The problem is that people treat "Linux" as an OS, which is just crazy when you think about it because as we all know Linux is just the kernel. So when someone is targeting "Linux" it's almost as vague as targeting "PCs". It's not surprising that there is no standard GUI toolkit across multiple operating systems. You might as well complain about all platforms not using Win32. 

Personally, if I was a company developing cross-platform software trying to hit the largest audience, I'd likely target Ubuntu (the most widespread Linux-based desktop OS) which means GNOME (as it's the most widespread desktop shell) and so would therefore use GTK+. 

In that case, I would choose to use wxWidgets because that will give me native widgets on Windows/OSX/GNOME saving a tonne of work. It would be nice if someone wrote a QT4 backend to wxWidgets, then KDE users would be happy; you'd just need to customize the menus/keyboard shortcuts etc. per-desktop.
