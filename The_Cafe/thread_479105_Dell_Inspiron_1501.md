### Post by twright on 2007-10-24
hi
the restricted driver is slow, the only reasonable way to do it is with ndiswrapper, try typing this into terminal:
```
sudo -i

apt-get update
apt-get install build-essential linux-headers-`uname -r`
wget http://easynews.dl.sourceforge.net/sourceforge/ndiswrapper/ndiswrapper-1.48.tar.gz
wget ftp://ftp.us.dell.com/network/R151517.EXE; 
mkdir .drivers
mv ndiswrapper-1.48.tar.gz .drivers
mv R1*.EXE .drivers
cd .drivers;
tar -xzvf ndiswrapper*.tar.gz

echo blacklist bcm43xx >> /etc/modprobe.d/blacklist
echo ndiswrapper >> /etc/modules

***restart now***

sudo -i

cd .drivers/ndiswrapper-1.48
make uninstall
make uninstall
make uninstall
make
make install
cd /root/.drivers
unzip -a R1*.EXE; cd DRIVER
rm /etc/ndiswrapper/bcmwl5/*.*
rmdir /etc/ndiswrapper/bcmwl5/

ndiswrapper -i bcmwl5.inf

***almost done***

test it:
ndiswrapper -l

it should display this:

bcmwl5 : driver installed

        device (14E4:4311) present (alternate driver: bcm43xx)

***finish it off***
ndiswrapper -m
modprobe ndiswrapper
```64bit vista comes on the same disk as 32bit vista, no manufacturer ships it by default as unlike 64bit ubuntu it is not backward compatable, the only antimalware you can get for it is trend micro (not nice) and it may actually run slower than 32bit as all of the drivers are second rate

the only way to get the brightness working is to switch back to bios version 1.7 (you will need windows or freedos to do so)
