### Post by John Phoenix on 2009-05-16
Sorry but i didn't find any place on the forum or the website to request features, so i'm posting this here in hopes that the Ubuntu fairies  will guide this message where it needs to go.

Ubuntu prides itself on being a very good desktop environment. Of course the top to compete with is microsofts and I suspect Ubuntu pulls a lot of windows users away. Well there is a feature in every windows from 98 to windows 7 that would be so simple and easy to do.. plus all windows users who switch to Ubunto would really miss this feature. Linux users who don't have this feature don't know what they are missing. Well perhaps some do.

I am talking about moving the file and the folder together like windows does for html files. In windows when you copy just the html file to a folder drive etc, the folder that belongs to it always gets copied with it. In unbutu if I want to save a web page from firefox then copy it to a usb drive, the file gets put on the desktop first along with its folder of the same name that has all the extra data in it.. imagaes and such for the web page. Then I have to copy the html file to the usb and then copy its folder to the usb, each in a seperate step. That way when I open the web page offlne I will have all the imaages in place.

But what happens now? if you copy the html file and forget to copy the folder, when you open the page all you see is the text and a bunch of ugly blocks where the images should be.

These days a lot of things you read on web pages are actually images of documents. If you then open the html file without the folder with the images, you will have cut off your chances of reading what you wanted to save in the first place.

Keeping the html file and it's folder together when ever you copy and past or drag and drop just make the most sence. This ensures your offline copies of the webpages are always complete.
