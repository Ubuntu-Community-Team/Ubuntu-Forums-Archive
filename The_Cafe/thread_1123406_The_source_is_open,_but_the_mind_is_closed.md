### Post by Methuselah on 2009-04-13
Closed source can exist.
People don't *have* use it.

Closed source software is an inherent security risk because it's not so easy to audit what the hell it's doing.
In fact, the makers might even tell you that it's against the EULA for you to reverse engineer it.
That's why not a few governments are favouring open source software for their critical military and civilian software.
In fact, it's my understanding that for some types of contracts even Microsoft has to release source code (under extensive Non-disclosure agreements of course).

So the question of why I should pay to allow arbitrary, undisclosed code to run on my computer is indeed one worth asking.
Especially since some commercial software has gotten into the nasty habit of installing rootkit-like DRM schemes that don't even pay you the courtesy or removing themselves when you uninstall the program.
Ironically, the closed source PC software I'm most likely to install is reputable software that is free as in beer.
Not just because I'm cheap, but because I know that if they're giving it away they're not going to invest in invasive copy protection.

Also, I think part of the reason viruses flourish so much on the windows platform is:

1. People have to pay for everything so they're more likely to be duped by 'free' downloads
2. There is a culture of allowing arbitrary binary code to run with full privilege.

So yeah, it's much easier for someone to make a quick buck selling binaries.
But I question how often it's better for computer users.
And it's definitely within any customer's right to make a usage decision based on whether source is available or not.
If I had stuck to my principles with the NVidia binary driver I probably wouldn't have a screwed up file system right now.

Closed source:
Disadvantages

Open Source:
Disadvantages:
1. Harder to get vendor device info so drivers, etc can be implemented.

2. Good software can die out, or become obsolete, very quickly when the developers quit/die/move to other projects, etc.

3. Difficulty reverse-engineering propritary software functions a la MS Office (which won't be as much of a problem when the Wine developers finish the MS Office support.)

Advantages:
1. When a flaw is found, it's fixed pretty quickly.  Those fixes are reviewed, improved if possible, and sometimes even applied to similar situations in other code.

2. Anybody can work on it. Each new user, if they wish, is a new developer -- even if only submitting bug-reports (to which there are many people to help fix the problem).
