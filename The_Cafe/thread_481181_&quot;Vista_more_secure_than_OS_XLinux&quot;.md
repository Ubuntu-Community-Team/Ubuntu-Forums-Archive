### Post by prizrak on 2007-06-22
Didn't Vista have a pretty serious hole in it as soon as it came out? Something to do with the way it handles animated mouse cursors I believe. To make matters worse the entire Windows line had the same bug starting from like 95. 

Most security reports are pretty pointless either way. You can either go by amount of bugs vs time it takes to fix. Considering that some of those bugs are obscure they can't really be used pretty much makes that an idiotic approach. For example in Breezy I believe there was a bug where the sudo pwd was saved in a log file. While technically a security vulnerability it could only be exploited by someone who has access to the machine already and at a level high enough to view system logs. Dunno about you guys but I wasn't really worried.

Then there is the severity and number of bugs vs time it takes to fix approach. That's better but also doesn't work all that well. BeOS could have 3million critical vulnerabilities but no one is going to give a damn because no one is targetting it. 

In effect the only real measure of vulnerabilities is statistics on what machines get compromised, how often, in what environment (if you have a protected network and still get viruses that's a bit different from getting them when you are fully exposed) and how easy it would be to protect against it.
