### Post by NiceGuy on 2006-09-14
**First, a disclaimer: This post is a response to the many posts/blog entries to be found on these forums and across the net where the user has has a hard time installing Linux and so has a rant about how bad Linux is and gives up. This post is not designed to start a flame war. My hope is that if someone who is having problems they don't just give up and go back to windows, but instead ask for help on the forums, read the wiki's etc. and generally stick with it because you might not have such an easy time installing windows either!**

One of the biggest complaints I hear about Ubuntu (and Linux in general) is how difficult it is to install compared to windows. I'd like to dispel that myth right now!

Please bear in mind that I'm in no way new to the world of windows and I actually count my self to be a bit of an expert as I spend most of my time fixing it when its borked/infected with virus and malware etc. (its what I do for a living).  

Also bear in mind that this was a relatively 'trouble free' install.


10.30am - Work begins! I assemble the PC. This is the easy bit and went pretty much without a hitch (I did have trouble with some of the packaging though :p ).

1.00pm - PC is built. All basic checks done, BIOS auto detect has picked up everything its supposed to. Great, time for lunch.

1.30pm - Right now the 'fun' begins. I pop in the windows CD and boot the pc from it. I choose the standard option of use entire drive for primary partition, format it as NTFS and then install windows.

2.30pm - First bit done, enter locale data. Everything seem normal.

3.30pm - And oh my gosh we are actually there, Enter user name etc. etc. 'wow - that was easy'

3.35pm - hurm... no network connection to activate windows? No network interface either... I know - it'll need drivers for the network card, forgot about that (Ubuntu has spoilt me!)

3.40pm - uh oh! Driver installation program has crashed. Ah well I'll just reboot and try again.

3.45pm - No its done it again... hurm... *checks cd for burn marks etc.* nope all good on the cd front. Lets have a look at the documentation on the cd, see if it gives any clues.

3.50pm - No good, nothing like my problem there. BUT WAIT! My main (and only) partition is not c:\ as expected but f:\ !!!! I wonder...

4.00pm - It's no good I can't change the drive letter of the windows partition :( Looks like I'll have to reinstall the os.

6.00pm - Oh good! Its decided to use the correct drive letter this time, and lo and behold the drivers install! 

Ooo. Now that's pretty cool - this particular mobo comes with its own hardware based firewall (its an nforce mobo if you are wandering).  Install, reboot.

Next sound drivers. Done. reboot. What next - oh graphics drivers - install, reboot. Monitor drivers – install,reboot. Yey - a decent screen resolution.

6.30 What's this annoying bubble saying? No I don't have anti-virus yet, give me a chance! Oh aright I'll Activate windows. Done. Now lets go get a browser which isn't one big security flaw. Right firefox. Downloaded. Installing...error. Oh this just isn't my day! Must be a corrupt download, I'll download it again. Installing... Nope still no good :( 

Google!... hurm it seems to be that nforce firewall. It seems the version I've got corrupts all downloads. Oh thats nice isn't it! Turn of firewall. Go to nvidia website. Download new version of drivers etc. Install them... oh it seems I need to un-install the old version of the firewall before I can install the new one. Un-install. Reboot.

Now lets try that again... install done, reboot.

Now lets re-download firefox - third time lucky. Yey it works. Now lets get some anti-virus, AVG... download... install... reboot... scan. Done. I'll just set it to download OpenOffice and Thunderbird and start it defraging  (I can't believe its fragmented already!) whilst I get some dinner as its 7.45 already.

7.45pm - Have dinner.

8.15pm - I'm back. Install Thunderbird... done. Install OpenOffice... done. Now lets do all those updates it keeps nagging me about...download, install, reboot... download, install, reboot... download, install, download, install, reboot... download, install, reboot... download, install, reboot... download, install, reboot... And done. Phew - I'm glad the install cd was SP2 or and not SP1 or lower or I'd have been here all night!

9.00pm - Sooo what's next. Printer - install, reboot. DVD software – install, reboot. Nero – install, reboot. Acrobat – install,reboot. Other assorted software – install, reboot., install, reboot, install, reboot.

10.00pm – And done, just set it defraging again.... So that's over 8 ½ hours to install windows and required/common software. Well it was less time than I was expecting and only about 4 times longer than Ubuntu. Sigh.

So there you have it – still think windows is easier to install?

ps. Ok, ok. So it was a bit of a rant but hey!

### Post by the lush on 2006-09-15
I started using Ubuntu a few weeks ago as a completely new user, I had tried to install Red Hat a few years ago, but couldn't work out a way to use my monitor with it, so after a few days of trying things found on the Internet I gave up. Luckily Ubuntu has far more supportive people, as well as plenty of inexperienced folk like me asking the same questions, and of course Linux is far better now than it was back then. In my opinion the actual installation of Ubuntu 6.06 using the live CD is mostly excellent, but it is what comes after that that caused me to vote for Windows. A few examples:

1. Why do my screen resolution options only go up to 1024x768? I still haven't been able to correct this on one of the systems I installed. Please, in the next version include higher resolutions in the default installation.

2. Why do I have to use the command line pppoeconf command to set-up my ADSL connection? Only having one machine at home meant having to log-out of Ubuntu, boot into Windows, search the Internet for the solution, return to Ubuntu, and mess about with the terminal. This function should be included in 'System>Administration>Networking' with the tools to set-up other network protocols. Way too complicated considering the majority of home users have to use PPPOE to connect to the Internet. Worse still, I can't get Automatix (an absolute essential in my opinion) to work correctly when using PPPOE - I have now found some instructions that I will try tonight which claim to solve the problem by disabling IPV6 and editing some conf files using the blasted terminal - again too complicated. Furthermore, if I didn't have access to other machines at work, how would I ever get the answers I need? Using the System Documentation and searching for PPPOE (and how many completely new users would even know to search for that) I find:

> 4.3.5.2.&#8195;ADSL Modems

All PPPOE and router-style ADSL modems are supported by Ubuntu (that use Ethernet for the connections), and some USB ADSL modems are supported too. For router-style ADSL modems, just connect it up, configure the modem as per your ISPs instructions and configure networking in Ubuntu. For information on PPPOE modems see this guide on the Ubuntu wiki.

How would I look at the Ubuntu Wiki without Internet access? I also found this less than useful tip:

> NAME

pppoeconf — configures a PPPoE (ADSL) connection 
SYNOPSIS


pppoeconf 
DESCRIPTION


The  
pppoeconf program is userfriendly dialog based setup 
tool for pppoe and pppd. It will 
look for existing ethernet cards and look for ADSL hardware connected to 
one of them. Then it will get some login info and do some minor 
modification to make working settings.

Right... Luckily I had already been using Ubuntu for a bit at work, so I guessed that I should open a terminal and type 'pppoeconf', but most completely new users would have no idea that they should do this. 

3. Why does my external hard drive mount sometimes when I boot the system, yet other times it claims that as it belongs to root I am not allowed to mount it? This is inconsistent. The shell scripts in Automatix allow me to get around this without using the damn terminal/editing files, but the inconsistency between installations means that I never know how long installation will take. Note, this also applied to internal drives - my main office machine refused to Mount the other drives and partitions without tedious messing about, yet using the same installation media at home my home machine saw the internal drives and mounted them immediately.

4. The first time I installed to a laptop, running at 1680x1450 on an ATI X700, lead to black screen. Reading forums I found that this was caused by the driver outputting the main display through the VGA port. Solution? Download the alternate installation CD, edit xorg.conf, cross fingers, hope for the best. It worked, but was a lot more complicated than installing Windows on the same machine.

5. The second time I installed to the same laptop, and using the Live CD, even though I told Ubuntu to format the drive (after a disastrous attempt to install XGL/Compiz) the resolution was perfect the first time! Why did it not work the first time? I am obviously not the only user to have experienced the problem (just search for Acer Ferrari Ubuntu Display to find hundreds of posts devoted to the issue). Is it a problem with Ubuntu's ability to detect hardware? 

6. Mice! Really, this one is crazy. The mouse is a pretty standard part of any computer set-up, at least for those of use who use GUI's, yet it was more fiddling about with the command prompt to set-up a Bluetooth mouse, and although the commands were simple, the ease of doing this does not even compare to setting up a Bluetooth mouse using the Windows GUI based application. Worse than this though was that I had to use the terminal just to get the side buttons on my wired mouse to work. This is just silly, I would not describe installation as complete when even the mouse buttons don't work.

7. Wireless networking: Works perfectly as soon as the Windows installer has completed, yet so far I have had to concede defeat on ever getting it running under Ubuntu. 

8. All praise to Automatix! Yep, I am not an experienced user, but I want to play MP3's, watch movies, use Flash and Java, as well as find my installed programs easily (the Debian Menu thing is essential). The only problem is, I can't use it due to PPPOE. Until I am able to listen to music, and waste time on Youtube, I do not consider installation to be complete. While it is true that Windows does not include Flash or Java by default, the web installers are childsplay compared to the Linux solutions.

9. OK, I give you OpenOffice, it is great that it is installed by default. Much better than waiting to install MS Office, but I think the antitrust courts would have a field day if MS built Office into Windows.

10. Just an idea, Suse offer a non-oss disk to download for use with their distribution. It would be really great if Ubuntu did this too. If I hadn't found Automatix I would have given up within the first week of trying Ubuntu. Having all of the stuff that Automatix installs on an extra CD would be great.

I could add many more points to this list, but this post is far too long. I am persevering, but I could never recommend Ubuntu to people with less computing experience than myself at this time. It seems that the first requirement of any user is that they read the entirety of the System Documentation before they even begin, and even then it will take many hours to achieve the same level of functionality from their machine as they would have immediately after the Windows install. Perhaps the guys and girls in charge need to hire a few HCI/Usability folk, and do some testing with novice users to identify the issues that they have.

### Post by kripkenstein on 2006-09-15
I think that the main difference between Ubuntu and Windows is the *type* of installation. Generally, people do a simple install of Windows, overwriting the entire hard drive (mostly after some problem with the installation, spyware etc.). But with Ubuntu, I think that many people do a dual-boot installation, that is they have a pre-existing Windows installation that they want to dual-boot with. And this is a bit trickier, since you have partitioning and so forth.

Not *much* trickier, but it was a bit of a hassle for me, the first time I did it.

When doing a basic installation, no dual-booting, I think Ubuntu is easier to install than Windows. The **post**-installation may be harder or easier, depending on the level of hardware support; on desktops I find Ubuntu to be easier (since I often need to look for Windows XP drivers on the web); on laptops, Linux in general is still a bit of a pain.
