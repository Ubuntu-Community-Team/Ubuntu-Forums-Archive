### Post by Namtabmai on 2007-10-31
Every time a Linux distro releases a new version we get a load of blogs/report/posts asking the same question "Is Linux ready for the desktop". While some do actual take the time to evaluate Linux as a different OS most really just review Linux as a clone of Windows, and ultimately for these people Linux will never be ready because it's not a clone of Windows and never will be. Using a similar methodology I could say that Windows isn't ready for the desktop because it's not a clone of Linux, but that's certainly not the case.
Anyway this post isn't about if Linux is ready for the desktop or not, so please don't let this thread descend into that discussion. There a thousands/millions of happy Linux users. 
But where Linux really fails at the moment is actually being recognised as a signification portion of computer users operating systems. There are various projects out there that try and count the number of Linux users, but they are all guesses really as they all are opt-in programs. Other distros try and count users by downloads but this isn't that accurate either for so many reasons.

What we, as the Linux community, need is a method of counting the number of users with little or no intervention from the user themselves.

The Fedora project manage this by providing active statistics from their yum package manager.
[http://fedoraproject.org/wiki/Statistics](http://fedoraproject.org/wiki/Statistics)
Now this still isn't 100% accurate, due to web proxies, people running multiple machines, people running servers etc, but it is a much better method than those mentioned above. Currently the number of Fedora users according to this is around the 2 million mark.

If we can convince other Linux distributions to provide similar information based on their package management system we could start to see a clearer picture. Of course we could never account for those small distros but if we could at least get the bigger names to provide information such as this we could at least have a figure that would give the magnitude of Linux users. e.g. 5 million? 10/20/30 million?

So what about Ubuntu?
Given the nature of apt, Ubuntu could provide similar information, but it does raise some questions.

1) How many of you actually 'apt update'/'apt upgrade' regularly?
2) Does the idea of Ubunut/Canonical using their apt server logs worry you? If so why?

Again, this thread isn't asking "Is Linux ready for the desktop", I'm simply trying to find away to provide an accurate-ish number on the amount of Linux users. I obviously don't expect the number of Linux users to even come close to that of Windows users. BUT how close are the number of Linux users to the number of Mac users? Apple estimated the number of OS X users at around 22 million last year, and this seems enough users for companies to actively develop software and hardware drivers for them.

If we can show from reliable sources that the amount of Linux users is close or even more than OS X we stand a better chance of convincing various companies that Linux should be taken more seriously.

[sidenote]
Currently there only vague figures floating around regarding the number of Linux users. The LinuxCounter project guesses at 29 million. Ubuntu claimed previous that there where around 8 million users, and Fedora shows around 2 million. Distrowatch claims that Ubuntu accounts for about 30% of the Linux market and Fedora around 4%. Using these figures you could estimate the Linux user base anywhere between 26-50, but with the exception of the Fedora number there is nothing to backup any of these other figures.

### Post by kebes on 2007-10-31
I'm also curious about the number of Linux users, and so I've accumulated some stats about this issue (thanks for the link to Fedora stats, which I've added to my list!)... Here's my list of estimates:

1. The [Linux Counter]("http://counter.li.org/estimates.php") estimated 29 million installs in 2005. This estimate involved numerous assumptions, such as extrapolating from 8 million installs reported by Red Hat in 1998.
2. There are [about 1 billion Internet users]("http://www.internetworldstats.com/stats.htm"). Browser logs indicate that Linux accounts for [~0.8% to ~3.4% of web traffic]("http://en.wikipedia.org/wiki/Usage_share_of_desktop_operating_systems"). This gives us an estimate of 8 million to 34 million Linux users. (The upper estimate is undoubtedly an over-estimate since the value comes from [W3Schools]("http://www.w3schools.com/browsers/browsers_os.asp"), which probably has a greater fraction of 'technical' users.)
3. According to Canonical's server logs from OS updates, there are approximately 6 million active users of Ubuntu (see [here]("http://www.linux.com/feature/119865") and [here]("http://coldfusion.sys-con.com/read/446538.htm")). Assuming that Ubuntu represents 30% of Linux usage (based on [this]("http://www.desktoplinux.com/news/NS8454912761.html")), you can come up with an estimate of 20 million Linux users.
4. [According to Fedora's logs for OS updates]("http://fedoraproject.org/wiki/Statistics"), there are approximately 2.8 million installations of Fedora 6, and 1.6 million of Fedora core 7. Assuming Fedora represents 9% of Linux installs (again, based on [this]("http://www.desktoplinux.com/news/NS8454912761.html")), you can estimate 48 million Linux users.

Obviously all of these methods have their own problems. None are perfectly robust. However they do at least suggest a range for the number of Linux users (~20 million) and the marketshare of Linux (~1% to 2%).



As you can see, the Canonical estimate of 6 million users is also based on repository access (although unfortunately they don't provide a nice stats page like Fedora). I agree that it would be interesting if each distro submitted monthly (anonymized) stats reports about the number of unique IPs that accessed their repositories for updates... the collection of all this data could provide a fairly robust indication of the number of active installations.

### Post by aysiu on 2007-10-31
Because for businesses, it's not just about raw numbers of users. It's about potential customers.

Stigmas, perceptions, and prejudices all play into this.

Big businesses tend to have, along with the general public, a misconception about Linux users that they pirate software or avoid purchasing commercial software. To these big businesses, it won't matter if you prove that Linux users number as many as 50 million. If those Linux users are happy dual booting or using Wine or Cedega for Windows-only software, the software companies have little financial incentive to port to Linux. If Linux users would prefer to pay less for a Windows-preinstalled computer on which they'll later install Linux, OEMs (and their suppliers) have little financial incentive to preinstall Linux on their main computers. Yes, Dell has preinstalled Ubuntu on a couple of models in a few countries, but the page isn't prominently displayed, and they still "recommend Windows Vista" all over the place.

The best way to twist the arm of big business (gaming companies, other software companies, hardware manufacturers) is to vote with our wallets. If we buy only Linux-compatible hardware and buy only software with native Linux ports or full Wine compatibility, then big businesses will see a *tangible* number: "Hey, company X that supports Linux is making more money than company Y that doesn't support Linux." That's a much better sales pitch than "Hey, do you know that 30 million Linux users access our Free software repositories every month?"
