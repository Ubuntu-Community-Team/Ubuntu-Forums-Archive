### Post by iceportal on 2007-06-23
My answer in short: Hell yes.

My answer in not-so-short: Hell yes. I have always loved the work Adobe has done with their software. Photoshop is an amazing tool, despite it's outrageous price. Granted, the Gimp is nice, as is Krita, but truth be told, if I had the money for it, and it were available, I'd dive in and buy CS3 for Linux.

My decision is based on a number of reasons. First of all, Photoshop is the industry standard. It's the best thing out there. And for the graphics professional, nothing beats it. The Gimp is great, but in the long run, doesn't come close. Krita is on its way, but has yet to match Photoshop. It is, quite simply, the best.

Granted, if a friend of mine was inquiring about photo editing software on a budget, the first words out of my mouth would be "get the Gimp." And if they were using linux, I'd suggest Krita as well. I love Open Source, and I support it wholeheartedly. If they're an average user, and would be happy with something as simple as Photoshop Elements, I'd suggest they save their money and get a FOSS alternative.

But as all graphics professionals know, Photoshop is the way to go.

My second reason for diving in and purchasing the Linux version of Adobe CS3 would be that it would send a clear message to the software world: there IS a Linux market for your software, and we would be happy to have your support. If a software giant like Adobe adopted Linux support, many more would follow. And I would be more than happy to see all of that wonderful software available for Linux.

Yes, I support Open Source. But I also support the freedom of choice. And I believe that more users would choose Linux if they knew their favorite software ran on it. The biggest reason people haven't switched to Linux isn't a fear of using a new OS (especially when the differences are minimal). The biggest fear is that they'll have to change all the software they use. Games, Photoshop, Office software... There are so many tools people NEED that just aren't supported in Linux. And if suddenly support DID exist, there'd be no turning back. Linux would explode with new users.

So, despite the fact that there are Open Source alternatives (which deserve our support very much), I must say I would gladly adopt Photoshop and other Adobe products for Linux, despite their proprietary, commercial nature.

### Post by geoken on 2008-04-21
> **Dragonbite said:**
> Can I make the SVG in something like Inkscape and pull it in (will it translate or be too much work to change)?

You can. That's what I do. SVG is a natively supported format so you don't need to translate anything, the flex compiler handles the task of translating the SVG into a Flash object. With that said, the official docs say that the compiler only supports a subset of the full SVG spec. 

I personally haven't run into any problems, the compiler has been able to pull in any SVG I've thrown at it. At the same time, I usually only use SVG's for the simpler graphical assets (ie. header bars, tooltip boxes, page backgrounds, scrollbars, etc.). I pretty much keep it to things that I know I'll have to scale. When it comes to more complex graphical assets (ie. icons, really detailed widgets, etc.) I usually export them from inkscape as png's. The main reason for this is because your apps run a lot faster when you're using bitmap graphical assets rather than vector based graphical assets. My point is that although I've never had any problem importing an SVG, but I've also never attempted to import an extremely complex SVG.


//edit
I just did a bit more digging and this;

*Flex supports a subset of the SVG 1.1 specification to let you import static, two-dimensional scalable vector graphics. This includes support for basic SVG document structure, Cascading Style Sheets (CSS) styling, transformations, paths, basic shapes, and colors, and a subset of text, painting, gradients, and fonts. Flex does not support SVG animation, scripting, or interactivity with the imported SVG image.*

This pertains the the Flex 2 compiler. We're currently on Flex 3, so my inability to find a current list of supported/unsupported parts of the SVG spec may be evidence that they now support all that stuff (with the exception of animation, ineractivity, etc.).

### Post by geoken on 2008-04-22
> **Dragonbite said:**
> Currently [[COLOR="Blue"]Pencil 2D[/COLOR]]("http://www.les-stooges.org/pascal/pencil/index.php?id=Home") Windows version (*there is an Ubuntu .deb  package available, but I haven't tried it yet*) has a export to SWF option.

EDIT: I just installed Pencil and it turns out it can export to an SWF file. I don't know if this is usable in Flex or not though.

Yeah, Flex can also make use of graphical assets pulled out of an SWF file. The problem with SWF files is that the graphical asset needs to be exported as a class and available for runtime sharing. These 2 things are easily done in the Flash IDE (simply click 2 checkboxes) but I'm not sure how other apps handle it. It may be easier to simply draw out the image in inkscape and save it out as an SVG. 

Flex's use of SVG's is an attempt to let people jump into Flex using their design app of choice. For that reason it tries hard to put few requirements on the SVG's it'll accept. In contrast to that, Flex's use of swf based assets is designed to allow Flash IDE users to build graphical assets for Flex. This means the swf files it accepts are very specific and must meet some strict criteria, making it less likely that other apps will be able to output the swf's it needs.
