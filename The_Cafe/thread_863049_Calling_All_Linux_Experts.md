### Post by MaxCarnage on 2008-07-28
> **lisati said:**
> The reason I said that it's a trick question is that when all the packets are forwarded (no matter what) there is no great security advantage of using a switch over a hub. The main difference between a hub, a switch and a router is one of *intelligence* - the more sophisticated devices are able to determine the destination address of a packet and only send it there, without the other ports being able to see them.

There is a distinct difference between hubs, switches, and routers. Hubs are layer 1 devices that forward data received in one port out all others. Switches are (generally) layer 2 devices that forward frames based on destination MAC addresses to a specific port. The only time a switch sends data to all ports is if it's a broadcast. Routers are layer 3 devices that perform packet switching and route determination.

It goes beyond a simple matter of "intelligence"; it's a matter of function and at what layer of the OSI model the device operates at. Switches don't bother with IP addresses, and routers don't care about MAC addresses. Hubs are physical devices that don't care about anything.

Sorry, I know this is not the point of this thread, but I felt a need to respond to this statement because I'm a geek.

I've always seen that "disclaimer" - that not all switches are layer 2 - but I never understood why. If it works with layer 3 information doesn't it stop being a switch?

Thanks!

All switching is layer 2; that is, if destination determination is being done with MAC address, then it is a layer 2 function. However, some switches have the ability to also perform layer 3 functions; that is, the destination is determined by IP address. These devices are usually called multi-layer switches.

It's actually less common nowadays to have pure layer 2/layer 3 devices. Most newer Cisco switches, for example, are multi-layer and are used for both switching and routing. So I can create a virtual layer 3 interface on a switch and as long as the network it's routing for is connected (usually a VLAN) it can now talk to other networks.
