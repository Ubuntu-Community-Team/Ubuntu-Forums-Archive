### Post by Eviltechie on 2009-06-08
I would love to switch my school to linux, (or at least part of it) but the more I think about it, the more impossible it seems.

Our school IT person has no gripes with linux. He would love to switch to linux just as much as the rest of you. He has a Cent OS box sitting in the floor of his office that runs Apache and stuff for the webmaster club.

Unfortunately there are some major problems. The school system just spent more $$$$$$$ on computers with windows XP. Naturally they want to get the most out of their investment. Software is also an issue. Every class that uses the computers (except programing) has their own propitiatory windows only app. The design students use the adobe suite, the drafting students use CAD, the teachers use grading programs, etc. (the programing classes use eclipse, so we're good there) The video and audio editing classes use macs, and I'm not worried about them too much.

Another big problem with software is that every single computer in the school has exactly the same software on it. Instead of a windows login screen, I get a novell login screen. Instead of a desktop, I get an application launcher. It's the same for all the computers. That means I could go to any computer in the school and get to photoshop, flash, or cad, no matter what room it's in. They accomplish this with some novell application distribution zen works thingamabob.

Let's not forget file sharing. Every student's documents are stored on a server. I can get to my documents from any computer in the school.

And here's the best part, it's like this on every computer in the entire school system.


I was originally thinking that the best way to introduce something like this would be to just have linux on one computer lab in the library. But if Joe Student can't access power point, or Mary Teacher can't access her grading software, what do you do? No matter how good it is, wine isn't an option here. It just doesn't scale.

Any suggestions from the linux people here, or is my school to deeply entrenched in windows.

### Post by Ocxic on 2009-06-08
> *Whats the problem...?*

Adobe suite------->**Gimp + BlueMarine + Inkscape + Scribus + XaraLX**
CAD--------------->**BricksCAD**
Eclipse----------->**Eclipse**
grading programs------->**Moodle**
Macs--------------->**Macs**
Flash------------>**Flash4Linux**
Server----------->[B]Ubuntu Server LTS
[/B]The problem is "Industry" does not use open source software, graphic designers use the Adobe suite, not gimp, etc...  A drafts person will use AutoCAD not BricksCAD,, and it's not that these programs wouldn't do the job, but that the job uses Adobe, AutoCAD, etc.. Employers and people in industry don't care that you can use GIMP to retouch a photo, you will need to use PhotoShop because that is what they use, and what you are givin to use. An employer will not install GIMP for you to use, just because you don't use photoshop.
The schools use the software it does not because it's the best per-se but because that is what the industry uses, and you being able to use those applications, is more important then the school saving some money with open source apps. 

When Windows specific Apps are in use, you really can't do much unless you want each computer to run a Virtualbox setup overtop a light wieght linux install. 
While this would not be the most practical way of setting up a computer system, it does allow for the system admin to create snapshots of systems(great for testing), aswell as make it easier to re-image/repair a system if something goes a-rye.

Try and look for computers that don't require the special software, if students only need, say the library computers to do research and type out papers, and can get to the design/CAD computer lab for those projects, then the library computer could run a linux setup with minimal problems.
