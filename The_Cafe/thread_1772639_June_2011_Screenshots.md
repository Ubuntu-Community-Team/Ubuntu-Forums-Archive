### Post by Copper Bezel on 2011-06-01
[quote=Giant Speck]I've got AWN up and running. It was a hassle to get it to look like the Gnome-Panel setup I had in my last screenshot, but it looks fine now. I'm a bit annoyed at the complete lack of customization options for the "digital clock" applet and the fact I can't reposition applets as precisely as I can with Gnome-Panel.[/quote]

It looks good, though. I like the whitespace. AWN's positioning is theoretically better than Gnome Panel's in that it keeps the spacing uniform, and I appreciate that, but I do wish there was a spacer applet that would just take up X number of icon slots, particularly for layouts like yours. 

Clock applets in general aren't as customizable as I'd like, but the only thing I really want with AWN's is a way to get rid of the little AM / PM tag without using twenty-four-hour time. Like many things in AWN, it's really designed to work best with a fat 30px panel, but I'm comfortable with the look with 16px icons, myself. I was irritated to find that it's the one applet that actually uses AWN's "text" color from the theme tab.

I do like that, once you *find* them, the configuration options for the indicator applet and system tray are better than Gnome Panel's, in that you can do multiple rows or scaled-down icons, split out or turn off the Me Menu, etc. Better Session applet than Gnome Panel's, too, and I've taken to using the included Sound Menu instead of the one that comes with the Indicator Applet. One bit I don't much care for is that an empty Indicator Applet won't disappear, but leaves the last icon displayed there, which is irritating when I've put my Sound and Session menus at specified places in separate applets, then switched to XFCE's power applet, which appears in the notification area, so that Dropbox is the only thing in my regular Indicator Applet area and doesn't disappear when quit.

arzali and el_koraco, goddamn, I really like Elementary. So pretty and simple. And el_koraco, yeah, I never actually got around to putting together that concept on the feeling that it was perhaps a bit overworked, and I was really impressed with Slingshot as soon as I saw it in action. It's simpler than what I had in mind and just as functional, and it would be just as touch-friendly (it's almost identical to Blackberry OS's launcher, which I also found shockingly ergonomic.) Elementary is, in general, much more what I'd want out of a desktop than Shell or Unity.

 BTW I am really digging your bright theme.[/quote]
Very cool - I *love* the transparency and the little curls that blend the whole shape together. 

Yeah, I'm a big fan of dark themes, which I still think look more striking and attractive at first blush, but I like the clarity and lack of a sense of weight I get with Orta. I tried setting up my panels to look more like these Shell-style black overlays instead of matching the windows, but I went back to the more substantial-looking ivory theme I had set up for AWN - I really like how unified everything turned out to be.

Openbox is a light window manager you can run on its own (have a look at the somewhat outdated guide linked to in my signature), or as the window manager in a desktop environment like Gnome, KDE or XFCE. If you install Openbox, you can select a Gnome/Openbox or  KDE/Openbox session at your login screen to use Openbox as the window manager with Gnome or KDE. TO use it with XFCE, login using the XFCE session, open a terminal and type

openbox --replace &
```

This will replace Xf4win with Openbox. Log out, and tick the box "save session" to have Openbox load again as the window manager on your next login.

### Post by FlameReaper on 2011-06-15
Adding Amarok and Audacious to Conky to display the current playing song.

If neither of these two runs, the Now Playing sections do not appear.

Also, my current Conky, this is for my Openbox setups. Feel free to use it, but remember to match the variables with what your system uses.

Oh, by the way, I also enabled RGBA on the theme I'm using.

```

update_interval 1
total_run_times 0
net_avg_samples 1
cpu_avg_samples 1

imlib_cache_size 0
double_buffer yes
no_buffers yes

alignment top_left
background no
border_width 1
cpu_avg_samples 2

default_color black
default_outline_color black
default_shade_color black

draw_borders no
draw_graph_borders yes
draw_outline no
draw_shades no

gap_x 20
gap_y 180

minimum_size 450 5
maximum_width 600
net_avg_samples 2
no_buffers yes
out_to_console no

own_window yes
own_window_class Conky
own_window_type desktop

stippled_borders 0
update_interval 3.0
uppercase no
use_spacer no

use_xft yes
xftfont Kozuka Gothic Pro:size=10
own_window_transparent yes


TEXT
System uptime: $uptime
  Current CPU frequency: $freq (MHz) ($freq_g GHz)
  CPU Usage: $cpu% 
  RAM Usage: $mem/$memmax ($memperc%) 
  Swap: $swap/$swapmax ($swapperc%)

${execi 360 aptitude search "~U" | wc -l | tail}${font} updates(s)

Temperatures:
  ACPI - ${exec sensors | grep "temp1:" | cut -c 15-21}
  Core 1 - ${exec sensors | grep "Core 0:" | cut -c 15-21}
  Core 2 - ${exec sensors | grep "Core 1:" | cut -c 15-21}
  Core 3 - ${exec sensors | grep "Core 2:" | cut -c 15-21}
  Core 4 - ${exec sensors | grep "Core 0:" | cut -c 15-21}
  GPU - ${exec aticonfig --od-gettemperature | grep Temperature | cut -c 43-47}°C

GPU: ${exec aticonfig --od-gettemperature | grep Default | cut -c 19-52}
  GPU Clock: ${exec aticonfig --od-getclocks | grep "Current Clocks" | cut -c 32-34} MHz
  Mem Clock: ${exec aticonfig --od-getclocks | grep "Current Clocks" | cut -c 46-48} MHz

${if_running amarok}Now Playing:
  ${exec qdbus org.mpris.amarok /Player GetMetadata | awk '!/album/' | awk '/artist:/{print substr($0,9)}'} - ${exec qdbus org.mpris.amarok /Player GetMetadata | awk '/title:/{print substr($0,8)}'}
  on ${exec qdbus org.mpris.amarok /Player GetMetadata | awk '/album:/{print substr($0,8)}'}
  ${exec qdbus org.mpris.amarok /Player GetMetadata | awk '/audio-bitrate:/{print substr($0,16)}'}Hz, ${exec qdbus org.mpris.amarok /Player GetMetadata | awk '/audio-samplerate:/{print substr($0,19)}'}Kbps, Amarok ${exec amarok --version |awk '/Amarok:/{print substr($0,9)}'}
${else}${if_running audacious2}Now Playing:
  ${exec audtool2 current-song}
  on ${exec audtool2 current-song-frequency}Hz, ${exec audtool2 current-song-bitrate-kbps}Kbps
  Time: ${exec audtool2 current-song-output-length} | ${exec audtool2 current-song-length} on ${exec audtool2 version} 
  ${else}
  ${endif} 

```

Why doesn't gnome-power-manager work? I don't use it, but should be a simple matter of adding a "gnome-power-manager &" line to your .xinitrc. If it's some other problem that's preventing gnome-power-manager from working, then maybe you could try an alternative like xfce4-power-manager.
