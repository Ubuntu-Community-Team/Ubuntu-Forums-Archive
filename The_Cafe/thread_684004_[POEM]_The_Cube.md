### Post by fatbuttlarry on 2008-01-31
[[IMG]http://img156.imageshack.us/img156/4537/screenshotkn5.th.png[/IMG]]("http://ubuntuforums.org/attachment.php?attachmentid=58219&stc=1&d=1201858937")

See... there was this cube...
The cube had many sides, but we can only see two sides.
Lets call them "left" and "right".

In the "left" is a space to run an application.
This application is not designed to run in the cube.
If the cube had a choice, perhaps it would never run this application.

But it does.
And it runs it well.
And it runs it fast.

In fact, this cube can run this application faster than the space it was
originally designed for.
The cube must be powerful.

In the "right" you'll notice a site.
This site runs a space.
The application in the "left" was designed to run in this space.
We don't run the application in the space it was designed for.
The cube must be versatile.

Look around the space. The space is running inside an application.
The application is designed to run in the cube, but the application was not
intended for that space.
That must be a good application.
It was written for a good cube.
The cube must be scalable.

This space running in this application is a virtual space.
And this virtual space is controlling a remote place.
The cube must be recurseable.

I wonder if the remote place can see the cube?

:popcorn:
