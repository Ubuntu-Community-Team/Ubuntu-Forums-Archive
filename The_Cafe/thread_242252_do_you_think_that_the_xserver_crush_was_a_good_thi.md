### Post by Miguel on 2006-08-25
Updating the X server and breaking it in the process is never a good thing. Period.

However, I was shocked to see the number of people suggesting reinstalls. One thing that differentiates *NIX from that legacy operating system (keeps most ABI's since 95, is very tricky to get "focus follows the mouse", has 4 USB stacks, etc) is that it is perfectly usable without a mouse and nice graphics. You know, it's asking for a login.

Some people here think CLI=MS-DOS. Well, both are black and both interact via text and keyboard. But one of them has tab completion, wonderful scripting abilities, thousands of utils for manipulating archives, is multitasking and multiuser and the other is, well, MS-DOS. 

What do I suggest? The following: [list]
[*] Tell people that only kernel upgrades need a reboot. HAL also asks for a reboot, but I don't understand why a service restart wouldn't work.
[*] Tell people to hold xserver-* updates for a couple of days. If a new kernel fails, you still have the older entry in grub. If X fails, only decent apt knowledge will save you.
[*] Teach people decent apt knowledge (yeah, the one from the commandline).
[*] Write a troubleshooting guide, both in pdf format and in text format (just in case X breaks and you don't have a printer).
[*] Finally convince people that you need to try very hard to require a system reinstall.
[*] And last but not least be more careful with the updates that will left lots of people helpless if they break (i.e. networking, X server, gnome, KDE).
[/list]


I probably sound a bit too harsh up there. The truth is that I am disappointed with the flaw the dev did but, at the same time, I am disappointed to see so many reinstall suggestion.

*EDIT:* Once upon a time, a Debian update (unstable, IIRC) broke apt. THAT is a bad one. I personally don't know if dpkg will still work independently from apt but, if it doesnt, you are basically screwed. I have only been in linux for 2 years so I don't know if networking has ever been broken with an update in a major distro. But, again, it was a testing branch.
