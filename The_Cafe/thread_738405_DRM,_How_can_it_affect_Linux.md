### Post by ph1 on 2008-03-28
In my opinion, one of the most attractive things about computers is the flexibility--the user can define a great deal of what he/she needs and uses.  Software is all about malleability--you have a hardware platform that gradually improves, but radical changes to software are occurring all the time, which is what makes it one of most dynamic fields out there.

DRM kills the flexibility.  It forces me to use whatever media/file/program I have in only certain situations, only with certain software, only for certain purposes, and only if I've legally obtained it.  What if a schoolkid wants to use some music as background music for a video, for fun or for school?  If it's DRM protected, no can do.  What if my friend wants to give me a music track as a present?  If it's DRM protected, I can't play it.

This is one reason why I moved from Vista to open source.  Open source lets me prune what I don't need and add in what I do.  It lets anyone, anywhere see what's "inside" the program and thus improve or extend it.  Vista has tons of "features" that are more like "encumbrances" that I'll never use, and there's no way for me to affect them.  And, no matter file I have in Linux, there are certainly no barriers to how/when/for what I can use it.  

Down with DRM, long live Open Source.
