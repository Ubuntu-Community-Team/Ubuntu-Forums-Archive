### Post by viperdvman on 2012-05-02
Got a screenshot of another Live USB test run.

Full-size screenshots here:
[http://img.photobucket.com/albums/v245/viperdvman/Screenshots/ScreenshotAtlasX1.jpg](http://img.photobucket.com/albums/v245/viperdvman/Screenshots/ScreenshotAtlasX1.jpg)

This is a Debian-based distro called AtlasX ([link here]("http://www.atlasx.org/")), running the Enlightenment desktop environment. It was featured as being newly added to Distrowatch's waiting list last week. And being an Enlightenment-based distro, it naturally grabbed my attention. I like that comes with more pre-installed apps than Bodhi does, but that's really about it. I can customize it very much the same way I can customize Bodhi. It does run a lot of GNOME apps, including Nautilus, which leads me to believe that it's actually running Enlightenment as its window manager on top of GNOME 2. One problem, though, is that its multi-monitor capabilities absolutely suck. It detected and ran on one monitor, and ONLY one monitor... my secondary one *(reason for the screenshot's 1600x900 resolution)*. I hope Bodhi doesn't do that when I finally decide to install it to my hard drive.

Other than adding a couple of pre-installed apps to the iBar on the bottom, all I've done was change the wallpaper to my Pokemon "Eeveelutions" (custom-made by me). It's not bad if you're looking for a Debian-based Enlightenment rather than an Ubuntu-based (Bodhi) or Puppy-based (Macpup) Enlightenment distro.

### Post by zombifier25 on 2012-05-08
> **Prescilla said:**
> Hey there.
How do I add applets like that on my desktop?

You need to install conky:
```
sudo apt-get install conky
```
Or do so, the graphical way, via the Software Center.
However, if you're not an experienced user, **don't** use Conky. You'll have to deal with a lot of configuring, editing,... in order to set it up the way you want it.
(of course, if you ARE experienced, wait for him to give you the configs)

Instead, you can use screenlets:
```
sudo apt-get install screenlets screenlets-pack-all
```
(include screenlets-pack-all if you want all the screenlets, which will take up about 50MB)
Screenlets are pretty much like widgets in Windows Vista/7, just start it, add the screenlets, and drag and drop the screenlets to move them.
