```Here are more alternatives to try:
### Post by MG&amp;TL on 2013-01-12
> **Cheesemill said:**
> I've been thinking of doing exactly the same, I could do with a fresh installation. I'm still in two minds about my filesystem choice.

Would you recommend btrfs + compression and did you come across any issues? I do have proper backups :)

Yup, I'd recommend it. The Arch wiki isn't very clear on this, but when it tells you to edit the fstab, that's when you need to edit the mount flags. Add

```
compress=lzo
```onto the existing flags (you can use gzip I think instead). Here's my fstab:

```

# 
# /etc/fstab: static file system information
#
# <file system>                    <dir>        <type>        <options>                <dump><pass>
# /dev/sda2
UUID=7ead097b-5515-4ee4-8ed2-d87a8f5231b2    /             btrfs         rw,relatime,space_cache,compress=lzo    0 0

```NB: no *fsck.btrfs* yet, so to stop accidental non-boots by having an unexpected fsck, I set the sixth field to zero, to stop the boot process trying to check the disk. Not the most maintainable solution, but I hear btrfs is mostly self-healing anyway.

As for issues, it's been on my machine all of about four hours, but no problems so far! Just really zippy boot times.
