### Post by ZankerH on 2009-11-24
So I got tired of waiting for someone to hack together a decent, secure wave client that supports an offline mode and strong encryption, and I tried out Google's web client again.

-It's still slow
-It still lags like hell, especially in longer conversations
-Bigger waves still take forever to load
-As soon as there's more than two people in a wave, the lag is so bad you can't even see *yourself* typing in real-time

So yeah.

While I believe wave will revolutionise online communication, the following things have to happen first:

Things google promised would happen eventually:

-The protocol must be optimised, and made more stable
-A Free wave server has to be released so people can set up their own wave servers and not have to rely on Google's monopoly
-Google's implementation has to be opened to the general public. Like gmail did, this should create an enormous amount of publicity and lead to more people adopting it, and hopefully other companies that offer webmail also offering their implementation of wave

Things that might happen, and should before wave is taken seriously in any production environment:

-A secured client that doesn't require a browser and a constant connection
-A way to give authors of a wave more control (ie delete/move sub-waves, assign moderators, wiki-like revision control)
-Server-side encryption
