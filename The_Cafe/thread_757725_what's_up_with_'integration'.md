### Post by NyquistLimit on 2008-04-17
I think some of you are confusing application integration with GUI consistency. Consistency means having the same look and feel in all apps e.g. icons and widget layout. Its important but only a skin deep part of an OS.

Integration is how well applications work together e.g. dragging an album from the Rythmbox playlist into Brasero to burn an audio cd. Another good example is the way calendar events from Evolution show up in the calendar in the top right panel in Ubuntu.

Windows for example has pretty terrible integration with 3rd party apps. Try dragging a song from Windows Media Player into Nero to burn an audio cd, you cant. You have to find the file in file explorer and drag it in from there.

With all this in mind MacOS has amazing integrating, try dragging something onto an application icon and more than likely it will do something sensible with the thing that you dragged.

Integration makes our lives easier by making the way we use our computers more natural and seamless.

### Post by DoctorMO on 2008-04-17
Some claim that the ideas of compatibility (having a number of applications able to use the same data source) and integration are natural apposed to each other.

For instance, web pages are very compatible with lots of different browsers. But they are arms length, no integration with the desktop.

On the other side of the spectrum you have evolution which despite it's heavy integration into gnome is no more compatible with kde-pim than microsoft outlook.

I see the inability to do both as a failure, evolution should give up trying to control the data source. much the same way as browsers are starting to implement context using formats such as RDF which are hoped will give better context and control to web browsers to add feature integration with the desktop.
