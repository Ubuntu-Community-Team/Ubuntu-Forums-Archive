### Post by angryfirelord on 2008-10-18
> **ronnielsen1 said:**

I'd hate to spend months (not that it wouldn't do me good to learn something new) learning something and then have someone say why did you get a certification in that? Is Linux + a good one to start or are there other programs that would be better to learn if my end results are hopefully work?
If you don't have any certifications, the first one you should take is the A+ certification. It's basically a certification that says you are able to be proficient as a computer technician and almost everyone starts out there first.

[http://en.wikibooks.org/wiki/A%2B_Certification]("http://en.wikibooks.org/wiki/A%2B_Certification") 

From that point onward, it doesn't really matter where you go. If you want to use it for your career, then you should get a certification that applies to you in that field. For example, it doesn't do you much good to get a Linux certification when your business is running Windows Server for everything. But if you're looking for a more general purpose certification, Linux+ seems good enough, so I would recommend that.

Now, if you wanted to up the challenge, Red Hat's exams are very tough, but they'll definitely make you look good in that field. But be aware, you will need to allocate a lot of time for something like the RHCE.
