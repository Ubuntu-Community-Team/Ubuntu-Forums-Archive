### Post by Sublime Porte on 2009-07-04
> Briefly, what's the difference between Java and JavaScript?

Java is a cross-platform programming language developed by Sun Microsystems. It can also be used to make web applets, although it's not used for that much anymore.

Javascript was actually called ECMAScript, and was developed by Netscape, but because the name sounded too much like a common skin ailment (Eczema) they decided to make it sound a bit more tech-oriented, and since Java was all the rage at the time, they used that name, with the blessing of their corporate ally Sun. Basically they wanted to trademark the name of the scripting language, to exclude Microsoft, even though ECMAScript was supposed to be an open standard.

Pbpersson,

> Reportedly someone thought it would be nice to create a scripting language that could be used on web pages with a syntax similar to the Java language and then named it "Javascript" to reflect the fact that it is similar to Java.

Actually it shares very little in common with Java. The name doesn't really have much to do with any similarities between the languages. And originally it was called Mocha, and then LiveScript, it wasn't until later when the Sun/Netscape alliance came about that it was renamed Javascript.

> You can create desktop applications using Java but not with Javascript.

Not true. Javascript can be used to write desktop applications too. In fact if you mix it with XUL for GUI building, you can use xulrunner, to build Mozilla-style apps, with javascript performing pretty much all of the work of the app.

Also projects like Sun's [Lively Kernel](http://research.sun.com/projects/lively/), fully written in Javascript, or [EyeOS](http://en.wikipedia.org/wiki/Eyeos) certainly blur the lines between desktop and browser, the latter being a complete operating system, albeit one restricted to the browser.
