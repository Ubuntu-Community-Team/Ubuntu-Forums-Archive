### Post by gutterslob on 2011-02-26
> **Azren0 said:**
> Hey, I've tried a number of times to get this font pack working under Ubuntu 10.10 XFCE with no luck, any suggestions?I'm assuming you're doing the thing where you unzip the archive and drag it to your ~/.fonts folder? It probably won't work like that.

I'm not on Ubuntu, but on my Debian boxes, I just extract the "artwiz" folder to my /usr/share/fonts/X11 or /usr/X11R6/lib/X11/fonts (you'll probably need root permissions). You're probably more comfy with Thunar since it's Xfce, so make sure you open Thunar as root to move the folder.

After that, fire up your terminal and go root. For *buntu, it should be;
```
sudo su
```Enter your password.

Then renew and reconfigure your font cache with these 2 commands
```
fc-cache -f -v[FONT=monospace]
[/FONT]dpkg-reconfigure fontconfig
```... and you should be able to use the fonts now (you might need to log out and back in first, though ... not 100% sure)

Remember, I do this for my Debian installs, so the font paths might not be 100% the same for Ubuntu.

Edit: Goes without saying that you should inspect the contents of the artwizx folder before you move them as root.

Edit2: Googling "Artwiz Ubuntu" led me to this page on Urukrama's site. It's for Hardy Hardwood, but should be the same for Lolita Lempicka or Masturbating Mosquito or whatever version you're using. Seems you'll need to reconfigure your fontconfig as well.
[http://urukrama.wordpress.com/2008/05/25/artwiz-fonts-on-ubuntu-hardy/](http://urukrama.wordpress.com/2008/05/25/artwiz-fonts-on-ubuntu-hardy/)
Use this instead of my guide. Urukrama's obviously more reliable than I am.

I'm not on Ubuntu, but on my Debian boxes, I just extract the "artwiz" folder to my /usr/share/fonts/X11 or /usr/X11R6/lib/X11/fonts (you'll probably need root permissions). You're probably more comfy with Thunar since it's Xfce, so make sure you open Thunar as root to move the folder.

After that, fire up your terminal and go root. For *buntu, it should be;
```
sudo su
```Enter your password.

Then renew and reconfigure your font cache with these 2 commands
```
fc-cache -f -v[FONT=monospace]
[/FONT]dpkg-reconfigure fontconfig
```... and you should be able to use the fonts now (you might need to log out and back in first, though ... not 100% sure)

Remember, I do this for my Debian installs, so the font paths might not be 100% the same for Ubuntu.

Edit: Goes without saying that you should inspect the contents of the artwizx folder before you move them as root.

Edit2: Googling "Artwiz Ubuntu" led me to this page on Urukrama's site. It's for Hardy Hardwood, but should be the same for Lolita Lempicka or Masturbating Mosquito or whatever version you're using. Seems you'll need to reconfigure your fontconfig as well.
[http://urukrama.wordpress.com/2008/05/25/artwiz-fonts-on-ubuntu-hardy/](http://urukrama.wordpress.com/2008/05/25/artwiz-fonts-on-ubuntu-hardy/)
Use this instead of my guide. Urukrama's obviously more reliable than I am.

Thanks for the suggestions Gutterslob, Ive followed Urukrama's tutorial before posting here and didn't have any joy; turns out the steps are correct but you are not able to enable the use of bitmap fonts (sudo dpkg reconfigure fontconfig-config). 

After a little more digging around I [discovered]("http://demonic.cc/?p=21") that you can do this manually by removing the "no-bitmaps.conf" located within /etc/fonts/conf.d and replacing it with "yes-bitmaps.conf" located in /etc/fonts/conf.avail then all you need to do is restart x and your good to go.

Now that you have conky installed, 
You can start it simply by typing conky into terminal, and pressing enter.

but you can change it to look any way you want by changing the script.

Simply download this~ [http://gnome-look.org/content/show.php/Conky+lua?content=139024](http://gnome-look.org/content/show.php/Conky+lua?content=139024)

 Once you have that,

 Then when you start conky again, 
It should come up as mine does. 

ohh, You will need to install a package called 
> liblua5.1-gnome-0

You can install it from synaptic, or from the terminal. 
To install from terminal just type 
```

```

That will allow the rings to display properly.
