Gaim requires the same GTK runtime environment that makes the GIMP so sluggish in Windows, but the Gaim application itself is nowhere near as resource intensive as the GIMP.  So it's not really a sluggish piece of software as far as software goes, but considering it needs the GTK runtime loaded just to send some instant messages, I think it's pretty excessive for what it does.  If you've got the computer power to run it though, and you need some of it's features (full-featured AIM protocol, for example), Gaim is very nice.  But if you don't need those Gaim-specific goodies, it's hard to justify using it when Miranda is lightweight, incredibly capable, and also open source.

### Post by hizaguchi on 2006-08-31
AIM has 2 protocols, TOC and Oscar.  The Oscar protocol is the official one and it supports things like buddy icons and direct connect, where the TOC protocol is free but doesn't have those things.

Gaim supports the Oscar features (I guess they reverse engineered it) but I've found file transfers to be sketchy.  I don't remember how installation went, but I think it's the same deal as GIMP (install GTK, then install app... don't need GTK again if you already installed it for GIMP).

Miranda is very easy to install.  You just unzip it into a folder (or let the automated installer do that for you).  The semi-tricky part comes when you wanna add plugins later.  You have to unzip those into a particular folder under the main Miranda folder.  The really tricky part is when you want to get fancy and do stuff like transparent buddy lists and fancy chat themes.  But even that isn't too bad, and just to get something that lets you chat is very simple.

Trillian is the easiest of them all and supports everything as far as I know, but it is closed source and comes in a basic and "pro" version.  Basic is free, pro costs money.  I think Jabber is only available in pro, but I'm not sure.
