### Post by eentonig on 2007-06-14
As an end-user, it's about the same.
If I want to browse, I click the browser icon; If I want to write a mail, I click on the mail app icon; if I want to write a cd, I click on the CD burning icon..... etc. I don't see the difference.

As a sysadmin/administrator: Yes and no. 
- It's harder because a lot of software hasn't been written with an end-user in mind. Therefor, foolproof methods usually don't exist.  And a lot of apps force you to struggle to get them working. And then they break again after an upgrade. ....
It's a lot easier, because there's a common file structure for all applications, documentation is usually extensively available, rtfm is a wise advice because 'tfm' is usally a 'man' or '--help' away. and it's easier because the debugging options are a lot more present. Whenever a program gives you problems, you run it from commandline and paste the error messages on a forum. Chances are great someone will point you int the right direction.
