### Post by Outworlder on 2006-11-19
I've got some ideas for this thread I'd like to share.

I am no Gaim developer(even though I'm studying the source for some time now), but I've been a Gaim-devel subscriber for quite some time, and I have some idea of what is going on.

First of all, Gaim supports Jabber, not GoogleTalk. Yes, GoogleTalk is Jabber(XMPP), but it does use some non-standard extensions. Which is fine, since Jabber allows for that. Some of them were created by Google and are standard now. If anyone is interested I can dig up some links.

There's a discussion right now on Gaim-devel about creating a new GoogleTalk entry, in the protocols list, splitting it from the Jabber one. There are some reasons for that, mostly because people do not know that GoogleTalk is really Jabber and keep asking Gaim to support it. The other reason to split the "protocol" (or rather, the protocol plugin) is that right now it is unclear how the GoogleTalk support should be implemented.

The ideal way would be to detect such extensions and just enable them if the server supports them, regardless if they are the GoogleTalk servers or not. The other way is to enable them if the user explicitly requests GoogleTalk. This is under discussion.

Sean said this, 4 days ago:

> 
If there were a Google
Talk prpl, and I could extend the Jabber protocol with signals, I
would not hesitant to make Gaim a really top-notch Google Talk client,
supporting everything a Google Talk user could ever want, the instant
it appeared in Google Talk.


So, if all goes well we won't need this petition in the near future :)

As far as video support goes, that's a bigger effort. Even though it is not apparent on the surface, Gaim 2.0 is a major milestone. I am not talking just about bug fixes or features. The libgaim separation is now complete*, and the code had some major refactoring. Now that the foundation is solid, we can look forward to audio/video support.

About the Win32 GoogleTalk client... guys, you are all aware that GTalk uses *_**[COLOR="Red"]Internet Explorer[/COLOR]**_* to render the conversation windows, right? I don't know exactly how tight the dependency is, but keep that in mind.

Well, I am requesting the linux version anyway. In Google We Trust! ;)


[I]*Before anyone says anything, I am aware that Adium is using libgaim even before the core separation. Just ask them the pain it is :)
[/I]
