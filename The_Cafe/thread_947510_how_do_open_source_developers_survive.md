### Post by beno1990 on 2008-10-14
You have to remember, for many programmers, programming is a hobby as well as a career. Someone who goes into programming for a company might find they don't get to make software they'd like to while working in the company, and so do it in their spare time instead. Others do it because they believe in the notion of free software being available for all.

There are also companies which produce free/open source software to make a profit. Mozilla do this a lot (Firefox and Thunderbird), Sun Microsystems (OpenSolaris and Open Office) Red Hat do, and many others do so too. And there's a real advantage to it. Your company might be able to hire 80 developers, but if it's open source and people want a working product, they're likely to contribute to the code too, so they get free labor in a sense, but it was because the community wanted to co-operate and make software that does what they want, instead of a company telling them what they want.

### Post by david_lynch on 2008-10-14
> **Dr Small said:**
> They work like normal citizens, and code their open-source projects in their spare time. 
It depends on which "open source" you are talking about - there are different categories.

For instance, the linux kernel developers are for the most part technical professionals who are paid by their employers to write linux kernel code as part of their job. 

OTOH, there are a good number of open source projects which are done on personal time, where programmers are "scratching an itch" by writing code to solve a problem.
