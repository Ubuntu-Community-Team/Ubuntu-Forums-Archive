### Post by userundefine on 2007-03-29
Well, browser devs (I mean non-IE devs) need to keep improving their rendering engines and adding features.  The W3C needs to get CSS3 shaped up so browsers can start implementing it.  And the browsers have to keep innovating to force IE to either try and do "as much as it can" or sink permanently.  Personally, I'm for the latter and wish MS would embrace FF codebase.  The problem (among others) with this is that so many businesses are running intranet apps based on crappy IE.  Firefox et. al. need to become not only the best browsers for the web, but the best for offline usage too.  This is a lofty goal and won't happen, imo.  The other, more realistic option is that the browsers keep innovating, devs keep coding to standards and just backporting code to keep IE functional (and not trying to hack sites to display exactly the same across browsers--saving lots of time for devs), and get more promotion/endorsements of FF, Safari, etc., from the big boys like Google.
