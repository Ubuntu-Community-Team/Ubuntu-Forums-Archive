### Post by popch on 2007-11-23
This is one of the greatest strengths of corporations writing software. They can and should employ people specialised in defining and measuring the usability of products.

Apple has major guidelines and has been known to conduct numerous studies. Rumours will have it that MS does much the same, with partially different results.

Many parts of Linux and of Open Source applications are done by smaller firms or private individuals. Some of them lack either the means or the inclination to invest in the usability of their products. In some cases, it shows very drastically.

This does not, however, equate to 'usability proportional to size of corporation'.

Also, usability is in the mind of the user. One user's help is the other's hindrance.

### Post by samjh on 2007-11-23
"Usability" is defined by the target users.  Nothing more or less.

It is not defined by developers.

It is not defined by anyone outside of the target group of users.

So if you have a software product that is a one-size-fits-all, then you will never have perfection from a usability point of view, because it covers too large a demographic.

Generally usability considerations deal with efficiency.  How efficiently does a machine or a user interface help to solve a problem for its users?  When the usability of a particular project is tested, a sample of the target users are chosen to test a user interface and various observations are made, which include:
1. Time it takes for a user to find relevant information from the user interface.
2. Time it takes for a user to utilise relevant information.
3. Time it takes for a user to become familiar with the user interface.
4. Ease of transition between parts of the user interface.
5. Frequency of errors made by a user when using the user interface.
6. Frequency and magnitude of physical movement by the user when using the user interface.
7. The user's mental load when using a user interface.
8. User's ability to retain the knowledge of how to use the user interface.
9. Onset of mental or physical fatigue when using the user interface.
10. Whether the user interface presents and allows the use of all desired functionality of the machine or software.

This means that usability issues vary widely between target users

For example, fast jet pilots experience large amounts of physical stress during combat flight, so controls and instruments needs to be designed to be used with minimal physical effort without being excessively fragile or loose (fragile interfaces cause malfunctions, loose interfaces contribute to erroneous user inputs especially during high-g manoeuvres).  However pilots of large passenger airliners suffer comparatively little physical stress, so the controls and instrumentation can be designed more liberally, but consideration needs to be given about fatigue caused by long-distance flights and durability of the user interface against physical collisions by moving crew members, spillage from drinks and meals, etc.

For computing, one of the biggest considerations is with the expertise of target users with computer systems.  For operating systems or application software targeted for general computer users who do not possess advanced knowledge or experience with computer systems, a simpler interface is usually preferred over complex ones.

Ubuntu has become a popular distribution largely due to its ease-of-use, especially its installation process.  If it wants to retain its spot as a première distro, it needs to keep making itself easy to use for inexperienced computer users.

Perhaps for a related reason, Ubuntu chose Gnome as the default desktop.  The decision has merit on HCI grounds, since Gnome is a very simple desktop environment and presents mostly good defaults for general computer users.

However, Gnome's weakness is that it doesn't cater well for advanced users, due to the tendency to hide in-depth features from its graphical user interface.  Instead, an advanced user needs to dig into configuration files.  This is the opposite of KDE, which tries to present as many features graphically, at the expense of overwhelming inexperienced users.  Another problem with Gnome is its lack of consistency across Gnome applications, unlike KDE's highly modular application framework which provides a consistent layout and intuitiveness for most KDE applications.

If you feel that Ubuntu is making life too hard for advanced users or users who like to tinker with advanced features, you're probably right.  That's because Ubuntu developers are probably trying to appeal more and more to general computer users.  You could either join the development team and try to add features which will cater for power users, or try alternative desktop environments or distros.

[quote=Jimmey]The other day I was horrified to see a little pop up that told me an application had crashed and that I could send an error report, or similar. It stank of Windows[/quote]Don't reject a good idea just because something similar is featured on Windows.

From a HCI point of view, Microsoft has done tremendous work on usability, especially in business environments where most users are either computer illiterate or barely computer literate.  The success of the Windows 95 paradigm (the famous task bar and start menu) bears testimony to their work.  Some may argue that the idea wasn't originally Microsoft's, but regardless of such claims, it was Microsoft who made the idea work.

Sure, they haven't always hit the mark (clippy, anyone?), but usually they've got usability right.  The ability to send self-generated error reports is an excellent QA tool, and should not be shunned, especially when the user clearly has the option not to send such reports.

### Post by DrMega on 2007-11-23
I like GUIs. Sometimes they make you aware of options that you din't realise were available.

That said, they MUST be well designed or they are rubbish.

I develop under Windows for a living, for a large corporation, and have the joy of having deal with the end users regularly, so I see the effects of my (and my colleagues) work quite quickly. For a GUI to be worth while you need to work out what the most common set of requirements are, how to put them in the least technical terms possible, how to guide the user to make the right choices for them, and how to ensure they can't screw the whole thing up in one swift mouse click.

The trouble with many GUI tools is that the very technically minded people that develop them, maybe forget what it was like to thing that a PC was copper, and a mouse was a small fury animal.
