### Post by murderslastcrow on 2010-08-24
I don't think KDENLive has an official Windows package outside of the Windows KDE 4 installer.

I showed him GIMP- he really likes it. I don't get why people think 3 windows sucks, it's more versatile. Just like how the OS X versions of Adobe products have docks above the desktop, allowing you to drag. They're quite reminiscent of each other.

But yeah, I showed him all the filters, including IWarp, the fact that he can save to PSD, and using Photoshop brushes and plugins, and he just said, "why the **** do people still buy Photoshop?"

I told him it's because of CMYK, non-destructible layer effects, and unfamiliarity. Hey, you've gotta' be honest. But I also showed him GIMPShop for the sake of not scaring him or whatever- he didn't like it quite so much. It looked more like a rip-off to him, so he's keepin' GIMP.

Anyway, thank you so much for your responses! I got him playing some of those games, too, like Tremulous. He's kinda' addicted now. >> And Nexuiz is one of his new favorites. Kinda' makes me wanna' develop an open source game some day (I do commercial programming as it is- shoot me if you must, but I have bills to pay, it's not like I don't like contributing to the community as a whole and protecting freedom).

But yeah, especially opensourcewindows.org- that one totally hit the spot.

I'm afraid that it's an incorrect assumption to say most open source software is available for Windows. The truth is that more than half of what's available in the Software Center has not yet been ported to function on Windows, even if you count the Qt applications. There's a wealth of things you just can't do anywhere but Linux, too.

To be honest, OS X probably provides the most software, as it can use Darwine as well as Fink, as well as native OS X applications. However, I hear that Fink isn't perfect in some regards, and Darwine and Fink don't seem so user friendly as Wine and the Software Center.

Anyway, thank you very much for your input- I think his fears are reduced. Now, if I can get him to use Emesene or something similar, he'll be able to use all of his favorite applications AND use Linux, which is itself one of the most robust open source projects available. I think building up this anticipation is a good thing, while I always continue to remind him of the software availability issues as not to make him unduly disappointed.

Just be honest with people, and even if they won't use it full time, they'll respect Linux as a platform, and have a greater appreciation for open source. And really, if you use a closed kernel, some closed libraries, and a closed window manager, but everything else is open source, is that really so bad?

I mean, a few of us use Wine applications, Flash, and other closed applications for Linux. I think we might actually have a bit more closed dependencies than my brother has now. XD But yeah, either way, it's good to have as much control as you can in whatever environment you decide to use. Also, Microsoft executives are embracing open source on that platform and admitted they had made a mistake. I think using Linux is going to be a lot more pleasant in the next few years.

**TL|DR Thanks again! :D This is working out pretty well.**
