### Post by murderslastcrow on 2011-05-10
Unity 2D is awesome, except for the workspace manager feature- it's great that it's even there to begin with in a 2D environment- great implementation- but it's not very fast. It'll get there, I'm sure. For the most part it shares the functionality very well with the Compiz-based Unity.

Also, for many computers I'm helping to upgrade, Unity 2D is the only option (graphics drivers won't support Unity, they're too old I guess). Of course, I think you can use vanilla Compiz with Unity 2D, without the Unity plugin. If you use xcompmgr or a similar compositor, you can get most of the antialiasing and transparency effects from the official version. And I suppose it's faster, too, from what I've heard- I'll have to wait for my new video card to come in the mail to test the Compiz version.

I think it's great that, like KDE 4, Unity has a 2D implementation that stays true and implements most of the features from the original. KDE 4 doesn't have any effects for workspace switching under the 2D functionality, but that's the main difference. Gnome 3's 2D version is basically Gnome 2, with configurable panels and the such, so I'm not entirely sure if it counts (it seems like a whole separate environment, in some respects, but well-integrated).
