> **MyNameisTheStig said:**
> 
There's three bad things about those (besides the name):
1. The Linux it comes with is rubbish, cutom to the EeePC and cannot install anything extra. Any programs it runs are pre-installed
2. There's no CD drive, so if you want to run a Live CD you need an external CD/DVD drive (my friend got one of those, so he could install Hardy Heron on it)

1. Yes, the Xandros it comes with is rubbish, and software installation in it is certainly more difficult than in Ubuntu, but it isn't impossible. You just have to be very careful about the repositories you add and make sure you don't upgrade with extra repositories enabled. I use the Debian repositories to install new software, but when I want to check for upgrades, I always remove the Debian repositories first and refresh the repositories list first.

2. While it's not as simple as booting a CD, it's also not extremely complicated to boot a USB stick as a live CD. You have to format it as FAT16, make it bootable with syslinux, copy over the live CD files, and rename two files. I have done so with Ubuntu and made my Sandisk MP3 player into both an MP3 player and live "CD." Again, not convenient, but not impossible either. The screen size isn't impressive, but it's certainly usable. Only occasionally do I have to side-scroll, and full-screen mode in Firefox helps a lot with browsing usability.

You bring up some valid criticisms, but they aren't absolute downsides so much as inconveniences with workarounds.

So long as it is developed and distributed as free software, there is little chance that any GNU/Linux OS will end up in the malware boat with Microsoft (socially engineered exploits notwithstanding).  Its very system of development prohibits this for two reasons.

Firstly, universal access to the source code empowers a much larger percentage of programmers worldwide to fix bugs and patch exploits.  This in itself is an incentive to desire a larger user base, since the percentage of technologically capable persons will increase both because of those who are drawn as users and those who are drawn to work on the platform.

Secondly, the Linux kernel maintains no stable ABI (and therefore implicitly no stable API either), thus it is very unlikely that any exploit will effect a very large number of Linux kernel versions.  Other free software programs can enjoy the same dynamic, at least those that are actively developed.  Again, this is an incentive to increase the user base, since popularity will also draw more developers to actively develop free software programs.

The advantages that free software has with which to insulate itself from malware are entirely dependent on its freedom.  Proprietary software can never enjoy any such protection because the source code is not made available in a way that ensures that the person who is able to do the work can do the work, unless they are already employed by the entity that controls the licensing of the program.  Because it maintains a stable ABI with which to ensure promiscuity with other non-free software, Windows is incapable of avoiding setting parts of its implementation in stone.  It will consistently be vulnerable where those parts are exploitable, and cause havoc with drivers and software compatibility whenever a new release is made.  This is a critical example why it is technologically undesirable to add _anything_ non-free to GNU/Linux, especially where a free implementation is possible.
