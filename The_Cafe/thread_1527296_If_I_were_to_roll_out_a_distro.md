### Post by nerdopolis on 2010-07-09
It would be KDE based 

it would use NX from no machine to  make apps not go down if the X server dies in a driver accident. (and so  that you can disconnect from a box, then reconnect to it later, you can't do that with native X).

it would have the pacman package manager, as it  supports multiple versions of packages

it would have something  similar to PBI from PC BSD (and try to make it distro independent)

It  will try to add feature integration between packages, such as  integration between dolphin (with a KIO) and rdiff-backup, to allow a  time machine like thing, and other stuff that would make use of the  multitude of 'hidden' functionality within open source programs. It will  use packagekit to try and detect if the package that bring out such  features are installed, before trying to  use them.

it would have  release frames at about a year. It would aim at an approximate time  frame, instead of a set date.
 
It will be user focused. It will  try to have some graphical configurations, as well as a few guides for  commands (like it telling how to revert back to the gui if you switch  tty's), and examples in the man pages.

would not try to shun  features out be different from other OSes. Of course it will be  different in many ways, but it doesn't have to be totally different.

EDIT!: I forgot to mention that toolkits would be try to be ported to XCB (it would be interesting to see if it could be binary compatible with the normal toolkits) It would have to do it to older version of the toolkits as well...

That  is if I had the skill to do so, and a team... sorry for the length, this thread kicked off my imagination juices.
