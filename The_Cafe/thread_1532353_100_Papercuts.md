### Post by duanedesign on 2010-07-16
[One Hundred Paper Cuts]("https://launchpad.net/hundredpapercuts") is  a project led by Canonical&#8217;s Design and User Experience team to improve user experience in Ubuntu by identifying 100 small points of pain for users, or &#8220;paper cuts&#8221;, and healing them! A paper cut is a easily fixable usability bug that the average user would encounter in default installation of Ubuntu or Kubuntu Desktop Edition. Many times users become accustomed to ignoring minor glitches to the point that such problems practically become invisible. The result is that there are a lot of really subtle deficiencies that have long been overlooked by developers but are immensely frustrating to new users. The One Hundred Paper Cuts project aims to identify and fix these issues.


**The papercut project needs your help.** Papercuts project is a great way to get involved in the Ubuntu community. Several members have found papercuts a stepping stone to get involved with the Ubuntu community. There are several ways you can contribute. 

**Hacker**
Have a look at the [milestoned/triaged]("https://bugs.launchpad.net/hundredpapercuts/+bugs?field.status%3Alist=TRIAGED") papercuts, grab some papercuts and help fix them. You can get more info on the [wiki page]("https://wiki.ubuntu.com/PaperCut").  You can also ask questions and get clarification on the [mailing list]("https://launchpad.net/~ayatana") or on irc.freenode.net #ayatana.


**Non Hacker**
No Problem , if you just love the initiative and want to make Ubuntu more awesome there are lots of things you can do. Test your favorite default applications and see if they have any papercuts that need fixing. Report these bugs and bring attention to these usability issues.  Please review the documentation on what is considered a papercut very carefully if you are going to report a bug as a papercut. David has a couple of nice blog posts helping to explain what makes a good papercut, [here ]("http://davidsiegel.org/paper-cut/")and [here]("http://davidsiegel.org/calling-all-paper-cutters/"). You can also help to triage existing papercuts. This is very important to help reduce the number of non papercut bugs that have been reported as papercuts. As well as make sure all the reports have enough information for a developer to fix the papercut. You can get more info on the [wiki page]("https://wiki.ubuntu.com/PaperCut").  You can also ask questions and get clarification on the [mailing list]("https://launchpad.net/~ayatana") or on irc.freenode.net #ayatana.
