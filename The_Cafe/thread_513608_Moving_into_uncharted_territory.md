### Post by Palmyra on 2007-07-30
The programmers forum here should be useful.  

It would be a good idea to become familiar with the terminal if you want to make good use of Linux, and although there's no real relation between the terminal and programming languages, the terminal may help you become more comfortable with the unknown (the computer and everything you're not familiar with).

Most beginners are encouraged to learn Python.  Don't bother with C++, C, etc.  

You don't necessarily have to buy a book.  Go to your local library, see which Python book you're most comfortable with, and if you like a book, buy it online.  Just make sure you use the library to determine which one is best for you, but don't forget to use Amazon.com to read ratings.  

Your intentions are very worthy, because GNU/Linux is only as good as the effort put into it.  Whenever you have spare time, please spend it on learning something useful so you can contribute to FLOSS.  

Get started by reading Python books.

Eventually language choice *shouldn't* matter, but starting out it could.  I'd seriously hesitate to start with something like C++, because of how twisted it can get.  It could lead to early frustration.

As mentioned before, Python is a great choice.  Here is one [tutorial]("http://wiki.python.org/moin/BeginnersGuide") among many for learning Python as a new programmer.  It's a language that allows you to start out just using it for simple scripting tasks and end up writing complex object-oriented GUIs or web sites.

It's not the only choice.  Java is at least a more consistent and clean language than C++.  There are also languages like Logo that are designed to teach kids to program.  A variation of BASIC is also not a bad starting point, but I'm not sure which Open Source implementations are best.

I'd avoid unclean hybrid environments like the Web until you get more comfortable.  You need to learn a lot of tricks to get things to work well.

Good luck, and ask a lot of questions.:)

### Post by IanH on 2007-07-30
[COLOR=Black]Some general tips:
1) Come up with a project. You want to keep it simple (you're not going to be creating a MMORPG off the bat), but I find it's far more motivating when you have a concrete goal than when you're just doing contrived examples. Think of a _simple_ utility (or possibly a very simple game), and work towards this goal. You might want to consider writing an extension for firefox/epiphany/rhythmbox/etc rather than a full fledged program.

2) Once you have an idea, think through it logically. Try to make a flow chart or step-by-step instructions (in plain english) describing what has to be done.

3) Take these instructions and break them down into small, separate tasks, which you can put together like legos to build the final program. For instance, if you were writing an instant messenger, one task would be to make a window for the user to type a message, another would be to send a text string to someone, etc. This is not only good programming technique, but it will also give you immediate gratification (since you'll have something finished sooner).


Some more specific advice:
The terminal is important, so at least get comfortable with the basics. You may want to make your first few programs command-line ones, so you don't have the extra complication of a GUI.

Language-wise, pick something high-level, so you don't have to worry about so many technical details. Python is probably a good choice. It seems easy (although I haven't really used it), there are lots of beginner's tutorials, and it's widely (so you'd be able to contribute to many projects without having to learn a new language). Plus, it's interpreted, which means it's easier to mess around and see what commands do.

Well, hope that helps. Good luck.
[/COLOR]
