### Post by Qrk on 2006-04-06
It depends. For a new user, Linux distros seem entirely different. People coming from windows and mac probably will consider the largest differences in new operating systems by the desktop. By this standards, different Linux distros are even more "different" than, say, Windows 98, 2000 or XP. Depending on the distros in question, say the difference between Linspire and Gentoo, it can be even greater than the difference between Windows and OS X. 

But once a user has used Linux for a little while, she or he will know how to change the theme. They've probably tried out KDE and Gnome, and many have tried out new distros. You begin to recongnize that there are really on four differences that matter when choosing most distros.

1) Installation- Many different methods. I think it used to be a deciding factor in choosing a distro, but its all usually so easy now it doesn't make much of a difference. I'll include hardware detection, but even than becomes less and less relavent as you get more experience. When I first started, if a distro didn't detect my monitor resolution, I'd move on to another. Now I use dapper... which found 640X400 and no network connection.

2) Package Managment - This is the big difference. A distro with good package managment can be anything you want it to be. Its important to have access to the DE's and WM's that aren't installed by default as well.

3) GUI tools - Arguably what makes SuSE popular. Ubuntu has been adding them slowly, and has gotten to the point (in Dapper) where I took my terminal launcher off of my panel. Does that make me a bad person?

This is probably an incomplete list, so feel free to add more. But the things that most people think about as the "difference" in OS's, the default theme and desktop enviroment, aren't on it.

### Post by bonzodog on 2006-04-06
If you look at the linux distro tree, you will see that it is very much a tree. There are Just 4 Distros at it's roots - all others are forks of these and of each other. BSD doesn't count, it is a true unix, like Solaris. 
At the roots you have:
**Slackware** - this distro was has been classed as THE granddaddy of them all, and doesn't use the System V init, it uses a BSD-like init system written by it's sole Maintainer, Patrick Volkerding. Slackware started out as SLS linux way back in 1992, and has it's own unique, if confusing, Packaging system. I used this from 1997 until last year, and it's current stable release is 10.2. Forks from this include Zenwalk and Vector

**Debian** - Another one of the real oldies, Debian has always strived to be be pure GNU/Linux. It uses the System V Init in it's entirety, and is famous for having come up with the Software repository idea. It is possibly also the most forked distro, and has many "children" including Ubuntu, Linspire and Knoppix.

**Redhat** This is possibly the most well known Distro, and again uses the System V-like init system, and has the RPM packaging system. It was the first real 'commercial' distro, and is in Wide usage in large companies, and on internet servers. It's Children include Mandriva and CentOS. Suse was originally based on it, but has since changed track drastically and in a way, become it's own root.

**Gentoo** - Although a relative newcomer to the scene, it planted new roots when they decided to use the 'Build Entirely from source' ethic to really fine-tune linux to the machine on which it is running. Gentoo also uses the highly acclaimed 'Portage' system to install software. It has no children AFAIK, although other distros have copied the 'build from source' idea, based around other distro root trees.

So, you see there are only originally 4, but they have had many Children, and we currently have about 140 distributions available to us. Some distro's specialise in one area, like Live CD's, or High-security, or Audio.
They are all forks though.
