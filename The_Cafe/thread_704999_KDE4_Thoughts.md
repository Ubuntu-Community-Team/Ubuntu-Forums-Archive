### Post by ilikenwf on 2008-02-22
I actually got KDE4 to run on my Kubuntu based laptop. It is very pretty, but very hard to use. Right click doesn't behave properly, the taskbar randomly disappears, you can't actually delete files off the desktop without using a console, you just close them like widgets, which is weird and makes it unusable. 

KDE4 seems to be much faster.

They need to refine the current widget system and split icons and files as separate entities from the widgets that behave like the ones in KDE3, so you can easily rename, move and delete them, not just close them. Then, have the widgets be draggable, resizable, etc, like they are now, with more configurables...for crying out loud, I can't get anything to look the way I want it to, and right click simply doesn't do much of anything for most items. The taskbar is not able to be added, you can't change the kicker, and you can't easily add icons for programs and applets to it. To get the bar back if you accidently remove it, you have to delete the .kde4 folder. Furthermore, I can't get custom icon sets to actually work with it. They'll install and take my space, but won't display.

I only have kde4-core installed, I don't know of much more I need to install to have it fully function, if that is part of my problems. I do know that I need to install kmix-kde4, among others...

Help/opinions appreciated.

Oh, and a nitpick - KDE devs aren't "making their own window manager" - they are still using KWin, like they have for years.  They have simply added a small compositing core and an API that allows extra effect plugins to utilise this core.  The GNOME guys are doing much the same thing with Metacity, IIRC.
