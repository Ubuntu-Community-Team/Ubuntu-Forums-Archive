### Post by akiratheoni on 2008-04-14
I'm writing an essay (some of you already know) but after an interview with my teacher it turns out I'm going to have to rewrite massive portions of it.

Basically now my essay will be about proposing what the 'perfect software developmental model' should produce and evaluating which model (FOSS or proprietary) is closer to it.

What should the ideal developmental model have? So far I have:

Stability
Fast bug fixing
Innovation
Ease of use

If you don't understand what I'm trying to say, the perfect development model would produce software that is:

99.9% to 100% stable
Bug fixes limited to hours
Innovation is rampant 
The software is easy for anyone to use

If you guys help me, I need non-biased categories. In other words, 'transparency' will NOT be a category because it obviously favors free and open source software. Now, at least for us, the first two categories (stability and fast bug fixing) seem like an obvious bias towards free and open source software. In truth, it probably is -- but it IS essential for software so it has to be included.

I need help for more categories if possible for me to evaluate. Thanks!

### Post by koenn on 2008-04-14
how about portability ?
operating systems change over time, and you want your software to be able to follow these changes. 
eg Windows XP introduced slightly stricter security than its predecessors, and it turned out 'older' applications had trouble dealing with that - you want a development that is capable of dealing with such changes, or you want a development model that produces software that us immune for such changes

maybe you also want software that can easily be ported to different platforms, either hardware platforms  (use the "same" program on an Intel PC and on a handheld device) or operating systems ("same" program on Linux, Windows, Mac, ...)
