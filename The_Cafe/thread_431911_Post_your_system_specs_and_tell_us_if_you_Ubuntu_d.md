### Post by kleeman on 2007-05-26
OK the var log (part 2) has one key line in it:

(EE) GLX is not supported with the Composite extension 

If you are looking for problems with X in this file they always start with (EE)

I googled this error message and came up with this page:

[http://wiki.java.net/bin/view/Javadesktop/Installation](http://wiki.java.net/bin/view/Javadesktop/Installation)

and they give a fix at the bottom of the page. It involves modifying your /etc/X11/xorg.conf
file (you need to use sudo to do this e.g. sudo gedit /etc/X11/xorg.conf):

Find these lines:

Section "Device"
	Identifier	"NVidia Riva TNT 2 Model 64"
	Driver		"nvidia"
	BusID		"PCI:2:0:0"
	Option		"UseFBDev"		"true"
EndSection

and add above the last one the following:

       Option "AllowGLXWithComposite" "true"

save the file and then reboot.

check the output of glxinfo again and also look in the log file you posted above to see whether the (EE) message has gone....

Edit: It looks like your symptoms are identical to this bug:

[https://bugs.launchpad.net/ubuntu/+source/restricted-manager/+bug/91064](https://bugs.launchpad.net/ubuntu/+source/restricted-manager/+bug/91064)

Strangely the bug claims that a fix was released in March. In anycase the fix in this bug report is the same as what I posted above......
