 [FONT=Arial, sans-serif][SIZE=4]**[FONT=Verdana, sans-serif][SIZE=2]An essay from Ivo - 17/05/11[/SIZE][/FONT]**[/SIZE][/FONT]
  [FONT=Verdana, sans-serif][SIZE=2]Have you ever asked yourself: Why do I pay for software? Probably not. The most people think it is naturally. Or they even never think about it, because the price of your Windows license is mixed with the price on a new PC. And if you buy a PC without an operating system, you can save €50,- easily. Then you just use a free, Open Source operating system, like Ubuntu. And instead of Microsoft Office you use LibreOffice.[/SIZE][/FONT]
  [FONT=Verdana, sans-serif][SIZE=2]But does the world need Open Source software? Most people use the paid versions, and because most people want to pay as less as possible, that software has to be bad?! I do not think that is true, and sometimes the Open Source variants are even better. So, does the world need Open Source software?[/SIZE][/FONT]
  [FONT=Verdana, sans-serif][SIZE=2]The first advantage of Open Source software is that everyone can see the code. In this way everybody can improve it, and fix bugs in it, little problems in the code. In this way you have less crashes and safety problems. Red Hat (my first source) is confirming this.[/SIZE][/FONT]
  [FONT=Verdana, sans-serif][SIZE=2]The second one is that Open Source software the big companies awake keeps. A good example is Microsoft with Internet Explorer 6. They were losing market share fast, much people where moving to Firefox or Opera. The same applies to Google and its Google Chrome. Browsers are improving really fast and are following Chrome design, minimalistic. If those companies not existed, we maybe still used Internet Explorer 6.[/SIZE][/FONT]
  [FONT=Verdana, sans-serif][SIZE=2]Also with Open Source software you can spread computers and software easier. In developing countries they do not have enough money to buy computers for education. Much Open Source organisations accept donations, and partly pass it on to projects to help children in developing countries so they can buy laptops. In this case you spare much money on the cost of the software and you give money to a good purpose. Red Hat confirms this again.[/SIZE][/FONT]
  [FONT=Verdana, sans-serif][SIZE=2]Now that are the advantages, but every kind of software has disadvantages, including Open Source. All this disadvantages comes from Open Source Technologies (my third source).[/SIZE][/FONT]
  [FONT=Verdana, sans-serif][SIZE=2]The first disadvantage is that there is no real support for Open Source. This is not actually true. Almost all of the Open Source software has a community, in many languages that always want to help you. Also there are much websites which offers a manual for the program. Also almost all of the big companies offer official support. But then you have to pay for it. Not really weird, in many cases that is their only source from income.[/SIZE][/FONT]
  [FONT=Verdana, sans-serif][SIZE=2]Also some people say that you have with Open Source software no guarantee of updates. I totally disagree with that, of course there are programs that are not updated any more, but the same problem you have with paid software. Some companies, like Canonical, the company behind Ubuntu, is running on the money from a big millionaire, Mark Shuttleworth. And even if he goes bankrupt, there is a big community that offers updates. And even if that stops, you have Debian, which Ubuntu is based on. They have to go bankrupt to and I do not see that happen soon.[/SIZE][/FONT]
  [FONT=Verdana, sans-serif][SIZE=2]Then there is one last problem. There are higher installation costs. The cost of licenses and the installation is not the problem, but the staff. People are used to use Windows, so they need training if you want they use a Linux Based Operating System. There are free programs that offer training, but then you still need a short training to check if they understood it or if they still have questions. So the cost are not higher, but more equally high. But if your employees are used to the Open Source software, it much cheaper on the long-term.[/SIZE][/FONT]
  [FONT=Verdana, sans-serif][SIZE=2]But is Open Source totally really free? Yes, and it always will be free. But much people that like Open Source helps the project, in any way whatsoever. Translating, bug reporting and fixes, ideas for new functions or even add new code. Normally a company hire person for such jobs, but with Open Source there is a community for it. So Open Source is actually free, but much people do something back to the company. My second source explains this in more details.[/SIZE][/FONT]
  [FONT=Verdana, sans-serif][SIZE=2]I could write a much longer essay, but it should not be too long. So here is my conclusion. Does the world need Open Source software? If we want to still want to use computers, want innovative and low cost software (especially for home use), then yes! The only big disadvantage is, and that apply only companies, that the employees needs training to get used to the software. But if you look on long-term it is cheaper.[/SIZE][/FONT]
  [FONT=Verdana, sans-serif][SIZE=2]So Open Source software saves us money, educate the developing countries, keeps the big software companies awake and it is safe. So why is it still so little used? Big companies like Google or Wikipedia already use Open Source software, but people at home not. So it is a question that I cannot answer, but I hope it changes soon![/SIZE][/FONT]

Secondly, your sourcing is far too basic (again, not sure how much this is going to matter). As you are using web pages as references (which when you get to university is practically a universal No-No), you need to record the date at which you accessed the page, and when the page claims to have been last edited/created/added (if poss.). The obvious reason being that unlike physical texts, web pages can undergo complete changes or can simply disappear off the web without any warning. 

Also, don't let your sources take the slack for you. Simply citing other people saying 'Go read them for the reasons why X' is really bad practice. You need to show that you understand what is being said, so summarize the arguments or quote select key parts.

Third, your arguments are vague & speculative with no real justification for why they should be accepted. 
You need to consider what your opponent might reply with, and deal with any possible objections. For example, a critic might say (and usually do say, for that matter) that open source software is less secure, because it makes any flaws in the code easily accessible to those that read it; not everyone looking at the code will have good intentions, and there is no obligation that they have to tell anyone about security flaws. What's to stop them from exploiting it?
