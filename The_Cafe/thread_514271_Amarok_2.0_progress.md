### Post by igknighted on 2007-08-01

Looks very promising anyway. But I'll be sticking with Exaile/Songbird until someone ports a GTK version :P

The problem is that qt is more advanced than GTK at the moment.  You would loose some of the pizaz if you switched to GTK.  The biggest stopping point is that Amarok itself is mainly a front end for the KDE audio playback subsystems (Phonon is the core in the new version, IIRC).  You need these core KDE subsystems to use amarok, which basically means you need all the KDE libraries.  Unless you had compatible gnome equivelents, it wouldn't be Amarok for GTK, it would be an Amarok-like shell over a completely different media player.
