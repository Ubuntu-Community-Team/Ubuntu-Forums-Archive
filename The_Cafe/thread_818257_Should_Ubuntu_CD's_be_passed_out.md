### Post by aysiu on 2008-06-04
I think if we want to keep new Linux users for the long-term, more efforts should be put into switching Windows users to open source software available for Windows (of which there is plenty).

Switching them to open source Windows applications does two things in preparation for an eventual switch to an open source operating system: [list][*]It gets them more trusting of the quality of open source applications. [*]It makes their transition to Ubuntu easier later by making the focus on acculturating to the new operating system and not the new applications.[/list] A lot of people don't even know what open source is, and if they can experience several open source applications on Windows (VLC, Firefox, OpenOffice, GIMP), they'll see that they can accomplish 99% or 100% of what they need to without the need for commercial and/or proprietary applications.

And if they're used to OpenOffice and GIMP and Firefox on Windows, using those same applications on Ubuntu will make the switch that much smoother, as they'll have less to get used to.

I'm glad it seems ARhere is starting to modify the original proposal so that CDs aren't just left around, but even left around with a flyer, the CDs will probably lead to more bad than good. Yes, you'll have a handful of success stories, but most people will psychologically associate the left-around CDs with the annoying AOL CDs that pop up everywhere, or they'll actually try to install Ubuntu themselves and run into the usual problems (incompatible hardware, inability to play songs bought from the iTunes music store, culture shock) and then tell all their tech-savvy friends Ubuntu isn't ready and then post a "Ubuntu isn't ready for the desktop" thread on these very forums.

### Post by Mr. Picklesworth on 2008-06-04
Tyler H actually has a pretty good point. I don't think anyone can deny that there is work to be done. For example, GNOME's behind-the-scenes infrastructure has changed a lot over the years but existing applications need to be caught up with it. The computer desktop has evolved a lot, yet we are still working within the bounds of "file manager is the shell", which is neither Unixy nor intuitive. Fixing these things (see KDE4, to some extent) will create a large disturbance, but there will be a wonderful outcome if the audience does not push back.

I think it would be best for those things to be fixed before *all* the average users start using a desktop distro like Ubuntu. Otherwise, such change will not be as acceptable :)
Not that they can't use it now or that it would not be worthwhile for people to use Ubuntu, of course.

I do not think that ARhere's is a bad idea. I think the official Ubuntu CD does pretty well by boldly saying "pass this on". With that in mind, comparing this to AOL is entirely unfair! AOL CDs are notoriously the type of thing one would only give to friends for the hard plastic (in order to assemble chairs and whatnot). In this case, the CD is infinitely reusable -- just as software should be. One thing I noticed is that your own CD label does not contain that text. May I recommend adjusting it accordingly? (The CD-ROM text is way more than it needs to be, so space can be cleared).
People just aren't used to the idea of software CDs that are not wasteful. In this case, while it is still made of all manner of odd substances, the Ubuntu install CD is environmentally friendly because it is able to last a very, very long time.

Still, problems:
-Install fails: Person throws out disk.
-Install REALLY fails: Person throws out disk and curses the invisible entity that convinced him to use it by putting it there.
-Most computer stores will not give away free operating systems that are alternatives to ones they sell. I have noticed a tendency for salespeople to spread FUD of more epic proportions than "Get The Facts". (Particularly when said stores have failed miserably at having any small & cheap EEE-style laptops whatsoever).

What this particular idea lacks is a human in between it all. My favourite way of promoting Ubuntu is using my computer in a public place (eg: A library), as I would anyway, but keeping in mind to be very courteous and answering questions people may have... as well as having a generic install CD on hand. I have noticed a surprising tendency for people to gain some interest when I run, for example, Compiz, although I must admit to not being a huge fan of the thing. Crazy, yes, but I think this works better than randomly handing out disks. This way, people gain some interest on their own effort, so they're more likely to remember and care. That, and they have someone to ask questions of if it fails on them.

It's slower than the "go to library, add stack of disks to a table and watch them disappear" method, but a good thing here is that I can just keep "promote Ubuntu" in mind as I go about my everyday business; I don't need to directly focus on it.
