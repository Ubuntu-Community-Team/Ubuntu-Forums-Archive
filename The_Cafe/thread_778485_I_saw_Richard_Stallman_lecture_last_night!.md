### Post by az on 2008-05-02
> **dougleduck said:**
> Can you give an example of things missing from open source that would make it free?

The GPL defines software freedom.  You are allowed to obtain and use the software for any purpose.  You are allowed to obtain and study the source code.  You are allowed to modify it.  You are allowed to redistribute it.

There are lots of open source licenses out there that are non-free.  For example, distributing it under a modified GPL license which adds restrictions makes it non-free.  For example, adding a "non-commercial" clause prevents the user from using the code for any purpose.  Adding a clause whereby you are not allowed to redistribute modified code because the authors doesn't want to be bothered by support questions is another example.

Some licenses simply state you are allowed to study the code, but are not entitled to use it for any other purpose.  That's open source, but completely non-free (as well as pretty useless - nobody is going to pour over hundreds of thousands of lines of code unless they have a good reason to;  for example needing to modify the program to suit their needs)

While Software Freedom zealots are often ridiculed, the GPL takes a very strong stance on Software Freedom and it is the most widely used open source license in use.  So the belief in software freedom is something a lot of people take seriously.

### Post by howlingmadhowie on 2008-05-02
free software and open source are two ways of looking at the same thing.

free software is software which respects the users' freedom. it has nothing to do with money. stallman formulated 4 basic freedoms a computer user needs in order to be able to use their computer in freedom (in other words: know what the computer is doing, be allowed to help themselves, be allowed to help others, etc). you can find the 4 freedoms here: [http://www.gnu.org/](http://www.gnu.org/)

open source software is an attempt to tone down the ethical side and make free software palatable to big business. if started with an essay called "the cathedral and the bazaar" by eric raymond. you can find the essay here: [http://catb.org/~esr/writings/cathedral-bazaar/cathedral-bazaar/](http://catb.org/~esr/writings/cathedral-bazaar/cathedral-bazaar/)

open source stresses the alleged technical advantages inherent in an open system. free software stresses the way free software enables the end user to avoid ethical dilemmas.

the main bridge between the two movements is probably bruce perens, one of the founders of the open source initiative and regular poster on slashdot.
