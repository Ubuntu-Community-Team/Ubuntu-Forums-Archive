### Post by chessnerd on 2010-05-14
This isn't a bad idea, and if there was a vote I would take part in it. However, I do notice some issues:

1. People have different philosophies.

Ubuntu is supposed to be both free as in beer and free as in freedom, but I use several closed source programs. For example, I prefer Opera to Firefox and Chrome. If Opera won a lot of "free as in freedom" people wouldn't be too happy about it. On the other hand, if proprietary programs weren't included for voting, then their users would be upset.

2. People have different preferences.

a) Some people greatly prefer stability to feature-richness and vice versa. I'm a "balance" kind of guy who uses the occasional beta, but only if it works. Some people might support including beta software with more features than the current stable builds and this could lead to more ugly debates. Plus, if one side was heavily represented, we could get either a feature-poor distro or an unstable one.

b) Some people prefer a small resource footprint to feature richness whie some don't. Users might request lighter programs, like AbiWord over OpenOffice Writer and Mirinox over Firefox, even if these programs lack features users have come to expect. These are great programs, but they might not belong in Ubuntu.

3. Some things aren't allowed.

Ubuntu Restricted Extras. Most people install it right away, but it isn't legal to include this in some countries. Then, Canoncial would either have to have an "International Edition" which lacked these features or people outside of the US couldn't use Ubuntu.

4. Some good programs are bulky.

Ubuntu is supposed to fit on a CD. Some programs (such as the GIMP) have been removed to keep this small size. If people all voted on their favorites, Ubuntu might no longer work as a CD distro.
