### Post by Mr. Picklesworth on 2008-11-28
That's actually one of the reasons I'm going vertical. With a horizontal layout, it can't seem to decide between screen 0 or screen 1 for either panel and they ended up being lost at some point. Further issues arose because one monitor is taller than the other, while they share the same width in pixels... so it's more balanced and symetrical to have the desktop more rectangular shaped. (Surprisingly, a good half of the GNOME wallpapers fit the tall screen thing perfectly!). Nautilus is really messed up either way, though; the desktop starts on the secondary screen instead of primary and the windows (in spatial mode) end up appearing at coordinates global to both monitors as a whole instead of a specific screen, which means stuff always appears on the top when I'm working on the bottom.

I definitely would prefer one huge monitor over two here, but I had this one sitting around as a glorified headphone stand so it had to be used :P
Besides, it's a good USB hub!

Oh, and I came up with what would be a wonderful innovation: A sticky mouse pointer (like we have sticky windows) for each screen. At the moment, Fitt's law is entirely broken with dual monitors.

The window list, however, I find astoundingly cool. If you place a window list applet on the second monitor as well as the first, they will change to only display windows on the current screen. That way, the top window list has windows on the top window and the bottom window list has windows on the bottom window. Very neat :)
