### Post by rosenkavalier on 2011-03-06
no, conky won't do... :(

I don't mind with text-only config. I don't mind learning some scripting to get what I want.

but there are problems

first, by default conky doesn't support animated gif. the gif file has to be exploded for each motions. I don't know how complicated the config script will be if I need an animated character who changes her expression depending on the system load

the next, AFAIK, a running conky always has /that/ box. we don't need, and we don't want a box. what we want is a cute little girl saying things like 'what do you want to do, big brother?'
if we hide the box by setting the opacity to 0, the image opacity will be greatly reduced, and it would blend with the background or completely disappear. cairo-lua did't help, all things drawn up by cairo became transparent too.

here, hina-chan's lucky enough to have dark background behind her, while the right corner of the pink box is barely visible with bright white background behind it.

[screenshot]("http://s.itb.ac.id/home/helmi_moore@students.itb.ac.id/webposts/maid.png")

maybe I didn't do enough research yet... but if someone come up with a solution (at least transparent/invisible conky background box with opaque images on top of it), we can make conky as cute as rainmeter. :KS
