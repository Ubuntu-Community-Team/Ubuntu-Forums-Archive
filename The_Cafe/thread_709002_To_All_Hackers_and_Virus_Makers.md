### Post by k2t0f12d on 2008-02-27
For those that do not know, 's/hackers/crackers/' is a regular expression, that when passed through a program like the stream editor called [FONT="System"]sed[/FONT], replaces the first discovered instance of the word "hackers" with the word "crackers".  Regular expressions are endemic in UNIX-like operating system programs and programming languages.

The implication is that the use of "hackers" to describe persons who generate and/or distribute malicious code is not appropriate.  Such persons are more accurately referred to as "crackers".

You can find out more about regular expressions [here]("http://info2html.sourceforge.net/cgi-bin/info2html-demo/info2html?(sed.info.gz)Regular%2520Expressions").

That page should also be accessible by issuing the command
```
info sed
```
from the shell, moving the cursor to the line with the words *Regular Expressions*, and then pressing [ENTER]

Emacs users can also access the page with [FONT="System"]C-H i[/FONT] then scolling to the bottom and selecting [FONT="System"]sed[/FONT] then selecting [FONT="System"]Regular Expressions[/FONT]
