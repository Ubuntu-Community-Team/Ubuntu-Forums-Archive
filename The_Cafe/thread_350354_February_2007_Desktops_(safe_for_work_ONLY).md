February 2007 Desktops - Posting Guidelines

* Stay within the board rules, no matter how hot you may believe the wallpaper/screenshot to be.

* If you are going to quote, there isn't a need to quote an img-embeded picture - Just don't do it. Please just reference the post by page number/page link/etc.

* Off Topic Posts will be edited or removed.

* The desktops thread is not to be used as a means to post explicit or suggestive images (they must be safe for your kids and/or boss to see). Please help to keep the thread relevant and safe to browse for everyone.

* The moderators of this forum reserve the right to edit or remove your posts as necessary so they may comply with our Community Ruleset.

* Please use reduced-size thumbnails, where possible, out of respect for people running on lower bandwidth or screen resolutions.

* In the interests of keeping the thread on topic it is strongly urged that you format your posts in the following order:

Themes: Name/where you got it (with link would be nice).

Wallpaper: Name/where you got it (with link would be nice).

Icons: Name/where you got it (with link would be nice).

Programs: Name/where you got it (with link would be nice).

Important Note: Any image posted that is over 800 x 600 will be removed! Use a thumbnail or use a text link to your image. If your image is removed, this is why.

No offense taken. I know my machine **is** a dog, but still ... my 300Mhz Pentium II is booting Grub to desktop 6 seconds faster than your 2.4Ghz. :mrgreen:

The thing is, a server install of 6.10 boots on this in about 1:10, without any special tricks or tweaks. A base installation of Arch on the same machine will boot in less than 45 seconds without tricks or tweaks. 

If I only use Openbox and tweak everything short of recompiling, I can trim Ubuntu's boot down to around 61 seconds at best. But with Arch and even fewer tweaks, I can knock it down to about 36.75 seconds (Timex method ;) ). 

It's not that Ubuntu is necessarily bloated, it's just that Arch starts with *nothing*, and you build up from there. So there's no added weight. For what I've seen, Arch also gives you better control over the initial booting phases, and you can carve out unnecessary things that you don't need or don't want. (And there's that whole precompiled for i686 thing. :roll: )

So if you know your hardware, you can win back a *lot *of speed. It's worth looking in to.I'm sure if you could get your machine running with Arch, you'd probably boot in less than half the time it takes me.

@ darkhatter- That is so funny. I like the duck

@ m.musahi- The gtkrc was written for the xfce-engine, I don't know how similar the engines are, but I'm sure they must be close. I'm going to try and learn how to write for some other engines for gtk2.0 like clearlooks so others can use them, I think more people use ubuntu than xubuntu, so it might be nice if I could make a theme that works in both operating systems for multiple engines and more ubuntu/xubuntu users.

I don't know what the problem could be for the Emerald theme, if I remember correctly, you download/extract a theme to your home folder, and then import it into the Emerald Settings Manager where it says import. (I think, I haven't imported a theme since Beryl/Emerald 1.1)

If you keep having a problem with it, I can just give you the colors and percentages for the glows and sizes of the theme Minimal_Glass, and then you can just use a different theme that has the tru glass engine as the installed engine, and you could re-create it and then save it to your themes. As for the buttons they are really easy to make in GIMP (or use the nice ones from moonglow).

For Firefox, what I did was unpack the jar file of Cylence:Obsidian (you have to have the theme Cylence:Obsidian installed), and replace all of the Default Classic theme's .png and .gif images with the black ones. You can find your jar files for your themes and extensions in your home's /.mozilla/firefox/th11a80d.default/extensions/{34c409b0-936e-11db-b606-0800200c9a66}/chrome 

That long number is actually the folder name for Cylence:Obsidian

You can find all of your themes there and decide which icons you want to substitute. What I have done is to back up the original classic theme, then I copy it and replace the icons with my new ones so I can easily change my "modified Default theme" by just changing the classic folder with my "new version classic" folder (still named classic).

This is where your Default classic theme is: /usr/lib/firefox/chrome/classic/skin/classic

(back this up before you start with this):

```
blah@blah-blah:~$ cd /usr/lib/firefox/chrome/classic/skin
blah@blah-blah:/usr/lib/firefox/chrome/classic/skin$ ls
classic
blah@blah-blah:/usr/lib/firefox/chrome/classic/skin$ cp -r classic ~/
```

Then just open a folder with your newly unpacked icon themes from Cylence:Obsidian (or where ever), and then next to it in a different File manager, open up /usr/lib/firefox/chrome/classic/skin/classic and start with one of the 5 folders there (start with browser) 

So open the browser folder up and look through it.

Now in the browser folder you will see Toolbar.png and a list of other .png and .gif images. Just start at the top and go down the list and replace every single .png and .gif using copy and paste from your unpacked Cylence folder. Then do that in every folder and sub-folder of each of the 5 folders that are 1)browser 2) communicator (don't worry about that one) 3) global 4) help 5) mozapps

I'm sure there is an easier way but I couldn't find it, so I just did this instead, since I saw that the images for toolbar icons were a matching 24 x 24 pixels (each) and the themes all had the exact same icons as the default's icon theme, so you don't really need the .css.

I tried to find info about doing this with a userCrome.css but I could't find anything about toolbar icons.

And when You're done, I usually make another copy of the classic theme (this time it's the new version) and then I rename it and keep it in a new folder with my other renamed classic skins like classic_diamond, classic_obsidian,  and classic_saphire, since I really like the Cylence icons.

Sorry for writing a novel.
