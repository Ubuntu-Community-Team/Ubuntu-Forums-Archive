### Post by BoneKracker on 2010-06-02
> **yester64 said:**
> is wmii still developed, or how does it look in generell with support. xcfe is in development and it look kinda cute, that why i chose it.
Xfce is a good way to lighten your load while maintaining as much freedesktop.org (and Gnome) compatibility as possible

e17 is actually quite easy on resources as well, and very pretty (although it seems to be in a permanent "alpha" status).

All the *box variants are good choices for people who want a light environment but would like to be able to customize a pretty desktop.

For advanced users who are keyboard-centric and more interested in productivity than prettiness, a tiling window manager is a good choice.  The best ones (in my opinion) are Xmonad and Awesome.  Xmonad forces you to use Haskell.  I believe Awesome forces you to use Lua.  The most innovative is wmii; a nice thing about wmii is that you can customize it in your scripting language of choice.  The fastest/most minimal is dwm; you can script it in your language of choice, but you can also customize the very guts of it easily if you know a bit of C.

If you've never used anything but a "Desktop Environment" (i.e. Gnome, KDE, Xfce4, e17), then the first time you try using a Window Manager, it's probably best to go with something like Fluxbox, Openbox, etc.  When you've got that figured out, then you can add the additional confusion of "tiling".

One of the nice things about X is that you can switch between different graphical environments on the same system.  However, if you like using a Window Manager and really want to realize its "load-lightening" benefits, then you should start with a minimal install and build up from there (or at least uninstall all that Gnome/KDE crap).

Dwm and wmii are part of this project:
[http://suckless.org/](http://suckless.org/)
