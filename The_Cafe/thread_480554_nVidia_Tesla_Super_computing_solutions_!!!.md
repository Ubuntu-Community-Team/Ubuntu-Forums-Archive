### Post by markitect on 2007-06-21
I have some reservations about this being of much scientific use.  While I acknowledge that folding at home has done very well with the gpu usage, i think most scientific projects, particularly those numerical in nature won't be able to use this.  While it crunches through floating point ops in a hurry, at what precision does it do it in? If they are only running 24bit floating points there are some serious accuracy issues compared to every platform that has come out since the 486.  I'm sure some people will be ok with those approximations, but it will impose the need for multistage calculations to have enough precision for a lot of scientific work.  Think about super pie, but now you have less precision so each of the steps takes longer to calculate.  Now 128 processing cores makes up for that, but i think for some things 8 or 16 64bit processing cores will still take the pie. (yes thats a really bad pun)  On the other hand for massive movie rendering and things like that where nobody will notice if you number is wrong this will be great.
