### Post by Tomosaur on 2007-05-30
By dropping support, you encourage people to upgrade. In a world where your browser is quite possibly the most used, and most important piece of software you're likely to run, keeping it up to date is very important. If everyone just ran old versions of browsers all the time, then:

1) You'd be open to security risks which have been fixed in later versions.
2) You are 'left behind' in terms of features etc.
3) You are incompatible with everyone else.

I'd say the most important is number 1. Since the primary reason to use Firefox is to connect to a network, it is therefore important for you to watch out for security risks etc. Aside from making sure YOU are secure, it is also possible for your own unawareness to cause problems for other people. Linux machines may not  be infected by Windows virus, for example, but it's certainly possible for a Windows virus to use a Linux machine as a bridge between two Windows machines. If everyone just used old software with well known security holes, then even old viruses would still be prevelant. It's important for a majority of people to keep up to date, to limit the damage all the old duffers do without realising it :P

I'm afraid short cycles are just the nature of the beast. The whole software / computer industry moves so fast that what is bleeding edge today is old hat in a week's time. In any case - only supporting one version of Firefox is probably the best way to ensure that the product is the best it can be. Microsoft support old technology for far too long. In fact, many of the most common complaints about Windows stem from the fact that Microsoft made a conscious decision to allow backwards compatibility, because they were tied into supporting old versions. With Firefox, it's your problem if something goes awry and you're using an old version. There's no excuse for not upgrading a free product. With software you've paid for, you can normally expect that product to be supported for a while, even if new versions become available. This requirement to support old versions can really slow down development, particularly if new technology becomes available, or when standards change (and standards are a BIG, BIG thing in web browsers). If Mozilla had to support old versions of Firefox, then we'd have more bugs, more dodgily rendered web-pages due to standard conflicts, and, of course, it'd take longer for these problems to get ironed out.

In any case, upgrading Firefox is hardly brain surgery.
