### Post by kk0sse54 on 2009-08-21
> **Grifulkin said:**
> What exactly is pkgsrc I wikipediaed it and it said it was some sort of package manager that works with just the source code?  So kind of like APT?  But it doesn't find the dependencies right?

Pkgsrc is the package management system of the NetBSD operating system. It has a ports like source tree that enables you to compile all your software or you can install binary packages. If you want the most up to date and largest selection available (WIP branch) you need to compile everything. Like NetBSD, pkgsrc focuses on portability thus allowing it to run on a number of different platforms including linux or specifically in this case, Slackware. The only relation it has to APT is that they are both package managers other wise they are completely different.

########## cut here - deb2tgz #################################

#!/bin/sh
directory=`echo $@ | sed -e "s/\.deb//"`
mkdir $directory
cd $directory
ar x ../$@
tar vxf data.tar.gz
tar vxf control.tar.gz
mkdir install
cat control > install/slack-desc
cat postinst > install/doinst.sh
rm *.tar.gz
rm control postinst debian-binary md5sums
tar cfvz ../$directory.tgz *
###################  cut here  #####################
