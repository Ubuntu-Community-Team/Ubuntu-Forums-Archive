### Post by rg4w on 2011-04-16
> **polardude1983 said:**
> So the GPL license basically allows you to use, change, share whatever it is. But isn't this also what Free means when talking about linux? I know free also means the freedom to choose.

Also the word Open Source, means that the source code is available, but also Open Source means the right to sell it, or give it away too?
The freedoms of the Free Software movement have been defined by Richard Stallman as:

[LIST]
[*]Freedom 0: The freedom to run the program for any purpose.
[*]Freedom 1: The freedom to study how the program works, and change it to make it do what you wish.
[*]Freedom 2: The freedom to redistribute copies so you can help your neighbor.
[*]Freedom 3: The freedom to improve the program, and release your  improvements (and modified versions in general) to the public, so that  the whole community benefits.
[/LIST]
So yes, making the source available is a key requirement of the GPL.  Choice is not addressed by the Four Freedoms, suggesting that perhaps it's self-evident that you can choose to participate or not, to make, use, or modify free software if you choose, or choose to use proprietary software if it does what you need (which I like to think of as the Fifth Freedom <g>).

### Post by undecim on 2011-04-17
> **Ctrl-Alt-F1 said:**
> Close.  Free means that you can sell it or give it away.  Open Source just means that the source code is available.  Sometimes it also Open Source projects are also contributed to by the community, which in my opinion makes them even better than if a single entity controls them.

"Free" and "Open Source" are generally considered the same thing.

That the software is free means the same as that a person is free.

For software to be free, users must be able to modify it without restriction or obfuscation, which implies that it's open source. Also, anyone can use it (i.e., the creator can't say "only US citizens can use it" or "US citizens can't use it")

The reason the phrase "open source" was created is that "free" is ambiguous, and during the early days of Linux, CEOs and other suits needed to be convinced that it was worth using in their companies.

Free Software and Open Source Software are two names for the same thing, though many people prefer to use (and even argue vigorously in favor for) one or the other, because of what each emphasizes.
