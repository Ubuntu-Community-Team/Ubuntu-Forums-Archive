### Post by Xavieran on 2008-01-17
INTRODUCTION

wbar is a simple quick-launch bar written in C++...it is intended to 
be used in FluxBox or similar,but can (with a little coercion) be used in Gnome or KDE...
wbar interfaces directly with the X Server in order to not go through many layers 
of processing and saves a heck of a lot of speed this way!

In this how to I will show you how to set it up and configure it...

INSTALLATION

First things first :)

You will need the build-essentials package, if you don't already have it and also imlib2-dev
which is one of wbars (few) dependencies...
```
sudo apt-get install build-essentials imlib2-dev
```
You will also (obviously) need the source code for wbar...
```
wget http://www.tecapli.com.ar/warlock/wbar-1.3.3.tbz2
```
wget will download the wbar .tar.bz2 into the directory you are currently in...

Then execute ```
tar -xjvf wbar-1.3.3.tbz2 && cd wbar-1.3.3
```
This will decompress the tar.bz2 file into your home folder ,the && 
means that if the previous command was successful,execute the next command,
in this case cd...

You are now in the source directory of wbar.
With most programs you should type in ./configure to see if you have 
satisfied all dependencies, but with wbar there is currently no configure 
file(I guess it doesn't really need one because it's only main dependency is imlib2...)
Type in ```
make
```
You should see some nice code go down the terminal window 
and assuming nothing goes wrong you can now install it with 
```
sudo make install
```

CONFIGURATION

Now that we have installed it,you would think that it's all over.
It isn't.
We now have to create a config file to hold our settings for our launchers.
The first thing to do is to copy the default icons into a separate folder...I made 
one called .wbar (the dot will make it hidden) and then made a folder inside that 
called Icons...

Making a config file is pretty easy,we just have to follow a few syntax rules 
given to us by the author...
So the very first launcher definition must be for the background image:

i: /home/$HOME/.wbar/Icons/backgroundosx.png
c: --
t: /usr/share/fonts/truetype/freefont/FreeMonoBold.ttf/8

i: Gets the image to use for the background...
c: There is no command,so we put --
t: The location of a TTF (TrueType Font) and the size

Now that we have that done let's make a quick launcher for,say Firefox...
Thes syntax is nearly the same...

i: /the/path/to/your/icon
c: The command to be executed
t: Any text you want displayed when you hover the mouse over an icon


So for Firefox we could make:

i: /home/your-username/.wbar/Icons/firefox.png
c: firefox
t: "Web Browser"

So,that's our config file done,now lets start with the fun stuff!
To execute wbar you run the command ```
wbar
```
But we still need to append a few arguments to tell wbar what to do...

This is what each argument does:

	-filter 
	What type of colour filtering to use...
		0:None 
		1:Icon hovered over 
		2:All icons other than the icon hovered over 
		3:All icons

	-fc 
	The colour of the filter to be applied...0xAlphaAlphaRedRedGreenGreenBlueBlue
	If you use the GIMP you can use the colour chooser included to get the html notation
	for a colour you like and then append it after 0xAlphaAlpha...

	-bpress 
	Icons get "pressed" when clicked

	-zoomf 
	How much the icons will zoom in when hovered over...

	-jumpf 
	How much the icons should "jump out" when hovered over...

	-pos 
	The position of the bar...left,right,bottom,top and center...
	Note,You will probably want a vertical bar if you choose to the 
	left or right...

	-vbar
	Makes the bar vertical

	-balfa 
	The bars normal transparency


	-falfa
	The bars transparency when hovered over

	-config
	The location of your config file

	-idist
	The distance between the icons

	-isize
	The size of the icons

	-dblclk
	Not entirely sure what this is for...

	-nanim
	The number of icons to be animated

	-nofont
	No text rendering is done...
	This is useful if you have a vertical bar
	and the text falls off the edges of the bar

	-above-desk
	This is used for Gnome or KDE
	Note,I haven't gotten wbar to work 
	properly with Gnome or KDE...
	I had to launch it from a terminal
	and it wouldn't show if I added it to Sessions

So the command I use is ```
wbar -filter 1 -fc 0xf065dcc7 -bpress -zoomf 1.5 -jumpf 1.5 -pos left -vbar -balfa 100 -falfa 80 -config $home/.wbar/conf
```

There are *heaps* of different combinations that you can use, and I have barely 
touched the tip of the iceberg...try out different settings with the arguments
and mainly,have fun...
