Nice comments.

As for the not installing in Debian, you must understand Debian. Ian Murdock specifically mentioned Sarge (testing) (which they are trying to release). However, Ubuntu has never tracked Sarge. It tracks Sid (unstable), where packages are uploaded to. (I think they actually go through experimental first, but I digress). Anway, the point of this is that Ubuntu has never had compatiblity with Debian Sarge only Sid and that there are packages on Sid which won't install on Sarge.

This brings us to today, where, after every Ubuntu release, the next version of Ubuntu (this one is called Breezy Badger), syncs up to Sid via an automated process. 

Is that information make it more clear?
