### Post by tiglionabbit on 2008-07-03
When I go to ubuntu.com, I have to choose a mirror to download it from!  That's silly.  Places like SourceForge.net have had mirror balancers for years.  How much work would it take to implement something similar on ubuntu.com?

benefits:
1) takes away a meaningless choice the user shouldn't have to make
2) results in a generally faster download speed, since all the servers would be balanced

Now, I've never really done something like this before, but [balancer]("http://sourceforge.net/project/showfiles.php?group_id=6355") sort of looks like a solution.
