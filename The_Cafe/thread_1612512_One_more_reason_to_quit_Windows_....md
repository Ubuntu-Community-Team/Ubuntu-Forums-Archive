Today I stumbled to find a strange bug in the windows calculator. A google search just told me that this bug is well existent... 

But I think its the most awful error of Microsoft. It shows 0 = 4.74665 ...

We all know that,

[IMG]http://latex.codecogs.com/gif.latex?%5Csqrt%7B%2810%5E%7B37%7D%29%5E2%7D%20-%2010%5E%7B37%7D%20=%200[/IMG]

[IMG]http://latex.codecogs.com/gif.latex?%5Csqrt%7B%2810%5E%7B37%7D%29%5E2%7D%20-%2010%5E%7B37%7D%20=%204.7466522942022786223711731575919[/IMG]

Using translative property (if a=b, and a=c, then b=c), one sees that

[IMG]http://latex.codecogs.com/gif.latex?0&space;=&space;4.7466522942022786223711731575919[/IMG]

Note that we have not used any undefined operations here. 

This is a known bug, and I googled to find a youtube video.
[Link]("http://www.youtube.com/watch?v=oBNyBhzu8zI")

To be noted here is that Linux does not produce this bug.
