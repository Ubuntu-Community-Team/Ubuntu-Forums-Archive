### Post by beccatoria on 2010-01-15
I'll be honest, I'm slightly confused about the premise of this poll, although I did vote.  
 
If it's about lobbying closed-source companies to support Ubuntu specifically or GNU/Linux in general, then I am not against this in principle.  I am very supportive of the philosophies of FOSS and indeed very much respect Ubuntu's decision to ship only with with open software, allowing the individual to make any decisions beyond that point.  
 
That said, there is space for paid software in GNU/Linux.  I don't have any moral qualms with paid programs existing for the GNU/Linux and don't see how that would compromise its goals as an open source operating system.  It will still exist as such, and the FOSS software movement will still exist.  If Photoshop were available for purchase for Ubuntu, GIMP would still exist (and indeed it is available for non-open OSes), it would still have its advocates and its detractors, and I would still choose to install it over Photoshop for both philosophical and financial reasons.  
 
So, I have no problems with Ubuntu lobbying companies to provide Linux support so that its users then have the option of purchasing a variety of programs that would otherwise be denied them.  I do not think that the argument here is that they should be installed in Ubuntu by default (nor would I support that), or even that Canonical should pay to license these programs so that they can be made available "free" to the users.  But I admit I'm a little confused about the premise of this issue.  
 
And...as I said, I probably wouldn't really use any of these programs, if only because I couldn't afford them.  In general, when there is no alternative program available for Ubuntu, I would prefer a FOSS equivalent to a paid-for port of the original.  
 
I would also note that I would be against including programs such as Hulu by default because it does not work outside of the US.  Put it in the repositories, by all means, but I'd be really annoyed to lose space and have it cluttering my OS when I can't even use it.  
 
Somewhat more usefully, I do actually have an opinion on what software should be included in the next Ubuntu default installation.  
 
I was very excited to see that PiTiVi was being included by default in the new Lynx alpha release, because Ubuntu really does need to ship with a video editor installed by default.  
 
That said, I can't understand why PiTiVi was the choice made.  I sincerely hope that a new release of PiTiVi will be available when 10.04 goes live because currently it does not even have any effects or transitions.  It is a very promising project, but is NOT currently anything like an ideal simple, basic video editing program.  
 
Personally, I use Cinelerra, but like the decision not to include GIMP, Cinelerra is far too complicated for the average beginner (although I would very much like to see it included in the Software Centre rather than as a third-party ppa).  
 
But I also have OpenShot installed and think it is a very nice piece of software.  Certainly it is no more complicated than PiTiVi and has vastly superior functionality.  There are also a few other basic video editors that at least offer basic effects and transitions (Kdenlive and Open Movie Editor to name two).  
 
My vote is for OpenShot rather than PiTiVi, but really, ANYTHING would be better than PiTiVi, unless the alternative is no video editor at all.

I realize that many people don't share my opinion on this issue, but there are also other reasons I feel like adopting proprietary software for use in the Ubuntu platform is a bad idea. Perhaps the best reason is that it "locks" you into using more software from the same vendor. A good example is Microsoft Office. Microsoft recently devised a new file format for its popular productivity suite. As it is a secret format, only Microsoft Office can read these files. Therefore, people who use Microsoft Office are stuck using it, unless they want to toss all their old files, which is not usually an option. This gives Microsoft a ton of control over its users.

Another reason is that without being able to look at the source, there is no way to tell that vendors aren't putting a back door in the software, compromising users' security. It gives vendors a way to monitor everything you do and force your computer to do things without you knowing. With free, open source software, even if you don't have the skills necessary to read source code and look for security flaws, somebody else can and most likely has.

Ubuntu's philosophy toward free and proprietary software has been one of my favorite reasons for using it. If the developers start adding proprietary software, beyond what is necessary, to the repositories, I may have to reconsider my choice of operating systems.

I've read through the past few pages since I last posted in this thread, and the comments are puzzling.  It strikes me as odd that people would prefer locking out certain applications from availability to Linux simply because they personally do not like them.  I don't see the point of restricting choice like that.  It's a real turn off for people willing to make the switch from Windows to Ubuntu/Linux.

It may be hard for some people to either understand or accept, but for a lot of people, proprietary applications are effective at getting what people need done, done.  There are plenty of open-source applications out there that are just as effective, if not more effective, but the fact that their source is available doesn't even cross the mind of the user who uses them.  A user with no programming experience is probably more likely to use a free, open-source application, because it is free, not because its source code is made available by the developers.

On Windows, you have the choice to run proprietary and open-source software side-by-side.  The same is mostly true for OS X, as well.  You can run Internet Explorer or Firefox.  You can run Adobe Photoshop or GIMP.  You can run MSN or Pidgin.  For almost every major proprietary application on Windows, there is often a free, open-source alternative, which can meet some, most, or even all of a user's needs without cost or proprietarity (that's so not a real word).

When users attempt to switch to free or open-source Linux-based operating systems, such as Ubuntu, they aren't afforded the choice to run the same software they can run on Windows and/or OS X.  They do not have access to the proprietary software they are used to; they simply have to use the open-source alternatives given to them.  Adobe Photoshop isn't available.  iTunes isn't available.  Spotify isn't available.

Let's face it:  Ubuntu has never been and will never be a completely open-source/free software operating system.  There will always be proprietary applications that run on Ubuntu (ubuntu-restricted-extras comes to mind, as well as others that are included in the repositories by default).  If you don't like the idea of having proprietary software included side-by-side with free or open-source software, you don't have to use it.  No one is forcing you to use Ubuntu or any proprietary software that may or may not be included with it or available to it.  There are plenty of Ubuntu derivatives (e.g., gNewSense) that do not include proprietary software at all.  There are plenty of non-Ubuntu distributions that do not include proprietary software at all.  You can even *make* a new distribution that does not include proprietary software.  That's the beauty of Linux: you can make it almost completely what you want it to be.

You cannot say that you are offering users freedom by giving them a free operating system and then forcing them, by lack of choice, to use only open-source applications.  Software should be about freedom *of* choice, not freedom *from *choice.  One user should not be given the power to make a decision for another user.  It is up to the user him/herself to decide what software he/she wants to use, how much money they want to pay for it, and why they want to use it in the first place.
