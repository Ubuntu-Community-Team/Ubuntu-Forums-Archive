### Post by MisterGaribaldi on 2012-03-31
I know this was probably intended as a light-hearted thread, but I'm going to jump on my soap box for a second.

Competition is a very healthy thing because it helps to promote excellence within a given market, or a given "area". Too many of the programs for Linux look like amateur-hour efforts (Gimp, F-Spot, Scribus, various CD/DVD burning apps, some of the media apps, for example) and, IMNSHO, that's the direct result of the myopic "written to scratch my own itch" view many (most?) Linux / F/OSS devs seem to have of software development.

And if this sort of "trash talk" that's going on as described by the OP pisses off the Brasero project members enough to get them to improve their product, then so much the better.

I've said this before, elsewhere: we have to attract more and better developers to Linux, and we also need to attract professional software developers to Linux (beyond those who already are here) if Linux-on-the-desktop is to ever gain traction and relevance out there, which is to say that it becomes a legitimate competitor to both Microsoft and Apple.
