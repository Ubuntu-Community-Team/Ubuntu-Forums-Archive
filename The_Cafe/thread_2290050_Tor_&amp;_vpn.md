### Post by Welly Wu on 2015-08-09
Here are some do's and don'ts for Private Internet Access VPN:

DO:
Download and install the Private Internet Access GNU/Linux desktop client for advanced cryptography and hashing algorithms along with key exchange and use it

DON'T:
Don't use an ecryptfs encrypted /home folder or partition with the PIA VPN GNU/Linux desktop client. It won't connect to PIA VPN gateway servers in this unsupported configuration. You will need to backup your private user data within your /home folder or partition and delete it and create a non-encrypted one to replace it in order to get the PIA VPN GNU/Linux desktop client to connect to a VPN gateway server. Or, you can re-install the latest version of Ubuntu 64 bit GNU/Linux and choose to enable full-disk encryption WITHOUT ALSO encrypting your /home folder or partition.

Don't use sudo chown -R yourusername:yourusername /home/yourusername. Don't use sudo chmod -R 700 /home/yourusername. This will break compatibility with the PIA VPN GNU/Linux desktop client and it will not connect to a VPN gateway server. Choose the default Ubuntu ownership and permissions structure for your /home folder or partition to get the PIA VPN GNU/Linux desktop client to just work right out of the box. This may require you to reinstall Ubuntu 64 bit GNU/Linux bare metal from scratch, but it is worth it.

That's it.
