### Post by Mr. Picklesworth on 2008-07-31
> **waapwoop1 said:**
> why don't people like evolution?
It just receives and sends email with a few extra features, what is wrong with that?

Because those few extra features mean that, in doing so, it is remarkably slow on otherwise zippy computers. Its interface looks and feels like it was cobbled together in the 80s, and it sees fit to duplicate the design blunders of Outlook such as having sixty different ways to file email. Flags, tags, categories, folders, filters, search folders, mark for followup, convert to task... What The Heck?!
Now, I can see that work is being done to comply with GNOME's HIG and to look / feel more like a normal desktop application, but there is still a ton of work to be done. It still is beyond me why email applications can't use GTK's List View widget, instead employing their own really ugly looking list views that behave unlike any others...

Now, the backend of Evolution - the data server we see on nice suites like Pimlico - is excellent. It's great to have that tool to create well-integrated desktops. I look forward to the day someone builds a desktop-friendly Evolution shell more suited for home use instead of for businesses that are infatuated with Microsoft Outlook.

Having said that, I for one do use Evolution right now because it gives me tools to organize the many mailing lists I use via search folders.
I think someone could design a pretty great email program if it was to focus entirely on mailing lists, for example separating list mail from regular mail and automatically creating folders for the different list subscriptions. Would be nice if bulk messages like those generated from Launchpad teams subscribed to bugs were filtered; they have a nice header for the job, but actually getting said header and removing it from search results is a bizarrely complex task in Evolution right now.

Sorry... I'm, err, cranky.
