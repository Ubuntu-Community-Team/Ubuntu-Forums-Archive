### Post by fluffman86 on 2009-11-29
False.  Now where's my cookie?

Really, though, Linux (the kernel) does indeed support a *HUUUUUUGE* amount of hardware.  In fact, if you count up sheer number of devices that work (even if it's just basic functionality), then the Linux kernel is head and shoulders above what's offered in XP, still more than Win7, and miles ahead of Mac OSX (since it only supports some peripherals and their own Mac Hardware).

Remember, though, that those numbers only consider what is built into the system and comes with it at install time.  A lot of stuff simply can't be distributed with the system due to licensing restrictions.  Things like the proprietary nVidia drivers fall into this category--you have to download them later.  

It really boils down to whether the hardware manufacturer supports a given OS or not.  HP printers, for example, use Open Source, or at least somewhat standardized, printer drivers.  Lexmark uses entirely closed drivers, and their drivers don't run on linux.  So a linux developer has 2 options: either spend all of his free time writing drivers for lexmark printers, or do what you want to do and just buy an HP.

So again: 
base install -- linux wins in hardware support and included software.
additional software -- well, everybody writes their software for Windows, then Mac, then Linux.
