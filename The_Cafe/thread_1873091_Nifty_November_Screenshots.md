### Post by Copper Bezel on 2011-11-09
Depends on the theme. The CSS theming style is modular - a theme can be all in one flat file, or 15 and include whatever it wants. A well-written theme declares the selected bg color in gtk.css and refers back to it everywhere else, but not all themes are well-written.

And you have to make the same edit in the GTK2 theme's gtkrc file, or only your GTK3 apps will be affected.

[quote=Roasted]I'm seeing a lot of lighter themes up here. Anybody have a reason why they prefer what they prefer?
Contrast with the Shell overlays, for one thing. White on black seems more visually emphatic than black on white, so it looks very good with a simple UI but starts to look cluttered faster. The effect for me is that an all-dark theme with only light input areas is more attractive at a glance but not as transparent to work with.

You can split the difference, but I don't like the division in themes like Ambiance and Hope between the title and menu bars and the content area, because it adds an extra complication to the UI, even though it probably does have a readability advantage. I think they make more sense under Unity than under Shell, though, since the panel and title bar are linked. Shell's title bars, equally, have to be *distinct* from the panel, and the most readable option seems to be white on black for the panel, black on "white" for the rest.

there are some core X11 utilities that use a different kind of theme setup than normal apps.  xfontsel is one of them. xterm is another (xclock and twm are also in there).  These programs only use bitmap fonts, and must be configured through a file called ~/.Xresources (formerly .Xdefaults).  Some terminals and programs (like conky and urxvt) can use X core fonts or normal fonts, and most bitmap fonts can be invoked either way in these programs.  Calling a font as an x core font takes fewer system resources, but calling it the normal way (xft) allows your font rendering engine to automatically generate bold and italic versions of the font in question.

This is kinda random esoteric knowledge unless you're into really minimalist setups or have other reasons for using an xterm-style terminal (like URxvt, which is almost infinitely tweakable).  The info and launch bars in scrotwm, for example, could only use x core fonts, last I checked (but the latest version of dmenu and dzen apparently have xft support, so this will probably creep into scrotwm soon too, if it hasn't already).
