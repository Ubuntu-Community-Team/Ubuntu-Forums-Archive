INSERT INTO TableName (Column1, Column2) VALUES (Value1, Value2);

-- Means comment, and DROP means delete

So, it commented out the rest of the query and deleted the table called Students.

Really, making SQL funny is a challenge, and XKCD did it IMO. :p

### Post by doas777 on 2008-12-08
RiceMonster beat me too it. I should have left it to him in the first place, but 'tis my destiny to tread where I should not. 

it's a SQL injection command. the database operator enters the childs name. into the table Students. the parameter 'Robert' is expected, but instead it gets a command string, that terminates the current command with the ';' and then executes it's own command:
```

Drop Table Students

```
. That would not be good as you would no longer have a table called Students.
