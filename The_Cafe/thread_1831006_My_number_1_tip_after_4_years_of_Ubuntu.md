### Post by mrming on 2011-08-22
It took me a long time to learn this one, but here it is:

**Never install a non LTS version.**

After wasting many hours of my life trying to make my AMD64 installs work properly, it's back to LTS for good.

If you're starting out with Ubuntu (particularly with AMD64 / ATI graphics), do yourself a favour and don't be tempted by the fancy new features in the bleeding edge version. The chances are (unless you have very common hardware) that it won't work properly. Stick to the LTS version where people have already worked out the kinks.

### Post by el_koraco on 2011-08-24
> **beew said:**
> If you use so many ppas and new kernels that basically defeats the purpose of LTS.


Not really. As long as a kernel upgrade doesn't force you to upgrade gcc or something, your system doesn't change that much. The kernel-userland functions pretty much stay the same. The basic purpose of the LTS/Deb. stable variety is to have a system that will stay the same for a prolonged period of time in its core. 

Take my example. I can install kernel 2.6.39 or newer versions of X and its drivers on Squeeze with no problems. This constitutes stability. 

But, if i wanted to install kernel 3.0 and headers from Wheezy, I'd have to upgrade gcc, by pulling it in from Wheezy, along with dependencies. This constitutes instability, although it doesn't mean the system will necessarily become less stable for general use, but I might run into trouble along the way.  

The same would go for Ubuntu LTS, only instead of apt-pinning Maverick and Natty repos, you'd use ppa's.
