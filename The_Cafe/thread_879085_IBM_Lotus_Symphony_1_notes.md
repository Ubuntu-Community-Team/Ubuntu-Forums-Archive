WHAT A JOKE !!!!!

As an I.T. Professional I was hoping for some months this would be the saving grace, the holy grail - the answer to our dreams in providing or satisfying the last caveat in affording a full migration away from Microcrap. I guess I was wrong for holding my breath.

While OpenOffice is the clear front runner, the problem doesn't lie with Sun, it's the stupid ISO. So until they can get their act together, the world must suffer longer.

- App install was... well I think I can install Linux faster than this app installed.

- It required 3 clicks per dialog box in order to activate the selection - Do we have an application coding issue?
- It includes it's own JRE. Many questions about this. what's wrong with the versions everyone has already installed? Why must an app cludge up a system with a duplicate install? What happened to autodiscovery and detection?
- App never opened after install which took almost 20 min. Like I said, I can almost install Linux faster.
- I figured maybe rebooting might clear everything and get the app running. WRONG !!!!! My reboot hug before the GRUB Loader. S C A R Y !!!!!!!

Rebooted with a power cycle and got the system back up and immediately removed the install without delay. I can't say how disappointed I am with IBM for this however my experience with IBM supports the phrase - (I)v'e - (B)een - (M)islead. 

BEFORE you install the IBM Lotus Symphony app, be forewarned!!!!!!

I've never encountered such a piece of garbage in all my life and am very disappointed that a firm such as IBM even chose Java which anoune outside a developer knows is crap to begin with. It may be good for some things but it's not in this instance.
