### Post by toupeiro on 2009-10-26
Its pretty easy to determine which are behind.  Some are not behind at all.  Pick a main repository, run update manager, it tells you how many packages are available.  Now, pick a mirror, run update manager, see if that number is the same.  Once you find one that is and is fast, stick with it.  5-10 minutes of this procedure one time has saved me that much time by 100 fold easy.  On one of my systems I use the easynews repositories.  Those are about 3 days behind but they are the the fastest mirrors I've ever connected to.  I'm using a different one on my main system.  Not as fast as easynews but it appears to be same-day and I bet anywhere from 1.2Mbit to 2.7Mbit/sec
