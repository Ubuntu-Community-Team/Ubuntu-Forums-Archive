### Post by mehaga on 2009-03-12
Disclaimer: I'll give you my personal opinion and I might be wrong about pretty much anything I say :)

I had similar 'problem' about 2 years ago, and moved to .NET from Java. I hate to say this, but... I like it better in .NET world. Often easier to work with, and jobs seem to be much easier to find. You end up contributing to Windows while preaching Ubuntu, but we all have to make a living, don't we?

If you forget about single-platform nature of .NET and just look at the languages, you'll probably love C#. I really, really hate to say this, but, IMHO, it's now a much better language then Java. You will probably love C# generics, collections, closures (ha ha, so funny :D), partial classes and methods, properties, just to name a few things I believe are better implemented in C# or not (yet) available in Java. Also, there's ease of GUI building and binding to data, availability of commercial controls/components (it's very easy to fall in love with DevExpress)...

While Sun and Java community wasted years discussing things like whether next version should have closures or not, MS implemented them and moved on to working on C# 4.

What you might hate about .NET is... well, typical Microsoft stuff. They're giving their best to lock you into their platform. It will, ofc, only run in Windows. Or, you really like Linq to Sql (although it's hard thing to like once you get to know it, but that's another story), but you can only use it with Sql Server. 

There's other, purely technical stuff, not related to MS policies, I don't like. I was introduced to WF and ended up with "WTF?!". For 3-tier apps, you might end up having to develop your own application server from scratch, using tools that, IMHO, suck or are really difficult to use properly (like WCF). 

One more thing... If you move to .net, you might soon become very curious about possibilities of .net - java interoperability :)

Phew! This is probably the longest post I ever wrote :p

Good luck, whatever you decide.
