### Post by draperdt on 2010-12-17
You can open however many new terminals using Cntrl Alt t

You can switch between the terminals using Alt Tab

The top row keys are also functions keys..ala F1 Through F10
You can switch to another tty using 'Cntrl Alt F*' 

Kernel Messages are on Cntrl Alt F8 [mute key] To go back to original terminal Cntrl Alt F1 [back arrow key]

You can enter shell with out logging into chrome aswell.

When logged in the browser and hit Cntl Alt t, the default shell is crosh

To get to bash, type shell

There is no password for sudo which you have to assign

### Post by libssd on 2010-12-20
People may be reading too much into the selection process; at least one forum has suggested that Google is selecting by zipcode. Think about it: let's assume 100,000 people apply. Are human beings at Google going to go through 100,000 applications of 140 characters each? That's not a job I would want. With their resources, I suppose Google could throw software at the job, but 140 characters isn't much for machine-based decision making. 

Far simpler would be to select by zipcode (which is more or less population based) with a limit on how many machines go to a given zip code, or to exclude specific zip codes, such as, to use a doubtful example, Microsoft's headquarters. I'm not saying this is what Google is doing, but it seems as good an approach as any, and is eminently applicable to machine selection from a large number of entries in a short time.

> There are four types of ZIP codes: Unique (assigned to a single high-volume address), P.O.-box-only (used only for P.O. boxes at a given facility, not for any other type of delivery), Military (used to route mail for the U.S. military) and Standard (all other ZIP codes). As examples of Unique ZIP codes, certain governmental agencies, universities, businesses or buildings that receive extremely high volumes of mail have their own ZIP codes...

Despite the geographic derivation of most ZIP codes, the codes themselves do not represent geographic regions; in general, they correspond to address groups or delivery routes.
**Source**: [http://en.wikipedia.org/wiki/ZIP_code](http://en.wikipedia.org/wiki/ZIP_code)

### Post by stmiller on 2010-12-30
> **libssd said:**
> Can you elaborate on this? I have an Acer netbook with a 32gb OCZ Vertex SSD. I have made a number of filesystem tweaks to minimize writing to the drive, but have done nothing re TRIM.

This is all over this forum in many other threads, but here are the basics:

ext4 only
kernel 2.6.33 or later
add the option **discard** to your /etc/fstab for the SSD to enable trim
(SSD model you have must support trim)

Here's my /etc/fstab where I have an SSD as the boot drive:
```

stmiller@brahms:~$ cat /etc/fstab 
# /etc/fstab: static file system information.
#
# Use 'blkid -o value -s UUID' to print the universally unique identifier
# for a device; this may be used with UUID= as a more robust way to name
# devices that works even if disks are added and removed. See fstab(5).
#
# <file system> <mount point>   <type>  <options>       <dump>  <pass>
proc            /proc           proc    nodev,noexec,nosuid 0       0
# / was on /dev/sdd1 during installation
UUID=58771c75-9024-4777-b992-19aaf2671a23 /               ext4   defaults,noatime,discard,errors=remount-ro 0       1
# /home was on /dev/sda1 during installation
UUID=137930f8-eadb-4aa0-be38-fc02cbea23ad /home           ext4    defaults        0       2
# swap was on /dev/sda2 during installation
UUID=54ef9b16-4d8d-404d-9317-c85dc92081dc none            swap    sw              0       0

```

The commands make use of **xvkbd** to send the keystrokes and **xbacklight** to control backlight levels as previously mentioned in this thread. They are assigned using the built-in Gnome **Keyboard Shortcuts** panel by adding custom shortcuts.

```
Browser fullscreen
/usr/bin/xvkbd -xsendevent -text "\[F11]"

Press 'Del' key
/usr/bin/xvkbd -xsendevent -text "\[Delete]"

Backlight up (note the previously mentioned bug in xbacklight needing 15 instead of 10 for incrementing)
xbacklight -inc 15

Backlight down
xbacklight -dec 10

Browser back
/usr/bin/xvkbd -xsendevent -text "\[Alt_L]\[Left]"

Browser forward
/usr/bin/xvkbd -xsendevent -text "\[Alt_L]\[Right]"

Browser refresh
/usr/bin/xvkbd -xsendevent -text "\[F5]"

Press 'Home' key
/usr/bin/xvkbd -xsendevent -text "\[Home]"

Press 'End' key
/usr/bin/xvkbd -xsendevent -text "\[End]"

Press 'Page Up' key
/usr/bin/xvkbd -xsendevent -text "\[Prior]"

Press 'Page Down' key
/usr/bin/xvkbd -xsendevent -text "\[Next]"
```

See the attached screenshot to see the button assignments I've used (note Mod4 = Chrome Search key, a.k.a. capslock), or come up with your own. ):P

When you mouse over Scratchpad on a Chrome "Apps" page, its URL appears at the bottom left, and looks something like this (I assume the actual string is tied to individual user identification, so I substituted the alphabet):

chrome-extension://abcdefghijklmnopqrstuvwxyzabcdef/html/view.html

It's a little tricky, since the type is small, and you have to enter a 32-character string (I couldn't figure out any to copy/paste), but it's possible to save this as a new bookmark. When you do, Scratchpad opens full-screen; if you right-click, you can choose to open it in a new tab or window. Woo hoo! No more resizing the tiny window. Add the bookmark to your Bookmarks bar, and it's always accessible, without having to go to the Chrome Apps page.

> I want to share my keyboard mappings which enable some of the functionality of the special Chrome keys (function keys) as well as some other useful keys that are not on the CR-48 keyboard. Browser fullscreen
/usr/bin/xvkbd -xsendevent -text "\[F11]"

Press 'Del' key
/usr/bin/xvkbd -xsendevent -text "\[Delete]"

Backlight up (note the previously mentioned bug in xbacklight needing 15 instead of 10 for incrementing)

Backlight down
xbacklight -dec 10

Browser back
/usr/bin/xvkbd -xsendevent -text "\[Alt_L]\[Left]"

Browser forward
/usr/bin/xvkbd -xsendevent -text "\[Alt_L]\[Right]"

Browser refresh
/usr/bin/xvkbd -xsendevent -text "\[F5]"

Press 'Home' key
/usr/bin/xvkbd -xsendevent -text "\[Home]"

Press 'End' key
/usr/bin/xvkbd -xsendevent -text "\[End]"

Press 'Page Up' key
/usr/bin/xvkbd -xsendevent -text "\[Prior]"

Press 'Page Down' key
/usr/bin/xvkbd -xsendevent -text "\[Next]"
```

> **xak said:**
> I want to share my keyboard mappings which enable some of the functionality of the special Chrome keys (function keys) as well as some other useful keys that are not on the CR-48 keyboard. The commands make use of **xvkbd** to send the keystrokes and **xbacklight** to control backlight levels as previously mentioned in this thread. They are assigned using the built-in Gnome **Keyboard Shortcuts** panel by adding custom shortcuts.

```
Browser fullscreen
/usr/bin/xvkbd -xsendevent -text "\[F11]"

Press 'Del' key
/usr/bin/xvkbd -xsendevent -text "\[Delete]"

Backlight up (note the previously mentioned bug in xbacklight needing 15 instead of 10 for incrementing)
xbacklight -inc 15

Backlight down
xbacklight -dec 10

Browser back
/usr/bin/xvkbd -xsendevent -text "\[Alt_L]\[Left]"

Browser forward
/usr/bin/xvkbd -xsendevent -text "\[Alt_L]\[Right]"

Browser refresh
/usr/bin/xvkbd -xsendevent -text "\[F5]"

Press 'Home' key
/usr/bin/xvkbd -xsendevent -text "\[Home]"

Press 'End' key
/usr/bin/xvkbd -xsendevent -text "\[End]"

Press 'Page Up' key
/usr/bin/xvkbd -xsendevent -text "\[Prior]"

Press 'Page Down' key
/usr/bin/xvkbd -xsendevent -text "\[Next]"
```

See the attached screenshot to see the button assignments I've used (note Mod4 = Chrome Search key, a.k.a. capslock), or come up with your own. ):P
