### Post by Tomosaur on 2007-03-09
This will be very easy to do with Linux - it's pretty much designed for this kind of thing anyway. The hardest part would be finding a big enough desk :P

X server is naturally multi-client capable. As long as you have enough RAM and processing power (and really, it doesn't need that much, all things considered), then this is not a problem at all.

EDIT:

By the way - you will still need two machines. One needs to act as a server, one as a client. The server can also be used as a client. Those pictures show multiple machines. The second machine only needs to be very basic, as it can draw resources from the server, if you set it up correctly. All you will need as a bootable network card - plugged via ethernet into the server machine, and some configuration to make the server accept connections from a LAN, and actually act as an actual server. There are tons of tutorials on the web, just google for "home linux multi client server"
