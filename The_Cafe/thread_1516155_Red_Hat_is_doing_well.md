> Red Hat reported GAAP earnings of 12 cents, up 20% from 10 cents reported in first quarter 2010. The year-over-year growth was primarily driven by strong organic revenue growth based on robust bookings and backlog, higher subscription renewals and strong results from the services business.

### Post by Dragonbite on 2010-06-23
> **Legendary_Bibo said:**
> does redhat make a desktop with a package manager that's better than Ubuntu's?

For the most Red Hat equivalent, look at CentOS, which bills itself as being 100% compatible with Red Hat RPMs and binaries.  Oracle was trying to copy-cat CentOS when they came out wiht Unbreakable Linux, but didn't realize how difficult it really is to be compatible, patched and not fork.

CentOS applications are roughly similar to Ubuntu LTS releases in version, if you compare version that come out about the same time.

Fedora will give you a more desktop-like feel and have more up-to-date (or beyond) versions but the price you pay is a greater chance at instability.

CentOS, Red Hat and Fedora use RPMs which, if I recall correctly, has recently had an upgrade.  I believe all of them use Yum, which isn't bad, really, though their GUI connection isn't as polished as Synaptic or Ubuntu Software Center.

### Post by Old_Grey_Wolf on 2010-06-23
Large companies are using RHEL a lot in their data centers. Part of the success of RHEL may be that companies like HP, IBM, etc., provide data center management software that runs on Windows Server Editions and RHEL. I can have Ubuntu running as Virtual Machines on my data center, however, the software I use to track usage, performance, uptime, user satisfaction, response time, etc., would most likely run on a Windows Server or RHEL Server.
