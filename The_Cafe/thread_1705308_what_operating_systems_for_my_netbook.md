### Post by ugm6hr on 2011-03-12
My advice... Decide what you are going to use the Netbook for first.
If the usual surfing / picture-viewing etc - then use whatever you like the interface for (and whatever boots fastest).
If you need to do actual work - Ubuntu works well.

I have a Dell Mini 9 (had for about 2 years now). I use it to surf (e.g. using it right now), skype (occasionally), but also edit images on GIMP, flow diagrams in Inkscape / OO Drawing, edit manuscricpts in OO Writer etc.

I found the Netbook interface limiting - since it is designed for 1 window to be open at a time, whereas I like to multitask, and have multiple windows open. The Compiz "expose-like" feature I find very useful. I have edited the Gnome desktop into a single lower panel, which I can manually hide from view, giving me the true full-screen appearance when necessary, but still allowing swapping between windows with "expose". Also - try Chrome/ium browser - it makes better use of the vertical space on screen than Firefox.

Given you're considering open source options - you could just try them out. I decided to make both my proper computer and netbook to have an almost identical desktop - I just find it easier. My minor edits to default desktop Ubuntu make it work with both netbook and larger screen.

Also - from 11.04, Ubuntu is no longer going to have a separate Netbook version - Canonical have decided there is no need for it.

### Post by Shining Arcanine on 2011-03-12
I assume that Canonical is upgrading Ubuntu's glibc to reflect the changes being made in Debian's unstable tree. glibc is not designed to be forward compatible so you might be better off switching to a distribution like OpenSUSE or Gentoo where you can resist the upgrade until all of the userland software has been properly patched than you would be by downgrading to an older version of glibc.

Alternatively, you can hack glibc 2.13 to remove the reverse memcpy() routines. That will not address the issue in that glibc 2.11 is the best maintained version of glibc at the moment and could introduce new issues if you make a mistake in hacking it.
