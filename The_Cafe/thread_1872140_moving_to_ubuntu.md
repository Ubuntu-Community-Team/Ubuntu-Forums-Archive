### Post by moldaviax on 2011-10-30
I think to be fair some of the problems stem from patents and other legal issues etc  which mean some facilities won't work out the box (eg dvd playing)

I transitioned to ubuntu in a day, though in fairness I had some experience of solaris so it made this slightly less intimidating (still frustrating when something doesn't work though:)

Since then I have installed ubuntu several times, mostly without problem. If I have one specific issue it is that sometimes fixes seem to require version updates, and as we know the recent non LTS versions have had a few probs. Also there occasionally seem to be regression issues leading to recommended updates breaking things - not good for the inexperienced user. 

Personally I would prefer fewer releases and more focus on stabilising the system,  perhaps through even greater contribution by Canonical engineers to some of the key projects they integrate. 

Still I am very grateful for the work done by the community on Ubuntu, and I still recommend it or other Linux distros :)

### Post by grahammechanical on 2011-10-30
Linux uses different language terms to Microsoft. It takes time to learn the differences. Likewise it would take time for someone who had never used a computer to learn the meaning of terms and how to do things. Things are equal.

Since 2007 when I started using Linux/ubuntu I have not found it buggy. 

You say:

> seems as though ubuntu has so many bugs and problems

It seems to me that problems are caused

a) by inexperienced users trying to do things that they are not yet skilled enough to do.

b) excessive modifying of the system. Changing things and then changing things back. And then finding that things do not work so well. And then Ubuntu is called buggy and problematic.

I agree with the person who spoke of the number of nuts and bolts and screws used to hold a motor vehicle together and yet it only takes one nut to break the vehicle apart - that is the nut behind the steering wheel.

c) the failure of computer equipment makers to work with the Linux kernel developers. If we used computers with Linux/Ubuntu pre-installed and configured so that all the hardware worked, then it would not seem that Ubuntu or any of the Linux distributions was buggy.

d) unreasonable expectations from some users. The user interface gets modified. Something is changed. Someone cannot work out how to do something that they were able to do with Ubuntu from 2 years ago and they say it is a bug.

I would rather the Ubuntu copyright owners spend money developing Ubuntu than spending it on advertising. Yes, there could be a billion dollar world-wide advertising campaign but then we would have to pay to get a copy of Ubuntu and we could only use it on one machine.

No thanks.

### Post by cariboo on 2011-10-30
I've been using Ubuntu since 2006, and haven't found it to be that buggy at all. The thing to do is when buying a system, choose your hardware carefully, make sure it works with Linux. The other thing to keep in mind is that the interim releases between LTS release are more experimental in nature, so expect things not to be finished, or working exactly the way they should.

If you want a trouble free experience, stick with the LTS versions, as they are around long enough for most of the bugs to be fixed.

Another thing to keep in mind is that as you get more experienced using a Linux Distribution, the problems see to get a lot smaller, and more easy to fix. Most users have been using some form of windows all their computing life, they actually never think about how long it actually took them to learn how to use it, and in many cases, they're still learning how.
