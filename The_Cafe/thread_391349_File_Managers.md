### Post by Wolki on 2007-03-26
> **daz4126 said:**
> This was the problem - can I use nautilus in different ways for different folders or do I need to use one file manager for file browsing and use nautilus exclusively for viewing rails files?


Nautilus has two modes, the one Ubuntu defaults to (called the browser mode) and the one official GNOME defaults to (called the spatial mode). In spatial mode, every Directory is its own window, and they always remember their position, size, zoom, background, and many other things. A spatial window has no toolbar, no sidebar, only a menu and a statusbar with a button that allows you to access the parent folders; it looks similar to the nautilus window you set up in that screenshot, and works like you described - that folder stays in 'sidebar' shape and on the side of the screen, other folders are unaffected (and remember their own state).

Many people hate the spatial mode since it works differently from the way they are used to, and it takes some time to understand the paradigm. I use the spatial mode, and find it very convenient, but many people do not; if used incorrectly it will lead to a mess of windows. In classic spatial mode, opening a new folder will keep the old window around, but you can close the old folder when you open the new one by using the middle instead of the left mouse button: and there is a variant with the buttons switched.

You can switch between the modes in the nautilus settings (on the 'Behavior' tab). Unfortunately you can not tell nautilus to only open a certain folder in spatial (which would completely fill your use case), but you can open a file browser window while Nautilus is in spatial mode (right click -> open in file browser). I suggest you try it out to see if it fits your work flow.

### Post by Wolki on 2007-03-26
> **daz4126 said:**
> 
It is a bit wierd to get your head round, but possibly better on balance. I want a mixture really. I like the fact that it remembers the size AND position of windows, but not so much that a new window opens each time (although the middle-click option seems a good workaround). 
It would be good if nautilus was more configurable - why only one mode or the other, black or white, I want some grey!!


The problem is that you can't have one without the other - if the window remembers its size and position, it can't open in the same window as the parent folder.

You may want to try ubuntu spatial, though - open gconf-editor and search for the "no ubuntu spatial" key somewhere in the nautilus options. Using this, the parent folder will close on opening a subfolder unless you use the middle mouse button.

Personally, I find spatial mode much more convenient after getting used to it. Some advantages are:
- Some folders contain lots of things, some only very little. With spatial, you can customize them to optimal size and position
- It's often convenient to have some folders side-by-side. If setup to correct positions, they stay that way
- I tend to use some folders very often, and in the past I often had the same folder open multiple times. WIth spatial nautilus, instead of opening the same folder more than once, it will bring that folder to the current workspace (if neccessary) and raise it. You end up with less unneeded windows without managing them yourself, and if the folder is already open it can be a good bit faster.

### Post by siddartha on 2008-02-22
*PCman* - I just installed it. It's clean and doesn't use much of the resources.
*Thunar* - it's a mess. A nice option from what I read, but in order to have it you have to install half of the XFCE system, at least this is how it is set up in the Ubuntu repositories. So I'm not even going down this road as I already have two systems (Gnome and KDE) that overlap.
*Dolphin* - nice, never really used it seriously.
*Konqueror* - wonderful. Has everything but the kitchen sink. And it surely works better than the Nautilus. In fact, although lately I used to run Gnome it was quite an irony that the best apps were in KDE. Can do thumbnailing as well as Nautilus, even better: you don't have to mess with the default install by replacing the totem backend. The only thing I like and it still missing it's the ability to label and comment on each file or directory. And this is a Nautilus only feature.
*Nautilus*. Back in the old days it was unusable. The only option was Gnome commander. Even when the Nautilus was imposed as the Gnome File Manager it was still barely usable. Now it is good. It's huge and somehow slower than the others. It needs tweaking to get to the functionality other FMs have (like movie thumbnails). Yet, it's the only one that can stick labels on the files and directories. And that does help a lot sometimes.
