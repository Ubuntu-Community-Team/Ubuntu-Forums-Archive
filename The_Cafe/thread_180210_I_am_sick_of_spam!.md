### Post by Dr. C on 2006-05-21
My suggestion is to use a statistical (sometimes known as Bayesian) spam filter. Basically you train it on what is spam and then it sends that spam to the spam / delete folder

This can be done in one of two ways

1) Use a mail client that has statistical spam filter built in or can accept a plugin. I now use Thunderbird.

or 

2) Run a spam filter as a proxy between the mail client and the POP server. I used Popfile [http://popfile.sourceforge.net/]("http://popfile.sourceforge.net/") in the past with Outlook 98 /  Windows 98. Popfile can be installed in Ubuntu using Synaptic and should work with any mail client.
