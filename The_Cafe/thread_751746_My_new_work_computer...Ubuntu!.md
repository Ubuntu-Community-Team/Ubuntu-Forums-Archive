### Post by tact on 2008-04-10
The  company I work for (small multinational operation) is pretty much all microsoft based clients and servers.  I am based in KL office and HO is in NZ.  

One day after (yet another) disastrous XP operating system crash on my office issued laptop I tried to get access to the data on the HDD using a few different windows tools (boot disk, rescue disk, even BartPE boot disk and several rescue tools) and all of them reported the drive as "unpartitioned space".  

I booted ubuntu liveCD and could mount the NTFS partition AND connect to the corporate network.  So I was not only able to recover all my data (copying directly to corporate windows file server using samba etc) I was able to access corporate mail server and work on spreadsheets and word documents while the recovery was happening.  hehehe.  Nice!

Then...all data safe...  decision time:
- format the drive, reinstall XP, locate and install all hardware drivers (video, chipset, bluetooth, network, etc etc), reinstall corporate AV, reinstall Office, and more....   OR.....
- double click the "install to HDD" icon on the running ubuntu liveCD.

Given that the ubuntu option would be so much simpler and have me working much faster....(in fact could KEEP working while ubuntu installed to HDD)...  II called the head office IT department in NZ and asked permission to run linux instead of the corporately sanctioned windows platform.

They agreed.   Woohoo!   Colour me "happy!" The only condition was "you can use it, but will have to support yourself as we have no one familiar with linux to offer you company support".    No problem!  hehehe

That was over a year ago and Edgy (v6.04).  Now i am running hardy beta on the same machine and happier than ever.  :)
