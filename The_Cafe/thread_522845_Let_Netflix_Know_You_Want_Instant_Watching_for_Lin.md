### Post by directhex on 2008-12-01
> **zmjjmz said:**
> I'm pretty sure this is what's going to happen. Silverlight's DRM system needs to be implemented through Moonlight first though.

Correct. Here's how it is right now.

Moonlight supports Silverlight 1.0 (and will be released imminently). It gains media capabilities either by being compiled against a libavcodec with sufficient support, or by prompting you to download (when needed) a binary codec pack for i386 and amd64 Linux, from microsoft.com.

Netflix uses Silverlight 2.0. Yeah yeah, moving target, etc etc. 2.0 is a pretty major change from 1.0. From a programmer perspective, the big change is the use of CLI things like C# to program your content (SL1.0 is all Javascript) - and from Netflix's perspective, the big change is 2.0 supports DRM (1.0 has absolutely zero support for DRM anywhere in the API). Unless ffmpeg gains the ability to decrypt the DRM, then at this point you'll be stuck using the codec pack from MS.
