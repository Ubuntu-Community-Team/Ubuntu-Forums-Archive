### Post by BuffaloX on 2006-08-24
From what little I know, it seems like Linux has the greater potential for fast development, and has shown this many times.

Codebase for Windows, started WAY back with QDOS which is from 1979.
Microsoft bought QDOS, and renamed it to MS DOS in 1981.
But the knowledge base on which current Windows is build, can be said to start here. That is a whopping 27 years of development.

1979 QDOS             = The absolute basics for an OS.
1981 MS DOS 1.0       = Support for directories.
1985 Windows 1.03     = graphic User interface
1987 Windows 286/386  = extended memory, multiple DOS apps, 32 bit.
1990 Windows 3.0      = 32 bit now optimzed?, Multitasking (sort of)
1992 Windows 3.1      = No NOW 32 bit optimized?, True Type
1993 Windows 3.11     = No no Now 32 bit optimized?, + Network
1993 Windows NT 3.1   = Micro Kernel, SMP, 32 bit.
1994 Windows NT 3.5   = Ups .:rolleyes: 
1995 Windows 95       = Plug'n Play? + Now real Multitasking.
1996 Windows NT 4     = Plug'n Play
1998 Windows 98       = Guess What. + Fat32 + internet.
2000 Windows 2000/ME  = Roll Back, encrypting file system
2001 Windows XP       = improved stability + MS call home.
2006 Windows Vista    = Aero + DRM.

Counting from 1981 when Microsoft released MS-DOS, they have had 25 years to get to where they are today.

Linux Started 1991 Which is "only" 15 years, and has almost all Windows OS features. Those not available in Linux, is generally because of patents and closed architecture.

Many things in Windows are not really implemented by Microsoft, but by 3rd party vendors.
Linux has several tecnologies before Windows, and even some that Windows don't have yet. Like clustering,  protection from malware by design. Support of a wide range of CPUs, Realy optimized to platform. Support for large disks (for 20 years MS were constantly behind industry standard), efficient filesystem.

GNU/Linux started as a 1 person project, then gained momentum due to the GNU license, and a good project.

Microsoft started with a golden deal with IBM, and had huge resources from day 1.

Linux is already the better OS in many ways, and has used 10 years less (40% less time) to get there.

### Post by majesticturkey on 2006-08-25
It's also important to realize not how old each OS is, but how long it's been popular. Linux was just the pet tinker toy of hardcore geeks until relatively recently. MS-DOS was very popular in workplaces in the late 80s.

But, however, a lot of it has to do with the support Linux has gotten recently. For the longest time, people defined their computer by its software. When Windows crashed (and we're talking the 9x days, when Windows was just a shell, not a real OS), people would exclaim, "Oh! My computer crashed!" Here, people do define that difference between the computer crashing, and Xserver crashing.

The difference recently is that, before, people said "open up internet explorer" when they wanted you to surf the web. Now you're as likely to see Firefox (my website has a vast majority of its hits as firefox, but hey, Lynx has a few hits too). OpenOffice.org has found its way into common places - my mom's school (she's a middle-aged teacher-slash-librarian at an elementary school) has it on all their computers. And with all this open source stuff, people finally realize that there are alternatives, and these alternatives don't sacrifice anything, now that you have Linux equivalents for nearly everything, and many programs not found in Windows or Mac.

So you get more people willing to shift over. (And to return to my mom and her media center, these outdated computers the school can't afford to keep updated are very close to having Edubuntu installed..."Wow, some of this software sounds very good.") As more people shift over, unlike with Windows, you get a larger developer pool. At Microsoft, more support means bigger paychecks. Linux is not looking for paychecks, it's looking for progress. So you get development that increases in speed.

Long winded, I know, but it explains what I think is going on with OS development recently :)

### Post by ago on 2006-08-25
Linux follows an exponential path. 

user base increases => some new users are developers/contributors => more developers get involved => code gets developed faster => user base increases => ....

To be more specific though in the first phase a lot of users were developers, which lead to very fast acceleration. Nowdays probably a lot of users are not developers, but that is compensated by the fact that linux is so polished that on each imporvement far more users join the boat. These days we are attractive even for the mac crowd. Moreover non-developer users found other ways to contribute, like this community.

No company can match an exponential growth path. Nor can they compete with something they cannot buy. The only way to stop it, is via legal impediments. By restricting use of code, use of propietary formats, locking driver development, locking resellers, breaking the standards, filing patent infringments... ...all things that are in fact happening.
