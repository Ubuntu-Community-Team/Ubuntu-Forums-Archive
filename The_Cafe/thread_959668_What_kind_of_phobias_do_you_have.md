### Post by nixscripter on 2008-10-27
Generally, I am phobia free.

However, I do have one serious fear, though not as personal as many in this thread: getting handcuffed... to a windows box in my next job. Figuratively, of course.

It's not just "because windows sucks"; I can live with a lousy UI and an unstable system with a few neat tricks I've picked up by being forced to use it in the past. It's because a job means you have to do actual work, and you will spend a lot of your time during a day doing it. This leads to two specific bad things which are the cause of this fear:

1. Some unscrupulous employer could actually use manditory, in-company spyware to monitor people. API vulnerabilities in the OS are why you can't turn it off even if you have administrator access to the machine.

2. A programming job with such bad software could turn a software fault into a deliverable being late. This is always true, but it just seems like data corruption is more possible on M$ Windows than Linux, just from my experience. Microsoft Visual Studio .NET and I have fought many battles, and its thrashings of triumph have included loss of my entire project. Fortunately, it wasn't very big or for an impending deadline.

The fact that either of these things is probably quite unlikely, but I worry about them anyway, is probably what makes it even worth mentioning in this thread. I could also start ranting about Windows Domains, but I think I'd better stop here. ;-)
