### Post by Lafunamor on 2008-08-05
or you can modify the Flower.ini file
(it's in $HOME/.config/Screenlets/Flower/default)
just add this code:

```
state=1
```

you can choose a number between 0 and 8 (0 is nothing; 8 is the full flower)

It may help if you stop the flower before you edit the file and start it again, because otherwise it will overwrite your changes.

### Post by moldovanus on 2009-04-16
I have changed the source file in order to eliminate the need of 24 hours up time in order for the flower to get to the next state. I have added a method that increments the passed time at every 10 minutes and writes it in the .ini file, such that on resume the count continues from there. 

The rest of the behavior is unchanged. After 24 hours ( not necessarily continuous ) the flower will advance to the next state.

Of course, all the credit for this great screenlet and my respect go to the author RYX (aka Rico Pfaus) <ryx@ryxperience.com>. I have just made a little modification for which he probably didn't had time. 

Just open "usr/share/screenlets/Flower" as root and replace the FlowerScreenlet.py with the one from the attached archive. Then start the flower and wait for it to grow :).

Open as root from terminal:

 ```
sudo nautilus /usr/share/screenlets/Flower/ 
```



Enjoy.
