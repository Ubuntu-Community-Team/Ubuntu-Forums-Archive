It starts with ":", which tells vim you're entering a command, and ends with "q", as in quit. Not hard to remember. ":wq" means write and quit. To insert text, you hit "i", for insert. The commands really are not as obscure and impossible to remember as everyone seems to think they are. 

For example, if I type d5w, that will delete the next 5 words away from the cursor, or if I type c5w, that will delete the next 5 words, and put you in insert mode (ie, changing them. d5w = delete 5 words, c5w = change 5 words. It's really not all that hard when it comes down to it, just different (and more efficient at that) than the usual way of doing things. I'll admit though, you do have to learn about it before you use it because one can't just simply open vim with no prior knowledge and just start using it.

### Post by urukrama on 2008-11-10

**cplay or mocp**: both simple in looks, and light audio players.

**htop**: nice and simple system monitor, for when I need it. Once I discovered htop, I couldn't understand why someone would still use Gnome's very heavy system monitor...

**canto**: an rss feed reader. Clean interface, easy to set up, good documentation, actively developed.

**zsh**: an alternative shell to bash. It's autocompletion and better history searching is all I really use it for.

**dvtm**: tiling window manager for the console!

**rtorrent**: I rarely use torrents, but rtorrent is light, configurable and easy to use.

**mplayer**: (without a graphical frontend.) The lightest and one of the most versatile media players.

**alpine**: because mutt is way too complex and alpine looks so much nicer.

**nano**: I learned vim (vimtutor is really helpful), but still prefer nano for most of my simple text editing.

### Post by MaxIBoy on 2008-11-12
And bb.

I recommend running bb with the root prompt (sudo won't do.) And set the height and width so it won't default to 80*25. For my 1280*800 screen, the following set of commands works well:
```
maxtothemax@maxtothemax-laptop:~$ su
Password: 
root@maxtothemax-laptop:/home/maxtothemax# bb -width 160 -height 55 -driver linux
root@maxtothemax-laptop:/home/maxtothemax# exit
```

The reason for using the root terminal is that otherwise, the sounds cause the display to lock up.
