### Post by 23meg on 2010-09-21
More like, journalism fail. 

> **Madspyman said:**
> 
So how do you explain this?

Explain what? The application review process has nothing to do with the partner repository that's meant for sales of proprietary software; that's meant to happen via a separate process which ISVs can take up with Canonical. The review process is meant for community application developers who'd like to get their software into supported Ubuntu releases without having to wait for a certain time window and go through the Universe review processes, which requires more dedication and can be more cumbersome.

And none of this is news; the [blueprint]("https://blueprints.launchpad.net/ubuntu/+spec/community-m-post-release-app-process") for the process has been public since the last UDS, for months.

[The goal for the Maverick cycle]("https://blueprints.launchpad.net/ubuntu/+spec/foundations-maverick-buy-something") was to get the infrastructure in place to sell *one* single piece of software; that happened to be Fluendo DVD Player.
