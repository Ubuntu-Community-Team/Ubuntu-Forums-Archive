Microsoft has generally not made public statements about Wine.  However, the [Microsoft Update]("http://en.wikipedia.org/wiki/Microsoft_Update")  software will block updates to Microsoft applications running in Wine.  On February 16, 2005, Ivan Leo Puoti discovered that Microsoft had  started checking the [Windows registry]("http://en.wikipedia.org/wiki/Windows_registry") for the Wine  configuration key and would block the Windows Update for any component.  Puoti wrote, "It's ... the first time Microsoft has acknowledged the  existence of Wine."[[38]]("http://en.wikipedia.org/wiki/Wine_%28software%29#cite_note-37")
 The [Windows Genuine Advantage]("http://en.wikipedia.org/wiki/Windows_Genuine_Advantage") (WGA)  system also checks for existence of Wine registry keys. The WGA [FAQ]("http://en.wikipedia.org/wiki/FAQ") states that  WGA will not run in Wine by design, as Wine does not constitute "genuine  Windows".[[39]]("http://en.wikipedia.org/wiki/Wine_%28software%29#cite_note-38")  When WGA validation detects Wine running on the system, it will notify  users that they are running non-genuine Windows and disallow genuine  Windows downloads for that system. Despite this, some reports have  circulated of the WGA system working in Wine,[[40]]("http://en.wikipedia.org/wiki/Wine_%28software%29#cite_note-39")[[41]]("http://en.wikipedia.org/wiki/Wine_%28software%29#cite_note-40")  although this loophole has now been closed with the next WGA component  update. In the case of [Internet Explorer 7]("http://en.wikipedia.org/wiki/Internet_Explorer_7") and [Windows Media Player]("http://en.wikipedia.org/wiki/Windows_Media_Player"), Microsoft has since removed the  WGA requirements.
 **[[edit]("http://en.wikipedia.org/w/index.php?title=Wine_%28software%29&action=edit&section=12")] Security**

 Because of Wine's ability to run Windows binary code, concerns have  been raised over native Windows viruses and malware affecting Unix-like  operating systems.[[42]]("http://en.wikipedia.org/wiki/Wine_%28software%29#cite_note-41")  However, programs running in Wine are confined to the current user's  privileges, restricting any undesirable consequences. For this reason  Wine should never be run as the [superuser]("http://en.wikipedia.org/wiki/Superuser").[[43]]("http://en.wikipedia.org/wiki/Wine_%28software%29#cite_note-42")
