### Post by Gouki on 2007-01-12
Hi,

I would like to tell this community of a project I started working a couple of weeks ago.

[www.UbuntuWeblogs.org](www.UbuntuWeblogs.org)

UbuntuWeblogs is an aggregation of public weblogs written by Ubuntu GNU/Linux users. Unlike the official Planet Ubuntu, anyone who writes about Ubuntu on their 'blog' has a chance of being aggregated there too.
There really isn't much more to say. The site is powered by Planet (aka PlanetPlanet) on a shared host environment (running Debian Sarge, w00t!). We still use the default CSS, which I'm sure you'll find ugly.

Feel free to help on the web-design (=

So, if you're interested, you can use [this submit form]("http://ubuntuweblogs.org/submit.html").

Hope to hear from you guys soon!

### Post by Gouki on 2007-01-13
> **luca.b said:**
> I don't understand the "do not put the "read more" link in the rules. I often (but not always) do that to avoid unneded overflow of text. Why should it be considered spamming?

Simple. People use this to write only 4 or 5 sentences and then the user has to click the link to read the rest. This is used to increase the number of unique visitors.

A Planet is a central place to read the posts, so the visitor shouldn't had to go anywhere else to do this.

I do understand that, if you're going to write a *very* long text this can be useful, but so far, I haven't seen a post on the Planet that required the --more-- tag.

Please contact me for questions. I rarely check these boards, so a prompt reply will be given by eMail when a question comes up.
