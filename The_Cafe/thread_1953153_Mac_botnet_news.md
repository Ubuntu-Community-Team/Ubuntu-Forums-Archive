### Post by MisterGaribaldi on 2012-04-06
Long term, this may not be such a big problem.

Linux, Mac OS X, and Windows all allow you to D/L an installer from some location (web site, FTP server, etc.) and run it to install a program. But, Mac OS X and Windows have always encouraged you to do this, whereas most software in Linux gets installed through repos.

Apple and Microsoft are both in various stages of using App stores, but the thing is most people who use those manufacturers' OSs are used to this sort of thing. People getting into technology by way of Android phones, iPhones, and iPads are all used to a centralized conduit system for getting software. As time goes on, almost certainly the computer-using public, who's demographic is largely made up of those who are not tech enthusiasts and, frankly, those who just don't know any other way, will be clamoring for this "easy way" to install software. "What do you mean I have to 'go to a web site' and 'download an installer', whatever *that* means, just so I can get a program?" I think we all know this is coming.

So yes, as Apple really ramps up the App Store under Mountain Lion, and as Microsoft does the same thing for Windows apps, it's going to be a lot easier to control what software gets installed on any computer running any OS, and this is going to make it a lot harder for scam artists to get people to install software.

Just my 2¢.

### Post by Copper Bezel on 2012-04-06
Yes and no. If I want to install software, I have to find out what I'm looking for, first, so I'm already in a browser. If I happen upon (or am referred to) a corporate website, that site is meant to be a gateway to the business's products, so they might find a big one-step "Install" button on the product page just as nice as a Featured spot in the customer's app store.

Unquestionably, the app store model is a security advantage on Linux and iOS and will be on Windows as well. (It needs mention that this is a power that can be abused, too, but since most consumers simply don't know or care, it's not relevant.) However, I don't think it's inherently more "natural." And this isn't yet considering web apps, which muddy the distinctions between "navigating" and "installing" or "running" (from the consumer's perspective) even further (and which don't - yet - require Microsoft's clement approval.)

So I'd honestly expect a shift, but not a permanent one. It'll be the new thing back the other way round again as web apps become a bigger part of the ecosystem.
