### Post by etnlIcarus on 2009-01-23
- Start with a brief history of Multics, UNIX, GNU and Linux - a *brief* history. Just give the dates and notable innovations, etc. Might be good to frame it as a timeline comparison eg, "while DOS or Windows version X was being used on X hardware, ...".

- Briefly talk about the GNU philosophy but don't do the hard idealistic sell.

- Don't just mention the customisability, also mention the scalability and portability (works on more hardware than anything else, can be scaled for embedded/micro devices, old PCs or cluster super computing (eg Large Hadron Collider uses Linux)).

- Now discuss customisability for desktop application. Different *types* of window management, different DEs, different apps, tweakability, etc.

- Then return to scalability and demonstrate how easily you can strip-back all the fluff and get the system back to it's roots: a clean, fully-functional CLI.

- talk about how distribution and installation of software differs to Windows on modern *nix systems. Also discuss the user-security model which is increasingly being adopted by even MS.

- Then discuss Linux's shortcomings. Hardware compatibility often depends upon how cooperative the manufacturers are, creating commercial-equivalent software takes time and tends to be a bit behind, very little commercial software is ever ported, Linux never seems to reach a point where you're guaranteed a GUI for everything you need/want to do. Make it clear that these generally aren't massive issues but don't excuse Linux on every count, either.

- Finish with why you personally prefer Linux. Give them a more subjective insight into how you do things differently and how it makes using a PC more enjoyable for you. This might also be a good time to incorporate the community aspect of Linux usage.

### Post by tsali on 2009-01-23
You should talk about the completely fair scheduler...and, and...yeah...stay away from the GUI cuz it's for babies...and, and, and...oh yeah, emacs and vim will knock their socks off...

Having taught a couple of "build you own" classes at community college, I know the vast majority of these folks care nothing about development. Most really aren't even into tweaking...they are "follow the instructions" type folks looking to expand their horizons...but only some. Linux has so much customizability that it will overwhelm them in 30 seconds. If they're interested, they'll discover it on their own.

OR...you could talk to your instructor about how he intends to present Windows and structure your talk in a similar way to make it easy for the students to compare and contrast the two.

I think cost, relative security and software availability are strong points. You might talk about free software briefly, but don't turn it into a sermon. Point out how to contact the local LUG. Answer questions about hardware support honestly.

Above all, DO NOT BASH WINDOWS! You'll come across as a zealot and lose credibility.
