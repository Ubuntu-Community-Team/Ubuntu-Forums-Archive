Essentially, the original expression becomes 48/2*12 after performing the operation inside the grouping symbols.

Now, what do you do next? Does it matter?

Is (48/2)*12 = 48/(2*12)? NO. The reason is the associative property (changing the grouping) does not apply with different operations (mult and div). (And, if there were any, you would then do any additions and subtractions in order from left to right).

So you get 288. The division MUST be done first in this case, and then multiply the result by 12. There is really no ambiguity here.

### Post by Paddy Landau on 2011-04-09
The rules are very simple, as others have pointed out. Starting with 48 ÷ 2(9 + 3)...

[LIST=1]
[*]Make assumptions explicit:
48 ÷ 2 x (9 + 3)
[*]Evaluate parentheses first:
48 ÷ 2 x 12
[*]Do higher priority items first. In this case, multiplication and division have equal priority, so no change:
48 ÷ 2 x 12
[*]Where items have equal priority, go left to right (except for unary operators, of which we have none):
( 48 ÷ 2 ) x 12
[*]Parentheses first:
24 x 12
[*]Solve:
288
[/LIST]
To those who say it should be 2, it would have been very different if the equation had a + instead of a ÷, because x has a higher priority than +:
48 + 2(9 + 3) = 48 + 2 x (9 + 3) = 48 + 2 x 12 = 48 + 24 = 72

That should make it clear.

When you distribute, you must distribute the *entire quantity*, not just one of its factors. If you want to use the distributive property in this problem then you should do it like so:

48÷2(9+3) =
48/2(9+3) =
(9*48/2+3*48/2) =

432/2+3*48/2 =
216+3*48/2 =
216+144/2 =
216+72 =
288

Stated more concisely, **48÷2**, or **48/2**, is a quantity unto itself. You cannot claim to have distributed that quantity until you have distributed *both factors*, **48** and **1/2**.

Actually, I have worked it out for myself.  There is implied grouping in other situations as well.

Both addition and multiplication are commutative (also associative but it's not needed here).
(i)   A+B = B+A
(ii)  A*B + B*A
So A+B*C = B+A*C by (i)
while A+B*C = A+C*B by (ii).

We all know that the first result is false and the reason is that there is an implicit association corresponding to the order of operations.  Multiplication has a higher precedence so the two operands of the multiplication operator are implicitly grouped together: A+B*C = A+(B*C), so the result that follows from (ii) is allowed but the one following from (i) is prohibited.

This is also the reason that ax^2 is not the same as (a*x)^2.  The precedence of exponentiation means that there is an implied grouping and ax^2 = a(x^2) = (a*(x^2)).  So, I agree that arithmetic does possess certain implied grouping rules.
