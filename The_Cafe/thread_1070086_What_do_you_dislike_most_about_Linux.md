### Post by Rawit on 2009-02-14
I agree on the game support. Linux would be much more popular if there were more games that run on it. Fortunately Wine is making steady progress, and I can play most of the games I want. Enough for me to drop Windows totally, except for my work.

Hardware support is "different" then Windows. I mean, I re-installed Vista on my work laptop after it refused to run anymore... I spent quite some time installing drivers, updating, and installing drivers again, only to do it in the wrong order, and now I can do it all over again. Ubuntu and openSuSe detected all the hardware I installed it on every single time, having a fully working OS right after the install. However, when a piece of hardware isn't supported, chances are slim that it will be supported soon.

Packages for Linux are diverse. There are lot of developers working on new features, one of the reasons there are formats like RPM and DEB. Fortunately. the repositories are getting better and better, one of reasons I use Ubuntu, getting the software I want without hassle.

Bottom line is that you have to tweak a lot, edit scripts, compile and such when you have requirements that aren't supported directly. Usability goes down the drain when a problem occurs. But when everything is supported, and your system is set up as you want, it's the best OS you can use.
