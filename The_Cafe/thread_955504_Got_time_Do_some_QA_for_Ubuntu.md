### Post by Vadi on 2008-10-22
**Website**: **[SIZE="4"][http://iso.qa.ubuntu.com/]("http://iso.qa.ubuntu.com/")[/SIZE]**

**In short**: go to website, download CD or DVD, read and follow instructions for the test, say if it passed/failed, if failed report the bug. Repeat when CD images are updated.

**Original announcement**:

> Good day!

Intrepid is now in RC freeze, and we have images available to test. As
usual we'll be asking everyone on the development and QA teams to
participate in the image testing to ensure we have good test
coverage.

The procedures for testing ISO images and reporting results are
explained on [[1]("https://wiki.ubuntu.com/Testing/ISO/Procedures")] and the test schedule is on [[2]("https://wiki.ubuntu.com/Testing/ISO/Schedule")]. In short, we start
with smoke testing the daily ISO images to flush out bugs in the
installer, CD builds and fresh install instances. We then perform
comprehensive ISO image testing covering all ISOs, install methods and
test cases.

As we approach release we may need to redo the images several times and
are therefore dependant on short turn-around times on testing. The smoke
testing is already under way in the QA team while for ISO validation
testing we will require everyone's help.

Test results will be tracked on [[3]("http://iso.qa.ubuntu.com/")]. Please register if you do not
already have an account (same login as brainstorm). The tracker also has
direct links to the candidate images (and appropriate rsync commands) as
well as links to the test cases for each ISO. Please download images
ahead of time so you can refresh them quickly with rsync. You may want
to use the handy download script [[4]("https://wiki.ubuntu.com/Testing/Isoscript")].

If you have any questions about testing or result reporting stop by
#ubuntu-testing. Happy image testing!

Sarah Hobbs
(On behalf of the Ubuntu Release Team)
