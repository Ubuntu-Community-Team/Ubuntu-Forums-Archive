### Post by tyggna1 on 2007-08-24
Just to elaborate:  I am talking about a .rpm, and a .deb that comes with the software packages required--and a list of open-source packages that would be needed included on the website.  

They (Blizzard) already design their games in a format that isn't too far off from packaging.  They have a main executable--a construction tool, and then a master data file (mpq) that all their executables call from.  Then it loads the separate map files, and get the specific information from those.  All there games have followed this pattern-as it's an efficient way to run a game.  Ur-Quan masters has the same setup--hence, only about three packages and a compressed archive would be needed: The game, the config, the construction utility, and an archive of maps.

Given that, all they have to do is give an online-list of what packages the game would depend on, and that should allow the Linux user to customize his OS to run it.  We're all used to doing this by now anyway--so Blizzard wouldn't need any kind of end-user support for linux--just a support for online accounts.  Not a huge deal for them, but I think it'd be a huge deal for thousands of linux users. 


Please vote on the poll--I'd like to present some impressive numbers to Blizzard when I write a (nother) letter petitioning them to make a linux version.
