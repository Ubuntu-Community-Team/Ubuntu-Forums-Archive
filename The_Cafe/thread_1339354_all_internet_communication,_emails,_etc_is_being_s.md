### Post by red_Marvin on 2009-11-27
> **Bunnybugs said:**
> Yeah, it sure is possible. Important is that you find the right encryption code;) nothing is unhackable.
Feeling like a smartass here, not intended though.
Using a randomly generated one time pad the same size as the message makes it unbreakable. Mathematically unbreakable. As long as you manage to keep the key secret, whoever tries to break it can, if they are lucky or it is a short message, find the correct key, but they can not *know* that it is the correct key.

As an example:
the encrypted sequence (hex) *83f97d8214a62f25781ab2dab4a4dbb032*
will decrypt into
"Kill everybody!!!" if you use *c89011ee34c35940a63d0b5d0ddfa9113* as key
or into
"Kittens are cute." if *c8909f671c85c51968d7fad7d1afd51c* is used as key.

If you don't have any additional knowledge it is just as likely that either one is the correct key.
And there are 87112285931760246646623899502532662132736 different keys for that message if we assume that any ascii character might be used both as key and plaintext.
There are however reasons that this kind of encryption is not often used. To start with, a 1Gb file, would need a 1Gb key and since the encryption is symmetric, there is a vulnerability in the beginning of the communication where you somehow need to transfer the key to the intended recipient securely.

Public key encryption relies on the fact that we have no classical algorithm for computing prime factors.  About all we can do is guess and check and this is not efficient when number space gets large.  You can get lucky and get the right answer in a short period of time, but on average you will have to guess an exponentially increasing number of times as you make the length of the key longer.  This is a matter of probability and statistics and isn't something the NSA can overcome without the development of a quantum computer or the discovery of a, so far, unknown algorithm for finding prime factors.
