### Post by TheFu on 2023-03-18
I was a Starflight player in the 1990s.  That game had huge maps, 150 star systems, countless planets, many races, with everything in the game fitting onto a single 1.4MB floppy disk.  If you are smart at packing bits, anything is possible.  I lost far too many weekends for a few years playing Starflight.  Far too many. [https://linuxgameconsortium.com/starflight-3-campaign-consider-linux-support/](https://linuxgameconsortium.com/starflight-3-campaign-consider-linux-support/)  The real nag in Starflight was their DRM, even for paid customers.  Every few hours, the game would ask you to look up a word on Page X, line Y, word Z and type it in (or was it picked from a list?).  I never remember any of the questions being the same.

I also remember playing WoW as a standalone game for a few years. This was before it had any networking. It all fit on a single CDROM. It was fun. 
C&C (Command & Conquer) had similar graphics to WoW, but also had 2-player networked play.  For something like C&C, on Android I have GLWG, though it has tiny maps in comparison. The gameplay is simpler, but similar. A way to kill time waiting in an airport for a connecting flight; better than doing any work.

As soon as network play was added to any games, play maps drastically shrank.  I ran a TF server for friends to play on in the late 1990s.  This was a FPS with teams playing against each other in multiple scenarios.  Fast play was more important than graphical details. That game convinced me to get a 5.1 surround system after I had a LAN party and one of my friend's daughters snuck up behind her Dad and knifed him in the back for the win.

TF was a native Linux server.  I think the clients were all Windows, but I'd be surprised if native versions weren't ported.

I really need to spin up an OS/2 VM and see if I can get all the MS-DOS games running again.  Windows at that time was too unstable for gaming.
