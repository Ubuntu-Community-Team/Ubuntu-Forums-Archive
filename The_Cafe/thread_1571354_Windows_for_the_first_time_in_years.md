### Post by Shining Arcanine on 2010-09-10
> **garyedwardjohnston said:**
> I'm having issues installing Ubuntu (hopefully the Maverick beta will work) so have been playing around with Windows 7.

What issues are you having?

By the way, have you considered seeing if the same issues exist in another distribution like Gentoo Linux? By virtue of letting users take more control of things, Gentoo Linux tends to have better hardware support than Ubuntu because it has access to all of the same code, except in a more up to date form. If doing things yourself is a problem, you could try Sabayon, which is a Gentoo-derivative, although if you work within Sabayon's framework, you lose the benefit of being in charge of things, so hardware support probably is not much better under Sabayon than Ubuntu. If you try Gentoo, I suggest using System Rescue CD to install it. It is another Gentoo-derivative distribution, but it exists solely as a LiveCD, so it tends to do things better than the official Gentoo Linux LiveCD/LiveDVD. You also might be able to get some idea of whether or not your issue is present in Gentoo from using the System Rescue CD, because you can run a X Windows environment from it, although without knowing the exact nature of your issue, it is hard to say whether or not that would be the case.

Now go to the chipset manufacturers website and download the latest drivers direct from them. It's more up to date than that of the PC/Laptop manufacturer which usually stop updating drivers after a certain period anyway. Chipset manufacturers are companies like Intel, AMD, Realtek, Marvell, Broadcom etc
