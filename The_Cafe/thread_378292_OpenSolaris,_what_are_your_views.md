### Post by G Morgan on 2007-03-07
Very different yet similar in some respects. Firstly Solaris is meant to be ridiculously stable so as default they use the original bourne shell rather than bash (which stands for bourne again shell). The bourne shell has no coloured ls, no command history and no tab completion. It is installed by default though and you can set bash as default.

DNS doesn't work as default, you have to enable it by editing a text config file.

Also the driver model is different. They focus on a stable binary interface for drivers and distribute drivers as packages rather than in the kernel. I'm not sure how far back this compatibility goes but I managed to install Solaris 8 drivers for my NIC.

Also there are no repositories and the packages are similar to what Slackware uses. There aren't a huge variety of packages for all purposes but most things will compile from source.

On the other hand the shell uses the same tools as Linux generally.

It's a good, solid system but is not easy to use.

### Post by G Morgan on 2007-03-08
> **Somenoob said:**
> Sun is calling it the best OS in Open Source, please tell if it's on/near the Ubuntu level.

The kernel is probably better but lacks the number of device drivers. The userland can be GNU so it's equal to Linux there. The applications will all work (those that have source in any case).

The problem with Solaris is defaults and ease of configurability (Solaris 10 is at the slackware level in terms of easy of use). It is all text edit config because you will regularly run it without a GUI so are better off without fancy tools.

If someone formed a large project around it then it could be made as good as Ubuntu while maintaining it's advantages but it's no mean feat to do that.
