### Post by neymac on 2006-10-30
I am running Edgy Eft.
With Firefox 2.0 I can't open sites that start with https. With Epiphany either.
With Swiftfox 2.0 and Opera I have no problem to open such sites.

Searching wikipedia I found:
"Strictly speaking, https is not a separate protocol, but refers to the combination of a normal HTTP interaction over an encrypted Secure Sockets Layer (SSL) or Transport Layer Security (TLS) transport mechanism."

and:
"Web browsers are generally distributed with the signing certificates of major certificate authorities, so that they can verify certificates signed by them."

Then it seems to me that the Firefox installed here has no such signing certificates and as Epiphany uses things  common to Firefox it has the same problem.

How do I fix this?
Where can I get these certificates?
Or anyone knows how to fix this by other means?

PS: I am using the FF2 built by ubuntu (that with the blue icon).

============================================================================
Did this and fixed the problem:
"This means you've experimented with Firefox from Mozilla website previously. There is nothing wrong about it, but Firefox package from Mozilla.com (not the one you install or upgrade with Ubuntu) installs own libraries in /usr/lib/firefox and not in /usr/lib. Given that Edgy upgrades Firefox to newer version, the libraries residing in /usr/lib/firefox are no longer compatible. And since Ubuntu scripts are not aware of what may be in /usr/lib/firefox there are left there. What makes troubles is that libs in /usr/lib/firefox take precedence before those in /usr/lib, as you can guess it, old libs are loaded before new ones are even looked for.

To fix it, delete all files from /usr/lib/firefox and reinstall firefox from synaptic. You may leave plugins directory if you installed custom plugins before."

Thanks
