> **rai4shu2 said:**
> Bug #1: Beryl by default takes over too much of the keyboard.

I've seen work being done on Bug #2, but is anyone working on #1?

You can turn off what parts of the keyboard it takes.  I added more shortcut keys to it so I can start apps without the menu.  As far as I'm concerned, EVERYTHING should be done using the keyboard.  Mice suck.


Other than that, if you have an older, slow computer, of course it won't run Beryl well.  You should have >512mb of RAM.  Yes, 512 is the minimum.  I'd go with 784+ I have 1gb.  If you have 512 or less, it will be slow because most of your resources will go to it.  Second, if you have ATi or nVidia and are using their beta drivers to run Beryl, please note that you have beta software on top of beta drivers, multiply that and I think you'll get an alpha (assuming alpha = 1/4 done, beta = 1/2 done, release candidate =3/4, and final = 1).  If you're not using the beta drivers, well, check the boards.  There are a TON of people having issues with a bunch of other nVidia/ATi drivers.



Puppy fam, when Beryl crashes, it just switches back to Metacity, the regular GNOME window manager.  It doesn't take down the whole system, and it certainly doesn't destroy your hard drive and eat babies.  Absolute worst that will happen as far as I can figure: you have an old computer, Beryl uses up so many resources that it slows to a crawl and you need to ctrl+alt+backspace to restart X.
