Of course, if the target application was running as root, that was basically an easy way for local users and malware to gain root access. Especially in Leopard, where there were nonessential programs running setuid root all the time.

There are also problems with various locations where libraries are stored (not sure if that's been fixed).

And if we're comparing old versions of operating systems: The iPhone OS 1.0 ran everything as root. In early Mac OS X, if you ran a configuration program and then ran another program from the Apple menu, that other program would run as root. In early Safari, if you downloaded a compressed archive, Safari would automatically run any shell scripts that were inside the archive.

Mac OS X is a security shambles. Windows Vista and 7 are far superior in their security design.
