I have three browsers installed. It's overkill, yes, but that's a character flaw I'll just have to live with.

My main browser is FF. I've got it more hardened than Fort Knox and use it for most of my browsing. A close second is Links2 in graphical mode. This is a super-primitive browser that cannot even accept cookies and has no scripting ability, so it is perfect for vising pay-wall sites or unfamiliar sites that I don't entirely trust. I have recently started to explore setting up my browsers within a LXC container. This should provide an additional layer of security over what I've already got.

For me, the most important aspect of the browser is security. It is the most exposed app on most systems and by far the biggest vector by which malware infects most systems, so it's the one I spend the most time and effort hardening.

### Post by DuckHook on 2016-01-03
> **mikodo said:**
> Do you have any security tips for using Links2 and Chromium browsers and how to set them up safely as, I am not familiar with what security features I should employ with them?Hi mikodo,

You've got things pretty hardened already. Not much more I can add, actually. On FF, I assume you've got a cookie manager, Adblock plus, noscript, Better Privacy and WOT installed?

Re: Pepperflash.

Unfortunately, flash is still a necessary evil. It's on its way out, but until all significant sites go purely to HTML5 for their A/V, we end up having to live with it. I've heard varying reports about YouTube. One imagines that Google runs a tight ship for malware, but then, you read these stories about malformed video headers that inject code into unsuspecting browsers etc, and can't help but wonder. Still, if one sticks to reputable video providers, I can't see too much going wrong. I just learned that pepperflash has been deprecated and now the installation of flash couldn't be easier. Just add the Canonical Partners software source and:```
sudo apt-get install adobe-flashplugin
```The fact that I'm still on pepperflash months after it has been rendered obsolete just tells you how little I rely on it.

Links2 is also easy. Just:```
sudo apt-get install links2
```No security tips for Links2. You'll see. It's so primitive and hobbled that it is its own best defense. No scripting functions so scripts simply cannot run on it. No cookie functionality either, so ditto for tracking. No pop-up functionality, so no annoying pop-ups either. I suppose that it is still susceptible to some exotic form of attack, but it's so insignificant with such a tiny installed base, who would take a run at it?

Security tips for Chromium. That's a tough one. My use case is one where I leave that one more insecure precisely for those sites that trade security for functionality. Therefore, I'm not that familiar with hardening techniques for Chromium. That is the browser that I am considering running in a container or a VM. I could do the VM now, but have been too lazy to set it up.

Re paywalls.

Links2 doesn't work on all paywalls. Just the simpler ones. But try it out on [economist.com]("http://economist.com") and [nyt.com]("http://nyt.com") for starters. On a regular browser, it seems that these sites test for cookie and scripting functionality and will refuse to display unless you either turn them on or will keep popping up overlay screens. But on links2, these sites somehow detect the lack of cookie and scripting support (vs seeing that the support is available but turned off) and then displays its content. You won't like what you see. The quality looks like something from the Web's infancy. Like Netscape Navigator 1.0. But if all you are interested in is the content, then the substance over style philosophy fits the bill.

Last but not least:

I've tried DuckDuckGo and can't say that I'm really impressed. I tend to search for technical topics like Linux issues or HW problems, and DDG is clearly less capable than Google. Will continue to use it to see if it improves, but it's too easy to default to Google for all searches.

### Post by night_sky2 on 2016-01-08
> **shantiq said:**
> Thanx Night Sky will try that ..could I ask you to detail which features can be *safely* unclicked and a couple of words to say what they are if you have the time and inclination ...   I like to know what I am knocking out

Firefox Hello (built-in app for voice chat and video call): **loop.enabled** -> *false*

Pocket (built-in app to manage reading lists and internet articles): **browser.pocket.enabled** -> *false*

Reader View (extracts web articles and display them like on Android):** reader.parse-onload.enabled** -> *false*

WebRTC (browser-to-browser app to allow voice chat and call): **media.peerconnection.enabled** -> *false*
                                                                                      **media.peerconnection.identity.enabled** -> *false*
                                                                                                                                                                            **media.peerconnection.use_document_iceservers** -> *false*
                                                                                                                                                                            **media.peerconnection.video.enabled** -> *false*
                                                                                         
PDF Reader (built-in reader in Firefox for PDF files): **pdfjs.disable** -> *true*

Tab audio indicator (display icon on a tab when sound is playing from the website):** browser.tabs.showAudioPlayingIcon** -> [I]false

[/I]Geolocalization (allow some websites to request and pinpoint your geographic location):** geo.enabled** -> *false*


None of these features are enabled in Pale Moon. This will make Firefox quite a bit lighter too.
