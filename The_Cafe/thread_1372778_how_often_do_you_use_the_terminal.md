### Post by t0p on 2010-01-06
I use the terminal frequently.  I don't think a day passes when I *don't* use the terminal.  I always have a terminal open.

Those users who claim that the command line approach is faster are right; but so are those who say the GUI is quicker.  It all depends on what you want to do.  There are lots of single file operations that are most simply dealt with through Nautilus's right-click menu - like renaming a single file, or encrypting one.  But many operations are better dealt with through the terminal: if you want to install a package and you know its name, it is much quicker and simpler to open your terminal and type in
```
sudo apt-get install <package name>
```
than to open up Synaptic etc. Or suppose you have a bunch of files whose names are in all uppercase and you want to change them to lowercase.  You could open Nautilus then right-click on each uppercase name file, select *Rename* and type in the new lowercase name. Or you could simply type into the terminal the command
```
 rename 'y/A-Z/a-z/' *
```
Horses for courses.

As for the idea that you can't run a Linux box without the terminal: that's becoming increasingly untrue.  Maybe I *could* administer my computers solely through the GUI, I don't know.  And I don't particularly need (or want) to know.  I'm happy using the terminal.

### Post by eriktheblu on 2010-01-06
I use the terminal infrequently (couple times a week maybe), but there is no way I would give it up. This goes not only for Ubuntu, but the command prompt in MS Windows as well.

A GUI is great for simple or routine tasks if it is set up properly to do so. When I want to browse the web, I'd much rather click an icon than type a command.

However, when I want to change the permissions of my WoW folder, it's much easier to type chmod -R +x ~/games/WoW than to navigate to the folder, right click, open properties, change the permissions, check apply to subfolders, and apply. When you know exactly what has to be done, the CLI saves time and system resources.

The CLI is only limited by your system specs; the GUI is also limited by your monitor. There is no possible way everything that can be accomplished from a single terminal prompt could ever be displayed on a single monitor. GUI alternatives either have limited functionality, or bury functions deep in menus. 

Scripts make more complex routine tasks much faster especially when run from shortcuts in the GUI. One that I use sets folder permissions, opens 2 instances of WoW, starts Keyclone and g15macro.

At work on MS machines, I have batch files that copy databases, backup files, map network drives, etc. I defy anyone to show me a backup program more efficient than xcopy "k:\" "R:\Kback" /e/c/y/d.

Sort of hard to automate things with scripts/batch files unless you know Bash/DOS.
