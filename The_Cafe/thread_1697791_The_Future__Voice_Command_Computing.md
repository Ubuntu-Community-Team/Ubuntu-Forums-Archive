### Post by barryfx on 2011-03-01
[SIZE=2]The time is right for a new paradigm in computing.  Speech recognition technology has advanced to the point where we should be talking to our computers and they should be talking back.  The technology is available for at least a fixed command structure computing environment.  I would like to welcome a discussion on how it should work and how it would best be implemented technically.[/SIZE]
 
[SIZE=2]My concept is to have the computer in a room with a large screen display hanging on the wall.  An omni-directional microphone will receive spoken commands.  The computer will respond or ask questions with speech.  The computer will display requested information if appropriate.  The computer has a fixed command structure.  The option exists to use a keyboard for direct input like we do today.  The system will start in voice command mode but for keyboard input, a desktop will be displayed.  Initially single room operation is envisioned.[/SIZE]

### Post by Blutkoete on 2011-03-02
Voice Command Computing will come for things that only need short commands. In all other cases - talking is slower than typing or clicking.

"Coffee please" (or "Earl Grey, hot") might turn on the coffee machine, but for a direct computer experience just browse the web for ten minutes and say everything you do in the browser before doing it and measure the time.

What most people underestimate: Speaking is comfortable, but speaking is slow. And it's still impossible for a machine to distinguish between your husband saying "And then he said 'Coffee please' as if I'm his coffee maker!" and your husband ordering the machine to cook coffee via a simple "Coffee please.".

Even in Star Trek they typed things they wanted to be done fast. There is a big difference between speech recognition and what-I-want-you-to-do recognition.
