Beta 1 June 2006 Feature Frozen

* all feature work is complete, including UI
* all strings are complete and available for l10n to start
* intended for beta users

* based on feedback from Beta 1
* all feature work is complete, including UI

* intended for beta users

* based on bugfixes from Beta 2
* should represent final product
* intended for wide audience of early adopters 

RC 2 August 2006 Release Candidate (if needed)

* based on bugfixes from RC 1
* should represent final product
* intended for wide audience of early adopters 

FX 2 August 2006 Final Release

* based on bugfixes from RC 2

### Post by Mr. Picklesworth on 2006-07-12
Sounds good :)

A shame to lose the Places feature, but I bet someone will write an extension that does it.
A simple implementation isn't too painful, after all; it only gets troublesome if it's a built in feature, since it then needs all sorts of configurability.

In a way, I'm glad the noteable features list hasn't grown very much.
This means they've taken a lot of care to just clean up what they have.

Downloading now so that I can test it and report everything that goes wrong in the hopes of this becoming a fantastic improvement over 1.5.
One of the first things I'll check for is improved crash handling (I hate it when Firefox crashes and leaves its old unresponsive process running!).
Even better would be just not crashing when viewing Quicktime movies.


Edit:
Yep, as I suspected.
It's feeling very snappy, and the UI has really been improved.
Now I can uninstall about half of my extensions!
I'll miss the All-In-One Sidebar, though. (Maybe I can get it to let the Add-Ons window not open in the sidebar when it gets updated to FF2).

One noticeable improvement is the ability to override the default actions for certain file types.
I can now get the browser to download PDF files instead of open them in Acrobat without any extensions!
Also the search engine sorting is nice, but I had to add another Google. Made me miss an extension which gave me a point-and-click interface for adding new search engines. (All I had to do was choose a form, right click and press Add To Search Bar).
It was pretty cool :)
Hope to see it updated soon!
(Too bad something like it wasn't built in).

A few things I just can't figure out, so I'll report them as useability errors.
First, I can't tell how these RSS feed readers work.
How do I get the previews displayed within the browser content window instead of just having their titles in the menu? It seems utterly hopeless, as I can't find a button anywhere. (Should be a shortcut right below the bookmarks entry for a Live Bookmark, perhaps).
Then there's a mysterious right-click menu option that comes up when I right click on a single-line text field, saying "Add a keyword for this search". It seems to open up the Add Bookmark window with an extra option for Keyword...
I also seem to be the only person who isn't getting spellcheck out of the box.

Microsummaries seem like a good idea, but a bit complicated at the moment.
Hopefully web developers will start building their web pages with the feature so these Microsummary Generators don't have to be used.
I'm hoping they have a button for them like the Live Bookmark button...
I think bookmarks should be given two lines in the menu with the summary on one and the title on the other. Would use more space, but be quite attractive.
It will probably sort itself out nicely :)
However, one problem: Now that Search Plugins can be removed, they had to make another mysteriously unremoveable feature.
Yep, it seems that microsummary generators can not be removed. (Heh...)

I guess some quickhelp things or a quick introduction to new features on first start would improve the whole matter.

Search suggestions (not autocomplete) are concealed by the autocomplete.
I guess that's a bug, and a darn silly one at that :P
