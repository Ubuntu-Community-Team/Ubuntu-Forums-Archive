### Post by mgmiller on 2008-03-29
Mac OS/X Vs. Vista Vs. Ubuntu 7.10 at the annual CanSecWest conference.  

The Mac went first on day 2, but the exploit was discovered and kept secret months in advance of the conference, while setting up a specially rigged web site with the Safari only exploit in place.  

The Vista machine finally went on day three.  The successful hackers said they weren't expecting SP1, so it took longer.

The Ubuntu 7.10 machine remained unhacked.  Although attendees claimed they knew of some bugs in the Linux code, they didn't want to put in the work needed to exploit them.

Too bad, if they did find some exploitable code, it would have been patched very quickly by the open source community.  The winners of the Mac and Vista hacks had to sign non-disclosure agreements with Apple and Microsoft so the parent companies could "work on" fixing them.

Read about it here:

> **EdThaSlayer said:**

Ubuntu has nothing listening on any ports by default.  It does have a firewall (iptables, part of the Linux kernel) by default, but it isn't blocking anything because nothing's listening on any ports.  The way into Windows is usually through the fact that it listens on lots of ports by default.  In the case of OSX, it was a bug in Safari that allowed another port to be opened.  For Windows, the exploit went through port 80, the regular web traffic one, through Flash, so most firewalls wouldn't block it either.  Most exploits are *not* things that a firewall will protect against because they attack things like buffer overflows in already-running programs.

By Emil Protalinski | Published: March 30, 2008 - 06:42PM CT

At the PWN2OWN contest this year, hackers were given the chance to circumvent security defenses in the latest versions of Mac OS X, Windows Vista and in a distribution of Linux. ### Post by ukripper on 2008-04-11
Few basic things you can do to protect your ubuntu box from unsolicited attacks/hacks/cracks:


use nmap to scan open ports example:```
nmap localhost
```
or for your router if it is configured as 192.168.1.1 then ```
nmap 192.168.1.1
``` and check what ports are open on your router.

Keeping unnecessary port closed using iptables or with some other firewall is the key. 

Also if you have webserver make sure it is configured in DMZ which will prevent crackers reaching your internal network.

 Secure your access points and routers especially secure wireless with WPA2 and register mac addresses of wireless PC/LAPTOP (you want to use) in router settings  so no other unauthorized machine gains access through wireless.

Make your root and router passwords hard to crack.

Secure grub using this guide [http://ubuntuforums.org/showthread.php?t=715630](http://ubuntuforums.org/showthread.php?t=715630)
which will prevent anyone getting root access through grub.

Regularly monitor your router arp to see who logged in recently.

I hope this will help any one looking to secure their  ubuntu box.

EDIT:

Forgot to mention the most important point - Keep your box updated via regular updates received from your specific distro maintainer (I use ubuntu so i get updates via update-manager).

Always try to find  apps in your repos as they are trusted source and you will get regular updates in case of any security vulnerabilty specific to app.

Use firefox addon noscript to block any untrusted java or perl based scripts running in the background of untrusted website.

Use CLAMAV antivirus to scan (I do it weekly).
