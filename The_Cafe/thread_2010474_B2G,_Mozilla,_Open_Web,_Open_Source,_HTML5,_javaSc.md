The key here to avoid a "closed garden" is to have GPL v3 or AGPL v3 code buried deep enough in the operating system that it would require root access in order to modify the GPL v3 or AGPL v3 licensed components. Ubuntu meets this requirement and I suspect so do Meego and Tizen. Android does not which is it can be turned into a "closed garden"; however it does not have to be. In fact a significant impact of Ubuntu for Andriod is that it will force a rooted and hence open Android platform. I would not place Android in the same category as IOS or Windows 8 RT which by design are closed gardens. 

Cloud based solutions are a lot less free than a propriety OS. I mean what is more "free" a dumb terminal connected to a mainframe running Free Software or a PC running MS DOS and Windows 286/386? That was the choice back then and in many ways is still the choice today.
