### Post by monsterstack on 2009-05-22
Reasons to think Linux might benefit from Netbooks:[LIST]
[*]it's mostly free as in beer, cutting costs

[*]Microsoft haven't decided whether they will support ARM-based processors yet
[*]you can get more whizzbang effects and speed with Linux on crappier hardware
[/LIST]

Reasons to think Linux won't do so well:
[LIST]
[*]many places apparently refuse to stock Linux netbooks
[*]bizarrely, Linux netbooks are often priced more expensively than their Windows counterparts
[*]OEMs often package netbooks with truly awful distributions that don't work properly and don't even have proper repositories
[*]currently, Microsoft have guidelines as to how good a netbook's specs *can't be* in order to be allowed to run Windows XP. A side effect is that there is much less benefit in creating a decent, midrange netbook. It either has to be crappy to run XP, or awesome enough to run Vista. There is no reason to go in-between. Linux could fill that gap with ease.
[/LIST]

If some retailers have the balls to price their Linux netbooks honestly, don't put on crappy distros nobody wants, and if ARM-netbooks really take off, then Linux might just have a fighting chance. That's a lot of ifs.

### Post by jespdj on 2009-05-22
Linux is quickly loosing the battle on netbooks.

Microsoft has successfully pushed Windows XP forward as the OS for netbooks, and most netbooks today are sold with Windows XP. Windows 7 is also supposed to run fine on netbooks.

Linux has lost its chance to become the dominant OS on netbooks. Partly this is because Windows is the de facto default OS for almost all users so people expect to get Windows when they buy a computer, and it is also partly because some of the Linux distros used on the first generation of netbooks were not really great.

So, it's good that Linux gets attention via netbooks, but it doesn't look like it's going to be the dominant OS on netbooks anytime soon.
