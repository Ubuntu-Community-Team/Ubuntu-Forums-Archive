### Post by grahammechanical on 2016-01-22
> Does It use a specific version of Ubuntu, or the same as laptops & desktops?

It will use whatever is the latest code for a Ubuntu phone. The Ubuntu phone OS does not follow the standard Ubuntu desktop release pattern. The phone OS has transactional updates which are also called Over The Air updates (OTA) every six weeks or so. The phone OS does not use apt-get to update package by package as the Ubuntu desktop OS does. With Ubuntu phones and now tablets and in future Ubuntu Personal we should not think in terms of version numbers but in terms of OTA numbers.

I know that the Ubuntu tablet will have MIr & Unity 8 but will it have an app store with click packages? Or will it be built on snappy Ubuntu core and come with an app store that has snappy packages? This is what I am keen to find out. 

I know that to get traditional apps like Libreoffice running a snap package called Libertine is used to install deb packaged apps that run in a Linux container on xmir. So, I am guessing that this converged device will have to be built on snappy Ubuntu core.

Regards.
