### Post by kbielefe on 2009-10-23
Well, this thread has gotten a little heated, but I believe the OP's original question was sincere, and I would like to try to answer it, as someone who switched to Linux from Windows 98, and hasn't had a Windows partition at all in a decade or so.

Try to think back to the very first time you bought a digital camera.  Did it even cross your mind to wonder if there was a better camera that was only compatible with Mac or Linux?  No, that would have been pointless.  You wanted it to work with the OS you used, and picked up the best one that met all your constraints, windows compatibility being one of them.

Think back to the very first time you installed photoshop.  Did it cross your mind to research if better image editing software was available for Mac or Linux?  Of course not.  You installed the best software you could for the platform you used.  Was your photoshop then as good as photoshop is now?  Again, no.  Adobe added new features over the years to meet the needs of their customers.

Why should the experience of a long time desktop Linux user be any different?  When I bought my first digital camera, I got the best one I could that worked with Linux.  That may have been a very different choice than the best windows compatible camera, but that barely crossed my mind because I didn't use windows.

Likewise when I have needed to perform image editing.  When my wife shot her first wedding and didn't notice lens flare in around 100 outdoor shots, we didn't wonder how to fix it using photoshop, because we have always used gimp.  Just like you didn't wonder doing a similar task if it might have been easier in gimp.  That's the tool at hand so you use it.  If a feature is missing that you need, you work with the vendor to get it added.  Bonus for a software engineer on open source is that I can add it myself, and yes, I have actually done that on an occasion or two.  If I weren't a software engineer, I could have paid to have a feature added, for probably close to the same amount as a couple of license upgrades for proprietary software, except I get the exact upgrade I want.

After over a decade of using gimp, if I were to try to do something in photoshop, it would be very difficult for me.  That's not because photoshop is subpar, it's because the Linux way is deeply ingrained into this particular user.  

If you have always used something and then suddenly you can't, that's a huge loss.  If the first time you hear about something and try it out, it doesn't work for you, you dismiss it and don't give it a second thought.  We don't miss the things you would miss because we've never become attached in the first place, just like a long-time windows user wouldn't miss features unavailable on windows, like a package manager for example, that feel indispensible to us.  I don't miss windows games because I've never played any and I've been satisfied with standalone console games, it's that simple.

As an aside, I work for a small Cisco competitor, and based on requirements from customers we have in common, and their own press releases, I have a hard time believing Cisco doesn't have a fair bit of support for Linux and Unix.
