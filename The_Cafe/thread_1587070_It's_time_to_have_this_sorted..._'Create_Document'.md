### Post by bra|10n on 2010-10-02
I am referring to the 'Create Document' option, right mouse click and the lack of functionality this feature currently offers in Nautilus. Discussions in no fewer than six bug reports in the past have circled without progress and sadly at the moment activity has ceased entirely. 

[IMG]http://img826.imageshack.us/img826/4471/screenshotqmn.png[/IMG]

From memory the oldest related bug goes back as far as 2007.
In summary there seems to be a number of views on how this feature should function and what options to offer the user. They are:

1. That the current behaviour of the feature be retained, perhaps with a few generic templates of the most common applications included, thus avoiding the blank folder scenario we currently have upon new install.

2. That the concept of template files be totally re-thought, and a true document-template function be created that adds usability, gloss and polish to the gnome desktops we use.

I make no secret of the fact that I am an advocate of the latter option, and somewhat bewildered by past discussion on this matter.

The current bug discussion can be found here:
[https://bugs.launchpad.net/hundredpapercuts/+bug/372132](https://bugs.launchpad.net/hundredpapercuts/+bug/372132)

So I wanted to survey the forums for their thoughts and ideas on this, what would you like to see, how would you foresee using this feature etc...
