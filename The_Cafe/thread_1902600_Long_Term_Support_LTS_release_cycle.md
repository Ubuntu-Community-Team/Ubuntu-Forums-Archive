### Post by markellse on 2011-12-31
The announcement that future LTS desktop versions will be supported for 5 years is wise. Can this start with 10.04?
Reasons:
1. If you have 50 Ubuntu machines in a busy organisation like ours, with testing and training, it takes 18 months to deploy a release. Supporting 10.04 for longer gives valuable time to do the job properly.
2. 10.04 server already has 5 year support. The extra for the desktop is not such a big ask.
3. CRUCIALLY, the change to Unity is a big one: it simply needs more more time for us techie chaps to get familiar before we can even think about rolling it out in an office. It's not ready technically, nor is the UI yet generally acceptable.
4. Ubuntu needs to get a name for solid reliability and hardware support over time. 10.04 is only just getting there for things like current Atheros network cards or Sandy Bridge. We simply do need longer with our current, very competent and accepted LTS.

### Post by markellse on 2012-01-15
(markellse said: If you have 50 machines, a release cycle takes 18 months.)

> **mips said:**
> What?

Why should a release cycle take 18 months. I don't think that anyone using Ubuntu with a large number of users and different machines will be surprised. 

In my own organisation, we were getting on very nicely with Jaunty, which largely worked fine. The trouble was that it was out of date and not supported with newer hardware. Hence we tried to manage the shift to Lucid.

Lucid moved to xfree. Along with that were quite a number of bugs associated with graphics drivers. For instance the Intel 845 graphics chips crashed randomly - still do under Lucid. These were a quarter of our machines, an new ones that we had bought for Linux machines. That required a hardware workaround (i.e. we didn't use those machines!!) Hardware problems like this slow down the implementation of any upgrade. 
Early versions of Lucid would not work with Realtek gigbit network cards, which delayed us using more modern motherboards on our LTSP servers. It was Jan 2011 before Lucid had sorted out this network card issue and we had had time to check that such servers were reliable - nine months after Lucid roll  out. At that time, we were still using Jaunty on the machines with Intel graphics. 

For any Ubuntu machine, one needs a testing period for any new hardware to ensure compatibility. We struck lucky with Gigabyte G31 boards, which worked as stand alone machines, LTSP servers and clients. This was great and we started to roll these out to replace the Intel boards that didn't work with Lucid. The problem was then that there was a general move from Realtek to Atheros network cards and good old Gigabyte moved their G31 motherboards to Atheros without changing the board number (Revision 1 is Realtek, revision 2 Atheros. We now have a stack of Atheros boards that we can't use with Lucid!!! 

Lucid, of course, does not yet have support for Sandy Bridge. In other words, the current LTS version of Ubuntu does not work with a very large number of pieces of current hardware. Hence my original post for longer support for this.
