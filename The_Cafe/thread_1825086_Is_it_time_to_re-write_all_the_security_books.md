### Post by whatthefunk on 2011-08-15
The name given to the malware depends on what the malware does, not on how it gets on your system.  A trojan can get on your system in hundreds of different ways.  Through an email attachment, clicking on a popup, an infected USB stick, Active-X, installing malicious software etc.  Some definitions from wikipedia:

Trojan: A Trojan horse, or Trojan, is a destructive program that masquerades as an application. The software initially appears to perform a desirable function for the user prior to installation and/or execution, but (perhaps in addition to the expected function) steals information or harms the system.[1] Unlike viruses or worms, Trojan horses do not replicate themselves, but they can be just as destructive.

Virus: A computer virus is a computer program that can copy itself[1] and infect a computer. The term "virus" is also commonly but erroneously used to refer to other types of malware, including but not limited to adware and spyware programs that do not have the reproductive ability. A true virus can spread from one computer to another (in some form of executable code) when its host is taken to the target computer; for instance because a user sent it over a network or the Internet, or carried it on a removable medium such as a floppy disk, CD, DVD, or USB drive.

Worm: A computer worm is a self-replicating malware computer program, which uses a computer network to send copies of itself to other nodes (computers on the network) and it may do so without any user intervention. This is due to security shortcomings on the target computer. Unlike a computer virus, it does not need to attach itself to an existing program. Worms almost always cause at least some harm to the network, even if only by consuming bandwidth, whereas viruses almost always corrupt or modify files on a targeted computer.

### Post by Dangertux on 2011-08-17
I think you got pretty good answers to all of your direct questions. 

But on the topic of the original question you posed "Is it time to re-write all the security books?"

Not really, IMHO it's the same stupid stuff that's been going on for years. Let's be realistic, the technology changes but the principle doesn't. 

As someone above said, most vulnerabilities still lie in terrible coding, unpatched systems, weak passwords, unvalidated user input and vendor device configurations. This hasn't changed in a decade. Sure, as the technology changes the methods for exploiting vulnerabilities change slightly. By and large, it's mostly all the same though. 

Oh yeah, let's not forget the biggest risk to security (in a corporate environment), the employee. Regardless of what your technology is doing the most easily exploitable device in any work setting will always be a person. Why? Because, companies pretty much refuse to pay for training for their employees to familliarize them with common social engineering vectors.

Security has never been a checklist, it's a way of thinking and a process. That will never change, old security books can be incredibly useful to getting your mind used to working in that way. Of course if you want to learn about the latest methods for scooting under a NAC's radar or the best technique for Blind SQL injection, you should probably read the newer stuff.
