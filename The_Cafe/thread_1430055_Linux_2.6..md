### Post by blueshiftoverwatch on 2010-03-14
Software quality is what matters, not arbitrary version numbers. For example, it took [FlightGear]("http://en.wikipedia.org/wiki/Flight_Gear#History") 11 years to get to version 1.0 and only 3 years to to 2.0.

Versions (as well as changes in the GUI) are often a scheme of commercial software companies to pass an incremental update off as a whole new experience. To get the user to buy what is essentially almost exactly the same piece of software all over again only a year or two later. Or even by non-commercial software developers as a means to drum up excitement about a new release. If a developer was currently at 1.0 and was ready to release a new version a year later. Calling the new version 1.1 wouldn't sound nearly as exciting as calling it 2.0.

Sort of like how Microsoft called their second generation console "Xbox 360" instead of "Xbox 2". Sony was already out with "PlayStation 3" and MS didn't want people to subconsciously think that the PS3 was better because it ended with a bigger number than their console. So they called it the "360" instead so that the first number in their console name would be the same as the first number in Sony's new console name. Some large corporations probably have entire marketing divisions who do nothing but sit around and think about stuff like that all day long.
