### Post by rko618 on 2007-04-21
I was wondering why the Ubuntu website makes it so hard to download Ubuntu using BitTorrent?  Just as a little test I steps one has to take from ubuntu.com to reach the .torrent files.  In all 5 steps, and that's only because I knew where I was going.  For a first time visitor to the site they would likely never find the torrent download.  Here are the steps:

From ubuntu.com
1. Click download
2. Click the link in the little paragraph at the bottom "Feel free to view the complete list of download locations and other download options. This list..."  <-- Would help if it mentioned BitTorrent specifically. I had no idea this was even the right link to click.
3.  Click on which mirror you want.  NOTE: It has the BitTorrent options at the bottom but 7.04 isn't even listed there! That made me think that there was no BitTorrent option available for Fiesty.  Again I had no idea whether this would lead me to a BitTorrent link.  7.04 should be added here!  And the BitTorrent section should probably be moved to the top of the page.
4. Click your desired ubuntu release
5. Finally squint your eyes and pick out the correct torrent link from the ugly list presented.


5 steps isn't take many but considering it only takes 1 if you want to download over http (click download on the main page), taking 5 is ridiculous. 

So since Canonical could save so much money by pushing BitTorrent more why don't they?  I understand BitTorrent can be really confusing for beginners and http would be better for them but at least make it so that the people who want to download it over BitTorrent can easily do so.

### Post by jerrylamos on 2007-04-21
I double click my simple 4 line rsync script and away it goes.  

It downloads the differences between the .iso I have and the current, downloads the md5 sums, does an md5 sum on the updated .iso, and does less on the concatenated results so I can see the md5's compare.  

Any time I've run bit torrent it takes much much much longer than rsync.

Now as an interest I've been doing rsync on Feisty Alpha and Beta Ubuntu, Kubuntu, and Xubuntu so the rsync doesn't have that much to download.  For example I was all up to date before the flood gates opened so I haven't had to face the crunch.  If  you don't want to pay that much attention, I'd advise about a week before a scheduled release date start doing rsync's.  I think you'll have a good copy before the crunch hits.

Cheers, Jerry :)
