### Post by Dangertux on 2012-04-08
Here are my comments, I joined the chat for this "destro" ;-) last night, and asked a few questions. My experience/impression was not particularly good.

Let me go ahead and preface this by saying the following : To the OP , learning linux is a unique journey and if this is your endeavor to do so, may the source be with you. That being said, the following criticism is intended to be constructive, and though it may seem harsh at times I am not picking on you instead trying to offer some advice.

I asked three major questions last night. The first was the same I posed in this thread, how are you different better etc than Ubuntu / Debian / Fedora / Bob's Linux.

To that I received the answer : We're more secure, and easier to use.

Okay , fair enough. So I asked how you are more secure.

I was greeted with two answers. The first that you would be implementing "extra firewalls". I asked for clarification on if you really intended to run multiple layer 3 firewalls. I was told yes.

You added that , the hard drive is "protected" I asked how, and was told it's encrypted from the start.

I asked how this is different from LVM with LUKS encryption. I was told that it is using AES, "which is used by the US DoJ".

That is true, however LUKS already supports AES by default. So it's no different, I was then told that the authors of this distro would be building a new crypto-graphic cipher for use with this hdd encryption.

Lofty goals.

However, I'm left with two problems. One the promises of "vague" security measures in the form of more firewalls and a magic cipher? 

I was also told that it would be tested against a "virus". When I asked what guarantee the authors would be able to provide that the modifications they were making to the kernel in fact bolstered its security as opposed to weakening it.


My overall view of this project from the hour or two I spent with what I assume is the developer and also OP is this.

1 - Stop, reorganize, learn more about linux.

2 - Come up with VIABLE, and pertinent changes you SHOULD actually make.

3 - Come up with a target audience. Stop touting the typical "we're secure" tag line, because truthfully, I'm not sure you're able to offer any additional features. As I stated in chat last night, if you want to add actual USEFUL security features to a Ubuntu-like distro. I would suggest something like a GUI frontend for Apparmor (See YaST) or maybe something like SELinux troubleshooter (in Fedora)

4 - Your marketing is great, but you need to lose the shady looking chat , and start using standards like github etc.

5 - You might want to inform people that this is a "programming project" for you, and that if it doesn't get off theground not to be surprised. 

6 - Play to your strengths. If you truthfully don't know much about security (didn't seem like you do) don't build a "security oriented distro". If you DO know about building GUI's build some, add them to some packages you feel need a GUI configuration manager. If you're great at math, you don't need a distro to build a cipher and truthfully those are two VERY different goals, equally time consuming if done properly.

7 - Stop trying to reinvent the wheel. I was told you would be redoing a lot of device drivers. Why was never answered. If it works, don't mess with it, fix broken things. 

8 - If you want some linux programming experience, you don't need to build a whole distro, there is a bug tracker full of issues you can help with.



Those are my bullets from what I've seen. Again, if this is what you want to do more power to you and please don't take my criticism as I'm picking on you. However, these areas definitely need development if you're going to be taken seriously.

Thanks
