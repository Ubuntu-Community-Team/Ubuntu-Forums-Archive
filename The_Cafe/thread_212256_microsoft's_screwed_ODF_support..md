### Post by rcarring on 2006-07-10
The writer of the article in e-week makes some valid points, but honestly, unless a user is getting documents sent to them in odf (e.g. odt) they're not going to be bothered. In Office 2007 the default file format is docx (which really is a zip file containing several xml documents) and this means that recipient users of a docx file would need to download a convertor tool if they are still running Office 2003. 

I expect that OpenOffice.org will eventually incorporate support for the docx format.

### Post by kripkenstein on 2006-07-10
First, Microsoft have their plugin on the BSD license... so other people can improve it, and make using ODF inside Office not so much of a pain. I am betting this will happen. With any luck, there will be a plugin that makes ODF the **default** format for Office, and governments, schools, etc. will use it.

Second, as others mentioned, Microsoft's XML format will be supported by OpenOffice, Abiword, etc. - I assume. Probably soon. So, even if Microsoft win the format war, we open-source people may still be able to interop with Office. Yes, MS will try to make things hard for us, but given their format is openly documented, it won't be as easy for them as in the binary file format days.
