It's not *inherently* a bad standard *because* it is pushed by Microsoft. To be honest, the CLI VM is way better designed (from a language POV) than the Java VM.

> **ryno519 said:**
> Can you say that Mono *is* a Microsoft technology or can you say Mono *is* based on a Microsoft specification?

The ECMA standard only describes the original MS .NET 1.0 implementation (well, the CLI part to be precise, not the accompanying libraries). And Mono is just another implementation of this standard.

So yes, Mono *is* a Microsoft technology (designed by MSFT although not written by them). Like it or not, you can't do much about that.


EDIT: BTW, what is exactly the difference between technology and specification in this context? I can't see one, since the specification makes the technology (unless you're one of those bottom-up proponents? :p).
