### Post by MaxIBoy on 2010-05-07
I personally am not at all involved in Compiz or its development, not even in a "bacon number" sort of way. However, I noticed this in the Compiz development blog, and I figured I could at least help spread the word. So I'll just quote SmSpillaz's blog post in its entirety:
> 
**[Compiz master testers needed]("http://smspillaz.wordpress.com/2010/05/07/compiz-master-testers-needed/")**

                   Posted on May 7, 2010 by smspillaz                     
                                        Hi everyone,
 So we have about 95% of all the bugs fixed for a release and now we need YOUR help to identify the remaining bugs in git master before making the new development branch *official*.
 Testing is relatively easy, and can be done without detroying your existing compiz install.
 Instructions [here]("http://forum.compiz.org/viewtopic.php?f=112&t=12565").
 P.S: This /is/ a rewrite from a code perspective, so it&#8217;s basically supposed to be functionally equal to 0.8 (with the addition of pluggable rendering backends and reparenting decorations among others). There isn&#8217;t any major new functionality, so Phoronix please don&#8217;t run a story about how there isn&#8217;t any progress whatsoever, because once a release happens you can definitely definitely definitely expect progress [IMG]http://s.wordpress.com/wp-includes/images/smilies/icon_smile.gif[/IMG]No, I don't even know SmSpillaz personally or anything, I just want to help get Compiz 0.8 finished as soon as possible.

And the directions he linked to are working fine for me. Performance seems a little bit better compared to Compiz 0.7 and so far no major showstoppers I've run into. But there is a little glitchiness so you should keep that in mind.
