### Post by pp. on 2008-09-12
> **clanky said:**
> I have never really seen the point in wine, surely the best way to run windows applications is to run them in windows?

It's a typical working pattern for me to use a number of applications at the same time. Thus, I might need some particular data about real estate which is available (or which I keep up to date) in an Access database and a number of spreadsheets to reply to a mail. Also, I might have to add some of the data received by that mail to one of the projects I am managing at the time.

Since Access and Project Scheduler run with little fuss and quite acceptably in Wine, I am happy to run all the rest of my desktop at home with FOSS. 

Multibooting is not acceptable since I need those applications running at the same time. My laptop is elderly and I don't plan on putting all that RAM into it just in order to run all that stuff in VMs. Besides, I'd have to maintain Windows if I did that which I'd rather not do.
