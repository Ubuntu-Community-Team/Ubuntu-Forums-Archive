### Post by jomiolto on 2010-03-28
> **falconindy said:**
> Actually, I didn't want to be pedantic and point it out (assumed it was a typo), but **you're** the one confusing bits with bytes. The only significant quantities of bits are:

4 bits = nibble
8 bits = byte
16 bits = word
32 bits = double word
64 bits = quad word

1024 bits is insignificant, particularly in the context of this thread.

Nitpick: byte does not necessarily mean 8 bits. It was traditionally defined as the smallest amount of data that could be conveniently addressed in a computer architecture and there have been many architectures that used various sized bytes (6 bits, 8 bits, 9 bits or even 12 bits). Octet means exactly 8 bits, but unfortunately it's not used very much.

Similarly "word" was the usual data size for doing arithmetic and addressing memory and such. The only reason word is so often used to mean 16 bits is because 16-bit processors used to be so common. But again different computers used different word sizes; 16 bits, 24 bits, 27 bits, 32 bits, 36 bits, etc. With modern processors it would make more sense to talk about 32-bit or 64-bit words.

Back to the topic: the problem with base-10 prefixes is that they're pretty much useless in many cases. You can never have exact counts with them (unless you want to have 2.147483648 gigabytes of memory), so you'll have to do some rounding anyway, which means that you're losing precision. You might as well (unknowingly) use base-2 units and *think* that they're base-10, because your counts wouldn't be precise anyway.

I think there would be much less confusion if the base-10 prefixes would have never been used at all with computers and everything would have been base-2 from the start (and the decimal prefixes, "kilo", "mega", etc. should have never been used in their base-2 meaning). Base-10 just does not make sense when you're often counting in units of 256, 512, 4,096, 65,536 or whatever. (It's especially "nice" how sometimes both of the units are mixed: for example, 1mbit is usually neither 10^6 bits nor 2^20 bits, but 1024*1000 bits. Similarly "1.44MB" floppies were neither 1.44 * 10^6 bytes nor 1.44 * 2^10 bytes, but 1440 * 1024 bytes).

I have no problem with calling the binary prefixes "kibi", "mebi", "tebi", etc., and I actually support their usage, but I do not want base-10 prefixes.
