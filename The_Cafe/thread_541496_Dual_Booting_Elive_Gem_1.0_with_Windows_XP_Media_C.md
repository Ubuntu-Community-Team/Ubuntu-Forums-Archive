### Post by RAV TUX on 2007-09-02
So today I am doing a bit of a change:

I first re-installed Windows XP Media Center Edition 2005 this went easy enough. I erased both hard drives while doing the windows install, intentionally of course.

I Installed Windows XP on the first hard drive.

I then popped in my 8GB KNOPPIX DVD the older version of KNOPPIX 5.0.1, the KNOPPIX installer works better for me with this version. I used the KNOPPIX installer to convert the second hard drive to ReiserFS, I then popped in my Elive Gem 1.0 CD and installed Elive on the second hard drive.

(using KNOPPIX to convert to ReiserFS is an unnecessary step, since the Elive Installer will do this, I just do it out of habit)

This again went with out event, I was really impressed that Elive set up a custom grub menu that gives me 3 options 

1. Elive Gem 1.0
2. Windows XP Media Center Edition
3. RAM Memory test

I was even more impressed that it prompted me for an update upon first booting into Elive.

To make a long story short, I made the decision to reinstall Windows XP Media Center Edition and create a dual boot to Elive Gem 1.0.

I honestly like e17 better then any other Linux DE, the old debate about which Linux DE looks most like Windows; KDE or GNOME, I think they both look like Windows, yet they are both first-class awesome Linux DE's. e17 to me has one of the true Linux DE looks, much like Fluxbox. 

I made the decision to reinstall Windows for personal reasons, which I won't get into but it is needed and is my choice of course. I already own about 3 copies of windows.

How does running Elive instead of Oz or Linux Nirvana effect development of those OS's. I honestly would like to have e17 the default DE on Linux Nirvana, so I thought I should become more familiar with an OS that exclusively uses e17, and Elive is the best OS for e17. Keep in mind that Elive is developed by the e17 team.

I know you can load e17 on any OS, but believe me it does not seem to work as smooth as it does in Elive.

I may pitch in to help with the e17 and Elive projects to help further the overall development.

I suspect there will be big changes in the future for Linux Nirvana. Oz I will probably keep it with GNOME and KDE.
