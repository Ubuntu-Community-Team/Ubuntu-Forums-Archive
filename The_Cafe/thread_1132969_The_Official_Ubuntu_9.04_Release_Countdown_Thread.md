    *

      Some users have found improved performance by using the "greedy" migration heuristic. This can be done by running "sudo gedit /etc/X11/xorg.conf", and adding Option "MigrationHeuristic" "greedy" to the Device section of your xorg.conf.
    *

      Alternatively, a new experimental acceleration architecture option, "DRI2/UXA", is available for Intel graphics users which our testing has found provides significant performance improvements in some cases, but has also shown risk of severe stability problems. You can opt-in to enable this by running "sudo gedit /etc/X11/xorg.conf", and adding Option "AccelMethod" "UXA" to the Device section of your xorg.conf. Users wishing to maximize stability should stay with the standard default acceleration method, "EXA".

      /!\ In some cases this will lead to the graphical environment not starting at all or becoming entirely unusable. In that case, start into rescue mode or press Ctrl+Alt+F2 and log into the text console, and use sudo nano /etc/X11/xorg.conf to revert the UXA option.
    *

      If none of the above helps, some users reported success with using an older driver version.

Display freezes with Intel graphics cards

Users of Intel video chipsets reported that the display freezes a few minutes after resuming from suspend or hibernate (bug 339091), or under other circumstances 359392. In many cases, switching off desktop effects in System &#8594; Preferences &#8594; Appearance was reported to help.

If it still happens without desktop effects, you can add Option "DRI" "off" to the Device section of /etc/X11/xorg.conf, as described above. This will disable 3D acceleration and desktop effects, but makes suspend work reliably again and also avoid many types of crashes.

Kubuntu 8.10 came with KDE4 as default.  Kubuntu 9.04 will continue that tradition.  If you still want KDE 3.5, there's a strictly unofficial remix [available]("http://www.kubuntu.org/node/77"), but all the development focus is on KDE4. KDE or GNOME being better/faster than each other pretty much depends on your own feel for each system.  For some people it's a love one/hate the other situation; others use both and switch between them all the time.  The great advantage of having official KDE and GNOME support from Ubuntu is that installing whichever one you don't have is as fast as typing one single command.

If you've never used either, then I suggest trying both.  KDE4 is definitely a leap in mindset, but as a Kubuntu user I've found that the more I use it the more I appreciate the changes.  I've also found that most of my annoyances with it are disappearing with each new release, so apparently there are plenty of vocal people out there chiming in with how much they love or hate certain features.  KDE4 is still not as stable for me as KDE 3.5 was, but unlike KDE 3.5 I can't manage to kill the thing.  If it crashes, it essentially restarts itself without any input from me, and the system is just as it was before the crash.  Crashes are becoming increasingly rare; I may experience one every two weeks or so.

I can say that my primary reason for preferring Kubuntu is that I prefer the KDE apps, like K3B, to their GNOME counterparts.  I was using GNOME and realized that almost everything I wanted to use was a native KDE app, so I made the switch.

Install both and give them a good thrashing.  Then delete the one you don't like and proceed to bash the heck out of it on the Ubuntu forums so you can fit in with the rest of us!

### Post by Geochelone on 2009-04-23
> **razielgn said:**
> I own an eeePC, should i get the UNR version? How can i burn the .img file on my USB stick on windows?

*1. Download the img*
[http://releases.ubuntu.com/jaunty/](http://releases.ubuntu.com/jaunty/)

*2. Get dd for Windows (well, assuming you are on Windows right nao)*
[http://www.chrysocome.net/dd](http://www.chrysocome.net/dd)

*3. Copy downloaded image to the extracted 'dd for Windows' folder.*

*4. Run cmd.exe, go into 'dd for Windows' folder.*

*5. Execute the dd commands*
dd if=<location of the image> f=\\.\<drive> bs=1M

*5a. Example*
dd if=c:\dd\unr-9.04.img of=\\.\g: bs=1M

You might want to run them with Admin right.
