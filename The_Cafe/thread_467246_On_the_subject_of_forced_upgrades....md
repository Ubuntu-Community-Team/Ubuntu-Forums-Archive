### Post by blueturtl on 2007-06-07
I'm sure someone has probably already pointed this out but a lot of us are here because we couldn't live in the Microsoft world anymore. One aspect of that world was the never ending cycle of forced upgrading. What I'm getting at is the fact that even in Ubuntuland some upgrades are forced! People make less fuss about it though because the upgrades often bring about improvements and don't necessarily require any changes to current system hardware. In our world newer IS better or atleast so we like to think.

Just today I glimpsed at a post where someone complained about being unable to install software after the repositories for his/her version of Ubuntu had gone offline. Since everyone using Ubuntu is so dependant on the repository system for software installation it does create an incentive to upgrade even when the upgrade is not necessary. Now I realize this is not a problem for most of us since we'd like to upgrade anyway, but it does create a small chink in the Linux armour to think that the choice of holding back an upgrade is infact not an option. Eventually everyone has to upgrade and let's not pretend upgrades in Linux don't break things; they do sometimes. Even though the quality of open-source software is very high I sometimes find that things that used to work before stop working in new releases. New features introduce new bugs and sometimes features that are well liked seize to exist. If a system has been configured and setup and everything works just as intended you still have to upgrade for the sake of upgrading.

I don't really know what to think of this. It's a common joke (joke to some that is) that you have to reinstall Windows every 6 months. Has anyone ever run into a situation where upgrading your Linux box was found to be a nuisance instead of a blessing? Am I a lone looney for having these thoughts?

> Ok, lets get it straight first. In your example in the middle their, the version did not go "offline", the support cycle met its end. All products in the software business work on this concept to my knowledge. You are absolutely right and I'm not claiming this to be a huge issue. However the repository install system introduces a new kind of problem. Let's say I install the current version of Ubuntu and upgrade it as updates are released. After a while the support cycle ends and if after this time I choose to reinstall that release or install more software I am under the impression that the repo's for that release do go offline. I would be unable to install any software or even previously released updates to that system. The difference between this and what most commercial vendors (you mention Microsoft) do is that even after support is discontinued for a product the software and updates previously available to that product do not disappear off the face or the earth.

I can still download all the patches previously released for Windows or WordPerfect even if new ones are no longer released. I can still download older versions of software off the internet that run on older systems. Windows 9x is grossly outdated and doesn't get new software or patches, but I can still run the latest incarnation of the release. With Ubuntu I will be unable to attain the latest version of a release after the support cycle ends which means I will be running the version that was released on CD at the very beginning of the support cycle. No patches available, not even the ones that were already released. Also installing software not on the install CD becomes near impossible (I suppose I can always download the source code and compile but who wants to go to all that trouble). While this can't be called forced upgrading I think the user is given a good nodge in that direction.

Why would I want to run a particular version of Ubuntu or software? Maybe the old updated version did everything well or better than a new release. Maybe I can't be bothered to deal with the new features or bugs or what ever cost that comes with the new system. Who knows. I willingly upgrade every time but what if a time comes when I have to stick to an older version of a system for what ever reason? Surely at some point even Ubuntu will require you to upgrade your hardware, then what will you do?

I think each install CD for Ubuntu should be updated at the end of the release cycle to include the latest patches and updates. This way even without the online support people who need the older version would also have their needs met.

### Post by amac777 on 2007-06-07
> **blueturtl said:**
> I think each install CD for Ubuntu should be updated at the end of the release cycle to include the latest patches and updates. This way even without the online support people who need the older version would also have their needs met.

I understand what you're saying and I think I remember reading somewhere that the Dapper install cd was updated to include more recent versions (and thereby prevent a new user having to download 700MB of new updates immediatly after a fresh install). Maybe I'm wrong about that but it would be a good idea like you said.

But just updating the install CD would probably not be enough to satisfy the needs of a person who is planning to continue running Dapper (for example) even after its support cycle is over. That person would probably also need a full mirror of the final state of the Dapper repositories right before they went offline. The reason is that just the install cd doesn't necessarily automatically install all the software that you might be using. So if you had to reinstall Dapper from scratch after it was no longer available from Canonical, you'd still need to be able to run 'apt-get install' to install the programs your system used before.

So I guess if you are planning to use Dapper past its support cycle, the solution might be to download your own full mirror of the full Dapper repositories and the install CD for dapper right before they go offline at the end of Dapper's support cycle. I guess this would be a couple DVDs worth of data. If you had the full repository and install cd yourself, you could reinstall from the CD and then use 'apt-get install' to get the most recent version of programs certified to work with Dapper from your local repository forever (or as long as you needed to) even if Canonical's go offline.
