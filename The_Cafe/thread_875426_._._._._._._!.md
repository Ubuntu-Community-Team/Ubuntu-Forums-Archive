Really alot of the big Vista problems ( beside the DRM/antipiracy/controll stuff ) are not caused by Microsoft, but by the hardware venders.  People keep talking about how much SP1 fixed things, and it did help, but the real reason Vista seems better now is that Hardware venders, for the most part, are no longer putting it on single core 512mb RAM PCs with shoddy integrated video.

If they would have not sold Vista on low end machines, and would have been a little more forthcoming about how much of a machine it would take to run, alot of people wouldn't have put it on machines that shouldn't have been running it, and popular opinion would be a little more proVista.

But on my newest machine both Vista and the older XP run about the same, I actually see slightly better performance in Vista** for some tasks.  Also alot of things in Vista show that Microsoft is trying to slowly move Windows into a better place, and the only reason they didn't jump there right away is there precious backward compatibility.  If they didn't have to worry about that we'd have seen a filesystem with permissions and a root/sudo like setup in Vista.

Now, I'm not saying that Vista is a good OS in it's own right.  I'm not a huge Windows fan.  But I'm just saying that it's getting the unfair end of the stick when compared to XP.  If someone tries to run GNU/linux on hardware that isn't really fit for it ( due to lack of drivers or some other reason ), that doesn't make linux a poor OS, it just means that the user was loading it onto the wrong kind of system.  The same thing is true for loading Vista on underpowered hardware.


** I should point out that both my XP and Vista installs where heavily modified while I was messing with them with a lot of unnessisary services turned off.  At the time I was using it Vista's indexing service was 'broken', though I have no idea if it's still as bad now.
