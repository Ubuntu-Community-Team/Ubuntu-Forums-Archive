### Post by FoolsGold_MKII on 2007-07-11
Microsoft (and Apple for that matter) have, for better or worse, successfully sold the idea to the public that computers can and should be thought as a household appliance, much like a toaster. You plug it in, turn it on and do your thing. In theory.

Now that's all well and good, but it also means that people will *treat* the computer as a typical appliance. Most appliances in the household are fairly simple to operate - computers are not. Given the vast functionality of a computer, there's a lot more things someone needs to educate themselves with if they are to avoid the pitfalls of the computer. I'm not saying people need to reach the level of Linux geek, I'm just saying... by labeling computers as appliances, this somehow means people believe they don't have to learn basic operation and maintenance skills as part of their experience in using the computer.

This is rubbish of course; failure to learn results in virus/spyware infestation and stolen back account details. It also results in a shorter lifespan of the computer, particularly if the user isn't aware a system can be reinstalled and instead throws out the computer, like "any other appliance", and gets a new one.

I say, the idea of a computer = appliance is wrong. I say, treat the computer like one treats a car. Cars aren't appliances, they are machines that need to be treated with a certain level of respect. They're straightforward to use, but educating yourself on basic maintenance skills and troubleshooting will go a long way to extending the lifespan of the vehicle. With these requirements no-one would define cars as appliances... so why are computers?

### Post by mangar on 2007-07-11
Computer are appliances since the early 80's.
Why? Because almost anyone can afford them, and almost anyone one who can, does.
(With the usual exception of half of the world).

Computes can be seen as a Meta-appliance, or if to simplify,as an information appliance
Since information comes in many forms, and represents many things, handling all those forms is complex task - lets say it is an order of magnitude more complex than a transportation appliance (a car, for instance).

IMHO, security concerns, system maintenance, fine-tuning configurations, resolving system errors etc. should not be part of the problem that the user have to solve. It is totally up the the appliance (or OS) manufacturer to fix, and the second it spills over to the user domain, than the appliance failed its task. 
osX does this mostly well, but has some limitations on the types of information it can process. Windows is the most versatile of the bunch (can process the widest array of information), but the spill to the user domain is terrible, and implies broken design (UAC, anti-viruses, ad-ware - all those should not be users problems!). GNU/Linux is very limited on the type of information it can process (see the desktop readiness for detailed analysis), and while trying to minimize the spill to the user domain (package management, automatic codec and driver installations, automatic updates), the coverage is far from complete.

(Just for clarification, IMHO, the correct way to set up a software piece is for the software to set up all 
the prerequisites, and do everything (or the absolute closest approximation to everything), rather than delegate the responsibility to the user using a configurable option).

The end result is not what the API is, or the underlying architecture, but rather - does the Information appliance does a good job in storing, processing and presenting information?
