### Post by 50words on 2007-09-26
One of the greatest features introduced by Gmail was the move from folders to tags as a way to organize e-mails. Then Thunderbird added and expanded on the same feature, and blogging software has jumped on the tagging bandwagon, as well.

As an organizational tool, tagging is incredibly useful. It does away with the need to store everything in a hierarchy or database and gives the user the ability to instantly find all relevant information.

So why not allow system-wide tagging? In other words, I would love to be able to tag everything related to a subject. Say I am working on a project--the Smith project, a blog post related to Exaile--and want to find everything related to it. What if I could search by the "Smith" or "Exaile" tag and pull up  every e-mail, contact, task, appointment, and--this is the key--document, image, mp3 file, etc. on my computer or network?

In other words, take one of the best features of Web 2.0, the ability to organize with tags, and integrate it into the operating system.

Now that would be one heckuva feature. I can't be the only one interested in this. Can we pioneer this on Linux before someone else jumps on it?

### Post by 50words on 2007-09-26
You can only put things into one folder without doubling, tripling, etc. your storage space. You can assign multiple tags, and can then store the document/thing anywhere you want to.

For example, you might not have room to put a 100 GB movie file on your laptop's hard drive, but store it on an external drive. With tags, you can still associate that with, say the "home movies" tag so it comes up when you search by that tag.

Coupling tags with search folders would be even better. Instead of using folders to sort things, you could use search-by-tag folders as a convenient way to pull up all things with a tag, no matter how they are stored on your drive.

The directory structure works perfectly well, indeed. For a directory structure. Tags are a completely different, dynamic way of organizing content.

### Post by 50words on 2007-09-27
MetaTracker and Beagle together make a powerful indexed search, but they don't have anything to do with getting the tags on the files in the first place. Leaftag looks promising, but I haven't tried it out, yet.

But even Leaftag is not integrated. You have to go to the actual file, which means if you have a paperless office, for example, you have to scan the document, save it, then navigate to the document and tag it there using document properties. That is MORE work, not less.

A tagging system needs to be integrated so that you can tag from OpenOffice.org just as easily as from your browser, Evolution, Thunderbird, or whatever.

So I pose this as an Ubuntu project because it would mean integrating the feature across the operating system, not just in Nautilus.
