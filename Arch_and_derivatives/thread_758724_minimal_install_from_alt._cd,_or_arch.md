### Post by fwojciec on 2008-04-18
IMO, it's not about how much stuff there is installed but how easy it is to set up and maintain a highly customized system.  The farther away you stray from the default configuration the more messy and more difficult to maintain the Ubuntu solution will get.  It's little things like overwriting my custom configuration files automatically that drive me nuts in Ubuntu.  The way Arch handles this is quite exceptional, especially once you learn how the system works -- it's simple, elegant, transparent, perfectly predictable and designed to empower the user.  Using Arch will give you an entirely new perspective on how the system is working together to do what it does.  The fact that Arch doesn't get in your way is, IMO, it's greatest advantage -- this rather than being lean or fast or anything of this sort that people often get excited about.

And I wouldn't worry about the difficulty factor -- as long as you have some patience and you're willing to do some reading every once in a while you'll learn what you need to know to be able to use Arch comfortably within a few days.

The most comprehensive source of installation/initial set up information is probably the [Beginners Guide]("http://wiki.archlinux.org/index.php/Beginners_Guide").  Arch devs have just released a new RC ISOs which are a different design than the previous ISOs so there is a possibility that some information in the Guide is not up to date, but it should be possible to follow it nevertheless.
