### Post by Rumor on 2008-09-25
I've been using Arch for a little over two years and yes, I am very happy with it.

I don't know that I would say you have to be terminal savvy, but yes, you should be comfortable working with and editing text configuration files. There are no gui configuration tools that come on a base install. 

Your initial install will leave you with a command prompt login taking you to a command prompt from which you will build your system.

The conf files you'll be editing are all well commented and discussed in the Beginner's guide in the wiki. If you are patient and take the time to read in advance of what you are doing, you should have no problems.

The package manager for Arch, pacman, is a command line app. However, it is no more difficult to type "pacman -S package" than it is to type "sudo apt-get install package"

The Arch community is quite friendly and knowledgeable. There are many former Ubuntu users there who can help you through any hiccups you encounter. Some of the developers are among the regular posters, so you will find some very competent help.

Arch is what you make it. There are no default desktop environments, file managers, web browsers, etc. You get to choose and decide which applications you'll use, which daemons start on boot up, which modules you want and which you don't need. Arch gives you the tools to build your system the way you wish it.

### Post by MisfitI38 on 2008-09-26
Arch requires a certain approach.
Before switching to Arch, it is recommended you read the entire Beginners' Guide. 
The entire guide.
If the guide is uninteresting, painful to read, or not appealing, then Arch may not be for you.
If the guide makes good sense, you follow a good deal of it, and you persist in your interest to try it, then go forward.
The reason that Arch users point to wiki articles to answer questions is *because that's why they were written in the first place.*
Thousands of hours have been spent compiling and maintaining these excellent articles, which cover a multitude of topics. 
If the articles are beyond your technical ability, you may either choose to learn more about it, or you may decide it is not for you.
The Arch way is *your way*. There is minimal abstraction in an Arch system; Arch is sometimes described simply as *Linux, with a nice package manager*. This saying has a lot of truth to it. 
People who dislike Arch are often those who don't necessarily want the amount of freedom or responsibility that come with the system. 
Many users do not want to set up configuration files or partition their systems from the command line. They crave sane defaults and GUI tools. This is simply a different approach, nothing more.
What it always comes down to is personal preference based on your computing habits, and whether you are open-minded enough to try something new.
As for the comparison between Arch and Ubuntu; I think it is unfair to compare the installation processes, since the Ubuntu mini.iso does not have a guide comparable to the Arch Beginners' Guide. Nor is the Ubuntu base system simple, with its more complex SysVinit directory structure and accompanying symlinks.
Ubuntu does not offer a ports packaging system. 
It is simply designed for a different crowd.
