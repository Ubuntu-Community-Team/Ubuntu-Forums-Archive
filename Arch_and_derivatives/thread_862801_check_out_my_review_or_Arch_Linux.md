I don't know what you're so upset about ;)  If I want the sources for "bash" package, for example, all I have to do in Arch is run "abs" as root to sync the ABS tree, copy /var/abs/core/bash folder somewhere, navigate to the copied folder and issue "makepkg -o".  This will download and extract sources of bash for me.  I can easily make a package out of these sources (much more easily than in Debian!), with my modifications and patches if I want -- afterwards my customized package will be available for pacman to manage (with dependency resolution and so on).

All Arch packages are built using ABS, and ABS scripts for building packages are available -- so it's completely transparent what it is that ends up in the binary packages.  Arch, in essence, has both a binary package system and a ports system which are merged together and, to an extent, interchangeable, so you are always as close to the source code as your heart desires.  Hell, if you really wanted to, you could use Arch the same way one uses Crux, for example: building all packages from source yourself with ABS/makepkg, and then managing them with pacman.
