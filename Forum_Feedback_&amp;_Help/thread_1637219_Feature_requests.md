### Post by Oxwivi on 2010-12-04
I'm not exactly a new user but neither am I a member from olden times. However, I'm aware of the recurring discussion status of moving to open-source forums just because Ubuntu is open-source. On the contrary, I've got nothing against vBulletin for not being open and such. But I've some feature requests that this current vBulletin forum cannot provide:

[LIST]
[*]I would like to request an API to make creating add-ons possible. Forum member, lovinglinux, [created an addon that allows us to keep track of our AskUbuntu accounts]("http://ubuntuforums.org/showthread.php?t=1598882"); and a similar feature for Ubuntu Forums would be more than welcome, specially since it's difficult to keep track of all the threads we are getting support from, or giving support.
[*]Becoming Ubuntu Members requires being active in the Ubuntu Forums, and I'm sure statistics of a prospective Member's activity on the Ubuntu Forums associated with their Launchpad account will help in such decisions.
[*]A separate area for how-tos and other guides can be created which displays them something like blog articles would be much more convenient. And instead of having them in a separate forum section, associating them with their respective support sections will be extremely helpful.
[*]Finally, if an issue involves more than one support section topics, the user can decide to make just one thread and have it appear in multiple sections so users specialised with each of their section can together help on the issue. Otherwise it maybe unresolved due to incomplete understanding of the situation.
[/LIST]
Of course if vBulletin can do these in the next versions, then all the better. But for now, as far as I know, open-source would the solution.
