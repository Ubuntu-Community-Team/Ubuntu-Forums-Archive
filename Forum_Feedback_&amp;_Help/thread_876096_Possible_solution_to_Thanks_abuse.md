### Post by forrestcupp on 2008-07-31
In light of all of the complaints about abuse of the Thanks system, I have a possible solution.  I have been to many other forums that have a similar system only slightly different.  In those forums, there are two choices to honor someone's help.  You can either mark a post as the accepted solution, or as an assist.  Only the thread starter can mark a post.  You can mark as many as you want as an assist, but only one post can be marked as the accepted solution.  After a post has been marked as Accepted, no other posts can be marked in any way.

You can take it or leave it, and I don't even know how hard it would be to implement something like that.  But it's a pretty good system that seems to work on other tech forums.

### Post by kko1 on 2008-08-02
The OP's suggestion wouldn't change the thread starter's options ("assist" = thanks, "accept" = solved), but I still wouldn't like the thanks removed from non-thread-starting users - also they may find something valuable that they're grateful to the poster for.

However, having two options (I'll sum them up as "assist" + solved) that **only** the thread starter could use *would* make it easier to *search for successful threads*, that is, threads where the OP has received help (and where thus useful advice is more likely to be found). (You could ignore the thanks while searching then, which means they'd be more for flavour.)

*

Apparently the "thanks" feature is much more visible if there is a problem of "thanks abuse". The problem with "solved" seems to be quite the opposite, namely that it doesn't get used enough. If only people abused that feature as willingly...

Because of this, having both "assist" and "accept" could work. "Accept" should mark the thread SOLVED automagically, and cancelling an "accept" should unmark it.
