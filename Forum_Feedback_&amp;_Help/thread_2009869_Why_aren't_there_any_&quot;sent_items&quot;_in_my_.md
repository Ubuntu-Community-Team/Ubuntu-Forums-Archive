### Post by schulwitz on 2012-06-25
Ok, options locked down for spam reasons, I get that.

What I don't get is why ubuntuforums.org (in contrast to virtually every other PM and e-mail platform out there) prevents a user, _**by default**_, from viewing his own messages?

Maybe this made sense back in the pre-8/11 days when users were allowed to configure their accounts, but now that configuration options are restricted, what purpose could this possibly serve?

And if these settings are really the intended result that the ubuntuforums.org programers had in mind, we should at least tell the user what's going on instead of tricking them.  When a user clicks on the 'sent items' button, all they see is a screen that tells them** "Sent Items contains contains 0 messages"**, indicating to the user that all of his messages have failed to deliver, and causing that user to scream and curse at his monitor because now *he thinks* (since his 'sent items' folder has told him so) that he needs to re-type and re-send all his PMs.

This should be changed.
**Proposition A**: Set the 'sent items' folder to be active by default
**Proposition B**: Inform the user that the 'sent items' folder has been disabled because he's not ubu-cool enough yet.
