### Post by kassetra on 2006-07-08
Hiyas, I have a few comments here - 

The forum software we use has a truckload of functionality that we require, but sometimes, things that you might consider "small issues" or "easy to fix" .... aren't.

For example, the theme/functionality of this forum is controlled by no less than 100 templates and partial-templates.  This is a great thing!  It allows us to make customisations so that forum fits our needs!  Yay!  The down side to this is *maintenance* .... if we hack up say, 20 of those templates, to do many of the things people request (if it's even possible!)... we will most likely have massive amounts of trouble upgrading the forum software without destroying all of those changes.

So when we say, "that's not something we can do." ... It's not because we don't value your input, it's because either 1. the amount of changes necessary to fulfill that one request would stop any upgrades to the forum software dead in it's tracks and take weeks of work, not to mention it would destroy those changes and then we'd either have to make people angry by not re-implementing those changes or work for more weeks in order to put those changes back in only to have the next upgrade destroy them and start the cycle all over again... or 2. the forum software simply isn't designed to do that feature.

We try and balance all of the requests with the number of massive changes it takes for us to do it.  For instance, we'd absolutely love to give tons more options in the UserCP to control all kinds of things - but until the forum software supports that kind of user-level customisation, we can't offer it.... 

Please keep these thoughts in mind when reading our replies to your requests.  

Thanks!  :KS

### Post by LKRaider on 2006-07-09
I know of two CSS fixes for the hovering issue.

The first involves using a transparent border, instead of no border at all. But I belive IE has issues with that.

The second is to add a padding to the hover at the same time.

This is the relevant code from the forum CSS:
```
.ubuntu_messagebody a:hover {
	color: #800000;
	border-bottom-style: none;
}
```

The first solution would involve replacing the border code with this:
```
border-bottom: 1px dotted transparent;
```

For the second solution, you just add the following line:
```
padding-bottom: 1px
```

I think it would be better to adopt the second, since some non-complient browsers could have problems with the first.
