### Post by GeneralZod on 2006-04-09
I quite like this idea, *especially* if there was a decent stickied FAQ detailing common criticisms and either justifications for the way things currently are, or, if it's so egregious that it's impossible to justify, at least an explanation on why we're stuck with this and what possible solutions are being tried. 

Installation of software would be an *excellent* candidate for this FAQ: many people complain that there's no setup.exe for Linux programs found on the web, and we could write a whole bunch fo stuff about this issue: That at least some of it is a design decision (UNIX in general has opted for heavy use of shared libraries rather than the statically-compiled strategy employed in Windows (and Mac...?) land; that there are technical difficulties with implementing it properly; that the use of repositories offer many advantages (Aysiu has written some good stuff on this topic); that there are candidate solutions (e.g. autopackage, klik, zero-install, etc).  Poofy's "Why Linux doesn't have something like a .exe" would be a good reference for anyone wanting to research this, especially as mhearn (one of the developers of Autopackage) provided a lot of insights.  

Others could include "(GTK/KDE) apps look weird in (KDE/GNOME) - why can't you Linux guys pick a single look-and-feel?!"; "Why does Linux's hardware support suck?!"; "Why are there 400 distros!" etc.  The list goes on, and I really think a central database with all of these commonly made criticisms and counter-criticisms would cut down on some of the frustrations new users have.

Thoughts? Has anyone tackled something like this yet (besides aysiu, of course :))?

### Post by aysiu on 2006-04-09
It's all in how you phrase is Kerberos. It really is.

If you say "Linux isn't ready for the desktop" and then proceed to list what you perceive as faults because they don't match up to Windows users' expectations for what a desktop should do (and not just Windows users but Windows power users), then, yes, you will be called a *troll*.

If you say "Wouldn't it be cool if we could do such-and-such?" then you'll probably get a lot of support and even some suggestions for how to go about doing that.

You also can't suggest that GUI is always "easier" than command-line tools. The best thing to do is to say that it's best to have both a GUI *and* command-line way to do things--that way everyone's happy. And, yes, there are GUI frontends for many things that are lacking.

The problem isn't that people say, "I think Ubuntu should have this feature." The problem is that people make outlandish statements that incite debate: "Linux is for only computer experts." "Grandmas can't use Linux yet." "Windows is sooo much easier to install than Ubuntu." If people just stuck their comments to actual concrete *constructive criticism*, they would not be labeled trolls.

For example, I don't believe anyone would consider [this set of criticisms](http://mpt.net.nz/archive/2006/01/01/breezy) to be a troll--maybe a little too nitpicky (most of the criticisms are things for which I'd say most people don't care about, but if they were fixed no one complain either), but not a troll.
