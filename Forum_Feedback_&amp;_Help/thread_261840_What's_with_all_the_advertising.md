### Post by marianom on 2006-09-20
I noticed this too and I reported the problem a few days ago.

Since then I come with this idea to stop this kind of behaviour (I tried to search for it in the forums without luck). Let me know what you think:

Since these spammers just post one message and they don't use the same account again I think that when you create an account, your very first message should be blocked and wait for admins verification. An admin logs in, read the message and if it's spam it should never see daylight and the user should be ban for good.
Two problems I see:
1- What if he/she post a real linux question for the first time and then start attacking with the second message? We just have to report him (the second time he'll be banned). Granted, there's a chance we'll get spam but at least it will be more work for him/her, somethin they don't really appreciate.
2- Is this more work for the admins?  Well, yes... and not only that but there'll be an expected "service time": new members can get very frustrated if their first posts take too long to get publish. 
To be honest, I don't exactly know how much work this might represent and even if it's worth the effort considering that regular users tend to recognize spam just with the thread name and never check the message.

Just my 2 cents.
