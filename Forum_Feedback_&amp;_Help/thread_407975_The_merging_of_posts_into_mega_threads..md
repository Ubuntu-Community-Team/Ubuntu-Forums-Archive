### Post by maniacmusician on 2007-04-12
:) wasn't afraid to state my views, just didn't want to be a jerk about it. Some of the mega threads can be justified; If you really want to address this, then I'll expound a little on what I think the right way to do it is.

I think that merges and mega threads should really only be created when a bunch of new threads about something are started around the same time (in the space of a week or so). This is exemplified with the Dell/Linux Megathread. Since it was a period of a couple of weeks and people kept posting about it, it was a good idea to lump them together. Other than that, I don't think they need to be merged; if people feel that it's already been discussed and it's an old topic, the new thread about it should die out by itself anyways. 

Understandably, there are some things that are reposted again and again every couple of weeks, such as the "Linux is not ready for the Desktop" sort of thing. For something like that, I think a more ideal situation would be to create a sub-forum for those types of complaints and move threads into there. It's a little less hurtful to users who feel like they're not being held in regard when their posts are merged into a huge thread. Also, there should be some stickies in that forum addressing all of the common complaints that those people have (nonfree codecs, hardware support, etc), and explain to them that the reason their posts have been moved there is because all those questions have already been answered (and show them the answers).

I've probably missed some things, but that seems like the smart thing to do in my opinion. If you see a flaw with my reasoning, point it out and we'll discuss.

### Post by TravisNewman on 2007-04-12
In many cases I think closing duplicate threads and redirecting would work, in many cases letting both go would work, but there are some threads that should be merged. For example, if George Lucas died (heaven forbid) there would be a LOT of threads started about it within the span of hours, and these would be good candidates for a merge.

If someone starts a thread asking about the security of Linux (to use the current example) and then one starts a month later, merging the two could cause some confusion, and people would miss timestamps and reply to points made a month ago. This goes for resurrecting old threads as well. So I can also see auto-locking threads after, say 90 days to be generous.

Although merging threads does keep confusion down so there aren't multiple conversations about the same thing. So there's definite good and bad to it. Also, something hard to write guidelines for since each instance is unique.
