### Post by Giant Speck on 2008-12-27
I think it should greatly depend on the content of the thread.

For example, if it is a thread about a bug in a program, and a solution is not found to that bug for more than a few months, then posting a solution to the bug should not be cause to close the thread.  There are probably more users besides the OP that are interested in that bugfix.

A user using the search feature is more likely going to be searching for the problem, not the solution.  They want to find a thread where someone is experiencing the same problem as them in hopes that the solution will be in the replies.  They aren't going to be searching for a thread where the solution is given upfront and then everyone comments on the solution.  That'd be a waste of webspace.

However, if it is a thread about something random like "Ubuntu is brown," then it makes sense to close the thread after two years if all the comments consist of "I agree, Ubuntu is brown" or "I don't like brown."  If a user seriously finds it necessary to dig up a "Ubuntu is brown" thread that's been dead for two years just to post "I like brown" then the user has too much time on his/her hands.
