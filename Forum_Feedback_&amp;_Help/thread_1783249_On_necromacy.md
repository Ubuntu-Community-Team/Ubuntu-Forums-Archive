### Post by nothingspecial on 2011-06-15
This is just something that occurs to me, not something I have particular views on one way or the other.

I understand the reasons for closing  old threads that are bumped up due to a new post in them. In almost all cases, it is preferable to create a new thread.

However, in the case of this one

[http://ubuntuforums.org/showthread.php?t=621320](http://ubuntuforums.org/showthread.php?t=621320)

almost 4 years old, someone asked a question about the subject of the thread and received an answer from the original poster within 24 hours.

It seems to me, in this particular case, the thread should not have been be locked/closed.

The reason is that, looking at the OP's profile, s/he rarely visits the forums, but judging by the post is very qualified to answer questions on the subject raised.

So the OP in question (unless there was some unquantifiable amount of luck involved), clearly has email notifications set for that thread. And is therefore available to answer. But would probably not see a new post on the subject.

So, even though the thread has had no activity in over 6 months, it is still a useful thread......

....if you see what I mean.

### Post by Dngrsone on 2011-06-15
I have to refresh my memory on the specific rules pertaining to old threads here on Ubuntu Forums, but the common-sense rule should apply:  if an old thread fits the problem/solution, then why not allow use it rather than create a duplicate one; particularly if there is a lot of useful information in the original that would be lacking in a new thread?

These forums gain new threads at the rate of like one or two a minute, anyway, so requiring a new thread every time compounds a problem that I have with this site:  too many threads to track, too many hits when doing a search for answers.
