### Post by abhiroopb on 2008-05-31
Although the ubuntuforums is great, and has helped me immensely I still find that the search option is practically useless, and I end up using google to search. I think the problem is that there are so many repeat threads, incomplete threads, and general threads that may mention my issue but not actually address it.

Here is what I propose:

Firstly I think there should be sub-directories according to hardware. I currently own an HP dv5242ea, iPod 80gb classic, Nokia N95, Sony Ericcson w850i. What would be great is if there was somewhere on the forum I could go to that would list (neatly) all the threads that related to (say my iPod). Now most people would argue well why don't i just use the search and look for all iPod related topics. My idea is vastly different. Basically if anyone ever has an iPod 80gb issue AND it is conclusively solved then it would appear in this sub-directory. This would mean that if I ever had an iPod classic issue I could look through JUST this sub-directory and if there were lets say 100 threads, it would be a lot, but at least each one would pertain to an important topic or issue that could help me better manage my iPod. Further if this was done for EVERY single device on ubuntuforums that has ever come up in the threads, it would mean that double posting would significantly decrease, as the primary reason for double posing is that the OP does not find the issue (even though it may have been solved). This "project" would take a huge community effort, but I think it would be very useful. Consider this, a new user who has always managed to sync their phones/mp3 players/blackberry's/palm/bluetooth, etc comes to ubuntuforums and can find EXACTLY the devices they have and any issues relating to those devices.

I think the hierarchy could be something like this:

Hardware>Manufacturer Type (Logitech)>Device (Logitech Messenger Pro)
OR
Hardware>Device Type (e.g. Webcam)>Device (e.g. Logitech Messenger Pro)

Carrying on from this idea I also propose a consolidation of threads relating to all software. So everything that has been solved relating to amarok (example) in one giant sub-directory.

Both these ideas would need a LOT of help:
1. Threads that have been solved would need to be identified.
2. Some thread titles may need to be changed so that the issue is better represented (as some posters don't use the best titles).
3. If there are multiple devices/issues these may need to be split into new threads.

What do you guys think? I know this seems very fanciful and far-fetched but I thought about this today when I realised how much easier my transfer would have been if I had a easy to find repository of information rather than chancing upon information.
