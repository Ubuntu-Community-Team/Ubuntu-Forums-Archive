### Post by heimo on 2007-04-10
Idea is good. Some things that need to be considered:[LIST]
[*]not everyone knows their hardware specs
[*]many of us have more than one system
[*]hardware specs change and information of hardware is important part of discussion threads, so the information shouldn't change or disappear afterwards[/LIST]Sometimes it'd be very helpful to have some kind of script, which users could run to gather information about their system, and then attach to their question. Too many questions are so vague.. Would be much easier to help if all the necessary information was there - version of Ubuntu and kernel version (uname -a), output of lspci / lsusb, perhaps dmesg and couple thousand lines from end of /var/log/messages grepped with some keywords (fail/error/warning...) - personally, I'm helping with a lot of Xorg resolution problems and it'd be nice to see xorg.conf or parts of it. Of course all this should be transparent to the user, so that they can see what information gets sent. And as easy to use as possible - with or without graphical user interface, perhaps even from Live-CD in case problems are related to booting or network problems.

I notice I turned this also to question about software "specs".
