It has nothing to do with server space or script.

It has to do with what is the most appropriate format for documentation and how-tos.

Every solution has advantages and disadvantages and all documentation is only as good as the community that maintains it. You can find good forums and bad forums, good wiki and bad wiki, etc.

When you weight the advantages / disadvantages of wiki vs forums for documentation, wiki wins.

For details see: [http://ubuntuforums.org/showthread.php?t=1949027](http://ubuntuforums.org/showthread.php?t=1949027)

This endless debate and personal insults are counter productive.

If you would like to help with the wiki, great. There are people here who can help you learn to wiki.

If you are not going to do wiki, that is also up to you.

If you would like to help, join #ubuntuforums , #ubuntu-wiki, and #ubuntu-doc

See also :

[https://help.ubuntu.com/community/ForumWikiTeam](https://help.ubuntu.com/community/ForumWikiTeam)

[https://wiki.ubuntu.com/DocumentationTeam](https://wiki.ubuntu.com/DocumentationTeam)

[https://help.ubuntu.com/community/ForumWikiTeam/ImportTool](https://help.ubuntu.com/community/ForumWikiTeam/ImportTool)

[http://www.ubuntu.com/project/about-ubuntu/conduct](http://www.ubuntu.com/project/about-ubuntu/conduct)

In particular the sections on colaboration apply. Documentation is a colaborative project. Individual work goes to private web pages, not the forums.
