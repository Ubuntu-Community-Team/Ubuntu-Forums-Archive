### Post by popch on 2007-12-23
In a very limited way this forum already does have a bot (in addition to LaRoza). It searches for threads with titles similar to the one you are about to start. That's by intent a bot, if not by implementation.

I quite agree that a bot would be irritating if it was made to 'feel' like a peculiarly limited member. So can a machine answering the phone.

Let's consider for a moment the needs of a person visiting this forum. A certain segment of the visitors would be perfectly willing to contribute to the solutions of their problem, be it by doing research of by filling out a questionary or whatever. There are others who just say 'help me' or 'this sucks' and still others who silently lurk and browse.

The visitors who are unfamiliar with the 'structure' of Linux or with the nomenclature are in a somewhat vexing situation because they lack the basic premises to ask a question which makes sense to those who know.

That's where the notion of a bot might become useful. Put a button on the web site labelled 'your helpful bot' or some such, just to make it clear that we offer an automated tool which can help some users find solutions to their problems. Behind the button there would be some kind of software which would 
[LIST]
[*]either analyze a written problem description
[*]or launch a dialog, driven by what used to be called an expert system.[/LIST]Since any problem determination begins with the gathering of relevant information, and the problem domain should be a well-known environment, I could also envision that parts of the data gathering could be done with a package installed by default into each Ubuntu machine. The package could interact with the expert system mentioned above and supply answers which are easily obtained by the software but not by a n00b. Care has to be taken, of course, that we do not create the impression that Ubuntu phones home undesirable information. 

Obviously, that process would not work as outlined for problems involving networking or browsing.
