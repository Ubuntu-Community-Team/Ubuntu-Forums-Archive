### Post by tgalati4 on 2010-05-03
It's interesting that 2 weeks before the Lucid release, the forums were actually kind of quiet.  Only developers and daring folks were using Lucid Alpha/Beta and they mostly don't use the forums because how can anyone help them if they are on the bleeding edge?  But the launchpad bug forums were quite busy during this time.

Now that Lucid has been released, there is the usual firehose effect in the forums.  Way more "My system is hosed!" posts and again few people to help because the ones with Lucid problems know that they have to file bug reports and work with the developers to get work-arounds.

Only after several weeks do these work-arounds get spread to forum members and by that time there are system updates in the pipelines.

If Hardy was rolled out today, there would be little fanfare because hundreds of patches have been released.  (And it has been respun a few times--up to 4 now?) So a lot of things that were broken are now fixed and when things are working, people don't waste their time posting.

I'm running Jaunty on this laptop (thinkpad t43p), so I can't really help anyone with a graphics issue in Lucid on an MSI laptop.

So until there is greater uptake of Lucid and some patches roll out to fix the big showstoppers, there will be a sea of screamers waiting to be rescued.

I would like to see some graphs of either # of forum posts vs release date, or # of unanswered posts vs release date.  I don't know if the vBulletin system can pull out that data.  It would be interesting.

Bottomline, if Lucid is not working for you, then try Karmic.  If that doesn't work try Jaunty.  If that doesn't work, try Intrepid.  If that doesn't work try Hardy.  At least you have a choice.

Spend a few days with the Live CD first.  Make note of what doesn't work using the Live CD.  Those same things will probably not work after installing and will require some tweakage.

### Post by P4man on 2010-05-05
What I think ubuntu (and linux in general) really needs, is an integrated hardware compatibility/drivers/bugs/howto database, Most of the elements exist:

- all PCI and usb devices have a unique identifier (see lspci and lsusb output)

- jockey helps finding proprietary drivers. Simple but oh so useful for new users

- checkbox ("system testing" in system > administration. you probably didnt even know it existed) does a fair bit of hardware testing, but what gets done with the results?

- there are several laudable attempts to make a hardware/driver database like here:
[http://kmuto.jp/debian/hcl/](http://kmuto.jp/debian/hcl/)
and many others elsewhere but none are done well, they are not integrated, not moderated or updatable, not fed by enough tested systems, and not even properly structured databases..

- we already have launchpad and community docs with tons of documentation on how to install or work around issues with specific hardware.

Now lets combine these!

Imagine you combine these properly. During (or even before) install, an app checks all your pci and usb devices, checks against an online database and reports back what will and wont work, along with open bugs, workarounds, links to drivers, to howto's, and community support pages. Or to a proper wiki that replaces most of these.

Make the database clever enough to combine components in to (eg) motherboard and into entire systems, so if you are planning on buying a certain laptop or pc, check its compatibility before you buy and it will spit out the results for its gpu, acpi bios, sound, webcam, wifi and bluetooth as well as PC specific things (multimedia keys or whatever)

You can feed such a database from an upgraded checkbox (and from canonical's own hardware certification) as well as by allowing a wiki style editing. Rather than writing how to's in community support pages, you write them there, or you link the howto's to the appropriate hardware.

You could even allow it to actually install software or run scripts to fix things (properly moderated obviously).

open it up to other distro's. If you got a laptop, find a linux distro that is known to work well with it.

Use the database to extract "recommended" motherboards, laptops, webcams, printers, wifi cards,.. Put these on canonical's homepage, or give the vendors a "Ubuntu approved" label,  and maybe you will also give an incentive to OEMs to better support linux (we are a small minority, but its still worth something if you laptop or webcam is one of few to be recommended to 10 million ubuntu users)

Got more idea's than that, but the above would seem doable and I think be a vast improvement (and result in a vast reduction of users needing the same help on this forum over and over).
