### Post by gord on 2006-04-06
i have recently been helping out some friends with their Vbullitin installation, I was poking around the admin interface and came across the BBCode manager, which lets you add your own BBCode that gets replaced with whatever you want. As someone who mainly comes around the Ubuntu forums to give support to new people, i'm often having to drag up links for Sudo and enabling extra repositorys and the like, so I was wondering weather it would be a good idea to perhaps have certain tags that link to help pages on the ubuntu wiki?

for example:
```
using ubuntu you don't have to use a root account, you mearly prefix any command that you wish to run as root with 'sudo'; 
see this link for more information: [sudo]
```

would get replaced with:
[INDENT]using ubuntu you don't have to use a root account, you mearly prefix any command that you wish to run as root with 'sudo';
see this link for more information: [Sudo on the ubuntu wiki](https://wiki.ubuntu.com/RootSudo)[/INDENT]

as an added bonus, it also helps preseve dead links. if, for example, the ubuntu wiki moved from "wiki.ubuntu.com" to "ubuntu.com/wiki/" for some reason, everyone who had hard-linked to wiki.ubuntu.com would end up with dead links, with the bbcode all you have to do is change what the bbcode is replaced with :)
