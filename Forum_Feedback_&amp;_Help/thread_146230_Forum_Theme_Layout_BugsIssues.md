### Post by LKRaider on 2006-03-20
A bug I have been noticing ever more is the one relating to the **CODE** and the **INDENT** tags: they seems to disrupt the message layout everytime! (I'm on FF 1.5 here)

Example, check this thread: [http://ubuntuforums.org/showthread.php?t=142481](http://ubuntuforums.org/showthread.php?t=142481) (yes, it's one of mine :P)

You can notice that, on every post that the CODE/INDENT tags were used, the message got it's layout distorted in some way.

On the first post, you can see the message is larger than the forum borders (INDENT cause). Not only that, but the column that shows the user info is compressed, compared to the default (CODE+INDENT). The CODE tag used by itself displaces the left column by just a few pixels (compared to posts that didn't use it).
I believe this is buggy behaviour and should be avoided, as it disrupts overall forum layout and decreases readability.

On the flipside, the QUOTE tag works very well and doesn't disrupt the layout at all, even for long texts. (unless there is a CODE or INDENT tag inside the quoted message...)

Based on this buggy layout behaviours, I believe the CODE and INDENT tags should be revised.
