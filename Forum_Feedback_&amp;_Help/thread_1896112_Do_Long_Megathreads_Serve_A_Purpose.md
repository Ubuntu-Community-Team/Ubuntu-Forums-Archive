### Post by SteveDee on 2011-12-16
This post is basically a response to amjjawad concerning post#12 here: [http://ubuntuforums.org/showthread.php?t=1895005&page=2](http://ubuntuforums.org/showthread.php?t=1895005&page=2)

I started to reply on that thread, but quickly realised I was running off-topic. So starting a new thread here not only overcomes that problem, but also allows any "One-Stoppers" to criticise me.

I think the Lubuntu One-Stop thread is, at best, misguided. I applaud the amount of effort that amjjawad has put into this, but I neither use it nor support it.

My understanding is that forum threads should stick to a single topic (within reason) and there are benefits in keeping them short (again, depending upon the nature of the discussion) meaning that mega-threads should be avoided.

I would argue that the One-Stop thread is not about a single topic. "Lubuntu" is not a single topic, its a distribution with its own category. And with over 350 posts, surely this is now a mega-thread.

I tend to use Ubuntu forums in 4 basic ways:-
1. Looking for specific information; I generally do this by using Googlubuntu to search.
2. Posting; I try to use the most appropriate forum, with a meaningful title, prefix and tags (when I remember)...but I don't always get this right!
3. Lubuntu "news"; I keep an eye on reported problems for my distro by searching using prefix=lubuntu
4. Reading unanswered posts; when I have the time I search all ?buntu forums by "At Most 0 Replies" and offer help where I'm able.

I very rarely read or even try trawling through mega-threads (an EzCap thread being the only exception that I can remember!).

It seems to me that we don't need a One-Stop or a forum index, although improvements to the forum search facility would be welcome.

Any really useful and reliable information from any ?buntu forum should be added to the Community Documentation (a valuable resource which is often over looked).

So is the One-Stop bad or harmful? Probably not harmful, but to me, it just generates more forum "noise".

### Post by SteveDee on 2011-12-18
One of the problems with mega-threads is that it makes life difficult for me (a self confessed "Lazy Searcher") to find information.

If I just search on a keyword like "minitube" the presented search list is full of mega-threads like:-
"Guide to Get the Best Performance from the GMA 500" (4,800+ posts),
"Things that you hate in Ubuntu" (2,200+ posts) & 
"What are you listening to right now?"(12,000+ posts).
The reason is that the longer a thread, the more likely it is (by chance) to contain your keyword.

A single topic (technical) thread is more likely to burn itself out after 20 or 30 posts, especially if a solution is found to a problem. And I think it MAY be worth reading the whole thread to explore all options if you are trying to fix something.

Whereas a thread like "What are you listening to right now?" is never going to be read from start to finish (in fact, if it was held in a rolling buffer, where only the last 500 posts were kept, I doubt if anyone would notice!). Surely this is just another blend of Spam?

The question of whether Ubuntu derivatives (such as Lubuntu) should be given a separate forum is an interesting one. All ?buntus are just like templates. You pick a starting point (your installed ?buntu) and then add or remove stuff in order to meet your needs. Some problems you encounter are fundamental (or fundabuntu) relating to Linux/Buntu/apps/drivers, some are version related, and some are distro/desktop related. So although the desktops can be very different, there is still a lot of common ground. I wouldn't want to lose that knowledge pool by breaking out Lubuntu from the other ?buntus.

So I guess there's nothing for it, I'm just gonna have to knuckle down and use the forum search features properly.

### Post by SteveDee on 2011-12-19
Hey guys! The more I think about this rolling buffer idea (post#18 ) the more I like it.

The idea would be to limit all threads to (say) 100 posts. Once this number is reached, the first post would be transferred into an archive and its position in the thread would be replaced by a link.
For each additional post, the oldest post would be archived with the first, so the maximum number of posts does not exceed the limit.

To keep thread spammers happy, the actual post number will still be used (e.g. 11,999) even though only the most recent posts appear on the thread.

The default search feature would not include any archives, but there would be an option to search archives.

This should have the affect of improving the relevance of basic searches.
