### Post by fourchannel on 2006-08-02
I normally do not like having my posts come up as "Edited" whenever I forget to add something. But I've learned to live with it.

Plus, for the people who say some objectionable things, it lets the readers know that the poster went back and changed his post, otherwise there would be mass confusion if someone quoted, OP changed it, no evidence of editing, so ppl are confused if the quote or the original post was forged/altered.

So I guess I'm saying it's annoying to look at, but it stops confusion from running rampant.

### Post by coffeecat on 2006-08-02
Another Linux forum I use is powered by [PNphpBB2]("http://www.pnphpbb.com/"). I don't know whether it's a default setting of that software or whether it's a setting that forum has chosen, but it has a simple rule for the 'last edited by..' message.

If no one has yet pasted after you, you can go back and edit as often as you want without the 'last edited by...' message appearing. Useful for correcting typos or adding afterthoughts without cluttering up your post with an unnecessary message. But if a post has been added after yours, the 'last edited by...' message appears. That covers the situation fourchannel mentions when someone quotes something that the poster has subsequently edited out.

Would this be a useful compromise?
