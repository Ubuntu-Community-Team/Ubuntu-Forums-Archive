### Post by malarky on 2007-03-25
I think the potential for linux on ps3 is great; it's a fixed target, so a stripped down version could be developed for it that 'just works' and is optimised for the platform. 

I've tried the latest feisty ps3 live CD in my ps3 and I'm really impressed with it - it took very little to get it installed and working as a desktop machine (i'm posting from it now), and I'm now installing emulators and MMS on it.

To dismiss ps3 as an unsuitable platform for linux because it has 256Mb of memory is a bit silly. 256Mb is more than enough to run most 8bit/16bit emulators, decode sdtv resolution movies, run firefox etc, and I bet there are plenty of people out there running ubuntu on x86 boxes with 256Mb with no problems at all. See how much use people have had out of XBMC for inspiration; The original xbox has 64Mb of memory and a cut down PIII 733Mhz cpu. 

The inability to use the RSX chip is an annoyance, but CELL could be used in its place for many tasks; it's been suggested that sony originally intended CELL to be the graphics processor before they turned to nvidia. There are folks attempting to use the CELL SPEs for decoding and encoding h264 video, which would make the system very useful as a HDTV home theater box.

There's a lot of information out there at the moment, but no one place to look for newbies. I've decided to start a website called ps3linuxgeeks.org to pull all this together into one place, I should have the site up and running within a few days. 

In the meantime, I think establishing an ubuntu ps3 specific forum here is a great idea, and you'll be surprised at the amount of interest. It might not happen straight away, but if we can make it as easy as possible to run ubuntu on ps3, people will use it.
