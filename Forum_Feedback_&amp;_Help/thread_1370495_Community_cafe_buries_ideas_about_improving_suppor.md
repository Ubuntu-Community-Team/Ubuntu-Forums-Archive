### Post by peepingtom on 2010-01-02
Howdy.

I think there is a bit of a cultural problem on these forums. We have no place to discuss how the community can improve its support, and there is nowhere for potential tutorial writers to ask for advice.  Where we observe that bad advice is endemic, there is nowhere to put this on the record and try to solve the problem.

Ubuntu Forums is a good barometer for common user problems. We need a place to discuss trends because when a question is asked hundreds of times, it's clear that a tutorial belongs on the forum or wiki. We can't file bug reports for other users, especially when we need to determine if there's a bug, or if they badly followed a tutorial (or followed a bad tutorial). This type of problem is especially apparent during the upgrade cycle. Many PulseAudio problems were caused by little changes that users made before upgrading. I now see a lot of this with NetworkManager, with users being encouraged to use command line tools they don't understand, to replace important parts of the distribution ("Fixing" nm by replacing it with wicd, uninstalling PulseAudio, etc). We need a way to tell the Ubuntu Forums userbase that they are giving bad advice, and that it is hurting end-users.

The Community Cafe is the catch-all general discussion forum, and the threads full of banter will move up the queue so fast that important discussion is buried. There should be a place to chit-chat about important things related to Ubuntu support. I have several tutorials in-process, but I sometimes need small amounts of help finishing them. I already help people on the forums, but I could do a lot more if I could sometimes get attention from an expert! Ubuntu Forums should be training people to make decent documentation, you need only look at the wiki to see how often the documentation must be radically changed between releases!

There are a lot of people in this community who want to help others. Help them help them!

### Post by noway2 on 2010-01-07
I am a little disappointed that this discussion got dropped from the forum council today.  I think that the post alludes to a few points that could use some discussion.

1) The first comment regards places where individuals can learn to better contribute.  This subject was addressed and it was mentioned that these tools do exist in the form such as the wiki pages, etc.  As I have become more versed in Ubuntu and Linux over the last couple of years, I find that I too would like to make a bigger contribution and I would have liked to have discussed means and opportunities of doing so.  I have begun reading the developer and wiki pages, but even still find the how to start curve somewhat daunting.

2) The post is spot-on with the statement that the forums and the community cafe are an excellent barometer of what is happening in the user community and what kinds of issues users are running into.  I may be wrong, but the impression I got was that the author seems to believe that there is a need for a different or separate set of forums, possibly ones that are moderated that would be more geared towards "advanced" users.  It is this idea that I would like to address.

My first thought was that I can see his points and I even agree that there is a bit of a problem, especially in some of the more topic oriented forums like the server forum that the same questions are getting asked again and again.  I also agree that many of the threads are quickly getting swamped and that there is a problem with the amount of thread bumping going on.  I even think that the forum admin should consider addressing these "problems."  However, after thinking about it for a while, I have come to the conclusion that splitting the forums in the manner that (appears to be) suggested by the OP is not the right solution, for several reasons.  First and foremost, I feel that this goes against the whole Ubuntu Community philosophy.  Second, this would be counter to what sets this forum community apart from so many others: that it is a place that welcomes beginners and experts alike in a respectful and educational environment.  

On further reflection, especially about the part that the same questions are routinely asked multiple times I think that "We" as a collective group need to take responsibility for this.  In this regard "We" could make a more involved effort to direct the person asking the questions to the wiki and how to pages, we can  politely remind them that these questions have been asked on multiple occasions and point out the search functions, and once this has been done the administrators could consider marking the threads closed.

At the same time, I would like to propose to the forum staff that some of the forums that seem to be gaining in popularity, such as the security and server forums could benefit from some sub division.  For example, the server forum could have a sub-section for web servers (apache), DNS and DHCP related problems, etc.  I suggest this because I think it might help to keep some of the discussion threads from loosing scope so quickly as well as helping to direct queries regarding frequently asked topics towards a common area.
