That is very strange. I presume you have your message editor set to WYSIWYG mode? I usually have my editor set to Standard Editor. If change my settings to Editor = WYSIWYG and then try to edit post #3 I get a blank like you. If my message editor is set to standard editor mode, then I see all the text of your post, including BBCode, and would be able to edit it. If I change to WYSIWYG while viewing the source of your post, it goes blank. I guess that QIII has their editor set to standard mode, which is why they can see your text in the edit box. Most of us old-timers here use standard mode - WYSIWYG in the message editor is simply an annoying nuisance and I do not recommend it. 

This is just the latest nonsensical bug involving the WYSIWYG editor to come to light recently. Let me give you a workaround, since I very much doubt this obscure bug in what is now an obsolete version of vBulletin would get addressed by the vBulletin developers.     

Click on Settings top right of page. Then General Settings under My Settings in the left pane. In the main pane of General Settings scroll down quite a way to find Message Editor Interface. Change your message editor to Standard Editor (not Basic editor which is far too spartan). Don't forget to save changes. Now explore that a bit with new posts and in edit mode. Advantage - you see the actual BBCode. Disadvantage - you don't see what you get. :wink: But you can always toggle between source mode and WYSIWYG mode with the first A/A icon in the message toolbar.

Oh, and did I mention that WYSIWYG in the vBulletin editor needs to be avoided like the proverbial plague?

**Edit:** and now I'm getting some odd happenings in WYSIWYG mode. However I haven't been able to reproduce the blank page problem with posts with code tags other than your one. But the problem persists in yours. I'll do a bit more digging when I have time but in the meantime my advice is the same: use standard editor mode.
