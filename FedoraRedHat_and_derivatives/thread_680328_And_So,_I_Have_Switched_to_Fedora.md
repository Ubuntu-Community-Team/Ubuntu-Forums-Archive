### Post by mrgnash on 2008-01-27
[img]http://img146.imageshack.us/img146/4881/fedorasu9.png[/img]

It just seemed like the right time. With each major release of Ubuntu, I find myself frustrated by the emergence of new stable release of some or other application which is often not included in the backports, yet alone the main update repositories. I understand and certainly approve of the reasons that the Ubuntu developers do this, but it doesn't suit me. 

What ends up happening, is that I migrate early to the development version, and wade through the various bugs just so I don't have to wait for all the new improvements and features. This is a pretty extreme route however, and it has often compromised my productivity, so Fedora seemed like a good compromise. 

After having installed Fedora 8, and used it for a couple of days now, I must say that I am very impressed. It seems a whole lot snappier than Ubuntu, and I appreciate the fact that there seems to be an even more Gnome-centric approach than the one adopted by Ubuntu.

To be fair, I do have a few minor quibbles with a number of bugs I have encountered, and the package management takes some getting used to after the joy that was apt/Synaptic... Codeina is also a pain in the behind, even though I understand why they are going down this road (it wasn't very hard to get around, however). But those things aside, I am really impressed by the overall polish, and the way it correctly identified my hardware from the get-go. The added security features are good for peace-of-mind, and I love being able to use the latest versions of my favorite applications without all the fuss. 

I am keen to see how Hardy turns out, because I very much enjoyed my time with it, but for now, I think I'm going to stick with Fedora :)

### Post by igknighted on 2008-01-30
> **tubasoldier said:**
> I'm using Fedora on my laptop. I took Ubuntu off of it with Gutsy, too many bugs to sort through.
I also agree that they are pushing the distros out too fast. Once a year is plenty, with their three year LTS version. I would love to use the LTS version but the problem is that it is three years old. There is updated software that I have grown quite accustomed to and it would be a bit hard to go back that far.
Perhaps six months after Hardy comes out I'll upgrade to that and then stick with it until the next LTS comes out.

See, thats the thing... 6 months after hardy, very little will have changed.  So far as I know, Ubuntu doesn't push bug-fixes through, only security fixes.  For all intents and purposes, once a release is out, all development focuses on the next one.

Fedora, OTOH, doesn't forget a release once it is out there, however the constant upgrading of applications can lead to bugs as well (the focus is on adding features, not necessarily fixing bugs).

For the second time now, both Ubuntu and Fedora, when released, could not properly run on my laptop.  However, by mid-life, upgrades that hit Fedora made it work perfectly, while in Ubuntu nothing has changed.  Both times the kernel was the issue, and Ubuntu doesn't change that.  In Fedora, a new kernel is out roughly every couple weeks.  Remember the "crisis" last fall about everyone's nvidia drivers breaking?  Well, that was a kernel upgrade.  Imagine if Ubuntu upgraded every other week?  It'd be a nightmare.

Whats my point?  There are really good reasons to do things both ways.  Obviously there are advantages to upgrading mid-release like Fedora does, but it takes much more knowledge of what to upgrade and when in Fedora.  It just adds to the complexity new users are faced with.  By releasing only security patches, Ubuntu cuts down on the complexity, and by pushing out releases every 6 months you are never too far behind.  It's a compromise, no doubt.

### Post by pelle.k on 2008-04-04
I thought i'd throw in my 2 cents, since i have been enjoying fedora for a while now.
1. it *diffrent*. some may call fedora broken, but it's just different. When you got the hang of it, it's just like when you're administering debian/ubuntu. I do prefer the debian packaging helper scripts + pbuilder to the rpm build tools and "mock" though.
2. Livna isn't always in sync with fedora, so you may not be able to install nvidia-kmod. I used a dkms package on freshrpms.net called x11-nvidia-drv (i think that was it), and it compiles a new nvidia modules in case a new kernel is installed automatically instead. very cool.
3. There is no gksu(do) in the repos, and that sucks. Fedora uses consolehelper, and that has to be set up with pam for every binary that is run with it. Hopefully policykit will replace this need in the future.
4. the yum gui (pirut?) component sucks, but yum itself is rather nice with the plugin system it uses(delta rpms, with yum-presto is cool).
