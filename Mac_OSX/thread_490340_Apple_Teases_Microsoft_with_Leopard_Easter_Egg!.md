### Post by izanbardprince on 2007-07-30
Actually, Apple is a little over 7 years late with that joke, I can count on one hand the number of times I've seen the BSOD on Windows XP, and I have yet to see it on Vista (although I saw it once when I was testing Aero Glass in Vista Beta 2).

As far as the whole OS locking up, I've experienced it more with OS X Tiger more than with XP and Vista put together, and as far as programs hanging, I'd say Tiger is up there with Windows, I've had to pull out "Force Quit" (Which is prominently displayed in the OS X interface) quit a few times, and thats just with running official Mac OS software, not to count what's happened when I tried to use the Linux compatibiility layer.

As far as viruses, Apple is correct that Windows has significantly more, except that 9 times out of 10, the virus is designed to exploit a security problem that Microsoft fixed like, the year before, that people didn't patch, another reason there's so much malware running around is other poor computing practices like disabling UAC and/or not bothering to run anti-virus software at all, and even though the viruses on Mac OS haven't been able to propagate quit as easily, a Mac OS virus is entirely do-able, Apple releases  dozens of security patches each year, the difference is that when a hole is found in OS X it's because of a proof of concept worm, nobody really bothers to write a virus to turn OS X into toast because it has something on the order of a 2% market share of home computer users, and probably around half a percent on server use, for the longest time after OS X was out, Apple was running it's own website off of Red Hat Enterprise, and was only tacitly shamed into "switching" to OS X Server because it was embarrassing not to.

As long as Apple keeps OS X overpriced, closed source, and bound to ridiculously expensive hardware thats 99.9% identical to a PC you could build for half the price, OS X will remain entirely marginal.

I'm not saying Windows or other operating systems are perfect, but when you run into a problem with them, it's usually because they have to run on all the hardware under the sun, while OS X is relatively easier to use because it's essentially an ivory tower in a walled garden.

### Post by izanbardprince on 2007-07-31
Vista's new driver model fixed a lot of that, Windows XP had become so stable that the only thing left crashing it was poorly written 3rd party drivers corrupting the kernel, but now that Vista runs most drivers in user space, the BSOD should largely be a relic.

Windows 98/Me were a house of cards, Microsoft kept trying to get an obsolete kernel running on top of an OS from 1980 to keep up with the demands of modern hardware, because if they released a better OS, people would complain that their applications didn't all work anymore, so really they had to prioritize backwards compatibility over technical superiority and QA concerns, I don't think most people appreciate the level of work it took to get the NT kernel to run most consumer programs and games and legacy apps from Windows 9x.

The situation is really no different than Mac OS X vs the totally horrible Mac OS Classic, of course Mac fanboys were willing to apologize for the Mac Classic kernel getting obviously long in the tooth, and increasingly unstable, of course OS X made everything better by switching to a modern kernel that was appropriate for modern hardware, but I've used Mac OS back to System 7.5.2 and I'll tell you that until OS X was released, Mac OS made Windows 98 look like a picnic, I had problems getting Mac Classic to mount Iomega Zip disks, I had trouble with the multitasking system, it didn't schedule CPU timesharing right and made the whole OS feel clunky no matter how much RAM you had, and god help you when it DID finally run out of RAM and went after the swap file, a lot of times the system would just slow down to a crawl and then lock up right in the middle of something, and you had better hope you had an autosave feature in your office suit, because if the OS didn't lock up, it would sometimes just decide to reboot for no reason at all, then when I had a nice collection of software and games for my Motorola Mac, they switched to PowerPC, then they did the same thing to their PowerPC users and switched to X86-64, so there's practically no backward compatibility at all on a Mac, and if your program is more than 2-3 years old, it's a coin toss on whether it'll even run at all.

From an objective point of view, the Mac has tended to be the most expensive consumer platform to choose, with the least return on investment, and the user experience goes from mediocre to nerve wracking.
