### Post by kingleer on 2008-04-07
Good job on Ubuntu 7.10 guys. I'm eagerly awaiting Ubuntu 8.04. I'm a linux noob, but I've been using Ubuntu as my only OS since 6.04, and I've learned everything I know about linux from Ubuntu. 



THE PROBLEM: 

One thing I really like about Ubuntu is how everything works out of the box (sound etc) no matter which computer I install it on.

Everything except my wireless card that is. I know how to use ndiswrapper now, but I remember it taking some kind soul on the Ubuntu forums days telling me how to get my wireless card working using ndiswrapper. I was pretty close to giving up on Ubuntu until three days after my first install I got my wireless card up and running. I would have given up if it wasn't summer vacation. I need a working computer with wifi especially if it's during the school year. 

It was definitely very frustrating, and I'm a very patient guy. I'm fairly sure that I'm not the only one who has had trouble using ndiswrapper for the first time.  The networking board is full of people new to Ubuntu struggling to get their wireless card up and running. 

Today, I have no trouble using ndiswrapper. It only takes me a couple of minutes to get a wireless card working on a new computer. 

I know Ubuntu has several guides online for specific cards and the forums are great, but it'll seem like a lot of work to someone new to linux. People want things done instantly these days and Ubuntu does a good job of that. 






SO WHAT I PROPOSE: 

Is an extra "diagnostic program/diagnostic disc" to come default with ubuntu.  This tool could be something included with Ubuntu or released as another iso depending on how big it would be. 

WHAT THIS DIAGNOSTIC TOOL WOULD COME WITH: 

Ndiswrapper, a tool for identifying wireless cards, and a complete set of instructions for what to do to get every wireless card up and running that is known to work in ubuntu (a summary of all the tutorials already available on the Ubuntu hardware support site). 


WHAT THIS DIAGNOSTIC DISC WOULD DO: 

The user would have to at least be connected  to the internet via an ethernet cable (not wirelessly). This diagnostic tool would make everything automatic. 

It would automatically install ndiswrapper.

It would automatically determine the wireless card that comes with the computer. 

It would inform the user which driver they would have to download or attempt to use the one that came with their windows machine (which sometimes works). 

This tool could attempt to download the appropriate driver automatically from a database/list of links.  

After providing a driver for ndiswrapper, this diagnostic tool would take over. 

This diagnostic tool would complete the appropriate procedure needed to get the users specific wireless card up and running. 

"If wireless card equals b then 
use provided driver to do procedure C." 

So this diagnostic tool would know what to do for every wireless card supported by ubuntu to get it up and running (these instructions would come included with the program or it would download them once the user was connected to the web).

WHY THIS IS GOOD 

All the user has to do is click a button after providing a driver or being directed where to find a driver by the the tool. 

No mucking around in the terminal which would be very intimidating for noobies to linux. It would make installing a wireless card about as easy as it is on windows or even easier since everything is being done for the user. 


SUMMARY:

This tool I'm proposing would be  like the "Add/remove menu" for installing programs except for installing drivers for wireless cards. 


Conclusion: 



So, is my idea feasible? 

Even if this course of action isn't chosen, SOMETHING NEEDS TO BE DONE. 
Getting wifi working in Ubuntu is too difficult for someone new to linux. 

I can remember trying to get several of my friends to use Ubuntu, but all of them telling me that getting their wireless card up and running was impossible so they gave up.

Also, I'm not a programmer. I don't know anything about programming. I'm just telling you what I would like to see from my perspective as an Ubuntu user.
