### Post by ddaavviidd on 2009-01-14
One thing I lack in Linux is a good information organizer like the one Mac users seem to have in [Yojimbo]("http://www.barebones.com/products/Yojimbo/"). It's an application where you can throw all kinds of files, text snippets, links or what have you, and organize them according to subject.

The only thing I've found that is remotely similar is [Incollector]("http://www.incollector.devnull.pl/"), but it only organizes texts and url links, not images, PDF's or other things. I'm studying urban planning, and I collect a lot of items on that subject, both in form of texts and other documents I've downloaded or created myself, and also in the form of links to articles on the internet. When it's time to write an essay or something, some of the information I will need are in folders on my computer, and some is on the internet. I'd like to collect and review all of those pieces of information in one place, in a fast and intuitive way.

I've come to think Nautilus could actually do the job for me, since you can drag links from Firefox into a folder. But it seems wasteful to both bookmark a site in Firefox and then also drag the link to a folder in Nautilus. And you can't even drag the URL from the adress bar or choose "save as link" in Firefox, you have to find a link to the page to be able to save it. For those reasons, I hardly think many people find it useful to save web links in folders locally.

So I had an idea for a plugin for Nautilus. I say plugin, because I'm not sure everyone would appreciate it as a default 'feature'. I couldn't program it myself, but maybe other people will find this idea useful. Or maybe it already exists??

The idea is that when you're in a certain folder, Nautilus would (in a column for themselves probably) find and display all browser bookmarks that are tagged with a certain word; by default it would be the name of the folder.

So that when I go to ~/urbanplanning, where I keep all PDFs, OpenOffice documents, sound recordings and images relating to that subject, Nautilus would also display a list of Firefox bookmarks tagged with "urbanplanning".

How about it? Any thoughts? Is it a horrible idea? Could I achieve this sort of "local documents/bookmarked documents" integration some other way?
