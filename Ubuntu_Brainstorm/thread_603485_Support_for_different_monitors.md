### Post by Zugzwang on 2007-11-05
Hi all,

many Laptops, especially those with built-in 16:10 LCD come with a Windows version & display drivers installed that manage to adjust the screen resolution/refresh rate automatically at boot-up depending on whether the internal LCD is used or an external LCD is used (which happens whenever one is connected before booting the computer). Dual-Head is only used when actually switched on.

This doesn't work in Ubuntu so far. The X-Server screen resolution is always that that was used the last time. 

It would be pretty nice if the GNOME screen resolution GUI that's new in Gutsy would provide the possibility to change the settings for different monitors and then checks at boot-up time which of those is actually connected.

By the way, the 16:10 resolution does not appear in the GUI at all at the moment, even when selecting a 1280x800 LCD screen as monitor type manually, but that's another issue. See: 
[http://ubuntuforums.org/showthread.php?p=3652697#post3652697](http://ubuntuforums.org/showthread.php?p=3652697#post3652697)

This feature appear to be missing for other people as well:
[http://ubuntuforums.org/showthread.php?t=352902](http://ubuntuforums.org/showthread.php?t=352902)

Again, this is *not* a dual-head issue.

On my Toshiba Sattelite PRO A30 running Windows XP, it's also nice to switch the resolution by just pressing FN+SPACE, which is quite useful when preparing to show a presentation or so on. But this is not so important. :-)
