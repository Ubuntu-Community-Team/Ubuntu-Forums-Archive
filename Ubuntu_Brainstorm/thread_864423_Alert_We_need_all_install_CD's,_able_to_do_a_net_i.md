I am convinced. We need all (next) install CD's, able to do a net install; when aged.

Pre-installation, may not be the panacea that many dream it will be. The time is now, to increase needed users and they will try open software when they perceive no losses, much more benefits, and a no-risk dual-boot, easy install.

The problem is, all this saturation of old install CD's (live or not) is near useless. Installing old stuff is backwards, compared to showing the newbie what open software has to offer today.

Respecting the hard work of managed release, dist-upgrades. A clean install is **ALWAYS** better. We are (in effect) asking new users to work it out, with an old CD (too often) and then the managed dist-upgrade which then too often fails. Even a 70% dist-upgrade "success" rate (worse when new and needed) is not good for open software promotion. Even with a current and new install CD (download time required) working out imperfections is a lot to ask the seeker. 

Great success and progress has been made with the current and great (K)ubuntu install CD's. Yet, where has the no-risk (once backed up) dual boot install option gone (on the "Live" CD?) Let people see the difference and on the exact same hardware! We can take it. Microsoft can not.

You see what happens is, a user gets peeved at one and then uses the other. Then, only to get peeved at it, and then go back **TO THE BEST**! This is the only way people are going to come to the conclusion, (K)ubuntu is the lesser of evils. Then the user will come in droves. We simply offer to many more advantages! Most people would say, "You had me at no anti-virus or Spy-ware checks needed".

Of course great care with release progression is needed. We can't leave wanted benefits behind with new (better) releases. I mean, if we can't keep the real benefits **and also** get new ones, why do a new release? 

People need help getting over their fear of change. That is, once the need is realized. No perceived need equals no adoption. No adoption by the 75% of people that are not technical, equals obscurity and stubborn standards battles.

We need this. Open software will prevail. Yet, at what speed? That is the question.

Please make a simple, universal, net boot type install (as a simple addition to all Ubuntu CD's), and have it automatically kick in after a set amount of time. This way, everyone with broadband may use even old CD's to install the current release without any more needs or complications.

I note that newbies are way to critical at the initial install and often don't make it into, the say, month or so of heavy use, before they see the greater number of benefits, only (K)ubuntu (and family) provides. 

That's current reality. What are we going to do about it?

Why not, do it?

It would lead to higher bandwidth costs, as well as higher CPU consumption. A LiveCD as it is now is compressed to 1/4th the size of the installed system. If you had to download each package from a repo, you would be downloading at least twice that in debian packages, which can't be compressed that much currently. Also, you would have to install each debian package in order, which takes a lot more CPU power than installing an image off a disk like is currently done. The cost of more CPU power means that it would take more time for it to install off the disk. The biggest reason I am against this is that we wouldn't know what the name was for the latest release until a month before it was going to be developed. This would make it hard for a newbie who doesn't know of Ubuntu except by "Ubuntu 8.04", or "Ubuntu 7.10".

Also, what if the newest release doesn't work on their hardware? They won't know until after it is installed whether it works or not, and if it doesn't then they would most likely say that their time was for nothing and be frustrated. Also, if they use the live portion and that doesn't work, they might think that the latest one wouldn't work either, thus why would they take the time to install it via the net?

Another thing, what would happen if, after a while, the net install portion of the CD became corrupted and no longer worked? They would have to download a new ISO anyway. And if the servers change after a little while so that the list of servers is outdated? Again, they may not get it to install and would thus become frustrated (especially if they were a newbie). Since it is an old CD, there are bound to be incompatibilities after a while, which Canonical would have a hard time controlling since there can still be some with a few year old Ubuntu release on them. :(
