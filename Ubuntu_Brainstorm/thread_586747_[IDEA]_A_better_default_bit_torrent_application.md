I'd like to comment here as well, because I see this as an important dilemma. First, let me state that I'm in favor of Deluge. I agree that Deluge is far too complex for the casual user. The casual user only needs to know how long it takes to download the torrent, how fast it as downloading, what % has been downloaded already, size of the torrent, and how many peers/seeds are available i.e. how 'healthy' the torrent is. They only need to know 5 things, no more. In that respect Deluge is overkill, and lacks the simplicity (and possibly usability, depending on how you see it) associated with GNOME applications. A more simple client is needed.

However, the current client, gnome-btdownload, might be simple, but it's totally useless in certain cases. Nowadays lots of people are behind routers, and routers require that you either forward ports or use a Bittorrent client with UPnP enabled if you want the client to download/upload with proper speeds. Otherwise, your download/upload speeds are close to zero. With so many people behind routers, including me, and the difficulty of forwarding ports for the casual user, that makes the current client gnome-btdownload effectively a useless piece of software. In other words, Deluge currently is the only viable alternative, even if it's not the best for the casual user.

What zachtib, the lead developer of Deluge, is proposing here, is probably the best idea. Make a simple GUI for Deluge which is completely GNOME-ified and more suitable for the casual user, and use that to replace the current default client.
