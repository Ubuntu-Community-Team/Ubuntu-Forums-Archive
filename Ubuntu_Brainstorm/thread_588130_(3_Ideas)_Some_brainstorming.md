### Post by sweRocker on 2007-10-23
[LIST]
**"Take a course in something": **a platform to collect tutorials and making them look similar to steepen the learning curve for new Ubuntu/open source users. It could also be a way to organize the many hardware how-to:s in all the forums, where it is sometimes hard to find what you are looking for. Facts and knowledge can be found everywhere, but it requires hours of searching and once found its usually organized in a step-by-step guide without telling much about it, or it is pure facts. A course should be something in the middle. Facts and step-by-step guides should generate automated tests and trials, just to make the info stick better. 
Also a learning tree with the courses as nodes so that things are learned in a thought-through order. Adding a new "course-node" should be very simple and also possible for different applications and combinations of them. 

** "Desktop-Gimp":** applying setting "to" desktop stuff instead of "through them", using a palette similar to the one used in gimp.
       *Painting a custom folder: choose tool "paintbrush"> chose icon>klick on a location, write a name> done 
       *Paint the background: choose the tool "paintbucket">choose a image/color>click some area (could be desktop background, a panel, a folder etc..)> done
       
[*]Project folders: Project folder are just like any other folders, but makes backups of themselves each time its closed. Later on, its possible to go back to a previous version. (euhm, see Leopard for further reference...) 
[/LIST]
