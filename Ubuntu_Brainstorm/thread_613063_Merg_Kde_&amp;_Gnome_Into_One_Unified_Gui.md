### Post by Rul on 2007-11-17
The thing is that it's not only technically difficult to merge KDE and GNOME, the thing is that they have different goals and they target different audiences.  

KDE aims to be highly configurable and really cares about integration, KDE apps tend to be powerful all in one apps. So KDE users normally want apps that work exactly as they want and like to be able to do a lot of things with the same application.

GNOME on the other hand tries to be as simple and intuitive as possible and follows the philosophy that each program should do only one thing but do it good. So GNOME users like simple UI and really don't care how things are done as long as they are done right.

Obviously this are just generalizations and are not always true but I think it pictures some of the essential differences between KDE and GNOME.

 In my case, I just love the way KDE works and how I can make it do exactly what I want, and I just hate the way GNOME does things and the lack of configuration options and the boring UI, but I can understand that some people might feel just the opposite way.

I don't think that having 2 (or more) UI makes linux more complex. I mean if users feel confused by computers and don't like having to choose they can just stay with defaults and that it's. On the other hand if you only have 1 UI and follow the idea that 1 size fits all you are always going to find unhappy users which is one of the reasons why people try using linux.

If GNOME were the only thing linux had to offer me I would be a really unhappy user and maybe I would be using windows again now. But with KDE and really happy and I believe I'm going to stay with linux from now on.
