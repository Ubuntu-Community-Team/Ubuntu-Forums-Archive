### Post by SunnyRabbiera on 2008-11-16
From ibex and up the useful took of displayconfig-gtk is removed in favor of a lame monitor tool that can barely get the job done right.
The default screen resolution application in gnome is pitiful and there is not a way to easily configure a monitor in KDE4 either.
Now I heard the excuse that displayconfig would mess up xorg and make the system buggy.
Well my reply to that is so can some newbie who has no idea how to manually edit xorg.
I mean come on we are not going to encourage new users if they find out they manually have to edit a file they have no idea about.
My case is that editing xorg is just as dangerous as using displayconfig and tools like it, we need a proper gui frontend for xorg to help newbies again or Ubuntu will loose out.
