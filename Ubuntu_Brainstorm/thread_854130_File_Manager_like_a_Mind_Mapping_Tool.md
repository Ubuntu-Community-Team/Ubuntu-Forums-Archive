### Post by plasmatron on 2008-07-09
Hello everyone. 
I will throw my idea/wish at the wall and see if it sticks.

The idea/wish is a file manager that presents directory structure similar to that available at mind-mapping tools. I know most of you would think that this is already present in tree-like form in almost every file manager, but let me tell you it's quite different. 

If your dirtree has many branches and many levels of sub-branches, a side-panel view is not practical. And not only it is not practical, it is difficult to get the whole picture, i.e., the idea or logic behind the tree structure. Moreover, this kind of view could easily allow for showing linked leaves indicating the presence of symbolic links in the tree structure. 

I think of it as showing a "mind-mapping tool--like" (i don't know how to call it) structure of the directory tree up to a certain (customizable) degree of detail, and then the user could possible expand the view in order to attain further detail in a particular branch of the view, but always having the whole-dir-structure view at hand.

This would also make far more easy to re-structure your dir-tree, simply by moving a branch of the diagram. (It is obvious that it should also have other standard capabilities i don't explicitly mention like delete, rename, etc.).

You see, the idea behind this is ORDER. Being able to naturally know where a file is without having to run a whole-disk search.
Being able to know where to put a particular file you just created or received from a collegue. Knowing which part of your hd you need to backup. 'Order' and 'the full picture', which i could also translate as coherence. 

As an physicist and, mostly as an experimentalist, i deal with many many files belonging to different projects. And even inside a project, I manage a huge variety of file content, such as: electronic receipts of an equipment bought for the lab, results of experiments, drafts, notes to papers, notes of seminar, my own lectures, articles from journals, computer code, results from simulations, raw data, processed data, graphs, etc.

And i've learnt that efficient file-managing is not about the files themselves, but mostly about the directory structure you lay out. 

Thanks in advance, criticism will be a lot more appreciated than just plain lack of enthusiasm.
