### Post by Turgon on 2007-10-22
When you plug a digital-camera into a ubuntu computer you will automaticly be asked if you want to import pictures from it. It then uses some gnome tool to do that and it does the job and gets the pictures to a folder of your desire.

But then, if you are like me, you got 8000 pictures and a desperate need to organize them. F-spot is a nice tool for this and has been included in ubuntu for a long time and is meant to be the way to work with pictures in ubuntu. To me, it therefor seems a little strange that f-spot is not used to import the pictures by default. It would be nice if  could add tags, perhaps remove some red eyes and adjust some colours at importing time, instead of importing them to a folder, then import the folder in f-spot after that. 

So my idea is that f-spot is made the default import tool for digital-cameras and by that make it quicker and easier to import and organize new pictures.

Edit: Since some people doesn't use f-spot the default behaviour should be changed from the date based folder structure to an event (or whatever you like) folder structure so that thoose who dislike f-spot doesn't loose anything by having it import their pictures. I think this is a better way to do it anyway, since it gives better compatibility for other apps.
