### Post by Habbit on 2008-07-05
> **Robin T Cox said:**
> There's just one or two small practical problems about rolling your own 2.6.25 (etc) kernel in Hardy: primarily that (on my system at any rate) Hardy randomly locks up long before the process can be completed. And I doubt that it's hardware, since I'm running Gutsy in the same box from another drive, and it operates flawlessly.

1.- Does this also happen with the Hardy Live CD?
2.- You can roll your own Hardy 2.6.25 kernel from your Gutsy installation: mount all your Hardy partitions under some mount point (I'm assuming /mnt) and do the following:```
/mnt$ sudo mount -t proc proc ./proc
/mnt$ sudo mount -o bind /dev ./dev
/mnt$ sudo mount -o bind /boot ./boot #Assuming the boot partition is shared
/mnt$ sudo chroot . /bin/bash
/#
```You are now within your Hardy environment as root (if you want to switch to your user do: "sudo -u yourHardyUser -i"). Now you can configure and compile anything you need without risk locking and then use the Debian kernel packaging utilities to build a .deb package for Hardy.

PD: This might be overkill, maybe you can compile the kernel from Gutsy and then chroot into Hardy to build the .deb package, but I don't know...
