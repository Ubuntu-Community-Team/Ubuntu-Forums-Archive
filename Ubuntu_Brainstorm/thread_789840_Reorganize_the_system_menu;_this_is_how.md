### Post by Redrazor39 on 2008-05-10
GNOME-Control-Center (Located in System > Preferences > Control Center for some reason) is a well organized settings manager for the GNOME desktop and Ubuntu. The System Menu itself is very concise, including only six options, two of which are the actual settings menus. When you get to the submenus (Preferences and Administration) you hit a nightmare of a menu. There are so many options that all sound the same. It's just too confusing. In the control center, everything is organized into six meaningful and informative categories that are organized as one would expect, just as the Applications menu is now.. The System menu should be the same. I propose that when the System button is clicked, the user sees the submenus "Personal, Look and Feel, Internet and Network, Hardware, System, and Other." The category called System may have to be changed to something different from the main menu option in order to not confuse people, or the System option in the main menubar should be changed to "Settings" instead. Then, after those six options are included with their options within, there should only be a "Help" option in the main System menu. This should have a button to open a window that is tabbed to include both About Ubuntu and About GNOME all in one. Lastly, the "Quit..." button should remain. Some people do not like the shutdown button and remove it to use the quit button. There should be an option for the "Quit..." button to either open the window to Log Out, Lock Screen, Switch User, Suspend (which should be renamed to Sleep or better yet, something unique; Pause. The icon should be a green vertical equal sign), Hibernate, and Shutdown, or turn the "Quit..." button into a "Quit" menu that includes all of those options and icons within a submenu. After clicking on one of these options in the submenu, there should be no confirmation; the action should begin immediately. It is extremely rare for someone to navigate two menus and click shutdown by accident.

That is how the System menu should be reorganized. It would increase efficiency, ease of use, and would look much better.


It has already been filed as a blueprint.
