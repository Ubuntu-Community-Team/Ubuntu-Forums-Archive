Welcome to the wonderfull world of ubuntu :D Its nice to see some newcomers to ubuntu/linux understand that emulation isnt the way forward when it comes to getting gaming up to scratch on Linux.. 
I think what you were trying to say when you compared the current state of gaming in linux to games for windows is that it would be nice to have some sort of framework for game developers to use to make developing games for linux more lucrative, transparent and enjoyable. 
I have had the same idea and discussed it for awhile over at the ubuntu gamers arena thread. 
When I think about it some more I think that this type of effort shouldnt be ubuntu exclusive, if you were to make it ubuntu exclusive you would be just as bad as microsoft. 
Maybe an open gaming inititive should be founded. 
If your interested in games development under linux then some interesting things to look up right now are telepathy and ggz.. GGZ is a networking framework for multiplayer games and would take some of the hastle of yourself because you wont have to recreate the wheal by creating the networking to your game from scratch. 
telepathy is an instant messanging framework for linux, I wont go into details but what you should look up is telepathy-tubes.. Its still under development but it is somewhat useable. It allows you to add support for sending game invites and joining sessions over the supported instant messanging protocals supported by telepathy. AIM, MSN, Google Talk etc. 
I hope you enjoy making games for linux, and yet again... Its refreshing to see a thread on gaming under linux which doesnt focuss on wine :D

oh yeah, one more thing.. The guys over at linux-gamers have set up linux gaming live cd's.. This was also discussed in ubuntu-gamers-arena thread under a slightly different use case... Bassically we wanted the ability to boot directly into the game from a live cd or play the game from the cd in your gnome session.
