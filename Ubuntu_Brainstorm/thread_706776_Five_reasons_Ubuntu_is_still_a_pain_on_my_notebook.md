### Post by TonyS on 2008-02-24
Hi all,

I've been an Ubuntu user since the beginning and am a loyal enthusiast. I use Ubuntu widely in my small business, and look forward to doing so more in teh future.

I note that Intrepid Ibis is focusing heavily on ubiquitous internet access, which implies a focus on notebook users.  But on the computer I personally use the most (a Thinkpad T61) Ubuntu continues to be a pain, relative to XP.

I know its easy to whine... I suggest its also easy to dismiss usability feedback as whining. Real, user-case experience is pretty important in designing a world-beating OS, right?

So, here's the "Top 5" things I wish were different about my Ubuntu-on-a-laptop experience:-

1. Suspend &/or hibernate just won't work correctly. Have tried everything on Thinkwiki, no dice. This is a basic user requirement on a notebook. 

2. Despite over a year of tinkering I simple CANNOT get Ubuntu to use my Treo 650 for DUN internet access using bluetooth. MAN I'd be happy if this worked. I have been surprised that, with bluetooth moved into teh foreground in Gutsy as a feature there seems startlingly little documentation available to allow users to actually use it.

3. Network Manager has dramatically improved, and (though its still too fiddly) establishing a working PPTP VPN connection is now fairly easy and works well. I wish there was a way (ANY way) to get the VPN to automatically connect whenever a network is available... this seems to be quite impossible. Ubiquitous internet would be greatly leveraged by ubiquitous network presence. IBM's excellent Access Connections is totally worth copying some ideas from!

4. I can't play encoded DVDs since moving to Gutsy, despite hours spent trying. I realise css is a hack, sort of..  but again, its pretty important on a laptop. Reading the forums its obvious this issue is widespread, but despite gutsy being two-thirds of the way through its primary lifespan I can find no solution, or no reliable documentation on this important functionality problem.

5. A common thread in most of the points above is issues in documentation. I have noticed that one nasty side-effect of the aggressive release schedule for Ubuntu is that documentation rapidly becomes obsolete, and in many cases you can't quickly tell which version it refers to. Over time my impression is that less is being documented, perhaps as its would-be authors reason that it has too short a lifespan to warrant the time. I'm hoping that Heron, as an LTS, will strike people as worth documenting more thoroughly... is there another system that might help? I find myself wishing Canonical would reduce to annual releases.


Cheers, guys. Perhaps forlornly, I hope that this report helps in some positive way.

Tony
