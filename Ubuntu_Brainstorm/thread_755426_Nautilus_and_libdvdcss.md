### Post by Sabru on 2008-04-14
After installing the dvdcss library files from the Medibuntu repository, DVD playback works great in most media players in Ubuntu. 

Cool. :)

However, you still can't do anything with a scrambled DVD under the standard file manager (nautilus). You can see the files, but there's no dragging and dropping as you would any other mounted drive, and certainly no "Copy Disc" function (which is incredibly useful). You just get an error, and the disc unmounts itself. 

Not Cool. :(

I'm not aware that this has been suggested, or is even being implemented by anyone, but wouldn't it be nice if Nautilus supported libdvdcss? Obviously, we couldn't ship it with the extension due to licensing problems, but if it used the library files the same way as, say, totem or VLC does, then all you would need to do is install libdvdcss from elsewhere at your own risk, and the disk encryption would be completely transparent!

How hard would this be to do? Should I be talking to the folks at GNOME instead?
