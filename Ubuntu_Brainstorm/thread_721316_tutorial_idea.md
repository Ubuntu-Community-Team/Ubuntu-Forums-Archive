### Post by Choad on 2008-03-11
ok i play a lot of a game called warhawk, and the tutorials it gives you are so simple but really effective.

basically, whenever you do something you haven't done before, it gives you information about what you've done.

example: bob logs in for the first time - the tutorial program tells him what he can do from the first screen, where the trash icon is, where to get to programs, where the system tray is etc. etc.

example2: bob plugs in his removable storage device - the tutorial program tells him that the device has been connected automatically, and that when he wishes to remove the device, he should unmount it by following procedure x, y, z to prevent data loss (click for a more detailed explanation??)

example3: bob clicks on <any internet based app> but his computer isn't connected to the internet bla bla bla

that third one you might want to remain active just because it would always be helpful to be informed of this. there could be some options for each tutorial tip like "always show this tip" and "don't show any tips" so that people who don't want them can easily disable them.

it would be a lot of work, but it would go a long, LONG way to making ubuntu truly accessible to everyone.

also, this could really take advantage of compiz to make it look brilliant. i don't think the default gtk toolkit is particularly suited to this job so a custom gui using cairo (?) or something would be cool.

### Post by coolen on 2008-03-13
I doubt it. Remember, Ubuntu is a multi-user system. Unless you wanted to add this information for every user during the installation process, you'd have to ask for it on their first login. It just seems messy.

The goal here is not to get the user doing all of the advanced stuff. You just need to let them know where everything is, to bridge the gap when migrating from Windows.

I'm just afraid they'll start pointing out the obvious. Just point them to the top bar, tell them that it's used for "accessing different programs/parts of their system", but don't tell them the difference between Applications/Places/System. They should be able to figure that out on their own.

It's just a hard balance to strike, and if you get it wrong, you'll either come off as arrogant ("you should *know* how to use the command line!") or sounding like you think the user's a moron ("did you press the enter key? I didn't mention that. You need to press the enter key here").

### Post by st0n3cutt3r on 2008-03-13
it is unlikely that we will have anybody converting to ubuntu who doesn't know how to copy and paste.  It would be really nice if we could have the system so user-friendly that computer ignorant people could use them, but....    you can only take that so far before you're better off handing them a lolipop, patting them on the butt, and sending them on their way.

A completely ubuntu ignorant person who has used windows before will be able to handle the basics of the interface. On the other hand, they're not going to (possibly ever) know about things like AWM, nor any of the non-obviously-named software, and they're not going to see much besides the basic installed software that comes with the OS.

Something that could be VERY useful would be a welcome screen, like vista, where it just pops up once and says "Hey, these are the new features!" with links to longer explanations, and just generally welcomes the new user. Included on that could be a link to a very friendly welcome to ubuntu that new users could click on to get a quick run-down on the OS, and then at at the bottom of the screen, a very obvious 'Click here to never see this again' and a tooltip that says where to find it again (or make it easily accessible... like in accessories?)

You'd just show it to each new user, really not "messy" at all, because each user will only see it once.  It's not like the administrator is going to sign in and configure everyone's accounts and then turn off the welcome screen for them... that would be ridiculous.

1 splash screen with links to more info for ppl who want/need it, and then....   I still like the idea of having different things explain themselves to you for the first time, although I suppose that for programs and things that's more up to the developers........   

Hmm... seems like there could be more done to explain/set up some of the system configuration options in a more user-friendly way, but my ubuntu's a bit dead atm, and I'm too lazy to list off a bunch of stuff in a thread right now...
