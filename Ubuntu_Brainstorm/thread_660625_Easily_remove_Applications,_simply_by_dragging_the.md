It does not matter whether the user already has rar, winrar or unrar installed, because it would be too tedious for the application developer to check for their presence and then download and install unrar if it is not present, so the application developer just includes it along with the application.

When this happens, it inflates the install size. If multiple applications installed on your system include the same core application in them (for example a lot of video conversion software on windows include ffmpeg instead of just having a single dependency on ffmpeg like on Linux) then that increases your hard disk usage exponentially.

This is why a lot of programs that are quite big in Windows (installation size wise) are only a few MBs here.

And then there's Perpetual's point - Windows does not handle dependencies. That is why the application developer has to give a warning in the uninstall part of his/her program that removing a particular dll might damage working of another application as it doesn't have a way to find out whether there is any other application that needs the dll or not.

These left over dll files, useless entries in the registry, multiple applications including the same core application, left over configuration files, and all the other useless stuff bogs down Windows over time.

I have never seen a mac in real life so I have no way of commenting about it.

Installing applications on Ubuntu is as easy as it gets. Uninstall is exactly the same procedure. What exactly is the problem here?

To make it uninstall like you said in the first post, I imagine a new entry would have to be made in all the .desktop files (and then there's the problem of some applications not making the .desktop files) that contains the information about which packages to remove when the menu entry is deleted.

Currently there are more than 22000 packages in the official repositories (and there are many unofficial repositories as well, but they are not in the base install of Ubuntu). Since individual application developers are not making the entry in .desktop files, I imagine this would be dumped onto the Ubuntu developers to add this information to all the packages. Boring task.

I have no idea how gobolinux implements this. I'll have to look into it.

Sometimes I just want to remove a menu entry but do not want to uninstall the application. Do you suggest having separate methods for both of these?

I may be wrong about some of what I said above though. If I am wrong about something, then please correct me.
