### Post by rockin_goliath on 2008-11-15
Absolutely not. I love the fact that I can just pop in an Ubuntu Live CD and have ALL my hardware work. No drivers to download, no extra OEM CDs to load (Dell's Windows drivers ](*,) ) and no trial-ware. If there is a driver I need, it's conveniently in the repositories.

When you start asking hardware makers to write their own drivers, and then tell them that they are also responsible for distributing them, then the drivers they write will inevitably be proprietary. If there is a problem with the driver, then the open source developers have no way of understanding what the problem is. We also have no way of overseeing the design of the drivers, so hardware makers will be able to make drivers as crappy as they want.

Finally, it will undermine the keystone of the open source movement: CHOICE. If we buy a computer pre installed with Suse, and we want to start using Ubuntu, we will be up **** creek if the laptop maker only provides/guarantees their hardware's functionality with Suse. Then, as more hardware makers start writing drivers only for ONE distro, the hardware support of all the other distros will be at risk.

We currently have the absolute best model for hardware support. It may not be super convenient for hardware developers, but it ensures that we have the best hardware support on the block. If you want your hardware to work on Linux (and more and more hardware makers do), you have two choices: Send the driver upstream to the kernel developers (as Intel does), or send it to the repositories of the individual distros (and make damn sure it works on the distros you package it for, I'm talking to you ATI).
