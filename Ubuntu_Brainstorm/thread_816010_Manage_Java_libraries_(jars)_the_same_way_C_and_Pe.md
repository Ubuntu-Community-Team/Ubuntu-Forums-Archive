### Post by rengolin on 2008-06-02
Every Java program I install on my machine has it's own /lib dir with the same jars, over and over. Why not have packages with the versions of jars and install them globally (like /usr/lib/java) and add that as the default class path?

I know that there are some frameworks that does that but not for desktop machines. Also, the maintenance would be big, but at least would be a single point maintenance.

I know that Java people like to work in parallel with the rest of the universe (some Java programmers told me absolutely not to think about managing libs because it's against the whole Java philosophy) but having multiple libraries in the same system (aka, Windows way) is just wrong and they're getting it worse with OSGI.

I guess it's time to clear their minds for a change...
