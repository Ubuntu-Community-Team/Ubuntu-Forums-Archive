### Post by Kazade on 2007-10-23
Just an idea that I have been thinking about lately. Where I work we use Active Directory for authentication when logging into Windows computers. Most of the staff have now switched to some form of Linux so now the set up is that Windows servers and the remaining Windows users authenticate with AD, whereas Linux users simply have their own account on their own machine.

I know its possible to configure Ubuntu to authenticate with AD and also to configure Samba to authenticate with AD for file sharing. But I've never managed to get it to work properly. My suggestion is that there should be a tab on the Login Window preference application that enables authentication with AD and or samba authentication. Something like:

[ X ] Authenticate with an Active Directory server?
[INDENT]     
Server name: _____________

[ X ] Authenticate shared directory access
[ X ] Authenticate users [/INDENT]

This (IMO) would help increase Ubuntu uptake, if users can log-on to their work network with no hassle at all they can convince bosses and others to switch. I know it would be useful where I work, I'm sure I'm not alone :)
