### Post by kragen on 2008-01-21
For experienced users, welcome screens are just annoying, and for new users welcome screens generally always fail to meet expectations, and shouldnt really be necessary in the first place anyway.

Generally speaking a welcome screen will only ever contain information which already exists elsewhere in the documentation - if this information (alonside the wider spectrum of documentation topics which cant be mashed into a single welcome screen) aren't prominent or accessible enough to warrant having the welcome screen in the first place, then that needs adressing before a welcome screen is really in a position to solve anything.

If people really feel the need to do a welcome screen, then I'd reccommend that some prototypes be made up and some proper usability testing be done, to identify what aspects of a welcome screen users find most useful (shortcuts to common tasks, or links to newbie documentation). My personal belief is that welcome screens cause more harm than good - as pieces of UI, theyre never designed well enough to be around the whole time, and so eventually whenever a new users gets rid of them, it causes more harm than good when they cant work out how to get it back!

The one acceptable exception to this rule would be the welcome screen you get at the start of apps like Visual Studio, Word, or Netbeans - the ones that are always going to appear when you first start up, and provide options that are are going to be aplicable to about 95% of users 95% of the time, but I gather thats not the sort of thing this thread is about.

### Post by Mr. Picklesworth on 2008-01-25
Hm... the installer *would* be a good place to put this stuff!

I was rather keen on a welcome screen myself, but the arguments against it are not bad at all. One thing such a screen could be really good for, though, is quickly setting up a desktop that is accessible for the user. (For example, if he has trouble reading the text). That shouldn't be considered a big issue, though; getting to the place in the menu is pretty straight-forward, if you know that the option is there.

I had been working on such a tour / setup tool, and can definitely see what you folks are getting at; even if it is trying quite hard to be unobtrusive, it still looks like an annoying, unwanted popup. Unless it serves many purposes (user account setup, for example), it will have that effect.

The idea of a desktop tour could be handled by a little slideshow over the live CD installer's progress bar. That would give people something to look at, be reasonably unobtrusive, and also be very scalable! We have about 20 minutes there for the slideshow, so quite a bit of handy information can be shown.

There wouldn't need to be much extra infrastructure, either; just a whole lot of images with accompanying plain text information. A nice bonus with that approach is there would then easily be room for community projects / contests to create slides for that presentation, and changes could quite quickly be thrown in for each release. Space shouldn't be a huge concern, either. Just a bit of text and some little screenshots or icons.

Really, the slideshow should do all that's needed, just so long as it mentions the right key words so finding the features later is easy. The important thing is simply that the user know what to look for, and an installer slideshow would do that fine in a way which demands no immediate interaction.

### Post by Mr. Picklesworth on 2008-01-25
Already hacking together a mockup in Python :P

My current hope is to keep it very, very simple. There is one special slide hard-coded into the app (special:welcome). The rest are selected at random, hopefully (some day) never repeated.
An important implication of that is each slide must be standalone. This means no technical mumbo-jumbo; just a quick glance at a component of Ubuntu.

Ubuntu provides quite a lot of unique things out of the box, whereas Windows' slideshow is built for Windows users, so there isn't much "new" they can talk about. On the other hand, this thing can sound really exciting to new users!

I'm thinking it would do to have an "option" file where slides can set up some extra features. Right now, "some" is one, and that probably won't need to change. Anyway, I'm pondering an option to add a button to the slide, with a title, which launches an executable file (be it gnome-app-install or a script attached to the slide). This way, the feature can be explored further, but not via lofty documentation; simply by seeing the program!

Each slide displays a single image, but I want to support mini-slideshows. That is, if the slide *does* have more than one image, they should each be shown in succession. In this way, it would be easy to demonstrate launching a particular application from the menu without causing visual confusion. (Frame 1: The menu item. Frame 2: The application. Frame 3: The application doing its magical job).

As for translations, I still have no idea how that works. I'm thinking it would be easiest to have an independent slide library for each language, but then again that could mean duplicating images, and it could be a completely bad approach because of how translation actually works...


Edit:
Good idea to end with that type of link, by the way. I must say I kind of like the modest "thank you" at the end of Windows' installer, (It's the best part!) but perhaps that type of "Learn Even More!" thing could be done tastefully.

### Post by germclown on 2008-05-14
When I'm setting up new computers of any OS for the technologically challenged, I just cover their desktop with icons. I rename the icons according to what they do, not what the program is named ('Evolution' doesn't shout 'e-mail' to the computer illiterate). I'll even put multiple instances of the same thing but with different names just so there's no confusion (Example: mother-in-law doesn't know yet that her new gmail account is internet-based, so I have two links to firefox: one for 'internet' and one for 'e-mail' which goes straight to gmail).

It's ugly as all hell to start with, but I've never had any complaints. Some of them eventually develop enough savvy to use the Applications or Start menus and the desktop starts clearing up.

The only thing I still can't get them to do is run the freaking updates (auto-update doesn't seem to always work). They usually have slow connections which would be fine for short doses of update, but most of them I only see every few months or so. I can't set them downloading for days on end and then leave (one elderly woman thought the delete key would erase anything that was on the screen at the time, no way am I leaving her to deal with any download/installation notifiers).
