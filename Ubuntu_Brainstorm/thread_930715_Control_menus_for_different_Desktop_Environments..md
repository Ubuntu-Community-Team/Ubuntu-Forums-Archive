### Post by keeler1 on 2008-09-26
Last time I checked, if you had KDE and Gnome both installed, all of the applications would show up in all of the environments.  For example applications like amarok (KDE) and rythymbox (Gnome) would both show up in KDE's menus and Gnomes menus.

It would be great if there was a way to have only the applications for a particular desktop environment available while in that desktop environment.

Possible ways I could think of to do this would be:

1.)Have separate locations for where KDE and Gnome apps get installed
2.)Have a Ubuntu control center which a user could turn this on or off or specify which apps to be available in each environment.

### Post by smartboyathome on 2008-09-26
The reason this happens is because GNOME and KDE use the same menu format (as do most of the DEs/WMs). And if someone wanted to use Amarok (for example) in GNOME, they shouldn't have to edit menus. It is annoying, I know, but when I install KDE/GNOME I just hide KDE's stuff using GNOME's menu editor and hide GNOME's stuff using KDE's menu editor.

### Post by dawynn on 2008-09-29
Bwaa!  I don't see the point.  But then, I also don't try to keep multiple bloated environments on my system at one time.  I used to be purely KDE, and I would just strip out everything I didn't want from Gnome -- including as much of the infrastructure as I could unload.  Now, I've swung the other way.  I use Gnome, and try to purge as much of KDE as I can from my system.

But either way, I still use Synaptic.  And I want that available to me, no matter what environment I'm in.  I also use Soundkonverter (KDE-based) and Kmymoney2.  Whichever environment I'm in, I'd like to see these tools that I consider important.

So, while for you, it may be nice to hide tools based on which GUI you're using at a given time, for me, I'd rather see every usable tool have appropriate links in the menu, regardless of which environment I'm in.  (So, if I choose to have Kate and Gedit installed on my system, I'd want to see both in all menus)

Please, please, please -- if you choose to go ahead with this, do not make it the default.  Some of us like to know *all* the software on our system.

In my case, my biggest lament is environments that do not have such well-developed automatic menus.  icewm and e17 come to mind.  When I fire one of those up, I'd like to see the same menu entries as I see in KDE or Gnome.

I'd much *much* rather see Linux move toward an integrated menu system where each environment reads from the same menu structure as every other.  If I create a new menu entry in Gnome, why should I have to rebuild it in KDE, icewm, xubuntu, fluxbox, e17, or whatever other environment I load onto my system?
