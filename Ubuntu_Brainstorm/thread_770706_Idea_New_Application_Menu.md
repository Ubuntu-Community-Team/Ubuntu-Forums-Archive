### Post by kragen on 2008-04-28
I completely agree: The Gnome menu remains difficult to modify and rearrange, and doesn't contain any usability aids (search or recently used icons). 

Also, when installing new programs, I never used to be sure if (and where) new launchers had been added to the menu, and generally speaking this is still the case. Although I hate the windows "new programs have been installed" highlight-thing, the windows programs menu naturally brings new launchers to attention, (because usually they get added to the main "programs" menu, which is probably just as annoying as the "new programs" thing).

If the classic menu structure is to remain, then at the very minimum:
 - It should be possible to drag and drop launchers between menus
 - It should be possible to delete and add launchers by right clicking on menus or launchers
 - It should be possible to view and change a launchers properties by right clicking on a launcher

A recently used launchers list and / or a search of launchers would also be a good idea, but I'm not  convinced that the classic gnome applications menu is the best starting point for an easily navigable menu system.

A good menu / launcher system is really important to both new users and power users alike - you need to be able to easily launch frequently used apps (which will change from user to use, for example I use Ardour a lot), and easily find apps that you don't use regularly, or might be unfamiliar with.

Sorry I don't have any really constructive suggestions at the moment :)

### Post by rockin_goliath on 2008-05-04
I think Hardy's menu system is the best out of all the distros and operating systems for two reasons.

First, out of all the operating systems, it takes the fewest clicks (2) to get to each option.

Second, all the functions are centralized. With OSX, you have to go to one side of the screen to change system preferences, and another side to access your application shortcuts, and then click on an icon within the dock to see a full applications list, ambiguously called "Finder" and not "Applications".

Third, the categories "Applications", "Places", and "System" are as descriptive and straight forward as you can get. The Applications menu is also divided nicely into its subcategories.

Fourth, all the items are shown as "cascading menus" so that you can see the path you have taken to get to that option. This is not the case with OpenSUSE, Windows Vista, or Mac OSX.

What needs to happen: "Preferences" and "Administration" need to be cleaned up a bit. They are the biggest menus in the entire applet, which makes it hard to find menu items. Possibly grouping them into subcategories the way the applications menu is.
