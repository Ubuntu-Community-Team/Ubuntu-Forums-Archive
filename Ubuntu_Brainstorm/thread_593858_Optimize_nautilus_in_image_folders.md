### Post by Can+~ on 2007-10-27
When opening an image filled folder, nautilus immediately tries to create the thumbnails for each image; of course, this can be deactivated or configured. But my point is:

Thumbnails should only be created for the visible section of the folder. For instance, if am seeing 10 elements of a 1000 picture folder, only those 10 should display a thumbnail. And the others should be loaded later as I scroll down on that folder.

Or, an alternative way to fix this, would be to auto deactivate the thumbnails when a folder is too fat. Or somehow, reducing the priority of the "thumbnailing process".

I like thumbnails, but recently I saw a post ([this one]("http://ubuntuforums.org/showthread.php?p=3646601")) where user "dbvanhorn" couldn't load a big image folder because of this issue.
