### Post by smurtguy on 2007-10-29
For me, I think the one area that linux in general has problems with is in the installation and initial configuration.

There are still too many "small" things that have to be setup manually, and I think a lot of the people that use linux and contribute to it, tend to forget to bring these things up as things that should be fixed.

We're used to having to do these things, and as long as it still works the way it did before, we know what to do.

The problem is that new users, you know, the ones we want to adopt linux as well, do not know how to deal with these things. And in a lot of cases these things may not be well documented either.

What would be useful to help identify these things is the option to have a nag process the keeps asking if the system is completely configured yet. If not it asks the user what they need to do still, and keeps track of it.
Once they are done, then this log can be used to figure out what the slew of small config steps are that still make linux a techie only OS.

Hmm its that are take a step back and look at what areas a user might have to configure still via an editor and make sure there are tools for all of these, that include all possible options for the configuration. (think xorg.conf here)
