### Post by enchantedsky on 2008-05-01
I think it's a great idea if Ubuntu's installation program actually tests a computer's hardware before installation, and make it clear to the user if something is not supported due to a lack of driver (e.g., graphics card, wireless card, sound card, etc).  The point is that users won't be surprised when they start up Ubuntu with 640x480 resolution, or some other quality of use issue on the computer.

The problem is that when users are surprised that X and Y don't work in Ubuntu after they partitioned their hard drives, they blame Ubuntu more, and it hurts their overall Ubuntu experience. If they know in advance something isn't going work, they will be more motivated to find drivers to fix their problem and won't blame Ubuntu overall as much. 

By the way, simply having a Live CD option to test out Ubuntu isn't good enough; that is more of a trial and error way of seeing if an OS works. It is better to have a dedicated program test your hardware to see how well Ubuntu runs under it. 

I got this idea from an old computer game called King's Quest. If you buy it on CD these days, the installation program makes it clear to you if you'll have sound problems because the games are old, and may not be compatible with newer computers. The same thing applies to Ubuntu, which has to run on many different computer configurations.
