> **Mr.mouse said:**

I agree, though that does not mean every aspect of the OS should be geared solely towards newcomers.  But, as I said, I agree with this point.

> **Mr.mouse said:**

I agree with this as well; however, I never said that Synaptic is crystal clear for me, and that it should be for others as well.  I do think, however, that the tooltip is about as clear as can be.

> **Mr.mouse said:**
> if Synaptic is advance add/remove tool call it that way

This is a better alternative to "Synaptic" than originally suggested in the first post.  

The issue here, I think, is with the term "Package Manager."  As I understand, naming conventions in Ubuntu follow a compromise between two options--giving the name of the program itself, or indicating the function (i.e. "Firefox" or "Web Browser").  So, Synaptic is the name of the program, and the function is "Package Manager."  I think it is ok as it stands, since the simple approach to adding and removing programs is placed upfront on the bottom of the applications menu.  Since Synaptic is the name of the program, it should probably be kept; however, if we want to call "Package Management" something else, that would be ok, but I don't know how people feel about that.  I happen to think that it manages packages, and that's the correct thing to call it.  

That's just my opinion.
