### Post by justinmiller87 on 2007-10-20
A discussion was going on in [this thread](http://ubuntuforums.org/showthread.php?t=582527) about being able to easily return to the prior version after a distro upgrade (e.g. Gutsy to Feisty) if so desired. When one gets security updates to the kernel, Grub contains both, just in case something goes wrong, so what is stopping Ubuntu from being able to do something similar for a complete distro upgrade? About the only limitation I could see would be disk space, and that would be on the user. For example, copy the entire current distro over somewhere, then upgrade that, and offer the choice to be able to boot into either, then be able to remove the old or new version if so desired. It sounds like something that might be nice to be able to do.
