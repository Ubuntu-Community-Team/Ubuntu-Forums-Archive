### Post by gnomeuser on 2008-11-04
No, the target should be perfection over a few cycles. What I absolutely hate the most about Fedora is that every 6 months I get a completely new half baked visual experience. Underneath all the code has been worked on for years to reach the point it and then the artwork comes along to make it all look inconsistent and ugly..

I really like the fact that Ubuntu builds upon an existing look, it helps with branding and the additional time means we can focus on ironing out bugs and perfecting the look.

Artwork should start with a mockup of where we want to be 2 years from now (next LTS basically), from bootsplash to backgrounds and icons. Then we figure out what technology we need to get there, what we can do the first time around and build on it till we have a satisfactory look and feel.

What we need is direction not constant change. If you want constant change there are 3rd party theming and we should also work on making that easier to apply to the whole system so that these adventurous souls might one up the existing design or start work on new ideas. Hopefully if they are good they will be integrated into the next theme mockups when time calls for it, if they are bad then we had that playground to test them.

### Post by gnomeuser on 2008-11-08
> **MadsRH said:**
> I agree with **gnomeuser **above.

Ubuntu is in the need for a look, but not every LST release.

Actually that is the opposite of what I said. I would like us to start every LTS cycle with a mock up of where we would like to be 2 years from now. This does not have to be a reinvention but to be seen as a way to push technology and identify areas where we need to do work. We also need a playground for people to try out new ideas. What I would be against would be the unsophisticated approach the Fedora artwork theme takes to this (As much as I love Fedora and love being a developer in the Fedora community, our artwork team are dangerously incapable of seeing the big picture).

It should not be a theme or an iconset. It should be a unified look for the entire project, complete with UI changes. We won't be able to do all of it but in each cycle we can add a few things, maybe one cycle get changes into GTK to accommodate the features we need, then the next cycle start adapting applications to using it as well as crafting the look.

My problem with peoples look on theming is that either it's to simplistic (aka. looking only at the gtk theme and forgetting backgrounds, icons, splash screens and UI) or they assume we can just make things look like a given mockup in one 6 month cycle (basically a problem of estimating the amount of work needed to accommodate the art becoming reality).

This will be work more in the search for standards, code changes and guidelines than in artwork. The artwork exists to give us a reference implementation of all the other hard work. In the end I hope we can fashion a way to theme the entire OS, from bootup till shutdown, for this set of standards we will offer a good reference theme.

Doing this work will require working with upstream, working with other distributions - if this work does not get buy in from both it is doomed so I would suggest standardizing via freedesktop.org.
