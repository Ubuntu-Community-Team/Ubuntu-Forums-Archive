### Post by altonbr on 2007-11-16
Editing files with gEdit and Nautilus over SSH or FTP ([http://ubuntuforums.org/showthread.php?p=3770166](http://ubuntuforums.org/showthread.php?p=3770166)) is a pretty great combination. No need to use an external program and since I'm using Nautilus, it's integrated directly into my desktop. No awkward GUIs like FileZilla3.

What's annoying though, is using Nautilus' "tree arrows" that allow you to see inside a directory with even going into it and then trying to add a new file to the folder you're looking at. You can't do it.

In Mac OS X, they have the same "tree arrows" feature, but you can right click a directory and say "Create new file in folder".

See my image attached. How would you propose adding a file or folder inside controllers? If you hit "ctrl + alt + n", it will create a new folder in the root directory because technically you're still in the root directory looking inside it's children. Nautilus doesn't know what you're looking at, nor the folder you have selected. That is why I propose a GUI feature to insert a new folder and/or text document inside a child folder either by (a) letting the user right-click and folder and selecting 'create new document in...' or (b) creating the new folder inside whichever directory is highlighted/selected with the mouse.
