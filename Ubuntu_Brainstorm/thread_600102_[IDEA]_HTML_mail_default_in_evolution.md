### Post by whitewizardcoder on 2007-11-01
Currently, evolution is defaulted to produce plain-text emails, which is fine if you are sending to a text-based (ie terminal) client.  However, most people in the desktop market these days do not use a terminal to do their emailing.  HTML mail composition should be turned on by default in evolution.  Most other email clients already have this as a default and evolution should be no exception.  Also, if a user needs to send to a text-based client, they can easily disable the html encoding.
