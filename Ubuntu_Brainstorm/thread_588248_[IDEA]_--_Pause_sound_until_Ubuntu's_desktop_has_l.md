### Post by altonbr on 2007-10-23
Now that Gusty is out and the desktop takes twice as long to load, I think it is vital to "tighten" and "polish" the appearance of Ubuntu. This goes along with the "slick-boot" blueprint ([https://blueprints.launchpad.net/ubuntu/+spec/slick-boot](https://blueprints.launchpad.net/ubuntu/+spec/slick-boot)) that cleaned up the boot process by using clean transitions between boot stages.

Thus I propose that even the sound the user hears when logging in should play ONLY when the desktop is ready to appear. Currently (for me) the sound plays when there is a blank brown screen and takes a further 5 seconds until even the panels start to load. This is without Compiz mind you, on a powerful computer.

Therefore the desktop should be loading with an Ubuntu logo showing the whole time until the it's ready to fade (literally) right into the desktop, then playing the sound.

May sound like an absolute waste of programming, but giving the user a richer experience on there computer never is.
