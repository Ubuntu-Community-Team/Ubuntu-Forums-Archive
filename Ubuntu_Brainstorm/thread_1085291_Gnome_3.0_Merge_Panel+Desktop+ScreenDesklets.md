### Post by BGFG on 2009-03-02
Hi All,

Was thinking about it last night and this occurred to me. I like screenlets and desklets but i don't use them because they are external to the Gnome and thus use memory outside of gnome processes and potentially can cause system instability.

I personally would enjoy an experience where the desktop becomes the Dock/panel as well as a file storage area.

Outlines:
* Merge Screenlets/desklets/Panel Applets into Shell

*Right Click menu offers "Add to desktop" listing of all our  useful tools

*Tools such as 'Gnome Menu' and 'System Monitor'can be placed directly on desktop and 'dragged' around according to user preferences including size, 'always on top' etc.

*Files can still be stored and viewed directly on the desktop or accessed via an applet/desklet, for neatness.

*setting like 'click on edge of screen' or a simple keyboard command give instant access to desktop (obviously :))

What do you think ? 
Also, from a programming framework standpoint how feasible would such a merge be ?
Clearly my idea would require tight compositing integration.

My Thoughts...:P

Edit; would be funny if this was already in the pipeline.....
