### Post by benanzo on 2008-04-30
I've suggested this for the last two releases and both times it seemed to have gotten a welcome response so I'm a little perplexed as to why this hasn't been done yet, considering it's so simple.

Suggestion for Gutsy:
[http://ubuntuforums.org/showthread.php?t=433384](http://ubuntuforums.org/showthread.php?t=433384)

Suggestion for Hardy:
[http://ubuntuforums.org/showthread.php?t=652158](http://ubuntuforums.org/showthread.php?t=652158)

Basically, put some default file templates in users' $HOME/Templates folder.  This allows for easy creation of common filetypes like OpenOffice Writer, Spreadsheet, Presentation etc.  A fully-featured OS should not say that it has "no templates installed" when a new user right-clicks for the first time and goes to create a new document.  On top of that, it's not even remotely obvious that in order to add new filetypes to the "Create Document" menu you must first save a blank file from whichever app you want to use and place it in the ~/Templates folder.  Who is supposed to know that?  Frankly, it looks slightly shoddy and incomplete to have a multitude of perfectly capable applications preinstalled but the most obvious way to create a new document with them simply says "no templates installed" --  Perhaps there is a good reason why this has not been done yet?  I have no idea.  Maybe someone knows why.

Also (this might be more of a Gnome problem) why is the Templates folder in the root of the user's $HOME?  Shouldn't it be in $HOME/.config/Templates or something?  I fear that new users will, while browsing their $HOME directory, accidentally delete the Templates folder or it's contents because they notice that they're just blank files.  They may not know the relationship between the ~/Templates folder and its contents and the nautilus right-click menu.

Ben
