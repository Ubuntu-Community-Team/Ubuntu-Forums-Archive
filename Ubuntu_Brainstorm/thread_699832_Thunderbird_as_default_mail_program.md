### Post by 23meg on 2008-02-18
You're not forced to use Evolution. You can simply remove it, and install an alternative. There will be no trace of its integration with the DE once you remove it.

I'd also like it if it were more practical to integrate other calendars (evolution-data-server has a Google Calendar backend, as far as I know, so it should integrate with the applet), but desktop environments work this way: there's one app for each task (for the most part), and each use case is handled with the app that deals with it. Alternatives can of course be introduced, but DE people will give first priority to the app that's actually part of the DE.

e-d-s uses iCalendar, so you should be able to get the applet to utilize any compatible source. Here's how to do it with Thunderbird / Sunbird:

[http://www.michaelwood.me.uk/wordpress/2006/09/04/gnome-clock-applet-using-ics-calendar/](http://www.michaelwood.me.uk/wordpress/2006/09/04/gnome-clock-applet-using-ics-calendar/)
