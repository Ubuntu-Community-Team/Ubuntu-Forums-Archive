> **Brinstar said:**
> wine does a good enough job, which is what matters for the time being. yes there may come a time when linux/ubuntu can do everything, unfortunately that time is not 2009 (or even 2010 for that matter), so lets start being realistic and admit the wine project is fulfilling a *present* major need of the community (and doing a pretty good job at it).

Doing a good job *for you*? I have quite a few programs Wine can't handle, does that mean I'm the exception? For example, SimCity 3000 won't work with Wine, nor will my SimGolf or Sim Theme Park games (I got them all as a pack when I was younger). I know there is Lincity, but it isn't nearly as good as SimCity, imo. Also, some of my school programs don't work with Wine.

In summary, my biggest gripes about putting Wine in is that it is still very buggy on many programs and encourages people to keep using the same programs they used on Windows rather than try the alternatives. Putting Wine in as default basically says that Ubuntu is supporting it, and it would be a lot of maintaining to keep updating Wine every few weeks for each release it has *for each release which is still supported*. Snapshots wouldn't work, since Wine gets updates frequently to fix bugs, and supporting it here on the forums could be a nightmare. Also, people can install it if they absolutely need it (no other alternative works for them), but it should be left out to encourage people to at least try the alternatives.
