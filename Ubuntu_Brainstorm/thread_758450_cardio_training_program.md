### Post by bikeboy on 2008-04-19
The ability to easily interpret the data (I presume you mean Heart Rate Monitor) will depend a lot on the data transmission method. My HRM uses a series of very fast sounds that the computer picks up through a microphone, I can't see that being easy to reverse engineer. Not sure if other HRM's use infrared, or some other system that might be easier to work with.

I know very little about programming etc., but would always be willing to provide some expertise on what the program should do, provided someone can get past the data collection problem.

If you have been able to get some sort of data file from your watch, I'd be willing to see if I can interpret it. In the meantime, running Windows in a virtual machine, or less sophisticated training programs in spreadsheets would be the go.
