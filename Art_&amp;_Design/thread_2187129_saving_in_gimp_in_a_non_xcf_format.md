### Post by Toz on 2013-11-11
[Here]("http://gui.gimp.org/index.php/Save_%2B_export_specification") is the save/export specification that gives the rationale for the way that things are. And [here]("http://www.gimpusers.com/forums/gimp-user/14339-hate-the-new-save-vs-export-behavior") is a good thread that discusses it - where you can read about the controversy surrounding the decision.

For times when I need to make a quick, simple change to an image, I use [Pinta]("https://apps.ubuntu.com/cat/applications/precise/pinta/").

### Post by SeijiSensei on 2013-11-11
If you open a non-XCF image in 2.8, you'll see that one of the export options in the File menu is to save back to the same file name.  I, too, found the change in 2.8 where "Save" = "XCF" and "Export" = "Everything else" a bit annoying at first, but now it's second nature to me.
