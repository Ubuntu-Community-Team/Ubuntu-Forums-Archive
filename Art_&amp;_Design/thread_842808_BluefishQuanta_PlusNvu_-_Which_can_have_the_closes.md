### Post by grimdeath on 2008-06-27
Let me start off by saying I am not a hardcore coder. In fact I have worked as a graphic artists for at least 5 years and got into web design as a hobby during that time. I now work as a web designer full time and some extra freelance on the side. I work with cooperation with a guy that is a hardcore coder and we really compliment each other with design/coding.

I by all means do not want this thread to become a "web designers dont use WYSIWYG" thread, I can totally agree that coders do not and should not need a program like Dreamweaver but personally I like it and work much faster with the graphical setup of it. I get to spend more time focusing on my design, layout and overall feel while letting Dreamweaver help pull it all together. If you do not have anything nice to say then don't say anything at all applies here ok guys!

Now my dilemma, I mainly use html/css with the occasional php/ajax/javascript/flash added in for specific functions. My reason for this thread is to find a program that works like Dreamweaver (not just has similar abilities) but actually has a similar workflow of Dreamweaver. So far I have installed and ran Quanta, Bluefish and NVU and I like them in that order (best to worst) with VERY QUICK test runs with each. However, I am having trouble figuring out which is the best for primarily CSS based design.

I am so use to having my CSS embedded in a window on the side of my app and being able to double-click and seeing tons of customizable options. I mainly lay the groundwork this way and then customize things from that point by hand in my pages and stylesheets. The CSS interfaces of the programs I have used seem REALLY wonky causing me to dig through long scrollboxes/drop-down menus to find the option I want. I am also noticing it is hard to add divs in the latter two apps, the div options seem almost hidden. My last problem is trying to figure out the quickest way create/attach new styles that are placed in the stylesheet rather then added to the specific tag on my html pages. I am completely spoiled to how dreamweaver lets me select a div for example, click "new style", customize a few options and click ok and the style drops onto my attached stylesheet instead of the specific tag I selected. Then I can go in and customize the code by hand on my stylesheet and pages. My quick tests has similar results with styles being created but dropped into either the specific tag I was targeting or the style being placed at the top of the page rather then my attached stylesheet.

Again I stress that I DO REALIZE that this is not how the "pros" do it, but it is my style. I would rather spend my time clicking a few buttons and creating the beginning style in a few seconds rather then hand coding it all. Perhaps this is a good push for me to try to hand code more from scratch but until then I would love to hear you guys opinions on the best way to setup/improve one of these three apps to work like this for me.

This is the first of my 4 big steps to trying to move to linux and free/open source apps full time. I still plan to try to get dreamweaver working in ubuntu with either the wine or virtualization but I would like to give native apps a chance first.

Thanks for any help!

### Post by itsols on 2009-05-12
Hello everyone. I know this thread is pretty old but every now and then I'm forced to re-evaluate which of the said tools are best.

As for me, I'm a designer AND coder. So I need both features - WYSIWYG and Design views.

Now, IMHO, I've found NO match to Dreamweaver on Ubuntu. But I honestly feel that Quanta Plus is THE BEST alternative, full stop. Here are my reasons.

1. In both NVU and Kompozer (probably since they're the same core), IT IS NOT POSSIBLE to include server codes or php scripts before the opening HTML tag. It is very common to do this in programming. I don't know if I'm missing something but I'm pretty sure these two packages don't support code before the html tag. Here's an example:

<?php 
//some code to run before rendering the page
?>
<html>
...
</html>

Now, when I type this code in and come back to it after looking at the design view, it's VANISHED! So I've decided NEVER to go back to them.

2. Quanta has a really cool feature in code view where I can SHRINK or HIDE blocks of code that lie within others. This is the BEST editing feature for coding so far and even Dreamweaver hasn't got this (at least my old version 8).

3. Much of the rest of the features are somewhat similar to Dreamweaver but like I said, it's no match.

Happy coding
Khalid
