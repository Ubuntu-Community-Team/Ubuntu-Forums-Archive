### Post by mpt on 2010-04-23
[cross-posted from [Canonical Design]("http://design.canonical.com/2010/04/battery/")]

As I explained on Wednesday, [we’re replacing Ubuntu’s notification area with a consistent set of menus]("http://design.canonical.com/2010/04/notification-area/"). As part of that transition, we need to adapt system components that have previously been using the notification area.

One of the simplest of these belongs to [Gnome Power Manager]("http://projects.gnome.org/gnome-power-manager/"), and displays the charge status for the batteries in devices connected to your computer — the computer itself, any wireless mouse, phone, and so on.

In Ubuntu 10.04, Gnome Power Manager has already been patched to use a custom status menu (“application indicator”). So, probably, not much will change when it shuffles over to the system area. But we’re still interested in feedback on the design and any improvements we can make.

[IMG]http://design.canonical.com/wp-content/uploads/2010/04/power-menu-no-brackets.jpg[/IMG]

Once you’ve checked out the full [battery status menu specification]("https://wiki.ubuntu.com/BatteryStatusMenu"), we’d be glad to have your comments either here or on [the Ayatana mailing list]("https://launchpad.net/~ayatana").
