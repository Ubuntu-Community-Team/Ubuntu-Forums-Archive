### Post by kayosiii on 2010-03-09
It would be harder to get the OSX version of ZBrush to work under Linux. than the windows version.

Although OSX has a unix core, The UI is very proprietary and specific to apple though you can create applications that will run on both using openstep. (you would need to recompile for this)...

If you are looking for ZBrush style applications for ubuntu.
I think that Mudbox is or is going to be ported to Linux.
3DCoat has a Linux version (this is a good one to look at regardless)
and Finally Blender's sculpting features are improving rapidly. You could grab either one of the recent blender 2.5 alphas or build from graphicall.org

If it has to be ZBrush then you are going to need a Windows or a Mac Box for the forseeable future

### Post by kayosiii on 2010-03-12
In the case of desktop Linux I would say that trying to do professional 3d graphics with ATI is a brave proposition. In short you are going to want to be running nVidia at least for the next year or two. (AMD/ATI are improving but not to the extent that I would use them in a Linux based production environment). 

Autodesk is a problem... but then again so are Apple and Microsoft. I died a little inside when Autodesk aquired mudbox.

Blender does have a different sculpting workflow to ZBrush this has a lot to do with the fact that it is also a general purpose 3d application. The best way to think of Blender is that of a shared inhouse developed tool. (if you have ever had to deal with inhouse software).

I don't see ZBrush producing a Linux version in the foreseeable future even the Mac Version lags noticably behind the windows version.

It sucks but those are your options.
