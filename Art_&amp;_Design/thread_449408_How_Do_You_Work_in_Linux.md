### Post by regomodo on 2007-05-30
tbh, i haven't really used Linux to do anything purposeful yet. I thought i would when it came to doing my 3rd year Elec Eng dissertation but i was in XP for the majority unfortunately.

I'm used to CAD, PCB design, elec schematic/vhdl simulation, photo retouching, scanning, printing, book designing, DVD backup, and some web designing. So far i've done none of my work in linux  or any to a satisfactory level. 

TBH i've never realised this and looking back i've spent most of my time using linux trying to get things to work. Eep, i'm worried.

**Photoediting** -  i use Lightroom/PS&Bridge CS3/PTAssembler.  I have dabbled Lightzone but it runs so damn slow (a java issue probably). I don't use GIMP as their is no 16bit and profile support (that i can find). I started off with Gimp and used it for a year. Then i discovered Photoshop.
**Scanning **- Vuescan. XSane doesn't support my scanners either at all or fully
**Printing**-linux doesn't have drivers for my Canon so i bought Turboprint. Not bad at all
**PCB design** - Eagle. Although in Linux it doesn't like the nvidia drivers and alt+f2 (eagle zoom to fit shortcut) doesn't work, for obvious reasons. PCB designer is terrible
**VHDL design/sim** - Xilinx/Modelsim. Neither are in Linux and i desperately need them
**Electronic sim **- Multisim. Haven't tried spice properly. Looks daunting
**CAD** - only solidworks, but blender (although not engineering suitable) looks good
**Book designing **- crudely in OO. Looking for something A LOT better
**DVD backup** - Gordian Knot. Not impressed by Acid Rip
**DVD creation** - Nero Vision, tried transcode in terminal mode. Not fun.
**Video reencoding** -Virtualdubmod - it rules

So looking back. I don't work in linux
