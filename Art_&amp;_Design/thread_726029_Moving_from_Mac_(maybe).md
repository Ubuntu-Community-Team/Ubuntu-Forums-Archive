### Post by kayosiii on 2008-03-21
>> Macromedia Freehand / Adobe Illustrator

Closest native app is Inkscape... Workflow feels a bit more CorelDraw than Freehand/Illustrator so might feel a little strange to begin with for a mac user - But honestly is a much faster interface than Illustrator (I use both). Also note that Inkscape on Linux works a lot better than Inkscape for Mac ditto for gimp.

>> Adobe Photoshop (GIMP is not there yet, IMHO)
The only thing really holding me back is lack of 16 bit support,  Though I would miss descent 4 colour separation support if I used that feature. Workflow is good but different enough from photoshop to cause much irritation from long term photoshop users - Therefore I don't recommend it unless you can find an experienced gimp artist to show you how to get the most out of the application. You have other options versions up to CS2 currently work under wine. Krita also shows promise...

>> Quark
closest thing on linux is Scribus - it really isn't too bad... Has good PDF output... There is a Mac Native version of this you can test.

 Dreamweaver
 Flash
>> You will not find anything close native especially flash - (you can get by with a good HTML coding program like quanta or bluefish - and something like Kompozer or NVu to do things that really nead wysiwyg. Wine or virtual machine will be the only way around these.

Basically if you are already invested in the tools listed then stepping down is going to be at least a little bit painful - especially if you are also financially invested in those tools. 

On the other hand if you are looking to get away from unlicenced software and are prepared to rough it a bit - with the exception of flash you can do most tasks that your list of applications are designed to do.

Regardless of whether you decide to go Linux or not I suggest checking out at least Inkscape and Scribus - both are very competent tools and a worthy addition to the artists arsonal.
