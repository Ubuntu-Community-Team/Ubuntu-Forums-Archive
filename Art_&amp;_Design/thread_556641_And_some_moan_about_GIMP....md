### Post by DirtDawg on 2007-09-22
I personally don't think this is as impressive as it looks. Most likely, the artist is using a Wacom tablet with a picture of the car placed inside of it and simply traces it. The rest is pixel art, and some artists may not even classify it as that, considering the artist's dependency on line tools. There's a whole pixel art movement happening right now that's pretty cool. You can see some extremely beautiful examples here:
[http://www.pixeljoint.com](http://www.pixeljoint.com)

I mean, the video (and others like it) is fine, but it is designed to look cooler than it actually is, IMO.

Most Pixel artists use MS Paint to create their work so the fact s/he's using that to work with is not unusual. In Linux, there is MT Paint (in the repos) that is designed specifically for pixel art, but when I dink around with the technique, I find that Gimp excels.

[Here]("http://derekyu.com/extras/pixel01.html") is a *fantastic* tutorial for anyone interested in giving this relatively new artform a try, and [here]("http://www.eglug.org/gimpixel") is a more-or-less good way to set GIMP up for working with pixels, though I prefer not to mess with palettes. 

I think it's a pretty fun way to pass some time and the results are neat-o.
