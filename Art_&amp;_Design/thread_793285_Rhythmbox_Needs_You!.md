### Post by tretle on 2008-05-13
Right, where to begin :D

During my first year at college studying networks and software systems one of the projects I had to do was create a simple html website. I chose Rhythmbox because I really love it and because I would like to see if I could do any better. Unfortunately, Web design was not my strong point. I created a simple website. nice looking but it couldn't really replace what is currently there. 
Programming is more my thing. 

So first year is over but I still think about the Rhythmbox website and see it as a huge flaw in Rhythmbox's armor. Banshee have a much nicer site but still not the best that can be done.

Whats the problem with the Rhythmbox site? its old. its not designed very well. The content is really, really bad. It has no sense of community. Its not a good place for developer interest etc.

So I thought how can we do something to change this. Hmmmmm... Ubuntu users are creative people. Lets start there.

So the object of this thread is to try and get input from ubuntu users and for the community to create a design.

I have spoken briefly to moch on #rhythmbox about this and he said that if the end product is up to scratch it can be used.

What needs to be done? 

*New graphics
*New design
*New content(description/words for each corresponding page)
*Community(Maybe a slick looking page with links to forums, mailing lists, irc and some cool stuff like lastfm widgets for the rhythmbox group on lastfm)

When it comes to graphics keep in mind this is a gnome project not just ubuntu so browns wouldn't be the best option. Tango is a good was to go in terms of color scheme. Bright pastel summer colors would be nice. 

So thats its really, if you like rhythmbox and want to help it grow a new site is needed. The easier it is for developers to learn about rhythmbox the better, the broader the community the better the feedback.

P.S. Currently there are no Rhythmbox forums, This is a problem. After the site redesign maybe another thread could be started with a poll on what forum software should be used.

### Post by tretle on 2008-05-14
I don't think we need black and grey. The current site doesn't even follow that color scheme. The only time you see it is when you go to the subversion repo. I think as much info on development should be kept in this new page design. I do think that there should be seperate sites for developers and users because both want different things. Users want screenshots, news, videos? synopsis of project. available plugins, where to get it.

Developers want documentation, documentation and documentation. The more documentation the better. Separating the documentation into different parts (python plugins, C plugins etc) seems like a good idea because people that are interested in achieving a certain goal can quickly find the information they need. 

They wont be looking for screen shots, videos, or possibly community forums. 
They will be looking for irc, mailing lists and information on where to grab the latest and stable code and builds.

Before freaking out about the absence of forums for the developer section think about the amount of developers that actually use forums. Not that many. Might be a better option to start user forums where data is gathered by the forum admin and sent in a newsletter style post at the end of each month.
