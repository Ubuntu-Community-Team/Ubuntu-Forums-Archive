### Post by kebes on 2007-02-23
> **encompass said:**
> Here is my current challenge.  For some reason the head that I am working with is always on top of the beard I am trying to place.

To control the order of overlaying objects, it's easiest to use the 'g' SVG element:
[http://www.w3.org/TR/SVG/struct.html#Groups](http://www.w3.org/TR/SVG/struct.html#Groups)

The 'g' element is a group, and you can think of it like a layer. You can put a bunch of items inside of a 'g', to group them together. Then you can set the 'height' of the various g elements based on the order of declaring them in the SVG document.

So the SVG doc could have a bunch of layers:
ears
head
hair
eyes
glasses

The order controls which one ends up being "on top"...



If you're using the fragments from the working file I posted, be aware that I grouped the elements into layers like "pieces", "working", "examples" and so forth... so it may be that's the problem.

Sounds like you're making good progress. I can't wait to see a beta version!
