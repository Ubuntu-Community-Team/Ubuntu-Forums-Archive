### Post by Rebes on 2007-10-31
I am looking for a program that has more capability than OOo Draw and Dia... I am drawing an assembly breakdown, where top level items have subcomponents, and those subcomponents may or may not have different subcomponents. I put each assembly on a separate page, so that Final Item A will have certain sub items, and Final Item B will have it's own sub items. In some cases, subitems for one product might be the same as sub items for another product.

Visually, this is all done with just boxes in (for example) OOo Draw, but I need some more advanced functionality...

I would like to "draw" subitems just once, and then have the ability to link them into top level items.  Then, if I edit the sub item, the changes would propagate to all of the drawings that are using that item.  I would also like the ability to link in text from a spreadsheet of some sort into those subitem boxes, so that if I change the text in the table/spreadsheet, it then in turn changes in all of the subitems, which in turn propagates to the product assembly diagrams.

Basically, I want to edit production information within the spreadsheet/table, and then see from the assembly diagram if a subcomponent for that assembly is in low stock, or if it's in production...

I know this sounds complicated, but it's really just a bunch of cross linking with a visual representation... Not much more than a graphical database.

Does anyone know of a program that might be able to do all this?

Thanks,
