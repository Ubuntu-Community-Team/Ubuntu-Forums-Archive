I observed some beneffits:

- It seems gnome renders better the svg than png. Maybe the quality of png visualization depends in the moment of png creation. But png it's not a losseless graphic format...

- An svg icon can be resized. A png image cannot be resized itself more than 128x128,, including when applying the file as icon for a folder. I'm not a ver big fan of oversizing icons (more than 128 ), but it's an issue.

- Smaller file format in svg.

The most important beneffit to use svg seems the better rendering quality. The smaller file size don't seems enough difference to use svg (the same file, as svg= 5KB, and png=12.7KB).

At this point. the SVG file format dosn't seem a great beneffit to use svg. PNG, as bitmap, can handle much more graphic details, some of them very hard to obtain in svg format: blurs, shadings, photoreal bitmap distortions (unless you insert bitmaps into svg files).
