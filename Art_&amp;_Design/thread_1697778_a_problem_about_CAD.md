### Post by Gemnoc on 2011-03-27
> **23dornot23d said:**
> it writes STL so why will it not read them .... ?
Did you know that it's the same thing with AutoCAD? You can create STL files, but not import them. An STL file's purpose is only for stereolithography (STL) and other rapid prototyping processes. It strips your solid model and converts the surface geometry to triangulated surfaces (polygons). In CAD software, an STL file is totally useless (unless as reference geometry), even if in some CAD packages you can import it back, you cannot do anything with it, not even measure it.

(BTW I use Solid Edge at work, but I'm familiar with how Inventor works. The workflow is very similar, as it is to SolidWorks. The first parametric CAD package I learned was Autodesk's Mechanical Desktop)
> Would love to get a parametric system on here ...... Blender is the closest I have seen to having the capability to do it ..... 
There is a link somewhere to where a guy did some impressive things in Blender ...... Will find it and post here ....[** LINK HERE**]("http://yorik.uncreated.net/tutorials/parametricobjects-blender.html")
That's Yorik website! He's one of the 3 main FreeCAD developers. He's an architect very proficient in Blender, and he's built the 2D Drafting workbench in FreeCAD. He's also responsible for the icon design and splash screen for the app. He started work on an Arch module, with the intent to bring fully parametric BIM into FreeCAD. These guys dream big! :)

Blender is an amazing app, but its focus is not on MCAD. The fact it uses polygon modeling means it's not really useable in that field.

> Can you produce useful file types from FreeCAD , would like to learn more a bout the guitar you have designed and how you went about it ......

Can it be output in a usable format for other 3D software 3ds or obj files ..... ?
FreeCAD imports and exports IGES and STEP, which are standard exchange format between CAD software. For polygon modeling apps, it can export to STL, and it imports STL too by the way (just tried it). Not bad, considering some commercial CAD apps don't!

The guitar is made of two components: the body and the neck. For the body, I drew a profile in the XY plane in the 2D Drafting workbench using the BSpline tool. I actually drew half of it then mirrored it to have a closed profile. Then I extruded it in the Part Workbench. For the neck, I drew another profile in the XZ plane with only straight lines. Extruded it, then added the fillets. Both objects were then fused together with an union boolean operation.

The neck is not realistic: it would be better with a loft feature, but it is not implemented yet in FreeCAD's GUI. It is available through Python scripting though.

The FreeCAD project's leader aims to offer an app that would have similar features to CATIA and other parametric CAD packages. It may take a few more years though. For now, there is no assembly environment, but it is planned. But if you're doing prismatic parts, all the tools necessary are there. And it is parametric.

### Post by Gemnoc on 2011-07-31
Hey guys,

Just FYI, I don't think I've mentioned it previously, but the FreeCAD project now has a **Daily Build PPA** with packages built from the development branch. You get daily updates of FreeCAD through the Update Manager.

A second PPA, **FreeCAD Devel** offers packages updated less often, between 2-4 weeks. I'm actually the one who takes care of that one, the Daily Build one is an automatic process.

Both PPAs support Ubuntu 10.04 LTS, 10.10 and 11.04 in 32 and 64-Bit. For info on how to install, check the [Download Page]("https://sourceforge.net/apps/mediawiki/free-cad/index.php?title=Download#Ubuntu_PPA_packages").

In the last few weeks most of the development's focus has been on the Sketcher, it's evolved fast even only in the past week. They're going to work on the Part design module next to add more tools (sweep, loft), because right now you can only extrude or make cutouts through the GUI.
