### Post by kid-pro-quo on 2008-06-29
If my memory serves me correctly the dark theme was never intended to be the default. The idea was that a dark theme would be used during the alpha (and perhaps beta) phase of development over a couple of release cycles so people could report bugs about how gtk renders dark themes in general so those bugs can be sorted out.

Basically they wanted to be able to identify applications which make incorrect assumptions about the user's theme and get them patched to behave properly. This was so if we decided to go with a darker theme in say Intrepid+2 a lot of the compatibility leg-work would have been done already.

I expect the theme for Intrepid final will be very different to whatever is in the alpha releases.

### Post by kid-pro-quo on 2008-06-30
Let's just remember that this is an alpha release, targeted at *developers*. The idea was to use a dark theme, any dark theme, so the *developers* can see which applications have problems rendering dark themes.

For the record, I am not a fan of the theme in the Interpid alpha, but it serves a purpose. Even if Ubuntu itself never decides to go with a dark theme the fixes that result from these trials will benefit any users who do decided that they want to use one.

It's not intended to look pretty, it's intended to help solve a problem. I expect we'll start seeing a better indication of what the final theme will look like in the late betas / release candidates.
