### Post by nshiell on 2010-11-01
A reoccurring design over-site that I often see in Open Source software is that the GUIs will hard-code one colour and leave the other to the operating system theme.

For example a button will hard-code black as the button text colour expecting a white-ish colour to be set by the theme for the background.

What if it isn't? This is such a silly issue and easy to fix if developers code their GUIs correctly.

This used to be a common thing in the early days of CSS on websites, as a rule of thumb either leave ALL the colours to the theme or set ALL of them in the application
