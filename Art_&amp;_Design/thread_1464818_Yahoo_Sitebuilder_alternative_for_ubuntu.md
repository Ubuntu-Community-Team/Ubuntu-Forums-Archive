### Post by candyiz4kidz on 2010-05-04
Neither reply really has anything to do with the original post.

Unfortunately, when this question is usually asked you get 20 people who want to tell you that "wysiwyg drag n drop editors" are garbage. However having actually worked for Yahoo! I came to love sitebuilder. It's ease of use makes creating a website so simple my grandmother could probably do it. Having left Yahoo! I kept the program and used it a few times, each time having to re-edit the pages it created with dreamweaver. 

You see, Yahoo! Sitebuilder is created so that you can only publish your website to a Yahoo! web hosting server. When the XHTML pages are created by the program it creates a sort of "watermark" that when published to your server with Yahoo! Sitebuilder it removes. However, when the pages are created and stored in ...

C:\Program Files\Yahoo SiteBuilder\sites\"websitename"\ 

There is this little box that says something along the lines of "This page is not meant to be viewed through a browser, please publish your website using Yahoo! Sitebuilder" or something like that.The way around this is to open these pages in dreamweaver or another html editor, select this stupid box with the text, hit your delete key, and save the page.

So there's that...

However, if your looking to use something open source or something made for the Linux platform, I still have not found anything that has the quality of Yahoo! Sitebuilder and I have been looking relentlessly. You may be able to run Sitebuilder on Ubuntu using WINE, i've never tried, but 75% of Sitebuilder is Java, and if i understand correctly java is almost 100% cross platform. So WINE may be worth a try.




I have however found some other types of editors that are more design/visually oriented sort of like Sitebuilder. 


-Website X5 Evolution (Windows)

Very very easy to use website creation tool, creates from templates, has a shopping cart script built in. Not exactly the most customizable websites, which is the reason I never really used it much.

-SandVox (Mac)

I just started using this. It's almost the same as Sitebuilder but with less customizable features. Has a built in image gallery script.

-SiteGrinder (Windows/Mac)

This has to be the best tool I've found yet. It's actually a plug-in for Photoshop. What it does is allow you to create a website from photoshop PSD files. By creating a layered image and running sitegrinder from the "file>automate" menu you can assign the layers as links and create true HTML pages from images, which is great because if you can setup a site with sitebuilder, you can create and image of your website and turn it into html.

-Nvu (Linux/Mac/Windows)

Nvu is a good WYSIWYG editor. If you don't prefer text editors, then Nvu is a great choice, especially considering  that it's free. But still no drag n drop capability.



I'm sorry I cannot be more helpful, but I've run into the same barrier and I feel your pain. Through hours and hours of searching for a replacement (preferably open source) I have run into the same thing over and over, people suggesting the same garbage apps that do not have the drag n drop design feature and people calling Sitebuilder garbage software because it uses absolute positioning (which can look funny in some browsers). The truth is for anyone who knows a bit of HTML but wants to save time/money by not typing out an entire website, Sitebuilder is pretty badass. I've built literally hundreds of websites using Sitebuilder/Dreamweaver, and Unless I find something comparable to Sitebuilder soon, I may have to start using Yahoo! Sitebuilder again. The only problem with that is, I no longer host with Yahoo!, and my current host has some garbage for free use. However I do prefer the Cpanel over Yahoo!'s Web Hosting Panel.

### Post by candyiz4kidz on 2010-05-04
I should probably also mention... In order to drag and drop an item, you have to right click and select "Float Element"

So you would ... 

Select "Insert" from the menu, Select "image", choose your image, it will add it to the project in the top left corner or first available space, right click on the image, select "float element", now you can move it anywhere without having to create tables or know the exact positions. voila!

other than that the learning curve for this software looks as though it IS Yahoo Sitebuilder with a funny hat and some sun glasses. So you should be able to adapt to it very quickly.
