### Post by orethrius on 2009-06-01
I'm taking a look at this now, too.  I figured out a bit of the wifi indicator; however, I've yet to delve into **sed** to finish it off.  
Here's what I have so far:
```
iwconfig 2> /dev/null | grep ESSID | awk -F: '{printf $2}' | sed "s/\"//g" 
```

Of course, those more versed in the system already know what this does; I'll provide the once-through for those who are stymied by it.

**iwconfig**, of course, calls the general wireless status tool.
**2> /dev/null** sends the error to the bitbucket; this prevents extraneous output from appearing on-screen.
The **pipe '|'**, as many of us know, passes that result to the next command for further processing.
**grep ESSID** returns only the line with the access point name on it (the Extended Service Set ID).
Then another **pipe '|'**.
**awk -F:** further seeks within the line returned for a colon ':', and
**'{printf $2}'** returns the text after the colon - in this case, the access point name.

Now, my only remaining challenge is to strip the double-quotes from the result and figure out a way to work it into the panel (though I suspect the latter is easier, and that the former is more elegant in **sed** than with choice applications of **grep** and/or **awk**).  Then I'm going to look into whether the "numbered workspaces" concept is feasible; and, if so, what kind of effort that might entail.  This is all very cool, and if I can help in any way it'd bring a little excitement to my otherwise dull life.

:cool:

UPDATE: **sed** seems pretty straightforward once you get used to it.  I'm sure there's a way to pare the code down even further, but for right now, this'll work.  Also, note that - unless I'm hugely mistaken here - the fact that **awk** is searching for whatever field(s) occur after the colon on the line returned should avoid the issue of spaced ESSID names altogether.  Perhaps somebody can verify, assuming I don't get a chance to try it myself?

Also, here's the quick primer on the **sed** code:

**sed** invokes the **s**tream **ed**itor.
**"** is used at the beginning of the command to slip a literal string to **sed**.  I'm not sure that it's strictly necessary here, but it helps when matching patterns with spaces in them (among other things).
**s/** is a signal to **switch** the text immediately following the **/** with the text following the second **/**.
**\"** is an escape character needed to correctly pass the double-quote **"** as a single literal character.
**/** with no text after it indicates that the matching text should be removed from the text stream.
**/g** indicates that the text, in this case **"**, should be removed globally (from the entire prior result).  Without this, the ESSID would appear with a trailing **"**.
