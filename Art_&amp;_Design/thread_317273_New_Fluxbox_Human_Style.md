### Post by thierry on 2010-02-19
Hi there,
 
Before the release of a new LTS version of Ubuntu I wanted to update my desktop in Fluxbox, so I created 3 new styles to use with the default gtk themes of Ubuntu (Human as of 9.04 and 9.10 + DarkRoom). IMHO the Ubuntu artwork team is doing a very good job (Ubuntu is to my knowledge the only operating system providing by default a desktop that has a professional looking design with warm colours), and that's what I really like to use so here it is ported for Fluxbox, which I use since Ubuntu 4.10 (always with modified styles).
 
The link on Boxlook : [http://box-look.org/content/show.php/Humanity?content=120172](http://box-look.org/content/show.php/Humanity?content=120172)
 
The menu : [[IMG]http://img535.imageshack.us/img535/276/menuv.th.jpg[/IMG]]("http://img535.imageshack.us/i/menuv.jpg/")
 
Tabbed windows : [[IMG]http://img69.imageshack.us/img69/3804/nautilush.th.jpg[/IMG]]("http://img69.imageshack.us/i/nautilush.jpg/")
 
This might need a few words of explanation :
 
- On the top of the screenshots is the Fluxbox toolbar
- On the left gkrellm with the plugins gkrellkam (a live picture of the Sun from here: [http://sohowww.nascom.nasa.gov/data/realtime/realtime-update.html](http://sohowww.nascom.nasa.gov/data/realtime/realtime-update.html). Feh displays a slideshow of the 4 EIT pictures when I click on it), gkrellmoon and gkrelltop. The theme is a modification of invisible-white (just a little bit more colours in the graphs)
- On the bottom right is Fbpager
- On the menu screenshot the New York Times with w3m in RoxTerm (a light tabbed terminal with utf-8 support). If you wonder why there's no terminal in the main menu, well I launch it with F12.
- You will notice that I group nautilus, eog, evince, gedit, totem and epiphany (which I use only for this purpose), this turns nautilus into a powerful browser better than konqueror and lightning fast. I can't use light alternatives like thunar because I need the thumbnails (for example large collections of files with names like s1050763.avi, s1050765.avi, s1050770.avi,...)
- You will find on Boxlook the names of the extra fonts I use.
 
The intention here was not to imitate Gnome, but rather to take advantage of the Fluxbox specifics : transparent menu and panel, shade/unshade buttons (I don't like double clicks)... while integrating elements of the metacity themes in order to get a consistent design.
 
About the license: the human-theme package from which these styles are derived was released under a Creative Commons license (you can read it on packages.ubuntu.com). Modifications have to be released under a similar license and contain a reference to the original authors, this is what I did (I thought a reference to Ubuntu was enough please correct me if I'm wrong). Actually I just took the license from a Flubox style by tenr ([http://www.tenr.de](http://www.tenr.de)) he also uses a Creative Commons.
 
Anyway now my desktop is ready for the Lynx !
 
Cheers
