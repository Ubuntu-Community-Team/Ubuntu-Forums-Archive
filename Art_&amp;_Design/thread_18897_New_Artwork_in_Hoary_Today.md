### Post by bvc on 2005-03-09
'Stock Icons' in 'Icon Themes' have been broken for years, which is why they all do not change when you change icon themes, and why we've had to tediously put iconrc's in gtk themes to get the proper stock icons for a uniform look. This happened when someone had the bright idea that nautilus/gtk should control the stock icons, the thought died, and the stock icons were not restored in Icon Themes. So the default is loaded.

This did a lot of damage because now that stock icons are being restored with the new gtk dir in icon themes....
[http://www.gnome-look.org/content/show.php?content=14260](http://www.gnome-look.org/content/show.php?content=14260)
...so many themes do not have these icons at all much less with the right names. I mean, why make stock icons if they do not work right?

So now there is no excuse for a distro to NOT release even a default Gnome Icon Theme with the blue highcolor navigation stock icons so that we don't get the green. I still to this day haven't found the location of these green stock icons despite using slocate and doing endless searching. Anyone know?

Of course for a distro, this is great because they can now make stock navigation icons and put them in their own default icon theme so that we get a nice uniform look by default
example:
Human gtk
Human Icons or Etiquette Icons
if they had stock would mean no more green!

Gnome devel have already decided the default army icons will remain for the 2.10 default.

I installed Ubuntu's Clearlooks engine pkg and was disappointed to find they didn't include a 'Human" version. I don't get that. Haven't made it yet? Takes all of 2 minutes so I can't imagine that is the case. Not going to? OK.
