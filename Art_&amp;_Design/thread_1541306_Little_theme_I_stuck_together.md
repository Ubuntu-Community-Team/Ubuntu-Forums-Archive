### Post by Mr. Picklesworth on 2010-07-29
I came across, and quickly fell in love with, the [Victory theme]("http://gnome-look.org/content/show.php/Victory+(Strikes+Again)?content=123936") for Gnome.

I took some time to modify it to my liking. All little modifications (I was learning as I went), which led to the following:

[SIZE="3"][http://ubuntu-art.org/content/show.php?content=128211](http://ubuntu-art.org/content/show.php?content=128211)

[http://people.ubuntu.com/~dylanmccall/themes/monoblin/](http://people.ubuntu.com/~dylanmccall/themes/monoblin/)[/SIZE]

[IMG]http://people.ubuntu.com/~dylanmccall/themes/monoblin/Screenshot-Desktop.png[/IMG]


There are a few details I am really happy with here :)


First of all, of course, the majority of this work is from the original Victory theme, which is a really beautiful theme on its own. I added my preferred look for expander widgets, and I messed with tooltips, basically to bring the theme a little closer to stock Ubuntu. (Conveniently enough, it blends with the Ambiance window theme if you are so inclined).

I spent a bunch of time trying to like Small Victory, but I didn't like its smallness. Eventually I realized that I just liked the Moblin icon theme, but only half of it. I did some pondering and came up with the Monoblin icon set, which is basically a subset of the Moblin icons (with one addition), providing *only* icons that appear in toolbars. Those are Action icons, with a few exceptions for everyone's favourite mail client.

I was inspired by recent movement in Gnome. At this point icons are being used quite sparingly, and the theory is they should only be used for objects; devices, applications and places.
However, right now that theory falls apart when we hit toolbars.
By adopting Moblin's symbolic style for action icons, we firmly establish that style which was halfway implemented in stock Gnome. This cleans up the screen, and, like Ubuntu's Mono icon set does for the panel, it gives colour a lot more meaning. Now elements can be selectively emphasized, removing unnecessary weight from chrome and drawing the user's attention towards content.



I also messed with the menus a little. Menu dividers, in particular.
Conventionally, those are drawn as unnecessary lines that conflict with their own intended meaning. Instead of separating two blocks of content, a menu divider adds a bunch of noise between them. The two elements are moved about about three pixels further apart (sometimes less), and the eye is drawn to the gap between the two elements instead of the two separate groups being created. In essence, they loudly tell the user to *imagine* that two things are detached instead of just doing it.
So, I removed those unnecessary lines in favour of *actual* separation via whitespace.

### Post by Mr. Picklesworth on 2010-08-06
> **schreber said:**
> Overall I like the theme but I have to ask, and it might just be me, but is there a particular reason why the empty trash can and the full trash can icon appear to be from two different themes?

Unfortunately, there is :(

GTK comes with its own set of stock icon names. Works well for portability, but it hurts the awesome potential of the Freedesktop icon naming specification. GTK's standard icon names have no split between device and action icons; context is of little importance to them. (I think it's poorly thought out, but then it does predate the desire to have context-specific icons, and I assume it predates Freedesktop's spec).

So, thanks to that, there *are* a few glitches like the one you described.


Evolution, being the wonderful UI marvel it is, [uses the wrong icon for the "delete message" action]("https://bugzilla.gnome.org/show_bug.cgi?id=625258") on its toolbar. (It should be using the action icon, but it doesn't). So, to maintain the integrity of that, I needed to keep that one place icon (called "user-trash") from the Moblin theme.

I've started using my trash and never emptying it lately, so I didn't notice it also affected the trash place. I had hoped it used a different icon name than Evolution, but now I know I'd hoped wrong.

What do you think? Should I keep catering to Evolution's misbehaviour, or remove that Place icon?
