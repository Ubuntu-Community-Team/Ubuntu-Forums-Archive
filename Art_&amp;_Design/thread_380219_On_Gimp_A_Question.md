### Post by jem7v on 2007-03-20
When I want to remove red-eye I usually just zoom in on the eyes and take the Brush tool...

Then I set the brush color to black (or something close to it) and change the brush mode to Overlay and reduce the opacity to 10-20%, and gradually darken the pupils until they're dark enough without actually being black.  (Leaving a little bit of color leaves them more natural looking than pure black, and the Overlay mode will leave highlights intact.)

At least, that's what I do.  I'm not a big fan of automated processes for stuff like that.

Hi,

The biggest downside of using the GIMP on Windows is the jagged and rough Fonts. On a native Gtk system sush as Ubuntu the Fonts are produced much smoother. When I have to draw using Windows the trick I use is to create the Fonts several times larger than I need on a transparent canvas and scale them down before applying them to my art.

GIMP on a Linux system is much better than it will be on a Windows system in my opinion.

On a side note in regards to the other question of using the Live CD, I concur that is not a good option if you plan to create large images since they will be stored in Ram memory while you are drawing. If you work as I do with many stages of the art open at the same time, you will quickly run out of system memory. I highly recommend an installation of Ubuntu. Your art fans will thank you.

### Post by dmn_clown on 2007-03-20
> GIMP on a Linux system is much better than it will be on a Windows system in my opinion.

But the necessary functionality for it to be considered as a serious image editor (Namely for high end images and digital photographs) is not there.  Color profiling has only recently been introduced into the unstable development branch, and many things that are reasonably simple in a proprietary image editor require a lot more work with the GIMP.

For high end images and digital photography in Linux you should be using Cinepaint (Film GIMP) which supports the high end image formats and color profiles.  Though the MS Windows(tm) build is currently stuck at 0.18 due to a nasty GTK+ MS Windows bug.
