### Post by aysiu on 2007-11-02
I can understand why they wouldn't bother porting CS3 to Linux.

Here are two reasons: [list][*]**No increase in income.** Pretty much those who find a need for CS3 use it on Windows or Mac, or they find some way to install it through Wine or Crossover Office. [*]Needs Photoshop but runs a pirated version in Mac, Windows, or Wine.[/list] There is no [list][*]Needs Photoshop but refuses to use it until Adobe makes a native Linux port, at which point, will pay for the native Linux port[/list] segment.

It's not about personal liking or not. There are ways to evaluate the efficiency of an interface (most notably, Hick's law, Fitt's law and GOMS model) and GIMP has several flaws that you can find using those elements.
To put it shortly: if an interface requires more clicks or keystrokes to do a certain action than another, then it's less efficient. If you have to continually click on the apps bar to switch from one window to the other (or do a key-stroke) then it takes more effort to use that application than one that doesn't have that "need".
Adobe spends a lot of money and resources to continually improve its applications' interfaces, it's not just luck or because designers happen to like the brand.
GIMP developers are just motivated by being "not like Photoshop". The result: an very unefficient interface, hard to work with, that gets in the middle of your workflow.
I'm sorry if you happen to like GIMP, I'm not saying that it's totally bad, but the interface **is** flawed. And if you're trying to just get the work done, you'll always prefer the most efficient interface, unless you're an amateur or a hobbist with lots of time.

Back in the day Adobe was burned quite badly when they did a port to Irix (SGI os) and so I don't think they are going to run into doing another unix port, If Mac wasn't where they started out I don't think there would be Mac port at this stage either.

Adobe already has one application ported to linux (acrobat reader). They are also one of the companies that are behind an initiative for a desktop LSB which would require that particular building block libraries be present (and binary compatible). The big problem is library binary compatibility - a qt app compiled for Ubuntu may not work on fedora due to incompatible changes made to the QT libraries. Binary only apps are second class citizens under Linux at present which is something of a double edged sword.

I Believe that Adobe plans to build Photoshops interface from scratch next version - this would give them operrtunity to do something a bit more cross platform friendly since they already have experience with QT (Adobe Photo Album)  and GTK (Acrobat Reader).

There are quite a few companies that use Linux exclusively or predominantly for media production. This includes most of the big end of hollywood - anybody who used to own SGI boxen. The reason that Photoshop works under wine is because Disney and ILM both payed Codeweavers quite a bit of cash to make it happen. Photoshop under wine has been used in some very serious production environments.

I believe that Adobe will port when....

1) They believe there is enough demand for a linux native version of the software.

2) They can say that Photoshop will work on any linux box that supports a particular standard.
