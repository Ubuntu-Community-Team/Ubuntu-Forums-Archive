### Post by jviscosi on 2008-03-12
> **Medieval_Creations said:**
> What are some of the differences between Flux & Openbox?  Is it really that much different than Flux?

I used Openbox for a while and eventually went back to Flux.  I found them very similar.  Some things I liked about Openbox were XML configuration files, an actual ALT-TAB window list, a slight (possibly imaginary) speed advantage, and a generally more polished feel.  But I run Fluxbox SVN/GIT so take those last two with a grain of salt.

The reason I eventually went back to Flux is that Flux has the built-in ability to remember your window attributes (size, position, workspace, layer, sticky, decorated, etc.), and it has the Slitlist to keep your dockapps in the order you want without having to introduce a delay after starting each one.  (This is important when you run like 25 dockapps.  I have dockapp impulse control issues ...)

But like Red Squirrel said, there is no "best" Window Manager.  There's only the one you like best at any given point in time.  ;-)

### Post by RedSquirrel on 2008-03-12
> **Medieval_Creations said:**
> What are some of the differences between Flux & Openbox?  Is it really that much different than Flux?

I agree with jvisosi. Openbox and Fluxbox are very similar. They both end in "box" after all. ;)

The main things I like about Openbox (compared to Fluxbox):

- standards compliance
- feels a tiny bit faster on my old hardware
- no longer based on old Blackbox code

I'm mainly sticking with Openbox now because I have tweaked it to my tastes so fully that switching to something else would be a pain. I had done that with Fluxbox too but it seems that my Openbox tweaking has surpassed even my Fluxbox tweaking!

One of my Openbox configs looks like evilwm. No window decorations, fancy one-pixel orange border for active windows... :twisted:

Openbox doesn't have as much eye candy. No icons in the menu. No transparency.

It doesn't have a toolbar/panel (I use fbpanel or no panel at all). 

It uses XML for the configuration files. I'm not a big fan of XML, but I can deal with it.

### Post by vvlist on 2008-03-14
I've tried Openbox, Fluxbox and IceWM. They're all very good in my opinion. Openbox is great when used with PyPanel, Fluxbox is awesome for keyboard shortcuts, but I think I might be settling with IceWM, because I like it's configuration files are better (no XML BS) and it's panel is better (hard to explain). Anyone looking at WMs should give all of them a try. It was very difficult to choose between them, and only after completely customizing each one to be the same setup was able to make my decision. In case anyone is wondering how much configuring I did when testing them out: I added a dark theme, changed all visible fonts to FreeSans, edited the menu to be similiar to Ubuntu's application menu with Tango icons (icons aren't possible with the openbox menu), edited the clock format, changed the bar to be on top of the screen, etc. IceWM was easier when trying to attain these things.
