### Post by Aelfric5578 on 2006-08-13
The problem with learning to use Gimp is that it's just different.  It's not any more or less difficult than Photoshop.  In the end it all comes down to the old "is user-friendliness a myth" debate.  If you're a graphic artist, you've probably been using Photoshop for awhile.  As a result, you find it's interface to be straight forward.  

The best thing to do is to just experiment.  It can take some time, but I'm sure you'll be able to learn.  I've used both programs--Gimp at home and PS at school.  When I came back from summer break last year, I realized that I had gotten so used to using Gimp that I had to quickly relearn Photoshop.  The same thing happened with Gimp after I went back to using Photoshop for awhile.

Also, I don't know if you ever found out the answer to your deleting problem so: the command to erase the selected area, either making it transparent or filling it with the background color, is called 'Clear.'   Since Ctrl+C is reserved for copying, the shortcut for 'clear' is Ctrl+K.  (On a side note I've noticed that delete doesn't always work as I would expect--i.e. it sometimes either does nothing or simply performs the same function as Backspace--on a several Linux programs that I've used.)

### Post by CarpKing on 2006-08-13
Aelfric, thanks for the Ctrl+K tip. Before I was using Ctrl+X, which has the same effect but steals your clipboard.  Alternatively, if you go to Select->Float, it will treat your selection as a separate layer until you anchor it.  You can then use the "Delete" button in the Layers Dialog (not the keyboard).  

I've read that for GIMP 2.4 they are rearranging the menus a bit and renaming some of the more cryptic stuff, which should make it more intuitive.  For a preview you might try installing the unstable 2.3 releases, though I haven't, so I don't know how much of the new stuff is implemented there.  One of the specific changes I've heard is that the delete key on the keyboard will work like one would expect it to.
