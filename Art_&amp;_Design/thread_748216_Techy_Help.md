### Post by duckgoesoink on 2008-04-07
> **pseudo-random said:**
> The single best site on the net for learning website building is
w3schools.com.


Agreed - [http://www.w3schools.com/html/default.asp](http://www.w3schools.com/html/default.asp) to learn HTML, then [http://www.w3schools.com/Css/default.asp](http://www.w3schools.com/Css/default.asp) to learn CSS (to add visual style to your pages). It won't take long, and you can practice while you learn.

To create HTML and CSS files, all you need is a plain text editor - don't worry about WYSIWYG editors - you'll save time in the long run by not having to unlearn techniques you learn using them. Plus you'll have a greater understanding of what goes on under the hood, and your pages will load faster (no excess code).

To design the graphics and mockups, you can use Inkscape and/or GIMP. You can easily find good tutorials for these programs on google. If you like video tutorials [http://screencasters.heathenx.org/](http://screencasters.heathenx.org/) is great for learning Inkscape, and [http://meetthegimp.org/](http://meetthegimp.org/) for GIMP.

Look at the source and CSS of other sites to learn more about how sites are put together. The web developer toolbar for firefox ([https://addons.mozilla.org/en-US/firefox/addon/60](https://addons.mozilla.org/en-US/firefox/addon/60)) is extremely useful.

### Post by duckgoesoink on 2008-04-11
> **dobroschi said:**
> Well the thing is .. in my poinion WYSIWYG is better because its easyer to learn but yes in the long run doing the text way is better b/c like you said you get a ideea whats happening 'under the hood' ...

Any other opinions? Is the text editing better for starting off with or with Dreamweaver like sofware?

To use Dreamweaver *properly* takes a while to learn, and lots of practice. And you will still need to learn coding anyway, to clean up all the coding mess these kinds of programs leave behind. (I learnt Dreamweaver at art school, but I wish I didn't bother - I could have been improving my hand coding skills.)

Honestly, I think if you want to use Linux and open-source software to do medium to high level websites, you'll need to use a text editor - Gedit is sufficient, it even has syntax highlighting (my favourite text editor at the moment is Geany). There are some WYSIWYG programs, but not many people seem to like them. 

You can design the "look" of your site in Inkscape, and slice up the images for use in the CSS.

If you can, gather the content and write the XHTML first (that's the easy part, no visual stuff to think about - just logical structure). It can help with ideas about how you'll arrange the content afterwards, knowing already what has to be laid out.
