### Post by Sand &amp; Mercury on 2009-07-27
I find a really good tactic to making a good design is to have a 'working mockup', code just a bit of a page to get the layout done, just using CSS divs with 1px borders so you have a skeleton of the page and get all the metrics right before you open anything. Once you're happy with the layout and it validates, screenshot it and paste it into GIMP, then set rulers as you require. That way you have an accurate framework immediately, that you know is going to work.

After that, you can design all the graphics as you desire and slice them as necessary.

to the OP:
I had the same question you did a few months ago and, i can say pretty confidently that there is much better software available for windows as far as graphic design goes, but if you reeeeeally want to spend some time learning inkscape and gimp, it is possible to turn out some beautiful designs.  i broke down and set up creative suite 4 on a virtual machine because all of the designers at my company use photoshop.  the biggest drawback is the fact that there is no concept of a "slice" within GIMP or inkscape, you have to copy paste sections of the image into separate files and then save them.

god speed, it's not an easy road you're going down :)

Actually slicing in Inkscape is quite easy.. Just create a new layer for your slices and then use transparent rectangles to mark your "slices". Then use "Export Bitmap"->"Selection" to save the sliced image. (you can also use semi-transparent slice rectangles to make them easier to see, and just hide the slice layer when exporting the bitmaps)

Inkscape even stores the different filenames you use to export each selection so after you've created the slices it's really easy to export them again if you need to change something in your design.

Inkscape works really well for web design, I'd choose it over Illustrator any day. (well, actually, I choose it over Illustrator *every day*..) :)
