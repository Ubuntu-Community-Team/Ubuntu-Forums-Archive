### Post by dylbud on 2011-01-27
Most people forgot the "why" part of your question.

Inkscape and Gimp are the most useful and would make a solid foundation for any design suite. Gimp can do all of the standard operations that photoshop can do. It has most of the same filters, but that's not really what's important. Mainly, the things you need are selection tools, layers, blending modes, paths, channels, clone tool, etc. All these are available in GIMP. 

Aside from photo manipulation, there's also digital painting. Gimp is great for this, and under the brush tool you can set various settings for optimum use of a tablet. There are also lots of cool brushes to download, as well as add-ons for Gimp in general. Although, for digital painting, neither GIMP nor Photoshop are as awesome as Corel Painter. I don't know of any current open-source project that simulates traditional media like Corel Painter. Would love to find one though.

Inkscape is my absolute favorite, and I prefer it to Illustrator for several reasons. A few examples of why: you can set different opacity levels for fill and stroke on a single vector -- you can't in Illustrator.

You can create a shape such as a star for example, then do something else, come back to it, and still be able to change properties of the shape (such as change the number of points on the star, or make it a polygon, etc) -- in contrast, with Illustrator, once you create a shape, that's it; it can only be manipulated by its anchor points after that, you can't manipulate its "polygonal" properties with the polygon tool.

The node tool in Inkscape is easier to use than its counterpart, the Direct Selection tool in Illustrator. It's easier to see and select nodes, and to changes a node's type.

One thing that Illustrator has for serious graphic designers, which inkscape lacks, is a selection of Pantone swatches. Pantone is proprietary, so I don't see Inkscape ever adding that feature. But you really only need Pantone colors if you are doing traditional offset printing. Traditional printers have Pantone swatch books you can pick the color from, so you don't necessarily need that feature in your application.

Speaking of offset printing, many people have said the main thing GIMP lacks is the ability to convert a photo to CMYK. But again, you really only need to do that if you are using offset printing. Even though color laser and inkjet printers use CMYK inks, any digital printer can print RGB images with no problem. Still, if you have a job that will be offset printed, the Printer company will likely have software to convert it for you, it just might not look the way you imagined it. But discrepancy between print and on-screen appearance is always an issue no matter what.

If you are creating a single-page document, Inkscape is also very good for page-layout. I haven't really given Scribus much attention, but the one time I tried to use it for a real job, I gave up and used Inkscape instead. The latest version of Inkscape (0.48) has made some pretty good improvements to the text tool and its treatment of text in general.

In general, I don't really like Adobe CS products, preferring Corel to them and GIMP/Inkscape to Corel. The exception is InDesign for multipage document layout. Maybe I should give Scribus another serious try. I'd love to here everyone else's reasons for why they like it.

Blender was not mentioned by anyone else here, and it should be mentioned because it is an extremely versatile program for 3-d design. For graphic designers, Blender is great for creating mock-ups of package designs. I am not a Blender wiz, and don't do much 3-d, but I've fooled around with all the 3-d programs available in the repository, and Blender seems to be the best and most reliable. The main problem is the learning curve -- it's not as easy to just pick up and start using, like Google Sketchup is, for example. (By the way, you can run Google Sketchup on Linux using Wine).

So in the end, GIMP/Inkscape is my suite, with Blender if I ever need 3-d. There are sometimes a few problems with creating a good, reliable PDF file from these programs -- that is, one that displays correctly on windows and mac machines (which it needs to do if you are collaborating or sending files to commercial printers). I'd love to hear suggestions on any good software in the repository for creating PDFs.

I'd also love to hear more about what everyone else likes about the programs they mentioned, and what they use them for.

(PS -- I don't really have a WYSIWYG editor for HTML that I like, not even Dreamweaver. I prefer to code by hand at this point, since I learn more that way)

Krita has a good selection of advanced brush engines including Sumi-e. Aforementioned MyPaint has a generic and by no means "media simulating", but very powerful brush engine.

> **dylbud said:**
> One thing that Illustrator has for serious graphic designers, which inkscape lacks, is a selection of Pantone swatches. Pantone is proprietary, so I don't see Inkscape ever adding that feature.

Inkscape doesn't need it really. What it needs is color-managed imnplementation of Cairo with support for spot colors and CMYK definitions **and** adjusted EPS/PS/PDF exporters that will use it. Shipping Pantone swatches is absolutely not neccesary. There are at least two legal ways to fetch them independently: 1) downloading archive from Pantone and using palettes from it, 2) fetching whole palettes or selected colors via X-Ref. Both ways are [available to Scribus users]("http://libregraphicsworld.org/news.php?readmore=684") already. There is no reason Inkscape shouldn't go the same way.
