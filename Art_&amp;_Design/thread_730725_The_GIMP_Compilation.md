I&#8217;ve seen a lot of users whining and crying that GIMP doesn&#8217;t do X and Y thing. I do admit that a lot of things can be improved but GIMP Developers can&#8217;t read every distro forum to see users problem and &#8216;wish list&#8217; for upcoming versions. 
 I will explain how to do proposals(either for features or improvement), because it&#8217;s something that most of the general users skip. Please be reminded once again that GIMP Developers are not mind readers (as far as I know, although I think [Øyvind Kolås]("http://pippin.gimp.org/") might be one). If you are a serious user who want to take time writing a proposal for an addition then please follow this:[LIST=1]
[*] Sign up in [GIMP mailing list]("http://www.gimp.org/mail_lists.html") (I used the developers one to make mine)
[*] Use an alternative interface like nabble to make your post along with your proposal.[/LIST]When proposing a new feature or improvement, please be detailed as possible. For example, you can&#8217;t just say &#8220;hey, add Transform Tool because I want it&#8221;, you&#8217;d need to think globally. How many users will use this feature? GIMP is a software that thousands of computers have installed in their linux and some windows computers. Note that it&#8217;s not limited to features but you can also request filters and other things (as long as the it&#8217;s on GIMP&#8217;s range). 
 **Things that you should know**[LIST]
[*] GIMP is not a clone of photoshop and it will never be, it could be better than photoshop if you decide to help. (Either by creating work with the GIMP, developing the GIMP, help debugging, reporting bugs, motivating users and developers)
[*] Before asking about layer &#8216;blending options&#8217; you should know that [GEGL]("http://www.gegl.org/") is on its way, it&#8217;s GIMP future image processing framework that will be implemented in v2.6. GEGL is also a created by the GIMP team. Apparently something happened that caused GEGL to be dropped (in mid 2003), they&#8217;ve recently picked it up.
[*] We all know Photoshop have a lot of helpful features, but please do NOT flood GIMP developers with so many photoshop features!
[*] If you are a developer with knowledge in C and linux system and want to help develop GIMP in Linux (or Windows Builds) I guess you are more than welcome.
[*]&#8217;nuff said there is a [bug tracker]("http://bugzilla.gnome.org/browse.cgi?product=GIMP")[/LIST]Why you should help? It&#8217;s not a matter of helping or not, it&#8217;s a matter of supporting open sources software like the GIMP([OpenOffice]("http://www.openoffice.org/"), and many more). Right now you guys have a free image manipulation program that might be capable of reaching PhotoShop or any other image manipulation software. Look at how successful [Blender3D]("http://www.blender.org/") have been. 
 And now, without further ado: The GIMP Compilation!
 The moment you thought that GIMP lacked features like color overlay, inner glow and so on it&#8217;s the moment you said &#8220;Okay, uninstall time!&#8221; It&#8217;s true that GIMP needs to be more of a &#8220;out of a box&#8221; term but here is a big one for you:
 **[GIMP Plugins]("http://sourceforge.net/projects/gimpfx-foundry/")**

 Over 50+ scripts that are commonly used in graphic programs, that project is called [GIMP FX Foundry]("http://sourceforge.net/projects/gimpfx-foundry/"). Inner Glow, Inner Shadow, Color Overlay, Photo Effects and many more are included in this project. You could think that GIMP is a little like firefox, except not many people is aware that GIMP purely runs on script-fus and plug-ins.
