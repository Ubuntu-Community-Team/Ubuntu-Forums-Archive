### Post by mendozaro on 2008-03-06
Moving from windows to linux is really easy.

The only real problem is that you must find the proper reasons in your head to do this... THIS IS THE HARDEST PART!

Lets look at my moving experience...
If you read this entire thread... you'll see all the steps. But here is a short resume of the entire process:

1. I am using windows - Fireworks, Dreamweaver, Flash, Sony Vegas...
2. I am trying to move to ubuntu but fail to find a reason to move to all this new free software. Fail to find a reason to start learning all this. Fail to find any resemblance between the programs used on windows and the ones available on ubuntu (linux). 
3. I am stubborn... i try harder to find reasons to do this.
4. I find them:
- no cost, all the software i need can be found in ubunutu for free (except flash)
- move to a non wysiwyg html editor because of the w3c standards. You cant make a xhtml 1.0 strict web site using Dreamweaver.
- start learning GIMP and other image editors for cost savings.

So... now: i am using ONLY ubuntu, i am make only xhtml 1.0 strict validated web sites, i remembered almost all the html language (still not very sure on css).

Now, this is my case... i am really stubborn and determined. Maybe some day you'll get to my words to.

### Post by akohler on 2008-09-05
I have been thinking this for awhile too. 

I made my first web page in 1994 and I've been using Linux since 1999, so I don't really count as a designer moving to Linux as I've done the majority of my design work on Linux. I've used Adobe and Macromedia tools now and again, but never really got hooked. I use Flash through [CrossOver Office]("http://www.codeweavers.com/"), and otherwise I primarily use [The GIMP]("http://www.gimp.org/"), [Inkscape]("http://www.inkscape.org/"), [KolourPaint]("http://kolourpaint.sourceforge.net/"), [Bluefish]("http://bluefish.openoffice.nl/") for my Web design, [Blender]("http://www.blender.org/") for 3D, and [Scribus]("http://www.scribus.net/") for DTP/PDF authoring.

For Web design (not print, which is a different animal), free software has most of the core features that the big boys are playing with. ["With Inkscape an artist can create most of the same illustrations that can be made with Adobe Illustrator."]("http://wiki.inkscape.org/wiki/index.php/Inkscape_for_Adobe_Illustrator_users") Also, what The GIMP is most lacking compared to Photoshop (besides pre-press features irrelavant to Web Design) is layer grouping and integration (import/export and interface wise) with other free software tools.

What all of the free software web design apps are lacking is integration and convenience (which, in reality, is what most designers are looking for). Combinging existing tools (gimp, inkscape, bluefish, etc.) into a unified, compatible suite ala abobe and macromedia is certainly a daunting undertaking, but is something that could easily take the creative corporate/freelance world by storm. thus, it is a goal worth pursuing.
