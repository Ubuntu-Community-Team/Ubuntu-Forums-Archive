### Post by mpfilbin on 2007-05-29
I am kind of surprised that no one mentioned Komodo Edit from ActiveState which is a free application that can be downloaded from: [http://www.activestate.com/products/komodo_edit/]("http://www.activestate.com/products/komodo_edit/")

This is an editor more so than an IDE and as far as I can tell there isn't any debugging but on the plus side it does have code insight and autocomplete for mostly all PHP 4.1 to 5.0 (I think ?) php functions.  There are project management tools, syntax highlighting, and automated tab control.  There is a commercial IDE offered by ActiveState, but it is fairly pricey.  I've tried to wean myself from WYSIWYG apps, those things became more of a crutch for me.  As for HTML, I personally like BlueFish, that's a nice application.  Quanta is also very robust, but if you are just working with HTML files, BlueFish is probably the way to go (wizards to help with automation).  Oh, I almost forgot to add, you can run terminal commands from within Komodo Edit and save them as "Macros" for use later.  This is a nice way to manage Apache and MySQL on the fly (versus opening a terminal window).

As for FTP clients, I am font of gFTP, it may be small, but it suits me well. As for Graphics manipulation I agree with all stated above GIMP is very robust, but perhaps you would want to check out Xara Xtreme with the ImageMagick plug in, does have some nice rendering capabilities.  I export the bulk of my graphic files in .png or .svg extensions and fairly confident that Xara handles both.

I would try to caution you away from PHPEclipse, still seems buggy (at least for me) on Fiesty, but before I converted to Linuxism, I ran it on a windows machine and found it quite nice.  I agree with jbaxley regarding Aptana however; for an Eclipse derivative (plugin) they did quite a nice job (solid for AJAX).  Anyway, that is my two cents worth, hope it helps!

- Mike, FNG :p

### Post by airtonix on 2007-06-14
i stopped relying on the embedded html renderer in wysiwyg progs cause they dont work.

there are four major html engines : 

+ Trident (IE)
+ Gecko (firefox, epiphany)
+ HTMLkit (konqoueror, safari)
+ Opera.

now the best way to see how your html/css/javascript/xml/xslt code will end up rendering and behaving in each of these four environments.......is too actually load it up in each of the four environments.

I am sorry for this but: measure 4 times cut once. stop being lazy and become a pro.

use gedit for the text editing : it has quite cool features now : 

+ tab to indent highlighted blocks.
+ customized tabSmart templates. 
   (tabSmart is my pseudo word for being able to 'tab' through placemarkers in the template and enter data)
+ file-browser (ssh,nfs,smb,symlink savvy)
+ session saver, re-opening your gedit will open last documents.
+ line numbering
+ code colouring
+ tabbed documents
+ code API quick look up...by typing the start of a correct sequnce of syntax you can complete the possible end sequnce by pressing alt+space. same as dreamweaver/flash api completion tool that pops up at cursor in code editor.

Use Inkscape for graphics (visit adobe's svg demo site for a reason why to get with the program now before you become stone age)

svg is xml. 
xml is eXstensible Markup Language. it can be used as flat form data storage format, microcontent data-transfer format and many other data-formats requiring platform promiscurity.
xml can be trasnformed by another xml format called xslt. 
xslt or xsl is eXstenible Styling Language for Transforms. or an exstensible transform language for styling.
they are both xml but they use different namespaces, 
a namespace is a reference to a collection of definitions that make up the schema of elements in the extensible markup language being defined.

the namespace of html for xhtml format is html...lol

point here is that inkscape can be used to draw your xslt templates as xml, which you then use in a php driven xslt environment to process other xml fragements into ready to use svg fragments.

data-xml -> inkscape-xslt -> w3c-svg
