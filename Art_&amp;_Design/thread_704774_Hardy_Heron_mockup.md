### Post by stephantom on 2008-03-05
Your work is heading in a good direction.
Here are some pointers to make it great:

- Use Inkscape (or any other vector edition application of your choice). The Heron on the front side is blurry because of aggressive resizing. Vector images can be resized endlessly without loss of quality. There are no objects in your composition that would require a pixel application. If you need any help converting your document to vector, feel free to drop me a line, I'd be glad to help.

- Unify the "Provides" section. The Compiz icon has a reflection, OpenOffice.org has a somewhat mangled shadow, Firefox/GNOME have nothing. All of these icons should have vector version readily available (check Wikipedia). IMHO no shadow/reflection at all would look best because the space is really limited. Without any gizmo underneath the actual icon you can align them properly and use the space best.

- Expand the "What's new" section. That is not in "add more stuff" but use less stuff and EXPLAIN what these applications/features enable the users to do. Instead of Wubi write something like "Installation from within Windows using **Wubi**", instead of Xorg (the name does not really matter to the average user) tell them what it actually does for the user (i.e. support more graphics cards, better speed - whatever!).
Look [here](http://blog.gobanquet.com/index.php/why-ubuntu-804-needs-better-marketing/) for more examples (this post refers to the release notes, but your project is, although limited, basically the same).

- If you need more space, reduce the font size of the legal stuff. That's the least interesting part, but I understand that it needs to be there.

- I like the idea of the dirve/memory icon, but I'm afraid that many people won't be familiar with that. You'll need to add text somewhere. Maybe a table/list based solution would work here.

- I absolutely LOVE the foot prints. Just got to say that out loud.


Thanks for reading up until here :-)
