### Post by tretle on 2009-02-21
Ok, just to begin with I don't care that it has been postponed nor am I really that surprised. Theres no point in changing it for the sake of changing it regardless of how much it is needed.
Mark has hired more people as part of a new direction towards achieving user experience goals, one of which being a new theme.

I'm getting really sick of seeing the same arguments crop up every release cycle. People that want a new theme now or else and then others who argue that there is no need at all for a new theme.

Yes there is a need, one need only compare an icon set like gnome colors to the current human icon set to see that the former really doesnt compare as gnome colors offers a better visual representation of files depending on the file extension.

A new theme is badly needed, Mark listened and is doing something about it.

And for all those "elite" guys that don't think it matters pft to u, yes I said pft because this will be my last comment on a thread like this targeted at guys like you.

How many people have you converted to using Ubuntu, now how many of those didn't react well to brown when you were endorsing it. 

Now imagine people who don't have friends that use Ubuntu, that don't know how great the Debian package system is. That don't know that the Ubuntu community is the largest of its kind. Imagine the people that investigate, see the horrible theme and stop right there simply because they don't have anyone to tell them the theme is not relevant in terms of the entire experience. 

This is why people want to change the default theme, this is why mark wants to change it. To try and bring in a larger audience.
Its not even like brown is a horrible color choice either, you can get some really nice brown themes like oransun etc.

So this is an issue and its one that is being worked on, no matter how many duplicate threads people start with different releases or how many rehashed arguments unfold on the discussion it make it come faster or change Canonical's minds on the situation. 

*And to give people an idea of why it may be taking so long here are some possibilities.*
[B]
1. Mark has only hired people for the new team since prague
2. Stability of upstream[/B]

Right upstream projects I am referring to are:

**Plymouth** - *for new sexy startup splash*

**GDM** - *so macslow can start integrating his clutter work(but first he needs to wright a new greeter)*

**GTK** - *the gtkhackfest has only just come to an end, some things we can expect with regards to theming are complete migration to cairo, gtk-animations(for stuff like buttons and transition effects for sidebars), and also css theming.*

**Panel** - [I]The panel will be replaced with gnome3.0, Canonical are unlikely to stray away from using the current gnome panel until then. The panel is likely at this point in time to be replaced with gnome-shell which in my opinion is the wrong choice.


[/I]

There are lot of other things for people to get excited about and try and endorse on these forums, packagekit being one of them. Packagekit would reduce the amount of work on developers so that they don't need to right support for multiple package managers for certain features. This way the developers could use this time to work on other more important features or refinements.

Some other things to look out for would be telepathy, conduit and gnome online desktop.
