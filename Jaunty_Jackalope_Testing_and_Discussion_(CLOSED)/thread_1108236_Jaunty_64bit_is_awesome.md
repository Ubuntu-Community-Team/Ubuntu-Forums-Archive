### Post by Reiger on 2009-04-18
> **toupeiro said:**
> I agree.  Not to turn this thread into a MS bashing talk,but I very distinctly remember in the windows world of support, even Microsoft would tell you that the 4gb was an "architecture" limitation of 32-bit hardware and not the fault of Windows.

A single 32bit CPU + Bus will only ever address up to 4GB of total memory. That does not prevent you from installing more memory, or running the system with more memory: it only prevents you from using the excess memory. 'Tis very simple: every single byte is addressed individually so you are limited to 2^32 addresses => 4GB.

If you want to get around that (and still use only 32bit hardware), in theory you could have multiple CPU sockets on your mainboard and have each CPU be responisble for its own 32bit address space (and hence its own RAM): meaning that with n such sockets you can address up to n * 2^32 for the total address space is not continuous but separated in distinct blocks per socket. Of course this adds some funny concerns with how you are going to transfer data from one socket + RAM to another, so you'll end up with a bit less still...
