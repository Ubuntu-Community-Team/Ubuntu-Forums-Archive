### Post by pferraro on 2009-04-08
From `man radeon`, regarding DynamicClocks option:
"Some users report reduced 3D performance with this enabled."

You may also want to experiment with the following options:
Option "AccelDFS" "on"
Option "AGPMode" "8"
Option "AGPFastWrite" "on"

### Post by pferraro on 2009-04-08
> **alternativ said:**
> It's not the 3d performance that is bugging, 3d performance works fine however (dynamic clocks disabled) still leaves me with troubles. Next to that my card is a pci express card, I don't think the agp modes will do much good if I am not mistaken.

True.  A few other tips:
* Try Option "FBTexPercent" "0"
"Specifying 0 results in all  offscreen video RAM  being reserved for EXA and only GART memory being available for OpenGL textures.  This  may  improve  EXA  performance,  but beware  that it may cause problems with OpenGL drivers from Mesa versions older than 6.4."
* Verify that your BusType is detected correctly.  Manually set to "PCIE" if not.
* You might want to try MigrationHeuristic "greedy" instead of "smart".
* See if AccelMethod "XAA" gives you better results.
* Try enabling "BackingStore"
