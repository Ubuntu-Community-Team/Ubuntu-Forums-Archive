### Post by gunksta on 2009-03-18
KDE 4.x replaces the infamous kicker panel with plasma. Plasma makes heavy use of argb, which hammered my ati card. It's an old card and the proprietary drivers don't support it. The new radeonhd drivers don't support it.

I'm using a FireGL 9000 (chipset r250).

Using the autogenerated settings, my CPU was running at anywhere from 30 - 80 percent load, just to support the desktop. Needless to say, actually using the computer was a real challenge.

After doing some looking around on the net and experimenting with some different values, I have come up with an xorg.conf file that lets me use the radeon driver for KDE 4.2 without stressing the hell out of my CPU.

```
Section "Device"
    Identifier    "Configured Video Device"
    Driver        "radeon"
    Option         "AccelMethod"         "EXA"
    Option         "MigrationHeuristic"     "greedy"
    Option         "AccelDFS"         "true"
        Option         "EnablePageFlip"     "true"
    Option         "EnableDepthMoves"     "true"
    Option         "AGPMode"         "8"
    Option         "ColorTiling"         "on"
EndSection

Section "Monitor"
    Identifier    "Configured Monitor"
EndSection

Section "Screen"
    Identifier    "Default Screen"
    Monitor        "Configured Monitor"
    Device        "Configured Video Device"
EndSection

Section "Module"
  Load  "glx"
  Load  "dri"
  Load  "drm"
EndSection

Section "DRI"
 Group        "video"
 Mode         0666
EndSection
```My CPU is now running 1 - 5 percent, to support the KDE desktop. Needless to say, I am pleased. I do not claim that this is perfect and I would certainly appreciate any feedback / advice on what I've come up with. When jaunty hits gold, it would be nice if we could have some documentation to help people use older ati cards with modern desktops.

### Post by zika on 2009-03-22
that was a good starting point. I do not need this option but, since You want to reduce CPU usage, Ypu would like to try (from *man radeon*)```
Option "DMAForXv" "boolean"
              Try or don&#8217;t try to use DMA for Xv image  transfers.  This  will
              reduce  CPU  usage  when  playing  big videos like DVDs, but may
              cause instabilities.  Default: on.
```my version of xorg.conf is now:```
Section "Device"
    Identifier     "Configured Video Device"
    Option         "AccelDFS" "on"
    Option         "AccelMethod"  "EXA"
    Option         "MigrationHeuristic"  "smart"
    Option         "EnablePageFlip"  "on"
    Option         "EnableDepthMoves"  "on"
    Option         "ColorTiling"  "on"
    Option         "DisplayPriority"  "HIGH"
    Option         "DepthBits"  "24"
    Option         "SubPixelOrder"  "RBG"
    Option         "DRI" "on"
    Option         "FBTexPercent"  "0"
    Option         "RenderAccel"  "on"
    Option         "DMAForXv"  "off"
    Option         "BusType" "PCIE"
EndSection

Section "Monitor"
    Identifier     "Configured Monitor"
EndSection

Section "Screen"
    Identifier     "Default Screen"
    Monitor        "Configured Monitor"
    Device         "Configured Video Device"
    SubSection "Display"
        Viewport   0 0
        Depth     24
    EndSubSection
EndSection
```I must admit that with all this tweaking (lot of values of options given above are defaults that are OK with me) I did not gain much, if anything, really, but it was a good experience in refreshing the state of radeon ...  radeon is just OK "from the box" for 2D, now ... waiting for next steps ...
