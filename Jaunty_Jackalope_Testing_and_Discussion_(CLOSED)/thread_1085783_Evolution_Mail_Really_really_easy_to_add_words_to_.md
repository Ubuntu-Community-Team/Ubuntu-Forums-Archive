### Post by Mr. Picklesworth on 2009-03-03
Has anyone else noticed that it is ridiculously easy to add a misspelled word to the aspell dictionary with Evolution Mail, yet ridiculously hard to remove it?

I've done it at least 5 times now. I misspelled something, so I right clicked for spelling suggestions. My finger slips and EVERY TIME I end up releasing the mouse button by accident over "Add word to dictionary." Even though that option is totally destructive and unusual, Evolution does it unquestioningly. It does not create an Undo item, and it seems the only way to Remove the word from the dictionary is to directly modify aspell's configuration files.

I can't decide where to file a bug report, or if there even is a bug to report. So, first of all, has anyone else encountered this issue? With regards to hitting that item on the menu all the time, it happens when the mail composing window is near the bottom of my screen so the menu opens with its middle below my mouse pointer. Maybe there's some bug-report-worthy scientific reason that applies more universally than I think for why I keep doing this :/

(One possible solution that would be totally awesome is a global spellchecking preferences tool in GNOME, and a libnotify bubble when things are added to the dictionary. After all, it's really cool that everything uses Aspell and it would be nice to see that consistency utilized to its fullest).
