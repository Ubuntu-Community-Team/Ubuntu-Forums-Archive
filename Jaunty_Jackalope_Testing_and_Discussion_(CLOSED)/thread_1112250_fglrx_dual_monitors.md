### Post by Woody1987 on 2009-03-31
Ive installed the fglrx 9.4 drivers in jaunty and have managed to setup dual monitors using the display utility in system->preferences. Problem is, despite that, everything is stretching across both monitors. This doesnt happen on the open source radeonHD drivers they work fine in this respect, in that programs occupy only one monitor. I would really like to see the fglrx drivers working like this to. Its as though programs are seeing my monitors as one big screen rather than two next to each other as is clearly shown in the display utility. Im using a radeon HD 4870, jaunty 64bit fully updated. Any help appreciated.

### Post by rolandrock on 2009-03-31
I think you should use "ATI Catalyst Control Centre" to configure dual-head.  Also, Xinerama can cause this effect so make sure that is not installed.

Please post /etc/X11/xorg.conf and output of 'fglrxinfo -n' and 'glxinfo'.

Cheers

### Post by rolandrock on 2009-03-31
Be careful installing older drivers on top of newer ones.  You will have to be very sure that you have completely uninstalled the newer ones.

The xorg.conf above indicate that you have a large virtual desktop like you are seeing.  This means that the config file is being interpreted correctly.  So you have two options:

1. Get the ATI Catalyst Control Centre running in super user mode.
2. Hack the xorg.conf yourself (here be dragons!).

Option 1.  Run this in a terminal (works on my quite old installation):

amdxdg-su -c amdcccle

Or if that fails, try:

gksudo amdcccle

Hopefully one of these will start the control centre with all options available.  Set up the correct config (WITHOUT Xinerama) and save.  Restart X.

If this still does not work.  You will have to start playing with xorg.conf.

Option 2.

Compare my (untested!) attached xorg.conf with yours.  This is an example of how to contruct a basic dualhead with fglrx.  You should merge any necessary sections from your existing xorg.conf into this file (e.g. keyboard and mouse settings if they exist).  You should backup your existing xorg.conf and replace it with your new one (assume new xorg.conf is in home documents folder called new_xorg.conf)...

sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.bak1
sudo cp ~/Documents/new_xorg.conf /etc/X11/xorg.conf

Restart X.

On failure, to revert:

sudo cp /etc/X11/xorg.conf_bak1 /etc/X11/xorg.conf

If this whole issue is caused by the super user version of catalyst control centre not being installed, then it should be easy to find a permanent fix for this.

Hi arrow.69,

What you are experiencing is dual-head.  Two independant desktops that you control from one computer.  You can move your mouse cursor between the screens but not applications.  If you prefer the 'extended desktop' (more like the Windows experience) then investigate Xinerama.

As for Compiz, I always disable effects on machines that use dual-head because it has caused me problems in the past (expecially with Video playback).  BUT it has been about a year since I did this with an ATI card and drivers are improving all the time so perhaps you will have more success.

If you start firefox from a panel launcher, then right click on the panel icon, in properties,change command to 'firefox -no-remote' (repeat on both heads).  That should fix firefox.

In a dual-head setup main display is called :0.0 and secondary :0.1.  e.g., running this on your laptop should start a terminal on the other head:

Not very convenient but a place to start in your investigations.
