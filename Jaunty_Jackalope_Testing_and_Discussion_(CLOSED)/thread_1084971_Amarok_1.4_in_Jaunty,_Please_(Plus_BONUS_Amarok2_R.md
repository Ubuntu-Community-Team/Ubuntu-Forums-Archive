### Post by dentaku65 on 2009-03-08
There is another way to install amarok 1.4.x
Add this repo;
```
###KDE3 & amaroK 1.4
deb http://ppa.launchpad.net/kb9vqf/ubuntu intrepid main
deb-src http://ppa.launchpad.net/kb9vqf/ubuntu intrepid main
```
Be sure that is pointing to intrepid, the jaunty pkg is still not available.
Add the GPG signing key:
```
wget http://apt.pearsoncomputing.net/public.gpg
sudo apt-key add public.gpg
```
Install amaroK:
```
sudo apt-get install amarok-kde3
```

Enjoy amaroK 1.4

### Post by george1984 on 2009-03-17
Hello all. Please let me tell you what Amarok 1.4 means to me and why I need it in Jaunty.

                                                                                                                                               Amarok 1.4 is fundamentally a music library and catalog management system, which is also a music player. It allows the catalog to be classified, sorted, examined and displayed umpteen different ways at the user's discretion. The GUI is designed for this purpose, giving priority to the comprehensive display of catalog information. Not a square centimeter of screen space is wasted. Other secondary information, such as cover art, is relegated to lower levels or alternative modes.

These characteristics have made Amarok unique. There is no equivalent alternative that I know of.

The developers of Amarok2 have with what appears to be a deliberate policy rejected these characteristics, instead producing what is in effect a new and different app. 

 This is not a question of bugs to be fixed, or functions added. It is unlikely that future development of Amarok2 will significantly meet the requirements of Amarok 1.4 "library catalog" users, as this would require a major shift of policy. Also, the developers have responded to considerable criticism with a rather insular attitude.

It would have been better if they had converted Amarok 1.4 to Qt4 first and then produced their new app. as it might be that Amarok 1.4 and Amarok2 have conflicting objectives which cannot be satisfied in single app. Consumers would at least have been given a fair choice. 

As there is no adequate substitute I will need to keep Amarok 1.4 running into the foreseeable future. Given that I have no technical ability, I had thought that the only way that I could do this is by running Hardy indefinitely - that is beyond its support date - assuming that this will be possible, even though I know this to be bad practice. Hopefully some of the info in this thread will enable me (a non-technical user) to run Amarok 1.4 in Jaunty. However, I am worried about the future.

It really does seem astonishing that one of the flagship linux apps. can be summarily abolished in this way - yes I know it can still as yet be built from source, but you have to know where to look and how to do it. One has the feeling that the developers would like to see the back of Amarok 1.4 because of the unfavorable comparisons being made.

	Sorry about that last remark which was a little intemperate. The aim is to be constructive.


After Jaunty is released it is possible that a significant number of users will be not just  be disappointed but even distraught to find Amarok 1.4 absent from the repository.

Is it possible that some official or semi-official means can be found to give these users the choice of easily installing  Amarok 1.4 if they wish?
                                                                                                                                               Amarok 1.4 and Amarok2 really are distinctly different apps.


I would like to stress that developers have every right to develop anything that they choose. However, I believe that consumers should also have a right to a real choice.

Please, I do not wish to start a war. I just want to continue running Amarok 1.4

### Post by sigi23 on 2009-04-14
Here's yet another way to have Amarok 1.4 on Jaunty, if you don't want to use a PPA:

[LIST=1][*] Add the intrepid repository to your /etc/apt/sources.list
[*] Add the following to /etc/apt/preferences:
```
Package: amarok
Pin: version 1.4.10*
Pin-Priority: 1001

Package: amarok-common
Pin: version 1.4.10*
Pin-Priority: 1001

Package: amarok-engines
Pin: version 1.4.10*
Pin-Priority: 1001

```
[*] Run 'apt-get update; apt-get upgrade'. This will downgrade amarok and friends to 1.4.10 if it's installed.
[*] From now on, only 1.4.10 will be visible, you won't even see Amarok 2 in any apt-based Tools.[/list]

### Post by chensamurai on 2009-04-16
Hey **tubegeek**, I had the exact same problem as you with Synaptic Package Manager not listing the Amarok14 package and having to use apt-get install to actually get the package.

I started a thread about this and found a solution for it here:
[http://ubuntuforums.org/showthread.php?t=1125833](http://ubuntuforums.org/showthread.php?t=1125833)


Basically, the cached list of packages that Synpatic uses does not always get updated with repository-add updates. Bug is reported here:
[https://bugs.launchpad.net/ubuntu/+s...ic/+bug/288797](https://bugs.launchpad.net/ubuntu/+s...ic/+bug/288797)

Must run this "***_sudo update-apt-xapian-index_***" to re-index the package lists for the quick search feature of Synaptic Package Manager.
