### Post by mikechant on 2009-03-31
An upgrade should retain all your data and settings. But with any system it's always advisable to back up important data before making a major change.
Possible issues with upgrading:
a) Software you have installed from outside the Ubuntu repositories may break and need manual attention
b) System-wide configuration files you have edited may get replaced with default versions during the upgrade and need changes reapplying
c) Upgrading involves changing your presumably correctly running system to a new version which may have problems, and you can't easily and quickly go back to the old release or switch back and forward.

Personally, because I want to be sure that I'll always have a running system, I always install a new release in a separate partition and dual boot until I've tested it; when I'm happy with the new release, I start using it with my separate /home partition.
