### Post by 4Orbs on 2009-04-05
> Error: API mismatch: this NVIDIA driver component has version
96.43.11, but the NVIDIA kernel module's version is 96.43.10

The problem here is that you should have uninstalled the 96.43.11 driver before you went to the Hardware Drivers Mgr and enabled the 96.43.10 driver. I suggest trying this:

1. In the Hardware Drivers Manager, disable the 96.43.10 driver.
2. Reboot, login, then stop the GDM by entering in a terminal:
```
sudo /etc/init.d/gdm stop
```
3. Log in to the console (black) screen. You might need to hit CTL + ALT F1 to get the console login to come up.
4. Uninstall the 96.43.11 driver by entering:
```
sudo nvidia-uninstall
```
5. Start the GDM by entering:
```
sudo /etc/init.d/gdm start
```
6. Reboot. You'll probably be in low-graphics mode, that is ok for now.
7. Login and go to the Hardware Drivers Mgr and activate the 96.43.10 driver. HINT: The progress bar may not show any indication of activity while it downloads and installs the driver. Don't give up or cancel the operation. I had to wait a few minutes before the driver actually finished installing. You will be prompted to reboot.
8. After rebooting, log in and open the nvidia settings manager as root so you can set the default resolution system-wide.
```
gksudo nvidia-settings
```
9. Save the resolution settings to the xorg.conf file by clicking the button labeled as so.
10. Reboot, log back in. Everything should now be the bees knees.

### Post by 4Orbs on 2009-04-05
Ubuntu Hardware Drivers Mgr will install the 96.43.10 nvidia proprietary driver, it is not the open source driver. The open souce driver is what you have if no proprietary drivers are activated (low graphics mode).

The steps I posted were intended to first uninstall the .11 driver which you had manually installed at some point. Then secondly to re-install and activate the .10 driver which works by using the Hardware Drivers Mgr.

For some reason when you have desktop effects (compiz) enabled, the Window Manager (metacity) does not create the frame around your windows. I have the same problem, and have tried using emerald rather than metacity with no success. I also tried fixing this by manually installing the nvidia .11 driver, again no success. Maybe someone has had success using compiz with the nvidia legacy drivers. I would love to hear their method for making it work correctly.

As for the nvidia settings manager, since you don't have it on your system, then maybe you never really had finished manually installing the .11 driver. Have you looked for the nvidia settings manager in System->Administration menu?

You can install the nvidia settings manager through Synaptic Package Mgr or by entering in a terminal:
```
sudo apt-get install nvidia-settings
```
