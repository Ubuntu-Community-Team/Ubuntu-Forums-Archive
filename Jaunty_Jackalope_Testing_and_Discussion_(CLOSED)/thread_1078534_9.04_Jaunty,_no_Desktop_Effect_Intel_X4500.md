### Post by klange on 2009-02-24
> **Stetzen said:**
> Try to run compiz **--replace** from the terminal and post the output here, I think, it will help us to solve a problem. /var/log/Xorg.0.log will also be very useful. It looks like your 3d acceleration is disabled for some reason.
I'd really like to see this sort of thing stop here. Ubuntu has shipped with compiz-manager replacing `compiz` since the beginning, and it automatically executes compiz with the `--replace` option. (The actual compiz binary executable is /usr/bin/compiz.real, compiz-manager is /usr/bin/compiz).

So, in conclusion, in Ubuntu, compiz **--replace** is redundant.

Anyway, on to the point: You have some old "config" files (not really config files, more like trash from a custom install or another repo) from some form of previous Compiz installation. Ask on #compiz-fusion@irc.freenode.net, as I don't have the exact file paths with me at the moment. Basically, one of these files is redirecting the script in /usr/bin/compiz from /usr/bin/compiz.real to /usr/local/bin/compiz (which happens to not exist).

You lack acceleration due to some major issues in EXA on the current drivers. A noted workaround is to disable vertical sync blank (search for it), but you can also help test UXA (again, search for it), which is unstable but faster and has a number of other advantages (which you'd notice once you get Compiz back up).

### Post by starfly on 2009-02-25
YSEESESESEESS!
GOT IT !

change gedit /usr/bin/compiz-manager

change lines 30 and 31 to:
COMPIZ_BIN_PATH="/usr/bin/" # For window decorators and compiz
PLUGIN_PATH="/usr/lib/compiz/"

and then a few lines down, change to:
COMPIZ_NAME="compiz.real" # Final name for compiz (compiz.real)

--------------------------

In the xorg.conf section the Device must be :

Section "Device"
	Identifier "intel"
        Driver  "intel"
        Option  "AccelMethod"   "UXA"
        


Thanks a lot and hope someone will need this someday ;-)
