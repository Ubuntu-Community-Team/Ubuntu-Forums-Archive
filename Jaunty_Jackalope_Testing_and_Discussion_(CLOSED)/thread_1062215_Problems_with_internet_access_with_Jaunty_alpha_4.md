### Post by BobCFC on 2009-02-08
This happened to me as well.  (Starting with Alpha3 may have been fixed since)

When dhcp obtained an ip address it wouldn't fill in the default gateway and I noticed that a route command would allow me to surf and ping etc:

```
sudo route add default gw 192.168.1.254
```

But this is not persistent and had to be done after every reboot, so I edited the **/etc/network/interfaces** file, in the same way as if I were setting up an IP in the old days:

```
# The primary network interface
auto eth0
iface eth0 inet dhcp
gateway 192.168.1.254
```

Now it remembers the default gateway when I reboot and I don't even need network manager now because I am not using wireleess. (You can disable NM in the startup programs Sytem->Prefs->Sessions)
