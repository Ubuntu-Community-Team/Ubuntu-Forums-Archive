### Post by skern03 on 2008-12-20
IMHO, we should not be chasing people back to Windoze, and I've run Ubuntu 8.04 on systems with far less power than the AMD 2000.

Regarding the Samba shares, do yourself a favor and from Synaptic, install system-config-samba. It's a very nice configuration utility for Samba that will help you quickly add shares, and set permissions (like read/write).

Regarding speed, memory (as someone else mentioned) may be a factor, as may be how you installed it. I personally recommend installing it on a separate drive if you are going to dual-boot. I know you can do it on the same drive, this is just my own "hard-won" experience. When you set it up, select manual install, and then set root (/) to 5 GB, swap to 1 or 2 GB and set the rest of the drive to /home. I have a bunch of old drives around, so this is easy for me. But even purchasing one as cheap as they are today, is preferable. Since I've been slammed for this line of comment, THIS IS JUST MY OPINION.

### Post by yeaitsdark on 2009-01-04
For RAM I am partial to Corsair. Also the big think is to make sure it works with your motherboard, and it's important also to know the speed of your current ram otherwise you might spend money getting "nicer" RAM only to have the motherboard slow it down to match the currently installed RAM.

To help you with the fact crucial has no support for using the tool in Linux instead you may use this command to determine all you need to know about your current ram.

```
sudo lshw -class memory
```

Main thing here to look for is "capacity" and "clock" for instance on my server machine it happens to be 1GiB and 800MHz times 2,so I would then ADD 2 1GB sticks at 800MHz.

Most RAM manufactures will include a "compatibility" list and simply check to make sure your motherboard is in that list. (9 times out of 10 as long as you get the same speed it will work just fine, but better to check first)

Hopefully that helps you a bit.
