### Post by almigi on 2009-07-06
Distro Hopping is fun!  I used to have a tendency to try a distro, love it, and post a nice message in the forum for that distro talking about how I finally found the Linux for me and blah blah blah...

Then after a few days, I'll read about a new distro, or because of fond memories of a previous distro want to go back to something else.  All my distro hopping has taught me a few things though...

1. Linux is Linux. In many cases any distro can be any other distro with the right selection of packages.  In my more noobish days I would fall in love with a distro only to realize that everything I loved about it could be duplicated on a previous distro I used, if only I installed the right packages.

2. Community is vital. Emply forums (or forums with more spam then actual messages from users) are a major turn off.

3. Good repositories.  It makes sense to have a package manager handle as much of your installation as you can.  A good official repo managed by the people behind the distro (and maybe some good community repos) can really enhance the Linux experience.  Read about a cool game or handy utility in Linux Journal or Linux Format?  Nice to be able to just "sudo apt-get install <package>" instead of manually tracking down the .deb, .rpm, or source and all the dependencies.

4. End User philosophy.  I'm pretty good with computers, and I'm getting better with Linux every day.  But, I'm also kind of lazy. If the distro can automatically detect my hardware and set it up for me, that's a huge plus.

5. Longevity.  I don't want to use a distro that I think is going to be dead in a few months.
