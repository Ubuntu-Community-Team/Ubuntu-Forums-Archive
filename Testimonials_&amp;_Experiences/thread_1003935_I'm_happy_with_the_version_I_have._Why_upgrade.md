### Post by Old_Grey_Wolf on 2008-12-06
Sorry for the long post. Yeah, I hate long posts. :)

I am tired of reading the testimonials of people having problems when they upgrade to the latest release without testing it first.

I have been using Ubuntu since 6.10. It wasn't until 7.04 that I started using Ubuntu as my primary OS. Then at 7.10 it was the only OS I use on my own computers. The family computers are another discussion.

I have a desktop running a LAMP stack on 7.10 as a server. I have tested it with 8.10 by booting from a USB thumb drive. It worked; however, I don't plan on upgrading it until support for 7.10 ends in April 2009. It is working fine so I don't see that there is a need to upgrade right away.

My daughter got divorced, and moved back home along with her three children. I set up a desktop in the living room for everyone to use. The grandchildren use it to play games; therefore, it runs MS Windows as default. For my own convenience I installed Ubuntu 8.04 on it with WUBI. It works for everyone. So I don't see a need to upgrade. 8.04 is supported until 11.4 is released by-the-way. 

I run 8.10 on my laptop. I tested it before upgrading from 8.04. The laptop is my test machine. That is the only reason it was upgraded. It is working fine. No problems. All hardware recognized. Easiest upgrade ever. Yeah, I know, your mileage may very.

On the MS Windows machines that I have been maintaining, I don't install the patches until I read reviews or try them myself. So, with Ubuntu I don't expect it to be any different. Edit: And I see a lot of MS Windows machines running very old versions of MS Office. So, getting the latest applications isn't necessarily the reason to upgrade.

I don't see the need to go to the latest release when it is available. If an older version works, and still gets security updates, I have no problem staying with an older version. 

I guess there is just something wrong with me.
