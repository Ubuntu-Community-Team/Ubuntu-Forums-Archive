### Post by GolanTrevize on 2008-05-09
Hi,

I want to express my deep disappointment and my raising rage, regarding the way Ubuntu and its descendents are packed, tested and released for updates during the last year.

I did choose this distribution, because I like the community way of working, because I like Linux and because everything seems to work out of the box. I started using it about 2 years before now.

During the first year everything was fine. I did not care on updates - they simply worked. This did not surprise me so much, because I was used this behaviour from Knoppix and Debian. Anyway I also realize that this is a huge achievement of the community and a terrible pile of work.

The last year changed my attitude. To sum up why:
[LIST]
[*]I graded up from 6.04 to 7: My scanner ceased working
[*]Some day my eclipse did not work any more. Big deal - half of the world could not work anymore
[*]Some update broke the usb-connection to my camera - I used this for five years without any problems before
[*]The last 7.10 update broke my emacs - had to completely remove and reinstall it
[*]the emacs jde package is still broken in 7.10 though the first bugreport was received a long time ago and the solution consisted only of switching the position of two lines
[/LIST]

Now I hear hardy is out. What I hear first from hardy is, that it ships with **Firefox 3 _Beta!!!!_**. 8.04 is a long-term support version. 

The irregularities and the subsequent quarrels I have with the distribution sum up and grew more frequently within the last half of an year. And this are only my experiances with the distribution. Summing up everything in the forums and on launchpad leave a very desolate picture.

What the heck are you doing? Don't you test anymore?
Do you miss people? Does noone control the projects? Do you leave metrics be metrics and ignore the increasing pile of crap?

If kernel 2.6.22 is not apt for lots of hardware - as it is -, because ignorant kernel teams try put their sight of the world on reality - leave it out.
Too many people depend on their system nowadays. 
**Linux is no toy system any more - it is a productive system with thousands of users!!!**

Who the hell did suggest using firefox 3 Beta as standard for a long term release. If the release date of the firefox team does not match the release-date of Ubuntu: Leave it out

Regarding the scanner problem: After some update cycles it worked again. Why not testing before releasing?

I do not like to say this - but I have got a very, very unprofessional impression of the way you do things.

I will definitely not update on hardy - Who knows if my laptop still works at all then..... And many threads in this forum confirm this suspections.

If anybody here knows a good and easy to use distribution aside ubuntu - let me know, please. 

I need a system to work with - not to play with. If I would like to toy around, I'd use development releases.

What about debian - has anybody experiance with current Debian ?

Regarding the Ubuntu situation: nice flyers and advertisement is not the complete picture. Please try to improve your processes and releases. It would be a pity if such a grand distribution would change to meaninglessnes, because it is released headless, with too few testing and simply really really buggy.

This is the M$-way - and these guys start improving. If you want to do a favor to Linux: improve!!!

And - if any kernel developer would read this post by accident:
Stop beeing such sticklers of principles. Recall Linus' pragmatism, that made Linux what it is.

regards
golan

### Post by luvr on 2008-05-25
I've been wondering lately...

I have no idea if I'm talking any sense here, but couldn't something like a *"pre-LTS"* release be helpful? Let me explain:
[LIST]
[*]If some release ***"N"*** is to be a Long-Term Support release, then ***"N-1"*** will be its *"pre-LTS"* release.
Thus, as a hypothetical example, if 10.04 were to be an LTS release, then 9.10 would be its *"pre-LTS"* release.
[*]The *"pre-LTS"* release is meant to *"morph"* into the next LTS release.
That is, users who install the *"pre-LTS"* release and keep their systems current (i.e., install the updates) will end up with the actual LTS release when that comes out.
[*]The LTS release is really just a remastered *"pre-LTS"* with all updates, available at cut-off, applied to it--meaning, all updates that are available when it is considered ready to be assigned LTS status.
[*]When the LTS release comes out, *"pre-LTS"* effectively ceases to exist (since it actually morphed into LTS).
[/LIST]
The most obvious advantage, in my opinion, would be that the (developing) LTS release gets tested by a wider audience. Furthermore, the *"pre-LTS"* release can include, e.g., a Firefox beta release, with the aim of upgrading to its final release by the time LTS comes out.

Does this make any sense at all? Could it work? Would it help any?
