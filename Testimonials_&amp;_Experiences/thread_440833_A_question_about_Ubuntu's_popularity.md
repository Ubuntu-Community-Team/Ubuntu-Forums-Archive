### Post by laptoplinux on 2007-05-12
The purpose of the post is to ask an honest question to the Ubuntu fan base, but to fully understand where this question is coming from, perhaps a brief history is in order: 

I'm coming here as someone new to Ubuntu, but with roughly 3 years of experience using Linux for scientific and programming work.  I came into the Linux scene around the time Fedora Core 1 released.  That was my first distro, but I soon ditched it for Mandrake 10.0.  I stayed with Mandrake/Mandriva until SUSE went open, and then started using openSUSE, which I've been more impressed with with each new release.  Over time, I've tried going back to Fedora a time or two, but until recently, the relative ease of use and driver support of Fedora just hasn't been up to the level of the other top distros, most notably in the wireless department (cough 3945 cough).  And throughout all this time I've occasionally dabbled with other minor Linux distros (Yopper, Knoppix, DSL, Mepis for example)

I came to Feisty as my first go at an Ubuntu distro, and with great expectations.  After hearing for years about how much better Ubuntu was supposed to be in all areas and seeing the excitement build more with each new version, I thought perhaps the Ubuntu fans knew something I didn't.  Although Linux on the desktop had made great strides since 2004, perhaps the answer to those few last remaining annoyances was just one disc iso burn away in Ubuntu.  

But sadly, that was not the case for me.  

I was pleased to note that installation went smoothly (something I expect from all distros these days) and that my Intel Pro Wireless 3945 card was detected on boot and smoothly integrated into Network Manager (something only Ubuntu and openSUSE have been able to do so far to my knowledge), but things went downhill from there.  Using the nvidia drivers resulted in the refresh rate bug that seems to be a common problem in 7.04.  And regardless of what refresh rate the dialog box or the operating system though my computer was running at, my eyes could clearly and almost painfully tell that something was not right.  Putting that problem aside for the moment, I went looking to other areas that I generally customize soon after an install.  But where distros like Mandriva, openSUSE, and more recently even Fedora offered clean and easy to find configuration options, I spent a good deal of time searching Ubuntu (and Kubuntu -- I tend to prefer KDE over GNOME), and came away disappointed.  Some configurations were hard to find, some configuration options existed in the Ubuntu GNOME desktop but were nowhere to be found in the Kubuntu KDE desktop, and other options didn't exist at all.  

Moving back to the refresh rate problem, I decided to disable the nvidia drivers until a fix came out.  Problem was, I forgot to disable the compiz 3D desktop as well.  So on next boot, GNOME turned up as a "White Screen of Death."  Realizing Compiz to be the problem, I rebooted into KDE to try to disable the 3D desktop, but could not find any option to do so under that desktop manager, leaving GNOME completely maimed and useless.  Editing xorg.conf proved fruitless as well.  That was the final straw for me.  I prefer KDE, but I like having the freedom to go back and forth between them.  It seems odd that compiz would even activate without the nvidia drivers, but that is beside the point.

So, after all this, here is my question:  why exactly is Ubuntu so popular?  I've used many distros and found nearly all of them to show more overall polish and usability than Feisty.  I'm not trying to bash this distro.  I had hoped it would solve my remaining Linux issues, and if I really hated it that much, I wouldn't waste my time typing this long post; I'd just go back to my openSUSE and get on with my work.  But I'd like to try to understand Ubuntu.  What is it that makes it so popular when many other distros do things at least as well as it does or better?  

A friend of mine, a Linux user for about a decade and staunch Redhat-Fedora fan (I think he started with Red Hat version 5.0) had a theory about it:  he thinks that perhaps Ubuntu is the Windows of the Linux world:  simplistic, easy to use, for the masses.  

While I don't necessarily share his views, I did find that Ubuntu tried to bend over backwards to make things simple, but often it made things so simple that doing any advanced configuration or customization was next to impossible.  It all "just worked," but if I wanted to do anything out of the ordinary, tough luck.  That was my take on the day I spent with Ubuntu, and I really can't use a distro unless I know that it can be counted on for anything.  

In the past, in addition to all of the normal computer things that a Windows or Mac user might do, I've used my Linux box to write scientific papers, design images and schematics to use in papers and posters,  build, compile, and run high energy particle physics simulations, write code to do non-trivial quantum mechanics calculations, and maintain web sites, so I need to know that my Linux distro has enough flexibility to handle anything I throw at it.  

So please, enlighten me.  I really do like Ubuntu, and I respect its accomplishments in the last few years.  I applaud its stance on giving users an easy way to include non-OSS drivers because in the real world, getting the job done with tools that work is more important than silly philosophical quibbles.  But I need a distro that is easy to use at the beginner level **and** at a more advanced level .  I seem to have missed the magic of Ubuntu, so please explain it to me.
