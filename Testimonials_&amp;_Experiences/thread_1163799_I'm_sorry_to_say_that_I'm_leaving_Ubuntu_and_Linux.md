### Post by GL_NORWAY on 2009-05-19
First off I must say that I was impressed by Ubuntu and what Linux in general can offer for 
free of charge. Not to mention all the supported software.

My experience after some more in-depth use was not that positive though. I noticed how well 
things worked in the windows mode, and I suppose that most users can stick to that as long as 
they use Linux for very simple and fundamental tasks. Windows/GUI makes things easier and 
faster, something I loved about MS Windows 3.1 and eventually 95 going from MS DOS 5. 
So I'm used to use a console/shell to do stuff on a computer -- but I do not miss it! The 
problem with Ubuntu/Linux is that as soon as you start to do some things that go a little bit 
beyond very simple tasks like browsing the web and writing documents, you often meet a 
wall that is labelled "root". So to gain access you need to open the console and start to write 
sudo commands and navigate trough directories by writing. The root security is so darn strict 
that I felt like loosing much of my beloved control. I know that this is a security matter, but 
never ever have my Windows XP been cracked, hacked or gotten infected by its way less file 
system security policy -- even I who always run as the XP Administrator. So when I started to 
use Ubuntu like I use XP, I too often found my self being forced to run the terminal and start 
to write sudo commands. And even that command did not give me the file system access that 
I wanted in the terminal. So being so depended on a terminal felt like going back to an 
computer age that I really don't miss. And of course the lack of control by being bullied by this 
ghost named "root" -- in Ubuntu you can't even log in as root in the fashionable way.

Another thing that I didn't like was the privacy and personal security aspect. As some of you 
may remember, I've asked a lot about privacy software like OS cleaners and especially 
encryption software that can fully encrypt the system. I run a business that requires strong 
client confidentiality and all client data is stored on my computers -- a laptop included. With 
no real solutions for any of those, and in the same time a OS that is very fond of logging (it 
crept me out seeing all those log files in the log file viewer) and all where protected and 
owned by the Ubuntu Ghost (Mr. Root), I did not feel any secure as a new Ubuntu user. And 
in the same time I never got any 100% insurance nor confirming answers when I wondered if 
and where ubuntu logged user activities or data -- it was more like "I think all private data is 
stored below /home/user".

I also met on some serious obstacles when trying to install drivers -- that generally took a lot 
of time and terminal writing.

To sum things up: Ubuntu Linux felt like going back to a time (as in retro) that I don't miss 
nor want to go. I also felt very naked and unprotected with no easy and user friendly way to 
clean the OS or install system encryption -- in the same time as logging is a huge part of the 
OS. And installing drivers to different devices got to be a huge and time consuming task.

Compared to Windows: In XP/Vista you very rarely need the console, but in the same time 
the security is good enough for most people -- everything, even advanced system 
administrator tasks, can be done quickly and easily in a GUI environment. Windows also logs 
stuff, but I don't think it does it in the same grade as ubuntu/Linux. Anyway, for Windows 
there's a huge ocean of different OS cleaners and full system encryption software that is as 
quick and easy as it gets to use -- and they do work. Installing anything is just some few 
clocks on the mouse and it always works.

I really don't mean to bash Ubuntu nor Linux, this is only my personal experience and opinion 
about the system. To be honest, I can't see any good reasons to use Ubuntu/Linux for default 
home tasks or even some advanced (server/admin/business) tasks. It feels way too retro and 
unnecessary complicated -- like cycling around on a monocycle when you have a car that you 
can use, or using an old typewriter to write a novel when you can use a laptop with a printer. 
If I can do the same things way more easily on another system, I'll do just that, and I'll not 
hesitate to do so even if it costs me money. The only reason for needing Linux is if I had one 
particular application by UNIX or anything that required Linux to work and I needed it to 
either make a living or do some very important stuff. It's free though, so I'm not complaining or 
anything, just expressing my opinion.

Thanks to all who helped me and I may come back in the future, but I doubt it really. :P

):P

Chances are that your copy of XP does have viruses, if you have been running it under administrator. Whatever anti-virus program you are using doesn't have your collection of viruses in its definitions.

The security system is a necessity. Running full-time as root is just dumb, because viruses need that level of privilege more than you do. This is the only time I'd say "Go back to Windows" - but if you can't handle the idea of seperation between root and user, then go back to Windows XP. But don't upgrade past Windows XP, because even Microsoft has moved to a very similar security system to what Ubuntu employs.

### Post by 3rdalbum on 2009-05-19
Also: Log files do not store sensitive data - in fact, sensitive data is never stored in world-readable files. Developers on Linux and Unix are very careful about this.

You may be suspicious of log files, but when something goes wrong you'll be glad for their existence. You're correct in saying that Windows doesn't log as much as Linux, and if you said that it's one of the things system administrators would love to change about Windows, then you'd be right there too.

It is easy to set up home partition encryption - use the alternate CD to install, that's what it's there for (read the Ubuntu release notes). You don't need the main system partition to be encrypted, obviously, as anything user-specific is stored under /home/username, not in the same locations as normal system files.

Heck, don't take my word for it: Linux is an open system. You can explore it as you want. Make some "sensitive" data as a test, and then try to find it when you are logged in as a different user. Take a look in the Gnome System Log program and see if the system is logging your sensitive data (it isn't, but you're free to confirm this by yourself).

To answer another assumption of yours: User data is stored, but user activities are not logged. The system does not find them interesting, and they are not important for troubleshooting, so they are not logged. Don't take my word for it - look in the system yourself!

I personally think you're fussing over details and missing the big picture. Drive encryption will slow down someone who has stolen a computer in order to extract its information, but running Windows as root will end off with that same information being sent to Russia, China or Nigeria; drive encryption or no drive encryption.

Or, humourously enough, the developer of your drive encryption software may have a backdoor in their software that triggers the software to send your data to the developer. You wouldn't know, because the software isn't open-source and therefore isn't subject to peer review, and also because Windows won't enable you to see outgoing socket destinations.

I'm also concerned that you described Windows XP's security as "good enough". Windows XP was designed to deal with the threat of Word macro viruses. Yes, you heard me correctly. Its "sophisticated-for-the-time" anti-malware measures are completely useless against today's viruses and today's crackers, and in fact XP predates the use of the word "malware". Yes, Vista is a lot better, but not if you change the configuration of the security which I believe you would probably do. And the biggest vulnerability of Vista is the third-party applications; unlike in the Linux ecosystem, third party developers on Windows generally don't write secure code.

I'm not sure what drivers you had to install, but it's getting very rare to have to install any drivers manually these days.

If you want to give Ubuntu a fair test next time, I'd be happy to help you out.
