### Post by matthanielcm on 2009-06-12
I am currently working on fixing a computer for a friend (who is working on my car for a fair trade) and I designed a dual-boot system. So I first installed Ubuntu 9.04, which was the easiest part of the install. Now, my friend said he wanted Windows however, as he doesn't use his computer for much and I think he may like me presenting it as a "backup" option, I don't think it will hurt. The Ubuntu install took a total of about an hour (pretty good considering he only has 256 MB of RAM). Everything worked right out of the box.

Now, the Windows XP install, that was an entirely (I hope I spelled that right, if anybody cares) different story. First, I didn't mention before that I completely wiped the drive. Then, the install went smooth until the Activation thing came up. It asked for the key and I ignored it for the time being. The Office 2007 suite installed smoothly, as did the other necessary softwares. Then, I tried to install some video drivers supplied to me by the Compaq website (as the computer is a Compaq). It turns out, they have the wrong video drivers posted on their website for that particular model (Presario SR1616NX). Then after installing everything, I attempted the Activation thing.  There was an key taped to the side of a computer whose motherboard had fried, so I figured, hey I have a key that's guaranteed to work. Apparently, OEM keys don't work with the Retail Windows copies. So I called Microsoft to make my case, the first operator gave me another key that also didn't work, who then transferred me to another hotline. Then they told me that the key was not transferable. So, pissed off and angrier at Microsoft that I had ever been (the key was to a computer that couldn't be ever used again!), I decided to find a keygen for Windows XP. I finally found a key that worked and went to activate it. Well, actually through phone activation, this worked flawlessly, I got the installation ID very easily (maybe it helped that I was calling at 3 AM MDT).I was very surprised that I was allowed to activate a blacklisted key (as I later learned it was). However, I then tried to get updates which requires WGA (Windows Genuine "Advantage") which obviously failed. So, then I remembered I had a couple keys I wasn't using from MSDNAA (MicroSoft Developer Network Academic Alliance) through Penn State. I got one of those keys and and it was fine after I reinstalled Windows, being that for whatever reason, I couldn't change my key despite the explanation on the Microsoft support site telling me how I could do so.

So now, everything is working fine. For awhile, I had forgotten why I left Windows, now I remember why. Heck, I was even considering trying out Windows XP on my Linux laptop again. No, it will never again know what Windows XP is. It's time for a beer.

### Post by decoherence on 2009-06-12
Windows XP, like OS X, is really not intended to be an "after market" operating system, even though it is sold as such. It is completely up to the user to install any additional drivers that may be needed to support their hardware.

Because linux was designed to be as 'hardware agnostic' as possible, much effort goes in to auto-detecting hardware and getting it up and running.

If your typical modern distro can't get something up and running it generally means one of two things: 1. there is a legal issue preventing distribution of the hardware's driver, or; 2. the hardware is very new/obscure and nobody has written a driver for it. If it doesn't work for some other reason, that's generally a bug.
