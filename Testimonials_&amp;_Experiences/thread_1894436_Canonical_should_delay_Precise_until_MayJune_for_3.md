### Post by chazinams on 2011-12-12
I recently stopped using Ubuntu because of excessive bugs coupled with all the Graphics (drivers) and excessive Power-use problems that Linux has. Although I now use Windows 7, I developed an affinity for the Unity interface and hoped to return to ubuntu once precise was released. But it's looking more and more like I'm going to have to wait for 14.04 before I can return to ubuntu.

The 3.3 kernel will get a bunch of power saving and graphics driver improvements that just missed the 3.2 window. It's a shame to be stuck for 2 years (LTS) with the 3.2 kernel. I just don't understand why Canonical won't delay Precise for a month or two so as to 1.) deliver Precise with the 3.3 kernel and 2.) spend even more time fixing ubuntu and gnome3 bugs. A short delay would = a better LTS for everybody to use. 

I only use the LTS releases because I've found the non-LTS releases to essentially be unlabeled Beta software that lacks stability and bug-free experiences. And for simplicity and stability assurance, I don't mess with installing new kernels. 

So unless Precise is delayed, I'll continue to use Windows for the foreseeable future (until 14.04). I expect many others will share my choice (although I don't expect anyone on this forum to vocally agree with me :)

I like Ubuntu, but I feel like bad decisions (like not allowing a short delay for an LTS release) is seriously hurting Ubuntu's success.

While I do understand your frustration; there is a truth I have learned in 30 years of software development: You cannot ship what does not yet exist.

The worst offender for creating this situation is actually Microsoft: They are always offering an new 'coming' release of development tools that folks are always temped to delay their releases for.

I know one shop that choose to simply ignore 'odd' numbered releases of visual studio tools just to keep from driving themselves crazy.

In the same way, waiting for 'the next' Kernel/Gnome/X11/whatever release will keep you from ever releasing your software.  Honest; I have been there.

I will not deny that the last few rounds of Ubuntu Non-LTS have been bumpy (this is the nature of large change - and remember that the dropping of Gnome 2.x and Kernel issues are not Canonical's fault).

I too have grown to like Unity very much (while happy to see other adopt GS3 if they prefer it).

In the end, you must use the OS that works for you now.  My own situation is a mixed OS reality:

* Netbook runs 10.04.03 LTS
* Desktop runs 11.10 and Windows 7 (I spend very little time in Win7)
* Work Desktop was just upgrade Friday from XP to a new Win7 box, but I spend all day using connected to linux systems from it.
* My 'massive' laptop (need a big lap for this baby) is running 12.04 
* My Toshiba Thrive tablet is running Android 3.2
* My Synology NAS server is running a custom Linux install; it allows me to share music, movies and photos to all my PCs/Tablets and DNLA equipped TVs and Blue Ray Players.

I guess to say that I use any **one** OS would be false; but most are a linux variant of some sort.

Honestly, there is a good chance you can install 12.04 LTS and then load the new kernel and be golden.

Who ever thought having plenty of choices could be so hard?
