### Post by 3rdalbum on 2011-07-06
> **4Orbs said:**
> These are things you've probably already tried: turn off automatic updates (settings in update-mgr)

Er... no, don't do this. Not only will it NOT help your problem at all, but it will prevent bugfixes and security patches from coming to your computer.

It sounds to me like you've got bad RAM. Windows reserves the upper 2 GiB of RAM on high memory computers for itself; most of this memory never gets used. If this part of the RAM is faulty it might never cause a crash on Windows. However, Linux uses as much RAM as possible for useful things and cache, so a fault in any part of the RAM could cause crashes or freezes.

Try running your computer less one RAM stick to see if you can narrow down whether the crashes occur only during the presence of one particular RAM stick. Then call the RAM stick's manufacturer and claim on your warranty - some (most?) brands give you a lifetime warranty.
