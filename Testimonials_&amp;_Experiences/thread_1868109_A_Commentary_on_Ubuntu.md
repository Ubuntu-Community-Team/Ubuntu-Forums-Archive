### Post by SyntheticShield on 2011-10-23
For about three years I have been using Linux pretty much exclusively.  I have a couple programs that I work with that are related to my love of automotive performance that I have to run in VirtualBox running Windows.  But other than that its been all Linux.  I played around with it off and on maybe a year or so before that.

When I first started using linux everyone that I sought beginner advice from stated, urged, passionately encouraged me to make myself very familiar with the terminal.  I initially resisted of course, but over time as I got more familiar with it the more I used it, the more I went to it first.  While I dont use it exclusively obviously, I do use it and try to find solutions there first.

I have come to absolutely LOVE the flexibility, power, control and speed that it gives me in computing, particularly when it comes to troubleshooting and fine tuning things.

For the period of time that I have used linux I have owned exactly one desktop PC and everything else has been laptops.  I recently counted up five laptops in my house, all less than two years old, and I decided that I should probably consolidate down to one laptop and one desktop.  I have not owned a desktop for a long time and thought it time to build another that was modern and had lots of bells and whistles.

However, I decided to sell some of my laptops and purchase a new one.  Though this time things would be a little different.  I usually purchase laptops with an eye on three things.  Processor, RAM and battery life.  Everything else seemed less of a priority.  This time however I decided to splurge a little and buy I higher end laptop that had better graphics and quality sound.  So I got a Dell XPS with Nvidia graphics card and JBL sound.  I was particularly excited about this.  That was until I tried to install Ubuntu.

I had been primarily a user of Linux Mint but Ubuntu swayed me with the Ubuntu One 5GB of storage.  I do a LOT of word processing on my computer, and create a lot of documents and wanted a way to keep them synchronized between the laptop and the Desktop PC that I was planning on.

Im not a hardcore gamer.  I dont really play a lot of games.  I do from time to time but its not my primary thing.  I do listen to a lot of music as I work on a computer but not so much on the gaming side.  And its not that Linux Mint is all that different from Ubuntu since Mint is based on it but I thought if I could shrink down the Unity bar thing so it was not so intrusive and get everything else working Id be happy.

Well a solid two days trying to get the Nvidia driver working and Im having very serious second thoughts about buying that higher end desktop PC.  Everything not working out of the box has revealed some changes in Ubuntu that have me troubled.

I understand the goal is to make it as usable for as many people as possible and to provide as pleasant of a computing experience as possible.  I get that.  I understand that.  However, it seems that in the process, those that prefer more flexibility have been left behind.  In fact, I draw a parallel to the way Windows has sort of left DOS behind and moved more to a pure graphical environment, at least as far as the everyday user is concerned.

It should not be the chore it is or as hard as it is to drop out of a GUI in linux and to a purely text/terminal environment.  This is a core function of linux in my opinion.  However, booting to a purely terminal environment has been the main road block in me installing the nvidia driver that I downloaded from their site.

I havent even started on the JBL sound yet.  The laptop has JBL sound with the Waves Maxx Audio 3 and I have not been able to find anything for it in Linux.  It has a subwoofer that seems to be working but it would be nice to be able to take advantage of this.

I did eventually get the third party restricted nvidia driver installed and it not cause issues with the boot process.  Though its taken me at least a half dozen or so fresh installs to get working.  Another issue stemming from not being able to drop to a terminal environment.

Look, I understand that Linux not being the dominant OS out there that manufactures dont necessarily produce drivers and such for everything that needs to have one.  Nvidia does, however, and yet I still cannot install it.  So what does this say about Ubuntu users?  Are they left with having to compromise on system specs in order to use it?  Do we have to use systems that dont have the higher end components in order to continue using Ubuntu?  

I am passionate about open source software.  Anyone I run across that is looking for alternatives I preach to them Open Source.  I LOVE open source.  It epitomizes those human qualities most enduring.  To give of ones self, time, talent, skills for the benefit of others with absolutely no guarantee of anything in return.

However, in the advancement to include more in more in the Ubuntu user base let us not leave that which produced such an awesome operating system in the first place.  Let us not forgo something so basic as the terminal environment for the sake of inclusiveness. There is no reason they cannot exist together.

There should be no reason I can tell the OS that on the next boot only I want to drop to a terminal environment.  There is no reason I should not be able to drop back to a standard menu bar if I cannot make the unity bar work and look the way I want.

The openness and freedom to customize the linux system to ones tastes is what makes Linux and Ubuntu so great.  I dont have to contend with searching through piles of registry entries, I just make changes.  If I want a little more control, then dig into the config file that controls what it is that you want to change.

It is a shame that Ubuntu has moved away to some degree from this freedom.  There is no reason they cannot co-exist.  As such, this restriction if you will has given me pause in buying a desktop.  Why spend the money on higher end components if I cannot take advantage of all their function because I choose to run Ubuntu or some variant thereof?

The next release in the works, 12.04 will be nothing more than a polish and stability release so nothing significant will change there, at least I dont think so because my understanding is that it will be a LTS release.  But come on Canonical, dont continue down this path.  It should not be this hard to do some of the things I have needed to do over the last couple of days.  I dont like windows, have hated it even more since Vista and Windows 7 is nothing more than a bit more efficient, marginally less resource intensive version of Vista.  And Ive already tried out Windows 8 and its not going anywhere that I care for either, though I do think they are on to something with an OS that will work on either a laptop/desktop and tablet PC.

What choice am I going to have, however, if I continue to face this choice of buying hardware that has specs I like vs lower end hardware to run an OS I like?

You have come a LONG way Ubuntu, since what, version 4.04 or something like that?  However, your trajectory is taking you down range to such a point you are, I believe, in danger of missing the target.  You want to be out front for sure, but not so far in front that you loose sight of who you are out front for.

I have tried other distros.  And I settled on Ubuntu and in my early use of it, it was very easy to drop to a terminal session, though I rarely had need of it at that time, it was there.  I consider this a core function of Linux.

I agree, however, that ubuntu is about the easy recognizable default for the larger general user base.  I get that, I understand that and I support that.  It can hardly be expected that Ubuntu advance, grow and attract more users without addressing, indeed, doing that.

That said, I do not believe that it has to leave those more advanced features behind or even make them more difficult to access.  I love Ubuntu, but Ubuntu itself with the advent of the Unity interface seems to left many of the more advanced stuff behind.  I think this does more harm than good because at the time you are coddling to the more general user you may drive away the more advanced user (not that I consider myself that advanced of a Linux user) which ultimately, in this open community of support, will be the ones that help the general users along.  It should be like this, it is the spirit of open source.  It should be that the more advanced and knowledgeable users give where and when they can to help those that are trying to get started.  But that does not mean that the tools for both cannot co-exist within Ubuntu and have the same easy access for use.

I ultimately had to install Xubuntu so Im still in the Ubuntu family because I like Ubuntu.  Ive tried and used many of the other distros and I like Ubuntu the most.  But with Xubuntu it took me about two seconds to drop to a purely terminal environment, stop the x-server and try out my nvidia drivers.  Did they work?  NO!  But I was easily able to get to the tools and resources that allowed me to find out instead of spending way too much time and trying out a myriad of solutions scattered around from google searches.

It does peeve me to some extent that the first laptop I have ever bought in my life with higher end gear and I cannot simply get the nvidia driver working when System 76 is selling a machine with higher end specs, including a Nvidia card and running Ubuntu with Unity and it all works together.  But that is another issue altogether.  But it makes no sense that it be so difficult to use basic, core functions of Linux in Ubuntu so that an appeal to more general users be made.

If Ubuntu continues down this path and moves further and further away from those core functions and it filters down to other official Ubuntu distros then I may have no choice but to learn another distro or as you suggested use one that lets me build one to my taste.  That choice is what makes Linux so grand, you can try a number of different things that suits how you use a computer and/or work.

I hope, though, that Ubuntu would consider the direction they seem to be heading and the impact it is going to have.  Ease of use is fantastic.  Up and running out of the box is Excellent. But what about in a larger, more multi-user setting?  That time consumed trying to find the tools to fix such issues becomes more of a factor and perhaps more of a decision making factor in which distro to use and which support services to employ.

### Post by 3rdalbum on 2011-10-26
Nvidia not making an easy-to-use installer is not really Ubuntu's fault.

After all, Ubuntu makes an easy-to-use installer for Nvidia's own driver (albeit one that Ubuntu has packaged itself). That's definitely the preferred way to install it anyhow.

As for installing the Nvidia driver, it's not too difficult when you know how.

First, you need the 'build-essential' package and the 'linux-headers-generic' package.

Secondly, you need to make the Nvidia driver installer executable by root. The easiest way to do this is to type 'sudo chmod a+x ' and then drag the file into the terminal and press Enter. Make sure the Nvidia installer is directly in the home directory.

Thirdly, you need to turn off X and get back to a text terminal. You can do this by typing:

```
sudo service lightdm stop
```

if on Ubuntu 11.10, or:

```
sudo service gdm stop
```

if on Ubuntu 11.04 or earlier.

Log into the text terminal and type:

```
sudo ./NV (press Tab here to fill in the file name
```

When the installation is finished, type the 'sudo service ' etc command, but instead of "stop" put "start".
