### Post by ThanksForNothing on 2010-09-12
I decided to remove Windows 7 Starter from my Acer Aspire One D260 in  favor of a smaller and (hopefully) faster and snappier OS (for a 1.66GHz  Atom with 1GB of RAM) and decided to try Ubuntu Netbook Remix.  Nevertheless, I immediately got the feeling that this was probably not  going to be a good decision and that I would probably run into several  issues when simply trying to install it on my Netbook. Strange, don't  you think? Maybe it has something to do with the fact that I didn't have  that much luck in the past getting various Linux distributions to work  (fully).

But Ubuntu (Netbook Remix) is such a user-friendly desktop OS that even  non-computer-savvy users can work with it (and allegedly even install  it), so what could possibly go wrong, right (FYI: I have more than 20  years experience with computers and code in more than half a dozen  languages on a regular basis)? After all, it's 2010 and most of the  hardware is probably going to work right out of the box, especially if  you install a *NETBOOK* edition on one of the most popular *NETBOOKS*,  the Acer Aspire One. They probably won't expect users to configure an OS  using cryptic command-line expressions anymore if the OS is actively  presented as a user-friendly desktop OS. So let's not worry about it.  Let's just install it. 

The installation went fairly OK; some cryptic messages after  installation which were probably due to the USB drive still being in the  netbook (a simple "please remove USB stick" would have been nice  instead of throwing a stream of cryptic text-mode messages at users),  some strange message about a "WMI device" (don't remember, don't care)  not being installed as well as some strange questions about installing a  keyboard with "dead" keys (because users love confusing stuff like that  - I still don't know what these "dead' keys refer to - Win/Fn keys,  perhaps? Whatever).

After that, there was no way to get the Wireless network connection  working. Great. Then the typical - and fully expected - "Linux" horror  began: cryptic ndiswrapper stuff, using the command line for EVERYTHING  related to the installation of the network adapter, actually wrapping a  Windows driver to get it working on Linux (are you serious?) and in the  end... Nothing. Driver installed, hardware not present, while I'm  absolutely sure that this is the correct driver.

I'm fed up with it. It's the same all over again. It's 2010 and Linux  still doesn't "just" work, it's still not ready for non-computer-savvy  users (and even computer-savvy users still have problems with it) and  it's still not ready for the desktop. I could go on trying to solve it  and I would probably be able to do it after hours of digging on the web,  wrestling with the command line, compiling and configuring stuff, etc.,  but I just don't feel like it anymore. It's not worth the hassle. Just  another one to add to my "bad Linux experiences". Back to Windows...

### Post by Tamlynmac on 2010-09-12
> ThanksForNothing

When deciding which OS to use, one can generally test Ubuntu  compatibility prior to installation. When put in perspective, IMHO one  should consider (prior to purchase) the OS that is to be installed and  not purchase any system that isn't compatible. Obviously, if I preferred  to use Windows - I wouldn't purchase a system designed for Linux or vice  versa.

I never justify poor behavior or performance by referencing other poor  berhavior or performance. Using one OS to define poor performance based  on another seems disingenuous. All OS's have both benefits and faults.  Comparing the merits of one OS to another should include the  identification of not only the benefits, but the weaknesses. A true  comparison requires a controlled environment where compatible systems  are tested and the results should be based on pre-established  benchmarks. Anything less (IMHO) is simply individual assessment or  opinion and thus suspect to interpretation. :-k

Prior to installing Ubuntu on any system (especially for someone else),   I always confirm compatibility and never assume anything. For some  (systems/users) Ubuntu/Linux is not an option. Often it depends on a  number of variables and not just system compatibility. Time investment  in the learning curve, specific software, etc.

I think it comes down to one simple solution. Use what works best for  you and don't waste time or energy on that which doesn't. Personally, I  wouldn't waste time posting, but I do understand the motivation.

I wish you the best of luck and hope you find satisfaction with  whichever OS you prefer and meets your expectations. I see no reason to  admonish or debate your "choice". It's always nice to have a choice, as  the other option is domination and no choice.

Enjoy.
