### Post by calmness on 2011-07-10
I am no geek, but I am not a complete newbie either.  Been using Windows from 3.1 onwards.  When Linux burst on the scene, I thought it was cool and wanted to switchover.  But my experience with all flavors of Linux has not been good.  It is far too complicated for amateurs which is why many people avoid it like the plague.

I tried to have a dual boot of Windows and Linux on my PC.  With a lot of expectations, I tried out different flavors of Linux which I came across: Red Hat 9.0, RHEL 4.0, another distro whose name I don't remember and finally Ubuntu.  I even got a friend to help, but in vain. I never could make my system show up the dual boot menu.  It either booted straightaway into linux or windows.  With Ubuntu, it was worse:  I downloaded 10.10 and it failed to even install. 11.04 did install but never showed up in the boot menu.  I deliberately did not install the boot loader in the root partition as I did not want to mess up my Windows, but on going through the various linux discussions, I was told that I can even install the boot loader in my 2nd HDD and I would still get the option of Windows or Linux.  It didn't work and worse I lost those two partitions on my 2nd HDD where I installed Ubuntu and the swap partition.  Using Linux Live CD didn't work.  I am still unable to access those two partitions.  Some of my hardware did not work with Linux either. After hours of toil, I have decided to let things be. I no longer have the interest or time to waste on installation and configuration of Linux, however good a product it is claimed to be.  

Windows may be bloated and prone to viruses, but at least it does have plug and play capacity which Linux seems to be missing.  And you don't have to go the console window and type out unintelligible commands (It may not be necessary always but it is still required at times).  

You will probably say that all my troubles are because of something I did wrong during installation.  That is precisely the point:  I am no geek and Linux is not as idiot-proof as Windows whether during installation or after.  Ease-of-use is definitely not the USP of Linux.

For those of you who are committed Linux fans, I respect your decision to ditch Windows, but Linux is not my cup of tea.  

So it is good-bye from a one time Linux fan.

### Post by akand074 on 2011-07-10
Most of your problems seem to just be blunders you made. Why didn't you just use the autoinstaller and let it do it's thing for you. If you don't let it automatically do it for you, then it's assumed you know what you're doing. You installed the bootloader on the second HDD, did you switch your BIOS to boot from your second HDD first? Are you using supported hardware? Linux has better plug and play support than Windows actually. With Windows, you have to install your drivers before you can plug it in, or if your device has a storage area it may install the drivers when you plug it in, but those drivers are provided by the device manufacturers. (i.e it's not Windows itself that supports it) while in Ubuntu you just plug it in and you're good to go.

It seems perhaps it's not that Linux isn't ready for you, but that you are not ready for Linux. If you're happy with Windows, there is no point switching over other than to see if you can get a better experience here. That said, most commonly, the people you see here trying to switch over to Ubuntu is because they are unhappy with Windows while a small minority are here just for curiosity. If Windows works for you, by all means, keep using it. But otherwise you should probably seek help when you are having issues rather than blame it on the OS off the bat.

Installing dual boots with Windows is prone to difficulties. Just read through the pages and pages of issues in the installation section of this forum.

The problem is caused by Ubuntu not complying with the MBR System. Windows does comply. Ubuntu wipes it out and bypasses it; Windows has to fix it again. ### Post by YesWeCan on 2011-07-23
> **el_koraco said:**
> From my limited knowledge of bootstrapping, Windows overwrites the MBR with its bootmanager (BOOTMGR). How's that different from GRUB?

It is completely different. The standard MBR code is universal because it will boot any primary partitions PBR. The Grub boot code only runs the Grub bootloader. The standard MBR code is put in by Windows but is not restricted to booting Windows partitions.

If Ubuntu/grub used proper PBR code then it could be booted by the standard MBR just like Windows is, selected by boot flag.

There are excuses why Ubuntu/grub is not installed this way but I think they are essentially political rather than technical.

### Post by YesWeCan on 2011-07-23
> **el_koraco said:**
> I wasn't stating this, I honestly don't know if there are differences. But I don't think the MBR was designed for Windows? The usual partition table is ms-dos, though, so that might make a diffrence?

I think the way to think about it is that the IBM PC bios standard defines an MBR with boot code, partition table and boot flags. This has nothing to do with Windows per se in that it is a universal partition booting system. Very simple, very effective and can be used to boot extended/logical partitions by daisy chaining EBRs.

Ubuntu/grub as normally installed does not choose to comply to this system. Instead it replaces the mbr code, ignores the partition table and boot flags. It sticks a boot program in a ill-defined area just after the mbr which sometimes causes conflicts with other apps. It would be much, much better IMO for dual booting on the same disk if Ubuntu was MBR compliant. There arent any insurmountable technical reasons it cant...after all, Windows has always managed it.

### Post by YesWeCan on 2011-07-23
It is revealling that for GPT formatted disks the Grub folk could no longer use the undefined sectors after the mbr because they are now all used for the much bigger partition table. So they had to bite the bullet and make the grub boot program reliably installable to a partition (has to be tagged bios-grub). This they should have done in the first place for MBR disks but didnt. And for some reason i have not got to the bottom of yet they have disallowed this partition install for MBR disks. If they just allowed it then Ubuntu and bios-grub partitions could be logical and with the right EBR code, no grub replacement of the MBR code would be necessary.
