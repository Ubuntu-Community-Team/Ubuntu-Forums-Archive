### Post by TigerEar on 2010-04-07
I really, really wanted Ubuntu to work out but I think I may have to go back to Windows 7.

it started when I used an Ubuntu 9.10 distribution on a desktop computer at a hotel I was staying at in Japan. I loved the visual effects and the styling - everything was simple and looked beautiful, like a cross between OS-X and Windows.

That, for me, is still my favourite thing about Ubuntu. It's just so pretty to look at. In my eyes it beats Windows and any Mac OS hands down as far as the visual theme goes. So when my Windows 7 began having problems (my fault, because I screwed around with the registry when I didn't really know what I was doing) I thought back to my Ubuntu experience with longing. It was a free replacement. 

There are four main reasons why I'm going back to Windows (I believe, if I can tear myself away from the great looks). Just for future reference and in case anybody involved in the development of Ubuntu is looking. 

1. ** Firefox is awfully slow**
I was a dedicated Firefox User for years until I tried a friend's installation of Google Chrome. It was so much faster and more intuitive than anything Mozilla has done. 3.6 still hasn't managed to catch up, Google Chrome is slow in Ubuntu (nobody seemed to know why, Flash especially was terrible) and the solution offered was "Change to Opera or Firefox." When Opera was also slow with Flash, the solution offered was change to Firefox. No way, I can't go back to that.

2. **Lack of a good movie player**
I know a lot of people here love VLC, but the quality is very, very poor, despite a lot of tweaks that I've been advised to do. Picture is grainy and sound is bad compared to either the KMplayer or MPC on Windows - I hate to say it, but even Windows Media Player does a better job. 

3. **Open Office**
It doesn't work as well as MS Office. I use both extensively (work computers have OO) and going back to Open Office makes simple tasks difficult. 

4. **I use a laptop, not a heater**
I use a notebook and cooling is everything for me. A noisy system makes work a chore - not to mention potentially painful. I manage my temperatures and power consumption through a GUI program in Windows. To achieve a similar result with Ubuntu, I've had to teach myself how to compile scripts, install daemons, read documentation (who does that?) and all sorts of other things. Despite following all of these things to the letter, I still have a noisy, hot laptop with half of the operating time of my Windows system.

Let me make things clear - I was not at all expecting Ubuntu to be similar to Windows. I was expecting to have to learn, and I'm not leaving because I'm not willing to put the effort in (although I think most casual computer users would agree changing OS shouldn't be quite so much effort). I actually enjoy using the Terminal and feel great when I get something right. But all I use my computer for is to browse the web (which isn't as good because it is slow), watch movies (which isn't as good because nobody can suggest a player other than VLC which is poor quality in my opinion) and do work (which isn't as good because Open Office is less intuitive than MSOffice). 

I've read a lot of these threads where people say, "Oh, that's okay, if Windows works better for you go back to it then." I don't think that's the right attitude to take. Ubuntu *should* work better, because God knows Windows isn't perfect. 

I'm writing this to give feedback. I don't particularly want to go back to Windows, but after a week of solid effort and a lot of education (of myself and from others) there are certain, basic things which I think an OS should deliver which Ubuntu hasn't been able to.

I love the way Ubuntu looks. I love the philosophy behind it. I really appreciate the community support. But I don't think, yet, that it is actually Linux for "real people". I'm not a complete computer idiot. I'd put my skills at around moderate. And after 7 days of almost full time effort, I don't think I can do it anymore. It is actually just too difficult.

Thanks for reading,

Tiger Ear

### Post by 2hot6ft2 on 2010-04-07
Let's just cut to the end and install it all at once so you can play pretty much everything.

I could spend a half hour saying go here click this open that  yada yada yada instead do this.

Open a terminal
Applications > Accessories > Terminal
and run these commands one at a time and your password wont be shown when you type it in just type it in and hit enter.
This first one enables the medibuntu repository for restricted codecs and such
```
sudo wget --output-document=/etc/apt/sources.list.d/medibuntu.list http://www.medibuntu.org/sources.list.d/$(lsb_release -cs).list && sudo apt-get --quiet update && sudo apt-get --yes --quiet --allow-unauthenticated install medibuntu-keyring && sudo apt-get --quiet update
```
Remove any browser media player plugin you currently have installed (totem-mozilla is installed by default, but you may have installed mozilla-mplayer or mozilla-plugin-vlc) like this:
```
sudo apt-get remove totem-mozilla mozilla-mplayer mozilla-plugin-vlc
```
then for all the multimedia, mp3's java, divx web player, and dvd playback
```
sudo apt-get install ubuntu-restricted-extras libdvdcss2 gecko-mediaplayer vlc
```
Use your arrow keys and Tab key to select and Enter to accept the Java terms of use.
And to get rid of one package in the restricted extras that causes problems:
```
sudo apt-get purge ttf-mscorefonts-installer
```
ttf-mscorefonts-installer bug report
[https://bugs.launchpad.net/debian/+source/msttcorefonts/+bug/431217](https://bugs.launchpad.net/debian/+source/msttcorefonts/+bug/431217)

[And to install Flash just click this link if using 9.04 or up, then on OK]("apt:adobe-flashplugin?channel=$distro-partner")

And you're done.

### Post by leclerc65 on 2010-04-07
To speed up Firefox, try this:

Open Firefox.

Type about:config in the address bar and hit the Enter key. Find these options, right click each one and change the default values to the ones listed below.

network.http.max-connections: 48 (30)

network.http.max-connections-per-server: 24 (15)

network.http.max-persistent-connections-per-proxy: 12 (8 )

network.http.max-persistent-connections-per-server: 6

network.http.pipelining: true

network.http.pipelining.maxrequests: 32 (4) 

network.http.proxy.pipelining: true

Lastly right-click anywhere and select New-> Integer. Name it "nglayout.initialpaint.delay" (without the ") and set its value to 0 . This value is the amount of time the browser waits before it acts on information it receives.
