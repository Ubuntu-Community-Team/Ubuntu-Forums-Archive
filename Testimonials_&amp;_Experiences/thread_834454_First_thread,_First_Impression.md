### Post by stormbolter on 2008-06-19
This will be a long reply, so brace on: :D

There are many ways to enjoy games under Linux.

**Windows Games**

Wine is one of them. Wine is optimized on a case-by-case basis, so the most popular requests are the ones that work better. Half-Life 2, World of Warcraft, or Oblivion are some of the games that works under wine without much problems.

If you plan to use wine, [the wine application database]("http://appdb.winehq.org/") is your best friend. Most of the time contains good advice and hints on how to make your favorite game work. But, as the database is contributed mainly by volunteers, it's not strange to find contradicting information. Usually, if the game worked for someone, it can work for you. The amount of work required vary. On some games is nearly null (steam games are super-easy to install, for example), while others may involve super-über hacker skills (that will made you ready to be part of the wine team ;) )

Then there is Cedega. Cedega is the commercial version of wine, oriented to games. It may make your life easier as it automates some of the "hacking" you will have to do to get some games running.

Personally, like jbaerbock, the easier option to run windows games is to have a windows partition hidden somewhere, and play the games there. :lolflag:

There's also a free alternative to Cedega, called [PlayOnLinux]("http://www.playonlinux.com/"). It didn't work to me, but supports 60+ games and is relatively easy to use.

**Native Games**

There are not many native releases, compared to windows. Heck, there are not many native releases compared even to MacOSX. That does not mean that there are no native linux games.

As mentioned above, ID releases all its games under linux. There are also native versions of UT. And recently, for example, Penny arcade released it's premier adventure "[Penny Arcade Adventures: On the Rain-slick Precipice of Madness]("http://www.rainslick.com/")" on linux too!

There are two good webpages to know what's running on Linux. One is [LinuxGames]("http://www.linuxgames.com/"), which is a news oriented webpage. The other is the [Linux Game Tome]("http://www.happypenguin.org/"), a database of Linux Games.

As for community-provided games... well. 
First of all a disclaimer: I'm spoiled rotten by commercial games.
Ok, that said: there are not a lot of good games for gamers. Most of ones have an incomplete feel, are buggy or finish abruptly.
There, I said it. I'm a bad, bad person.

There are a couple of really great games, though. One of my favorites is "[Battle for Westnoth]("http://www.wesnoth.org/")", a turn-based strategy game that rocks!

**Emulation**
One great way to game under linux is through emulation, as it is based mostly in opensource efforts, and that means that the linux version of the emulator often is as good as the windows version.

DosBox is a great MSDOS emulator. It supports games like the Ultima saga, Colonization, One Must Fall or Tyrian. There are a lot of good games, specially oldschool RPG's, that work perfectly on this emulator.

For consoles, VisualBoyAdvance (GB, GBC, GBA), SNES9x or ZSNES(SNES), dgen (Genesis) and pcsx (PS1) are all available on Ubuntu repositories (just add the games!). As some of them are command-line based, be sure to get the frontend. KameFu is a frontend for most of them, and it keeps tidy your collection.


**In Conclusion**
There are a lot of good ways to waste your time with Ubuntu. And remember that UbuntuForums has a section dedicated to [Gaming ]("http://ubuntuforums.org/forumdisplay.php?f=93")(altough it doesn't cover wine questions) and [Ubuntu Gamers Arena]("http://gaming.gwos.org/doku.php") for Hardcore Gamers.

Hope this helps!

### Post by mivo on 2008-06-20
Welcome to Linux, and Ubuntu. :)

As for games, well, the older stuff usually works in Wine, but for the newer games the situation is a little dire. If you absolutely need your PC games, dual-booting is a viable option. Personally, I found dual-booting to be a pain, so I just moved my gaming to consoles. Since I'm mostly into RPGs, that worked well. The PC gaming market is really shrinking, unless it's MMORPGs.

There are some really great native Linux games, though. I spent some 300 hours on Quake Wars (they have a Linux client), and also enjoyed Savage 2 (they have two Linux clients, 32- and 64-bit). Generally all the id Software games have Linux versions, so there's actually no shortage, depending on the genres you're into. Games like Mass Effect don't run, or only poorly, so either a console or dual-boot is a good approach.

The nice thing about Linux is that you actually don't have to upgrade your hardware every time a new OS version is released. So if you do shift some of your gaming to consoles, you'll basically only need a new computer every five or so years (you'll find that Linux and Linux applications tend to use your resources well), and a console every 3-5 years. With Windows and Windows gaming, you tend to have to buy a new video card every two years, and RAM, and a new CPU, not to mention all the issues with Windows games being released in a half-finished state that either require tons of tweaking or half a dozen of patches (that's been a huge turn-off for me in the past few years).

### Post by 3rdalbum on 2008-06-20
I noticed that the original poster said "And if not, why not?". The "why not" hasn't been answered for him yet :-)

An operating system not only contains straight out instructions, but also "libraries" - pieces of code that are designed to be used by any program that needs them. A computer program will be written around the libraries that come with the target operating system. In turn, the libraries are written specifically to run with that particular operating system.

Windows programs (including games) depend on Windows libraries, and Windows libraries depend on Windows. On the same token, Linux programs depend on Linux libraries, and Linux libraries depend on Linux. The major difference is that the Windows libraries are closed-source, so are not able to be modified to work with Linux.

The Wine project has reverse-engineered the Windows libraries though, and written equivilants that convert the Windows function calls to Linux calls. The translation is certainly not perfect though, and it's very difficult to marry Windows libraries to the vastly different architecture of a Linux system, so a lot of things don't work at all, and many things don't work properly.

But it's an amazing achievement that Windows programs can be run on Linux at all.
