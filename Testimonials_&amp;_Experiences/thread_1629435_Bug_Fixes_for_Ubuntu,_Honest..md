### Post by ventrical on 2010-11-24
I beg to differ. I think it is exceptional programming. I think the *luck* part may come while you are downloading the actual .ISO file. Just make sure that  part of the process is verified and , for the most part, a ubuntu install will work exceptionally,(depending on if you are using a Windows install to download the .ISO and it doesn't get infected with malware enroute to the harddrive).

However, as I said, exotic systems are the exception to the rule, especially those 'Designed for Microsoft Windows' that have an OEM ID byte-key previously worked out in an earlier partnership - and it is reasonably most likely because of this that many Ubuntu/Linux installs may fail. I recall an incident back from 1990 and a company called Tallgrass that built 8-bit back-up tapedrive adapter cards for IBM clone XTs. It was all about that *undocumented interupt* and if you didn't have the driver for that card then there was no way  one could get it to work.

 Eventually , a level playing field will prevail where PC manufactuers will design their systems for Linux/Ubuntu OSes. These manufacturers will see   that there is a lot less hassle in the overall Open Source concept.

More and more Dell Computers are moving towards this type of thinking. Have been for a while I think.

Lenevo may be next.
