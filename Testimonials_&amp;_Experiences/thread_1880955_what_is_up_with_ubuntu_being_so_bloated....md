### Post by snowpine on 2011-11-14
1st of all, Ubuntu supports this already, it is called the [Minimal Install CD]("https://help.ubuntu.com/community/Installation/MinimalCD").

2nd, unless you have a *very* tiny hard drive, there is **no benefit** to uninstalling apps you don't use. Ubuntu's decision to include bluetooth, Libreoffice, etc. in the default install does not slow down your computer's performance in any way, shape, or form.

3rd, just because you don't need the features, there might be other users out there who have a Bluetooth mouse or need to open a .doc file for work/school. These users would see it as a big FAIL if Ubuntu cannnot meet these basic needs.

### Post by 3rdalbum on 2011-11-15
Honestly, if you think Ubuntu is bloated, you're obviously very new to Linux.

Back in the mid-2000s and presumably earlier, most distros came on DVD. By default they would install four text editors, three image editors, an icon editor, a text-to-speech program, at least two email clients, and three web browsers. If you were lucky you'd also get a few 3D accelerated games that few people could even play out-of-the-box due to missing drivers and no easy way to get them.

Ubuntu changed this within the course of a few years. It was the first really popular distro to have only ONE office suite included, ONE e-mail client, ONE web browser, and no silly frills like icon editors or a little animal that would walk along your desktop if you launched it from the menu.

Asking "Do you want to install Bluetooth, do you want to install Wifi, do you want to install x and y and z" will just complicate the installation process, and would cause a lot of annoyed users when they tried to plug in a Bluetooth adapter later on. Oh sure, you could download the software automatically when the adapter was detected... but that would require a logout first, and an assumed internet connection for the downloading anyway. Remember, some Bluetooth modules are only detected when you turn them on via the physical switch on the computer; you couldn't detect it foolproofly at the installation process.

Why not just have that two megabytes of Bluez installed to disk? It's barely using any memory when not in use. It would save a lot of engineering effort and annoyance for the user.
