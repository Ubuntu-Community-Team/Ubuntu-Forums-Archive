### Post by jlhughes on 2006-08-09
Bought nifty [Hal 9000 Memory Unit]("http://www.thinkgeek.com/computing/drives/806d/"). Plugged it into USB port in Ubuntu box. Drive window openned on desktop. Copied document to USB drive. RMB-click eject disk. 

Off to work.

Take nifty Hal 9000 Memory Unit and insert it into Microsoft XP Pro USB port. Hear sound associated with USB port. Watch messages saying the new device is a USB Mass Storage device. See the "Device is working" message.

But no window. Look at "My Computer." No drive.

Sigh. 

Used "troubleshooting" guide to walk through steps to check driver status. EVERYTHING looked great. Uninstalled and reinstalled driver. Even rebooted the box. Nothing worked. 

Walked over to co-workers Microsoft XP Pro box. Inserted Hal 9000 Memory Unit. Boing! Up pops drive contents window. There's document.

Return to my computer. Finally notice that Hal 9000 is loaded as Drive E: but Drive E: is already mapped to a network drive. Disconnect network Drive E. Insert Hal. Now I can access USB card on Drive E.

Tell me again how a Linux desktop isn't as easy as Windows. 

Ubuntu: It just works!
