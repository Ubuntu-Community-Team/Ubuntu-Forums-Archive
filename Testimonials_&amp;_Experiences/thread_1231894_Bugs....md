### Post by Keyper7 on 2009-08-05
This is a common misconception among non-programmers. You think that, if something is working, it's easy to freeze it and keep it that way forever. Unfortunately, the real world is much more complicated than that. Every new feature and every bug fix involves writing new parts but also, inevitably, changing parts already written. And not even the best test-driven development is capable of considering every single situation and every single hardware the program will run on.

There's only one 100% guaranteed way to avoid introducing new bugs: not touching the code. Meaning: not fixing existing bugs and not introducing new features.
