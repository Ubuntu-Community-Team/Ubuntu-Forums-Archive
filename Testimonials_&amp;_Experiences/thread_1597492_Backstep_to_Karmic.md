### Post by perspectoff on 2010-10-15
Unfortunately, I agree with you.

The reason I say unfortunately is because I upgraded one of my main desktops to Lucid with great difficulty (due to graphics driver/Plymouth splashscreen problems).

Karmic is a great, stable system. One of my favorite teleconferencing servers (BigBlueButton) is not available for Karmic, though, which is why I made the switch to Lucid. Problematically, though, some of my Drupal website modules only run in Karmic.

So I tried running Lucid with a Karmic virtual machine for my Drupal website. Painful, inefficient, and slow.

Now I split the machines. One machine has Lucid with the BigBlueButton teleconferencing server and the website server machine is now reverted back to Karmic (I never do a full upgrade -- I install parallel systems until I'm sure everything works so that I can revert back if it doesn't).

Having said that, on my laptop, which is only 3 months old, I love Lucid (and the only server on the laptop (in addition to the Kubuntu desktop) is a game server). Lucid is a bit faster on newer equipment than is Karmic. On older equipment Karmic is faster.

My nVidia graphics problems have now disappeared with current Lucid updates, as have my Intel graphics problems. Plymouth splashscreen still doesn't work on any of my 10 computers at home, but in reality that is a trivial thing. GRUB2 is now even more of a PITA (it seems to now resist chainloading without a workaround), but since I have it quarantined to do only one job, I don't have to fiddle with it much.

I never consider a version ripe in Ubuntu until the next version comes out. Now that Maverick is out, Lucid is just about ripe.
