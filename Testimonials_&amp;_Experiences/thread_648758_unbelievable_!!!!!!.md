### Post by shadetree physicist on 2007-12-24
First...I must say that the Ubuntu development team is awesome. I can easily see myself using this system instead of windows. Actually, I started my career on unix programming MR techniques on the 1st gen scanners and moved to windows roughly 10 yrs ago...in retrospect, unix based systems were superior systems to develop on!

My experience with bringing up ubuntu has been a good one ... however, it was not without its puzzles.

I initially tried the installation on an aging asus P2-99 m/b with intel P3-450/500mb memory and matrox millenium video card. It ran at least as well as win2k, which in my estimation was the last non-resource hungry OS from microsoft. The problem surfaced when ubuntu would lock up in a kernel panic (flashing caps-lock and scroll-lock leds) seemingly at random. I was getting boot error messages to the effect that bios date (1999) was beyond range (2000) and that certain things had to be *forced*. 

Given I had already been bitten by the ubuntu *bug*...I decided to upgrade the hardware to an asus P2LD2 m/b with intel E4500 duo core/2gb memory and msi geforce 7300 video card. To my amazement, it behaved EXACTLY the same way.

I had seen numerous messages on this forum about duo core / geforce 7300 lockups but there was so many different hardware components involved...it seemed things weren't far from *phase of the moon* type theories. When I thought about it, there was absolutely no connection between my 2 systems hardware-wise except for a linksys wireless usb adapter...which I have been using for years under windows...so I really didn't even think about it when upgrading. Pulling this device off the system and using the onboard marvell gigabit lan port showed it was the cause of the lockups!!!

Ubuntu has been running for 4 days now hardwired to the router...however, I really needed the wireless convenience wrt location. The solution (again) came from the open source community in the form of firmware for wireless routers that permit easy implementation of wireless bridging...and I am running now using a wired connection to a router - which connects wirelessly to the *main* router. Ubuntu has been running rock solid now for 4 days. Contrast this with 4 hours max before kernel panic with the wireless usb adapter.

With retirement on the horizon...I was wondering what I would *play* with to keep mind active...I have found that outlet...The opensource community has my support...and my umpteen years development experience when that time comes

 :guitar:
