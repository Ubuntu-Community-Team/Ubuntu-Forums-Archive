### Post by DMK62 on 2007-07-29
Before anything else I want to thank all the posters / responders here at the Ubuntu forums. They have made the transition over to Linux and Ubuntu an enjoyable experience . Thank you !

A little background ... I have been working with computers since I took my first programming course in Fortran back in the early 80's. Since then I have worked with all the Windows variants as well as Macs and Unix ( Solaris ) and I have to say that Ubuntu and Linux in general has been the one that gives me the most control over my computing environment and allows me to tinker to my hearts content. 

I started with Breezy almost 2 years ago and I am currently running Feisty in a single boot configuration ( upon realizing that I only used XP to run a virus scanner earlier this week I decided to go just with Ubuntu ). I have seen consistent improvement along the way as I upgraded to each new version of Ubuntu. Some things along the way have got broken but I used those moments to learn more about what caused it to happen and the solutions were never far away in the forums here. 

For those considering trying Ubuntu I hope the following information helps. Windows and Linux is like comparing apples and oranges and keeping that in mind when you first start with Ubuntu is very important. You may be trying to do the same task etc on both operating systems but the method to get it done is not always the same. Keep an open mind and be willing to learn. An analogy I use to explain the learning curve is to use the example of an automatic and standard transmission ( Windows using an automatic one and Linux using a standard one ). If you are from a Windows only background don't be surprised if you find yourself grinding gears / clutch as you begin with Linux. Give it some time and more and more will become second nature to you in Linux. Do not feel that you must only go with one operating system. One system will do some things better or easier than the other system and vice versa. Trying over and over again to get something working in Linux but always ending up hitting a brick wall,  has and will,  cause many users to abandon Linux and go back to Windows only when there is the option of running both systems in a dual boot or VM.  Set things up for what you want your computer to do for you and not the opinions of others stating which configuration is best for you.  Browse the posts here on the forum on a regular basis and you will be amazed at how much you can learn in a short time.  The same also applies to those coming from a Mac environment ( I wanted to keep this section as short as possible ).

Mark Shuttleworth and the Ubuntu team and Linux / foss / etc developers have given you the opportunity to use a powerful system and I encourage you to try it. If it works for you then that's great and if it isn't what you need then you will have at least learned along the way.

Dale
