### Post by Tamlynmac on 2012-05-30
> mastablasta
yeah but updates shouldn't do that. they are supposed to be tested before release.

I totally agree.

One can try and justify, but IMHO there's really no excuse for an update  failure. I've only ever experienced one and that was enough. Caution  became my mantra after that update. As the update impacted a number of  our PC's, before I could stop the family from accepting the update.

Prior to updating all our systems run a backup. That way, should a  failure occur recovery time is minimal. If one sets up their drive  appropriately, backups only take a few minutes and can significantly  minimize the frustration of an update (or most other) failure.  Especially, if one only backs up new data - since the last backup. Or,  simply perform an auto backup periodically and don't update until after  the backup has been completed.

It's amazing that after I implemented a solution, the issue never occurs  again (typical). But IMHO it better to be safe than sorry. It's not the  only reason to run a backup, but it is one. Guess it might depend on what you think your time is worth. :smile:

Just my $0.02
