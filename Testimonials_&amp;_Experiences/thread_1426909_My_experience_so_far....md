### Post by dbuck53 on 2010-03-10
A few days ago I was browsing the Ubuntu site and had an interesting idea.  Being a long time, relatively satisfied windows user (since version 3.1) there was no apparent reason to change or experiment with Linux.

But curiousity got the better of me when I remembered the old 900mhz system I have had in my back closet for 5 years or so. This system was too old to upgrade to Windows XP, and so was stuck wearing an unsupported ME release incapable of running most current software releases.

The old computer was not happy to be disturbed.  The CMOS battery had died sometime after retirement, and had to be replaced. It was neccessary to locate my old SIS video driver so I could leave the grey 640x480 default screen resolution. And the cd rom drives seemed reluctant to go back to work, failing to read the boot info on the Ubuntu install disk beyond the start menu for the first two hours of operation.

I finally dropped the master cd drive, promoted the slave, and burned the install disk at 4x (on my real desktop machine). Things started to move from there.  In a very short time I had a hard drive partition and install of Ubuntu on the old beast, effectively removing Windows.  This is an all-or-nothing experiment, no namby-pamby dual boot cop-out!

During the first install, several text notices of errors showed briefly on the screen, although the install continued undaunted.  When the beautiful Unbuntu desktop appeared it seem I was in business.  I played with the interface until the update manager popped up, prompting me to download and install the up-to-date files--which I dutifully did.

Several times during the update and message about an altered file about to be replaced appeared on the screen, and I did the windows thing, opting to replace the older with the newer.  This was a fatal mistake for this install effort. Upon rebooting, the desktop would no longer load and I was given recovery choices, none of which worked.

Since the options for this system were Linux or garbage can, there was nothing lost in trying again. So I did a complete new re-install. Now, maybe it was all the years of inactively have to wear off, but this time everything went perfect, including the after-install updates.  I am typing this message on that machine now, staring at an ancient IBM G40 13 inch monitor.

Ubuntu is a nice looking package, and seems reasonably intuitive, although it took me some time to locate the "shutdown" command link.  I will admit it looks like finding software will be a tedious process.  But the pluses are many. I was immediately on the internet and passed the [www.grc.com]("http://www.grc.com") Leak Test. My video display resolutions all work, my sound card works, and my windows computers can even access this linux machine on my home network.

But most importantly, the Ubuntu install just put an outdated and fairly useless windows machine back online and up-to-date...for free!  You see, my other desktop and my laptop run Windows XP, which will soon also be unsupported.  The only option for windows at that time is to spend hard to come by money on upgrading my windows os and/or my systems.  Ubuntu has just shown me another way that is delightfully inexpensive and effective, and apparently will extend the usefully life of my current machines for years to come.

Next step, a separate partition install of Ubuntu on my main desktop in dual-boot configuration.  Call me namby-pamby...
