### Post by jgfoot on 2010-01-24
I have a 4-year-old Panasonic R4 subnotebook, with Japanese keyboard.  A very nice computer, but it had been running Windows XP, and had managed to accumulate a lot of crud.  Tired of the long boot times, I decided to switch things up and make it dual boot Ubuntu 9.10 and also Windows 7.  (Why bother with Windows?  Because I had never used Windows 7 before and decided it was time to give it a try).

Which OS did you think had the better hardware support?  Answer:

On Ubuntu, I was able to get the Japanese keyboard working.  Wireless worked without my having to install anything or change anything, other than entering in the WPA password.  So did printer sharing.  The laptop went to suspend mode when I closed it.  I didn't test the phone modem, because I haven't used a phone modem since 2002.  The one problem: Sound did not work.

On Windows 7, the keyboard didn't work correctly.  The letters were all in the correct place, but the punctuation was all wrong.  This happened even though I set the keyboard to "Japanese."  (Unlike Ubuntu, which knew about 3 different Japanese keyboard variants, Windows only seemed to know that one).  Wireless did not work at all.  And, sound also did not work.

So, the one thing Ubuntu couldn't do (sound), Windows couldn't do, either; and, Ubuntu could do two things Windows 7 couldn't: keyboard and wireless.

So much for superior hardware support from Windows...

(By the way, it looks like I'm out of luck for sound.  The reason neither OS could do it appears to be because the R4 used proprietary drivers that ran on Windows XP only).

Getting hardware to be working right after installing is arguably better in Linux than in Windows, specially if the hardware already has some years.

The problem is if Ubuntu (or any Linux distro) doesn't recognise the hardware out of the box, it won't be easy (maybe even possible) to get it to work properly.
In Windows, all you have to do is search for drivers in the internet that you'll find them.
