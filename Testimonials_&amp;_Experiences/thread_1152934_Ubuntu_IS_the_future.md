### Post by AlesUbu123 on 2009-05-08
Lately, I have been performing analyses of data from a couple of genetic experiments, and when working in R on XP, i had great trouble with R running out of memory. I needed a 64-bit operating system. 
With ubuntu, it was simple, I downloaded  64bit ubuntu for free, ran it, and I could complete my analyses by running the whole system from a usb key on a server in our genetics lab. 
Although I still can't use Ubuntu as my primary system due to specific caveats, it has become clear to me that linux is on the right way. And it doesn't matter when, developers can take their time, what matters is that foundations are well made. 
The centralised packaging system, fact that it's free, better at managing memory, customisable, this is the future, I think. Windows with latest subscription ideas, slower software and pay-for-new-software-every-two-years ideas just seem go into the wrong direction.  
I can't wait for the time hardware issues are resolved and some (significant) software gaps become filled, so I can run away from Microsoft madness...

### Post by ubuwatson on 2009-05-08
I used to run Virtualbox under Vista and I always received a warning from the system tray that the system was running low on virtual memory - eventually everything would slow to a crawl and I would have to reboot, during this process I couldn't run other applications inside Vista as the system couldn't seem to multi-task.  Now running Ubuntu on the same machine, I can dedicate a full gig of ram to Windows with an ample amount leftover and I am able to surf the web, play music via Rythmbox and run other applications with no slow downs - it's simply amazing how good the Linux kernel is.
