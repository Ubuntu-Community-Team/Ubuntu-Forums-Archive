### Post by InfernalNeutrino on 2009-02-27
Perhaps I can offer the perspective of another relatively new linux user. The one thing that *buntu has going for it is ease of setup, no more, no less (IMHO anyways). I have been coming to realize more and more recently that fundamentally, linux is linux is linux. The point being, as has been said above you can get any distro to do what you want if you learn how, but some just fit certain user styles.

So what is my point? If you have sufficient skill to move past the (relative) hand-holding of ubuntu to something that requires you to do a lot more yourself (like arch, or for me my objective is gentoo) and can benefit from the features of said more advanced distro (rolling release, pacman vs. apt, or whatever you find attractive) there is no reason not to switch. Just be wary of the points people have been making... dual booting or a virtual machine are good ways to try stuff out, and it's always at least a little risky to change from a system you have running nicely to something new. Trust me. I broke my system several times because I got gung-ho about distro-hopping, and ended up having to re-set up my previously existing system just to be able to get work done. Cheers!

### Post by jbysmith on 2009-02-27
I use Arch myself, and I *really* like it a lot.  Ubuntu is an excellent distro to be sure, especially 9.04 which is making me seriously consider switching distros again.

But you're comparing two radically different "OS philosophies".  No real good way to answer the question though.  It depends on what sort of computer user you are.

Do you like to get your hands dirty?  Do you want to learn how the guts work?  Do you want to start with the absolute bare minimum and build your desktop from the ground up?  Arch fits the need there spectacularly.  Since it's a minimalistic approach, by nature it runs very fast; if there's any bloat, you put it in there.  For a "tinkerer", this is an excellent pick.  Just make sure you have wiki.archlinux.org bookmarked as you'll be reading it.. a *lot*.  There's very little hand-holding going on.  It's not hard, but takes significantly longer to set up, as once the installer is done you're looking at a terminal and nothing more.  Need to manually install audio, video, X, your window manager of choice, configure users, etc etc.. if you like tinkering you got it.

Or are you the type who just wants it to work and not deal with the internals? If you just want a desktop that just plain works out of the box with minimal fuss, Ubuntu's the excellent choice for that.  It's more of a "everything and the kitchen sink" approach.  Quick install, quick update, and off you go.

They're both outstanding distros, just wildly different ways of doing things.  If anything, set up VirtualBox or VMWare, and fire up Arch in a VM.  Look before you leap, it's an excellent distro but most certainly not for everyone.

Both distros have advantages, the only way you'll know is by trying them. I would be very unlikely (never say never) to go back to Ubuntu having made the switch to Arch. You may not love it but even if you don't want to keep it, it's a great learning opportunity. The initial setup is longer (you will probably need about a day to do the setup) but in the long run you will save time on not having to do 6 monthly reinstalls. I detest any kind of automatic updater and on Arch I find that updating is easy, you just run a pacman -Syu when you want to. If you run it daily it will normally find upgrades, just make sure you check the Arch homepage and forum before doing it because some updates will cause breakage (which is usually easily solved). There are automatic updaters available for pacman but I'm not sure how well they'd work - Arch users are encouraged to "intelligently upgrade" by checking for any known issues first.

I thoroughly second the idea of a separate partition, that's what I did as I wasn't sure whether I would like Arch and whether I could get it all working. After a day or so I didn't boot back into Ubuntu and when I was sure I didn't need it I wiped the Ubuntu partition. Setting it up was actually much easier than I could have imagined.

### Post by RedSquirrel on 2009-02-28
I haven't used either of the CDs recently. If memory serves, both the Minimal CD and the alternate CD allow you to choose whether you want the full Ubuntu desktop or a command-line system.

The "ideal" aspect of the Minimal CD I was referring to is that you will pull down the latest packages from the mirrors. This means you will have an up-to-date system once you complete the installation.

If you use the alternate CD, you have all of the packages on local media (the CD! :P), which can be advantageous if you have a slow or unreliable internet connection. However, after you complete the installation from the CD, you may have a large number of packages to update.
