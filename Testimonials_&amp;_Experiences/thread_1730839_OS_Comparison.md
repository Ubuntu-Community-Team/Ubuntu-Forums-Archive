### Post by Kururu Souchou on 2011-04-16
Computers. A breakthrough tool that we use in our daily lives. For work, school, entertainment, and many other things. We want the most of our computer. That is what you are reading about. What the best option is for you.

     First, an introduction to the operating systems that you see the most; Windows, the most seen one; OS X, the popular one; and Linux, the small but open source one. There are many versions of each operating system. The most popular windows systems are XP, Vista, and 7; all supporting either 32 or 64 bit processors (these will be discussed later). OS X would be OS X Snow Leopard. Linux has many distributions of itself, including Ubuntu distributions, Debian distributions, Fedora distributions, and Red Hat (mainly used for companies).

     On to the users! There are many types, but I will only name a few, office workers, mathematicians, hardcore gamers, students, the developers, and the hackers (the people who tweak, not break or do illegal stuffs). The office workers will need a user compatible graphical interface, the basics: web, doc, and communication. Mathematicians will need to have a computer capable of doing really large calculations (ie. weather people who need to calculate weather). Hardcore gamers would need something capable of supporting their games, and a suitable. Students need all that an office worker usually needs, plus some educational software. Developers and hacker usually have the whole nine yards.

     Requirements. One of the most important parts of the technology world, they have to all fit. Windows would be a mid-size OS. It can run on most computers. It uses the most popular processors, AMD (mine) and Intel processors. But that&#8217;s not it. These processors all have different architectures. A main difference is the speed in which the processor runs. The main types are 64bit and 32bit. Mine would be called an AMD64. Intel has their names. But you wouldn't call it Intel32. It was just naturally called AMD64. Not all OS's support 64bit computing, because all average users have a 32bit processor. RAM (random access memory) is also a very important part of choosing between 64 and 32bit processors. 64Bit processors would require more RAM because it controls more data, while 32 would require less. But buying ram is easy and cheap. So it&#8217;s not much of a worry. A hardcore gamer, mathematician, and developer/hacker would all go for more as they all run many things. Yes, 10 times more than you. But a weatherman wouldn't be using a PC; he would be using a supercomputer. That's 100+ times the CPUs. Then all those people would also be using lots of HDD space also (hard disk drive). They would be saving many things. But a gamer would be a bit more different. He would also need a real good graphics card. With HD support at least more than 1 gigabyte of memory (this type of graphics card memory is only used for the graphics card; won't work for anything else) and definitely with 3D rendering. Some even have a water-cooling system that makes you need to waterproof your computer and the put it in, like, a fish tank full of cold circulating water. Most gamers would be using Windows, as many of the most popular games support it. The main graphics card manufacturers are NVIDIA and ATI, but there are others. The OS X system can be for office and students, but may be a bit slower. Linux is for everyone who is daring enough to use it. Ubuntu's interface is mostly like windows, except many store bought products don't support it, that's why there are many free alternatives. Excluding games. The sound chipset is also very important to gamers as sounds make the game realistic. Also, you need the different hubs to support your hardware. The hubs are all connected to the motherboard.

     Windows requires a one-gigahertz or higher 32 or 64 processor, which is fine from both Intel and AMD are fine (&#8220;Windows 7&#8221;). One gigabyte of RAM is required. If you want good, useful software, you need to buy your software.

     OS X only supports Intel processors; what sucks even more is that the OS X kernel only barely supports 64bit computing. So Apple is all locked up hardware-wise. Most large companies&#8217; software sometimes works with OS X. But since both Windows and OS X are both written from scratch, so not many non-Windows/Apple workers/developers know its language. Which means that these are OS&#8217;s are fairly closed source. Now Linux distributions are different. The kernels are almost all the same, and its code is open source. So many developers know the language. 
 
     &#8220;A good &#8216;rule of thumb&#8217; is that machines that could run XP, Vista, Windows 7 or OS X will almost always be a lot faster with Ubuntu&#8221; (&#8220;UbuntuSystemRequirements&#8221; 1). That is a summary of the speed of the Ubuntu system. It has many features, and drivers are mostly universal, and free. Also, with most drivers, you don&#8217;t need to find them yourself; it finds it automatically. Also, it automatically updates all your programs. With any bug and problem, it&#8217;s fairly easy to find online, and fix, with no outside help. A plus is that you can do almost anything in the terminal, and many help people tell you to use it. They just type to you a box with the code inside and you copy it into the terminal. The code is fairly easy to learn, in a ways. You can update, for example, Firefox, with the code _sudo apt-get firefox_ (sudo runs things as root, as there are not many system changes allowed when running as a normal account; you need an admin password to use sudo). And, as a plus, if you don&#8217;t have Firefox, it installs it with that command. But that&#8217;s for later. Also, many software or 20+-disk space required. Drivers will be needed to support some of your hardware. Windows supports lots of software, but to unlock most of your software, you may need to require a license or you have re except government or gaming software won&#8217;t work for Ubuntu systems. But almost all other software has good or even better alternatives.

     My hypothesis is that Linux is the fastest, Windows speed medium, and OS X and OS X tying in last. Microsoft has the most developers, so their work is pretty much pieced together, not always working in synchronization, which makes it have the most bugs and crashes. They have 89000 people working full-time there, so you don&#8217;t expect everyone to cooperate that well. Apple has 4600 people working, and most of them are working on improving their iDevices, so the people assigned to the OS X system have to really cooperate to actually make money off it (&#8220;Microsoft Cooperation&#8221;, Profile; &#8220;Apple Inc.&#8221;, Profile). There&#8217;s hundreds of Linux distributions, so I&#8217;ll just be talking about Ubuntu today. Many developers use Ubuntu, and they don&#8217;t have a large company, but they have a large community of people helping them solve problems, so there are a few bugs, almost never crashes, and when there are bugs, they can be resolved easily by doing some research. Much less research than when you have Windows or Mac bugs. 
 
     Windows is very pricey, at around $200+, but you can get Mac at around 20 bucks. But, most popular distributions of Linux are completely free due to the Open Source Initiative, meaning that you can do whatever you want to it. That's because of the GPL (GNU General Public Release).

     Windows is for mostly gamers and other people, as most games only support Windows. Mac is for anybody who plays around with media; ie. ITunes , Photoshop, iWeb, and many other media editing softwares. Linux is for Office workers, and anyone else who decides to use it. Many popular softwares also support Linux. It can be made for windows people (only sometimes) too, just use Wine [Windows Emulation (Emulation means virtual running)] for some games. Linux is for people who have low budgets, and can't afford to lose anything. Since a majority of people use and store things using Windows, hackers mainly make viruses for Windows. Mac and Linux are basically equally secure, but Linux is the on with the most derivatives, so it's hard to make a generic hack. On my account, Ubuntu is the best option, as I don't want or have time to, play games.
