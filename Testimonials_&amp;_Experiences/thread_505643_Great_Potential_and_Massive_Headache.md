### Post by newbiefromwindows on 2007-07-20
I installed Ubuntu yesterday. I installed it using Wubi which lets you install Ubuntu on your windows computer without having to partition your hard drive or create a boot disk. This is incredible! This is exactly what it will take to move windows users like me over to Ubuntu/Linux. The install went great. Ubuntu and Windows can both run, I choose which one I want when the computer starts up. I understand that Wubi is going to be apart of the next official Ubuntu release, which is a really good idea.

Ubuntu has many great features which have impressed me. The idea of having multiple work stations is incredible. I love this little toggle that moves me between different windows. It is like the tab in firefox, once you have it you can't live without it. I like that you can put a toolbar on any side (or all four sides) of the screen. Way to think inside and outside of the box! I like the way it is extremely easy to install many applications from the applications menu. However, now we have arrived at my Ubuntu is not desktop ready rant.

I read the criticismFAQ and one of the things it says is:

> 
The software is not easy to install/uninstall/update

Quite the opposite, the software installation process is extremely simple: it just isn't the same as Windows. In Ubuntu there is a list of most applications available, you simply click the ones you want and they are automatically downloaded, installed and configured for you. It does not get any simpler than this. Moreover the downloads are signed, which means they are safe. You do not need to fetch programs on the web, trust unknown sites, download unknown binaries, and execute setup.exe programs. To uninstall, you simply uncheck the programs from the list. For updates, you are notified when they become available and all applications are updated in one go. There is no need to go through each application individually and update it using special procedures.

In other words, all of your programs are 'centralised'. You have one package manager which knows about available software, it remembers which files were installed by which package, and this makes reinstallation, reconfiguration, and removal a simple task.

With other operating systems, when you download a program from a website, you are putting a high degree of trust in the provider that the program is not malicious, or has not been modified from the original file to make it malicious. The Ubuntu repository system (official websites from which all installation packages are automatically downloaded) addresses these security concerns: the packages are compiled from source code and are screened to make sure they are free of malicious files. Moreover, the downloads are signed, so that the files cannot be swapped when you download them. 

Now in many ways this is true. If a program is under the applications add/remove thing, it is easy to install. However, many programs are not under this, but instead must be found using Synaptic or another program like it. I have tried to download programs (or their packages) through synaptic, but once I do so I cannot find them. They don't show up in the applications menu and I can't find anything to launch them with. There is something wrong with a program that can't put itself in the applications menu/on the desktop with a shortcut. If I can't figure out how to open a program, I can't open a program. Period. Many helpful people have explained how to do this with command in the the directory. I'm still not really getting this. Someone told me that many programs are "command only" Command only?????

The whole idea of regularly using/ using command in the directory drives me crazy. Ubuntu is supposedly a desktop ready people friendly version of linux. In many ways it is, but this aspect just blows my mind. I have found that people are very helpful (even if their help doesn't produce results), but that what they almost invariably tell me to do involves messing with the directory/entering code etc. To call anything which uses command code directory whatever "desktop ready" is just insane. This aspect has driven me nuts. No average user wants to touch it with a ten foot pole. It is like telling a windows xp user they are going to have to use MS-DOS to get the computer to work at certain points. Microsoft would go out of business.

Sort of related but different is the issue of downloading software from the web. I agree that as this FAQ explains it is safer to set up the system as Ubuntu does. However, I want to be able to see a program online, click the download button and have it download. It should be possible to do this, even if it uses the repositories/ centralized process. One of the lines I find comical from this FAQ is
> 
It does not get any simpler than this... You do not need to... execute setup.exe programs. 


Now, I understand the advantage of the way ubuntu does it, but this is a joke (yes, it has been explained to me that that .exe isn't necessary and it takes up space, I am talking about the convenience  not the technical details). Every average user wants something they can click when they find a program online and presto, the program installs. Would it be possible for Ubuntu to do this using its current system but such that a website could put a download button on their page and when you clicked it would open the add/remove box pointing to the exact application you intend to download. Thus, it would still have the advantages described in the FAQ, but it would be just a click away on a web page.

There is a reason for this beyond my lazy self. It shows Windows users that the programs which they like are also available for download with Linux. Take BitTorrent for example. I use Azureus to download torrents. I never knew it was available for linux. Take a look at their website: [http://azureus.sourceforge.net/#](http://azureus.sourceforge.net/#) I don't see where it says you can get it for anything but windows. It just has a big download button for windows. If there were a smaller download button for linux, even if it just pointed you to where you could get it in the applications thing, it would show windows users that their favorite programs can also be easily downloaded for Linux.

Overall I am very impressed. I like free software. Period. I am hoping that future versions of Ubuntu are even better. For now I will keep both Ubuntu and windows on my computer as this offers me the best of both worlds. We'll see what I'll do when I get a new computer in a few years time.

Big Sigh!  When a linux person helps you to fix a problem or writes a howto for you to follow, why should they have to type out a 1000 word description when a simple one line command in a terminal does the same thing?  if it were not for the power of the command line in linux, some of the help and howto's on this site would take people weeks to write and read.  The command line in linux is one of its greatest strengths!

You're just use to doing everything the hard way in windows, like digging a swimming pool with a tea spoon.  Everything that can be done on the command line can be done with the gui, it's just not practical to do so.  Even the more advanced windows user wishes for greater power from the windows command tool and from what I understand, microsoft have to a degree addressed this in vista.

Windows users need to stop thinking everything has to be done the long and hard way or they'll never be comfortable in linux and that's fine with me to be honest, leave my command line alone, it's what makes me love linux the most!

Now as far as being able to install programs like in windows, linux can do that also, it's just the software writers prefer to do it the way they do, it's more efficient for them.  There is nothing stopping a programmer from writing a complex script to do all the hard work for you as in an msi file from the windows camp.  But they do this for free, most don't get paid for their charity so doing a little work yourself to install their software is a small price to pay for the freedom you enjoy from linux software.
If you don't like it that way, then that's fine, use a  commercial offering, no one's stopping you.  But please try to understand why some things are the way they are in linux, it's because we want them that way!
