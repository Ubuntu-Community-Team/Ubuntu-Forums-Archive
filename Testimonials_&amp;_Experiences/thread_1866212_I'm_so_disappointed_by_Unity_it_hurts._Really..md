### Post by idefixx on 2011-10-21
Unity doesn't work for me. Not even if I try really hard. 

Here's why:

I'm using nomachine to log into my Ubuntu server. I have the following problems with Unity:
- No way to change font size (*). It's way too big for me.
- The sidebar (called launcher?) disappears when windows are maximized. No way to change that behaviour (**). The sidebar reappears when the mouse is on the left edge of the screen but since I'm running Unity in a window I cannot easily position the mouse at the left edge of the Unity screen, only the host screen.
- I cannot change to an open but covered window. Really. If for example I have a terminal window open and it's covered by the browser, I have not found a way to bring the terminal window to front, using the mouse (only alt-tab works, but that doesn't count). 

Minor gripes:
- Menus are visible only when the mouse is hovering over the top panel. So when I have the mouse cursor in the application I'm actually working on I cannot see the menu for it. Really unhelpful.
- Menus are visible only when holding down the mouse button. This is a bug, I think since it is inconsistent.

Those are only the things I found in the short time I had with Unity until I was forced away to something else. 

You can argue that since I'm logging in via nomachine my concerns aren't really valid. Well that invalidates also virtual machine users and I really think it should not matter how I'm accessing my computer. Linux has and always will be a networked citizen. Why and how and where should be irrelevant.

I was really hoping for Ubuntu to finally take Linux to the desktop. I can't even use Unity and thus I can't recommend it to anyone else. That really hurts because Unity takes some hope away from me that Linux would finally make it to the desktop. Ubuntu is now a dead end for me and I really, really did not want that to happen.

Sincerely,
-- Henrik

(*) Only by installing gnome tweak tool. Completely unacceptable for such a basic feature. 
(**) I did try to change it by installing ccsm and configuring Unity but that didn't work due to bugs.

### Post by Larkspur on 2011-10-21
I've a couple of possible solutions to two of your issues:

> **idefixx said:**
> - No way to change font size (*). It's way too big for me.

While nowhere near as finely-grained as gnome-tweak, there's an option to reduce text size in Universal Access.

> **idefixx said:**
> - I cannot change to an open but covered window. Really. If for example I have a terminal window open and it's covered by the browser, I have not found a way to bring the terminal window to front, using the mouse (only alt-tab works, but that doesn't count).

Try double-clicking on the panel.  It should bring non-focused windows forward. Another way would be to pin the Terminal launcher to the bar in one of the first 10 spaces.  That way you can either click that to bring the window forward (once they fix that bug for your ccsm problem) or use super+[number].
