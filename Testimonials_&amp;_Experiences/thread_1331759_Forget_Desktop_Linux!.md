### Post by WickedAx on 2009-11-19
I've been an avid Linux enthusiast since Red Hat 5.1 (Manhattan). I went with Fedora once Red Hat went enterprise and then I moved onto Ubuntu starting with Edgy Eft. I always held out hope that things with Linux would get better so I never minded having to constantly screw with video drivers and wpa supplicant just so I could get the basic essentials functioning properly; however, that all changed with Karmic Koala.

I run Linux on my VPS and it's rock solid; however, as a desktop operating system it sucks! I thought Ubuntu had finally cracked this nut but I was wrong. With Intrepid Ibex (Ubuntu 8.10) the proprietary ATI drivers provided terrific hardware acceleration and the wifi drivers for Atheros chipsets worked out of the box. Along comes Karmic Koala (Ubuntu 9.10) and suddenly the wifi networking stack crashes every 5-10 minutes. Unloading/reloading the Atheros modules won't correct the problem so the only solution is a reboot. How is it with every successive release of Ubuntu they manage to screw up something that was functioning perfectly fine in a previous release? I understand they're trying to make changes to the underlying kernel structure to make the operating system more robust/stable but you can't keep breaking previous functionality if you want people to adopt your distro.

In addition, ATI's Catalyst 9.10 drivers no longer support the ATI FireGL v5250 card in my T60p (the laptop is from 2007...What gives ATI?) I understand this isn't the fault of Ubuntu but it's just another case of why Linux in general is too much trouble. What good is a desktop operating system that doesn't have reliable wifi and 3D acceleration?

I've spent years tinkering, modifying source, and recompiling kernels just to get basic functionality working and I'm sick of it. I had hoped Ubuntu would be the answer to making Linux an operating system that I could be productive with but it's obvious that's not the intent. I've been so impressed with Windows 7 that I've completely done away with my Linux partition and kissed desktop Linux goodbye.

In all fairness to Ubuntu and the other distributions, part of the problem with the lack of good drivers and hardware support results from the hardware manufacturers unwillingness to play ball with the Linux community. A lot of what's been done in terms of drivers support for Linux has been the result of painstaking reverse engineering of the hardware. While the community is starting to see increased support from manufacturers, there's still a lot of reverse engineering required to make things function. With that said, I'm sick of waiting for better hardware support.
