### Post by synapse13 on 2009-02-20
I am a computer user.  I know a bit more than the average user maybe, but not much.  I am not a developer or programmer, and frankly I've got to say that I admire those of you who have those skills.  What I am is just a normal computer user, one of the many, and I'd like to be an Ubuntu advocate to friends, family, and anyone who will listen.

Don't get me wrong.  I am already a minor Ubuntu supporter.  But at this point my enthusiasm is tempered by some negative experiences that I've had.  Here is the primary reason for my lack of full-on zealotry:

- Games. I have read the arguments... "games aren't such a big deal", or "just use Wine".  To me, the argument that games don't really matter is just disappointing.  What I'd like to see, as a user, is a system that "just works", or does so with only very little need for tinkering.  With Ubuntu, most things "just work", and they do so very well (in many cases better than Windows), but game support still lags far behind.  I find arguments that dismiss the gaming community disappointing because I think that the Ubuntu community at large wants a system that is grandma-ready.  What does that mean exactly?  Doesn't it really mean ready for the average user?  

For me, the sort-of-upper-edge-but-still-average-user, I would use a solution like virtualization, but from what I understand 3D acceleration and general game performance doesn't work in those setups.  Next up is Wine.  Wine is great, and is getting better, but doesn't support nearly enough games for me to completely turn my back on Windows.  That leaves me with dual-booting, and with only a half-hearted ability to sing the praises of Ubuntu to my friends and family.  Because really, as much as I dig Ubuntu my praise sounds hollow when they hear that I still have to boot Windows regularly.

I can tell my friends and family all about the things that I have seen that I believe are awesome (please note that none of these things are really necessary to have a working OS, but they are attention-getting and build excitement, just as expanded game support would): OpenOffice; fancy desktop visual effects that can be easily turned on through the GUI; readily installable graphics drivers (w/ easy dual-screen setup through Nvidia-settings!); multimedia options and ease of use (I love you, VLC!).  There are more, I just can't think of them now...

As I said to start off my post, I am not a developer or programmer.  I don't know what sort of work goes into making Ubuntu run like it does.  I've only been an Ubuntu user since Gutsy, and I've got to say that I am in awe that things have progressed as quickly and as well as they have.  I don't know how you do it.  All I can think is that it must be exceedingly difficult to build support for games, or it would have been done by now.  But if it can be done... please try to build Wine further or build up 3D gaming support.  

I love Ubuntu.  I want to love Ubuntu more, leave Windows behind completely, and share the love with those I know!

Peace.

### Post by 3rdalbum on 2009-02-21
There are already paid professionals working on Windows gaming on Linux, and many of the best open-source programmer/reverse-engineerers in the world are working on Wine.

The whole job would be much easier if Windows was not buggy and had a fully-documented API. Unfortunately, this is not the case, and Wine developers have to actually reimplement Windows bugs in order to get some things to run.

There are two solutions to games on Linux:

1. Buy a console for big-name games, and use Linux for original open-source games
2. Campaign for game development houses to port their games to Linux. Wine's own API is available for Windows, meaning that developers can write their programs using Winelib instead of Win32, and the resulting programs can be run flawlessly on Windows or any Wine platform (Linux, FreeBSD, Mac OS X etc).
