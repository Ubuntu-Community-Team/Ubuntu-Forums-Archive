### Post by Methuselah on 2009-08-10
Yoo have a better chance getting linux to work on a PPC based mac than windows.
Linux is embedded in phones and installed on Sony Playstations and Gamecubes and things you wouldn't imagine.
But..it doesn't work on some common computer harwdare.

So if it's not a matter of lack of flexibility, where does the problem lie?
Well, there are many different consumer peripherals mostly designed for windows based PCs and which are supported on linux by the linux community and not their manufacturers.
Understandably, every new fangled device might not be supported upon release given this state of affairs.

I consider it quite a miracle that with one liveCD and, in many cases manufacturer indifference, as much hardware works out of the box as we have it now.
However, I cannot stress enough that if one intends to run linux he'd better do some research on the planned HW configuration or you're dealing with a lottery.
There are websites that assist in performing this check.

Granted, this might seem like too much effort for some in which case there's always the option to buy linux pre-installed. 
One's judgement of linux is often skewed by the fact that one frequently installs linux himself but seldom *has* to do so for windows.

I still maintain that modern linuxes are not at all difficult *to use* but installation may often require too much from the casual user despite the fact that Ubuntu has one of the easiest OS installations in existence.
Then they might say linux is 'not ready' because they were instructed to compile/install some modules to get support for hardware not autodetected or edit xorg.conf to fix a display issue.
However, these are things they would not have to do repeatedly during ordinary use and which technicians would have already done had linux come on the machine like windows often does.

BTW, I don't think any SATA issue you're having is universal since I've been using SATA with (fake)RAID capable hardware since Hardy.

### Post by Methuselah on 2009-08-10
> 
It would be hard work.. but having a team working on Ndiswapper or tools around like that would surely be beneficial to all Linux distributions. Once perfected instead of putting large amounts of effort into re-writing thousands of drivers extra man power would be free to improve the distribution itself... 

Just an idea... Probably already been thought off but Vendors are not going to suddenly start writing for linux.. they are lazy.


ndiswrapper is not the future.
It's an expedient solution in some cases and I have used it in at least one case but it is fundamentally a kludgy workaround...as wine is (which explains why it didn't work for you).
Neither is useless, but both are essentially bandaids for a symptom.
The future is native apps and drivers not constantly increasing investment in imperfect usage of windows hand-me-downs.

For one, proper linux drivers are open source like the kernel.
This allows them to be legally distributed with the OS (which, for the end user, means supported hardware works 'out the box' without them having to hunt, download, and install drivers).
It also means devices can be supported as long as the community wants rather than until manufacturers decide to pull the plug on providing drivers.
Additionally, a kernel strewn with binary blobs meant for windows seems like a recipe for hard to fix instabilities since bugs could be in closed components (which are being used in ways not intended by the provider so we can't as them for help). 
It seems that the emergency stopgap that ndiswrapper represents is as far as we want to go down that route.

However, for almost every class of hardware (wireless adapaters, webcams etc.) there are multiple choices that work on linux with native open drivers or manufacturer supported closed ones.
The solution is to buy properly supported hardware if you want the flexibility of being able to run linux.
It is not to your advantage to buy hardware from people who do not care about your use case while you ignore others that are trying to please you.
Voting with your wallet has a higher change of effecting change than complaining here.

But, if it's not really important to you that your hardware runs linux properly, then it's no real loss when it doesn't, right?
The cost of downloading/burning an Ubuntu CD or ordering one from ShipIt is minimal so that's just how the cookie crumbles.
