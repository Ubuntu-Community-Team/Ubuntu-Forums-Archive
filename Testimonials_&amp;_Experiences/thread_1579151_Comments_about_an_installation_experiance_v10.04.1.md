### Post by anotherbigal on 2010-09-21
Hi all
 

 Well, not for the first time I have just installed Ubuntu on a customers machine. However, this is the first time that the customer has actually sat alongside to watch. So here are the observation passed to me. This came about as the customer has been a Windows desktop user for many years and is finally taking the plunge and wanted to start the practical, hands on, learning curve with the installation.
 

 During the installation process straight off of the 10.04.1 LTS CD there is a dialogue box on the screen that, initially allows input of the various user parameters; name, user name password, keyboard & etc. Once passed this stage installation continues. The box carries the process bar and gives little hints about what is going on. The process bar shows a percentage figure but that annoyed my customer. Especially latterly when it would climb to 75% then drop back to 72% then later on climb and drop back again.
 

 Another comment was that, as a first time user watching the install what are the various things that are happening. As an example when it got to a comment that said 'Running dpkg' the customer had no idea what that was. While I was able to explain it would be helpful to people if a little additional information was supplied. Something like 'Running dpkg – the software package manager'. Short but at least it gives a hint.
 

 The final part of the install runs out of advice while the various packages are being sorted out. The text above the process bar is headlined '**Getting help with Ubuntu**. There are two hints. Learn more at [COLOR=Magenta]ubuntu.com/support[/COLOR] and Let us know about your Ubuntu experiences at [COLOR=Magenta]ubuntu/community[/COLOR]. If you hover the mouse over them they appear to be links which of course cannot actually take you anywhere. What is more, neither addresses are complete so there is little point in writing them down for later use. From the new user point of view there is no point in having them. One additional comment was that during the installation of the language packs a 'Skip' box appears on the right. No idea what would be skipped if it was clicked, or why it was there at all.
 

 Once installation had completed we were told that the machine will restart. The message box does not say if we have to remove the disk and physically restart the computer, wait for it to restart itself or open the drive tray and take the disk out or whatever. No information. In actual fact, after a pause, the tray opens on its own to enable disk removal. What then happens is a screen full of errors. Although the numbers change they are all of the format[INDENT]  [3500.890769] end_request: I/O error, dev sr1, sector 519000
[/INDENT]Pressing Control C will restart the machine but of course, a new user would not know that. It not a good image of Ubuntu. What should actually happen is the disk should be left in the tray. Lack of clear, unambiguous information; if the disk should stay in the tray why open it at all?
 

 Once restarted however, we arrive at the desktop and the Update Manager auto starts. Great, apart from the fact that it auto minimises so it is not at all obvious that user input is required (password entry). The updated packages are downloaded and installation begins. Here is the next little hic-cup. I presume that it is a screen saver kicking in. All we see is a blank screen which, while something is going on, is worrying.  Lots of hard disk activity so we wait patiently. The blank screen persists, even after the HDD activity ceases. Moving the mouse reveals a input box that has a space for a password and four options to select from. None of them say anything useful. They are: Leave message, Switch user (there is only one at this stage), Cancel and Unlock. We presume that the latter is the one to click but 'Unlock'? Enter might be better. Anyway, why the second password entry. That was entered at the start. Anyway we are now presented with the 'Restart later' or 'Restart now' options. At least the message says that the restart is needed to complete the installation of the updates. All finally is well and we have a working desktop. But, from my customers point of view, not an easy experience.
 

 Well. That it for now. A long post I know but the installation did ask for user experiences so here it is. I am putting this on the forum as I have no idea where else to put it. If anyone want to advise me differently, please do (I worded that VERY carefully to avoid being told where to go :)).
 

 Cheers
 

 Alan
