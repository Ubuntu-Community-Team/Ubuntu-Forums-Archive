### Post by slakkie on 2009-10-20
Hello all, after 3/4 years it is time to write my experience of Ubuntu.

I started out with 5.10, which I quickly upgraded to 6.06 a month later or so, because 5.10 did not support WPA iirc, or at least, not for my wireless card at that moment. I did this on a box where Debian always gave me troubles with X. X has never been a friend of mine, Red Hat/FreeBSD all failed to get X working on other boxes I had owned. Mandrake saved the day for Linux distro's back then (btw, I know FreeBSD is not Linux!).

I installed it dual boot with Windows XP on my Dell laptop and was pretty happy about it. So I finally decided to remove XP and go fully Ubuntu. A pleasure it was.

At work I used FreeBSD on both of my boxes, one as a server install and the other as desktop install. When I had to replace my desktop machine I installed Ubuntu 6.06 on it (or maybe 6.10, don't know anymore). My problems at that time were that I couldn't get my Philips GoGear working ([http://ubuntuforums.org/showthread.php?t=303357](http://ubuntuforums.org/showthread.php?t=303357)) on any of my Ubuntu boxes.. Luckily my girlfriend had a Windows box so I could use my mp3 device. But boy, those MTP devices suck.

I also had a Compaq NC6XXX laptop from work, which ran 6.06/6.10 at that time. And it ran that version of Ubuntu for quite some years. I upgraded to 7.04 when 6.10 was EOL. But it was a great OS to use. I never had the urge to upgrade. 

In the mean time I requested a new PC at work to replace my server and I installed some Ubuntu version on it, 7.04 I think it was. I first planned on keeping FreeBSD, but lack of support for my gigabit ethernet card meant I couldn't use FreeBSD. That sucked, but since Ubuntu supported it out of the box, my problems were solved.

So now I was running Ubuntu at my work laptop, work desktop, work server and private laptop. And to make it more fun, all at different versions.
I don't know why I upgraded my 6.10 work laptop, maybe it was the booze... But I upgraded 6.10 to 7.04/7.10/8.04 in a day (including bug hunting because of an EOL upgrade). I started 1/2 am in the morning fell a sleep and booted 7.04 when I woke up and then continued the rest of the upgrade path. Somehow I always do upgrades in the middle of the night. When I did that upgrade, 8.10 came out not so long after, or was already released. 
This is when I started writing the EOLupgrades page on the wiki. Spent some days installing old versions and upgrading them. Fun to see all old gnome themes Ubuntu has made over the years.

I upgraded my desktop machine at work from 7.04 to 8.04 and then to 8.10 and boy o boy, that turned out to be a negative experience. There was some kind of Intel related bug that caused my X to crash/hang whatever, but it wasn't funny at all. Luckily, like real men do, I never created a backup of my old install.. I worked a week with a crippled system, not to interfere with my regular work and I reinstalled 8.04 on a Friday evening. Soon after I upgraded all machines to 8.04. 

On some afternoon I decided to upgrade my Hardy install of my work laptop (which had been replaced due to hardware failure) - a Dell Latitude D630 - to the latest Karmic release, alpha 2. That was a good ride. It was fun to see how many packages are upgraded on a daily basis, and the bug-hunting aspect is also very fun to do.
If you feel experienced enough I think everyone should give testing a development release a go. It is a fun ride and you'll learn some new things too. Although not being able to boot your machine when you really really need it is not fun (but I have learned from my mistakes, I have a backup on DVD which I can pop in restore to and use a working OS - Hardy/Karmic). Thank you clonezilla (which is both Debian based and Ubuntu based)!

But then the Beta period of Karmic started.. Since packages are not touched anymore, you won't see big changes etc etc I decided to have a look at our big brother, Debian. I already run Debian (stable) on a rented server, which runs just fine, not having to worry about it. But I wanted to see how Debian would be as a desktop OS.
Lenny got installed on my laptop (no problems with X this time!!), no support for my wifi card, but once connected to the web via a wired interface and enabling all of the repositories I could find (basically copying the sources.list from my server) I was able to quickly install the correct firmware. And now I'm running Debian testing. Like expected, it is the same as Ubuntu. What really surprised me, it is a lot quicker, a lot. I boot my machine and now I have to wait a while before my wireless network comes up. With Ubuntu that was never the case. I see my network coming up in conky (and I can hear it too, since the beeps are there, which I don't have with Ubuntu - my setup if you are interested can be found [here](https://help.ubuntu.com/community/NetworkConfigurationCommandLine/Automatic#Configure%20laptops%20for%20multiple%20locations)). Normally I booted, login to KDE and I was already connected, but with Debian, I'm logged in and then I'm connected via wifi.

Currently I'm having a discussion with myself, should I make the switch to Debian or do I keep at least one Ubuntu machine? Should I run Ubuntu LTS or use Debian stable/testing. Wait for 10.04 (well, if I wait for it, I will certainly run all the alpha's and beta's) or keep running Debian testing/unstable? I really like Ubuntu, it has shown me the power of Debian. But now that Debian is able to show me the power directly I wonder what I should do...

I guess this experience topic is not only experience, but also a testimonial to both Debian and Ubuntu. Without Ubuntu I guess I wouldn't use Debian and without Debian I would never ever have used Ubuntu.

### Post by slakkie on 2009-10-24
> **martrn said:**

For me, Debian compared to Ubuntu is the same, till a certain degree. With Debian you need to install more after the initial install. Proprietary stuff which is included in Ubuntu you will not have on a default Debian installation. 

The release cycles differs. Debian testing/unstable are so called rolling releases, where as Ubuntu isn't a rolling release, Ubuntu can be compared to Debian stable, only with more up2date packages.  

You can mix and match packages from Ubuntu in Debian. You can do it the other way around too if you want. I use some Ubuntu packages on my Debian testing system. 

Regarding Red Hat, I want to give it a try once Rawhide goes stable (Fedora 12). I've installed it during the Karmic alpha to confirm a bug and it looked good. One small issue with the Fedora 11 installer, which is supposed to be fixed in Fedora 12. 
I booted in Gnome whereas I normally run KDE, but yum works perfectly, like apt. It is a bit slower, but I understood that is because it does some magic under the hood, which apt doesn't do (although some of that magic might be a bit overkill for my situation). But anyways, Fedora looks like a good OS too. The upgrade from Fedora 11 to Rawhide went super smooth, although it took longer then with Ubuntu.
