### Post by thedragon4453 on 2007-04-30
Well, I've been using ubuntu for about 3 weeks, and only have a few complaints:

1. Ipod support. 3 apps to get the features of itunes (which isnt my first choice even on windows) kind of sucks. And that is with some tweaking in between.

2. Installation. I realize that ubuntu cant control how apps are packaged, but one of the biggest difficulties for me switching to ubuntu has been the lack of a standardized installer wizard. the add/remove programs function is great, but to install a simple dock not listed, you have to download, make sure i have like 6 dependencies installed first, drop to a command line, and pray. i have not had to use a command line since windows 95. Maybe ive been spoiled by simply having to spam the enter key to "next" until the install is complete.

3. the command line in general. I would really like to start recommending ubuntu to friends and family, but i think the first time i tell them they will have to use the command line, and a language that will appear to be gibberish to them, they'll laugh, and ill be reinstalling xp. I realize the benefits of the command line, but i dont think it should be required for anything at this point.

other than that, its been great. ubuntu worked with all my hardware out of the box. i had a little problem with beryl at first, but that was only because i was doing it the hard way. tweaking config files, and using the command line a ton at the instruction of various wiki's and forums. oh, also in upcoming releases, i would make beryl the default. that is really going to win users over from windows. i think that i will be happy with ubuntu, because i will be patient with it, but for the average user, i dont think its quite ready yet. which is actually more of a problem with linux in general.

### Post by zjameria on 2007-04-30
I have been a 'loyal' user of XP and MS for quite a while and been very happy too. It took care of most of the stuff.  Finally, I decided to give Linux a shot. I tried couple of distros including Redhat, LinuxExpress and some other one, I forgot the name of. Then finally I came across Ubuntu and was impressed by the support and forums. So I switched to Ubuntu. 

So far, I have been very impressed with Ubuntu. It worked smoothly on my celeron laptop, which was very slow with XP before.  However, one thing I miss about Windows is hardware support at tip of the fingers.  Its so much easier to get any hardware installed and working in XP. I have had considerable trouble getting things in order with Ubuntu. After spending hours on Wireless and widescreen resolution, I still cant get sound to work. Working with CL is a little frustrating, but I am getting used to it.

I guess a lot of the hardware troubles can be alleviated if the best suitable drivers can be automatically installed and configured.

### Post by hartz on 2007-05-01
> **zjameria said:**
> 
However, one thing I miss about Windows is hardware support at tip of the fingers.  Its so much easier to get any hardware installed and working in XP.

I feel your pain zjameria.  The answer to why the hardware support is in the state it is, is all about money.

The hardware vendors are businesses out to make money, not out to lose money. So unfortunately, with the relatively small size of the Linux community and the rapidly changing kernel scene, it just cannot be justify by many of these vendors to spend the time and money to develop and support drivers for Linux.  (Fortunately nVidia has got an active Linux driver development)

Sure, they can write a driver and post it on the web, but then they would need to support it.

So rather than open themselves up to difficult support and possibly even legal issues, they avoid it.  The result is that the Linux team turns out to be responsible for writing and testing drivers and then making them part of the Linux core code.

This team is mostly made up of volunteers who figure out how to write the drivers in their spare time, often by reverse-engineering the Windows drivers to figure out how the hardware work.

If the Linux kernel team (read Linus) would stabilize the device-driver interface, it would make it easier (or even possible) for hardware vendors to distribute binary, even closed-source drivers on the CD with the hardware, as it would not be Linux kernel version specific.  This would take away much of the load of the kernel developers because they would not need to test and include every single driver any more, and because the driver interface is standardized, it would make it easier for hardware vendors to support.
