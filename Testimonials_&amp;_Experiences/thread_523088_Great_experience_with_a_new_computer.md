### Post by Old *ix Geek on 2007-08-11
I recently bought an HP dv6000 laptop, and went through hell getting its wireless connection working properly.  There were other issues, too, and when it was all said and done I'd spent a good two weeks tweaking the system to my liking.  So yesterday, when I bought a Compaq desktop, I suspected I'd be in for a similar ordeal.

For one thing, I had never connected a Linux desktop computer via wireless before, so I was expecting problems with that.  Also, I've never had a flat-screen monitor before, but I bought one yesterday, too--and it's some brand I've never even HEARD of (Hanns-G).  When I opened its package there was a CD with prominent instructions to use it to configure the monitor...but, of course, it was for windoze.

The first thing I did (after putting in a wireless card) was boot up under windoze vista.  I wanted to verify that everything worked in terms of hardware--the monitor, the wireless connection, etc.  Then I used its disk tool to shrink the partition.  (If you're wondering why I set it up to dual boot with 'doze, it's because I CANNOT get Paint Shop Pro 8 working CORRECTLY using wine, and I need it for my business. Other than that, I NEVER boot 'doze on any of my computers.)  By the way, vista NEEDED the wireless card's CD and the monitor's CD.

Next I booted up with a Feisty CD, verified that everything would work (I did get a warning message about proprietary drivers that are not supported), and then proceeded with the install.  It was smooth to the Nth degree (except for one glitch trying to set up swap space--I'll post that on one of the help forums).

When I booted the new installation, EVERYTHING worked.  Just like that.  The monitor worked like a charm...no CD required.  Wireless?  No problem...and no CD.  My USB trackball?  Recognized straight off.  And so on.

To say I'm thrilled about how easy this was is an understatement! :lolflag:
