### Post by 2CV67 on 2012-01-28
In spite of my “respectable” age, I am all for improvements. 
Show me a better way & I happily jump at it. 

So when I recently upgraded to 11.10 & found myself with Unity, I was certainly disconcerted because nothing was where I expected it, but I was interested to see how it works. 

After about a week of pretty-full-time work with Unity, I have gone back, thankfully, to Gnome Classic. 

I cannot find a single advantage, for me, in Unity.

I should perhaps mention that I use a desktop PC with 17" 5x4 screen, so not short of vertical space. 
Also I only have 512M RAM & other vintage parts which mean only being able to use 2D Unity. 

What was good about Unity for me? 
1. The "Windows key" to show/hide Launcher & Dash. (rather like in Windows7...)
2. Search function available in Dash.
Nothing else. 

What was wrong with it? 
1. Sure, I did find I could add Application Launchers to The Launcher OK & could easily remove unwanted items, but the Launcher became very long so I was always scrolling up & down for what I wanted. 
2. All the instructions said you could re-order the launcher icons by dragging to the right then dropping back, but that just didn't work - I found I had to hold the icons down until they "dropped" then could slide them up & down OK. That method is very good, but needs instructions... 
3. To see more icons & avoid/reduce scrolling, I wanted to reduce icon size, but it is impossible in 2D (yes, I know it is OK in 3D). 
4. Finding anything (not on launcher) in the Dash was a pain. Open Dash > Think what item is called > start typing > select item. GUI Ergonomy = zero.
5. Impossible to pin FILE-Launchers in Launcher or Dash. 
6. Impossible to pin anything (File or App Launchers) to top panel. 
7. Need to drag windows to top panel (& preferably top left corner) to see & use menus. 
8. Can't see which windows are active – they are all grey in Radiance theme. 
9. Lousy choice of obviously-available themes. - I had to install & struggle with Gnome-Tweak-Tool to get a miserable extra selection & then could only see them one at a time as a test. Out of curiosity, I looked back to 10.04 where I simply go Preferences > Appearance to see a whole range of themes laid out for comparison. What kind of progress is this?
10. Need 3 clicks to shut down, and need to read titles on identical-looking buttons on the way. 
11. Need to Alt+Tab between applications. I need to set up different key combination for that, for when I use mouse with left hand. Can't have both combinations active simultaneously?
12. Sometimes need Alt + Right-Click to show context menus. No warning or explanation?

How should it have been done? (IMHO of course & thinking “desktop”) 
My vision:
1. You shouldn't NEED to remember anything, like key combinations or application names. GUI means choosing from what is offered. That does not stop you using key combinations etc if you want & where it is more efficient. 
2. Option to pin launchers to top & bottom panels for instant 1-click access at any time. I currently have, on Top Panel: Hide button; Stop button; Applications menu; Places menu; Firefox; Thunderbird; Picasa; Calculator; Text editor; LibO Calc; Draw; 2 regularly used files; Terminal; Root terminal; Root Nautilus; Synapt; Grsync; iScan; Shutter; System Monitor; Firetray. All in space wasted in Unity. On Bottom Panel I have: Show-Desktop. 
3. Windows key shows/hides Dash. (OK - they got that right...) 
4. Dash size can be set as I want. (OK?)
5. Dash includes Search. (OK) 
6. Dash can include any Application Launcher I want. 
7. Dash can include any Document/File/Folder launcher I want. 
8. Dash can include any Browser Bookmark I want. 
9. Dash can show Applications/Places/Preferences/Administration Menus if I want. 
10. Can arrange all the icons in Dash in any order I want, in fenced-off groups if I want. 
11. All windows can include attached Menu bars if I want. 
12. Can have a Bottom Panel with selectable open-application tabs if I want (avoids Alt+Tab). 
13. Can have 1-click button to Show/Hide Desktop on Bottom Panel &/or Top Panel. 
14. Windows can have traditional elevator bar if I want. 
15. Active windows are clearly indicated (colour). 
16. Can have Red Stop button on top panel at left (all window shut-downs are top left now) & again in Dash at top left. 
17. Stop button leads to clear dialog box, displayed at top left of screen, with big coloured buttons (Oxygen icons for system-shutdown etc) for Stop (red), Logout (blue), Reboot (yellow), Switch User (green) in that order. I can add buttons for Hibernate or whatever if I want & customize everything by right clicking this window. Shut-down is just clicking on 2 big red buttons close together. 
18. Right-click is enough for any context menu – don't need Alt+Right-Click.
19. Built-in GUI version of Autotrash to automate Trash Management. In System Settings.
20. Built-in GUI setting for default papersize. In System Settings.
21. Good selection of themes which can be seen together on one page & compared in standard format.
22. Good Unread-Mail Indicator – like the excellent, if ugly, FireTray one for FireFox. The 11.10 thing is not visible enough, does not tell you how many mails (so once you have had one you don't know if there are any more) & it does not extinguish reliably.
23. “Add Printer” button in System Settings > Printers.
24. “Users & groups” in System Settings.
25. Clear indication that double-click has “taken” - especially for launching slow LibO.
26. I can Shut-Down in an emergency with the universal Ctrl+Alt+Del combo.

Of course, you may, and probably do, have quite different ideas!
Let's see them...

Good Luck to Ubuntu – I really want it to succeed!
