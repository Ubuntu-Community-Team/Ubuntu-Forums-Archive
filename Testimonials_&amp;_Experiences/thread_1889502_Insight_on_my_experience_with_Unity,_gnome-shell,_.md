### Post by kio_http on 2011-12-01
First of all let me first state that I have always been a KDE user and  have never installed any GUI linux system with other environments other  than for testing it.

My test system that I used is actually an Intel Atom N280 based netbook  with 2 GB Ram and a GMA 950. Since Windows Vista Ultimate SP2  (I did not want to waste my vista license) performs excellently on this  machine, I assume that an environment that feels slow on the netbook has  performance issues itself, after all people hated Vista so much so if it performs well, everything else should also work.

I was waiting for the final release of Mint 12 to compare these  Desktop's. Also since I am a KDE user, I will try not to be biased but  you may find that I tend to compare things with KDE.

Note: the screen-shots were found on the web.

**Gnome 2**
[IMG]http://lh4.ggpht.com/_1QSDkzYY2vc/S6Kdc-BC7nI/AAAAAAAAAng/sEv6YQdl9_Q/SS_Selection_007.png[/IMG]
I've used Gnome 2 in Ubuntu 10.04 and I found it a bit minimalistic. I  am not a huge fan of Menu's being the primary way to launch applications  and configure the interface. The lack of a proper way to search for  things disappointed me here. Although I normally do not like dark  themes, I was pleasantly surprised by the fact that Ubuntu 10.04 was not  too irritating to the eyes. Another thing to dislike here was the  excess use of vertical screen space; it was not comfortable to use  Firefox on a netbook. While I am aware that netbook interfaces exist, I  prefer the desktop UI on a netbook. There is one thing in particular  that I admired here though; the ease of changing a chat status from the  top panel. Performance was quite good on my netbook and certainly better  than the version of KDE that came with Kubuntu 10.04 at the time.

 [B]Unity with Gnome 3 in Ubuntu 11.10
  [IMG]http://www.picamatic.com/show/2011/12/02/09/03/8056398_800x469.jpg[/IMG]
  
[/B]Note: I had tried Unity with gnome 2 in 11.04 and this is definitely an improvement over it.

Despite all the negativity about Unity, I actually found the design  to be quite ingenious and space reducing. I liked the way the lenses  worked especially the categorization in the app launcher and the ability  to access music. However there were some disappointments. Unity offered  me extremely poor customization options. There was no (intuitive method  not hacks) way to change the dock to the bottom and very few settings  to tinker with. While Ubuntu 11.10 with Unity consumed less RAM than  Kubuntu 11.10, it was less responsive on the netbook than KDE; typing  text in the Unity search could be slightly slow but it was bearable.  However GTK3 applications like the software center gave me really poor  performance; the UI took time to update itself, e.g the button hover  effect would come late and scrolling would lag and menus would appear late.

There seems to be a minor issue with the blur on the Unity Dash for me, if there is text behind the dash, the text behind appears corrupted i.e blobs of orange can be seen over the text. I've had this exact issue with a fake aero emulation program for Windows XP. I do not have this issue with KDE's blur or even the blur in Windows Vista and Windows 8 Developer Preview.

It is interesting to note that thanks to the performance improvements  KDE has been seeing recently, my KDE 4.7.3 desktop can look like this  and still perform almost as well as my high end Core i7 system on the  netbook. KDE responds to my typing and clicks instantly although Dolphin  takes time to load folders with thousands of files. (Dolphin 2.0  releasing in Jan 2012 is supposed to address this)
[IMG]http://www.picamatic.com/show/2011/12/02/09/03/8056399_800x469.jpg[/IMG]

**Gnome-shell with Gnome 3 in Ubuntu 11.10**

[IMG]http://www.picamatic.com/show/2011/12/02/09/03/8056400_800x469.jpg[/IMG]
I installed gnome-shell on ubuntu 11.10 and logged in to it. The default  Ubuntu theme and wallpaper looked terrible in gnome-shell so I changed  it to the gnome defaults (although this made Unity look terrible as they  share settings) Gnome-shell looks extremely minimal to me. Its not  exactly my favourite UI. It looks as if the Gnome team have attended to  make as little customization as possible to reduce the amount of code to  be written. Instead they let other do the work by creating "shell  extensions". Surprisingly here on the netbook GTK3 apps would run better  than in Unity (although still slower than KDE) but the gnome-shell itself was slow in some respects; searching in here was much slower than  Unity. Also if I click the battery notification and move the mouse to  the accessibility icon, the transition between the menus is quite slow. I  did like the way the chat system in Gnome 3 shell was. (I am waiting  for KDE to implement something similar called kde-telepathy and  discontinue kopete). The iPhone like on/off settings was also pleasant  to use.  There was nothing "advanced" in the gnome tweak tool which supposedly provides advanced tweaks.

It is to be noted that whenever Gnome 3, whether Unity or shell were slow  at something, my CPU usage would reach 80% which is quite high.  Although both idle with less RAM usage than KDE and have 2% CPU usage  idle like KDE, they both experience a rather high CPU usage when you are  using the system.

[B]Gnome 3 shell with LMGSE on Linux Mint 12
[/B][IMG]http://www.picamatic.com/show/2011/12/02/09/04/8056402_800x504.jpg[/IMG]

Given all the hype about Linux mint 12 recently, I downloaded the distro and decided to give it a shot. I did not bother testing Mate as I was interested in there implementation of Gnome-shell. I actually enjoyed what they has done. Overall Linux Mint 12 experiences the same performance problems within gnome-shell but the bottom panel was very responsive (except for the menu search). The GTK3 apps were slightly faster than they had been on Ubuntu with gnome-shell (maybe its just imagination) except that every time I clicked a menu there would be a delay of a second before the menu was visible. The software installer in mint was much more usable than the Ubuntu Software Center although it had a more basic UI. Again Linux Mint 12 like Gnome 2 was not good for vertical screen space although the ability to view currently open tasks was a big boon thanks to the shell extensions. When Mint 12 KDE is out, I will definitely give it a shot.
[B]
LXDE on Lubuntu 11.10
[IMG]http://www.picamatic.com/show/2011/12/02/09/04/8056401_800x500.jpg[/IMG]
[/B]I have lubuntu-desktop installed on my kubuntu system, on the netbook however since it is intended to work on old hardware, I installed it on my Pentium 3 600 Mhz with 768 MB RAM using an alternate installer. Lubuntu looked quite sleek and was able to give me a much better performance than any of the Gnome 3 based OS's / DE's for everyday tasks. My Pentium 3 system felt like it was brand new.  All hardware was detected and the system was very stable. While LXDE lacks customization as well compared to KDE, it is understandable considering the extremely low resources that it demands. It would be interesting to see what happens when LXDE goes GTK3.

I might add an Xubuntu review later if I feel like it. Concerning Xubuntu, I last tried it with version 7.10 and it was very similar to Gnome 2 but I preferred its defaults to Ubuntu 7.10. It would be interesting to see what it is today.

**Note concerning KDE**
Concerning KDE, I agree that it was not the best performing or friendly environment when KDE 4 was just released. From KDE 4.0 to 4.5 the primary focus was enhancing functionality. Since then the goal has changed to improve performance and reliability.  Martin Gräßlin (that's graesslin) has been doing a lot of work regarding kwin performance recently [E.g]("http://blog.martin-graesslin.com/blog/2011/07/running-kwin-with-opengl-es-2-0/") KDE 4.8 (release January 2012) plans to be the fastest release of KDE ever (including KDE 3), so it might be worth giving it a shot when it is out.

Since Gnome 3 is relatively new, I assume that it should see performance improvements over time like KDE 4 has.

I hope you enjoyed my short review!
