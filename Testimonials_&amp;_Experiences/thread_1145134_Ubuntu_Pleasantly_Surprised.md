### Post by blackened on 2009-05-01
> **SoftPops said:**
> The next step is to go for the upgrade to 9.04. Only problem is that when I kicked this off last weekend it told me it was going to take 7 hours to download. But thats my problem, if I must live in the sticks with a v. poor broadband connection!

Is this because of your connection or the download server?  The default archive servers are slow as molasses on all my machines, so one of the first things I do is go to System -> Administration -> Software Sources, hit the "Download from:" dropdown menu and select "Other...", then in top-right corner of the subsequent dialog press the "Select Best Server" button. It will test all the servers and find the one with the best connection to you. When it finishes, press "Choose Server" then close the Software Sources window. It will ask you to reload software and update information, so hit the "Reload" button.

After that, you can try the update process as normal.

Granted, if your problem really *is* your connection, then feel free to ignore me entirely. :P

### Post by 3rdalbum on 2009-05-04
You could do what I did, and download and burn the 9.04 Alternate CD. This handy disc can install Ubuntu in many exotic ways, but most importantly it contains all the default Ubuntu packages in their normal form. If you put it in while running 8.10, you'll get a dialog box asking if you want to use the disc to start an upgrade. Then as many packages as possible will be pulled from the disc, and any that aren't on the disc will be downloaded simultaneously.

This is incredibly good if your ISP has a free mirror for the CD images but not for the distribution's packages.
