### Post by zsugiart on 2009-05-23
Edit: The following honest opinion is expressed in the context of using Linux Ubuntu as a workstation OS. As a server, I wouldn't even touch Windows unless I'm under some serious influence of alcoholic beverages.

Nope it is not the best OS ever (yet?). Pardon me but I would like to share my honest opinion about the OS. Hopefully it can be viewed in a positive light:

[LIST]
[*]**It lacks killer apps**. 

Again, not ubuntu's fault, Linux have so many alternative apps but very very few can rarely be classified as a 'killer app' and most of them still can not compare to their counterparts from other OS. 

The only thing where Linux is a complete winner is in compiz/bling department. And its bling is sort of the poster boy that draws many new user to try out Linux/Ubuntu. But will it be enough to retain them? works for me, but again, not everybody's in it for the bling.


[*]**Production-quality apps generally lacks maturity in terms of usability concept**. 

My experience so far have always included some sort of 'magic installation steps' derived from scouring the net. Yes this also means that Ubuntu/Linux is very net-dependent. 

Quite a number of stuffs in Linux can be installed/upgraded/configured intuitively without needing to ask for help in Forums or look for guides on the net.

Think: Windows XP never ask users to edit any file from C:\Windows\System32. What would happen if 40% of 'advanced application usage and configuration' requires average joes having to edit numerous files deep in C:\Windows\System32\* (for whaetver reason) with a risk of crashing explorer.exe if they stuff up (which they will). Windows XP may be ancient, but they definitely get this bit right. Proof? my mom can drive the OS with very little help from me, and I get very little 'support call' in the past few years. I doubt the same can be said with Linux/Ubuntu.


[*]**Despite Ubuntu (and most Linux distros) having very rapid iteration cycle, there is no easy way to migrate from one version of OS installation to another**. 

Yes this is not easy, but I've spend 2 weeks configuring my Ubuntu to do it just the way I like it. 

Now, here comes the latest and greatest and I have to spend yet another 2 weeks to configure it? Not to mention I might have to re apt-get all of those packages.

This is not such a big problem for Windows because their iteration cycle are very long. But for Linux distros where the release cycle is quite often, tis a problem that must be solved. Yes I've been following Ubuntu since 6.*. Upgrades has never been easy and here we are in 8.* going to 9.* and just look at the posts above.


[*]**lacks industrial support, hardware integration does not all work 'seamlessly'**, 

Even when you compare the latest stable Ubuntu with the now 'ancient' Windows XP. Not Ubuntu's fault, but that is the after-effect we all must live with under Ubuntu and other Linux distro. 
[/LIST]


-- sorry for the willy nilly and long winded post --

But innn conclusion, I reckon what Linux community lacks the most is a unified, concentrated and coordinated effort in advancing the OS' capability. Look at how many ALTERNATIVE distros, window manager, applications (mind you, even for different windows manager) we have. Even within the Ubuntu banner, we have Ubuntu, Xubuntu, Kubuntu, and so on.

if all that effort have been spent in developing half of the distros, wndow managers, applications, etc we have in Linux, then surely each of those distros, window managers, applications, etcs will be much more awesomer.

That being said, I will testify that Linux Ubuntu is so far my favorite Linux distro to date out of the ones I've tried so far. Mind you, I have been using Linux quite extensively just to see what it's like, I have not booted my Windows XP since around September last year. I've used it extensively for work and non-work use. So yep, this is my way of giving back: by providing my most honest feedback. 

Linux have a very big advantage for now: that it is free of charge. But think again, how many 'free' copies of Windows XP are available on the net today? For corporate use, how many businesses are a 'microsoft partner' as opposed to a 'Linux supported'. The 'free' angle is not as effective as it appears to be, thanks to Software Piracy.

Now, with Microsoft and mac locked in an arms race to produce more and more killer apps, killer device, software in both PC and mobile realm, I wonder how Linux community's disparate approach will keep up?
