### Post by rliegh on 2010-04-18
> **p0cky84 said:**
> Why is it that every *bsd fans is so focused on making "tux" look bad?
History.

A lot of BSD users feel that they were unfairly hamstrung by the [USL vs BSDi lawsuit](http://en.wikipedia.org/wiki/USL_v._BSDi) and that Linux got it's mindshare through luck and marketing instead of any technical merit.

Also, there's resentment that Linux devs will sometimes agree to sign NDAs instead of pressing harder for techical specifications that would allow for other OSes to benefit from Linux's device drivers.

This resentment (sometimes) comes out in artwork.

### Post by rliegh on 2010-04-20
> **GodLikeCreature said:**
> 
I have not used BSD myself, so would be interested in learning more about it from an experienced user, and you seem to be just that.  Can you please go on about the reasons why you use and like BSD?  Do you agree with those comments about the limited number of applications available?  If so, how do you work around it?

Thanks!
I don't do a whole lot professionally or anything, so to me running FreeBSD is on a par with running Ubuntu -just a slight bit rougher around the edges (no auto-mounting of ntfs partitions in gnome, for instance).

I go back and forth, one day I'll use windows, another day I'll use ubuntu, another BSD. What I like about BSD is that if you're so inclined it's *very easy* to build the entire system -every thing is laid out in /usr/src (or /usr/ports once you get outside of the main OS -that's where stuff like gnome comes from). That makes it very easy to optimize. Also, to me file operations feel a lot faster in BSD than they do in Linux. 

If you know about gentoo, they got the idea from FreeBSD (portage was inspired by ports). They refined the ports and optimization idea and brought it to Linux. 

There isn't a huge lack of applications -as a desktop user anything that runs on Linux generally will have a BSD port (and if it doesn't, there's a compatibility layer that allows you to run Linux apps on FreeBSD).

The main problem, really, is device drivers. Nvidia's ported their graphics drivers to FreeBSD -but not OpenBSD or NetBSD. Wireless networking is a struggle because most vendors only deliver binary precompiled blobs that the BSDs can't work with. 

Where it works, it works like a champ -but Linux handles a lot more devices than BSD and Ubuntu's desktop is a lot more polished than any BSD desktop (and -imo- other Linuces). 

If you have a spare computer, you might try out PCBSD -it's based on FreeBSD but it's got a KDE desktop and it's designed to be easier for desktop users. 

I'm sticking with Ubuntu for my desktop, mostly, but if anything interesting comes out with FreeBSD I'll probably install it -and when I want a server I'll set up a virtual machine with OpenBSD on it (because OpenBSD is the most secure of the BSDs). As far as what I do with it, I usually will use it the same way as I use Ubuntu; as a desktop where I play with virtualbox and photo editing.


I hope I answered your question :guitar:
