### Post by 3Miro on 2010-12-04
Linux is not "behind" in tech, it is just different. Linux is build bottom up, meaning most of the innovations are hidden under the hood. Windows and windows apps in general develop top to bottom, killer interface first, then try to boot strap functionality behind.

For the drivers, it is good to note that windows is the only system that doesn't provide any drivers. Manufacturers make drivers for Windows, while Linux makes drivers for itself (Apple also make their own drivers, but they have a very limited selection of hardware). To get the best experience with Linux, you need to do a bit of research to get compatible hardware; also once something works, you will not suffer from such powerful regressions like in the case of XP to Vista.

If your printer runs only under XP, then you can consider Windows XP under Virtual Box.

### Post by 3Miro on 2010-12-08
Ubuntu uses a lot of technology from Debian, but Ubuntu and Debian releases are not in "sync", each does whatever the corresponding devs want. If you want LTS, but you need a newer version of ALSA, you can try an unofficial PPA. There are many community based PPA that provide additional and more up-to-date packages for any release of Ubuntu. The PPA will only upgrade ALSA and keep the rest of the system as is.
