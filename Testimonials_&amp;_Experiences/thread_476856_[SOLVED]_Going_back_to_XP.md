### Post by gcos7 on 2007-06-18
I know this is off topic, but there have been a couple of replies here that need addressing.

I am an ex-systems programmer and ex-systems administrator.  Unix, and its' off shoot Linux, were always an "engineers" operating systems.  The "Linux mentality" mentioned, about not posting some things here, is THE reason Linux still has not progressed to be usable by the "average" user.   By "average" user, I do not mean what has always been the Unix/Linux average user - someone with a technical background, familiar with operating systems, and thinking as a systems programmer.  Instead, I mean people like my friends, parents, etc., that don't know squat about operating systems and don't want to.  All they want is to be able to just run their stuff.  By saying that those users need to just go back to Windows, you are shooting your own project - making a free operating system for the masses - in the foot.  There are many comments about using "x" in Linux in place of "y" in Windows.  The average Joe doesn't want to learn "x" - he wants to us something that functions and looks EXACTLY like "y".  The LInux community passes the buck off to manufacturers and software companies, when in fact they need to start developing these things themselves.  Creating, maintaining and fixing an open source operating system is a wonderful thing, but so much is concentrated on the engineering aspects that the average Joe user is left behind.  How can that user choose Linux, as the Linux community wants, when the response has not been to cater to the user, but to cater to the operating system for the sake of the operating system.

Instead of insulting perfectly honest people asking perfectly honest questions, how about being like a lot of the responders I have seen on this forum - knowledgeable and willing to share it with those not as knowledgeable.  I have seen some replies on this forum that take a person hand in hand through the steps they need to solve their problem.  These responders are greatly appreciated.

I know from my years of tech support what kinds of questions, etc., come around that to someone knowledgeable seem ridiculous, but you need to remember when you knew nothing (and that day really did exist).  Not everyone has the ability, or the desire to learn technical things, to fix their problems or they wouldn't post their questions here.  I have posted many questions here that probably seem ridiculous to some, but they are honest questions on my part.

The idea is to be kind and willingly share knowledge, and to build applications, not just an operating system, that meet the needs of the average Joe.  Until then, even with as much progress as Ubuntu has made in make Linux "friendly", Linux will remain an engineers' operating system and will be used mainly by people running servers.  Since the push has seemed to be to make Linux an alternative operating system choice for the average Joe, Linux as a whole, and a lot of the "old guard" engineering types who do such a great job with the operating system, seem unwilling to really make it a replacement for the masses.

### Post by Nekiruhs on 2007-06-21
I from experience can recommend [System Rescue CD]("http://www.sysresccd.org"). I have used it to back up ext3 and NTFS partitions. Lemme show you some instructions so you understand what you have to do.
1. Boot the Live CD, it boots to CLI.
2. Mount the drive/partition you want to save your backups to eg ( mkdir /mnt/yyy && mount -t ext3 /dev/xxx /mnt/yyy) that makes a folder in /mnt called yyy, mounts an ext3 partition (you have to replace xx with hda, sda, hdb etc.) xxx at location yyy.
3. Now here comes the easy part. type partimage and watch as the GUI (ncurses) comes up.
4. Select the partition you want to back up and in the name spot enter /mnt/yyy/NAMEFORFILE to put the backup on your backup drive/partition. You probably want to back up your MBR too. 
5. Say yes to the file check report and wait.
6. Repeat for any other partition.

I used BootITNG as my partition managers, imaging and boot manager. Thanks to, [http://www.goodells.net/multiboot/index.htm](http://www.goodells.net/multiboot/index.htm) I was able to partition my HDD just the way I wanted (8MB Bootit EMBR, 10GB Ubuntu, 10 GB Windows XP and rest ~400 GB as extended partitions). In the extended partition, I have a 50GB OS backup for the images of OS, 10 GB Linux swap and rest for data. 

General steps that worked for me using BootitNG. I am sure there are other (better) ways to do:
1. Boot from BootITNG CD and install BootIT to HDD. In the settings, mark "Limit partitions", this will keep Ubuntu and other OS to create more than 4 primary partitions. Create partitions as desired making Ubuntu partition Ext3 and Windows as NTFS. 
2. Create Boot items from Boot Menu for Ubuntu and Windows. Make Windows item Active. From the Boot Menu, choose Windows partition to boot. You will be prompted for bootable media. Insert the Windows installation disk in the CD drive.
3. Install Windows from the Windows CD in its all glory (updates, drivers, patches etc). Took me good 10 hours but I have a lot of programs and patches. I did not install the data at this point.
4. Boot from BootITNG. Go to the partition works dialog. Select the WIndows partition and click image. You will notice a message "* paste pending...". Select the OS backup logical partition and past the image where you like. I created a folder for Windows and another for Ubuntu and used dates as file name. As a backup, I also saved my image to a network drive.
5. Now set Ubuntu as Active partition. This time boot Ubuntu from the Boot Menu and insert Ubuntu LiveCD.
6. In Ubuntu installation process, choose "Manual" partitioning instead of Guided partitioning. Choose Linux partition and mount to root "/". You have to format the partition from Linux and choose Linux logical swap partition for Linux Swap. As long as Linux confirms the paritition numbers and capacity in general range, you are good. Ubuntu will automatically import Windows desktop and accounts.
7. After the installation is completed, you should get GRUB menu prompting you boot from Ubuntu or Windows. This is good news. Try booting Windows. I got an error message "Windows could not start... hal.dll". This is because I moved Windows partition and restored the image in a new partition while Windows remember the boot information from last time. Boot from BootitNG disk, go to partition works and click details on the Windows partitions. In the the Details dialog, click Edit file and goto the Windows folders. You can edit "Boot.ini" directly from within BootITNG. Update the partition number as explained by Dan Goodell.
8. At this point of time, GRUB will allow you boot between Windows and Ubuntu.
9. Instead of using GRUB, I wanted to use BootITNG as my boot manager so I can image the partitions as well. Install BootITNG again to HDD to restore the BootIT EMBR. 
10. Now Ubuntu will refuse to boot. I found another pointer on Ubuntu forum (Thanks). Boot from the LiveCD and initiated the terminal and follow the steps below:
a) sudo grub
b) find /boot/grub/stage1 (Note:this will give you location of the boot partition) For me it came back as (hd0,2)
c) root (hd0,2) (note: use whatever comes up in b above)
d) setup (hd0,2)
e) quit

After executing all the steps above, my system shows BootitNG menu and allow me to boot between Ubuntu and Windows. I dont know if these exact steps will work for you or not.

### Post by trak87 on 2007-06-27
> **Wynne G Oldman said:**
> ...I still haven't found a program that will back up a dual boot system using EXP3 [ext3?] partitions to an image on a seperate drive, (that I can understand how to use) that I don't have to pay for.

Drop to the command line and do something like this to copy a partition at /dev/hda1:

```
dd if=/dev/hda1 of=/somedir/my_hda1_partition.iso
```

It's that simple. hda1 could be ntfs or ext3 or fat32 or whatever. You end up with a single iso file. To restore it, just reverse the "if" and "of" and remember the limitation that the partition sizes must be exactly the same size to contain the restored data because dd isn't sophisticated enough to adjust partition size of the target for a restore.

BTW:

if = input file (or partition)
of = output file
