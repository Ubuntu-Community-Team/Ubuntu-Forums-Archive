### Post by Ken W on 2012-06-09
Hi!

I'm new to Linux in general with new hardware also.  I'm running an Intel i5-3570K on a Gigabyte  UD3H Z77 motherboard with 8GB of Corsair MMS memory and a Plextor 128GB M3 SSD as a boot drive. I'm using the on-chip Intel HD 4000 graphics.

I'm coming from years on Macs and am very impressed with 12.04 in general. I first installed the 64 bit version  (this was in mid-May) and had frequent system freezes. Everything froze, even the Caps Lock light on the keyboard.

I noticed that the 32 bit version was "recommended" on the download page, so I fresh installed that.  It was more stable by far than the 64 bit for me, and this improved further in the following weeks, but I still get freezes. If audio is playing, about 1/8 second is repeated ad nauseam.

From what I've been able to dig up,  I'm not the only one experiencing these freezes. I'm chalking it up to a new release and maybe the new Ivy Bridge hardware.  I'm loving Ubuntu in general and expect get the stability issue worked out eventually, hopefully soon!!

In the near term, I plan on giving 64 bit another shot with a fresh install. I'm considering customizing from Mythbunu. I'll be using this build heavily, but not exclusively for multimedia. I don't  consider it an HTPC, but I want it to act like one some of the time.

My brother has been using Ubuntu for a few years now and has been a big help. He recommended a method of installing the OS which would place many configurations in a separate partition so that  an *almost* clean install could be performed in the future without having to reconfigure everything.

Anyway, that's where I'm at.  Any comment from the community is very, very welcome!

> **Ken W said:**

My brother has been using Ubuntu for a few years now and has been a big help. He recommended a method of installing the OS which would place many configurations in a separate partition so that  an *almost* clean install could be performed in the future without having to reconfigure everything.

I'm going to tell you something that your brother doesn't know :-)

Even if he had set up Ubuntu in one partition, so the operating system is on the same partition as your home folder and settings files, you could still do a "clean install" of Ubuntu without losing your home folders and settings.

You could use the "Something else?" option in the Ubuntu installer to perform that sort of clean install (but don't tell it to format your partition, of course), or you could use the "Upgrade" option if it was a newer version than what you currently had installed.

This feature has been in the Ubuntu installer since 2009, but it seems that a lot of people still aren't aware of it.

There you go, now you know something your brother doesn't. Be sure to let him know next time you see him. Have fun, I hope you get your issues sorted out too.
