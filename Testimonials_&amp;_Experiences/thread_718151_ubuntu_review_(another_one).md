### Post by Az_135r on 2008-03-07
theres probably a million of these so ill get straight to the point

what can i say, fantastic... ill get the usual stuff out of the way, gutsy + compiz - brilliant, it makes me laugh when i think about vistas attempt at spicing things up. My gui is tweaked to perfection, customised to a point which was not possible on xp (used to run blackbox etc), its a joy to use

right where do i start with the rest..

what ive got running: in the limited space of time ive used ubuntu, circa 2 months, ive got 2 ubuntu servers running with webmin for admin management... 1 x 32bit ubuntu home server, 1 x 64bit ubuntu running on intel 3000 series server... my desktop machines running 64bit, and ive got a dell x1 running 32bit

**every single installation was a breeze, no hardware issues, no installation issues... nothing.. seemed a little to easy to be honest!**

i even managed to get a sony eyetoy hooked up as a webcam with relative ease on the 64bit desktop... oh and dual screens with compiz + sw raid arrays

what i like:
> performance for a start, even 1.1 ghz runs compiz + extras smoothly
> terminal, sudo, repositories, theres a package for everything, great stuff
> linux mounting - fiddly at first, but i like it
> nautilus - brilliant
> workspaces
> installation process, both desktop and server versions
> endless possibilities

lots more, maybe ill update this post when they spring to mind

dislikes:
> desperate for an exchange client as stable and reliable as outlook
> cannot find the perfect cross platform remote desktop set up, im not getting along with vnc so far

anyway.. its going great so far, thumbs up :smile:

ps. i had never used linux before 2 months ago, except for the odd dabbling on my centos vps server

### Post by Az_135r on 2008-03-08
hi insineratehymn

hmmm, webmin memory usage seems ok, webmin processes -> 150mb combined at idle 

vnc - well i don't personally like the way vnc is bundled with ubuntu-desktop (very microsoft i thought) and cannot be removed... its surprising when i read "Well, the simple solution is not to remove vino." within the ubuntu community

also, its unencrypted, for me this is a big no no. But its not just unencrypted, its too easy to set up, the normal user can expose themselves to an unencrypted vnc server without no less than a mouse click and no second thought


Of course you can set up ssh etc but for me, i need a one click solution rather than having to set up ssh servers etc, and for the normal user, its too complex

oh, and a current session is required, this wont work at all for my needs

ive tried ultavnc with good success on windows platforms, yet of course there is no ultravnc server for ubuntu. so a workaround is needed... you can install tightvnc server, but ive found that doesnt work very well... plus it bugs the hell out of me that ive got to use session 1 rather than 0 which is being used by the bundled vnc server

all i require is a secure, fast remote desktop application (both client and server), that is deployable in 'one click' and is cross platform compatible... :lolflag:
