Any reasonable person that admits Linux has faults, particularly in the software install system, admits that dependencies cause a lot of problems. I'd even say they've made up at least 90% of problems. But when you suggest that Linux needs to copy over the self contained install files of other systems, your idea is plain insulted. The other OSes you're trying to compete with knew dependencies weren't worth their problems like 10 years ago, but somehow they're better on Linux? Especially now that we have gigs of space to make up for the extra memory? If you want to resolve bug one, You have to implement a single file install system where each file contains it's own dependencies.

Your community would rather enforce the status quo, completely killing the idea that open source changes for the better. Instead of the developers hearing the reasonable ideas, they're drowned out by people that just want to things to always be the same. Instead of admitting faults they would rather bicker about technical details like dll hell, even though that kind of thing is from bad programmers and can happen on any system.

Until Desktop Linux has single file installs that bundle dependencies like windows and mac, I'll regard it as something run by kids who never listen to anyone else and not even worth talking about.

### Post by jbysmith on 2009-03-29
Well to be fair to Windows, DLL hell died with Windows 2000.  They implemented the Side By Side system, so multiple versions of the same DLL can be installed.  (Check your Windows\WinSxS directory.)

That aside, missing dependencies and missing DLL's is the same.  Just for example, ever install something and it pops up you're missing a Visual C++ runtime library? Same damned thing. Missing dependency. Well, not quite.  Modern Linux package managers are smart enough to grab the missing bits.  In Windows-land, it's either packaged with the installer or you're left hanging in the breeze to get it yourself.  How is that better?

And I agree.. this poll is retarded as the OP apparently doesn't understand how dependencies work, in either Linux or Windows.
