### Post by garyedwardjohnston on 2008-05-29
Another one bites the dust.

My heart is with Linux and open source but I'm tired of fixing bugs.

Ubuntu is probably one of about 100 different distros.  It's the most popular and because of this, I'm guessing the best overall.  There are probably thousands if you include every single one.  I was even tinkering with the building of one myself and I am certainly NOT A PROGRAMMER.

The fact is that one of the arguments against Linux is that it is only for advanced users and programmers.  The Linux community tries to deny this but it is very much true.  You all got so happy when Ubuntu can now come preloaded on Dell's (I think there is a link to Dell from every Ubuntu blog now, lol).  Don't forget, that this means that you, the Ubuntu community, believes that Linux is ready for the mass market, noobs included.

Instead of Linux developers getting together and building one good operating system, everyone has their own idea of how they think it should work and they create their own distro and rename it.  This is very inefficient.  Microsoft only has to worry about one system and thats why they do it better.

There is a good video about the paradox of choice from a TED Conference worth noting here.

[http://www.ted.com/index.php/talks/view/id/93](http://www.ted.com/index.php/talks/view/id/93)

The Linux community thinks that choice is a good thing, when, especially in this case, it isn't.

The command line (terminal) is still FAR TOO COMMON, 1/2 of the software in Add/Remove Programs either screws up my computer, doesn't work at all, apps looks like crap, not all my devices work properly, etc., etc.

Fact is, Linux brags about being able to do all these amazing things, when in fact its not even getting the basics right.

The only people this so called freedom benefits are the lucky few with the right configuration of computer (most likely Dell computers) and advanced users.  I wish you would all just accept the fact that Linux is NOT user friendly.

Sure you try to make it user friendly and everyone tries really hard to help one another, a very nice gesture, but alas it's like trying to put a band-aid on a bullet wound.  As soon as I get one problem worked out there are 10 more waiting.  I finally gets things up and running ok then an update messes it up again.  I simply can't get anything done because there's always bugs to fix.

I really think for Linux to really become anything more than a dreamy techie's operating system, 90% of the projects should end and you should all create a way to work together on ONE version that works well.

Fact is, Windows works much better than Linux and with a couple installs of security software and a little intelligence regarding the internet, you're pretty much secure, have more and better software at your disposal, and spend less time fixing bugs and reinstalling operating systems.  Sure you have to reinstall Windows every few months, but I have been doing the same thing with Ubuntu, even more so.

Maybe if I go back to school and get a computer science degree and have a few extra hours per day to spend fixing bugs I will be back to give it another try.

Bye Ubuntu,  see you in a year or so.

Linux of any kind is and has been geared to the DESKTOP units.
