### Post by maniacmusician on 2008-08-24
> **cardinals_fan said:**
> I would be much happier with KDE 4.x if it had any major improvements over 3.5.x.  In my experiences, the only thing that has improved is the appearance of the KDE Games suite.
[shrug] I suppose everyone will take away something different from it. I make an effort to keep up with the developer's blogs, so I can see all the different things that are being put into motion. The developers have said this a lot of times, but they have to first fully implement new technologies under-the-hood before they can really start rolling out amazing new features. They don't want to start offering code based on half-finished services, which will just result in a lot more work for everyone involved.

That being said, there are still ways you can dig into it and preview the new stuff if you want. 

As I understand it, this was their basic plan:
> **4.0** - Start putting together the new and re-written components of the kde desktop for testing
**4.1** - Polish up and fix bugs in the new services , improve the UI, start work on feature parity with 3.5
**4.2** - Complete (or very close to it) feature parity with 3.5, major work on features utilizing the new KDE4 technologies
**4.3 and beyond** - ...who knows? it's a crazy world. I imagine we'll see the different services get more and more integrated, hopefully with some radically efficient workflows and exploring different ways of interacting with your information.

Even if the core KDE packages don't seem to be better or new enough to you (which I'd disagree with), there are still individual apps that are doing cool things. For example, **Gwenview** is spectacular; **Kwrite** got even easier to work with; **Marble** is really useful as a slightly stripped down Google Earth and is really powerful in its own right as well, doing things that Google Earth can't do; **Krdc** had an overhaul; **Okular** is miles ahead of KPDF; **Dolphin**, with its Nepomuk integration, even in its incomplete state, is a sign of big changes.

There's other stuff that I'm skipping because it's often mentioned, and still more that I'm forgetting. So I certainly don't think that KDE4.x isn't showing any improvement. When you take into consideration that most of these developers have their own real life schedules to work around, yet still manage to be so enthusiastic and make so much progress -- it feels like they're making pretty good headway.
