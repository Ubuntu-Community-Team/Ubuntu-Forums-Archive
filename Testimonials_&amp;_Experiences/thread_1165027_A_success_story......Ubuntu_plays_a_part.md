### Post by lisati on 2009-05-20
Some time back I'd made a backup of the XP recovery partition on one of my desktops, but misinterpreted the meaning of one the settings in Nero . The result was a DVD-ROM that, although bootable, would be of minimal value in actually restoring the system should the recovery partition be deleted or damaged. A week or so ago I deleted the recovery partition & made the machine exclusively Ubuntu. 

Being mildly bored, and having little in the way of potentially useful chit-chat for these forums, I decided to reinstate XP temporarily so I could make a proper DVD backup of the original recovery partition. Out with the set of recovery CDs that I'd made when I first purchased the machine, and just over two hours later the recovery partition was reinstated and the regular XP partition set to an "out of the box" state.  

Next step, figure out how to use the preinstalled recovery disk creation software when it already knew that I'd made a set of backup CDs...a carefully but clumsily framed question at a forum at the manufacturer's website, (posted while waiting for the recovery from CD to finish), resulted in a link to the answer. Easy enough to implement (a little CLI stuff here). 

But Aaaaargh! the backup creator complained about a missing file! A little more research via Google provided some clues what to look for and where....so out with the "botched" backup (thankfully it was still around), a Jaunty LiveCD, and a copy of the missing file was located and copied to the proper place.

XP duly restarted, the backup creation software restarted, and, hey presto, it's completing its task as I type this "rant"......

Now to see if the DVD works and, if so, put some flavour of Ubunt back on the machine.
