### Post by jamyskis on 2008-09-11
I've been a Linux user since I dug my first 3 euro Linux distro out of a bargain bin while living in Hanover. I had always used Windows out of necessity and considered Go! Linux, then Red Hat 9, then SuSE 9, then Mandrake to be mere toys. I couldn't really find a serious use for them. After constant switching between distros for about 2 years, I finally settled on Ubuntu in 2004.

After numerous problems with Windows at work (I worked for a publisher in Gütersloh, I asked my boss if I could install Ubuntu on my work PC as a dual-booter, as I was web-designing and I was quite fond of Quanta Plus and the GIMP. I did, and although we had a few hiccups with setting up a link to the network printer and SAMBA file shares, it eventually worked fine.

When I left the company in 2006, I couldn't take my Linux with me, unfortunately, except to use as a toy.

Ubuntu's day for me came when I finally started out as a freelance translator and English teacher in June 2007. I bought myself a cheap HP530 laptop, wiped Windows Vista off it straight away and proceeded to install what was then Ubuntu 7.04.

It installed like a dream. I had no problems with wireless LAN, and I went on to install my Oxford-Duden German-English dictionary CD-ROM (complete with Linux client!), OmegaT, Quanta Plus and Scribus - all essential tools for a Linux translator.

Until now, being a professional translator was an expensive business - 100€ for Windows, 200€ for OEM Office, 600€ for TRADOS plus whatever else, and mine was all free and completely open source. I've twiddled about with the OmegaT source code a little bit to add a couple of features for myself.

Because of the low IT costs, I've been able to keep my prices down and stay profitable for over a year now, and it isn't letting up, even in the recession.

So why the title? Well, Ubuntu is by no means perfect. I've had my fair share of problems with WLAN later on, and some terrible problems with USB speakers (necessary for my teaching, as I have all my listening exercises saved as OGGs). I've had a few run ins with clients about formatting problems arising from compatibility issues with OpenOffice's interpretation of the DOC format and it's been difficult to find a CAT program that supports TagEditor and actually works (I'm waiting for Heartsome to stop saving its config files in /lib)

But while it hasn't been all plain sailing, seems like I've got a strong wind following.

I don't get it really. Why is this a "success story"? 

As a translator your main tool is Trados or one of its competitors (I use Déjà Vu). Trados does not work under Linux. Tools like Déjà Vu even need a dongle for authentication. To make things worse, the same applies to dictionaries. The electronic versions of ALL important dictionaries (e.g. the Van Dale series for Dutch, which are about one meter thick when put together) are ALWAYS written for Windows.

When it comes to languages, there is only one thing that puts Linux on top of all the rest, and that's Chinese/Japanese/Korean support on non-Asian desktops. Writing Chinese on my Dutch Ubuntu system with SCIM is a joy.

### Post by frutavana on 2009-04-09
> **duott said:**
> I'm a freelance translator too, and I do this for a living, but I've never heard of this TRADOS thing. What is that and why does one need it? :confused:


TRADOS is a piece of CAT (computer assisted translation) software. This is my sixth year as a professional translator and I've yet to find a customer demanding TRADOS, and I've worked with LOTS of people. Still, from what I hear, in a nutshell, it is a "translation memory": i.e., it "remembers" how you translated certain things and provides that translation the next time you come across the same pattern. I suppose it could be handy for large documents with lots of repetition, or for very formalized texts, and also as a means of producing glossaries. Still, it is rather prohibitively expensive, and I understand many customers are using it to pay less for our services (basically, paying less -or nothing- for the text provided by the CAT).

I've toyed around a bit with Wordfast, which is much cheaper (and, if I recall correctly, free for schools, universities, etc.) with a few less features, but I didn't really find a use for it.

The only thing I REALLY need for my work (and probably the only reason I keep an XP partition) is MS word, and then again, only for some documents with extremely touchy formats.

Glad to see so many fellow translators in here.

### Post by Gvigo on 2009-04-24
I'm also freelance translator using Ubuntu + OmegaT and OpenOffice. 

It's nice to know I'm not the only one!

Personally, both Ubuntu and OmegaT and FOSS has been really important to me because they've given me a lot of autonomy, economically and otherwise.  I've been making a living at translating for a couple of years and I haven't had anyone demand trados so far. But, I have a feeling that other people may find trados necessary because it may be used a lot more often in the kind of translation they do, like technical translation for example.  As for me, I'm sticking with OmegaT for the time being because it gets the job done and I haven't had any real  problems with it (see below).   

That said, I'm not really a full fledged expert with OmegaT but I'll give a shot at answering a couple of the questions posted above.

Making glossaries is really easy: 
The idea is that you have a three column tab separated list that consists of SOURCE (tab) TARGET (tab) notes/comments, you can do this with gedit or any plain text editor. After you've got that, save the file with a .tab extension and make sure the encoding is UTF-8 and then save it in the glossary directory of your OmegaT project, (you have to reload the project if you've already got it open) and that's that.  There's also a way to make a glossary with OOcalc.  Save your speadsheet as .cvs(text) select separate columns with tab, then change the file extension to .tab.

You can't translate pdfs in OmegaT.  (Can you do that in trados?) I usually save the pdf as text or convert it somehow, translate the text, and then save it and proofread it with OO.  I think the larger issue here is editing/modifying pdfs with Linux.  I haven't had much luck with the OO plugin for importing pdfs, it would be great to have that working better...

There's a [yahoo group for OmegaT]("http://tech.groups.yahoo.com/group/OmegaT/") which can be helpful sometimes (but should not to be confused with the group for OmegaT plus, which is not exactly the same software).

Lastly, I have a problem with OmegaT that I have no idea how to solve.  I recently changed my desktop theme to a dark theme and now I can no longer see the text in the OmegaT menus, it's still there, it turns white when I open a menu, everything works perrectly.  
What I need to do is change the color of the text in the menu.  But, I don't know if I have to change something in the OmegaT code or if I could do something with some kind of gtkrc file or the like.  I'm using OmegaT 2.0.1 in Intrepid.
  I don't really know anything about programming, but I'm not afraid to try things out.  I'd be really grateful for any leads on solving my problem.
