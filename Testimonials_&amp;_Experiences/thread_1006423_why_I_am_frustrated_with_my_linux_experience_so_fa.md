### Post by aysiu on 2008-12-09
My guess is that you didn't buy your hardware with Linux-compatibility in mind. That's okay.

Sometimes Linux users get a little overenthusiastic about evangelizing and "converting" people.

If you want to explore Linux, take it slow.

Here are some tips for you and practical migration steps:

1. Start with [open source Windows applications](http://www.opensourcewindows.org). Use those for a while. Make sure they fit all your computing needs.

2. [Install Ubuntu as a virtual operating system within Windows.](http://www.psychocats.net/ubuntu/virtualbox) This will allow you to quickly switch back and forth between OSes and not worry so much about hardware support.

3. You can keep this setup indefinitely... until you're in a position to buy new hardware. When you do, make sure it is Linux-compatible hardware.
[https://wiki.ubuntu.com/HardwareSupport](https://wiki.ubuntu.com/HardwareSupport)
[http://www.linuxcompatible.org/compatibility.html](http://www.linuxcompatible.org/compatibility.html)
[http://www.linux.org/hardware/](http://www.linux.org/hardware/)
[http://tldp.org/HOWTO/Hardware-HOWTO/](http://tldp.org/HOWTO/Hardware-HOWTO/)

4A. Once you know you have all Linux-compatible hardware, go ahead and do a full install of Ubuntu.

4B. Or, better yet, when you're in the position to buy a new computer, buy one with Ubuntu preinstalled.

VirtualBox is a virtualization software package.  In other words, it lets you "virtualize" one operating system inside the application (fake hardware) while still running another.  Picture one operating system inside a window on another.

I don't much like VirtualBox, and recommend VMWare on Windows and Linux or Qemu on Linux.  Try them all and choose for yourself.

EDIT: Here's a screenshot of FreeDOS being virtualized with Qemu on my SliTaz Linux desktop.  I'm familiar with regular command-line Qemu, but there's a GUI called KQemu available, and both VirtualBox and VMWare Player have GUIs.
