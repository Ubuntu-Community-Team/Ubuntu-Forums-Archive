### Post by sandybuchan on 2008-12-30
OK, I am a self-confessed technophile. I've come from machine code on a Z80 then through Basic, Pascal, Lisp, Visual Basic, Delphi, and now a bit of Python.I even got some assembler published in the dim and distant, but trivial stuff.
I want to love Linux, and I extol it's virtues to all who will listen. I know it is the right way to go, not just from a software development point of view but from a moral standpoint too.
What I don't like is the ability of some programs to take control of the machine, causing a freeze. Try to play an mpg in the “wrong” app = freeze. Alt-Ctrl F1-2-3-4-5  - not playing.  Try to install OO.o = doesn't work, took 3 hours surfing to get the arcane Jave problem sorted.
I just tried "ecawave" for a laugh, and got "Error: error". Well thanks, that doesn't give me a lot to go on. That's just an example, I'm sure it's a good program when it works.
Compile a tarball from source? Try ./configure and it doesn't work half the time (yes, I have build-essential etc etc etc). I have read all the info, man, web forums. Hmmmm. Piddles me off. Hit setup.exe in Windows and it just installs. Spoilt? Me?
Why can't the OS tell you need something for a new software installation, go get it, and do what's necessary? Even at compiler level?
For the average user, this is a total showstopper.
I started Linuxing with Sabayon, which compiles everything, and it had a much better success rate than I do, but Ubuntu does everything 200% faster.
Don't get me started on trying to “upgrade” to KDE4
There are a lot of things conspiring to keep Linux from being the OS of choice for Joe Public. The likes of Asus have done great things in tailoring Linux to a low-power machine, but the Windows community are unaware that it also powers supercomputers. They also have no desire to change. I use Windows at work, because there's no option. I don't host a website, when I would probably have the ability to decide to use Apache. I just use about 10 office apps but need to use them well; I use bubble charts in Excel to compare projects to management. Can't do that in OO.o.
Or any other Linux app I know of.

I know Linux is a work in progress, I'm just impatient to make it so much better than Windows that there's no competition. At present my programming skills are such that I'm relegated to doing man pages though!

I can't wait to plug in my Linux laptop to the boardroom projector and blow everyone away with a presentation with the odd "oops, sorry that was Compiz" moment.

I will continue to work with Linux, this is written on OO.o on Ubuntu.

This isn't supposed to be a troll, just my observations, and I would be interested to hear others thoughts on the subject.

Disillusioned Linuxer!

To the original poster,

If you're compiling something and it doesn't work, often that means you don't have a dependency installed. I often get errors when running the configure script, I just look up the error and usually one of the first results is the website for some software I need to install before the program I'm compiling will work. ### Post by BIGtrouble77 on 2008-12-30
one suggestion, don't compile apps on your main machine.  It's better to do it on a dev machine or vm so you don't pollute your main install with tons of dev libraries.  

I can't tell you how often I try to sort out dependencies and end up installing the wrong libraries.  Now I have an old amd machine I do most of my compiling on and build .debs for my prod machines.  Just building something like AWN takes a ton of libraries for a simple dock.

If you are having trouble working out dependencies when compiling always look for the first error.  That's always the best indicator of the library that's causing issues.  You should be able to search through synaptic for the correct dev library.  You can also search through rpmfind, but that process is a bit more complicated.

If you are compiling crap you should try to understand what you are doing.  It will be much more rewarding in the end.

If it locks up to the point that it seems your only option is Reset, you can usually hold Alt + SysRq and type in R-E-I-S-U-B with about a second between each letter.  That tells the kernel directly to kill all applications, unmount the drives, and then reboot.  It's safer than Reset.

One of the most compelling reasons to use Linux is its flexibility.  The very features I like most are often the same ones that make casual users who happen to use my system go absolutely nuts.

In *theory* it should be possible to have the best of both worlds, but in *practice* that is seldom the case.  I've already seen over the years a few areas where Linux has gone down the path of losing features / flexibility in the name of "ease of use", and I don't like it.

There are a number of spreadsheet apps for Linux, if enough people say they would like this feature, it might get implemented (assuming the developers want to, or someone pays them).  I'd say this is not a critical feature for enough people.

The lack of features compared against MS Office has long been used as an excuse against Linux.  Thing is, very few of those features have really been _critical_.  They are just one of many ways to present the same information, and the other ways are available in other apps.


I'm not a rabid Linux supporter, or anti-MS zealot.  I *prefer* Linux (by a huge margin!) but can use Windows just as well if need be.  I even used OS X on a Mac Mini as my primary system for a few months once, and while I do like it (and even recommended it to my sister) it too restricts me more than I like, so I went back to Linux.  

I set my parents up with Ubuntu once, and they quite liked it - but eventually went back to Windows.  Primarily, they wanted to print photos with their WinPrinter.  Hey, the printer actually _worked_ in Linux, a far cry from the state things were in last time I used printers!  It just wouldn't go to the highest-quality photo setting, so they went back.

Use what works best for you, there's no need to condemn the OS after a short trial.  It took me a year or two before I could comfortably use it for *any* length of time!  (If you think it's hard to use *now*, try a distro from 1993! ;) )  I would install Slack from floppies, get the system running (text console only) then stare at it for a while.  Reinstall Windows.  (Didn't have a spare machine then either!)  Each time I installed Slack I'd get a little further before going back, and eventually I was comfortable enough to stay with it.

Note that I am NOT a programmer.  I know just enough of a few languages to stumble my way around a source file and get the gist of what's going on.  The limits of my actual programming "skills" are what amount to some fancy Python scripts.
