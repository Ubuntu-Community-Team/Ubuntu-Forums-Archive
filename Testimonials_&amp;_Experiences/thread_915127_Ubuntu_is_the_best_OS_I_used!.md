### Post by speedkreature on 2008-09-10
Yup, another plus one. 
I just discovered something neat that I could rarely get to work right in Windows.

I'm using a terminal program at work which communicates on one network and I was busy working and listing to music over this wireless connection.  Well, the security admin updated the rules on our IPS so streaming media is now blocked so I plugged in an ethernet cable fully expecting to lose my terminal connection while network manager figured out what was going on.  Nope.  Music is now streaming over our testing network and and the terminal application is still working...these two networks have completely different IP schemas and subnets (i.e. 172... vs 10...) and are physically (not virtually) seperate networks.  

Windows default behavior would be to switch from one interface to another, not use both simultaneously.  It just blows my mind.  I'm a little elated about it.

I've even got my girlfriend using it now.  She still uses the Vista machine I built her when she's at home, but when she's at my place she is _completely_ comfortable with Ubuntu and she's a very novice computer user.  What's more, she says she prefers using Ubuntu.

That is all.
