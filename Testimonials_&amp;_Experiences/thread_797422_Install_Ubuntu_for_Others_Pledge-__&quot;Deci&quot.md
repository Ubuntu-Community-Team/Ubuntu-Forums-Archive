As I was about to cut up and throw one away, it dawned on me to keep these as evidence of my Windows "kills".

In this year alone, I've "earned" about 6 since January.  And this isn't counting the DIY machine builds I've done for myself and others this year.

How many Windows "kills" have you achieved in recent years?

I propose a pledge to "Deci"-mate Windows this year, whereby you pledge/plan to install Linux on at least 10 (the "deci") computers this year, either replacing Win on OEM preloads or installing on DIY machines. 

The intent is to do this for 10 machines, preferably owned or used regularly by others (i.e your "Mates" for those down under ;)), doing the install and app/driver/peripherals setup for that machine to get it functional to the users expectations with appropriate alternatives when available (tell the gamerz to get a PS3/Wii/Xbox360 and leave the PC alone ;)), including Wine if needed, but Windows in a VM not allowed.  IMO, most "normal" people, regular everyday PC users who just want to play audio and video as well as web/OpenOffice, won't need Wine anyways.

The point is to eliminate the need for a new Win license.  Be sure that the Linux load you do does all the media playback expected- MP3 encode/playback, DVD playback of encrypted discs, video capture and editing if needed, etc.

If Win dll's are needed for Wine, often the machine you're converting had Windows on it, so you're entitled to those .dll's for that user, assuming it was an OEM preload or paid Win license from the past. So don't erase the Win partition- at least keep the windows directory and/or burn the Windows directory to a DVD-R just to grab dll's when/if you need them.

On December 31, or maybe one year from now, we'll see who has the most Windows "kills"...

..for  a more peaceful metaphor, consider yourself a "Linus Linux-seed" :D

[http://en.wikipedia.org/wiki/Johnny_appleseed](http://en.wikipedia.org/wiki/Johnny_appleseed)

For those that haven't had the time and/or skills to contribute to FOSS in other ways (programming, maintaining a project, writing documentation, etc), simply building boxes, either new or with leftover used parts, and/or installing Linux OS's (including Mythbuntu, Geexbox, the upcoming XBMC for Linux, Mint, Puppy) for others on their OEM-preinstall boxes (or BYOPC's) is as important a contribution.

For machines around 1.4GHz or less, I'd recommend Xubuntu and/or replacing the default window manager with OpenBox or similar lightweight wm. The Metacity wm in Gnome is one of the main culprits of sluggishness on lower spec machines, but switching to the Xfce environment (Xubuntu) vs Gnome or KDE is simpler.  I've never had a Gnome or KDE app that couldn't be run under XFce.

For machines less than 700Mhz, you might want to install PuppyLinux, unless you have suggestions for how to optimize Ubuntu/Xubuntu for snappy response on <700Mhz machines.

If current versions of Ubuntu/Mint/Puppy don't work on the hardware you're installing to, try a version one or two revs back- remember that 6 or 12 months in the Windows world is nothing- just use what Works.
