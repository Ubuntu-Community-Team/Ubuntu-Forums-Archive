### Post by arubislander on 2012-12-18
I am a software developer at a smallish sized company. So here I was needing to implement file transfer via sFTP in our application for a customer. The customer was going to provide credentials for a sFTP server for testing purposes, but I was anxious to get this done asap and did not want to wait for them. 

But what was I to do since our internal sFTP server was configured to only work with keys and I needed one configured to work with username/password. Also I did not want to bug our sysadmin seeing he was having his own set of issues due to our hosting provider migrating our servers...

Then I remembered I had a fairly up to date version of Ubuntu on a virtual machine. So I fired that up, installed openssh-server and presto, my own sFTP server ready to go.

That's why I love Free Software!
