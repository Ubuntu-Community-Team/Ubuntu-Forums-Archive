### Post by cwklinuxguy on 2012-02-07
Why would you not WANT to use the repos? The repos are much more secure than downloading files directly, and it's typically easier too. If you have an issue with the repositories, by all means inform Ubuntu. If you have an issue with the installers provided by a program developer, by all means tell the developer! But we're a community of Linux USERS--you're essentially preaching to the choir here.

 By the way, the RPM-tar.gz-DEB "confusion" as you call it is the result of multiple distros having different names for their package management systems (and actually I think a tar.gz will usually work with any distro, if I'm not mistaken). All in all, the systems are very similar with only minor differences in the technology used. But they ARE different enough to be incompatible with each other. Most programs are available in the repositories, and if they aren't there is probably a DEB file or the ability to compile from source. But expecting Linux to work the same as Windows is absolutely ridiculous and unrealistic. You don't expect Mac OS X to act the same way that Windows does, so why would you expect that from Linux. They are different operating systems that work differently. If that different way of working is not what you require to get things done, then don't use that particular OS--that's fine. But I think you should give it time. You may eventually see the advantages to why Linux does things how it does.

And by the way, the fragmentation between different installation systems isn't here for good--Ubuntu and all other major distros are working together and will eventually create a unified app store.

### Post by 3rdalbum on 2012-02-08
Linux won't suddenly sport binary installers such as the Windows-style ones.

Not because the Linux developers are blind or stubborn, but simply because there are better ways of handling program installation. On Linux, you use your distribution's repositories to download and install software. If you need a later version, just add the program's PPA to your system and you get versions as soon as they become available.

Coincidentally (or not?) Windows 8 is moving to a repositories system as well. It's not really a coincidence - Microsoft is doing this because it's simply a better way.
