### Post by viperdvman on 2012-06-03
*[SIZE="1"]Note: This testimony refers to my desktop computer, not my netbook[/SIZE]*

I've been using Ubuntu for over a year now. When I adopted Ubuntu as a secondary (but most-used) OS alongside my Windows, I had installed Ubuntu 11.04. Unity wasn't bad, but it was still in its infancy. So for the longest time, I used the Ubuntu Classic session, with GNOME 2.32 as my primary DE. I had modified it to look and act a lot like Mac OS X, with a top panel containing the mint menu, the global app menu, systray, time, and shutdown... and AWN dock at the bottom with a secondary app menu, a few applets, my launchers/task manager, and my trash can. For a long time, it worked very well for me.

But, Ubuntu 11.04 isn't going to last forever. It's dead this October. Also, GNOME 2 is completely dead in Ubuntu come April next year when Lucid finally dies. So I upgraded to Ubuntu 11.10 a few months ago. I liked Unity even better in Oneric than I did in Natty, but it was still a secondary DE for me. I had modified my GNOME 3.2 "GNOME Shell" with plenty of shell extensions and the AWN dock to once again look and act a lot like Mac OS X. I had all of the functionality that my 11.04's GNOME 2.32 had... sans global menu. I even installed a package and modified a Simple Launcher applet to activate the Activities Menu from AWN... giving it a pseudo Mac OS X Launchpad function.

Now, I have upgraded to Ubuntu 12.04 nearly 3 weeks after its release. I have yet to mess around with GNOME Shell the way I had messed with it in Oneric, but I have played around with the more mature Unity. I've gotten my previous Zelda themes to work very well with it. I like using the new HUD, and I like some of the extra stuff I can tweak in Precise's Unity vs. Oneric's Unity. I even added the "Dodge Windows" function back to Unity.

However, here's the kicker. I've been playing around with KDE Plasma since I still had Natty. In fact, I had installed it as a secondary DE and had even posted screenshots of it. It had pretty good functionality and lots of flexibility, especially when it came to the Plasma Activities. I had the main Desktop, a News Feeds, and a Fanfiction activity... each according to what I was using it for. I liked it so much that I had switched to it off and on according to my mood, but always went back to my Mac OS X style GNOME. I never did install KDE when I was running Oneric, not having the time nor being in the mood for it as I had an excellent Mac OS X interface in GNOME Shell.

So after playing with Unity for a bit, I installed the full KDE desktop on top of Ubuntu and trimmed away any unwanted KDE apps that it brought with it. Just like with my Natty's KDE, I tweaked it with oxygen-transparent, added a number of themes and plasma widgets, and created my News Feeds, Linux News, and Fanfiction activities (more to possibly come). On top of that, I had discovered when I played with a live USB session of Kubuntu that KDE actually handled dual monitors **very very well**. Not only can I have different resolutions and can designate a primary display like I can in Unity and Windows, but it has native support for different wallpapers on different monitors. 

After playing around with my new KDE install, **I have made KDE my primary DE**. It just does everything I want it to, especially when it comes to dual displays. I could make a Mac OS X style interface out of it if I want to, or even create a minimalist interface like I've seen some Ubuntu Forums members posting screenshot of. But right now, I'm fine with the Windows Aero style interface that I'm currently running. Don't get me wrong, I love how much Unity and GNOME Shell have matured in the last couple stable releases. And I still love using them according to my mood. I still love how I've made a Mac OS X style interface out of GNOME Shell. But... here's the downside to at least Unity's dual-monitor support: While it can be set to display each monitor in its native resolution, and I can choose which monitors show the Unity launcher, both monitors end up using the same wallpaper. Now this would be fine if my two monitors have the same resolution, but they don't. I have yet to find a way to have different wallpapers for each monitor in Unity and GNOME Shell like I can in KDE and with 3rd party or NVidia software in Windows. _I am open to suggestions._

[SIZE="1"]*note: I haven't tested this in GNOME Shell yet, nor have I tested AWN in a dual-monitor environment... on **any** DE*[/SIZE]

So yeah... when it comes to running Ubuntu, KDE is now my primary DE. Why not just install Kubuntu instead of installing KDE on top of Ubuntu? Simple... I still like using Unity and GNOME when I'm in the mood. But right now, KDE is my primary. Just don't ask me to run the KDE netbook interface... it doesn't play with dual monitors very well, and I doubt it was designed to.
