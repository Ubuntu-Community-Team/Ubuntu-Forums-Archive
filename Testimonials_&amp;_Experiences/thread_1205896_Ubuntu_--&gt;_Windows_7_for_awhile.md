### Post by jimv on 2009-07-06
I've been using Ubuntu for 3 years, but it's simply too much of a headache to deal with after the initial excitement is over.

Here's some of the problems I've run into over the last few years that just frustrate the hell out of me:

* Random lockups - this seems to come and go with kernel releases...some lock up, some don't

* Random program crashes

* Firefox is slow slow slow when compared to Windows.

* Little consistency in applications as far as networking.  I can set a proxy in Gnome's proxy settings, but as far as I can tell, the only apps that look at it are the terminal and Firefox.  Rhythmbox sporadically uses the proxy settings...LastFM will work, but not the radio or podcasts.  Synaptic has it's own proxy settings.  Pidgin has it's own proxy settings.  Thunderbird has it's own proxy settings.  It's tiring to deal with.

* Gnome's controls are way too big.  The File Open dialog box is ridiculous.

* Drivers - Why must I reinstall my drivers (Nvidia and VirtualBox) every time I install a new kernel?  Yeah, there are ways to get around this, but I shouldn't have to.

* Slow IO.  For some reason, copying files slows down my entire machine in Ubuntu.  Windows doesn't have this problem.

* Available games suck.  Don't argue...they're mostly like 1998 shareware.

* ACPI oddness.  I had to patch my DSDT to get my brightness buttons to work correctly in Ubuntu.  In Windows 7, I ran Windows Update and it installed an Nvidia driver that fixed the issue automagically.

* Overall lack of polish.  I installed Windows 7 last week.  It FEELS like a solid, well thought out, finished product.  Ubuntu feels like something patched together with duct tape and then spray painted to look pretty.

Anyway, I'm leaving Ubuntu for awhile.  I'll be back when 9.10 comes out to try it again, but I feel like I'm going to be sticking with Windows 7 for awhile.  It just works.  And I'm shocked to be saying that about a Microsloth product, but it's true.  It was a struggle to get Ubuntu working on this machine, but I installed Windows 7 and it was just so smooth.  It didn't have all my drivers at first, but I ran Windows Update, and it detected and installed the appropriate drivers for my hardware.  One reboot and everything was working.  And Aero has been much improved since Vista.  It's smooth and doesn't slow my machine down.  I also didn't have to go into CompizConfig-Settings-Manager and manually set refresh rates to get it to work right.

I'll still be using Ubuntu on my home server (I mean, you can't beat 80MB of ram usage on idle)....but I just can't take it anymore on my laptop when Windows 7 is so much better.

Well, I use both OS and they both do well. I am attending a Unix-influenced high-school, but at work and at home it's nearly (!) all Microsoft. I run both Windows and Linux based servers. The simplicity is a great pro for Windows, Linux being free is a great pro for Linux.

The reason Windows needs anti-virus and other related software is mostly because it's so widely used; the hackers/writers of virus target the platform used by most people, as simple as that. When/if Ubuntu gets as big as Windows it might very well have problems with viruses and trojans.

Also, my most recent experience with installing Ubuntu on a HTPC:
[http://ubuntuforums.org/showthread.php?t=1200868](http://ubuntuforums.org/showthread.php?t=1200868)
[http://ubuntuforums.org/showthread.php?t=1202832](http://ubuntuforums.org/showthread.php?t=1202832)
[http://ubuntuforums.org/showthread.php?t=1205184](http://ubuntuforums.org/showthread.php?t=1205184)
(notice the lack of replies =( )

The same setup just worked in Windows, no need to spend hours googling/configuring/trail-and-erroring. As I think it is quite fun to mess around with these things, I kept going. But as you can read in the last link, I was very close to giving up... If you consider the average user, they don't have my interest, and as such they will use the simple "just works" alternative.

I have also seen several strange crashes on Linux-based servers compared to Windows-based servers. Two examples:
1) I had a Ubuntu-based VMware Server host. "One day" I was about to add a new virtual disk to one of my VMs. I created it, VMware popped up a EMPTY error box and the whole system went down. It was not a hardware fault.
2) A friend had a RAID0-array (mdadm) setup in Ubuntu (about 1 TB of data). He accidently hard reset the computer. At next boot, he got some error about the partition. He tried running fsck and then mounted, but all files were gone. I have been running a RAID0-array in Windows Server 2003 for quite some time, and I have hard reset that server several times without any data loss.

However, now when I got the HTPC up and running, it's running as smooth as Windows ever was (not considering the slower Internet browsing experience, as mentioned by others). And it's totally free!

Both OSes have their uses, and it's quite pointless to argue "which is the best". It's a fact you will have to spend more time in configuration when running Ubuntu, but you can also fine-tune almost anything. It is a matter of what the user prefers.
