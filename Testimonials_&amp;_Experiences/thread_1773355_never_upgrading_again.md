### Post by Mariane on 2011-06-02
:lolflag: 

I know the feeling... And thanks for the warning :) . 

Before using Ubuntu I used Debian for 5 years. Debian is hell and brimstones to set up, I had to get someone to do it for me. But once set up, it stays up. I used to type: 

```

su
apt-get update
apt-get upgrade
apt-get dist-upgrade

```

every once in a while and that was the only maintainance it ever needed. During these 5 years it only crashed twice, each time because I did something really stupid with it. (Yes, I'm the kind of person who can even crash a stable and up-to-date Debian - in fact the second time I brought down the computer system in the whole office). 

Now dist-upgrading Ubuntu is always risky. Once I ended up with huge icons, huge everything, and only a small part of the desktop visible on the screen. Another time, when alsa was replaced by this <self-censored> pulse audio, I got into endless sound problems. These only got resolved by the next dist-upgrading which allowed me to have alsa back at the cost of no more Gnome interface but only KDE. AFAIK the Gnome interface still wants to force you to use pulse-audio. Beats me why we are not allowed to have the Gnome interface with simply no sound at all? I'll only let pulse-audio back on my machine if someone pays me $2000 or more to do it, in which case I'll install pulse-audio, trash the machine and go buy a new one. 

Maybe Debian would be good for you? If you can get someone to set it up for you and if you don't ever want to fiddle with your machine, but only to use it. Ubuntu is much better if you want to fiddle. When you fiddle you will mess things up from time to time, and when this happens you get much better support from the Ubuntu community than from the Debian community. Debian people are not unfriendly - that's a lie - but most of them are experts and when they answer one of your questions they answer as one expert talking to another. Which means you don't understand what they're telling you to do and even less how you're supposed to do it. 

Look on the bright side: today you have learnt something. During my 5 years as a Debian user I learnt nothing about trouble-shooting and that's a mixed blessing. 

Mariane

### Post by 3rdalbum on 2011-06-03
> **throatgorge said:**
> 
First off, that bar at the top with all the pull down menus?  When you're accessing one program, you can't use that bar to access anything else!.

Correct. It's a combination panel/menubar.

> When you try to open two windows to drag and drop files, surprise!  The first window you opens disappears and is replaced with the other!

That doesn't sound right. You can drag and drop files between two open windows even when the second one isn't visible. Just hold the file over the Unity launcher (it will swing out from the left side while you're dragging), hover over the program icon, and the program's windows will appear. You can now continue dragging your file to the window you want to drop to.

> There appears to be only two sizes for all apps-- fullscreen and minimize... but there is only ONE because when you minimize, it closes the program!

When you "minimize", the program becomes accessible only through the Unity launcher. Just like when you minimized in Gnome 2, the program would only be accessible through the bottom panel.

Unity is a different user interface. We can't stay with the old Gnome 2 forever. It's no longer maintained, it's not touch-friendly for touchscreen devices, it doesn't look very pretty and there are problems with it that have never been solved.

Of course, being a different user interface, Unity works differently to Gnome 2. You just have to keep with it to learn how to use it. After all, you gave Ubuntu a chance, you didn't just say "This doesn't work how Windows used to work! I'm going back to Windows!".
