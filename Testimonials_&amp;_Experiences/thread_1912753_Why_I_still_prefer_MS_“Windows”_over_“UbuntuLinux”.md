### Post by alkatraz on 2012-01-21
[B]Why I still prefer MS “Windows” over “Ubuntu/Linux” 
[/B]an open letter to the “Ubuntu/Linux” community from an average user

       Dear Ubuntu,
       I'm an average user who doesn’t have any special programming skills and/or knowledge in the OS area, but I'm still able to work my way and do what I have to do when I have to do it with computers. I've heard/read a lot that Ubuntu/Linux goal is to work out of the box and to be more user friendly these days. So here I am the average user and that's my humble opinion – It ain’t going to happen. 
 First of all just to clarify things up I'm not advocating for MS. No, Sir! Not at all. Ubuntu is a great system, and it has a lot of potential(I think). Some other major distros are too, but I’m still using MS “Windows” as a primary OS. So why is that? The one sole reason is because regardless of all the negative opinions in the Linux community, and all the small annoying bugs and flaws(the key word here is small) it's the system that still just  works(!). Plain and simple. And I don’t have to be a programmer or something else to use it. I like the idea of free software but when it came to usability and reliability I have no choice but to pay allot of money to MS for their OS. So it's somewhat confusing – on one hand I like Ubuntu and linux but on the other I’m forced to pay money to MS. And forced I am because Ubuntu/Linux just didn't do the job. Yes it didn't. 
 Things are simple. When I switch on my laptop on Linux is literally eating my battery's life. In Windows there are like almost three hours but in Ubuntu/Linux it starts of with two(!). I've read that it was a kernel problem and I've tried to fix it, but ... Oh dear, it was a Sisyphus task with no luck at all. I thought that there would be a nice little program to fix things up but no. And Ubuntu with the Unity interface didn’t help either. The lack of instruments to set things up is almost scary. There are only basic controls for power In the system settings menu and if you want to do something else ... 
 On every system start up Unity turns my bluetooth on, and switching it of from the panel is just a temporary measure till the next boot up (!). Again I've to spend allot of time to read in various forums to find a solution witch was to add a weird command line (com on guys I'm not a programmer ) in start up module or something like that (yikes!). I've set up some transparency to the panel but it randomly switches of, so I have to set it up manually again and again spend time to do it. 
I like the ideas in Unity, but (again but) I’m still not ready for it, or maybe it's not ready for me(?!). I don't know. In windows (7 it is) the transparency is working fine. All windows menus, and other stuff are behaving like thy are supposed to do. In Unity the transparency is linked to the panel but it's somehow linked with the windows too, so when I make the panel transparent the windows titles became transparent too and the buttons are disappearing (at least annoying). The GIU tends to crash when you try to set it up and there is no reliable way to recover it unless you open a console and input some mystical line in it, witch I don't know cause I'm just an average user. Actually its 'still very difficult to set things up in Unity. On the other hand Windows didn't crash when you customize it's theme. 
 Another annoying thing is the launcher. When you click on a program you didn't get the feel that it's going to work. No you have to wait to see if the magic will happen. Position. Come on guys even in Windows you're able to simply move the taskbar/superbar where you want it: left, right or at the top of your screen. I can live with that, but the feeling is strange. There are allot more examples: the software center (slow and buggy), the system browser (why the open as ... command or whatever it's name was is not present by default), the embedded music player and so on, but that's not my point. 
 My points are: If you really want to be user friendly… well start to listen to your users. It's ain't gonna hurt allot, maybe just a little but definitely not allot. Improve the system look and feel. It's a critical thing for every average user. Cause frankly now it looks like the apps and the window manager sometimes have different opinions on what they should do or look like. Make the system buttons to do their jobs. When you click on some button you expect something specific to happen. For example (just for example) when I switch off my bluetooth I expect it to stay that way until I switch it on. I think we all will agree that there's nothing wrong with that to have more tools to set up our systems like we want not with some mystique terminal commands but with a mouse or a touchpad (they’ve been around from quite some time now you know). Just a suggestion, try not to be so innovative but focus on polishing things up and removing bugs first. And if you must to, than slow down the release cycle. I (the average user) did not expect six months miracles, but definitely will appreciate if the new version of my favorite OS has less bugs and problems than the previous one. 
 Finally like I’ve said/wrote before I'm not advocating for anyone. I'm an average user and such as I wish  my operating system to help me to do my tasks not the other way around. For now Ubuntu/Linux doesn't seems to do that. And sadly until things improve I'll have to stick to MS Windows as primary OS. But regardless of all the problems I still have hope, and that’s why I'm writing this letter. So please Ubuntu be more reliable and help me to rid of MS Windows.
Yours sincerely,
The average User I.Ivanov
