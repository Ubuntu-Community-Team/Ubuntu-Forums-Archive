Are there really people out there who can identify icons faster than they can read?  And then, are there really developers out there who are so poor at reading they would make a menu system with pictures instead?  

Back when I began programming, and we're going a long long way back, there was a rule of thumb used by many developers.  If a user must open a book to figure out how to navigate something, then it needs re-designed.  I would say that today the same idea would be written, if a person must consult the web to understand a new interface, then it is not intuitive enough.  

All this hidden stuff all over the screen? What's up with that?  I'm with couling above who is unable to cope with the autohide stuff.  It drives me mad as well.  Like the lack of a menu isn't enough, they hide the program main menus, and then make it look like it doesn't belong to the window of its app.  

Windoze 7 has a menu that won't expand, must be clicked and scrolled all the )(*&%*$ time, someone should be put in front of a firing squad for that.  I simply don't use 7 for that reason.  Here, there are also choices, unfortunately, mine is to lag behind, or go back to the 'finally' functional Fedora.  

If the decisions were all mine, I would approach them like this:  "Can these changes allow me to get more work done in less time (Y/N)?"  Yes?  Then they must be intuitive,  implement them.  No?  Then they are a waste of time for no good reason, rip it out.   Can a person look at icons in no particular predetermined order and take that information in faster than they can read a smaller, text based, alphabetized list?  No? then lets use the alphabetized list.  

When using any other menu system, and there is an icon, and text to accompany it. Do you look at the pretty pictures, or do you simply read the text and forget the pictures?  I'm betting 99% read.  The others simply identify with a color, and don't actually use more than 2 or 3 apps on a system anyhow.

I like how easily it can be navigated with the keyboard. I like how you can hold down Super+Shift and a number key to open another window. It would be nice if you could shift and click to do the same as there currently isn't a way to open another window for an app from the dock with the mouse.

To summarise - I like Unity so far - it's far better than Gnome3 (which seems to assume everyone uses iPads) but it's not quite ready yet. It feels like I'm using an early beta release as it's quite buggy. Hopefully these issues will be fixed and they'll change their mind about keeping the dock on the left and just generally make it more customizable.
