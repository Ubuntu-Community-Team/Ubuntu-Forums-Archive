### Post by harishg on 2008-12-02
I was trying to install Ubuntu 8.10 from a live-cd on a quadcore intel processor with a silicon graphics monitor. I booted in the installation cd and the default choice of run from the cd failed. I tried doing startx in the command prompt and i got the message no monitor detected. so I again started the live cd and selected safe graphics mode and the installation went smooth. For beginners, I doubt if they will be able to figure this out. It would be nice to have Ubuntu automatically resort to the safe graphics mode when there is a problem with the default configuration.
