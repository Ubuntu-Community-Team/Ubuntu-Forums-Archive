### Post by theDaveTheRave on 2008-10-31
Hi all,

just thought I'd add this thread after a problem I Experienced a couple of days ago, partly to show any newbies what fun we have with Ubuntu even if we are "experience" users!

My problem. 

One after noon my networking failed on my laptop.

I checked all the obvious stuff, cable plugged into computer and into local "router"

Nothing worked to fix the problem.

Eventually I gave up, and after heading home and finding that all my networking worked as per normal I deleted all my "extra"  network setting, so now everything was back to a basic automatic network IP type thing.

so back to work in the Morning,

And still no networking via the Ethernet port!

I eventually decided that someone had done something to the local network, so as it only recognised certain hosts, or changed the location of the name server.

It was none of these.

So I unplugged my works desktop, and shoved the ethernet cable into the back of the laptop.... and amazingly everything worked just fine...

so now I was really confused....

The Solution.

The network cable, although plugged in at both ends has a "dodgy" connection at the router end, a little wiggle and everything was back to normal, untill the guy with the vacuum cleaner returns I guess!

So did I feel REALLY STUPID

You bet. :lolflag:

Was it a learning experience... OOH yead :guitar:

So now if something like that fails, I'll remember to try a "known good" configuration.

I'm sure I'm not the only one to have a daft story like this, so lets share them for the newbies.

David

So I was having a little trouble, if I didn&#8217;t wait until windows finished loading, [the last thing being a HP Scanjet driver which makes a startup noise] and started explorer the computer would freeze usually in the first minute of use.

The Scanjet driver is not a vxd so I looked for an update, no luck, talked to Microsoft, no luck, tried every thing I could think of including eventually resetting the BIOS default..

This provided the next opportunity for grief , the reactivated post test would, every once in a great while, show memory failure. Down to the basement to rummage. Got lucky , found one bank of 32mb edo enough to change out half at a time.

Instead of taking the box down to the basement bench, I just slid it out enough to remove the cover without unplugging everything, of course changing memory in a mid size tower with two HDs two floppies and two CDs requires shoving a mess of ribbon cable around.

Cover on, power up,Cover off replug loosened floppy line.

Cover on, power up, works great, for several days, then starts freezing under all new circumstances. Now after being on a while whenever I exit programs, try to access explorer or the my computer shortcut, freeze city , and it is a software freeze that can be cleared by re starting explorer under the cntrl alt del popup. Also freezes when I try to back up data to the d: drive, try scandisk through, find one bad sector, but problem continues. Try new HD, problem continues.

Finally after having the cover on and off a dozen times, I get a flashlight so I can see if the eide line is seated fully on the board. It is, but I find the ribbon cable is a little tight stretched over the exceptionally pointy sharp unused connector for an accessory to the video card, just above the fan exiting hot air from the processor, sufficient to soften the plastic.
