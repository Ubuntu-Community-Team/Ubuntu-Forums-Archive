### Post by rcasha on 2011-05-21
I doubt if the developers could have picked a more unfortunate name for the new desktop environment. Unity seems to be one of the most divisive changes in Ubuntu.

I've been using Ubuntu for a long time now - in fact ever since the shakeup that converted Mandrake into Mandriva. I've been quite pleased with every new release - until now.

Unity was one rather bad experience, and Gnome 3 shell didn't improve matters. The fact that the installation of the latter broke Unity didn't help either. Unity doesn't work well with two monitors of different resolutions, and the panels that I needed and used are gone.

Apparently the idea behind unity is to have one user interface across all devices. Huh? Why should a teensy pad with touchscreen have the same interface as a dual monitor setup with keyboard and mouse and ample screen real estate? In order to make Unity cater for both, it must necessarily be designed to the lowest common denominator. For instance, it doesn't have a right-click functionality because you can't right-click a touchscreen. Well my mouse can, so on my desktop I want an interface that takes advantage of everything that my mouse, screens and keyboard can provide. I'd rather have two slightly different interfaces for two very different devices, than one interface that does both jobs poorly.

Now, from the discussions going on, Oneiric Ocelot is going to be de-clawed. Choosing chrome by default is not yet a good choice. I have both installed, and I still find Firefox to be the better browser. 

Ubuntu should take a feel for its users' preferences before making such decisions final. Ubuntu is a big distro, but every other distribution is just a download away and I think that if people feel that Ubuntu is no longer about the users, they'll look for one which is.

"A try" is defined as "at least a week".

No, Ubuntu's telling them "here's another option for you, start with this". As has been pointed out time and time again, regular Gnome is still available. KDE, XFCE, and LXDE are still available. Run Window Maker if you want. In Ubuntu 11.10 you'll lose Gnome 2 but you'll gain Gnome 3.

> Then maybe they should have waited until it's a bit more complete.

Release Early, Release Often. In open-source you don't work on something for three years until it's perfect and THEN release it - you release it early to encourage participation. This especially makes sense for Unity because half of the picture is third-party developers implementing things like Quicklists and Lenses; and updating existing programs to use the Indicators API.

Unity is complete enough for the next five months. I've seen just as many rough edges in point releases of Gnome 2.

> Changing the UI completely is not "some small change". In fact it could be the change that makes Ubuntu no longer easy to deal with.

I wasn't saying that it was. It's a large change. But people have always been claiming "I'll move to Arch! I'll move to Fedora!" over small changes and most still don't move. In fact, just look at how Ubuntu 10.04 LTS is virtually revered now - whereas back in May 2010 it seemed like *everybody* was going to move to Fedora because they had their window buttons "the right way".

People always huff and puff about moving distros, but usually the biggest change they make is moving from one DE to another. Xubuntu is picking up greater market share because of Unity, for sure - but Fedora and OpenSUSE aren't.
