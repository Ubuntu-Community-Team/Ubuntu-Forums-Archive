### Post by jkxx on 2010-09-15
As far as apps that are holding ubuntu back, here's a few that I've noticed repeatedly. I'll try to simply list the problematic apps as this is not a troll, nor do I want it to become one.

(To those of you who say "why don't you code it yourself", please take a minute to consider what a poster is complaining about first before just throwing that out. For the record I can technically program but cannot actually do much of it help due to a condition of mine. Thus my help will have to be limited to finding and reporting bugs for now, sadly.)

Gnome: (Gnome is much improved in 10.10 vs what I encountered in 10.04) Upon seeing the sleek interface and the enormous amount of effort that was put into making Gnome as user-friendly as possible, I was shocked to find a number of very obvious bugs. 

The first one was the inability to set a default app for multimedia or much else. The dialogs were there but the same player would launch no matter what was selected.

The second problem was metacity/compiz's halved refresh rate - it was originally set to (I think) 60 fps, which halved produced a real refresh rate of 30 fps, which made the desktop appear unnecessarily laggy, including during game and video playback. Setting the refresh rate slider to double the real refresh fixes this problem real quick.

A third issue is that there is no shortcut for the CD drive (under places). Sometimes I have to open a terminal to mount/access CD/DVD media as there is no other way to get to it. Other times I do get an icon on the desktop and it opens the media just fine, although this doesn't always work.

Audio: alsa+pulseaudio, see my other complaints about this. I have often wondered why this is included at all given how broken it's been up to now, but the question is usually regarded with hostility so will not be asking that now :)

Lack of backward compatibility in APIs/file locations/file formats: this is self-explanatory but change for the sake of change is never good, and linux is unfortunately full of examples of this, some of which find their way into ubuntu. Any effort to standardize APIs and config file locations/formats or at least resist changing them will help massively as far as this being a problem.

And finally, one that is not really a fault of Ubuntu or even Linux: games!
Making games for only windows puts ubuntu at a great disadvantage, when OpenGL works just fine and can handle the latest gen graphics just as well as DirectX. Note that my compatibility complaint has been used as a reason for not porting many games to linux, although I suspect it's not the only or main one.

Anyway that's all I can think of for now, with kudos to the ubuntu developers for keeping at it with respect to improving ubuntu.
