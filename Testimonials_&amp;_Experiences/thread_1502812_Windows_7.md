Hi Orphanlast

I hope Windows 7 works out for you. A couple of points which may be obvious, but I think are worth making:

'Linux has no interest in ease of use':
If by 'Linux' you mean the Linux community, there are whole organisations (Canonical, for example) which pretty much devote themselves to this, so I won't dwell on it further.

'compatible with commercial software':
Windows, and other commercial software which has been built to run on it, costs money. Those who earn their living doing the sorts of things that these tools deliver may be able to make the case that these tools are worth it. I'm sure that many people buy these tools for the pleasure they get from them;  whatever, it is a personal choice and far be it from me to suggest how people spend their own money.

However I am incredibly impressed that there are free alternatives which I can choose to use.

(As mentioned previously, there are technical reasons which constrain what the Linux community can do regarding compatibility of these tools - that is not my main point. In fact, given these constraints, the fact that something like Wine exists, which allows many programs built for windows to run in Linux without the need for a Windows licence is amazing.)

On the VirtualBox subject:
As mentioned up thread, it is a commercial product (part of Oracle's empire) and, although it is open source, I suspect it is getting all the attention it needs without the significant input of Linux users (in their role as Linux users, rather than say Virtualbox developers who happen to be Linux users)

As the name suggests, Virtualbox is virtualisation software - it creates an 'imaginary computer' inside your real computer. This means you still need the Windows licence and other commercial software licences to use it to do what you suggest in your post. You really are better off running those products natively unless you want to take advantage of some of the benefits of virtualisation. For the home user the main advantage of virtualisation is to allow rapid switching between OS's and some ability to interact between two (or more) running instances of OS's.

From the perspective of keeping Linux distributions relatively lean, there is not much to be said for somehow 'integrating it deeply' into the distros as most people do not use it. 


Anyway, enjoy Windows 7, and perhaps keep in mind the fact that you can dual boot - have both! Maybe one day you won't be able to afford the commercial products and will find the alternatives are good enough, just as good, or even better!

You will be welcome back.
