### Post by Repgahroll on 2010-09-28
Hello there. I'm changing my main distro for basically half dozen of reasons. Ubuntu doesn't fit my needs very well any more.

Ubuntu is very aggressive in ease of use, but sometimes I just don't want to be bothered with massive easyness.

Updates should be better classified, because Ubuntu almost "force" you to install updates. I know they can be disabled, but it isn't good to be using unsecure apps, and Ubuntu almost every 3 days finds new "critical" updates (security-related), even on LTS versions. Why they don't choose better versions for packages instead of releasing lots of fixes for bad ones?

Updates mess some configurations, breaks compatibility, and when the app changes the configs directory it stays there forever. After 3 years of a new installation, my home folder has a ridiculous amount of hidden configurations folders. Maybe it's an APT problem, but should be priority to be solved.

The community is VERY bloated. It's impossible to get TRUE help on the forums / chat. The help you can get from community isn't much different than the one you can get on google instantly. So, if you need any solutions that isn't well-known, you can't. Need to figure out yourself.

Old hardware problems still unsolved. For example, Ubuntu 10.04 doesn't have laptop-mode by default anymore, and some laptops (pretty much every one with WD / Samsung HDs) are having the load_cycle_count issue (since the first Ubuntu).

Many non-free drivers for softmodems/wireless cards doesn't work. At least the ones installed using the "hardware drivers" helper. Some drivers are very bad, for example, the one for atheros B43 that has very little range, doesn't support leds, crash when disconnecting, so it's necessary to reaload it to reconnect, and keep the device always on (at least for pretty much every card on acer laptops).

And finally, the worse problem EVER in ANY LINUX DISTRO. CLI simply doesn't work any more on versions 10.04 or even 10.10 beta. I believe that silly plymouth eye candy is the cause, however, Ubuntu uses grub2 which is very new and isn't friendly to configure. The result is, even after 2 tutorials i couldn't get the CLI working on my main Desktop and now everything is messed. And It isn't possible to use CLI out-of-tha-box anymore on pretty much every nvidia/amd/intel/sis vga. And as peoples doesn't have much knowledge of grub2 for it's very new, Ubuntu is a real pain after version 10.04.

Ubuntu wanna be Windows imho. And Windows is a damn bad OS imho.

I'm switching to openSuSE (don't care if Novell is a damn bad enterprise), and if I figure out it's not good, i'll be using the good old Debian or Arch (because i plan to switch to KDE4 when it's stable).

After such experience is easy to understand why Mint isn't based on Ubuntu any more. Ubuntu is going the wrong way imho.

### Post by KdotJ on 2010-09-28
For arch booting slower, this is true (depending on the amount of daemons and modules etc you have to load at boot time). You may already know that Arch uses the older INIT method to boot, which you can see all of the things loading at boot time. Ubuntu on the other hand uses the newer upstart method where things load asynchronously and is covered over by the Plymouth splash screen. Arch Wiki has some great tips on how to improve your boot time and tweak your boot scripts. See you on the Arch forum lol.
