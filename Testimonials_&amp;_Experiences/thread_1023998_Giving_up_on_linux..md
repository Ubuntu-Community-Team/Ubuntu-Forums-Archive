### Post by gmjs on 2008-12-28
Hi marlocarloandro,

Hey - we all care about everyone's forum posts!  Personally, I've found this forum to be one of the best Linux-based forums.  I've had no responses at all on other forums.

Anyway, with regards to hardware support, I'd have to agree with you.  I'm without sound at the moment (I'm running Hardy, had a bad time with Intrepid and am waiting for the next stable release with kernel 2.6.25+).

Linux is great for converting your PC into a tool that can perform (selected) tasks efficiently and reliably.

I still run Windows (Vista).  After all, if I bought some software or hardware at a shop, I want to have an operating system they'll work with!  But I do turn to Linux for looking something up on the net, C and PHP programming, graphics work, and document and music typesetting.


**If you've had trouble with the Ubuntu Live CD, do consider the Alternate Installation CD.**

It's available for download by selecting the "[Text based alternate installer installation disk]("http://www.ubuntu.com/getubuntu/downloadmirrors#alternate")" hyperlink on the Ubuntu download page.  The installer's text-based, but it's no more difficult than using Ubiquity.  Just watch out at the partitioning stage if there's any data you want to keep!


**From memory, Fedora's package management system's command line syntax is very similar to that of aptitude.**

Just try:

```
**#** yum install <package name>
```

instead of

```
**#** aptitude install <package name>
```

Fedora doesn't (or didn't) come with a **sudoers** file configured by default.  Instead of using **sudo** for installation, you may need to use the command **su** and the root password (rather than your own).  To use the **su** command, type **su** then hit return, rather than placing it at the beginning of the command (as with sudo).

```
**$** su
Password:

**#** yum install <package name>

```

Skype does seem to have tried very hard to get out a good Linux client, so I would expect it to be quite straight forward, as long as the support is there for the camera.

Hope this helps and good luck.

Graham

### Post by gmjs on 2008-12-28
The package unetbootin doesn't exist in the Fedora repository.  Nor does it in the Ubuntu Hardy repository.

However, it looks like it's just as easy to download unetbootin from their website instead.  I'm not running a Fedora machine at the moment, but this works under Ubuntu and should under Fedora too:

[LIST=1]
[*]Download unetbootin for Linux from their website (link below).
[INDENT]```
**[http://sourceforge.net/project/downloading.php?groupname=unetbootin&filename=unetbootin-linux-304&use_mirror=surfnet]("http://sourceforge.net/project/downloading.php?groupname=unetbootin&filename=unetbootin-linux-304&use_mirror=surfnet")**
```[/INDENT]

[*]Use yum to install the packages mtools and p7zip-full.
[INDENT]```
**# yum install mtools p7zip-full**
```[/INDENT]

[*]cd to the downloaded unetbootin binary file and make it executable.
[INDENT]```
**# chmod +x unetbootin-linux-304**
```[/INDENT]

[*]Finally, run it.
[INDENT]```
**# ./unetbootin-linux-304**
```[/INDENT]
[/LIST]

The only reason I know to install **mtools** and **p7zip-full** is that, if not installed, running the program produces two error messages stating that these two packages need to be installed.

Hope this helps.

Graham
