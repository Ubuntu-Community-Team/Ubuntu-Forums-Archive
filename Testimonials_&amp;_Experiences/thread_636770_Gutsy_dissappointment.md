### Post by DrMega on 2007-12-10
For about a year, I have had Dapper installed on both my home machines (a media centre and a normal desktop). I recently upgraded first my desktop then my media centre to Gutsy. My desktop has no special requirements so I was happy with Gutsy. However when I installed it on my media PC, serious problems began.

1. It seems that the 'SPLASH' option in the boot options file 'menu.lst' killed my graphics setup. I use the ATI driver for TV out, and before we all slate ATI, it isn't their fault. The ATI driver works exactly as configured now that I've removed the 'SPLASH' option from the boot file.

2. XDMCP seems to be broken. There is a workaround, but in Dapper no work around was necessary. It just worked.

3. Flash movies spontaneously stopped working. A minor issue I know, and not Ubuntu's fault, but by the time I got to this bit I was already a bit dissappointed.

All in all, I can't really see what Gutsy offers that Dapper didn't.

Now before everybody has a go at me, I know Ubuntu is free so I have no right to complain (and I am not complaining). I'm just dissappointed and a little bit worried that the fantastic distro that is Ubuntu may be going downhill from Dapper, and I would hate to see that happen as Dapper was ace (as indeed is Gutsy once you manually tweak various bits).

I agree with the points made about sticking with a setup that works, upgrading just the elements I needed etc, but I think my point has been missed.

My point is that Ubuntu is promoted as the Linux distro that 'Just Works'. I seem to recall reading once that it was 'Linux for human beings'.

I take this to mean that it is easy to setup and configure. While this is no doubt true in most cases, it certainly isn't true in my case.

I fully accept that Gutsy is new, and therefore likely to have teething problems (this is true of anything new in the field of technology). The problem is that of three issues I encountered:

1. The remote login issue is being discussed a fair amount on the forums so will no doubt be resolved at some point soon.

2. The Flash version issue has been discussed to a lesser extent and a .deb package is available for download - shouldn't this be in the repository so it is picked up by the automatic software updates?

3. The ATI driver settings being blown away during boot with the default 'SPLASH' option is a major issue, albeit likely to affect a comparatively small number of users (those who use TV-out as their primary display). I've posted details on this forum abou that but with no reply, I'm not entirely convinced that the issue will be recorded as a bug.

I'm not asking for anyone to sort out my problems, I am quite happy to work through them (as I have done), but if Ubuntu is going to continue to be promoted as the distro that 'Just works', it really should just work.
