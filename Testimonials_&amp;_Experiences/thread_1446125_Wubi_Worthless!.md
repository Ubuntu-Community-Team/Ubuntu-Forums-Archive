### Post by 3rdalbum on 2010-04-04
I would agree, don't use Wubi. If you install your Wubi file on a different hard disk (not C:) then it seems that you can't access that hard disk directly from within Ubuntu; this has confused some people.

But of course the biggest problem with Wubi is that, if Windows won't boot, then Ubuntu won't boot either because the Windows partition has not been cleanly unmounted. I've encountered this problem on a friend's computer before.

Unfortunately I think many newbs these days don't even realise that you can boot up the Ubuntu CD, and they think that the only way to install Ubuntu is to run Wubi.exe.

### Post by Jguy on 2010-04-05
I am using Wubi on my laptop to run 9.10 x64 and I've had only one minor problem so far. When Wubi was installed, for some reason, the ATi crapdrivers for ATi were installed with it (without the control center, mind you). All I had to do was uninstall those via Synaptic and reinstall them using Hardware Drivers. I've had no problems whatsoever. KUbuntu via Wubi would not run Wireless very well (kept dropping out) and the ATi drivers would not install via Restricted drivers, so no luck there, Xubuntu worked fine but I wanted Ubuntu. I've had no problems with it and it sometimes out performs my desktop which only has Ubuntu on it, this laptop is booting Wubi Ubuntu 9.10 x64 with the main OS being Win7 home Premium x64, it runs very smoothly and boots fast. Overall, a very usable idea. I at first was very against it as well, but after giving it a shot (I don't have a restore disk for this laptop, so I didn't want to install Ubuntu on it as the main OS with the chance that it wouldn't work with everything, that and I need a Windows machine)

Absolutely perfect with no huge issues.

### Post by bark50 on 2010-04-05
I became acquainted with Ubuntu through Wubi.  This was back when Ubuntu 7.10 was out, I believe.  Wubi worked well for me, so I took the plunge and put 7.10 on its own partition.  That was when I discovered that for whatever reason my 5-button mouse which worked so well on the Wubi Ubuntu would not work on the Ubuntu straight install.  So, by scouring these forums and sudoing this and gediting that (and terrified the entire time I was going to screw up something royally), I eventually got my mouse to function like it should.  A few months later I upgraded to 8.10, and the blasted mouse wouldn't work again.  And not only that, but the fix that got it working in 7.10 didn't fix it in 8.10!

The moral of my story?  Forget Wubi; put Ubuntu on its own partition so you can really butt your head up against any problems that might arise.  Ubuntu is like a relationship:  it's going to aggravate the devil out of you before you learn to love it.
