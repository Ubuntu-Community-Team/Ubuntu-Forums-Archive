### Post by forrestcupp on 2008-03-11
Well, the thing is that you don't really need to know all of that CLI stuff.  There is a GUI solution to almost everything that a normal user would do in the CLI.  The problem is that in the forums, people always give CLI solutions because it's easy for someone to copy and paste into their terminal and have it done.

Here are a couple of examples:

instead of
```
sudo apt-get install
sudo apt-get remove   or
sudo apt-get --purge remove
```
You can do all of that in Synaptic, which is a graphical way to install any program in the repositories.  It even categorizes the apps, and allows you to search for keywords.

instead of
```
sudo chmod +x filename
```
to enable the file to be executed, you can open up Nautilus, which is your file browser, navigate to the file, right-click and choose properties, click the Permissions tab and check the box that says 'Allow executing file as a program.'

I could go on and on, but the CLI really isn't needed as much today as it was in the past.  It's a matter of personal preference, and for support in forums, it's easier to just give a command that someone can copy and paste.
