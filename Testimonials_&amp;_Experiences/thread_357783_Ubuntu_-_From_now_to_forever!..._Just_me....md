### Post by mostwanted on 2007-02-11
Linux was born as an Internet project and it shows.

The only way to standardise a way to install software would be if
a) There was a common Linux base which distros sprung from
b) This base was solid and did not change for several years

Take a look at Windows XP to Vista. I've read somewhere that approximately half the applications that worked on XP won't work on Vista. The only reason that XP has such a large application catalogue, which can be readily and easily installed, is because XP has haa a solid base for 5 years (even Mac OS X has had a relatively solid base for 5 years now, they just added components did not change any radically). When Vista arrived, incompatibilities akin to those we see between different Ubuntu versions arrived with it.

To get software installation like Windows or Mac OS we would need to halt software development of the base system libraries, which is something we can't do. The only reason Microsoft and Apple can do it is because they control the hardware in each their separate ways (MS do it through their monopoly, Apple through their one-vendor-only strategy). Linux needs to improve its hardware support all the time. The other reason we can't is because no one owns Linux, it's a free project.

We need to develop the current methods of software installation, not strife to copy Apple and Microsoft, because it is something we can never do as well as they can.

### Post by 3rdalbum on 2007-02-11
Also, Windows and Macintosh don't have the problem of supporting multiple architectures. Linux runs on more than just your standard desktop PC - it runs on a whole lot of computers with completely different and incompatible CPUs. An iMac G3 won't run programs compiled for an x86 PC, and vice versa. Source code must always be distributed, because source code can be compiled for all architectures.

Having said that, I believe there should be a universal GUI-based source installer.

Since you're a pro, maybe you have the ability to create an easy-to-use source installer that doesn't require an internet connection or any terminal commands? If so, that would be a great contribution for you to make to Linux.
