### Post by libssd on 2011-02-07
Back to the original thread, I have always disliked the "Windows" way of doing things; riffing on Dilbert, Windows (any version) always struck me as Mac OS built by Elbonians.

My Acer netbook came with a 160Gb HDD, which I partitioned 2 years ago to run Ubuntu in a 32Gb partition. Initially, I used Windows not infrequently, because there were certain things that just had to be done in Windows. Any operating system has some features/programs that are unique to it; in almost all cases (except when mandated by one's employer), you can do without, and have similar, sometimes inferior, sometimes superior, functionality in another operating system. Ultimately, any OS shapes what one does, and you learn to work around its limitations, and to appreciate its benefits. 

For me, the longer I used Ubuntu, the less I used Windows, to the point that the only reason it got booted was to apply patches and security updates, so I removed the HDD and replaced it with a 32Gb SSD, which runs Ubuntu only. Every 60-90 days, I swap drives to go through the tedious process of applying all the Windows cruft that has been released since last boot.

I have a desktop iMac which is also getting used less and less. For me, Mac OS X is still the most integrated, consistent, and complete OS, but you have to operate within Apple's rules, and the more I use Linux, the less willing I am to do so. This is the true meaning of "free" in OSS. I have gotten to the point with Linux, especially Ubuntu, that I am now frustrated by the things that I *can't* do in Windows or OS X. The worst thing about Linux is that, because it is so flexible, you can end up slipping down a rabbit hole, and spend too many of your waking hours exploring options and tweaking things. I *think* I have evolved past that point, at least with Ubuntu.
