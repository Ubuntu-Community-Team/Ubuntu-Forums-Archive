### Post by gladgrind on 2012-12-09
Being a civilian when it comes to computers, I expect things to
just work. 

When I was running a pc with a pentium 4 or pentium 3 processor,
most linux distros that I tried ran without a problem.

But since I got this I5 Ivy Bridge machine, and have been running
64-bit distros, it is a different story.

Here is what I have been seeing:

I tried Mepis 11. It installed ok, but was so ungodly slow that it
was useless. I remember how it used to be an up-and-coming os that
I liked, but no more.

I tried Magiea 2 today. It installed nicely, looked good, very
polished. When I booted up after install, it went to a terminal
login screen with a message about having to run drxxx to fix
problem Z, and that I should log in and enter command y.

I did that, and it said I did not have permission. I tried to run
sudo to log in as root and it said I was not in sudoers.

At that point I said to heck with this and dumped it.

I tried installing Debian, which I recall as a fine OS. It started
to install but could not see that I had a dvd drive, even though it
had already displayed a splash screen and started the install.

I installed Snowlinux, which installed fine, except that it had
only one resolution to choose from, one that I can not use because
of my eyesight.

I used to run Mint 13, which I liked for its looks and speed,
except that with the I5 machine, it was freezing once a day or even
oftener.

When Mint 14 came out, I installed that and it ran very well, and
I did not see any freezing after running it for several days; but
when I ran clonezilla to back it up, clonezilla came back with a
'something went wrong' message. So I tried the 14.1 version but
got the same error. Then I tried the 32-bit version, but
got the same error. That was it for Mint.

So I went back to ubuntu 12.04. It runs well after I installed 
gnome-panel and got rid of the cartoonish Unity interface. It 
seems a tad slow compared to mint, but it backs up ok. I tried to
get rid of Unity, hoping to speed it up, but that caused problems
so I re-installed from a clonezilla backup.

I suppose this is a back-handed way of saying that ubuntu is the
best of a bad bunch, but it will have my reluctant loyalty until
something better comes along, which looks like it may take a while.

### Post by ToddCannon on 2012-12-12
Ubuntu does work. 

In the past I found Linux distros to be very interesting but that was all. 

They were not ready for prime time. 

Ubuntu comes closest to being ready for everyday use but still needs some tweaking.

I don't have a huge problem with the Unity interface but I would like to have the option of using a different desktop if I want to.

I don't like feeling as though I am stuck with Unity.

Another thing I would place high on my list of things is driver support. Windows 7, for example seems to work with almost every device right away.

I'll keep using it!
