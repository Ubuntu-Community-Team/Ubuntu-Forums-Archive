### Post by a0u on 2008-10-19
> **cardinals_fan said:**
> 
Malware is entirely a user problem.  A foolhardy user can get infected on any system.
Well, not entirely...
In any operating system, most of the time the underlying processes are out of the user's hands. Thus, the design of the OS has a substantial impact on the susceptibility of the system.  Under normal usage, I would say that it's a whole lot easier to tank a Windows installation than a GNU/Linux one, simply because Windows suffers from inherent flaws that the user cannot control, even if he/she was aware of them (i.e, I cannot prevent Windows from using remote procedure calls).

In contrast, GNU/Linux comes from a solid design, and probably the only viable way one could permanently break a GNU/Linux system is by a *voluntary* action - i.e., intentionally running a script or command.

This [white paper]("http://www.theregister.co.uk/2004/10/22/security_report_windows_vs_linux/") is somewhat dated, but the fundamental differences should still apply.

So, malware should also be considered a programming problem.  Despite how careful a user is, an insecure design will always be insecure.

Sure, we can't prevent users from downloading dubious files, but DLLs in %WINDIR% wouldn't be infected so often if Windows didn't run with root privileges by default.  And I don't see that practice changing soon, because many Windows applications must now be run under an administrator account to work correctly.

Exploitations are inevitably tied to the shortcomings of the system.  If M$ adhered to *nix security standards, malware would be much harder to code; without an easy target, the incentive to develop malware decreases, and there wouldn't be so much harmful junk to tempt unsuspecting users in the first place.
My point is that developer incompetence empowers user incompetence.

But, I do agree with you that saying there are absolutely "no viruses or trojan horses" is misleading.  What we should emphasis is the greater protection a user gains from GNU/Linux's design philosophy, and how much easier it is to implement safer security practices.
