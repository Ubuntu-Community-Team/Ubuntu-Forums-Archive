### Post by warjowuch on 2010-04-03
Just some tips, in case you/someone else didn't know

IN synaptic, there is an option called "remove completely" (inc configfiles etc)

It is a more thorough removal: sudo apt-get remove --purge <program>

ALso, if you have ever installed a program with a lot of dependencies coming with it, you can't uninstall them from GUI in one action (at least not as far as I know), but in terminal:

sudo apt-get autoremove --purge <program>

> **sixthwheel said:**

I'm so excited!!!

Well yeah, that's good... but why did you install a program that you'd just removed?

If you're having problems with a program that you weren't having problems with before, usually the problem is caused by a corrupt preferences file. Per-user preferences are stored in hidden folders in your home directory, so have a look there and clean out any that belong to that program. Then when you start the program, it will truly be at its default configuration.

Reinstalling the program doesn't touch the per-user settings, so yeah.

If you just reinstalled the program as a learning experience for the terminal, then congratulations and I'm glad you're having fun with your computer.
