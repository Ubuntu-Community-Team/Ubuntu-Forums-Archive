### Post by caravel on 2009-04-23
To try the OS out without installing/changing anything on your system, that's what the LiveCD is for.

Personally I've never managed to accidentaly wipe out my windows partition when installing any Linux in dual boot.

As the others have said, from looking at this forum this problably seems like a fairly common issue, though in truth it probably isn't.  It's usually down to people rushing in and not reading the instructions.  The partitioner is very clear:

1) There is the default option to use entire disk.  This means erasing everything, setting up a fairly standard set of partitions and reformatting.  Windows (or any other OS on the drive) = hosed.

2) The second is to use the free space (resize). This means you need to know how much space is being taken up on the windows partition, defrag it and then work out how much free space to give to the Ubuntu partition, while leaving some for windows.  This is the option to give you a dual boot and is easy enough for a windows IT professional or advanced windows user to work out.

3) The last allows you to partition  manually.  Again, any windows based IT professional or advanced windows user should probably go for the last option after reading up on mount points and swap etc - it's clearly a no brainer for those that want to manage exactly what goes where, what get's removed and what does not.  This is always how I install.

Those that don't make much the same interpretations from these options are simply not reading the instructions, are just clicking through carelessly or are not windows IT professionals or advanced windows users.

The partitioning stage is always dangerous if the drive contains data.  Check, double check, rethink and if still not sure; ask.  Ubuntu is very different to Windows so it does not matter how much you know about the latter.

To be honest this thread comes across as a typical "It's no good because it's not like windows" whining session.  e.g. the installation process enabled you to wipe out your partitions whereas windows would have asked you "are you sure?" several times.

1) There is the default option to use entire disk.  This means erasing everything, setting up a fairly standard set of partitions and reformatting.  Windows (or any other OS on the drive) = hosed.

2) The second is to use the free space (resize). This means you need to know how much space is being taken up on the windows partition, defrag it and then work out how much free space to give to the Ubuntu partition, while leaving some for windows.  This is the option to give you a dual boot and is easy enough for a windows IT professional or advanced windows user to work out.

3) The last allows you to partition  manually.  Again, any windows based IT professional or advanced windows user should probably go for the last option after reading up on mount points and swap etc - it's clearly a no brainer for those that want to manage exactly what goes where, what get's removed and what does not.  This is always how I install.
