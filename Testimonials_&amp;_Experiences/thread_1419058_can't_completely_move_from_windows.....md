### Post by 3Miro on 2010-03-01
Faxes and Printers and things of that category still would not work often enough. Manufacturers don't make drivers and there is just way too much stuff out there for developers to keep track of all of it.

I had some trouble with my printer and I resolved it with VirtualBox. I am assuming the Fax machine connects to the USB. You can download the VirtualBox non-open source edition from their web-page (the version in the Ubuntu repos will not work for this), download the manual (a very easy to read and follow and you only need a few of the many topics in it) and then install some old copy of XP or Vista. Then you can run Ubuntu and Windows at the same time, using Windows for Fax and Printing and Ubuntu for the other things.

### Post by cariboo on 2010-03-01
Everything you are talking about can easily be fixed. There are database connectors to allow you to connect to an Access database. Networking is simpler running a Linux variant, than it is in Windows. There are now gui tools to setup file sharing using Samba. Have a look at this [document]("https://help.ubuntu.com/community/DialupAndFax") for help to set up faxing. The only fly in the ointment, is hardware that isn't very well supported, and that is more the fault of the manufacturer than Linux.

The main things is you have to spend almost as much time learning to use Linux as you did learning how to use Windows. Windows skills don't transfer very well to Linux, as the system is totally different, so even if you have been using Ubuntu 3 to 4 months, that isn't enough time to learn the ins and outs of the system.

Think of it this way. How long have you been using Windows, and how long did it take to be thoroughly comfortable with it. You can then estimate how long it will take you to get thoroughly comfortable with a Linux variant.
