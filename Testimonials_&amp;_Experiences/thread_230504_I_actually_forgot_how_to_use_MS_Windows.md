### Post by cptnapalm on 2006-08-09
A few years back, I only had Linux on my machine, but I wanted to play this new game, so I got out my dusty Windows CD and configured to dual boot, which Windows did not make easy.

I was having some problem and I needed to find a file in the system folder.  Long used to a Unix filesystem where I pretty much know where most things are intutively, I had to navigate my way in a now foreign filesystem.

Getting into the Windows folder, I was aghast.

"Its an atrocity."

A complete mess.  There were library files and configuration files and executables and every other thing, all in the same folders.  And not just a few, but hundreds upon hundreds of unrelated things all dumped irrationally into the system32 folder.

Later I had to find the folder with the game in it, to edit a configuration file.  I was at a complete loss.  I knew to go to C:\Program Files.  That wasn't the problem.  The problem was that there was no folder with the name of the game.  I had to go into each potential candidate's folder to see if the game folder was inside.  Eventually I did find it, in a folder under a folder named after the game's publisher.  THE PUBLISHER!!!!  To find my game I have to know who published it?  Utterly stupid.  The game was in a subfolder... that of the company who wrote the game... I had to know that too?

After going through all of this, I was telling a Windows using gamer friend  about my trials and tribulations and the HOURS it took for me to find the TWO files I needed to mess with.  He laughed and said that is why he made a C:\program files\games folder and installs all his games there by default.

It was my turn to laugh as my never-touched-a-Unix-in-his-life friend had basically thought in Unix terms as far as filefolder layout goes.  The Unix style of system layout is so much more rational that people come up with it on their own in trying to avoid the idiocy involved in installing stuff on Windows.
