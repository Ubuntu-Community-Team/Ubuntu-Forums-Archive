### Post by mauldor on 2007-10-28
**Compaq nx9105**
This laptop has Athlon 2800+, NVIDIA GeForce4 420 Go, 512MB ram and built in Broadcom Wireless of some sort. 
Getting 3D to work involved reading various guides and commenting out the 65535 memory line from Compiz, the wireless just worked and everything ran fine. The 3D part was fast enough to work with and could add reflections and so forth without massive slowdown. 

**Dell Latitude 840**
This featuring Pentium M 1.8 with the same NVIDIA GeForce4 420 Go as above (or so you would think) but after struggling to get 3D working, it was so slow it was almost Frame by Frame. How can the same Named Graphics Card and following the very same process as above yet it was un-usable?? I gave up in the end to get a PCMCIA Linksys WPC54G Wireless card working even after following many guides (boo Hiss Broadcom eh). 

**Athlon 3700+ Desktop**
This I thought would do the trick for the old 3D Desktop as it had a ATI X300 Card in there but here we went again with searching to death (thank god for having more than one machine) to find the answer. I got it working you might say but this really was Frame by Frame Movement, White Screen after trying one thing and eventually gave up. 

**Summary**
My Goal was to have a machine with 3D Desktop etc but no matter which machine I pick that I use, there is either a problem with the graphics Card, problem with the Mouse (MS 6000+ Set moves WAY fast) or Networking. I am lucky in that I am technical and can follow as many guides as needed but  the average Joe will have given up long before now. 

For now Ubuntu lives only on my Dell Laptop as a LAMP Server (The Compaq is a Works Laptop) - the rest have been reverted back to a "Working" O/S....
