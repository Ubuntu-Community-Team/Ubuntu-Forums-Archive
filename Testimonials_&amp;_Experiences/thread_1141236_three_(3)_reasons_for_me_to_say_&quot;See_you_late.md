### Post by Dyqik on 2009-04-28
On the version numbering of the network manager:

Free and open-source software usually has a very different approach to version numbering than propriety software, where the pressure is to get version 1.0 out of the door and on sale, despite the bugs.  For free software you should probably remove the first decimal point to compare version numbers directly (this might be a little bit optimistic, but I've used pretty flaky 2.1 or 3.2 version propriety software).

Generally a version 0.<high digit> version number like 0.7 or similar means that not every feature that is desirable is implemented, but that the software works well for a subset of the desired features.  Bugs tend to get worked out fairly quickly for each 0.n.m release.  Personally I find the network manager applet completely stable for wired and wireless lan and for VPNs since version 0.6.2 on several machines, including my main work desktop, home desktop, compute servers, telescope control systems and laptops.  I don't have a 3g card, so I can't comment on that particular application, but I wouldn't worry about using the applet if I found it worked.

There's nothing magic about a 1.0 version number (actually I'd be really suspicious of a 1.0.0 version), as the version numbering is set by whoever's releasing the code.

### Post by Kareeser on 2009-04-28
You, sir, like many other Windows users, have been trained to associate any version number under 1.0 to be inferior. This is simply not the case.

Whereas in Windows, developers are pushed to make a program that does EVERYTHING (and thus, version 1.0 is reached quickly, but all of the functions are half-assed), Linux developers make each program do one thing, and to do it well.

So if they managed to create a program that does one thing well, and it does it perfectly, why does it have to be v1.0? There's no speciality associated with that number, only what YOU perceive :)

Case in point, take a look at the package "mousepad". Quick, lightweight text editor, akin to Notepad.

It is version 0.2.

### Post by joeinbend on 2009-04-28
Ubuntu, or Linux in general, is not for everyone. You must see the value in fine granularity of fine-tuning, and the satisfaction of doing something yourself. If everyone said "Well, it doesn't work, back to Windows", then no community-driven OS, or software for that matter, would ever get off the ground. You have to be willing to work through problems, and share the results with the community.

I have an eeePC 1000h. When I first got it, I loaded it up with 8.04 and worked out some bugs and was very happy with it. Curiosity got the best of me with 9.04, and I decided to go for it, once I verified my hardware running the Live CD. The visual improvements are awesome, and of course I have a whole new batch of minor tweaks to work through. Maybe I'm sick, but I love solving those problems. 

This week I went for a clean install of 9.04 Netbook Remix, and I do see the value in its' interface, however I found the vanilla kernel to be more stable, for my particular hardware (my 1000h is highly modded -- Bluetooth, USB hub, GPS and Verizon 3G all hard-wired "under the hood"), so I'll probably be going back to that, and reporting my findings to the forums here.

The point is, "there's no such thing as a free lunch", and Ubuntu us no exception. You will have to work through little things, but you will also find one other truth that you do not find in Windows: "There's really no way to do that" is a Windows statement, not a Linux statement.

If your Ubuntu works straight out of the box 100% flawless, awesome... you're the exception that proves the rule :) To the rest of us testers, developers, and innovators, we'll keep modifying .conf files and tracking down oddities in hardware device ID's to get our systems tuned in.

One last comment about Ubuntu: My wife is not great with computers. She gets online to check her e-mail and find recipes, or read the latest Justin Timberlake gossip. I gave her her own laptop recently, running 8.04, and she NEVER has a seconds' trouble with it. Very user-friendly, and you don't have to be an engineer to get the thing to work. That's a testament to how far Linux has come in the last few years.
