### Post by bamboomy on 2009-10-16
Dear All,

I recently assembled a new pc (the first time I did this, ever) and happily went on after some beginner errors to insert the bright, shiny, freshly burned ubuntu 9.10 beta into the cd tray, closed it and was going to install my (I stopped counting after 10)th Linux.

I am, as opposed to the 'fresh' status this forum gives me, around in the Linux world since quite some time already (almost a decade now). I started off buying (you read it right, buying Linux) a copy of RedHat at a local bookstore to give it a try. That attempt quickly stopped after calling in the help of 2 friends both claiming to be able to install Linux on any pc.

Later on I gave it another try after moving to a corridor which was the habitat of a real Linux freak. He patiently learned me to work with Debian (this distro should sound familiar to you guys ;) and later on Gentoo. (The first thing I did was log in to his box and run 'startx', gives a funny result, you should try it some time :D. (I'm not responsible for angry box owners or possible seizures...))

Anyway, I'm not going to tell you whole my life but just the Linux part of it...

Back in those days you needed to write your own X config file. I bet that 90% of you guys don't even know what 'X' stands for but hey, modern times, modern knowledge, no?

(writing an X config file was dangerous because it could kill your screen. Killing in the sense that you would need to buy a new one afterwards, really killing it, screens where also half the price of a computer system so dangerous it was)
(No X basically meant not being able to use your mouse, not because it was not recognized or the sort, no, simply because there were no things to point and click, (all text baby :))

The Linux freak graduated and I was on my own again in the big Linux world. I stopped using Linux by lack of internet at that time. Later on, my father had heard the 'Linux' word quite often come out of the mouth of one of his sons in the mean time (guess which son, you get one shot ;) and he gave me a set of SUSE cd's he got from a friend.

Witnessing my first graphical installation on Linux was quite the culture shock but after messing around with Yast for some time a friend challenged me with the words: "You'll never be a true guru until you use Gentoo." After which I promptly performed, single handedly, my first full Gentoo installation. (Back then you didn't got the stage3 for free, you had to compile your compiler, compile your system and after that compile your software, back to the basics that was) you also had to think before you did, hard some times, luckily you had a lively community which helped you out with almost everything, not really willing to refer to myself as a non human being it was quite the opposite of Linux for human beings back then...

To conclude the story of my life part of this post (quite the intro I must admit) I later on converted someone who showed off his cracking skills to a hacker in the only and most beautifully sense of the word (a non human being as you will but even though a good person).

Why posting this story on an ubuntu forum, you might ask, the remainder of this post will enlighten you and the tone of the post might shift from nostalgic to somewhat pessimistic, maybe even bitter. (Is this the 'becoming older' part of my life?)

There is some hope at the end though...

So, to come back to my first sentence, I injected the bright new Ubuntu 9.10 beta cd in the tray and booted.

At first all seemed to be going well.

I glanced at the new features on the 9.10 release on the site and it was all pretty promising, I enjoyed the graphical installation screens which excited me more then ever whilst installing a non-Microsoft OS.

I was happy.

After the installation I rebooted, my system ran through the bios and I got a message 'no disks found'.

Panic.

At first I blamed my computer assembling skills, who would guess that an Ubuntu CD, assumed to be well-tested by many skilled people would fail, it must be me.

Being the persisting pain in the *ss that I am (and being too lazy to do a profoundly search for that long lost RC of Windows 7 a month before the official (paid) release) I burned a 9.04 on my laptop and inserted that one into the tray.

Age old grub1 seemed to be able to handle my SATA disks without a glitch. I even, being naive, formatted my partition that later on would be the habitat of a 9.10 upgraded system to ext4.

1) grub doesn't work

Before, when the software was still in alpha I burned a cd and ran that on my laptop to install Ubuntu inside Windows, which failed also at booting.

For the love of God, if there is one single thing an OS must be able to do.

And I can not possibly stress this enough.

Is.

Boot!!!

If the OS can not boot, well, 
_THEN_
_YOU_ 
_CAN'T_ 
_USE_ 
_IT_.

So the first thing that you suspect the good, brave and noble people of the Oh so human distro would do, if it is reported that the OS doesn't boot in alpha, that it does boot in beta.

Or, my conception of the world might be wrong. Am I the sole person on the whole world who has tried to download the alpha (or beta for that matter) and tried to actually boot it? Or, it seems impossible to me but, hey, who am I, did they all say: oh well, it doesn't boot, but, I will continue using it anyway...

Which brings me to point 2:

2) Over complicated bug reporting.

Ok, so I found a bug in Ubuntu, being the (still) positive person that I am I tried to file the bug (yes suree).

But I couldn't.

Oh, there were other bugs, and I could file them, but the boot problem, no, you had to give all sorts of information (that I didn't have) and it wasn't possible to file the bug (maybe that was the reason it wasn't solved? Who knows?)

Filing bugs seems to be one of these:
a) You almost have to know how to fix the bug yourself
b) One of the many things that's wrong with the beta crashes and you are guided into a huge system which eventually solves those bugs (or not).

Which brings me to point 3:

3) The beta isn't stable.

Ok, the beta isn't stable, duh, it's a beta. Yes, but it's not an alpha anymore.

There is one particular problem that just seems to not die: audacious2 kinda scrambles my mp3's, mostly at the start, ok, I want to file that bug. I look in the documentation how to file it, "click on the menu bar" it says. Audacious2 doesn't _have_ a menu bar, impossible to file the bug...

I updated my Ubuntu almost daily but no succes until present moment...

Things also keep crashing, luckily you have the 'do not report crashes from this package' checkbox otherwise I would have been angrier then I now already am...

...with apparently no solution, 2 weeks before the official release date...

A more serious problem is that after the last update my system is just broke. It keeps giving some kind of error indicating that something is wrong with my disks (I guess), I'm going to install 9.04 (again) but am doubting that the hardware is the problem this time...

(booting with an older kernel didn't pose the problem but I use the NVIDIA module (silly me) which isn't compatible across kernel versions and I don't know how to drop into a shell...)
(And even if I drop in a shell, Ubuntu masks everything so I wouldn't know how to compile my own NVIDIA module).

4) Ubuntu becomes more Windows-like

A bold statement, technically totally wrong (I know), maybe better rephrased as 'Canonical becomes more Microsoft-like'.

Not in philosophy (Microsoft is evil and all that, heard it, done that) and Canonical still fully supports the free (as in beer) philosophy, but it seems to me that Ubuntu is becoming the victim of its own success...

Back in the days all of the distro's had their fans, each being a relatively small group of computer enthusiasts believing that their distro was the best distro, all united in the Linux philosophy and whilst competing, though all supporting and being part of the 'big', happy Linux family.

Then came Ubuntu: "Linux for human beings", user friendly, not something you could say about any other distro, and with soon a big community helping out the poor human being in his/her first steps into the big scary Linux world.

As exact numbers are hard to estimate, Ubuntu is considered the largest open source operating system in the world.

Which comes at a cost.

More users, higher volumes, Ubuntu is facing a problem no other distro has ever faced before: too many users.

The dream every other distro always had is turning into a terrible nightmare for Ubuntu: a Linux distro, because it's popularity, is becoming like the arch enemy of Linux: the Big, Evil Microsoft Windows.

More users means you have to automatize the big input users give into managed solutions, no more hand-helping, no more mailing the writer of a script or application, no, Ubuntu now has a closed bug filing system that collects and manages solutions to this problems, problems not getting into the system don't get solved.

This is not the only resemblance...

5) Ubuntu becomes more and more monolythic

Partially overlapping with the previous point this is getting disturbing.

OK, other distro's emposed their likings and dislikings into the software they made available Ubuntu takes this to a higher level...

Where in other distro's it was quite easy to 'bake your own' installing software on Ubuntu with the good old 
./configure 
make 
make install

just don't cut it anymore.

Every little piece of Ubuntu has become part of a very complicated puzzle that looks very simple.

If you want to use, I give two examples:
a) Boinc with CUDA support you have to do more effort on Ubuntu then on any other distro.
The first (and logical) thing you do if you want to install a package is look it up in the handy (I must admit it is well placed) add/remove software option in the start menu)
If you don't find it there, you might use synaptic, I prefer going to the terminal right away (practice the command line commands I still know, many I have forgotten in the mean time).
Once (Boinc in this example) is installed, you run it, it is well integrated and everything (châpeau on dit en français) 
but...
If it is not totally like what you want (it _is_ the exact software you want, not the right version) you have to reinstall Ubuntu to clean up the mess (sounds familiar?).

Other example:
I have a thing for screensavers (addiction might be a better suited word) and I wanted to write my own, Xscreensaver, or better said, some mangled clone, needs to be de-installed and once you have every bit extinguished you want to lock the screen, doesn't work anymore.

This is even worse than Windows because you don't have to break windows functionality to be able to use your own choice...

So far my rant,

Conclusion:

I'm going to insert the cd I just burned into my pc, (my boinc stats are going to have taken a deep dive) reinstall ubuntu, maybe, if I'm not to tired yet again update to 9.10 (possibly against common sense) and I'm going to definately _buy_ the evil windows 7, if I feel like it I will download the release of 9.10, if it doesn't boot I will still try to update the 9.04 I'm now going to install

and I will pray

that Canonical and Ubuntu will handle things better then Mike and Soft did.

Thanks for reading...

S.  

ps: writing this post gave me some ideas for solving some of the issues, but others might find your input valuable

make

If the problem you are having is with dependencies, you can always search for dependency names with:
aptitude search <dependency-name>
or use Synaptic Package Manager if you want a graphical solution. Also note that some of the required dependencies start with the letters 'lib', even though the stated missing package name doesn't, and be sure to also install any 'dev' packages under the same names. I'm very new to compiling, but now that I know how to search for dependencies I am not having any trouble anymore. Some, if not all, of the building packages are not installed by default too, so installing build-essential usually helps get things started.

Any beta is bound to have some compatibility issues with some computers/hardware. On top of that, most hardware is built to run on windows. These two links below will help determine if your hardware parts will work with Ubuntu. ### Post by Vined Adobo on 2009-10-30
I also wondered where freecell went when I upgraded to Ubuntu 9.10  I found this bug on Launchpad:

[https://bugs.launchpad.net/ubuntu/+source/gnome-games/+bug/433330](https://bugs.launchpad.net/ubuntu/+source/gnome-games/+bug/433330)
Michael Martin-Smucker wrote on 2009-09-28
Just to clarify, Free Cell can be run with the command "sol --freecell", the problem is that there is no launcher for Free Cell

Okay, let's add a launcher.  Right click on applications in your upper panel.  Choose Edit Menus.  Click on games.  Click on + new Item
and you should get a screen to create a launcher.  Type should be application.  Name: type in "Freecell" (don't need quotes.)  Command: type in "sol --freecell" (no quotes.) Comment: type in anything you want--or nothing at all.  Your choice.  Click on the bolt icon in upper left of create launcher dialog to get browse icons dialog.  Click on down arrow on to get /usr/share/pixmaps.  Click on aisleriot.xpm to get logo.  Click OK.  Click OK again.  You should now have menu item in games for Freecell.  Click that to play freecell.

This worked for me.  Comments are always welcome!
