### Post by johann_p on 2006-05-31
Well, I had been using Suse and then OpenSuse 10.0 for years now. I have given previous versions of Ubuntu a try several times and always was shocked to see how much the hype differed from the shortcomings experienced in reality  (especially the installation procedure which was an embarrassment). 

Now I tried Dapper and I was quite pleased: installation from the Live CD is the obvious way to proceed and I am sure most distros will eventually provide this. Unlike in previous versions the installation is now much better, especially if messing with (existing) partitions is necessary. 

Overall, there are enough things that make me stay with Ubuntu so far, though there are still a couple of things that are better solved in Suse (or at all present). I like the way how Suse has nearly all configuration options combined under yast - even for things like Apache or NTP. I like the fact that Suse provides and interface to its firewall out of the box while I needed to install it for Ubuntu -- and that included figuring out what that interface is in the first place.

I do not like the way how Ubuntu divides itself into two distributions just because there is GNOME and KDE -- Suse's approach to simply allow both in parallel and to allow the user to easily switch between those and the different supported session managers is less confusing. 

The main advantage of Ubuntu is its package management system and the fact the repositories are faster and more up-to date. I like the fact that if you want, you can stay up-front very easily while Suse is more following the usual release cycle procedure. 

There are still a couple of things I have done in Suse but not yet in Ubuntu (setting up Mysql, installing more exotic programs etc.) but so far, I am quite pleased with the experience.
