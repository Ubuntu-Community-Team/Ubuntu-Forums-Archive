### Post by tyggna1 on 2007-08-25
I know how you feel.  I tried Mandrake years ago, and that was supposed to be the most "user-friendly" at the time.  It took me a solid week to get the wireless card working.  After I figured that out, I wasn't too impressed with the OS, in general.  

Red-hat was waay over my windows technical mind.  

Then I heard about Ubuntu.  As for "user friendly," it's a myth, but it's important.  User friendly is whatever you're familiar with.  In my case--the case of a windows to linux convert--that's a GUI with all the options being shown in a graphical interface.  Linux is a really powerful hybrid of a GUI and a CLI--you can do a lot in either.  Whatever is more comfortable.  Since I had some experience with DOS as a kid, I find myself using the CLI a lot more often.  

More and more people will design a GUI for linux--if it makes things easier.  I mean, for me, it seems easier to do a google search, read a forum thread, read through a spreadsheet, and then type in modprobe -i orinoco_pci on the command line rather than try to find which driver I have on a pre-made-but-not-comprehensive internal list--or rely on a large database that scans and compares my card with usable drivers (the windows method).
To me, it's worth saving the 2mb of space (multiplied by the number of types of hardware components, and then factor in the number of vendors who produce those components).  

Linux is user friendly--it just depends on the kind of user you are.  GUI's will come.
