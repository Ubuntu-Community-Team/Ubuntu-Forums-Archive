1. The old addage Linux is not Windows applies.

If you wish, there is paid support available for Linux, RHEL and Ubuntu offer such options.

2. Another part of your problems is you are unwilling to learn new ways ;)

It is much easier to solve a problem on Linux using the command line.

```
sudo apt-get install package_1
``` works on all versions of Ubuntu, in one line of code, and if it fails we get an error message. Now you may not understand the error message, but posting the error message will help us to help you.

Now imagine describing the same action on Kubuntu, Lubuntu. Xubuntu, and Ubuntu ...

Worse, the interface in Ubuntu changes with each release, so the description for 10.04 would not be the same as for 11.10 (which is in pre-release).

Worse if the graphical application fails, it may or may not give an error message. If it does not the graphical method is at a dead end.

3. Part of the problem is that the Ubuntu wiki is community maintained. Are you willing to donate your time to writing and maintaining such pages ? If not, what gives you the right to complain that they are lacking ?

4. Finally, it takes time to learn a new OS. Yes the command line is intimidating at first, rather then spending your energy ranging, spend your time and energy learning to use it.

[http://linuxcommand.org/](http://linuxcommand.org/)

Learn to read the man pages ;)

Every OS has it's unique lingo, Windows has the registry for example.

You need to be willing to learn the language if you want to talk the talk.

5. I see this is your first post, perhaps if you took the time to ask a question you would have more success.

6. Learn to use google. Plenty of screen casts and graphical pages out there.

[https://help.ubuntu.com/community/GraphicalInstall](https://help.ubuntu.com/community/GraphicalInstall)

[http://screencasts.ubuntu.com/](http://screencasts.ubuntu.com/)

### Post by Tamlynmac on 2011-07-19
> MG&TL

I agree. We should endeavor to enhance support whenever possible. However, one should keep in mind the varying levels of competency.

Not all users may even feel comfortable installing an OS, much less performing the setup and configuration. I don't agree with statements referring to the so-called "average user". It's been my experience that every user is somewhat different with varying levels of computing experience. Assuming that each user's knowledge level meets or exceeds a certain degree of competency - may be a false assumption.

IMHO one on one assistance significantly increases the potential for success. When users attempt to migrate, it's beneficial to engage another experienced user and request individual support. I've met few Linux users who wouldn't gladly provide hands on assistance if requested. Historically, I've assisted numerous friends, family and associates with excellent success stats. Think of it as a mentoring program, providing support even after installation.

Those users who come here requesting assistance should understand that the support sections of the forums are voluntary. They are under no obligation to provide assistance. Additionally, support may vary depending on the member that responds. There is a learning curve that all users must invest in. If a user has no desire to invest in said curve, then I always recommend they remain in the environment in which their comfortable.

When considering written help, we should all keep in mind the level at which help is written and how a new user would interpret it. Rarely if ever, have I read a well written help manual. They all seem to be written either in a foreign language or by someone who forgot what it's like to need an extended explaination. Not to mention, terminology that seems to target individuals who already understand the lingo.

Just my $0.02
