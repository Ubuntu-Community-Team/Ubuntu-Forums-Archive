### Post by wolfen69 on 2011-08-05
Not every OS will work on every computer, considering how many 1000's of different hardware configs there are. You just need to find what *does* work on it, and not worry about what *doesn't*. Mint didn't work well on my lappie, but ubuntu did. Oh well, it is what it is. But by no means does it mean mint is unstable.

Windows can be unstable too, depending on the hardware in question. I see it all the time, as I repair and sell windows machines all the time. You just need to match the OS to the hardware and move on. There is no such thing as a bad OS, just bad match-ups between hardware and software. But what the heck do I know? I've only installed *every* os (distro, mac, and win) on every kind of computer you can imagine. Not trying to brag, it just helps my point.

Find what works for *your* computer, and be happy you have choices. :)

### Post by el_koraco on 2011-08-05
Yes, but in your case it's an issue of hardware-software inconsistencies, which can probably be solved. 

Open a thread in the Absolute Beginners Talk subforum, and when you do, append the following information:

open a terminal (CRTL ALT T), type ```
lspci
```, copy the output and paste it between the code # brackets. Do the same with ```
lshw
``` and ```
lsusb
```
