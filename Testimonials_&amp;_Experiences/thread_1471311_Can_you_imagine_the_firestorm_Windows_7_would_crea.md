> **dnguyen1963 said:**
> Done that...did not solve the problem.  The only way that seems to work so far is to select "Flying Ubuntu Icon" in the screen saver.  Selecting anything else will lock up the computer.


On a side note before I get started, that was the worst possible thread title you could have picked to get support.  Insulting a product that people are passionately supportive of, then asking for assistance does not a good impression make.

Now, on to your problem.  I have seen issues like this being due to video card issues.  More specifically older cards that are clogged up with dust.  What happens is that the newer screen savers tend to push the cards a bit more.  This, along with clogged heat sink fins, cause minor issues that seem to show up in screen savers.

The first thing to check would be under System->Administration->Hardware Drivers.  If one is available pick it.  You may also want to step back to a different version depending on the card you have (Thinking nvidia here)

Next, crack open your case and see if it needs to be cleaned.  If you never have done it before then it is probably a good idea to take a can of compressed air to it.  Take it outside first or you will kick that dust into the air.

If that does not help we will need information on the type of computer you have, type of video card, version of Ubuntu (x32 or x64). and if you did an upgrade or clean install.

### Post by frappe1 on 2010-05-03
Windows is just an operating system with a browser and media player.

These forums are filled with bugs not just in the OS, but the literally tens of thousands of pieces of software installed through the repos.

Not only is there so much software, there is also so much diversity in software. Threads show up when program X doesn't interact with program Y on OS Z, with each component coming from a different developer. Microsoft isn't responsible for the remote management of your torrent client running on FDE and tunneled over SSH. That sort of problem doesn't even appear on Microsoft's radar.

The systems here are also disproportionately custom compared to the typically-OEM hardware Windows runs on. These systems have been built, rebuilt, overclocked, abused, and everything else possible. If someone has problems loading Lucid onto an espresso machine, you'll read about it here.

Look at all the NVIDIA problems with Lucid. Microsoft isn't responsible for developing unofficial drivers. All they do is bundle whatever a manufacturer can prove to work properly.

With regard to other drivers, and with the exception of purpose-built servers, I've never had a Windows system install with full driver support. Ubuntu/Debian Linux almost always provides full driver support out of the box.

I'd venture a bet that Boeing has more problems per unit than Ford. That comparison is irrelevant, due to the differences between the units.
