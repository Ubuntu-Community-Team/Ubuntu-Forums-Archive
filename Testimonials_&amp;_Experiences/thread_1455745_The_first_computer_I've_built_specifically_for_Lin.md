### Post by donovan1983 on 2010-04-16
A few weeks ago I got fed up with Mac OS X (again) on my Mac mini and put Ubuntu on there. I was quite happy with it but and although it generally worked well, that thing was obviously not optimal hardware to run anything but OS X on. Not to mention that since it was a late 2007 revision unit it was quite out of date. A 1.83GHz mobile Intel Core 2 Duo and GMA950 graphics is fine for basic use but not much more.

So I decided to just build a new PC, and one optimized for Linux. Luckily with the current broad hardware compatibility in the Linux kernel it is quite easy to build such a PC. I ended up with an AMD Phenom II X3 720 CPU and MSI 785GTM-E45 motherboard and have no issues using it with the Ubuntu 10.4 beta. Even the integrated ATI graphics were painless to get working. This is the first time I've ever focused on getting hardware that is compatible with Linux, though, so a bit of a milestone for me there.

The only real issue I have is not a Ubuntu-specific issue, but an issue with (at least) the 2.6.32 version of the Linux kernel having severely broken ACPI and APM with some hardware that worked fine in older versions. I cannot suspend or hibernate this thing and even shutting it down causes it to hang at a blank black screen. This seems to be an issue more than a few people are having in Lucid and other distros that are using the 2.6.32 kernel. I'm really hoping this will be fixed since I know this new machine will eat up far more power than my Mac mini if I leave it on all the time and I prefer suspending over powering off my machines. I guess for a bit I'll have to eat some extra electricity cost, though :P

I guess a big point I was going to try to make is that using Linux is not using a second-rate operating system at all. Ubuntu especially is quite user-friendly and polished out of the box. I get to use a lot of software I'm familiar with, like Google Chrome, the GIMP, and OpenOffice.org, and have found a good music player (Banshee) and other software that is useful. I used Mac OS X for a long time so am used to using software that may not be as popular with most computer users due to lack of availability on other platforms. Ironically, I can use more software that has quite good Windows ports in Linux than I can on Mac OS X despite Mac OS X also being Unix-like. So I have more in common with Windows users than Mac users. Of course I have VirtualBox and a Windows XP virtual machine installed, just like in Mac OS X, which in Linux I only need for Netflix and syncing my iPhone. This is a real, usable operating system as suitable for everyday use as Windows or Mac OS X. I believe it was the new COO of Canonical that mentioned in a CNET blog that the year of the Linux desktop has already came and gone, and I have to say he's right about that.
