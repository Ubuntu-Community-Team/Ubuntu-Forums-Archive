### Post by solwic on 2008-07-31
This is more of an experience than a testimonial, I guess.  I've been using Ubuntu for about a month or so now (steadily, anyway...have tried to get Ubuntu going since Dapper), and I can honestly say I like it.  It's a great OS.  

I posted a while back about how naked and vulnerable I felt using Windows online, and how that made me realize that my perspective had changed; I no longer thought of Ubuntu as "that other OS".  It was my default, my go-to OS.  

Tonight drove the nail fully into Windows' coffin, however.  

I think all new users are intimidated at first by the command line; I was, and I had a little experience with it from college.  When I first installed Ubuntu about a month ago I had resolved to learn the basic commands, choosing to accept the command line as a necessary evil rather than gripe about its continued existence.  

But tonight...was...magical.  I'm a writer, and I decided tonight that the bells and whistles on my main desktop were distracting me a lot from my work.  I didn't want to overhaul my main desktop (I love my setup), as it's still great for everything else.  So I decided to create a new account and then pull the relevant files over from there.

Little did I realize (and this is a big "duh" moment) that I wouldn't be able to access the /home directory for my main account while logged in to the new account, primarily because the new account wasn't part of the "sudoers" list.  

To cut all this short, it was five minutes or so before I realized I had automatically popped open a terminal to address this issue, and that surprised me as much or more than the fact that I was actually able to get the files copied over without having to log in to my main account!  I fixed an issue without help for the first time (woohoo!) and did it from the command line (woohoo!^2)!  

Ah, it's nice to finally be comfortable and happy with an OS.  

So one more time:  WOOHOO!  :lolflag:
