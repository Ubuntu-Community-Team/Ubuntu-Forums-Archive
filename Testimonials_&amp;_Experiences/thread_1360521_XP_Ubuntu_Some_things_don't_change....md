### Post by gwilson on 2009-12-21
Just an update to let you know, Ubuntu is the best and (sigh) some things are still easier in Windows. I actually choose Mint as my flavor of Ubuntu, which is just a personal preference. I have a fully legal and fully updated XP installation that I can dual boot. I have plenty of machine to run XP and have cleaned out all of the extraneous crap that clogs up Windows installations. Still, it seems to get more and more sluggish based upon the number of programs INSTALLED. Sometimes I think the number of installed programs affects performance more than how many programs are actually RUNNING! It will begin to run slowly. I check the resources being used. Not much. Still 600 MB unused. CPU at 10% but lots of disk access and waiting around for whatever is happening to finish. Nothing showing in Taskmanager to account for it.

Ubuntu loads fast, runs fast, almost NEVER fails unless I have been messing around above my pay grade, and has always proven to be recoverable with a few key strokes. I'm in Ubuntu 98% of the time. I'm not going to tell you which things I prefer to do in Windows because I don't want to get into that. Let's just say that I wish that the hardware manufacturers would make an effort to support Linux more fully.

So...

I'm thinking back to when Linux was a build it yourself kit, and I was thrilled to finally get X running a graphical user interface. Of course, there wasn't really any software to run at that time, but seeing that mouse move around the screen was sweet success! We've come a long way!

For another distribution that stays even more "young and vibrant" check out Tiny Core Linux - you can add programs by "mounting" them, which means there are never files left behind when you remove them : they don't put files in various places in your system but rather they put links to the application's own directory so that when you delete the app, the app's files disappear.
