### Post by Fraoch on 2006-08-29
I've tried Linux out for a couple of years now.  Knoppix, Knoppix HDD install (aka Debian), Mandrake, SuSE, Fedora, and now Ubuntu.   I ended up abandoning the other ones, although this was several years ago, mostly due to huge hardware detection issues.  Fedora ended up being the worst in this regard.  Those were really just experiments, I just wanted to see what Linux was about, I had no real reason to use it.  Now I do - my secondary PC (devoted to model railroading and Internet applications) was running Windows 98SE.  With the discontinuance of support recently I just wasn't prepared to have a machine with such unresolved vulnerabilities on my network.  Also I'm not a person who likes eye candy, but Windows 98SE is just looking way too ugly these days.

Plus the two model railroading applications I really needed, [XtrkCad]("http://www.sillub.com/") and [JMRI]("http://jmri.sourceforge.net/"), are both Linux-compatible.

Maybe it's because it's been a few years or maybe it's just that Ubuntu is really good at it ;) but the Ubuntu install went without a hitch.  Perfect hardware detection.  I decided to try Ubuntu due to the buzz it's getting, I figure it's more popular due to good performance, ease of use, etc., but would also mean better community support.

I really like the fact that the install disc is also a live CD.  Nice touch!

Anyway, as I said, installation went without a hitch and all my hardware worked fine the first time.  I really like the look of GNOME, I guess I'm subconsciously associating KDE with all those failed installs of the past.  I also get tired of k-everything in terms of the app's names.

Installing XTrkCad took way longer than it should, 5 hours, but I got it working although any time a font would try to be displayed, it would error out.  I decided to try Automatix to install TrueType fonts.  Worked fine and corrected the issue.  Got Samba working right away but due to some misunderstanding on my part, ended up getting an error message on login [which was promptly resolved right here]("http://www.ubuntuforums.org/showthread.php?t=245776").  I also [uninstalled Network Manager]("http://www.getautomatix.com/forum/index.php?showtopic=103") which I installed using Automatix and optimized things using the Boot-Up Manager which Automatix also installed.

I had a little list with Ubuntu to-dos.  It was 4 items in length - the error on login was a critical one.  Now it's down to 2 items, both of which are minor niggles.  I'll do some forum research in the coming days and may make a few posts, but even if they don't get resolved these aren't show-stoppers.

So I'm sticking with Ubuntu, I've finally got a Linux distro working exactly as I want it.  Whether it's that Linux has improved so much over the years or whether it's just that Ubuntu is so good, I'm a happy camper.

The last big hurdle is JMRI, which apparently isn't the easiest install.  I was considering switching to an RPM-based distro at first since the JMRI install guru uses RPMs, but now I'm willing to get this working in Ubuntu and I'm confident I can get the support both there and here to get it going.

Thanks everyone!
