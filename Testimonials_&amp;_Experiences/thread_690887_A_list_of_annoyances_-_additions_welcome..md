### Post by oskar2000 on 2008-02-07
There are some things that don't qualify as a bug, or they do, but no one resolves them - like the infamous evolution: "summery and folder mismatch" - error - that has been reported in the cretaceous.

So, I'm not asking for answers, I've already done that, or for some things I never really cared. I'm mainly searching for people who share some of those gripes with Ubuntu, or Linux in general.

So, let's get started:

In all video players the video turns into a mosaic of static colors, but the sound plays normally. Logging in and out resolves it. I've seen this on two computers. Both running Ubuntu 7.10. (screenshot attached). Might be related to compiz... I think I've had this on a couple of window managers, but I don't keep track.
--------------------------------

Cannot load settings deamon.
Gnome loads, but with a strange theme and tiny fonts. Can be resolved by logging in and out again. Also by adding a delay in some configuration file... I forgot which, and after a re-install I didn't bother looking it up, so I got it again. I've seen this on two different computers too. Both running 7.10.
Error message on startup:
```
There was an error starting the GNOME Settings Daemon.

Some things, such as themes, sounds, or background settings may not work correctly.

The last error message was:

Did not receive a reply. Possible causes include: the remote application did not send a reply, the message bus security policy blocked the reply, the reply timeout expired, or the network connection was broken.

GNOME will still try to restart the Settings Daemon next time you log in.


```
--------------------------------


Font zooming in firefox drags like hell. (ctrl mousewheel)
This is the worst on Suse. On Ubuntu it's tolerable, but it can take up to 20 seconds for a complex site like this one to go up 6 steps or so. I have set the minimum size so I can read it, but sometimes I'd still like to be able to zoom. It can lock up firefox and always takes 100% of my cpu. Same on a second computer (2x 2.2Ghz) Although not as bad.
I've never heard about anyone having the same problem which is truly odd, since I've had this on 4 PC's starting with Suse 8-point-something  a couple of years back.
--------------------------------


On the gnome desktop, If I drag a group of items over the edge, they overlay, and I have to drag them apart individually - that sucks.
--------------------------------

When I go from a big resolution to a small one and back again - the gnome-panel is a mess. People complained about that years back.
--------------------------------

I think that's enough so far :)
Wheee. Please refrain from flaming. I'm a long time linux user, and this is not a flamebait... I don't regard these things as major problems, but these are good examples for a phenomenon that I've seen all over open source software. The tiny 'bugs' are usually persistent... I don't know why... maybe because they can't really be categorized either as bugs or something else.

### Post by 3rdalbum on 2008-03-03
With the video problem, you have been using two Nvidia cards.

Firstly, if you have Compiz turned on, then turn it off:

```
metacity --replace
```

Try again. If you don't get the problem, it's a Compiz bug.

If you still get the problem, try switching your graphics driver to the basic "vesa" driver. If you still get the problem, it's an Ubuntu bug. If you don't, then it's the Nvidia driver.

The latest Nvidia drivers are a bit buggy unfortunately; video problems can sometimes be worked around.
