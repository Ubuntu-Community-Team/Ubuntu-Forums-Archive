### Post by empty_spaces on 2010-03-03
I have a toshiba laptop on which I already had 9.10 Karmic 64bit running smoothly with Grub2 on mbr. I decided to test out the new Lucid alpha3 on a spare partition.

I went ahead and installed Lucid, but chose not to install a new Grub at the end of the installation.
Then I restarted the laptop, booted into Karmic, and ran **sudo update-grub** in the terminal. This automatically created a new entry for 10.04 in Grub2.

Unlike the original grub, there was no need to manually edit the grub file for the new entries.
Methinks Grub2 is definitely the way to go.
