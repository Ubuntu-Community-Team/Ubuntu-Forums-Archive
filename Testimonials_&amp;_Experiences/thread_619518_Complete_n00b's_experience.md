### Post by decoherence on 2007-11-21
I'm dedicating this thread to a user I've introduced to Xubuntu. He is running an old Pentium Celery with 320MB of RAM. He's a Windows guy who wants a basic system for web, email and p2p and doesn't want to worry about viruses or get nagged to buy things and will 'just work.'

Somehow he managed to break dpkg right off the bat, and it throws the dpkg interrupted error and tells him to run "dpkg --configure -a" -- that's not "just working." There needs to be an obvious way to recover from interrupted dpkg without resorting to the command line. As it is, he's given it back to me saying "i think it needs to be reinstalled." I tell him no, run dpkg, run apt-get -f update, reboot and.... Kernel panic - not syncing: VFS: Unable to mount root fs on unknown-block(0,0)

SHEESH! Not making a great impression on this guy. So I look around the forums and I see this seems to be happening to many people with no useful help in most cases.

I'd like to troubleshoot the issue but the guy wants his computer back. So I AM doing a reinstall, update, etc. Fricking embarrassing for me since I've been running Debian Unstable for years and have never resorted to a reinstall. Well, we do what we have to I guess.  *Hopefully* it doesn't break again.
