### Post by ventrical on 2011-03-06
Iv'e always wanted to be able to build a program (or find a program) that would be able to manage files like a 3D virtual live Rolodex. One thing about Ubuntu default <image viewer> is that it will open any pic,jpg,png file in a seperate session. That is to say most files like pictures or .txt files coverted to jpg etc. This is very unique for many reasons because it uses minimal memory and processor resources.

Now, with the compiz-manager loaded and using the <windows-logo> key and the <tab> key , an end user can load up several files in one workspace on a cube or expo and then shuffle through them as if they were the cards of a virtual rolodex!!

Seeing this work in action prompted me to test the <hibernate> option and the <suspend> option.

 I would like to report that both <modes> work exceptionally well with <suspend> being the faster.

For whatever reason (unbeknownst to me at this time) after the fist hibernation session -the memory usage actually dropped and I cannot say if this is from intuitive  expert programming algorithms but it appears so.

 Then - the results are that using serial cue methods to load from disk the number of open imagefile_veiwer sessions can be very large indeed.

  I also cannot say for sure if this could be considered an assistive technology but it appears just by chance that the Ubuntu maveric Meerkat environment provides for this feature to be rendered.

  Using this method I find that it is very unique and more economic and graphically assistive than working with TABS which at  times an seem somewhat archaic after experiencing the 3d rolodex effect.
