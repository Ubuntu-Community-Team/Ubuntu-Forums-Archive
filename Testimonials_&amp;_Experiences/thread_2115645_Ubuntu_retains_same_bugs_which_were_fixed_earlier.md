Two fine choices. :)
Mint is based on Ubuntu, so it shares many of the same pros and cons.
Fedora is very "bleeding edge" so it's unlikely you would find it more bug-free than Ubuntu.

If stability is your number 1 priority, I recommend: Debian, Slackware, CentOS. 

But the important thing is to find a distro that works for **you**, not the one I personally use/recommend.

### Post by iamkuriouspurpleoranj on 2013-02-13
If you think Arch and OpenSUSE are what you want, give them a spin. If it wasn't meant to be, all you've wasted is a little time.

Funnily enough, I was trying to think of something bad to say about OpenSUSE - really hard - but I can't think of anything.

It didn't have all the software I wanted and I couldn't install XMind on it or NetBeans. You're meant to execute the NetBeans script with kdesu or beesu(or is it gnomesu?), otherwise you get a strange error associated with X.

It is fairly stable.

If you want to try CentOS, also consider going for Scientific Linux. They're a little bit more relaxed with regard to the software included and the forums are slightly more approachable than the CentOS ones. Otherwise it's another RHEL clone based on a version of Fedora from the time before aeroplanes flew.
