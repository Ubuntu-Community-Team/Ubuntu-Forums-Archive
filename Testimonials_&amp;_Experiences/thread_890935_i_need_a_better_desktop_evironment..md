### Post by pparks1 on 2008-08-16
> **charmint said:**
> I got to say no matter witch one i used it was a pain to install programs and getting those programs to work was even worse. Well, KDE, Gnome and XFCE are all simply Desktop environments which run on top of Linux.  The process of actually installing software and such is more a Linux thing and the reason the experience is more or less the same on all the different distros that you tried.

> **charmint said:**
> Why isn't there a desktop evironment like windows xp's start menu, control panel, installer? I'm sorry but even Mac's environment is easier than the three linux ones available. Well, 1 reason in particular is that Linux is a very open system.  Which means that you are always going to have alternatives and different ways of doing things.   That's just the way us Linux users want it.   With Windows, there really is 1 way and everybody does it the same.  Same thing with Mac...it's 1 way...Apple's way.   And I'm not saying there is anything wrong with the Microsoft or the Apple approach...but this openness and complete freedom of choice is one thing that I love about Linux and I wouldn't ever want that to change.

> **charmint said:**
> in time with more people having linux each day, it too will have viruses. That's a common argument that people unfamiliar with Unix/Linux bring up.   The fact that there aren't viruses, trojans and such in Linux is inherently based on the overall design of the system.   Files and such not being executable unless made executable by a root user or user with sudo rights.  With Ubuntu, users have to authenticate with sudo to do most admin level tasks...so most of these issues would never come up.   There is a whole lot more going on than just a smaller userbase.   Some of the most complex banking, healthcare and government systems run on Unix.  There is plenty of reason for viruses and trojans.   And developers and script kiddies have had about 30 years to do it...but yet here we are.

> **charmint said:**
> How do you not want to break your pc? with linux you have to type in commands with every program.
Almost everything can be done with the GUI.   Although there is nothing wrong with commands.   I maintain a comprehensive Ubuntu setup guide and the fact that people can simply cut and paste my commands right into a terminal makes setup a snap.  A whole lot easier than a ton of instructions like 1) Click here, 2) select that 3) go to this tab  4) check that box 5) next, next, next, install etc.   Not to mention, commands make system documentation a snap.   In fact, you could script out these commands and have every single thing setup on your system.  Imagine that, installing 10 apps, 5 drivers, and 65 patches all with 1 and only 1 command.  Pretty slick I'd say.


> **charmint said:**
> I'm surpised i dont have to type a very long command just to turn the thing off.Like I said above, what you can do in the GUI you can do from a command line.  You could have typed this;
sudo shutdown -h now


> **charmint said:**
> Dvd play back wasnt available for copy right protected dvds. All my movies are copy right protected. Bear with me for 5 seconds.  Imagine, you format your hard drive.  You insert your Windows XP cd and install the operating system.  Now, put in a commercial DVD that is encrypted.   Guess what....it doesn't play here either.   In order to get it to play, you have to install software (something like PowerDVD or WinDVD.  Why??? Because these applications have paid the licensing fees associated with decrypting these DVD's.   With ubuntu, you simply enable the multimedia respository, install libdvdcss and viola...you are now watching DVD's.

There is nothing wrong with using Windows.   Linux is certainly not geared towards everybody.  Nor will it ever become standardized to the point where it's simple for everybody to use.  That violates the true beauty and openness of the system.  It may seem like a catch22....but if they "watered it down to the point where everything was pre-configured and determined for you", most of the hardcore users would simply abandon it in favor of something that they could tweak to their hearts content.  Linux isn't striving to be a Windows replacement.  It's perfectly content with being Linux.
