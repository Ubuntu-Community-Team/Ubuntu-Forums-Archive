### Post by enchantedsky on 2008-05-03
3 points to make:

1) Firefox 3 runs faster, and requires less memory to run. It conforms to web standards *better* than its previous version, Internet Explorer, and Apple's Safari. It also has better integration with Ubuntu's standard GUI interface, Gnome.

2) It was the Firefox developers who delayed Firefox's final release until like June 2008. Ubuntu developer's had nothing to do with it. In fact, if Firefox's final release was in April, the same version you are using now would have been the final release anyway. They wanted to tweak out minor bugs that still exist; for Beta software, it runs like a release candidate

3) Your add-ons DO work in Firefox 3, but a compatibility file is disabling your add-ons until they are all 100% tested and approved to work in Firefox 3. If you want, you can bypass the compatibility checks by editing a file --- start a post or do a search on this forum to find out how, if you want your add-ons to work.

### Post by Bytesource on 2008-05-18
There is a simple solution to get around incompatible extensions. 

All you need to do to is install the [Nightly Tester Tools]("https://addons.mozilla.org/en-US/firefox/addon/6543") extension. After installation just click on "override all compatibility" in "Tools --> Add-ons" to activate your extensions again. 

I found this tip on Lifehacker some time ago and for me it worked like a charm :)
