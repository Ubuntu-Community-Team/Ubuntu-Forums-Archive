### Post by katasuka on 2007-08-19
Greetings,

I would like to say Ubuntu has come a LONG way since I first tried it with Breezy.

When I tried Breezy, it failed to install. Dapper installed and worked nice but needed some polishing. Then came 7.04, it was polished, well done and worked great. The only complaints I have at this time with Ubuntu is that it is really slow even on a 1.5ghz computer with 256MB ram.

I feel Ubuntu could be better, but then again anything could be better. I'd like to see more work in the field of network shares (samba) because I've yet to be able to connect to and view shares on both XP and Vista. It always asks for a password despite there being no password. Also, when I share folders on 7.04, my friends vista computer asks for a password despite there being no password set for the share.

Other than that and the slowness, so far Ubuntu is going strong and getting better and better each day.

Thanks to all the devs and volunteers for all the work they have done to make Ubuntu what it is today. Keep going and we all can take the market share from Microsoft!

### Post by Crick on 2007-08-27
> **ukripper said:**
> i am still confused was it a joke with "doh" command? or was he/she being serious about it?
A bit of both, really. I'm sure everyone has been at the CLI typing away on some long command, only to realize that one will now have to move the mouse, select, copy, type "sudo ", paste, press enter. And then say to oneself "Doh!", perhaps even verbalizing it.

I thought, "There really ought to be a command for this", since I'm sure everyone has had the same experience, and likely will as long as they use the CLI in Ubuntu. So I typed "doh" into a terminal, there was no command bound for it. I searched through ubuntuforums, and couldn't believe that no one had thought of it before.

It is short and fairly easy to type. I suppose that I could always just "sudo su", but I think sudo is a better practice.

So yes, it was intended to be humorous, but I would really find such a command useful. I suppose I could always type <up>, <home>, "sudo <enter>". But it would take about four times as long. ;)

Edit: Thanks to all those who found it amusing!

In the /etc/samba directory you will find a file named smb.conf

in this fiel you will find a line that says security = user
uncomment this line and change it to security = share

save and exit
