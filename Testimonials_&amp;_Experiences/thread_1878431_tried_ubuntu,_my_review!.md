### Post by steve12321 on 2011-11-09
hi!

i was told to try out ubuntu, so i downloaded. here are my thoughts

installation - super easy, really well done. no problems getting it up and running

navigation - pretty good. i like the layout!

speed - not nearly as good as windows xp. between 2-3 times slower. when i click on something in the "launched" it takes a few seconds to load up, in xp it's usually pretty instant.

internet - holy crap is it ever slow! it takes about a minute to load a webpage. i tried to search 'ubuntu internet too slow' and just got a lot of stuff about typing in instructions and copying and pasting stuff and code and god knows what else. this was the deal breaker for me. if this isn't something i can quickly fix, no dice.

free apps - awesome!

at any rate, the speed issue is what's killing this for me. it's twice as slow as xp, and i don't think there's an easy way to fix this. the internet is brutally slow, and i have no idea how to fix that. it seems like troubleshooting with ubuntu is a lot more difficult than with xp - or at least this is the impression i have looking up 'how to' websites. in windows all the 'how to' pages seem to say 'click here, click here, click here, delete this'. in ubuntu it appears to be a lot of 'open this, type this, copy this, type this, copy and type this, etc' which is a lot more formidable to someone new to this operating system.

on a more individual note, i use autocad a lot and also cubase. i was told there are some good open source alternatives to cubase, which i'm interested in.. i have a MOTU setup, and apparently MOTU hates linux and won't make drivers, and while this is motu's fault, it still doesn't help me. and then secondly i don't know if there's a CAD alternative to autocad, but even if there is, i'm really skeptical of using it and having drawings not transfer over so that autocad users see 100% the same thing i do. 

anyway, i hope this review is somewhat helpful to the people doing this. good job, but i'm going to pass largely because the learning curve seems way to steep, the internet is maddeningly slow, and the overall speed of the operating system is noticeably slower than windows.

i noticed a ton of 'internet too slow ubuntu' webpages - since this is probably the most important feature of a computer to regular people, i'd suggest putting more focus into ensuring things work nice and fast, right out of the box.

thanks for your time, and good luck!

### Post by JayKay3OOO on 2011-11-11
People often use terminal commands to fix things as the user interface is not quite as highly developed as XP and it's often easier and quicker to type or copy a few commands into the terminal to get a fix done where as windows hides all the commands behind user interface and everything is orientated for it to be this way. 

The user interface of Linux is still a bit stuck on top and everything can be done through the terminal.

When I first started using Linux more than a year ago I hated using the terminal & could not understand the reason why people always used terminal commands. 

Using Linux requires a different mind set. You, like many people (inc me) presume that it should work like windows. We bring our knowledge and assumptions over without starting out on a clean slate.

Without knowing the specifications of your computer I find it hard to know what you were expecting. XP is quite fast because believe it or not it's over 10 years old now and technology has advanced a LOT. By rights without the MS Vista screw up it should have been superseded years ago, but due to this blunder people have kept their rubbish low spec machines and forget that Ubuntu is moving to higher resource usage. If people want transitions, transparency, effects & other 'flashy' aspects in an OS then we have to accept a more resource hungry OS that generally requires better and better graphics acceleration.

But you think I must be running a quad core machine, while telling you to get a better PC? While I do have such a machine (that I don't use) I actually run Kubuntu (KDE version of Ubuntu) on a 2.6GHZ (single core) laptop with a small 2GB of DDR2 RAM and while it does not work as fast as XP it does work about the same if not faster as Windows 7. I can honestly say that the quad core runs the actual KDE interface only a tiny bit faster due in part to the DDR3 ram & faster hard drive.

But I'm not here to rant about what machine to use and I'll gladly admit that Linux caters for all specifications and Ubuntu has an XFC (Xubunut) derivative for an old (XP) machine such as yours although you will have to sacrifice some of the 'flash' that Unity has.

I really hope you can get your internet fixed in due course. Don't give up just yet. KDE took 2 versions for me before it finally decided to support my laptops wireless (although I've heard that was because network manager was broken!) I even went on a non-public rage quit and brought Windows 7 to not deal with the Linux problems and look where it got me. I'm back on Linux :lolflag: with no problems! Phew!

I've even come across the odd situation where Linux actually supports some of my hardware I have out of the box where Windows 7 does not and Windows 7 is pretty good for out of the box support.

I guess my message is simply this.

Where there is a will there is a way.
