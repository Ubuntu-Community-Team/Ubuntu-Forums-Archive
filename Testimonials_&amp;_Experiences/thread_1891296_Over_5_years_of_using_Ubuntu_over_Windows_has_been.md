### Post by Amurko on 2011-12-05
Contrary to popular stereotype of Linux (at least when I was considering it 5-6 years ago), it didn't prove much more difficult to use than Windows.  I also used mostly free software so the transition to Linux wasn't very drastic (so I continued to use the same open source progs that were available on many platforms.)  The app store, aka package manager, has been quite convenient in getting software updates when they're released..  wish Windows Update could do the same with your installed programs.

I first installed Ubuntu 6.10 on my newly built Core 2 2.4 Ghz desktop in late 2006 and have upgraded it all the way to 11.10 only to have to reinstall this weekend (the reasons are ahead.)

My main complaint about the system is that although it's relatively easy to get set up and working, there's little guarantee that any specific program, feature, etc. will continue to work in the future.  In other words, there's no guarantee that whatever's working now will continue to work in the future.

This is especially true after a software upgrade or especially a distribution upgrade.  I've had my Internet, sound, boot manager (Grub), Xwindows, desktop layout (esp change to Unity), etc. fail on various occasions after upgrading.  In many instances, having to boot on a Live CD while relying on another computer to read the forums' instructions to rescue the system has been an ordeal endured too often.

Sure, avoiding upgrades altogether or reinstalling frequently can help mitigate the problems with the system frequently breaking after updates.  But who wants to run a system that is never updated?

My suggestion for the upcoming releases is to make the upgrade process more **Stable and Reliable**, at least ensuring the critical components such as **networking, graphical display, and internet** are working (even if other components were to break.)  If those 3 are working, it makes it so much easier to troubleshoot whatever other problem is plaguing the system.

Hope that helps.

### Post by Iloru on 2011-12-05
> **Amurko said:**
> I've had my Internet, sound, boot manager (Grub), Xwindows, desktop layout (esp change to Unity), etc. fail on various occasions after upgrading.  In many instances, having to boot on a Live CD while relying on another computer to read the forums' instructions to rescue the system has been an ordeal endured too often.

I used to have that problem years ago, but these days I rarely encounter a problem that causes my wireless card or graphics card to not work.  And believe me, I'm no expert at working with Linux.

I often think of myself as part of the target audience that Ubuntu tries to appeal to.  I'm a casual user who uses Ubuntu the same way most people use Windows.  My husband (then boyfriend) was the one who introduced me to Linux.  It was Red Hat, and I couldn't set it up, couldn't fix anything that broke, but I enjoyed having multiple workspaces and high levels of customization.  I even remember that my very first desktop environment was Enlightenment, with its many shades of green and teal backgrounds.

Anyway, back in 2000, when I first started using Linux, it was next to impossible for someone like me to install it.  I knew nothing about how to build something from source and the communities seemed far less helpful and friendly toward newbies like me.  That's not to say that help wasn't available; I just didn't know where to find it.

Ever since my first Ubuntu install (2005, maybe?), I have been hooked.  It's relatively simple to install (once my husband explained the whole partitioning process to me) and within a year I was doing most of the work setting up a dual-boot Ubuntu/Windows machine.  

These days I install/reinstall various flavors of Ubuntu (I'm quite fond of Mint at the moment, though my favorite used to be Kubuntu) without a problem.  It's not a huge accomplishment that deserves any sort of congratulations, but I'm still proud of myself.  If a problem comes up, I just do a forum search and 99% of the time find a solution and detailed instructions that allow me to fix what went wrong.  I still couldn't tell you how to build something from source, but I sure as heck know where to look for the answers.  ;)

I guess that is the long way to say that I feel Ubuntu has come a long way.  It was always a great experience from the beginning, but the past year or two I've seen the releases become much more stable regarding internet support (wireless cards, routers, etc.) and graphics cards.  Upgrades often go through without a problem for me.  Now if hardware companies (I'm looking at you, Nvidia *coughOptimusSupportPleasecough*) and game companies would just start showing some Linux support, I could get rid of Windows again...
