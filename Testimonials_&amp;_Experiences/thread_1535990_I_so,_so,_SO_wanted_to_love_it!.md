> **Rhodophyta said:**
>

If I were in your situation, I'd get a backup of everything that you need and re-install Windows 7. You could run into serious problems trying to get Windows 7 back and it's better to find out about them now instead of later. If your CD has an option to re-install Windows or reset it to its factory default, then the quickest option would be to use one of those (but not without backing everything up).

I read about your problems in other threads. Unless either you have time to spare and want to troubleshoot computer problems or Ubuntu can offer you something significant that other OSes cannot, it's best to stick to a system that meets your needs.

Ubuntu isn't great for playing most commercial video games and if you can't do basic things such as connecting to the internet, viewing your desktop, or accessing devices (such as CDs/DVDs, your hard drive, or USB drives) without a significant amount of effort then it might be best to try these things from a Live CD when the next version comes out.

Don't get me wrong, I'm not saying that you shouldn't try Ubuntu or use it if you really like it. I use it for everything and it's my only operating system (unless you count my Windows testing system that is in a virtual machine). I'm just saying that now might not be the best time to continue with it.

If you want to use it with the minimal fuss, you might want to try installing Ubuntu in a virtual machine from within Windows in order get used to it in the mean time.

### Post by sailorboy on 2010-07-23
Have not read all your posts but I got a new Compaq laptop (Windows7) that I set up to  dual boot Karmic. I have boot choices  (aside from Ubuntu and memtests) of either a  'vista bootloader' (does nothing) & "Windows7 loader" (which works).
 I read up on dual boot posts B4 I started and was glad I did.
 First, I imaged the HD to an external usb HD with the provided Windows utility since no recovery discs were included with the PC.
 Understand, this PC would not accept a dual boot until I modified the HD. I deleted the 'recovery' partition (since HP had a check box somewhere to do that) rather than lose the "HP tools" partition. This appears to allow the HP 'support assistant' to work.
 Then it resized the windows partition to use the extra space freed up.
 This was done with the provided Windows utility under control panel> admin tools>computer management>storage.
 Believe I then defragged windows and installed Karmic as dual boot.
 My first reboot into  Windows requested a 'chkdsk' that I allowed.
 Then the Ubuntu boot failed- but I chose recovery or grub repair and that fixed it.
 After that, it worked fine- (later, I upgraded to Lucid) but admit the setup was quirky.
 I rarely use the Windows side, it just seems lame in comparison to Ubuntu eye candy.
 Hope your luck improves- it's worth the effort!
