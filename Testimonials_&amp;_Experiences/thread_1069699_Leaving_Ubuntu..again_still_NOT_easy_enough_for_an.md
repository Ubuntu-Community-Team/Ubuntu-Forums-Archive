There's a 'Linux is not Windows' site for that stuff.

Sometimes the solution or something related to your solution can be found in similar forums or even a forum for another distro.

Most of all, Ubuntu is like the other distros as there are times when things are working but when you make a 'change' something gets broken and then there is a new problem to troubleshoot.  That is really annoying and frustrating and another problem is that each distro has some sort of issue or 'differences' that makes you want to pull your hair out sometimes.  Anyway, to even give Linux a chance, you have to be prepared or willing to read.  Like I said, there is a learning curve and even Linux users have to confess to that. ### Post by Alterax on 2009-02-15
Two of the OP's genuine complaints are already taken care of:

USB wireless driver support is part of the 2.6.28 kernel.  I'm not absolutely certain, but I am fairly sure that will be the kernel used in Jaunty.

Bulletproof X came as a part of Intrepid, with the xorg update; this is usually sufficient for the safe mode.

The autosave of config files isn't a bad idea per se, but I've found that I can usually fix bad configurations when they occur by using the rescue options on the bootCD.  I also tinker A LOT with my config systems, and despite my best efforts haven't managed to render a system unbootable to a point that would require a "system restore".  I won't call it foolproof, as I know better than to not underestimate the power of a sufficiently motivated fool.  But it's pretty darn close.

At the same time, I do have to note to the OP that Linux wasn't designed to be Windows, so the expectation that it behave exactly as Windows behaves is somewhat silly.  Comparing the two is like comparing apples to Plato's Symposium.  Those fixes for a Windows system were developed to counter breakages that Windows is prone to encounter.  Applying these methods to a Linux system are not only unnecessary, but without a massive overhaul could cause more damage than it fixes.

Besides, I know very few that use Windows system restore because 3/4 of the time it doesn't work and 100% of the time it's a resource hog.  Most learn it works best when turned off.
