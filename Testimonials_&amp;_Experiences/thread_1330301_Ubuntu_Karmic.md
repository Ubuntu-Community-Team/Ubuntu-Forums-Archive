### Post by Methuselah on 2009-11-18
I've been using Ubuntu for about 2 years now and recently tried Karmic in a very limited way on a LiveCD.
I must say, I was both impressed and disappointed.

After having tried Feisty, Gutsy, Hardy, and Intrepid discs inthe same machine, Karmic is the first that actually had problems loading the LiveCD environment.
The first time it loaded the Desktop but warned of a fatal kernel error for which I restarted.
Then it failed to even enter the Desktop but froze due to an error while booting.
On the next attempt it loaded the Desktop background but no menus, panels or icons appeared.
Finally, on the fourth attempt, all seemed well and I am typing this message from the LiveCD.

The new Software Centre fails to open in the LiveCD environment.
It appears in the lower panel briefly as it tries to load but goes away before ever launching fully. 
This is something no other releases' Add/Remove utility ever did to me.

On the positive side, the new icon set is nice.
I especially liked how a mounted USB thumb drive got its own icon rather than a generic drive icon.
I was also rather impressed that the monitor came up at full 1920x1080 resolution without any tweaking on my part.
This is something no other release had been able to accomplish.

However, my overall impression from the LiveCD is that Karmic is indeed somewhat less stable than past Ubuntu releases. 
It could be that these symptoms only exist in the LiveCD but I would hesitate to install it given the mixed first impression.
I guess I'd recommend that anyone tempted to upgrade their main system exercise extreme caution.
Do appropriate preparation/testing and do not take things for granted.

### Post by Methuselah on 2009-11-19
IMO, a six month release cycle made more sense when Ubuntu was being born as there was much to improve and a smaller and more technical userbase likely to be more resilient in the face of bugs and inconveniences.

Five, years later, the situation is rather different. No one was *relying* on Ubuntu in the days of Warty but now people are!
As of Hardy Heron, Ubuntu has matured into an operating system that can and is used my non-technical people and became the most popular Linux distribution.
I think we are at a crossroads where we might have to rethink some of the old wisdom.
Once you have already earned respect and credibility it is very important to keep it.

Remember that new versions of Ubuntu aren't simply for a shiny new OS but is the supported way to get new application software.
If you had to upgrade windows to use the next Photoshop would you find it acceptable if the next windows broke so many things that you couldn't upgrade?
Every release does not have to be cutting edge and risky as they also provide a rather mundane but essential service.

IMO, releases might need to be slowed down or planned with more foresight than sprints to +1.
The trend of releasing versions with significant problems (eg. Jaunty Intel) will be more damaging to Ubuntu's image in the long run.

Lucid Lynx LTS, should really be an opportunity to step back and make things clear, functional and as reliable as possible.
After that a greater emphasis needs to be placed on quality maintenance even if it requires more long-term planning.
