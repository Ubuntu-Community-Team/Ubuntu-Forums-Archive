### Post by Hedgehog1 on 2011-10-24
Macs are great and can run Ubuntu as well.

I have found that a clean install of each new version of Ubuntu yields the best results.

Using a separate '/home' partition for your data and formatting and doing a clean install of 11.10 on my '/' (root) partition gave me a good solid base. I do this each time i move to the next version of Ubuntu.

Now, you may not have your '/home' as a separate partition, but you can do that to your system now and then re-install 11.10 (or 11.04) cleanly that way.

Just a thought.

### Post by TBABill on 2011-10-24
+1 malspa. Some things that can help or hurt, depending on hardware, are xorg version, kernel version, drivers and other things. For example, in 11.04 the STA Broadcom driver or firmware is borked for the BCM4311 Broadcom wireless device. Sure, it can use the b43, but why not the STA any longer in 11.04? So someone may find Ubuntu and derivatives not working while the same hardware works fine in Debian, PCLOS, openSUSE, etc. And xorg in Ubuntu stays closer to cutting edge than some other distros so some users experience video problems while a move to PCLOS not long ago may have exhibited very different behavior because the version it had been on was pretty long in the tooth.
 
I'm not saying at all that things shouldn't work in ubuntu because it seems to have some of the strongest hardware support, even if sometimes a users has to figure out the workaround to get something working exactly right. Rather, there could be a varied experience with hardware when trying another distro that is not Ubuntu based. I've had the best luck with Ubuntu derivatives, PCLOS and Debian, but others may find better luck on many of the others. Regardless, if you find one that works well in the live session it may be worth further exploration.
