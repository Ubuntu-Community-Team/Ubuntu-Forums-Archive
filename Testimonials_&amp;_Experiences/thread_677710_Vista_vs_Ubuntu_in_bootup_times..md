### Post by Dragon45 on 2008-01-25
I wiped my drive, clove it in two, and installed Vista Ultimate on one half and Ubuntu on the other. Default options and settings, no tweaks.

Guess which one boots faster?
Vista. By far. By at least 5 seconds.

This is with no additional services or programs; just a fresh Gutsy CD and a fresh Vista CD. Shocked? So am I. I had previously assumed that the longer Ubuntu bootup times were due to the number of daemons, services, and servers I had running from the machine. Alas, that was not the case.

To iterate: Clean Vista boots faster than clean Ubuntu. Used and worn Vista boots faster than used and worn Ubuntu.

I'll run some empirical measured tests later today. I hope I'm wrong and I'm simply being distracted by Vista's prettier startup animation. If I am wrong, I will admit it happily (after I get a stopwatch).

I'm running a Gateway laptop Pentium Dual Core (NOT Core 2 Duo) 32bit with 1 gig of SDRAM, a 100 gig harddrive, and integrated graphics.

So anyone have any ideas why this is the case?

### Post by OoooMatron on 2008-01-25
I am not shocked at all. I run windows xp on another partition for music software and with a clean system and only the apps I need installed it boots much much faster than Ubuntu. Should note that only the services in services.msc that are required are set to automatic. Most are disabled. This is the key to faster XP booting (that and the start up reg keys).

While I like gnome I still feel that XP running no themes is very quick with the user interface and faster than Ubuntu on the whole.

That said, I wouldn't go back to Microsoft OSs for every day use if you paid me.
