### Post by crhylove on 2008-02-26
The last few times I tried out Ubuntu, I'd have to have said I was new to it.  I started messing around with Ubuntu back in the Dapper days, and tried out Edgy, and then actually used Feisty for a while before I had an unrecoverable crash.

Currently I am using Gutsy 64 bit.  With all updates.

I am posting this review for public scrutiny, and because I believe in FOSS, and would like to see improvements to Ubuntu.

The last OS I was using as a daily driver was Windows XP x64.  It is blindingly fast and stable on this hardware (athlon 64, geforce 6600 gt, 1 gb ram).

I'm going to start with the positive experience that Ubuntu in it's current form provided.

To start, Gutsy seems even more polished and smooth than Feisty.  It seems more user friendly, and has some better organization in the menus and during install.  The disk partitioner is still a nightmare unless you select "Automatically use entire disk", but other than that the install was fast, painless, and easy.  I'd suggest a situation where it listed all your current partitions (in an easy to read format), and there was a selection that stated "Use entire Partition", and that left the other partitions on the disk alone.  Further, My x64 installation that I was hoping to dual boot to is completely gone.  Well, it's still available after I click on the hard drive and type in a password (why should I have to do this every time gutsy boots, by the way?  And why do I have to select the same drives in the same order in nautilus lest I have to redo my rhythmbox database and all my playlists, etc?), but there is definitely no selection in the grub menu allowing me to boot back into x64.  Which means this computer is now due for a reformat.... But I'm getting ahead of myself.

Here are some of the problems I've had with Gutsy 64:

1. There is no Skype client for gutsy 64.  Sad Face.  Not Cannonical's fault, but it still sucks.

2. When I left click in the URL area in firefox, it does not automagically select the entire line allowing me to quickly and easily type in a URL.  I have to delete all the letters and start over by hand.  Every time.

3. Virtualbox does not work out of box.  Apparently there is a way to do it, but despite my 20+ years of tech support experience, I cannot figure it out.  That means, no tinyxp, no gOS, no fall back plan so I can run the windows apps that won't load in wine.

4. After attempting to get virtualbox working, compiz died.  It will not come back.  Now every time I select "Extra" in the visual preferences (which turns on compiz), X restarts.  As a Windows user moving to Linux (for the 8th time!), without compiz there is little point in switching at all.

5. There is no "It just works" Nintendo 64 emulator in the repositories.  I was able to get a windows one to work under wine (after experimenting with several input plugins, there is still major joystick issues in wine, after all these years), but it doesn't run that speedily, and it was a real pain in the *** to set up.  Especially when I have to re-initialize all my old windows drives when I boot up, and if I do this wrong, the emulator cannot find my games (again).  Ditto for SNES.  I found a SNES emulator that was native, but it is drastically behind any of the windows ones.

6. Default to Evolution.  Are you kidding?  Thunderbird is so much better!

7. Default to Brown.  Hiddeous.  Why must I be forced to run through all the preferences just to keep from puking?  Here are some colors I would prefer to brown:  Black, White, Gray, Red, Blue, Green, Purple, Yellow, Orange, Silver, Maroon, Gold, Hot Pink, Puse, Vanilla, Teal, Turqoise, and Goldenrod.  If you can think of another color, please add it to the list, because brown has got to be the worst.

8. Difficulty burning cds.  I tried to make an mp3 cd, and after I select about 20 tracks, it tells me it cannot read from the source drive any more, until I reboot.  Then I have to begin selecting tracks all over again.  I have not found a solution.

9. Difficulty in watching some types of video.  I did eventually get the correct codecs installed (mostly), but I still can't watch .rm files, and it was a big pain in the ***, initially.  I had to reload like 6 different files from the repos, and I'm still not sure I got the "best" ones.

10. Pidgin crashes.  Most particularly if I try to add MySpace IM and MySpace IM buddies.  Very, very unstable.  Also slow.

11. Slow shut down time.  It's not horrifically slow, but it does seem slower to shut down than Windows x64.  Why?  Why not just turn off immediately?

12. No Urban Terror in the repos.  Urban Terror is free as in beer, and a GREAT game.  It is installable from their website, but if it was in the add/remove software gizmo, that would be a real boon for n00bs coming from windows, and impress them as it is a GREAT game.

13. Slowness with online video.  YouTube videos and the like seem to render a little slowly.  Don't know why.

14. Iffy photo import gizmo.  It seemed to be unable to remove pictures from one of my cameras.  It could copy them, but not empty the camera.  It worked well on my other camera though.

15. Constant unnecessary password entry.  I know that Linux is SUPPOSED to be more secure (and I guess it probably is), but I should really only have to enter my password once every half hour, pretty much regardless of other circumstance.

16. MySpace page crashes.  There are tons of heinous MySpace pages that will slow firefox to a crawl and then crash it.  Only under Ubuntu.  Everything renders correctly in FF in Windows (though, to be fair this is probably largely due to how horrific some MySpace pages are.

17. Messed up GRUB.  There are 10 different Linux options, no windows options, and this computer has a valid x64 partition still in it, and several VDI (virtualbox drives) images that it would also be nice to boot into.  Also, only two of the options work: Generic, and Server, though server seems even buggier and harder to use than generic.  Also, I can't access GRUB options with my USB keyboard, I currently have to have TWO keyboards on my desk, just for grub menu selection with the old purple ps2 jack keyboard.

18. Missing my very common ViewSonic monitor specs.  I tried to enter them by hand.  I may have gotten them right.  A VX924 is really not a rare monitor though.  It should have specs for this one automagically.

The last time I gave Ubuntu a serious run (about a year ago), I really tried hard to find work arounds and solutions for the (smaller group of) problems I experienced at that time.  I even submitted bug reports here in the forums and on Launchpad.  I got personally attacked by a few people (both on launchpad and in the forums), and I think I even had posts edited or deleted.  I don't remember the details as I wasn't that emotionally invested, but for the record, that kind of crap is also a major turn off for people trying out Linux coming from Windows.  There is no need to be a zealot, especially when 90% of these issues are (probably) easily fixable.

I'm not trying to start a flame war, and I really don't want to hear excuses or dissenting opinions.  What I'd like to hear is that these issues are being fixed for the next release.  Until then though, I will wait patiently for the hardy heron, and for now go back to nlite and x64 customized with all my favorite FOSS apps:

Open Office Writer (portable)
The Gimp (portable)
Thunderbird (portable)
Virtualbox (it works!)
Firefox (Beta 3 portable)
Pidgin (still buggy and crashes, but still better than MSN, AIM, or yahoo messengers)
Virtualdub (portable)
Filezilla (portable)
7zip
SumatraPDF (portable)

and the few non FOSS apps I am still forced to use in the windows world because there is no reasonable alternative:

Winamp 2.61 (with Milkdrop)
Nero (portable)
Grand Theft Auto San Andreas
Urban Terror
Skype

Maybe one of these days all that stuff will work reliable in ReactOS and I can go full FOSS!!!

Best wishes, and best of luck to you and yours,
The Captain

### Post by 3rdalbum on 2008-03-11
1. Skype can be made to work on 64-bit. Do a Google search. But I admit, the program probably only requires a recompile on a 64-bit OS, but Skype are too busy to do even that!

2. Double-click it. This is absolutely correct behaviour that you are describing. No other text field selects all the next when you click it - go on, try it! Click inside a currently-deselected text field and you'll find that it places the insertion point.

3. Your 20+ years of computer experience didn't tell you to read the manual :-)  You must add your user account to the "vboxusers" group. This is a security feature in Linux.

4. If the entire reason for you switching to Linux is to get funky desktop effects, I suggest you reinstall Windows, and come back to Linux once you have more of a reason. That's probably why you keep switching to Linux and then switching back - you have no reason to switch in the first place.

7. Some people like Windows XP-blue. Some people like OS X's titanium/aqua/platinum/brushed/whatever-theme-they-are-using-this-release. I like the Ubuntu orange and the brown wallpapers, and I hate the themes of the proprietary operating systems. It's such a ridiculously trivial thing for you to change, too!

8. Never had such a problem

9. Have you got the w32codecs installed?

10. Not had this sort of problem with Pidgin, only with Windows Live Messenger on my old Win XP box. Try Kopete.

11. If you want to turn off the computer immediately, just kill the power. I'd rather have my data stay intact. Ubuntu does shut down faster than Vista. The Gparted Live CD shuts down within 2 seconds, but that's because it's not mounting anything read/write.

15. You enter your password once every 15 minutes for sudo, and once every 15 minutes for gksudo. Even that is a little bit unsafe. It's not really "constant password entry". If you want to change the default action, it is possible - just look up a HOWTO.

16. It's probably something to do with Adobe's pretty-far-broken Flash player. I stopped surfing Myspace a long time ago because I was sick of having to download 10 megabytes worth of Flash slideshows, Flash audio players, embedded Myspace Video, etc.

17. Never had such a problem - it could be a problem with your BIOS settings. I've also got no idea why you installed a bigiron server kernel on what is obviously a desktop machine. I'm also not sure how you expect to "boot into" a Virtualbox image.

As for your other problems, yes the partitioner is pretty terrible. GRUB should have detected your Windows partition when it installed, but you can always add your Windows partition to it.
