No one forces us to upgrade or install the most recent release. It's done by choice. For those that are installing Ubuntu for the first time, I often recommend starting with 8.04 (if possible). It's a nice stable release and an excellent starting version for the new users. Once comfortable, they can upgrade and try newer versions.

I also recommend using an spare HDD for upgrade testing purposes and only installing the upgrade once tested and established that it contained improvements that enhanced one's computing experience. I advise the individuals that I assist, to be patient and test new released before committing.  With all the available hardware and different mfgs., the odds of eventually finding a hardware device that isn't supported - increases with every release.

Hope you enjoy the new OS and in the future test upgrade functionality and identify that the improvements will actually benefit you, prior to fully implementing. If not, reconsider and wait for the next release.

IMHO, when suggesting Ubuntu to others - start them with an earlier version rather than the most recent release (if possible). Explain the need for backing up personal data and using a spare HDD to test upgrades prior to installation. With just a few minor, logical steps - we can improve the process. A spare, used HDD is an inexpensive, yet viable tool that could be used to test multiple systems.

Good Luck and I'm glad you were able to resolve your issue. The right hardware can often make a big difference. I've always found it odd that some complain about hardware/drivers, yet every OS I've ever used had similar problems. It's been my experience that hardware compatibility is a must with every OS.
