The fix to get the computer to boot again was easy: put in the Windows install cd and reinstall the Windows mbr. What had happened was grub died with the Ubuntu erase, most likely.

Why was he using Ubuntu Server Edition? My guess is that it doesn't have many (particularly wireless) drivers. Additionally, in 7.04, it is very easy to get Wireless to work if you have the Windows .inf and use ndiswrapper. Using default drivers, like in Windows, isn't always the best idea. Additionally, his dhclient file may have been set up incorrectly with his network.I suppose you tried many drivers with Ubuntu, after reading this thread,  but you could've bought another wireless card. :/ I find Belkin wireless drivers work well.

I would have gladly taken your not working computer for free, or even pay for this so-called "not working" computer... I would have been easily been able fix anything you did to it.

Why did you have an x1650 with Linux anyway?

Also, let this be a lesson to you for installing Linux on the same hard drive as Windows... If you want to securely install Ubuntu or any other OS, you should've taken a spare hard drive and installed it there (making sure to disconnect your primary hard drive so that Ubuntu doesn't install grub over your Windows mbr--old Ubuntu bug.)
