### Post by bzamora on 2008-05-13
Is it just me or has anybody else been disappointed with the performance of the automatic package updater? In my updater, it is stating there is a new version of Ubuntu out (8.04). I am running 7.10. So I click the Upgrade button and get the usual prompts... the applications tells me that based on my connection speed, the upgrade should complete in about 55 minutes. Once the upgrade process starts, the download performance is terrible. I conistently see this characteristic. My throughput is being displayed in bytes/sec not kb/sec and the remaining time bounces around in terms of days to complete, not minutes to complete. What's up with this??? Is this tied in any way to the Ubuntu website? Too many people downloading updates and thus overloaded servers? I don't seem to have any options to switch to other mirrors for better performance and I thought this was supposed to be handled automatically anyway. Let's face it, if the program checks the connection speed and can determine that upgrade process should be able to complete within x amount of time, then while downloading components, it sees the measured throughput is drastically poor, shouldn't it compensate? If the software can't manage that process then maybe it makes sense to present a list of mirrors so that the user can pick one they know performs decently? The type of performance I'm seeing leaves me very irritated with this automated update process.
