### Post by Crusader_Tech on 2008-10-21
Hello everyone!

I would like to add myself to the list of Ubuntu users.

So far, things have been relatively painless.

Install 1

Ubuntu 8.10 Beta
Dell Inspiron 9300
Pentium M 1.6
1 Gig Ram
60 Gig HD
GF 6800 Go

This is my main Linux machine. It runs like a champ. The only thing I had to do was hard wire into my NIC to download the "firmware" for my wireless card. (still not sure I understand that) but its working great now.  Zero config. My 6800 runs great as well. I've got compiz doing some pretty crazy things, and they're all very smooth.

Also, much to my amazement, with VERY little work, I have World Of Warcraft running. I'm absolutely amazed. Gaming is still the biggie keeping me from ditching Vista on my desktop, but Linux seems to be making slow but steady progress in that area. 

Install #2

Wubi on my desktop
Asus A8N-VM CSM mobo
AMD 64 X2 4200+
3 Gigs Ram
GF 8800 GT

Ubuntu runs well, WHEN it runs.  It seems to have file corruption issues frequently. ( happens also on my work desktop, also Wubi inside Vista ) I simple check disk always fixes the problem.  It's just a bit of a pain, as it takes a while to load. 

Some personal Experiance Pro's Con's
(please feel free to comment)

Pros:
Rock solid stable
Amazing performance for such little resource usage.
Insanely easy software management via repositories.
Completely customizable
Free primary install. Free updates. Free software. Free freeness. ;-)

Cons: (feel free to help)
Flash - Wow, I didn't realize how much I used flash on the web until I started trying to watch flash videos in Ubuntu. Why does flash perform so poorly in Ubuntu? Youtube videos look fine in the web page window, but when I full screen, they get very choppy.  Same with other videos.  In the flash version, VERY choppy. But if I download it and play it, it's smooth as silk. It's not a download speed issue, as I can wait until the entire video is downloaded, and it's still very choppy.  Like my video card is not powerful enough. ( also not the issue )  And some flash pages just don't load right. ( of a positive note, I was looking at the Dell Inspirion Mini 9, and the flash wouldn't render right.  After a week of nothing but Ubuntu updates, the site suddenly renders right.  Grats Ubuntu!)

Main-stream game support - I know this is a chicken and the egg issue, but I wish it would crack ( I know, but I couldn't help myself ) Linux user base is too small for developers to bother. More gamers wont use Linux until more developers make games for it.

Familiarity - This is not a fault of Linux. Strictly my own shortcoming. If it's not a nice neat package in the repository, I can't seem to install it.  Make what? What do you mean, compile?

Wubi - This is the ultimate safe way to get people to try Ubuntu. Lets fix the file corruption issues.

Fonts - Some web pages look plain strange. 



CONCLUSION

I'm loving me some Ubuntu. After years of trying Linux here and there, and loosing interest in all the terminal time spent, I've finally found a distro that I can get working with little to no configuration. I just found out I don't have to defrag. If I don't have to reinstall every so often due to "bit rot" (already don't have a registry to mess with) I'll be very close to ditching windows completely. Come on, game companies. 

Thanks for having me community. I look forward to asking questions, and I hope I can give back at some point. The only thing I know how to do is run the software test, and send the report.

### Post by duott on 2008-10-22
>Make what? What do you mean, compile?

Compile means:
1. download the *.tar.gz archive with your software
2. extract it
3. cd /directory/where/you/put/it
4. ./configure
5. make
6. make install
7. enjoy :)

Not too complicated really :) In addition to this, there're many *.deb files around the web that are not in the repositories but which you can download and install by typing:

dpkg -i file.deb

### Post by MaxIBoy on 2008-10-22
A little bit more of an explanation:
[LIST]
[*]A ".tar.gz" file, otherwise known as a "tarball," is an archive (a ".tar") which has been compressed (a ".gz.") "gz" stands for GNU Zip. You may also see .tar.7z or .tar.bz2 files. 
[*]"cd" means "change directory." It's like clicking on an icon in the folder browser. "cd .." will take you to the parent directory. "cd ~" will take you to your home directory. "cd /" will take you to the root directory (think "my computer" mixed with "c drive.") "cd /whatever" will take you to a folder called "whatever" inside of the root directory. "cd ." will loop back to the current directory.
[*]"./configure" is a program that sets up a few files for "make," and customizes them for your specific computer setup. They're not always needed, but if your tarball comes with one, you should always run it. A different one has to be written for every program. It could be called something else, but it's always called "configure" for the same reason that there's always a file called "README." It's what people expect.
[*]"make clean" gets rid of older versions of the program you might have already compiled so they won't mess with the new version. You don't need to do this if you're installing something for the first time.
[*]"make" takes the code that the programmer wrote (which is basically in a human-readable text file) and translates it into ones and zeroes that the computer will understand.
[*]"make install" takes the newly translated stuff and puts it in the right directory (basically it moves into the program files, which in Linux is the "/usr" directory.)
[/LIST]
