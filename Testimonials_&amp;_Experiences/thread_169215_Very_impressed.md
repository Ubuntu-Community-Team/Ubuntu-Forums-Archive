### Post by zba78 on 2006-05-02
Hi to all.

Unlike salmon, I'd had enough of swimming against the tide (more like a tidal wave) of ubuntu popularity. So after years of SuSE, Slackware, Vector, Arch Linux I decided to give it a go.

I'm using 6.06 on an Acer Aspire 1692WLMi laptop and I must say I'm very impressed. Sound, video, intel pro 2200 BG wireless network, bluetooth, touchpad & USB mouse, USB all work perfectly fine. I've actually never used Gnome as a desktop before (always switched between KDE, XFCE & Fluxbox) but I'm managing ok with it. I guess gnome has enough there to get things done (although I have to use Konqueror with kbluetoothd to get bluetooth working)

I was really impressed to see ubuntu work 'out of the box' with the keyboard shortcuts (the 'fn' key + something) and the hotkeys.

All I need to do is find a gnome equivalent of KWIFIManager so I can tell how strong/weak my WIFI siganl is. Any thoughts?

Oh and I might replace the default battery monitor with KPowersave as Kpowersave has options for setting schemes & CPU frequency. (unless someone can suggest a gnome equvalent for doing the same)

Anyway. So far so good with ubuntu. Well done to all the devs

### Post by geek.de.nz on 2006-05-02
Yes, I agree. It's the best distro I've used so far (by ease of use) and has the best support (these forums) with the quickest reply rate by people browsing the forums. I've been using (K)Ubuntu for a while now and am looking for unaswered threads myself now.

Here my hint:
Use Kubuntu if you're used to KDE or install KDE by apt-get. I prefer KDE as well (because of the short-cut keys for desktops mainly and I don't really find that it's slower than Gnome although people say that). Also, I use 5.10 (Breezy), because 6.06 is still testing (release date 2006 June, that's how 6.06 comes about). The support is best for Breezy at the moment, so if you're more concerned with usability like me, then use Breezy, because the support is best. If you want to be a tester go with Dapper (6.06). I used Gentoo before which is a bit much for me to be honest. I liked that you compile everything yourself basically, but in the end it took too much time, just installing a little application. Now I compile my own kernels happily in Ubuntu, because it's so easy with all the HOWTOs here in these forums. Try suspend2 ;). OK, gotta go. Hope that helps.
