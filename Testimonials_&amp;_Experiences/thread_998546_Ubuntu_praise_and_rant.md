### Post by Holonet on 2008-11-30
First off, I love everything Ubuntu *stands for*, i.e. free, lack of corporate control over our computers, community banding together to sustain something that no one could do alone...etc...  I'm very fond of how it gives Windows a run for its money (factual efficiency and capability, not user base, unfortunately), even with things stacked against it like few making software for it, Microsoft deliberately making it difficult to run Windows programs, etc...  It demonstrates just how poor and apathetic is the effort to produce anything beyond satisfying a user's superficial needs when someone can do better without even being paid...that all said...

Ubuntu needs a philosophy check in some departments in my opinion.  The idea of taking Debian and bringing it to the bleeding edge is not working.  Every release, a bunch of stuff gets fixed, and more gets broken.  Silly things like British broadcasting through Totem is implemented while I've still got to unplug my USB keyboard if I want Ubuntu's driver to handle a webcam.  Pulse audio is a veritable disaster...Youtube flash movies have to be closed in order for other programs to be able to use audio sometimes, printer drivers need work...I personally get colors misrepresented, title bars disappear every other update...  These things are basic functions and need to be put ahead of all the bells and whistles.  Also, the applications "approved" for use and thus in the repositories have more bugs than the Amazon.  

Ubuntu is running so fast, it's tripping over its own feet.  I'm not really a programmer, but I'd venture a guess that Debian is all that's keeping it from being a complete train wreck.  As is the law of physics...there's no such thing as a free lunch.  You can't cram frilly features, fix bugs, release every 6 months and still expect to eat your cake and have it too.  So my message to the developers is SLOW DOWN...fix the things that don't work before creating more things that don't work, and not only that--implement features that matter.  I don't care about my bluetooth when I have to track down 76 libraries before I can compile my what-ever....Right now, the toilet's working great, but you've got no running water.
