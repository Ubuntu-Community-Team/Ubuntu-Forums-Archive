And that is the reason for the development of the Linux Standard Base (LSB). Hopefully, that will allow various Linux distributions to use the same install package to install programs.

Until then, however, have you heard of programs like alien? alien can convert between RPM's, DEB's, and TGZ's. I suggest that you try it.

Also deb packages that are made for Debian GNU/Linux will also work on Ubuntu GNU/Linux (in most cases, at least).

[QUOTE=olecg]
This is sad, because in the M$ world the prog / app you need IS working / supported by the OS with minimum hazzel. (in most cases)

This is not always true. 
In MS Windows, there are many programs that do not come with uninstallers. You only realize this, however, only after installing the program. Then it becomes quite a hazzle to uninstall the program.

Sometimes, uninstallers for MS Windows don't even uninstall the program fully. Some leave registry keys in the registry, for example, unecessarily. This bloats your registry, which then requires you to use something like Norton SystemWorks to clean it up - another hazzle, especially if you need to do this on multiple computers.


Continue using GNU/Linux for a while, and I think you will see, just as I did, the impracticalities of MS Windows. After using GNU/Linux for 2-3 years, I simply can't imagine using Windows on a day-to-day basis again. Once all software companies support it (including game developers :D), most of the "problems" that people see with GNU/Linux will be resolved.
