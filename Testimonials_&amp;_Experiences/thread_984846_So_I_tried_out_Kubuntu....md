### Post by Luke has no name on 2008-11-17
One word: Meh.

Cons:
---I couldn't figure out how to turn window effects off (Compiz or whatever equivalent). Also, the effects were rather laggy and unsmooth, even with my nvidia binary 177 drivers on my 8800 enabled.
---Plasmoids are weird, and I don't see much of a point to them (personal thing).
---I prefer Gnome's menu better than KDE's single menu with the back arrow navigation.

Pros:
---The system settings center. I really liked its groupings better than Ubuntu's "Throw everything under 'System'" philosophy.
---The power management icon allowed me to switch between dynamic, power saver, and performance modes on the fly. I don't know if you can do that in Ubuntu, but I never noticed it.
---KDE's keyboard shortcuts menu was rather easily found by default, unlike gconf-editor, which I use on every install. 

Overall, I know I'm biased because I'm comfortable with Gnome, but I do like GTK/Gnome better, as well as Firefox and Pidgin over Kopete and Konqueror. I know I can apt-get what I want, but why?

CONCLUSION:
---I think Ubuntu should use Fedora-style gnome menus, with tighter grouping of applications like [Video, Personal, Inputs, Hardware] instead of having 15 or so programs listed in the "System" and "Preferences" tabs.
---If it were more easily usable and accessible, gconf-editor would likely be more widely used to increase user navigability. It would also be easier to get to by those who already know about it, instead of having to apt-get it or add it to your "system" application menu manually.

Also, While Java and javascript didn't lag much in Fedora 10, Ubuntu has terrible performance from me. I will run tests in the next few days to determine what the problem is. If there is something amiss that is not my fault, there is a serious problem in Ubuntu if its JRE runs 10x slower than Fedora's.
