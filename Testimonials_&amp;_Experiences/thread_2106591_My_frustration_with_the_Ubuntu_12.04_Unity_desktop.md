### Post by Dr Kayak on 2013-01-19
This is a description of my frustration with the Ubuntu 12.04 Unity desktop.  I'm not seeking help with the specific issues mentioned, or advocating that others do their computing the way I do mine.  I just want to express my opinion, with the hope that it might make some small impression on a mover or shaker with the power to preserve my (apparently unusual) computing way of life!

I've been running Ubuntu 10.04 for years on three machines, and finally installed 12.04 on my laptop.  I've been trying for several days to customize the Unity desktop so it doesn't get in the way of my work, with many frustrations along the way.  I think the answer for me is to abandon Unity in favor of Gnome Classic, but I was worried that the latter won't be supported indefinitely, and I wanted to give Unity a fair shake after reading so many positive reviews.  Here are many specifics.

[LIST]
[*]I use many applications spread over four workspaces.  In 10.04, I had a small image of the workspaces visible in the bottom bar, in which I could see which app's were running in which workspaces.  The icon for the Workplace Switcher in the Unity Launcher doesn't give me that information, nor does it indicate which of the four workspaces is active. Instead, I have to click on the icon to get a detailed look at the workspaces, and then double-click to select a workspace (or use CTRL-TAB-arrow to navigate through them).  

[*]My bottom bar in 10.04 also had a listing of all the applications running in the current workspace, which was very helpful if one window was completely obscured by another -- I could just click on the icon for the hidden window.  When that happens in Unity, I have to move the windows around, or use keystrokes to switch between windows (ALT-ESC, I think).

[*]In 10.04, I put icons for commonly-used applications (e.g., terminal window, firefox) in a bar at the top.  Clicking on the icon would then launch that application in the current workspace.  In the Unity Launcher, if I left-click on an icon, it will take me to whatever workspace might be running that app -- unless I remember to right-click, which will bring up a menu with an option to launch a new window for the application in the current workspace.

[*]I used gconf-editor to get the windows buttons always put in the same place (I chose upper right), and then discovered that I have no choice about the Global Menu at the top, which inexplicably becomes the control bar for a selected application running in the current workspace.  

[*]Not liking the way the Launcher overlaps windows that I'm working in, I reduced the icon size and later chose to auto-hide the Launcher.  It's supposed to reappear when I move the mouse to the left side of the screen, but some fraction of the times I try this, it doesn't work, and I have to bump it again!  

[*]In 10.04 I knew exactly where to look for things in the Applications, Places and Systems menus that were fixed in a bar at the top.  In Unity I have to hope I can come up with the correct keywords for the Dash to help me find what I'm looking for.

[*]I can't save the settings for my workspaces, the way I used to be able to do in 10.04, so I have to set things up from scratch each time I log in.  

[*]Applications seem to launch more slowly in Unity than they do in 10.04 or in Ubuntu 2D.

[*]This last one is probably the most surprising to me (and the least important from a work point of view).  I can't play videos fullscreen in Unity, unless (i) I rapidly click the mouse on the fullscreen icon a large number of times, or (ii) I log in with Ubuntu 2D to disable video effects (which I could do via settings in 10.04), in which case the Launcher defaults to its annoying large size and the option to re-size it has apparently disappeared!  I explored several other options talked about on the web, but none worked for me.
[/LIST]

### Post by tartalo on 2013-01-19
A quick search in the forum will show that you are not alone. But Unity has been developed in an effort to create a single UI that will be usable in big screen PCs with keyboard and mouse, small screen netbooks, TVs, phones, (fridges?)... So it's unlikely it will change back to something similar to what you knew and loved.

I know you didn't ask for help, but you don't need to use something you dislike, people have dealt with this in different ways:

- Keep using Unity until you finally like it

- In 12.04 Gnome fallback mode is still available, it's similar to Gnome2 but it's a short term solution because Gnome has already abandoned it.

- Change the Desktop Environment (KDE, XFCE, LXDE, Gnome Shell, Cinnamon, Mate, Consort, Openbox+tint2, Enlightenment ... you name it, my personal choice is KDE)

- If you are a LTS user, better than just changing the DE switch to Kubuntu (KDE) or Xubuntu (XFCE), Kubuntu has 5 years LTS and Xubuntu 3 years.

- Switch to another Distribution that supports officially the desktop environment you like, for example some people chose Mint because Mate and Cinnamon are well supported.

- Finally, some people opted to switch to another distribution that promises more stability. Unity is not the only change to be expected in Ubuntu, another big change in the roadmap is Wayland. Ubuntu has been delaying this, but it's unlykely they will abandon the idea, because it suits the phone strategy well.

Hope this helped.
