### Post by michaelkahl on 2008-10-02
Ok, ok I know this is Ubuntu Testimonials.  

I work in IT for a school, and at one of our locations I was checking on some of our lab machines.  The PC's were left on over night so I rebooted them.  Everything went fine on the first machine, but the second machine was different.  I got about 5-6 error messages about .dll files.  Try opening any application and you get a .dll error.  I reboot the third machine and coloring is off in the windows.  A good portion of it is black, and there are other graphical errors.  It at least reboots just fine.

Back to machine #2, in the end it was easier to dump a new image onto it and call it a day.  These are XP SP2 machines setup with Cleanslate.  If you are not familiar Cleanslate is a nice piece of software that helps keep a Windows installation and local hard drives in the same state from the time you enabled the software.  

My point?

We all have bad days with operating systems.  Doesn't matter if it's Linux, OSX, or Vista/XP.  They all have their imperfections.  Unfortunately newcomers to Linux can be harsh because they really aren't familiar and expect Linux to function just like Windows.  Linux is not Windows and won't be.  People who choose Linux want something different in an operating system, and for one reason or another OSX or Windows don't meet that need.  There are always learning curves though, and each day you pick up on something new.  Let me share with you what I learned last night.
Improper file permissions on one file can prevent a user from unlocking a session.  My /sbin/unix_chkpwd had improper permissions set.  Once I set proper permissions I was able to unlock my session.

I don't see this as overly complex or ridiculous, just different from Windows.  I've used Ubuntu for a year now, and exclusive for about 5 months.  In that time I picked up on how on file can effect a system.  So in my google adventures I started looking for files and how they effect unlocking a session.  This lead me to a post on unbunteee's forums.  

I know it's frustrating when you are trying to configure or customize and you break the system.  Fixing it is better than reinstalling.  Reinstalling may seem easier/faster sometimes...but you don't learn as much.  So if you are enjoying and wanting to learn I suggest fixing it.  

I'm still learning and enjoying it.
