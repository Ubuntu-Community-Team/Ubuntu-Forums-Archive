### Post by spudw on 2006-08-22
well i was running kubuntu 6.06 and i liked it alot. completely formatted my hard drive and got rid of win2k for kubuntu. well i tried to get my bro and dad to try it. my bro likes it and hes willing to take the time to learn how to use it on a day to day basis. my dad on the other hand, well i gave him the cd 2 weeks ago and he still hasnt tried it out. after i thought about it, i was kinda glad he didnt. hes the kinda guy who thinks he should never have to use a command line. hed rather spend 200 bucks on xp pro than type sh launchEU.sh in a command line. ill keep my 200 bucks thanks. so anyway, i went looking for a distro someone like my dad could use. i read good things about pclinuxos so i thought id test it out myself before i burned him another cd that would probably end up as a coaster. i downloaded pclinuxos junior .93a, roughly 480mb. much smaller than a ubuntu/kubuntu iso. i liked that. install went smooth and quick. kubuntu install took 2 tries. i rebooted to find lilo as the boot loader. my dad would like them perty colors. logged in and a neat transparent theme for my desktop was default. sweet! mp3s played without codec download. very nice. all important settings have a gui frontend in pclinuxos control center. perfect so far!

ok now i want firefox cuz i dont much care for konqueror as my browser. open up synaptic, and i am greeted with my first disappointment. the only repository in my list doesnt work. cant get any updates! no!!!](*,)  ok so i go looking for mirrors and i add a few to my repos list. some work some dont, no big deal. i install firefox and nvidia-glx. cool now my screensavers are hardware accelerated! now that i have 3d, i wanna play glest. i try to run the loki installer from konsole and it tells me i need a newer version of glibc. i go looking in synaptic for an update to glibc and it cannot be found! so i scour the web to find that a newer version of glibc will be implemented in version .94 of pclinuxos, but that wont be out until LATER THIS YEAR! im a patient man, but not that patient so back to kubuntu i go.

well, now im back in kubuntu, and im happy again. so is my other bro, with whom i share this computer. he wasnt in to computers until i showed him kubuntu and now he wants to buy a system76. 

i liked pclinuxos for the most part. i dont like rpms. i dont like not being able run something as simple as a loki installer because i cant have a newer version of glibc until later this year. all in all, this distro is near perfect for linux converts and noobs. its perty and i had few problems with it. i think it would work great for someone like my dad. as for me, im stickin with kubuntu

### Post by msak007 on 2006-09-02
I'm also a Kubuntu user and always come back after trying other distros. My brother is still using XP pretty heavily, but he's started dabbling a little with Linux ever since I first started using it. He's tried Kubuntu, mostly stuck with SuSE, and today saw PCLinuxOS on DistroWatch and decided to try it. I saw a demo of the live CD on his laptop (I think he tried the biggest version of it, it's called BigDaddy or something like that) and the first thing I noticed is that there were *way* too many apps on there by default (maybe you get a choice when installing, not sure). The only thing that I saw that impressed me was their Control Center - it reminded me of SuSE's YaST a lot. Practically a GUI for most settings that you would have to hunt down using a CLI, for those that like that sort of thing. The default package manager is Synaptic, but I was quickly turned off from this distro when I saw it was RPM based. To be fair I didn't actually *use* the distro enough to get a feel for it, but didn't see much else aside from the Control Center to make me give it a second thought.

### Post by Neobuntu on 2007-02-03
I think you should install Kubuntu Dapper (Alternate CD for 128MB to under 256MB of RAM).

Dapper is much closer to stable perfection than Edgy (right now). Edgy pissed about half the people off (AT FIRST) with some growing pains. They say hey, it's "Edgy". I say it's "beta" and should have been released as such. Yet, they really pushed the envelope and laid the base for Fiesty coming up.

I would ALMOST say go ahead with Edgy now but it's stabilizing more and more every day. You can reinstall up to Edgy later as it gets even better. DO NOT (dist-)upgrade because sadly, it is not as easy, clean or time respecting as saving your stuff and reinstalling (Note: You might want to download and burn the live GParted boot-able CD for separate XP/NTFS partitioning and to have around. It's a quick download). This trouble is un-like the ease with upgrading from Breezy to Dapper (a sad digression but it may get better). So be patient and run Dapper today.

Addendum: For the unstable testers out there, Kubutu Fiesty Herd 3 (Alpha) "Alternate", actually installed well on a 128MB notebook and is working great (day one). This means the (second of dual boot) install went well (check your homemade burned CDs). 1. the md5 checksum of the iso matched the md5 test file online. 2. I checked "verify" the burn and it did. 3. Boot and run the CD integrity check. This is NOT my main system, though. Don't run it as your only system.

Kubuntu is faster than Ubuntu (especially if you use KDE apps) and you can turn off all the GUI crap (like menu roll out) and use Konqueror for memory savings (It is less than Opera, unless you run Ubuntu/Gnome). Use Kopete for IM(ing). It comes real close to Xubuntu's lower memory usage and if you are REAL carefull, one can even run it with 128MB (You have to install with non-live alternate Dapper CD). 

***Added: On my Kubuntu system I just ran some tests and Firefox 2.0 uses a little LESS (about 6MB less) memory than the newest Opera! Still Konqueror uses FAR less (about 26MB less). With Konqueror and Kopete running I'm using total, about 192MB (of 512MB) RAM. That's old computer teritory.

Get more memory. Yet many systems are limted to about 192MB MAX. Some 128MB and some 96MB(read below) Max.

Nothing against other DM's or distros but Kubuntu and added Automatix are the deal right now. Dapper has your LTS stability. After Edgy, your "Fiesty 7.04" upgrade future is the best. 

I do not recommend Ubuntu Lite at this time (though one could install Ubuntu server and add any lean DM and custom build it up, if you have the time). Starting at 128MB RAM or less, one could run Puppy Linux loaded automatically into memory for super speed but at a slightly less familiar interface and with much less community. Still, major apps are accounted for and readily addable once you understand the conceptual design. 

Anything less than 128MB might best look at Puppy Linux (Installed as if a CD on the HD; from the Live Puppy CD) and if that doesn't work, try DSL Linux. Though I find DSL Linux less real world usable than Puppy. It depends on how demanding you are. If you just want to boot the live CD and run VNC terminal from your faster computer set up as VNC server then you're good to go (and fast) for all but direct gaming/video. It's a zero setup game (on the old computer/client).

Puppy is a closer approximation and more similar to what you can actually get done with Kubuntu. Puppy can be tried in less time. It's a fast download. There are many Puppy releases/builds to choose from.

If none of these time saving ideas work on your hardware, there's always Debian. Download a "testing" tier CD with something less that KDE included (for really old systems.) With 128MB or more get the KDE one. The repositories (and ease of installing packages) are your friend. It's the only thing working on a (real) old thin Pentium Thinkpad notebook I have (24MB RAM!). The Video panel is a very proprietary 800x600. Slackware might work on SOME older systems better. Now we are getting uber geeky. Is it worth it? I'd try Debian before Slackware first, but that's me.

Do not use an old (version) release!

One could dual boot with their copy of Win98 but it's no longer supported (very not secure online) and takes FAR longer to build up, update, tweak and there's rarely room on the Hard drive for a dual boot on these old; less than 64MB systems. I'd choose only Linux; for an old system and dual boot newer boxes, if you must run native Windows apps.  Though, it's hard NOT to find a native Linux replacement today. 

So, there are options for just about any hardware you'd want to keep. Well back into the give-away set. Of course, there's no substitute for newer hardware, just don't *BUY* Windows with it. That said, don't trash any working system. At least give it away.

Oh, and you Quicken people, now is the time to start anew (with 2007 beginning balances) on Kmymoney. You can still load up Quicken for historical looks ups (Dual boot or even in Linux with WINE). Kmymoney is not as bloated as Quicken and that's a good thing. 

Get free.
