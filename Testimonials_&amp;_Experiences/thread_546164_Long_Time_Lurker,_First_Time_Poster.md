### Post by rmrector on 2007-09-08
So I've been a big fan of Ubuntu since I first heard of it with the release of Breezy Badger. I've attempted to go Linux full time with each release, but there was always something holding me back. Usually it was digital camera support, or printer support. I'd usually switch between Kubuntu and Ubuntu, and I could never decide which I really liked.

Though I had found success installing Xubuntu on my server (I was just a bit too afraid of installing Ubuntu Server, going all commando command line only was beyond my comfort level at the time) last year, I still couldn't commit to it on my main machine.

Then 7.04 came a long and made me a happy boy. Camera finally worked well. Can't say for certain about the printer, I don't have much use for it this time around.

Finally decided that Ubuntu was for me. I like the theory of KDE apps integrating together nicely, but in practice I didn't see it happening, and it was just... I guess it was too polished for me, I was unable to be productive. Gnome it is.

Though I have found some quirks in Gnome that still bug me a bit. No right-click and drag? Bah. Copy/Move was the only reason I use it, and I know, Ctrl/Shift drag takes care of that. But I still find myself trying to do it my old way every once in awhile. And no programs seem to have a default button? For instance if I've got the file open/save dialog open and I type in the path, then try to hit enter instead of clicking on the OK button, it just stares at me. I end up needing to tab to the button or click on it. Grrr. That's where I notice it the most, but it seemingly happens on everything. Though I am more likely to notice when it doesn't work rather than when it does, so maybe it works on some apps and I just don't notice. Decent photo management software for GTK/Gnome? F-spot is slow as molasses, and I've only got about 200 photos. I suppose that's to be blamed on GTK#/C#, though. Eventually when the Sharp interpreter grows up I'm sure it will speed up a lot, but developing applications meant for others to use with it at this point in time just seems... like a very bad idea. I've found that on Linux, Java is multitudes faster than C#. As is Perl (though it doesn't scale well when applications get larger) and Python (my language of choice at this point).

With the fact that my complaints/quirks paragraph is the longest of this post, you might think I'm unhappy. Not by a long shot. If I had written up a list of the complaints/quirks I had when working with Windows... well, I don't want to spend so long typing that up. Nothing is perfect, though with Gnome/GTK and Linux in general, I myself can do something about that. The beauty of Free software.

I also have just ordered a Lenovo T61 laptop (mmm... IBMers still get killer deals every once in awhile), and I'm looking forward to wiping Vista off that (with prejudice!) and rocking the Gutsy next month.

And speaking of rocking: :guitar:
