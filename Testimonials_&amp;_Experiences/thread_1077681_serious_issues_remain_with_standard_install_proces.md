### Post by nhcatsteve on 2009-02-22
I've just transitioned to a new external hard drive to boot Ubuntu
from, and upgraded to the 8.10 version.  It's definitely an
improvement... it supports the drivers for my integrated wireless card
pretty easily now.  I tried setting up connection sharing, but the
manager keeps setting eth0 back from static to dynamic, to make sure I
stay online in some fashion I guess.  Thus firestarter doesn't think
eth0 is ready.  I tried 3 different walk-throughs specifically for 8.10
and eventually gave up.

Also, I had to install with the "OEM Manufacturer install only" option
from the live CD, since the standard install would just hang part-way
through boot up (probably a hardware-specific issue).  I asked a techie
guy I know at the town dump if he'd tried Ubuntu yet, and he said when
he hit that glitch and a reboot didn't get past it, he just gave up.
I'm amazed at how far Ubuntu has come, but there's still a lot that can
be done for mainstream friendliness.

Everything should have a GUI at least for all common functions, fewer and simpler options should be given at a time, and it should be harder to mess up basic functionality by following the instructions in walk-throughs.  Such commands are usually very technical, and it takes a very thorough user to check for issues and undo whatever messed it up.  Clear, specific error checking and messages should be present at every level.  

I don't think Ubuntu is quite ready for the mainstream yet, but it's getting very competitive vs Micro$oft products - partially because M$'s approach has been so bad lately.
