### Post by flavouride on 2009-11-08
Thanks all for spending a few moments reading my little opinion.

The main reason behind switching to Karmic Alphas was out of my curiosity for ext4, which was a lot faster than ext3 for file transfers and system startup, yet in my personal experience not quite stable and often suffered from kernel panic in Jaunity's then 2.6.28.x kernel. This was concluded by my then first tune2fs and fsck upgrade and then it went so wrong that I had to perform a fresh install and manually chose ext4 as my fs for the single partition for / + /boot + /home but again it failed horrendously so I simply "update-manager -d" to opt for a change, that was Karmic Alpha 2.

As a USB audio device (Sony PC link kit to a mini hifi) user, to my surprise, the pulseaudio integration in Karmic means I could set my usb audio as my default audio device with a few mouse clicks and using it in everything.

Unlike Jaunty which requires the installation of pulseaudio volume control, default soundcard chooser, pulseaudio common, and so on, in addition to alteration to the default soundcard to pulseaudio, default device to usb audio in pulseaudio volume control, pulseaudio server to set "default sink" to usb audio and User Account to "allow myself" to use the usb audio device.

Too much work!? Not quite yet. I still needed to use pulseaudio volume control to manually switch the "soundtracks" of flash and java apps seperately to usb audio in order to get them to use my usb audio device. Yet after all there is still no way to get my usb audio to play the ubuntu startup sound in Jaunty.

But of course the majority of us would have known that a series of unfortunate blunders occurred in Karmic Alpha 5: I was eventually fed up with the daily routine of partial-upgrading to try out new stuff then downgrading a few modules to keep the system running(!!!!), and I decided to switch back to Jaunty with ext3 before the release of Karmic Alpha 6.

In doing so and according to my personal experience, I noticed the difference of the speed between Jaunty and Karmic in terms of start-up, processing and shutdown, where the latter outpaced the former by far yet I did not have the balls to even touch Karmic RC having read so many complaints about it in this site.

Nonetheless now having run the final release for more than a week now and everything has been quick, stable and utilised and there is no reason for me not to recommend it to everyone of us. :D
