### Post by BeWop on 2010-10-21
Running on:
Acer Aspire One
1.66 GHz Intel Atom N450 Processor
1GB DDR2 Memory
160GB SATA Hard Drive (5400RPM); 802.11 b/g/n Wireless Certified
10.1" WSVGA Acer CrystalBrite LED Display, Intel GMA 3150 Graphics

Fresh install, clean harddrive.

_**Unity**_:
*Pros:*
VERY impressed with the interface, love it. I can see how excellent it would be especially for touchscreen laptops or tablet PCs. Everything was clean, very easy to get to quickly, and screen real estate was great. I would imagine the experience would be even better for a touchscreen.
LOOKS really cool.

*Cons:*
My netbook couldn't handle it. The interface was really slow, and I'm pretty sure my netbook is generic and standard among netbooks. I would hit the app button, and that would load fine, but then all the tabs would take a while to launch, and the tabs would lag. Not nearly as efficient as GNOME, though it would be if it wasn't as slow. The flashyness is not a good trade-off for slow. 
Most programs load in full screen. This bugged me, as I would open a chat box, or rhythmbox, clearly not meant to be fullscreened. 


I ended up switching back to gnome, which ran gloriously. Seems much snappier than it did in 10.04.

_**Software Center:**_
*Pros:*
Much better, everything is central. I can run a .deb package and it will run in the software center and log it. A great way to organize and keep track of programs that I have seemingly installed. I love it.

*Cons:*
Really really slow on my little netbook. Software center was slow before, but this is even slower. When a program is installing, I can do barely anything else it seems, everything responds way slowly. It also seems to freeze a lot right before the programs get done installing. Quite annoying.
The only thing in the "For Purchase" is that silly little dvd player that seems to think that people will buy it. That right there looks really silly in itself. I would contact the companies that make software like "Crossover" or even "World of Goo" and get some more software in there.

**_Random_**

Power management is great! I used to get 3-4 hours on my battery, and now easily getting well over 5. Always good.

I love the new fonts. They create a whole new feel to it.

The music integration is amazing.

The installer is very a nice, was a bit slow and froze a bit on my netbook, but that is to be expected. I love how you can click on the "support" thing and firefox will pop up, so you can browse while you're installing.



That's all for now folks! I'll update if I see anything else I want to write about. Very solid upgrade, love it!

### Post by sunset on 2010-11-08
I installed Maverick Netbook Edition on my Asus 1000HA over the weekend.  It was previously running eeebuntu 8.10 (Intrepid).  Maverick went into a separate partition so I could still boot Intrepid while tweaking the new system.

Very nice to find that the internal wi-fi and my Brother network printers (HL-2170W and HL-4070CDW) were supported out of the box.  This was not the case with Intrepid.

I like the layout with the launcher on the left leaving more vertical real estate.  Yes you can do that with Gnome but it doesn't look as nice.  Also it's nice to see some apps putting their menu in the top bar to further save space; I assume over time more of them will do this.

Didn't notice any performance problems.  The Atom N270 is not very speedy, but DRI was enabled OOTB and everything seemed at least as fast as Intrepid was with Gnome.

This is a dual display setup, using an external 1680x1050 display with the netbook's screen still available.  Of course when mobile, only the built-in display is used and that works fine.  All that worked with Intrepid and also now with Maverick.

There are/were some oddities:

o After installing Maverick, trying to boot Intrepid would fail with a "missing file" error.  This problem went away after applying Maverick updates.

o The dual monitor support applet went crazy and locked up the first time around, perhaps because I did not attach the external monitor until after bootup.  But it worked nicely after rebooting.

o Gedit and the console window do not save their state on closing, but they do if the system is rebooted while they are running.

o There is no obvious way to add Picasa (installed via its .deb package) to the launcher bar, or even to run it from anything but a shell prompt.

o The desktop seems unavailable for any custom icons.  This limitation seems unnecessary.

o I could not figure out how to set up a CPU Frequency Scaling applet. This is a major problem since the system is geared for battery-powered devices.

o The printer applet was buggy and quirky when setting up network printers.

Everything considered, I like it and look forward to further improvements.
