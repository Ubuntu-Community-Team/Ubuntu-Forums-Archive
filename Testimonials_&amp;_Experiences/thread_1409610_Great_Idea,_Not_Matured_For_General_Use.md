### Post by sophicsage on 2010-02-17
I have used Ubuntu exclusively on my Sony Vaio laptop since November 09.  It works out of the box, in general.  However, there are serious end-user issues that can stand in the way of reliable, daily use for people who use their machines for work.  

Most workplaces use Windows programs, that's just a fact.  They can get reliable service from Microsoft and compatibility isn't an issue usually.  With Ubuntu, you must depend on other users and sometimes search _extensively_ for solutions on message boards.  For someone with a job working many hours a week, there just isn't time to play around with terminal code.  It needs to just WORK for everything.

While Ubuntu "works", it is not worth the price (i.e. free) to give up reliable program compatibility.  I'd rather pay for a tried and true operating system than get a free operating system that doesn't work like it should.

I would recommend Ubuntu to users whose older machines are on the brink of being obsolete.  But for users with newer machines who depend on them for work, I must recommend Microsoft.  

Sony is shipping me a recovery CD with Vista Home Premium.  I still have the Windows 7 Upgrade disc, so I can get back in business.  

I should have created a dual boot system, but on a leap of misplaced faith I created an Ubuntu-only machine.  I will attempt to have a dual boot system.  The Sony tech I spoke with today said that Ubuntu was a good operating system and that I shouldn't have any trouble having a dual boot system with Windows 7.  This is good news to me, because I want to help out with the open source OS movement.

The only reason I am putting Windows back on my machine (blah) is because I have to in order to work reliably from home or on the road.  There are programs that I simply must use to get paid, and that's all there is to it.

Here are my recommendations to the Linux programmers:

1) Be more flexible on using proprietary drivers.
2) Focus on webcam compatibility and usability.
3) Make it easy to install programs without messing around with   
   terminal commands.  Make as much as possible available through  
   a GUI.  Average users just WON'T use it for many things if
   they have to spend a lot of time setting things up.
4) Get Wine to work better.  Wine just doesn't work right much of 
   the time.  You'll get partial installs or Windows panes will  
   be incomplete.
5) Make it easier to use mpeg, avi formats out of the box.  
   Things like OGG files and Theora confuse people.  I've gotten  
   used to it, but most people won't tolerate having to convert  
   files.

Again, this is an honest opinion piece written not to criticize, but to improve the open source movement.  I hope this review helps to improve it in the future and eventually mature this OS line.
