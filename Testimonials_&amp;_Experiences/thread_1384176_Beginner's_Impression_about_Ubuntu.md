### Post by Vahagn_IV on 2010-01-18
Hi, all.

I want to share my impressions about Ubuntu Linux with community. Sorry for my awful English, but I really want to tell you what I have found good or... not so. 

**1. Why Ubuntu?**

Well, the decision to pass from old good Win XP to Linux Ubuntu was.... the Microsoft corp. It seems that they require people to buy their new products. The last thing that has completely broken my patience was *.docx  format of word files. It would be better for Bill just to ask people give him more money and let them use their old MS office files. 

On the other hand Ubuntu with most of its programs  is completely free and they have no reason to change their file types. Also, a friend of mine is experienced linux user and he has suggested to start from Ubuntu. "It is the most easy to learn linux distribution", he said."Start from Ubuntu and, I think you will like it". He gave me a boot CD with Ubuntu Jaunty 9.04.

So, injured by Microsoft's lawlessness  and inspired by my  friend's  advice I have started my linux way.

After a small fight with gparted (It was hard to figure out that one should Unmount drive before making changes) I installed Ubuntu on my HP Compaq 6720s(About swap partition I have learned from that friend).

**2. Terminal**

I suppose anyone, who had ever used any of linux distributions had to deal  with terminal. "That's what you have bought your keyboard  for" I read in linux.org. No, man, I have bought my keyboard because it was hard to find laptop without it! IMHO, in ideal case, user should use his keyboard only for typing passwords and text. I cannot pass to /root directory with Nautilus even if I know root's password. Don't you think, that it would be better to ask the password and enter, but not to  throw errors.

It seems as if people that think so have deal only with university  professors or so. Let me tell you a story I was told by a friend of mine that works in a computer store.

Few years ago a man entered in that store and asked for "A beautiful Pentium". By the way, he was dressed by Armani and wore huge golden chain. The seller was a bit confused by that unusual requirement but suggested to choose the good from vetrin. Our client chose a laptop and  then asked to try it. When my friend enabled it Microsoft's label appeared: "Windows loading". 
"This is not Pentium", said client with a face as if he discovered cheating , "It is Windows!!". The seller thought that if he will sell  him that "Windows"  he'll have great problems, afterwards. "Oh, sorry", he said,"we have no more Pentiums, but only Windows.". 

I'm sure, that guy bought his "beautiful Pentium" and now uses it. Do you really think he can use Synaptic and figure out what means "dependency"? Or can he type "sudo dpkg -i something"?? IMHO, terminal should be  for advanced users only. 

The suggestion to Ubuntu Community:

Do something with installation process and that great amount of packages with unintelligible names. 

**3. Media**

What usually people use their computers for? Of course, it is playing music and watching DVDs. So, I passed to my mp3 directory and double clicked on the first mp3 file(It was Metallica-The unforgiven II.mp3). Rhythmbox has appeared and what do you think he told me? Totem(what??) cannot play bla-bla-bla. In a few words he asked to download something. "Windows Media Player(WMP) does this by default", I thought. To be completely frank, should say that it has automatically downloaded and installed everything.

Honestly, I hate WMP and use it only if  there is no alternativity. Rhythmbox looks like it and makes me nervous.  So, what people usually do in common cases? Yes, they type google.com and start to look for  mp3 players. I have found that most of Ubuntu users use Amarok and they like it.  Damn, it is a monster!! It has more configuration options then MS Visual Studio. Where is my winamp? Installation of xmms leads to dependency problems with some libblibblubblab. The problem has not been solved up to now.

 The battle with DVDs was much harder. There were some restricted somethings, some repositories. But I have won it!!!

**4. Software**

Let me start this section from a suggestion to linux software developers. If you are not from Sun Corp., then you should not consider using Java. It is extremely slow both for windows and linux. 

 [CENTER][B]Java is for Sun use only. 
[/B][/CENTER]
There are many cross platform packages. Consider using them.  

I have already mentioned that installation process is very complicated in Ubuntu. Even after installation, it is hard to make changes in programs. A good example of this is Kile and latex. It does not install packages on fly. You should agree that it is annoying to pass the installation process every time you need to install packages. 
Even after installation of texlive-lang-armenian package(for typing armenian) it does not work properly- the beautiful letter "&#1390;" is missing. Since many of armenian abusive words use that letter, I cannot swear in latex now. This is the first one of two reason I have not completely  removed Windows installation from my computer.

I understand that this is not Ubuntu's issues, but you should agree, that the value of operating system is not only in it, but in its software also.

Now, let me  stop on program development environments of ubuntu. I have found three of them- Anjuta, KDevelop and NetBeans. After the first launch of Anjuta it was completely removed and saved in my blacklist. It is not environment but a template of it. The second one was a bit better but still  did not allow me to compile simple "Hallo world!" Console Application. There was a problem with libtool2. I had to downward  it to version 1.5. Although it has compiled the Console application, it threw errors every time I tried to do something new. The best choosing was NetBeans but, unfortunately it is written in Java and, as a consequence, is very slow. 

IMHO, Microsoft has made a great decision  to create its masterpiece- MS Visual Studio and MFC. It is a great tool and makes program development as easy as possible. 
Let there are two guys- mathematician and programmer. The first one has developed an algorithm and wants to write a program. One would say:"Let them write the program together". No, man. They live in the different sides of the world and do not know each other!!! The mathematician should be able to write that program by himself.

 I dare to suggest to Ubuntu developers:

Create something similar to MS Visual Studio and MFC and the number of Ubuntu programs will rise steadily.

I  really love playing games- strategies. Have you ever tried Creative Assembly's products from Rome Total War to Empire Total War? If not, try and you will like them. 3D military map, beautiful units, detailed graphics etc. They are really cool! But there are no linux versions of that games :(. And this is the second reason for I kept my Windows instance.

**5. And so, what is good in Ubuntu?**

Windows user would understand me. After installing any operating system user starts to install the applications that he will use afterwards. More applications installed more things user can do. This means that day by day computer must become better and better. But, while Ubuntu does become better, Windows proves to be heavier after any installed program. It becomes unstable and  throws his Send/Don't Send windows. Although one should make a small fight with dpkg or apt-get for program installation, there are usually no system problems afterwards. 

The best achievement of Ubuntu's developers IMHO is its stability.

**6. Summary**

Above were described most of problems(that, I'm sure will be solved) I have found in five weeks of using Ubuntu.
So, we have a good(still a little complicated) operating system with small amount of programs. Well done guys! Keep on the way you are doing!

### Post by 3rdalbum on 2010-01-18
This is computing. You're going to have to expect some jargon like the word "dependency", just like how if I started using Windows I'd have to learn jargon like "virus" and "safe mode".

You don't need to use 'sudo dpkg -i'; Debian packages are installable just by double-clicking on them. I haven't had to use sudo dpkg -i since 2006. However, on Windows I've had to use "ipconfig" and "rundll32.exe". Recognise those?

The reason why there's no MP3 support by default is because of software patents. The Fraunhoffer Institute and Thompson Multimedia could go after Ubuntu for patent violations if Ubuntu shipped MP3 support. Microsoft pays this extortion money, so they can include MP3 support with Windows (a paid product - in effect you are paying Thompson Multimedia for the ability to play MP3s every time you buy Windows).

Same problem but more acute for DVD decryption.

It's not hard to enable DVD decryption, it's just different to what you're used to.

Ubuntu already has something that, IMHO, is about as awesome as Visual Studio. It's called Quickly. It features the Glade interface designer and some documentation, and also includes a program template to get you up and running (with a main window and even DesktopCouch integration). The backend language is Python, which is easier and quicker to write in than Basic.

Quickly also automates version control, builds Debian packages of your program for you, and can optionally sync your changes with Canonical's Launchpad project hosting service.
