While your approach is admirable, it's going to suffer from personal bias -- folks are going to post their opionions -- which may or may not translate to the same performance on your PC.

The only way to see how well it will work for you is first, to run it from liveCD.  That will show how well it detects your hardware and installs the proper drivers.  If you're going to have video, audio, or networking problems, they will show up then.

The only way to test the real-time performance will be to install it yourself.

### Post by xyzzyman on 2011-12-15
I just switched back to Ubuntu from Windows 7 back in September and now only boot into Windows once a week at the most if I want to play a game for a few minutes. In the last two months I have tried Fedora 16, OpenSuse 12, Arch, and Linux Mine 11 and 12. I've found I like having Arch in my grub menu as my backup OS and it boots extremely fast, but in the end even after taking the time to tweak and configure the other OS's I just like Ubuntu w/Unity (Yes suprise someone likes Unity). 

In 11.10 I don't have any graphical issues that I would get once in awhile w/11.04. By default I think it's ugly but I throw in the Adwaita dark theme, tweak the fonts the way I like, shrink the icons down on the side and lock it in (I don't like the taskbar disappearing and reappearing) and install Guake and set it to use the tilde to open a console.

Memory being released seems to work as it should. Compiz floats between 100MB and 177MB's for me, but never goes above and I haven't noticed any memory leaks occuring. Sometime's I'll have a Win7 Starter and an ArchLinux virtual machine running in Virtualbox and when I close them all the RAM is back for my use. Without tweaking it is good on using my RAM without touching the SWAP. 

Now if I had to use Unity2D I'd just wind up switching to Gnome 3 or Gnome-shell. I wasn't too impressed with the performance of the RPM's repositories in Fedora and OpenSuse, so if that was the case I might just go to Arch over Ubuntu but as long as Unity keeps solidifying I'm here.

Plus now whenever I go into Windows I'm annoyed and just want to reboot back into Ubuntu!

### Post by 3rdalbum on 2011-12-16
> **hartz said:**
> 
Linux Mint is much better in terms of the memory utilization.  But it is worse in every other regard - stability of window / special effects, speed with coping suffers even on normal hard drives, I can't get the irritating Gnome 3 panels to get out of the way quickly enough, can't add things to the panels, dislike the way themes work severely, regularly need to disable and re-enable networking to get a connection working (particularly after resuming from a suspend), my list of gripes goes on.

Did you know that Linux Mint is based on Ubuntu? It's like a thin coat of paint on the surface of the Ubuntu house. If you've been having trouble with something technical on Ubuntu, you'll have that same trouble on Mint.

Unity is based on Gnome 3; the panels are similar but not identical. Your complaints about Gnome 3 infrastructure will be present on all distros that use Gnome 3. Gnome 2 is obsolete. Dead and buried.

If you'd like solutions to your technical problems, here's two ideas:

1. Submit bug reports, if you haven't already. If the developers have not heard of the problem, they can't fix it can they?

2. Try a distro that is not related to Ubuntu.
Debian, and anything based on Debian, is related to Ubuntu and might suffer the same technical issues you report. Maybe not, but probably. You'd be better off trying Fedora or OpenSUSE as these are not related to Ubuntu, and they're also not related to each other.

3. Chill out about Gnome 3. They should have given it a completely different name, because really it's a completely different desktop. As such, it works differently and you won't be able to use it exactly the same as the old Gnome 2 desktop. If you take the time to learn it or learn Unity and become familiar with it, you might find yourself liking it. You can't carry over old habits though.
