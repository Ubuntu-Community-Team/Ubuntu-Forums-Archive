### Post by stabu on 2007-05-12
Using Feisty - second time impressions

background: slackware command-line user, have used knoppix in past. Never used Gnome. I did use kubuntu a while back (I have used KDE ok) band I had a basic idea of what to expect.

Feisty very good at recognising hardware. In fact, more than good, it was brilliant. However, though the wireless network card was noticed, getting it configured was quite awkward (so say the least) in the GUI.

During the installation, all it asked to setup was the main user id and password. No mention at all of the superuser. I'm all for limiting access to superuser, but denying its entire existence is going too far.

On the top, to the right, there's an icon which looks like a terminal and clicking on this reveals that ubuntu has also done a scan of the wireless networks. This is all very promising, however this quickly turned to extreme disappointment as on inputting the WEP key (and seting the weptype to "open" - I admit to not knowing its meaning) a blue circular arrow started going arund the terminal icon mentioned previously, and it didn't stop. 

I decided to kill all this and I reckoned that it was my ignorance that had set the weptype to open, when really it should have been "shared" (well, this was all surmising on my part). However repeating the WEP key input process and setting its type to "shared" caused something even worse, the whole computer went into hang! Nothing worked. No errors, no keyboard, mouse not moving. This is a disaster. Ctrl+alt+del did not of course work, so I had to actually do the unmentionable: a hard switch-off.

Another thing that I have noticed is that other distributions on my hardware allow auto switch-off after logging out. Feisty hasn't worked it out unfortunately. It logs out and (presumably) kills all processes, but it cannot manage the power down.

In trying to setup some facilities I was finding that Gnome was asking me for superuser password all the time, so I went to the forums and foudn out that thenormal user password I had set up in the first place was the one I should use. Was I told that when I installed? I forget, but any way it turned out to be true and I go go about setting up WEP on my wireless card. Wanting to totally avoid the route I took last time, I went through the menu and found networks and my wireless card came up. I was able to input my WEP here and finally it came good. 

This whole experience has got me quite nervous about some of the more user-friendly GUI applications on Feisty. Just my 2 cents.

### Post by AlanR8 on 2007-05-12
Seems to be luck of the draw regarding wireless cards. I'm running Kubuntu Feisty on a Sony Vaio VGN-FS415M laptop. My experience in the past with Sony laptop's is they can be sensitive to hardware and a tad proprietary!

That said, Feisty went on a treat, it picked up everything, including the track pad. The wireless card was a breeze. I use WPA (?) not WEP and the manager loads every time I boot asking for the key. What I have noticed is that if I get the password wrong the easiest way out is to reboot otherwise you can be hanging around for a while!

Other than that, the Sony has a Nvidia card so Beryl works a treat and my sixteen year old son was so impressed he asked me to install Feisty on his Core Duo beast. Works very well and now I've converted all his music (was M4P) he's a happy bunny.

Also have it on an old desktop used as a "server" and again faultless install.

Maybe I've been lucky with my hardware.
