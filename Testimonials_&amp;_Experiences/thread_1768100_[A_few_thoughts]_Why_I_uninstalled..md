### Post by Saito on 2011-05-26
Every year for the last 5 years or so I take a month off from Windows and install a every day desktop distribution of Linux like Ubuntu, Mint and PCLinux and pretty much everything else under the sun.

So this year I installed Ubuntu again because I heard quite a few good things about the 10.04 version.

Sadly, as the topic indicates I'm switching back to Windows again.

But for what its worth I'll explain why I made this decision.

Ubuntu is made to be used by people that are not experts (it says so on the website). And for a part it does that very well.

The main problem with Ubuntu and every other Linux os that aims to be an everyday desktop OS like Windows has the same problem.

The gap between the easy and the hard is extremely big in terms of skill required but amazingly small in when its required.

A few examples:

Installing an application,

Situation one:
You start Ubuntu Software Center and you click what you want and press "Install" and wait. BRILLIANT! This is better then anything Windows has to offer.

Easiness: 9.5/10 (there is always room for improvement!)

Situation two:

You want to install an application that is not in the software center. You download the program archive and extract it and see files like "configure" "make" "install" and whole lot more. So you double click install and you get a text file full of strange stuff. Now if your an seasoned Linux user you might be laughing your *** of but your average user will not.

Easiness: 1/10


Using applications:

Situation one:
You install something from the Software Center and it appears in your Applications menu with a nice icon and you click it and the program starts.

Easiness: 9.5/10 (there is always room for improvement!)

Situation two:
You install something from the Software Center but (apparently) its an piece of software that has no GUI so no icon in your Applications menu. A new user will just think "its broken".

Easiness: 1/10

Configuring applications:

Situation one:
You install something from the software center and it appears in your Applications menu with either and configuration tool or the program has a good configuration menu build in.

Easiness: 9.5/10 (there is always room for improvement!)

Situation two:

You install something from the software center and it appears in your Applications menu with an icon. You start the program to find out it runs fine but you can't find any sort of configuration and its required to run the program. 

Easiness: 1/10

Hardware:

Situation 1:
You just installed Ubuntu and all your hardware is recognized and even your video drivers are installed automatically with the latest version. (again something that makes me love Linux).

Easiness: 9.5/10 (there is always room for improvement!)

Situation 2:
You start the music playback application that comes with the installation and load in your music and click play... no sound. 
So you open the settings menu of the program and click on the "output device" and get a huge list of devices (15+). It sees every one of your onboard audio outputs as a different device but with no sane description and to make things worse it also see's the 10+ hdmi devices from your video card.

Easiness: 1/10


Now I'll stop here. I already went way past the point of tl;dr.

I believe that the key to get more and keep more users on Linux is by decreasing the big and steep difference in difficulty of the every day tasks.

Ohh well... UNetBootin finished copying my Windows image so i'm signing off and will see you all again next year with the hope I'll never have to make a Windows USB stick again :)

Rock on! \m/ :guitar:

### Post by 3rdalbum on 2011-05-27
"I wanted to find out what Linux is like today, so I installed the third-last version".

Is really what it amounts to. Ubuntu 10.04 is more than a year old, which is quite old in Linux terms.

Ubuntu Software Center has been improved in 10.10 and 11.04, and you might find that your interesting sound problem has been fixed. Ubuntu Software Center doesn't, as a rule, show non-GUI programs anymore unless you specifically ask for "technical items".

Also, note that most things don't need to be compiled from source anymore. You can use repositories or PPAs to install programs that aren't in the official Ubuntu repositories. It's rapidly becoming the most common way of distributing programs.

I'll also point out that the solution to drivers is not to make driver installation easier. External drivers bring a whole bunch of problems themselves. The solution to drivers is for every piece of hardware to conform to a standard way of communicating with the computer, so you only need ONE driver for each class of device, and each standard driver comes with your operating system already.

Linux is very quick with adopting new standards for hardware support, so when the device manufacturers actually get together and work out the standards, Linux will implement those standard drivers before you know it.

### Post by lykwydchykyn on 2011-05-27
I think the problem is that, if you go looking for a piece of software the way you look for software in windows (i.e., google the software name, find the author's website, look for a download), you're going to end up with a source tarball 75% of the time.  That's just how open source software is distributed by authors -- nobody wants to waste time packaging it when the distros are all going to package it for you.

Not only that, but it wasn't so long ago when just about every review/writeup/how-to on a piece of Linux FOSS started out with a short set of instructions for compiling from source.  That's changed, but the internet is forever...

About all you can do is try to educate users on how to get software packages they need.  Basically:

 - Check the repositories
 - If it's not there, make sure universe and multiverse are enabled and check again
 - if it's not there, look for a PPA
 - if it's not there, look at sites like getdeb.net or playdeb.net
 - If it's not there, ASK ON THE FORUMS. 
 - Failing all that, if you just must have it, compile.  But at least use checkinstall so that it's registered with the package database.
