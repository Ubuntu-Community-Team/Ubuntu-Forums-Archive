### Post by mastablasta on 2012-03-17
Scribus is very good. i suggest using a PPA to get the latest version as it is improved a lot.

regarding puppy - there are other lightweight options out there that you can install to hard disk.
since i do not know all your components i will suggest only a few:
Slitaz - it's has a about 60 MB iso. well below 100 MB anyway. plenty applications available. and can work even on 16 MB ram. [http://www.slitaz.org/en/doc/releases/3.0/relnotes.en.html](http://www.slitaz.org/en/doc/releases/3.0/relnotes.en.html)
Lubuntu - Ubuntu with a light LXDE. will need 129MB ram and i think it takes up less than 2 GB hard disk. [http://lubuntu.net/](http://lubuntu.net/)
Bodhi Linux - uses Enlightenment desktop. uses ubunut under it and can run on old 350 Mhz CPU and 128MB ram, and needs a little over 1GB hard disk for install. [http://bodhilinux.com/](http://bodhilinux.com/)
Another relatively user friendly system is Chrunchbang Linux which is based on debian stable. very easy to install (compared to debian) as it uses backports and scripts for better hardware support, very stable and uses low ram. By default it uses openbox, but you could install LXDE or XFCE on it. [http://crunchbanglinux.org/](http://crunchbanglinux.org/)

another option is to try otu (but it is sitll in development stage) the RazorQT desktop environment.

### Post by varunendra on 2012-03-21
That is a graphics issue that often happens due to a resolution/frequency than the monitor can not handle. You may try temporarily booting with 'nomodeset' option as described here: [https://help.ubuntu.com/community/Grub2#Editing_the_GRUB_2_Menu_During_Boot](https://help.ubuntu.com/community/Grub2#Editing_the_GRUB_2_Menu_During_Boot)

Just add **nomodeset** after a space after the line that ends with 'quite splash'.

The nomodeset options lets you boot with a usable 'low graphics' mode. Once in usable graphics mode, you can make this change permanent, if required, by installing boot-repair as this guide suggests: [https://help.ubuntu.com/community/BootOptions#Changing_boot_options_Permanently_for_an_Existing_Installation](https://help.ubuntu.com/community/BootOptions#Changing_boot_options_Permanently_for_an_Existing_Installation)

However, you should first try to look in "Additional Drivers" (I'm not sure if it is the same term in lubuntu) to see if a suitable driver for your graphics is listed. If it is, installing it is all you need.
