### Post by simosx on 2010-08-16
The bugs section in Launchpad is mostly a staging section for issues that need reporting in an upstream project.

That is, if you report something at Launchpad, this does not mean that the upstream developer will necessarily take of it.
The same goes with other distributions and their own Bugzilla bug management services.

Compared to other bug management services in distributions, Launchpad offers some nice tools to link the Launchpad report with an upstream report, and follow automatically. If the upstream report is fixed, then the Launchpad report is auto-notified. 

Who deals with Launchpad bugs and help solve them? There are Canonical employees that help verify bug reports and try to identify an upstream bug (or they often create corresponding upstream reports and link to Launchpad).

In addition, there are these Bug Days where members go through bug reports and help clean them up. See [https://wiki.ubuntu.com/UbuntuBugDay/](https://wiki.ubuntu.com/UbuntuBugDay/)

If you are really interested in helping, you can join one of the Ubuntu Global Jam events where people like you and me are taught how to clean up bug reports.
You do not list where you are from, so find if there is a local Jam event near you from [http://www.jonobacon.org/2010/08/10/ubuntu-global-jam-we-need-your-events-2/](http://www.jonobacon.org/2010/08/10/ubuntu-global-jam-we-need-your-events-2/)

You use the word 'bloated'. In the open-source world, when a person says 'bloated' they mean they are unhappy about something but they do not know what's actually the problem. Seeing 'bloated' is interpreted in 'like the software is not used properly'.

With launchpad.net, the most common need for help is the management for the bugs rather than actual solving them.

For example, there are many bugs that are similar. So one task is to go through them and mark the same bugs are duplicates, letting the one with the best description stand out. This is a common and somewhat easy task.

Another task is to link a bug report with the relevant upstream project. There is a process in launchpad to perform this link, and after that the updates will come automatically. Once you have a report that is for an 'upstream' issue (such as a bug in GNOME), simply by perform the link to the upstream report is the way to solve the bug in launchpad.
