### Post by Xonara on 2008-10-18
As a free, open source OS, Ubuntu is a major accomplishment and I'd like to thank everyone who has taken part in it's development, from the programmers to the art designers to the documenters to everyone who helps out on the Support Forum, and of course the people who help fund the project. I can't say I've helped much myself but I can give a little feedback if it helps.

Ubuntu's *features* are numerous. When I say features I mean the different operations it can perform, regardless of how successfully it performs them or how easy it is to use the features. Ubuntu can run many different programs. Ubuntu can run all of the open source programs, of course, and can ran a good number of closed source windows apps. As far as I know running proprietary OSX apps are near impossible, but I couldn't care less. Nautilus does everything a browser should do. Really, you can do very nearly anything you could do on OSX/Windows. Despite Ubuntu and all the apps that are supported by it being a pretty complete heap of software the support for proprietary Windows apps is a much needed and much appreciated feature!

Ubuntu's *functionality*, or stability if you prefer, unfortunately isn't exactly impressive. Maybe it's just me, but I think all of the article's I've read about being able to run Ubuntu for months on end and never having to restart it aren't true at all. I've been bombarded with some some pretty frustrating problems. An example: I'm sharing an internet connection over a network but the connectivity is poor because the router is in a different building which is pretty far away. Naturally I frequently disconnect. On windows, I used the "repair" button so I don't have to wait for it to reconnect. On Ubuntu though, there's no "repair" button and it takes too long for it to realize I'm disconnected and reconnect. It wouldn't matter if there was a repair button though because once I've disconnected Ubuntu simply won't successfully connect again no matter how many times it tries unless I unplug the adapter, plug it back in, wait for Ubuntu to verify the network is there before it tries to connect and then I'm connected. Constantly doing this is pretty frustrating. And sometimes the the nm-applet doesn't recognize there's an adapter plugged in at all after I do that routine. Another example: Every now and then Ubuntu becomes rebellious and makes random apps crash. I can't think of any better explanation because I can go for a fairly long time without anything crashing and then firefox and the gnome bars and even solitaire will crash for no reason all within about 2-3 minutes tops. In fact, the apps Ubuntu decides it doesn't want me running will crash repeatedly within about 30 seconds every time I try to start them again until I reboot. I mean, what the hell?

Ubuntu's *usability* is great! I don't have any complaints :)


So my general opinion is, Ubuntu is a flexible and easy to use O/S, but stability seems to be a little neglected.
