### Post by K-Ghidorah on 2012-06-15
I'd like to open this by saying that this isn't a rant against Unity.  I actually really enjoyed using Unity in Ubuntu 11.10.

Like most here, I was chomping at the bit for the new LTS release of Ubuntu.  Well, actually Kubuntu (I prefer KDE), but the thought is the same.  Then it got released, and I was thrilled.... for a few days.  After several hard freezes in one day, I went to the main Ubuntu, hoping that it was bugs within the current version of KDE.  Same thing happened.  Went to Linux Mint 13 Cinnamon, which has a different default kernel.  Same thing happened.

I figured before anyone suggested changing desktop environments, I'd start off this rant by saying that I had changed THREE TIMES.  I even went as far as installing a newer kernel, namely 3.4, as per the instructions of quidsup (YouTube Linux reviewer and all-around classy guy).

NOTHING WORKED.

I even filed a bug here ([https://bugs.launchpad.net/ubuntu/+source/xserver-xorg-video-intel/+bug/993187](https://bugs.launchpad.net/ubuntu/+source/xserver-xorg-video-intel/+bug/993187)), and, noticing that many others are suffering from the same problem on a variety of computers with different hardware configurations, I thought that this would get Canonical's attention.

(For the curious, my computer is a Dell Latitude D620, Intel Core 2 processor T5600 1.83 Ghz, Mobile Intel 945GM Express Chipset video card, Sigmatel 9200 soundcard, and 3 GB RAM)

It did.  They won't fix it.

That was the last straw.

When an OS sets itself up as a speedier and more stable alternative to Windows, one should make sure that it is in fact SPEEDIER and MORE STABLE.  Ubuntu 12.04 hard freezes more often than Windows 98 SE!  Of course, I could change to a previous version of Ubuntu, but the point of the LTS was to have a stable release that I can keep in my computer for 5 years.

I liked Ubuntu because, out of all the distros, it worked the best while still being up to date, and it's a breeze to install.  Plus it's damned cool looking.  I've advocated the use of Ubuntu and its derivatives to my friends and family since I discovered it.  Until Canonical takes responsibility for this horrendous bug (or multiple bugs) and fixes it, I rescind my advocacy.  This sort of thing is unacceptable.

### Post by K-Ghidorah on 2012-06-21
First of all, I want to apologize to anyone that I might have offended in this thread.  I typed out of anger and frustration and, while my frustration could certainly be considered justifiable, I should've just let it go quietly.

At any rate, I may have stumbled onto the cure due to the fact that I really... REALLY....  wanted to use KXStudio again, and their low-latency kernel is only  available on 12.04 (or else I would've stuck with Linux Mint 12 KDE with the repos added).  Turns out the kernel is slightly older than the  original (3.2.0-23, as opposed to 3.2.0-24 or -25).  Again, I'm using  the low-latency kernel, so I don't know if this makes any difference (I  know more about popcorn kernels than Linux kernels).

If anyone else is having the problems I was facing, I recommend getting  the 3.2.0-23 generic kernel (or the low latency kernel if you're also a  musician).  I've not had one freeze so far and I've been running this laptop all night.  Also, as far as I know,  very few people have had the Live environment hard freeze on them, so the  3.2.0-24 generic kernel might also be a good one.

Be well, folks.
