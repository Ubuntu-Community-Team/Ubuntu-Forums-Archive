### Post by change_mode on 2010-07-20
Before leaving Ubuntu after more than 2 years, I want to write down the reasons for this decision, starting with the less important reasons and going on to the crucial ones, in the hope that maybe it'll help to improve Ubuntu.


**Bad design decisions**

Making pre-beta software like Empathy the default in LTS releases is bad. I don't want to start an argument about which messenger is the best, but let's just say I experienced and reported about ten bugs myself in only 3 months and there's a million other bugs in Empathy and the related packages. Its functionality doesn't come close to that of other IM clients and for some protocols it even lacks basic functionality, like adding contacts. 
Yes, I know the reason for the decision behind this, but the users are not beta testers, or at least they shouldn't be.

Moving the window controls to the left side. Why one would force such a change when only 10% of the community support it is beyond me. Not that it's hard to get used to, but it serves no purpose whatsoever. It doesn't make Ubuntu unique, it doesn't improve the functionality. All it does is adding an extra step to the installation, to put the controls back where they belong. I guess someone likes Macs and wants Ubuntu to look and feel similar. Whatever.

Social networking desktop integration. Just another thing I have to remove after the installation. If at least it wasn't integrated in the systray...

All of these things are easy to change, but I started using Ubuntu, because I wanted a usable system with good defaults that save me the configuration time. For my taste, Ubuntu clearly moves away from that. There's too much unnecessary and broken stuff included by default.

That being said, the themes and icon themes have been improving lately and except for the misplaced window controls, Lucid looks pretty slick out of the box.


**Bugs and response to bugs**

The amount of bugs and the time it takes to fix them is ridiculous. I realize that this is not always Ubuntu's fault, but more often than not, it is.

For the most time I used Ubuntu, I had essential programs that were unusable due to bugs. In one release it was Rhythmbox, in the other Pidgin and a few others. These bugs could all have been fixed by upgrading the respective packages to a newer version, but that's against the Ubuntu release policies that state packages/fixes will only be backported/upgraded/fixed when there are security vulnerabilities. 
That means the system will be partly unusable until the next Ubuntu release, which hopefully has a newer version of that package.
There are only very few exceptions to this and that's bad, because no part of the default system should ever be in an unusable state, no matter what.

Of course, I could have (and have, in some cases) manually upgraded the programs or switched to others, but the user really shouldn't have to do that in order to get things working.

Ironically, even the bug tracker, Launchpad, seems to be down half of the time. It's timing out yet again as I write this post.


**Stability**

The most important point. During the last 2 years, there was only one release that was perfectly stable for me (Karmic). The ones before that had occasional PulseAudio crashes which made the GUI completely unresponsive, now in Lucid the mouse and keyboard regularly lock up. It's 2010 and that simply shouldn't happen.

It wouldn't even be too much of a problem if the bugs were fixed swiftly, but see *Bugs and response to bugs*. For approximately 21 out of the 27 months I used Ubuntu, it would occasionally crash.



I really hope some of these things can be improved in the future. Good luck.

### Post by ghostborg on 2010-07-20
I usually find the culprits are the software packages and not the base system. For example on my system Firefox seems to occasionally turn dark grey and lock down other apps until it decides to return control to me. I use Chromium without those problems.
A Compiz setting can sometimes be a source of problems again not Ubuntu.

Some hardware is better supported and some people run great while others have nothing but headaches. Most problems I usually find the solution to. Sometimes failing hardware can pose itself as a software problem.

I have a few systems in the house and I could not believe the differences even with Windows7. Some machines frequently Blue Screened while others ran fine. Switched the machines that BSD to Ubuntu and they run great. 

I have tried other distro's in the past and always find my way back to Ubuntu. I feel more comfortable with it and the community is so large you can usually find the answer to your problem.
For me, Ubuntu 10.04 has been a pleasant experience.
