### Post by hakko on 2008-07-03
With a new Quad Core 8GB ram machine, I wanted a 64bit OS as a base and XP and Vista as virtual machines. Initially I wanted XP 64bit as a base, but I had an inclination for Linux:
* I like the command line (but I do prefer GUI for other things).
* I like the stability when it's running lots of applications and doesn't slow to a crawl.
* I like the workspaces idea.
* I like the general "look" of Linux.

I started with OpenSUSE 11, backed by Novell and assumed Groupwise would therefore work.  Not exactly. Eventually got it working and found the filter field is hidden away. I like using RPMs and one-click installs which SUSE supports.  However there was a general flakiness, panels would go blank, I couldn't click on apps or switch apps.  :mad: It was like using Windows 95/98 again LOL

Try Ubuntu... After installing Ubuntu, I found the resolution was an issue, especially connecting via KVM switch.  OpenSUSE allowed me to specify a resolution and it wouldn't change when I switched between fullscreen on the virtual machines.  After getting frustrated trying to install a program from the web (I mean why can't it be a simple download and click install?), I went back to OpenSUSE.  The flakiness returned, with panels going blank and so on.  

I think I had enough of Linux :mad::lolflag: So I was about to install XP 64bit but then Debian came to mind.  I used it at Uni, where all the lab PCs were dual boot Windows/Debian.  But then, Ubuntu is based on Debian so I went back to Ubuntu and installed that as I base OS.  I'm running two monitors, with one connected to the Ubuntu PC so I get the correct screen resolution and don't lose it. I've also switched from VMWare server to VMWare Player, the player gives me fullscreen within the current "desktop", and not restart X and so on. Overall Ubuntu is less flaky for me than OpenSUSE.

Thing is, I've got time at work to fiddle with these things.  But if I was time pressed I would have went with XP 64bit.  Linux is an amatuer OS after all and I think it's great that the OS can autodetect so much hardware, but unless you have time to deal with the flakyiness, then you are better off with Windows.  With Linux I feel you have to choose your hardware around it and "lock it down".  I think Linux can work in schools and offices, but you have to be strict with hardware and do a lot of pilot testing.  You can say that the time/money spent there is made up for from the money saved on XP licences :guitar:

Using Firefox on Ubuntu is really fast, it's like having a faster net connection compared to using Firefox on my Windows PC.  :lolflag:

I would like to see the following in Ubuntu/Linux:
* Better screen refresh/resolution options and gui interface.
* True one click install of software, I don't want to do packages, run command line installer, download kernel source, research error messages blah blah blah.  I really wanted to install VLC and VMWare just by downloading it off the web and double-clicking on it a la Windows and getting on with my life LOL.
* Windows driver conversion.  There must be a way to convert Windows drivers to Linux... 
* Merge distros, there's too many!
* OpenSUSE gave me a choice, KDE vs GNOME - you do know that too much choice can drive people away? I went for the stable one, which I was told is Gnome.

Anyway I feel like making my own Linux distro now, doh! :lolflag:
