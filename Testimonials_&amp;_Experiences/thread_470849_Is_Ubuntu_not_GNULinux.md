### Post by francisco_athens on 2007-06-11
I've been an Ubuntu user for a number of years.  It was the first GNU/Linux distribution that I was able to install on my computers.  It helped opened my eyes to the freedom offered by free software, but recently I've been troubled by something as I learn more about the origins of GNU/Linux.

Ubuntu has become a dominant force for introducing people from all over the world to GNU/Linux.  Ubuntu strives to be a humanitarian effort.  I think that with this comes responsibility.  Ubuntu is not just the Linux kernel.  It contains many essential programs from the GNU project and many more still that are developed under the GNU GPL license.   This license can be seen as an enforcible part of the the ideals that Ubuntu seems to represent.

It is then, with some concern that I find the opening statement on the Ubuntu.com homepage to read:
> Ubuntu is a community developed, linux-based operating system that is perfect for laptops, desktops and servers. It contains all the applications you need - a web browser, presentation, document and spreadsheet software, instant messaging and much more.

If Ubuntu is a "community developed" OS as it is stated, then should we not give some credit to the community that helps create it and acknowledge the reasons why GNU/Linux was developed?  That is to to create a Free Operating System that is licensed with the [four freedoms]("http://www.gnu.org/philosophy/free-sw.html") that the [GNU GPL]("http://www.gnu.org/licenses/gpl.html") provides.

Therefore should not Ubuntu be referred to as "a distribution of GNU/Linux", not just as Linux or simply as a "linux-based operating system?" Otherwise is Ubuntu not misrepresenting itself and denying the community efforts on which it is based and derives its success? It is true that Linux is a very important part of Ubuntu, but lets not forget that the Linux kernel too is released under the GNU GPL and that license has been part of its success because it allowed others the freedom to examine, modify and redistribute their changes and ensure that still others enjoyed those same freedoms so that the kernel could and will always be free to mature.

Thank you for your time and comments,

Francisco

### Post by francisco_athens on 2007-06-11
> **weblordpepe said:**
> Aaaaaaaaaaaaaaaanyway. Nah I dunno. Linux is GNU anyway am I right? Is there a scenario where Linux isn't GNU? I guess when the question about how much Linux costs comes up...thats when you could mention GNU and the licences...

To address the issue of "cost" and freedom in GPL licensed software, the freedoms you are guaranteed go far beyond the monetary. The issues of [knowing whether the software you are using is spying on you and the ability to make changes and share those changes freely]("http://www.gnu.org/philosophy/free-sw.html") can have greater implications.

Linux was not originally licensed under the GNU GPL.  The development of the [ GNU tools in the mid eighties]("http://www.gnu.org/gnu/gnu-history.html"), by Richard Stallman and friends, actually predates Linux by a number of years.  In the early nineties Linus Torvalds released an early unix-like kernel to the public with a license which prohibited commercial redistribution, but then changed to the GPLv2 [(Wikipedia,2007)]("http://en.wikipedia.org/wiki/Linux#History") and the GNU tools were added to it to create an functional operating system.  The GNU tools such as Emacs have been ported for use with other kernels like the ones from BSD and Solaris as well as the Amiga kernel (where I first learned of things GNU).  

A situation where "linux isn't GNU" can occur with a device like the TIVO.  It uses the linux kernel and GNU tools and they release the source-code.  However, if you modify that code in anyway and attempt to put it back into your TIVO, you will discover that it will not work. It has been called ["Tivoization"]("http://en.wikipedia.org/wiki/Tivoization") The TIVO checks to make sure that it is not running modified software.  While not technically in violation of the GPLv2, it seems to circumvent the spirit of the GPL by limiting your freedoms. 


To stray off-topic a bit. There is actually a GNU microkernel in development called [the Hurd](http://www.gnu.org/software/hurd/hurd.html), which would make for a GNU/GNU operating system ;)  
There are developing projects for OpenSolaris like [Nexenta]("http://www.gnusolaris.org/gswiki") to create a Debian GNU/OpenSolaris OS.
