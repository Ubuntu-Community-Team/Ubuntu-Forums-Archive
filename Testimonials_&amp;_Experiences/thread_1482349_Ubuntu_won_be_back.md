### Post by Elvish Legion on 2010-05-13
After using linux for a while I switched back to Windows due to some work software, and I'll admit I got complicit with Windows, things just worked and that was fine by me.  After about a year or so of not using Linux, I got a new job and thus no longer had a need for Windows.  I went right back to Arch where I had left from, I found my self kind of frustrated I couldn't even figure out how to install some stuff from their ABS, I read a little couldn't figure it out so I go discouraged.  

Went back to Debian testing an old favorite of mine.  With the (new to me) laptop I have having an nvidia card I tried installing it the Debian way...only to find that the kernel I was using didn't seem to have a Debian Driver. Did a little reading couldn't for the life of me figure the driver out (Going to try again soon though) Went and messed with the wireless and couldn't connect to my network, which ment being tethered to a wall or dragging 300' of cat5 behind me no thanks. Not to mention I was unable to find much help in documentation for Debian a lot of 3rd party sites hosted random howtos on this or that but nothing like Arch or Ubunut offered.

Dreading it I went back to Ubuntu.  Last time I had really used it it was kind new (5 or 6 I think) and it was pretty fat back then.  Boy was I impressed when I installed it, Nvidia driver and Broadcom wireless both right now, boot time was insanely fast, the few games I play work wonderfully, Everything on my machine works except the broadcom USH (universal security hub, smartcard/fingerprint reader) but I have a usb card reader and don't need the fingerprint reader so thats no issue.

Keep up the good work Ubuntu you have grown leaps and bounds!

### Post by MarcusW on 2010-05-15
Installing it manually isn't really THAT bad. :) 

sudo service gdm stop (assuming gdm is running)
wget [http://uk.download.nvidia.com/XFree86/Linux-x86_64/195.36.24/NVIDIA-Linux-x86_64-195.36.24-pkg2.run](http://uk.download.nvidia.com/XFree86/Linux-x86_64/195.36.24/NVIDIA-Linux-x86_64-195.36.24-pkg2.run)
sudo sh ./NVIDIA-Linux-x86_64-195.36.24-pkg2.run
sudo service gdm start

If on 32-bit, change pkg2 to pkg1.

### Post by cascade9 on 2010-05-15
I dont even know if that set of commands would work (I know sudo doesnt, unless you have configured it to work). 
If it was going into the debian wiki, it should be more like- 

su
apt-get install build-essential linux-header-$(uname -r)
cd /usr/src
In -s linux-headers-$(uname -r) linux
wgethttp://us.download.nvidia.com/XFree86/195.36.24-pkg1.run
/etc/init.d/gdm stop
cd /usr/src && sh NVIDIA-Linux-x86-95.36.24-pkg1.run

That should start the nvidia installer, when its finished 

startx 
or
/etc/init.d/gdm start

I'm 99% sure that is how I did things last time I installed nvidia drivers on debian lenny.
