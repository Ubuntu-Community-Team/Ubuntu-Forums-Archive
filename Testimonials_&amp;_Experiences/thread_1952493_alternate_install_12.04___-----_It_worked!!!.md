### Post by cloyd on 2012-04-04
Just gotta share this. I have a netbook that I thought was going to be useless. It worked with 9.04, and 9.10. Worked with 10.04, but with some problems.  Got it to working well with a little tweaking. Could never successfully boot a live usb that used unity. The problem seemed to be an old ati graphics card. I was running 10.04 and having no problems. A while back, I was playing with live usb"s, and something bad happened. The system would no longer boot. Since we were quickly moving toward my dist being no longer supported, I tried to install something newer, but with various less demanding desktops . . .  lubuntu, xubuntu and some others. The only thing that ran without crashing video was puppy linux, and I just wasn't happy with this.  Talked to my son, and he convinced me that I had a bad graphics card,m so I just gave up. 

(I bought a tablet to take care of my computing needs while traveling.  Love it for internet and reading, but it lacks somethings. )

The other day, I went back to the old machine, and reinstalled 9.04. It worked, but I couldn't uses software center or synaptic on it. Next, I tried 10.04 again. It worked. Nothing wrong with my card, just the drivers in newer kernels. 

Ok. I really want this machine to run an up to date LTS system. 12.04 is LTS, but beta . . .  but . . . 

One problem I had with the newer 'buntus ()on this machine) was graphics crash before the live usb could even boot. How do you install with crashed display?  Then I thought, why not alternate install iso?  I thought maybe I could boot in 10.04, and modify some config files from there and get 12.04 to work.

I did the alternate install, and got it installed  on another partition of the machine (still had 10.04 on the other partition).  Would not boot; the display would crash before I could even put the password in. 

I booted to recovery mode. Didn't know what to do there, so I resumed a normal boot. Lo and behold, it booted into a workable unity desktop.   I used it for a while, booting through recovery mode and resuming, and it always worked. At this point, I could not do a normal boot . . . it would crash.  

Then I got some updates. By accident, I did a normal boot, and it worked. It has been working ever since. I really don't know what happened, but 12.04 beta2 is working for me.  It would never have happened had I not tried the alternate install. The alternate install was really easier than GUI install. It would not have been easier if it had been my first install in Ubuntu or Linux. But once I had used Linux long enough to know what partition is what, the alternate is not difficult.

Nothing sensational here, I'm glad to know I'm going to be able to use this machine with 12.04.  I really do like unity. I have not modified any config files.   Some developers must have made a few changes that are good for my old machine. Thanks.   

If you have a ATI raedon x1200 series card, there is still hope.
