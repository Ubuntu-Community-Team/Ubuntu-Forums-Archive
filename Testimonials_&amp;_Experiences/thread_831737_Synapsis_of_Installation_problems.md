### Post by JeffSherlock on 2008-06-17
Synapsis of Installation problems

In running a Google search, I've found that all versions of linux - Redhat, etc...etc.. have the "EID BAD" problem which ubuntu varieties have. These postings go back to 2005, and no one ever fixes the screwed-up code within the Linux keral that causes the problem.

With several types of Linux - ububtu varieties included - I have seen that once Linux is installed on a hard drive, people find it very difficult to write over it for a clean install. I have no idea what causes this other than a complete lack of a simple formatting command.

Specifically regarding installation of ubuntu or xubuntu - latest three releases of each - the installation screen's first line is to choose to run or install. It then seems to be confused as to whether the user wants to run the live CD or install the OS. There are not seperate lines to choose to do one or the other.

For me, it is difficult when seeing a black page of code ending with "Kernal panic" to determine if this is the installation on the hard drive or an attempt at running live.

When forum contributors offer advice such as "remove your graphics card" - which would make it impossible to do anything, or "Type in the following commands" - when the user's complaint is that the keyboard is not working, are incredibly stupid. It gives me the impression that many forum members do not actually read the questions posted, but unstead simply brain-fart out these silly responses.

For me, every download I have made of ubuntu Desktop, ubuntu Alternate, and xubuntu, I ran an md5 check. Every copy said "Bad checksum" - the ones which would not install and the ones which installed just fine. Not great advice to always tell a user to do an md5 check as the Holy Grail answer to every problem.

FYI: I have succeeded and failed to install on old computers with only 128MB of slow ram and very high level computer with state of the art motherboards, processors, and 4GB of fast ram. It seems to make no difference, it will either instasll or it will not. Might be clever to stop blaming hardware for the endless list of problems in the Linux kernal, and the lack of any attempt by the programmers to try creating more sensible ways of writing the installation isos.

Would be nice if ubuntu/xubuntu allowed the user to control the software setups rather than requiring a world-class programmer to reverse-engineer and write new codes, especially when it comes to networking and hardware incombatablilities. With both ubuntu and xubuntu, I feel like I'm flying blind with no provisions to control much of it.

I do understand it is not just Linux, but all OSs. From the earliest computers forward, all OSs have been created on the basis that the OS must have a personal relationship with all the hardware rather than just running on the assumption that the hardware is correct (which it is). PSUs and cooling fans are the only compnents which do not have to have conversations with the software in order to be allowed to run.

Anyways, had to use XP Pro to write this posting. It always installs perfectly and never needs code editing to find and communicate with LAN hardware. But it does cost $150 US, and so many people - especially those who require using multiple computers would like to use free - as in Free Beer - Linux, but the endless problems push them to use Windows.
