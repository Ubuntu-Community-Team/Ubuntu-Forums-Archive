### Post by 3rdalbum on 2012-05-25
Just a note: You could have removed the global menu - either by switching to Locally Integrated Menus (it's a dconf-editor setting, but you might not like LIM) or by removing the global menubar package to get the menus back to the Windows way. Note that removing the global menubar package also disables HUD.

There's nothing in XFCE, or indeed in any operating system, like HUD.

Also there's a classic applications menu indicator for Unity, for the people who really can't get to grips with the way Unity arranges applications.

And finally, you didn't have to "remove Ubuntu" and "install XFCE" as though they were different operating systems. Simply installing XFCE on top of regular Ubuntu will give you Xubuntu.

I'm glad you've found what you wanted without going to an entirely different distribution, but you may have still gone to more trouble than you needed to. You know for next time :-)

### Post by 3rdalbum on 2012-05-26
> **Jose Catre-Vandis said:**
> Not trying to bash Ubuntu, it looks very pretty but...

I tried the HUD, maybe I wasn't using it right, but anything I typed into it just came up with nothing? Does it only look for files in ~/ ? I was expecting to be able to type "terminal" and to be offered a terminal icon to click on.

You've misunderstood something here. If you tap the Windows key, you can type something like "terminal" or the name of a file you've recently used, and then it will appear. This is the Dash; it's the same thing as if you clicked the Ubuntu button.

If you tap the ALT key, however, you bring up the HUD. If you start typing "bookmarks" you'll see menu items related to bookmarks (if you're inside a web browser).

HUD is new to Unity, but Dash has been there since the very beginning.

> All very confusing  - click or type - if Ubuntu as I understood it was heading towards a touch screen interface, the HUD leads itself to the keyboard.

You misunderstood. Ubuntu is moving to an interface that is:

a. Efficient in use of space
b. Modern and helpful
c. Equally usable on touchscreen devices as on desktop and laptop - keyword being "equally".

The HUD will surely be usable on tablets, when there are Ubuntu tablets. There will probably be a hardware button that brings up the Dash and another that brings up HUD. In both instances, an onscreen keyboard will appear. Ubuntu tablets might even have a small flip-out or dockable keyboard anyway.
