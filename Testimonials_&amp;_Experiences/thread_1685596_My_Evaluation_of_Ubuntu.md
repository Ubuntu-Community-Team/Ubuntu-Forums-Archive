### Post by Rasputin69 on 2011-02-10
Benefits

Free software-comes with a fully functional office suite, email-calendar, contact manager, photo editor etc.

Loads and executes fast.

Issues

I recently installed this product because I wanted to learn to use Linux and had read many suggestions by forum users regarding the security of and Linux and it's stability.

The product has been installed side by side with Windows XP Pro.

My first efforts to install it failed because the CD image software that they recommend would not successfully burn a CD. I used the USB installer and successfully got the program installed.

After the Update Manager ran the system froze and I had to reinstall it. I was told that this occurred because I had insufficient memory to accommodate the system with all the updates.

I turned the update manager off and the system appears to be stable since then.

My next challenge was to try and install my Brother MFC5490CN all/in/one which is mounted on a LAN.

The software manager was unable to locate drivers so after receiving recommendations from forum members I was able to go to Brother\support\drivers\Linux\Debian and down load the two drivers and a set of instructions.

I got the drivers installed and then had to add rules for the drivers to the libsane.rules.

After doing the above I was able to get the printer running.

I did the same for the scanner.

The system shows the scanner as installed but when I tried to use the scanner the scanner software pops an error saying the scanner is either not attached to the computer or is not turned on.

I have been on the Ubuntu help forms looking for a fix for this issue and so far I have found none.

I currently do not have access to the scanner.

I found that if you try to use sleep from the key board hitting the sleep button nothing happens.
If you try to put the system to sleep or hibernate from the shout down menu it freezes. I found that this issue is will known and widely discussed in the Ubuntu Help forums but so far none has come forth with a resolution. Some posters attribute this to an assertion that the computer manufacture does not use standardized hardware protocols. I don't know if this is true or not.

This sounds like the old merry go round that you get from programmers vs computers.
The Computer Mfg always say it's a program issue. The programmer always says there is something wrong with your computer. Mean while the user is left in the lurch.

The next problem I had was in Windows when I tried to back up with a software that I have been using about two years. It reported that HO Drive C was not available and I need to insert a disc.

Went through the set up wizard and the software backed up to an external drive.

The final issue is that I have a legacy data base in Quicken of about 9 years and you can't run Quicken in Linux w/o the extension WINE.

I have received reviews from people who said that they tried that and that Quicken never ran correctly.

At the present time I have down loaded and installed Chrome and several extensions.
Chrome in Ubuntu is stable and very fast.

I have been using OO.O Writer for word processing and as a spell checker for my forum post.

I have been using OO.O in Windows for more than a year so I am familiar with the product.

I finally got simple scan to quit reporting that there is no scanner attached. However, ehrn I launch Simple Scan it has a group of small circles that it displays and then it hangs,

Recommendations

My opinion is that Ubuntu Linux is acceptable for armatures who are geeky and don't mind spending time chasing down fixes for hardware issues.

My hourly rate for the services that I provided before my retirement was $ 85.00 per hour so if I had attempted to install this free software on my office computer I would have lost over a $ 1,000.00 and would have been fired.

I also would not recommend it as a business desk top because it's inability to run Windows software is a fatal flaw in my view.

That point is so important it merits printing and hanging on the wall.

Earlier this week I went to do a clean install on my Dell Vostro, having just put in a spiffy new hard drive.  This Vostro was originally shipped with Vista, and while I bought it second-hand and don't have the original restore CD I do have a licensed copy of Vista here - should be no problem, right? That day I learned what you described:  the original restore disk computer manufacturers put together for the machines they sell apparently take a LOT of time to assemble, time you'll have to put in if you try to use an off-the-shelf copy of Windows.

These things are so easily taken for granted until you find yourself needing to do it.  And when you do, if your experience is like mine you'll find Ubuntu may do a better job out of the box than a non-customized Windows does.

Everything works in VirtualBox - it is nothing like Wine. I keep hearing how great Wine is but my personal experience is that I have never got anything to work properly in Wine. I am sure it is my fault but in terms of ease of use it requires a leash and regular walks the park.

VirtualBox is actually a PC simulator. You install Windows into it as if it were a PC. Windows knows no difference and everything works, in my experience. VirtualBox is a proper software application written by Sun and now Oracle.

In your case, if your main platform is Windows and you want to learn ubuntu for the fun of it, why not install VirtualBox on Windows and then install Ubuntu as a VirtualBox guest? See: [http://www.virtualbox.org/](http://www.virtualbox.org/)

The easiest way to get Ubuntu is to boot it off a live CD or USB stick. You can just play with it that way.

The second easiest is to install it on its own hard disk, separate from your Windows disk. Then you can boot either Windows or Ubuntu separately. I would not advise installing both on the same disk because this can create bootloader complications.

The next easiest is to use VirtualBox. I use this. It is a little fiddly to set up but the documentation is excellent. Once you get the hang of it you can then install lots of different OS really easily. Each OS exists as a simple file in your host filesystem and it is really easy to delete them. You could install OpenSuSE linux and fedora and another version of Windows if you wanted. Some people use this to try out the latest versions of Ubuntu before they upgrade. Also, you can create any number of virtual hard disks and then try out RAID. All this without any danger to your host OS or your real disks.

Whatever you try, just be patient with it. Linux can seem impenetrable at times but my experience is that there is always a solution and it is usually simple, just need to look for it.
