### Post by dakke on 2007-09-08
HI,

Just a little post about the os of mac vs Linux/Kubuntu. I'm an end user on mac, do not have any experience with command line nor will I ever do. Why not? I don't have the time to do so. But, I really like the open source community, its attitude and the skillful development. Yes indeed, I'm one of those who know nothing about programming, are amased by the soft being developed and use it as much as possible. Thank you all for that.

A few weeks ago I decided to migrate to Kubuntu on an old ibook G3, 700 MHz. It was running mac osX 10.3, it really isn't worth an upgrade to 10.4 (let alone 10.5 later on) so it became an old machine with an OS that's outdated. So Kubuntu seemed a good alternative. Before I start sharing my experience, let me point out where I use it for:

 I'm an academic so,
- Office packages: something like microsoft word or openoffice
- reference managing: endnote or bibus
- webdesign: taco, amaya, simple CSS
- PHP/MySQL based soft development (I'm in a neuroinformatics team)

Other stuff:
- (s)ftp: cyberduck
- handheld: palm os
- ipod: songbird/itunes/senuti
- internet: firefox, adium

I checked and for most of these I was able to find a simular soft or at least one that came close. SO Kubuntu seemed a good alternative, yet it turned out not to be (at least for me). Why did I migrate back to mac OSX (or why didn't I migrate my other more recent computer to kubuntu)?

Reason is rather simple: mac OSX is a by far more user friendly.

**Installing the Kubuntu:**
I was on a mac 12'', tried to install ubuntu 7.04, but my screen was 'too small' being a 12''. I browsed the web for some solution to it, but didn't find a decent answer. Meaning: you had to go into command line and adjust it manually. But I'm really not good at it, so after trying and stressing on it for some hours, I just gave up and closed all the different taps in firefox who were talking about the issue.
So I tried another version, but on the ubuntu website there wasn't a good ref to the powerpc version of ubuntu. So I, again, browsed some and downloaded a kubuntu 6.06 for the powerpc. I took the text based version (alternate) and installed that one. All worked well, no problems with the basic installation.

**Hardware support:**
First issue I had was the airport extreme that didn't work wll. So, again, browse the net (day 3 by that time) and tried to check wether there was some solution to it. I ended up confused, since there was and there wasn't. Different sources convinced me to forget about it, others claimed the opposite.
I know this is due to the propriety hardware, and I do know that this is a lame attitude of the company and I do not like their selective protection. But, these comments did not help me out at all.
Apart from that, all worked well, and for that I'm impressed that people are able to write a os from scratch.

**Software:**
OpenOffice is preinstalled with the kubuntu 6.06, so that's cool. I tried to check wether there was some kind of reference soft I could install (in academics endnote is the standard, but I also use refbase for our neuroinformatics tool). Bibus seemded to be the best alternative. So I checked the website, but I did not find a precompiled package for that. So again, I had to go into some command line terminal and try to install it that way. The guide mentioned something like 'you need to have A installed and B and probably also C' and well that's not the way I kinda like it.
Yes, I'm an end user, yes I might be spoiled, but mac has a very cool way of installing soft: drag and drop. With mac it goes: download a soft, unpack it (double click), and drag it to the place you want it to, and that's it. How simple can it get? And that's exactly what I like about mac, user friendly way of installation/maintenance. Even most open source goes like this, so again, very cool.
That's not the case with ubuntu, so I wonder: why not?

**Updating the OS**
Again, on a mac you click: check for updates and it give you a list of all the updates for the soft that is standard part of the mac os. Click install and again: done.
On linux I found it harder to do so. I tried it, but it gave me this message: insert dvd with... and by that time I was fed up with it all that I simply skipped the whole thing and went back to my more recent mac.
The major advantage of ubuntu is the free/well supported updates. The downpart is that I do not know how and that I (after spending way too much hours on google) do not feel like browsing anylonger for each tiny problem. 

**Information about Ubuntu is spread all over the place**
And that's my final comment: why do I need to go through a massive amount of forums to get my problem solved? 
Why is it that hard to find a package for powerpc through the main webpage of ubuntu (for those who would point me out to the right dir, I already downloaded a version, it took me some time to finally find one).
How come that installing some more specialised soft is so hard? Take bibus, why can't it be a drag and drop thing?
...


So after giving kubuntu a go, I would consider it being:
- a good alternative for *real end users* who only need to use a microsoft word-like soft, some browsing and a little bit of instant messaging.
- everybody willing to do more, needs to know/learn some of the *basic command line* stuff
- a good and decent operating system, yet the style is *not as 'clean' as the mac osx*, as far as I know there is no real out of the box mac like style.
- the *learning curve is simply too high for end users* who do not have the time to learn some of the more specialised stuff. I can't start learning command line stuff, because that would mean that I do not have time to spend time on my 'real' job. So we end up having: IT guys, end users and in betweens like me. 
Take eg a webdesign: most of the PHP coders (or whatever they use) can code and do a basic design. Yet the websites become well less attractive if there's no graphic designer doing his thing (ps: I'm not  graphic designer). And there's no way that a graphic designer can do both: coding and desgning since coding takes some time to learn and to keep up to date with the coding part. Same goes for a designer: he/she must keep up with those evolutions and needs to become better at his/her job so there's got to be some artistic evolution. So you end up with a coder able to code (and often claim to have some artistic skills) and do a basic design. And a graphic designer with some basic coding skills (and also often claim to be able to code, hehe).
What about this little example? It indicates (for me) that an os need to direct itself to end users with high demandings. Otherwise they'll never migrate. It's just not worth it: they can not migrate since it would consume too much time learning those things. I would very much like to migrate, but it's just not yet as good as the other commercial alternative is. The commercial OSX comes out of the box, works and is very robust with other soft integration. The only downpart is its price.

To me, those are some of the reasons why it's not the standard os already. I really like the os, being free/stable/fast (my old ibook runs really smooth with it)... but.

However, this is simply my opinion/experience. Feel free to comment  or to reject my claims.
