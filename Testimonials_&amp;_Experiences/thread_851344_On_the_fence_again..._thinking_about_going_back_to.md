### Post by moore.bryan on 2008-07-07
> **Roasted said:**
> No, I haven't compiled the latest alsa from source. Got a guide for that?
...
EDIT - I'm going to try a fresh format tomorrow.
not really a guide, just a few steps:
download the latest alsa stuff:

[list]
[*][alsa-driver-1.0.17rc3]("ftp://ftp.alsa-project.org/pub/driver/alsa-driver-1.0.17rc3.tar.bz2")
[*][alsa-lib-1.0.17rc2]("ftp://ftp.alsa-project.org/pub/lib/alsa-lib-1.0.17rc2.tar.bz2")
[*][alsa-utils-1.0.17rc2]("ftp://ftp.alsa-project.org/pub/utils/alsa-utils-1.0.17rc2.tar.bz2")
[*][alsa-tools-1.0.17rc1]("ftp://ftp.alsa-project.org/pub/tools/alsa-tools-1.0.17rc1.tar.bz2")
[*][alsa-firmware-1.0.17rc2]("ftp://ftp.alsa-project.org/pub/firmware/alsa-firmware-1.0.17rc2.tar.bz2")
[*][alsa-plugins-1.0.17rc2]("ftp://ftp.alsa-project.org/pub/plugins/alsa-plugins-1.0.17rc2.tar.bz2")
[*][alsa-oss-1.0.15]("ftp://ftp.alsa-project.org/pub/oss-lib/alsa-oss-1.0.15.tar.bz2")
[*][pyalsa-1.0.17rc1]("ftp://ftp.alsa-project.org/pub/pyalsa/pyalsa-1.0.17rc1.tar.bz2")
[/list]

then untar/compile each package (e.g. "tar xvjf alsa-driver-1.0.17rc3.tar.bz2" will untar the alsa-driver package, there should be a folder now named something like "alsa-driver-1.0.17rc3." cd into that directory and type "./configure && make && sudo make install").
after you're all finished, things might be better!
