### Post by 3rdalbum on 2007-06-08
The typical Linux desktop, even running all the programs one would ever run simultaneously, will not use up even 2 gigabytes of RAM. 64-bit computing is only necessary for addressing more than 4 gigabytes of RAM.

Until Linux programs become overly bloated as to require such a large amount of memory, then 32 bits is fine.

In supercomputing applications and servers, then yes 64-bit is important. But in these situations, there will be a native 64-bit version of the program (or the program can compile on a 64-bit arch).
