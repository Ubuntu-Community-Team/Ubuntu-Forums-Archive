### Post by Boelcke on 2007-09-05
OK, before I get to the point, I'll be up front with my OSes.  I had completely switched over to ubuntu, running it on 2 machines.  Now, I've gone back to running one of them with XP.  Why?  Because it requires much less maintenance, and works with audio/video/digital cameras better.  I've found a lot of good things about ubuntu, and learned a lot of useful things too.  So, this isn't a rant about why it's bad or good.  It's a (hopefully) constructive point about why this user has found ubuntu to be a work-intensive desktop alternative.

In order to make a nice, stable desktop that wouldn't change much (read: my wife would accept), I decided to stick with Dapper, since it was the LTS (Long Term Support) release.  It has served me well, and I haven't wanted to mess with constantly (read: every 6 months) upgrading the OS.

Now, I'd like to install [Glubble]("http://www.glubble.com/"), which is a neat looking Firefox extension that uses a whitelist of websites to help control my kid's access (calm down, they're all under 5 years old).  Unfortunately, Dapper has Firefox 1.5 in the repositories, and Glubble requires 2.0 or better.

Reading through the forums, it seems that I can upgrade to Firefox 2.0 manually, but it doesn't seem entirely easy, simple, or foolproof.  It'd probably be easier to upgrade to Feisty, and then just use the 2.0+ version of Firefox that's in the Feisty repository.  I just haven't had the 2-3 hours to run through the upgrade, plus the extra hour to re-customize things that get messed up.  Yes, folks, admit it.  On a system that you use as your primary PC, with a lot of stuff on it, it's going to take an entire evening to upgrade from Dapper (to Edgy) to Feisty, and then get things tweaked back to how you like them.

I love the repository system.  I love opening Symantec up, finding something I'd like, and just installing it from there.  It's a sweet system.  There is, however, no simple way to upgrade from Firefox 1.5 to 2.0.  IMHO, that sucks.

Without attracting too many flames, that's something that XP does much better.  I can upgrade my Firefox on the XP machine within 10 minutes, and not disrupt anything else.  That's a serious barrier to me using ubuntu everywhere.  For something that I expect to be simple (installing software), coming to it with an expectation of simplicity from my Windows experience, it just isn't.

Your thoughts are appreciated.

Its Synaptic (Symnatec is shareware!!!:tongue:) and you can download Firefox 2 from the mozilla website. Ill tell the install instructions for every type of package (assuming it is on the desktop)

(tar.gz/tar.bz2/tgz/tar) Most have a configure shell script.  Right-click the package and select "Extract here". Open a terminal and do  these commands in order (after looking at  the new folder's name)
1) cd Desktop
2) cd <new folder's name goes here>
3) ./configure
If ./configure does nothing go to step 4
If ./configure gives an error after a few lines of output post it here and ill tell you what to do
4) make (most packages will say something like "Good, type make to compile the package)
If it is sucessful or does nothing, advance to step 5
5) sudo make install
It will ask for your password. That will do it.

(.deb) This one is easy. Right click the package and select Open with> GDebi Package Installer. If it has an error under Status: go to Synaptic and install or uninstall the package it wants/does not want.

(.rpm) It is unlikely they will give you an rpm, but you could use this for future reference for installing packages. Go to Synaptic and install alien. Then do:
cd Desktop
alien -i <package name>
The -i tells it to install the package immediately after it converts it. You can also list as many rpms as you want to install them all with one single command!

(.bin) The only binary installer i ever got was Java. Anyway, double click it. If it says it is an executable text file, click Run in terminal. If not i think right click, select Open With, and use the custom command "gksudo" and tell it to run that in the terminal.

Now you know how to install every package there is. If i missed one, feel free to PM me and i will edit this post. Same thing if something doesn't work or you don't understand something.
