### Post by jkxx on 2010-10-12
Been installing and trying the 64-bit versions of (K)ubuntu since 7.x and flash has worked very well since at least 9.x. I'm not exactly sure which flash gets installed and how it's connected to the browser but it does work so I'm not worried about that.

As for 64 vs 32-bit: If you just listen to music and browse the net you won't notice much of a difference. If you game but run 32-bit games, those will run just as fast as on a 32-bit machine. However, if you run native apps that transcode audio/video or do any number crunching, those will be significantly faster on a 64-bit system. 

With Linux all native apps in a 64-bit system can take advantage of the faster processing capabilities so you'll see a benefit in whatever CPU-heavy application you run.

### Post by cascade9 on 2010-10-14
> **ratcheer said:**
> Dumb question of the day: Is Ubuntu 64-bit version ***only*** for AMD processors? I have been wanting to buy an Intel i7 system for 64-bit Ubuntu, but the only 64-bit Ubuntu downloads I see seem to specify AMD64. If so, I am glad I didn't buy the system I almost bought earlier this week!

Its not only for AMD CPUs. AMD got the 1st 64bit extensions for x86 out, and since they made it, they called it AMD64 (x86-64). Intel 64bit CPUs use AMD64 as well. 

*edit- intel x86 64bit CPUs use AMD64, it wont run on non-x86 intel 64bit CPUs, eg, itanium. 

AMD putting thier name in is no different to i(ntel)386, etc.
