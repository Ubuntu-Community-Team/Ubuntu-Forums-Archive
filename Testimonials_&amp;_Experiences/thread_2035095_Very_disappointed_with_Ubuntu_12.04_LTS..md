### Post by levlaz on 2012-07-29
I am not a huge fan of Puppy, but thats just my personal opinion. Give Lubuntu a try, I honestly think you will be pleasantly surprised! 

Ubuntu USED to be great for old hardware, I ran Ubuntu 4.10 on a Pentium III with 512MB ram and it was a monster. 

But, as ubuntu has grown over the years it has gotten much more complex, and much less useful for old hardware. 

The problem isn't really Ubuntu per se, its the Desktop Environment. Unity takes a lot of resources because its designed to take advantage of the latest and greatest hardware. LXDE (The Desktop Environment for Lubuntu) is designed to work great on older hardware that requires much less resources. 

XFCE is another good choice (Xubuntu) I recently installed that on an old G4 Emac with a 1Ghz G4 PPC Processory and 1GB or Ram and it worked pretty well. 

At the end of the day, all of these various Desktop Environments sit on top of the Ubuntu base -- so many of the features will be the same. If you can go without all the pretty decorations that Gnome3, KDE, and Unity offer then you can have a rock solid Linux box running with LXDE or XFCE and that "old" Pentium D with 2GB of Ram desktop will last you another 10 years. 

Best, 

Lev

Despite people saying "You don't have enough RAM" and "Try Lubuntu", your experience definitely sounds out-of-the-ordinary. I use Ubuntu on a much lesser machine than yours and don't see what you're describing.

I think there's probably a background program running that is eating up your CPU power; if you run System Monitor you should be able to see what it is and then kill it, and hopefully find a way of stopping it from running.

Glitches like this can happen with any version of any operating system, but it usually indicates that this problem was not observed during the development phase of Ubuntu 12.04. Or, it was observed, somebody complained on the forums, and then was told erroneously that their computer is too slow for Ubuntu and that they should use Lubuntu instead.

In short: Check out System Monitor, you probably have a glitchy program that's running away with your CPU power. Kill it and Ubuntu will run fast again.
