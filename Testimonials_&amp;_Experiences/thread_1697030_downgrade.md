Puppy is also a good option to test if the user does not have more than 128 MiB of RAM.

There are still plenty of good options for safe use of an ancient machine (that is about ten years old) with some of these Linux distributions. My point is that Dapper Drake is probably slower and less appropriate than some of these other options - and they seem to be good, secure, current options - but I'm no real expert on the topic.

### Post by Kixtosh on 2011-03-01
> **Jerry N said:**
> ... although I don't understand how boot and shutdown times came to be performance benchmarks. ...

You're right, of course Jerry N! Boot times (and shut down times) are not true benchmarks for performance, but they are the most obvious indicator to the casual user, and also one of the most aggravating if you can't use suspend mode on an old clunker with no battery life left (replacement batteries are still expensive, and probably not what I would want to invest in for a ten year old laptop). The other "benchmark" I like is how fast the browser opens, and any other most used applications.

> I tried Xubuntu 6.06 on several old 400mhz class computers with 192 to 256mb RAM and found it to be useable but quite slow compared to Win 2k on the same machine.  I also found its boot time be be longer than Win 2k (which isn't much of a boot-time performer itself) ...Well, as I already stated, my boot times on a PIII 700 MHz were much faster with Lubuntu or any other LXDE environment than W2K, so maybe you had some sort of hardware issue that was slowing Xubuntu down. Let me point out, in case you didn't read the whole thread, that XFCE Xubuntu is considerably slower for me than LXDE Xubuntu. A fresh install of W2K is also much faster in my experience than (1) once any security software is installed, and (2) after several weeks/months of usage.

As for other performance pseudo "benchmarks", I can add that Office 2000 applications started much snappier with W2K than OpenOffice applications with Lubuntu, but then that's possibly not a very fair comparison since the OpenOffice applications were also much newer and up to date. Outlook also started much faster than Thunderbird. In general, I do find OpenOffice to be a bit sluggish to start, and Thunderbird as well.

It's all a moot point, though, since W2K is no longer supported and I don't think it can be considered safe to use. I used it and mostly liked it for five years. It was stable, reliable, didn't crash. I thought it was secure too, until the day that it wasn't. I like XP in the same way, except that I just don't know how to secure it to a point where I would be happy using it all the time. Boot times with XP also slow down to five minutes on the ultraportable laptops I always favor (and which tend to have lower specifications - although that is changing dramatically recently). I've been using Windows 7 as well on a couple of laptops too, and I have to say that I don't think it seems to slow down much, but those are young kids laptops so they don't do much of anything other than visit a few kid oriented websites. 

Once my laptop is up and running, I'm mostly happy with performance, with either Windows or Ubuntu. The start time and shutdown time is still what aggravates me most if it takes too long. The Toshiba Portégé R500 I use (Intel Core 2 Duo 1.20GHz, 2MB L2 cache, 533MHz FSB) boots Ubuntu in twenty-five seconds, which includes five seconds of BIOS before the GRUB menu appears. That's obviously as close to perfection as I need right now. What would be really nice would be if it could hibernate that fast, so that I could save work sessions, since I frequently use multiple desktops simultaneously.
