### Post by cwortman on 2009-01-19
Linux is not Windows. I wish everyone would stop saying how amazing it is as a Windows replacement. To those who have not fallen into this delusion, thank you. I tested Ubuntu for quite a while. I have found a lot of free applications I will be taking back with me to Windows. After using it I have learned to respect computer programmers and to better use Windows.
	I started using Linux back in 1998. Back then everything was unstable and barely a thing worked as it should unless you ran Red Hat. Even in Red Hat things as a desktop were not up to par. It is now 2009 and the story is the same for me. I never really learned how to use a computer until I came upon Ubuntu. Oh sure I knew how to install Windows, Drivers, and a few applications. I knew enough to stay alive but I did not know enough to know I was barely floating in my own feces. I am going back with sound mind, soul, and body with a sense and better respect for programmers. I now know not to take everything at face value and enough to buy hardware more sensible.
	Ubuntu fails for me as a desktop and as a server. I found Debian server, however, to be perfect for what I need it for. I have money stashed away and will be purchasing Windows 7 Ultimate non-oem. When I do install it a slew of FOSS applications have, however, replaced commercial ones. Open Office.org or AbiWord and Gnumeric replace MS-Office as this used to be a regular purchase with Windows and saves me a few hundred, Gimp to replace Photoshop, Pidgin to replace my 10 different instant messaging clients, I already was using Firefox but will continue to use it, VLC player and Fusion instead of WMP, the list goes on... I will be making an icon set out of the icon set I use here, and I am taking the fonts because they are the easiest font I have ever come across to read.
	It honestly feels as if the bridge to FOSS OS instead of Windows is being built fast, but there are a lot of things which I won't miss either. Sudo is a huge let down to me and was the deal-breaker for me. Relying on a password for an entire security model is really insecure and quite frankly scares me. If this catches on idiots will use it and will use idiot passwords easily broken, which means viruses and spyware when it does exist will be a rampant mess. Other small annoyances include how slow things drag. I don't ever remember Gnome being this slow. I wish I could install Gnome on Windows and call it good. 3D games FOSS and NON-FOSS run better on Windows with more FPS. Nvidia 7800GTS SLI 8GB RAM 10,000 RPM HDD 500G. Some FOSS games simply crash out and don't run at all but in Windows they run impeccable. Wine does not run every Windows application I need (hardly the fault of Ubuntu) but I need my applications. Desktop effects seem to break other 3D applications and cause them to crash so I have to turn them off. 
	If Ubuntu 9.04 is anything short of amazing I won't bat an eye at Linux for at least a few years. I will come back to it and try it again, but if it fails by then, it will never make it and people will not want to use it and it will be a server/toy forever. This does not seem to be a huge problem for some of you. I will use my computer to do what I want and need with the applications I have no issue paying money for. If a free program comes by that completely blows my mind like OOo or Gimp did sure I will use it instead. As for right now with my experience with people and computers, Linux as a desktop is just a pipe-dream for the mis-informed. Linux is not Windows and I don't see any programmers jumping through hoops to make it that way. I know enough now to remove Internet Explorer, Windows Media Player and a few other un-needed applications. I know enough to install sensible FOSS ones if they fit my needs. I will continue to follow Linux and other FOSS news as I am now a huge fan. Too bad I will be cheering from the stands.
	To those of you who are hoping it will replace Windows on every desktop. I do not think it will happen anytime soon. And I think for obvious reasons you might be oblivious to. The Open Source creed: I do it for me not for you. If it benefits you, then booley for you. Sure it might happen but it will only be a side effect if it does.

### Post by BIGtrouble77 on 2009-01-19
cwortman,
You don't really explain why you don't like linux.  You just say it didn't work for what you needed it for.  I have a feeling you were looking for a drop-in replacement for Windows, which you obviously found linux is not.

From my experience, out-of-the-box no distro is perfect.  It may take a while to mold it to your specific workflow.  Unlike  osx and windows, linux encourages you to do things your own way, not the way ms and apple want you to do things.  That can be hugely intimidating for novice users.  5 years ago it took me about 6 months and a ton of effort to really get comfortable with the transition.  

As for as 'SUDO' goes, I don't think you really understand the purpose of why we use sudo.  Linux only allows you to run certain applications and commands as root (the linux admin).  The old practice was to log in as root and run these tasks, but it's very dangerous to use the root account because you can easily make mistakes that will break your system.  Introduce sudo, you can now execute root commands with your current user as long as you belong to the sudoers group.  This is the same way osx authenticates.  It works and it's the best way to manage your system.

It doesn't sound like you are comfortable with the command line so just use that ADD/REMOVE programs feature, it was made for you!

I can't help much more if you don't tell us specifically what your issues were.

### Post by 3rdalbum on 2009-01-21
> **cwortman said:**
> Obviously you have never used Vista. It pops up a message asking for your input Yes or No. Clickable and easily disabled. Once disabled it does not disable other security protocols put into place.

Disable the UAC prompt and you disable most of the other Windows Vista security features. Disable sudo and you still get the execute permission, AppArmour, and the compiler's native anti-attack functionality (address space randomisation and stuff).

> It was vastly improved in Windows 7 as well.

No, it was just made more permissive.

Claiming that Mac OS X has a better security model is quite naive; we're talking about an operating system that could be "rooted" by any local or remote user regardless of their permissions; this security hole was present since the first OS X betas and Apple was warned about it four years ago; they finally pushed out a workaround in August last year.

I'm not sure what your beef with "sudo" is. Unlike Vista, there's no way a gksudo prompt could pop up onto your screen, triggered by a virus, and you possibly think that you've triggered it. Sudo has been in use in enterprise and national security organizations for decades without incident.
