### Post by k99goran on 2008-04-25
> **yorkie said:**
> I think sometimes delaying a release to get most problems fixed is better for everyone its been done before. Why have a fixed release date anyway?. Bring a new release out when its ready.
I think the problem is that:
1. Software is never really "ready" (as in finished, as in works all the time for everyone).
2. Developers are dependent on a larger group of people using the software in order to fix remaining bugs.

It's not like having a feature-based release indemnifies you from bugs or problems. Consider Windows Vista and KDE4 the day they were released. Whatever problems Hardy Heron may have for some people pales in comparison. Ubuntu is the only operating system I've used that I can actually install and use as my primary OS just hours after its release.

I think that having a periodic and frequent (counting alphas) release is a great marketing move. It means that when all the excitement about Microsoft's and Apple's new operating systems have died down, there are still people getting excited about the next release of Ubuntu just around the corner. Trickling out new features in smaller packages also ensures people do not miss them. (How many people actually read the entire 300 features list of Leopard?)

Making a release feature-based also has a tendency to increase expectations, which invariably leads to disappointment.
