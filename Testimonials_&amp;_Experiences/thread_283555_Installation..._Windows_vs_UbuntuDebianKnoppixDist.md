### Post by cunawarit on 2006-10-24
OK, in a way I sometimes feel like I am the Windows defender here. It had been a good few years since I installed XP from scratch, I think I have been spoilt by all the modern Linux distros I have installed lately... OH MY GOD! What a painful experience!

The desktop is your basic Compaq desktop machine, a P4 EVO D510, a common as muck machine. Did XP auto-detect everything? DID IT EVER!!!!

It booted into 640x480 4 bit colour, with no Ethernet driver, no sound driver, and clearly no video driver either. Thankfully USB worked so I was able to get the Ethernet driver to it that way. After enjoying the Compaq/HP site in 4 bit colour and getting the right drivers for the rest I wanted to install IIS, was it easy? 

HELL NO! Why? Because I had given back the Windows XP install CD to the Helpdesk team. And it expects the disc, no “apt-get install IIS” or anything quite as elegant.

This is frustrating! 

How come not many people sing praises about how easy some of latest Linux distros are to install? I’ve had a few dump me straight into the command prompt because X couldn’t work as the graphics card wasn’t detected, however, even that was a breeze compared to the XP install experience. That’s not even mentioning what the install looks like! It is like traveling back in time to the 80s!

PS: I do still like Windows BTW, and I hope Vista is a little less painful to install.

### Post by ericinwisconsin on 2006-11-01
I've probably tried every popular Linux distro and a few that are lesser known. I started with Redhat 5.1, many years ago, and continued from there. The easiest to install, hands down, has been Ubuntu. Detected everything out of the box.

I started messing with Linux for my home computer about two weeks ago. Before I was married last year, I ran Linux all the time. After marriage, I decided to stick with Windows since that's what the wife and stepkids knew.

I quickly learned that the sum total of their Windows use was web browsing, chatting, and music. I thought "Heck, I can get a Linux machine to do all that," and I started checking out distros.

I tried Mandriva first. I hadn't used it since it was Mandrake and always found it to be filled with software. It was always a bit more "bleeding edge" than Redhat, so I figured it would detect my Belkin F5D7000 wireless NIC more readily. Nope. I tried Mandriva and Redhat and MEPIS before I realized that Linux just wasn't going to support this card, at this pooint.

When I installed Freespire, it found the card. I could get an IP from the router, but I could not, for the life of me, get it to reach either the net or the router's webpage. In disgust, I went out and bought a Linksys WMP54G wireless NIC. Freespire found it immediately and it worked.

I stuck with Freespire for a total of two days. I just wasn't happy with certain aspects of it. I said "Let me try SuSE. It has all the packages I could ever need." Well, SuSE 10.1 had problems. It didn't support the NIC, but I managed to get it to. But I kept losing my gateway to the router, and every time I had to reboot the router (it's a Linksys, so that's a given), it locked up SuSE!

I finally tried Ubuntu and I love it. Easy to install, easy to add software. I'm happy.

I may throw another hard drive in the system and try another distro for fun, but chances are I'll stick with Ubuntu. It even detected my NTFS drive and configured Grub to properly boot Windows 2000; something no other distro I've tried has yet been able to do without me editing Grub.
