### Post by cas007 on 2010-02-03
Yes, this is a rant. So I really don't care who says what. And this post has come about due to trying to install a Canon MP 130 printer in Ubuntu 9.10 (and no, I do not want to install an iP1500 printer either) and failing, without any reason being given (by the software). I have spent over 2hrs on this bloody task which has sucked up MORE time than any interation of Windows ever did.

There is a fundamental problem with those who create this type of software. These developers hope that Linux or any distro of it, like Ubuntu, will unseat Windows as the OS of choice for everyone. Well, this ain't gonna happen in a hundred years. And here is why .....

Trying to install a printer is a very simple matter, or so it should be, at least in Windows it is. Yet, when being confronted with a situation where the printer did not work (yet is recognised by Ubuntu), I go searching an the net for a solution and get taken to a couple of posts here as well as one elsewhere where the poster talks about installing the driver through a terminal window. Well, the poster failed to convery ALL the steps that he took. ie. Where is the terminal program found? Spent 10 minutes looking for that.

Poster then said to go to /etc/apt/ and look for a file called sources.list failing to state how one goes about navigating to that file. I gave up in the end on trying to find it. The terminal window did not find it (or if it did it did not state that it had found it), nor was it found (after ticking 'show all hidden folders' was done)under the root folder.

Those that have written this OS (Linux/Ubuntu) seem to disregard what makes Windows so appealing. Namely that anyone can use it without being a rocket scientist; printers, and just about anything else, can be installed by clicking on a few buttons with or without an installation software CD; nomencalture used is something that anyone, with a modicum of reading/study, can understand; instructions are easy to follow; AND one does not have to go to a forum such as this and be presented with the problem of how to search for a problem/post which causes a search page to come up with a dizzying array of options that are hard to follow so that when the submit button is clicked it doesn't find anything and subsequently comes up with an error saying that one has to refine one's search parameters. 

I ended up using google to find what I was looking for in this forum. This is pathetic and is an example of the sort of stuff I am talking about; techies/geeks/nerds/what have you creating an OS and all the extras (forums, help files, etc) that make them feel superior to every other person trying to get a mundane task completed. This is accomplished by using terminology that alienates (deliberately so, it seems) regular users because they don't understand it, procedures that cause one to spend hours trying to figure out how to get something to work, and the list can be continued for a long, long time. 

I actually spent 1 whole week back in 1999 trying to install three different versions of Linux (Red Hat 6.0, SuSe, and Mandrake) on my humble 233mghz AMD machine that was custom built for me. Each of these attempts failed. In the end I gave up (after spending 6-7 hours PER DAY on this). Yet, I was able to learn from that experience, partitioning my hard drive to have a windows partition, a programs partition, a My Documents partition, a temp partition, two spare paritions, and a swap partition (and I have never had any more problems with my machine EVER again).

Now I am not a non-techie, having built and configured the first ever 12 monitor computer system in the UK in the same year. This was a system for a friend of mine who was trading the markets. Even his brokerage house had never seen anything like it. Yet it was easy for me to do. I used 3 Appian Jeronimo Pro Video cards (4 monitors per card, which acted independently of each other, 8MB video memory per channel), in one computer case that one had a Pentium II 500Mghz chip, 512MB memory, and an NT4 operating system. So, I can speak with some authority on my knowledge of computers.

If those who have created the Linux/Ubuntu OS had the right attitude, they would have made it easier to use this OS than using Windows, but this is not the case, is it? These forums wouldn't exist otherwise.

So there you have it. I could go on about this for quite a while, but why bother? And here is the sad thing, when I first read about what Linus Torvalds had done back in 1996 I knew that Linux would be a force to be reckoned with and would supercede Windows eventually. And so I was very enthusiastic about what he had achieved. Sadly, this has not come to pass, and with the experience I have had in trying to install Linux/other distros back in 1999, and the problems I am having with trying to install a printer in Ubuntu (my mum is the one having the problems after I told her to switch to Ubuntu after she experienced a virus in her old Windows XP system), I am definately NOT going to go to Linux any time soon. Maybe never. So, much as I admire what Linus has been able to do, his vision (whatever that may be) is just that at the moment. For me anyway.
