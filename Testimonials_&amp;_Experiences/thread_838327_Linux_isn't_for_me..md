### Post by wdaniels on 2008-07-08
> **duckling said:**
> The problem is, that my botable windows OS disc will no longer boot, it boots on all other pc's i have (4of5) but not on this ubuntu os system.

If your Windows installation disk just won't boot then wiping Ubuntu won't make any difference. Deleting the partitions is indeed not the same as formatting, but it will have exactly the same effect so far as "wiping" Ubuntu is concerned. You will probably need to delete the partitions to install Windows anyway and the Windows installer will format them itself, so there is not a lot of point in formatting partitions from Ubuntu if you want to install Windows. But if you don't believe me, the relevant commands are as follows:

If you want to format to NTFS, you first need to install the package ntfsprogs:

```
sudo apt-get install ntfsprogs
```

You can then use the command mkntfs with the device name for the partition to format (e.g. /dev/hda1). Read the mkntfs man page (using the command "man mkntfs") for more info.

Alternatively, if you want a FAT32 filesystem, you use the command mkfs.vfat (or mkdosfs -F 32) in the same way.

You can format for many other filesystems in Linux, but NTFS and FAT32 are the main ones used by Windows. Note that you should do this from a live CD boot since you cannot format a partition that's currently being used.

Be a little careful here - you seem to be under the impression that Ubuntu has something to do with being unable to boot your Windows  install disk - this is not the case, the BIOS only reads the disk it's trying to boot and it makes no difference what's on any other disk. If it won't boot then it won't boot. And if you wipe Ubuntu first then you won't have any OS at all.

I'm sure most people here will be happy to help you in whatever way we can, but ultimately, if you need assistance installing Windows then you need to ask on a Windows forum or contact Microsoft support directly.
