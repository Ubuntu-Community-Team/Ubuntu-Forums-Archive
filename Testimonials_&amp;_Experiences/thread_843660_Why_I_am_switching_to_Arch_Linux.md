### Post by jjk7288 on 2008-06-28
First let me say that Ubuntu has been a great distribution -- I started using it in January of this year so I could be more consistent with lab machines for my Scheme (yuck) programming course.

Since then, I haven't really looked back at Windows. I still have a Windows partition, but out of 1.5 terabytes of storage that I have, a mere 80 gigabytes belongs to Windows. I probably boot up into Windows on average once a month, and only to do some hardcore gaming.

Ubuntu has done a great job easing me into the world of Linux, and these forums have been an especially great place to get help and learn new things. More than anything, the Ubuntu community is by far the best thing going for it. I'll still be lurking. =)

That said, here's why I am giving Arch Linux a go. First, Ubuntu (as far as Linux distributions go) is **bloated**. Before trying Arch I gave Fedora 9 (which sucked) a go, but Fedora completely smashed Ubuntu in terms of speed. Arch Linux is even faster than Fedora. I believe that this is due to several reasons; one that Ubuntu is compiled for the i386 architecture (where Fedora and Arch are i686) and that Ubuntu comes with so much extra stuff that probably isn't needed.

Along those same lines, the customizability of Ubuntu during the install and initial setup phase really needs work. Why, after so many releases, is there no good method for selecting packages? I read somewhere that you can run Synaptic on the live CD and the packages you choose will be installed with Ubuntu, but this interface is clunky at best. In addition, I think that perhaps default installs should be more minimalistic. Maybe the installer could have a "newbie friendly" option that installs everything that usually comes by default so that newcomers don't get lost in the shuffle. Once people start becoming experienced, however, the lack of package selection is frustrating.

Finally, I have to say that as most people who start with Ubuntu are trying to learn about Linux, it can sometimes be frustrating that Ubuntu "breaks the standard" and does things its own way, for simplicity or otherwise.

That said, the primary reason I'm switching is for the speed boost. Most of the other stuff is kind of nitpicky.

Keep working on it, though -- I'd love to come back eventually, especially if there is some serious improvement on the 64 bit front or if you ever make the decision to start compiling for i686 (the speed difference is truly amazing).

Thanks for being a great community.

### Post by DrPppr242 on 2008-07-05
I started out using Linux with Ubuntu as well and eventually (after trying a lot of distros) ended up using mostly Arch on my system.  Ubuntu also feels very slow on my machines now to, and in my opinion this isn't as much due to the software bloat (although it is a little annoying how much it installs by default) as it is to the support for older hardware in the kernel and modules.  My point isn't that ubuntu is a bad distro because it does a lot of things very well, but if you want your system to feel much faster rather than trying debian or fedora use one that works well for compiling your own kernel and build support for your hardware into it rather than using modules. If you don't mind spending time for the performance gain nothing compares to gentoo for a stripped down fast system.
