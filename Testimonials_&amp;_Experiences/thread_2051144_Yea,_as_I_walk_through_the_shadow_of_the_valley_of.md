### Post by 3rdalbum on 2012-09-01
As you will see to the left of this message, I've been using Ubuntu since 2005. I started off on a PowerPC Mac dual-booted with OS 9, moved to a PC dual-booting with Windows XP (yuck) and then my next four computers were Ubuntu-only.

I'm taking the unprecedented (for me) step of installing Windows onto my desktop. Not in a virtual machine or "to run one program I need", but as a proper part-time, maybe full-time, operating system.

Linux seems to be a parade that goes like this:

"XYZ has a limitation! Let's write something completely new to replace it. We could call it XYkit".
"Okay, now the distributions have adopted XYkit, but it's buggy. Let's fix the bugs."
"Now we've fixed most of the bugs in XYkit, we've seen that XYkit has a limitation. Let's write something completely new to replace it..."

The users bear the consequences in the form of regressions, half-baked features and plain flakiness of the system. They also have to learn new ways of doing things every six months, and if they ask for help on the forums they get given advice by somebody who hasn't kept up with the changes, and whose advice is now worthless.

I used to find it exciting, but now I'm getting sick of everything regressing and breaking in every version. The typical response here is "Use an LTS, then, it's super-stable". I do use an LTS - 12.04.

Then the typical response is "Use 10.04 instead"; but do I really want to be stuck with ancient software? It's not like we can just install newer versions of your favourite programs on an old Ubuntu, because the program developers have decided to use the newest XYZkit in their program instead of the old XYZ that shipped with the old Ubuntu.

What if I want to use some older, unmaintained software with a newer version of Ubuntu? It probably won't work anymore. I wrote a program to use ffmpeg to encode some video and it partly broke in a later version of Ubuntu and then fully broke six months later; I don't have time to keep rewriting my programs to keep up with these changes. It just ends off with bloat if I have to have one code path for 12.04, another for 11.10, and another for 11.04.

Currently, although my netbook is running like a dream, my desktop is full of crazy regressions that I can't even begin to explain. I don't enjoy being frustrated by my computer.

The other problem exists when programs reach a state of maturity and developers lose interest. My father today was asking me when a particular feature to do with layers was going to be implemented in GIMP. My honest answer was "probably never" as the developers had been talking about this kind of feature for the last five or six years. I'd like to be using modern, attractive, and featureful software on Linux but it doesn't exist.

So I'm going to try Windows as a potential full-time operating system. I love Unity and wish I could take it to Windows with me. There are some parts of Ubuntu that I love, that are despicable and loathable in Windows; but I'm going to hope that they're better in Windows 8 (which is what I will try).

I'd love for Ubuntu and Linux in general to settle down a bit with its infrastructure and actually get some high quality application software written, but I have grave doubts that this will happen.

This certainly isn't goodbye, I'll still be around and I'll still even use Ubuntu from time to time, but I thought I'd put my testimonial out there and see if anyone else has felt this way.
