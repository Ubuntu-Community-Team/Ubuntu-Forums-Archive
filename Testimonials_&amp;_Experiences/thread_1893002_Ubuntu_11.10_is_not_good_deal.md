### Post by Bucky Ball on 2011-12-10
Reinstall 10.04 LTS. It is the most stable and was working. Not sure why it is not made more clear that the latest release is not (generally never) the best. These things will have bugs for a few months which may/may not get fixed before the next release is out, then support dies within 18 months anyhow. 

I try to stick with the most stable releases - LTS=Long Term Support - intended to be that way and supported for three years by which time the next LTS is out, tried, tested and generally stable, and my motto: If it ain't broke, don't fix it! ;)

Good luck.

PS: You can set your machine to only offer you LTS releases, not every damn unstable thing that comes out. I would wait a few months after the release of an LTS also but I require/need stability/reliability as production machines (they *can't* break).

### Post by Bucky Ball on 2011-12-11
> **wazupwiop said:**
>   I usually try to keep up to date with every xx.04 release.  That way I get the LTS support, but I stay updated yearly. 

Every **.04 release is *not* an LTS release. They come out every two years and are supported for three years. E.g. 11.04 is *not* an LTS release. 10.04 LTS is and 12.04 LTS will be (obvious as the release name finishes with 'LTS'). ;)

An update is not a fresh install. That is simply a fresh install. If you are installing a newer release that a release upgrade (as opposed to a package upgrade via 'sudo apt-get upgrade'). The wrong terminology can be awfully confusing, specially for newcomers to Ubuntu.
