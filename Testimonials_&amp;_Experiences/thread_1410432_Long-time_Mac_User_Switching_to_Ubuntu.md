### Post by bakelitedoorbell on 2010-02-18
I won't bore you with a list of all my old computers, but they have all been Macs since the mid 1990's.  Never had a home computer running Windows.  Last year I decided to switch to Ubuntu as my main OS.  Why?

1. I wanted a larger screen resolution than was available on any Mac portable.
2. Attracted to the idea of software freedom.
3. The challenge of learning something new.

In June of last year I bought a System76 Serval laptop with a gorgeous 15.4" 1920x1200 matte screen.  For the last 7 months I've been running two "main" computers, going back and forth between my 3-year old MacBook and the new Serval.  Now I think it's time to finish the switch and sell the old MacBook.

Ubuntu can do almost everything I want to do with a computer.  The only problems are:

1. Getting my USB video encoder box working so I can watch cable TV on my computer screen.  I have the driver and may yet get this going.  Or I might just use Hulu instead (which works great).

2. One of the TV shows I watch online is on CWTV, which streams video using the Move Media player that has no native Linux version.  I have had some success using Firefox2 in Wine with this player.  The image flickers too much, but I'm still tinkering with it.

3. I use an iPod Touch as my mobile pocket-size web browser, and run Apps.  iTunes 9 has no Linux version and won't run in Wine.  iTunes seems to be necessary to install firmware updates, manage Apps, and do backups.  So now I'm in the strange position of thinking about buying a Windows OS, installing it on a partition on my Ubuntu computer, so I can run Apple software!  I really am reluctant to buy Windows and give up 40 GB of my hard drive so I can run this one program.  I like my iPod Touch, but I'm trying to decide how much I like it.

### Post by marshmallow1304 on 2010-02-20
> **bakelitedoorbell said:**
> 4. On Mac I had Sleep mode.  On Ubuntu there is Hibernate and Suspend.  Two things which seem to serve the same purpose.

To clarify, Suspend saves your current state (open programs, etc.) to RAM and powers down everything else.  This uses a small amount of power, but resuming is very fast.  Hibernate saves your current state to the swap partition and powers down everything.  This uses no power, but resuming takes as long as a regular boot.

From what I've seen with my "new" old G3 iBook, Mac's Sleep mode is more similar to Suspend than to Hibernate.

### Post by 3rdalbum on 2010-02-22
> **marshmallow1304 said:**
> 

From what I've seen with my "new" old G3 iBook, Mac's Sleep mode is more similar to Suspend than to Hibernate.

I'm pretty sure that Mac OS X, like Windows, doesn't "either" suspend or hibernate.

I believe it "hybridnates". It save the running state to disk before suspending normally. If you don't lose power, you unsuspend. If you do, then you unhibernate.

You can sometimes accomplish this on Linux by issuing the command:

```
sudo pm-suspend-hybrid
```

Doesn't work on my computer so I assume that it works for some people, and I imagine that's the reason why it's not the default.

Oh, and if you like the name Hybridnate, please spread it so more people use the name :-)
