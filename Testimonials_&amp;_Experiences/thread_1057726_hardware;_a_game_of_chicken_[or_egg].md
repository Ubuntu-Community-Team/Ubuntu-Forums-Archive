### Post by jbysmith on 2009-02-04
> **RaiCoss said:**
> Yeah they worked fine with Compiz turned off, unless you tried to use WINE, or when they sometimes seemed to uninstall themselves, and how do you install the latest nVidia driver in Ubuntu?? Everytime I've tried I broke my install :(

Want to read a wall of text?  Course ya do.

It's not too hard really.  Some people swear by EnvyNG; I've had good results with the older versions in the Ubuntu 7 series; once it got to 8.04 I've had nothing but problems; it's never worked for me and I've had to do it manually.

I'm just waking up and currently hooking up my coffee IV, so if anyone noticed I missed anything, speak up :D  Also note that I've *never* had issues doing this.  But I'm not everybody.  Be prepared for breakage, explosions, crying fits in the corner and whatnot if things go wrong.

You'll want to print these or jot the instructions down; halfway through Gnome is going to get shut down along with your web browser. 

I've done this method with 8.04 and 8.10 on the x86 platform with no issues on various distributions.  Note that as I understand it, this does *not* currently work on 9.04's beta as it uses a newer X server that the drivers aren't ready for; I haven't tried it so don't quote me.

First thing you need is the driver from the nVidia site.  As I type this, current stable is 180.22, current beta is 180.27. You can find some links [here on nVNews.](http://www.nvnews.net/vbulletin/showthread.php?t=122606) Save it someplace where you can find it again.  If things go wrong, you should be able to uninstall the driver, and restore this configuration file getting you back to where you were if things go wrong.  Again in theory; be ready for troubleshooting.

Also note that this is for current hardware.  If you have something older (IE legacy TNT2 through GeForce 5's), chances are you'll want to use the 173, 96, or 71 series drivers depending on the age, and chances are those are already in the repositories.

You may or may not need the kernel headers too if I recall.  I typically build my own kernel and it's never said I was missing something,but reasonably sure it needs these.  The installer will give an error and quit, telling you if its missing anything that it needs.  I believe the headers package is linux-kernel-headers but double-check in Synaptic.  

You should also back up your X configuration as well, just in case.  If something borks badly, you can revert to it, and I typically let the nVidia installer rewrite this.  Copy the /etc/X11/xorg.conf file somewhere.  Note that X11 has a capital X, xorg does not.

Ok, now you need to shut down the desktop.  Make sure all your applications are closed.  Press Ctrl-Alt-F1 to get into one of the running consoles not attached to the X server, and log in.  This varies on whatever window manager you're running.  For Gnome, you'll want to do, sans quotes, "sudo /etc/init.d/gdm stop".  Replace gdm with kdm if you're running KDE.  Other environments you'll need to look up.

Run the installer.  Navigate to whereever it is you've stored it, and type, again no quotes, "sudo sh ./NVIDIA-Linux-x86-180.22-pkg1.run"    Change that version number if you're using the beta or x64 of course.  It's a fairly straightforward installation.  Just agree to the license bit, and pretty much say yes to everything including generating an X configuration file.  If the installer fails, take note of what the problem is, it's probably missing some files that you'll need to grab via apt-get.

That's pretty much it.  The official instructions say you can just restart things via a "sudo /etc/init.d/gdm restart"; personally I just do a sudo reboot to be completely safe, as old Winders habits die hard.

If everything's ok, the drivers should pick up on your monitor's ideal resolution and off you go.  (It picks up my LCD's 1680x1050 default res flawlessly.)  If it doesn't work, Ubuntu should eventually come up in a graphic fail safe mode; uninstall the drivers and copy your old X config back and you *should* be back to where you were.

It's really pretty easy, just be prepared to deal with a problem if things don't work as with anything you do outside the official Ubuntu repositories.  Also note if you later recompile your own kernel or if it gets updated if you use the stock kernel, chances are you *might* need to do this process again.

If things work properly, you should see your typical Gnome desktop, and Compiz will automatically kick in as it sees you're using a video driver that'll support it. Depending on your hardware and level of "tweak geekyness" you can edit xorg.conf to suit your tastes.
