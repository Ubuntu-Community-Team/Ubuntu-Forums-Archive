### Post by Novega on 2008-05-11
Almost everyone I've heard complain about Ubuntu (and linux in general) has said the hardware drivers/recognition was the biggest flaw. Having FF 3.5 _beta_ included might have been an unwise decision as well.
Hardware from the brand name companies is easily installed in XP... it usually works, where in linux you're really just rolling the dice.

Examples: 
Pick any wireless network card off the shelf, install it in Windows, it (usually) gets auto recognized and you install the *supplied* driver, product usually works.
Exact same card in Linux, install it, the system might recognize it...or maybe not. Ok, so now what? 
Let's assume it does get recognized, I still need drivers, so now I have to download the Linux driver (if there even is one) and install it using a command line??? Cross your fingers and hope it works...if not, prepare to do some reading and waiting.

Video Card:
Windows: Same as above. supplied driver, clicky clicky and away you go.
Linux: goodluck easily getting to the login screen of any distro besides Ubuntu with a Nvidia 9 series card. Once you're in, you have to jump through hoops to get the driver installed *(figuring out how to stop gdm, using command line to install it, stuff with the kernel, restarting the desktop,editing text files to ensure no conflicts)*


Thats not easy for most new users, so I can kinda understand why they feel that way.
PCLOS,Mandriva,Linspire,SuSe and Dreamlinux all were failures in one way or another for me. BUT, I happen to have only a few problems with my system when installing Ubuntu.

If the circumstances were different, I might feel the same way.
