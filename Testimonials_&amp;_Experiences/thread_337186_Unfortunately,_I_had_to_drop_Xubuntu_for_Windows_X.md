### Post by rodrigo666 on 2007-01-12
Greetings,

First of all, I am a big fan of Ubuntu and use it as my main OS on my PC and I am always telling others to do so. The only reason I still have a WindowsXP on my PC is that I use it to play NBA Live, my favorite game of all time.

This week, my girlfriend brought me the PC from her work place to me. She wanted to me install some aplications and, among other things, a Linux system.

The box was a Intel Celeron 567 MHz with 64mb of RAM. So I thought with myself, Xubuntu was made for this, right?

Unfortunally, wrong.

I installed a Xubuntu Dapper Drake in the box with dual boot and tried to upgrade it for Edgy Eft, I followed three diferents tutorials and none worked.

Okay then, Dapper Drake is just fine I thought.

But the thing is this, I realized that Xubuntu and the XFCE graphical enviroment isan't fast as Windows XP.

As a matter of fact it's far more slower!

Then I tried to Install and use IceWM, and I really liked it, since it is really faster than XFCE and looks a lot with Windows (that's a bonus, not for me, but for the people who will use the PC in question).

But there was another problem with that, neither IceWM or XFCE has half of the resources that WindowsXP has, like wallpaper (easy) selection, right button mouse features, and a lot of other important things, specially the speed.

And the major problem with IceWM was that the softwares such as AbiWord, Synaptic Manager and others, wasan't availables in the start menu.

Well, the bottom line is that altought Linux systems has a lot of good carachteristics like security, open source, coolness, etc, Xubuntu with XFCE or IceWM does not beat Windows XP for a low end system in terms of benefits.

I haven't tryed others distributions and I do hope they could be as good (or better) as WindowsXP for a low end system like a 500 MhZ with 64mb RAM, but I have to state that Xubuntu is not one of those.

Unfortunally.

### Post by revertex on 2007-07-12
never try install a full featured linux in a slow machine, it's a waste of time.

it's  better to use another distro that give you a better control ove what is installed, like slackware.

Start with a bare minimum command line only install, then install a lighweight WM like fluxbox or openbox, but never a DE like gnome or xfce.

you can do it with ubuntu if you prefer, but grab the alternate install cd, do a CLI only install, no GUI, then install xorg and your favourite WM.

Also a kernel recompilation can speed up things a lot, strip all useless junk from kernel config and install a lighweight one, but recompile it in a faster machine unless you don't mind to spend all day to recompile.
