### Post by Mr. Picklesworth on 2008-04-25
I've been quite happy with it (upgraded to a late alpha quite a while ago). Mostly happy improvements all around, even with the upgrader (as opposed to a fresh install). I am bothered by two new things that obviously need some work:

-Large file operations (in particular multiple large file operations) cause the system to hang up in a horrible way. I don't know whether to blame the kernel, gvfs or both. It does need to be addressed, though.
-PolicyKit is nice, but was poorly implemented from a usability perspective. The GNOME Usability team is aware of this, so hopefully we will see some nice changes. For one thing, the Unlock button is placed where people will not notice it. It would make a lot more sense at the top of a window where it could serve the dual purpose of notifying people that configuration is locked and being the method of unlocking. Alternatively, there could be no unlock button; it could simply pop up the unlock dialog when the user starts making changes... although that could be irritating.

Other than that:

-Yay, Wacom drivers in the repos work with my Bamboo tablet! No more recompiling drivers with every kernel update.
-xorg.conf is less ugly looking thanks to autoconfiguration stuff.
-The new kernel is visibly more responsive.
-Nobody is going to install Automatix on this release. It will be interesting to see what happens with bug reports and support requests.
-Etc...

But then they go ahead and in the late hour implement CFS and leave the desktop version with a [moronic (click here)]("https://bugs.launchpad.net/ubuntu/+source/linux/+bug/188226") CPU  schedule setting suitable for servers, and boom, bye-bye desktop responsiveness. Remember feeling Hardy laggy, or having stuttering audio whenever you ran anything as root, like the update manager? Was it annoying?

[Quote 1 (click here)]("https://bugs.launchpad.net/ubuntu/+source/linux/+bug/188226/comments/32"): "Can someone explain to me why a regression known to cause stuttering sound is not considered a show-stopper?"

[Quote 2 (click here)]("https://bugs.launchpad.net/ubuntu/+source/linux/+bug/188226/comments/42"): "I really do wish that desktop responsiveness gets the same attention in the future. Going from a clean install of gutsy to a clean install of hardy felt like going from win 3.11 to Vista."

My thoughts, precisely. But wait... that's not over. As if to create a perfect storm of audio problems, they also decided to go ahead and release the LTS with pulseaudio, clearly [not ready for primetime (click here)]("https://bugs.launchpad.net/ubuntu/+bug/190754") yet. It's no wonder even package maintainers couldn't predict [all the problems (click here)]("https://bugs.launchpad.net/ubuntu/+source/vlc/+bug/208579").

So, you see... A new interface design isn't suitable for a LTS, but a major new CPU scheduling strategy, untested, and a new sound system is fine. To sum up my disappointment, I'll say that Hardy is the first Ubuntu release that is seriously making me question whether the 6 month release schedule is a mistake.
