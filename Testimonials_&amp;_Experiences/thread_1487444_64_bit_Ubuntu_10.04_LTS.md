### Post by POWMS on 2010-05-19
My mum recently acquired an ASUS AMD64 desktop, with XP 32 bit pre-installed. It was slower than a wet weekend.
I was about to format and reload XP when I decided to try the new 10.04 LTS. I have Xubuntu on my notebook, that usually requires an extra hour or two of tweeking to get it to run as it should after a fresh install, so I allowed for extra time for this one.
To my great surprise, all I had to do was check the multiverse box, do an uptade, install a few codecs and it all works flawlessly. Well done Canonical on this release. Boot speed is in seconds rather than minutes.......and applications open in seconds too.
This is the kind of installation that will draw more users to Linux based OS.:)

### Post by rjbl on 2010-06-09
I have had linux boxes since redhat 5.5 but have been forced to be a Windoze freak by work and Flight Simulator needs for about the last 20 years. Just knocked together a new AMD64 3800+ box to see me through the the next ten years of my retirement, to fill all my computainment needs except flight simming for which XP remains the only show in town.

Of course I loaded up Fedora 13-64bit on the new box as first choice and for the first time *ever* had a tricky linux install, followed by a really difficult configuration. It is *so* buggy it cannot be true. So, despite Ubuntu's warning that the 64-bit version of 10.04 LTS isn't best recommended for Desktop use, I downloaded the Live 64-bit spin and gave it a whirl.

The result? The fastest, cleanest linux install I have* ever *experienced. Flawless. System configuration? Again, fast and flawless. Took two days of work to pull in and set up *all* the apps I am going to need and to get the Desktop looking just like I like a destop to look and I still had time to do a bit of gardening and shopping. Magic. Installing NVIDIA's graphics drivers? Trivial. no sweat.

I love the integration of Ubuntu into a wider, creative, computer using community. It is a brave new world and one so much more hopeful than Planet Micro$oft.

rjbl
