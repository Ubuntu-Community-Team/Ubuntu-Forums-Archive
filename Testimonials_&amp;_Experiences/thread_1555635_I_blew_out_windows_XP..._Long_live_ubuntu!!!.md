I hate to be the bearer of bad news, but if this is your approach you are likely to be sorely disappointed.

Whilst it is true that nowadays many, many things can be accomplished in Linux with a GUI, some things still need to be done via the command line.

Let me give you one little example to illustrate this point:

If you look in the Installation & Upgrades sub-forum you will find numerous posts from people who have run into problems with graphics issues or bootloader (GRUB) problems.

In almost all cases, the solution involves either posting the output of a command from the terminal to get more information or using the terminal to accomplish a certain task (such as reinstalling GRUB).

To repeat, you could, theoretically, use Ubuntu and never have to use the terminal.

But to suggest that in order for Ubuntu (Linux) to replace Windows you hope to never use the terminal shows a lack of understanding that Linux is not Windows.

This is just my opinion and I hope you will accept the criticism in the spirit in which it is given; friendly advice to a new user.
