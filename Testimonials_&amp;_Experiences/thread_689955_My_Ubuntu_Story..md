### Post by Dingbat on 2008-02-06
Well, how to begin.

A little bit over two year ago I started playing around with various distros, you see at home I have quite a few computers running (windows 2000 servers and workstations, a couple of WinXP boxes) and simply wanted to try something new.  

I've dabbled with linux over the years since redhat 4.0 so I'm not afraid at all of using the Shell.  What started pushing me over though was how crappy Vista was on brand spanking new hardware.  Having so many version of the OS with the UAC and other stuff just got me fed up with the whole microsoft experience.  The final straw was the inclusion of DRM built into Vista -- the basic assumption that all customers are thieves in other words.

The thing is -- I've always payed for my licenses.   Kind of a badge of honor.  I really really don't like it when a company starts assuming that we're all thieves and criminals.

So I started looking for an alternative.  I've always been tempted by Linux but it's only been during the last couple of years that linux has become easy enough to use to make it "usefull" (in the sense that, I don't need to fight with my machine to get something done).

First thing I tried was Ubuntu 5.04 (Edgy I believe).  It was an interesting attempt but at the time, the machine I was using had an ATI card and an old wireless-b card and I spent several hours [8+] to get everything to work.  That sort of soured me on Ubuntu 5.04.  Also having to fight for the right codecs for video or audio playback was a pain in the derriere.

Next thing I tried was Xandros -- everything worked!  But the software selection in their version of the CNR (re: Linspire) was pittifully inadequate, so unfortunately I gave up on it.

Tried Linspire, was nice -- didn't grab me though. 

Even tried Gentoo ... building my pleasure ONE .... BRICK ... AT .... A ... TIME.  Not my idea of fun (although portage is nice).

In the end ... went back to windows for a while, waited a bit and then heard that Ubuntu 7.04 was really nice and had automated many of the things that I found painfull.  And it did.  Installed it on my primary workstation at home -- worked beautifully from the getgo.  Rarely had to fight with the OS to have it DWIM (Do what I mean!).

That's when it hit me ... I think 7.04 was the first time that I admitted that Linux is finally ready for the desktop.  Even my girlfriend was impressed and she's a windows person.

Since then, upgraded it to 7.10 -- all is well.  My ONLY complaint is that very often, _something_ (I don't know what) keeps breaking Java.  You so much as sneeze at the damn thing and Java conks out.

Other than that though, no problem.

Today ... still running windows on some machine, running ubuntu on my primary workstation and xubuntu on my laptop.   Also running ubuntu on a small PC in the basement we use to play our music.

Is Linux ready for the desktop.  Yep.
Would I give a pre-installed UBUNTU machine to my grandma? Yep.

I can get things done with my machines without fight them.  That's what counts.    Will I give up on windows?  No.  It still has its uses.  However I'm stopping at XP.  Vista, not interested.

Server side -- I find Windows 2000 and Windows 2003 very usable and stable.  Never have problems with them. Secret with windows is to only run one thing and it'll be just as stable as Linux (eg: If you're going to run a windows-based mail server ... run just that.  Don't run MS-SQL on it and your FTP server and this and that and the other thing.... you're asking for trouble).

So ... there's room for both I guess. 

But Microsoft went very wrong with Vista.

There's just no excuse.

Even with everything turned off (UAC, Aero, etc ...) -- Vista ate up 600 megs and had a constant CPU "hum" at 5% on a dual core machine.  There's just no excuse.

-- Dingbat.
