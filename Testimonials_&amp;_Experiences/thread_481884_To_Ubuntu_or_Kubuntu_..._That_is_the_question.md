### Post by Geekkit on 2007-06-23
So I figured since I had some spare time (and had backed up all my data), I figured I would give Kubuntu a stab. Since about April I've set up Ubuntu about four or five times now and I wanted to set up Kubuntu just for the experience. I knew that if things didn't work I could easily reinstall Ubuntu back on since installation takes about an hour of my time (compared to Windows where you have to babysit the computer for hours punching in serial numbers and installing all sorts of software).

So, here is my experience and opinion of the comparison of the two. Keep in mind it is solely based on my experience and my opinion and as you know opinions are like belly buttons: everyone has one so mine is no better than yours.


**Installation**
Ubuntu
=====
Ubuntu has a live CD and a GUI with something that is really important: back buttons so you can literally go back to the beginning and change something if you made a mistake. All decisions are high level - no micro managing. Example is "what city do you live in?" which is used to pick your keyboard and locale settings.

Kubuntu
======
Text based installer, no GUI, no live CD, no feedback that anything is happening with a blank screen or a screen that says installing at 6% which it has already been sitting at for the last 10 minutes. No back button if you make a mistake - you'll have to reboot and start over, sorry. The installation requires you to micromanage things such as your keyboard (better know that you're using pc104 not pc105 otherwise Kubuntu doesn't recognize certain keys on your keyboard.


**Initial Desktop**
Ubuntu
=====
Simplified feature reports, updater asks if you want to update, system panels and windows present simple options, easy to figure out what it is that you need to do. If you try to play a media file that you don't have a CODEC for, Totem politely asks you if you want to download and install the appropriate proprietary CODECS at which point it does so and then starts playing the media you started to play. Sound works, network works, installation of printers easy since it seems to know what you have and then ask you if it's correct which it is and for this you smile.
 
Kubuntu
======
Assuming you know the intrinsics of your hardware (i.e., keyboard country code, monitor frequencies, resolutions, buffering abilities), your desktop should have a reasonable resolution ... but most likely stuck at a very low refresh rate which you may or may not be able to adjust based on the driver loaded. If however you failed at any point to specify some trivial piece of information about said video card, your desktop will either be set to 640 x 480 or Kubuntu will belch an error and you'll have to reboot, go into safe/rescue mode and roll back to a default value with your X conf file.

Sound does play, networking is up and running but don't double click on things in the system controls window quickly (it's single click by default although you can change this later), because that will crash the system panel. Printer setup will require you to tell Kubuntu what type of printer is hooked up.

 
**Twiddling Settings**
Ubuntu
=====
Most settings seem to be already the way you'd expect them (double click icons, not single click), but if they aren't, there are always help files, forum posts and little hints along the way. Soon you'll have all the appropriate CODECs you need, your printer will be set up and configured.

Kubuntu
======
If the keyboard mapping is wrong (see above for review), changing these doesn't seem to follow any rhyme or reason. Setting the language *and* setting the  country *and* setting the keyboard that you're using may have been what fixed it, but then again, maybe it was the reboot. No documentation really seems to point you to what it was that actually fixed it.

If apt-get accidentally belches, you'll have to reboot because you don't have privileges to kill the process in the process manager - not enough privileges. The system monitor looks like the war room maps in Star Wars: Empire Strikes Back and really makes no sense as to what is what. There are about 20 buttons that you can poke to change things like the color and where the lines go but it still fails to answer what those different layers are.

Sometimes changing property values for a Konsole updates the current console and sometimes not. It's never really clear why changing the background color this time worked and why last time it didn't. Was it because I also made it bold and that required an update because the update buffer needed to be flushed? Hmmm.



**Summary**
On a last note, I remember reading a bit ago on the interWebs an article about a flame war between Linus Torvalds and Christian F.K. Schaller, a senior GNOME developer had. The article is at: [http://www.desktoplinux.com/news/NS8745257437.html](http://www.desktoplinux.com/news/NS8745257437.html) and is by Steven J. Vaughan-Nichols. Steven summarizes the banter which had Linus not liking GNOME because of its "dumbing down for the masses". He focused on one particular thing that pissed him off which was in GNOME he couldn't reconfigure the right mouse button to do something else. Linus points out that clearly this isn't the only thing that keeps him disliking GNOME.

For me after using both I'd have to say I would/will go every and any day with Ubuntu. Why? Because it works. The installation procedure is completely dirt simple. As for lack of configuration in GNOME, I can have two panes in Nautilus: one for the tree and one for the icons/list, Konquerer? Nada. And in GNOME terminal I can change the font I want. Konsole? Maybe in some config file somewhere. But I have things to do.

### Post by GMU_DodgyHodgy on 2007-07-03
The argument between KDE and Gnome is alot like the argument between Chevy and Ford fans.  

If you like a clean and lighter desktop, Gnome will be your choice. If you want more eye-candy and something a little closer to Windows KDE looks good. 

I wil make one caveat however.  KDE is going through a major makeover right now with KDE 4.  In addition, it seems that there is a little more development being done on some of the core KDE based applications.  

For example, I prefer and use Gnome.  I like Evolution because it combines my email, calendering, to do list, and contacts in a single application.  In addition, I like Rhythmbox/Exaile - because they give me the functionality I want with out too much clutter and bloat.  


However, I have been disappointed in the mutlimedia production applications that are Gnome based.  F-Spot lacks the functionality to prepare phot slideshows with music or add titles to my photos.  Digikam does.  So I end up using that.  Kendlive has proven more flexible and more feature packed than Kino or Avidemux.  So I use that and it is a KDE application.  

If there are more developers working on the KDE based apps - adding quality, stability, and features - that will eventually attract more users to KDE.  In the end - its the applications that will matter most when making a decision between Gnome and KDE.

I will be interested in seeing Gnome's response to the release of KDE 4.
