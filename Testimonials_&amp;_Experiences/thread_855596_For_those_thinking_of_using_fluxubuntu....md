### Post by snowpine on 2008-07-10
> **PHATSPEED7x said:**
> DON'T!!!!! Not user friendly at all. I'm in the process of reinstalling xubuntu on my desktop...

Ah yes, I am glad this topic came up for conversation. I am typing this message using Fluxbuntu. :)

I am a relative "newbie" to Linux (6 months). I went through a period of distro-hopping, tried Fluxbuntu for a little while, then settled on Ubuntu. I couldn't stay away from Fluxbuntu, though, and I recently made it the primary OS on both of my computers. 

I think Fluxbuntu may be right for you if you meet some or all of the following criteria:

[LIST]
[*]Your computer is too slow for Xubuntu (it is ideal for 64-256mb of RAM) OR you have a faster computer and want it to fly.
[*]You like the command line terminal interface (or "console" as Fluxbuntu calls it).
[*]You think that editing text files is an efficient way to configure your menus and system.
[*]You have a half-dozen or so applications that you use every day, and don't want a lot of other distractions.
[*]You like to configure something once and then forget about it. 
[*]You love the green and purple theme (or are willing to learn how to change it).
[*]You primarily use a different windows manager (Gnome, Xfce, KDE, etc) but want something you can boot into occasionally when you need extra speed or simplicity.  
[*]You agree with the "Lightweight, Productive, Agile, Efficient" philosophy described on fluxbuntu.org.
[/LIST]

I also have to acknowledge that there are some really frustrating problems with Fluxbuntu:

[LIST]
[*]It is not an "official" *Buntu release, in fact, it isn't even a release at all--it's a release candidate, and the project hasn't progressed since last year. There's a forum at community.fluxbuntu.org, but it is really quiet over there. The best help for Fluxbuntu right now seems to be right here on these forums. I would love to see Canonical pick up Fluxbuntu as an official product and improve the user experience, fix some bugs, and continue to make it better. Every day I see people on these forums asking "what will run best on my old computer?" so I know the demand is there.
[*]You can't try it as an alternative session within an existing Ubuntu/Kubuntu/Xubuntu install. (Yes, I know you can install Fluxbox, but Fluxbox and Fluxbuntu aren't really the same thing, just like Gnome and Ubuntu aren't the same thing..) I think this holds a lot of people back, because you can't experiment with it as easily. (I would recommend allocating a small partition on your hard drive, 4 gigs is enough, as the best way to try Fluxbuntu.)
[*]The application choices are not to my liking. If you have an ancient computer and need to run kazehakase, nano, rox, etc., that's one thing. I am running Fluxbuntu on computers with 256-512mb ram, so I immediately (and easily) installed firefox, openoffice, thunar, gimp, gedit, etc.
[/LIST]   

Ultimately, the most important reason that I am using Fluxbuntu now? I would like to design my own distro someday, or be involved in a larger project. I know that I still have a lot to learn, but I feel that Fluxbuntu is teaching me some excellent skills toward this end.

That being said, what would you look for in a new release? Is a new release necessary?

Some of the default applications are terrible (unless you have a truly ancient computer), but we all know how to add new apps like Firefox using apt-get or synaptic, right? So that is not really a deal-breaker to me.

Fluxbuntu can be easily upgraded to 8.04 Hardy Heron. All you need to do is edit your /etc/apt/sources.list file (thanks Aisyu!) and change all instances of 'gutsy' to 'hardy'. Then, you can do 'sudo apt-get update' followed by 'sudo apt-get dist-upgrade'. 

Then there are a couple of bugs with easy fixes. If your menus don't auto-update when you add a new application, 'sudo update-menus'. If your USB devices are not auto-mounting, 'mkdir ~/.ivman'. The only one that still has me scratching my head is why most applications are in the "Applications" menu, but a few of them are in "Apps". Does anyone know how to fix this?

I guess my point is, it's a shame the Fluxbuntu project is stalled, but I do think we have a good little community here to keep it up and running, maybe make some noise so that the original developers or somebody like Canonical takes notice.
