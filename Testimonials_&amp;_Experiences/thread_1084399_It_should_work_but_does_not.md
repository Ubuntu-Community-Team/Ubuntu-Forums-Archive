### Post by armandh on 2009-03-02
One gets so use to ubuntu [and other live linux] being a slam dunk install one can go off the rails when it doesn't go well. This falls in to the category of "I should have known better, it has happened before"

The second most exasperating of install problems is [IMHO] no network. [first being no GUI]. Sure enough, expecting some parts from the used bin to "just work" when they were "ok when pulled" might be setting high expectations. Yep, no network on a common as dirt lan card. The leds on the card are flashing just like they do when the card is talking to the lan, so wazzup? I try a different live linux [DSL] no go. I try an old 16 bit ISA card  WOW it works????? Now I'm confused! It dawns on me when I try a different brand of "known to work" PCI nic and it still fails! At last I try a different MOBO slot and BINGO I'm on line. Lead down the garden path by a bad slot which I suspect once held a modem during a thunder storm. DUH!

I wonder how many of the really great "going back to M$" rants are hardware problems, and when there is no better results go in the trash. I fixed the MOBO with a blue sharpie, liberally marking the bad slot.

Actually 2 bad slots as I found when the sound would not work either.
