### Post by 3rdalbum on 2008-07-29
3. I used to look for programs on the web to install... but then I realised that everything I want and need is in the package manager. If it isn't, I look for a .deb.

Compiling isn't difficult if you know how to do it, but it's something completely alien to Windows users so it seems incredibly complex.

a. Use Synaptic to install the "build-essential" package, and for the next step install the "nautilus-open-terminal" package and log out and log in again.
b. Open a terminal window inside the source code's directory, and type ./configure
c. If the script complains that you're missing a library, go to Synaptic and install the appropriate package (one with "-dev" at the end of the name).
d. Repeat b and c until the configure script finishes successfully.
e. Run the command "make", and then "sudo make install". The program is now installed.

Compiling is necessary because Linux runs on so many different incompatible processors; source code is the only form a program can be distributed in that can work across all those different types of processors. Windows only runs on x86 processors, so software developers can distribute x86 binaries for Windows software and not miss anyone out.

4. I gather you're trying to find the program executables so you can add them to the Applications menu? That's not necessary - just find the name of the executable and type that into the "command" field when you're adding the program to the menu. If you need the location for some reason, look at the link in my signature of "Where did my program go on Ubuntu".

5. In the terminal: wget <address of file>

Very easy, reliable download manager. But I don't do lots of downloading, so I can't give suggestions for full-featured GUI programs.

6. Right-click a file and go to the Open With submenu; you can choose a different program to use there. If you go to Properties instead, you can choose the default program to open files of that type with.

7. If files and folders are locked, then they are parts of the system you should be careful of dealing with. You will need to open a file browser as "root" - do this by first installing the "nautilus-gksu" package, log out and in again, and then you can right-click a file or folder and choose "Open as Administrator".

### Post by tiachopvutru on 2008-07-29
Installing **ubuntu-restricted-extras**, **libdvdcss2**, and **w32codecs** (or **w64codecs** if you installed the 64-bits version) will probably fit all your codecs need. The last 2/3 require Medibuntu repositories, so make sure to add them. The instruction is [here]("https://help.ubuntu.com/community/Medibuntu").

For video player, I suggest smplayer (which is the frontend of mplayer). For audio players, beside Audacious, XMMS, Amarok, you can also try Exaile.

Also, some files that you compile may already be package into .deb files by someone else, so I suggest checking [http://getdeb.net/]("http://getdeb.net/"). All you have to do is open the deb file and clicking the "Install" button, and it will be installed.

If you are confused about "Linux directory structure," google that. Here's the first result: [http://www.tuxfiles.org/linuxhelp/linuxdir.html]("http://www.tuxfiles.org/linuxhelp/linuxdir.html")

Also, search inside Synaptic Package Manager for "nautilus" to get some plugins for something like, if you wonder why when you right-click a picture and there isn't an option to set it as background, there's a plugin inside Synaptic to do that.
