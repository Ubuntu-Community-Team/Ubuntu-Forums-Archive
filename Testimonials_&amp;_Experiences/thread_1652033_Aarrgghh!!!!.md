### Post by jgperez on 2010-12-24
Let me preface what I'm about to relate by saying I'm not an inexperienced computer user. I first went on personal computers more than 3 decades ago. I've used Apple IIs, Amigas, Trash 80's --the original laptops-- even done  a little recreational programming (basic back in the 80's) and been involved with software localization and documentation. And, oh yeah, I was the sci-tech producer for one of the CNN networks for a decade.

But ... how should I put this gently??? ... I have never seen such a viciously, relentlessly user-hostile program as Mythbuntu and such worthless documentation as is offered online for MythTV and Mythbuntu. 

I am on my FIFTH attempted installation of Mythbuntu/MythTV. I started out with Mythbuntu. Did two clean installs of that, tried to configure it, could not get it to work. Then tried the other way, (because, IMHO, the Mythbuntu distro's desktop is a POS -- why not use the more polished/finished KDE/Gnome desktops? I guess because that would be the EASY way). 

Did two installs of that, got one to where I could see (but not hear) a picture, when I went to fix the audio the next day, Mythbuntu would quit saying the backend wasn't running, and when I started the backend manually IT would quit saying something that seemed to be saying it couldn't find the front end.

So went back to Windows (dual boot machine) deleted the Ubuntu partitions, rejoined the disk space to the Windows partition, did a fresh new install of mythbuntu, using everything I've learned. Like updating the graphics driver to a proprietary Nvidia one, updating all the packages. And discovering that the setup that is wanted is NOT the setup that is in the program that covers your entire screen and has as one of the five or seven menu choices "setup," but an ENTIRELY different program. Then you have to discover that you have to get rid of the program that is running by ... clicking exit, or quit, or get lost? NO WAY! There is no quit menu choice. Somehow I lucked into figuring out that escape brings up an exit dialogue. 

So I configured the backend, again, reboot and ...

And all that to get to the following: "Error: MythTV is using all inputs, but there are no active recordings?"

I mean WTF?! WTF is an "input"?  And how could there BE any active recordings, I haven't used the MythTV for the first time yet! And why is there a question mark at the end of the sentence? Is the question mark meant to express astonishment, ask me whether it is really true there are no active recordings, or was it just an-end-of-the-sentence character, like a period or an exclamation mark or a dash or some other dingbat, selected at random by some subroutine in the program that a perverse programmer put in there?

So I go to the best documentation I can find and see it is a couple of releases old, was being updated for 10.04 release. Worse, it is missing ALL the screen shots, is so totally incomplete that I don't know if the configuration question before me is a new one or such an insignificant one the people who worked on the wiki just decided to skip it.

As I mentioned, I've worked on software localization and documentation. And I've seen this problem. The über-geeks are running the show and it suddenly occurs to them is would be REALLY COOL if the program did some totally random thing like brush your teeth or send your girlfriend flowers for Valentine's Day.

So they put it in there, even though it makes the program drop dead unless you know to press alt-mega-cokebottle so that you can write in the wave function for the electrons in the outermost shell of the einsteinium atom. And if you do it in classical greek then the program will run, otherwise it will give you error messages composed of 3-4 random computer error message strings from the good old DOS days put together into a sentence. 

The solution is simple. Freeze development of the program. NO new features, none AT ALL, until installation is, say, no more complicated than using early 80's word processing programs on text interfaces. IDEALLY, all features except tuning in, recording, and playing back would be STRIPPED OUT of the program until the effing thing was USABLE by normal geeks even if they don't have two or three PhD's.

But that's unrealistic. So work on a) Having the program PRESENT to the user as ONE program. IF Mythbuntu/MythTV is going to get even 1% or 2% market share, that would mean 99%+ of the users are using it on a single box. 

GET RID of everything about client-server architecture and everything else like that from what a new user sees in a fresh install and in its documentation. DO NOT ASK whether a database should be filled -- let the program figure out whether or not to fill it. IF THE DEVELOPER cannot write a simple, rules-based subroutine to decide whether to fill the database or not, how in the hell is a totally new user, who doesn't know what the database is for, what fields it has, why they are there, where the data is coming from and where it is going to, how is the USER supposed to know the answer?

UNLESS you guys are being paid off by Microsoft to DRIVE PEOPLE AWAY from free software, MAKING Mythbuntu/MythTV usable for (in a first stage) at least advanced computer users/enthusiasts should be the overriding, everything else be damned priority. 

Why is the documentation wiki so incredibly BAD??? Because there are no users, or to be more precise, no users like me, people who are NOT primarily computer specialists of one type or another, but enthusiasts who happen to be professional writers, producers, content providers. 

IF someone can tell me how to configure Mythbuntu to play and record over-the-air ATSC TV channels using a P4, 3.2Mhz computer with 4 gigs of RAM, an 80GB and a 2TB hard drives, a good NVidia PCI express video card, normal 2-channel audio, and a Hauppage 950Q USB stick tuner, all playing out through the VGA port to a 1920X1024p 42" TV, then I will work on fixing the documentation to THIS version of MythTV. If not, well, I've got a half dozen WORKING computers at home, most dual boot with Win7 ultimate.  Media Center sucks, but not nearly as much as MythTV seems to.
