### Post by Tamlynmac on 2009-03-18
> hyper_ch

Excellent question. 
 
So many new users are conditioned to either no choice or the requirement of purchasing new applications. In Ubuntu the choices are vast and majority of which are free. I have noticed that many new users struggle with the volume of available applications, and deciding which are appropriate for their use. When conditioned to accept only the status quo, it's been my experience new users quickly become overwhelmed with the number of choices in Ubuntu. I can't count how many posts I've read that have asked for assistance in selecting an application. They usually receive quite a few response, many of which differ in opinion. Many respond by advising the user to try numerous applications from the various postings and decide which one best fits their requirements. Selection of which application best meets the users expectations and requirements, should only be performed by the individual user after reviewing the available options.

Let us not bloat the default package just to acquiesce and endeavor to attract Windows users. IMHO when deciding to change OS's, new users should investigate and not just expect a Windows clone. Knowledge of their new OS must be acquired and they need to understand the changes. Creating a bloated installation package simply to eliminate this process is not in the best interest of either the new users or Ubuntu. In my opinion, a significant part of the Ubuntu experience is the customization and personalization of the OS and available applications. We must remember, Ubuntu is not Windows and new users need to learn how to use it, as they did Windows. Learning is a fundamental necessity to change and this experience improves new user comprehension of the OS.

I do not believe it's in anyones best interest to dumb down users any more than Windows already has. But rather enlighten users and involve them in the community, a truly unique experience for many. I know it was for me.

Just my $0.02

### Post by ve4cib on 2009-03-18
Personally I'd like to see a little bit less integration with the Gnome/KDE applications.  I use Fluxbox.  I don't *want* a whole lot of extra goings-on behind the scenes so that the few KDE and Gnome apps I happen to use can integrate themselves into a desktop environment I don't want.

Tying core parts of the DE together make sense; the example of being able to drag a file from File-Roller to a file browser (any file browser -- be it Dolphin, Nautilus, Thunar, PCManFM, EmelFM, or Konqueror) is intuitive and makes sense.  But that's (to my way of thinking) less about "desktop environment integration" and more about "making applications that do one thing, do that one thing well, and get along with each other when needed."

As for going with a clone if Apple's iLife (why must the prepend everything with that blasted "i"?!) mindset... huge mistake in my opinion.  And it would never last.  I wouldn't be surprised if way back in the dark and distant past of early Gnome/KDE development there were certain programs that would only work within one or the other.  But it's the nature of open source software to be ported to other environments.

Imagine, for example, that Gnome had an uber-awesome video editor that was brilliant and everyone liked using.  And this program could only work if you're using Gnome.  You can bet that some KDE user would hear about this app and want to use it.  If it didn't work on KDE he (or she) would get the source, find the parts that make it not work with KDE, recompile it and start using it.  *presto* iLife clone gone.

In the end if an open-source application, library, or tool is good and does what it needs to do well people will always port it to as many different platforms as possible.  Look at GCC and G++ for example.  Or VLC.  Or Firefox.  As long as it's open-source it will never remain "Ubuntu-only."
