### Post by ProDigit on 2009-11-04
I'm sorry to give some negative feedback...
As much as I love Ubuntu and Xubuntu, I've installed Xubuntu 9.10 and have noticed we're heading to making Xubuntu like a next Vista...

I mean, I installed Xubuntu, because the layout is simple, the OS is light, basic, it uses very few resources, and has all that I need to run an Os at acceptable speeds on most of my 5 years old devices.
But with the newer Xubuntu, they adopted the Ubuntu mentality a bit, in making effort to make Xubuntu more appealing, at the cost of performance.
The startup screen with useless (though beautiful) dots flying around wasting booting time, using resources and CPU; the layout (theme) is very nice of the newer Xubuntu; but my only concern here is, this new layout, does it require more system resources?
Xubuntu is not the only one who upgrades, Firefox too, with it's version 3.5 taking nearly twice the ram that version 2 took...

Perhaps you can see this as a small complaint; because not all these 'upgrades' are necessary... Java now has more nice buttons, instead of the plain grey ones they switched to XP like buttons. On a normal PC you won't notice the speed decrease, but on a Pentium 166 you still do!
And if on a pentium 166 you still do, then it must be one of the hundreds of things that slow down the OS on a regular computer.
Testing an OS on a P166 is a good way to test it's responsiveness,and see where an OS is being slowed down on the processor part.

Xubuntu would be an excellent OS, if it further focused on minimizing background disk activity, reducing all unnecessary graphical treats, and improve battery efficiency for Notebooks and laptops.
I think the Os itself can be pruned and trimmed down at least by 25% in size, by removing all extra's and unnecessary things; and instead offer these extra's to be selected in a menu before (or after) installation to be installed.

So, in short, I am less content about Xubuntu 9.10!

How many would agree that if I wanted a more flashy OS, I'd go for Ubuntu or Kubuntu... I loved Xubuntu for it's simplicity,and wanted to bring this to the developers attention, to perhaps keep themselves distinct from Ubuntu and Kubuntu, and keep the treats for a hogged down version of an OS like Vista.

Many would say I'd better go with DSL, or another lightweight OS, but I use Xubuntu because I want to stay in the Ubuntu family, with the ease of updating packages that work on all 3 OS versions!

I am less familiar with other Linux operating systems that often have a very small packages content,and use different programs to perform certain tasks. And since I'm not fond on learning to know all Linux apps and OSes out there, I wanted to stay with Xubuntu.

### Post by wrgb2 on 2009-11-04
I sympathize with your plight, but since Xubuntu 8.04, the minimum system requirements for Xubuntu have been a 300MHZ processor (that probably means a PIII or maybe a PII).  I'm running Xubuntu 9.10 on a Pentium III 650MHZ machine with 384MB of ram and it's snappy.  You can get one of those on Ebay for almost the price of shipping, so if you can at all afford it, you might want to upgrade your hardware just a bit.  BTW, I also tried installing Xubuntu 9.10 on a Celeron 1100MHz laptop with 256MB memory and it _is_ slow -- much slower than the PIII 650MHZ, so I think you need at least 384MB ram.

But I do agree with you about the boot process, it was faster in 9.04 and the new user chooser is just an unnecessary step.  And I think the desktop is a litte slower, too.

### Post by running_rabbit07 on 2009-11-04
Install startupmanager via Synaptic Package Manager or ```
sudo apt-get install startupmanager
``` and open it, turn off splash.

Install epiphany web browser via Synaptic or ```
sudo apt-get install epiphany-web-browser
```

These two changes should make a world of difference for you.

There are other distros made to be even lighter, such as, DSL (Damn Small Linux), Puppy Linux, or a few others.

### Post by ProDigit on 2009-11-05
I'm not running Xubuntu on a P166, but it is an excellent reference system to see how an operating system is running, and yes, we've managed to run Windows 2000 on a Pentium 100Mhz, with 256MB RAM I believe (though painstakingly slow;I forgot the exact amount RAM because that was about 8 years ago).

I am aware that people are able to purchase faster PC's, they sell P1Ghz + Mobo + Case + 300W PSU for $50 in a store around the corner.

I'm actually running Xubuntu from a Laptop Core2Duo 1,6Ghz, and want to shave off as much CPU usage, and footprint as possible, with still having the basics of the GUI functioning. I just had hoped Xubuntu would not step into the boat of bloated stuffed with eyecandy OSses.

All I wanted to do was send out a warning signal to the people who create this OS, not to end up like the other two! (Or like Windows for that manner);and now is a good time to tell them (better than when it'd be too late).

Also, Xubuntu might turn out to be an excellent OS to be run from netbooks and mids!
Netbooks run an Atom processor equivalent to a +1Ghz P3, or the older models go with a Mobile Celeron P3 CPU @ 630Mhz.

Mids often go with 300-600Mhz ARM/freescale processors, but there might be a future for i86 processors at that speed!

And for those who are not fond of Ubuntu Netbook remix, they might want to switch to an Xfce interface, having the looks of a modern OS with all programs installed, but still not lagging like the other OSes.

As far as the other versions of Ubuntu are concerned, they did spend a lot in the looks of them! There's the 3D Cube desktop, the special effects of Gnome, the nice icons (which use some cache) of KDE...
I would prefer a Windows 98/2000 approach in to minimize the footprint by reducing caching of icons (or disable some Icons, and force the use of shared icons, same icon for many file extensions), and the disabling or extremely optimizing themes on the OS.

[I]Perhaps it would be nice to convert configuration files in text format to slightly compressed files,with commands and characters that the OS understands better, and runs faster with; to reduce the total OS size.
Then, if a program could open those files and reconvert them to the screen as a regular text based configuration file, you'd win a lot of performance and diskspace. I know it does not exist yet in the Linux community, other than encryption or compression in zip/rar/bz2/... formats; or perhaps python or C# can do this, but they also are so complex to use.
It's much easier to have a 'mousepad' like interface (regular text editor) that can open compressed formats, optimized for use with a Linux kernel.[/I]

Like said before, this post is nothing more but a ring on an alarm bell to keep certain priorities, to perhaps make Xubuntu the fastest and lightest Ubuntu variant available!

Apple did something like this with their .plists (preferences files for OS X programs.) It used to be they were XML documents that could be edited in anything. For some reason they decided to convert them in to binary data and now you have to run them through a converter program to edit them, or use a program that can natively read the format.

Not being able to edit these files without a special program is a massive PITA from a tech support standpoint. The great thing about text configuration files is that they can be edited anywhere, by any program and any operating system, even if you have to rip the HD out of a non-booting computer.

I'm all for ecking out every ounce of performance, but not at the expense of elegance and simplicity.

Besides, config files generally get read once at runtime so I doubt you'd see much improvement in performance.

Still, if this were to happen, I would tend to think it'd have to be implemented at the kernel level, so that the binary files appear to be text no matter what program is reading them.
