### Post by Hagbard on 2006-11-14
I'm a long-time Mac user who has recently started using Linux thanks to the miracle of virtualization. I'm running Ubuntu 6.10 in a Parallels Desktop virtual machine, and it works beautifully! I never would have believed this before I tried it, but my system performs just as well within the linux VM as it does in OS X. Everything is fast and responsive, and my internet, sound, and printing all work fine. My main setup is GNOME, but I have separate KDE and XFCE virtual machines and setup and installation has been easy and successful for each of them. 

In many ways, Ubuntu is even easier to use than OS X, while providing more flexibility. The incredible Synaptic Package Manager makes it simple to customize my system to do exactly what I need. Once you start dealing with more advanced tasks, the tighter integration between the GUI and the base OS within Ubuntu really starts to shine. It's easier to get started with the shell in Ubuntu, because Apple's efforts to conceal the gears and cogs of Unix/Darwin from the standard user makes it harder to figure out exactly how system resources are being used by standard GUI applications.

Ubuntu and its applications also clearly use system resources efficiently. The 512 MB of RAM in virtual machine is able to support a large number of simultaneous applications and documents without the need for HD swapping. My Ogg Vorbis net radio stream never skips while I'm loading other web pages. In short, even in virtualized form, Ubuntu seems to equal the performance of the OS that is considered the leader in usability. 

The only area in which OS X is superior is the finest level of graphical polish on the GUI. Ubuntu is clear, well-designed and fonts are rendered very crisply. OS X, however, is just a little bit better in all of these categories and the overall presentation gels into something sublime. However, I'm sure that Ubuntu users with the correct hardware who put some effort into customizing their system and fonts could achieve equal results! Given Ubuntu's goal of having a lightweight and flexible core installation that can be used on as many computers as possible, I think the developers made the right choice in going for performance rather than glossy polish.

The strong community and wealth of helpful information online also contribute to the postive experience of Ubuntu. It's been very easy to google or link-hunt for anything I've needed to know, and the built-in documentation features are great. To summarize, I have found Ubuntu to be a high quality productive environment to work in, with no weaknesses and many strengths. It is just as functional for me as Mac OS X.

All of the above comparisons do NOT take price or ideology into account, they are purely performance-based. Once we take into account the fact that Ubuntu is completely FREE, and is based on a philosophy of trying to provide high-quality tools to help all human beings work together better, the true winner in the comparison becomes apparent.

It should be noted that none of this should be taken as negative comments toward OS X, or the Mac as a hardware platform, or Apple as a company. My Intel iMac is a beautiful and powerful platform, and I also think OS X is an excellent operating system. In fact, the excellent performance of Linux as a virtual machine inside OS X shows that OS X really is incredibly powerful as well! Being able to run Ubuntu and OS X side by side with full functionality in each gives me the most far-reaching and flexible set of tools that I have ever had access to at a computer. If I had to choose just one OS, I'd choose Ubuntu, on Apple's hardware platform.

### Post by Aurora on 2007-01-25
> **Hagbard said:**
> I'm a long-time Mac user who has recently started using Linux...It is just as functional for me as Mac OS X.

Wow, I'm genuinely surprised by this statement. 
Try this: go to [www.wholenote.com](www.wholenote.com), and try to get the MIDI guitar lessons working in Linux of any type.  If you can, please tell me what you did.  I've been completely stumped.

I've been using Ubuntu since Warty, and I just recently got a MacBook.  I got it under duress, as I'm involved in a very demanding audio production course that requires the use of a portable hard drive in HFS+ format, and a computer running ProTools audio software. In other words, buy a Mac or flunk.  I was very reluctant to spend the money, as I already have a desktop running three flavors of Linux.

Now I find myself using the desktop less and less, and using the MacBook instead.  No longer do I have to spend many hours a week trying to fix my computer, I just turn it on and it does what I need it to do.  I don't have to endlessly search and post on forums in order to do ordinary, everyday tasks such as saving to an external drive, which has been an ongoing project for the last several months on my Linux box. (Almost there; I've only been working at it since September).  I just plug my drive into my Mac and it works.  What I don't like about the Mac is Apple: they are not going to let me upgrade to the next OS version for free simply by typing sudo aptitude dist upgrade, or using a nice graphical package manager like Synaptic.  If you want to run Ubuntu on Mac hardware, you still have to pay for the Mac OS.  If I want a nice media player from Apple, I'm going to be restricted by DRM, be tied to their online distribution system, and have a limited choice of file formats.  Linux is freedom; that I miss in the Mac world.

But, from a function standpoint, I really can't say that Ubuntu works as well or as easily.  I am doing stuff with audio now on the Mac that I had not been able to do on the Linux box. I built the thing specifically for audio production, installed 64 Studio, an audio-optimized distro, and still can't get anything recorded.  I can't get Audacity to talk to Jack, I can't get Ardour to do much of anything, because the interface is so unfriendly it's hard to learn and get around on it.  Pro Tools is a pain, but still much easier than open source audio solutions.

I miss Dapper though, maybe I'll try to dual-boot or use Parallels.  I would love to have it working on the MacBook.  Please post further about how that works.  Parallels doesn't even mention Linux on their website that I can see, so I was peasantly surprised to see that someone was running Ubuntu on it.
