The problem is that Real Player is a proprietary app.  Any time a new version of Ubuntu comes out (every 6 months) the app may depend on some older version that isn't in the new release, and has to be fixed by the developer before it will work in the new release.

Also, this problem exists extensively on Windows platforms.  Just look at how much stuff runs on XP but not on Vista.  You may not have noticed it as much, as Windows has a much longer release cycle (what was it, about 6 years between XP and Vista?).

Ubuntu does provide a long term release cycle (LTS) for some of it's releases.  These releases are supported on the desktop for 3 years.  The latest release is 8.04 (the current release), but 6.06 will still be supported on the desktop in 2009.

Having proprietary software break on you from time to time is part of running a bleeding-edge OS.

In any case, I'm pretty sure the universe/multiverse repos have gstreamer codecs that will play .rm files.  Search for "gstreamer" in add/remove software, or in synaptic.  With the proper codecs installed you should be able to play your .rm files in totem.

### Post by decoherence on 2008-05-21
"Why has nobody replied to the key question ie why will Hardy not run a bin file? It seems totally bizarre behaviour to me."

Well, I'm just gonna bite the bullet and answer the question; why not?

But first, here's a question: Why **would** Hardy run a .bin file?

An .exe file is a Windows executable binary. It is a file that has been designed to run as a program under Windows.

A .bin is a file that is binary data. It may or may not be a program. If it is a program, it may or may not execute properly under Ubuntu, depending on how talented the people who packaged it were. Maybe it's meant for RedHat? Or OS/2, for that matter?

Furthermore (because you asked for it,) filename extensions such as .bin and .exe do not serve the same role in Linux that they do in Windows. In Linux and all UNIX-like systems there are only two different kinds of real file formats (excluding sockets, character special devices, magic numbers and other esoterica no ubuntu user needs to care about.) Those formats are text and binary. Everything beyond that, such as filename extensions, is implemented at the application level (generally GNOME/KDE or any other program whose job it is to support many different formats such as GIMP or Kino.)

Now, you seemed to be having some sort of difficulty earlier on in the thread having to do with RealPlayer. I don't use RealPlayer because of previous bad experiences with their software and because there are better alternatives. However, my suggestion would be that if you want RealPlayer that "just works" the way a Windows .exe does, you need to find it packaged for Ubuntu (just like .exe files are packaged for Windows.) Ubuntu installer packages typically end with a .deb extension. If you can't find such a package, let Real know you want it, then ask somebody how to install it in the mean time.

I hope this has been helpful in allowing you to participate in meaningful two-way discourse!
