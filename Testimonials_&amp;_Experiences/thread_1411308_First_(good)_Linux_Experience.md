### Post by Letrazzrot on 2010-02-19
I am writing this as feedback, in case it can be used in some way to further develop Ubuntu (or Linux in general).

First off, I consider myself an experienced computer user.  I've used a myriad of operating systems over the years, and done my share of meddling with programming/development.  I am usually the guy that friends/coworkers go to for help when Windows starts behaving badly.

The last time I tried any sort of Linux was almost ten years ago (I *think* it was Red Hat, but not sure), and it left me with sour memories.  It was difficult to get working correctly, and lacked any real motivation to use it, except for specialized tasks.  I never much though about Linux again, until recently.

In preparation of building a home file server project, I've decided to trial Linux as a possible OS for it.    Deciding on Ubuntu (for no other reason than it seemed "popular") for my first trial, I installed it dual boot alongside WindowsXP without any trouble whatsoever.

To make a long story shorter, I was hooked.  This was very unlike my previous experience (which was to be expected, in hindsight).  It was easy to set up, and as far as OS's go, fairly easy to get the hang of.  What started out as a server OS trial ended up as my daily OS, both for my desktop, and now also for my Dell laptop.

I should say that, while I am not scared to delve into shells and config files, I'd just as soon not if I don't have to.  I was surprised at the amount of tasks that could be accomplished with mouse clicks and GUI.  I also appreciate how straightforward the OS seems to built -- the /proc directory, for example, was an extremely satisfying discovery from someone like me who has been mucking around Windows for a decade.

I could go on about the accolades of Ubuntu (or Linux in general), but I have a feeling that I'd be preaching to the choir, so to speak, on this forum.  However, I will try to balance this with some criticisms, although they are minor.

I've noticed missing documentation when hitting some of those "Help" buttons on dialog forms.  This doesn't seem too widespread, and typically a small amount of poking around on the internet or the main help files nets me an answer, anyway.  But for a less tech savy person, it may be a source of frustration.  The various dialogs are usually somewhat self-explanatory, but context sensitive help just doesn't seem to be nearly as fleshed out as it could be.

Also, it would be nice to have some way to "unpackage" something.  For example, after installing KDE for a test drive, it doesn't seem possible to automagically remove that package and everything else it installed.  Instead, I had to manually remove packages separately.

Another thing that would have been nice during the initial learning and configuration of the desktop, is some sort of "Configuration Theme."  A "config theme" could set menus, various gui enhancements, panels and their placement, etc.  This would give some examples to work from, in setting up that perfect user environment for me.  Perhaps it could also be used to save/backup/snapshot the look and feel of a desktop, eliminating any fear of experimentation.  If nothing like this exists (or is planned), I wonder how difficult it would be to develop?  Perhaps I should have a look over at Mono...

So far, after about a week of using, that pretty much sums it up.  The real test, however, will be when my server hardware arrives, and it is time to set that up.  Perhaps I'll share that particular experience, as well.

p.s. Hello everyone, and sorry for the length...
