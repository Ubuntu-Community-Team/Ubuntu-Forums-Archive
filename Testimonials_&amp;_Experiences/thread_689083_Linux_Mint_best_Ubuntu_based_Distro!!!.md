### Post by themacmeister on 2008-02-06
I have been using (and improving) Mint Linux 4.0 for about 2 months now.

In my opinion, it is the finest distro I have used so far (and I have tried heaps). The only two other distros to work with my Dell Optiplex out of the box were:

Mandrake 9.0/9.1
Fedora Core 3

The only thing not working was my NT1 ISDN Modem. This was fixed with a kernel recompile (which I became very good at).

It is probably more a case of Xorg improving, and also the Nvidia linux driver. I can now log-off and force-quit Xorg, and it returns correctly to the login screen (the first distro I have used with both NVidia/ATI that has done so correctly).

There is no workable method (and never has been) of Sleeping/Hiibernating the machine, although the DPMS monitor power saving works well.

Sound/Video/3D etc etc all working out of the box. Apt-get install works well.

### Post by FrozenFox on 2008-02-10
Though Mint is compatible with gutsy stuff pretty much every time that I've found, it is not based on Gutsy itself according to the creator IIRC. From what I understand, it was originally made based on an older version of ubuntu (hoary?), then updated alongside ubuntu with most of the same packages but was (for some) much more stable. The most recent version was made from Feisty then updated to include most of the stuff from Gutsy. Somehow, again, significantly more stable for some. 

Mint uses grub gfxboot by default, and you can easily change this by moving a gfxboot .message file into the right folder and editing one line of your grub menu.lst. 

See their forums for more accurate info on all of the above comments ;P

Mint nor Ubuntu gutsy or hardy have ever randomly crashed for me though, so I couldn't say much on that issue.
