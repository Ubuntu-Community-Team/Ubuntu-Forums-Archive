### Post by Adamaeus on 2007-08-22
I installed Ubuntu on my aunt's old computer, mainly out of curiosity. I'd tried it on my laptop via LiveCD, but didn't think it let me do enough. So, I popped the disc into an old Pavilion and let it set itself up. 

And you know what? Even though this ancient dusty box is in desperate need of an overhaul, (a massive TWENTY GIGABYTES of hard drive space, 750 megahertz Intel processor and a whopping 128 MB RAM), Ubuntu still runs fine. Not exactly a cheetah, but it's steady. It runs smoothly. That is so much more than I can say for my Vista-run laptop.

I haven't fully tested Ubuntu's capacity, not by a long road. I have experimented with it and I find that it can replicate everything that I need it to do. Internet? Got it covered. Writing? It's got OpenOffice for that. Pictures? The GIMP ain't as gimpy as its name implies. Flash? Doesn't work as fast as I'd like it to, but I suspect this is more of a hardware issue than anything else.

I've decided that I'm going to assemble a computer from scratch and install Ubuntu on it. I am in love with this operating system. I have had it running for the full seven days, and it hasn't stopped to reboot, run out of memory, crashed unexpectedly...it just WORKS. It's so beautiful and good to be able to use something that does what you tell it to, when you need it do. 

This is not to say I haven't encountered a few bugs. For some reason I can't install VLC, Beagle or a number of other pieces of software--again, this is most likely (from what little my newbie brain can comprehend) an issue of hardware.

What tells me that Ubuntu is for me is this: I am using it on ancient hardware, and prefer it to Vista, which runs, if erratically, then speedily, on my laptop. I just need one Windows machine. 

Which brings me to a question, actually: My laptop has two hard drives in it. One is the operative drive (designated ACER), and the other is a drive marked DATA, whose purpose ought to be obvious. 

Both drives are 30 gigs. But the Vista/ACER drive is where all the work takes place. So--how do I install Ubuntu on my secondary drive and boot it up? I am rather new to this kind of thing, so, if I'm posting in the wrong place, direct me where I ought to go.

Thank you very much. And Ubuntu team? Thank you for making Linux easy for everyone to use. I am VERY confident that this is the OS form me. :D

### Post by arvevans on 2007-08-23
[INDENT]*Both drives are 30 gigs. But the Vista/ACER drive is where all the work takes place. So--how do I install Ubuntu on my secondary drive and boot it up? I am rather new to this kind of thing, so, if I'm posting in the wrong place, direct me where I ought to go.*[/INDENT]

With Windows installed first, when you install Linux it will probably recognize that you have Windows on the other drive, and put a stanza in the "grub" boot loader that will ask you at boot time if you want to run Linux or Windows.  

This gets a bit tricky though  because your Linux will be on the second hard drive, from what you are saying, and you want the boot track on the first hard drive to be written with the dual-boot pointer.  For this you will need to  be absolutely which drive is hda and which is hdb, and tell the install process to put the boot on the proper drive.

Good luck, and enjoy your Linux experience.
