### Post by oldsoundguy on 2008-11-08
This Friday nite (07 Nov) I proceeded to update my three Gutsy desktop machines and decided to go all the way to Intrepid.
First machine has been having some issues either with a memory chip or the hard drive and testing tells me nothing, so I went for a new install,
I can not BELIEVE how easy it was .. did not have to install ANYTHING hardware .. just updated the repositories, added my plug-ins to Firefox, and installed a couple of "got to have them now" programs.
Took all of about 2 hours including the updates and the program install!
NICE
computer number two was done via on line updating to Hardy and then to Intrepid .. NO ISSUES WHATSOEVER other than MAN, does it take a long time to do it on line (even with high speed cable) .. way too long IMO, but it works.
so, NICE
computer number three is my MAIN computer and the one I am using right now.
Same procedure as number two. 
Then the problem .. the box uses an old FX5500 NVidia card PCI (as there is no AGP of PCIe slot .. only way I could go)
This machine worked fine on Gutsy driving an HP LP2465 Cinema display and even ran the rotation (turns to portrait display)
HOWEVER, Intrepid and that card and it's drivers do not play well together .. have the 173 driver installed as per all of the instructions all over these forums to NO AVAIL.  Have come to the conclusion that it IS a bug but do not know how to do a work around (if there even is one).
Right now I have a standard LCD on it and it is working on that screen, so I am not dead in the water .. but sure would like a fix for the problem.
so, NOT SO NICE.
