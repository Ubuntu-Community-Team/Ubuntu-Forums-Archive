### Post by hoppipolla on 2009-09-07
Yeah just thought I'd bung these down, some are genuine problems, others are tiny things that I think would make huge improvements to the user experience!

Overall though I have just loved my Ubuntu experience this time round - it truly is a great and very friendly OS :)

Ok here we go :)


**1. Some applications I feel are strange choices, or are missing.**
- I have found I much prefer Banshee to Rhythmbox - is there a reason why Rhythmbox is the chosen app?
- Parcellite would be a cool addition by default, as would Firestarter I think :)
- It is a bit strange to have to wait until Karmic to get Firefox updated automatically to 3.5. I do have Shiretoko but it would be cool to have it auto-update when it's an app this important!
- What's this I hear about switching to Empathy over Pidgin? Isn't Empathy a bit simple? o.O

**2. Sound, webcam and wireless still a bit bumpy.**
- Wireless... I've had very little out of my card, but then again it is a rubbish card even under Windows, so I can let this one go lol
- Sound is very up and down - Pulseaudio seems awful! I even bought a Soundblaster to try to make my set-up as simple as possible and it still didn't work! So my sound is usually fine but sometimes goes completely wrong ._.
- Webcam I can only get to work on ebuddy on the internet ._.  It would be cool to have better Webcam for future :)

**3. Flash support is excellent but only when switched to the official flash plugin** as opposed to the standard swfdec plugin. Maybe we should give people the option, otherwise every Ubuntu install by default has a poor flash plugin.

**4. Themes a bit limited, and software removals require dropping to Synaptic sometimes.** I think both of these will be enhanced in Karmic though :)

**5. Tiny problem with desktop effects** - when I turn them on I can't select the Gnome menus by clicking the extremities of the screen, I have to basically click the text itself.. does anyone else notice this? I know it sounds tiny but of course when you're trying to get lots of things done it does slow you down a bit. It was actually the reason I turned desktop effects off ._.

**6. **Oh TOTALLY random - why don't messages in Pidgin cause the taskbar to flash? lol :)


I think that's about it! It's all like, little niggles really! For the most part a lot of my Ubuntu experience has actually been smoother and more friendly than on Windows! If you explained that to me about 5/6 years ago when I first started using Linux OSs I would have been shocked! :)


So yeah, well done Ubuntu guys and I'm sure these problems will gradually get ironed out with future releases - I can't wait for Karmic Koala!

Peace out!

Hoppi :)

### Post by Tibuda on 2009-09-09
**About Banshee:** Forget this Mono flamewar, use the application you like. I agree with you, Banshee is great. They have considered using it by default in karmic, but it missed some important features. Perhaps in future releases it will be the default music player.

**About swfdec:** There's no flash player installed by default in Ubuntu (as in Windows). The first time you open a flash-powered website (tested in jaunty and karmic), Firefox asks you which plugin you wish to install: swfdec, adobe or gnash. You probably just clicked "Ok" in that dialog without reading the selected option.

**About Firefox:** You can:
1. Use Shiretoko. It is Firefox 3.5 with a different name. You may have problems with some sites that don't like the name. Theres a workaround somewhere.
2. Use the version downloaded from mozilla. It is just a compressed file. You uncompress it to anywhere in your home folder, and double-click the firefox executable. You can create a launcher in your menu and set it as your default browser. It will be able to update itself.

### Post by ddrichardson on 2009-09-09
> **directhex said:**
> It's a development framework. It causes people to foam at the mouth because it implements the same standard as Microsoft.NET, and therefore is omg microsoft taint evil lulz
Wow. Talk about a spectacular over-simplification.

To the OP, Mono is a technology that allows .Net development tools and an implementation of a [common language runtime]("http://en.wikipedia.org/wiki/Common_Language_Runtime") on Linux. There are a number of advantages to this but the most commonly cited is cross platform compatibility and the potential to bring Windows applications to Linux more easily.

The downside and a very real concern, is not the applications written in C# but in the lack of a free implementation of the language. If someone was to claim patent rights we lose the software.

While directhex is right in so far as there are many who get very excited over this, there is a good reason.

[http://www.fsf.org/news/dont-depend-on-mono/view?searchterm=mono](http://www.fsf.org/news/dont-depend-on-mono/view?searchterm=mono)
