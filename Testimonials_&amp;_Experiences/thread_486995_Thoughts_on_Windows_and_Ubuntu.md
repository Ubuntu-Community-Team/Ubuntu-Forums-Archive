### Post by jasonrandolph on 2007-06-28
I've been reading the Goodbye Ubuntu thread, and a couple of the original poster's complaints hit me, specifically the lack of games and the tediousness of the command line compared to XP.

I'm now starting to think of Windows in a whole new light.  Maybe the abundance of games and the lack of a command line is Microsoft's way of saying that the general public is too dumb to handle an operating system where they have the power to control their entire computing experience.  By doing away with the command prompt, the general public is less likely to screw up their own computers, and therefore there will be less need for tech support.  Instead, let the problems be "limited" to those that Microsoft's own coders couldn't handle.  And by pushing games on the public, they will make the personal computer an entertainment center, and leave the "serious" computing to the anointed few (who certainly don't work in Redmond.

The reason I came to Linux is exactly because I wanted more control.  I want to learn how this thing works, and I want my computer to run the way I want.  I don't want to be "fed"; I want to feed myself.  And THAT is something Windows will never allow me to do.  So for now, Windows is a slowly dying annoyance.  Once GIMP reaches the level of Photoshop CS2, I'll never let Microsoft touch my PC ever again!

Anyway, just a few random thoughts.

### Post by weblordpepe on 2007-07-01
Hmm...
OK I might just butt in for a minute. Excuse me if I seem like a bit of a troll.
But I have a different opinion on the pros/cons of Windows XP over Ubuntu & linux.

XP is very much an official, defined install. Software on the computer is either an OS component, or it isn't.
In Linux,...any linux....there is no distinction. Really. There is the 'main' repository, sure... When you download software, its just as if it was part of the OS.
Software comes out for linux which has requirements xx, yy and zz. There is no way every linux machine is going to have every component needed for software to run.
On XP, however - the developers know what is going to be on the system. There is specific, documented ways to use features on the OS. Component improvements aren't very often but they are well established and somewhat uniform.

For example: You gotta upgrade your DirectX to play WeblordPepe Ultimate Tennis.
OK I can do that. The EXE comes with the game. do do do.

But in linux, even with good package management, there is dependancies all over the show. ALSA, OSS, Mesa versus something else. KDE or Gnome. GTK or QT. 
To get the lowest common denominator for all linux systems, you have to get simple. And thats the command line. Ouch.

Remember, with XP - there is a few hundred million computers with uniform GUI, settings, folder structure etc.
I know there is efforts underway to standardize the linux desktop. Thats a good thing.
