### Post by shateredsoul on 2009-02-15
Oh man man man... the best word of advice I would suggest is make sure you have time, but the time is well spent.

As a graduate I have little time to do side stuff, so I must admit that my usual work outs and some of my work were ignored for the sake of getting programs I needed working.

I was able to get vbox to run windows which was great to keep using some of those xp files I still need.  I can still give my students feedback on their paper with track changes and open office works great for typing my own papers.  Unfortunately it turns out that I installed an OSE version of virual box (open source), so it doesn't recognize usb devices (doh!).  So I'm currently figuring that out.  So I learned one thing... research before you install.

It took a lot of frustration for me to install a statistical analysis program because it didn't like compiz.  I did find instructions but did not understand them, luckily several nice people were patient enough to explain what the instructions meant on the forum (some of the stuff had to be entered into the terminal, and I had to go and edit the shell of the program), it still didn't work but then i reread the instructions and it turns out I had to launch from the shell spss instead of SPSS the main program.  It works fine now. So I learned 2 things from this.. the forum members are really patient and helpful and try and try again.  

I love ubuntu.. even my wife who thinks it really cool.  I dual booting my machine and have decided to single boot with ubuntu.  Here are a few things that may be of interest to graduate students (or students, teachers, professors).. and things to research and learn how to do.

tips

1. give your time to explore ubuntu.. I did this by installing wubi on xp.  Install a few programs, edit them, find out what the terminal is and does 

2. what programs do you need? stats programs? audio video editing? basic word? 

3. make sure you have the original install cds or files for programs you need to reinstall on ubuntu or vbox running xp (if you paid for a program you might want to find out if you will be able to reinstall it)

4. hang out in the ubuntu forum, ask questions, people are patient even with the newb questions.  At least try to google it before you ask though.  I have asked quite a few questions that never received respones.. but after googling I found out that the answers to my problems were a question that had already been asked several times.  Still, usually I received help through links and messages 

5. have lots of time, do it during the summer.. winter break... or weekend. Make sure not to be stuck in a situation where you need to run a program and you can't so test things out before meetings, presentations, etc..

6. be patient.. oh so very patient

pros
-it feeds your nerdy side, you can always find something new to learn 
-it's extremely fast
-supportive forum
-freeware! =)
-amarok has a lot of cool features.  I have so much music, itunes keeps repeating the same song.  Amarok gives you ways to explore (i.e. the pick 50 random songs option).
-virtual desktops (yes mac has it helps when you are student to have a data file on one screen, a word on on another, and facebook on a third)
-most of your usual procrastination tools work on here.. facebook, youtube.  Haven't tried chat, but chat through gmail works fine for me.

cons
-time consuming
-sometimes my newb-ness annoys ppl.. and me too. (ask yahoo is not your friend)
-relearning what you already knew how to do on xp/mac (it was fun for me though)
-sometimes you want something to work... and not have to go through forums


Basic tips for xp to ubuntu users
-Terminal is where you will have to enter a lot of commands.  To find this Applications<Accessories
-the program files folder is in home/yourusername/ then press ctrl+h to make the invisible files visible
-ctrl-alt-del, ok not sure how to get easy access to this but go to system<administartion<system monitor to get there
-sometimes, when you open documents using open office it won't let you edit them, save a file with a new name.. now you can edit.

Now go and do the work you were suppose to actually be doing.

### Post by SeePU on 2009-02-15
> **shateredsoul said:**
> Interesting, so if I wanted to build my own pc... would it be best to go for an older motherboard and cpu? 

Lets say I bought a new dell machine, would it most likely support linux? 

-shatered
It depends on your budget.  If possible, buy parts, mobo, cpu, GPU card, RAM etc.  Buy Intel, P35 mobo, etc.  If you buy a Dell system or something like that, it might work but Dell often includes their own hardware and sometimes Broadcom for the ethernet and/or wireless.  Broadcom is getting better at Linux support but before it was just lousy.  If you buy a system buy individual hardware if possible, as you can choose what is most compatible.  

If you buy a laptop, most of the newer laptops are very good at Linux.

I agree with the other poster.  It often depends on your hardware and then you have to worry about the learning curve and getting used to the differences between Linux and Windows (if you are used to and most familiar with Windows).

Also, be aware that besides CLI (command line), there are some nuances and other annoying situations you will find yourself in.  I mean, sometimes a solution will be found by a google search and the solution won't make sense or it will be some lines you edit in a file using an editor or it will be a bug you will have to deal with but there might be a fix.  It's hard not to get irritated with that but you have to think back to all the annoyances in Windows if you can.  Just remember that you have to learn some new things and that a lot of hardware manufacturers try to get their Windows drivers sorted out first before worrying about Linux support (unforunately) but the more we use Linux, that allows more chance for change in that strategy!
