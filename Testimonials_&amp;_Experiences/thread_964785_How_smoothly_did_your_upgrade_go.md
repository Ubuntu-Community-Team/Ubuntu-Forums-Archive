### Post by aaaantoine on 2008-10-31
I'm suffering from at least two new regressions since 8.04.  I don't think it's related to the upgrade process though.

The new Gnome bluetooth preferences window is crude; it doesn't even have tooltips for the buttons at the bottom.  I can only guess, from left to right, "New", Connect/Disconnect?, Not a damn clue, and "Delete".  Worse than that, my BT mouse keeps losing connection, unable to reconnect until I delete it from the list of known devices and reconnect it.

I've lost the ability to use gaussian blur in Compiz.  This is probably a regression in the FGLRX driver.

The new version of Kate for KDE 4 has some weird features that are bothering me.  Such as, hitting tab only moves 2 spaces at a time, even though a tabulator is set to 4 spaces wide in the preferences.  And also, it asks me for FTP credentials far more often than the previous version did.  Not on every save/load, but far more often than once per session.

As far as progress goes, my laundry list of hardware problems still goes unanswered:
 - Cannot record audio.
 - Headphone jack does not toggle external speakers.
 - Microdia webcam is still unsupported (though I know this driver is in development).
 - Still depending on fwcutter for my Broadcom wireless firmware.
 - Radeon Xpress 1100 driven by fglrx works, but is embarrassingly slow, even for integrated video.  I can't play Diablo II, an 8 year old game now, in a virtual wine window without choppiness?  Blasphemy.

And an 8.04 software regression -- where nautilus FTP does not show all folder contents -- has not been fixed for 8.10.

These are currently my greatest issues with Ubuntu and Linux software as a whole.  Not enough to deter me.  Though I'm tempted to switch to KDE so I can get full FTP support...

EDIT: And also, Pidgin has decided to start crashing on me, which I'm not happy about.

It's kinda funny to watch.

### Post by snova on 2008-10-31
I had quite a time.

Originally I was planning on upgrading. Then I ran into some major problems in Hardy that prevented me from booting at all (except in recovery mode). Live CD's still worked, thankfully, so I changed my plans and decided to reinstall.

So when Intrepid came out I downloaded it via BitTorrent on my mom's computer. Since that particular computer's cd burner doesn't seem to work right, I had to put the iso on a thumb drive and burn it with cdrecord on my laptop.

Booted it up, and got no furthur than the first menu. I tried it from text mode and found that the kernel was corrupted. Apparently I'd burnt a bad disk.

So I made sure that the download had been successful by checking the md5sum on mom's computer, and it was perfect. Then I checked it again on the thumb drive, and it was perfect there too. But I got I/O errors from the kernel when I tried to check the cd drive.

So I started over again, but this time I slowed it down to 4x. Booted it up, everything was perfect. Then it gave up halfway through the install because of a corrupt help file it couldn't copy.

By now, I was running out of disks. Mom helpfully pointed out a stack of them I'd not noticed before, so that problem was trivially solved. Unfortunately, I hadn't realized that the installer had wiped out my existing installation, so I couldn't even boot into recovery mode. Not that it mattered, because at this point I didn't trust my cd burner anymore.

Fortunately there is a third computer in the house, newer than both mine and mom's. Downloaded a program to burn the .iso, burned it, booted it, and it *finally* worked.

Hardly the way I intended things to go, but here I am.

### Post by Bronto on 2008-11-01
**[color=#000066]Dell Inspiron 1501 : BIOS Revision 2.6.2[/color]**
**[color=#000066]Ubuntu 8.10 upgraded from 8.04 (online).[/color]** 

**A. Old stuff still working after upgrade:**
**[color=#000066]1. Hardware[/color]**
- wireless (with ndiswrapper). One issue: NetworkManager kept asking for password on every startup. Solved by going to **Applications** -> **Accesories** -> **Passwords and Encryption Keys** -> **Passwords**, selecting the entry in the list and responding "*Always allow*" when "*seahorse*" asked for access.
- video (ATI Radeon Xpress)
- touchpad. One issue: GSynaptics is dead (because of xorg modifications) and the OS informed me that my touchpad is now managed by HAL (**System** -> **Preferences** -> **Mouse** -> **Touchpad**).
- sound. One issue: The system beep (from pc speaker) is back (must be somebody's obsession) and there is no longer an option to turn it off from System -> Preferences -> Sound. Problem solved by launching **gconf-editor** and removing the ***audible_bell** tick* from **apps** -> **compiz** -> **general** -> **allscreens** -> **options** and **apps** -> **metacity** -> **general**.

**[color=#000066]2. Software (not present in the standard install).[/color]**
- CHM Viewer
- Gnome Commander
- GVim Text Editor
- pyRenamer
- Ant Movie Catalog (with Wine)
- Radmin Viewer (with Wine)
- Picasa
- Fonty Python
- Inkscape
- Scribus
- Sun xVM VirtualBox
- Vmware Server 2.0 (reconfiguration needed)

**B. New stuff working after upgrade:**
**[color=#000066]1. Hardware[/color]**
- Backlight control keys

**[color=#000066]2. Software[/color]**
- OpenVPN connection to a Endian Firewall server from NetworkManager (using network-manager-openvpn).
- Alternative Find & Replace extension for OOo Writer (full width and height)

**C. Still not working after upgrade:**
- resume from Suspend (not tried Hibernate, I don't use these functions).

**Conclusion:**
Flawless upgrade, minor issues, nothing important. Definetly, a step forward.
