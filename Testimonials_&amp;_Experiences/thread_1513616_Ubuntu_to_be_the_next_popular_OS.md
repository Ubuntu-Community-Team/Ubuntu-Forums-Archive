### Post by milawynsrealm on 2010-06-19
I have used Ubuntu on and off for almost 3 years now, and I applaud the  developer's efforts. With each passing release, I see it more and more  as a potential Operating System that may very well join the ranks of  Windows and Mac OS X. If it wasn't for the fact that I was both a hard  core gamer and using some Windows-only programs for college, I would  have completely switched to Ubuntu a few years ago.

What I like  about Ubuntu:

[LIST]
[*]Fast Start-up times
[*]An Update system  that updates more than just the system itself (I'm talking about software)
[*]Hassle-free  network setup
[*]Free upgrades for life!!
[*]Easy install of  proprietary video drivers
[*]Straight-forward Software Center
[*]Wubi  Install makes installing Ubuntu in a windows environment easy
[/LIST]
Although  I like the Operating System a lot, there is always room for improvement  that would not only make the OS work better for me, but could attract  more people into trying out the Operating System:

[LIST]
[*]**Extend  the Software Center to allow the easy sale/use of Proprietary Software  via an integrated store system**
[LIST]
[*]Although Free/Open  Source Software is the reason software like GNU/Linux is so popular,  proprietary software such as Video Games should be allowed to be sold to the public in the Software Center in addition to the large list of free software already available for Ubuntu.  The reason behind this is that not only would it allow people who are  used to Proprietary Software and/or may or may not completely understand  the actual power of Free/Open Source, but it would also give them more  choice in what the users want to use. Software packages such as  OpenOffice.org may be a good replacement to other productivity suites,  but if someone wants to use Corel(R) WordPerfect instead, then they'll  go for the Operating System that supports it.
[/LIST]
 
[*][B]Create a  Developer Center to better allow software developers to find information  about creating software packages more easily
[/B]
[LIST]
[*]Not  only would it be a good source of revenue for Canonical, but it would  allow a centralized internet location for aiding in the development of Linux Software  (using GTK+, OpenGL, PulseAudio, etc.). The developers would have a central place to look  for API documentation and information about compiling on the system  (either through the CLI or through an IDE). I would imagine the basic  service being free for common users creating their own programs (full  access to system API's, tutorials, etc.), and a service that would cost a  monthly/yearly fee and most likely be used by software companies  (software certification, training discounts for development teams,  software testing, etc.). Both Windows and Mac OS X have something like  this, and it would be more familiar for professional developers to  develop software a lot easier.
[/LIST]
 
[*]**Focus development on  making users less dependent on using command line arguments**
[LIST]
[*]I'm  not saying to get rid of the console or anything like that (the CLI is  useful in many cases), but to make Ubuntu more user friendly and to  allow users to make the most of their system without having to type  anything in the console. In Windows, I can change many system settings,  and diagnose parts of my Windows System without having to know how to  use the CMD, or having to create BAT scripts to make my life easier. If I  wanted to create a shortcut in Ubuntu, I should be able to easily  choose between if I want to load a program from an easy list instead of browsing through the **/usr/bin** folder with applications too many to count, looking for a file, or have  the optional choice of typing in a command argument. The goal for Ubuntu  should be to allow basic options that common users would choose, while  allowing the options that make it easy to use for power users to change  the settings the way they want to. I have used Windows ever since I got  Windows 3.11, and I have never had to create a BAT file, or use most of  the power user programs found inside the **System32** folder (with  the exception of cmd.exe, and dxdiag.exe).
[/LIST]
 
[*]**By  default, the file browser should display more alternative,  human-readable folder names limited to the ones found in the root (/)  directory**
[LIST]
[*]Just so you know, I'm not talking about a  change to the default names in the root folder, or something that would  go against the Filesystem Hierarchy Standard, but rather, a way to show  where everything is on the filesystem in a GUI File Browser. In the  console, ***/bin*** is short in length and reduces clutter, but in a Graphical  User Interface, this actually offers more a confusion for some users  (especially for beginners), and not as necessary to have the folders 3-4 letters in length since the clutter of the file/folder name would almost be non-existant. By associating the folder name "Binaries" to  ***/bin*** for example, it would likely reduce confusion and  gives a name that would be more meaningful than simply **/bin**  alone. For something like this to happen, there could be either a  plug-in that comes with the distribution, or changes to the File Browser itself to allow this. It should also be  easy for users to disable this feature if they wanted to (to make the  power-users happy), and show the actual folder names (**/bin**;**/sbin**;**/usr**) if someone wanted  to.
[/LIST]
 
[/LIST]
These are the ideas that I had off the top of my head.  I may have more later, but for now, this is a good list that could make  Ubuntu more popular for the masses.
