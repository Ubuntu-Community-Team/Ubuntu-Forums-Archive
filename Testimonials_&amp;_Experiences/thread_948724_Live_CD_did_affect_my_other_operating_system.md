### Post by 3rdalbum on 2008-10-16
It probably wasn't a case of "Ubuntu screwed your networks up". Ubuntu did exactly what was meant of it: It sent a DHCP request to your network, just as Vista does when it boots up. That DHCP request contains a hostname, which on the Ubuntu live CD is "ubuntu". Your network remembered that hostname and then when you started up Vista (which doesn't send a hostname), the network sent Vista the "ubuntu" hostname.

That might have caused things to happen that you weren't expecting, but it's completely normal behaviour defined in the DHCP specification. Both Vista and Ubuntu were performing as they should.

As for your hardware, you mentioned that you built your own computer. Every operating system requires you to install it on hardware that is compatible with it. If you've built your computer with components that do not have Linux drivers or are specifically Linux-unfriendly, then naturally that's going to give you troubles. I *am* surprised that you couldn't get graphics acceleration working, and until I get more information I'll assume that it's user error or that you are using a cheap, obscure chipset like SiS.

You wouldn't try and install Vista on a computer with older components that don't have Vista drivers.
