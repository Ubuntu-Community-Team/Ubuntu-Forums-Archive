### Post by danomatika on 2007-04-29
It's all a matter of intent.  Do you want to change your preferences or not?

I think it's a mistake to require complete replacements for Windows applications in Linux before you switch.  If you want to switch, then do it and find the apps that work for you.  It may require more then a single application to achieve the workflow you may have had in Windows.

Example: Fruityloops -> Seq24 + Jack + Ardour + ASynth + etc ...

I wanted to switch for a quite a while last year, but I was held back by using one program: FL Studio.  When I decided "Ok, this music project is over, I don't need FL Studio now."  I bit the bullet and just lept into Ubuntu as my sole OS.

I have no complaints now, I can do everything I needed to do in Windows in Ubuntu and I feel far more creative in the end.

So the real question you should ask yourself is "How much do I really want to switch?".  If you spent the time to learn Fruity Loops, why can't you learn Linux audio?

### Post by Geekkit on 2007-06-09
> **barmazal said:**
> There is great progress going on with Ubuntu Studio, this is great you put it under same name as this friendly Linux but still Audio/ Graphics/ 3D / Web Development is on stone age level

Stone age? Your statements simply don't wash. Here's why:

1.) GIMP supports everything that Photoshop does except for three (important) things I've come across:

[LIST]
[*]Pantone support (proprietary)
[*]16 bit channel support
[*]Free distort
[/LIST]

The first is a limitation that only a professional graphic designer would spot - not your average Web page enthusiast. The second option (i.e., 16 bit channel support) is easily solved by using CinePaint (which does support 16 bit channels) although this is another feature that is not going to be used by your average shopping artist: 16 bit support is only a requirement in the film industry. The third feature is missing and not available. But give that a year. Behind yes. Stone age? I don't think so.

2.) Vector support ... Inkscape. Inkscape has features that compare (maybe Pantone support missing), and output to SVG which is more than I can say about Illustrator and Freehand. I've used Freehand for professional publishing and have found myself getting away from it because of it's mangling of EPS output. Inkscape was my savior at that time. Albeit it may not have all the features of Freehand and/or Illustrator but surely not stone age.

3.) 3D huh? For open source you have (same as you have on Windows): Blender (as far as full packages go). Not industrial strength? Okay, download Maya or Softimage for Linux. Same thing right? This is not only not behind, it's at par since it offers everything that you can get on Windows: same open source, same expensive commercial alternatives. Stone age? Not a chance.

4.) Web support. I have yet to see a Web WYSIWYG devel tool that actually does a decent job of XHTML (don't talk to me about VS, GoLive, DreamWeaver since they all commit crimes that make W3C Recommendation enthusiasts cry out loud). But anyways, there are several solutions which seem to take standards seriously (and don't horribly mangle XHTML or CSS). First, there's NVU. However, and even more powerful is the battle between NetBeans and Eclipse - both offering plugins for doing WYSIWYG drag-and-drop XHTML programming, with support for languages other than simple Javascript. Both NetBeans and Eclipse are cross-platform although NetBeans is more cross-platform since it's pure Java (i.e., no native libraries). NetBeans is also endorced and blessed by Sun as well so it's gaining momentum.

5.)  Audio support is there as well. Audacity which supports multi-channel sound editing with a time line. There are also several MIDI-based sample mixer programs (sorry I'm not terribly into this stuff but I've come across several links to pieces of software that support this sort of thing). Again, I'm not denying that there may not be the depth of support that you have in say Windows but surely not stone age.

6.) Last point video and 2D vector animation support (a.k.a. SWF output). This area is a definite blind spot. Video software seems to be at about 1999 to 2000 - I'm thinking of Premiere. Flash doesn't really have a contender although there is Synfig which seems reasonable but doesn't output to SWF.


- - -
With all of these available options I would definitely say that there's a lacking. But stone age? That's not an accurate assessment. You need to look harder but if you do I think you'll find that there are options out there. The way I look at it, I give it another 3 - 5 years before we're at par.
