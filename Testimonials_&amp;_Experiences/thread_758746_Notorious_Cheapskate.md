### Post by BradBrening on 2008-04-18
I just recently installed Ubuntu on an old machine and I thought I'd share my experience and observations.

First, let me say that my entire computer experience has been Microsoft based - from back in the old DOS 5.0 days.  I've grown through DOS 6.0 through the various Windows versions.  To this day I am a developer using Microsoft technologies.  My reasons for deciding to install Ubuntu, however, have nothing to do with any sort of disillusionment or complaints about Windows.  My reasons are because I'm a notorious cheapskate.

It basically started with getting an XBOX 360 for Christmas and a HDTV a little before that.   I wanted the XBOX to play Grand Theft Auto 4 when it comes out, and the HDTV because, well, I'm a guy.  I love playing the 360, yet I knew I was missing out on a lot of things because I didn't have it connected to the Internet - and I'm too cheap to pony up the $99 they want for the wireless adapter.

So, I'd been using my XP laptop (which connects wirelessly) which I'd configured as a bridge with my Ethernet port.  In this manner my XBOX could get online.  Needless to say, my XBOX became that much more enjoyable.  The problem is, though, that my girlfriend likes to use my laptop to play other games, meaning my XBOX was again detached from the Internet when she used my laptop.

So, I had the bright idea to take an old machine and install a copy of Windows on it (maybe 98 or 2000) - along with wireless and ethernet cards so I could use _it_ to create the network bridge.  This is when I decided that since I'd be installing an OS anyway, why not give Ubuntu a try?  What the heck, right?

Well, the install wasn't without hiccups - probably because I was trying to install 7.10 on an old Compaq with only 150MB of RAM.  There wasn't enough memory to run the Live CD and get the install started.  After fiddling for a while, I finally noticed that the Live CD said that it would use any Linux swap partitions if they were found.  Eureka!  So, I used the Partition Manager to create a large Linux swap partition and installation proceeded without a hitch.  This whole process took me a weekend.

After Ubuntu installed and running, I noticed that it recognized my Ethernet card, but my wireless card (based on the Broadcom 4303 chipset) wasn't working.  After many, many dead ends trying to get it going, and about the time I was going to throw in the towel, I noted a post on here about using ndiswrapper and some other driver to get it going.  Now, as I said - I'm a Windows guy - so here I am pecking away at a terminal like a total idiot.  Thankfully the steps provided in these forums were very concise - even for someone who'd never used Linux before.

After I got the wireless going (finally!), my troubles weren't done, though.  As I said, I'd been using my laptop as a bridge, so that's the direction that I tried to go with Ubuntu.  It wasn't working at all.  I then proceeded to try to follow another forum post that had me delving into the terminal again to try to configure port forwarding between the two cards.  No luck.  Finally, I installed Firestarter and with a couple of clicks my Ubuntu machine was finally doing what I wanted all along - connecting wirelessly to the Internet and sharing that connection via the Ethernet card with my 360.

My impressions since I've been running it now for a week (or so) is that it's a pleasure to use.  Since the Ubuntu  machine runs video through my HDTV, it's nice to be able to switch over and Wikipedia something interesting I just saw while watching something on TV.   I'd definately install it again - or recommend it to other users in the future.  And a great big thanks to this forum for all the help getting my little project up and running!
