### Post by jbysmith on 2008-04-29
Yea, its a nice feeling when you eventually realize "Why the hell did I put up with this garbage for all these years?" and blast Windows into dust.  I started computing back in the mid 70's on a TRS-80 Model 1, then eventually became a Microsoft drone when the PC-Jr came out.  I even toyed with Unix way back then with Microsoft's Xenix operating system on a Model 16, but I figured it would never catch on.  Big mistake on my part lol.  Been using MS operating systems ever since until about two years ago.  

For me, it wasn't the eye-candy of Compiz-Fusion.  (Beryl back when I switched)  Granted, it definitely helps lol.  It wasn't the price either (or lack of one); I'm a huge fan of the open source concept though, and can't wait for the day where the US legal system slaps Microsoft around for the whole "intellectual property" stupidity. 

For me it was the stability and security of an OS that I don't have to constantly babysit.  I don't have to load up a ton of security apps just to still worry about getting some random trojan. I don't need to do a major upgrade to my hardware just for a little bit of minor eye candy and a lot less compatibility.  I'm waiting for someone to notice that you'll need a high-end quad core with 4GB+ of memory just to run Solitaire on Windows 7.

It does everything I need it to do, and 1000% better than what I was getting in Windows.  FireFox, OpenOffice, Evolution, Exaile, MPlayer, programming, etc etc it's all there natively.  There's a couple applications from Windows that I can't do without, but Wine covers that very nicely.  I use The Journal (a diary app) to track work I do for clients; haven't found a native Linux app that works as nicely as it does.. may have to write one some day.  And when I'm goofing off, World of Warcraft runs amazingly well, even faster than what I had in Windows.

The console environment is amazing.. you can do anything in it. Think the only time I've ever used the command prompt in Windows is to flush my DNS once.. useless otherwise.  Yea, in Ubuntu for the most part you don't need to, but I actually like it.  Been brushing up on my Bash script programming.. neat stuff.

As a server it's downright awesome.  I have a couple servers at home that I use for web, programming, database and the like.  They were running Windows 2003.  I converted them to Linux and got a drastic performance boost, and the amount of memory it's using now is ridiculously low.  My one server runs Apache, MySQL, WebMin just for simplicity's sake,  automated backups for other machines via some Cron jobs, and acts as a file server as well.  Does that all under 100MB of memory.  Can't even boot a naked Windows 2003 install under double that.

I think the only thing I miss from Windows is the wide availability of gaming.  (Sooner or later publishers are going to notice there's more than 3 or 4 Linux users..)  But eh, most games are nothing spectacular.. same old story with a new coat of paint.  If there's something I desperately need, I could always make a small Windows partition for gaming, but in the past year or so, I haven't felt inclined to do so.

Granted, as a desktop OS Linux in general has a ways to go before  it'll be a serious threat to Microsoft stockholders, but it's definitely getting there.  The latest version of Ubuntu for example is causing some big problems for some people (myself included, staying with 7.10 for now till later, or at worst waiting for 8.10)  Wine needs to be able to claim it'll run everything before the die-hard Microsoft fans switch.  Plus, some people will never switch anyway. They use Windows, and expect other operating systems to work exactly the same way, and refuse to learn a different way to do things.  Their loss.

I'm learning something new every day about Linux, and loving every minute of it.

### Post by jbysmith on 2008-04-30
> **ejpyle said:**
> i would like to discuss some of these things further... I have aspirations of running my own Linux webserver....

I appreciate your comments...lot of interesting information,

Ask away.  There's plenty of guides on the net, and tons of helpful people on these forums.

But there's really not much to it.  Typical "LAMP Stack" consists of a Linux core, Apache, MySQL and PHP (or Perl or Python depending on who you talk to).  All of which you can find in pretty much any distro's repo.    As an alternative to Apache, which can be a little intimidating at first to configure, Abyss Web Server also works remarkably well and is a no brainer to configure as it has a web page you use to configure it.  But, the free version doesn't have all the features of Apache, nor open source.  (Namely missing multiple domains and a few other minor things.. very capable as a home web server for sure though.)  I've toyed with the XAMPP server stack a couple times.  It's actually not bad, but by default it's very insecure as it was designed for development use.  You can set it up for production use, but personally I'd stick with the repositories just to save yourself some headaches down the road.

You might also want to consider using WebMin to manage your server.  Simplifies configuring Apache, Samba if you're sharing files, your scheduled backup jobs, etc etc, pretty nice. I've heard of people having problems with it; I personally have used it constantly with no issues whatsoever and is a nice timesaver.  

As far as the distro's go, I've used three and they all worked very well.  All of them are not running a window manager, pure console to save memory, CPU time and just less things in general to worry about going wrong.  My home server is running Ubuntu Server 7.10.  (Same as regular Ubuntu, minus Gnome.)  One of my clients I set up with Arch Linux, another excellent distro that's stupidly easy to configure (with their Wiki open on another display). One config file to deal with, rc.conf, covers everything.  Network, daemons, the works are all set it one file.  Another client I set up with Debian Lenny/Testing.  It's what Ubuntu is based on, and the setup process for the software is pretty much identical.  I liked that one as it's constantly being updated vs a 6 month cycle.  

It's really not that hard to configure, just be ready to do a lot of Googling, especially where Apache2 is concerned as it's a little intimidating at first until you understand how the configuration files are laid out.  Or, if you just need something simpler for one domain, give Abyss X1 a try, stupidly easy to set up via a web browser, very fast, supports all the script languages, has built in "anti-hacker" and "anti-leecher" systems, etc etc. The only downside I found to it is that it doesn't support Apache's .htaccess files, but 99.9% of the time you can get around that.  Any configuration settings you can do via the admin console. The only semi-tricky ones are URL rewrites; Abyss supports them but you set them up a bit differently in the admin console.  I personally prefer Apache2 myself, but they're both very good.
