### Post by -kg- on 2008-10-31
*Oh Lord Yes!!*:roll:

I was heavily into Micro$oft before there was Windows.  I pretty well started off with DOS 2.X and remember seeing Windows 1.0 on the software shelves and thinking, "Oh great...another DOS Shell."  I was into it through the various Windows "distros" right up to Vista.  I had various forays into alternate OSes like OS/2, BeOS, Atari (Atari 800XL), and even Linux in it's earlier days.

I recently decided to make the migration to Linux after getting more and more disgusted with Windows and it's susceptibility to virii, increasingly poor design and ability to handle tasks, it's increasing "DOF" (Denial Of Features...unless you want to pay extra for them), and finally, their increasing use of built-in "spyware" to monitor your usage (***Validation Tool, my...***well, you get the idea.)

I have found that the recent Linux distros available are extremely easy to install (I recently installed Ubuntu on my laptop while at a Pool league match), most everything (if not everything) works, even off the LiveCD, and if it doesn't work it's reasonably easy to configure the OS so that it does.  When you install software (especially using the Package Mangers installing approved and tested software) it very often works without a hitch, and one is free to configure and write software to their heart's content.  There is no "hiding" of proprietary coding except to the company's "approved" third party software writers.  You are free to decompile and recompile to your heart's content.  No restrictive EULA (well, except for one case I've come across...FreeSpire)...the only "EULA" involved is the GNU EULA (in most cases...I'm sure the Enterprise versions might have a limited one).:cool:

Windows, on the other hand, has always been buggy.  While I must reluctantly say they have gotten better in the most recent distros, you trade buggy for bloatware.  It takes more and more hardware resources; more and more Internet resources (that is, if you want to be able to fix the bugs in the software);  third party vendors' software, while it might work fine, are "dirty," often not cleaning up after themselves until they or the OS starts slowing down and finally fail.[-X

Yes, every time I need to leave Linux and return to Windows for something (I still need it around for a while.  I use DeLorme mapping software and a GPS satellite receiver in my car, and I do like M$ Flight Simulator X), I think to myself, "I paid $200+ for ***THIS?!***"#-o

There are times that I go to Windows to do something and while I'm there decide, "Well, I'm here...I might as well do ***that*** too while I'm here" remembering the "convenience" of doing it under Windows.  Often I am forced to return to Linux to do those tasks because they aren't as easy and convenient as I remember.  I get it done under Linux every time.

Yes, every time I go to Windoze, I think how much it sucks, and every time I return to Windoze that feeling increases, almost exponentially.  *"I paid $200+...**FOR THIS?!!***#-o #-o ](*,)

Too bad Linux only has two syllables, because if it had three, instead of singing, *"I want my MTV"* I'd be singing *"I want my Linux!"*
