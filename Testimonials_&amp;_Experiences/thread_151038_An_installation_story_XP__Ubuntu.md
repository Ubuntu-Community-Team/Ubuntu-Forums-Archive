### Post by risbac on 2006-03-27
Hardware drivers suck on Linux. This is what everybody will hear when asking about Linux. If one can't deny that some drivers are really missing, all is not so bad in the hardware world on Linux. An exemple with a friend buying a new computer. He wants to install XP, puts the CD, boot, no harddrive found??? WTF???. Ok, I check the computer, Sata HD, but no floppy reader... I try to explain that you need a floppy drive in 2006 to install XP. 

As it's possible to add a driver on the CD, we burn a new version, try again, still not HD. I propose to install a floppy drive on the computer, but if we open the case, the warranty is cancelled. So he goes back to the store, ask for the installation of a floppy drive. 

Back home, we prepare the driver, put it on the floppy, boot on the CD, the HD is recognized, the installation goes fine, first reboot, second and final reboot and BOOOOM. Blue screen. Ok, a wrong driver, let's do a minimal boot, same. No way to boot this XP... The CD is maybe too old. 

I find prepare another CD, with all the possible updates: SP2, critical updates, add some drivers. We try again, beginning of the installation, ok, first reboot, BOOOOM. Blue screen again, can't even finish the installation. We check for the error message on internet, 12 different reasons, included on Microsoft website. Could be the power, could be the DVD burner, could be this or that. And we can't open the case to remove the hardwares one by one. Great. My friend spent  1000€ for a computer he can't use for more than a week. 

Just to check if it's really a hardware failure (bad ram???), I try my Dapper CD. installation is ok, first boot ok, system is working fine, all the devices seem recognized. Sadly, my friend wants to work on video on his computer, so I can't install Premiere on Ubuntu. 

Right now, we are still trying to find the origin of this blue screen online, but it's diffcult to know if it's a driver problem or a hardware problem. Great.

What else to add? I know it's maybe a particular exemple, but it still shows that everything is not black or white in the hardware drivers problem.

The truth is that it's very common to have hardware problems with Windows as well. These people, however, will say "I've never had any problems with Windows" and will be right, just as many people here have never had any problems with Ubuntu. 

As of today, neither world is perfect.
