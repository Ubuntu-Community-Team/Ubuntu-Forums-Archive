### Post by armandh on 2009-09-15
sometimes getting Ubuntu on to a computer is like playing solitaire with a deck of 51. mostly it is hardware, occasionally it is a bad disk. almost never Ubuntu.

my experiences.
looking back I find the most prevalent problem leading to total install failure is a bad cd drive. they may seem ok for most read/play where some missed read goes un noticed. they wont install an OS. one only knows for sure when, after substituting a known good drive, the install is slam dunk.

mouse pointer on a black screen and other X problems.
lets hear it for totally worthless on board video. go straight to xubuntu or puppy. Ubuntu is great for older equipment ONLY where it has or one can plug in decent video. I have an old 866 P-III with a 64mb AGP video card that will twirl the cube and play DVDs. [not at the same time] I have had to put a lighter OS on faster equipment that is limited by crappy on board video. some of this junk is so bad under XP with factory drivers I am sure it is not worth anyone writing Linux drivers for it. 

for example
there is actually an intel MOBO with an SiS chipset/video
[http://www.silentpcreview.com/article780-page1.html](http://www.silentpcreview.com/article780-page1.html)
ubuntu will run [2D vesa] with a 2 word edit but OOTB the screen has a lot of sparkles/drive lines. that one is back to XP and video is barely tolerable with factory drivers. [future server]

In summery, Ubuntu is the best all around of the linux distros IMHO and as evidenced by its popularity, but there are places where it will never fly. it is no fun trying to go there and easy to understand the frustration of those who try.
