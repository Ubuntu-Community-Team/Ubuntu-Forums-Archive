1) I have accidentally removed the battery several times while my laptop was running, and once I even pulled the disk drive while it was powered up. **Never** a problem. I'm not saying disaster couldn't strike 5 minutes from now, but in my experience, Ubuntu has been extremely robust -- far more so than any version of Windows I have ever used, on a par with Mac OS X, and way better than Mac OS9 or earlier.

2) In nearly 2 years as an Ubuntu user, I **have** shot myself in the foot in other ways that required recovery. Install **_[COLOR="RoyalBlue"][Remastersys]("http://www.remastersys.com/")[/COLOR]_** to make a bootable image of your system, then make backups to DVD at whatever interval you are comfortable with. In my case, that's before any kernel upgrade, and at least once a month. Remastersys will allow you to restore your system to its exact state at the time of the backup, which takes about 15 minutes on my netbook. I have restored more than once, but always in response to self-inflicted damage, the result of experiments gone wrong.

People offering support in these forums often refer to the command line because it's easier (and more idiot-proof) to describe what has to be done in a CLI than in a GUI (point at this, click at that, etc.) In point of fact, I spend very little more time in the terminal interface with Ubuntu than I do with Mac OS X (and appreciate that all *nix command line interfaces are roughly the same). I have had a few tricky problems with Mac OS that could have been far more easily resolved at the terminal command line, but it's very hard to find command line problem solving advice for Mac OS.

After running Ubuntu for nearly 2 years, my Mac feels awfully constraining, and is used primarily for updating my iPod.

Finally, if you cut the power to any computer with any operating system (except, possibly Chrome OS), rather than going through a normal shutdown, eventually you are likely to have problems. Get a UPS.

### Post by coffeecat on 2011-03-10
> **mastablasta said:**
> Acer to me is not really a quality brand.

As an Acer user I can concur. However, Ubuntu on my Acer laptop has been as solid as a rock. I think the OP's problem arose from the hard shutdowns. Hard shutdowns in any OS can lead to an unbootable system. 

@hyde11, what you experienced wasn't a grub problem or a corruption of the mbr. If the mbr had been corrupted you wouldn't have got a grub error at all. What I think has happened is that your hard shutdowns corrupted the filesystem of your Ubuntu root partition where some of the grub code resides. Grub in the mbr (and the embedding area) couldn't find the rest of itself in the Ubuntu root partition, hence the errors. This is exactly analogous to the situation already mentioned where a hard shutdown in Windows leads to corruption in the boot sector of the Windows partition. 

Your point about the situation where a battery dies is only half right. The default power setting in Ubuntu is to do a graceful shutdown if the battery reaches a critically low state, thus protecting the filesystem. However, if the battery were to fail suddenly, then this problem could occur. But so it could in Windows.

### Post by SeijiSensei on 2011-03-10
Next time try allocating a small primary partition during installation (512 MB is plenty even with frequent kernel updates), format it with ext2 (too small to require a journaling FS), and mount it as /boot.  That way you'll still be able to boot even if you corrupt the filesystem mounted as /.  In addition, you can share a common boot partition across multiple OS installations so you can easily try alternatives.
