### Post by Evil Overlord on 2009-03-03
or, **How I Learned to Stop Worrying and Tolerate Windows**

Ok, the peremptory tone was for brevity - really, this is about why *I'm* thinking about giving up on Ubuntu, not a command that others follow suit.

First, the background.  I'm a fairly advanced casual user, but not a programmer (macros excepted).  I'd been thinking about Ubuntu for a while, and finally decided to take the plunge after a disastrous (what else?) bout with Vista.  Below, some of my disappointments with Ubuntu 8.10.

*** Filing** - What I want is to have exactly three base level folders: \System, \Programs, and \Files.  Windows irritates with its dispersed DLLs, the fact that some programs just have to be in \Program Files, and the general lack of control over where files can go (even if you mess with the default location and name of My Documents).  To say nothing of the constant creation of useless folders such as \My Videos, \My Scans, \My Pictures, etc.  I long for the days of DOS, when each program stuck strictly to its own folder.
[INDENT]  **> Ubuntu** -  Much to my disappointment, I found that Linux is even worse about this than Windows.  Countless mysterious folders such as \bin, \dev, \etc, etc!  And it's even less clear what goes where and why!
[/INDENT] 
*** Arcana **- I'm not afraid of the command line.  It's a useful tool, sometimes.  But I'm not in love with it, either.  And my experience is that it's almost impossible to do anything meaningful in Ubuntu without constantly opening up a terminal and entering arcane commands.  I'm willing to learn a new set of commands, but in my view, I shouldn't have to.  There's no question that Windows is better at this.  Yes, you trade off some control, but Ubuntu just has the balance wrong.  There's frankly no way that Ubuntu is going to become a mass user operating system if constant terminal use is required.

I can already hear someone out there consigning me and other ignoramuses to the trash heap - "If you can't learn how to use grep, you're too stupid to use Linux anyway."  In response, I can only say that if you want Linux to succeed with the average consumer, you're going to have to be less elitist and more user friendly.

[INDENT] **> Ubuntu **- Here's a perfect example: due to problems with my test system, installs are frequently interrupted.  On resuming, the package manager says to me "you will have to manually run dpkg --configure -a".  Why?  The program knows what I have to do; why not do it for me?  Or at least, give me a button that executes the command.  Or opens a terminal and does so.  To me, this is the perfect example of a system designed for programmers and not for users.  I have to find and open a terminal, realize that I need to add "sudo" in front of the command, enter my su password, wait for the command to execute, close the terminal.  All when surely a simple click could have accomplished the same thing.
[/INDENT] 
*** Integration **- What's one of the most irritating things about Windows?  To me, it's the inability to get rid of Internet Explorer, even though I don't use it.  I don't buy Microsoft's argument that it's too tightly integrated into Windows to remove.  Same with Messenger (though that can be removed, if you're really, really persistent).
[INDENT]  **> Ubuntu** - I use Thunderbird, because I use Plaxo.  I was looking forward to Evolution, but chose not to use it in the end.  So, naturally, I tried to remove it.  Much to my surprise, I found that it is "too tightly integrated [into Gnome] to remove."  Seriously?!  What I want from an OS is very simple - yes, I want an e-mail program, but I don't want it built-in to the OS.  One of the big appeals of Ubuntu was to be the ability to load only what I want to, not what the OS insists on.  This forced integration directly counters that.  Yes, I could try Xubuntu, but at this point, I'm losing patience.
[/INDENT] 
*** Wireless **- It seems to be widely accepted that Ubuntu doesn't handle some wireless chipsets well, and especially those from Broadcom.  Unfortunately, virtually all my cards are based around Broadcom chips.  And, in fact, I haven't been able to make wireless work reliably with my test system (Gateway 7422GX).  I've spent a fair amount of time on this, and a) it didn't work, and b) your average consumer would have given up long, long, ago.

---

In short, I don't think Ubuntu is ready for prime time.  I'm disappointed; I was really excited about using it.  But with all the above issues, when I had to return my defective Eee PC 1000, I decided to buy a 1000HE (with XP).  After all, I know how to use XP, I already have a half-dozen copies of it lying around, and all my major apps (Firefox, Thunderbird, Open Office) work either way.

That doesn't mean I'll never use Ubuntu again.  I'll still keep the Gateway test system (sans functional wireless) to play around with.  But did Ubuntu miss a window of opportunity (with me and probably many others)?  Yes, definitely.  If Windows 7 turns out to be a decent OS, I'll probably go with that.

[Again, for those whose response is limited to "Fine, go back to Windows!  Who needs you?"  Well, you don't, of course.  But if you want Ubuntu to be useful to a larger audience, it needs changes.]

### Post by issih on 2009-03-03
FUD = Fear Uncertainty and Doubt - typically used in reference to marketing.

**Filing**

The filesystem layout is partly historical and partly to allow shared libraries between programs. The merits of that get debated all the time, and the arguments on both sides never go anywhere.

One thing you might want to look at is gobolinux:

[http://en.wikipedia.org/wiki/GoboLinux](http://en.wikipedia.org/wiki/GoboLinux)

Which is a project that addresses specifically the filesystem issues you have with linux.

**Arcana**
The terminal isn't going anywhere, its too powerful, and is better than the gui for a lot of tasks. Nonetheless, there are far fewer things that require it than you think, it is the nature of a text based support forum that it lends itself to support using a text based command shell. For 90-99% of things, there is a gui way of doing things. In a sense, the forums are pure help...you will hopefully get a fix, but it may be utterly incomprehensible to you. If you want to learn about the OS, use the ubuntu wiki and help docs, those will show you the gui way more often.

**Integration**
*Shrugs* that is gnome, in the same way that MS is MS and Apple is Apple. There are a squillion window managers out there that don't try to make a nicely integrated Desktop Environment with all the tight integration that that implies. Ubuntu made the decision that more people want the integration than don't, so they use gnome. If you don't like it you can always rip it out and install something more basic like openbox.

**Wireless**
This is not ubuntu's or linux in general's fault. Every time a new bit of hardware comes out (for which there is no manufacturer led linux support) the programmers have to sit down with nothing more than the basic hardware specs and start poking it with a sharp stick until eventually they find a way t make the damn thing work. In all honesty it is astonishing that as many devices work as well as they do. If broadcom would take the tiny effort to do this themselves (forearmed with an actual knowledge of how the damn thing is meant to work) none of this would be necessary. Don't blame the linux developers for something that they can't fix, and in fact spend countless hours trying to make better, blame the manufacturers for their arrogant complacent attitude.

Anyway, linux isn't for everyone, and you should use whatever fits your life best, but it is important that it is clear that there are often reasons for the differences, and usually good ones.

Hope that helps

I was able to remove most of Evolution.  It's hard to see why I shouldn't be able to remove it all, though.



Shared libraries.  That thing which is part of the basis behind all of the directories you don't like.  Little things like the C run-time libraries and the graphics run-time libraries are all dependencies for Evolution.  They are also dependencies for nearly every other piece of Gnome.  There is only one instance of them on the system.

If each application was in its own tree there would be many instances of these things scattered about a drive.  Downloads would be much larger, updates more difficult to synchronize, and let's not forget, each application would have to keep track of which one it was using.

We hit this problem back in the days of DOS.  32-bit DOS extenders creating overlays and trying to make everything execute in what was left of that precious lower 640K.  Every executable had every library statically linked into it along with its own set of overlay files.  Then the extenders started creating shared overlay files for application suites trying to reduce the number of 5 1/4 floppies needed to ship the thing and the amount of stuff to swap in/out of the lower 640K.

In order to keep track of what they were using, each DOS application had to have one or more symbols (environment variables) set up indicating where it should find stuff.  Pretty soon you ran out of environment variable space.  If you increased the space you took away more of that 640K.

From a development standpoint you would be burning cash and spinning you wheels pursuing that.  You would be much farther ahead to write an OpenSource gaming engine (world database, motions, etc.) which ran on both platforms.

Games come and go, but the game engines remain the same.  The biggest reason game developers don't release games on other platforms is the cost of developing a multi-platform gaming engine.

If you are as old as I am, you can remember that Quake and a few other big name games had game engines which allowed them to release on Windows and OS/2 simultaneously.  I believe one had a holy-trio engine which included the Apple platform of the day.  The platforms eventually moved on and when the engine had to be redeveloped, they developed for only one platform intending to "get back to it later", but that never happened.

Develop an OpenSource (or commercial) game engine supporting Worsta, 64-bit Linux, and MAC and you will move the gamers eventually.  Keeping an engine updated to support the new holy-trio will be a massive effort.  MS is killing off Worsta in a few months.  Apple will release yet another major design in a couple of years.  Significant kernel mods will occur in the 64-bit Linux kernel, especially now that Red Hat hired the guy who added a pathetic excuse for clustering to Tru/64.  Inside of 3 years the Linux kernel will migrate dramatically towards he OpenVMS platform because that is the only way to implement clustering.  Oracle, Red Hat, and a few other deep pocket players (OK, Red Hat not so deep) are pushing to add clustering to the Linux kernel so distributed databases can actually work.  (GRID has failed and companies need to distribute their database world-wide for fault tolerance.)

So, if you are serious about wanting to drag Windows users to Ubuntu or any other Linux distro, gather up the cross platform tool OpenSource developers and write a gaming engine supporting the new holy-trio.  If you poke around on-line you should be able to find old interviews and articles from the Quake engine developer where he actually described how they developed the engine.  The code won't help you today, but the design won't have changed much, simply the capabilities you can store in the world database.
