### Post by georgemc on 2010-06-26
As my journey with Ubuntu continues.
 

 Recap: Been in servers farms most of my career,  been using *NIX most of the time with a little MS server/workstations administration.  Cooperate communications and documents are mostly MS Office applications.
 

 Last year I resurrected my aging laptop with Ubuntu 9.04 and converted my main stay Desktop to Ubuntu 9.04 with Vmware and Vbox to deal with the applications that I use either on a daily basis or need every once in awhile.
 

 I use Vbox to run a Windows XP guest for Quick Books and Vmware to run my Magic Jack on an Windows XP guest, you just cant beat a $20.- a year phone line.  For the occasional document scan on my 10+ year old parallel port scanner I run W2000 in Vmware because Xsane doesn't support that make and model.
 

 All of this I was able to accomplish with out much fuss by searching these forums and time permitting  a daily scan and read of the help sections.  If you have an issue there is help available and you can actively post a help request, or it is already described and you just need to search for it and I did that many times.  I have yet to post any kind of help request.
 

 Now moving forward.
 I installed various networking tools and packages.  Networked all of my workstations with either Samba or just plain *NIX remote mounts.
 

 Used Vbox to setup and simulate a clients environment using guests of MS Small Business server with active directory and several XP Pro workstations.  What a neat way to simulate and setup an environment of  Networked server/clients.  Its a great tool to use for trouble shooting and learning.  Your only limitations are physical memory and your imagination.
 

 Being the Engineer and not the artistic type, I was able to created a Movie with fading in and out Titles of a dozen of home movies we shot of a family project that spanned a several months, and actually made NTSC and PAL DVDs for everyone to enjoy, I have family in Europe where they use PAL encoding.  Again searching and looking for solutions on these forums.
 

 Now the dreaded OS upgrades.
 As you read though all the issues posted you do get the feeling of OMG what will happen when the update support runs out and I am forced to upgrade the OS.  Recent events with other OS's tend to reinforce those fears.  And being an Engineer dealing with these issues on the job, I do know that any OS upgrade can go terribly wrong.  And yes if possible a new clean install is preferred.
 

 Since I started using Ubuntu I kept a record of the applications I installed on the various systems that deviated from the “factory defaults” so that when I upgraded I would be able to perform a fresh install and manually load the applications.
 

 It just so happened that I came across a post dealing with just that situation and it did help me formalize the upgrade process where A) I preserve my Home directories, B) perform a new clean install of the OS, and C) easily reinstall all of the applications I loaded using the existing tools and capabilities available.
 

 I pressed my “spare” desktop into the upgrade test bed.  I tested different upgrade paths and OS versions (9.04>9.10, 9.04>10.04, 9.10>10.04) and with the existing tools and capabilities I was able devise a path that works for me very well.  It preserves my customization, does a clean install of the OS, and leaves my data intact.  I put my plan in place and executed it on all of my systems with out any  issues.  Now all of my home systems, 2 desktops, 1 media center, and my aging laptop are Ubuntu 10.04.
 

 If you can take away any thing from this post it should be this:  The information you seek on Ubuntu is in these forums, and more importantly help is available for any and all issues you may encounter, all you need to do is politely ask for help.
 

 My sincere Thank You goes to the volunteers that make them selfs available and help, and to all the contributors that post the many tutorials.
 

 And the journey continues.


George
