### Post by 3rdalbum on 2009-01-21
Firefox offers binary downloads for i386 (ordinary PC architecture). Bad example, dude.

If the developer doesn't create easy-to-install packages, then it means that their newest work isn't for use by inexperienced users. I think you're greatly overstating the number of developers who don't package their software in some way.

The Linux method, of having source-code available to compile, is actually the better one. Linux runs on lots of different processor architectures (x86, x86_64, Itanium, MIPS, PowerPC, ARM, m68k, SPARC etc). If you compile a program for x86, it won't run on any other architectures except x86_64 (and then not in 64-bit mode). If you compile a program for x86_64, it won't run on any other architectures. If you compile a program for ARM, it won't run on anything else.

Windows only supports 3 architectures; x86, x86_64, and Itanium. Windows on Itanium sucks hard because every Windows-based developer pre-compiles their programs for x86, and therefore they won't run on Itanium processors. Windows on 64-bit (x86_64) is also a pretty bad experience because of the same problem - you might be running 64-bit, but everyone precompiles their programs for 32-bit!

Source code can be compiled on any processor, for any processor. If you're running a 64-bit Linux system and you want a new bleeding-edge program, you just compile it - and that way you get a native 64-bit program.

So, there you have it. It's certainly very possible to distribute x86 binaries for Linux programs, and lots of people and developers do it. If a binary isn't available, that means that the program is not ready for end-user use.

Also remember that in the Windows world, a program doesn't get released until it has reached version 1.0. In Linux, as it's all open-source, programs get released at every decimal point starting at 0.1. It's what they call "Release early, release often". But they are NOT READY for you to use until they are packaged either by Ubuntu or distributed with a binary installer.

2. Start Synaptic Package Manager
3. In search box type thunderbird
4. Click on checkbox next to package thunderbird

7. Click apply again.

1. Go to applications-> Accessories

3. type in "sudo apt-get install thunderbird"

The terminal is actually a vital component that will not be "phased out" and replaced with a GUI.  It's a very useful support tool and allows for things to be fixed cleanly and quickly.  Windows on the other hand has lot of built in config tools scattered throughout the UI or totally inaccessible unless you hunt down the .exe or run it from the command line.

For example if a Windows user has a problem, you have to tell them to "click this, click that, click the something tab, check that box, uncheck that box, click ok, click this, double click on that".

If a Linux user has a problem then it's a case of "what is the output of this:"

```
su
<command1>
<command2>
<command3>
```

It's a hell of a lot easier, faster and more powerful.  I support Windows users on a daily basis so I see it's many shortcomings constantly.

A lot of people base their criticism of Linux on the fact that it doesn't work like Windows.  Like anything in life you put in the effort and after a time you reap the rewards.  You had to learn Windows, that didn't take a week, the same goes for Linux.
