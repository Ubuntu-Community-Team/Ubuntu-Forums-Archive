### Post by Gausian on 2009-11-08
Yes, unfortunately Ubuntu has been downgraded to No.2 for me in favor of Windows 7.  From here on I will probably only be playing with Ubuntu as a virtual OS in VB.
My reasons are many but here are a few.

  1.Once Windows 7 is installed with all the drivers, I don’t have to worry about anything not working.  I’ve had endless problems in Ubuntu with wireless cutting out, sleep and resume not functioning properly and sub-standard power options and temp control for laptops.  None of these problems occur in any flavor of windows I’ve run.

  2.Software.  In windows I can without effort have the latest versions of software (like Firefox)running at my convenience.  Not so in Ubuntu. Skype is important to me since I live far from home.  Skype has never worked for me in Ubuntu.   In addition, a lot of the software in the repos is like freeware from the 90’s.
 
  3.Office.  Now OpenOffice makes a great effort, but to be honest MS Office makes it look like someone’s plagiarized Bachelor’s project.  Not to mention the endless formatting problems when dealing with files from people in the rest of the world who use MS Office.  Frustrating.
 
These are just some of my gripes.  I do think though that Ubuntu, or Linux, has a future, it’s just not going to be on my systems anytime soon.
Windows 7 has made some really nice progression compared to Vista, and has become the drop in the bucket that made me switch for good.

I’ll end with a quote I read somewhere.  “Use Windows if you like doing things with your computer.  Use Linux if you like doing things to your computer.”  This rings true with me.

### Post by Small_Nuke on 2009-11-09
1. Out of the box hardware support is actually pretty good in Ubuntu, but if your hardware doesn't work it can be tough to fix. Linux puts its best effort into making drivers for all the hardware, but the community can't code everything and also make it at least as good as locked drivers for Windows. The source of the problem lies in the fact that the hardware manufactures make drivers for Windows because it has the majority of users worldwide and they see that making drivers for Linux would be a net loss for the company since the user base is so small (their perception).
Your point is valid, but the finger should be pointed at hardware manufactures instead of Linux.

2. Use the same argument as above except substitute software companies. These companies do have a fairly good reason to not code for Linux however: risk of piracy, cost of support, most know that WINE can run Windows programs and put the task on WINE team, and time spent working with the Linux community just getting the program to run if they code it for Linux. The only software your going to see running good on Linux is open-source. I do agree that Ubuntu could do a better job with upgrading the software it comes with, but they have to take a much larger picture into account, and you, and every other user, can upgrade the software manually at risk of breakage.

3. Ok, I feel your pain on this one. OO.o is a huge pain to use just because of formatting. Hopefully MS will work harder to make open formats more usable, but for now using .doc or .docx is the defacto standard and it's one hell of a pain for OO.o users.

Overall Ubuntu is a decent OS. It just doesn't have a huge user base pushing for-profit companies for support. For me, it's relative small scale use nets me much more security than any other OS out there based solely on the fact that it's not a target.
