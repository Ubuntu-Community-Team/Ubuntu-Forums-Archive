### Post by Skripka on 2008-11-27
Sorry you've had a rough go of it--we all start out with a toughie sooner or later.


The only difficulty in terminal work for a basic Linux setup is copying and pasting, really.  It is no more difficult than that most of the time.  A little thought is sometimes needed, as to changing directories, but really-for Ndiswrapper it is a matter of cut and paste.


Ubuntu has NO SAY whatsoever in wireless drivers, there is really NOTHING Ubuntu or Linux can do about it-Ndiswrapper is NECESSARY as wireless card manufacturers _***refuse***_ to publish anything other than Windows drivers.  That is why we have Ndiswrapper-to fake Windows wireless drivers into thinking they are running in a windows environment.

Terminal windows do not show a confirmation of command, unless the command explicitly issues an output or there is an error.  No news is good news.  Every system does terminals differently--Mac OSX is the same way.


If you find Windows works fine for you, fine.  If you start feeling every bit annoyed at Microsoft and its mistakes as some of us do--you'll be back.
