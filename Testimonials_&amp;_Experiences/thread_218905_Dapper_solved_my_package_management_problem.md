### Post by Peter Mount on 2006-07-19
Hi

I've tried both Ubuntu 6.06 and Kubuntu 6.06 (as well as Xubuntu 6.06) the last few days. I prefer Kubuntu because I prefer KDE. But I'll say this about all versions of Dapper. Dapper has solved the problem with package management that I had with Fedora Core 5. With Fedora Core 5 it was like you could only easily install or update to the latest versions of packages. So if I installed LAMP it would be the latest versions of PHP5 and MySQL5. 

I needed to install PHP4.x and MySQL4.x. I found I would have had to recompile both of these to get it to work in Fedora Core 5 (at least that's what I read). I wasn't happy about this, especially with my old Pentium 2-350. It seemed like compiling such packages would have taken a very long time after I read that even on new computers you'd still need to take a few coffee breaks while compiling. I did read that it would be possible to use rpms from previous versions of Fedora Core but I also read the dependencies would take a very long time to sort out and download.

I found out that Ubuntu has ready made packages that I could download for my required version of LAMP. So I installed Ubuntu and tried the instructions at:

[https://help.ubuntu.com/community/ApacheMySQLPHP](https://help.ubuntu.com/community/ApacheMySQLPHP)

It worked. So I installed it on Kubuntu (on my other hard drive)

I also read you can use deborphan to remove orphan dependancies on Ubuntu. I hadn't found how I could do this on Fedora. 

One problem I have had with Dapper is an error message I got when installing, to do with not being able to create a file system. I got this even when I booted in safe video mode. I found when I started the install the second time the error message wasn't there. I don't know if my old hardware caused this but I'm hoping this bug is gone in the next release of (K)(X)Ubuntu.

Fedora Core 5 is still a great OS. If it wasn't for my need for a particular version of LAMP I'd probably still be using it.

I'm very glad I changed to Dapper. The package management seems so much easier to cope with and the help I've found online has been great.

Have fun
