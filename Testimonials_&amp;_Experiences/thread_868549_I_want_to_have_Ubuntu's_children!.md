### Post by sydbat on 2008-07-24
The longer I use Ubuntu (8.04 64-bit and 32-bit), the more I find it is **THE** OS for me.

I come from...come on...you know...yup...a Windows background. I had tried other Linux distributions (never inhaled), but found something...lacking. The GUI was always 'not-quite-right' and they relied on the terminal too much for Joe Average (the ONLY thing keeping most people from adopting GNU/Linux).

Then I found Ubuntu. It was an incredible experience. 7.10 32-bit for my test box...a PIII box with Fedora 7 and Windows 2000 running fairly smoothly along side each other. I had read much about the friendlier interface and higher speed, so I burned and installed. It was so unbelievable - an OS that made this old, tired processor, and the 486 MB of RAM run like the world was new again. The XP on my "main box" seemed sluggish and awkward by comparison (a box with AMD X2 4400 and 2 GIG's of RAM).

But I was still 'testing', still waiting to see if this Ubuntu distro would do what Fedora had done - bog down a bit after a few months of updates and configuring, needing a clean install of the next version to make sure it ran right (if at all...a completely separate story) and making XP attractive again. But it did not. The speed remained, no matter what I seemed to do to it. Then came the upgrade option in Update Manager.

I figured, what the hell, it is a test box after all. If it breaks, I'll just reinstall. So I took the chance and had a successful upgrade to Hardy (well, mostly...there are always a couple of things that need tweaking). I found, after the upgrade, that the opposite of what I expected happened - Ubuntu ran the same as before, if not a bit faster for some apps. Completely different than a Windows "upgrade" (or service pack). WOW!!

As I mentioned, I ran (and still have as a dual boot option) XP. XP Pro to be exact. My HDD partitioning scheme allows for a complete reinstall of XP if (really when) I need to, without disturbing any of my data. I had also created an empty partition for a future second OS when initially partitioning. This came in very handy when I downloaded and installed the trial version of XP 64-bit (which I found better/more stable than the 32-bit). However, this trial only lasted for 120 days, then asked for $$ to keep using it. I never used it as a main OS because I knew I could not afford it, but I wanted to know what my 64-bit processors could really do when given the chance to go full out 64-bit. Then along came Ubuntu (see above passages).

Because Ubuntu 7.10, then 8.04, ran/run so well on my old PIII box, and because I had an empty partition waiting to be used after removing XP64 (for several months), I decided to download 8.04 64-bit and see how it would run. HOLY @#^*! Ubuntu Hardy Heron 64-bit made XP64 look like a slow moving turtle about to get run over as it crosses a busy highway.

I know many people have had problems with certain things not working right in the 64-bit version (as evidenced by several threads on this forum), but I have not had anything bad happen...yet (that is a 'knock on wood' caveat just for me)...most things have 'worked out of the box' as they say (why do they say that?).

I have, and will continue to, recommend Ubuntu to people I know. It has turned out to be better for me (and the biggest stumbling block to new technology, my wife) than Windows ever was. Because we used FOSS in Windows to begin with, there was no major shift for either of us in adopting Ubuntu. It has been 3 weeks since I booted into XP and it looks like it will be longer still...hmmm...maybe I need an empty partition to put something in...

And this community is just the 'extra gravy and cheese' on the poutine! Mmmm...poutine...
