### Post by TB2 on 2007-11-01
Hi,

first off, two things I have to say about Windows XP:

**1.** I've been using Windows since 3.1 and I **really** found my way around all that stupid crap that it throws at the user. I've been tweaking my Windows XP installation so far that it doesn't really resemble XP anymore. Xplorer^2 for filebrowsing, theme patch, hundreds uf registry hacks and tools to "make it right", make some annoyances disappear and let some other useful features show up, because on every mandatory annual reinstall of XP the standart set of features made me throw up. But I really enjoyed my Windows environment in the end because it did what I wanted, and it did it fast. And btw, with nLite I can spare me a lot of that tweaking by only doing it once for every machine.

**2.** People like to talk about how horribly unstable Windows is and that it just crashes at random. This is definetly not true. XP runs very stable on any machine I've been using. And if there's a complete system crash, you can always turn on the Bluescreen and the error code information provided really helps solving the issue if you know how to use Google. It's also to mention that XP was the first version (besides of 2000 maybe) which provided this kind of stability, of course after SP1 and after a million patches. Oh and by the way: Even hard core windows users are hating Vista, yay... :D

And now about my Linux experience:

I tried it a couple of years ago, my first Linux was Mandrake and I found it was a bloated pile of crap. After that I tried a couple of other distros like Debian, Mepis, Arch and finally Gentoo. Gentoo was the first distro I liked and this being a total noob. Installing it took me 3 days but hell, that manual gave me some in-depth knowledge of the basics. What is fstab, what is X, what is whatever basic thing you need to run a Linux distro. I feel like it's the best thing for a noob to install Gentoo. After you manage to get a running Gentoo, you're free to use another distro... ;)

These days I'm using Ubuntu. I haven't touched the XP installation for weeks on my lap top and Ubuntu is such a great experience. If there's something wrong there are plenty of people to provide help and almost every problem is solved first on Ubuntu and maybe later on other distros. The speed, the wide range of useful applications and last but not least the freedom of not being bound to Windows are really making me happy. On my gaming machine on the other hand I haven't touched the Ubuntu I threw on for weeks, but the reason for that seems quite clear. But I'm really looking forward to Unreal Tournament 3, which has been officially confirmed for Linux. :)

Anyway, to come to an end and to the subject of the thread title: I was able to convert quite a wide range of people to Linux and this because of the simplicity and usability of Ubuntu. Ubuntu has provided a chance for every Joe Public in the world to install and use Linux. There are still a few quirks which prevent people from switching, like if some printer doesn't work, or the display doesn't work with the right resolution from the beginning so that people are simply scared off and running back to their fool-proof XP. But in my environment "Linux Days" or "We will help you install Ubuntu-Days" are popping out everywhere.
On one hand, this is a nice thing and reason to celebrate, but on the other hand I've met some people thinking Ubuntu = Linux, and not Ubuntu = "One Linux distro from a wide range of distros". The Ubuntu community is huge, and there is a big marketing machinery working on spreading Ubuntu as the "real deal". I myself am thinking of switching back to Gentoo, now that I have more experience to take that challenge, because I have the feeling that "the linux for human beings" is kinda trying to "take over" the Linux business. Of course Ubuntu is mainly targeted at Joe Public, and server administrators with a lot of knowledge might still use another more efficient distro, but "Joe Public" is in fact the biggest unexploited market for Linux. I really have no in-depth knowledge about anything really, but I fear that developers might fix their view more and more on Ubuntu rather than Linux as a whole. There are quite a few features that are stand-alone Ubuntu and not made for every Linux. As this is not unusual for distros, it might begin to hurt if Ubuntu keeps taking over shares. But as long as these shares are mainly taken from Microsoft, it's always a good thing. ;)
And just as general view of things, I just don't like it if _anything_ gets too big. Of course I have a lot of respect for what the Ubuntu developers have accomplished, as I said, it's mainly thanks to Ubuntu that I can actually convince some of my friends to switch to Linux, so Ubuntu is indeed a benefit to "all of Linux", but in the end, nothing comes for free and there shouldn't rest to much power in one hand.

But maybe all these worries only applie to my environment. I don't know anyone using a distro other than Ubuntu, and I guess in other circles there might be a lot of users of different distros. Maybe it's because we're all semi-noobs around here. ;)

Anyway I just felt like sharing my thoughts. I would appreciate if anyone would like to comment on this.

### Post by DMK62 on 2007-11-01
Hi

I also have worked with Windows for several years. Win2K and XP definitely increased the stability of Windows over other versions. On my home computer I only run Ubuntu ( more than one version ). My experience outside of Windows was primarily with Solaris on Sparc workstations in a testing environment. I fell in love with Unix then and I tried different versions since and have settled on Ubuntu as my OS of choice for now.

Although it would be great to see more users try or use linux I do not see it being a  reality anytime in the near future. Maybe with the current generation of kids growing up and being surrounded by technology all of their life. If you look back to the 90's and the introduction of the Internet to the general public there was a huge marketing campaign by the industry that computers were not that hard to learn and they made it appear that all you had to do was connect a couple cables and hit the power switch and you are off to the races. Not quite the case as we all know but people still expect that something should " just work " immediately. More and more things work in Linux now but there is still that barrier for many. Also the impression that it is just too difficult to learn. There was a nationally televised computer show here in Canada that recently did a show on linux ( their first attempt ). He did an ok job but his show is sponsored by companies with tight ties to MS and Apple. He said that programs are not easy at all to install and that he laboured for hours until he discovered that it helps to hit the Reload button in Synaptic.

I may get flamed for the following comment ( I hope you keep an open mind ). One of the dangers to linux that I see more and more is the adaptation of linux to cater to the users coming from MS products in particular. Although it does make the switch easier to some of them it is annoying to some users to have certain things installed and enabled by default. You can only go so far before it becomes bloated / buggy.

As far as one distro having a large part of the overall user base that tends to fluctuate over time. Ubuntu has been one of the top ones for a while but that can change. In your case you have considered going back to Gentoo and isn't it great that you have that freedom of choice ?

### Post by Johnsie on 2007-11-02
I dont think Ubuntu is too powerful. Ubutnu is only run on a small percentage of machines in the world. The fact that Ubuntu is open source means that people can take what they want from Ubuntu and branch it into other things. All Linux distros share alot of the same software so the rise of Ubuntu is good for the software they all use.

XP, yes I think it's a lot more stable than people let on. I think XP is better than Vista in terms of usabiliy and pretty close to Linux on stability. My problem with XP is that it easily becomes bloated and overloaded. Ubuntu has a few advantages like package management which makes it very easy for people to install and remove software and being almost sypware proof makes me less worried while surfing the net.

The biggest problem for Ubuntu at the moment is hardware issues. People are switching from Windows with obscure hardware and expecting it to work out of the box. Realistically that's impossible. Traditionally hardware manufacturers make drivers, not operating system developers. Hardware munufacturers need to start releasing open source or restricted Linux drivers so that Linux can support their stuff. The only way we can convince hardware manufacturers to support us is by increasing the number of people people who use Linux. At the moment you have to make sacrifices to use Linux but those sacrifices are rewarded by how well Ubuntu works on supported hardware.

### Post by Steve1961 on 2007-11-02
In general I think it's great that so many people have now heard of linux via Ubuntu, that oems are starting to take notice, and that Ubuntu, largely because of it's popularity, has just about the best community support forum around.  However, in the greater scheme of things the percentages are still tiny.  I spend my days visiting customers with broken windows machines and very few have heard of Ubuntu or Linux, and fewer still are prepared to try it - they'd rather pay me twice as much to reinstall windows.  So we're still a long way short of worrying about whether Ubuntu is getting too powerful.

Personally I've used Ubuntu since Hoary but I still play with other distros - Debian and Slackware being my personal favorites, with Suse (which I'm playing around with at the moment) coming third.  However, for the serious stuff (mostly work related) I stick to Ubuntu.  It takes less time to setup to my liking than, say, Slackware or Debian, and I've always found it incredibly stable (and I include Gutsy in that judgement).
