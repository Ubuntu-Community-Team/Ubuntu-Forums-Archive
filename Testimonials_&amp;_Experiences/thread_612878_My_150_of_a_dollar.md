### Post by xlr8ed on 2007-11-14
I have been using Ubuntu for a few months now, and while it isn’t my first Linux OS, I thought I would give my .02’s.

Let me preface this by saying that I am a total application fan boy, meaning that I could care less what OS the application I need, needs, as long as I can get it running and it does what I want, I’m happy. In my day to day life I generally need Apache, cron, Dreamweaver, Lightroom, MS Money, MySql, Outlook, Photoshop, PHP, rsync, Samba, Synergy, Tivo Desktop, VideoReDo, and Zune. The only piece of the software puzzle that I am missing is a centralized backup system (NOT BackupPC) that works on Linux and backups up both Windows and Linux clients (or if MS Home Server would just run the Linux apps I listed above.).

Likes:
It’s really fast and I would say that in day to day work, it just about keeps pace with my XP desktop. Compiz is wonderful and a really great little toy. 

Synaptic is generally fast and gives me a great deal of options when I am looking for something.

It’s a really well put together OS

Nice easy and quick install

More precise control over the look and feel of the OS

Dislikes:
Lack of customization during install. I keep an apt-get remove list to run the moment the install finishes.  While I know people like and use Open Office, I don’t need, don’t want it and it’s the first thing on the list to go, I would prefer to save the install time by simple not including it.

Real dual monitor support…I mean come on…it’s 2007. This should work with even the most basic drivers’ right out of the box…no need for command line anything. Even with the ATI or nVidia drivers, dual monitors mean just that, dual monitors…no more big desktop crap.

Administration control is a pain sometimes. My wife does not have the option of being an administrator on my machine, so if she had the option to update the machine, she should be prompted for the root password. Since I do have the option checked, I should not have to enter my password. I honestly do not feel this adds any level of security to the environment. If I can get access to your machine, it’s no longer your machine plan and simple. If I click on Screens & Graphics, Updates, Users & Groups or anything else on the Administration menu, just give it to me don’t prompt.

### Post by p_quarles on 2007-11-16
> **xlr8ed said:**
> I think you are missing my point about the security. For things like changing screen resolution, there should never need to be root access granted, the user should be able to change it. I've been working in Security for a long time, currently as a Security Engineer for the fourth biggest network in the world and I have yet to hear about someone rooting a box, Windows, Linux or otherwise, by changing the screen resolution. For some of the other things, I'll grant that they could be bad, while I could argue that they couldn't hurt, so we'll leave that one moot.
No, you certainly can't root a system by changing the resolution, but that actually is a root function for a good reason. 

When you use the GUI to change the resolution, you are essentially using a program that automates the process of editing xorg.conf, the video display configuration file. That file is relatively easy to break, and can be a pain to fix, if not outright impossible for beginning users. While the GUI program is *not* going to break it, it still has to be able to modify a file that is risky to modify by hand. So, it's essentially a compromise: make the GUI functionality available, but preserve the verbose text-based configuration system that many people, myself included, consider one of the greatest things about Linux.

Btw, it's pretty easy to give root privileges to any user you wish to. This way they can enter their own password instead of needing to enable the root account. (this command works for the default Ubuntu setup -- not for all Linux distros)
```
sudo addgroup *username* admin
```
The user will need to log out and back in, and will be all set.

### Post by CatKiller on 2007-11-19
Just a couple of quick comments.

It used to be possible (I haven't done any installs lately) to install the server version which came with no GUI and very few apps. You could then add what you wanted using apt-get/aptitude after install.

Dual monitor support is entirely in the hands of the people that make the hardware, until they release the specs enough to let more competent people do it.

Root access is generally required for things that will affect all users. Normal user privileges are sufficient for things that only affect that user. Changing the resolution of the desktop does **not** require root privileges. It is a bad plan to be logged in as root all of the time, but you can do it if you want.
