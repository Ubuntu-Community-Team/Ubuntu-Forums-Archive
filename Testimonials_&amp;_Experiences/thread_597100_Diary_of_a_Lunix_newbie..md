### Post by Metalslave on 2007-10-30
**Introduction.**
I have been a Windows user for years and years now and while I liked XP, I've become increasingly fed up with adware, malware, viruses, and general instabilities. I have been aware of Linux during all this time, but I have never actually tried it. Partly because it seemed complicated and partly because of the community (sorry guys) or maybe I should say *parts of the community* who have a tendency to come down hard on anyone who doesn't get it straight away. Oh, is this too complicated for you? Maybe you should stick to Windoze, then! You want to play games? Keep on feeding your M$ addiction, then! And so on. You know the type.

But I have been keeping my eye on Ubuntu and with the latest release (Gutsy Gibbon), I finally took the plunge and installed it on my second machine (the other one runs Vista) and these are my impressions. I'm not trying to (or want to) start an argument. If you disagree, that's cool, but keep it civil. If you can help, great, I appreciate and need any help I can get.

**Installing the Beast.**
The install was alright, apart from the fact that it wiped my XP install. I was under the impression that the installer would help me partition the drive, so I could dual-boot and it probably did, but I just didn't notice. I did keep an eye out for it, but appearantly it wasn't clear enough for me. No big deal, though. I have all my important stuff and gigantic MP3-collection backed up on an external Lacie drive.

**A new world.**
Getting used to the new environment has been fairly smooth. It all feels very familiar and also very new in a strange way. The file system took some getting used to. I don't feel like I have as much control over the harddrive as I did in XP (with Directory Opus), but that might just be because I haven't settled in quite yet. When I'm writing this, I've been using Ubuntu for only 4 days. 

**Confusion.**
Installing new software also works very differently. It works, I guess, but it feels very confusing at times. Case in point, I noticed on Blue's that ATI have new Linux drivers out (I have an ATI card). So I went there, but it was a little more involved than it is on the Windows side and I wasn't quite sure what to do.

How do I find out which version of XFree86 I have? The page said that if I wasn't sure, I could download something called "check.sh" and run it from a command line (which I'm guessing this is the terminal?) to find out. I downloaded it, but I had no idea how to run it in the terminal.  Am I supposed to know these things?

I wasn't sure that the above mattered, though, because there was only one set of drivers to download and they covered "XFree86 4.3 and X.Org 6.7, 6.8, 6.9, 7.0, 7.1, 7.2, 7.3". Do I need these or can I screw things up if I mess with it?

ATI helpfully pointed out that "the display driver requires POSIX shared memory to be enabled on the system. Kernel Source package is no longer required if Kernel Header package is installed". How on earth do I find out about POSIX and the Kernel Header Package?

In the end, I just didn't bother. The machine worked. I wasn't about to screw with it.

**Checklist.**
I started going through my personal checklist. I got the internet, I got mail, torrents work, I managed to make it play MP3's (Amarok), I installed Wine and got one or two of my old Windows apps working through that. I'm sure there are Linux equivalents to those apps, but I'm sticking with what I know, until I have learned more.

**Security.**
Security is a big deal on Windows, but there seems to be little talk about it in Linux. The Ubuntu website mentioned firewalls (firestarter) and some basic advice about email scams, but that was it. Some people told me that Linux simply didn't need things like anti-virus software, because the architecture and the way things worked here were so different from Windows. It sounds a little too good to be true, but maybe they're right.

**Progress.**
I eventually switched from Amarok to Rhytmbox and liked it a lot better. I hate the interface, but it hooks into Last.fm, which is nice. I missed my lists. My USB devices and my camera get along fine with Ubuntu and I started using Pidgin for my modest IM needs.

**Problems.**
Left on the checklist of things to sort out are my printer, scanner, 360 joypad, Wacom tablet, and some software. I did hook up my printer (HP LaserJet 1080) and Ubuntu identified it right away. This impressed me, until I realized that it didn't actually print anything, but I'll solve that one eventually, I'm sure.

**Games.**
I have to say it. This has been disappointing. I didn't expect much, but I expected *something*. Steam actually installed (through Wine), but I haven't tried to play any of my games there yet. The Dawn of War series started out promising, but it fell apart when the install asked for the second disc and Ubuntu refused to eject the first one, because it was appearantly still busy using it. I suspect the same thing will happen to Guild Wars, so I haven't even bothered with it. Eye of the North comes on the single DVD and should theoretically be fine, but the first two installments comes on multiple discs. Maybe if there's collection out there that shoves the games onto a single DVD, it could work. I don't know. Rogue Trooper refused to work, which is a shame, because I love(d) that game.

**Apps.**
Photoshop Elements quit installing, saying it wanted Internet Explorer on the system or it it wasn't having any of it. I didn't really expect it to work, anyway. I looked briefly at the Gimp and frankly, it looks like a pain in the ***, but it will have to do, until I find something better, prefferably something that at least approaches commercial quality. I haven't tried installing Manga Studio EX yet, but it's another one of those that I don't really expect to work with Ubuntu. Also, it comes on several discs.

**Conclusion.**
This Ubuntu box feels solid and reliable. There was one lockup where it somehow got confused by my external drive and wouldn't display it's contents anymore. A few seconds of icy despair (OMG, the **** wiped my harddrive, NOOO) and a reboot later and everything was fine again, however. :D

There, I'm done.

### Post by 3rdalbum on 2007-11-05
To be honest, I generally advise against trying to shoehorn Windows games onto Linux with Wine. Some people get it to work, but you'll be constantly nudging the limit of what Wine can do. And you'll jump through some hoops.

But of course, there's NO way of running graphical Linux programs on Windows; so it isn't a case of "Windows is better", it's just that Windows programs are built to run on Windows.

I'm not sure what sort of "art software" you're after; could you please describe what sort of things you want to do? Yes, The Gimp's interface does take some getting used to, but it's not much more of a jump than it was between earlier versions of Photoshop.

You'll basically want to ignore what was said on the ATI website. It's an example of a self-fulfilling prophesy - ATI says "We don't need to make our drivers easy to install, because Linux users are all hardcore geeks", so their drivers become too difficult to install for the *ordinary* Linux users, and some quit out of frustration. The good news is that your graphics card is working out-of-the-box - the Restricted Drivers Manager has taken care of that. See? An Ubuntu-developed utility can make the drivers a piece of piss to install, but ATI doesn't care to put that sort of time and effort in. ATI aren't the only ones.

As for viruses and malware, what you've been told is true. Just think about how much hassle it's been to run legitimate Windows programs in Wine - so there's little chance that viruses could successfully run in Wine. And as for native Linux viruses, think again - effective security policy permeates the entire operating system. When you tried to run the "check.sh" script, the reason why you couldn't is because you hadn't given it executable permission yet. A program cannot run until the user explicitly gives it permission to (through Linux's permissions system).

Yes, it's a learning curve, but ultimately it will be a satisfying switch, and you will come to appreciate the Linux and Ubuntu way of doing things.
