### Post by jebsector on 2007-12-07
[COLOR="DimGray"]
First off, I have been running ubuntu for the last several months after a windows virus shut down my system.

I want to be clear in that ubuntu has impressed me quite a bit.  I show off the neat features to my roommate, and people get to see all the neat stuff I can do when I sometimes work from home and connect to my unix/linux servers.

I have a couple of ticks, though.  When I clicked to upgrade from v. 6 - 7.1, it did not complete successfully and I had to do a full re-install. 

After the install, my periferals did not function properly, namely usb drives and flash memory.  I see this is a common problem.  It was alleviated after installing restricted packages (I am guessing here) from the package manager.

Another peeve was that flash would not install properly in the package manager or the firefox update.  I had to manually install it by downloading flash separately.

Another, cannot get .bin files to install from the command line.  Seems to work if I double-click on the file, though.  I also cannot get self-extracting files to run by chmodding +x it and then running a ./[filename].  It says ./ is a directory instead of running the script.  

Another, compiz is fantastic, especially when you download the preference tools for it, but a change I noticed that I am no fan of is the fact that it tries to stick to the edges of panels and screens.  This is really annoying to me.  The movement of windows is fine in compiz, but I would have preferred simply not allowing a drag to end up past a panel (making it unreachable), namely the upper panel's edge.  Past that, don't know why you would need to snap a compiz-enabled window to an edge, because it ends up stretching and not moving anywhere.

For the install procedure, it would be great if there were a failback, just incase something went wrong during the upgrade to the next major version of the operating system.  A failure here causing the system to be inoperable is not a good side effect, but it is very nice that the option to upgrade is there, without having to do a full re-install.

Now, to the gimmes.  

UI - The look and feel is fantastic now.  Compiz helped me out a lot, and for those that don't like animations, well, the option is there not to use it.  Like the ability to freely add or install user interface customizations in gtk, will probably make some ui's while I'm not doing anything.  The windows user (my current roommate-who is a windows fanatic-used to sell them, said they tried to sell redhat back in the early 2000s and nobody would buy them, but hey, linux has come quite far in those years) especially seemed to notice the ability to freely change the ui, as well as some pleasing animations, and especially the compiz ability of looking at thumbnails and choosing windows that way - (the pick windows feature in compiz) - particularly he said 'that is cool'.  Which is exactly what I want windows users to be caused to say.  But that reminds me, the screensavers gnome-screensaver is useless.  There is no options for settings, even for the individual screensaver.  Somebody needs to write a mod here-my screensaver is embarrasing.

I have tried the other UIs - KDE, Xfce.  Appreciate the functionality that comes with KDE, like the easy Xfce customization options, but Gnome is the winner for me overall.

Additional software - installing software can be a chore, especially when downloading software from the web and installing it.  Most non-computer savvy users will want nothing to do with the shell, sorry to say.  However, the option to use package manager and the installation/removal application are fantastic.  It is a simple way to install open source software, and it's like having an advising directory right there for you.  Great.  It's also nice that I can install what I need without forking over my left-arm for it.  The availability of software makes this system kick-a$$, however I would have gladly paid a little for each; someday I will have to give something to the software I enjoy using.  Throw a buck into the kitty.

Gaming software - all the best stuff is only made for windows.  Shame.  Good thing wine concentrates on running video games.  Cool stuff.

This is just my opinion on this system.  Overall, its very impressive-not at all what I had in mind.  I used fedora in college a couple years ago-was excited to try out linux but very dissapointed.  It was slow, lacked excitement, didn't altogether work right, difficult even for network admins to use, and the user interface was a flash-back to the late 80s.  But Ubuntu has provided a linux system that is just outstanding.  It's awesome that you can just plug it in, and it simply works.  Not a whole lot of setup here.  Security is just fine (hey its been running longer without security flops than it ever did with windows instaled).  The user interface is pleasant, wide availability of software, great features, etc.  

I think this is something I can start showing off as a linux machine.  Heck, the user interface is about as cool as the mac.  Well, the mac has some smoother minimizing algs and the panel is a little cooler, but as far as I am concerned it is pretty close.  I think this is finally starting to become a viable alternative, and that is outstanding.

:guitar:
[/COLOR]
