**Who am I: **
27 year, first computer = Vic 20, first PC = 8086. Soon to be employed as embedded real-time application developer:D in a linux enviornment.

**What I use the computer for:**
email, internet, calander, programming and play(=poke around different aspects of a computer just because I love it)

**What I like about linux:**
[LIST]

[*]It is highly configurable(with good default values provided from ubuntu)
[*]The interaction between gui and konsole provides a superior tool for solving complex tast at a minimum of effort.
[*]Package-managment is superb tool. I can (un)install software easily and the best of all I can try MANY  applications just for the fun of it.
[*]It gives you tools for daily usage(email, internet, office suits etc) but also has top quality developing tools(after all linux has mostly been a developers toy until recent).
[*]The development is ongoing and bleeding-edge is at your fingertip. But most important I think that the community and developers provides innovative solutions. This can not be stressed enough.
[*]The kernel is such a nice base that its usage is widespreed thrue different enviornments: embedded devices -> home computer (one tool to rule them all)[/LIST]**What I disslike about linux:**

### Post by aysiu on 2006-04-23
This was a well-thought-out critique of desktop Linux in general, and I agree with almost all of your points. [QUOTE=GoldBuggie]
Automatic Mount/Umount needs to be perfected(some effort has been put into this but not enough). I would like to plug in my flashcard and copy/delete some files. Then directly unplug it. Unfortunalty this sometimes gives error as "not mountable" or "in usage" or worse yet the write cache etc even after a umount gives corrupted mp3 files on my mp3-player.[/QUOTE] This one I only half agree with. I basically think it can be improved, but not for the reasons you say. It can be improved because when it's working it works just fine, but when it's not working, it's a disaster.

It works fine for me, but I've seen numerous posts on this forum indicating it doesn't work equally well for everyone's USB devices.

That said, I don't see--when it **is** working--how it's any different from OS X or XP. I actually unmount or "safely disconnect" devices in OS X and Windows, too. I don't just pull out the USB drive. And I've gotten that "cannot disconnect--busy" warning from Windows, too. 

In fact, in this respect, I think Linux has a one-up on Windows. Maybe I just don't know what the DOS command is, but if I can't safely remove a drive in Windows, my only two options seem to be:

1. Just pull it out anyway, possibly damaging data
2. Reboot the computer and pull it out during the shut down process

At least in Mac or Linux I can type ```
sudo eject /dev/sda1
``` or whatever if it's misbehaving. And I much prefer right-clicking and selecting "unmount volume" to finding a little green and white icon in the system tray and selecting which device to "safely remove."
