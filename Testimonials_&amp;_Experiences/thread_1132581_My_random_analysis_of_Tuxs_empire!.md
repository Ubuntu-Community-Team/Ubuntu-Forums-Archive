### Post by =^,^= on 2009-04-22
Ive been using Linux for about 3 months now, mainly in the form of Kubuntu.
In this time I think I have had enough time to pass an opinion on the pros and cons of this FOSS world I find myself in!

Most of the positives of using this non Microsoft/Apple product seem only make themselves known after a considerable while. For my experience it was about a month. I seem to be lucky in the fact that I chose an Ubuntu derivative as my introduction. It seems pretty widely accepted that this distribution is by far the most user friendly. I may have had less positive experiences if I had gone with another version.

The painful negatives are however the first thing new users will experience. Microsofts total domination means that a lot of the so called freedom afforded to me by using a GPL licensed OS are hard to find if I want to use things such as the many streaming media services that rely on Microsoft DRM. This alone would be enough for many to return to Microsofts cold embrace!

Gaming is something that is also something that is either painfully difficult compared to a MS product, or just plain lacking all together.

The sheer fragmented nature of Linux, means it can never really get anywhere. Its like the some huge well organised army versus a collection of confused infighting tribes.
Even if by some miracle Linux could gain a decent share of desktop use, developers would still have to compensate for RPM/DEM/TAG.GZ based distros. Therefore increasing development cost. I know the are RUN and BIN, but they have there own problems!

Anyway this post has turned into something different than I meant it to be, so I will just leave it here!

### Post by Ozor Mox on 2009-04-22
Good to hear that you have had a generally positive experience with Ubuntu. Thought I'd address some of your points though.

> The painful negatives are however the first thing new users will experience. Microsofts total domination means that a lot of the so called freedom afforded to me by using a GPL licensed OS are hard to find if I want to use things such as the many streaming media services that rely on Microsoft DRM. This alone would be enough for many to return to Microsofts cold embrace!

This is true for the most part. I have been using Ubuntu as my only OS for more than two years now, and I have once or twice come across a situation where a bit of hardware or software is not available on Ubuntu. Trouble is, as a minority OS we have to accept that for now this is going to happen, but I believe that hardware/software companies are slowly realising the importance of Linux and releasing drivers/codecs/software for us. Not to mention of course the amazing job that some Linux developers do on reverse engineering hardware where the company will not even provide specifications, let alone drivers. Compared with Windows where the OS market share is such that *everything* is released for them, and Mac where the platform is very limited so there is no driver problem, Linux could be seen to be at a disadvantage. Linux is so versatile though, I'm sure it could run on a toaster with a bit of effort ;)

That was a bit of a waffle, but generally the solution I use is:
Hardware: Buy pre-installed Ubuntu or other Linux, or check the hardware is supported before buying.
Software: Run a virtual machine with Windows in it if no Linux native equivalent.

I can probably count on one hand the number of times I've booted my Windows XP virtual machine however...

> Gaming is something that is also something that is either painfully difficult compared to a MS product, or just plain lacking all together.

I think there are loads of great games on Ubuntu, and I play them without experiencing any pain, so I'm going to disagree with it being "painfully difficult"! In terms of commercial gaming though, I'm not going to disagree with it lacking at the moment. That's one reason I got a PS3.

> The sheer fragmented nature of Linux, means it can never really get anywhere. Its like the some huge well organised army versus a collection of confused infighting tribes.
Even if by some miracle Linux could gain a decent share of desktop use, developers would still have to compensate for RPM/DEM/TAG.GZ based distros. Therefore increasing development cost. I know the are RUN and BIN, but they have there own problems!

This argument comes up very frequently and while I understand where people are coming from, I think it's a fundamental misunderstanding of FOSS. Think of FOSS as organic, and its growth as evolution. Sure, open source projects need someone to direct them (and probably the most successful do have good direction), but the code is free and open and all distributed changes must be published (if under the GPL, for BSD the code is free but changes aren't forced to be published). This means that all of the various distributions of Linux can benefit from one another should they choose to. Also, Linux can be literally anything you want it to be. A server, a modern desktop, highly customisable, easy to use, built from scratch, using the latest technologies, fitting on to a floppy disk, running a supercomputer... What if everyone was forced to build the exact same thing? It would be a one-size-fits-all behemoth that didn't do anything well, was chocked full of features and bloat, and the developers would never stop arguing about the right direction to take it in. Instead, the distributions work by natural selection. Who says some can't rise above the rest? I could pick out the top distibutions with ease (Ubuntu, Fedora, openSUSE, Debian, Slackware, Arch, ...), but the rest are there because Linux is all about choice. The same principle goes for whether rpm/deb/tar.gz/run/bin become the standard packaging format... It will all depend on the format that the most successful distribution(s) are using, which at the moment is probably a combination of rpm (Fedora, openSUSE) and deb (Ubuntu, Debian).
