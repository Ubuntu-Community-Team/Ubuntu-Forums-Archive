### Post by 3rdalbum on 2009-02-02
Dude, you're comparing moving between point releases of Windows XP, to moving between major releases of Ubuntu.

Also, if you think it's easier to solve problems in Windows, you obviously haven't looked at the Microsoft Knowledgebase. It seems like every solution in Microsoft Knowledgebase involves going to an MS-DOS prompt and copying in "rundll32" commands that are designed to be incomprehensible to anyone who isn't a Windows programmer; these commands end off wrapping over three lines, because they are actually C function calls!

Windows has the benefit of only really running on one architecture of processor, that's why all Windows programs are installed pre-compiled. Ubuntu (and especially Linux) runs on a number of different processor architectures that do not understand eachother's instructions (e.g. i386, ARM, SPARC, Itanium, PowerPC) , so source code is the only thing that all the architectures can work with.

Windows used to support the Itanium architecture, with disasterous results. Windows-based developers only ever distributed i386 binaries which Itanium users could not run. Being able to compile everything from source code is why Itanium-based machines generally end off running Linux, and it's also why Linux's 64-bit support is so much better than Windows'.

Compiling can be made easier, and definitely should be made easier, but you've tried to fight something you don't understand there.
