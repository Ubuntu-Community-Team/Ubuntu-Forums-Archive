### Post by El Bartolomew on 2011-12-28
Linux rulez / Linux is safer / Linux is faster
or 
Ubuntu : Fast, secure and stylishly simple

These are nice slogans but Linux how does it perform in real life?

I decided to update Kubuntu 11.10 on 1 of my computers last week. 
I was forced to do a hard reset because Muon kept hanging on 55%!
When checking Google I found quiet some bug reports from people with the same issue... 
Indeed Linux starts to beat Windows on this point. I actually never had with XP such serious issues.

Today I decided to install Kubuntu on another computer with another hardware config.

I went for Kubuntu 10.04 LTS because I do not like upgrading every 6 months. 
I ran in some serious issues when updating, the total system kept hanging. I already was surprised to see over 170 security updates. 
As I am also used working with XP I immediately felt at home, I thought, maybe this is the purpose of these Linux guys. You have to make Windows users feel at home.
 
I decided to go for version 11.10 because new version should be much better, but the fact was, new version new and above all, more bugs.

I really like the new installer of Kubuntu It is without a doubt one of the best on the market (and I tried almost entire Distrowatch).
But when you have an installer where you can select the option 'to download updates when installing Linux', I actually do not expect when rebooting after installation to have a bunch of updates ready for install. This means the installer is not working.

So when you decide to install these updates, and Linux is so good, you do not expect Muon to keep hanging a second time on another pc with the same problem.

So I decided to go for a second install ... and a fatal error occurred when installing Grub.

Install 3, all went well but again same issue with Muon ... keeps hanging on 56%

Personally I am wondering if Linux users are growing arrogant. When you mention this kind of stuff I always receive remarks like; 'I never had such issues, Linux works perfect with me'. The problem is, Google proofs different.
(Btw, I had to use Google to get to Ubuntu forums, what is wrong with Mark Shuttleworth? Is the word Forum to difficult to put it on the Ubuntu website? I do not  like reading pages full information before I get to the important stuff)

- KDE is nice but still some serious issues with ergonomics from version 4 till 4.7.
- Ubuntu with Unity which is cute but not appealing and also lacks ergonomics. It's design is nice for my Grandma if she would not have been blind.
- New Gnome version which pushes their uses such as Linux Mint into developing their own Desktop Interface.
- Another split at Openoffice, beside Libreoffice we have White Label Office (Maybe there are still some ego's left at Openoffice to start a third or fourth spin off)

After all these years I have the impression Linux is going backwards instead of forwards.

Personally I think developers should rethink the essence of their work. No bugs, safe, ergonomics and above all the stuff must do what it is designed to do before released into the wild because my personal impression is that Linux no longer walks the talk.

### Post by QIII on 2011-12-28
Why all the updates?  Because you get a stream of continuous improvement not only from the OS distributor but also from everyone who develops the software you have.  Realtime improvement.  

Why so many updates when you first install?  Because the installation iso for distributions like Ubuntu is fixed at release time.  See previous paragraph.

Why split OpenOffice?  So it could remain open source as LibreOffice before Oracle had a chance to make sure you had to pay out the nose for it.  Why split further?  Because someone else wanted to do something with it.  Homogeneity is the playground of monopoly.

Don't like Desktop Environment X?  Then use Y.

Don't like distro A?  Use distro B.

If you ask me, that is what is RIGHT with Linux.

It's actually sort of nice to walk where you want and not be forced on to someone else's path.

And if you are waiting for a "no bugs" condition, pull up a chair and grqb a bowl of peanuts.  You'll be waiting a while.
