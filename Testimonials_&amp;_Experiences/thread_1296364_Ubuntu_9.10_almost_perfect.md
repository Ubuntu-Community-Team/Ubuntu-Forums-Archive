### Post by 3rdalbum on 2009-10-21
9.10 so far has a whole bunch of broken features and broken packages:

1. Totem's new "Create Video Disc" function does nothing (works upstream, but not in Ubuntu)

2. Tangerine (DAAP sharing server) doesn't work; it complains of a missing DLL

3. Rhythmbox's DLNA plugin won't enable unless you manually install "python-coherence". Even after you install that package, DLNA still won't work. This hurts; there's virtually no media streaming on Ubuntu 9.10 with the Rhythmbox and Tangerine bugs.

4. Dragging files to a Nautilus window to decompress them will crash Nautilus. Fixed upstream, but it's not in Ubuntu yet.

5. FFMPEG does not have AAC encoding support, even if you install the "libav*-extra" packages. This is due to uncertain licensing of FAAC, but it still hurts because iPods and Walkmans (and probably lots of other portable media devices) accept only AAC audio tracks in their videos.

6. How the heck do you disable Apport? Whenever I unsuspend it reports a kernel problem. I can't report the problem because it claims that the Linux Image package isn't an Ubuntu package. I just want to disable it now, but I'm scared of touching anything in BUM because I know the Ubuntu devs have gone over to Upstart jobs.

I still like Karmic besides these problems (and a couple of other minor bugs), but I'm *really* hoping these issues get sorted out straight after release, because I'm not sure I can live with them for 6 months.

On a positive note, kudos to all those people testing the beta; all but one of the bugs I've tried to report have already been reported at Launchpad. That's some impressive testing from all involved.
