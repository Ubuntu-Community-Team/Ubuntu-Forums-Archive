### Post by stevenwscott on 2007-11-16
*....Ahem...(clears throat, steps up on soap-box).....*
Reality check, folks: Yes, Uber-geeks will swear that Linux rocks, and it does. The question really is, is Linux ready for High-end multimedia production? I will be the first to say no, having attempted to use open-source software for audio production, MIDI sequencing, triggering, and virtual synthesis. Yes, people will swear you can do it, and open-source projects exists covering the major areas, but you need some fat hardware and a team of geeks to tweak out the implementation on your particular hardware, and ultimately, it's never quite the quality of the commercial products available.
       Having said that, the offerings on Linux still give them a run for their money - I actually like using GIMP better than Photoshop (though it needs some maturity in the cache/internal image handling dept), and as far as I'm concerned, MPLAYER/MENCODER beats the pants off of a lot of similar products for windows. 
    Bottom line is, If I'm producing media professionally, I'll probably be using professional software to do so, so you may be expecting a bit too much from Linux.
*....(steps down from soap-box, put on propeller-hat)....*
  You're command prompt issue is because your X config is goofed up after the driver switch. I don't know what level of experience you have so I'll just give you the basic steps.
1. Issue the command "ls /etc/X11"
2. You should see a several xorg.conf files, some with numbers after them. The xorg.conf without the numbers is the current configuration for X, the ones with the numbers are backups, the first being 1 and the most recent being the highest number found. 
3. copy the highest number config file over top of the current, non-working config file by issuing the command "cp /etc/X11/xorg.conf.n /etc/X11/xorg.conf" where the n in xorg.conf.n is the ighest number found.
4. reboot with the reboot command. Yes, there's a way to recycle X, but I don't remember off hand so I'm cheating. :)
5. If that doesnt work, drop back to the next highest number and repeat the steps. Eventually you should get to a working configuration.
6. Have a beer, bask in your success.

SWS

### Post by sirdilznik on 2007-11-29
I'm glad you're back with Ubuntu.  Honestly video editing/production is one of the biggest weak spots in the Linux repertoire right now.  It can be done but the level of available tools (and the ease of use of those tools) is nowhere near the level of the tools available on Windows or Mac.  It sucks, but it is what it is.  Linux has made tremendous strides in all sorts of areas (including video editing, not too long ago even the few tools we have didn't exist) and I'm sure this particular area will continue to improve as well, but for now we're stuck with what we have.

From reading your original post, whoever told you to go with GNOME for speed needs to lay off the wacky stuff for a bit.  I'm not going to go into a GNOME vs. KDE flame war here, but the fact is that both of those DEs are at the bottom of the speed spectrum when it comes to Linux.  If you want something faster you should check out Fluxbox, Openbox, or Fvwm.  Of course these window managers are nowhere near as user friendly as KDE or GNOME and will require "getting you hands dirty".  A good "middleground" compromise is Xfce.  It is still relatively easy to use and comes with a decent set of tools/apps and is significantly lighter and faster than either GNOME or KDE.  The great part about Ubuntu (and Linux in general) is that you can try any of these out at any time and if you don't like them, switch back to your old desktop.  In Windows and Mac you're stuck with what you've got and in Mac you can't even switch the theme.
