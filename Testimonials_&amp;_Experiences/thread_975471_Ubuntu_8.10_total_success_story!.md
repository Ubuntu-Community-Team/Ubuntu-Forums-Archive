### Post by pablolie on 2008-11-08
i see quite a few threads going into problems with 8.10. i for one must say that the installation of 8.10 was far easier onto my machine than 8.04 was. i am now finally running the 64 bit version, and that of the standard installation disk just hitting "install"... last time around it was far more of a struggle, and i had to settle on the 32 bit verion.

this is a Shuttle XPC SG33G5 with an Nvidia-based Asus EN8500GT Silent graphics card, an Intel E8400 Core Duo, and 4GB of DDR2 RAM. this little system flies. even the Nividia driver which causes so much discussion updated itself to (admittedly the Nividia proprietary, but Ubuntu developer tested) version 177 basically by itself.

congratulations to the development comunity for making this more and more usable. in fact i am going to take this system into the office to work on it and try to utterly do without Microsoft, and evangelize that around me. 

a neat utilty for ubuntu would be something that monitor CPU temperature, i have not yet found one that truly worked with my computer.

anyhow, just a success report and a "thank you" from the trenches. i find 8.10 the best flavor of ubuntu yet!

### Post by tjwoosta on 2008-11-08
> a neat utilty for ubuntu would be something that monitor CPU temperature, i have not yet found one that truly worked with my computer.

i used to have a temp monitor on my gnome panel that was pretty accurate

it could monitor cpu temp as well as hd temp

```
sudo apt-get install sensors-applet
```

then reboot

after rebooting do this to configure the sensors 

```
sudo sensors-detect
``` (read everything carefully)

now you can add the sensors applet to the gnome panel the same way you would with any other applets

or you could simply type "sensors" into the terminal (without the quotes) to get the temps
