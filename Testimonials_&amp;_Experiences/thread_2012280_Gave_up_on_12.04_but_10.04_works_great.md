### Post by houseworkshy on 2012-06-29
Ubuntu 10.04 is very good, and it will continue to have support untill 2013.04 as it was a long term support one. The long term support versions have now got five years support which is even better but Ubuntu is getting heavyer and more bleeding edge as it is now aimed at newer machines, especially tablets.   If you are not intending to get new hardware soon you may want to have a look around for something else next year.    Debian is excellet for older machines and runs very light, it also has gnome 2 which is the same user interphase you're using in 10.04.  If you go for the stable version it will change only very slowly and bugs will be very rare indeed, they try not to have any at all and only release when it's right no matter how long that takes, in debian stable really does mean stable.  The next level is unstable which is mostly ok and bit more up to date but will have the occasional glitches, Ubuntu is based on debian unstable, then testing where rather a lot needs fixing, then experimental which is for development pioneers. It is a bit more of a fiddle to set up but not much and once done that's it, if you stick to stable updates will never break the system which is nice.  A novice level distro has been made which is based on debian stable, mepris [http://www.mepis.org/](http://www.mepis.org/) The default GUI is KDE though one can change that if one wishes. I've been using it as the master system on my ageing machine for a while ( until I'm ofey with debian I'll use a debian set up by people  who are and learn from that )and it runs amazingly light for a KDE desktop and I've had no problems with it at all. Meanwhile enjoy Ubuntu 10.04lts to the last dregs of it's funcional support, I regard it as one of the all-time classic opperating systems for a desk/lap top, suspect I'm not alone in holding that opinion.

### Post by houseworkshy on 2012-06-30
There are many alternatives.  One thing which is nice about linux in  general is that all most all of what one learns in one can mostly be  used in another.  Especially within the few linux lineages Slackware and  Arch or Debian and Ubuntu for example.  Package management varies as  does where things are in the various GUI's, but that's pretty trivial  and usually find-out-able within a few minets.  The command line and  it's excellent built in help is never far away anyhow so even if one  does encounter some dreadfully none intuative  gui one isn't quite lost  if fairly familliar with that. 

To access the built in help for the command line simply open a  terminal and enter "man" in front of whatever command you are interested  in.  So for more details on this just enter "man man" in a terminal.   When feeling around not knowing much I found that "apropos" was a very  handy command.  One can open a lot of terminals so one way of doing  things is to have one for the manual page and another for whatever one  is doing, one can copy and paste between them too.

There is some excellent documentation for Ubuntu, I'd recomend going through this first ;- 

[https://help.ubuntu.com/10.04/index.html](https://help.ubuntu.com/10.04/index.html) 

As it gives a brief overview and contains trustworthy apt;urls (  another way of installing, which one has to be cautious of elsewhere )  By the time one has worked through it one has had a good brief overview  and mostly set up ones system plus put in some of the programs you like  the look of.  Could easily go through the whole thing in and hour or two.  Time well  spent.

This is a very good and free downloadable manual.  
[http://ubuntu-manual.org/downloads](http://ubuntu-manual.org/downloads)
I  like to have a few guides on the drive for reading offline, especially  if ever faced with the problem of not being able to connect.   Remember having that problem with one of the microsoft products once,  clicked on the help and the networking help was online! aaarrrhhh!.   

This one is a bit more high end and not for now really  but superb should you ever want to be a command line guru.  
[http://sourceforge.net/projects/linuxcommand/files/TLCL/09.12/TLCL-09.12.pdf/download](http://sourceforge.net/projects/linuxcommand/files/TLCL/09.12/TLCL-09.12.pdf/download)

When  the support runs out for 10.04 if you still can't get 12.04 on or don't  like Unity and want to go hunting this is my favorite site for distros  and links to reviews 
[http://distrowatch.com/](http://distrowatch.com/)

Have fun
