### Post by MakOwner on 2011-12-06
The only constant is change.
That said, I don't dislike Unity just because it represents change - The design of the interface just doesn't work for me - for lack of a better way to put it, the work flow through the interface is just a broken concept for me.  The closest paradigm I see to it is (roughly) the MAC OS interface.
If I liked that interface, I'd be using a Mac. And yes, I have one or two and I have to work with them sometimes but for me it's not something that works well. Probably a big reason for that is that I don't find the need for floating-cube-3D-wonder-wheel workspaces on my computer screen to be of much use to me.  I get along fine with a 2 or 4 workspace pager and quick access to the application I use the most.  Digging through menus for an application or function is annoying.  I don't even like the fact that in the current iteration of KDE I have to dig through four menus to find the shutdown button.  

I have a lot of (to me) logical issues with the way the Unity desktop works. I have been using Ubuntu for a long time. Just anecdotal evidence leads me to believe that I'm not the only one that wanted to see a more stable 10.10 rather than start over with something less.

There are other variants out there, thank-the-spaghetti-monster.  
I'll wander off in search of a configuration or some variant that I can modify to meet my needs until such time as something changes.  If things make a turn for the useable maybe I'll try again.

Either way no one has to listen to me yelling "Get off my lawn!" which at the end of the day pretty much what the whole thread is, I suppose. :lolflag:

Out of 3 of the packages you listed Ubuntu Tweak and Myunity (don't know how it is different from CCSM's Unity plugin  really) are both third party tweaks . If those count you can aslo include that thingy that let you put the Unity bar at the bottom. :)

I guess a problem is that the configuration options are scattered all over the places (Gnome-tweak-tool, Gnome-tweak-tool extensions, CCSM, gconf-editor, dconf-editor) and neither of them are  even installed by default. That makes configuring confusing for new users and even more experienced ones. In Natty at least they all appear in "System Settings" if you install them but not so in Oneric.

You don't make a DE more friendly to new users by making it difficult to configure (because the options are either hidden or not installed), but by making the default as sensible as possible to minimize the need to configure, and if the need arise, make the options organized and easily accessible.

Right now the default configurations sometimes don't even work (e.g gnome-search-tool, LibreOffice quick start tray icon) and the tools to fix them (rather easy) are not installed. "Zero configuring experience" only makes sense if the default is so nice that there is no need to configure, it is perverted to try to achieve this by removing or hiding the options to configure.

Incidentally, I have no complaint about speed. 11.10 may be very buggy but it is very fast (11.04 is fast too) Speed has never been a problem with Unity for me.
