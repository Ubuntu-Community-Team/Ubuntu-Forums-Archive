### Post by Scarlett on 2007-01-10
A few months ago, on a whim, I decided I was going to finally make the switch to Linux because I was sick of Windows.  It kept wanting to phone home, it had massive vulnerabilities, my anti-spyware and virus and firewall all needed renewing and their logo isn't nearly as cool as Tux.  Who can resist that cute little penguin?

I intended to try it from a live CD but it was missing a file which for some reason made it impossible to run from disk but it installed just fine.  The installation was the easiest thing and it just created a user profile and a partition and it took a fraction of the time it had taken me to install Win XP.  Of course, there was a small problem with the refresh rate and I came here for help where I got my first introduction to the command line voodoo.  But now I've got everything (almost) up and running, I'm getting to be better friends with the console and I'm starting to figure out how to make it play the way I want it to.  And it just works and it's free as in speech and free as in beer!  (Still not quite sure how anyone gets paid that way...)

So I've got this great OS, very stable, very unintrusive, I LOVE the repository, and the support here has been amazing.  Really, you guys are the best!  But... I feel guilty.  I haven't paid a dime for any of this and I think it's well worth investing in.  Is there somewhere I can send a check?  I don't have a card but if there's an address I can mail to I want to let the developers know their work is appreciated.  (Seriously, how does this business model work?)

### Post by Jussi Kukkonen on 2007-01-10
That's just it - it's not a business model. It does prevent you from using some business models, but so does every other decision you do. There are still several different business models you can use with free software: Mozilla corporation is currently getting rich with the deal from google; Sun sells machines, support and services with Java; MySQL sells support and a closed source version of the database. Free software has also been used as a loss leader for traditional commercial software. Then there is of course the Ubuntu business model where you find a billionaire who is willing to pay for it alll :)

Still, probably most of the software on your machine hasn't been developed because it fits the business model, but because the developer had an itch to scratch AND[list][*]he thought freeing the code was the right thing and/or
[*]he knew he wouldn't get the help he needed if he didn't free it.
[/list]The last point is important: I doubt Linus would have freed the code if it hadn't been the only viable way to develop linux further.

About the payback: I'm not opposed to sending some poor developer a check, but I've always thought that the best way to thank the whole "enabling community" is to do something for the community -- that's why I file bugs (even if I've already solved the issue for me) and help people on these forums.
