### Post by eigenman on 2008-01-01
Hi all,

so I recently purchased a new HP laptop (DV6607nr), and as my first real work on the new shiny toy was to install kubuntu (amd64 version). However, there were quite a number of problems with this distribution on my laptop. The worst, in some ways, was the fact that the CD would crash with the default boot option, and I had to choose the "Safe" option. Eventually I did manage to boot and install kubuntu, and make most of the computer work, although I gave up on the wireless and everything felt a bit flaky. Currently I'm using gentoo, and the system seems much more functional, although there are some small issues that I haven't started worrying about yet. 

To be honest, if it wasn't for the fact that I've installed kubuntu/ubuntu on some other computers with almost no problems, I would have advised people to avoid kubuntu, or at least this specific version. However, I have a feeling that the problem is at least partly on my laptop's crazy hardware. Still, I feel like many of these problems can be resolved if few packages (such as the kernel) existed for specialized hardware. I'm wondering if it is possible to have a (online) database where the installer checks against before hand? In some ways I feel like I can make kubuntu work well on my laptop by putting enough effort in it. However it would be really cool if this can be done automatically for other people with the same hardware.

I'm curious if anybody has any idea if such a thing is possible, and if so how it might be implemented.

Cheers,
Eigenman
