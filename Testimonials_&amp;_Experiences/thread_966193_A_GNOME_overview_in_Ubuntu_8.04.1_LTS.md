Fonts
=====
One of the differences between Windows and Linux is using of a feature called "Smooth Edges" in Windows, and "Best shapes" or "Subpixel Smoothing" in Linux. it makes font consists of black and gray pixels, not only black pixels.

This feature is used in Linux everywhere; GUI items, web pages, and office documents... etc. But in Windows it excepts  GUI items, since the font used in GUI items is natively designed for GUI items only, it consists of only black pixels, there's no gray pixels at all.
[http://www.linuxfuture.org/media/Fonts_win.png](http://www.linuxfuture.org/media/Fonts_win.png)
[http://www.linuxfuture.org/media/Fonts_linux.png](http://www.linuxfuture.org/media/Fonts_linux.png)
Linux is strongly recommended to follow this principle:
"GUI font should consist of black pixels only".

### Post by 3rdalbum on 2008-11-01
Nautilus is designed for file management. I don't think sound recording or scanning is in the scope of Nautilus.

You might be more interested in trying KDE. There's a program for KDE called Konqueror, which is a file manager, web browser, CD ripper, image viewer, and iPod loader all in one. (CD ripping and iPod functionality requires extra plugins).

In particular, the CD ripping just shows the CD as a set of directories - one called MP3, another called Ogg, another called FLAC, and another called WAV. You just drag the appropriate folder to the part of the hard disk you want the ripped/encoded files to go to, and Konqueror does the rest.

Gnome developers have also been thinking of adding a file format converter to Nautilus, which would operate when you change the filename's suffix. No progress on this yet.
