### Post by SandyJ on 2008-02-12
I suspect many will look on this as a rant.  Please believe this is not my intent.  Try to look on it as an appeal for information: perhaps on what my expectations should be, or what basic concepts I lack.

I've used PCs since before some reading this were born.  (How long? If I look I can probably fond a DOS 2.1 operating system on a 5.25" floppy in my basement.)  I've pretty well worked exclusively in the DOS/Windows world all that time.

Throughout all this I recall favourable reports about Unix.  I remember the explosion of development when Linux appeared, and people began to realize what it could be the foundation for what we see today.

I've been wanting to play with Unix for some time but was unwilling to make time to master it.  As Linux-based systems began to proliferate and develop GUIs I began giving stronger consideration to doing something more than read the odd article.  Receently someone whose expertise and opinion I have come to respect said that Ubuntu v7 was well worth looking at.

I took the plunge.  I downloaded an ISO for v7.10 and burned it onto a CD.   I took my old laptop, wiped the hard drive, loaded the CD, re-booted  --  and nothing!  A blank screen.  Frozen.  Not even the power button worked.  In the end I had to pull the battery to get my machine back.

The laptop was one of the first Centrino laptops ever built, based on an ASUS M300n board.  Google revealed there were problems with Ubuntu (and other Linux-based systems) and this motherboard even with the most recent BIOS loaded.  Some reported success with the inclusion of  acpi=off  and other parameters.  This helped, but the resulting system still was far from stable.  As well, letters on the screen lacked pieces if themselves.  For instance I saw the word "live" in which the first 2 letters appeared to be identical.

I then read a bug was found and supposedly fixed in the Linux kernel that overcame this problem, that this was included in the new v8 code (Hardy Heron), and that alpha code was available for testing.  I found the code for v8.04 (alpha 4) and tried again.  It worked, without the need for parameters.  I installed Ubuntu and sat down to play with it.

The first thing I did was to install all the security patches.  When I re-booted the system was completely unuseable: the screen seemed to be spinning upwards to the point that nothing could be made out.  After some effort I identified the problem patch, applied the others, and again sat down to play.

It soon morphed from play to pain.

The screen was, if anything, worse than before.  I played with dots, taking it up to 120 and beyond.  I played with fonts.  It made things better, but by no means could it be called good.

I knew the problem wasn't my machine.  To prove it at one point I re-installed XP on the box; the display was fine.  I wondered if there's a better video driver available; if there is it's proprietary and hiding out of my sight.

I put that aside for a moment and went to other areas.

I couldn't connect wirelessly to my router!

Most (all?) routers have room for more than one WEP.  I had my system set up to use one other than the first, to make it harder for anyone trying to break in.  I could find nowhere in the wireless setup to tell it which key to use.  I wasn't about to reconfigure my home network (and weaken security to boot) to acxcommodate my Ubuntu machine.  So for the moment I put this aside as well, settling for running a Cat 5 cable through the house to where I happened to be working.

I then set up an email account in Evolution.  It downloaded messages when I set it up, but ever since then the send/receive button is greyed out and try as I may I cannot use Evolution.

No problem, I thought.  I like Thunderbird anyway, and there's a Unix version.  I downloaded it.  Surprise number 1: what I got was not an install program that I've gotten to take for granted in the Windows world.  Instead it was an archive.  I extracted the files.

Surprise number 2: no install program in the archive.  No problem, so I thought.  All the files seem to be there already.  I clicked on the Thunderbird icon.  Surprise number 3: nothing happens.

At this point I gave up on Ubuntu.


A day later I'm willing to re-consider.

Perhaps it's the laptop.  My son will probably be getting a new machine.  If he does I'm considering grabbing his old machine and trying to turn it into a Unix box.  It certainly is not worth my while to buy a new box for this purpose, even not taking into account the reservations I'm starting to have about Unix/Linux/Ubuntu based on this last little while.

The unfortunate thing is that I'm probably someone that would be an asset to Ubuntu.  I'm a techie.  My people trust me.  I am in a position to convince people to try it, and to hold their hand while they do so.  More than this: I am in a position to influence literally hundreds of people like (better than!) me, across 4 continents, many of whom are in a more influential position with even more companies than I am.  If I were convinced that  Ububtu Is A Good Thing  I might help it to spread in some small way.


Anyone want to try to convince me?  I'm open to discussion.



P.S.  I'm typing this on my Windows laptop.  I'll be using it to take part in any discussions here.    :-(

### Post by igknighted on 2008-02-12
1) laptops are notoriously difficult to get working properly.  They are loaded with proprietary pieces that are sometimes difficult to get working properly.  Try a desktop to learn on, or be ready for a little challenge.  Also, try a few distributions.  Ubuntu is popular, but if there are bugs with ubuntu and your system, try other distributions.  I use Fedora because ubuntu just doesn't work on my laptop and Fedora does (albeit with a little work).

2) you are using an "alpha" release.  Being in computers for a long time, surely you are aware that an alpha release should never be considered stable.  There is a warning not to use it as an every day system, because of things like this.  Updates often do break things.  They are not security updates (ok, some might have been), but rather upgrades to applications.  New versions = new bugs.  They will be (hopefully) worked out by final release, but thats the purpose of the alphas and betas... to find them.

3) As for email... evolution is crap, most here will agree with that.  It's default because it's a do-all application like MS Outlook, but it seems to fall short at everything.  As for getting Thunderbird, you have fallen victim to "bad windows habits-itis".  One huge strength of linux distributions like Ubuntu (and suse, fedora, mandriva, etc.) is the centralized software repositories.  Go to the add/remove programs application (in the applications menu) or the synaptic package manager (in system->administration) and you will find tens of thousands of programs available to download and install automatically... all for free.  Search for thunderbird there.  This is also available from the command line as well (most linux vets prefer this because its simpler and faster), just type "sudo apt-get install <package name>".

That's a start.  Second, I would really spend time doing research.  All these things are well documented, especially in these forums.  And post for help, we all do it when we need some advice.  Chances are whatever you are trying to do someone has done before and can advise you on what to expect.  We're here to help... make use of us.

### Post by steveneddy on 2008-02-12
To get to Synaptic, the software packaging system for Ubuntu:

At the top of the screen on the upper task bar - 

System --> Administration --> Synaptic Package Manager

when it opens, click inside the upper right box/window with the program names and start typing Thunderbird. Synaptic will catch up and you will see the program ready to be installed. Right click and choose install. Go to the top and click the green check that says apply.

That's all there is. 

There are other ways to install software on Linux, but this is a start. these are programs and software that was compiled to run on Ubuntu and have specific Ubuntu packages here for you to use.

I don't know what problems your still having with your monitor and I am very much *not* an expert but I have some ideas.


Restricted Video Card Driver?
-------------------------------
hit ALT+F2 and type in 
restricted-manager
enable your video card if it's not enabled already



Proper Monitor Refresh Rates?
---------------------------------
hit AL+F2
type in gnome-terminal
in the terminal, type in sudo gedit /etc/X11/xorg.conf  
and enter your password
click SAVE AS at the top and rename this xorg.conf.backup (or something) in case of disaster

Now, find and alter the areas...
> 
Section "Monitor"
	Identifier	"Generic Monitor"
	Option		"DPMS"
	Horizsync	28-96
	Vertrefresh	43-60
EndSection

Change your horizontal and vertical refresh rates if they need it.  Both my desktop and laptop machines needed this (old monitor, and typical laptop problem)



Decent Resolution?
--------------------------------
hit ALT+F2 and type in
gnome-display-properties
if your refresh rates and driver are correct you should have some decent resolution and refresh rate options



Select Software Sources and Install Fonts?
---------------------------------
Click the little Ubuntu Start bar (not sure of the proper key commands for this one so I'll give you the long mousey way)
Start > System > Administration > Software Sources
check the first four options under the "Ubuntu Software" tab.
You might get a message saying "sources have changed" or some such, and you'll probably have new updates waiting for you after this.  Just ignore those updates for now.

Next, hit ALT+F2 and type in
gnome-app-install
On the right, select "All available applications" from the dropdown list.
In the Search bar, type in 
fonts
and select "Ubuntu restricted extras" and if you like "Microsoft Core Fonts"
install, apply.


Misc
----------------
If any of the above commands give you a message about needing to "be root", instead of the straight command, put gksudo in front of the command.  So for example "gnome-app-install" would be "gksudo gnome-app-install".  Do *not* use sudo or gksudo in front of anything if you can do without it though, or you might end up essentially adding locks where you don't want them.

All of those commands above can be found with the mouse in the menu, but for instructions it's just easier to give a command line.

To APPLY changes to your xorg.conf file you need to restart X-server.  Do this by hitting CTRL+ALT+Backspace and log in again.

If you type in something wrong in xorg.conf file and then restart your x-server you'll be in for an unpleasant surprise when you reboot, stuck with a text-prompt only.  This is why you needed the xorg.conf.backup, and you might want to write this next command down.  If your stuck with a text-only promtp, type in
sudo nano /etc/X11/xorg.conf.backup
Make a change somewhere in your backup file, I suggest just typing a word in the commented out lines (the lines with # in front of them) just to get the program to promt you for a save.
Hit CTRL+X
Do you want to save? Yes
File name to write?  Just erase .backup from the filename, hit return
Do you want to overwrite?  Yes
Then hit CTRL+ALT+Backspace again  

Chances are you wont have to deal with a text-only prompt but I still suggest you write down "sudo nano /etc/X11/xorg.conf" for reference somewhere if you ever, ever mess with your xorg.conf file.

### Post by hugmenot on 2008-02-24
About your key number issue. This is not supported in the GUI tools. I assume because barely anyone uses numbered keys. It definitely is not an effective security measure. Trust me. You can just as well forget about it and set your key to be the default #1.

You can set it with the command line interface. Here's what "man iwconfig" says:

```


[... down low ...]

       key/enc[ryption]
              Used to manipulate encryption or scrambling keys and security mode.
              To set the current encryption key, just enter the key in hex digits as XXXX-XXXX-
              XXXX-XXXX or XXXXXXXX.  To set a key other  than  the  current  key,  prepend  or
              append [index] to the key itself (this won&#8217;t change which is the active key). You
              can also enter the key as an ASCII string by using the s: prefix.  Passphrase  is
              currently not supported.
              To  change  which  key  is  the currently active key, just enter [index] (without
              entering any key value).
              off and on disable and reenable encryption.
              The security mode may be open or restricted, and its meaning depends on the  card
              used.  With  most  cards, in open mode no authentication is used and the card may
              also accept non-encrypted sessions, whereas in  restricted  mode  only  encrypted
              sessions are accepted and the card will use authentication if available.
              If  you  need  to  set multiple keys, or set a key and change the active key, you
              need to use multiple key directives. Arguments can be put in any order, the  last
              one will take precedence.
              Examples :
                   iwconfig eth0 key 0123-4567-89
                   iwconfig eth0 key [3] 0123-4567-89
                   iwconfig eth0 key s:password [2]
                   iwconfig eth0 key [2]
                   iwconfig eth0 key open

```

That means, you make a series of commands (sudo prepended):
```
sudo iwconfig eth1 essid "YourWIRELESSName"
sudo iwconfig eth1 key [3] 12345678900987654321123456
sudo iwconfig eth1 key [3]
```

with eth1 substituted for your actual network card and the key number and key itself.

### Post by hugmenot on 2008-02-28
Sorry for returning to this thread.

From your description I realized that NetworkManager still has its clutches on the network cards. This is an old source of confusion for Ubuntu users. You can either have NetworkManager manage your cards, or do it manually, but not at the same time. 

If you want to see whether command line config works for you at all, try to kill NetworkManager first. Just give it another shot. We have more infos now, so I would propose this sequence (later we'll write it to a file, so you will not have to do this every time):

```
sudo killall NetworkManager wpa_supplicant
sudo iwconfig wlan0 essid Sandy
sudo iwconfig wlan0 key [2] 2222222222
sudo iwconfig wlan0 key [2]
iwconfig wlan0
sudo dhclient wlan0
ifconfig wlan0
```

Explanation:
First we want to release wlan0 from being managed by NM. Its automatic logic doesn't help here (key index 2, etc).
Then we tell wlan0 to associate with "Sandy", and give wlan0 a key with index 2, then tell it to use this key number 2.
Now running "iwconfig" should show wlan0 being associated with 'ESSID:"Sandy"', but wlan0 doesn't have an IP address yet. I guess you have DHCP on the router, so we just use dhclient for this.
Check with ifconfig whether wlan0 has an IP adress now, if it has, the wireless key has been correctly set.

If this works you can write very similar lines to the file /etc/network/interfaces and the WLAN will come up every time you boot, by itself.

Without the key index this would be a lot simpler. But anyway.

EDIT: And to get the NetworkManager back, if it didn't work of course, you just restart it with
```
sudo NetworkManager
```

No not exactly saying that. My understanding is that Sandyj's network set up is already insecure and unsatisfactory due the one machine having an adpter that can't handle wpa. So it's a mistake to try to find some elaborate workaround to preserve the current setup. *The current setup is the problem* and even if no new PC or OS was introduced the most pressing issue is to rectify that problem i.e. change from wep to wpa immediately. This can be done in two ways: 1: remove the machine with the inadequate adapter or 2: add a modern wireless adapter to that machine (at very small financial cost). The immediate and direct benefit is a secure network (wep is about as good as no security at all) and the additional/side benefit is that suddenly the new Ubuntu machine/OS (and any modern Mac) can now connect with ease using network manager/wicd or even wpa-supplicant config'd manually. All this effort being made to preserve a really bad and insecure wireless setup is just adding another problem which, unfixed,  will doubtless rear its ugly head again in the future in the same form, or worse because some ill intentioned person has accessed the network and taken advantage of its resources.
