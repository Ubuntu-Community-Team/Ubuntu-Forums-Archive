### Post by ve4cib on 2009-06-22
> 1. With each release there is always new problem that was not there before... Whats with that other distros are also on a 6 month cycle and don't break something new every time..

From what I've read most distros with a rapid release schedule encounter problems like that.  You need to remember that Ubuntu is based on Debian Unstable.  While generally stable for most users, some users will encounter problems.  That's a fact of any software, really.

Also, if you really need stability consider limiting your use to the LTS releases.  They are (in my experience anyway) much, much more stable and robust than the regular 6-month releases.  Think of the six-month releases as betas for the next LTS and you're not far off.


> 2. Doesn't come with every tool needed by default or atleast not enabled. A perfect example is the control panel.

What constitutes "every tool needed"?  Some people would categorize NetBeans, Eclipse, and Anjuta as "needed".  Other people really need the entire OpenOffice collection.  Others need every single eye-candy desklet they can find.

With any OS there is always software you don't have that you need to install, and superfluous software that you need to uninstall.  Fortunately with Ubuntu the repos contain pretty much everything you could need, so you're a simple apt-get (or a few clicks in Synaptic) away from getting what you want.

In my opinion Ubuntu strikes a good balance between a usable, complete desktop environment without cluttering everything up for those of us who tend to customize everything much more heavily.


> 3. Almost every god dang tutorial I read it leads to the CLI. I will first say I do not care about using the CLI. However others do... Infact some people are scared of the CLI.

Most people who write tutorials prefer the command-line.  It's more expressive, and it gives you a better feel for what exactly is going on in the computer.  But I agree with you that it can be intimidating to newcomers.

If you find that this is such a problem why not write some GUI tutorials for these issues?  I'm sure CLI-phobes will appreciate your efforts.


> 4. Some releases are known for coming default with broke software....

Examples, please?  And see #1, above.


> 1. The distros i mention and some others do have a control panel by default which many people like myself find useful.

See #1 above.  Yes, it's nice.  But many people do just fine without it, and it's easily available those who really want it.


> 2. I know mandriva comes with a great installer on all three of its versions that is very easy to use... Ubuntu's installer is not so easy..

What's not easy about Ubuntu's installer?  The Live Disc one is about as easy as it gets.  Enter information, click Next.  Repeat a few times.  Wait until install is complete.  Reboot.  Done.

I've never used Mandriva, so please tell me what it does that makes it so much easier than Ubuntu?  And have you pointed any of those differences out to the devs so that they can see about integrating those changes into Ubuntu to make it easier?


> 3. Many distros have ndiswrapper built in so if it can find a Linux wirless atleast you can get a windows one working just fine.

Provided you've got an ethernet connection, or another computer connected to the internet and a flash drive, you can easily install wireless drivers.  Most common ones (including the notorious Broadcom b43xx) are set up with a single package.


> 4. About every other distro besides Ubuntu has a gui tool for everything. Although some smaller distros do not either.

Examples, please?  And what specific GUI tools is Ubuntu lacking?  Since 6.06 there haven't been many times where the CLI is needed.  Since 8.04 the CLI can be avoided entirely if you want to avoid it.  (Barring any particularly arcane operations, of course.)

> 5. Last,but not least when asking for help on lets say opensuse or mandriva... you normally get a gui tutorial not a cli one..

See #4 in the first portion.  Please, feel free to write your own GUI tutorials.  Your efforts will be appreciated.
