### Post by Keyper7 on 2008-10-02
For the past few months, I've been considering moving to Debian. Not because I'm not satisfied with Ubuntu, quite the contrary: I'm **very** satisfied with it and my hardware works perfectly (even hibernation). But I had two main reasons:

1) Pure sentimental crap
Debian is the distro that introduced me to Linux in the first place. I'm very attached to it (and to apt-get).

2) Ubuntu offers me too much
Being mainly a command-line user and a "bleeding-edge compiler", I don't use many, I think most, of the features that Ubuntu adds to Debian.

However, I've been changing my mind in the last days, because of one reason: conversions. Nowadays, Ubuntu, as a brand, is very strong. The existence of several different distros usually is more information than the average Windows user wants, and is usually much easier introducing Linux with "that Ubuntu thingie they've certainly heard about" than with a perhaps-easier-but-less-known distro such as Mint. People tend to be less suspicious when facing a known brand. *Specially* if you have something professionally packed such as a ShipIt CD at hand.

So what does this have to do with **me** and **my** computer? Well, quite simple: depending on the hardware, Ubuntu has issues, and regardless of hardware problem there are hassles (depending on the user) such as proprietary drivers and codecs not being installed by default. Therefore, I want to keep using Ubuntu, the latest version, to be *fully aware of all necessary tweaks to install it perfectly*. I don't want to give someone a Mint CD saying "this is even easier than Ubuntu, try it", and then not knowing what to answer when a Mint-exclusive issue arises.

Bottom line: Linux is not Windows, it's different. I don't believe in giving a Linux CD to a Windows user and saying "try it". I believe in staying at the user's side for a while and explaining patiently all the important differences (as long as he's interested in handholding to begin with, of course). And if I want to do that, I want to make sure I have, from personal experience, relatively deep knowledge of what I'm introducing him to.

Your thoughts?
