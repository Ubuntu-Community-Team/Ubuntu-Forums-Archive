### Post by Christian Knudsen on 2009-02-24
I thought I might as well add my experiences to those already posted here...

I'm a hobby programmer and have been working on a game for a while now. A year or so ago I finally decided that I had to port my game to Linux as well (instead of forcing users to use Wine). I decided on Ubuntu as it seemed to be the most used/userfriendly/supported. Like many other Linux novices I feared an operating system consisting mostly of a terminal and having to type in most commands. After the smooth installation of the OS (dualboot with XP), all my fears were shattered. Yes, the desktop may be a bit orange, but I really found it slick and extremely easy to use. Having never used Linux before, I spent a great deal of time just playing around with stuff before installing my programming language (FreePascal with SDL). This of course wasn't quite as easy as it had been on XP, since I was used to XP and still had to wrap my mind around dependencies and repositories. But I finally managed to get it working and getting a good feel for Linux and Ubuntu in the process.

Now, I still mostly use Ubuntu for when compiling new versions of my game for Linux (I have both 32 and 64-bit versions installed), but now and then I boot into Ubuntu just to play around a bit, or when I need to use Gimp for some graphics stuff (I tried installing the XP version, but it felt pretty buggy compared to the Linux one). Overall, my Ubuntu experience has been extremely positive. I love how easy it is to add/remove programs: just select it from the list and it installs automatically! Brilliant! However, it's the "small" things that constantly surprise me and continue to turn me to Ubuntu. Like for instance: I've installed a plugin for Firefox (yes, I've also switched from IE to Firefox in XP) that allows me to download a HD youtube video and view it offline. In XP I click the download link, select save file, and then select the default file name and overwrite it with my own. Easy? Well, yes, but I'm annoyed that I have to press delete or manually highlight the default file name with the mouse to rename it. In Ubuntu, when I click save file, the OS automatically highlights the first name of the file, so I don't have to press delete or select it with the mouse; I just type in the new name! It's such a small thing, but it streamlines the usage so much when I don't have to manually delete the default file name - I just go straight to typing in the new name. Also, I just had to print something from Ubuntu and I hadn't installed my printer yet. I thought it wouldn't be a problem - I just had to find and install the driver. Nope. I plugged in the printers USB cable, watched the harddrive LED blink for a short while (so I knew something was happening), and then a popup told me that the printer had been installed. What? I didn't need to do anything. Not even click on a "Yes, install these drivers" button. It just did it automatically in a matter of seconds. I know I'm probably just lucky in having a printer that's supported by the default Ubuntu install, but I was definitely surprised.

Yes, there are some minor things in Ubuntu that can annoy me once in awhile, but the fact that this is a completely free OS, that I don't need antivirus software installed, that I can install new free programs with two clicks of the mouse, that it looks good and is relatively user friendly, and that it seems snappier and more stable than XP just continues to blow my mind. Did I mention that it's free!

Reading through some of the testimonials here, there are some users that have become frustrated with installing or getting Ubuntu to work. I suppose that's only understandable as most of these users are probably used to the Windows way of doing things; or they haven't installed an OS before and can get daunted with the task; or they're unlucky in that their hardware isn't supported. I can understand their frustration to a certain extent, but I also understand that the Linux community is in a constant struggle with hardware manufacturers to release their drivers for Linux. There's only so much you can do if some hardware manufacturers decide not to support the OS you're developing. I guess I've been lucky that way. Most of the hardware on my laptop is supported (except the wireless network card, but I knew that when buying my laptop and I can still connect with a cable), I've installed various operating systems in the past, so I had an idea what I was doing when installing it, and I was willing to let go of the XP way of doing stuff and learning how it's done in Ubuntu.

I believe and hope that with time all hardware developers will support Linux, more computers will be sold with Ubuntu/Linux as the preinstalled OS, and users will get more used to (and be more willing to learn) the Linux way of doing things.

I still really like my XP and it's still my primary OS (and I have no ill feelings towards Microsoft; it's a business like any other and I love most of their products), but that doesn't mean I can't be amazed by Ubuntu as well and hope the best for its future. It has certainly taken me by surprise already!

### Post by jbysmith on 2009-02-25
Good read, nice to see people keep an open mind. And +1 for the FreePascal nod; I mostly work in C++ (Visual Studio 2008 Ent) lately for work, but I grew up with Turbo Pascal back in the DOS days, and Delphi seems to have lost its way since D7.  I use FPC any chance I get heh. Sometimes with Lazarus, sometimes with Geany for console-only stuff.

And definitely picked the right OS as a programmer.  Windows has some nice stuff, granted, but Linux just reeks of "programmer heaven."

But yea, there's a few things that'll stop people from really enjoying Linux.  And I'm not going into the Windows vs Linux thing.  That's on you; Windows does have its merits too.  I have a gaming partition, I've no problems with that. 

First is the hardcore gamer crowd.  Sure, Wine/Cedega runs some.  Sure you can set up a dual-boot and just play them in Windows.  But some people just think that just because I can't play Dead Space on Linux, that means Linux must suck and won't even try it. Period. Nothing you say will convince them otherwise.  Give the token "enjoy the spyware" comment and move on.

Some get irked by driver support.  That's something that can get anyone.  It's not Windows, so need to research your hardware to make sure it works.  (If you're in doubt, Wubi is a great way to test a live install without commitment.) This isn't unique to Linux.  Got an older nForce2 motherboard for example?  Good luck with Windows 7.  (You can try the XP drivers, but performance is laughable, and quite probably unstable as hell.)  Look before you leap.

Then you got the people who have been using Windows since forever, and are hard-set in their ways.  If it doesn't work like they think it should, it's obviously a bug. They just don't want to learn something new, even if in reality it's actually a lot better way of doing things.

Some are turned off by the terminal.  If it's not click-click-done it's obviously junk.  I find it a blessing.  The GUI is nice for the really simple stuff; I'd rather drag and drop than type a cp command any day, especially if it's a bit selective.  But anything beyond that, the terminal is amazing.  Bash scripts are absurdly powerful, and if thats not enough, kick it up a notch with Python or the like. Throw in the fact that I can do *anything* from the console that I can do with the GUI, and anywhere in the world for that matter with a properly secured SSH server, it doesn't get any better than that. Expose Winder's Remote Desktop to the world? Uh-huh.  Drop your pants and bend over. I'm sure they'll be gentle.
