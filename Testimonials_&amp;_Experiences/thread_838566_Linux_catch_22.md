The case is that Linux is not Windows. Windows has its way of doing things and Linux has its own way which just happens to be different. Installing most programs/applications/whatever is extremely easy in my opinion. It is simply just a matter of going in to the Synaptic program, searching for a program, checking it off and clicking apply. That's not difficult at all :)

If you can't find the program through the default repositories, then you might be able to download a .deb file. This is pretty much like the .msi files that you find some people use to pack their applications into for installation.

I was like you at first--not being able to get a lot of work done in Linux. This was because I was trying to use Linux distributions like Windows. When I realized that things are done differently, I had to do some playing and learning to discover how to accomplish certain tasks. It wasn't hard, it just took some time.

Personally, I prefer the use of software repositories and .deb files. I my opinion, I think it is a lot easier to download and install programs like this instead of hunting down the application from a website and trusting that they have not included any additional 'gotchas' in it.

It is all about learning the differences and using Linux as Linux and not Windows. The same principal applies to Mac as well. You don't install Mac based programs like in Windows. Correct me if I am wrong, but to install a program you drag the application to an application folder ?

As far as I know, GTK+ is a framework of libraries that programs use. Depending on the language you want to work in, you will have to find bindings that apply to that language. For example, say you want to use python to create a GTK application then you'll need *python-gtk2-dev*. Basically, go in to Synaptic and look for gtk with an extension that says **-dev**.

Perhaps: **libgtk2.0-dev **and from **apt-cache show libgtk2.0-dev** :
> 
Description: Development files for the GTK+ library
 The GTK+ is a multi-platform toolkit for creating graphical user
 interfaces. Offering a complete set of widgets, the GTK+ is suitable
 for projects ranging from small one-off tools to complete application
 suites.
 
 This package contains the header files and static libraries which is
 needed for developing the GTK+ applications.
That looks like what you need :)

Software installation is probably one of the easiest task in Ubuntu.
