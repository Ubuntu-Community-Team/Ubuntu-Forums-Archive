### Post by weblordpepe on 2007-05-11
Thats quite alright.

You have touched on an issue which I feel is quite an interesting one when it comes to designing an OS:
Can you make an OS easy to use without hiding how the computer works?

Some people seem to think that they are polar opposites. Personally I think it all nails down to the best choice in defaults.

My opinion is that using proprietary systems (Eg Windows on any platform) will only allow you to go so far as the developers let you.

Eg: You can never find out -exactly- what your program is doing. You only know that it is accessing xx API provided by the MS gods.
Eg: Drivers: 'Driver tweaking' in Windows consists of 'does this one work? try this one instead'. There's no 'what if I change this line of code'.

Now I know there is closed-source and open-source drivers/software for both platforms but my point is regarding the openness of Linux versus the 'no we wont tell you how it works. Use our API which will do it for you'-ness of other systems.

Ubuntu is a prime example of a system which is idiot-friendly by default, yet you can alter it to be the geek-system of your choice. End result is: Everybody wins.
My guess is that its the middle-ground - the power users who are having the Windows-Linux teething problems.
