### Post by llimaa on 2008-10-23
Hello all...

... I would like to discuss/share a little bit about the problems I had and still have with Linux in the last +/-4 years. Sorry for my bad English in advance.

First I want to say that I love the GNU,GPL etc. idea and think that most software should be free. But everybody also have the freedom to decide to sell his software or not. 

About 4 years ago I decided to install Linux as my main OS and completely deleted Windows XP from my Hard disk. I had my private reasons for this decision. I tried many key distributions of the top 10 Distrowatch charts..Fedora, Suse, Debian, Kubuntu, Ubuntu. I found out that Gnome fits more to me.

I did like Debian a lot, used (stable,testing,sid)for 2 years and then changed my Desktop Systems to Ubuntu because Debian was taking to much time of my life. I wanted to be more productive and spend less time with the OS. I started to use Ubuntu and found out that Ubuntu is better packaged and preconfigured than Debian. Both have their strengths but for me Ubuntu was simply the better choice for the Desktop PC.

And now after using Linux for a while (not only related to Ubuntu) the problems did appear. I noticed that Linux Software does not always works very well. Things often break that was working fine one release before. E.g. took me a whole week to get Wlan working again when 7.04 got released. I even bought a Atheros card. This was not the only Problem I had and all this stuff took a lot of time of my life. Notice: Was just an example and I upgraded the distribution just because of a hand full of desktop application.

I often wanted to have the by their developers as stable released applications and did run into several problems.

1.) I don't want to install software from repositories made by people on then web that I can't trust 100%. Specially not as root, into /home I would make an compromise.

2.) But I also would like to have the latests stable version of applications I use daily. I trust the app developers and if they say its stable then its stable for me. I think they know best when their application is stable or not.

3.) I also don't want to compile applications...remember I'm a user and not a developer nor tweaker. And I want to spend my time with things I think that is important for me.

Let me try to let you people understand me more. Imagine an Gimp artist, of course he wants to have the latests version of Gimp and use all the nice features the developers created in the last 6 months. Like an database developer that want to have the latest stable Mysql version. And a Office Guru want to install the latest OpenOffice.org version and so on. From my point of view this is a very natural behaviour.

And that's the main Problem have with Linux right now. Applications are tied to the whole Linux OS, often I need to upgrade the whole OS just because of a few Desktop applications and then run into new problems because of that upgrade. After solving the problems I look to the date and wonder how many days passed and in the meantime new version of my favourite application appeared. It's a never ending story.

Since I use Linux I had not a single day to just sit down and think hmmmm "what's do next".

That's my main problem with Linux at all, I don't mind things like buying a new piece of hardware because the one I actually have is not supported. I also don't mind to wait 1-3 days till I can install the new released application but 6 months is too long for my taste. And I also don't want to upgrade the whole OS if everything is working fine just because of the wish to run a newer version of my favourite application.

Can't be that hard to bring all distributions closer together, define a new package concept, making software installers cross distribution compatible and untie applications from OS libraries and OS base.
Make they install so, that can't break the whole Os by overwriting other distribution related files.

I'm starting to think it's better to use openSource Desktop Applications on Windows than on an openSource OS like Linux. At least I can enjoy new versions of my favourite applications. And probably I would not even mind to deal with malware,virus etc. because of this benefit.

And trust me, I learned to love Linux like Ballmer his company ;)
But sometimes... there is no other way.

What do you think ? Any solutions ?

PS: I'm sick of flame wars etc. so don't even start to think about it. :) 
It's not about Windows, OSX vs Linux,etc. it's about how to enjoy the latest application improvements while running a productive environment.
 
thanks in advance
llimaa

### Post by 3rdalbum on 2008-10-23
> **llimaa said:**
> 

Can't be that hard to bring all distributions closer together, define a new package concept, making software installers cross distribution compatible and untie applications from OS libraries and OS base.

1. Then you get huge executable files, if you put all the dependencies inside the program. It's also less flexible, more work for program developers, frustrating for users who want the new features of a particular library and then have to wait for a new version of the program to be released, and a security hazard.

2. You cannot create a binary package that can run on every Linux system out there. For instance, a PowerPC processor cannot read x86 instructions, and an x86 processor can't read PowerPC instructions. An x64 processor can read x86 instructions, but not the other way around.

Windows on an Itanium processor is a depressing experience because you can't run x86 programs; all programs on Windows are distributed as compiled x86 binaries, whereas if they distributed source as well, you'd be able to compile the source for Itanium.

In short, things are done this way on Linux for very good reasons.
