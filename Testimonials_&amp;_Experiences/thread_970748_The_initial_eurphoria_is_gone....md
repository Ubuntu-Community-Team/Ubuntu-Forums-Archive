### Post by moviemaniac on 2008-11-04
Well, I, too have to rant a bit about Intrepid.

But let me say this first: I won't say Ubuntu is crap, because it isn't. I have loved it from the very first steps with 7.04 and am still in love with it. However, in the last week with 8.10 I've had more issues than I've ever had with 7.04, 7.10 and 8.04 combined.

The system's very usable for me, but there are quite a few things that used to work and don't work anymore - regressions. I do hope and believe that all my problems will eventually be solved. 

Here's the problems I ran into (just a small overview, click the links for details where provided)

- Feature missing in NM 0.7 that allows me to get an IP on eth0 when ppp0 over eth0 is activated ( [https://bugs.launchpad.net/ubuntu/+source/network-manager/+bug/290639](https://bugs.launchpad.net/ubuntu/+source/network-manager/+bug/290639) )

- Cheese is not working ( [https://bugs.launchpad.net/ubuntu/+source/cheese/+bug/290506](https://bugs.launchpad.net/ubuntu/+source/cheese/+bug/290506) )

- Photoprint doesn't start ( [https://bugs.launchpad.net/ubuntu/+source/photoprint/+bug/260849](https://bugs.launchpad.net/ubuntu/+source/photoprint/+bug/260849) )

- Live-CD video-card-detection-regressions on many machines including my iMac 8,1 ( [https://bugs.launchpad.net/ubuntu/+bug/289999](https://bugs.launchpad.net/ubuntu/+bug/289999) )

- mail-notifiation crashes Evolution ( [https://bugs.launchpad.net/mail-notification/+bug/251031](https://bugs.launchpad.net/mail-notification/+bug/251031) )

- DVD-playback unusable in Xine when using the fglrx-driver ( [http://ubuntuforums.org/showthread.php?t=970653](http://ubuntuforums.org/showthread.php?t=970653) )

- Kaffeine locking up the machine when messing with Deinterlacing-settings (related to the previous problem, [http://ubuntuforums.org/showthread.php?t=970653](http://ubuntuforums.org/showthread.php?t=970653) )

- Broadcom STA-driver (wl) should work OOTB but it doesn't

- German translation was great in hardy, many things are now untranslated in Intrepid (purely cosmetical, but nonetheless)

- Bugs concerning system sounds (i.e. the logoff-sound not playing)

- and a few smaller glitches


All these things used to work perfectly in Hardy and are now broken in Intrepid. Applications I frequently use (photoprint, Cheese, Xine) don't work at all or are unusable, the new network-manager is completely unusable for me and the new fglrx-drivers seem to be buggy when it comes down to playing back video (Not Ubuntus fault, but anyway).

We'll see how soon these bugs get fixed. Until then I'll still have much fun with Intrepid but considerably less than I did when using Hardy or even older versions.

The way I see it, a new version of Ubuntu "should" offer more stable, improved versions of programs and not inferior, not-working ones. I know that there are problems at the beginning of every new release but I have never experienced this many and critical bugs before at the beginning of 7.04, 7.10 or 8.04. I'm not talking about small bugs, I'm talking about whole programs that don't even want to start or fail to work generally (cheese, mail-notification, photoprint...)

While staying with 8.04 surely is an option, it is certainly not the point of releasing a new version of the distro in a state that would at best be worthy of a Beta. These are bugs that should be fixed before the release, not months afterwards. But, well, that sure is the downside of fixed release-cycles.

//edit: I think my whole point is this: While Linux-veterans know to live with these things ( I most certainly do ) I am thinking abount the tens of thousands of users that are willing to switch to Linux and try their luck with the new Ubuntu 8.10. These bugs will most likely be what drives them back to Windows saying "Linux is unusable, unstable, there's software included that's not working and this and that. What's the hype all about?" That's what my main concern is about.
