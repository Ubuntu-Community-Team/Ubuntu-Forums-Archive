### Post by jmat on 2010-01-01
After a great deal of thought, pondering and contemplation, I decided to comprehensively migrate from Windows to Linux, and from either Open Suse and Mandriva to Ubuntu Linux.  Part of the reason for the migration was political: a reaction against the Windows Monopoly position in the market.  Yet, the other core reason is due to the fact that Linux in general, and its Open Source software I've found to run just as good, if not better than most of what Microsoft has to offer.
 

 The migration came at a busy time as I have been updating, rewriting, revising and preparing the 2nd edition of my book, which to this point, had been done on Word and Windows.  The complete shift to Ubuntu thus required me to switch to Open Office.Org.  Thus far, the transition to Open Office appears painless and smooth, and the publisher will accept Open Office documents, although they prefer it to be saved as a .doc file.  So with all that said, and with the fact that one of my chapters contains a strong critique of the Windows Monopoly, I feel much more comfortable working on Open Office.
 

 One thing that held me back from full migration to Linux was my Music collection—done comprehensively on Windows Media Player.  I had no success in  playing the files on Banshee, Amarok, or Rhythmbox, so I decided to do a search for .wma compatible players and found a neat little player call Audacioius2 that works quite well! Hence, the 20+ Bob Dylan albums I had created on Windows Media Player now can be utilized quite successfully on Ubuntu!
 

 Some other initial thoughts on Ubuntu 9.10:  It not only boots and shuts down faster than Windows, but also much faster than Open Suse, Redhat, as well as Mandriva.  The one problem I encountered when I initially tested the 64 bit LIVE CD version I got a “kernel problem” and “potentially unstable” notice on the test drive. So I then tried the 32 bit version and didn't have that problem.  Weird though—cause my machine is a duel-core 64 bit AMD.
 

 So at this point, I have 3 machines running Ubuntu with one machine being an old 800mhz with a half-gig of ram and it works fine as a back-up system.  The other two machines have much greater processing speed and memory, and additionally, in the garage I have 2 refurbished puters with Open Suse 11.2 on 800 mhz with only 400 mb ram.  I couldn't run XP or Vista or Windows 7 on those old machines, but surprisingly Open Suse runs a bit sluggish, but fine, on those puters.  One question I have is if Ubuntu would run on the refurbs—as I don't know whether Ubuntu 9.10 requires more memory than Open Suse 11.2???
 

 So, all in all, my initial reaction to Ubuntu is quite good and merits a 10-star rating. I was given the recommendation to try Ubuntu from several different sources, so I'm glad I finally made the transition. With that said, I hope everyone enjoys a happy and prosperous New Year 2010!
 

 Joe

### Post by Fri13 on 2010-01-01
Hi! And welcome to wonderful world of Linux operating system, GNU philosophies and Ubuntu distribution and it's community.

> One question I have is if Ubuntu would run on the refurbs—as I don't know whether Ubuntu 9.10 requires more memory than Open Suse 11.2???

That is totally up about multiple different things. But first I just want to remind you that the correct names are "OpenOffice.org" (if not wanted to use whole correct name, then just OpenOffice. Official short versions of the name are OO.o or OOo) and "openSUSE".

The openSUSE use Linux (the kernel) as it's operating system, same way as the Ubuntu does. And both have the same Desktop Environments and all application programs available. So it is not matter of actual software what is available, but what is running by default. The biggest difference between all distributions are the default configurations.

The Ubuntu can be as fast and have as small memory (RAM) usage as  openSUSE does. But question is does the user (you) need to do anything configurations for default settings to get that.

Like you can need to turn off some services what you do not need. Like MySQL (database-server), Apache (WWW-server), CUPS (Printing-service) and so on. If you do not want to spend time to configure settings of your software system (what is possible because you are writing a book after all and you want just to use computer as the tool, not as the work), then you need to change the distributors (Canonical, Novell, Mandriva) by that who just offers best distribution in your situation. That is just the one great feature of open source and Linux that there is hundreds of distributors who packages own distributions of Linux OS and thousands of softwares as small packages with it.

To get the memory usage footpring for your machine, you could always just run the LiveCD from all of them and check the RAM usage. That is just directive how much they would take when compared to each other, but it will be less when installed.

One of the softwares what consumes most RAM, is the desktop environment. There are four desktop environments what you can choose. KDE4 SC, GNOME, XFCE4 and LXDE. They draw for you the desktop and has own application programs to tweak the desktop and offer some basic uses, like web browsers, mediaplayers etc. But then there are few dozens windowmanagers, what just draws the desktop and allows you to start application programs and use them normally. Like Blackbox, Windowmaker, FVWM, IceWM. Those consumes very little RAM (mayby few megabytes, when compared to desktop environments what can take few hundred) so using those on older computer makes them "fly".

If you want to use desktop environment on old computer, I suggest to use LXDE. It is very lightweight, fast and has lots of nice features what are on all three other heavier desktop environments. The XFCE4 use more RAM than GNOME, but GNOME needs faster CPU. KDE4 use as much RAM as GNOME but needs littlebit faster CPU than GNOME, it is the most heaviest desktop environments from all, but most flexible of all of those.

So if you do not need so powerfull desktop environment, use plain window manager on the older computer. 

> One thing that held me back from full migration to Linux was my Music collection—done comprehensively on Windows Media Player. I had no success in playing the files on Banshee, Amarok, or Rhythmbox, so I decided to do a search for .wma compatible players and found a neat little player call Audacioius2 that works quite well! Hence, the 20+ Bob Dylan albums I had created on Windows Media Player now can be utilized quite successfully on Ubuntu!

You should not need to choose the music player to do that. The idea of Unix-like systems is that you have many backends and frontends how the application programs are developed and used. Like you have a multimedia backend what can play all needed multimedia files. But you have then many frontends what use those backends to play the mediafiles. Frontends are like Amarok, Banshee and so on. If your backend does not have the correct *codecs* installed, it can not play the music. And then you need a such mediaplayer what decodes the mediafile itself, like VLC or Audacioius2. Fix for such problem in the future is just to install needed codecs and all mediafiles can be played by multiple different mediaplayers and you can choose what you like from them. My own combination is GStreamer + Amarok/Bangarang for music and videos. And DRM protected WMA files just does not work as far I know. That is the bad side of DRM! :(

First things First Happy New Year and welcome to the land of Ubuntu (We all help)

open a terminal (Applications/Accessories Terminal) and copy & paste this in it (if you haven't already)

```

sudo apt-get install ubuntu-restricted-extras

```

It'll ask for your password (the same one you log in with) but you **WONT** see anything when you type your password in. So get it right !!

That'll install various codecs & fonts & flash etc.

Commercial DVD playing..... Terminal time again

```
sudo /usr/share/doc/libdvdread4/install-css.sh
```

If you click the little blue button next to the firefox button and click **Music, Video and Photos** then click playing DVDs you'll see where I got that terminal command above and you'll get a better picture in "your minds eye".

good luck

Since Canonical actually sells support for Ubuntu, they make sure that the default edition has only Free Software (Open Source and all) so that they can fix it if something goes wrong. That is why NVidia, ATI, Broadocome and other drivers, as well as flash, mp3 codecs and encrypted DVD support are not installed by default. Some other distros (such as Sabayon) do come with many of those by default.

BTW: do you know that you can save OpenOffice files in .doc format.
