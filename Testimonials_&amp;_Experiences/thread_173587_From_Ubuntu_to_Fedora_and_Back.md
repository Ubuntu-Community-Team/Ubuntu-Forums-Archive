### Post by DigitalDuality on 2006-05-10
After trying Breezy with Gnome and KDE, then trying Dapper and playing wit XGL/Compiz, then back to Breezy, then back to Dapper to Xfce, and again back to Dapper.

I finally decided i'm comfortable enough to switch distro's entirely and wanted to get a taste of an rpm system, and play with yum.

Burned some fedora discs, install process was kinda long, but prettier than Ubuntu. The bootup process was pleasant looking. It recognized my new Nvidia card on first boot (something Ubuntu doesn't do) and recognized the fact my monitor could handle 1600x1200 (also something ubuntu doesn't do on boot). Started installing restricted formats and almost everything was working without a hitch.

But there seemed to be problems across the board.  Small differences would throw me for a loop for quite sometime, such as the location of "ifconfig" was not where i was used to it being (and i couldn't call the app directly from the command line without knowing what directory it was in).

I honestly feel far more secure using sudo rather than using su, and having the root account disabled from the start.

My 2nd hard-drive would never mount on boot.  I checked, re-checked, triple checked.  My fstab file was perfectly correct.  I posted on the forums and asked people i knew that know Red Hat systems, and know one could tell me what the problem was.  I figured it my be permissions.  So i ensured that my user account had permission to the drive, had permission to every file on it (permissions on all my files on that drive were really out of wack when switching distro's, and became out of wack when i switched back).  It still never worked in 2 weeks of playing with it.

I did think the gui means of installing rpms was a bit quicker than debs which was nice.  But doing updates and downloads/installs through yum, wow.  That is sluggish as hell compared to what i've been spoiled with here.  And their repo's suck compared to the repo's here.  The number of packages included is extremely limited to what i've become accustomed to.

Almost every program i installed not through the repo's there seemed to be permission problems.  This goes for bluefrog, avg anti-virus, electric sheep, etc etc.

I was able to get Democracy Player running beautifully though, something i still have no accomplished in Ubuntu.

The Firefox build that comes with Fedora does seem a bit faster than the build that comes with Ubuntu.  No idea why though.

So back home i am, where i'm comfortable.  I might try Suse and Debian pretty soon.  But i want a machine that just works for a little bit before i go playing again.

### Post by groggyboy on 2006-05-10
Neat story.  Ubuntu is my first taste of linux.  I too have toyed around with KDE/GNOME/Xfce.  I currently run Dapper Beta, but started with Breezy.

I love to tinker.  I came over from Windows XP (and before that Mac System 7), which was much harder to mess around with, being closed-source and all.  Linux is a wet dream come true as far as tinkering goes!

But now I'm getting the itch to try out other distros, just like you.  After all, Ubuntu is all I know of linux.  So the other other day, I cleared out a five gig partition, and set to work installing Gentoo.  Things were going fine, I was enjoying setting up *everything*.  But then I had to emerge X.org from source, and I got scared.  It was taking forever!  So I quit the install right there, booted up my gParted LiveCD, and wiped the Gentoo partition clean.

Maybe Gentoo is a little too complex for me right now, but I still have that empty partition, waiting for a new installation.  OpenSuSE 10.1 comes out in a couple days...

cheers,
groggyboy

How long is forever? The Xorg took me a few hours to download the source, and maybe around an hour or so to compile on my AMD64 3500+ machine. So if you were grabbing Gnome or some other WM at the same time, then it probably would take quite a while.

I ended up sticking with Ubuntu, though, as the Gentoo stable had older versions of things than Ubuntu, and the unstable was... well... unstable. Probably perfectly workable if you know the right tricks, but I, too, chickened out. 

You might wanna check out Sourcemage, though. Source-based distro similar to Gentoo. Holds your hands more in some places, less in others. But if you can get a desktop up and going, you'll probably have learnt a lot.

### Post by kolohan on 2006-05-20
I currently use both Fedora Core 5 and Ubuntu Dapper. In my opinion, Fedora is a lot more responsive than Ubuntu. I admit, Yum is a lot slower than APT, but hey, can't win 'em all. The driver support for Ubuntu was also kinda nice, but you can't beat Fedora's ability to keep up with the bleeding edge. Some people prefer stable releases, I like to live on the edge, though ;) . I say that the best way to enjoy linux is to have a dual-boot of two of your favorite distros. I use Fedora for email, internet, videos, and pretty much everything else. I use Ubuntu for some of the same things, but gaming is a lot better on Ubuntu. So, all in all, it just depends on how I feel that day. Fedora is a lot faster to me, and Ubuntu is a lot easier to maintain.
