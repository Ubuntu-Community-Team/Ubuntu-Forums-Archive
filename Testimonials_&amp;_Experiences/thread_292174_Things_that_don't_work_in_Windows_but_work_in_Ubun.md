### Post by Tweedicus on 2006-11-03
Hi,

Often the forum has posts frustrated because things don't always work in Ubuntu, or straight out of the box.

There are two sides of the story.

I'm a recent convert to Ubuntu, it's my sole OS, I don't have another OS on my computer. When my laptop was running XP it wouldn't go to sleep properly, I'd close the lid, it'd fake sleep for two minutes, then when I was out of the room, it'd wake up, and leave the screen burning under the closed lid. 

Second, there's always posts about wifi cards on the forums. My didn't work straight away on Ubuntu, a little digging around and fiddling corrected this, it now works all the time. In contrast, the same wifi card on the same computer hardly ever worked on XP, I'd have to reboot, repair, unplug it, sit around waiting for it to work. I don't have this hassle anymore.

So there are things that will work on Ubuntu that don't work on XP.

I have a laptop, that is now super silent, it runs cool and fan only comes on occasionally when it needs to. The system is more stable, and a lot faster than XP.

Finally, a recommendation. My machine is an IBM T23 Thinkpad, 640mb RAM, 1.13 GHZ processor, and 30 gig HD. I bought it for £200 off ebay about four months ago, they're now even cheaper. I run Edgy on it. This Thinkpad is faster than my 3.4 GHZ P4 Toshiba, whether running XP or Edgy. So if you were on a tight budget you can get a great machine complete with all the free software you need, that works fast, for around £150-£200 --which will probably be about the same price as Vista alone.

I google, and I find that there are 3 reasons why files are "hidden" in add/remove (as if it was a normal thing...): 

1) "show hidden files"+"show extensions" in file browser > folder view (how obvious!) 
2) id/display name longer than 63 chars (to be edited in the registry)
3) sysoc.inf file where you need to cancel the "hide" flag (behold: you need to edit a text file!!!)

Hidden programs to uninstall! How clever!

But I soon found out that there must have been also a 4th reason because none of the above methods helped. I also tried to go to the registry, find the uninstall command and execute it directly, but it did not work since it said that the uninstall executable had to be run from setup. Sure! If only the software was listed in the stupid thing!!!

After 1 hour I gave up and decided to use a "program to uninstall software". Apparently in windows, a lot of companies make money to allow you to uninstall software... I guess I was not the only one... I hate using binary blobs, but I did not see any other route. I picked a 30 days trial one (I had forgotten what a bliss it is not to have to look at the calendar to run apps...) and quickly googled to see if there was some malaware in it... Cannot trust anything in MS word, have to double check all damn binaries... The uninstaller at least came up with a decent list, Norton was in there. I had hope. But when I clicked "uninstall" it failed with exactly the same error I had... Obviously this great uninstaller tool was simply reading straight from the registry and trying to execute the uninstall command in there, exactly what I had tried to do 5 minutes before... 

When I was about to throw the machine, I stumbled upon an application from norton to remove norton!!! Wow, I did not know that software houses had to provide removal tools on top of uninstall software... That was cheating, all other programs were still hidden, but hey, finally Norton was not there anymore!!! I did not want to use XP one more minute. I was sooo glad not to be able to come back to my sane Linux machine that I did not care about _really_ fixing the problem... In Ubuntu I can pick any of the 20K+ programs provided, I install/uninstall any of them with one click... no hidden programs, no calendar triggers... and even if I wanted to play with unsupported software from source, I could easily use checkinstall to package them on the fly and have all the one-click advantages... or at the worse look at the installed files and simply remove them manually without worrying about left overs in the registry and other unknown places... Boy I missed Linux.
