### Post by kurt18947 on 2013-03-13
The O.P.s problem may have to do with the drive if it's an older machine.  I have a 2001/2002 laptop with a CD/DVD ROM drive (no write capability).  It's pretty fussy about which CD/DVD it will play nice with.  Some discs it will work one time and not another.  I attribute it to being old and crotchety:P.  Plus early generation DVD-ROMs were known to have 'issues' with reading writable media.  I also had one alternative installer refuse to work unless there were a network connection.  The installer would search for a network connection and would refuse to procede if it didn't find one.  Without a functioning live session, there's no way to get a wireless network connection.  Some users here only have wireless, no ethernet connection but can't establish a wireless network connection without a live session which presents a conundrum.  I converted an old router to a wireless bridge for just such occasions.

As far as the installer itself goes, I have no issues with it.  I use a 3rd party boot manager which requires the grub files be placed in the partition, not in the 1st sectors of the disk.  This is pretty simple and reliable using the 'something else' or advanced option.
