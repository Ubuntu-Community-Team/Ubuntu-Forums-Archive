### Post by theunixgeek on 2008-08-09
Don't get me wrong on this - I love Linux and [Ubuntu is one of my favorite distros]("http://theunixgeek.blogspot.com/2008/03/why-i-switched-back-to-ubuntu.html"), but there were some problems that temporarily turned me away when I was introduced to Ubuntu (and Linux in general) that I've noticed are affecting a lot of newbies on #ubuntu, too.

Intrepid Ibex (8.10) is coming this October and here are some things that should be brushed up, especially for new users:
[LIST=1]
[*]Screen resolution and desktop effects are really hard to configure out-of-the-box with NVIDIA and ATI hardware.
[*]In package management, the names "universe", "multiverse", etc. should be replaced with something more understandable, like "free with source, proprietary, etc"
[*]It would be nice if it were more obvious on how to install themes, because the first thought is to extract the (usually) .tar.gz compression.
[*]Please, more support for networking devices!
[*]Make sound configuration easier and better support for some speakers that might otherwise reproduce very very soft sound. Forget alsa, pulseaudio, and all those names that don't make sense; make it simple.
[*]Imitate GoboLinux and Mac OS X filesystem-wise.
[*]Better built-in documentation would be greatly appreciated by new users.
[/LIST]

One good idea would be to do like the early Macs and Windows versions: include an optional tutorial on how to use Ubuntu; even though the interface is simpler, it may be unfamiliar on some terms. I was switching from Windows and I was surprised that Preferences was under the Edit menu instead of the Tools menu. It's those tiny differences that count. (Maybe offer a separate tutorial CD?)

I'd like to see how well Ubuntu would do on a [Mojave Experiment]("http://www.mojaveexperiment.com/") test. That would bring more results as to what needs to be reworked.

### Post by PmDematagoda on 2008-08-09
> **iaculallad said:**
> To taunt the forum members perhaps :lolflag: This must be placed/moved in it's proper category: Testimonials section maybe.

And so moved.

About your points:-
> Screen resolution and desktop effects are really hard to configure out-of-the-box with NVIDIA and ATI hardware.
That is because of the proprietary nature of the Nvidia and ATi's fglrx drivers, since the Ubuntu developers can't look and change the source to make it easier on new users, its up to Nvidia and ATi themselves to do it.

> In package management, the names "universe", "multiverse", etc. should be replaced with something more understandable, like "free with source, proprietary, etc.
That is if you interact with the command line interface, if you decide to use Software Sources then the names of the repositories are a bit more "sane". And I don't think complete beginners would want to use the command line interface as soon as they switch to Ubuntu;).

> It would be nice if it were more obvious on how to install themes, because the first thought is to extract the (usually) .tar.gz compression.
This suggestion would be better if you had told it to the GNOME people themselves instead of asking Ubuntu to do it.

> Please, more support for networking devices!
Already in progress with the inclusion of Network Manager 0.7 in Intrepid, one of the goals of Ubuntu Intrepid is to make networking easier.

> Make sound configuration easier and better support for some speakers that might otherwise reproduce very very soft sound. Forget alsa, pulseaudio, and all those names that don't make sense; make it simple.
While this seems to be a present problem, it is not all that common either, so it is not a very valid point. If you want Ubuntu to create a sound system by itself just because the name is bad is really not worth it, look at what happened to KDE's aRTs sound system.

> Imitate GoboLinux and Mac OS X filesystem-wise.
I'm sorry, I don't know much about these file systems, could you tell me what makes them so special compared to the normal file systems?

> Better built-in documentation would be greatly appreciated by new users.
Once again, this would be better if suggested to the Desktop Environment projects such as GNOME and KDE themselves, although I don't think it would have a great effect since better documentation is a goal of everyone, with the only problem being the lack of man power to make it happen.

### Post by huxterby on 2008-08-10
I am a life long Debian user and have recently installed Ubuntu after not using it for a few years. There seems to be some confusion here on who the problems listed should be directed at.

Anything that relates to hardware such as graphics cards (Ati, Nvidia), Wireless cards (Atheros, Broadcom) should be directed at the hardware manufacturers.

It is very difficult to make a distro work with every type of hardware out-of-the-box if the manufacturers will not release the source to their drivers to the linux community. Nvidia for example.

The Ubuntu developers have done the best they can with what they personally control. They cannot however, take the responsibility for the lack of support by 3rd party vendors.

We have recently seen issues with Foxconn motherboards and their so called anti-Linux BIOS. All rosy now, but how many people with Foxconn boards over the last two years rejected Linux because it wouldn't install?

Users need to choose their hardware very carefully. Make sure it is Linux compatible FIRST.

Then start sending letters and boycott hardware which isn't Linux-friendly.

Huxterby
