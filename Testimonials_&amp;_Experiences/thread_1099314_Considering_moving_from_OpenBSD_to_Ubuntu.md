### Post by Alex Yeh on 2009-03-18
//begin late night rant...

I recently went through another kick of trying out various open-source desktop-oriented OSes, and tried Ubuntu again. Compared with my last two experiences with Ubuntu (trying it on ppc with a live cd - the X11 setup was unimpressive, and I couldn't reconfigure, then trying to install ubuntu x86 server edition on a pentium m laptop and booting to an error message saying that this version of the linux kernel does not support my cpu) I was pretty impressed. For one thing, I tried installing the desktop version instead of the server version. The desktop was very pretty and highly functional. I noticed that Compiz is at work behind the scenes making the desktop prettier without going all-out like Mandriva does.

The main thing, though, is the tight system and package management features (and apt-get ;)). Automatic, regular binary system updates are a nice luxury, and having access to such a huge list of binary packages of open source applications, updated regularly, is also nice. Being able to view Adobe Flash(tm) content is particularly enjoyable. Also, my girlfriend liked Ubuntu ;)

Installing patches and recompiling system tools in OpenBSD is actually much easier than it sounds. It takes maybe a minute or two, and is completely painless (if you are fluent in the command-line). I actually really like my little OpenBSD setup, running fluxbox and a customized pf firewall and conky and OpenBSD's custom pdksh. But, OpenBSD simply doesn't have the community or the developer resources to match Ubuntu. I will probably always use OpenBSD to some extent, since it is such an excellent system. But perhaps it is time for me to use a "proper" desktop system. I have been hesitant because of the Debian openssl disaster, but I guess everyone makes mistakes; hopefully the Debian folks have learned from that one.

### Post by Thelasko on 2009-03-18
If you're happy with OpenBSD, I would stay with it.  OpenBSD has a reputation for being the most secure and stable operating system around.  You might find Ubuntu to be "dumbed down" compared to OpenBSD.

In the end, I would say the optimal thing to do is have Ubuntu as a desktop system and OpenBSD on a server.  Ubuntu is very good in being capable of running programs like Flash, but also more secure than Windows.  I don't think Ubuntu can compete with OpenBSD as a sever, especially since you already know how to use it.
