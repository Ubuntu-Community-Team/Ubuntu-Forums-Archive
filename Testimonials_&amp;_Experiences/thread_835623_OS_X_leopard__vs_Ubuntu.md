### Post by jman623 on 2008-06-20
Hi All,

I have been running ubuntu as my only OS for a quite a while, and I am very pleased with it. However, I felt a little bit adventruious and decided to attempt to install OS X leopard through the OS X86 project.

I got everything to work except my resolution settings were messed up and I could only get one resolution. I guess it had something to do with my video card. I also had trouble with the sound but was able to fix that quickly with a patch. Surprisingly it picked up my wifi card straight away:guitar:

Other than that I was very disspointed with OS X. I found the buttons, menu bars, and location of everything to be very counter- intuitive. Despite the fact that the dock thing is aestically pleasing I didn't like using it too much, I'll take my taskbar thank you very much.8-) I also found OS X didn't offer as many apps straight away. For example it did not offer an office suite like ubuntu (and most linux distros) do.ichat was nice but didn't support that many protcols including msn.  

I also had trouble connecting to my printer which is hooked up to a windows machine.

I will admit though, it is very aestically, but I found it didn't offer me offer me much more than ubuntu can.

So I wiped my system and re installed ubuntu and I am happy again.:)

I might install OS X in vmware cause I like garage band other than, I'll stick with ubuntu thank you very much.

### Post by darrenleeweber on 2008-07-02
I've been using Ubuntu gutsy (amd64) on a thinkpad T61 and several desktops for about 1-2 years now.  I'm often frustrated by little things, like switching from work to home networks on my laptop.  I use Gnome (gdm) without any compiz/fusion.  Often I find that some applications are not placed into the 'intuitive' sections of the application menu.  My account is configured for sudo access, but I constantly get prompts for my password to authenticate admin changes - isn't it enough to login!  I use emacs a LOT and I hated the default emacs installation - I found the pretty emacs package finally resolved my problems (thanks to Andre for that!).

I love the debian package management system and I like the way Ubuntu try to keep up with recent developments, without going to the bleeding edge.  I love the linux kernel for the most part.  All my processes run very nicely.

In my new job, I have to work in a mac network environment.  So I'm trying to quickly learn the osx leopard system.  It's not difficult to learn.  I am struggling with keyboard and mouse conventions, but that should pass eventually.  I found some tips on customizing the keyboard mapping for terminals - that helped me a LOT.

I've found the macports system and my initial experience is good, after a minor hiccup.  It auto installs some environment variables into ~/.tcshrc - why not ~/.bashrc?  So I transferred the environment variables to .bashrc and that helped.  It seems to be an entire build system, so every package is actually compiled and it seems to result in /Applications/macports/*.app - I did this with emacs and the default installation is a very nice looking emacs (but I hate the way the right-delete key actually does left-delete!).  I found a way to launch these .app software from the terminal using 'open -a emacs' so I created an alias for that.

I'm considering using Xtools for development, but I'm hesitating because of license issues.  I really want to do open-source development so I'm wondering if I should stick to emacs or take a plunge into eclipse with c/c++, python modules.  I'm a bit confused about the mix of open-source and commercial libraries for OSX development.

I intend to install ubuntu as a boot option, so I can always fall back to this great platform if all else fails.  My main concern is compatibility of the file-system format.  I think mac is using some kind of file-system that might not be handled by linux - any tips?  I'm wondering if I will have to do a complete, clean install of ubuntu and osx after designing my own file-system partitions and formats.
