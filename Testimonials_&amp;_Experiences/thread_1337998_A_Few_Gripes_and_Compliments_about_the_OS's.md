### Post by Gene Rodgers on 2009-11-26
Well it's only my opinion, but Ubuntu 9.10, Kubuntu 9.10, and Xubuntu 9.10 all suck on my computer. However, Xubuntu 8.04.1 works like it should with a small amount of tweaking.
The problem I encountered with Ubuntu 8.04 & 9.10, Kubuntu 8.04 & 9.10, Ubuntu Studio 9.10 and Xubuntu 9.10 (I tried them all and all kinds of tweaking) Amarok does not have any sound (Amarok 2.2). Audacity won't record properly even with a sound mixer. Amarok 2.2 is missing some of the streams that Amarok 1.4 has. Shoutcast in particular. K3b burner will not burn mp3's to music cd unless you convert them to WAV format. I encounter no such problems with Xubuntu 8.04.1 (Hardy), everything works and works like it should.

My computer CPU is AMD Sempron 2200+ (1.5 ghz), 500 mb memory, 40 gig HD, sound card is built in SIS SI7012 and I have 2 cd/dvd burners. 
 
I've tried SuSe, Mandriva, Red Hat (Fedora), Whitebox, Slackware, BSD, aaaand the list goes on. They all have some good features and a few bad ones which I just couldn't live with.

P.S. There is another distribution which is running a close second to Xubuntu and that is Linux Mint. Everthing works without having to search the web for codecs and other needed files to make an application work. It also has a program which is a P2P file sharing software called FrostWire, which is included in the installable software package.
It will install on a Windows OS and dual boot. It builds a file on root of Windows which can be deleted if you decide it isn't for you. It is also a live cd like Ubuntu.

Gene - W5FFA

### Post by cariboo on 2009-11-26
You do reralize that most of the codecs you need to play most audio/video media, can be installed from the repositories. Open synaptic and search for ubuntu restricted extras. To play dvd's and a few propriatory MS formats. Enable the [Medibuntu]("http://https://help.ubuntu.com/community/Medibuntu") repositories and use synaptic to search for libdvdcss2 and w32codecs for 32-bit, w64codecs for 64-bit and non-free codecs.
