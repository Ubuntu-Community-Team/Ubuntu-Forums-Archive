### Post by pug on 2012-05-15
After using ubuntu almost since the very beginning I have sworn to myself that next time I reinstall it won't be Ubuntu.

Why? Because of so many frequent reboots its driving me insane.

I use Ubuntu as a combination of workstation and server and due to some finicky cryptography issues it is a big deal for me every time Ubuntu needs to be rebooted.

Rebooting any sort of linux workstation/server used to happen quite infrequently and as such was not a real problem. However, the last year or two the amount of reboots required to keep your Ubuntu up to date has been steadily increasing for no apparent reason at all.

Yes, I am aware that sometimes there are security bugs that need to be addressed, but there is just no way it can be this many.

I will use 12.04 until I am to do a hardware upgrade (in ~6 months) and if things have not changed in the mean time its bye bye Ubuntu.

This issue is of course not the only thing that bothers me, but it is simply the last straw. 

The lack of listening to the users, going off in a direction nobody really wants because of some "vision" - the inconsistency of upgrades (turn on default X screen blanking without an option anywhere in 12.04 to turn it off? Come on! No user should need to fire up a shell and do "xset s 0 0" in order not to have screen blanking!) the whole Unity debacle and last but not least the consistent degrade in desktop performance over time (even using XFCE!).

So, thank you for the ride Ubuntu, you used to be quite perfect.

-pug
