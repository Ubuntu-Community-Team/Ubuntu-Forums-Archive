### Post by cariboo on 2009-10-08
Your assumption the Linux uses memory the same way as Windows is wrong. Linux will cache a program in ram, until another program needs the space, in Linux we say free ram is wasted ram.

Instead of using gnome-system-monitor, which adds it's own over head, open a terminal and type:

```
free -m
```

here are the results from my system:

```
free -m
             total       used       free     shared    buffers     cached
Mem:          3019       2993         26          0         46       2370
-/+ buffers/cache:        576       2443
Swap:         3812          1       3810
```

According to the listing, all my ram is being used, but if you look at the line that starts with -/+ buffers/cache, my actual ram usage is 576Mb.

You can't really comapre linux to Windows, as they operate totally different.

### Post by cariboo on 2009-10-08
Linux will use ram to cache programs that you have run but closed just in case you need the program again, it will release the ram for other programs as needed. To try it, open your favorite browser, close it, then reopen it agian. I t will start up faster the second time because it is cached in ram.

Windows XP treats ram as a resources, to be used only when needed. This bahavior has changed in Vista to be more Linux like in the way it uses ram. Vista uses ram as cache and uses almost all of it as soon as it becomes available.
