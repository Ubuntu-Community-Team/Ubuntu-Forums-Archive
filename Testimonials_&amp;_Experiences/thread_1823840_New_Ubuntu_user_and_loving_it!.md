### Post by decoherence on 2011-08-16
In general, you can use either aptitude or apt-get but use apt-get for the sake of consistency. You are correct that aptitude is like a console version of the software centre though as you've seen it can also be used to directly install a specified package right from the command line.

Aptitude can come in handy if you really stuff up your dependency tree, generally by trying to run different releases at the same time. This is not really a problem in the ubuntu world, it's mostly for debian people who are running testing/unstable and got stuck because they ran dist-upgrade -fy and ignored warnings about breakage. Or something like that ;) Anyway, aptitude can often fix a botched dependency tree when all else fails but Ubuntu users should never need to unless they're doing something very strange.
