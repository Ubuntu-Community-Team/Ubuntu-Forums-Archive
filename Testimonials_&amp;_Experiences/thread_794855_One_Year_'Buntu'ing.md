### Post by Geekkit on 2008-05-15
Well, I hit the one year mark ... been using it since the same time May last year. I had tried 6.10 but had kernel freezes when I plugged in my Nokia 6126 via USB. When 7.04 came out a kernel patch fixed that problem.

When 7.10 came out I immediately migrated (with excitement) although I did break my own rule which was not to adopt anything too new (I usually wait several months). At home I am still running 7.10 and am happy with it. Currently running an older system with only 512 MB RAM but 7.10 fits quite nicely in it. Run VMWare for the wife (she now has an account and is okay buntu'ing as well but jumps into VMWare when she needs to use Simply Accounting).

Can access my Nokia phone via USB using ObexFTP and ObexFTP Front-End, using qtpfsgui for HDR images and tone mapping. Using Netbeans for development, Geany for quick and dirty, GANTTProject for project files, Scribus for DTP, OO for office applications, Inkscape for vector graphics, GIMP for images, Brazero for CD/DVD data backups, and Glade 2 (don't want 3 ... I'm a control freak and want to keep track of all the code). I've only had a few problems with it (7.10). A few glitchy things I notice with it (and the software in it) are:

- FF 2 chokes on large pages or several tabs from YouTube.com
- OO sometimes goes nuts with the CPU and doesn't exit properly
- The Tracker daemon sometimes goes nuts with the CPU (I've disabled it)
- Virtual box seems unable to install and not mess up the kernel or video card driver (or both) ... VMWare works perfectly though so I have that need satisfied

My office computer switched over to 8.04 but I've only been running it for a few days and not really taken it out for a serious drive but it does indeed seem more peppy. I was disappointed that the undo/redo feature that they promised in Nautilus isn't there. :( I am a klutz and being forgiving is a real major step for Linux in making a dent in the desktop market (and helping me undo silly things).

I think that there will be three things (and in the listed order) that will help ensure the success of Ubuntu:
1.) Stable kernels
2.) Easy-to-use UI
3.) Supportive community

While Ubuntu enjoys most of this, I think the one item above that needs the most work is number three. One thing I have noticed in this last year visiting here is that it is apparently no longer cool to post gripes or negative feedback about Ubuntu in these forums.

There needs to be more listening instead of "don't let the door hit your *** on the way out" comments or "it's your fault for buying incompatible hardware" comments. Chances are, if someone is already here trying stuff, it's because they are looking for an alternative. Best thing you can do be supportive.

Anyways, my two cents. Either way, it continues to be interesting and I am using Ubuntu because it allows me to be productive, something that drew me to it in the first place.
