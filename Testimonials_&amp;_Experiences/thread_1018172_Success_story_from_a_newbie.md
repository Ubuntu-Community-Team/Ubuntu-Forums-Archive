### Post by pshootr on 2008-12-21
I am happy to report that as a linux newbie, I have managed to setup a pretty nice file server for my local network to use. Within a couple of short weeks I have turned a old 500mhz gateway pc in to a fine linux file serving machine. My whole motivation was to be able to download off the net from my laptop, but send the data to a networked pc (the linux file server just built) instead of saving the files to my laptop which has limited space. I learned how to configure samba to share the servers storage drives with my windows machines. So now I can sit on the couch and download torrents with out worrying about how much space I have left on my laptop. It is awsome. I have also learned how to configure my server from my laptop using a secure ssh protocol 2 method which involves a secure authentication key. That means I can use putty from my windows machine and log in to the linux server and do anything I want to do with the server. Another very cool thing I have learned is how to use WinSCP to do secure log in and file transfers to and from the server from my laptop or from any other pc connected to the internet. Anyways the point of this post is to show that with some desire and patiance, you can learn alot. After all look what I was able to do in two short weeks. Hope this inspires some of you newbies like me, to keep at it and have fun with it. I would also like to thank this community for all the help that made this all possible. :P

  For this server I used: ubuntu 8.10 server (intrepid)

### Post by Liviu-Theodor on 2008-12-22
> **bgast1 said:**
> I guess I must be really dumb, because I haven't got a clue what you just said.:) I don't know what a file server is, so I guess I really don't need it, except that I am always curious and always want to learn more. That happens to be something that I want to dig into.

You are not dumb, just because you don't know what a fileserver is. But I explain in a few words its role. For example, in a school you have a lab with 30 computers for pupils. They must access the same files on each computer. If the files are big, there will be a waste of storage space, if you store a copy on each computer. Instead, you could have a fileserver, which stores the files needed in common, and shares access to each computer. That is the role of a fileserver: to store files needed in common by more computers. You need a fileserver only if you have more than one computer (yes you can have a network of just two computers, one being used as a fileserver).
