### Post by nestawasright on 2011-02-26
Lots of people are sold on this OS but very basic things go unfixed from one flashy version to another. Here's a list and brief description of consistent problems:

**Nautilus**: try copying large files (say 40GB) from an external harddrive to an internal harddrive using USB 2.0. Results, it takes forever. I started this this morning at 7 am, it's coming up to 2 pm and it's still copying.

Why can't **Nautilus** be fixed and be able to mount Windows NTFS network shares? It says there clearly, Windows Network, but time and again, it simply won't: "Unable to mount location...failed to retrieve shared list from server," is the constant cry on that one.

**Wireless**: Why can't it be that the OS support wireless cards natively? Why would it be that a kernel update should mess up an existing working wireless card?

**Network manager**: Time and again people complain about Network manager. And like all of these from version to version, it goes unfixed. Yes, WICD is good, but it's shame that the native app in the OS time and again, does not work well, or work at all. Meanwhile, the OS marches on to the next glossy version. 

**GRUB** The promise was that GRUB2 will fix much of the problems with earlier versions. Most people are left praying that they have the option to go back to that earlier version now, where you could edit the file easily, for instance.

A GRUB GUI would be nice too, as it's quite insane the option to delete Kernel version using Synaptic Packet Manager. The chances of making a mistake and deleting GRUB and not been able to log back into the machine are, well, great. 

These are just a few annoying things about Ubuntu and I'm sure there are tons more out there. You only have to search the threads for the seeming timeless problems that continue on, from one glossy, flashy version of the OS to another. 

These are just the problems I keep having year on year, release after release. I'm sure there are others out there with similar ones. These, to my mind, are very, very basic issues that should be fixed. What's the point of spending time creating a newer OS of the earlier broken one, without fixing the earlier version? Makes not sense to me. 

Instead of the next great release, perhaps it would be good to work and fix these persistent problems and make the user experience a pleasant one. When I use Ubuntu knowing that I was simply toying with it, all these problems were OK. But for a matured OS, this is a shame.

### Post by Fran7842 on 2011-02-26
Wireless Support seems to be much better in the past year.

I remember a time when I had to manually install ndiswrapper to get my linksys driver to work.  Nowadays, I can usually connect wireless from a live CD of 10.04 Lucid Lynx.  If I choose to install Lucid Lynx, it seems to install automatically the exact drivers needed for my wireless and wired Ethernet adapters.

I haven't noticed any improvements to grub.  But there is very good documentation available online for free.  I once got the dreaded Grub rescue screen at boot-up.  I went to the forums and was able to purge and re-install grub via my live CD.  Now, the thread I used was two years out of date, but it was not that hard to figure out how to modify the listed steps to fix the current version.

I don't know if I've ever transferred 40 GB of data quickly, but I'm willing to bet that if the file being transferred were saved to any of the open source systems common in linux rather than FAT32 or NTFS, it would have transferred faster.

I am very thankful that since switching to Ubuntu, I have never had to defragment a hard drive.  There is something horribly wrong with a world where free software for file systems functions drastically better than locked down, proprietary, and over-priced filesystems don't function properly.

### Post by Vaphell on 2011-02-26
> The only thing I don't like is GRUB, especially GRUB2. If you have two hard drives with Win on one and Ubuntu on the other, it is a job and a half trying to boot into Ubuntu if the HDD with Win on it goes down, especially for a beginner like I was when it happened to me last year.

reason for that is that boot info (with pointer to the ubuntu partition) is usually stored on a primary drive (windows partition in your case). When it fails, boot order of drives changes or device identifiers change for whatever reason, you are left with invalid configuration - there is no grub launcher at all or data is messed up and pointers don't point to where they are supposed to point. There is not much fault in grub itself, more like a consequence of PC architecture.
