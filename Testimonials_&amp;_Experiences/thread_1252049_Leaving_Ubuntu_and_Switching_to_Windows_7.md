### Post by Wlo on 2009-08-28
With the official release of Jaunty I switched my desktop from Windows XP to Linux (64 bit Jaunty on my quad core, 4gb RAM, 128gb SSD).  It would be the third time I tried to switch to Linux, the last time also being a switch to Ubuntu around the time of Feisty, and the time before that being RedHat well before that.  After a few months I'm now switching back to Windows 7.  I'm not trying to knock anyone here, I'm just providing my feedback incase it might be helpful.

There are so many things I love about Linux, but some things which seem fundamentally simple with Windows are an uphill struggle.  I'll list some of the problems I've encountered below.  Also, there are some apps I just need Windows for (Photoshop, visual Studio, Ableton - first two are fine in a VM, but Ableton needs to be native), and I know that's not Ubuntu's fault, I'm only adding it because maybe it makes me feel easier about switching back.  I'm a Python and .NET developer, and for Python dev I love Linux, I think that's another reason, it's stuff I can run easier in a VM the other way round (ie, native Windows, Linux guet), and I'll miss the command line, and how everyhing is only a bash script away.

I love the idea and philosophy of OSS, only, not for my native desktop OS just yet.  Hopefully I'll try again in a couple of versions, maybe when the next LTS is released.  And I wouldn't dream of running anything else on a web server, obviously.

Anyways, keep up the great work, my issues are below.

- Firefox and flash are so unstable.  32 bit Firefox 3, 32 bit Forefox 3.5.  64 bit Shiretoko, 32 bit Flash plugin, 64 bit beta Flash plugin, a million different walkthroughs and tutorials.  It kind of works, but crashes frequently.  I realise this is one app among many, but the fundamental practicality of it is, it's the app I need most frequently.  Whilst I would have thought it not important, actually I found many times when missing flash caused me probs (GitHub graphs, music demos, youtube).

- The VNC bug.  Hey, there was a workaround in the end, to use FreeNX, but it was frustrating until then all the same, and there doesn't seem to have been any aknowledgement, let alone a fix.

- This Brasero bug.  [https://bugs.launchpad.net/ubuntu/+source/brasero/+bug/385396](https://bugs.launchpad.net/ubuntu/+source/brasero/+bug/385396).  "Error While Burning" - You do not have the required permissions to use this drive.

- I can't print properly in Text Editor or Open Office Word.  It was working fine before.  No doubt I'll fix it with a couple of hours of Googling, but I've had enough of doing that.

- Sound problems.  This bug, [https://bugs.launchpad.net/ubuntu/+source/pulseaudio/+bug/366708](https://bugs.launchpad.net/ubuntu/+source/pulseaudio/+bug/366708).

It mostly works with the workaround, but not always.  The decision to actually switch back actually came when I just wanted to put some tunes on but couldn't.  This is stuff which is fundamental to me.  Someone suggesting using OSS to get sound from one app at a time (but not flash), but that's not really an answer.

wlo.
