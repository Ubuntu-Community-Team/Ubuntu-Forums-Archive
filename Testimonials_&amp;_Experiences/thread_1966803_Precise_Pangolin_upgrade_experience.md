### Post by doorunrun on 2012-04-27
I took the plunge last night and did a fresh-install-upgrade of my system from 11.10 to 12.04. I used the alternate install CD. 

This was my first time not doing an in-place upgrade. I used Deja-Dup to backup my home directory onto an external USB hard drive. This system is not too elaborate and the home directory didn't wasn't too bloated; but it did have some important documents I didn't want to lose.

During the installation of 12.04 a message popped up wanting to know it I wanted to supply a driver, rt73.bin, on a USB stick. I searched around and thought I had the right thing, but the installer couldn't use it. It was a driver for an internal USB wireless controller. Since I had a wired connection, I finally chose to ignore the message and press on.

The rest of the install went by and I booted into the Precise Pangolin world. I then went back to Deja-Dup to restore my home directory.

Not too far into the restore process, I lost Unity and got a blank desktop. I guess I had just written over my Unity settings. But it restored just about every setting. I had to go through the process of logging out and use a virtual console to remove the existing Unity settings and then log back in to get the defaults rebuilt.

After I had regained a normal desktop I found the system was very functional, the application's settings were all there. I got Chrome (my default browser on the old set up) installed and there were all the bookmarks and extensions working.

I'm now more confident in the fresh-install-upgrade. Over all it was a good experience. AND, the wireless was also working! I'm still trying to figure out why I got prompted for the driver, but there must be some other way to get the hardware working.

Congratulations Canonical and all the hard working development team!!
