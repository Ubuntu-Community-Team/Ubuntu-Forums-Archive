### Post by primary0 on 2006-11-20
After installing Windows, you need to go to websites to get WinZip, WinAmp, AdAware, Firefox and the rest of whatever you need; after installing Fedora (for a desktop) you need to go through a similar but more tiring ritual to install mp3 support, VLC, Java for Firefox, Flash, Acrobat, groom the startup services and you will need to go to rpm.pbone.net (with fingers crossed) everytime you need to find something. Using a computer has been easy since Windows 95 (we were all in "awe" remember?) but setting things up never got easy and it grows on you if you have to do it often.

Setting up a server on Fedora is just as easy or hard as it always has been (sendmail setups are headaches, apache, squid, postfix, etc much simpler, etc goes on). There is very little or no progress on what i call "Usability for the Sys Admins", especially in the Linux world. Microsoft server setups have alwasy been very Window-ish, and it suddenly became cooler to manually edit config files using vi. From a usability point of view, Windows users were a step ahead in the evolutionary process of user interfaces. Even now, some simply play down even using gEdit to change an apache configuration file, using vi is STILL cool. But it is time many stopped being so anal about it. Before vi it was cool to plug vacuum tubes but come'on already! GUI based admin tools are COOL, and its perfectly OK if you do not know how to use vi.

Ubuntu was the biggest surprise since Windows 95. Now setting things up can be easy too. Especially on the parts where you have to look for it, find it, get it, install it, keep udpating it. After enabling the Univsersal Repo - its really a 1 click process to get almost ANYTHING on Ubuntu. No more rituals. I had a cheap USB bluetooth dongle (which happened to be connected when I setup) and I was able send files to my phone already, (I MUST get a specific driver from a specific website for Windows). Yes all that can be done on other OS'es too, since long time. But this is about a little bit of setup effort compared to no setup effort! Ubuntu is the only bundle that stretches the "good user experience" thing to ALL types of users. I am not talking about the grandma stereotype, I am talking about the advanced/expert users who would certainly feel a lot at ease with Ubuntu than on any other OS. You won't regret it.

### Post by weresheep on 2006-11-20
Hello, I am glad you are finding Ubuntu such a breath of freash air.

I would like to point out that while some people you may have encountered were being anal about using such tools as vi, I suspect many have very good reasons for their opinions.

Vi (or its modern equivalent, VIM) is a powerful text editor which, once learned, offers a fast and flexible environment with which to edit text files. Whilst Gedit may be a fine text editor it is limited by being a GUI program. Many *nix systems run without a GUI environment and therefore are restricted to running command line programs. In addition vi can be relied upon to always be part of a *nix system thus offering easily transferable knowledge between various *nix computers.

For a desktop system though you are quite right in saying users should be free to use any editor they wish but I don't think learning vi would ever be a waste of time.

As GNU/Linux and the BSDs move more into the desktop world the addition of GUI interfaces for administration tasks is inevitable and, I believe, a good thing. However, GUI interfaces are not a silver bullet or magical fix and wanting everything to have a GUI may ultimately lead to dissapointment. Some computing tasks are complex and sticking a GUI in front will not make them simple.

However, so long as all configuration is stored in text files we can enjoy the best of both worlds.

-weresheep
