### Post by Dweomer on 2010-01-20
I've been using Ubuntu for a week now. I've had no shortage of problems, but that is to be expected when making such a major switch from Windows to Ubuntu. There are pros and cons, and those pros are excellent, while I believe there is a chance the cons could be reduced or eliminated in time.

Pros:

It's FREE. This was the biggest draw. Microsoft abuses its monopoly in startling and sometimes illogical ways. If the cons are fixed to make a truly user friendly OS, Ubuntu could eventually be a viable alternative for the vast majority of the computer-using population uses PCs for - internet and multimedia. Its only long-term hindrance is the additional computer literacy required to run and configure a virtual box or compatibility layer like Wine.

It's FAST. It's light and quick and you don't even have to defrag it. It boots fast, it loads programs fast, and from what I hear it stays fast for much longer than Windows.

The Software Center makes getting set up easy. It could be improved further, but that doesn't keep it from still being awesome right now.


Cons:

The biggest problem right now is the terminal. For non-programmers this is a nightmare. The terminal would be a nice opt in for all the advanced users out there, but right now it is the primary means of input and the GUI is secondary. This is a problem for true usability for the less savvy users. I've made an effort to use it -- I even keymapped Alt + ~ to open it for all that I've been trying to use it -- but to no avail, or very little anyway. This brings me to the second con.

Ubuntu is impossible to learn without help from the Ubuntu community. These forums, while not part of the OS, are integral to the success or failure of a new user to be able to learn enough to get to the point where they can start learning the little stuff they don't know on their own. It's almost like learning a language - they don't teach all the words, they teach enough to learn the rest on your own. To put it another way, there's too many fish-dinners and not enough fishing courses. An example is that in Windows you can simply look at a shortcut's properties to see exactly what directories a program has been installed to; in Ubuntu not only can you not do this, but neither google nor the community was able to tell me an Ubuntu alternative to finding out such critical information on one's own. A traditional forum wholly inadequate for the purposes of growing and maintaining a larger (if less computer savvy) Linux community. I would suggest a better format, possibly integrated into the OS itself, and hell, since I'm in dream-land already, why not throw in some light organization too? It's certainly not impossible, but lacking the community will to rectify this problem, it would probably just be easier to solve the first problem and thus nullify this one.


The short version:

Ubuntu is free, fast, and has some convenient features, but it has an impossibly steep learning curve and lacks the logistical means to bridge that curve. All in all it's a viable alternative to Windows, but needs a lot of work if it's going to become truly user-friendly. At the moment, it simply isn't.

### Post by caravel on 2010-01-20
The terminal is not a problem - you simply need to shake off the Windows mindset.  In day to day use I hardly use the terminal.  I only use it when I am fixing or installing something.  It's rare that I am doing the former these days and the latter can be done through synaptic in most cases.

Windows lumps programs into their own directories, whereas Linux keeps program files separately according to what kind of files they are and who is allowed to access them.  This is not only a better way to store and access files it's also vitally important for security reasons.  Most of the config files that you might want to edit for a given program will be in your ~ directory anyway.

None of this is "wrong" because it's not windows-like, it's just different and, in my humble opinion, better.

It's also very possible to learn how to use Linux without the help of this forum.  Most sites can help you out with Debian-like Linux OSes because Debian is so common.

I run Debian, which some consider "harder" to get to grips with than Ubuntu.  Not true.  It's ridiculously easy to install and use.
