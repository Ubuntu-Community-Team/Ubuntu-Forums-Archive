### Post by Spr0k3t on 2010-06-12
So let me put this out there first and foremost.  I've converted many people over to using Linux in general.  Most of the users I've converted over use Ubuntu as their primary operating system.  Some of the users have gone completely cold turkey, while others remain with a dual boot system.  In total, thirty-eight individuals and 47 computer systems.  Of those, 42 (perfect number) are loaded with Ubuntu as the default system.

The upgrade process from 9.10 to 10.04 has been horrid.  So much to say that I've asked the end users to hold off on the upgrade until after I can be there for the change over.  The success rate of a clean upgrade (methods stated later) has been closer to 27%.  The 100% success systems are hand built using hardware that is known to function 100% in Linux through support of the hardware company or support of the Linux community in general.  However, there are two of these hand built computers that required backup and clean install due to the excess jump of the last LTS release.

Two of the computer systems were netbooks.  One an Acer Aspire One (110L) which I had to wipe clean and install new.  The upgrade, for some reason, forced the keyboard inactive after the kernel was loaded... so even trying to access any keyboard to go into recovery mode from the grub menu was not possible.  Instead, the backup was a snap so I installed clean.  Instant success afterwords.  The other netbook was a Dell Mini9, that went through without any problems... with the exception of the audio microphone... but that was a simple configuration change.  Both systems had been installed from scratch using 9.10.

I had three laptops which used the Intel i815 chipset.  These systems were mind numbing to resolve.  After much hair pulling and various four letter words thrown out there, I was able to fix them following a post in the sticky of the general forums.  Many kudos to those who found the issues with the i815 chipsets and resolved them.

The upgrade paths taken were all done one week after (with the exception of my own personal system) the release of 10.04.  Most of the upgrade bugs had been found, posted, and workarounds given.

With all the headaches I've had in support of the end users in my area, I've started looking at other distributions to see if there are better solutions for general consumers.  What I've personally found, I've come full circle every time back to Ubuntu.  Mint is about as close as I would come to selecting of a different distro, but it would require reconfiguring the methods used in the panel/launcher.

So to recap... my support to Ubuntu is going to stay even with all the problems and issues I've seen, now stronger than ever.  There in simply put: "Elegance through simplicity" that I can't find anywhere else.  Not to mention the end users out there who are willing to go that extra mile in lieu of support.  Kudos to everyone who keeps Ubuntu going.

If you have helped another find a solution to their problem, thank you.

### Post by tgalati4 on 2010-06-12
I find that linux mint is polished with all the codecs and stuff that people have to load anyway to use their computer.  That can save time if you have a lot of users to support.  Also, because mint is released later than ubuntu, it tends to have a month or two of patches which can make the upgrade process a little less painful.

Because Lucid is an LTS, I would have waited 6 months before subjecting new users to the upgrade path.  If you get inquiries, then it's time to perform a complete backup for the user before trying to perform the upgrade.  I prefer dual boot with new distros because of the breakage with new releases.  Also, dual boot gives you a clean installation, but it takes up some disk space which can act as a backup and be reclaimed later.
