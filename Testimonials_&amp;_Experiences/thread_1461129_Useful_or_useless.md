### Post by akand074 on 2010-04-23
"Linux assumes you know exactly what your doing"

That's a quote I've seen around often in the linux community. Now in operating systems like Mac OSX and Windows its quite the opposite. More along the lines of it "assuming you have no idea what your doing". Hence I find its easier to get started, easier to do the basics. But the more user friendly something becomes, generally the less efficient it also becomes. Its not always about the first impression, its about taking the time to learn and really know the operating system before judging. 

When I first got introduced to Ubuntu, I just really liked how cool it was. I had lots of issues installing it on my computer at the time (it was very old it could barely run the live cd). When I got my new computer it installed and ran well but I never really used it I just liked the novelty of it, mostly Compiz Fusion. Anyways one day I got fed up with all the issues I was hitting with Windows Vista and booted into Ubuntu and swore never to go back. And I haven't since (on any of my desktops I've had since anyways). Anyways, by default Ubuntu uses open source drivers and such you have to install/activate proprietary drivers and pretty much make it the way you want. 

I've never had the issue you've had even before I knew anything about Ubuntu, but its unfortunate that it happened and you had a bad first impression. But it really is a great operating system. Windows and Mac OSX are both great too depending on the user. For me, they don't quite compare. For a more simple-minded user, something like Mac OSX would be great, and for software nothing beats Windows just yet. Main thing, I wouldn't let one try put me down, I personally would be troubleshooting right now because more often than not its MY OWN fault for the problems that are happening and not the operating system.

EDIT:
Also, you can't watch a DVD in Ubuntu by default because of piracy and other laws, so you have to manually install the software you need to allow it (also Ubuntu only installs open-source stuff like I said before so if its not open-source you have to install it yourself) read here:

[https://help.ubuntu.com/community/Medibuntu](https://help.ubuntu.com/community/Medibuntu)

Also menu.lst is no longer used in the new ubuntu, that file has been moved, read up on the new Grub here:

[https://wiki.ubuntu.com/Grub2](https://wiki.ubuntu.com/Grub2)

Premissions is never a problem, whenever you want to do anything that could affect your system (installing new software, playing around with drivers, mounting a drive) it'll prompt you for your password, type it in and your good. Makes it much safer so drives aren't mounted and information stolen without your permission, software you don't want on isn't installed without you allowing it first. If you want to run your file browser as root so you can edit system files just type:

gksudo nautilus

into the terminal and you'll be in root. Hope this helps.

### Post by 3rdalbum on 2010-04-24
Sorry, but if you want to use something different to what you're used to, then you will have to learn.

Vista should have been added to the boot loader, no idea why it wasn't. I've never seen this problem occur.

For security reasons your default user account doesn't have full access to system files. However you can temporarily  become root through use of the sudo command - just put the word "sudo" before any command, or put "gksudo" before any graphical program. If this is too difficult for you, install the "nautilus-gksu" package and you'll be able to right-click any file or folder and choose "Open As Administrator".

The default security policy will not be changed in Ubuntu. It's a very safe and secure policy, and you'll notice that Mac OS X and Windows have a similar security policy.

As for DVDs, you need to install DVD decryption software. This is exactly the same situation as on Windows. There are plenty of guides around about how to do it. Through legal reasons you can't ship DVD decryption unless you've paid for a license, and of course since Ubuntu is free there's no way Canonical could pay for this (buying ten million licenses is a bit costly for a free product).

You really need to get used to the security policy - because this is the way computing is moving. You can't stick with Windows XP forever.
