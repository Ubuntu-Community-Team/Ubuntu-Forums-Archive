### Post by goran01 on 2012-06-30
new to Ubuntu forums

well, it is hard to say this but 12 is not exactly what I was looking for. there is at least one excellent improvement... better virtual driver support. in hyper-v the mouse works and there is a generic network driver. very good.

now the bad. i have no idea what drives our fellow developers to concentrate so heavily on GUI changes. i think this has gone overboard. each time i try out a new operating system from Linux or Windows series, the things are just shuffled around, from my experience mostly without any sense or benefit to the user. but there is a steep learning curve and a penalty for such mindless work. 

in the last several years i tried each new OS with excitement and hope that essential features will be easy to setup and use. and each time i am disappointed. instead of ease of use I see a new GUI. This desktop, that desktop, classic, new, futuristic. Like all the computers are for is web surfing and social media. Come on guys....

with Ubuntu, I could use a simple FTP, VNC or LAMP (basic developer tools) setup. click on a button and it is done. nothing of that is available. why not?

system help is in accessories! accessories? it should be in it's own main menu option, not hidden somewhere below.

I do not really understand what is happening... i really do not. we had exciting progress in OS up until perhaps 2005 and then more or less just goofing around. i know it is not always easy to perfect lower level system features. but that is not a reason to then focus on GUI. please no more GUI changes and nice pictures. pretty please!

### Post by TheFu on 2012-06-30
I'm a CLI user mainly, but I do run a few Ubuntu-based desktops.  The fantastic thing is that you have a choice.  You don't need to use the default GUI of you don't like it.  I don't.  In Linux, the GUI is like a paint job on a house, not the house itself.  The A/C, heater, stove, floors, walls, windows, stairs, bathrooms, kitchen are all Linux underneath.  

The base OS in Ubuntu 12.04 is solid - seriously solid.

Sure, I'm not a Unity lover, but changing to a lighter GUI - LXDE - is pretty bonehead.
(sudo apt-get install LXDE), so I'm not complaining.

As you the "Help System" - Linux is more _UNIX-like_ in that aspect.  UNIX has "man pages" which explain everything - - EVERYTHING.

It is hard to explain how much real information is contained inside the man pages.  Open a terminal and type 'man sudo'  - READ the entire article.  Sudo is amazing. It does things you never knew.  This is actually one of the best man pages around.  There is a man page for almost every command or program or configuration file on UNIX systems.  A few non-compliant projects still insist on using other formats for their help - 'info', textile and HTML are also common.

Life is about change. Humans handle change better than others.  If you prefer to stay with the older version of Ubuntu, go for it. It is still stable.
