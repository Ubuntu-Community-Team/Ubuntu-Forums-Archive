### Post by Stoneface on 2009-07-19
Well, I read one rather long post by a very disappointed guy on Ubuntu Testimonials & Experiences and other really positive ones and I thought... let me add an average one. Mine to be precises. 
I have been using Ubuntu parallel to Windows XP on my Acer Aspire 4520G laptop with 2GB RAM, 160 GB Harddisk and Nvidia Geforce 8400) (and a basic Desktop which I don't use to list pros and cons here) for more than 2 years now and no I have not decided to pass over to the other side. 
FYI The main thing is that webdesign is just great using Adobe packages under a MAC or Windows. I design and develop websites for a living.

Let me make a list of my pros and cons of the Ubuntu Jaunty Jackalope | Linux | Gnome combi:

Pros

[LIST]
[*]Shell works really well and fast (more of a cross Linux command line tool, but I thought I'd mention it anyways.
[*] Firefox is a great choice. I love it. What happened to Firefox 3.5. Where's the automatic update option?
[*]Rhytmbox box works really well. I like it a lot!
[*] DeVeDe and Brasero work well, but need more experience before I can give a balanced opinion.
[*] Appareance preferences & options ae awesome (splash screen, mac like effects)
[*] I run a complete LAMP on my Ubuntu Desktop and it works like a charm. Awesome!
[*] Virtual Box works really well with the help of a tutorial I found. This was vital for me to test sites under Internet Explorer.
[*]Gnome is a nice GUI and well chosen by Ubuntu
[*]GIMP is great. Except for: see point under cons
[*]Aptitude is a great package manager
[*]Synaptic Package Manager works really nicely as a GUI package manager
[*]Open Office is a great for text editing (I know this is separate OS Software, but part of the Gnome package so I stated it anyways)
[/LIST] 

Cons

[LIST]
[*]Open Office Spreadsheet and presentation is not like its M$ equivalent yet
[*] GIMP is quite ok, but as long as it cannot work together with Photshop - read open psds properly with all layers present - its worthless in my line of work. I receive designs as psds all the time.
[*]Wireless works half of the time using my Atheros wireless card and I had to use madwifi which I had to compile quite a while before Ubuntu supported it. This is a big turn off. To compile a wireless driver is a big hurdle for an end user. I might be a bit more experiences by now, but I know many people won't have gone this far. And now that compiling is no longer needed it works half of the time. I need to reboot often just to get it to connect to my Linksys using WPA2.
[*]Kompozer is nothing compared to Dreamweaver and I prefer Gedit to it so it doesn't need to be added to Ubuntu/Gnome until they've improved it. Why this has not happened already I don't know. Maybe they could join forces with Firebug (css editor + basic gui for page lay-out)?
[*]NVIDIA Accelarated Graphics (propriety hardware) sped up things, but damaged a proper shut down. I see 9 command prompt after shutdown since I activated it in hardware drivers..
[*]Skype calls and Skype out has never worked well. Believe me, I've been at this for a long time. It doesn't function. Why not use Gizmo5 and make it part og Aptitude as regular package? Waiting for a new Skype for Linux is a waste of time.
[*] Sound under Ubuntu using PA and ALSA just is no match for Windows XP. Some people recommend another distro. I have not made that decision yet. On my Acer the sound is too soft and Skype sound is crap. Ubuntu audio (ALSA + PA) has never functioned well.
[*] Every time I shutdown I get an annoying beep. Why?
[*] Every time at startup brightness is lowered, why?
[/LIST]

NB Inkscape as alternative to Illustrator has not been added to the pros or cons list as I haven't tried it yet.

If I cross over this year or later on or just keep it using for web development at the side will depend on progress on audio/communication (Skype + Pulseaudio) and wireless. I can (more or less) live without Photoshop and Dreamweaver, but I need well functioning working audio and Skype (or Gizmo5 maybe?) and I need my wifi to work out of the box.

Comparisons have no value unless done in a controlled environment with guidelines and matching hardware. Expectations must be minimized and generally when endeavoring to compare one system (OS) to another - it's the human aspect that fails. We often cannot accept the fact that something different may be better. But rather choose to berate it simply because it's different. Or just deduct points for being different.

Every OS had it's benefits. Comparing one over the other is generally a waste of time, unless performed in a controlled environment without an objective (other than those defined by the comparrison). We simply bring to many opinions/expectations and not enough benchmarks to the table. IMHO being completely objective and performing a quantitative analysis is almost impossible, when it involves human interaction. Especially, when interrupting the results. IMO few if any can be completely unbiased.
