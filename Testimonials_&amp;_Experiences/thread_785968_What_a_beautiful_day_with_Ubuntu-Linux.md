### Post by floreal on 2008-05-07
Hi, from Turkiye, to great Ubuntu community; 

I' m a person of your community for 2-3 months. And i' m proud of it,  
not only to be a member of Ubuntu users, a member of open source world. 
I have been in computer world since 2002. It' s not too much but it' s enough to understand, Microsoft and the other big brothers suck.  

I' m a eee student but i' m more interested in software programming. It' s easier to programming on Linux than windows. And on Ubuntu, it' s really an amazing job on this OS... so it makes you write the code (with SciTE) and debug it (with gdb) and release it. It' s really simple. All you need is at a step forward, on Package Manager. No more unknown compiling processes on Borland, Visual Studio or others. No more "i want you GNU, not in my name (so my name is windows)" like cygwin compiling errors (sometimes i really cant managed to compile) . No more minimalist GNU because i want all of the GNU. "Not to have it, to share it." This is one of the reasons for passing Ubuntu. 

It' s superb for a programmer or trying to be a programmer. 

My computer (Laptop- Acer Aspire 5570Z) said to Ubuntu: "You are the only OS that understands me ever", and they married. (I' m a Turk, you know them!) It' s true, it understands all the transistors of my computer. (Except wireless, i' m using madwifi) No more driver DVDs. I used to be a newbie while installing drivers on windows. After a fresh install of windows, on driver installing it blows away (it blows, data on hard disk blows, my registry data blows) like spring leafs and toasted under a fat women' s shoes' heel. Then you need a new fresh (what fresh, if you install an OS third times on a day, it losts its freshness) install. It' s better to be a newbie on Linux than always being a newbie on windows. After using Ubuntu i realize that, if i' m given a chance i can be a computer guy. Not a registry guy... 

And programs i use on Ubuntu. (top 5) 1. Amarok (i really really like it) for music, 2. SciTE (it' s very fast) text editor for programming, 3. KTechLab (ok, it has some bugs) for pic and electronics simulating, 4. Nicotine-plus for music sharing, 5. X-Moto (it' s like a plug'n-play hardware) for having some entertaining time. 

/* Some of the programs i used on Linux also have windows versions. I decided that if open source programmers has not build versions for windows, people would pass to Linux for these programs. Who wants to be a "pirate". (In my country they call cracked program users as pirates, not Pirates of Caribbean, pirates of faithless lands) Who wants to be disloyal. Disloyal to corporations... Don't say i want, it' s really a difficult situation to be outlaw. (Sometimes i see a commercial, it advises people to inform against unlicensed software users, this made me afraid of Uncle Sam and his brother Bill and people believed them) People like to use free programs. Most of them are open source. Why programmers don' t make these only for open source OSs?  */

And conclusion; 
I tried it and liked it. My opinion about programming and computer systems has changed. Before Linux i thought linux is only for super computer users, after Linux i saw it' s not difficult to use. Before Linux i thought i cant manage anything about programming untill i become a employee of a corp., after linux i realize that you don' t need to be a programming genius and you can be a part (small or big part) of this development world by your small scripts. Before Linux i thought i have to downgrade my XP-SP2 to XP-withoutSP2, after Linux i saw that i must upgrade to Hardy (i did). 

Last word; 
Use windows for entertainment, 
Use Linux for more entertainment.
Use windows for its "usability", 
Use Linux for its all ability.
