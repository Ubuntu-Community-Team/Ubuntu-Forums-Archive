### Post by DMK62 on 2008-05-01
ditto on what kingtaurus had to say.

The Gnome version of Ubuntu is where I would recommend someone start with Ubuntu. Almost everything needed to get going with Ubuntu is included with the Gnome version. Although the Kubuntu forums are a great resource you will find the ubuntu forums more active and almost every problem you may encounter has been asked here.

### Post by perspectoff on 2008-05-01
I tried the experiment of transitioning from Windows to Ubuntu with my 8 year old daughter, but using Kubuntu (the KDE desktop) instead of Ubuntu (the Gnome desktop).

She has been using Windows XP for about 2 years, mostly for games, school projects, looking up stuff on the Internet, graphics and posters (for science fairs and stuff).

I set the menus in Kubuntu to look nearly identical to Windows.

She transitioned without batting an eyelash.

When I had tried the experiment with Ubuntu (Gnome), it was frustrating for both her and for me.

I have used Ubuntu for the past 2 years. When I first looked at Kubuntu I also initially thought it was too complex and daunting. But after finally trying it one day (which I was led to do after installing LinuxMCE -- the Media Center Edition of Linux which uses Kubuntu), I found out what a joy KDE actually is.

There are a lot of irritating things in KDE (Kubuntu), such as the fact that all their applications start with the letter K. How stupid. But it turns out you can turn off the application names in the menus, and then it doesn't look so stupid.

Now my menus look almost identical to my Windows menus. In fact, my desktop is quite similar (but way better on Kubuntu, actually).

The similarity was necessary to convince both my business employees and my family to transition to Ubuntu (that is, Kubuntu).

I could never do it with the Gnome desktop that is the default in Ubuntu. Although it appears "simple" initially, it isn't flexible enough and is often quite frustrating in the long run. There are a lot of things missing, such as an icon that tells you when the system is busy, that is missing. KDE provides those little things that end up being essential for my peace of mind when using a GUI.

There is some "experiment" floating around about some guy trying to get his girlfiriend to accomplish tasks with Ubuntu (the Gnome desktop) and she fails, for the most part. I very much understand his perspective. Gnome is clean, but not very powerful, is not easily customized, and is much of the time quite annoying.

Had he done the experiment with Kubuntu (KDE) the outcome would have been far different.

My very, very strong advice is to choose Kubuntu.

### Post by Zorael on 2008-05-02
I strongly advocate KDE. At the very least, I urge new linuxees to burn live cds of both Kubuntu and Ubuntu and trying both out before deciding. Ubuntu (and Gnome by extension) is what has been getting in the headlines, so most likely don't give the alternatives a second though. Except perhaps those on low-end systems who quickly realize they might be best off using Xubuntu(/Fluxbuntu etc) after seeing Ubuntu's performance.

I read that many think that Gnome feels more well-polished, which is curious, since I think the contrary; that KDE feels more well-polished, fleshed-out, mature. Gnome prides itself over its minimalism and small memory overhead, but benchmarks(tm) have shown that KDE uses even less memory, while still packing more features. Gnome feels redundant whatwith it being in the same ballpark as KDE is (system requirement-wise), but then, I'm biased. :>

I **don't** think you should try out KDE4 yet; it is still too early to be considered desktop-ready, and it might prove counterproductive, giving you a false negative image of KDE! rarr

It needs time to mature, measured in months, but I have high expectations. Seems very promising.

Brilliant- thanks vvvladut.  It's probably worth mentioning here that if you are planning to give KDE a try *don't* use ```
sudo apt-get install kubuntu-desktop
```

Instead, use ```
sudo aptitude install kubuntu-desktop
``` so that if you want to remove it later all you have to do is run ```
sudo aptitude remove kubuntu-desktop
```

Otherwise, you can follow the directions in the URL in the above post.
