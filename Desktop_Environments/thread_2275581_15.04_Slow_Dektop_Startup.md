### Post by dino99 on 2015-04-27
That first error "ACPI PCC probe failed" is fully harmless (recent new kernel feature that should be silenced when the hardware does not have the required chipset; so quite 99 % of the users on earth at time). This does not affect the usual ACPI system.

but you get a real error: 
> Dependency failed for Swap.
Job swap.target/start failed with result 'dependency'.

you are not alone meeting that issue; did you use ecryptfs ? What's the 'sudo blkid' output ?

related bugs:
[https://bugs.launchpad.net/ubuntu/+source/ecryptfs-utils/+bug/953875](https://bugs.launchpad.net/ubuntu/+source/ecryptfs-utils/+bug/953875) (should be okay now)
[https://bugs.launchpad.net/ubuntu/+source/ecryptfs-utils/+bug/1447282](https://bugs.launchpad.net/ubuntu/+source/ecryptfs-utils/+bug/1447282)

is your system using bios or uefi with/whitout gpt ?

sorry about the "clean/autoclean/autoremove/gtkorphan" its not a unique command indeed, but im lazy when im seeing experienced user (2006) 

do:
> sudo apt-get clean
sudo apt-get autoclean
sudo apt-get autoremove
sudo apt-get install gtkorphan
sudo gtkorphan

### Post by monkeybrain20122 on 2015-09-12
Go to link in post #12 above, here you see a list of kernels available. Choose the one you want. e.g V4.2 unstable. Click it and you see this [http://kernel.ubuntu.com/~kernel-ppa/mainline/v4.2-unstable/](http://kernel.ubuntu.com/~kernel-ppa/mainline/v4.2-unstable/)

Create a folder, say, in your home and give it a name, say ker

From the above link download three .debs into ker
1. linux-headers-generic (choose amd64 or i386 depending on whether your OS is 64 or 32 bit)
2. linux-headers-all
3. linux-image-generic (choose amd64 or i386 depending on 64 or 32 bit)

Now you have the three files in ker, open terminal and change directory into the folder where the three .debs are, in this case /home/username/ker

```
cd ker
```

then run

```
sudo dpkg -i *.deb
```

When done, reboot. You can choose your version of kernel. I picked 4.2 as an example but you may want something a bit older, like 4.1.6

If anything breaks. Reboot, hold shift key to show grub menu. then choose advanced options then you will see a list of kernels installed. Choose the previous one, boot into it and then remove the three files you just installed from synaptic (or sudo apt-get remove packagenames, but using synaptic is easier. If you don't have it install it with sudo apt-get install synaptic)
