### Post by zarqoon on 2007-12-29
I have some very weird problems. So let me describe from the beginning
I am using gutsy and i have a ati x800 graphics card.
I use the fglrx driver from the restricted drivers manager.
If i dont use xgl everything works fine, my second screen works fine in dual head mode with  2 seperate xservers. 
With xgl enabled the first screen works fine desktop effects and everything, but the second screen is black and displays the standard mouse cursor (the one looking like an X) if i move the mouse there. About this however i dont care. I just use it to watch movies on and i start them using this script
```

killall wxvlc
DISPLAY=:0.1 wxvlc --fullscreen "$*"

```

as a nautilus starter.
This opens a new vlc player on the second screen in fullscreen and plays the movie i klicked. This vlc has no window borders which is not surprising because there does not seem to be a window manager running on :0.1
Now the weirdness starts.
As soon as i do this all my keyboard entries go the this vlc player. Clicking another window or starting other programs with the move does not change anything. Even closing the vlc player makes no difference.
So i did a little experimenting. I now have this script on my desktop
```

DISPLAY=:0.0 wxvlc

```
this opens a vlc player on my first screen without window borders which i have to manually close using the mouse. Then i get control of my keyboard back.
If i change the second script to
```

DISPLAY=:1.0 wxvlc

```
i get window borders but i dont get the keyboard back.
After noticing that my compiz seems to be running on :1.0 not :0.0 i naturally tried using :1.1 for the second screen but that does not seem to exist.

So to summarize my problem. I want my keyboard to work on the first screen while using the second to watch movies. The script clicking i do now is annoying me. Getting compiz to work on the second screen might be the solution but i could get it to work and i dont really care if it does (dont need it for movies).
