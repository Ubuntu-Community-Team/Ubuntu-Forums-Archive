### Post by Copper Bezel on 2011-02-02
Not a support request, but a "problem" I "solved" others might be interested in, since I haven't seen it around.

I just set Thunar to the default application to open folders in my Nautilus preferences. Nautilus preferences. Like, the preferences within the default file manager I was displacing.

It worked.

The practical upshot for me is that any folder I open with more than 100 or so items takes several seconds to render in Nautilus but does so instantly in Thunar. Anything that opens a location, like my Places menu in my AWN dock, now opens in Thunar. Obviously, it has no effect on opening folders on the desktop, since they're already *in* a Nautilus window, but that just gives me options.

Along with my system figuring out - I don't honestly know *how* - that I was using xfrun4 as my Run Dialog, so that that's what launches when I select Launch in the Gnome Menu in AWN, and the fact that Beagle replaced the default search tool there, I'm once again just amazed at how well Linux apps learn to play nicely with one another.

I love Linux. = )
