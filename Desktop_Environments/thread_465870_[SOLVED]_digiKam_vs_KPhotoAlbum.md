### Post by steevc on 2007-06-20
As far as I'm aware they are different applications by different authors, but they have similar aims. Kipi is a standard feature of KDE, so it's natural for them to use it.

I was playing with KPhotoAlbum last night. I'm trying to find the optimal application for tagging my 5000 photos. This one looks pretty good. It's quick to tag one or more images and to add new tags. The timeline is nice.

There are comments in other threads that Digikam can store the tags in the image files themselves, so that you can use them from other applications and they stay with the images if you move them. I've not really tested the latest Digikam, so I need to have another look.

I'm after something that lets me quickly tag images as I add them and provides good search tools so I can do things like find a picture of my son on a beach.

Update: I've been playing with Digikam. It's different. Adding tags is slightly more work, but it has the advantage that you can add people who are already in your address book. Tagging and searching are not as smooth as KPhotoAlbum. I also could not work out how to make Digikam write it's tags to JPEG EXIF data. I couldn't see much documentation on that.

So for now it's a draw. Neither quite does all I would like.

### Post by siddartha on 2007-10-16
Hmm...

Yes, they are different. After using both in parralel my oppinion is that KPA is a nice alternative to Picasa if you don't mind not being able to upload to Picasa web as it is faster and native. It has the advantage of not changing anything in your files. Also the database is usefull but that is ok as long as you can dedicate a whole hard drive to the whole photo collection. As I work with larger files (3-4M in size each) and the collection is 5000+ I discovered that it it taking about three quarters of the laptop's hard drive. Backuped that on DVDs, but KPA doesn't know how to handle more than one 'photo root'. So the use is quite limited. It could have been useful to have a function like copy the database in the files' IPCT for example so it would be portable at least for other apps. Sadly it doesn't. So it's just a waste of hard drive space for me. Dumped it.

digikam has the same problem with the different photo root - it can't do it. But it comes with a nice editor and tools so I'd keep it.
