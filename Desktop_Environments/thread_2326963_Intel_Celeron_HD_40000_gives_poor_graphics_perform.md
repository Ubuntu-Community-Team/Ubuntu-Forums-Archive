### Post by grahammechanical on 2016-06-06
This device?

[http://www.intel.co.uk/content/www/uk/en/nuc/nuc-kit-nuc5cpyh.html](http://www.intel.co.uk/content/www/uk/en/nuc/nuc-kit-nuc5cpyh.html)

"running at > 50%" But of what? RAM? How much RAM?

If it is 1GB RAM than 50% is better than I am getting with 1GB of RAM. If it is 2GB then that sounds about right. But if it is 4 or 8 GB then you may have a point. An OS will use as much memory as it needs and then it will take some more memory and set it aside as cache memory just to be on the safe side. That can make it seem that the OS is using a lot of RAM. 

 As for which 16.04 flavours use the least RAM see this,

[https://bryanquigley.com/memory-usage/ubuntu-16-04-livecd-memory-usage-compared](https://bryanquigley.com/memory-usage/ubuntu-16-04-livecd-memory-usage-compared)

We can compare the results of the same tests run using 14.04. Things have improved for Ubuntu in 16.04. 

[https://bryanquigley.com/uncategorized/ubuntu-14-04-livecd-memory-usage-compared](https://bryanquigley.com/uncategorized/ubuntu-14-04-livecd-memory-usage-compared)

The fact is, if we want fancy effects then we need a window compositor/manager. And window compositors like Compiz use more memory than other window managers that do not do window compositing.
