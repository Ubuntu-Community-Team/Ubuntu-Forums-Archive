Hi,

I was originally going to post a thread asking for help, because my Gaim kept quitting abruptly after completing log-in procedures to the MSN servers. However, I managed to solve this myself, and I hope that this post helps others who had the same problem.

The problem first appeared after I added a certain contact to my MSN buddy list: the "Play Q6" assistant (chat bot)^1. I did this in Windows Live Messenger under Windows XP. Immediately afterward, shutting down, booting into Ubuntu 6.06 and launching Gaim produced the problem.

As far as I can tell, the solution was boot back into WinXP, open Windows Live Messenger, and remove that contact (to be safe, I also blocked it and removed it from my Hotmail contacts, though this may not be necessary). After doing this, I returned to Ubuntu and Gaim was back in working order.

I am not sure that this solution will help that many people, since I do not know how many people use a dual-boot WinXP/Ubuntu system, let alone Windows Live Messenger _and_ Gaim, but felt I should post it anyway.

^1 Can it be safely assumed that other chat bots (with enhanced capabilities which only work with the native Windows Live Messenger) may also cause this problem if they are on a buddy list loaded in Gaim? I don't know of any other chat bots to test this theory with. Anyway, it may be something to consider.
