### Post by laodan on 2006-02-04
I found the solution to my problem and here it is. It is DHCP related (networks)

I went in the following folders. (they are read-only so I entered them as root-user. I prefer to do changes manually than through a terminal, it gives you a better visualization of what you are doing...)

[COLOR="RoyalBlue"]etc/dhcp3/dhclient-enter-hooks.d
etc/dhcp3/dhclient-exit-hooks.d[/COLOR]

Each of those folders has a debug file that in the default installation is set so as not to run the "debug-script"...  as in the following code:

[COLOR="RoyalBlue"]# To enable this script set the following variable to "yes"
RUN="NO"

if [ "$RUN" = "yes" ]; then
	echo `date`: entering dhclient-enter-hooks.d, dumping variables. \
		>> /tmp/dhclient-script.debug

	for i in reason interface medium alias_ip_address new_ip_address \
		new_subnet_mask new_domain_name new_domain_name_servers \
		new_routers new_static_routes old_ip_address old_subnet_mask \
		old_domain_name old_domain_name_servers old_routers \
		old_static_routes; do

		echo $i=\'${!i}\' >> /tmp/dhclient-script.debug
	done

	echo '--------------------------' >> /tmp/dhclient-script.debug
fi[/COLOR]

[COLOR="Red"]I changed the NO into YES in the debug files in the 2 folders mentionned hereabove and Oh! miracle my access to the net was restored... after restarting the computer.[/COLOR]

I'm wondering why these debug files are there and why are they by default inactive? Jeez, it took me an eternity to find them and then it's still only by accident that I found them and changed the option so that the debugging could work...
