### Post by AngryPanda on 2006-08-26
I've run into the same problem. The tracks will encode if there isn't a pre-existing U2 directory (i.e., this is the first U2 disc I'm ripping), but if there is already a U2 directory, no go.

The workaround I've been using is to rename the U2 directory (I used "youtwo", for instance) before ripping the CD. I'm sure you can guess what happens next: move the songs from the newly-created "U2" directory into the original-but-renamed directory, delete the now-empty "U2" directory, and repeat, ripping the next CD featuring everybody's favorite Irish rock band. When done, rename "youtwo" back to "U2" and you're in business.

This is admittedly a pretty ugly hack, but it gets U2 on the hard drive, anyway. I was searching the forums for a better fix--*Rattle and Hum* is ripping away in the background--when I came across your post and thought I'd post my band-aid. ("Pretty ugly"--that's some quality word selection there!)

I have an embarrassing number of U2 CDs, so I would certainly appreciate a more elegant solution than this, if anyone has one.

Incidentally, the same thing also happens with REM.
