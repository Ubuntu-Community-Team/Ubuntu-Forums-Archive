### Post by stoanhart on 2006-06-07
Hey,

I am trying to install XGL/Compiz on my laptop. I have used the following method.

1) Install the CVS version using the CVS Howto in "One Thread to Rule them All".
2) Instead of adding my "Xgl :0 -fullscreen ...." line to /etc/gdm/gdm.conf-custom, I have used a script called startxgl.sh which is called by /usr/share/xsessions/xgl-gnome.desktop whenever I have the xgl session selected.
3) Once I am on my desktop on the Xgl x server, if I want compiz stuff (when I am on AC power), I type 'xgl', which runs a toggle script that starts compiz.

This is exactly how I have set up my desktop, which works just fine. Both my laptop (Asus z71v) and my desktop use nVidia cards, all my drivers are installed, and my 3d accerelation works. 

The problem I have is this. When I try to start compiz with my 'xgl' command (not to be confuse with Xgl, which is the Xgl X server), I get:

```

```  

After a a trip to google, I added 
```

  Section "Extensions"
         Option "Composite" "Enable"
  EndSection

```

```

compiz.real: GLX_EXT_texture_from_pixmap is missing
compiz.real: Failed to manage screen: 0
compiz.real: No managable screens found on display :0.0

```

and if I try to run 'gnome-window-decorator', x restarts.

I have checked forums and the web, and usually when this happens, someone isn't Xgl, but rather just a plain X server, or they don't have the mesa libraries installed. I am most definately running Xgl, and I have mesa.

One thing I notices is this. If I open a new XGL server on top of my current one, and start gnome-window-decorator and compiz on that, it works, and it runs full speed (so I guess I could always do that). Also, if I change my start-xgl script to run my Xgl server on display 1 (instead of 0), it works, but is VEEEEEEEEERRRRRRRRRRRRRYYYYYYYYYYYYYYYY SSSSSSSSLLLLLLLLLLLLOOOOOOOOOOOOOWWWWWWWWWWWWWW.

Any help would be appreciated! Thanks!

startxgl.sh
```

```

[Desktop Entry]

### Post by matrooswolf on 2006-06-07
Hi I do not have the answer to your problems, but I do know that if you use the latest compiz (which runs very well with me) form Quinnstorm (see: [http://xgl.compiz.info](http://xgl.compiz.info)), you do not have to load all the different plugins, just ```
gnome-window-decorator &  compiz --replace gconf 
```should do the trick.  Gconf loads the plugins automatically.

Try also the compiz forums for more info:
[http://www.compiz.net/](http://www.compiz.net/)

### Post by slinky on 2006-06-08
Hi have come accross a work around hope this is some use to you?

first off i did a dist upgrade.
then i took out the section in xorg.conf  
#Section "Extensions"
#	Option "Composite" "Enable"
#EndSection

then i created a file like so:
sudo nano ~/.Xsession 
 #!/bin/sh
# Start up Xgl, compiz, and GNOME
# Run Xgl server on :1, on top of normal X
Xgl :1 -fullscreen -ac -accel xv:fdo -accel glx:pbuffer &
# Tell subsequent X programs to access the Xgl server at :1
DISPLAY=:1
# Start Compiz window manager
gnome-window-decorator &
compiz gconf decoration wobbly fade minimize cube rotate zoom scale move resize place menu switcher &
# Start GNOME 
exec gnome-session

i then did:   chmod +x ~/.Xsession

then when you log in choose default session it worked for me although Gnome and your desktop applications will be running on the Xgl display :1 instead of the usual :0

seems to be working fine :)
