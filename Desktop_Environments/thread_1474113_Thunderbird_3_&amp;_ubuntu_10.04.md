After the upgrade to 10.04, Thunderbird was updated to 3.0
Apart from losing my calendar (Lightning) the spam filtering has stopped working.
Well it sort of works, sometimes.
I am using Thunderbird to pick up email from several gmail (with imap) and pop accounts and it does that well.
Gmail does its own spam filtering and puts the spam it finds into a spam folder.
With v2 of Tbird I had it configured to grab the spam from the gmail accounts, mark it as read and drop them all into one local junk folder. Every now and then I check this folder to see if there are any genuine emails among them and clear the folder out.

However with the upgrade to v3 of Tbird nothing much happens. If I click on one of the gmail spam folders and then select Tools | RunJunk Mail Controls on Folders then messages in the folder get marked as junk. In v2 that would happen automatically. But also in v2 the messages would get moved to my local junk folder as set up in
Edit | Preferences | Security | Junk 
it would move them to the junk folder as defined in
Edit | Account Settings | Junk Settings | move to Junk on local folders
