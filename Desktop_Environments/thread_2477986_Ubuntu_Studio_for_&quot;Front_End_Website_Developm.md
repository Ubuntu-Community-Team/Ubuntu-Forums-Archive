### Post by TheFu on 2022-08-15
"Desktop" isn't a concise term.  
DE - Desktop Environment
WM - Window Manager
are the proper terms.  Historically, the WM has been used for 30+ yrs in the X/Windows system. This provides a root menu and window decorations. Most DEs run on top of a WM. They are different layers.  
Some DEs have become tightly coupled to the MW they use/ship with at the expense of flexibility. WMs have certain standards they follow to ensure inter-window communications and DE compatibility.  Of course, if a DE chooses to ignore those things, we cannot do anything about it, except agree to be trapped by by that DE into using only the WM they support, losing flexibility.


WMs and DEs are just programs. They are not the OS.  We are free to change those on a per-username basis.  There shouldn't be any issues in having 20 different WMs and DEs installed onto a single workstation and many can be used by the same userid, switching just by logging out, picking the next, and logging back in.  Alas, some related DEs that use the same underlying toolkits - GTK+ (Gnome) or Qt (KDE or LXQt) can run into conflicts since these toolkits will read and write settings to the same config files/DBs and those settings may have different meanings between different DEs.  Does that make sense?  Anyway, the way to avoid that is to use 1 useraccount per DE.  That's not an issue, since all Unix systems are multi-user from the beginning and have always been multi-user going back to the first pdp8 Unix OS.

However, in 22.04 and later, Canonical has changed some standard Unix settings (not GUI stuff), which wouldn't usually matter for the typical single-user way that Ubuntu systems tend to be used these days, but which make working together between different userids not quite as easy for someone unfamiliar with normal Unix file/directory permissions.  I can only say that you should learn those ASAP, spend the 45 minutes of concentrated learning that is required, which requires about 30 minutes of hands-on testing and fooling around with multiple userids, multiple groups, in multiple directories AFTER working through a 15 min tutorial to gain the basic understanding.  Do that to avoid hours, days, weeks, months, of frustration ASAP.  There's no way around it and basically, every popular OS in the world today uses the normal Unix permissions model as their first line of security, except 1 (which happens to be the OS you are probably most familiar about).

Suggestion #2.  80% of the power of a Unix system comes from non-GUI programs.  Grandma can use Linux systems fine just by pointing-n-clicking, but that's because she has minimal needs.  For everyone else, don't fear the shell and learning how to create 1-10 line scripts will really help your productivity.

I've never used Ubuntu Studio, but suspect it is like any other flavor - it has specific default applications pre-installed and setup to work within the environment.  That's what every flavor does, but any GUI program is just another program. They can be loaded onto any other flavor, though many tightly integrated workflows that cross between different programs will need to be manually integrated.  I understand that "Studio" uses the Jack audio subsystem.  All other Ubuntu Flavors use PulseAudio.  That means that you'll need to be aware of this if there are any audio issues.  Some distros have already dropped Pulse for a "new" audio subsystem, not Jack.  The early adopters seem to like it.  All that I know is that pulse has been full of issues all this time, but tends to work for most people and doesn't usually require any special setup for most people.  Before Pulse, we had to configure ALSA or OSS audio on our Linux systems.  They never "just worked", without some manual config, in text files, at least not for me.  I think ALSA is still under Pulse, but haven't had to touch any ALSA configs the last 7+ yrs (I don't really recall).  I do know that Pulse does crash a few times each month around here.  Pulse was released initially in 2004 and still isn't stable.  The guy behind Pulse is also the guy who created SystemD .... and fortunately, recently changed employers to MSFT, so now they get to be inflicted by all the "change for the sake of change" code he's been inflicting on the Linux world.  

"Studio" isn't about web development. It is about audio/video stuff.  On the main page for Studio, they list "publishing", but the main applications there aren't anything special.  LibreOffice is probably the default office suite for most Ubuntu flavors that aren't trying to run on 10 yr old hardware.  Calibre is an ebook library manager and web-server. I use it.  Scribus - I've never used.  I used to work in publishing and document management. Seems to be a page layout editor.  Meh.  IMHO, only advertising people should care about page layout.  Everyone else should be using HTML to share information that can automatically flow as needed for the viewing hardware.  Sorry - too much rant.

Always remember the Linux isn't Windows.  Things that might appear similar are probably not.  Unix has a different core set of philosophies than MS-Windows.  The way to solve many problems is by putting lots of little tools that do 1 thing really well together into the solution.  [https://blog.jdpfu.com/2012/02/15/beginning-linux-thought-shift-needed](https://blog.jdpfu.com/2012/02/15/beginning-linux-thought-shift-needed) tries to explain how a problem may be solved thinking the Unix way or thinking the MS-Windows way. Anyway, hope this is helpful AND that others will chime in. We all have different experiences, so different views on things.  I've installed, deployed and created hundreds of webapps over my career, so some things are warped by that background.

### Post by TheFu on 2022-08-15
For all my website development, I use vim for html, css, javascript, ruby, perl, php, python development.  Also use vim for C/C++ and anywhere a text file exists.

I subscribe to the "Unix is my IDE" method, because that's what the gurus I learned from did.  You can google that, but I don't consider HTML/CSS languages. They are markup tools.

Be careful with Wordpress.  Others here know much more about it than I do.  Since 2008, the company were I work has forbidden any php webapps be allowed directly on the internet because it takes really good php developers to make secure code.  I have a few internal php webapps, but access requires using a full VPN. That's mandated by the security team.  

SeijiSensei has posted a number of Wordpress techniques to these forums to limit security failures.  On any day, over 1M wordpress sites are hacked.  Beware.  That isn't to say that all other webapps are security, just that wordpress is a common target.  Beware of themes and addons which seem to be where many of the security failures occur besides misconfiguration.

IMHO, you should only run LTS releases unless you will commit to replacing your OS every 6 months.  Few people will do that.  New Ubuntu releases are released every 6 months and support for non-LTS versions end shortly after. For example, support for 21.10 (Oct 2021) release ended last month.  OTOH, LTS releases for servers have 5 yrs of no-hassle support and with some cash, you can get longer periods. Generally, I think 5 yrs is more than enough and waiting much over 4.5 yrs to migrate to a new OS just makes the job nearly impossible due to all the changes and incompatible configurations.  Most people don't learn this lesson quickly enough and end up with an unsupported server system that won't boot and nobody can help ... because it isn't still supported.   As much as I loved 10.04-16.04 servers, I can't recall all the little details that have changed over those times anymore.  OTOH, I'm still running a few 18.04 servers and planning to migrate them to 20.04 shortly - definitely before Xmas.

Please learn from what the folks here have posted. I think we all have over 20 yrs experience doing these things.

People always want to simplify LTS to be "Supported for 5 yrs".  This isn't the real situation for most people.  It applies to Ubuntu Server (the distro without any GUI) and the default DE shipped with Ubuntu Desktop.
