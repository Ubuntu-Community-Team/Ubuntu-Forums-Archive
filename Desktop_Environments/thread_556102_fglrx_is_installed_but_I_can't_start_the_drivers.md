You're trying to use fglrx, right? Your xorg.conf tries to use opensource (radeon) driver though.
Here's my device section from xorg.conf, if it helps any
```

Section "Device"
        Identifier      "ATI RADEON 9550"
        Driver          "fglrx"
        BusID           "PCI:1:0:0"
        Option          "VideoOverlay"  "on"
        Option          "OpenGLOverlay" "off"
#       Option          "UseInternalAGPGART" "no"
#       Option          "XAANoOffscreenPixmaps"
        Option          "backingstore"  "true"
EndSection

```
You should keep your "Identifier" and "BusID" values at least.

fglrx module has problems to load if you already have radeon module in your kernel. Shutdown your display manager (gdm?) and remove the radeon and/or drm module from the running kernel. then try to modprobe fglrx module. From a tty1 (ctrl-alt-F1), type
```

sudo /etc/init.d/gdm stop
sudo modprobe -r drm
sudo modprobe fglrx
sudo /etc/init.d/gdm start

```

You should backup your current xorg.conf before doing anything though. Doesn't "Restricted Drivers Manager" know how to handle fglrx loading btw?
