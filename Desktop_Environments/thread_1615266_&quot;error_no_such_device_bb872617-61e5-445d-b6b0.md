### Post by drs305 on 2010-11-06
Edit: Post revised following posting of RESULTS.tx in Post #5.

Your Ubuntu install is on sdb5, you could try typing the following at the grub prompt:

```

ls (hd1,5)/boot/grub   # Note: if you don't see a lot of files after you press ENTER, the rest of the commands won't work. You can try (hd0.6).
set prefix=(hd1,5)/boot/grub
set root=(hd1,5)
insmod linux # You can also try "insmod (hd1,5)/boot/grub/linux.mod  Continue even if doesn't work.
insmod ext2  # You can also try "insmod (hd1,5)/boot/grub/ext2.mod  Continue even if doesn't work.
linux /vmlinuz root=/dev/sdb5 ro
initrd /initrd.img
boot
```

If that doesn't boot, boot a LiveCD and run the boot info script from the following site. Post the contents of the RESULTS.txt file which is generated by the script.
[http://bootinfoscript.sourceforge.net]("http://bootinfoscript.sourceforge.net")

### Post by drs305 on 2010-11-06
> **MdNtSnow said:**
> 
I have rebooted it back into ubuntu. Perhaps I should reinstall grub?

If you were able to boot via the grub prompt then you can try reinstalling grub and regenerating the grub.cfg file. If this doesn't work, the next step would be to purge/reinstall grub. 

Let's try the less drastic step first. From the grub prompt boot:
```
sudo grub-install --recheck /dev/sdb
sudo update-grub

```

Make sure your computer's BIOS points to sdb as the first drive to boot.
