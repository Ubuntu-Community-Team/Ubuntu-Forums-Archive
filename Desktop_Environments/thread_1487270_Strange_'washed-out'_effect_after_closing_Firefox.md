The screen does not return to normal until it is redrawn by one method or another. As you can see in the second picture, the screenshot app and its save dialog have redrawn some of the screen as has conky and a couple of other tidbits. The way I have been redrawing the whole screen is to quickly click and release the left mouse button with the cursor at the left edge of the screen which is the configured action to initiate Compiz Fusion's desktop cube effect (cylinder in my case). Doing it quickly doesn't actually start unfolding the cube/cylinder but does cause a redraw of the whole screen. Still it's a really sucky 'workaround' and a bit embarrassing when showing of my wicked cool Ubuntu desktop to prospective new Ubuntu users.

I do know what the source of the problem is. I am using the now-defunct Greenscreen plugin for Compiz Fusion to get true transparency in conky (can also be used for the panels as well as practically anything that allows the background color to be explicitly set. It's known to be problematic but I'm not aware of any good alternative for getting true transparency with my current desktop environment (gnome/compiz/metacity).
If I disable the plugin the problem goes away. I'm now looking for a better workaround. If you have any ideas or need more info on my current setup, I'd love to hear from you.
