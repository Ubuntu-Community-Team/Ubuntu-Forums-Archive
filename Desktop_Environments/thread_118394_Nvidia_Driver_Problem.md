### Post by fourchannel on 2006-01-16
k geforce 2's need the legacy driver. In synaptic, search for 'nvidia' and install the legacy stuff.

if you have a 386 kernel, install the 386-legacy restricted module, naturally, a 686 kernel would be 686-legacy. you can find this out by typing in a console: ```
uname -r
``` 
and also don't forget to get the nvidia-glx-legacy driver. go ahead and restart for the hell of it.

when you get back to the desktop do..

```
ctrl-alt-F1
``` to get to a virtual console. login and then run...```
sudo su       *to give you root access*
/etc/init.d/gdm stop     *to shutdown X*
nvidia-glx-config enable   *enable your driver*
(it is *nvidia-glx-config* even if it is the legacy driver.)
dpkg-reconfigure xserver-xorg  *to setup X again, for your driver select 'nvidia' not 'nv' and also on load disable dri and GLcore* IF *you have problems the first time round.*
/etc/init.d/gdm start    *fire it back up.*
``` 
and you should be good to go. 

if not, then paste your /etc/X11/xorg.conf and your /var/log/Xorg.0.log here so we can see what the hell went wrong.

EDIT: just remebered.. copy your /etc/X11/xorg.conf file before messing with anything. you can fix X by simply replacing the file with the backup and hitting ctrl-alt-backspace in X. ```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf~
``` and also you might need to add the extra repositories before you get synaptic to show you the nvidia stuff... [http://www.ubuntuguide.org/#extrarepositories]("http://www.ubuntuguide.org/#extrarepositories")
