### Post by Luke771 on 2006-06-19
I like the Xpenguins applet, sometimes I used to let a couple of penguins and maybe one Bill or two run around my desktop, that was fun. And it worked perfectly on Breezy.
On Dapper it deleted the last panel leaving me without an interface.
I could have tried to find a solution but as I said in another thread, I have my data on a dedicated data storage partition together with a backup of some configuration folders (those with the dot before the name), that way I can reformat, reinstall, not lose any data and restore my settings in a matter of minutes.
So that's what I did.
Reinstalled, restored settings, installed a bunch of stuff including xpenguins, tried to add the xpenguins applet to my panel... and got the same problem all over again:

What happened both times is this:
I searched xpenguins in Synaptic, installed it and all went fine.
Then I wanted to add the Xpenguins applet to the panel, so I clicked the panel and chose 'add', and then I selected 'Xpenguins'

And then it froze.

I couldn't click anything. After a while I even got a message like *'there is another panel running, so I'm shutting down' * or something like that. I didn't write it down, sorry.
Then the panel disappeared.  I only run one panel at the bottom of the screen with  buttons, window bar, sytem tray etc (I don't like the upper panel) so when my only panel disappeared I couldn't do anything but reinstall everything from scratch (no xpenguins for me this time). I'm pretty sure that there must be another solution (for ex run with two panels) but reinstalling was the faster one for me (data and settings already backed up).

I'm posting this thread to warn people about xpenguins: be sure you can 'afford' reinstalling Linux (in the sense of the data loss) before trying to run the xpenguins applet.

My sistem is an old AMD Duron 1600MhZ with 512MB RAM, Video nV GeForce4/64, WinXP ro on hda1 Dapper on hdb1
