### Post by davarino on 2008-01-09
Your hardware configuration is perfectly fine for a regular Ubuntu installation, or a Kubuntu installation. I would recommend that you switch over to Ubuntu for several reasons:

[LIST=1]
[*]Your "Help" information is directing you to the (Ubuntu) Gnome menu. It's going to be hard to learn what to do if you can't easily find the help that is offered.
[*]There is much more discussion of Ubuntu-specific problems than there is of Xubuntu-specific problems... thus it's probable that you'll get your system up and doing what you want it to much more quickly.
[*]You can switch back to Xubuntu in the future, easily. After you've gotten accustomed to the way we do things and communicate.
[/LIST]

Give yourself a break and take the easy way on this. There is a lot to learn... but it will probably be worth it to you. But in any case, at least set yourself up for easy successes at the beginning. Xubuntu is just a little too raw for some Linux beginners - the documentation is a bit too terse and the interface seems a little sticky to many folks - Ubuntu may be easier for you to start with 

You can easily install Ubuntu by loggin in to the account that you first established when loading up Xubuntu. Then, open your terminal program and type in

gksu /usr/sbin/synaptic

You will be prompted for a password: use the password that you gave for this account that you had established and are logged in as. (This is the "sudo" hat trick. You use the user password to gain temporary administrative privileges. It works with sudo and gksu.)

Select the package "ubuntu-desktop" and mark the check-box (tick-box) for installation with a right-click. Then click the menu item "Apply". It will take some time to download the files and update the system, but after probably 15-30 minutes you will have an Ubuntu system... which will be much easier to learn.

You can easily revert to Xubuntu - if you want to.

### Post by airtonix on 2008-01-09
your missing hard drive space can probably be attributted to the fact that windows was left on your laptop?

do you gt a choice on boot wether you want to boot into ubuntu or windows?

if so then thats where your hard drive space is at, being used by windows.

Also a partition is used for "virtual memory", unlike windows which just plonks a file on your C: drive, which  creates a lot of fragementation. 
On linux the standard file-system's natural design mitigates fragmentation by starting writes at center of file table.

It also uses a swap partition instead of a swap file for virtual memory...so any possible fragmentation is contained to that partition(which is re-imaged each boot).

If you want to share files with local linux machines on your network....use NFS or SSH.
If its for other windows machines, investigate the usage of : 

* smbfs
* fusermount

Then you can "Map a shared Drive' in windows talk. this is done by : 

* creating folder under "/media" (ie "/media/deepBlue-moviesArchive")
* making it owned by a certain group that is for share users. (ie: sambaUsers)
* mounting the remote machines share on this folder
(ie:  sudo mount \\192.168.0.100\moviesArchive)


Of course it would be much easier for you to install ubuntu-desktop and use the native network browsing features in nautilus.

locations can be ssh://, smb:// ftp:// network://, computer:// (might be more that im not aware of)

ps. you will get higher transfer speeds with NFS than windows-networking(aka samba)


File Searching & Indexing.

Strigi, Pinot meta-search, Beagle, Tracker.

Of all of them, pinot seems the lightest on resources. the other two slow my machine down to an in tolerable crawl.
Tracker gives the most useful results.

or you could use Strigi, and its web-interface (which runs off your 127.0.0.1 ip) it is the lightest and fastest. There are guis for gnome and kde.

Lastly, have a look at Opera, supposed to run on windows 95. and is currently the fastest web browser on the planet. it loads faster for me compared to firefox, epiphany and konqueror.
