### Post by Bleppe on 2006-05-04
i've tried both amule and xmule (and yes ml-donkey too) so far, they seem to work fine, but both seem to be lacking the nice search function of the emule client 

you don't have to know the name of the file, you can just specify a file format, an approximate size and you get something relevant, let's say that you're looking for gamecube isos, their file-format is gcm, and their maximum size is 1,35 gb. you could just use that as a search criteria and only get gamecube isos back as an awnser... and this principle you could apply to just about anything.

it's currently only avaliable for winblows. i can run it through wine, but it has some stability issues concerning the gui - leave it alone and it's as solid as a rock, mess about with it and it all comes crumbling down... 

(edit) apparantly i didn't look hard enough. amule does indeed have something similar, if you click on extended parameters it all appears before you... it does seem rather snappy but i have yet to succed in connecting to multiple servers. the current cvs version is supposed to support cad as well, i'm going to have to learn a bit more before i can use that though...

i halve also tried bittorrent and bittornado, oddly enough both have a problem interpeting torrent files that are tagged with login/pass information. i can download from mininova and tpb just fine, however if i login to tvtorrents.com for example all i get is a red warning saying "Bad data from tracker - bad bencoded data" i believe that this has something to do with "UTF-8"  maybe this issue is in fact browser related?
