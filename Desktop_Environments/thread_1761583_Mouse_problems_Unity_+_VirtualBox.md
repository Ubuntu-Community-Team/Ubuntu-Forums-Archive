### Post by rtobyr on 2011-05-18
I know that there are documented problems with Unity with the "wobble" effect. I am not using the wobble effect. I'm running Ubuntu 11.04 as a host with Windows 7 inside VirtualBox. My preference is to have the virtual machine in a maximized window. I don't like full screen because I like to have access to both environments at the same time. I don't like seamless mode because a Windows app called Object Dock causes problems with that.

There are times where I cannot click on certain locations in Windows. I think that it's trying to click in Ubuntu. If I switch to full screen mode, then I can click. Another workaround is to move the Windows window in which I'm trying to click to another position on the screen. Similarly, if I have an Ubuntu window in front of the maximized Windows VM, there are times that I cannot click in the Ubuntu app unless I minimize the VM or move the location of the Ubuntu window.

My working hypothesis is that because of Compiz Unity, the computer doesn't know whether I'm trying to click in the Ubuntu host or the Windows VM.

Has anyone else experienced this? Does anyone know of a fix or at least a bug report that I can follow?
