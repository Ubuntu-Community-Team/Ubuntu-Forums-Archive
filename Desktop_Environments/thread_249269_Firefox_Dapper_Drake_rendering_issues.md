### Post by Noobmeister on 2006-09-02
Hi, I've been using Ubuntu (Dapper Drake) for about 2 weeks and have noticed that the bundled version of the Firefox browser renders an unhealthy percentage of websites incorrectly, compared to the same version on Windows. Most of the time, the problem is just cosmetic (columns out of whack, garbled text), but in some cases, which usually involve javascript, site functionality can actually be broken.

Sites I've noticed a problem with are [www.Gamespot.com](www.Gamespot.com), [www.CNN.com](www.CNN.com) and [Xbox360.ign.com](Xbox360.ign.com). I've also experienced a problem with sites that use the estara service, which uses javascript popups. You can check this functionality yourself by going to [https://www.amazon.com/gp/help/contact-us/call-me.html/103-9688541-0000611?](https://www.amazon.com/gp/help/contact-us/call-me.html/103-9688541-0000611?) and clicking on the 'Call me' button at the top left of the screen. This popup normally appears in a completely corrupted state when I attempt to view it.

Now, what have I done to diagnose the issue you ask? Well, I first logged this issue at the Mozilla.com site, and had one of the mods there try and reproduce my estara test case, which he said rendered fine on his Suse/Firefox 1.5.0.6 install. I've tried disabling all extensions and themes from Firefox to see if that resolved the issue. I even upgraded the version of Firefox to 1.5.0.6, to see if that resolved the issue. None of these things helped.

Finally, I downloaded the Opera 9.0 package and confirmed that all of these problematic websites **do** render correctly using that browser, which leaves me somewhat confused as to the nature of the problem.

Has anyone else seen these types of issues when using Firefox on Ubuntu 6.06? Any thoughts would be appreciated.
