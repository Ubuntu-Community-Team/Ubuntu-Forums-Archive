### Post by az on 2005-07-25
They are two different desktop environments.  One is based on commercial libraries called Qt, and the other is based on Gnu libraries called GTK.  

KDE built itself up faster since its libraries had more functionality from the beginning.  Gnome developers had to develop both the desktop envrironment as well as the libraries.

They both are roughly on par, today, in terms of functionality.  Gnome has a tendancy to push the useability envelope more than KDE.  KDE has more trinkets than Gnome.

If anyone can add to this, please do so without getting into a debate over which is better.  Many new users cannot grasp the idea of more than one desktop environment and they deserve an answer.

### Post by Terracotta on 2005-07-26
Since they are considered to be equally usefull, I'd rather recommend to choose the one, who's filosophy suites you best, rather than which one looks prettiest, since they are both customizable to act and look the way you want it. I chose KDE since I feel that the filosophy of using libraries that a company gpl-ed for the non-profit developers is much better since it has a license for companies who wants to make money out of the program they write. Qt is also available for a certain non-unix OS, which makes it easier/cheaper for programmers to write a program for all different OS's. So the base is larger, and the code will be better sustained as well. Gnome on the other hand, created their own set of libraries, in the spirit of the Free Software Foundation, who thinks software shouldn't be used to make money out of it. The only problem was that only open-source programmers for linux would create programs with the gtk-toolkit, so they gave the gtk-toolkit the lgpl-license. This gives a programmer the right to use a certain toolkit without open-sourcing it's program. 
This is what I think are the main filosophical way both think, if I'm wrong, correct me, I'm just trying to explain how I feel about both of them, no flame intended, of course I'm biased because I use only one of the two, but I explained the reason why, if two products are equally good, I choose the one that I agree with the most, which is in my case, KDE.

### Post by Sushi on 2005-07-27
[QUOTE=robins_web]KDE is too much like Windows.[/QUOTE]

How, exactly? People love to make that claim, yet they provide very little facts to back it up. And besides, being like Windows is not bad as such. Yes, Windows is a crappy OS and MS is even crappier company. But there's nothing fundamentally wrong in the UI of Windows. if you really look at it, you will see that even Gnome looks like Windows deep down.

I can think of several differences between KDE and Windows from the top of my head:

1. KDE is single-click, Windows is double-click
2. KDE has multiple desktops, Windows has one
3. Kmenu is different from Start-menu
4. By default, Kicker is double-height, Windows Taskbar is single-height

And so forth. What similarities are the between the two?

1. By default, both have Kmenu/Start-menu in the bottom-left corner of the screen
2. Both have "Close-window" button in the top-right corner of the windows
3. Both have system tray and clock in the botton right corner of the screen

That's about it. Of course there are other similarities like icons on desktop and the like, but those exist in just about all GUI's. Gnome is quilty of nr.2 as well, and they "solved" 1. and 3. by placing them to upper corners of the screen instead. But the basic idea is exactly the same. Should KDE also move the two to the upper parts of the screen, just so they could be different from Windows?

it seems to me that many people think that one of the primary design-criterias of a Desktop Environment must be "it must be different from Windows". That is, it must look different than Windows, for the sole reason of being different. I have seen this line of thinking go pretty far recently. Just yesterday I saw a KDE-user who told how great the "Autostart"-feature in KDE is. But then he started to complain that that feature is bad, because Windows has similar feature as well. So apparently KDE should remove that useful feature for the sole reason that Windows has something similar as well. Maybe KDE should remove icons and taskbar as well, because Windows has them as well....
