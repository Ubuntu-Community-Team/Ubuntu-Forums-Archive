### Post by tact on 2007-05-03
Regarding virtualisation in ubuntu:

The company I work for makes windows based applications and I do sales demos etc.  I used to have a heck of a job installing our products on different machines and networking them and getting servers talking to clients etc.

We looked into using VMware on windows boxes to ease the installation/configuration of the applications we sell and make repeatable demos easier - but you need real processing horsepower with GIGs of ram to even think about a smooth demo if windows is the host machine.

On a whim I used the linux version of VMware player on my little laptop.  Its no horsepower machine and 1.5Gb Ram only - running Edgy (at first), and later Fiesty.

Magic!  

Believe this - on my underpowered linux laptop I can have  MS-Server 2003 running in one VM window with  our server application.  PLUS in another VM window MS-WinXP Pro running our client application, and a THIRD VM running a "headless" linux comms controller.  

With the THREE VM's running and communicating to each other in real time via TCP/IP my system still runs nice and fast.  Positioning each VM in a different face of the Beryl cube makes for a mindblowing demo.

Now none of this means YOUR applications will work well in a virtualised windows running in VMware player on YOUR linux desktop.  But have I whet your appetite to give it a shot?

:)  Oh ya...  in my estimation Fiesty is better at runing VMs than Edgy.

### Post by tact on 2007-05-03
Another beautiful thing about using VMs - you defeat the old problem of windows, over time it becoming unreliable.  You install your version of windows in a VM, configure it how you want it, install your applications etc.  Then log out and shut it down.  Then save a copy of the (huge) VM file somewhere as a back up.

When dear windows gets old and tired after a while...  it only takes as long as a multi-gigabyte file copy takes to complete and you have your "original" cleanly installed windows back again.  ;)
(of course you save any work in progress to a folder shared between ubuntu and the VM first).

### Post by jimbo99 on 2007-05-08
This is in reply to the guy that was wondering why most commercial apps aren't finding their way to Linux.

First, there are estimated to be 3-5 times as many (world-wide) Linux boxes out there as there are Apple OSX boxes.  So the issue isn't one of market.  If someone had created a killer Linux program that program would be available to a far wider audience than many other products that were released when Windows was being developed.

With so much borrowing from one package to another with dependencies tied to so many other packages just to operate the larger development houses have to be feeling that anything they develop may subject them to a license they don't want to be bound to, especially for some potentially hugely popular product they might create.

I'm sure that is more of a lack of knowledge on their part than it is practical experience with Linux.  But it sure is a stopper when you have Microsoft FUD about every Linux user having to pay some final price to Microsoft for IP violations in Linux.  This FUD is totally false and Microsoft has been asked to put up or shut up, but this isn't their goal.  Their goal is to never actually sue Linux users, it is to create FUD so development doesn't switch to this platform.  In fact, if they sue they loose and they loose the ability to use FUD in the future.

Examples of companies that have successfully created Linux games are the people that created the Quake and the Unreal Tournament Series.  These companies have encapsulated their programs so that they aren't touched by other parts of Linux except maybe OpenGL and Xorg.

The Linux zealots have been pushing for *all* free software without understanding that even though open source products are excellent they are mostly incomplete and unpolished.  Take for instance the new Pidgin.  It is a nice product and it run extremely well under Windows and uses the nice transparency plug in.  But under Linux it has no transparency and is extremely hard to get installed on any other platform except one based on RPM.  You can compile the source but it is hard  for the average  user and that falls clearly in the realm of the Linux Zealot.  When you try the output of their ./configure generally displays a large number of lacking programs and it is hard for the average  user to know what those are or how they affect you.  Then if you take the default and install anyway you can't log in under Yahoo or MSN IM due to SSL and TSL dependencies missing, among others.

Pidgin is such a perfect example that even Linux apps are slow to be released on the various distros while the windows version is immediately available the day of the release.

The point being, that even if commercial apps were to be released modifications to the environment are slow in coming, lack the quality assurance testing necessary, and any change could kill their app if they rely too much on Linux provided libraries, etc.  This cuts into their revenue.

No matter how you look at it, even with the advent of Feisty Fawn, it is still hard to get some video drivers working properly under Linux.  This has come a long way but when you look at the drivers provided say for the nVidia cards they put in an older set of drivers so the user is still forced to go back and install the most recent in order to take advantage of the faster drivers with bug fixes.

This isn't to say that Windows doesn't have the same or similar problems.  Certainly development under Windows has made the applications mostly encapsulated with very little inter-app sharing of libraries.  One update to a core library could harm a bunch of Windows apps but that is rare and they are alerted fast enough to get fixes out ASAP.

The example of removing GAIM is on point.  If you try to remove GAIM you get prompted to also remove many other aspects of the gnome desktop. I was trying one day to resolve some GAIM crashing issues and when I tried to remove it I was prompted to remove a lot more than just GAIM.

Essentially you have so much borrowing that dependencies take on a new phase that complicates commercial development, you have people professing they only want free and open software on Linux--which is an all or nothing attitude.  Then you have the likes of Microsoft spewing their FUD.
