### Post by realkerim on 2020-04-05
Hi all, 

Ubuntu 20.04 introduced a very annoying feature, where when you want to close an application and it prompts you for various reasons (e.g. closing unsaved file, closing multiple tabs in Firefox, etc.) then the active application slowly darkens and the prompt is shown like this: [https://i.imgur.com/MOz3LnI.png](https://i.imgur.com/MOz3LnI.png)

Unfortunately, this whole darkening until the prompt shows takes way too long, 1 to 1.5 seconds, which is annoyingly slow. This is especially noticeable when programming and closing unsaved/untracked/modified files but, but also when closing terminals with active sessions or Firefox with multiple browsers. 

I think it may be a bug, because in VS Code at least I noticed that the first time the prompt is shown, it's slow (the application darkens and then when that animation is finished the prompt is shown) but after that it's fast (prompt showing and application darkening happen at the same time).

Is there a way to fix this? Or to disable it altogether without disabling all other animations?
