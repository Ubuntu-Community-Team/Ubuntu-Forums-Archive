### Post by HermanAB on 2005-11-25
Here is your problem:
"[4294698.984000] ipw2100: eth1: Firmware 'ipw2100-1.3.fw' not available or load failed."

The Intel IPW adaptor contains a little ARM processor that needs firmware to do its work.  The device driver is supposed to download this firmware to the device, but the driver cannot find the firmware file.  The result is that the IPW device remains dead.

Go to the Intel IPW2100 site and start reading.  You need to download the firmware file from the web site and put it in a very specific place, where the driver can find it:
[http://ipw2100.sourceforge.net/](http://ipw2100.sourceforge.net/)

I don't have this card, so I can't tell you more.

### Post by kxmas on 2005-11-25
I have a 2200, so it's not quite the same, but I think this will work:

copy these files in /lib/hotplug/firmware so that it ends with your kernel name
ipw2100-1.3.fw-2.6.12-9-686
ipw2100-1.3-i.fw-2.6.12-9-686
ipw2100-1.3-p.fw-2.6.12-9-686


for example, if "uname -r" is 2.6.14-ck5 then one file would be ipw2100-1.3.fw-2.6.14-ck5
