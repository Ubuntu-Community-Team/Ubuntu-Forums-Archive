### Post by ldrn on 2007-07-14
I really think so... well, I'd call it more of "knowing" so; I mean, it's common knowledge that Gnome's goal is to focus... I forget the nice quote, but it's something along the lines of easy to use defaults and not confusing anyone with being able to change them.

That isn't to say there are not some cases where the opposite isn't true, although I think you'll find the list of things you can do on KDE but not gnome is very large, while the opposite list is small. 

I don't know of any emblem equivalent; I don't think one exists.  KDE does not have a Windows-like registry like Gnome does, so there is no Gconf tool.  There's kcontrol, which is a gui; for changing and accessing configurations from the command line, KDE acts like most Unix programs -- the configuration files are in the .kde subdirectory of your home.  If you've ever set up a non-gnome application, you'll know right away how to do it with KDE from the command line; that being said, KDE's graphical configuration tools are much better for tweaking than those in Gnome; none or next to none of the options are hidden. 

Window management is where KDE really shines -- you can add toolbar buttons (always on top and shade are two I don't want to deal with a WM without) and apply custom rules to windows.  There is a similar application called Devilspie that works with Gnome, but KDE's is integrated and incredibly easy to configure.  Compiz adds a bunch of these to features to gnome -- I use Gnome with Compiz on my desktop, and I find it a great combination -- but I don't know if they'll last.  In the past, Gnome has refused patches that add those kinds of features or the option for them.

Another big one: almost all KDE applications have a "Settings" option in the menu, where you can Configure the application, the keyboard shortcuts the application uses, and its toolbars. If you don't like the default settings, it's simple and quick to change them.  You can add or rearrange buttons on the toolbars and whatnot as well; one place this is very noticeable is when comparing Nautilus (Gnome) and Dolphin (KDE).

You can also configure your desktops to have different wallpapers in KDE, too, and they'll show up as such in the pager.

If you do try to stick with it, I think you'll find that KDE's Applets don't look quite as nice as Gnome's or work quite as well -- but that's probably just my own impression based on the ones I frequently use in both Environments. :) Anyway, if you've been using Gnome a long time, is there a particular reason you are trying KDE out?

### Post by aysiu on 2007-07-14
Konqueror has all sorts of plugins to handle what Nautilus scripts do (Edit as root, for example). KDE allows you to specify what pictures you want in a screensaver slideshow. You can easily have different wallpapers for each desktop. You can have full transparency in the taskbar. You can have a universal toolbar. You can change the color of any theme. You can preview images before uploading them in the web browser. Oh, the list just goes on and on. Of course KDE is more customizable than Gnome is, and if you include only customizations done through the GUI, then even more so!
 
Does that make KDE *better* than Gnome, though? Not necessarily. In fact, many would argue that it is the simplicity of Gnome that is its strength. Linus Torvalds wouldn't, but others would.

### Post by Afoot on 2007-07-16
Oh yeah. KDE is definitely more customizable than GNOME. Say, for instance, that you want to change what happens when you right-click the desktop. In GNOME, you can't change it (except perhaps if you're willing to look through gconf-editor for like an hour before finding the right options. That is, if they're even there). However, it's super-easy to change in KDE. You can change what happens on left mouse button click, middle click, and right click. You can even customize the menues themselves. Just try to change something in KDE, you'll surely be able to do that. However, if you try it in GNOME, you're out of luck. That's how it's been for me anyways.
