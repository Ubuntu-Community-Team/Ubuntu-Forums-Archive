### Post by LexRoss on 2008-03-07
The thing I like most about Ubuntu is that it is optimized for usability and is logical. That made me switch from KDE I used to love so much to GNOME. Now I noticed that installing kubuntu-desktop package renders Ubuntu unusable, as its applications menu becomes cluttered with the mix of GTK and Qt apps doing the same thing.

What I want is KDE desktop installation to leave my existing GNOME environment intact, and vise versa. In fact, this should be the default system behaviour, and here is why.

Let's say there are Ubuntu users and there are Kubuntu users, both used to their default application suite. As they log into the machine with both GNOME and KDE installed, they should get just that - the desktop they know of, and not the one cluttered with foreign apps. If the user wants particular application from another set then, it is very easy to add it to the menu list, and keep the system usable.

So once again, how do I instruct the system to log in user into "clean" Ubuntu or Kubuntu on a machine which has both?

Thanks.
