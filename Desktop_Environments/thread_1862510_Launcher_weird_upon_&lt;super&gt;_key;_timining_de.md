### Post by DrJohn999 on 2011-10-16
Using 11.10 Oneiric desktop; this is an in-place upgrade from 11.04. When I press and quickly release <super> (currently at the default left-Win key but same thing happens if <super> is remapped to, say, the right-alt key) the launcher opens and displays the associated numbers on each icon immediately. At this point the launcher will never autohide (it's set to autohide in CompizConfig / Unity Plugin).

Holding <super> for about 1 second causes the numbers to disappear and the launcher then autohides.

Holding <super> a little longer sometimes displays the dash, sometimes not.

Holding <super> for a few seconds causes the dash / launcher areas to flash and toggle rapidly between different states (of dash or launcher or mayber launcher w/ numbers -- it's hard to tell).

Sometimes, when the launcher is stuck thus (again, hard to duplicate on purpose), all of the application menus become inactive, including the Desktop menus. Holding <super> for a while sometimes re-activates the menus, sometimes not... Yuck! What a mess!

Is this the intended behavior for Oneiric Unity? It becomes quite difficult to use with a key-press timing dependency especially for someone with mousing difficulties.

Or is this a bug? Or is this a problem w/my system? 11.04 does not do this at all. Hold the <super> key down and the launcher stays on numbered icons. Release <super> and the launcher autohides. Simple, works.

In any case, I won't be upgrading any other systems in the foreseeable future. I don't see any benefits at all to 11.10 over 11.04. Seems more like it was released because the 6-months release schedule said so.
