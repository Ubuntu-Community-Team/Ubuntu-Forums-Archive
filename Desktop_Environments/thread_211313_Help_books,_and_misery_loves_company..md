### Post by Kizzume on 2006-07-08
This is my 5th attempt at working with Linux, and I can't say it's been a pleasant one.  For Windows, I've worked helpdesk technician jobs and got known for being able to fix the weird problems, I've been going to people's houses to fix their Windows computers for over 6 years (and macs for 2 years), and I've been using computers in general since 1984.  

Nothing has ever been as much of a challenge for me as trying to just get normal things working under Linux.  The reason this distribution has been so frustrating is that everything that I want to use I have to apt-get first before it's even available, but then have to do a bunch of other things so apt-get even finds anything.

I had to go to the chat room to find out how to get commands like "make" installed.  I have a delta 66 sound card, and it took 6 hours of troubleshooting with a system administrator (who has also been a programmer for 10 years) on irc just to get the Delta 66 working, and then it stopped working once I restarted, even though the command to start it was in /etc/modules.  It still isn't working now because I didn't write down the commands that I didn't know I'd need.

About half of programs for linux are only available as the source code.  Vice emulator--I've tried getting that going every time I've worked with linux, and I would have assumed that compiling tools would have been improved, but it's been the same since 1999.  Oops, I have the wrong version of libsigc, I don't have that directory already made, I'm missing that library that can't be found anymore, the syntax for that library has changed... blah blah blah..    In seven years there aren't any more tools that help this sort of thing.  It's bizzare to me.  Are most people satisfied with just using the programs that come with a linux install, or what?  

Today was the first time I've flat-out been in tears over an operating system.  Just flat-out bawling.  It's been a very humbling experience.  It's made me feel really really stupid.  I have never struggled this badly with an OS.  Microsoft is messing up badly with their WGA and some of their poorly-written updates that seriously hamper the efficiency of one's computer, and I'm willing to keep trying to work with Linux because it's the only alternative to Microsoft unless Apple gets smart and releases their OS for non-Apple hardware, and I'm certainly not going to pay double the amount for Apple computer hardware.

Does anyone have any recommendations for books on learning how to compile programs?  On learning troubleshooting techniques?  Books that have short, easy to understand definitions of the most commonly used commands (instead of the usual descriptions one finds where it's either written in short gobbly-gook acronyms or is such a long explaination that one falls asleep while reading it)?

I'm used to tweaking with text files and using memory hack programs like T-Search and the such from being the type of person who plays games to find out how to get them to do things they weren't designed to do--I've spent 20 hours in a row in front of the computer making games play the wrong way, modifying terrains, changing things, messing with game physics.  I'm not used to -having- to tweak with text files and the such to get things working the -right- way, and I don't know how to change my attitude about the whole thing.  If I could somehow make this fun, there'd be no problem and I'd be happy-go-lucky about the whole process.

What I'm hoping for out of this post is information on books, and possibly some responses that show how frustrated other people have gotten while trying to learn how to work with Linux.  Misery loves company.

Thanks.

### Post by matthew on 2006-07-08
I can sympathize. Learning a new OS is never easy, but with some time and a good attitude you will get the hang of it. A willingness to study and learn will take you a long way. If you want actual paper books, anything by O'Reilly is likely to be good. Here's their web site:

[http://oreilly.com/](http://oreilly.com/)

While not specific to compiling software I am still going to recommend you start with these four books, in this order from most basic to serious power user stuff:

[http://www.oreilly.com/catalog/runlinux5/](http://www.oreilly.com/catalog/runlinux5/)  *Seriously, every Linux user should read this book.
[http://www.oreilly.com/catalog/linuxckbk/](http://www.oreilly.com/catalog/linuxckbk/)
[http://www.oreilly.com/catalog/linuxnut5/](http://www.oreilly.com/catalog/linuxnut5/)
[http://www.oreilly.com/catalog/upt3/](http://www.oreilly.com/catalog/upt3/)

EDIT: if you really want to get into the programming aspects on Linux this is a good one as well. [http://www.oreilly.com/catalog/prognu/](http://www.oreilly.com/catalog/prognu/)

EDIT #2: I forgot to mention that you can often find these books used on Amazon.com for a mere fraction of the cover price. It's worth a look.
