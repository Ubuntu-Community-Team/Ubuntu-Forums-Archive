### Post by radl33t on 2006-08-30
I updated xgl server with compiz on two machines via Automatix Bleeder after my first update since installation of LTS back in June. This XGL server seems to have serious issues with keyboard input.

1 AMD64 (i386 kernel) w/ Nvidia 7800GTX
1 Centrino w/ Nvidia 5200FX

Both machines exhibit the same symptom. X crashes with fast input. Too many keys at the same time or two many in a given time period, the former being more sensitive. This happens most frequently when holding shift for several letters in a row (i.e. acronyms). Several applications have instigated this including evolution and firefox/swiftfox. This issue is not dependent on the window manager (compiz and metacity behave the same), leading me to believe it is from the Xgl package or a wider problem with the last update.

I am surprised to not have found this problem documented given its repeatability on both of these machines. Of course this could be more a function of my inability to express the issue in my defunct searches.

Suggestions Please.
