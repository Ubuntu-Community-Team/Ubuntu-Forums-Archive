### Post by louis_nichols on 2006-08-10
the w32 codecs are pretty useless for amarok.

What amarok can or cannot play all depends on the version  and the engine you use. The version of amarok in dapper can only use xine as an engine, whereas previous versions could use gstreamer also.

Now, if you have xine as an engine (which I personally use and recommend), then you need to also install the package libxine-extracodecs. If you use the gstreamer engine, you need the gstreamer-bad and gstreamer-ugly packages (and maybe others). Now, if these can play wma, that's all good. But if they don't, then I'm afraid there isn't very much we can do.
