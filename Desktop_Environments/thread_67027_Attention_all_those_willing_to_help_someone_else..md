[i]
The unnoficial ATI ubuntu howto:

1)If you have installed the drivers through synaptic or apt-get, uninstall them using apt-get or synaptic(or kynaptic).
2) Download the 50MB(!) installer from ATI ([http://www2.ati.com/drivers/linux/a....16.20-i386.run](http://www2.ati.com/drivers/linux/a....16.20-i386.run))
3) check that you have your kernel headers in /usr/src/linux-headers-<version> and that /usr/src/linux is a symbolic link to that directory. This is needed to compile the fglrx kernel module.
3.1)If you don't have the headers get them by finding the package in synaptic called "linux-headers-(the name you got from uname -r)". Then "ln -s /usr/src/linux-headers<version> /usr/src/linux".
4) change to single user mode via "sudo telinit 1", this will drop you to the console.
5) start the installer as root: "sudo sh ./ati-driver-installer-xxxxx"
6) most people can just accept the defaults in the installer (just hit enter at each screen) until the driver starts installation.
7) wait for installation to finish (shouldn't take too long, 30sec at most maybe).
run "sudo fglrxconfig" to generate your /etc/X11/xorg.conf file or simply change "radeon" to "fglrx" in your Device section if it exists already. It might be a good idea to generate one even if you have one already just to see how they are different. Make sure you don't copy over the original!
9.)reboot (I recommend a reboot, even though it is technically not necessary, as I've hit problems before when not rebooting).

Troubleshooting:
Direct Rendering doesn't work:
Try "ls -l /lib/modules/(kernel version)/kernel/drivers/video/fglrx.ko". It should say the same date as the date you ran the ati driver installer. If not you will have to copy the fglrx driver from /lib/modules/fglrx/. First rename or delete /lib/modules/(kernel version)/kernel/drivers/video/fglrx.ko and then "sudo cp /lib/modules/fglrx/fglrx.(kernel version).ko /lib/modules/(kernel version)/kernel/drivers/video/fglrx.ko". Then, make sure you REBOOT.

### Post by snoochems on 2005-09-21
Okay, thanks for your help so far, (*rolls eyes*) but i'm about to try this method i found here:  [http://ubuntuforums.org/showthread.php?t=64234&page=3&pp=10](http://ubuntuforums.org/showthread.php?t=64234&page=3&pp=10)


[I]
So to sum everything up, here are all the steps I took from a fresh install:

Uninstall xorg-driver-fglrx (if you have installed it)
Uninstall linux-restricted-modules-<kernel version>
Delete everything in /lib/modules/<kernel version>/volatile, or at least just fglrx.ko

Install (if you haven't already): build-essential, linux-headers-<kernel version>, gcc-3.4, libstdc++5

Download and run the ATI installer (with sudo).
Choose automatic installation, blah blah.

When that is done, log out and then hit CTRL-ALT-F1 to get to the console.
Login on the console and run "sudo /etc/init.d/gdm stop", and then "sudo modprobe -r fglrx"

Build and install the new fglrx module by doing:
cd /lib/modules/fglrx/build_mod
sudo ./make.sh
cd ..
sudo ./make_install.sh
depmod -ae

Make sure that "find /lib/modules | grep fglrx | grep ko" reports that only one fglrx.ko exists, and that it is at /lib/modules/fglrx/fglrx.ko. Specifically, make sure there are is no fglrx.ko under /lib/modules/<kernel version>/. Those other modules under /lib/modules/fglrx are fine.

Make sure "fglrx" is listed in /etc/modules

Reconfigure xorg.conf if necessary:
sudo dpkg-reconfigure xserver-xorg

Most things can probably be autodetected, but for device choose fglrx instead of ati. After writing xorg.conf, you may need to edit it to add a few options to the Device section, such as:

# To use the xv extension
Option "VideoOverlay" "on"
Option "UseInternalAGPGART" "no"

Alternatively, you can use "sudo fglrxconfig" to write your xorg.conf, which should ask lots of questions about driver-specific options.

Reboot and pray for the best!

EDIT: I forgot to include the libdri.a thing. Be sure to copy over an older libdri.a (from Hoary) to /usr/X11R6/lib/modules/extensions/libdri.a (back up that file first). This is explained in some other threads.[/I]
