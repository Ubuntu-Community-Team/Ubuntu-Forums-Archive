### Post by VincentVX73 on 2007-06-22
Good news! I found a startxgl.sh script that works with the dual-head setup.

```
#!/bin/sh
DISPLAY=:0.0 
Xgl :1 -fullscreen -ac -accel xv:pbuffer -accel glx:pbuffer -br &
DISPLAY=:0.1 
Xgl :2 -fullscreen -ac -accel xv:pbuffer -accel glx:pbuffer -br &

export DISPLAY=:1
exec dbus-launch --exit-with-session gnome-session &

export DISPLAY=:2
exec dbus-launch --exit-with-session gnome-session
```

Using that, the screen designated as screen 0.0 will start up its xgl session, then screen 0.1 starts it's own xgl session.

### Post by javroch on 2007-09-11
by the way.. if the "Hibernate" or "Restart" options do not show up for you when you click to log off using Xgl and dual-head, you can use this startxgl.sh instead and it will put the "Hibernate" and "Restart" options back for you:

```
#!/bin/sh
DISPLAY=:0.1
Xgl :1 -fullscreen -ac -accel xv:pbuffer -accel glx:pbuffer &

DISPLAY=:0.0
Xgl :2 -fullscreen -ac -accel xv:pbuffer -accel glx:pbuffer &

#for hibernate / restart menu
cookie="$(xauth -i nextract - :0 | cut -d ' ' -f 9)"
xauth -i add :1 . "$cookie"
xauth -i add :2 . "$cookie"

export DISPLAY=:1
exec dbus-launch --exit-with-session gnome-session &

export DISPLAY=:2
exec dbus-launch --exit-with-session gnome-session
```
