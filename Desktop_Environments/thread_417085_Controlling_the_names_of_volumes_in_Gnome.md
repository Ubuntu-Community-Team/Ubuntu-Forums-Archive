### Post by kairn on 2007-04-21
I'm able to mount my two NTFS drives with no problems.  On the desktop, the first drive's name matches the associated folder name in my media directory (windows), as I would expect.  But the second drive's name is always "New Volume" rather than matching the directory's name (backup).  Does anyone know where these names come from and how to change them?  Just a minor annoyance, but I thought I'd ask.

Thanks in advance.

---

### Post by CameO73 on 2007-04-21
The names for the icons on the desktop represent the **volume label**. This label is usually set when formatting or creating a partition. In Windows this can be set by right-clicking on the drive --> Properties and entering a name. No idea how to do this in Linux, though.

---

### Post by kairn on 2007-04-21
Yep, you're right.  "New volume" is the volume label.  When I changed it in Windows, the name of the drive in Gnome changed to match.  I noticed that the name of the first drive (the one whose name tracked the mount directory) just showed up as "local disk" in Windows.  So perhaps it didn't have a volume label, potentially explaining why it tracked the mount directory in Gnome.  Anyway...thanks again for the assistance.
