### Post by TheFu on 2021-07-08
> **ml9104 said:**
> I always thought that that's what Alt+Tab is supposed to do. Am I missing something here?

Alt-tab is just another chord. It can be anything your mind and WM can create.  On Unix systems, any GUI is just another program. It isn't part of the OS, so there aren't any mandated key chords outside the power button which usually needs pressing for 5-10 seconds before the HW aspect kicks in and it powers off a system.  Everything else is optional and configurable.  

I guess kids these days don't learn that.  In the mid-90s, we had to setup any chords that we wanted. 

Even the hot areas in the L/R corners of titlebars for a window are configurable too.  We can have 0-6 "buttons" in the corners, each can be programmed for anything - add in the 3 main modifiers and that's 18 possible window commands.  Add in the possible chords (shift or shift-alt or shift-ctl) and the possibilities explode.  And that's with just 1 mouse button.  Some mice have 7 buttons. Imaging the possibilities!  All of this is configurable ... well, maybe not using Gnome, but if other WMs are used, all of this is configurable.

Odd thing. I never setup alt-tab, so that does nothing here. If I right click on any part of the empty desktop a list of all windows gets displayed which can be selected via mouse or arrows.
