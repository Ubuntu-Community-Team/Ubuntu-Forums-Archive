### Post by n-alexander on 2008-02-25
1) edit /boot/grub/menu.lst

2) find your Linux section, the one you boot, something like this:

title		Ubuntu 7.10, kernel 2.6.22-14-generic
root		(hd0,2)
kernel		/boot/vmlinuz-2.6.22-14-generic root=UUID=f622900b-4696-46a4-8384-f3e07c772aa7 ro quiet splash vga=792
initrd		/boot/initrd.img-2.6.22-14-generic
quiet

3) check that you do have the vga=792 at the end of the kernel line. Below is the table for other resolutions vs. color depth:

+ | 640x480 800x600 1024x768 1280x1024
+---------+----------------------------------------------------------------------------------
+256 | 769 771 773 775
+32k | 784 787 790 793
+64k | 785 788 791 794
+16M | 786 789 792 795

thanks Thomas from Fedora forums [http://linux.derkeiler.com/Mailing-Lists/Fedora/2007-03/msg00062.html](http://linux.derkeiler.com/Mailing-Lists/Fedora/2007-03/msg00062.html)

This did it for me.

4) if you want it from GUI, go to System/Administration/Start-up Manager (install if missing).

If it shows an incorrect resolution - just fix it.

Otherwise select a different resolution/color depth and close it - it'll make it write out that "vga=" info. Then you can go back and get the desired resolution back.

### Post by erginemr on 2008-02-28
OK. Can you please try these:

1. First, open a terminal and back up your current X config file:
```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.mybackup
```

2. Edit your X config file with:
```
sudo gedit /etc/X11/xorg.conf
```

3. In your xorg.conf, look for Section "Monitor" and insert this line:
```
Section "Monitor"
Identifier "SyncMaster"
Option "DPMS"
**[COLOR="Red"]DisplaySize 	270	203	# 1024x768 96dpi[/COLOR]**
EndSection
```
You can omit the part starting with (#), which is a comment.

4. Save and close the file. Restart your X server with Ctrl+Alt+BackSpace or better, by passing to a virtual terminal with Ctrl+Alt+F1-F6 and writing
```
sudo /etc/inidt.d/gdm stop
```
to stop the X server and
```
sudo /etc/inidt.d/gdm stop
```
to restart it, in order to see if you achieve 1024x768 at login screen.

5. If not, then edit xorg.conf again and look for Section "Device" and insert this line:
```
Section "Device"
Identifier "Intel Corporation 82865G Integrated Graphics Controller"
Driver "intel"
BusID "PCI:0:2:0"
**[COLOR="Red"]Option "MetaModes"	"1024x768"[/COLOR]**
EndSection

```

6. Save and close the file. Restart your X server again to see if you achieve 1024x768 at login screen.

### Post by darkside299 on 2008-03-14
I got the solution from Bhavani Shankar and Max.

Use the following procedure:

To make a backup copy of your actual /etc/X11/xorg.conf type:

cd /etc/X11
sudo cp xorg.conf xorg.conf.bak

give your user password when requested, you don't see nothing when you type it, then press enter.

Then try to reconfigure your xserver-xorg

sudo dpkg-reconfigure xserver-xorg

answer to the simple questions, then restart the pc.

If something goes wrong you can restore the saved copy of your old configuration, open a terminal or login to a virtual terminal by pressing ctrl+alt+F2 and type:

cd /etc/X11
sudo cp xorg.conf.bak xorg.conf

and restart the pc, type:

sudo reboot

For more info: [https://answers.launchpad.net/ubuntu/+question/27026](https://answers.launchpad.net/ubuntu/+question/27026)
