Hi.

I'm running Feisty. Recently, I've encountered some strange behavior in the panel.

This started when my keyboard indicator froze a couple of times. Clicking had no effect. Refreshing the panel solved this ... at least temporarily. However, on some occasions, the indicator shows English, but the keyboard in use is Hebrew. Clicking on the switcher has no effect and I can't type a killall command in the terminal because it will only write Hebrew. (The default language is English, but Hebrew, French and Spanish are activated and accessible from the switcher)

I put a drawer in the bottom panel and added a custom launcher to run the killall command in terminal. That part works OK and the language indicator resumes functioning normally.

BUT, I also put launchers for Desktop Search and System Monitor in the drawer. Sometimes, even after a reboot, these disappear. They are really still there, but not displayed. If I add another copy of the launcher to the drawer, I suddenly have two (the original comes back as well). Then I remove one and am left with the launcher I had before. Refreshing the panel alone does not bring back the launcher icon.

I add these launchers (Search and Monitor) by first placing them on the Desktop and then dragging to the drawer. Using the "Add to Drawer" function has wiped out the panels completely on several occasions. There is an error notice about a script problem and everything disappears and can't be refreshed. I needed to put both panels back together manually, even the main menu, notification area, logout icon, etc.

I haven't found mention of a bug anywhere.

Does anyone know anything about this behavior or have an idea as to whether it's fixable?

Thanks.
