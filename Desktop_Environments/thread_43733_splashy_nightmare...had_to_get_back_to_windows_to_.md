### Post by tristan on 2005-06-23
Hi byen

Sorry it didn't work for you. Don't punch yourself in the face just yet...

I'm mystified as to why splashy would hose down X for you, all I can think of is that framebuffer gfx and X don't get along nicely on your system. Splashy most definitely doesn't touch your X config in any way. Is it possible for you to summarise the Xserver output from the error box, that might give us a clue as to why X spat the dummy for you. Also, what video card and drivers are you using?


The main thing is that your system has still booted OK. Did you set up your grub so that it defaults to booting with splashy?

If you can boot your system to a command line (or press ctrl-alt-F1 and log in), you should be able to edit /boot/grub/menu.lst (using vi, mc etc) and remove the offending **vga=792** which makes your system use a graphic framebuffer needed by splashy.

```
kernel		/vmlinuz root=/dev/hdaX ro quiet splash vga=792
```

When you reboot from this, splashy will still be present but it won't be able to display anything because there's no framebuffer.

Hopefully this will get you to gnome, from where you can uninstall splashy properly.
