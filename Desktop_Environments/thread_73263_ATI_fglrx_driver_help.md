> Does anyone have any ideas what is going on?  I noticed there is some stuff missing from the fglrx enabled one, maybe I have to combine some stuff.  If I can get this running and get Cedega working with it, I will be making my PC a Linux only box.  Can anyone help?

Try booting up with the fglrx driver and once it has failed use the Ctrl-Alt-F2 keys to switch to a terminal.  Copy the /var/log/Xorg.0.log file to your home directory or somewhere else.  Then you can switch back to the ATI driver and log in.

```

cp /var/log/Xorg.0.log ~

```

Post the Xorg.0.log file (if you rename it to end in .txt then you can attach it to your post) that gets created and we'll take a look at what it is trying to do.  There are known issues with Samsung displays and the latest driver, but normally you'd just end up with a 1024x768 display.
