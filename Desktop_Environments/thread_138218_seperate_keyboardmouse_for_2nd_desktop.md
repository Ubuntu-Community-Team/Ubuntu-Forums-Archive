### Post by ArtInvent on 2007-04-20
Multi-seat seems to be the general term for using one computer with more than one user at the same time. The technique works under Ubuntu and Linux in general but it is at this point (early 2007) in a non-user friendly state with lots of manual configuration. There is a lot of work being done on this front as modern CPU's have plenty of power to run more than one user, and the economics and administration of the system are way better than having a single freestanding pc for each user. Schools, libraries, and internet cafes are the initial places where these setups are appearing, but general home and business users will also be interested since the exact same advantages apply. 

There is also the question of sound and speakers and how to assign these - this is generally a weak spot, but you can see that unless you had multiple sound cards or separate sound outputs this would be difficult. Multiseat is still extremely useful however. 

There are a number of methods floating around. You can also search for 'multsieat' or 'multi-seat' and 'linux' or 'X' or 'Ubuntu'. Here is one that looks most promising to me:

[https://help.ubuntu.com/community/MultiseatX#head-2101e5aacc76b7db56a5c259dc0eb36233401128](https://help.ubuntu.com/community/MultiseatX#head-2101e5aacc76b7db56a5c259dc0eb36233401128)

Two other methods that look a little more difficult:
[http://www.c3sl.ufpr.br/multiterminal/howtos/howto-xephyr-en.htm](http://www.c3sl.ufpr.br/multiterminal/howtos/howto-xephyr-en.htm)
[http://netpatia.blogspot.com/2006/09/multiseat-computer-with-ubuntu.html](http://netpatia.blogspot.com/2006/09/multiseat-computer-with-ubuntu.html)

All the above methods basically involve - 
- first configure the multiple screens in xorg-conf. This would be set up initially as if running a single user desktop with multiple monitors.
- next use various methods to assign a keyboard and mouse set to a single monitor. This involves at least editing gdm.conf, and can get more elaborate.

Then there is the Ndiyo project [www.ndiyo.org](www.ndiyo.org) which is a fantastic idea just getting off the ground. It's basically somewhere in between a thin client and multiseat, using just a network cable hooked up to a small box at the client station containing a graphics card with monitor jack, usb ports for keyboard and mouse, and sound jacks. Sort of a detached, network-connected multiseat or 'ultra thin client'. It's not a nomal thin client because the client has no processor or RAM or hard drive, but uses the computing resources solely of the server machine.

Should also mention Userful, a commercial, non-free system that is really more of a service marketed to libraries supplying all hardware and software. But they sell the software alone as well, and it is actually in the Ubuntu repos. Have heard there are some issues with it actually working, but apparently it offers a simple GUI to assign keyboards and mice to various displays.

Note that the package available in the Ubuntu repos called 'multiseat' is not real helpful in setting this up, it seems to be more like utilities for multi-seat users once you get it set up. Also, there is very little documentation that I could find on that package. It is definitely not a matter of setting up two graphics cards with monitors, plugging in an extra keyboard and mouse, and install the 'multiseat' package. That will do nothing that I can see.

I look forward to a better method and hopefully a GUI utility for setting this up in Linux. It's really not much of a stretch as Linux and X in general has always been well designed for remote and multiple logins, users, and desktops.

### Post by ArtInvent on 2007-04-23
Yes, X seems incredibly flexible and capable, but unfortunately xorg.conf is a very critical setup file, and it is still required to edit it manually for the most part. I feel that is one great weakness of Ubuntu. Just to get all your mouse buttons working you have to manually edit this cryptic text file? And making a small error in this file can make your system unusable? Bad.

Once you get to know the rules of xorg.conf, you see how to link and assign graphics cards, monitors, and 'screens' and how to get multiple monitors working. Then it might seem that to assign a keyboard and mouse to one particular screen would be a simple matter of having two separate 'layout' sections. Unfortunately it's not that simple, X is really not set up for simultaneous users. Thus you have to look at ways to have xorg.conf work with gdm.conf.
