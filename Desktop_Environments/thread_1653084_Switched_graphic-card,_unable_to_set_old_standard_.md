Earlier I had 1680x1050 which to me seemed to be the genuine resolution. But after switching to this card, the closest and highest I can set is 1600x1200. Which makes my screen "fuzzy".

I got the Nvidia 173 drivers, and installed them from the hardware driver application under System->Administration.

So I've tried nvidia-xconfig and rebooted. No luck.

I've tried to edit my xorg.conf file and manually adding "1680x1050" to modes. Also tried adding ```
Option "metamodes" "DFP: nvidia-auto-select @1680x1050_60 +0+0"
```

This at some point gave me a "out of range" message on-screen. But after lowering horizsync and vertrefresh to

```
    HorizSync       30.0 - 60.0
    VertRefresh     50.0 - 60.0
```

I managed to get back into the Ubuntu GUI, but still with 1600x1200 resolution.

Also tried something like

```

### Post by DarkAmbient on 2011-01-05
Haven't touched this computer in a week, but got back to it now. And I might add I got 1680x1050 resolution to work. Of-course the solution was easier than I though. Backed-up xorg.conf, Installed "nvidia-current", made them active. Worked like a charm. *a'lil embarrased*
