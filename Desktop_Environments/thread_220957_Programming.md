### Post by jayemef on 2006-07-22
There are lots of books out there that are designed for picking up a language. I would suggest picking one up that has assignments at the end of the chapter, and just go through it, doing an assignment or two from each that sounds interesting to you. Deitel has many such books, and for the most part get pretty good reviews.

As for which language to start with, I would personally recommend either C++ or Java, as being familiar with C-type language syntax will really help you out down the road, as well as give you a greater appreciation of what interpreted languages are all about.

### Post by Thund3rstruck on 2006-07-22
If you indeed have no programming experience then there's no need to dive into a high level language such as Java or C++ and bore yourself to death learning constructs, object creation/management, garbage collection, inheritence, polymorphism, encapsulation, and all the other object oriented pieces right away. In my experience this more than anything else will frighten new coming programmers away more than anything else. 

What you need is instant gratification so start playing around with scripting preferably on linux bash shell scripting and on windows Windows Script Host (via VBScript or WMI). This will have you writing and running functional code in minutes. Once you create some scripts that do useful things such as enumerate the CIV2 namespaces (WMI) or a vbscript that finds cache and deletes it, you should be ripe with motivation to continue with programming and then you can start looking at a higher level language. 

Just my opinion...
