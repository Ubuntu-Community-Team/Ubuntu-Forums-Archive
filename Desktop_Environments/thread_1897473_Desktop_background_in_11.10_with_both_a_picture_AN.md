### Post by StewartM on 2011-12-21
Well, I feel like an idiot, for I missed the obvious solution. :roll:

The simple solution (if you just want to use custom wallpapers for yourself) is to simply create them with GIMP and either plop them in the /home/Pictures folder, or create a /wallpapers folder in your /home directory, select "pictures" in the GUI option, and navigate to that.

The simple solution (if you want to share wallpapers among users) is to likewise create them then either plop them in a folder accessible to all users (like usr/share/backgrounds). Even if GNOME/Unity won't recognize them as backgrounds per se, it will see them as pictures. Then you select the "pictures" option, navigate to usr/share/backgrounds, and you add your custom wallpaper there (clicking on the "zoom" or "scale" option to fill the screen, assuming you've taken the care to give the picture the approximate dimensions of your screen.

I've restored the original /usr/share/gnome-background-properties/ubuntu-wallpapers.xml file and deleted the new. Now all I see are the default wallpapers. However, that means I have gone from seeing *three* copies of the default wallpapers to *two*. Again, this is not a huge problem (it doesn't hurt to see two copies) but I will try to figure it out. Maybe I should have used a custom .xml editor instead of gedit, though none of the sources I read warned me against doing that.

As for your software suggestions, both of these attempt to do something more ambitious, which does require referencing an .xml file--to create desktop slideshows (i.e, the wallpaper changes given a certain time period). Only Desktop Nova is now available in 11.10 (for good reasons).

Oh and BTW: KDE programs generally run fine under GNOME, and GNOME programs run fine under KDE (at least you can have Ubuntu and Kubuntu installed, as I do, and the programs show up in both menus and work). Wally however, was much more ambitious--it advertised to work on Macs and Windows machines as well as Linux desktops.
