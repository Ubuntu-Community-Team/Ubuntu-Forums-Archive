### Post by tonyr on 2006-07-03
This **Makefile** does not seem to be in tune with the times.  There is a hardwired
requirement for **make** version 3.80 in there.  Looks like you'll have to edit
the Makefile to force it into the present.

Alternatively, you could remove **make** and force install an older version.
But, (and it's a BIG but), you will continue to run into requirements problems related
to versions and missing packages.   I don't know what the development environment
was where this **bulldozer** tarball was put together, but it resulted a not very flexible
environment on the downloader's end.

### Post by tonyr on 2006-07-03
[QUOTE=arthurbarnhouse]How do I change a "Makefile?"[/QUOTE]

Weeellllll.....
You have to edit it.  It's one of the files in the **bulldozer** directory.  You have
to be familiar with an editor.  The standard one for Ubuntu is **gedit**,
but there are many others. Then you have to know all about makefile syntax,
and shell syntax (it looks like the expected shell is **sh** or **bash**.
Then you have to figure out how exactly the author went about putting
this **Makefile** together with his/her particular interpretaion of 'how it spoze to be'.

This endeavor is not to be entered into lightly.
