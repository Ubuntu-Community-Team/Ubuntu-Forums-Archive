### Post by bhultin on 2008-10-17
With KDE3.5.8, and Dolphin 0.9.2, you can delete files that are read-only!!!  This same behavior happens in konqueror too!!

How can this be possible??  "ls -al" and permissions window in Dolphin both show r--r--r-- on a file.  From command line "rm file" is rejected because the file is read-only.  However in KDE Dolphin (and Konqueror), the file can be deleted with no problems!  This seems like a serious bug.
