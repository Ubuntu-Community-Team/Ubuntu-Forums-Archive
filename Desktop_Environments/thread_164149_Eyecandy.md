### Post by GeneralZod on 2006-04-22

[/QUOTE]

To expand on this: xcompmgr was written mainly as a proof-of-concept not really intended for "mainstream" use, and then pretty much abandoned.  KDE's compositor (or should that be Kompositor? :)) is intended to be a "proper" way of using the Composite extension intended for widespread use, rather than a throwaway tech demo.
