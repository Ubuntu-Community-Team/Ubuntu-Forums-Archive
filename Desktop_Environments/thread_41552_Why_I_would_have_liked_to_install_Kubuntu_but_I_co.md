### Post by jyp on 2005-06-13
Being a newbie who had ventured to install Debian on 2 machines, I came to Kubuntu (and Ubuntu) mainly because it was Debian based and also because the release cycle made sense. Also there was the lure of an easier distro where multimedia functionalities would be easier to get

I run an older machine (I686) with 512M of ram, 2 HD, cdrom and burner. The installation of Kubuntu was very simple even using the expert mode (doing the expert install gives you a root account which is a nice thing to have and does not ask that many questions either). But since Kubuntu is quite a new project, in spite of the fabulous work done on it, it seems to me a little bit skinny and most of all, it has a nasty tendency to freeze. I tried many solutions with not much success. So I installed Ubuntu and apt-get kubuntu-desktop over it.  Now it seems to be pretty stable.

The story is not at all the same on the newer machine (MSI k8n neo2, amd64, 2GB of memory, Raptor 36G, 2 Caviar 200G, cdrom and Pioneer DVR) . Kubuntu (amd64) would install easily but kde would freeze almost instantly and there was no way to have the system detect my burner (Pioneer-108) ; I had to go the hardware way and connect both the cdrom and the burner as masters, each on its own ide. 

I  tried all the combinations with not much success : Kubuntu (amd64), instant freezing; Ubuntu(amd64) + kubuntu-desktop, random but frequent freezing ; Ubuntu(i386) + kubuntu-desktop and the freezing kept on.

BTW, I gathered as much information as I could on forums, but no way. 
Too bad because I like Kde a lot ; now I try to live with Ubuntu on my amd64, which for the moment seems pretty stable. :???: 

So for now, I do not see how I could run Kubuntu on my amd64 machine; I wait for the Debian amd64 cd images ... they are not there yet!
