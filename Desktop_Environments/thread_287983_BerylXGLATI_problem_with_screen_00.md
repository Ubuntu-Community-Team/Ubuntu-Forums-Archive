### Post by synthaxx on 2007-01-11
Same "problem" here, but i think i figured out whats causing it.
If you followed the guide that makes you add an Xgl option to your startup managers, you're starting a "display 1:0" in the startxgl script.
But! Your desktop is still run on ":0.0", so if you run "glxinfo -display :0.0" you get the message saying your direct rendering is enabled.

Now maybe, if you change the startxgl script to run on display 0.0, the programs that rely on this would work?
I can't really try this, since it makes my dual monitor setup go ape.
