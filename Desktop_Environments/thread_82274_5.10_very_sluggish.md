I think Linux is only good with resource usage if you set it up to be.  If you want your computer to do the same things that an installation of XP will do, then the resource usage is usually pretty comparable.  At least that's what I've found.  But with Linux you have other options.  New versions of "full service" desktops like Gnome and KDE are very resource intensive (just like new versions of Windows), but you can always install older versions of them, or try desktops like Xfce or Fluxbox, etc.  They don't include the eye candy, or some of the integration features, but they usually get the job done.

### Post by skylark on 2005-11-02
Even if you have used automatrix try testing the speed of your harddrives:

sudo hdparm -t /dev/hda1
(it might be sudo hdparm -t /dev/sda1 if you use sata harddrives)

My relatively modern sata drives return a figure of approx 56 MB/sec. Anywhere >25 MB/sec should be okay I think. If it is under this then you might consider manually tuning your harddrive settings with hdparm to see if you can boost performance.

That alone has boosted my computer's performance incredibly.  If anybody is running an Athlon XP processor, I would highly recommend installing this linux kernel image.  If you are not using an Athlon processor, you might want to consider the 686 kernel image.
