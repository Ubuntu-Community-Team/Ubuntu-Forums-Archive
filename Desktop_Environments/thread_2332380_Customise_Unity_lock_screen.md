### Post by NZLamb on 2016-07-31
I'm a big fan of the Unity desktop and it's been a big hit in my work environment. One issue is the lock screen in Unity - it displays the current user name which is a no-no in our company security policy. I also had issues with it displaying faint ghosting of the secondary monitor desktop contents when running multiple monitors. It's getting increasingly hard to avoid these issues and we may have to abandon Unity altogether.

I noticed that the 14.04 beta had the option to fall back on LightDM when the screen was locked which is great as it can be easily customised to fit the company security policy, but this appears to have been removed in the later Unity 7.2 versions. 16.04 now uses Unity 7.4 and so this option is long gone.

Is there any way to either customise the Unity lock screen, or better yet still force Unity *not* to use its own lock screen and fall back on LightDM instead these days? If not is there somewhere I can properly request that this feature be added back to Unity?
