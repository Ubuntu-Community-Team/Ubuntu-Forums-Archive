### Post by amsterdamharu on 2010-12-20
The i915 drivers are loaded supporting the video card if the Intel drivers were not loaded.


Trying to find information on the Intel driver package: xserver-xorg-video-intel but it doesn't mention a GM965/GL960 all it says is that it supports i8xx and i9xx cards.

Lets give it a try anyway:
Disable the i915 driver:
Press alt + F2 and type
```
gksu gedit /etc/default/grub
```
There is a line like
```
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"
```
Replace with
```
GRUB_CMDLINE_LINUX_DEFAULT="i915.modeset=0 rdblacklist=i915  blacklist=vga16fb i915.noaccel=1"

```
Now give the following command:
```
sudo update-grub
```

Reboot but don't expect to get into a gui so write this down:

If the boot is finishid and you end up with garbled or black screen just press control + alt + F1 and login to the terminal.
execute the following commands to stop gdm (the gnome login screen) and install Intel drivers:
```
sudo service gdm stop
sudo apt-get install --reinstall xserver-xorg-video-intel
```
You might need a network cable for this because wireless usually is started by network manager after you log in.
When it's done press control + alt + delete to restart. Later you might revert to the old kernel commands:
Press alt + F2 and type
```
GRUB_CMDLINE_LINUX_DEFAULT="i915.modeset=0 rdblacklist=i915  blacklist=vga16fb i915.noaccel=1"
 
```
 There is a line like
```
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"
```
Replace with
```
gksu gedit /etc/default/grub
```
 Now give the following command:
```
sudo update-grub
```
