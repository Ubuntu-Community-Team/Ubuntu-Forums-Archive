### Post by mannheim on 2006-04-23
There are "howto" threads on the forum for using either kwin or xfwm4 with gnome:

[http://ubuntuforums.org/showthread.php?t=115974]("http://ubuntuforums.org/showthread.php?t=115974")
[http://ubuntuforums.org/showthread.php?t=102875&]("http://ubuntuforums.org/showthread.php?t=102875&") 

You will need suitable graphics hardware and drivers to get transparency and drop-shadows working with either of these. Wait and upgrade to the "dapper" release if you haven't already, because these things seem to have improved quite a bit.

I use kwin for various reasons. It is very configurable, gives transparency and shadows without problems, and looks good with the "Plastik" window-border theme. kwin allows you to set up your own color schemes; so I made a color scheme for Plastik that looks right (to my eyes) with the new orange theme in ubuntu/dapper.

The xfwm4 howto is worth a look even if you want to use kwin, because you can use the installation tips there. That howto explains how to edit gnome-wm to enable the use of xfwm4 as an option. The corresponding steps for kwin are:

0. Follow the above kwin howto, for all the xorg stuff and installing the packages for kwin, kcontrol, kdeartwork-theme-window. 

1. Edit /usr/bin/gnome-wm, either in place, or by creating a local version in /usr/local/bin/gnome-wm. You need to do add the lines

```
    kwin)
      OPT1=--session
      OPT2=$SMID
      ;;

```

before the line that begins "##FIXME" at the end of the "case" statement.

2. Edit or create a file ~/.gnomerc and add a line
```
export WINDOW_MANAGER=/usr/bin/kwin

```
to tell gnome that you want to use kwin. Then log out and log back in. If it doesn't work, log back in again with the "Failsafe Gnome" option, and edit out the line you added to .gnomerc.

### Post by mannheim on 2006-04-24
You need to add the lines:

```

Section "Extensions"
       Option  "Composite" "Enable"
EndSection
```

at the end of your xorg.conf.
