### Post by mcduck on 2010-11-10
yes, it is just a part of the file name and not actually required.

Like I said, it's hidden, not because of any special meaning to the OS itself, but just so that the entries in the menu can be more clear and explain better what each template is.

Besides, at least I have many different templates that use same file format (and thus have same file name extension) so separating them by extension only wouldn't really work at all.

Actually I'd say using a descriptive file name instead of relying on file name extension is *very much* Linux-like. :)

What comes to spaces in  file names, that's really simple even with scripting, just remember to surround such file names (or variables used for file names) with apostrophes. Besides, the name used for the template is intended to be a descriptive name *for the template*, of course you'll name each file you create based on that template yourself so I don't see what the template naming has to do with scripts or the web. (I bet you aren't now trying to have several "empty.html"s but instead give the files appropriate names afterwards, same thing with descriptive template names.) :)
