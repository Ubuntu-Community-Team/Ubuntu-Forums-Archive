### Post by Zorael on 2010-01-12
It's a panel. :3

The recent rebranding efforts confuse me as well, but basically there's this KDE Plasma Desktop framework. With it, you can create widgets, and panels that also contain widgets. Everything that you think of as "your desktop" (as opposed to an application) is a widget in KDE. I guess the wallpaper is an exception.

The default panel contains an Application Menu (of which there are two kinds; a modern and a classic KDE3-like menu), a Task Manager, a System Tray, a Device Notifier, a Battery Monitor (if laptop) and a Digital Clock. But you can remove it all if you wish, or add a second panel like in GNOME and spread out the widgets as you please.

There's a [vocabulary page](http://techbase.kde.org/Projects/Plasma/Vocabulary) over on the plasma development wiki, but part of it is a bit technical.
> A widget is a single component on the canvas. Other common names that are analogous are "applet" or "gadget". Superkaramba Themes, Apple's Dashboard, Google Gadgets, Yahoo Widgets, Vista Sidebar Widgets, Opera Widgets are all examples of other widget systems (some of which are supported by Plasma as well).
