### Post by abaddon80 on 2006-01-04
My company has started to do more frequent code reviews, and gEdit coupled with gtklp has been nice.  gEdit gets us the syntax highlighting and the line numbers, and gtklp allows us to do a two-up print out (two pages on single side) and double sided printing.

However, it gets a little tedious with large classes with many functions to "navigate" the code.  While going through code, we're often looking for where functions are, losing are place, jumping back and forth, etc.  Personally, I use vim and use folds all the time to navigate large classes.  I thought it would be nice if I could turn on folding, print from vim and have the actual folds print out.  This would be a quick way to get an "index" of functions.  A co-worker, who comes from the Windows dev world, says that Microsoft Visual Studio actually did this nicely.  :-| 

I've found a few other editors (Eclipse, Anjuta, Bluefish, etc), that are also capable of folding functions, but it seems like none of them are capable of actually printing out the file with the folds intact.  Does anyone know of any editors that are capable of doing this?
