### Post by johann_p on 2009-10-21
I am using Ubuntu 9.04 on a Lenovo ThinkPad T500. I am using the GNOME window manager and have buttons for starting Firefox, Eclipse etc. added to the panel.
All updates are installed.

At irregular intervals but fairly frequently (about twice a day) the whole screen locks up when I start an application  by clicking on one of these buttons (does not matter which).

The mouse is still active, but clicking on anything has no effect. The screen is unusable and dead. There is no more way to close windows, start other applications or log out.

The only way to get out of this is by power-of/power on or by Ctrl-Alt-Backspace (which I have enabled). If I terminate the X Server using Ctrl-Alt-Backspace I get back into the login manager and can re-login and everything is fine for while again.
There seems to be no way to make this happen deliberately.

It also seems to be independent of the graphics hardware: I have had this happen with the Intel graphics subsystem enabled in the BIOS, with ATI enabled and either the default or the proprietary driver ... none of this made a difference.
