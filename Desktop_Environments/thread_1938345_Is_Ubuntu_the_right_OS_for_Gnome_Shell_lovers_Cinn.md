### Post by Scott Baker on 2012-03-09
A couple of thoughts here. First, is your Ubuntu installation a fresh install or is it an upgrade from a previous version? With the last couple of releases, upgrading seems to be a bit buggy at times. Next, if this is a fresh install, did you run your update manager right after? A lot of users miss this step, believing that when they marked "add upgrades while installing" that would be done. Actually running update manager will normally show 300 to 400 updates that weren't added during installation. If this is a new install, run your update manager to make sure this wasn't missed. Next, Unity 2D seems to be easier on the machines. I've seen that Unity 3D seem squirrley unless you're running either a dedicated graphics card, or really good on-board graphics. Finally, I know this will sound strange, but I've noticed that when I first started running Unity, it seemed pokey at times, but as I continued to use it that the pokeyness seemed to go away. (as if the software is adapting to my usage habits). Check your updates to make sure they're all installed, then try 2D and see if this is better. :-k

If your gnome-shell environment is crashing frequently then it is likely a rogue extension. Prune your extensions back to find the culprit.

I like gnome-shell, and currently prefer it over Unity, but I personally think it is a poor design that gnome-shell allows (indeed encourages) extensions to "monkey patch" gnome-shell internals. That causes very brittle systems prone to unreliability and crashes.
