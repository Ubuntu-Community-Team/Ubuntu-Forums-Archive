### Post by booljayj on 2007-10-28
I am not going to lie, I am growing more and more angry at this community. If an application works well, then great, but if it can work better, why shouldn't it? So many people switch to other e-mail clients because Evolution doesn't work/look/feel like they want it to. Evolution is:

A) Monolithic. It should be a group of integrated applications instead of one giant beast of a program. Good programmers don't make code that is static; rather, code should be extensible and flexible in order to reflect changes in technology, society, and humanity. I don't ever use notes or tasks, why do I need them taking up valuable space on my hard drive? I usually have Evolution running all the time for my e-mail, so why does the calendar need to be running as well and taking up all that memory? Kontact has the right idea (even if it is ridiculously complicated), and Evolution could benefit greatly from studying how it works and improving on its architecture. Sylpheed Claws is excellently fast, and why shouldn't Evolution be like that? Seriously, how much memory and time does it take to connect to a server and display text?

B) Badly Designed. The program has very little flow, very little pizazz. Evolution could be made to look and work better without using as much space or memory as it currently does. Cairo, yes, that was a good move. I'm sure that the Evolution developers are looking into adding cairo graphics to the other parts, like address book and email, and I fully support that. The overall design needs to be adjusted, though. The calendar redesign I posted above allows much more functionality for less space (but not too much, the opposite is just as bad). By making six rows of boxes in the month view, you get more days visible, but less information for each day. Use a smaller font size, delete one row to make five, and you have much more information available. The right-side box containing tasks and memos could be integrated into the left-side box that shows calendars, using a tabbed interface. Think smart, Evolution. Why not make all of these things fully customizable? And why can't I customize the top toolbar?

Fixing these two things will make this come closer to becoming the #1 preferred PIM software available. Don't you guys want that?
