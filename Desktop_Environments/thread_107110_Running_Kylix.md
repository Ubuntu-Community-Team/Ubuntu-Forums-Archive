### Post by back2ubuntu on 2006-02-22
Hi,

The problem with Kylix3 is that it's no longer developped and only patched.
The problem isn't that of the linux distribution, but of the kernel. Kylix is no longer updated along the linux kernel.

Visit the page [http://groups.google.fr/group/borland.public.kylix.ide/browse_thread/thread/be4693d9eb5c70c8/c239b607f10f2caf?hl=fr#c239b607f10f2caf](http://groups.google.fr/group/borland.public.kylix.ide/browse_thread/thread/be4693d9eb5c70c8/c239b607f10f2caf?hl=fr#c239b607f10f2caf)
 for more info.

I've been using kylix3 for a few years, and now I find it so excrutiatinI to switch back to windows and BuilderC++ for programming.

Anjuta & kdevelop are fine but don't offer the same as kylix or BuilderC++, and anyway it's not possible to import your kylix/Builder interfaces within anjuta/kdevelop.

Unfortunately, there is no solution for the time being.

Just grin and bear, and hope that someday some smart guy will come up with a solution.

cheers,
hk
