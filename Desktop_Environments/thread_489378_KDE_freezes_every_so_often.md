Hi there!

In the run up to KDE4 I have installed the kubuntu-desktop package so that I can experiment with KDE to decide whether I want to switch over completely. I have grown to like KDE a lot, although some things still annoy me.

Anyway, the real issue is that every so often, the environment just freezes. I can still move my mouse around, and after a while (it can last up to about 10 seconds), everything goes back to normal. It seems to happen mostly when I open or close whole windows, and even just tabs in firefox or konsole. I have installed Compiz-Fusion, but this was happening even before I did that.

System Specs:
Ubuntu Feisty (with kubuntu-desktop installed over that)
512mb RAM
nVidia GeForce FX 5200 128mb
Two Intel Pentium processors, both 2.6Ghz
Kernel: 2.6.20-16-lowlatency

Sometimes when the freeze ups occur, I can hear some hard disk activity, and when it happens as I open a menu, it sounds (and looks) as though it is fetching icons. However, it doesn't ALWAYS happen, and most of the time the system behaves itself and works perfectly normally. I don't experience slow-down or jerkiness, as if the system is struggling with the hardware (indeed, using Gnome, everything is smooth and works perfectly well), so I'm just a little puzzles as to what's going on. If the behaviour was consistent, I could at least make an assumption that there was some hardware issue, or something like that, but there just seems to be nothing obvious which is causing the problem. I have even tried disabling the built-in KDE eye-candy (fading menus etc etc), but the freeze-ups still occurred.

Does anybody have any ideas? It's not exactly a serious issue, because nothing crashes, and I can still use everything perfectly well, but it can be a bit annoying!
