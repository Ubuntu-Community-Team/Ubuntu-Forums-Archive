I created two dapper Desktop liveCD's not log ago from the ISO.  I created the two CD's using Dapper to burn the disks.

I have a laptop at work I would like to use with Ubuntu, but I run into errors so I ran the "check CD" option on boot and the computer told me one of the check sums was wrong.  Strange I thought.

I burned the second disk, same result.  I thought I had a problem with the ISO image, but I had used the image many times before with no problems.

I put the first CD into another computer, this time a desktop machine, not a laptop.  And ran the "check cd" option on boot and it said everything was A-OK.

Why would that happen?   Installing on the laptop stalls, I guess because of the checksum error ONLY THE LAPTOP FINDS.

### Post by CatKiller on 2006-09-30
It sounds to me like a problem with the cd drive in the laptop. Since the checksums are correct on a "reliable" drive, both the image you downloaded and the image on the cd are OK. It's either that the drive is failing, or dirty, or the cables aren't connecting properly, or it's that the laptop drive can't read the media that you've burned it onto. Try a different brand of blank cd and see if that helps. If it doesn't, then you'll need to look at cleaning the lens, checking the cables and whatnot.
