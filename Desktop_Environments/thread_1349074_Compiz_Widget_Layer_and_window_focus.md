### Post by pataphysicianu on 2009-12-07
Having a widget layer issue and just want to see if others are having the same issue. 

First the defaults for the widget layer in Karmic are stupid, "end widget mode on click" is selected, as well as "background brightness" 50. So if you normally don't use widget layer and want to  test this you might want to turn off end widget mode on click, at least.

If I turn on widget layer with F9, then select another window, that windows title bar won't change to what it should be if it has focus, other subtle focus issues are their as well, though one can use programs normally, it's hard to tell which one has focus, and trying to us group and tab compiz functions become rather impossible.

Firefox is sometime the exception to this, so test with true gnome applications.

It seems to me like "end widget mode on click" was set as default to cover up this problem. This means that you can only use your widgets for a quick peek, and then let them disappear, which is not the way most people want widgets. We want them on the desktop most of the time and then toggle them off the rare times they are in the way.
