### Post by ecadre on 2005-07-08
OK then, I use KDE and make no apologies for that. I like it and the way it works etc. That doesn't mean that I HAVE to dislike Gnome.

I find the comments that KDE is like Windows XP quite extraordinary.  Would people like to be specific about that because I cannot see it?  Have any of you used XP and KDE side by side for any period of time?

I suspect, however, that this is no more than self-justification parading as prejudice.  It implies that KDE users are not "proper" Linux users because they need to have their hands held by a desktop environment that is the "XP of Linux".  It is demeaning, unnecessary and wrong.

Gnome users need no particular justification for using it, Gnome is obviously a great desktop environment.  They certainly do not need to start misrepresenting the other major desktop environment to explain their decision.

I recognise that I probably use and continue to use KDE because it was the first desktop that I was introduced to on Linux.  As far as I understand, at that time Gnome was not really usable (how things change :))  Perhaps, if I had been introduced to Gnome at that time, I would be a Gnome user.  Who knows.

So, to the new user, what would be my advice?  Start somewhere (Gnome, KDE. whatever) and then experiment if you have the time and inclination.  If you start on Gnome and most of your favourite applications start being from KDE, maybe it's time to try a swap.  The same is true vice-versa.

The sordid little truth is that for most users (ie. those not obsessed with computers and computing), there is very, very little difference between Gnome and KDE.  Sure, if you start off on one then you may find the other a bit odd (different).  For instance, my mother uses a KDE desktop and when I showed her Gnome, she said it was very interesting.  However, she didn't see much point in investigating it further since it did basically the same things as KDE.

Linux developers in different environments learn a lot from one another.  Different approaches problems, ways of doing things, interfaces etc. strengthen the free software movement.  After all we are meant to be allies.

### Post by poptones on 2005-07-08
I used XP at the office because it was forced upon me. I hate it and find it ugly.

*I find the comments that KDE is like Windows XP quite extraordinary. Would people like to be specific about that because I cannot see it?*

Asethetically, KDE has more "edges" that one can only get rid of by way of assigning colors to objects that "match." But this never really works because some of those colors are used for other things, so you may suddenly find youself in an app with text that is "invisible" because the person who wrote the code used the color of the page gutters or something. 

And even if you try the above, you still end up with lots of "edges."

The icons are tiny and bunched together. The default Mandrake desktop is, to me, the pinnacle of this sort of ugliness. It's amazing to me such an ugly desktop should come from the French, a group of people known for their zeal for the arts. 

The worse part though is that all the stuff is wedged together. This part fits that part - it's designed for this, but it doesn't work well because there are no firm guidelines. So one tiny bit of ugliness propogates across the whole desktop. It's like having gtk1 apps on a gnome2 desktop - it sort of fits bit not really.

That damn browser. It's faster at opening very large folders than nautilus, but it never seems to make the correct "guess" as to which "profile" it should be using to display content. One day I realized how many times I was having to go up to that "profile" setting and correct the redraws, and that was when I installed mozilla. But that didn't work quite right either because it doesn't integrate well with kde - so now my desktop was even more mismatched and incoherent.

In spite of the "kparts" bit, the KDE desktop does not have a feeling of "unity" for me. As I said before, it feels like a bunch of boy racer parts cobbled together from a parts catalog. There is no sense that someone designed this stuff to *work* together, but only to allow it to *fit* together. it's like having a dozen different jigsaw puzzles all cut from the same die and a few pieces from each thrown into a box together. All the parts may fit one another, but when it's complete the picture is just bits and pieces of a dozen very different views of the world.
