Hey guys,

I'm setting up some Ubuntu computers for some older friends who have never used Ubuntu or any type if Linux in their life. They mostly have Windows Vista, which is very slow and buggy. One of my major hurdles is getting these completely non-technical people to use Ubuntu like they were able to use Vista. If they hate it, they're going to probably hate me too!

I just did a test with my Grandma with 12.04 with Unity, and I already struck some issues:

**1.**  Window action buttons(minimise, maximise, close) problems:

[LIST]

If you want to make something easy for (former) Windows users, regular Gnome is probably preferable to Unity*.  But it's unfortunate that the Windows GUI is so bad because many people think "this is how computers work" because they've never seen anything better. "Windows" is mostly a platform for computer-illiterate people to run MS  Office (IIRC, Office is where MS makes their money, not from the Windows  OS itself).

I came to Windows from Unix/Motif and have always considered windows to be a crude imitation of that, and their new GUIs are just minor updates to Windows 95.  They don't even supply multiple/virtual desktops! (I use blackbox or EmergeDesktop on Windows and don't see their goofy "Start" button anymore).

Over the years I've become less enamored of GUIs to get to and run software unless you're only using a few programs. One thing MS finally did right was putting a search function on the "Control Panel"; type a keyword like "disk" and it provides a list of related functions you can select. Things like "Launchy" are similar but don't work well in Linux because the applications usually have so-cute but REALLY STUPID names that have NO relation to their function: you don't get a list of DVD-related programs by searching for executables with "dvd" in the name.  That's a fundamental linux problem that very unfortunately probably won't ever be fixed.  I use fluxbox with 9 common programs launched by pressing buttons on the numerical keypad, and others launched with fbrun or a terminal; I hardly ever use menus or desktop icons.

*Anyway, enough ranting, you might look at the "Cinnamon" desktop mgr.  It's like "classic" gnome but looks nicer and seems to work better on 12.04 - the desktop appearance settings actually worked! It's from Mint.

### Post by chocklet on 2012-05-23
A similar decision between Classic and Unity was made here, but for my wife.  And yep, it's not my/your priorities that count, but hers.

The final decision was the result of a seemingly insignificant factor... Firefox bookmarks menu functionality.  Firefox bookmarks menu operates slightly different in Unity.  In Classic, the response to a right click on a bookmark is a menu of 11 options ranging from "open" to "properties".  In Unity, the response to a right click is not a menu, it opens the bookmarked url in the current tab - drives her crazy.

Similar to you, Smokin Whale, my concern with Classic is that "...future major updates may break things."  In particular, there is a warning in the Software Center description about gnome-panel... "Canonical does not provide updates for launcher and docking facility for GNOME."

However, the immediate need is to minimize frustration today.

Small things sometimes drive big decisions.  The final  functionality that my wife needed to sell her on Ubuntu was the ability  of a user-friendly photo editor to reset exif rotation data when  rotating a photo... gThumb's "Reset EXIF orientation" did it.  That  little piece of functionality was huge.
