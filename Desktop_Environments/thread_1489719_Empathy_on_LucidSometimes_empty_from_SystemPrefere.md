### Post by T.E.N. on 2010-05-21
According to [http://ubuntuforums.org/showthread.php?t=1451142#4](http://ubuntuforums.org/showthread.php?t=1451142#4)
and [http://ubuntuforums.org/showthread.php?t=1403848#3](http://ubuntuforums.org/showthread.php?t=1403848#3) Empathy is supposed to autostart with the Contact List always open if entered as simply "empathy" (without the quotes) into System/Preferences/Startup Applications, or with the Contact List always minimized if additionally " -h" is appended.

However, the above is not the behavior I see:
[list][*]On some reboots, the Contact List will display in spite of -h having been specified
[*]On yet some others, the Contact List (and rooms it tries to join) will be empty (in spite of View/Offline Contacts being ticked).
[*]The lack of "last seen" information (or "last active" in IRC after *apt-get install telepathy-gnome*, which still does not allow to specify IRC accounts when first started) significantly reduces usability (cf. [http://ubuntuforums.org/showthread.php?t=1268948&page=3#30](http://ubuntuforums.org/showthread.php?t=1268948&page=3#30)), if not to say makes its users look stupid to the greater Net, as they'll have to "blindly" write to others e.g. every Pidgin users knows have been away/inactive/offline for many hours.
[*]Auto-connect will only work if the key-ring has an unsafe blank password: [http://ubuntuforums.org/showthread.php?t=929566#4](http://ubuntuforums.org/showthread.php?t=929566#4)[/list]These issues need to be addressed if Empathy is to adequately replace other Instant Messaging clients.
