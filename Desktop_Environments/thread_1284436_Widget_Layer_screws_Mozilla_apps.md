### Post by nikolardo on 2009-10-06
Hello there.
I'm running Jaunty and have been enjoying it for quite some time.
I recently got gnome-do and my widget layer up and running well enough for me to get rid of all my docks.  I run stalonetray as a widget so it shows up there as well.
In my widget layer, I have stalonetray, fuzzy clock, the conky-based system monitor, two graphing CPU system monitors (one for each cpu), and a folder view widget.
My problem is this:  with the widget layer enabled, mozilla applications (for me, firefox and songbird) behave in a rather poor graphical manner:  clicking "file" or other menu buttons cause the window to redraw itself fairly slowly and choppily, meanwhile showing me my desktop background.  Also, when switching desktops or having a plasma notification appear over either of these windows, they have this same enormous flicker.  Sometimes it takes a number of seconds for the window to redraw itself.
If I turn the widget layer off, this problem goes away, but I also like my screenlets.  I could live without the folder view, the CPU monitors, and maybe the fuzzy clock, but I really need the system tray (as I have no docks and don't want them) and the conky-based system monitor is nice, but I also really enjoy having them on a separate layer so I don't have any trouble summoning them (as in, I don't have to minimize any programs in order to see them).  If anyone has any suggestions, I'd really like to be able to use both Mozilla applications and screenlets.

thank you in advance.

EDIT
this is not true only for mozilla apps.  some things are fine, such as pidgin and supertux2.  However, Openoffice becomes almost unusable because of all the flickering when, say, going through a font menu.
