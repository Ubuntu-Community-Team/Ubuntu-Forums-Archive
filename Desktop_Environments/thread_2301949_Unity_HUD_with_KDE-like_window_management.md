### Post by knuddeldrache on 2015-11-06
I love Ubuntu **Unity**'s ability to reliably search any menu item in any program and redirect commands into those menu items. Appmenu search is the single most important invention since WIMP itself.


However, i hate the window management options available in Ubuntu/Unity. 
So currently, i am on a KDE desktop environment, because its window behavior lets me do everything i expect from an interruption-free window manager:

[LIST=1]
[*]Clicking inside an inactive window does not raise the inactive window 
[*]Clicking inside an inactive window does not pass click (but focuses) 
[*]Clicking inside an inactive window focuses it (but does not pass click the first time) 
[*]Clicking inside an active window does not raise it (but passes click) 
[*]Clicking the window decorations of any window raises it 
[*]Full focus stealing prevention 
[*]Alt-drag does not raise or focus a window 
[*]Alt-resize does not raise or focus a window 
[*]I can make window stay always on bottom 
[*]I can make special application rules and window rules so that windows are created on bottom when a program starts. 
[/LIST]

Is there ANY way to get both the benefits of menu search **and** window behavior that loves me? And no, appmenu-krunner is not the answer. that one doesn't work and hasn't been updated in 3 years.
