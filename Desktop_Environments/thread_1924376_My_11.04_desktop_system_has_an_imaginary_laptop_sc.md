### Post by bturrie on 2012-02-12
I have a Intel D510MO desktop system running under 11.04 classic. Using grandr I can see that Ubuntu thinks my computer system is a laptop and that my monitor is an external monitor. It thinks the max resolution on my non-existant laptop screen is 1024x768. 

The only way I've found so far to get native resolution on my actual monitor (VGA1 in grandr) is to go open the Multiple Screens tool under System and disable the fictious laptop monitor LVDS1.

This works okay except that Firefox still seems to be looking to the non-existant LVDS1 for screen resolution. When I try to display sites they seem to be limiting the width to 1024. I end up with a tiny site display in firefox with large white bars on both sides. I can make the font bigger or zoom in but that doesn't work so well because the lines begin to overlap each other pretty quickly. That takes away a big part of the advantage for using my new 23 inch monitor.

So is there a better workaround to get rid of the "laptop monitor". Alternatively is there a way to make firefox use my actual screen resolution?
