### Post by lhtown on 2006-10-03
I finally found dvd-slideshow. I recorded the voice over and mixed the audio with audacity. I used Inkscape for a single text slide and Gimp for some very minor image manipulation. 

dvd-slideshow is a command line program, but don't let that scare you unless you are a newbie to the command line in which case, you can still do it, but it will take some time and patience. The documentation is pretty good. 

I spent a week or so working with this program and did a really cool semi-professional slide show for a presentation for my non-profit organization. Saving it to a DVD for playback was a breeze.

It would be nice to have a good gui, but really, it was easy to use and pretty powerful. I used the kenburns effect as well as the crossfade transitions. It does take a long time to compile the presentation. My presentation was about 110 pictures and about 10 minutes long with three or four kenburns and a number of crossfades. Also, it had a single audio track through practically the whole thing. (I had already mixed the audio in audacity.) It took about 10 minutes to compile a version suitable for debugging and maybe about 30 minutes for the final version on my dual Athlon 2g processor.

Another note, I have a dual processor and tried to enable the option to allow it to use both processors. It almost worked, but kept giving me errors towards the end (no, it didn't appear to be a memory problem). When I did it normally, it was a bit slower, but finished with no errors.

All in all, this is a very promising program that is well suited to developing semi-professional stuff. The dual processor thing was the only bug I encountered, and the author as much as said it was a somewhat crude implementation that should generally be avoided.

I look forward to using this program in the future and really, don't much care if it has a gui or not.
