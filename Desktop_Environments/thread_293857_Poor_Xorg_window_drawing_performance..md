### Post by Xzyx987X on 2006-11-05
I've had no trouble getting Xorg set up to use OpenGL composite managers, but for some reason getting simple hardware accelerated 2D window drawing set up doesn't seem to be possible. Whenever a window on my desktop has to be drawn (i.e. when overlapped by another window), it takes at least 100ms for the contents to be updated, even for windows with few elements. With windows that have a large amount of elements, it can take over a second for complete updating. This is on an AMD Athlon64 4000+ with 1GB of RAM and a Geforce 6800 GPU. I'd hate to think what my performance would be like on a slower computer. With Windows XP on the other hand, window redrawing happens instantly in all but the most extreme cases.

I'm sure the fault lies somewhere with Xorg, since it's CPU usage shoots up to 100% as soon as window drawing starts. But, I haven't been able to find any obvious problems with my configuration. I really need to get this fixed though, because unless I use a composite window manager the redrawing that occurs when windows overlap is so annoying that it seriously inteferes with my ability to operate. And I can't use a composite manager for regular use due to lack of support for seporated desktops in KDE and choppy video playback performance. I don't want to have to switch back to Windows XP, but if I can't figure out what's wrong here it may be my only choice...
