Here's the deal. A window manager does just that: it manages your windows by putting frames on them and letting you move 'em, maximise 'em, iconify 'em, close 'em, and shade 'em. A desktop environment, like GNOME, *includes* a window manager but also includes lots of other things as well, like panels.

You can install window managers with apt-get install, and then log out, pick the new window manager from GDM's "Session" menu, and log in again. GDM will ask you if you want to use the new window manager just once or make it your default session. If you don't like it, you can always go back to GNOME.

If you use Openbox by itself, all your apps will still be available, though you might have to edit Openbox's menu. You will also have to set the GTK theme manually; you can run gnome-theme-manager from a term to do that. Lastly, if you don't want OpenOffice.org to look like shit, make sure your /etc/profile has the line **export OOO_FORCE_DESKTOP="gnome"** at the end. Also, you'll have to configure Nautilus to refrain from taking over the desktop; use gconf-editor for that. You'll have to set the background manually as well; **feh** is a good tool for setting backgrounds if you don't want to get stuck with Eterm along with Esetroot.

To replace metacity, just hit ALT+F2 to get a prompt, and type "openbox --replace". I recommend installing obconf; it offers a GUI for most of Openbox's settings.

### Post by speedman on 2005-04-15
Options presented in the GDM sessions menu (other than failsafe and default) are parsed from the /usr/share/xsessions directory in Ubuntu.  To remove an entry from the GDM sessions menu just rm the corresponding file from this directory.  The gnome.desktop file that resides there by default will give you an indication of the syntax for creating a new entry from scratch.

Speedman answered the first query nicely!!
As for whether or not to replace openbox as the window manager for GNOME, I do not see any advantage to that because metacity is perfectly integrated by the GNOME folks and especially the work of the Ubntu developers that brought us the distro. And you will loose some of the GNOME gui config options.

That said, The advantage to Openbox running all by its lonesome is speed and light--- very light-- load on the system. This is especially handy when one wants to have many editing applications running at the same time plus a few file managers plus several terminals while listening to music. :) If unnecessary or never used GNOME services are dormant, then this makes for speed and response in editors that get sluggish when editing big blocks of text, code, or mathematical equations.

Also, another reason to have a super light window manager installed is in case of general GNOME or KDE system freezes or complete crashes that carry over to the next login. OpenBox will always start up, and give you the chance to correct whatever needs configuration from within openbox...

OpenBox is good to have as an option even if one does not normally use it.

SessionDesktopDir=/etc/X11/sessions/:/etc/dm/Sessions/:/usr/share/gdm/BuiltInSessions/:/usr/share/xsessions/
