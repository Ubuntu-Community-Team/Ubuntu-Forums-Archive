Hi all, I understand this can seem a very silly even philosophical question, please be tolerant :p

    I'm total new to gnome, and it puzzles me a lot that most of preferences/options windows lack a "cancel" button. I mean: if a user plays with the settings and then, unsatisfied, she wants to exit without saving them and keeping previous ones instead, there seem to be no way to do it. Even closing by the "x" seems to save the settings.

I'm really curious: is this intended by design for some reason? If it's the case, is it a gnome design choice or an ubuntu design choice? Is there, I don't know, a special shortcut to exit without saving?

### Post by Hanj on 2005-12-29
Yes, it is a Gnome thing. The part about preference windows in the [Gnome Human Interface Guidelines]("http://developer.gnome.org/projects/gup/hig/2.0/") states that threre should only be a close button in preference windows. I guess there's some thought behind this, but I agree that it's quite confusing not to be able to click cancel.
