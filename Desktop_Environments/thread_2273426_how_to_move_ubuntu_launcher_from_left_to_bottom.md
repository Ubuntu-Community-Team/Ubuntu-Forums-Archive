### Post by buzzingrobot on 2015-04-13
The strip along the bottom in Windows and old Gnome 2-based Ubuntu is a 'panel', which behaves differently than the Unity Launcher, which is essentially a dock. The primary difference is that a panel traditionally allow windows to be 'minimized' to tiny, labeled, rectangles in the panel (it's a bit of an illusion, but so is most everything in a GUI.)

If a user is locked into the Windows-esque requirement to constantly minimize open windows to free up screen space for new windows, and if they can't break the habit of pushing the mouse cursor to the bottom of the screen, then the Launcher or any other dock will likely annoy them.

I find the Launcher, and other docks, very efficient to use in conjunction with  virtual workspaces. I place applications and windows on individual worskpaces and move between open apps/windows via the Launcher. Minimizing is seldom necessary. It's all enhanced when I can use the scroll wheel to move between virtual workspaces.

(The panel you remember was a feature in Gnome 2, which was developed and maintained by the Gnome project, an organization independent of Canonical.  That project abandoned Gnome 2 in 2011. All distributions that had used Gnome 2 were compelled to use an alternative.  Canonical decided to use its own Unity interface in Ubuntu.)

(I recall seeing usability studies that indicate that dock interfaces -- the Launcher, Gnome 3's Dash, OS X's dock, etc. -- are accepted more rapidly by users with no previous Windows experience, while people with Windows experience take longer to use them effectively rather than trying to make them work like Windows.)
