### Post by stefan1975 on 2007-07-06
Hello,

i am looking for a new linux experience with a fast, lightweight desktop manager that is slim but still nice to look at. XFCE with compiz was still a resource hog so now i am trying E17, it is beautiful really fast and still has lots of "bling".

However i am unsure as how to install a "perfect" E17 desktop. I now have two pc's running E17. One is Elive gem 1.0. This is really pretty out of the box, very stable and fast even running from the live cd, with enough applications onboard from the default install and supposedly (since they are at 1.0 stable now) ready for a production desktop. However i have the ata_piix kernel bug on it, the forum is reeaaallly slow, you sort of have to "donate" for it and they say apt-get will not work (all the time) on Elive.

The other setup is ubuntu server 7.04 x64 with gdm, xorg and Ebuntu. it is running quite fast too, it is x64 so that should be even faster, i can use automatix in it, i can use the ubuntu forums, it has a fixed release schedule, it has more modules but it is also more work to make it look and feel the way Elive does. and it might be just the idea but i have the feeling that Elive is a little more snappy even running from cd when it comes to apps, even if it is x86 and a live-cd.

are there any opinions about which is the best way to go, what is Elive like? It does not feel  like a free, community project but the result is great. Ubuntu server does boot very fast (compared to the other family members).

### Post by gspearing on 2007-07-06
I am currently staying with Ubuntu 7.04 with  E17 on top on both my laptop and desktop PCs.

WICD ([http://wicd.sourceforge.net](http://wicd.sourceforge.net)) sorts out the wifi issue and creating the default .gtkrc-2.0 in my home directory controls the font size, theme and icons for the gtk applications.

I have downloaded the Elive ISO and briefly played with it on the laptop. They've done some very nice work with it and it is well worth look. However, for now, I see no need to move from using .deb E17 files from a repo and ubuntu underneath.
