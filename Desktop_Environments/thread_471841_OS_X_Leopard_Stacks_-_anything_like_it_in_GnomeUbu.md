### Post by gfixler on 2007-06-15
I know this doesn't answer your question, but WOW! I didn't know about Drawer. That's awesome :) Thanks for the heads up.

Something like 6 years ago, I had an idea (for Win2k) to make a little 16px tall grabber thing, to which you could drag icons from anywhere, and they'd all be 16x16, so it was tiny, and they'd act like launcher buttons. As you did, it would grow to the right of the grab handle with icons. If you right clicked on the grab handle, you'd get a dropdown of choices, and one would be to add a folder, which when clicked would drop downward into a vertical version of the bar to stack in more icons.

You'd also be able to add folders to that, and so on, making any number of 90° angle dropdowns/drop-rights to categorize things in hierarchical ways. I just tried to add a drawer to the drawer, and it worked, and I could keep doing that recursively. It's basically exactly my idea, except contained on the panel, rather than free-floating (which is better). Hooray.

When I watched the WWDC the other day, the only thing I could think of was how Mac folks have to wait a year or two for Apple to invent these things for them, and hope they get what they want, and take whatever they get, whereas we have coders working daily on fun stuff for us, like the Beryl/Compiz folks.

I figured it would be a few weeks before someone just emulated all of the new things shown into Ubuntu through the power of open source. I wanted to do it, but I don't yet know the coding kung fu. Interestingly, to test out the drawer, I dragged the first thing on my desktop to it - a mov file - and clicked it to test how it launched, and it was the streaming WWDC conference video :)

Oh, and best of luck in finding what you want.

### Post by mkantor1 on 2008-03-26
I've actually been thinking about this a lot recently.  Drawers are only about halfway to what I'd want.  Additionally, I'd like to see:

[LIST]
[*]A more informative display, not just icons.  A common use case for me would be to keep a "working set" of files in a stack and many of these may have the same icon, which makes icons-only not very useful.  Maybe the stacks could look something like the main menu? Having a customizable name and icon for the stack as a whole would be nice too (ie "Downloads" with a down arrow icon).


[*]A folder-based backend (like OSX stacks).  Everything in the stack is just pulled from a folder that contains regular files, launchers, etc.  Dragging and dropping something into a stack would either make a link to the file in the stack's folder or move it there.  Subfolders could even expand when moused over like items in the main menu do.  The location should be customizable.
[/LIST]
I know that AWN's stacks already offer most of this, but it chugs on my system. Also, having stacks in the panel means they'll integrate nicely with GNOME and be always visible, which makes them a lot more appealing.  I could definitely see myself filling up all of that wasted space in my top panel with a bunch of these guys.

Anyways, it seems like something like this would be relatively simple to implement and pretty damn useful.  I don't have any experience coding for GTK, but I might take a crack at this once I graduate college in May and have some more free time.  If anyone would be willing to help me out, let me know.
