Folks,

Each time I login I get a message saying that the permissions for .dmrc need to be 644 and owned by user. 

Have ensured that my home directory is owned by me as well as .dmrc which they are. The permission settings are at 600. I use chmod to set it to 644 but even while I am logged in it gets reset to 600 before I ever log off. Obviously it is being updated and reverted back to 600. 

Any clues how to stop this so it retains the setting of 644?

### Post by edwardTheGreat on 2008-12-15
Have a look at this link... It should provide you with a bit more information.

[http://ubuntuforums.org/showthread.php?t=796628&highlight=.dmrc+644+login]("http://ubuntuforums.org/showthread.php?t=796628&highlight=.dmrc+644+login")

It happened to me last night and I did this via the recover mode and it sorted it out for me([taken from OldSoldier2003's post]("http://ubuntuforums.org/showpost.php?p=4974555&postcount=10")).

```
chmod 700 /home/<yourusername>
chmod 644 /home/<yourusername>/.dmrc
```
