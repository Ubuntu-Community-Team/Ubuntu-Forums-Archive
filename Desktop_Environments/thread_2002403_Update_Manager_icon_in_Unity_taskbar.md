When Update Manager automatically checks for and finds updates, it displays the Update Manger icon in the Unity left taskbar. I have to go into Dash to find & open Update Manager to view and install the updates. Right -click to display Update Manager on the menu also does not work. This is not consistent behavior.

Note that this only occurs when Update Manager runs automatically. If I open Update Manager manually, the icon behaves correctly.

This behavior started in Oneiric and has continued into Precise.

Has anyone else noticed this inconsistent icon behavior?

### Post by efflandt on 2012-06-30
This is apparently NOT a new issue, since I have had this same issue in 64-bit 11.10.

Simple workaround: Go to a different virtual desktop, and then you can open Update Manager from the icon in Unity bar and it should pop in from wherever it is hiding.  Although, I forget if you can just left click it at that point or open it from right click menu (which does not work either when hiding, until switched to different desktop).
