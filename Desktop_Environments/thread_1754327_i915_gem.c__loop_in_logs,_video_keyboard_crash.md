### Post by harrismh777 on 2011-05-10
Looks like this freeze up in the GPU (mouse, keyboard, lights, video all dead) may be related to Intel's DVMT (915 graphics Dynamic Video Memory Technology) found on machines esp with integrated Intel chipsets like the 945, and 915.   The ,915_gem.c  may not be handling DVMT correctly. 

The DVMT can allocate dynamically up to 128M of system memory for video to balance between 2D -3D graphics performance and then return the memory to the system when not needed.   My machine defaults to 8M preallocated, 0M fixed, and 120M DVMT.   

I changed the setting in BIOS for 8M preallocated, 120M fixed, and 0M DVMT and the freeze in the GPU is gone.   I left the silly thing running all night with compiz.real running, as well the cosmos slide show screen saver... ran fine for more than 8 hours... no evil logging, and no freeze.
