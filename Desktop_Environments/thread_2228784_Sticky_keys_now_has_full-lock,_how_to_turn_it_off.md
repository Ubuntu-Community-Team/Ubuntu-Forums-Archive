### Post by avastmc on 2014-06-09
This may be a gnome thing. Sticky keys *used* to operate as follows:
On first press of shift (or other modifier), the next key pressed is shifted, and the lock released.
Pressing shift twice is just like single-pressing that key without sticky keys.

Now this feature operates like windows, except the third part is not optional. That is:
First press: Functions as above
Second press: Locks modifier, every next key is modified and lock doesn't release.
Third press: Functions like pressing shift normally.

This is annoying for a few reasons, one being that I've got short fingers and sometimes accidentally press a modifier. Before, this wasn't such an issue because the next key reset whatever modifier was locked by the sticky keys feature. Now, I have to figure out what is pressed and this wouldn't be so bad except there's no visual feedback as to which modifier key is pressed.

Shift seems to be the main offender for me. I can't describe how annoying it is to click on a link and a new browser window pops up, or the keyboard is behaving funny and I have to figure out which modifier key I accidentally hit, in order to remedy the situation.

This modification may also have to do with [a shift sticky key being stuck on resume]("https://bugs.launchpad.net/ubuntu/+source/unity/+bug/1311344").

I couldn't find any details in configuration editor. I'm sure I'll find time in the next few days to go nag the gnome people and blame them for me typing too fast.
