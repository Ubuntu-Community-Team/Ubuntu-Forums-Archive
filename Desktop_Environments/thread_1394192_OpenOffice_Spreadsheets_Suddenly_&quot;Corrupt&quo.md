### Post by varanasi on 2010-01-30
Anyone else having this problem?

Since the very recent (the morning) samba/fuse update to Karmic, saving OO spreadsheets to a NAS has stopped working.  

I open a spreadsheet from the NAS, work on it, save it, the save seems to work fine, but then there is a long pause.  When I close everything and reopen the spreadsheet, OO says the file is corrupt and asks to repair it.  Re-saving the repaired file to the NAS doesn't solve the problem: opening it again results in another corrupt/repair message/process.

Here's a clumsy workaround. . . . Saving the repaired file locally works and the file can be re-opened without the corruption/repair message/process.  And then, of course, I can copy the file back to the NAS.  Opening the file from the copied file from the NAS works fine.
