### Post by stapler123 on 2007-10-19
Hi, I think I may have had the same problem, but I'm by no means an expert.
It seems that Twinview worked much better for me with compiz than xinerama. The only problem I have with it is that full screen games span both monitors (if you want to watch a 8600gt choke try quakewars at 2560x1024). I am on gutsy now but I did the same thing when using feisty and it worked well then too.

This is the video bits of my xorg.conf as set up by the automatic nvidia-settings program 
```


Section "Module"
    Load           "glx"
EndSection

Section "ServerFlags"
    Option         "Xinerama" "0"
EndSection

Section "Monitor"
    Identifier     "Generic Monitor"
    HorizSync       30.0 - 70.0
    VertRefresh     50.0 - 160.0
    Option         "DPMS"
EndSection

Section "Monitor"
    Identifier     "Monitor0"
    VendorName     "Unknown"
    ModelName      "LG L1952TX"
    HorizSync       30.0 - 71.0
    VertRefresh     56.0 - 75.0
EndSection

Section "Device"
    Identifier     "nVidia Corporation G80 [GeForce 8600 GT]"
    Driver         "nvidia"
    Option         "AddARGBVisuals" "True"
    Option         "AddARGBGLXVisuals" "True"
    Option         "NoLogo" "True"
    BusID          "PCI:1:0:0"
EndSection

Section "Device"
    Identifier     "Videocard0"
    Driver         "nvidia"
    VendorName     "NVIDIA Corporation"
    BoardName      "GeForce 8600 GT"
EndSection

Section "Screen"
    Identifier     "Default Screen"
    Device         "nVidia Corporation G80 [GeForce 8600 GT]"
    Monitor        "Generic Monitor"
    DefaultDepth    24
EndSection

Section "Screen"
    Identifier     "Screen0"
    Device         "Videocard0"
    Monitor        "Monitor0"
    DefaultDepth    24
    Option         "TwinView" "1"
    Option         "metamodes" "DFP-0: nvidia-auto-select +0+0, DFP-1: 1280x720 +1280+0"
EndSection
```
It's that twinview 1 stuff that seems to be different than yours, also dfp-1 is ofset 1280 pixels to the right of dfp-0 where yours seem to start on the same bit of the desktop. Perhaps thats where your black screen with a x for a cursor problem is popping up, though it's just a guess, I'm clueless.
