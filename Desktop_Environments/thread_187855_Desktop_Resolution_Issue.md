### Post by eecscommando on 2006-06-03
Hello, I recently installed 6.06 (clean, not upgrade) on a PC with a NVidia Geforce 6800 GT. In the past, this PC was hooked up to a CRT monitor and was running 5.10 and was working with the expected resolution options.

Now, it's hooked up to a 19 inch LCD (native rez 1280x1024) using a non-DVI cable. If it matters, the monitor is a Samsung SyncMaster 191T. My Windows PC is hooked up to the same monitor using the DVI cable, this monitor has both types.

After installation, Ubuntu correctly detected my videocard and setup xorg.conf to use the NVidia driver. For some reason though I couldn't raise the resolution of the desktop above 1024x768 and the left hand side of the desktop - a 15 pixel column or so - was cutoff (not just failed to be centered, literally not rendered). Investigating in xorg.conf I noticed there was no "1280x1024" option in any of the "Display" groups. But after adding them, I was still unable to change the desktop to 1280x1024 mode.

What gives? Do I need to use the DVI cable of the monitor to get this working? Any help would be appreciated, thanks.

### Post by tommcd on 2006-06-06
Did you try:
sudo dpkg-reconfigure xserver.xorg
either boot to recovery cosole and run that or stop X first:
sudo /etc/init.d/gdm stop
At the end of reconfiguring Xorg you can set resolutions and refresh rates. I had to do this cause I had the same problem. You will need to reinstall your nvidia drivers after this.
To restart X: sudo /etc/init.d/gdm start

There may be an easier or better way than reconfiguring X, but this should work.
