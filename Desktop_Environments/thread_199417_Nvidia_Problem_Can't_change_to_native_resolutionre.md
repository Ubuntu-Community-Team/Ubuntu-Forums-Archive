### Post by mlind on 2006-06-18
is this section correct? Especially HorizSync and VertRefresh
```

Section "Monitor"
	Identifier	"Rosewill R912E"
	Option		"DPMS"
	[B]HorizSync	30-83
	VertRefresh	55-75[/B]
EndSection

```

You could also try these on your Device section
```

Option "IgnoreEDID" "1"
Option "UseEdidFreqs" "0"

```

For requesting 60Hz
```

SubSection "Display"
	Depth		24
	Modes		**"1280x1024_60"** "1280x1024" "1024x768" "800x600" "640x480"
EndSubSection

```

### Post by TheEvilHammer on 2006-06-19
I think it's a problem with the "nvidia" driver; I tried using the regular "nv" driver, and it detected my monitor correctly with the entire gamut of resolutions and refresh rates.  However, I was able to fix the problem with the "nvidia" driver rather easily: I just edited my xorg.conf file and manually set the vertical refresh range to 60-60.  Once I did that, I had access to 1280x1024.  Honestly, this is fine by me, since LCD's are supposed to be run at 60 Hz anyway...

Also, I browsed that Nvidia page briefly, and it says that "CRT" denotes any monitor that is connected by a 15-pin VGA port, whether it's really a CRT or not.  "DFP" is actually reserved for monitors that are connected by DVI.  So, it looks like "CRT" is correct after all...

### Post by fragos on 2006-06-29
I may be mistaken but it appears to me that your nvidia chip set is on the mother board and not a separate card.  When embedded on the mobo, the BIOS determines how much of your system memory is allocated to video.  You can set the BIOS to use more memory and that might enable you to use 1280x1024.  Separate video cards have their own memory.
