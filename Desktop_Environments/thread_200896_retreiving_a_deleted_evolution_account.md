### Post by aysiu on 2006-06-20
A lot depends on how you check your emails and what settings you have.

POP3 retrieves copies of your email messages and puts them on your hard drive. You have the option, as ubman pointed out, to leave a copy on the server as well, but I think you have to enable that option.

IMAP just takes little snapshots of what's going on at the server end, so that if you see a message, it's only temporarily downloaded to your hard drive, but if you move or delete a message, it's moved or deleted on the server end.

Now, if you're using IMAP, you have nothing to worry about. "Deleting" an account that's IMAP simply means removing it from Evolution. The account still exists, and so do the emails.

If you're using POP3 and did not opt to leave copies of messages on the server but then deleted the account... well, those messages may be lost forever.

### Post by aysiu on 2006-07-05
Unfortunately, there is no help. When something is deleted, it's gone.

This is the main reason I use IMAP, as I'm far more likely to accidentally delete my email than my mail host server is. Even though I generally trust the servers that host my email accounts, I also make regular local backups of all my messages, too... just in case.

I've lost many an email message over the last eleven years, and I'm sad about it, but you live and learn. Use IMAP whenever possible. If you must use POP3, leave a copy on the server.
