### Post by elreteipos on 2007-04-05
Nobody on the chat was able to help me out, but I solved it by myself. Here's what you should try.

[list][*]Make sure that you're not using GNOME or any GNOME-enabled applications (GTK).
[*]Open Konqueror or any file browser/terminal that's doesn't use GTK.
[*]Go to your home directory.
[*]Rename the following folders/files to *.filename*.old: .gconf, .gconfd, .gnome, .gnome2, .gnome2_private, .metacity, .nautilus and .gtkrc-1.2-gnome2.
[*]Cross your fingers and start GNOME. If everything goes right, GNOME should recreate the setting files/folders you just renamed.
[*]If that didn't work and you want your old settings back, rename *.filename*.old to *.filename*. You might have to delete the newly created *.filename* before you can rename *.filename*.old back to *.filename*.[/list]

Please let us know whether that worked or not. :)

### Post by elreteipos on 2007-04-12
So you're saying that you cannot log in with your admin account because gnome-settings-daemon crashes every time? In that case, try this:

[list][*]When at the login screen, press CTRL+ALT+F2.
[*]Log in using your admin account.
[*]Type *cd ~*
[*]Type *mv .gconf .gconf.old*
[*]Type *mv .gconfd .gconfd.old*
[*]Type *mv .gnome .gnome.old*
[*]Type *mv .gnome2 .gnome2.old*
[*]Type *mv .gnome2_private .gnome2_private.old*
[*]Type *mv .metacity .metacity.old*
[*]Type *mv .nautilus .nautilus.old*
[*]Type *mv .gtkrc-1.2-gnome2 .gtkrc-1.2-gnome.old*
[*]Switch back to the login screen by pressing CTRL+ALT+F7.
[*]Press CTRL+ALT+BACKSPACE. Don't worry, the login screen will come back in a few seconds.
[*]Log in.[/list]

Did that work?

### Post by ugm6hr on 2008-03-24
> **munceenuts said:**
> Oh, I should've said why. I got the black screen before I tried your method. When I went to log in and edit it, it gave me a black screen.

OK.  Remember to make backups before editing anything in future!  In fact, it might be worth running those 2 backup commands now, before editing again, just in case things don't improve...

But, yes, you can revert from Terminal:

```
sudo vi /etc/X11/Xsession.d/99x11-common_start
```

Delete everything here, and replace with:
```
exec $STARTUP
```

Then:

```
sudo vi /etc/X11/Xsession.d/55gnome-session_gnomerc
```

Remove the 2 extra lines you added.
