### Post by Severun on 2007-12-03
Came back this morning after I left myself logged in last night, and the Xsession had restarted.

It appears that fglrx +Compiz + BigDesktop + Gutsy is a no-go at this point.

To get it to work at all in a stable manner you'll need to run "gnome session safe mode" from the options at the gdm login screen. (This of course disables desktop 3D support)

It's the only thing I've found will work consistently on Gutsy w/ the fglrx driver.  Long and short is it appearts fglrx +Gutsy + BigDesktop is pooched.

I haven't really done much testing w/out BigDesktop.  So no strong conclusions about if it is fglrx w/out BigDesktop works any better.

### Post by Jive Turkey on 2008-01-07
I've been having some similar problems.  I have some advice for you though.

Compositing (compiz) will not work with big desktop if you have a resolution that exceeds 2048 pixels in either horizontal or vertical directions.  There is a dirty fix for this but it creates even more problems.  If you can get your to 2048x768 ie: pairmode: "1024x768+1024x768" for big desktop you should be ok to run compiz in big desktop.

The xorg.conf you posted has duplicate sections for device, screen, and monitor.  This is probably from running "aticonfig --initial --dual-head" In any case the duplicate sections should not be necessary in my experience.
