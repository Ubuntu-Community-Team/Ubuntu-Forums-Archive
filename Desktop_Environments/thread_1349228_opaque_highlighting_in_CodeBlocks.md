### Post by r.stiltskin on 2009-12-08
Recent revisions (nightly builds more recent than the official release) of Code::Blocks IDE include a "Highlight occurrences" capability which highlights all occurrences in the current file of a selected term.  This term can be selected with the mouse or by using the ctrl-F or ctrl-R find/replace functions.

Problem is, in Ubuntu 8.04 the selected term is "highlighted" by an opaque block of color which makes it impossible to see the search term  (i.e. the highlighting is displayed as opaque rather than transparent).  It also happens when using incremental search.  This is happening on all 3 of my Ubuntu 8.04 machines -- 1 of them has an ATI graphics adapter and the other 2 are Nvidia.

I have been told on Code::Blocks forum that this error is not occurring with other versions of Ubuntu, nor with any other distros.   Any ideas as to what might be causing this problem in Hardy?
