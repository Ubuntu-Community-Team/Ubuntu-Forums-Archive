I think I know what he means because I am experimenting with WM's for my own amusement and have an idea I could call 'desktab'

**think of a fusion of tabbed windowing / multi desktops / taskbar**

My idea was, for greater compatability with non-globalmenu apps, to allow maximize to make the client fill the screen, but the taskbar remains, like a form of Tabbed windowing. virtual desktops themselves would just go on the end as new tabs.
(I like focus-follows-mouse so want to keep non-globalmenu, best would be rightclick context menu for everything for me)

initial state:textual launchers on the taskbar. 
click one launch/select existing instance, mac /win7 style. double-click to toggle it's instance fullscreen; in the fullscreen state *there is no titlebar*, the usual GTK menu is on the screentop ready for fits law, **and the taskbar is drawn like a Tab**

midclick or rightclick in the maximized state pulls up a new window **tiled** (i think people are used to mid click for launch new instance)

right most corner shows a single button that says 'new desktop'. Same location as win7's 'show desktop'. click it, you go to a new empty desktop, BUT the previous 'state' is now remembered in a new tab next to it, again drawn as a 'tab'.

Hotkey per tab, visually indicated all the time e.g. f1 f2 f3... for each launcher (and on your laptop screen, those are physically near by :) )  
Desk-'tabs' would count down from the end f12, f11 ,f10...
