### Post by arjaybe on 2005-07-26
"I was talkng about others KDE-centric distributions that simply focus on having the latest KDE."  - bored2k

I have Kubuntu installed.  I'm grooming it as my next "main" OS.  My present default boot is MEPIS, which I recommend for at least a live CD tryout.  My previous working system was Libranet, which you might find interesting.  It would be alright, but I don't think it fits your criterion above.  Another nice KDE implementation is PCLinuxOS.  It's based on RPM, but it's still okay.  SLAX is really nice, but unless they've fixed their funky install routine I'd recommend just looking at the live CD.  If you like to experiment you could try out PC-BSD.  It's a KDE-centric release of FreeBSD.  It has a "ports" system which is just as powerful as APT.  But you need to be prepared to put in a bit of work learning the ropes.  And the people involved seem to be hung up on comparing BSD to "Linux".

As far as the support communities for each distro, I haven't been very involved in all of them.  The exception is the MEPIS community and I can say that I enjoy it just as much as I enjoy the Ubuntu one.

Good hunting.-)
