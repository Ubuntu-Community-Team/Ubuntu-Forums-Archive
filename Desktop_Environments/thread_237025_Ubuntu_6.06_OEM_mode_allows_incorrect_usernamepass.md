### Post by TwistedLincoln on 2006-08-15
It would seem that the OEM mode in Ubuntu 6.06 allows the user to choose a username and password using upper-case letters.  It also allows you not to have a password.  In either event, the user is unable to login...ever.  In fact, it would seem that no new user is created.  But the OEM user is removed.

The only workaround I've found is to boot into single-user mode and fix things via the command line. 

Does anyone have a fix for this?  As it is, the OEM mode is basically useless, because most users who are used to Windows will capitalize their usernames, and/or disregard the password part.  Any suggestions would be greatly appreciated.
