### Post by glabouni on 2007-02-01
For those of you with beryl difficulties, beryl installation problems or any other beryl related question, before asking any question, be sure to read [How To Ask Questions The Smart Way]("http://www.catb.org/~esr/faqs/smart-questions.html") first.

would you post a question about your car on a cooking forum ?
so why so do you post beryl related question in a ubuntu related forum ?
these post that belongs more to [beryl forums]("http://forum.beryl-project.org/"), than to ubuntu forums. 

by combining these two advices chances are you'll find help. but do keep in mind that beryl is not to put in all hands. if you're new to linux, you should definitely start by learning the basics of the OS and xserver. save the beryl part for later.

> If that doesn't work, there's a problem somewhere! Hopefully, useful debugging output will be in the terminal session; Ubuntu users can ask for help at #ubuntu-xgl, or the stock #xgl channel on irc://irc.freenode.net/ or search the [beryl forum]("http://forum.beryl-project.org/"); others have probably had the same problem.


if you end up with xorg not starting because you have set beryl-manager to start automatically at session startup, here's a useful tip:
> **Ramses de Norre said:**
> I think the fastest way to stop beryl from starting is aliasing beryl-manager to ```
beryl-manager --no-force-window-manager --no-force-decorator
``` which would make that although beryl-manager is started it wont load his window manager nor emerald.)

if you had a beryl problem ubuntu related that you haved fixed, you're very welcome to post the problem how you fiwed it in this thread. having a small knowledge base available wouldn't hurt.
