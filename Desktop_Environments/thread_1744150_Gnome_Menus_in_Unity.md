**Flat menu structures.** I have to agree with all the users here that have complained about this. just add "cardapio hidden" to startup & "cardapio" is the command for the menu.

For those that want a more 'traditional' menu available in  unity it seems to work as expected
Can be added to the launcher and works fine from there
the ppa which includes a natty package
[https://launchpad.net/~cardapio-team/+archive/unstable](https://launchpad.net/~cardapio-team/+archive/unstable)

For the launcher just used a simple .desktop like this, icon and launcher display name can be whatever desired
```
[Desktop Entry]
Version=1.0
Type=Application
Terminal=false
Exec=cardapio 
Name=menu
Icon=gnome-main-menu

```
I place created .desktops in ~/.local/share/applications, though where doesn't particularly matter, just place before adding to the launcher

For unity don't think you need the hidden start up command - there is no panel for it to start up in 

(while I like icon quicklist menus because they're a bit faster and more specific, cardapio is very complete, familiar and doesn't appear to suffer from the very small per use memory leak that can occur with multiple quicklist icons, or if it does it's an even smaller irrelevant amount if that's a concern
