You're having this problem because you changed from the normal VGA connector on your graphics card to the DVI port... try adding this to your xorg.conf:

in **Section "Device"** of your **videocard**(!)
> Option        "DCCMode"        "on"

this worked for me! (if you changed from DVI to VGA you should maybe try "off" but I think off is the default setting...)

### Post by pactux on 2006-06-21
There are three drivers for ATI videocards.

**xorg-driver-ati:** This one works with all ATI-cards before Radeon X850-series.
(Somebody told it won't work with his X700 either, I don't know it - I don't use it.)

**radeon:** This one builds on the ati-driver and gives 3dacceleration for older videocards before 9250-series. It doesn't give 3dacceleration for newer videocards such as X700.

**xorg-driver-fglrx:** This is the official driver from ATI, which isn't installed by default (!). It will work with all cards above Radeon 8500. (This is the one I use and the one you should get!)

_**How to install?**_

In a terminal window type the following:
> sudo apt-get install xorg-driver-fglrx

The driver should now be installed. Afterwards you have to link it to your videocard by changing the coresponding line in your xorg.conf like this:

**/etc/X11/xorg.conf**
> Section "Device"
	Identifier	"ATI Technologies, Inc. R420 JJ [Radeon X800SE]"
	Driver		"fglrx"
	BusID		"PCI:1:0:0"
EndSection

Just change the Driver (!!!) and leave the other things as they are, don't use my BusID (!!!).

Ok now you should restart your Xserver so that it runs with the new drivers.

Afterwards you can configure Dualscreen with the self-explanatory tool **fglrx-control**. You have to install this (just like you did for the driver using apt-get!).

Hope this will help. Cya!

PS: You can start the fglrx-control by typing **fireglcontrol** in a terminal.
