### Post by jem7v on 2007-01-11
Lots of people will probably suggest KDE (Kubuntu) because it has a reputation for being "more powerful" than GNOME (plain ol' Ubuntu), and there are a lot of coding programs written for the KDE environment... but quite frankly, they'll all run in GNOME too!  They just won't match the color scheme of your other programs.  I prefer GNOME because it's blastedly simple to use and apparently it's less resource intensive than KDE.


As a side note, if you make the move to Linux, try Swiftfox instead of Firefox.  It's a processor-optimized version of Firefox (i.e., it's not a Firefox-derived browser... it Is Firefox.  Only faster.) only available for Linux.  It renders pages significantly quicker than Firefox does, and it's fully compliant with all its extensions (as well as being fully up to date with Firefox).
[http://getswiftfox.com/](http://getswiftfox.com/)
(You'd want the Debian package if you decide to install it.)
