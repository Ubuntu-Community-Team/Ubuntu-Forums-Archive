### Post by supercompman on 2007-10-28
I just installed Ubuntu 7.10 (Gutsy) on both my desktop and laptop over the last few days. Both installs went off without a hitch and the HP Deskjet 932c connected to the desktop was detected perfectly. Using the printer configuration tool I configured the desktop computer to "Share published printers connected to this system" and made sure that the printer had "Shared" checked under the "Policies" tab. On the laptop I used the printer configuration tool to enable "Show printers shared by other systems". Initially this worked exactly as expected, but after rebooting both systems and starting the printer configuration tool on the laptop, I received a CUPS error saying that the printer was no longer available. Since then, regardless of what I do, including rebooting each system, restarting CUPS on each system, removing the printer configuration from the desktop and reinstalling it, and many other things, the printer is not seen by the laptop. I will mention however that if I try to add a printer on the laptop, select "Windows Printer via SAMBA" and brows for the printer, I can see the printer without a problem, but even if I add it through this method, printing is unsuccessful. Any help would be appreciated. Thanks.
