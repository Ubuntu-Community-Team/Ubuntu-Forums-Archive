### Post by Spark* on 2005-04-12
[QUOTE=danip]Someone mentioned there is a browser mode for nautilus?  How do I set that up?[/QUOTE]
Hmm, it shouldn't be hard to find, do we have a discoverability issue?
Anyway, you can open the browser from Applications -> System Tools -> File Browser (drag this item to your panel or desktop for quick access). You can directly open a certain folder in the browser by choosing "Browse Folder" from its context menu. And if you want to make "Browse Folder" the default action for double clicking on folder icons, you can go to the File Management preferences and select "Always open in browser windows" from the Behavior tab.

### Post by Julius on 2005-04-13
More than a "solution" to a "problem", is changing an option, from spatial mode to browser mode. It's not a "problem", nor a mistake. The devs wanted the spatial mode to be default. Browser mode it's still avaiable, and IT'S NOT hard to use it instead of the default. Just look in the preferences thing, it's easy!

I remember having read a very huge article about this spatial mode thing. It looked like gnome devs had really philosophic and spiritual reasons or something to set it as default. 
Personally, I don't thing there's any reason to go back to 1995. Using a different window for each folder sucks, specially when the buttons in the taskbar behaves in a strange way (sometimes a single taskbar item takes 50% of the taskbar), and when you have too many folders. (In windows 95 it probably was "C:\My Documents" to get to the personal folder. But spatial mode with c:\Documents and Settings\username\my documents" sux :P). Browser mode is cool, specially nowadays when almost everybody is familiarized with browsing. Konqueror is cool, it works well as file browser and file viewer, and KHTML is a potent html renderer. Anyway, I still don't see why people would need to use the "back button" while browsing files, or why they'd need to use the "up button" when browsing the web. *Now that's confusing*. Windows/IE do a lil' better job at this, because the file explorer/Interet explorer are quite differenced, while still kinda integrated.  In konqueror you basically don't know where you are :P Anyway, tabs in file browsing  can be VERY helpful :p As for windows file explorer, I like the way it is. While I think nautilus/konqueror offer better 'file preview' features, they're not 'well' used.  I think noone of them are close to the way windows 'lists' the files. I like the way windows list the files and folders.  C'mon, having to leave the pointer over an audio file it's not the best solution, and the icons don't have any fixed size. I think that better file browsers can be done, more 'intelligent' ones, according to the contents of a folder and the amount of files. What's the point of having an icon view when there are 800 files with the same extension in the same folder? Or why aren't folders that only have music show the files with their ID3 tags info? 

^ THat wasn't the point of this topic, anyway :P


For me it's simple: if you don't like it, change it  :P  Most average users probably don't know that they can change that using the gconf editor.. but *most* of them know how to do a "Edit > Preferences" thing :p
