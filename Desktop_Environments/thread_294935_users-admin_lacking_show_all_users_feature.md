### Post by mikelito on 2006-11-07
Please correct me if I'm wrong, but it seems to me that in edgy version of users-admin utility the "show all users" checkbox has gone; how is it then possible to graphically edit the properties of "non-user" users (i.e. users with an userid < 1000)?
they don't get listed anymore....
this is particularly annoying because I have some old users over my network who had userids < 1000, and I have to keep them for compatibility (e.g. NFS shares...)
I understand the GNOME phylosophy to keep the user interface as simple as possible, but this is really taking feature away in newer releases....

### Post by mikelito on 2006-11-08
if someone else is having this problem (better, if someone else feels like this change is interface is a step backwards), a quick-and-dirty workaround is to edit /etc/login.defs file:

sudo vi /etc/login.defs

and change the value of UID_MIN to 1 (or to the lower UID you want to be listed in users-admin)
BEWARE that this will also change the range of UIDs automatically generated by useradd, and giving to normal users UIDs > 1000 is a good policy anyway.
