### Post by ajgreeny on 2014-10-14
If you are happy with the way you do it now using VMs in VBox (as do I with Lubuntu 14.04, Xubuntu 14.04 and Ubuntu 14.04 VMs available), I would stick with it.  Your hardware shown is certainly good enough to run VMs with no discernable lag, assuming you are not doing any graphically challenging activities, and using a VM you will avoid any clutter to your current Ubuntu system that would certainly be dragged in with the addition of the other *ubuntu-desktop packages, giving you a large choice of, for example, text-editor, GUI terminal, etc etc.

I used to always install the vanilla Ubuntu when it used gnome-2 and then add kubuntu-desktop to it, giving me what I thought was the best of both worlds, but it did clutter up the menus of both DEs.  When gnome-3, unity and kde-4 turned up, all of which I disliked, and still do, I realised what I had been missing with Xubuntu, which is now my system of choice, and with a left hand panel full of launchers, a bit like the unity launchbar, but with all the other parts of a standard desktop that unity does not give me, I now believe I have the best system possible for my needs.

### Post by Bucky Ball on 2014-10-14
Don't install *buntu-desktop! That will/can cause conflicts and duplication. If you want a different desktop environment, install JUST the desktop environment, not all the default apps you will drag in with *buntu-desktop. xfce4 (used by xubuntu) and lxde (used my lubuntu) rather than xubuntu-desktop and lubuntu-desktop respectively. With both installed, nightmare, bloatage and possible slower machine. That makes three.

If you install xfce4, for instance, log out, choose it from the 'Sessions' menu, log in.
