### Post by [S|G] on 2006-06-02
Is hda5 a NTFS partition? If it is, you really can't write to it. Write access to NTFS partitions is still experimental and *DANGEROUS*. You can only read from it in its current state. There are ways to write to a NTFS disk, but I wouldn't recommend it, as you might lose all your data.

Edit: If you want to share files between Ubuntu and Windows, you'll probably find it useful to create a FAT partition, as both windows and linux can read and write to it.

Editing again: Mount your NTFS partition with option umask=000. You'll have a line like:
```

/dev/hda5 /home/yourhome/directory ntfs defaults,umask=000 0 0

```

### Post by [S|G] on 2006-06-02
if TEMP directory exists, is it spelled exactly like that? Linux is case-sensitive, and if you created it as Temp or temp then it won't work. Try these (from your home folder):
```

ls -l|grep TEMP

```
If the directory exists it'll appear like this (notice the first character: if its a directory, it'll be a "d"):
```

drwxrwxrwx  15 diego   500     4096 2006-05-16 12:33 public_html

```

If it doesn't have a "d", it is a file and not a directory. If nothing shows up, try this:
```

mkdir ~/TEMP

```
You can also try this to make sure TEMP exists in the proper place.

After using the mkdir command, try sudo mount -a again

### Post by [S|G] on 2006-06-02
What filesystem is sda3 formatted with?

If it is ext3, place this line on your fstab:
```

/dev/sda3 /MountPoint ext3 defaults,atime,auto,rw,dev,exec,suid,nouser 0 2

```

Make sure that /MountPoint exists (change it as you wish). Then go to a terminal and type
```

mount -a

```

I did a quick search for pmount:
pmount: mount removable devices as normal user

It mounts devices that have no matching fstab entry, particularly removable devices.
