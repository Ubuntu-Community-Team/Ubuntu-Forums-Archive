### Post by JAwuku on 2006-08-10
Hi all,

I took a look at the arabeyes.org site for qtquran, and I may have found a possible solution, though I can't guarantee results. This I have run successfully in Gnome Ubuntu, KDE Kubuntu and XFCE Xubuntu.

[http://lists.arabeyes.org/archives/developer/2005/December/msg00012.html](http://lists.arabeyes.org/archives/developer/2005/December/msg00012.html)
[http://lists.arabeyes.org/archives/developer/2005/December/msg00013.html](http://lists.arabeyes.org/archives/developer/2005/December/msg00013.html)

First, I found problems compiling under gcc4.0, so I installed gcc-3.3:

```
sudo apt-get install gcc-3.3 g++-3.3 libogg-dev libspeex-dev libexpat1-dev build-essential autoconf automake1.9 libtool libqt3-mt-dev
```

(you already may have some of these packages installed) 

Download the source .tar.bz2 files for libquran, and qtquran, and extract to separate directories.

then download Quran components (N.B. Large files!)
from [http://www.arabeyes.org/project.php?proj=Quran](http://www.arabeyes.org/project.php?proj=Quran)

1. Quranic Arabeyes font
2. Text Data Components (Quranic Text)
3. Audio Data Components (Audio Recitation)

The audio files are quite huge, you can download the small file of the recitor Mohammed Sayeed Khayat which is 25MB instead of 180MB!

[http://prdownloads.sourceforge.net/arabeyes/mohammed_saeed_khayat.tbz2?download](http://prdownloads.sourceforge.net/arabeyes/mohammed_saeed_khayat.tbz2?download)

```
tar xvzf libquran-0.3.tar.bz2
cd libquran-0.3
```

Because the development version of speex removes le_short, I had to add it to
quranaudio.c (insert just after the #include declarations).

```
unsigned short le_short(unsigned short s)
{
  unsigned short ret=s;
#ifdef WORDS_BIGENDIAN
  ret =  s>>8;
  ret += s<<8;
#endif
  return ret;
}
```

Save quranaudio.c, then go back to the command line.

type:

```
export CC=gcc-3.3
export CXX=g++-3.3
```

This directs the compiler to use gcc3.3 instead of 4.0

**[COLOR="DarkRed"]N.B. you can use gcc/g++ 3.4 instead if you like, just substitute 3.4 wherever you see 3.3 above, and also in the apt-get statement at the beginning. The program works equally whether under 3.3 or 3.4.[/COLOR]**

**_Speex compiling workaround_**

```
CPPFLAGS=-I/usr/include/speex ./configure --prefix=/usr/local
make
sudo make install
```

**_Installing components_**

This libquran has actually installed a program called **quran_install_component**

so go to the directory you downloaded the Quran text and audio components, and type

```
sudo quran_install_component quran_original_text_INCOMPLETE_AND_UNVERIFIED
sudo quran_install_component mohammed_saeed_khayat

```

Install the fonts [http://prdownloads.sourceforge.net/arabeyes/arabeyes-qr.ttf.bz2?download](http://prdownloads.sourceforge.net/arabeyes/arabeyes-qr.ttf.bz2?download)

in the gnome/kde control center.

_**Now for the Qtquran, the frontend for libquran**_

cd to the extracted qtquran-0.1 folder.

generate Makefile:

```
./configure --prefix=/usr/local
```

Now, it gets a bit complex. The Arabeyes website says:

in qtquran-0.1/src/plugins/audio/alsa/Makefile , find the line which says **CPPFLAGS =** and insert **-I/usr/include/speex** in that line, (if it is not already present.)

Do the same for qtquran-0.1/src/plugins/audio/oss/Makefile

then go to the top qtquran-0.1 directory, and type
```
make
sudo make install
```

If everything has gone alright, if you type qtquran, you should see a quran window pop up. Good Luck!
