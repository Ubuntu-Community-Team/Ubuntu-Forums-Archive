This patch will add to kmail the hability to show inline images in html mail instead of showing a blank space and the images as attachments. If the images are remote, kmail works nice, but if the images are sent with the email they are showed as attachments.

I tried this patch (for KDE 3.5) when I compiled the latest stable kdepim today on a Debian sarge (my machine at work) with KDE 3.3 and it worked. On Kubuntu I am using the 3.5 packages from the announcement [http://kubuntu.org/announcements/kde-35.php]("http://kubuntu.org/announcements/kde-35.php")
 but kmail does not have this feature.
