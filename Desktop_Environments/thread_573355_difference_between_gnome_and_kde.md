### Post by eurgain on 2007-10-11
**As to the differences**

Answer 1:

There are several differences:
[LIST]
[*]KDE and Gnome are spelled differently
[*]Most users use Kde XOR Gnome, but some use Gnome OR Kde and others (experts) use Kde AND Gnome
[*]Capitalisation!  Some prefer KDE to Kde or kde, but few like gNOME or GNOME; most like Gnome.
[*]One may sometimes be better than the other (but occasionally the opposite may apply)
[*]Only one of them may be available to users to whom both are not unavailable.  But to users to whom both are available, neither cannot be used.
[*]Sometimes one of them runs an application that is designed only to run on the other of them.  Mostly, this is not the case, because normally neither cannot run an application that is intended (or is not intended) to run on the other.
[*]Some people who always crack their boiled eggs at the pointy end always use Gnome.  Other people (and some or all of the same people) take the same or the opposite view as to the use of KDE.
[/LIST]

Answer 2:

Llareggub  (*per* Thomas)

Answer 3:

There *is no* answer 3.

Answer 4 (the *real* answer)

You may find you like one better than the other.  If so, use that one!  Functionally, they are mostly the same, so choose whichever you prefer, and you will get the best from your computer.  Even better, if you spend a lot of time every day working on your computer, switch between the two for variety.  I earn my living at a Linux workstation, and I switch between Gnome and KDE regularly for the hell of it, and even use WindowMaker, Fluxbox and others from time to time, for a change of scene.

**To switch**

Install the kubuntu-desktop metapackage using Synaptic or:

 $ sudo apt-get install kubuntu-desktop

When you next log in, you can click on Options in the logon screen, select Session, and then choose KDE.

The metapackage installs a basic kde system without lots of things that you would like (which you can install browsing through Synaptic - most of the missing packages begin with 'k') and some you probably don't need or dislike which you may have already scrapped (like the dreadful, imo, avahi).

If you prefer KDE, to be honest, you are probably better served by other distros; Sabayon (if you have a modern computer) or SUSE are more KDE-centric.  Kubuntu always feels to me to be a poor cousin to Ubuntu; it really is not as well developed as Ubuntu.  As a concrete example, the main application menu, is an almost unusable disaster that needs completely re-organised to be useful on a moderately busy workstation.  This is not the case in Ubuntu nor SUSE Kde (but SUSE Gnome is a disaster of an almost equal magnitutde).

Because I like both, and I am geeky enough to do it, my approach is to install Ubuntu, followed by kubuntu-desktop, then a *lot* of tweaking the KDE side to get it as I like it.

**Well???**
If in doubt, try it out!  If it works for you, use it; if not, don't!

GNU/Linux gives you choice in computing! 

HTH
