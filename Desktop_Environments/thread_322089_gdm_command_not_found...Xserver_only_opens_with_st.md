assuming you are using ubuntu and not kubuntu, try this (which most probably isn't the standard thing to do, but I don't know any better and google didn't help):

double-click data.tar.gz
double click etc
double click init.d
move the "gdm" file to your desktop
then 
```
sudo cp ~/Desktop/gdm /etc/init.d/
```
make sure you're not typing anything wrong... 

verify that it's there:
```
ls -l /etc/init.d/gdm
```

log out of gnome
type 
```
sudo /etc.init.d/gdm start
```
did it work? If it didn't, make sure to move back the gdm file to your desktop bf restarting:
sudo mv /etc/init.d/gdm ~/Desktop/

so that you won't get into some loop upon reboot etc

> 
Thank you for your bug. That's not specific to gdm and a feature, files
installed to /etc are usually "conffiles", which means they respect
administrator changes and are not replaced by the package version. You
can try the dpkg --force-confnew option to reinstall it by example

** Changed in: gdm (Ubuntu)
     Assignee: (unassigned) => Ubuntu Desktop Bugs
       Status: Unconfirmed => Rejected
