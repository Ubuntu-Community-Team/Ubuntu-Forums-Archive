### Post by ajroach42 on 2008-12-23
I want to set up a few basic keyboard shortcuts. I am migrating from years on an outdated slackware box running the various *box window managers, and as such I have grown accustomed to some rather esoteric keyboard shortcuts. 

The problem I'm having is that (when using system>preferences>Keyboard shortcuts) when I press the windows key it is used as the primary key for the shortcut rather than as a modifier. 

I am using the standard Ubuntu desktop, (with Compiz, if it makes a difference). 

Is there a way to fix this, or another way to approach this problem?

---

### Post by ajroach42 on 2008-12-23
Allright, i fixed it. 

Under System>preferences>keyboard choose the tab layouts, click other options and choose the option windows key is mapped to meta. 

This fixes the problem.

### Post by ajroach42 on 2009-01-06
The best thing to do is to play with 
System>preferences>keyboard -->layouts-->other options--> alt/win key. 

try mapping just the left win key to meta, failing that, choosing the option win key to super or win key to hyper.

Then try setting the keyboard shortcut to super+R or Super+L (or hyper if that is what you have win mapped to.) 

Look around on the forums, there are various other methods for checking which key is being registered, and changing what key maps where.
