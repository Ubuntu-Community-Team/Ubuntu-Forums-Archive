> To fix the wrong behavior of the OpenOffice with the dark themes, follow the next steps:
1. Start the OpenOffice.org
2. Go to Tool -> Options
3. Then OpenOffice.org -> Appearance
4. For the "Document background" choose - "white"
For the "Text boundaries" - "Gray 80%"
For the "Application background" - "Gray 10%"
And for the "Font color" - "black"

### Post by quinnuendo on 2011-02-06
I have the same problem.

Checking around, I found some other topics, and some links in this topic: [http://ubuntuforums.org/showthread.php?t=1299925](http://ubuntuforums.org/showthread.php?t=1299925)
Unfortunately it seems that eclipse is very bad at customizing the colors. You can export and then import the preferences, but it seems that you can only do that with the full preferences, which include everything not just the colors.

My quick fix was: go to window > preferences menu:
General > Editors > Text Editors (more settings here: [http://pydev.blogspot.com/2009/07/creating-dark-theme-and-exporting-and.html](http://pydev.blogspot.com/2009/07/creating-dark-theme-and-exporting-and.html))
There you can change the basic colors. Next to the color list is a button for selecting the color, and a check box called "system default" (or something like that), just uncheck it for all of the main colors, and you end up with the default white eclipse. That kinda hurts my eyes, so I changed the background color to something more gray.

Hope it helps, and I even more hope they will add more support for themeing eclipse in the future.
