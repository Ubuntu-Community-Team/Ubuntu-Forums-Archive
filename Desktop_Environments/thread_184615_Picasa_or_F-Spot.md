There are several complimentary/competing interests running through my mind when I look at something like Picasa.

Open Standards v. Open Source 
Which guarantees Our Freedom of Choice?


In the end, maintaining open standards is **THE** most important goal. Picasa doesn't add to or change the existing photo standards. Picasa still uses JPEG and RAW  (OK, not sure about the last one) photo files. In my mind, if someone wants to use a proprietary program that maintains existing open standards, fine. Thus, in the interest of "freedom" as in "freedom of choice" I don't mind and I might even use such an application myself. 

I'll admit that I often use Adobe Acrobat. I know, I suck....but people can choose to use Evince, KPDF, etc. because it is an open standard that anyone can choose to implement.

What I DON'T approve of is a program that creates a new "standard" or closed file type that open source software can't manipulate. Microsoft media and office files come to mind immediately. These "standards" remove MY freedom to choose the software I want to use. Picasa doesn't do that. Essentially, I am arguing that Open Standards are even more important than Open Source because Open Source will fail to thrive in an environment dominated by closed standards.

I think we should save the bile and anger for programs that want to establish closed "standards" that limit our choices.......like Windows Media Player, ITunes, M$ Office, etc.

Open standards, like W3C HTML, Open Document, Flash, etc. give developers and users the option of developing open source tools to do the job AND they maintain people's right to pay for software if they want to / can afford to. I think we only get into trouble when there is a standard like mp3 that isn't open and is therefore impossible for OSS to compete on a level playing field.
