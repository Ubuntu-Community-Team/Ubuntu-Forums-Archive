### Post by glave on 2006-12-19
I switched from gnome to kde, but I'm finding one feature of gnome that I sorely miss. In gnome, if I use synaptic or something else that asks for a password to sudo, it remembers the password for a little while or until I go inactive for a bit.  

In KDE, there seems to be no such beast. Everytime I turn around, I'm getting the 'Run as root' dialogue and I have to re-enter my password. Annoying as can be if you're installing a few things in adept, close it, and run it again for some reason a few minutes later.

Is there any way to somehow wallet this, or cache it some way to mimic both terminal and gnome behavior?

### Post by milton1 on 2006-12-19
make your user part of the "sudo" group and "admin" group.
