### Post by |cassidy| on 2005-09-17
Take a look at this
Explains how to install/run ubuntu on a low memory system.
[http://www.ubuntuforums.org/showthread.php?t=42873](http://www.ubuntuforums.org/showthread.php?t=42873)

If your system doesn't work as fast as you want it to after following the above instructions, then you have two options

a) try finding another 128MB of sdram. People give it away these days, since they have it on old computers they're not using anymore. This should be enough to have a functional ubuntu machine (still following the above guide though)

b) you can try Vector Linux
[http://www.vectorlinux.com/](http://www.vectorlinux.com/)

The whole idea it's based on is running on computers with very few resources.
Problem is it's not as user friendly as ubuntu and it may give you a hard time.

Still, better than running windows 95, right?

### Post by az on 2005-09-17
128 megs of ram is just enogh ram.  If you had any less, your perfomrance would be worse, and any more will help you out if you do a lot of things at once.

If you don't have high demands from your computer and do not tend to do a few things at the same time, you should get pretty good performance with that.   The problem with the live version is that much of your ram is used up as a filesystem.  You do not have this problem when you run the real version.

If you still find that slow, yes, using a lighter desktop will help.  You can try XFCE4 or ICEWM.  You can have tham all installed at the same time.
