### Post by Nick Payne on 2011-10-11
I have a two monitors configured as separate X screens so that I can have the right-hand monitor in portrait orientation, and I have Nautilus setup with its left-hand pane showing tree view. If I open Nautilus on the right-hand monitor and then right-click on one of the folders in the Nautilus left-hand pane, the right-click popup menu appears in the middle of the other monitor. 

If I open Nautilus on the left-hand monitor and do the same, the popup menu opens as expected at the mouse position.

If I have the left-hand Nautilus pane at the default setting of Places, then the right-click bug doesn't happen - the popup menu opens at the mouse position. I haven't found the same bug with the right-click menu in any other application that I've run on the right-hand monitor.
