### Post by Nick Payne on 2011-08-26
I have 10.04 amd64 installed on a fairly powerful machine: Intel i7 processor, nVidia 7600 GT dual head video card, 12Gb RAM, everything except /home installed on an SSD. Everything runs satisfactorily fast except for Nautilus. 

When I click on a directory in the Nautilus LH pane, the time taken for the directory contents to appear in the RH pane varies from a couple of seconds if the directory only contains a few dozen files and dirs up to five or six seconds if the directory contains around a thousand items.

I have all previews set to "Never" in the Nautilus file management preferences, so I don't think that's the cause of the problem.

The same directories show their contents much much faster from other applications, such as the file open/save dialogs in Libreoffice and Firefox. Indeed, I can view directory contents much more rapidly with Windows Explorer on a remote Windows machine if I share the dir with Samba.
