### Post by picpak on 2007-03-31
This has been brought up again and again, but it's getting tougher to really choose one. I've gotten tired of the ugly X screen and other quirks that you get when using an autologin with mingetty, so I've switched back to display managers. But here's the thing:

My requirements are simple -- lightweight, fast, and nice-looking. This doesn't mean I want some 1995-era blocky gray interface like xdm or a big bloated one like gdm.

Which brings me to my point: gdm is crap. My system tends to be faster and more stable when I'm not using it. After a few minutes, the whole system just goes slow...and it's because of gdm.

Kdm is faster...but I don't like loading all that KDE cruft.

I don't have a framebuffer, so Qingy is out of the question.

Slim seems to be the best of both worlds, but I can't get it working. It shows up for a few seconds and disappears.

I shared my thoughts on Xdm already.

Wdm - if I could get it to look like Vector Linux's, I'd switch to it in a heartbeat. Thing is, I don't want its default clunky gray interface...so it's also out of the question.

Does anyone have any others they'd like to share? Should I just stick with Mingetty?
