### Post by gatman3 on 2008-11-16
Intrepid 8.10 uses Xorg 7.4, and I don't know that much about 7.4 but it seems to provide much better automated attempts to load drivers and configure devices.  I'm not exactly sure what it will do with no xorg.conf, but I suspect it will create a bare bones version and then attempt to load the open source ati or radeon driver.  That would be the place to start: delete your /etc/X11/xorg.conf and reboot (or maybe just restart X).  Then see what it does.  If X starts OK with the open source driver, then you should be able to go into the System menus (I run KDE and it's Applications->System->Hardware Drivers) and enable the proprietary fglrx driver.

I used to have a custom xorg.conf to set up the fglrx, etc., but I have found for the first time that with 8.10 it is not necessary, at least for my setup.  I have a Lenovo W500 with ATI Radeon HD 3650, and it runs fglrx perfectly.  Even suspends and hibernates as long as the desktop effects (compiz) are not enabled.

### Post by gatman3 on 2008-11-19
Hmmm... not sure about 64-bit.  I haven't messed with the 64 bit stuff for a couple of years.  It's possible there are other "issues".  But presuming not...

It seems to me that there was some trial and error involved.  I definitely experienced a few reboots where pushing the button to enable the proprietary ATI drivers did nothing.  I wish I could remember more details, but I can't.  One thing you might try is enabling the pre-released updates in Adept or Synaptic.  In Adept (KDE) click on Source->Updates-> check "Pre-release updates".  Then "Fetch current package lists" again, and see if there is an update under "System" for the ATI binary X.Org driver.  Under Synaptic (GNOME) it's Settings->Repositories and then the same procedure.  Adept seems to hide the actual package names, though, and in Synaptic the package you want is called "xorg-driver-fglrx".

Another thing to try... if that doesn't "just work"... is to use the aticonfig utility.  Unfortunately, this requires command line activity, so hopefully you're comfortable with that.  First, make sure you have the fglrx package installed (as described above).  Then, at a command line, type

```
aticonfig --help
```

That will show you all the options.  For a basic setup, the one you probably want is:

```
sudo aticonfig --initial
```

Make sure you run as root or with sudo as shown above.  If that appears to run happily, then you'll need to reboot.  If the aticonfig command is not found, then you probably haven't gotten the fglrx driver installed properly.

Once you reboot, to determine if it worked... you can try:

```
fglrxinfo
```

Here is my output:

> fglrxinfo
display: :0.0  screen: 0
OpenGL vendor string: ATI Technologies Inc.
OpenGL renderer string: ATI Mobility Radeon HD 3650
OpenGL version string: 2.1.8201 FireGL Release

At this point you should also have access to the ATI Catalyst control center.  Not sure where it is in GNOME (I'm sure it's there), but in KDE you find it at Applications->Utilities->ATI Catalyst Control Center.  You might first just try the normal GNOME or KDE display settings to set the resolution, etc.  I use dual head, and I have other configuration complexities, and for some reason I have found that the ATI control center is the only way to set up the resolution properly for my dual head setup.

Good luck.
