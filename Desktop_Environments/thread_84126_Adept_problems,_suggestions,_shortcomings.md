### Post by Matchless on 2005-10-30
Hi,
   I have found that Adept which ships with kubuntu still needs a lot of work and at the moment it seems very basic compared to Synaptic. I feel the user side needs a lot more work and the author must be supported by us and be made aware of the requirements. Adept is only going to make it, if it is an improvement to Synaptic from the users point of view. I have been forced to swing back to Synaptic untill more features are built into Adept. I think the auther is doing great work and this is not any criticism, but aimed to help in improving the new wheel. I have added some rough ideas and would appreciate it if anyone can add to this thread:

1) Adept should have an add CDrom facility same as Synaptic.
2) Adept should have an example note (hover?) when adding a repository to show the proper syntax and to give the user more details.
3) Adept should have a facility to create own local repository from cache and allow downloads  to the repository for later offline work or to burn to a cd.
4) Adept should have an edit button where you enter repositories in case of a typo or change?
5) Adept v 1.0 is not changing the sources.list file it seems when enabling or disabling a repo- bug? 
6) Adept should also show from what repository it has found a file and allow preference selection.
7) Does Adept resume after partially downloading? How does one interupt a download to do this? Maybe a button added?
8) Adept - stop/resume button for long downloads?
9) Adept should give an indication like Synaptic how long the update of the packages list will take, the present percentage does not tell the user much.
10) Adept should also be able to sort the repositories in order of preference in the sources.list file. Manually move up or down.
11) The details of a file is awkward to read as it needs to be clicked on and opens in another layer, see how Synaptic does it?

It would be nice if the author also partakes in this thread, so that we know what is planned or has been noted.
