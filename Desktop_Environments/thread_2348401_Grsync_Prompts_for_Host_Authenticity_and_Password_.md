If I run Grsync to external USB drives where host authenticity and passwords are unnecessary, it runs just fine. If I run Grsync from Terminal, I am prompted in Terminal for the host authenticity and the password of the target machine, and if I enter them in Terminal, the rsync command works, so it appears to just be an issue with the prompts not being displayed in dialogs as they should be.

I've used Grsync for years, in a variety of flavors of Ubuntu, but this is my first time with Ubuntu Gnome. I don't know if there's a setting I'm missing in Gnome, or if this is a bug. I've tried the same Grsync script in Ubuntu Gnome 16.10, with the same issue. In Ubuntu Mate 16.04, it works just fine, as it did in Ubuntu 14.04, so it appears to be an issue confined to Ubuntu Gnome.

Does anyone have any ideas?
