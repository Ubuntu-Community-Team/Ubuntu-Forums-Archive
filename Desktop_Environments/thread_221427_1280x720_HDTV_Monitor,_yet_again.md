I can only really offer two more suggestions, I'm afraid. The first - I have seen these exact problems with onboard graphics be resolved after a BIOS upgrade. Worth a try, I suppose?

Before that, though, you may try changing the BIT line to 
```
BIT=24
```
and see if X is happy at this Depth.

### Post by hotani on 2006-07-25
I've been through this with my display which is 1920x1080. Here is the xorg.conf that works for mine:

```

Section "Monitor"
        Identifier      "LVM-37w1" # this is the model name of my monitor
        Option          "DPMS"
        HorizSync       30-80
        VertRefresh     50-75

	Modeline "1920x1080" 172.81  1920 2008 2216 2616  1080 1080 1082 1119 
EndSection

Section "Screen"

        Identifier      "Default Screen"
        Device          "NVIDIA Corporation NV25 [GeForce4 Ti 4600]"
        Monitor         "LVM-37w1"
        DefaultDepth    24
        SubSection "Display"
                Depth           1
                Modes           "1920x1080" "1680x1050" "1280x1024" "1024x768" "800x600" "640x480"
        EndSubSection
        SubSection "Display"
                Depth           4
                Modes           "1920x1080" "1680x1050" "1280x1024" "1024x768" "800x600" "640x480"
        EndSubSection
        SubSection "Display"
                Depth           8
                Modes           "1920x1080" "1680x1050" "1280x1024" "1024x768" "800x600" "640x480"
        EndSubSection
        SubSection "Display"
                Depth           15
                Modes           "1920x1080" "1680x1050" "1280x1024" "1024x768" "800x600" "640x480"
        EndSubSection
        SubSection "Display"
                Depth           16
                Modes           "1920x1080" "1680x1050" "1280x1024" "1024x768" "800x600" "640x480"
        EndSubSection
        SubSection "Display"

        EndSubSection
EndSection

```
