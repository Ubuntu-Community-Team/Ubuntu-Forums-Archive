### Post by fjm03 on 2007-04-25
OO2.2 won't open a document stored on a network server. Attempting to open a document, whether by mouse or menu command, whether through OO or through Nautilus, results in:

1) In the case of an OO file extension:

a) The OO splash screen appears
b) The progress bar proceeds at a normal pace from left to right ..
c) and then, instead of the document appearing, OO closes (not sleeping, but gone).

OR

d) OO advises that the document, because it's an OO extension, represents a security risk and demands the file be renamed with an OO extension before it can be opened

2) In the case of an MS file extension:

a) OO advises that it doesn't recognize the mime type.

OO behaves normally if the documents are stored on the same drive as the O/S.

This is a serious problem for any office environment. Any help resolving the issue would be appreciated.
