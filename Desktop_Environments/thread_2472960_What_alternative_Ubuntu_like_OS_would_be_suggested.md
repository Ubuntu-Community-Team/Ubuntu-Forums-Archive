### Post by guiverc on 2022-03-18
You've provided few specs; that laptop is newer than the box I'm on which to write this reply; but machines could be purchased with 2GB to 8GB of RAM which is something I'd consider in what OS & desktop I'd use, as well as what apps you'll actually want to use.

If RAM is tight (ie. *limited*) I'd be matching DESKTOP & apps you're going to use to ensure you're not wasting RAM by requiring multiple *libs* in RAM that do the same thing slightly differently to co-exist in the *limited* RAM.  Here details are no different to using windows & apple OSX, except they avoid the technical detail & just tell consumers you need more RAM (ie. *skipping the reason why*).

Lubuntu is the *lightest* and *fastest* flavor of Ubuntu; HOWEVER it's Qt5 based meaning the *light & fast* will be lost if you're using GTK3 apps. Xubuntu will win out with GTK3 apps... ie. each has it's *pros* & *cons* so you should match the *apps* you'll use with the desktop esp. if resources are limited.

I use devices still with 2GB of RAM (eg. a core2duo laptop with 2GB of RAM) and have both Lubuntu/LXQt installed & Xubuntu/Xfce installed/co-existing on the same install (*not dual-boot*) as I'm not worried about disk space on the box; but I'll login and use whichever will be most efficient for what I'll do that session (ie. LXQt or Xfce). For your Aunt I'd likely recommend you avoid my multi-desktop system too (*it adds complexity*) but my point is choose what will be most efficient for your end-use case. 

If you've plenty of RAM (ie. 5GB+) you can pretty much disregard *libs/TKs* co-existing in RAM & consider *tastes* & what will bring you the most pleasure..  ie. weigh up all facts in your decision.  I'm using the 2009 dell as it's more RAM than much newer boxes I have, plus I like it's form-factor as well (ie.* extra RAM on this core2 box beats my newer i5 options*)

My primary point is that you've not provided what I'd consider in which is best for you.  Even if I rarely actually do it; my mind weighs everything out like I was mapping out a *kepner-tregoe *chart & details such as use-case; apps to be used; RAM etc. strongly influence the weighting of your options.


Slow to boot; yeah Ubuntu (and *flavors*) are somewhat slow to boot, which can be sped up, but as with everything, there are pros & cons and I just *sleep* my machine (*when at home & thus aren't worried about security with regards others*) if I'm going to use it later.

On a resource challenged box; GNOME isn't my choice. Sure it has nice features that make it *shine* on touch-sensitive & gesture capable screens on newer boxes.. but extra code is required to make those features work which will slow down a resource-challenged box.

System boot times are not that different between Ubuntu and the various *flavors* of the same release, as underneath they are all the same Ubuntu base system booting/starting up.  Differences are largely related to release in my experience.

The same occurs with regards opening `firefox` or a web browser.  Of note some package versions (eg. *snap*) are slower to start the first time they are run when compared to *deb* packaged version (Lubuntu *jammy*, Ubuntu-MATE* jammy *& Ubuntu-Desktop *canary-jammy* contained both on yesterday's *daily *allowing easy comparison, but does it matter? as you only start the program once per session).

The difference is *speed* and *lightness* relate to RAM usage after starting, and what is available for user-apps; start up time & allocation load time are pretty equal EXCEPT that if you're using a Qt5 desktop, the first GTK app started will be slower; and likewise if using a GTK desktop the first Qt5 app will be slower (*but that's a user choice because of the mismatch*) unless those services are made to start at boot/login time - meaning a slower boot/login anyway.
