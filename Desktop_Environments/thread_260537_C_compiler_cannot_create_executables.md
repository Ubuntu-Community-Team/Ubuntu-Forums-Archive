### Post by chavo on 2006-09-19
Ubuntu doesn't come with a compiler by default. You have to install it and a few other things to get it working. Luckily there's a meta package for that.

Just do this ->
sudo apt-get install build-essential

This will give you what you need to compile. Now every program you compile is going to depend on different library headers being in the system. You can use the build-dep argument to apt-get to get waht you need. 

For example ->
sudo apt-get build-dep wine
Will install everything you need to be able to build wine.

### Post by CompShrink on 2006-09-24
Trav, you do realize that using the apt-get build-dep just does a source build of the same version of wine as the pre-made binary. It won't give you any new features or whatever you expected from the CVS version you were going to compile.

What it does, is gives you a slightly more optimized version built for your hardware, although the difference is probably minimal.

Once you have build-essentials, you can try ./configure on the CVS like you started with. Assuming you had a reason why you were using CVS.
