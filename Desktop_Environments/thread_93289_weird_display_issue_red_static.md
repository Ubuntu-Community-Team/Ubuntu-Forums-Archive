I noticed this "red static" didn't happen with kernels of 2.6.11 and earlier, but only with 2.6.12 and higher. Also, I canged my monitor from the Apple 17" Studio Display to a Samsung SyncMaster 914v. I use a Radeon 9800 Pro graphics card, which has ADC for connection. The Samsung monitor has only VGA connection, so an adaptor was used to connect it to the ADC on the graphics card (Apple monitor is ADC).
NO RED STATIC!! Yay, I'm happy, but, you know, "what about it"? From the eyes of a very inexperienced GNU/Linux user, it would appear that there is some sort of issue with kernels 2.6.12 and higher and Apple's ADC connection. But, that's about all I could surmise from this experience. A workaround is nice, but never a substitute for a real solution.
