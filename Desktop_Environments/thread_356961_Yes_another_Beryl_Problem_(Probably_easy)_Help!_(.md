### Post by shareMenaPeace on 2007-02-09
Sorry xcant help you for sure best is put the settings back and checkout this guide
```
 ATI

There are three different drivers for ATI cards: "ati","radeon" and "fglrx". The "fglrx" driver is ATI's closed source driver and you will find instructions on how to install it below. This driver will *not* work with AIGLX however as it does not support compositing (as of version 8.30.3). If ubuntu installed fine you should already have the "ati" driver loaded. If ubuntu installation seems to freeze instead of showing you the gnome desktop odds are that the X server failed to start with the "ati" driver and you will need to change to the "radeon" driver. If you have an X1K (eg: x1600) series however, use "vesa" instead of "radeon" as the open source radeon driver only supports cards up to the x850 and is very unstable on cards above the radeon 9800 series (ATI is directly at fault for this because they do not respect you the user and your choice to use a powerful operating system such as Ubuntu). To do this restart the live-cd and press F6 and remove "quiet splash --" from the boot options. Now you will get a command prompt when X fails to start. Once you are at the command prompt type "nano /etc/X11/xorg.conf" and locate the line "ati" and exchange it for "radeon". Save and start X by typing "startx". All you have to do now is to install beryl. 
```
[http://wiki.beryl-project.org/wiki/Install_Beryl_on_Ubuntu_Edgy_with_GC_Drivers#ATI](http://wiki.beryl-project.org/wiki/Install_Beryl_on_Ubuntu_Edgy_with_GC_Drivers#ATI)
