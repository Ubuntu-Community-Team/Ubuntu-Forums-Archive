### Post by wildepiet on 2007-12-07
I use eclipse on kubuntu gutsy.
A shortcut I regularly use is Ctrl+Shift+I
for some reason, since using kubuntu (I was previously on redhat with KDE), pressing these keys does nothing.
I had a look at the keyboard shortcuts in the system settings but this shortcut is not assigned anywhere.
In fact, If I assign it to, say for example, maximize a window, the assignment goes fine, but pressing the keys still has no effect.
It seems 'something' is interception the keystroke. 
I've got a fairly clean kubuntu install, So I've got no idea what that 'something' might be.

Any help will be greatly appreciated

### Post by wildepiet on 2007-12-07
And he solves his own problem :

I started stopping all the things in system tray to see if something there was intercepting my keystrokes, It turns out to be Kopete that has this shortcut in it's 'global shortcuts' list.

What would be handy is a kde utility to see which program handled any given keystroke. There might be something like that already...
