### Post by jrb114 on 2007-11-28
I don't know if anybody else thinks this, but here's a couple of feature requests/bugs I've found in tracker.

Ignoring the large number of bugs in it generally:
[https://bugs.launchpad.net/ubuntu/+source/tracker/+bug/130935](https://bugs.launchpad.net/ubuntu/+source/tracker/+bug/130935)

and the typo in the config file:
[http://ubuntuforums.org/showthread.php?t=520853&highlight=tracker&page=4](http://ubuntuforums.org/showthread.php?t=520853&highlight=tracker&page=4).

My own wishlist would consist of, for now, fixing these two things.

Say I write my CV. It has my name on it, but nowhere does it say, "cv". The file is called cv.pdf. I would expect tracker to find this file when I search for cv. It doesn't. If I search for my name, then all the iterations of the file pop up. Is this expected behaviour? I'd say it shouldn't be, even if it is.

=edit=
This has already been reported. Apparently svn fixes it.
[https://bugs.launchpad.net/ubuntu/+source/tracker/+bug/148114](https://bugs.launchpad.net/ubuntu/+source/tracker/+bug/148114)
=end edit=

Second on my wish list. Another way to sort the results in the display window. I think the most useful case would be to show results that have been modified most recently at (or close to) the top of the result list. If there was some was to do that, it'd improve the functionality a lot.

=edit=
And this has, too. Although there may be a better bug report somewhere.
[https://bugs.launchpad.net/ubuntu/+source/tracker/+bug/145367](https://bugs.launchpad.net/ubuntu/+source/tracker/+bug/145367)
=end edit=

I'm not trying to criticise tracker or the developers. I think it will be extremely useful and provide a good source of integration for all the apps. It runs so much more effectively than beagle did when I tried it. These are just my pet hates about it.

Comments welcome :)
