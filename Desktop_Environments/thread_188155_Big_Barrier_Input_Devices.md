### Post by Gtaylor on 2006-06-03
One area that I feel Linux really needs to focus on in the coming years is getting a wide variety of input devices easily configured. Right now, many mice, for example, require lots of manually editing xorg.conf, using xmodmap, imwheel, and any other number of really dirty hacks. This usually involves a lot of X session restarts and stuff that you shouldn't have to do. Of anything that should "just work", it should be keyboards and mice. Windows has this down pat for 95% of the typical input devices. I shouldn't have to do anything special at all to use some of the extremely common mice out there. I've found keyboard support to be excellent, but mice seem to be far behind.

I hope that there are better mouse configuration utilities in the future, or better yet, we achieve that level of "just works" that Windows currently has with regards to input devices. No holy war starting intended here, this is just something that **really** irks me and I am embarassed to have to tell complete newbies in our LUG when I'm installing on their machines that Linux can't handle configuring its own mice :)
