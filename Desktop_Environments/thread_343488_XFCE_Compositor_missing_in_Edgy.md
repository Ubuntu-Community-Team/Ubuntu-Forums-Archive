Thanks for the effort! Yeah my nothing really concrete came up in my google searches too hence this post.

Couldn't sleep coz of this, and so finally came up with a solution as follows:

[LIST=1]
[*]Check if the file ~/.config/xfce4/mcs_setting/wmtweaks.xml exits.
[*]Delete it if exists. If it doesn't go to next step.
[*]Open Window Manager Tweaks settings and fiddle with some settings. This will ensure xfwm will generate a new wmtweaks.xml file
[*]Logoff/ login again. Composite effects should now be enabled, and Window Manager Tweaks should have the compositor tab back.
[/LIST]

The solution above i've already read elsewhere, but in a different  context which initially confused me. As I read it, this situation happens in the ABSENCE of the wmtweak.xml file. In my case, I still had a carryover from the dapper install, so I assumed that something else was amiss. 

As a next step I found out that this line:
```
<option name="Xfwm/UseCompositing" type="int" value="1"/> 
```
should be existing in the .xml file. My file didnt have it, so I proceeded to edit the file manually to add the missing line, restarted my session but still nada. So it appears that the edgy xfwm will need to regenerate a new xml file to function properly.

When I installed Edgy, I had the same trouble. ### Post by kerry_s on 2007-02-22
Add this repo to your /etc/apt/sources.list and update-> 

```
deb http://ubuntu.tolero.org/ edgy xfce-4-4-0 
```

Then make sure you turn the composite on, add this to /etc/X11/xorg.conf->

```
Section "Extensions"
    Option         "Composite" "True"
EndSection
```

Then you'll find a new tab under the windows tweaks in settings to control composite.
