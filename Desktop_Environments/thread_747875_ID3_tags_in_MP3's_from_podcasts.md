### Post by Levander on 2008-04-07
I've started downloading a bunch of podcasts with Rhythmbox.  Apparently, about a third of the people doing podcasts are not keeping up with the ID3 tags in their MP3's.  So, when the MP3's go into my music collection, they show up and Unknown Artist, they're missing album names, sometimes the same podcast will have different (but synonymous authors), etc., etc...

I looked at the package description for some program in the repositories called hpodder.  It includes this:

```

 Automatic updating of ID3 (v1 and v2) tags based on metadata in the
 podcast feed.  This important feature is available through iTunes but
 is often missed by other podcatchers.

```

Now, I'm thinking I don't want to use hpodder for podcasts because all it does is podcasts.  I'm wanting something that does more than just podcasts, I want to be able to control my whole library.

I know there are a bunch of up and coming music library managers for Linux.  Do any of them have this feature where they will populate ID3 tags with info from the podcast feeds?

I'm pretty happy with rhythmbox except for the lack of this one feature and what seems like a few bugs.

I'm interested in whether or not amarok supports this feature, but really don't think I'll use it because I don't want to install all the kde libraries.

Has anyone tried using some combination of hpodder and rhythmbox, or other music collection manager?
