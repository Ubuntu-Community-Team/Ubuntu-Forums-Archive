### Post by rplantz on 2005-12-28
[QUOTE=kabus]I use mutt with getmail to, er, get mail, msmtp to send mail and procmail to sort mail into folders.
I chose that setup mainly because I like mutt, for its amazing configurability and the keyboard-driven interface (I don't like mice :p ).
I chose the other programs for their simplicity, because I think stuff like postfix is a bit overkill for my single user needs.[/QUOTE]

General problem: I wanted to use mutt to read several email accounts, both imap and pop. I use a "wireless dsl" to connect to the internet. I shut down my machine when not in use.

This is very easy with Evolution, Thunderbird, Opera m2, etc., but I want to try mutt.

I did much searching and reading, both online and in several books. They all go into great detail. Most of them suffer from the too-much-detail-too-soon syndrome. That is, I want to get it to work before I worry about text colors, etc.

After spending many hours on my postfix configuration file (/etc/postfix/main.cf), I almost had it working. But I had the general sense that I wanted to "wipe the slate clean" and start all over again, armed with some knowledge from all my reading.

Here are the steps I took:

1. Make a backup of my current postfix configuration since it was almost working.
```
sudo cp /etc/postfix/ postfix_backup
```

2. Use Synaptic to do a complete removal of postfix. This also removed mutt and muttprofile. I checked, and the entire /etc/postfix/ directory was removed.

3. Use Synaptic to install mutt, which causes postfix also to be installed. (I also installed muttprofile again; this might be useful to me after I learn more about using mutt.)

This causes a Postfix Configuration dialog to appear:

a. For "General type of configuration?" I selected "Internet with smarthost."

b. For "Mail name?" I used the domain name of my isp email address. That is, everything following the "@".

c. For "SMTP relay host?" I used the name my ISP provided me. E.g., mail.cds1.net

4. I had already configured my ~/.fetchmailrc file to include things like
```

# poll every 5 minutes
set daemon 300
# department mail server
poll "cs.sonoma.edu"
protocol pop3
username "my_user_name_for_this_account"
password "my_password_for_this_account"
fetchall
# my isp mail server
poll "mail.cds1.net"
protocol imap
username "my_user_name_for_this_account"
password "my_password_for_this_account"
#
mimedecode
mda "/usr/bin/procmail -d %s"

```

I then started mutt from the command line and everything works!

If anyone has comments about how this can be improved, any errors I've made, etc., please let us know.
