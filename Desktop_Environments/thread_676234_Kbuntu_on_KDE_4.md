### Post by jacksaff on 2008-01-26
I don't think July is unrealistic. 
The differences in functionality between each beta, RC and 4.0 were all huge and they are going to release monthly 4.0.x releases this year as well.
Except for plasma, all of the under-the-hood work has been done.
Plasma itself is nearly there. 
Most of the functionality will be added by the wider community writing plasmoids and that's already happening pretty fast now as plasma itself makes development fairly easy. Much easier than superkaramba for kde3 anyway, and with a lot more features available too.
The main trouble with July to me is that they will keep a lot back from 4.0.x releases.
I think they should just treat all releases up to 4.1 as testing releases and push as many features as possible, bugs included.
They've already scared off everyone who won't or isn't capable of testing and bug-reporting so there's nothing to lose by doing this and it will getthe new features out there, used and tested sooner.
