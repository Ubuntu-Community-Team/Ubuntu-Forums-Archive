### Post by eldragon on 2008-08-26
the ubuntu codebase is usually updated every 6 monthes. it usually happens every april and every october.

so 8.04 was released april-2008, next release will be in october, and will be numbered 8.10

you will notice open source software is under constant development. so distros trend to freeze the version number before release, and only update security fixes / bug fixes to those versions until the next release.

if i were you, i wouldnt rush it to 2.22 since its fairly new and you might see things dont work as they are supposed to. if you wish, you could always download and install 8.10 alpha3 which is the latest snapshot from the next release.


i hope i cleared some stuff up

good luck.
