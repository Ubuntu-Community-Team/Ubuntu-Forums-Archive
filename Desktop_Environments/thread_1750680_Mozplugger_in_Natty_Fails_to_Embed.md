### Post by Nisitiiapi on 2011-05-05
I've always used mozplugger to have pdf, doc, odf, etc. open in Firefox and it's always worked beautifully.  It even worked with Firefox 4 in Maverick.

But, now, with Natty -- a clean insall, in fact several clean installs -- mozplugger isn't working well at all.  No matter the document type, Firefox won't open it within the browser.  Instead, it opens it creates a blank page and opens outside the browser.  It doesn't matter whether it's a pdf, doc, odf, etc.  It happens with all things handled by mozplugger.

Interestingly, the document remains attached to Firefox.  If you close the blank tab, the external window with the file also closes.  So, it's just that the apps won't load within Firefox.

I've tried messing with mozpluggerrc.  I've created clean Firefox profiles with no add-ons, etc. and nothing fixes the wrong behavior.  It might not be horrible if the files just opened as external application windows, but the fact they are tied to browser tabs that can't be closed makes working in Firefox and Natty a nightmare with embedded documents.

Has anyone else come across this problem or found a solution or is this?  I've filed a bug in launchpad, but am hoping someone may have found a solution.  Thanks in advance.
