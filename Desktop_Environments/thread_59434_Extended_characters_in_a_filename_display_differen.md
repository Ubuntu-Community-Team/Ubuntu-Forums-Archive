### Post by Corlian on 2005-08-24
In a given filename, extended characters such as accented vowels display correctly in XP, but not in Kubuntu (they show up as the empty boxes).  What's more, it seems that I can type (or copy, so far) extended characters into a filename in Kubuntu and it displays correctly, but I already have quite a lot of them and it would be ridiculous to retype them all.

Is there any way I can fix this?

---

### Post by Corlian on 2005-08-24
My problem fixed itself.  I read one thread about a different problem which suggested changing to system fonts that had more friendly extended characters (such as FreeSans).  I did that before I posted, but my problem didn't change.  However, the next time I booted the computer after changing to the new fonts, everything was fine.  When I was having the problem, I even unmounted and remounted the drive with all the files on it, and it still read them wrong.

But now everything's okay, hurray!  I hope it helps someone in the future.
