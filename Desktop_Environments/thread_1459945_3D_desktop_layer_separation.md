### Post by jlebrech on 2010-04-22
The user will be able to zoom in and out to different sections of usability thse would include.

starting from the top layer:

widget layer - when the user zooms right up to the top he will see the widget he has installed.

icon layer - rather than have icons behind the windows and on top of the wallpaper they wil appear when zooming away from the windows.

window layer - the windows will appear within that layer, they can shuffle around for task switching etc, within that layer.

wallpaper layer - the wallpaper will appear cleanly within that layer without ugly icons ontop of it, and an idle desktop will automatically zoom into that screen before eventually locking

technical stats layer - applications such as conky and superkaramba with display within that layer.

lock - when zooming in from tech layer, this will lock the user (maybe with a time out to enable zoomin back out, to not lock)

logout - when zooming this will log the user out (maybe with a time out to enable zoomin back out, to not log out)


the advantage of this is that, control panel be have different contexts. launchers can be on the icon menu and window arranging panels can be on the window layer.
