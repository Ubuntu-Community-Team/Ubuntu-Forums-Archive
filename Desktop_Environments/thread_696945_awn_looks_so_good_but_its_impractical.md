### Post by chipants on 2008-02-15
the awn configuration utility would be good to explore.

with the correct settings, the 'dock' will show a little triangle underneath any running app.

to show your launchers as well as apps and windows running that don't have launchers in the dock, open the awn manager and click on 'applets' in the left pane and make sure 'launcher/task manager' is one of the active applets. then, add launchers for your favorite apps by clicking 'launchers'. 

then, also in the awn manager, click on the 'task appearance' tab, and make sure "tasks have arrows' is checked. that way, any running show an arrow underneath it much like the OS X dock. if you open an application that doesn't have a permanent launcher in the dock, a new icon will pop up, with an arrow underneath to show that it's running. 

in the 'general' tab, you can choose whether or not you want to be able to see tasks and running apps for the current workspace only, or for all workspaces.

there are several applets for awn that provide a way to navigate between workspaces. just pick one!

### Post by pebo on 2008-02-15
> **Marnijt said:**
> I've came across the same problem. It's to bad that when you click an launcher it not ads another icon. 

For example. I have a launcher for firefox. When i click it once it will open an instance of firefox. I can't click it another time to open another instance of firefox. It just pops up the first instance.


You can change that behaviour - see the [awn wik]("http://wiki.awn-project.org/index.php?title=FAQ#Why_do_some_launchers_make_new_tasks_appear_and_some_reuse_the_launcher.3F")i>  Why do some launchers make new tasks appear and some reuse the launcher?

    This is because AWN matches precesses to tasks by looking at the task name. Example: You have a launcher named "Terminal" in my dock. This launcher launches a program whose title (the text at the top of the window is "~ - Terminal"; AWN uses launcher as task icon. If you rename "Terminal" launcher to "Command Line", AWN does not use launcher as task icon, because "Command Line" is not part of the program's title. If you configure the terminal to have the title "~ - Command Line" (from its preferences), AWN uses launcher as task icon. If you rename you Bash Command Line launcher to "Terminal" or "Konsole" or whatever it has in the title, it will not launch a new instance. If you want the opposite behaviour, make sure the launcher name is not part of the window's title. Lastly, you can middle click to launch a new instance of a used launcher. 

-- (Semi-)Quoted from: [https://answers.launchpad.net/awn/+question/11017](https://answers.launchpad.net/awn/+question/11017)
