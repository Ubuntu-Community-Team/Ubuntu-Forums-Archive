### Post by kidders on 2007-06-09
Hi there,

> **besson3c said:**
> I know I should probably just get used to doing things the Ubuntu/Gnome way, but if there are ways to kind of ease me in that would be fantastic!That'll be the answer (at least the *easiest* answer) to some of your questions, but Linux is supposed to be configurable ... we should be able to find a few ways to make it feel more Mac-ish.

Mostly I only ever find myself saying this to Windoze users, but having said the above, it is still important to remember that Linux & MacOS are not the same.

> **besson3c said:**
> an OS X dock that doesn't require BerylI love the OSX dock. Although I'm using a CVS version of avant-window-navigator at the moment, I find it totally unsatisfactory. I haven't been able to find anything (whether it requires Beryl or not) that works the way we Mac lovers would want it to. :-(

> **besson3c said:**
> control + A to go to the beginning of a line like in emacs, rather than the "home" keyI wouldn't recommend pursuing that one. CTRL+A is a cross-OS de facto standard shortcut for selecting things.

> **besson3c said:**
> backspace key for deleting mail in Thunderbird rather than delete keyHere, I would recommend trying Opera, rather than Mozilla products. Opera is the configurability king! Along with well-tuned Safari- & Apple Mail-like skins, you will probably find that other users have already designed Mac-ish key bindings for it.

> **besson3c said:**
> switch between windows within an application (apple + tilda on the Mac)I'm not sure about this one. Can KDE do it? (It's been a while since I've used KDE.)

> **besson3c said:**
> control T for new tab in Gnome Terminal rather than control + shift + TAgain, I would recommend against making that change, since it would prevent you from sending the CTRL+T key sequence to anything you ran in a terminal window.

> **besson3c said:**
> A consistent key command to switch between tabs in Firefox and the Terminal (e.g. Control + Alt + cursor arrow rather than page up/page down)I notice that you don't make much mention of the "Command" key, which is essentially wasted in Linux. (Doesn't your keyboard have one?) I use an Apple bluetooth keyboard with my Linux, so I have been using "Command" to extend the range of Linuxy things I can do in a single keystroke ... but I'm sure you could use it to set up some of OSX's shortcuts.



> **besson3c said:**
> I apologize if this is the wrong forum section for this, but it seems like the Mac sections are better suited for installation type questions.This seems to be quite a lax forum, in that people don't seem to get too hung up about stuff being in the wrong places. The advantage to posting in the Mac sections is that your post may have been answered more quickly ... since that's where most of our Mac guys probably hang out.

Anyhow, the one general comment I would make is that your first job should be to choose the right applications. For instance, I find Gnome rigid & inextensible by comparison to KDE. The same goes for Firefox vs Opera. (Having said that, I've been using Gnome consistently for the last 3 or 4 months, to try to force myself to like it lol!) Extremely configurable apps tend to be the ones people find the most confusing ... but they offer the most possibilities, when you want to do something strange or special.

I hope something here is useful!
