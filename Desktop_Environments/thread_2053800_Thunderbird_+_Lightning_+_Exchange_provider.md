### Post by sundman on 2012-09-06
The latest release of Thunderbird, 15, which rolled out last week made the excellent add-on "[Exchange 2007/2010 Calendar and Tasks Provider]("https://addons.mozilla.org/en-US/thunderbird/addon/exchange-20072010-calendar-/")" unusable due to some internal changes in Thunderbird.

The add-on is an one man's project but it is really a key component if you are working in a enterprise environment with mail and calendar services provided by Exchange. AFAIK there is no other tools that allows me to work not only with my own, but also shared calendars on Exchange 2007. Evolution is too buggy and DavMail only gives me my own calendar.

The solution, for the moment, if you need your calendar back, is to rely on the mozilla-stable ppa and stick to version 14, as outlined in [this post]("http://forums.mozillazine.org/viewtopic.php?f=28&t=2529599") on mozillazine.

Also I read some discouraging comments around the future of Thunderbird, what if Mozilla stops development on it? A full-featuered mail and calendar client is a must for the Enterprise Desktop.
