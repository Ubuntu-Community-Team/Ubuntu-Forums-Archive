### Post by valadil on 2005-01-31
This may sound stupid, but have you tried cd to change directory?  Also you shouldn't need to boot into single user mode to get out of X.

Just hit ctrl-alt-f1.  That will bring you into virtual terminal 1.  Try the following command:
sudo /etc/init.d/gdm stop
It will ask for your password.  Then you should be able to install your graphics drivers.  Use the following command to bring back x:
sudo /etc/init.d/gdm start

For part 2, edit your /etc/X11/XF86Config file (or xorg.conf if you're in hoary with xorg).  You need to change the monitor refresh rates to reflect your monitor.  Googling for its model will give you the refresh modes you need to use.

### Post by valadil on 2005-02-01
The part about 1280x1024 is correct, but you may need to change HorizSync and VertRefresh (a few lines above the resolution) if you want your monitor to behave.  If horizsync needs to be 60 for a resolution that high it can't reach 60 with the current values you have set up.

Someone sitting next to me in class says you can comment out both lines and the comp will ask the monitor what it wants.  You can try that too.

As for the other question, you need to install a compiler if you want your comp to compile stuff
apt-get install gcc make automake autoconf gpp g++
I think that'll get everything you need.
