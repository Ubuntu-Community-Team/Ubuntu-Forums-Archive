Under 11.04, using the fallback GNOME 2 environment, I am trying to set up my Compose key. I have a sample set of shortcuts in `.XCompose`:

```
james@thoth:~$ cat .XCompose 
include "%L"
<Multi_key> <Right> <c> :               "&#8635;"    U21BB    # CLOCKWISE OPEN CIRCLE ARROW
<Multi_key> <Right> <Right> :           "&#8594;"    U2192    # RIGHTWARDS ARROW
<Multi_key> <Right> <bar> :             "&#8677;"    U21E5    # RIGHTWARDS ARROW TO BAR
<Multi_key> <Right> <f> <less> <3> :    "&#9753;"    U2619    # REVERSED ROTATED FLORAL HEART BULLET
<Multi_key> <Right> <o> :               "&#10227;"    U27F3    # CLOCKWISE GAPPED CIRCLE ARROW
<Multi_key> <Right> <asciitilde> :      "&#8669;"    U21DD    # RIGHTWARDS SQUIGGLE ARROW
<Multi_key> <Right> <minus> :           "&#8674;"    U21E2    # RIGHTWARDS DASHED ARROW
<Multi_key> <Right> <Left> :            "&#8596;"    U2194    # LEFT RIGHT ARROW
<Multi_key> <less> <bar> :              "&#8629;"    U21B5    # DOWNWARDS ARROW WITH CORNER LEFTWARDS
james@thoth:~$
```

These are not being recognised.  I only have the standard shortcuts.  I've set up my compose key as `Alt Gr` under keyboard layouts (I'm using a Dvorak UK layout).  AFAICT it is not recognised under a standard QWERTY either.

### Post by lpietsch on 2011-10-24
Very late reply: Setting "export GTK_IM_MODULE=xim" in /etc/environment has worked for me in a series of Ubuntu versions, including 11.04 with fallback Gnome. (Although, strangely, it didn't seem to work in a Xubuntu 11.04 installation with Xfce desktop environment, where I've had to set xim through the Xfconf settings editor instead, under /xsettings/Gtk/IMModule=xim.)

### Post by rrt on 2013-04-29
The problem with setting the input method to XIM is that it's quite primitive, and many other things don't work: for me, the important one is the ability to enter Unicode code points using Shift+Control+U <code> Return. I just found, after much inconclusive googling and a little conclusive experimentation, that it is possible to get both this AND ~/.XCompose support by using UIM, which is in the "uim" package.
