### Post by ankspo71 on 2011-03-03
> **DMGrier said:**
> So I have read on a lot of articles where people talk about making Ubuntu operate faster with messing around in the setting and I was curious how I would go about doing this?

What part of Ubuntu do you feel needs speeding up on your system? I ask because there are lots of things that can be tweaked that can make Ubuntu faster. 

If you would like it faster on the desktop in general, I would start with modifying the desktop effects. Desktop effects can be disabled completely, or you can use metacity's compositing instead, or you can install compizconfig-settings-manager to adjust all of the compiz 3D effects.

The gnome menu popup delay and menu speed can be modified too, to make the menus much more responsive. The panel auto-hide speed can be adjusted as well.

Lightweight themes (and theme engines) can be installed too. As a rule of thumb, any themes that use images (eg panel backgrounds, application backgrounds) generally take more resources to display those images - which can be very noticeable on a low end system. You can make your existing themes faster by installing gtk2-engines-xfce and choosing them as the "controls" when you click on "customize" in Gnome's/Ubuntu's theme selector window (gnome-appearance-properties). They will make your themes look a little different though.

Replace existing applications with lighter alternatives. 

Swap is so much slower than memory is, so you can disable swap if necessary *but* only if you have plenty of memory. If you still need swap, but you feel Ubuntu is using too much swap and is slowing you down, you can adjust the swappiness value. I have 2GB of memory, so I don't actually need swap, but I choose to keep it enabled so I can still be able to rarely use "suspend to disk" or run memory intensive tasks. I have turned down my swappiness value to 10 though, and I can see the difference it makes when I run applications that use alot of memory.
[https://help.ubuntu.com/community/SwapFaq](https://help.ubuntu.com/community/SwapFaq)

Disable any services you or Ubuntu doesn't need in Startup Applications and in Services. For example, it is safe to disable bluetooth and cups if you don't use bluetooth devices or printers. For even more control over services, you can install "bum" (bootupmanager).

Being that I don't use Ubuntu (I use Kubuntu), this is all I have for now. Searching Google turns up many tips for _["make ubuntu faster"]("http://www.google.com/search?q=make+ubuntu+faster")_ and _["ubuntu speed"]("http://www.google.com/search?q=ubuntu speed")_. Some of them might be really useful but some might be a little outdated though.

> **DMGrier said:**
>  Also would my OS stay as reliable even with doing this? 

This would depend on type of tweaking being made. Sometimes accidental modifications happen too, so be careful. :)

Hope this helps.

### Post by Hreinsi on 2011-03-18
1. Press alt+f2, a dialog box should come up. Type:
Code:

gksudo gedit /etc/sysctl.conf

2. Type your password when prompted, and a text file should come up. Scroll the whole way to the bottom of the file and add this on a new line to the bottom of the file:
Code:

vm.swappiness = 20
vm.vfs_cache_pressure = 50

3. Save the file, reboot and enjoy!
