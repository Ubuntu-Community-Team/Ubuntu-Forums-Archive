### Post by Fyda on 2006-11-04
I came across this same behaviour when I was experimenting with adding Option lines to **xorg.conf**. I found that this behaviour was triggered by the following line:
```
Option "EnablePrivateBackZ" "yes"
```
As soon as I commented that out, everything was back to normal again (ie. windows were properly refreshing).

It's strange that it would go away after a few reboots, though. It could just be your particular card; I'm using Edgy+GLX+Beryl with an ATI Radeon 9550 GE. :-k

Perhaps if you post the Options in your xorg.conf file, someone can spot any problems in it...?
