### Post by AgentZ86 on 2011-08-08
I'm assuming you selected exactly the same setup network printer / windows samba share ? and not the jetdirect section of the ubuntu printer setup right ? 


And assuming you selected the windows samba did you see your printer in the /smb when browsing the network group etc ? 

At this point there is a process for prompt for authentication which allows the printer to print and if you accidentally typed the (incorrect user/pass or mistyped it, AND selected (set authentication destails now) ) then in this case you would not be prompted and it would be incorrect authentication everytime; and would not allow you to print to that printer.

I can only speculate this may be what happened but delete that printer from you workstation and start again and make sure you have selected (prompted for authentication) if you want a prompt

Additionally some printers for some reason won't print a test page, but otherwise print just fine. 

You could just continue and try printing a normal document and see if there is any effect. I have a desktop right now that will to this day NOT print the ubunty test page but otherwise works fine everyday.

I hope this helps.
