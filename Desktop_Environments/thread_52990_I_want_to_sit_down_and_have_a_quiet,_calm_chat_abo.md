Tried **Kubuntu 5.04**, liked the fact that it was just one CD and did apt-get during the install process to install any packages it was installing.  I liked the fact that it had more of the latest software including KDE 3.4 already installed.  The one thing I didn't like was the fact that it didn't have a pre-configured firewall and being both a Linux newbie and while I am a programmer, I am not a network specialist, I found myself running MEPIS from its LiveCD to get the settings I needed to start out with for the GuardDog front-end I installed... not very friendly.  However, with that aside, Kubuntu has been the most stable Linux Distro I have found.... and upgrading to KDE 3.4.2 was relatively painless leaving my system even more stable.

I think one of the main things I really like about Kubuntu is because it is made by one of the Developers from the KDE system and the base Ubuntu is used by a lot of the Mono Development guys and picking up steam there and since I am all about C# Programming, Mono and KDE, this looks like a great place to live  :) 

Is Kubuntu for everybody?  Well, personally I think it needs some work in terms of the packages installed and pre-configured but otherwise, yeah, I think Kubuntu is great!

My recommendation to the Kubuntu guys is to work on your default installed packages and maybe put a "grouped" Development package that will install all the necessary Development apps and libraries to help us Linux Noobs up and running easier.  It took me quite a while to get all the Development Tools and Header files (libs) installed to do everything I wanted.... would have been nice if there was a "kubuntu-development" package to get people like me started with the basics.

Okay, sorry for the long writing... just wanted to share my thoughts on the subject.
