### Post by adamkane on 2006-07-31
I've seen this using Microsoft Access. Unfortunately this would require a company to design and sell something similar for OpenOffice, and I don't think this has happened yet.

An alternative is to install osCommerce along with Apache2/PHP4/MySQL4.

You would have your own little ecommerce store, where you could place orders like a customer, and then process the orders on the back end.

There are a ton of osCommerce modules to handle almost anything, so I am sure someone has created a POS module to handle counter sales.

It sounds daunting, but it is all very straightforward, if you take it step-by-step. I would be very willing to help, if this is a path you would like to explore.
