### Post by wmog on 2018-03-06
Hey everyone,

To set my environment, in `.profile` I source the file `~/.envr`:

```
# ~/.profile

. $HOME/.envr

```
that executes some tasks (environment variable setting, etc.). `~/.envr` uses the second positional argument `$1`. If it is assigned a value, this one is used for further tasks. 

Under ArchLinux or Ubuntu + login shell everything works well, but does not work with Ubuntu + Lightdm. Indeed, it seems that when sourcing `~/.envr` in `~/.profile`, Lightdm passes the file path of `~/.profile` as argument `$1` in `~/.envr`: when I prompt `$1` in `~/.envr` I get `~/.profile` i.e. /home/user/.profile:

```
# ~/.envr

echo "$1"  # output: '/home/user/.profile'

```
 whereas I pass no argument to `.envr` when it is sourced in `.profile`.

How is this behaviour possible and how to fix it please?

Thank you in advance
