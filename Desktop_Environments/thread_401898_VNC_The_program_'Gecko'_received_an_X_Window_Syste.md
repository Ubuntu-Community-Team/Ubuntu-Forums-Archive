### Post by flabdablet on 2007-07-18
Until this is fixed, you can force vnc4server to behave properly by starting it with
```
vncserver :1 -extension XFIXES
```
See [Bug #110263]("https://bugs.launchpad.net/ubuntu/+source/vnc4/+bug/110263").

### Post by geek.de.nz on 2007-08-12
If you are running KDM or GDM and you are using xinit.d make your  /etc/xinetd.d/Xvnc file look like this:

service Xvnc
{
        type = UNLISTED
        disable = no
        socket_type = stream
        protocol = tcp
        wait = yes
        user = root
        server = /usr/bin/Xvnc
        server_args = -inetd :1 -query localhost -geometry 1024x768 -depth 16 -once -fp /usr/share/X11/fonts/misc -DisconnectClients=0 -NeverShared passwordFile=/root/.vncpasswd -extension XFIXES
        port = 5901
}

(for now). It worked for me.
