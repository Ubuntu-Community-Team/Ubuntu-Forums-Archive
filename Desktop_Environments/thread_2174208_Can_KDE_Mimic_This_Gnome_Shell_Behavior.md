### Post by buzzingrobot on 2013-09-13
Gnome Shell plus the DashToDock extension offers some behaviors I'd like to mimic in KDE.

Here's what happens in Gnome Shell.  Let's assume we start with no open applications.

1.  A left click on an icon in the dock opens that app in the current workspace.

2. A middle click on an icon in the dock creates a new workspace, moves to that workspace, and opens the app.

3. A left click on the dock icon of an app that is already open moves the focus to that app, changing workspace if necessary. 

4. If I want to open a new instance of an already running application, I need to do that explicitly, via a right-click on its icon,

5. If the default browser is open, a left click on a link, say, in an email, will open that link in the open instance of the browser. However, the focus does not move to the browser, either in the current or other workspace.  I need to explicitly do that, which I prefer, per Number Three above.


Can KDE mimic this with icons in the panel?
