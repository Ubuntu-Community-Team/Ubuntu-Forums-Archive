My fix was to manually edit grub.cfg to include the 2.6.32-22 kernel.  NOTE if you screw something up here, you may well end up with an unbootable system, so don't do this unless you are prepared for taking full responsibility for you own actions.

I took the lines from the grub-mkconfig output that begin immediately following these 3 lines:
```

### BEGIN /etc/grub.d/10_linux ###
Found linux image: /boot/vmlinuz-2.6.32-22-generic
Found initrd image: /boot/initrd.img-2.6.32-22-generic

```
and paste them into grub.cfg, then I added the appropriate code (from an existing entry in grub.cfg).

I ended up with this (my UUID has been obscured) in my grub.cfg:
```

...
### BEGIN /etc/grub.d/10_linux ###
menuentry "Ubuntu, Linux 2.6.32-22-generic" {
        recordfail=1
        if [ -n ${have_grubenv} ]; then save_env recordfail; fi
	set quiet=1
	insmod ext2
	set root=(hd0,1)
	search --no-floppy --fs-uuid --set c30b2d07-...
	linux	/boot/vmlinuz-2.6.32-22-generic root=UUID=c30b2d07-... ro   quiet splash
	initrd	/boot/initrd.img-2.6.32-22-generic
}
menuentry "Ubuntu, Linux 2.6.32-22-generic (recovery mode)" {
        recordfail=1
        if [ -n ${have_grubenv} ]; then save_env recordfail; fi
	insmod ext2
	set root=(hd0,1)
	search --no-floppy --fs-uuid --set c30b2d07-...
	linux	/boot/vmlinuz-2.6.32-22-generic root=UUID=c30b2d07-... ro single 
	initrd	/boot/initrd.img-2.6.32-22-generic
}
...

```
