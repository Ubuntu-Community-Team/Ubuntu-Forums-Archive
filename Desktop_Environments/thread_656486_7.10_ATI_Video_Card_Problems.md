### Post by tgilber1 on 2008-01-02
Latest drivers from whom, ATI or Ubuntu/xorg?  In other words, did you install the proprietary verision that does not use the apt-get package manager?  In any event, I noticed that you're using ati driver.  I did not have very much luck with it.  The only thing that worked with my Radeon 2400HD Pro, was the Ubuntu vesa and ATI's proprietary driver (fgrlx).

To use Ubuntu vesa

1.  sudo apt-get install xserver-xorg-video-vesa

2.  Edit /etc/X11/xorg.conf and change driver name in device section

Section "Device"
        Identifier      "ATI Technologies Inc ATI Default Card"
        Driver          "vesa"
        #BusID           "PCI:1:0:0" 
#If you use BusID, uncomment BusID line - make sure to use your config, 
#"sudo lspci" and look for the line with your graphics card
#Then, populate your info (e.g, PCI:2:0:0)
EndSection

To use ATI's proprietary version

1.  Download from ATI's website, their latest driver - [http://ati.amd.com/support/driver.html](http://ati.amd.com/support/driver.html)

2.  Remove Ubuntu's fglrx driver to avoid conflict.  sudo apt-get remove xorg-driver-fglrx

3.  Install ATI's driver sudo ./ati-driver-installer-7-11-x86.<platform>.run

4.  sudo  aticonfig --initial -f

5.  sudo /etc/init.d/gdm restart


To uninstall proprietary driver, 

1.  cd /usr/share/ati
2.  sudo ./fglrx_uninstall
3.  Change xorg to desired driver 
4.  /etc/init.d/gdm restart or reboot computer

You can also install latest radeon driver from xorg, which has the installation steps

[http://www.x.org/wiki/radeon](http://www.x.org/wiki/radeon)

Whatever driver you decide to use, you must change the device driver in xorg.conf file to reflect the driver you want to use.

Lastly, except for vesa, I did not have much luck using Ubuntu's video display drivers with my radeonhd on a x86_64 platform (AMD 3700+).  However, this issue should change in a future update because xorg has implemented a fix in their latest release.

### Post by tgilber1 on 2008-01-03
A quick note, the radeon driver that comes with Ubuntu are coded by the xorg developers.  Furthermore, they are not the latest.  Hence, they may not work with your card, since ATI has many different flavor of cards. If your card does not work, then you may only have two options:  

1.  Install proprietary version from ATI, which I believe uses the fgrlx driver.

2.  Go to [http://www.x.org/wiki/radeon](http://www.x.org/wiki/radeon) for the lastest radeon driver.  Installation instructions are available from this website

Additionally, the xorg driver probably may not be able to do 2D/3D.  Hence, check latest release information.  If you want 3D/DRI, you may have to resort to the proprietary version.

### Post by tgilber1 on 2008-01-04
The proprietary driver builds loads a fglrx.ko (kernel object) module.  It does not build an ati.ko driver.  Hence, you will not be able to build one with the proprietary setup program from ATI/AMD.  If you want to load an alternative driver, you will have to load the xorg drivers.  You can check out [http://x.org](http://x.org) site for drivers

[http://x.org/radeonhd](http://x.org/radeonhd)

[http://x.org/radeon](http://x.org/radeon)

[http://x.org/ati](http://x.org/ati) # ati driver is a small wrapper than can be found in three xorg drivers; radeon, r128, or atimisc.  If you load any of these drivers an ati card should work.

Lastly, you can read the development release information for both the features and limitations.  Additionally, the news from the AMD/ATI folks is they are working with the xorg developers by providing some information so xorg can develop open source drivers.
