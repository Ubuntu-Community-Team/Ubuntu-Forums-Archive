### Post by wsmoser2004 on 2007-01-31
I've got a laptop with an ATI Mobility Radeon 9200 which works with the open source drivers and AIGLX in Edgy.  I installed Beryl, and it runs very well, except that the desktop background and kicker seems to "cut off" about 2/3 - 3/4 of the way across the screen.  The laptop screen is a widescreen (1280x800) and the resolution is correct, it's just as though Beryl doesn't seem to notice that it's a widescreen.  It just seems to cut it off where a normal-sized screen would end, and so I have a few inches of white-ness.

I can drag windows over there, but when I do they seem to "tile"...I see multiple copies of the window over the white area where I dragged it, and they never go away.  Maybe Beryl isn't redrawing the screen there for some reason?

If anyone else has had a similar problem and fixed it, please let me know...I'd greatly appreciate it, since beryl is pretty awesome even though it's not working perfectly :D .

### Post by wsmoser2004 on 2007-02-06
I think I've fixed it...I'm not sure which setting actually did it, but I changed 

```

Option "AccelMethod" "EXA"

```

to

```

Option "AccelMethod" "XAA"

```

and also added 

```

Option "AGPMode" "8"

```

to my xorg.conf, and now it works pretty well.  Beryl lags a little bit, especially when rotating the cube, but I guess I shouldn't expect too much from a laptop card with 32MB of memory :) .
