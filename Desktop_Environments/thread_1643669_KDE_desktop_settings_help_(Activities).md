### Post by stayahead on 2010-12-12
Hi,

I'm trying to get to grips with KDE desktop settings with regards to Activities. I understand how they work etc but would like to change some things.

Basically the default behaviour seems to be that if I change activities, the apps I'm running "move across" from one activity to the other. ie: at the moment I have a Google Chrome window open. If I switch activity the chrome window remains and only the activity in the background is changing. For example: from a Search and Launch container to a Desktop container.

What I would like to happen is when I switch activity I get a blank workspace. So what I'm asking: is there a way to keep applications that are opened in one activity to remain "locked" into that activity? 

Not sure I explained that very well. But thanks for any tips on how to configure this behaviour.

---

### Post by stayahead on 2010-12-12
Its ok I achieved the effect I wanted, by ticking "different widgets for each desktop" option. Which seems to lock an app to a desktop. Also allows different wallpapers on each desktop and a different activity per desktop, which exactly what I wanted!
