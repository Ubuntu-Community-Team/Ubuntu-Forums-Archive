### Post by _UsUrPeR_ on 2018-03-06
Hey all.

I'm having some real trouble getting X working on a desktop I'm using for work. I'm running Ubuntu 14.04, and the latest NVidia "nonfree" driver 390.25 downloaded from nvidia's driver web site. The two video cards are a Nvidia GTX 570 and GTX 580. I'm not trying to use SLI or anything due to the hardware mismatch.

Here's the issue:
[LIST]
[*]Xinerama causes hardware accelerated programs like chrome to not display on any other screen aside from the one originally opened in
[*]Xinerama causes click-drag movement of windows to jump to the right 3840 pixels, which makes the leftmost two monitors useless
[*]Without xinerama enabled, two of the monitors on "screen1" (gtx 570) are unusable. I can move the mouse cursor over there, but I can't drag windows over.
[*]Maximizing a window on the working monitors spans both monitors splitting them down the middle. This wasn't a problem when running a single GTX580 card, but with two cards, there seems to be a loss of the ability of X to discern the breaks in the monitors.
[/LIST]

What I *really* like: I would really like this to work like a single X desktop with a 7680x1080 footprint, and some sort of demarcation for maximizing windows so that when maximizing one window would only cover a single monitor. Sadly, it appears that I am unable to have one virtual screen due to the second video card. I would settle for two X sessions, one running on each virtual screen, but whenever I try to start an xsession, it starts on the primary screen. I don't want to use nouveau as a driver because nouveau isn't supported by Steam. Nouveau has been blacklisted.

Any help would be appreciated. I've been at this for days. :(
