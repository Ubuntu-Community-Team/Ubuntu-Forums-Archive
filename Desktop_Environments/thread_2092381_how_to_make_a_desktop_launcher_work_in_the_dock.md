### Post by grashdur on 2012-12-07
I am having this problem with a desktop launcher I created for OmegaT (a translating tool I use all the time for my work): When I click on my launcher icon in a folder (launchOmegaT.desktop), it opens the program just fine with all the options properly loaded. But when I lock the icon to the dock, close the program and then try to open it by clicking on that launcher on the dock, nothing happens. 

I had this problem before, but I just looked thru my notes and see I didn't bother to write down what my solution was.

I'm using Pangolin.

---

### Post by grashdur on 2012-12-07
Solved! Instead of locking the icon when the program is open, do this: 
1. Turn off the dock's auto-hide feature
2. Drag the desktop launcher onto the dock
3. Turn the auto-hide back on

And then it works just fine!
