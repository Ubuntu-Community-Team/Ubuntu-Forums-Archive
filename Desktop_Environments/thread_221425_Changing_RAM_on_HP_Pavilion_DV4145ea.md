### Post by adam.tropics on 2006-07-23
I know this sounds rediculously obvious, but it's happened to me before....make sure it's properly seated in the memory slot. Also, if you can find any documentation for your motherboard, often it will give a beep code of some sort on errors. Finally, I suppose you could try it with one of the other memory modules in as well, and see if you can get as far as grub to run memtest.

### Post by adam.tropics on 2006-07-25
Ok so it seems that you are able to have memory in your machine in any combination upto a max of 2gb. As for the 400-333Mhz, it was always my understanding that most times, the 400 should underclock itself to match the rate of the machine, or the other memory module if one is present. So it would seem that either you have a faulty module, in which case swap it, or it won't run at 333, in which case I'd swap it. Sorry I can't give any better news!
