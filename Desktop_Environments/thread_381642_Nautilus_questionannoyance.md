### Post by tbraun on 2007-03-11
Hello!

There is this annoyance in Nautilus, which keeps bothering me. Maybe someone
can help me with this?

I am using the 'list view', not the 'icon view'. Now let's say I look at a folder's contents
with Nautilus, and I would like to create a sub-folder in there. I can right-click on some
empty space in the folder window and a pop-up appears, which gives me the option
to create a new file or folder right there.

However, if the window is full (because the folder contains lots of files), I can't
seem to find a place to right click into, to get the same menu. If I right-click
anywhere, I will always end up right-clicking onto an already existing file. Thus,
I don't have the option to create a new folder now. I know, there is the 'File'
menu, where I will have the option to create a new folder, but this means that
suddenly I have to work differently, just because the window is full? That shouldn't
be the case,  no?

The same for dragging and dropping a file into a folder. If I have that folder open
already, and there is some empty space in the window, I can easily do that. But
if the folder has enough contents so that the Nautilus window is full, I need to
go up one level, and then drag and drop the file onto the folder icon/name. I
cannot drag and drop it into the folder, if the folder is already opened and there is
no empty space displayed in the Nautilus window (since otherwise I would drag
onto an already existing file).

It seems to me that this is not how it should work. I shouldn't have to change the
way I do things, just because a window is full. There should always be some
'empty' area where I can drag to or right-click into, to get the folder-specific
actions. For example, I should be able to scroll to the end and see at least one
empty line, or some such thing.

Or am I missing something? Is there a way to do this?

Thank you very much...

Tom
