I've been using Ubuntu 9.10 (as part of Mythbuntu) for a few weeks, following an upgrade from 9.04.

When I started, I was using the nvidia 6150 video chip built-on to the Asus A8N-VM-CSM motherboard and an Acer 23 inch LCD monitor with a full resolution of 1920x1080.  As soon as I booted up with the 1920x1080 monitor, Ubuntu knew that it was an Acer monitor and that it was capable of 1920x1080.  

At that point, my xorg.conf file was this:
```

	Identifier     "Configured Monitor"
EndSection

Section "Screen"
	Identifier     "Default Screen"
	Device         "Configured Video Device"
	Monitor        "Configured Monitor"

	Driver	"nvidia"

Section "ServerFlags"
	Option	"DontZap"	"False"
EndSection
```

Today, I disabled the onboard nvidia 6150 chip and added an nvidia 8400GS card to the system (to try out VDPAU).
I moved the monitor cable from the onboard VGA connector to the VGA connector of the 8400GS card and powered up.  Ubuntu booted up and the resolution was down at 1024x768.  I activated the nvidia-settings program and tried to change them to 1920x1080 like they were earlier today - NOPE.  There were no choices higher than 1360x768, and the nvidia-settings did not report the type of monitor (an Acer), as it was earlier today.

What do I have to do in order for Ubuntu to communicate with the monitor again, so that I will get the 1920x1080 choice as before?

### Post by Bunnybugs on 2009-11-14
Did you activate the right driver for the new Nvidia card???

As far as I know, the 6xxx series use another driver then the 8xxx series... (i have an 8400M, my GF has an 6400M) and we use both different drivers...

Go to System>Administration>Hardware Drivers, Remove the current NVidia-driver, and activate the 'Recommended' Driver:)

That should work!
