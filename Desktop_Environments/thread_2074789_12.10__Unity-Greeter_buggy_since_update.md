### Post by Bodifar on 2012-10-28
(strikethrough)I have exactly the same problem.(/strikethrough)

I have a different problem, considering that I couldn't login. The following 2 posts allow you to boot into your system again using lightdm, if you have problems. Don't know if it works for the original problem, which I guess I didn't read properly.  

You can switch DM to GDM (or whatever suits you), that works for me. 

Just run these commands: 

```
sudo apt-get install gdm
sudo dpkg-reconfigure gdm
```

In the resulting terminal dialog, you can switch to gdm. Tip: to get to the "OK-button", you need to press TAB. 
So, when you get the first screen, tab to OK, press enter to confirm. 
On the second screen switch to gdm using the arrow keys, press enter to select, and then tab to OK, and press enter. 
Reboot, and you'll get the GDM login page. 

P.S. If you cannot logon at all, press CTRL+ALT+F1 to do a terminal login.

### Post by Bodifar on 2012-10-28
Found a solution. 

First, you can switch to GDM following my previous post. 

Second, boot into your account, open up a terminal, and run the following command:
```
rm ~/.Xauthority
```
Respond with **y** when prompted to delete the file.

Third, in the same session, switch to lightdm again: 

```
sudo dpkg-reconfigure gdm
```

... and follow the same flow as in the first step, but selecting lightdm as your desktop manager now. 

Reboot your machine, et voilà! \\:D/
