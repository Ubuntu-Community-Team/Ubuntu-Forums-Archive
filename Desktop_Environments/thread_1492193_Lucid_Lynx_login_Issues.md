[SIZE="4"]Solution:[/SIZE]
Replace your etc/X11/xorg.conf file with a new one. I copied mine over from Hardy and it worked on lucid. There weren't many devices specified specifically, so it has many defaults.

I have a Via Unichrome Pro on-board graphics card called. Seems to be called 'vesa' as you'll see.

Thanks to Barafu for telling me what the issue was.

**Some links to get you started:**
[http://www.cyberciti.biz/faq/howto-linux-configure-xorg-x11/](http://www.cyberciti.biz/faq/howto-linux-configure-xorg-x11/)
[https://wiki.ubuntu.com/X/Config](https://wiki.ubuntu.com/X/Config)
[http://ubuntuforums.org/showthread.php?t=83973](http://ubuntuforums.org/showthread.php?t=83973)
[http://hubpages.com/hub/How-to-configure-Xorg-in-Ubuntu](http://hubpages.com/hub/How-to-configure-Xorg-in-Ubuntu)

[Google it]("http://www.google.com/search?hl=en&client=opera&rls=en&q=xorg+configuration+ubuntu&aq=f&aqi=&aql=&oq=&gs_rfai=")

### Post by Khopstick on 2010-05-24
Do you have an NVIDIA or ATI video card? You may have a driver issue.
(You can get to the log-in screen without OpenGL etc., so it only crashes when the desktop starts.)

If you have an NVIDIA card, you may want to check out this thread an see if it applies to you - the fix is very simple.

If you are using a proprietary driver and have an older card, it's possible ATI of NVIDIA discontinued support for the latest X-server release. Then you would have to install the open source driver; that's worth a try, to see if it is a driver issue, and you can do it from the command line (or use 'envy-ng', that's very simple and has a text interface).

Video driver issues can be pretty annoying; happened to me a while ago; I could not use desktop effects for a while, but in the meantime the open source driver caught up and all is well now.

Hope that helps...
Khopstick
