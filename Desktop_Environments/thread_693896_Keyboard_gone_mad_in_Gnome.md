### Post by Vaftrudner on 2008-02-11
Hello all!

I currently have a problem with my keyboard in Gnome. I'm using Ubuntu 7.10 on a Philips 15NB57 laptop. I can't seem to find more detailed information about the keyboard, and I don't know what hardware information could be relevant, however, I'm using a Swedish keyboard layout and a *Generic 105-key (Intl) PC* keyboard model. As I'm sure that you're aware, a laptop keyboard does not have a numpad, but uses Fn-combinations for this (example _[here](http://thecaseagainst.net/wp-content/d610kbd.jpg)_, not actual laptop but you get the idea). I was working in QT (I am a novice programmer) when suddenly the 'U', 'I', 'O', 'P', 'J', 'K', 'L', ';', 'M', '.' and '/' keys started giving their Fn output instead of the original letter. For example, when pressing the 'I' button, I get a '5' as input in all programs, when pressing 'L' I get '3', and so on (see the picture above). Pressing the actual Fn button simultaneously does not affect this. Thinking that the Fn button may have been registered as held down by mistake, I restarted my computer, but in Gnome, the same error is still going strong. It does *not* appear in the login screen, nor in xfce where I'm currently writing this.

Does anyone know what might have happened? Thank you for your time and help in advance!
