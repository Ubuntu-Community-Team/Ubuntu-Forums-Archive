### Post by Dragonbite on 2011-06-06
I had digiKam not recognize my camera.

Now I use Shotwell (ver 2.0?) and it's doing alright.

It allows me to [LIST=1]
[*]Import from my Nikon D90
[*]Option to view only the pictures that haven't been imported yet (simple checkbox)
[*]Allow selective or import all (and recognize duplicates)
[*]Offers to remove the pictures downloaded from the camera (haven't really used it... too paranoid so far :) )
[*]Allows for Importing from file folders
[/LIST]I also like it because it is more visual[LIST]
[*]Events (days) are displayed with a selected picture as the front
[*]Event listings (tree listing) default to the day of the picture, but can be renamed for something special
[*]Tagging is available, but isn't the primary means of categorizing
[/LIST]And yes, there is exporting data. It isn't as easy as right-clicking and selecting "Copy" which can then be pasted into a document/email/etc. like DigiKam provides, but it isn't toooo bad.[LIST]
[*]Publish to PicasaWeb, Flickr, and more (I use PicasaWeb, which is why I don't know what else it offers)
[*]Export selected pictures to a specified folder (handy to pull them aside for placing in emails, etc.)
[*]open in file manager allows at least opening it without having ot navigate to yyyy/mm/dd yourself
[/LIST]

### Post by eric-yorba on 2011-06-06
> **lucacerone said:**
> What I'm really interested in is handling RAW format (.cp2 if I'm correct?)...
Any RAW format supported by libraw will open in Shotwell.

Advanced RAW processing isn't implemented (yet.)  We do have a feature that lets you open in an external RAW processing program, like RawTherapee or ufraw. We're working on improving this a bit for 0.11 and 0.12.
