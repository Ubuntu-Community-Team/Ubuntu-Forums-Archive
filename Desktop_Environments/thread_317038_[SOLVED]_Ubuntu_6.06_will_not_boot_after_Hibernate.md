### Post by JawsThemeSwimming428 on 2006-12-11
I recently had an issue while trying to Hibernate my Ubuntu 6.06 (Dapper) machine. I had never used Hibernate until this point. Today, I had two Firefox windows open, and a OpenOffice.org Word Processor document. Instead of rebooting and having to reopen everything I took the easy way out and chose to use Ubuntu's Hibernate feature. (Just an FYI, I run Gnome) Anyway, I chose Hibernate from the shutdown box and after a few seconds the monitor shut off, however the machine stayed on. I waited a few minutes and the machine was still on, but when I would turn the monitor on there was nothing on the screen. So after a few more minutes I figured it was just hanging so I did the only thing I thought possible and manually shut off the machine. Upon reboot, I was distraught. The machine turned on and loaded the Intel Desktop Board screen which it always loads first, (I have an Intel D101GGCL Desktop Board). But nothing happened after that. The machine was running for a few minutes and remained on the Desktop Board screen. I tried hard-booting the machine again and got the same result. I did not even see an option to get into the BIOS or any type of boot menu. I also tried booting to the LiveCD for Dapper after this and I can not get that up either. This machine runs Dapper only. I am not quite sure what to do. If anyone has any ideas I would be very grateful!!!!! Please help me!

### Post by scrooge_74 on 2006-12-11
Had a similar experience with machines using XP, or with laptops that refuse to shutdown.  Take all power out, if it is a laptop take even the battery out. If it is a desktop, not just shut it down, un plug it so it will loose all energy to the motherboard.

I once had to reset the BIOS so I could get in a make some changes.

### Post by JawsThemeSwimming428 on 2006-12-11
Thanks for the reply, as I was waiting I was trying a few things and I actually unplugged the power supply from the motherboard and that seemed to reset it. Thanks again!
