### Post by beastly on 2006-12-01
Hi all,

I have two TFT screens running off an Intel GMA950 graphics controller. The primary screen is a 20" 1600x1200 (DVI), the secondary a 17" 1280x1024 (VGA).

For the most part everything works fine, my 20" screen shows the Gnome desktop, menu bars etc. The secondary screen is an extension of the desktop (except without the menu bars etc) and I can drag windows between them. That much is great.

The problem arises with the different resolutions of each screen. Both are physically running at their native resolution, however the smaller screen has a "virtual" display of 1600x1200, presumably to match that of the larger display. This means that the visible part of the screen moves around when the mouse pointer is at the edge of the screen (only on the secondary monitor).

I realise this is probably due the the limitations of Xinerama with non-rectangular displays, therefore I can't have one display with height 1200 pixels next to a display with height 1024 pixels using a horizontal layout. However I would at least like to reduce the width of the secondary display to that of the physical screen width - that is 1280 pixels. I see no reason why this can't be done, resulting in an overall spanned display of 2880x1200 pixels, with the only part not visible being 176 pixel rows on the secondary display.

Any ideas how to achieve this? I can post up my xorg.conf file later if it helps (not at my machine just now, and the sshd isn't running).

Thanks.

### Post by beastly on 2007-02-09
Hi

Yes I did get it working just recently actually, and I stumbled across the solution through trial and error.

All I needed was to add this to my **display** section for the second (smaller) screen:

virtual 1280 1024

That forces the second display to have a virtual area of 1280x1024, which also happens to be its physical display size. No more panning around when the mouse gets to the edge of the screen. It works fine with my main display still at 1600x1200, the result appears to be exactly how both Windows XP and Mac OS X handle multiple displays with different resolutions. Much more useful, as I can do stuff like run Windows or FreeBSD in VMWare Player and just maximise it on the second display giving a 1280x1024 Windows/BSD desktop with my main one left over for doing everything else.
