### Post by PhilGil on 2013-03-02
> **zissshh said:**
> 1) Slow Boot of GNOME 2) The login screen to choose Gnome or Unity does not appear,,it gets skipped 3) The Nautilus Mennu Bar issues 4) Optimising my system to give faster process
5) No extensions in Gnome are working 6) I couldnt understand where to put my queries in ubuntu forum,,,coz i had to use it after a long time,,, 7) Iam a using gimp as a graphic artist,, so i need better optimisation,,,pls help me out with theeses issues

1) I haven't found a solution to Gnome 3's slow boot. A good work-around is to Hibernate instead of Power Off. It takes a bit longer for your machine to shut down, but Gnome 3 loads significantly faster.

 2) Do you mean it doesn't appear at all or that Unity is no longer a choice at login? You wouldn't be able to choose Unity because you uninstalled it. If no login screen is appearing at all... First, check that gdm3 is installed. If it is, you may have auto login enabled &#8211; I haven't used Ubuntu for some time so someone else will have to walk you through turning it off.

 3) Can't help you with this one

 4) Ubuntu is tuned for ease of use, not to be the fastest distro out there. I know there have been threads posted here that talk about ways to fine tune Ubuntu for better performance.

 5) Can't help much here, will need more details (they won't install from extensions.gnome.org, they will install but won't activate, they install and activate but don't work, etc.)

 6) You should probably ask about each issue separately (in the appropriate sub-forum) rather than dumping them all into one post.

 7) I have a computer of Core2 Duo vintage myself, and I've found that GIMP brushes can be laggy on any desktop with 3D compositing enabled. If you really want to tune your OS for GIMP performance I'd turn off the shiny and use a non-composited DE (Gnome Classic, XFCE, LXDE, etc).
