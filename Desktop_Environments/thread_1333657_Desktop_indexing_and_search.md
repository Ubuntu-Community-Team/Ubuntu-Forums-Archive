### Post by minaev on 2009-11-21
I'm looking for a good local search engine.

What I want from the indexer is: indexing zipped files, indexing PDF, RTF, Epub, or at least the ability to define new file types.

What I want from the searching interface is: phrase search ("as in Google"), regexp search, stemming support (ideally with the Russian stemming :)), limiting search with a directory tree and file type.

I have checked Google Desktop Search, Recoll and Beagle. GDS is absolutely closed and unextensible. Recoll, when indexing RTF files, extracted a huge number of images from them and put them into my root directory. Now, I'm playing with Beagle. By now, there's only one bad thing – I cannot limit the search with a directory tree, only with one certain directory. Besides, it doesn't understand Epub and I'm not sure I will manage to extract the book text from Epub :).

### Post by HTML-insane on 2009-11-21
I used to use Strigi with KDE 3.5 and was very satisfied with it (unfortunately, the Kubuntu guys have managed to screw up Nepomuk somehow, even though it worked with every other distro I've used). You can install it by installing deskbar-plugins-strigi from a package manager, which is the Gnome client.
