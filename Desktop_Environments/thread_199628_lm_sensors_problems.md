### Post by hayalci on 2006-06-21
The driver module for one of your chips is not present in your current kernel, but from my experience with lm_sensors, it may not be essential.

try loading each module it says using commands like "sudo modprobe w83627hf"
then run "sensors" command. The changes in /etc/modules are applied on your next boot and every boot after that

### Post by a-musing amazon on 2006-06-24
You could try adding i2c-dev *first* in the /etc/modules file before the two you mention. I needed to do this when I got lm_sensors working on my installation. You will need to reboot to check whether this is working.

This is how my /etc/modules file ends:
# Generated by sensors-detect on Sat Jun 10 20:12:15 2006
i2c-dev
# I2C adapter drivers
i2c-isa
# I2C chip drivers
w83627hf

(obviously my adapter and chip drivers are different from yours).
