### Post by royg1234 on 2005-05-07
Solved.  I intended to uninstall all the nvidia stuff, and re-install, all while running the 686 kernel.  So I opened Synaptic, searched "nvidia", and marked for complete removal "nvidia-kernel-common", and then the "additional required changes" dialog came up, and on this list:
[list]
[*]linx-386
[*]linux-restricted-modules-2.6.10-5-386
[*]linux-restricted-modules-386
[*]nvidia-glx
[/list] 
This is the solution, these "restricted modules" (I'm a newb, and have no idea what any of this stuff means) apparently have to be installed in my current kernel flavor so that the nvidia stuff would work.  So from there I:
[list]
[*]install-marked "linux-restricted-modules-686"
[*]copied xorg.conf_386-->xorg.conf
[*]rebooted
[*]NB this xorg.conf_386 was the xorg.conf that was edited by the nvidia configuration stuff (while running under 386 kernel); there's nothing I could find that was 386-kernel-specific in it, it was just the xorg.conf that worked under 386 but not 686...
[/list]
Now everything's working fine.  Plug: I just hope that [screen-jumbled-with-noise problem](http://www.ubuntuforums.org/showthread.php?t=31015) doesn't happen as it did in my old install, for which I haven't done anything that used to trigger it (stayed logged on for a while, ran KDE apps).

### Post by kosmic on 2005-05-07
the easy way was :

in a terminal window :

sudo nvidia-glx-config disable

and then

sudo nvidia-glx-config enable
