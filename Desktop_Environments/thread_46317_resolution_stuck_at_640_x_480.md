### Post by frodon on 2005-07-04
well
In order to be sure to specify exactly in xorg.conf your screen resolution follow these steps : 

1- Go on internet and have look to your screen specifications, particularily on the allowed resolutions : [http://www.lemcomputers.com/kdsvs19.htm](http://www.lemcomputers.com/kdsvs19.htm).
2- Choose between these resolutions the one that you want.
3- Paste the result (in bold) of this command in the monitor section of your xorg.conf file :
I give you an example of the command if you want 1280*960 80Hz resolution
```
gtf 1280 960 80 
# 1280x960 @ 80.00 Hz (GTF) hsync: 80.40 kHz; pclk: 140.22 MHz 
**Modeline "1280x960_80.00" 140.22 1280 1376 1512 1744 960 961 964 1005 -HSync +Vsync**
```
4- Ajust the parameter in bold to fit with hsync: 80.40 kHz and pclk: 140.22 MHz and respecting your screen specification (I put here the exact parameter for your screen)
```
Section "Monitor"
	Identifier	"Generic Monitor"
	Option		"DPMS"
        Modeline "1280x960_80.00" 140.22 1280 1376 1512 1744 960 961 964 1005 -HSync +Vsync
	HorizSync	**30-95** 
	VertRefresh	**50-160**
EndSection
```
5- Modify the screen section :
```
Section "Screen"

DefaultDepth 24

EndSubSection
SubSection "Display"
Depth 8

EndSubSection
SubSection "Display"
Depth 15

EndSubSection
SubSection "Display"
Depth 24
Modes** "1600x1200"** "1280x1024" "1024x768" "800x600" "720x400" "640x480"
EndSubSection
EndSection
```
6- restart xserver (Ctrl+Alt+Backspace)

[QUOTE=colkowalski]I can't modify my xorg.conf file, or change the permissions...how do I log on as root[/QUOTE]

If you want to log as root in a terminal : ```
sudo su
```
If you want to open your xorg.conf as root : ```
sudo gedit /etc/X11/xorg.conf
```
add "sudo" before each command you want to execute as root.

### Post by pezaremba on 2006-06-14
I was having a similar problem after upgrading to dapper.

I went through the xorg logs and noticed it was trying to enable Xinerama even though I didn't have a monitor plugged in to the other port.

At the bottom of my xorg.conf file I noticed that there was a "ServerLayout" section.
I commented out that section by prepending # to every line and it started using 1024x768 on my LCD again.

I see you also have a "ServerLayout" section.  Try commenting it out:

Before:
Section "ServerLayout"
       Identifier      "Default Layout"
       Screen          "Default Screen"
       InputDevice     "Generic Keyboard"
       InputDevice     "Configured Mouse"
       InputDevice     "stylus" "SendCoreEvents"
       InputDevice     "cursor" "SendCoreEvents"
       InputDevice     "eraser" "SendCoreEvents"
EndSection

After:
#Section "ServerLayout"
#       Identifier      "Default Layout"
#       Screen          "Default Screen"
#       InputDevice     "Generic Keyboard"
#       InputDevice     "Configured Mouse"
#       InputDevice     "stylus" "SendCoreEvents"
#       InputDevice     "cursor" "SendCoreEvents"
#       InputDevice     "eraser" "SendCoreEvents"
#EndSection

Hope this helps.
