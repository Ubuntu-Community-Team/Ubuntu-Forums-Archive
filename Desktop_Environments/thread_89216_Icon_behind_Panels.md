### Post by asimon on 2005-11-12
This sounds like a bug. Icons behind the panel should never happen*. As a workaround you can try to change the icon placement mode, right-click with the mouse on the desktop, there is a menu "Icons" or something (sorry, I am on a gnome machine currently and don't remember the exact menu layout) ... 

*) The general bug is already fixed in KDE, the panel now informs the desktop where icons can be placed and where not, but it seems the code which creates icons for mounted media doesn't yet use this new functionality.
