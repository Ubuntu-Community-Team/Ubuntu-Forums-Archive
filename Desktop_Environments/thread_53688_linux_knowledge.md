### Post by cwaldbieser on 2005-08-02
Read everything you can get your hands on!  The Linux Documentation Project is full of HOWTOs and Tutorials.  O'Reilly carries a bunch of great books: "Linux Cookbook", "Linux Server Hacks", "Linux Network Administration", "Running Linux", "Classic Shell Scripting", and tons more.  Other publishers put out some good material, too: "Linux System Administration" and "Moving to Linux: Kissing the Blue Screen of Death Goodbye" are good ones from Addison Wesley.

Then of course, the man pages are built in-- In KDE you can even use the man:/ URL in Konqueror.  Info, too! (e.g. man:/tar, info:/tar).  Of course, many prefer the command line: "$ man tar", "$ apropos usb".

### Post by maruchan on 2005-08-02
Just run Linux, that's the best way to get the hang of it. Fix things that are broken, try to enhance things that already work, etc.

Pretty soon you'll start putting the pieces into place, and you'll see how things fit together. This comes with time. If you see a command you don't fully understand, try typing "man command-name" in a terminal, and read what comes up.

I think most users started this way. Just fooling around. You don't necessarily need to read a book, but it might help speed things up.
