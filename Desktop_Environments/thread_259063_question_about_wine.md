### Post by mikeoh on 2006-09-16
You can only try...  Before installing use 'winecfg' to set up wine, especially the drives.  Also use 'winecfg' to change to a different version (like 95 or 98) if the default doesn't work (2000 I think).  Printing does work in wine, so if your program works than printing should be fine. 
 
Wine is not an emulator, it is only a compatibility layer. The performance is not that much worse then windows (sometimes better).

Good luck and tell us how it turns out.
