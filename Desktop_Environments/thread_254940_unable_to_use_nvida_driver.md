### Post by Fully216 on 2006-09-12
It sounds like a hardware issue.  I would recommend testing the motherboard and components.  You have already swapped out RAM, so that is one down.  Unfortunately, MBs are too complicated to test outright.  Using the nv driver, you could try running different utilities.

Sisoft Sandra can be used for an overall test/benchmark, Prime95 for testing the new processor, Memtest is good for RAM (although sound like it is okay), and USBReady is good for USB ports.  Lucifer is one of the tools on the Ultimate Boot CD.  It's an obsolete burn-in program, but some folks still like to use it.  There is also PCMark05 suite [http://futuremark.com/products/pcmark05/](http://futuremark.com/products/pcmark05/) which will run benchmarks.  

If your computer is really 5 yrs of so old, it might be a power supply issue.  I think at this point, it might be a long, arduous process to narrow down the error.

### Post by ls8 on 2006-09-20
There is a well documented incompatibility of ASUS K8V-X SE in Linux.

CMOS Checksum Bad after (almost) each restart: fixed by the latest BIOS v.1402 ( [http://support.asus.com/download/download.aspx?modelname=K8V-X%20SE&SLanguage=en-us](http://support.asus.com/download/download.aspx?modelname=K8V-X%20SE&SLanguage=en-us) )

Xorg freeze at startup: vesa driver works, drivers ati, radeon and fglrx dont. Somebody reported that it can be fixed by changing some BIOS settings: AGP 8x to 4x, AGP FastWrite enabled to disabled. Somebody else reported it can be solved by disabling DRI in Xorg.

Same problem occurs on freebsd: [http://lists.freebsd.org/pipermail/freebsd-i386/2005-November/003240.html](http://lists.freebsd.org/pipermail/freebsd-i386/2005-November/003240.html)

[http://www.fedoraforum.org/forum/showthread.php?t=93478](http://www.fedoraforum.org/forum/showthread.php?t=93478)
