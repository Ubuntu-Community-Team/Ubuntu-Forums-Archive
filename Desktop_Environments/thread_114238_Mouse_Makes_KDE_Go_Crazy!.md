### Post by tmccollum on 2006-01-08
I added KDE and XFCE to my options for window managers to check out GNOME's competition. XFCE works fine. However, in KDE, as soon as my mouse gets to the bottom of the screen to check out the taskbar, the screen seems to go nuts and starts scrolling rapidly both vertically and horizontally, depending on how I move the mouse. A mouse cursor is suddenly nowhere to be found, but right-clicking pulls up the standard "right-click on panel" menu, which I can move through using the arrow keys, but that doesn't help me much. I seem to recall having to install the i810 drivers for GNOME. Can someone tell me if I will have to do this again for KDE and, if so, point me in the direction of a HOWTO?

Thanks for any help! :D

### Post by tmccollum on 2006-01-10
Just as an FYI to anyone who may run into the same problem, I was able to solve this by simply changing the screen resolution through the right-click menu before moving the mouse anywhere near the edges of the screen.
