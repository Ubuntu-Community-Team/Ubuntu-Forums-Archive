### Post by Anduu on 2006-07-17
Yup I ran into exactly the same problem with fglrx and the big desktop.

Try adding this to you xorg.conf

```
Option	    "EnablePrivateBackZ" "yes"
```

Add it to your Device section...the one where fglrx is specified...like so

```
Section "Device"
	Identifier  "ATI Graphics Adapter 0"
	Driver       "fglrx"
	Option	    "VideoOverlay" "on"
	Option	    "OpenGLOverlay" "off"
	[COLOR="Red"]**Option	    "EnablePrivateBackZ" "yes"**[/COLOR]
	BusID       "PCI:1:0:0"
EndSection
```

Reboot

### Post by BlackwaterPark on 2006-07-18
I finally got it!

It ended up being a problem with my monitor settings in xorg.conf, I added the highlighted lines to the monitor sections in it:

```
Section "Monitor"
	Identifier   "VX2025wm"
	[COLOR="Red"]**DisplaySize  592 210**[/COLOR]
	HorizSync    30.0 - 82.0
	VertRefresh  50.0 - 75.0
	Option	    "DPMS"
EndSection

Section "Monitor"
	Identifier   "aticonfig-Monitor[1]"
	**[COLOR="Red"]DisplaySize 592 210[/COLOR]**
	Option	    "VendorName" "ATI Proprietary Driver"
	Option	    "ModelName" "Generic Autodetecting Monitor"
	Option	    "DPMS" "true"
EndSection
```

I am running a 1680x1050 monitor + a 1280x1024 monitor, that ends up being a 2960x1050 desktop. 

```
	DisplaySize  2960 1050
```

That will make video play back at the right aspect ratio, but some text will be ultratiny (some sort of function of DPI). If you reduce the ratio down as much as possible, you'll get 296x105.

```
	DisplaySize  296 105
```

That will make that same text that was supersmall before, superbig. So, just multiply back up by 2 and you'll get 592x210

```
	DisplaySize  592 210
```

That will make all of the DPI-dependant stuff look right and the video aspect ratio is still right.

Right now, the only (very minor) issue is that video in totem-xine won't play on both monitors simultaneously, but it will play on either one independently. Which is something I can deal with :)


Thanks for the help!
