The default Ubuntu installation has 4 virtual desktops. I use them to the full capacity; indeed, I use up to 8 desktops normally.

Usually I have a terminal open in each desktop from which I launch GUI applications.

My question is, how can I constrain GUI applications to the specific desktop from where I launched them?

For example, I'm in Desktop A. I launch Application A. While waiting for it to load, I switch to Desktop B and start to code in Emacs (for example). Some time later, App A pops up on Desktop B, steals the focus, and since I was coding at that time I start to type glibberish into App A. 

One limitation is that the applications I'm launching sometimes require waiting for licenses, so it could be several minutes between the typing of the command and the opening of the GUI, while it communicates with the license server.

Ideally what I'd like to do is go from Desktop A to Desktop D in order, launching 4 instances of this application, the go to Desktop E and do stuff while the applications pop up in the background in their respective desktops.

Is there any way to accomplish this? Or am I forever doomed to have applications pop up unexpectedly in my current workspace? :-k
