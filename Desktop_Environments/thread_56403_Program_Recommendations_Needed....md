### Post by Mishura on 2005-08-13
> I used to use EAC to rip and encode my CDs, and whilst programs like Sound Juicer work well enough it appears they're very limited in what you can do. Ideally I'd like just to rip a CD to .wav, and then batch encode them, instead of ripping and encoding at the same time. Even though I am a KDE user, I use Grip which is a GTK2 app.  It has a lot of options, but the user interface leaves something to be desired...  You can also use Konqueror to rip to wav, ogg and mp3 (with lame installed).  Insert audio CD and browse Konqui in File manager mode.  Look under devices and you'll see Audio CD.  Just drag the wav folder to where you want it.  Keep in mind, that it rips slow...  which is why I use Grip.. its faster.

> 
I think MP3 Tag Tools is by far the best tagging program available. It can derive Tags from any filename, supports batch processing, can load files from playlists, can mass edit just one tag element (ie album information) and much more. I've tried Cantus, Cantus3, and EasyTAG, but none of them come close.

Hmm.. I use amaroK to edit tags.. even though its a bit overkill since its a audio player with a rather insatiable CPU cycle hunger..  If you want an entire album tagged, load up the audio files, SHIFT+click all the files and right-click Edit tags and edit the things that relate to all the files (Mostly:  Artist, album, and genre) but you'll have to edit song tags individually.  Not necessarily the best, but it works.  EasyTag is good, but I can see you've already tried it.

Just some ideas...
