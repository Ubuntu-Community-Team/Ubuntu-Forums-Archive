### Post by David Marrs on 2005-10-18
I don't think I'm revealing extreme ignorance by admitting that I'm a web designer and I've never even heard of VML. I'd write an email to the webmaster asking for an internet ready website. Just looking at the opening tag, it's clear that this site has been designed for an *intranet* situations where all users can be guaranteed to have the necessary software installed. It certainly shouldn't be on the www. From the code, it looks like it was auto-generated by Powerpoint.

Web sites should really only use accepted internet standards, like standard HTML, XHTML, jpeg, gif, png, and so on. I wouldn't even put SVG on the web, let alone VML.

PS. It just occurred to me that OpenOffice will export Powerpoint presentations to Flash, which is a much more suitable format for the web. If powerpoint doesn't support this option, then maybe they could republish the slides using Impress.
