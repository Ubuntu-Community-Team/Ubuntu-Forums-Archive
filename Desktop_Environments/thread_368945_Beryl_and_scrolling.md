### Post by sloggerkhan on 2007-02-23
My beryl settings are fairly typical. 
Some actions and FPS impact:

Scrolling wildly back and forth: eventually drops FPS to 10-15, will keep scrolling for a while as screen gets cot up with actions. (so 15-50 fps hit, depending)

moving mouse back and forth over screen: 1-2 fps drop

moving mouse back and forth over 3 windows: 20-30 fps drop until mouse still

(have focus set to follow mouse. Changing single window focus focus causes about 5-10 fps drop for a moment.)

Typing this message: FPS drop in proportion to typing speed. 0-7 fps drop

Create new window: momentary 5-15 fps drop momentary

Closing window: 5-15 fps drop momentary

Waving a window back and forth: 0 fps drop.

Cube free rotate: constant 32 fps (actual fps, so about 30 fps drop)

maximize/minimize: 0-10 fps momentary drop

alt-tab: no fps drop (maybe 0-1)

super-tab: intitial short 3-5 fps drop, then constant fps are regular (about 60 fps) framerate until window selection made, then another short 3-5 fps drop

Open/close applications menu: 5 fps drop momentary


So scrolling causes possible worst FPS hit out of any activity.

Beryl settings, specs:
ati x700 mobility 128 meg, using ATI open source driver, AIGLX, 1680x1050 res on 60hz lcd, framerate limited to about 60 fps

Using standard beryl with snapping windows instead of wobbly windows. (mostly cause I got annoyed with wobbly windows and switched to using a more productivity oriented setup.)

So pretty much I maintain 30+ fps unless I scroll wildly....
