### Post by harold4 on 2008-02-27
The following does not give front/back in Konqueror, but does work in Firefox/Swiftfox.  Not sure about Nautilus.

```

Section "InputDevice"
        Identifier      "Configured Mouse"
        Driver          "mouse"
        Option          "CorePointer"
        Option          "Device"        "/dev/input/mice"
        Option          "Protocol"      "ExplorerPS/2"
        Option          "ZAxisMapping"  "4 5"
        Option          "Emulate3Buttons"       "true"
        Option          "Buttons"       "7"
        Option          "ButtonMapping" "1 2 3 6 7"
EndSection

```

### Post by der_joachim on 2008-02-28
Pasted below is the config for my mouse, copied from my xorg.conf.  It's a Logitech MX440. Note the "evdev" driver, which is apparently specific for Logitech.

There's a number of HOWTOs for the many many Logitech mice in the HOWTO section. If the snippet below does not work, try reading one of these.
```

Section "InputDevice"
	Identifier      "Configured Mouse"
	Driver          "evdev"
	Option "evBits" "+1-2"
	Option "keyBits" "~272-287"
	Option "relBits" "~0-2 ~6 ~8"
	Option "Pass" "3"
EndSection

```

Backing up the xorg.conf means you can put it back to the way it currently is ;)

```
sudo cp /etc/X11/xorg.conf ~/xorg.conf.bck.pre518
```

then

```
sudo dpkg-reconfigure -phigh xserver-xorg
```

Add the mx518 confi to /etc/X11/xorg.conf, then restart X

If the mx518 buttons are working, there is an issue with your backed-up xorg.conf, otherwise it's something else and just do 

```
sudo cp ~/xorg.conf.bck.pre518 /etc/X11/xorg.conf
```

restart X and you are back to where you were.  Working dual monitor and broken mx518 :-P
