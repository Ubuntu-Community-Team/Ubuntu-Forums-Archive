### Post by benhall on 2007-08-27
Just to add to what has been already said, they add to both the look and feel of the desktop in terms of behavior of individual windows and responses to different input. As such, they can have different resource requirements (xfce being lighter than Gnome or KDE), respond differently to common gestures (last time I used KDE it defaulted to using a single click to open a folder, whereas in gnome I do this with a double click, though this may reflect the default settings used for that distro)  and may be differently configurable (key here is that an explicit aim of Gnome is to minimise the configuration options available to make common choices simple and is hence difficult to do for non common tasks, whereas it is, as far as I am aware, easier to do with KDE). The desktop environments are also associated with different libraries of widgets which lead to sometimes poor integration of components built for one with the other - I use a program called Kile (built for KDE) in Gnome, and I'm aware that the icons look slightly out of place and I believe it starts slower than in KDE.

Personally, I recommend Gnome or Xfce. I do not have a specific view on KDE, but I have never felt a need to move away from either of these. I tend to use Xfce for remote sessions and when I'm doing something resource intensive and I want the programs I'm using to have the maximum available to them (audio editing), and Gnome as my default desktop. I have also used blackbox as well in the past as an extremely resource light window manager, but this has a steep learning curve. Peoples viewpoints can differ strongly though on preferred window managers, so if you are interested try different ones out; if not, I would just use the default which is installed. None of the common ones are bad (in my opinion).

### Post by benhall on 2007-08-28
Gnome is installed by default in an Ubuntu installation, KDE by default in a Kubuntu installation and Xfce by default in a Xubuntu installation- you do not get an option to install the alternatives. After installation you can install them using synaptic; look for packages with the names ubuntu-desktop, kubuntu-desktop or xubuntu-desktop.
