### Post by sal on 2005-08-23
I edited the ID3 tags of about 1,000 tracks (MP3 Format) in a windows only application called MediaMonkey ([http://www.mediamonkey.com/)](http://www.mediamonkey.com/)).

After I synchronized the ID3 Tags against the files in MediaMonkey, I decided to load them into WinBlows Media Player v10 just to see if the tags where appended to the files before I loaded them on to my media player.

WinBlows Media Player v10 only got the ID3 tags of 258 files out of 1,000. So, I got concerned for a moment. Then I told myself **** it!, let me load them on to my Ubuntu Box and see what would happen because we all know how “Standards Compliant” Micro$cum has been threw out the years.

Here are the results of my Tuesday morning experiment:

1,000 MP3 files

Windows Media Player v.10 (Window$ XP Pro $P-2)
Recognized the ID3 Tags of 258 files

Rhythmbox v.0.8.8 (Ubuntu Linux 5.04)
Recognized the ID3 Tags of  all 1,000

Juk v.2.2.1 (Ubuntu Linux 5.04)
Recognized the ID3 Tags of all 1,000

AmaroK v.1.2.4 (Ubuntu Linux 5.04)
Recognized the ID3 Tags of all 1,000

So,  this I find funny for a few reasons.

The Windows only application I used (MediaMonkey on Window$ XP Pro $P-2) was able to use standard ID3 tags to append to the mp3 files. Microsoft's own Media Player was not able to grab the ID3 tags from the majority of the tracks even though the files where edited on the same system. I would also like to mention at this point that, The Window$ XP Pro $P-2 box I used had an NTFS files system with Indexing on.

Now, what I plan to do next, is to edit the ID3 tags of the same tracks in Ubuntu Linux 5.04 and then transfer them over to the Window$ box and see what both Window$ Media Player v.10 and MediaMonkey report for the ID3 Tags.
