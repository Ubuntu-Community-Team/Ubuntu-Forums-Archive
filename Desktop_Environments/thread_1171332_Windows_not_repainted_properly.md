Hello,
this thread is about a problem I'm currently having with Kubuntu 9.04, but it happened with Ubuntu 8.10 and 8.04 as well, so it's not a Kubuntu-only issue, though it seems to happen a lot more frequently now.
The problem is as follows: Whenever I keep a window open for an extended period of time (from 15 to 30 minutes), I get redrawing problems. This happens especially in the Terminal (Konsole in this case) and in Firefox. In the former, after typing a few commands, the window doesn't get redrawn, so I still see the command I've entered hanging in there on the last line, looking as if it hadn't started doing its magic yet. It's funny when that command is a cd or something of the kind, because you expect it to be instant, right? To force the window to be repainted, I either have to resize it, or to highlight some text in it. If I do this, only the text I have highlighted is refreshed, while the text I haven't still shows what the Terminal showed before the drawing problem.
In Firefox, when this problem happens, I can still see the complete Firefox window, and my cursor turns into text mode whenever I'm hovering some text or a textbox, it turns into the "waiting" mode when there's a page loading, etc., but clicking anywhere doesn't give any visual feedback, which is confusing. I can load a page by clicking in the address bar and typing it, but the Firefox window always stays blank, doing nothing, until I resize it to force it to be redrawn.
Only workaround I've found to fix it is a quick resize, or restarting the program in question.
I'm using compiz-fusion from the repos, Kubuntu 9.04, NVIDIA GeForce 9800 GT with the latest (180) proprietary NVIDIA drivers. Single screen, 1920x1200, 50 Hz (although Windows lets me go up to 60 Hz at that resolution... :?)

Edit: Also happens often in Kate. Does not happen in 3D (games) windows though.
