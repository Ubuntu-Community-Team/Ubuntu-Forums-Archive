### Post by tomski on 2007-05-25
looks like i was close!!!!

i basicly had to:

backup my folder
log out & back in as a different user
open users & groups in admin menu
delete my username (the one with the issue)
delete my folder in /home/
reboot or telinit 1
log in as different user
open users & groups in admin menu
add my user name again
log out & back in as my new user name to create all gnome files
log out & back in as different user
move all backed up files to new home folder
open terminal
and enter the following ( [thanks to this thread]("http://ubuntuforums.org/showthread.php?t=371052&highlight=dmrc") ) just in case we end up in permission hell!!! as i did with .dmrc & home folder

```

sudo chmod 644 /home/**username**/.dmrc
sudo chown **username** /home/**username**/.dmrc
sudo chmod -R 700 /home/**username**
sudo chown -R **username** /home/**username**

```

change **username** for the one you have been having issue's with and all should be ok
I know this is probably the long way of dealing with this issue because like most things linux there is allways another way to accomplish a multitude of tasks but hey i have been searchin the web to try to fix this & this is what i have done and it worked.

If you have another solution to this problem or a quicker way of getting out of this mess then please post it

thanks to those who do reply to this
