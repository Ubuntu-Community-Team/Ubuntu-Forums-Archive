### Post by djmadkins on 2005-08-17
to provide some level of protection for your SSH you may want to look at using publickey authentication.. check in the "How-To's" for info on that.

Also you may want to get DenyHosts from Sourceforge. This is a python based program you use to analyze your server logs and automatically add entries to /etc/hosts.deny . This will block the IP's attacking you from connecting via sshd (or ALL services if you choose) for any length of time you choose. It can also look at the hosts.allow file for specific ip's (or ranges of ip's) to ignore in case you or your friends make mistakes logging in.

I run denyhosts every 2 minutes because even tho I use publickey authentication and have disabled keyboard-interactive I still get the script kiddies trying to dictionary my system.

The other entries are most likely cron related.. The system launching cron for the hourly, daily, weekly and monthly directory entries.

### Post by mtelewicz on 2005-08-17
Dictionary attack indeed.  I started having serious problems with dictionary attacks on a Mandrake server I was running.  To echo the above, I highly recommend using public key authentication.  It is much more secure.  Just type "openssh key authentication" in google and you'll get plenty of information.  It is pretty easy to do if you basically know your way around Linux.

Also, you should take advantage of AllowUsers in /etc/ssh/sshd_config.  Only allow the users you want, that way nobody can get in through some strange user account you didn't even know about that has a weak password or no password at all.  Type "man sshd_config" for more information.

After I started using these two things, attacks on my ssh server basically stopped or were completely ineffective.
