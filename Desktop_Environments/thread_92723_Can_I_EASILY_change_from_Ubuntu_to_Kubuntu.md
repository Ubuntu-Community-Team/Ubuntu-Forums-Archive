### Post by aysiu on 2005-12-13
[QUOTE=subscrive]When I do "sudo apt-get remove xyz", am I removing the setup files or removing the complete programme xyz?[/QUOTE] It depends. For **most** programs, *sudo apt-get remove xyz* will, in fact, remove xyz (not the setup files, just the program).

However, ubuntu-desktop, kubuntu-desktop, and xubuntu-desktop are not programs. They are what are called "meta-packages." When you install ubuntu-desktop, you're not really installing anything directly. By marking it for installation, you're marking a whole slew of programs to be installed.

When you mark ubuntu-desktop for an uninstall, *nothing* really gets uninstalled except the pointer--all the programs it originally installed will still remain.

apt-get clean is the command that gets read of the setup files.

If you don't believe me, do a fresh install of Kubuntu (something which I took the pains to do--I'm not just talking theory here), and do a ```
sudo apt-get install ubuntu-desktop
``` Take careful note of all the programs that get installed. Copy and paste them to a text file, if you want.

Then, do ```
sudo apt-get remove ubuntu-desktop
``` and then check how many of those programs that you installed get removed.

### Post by aysiu on 2005-12-13
I have [url=http://www.ubuntuforums.org/showthread.php?t=96048
]a HowTo that is generic and will help you uninstall the kubuntu-desktop packages[/url] (*sudo apt-get remove kubuntu-desktop*, as you may have gathered from this thread, will **not** work).

However, if you know in advance that you want a safeguard, the best way to do this is to open a terminal, type ```
sudo apt-get update
sudo apt-get install kubuntu-desktop
``` but before you say yes to installing everything, highlight all the packages that are going to be installed and copy them to a text file and save the text file.

Then, say "yes."

If you later decide you don't want Kubuntu, open a terminal and type ```
sudo apt-get remove
``` and paste in all the packages from your saved text file.
