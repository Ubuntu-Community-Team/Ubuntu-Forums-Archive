### Post by joscar on 2005-10-13
you need t make sure the kernel headers are installed for the kernel you are using. do a : uname -r and then go synaptic and grab the headers that match that kernel version. Also make sure the linux-restricted modules for the current kernel are installed. Once these dependencies are satisfied, simply do an apt-get install xorg-driver-fglxx. that will do the complete install. next and last step is to do a :  gedit /etc/X11/xorg.conf and under modules make sure the driver is fglrx. if not chaange it and save. now when you do fglrxinfo you will get the ATI response you seek.

### Post by andrego on 2005-10-13
Before you compile a custom kernel (which may, or may not be, a good idea...I find stock works fine!), you'll probably want to know:

1) There are pre-compiled fglrx (ATI driver) kernel modules for the stock kernels in Ubuntu.  This means you don't have to do anything like compile custom kernel modules to use 3d acceleration, if you use a stock kernel.

2) The 686 kernel will be totally fine for your needs.  Custom kernels for K7 won't yield any significant speed gains for everyday use.  However, feel free to 'apt-get install linux-k7' and reboot, if you want to ;)

3) Installing linux-restricted-modules-686 (or, k7 if you so desire) will get you the fglrx kernel modules.

So, technically if you followed the above, all you *should* have to do is 'sudo fglrxconfig' at the command line, enter the password for *your* user account and walk through the instructions. (In Ubuntu, there is no root account enabled for login by default.  So, you can use "sudo" to for administrative access.  It might not have worked before, because you didn't have permissions to access/write to the files.)

Also, as others mention, XF86Config-4 is obsolete.  It is for XFree86.  Ubuntu Breezy ships with the forked (and much improved!) Xorg version of the X Window System.  xorg.conf is the Xorg equivilent of XFree86Config-4.

Hope this helps - I know what a PITA the fglrx drivers can be to get running, so I'll stick around and see how it works out for you!

P.S. If you followed the other guides (compiling kernel modules, etc...you might just need to REMOVE the linux-restricted-modules-xx packages.  That fixed it for me, after a day of pulling my hair out a few months back).

EDIT: mlomker says it better than me: [http://www.ubuntuforums.org/showthread.php?p=408111#post408111](http://www.ubuntuforums.org/showthread.php?p=408111#post408111)

### Post by joscar on 2005-10-15
I must have done about 10 installations of ubuntu and I have yet to fail at getting this to work. If you follow these directions you should have this working. I will do a step by step for you of what got this working for me. Follow exactly please. do as root

1. After a clean install enable all necessary repositories. 

2. apt-get update and reboot

3. go to synaptic. System - Administration - Synaptic Package Manager.

4. do uname -r to verify the current kernel

5. search for and install : fglrx-kernel-source,  linux-headers and linux-kernel-headres for the kernel version uname -r gave you.now reboot 

6. apt-get install xorg-driver-fglrx

7. gedit /etc/X11/xorg.conf   and change the driver under device to " fglrx "

8. at this point log out or reboot . doing a fglrxinfo should tell you the ATI drivers are there. This is exactly how i have done it many times so give this a try after a clean install and you should be ok.
