### Post by hizaguchi on 2006-08-31
I kinda float back and forth between Gnome and KDE.

KDE has all that functionallity and integration between apps, and that makes the user experience feel very elegant and efficient.  Everything uses the same libraries and so once you've opened Konqueror once everything else just snaps right up, ready to go.  Plus Katapult gives me the convienence of a ton of launchers without eating up the screen space.  And I use Kate constantly... it's love.

Gnome, though, has a different kind of appeal.  The DE itself, along with most of the gtk apps, is not so integrated and lacks some handy features that I use often in KDE.  When I switch to Gnome I feel crippled for the first week or so.  However, after I get used to it I enjoy the fact that 90% of what I want to do is available in a few simple buttons, and the interface is delightfully uncluttered.  The themes are also very smooth and easy on the eyes, with gradients and plain colors instead of fancy pixmaps.  It's a more relaxing environment.

I love em both, but if nautilus had just a few of the Konqueror features, and there was a gtk amaroK and Kate, I'd say Gnome all the way.

### Post by donkyhotay on 2007-01-03
I started with KDE when I first started using linux because it was the default of the first distro I tried (mandrake 7.0). I gave up linux for awhile (call of the games was too much) but when I went back I used FC4 which defaulted to gnome. I messed around with both and there were things I liked about both. As a windows user transitioning to linux I found KDE to be more "friendly" however after becoming more familiar with linux I found gnome to be cleaner and more efficient. Specifically KDE has multiple programs built into it for system administration (similar to windows control panel) while with gnome these programs are seperate and not part of the desktop itself. Maybe it's changed but that to me has always been the primary distinction between KDE and gnome. Since I prefer having my programs as seperate as possible from my OS/desktop I use gnome. This helps keep the system efficient as unnecessary programs can be uninstalled without affecting my entire desktop, also if a program crashes it's less likely to affect my desktop.

For those who have said such things as IceWM and Fluxbox: those aren't desktop environments, but mere window managers.
A desktop environment, is just that, a single environment for your applications to live in. The two biggest ones are KDE, which uses Qt, and GNOME, which uses GTK.
A window manager is just one application, in fact, KDE and GNOME both use window managers to manage the windows in their created environment.
