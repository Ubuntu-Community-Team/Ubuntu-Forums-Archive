### Post by john5 on 2006-11-05
In upgrading to Edgy I can't work out how to type in a path in the gnome/gtk file chooser.  In the past, you could hit ctrl-L and it would open up a path dialog where you could type in with auto-complete.  But now if you hit ctrl-l  it clears the default filename, which is really frustrating, because then you have to remember what the filename is and type it all in again.  

Anyone know some way around this?  Being able to type in the path without losing the filename?

---

### Post by john5 on 2006-11-07
After trying all sorts of keystrokes, I realised that the cut/paste behaivour can offer the solution to this problem.  Instead of hitting ctrl-l for location, instead you pres ctrl-x to cut the filename to the clipboard.  Then  the path entry works as normal with the auto-completion.  Once the path is in, then ctrl-v can restore the filename component.
