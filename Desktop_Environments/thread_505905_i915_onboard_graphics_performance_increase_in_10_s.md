### Post by richardq on 2007-07-20
I've been having intermittent problems with GUI performance in Metacity (and Beryl). And while I likely haven't solved that exact problem yet, I did manage to find out something very very useful along the way (after having this machine for over 2 years!)

I have onboard Intel i915 graphics. This uses shared RAM. The default on my machine was 8MB. In two years of running Linux and trying to tweak my system and solve various performance related problems on forums like this one, I NEVER realized that my bios would let me increase this amount.

I changed it to 32MB (the max allowed in my bios) and the performance increase all around the desktop is very very signficant. 

So if you've got onboard video using shared ram, check your bios to see if you can increase it. It made a world of difference on my system. 

Sorry if this is a well-known fix. Maybe I'm just an idiot for never knowing it. ;)
