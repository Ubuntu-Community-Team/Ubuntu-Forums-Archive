### Post by cyclingman on 2008-01-19
Normally, when you are running a program such as Pidgin, or Azureus, when you close the program, it remains running in the background on the top panel.

However, recently, I right clicked on pidgin, while it was running in the background on the panel, and click "Remove from panel" and then the program shut down, and it disappeared. And now, whenever I run a program,  which by default runs in background when you click the "X" button at the top run hand corner (e.g. pidgin, azureus, etc.) it closes completely, and doesn't run in the background.

When I right click, under "add to panel" I can't see the option to add any of these programs. Is there a setting somewhere that I can change? HELP! (please) I'm sure this is a very simple and obvious problem (I'm a beginner)

---

### Post by mcduck on 2008-01-19
You just removed the Notification Area-applet from the panel. That's where all programs put their notification icons.

Right-click on the panel, select 'Add to Panel', find the Notification Area-applet and drag&drop it into your panel.
