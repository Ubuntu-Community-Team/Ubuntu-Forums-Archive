### Post by almahtar on 2006-11-01
Beryl is not seeing any displays that have compositing.  There are a number of things that can cause this.

Sometimes you need to reinstall libmesa-gl, copy its libGl.so somewhere (like /opt) and pre-link it when you load beryl.  There are walkthroughs on the Beryl forums for that.

It could be a driver issue - you should verify that your 3D is working because sometimes the process of tinkering with XGL, AIGLX, and Compiz/Beryl can break a thing or two.

The best place to go for help with this is the beryl forums, but I do have some advice that'll help you tinker with less tedium.  Open the "Beryl Settings Manager" (in System->preferences), right click the red gem icon in your tray and make sure "Launch fallback window manager if beryl crashes" is checked, then right click it again, go to "select fallback window manager" and make sure metacity is selected.

That way if beryl tanks on you, at least you won't have to restart X.  Sorry I don't have a solution for you - I'm still pretty new to this stuff myself.

### Post by Beowolf on 2006-12-19

I believe the flashing window border means that there are more than one instances of emerald working in the background and they keep replacing each other. Try killing all instances by
```
killall emerald
```
and then restarting with
```
emerald &
```

I hope it helps.

### Post by jbuberel on 2007-03-25

```
Section "Module"
    Load           "i2c"
    Load           "bitmap"
    Load           "ddc"
    Load           "extmod"
    Load           "freetype"
    Load           "glx"
    Load           "dri"
    Load           "int10"
    Load           "type1"
    Load           "vbe"
EndSection
```
