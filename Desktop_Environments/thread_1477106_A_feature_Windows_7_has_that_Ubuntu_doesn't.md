### Post by N00b-un-2 on 2010-05-08
I've been using Ubuntu since Gutsy Gibbon and I have watched it grow and evolve a lot in the last three years.  Ubuntu, in my honest opinion, while not being a "perfect" Linux distro (since Linux is a very fluid concept, it's safe to say that there is no perfect distro), is by far and away the best one around.  The quality of work put into every new release is evident by Ubuntu's success.  Sure, sometimes there are some growing pains as the kernel evolves or changes are made to Xorg. Like almost every other Linux user, I would throw a fit when I couldn't fix something and I've done my fair share of distro-hopping, but in the end I keep on coming back to Ubuntu.
One thing that I find though when trying to use proprietary software is that the packages are often very outdated and often require some amount of tweaking in the terminal to make them work, or sometimes just won't work at all.  Say what you will, but there are many very useful apps out there that are not necessarily in the repos.
On Windows 7, I've found that when a program will not run natively, you can always run the program in compatibility mode (typically it just defaults to Windows XP SP2 mode) and otherwise useless software becomes usable again.
I realize that the way that Linux works, to implement this type of functionality would likely mean that you would need to basically have a whole other operating system "sandboxed" on your hard drive, and to include multiple compatibility modes would mean multiple sandboxes.
I just wish that there was a better way to do this.  Maybe some guru out there will figure out legacy software compatibility for Ubuntu.
