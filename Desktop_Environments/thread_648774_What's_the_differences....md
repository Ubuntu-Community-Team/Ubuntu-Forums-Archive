### Post by vek on 2007-12-27
They are actually very different under the hood.

On top of the usual linux layer, at least.

For example the gnome system uses gstreamer for media handling and playback, the KDE can use it, but also uses other engines.

The KDE system uses QT, the Gnome system uses GTK (But both systems can run each others widget apps).  This leads to differences in look and feel, and the way hotkeys work, and so on.

They use different underlying messaging interfaces, and KDE uses different url handling type stuff.  A lot of pipework is very different underneath the hood, but a lot of it is worked in such a way as to present the same kind of interface, despite its differences.  

They also have a different philosophy.   Gnome is one of simplicity, and minimalistic design and uniformity (it seems), and KDE is one of customizability perhaps at the cost of simplicity, and sometimes it loses uniformity because of that too.  There are genrally more knobs and dials in a KDE application, which can be intimidating and confusing, but at the same time, the problem with removing those knobs and dials is there comes a time when you need to adjust them... and in Gnome they might not even be there.  Config file editing time (sometimes), other times, its just not possible to tweak those things.

So theres pros and cons either way.  I find that for what I need to get done, Gnome does it just fine, but I end up using quite a few KDE apps inside Gnome because it simply lacks the configurability.  Sometimes, sacrificing it in the name of simplicity is taken too far, and you end up losing functionality rather than just getting simpler.

It all depends on the skill level and the desire to customize for an individual.
