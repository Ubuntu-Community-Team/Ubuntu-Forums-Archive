### Post by snowpine on 2011-07-01
> **Yuhed said:**
> Just a faster system to do normal stuffs in this old PC, like watch videos(flash is really slow, even after all Optimization tips, etc.

Adobe Flash Player is proprietary "closed source" software. It is exactly the same in every Linux distro because nobody can edit the source code to "optimize" it.

Does your computer meet (or preferably exceed!) the [hardware requirements published on Adobe's site]("http://www.adobe.com/products/flashplayer/systemreqs/")? What is your graphics card chipset?

> **Yuhed said:**
> And it slow down all system, haveing to restart to back to normal), use some programs that require more ram from system, etc.

You can use the terminal command "top" to check what's using your RAM. You may find that the heaviest RAM users are not the basic system itself (the "distro" and its desktop environment) but rather individual applications (like Firefox). In that case the best way to improve performance is to use different applications that use less resources.

### Post by 3Miro on 2011-07-08
For speed, you first look at the RAM. You need a system that can fit entirely in the RAM without touching the swap. The gradation usually is: LXDE < XFCE < GNOME + No Effects < GNOME + Compiz/Unity < KDE.

Once the system fits in memory, it all becomes a matter of CPU and GPU. No effects should be faster, but other than that you should look at the specific applications that you are using (i.e. the rest of the DE doesn't matter much).
