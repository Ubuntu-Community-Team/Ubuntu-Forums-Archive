### Post by entangled on 2005-12-11
I'm a long-time kde user on various distributions, but when I switched to ubuntu I gave the default gnome desktop a try for about 6 months. It was simple, and things mostly worked, but a bit dull. Now I use Kubuntu (KDE 3.5), which is immediately more appealing, and that counts for a lot.
However, some things don't work well on Kubuntu.

Local windows network browsing is reliable and good in gnome; it is flaky in Kde (it always has been in my experience). In kde now I get intermittent connection failures, messages saying the Lisa server is not running (it is, and was working a few seconds ago, and it will be working again in a moment), double workgroup icons in the Remote Places window, only one of which works, indefinite workgroup icon replication in the Remote Places window. I haven't seen any of this behaviour reported, but lack of this basic functionality makes me wary of using kde.
As I said, gnome seems to have this area under control.

Kde menus are chaotic and therefore confusing. There are whole sections replicated - or is this just the merge of gnome and Kde menus?. This just needs someone to sit down and sort it out.

Kde sound server (Arts) is just a liability, but its OK if you just turn it off and forget about it. This seems to help reliability too, but I have still had crashes on logout.

On the other hand I think gnome desktop is just depressing, visually and even functionally. It can be a virtue to be simple and clear but not at the expense of user satisfaction. I just don't want to see a dull brown desktop and a couple of plain menus on my expensive colour monitor. It looks like the hair shirt brigade won out. Gnome doesn't deserve to win fans from Kde, which does a much better job at the desktop.

Kde also has the best CD/DVD burning tool, in K3b.
Kde runs openoffice using 'normal' icons and it looks a lot better than the gnome stuff.

Overall then, I prefer Kde over gnome because it looks and feels better, but I'm worried by some flaky basic functionality in Kde.
