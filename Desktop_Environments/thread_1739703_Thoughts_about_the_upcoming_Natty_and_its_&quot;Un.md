### Post by 3Miro on 2011-04-26
Unity runs on top of Gnome2 and still uses Nautilus. Your scripts should work just fine. You can test that with Virtual Box, just make sure to get the latest version on the Virtual Box web-page (not the Software Center) and the older versions do not support effects under Xorg 1.10 and hence no Unity.

You like Gnome2 but Gnome2 is a dead project. Gnome developers killed it and replaced it with Gnome3, which is at least as big of a change as Unity (probably more).

In the future, Unity will run on top of Gnome3. Gnome-shell (the default UI for Gnome3) will be available either as a startup option or download from the Software Center.

You do have the choice on Natty of running either Ubuntu Classic or Ubuntu Classic with no effects (both of those disable Unity and get you to Gnome2). Note that Ubuntu Classic may need couple of tweaks to get it to look exactly like 10.10.

Hi,
> And stuff on our OS needs to be working fine in order for me to concentrate on my job, I can't spend my whole day trying to tweak and make things work.

You can use the "Ubuntu Classic" desktop for another 6 months or so, which will give Unity some time to become twice as good. After installing (or upgrading to) Natty, you just need to log out, then first click on your username in the login screen, then select "Ubuntu Classic" in the menu at the bottom of the login screen, then enter your password and log in. There is also a Unity-2D version that can be installed if you can not run the 3D compiz version. You will still be able to install XFCE, KDE, LXDE, E17 and many other desktops through your package manager too.

### Post by lyschoolsocool on 2011-05-04
I can be a very superficial person, so there are some inconsistencies that makes me feel as though the polish isn't there for Natty.

1. I love the global menu, it's aesthetically pleasing and for me, has been functioning very smoothly. I particularly like how the title bar is slightly visible when hovered over; it lessens the likelihood of clicking on the wrong application. I like the integration between the menu and the notifications; dragging my mouse across the screen to get to the notifications has never been so accommodating.

I don't like the empty space between the Ubuntu logo and the appmenu when you hover over it in the desktop. I know it's that way because of logistical issues, specifically to leave space to show the title bar when there is a title bar, but for someone bordering on OCD like myself, it's highly irritating. I would like to see something along the lines of "Desktop" as the title so the gap in the panel wouldn't be there, but I know people have better things to worry about.

2. I like the overlay scrollbars, but its implementation is inconsistent depending on the application. I've disabled them because of this reason, but if they were one day implemented throughout, I would be happy to re-enable them.

3. Root gtk does not use the global appmenu. I've read that this is because of security concerns, but the best system is a consistent one.
