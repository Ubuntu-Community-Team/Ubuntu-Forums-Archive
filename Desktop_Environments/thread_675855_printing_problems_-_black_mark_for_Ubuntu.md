Hi all,

I've just tried and failed to print correctly a simple text document using gedit. I printed the same document on the same printer using RedHat EL and the printout was flawless.

To be specific about the problem, the text file is plain ASCII text with about 15 lines each about 180 columns wide.  To print this, I switched to landscape mode. Several things went wrong:

* the printout was landscape OK, but the _text on the right was cut off_ at about column 92, leaving a couple of inches of blank paper down the right hand side; a significant part of my text did not get printed. I expected the printout to be landscape and to show all 180 columns of characters.

* the print preview page was badly flawed - the paper was correctly depicted in landscape orientation but the text was _vertical_ with each line running _off the bottom_ of the 'page' on screen. The text should have been horizontal and should have shown all 180 columns of characters.

This is not a good day for Ubuntu because my installation is being watched with interest by other people here with the potential for switching all our desktops to Ubuntu. Whilst the Debian package manager is seen as a great benefit, problems like this one are likely to stiffen resistance to change.

Is this a known problem?
Did I do something wrong perhaps?

Rick

I am using Ubuntu desktop 7.10. I have gedit font set to the default Monospace 10. Gedit v2.20.3 on Ubuntu and v2.8.1 on RHEL AS4.
