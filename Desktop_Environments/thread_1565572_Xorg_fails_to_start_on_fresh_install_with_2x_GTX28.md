### Post by pepolez on 2010-09-01
Following an absolute nightmare getting FakeRAID to work on Ubuntu, I've now launched in to a whole new world of pain trying to get Xorg to start for the first time.

When I booted initially, my monitors would turn off and just sit there doing nothing. This was an issue I've experienced before, and was easily fixed by adding **vga=791** to my boot options in grub.

I'm now at the stage where my PC is attempting to start Xorg for the first time, and it's presenting me with a dialog saying that my devices could not be autoconfigured.

So, I downloaded the latest NVIDIA drivers from the NVIDIA website (256.53), disabled **noveau**, and installed the drivers. I then ran the nvidia-xconfig utility and rebooted.

Xorg still refused to start, with the same error message, despite having the driver set to **nvidia** in **xorg.conf**.

To simplify troubleshooting, I disconnected my second monitor and rebooted. This did not change the outcome.

My question, with reference to the information below, is how do I get X to start (with just one monitor for now)?
