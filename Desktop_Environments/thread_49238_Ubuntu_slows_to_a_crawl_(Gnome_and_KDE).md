### Post by ClintNoSleep on 2005-07-15
Hello!  I am a new Ubuntu user.  I installed Ubuntu a couple of days ago.  The first thing I did when I installed it was to try to change the background image.  When I right-clicked on the background and tried to select to change the background image, my system slowed to a complete crawl and I had to reboot.  I thought this was just a one-off thing and didn't think much of it until a day later I started using my machine after the screen saver was on, and again, everything was at a complete crawl.  I thought maybe the computer was just a bit slow after coming out of the screen saver, but the problem did not go away after waiting for 15min.  I could not even open an terminal and I had to reboot again.

I tried installing Kubuntu next, but I had similar problems, so I reinstalled Ubuntu.  Everything was going okay for one day, but I left my computer on overnight and when I came back this morning, the computer was dying again (I had turned off the screen saver option, so my monitor was just turned off while I was asleep).  When I say "dying," I mean the following:

- I could move the mouse pointer around fine
- When I clicked on menus, they sometimes never came up, sometimes came up after a *really* long time
- When I did get a menu and try to launch a program, no program ever showed up
- When I right-clicked and tried to get a terminal, no terminal ever appeared
- Logging in through ssh was incredibly slow
- When I ran a program (such as top, or "man") through ssh there was a *very* long delay before the program would do anything (on the order of minutes to run top, for example)

The other strange thing that I noticed this morning was that the clock on the computer was behind by three hours!

When I ran top it showed very little memory free (on the order of 5-10MB), but CPU usage was almost zero and there was no one program that was using a lot of memory (I think Thunderbird was using the most, it was using about 18MB).

I am using an IBM ThinkCentre computer with a 2.4 GHz P4 and 256 MB of RAM.  I thought that maybe Gnome/KDE uses too much RAM so I am running WindowMaker now.  I haven't had any problems yet but I ran Gnome for a day without encountering any problems, so I am not sure if I have really solved the problem or not.

Does anybody have any suggestions about what could have caused this problem?  Most of the searches that I did suggested that this was some kind of video-card issue or power-management problem, but I have a generic Intel video card and I turned off all of the power-management stuff on the computer.

I would greatly appreciate any help anybody can give me!!!!!!

-Clint
