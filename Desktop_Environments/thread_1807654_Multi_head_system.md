### Post by aphatak on 2011-07-19
I don't think you are the only one to have trouble with Ubuntu 11.04 and Xinerama - I was trying to get Unity working with Xinerama, and ran afoul as well.  When Xinerama was switched off, the Unity desktop starts up fine; if I switch Xinerama on, the system allows me to log on, but the desktop never gets loaded.  I can switch to a console (ctl-alt-F1 through F6), and confirm that the beast is working alright, but hit ctl-alt-F7 and I don't see a graphic desktop.

I am using an Intel quadcore Core i5 2500K, 16GB RAM and two ATI Radeon HD5450 video cards driving two monitors each, with the ATI proprietary driver which was downloaded from the ATI site, not from Ubuntu repositories.  The set up works sweetly under Ubuntu 10.04.2 LTS, and Windows 7.  I am pretty sure it works with Kubuntu 11.04 (I will confirm this when I get home and try it out tonight), but bombs with Ubuntu 11.04 Unity.

Any suggestions / pointers, anybody?  What more info should I upload for people to make sense of this problem?

Incidentally, your xorg.conf code shows 'Option         "Xinerama" "0"', which would actually switch Xinerama *off*, not on.  You need "1", "On" or "True" in place of the "0".
