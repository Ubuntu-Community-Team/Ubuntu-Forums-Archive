So what happens? Well, the system just randomly hangs. Sometimes while I'm using it and sometimes not. It's not the xorg issue where X hangs and the machine is otherwise responsive; everything is dead.

I ran memtest twice -- 10 or 11 passes each time. No problems reported there. I also upgraded the BIOS.

I've tried disabling ACPI (noacpi acpi=off) as well as local APIC (nolapic noapic). Disabling the APIC seems to make the system much more unstable than it was to begin with.

Since there's very little hardware in the system (motherboard, cpu, ram, video card -- apart from the drives, which I doubt are causing a problem), I'm trying to rule out components piece by piece as I expect this is likely a hardware issue (or at least a hardware/software combination issue). I plan on picking up a nvidia card with a different chipset tomorrow just to test. In the meantime, I'm running off the Kubuntu Live CD (w/ 386 kernel) to see how long this remains stable for. Of course, in this mode, I'm only using the open source 'nv' drivers for X.

After running memtest without issues, I figure the next best bet is the video card.

Temperature wise, the system seems well-cooled. I certainly don't put it under heavy loads. Normal conditions are around 40-ish degrees celsius on the cpu and motherboard temp sensors (either via BIOS or mbmon).

Any other suggestions as to what I can do? If I have to take the box back to the shop, I'd like to be able to tell them what specifically to look at.
