### Post by raziv on 2008-01-16
Solved!
I created a script called "beforeX": ```
#!/bin/sh -e
#
# beforeX
#
#	All you need to run before X starts during boot.
#

#--------------------------------------------------------------------------------------------
# add resolution modes for Samsung SyncMaster 206BW
# see http://cowbox.coe.uky.edu/stu/linux/Xat1680x1050.html
#--------------------------------------------------------------------------------------------
915resolution 50 1680 1050

exit 0

```and put it in /etc/init.d. Then I saw that x11-common is called at rcS.d through the link S70x11-common, and assumed that this is where(=when) X is being started. In order to execute my new script before that, as its name suggests it should, I added a link to it in that same folder, called S69beforeX. At first it didn't work. Then I remembered that in linux you must mark a text file as executable in order to ..ahem.. exeute it, so I looked at the permission marked for x11-common and marked the same Vees in my script's "permissions" panel. Rebooted and WOOHOO! got high resolution on boot! :)
