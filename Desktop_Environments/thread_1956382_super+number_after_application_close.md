### Post by gleedadswell on 2012-04-10
To start with, I'm loving the Unity desktop.  It took me a while to warm up to it, but once I learned a few hotkeys I suddenly started finding it very efficient for rapidly switching between a large number of applications, which I do a lot.  I think its window management is way ahead of anything else I've seen.

Lately I've noticed a minor hitch.  I use super+number a lot to switch between applications.  When I close an application and there is nothing else in the current workspace I find that super+number, ctrl+alt+arrow and alt+tab don't work anymore.

In trying to figure out what was going on I noticed the following:

1. If you close an application and nothing else is in the workspace then if you hover over the panel you will see the menu bar for the application you just closed, including the close, maximize and minimize buttons.

2. If you click on the desktop then the incorrect menu bar disappears (replaced with the desktop menu bar).  Now super+number, ctrl+alt+arrow and alt+tab will work again.

3. Alternatively, if after closing the application, you get to another application by clicking its icon on the launcher then next time you try them the hotkeys work again.

Both of the ways to recover the hotkeys involve going to the mouse.  What I really like in Unity is that it works so well with hands on keyboard, so this little hitch bugs me.  I think this is a new behaviour (as of a week or two ago??).  Does anyone know a fix?  I searched launchpad for this and nothing came up.
