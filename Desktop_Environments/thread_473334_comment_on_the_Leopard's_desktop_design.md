### Post by bittergourd on 2007-06-13
I just saw this
[http://youtube.com/watch?v=xrlVgXxjuR0&mode=related&search=](http://youtube.com/watch?v=xrlVgXxjuR0&mode=related&search=)

I don't know if OSX is "supposed" to be elegant, but the stack feature looks not only nice, but also very handy.  It is true that there are differences in the spirits behind OSX, windows and linux; they are developed towards different goals, and for different kinds of users.  But now that ubuntu is trying to become a user-friendly os (it is already doing very well), i think the community should focus more on the usability.

I will take two examples.  compiz/beryl are remarkable, great projects.  But apparently i am only using a few features, like scale and expand cube.  Things like wobbly, i think it is just totally USELESS.  Other things like animation, cube rotation, transparency, they are nice to have only if one wants to kill free CPU time or to quickly drain  battery power.  Another example, kiba-dock, i almost abandoned it because i didn't know how to turn the stupid physical engine off in gconf-editor, until the new version had it off by default (and came with a nice configuration tool). 

compared with stack, expose etc etc in OSX, i believe compiz/beryl and kiba-dock are no less challenging from  the technical point of view.  But I just think the effort is wasted on those less useful stuff.

It seems to me that there is an intrinsic paradox.  If someone has a great idea, he/she may attempt to contribute (sell) it to a commercial software simply because a great idea deserves its value and so $$ is where it should go naturally.  Even if it is kindly "donated" to the open source community, the licenses will not protect these IDEAS from being stolen by companies since they are not directly copying codes.  This is an essential difference between a patent and a open source license. (correct me if i am wrong.  i am not an IT lawyer.)  The argument above may explain why linux community isn't so creative -- it is not money driven.

### Post by tgalati4 on 2007-06-14
My understanding is that Leopard is a full-fledged 64-bit rewrite of OS X.  Besides all of the eye-candy, I would expect better performance.  Also, since Apple dropped PPC, I believe that Leopard will drop all traces of PPC code in the OS, so it will be smaller than Tiger.  Tiger is certainly bloated since most applications contain both PPC and Intel binaries.  There are utilities that can remove some of this bloat, but then you risk borking your system.  The iDVD application (for making movies and burning them to DVD) is about 1 GB in size.  That's a large application.
