### Post by Xianath on 2007-07-30
Hi all,

I added the kubuntu.org repos to Adept and upgraded. Now Alt-Tab and Win-Tab work differently than they have always been working.

Alt-Tab would now always cycle between all windows on the desktop. Before, if I only hit it once, it would alternate between the two windows most recently used. The new behavior is really counter-productive. For example, when I am typing an email,  I'd switch back and forth between Kontact and Composer as new mail comes in. Now I need to cycle through all open windows which wastes a lot of time (not to mention making it likely to send the wrong content to the wrong recipients). And that's with a desktop specifically dedicated to mail (forced via KWin based on window properties).

Win-Tab used to switch desktops immediately. Now it pops up a list of desktops for me to choose from. I could live with that if it activated the selected desktop when releasing the keys, or at least if it selection with the mouse worked. As it is now, I have to get my right hand off the mouse to hit the Enter key every time I need to switch desktops.

I'm in the habit of having a separate desktop for every type of task (Mail, Web, IM, Eclipse, VMWare, krdc, Console, amaroK) and this makes multi-tasking really easy with the proper shortcuts. Now all of a sudden there's this behavior change that totally ruins the smoothness of my workflow. Is there a way to configure it back to what it was before the upgrade?

Thanks,
