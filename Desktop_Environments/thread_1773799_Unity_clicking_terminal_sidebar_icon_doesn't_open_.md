### Post by sarahsharp on 2011-06-02
My workflow involves keeping several terminals open on different desktops.  With gnome, I could click a taskbar icon and launch a new terminal window in whatever desktop I'm in.  With Unity, I can't open a new terminal window if a previous terminal window is open.

The behavior for clicking icons in the sidebar seems to have different behavior for different applications.  Clicking some icons will always open a new window, and clicking other icons makes the window focus jump back to a previously open window for the same application.

I almost never want to jump back to the open application (aside from firefox).  I always want to open a new invocation of the application when I click the sidebar app icon.  I can't seem to find a setting to control whether the application is launched again, or it jumps to the previously opened application.

Steps to reproduce:
----------------------
0. Go to Desktop 1.

1. Add a terminal icon to the sidebar by dragging the terminal icon from the search window to the sidebar.

2. Click the terminal icon to open a new terminal.

3. Switch to Desktop 2.

4. Click the terminal icon on the sidebar again.

This jumps back to Desktop 1 and displays the previously opened terminal.  The behavior I want is to open a new terminal window in Desktop 2.  Is there a setting somewhere to turn on this behavior?
