### Post by Jucato on 2006-06-10
Take note that Linux and Windows use RAM/Memory differently. Windows doesn't only uses RAM when needed. Linux, on the other hand, uses up the unused/free RAM to speed things up, freeing it only when something else needs it. As someone in the forum once said, RAM is meant to be used, not to be left lying around (not exact quotes).

That said, Xubuntu is very good when you want something minimal and fast. On your specs, you could run GNOME and KDE without problem. I have a 1.5Ghz AMD processor with 640MB RAM (512+128), and KDE is doing well, though not as fast as Xubuntu.

Just to give you an image of how fast Xubuntu can be, with those specs, Xubuntu runs like the wind on the Live CD. And Live CDs are supposed to be slower than installed systems! So just imagine how fast it will be once installed. :D

### Post by Jucato on 2006-06-10
Edited the first link. OSdir seems to be having some trouble displaying the full sized screenshots. I'm afraid you'd have to browse through the full slide show.

@Sutekh: AFAIK, that's how RAM/Memory is managed in Linux. Linux allocates RAM to the applications that need it, then uses whatever is free for Disk Cache. If more RAM is needed by apps, Linux then frees some of the Disk Cache for the apps to use. I think it also keeps a minimum amount of RAM for Disk Cache and starts to use the swap if the apps need more than what the physical memory can provide.

I have 640MB (512+128). According to KInfoCenter, 55% is being used by apps, and 39% is used as disk cache, 6% is free (around 11MB). And that's with Konqueror with 3 tabs, the GIMP, the whole Kontact suite (RSS, Mail, etc), KTorrent seeding the *buntu ISOs, superkaramba, kompose, katapult, and 3ddesktop all running.
