### Post by motorcity909 on 2009-09-13
Hi all.  I've used Thunderbird for many years so when I recently dumped WinXp for Ubuntu, I made sure I installed Thunderbird and copied my profile over.  All messages, extensions and settings were fine.  However, I noticed that when I started Thunderbird, it would freeze so I would have to force quit it, then re-start and it was okay.

I've uninstalled and reinstalled it and it was the same.

With a bit of experimentation. I've established that this happens whenever a new email comes in.  So when I first open TB in the morning there are inevitably new emails, so it freezes, I quit and reopen, the new emails are all there but TB no longer considers them a 'new' so it works fine and I can read the mails.

Once a new email comes in though, it will freeze again.

I've disabled the 'check for new messages at startup' option but as soon as I click 'Get Mail', it freezes (if there was some new mail).

Any help on this would be much appreciated.  I like using Thunderbird and have got it set up just how I like it, mail folders, rules to filter mail, etc so would like to continue using it.  

Thanks in advance.

---

### Post by motorcity909 on 2009-09-13
Hi again, all sorted.  I found a known TB bug with the same problem and some suggestions that disabling the new mail sound alert would fix it.  I tried that but it still froze.  I then disabled the 'show message alert' function (which hadn't worked anyway) and now new emails are coming in fine, no freezing.

Cheers, Dave
