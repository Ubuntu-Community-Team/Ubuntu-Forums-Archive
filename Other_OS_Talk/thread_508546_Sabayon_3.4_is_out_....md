### Post by handy on 2007-08-01
I have all but finished setting up SL.  

From the little experience that I have had with package handling, it is not a problem, it is very similar to using apt.  You do need to keep your portage tree up to date via the internet.

In a root terminal you should type:

***emerge --sync && layman -S***

***emerge --sync***  syncs your computer to the latest portage tree.

***layman -S***  syncs all your overlays such as our own Sabayon overlay. 

Using the following:  ***emerge eix && update-eix*** downloads and installs ***eix***, which searches portage via the terminal, making searching through the portage tree easier and faster.

***update-eix*** of course updates ***eix*** to your last sync. 

You can then use ***eix {program name here}*** to find if it exists in portage.

If it's not in portage then you have quite a bit more work to do, options being:  

Manually install from source code.  

Build an ***ebuild***. 

Search the net for a 3rd party ***ebuild*** placing it in ***/usr/portage/{it's location}*** and then ***emerge*** it.

Not *quite* as simple as apt or synaptic when portage ***does*** have the desired files. Potentially a whole lot more work if your desired files are not residing in the portage tree.  

There are over 11,000 titles in portage, so doing things the hard way shouldn't be a common occurrence I would think.  Apart from the fact that there are so many files supplied on the SL LiveDVD.

I installed NeroLinux 3, via emerge today, it was alive & well in the portage tree.

So apart form the above mentioned handling of individual package installation, there is the altogether more complicated problem (to the Gentoo ignorant) of keeping your ***entire*** SL machine up to date.  If you don't do it right you can trash you system.  Or give yourself a great deal of work trying to repair it.  

My philosophy on updating SL,  is to not worry about being on the cutting edge, I will wait for the next official update (probably about 4 months) & download the DVD which has an update option & all the needed files on it.  This way I will avoid possible dependency hell, which I don't find an attractive proposition at all.  

As time goes by I may become interested in learning the intricacies of Gentoo, though really I don't have to in Sabayon Linux, which is nice. :-)

The only time torrents are really useful is in the first few hrs of release to offload some of the server strain, and thats when they don't have metalinks.
