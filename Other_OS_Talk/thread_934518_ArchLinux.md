### Post by Rumor on 2008-10-01
> **Mr.Macdonald said:**
> I have been reading about linux stuff and found archlinux. I seem to like its ideas more than ubuntu's. how do those who have experiance with it feel it compares to ubuntu

Both as a desktop and a server

I've used Arch for a little over two years, both as a desktop (currently on three computers) and a server (a private IRC server).

it is hard to [compare]("http://wiki.archlinux.org/index.php/Arch_vs_Others") Arch to Ubuntu because the [philosophies]("http://wiki.archlinux.org/index.php/The_Arch_Way") behind them are so radically different. Ubuntu gives you a fully functioning desktop system with GUI tools out of the box. Arch gives you a fully functioning command line and the tools with which you build your system to be whatever you desire. Ubuntu has default programs and Arch does not.

I prefer Arch because it is a rolling release. I am not a fan of large updates every six months, preferring to stay current with my weekly system updates.

Arch has a great [community]("http://bbs.archlinux.org/"), and an excellent [wiki]("http://wiki.archlinux.org/index.php/Main_Page")

Arch is what you make it. See the [FAQ]("http://wiki.archlinux.org/index.php/FAQ") for more info.

### Post by SomeGuyDude on 2008-10-08
If you have the time to get it set up, Arch is king. I've used a baker's dozen Distro's so far if you count each Ubuntu type as different: X/K/Ubuntu, Crunchbang, Mint, Fedora, OpenSUSE, DreamLinux, Xandros, Sabayon, Arch, Mandriva, SimplyMEPIS, Zenwalk.

To me, there's a huge sense of accomplishment in getting Arch set up. Since you build it, there's kind of personal responsibility going. With most OS's, you install it, and if something doesn't work you get annoyed because you have to fix things that the devs should have done. With Arch, the responsibility is more on you, so there's this feeling of "that's broken because you did something wrong" and the converse of "everything's gravy because I did a good job."

That said, you have to remember that there will be a LOT of things you probably didn't realize you had to consider. Here's a small list of things that completely slipped my mind when it came to manually building Arch that worked under Ubuntu right off the bat:

1) Media hotkeys
2) Making the PC sleep, particularly when I closed the lid
3) The touchpad's more complex features
4) DPI resolution

Then there were the things that turned out to be more complicated. MPD's a good example, installing that was effing annoying because you had to toy with permissions and the mpd.conf file more intimately. The Arch Wiki is downright crucial to the process.

You'll also need to steel yourself because if you've never done it before, setting up Arch will result in a heap of frustration when you realize you didn't read the instructions as closely as you thought you did and now something's broken. Wireless seems to be a HUGE issue for lots of people.

If you're a .deb and Ubuntu repo addict like me you're going to have some uphill work, too, because Arch just doesn't work like that. As with other things, there's more manual setup. That's fine too, though, since it's just extra steps. As someone said to me, the only hard part of Arch is getting it started. Since it's rolling-release, once you have it set up, all you ever have to worry about from then on is updating periodically.

So my advice: do it. Totally worth it.
