### Post by Linux&amp;Gsus on 2008-05-11
OK, I try to through in some ideas, suggestions, things to think of, what ever you wanna call it.

**DE:**
I know you are pretty fixed on XFCE already and for that you have good reasons, being light-weight is a good thing. Although I must say most offices run MS XP. Means, there is more than likely to run some heavier DE smoothly. Those on the road (with laptops) have that capability for sure. They simply can not afford to show up at their business partners and have a slow / bad responding computer with them. So, there is more than enough computer power to not worry about whether Gnome or KDE will be sluggish.
With that, I do understand the desire of using XFCE or Gnome, but please have in mind that a business simply wants to keep going. The way KDE is laid out is very familiar to Windows users. Now, just think of someone working in a office who is told from one day to another that there is no more Windows on his/her computer but Linux and all looks different...

It might be OK for a dedicated Start-up but for the rest it's debatable if they are willing to transition their folks to a new OS that looks (and works??) totally different. Remember that it cost a business real money to train all their folks when introducing a new platform. My suggestion would be to make it as easy as possible for them. E.g. by seriously considering KDE or the way KDE is laid out.

**Integration:**
Having said that, what is your main focus anyways? Start-ups, SOHO, Fortune 500, Education, Government,...? Or simply "all" offices?
If you target an existing office (basically every company except Start-ups) then they most likely run Windows and also they most likely don't through out all their stuff and get a 100% Linux environment over night.
Therefore a very good integration into the existing (Windows) environment is a must. Webdav, Active Directory, LDAP, Intranet, etc. What about connecting to Windows Terminal Server, shared directories and I don't know what.

**Applications:**
Since people most likely prefer a native app instead of using a web interface Evolution seems to be the only option for connecting to MS Exchange Server (correct me if I'm wrong).

As browser I would say Firefox should be installed by default since people most likely know about it. Less people know Opera and even less know the other browsers mentioned. Remember, if you target businesses you'll deal with many people who thing that Internet Explorer is the internet. I would  have multiple browsers installed so that people can choose easily.
Also, you should think about having IE4Linux on there by default. People might come across IE-only-sites which are crucial for the company or they might have IE-only-apps in their intranet.

Something else to think of is synchronization with PDA/Smartphones.

Then, again depending on targeted businesses, there are a lot of tools to consider. Software & web development, accounting, video editing, etc. You name it.

Personally I would not worry too much about games. I'd rather prefer someone having a 10min focus shift a couple times a day with something like Mines or Solitaire then surfing porn sites or downloading pirated music. This is not about ones personal opinion on that matter. It's more about what the law says about it and the potential dangers for the corporate network. Ya ya ya, I know, Linux and security. But you need to think from a companies perspective...

For bigger companies a easy remote connection to the desktops of the employees is a real winner for the sys-admin.

Many things in Linux are about choice. Give people the choice of apps but very well sorted. Too much choice is overwhelming for a Linux first-timer. But a very well sorted set of applications can be the winner when choosing between Windows and Linux.

"Choice" could also be the solution for the DE. At the end of the day I believe for a project like this to succeed it would need a good, clean, easy to  access web site that shows in a few screenshots and videos the potential of it.

**Server:**
I know there is a server version of Ubuntu. But I would guess that a modified version with installed apps and pre-configured would be a great help to make the office/business version play well together with the server. Specially having something like a groupware installed and easily connected with something like Evolution/Kontact/Thunderbird/whatever-other-app.

**Desktop Backgrounds:**
Instead of choosing one default background, like pretty much every distro does, why not having 4 different ones on the different virtual desktops.




So, this is by no means a complete list of thoughts but I guess I finish my book here for now... ;)

### Post by atomkarinca on 2008-05-14
Basically a meta package includes the names of the packages it will install. For example when you type in a terminal

```
sudo apt-get install ubuntustudio
```

it installs ubuntustudio-audio, ubuntustudio-video, ubuntustudio-desktop so on so forth. And even those are meta packages including jack, lilypond, ardour, hydrogen etc. The meta packages can also include themes, icons, backgrounds, gdm's, usplashes, and any other eyecandy you would like.

Starting with a meta package sounds like a very good idea. You should consider it.

### Post by cardinals_fan on 2008-06-24
> **volkswagner said:**
> I can't remember the exact problem I had.  It was with Xubuntu.  Either adding a shortcut on the desktop, or a newly installed app to the menu.  So, are the steps to perform the above the same for XFCE and Gnome?
It's adding apps to the menu.  It's actually quite easy to do, but they don't make it obvious.  From the Xfce wiki:

> How to edit the auto generated menu with the menu editor?
```
cp ~/.cache/xfce4/desktop/menu-cache-name-of-the-generated-file.xml ~/.config/xfce4/desktop/menu2.xml
cd ~/.config/xfce4/desktop/
cat menu.xml > menu3.xml
cat menu2.xml >> menu3.xml
mv menu.xml menu.orig.xml
mv menu3.xml menu.xml
```
Now, you already have a menu with all the categories in the main tree with some duplicates, but you must first edit menu.xml with your favorite editor and remove the 4 following lines in the middle of the file, otherwise the menu editor will complain about a wrong format: 
```
</xfdesktop-menu>
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE xfdesktop-menu>

<xfdesktop-menu>
```

That's all. Now you can run the menu editor, remove the few duplicates and edit all as you like. 

Settings > Desktop > Menu > Menu Editor 

Notes: by removing the &#8220;system&#8221; line, you will remove all the duplicates menu entries from the auto generated file. So, if it is changed in this auto generated file, they don't appear anymore, but you will get rid of most of the duplicates. 

To restore the original menu, just do in a terminal: 
```
mv menu.xml menu3.xml; mv menu.orig.xml menu.xml
```
