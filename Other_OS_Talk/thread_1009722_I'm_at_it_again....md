Otherwise, anything goes.  Here's some of what I'd like (a bit different from my quests in the past):

- Heavy emphasis on minimalism and the CLI **above all else**.  I want something insanely lean and manual.

- If it uses binary packages, I don't care whether it handles dependencies.  If it is source-based, I want the system to handle deps.
- Binary packages are preferred, but I can deal with source-based.
- While I prefer vanilla packages (ie not Debian), I will accept distros with any base.

Here are some distros I **don't** want (because I've used them, not because they're bad): Slackware, Arch, SliTaz, *BSD, Gentoo.  Also don't recommend LFS.

I'm especially interested in CRUX, Lunar, and BasicLinux.

All thoughts (even "you're insane") are appreciated.

I know you wrote "no Gentoo", but honestly, all of the above sounds like Gentoo. I understand the idea of wanting to try something that you haven't tried before, though. I just have the feeling that after all your travels, you will end up with Gentoo on the box. :)

As you may know, you can install Gentoo from Slackware, so there's no need to download any images and burn yet another CD. You can either add Gentoo to your bootloader for Slackware or install a bootloader for Gentoo and add Slackware to that. Chainloading is also a possibility. (I often do that.)

In spite of being source-based, you can limit the amount of compiling you have to do. As you likely recall, it does take some time to get Gentoo setup (compiling xorg-x11 takes a while on my old hardware), but once you get the system the way you want it, you don't have to compile new packages all the time. In my case, I keep an eye on the security announcements and I upgrade any affected packages ASAP. For everything else, I only update the system about once a week.

I run a minimal setup with the base system, a WM, and a few light applications. Upgrading once a week only takes an hour (or less). For the rare cases where it's longer than that, I just watch a movie or a hockey game or something. ;)
