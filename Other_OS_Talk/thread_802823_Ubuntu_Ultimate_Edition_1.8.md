### Post by SaddleTramp on 2008-05-23
Hey ya'll.  It's only been a few months since I made the crossover from MS to the *nix side.  Ubuntu Hardy 8.04 (then while still in the test phase) was the very 1st distro I tried and ended up being the only OS installed on my system (totally dropped MS).  I'm not that easy to impress, but I gotta tell ya Hardy blew me away.  I never thought any OS could be so compact and still be functional.  And the speed and performance level increase in my system just absolutely amazed me.  For what very minor problems (i.e. finding and setting up some apps in Hardy) I had, it was easy enough to find the answers here in the forum.  The people here were great and there to assist when I needed it.  Shortly after my introduction to Ubuntu, I came across a thread here that mentioned Ultimate Edition and how it was loaded with additional apps, games, and such.  Thought I'd give UE a try and here I am at present.  I have UE 64 & 32 bit versions on my system now, and have learned a great deal about *nix from Hardy & UE both.  I settled on UE as a permanent OS for several reasons but most importantly because of the UE team.  Someone here asked about the update/upgrade.  Yes it is.  Since being built on top of Ubuntu, when a new release comes out, UE will follow **sometime** thereafter.  The reason I say sometime is due to the fact that no release of UE goes without extraneous tweaking, testing, and streamlining by TheeMahn and the staff of the forum.  The decision to release it is made by only one...TheeMahn.  Not until he is absolutely satisfied that it is the best it can be will he let it go.  This policy ensures that we as users are getting a release that will impress even the most hardened critic.
You can ask for and receive all the opinions of others that you want.  But until you experience first hand what UE is all about, you'll never know what you are or aren't missing.  I personally invite anyone to come and try it out.  Whether it's by running the liveCD or by a full install.  Running the liveCD, while can be impressive in itself, will not do your system justice as you won't be able to see the full capability of your hardware as you would with a full install.
As I said earlier, Ubuntu Hardy was my first "home" in the *nix world.  But I have permanently settled down in UE.  Thanks to all and anyone that take the time to read this post and consider giving UE a try.
S'Tramp :)

function wopinstall {
#make sure no other package manager is running
dpkg_check
#see if it has already been downloaded
if test -f worldofpadman.run
then
#if it has install as requested
./worldofpadman.run 2>&1 | zenity --window-icon=/usr/share/ultamatix/pixmaps/wop.png --width=600 --height=100 --progress --pulsate --auto-close --title "Installing World Of Padman..."
else
#download it
wget -c http://thilo.kickchat.com/download/worldofpadman.run 2>&1 | sed -u 's/.*\ \([0-9]\+%\)\ \+\([0-9.]\+\ [KMB\/s]\+\)$/\1\n# Downloading \2/' | zenity --window-icon=/usr/share/ultamatix/pixmaps/wop.png --width=600 --height=100 --progress --auto-close --title="Downloading World Of Padman..."
#set permissions to install
sudo chmod +x worldofpadman.run
#install it
./worldofpadman.run 2>&1 | zenity --window-icon=/usr/share/ultamatix/pixmaps/wop.png --width=600 --height=100 --progress --pulsate --auto-close --title "Installing World Of Padman..."
fi
#create Icon since they cant seem to do it right
#since we have root we need to discover normal username so we can create the shortcut & set proper permissions
NU=$(cat /etc/passwd | grep 1000 | cut -d: -f1)
#set proper permissions
sudo chown -R $NU:$NU "/usr/local/games/WoP/"
if [ "$architecture" = "x86_64" ]; then
#64 bit icon creation
sudo echo "[Desktop Entry]
Name=World Of Padman  
Comment=World of padman 
Exec=/usr/local/games/WoP/wop-engine.x86_64
Icon=/usr/local/games/WoP/wop.png
Terminal=false
Type=Application
Categories=Application;Game;" >> /usr/share/applications/worldofpadman.desktop
else
#32 bit icon creation

Exec=/usr/local/games/WoP/wop-engine.i386
Icon=/usr/local/games/WoP/wop.png
Terminal=false
Type=Application
Categories=Application;Game;" >> /usr/share/applications/worldofpadman.desktop
fi
}

function wopremove {
#created custom uninstaller since thiers does not work.  kicks back "Could not find a usable uninstall program. Aborting."
#find username it installed under
NU=$(cat /etc/passwd | grep 1000 | cut -d: -f1)
#create directory incase they installed it elsewhere, no sense in scraping all thier games
sudo mkdir /usr/local/games/WoP/ 2>/dev/null
sudo mkdir /home/$NU/WoP 2>/dev/null
#remove it
echo "Removing World Of Padman please wait ..."
sudo rm -R /home/$NU/WoP 2>/dev/null
sudo rm -R /usr/local/games/WoP/ 2>/dev/null
#remove shortcut created earlier
sudo rm /usr/share/applications/worldofpadman.desktop
echo "Done."
}

>        --force-yes
           Force yes; This is a dangerous option that will cause apt to
           continue without prompting if it is doing something potentially
           harmful. It should not be used except in very special situations.
           Using force-yes can potentially destroy your system! Configuration
           Item: APT::Get::force-yes.

For example, if I have an unofficial repo enabled (perhaps from another script or something) that conflicts with the packages Ultamatix wants to install, Ultamatix will install those packages anyway and BREAK THOSE PACKAGES!

--assume-yes is safe; compare this to --force-yes:

>        -y, --yes, --assume-yes
           Automatic yes to prompts; assume "yes" as answer to all prompts and
           run non-interactively. [b]If an undesirable situation, such as
           changing a held package, trying to install a unauthenticated
           package or removing an essential package occurs then apt-get will
           abort.[/b] Configuration Item: APT::Get::Assume-Yes.

Note the bolded line.

In short: --assume-yes, safe, --force-yes, no. I'm not saying that the script hasn't been improved since Automatix; I'm saying that a 's/--force-yes//' is in order.
