### Post by Down8ve on 2009-02-07
I have to come to Sabayon's defense.  The Gnome-based Sabayon4 MCL edition is a 2-gig version with the XBMC Media Center.  I've only used it for a day, but it is really nice.

There is a new community repo you can add called "Naendo," which adds KDE4.2 to the mix easily.  The only thing I've seen with their KDE4.2 is that you need to configure the fonts to render correctly, they are a bit blocky out of the box.

Still, I have had no problems with this "lite" edition, contrasting my experience with the extremely heavy full KDE 3.5.10 edition.  If you want to use something very up-to-date and a bit different I'd encourage you to try it.

### Post by handy on 2009-02-07
I had the two previous versions of Sabayon installed for many months on my 2nd machine.  I liked it, it was stable & worked on that machine with its difficult graphics card straight out of the box (so to speak). 

The latest version fails very quickly in live or installation mode.  So, I struggled & got it to the point where I took a chance on installing it to the HDD in upgrade mode, (I was prepared to loose the lot) where it took a loooong time & trashed the previous install of Sabayon.

I only really used Sabayon on that machine as internet reference when working on my main machine, & for playing Oblivion via Cedega, due my having never been able to get Cedega to work on my Arch machine.  It also played vid's, ripped DVD's via wine/DVDRipper & burned them with NeroLinux.

If Sabayon likes your hardware, & you like it, go for it.

I can't comment on the new package installation method the current version is using, though from what I have read, it looks promising; kind of like making Gentoo more like Arch in the package management department. ;-)
