### Post by igknighted on 2007-04-20
I ran across SAM Linux a few weeks ago while looking for a distro for my older (1ghz) desktop.  At this point I was not a fan of Xfce, but I needed something light, so I gave it a shot.  Initial, I was stunned.  I have used Xubuntu before, but Xfce comes so plain in Xubuntu that I asumed it to be a very plain desktop.  Even popular distro's like Zenwalk have a very plain Xfce look.  SAM is different.  It has only one real panel, and a tremendous dock which zooms extremely smoothly and has nice effects.  The default theme is green, which I believe is a great color for themes that more distro's should consider.  Finally, the selection of applications is great.  Bluefish, Frozen Bubble, Exaile and Opera are all there by default.  Also, wine is installed and configured already.

It is not without issues however.  It uses the mandriva/pclos drak control centre, and some tools there are KDE specific.  Like when setting auto login with the drak tool it tries to set kdm, while SAM uses gdm.  Also, when I tried to install onto my main desktop, I ran into the same bug as when I used Mandriva 2007.0 and PCLOS 2007.  My network card and my sound were unrecognized.  As a workaround I copied the modules folder from my fedora install into SAM, then booted the fedora kernel into sam, which worked perfectly, and then compiled a vanilla kernel for SAM so it didn't have to piggyback Fedora's.  The bug is upstream with Mandriva, and they have fixed it for 2007.1, so I am not worried.

Overall, I think SAM is an amazing distro.  It made me actually want to use Xfce.  In fact, I liked it so much I am now helping Nico (the founder) write wiki documentation and helping out on their forums/irc.  If you are looking for a distro that is very user friendly and works great on older hardware, give SAM a try.
