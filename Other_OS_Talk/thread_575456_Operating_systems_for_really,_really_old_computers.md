### Post by darrelljon on 2007-10-12
Here's some suggestions in order of their size (as a compressed image rather than install or [memory requirements according to free -m]("http://ubuntuforums.org/showthread.php?t=814433")). Most run on x86 hardware and are graphical.

[SIZE="4"]Not Linux but worth mentioning[/SIZE]
   (<2Mb?) [Contiki]("http://www.sics.se/contiki/")
   (<2Mb) [QNX]("http://toastytech.com/guis/qnxdemo.html") Floppy demo
   (<2Mb) [KolibriOS]("http://kolibrios.org/") (based on [MenuetOS]("http://www.menuetos.org/")) a GUI OS
   (<2Mb) [Balder]("http://www.finnix.org/Balder") (based on FreeDOS)
   (~11Mb) [FreeDOS]("http://www.freedos.org/") + [OpenGEM](http://gem.shaneland.co.uk/) (4.4Mb) - DOS compatible GUI
   (~23Mb) [ReactOS]("http://www.reactos.org/") - Windows compatible but still in alpha
   (~30Mb) [KolibriOS]("http://kolibrios.org/") CD version (based on [MenuetOS]("http://www.menuetos.org/")) a GUI OS
   (<50Mb) [DamnSmall BSD]("http://www.damnsmallbsd.org") - based on FreeBSD, but with Fluxbox
   (<100Mb) [Milax]("http://milax.org/") - Solaris live CD

[SIZE="4"]Unknown size[/SIZE]
[Deb-Ice]("https://sourceforge.net/projects/deb-ice") netinstall

[SIZE="4"]Special mention[/SIZE]
[INX](http://inx.maincontent.net/) Command line Ubuntu designed for usability.

[SIZE="4"] Under 50Mb[/SIZE]
Distros under 50Mb fit onto a MiniCD or sometimes floppy disks (usually 1.44Mb each). Best in category: Damn Small Linux.

   (<1Mb) [muLinux]("http://mulinux.dotsrc.org/")
   (~3Mb) [BasicLinux]("http://distro.ibiblio.org/pub/linux/distributions/baslinux/") - Slackware 4 + X + e-mail + presentation
   (~3Mb) [Mungkie]("http://bit.ly/40Kn8B") - (aka 2diskXwin)
  (~4Mb) [Blueflops]("http://blueflops.sourceforge.net") - 2 floppy graphical web browser
  (8.9Mb) [Geexbox](http://geexbox.org/) (Media Center only)
   (10Mb) [Tiny Core]("http://www.tinycorelinux.com/") - Linux 2.6 kernel, Busybox, Tiny X, Fltk, and Jwm
   (12Mb) [Grey Cat Linux]("http://bit.ly/4ySJj") - Slackware based with IceWM
   (<20Mb) [TINY]("http://bit.ly/2IpvVG")
   (~25Mb) [SliTaz]("http://www.slitaz.org/en/") with JWM
   (29Mb) [LinuxConsole]("http://www.linuxconsole.org") Micro - GUI IceWM
   (<50Mb) [DamnSmallLinux]("http://damnsmalllinux.org/") - based on Knoppix, but with JWM and Fluxbox
   (44Mb) [Jinx]("http://jinx-linux.sourceforge.net/") - with Opera 9, Flash, Textmaker and Planmaker.
   (<50Mb) [Austrumi]("http://bit.ly/1nsXrF") - A business card size bootable live CD Linux distribution based on Slackware.
   (<50Mb) [MeanPup]("http://puppylinux.org/wikka/MeanPuppy") -A live distro based on Puppy
   (<50Mb) [Mitrax]("http://www.mitraxlive.com/") -A live distro based on Slackware
   (<50Mb) [Luit]("http://bit.ly/5jWbE") - XFCE and ROXFile 

[SIZE="4"] Under 128Mb[/SIZE]
Best in category: Puppy Linux.

   (80Mb) [LiMP]("http://limp-vkk-ver1.sourceforge.net") - Linux Media Player turns your PC into a jukebox.
   (<100Mb) [NimbleX sub100]("http://www.nimblex.net/") - Slackware based with KDE (or 69Mb with no GTK or multimedia)
   (<100Mb) [Puppy Linux]("http://www.puppylinux.org/") - live with JWM (see also eBoxPup, IcePup, LitePup, MiniPup, PizzaPup, SafePup, SurfPup and XPuppyPro)
   (82Mb) [KioskCD]("http://kioskcd.com/") - web browser only
   (84Mb) [DamnSmallLinux-Not]("http://damnsmalllinux.org/dsl-n/") - distro using Knoppix hardware recognition with Abiword and Gnumeric.
   (<128Mb) [DeLi]("http://delili.lens.hl-users.com/") - Desktop Light
   (<128Mb) [SLAX Popcorn Edition]("http://www.slax.org/")- A smaller version of SLAX with Xfce

Best in category: Ubuntu GTK1.2 Remix.

### Post by stranger on 2007-12-22
You've got several Slackware-based systems on there, but I don't see any mention of the options Slackware itself gives you.  First, there's zipslack, which fits on a 100MB zip drive or a FAT partition.  And, starting with 12.0 Slackware has USB and PXE installation instructions.  The USB instructions are for a 24MB partition.  PXE is for a network install.

Plus, you can just install Slackware and install a very lean set of programs.  It takes a little choosing, but I've installed it with basically X and Fluxbox with a half-dozen main programs -- firefox, etc. -- on an ancient AMD K6-2 laptop.  It ran very well with that, considering.  I also considered just downloading an older version of Slackware that was smaller and lighter, then upgrading just the parts I needed to.  In the end, it was easier to go with the current version, but that is an option.

If you remember back to MS's attempt to discredit the idea that Linux runs on anything, the one distro (IIRC) that ran OK on a 486 was Slackware.  It takes a bit more doing that Ubuntu, of course, but once it's set up, it's a great system for an older box.
