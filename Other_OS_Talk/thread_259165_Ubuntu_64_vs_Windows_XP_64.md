### Post by kopilo on 2006-09-17
First of all, this is like comparing apples and oranges. In other words, 'horses for courses' each person will find that a difference operating system will fulfil their needs and wants 'better'.

Anyway, Windows is known for being able to natively run 'popular games' such as GunZ etc, and for which _some_ games do not have a real alternative in linux.

So I've been considering to run the 64bit version of Windows XP as a dual boot with Ubuntu drapper AMD 64. When I came across a statement in the latest version of PCAuthority (October 2006).

[quote=Nick Ross] Unfortunatly, the 64-bit version of Windows has always been intended for the corporate market and the lack of drivers is a natural result of this. It's simply not worth the manufactur's time creating drivers for an operating system that is not mainstream.[/quote]
Found on page 22, under 64-Bit Isolation.

Whilst the latest version of Ubuntu 64 has been having some issues, personally I have found there to be some resolve and I'd rather be able to use a device with possibly considered not the 'best' software, then not to be able to use the devices at all.

I'll keep running a 32 bit version of windows on the other partition but certianly won't be using it for mainstream operations, such as programming, compiling and running my own developed software.
