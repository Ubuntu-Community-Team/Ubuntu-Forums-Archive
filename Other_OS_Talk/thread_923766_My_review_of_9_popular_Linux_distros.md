### Post by Timberwolf5578 on 2008-09-18
I am new to Linux, and this is a review of 9 Linux distros I have tried over the past 3 weeks.  This review is just **my opinion** of these distros:


**_Mandriva 2008_**

Installation = Easy, no problems.

Hardware detection = Very good, even installed Nvidia drivers for my video card.

Stability = Seemed very good.

Performance = Somewhat slow even for a KDE distro.  Seemed like too many services were on by default.  A small bug with the package manager which caused it to be too "busy" to access the repos sometimes.

Looks = Very nice looking.

Overall = A decent distro.


**_OpenSUSE 11_**

Installation = Easy, no problems.

Hardware detection = Good, but didn't install Nvidia drivers for my video card, and I just couldn't get them to install properly.

Stability = Seemed very good.

Performance = Slowest of all the distros I tried.

Looks = Nice looking.

Overall = A decent (but slow) distro.  I wasn't happy that I couldn't install the Nvidia drivers for my video card properly.


**_Debian Etch_**

Installation = Not as easy as the other distro I tried, but ok.

Hardware detection = Seemed good overall, but I had to install Nvidia drivers with EnvyNG.

Stability = Seemed very good.

Performance = Relatively fast.  But a very barebones distro.

Looks = Very plain looking.  Worst looking internet browser fonts of all the distros I tried.  I think the internet browser in Debian was called Iceweasel, which is, if I am not mistaken, similar to Firefox.

Overall = Just ok.

[U][B]
Sabayon 3.5 POD[/B][/U]

Installation = Easy, no problems.

Hardware detection = Very good, even installed Nvidia drivers for my video card.

Stability = Seemed very good.

Performance = Fast, but I think mainly because it uses XFCE.  But Firefox was slow for some reason.

Looks = Very plain looking because it uses XFCE.  I tried to install Gnome from the repos, but it didn't work.

Overall = A decent distro, but not being able to install Gnome was a big letdown.


**_PCLinuxOS 2008 Gnome_**

Installation = Easy, but partitioner had problems.

Hardware detection = Very good, but had to install Nvidia drivers from package manager.

Stability = Seemed very good.

Performance = Seemed about average speed for a Gnome distro.

Looks = Nice looking.

Overall = A decent distro, but I wasn't happy with the partitioner.


[B][U]PCLinuxOS 2007 KDE
[/U][/B]
Installation = Easy, no problems.

Hardware detection = Good, but I had to install Nvidia drivers from the package manager.  And I couldn't get my DVD/CD writer drive to get recognized in the DVD/CD writer program (forget what it was called).
 
Stability = Seemed stable, but had one application crash.

Performance = Fast for a KDE distro.

Looks = Very nice looking.

Overall = A very good distro, but I wasn't happy with my DVD/CD writer not being recognized.


**_Pardus 2008.1_**

Installation = Easy, no problems.

Hardware detection = Very good, but I had to install Nvidia drivers from the package manager.  

Stability = Seemed very stable.

Performance = About average speed for a KDE distro.  But video card performance was slow for some reason.

Looks = Very nice looking.

Overall = A decent distro, but I wasn't happy with the video card performance.


**_Ubuntu 8.04_**

Installation = Easy, no problems.

Hardware detection = Very good, but I had to install Nvidia drivers with EnvyNG.  
 
Stability = Seemed very stable.

Performance = Very fast for a Gnome distro.

Looks = Decent looking.

Overall = A very good distro.


**_Linux Mint 5 Elyssa_**

Installation = Easy, no problems.

Hardware detection = Very good, but I had to install Nvidia drivers with EnvyNG.  
 
Stability = Seemed very stable.

Performance = Very fast for a Gnome distro.  The fastest distro I tried.

Looks = Very good looking.

Overall = An excellent distro.
_________________________________________________________________________

After trying all these distros, I decided to use **Linux Mint** for now because, in my opinion, it is the best distro of the bunch.

### Post by Greyed on 2008-09-19
Sorry, you're both wrong.  Yes, they can have different kernel versions.  Yes, they can have different patches.  But there is absolutely nothing saying that the defaults must be adhered to forevermore and evermore.  Since you're talking kernel level and the kernel is GPLed then those patches must be released with the binaries.  Any other distribution can take advantage of it and any end user can take advantage of it.

Differences in hardware support simply does not exist because in the end they are using the same code, period.  When you get to the domain of options turned on-and-off that is the domain of the user tweaking their system to taste, period.  It's just that simple.

The differences in distributions come from things unrelated to code.  Release cycles, quality of their packages, the capabilities of their packaging system (which depends heavily on the preceding item) and the testing they put into their releases before booting them out the door.  But at the end of the day if you take 3000 programs, compile them from the same source and shove them out the door as a distribution your 3000 programs should (and probably do) support the exact same hardware as any other distribution on that path.  Anyone who thinks differently is fooling themselves with placebo effects, nothing more.
