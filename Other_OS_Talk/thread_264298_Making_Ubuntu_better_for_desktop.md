### Post by WebCM on 2006-09-24
Ubuntu on other desktop distributions' background.

Ubuntu is a good and fast-growing distribution but it isn't **enough** user-friendly. What are advantages and disadvantages?

**Introduction**
I've had MandrakeLinux 10 (long ago) then OpenSUSE 10. They were installed from Internet (FTP). I've seen screenshots on OSDIR. The  second has more admin tools.
Today, I installed Ubuntu from CD.

**Installer**
Ubuntu booted from CD. Okay, it's possible to see the system before installation. Without thinking, I clicked "Install". The installer is very simple but...

_1. Where can I select programs to install?_
I think all programs are installed. It isn't so much but it should be possible to control it.

_2. Changing filesystem._
I installed Ubuntu on OpenSUSE's reiserFS (is it good choice?). Changing filesystem (e.g. on EXT3) wasn't possible.

_3. Boot manager._
Any tool for configuring boot manager doesn't exist! Why must I change booting menu manually in .conf file?

**After the installation**
GNOME is quite light desktop enviroment. It's an advantage. :) However, lack of important configuration tools makes Ubuntu less user-friendly. It will be good if you increase **operation's performance** (mainly system start-up).

Where is control panel to Gnome configuration (choosing from menu is so uphill)?

I propose to make Ubuntu Control Center (e.g. installed from the Internet if there is so few space on CD).


What do you think about it? I wish I hadn't erase OpenSUSE. Maybe **KUbuntu** is more user-friendly (KDE have a lot of configuration options and tools but it uses a lot of RAM / swap memory). :)

> _1. Where can I select programs to install?_

A default suite of applications is installed. It's perfectly possible to change this later with GUI tools or by the command-line.

ReiserFS is a decent choice, though I made all of my partitions ext3. You say ext3 wasn't possible? Were you installing on a previously formatted partition?

> **WebCM said:**
> _3. Boot manager._
Any tool for configuring boot manager doesn't exist! Why must I change booting menu manually in .conf file?
I believe it's a very rare operation, and the Ubuntu installer normally takes care of things perfectly well (certainly better than the heavy-handed approach of the Windows installer).

Erasing an old OS before you're *sure* that you're happy with a new OS is a pretty hasty thing to do in my opinion. I dual-booted Windows XP and Kubuntu for six months before deleting my Windows partition.

> **WebCM said:**
> Maybe **KUbuntu** is more user-friendly (KDE have a lot of configuration options and tools but it uses a lot of RAM / swap memory). :)
I thought we'd already debunked the 'KDE is a memory-hog' thing ages ago? Today, KDE and GNOME take up comparable amounts of memory, with KDE even beating GNOME on some occasions (though lightweight desktop environments like Xfce or simple window managers like IceWM beat both).
