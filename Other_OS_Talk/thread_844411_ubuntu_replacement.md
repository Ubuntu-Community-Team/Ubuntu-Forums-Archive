Fedora - PackageKit for graphical installer; Yum - very fast command-line installer (very similar to apt)

openSUSE - Yast - best control center and best configuration GUI in the Linux world; zypper - command-line install tool (the version in openSUSE 11 is faster than apt and probably better)

RPM has improved a lot lately - better compresion, faster install times... and so far RPM based distros have done their job for me better than Debian or Ubuntu or other Debian based ones. Also Fedora supports a version for 13 months and openSUSE probably longer... since a 10.2 install still receives updates. While Ubuntu only supports LTS versions more than 6 months... and bugs are taken more seriously by Fedora or openSUSE developers, Ubuntu devs are more concerned with the next release than the current one.

That's my opinion about what you said... (with a lot of mistakes I might add... but nobody's perfect and they will become less in time :D). As for a more on topic answer... my distro of choice varies between Fedora and openSUSE and currently I'm on openSUSE 11. I like Ubuntu, I used Ubuntu and I've learned one thing... If something doesn't work in this release it might work on the next... so I gave up on it for the moment... at least until they start concentrating more on bug-fixes than on the next release.

PS: I might add that even with this approach it's not a bleeding edge distro and it's not even the most stable one out there... that should get someone thinking.

I would like to suggest you have a look at Arch Linux. Yes, it's not Debain based but a very nice distro.

It's VERY stable.

Software is new as it uses a rolling release system. 

It has no default DE, you pick your own which would be gnome.

> **oxtail75 said:**
> what would a netinstall intail the reason i don't wont to use sumthing like suse or fadora is they do not offer the long term updates that i really need as i am about to start university and wont have the time to install a newos ever year  do rpm distros have anything smiler to apt-get as i am not a fan of graphical package managers

Arch offers a FTP(net) install as well as a Core install. You build your own and it is really not hard, just follow the beginners guide to the letter. A LiveCD will be available later, still in the works.

The package manager is Pacman and it's as simple as apt-get to use but more powerfull. Then their is Yaourt which uses the same command syntax as Pacman but also offers installing from AUR source repositories. If you want a gui interface there is Shaman which is relatively new (RC2) but helluva nice to use.

Keep in mind Arch uses a rolling release system so packages are constantly being updated (and I'm not talking about unstable & testing which is also an option) so you never have to reinstall the entire distro. A simple "pacman -Syu" or "Yaourt -Syu" will keep your system 100% updated.

Yes, the initial install takes a bit longer than your average ubuntu etc. but it is well worth it. No bloat, no crap, just what you want. The Arch community is also a nice one and then you will also find a lot of ex-ubuntoers in the Arch subforum right here.

### Post by Sorivenul on 2008-07-01
I second the opinions of Arch or openSuSE. 

Arch, while not for the absolute beginner, is still very easy to install, use, and maintain.  The Arch community is awesome, as is the Wiki there.  Check them out before an install for more details - and don't forget the beginner's guide. 

openSuSE is my current main OS (11.0, KDE4), and runs beautifully. The forums are more sparse than the offerings of Ubuntu or Arch, but overall documentation is still excellent, and system configuration is breeze.

Neither of these distributions use apt-get, but that is not much of a hindrance, and IMHO is a blessing. Arch's pacman package manager is possibly the best I have used next to the FreeBSD ports system.  But that's neither here nor there.

The important thing is to find what works for you. The idea that Gnome my be causing your problems is very real, so definitely follow the advice to try other DEs as well before wiping your Ubuntu install. The other thing is to consider hardware incompatibility. Don't rush out for new hardware.  A spindle of blank CDs or DVDs is much less expensive than new hardware, and trying new distributions can be as much fun as it is annoying sometimes.

I understand this probably seems like a short summary of the thread so far, but sometimes a nice clean presentation helps put things in a new light. 

Good luck to you!
