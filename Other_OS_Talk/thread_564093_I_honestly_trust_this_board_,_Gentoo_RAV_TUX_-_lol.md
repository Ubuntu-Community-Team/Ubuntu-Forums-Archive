### Post by fistfullofroses on 2007-09-30
Honestly, any system can be configured to perform better. In some distributions it is merely flaunted a lot more. Also, no distribution is more "innovative" than others. You may just have a program or two that performs a certain task with more automation than another. So, what it comes down to is preference. I could compile everything from source, on Ubuntu. I could set default make flags. I could compile my own kernel. I could do whatever. I choose not to do so though. I chose Slackware with a very customized package set, customized kernel. Other than that I set my daemons and services myself, instead of defaults. I have extremely goof performance... without the wait time of a Gentoo system. So, figure out the distribution you prefer, and proceed to optimize that system. If you love emerge, Gentoo may be for you. Also look into Sabayon. If you like apt stick to Debian derivatives. If you like pacman, stick to Arch. It is all about preference when it comes to Linux. You have choices.

### Post by yabbadabbadont on 2007-09-30
What I really enjoyed while using Gentoo (for several years), was the ability to include/exclude whichever optional features a package may offer.  If you configure your USE flags properly, you can end up with a very lean system that still has all the features that you want.  Take Inkscape as an example.  In Gentoo, I could install it and only the minimally *required* dependencies.  In Ubuntu, it will pull in pretty much all of Gnome.  That, and the ability to always have current program versions, are the only things I miss.
