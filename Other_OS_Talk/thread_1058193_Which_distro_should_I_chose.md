### Post by psycho on 2009-02-03
all of the distros you've listed are "newbish" in the sense that they try to make installing, configuring and maintaining your system as painless as possible: in my opinion ubuntu does the best job of this, so if this makes it the most "newbish", ok...but that's a good thing, isn't it? or are you wanting some constructive pain?

if you want to learn linux any distro will do fine (they're all basically much the same under the hood) and personally i'd recommend NOT making the learning curve any steeper than it needs to be. if you are new to GNU/Linux, why not use a "newbish" distro? there is nothing stopping you from taking ubuntu to bits and editing the source code of everything from kernel drivers to word processors if you want to customise things and get into the "expert" stuff; on the other hand (and unlike some other distros), if you suddenly need to do something with your computer that you haven't done before (say, upload some videos from a friend's camcorder before s/he heads back overseas) you're more likely to be able to "just do it" with ubuntu, rather than having to take a crash course in the kernel's ieee1394 subsystem or whatever (great, you "learn more linux", at the expense of actually doing what you needed to do).

admittedly debian-based distros like ubuntu are a bit clumsier than others (gentoo, for example, is a tweaker's dream) when it comes to customisation, but they more than make up for this by working dependably and automating hundreds of tasks that you really don't need to be worrying about. i started out many years ago on slackware 3, but much as i love slackware and would recommend it if you want to "learn linux" in a general sense rather than learning any particular distro, i think you're probably much better off letting some of the "under the hood" stuff STAY under the hood. most people drive perfectly well without knowing how every tiny component of their vehicle works, and the same principle can apply to computers. if you want to spend hours and hours immersed in code and learning complex obscure hacks that will be obsolete and completely useless in six months, cool, avoid all things "newbish". on the other hand, if you want a life, it can be nice to have a computer that just does what you need it to do without your having to spend a day researching how.

the only reason i'm not using ubuntu myself now (i use debian) is that debian has a slower release cycle and stability is more important to me than bleeding-edge technologies (the "long-term support" editions of ubuntu are much the same: if that's what happened to be on my computers right now i'd leave it there). of the distros you listed, fedora is probably the most "bleeding-edge": radically new stuff tends to arrive there first as the fedora/redhat community has lots of kernel developers etc. for that reason it's prone to the occasional serious bug and so i wouldn't recommend it if you're really new to linux, but it's still a lot easier to pick up and run with than slackware, gentoo, arch, etc. as for opensuse, it's another good distro, though personally if i wanted to learn the redhat-style admin stuff (rpm and so on) then i think i'd go with a true redhat clone like centos (or with fedora, if i cared less about stability).

there are so many distros now and so many new ones arriving that you could distro-hop forever without ever spending enough time with one to learn it properly. my advice to linux newbies is usually "try ubuntu", but really the best thing is probably just to make the most of whatever distro happens to be in front of you; they're all fundamentally very similar. if the distro that happens to be installed on your computer right now is ubuntu, then i would definitely recommend leaving it there and enjoying it. it's an excellent distro and the fact that it works so well even for new users (its "newbish"-ness) is a point in its favour, not a point against it!

### Post by psycho on 2009-02-04
[quote=cardinals_fan]I disagree. I find more manual tasks to be considerably more complex and difficult on "beginner" distros with lots of automation. That's why I recommend Arch, SliTaz, and Slackware. They offer a system that is very easy to learn and truly understand.[/quote]
undoubtedly slackware is simpler (and so easier to "truly understand") than ubuntu; but much of this understanding is achieved out of necessity. great if you're using windows (or whatever) as your real os and just tinkering with linux in a vm or another partition in your spare time. not so great if you're a newbie using linux as your day-to-day desktop.

it's so easy to fire up synaptic and browse through the package descriptions, grab what you need (a gui video capture tool for example) and within seconds the system has not only fetched and installed all the necessary libraries etc., but made configuration changes all over the place (not just for the app itself but often in terms of device permissions, etc. etc.) and you're ready to do what you need to do. sure, this "behind the scenes" or "under the hood" stuff doesn't get learned (as it would on slackware, were you downloading source tarballs, going through the readme's etc. for dependencies, hunting them down, configuring building and installing them, finally configuring building and installing the app you wanted in the first place...maybe several hours later...) but the point i'm making is that there's still nothing STOPPING you from learning it; it's just that you're not FORCED to learn it. with ubuntu you can still read the docs and figure out what's going on under the hood; you can even do it all yourself manually and shove it into /usr/local if the automation is too "newbish". it's nice to have the choice of using the automation when you want it.

you're right that "manual tasks" are easier on a distro like slackware, but this is because they are built with the expectation that their users will be doing lots of "manual tasks" and so will appreciate the simplicity. for a newbie i think this "simplicity" can look more like complexity: the advanced user sees the bsd-style init scripts and thinks "cool, this is so easy to edit" and then comes to debian and thinks, "geez, what a mess..."; but the newbie is the other way around, and experiences automated bootscript changes as "simple" and having to understand and manually edit the scripts as "complex".

maybe it all comes down to whether or not the user is learning linux on a "play" system or using it as his/her actual productive pc? for newbies who want to actually USE the os they're learning, i still think ubuntu (or similar) is the way to go. sure, the stuff under the hood is a bit more complicated, but the user has more freedom to tinker with it or ignore it. slackware and similar "clean" distros (i hear arch is similar, though i haven't used it) don't really offer the "ignore it" option: the "manual tasks" that are so nice to carry out on them are also pretty much compulsory.
