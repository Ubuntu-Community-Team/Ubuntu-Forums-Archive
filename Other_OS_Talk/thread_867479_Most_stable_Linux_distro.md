### Post by GeigerRulesBig on 2008-07-24
To put down some general baselines for what I (perhaps somewhat redundantly) am looking for in a stable Linux distro. 

Desktop Linux distro with easily understood installation, especially when it comes to partitioning a drive.

Easily understood package management that works. (Feisty's Synaptic used to crash on me unexpectadly, very annoying. Gutsy had trouble doing lots of upgrades/installations) Easy to update and maintain all software packages.

No crashes while doing basic desktop tasks: E-mail, internet, IM, music/video playback. (The Ubuntu Gmail bug where your browser would crash if you closed an open Gmail tab almost made Hardy unusable for me.)

All officially supported packages/options work, regardless of almost any hardware peculiarities, without sacrificing options and usability.

Easy to get a system up to date and usable for all basic desktop uses. (For Ubuntu I download the restricted extras, VLC, and Amarok and my system is ready.)

No crashes with any officially included software. (Brasero crashes for me quite frequently.)

Good and easy hardware recognition. All hardware should work out of the box, or with minimal configuration. (I do not want to have to compile source code.)

System should not crash while being used for any normal desktop operating systems for extended periods of time. (My current problem is that my system freezes when left on overnight while running a torrent, pidgin, and Amarok. Unacceptable in my view.)


These are the things I would consider in a stable distro release. At the moment, I see no reason for keeping Hardy on my computer past Intrepid, even if Hardy is LTS. I have too many problems. I think these parameters outline a mature, stable distribution. Not bleeding edge. I think sometimes there is too much innovation in Linux distros, which keeps good distros teetering between the bleeding edge and stable. Or, there are distros like Debian, which while I have never used my impression is that it is stable and stagnant, un-inovative to the point that usability suffers. 

Perhaps these guidelines are too steep for any distro, but I would like to hear any thoughts regarding them.


-Geiger
