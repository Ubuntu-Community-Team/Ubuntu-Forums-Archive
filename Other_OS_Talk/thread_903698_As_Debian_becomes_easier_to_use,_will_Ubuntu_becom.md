### Post by Xenix on 2008-08-30
There is one key feature that Ubuntu really lacks in that Debian has, and that is the rolling release repositories of Debian Sid. I haven't had a breakage and I never need to to re-install if I don't wish to. Packages are updated constantly to, not just sitting there for 6 months becoming out of date, like Ubuntu does. Yes it is possible to update Ubuntu to the next version, however (and PLEASE prove me wrong) I hear of so many reports where people have had issues trying to upgrade Ubuntu to the next version.

Especially when using 3rd party repositories. I see the warnings where you should disable such repositores etc before upgrading? Why? What an annoying task to do. Once I get settled in, I don't want to have to re-install every 6 months just because upgrading to the latest version of Ubuntu might break the system.

In my opinion these are the 2 biggest flaws of Ubuntu:- 1) Out of date packages and drivers between releases 2) Having to re-install every 6 months simply becuse upgrading can cause breakage.

This is why I use Sidux now. I would really like to come back to Ubuntu, however I just can't face these two issues that Ubuntu suffers from. Even out of date software in the repo's wouldn't be so bad if someone could prove to me that upgrading from Hardy to the next... 10 releases or so would not break my system.

Yes Sid can break, but I have never had any breakages and they usuually have warnings plastered all over the place if a breakage is to occur. This is Debian's biggest advantage:- Rolling release.

This is not a troll, just my frustration wtih Ubuntu. If someone could correct my views (with proof) then I would be more than happy to come back to Ubuntu.

### Post by benerivo on 2008-08-31
First off, debian stable and testing/unstable are so different they are almost like separate distros. Ubuntu will never be pointless as long as canonical sell their support service (but debian / ubuntu may steadily get very similar), but as of now linux as a whole is still trying to have a point for 99% of computer users, and ubuntu seems to be leading the way. Debian doesn't try that hard to be easy to use for the desktop user (mainly due to it's stance on open source), and i'm not sure it ever will, but certainly some of the packages are individually getting more user friendly - eg. youtube played out of the box when i installed unstable last week.

This whole question depends on who you are. I think the original poster used the term 'pointless' in the context of 'it just works for what i want to do', but this depends on who you are. Me, i'm using debian unstable. I consider my setup better than what i can achieve with ubuntu and i'm tempted to say it's all quite easy, but for someone else, it is not a viable option.

The comments on the rolling updates are interesting. The daily updates are one of the best things about debian testing/unstable. Ubuntu does get very slightly outdated over six months but insignificantly so. It's main problem is having a huge user base,trying to do an automatic upgrade from one version to the next.
