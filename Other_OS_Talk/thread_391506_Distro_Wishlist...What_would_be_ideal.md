Gentoo based.  Primary install method would be a live-cd which starts a minimal desktop (maybe fvwm-crystal or fluxbox?).  This way config files would be generated and you would have the option of changing things as you wished.  The install would be from source.  But it would be done while in a graphical environment, so the computer wouldn't be worthless during install.  The install would go a lot like the current Gentoo live install, but work.  Nice gui's to set compiler flags.  Good help documentation (not just RTFM) in the installer to help users pick the right cflags.  Perhaps the installer would even set the flags based on the level of aggressiveness you chose to use.  Then there would be some "entire system meta packages" to choose from.  You could pick an entire working KDE OS, or Fluxbox, or whatever you wanted.  If you chose not to use on of the stock setups, a (working) Kurooish package selector would come up and let you choose there what ebuilds to use.  Then it would go do the compile/install after all setup, using the config files on the system, or that were affected by install options (eg make.conf).  After this long but rather user friendly install you would have a nice custom-compiled desktop all ready to go.  Proprietary drivers would be installable if needed, as well as custom compiling the kernel would be an option (with an implementation of xconfig to reconfigure it).
