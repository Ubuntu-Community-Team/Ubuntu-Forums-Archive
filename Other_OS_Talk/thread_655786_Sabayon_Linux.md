### Post by RS3York on 2008-01-01
I used it but not for too long.  I got caught trying to update the install to the most recent packages. 

If you plan on updating the entire system you should follow the instructions here : [http://wiki.sabayonlinux.org/index.php?title=Unoffical_Guide_To_World_Update](http://wiki.sabayonlinux.org/index.php?title=Unoffical_Guide_To_World_Update)

If you're interested in Gentoo then Sabayon is a nice step toward it.  You get a Gentoo system in the end but they save you a nice amount of effort.  Although you should appreciate the counter-argument that they are also preventing/slowing the knowledge you would acquire by going straight Gentoo.

The biggest thing you should prepare for is that even if you have a Quad-Core PC with spades of RAM and 10,000rpm hard drives arranged in stripped RAID config...it's still going to take you longer to do something like update OpenOffice because you compile everything from source.

Typically, the hold up in updating with a binary distro (like Ubuntu) is how fast your internet connection is.  Once you get the packages things move along quickly.  The hold up in updating Gentoo (and therefore Sabayon) is how fast your system is able to compile (and you're still going to want a nice bit of bandwidth to get the sources to compile in the first place!).  The posts you'll see about starting to update something big before going to bed with Gentoo are no joke.

To be fair there is an option to do some package management with binaries instead, but that sort of defeats the purpose since most distros are binary anyway.

As the Wikipedia article suggests there is some debate about whether the performance boost gained by compiling is worth it and if there is a gain at all.  If you want more power to play but in a binary distro then you might want to consider Arch Linux.  In addition the folks behind Vector Linux *claim* they have one of the fastest binary distros available (I know they used to claim to be the fastest not sure if they are still saying that).
