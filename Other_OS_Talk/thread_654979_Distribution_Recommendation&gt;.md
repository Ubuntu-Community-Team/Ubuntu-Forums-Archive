### Post by julian67 on 2008-01-06
Another vote for opensuse if it's important to make the package choices at install time and to manage the PC in GUI.  

Zenwalk is going to be one of the fastest distros you'll find, install is fast and easy, hardware detection is good and it works brilliantly..... but it's not for people who want to avoid the terminal. While the netpkg package manager is very good and you can install anything from source more easily than with Ubuntu you'd probably miss the huge range and diversity of pre-packaged applications that are available for the really big distros like/based on Ubuntu, openSuse and Mandriva. Zenwalk is maybe one to try after a few months of feeling comfortable with a more mainstream distro. 

Imo the best compromise between user friendly GUI and speed is found  in the Xfce desktop. My 1.6 GHz Pentium-M centrino laptop runs Xubuntu fine. Performance can be improved noticeably by adding the noatime option to /etc/fstab. This is the default on Zenwalk which I guess is one reason why it's feels snappier than default Xubuntu.  I think Ubuntu isn't necessarily the outright best in any one area but it is very, very good all around and doesn't have too many serious deficiencies. The other distro you should seriously examine is Debian 4.0. It definitely feels faster than the Ubuntu equivalent and is available as a 1 CD Xfce version. The Debian menus combined with the Xfce menus are not a pretty or coherent experience. I think the Shuttleworthism would be  "lacks polish". But it is lean and fast and relatively easy to run (if you read the docs!) and if you update the repo list to testing/lenny it will have much newer versions of most apps than the current Ubuntus while being at least as stable.
