### Post by MattBD on 2008-03-08
I often try other distros. If you have enough power, VirtualBox is a great way to try them as if you run into problems during installation you can still use a web browser on the host system to find the answer, and it lets you have a trial run before installing it on your hard drive.

In the last few weeks, I've tried Debian, Slackware, gOS, Open Geu, Xubuntu, Fluxbuntu and many others in VirtualBox. I've also tried putting on a command-line install of Ubuntu and adding window managers in an attempt to get a really fast desktop for low-spec computers. It's been very educational.

I highly recommend this method as if an OS is installed to VirtualBox you can play around with it to your heart's content and feel free to completely trash it. This is actually one of the best ways to learn as you can learn from your mistakes but also don't need to worry about breaking anything. You don't even need to burn an iso image to CD.

### Post by kazuya on 2008-03-12
> **mips said:**
> No I don't, arch is not ubuntu.

The documentation is pretty clear if you want to install the complete kdemod desktop, **pacman -S kdemod-complete** will do the trick but why anyone would want to do that is beyond me as it kinda defeats the purpose of kdemod.

First user has to enable those sources in the repo list via CLI. Not GUI. I'm sorry there are those not too comfortable with gui especially when they do not yet have the option to use the gui. They are still at the shell stage.

Kdemod-complete is awesome. Why you ask? I want as much things as possible already included, what if a user wants k3b already on there and a vast variety of other kde applications they love. Now, the user has to go searching for these apps individually. That is assuming the user understands their names and how to type them when doing pacman -S XX.  besides kdemod is still very light and superfast to use. I love the implementation of kde better on there with the nice domino theme. The question is why install kdemod-light? That does not make sense to me. I take functionality and features over just bare minimalistic especially when it does not really buy you any speed gain on your new PCs.

Arch is not Ubuntu - no one is arguing with you on this.; It has a different mindset and target users, however, it does not target most of the type users that distros like Ubuntu, Mepis, Zenwalk, Vector, etc do. These other distros are very simple to get started on. Once users get comfortable with these and attain slightly more understanding of linux then they may look for something else that yields more in the long run. Archlinux is a gem for those who give it a try and put a little effort. Also, a newbie cannot install Arch without some sort of print out handy. It is a fast install and the reward at the end is great, but it is no where near as easy as the Buntus, etc for beginning users.

This is from experience and also from basic common sense which is not so common. Do not assume everyone knows !wq is save and quit in vi   or that everyone knows the CLI, shell commands, etc.. They do not.
