Gonna have to disagree with you on the Knoppix choices. Package management, especially dist-upgrades, are horrible because the packages in Knoppix are mix-n-matched from all different branches of the Debian tree for the best custom LiveCD environment. ### Post by Iandefor on 2006-10-26
I'll post not only my selections, but *why* I chose them, too![LIST=1]
[*]Fedora Core. Fedora Core has been less kind to me than Ubuntu, which is *exactly* why I'm putting it at the top of the list. Using Ubuntu has done the exact opposite of expanding my Linux knowledge- it's too freaking easy. The problems I have with it are so routine and easy to fix that it's no longer fun or educational. Fedora, on the other hand, is more interesting. It's broken more often than Ubuntu has. And, when something breaks, I get to learn that much more about how Linux works. Why *wouldn't* it go to the top of my list, especially when it's not breaking so often I can't use it?
[*]Ubuntu. My old standby; what I use when I need something that just *works* without too much hassle.
[*]GeeXboX. Nobody with a computer that can play DVD's and has a decent speaker/screen setup should go without a 20 MB GeeXboX partition, in my opinion. It's especially good for laptops- there's nothing but media-playing utilities, so no cycles wasted on stuff you're not using when you want to play a DVD. AND you can customize it.[/LIST]So, there you go. It's not much, but it's what I've got.

### Post by danielph on 2006-12-02
> **one_stinky_bum said:**
> Just to get a quick opinion... which one (on the list or otherwise) is the fastest Gnome based distro for centrino laptops?

From my experience the fastest distro's are (in no order) Archlinux, Zenwalk and Gentoo. There are of course many others. These are just the ones I have had personal experience with and can say that they do boot up quicker and run with less bloat.

Archlinux is optimized for the 686 platform. I believe the Centrino will come under that umbrella or is it a 586 I am not sure.

Zenwalk is as fast and designed around a small package set, although I think you can use Slackware's packages. If you are familiar with Slackwares way of doing things it could be an easy distro for you. Or of course there is also Slackware itself as another option

Gentoo can be customized and optimized as required when you walk through the install process. It leaves you with (hopefully) a working system that has been tailored to your requirements and fair amount of customization to your hardware. Remember it is a source based distro and although it is fast, if you tend to upgrade or try  different applications, it may not be for you as you will spend a lot of time compiling. It is all taken care of through emerge, which is a great bit of software.

The faster distros tend to be the hardest as you have to work to build them and potentially maintain them. A distro like Suse will work well out of the box with a lot of hardware, but there is a price to pay for that. It is carry some extra weight to make it all work well. Some of that is true of Ubuntu also. Depends what you want.

None of these are "Gnome based", but you wanted speed :). You can install Gnome on top any of these, I tried it with Gentoo and Arch (I didn't try it with Zenwalk, but I don't see why it wouldn't be possible if you really wanted). I did notice the speed difference. If you really want speed then you are better to build from the ground up and add the Gnome components you want rather than going for a Gnome distro IMO. Or better still, go for a fast distro and put one of the boxes on it - Openbox, Fluxbox, Blackbox or Icewm for example. Next best XFCE4, Gnome and finally KDE, again IMO.

I know I can set this up manually, but it would be a nice, easy touch for the devs to add.  Also, if I click an executable fron the GUI in my Fedora install I can run it, rather than opening a terminal to run it, a really little thing, but a nice convenience.  These little details being missing adds up to Ubuntu dropping to this spot on the list.  I hope these types of things get addressed in future releases, as I think they are way more important than running Beryl by default (I love beryl, but these little details affect my productivity much more).

Just make sure you have a lot of time to be without that particular computer.

Honorable Mention, in no order: PC-BSD & Solaris (there is more to Unix OS's than linux), Symphony OS (if they ever go final it intrigues me), rpath (seems like great 64bit support), and Dreamlinux (Only Xfce desktop I've really liked)

Frankly, all Linux distros "feel" the same for the most part. If you're using GNOME, it's GNOME. Whether it's Mint, Ubuntu, Arch, Suse, Fedora, or whatever, it's still GNOME. The difference comes down to basically three things:

1) Hardware detection

2) The package management system

3) How much control you had over what all got installed

Once you've spent some time tinkering with things, there's not a huge difference between various distros given the same DE or WM in terms of "feel". That's one reason I couldn't bring myself to switch amongst the "big-time" distros. Mandriva or Fedora had no real benefit over Ubuntu since it's basically the same thing.

The only way to get a different "feel" is to use a system that really approaches things differently and for that I think you'd have to go for Gentoo/Slack/Arch, or anything that goes beyond "insert CD, hit next, reboot" for system installation. Even then, the difference in "feel" will largely be how you melded with the system during setup and such.
