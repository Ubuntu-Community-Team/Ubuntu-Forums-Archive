### Post by pelle.k on 2007-02-28
Ubuntu, SUSE, Fedora is built on top of a linux kernel. OSX tiger / panther is built from a darwin kernel, xp / vista is built from an nt kernel (well, sort of).

I guess you can say ${linux desktops} is a scattered effort in bringing a nice coherent enviroment to your pc for free. You have much software freedom/choice, but driver support pretty much sucks. Everything is done in diffrent layers, so it's dynamic, but not very efficient/fast. A jack of all trades.

OSX (apple) got cash flow, they've really concentrated on a single platform desktop. That make is easier to make it coherent even though it's (more or less) a unix platform. They got some support from vendors and the buisness world, so that one reason there's no "why isn't mp3 support included by default <whine>". Good defaults, good multimedia support.

Windows. Need i say more. They've got 99% vendor dupport, lots of cash and employees. But they also got viruses and malware to fight against. They provide a system with little flexibility, but if you fit their profile, you're good to go. That is install software. don't tweak the system. no flexible scripting system (though vista is rumored to have something new built in). No package manager. windows is ugly but functional (because of hardware support).

BEOS/Haiku is a little of all above. An attempt to take the best parts of existing OS:es. It does _not_ try to be a jack of all trades, but a desktop with a rich API (like qt) with most functions you will ever need built in (sort of if like qt + alsa + xorg + a lot of widely used libraries where a standard in linux). This API is built to be _very_ fast (and easy to use). That brings coherency and speed. Solves a lot of dependencies on exotic libs like you've got in linux, and you can more easily develop binaries that can be installed without resolving dependencies. Bash is built in for your scripting needs. There's no ugly xorg.conf to edit. Did i mention it's free and open source?

To answer your question: atm not very much, but it has the potential to be the very best desktop OS in the future.

      [IMG]http://haiku-os.org/themes/shijin/icons/folder_development_64.png[/IMG]The Google Summer of Code 2007 is on, and we want to be part of it. We are therefore starting this drive, with the ultimate goal of getting Haiku to participate as a mentoring organization. As part of the drive, we want to mobilize the community to help us better articulate our intentions to participate in the GSoC. In general, we want to create more favorable conditions to help us reach our goals.
  **What is the Google Summer of Code and why does Haiku want to participate?**

  The Google Summer of Code is a program that offers student developers stipends to write code for various open source projects. For Haiku, this not only represents an opportunity to have a paid for engineer for three months that would contribute with actual code, but it also means that we can raise our visibility in the open source world. It is also common for a student that works on a project through the GSoC to stay as a community member of that project, and that is obviously a welcome benefit.
  **Will Haiku be able to participate?**

  We really don't know. Our participation depends on whether our application as a mentor organization is accepted or not. The application period is from March 5th through the 12th. We expect to submit our application on March 5th or 6th.
  The list of accepted mentoring organizations will be announced on code.google.com on Wednesday, March 14th, 2007.
  **How you can help during the application period**

Now what is the draw to BeOS or Haiku. The draw is it is faster than you are probably use to using. It is easy to use or user friendly without having to know a lot about computers and it does most of the work for you like installing apps you want. It has a terminal or command line but you may never have a need to use it. Now the reason it is called a "media OS" was that it is designed to crunch a lot data FAST. Graphics, video, and music all require large amounts of data be processed and most computer systems or operating systems and software were designed for text, or the crunching of small amounts of data such as emails, chat, or simple text on a page. 

Most apps and operating systems were designed to use only one cpu or processor and has one single lane highway or thread they can go down. Fine when there is little data to be processed like typing an email. BeOS was designed from scratch in the 90s with no past system it had to be compatible with so they were free o design it anyway they pleased and what they designed was a system to use not one highway but 8 of them all at the same time and they designed to to use as many CPUs as the machine had installed in it with really no limit to the number of CPUs it could use all at once. This is true for not just the operating system but also for the software. Intel tested BeOS on up to 8 CPUs through simulations back in the 90s. Here is a demo film the company made in the 90s using some really pathetic hardware compared to today but the machine did have 2 CPUs. This is the real secret to unleashing the power in BeOS or Haiku. They run well on just one CPU but if you increase it to two, 4, or 8 CPUs it just gets that much faster. This will NOT happen on other systems. They will gain a "small increase in speed" overall.

BeOS saw this coming in the 90s just to try and handle multimedia demands on modern day computers and designed their system from the ground up to use them all at the same time, but they were just way ahead of it actually happening. The day of just one CPU machines is drawing rapidly to a close and Intel for one is talking about many more than just 8 CPUs per machine in the future.

Yes Linux, OSX, and Windows will run on these new machines but they are not designed too, and neither is there app software to effectively use all these chips and almost all of it was designed for a single CPU only. BeOS and Haiku is and was designed to do exactly this from the ground up and to be extremely user friendly.
