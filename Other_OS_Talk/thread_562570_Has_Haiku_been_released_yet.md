### Post by koki on 2007-11-20
> **jrusso2 said:**
> There is a vmware image that i downloaded and played around with.  I used to have BeOS long time ago, and Haiku is less like the old BeOS and a lot more like Linux.

In fact it seems they took a lot of Linux stuff and just changed it around a bit to make Haiku.

Haiku is pretty much the same as BeOS in terms of architecture, API as well as look and feel. The Haiku kernel is a fork of the NewOS kernel, and not linux, nor does Haiku use the X Window System or toolkits such as GTK+ or Qt (the Haiku graphics system is based on the AGG library). Haiku also uses OpenTracker, the file manager that was open-sourced by Be Inc. before they went under, and OpenBFS, a written from scratch open source implementation of the BFS file system that Be Inc. developed for BeOS in the early 90s.

Haiku does include the GNU tool chain, and a bash terminal. But when it comes to drivers and other areas of the OS such as the various kits (media, network, etc.), there is a lot written from scratch. Where the Haiku devs did get inspiration or borrow code, it is mostly from the BSDs rather than Linux, due to license compatibility, as Haiku is licensed under MIT.

There are nightly VMWARE image builds that you can play with here:

[http://haikuhost.com/housestrain/](http://haikuhost.com/housestrain/)

A sample VMX configuration file can be found here:

[http://haikuhost.com/haiku.vmx](http://haikuhost.com/haiku.vmx)

There is also a Weekly SuperPack put together by Haikuware.com consisting of a VMWare image with Haiku plus drivers and applications.

[http://www.haikuware.com](http://www.haikuware.com)

Please note that Haiku is still pre-alpha software. For geeks only. ;)
