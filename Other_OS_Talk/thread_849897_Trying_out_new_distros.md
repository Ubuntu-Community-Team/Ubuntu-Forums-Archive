### Post by DrPppr242 on 2008-07-05
I've run all of those except crux on my primary machine at one point or another and my personal opion is slackware's package management (or lack there of) is too inconveint for day-to-day use, FreeBSD lacks support for a lot of desktop programs/features that I use (networkmanager w/ wifi support was huge for me), gentoo is great and a good learning experience, but compiling every program when I install it took too long for my level of patience and finding out that you missed important features in your kernel (i.e vfat and ntfs support) gets annoying quick if your inexperienced like I am, and as for Arch that's what I use now, it takes a couple hours to set up properly but then it's great and has what I consider to be the best package manager available.  If your looking for something more familar though frugalware is based on slackware and uses pacman, Arch's package manager, without having to learn how Arch and BSD use rc.conf and rc.d.

It's mostly person prefernce, but the biggest reasons I like it are:

1. rolling release system, I personally hate the 6 month release cycle and the need for dist-upgrade vs upgrade. In Arch pacman (or yaourt) -Syu always updates the repos and upgrades to the most recent packages,

2. ABS allows the user to make a package from source and install (and later remove) packages that are in the repos.
