### Post by MethodOne on 2008-01-31
Look at my sources.list:

```
#official main
rpm ftp://mirrors.usc.edu/pub/linux/distributions/mandrakelinux/official/2008.0/i586/media media_info/hdlist_main main/release 
rpm ftp://mirrors.usc.edu/pub/linux/distributions/mandrakelinux/official/2008.0/i586/media media_info/hdlist_main_updates main/updates 

#official contrib
rpm ftp://mirrors.usc.edu/pub/linux/distributions/mandrakelinux/official/2008.0/i586/media media_info/hdlist_contrib contrib/release 
rpm ftp://mirrors.usc.edu/pub/linux/distributions/mandrakelinux/official/2008.0/i586/media media_info/hdlist_contrib_updates contrib/updates 

#official non-free
rpm ftp://mirrors.usc.edu/pub/linux/distributions/mandrakelinux/official/2008.0/i586/media media_info/hdlist_non-free non-free/release
rpm ftp://mirrors.usc.edu/pub/linux/distributions/mandrakelinux/official/2008.0/i586/media media_info/hdlist_non-free_updates non-free/updates

#official backports 
rpm ftp://mirrors.usc.edu/pub/linux/distributions/mandrakelinux/official/2008.0/i586/media media_info/hdlist_main_backports main/backports 
rpm ftp://mirrors.usc.edu/pub/linux/distributions/mandrakelinux/official/2008.0/i586/media media_info/hdlist_contrib_backports contrib/backports 

#plf free 
rpm http://ftp.cica.es/mirrors/Linux/plf/mandriva/2008.0/free/release/binary i586/hdlist i586

#plf free backports 
rpm http://ftp.cica.es/mirrors/Linux/plf/mandriva/2008.0/free/backports/binary i586/hdlist i586  

#plf non-free 
rpm http://ftp.cica.es/mirrors/Linux/plf/mandriva/2008.0/non-free/release/binary i586/hdlist i586 

#plf non-free backports 
rpm http://ftp.cica.es/mirrors/Linux/plf/mandriva/2008.0/non-free/backports/binary i586/hdlist i586 
```

Make sure you add a line somewhat like this to /etc/apt/apt.conf to prevent apt-get update from failing:
```
APT::Cache-Limit 212582912;
```

Remember that APT on Mandriva will be slower than on Ubuntu.
