### Post by Jack Waugh on 2008-11-19
//GO.SYSIN   DD     *

Best:  I never used it, but a friend of mine used to swear by Multics.

Best:  From reading the (August?  October?) 1981 _Byte_ Magazine special issue on the Smalltalk language, Smalltalk 80 on bare hardware seemed to be a pretty good OS.  It could no doubt be improved by adding multiuser support, adding finer grained concurrency, enlarging the maximum numbers of objects and maximum information that could be stored, and adding some kind of checkpointing, but I think it was interesting how the "OS" as the user saw it was made up of the concepts of the programming language and nothing else.

Somebody said that an OS exists only to make up for the deficiencies of the programming language, and that there shouldn't be one.

Best for its Time:  The Unix system.  Ritchie and Thompson developed it on a DEC PDP-7.  This example suggests to me that the best software is made on the shittiest hardware.

Best for limited hardware: FORTH.  Where else did you get a useful and complete programming system in 4 KiB of main memory?  It included an assembler, a compiler, a text editor, and device drivers, all in 4K.  File system?  One didn't need no steenking file system.  One threaded together disk pages by their index numbers from the start of the disk.  This worked fairly well for source code.  A disk page corresponded to a "screen" in the editor.  The code would explicitly refer to another page by its number.

Worst:  No matter how bad an OS you point to or devise, I think I could probably devise a worse one.  What about a program that runs on the bare hardware and does not interact with any peripherals, just an infinite loop that does nothing useful?  That's pretty bad as an OS.

Worst:  The systems that computer makers used to put out in the 70's had some pretty bad features.  As I understand from listening to people (again, not personal experience), some versons of OS/360 required you to specify the disk allocations for new files, in physical disk dimensions, such as cylinders and tracks.

Worst:  I thought 8.3 filenames and non-hierarchic file systems rather sucked.
