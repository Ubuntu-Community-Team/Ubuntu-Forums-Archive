### Post by smoker on 2007-07-29
the 'gparted live-cd' loads up gnome (gnome-partition-editor), which is why it will take a bit of time to boot up. the gparted included in puppy runs on the puppy window manager which is a lot lighter, jwm, i believe, so i think that will account for the speed of the puppy gparted, plus of course, puppy loads itself into ram so will run much faster than a distro that has to read from a live-cd, or read and write to a hard drive.
