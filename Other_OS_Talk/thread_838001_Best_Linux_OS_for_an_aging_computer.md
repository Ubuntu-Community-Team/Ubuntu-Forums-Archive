### Post by timzak on 2008-06-23
Ubuntu 8.04 can be configured to use about 130-140 MB of ram on bootup by disabling some services and auto-started apps.  That would leave you a little over 100 MB, which would let you run one big app at a time or several small apps at a time.  

Xubuntu can be tweaked down to 85 MB or so.  

I'm partial to *buntu because of ease of configuration.  If you want to stretch your wings and learn more, try other distros that are suggested here.  I've heard good things about Zenwalk and AntiX.  

Because my free time is so scarce, I always end up coming back to Ubuntu after trying other distros.  The amount of time I need to learn the various nuances of other distros just isn't worth it for me.  I dig the familiarity of knowing how to set up Ubuntu quickly, and exactly how I want it.

Edit:  733mhz is plenty of cpu for modern distros, you're mainly limited by your RAM and the i810 graphics subsystem.

### Post by mikjp on 2008-06-25
Any modern Linux will do the trick, as long as you do a custom install and do not install GNOME or KDE. I would forget also XFCE. And of course, forget the graphical installations offered by live CDs. You want a real installation media, not a live CD.

You might install Ubuntu server edition, and then ad some lightweight window manager + xorg. Openbox, Fluxbox, Blackbox and IceWM would be obvious choices.

If you want icons on the desktop, use idesk.

Dont use OpenOffice.org for writing -- use Abiword. But anyway, 256 MB should run fine as long as you dont spend the RAM for KDE or even worse, GNOME.
