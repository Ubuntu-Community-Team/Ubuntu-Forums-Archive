**Synopsis**

 Sorcerer is a source-based Linux distribution. Source tarballs are downloaded directly from software project homepages or as patches when an old source was previously downloaded. They are compiled for the architecture and with the optimizations that the system administrator specifies. Finally, it is installed, tracked, and archived for easy removal and upgrades. Sorcerer has both command line and menu driven source mangement programs.
 The 10 most compelling [reasons to choose Sorcerer.]("http://sorcerer.aakin.net/top10.php")

**Getting Started**

 Getting started with Sorcerer is easy:
  [LIST=1]
[*]Download a compressed bootable Install/Rescue ISO9660 CD image.
[*]Download a compressed sources CD image.
[*]Decompress and burn both to separate CDs
[*]Boot the Install/Rescue CD.
[*]Login as root with empty password
[*]Invoke the menu driven installer /sbin/sorcerer.install[/LIST]

### Post by RAV TUX on 2007-01-01
> **Top 10 Reasons to Choose Sorcerer**

 [LIST]
[*]Easy Management Sorcerer boxes are managed through dialogs or by a handful of commands with intuitive and easy to remember options. It is very simple to automatize the management tasks by writing a few custom scripts and/or crontab entries.
[*]Self-Heal Feature The system is regularly checked for missing libraries and/or dependencies. If a problem is found, the system is automatically repaired by (re)installing missing libraries/dependencies. Of course, you can heal your box any time by invoking the command heal.
[*]Easy Updates Worried about updates? Don't, Sorcerer way is to run augur easy.
[*]Dependencies are under Your Control Choose the dependencies you like or need, not the ones that somebody else have chosen for you.
[*]Best Uninstall (dispel) Tool Available Are you tired of cleaning a box after installing a rpm package that did not remove installed dependencies? It is a no-brainer with Sorcerer: All dependencies are dispelled unless required by another spell.
[*]Easy Spell Addition Is Sorcerer lacking your favorite application? Simply write the new spell yourself (and contribute to the grimoire if you like). Sorcerer spells are usually trivially easy to create.
[*]Security Patches and new versions are available as quickly as for major Linux distributions. Furthermore, Sorcerer offers safe holds that enable the software to be held from upgrades unless the installed version becomes a security risk.
[*]Standard Compliance Sorcerer offers the best Linux, *NIX, POSIX compliance. It also conforms to the Linux filesystem tree standard.
[*]Keeps the /etc Directory Clean Sorcerer installs only needed configuration files not all the possible ones for every potential package/package configuration.
[*]Best Development Platform Development tools are a necessary part of the default setup. Sorcery tools help you to focus on the development without spending much time keeping the system clean.
[/LIST][http://sorcerer.aakin.net/top10.php](http://sorcerer.aakin.net/top10.php)

> **Sorcerer Documentation**

 **README FIRST**

 Welcome to Sorcerer documentation. Please read this page through, even if you have no intentions of reading anything else.  
 ***CAVEAT***

 It is important to have sufficient RAM available. If you keep /tmp as a recommended tmpfs file system then you should have RAM should probably be at least 256 mB.  
 It is also vitally important that the kernel have both the EXPERIMENTAL devfs and tmpfs support compiled directly into the kernel. Do NOT de-select these options during kernel configuration or the agetty processes will go into a respawning loop, and login will not be possibile. 
 To avoid problems after install, you should follow these instructions: 
 
 ***RECOMMENDED READING***

 For BEGINNERS, we recommend reading through all of the documentation at least once. preferably twice. Since we realize that many people will not be doing this, we recommend that at the very least you read the system administrator, install, and the first two parts of the FAQ.  
 For MORE ADVANCED users and ANYONE HAVING PROBLEMS , we would add the rest of the FAQ, and  the spell documentation.  
 If you desire extra security on your box you will want to read how to set up encrypted loopback filesystems in the newbie FAQ. Please see these FAQ items both [here]("http://sorcerer.aakin.net/docs/newbie.faq.html#elfwhy") and [here]("http://sorcerer.aakin.net/docs/newbie.faq.html#elfhow") for more information.

 Forks of the GNU/GPL rough draft of the sorcery tools are available at the following websites. The implementation of sorcery available at them is based upon old, buggy, experimental, depreciated sorcery.
