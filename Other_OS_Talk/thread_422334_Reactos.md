From a corporate point of view you don't really need to be on the Microsoft cutting edge around 50% of the time you are going to be running legacy applications in a highly controlled environment and not neccessarily using modern hardware. But if Microsoft decide to stop supporting the version of Windows you use then you have two options you can upgrade your hardware and software at great expense possibly losing the compatibility of your legacy software or risk running on an unsupported operating system. ReactOS adds a third choice because it is open and free as in speech it will be adapt to the user's needs rather than the user adapting to the software, if you want Windows 98 compatibility you got it, if you want to optimise to your lower spec machines well then it'll happen, just like it has happened with Linux. You have to remember Linux started out as a clone of a commercial closed product so there's no reason to think that ReactOS can't grab some of the OS market in the same way Linux took marketshare from UNIX.
