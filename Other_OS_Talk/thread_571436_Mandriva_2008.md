Here's the official community release announcement (there'll be a press release with more commercial stuff in it in a few days):

We're proud to announce that Mandriva Linux 2008 is now [available for download](http://www.mandriva.com/en/download.html) via BitTorrent, and on the network of public mirror servers.

The result of six months of heavy development and testing, 2008 includes all the latest software and many enhancements over previous Mandriva Linux releases. You will find KDE 3.5.7 and the new GNOME 2.20 already integrated, a solid kernel 2.6.22.9 with fair scheduling support, OpenOffice.org 2.2.1, cutting-edge 3D-accelerated desktop courtesy of Compiz Fusion 0.5.2, Mozilla Firefox 2.0.0.6, and everything else you've come to expect from the latest Mandriva Linux release. We have integrated a reworked hardware detection sub-system, with support for a lot of new devices (particularly graphics cards, sound cards and wireless chips). Also, the addition of new features and ongoing improvements to the renowned Mandriva tools continues, including a wizard to import Windows documents and settings, the new network configuration center, and improvements to the Mandriva software management tools. 

You can read about the most exciting new features of Mandriva Linux 2008 in depth in the [Release Tour](http://wiki.mandriva.com/en/Releases/Mandriva/2008.0/Tour). The [Release Notes](http://wiki.mandriva.com/en/Releases/Mandriva/2008.0/Notes) contain important information on changes from previous releases. The [Errata](http://wiki.mandriva.com/en/Releases/Mandriva/2008.0/Errata) will contain information on any future known issues and solutions for them.

Mandriva Linux 2008 was developed by the Mandriva development community, based around the Cooker development distribution. This open, community-driven development system has been in place since 1998, making it one of the longest-standing open source development communities around. The involvement of the Mandriva development community helps us to make Mandriva Linux one of the largest, most up-to-date, integrated, internationalized and standardized distributions available, for which we thank our contributors greatly. We're proud of the standardization efforts included in Mandriva Linux 2008, including support for the Freedesktop.org XDG menu, user directories and icon theme specifications.

Mandriva Linux 2008 is [available for download](http://www.mandriva.com/en/download.html) free of charge at the Mandriva website. The One installation CD is the recommended download with everything you need to start using Mandriva Linux 2008: it comes with a full KDE desktop and application suite, NVIDIA and ATI proprietary video card drivers, Intel wireless firmware, Adobe Flash and Sun Java browser plugins, all included.

Enjoy your brand new Mandriva Linux 2008 system!

Please refrain from submitting this release announcement to general news sites for now. We are waiting for a couple of things before sending it to them as an official story in a few hours' time. News site editors, if you come across this story, please wait for us to submit it as an official news story. Thanks!

Okay, from the top :)

I know about the odd browser window bug now. It's the first time wizard screwing up. It gets its content from a page on [www.mandriva.com](www.mandriva.com) which has been broken for some of today. It should be okay soon if it's not already.

On the Club - we haven't quite completed the planned changes yet. As has been mentioned in various places, paid Club membership is being abolished. Anyone with a my.mandriva.com account (which is free) will be a 'Club member'. The paid Club membership levels and privileges are abolished and replaced by a simple Powerpack download subscription service: pay €59 / $69 per year for Powerpack download access, no relation to the Club concept any more.

These changes will be completed within a week or so, I believe - right now they're not complete and the paid membership system is still in effect for the moment.

No, synaptic is not in task-gnome, and it does not replace rpmdrake / urpmi. synaptic is not a part of GNOME in any way, so there's no reason it would.

### Post by AdamWill on 2007-10-10
This is for anyone getting NOKEY errors trying to install packages from the online repositories for Mandriva Linux 2008.

The problem was that the pubkey files, which contain the keys used to sign packages in each repository, were missing from the media_info directories in the 2008 tree, so when it was adding the repositories, Mandriva could not get the key for each repository. Since the repository seemed to have no key associated with it, but the packages were signed, when you tried to install any package, Mandriva would give you the NOKEY warning.

This has been fixed on our master mirror around two hours before this post. I have checked several mirrors and found that at the time of writing this post, a couple of mirrors have synced up with the fix already, but most have not.

Once all the mirrors sync up with the fix, the problem will not occur again for anyone adding repositories. However, if you have already set up your repositories, the problem will not be fixed automatically. You must remove your repositories using the repository configuration tool, which can be found in the Mandriva Control Center, or the urpmi.removemedia command line tool. You can then set the repositories up again (using the repository configuration tool, or urpmi.addmedia) and you should no longer see the errors.

I will post a follow-up message when my checks indicate that most mirrors are synced up with the fix.

We’re sorry for the inconvenience.

### Post by AdamWill on 2007-10-11
Igknighted: you touch on an excellent point. :)

Handling external kernel modules (those that aren't built and updated right inside the kernel package) is a nightmare for all distributions. Our system is based on DKMS. And yes, it does have some limitations, most of which you identified.

Unfortunately we just didn't have time with 2008 to make switching kernels a completely trouble-free process. So yes, when switching kernel flavors in 2008 you will have to be a bit careful about external modules. The reason this is not yet documented anywhere is simply, frankly, because I didn't yet have time to test it out carefully and document the process precisely. I was just talking to someone about it in IRC, though, so now I more or less know.

First off, switching kernel flavors is only something you should really worry about if you installed from One and you have over 1GB of memory. If you have 1GB or less the kernel installed by One is fine. If you installed from Free or Powerpack, it'll have selected the best kernel for your hardware automatically. It's only One that installs the desktop586 flavor regardless of your hardware. That's because One doesn't use an installation system like the other flavors: One's 'installation' really just consists of copying the entire One image to the hard disk (that's why it's so fast). One uses the desktop586 kernel flavor in order to support the widest possible range of hardware. So no matter what CPU or how much RAM you have, you'll get desktop586 if you install from One.

So, assuming you installed from One and you have several GB of RAM (you lucky, lucky person), how do you switch kernels?

In a nutshell, you just have to install the -latest package for the kernel flavor you want to use, and the matching packages for any external modules you use. In most cases the flavor you want to switch to is kernel-desktop. kernel-laptop has a few tweaks that are supposed to reduce power consumption on laptops (mainly, USB suspend and something to do with the timer). kernel-server has 64GB RAM support, I don't think it has any other difference from kernel-desktop (which supports 4GB) any more. So assuming we're going with kernel-desktop, you should install:

kernel-desktop-latest

and then all packages named:

(module)-kernel-desktop-latest

where (module) is the right name for the external modules you use. Here's the names for the most commonly used external modules:

nvidia-current (the NVIDIA 100.xx driver, for all GeForce FX and later cards)
nvidia96xx (the NVIDIA 96xx driver, for GeForce 2MX, 3 and 4 cards)
nvidia71xx (the NVIDIA 71xx driver, for GeForce 2 other than MX and earlier cards)
fglrx (the ATI 8.40.4 driver, for all Radeons from 9500 through to X1950)
fglrx-hd2000 (the ATI 8.41.7 driver, for Radeon HD 2xxx cards)
madwifi (the madwifi driver for Atheros wireless chips)

there's some other less commonly used ones. You can get the full list by doing rpm -qa | grep desktop586-latest .

So say you want to switch to the kernel-desktop flavor and you have a GeForce 6600 card. You'd install these packages:

kernel-desktop-latest
nvidia-current-kernel-desktop-latest

say you have a Radeon X850 and an Atheros wireless card, you'd install these:

kernel-desktop-latest
fglrx-kernel-desktop-latest
madwifi-kernel-desktop-latest

then you reboot, the new kernel is in the boot menu (I *think* it would also become the default kernel too), and all should be good.

Yes, I know this process is something of a pain. :( We wanted to make it work more easily for 2008, but just didn't have time. We at least improved on the situation in 2007.1, where just doing an official kernel update tended to break external modules unless you happened to have exactly the right packages installed for DKMS's dynamic recompilation to kick in. I've heard this can happen on Ubuntu too, it's a pain for everyone. We will hopefully have time to make switching flavors easier on 2008.1.

One last note on the other kernel varieties available: only -desktop , -desktop586 , -laptop and -server are different flavors of the official Mandriva kernel. 

The other different types of kernel package are really *different builds*. -multimedia and -tmb are the two principal volunteer alternate kernels for Mandriva, maintained by Thomas Backlund and Danny Tholen. Thomas and Danny both help the Mandriva kernel team out with the official kernel build now. -multimedia is a bit deprecated at the moment and is only really useful if you need a 2.6.17 kernel for some reason. -tmb is good if you need a really bleeding-edge kernel - it's kept up to date with the latest upstream RCs (in 2008, it shipped as 2.6.23rc7 or rc8 I think) and it has some experimental patches that aren't tested enough to go in the main kernel. It's basically used as a testbed for the main kernel these days. 

-linus is a stock upstream kernel with no Mandriva patches. The point of this is so when people hit kernel bugs we can ask them to boot kernel-linus and see if the bug happens there. If it does, it's an upstream bug and we send them off to the kernel bugzilla to get it fixed. :)

-mm is the upstream -mm kernel patchset, again with no changes from upstream.

-rt is Ingo Molnar's realtime patchset, again with no changes from upstream.

-vserver is built with the VServer virtualization patchset, again otherwise unchanged from upstream.

-kerrighed is built with the Kerrighed patchset used for clustering, again otherwise unchanged from upstream.

Most people shouldn't need to touch -linus, -mm, -rt, -vserver or -kerrighed. The only alternative kernel build most people would ever really be interested in is -tmb.

I'll be adding the procedure for switching kernel flavors to the release notes soon, I think.
