Hi dmiller40,
The main issue with OzOS on gOS base, it's that gOS uses an e17 not  compatible with the one OzOS uses (e17-cvs). 

So it needs to remove that e17 and consequently gos desktop meta-packages. The problem it's that apt it's not the best "kid on the block" for that purpose :(
Typically things was left beyond on your disc and starts to conflict...
Lets hope that your e17-cvs it's installed ok, and linked against the correct libs (not some phantoms ones lurking mixed in your system...)  you need to clean that old reference to enlightenment_start.
Check what package install it:
dpkg -S /usr/bin/enlightenment_start

remove any package it list with:
```
sudo aptitude purge <name_of_package>
```
```
sudo apt-get autoremove
```

If possible, open Synaptic and removed any e17 things listed, **except e17-cvs**.

Check if any /usr/bin/enlightenment_start has been removed:
```
sudo rm /usr/bin/enlightenment_start
```

Do a: 
```
sudo aptitude resinstall e17-cvs
```
(to ensure that correct enlightenment starter it's installed)

Should be working by then.

Post on any problem.

Rui
