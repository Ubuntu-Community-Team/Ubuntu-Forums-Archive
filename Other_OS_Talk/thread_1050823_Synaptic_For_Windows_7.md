### Post by fistfullofroses on 2009-02-04
Cloud computing model is a maybe. I tested both Win7 32bit and Win7 64bit. The main thing that I saw was automatic driver installation through Windows Update, and the ability to purchase and buy Microsoft products through Windows Live. While it is closer to the Linux model, it will never be as major. The main reason for things like apt-get, yum, adept, slackpkg, swaret, netpkg, pkgsrc, emerge, and synaptic in GNU/Linux is the massive amount of dependencies, which come about by massive object sharing and code sharing; this in-turn comes about due to the nature of open source software. Unless Microsoft is planning on going open source, I wouldn't expect they'd need anything like auto-dep resolving software management. It would be useless. It also goes without mention that the number of software vendors for the Windows platform is so large that no one repository would be able to handle that volume, and the customer base of over one billion users would also be a problem. Were Microsoft to implement such a feature it would need to be decentralized. Decentralization would also lead to version control issues and mirror-update issues. The over-head to run something like that would simply be to high.
