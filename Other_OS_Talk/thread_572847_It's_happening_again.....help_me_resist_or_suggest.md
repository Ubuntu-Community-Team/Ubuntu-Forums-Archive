### Post by the_darkside_986 on 2007-10-12
Have you tried FreeBSD? Not a linux distro, but it is open source and actually based on 386BSD, a version of Unix developed at a university. It has an excellent package management system that offers the options of building from source tar balls or using binary packages using a very simple command(s) (portinstall pkgname or pkg_add -r pkgname).

It took me a while to get things set up the way I want, and I'm still not done yet. I had to manually edit the source code of SDL to get sound to work in SDL games :P. Getting sound to work in general is simply a matter of grabbing the opensoundsystem binary package. It is stable and solid once it is setup. I use KDE because I like to try different desktop environments, and the fact that the version of Gnome shipped with FreeBSD is hideous and its file manager is barely usable.

Thats an incorrect opinion. You cannot call a decent distro like Arch "yesterday" while it has a great package management tool, highly up to date packages that include the latest Linux technologies, and a simple installer. Also there is no updating between "releases" (a huge mistake in most distros if you ask me), so you don't have the headache of doing a system upgrade every 6 months or so.

Arch installs itself and detects your hardware, all you need to do from there is read the wiki and install whatever you need (xorg, gnome, kde whatever).

It is not for absolute beginners but for someone who knows his way around it is a very fast and usable system while ensuring that you install only what you really need.

I used to be negative for distros like Arch (or Gentoo, Slackware) but only when you try it yourself (and invest some time in it) you can understand the benefits of using such a system.
