I'm afraid you might be further disappointed with Slackware, because it doesn't even have a proper dependency checking in a package manager.

I would instead recommend something that is closer to Ubuntu in terms of ease of use yet quite lightweight, clean (not bloated) and powerful:

[Zenwalk](http://zenwalk.org/) or [Frugalware](http://frugalware.org/)

Zenwalk is based on Slackware, but uses a netpkg package manager which is less powerful than APT used in Debian and Ubuntu, but fairly easy anyway. It also has a GUI. Either of these 3 options may be good to check out.

### Post by fwojciec on 2007-12-20
Given your expectations I'd recommend either Arch or Gentoo (or Slackware as well, but personally I'd choose between the first two).  All of these are pretty much the opposite of Ubuntu and focus on empowering the user rather than trying to make system "just work."   

I can only really speak for Arch since this is what I use: you'll get to choose what you want installed, nothing you don't want will be installed by default.  You'll be doing most of system maintenance and configuration from the CLI rather than using GUI tools.  You'll have all the essential dev tools - compilers and such - ready to go the moment you install the base system.  Arch is a binary distro but you have access to tools that make it easy for you to reconfigure/recompile any application -- and it's all done in a very elegant way as well.  If you know how Linux works (or if you're willing to learn) your system will be pretty much completely transparent to you - you'll be in full control.

I can't say if your particular hardware is going to work with these distros out of the box, but if a particular component works with Linux you will be able to make it work with them.  One of the best things about Arch, from my perspective, is that it is a rolling distro that tries to use the latest stable versions of software, so with Arch you're generally at the cutting edge of open source development (Gentoo is similar in this regard, perhaps slightly more conservative with some upgrades); using the latest versions of kernel/drivers tends to improve hardware compatibility but you might also experience regressions before they're fixed upstream.
