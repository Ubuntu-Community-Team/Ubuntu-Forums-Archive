Hello All,

Yes, There will be a release for 7.10. We are currently verify and testing the RC1 images. As you can see here:

[https://code.launchpad.net/~fluxbuntu-dev](https://code.launchpad.net/~fluxbuntu-dev)

1. Archive sync
2. Package rebuild
3. Package Testing
4. CD Image Building
5. CD Image Quality Assurance
6. Publish

Right now we are doing step number 5 mostly and going back to number 4 to include fixes to bugs we find with the images. Once this process is complete we will publish the images.

Now on to the topic of 8.04. We will be having a meeting on October 27 @18:00 EDT to discuss the next release cycle. All are welcome to attend.

I hope this post clears up any rumors, misinformation or blatant mischief that has been spread about the Project and its standing. :)

Joe Jaxx

If you like the livecd but don't care for the performance, consider installing on a USB flashdrive.  Much faster than livecd, but your hard disk's Linux/Windows partitions will never know.  (You can do a pendrive install from the livecd's menus.)  An old 512 MB pendrive will be fine.

If you have more than 128MB of RAM, boot your USB pendrive (or livecd) with the "toram" boot option.  The initial boot will be a bit slower, as the entire distro will be copied to your system's RAM.  It then runs entirely from RAM.  With a pen drive, any changes are automagically backed up to the first pendrive partition on exit from DSL.

* enabled all sources in apt... then dist-upgraded... got 3 packages...

* the thing that annoys me most is the menu... it has Applications and Apps submenus... so the apps are devided between two menues... to get to pidgin i need to go Applications > Network > Communication > Pidgin ... a bit of a long way :)

besides the menu, which is really getting on my nerves, i find that other small problems are easily overcomed by simple fixes found in ubuntuforums or ubuntuguide (feisty one is still much more helpful at this point)

so i will not lie, this distro is definetly staying where it is, that is on my lappys harddrive :D

thanks guys for a great distro!

### Post by arron on 2007-11-04
> **meborc said:**
> yeah, how did you do that? i'll trade you a tip on how to make your OOo look beautiful... small icons and integrated fluxbuntu GTK2 look :P

Sweet, i was wonderen how to make OO look better... anyhow... here we go... Borderless terminal in Fluxbox..

First you need a ~/.fluxbox/apps file.  There is a lot of info on  the web about the options for this file, but here is mine:
```

[app] (systerm)
   [Deco]       {NONE}
   [Workspace]  {N}
   [Hidden]     {yes}
   [Sticky]     {yes}
   [Layer]      {12}
   [Tab]        {no}
[end]

```

then you need to run the the terminal with the name "systerm" (or whatever you name it):
```

sleep 4 && Eterm --trans --borderless --scrollbar no --double-buffer --name systerm -x -g 130x32+10+10 --shade 55 -O -a "toggles buttonbar false" -f white &

```

The sleep is not needed, i put that in there to give fluxbox time to boot to render the background ok (not sure if thats needed for terminal, but is for conky with rox desktop).  I put that in ~/.fluxbox/startup .

To have a nice system log display, install grc ( sudo apt-get install grc ) then run something like:

```

Eterm -Ox --shade 0 -g=200x20+30+470 -q --name tailsyslog --no-cursor --scrollbar off -f yellow --buttonbar 0 -C -e sudo grc tail -f /var/log/syslog &

```

There can be some sudo issues with that (note the different name used for this), but there is ways around it. (sudoers, chmod etc).  But you can get some cool output like these pics.  Hope it works for you!
