### Post by pelle.k on 2007-07-08
I have had the pleasure to use a debian, arch and kubuntu (from server install -> kde metapackage), and i am often amazed how much people "feel" noticeable differences in speed. I'd say the difference in reality is tiny.
Also, I've been hunting for the ultimate kde distro for quite a while now. Tried pretty much all of them, and you know what; in the end i realized that unless a new, clearly superior distro, comes along, those three i mentioned are my favorites.

My criterias was;
1. stable / bug free
2. vanilla
3. large repos
4. sane defaults
5. easy to build packages for
6. good package manager
7. good forums
8. good documentation
9. easy to expand (you can have a small base to start with)

Most distros fail pretty much fail this list. Arch might not be as "stable" as debian/(k)ubuntu, but on the other hand building packages with arch is very easy. As i said, those three pretty much cover my need until something else comes along, and i have a feeling it's gonna be a long time until then. Too bad it took me forever to find out.

My advice to you, dear miggols99, is to learn about the distros you already know. This will probably make you more comfortable in the long run. Learn some bash scripting (addictive!). Try python (maybe even wxpython). Learn about stuff you encounter every day, like everything stuff from freedesktop.org
Learn how daemons work.

I won't discourage you from trying out new distros though. When you are feeling weary about it, remember my advice. :)
