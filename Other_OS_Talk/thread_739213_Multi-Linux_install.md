You may have had more than one package manager (ie Synaptic) but it's very unlikely that you've used more than one package format (ie RPM, deb, tgz, etc.).  And many distros handle DE's better than others.  For example, Zenwalk is a much lighter execution of Xfce than Xubuntu.  And distros like Arch are designed to land the user at a working command prompt with network configuration tools and a nice package manager.  The user then installs all the gui apps that they want, rather than those installed by the devs of a more pre-configured distro like Ubuntu.

### Post by cardinals_fan on 2008-03-29
> **Drmgiver said:**
> Well, with the right commands, to my knowledge and correct me if I am wrong, you can convert package formats to the needed .deb for Ubuntu right?
As already noted, this doesn't work perfectly.  And it is also worth considering that these distros evolved very differently.  Some, such as Arch and Gentoo, attract advanced users who want a blazing fast and customizable system.  Others, like Ubuntu and openSUSE, aim to make Linux painless to set up.  These distros tend to include many apps on the install cd.  Good if you want to set up your desktop fast, bad if you want to choose your own apps.  Still others, like Zenwalk, take a middle road.
