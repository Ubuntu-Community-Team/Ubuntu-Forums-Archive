### Post by mozetti on 2007-05-30
IMO, the main reason this occurred is because MS has always taken the stance that, in order to improve security we need to proect the customer from themselves. And, in fact, that can be an effective measure. The problem is, that only works when you also protect the customer from others. I think MS has the priorities out of balance. Protecting users from themselves is only effective if you also protect users from outside interference first. If you don't do that, you end up where XP/Vista is -- still vulnerable and pissing off people that are knowledgable enough to know when their hands are being tied by restricting what they can do with the software.

### Post by 3rdalbum on 2007-05-30
Microsoft uses the Windows Attitude To Security, which is: "Limit the threats once they start running on the computer".

Linux distributions use the Linux Attitude To Security: "Stop the threats from getting into the computer in the first place".

So it's no wonder that Vista is not more secure than XP - the attitude to security is almost completely wrong. What's completely wrong is this:

Mac OS X Attitude To Security: "Viruses don't run on our system. We're too small to be a target. Bits of FreeBSD in the kernel will protect us. Hey, look, someone claims to have written a virus for OS X - they're obviously lying, because OS X is invulnerable, so I'll download this program and run it."
