### Post by yoshida on 2007-04-26
For a long time, the root acount in Linux has been seperated from the user account, as a safety measure. Most distros merely seperate it by password, so only one person in the world can access it on that PC, and in terminal mode the sudo prefix will do with the password stored for 15 minutes.

Windows 98/2000/XP users have been administrators more often than not. There's not much difference in functionality between regular Windows users and administrator accounts, which is ofcourse very unsafe. Microsoft decided to put into Vista what XP SP2 has been doing with Internet Explorer: safety notifications. Instead of using a password all one needs to do is click a button, effectively (attempted) generating protection from scripts and such (but not from people who actually have access to the terminal).


I'd like your opinion on this issue. Is Vista 'stealing' Linux philosophies without taking away functionality and responsibility from regular users? Or is it just a somewhat failed attempt to 'clone' one of the biggest advantages from certain distros? Also, which would you prefer if you'd approach it open minded? I don't need 'Windows is Evil' geeks.
