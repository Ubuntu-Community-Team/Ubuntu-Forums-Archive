### Post by Sorivenul on 2009-01-29
For servers, I have used:
[LIST]
[*]CentOS
[*]Debian
[*]Ubuntu Server Edition
[*]Slackware
[/LIST]

And I currently run my server on FreeBSD.

The definition of "user friendly" on these Forums is disputed. 

If you are new to Linux, I suggest setting up Ubuntu Server Edition, simply for the support offered on these Forums - it is second to none, IMO. You won't have a GUI (Graphical User Interface) to start with, but as long as you are connected to a network, a few commands should get you one should you want it.

If you have some experience, pick whatever you wish, but with a server, stability is key. 

PS: That machine should *fly* as a server.

### Post by Dragonbite on 2009-01-29
Can also try openSUSE.  One advantage with that is Yast.

Yast handles most settings (hardware, network, software, etc.) and is fully functional and available whether you use KDE, Gnome, FVWM or CLI.

I had at one point a headless openSUSE server set up which I would connect to via SSH and run Yast to configure things.  This alone made it very handy, probably one of the easiest to work with though you don't learn as much as rolling up your sleeves and modifying some config files.

Oh, and just for the record, I am very new to networking and am learning the (very) hard way!
