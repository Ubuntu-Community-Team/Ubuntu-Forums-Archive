### Post by handy on 2008-05-26
I must say that I was incredibly surprised to see just how much faster Arch is than any other Linux distro' or operating system I have ever used.  (Not including the various Windows systems that I once was very familiar with), I must have tried at least 20 of them including PC-BSD, Solaris & Zeta.

I'm using a standard (1Gb RAM) aluminium 24" iMac & a home built job with a 64bit AMD 3500+ single core, with 2Gb RAM & an AGP XFX-7950GT /512.  Both of these machines have never been as sprightly as they are with Arch on them; & that was with Gnome as the desktop!

When I changed to Openbox the next step in speed increase has just blown me away. The inherent simplicity of Openbox (compared to the complexity & size of Gnome), which sits on top of Arch, (which must be one of the simplest Linux systems available), allows the user to more easily understand how their system works.  

Once you have learned how to set up Arch & Openbox, which due to all of the brilliant documentation available & the two magnificent forums - Arch & Ubuntu, it took a novice like myself 3 or 4 days to have Openbox on Arch set up how I like it.  Then due to the brilliant Arch rolling release update system, there is no need for me to have to do all of this setting up again because a new version of Arch has been released; all I have to do to update, sync, and upgrade my entire system is type: *sudo pacman -Syu* at the prompt whenever I feel like bringing everything up to cutting edge.  The more often I do it the less time the process will take.

The other great thing imho about the Arch way, is that you have to set everything up yourself, so you learn about Linux, you get to configure your system how you want it to be, you gain knowledge that allows you to be able to modify your system to suit yourself.  Once you have your system the way you want it you can make copies of all the configuration files that you have created/modified & if a hardware failure or whatever happens that causes you to have to set up your system again, you will be able to do it so much quicker having these files on hand.

It makes my iMac's native Leopard seem like it is trying to run in sand filled gumboots. Not just because of how much slower Leopard is but also with regard to the limitations regarding the upgrading of software in Leopard; Apple only looks after the Apple branded software, so it makes the Leopard update seem comparatively very unsophisticated when compared to software/system updates/upgrades made the Arch way.  

I just hit *sudo pacman -Syu* & am doing (in the background) a complete OS & software upgrade right now as I type this:  First my package database is synchronized for all of my enabled repo's (which is all of them).  Then it identifies which packages have upgrades available & lists them informing me of the download size which was 96.06Mb in this case, & the total installed size being 392.38. After confirming that I want to continue it then downloads & installs the upgrade.  At the end I will have the latest most current version of all of the software installed on my system.  No need to go through all of the time consuming steps that we normally do to upgrade our OS, reinstall much of the software & possibly drivers, blah, blah, blah... My full system synch' & upgrade took approx' 10 minutes!

It also makes Leopard feel like it has been designed to be an OS that is not to be used in any more than a very superficial way.  Which is why the majority of Mac owners like Macs I think.  They like the look of Leopard, but the OS really only exists to look good & enable them access to the applications & peripherals that they use.  I have no problems with how anyone else likes to use their computer, my wife uses her Macs just like that & she is generally a very happy Mac user.

For any alu' iMac owners that do want to give Arch a try, I have made a guide that is now on the Arch wiki that is fairly comprehensive, it doesn't cover wireless as I don't use it, & I haven't bothered to set up a printer yet, when I do I'll add using CUPS to the wiki:

***_[http://wiki.archlinux.org/index.php/IMac_Aluminium](http://wiki.archlinux.org/index.php/IMac_Aluminium)_***
