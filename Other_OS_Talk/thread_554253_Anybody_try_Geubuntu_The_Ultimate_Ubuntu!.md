Seems like this point is kinda confusing to a lot of people. I'll explain.
E17 is a Window Manager, not a Desktop Environment. So there's no reason why E17 should work on its own. E17 is not fully capable of giving users every function they need to run a desktop TODAY.

For example, how could you live without a system tray? E17 hasn't got one. Plus, in E17 alone you cannot configure GTK themes for every app running GTK. Elive is wonderfull and usable and it runs a very old and heavily patched version of E17 to achieve its level. In Elive there are also a lot of tools that in E17 does not exists. That's why Elive is "Kinda" usable, but not totally by a newbye aniway. After all, for example, there's no easy way to set GTK themes or have a system tray and there are no desktop icons.

That's where Geubuntu comes. Gebuntu considers E17 as a Window Manager and that's what it is. Surely there are a lot of ETK apps around too, you can find many of them in Geubuntu infact. But Geubuntu is an experiment trying to integrate Gnome with E17. And there is absolute need of the Gnome Panel at the actual stage of Enlightenment DR17, in our opinion. Gnome panel gives you a system tray and a series of tools (Tipical of Ubuntu) to manage GTK themes and every other aspect of your computer! You cannot find those tools in E17 because E17 is NOT A DESKTOP ENVIRONMENT.

That's the simple point :)
So it is not odd to see a gnome panel in a E17 installation. Sure you could argue that Geubuntu could have used a third party sstem tray and others solutions but Why? Why should it have done it? Geubuntu has all the basic caracteristics of Ubuntu, such as the restricted manager, gtk theme editor, nautilus (E17 hasn't got an usable file browser for example), ecc. If you whant a list I can provide one :D

As time passes, if new E17 apps will appear beying able to substitute the corresponfing Gnome apps integrated in Geubuntu, then Geubuntu will use and prefer new E17 apps over those from Gnome but today E17 is not complete and that's all :)

So Geubuntu gives users a fully customized and working E17 desktop, integrating the E17 WM with Gnome. So the goal is different from Elive and Elbuntu, both trying to run E17 on its own.

Also, Geubuntu is trying to build a community around the project to create native E17 apps to complete the losses in E17. The goal is to create a fully functional merge of E17 WM with Gnome. Redundant would be to create an infinite number of E17 apps able to do things a normal DE does and not a WM. Since E17 is intended as a WM and not a DE (And a set f libraries, etc. Elinghtenment is really a very big and complicated project) :D !

Hope this gives you some explanation about the Geubuntu project. If you don't think that E17 should be integrated with Gnome and don't whant to consider the idea even knowing that E17 is NOT a DE, than Geubuntu is just not your kind ;)

But if you whant a newbye oriented easy to use and complete E17 based distro and desktop, then Geubuntu is for you ;)
