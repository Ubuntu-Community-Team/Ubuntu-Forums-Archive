To elaborate on this, the very first part (the part before the colon) of a URL address is the protocol that should be used to access the file. This is typically "http:" which stands for HyperText Transfer Protocol; but it can alternately specify other protocols such as "https:" (HTTP with secure encryption), "telnet:" (remote login), "rtsp:" (time retrieval), and "ftp" (file transfer).

It is left to the webbrowser (Firefox, Internet Explorer, Safari, etc) to provide support for the protocol and perform the appropriate operations when the link is clicked on. In the  the case of URLs which start with "ftp:", browsers are designed to download the file when it is used as an address (or list the contents if it is a directory). 

Clicking on a link which points to a Slackware ISO such as:[ftp://slackware.mirrors.tds.net/pub/slackware/slackware-12.1-iso/slackware-12.1-install-d1.iso](ftp://slackware.mirrors.tds.net/pub/slackware/slackware-12.1-iso/slackware-12.1-install-d1.iso) should popup a dialog to download the file. It is possible that your browser doesn't support the "ftp:" protocol (though almost all of them do). If you can't get the FTP download to work, there are also some sites that support HTTP downloads listed [on this webpage](http://www.slackware.com/getslack/list.php?country=USA).

### Post by Whiffle on 2008-09-08
Go to [www.slackware.com/getslack](www.slackware.com/getslack), pick your country.  Then, it'll bring up a list of servers.  Pick a server.  Now, it'll give you a list of every slackware version.  The ones with -iso at the end are images, like Ubuntu gives.  The ones without are the individual files.  Find Slackware-12.1-iso, click on it.  It'll give you another list.  Download slackware-12.1-install-d1.iso and slackware-12.1-install-d2.iso, and burn them to CD.  I don't think you need the third disk, but you can get it if you want.  There is also a DVD if you want to do that.
