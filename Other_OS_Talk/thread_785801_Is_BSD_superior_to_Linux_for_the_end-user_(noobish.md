### Post by Phenax on 2008-05-07
BSD flavors are just a different kernel with the same tools Linux users know and love plopped on top - Gnome, KDE, X, Easytag, Firefox, etc. Usually BSD uses a BSD tool-chain instead of a GNU tool-chain for compilers, linkers, tar and other programs, although they are very similar and it is possible to use GNU toolset with BSD too.

The end user probably wouldn't even notice much of a difference between BSD and Linux as for the most part it's just a different kernel.

### Post by izanbardprince on 2008-05-08
> **LaRoza said:**
> Linux is used more (on the desktop at least) so it probably has the benefit of a larger user base. On servers, I think BSD flavours might be a tad better, but the differences between BSD and Linux are nowhere near the differences between Linux/BSD and Windows.

Linux and BSD are the same as far as they're both unix-like, they both have comparable features and performance, and they're both free and open source software.

The reason BSD can sometimes be slightly more stable is because the kernel and the base system are all developed by the same people, with Linux your kernel is Linux and the rest of the core system is from GNU.

With BSD, it's the BSD kernel/core, and then of course your add-ons like X, a window manager, etc.

Until PC-BSD, the concept of different package managers on BSD systems just wasn't there, which was good in that Ports was slow and difficult to use, but consistent.

The BSD license and the GPL are both Free Software licenses, but BSD is favored by companies like Apple and Microsoft because they can link proprietary libraries to it, and so on.

On the desktop, Linux is usually the better choice, because it has more software. BSD does have a Linux compatibility layer, but it's not completely compatible with newer Linux binaries, and if you want Flash 9 on BSD, you hafve to run Firefox for Windows on Wine.

As a user of all the *BSD's, and many Linux distros, (jack of all-trades, master of none! :), I have a philosophical outlook about this - meant for contemplation, not being argumentative.. 

When the CSRG ripped out all the ATT code from BSD in 1994, could not then BSD be considered a "clone" of *nix? as well?  Certainly up until that time, a lot of BSD code intermingled with the ATT code, and even influenced it, but once the ATT code was removed, wouldn't the only thing left being that of following the initial rules:

Write programs that do one thing well.

Use text as the universal interface (as in not having a "registry" for control - gui's under this control ok)

And following the other traditions of the mind-blowing work in the 1974-era "Software Tools" philosophy?  (Imagine the *nix philosophy written in ratfor!)

I guess the point I'm getting at is that a clone is as good as or even better than the original, parental version, if it follows the initial guiding principles.  A similar case is made for the bug-for-bug compatible clone of Bill Joy's vi editor done by Keith Bostic's NVI (new vi), which is what you get by default in the *BSD's.

Thus aside from licensing issues, or specific code, both bsd (post-1994) and linux could be considered clones and are very succesful - but merely following the philosophical guidelines doesn't guarantee adoption.  I don't see any Plan-9 distros out there these days. :)

In the end this isn't meant to be a "who's better" response, but just something I think about.  I learned the basics of *nix on BSD and happily use both "clones." with Linux as well. :)

And don't get me rambling thinking about how the BSD kernels are still being compiled by GCC, although I hear that the bsd-specific PCC compiler is making progress..  I dig ALL of *nix!

### Post by Luke has no name on 2008-05-22
I *really* started my Non-Windows experience by picking up "FreeBSD 6 Unleashed". I read about the advantages of FreeBSD over Linux, and it was pretty interesting. There are pros and cons to both systems.


---FreeBSD is a complete OS, not just a kernel.

---FBSD is centrally managed. Any changes to any code in the OS has to be approved by "maintainers" of the OS.

---It's more like Debian than Ubuntu in that it's very stable, and moves slower to achieve that reputation. FreeBSD's kernel is compiled to run well on low-end hardware, so if you're running a high-traffic quad-core server, you'll want to recompile the kernel.

---BSD is licensed under the BSD license (shocker) so it can be more easily used and modified by private companies (OS X is based on FreeBSD). This right here is where a disproportionate amount of disagreement exists between the two communities.

---Their package system is pretty interesting. As someone said, Gentoo is a hybrid of BSD (package management, compile when possible philosophy) and Linux (Licensing, software).

---The Linux community is larger and thus more helpful. In FreeBSD, forums are hard to come by; you usually have to get on mailing lists. OpenBSD will very bluntly shut you down if you ask noob questions. their intentions are clearly not to help new users. That being said, it's a very secure OS and you should try it out if that's what you're into. 


I'll say right now that I'm a bigger fan of Ubuntu Server now, more than anything. Apt-get/Aptitude is great, the instant server setup is fantastic. However, FreeBSD does allow you to choose EXACTLY what packages get installed when you are setting up your system. Ubuntu doesn't do that. 

If you want pure flexibility and power, go FreeBSD (You'll learn more about CLI learning BSD, I can almost guarantee it. It's not a plug-n-play environment...). If you want ease-of-use, intuitive setup, go Ubuntu.

Something to know: BSD can run practically all Linux apps. It's claimed by BSD that they can even run some Linux apps faster than Linux! if it's not in the ports tree, you can download and compile the source.

Books on FreeBSD: **FreeBSD 6 Unleashed **was really helpful in getting me acquainted with the system. I didn't completely ustilize the last 1/4 of the book, but I did read through a lot of it, and I got to where I could config and setup a BSD server with no hassles, no looking up help.

**Building a Server with FreeBSD 7** gets good reviews.

***---FreeBSD is a complete OS, not just a kernel.***

Any distro is a complete OS. 

***---FBSD is centrally managed. Any changes to any code in the OS has to be approved by "maintainers" of the OS.***

Any distro has a core group of maintainers who approve software for inclusion. It might be argued that derived distros, such as Ubuntu, don't maintain the entire codebase, or that some distros are lax in there control; but if a comparison is made between FBSD and Slackware, or FBSD and Debian, there is no real advantage WRT to code maintenance.

***---It's more like Debian than Ubuntu in that it's very stable, and moves slower to achieve that reputation. FreeBSD's kernel is compiled to run well on low-end hardware, so if you're running a high-traffic quad-core server, you'll want to recompile the kernel.***

I agree with your point on stability. The distinction about kernels should, again, be more of a distro-specific comparison. Each distro chooses what kernel configuration they wish to support and a re-compile is sufficient to target more accurately the end machine. Most Linux distros are compiled to support low-end processors (486 or even 386), but are optimized for more modern hardware (they run slower on older H/W than if compiled targeting that older H/W specifically). Some distros, such as Zenwalk, do indeed target only newer hardware. 

***---BSD is licensed under the BSD license (shocker) so it can be more easily used and modified by private companies (OS X is based on FreeBSD). This right here is where a disproportionate amount of disagreement exists between the two communities.***

From a practical standpoint, I don't think the disagreements are that significant. The GNU project recognizes BSD as Free Software and will accept BSD-licensed code, as well as any other Free Software. Though BSD won't accept (to my knowledge) GPLed code, I get the impression that most BSD users are willing to run it (and isn't it in the Ports system?).

There are indeed argumentative folk on both sides; and both seem ready to engage in polemic debates and banty about labels of hypocrisy. It all boils down to a developer releasing his code under whatever licensing he wishes. If a BSD developer wishes to permit private companies to relicense his code under a proprietary license then is not it hypocritical to complain if a GNU developer relicenses it under GPL. Of course, the GNU developer's behavior is considered to be hypocritical, but then considering it such is also hypocritical... OK, I have a headache now and will stop. 

***---Their package system is pretty interesting. As someone said, Gentoo is a hybrid of BSD (package management, compile when possible philosophy) and Linux (Licensing, software).***

I think the BSD Ports system (is that what you're referring to?) is indeed a cool approach to package management. I don't see a big disparity between particular Linux package tools and Ports; and some are quite similar -- not just Gentoo, but Crux, SlackBuilds, and even apt-get can build from source. 

***---The Linux community is larger and thus more helpful. In FreeBSD, forums are hard to come by; ...***

I think that the comparison should be such as though FreeBSD were a particular "distro". I imagine the FreeBSD community is larger than some particular GNU/Linux distros; smaller than others. Followers of these smaller distros experience the same disadvantages, and advantages, as a BSD user. The entire gamut of GNU/Linux distros may be larger, but areas which differ between BSD and GNU/Linux are not significantly greater than those between individual GNU/Linux distros.
