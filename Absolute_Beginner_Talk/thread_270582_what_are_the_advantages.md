### Post by Iowan on 2006-10-03
I don't mean to be the "stick in the mud", but every rose has it's thorns.  Linux (including Ubuntu) is "different" from the Windows you're used to, but Windows 3.1 was different from Windows 95, which was different from NT or XP... 
It's still a Windows world - some hardware manufacturers don't (yet) support Linux, and equivalent software packages don't necessarily exist (not discounting Wine).  It may take some tweaking to get some hardware to work with Ubuntu. 
Unix-type "malware")*has* gone around.  I suspect that there's still a fair amount of "security by obscurity" - when Linux eventually rules the marketplace, it'll be more worthwhile to virus programmers.  I'd rather not claim "There's never been - and never WILL be a Linux virus".
That all said, the support you get from this *free* OS is unbelievable.  They can't necessarily solve a proprietary driver problem, but may be able to point you to the solution (if one exists).
Again, not to be the "wet blanket" (I've evolved from a muddy stick), but be aware that Ubuntu and Linux are not guaranteed problem free, do-anything-without-even-asking solutions to the world's computing problems. I like it, you probably will, but don't believe it's perfect - yet!

### Post by sbergman27 on 2006-10-03
> **Iowan said:**
> I suspect that there's still a fair amount of "security by obscurity" - when Linux eventually rules the marketplace, it'll be more worthwhile to virus programmers.  I'd rather not claim "There's never been - and never WILL be a Linux virus".

Agreed, in principle.  However, it is important to stress that there is a vast gulf between the possiblility that POSIX-like OSes might someday have a noticeable amount of client-side malware, and the rampant problems that Windows has had for years, and in fact, all Microsoft OSes going all the way back to when we were all using DOS and we were dialing in to local BBSs because home internet connections were unheard of.

Plus, it's not like we have no reasonable points of comparison.

Apache web server, from a Unix background and usually run on Unix like OSes has been running between 60% and 70% of sites for years, compared to Microsoft IIS's 20% to 30%.  Apache is the logical target for attackers, based upon the number available targets.

[http://news.netcraft.com/archives/web_server_survey.html](http://news.netcraft.com/archives/web_server_survey.html)

And yet the security exploits are comparable in number and severity.

[http://secunia.com/product/1438/?task=statistics](http://secunia.com/product/1438/?task=statistics)
[http://secunia.com/product/9633/?task=statistics](http://secunia.com/product/9633/?task=statistics)

On the client side, Windows security philosphy and practices evolved out of single user, non-network-connected DOS, and a design goal of user convenience over security.

Linux/Unix is coming from the opposite direction, having been paranoid about security for decades, at the expense of user convenience, and now learning how to combine the two.

We do not, in general, have a legacy of security through obscurity to overcome, as Microsoft does.  However, the possibility of poor security practices being introduced by software makers out of a goal of user convenience, is a valid concern.  I have not seen too much of that occurring, though.

Not to argue, but it is important to point out that the concept "No OS Is 100% Secure", which is valid, covers a *very* broad range of severities.
