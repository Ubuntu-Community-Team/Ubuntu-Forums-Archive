### Post by cotcot on 2008-03-31
Both are possible. Install from scratch is safest.
I normally install the newer version in multiboot with a new /home. It is not a big job when you make a text file with what you have done to install. So I only need to open my text file and copy paste the commands. Then I upgrade the one i had before. When the upgrade is OK I keep it as default; when it is not ok I move my essential files to the new installed version.
This leaves with a default OS and an OS that I use to do test.

### Post by abhiroopb on 2008-03-31
There's a HUGE debate over this. Here is my honest opinion:

1. If you have a lot of things that you have tweaked changed over time, it may be less hassle to simply do an upgrade (I upgraded from fiesty to gutsy without any problems to date)

2. Backup your data so that IF there is a problem you can always do a clean install.

Basically, upgrading is less hassle as you can set it to start and it will just get going, and your settingss remain (e.g. wifi card, and video card drivers). On the other hand a clean install often sorts out any problems you may have been having and it is usally better for any OS.
