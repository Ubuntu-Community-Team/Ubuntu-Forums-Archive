### Post by jasmuz on 2006-01-15
Live CD versus Full Install?

Well Live CD's are mainly for testing your hardware to see if everything works, as it runs off RAM it will certainly slow depending on your ammount available. You wont be able to modify certain configurations, nor install and uninstall software.

My recommendation is to slap another Hard Drive into your machine and install Ubuntu GNU/Linux to it, doing so..you will able to dual boot between it and the other operating system.

### Post by jimrz on 2006-01-15
Hi there and welcome. It is true that the "live cd" will run without touching your current os. Make sure you burn the downloaded iso "as an image", then that your computer is set to boot from cd 1st, put the burned cd in your drive and reboot should take you into ubuntu live. "live cd" does not touch your os because everything runs from RAM, and so it does not run as fast from the cd as would from a full install. Try it out...have fun...come back here to ask questions. You can do quite a bit with the "live cd" in addition to just getting the feel of the os, also, you can search these forums + the wiki for basic answers and the folks (lots of them) on these forums can and will gladly provide plenty of good advice/instruction/help. Enjoy

### Post by Qrk on 2006-01-15
There aren't really any limitations in the Live environment, its the same as the install CD. There are things that you can't do, such as

Save your configuration (use a pendrive for data, but it won't save the filesystem)
Install much software (small things work OK)

Its also much slower than the harddrive version. 

Ubuntu's Live CD is great for checking to make sure that the installed version will run on your hardware. But if you want a live environment to do work in there are better alternatives.
