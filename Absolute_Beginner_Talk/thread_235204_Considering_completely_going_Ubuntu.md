### Post by kebes on 2006-08-12
> **mengfei said:**
> I'm considering completely getting rid of windows and going Ubuntu.

Very cool idea.

> After all, from what I understand, you can run most Windows programs under Wine?

Well "most" is an over-statement. Many work, some do not. I think it would be a good idea for you to write down a list of the types of programs you'll need to run while at college. For most of them, there will be a Linux equivalent. If there are any windows-only ones, check the Wine HQ database and see if they tend to work under Wine.

I know some college programs require you to use specific software (usually Windows-only) so maybe find out about those first, to make sure that you'll have what you need.

Like I said, for most software types there is a Linux equivalent for anything you need to do. But some schools/professors can be fussy about using the *exact* piece of software they specify.

> Would you guys recommend this?

Linux is really amazing. It's powerful, stable, and fun (if you like computers!). However as I already said sometimes the whole "compatibility" issue does come into play.

> Right now I'm running with my OS on a 20gb hard drive, and a 300gb hard drive for my Digital Camcorder. At college I won't have the family Digital camcorder, so I was also considering just using the 300gb hard drive. Good idea?

Sure, why not. A full Ubuntu instal only requires ~2Gb... so you're obviously not lacking for hard drive space. Even the 20 Gb drive would be enough. Of course if you're planning on accumulating lots of media files, then the bigger drive is better. Typically it's a good idea to keep the operating system and the data separate, so why not keep both hard drives?

> A final question is, as I have an Athlon 64 processor, is it worth going with the 64-bit version of Ubuntu? I only have 512mb of RAM, and I heard that under 64-bit, it fares better or something like that? But then I also heard there are compatibility issues.

I've never used the 64-bit version, but I've heard that it runs well. CanonD said the 64-bit version didn't work for him, which is too bad. However a Ubuntu instal is pretty fast, so it's no big deal to try out one version and then reinstall the other. I would start with the 64-bit version and see what happens.

### Post by umonkey on 2006-08-12
I would recomend that you dual boot and run Ubuntu for a while to make sure that it does everything that you want/need it to do before you decide to totally abandon Windows. If you have a 300GB hard drive you will have plenty of space. 

Even if it does everything you want right now, I would still keep a Windows boot partition on your machine, even if it is just a small one. You might want to fire up a video game to play online with some friends you met at college. Maybe a college class will require the use of a program that doesn't work well under Wine. Or you might pick up a gadget like a cell phone or mp3 player that doesn't have good support under Linux. What if your motherboard/video card/[fill-in-the-blank] needs a firmware upgrade and the manufacturer only provides a Windows version of the update program?

It doesn't hurt to be safe. I believe that most Linux users have to boot up Windows at least occasionally for one reason or another. 

Of course if you are really hard core you can completely nuke Windows off your machine. You just have to be willing to put up with the limitations of using Linux in Windows dominated world. Would you mind using the computer lab to do your class work if a class requires a program that doesn't work well under Wine? If you have friends who are all playing a cool new video game online do you mind not being able to play with them? Are you willing to support your machine yourself if the college IT department tells you that they don't support Linux? 

For example, at our college you must connect to their VPN if you want access to the Internet over their wireless access points. They had instructions for setting this up under Windows, but I had to figure out how to make it work under Linux myself.

Another personal example: I am also going to be taking a class on .Net/C# this semester and I want to be able to use my laptop instead of the computer lab. (It really sucks to be kicked out of the computer lab at midnight when your program is due tomorrow and you only have it 90% finished.) If I am debugging a homework assignment/project I don't want to have to worry about the possibility that my program is not working properly because of a problem with Wine. I have a full load of classes so I don't want to waste my time on problems like that. So I will definitely be running Visual Studio and my programming assignments under Windows.

Anyway, those are my thoughts. Good luck.
