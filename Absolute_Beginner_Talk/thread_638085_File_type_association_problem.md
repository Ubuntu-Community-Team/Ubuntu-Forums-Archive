### Post by mnb on 2007-12-11
Hi all,  this one has me stumped.  I installed a mind-mapping application called kdissert (really neat program), and it uses a file type of ".kdi".   When I double click on a .kdi file the Archive Manager program runs and says "File type not supported".

Thinking this would be an easy fix, I right-clicked on a .kdi file and went to properties and the "Open With" dialog.  BUT.... there it thinks the file is of type "gzip archive" and the Archive Manager is set as the default open application - which explains why the File Manager is coming up in the first place...

I don't want to change the default behavior for my gzip files.... why does Ubuntu think this file type ".kdi" is a gzip archive file and how do I associate ".kdi" files with my kdissert application instead?  Any thoughts?

Thanks for your help,

Michael

### Post by drs305 on 2007-12-11
My advice is to change the association. I've not seen a compressed file with a .kdi extension and a google search doesn't readily show kdi extensions. If you do come across a compressed file, it would be easy to change the association back. Chances are you are going to need your mind-mapping app with a kdi extension than anything else. It won't change what happens to gzip files, which don't have that extension.
