### Post by poofyhairguy on 2005-05-19
[QUOTE=nevets68]

[/QUOTE]

I will answer here even if the original poster doesn't need it, because other might want to know.

I install Ubuntu on a bunch of computers, and this happens to me often. X server messing up on the install. You know what this tells me everytime? it tells me that my install CD is messed up. 

I never know why. Unlike a music CD, a very small: scratch, error while burning (usually because of high speeds), gunk, bad blank to begin with , etc will make it so that Ubuntu won't install. In every case it manifests itself in the xserver not working.

Today this very thing happened to me, and like the other 20 times my response was to burn another ubuntu install CD. Sometimes CDs can go bad right after they work on another computer, sometimes a install CD I burn doesn't work the first time, and sometimes I have install CDs that will install on a dozen computers without any problems (I treasure those). In every case, the solution is to burn a fresh install CD. In EVERY SINGLE CASE (eventually after enough were burned) the new CD worked. I have never met a computer that the xserver won't work right on if the install CD is a good one. Funny thing is, I think I once had one of the CDs that were shipped to me do this too.

So, if you have this problem after the install than burn another install CD and do it again. It kinda sucks that I have gone through so many blank CDs, but its the tradeoff for a free operating system.
