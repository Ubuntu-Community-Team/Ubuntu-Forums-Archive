### Post by Cato2 on 2007-10-28
Try adding the following line to /etc/fstab, given that /dev/sda1 is your NTFS partition:

/dev/sda1    /win             ntfs-3g    noatime,errors=remount-ro    0    1

(To be extra sure, type 'fdisk -ul /dev/sda' which shows the partitions, sizes and types - always a good idea before any disk operations, although here we are simply mounting a disk which is quite safe.)

Then type this to create the mount point and mount the filesystem:

    sudo mkdir /win
    sudo mount /win

Then do the following to confirm all's well
    
   df -Th

You should now have read/write access to your NTFS partition, and it should be mounted on boot.

NTFS-3G is the filesystem type to use, as it permits true read/write and is very well tested - normally Ubuntu 7.10 should enable this for your NTFS partitions on installation.

I was able to get it to work.

You can see my drives above.  After having that info, I did the following...

Added a mount point:
```

sudo mkdir /media/data

```

Backed up my fstab file:
```

sudo cp /etc/fstab /etc/fstab_backup

```

Edited my fstab file:
```

sudo gedit /etc/fstab

```

It showed this:
```

# /etc/fstab: static file system information.
#
# <file system> <mount point>   <type>  <options>       <dump>  <pass>
proc            /proc           proc    defaults        0       0
# /dev/sda1
UUID=de6008a8-1608-4dab-aa1c-83a31ee42d63 /               ext3    defaults,errors=remount-ro 0       1
# /dev/sda5
UUID=bd2d95ba-3379-495c-bb83-16a4c09a9d8b none            swap    sw              0       0
/dev/hda        /media/cdrom0   udf,iso9660 user,noauto,exec 0       0
/dev/hdb        /media/cdrom1   udf,iso9660 user,noauto,exec 0       0

```

I added a section under **# /dev/sda1** so it would mount **/dev/sdb1** which made the file show this:
```

# /etc/fstab: static file system information.
#
# <file system> <mount point>   <type>  <options>       <dump>  <pass>
proc            /proc           proc    defaults        0       0
# /dev/sda1
UUID=de6008a8-1608-4dab-aa1c-83a31ee42d63 /               ext3    defaults,errors=remount-ro 0       1
# /dev/sdb1
/dev/sdb1	/media/data	ext3	defaults	0	0
# /dev/sda5
UUID=bd2d95ba-3379-495c-bb83-16a4c09a9d8b none            swap    sw              0       0
/dev/hda        /media/cdrom0   udf,iso9660 user,noauto,exec 0       0
/dev/hdb        /media/cdrom1   udf,iso9660 user,noauto,exec 0       0

```

I wasn't 100% sure but I placed tabs between each entries so those aren't just plain old spaces you see there; they are tabs.  It seemed to do the job.

Then I made sure Ubuntu could do what it was told:
```

sudo mount -a

```

Then I made sure I could use the drive:
```

sudo chown -R rob:rob /media/data
sudo chmod -R 777 /media/data

```

I logged off/rebooted a few times to test and it worked fine.  Thanks for the help and I hope my step-by-step can help others.
