### Post by vayde on 2007-07-09
There are many great explanations on this page about what makes up an OS.  One thing that's confusing coming to Linux (there are plenty) is that each piece of the OS is modular.  If you're coming from Windows, you're used to having every component is compiled into one big block, and thinking of that whole as the OS.

Linux does things differently.  It's all made up of modular pieces that can be put together in many different ways.  This allows for greater customization, and also greater reliability, as if one part of your OS develops problems, it's just that piece that doesn't work.  You still have access to the other parts so you can fix the problem.

Most of the differences between Distributions or flavors of Linux are differences in the specific modular components that go into making a complete OS.

GNOME and KDE are user interfaces to let you control the OS and runs software.

Ubuntu, Red Hat, etc. are "distributions" ... the Linux OS, some interfaces, and a bunch of software that is all bundled up to make it easy to install.
