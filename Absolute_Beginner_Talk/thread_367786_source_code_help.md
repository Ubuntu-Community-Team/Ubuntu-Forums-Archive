### Post by louis_nichols on 2007-02-22
Commands are not source code. Although one can create files, called *scripts* with series of commands to accomplish a certain task, and in that case the commands might be considered source code. A [definition of source code]("http://en.wikipedia.org/wiki/Source_code") is, of course, found on Wikipedia. Hope it's simple enough for you to understand.

Now, commands are many. There are some basic ones and besides those there is a lot of software that does not have Graphical User Interfaces, but can be manipulated by commands typed in terminal. There are many lists of commands out there. A [search on google]("http://www.google.ro/search?hl=en&client=firefox-a&rls=com.ubuntu%3Aen-US%3Aofficial&hs=9ff&q=+linux+terminal+commands&btnG=Search") shows some. There are also many books, and I think a book is the best way to start. I started out with *Linux for dummies*, which I recommend, but I'm sure there are many other very good ones, too.

Anyways, this is a long story. I suggest you read about it to learn more. I've been using Linux for almost 3 years now, and I still feel like a beginner many times. I'm pretty sure no one can learn everything about the command line in a lifetime. :)

EDIT: I should not forget. Ubuntu has a very brief description of the most basic commands in its documentation. That can be browsed [here]("http://https://help.ubuntu.com/6.10/ubuntu/desktopguide/C/terminals.html")
