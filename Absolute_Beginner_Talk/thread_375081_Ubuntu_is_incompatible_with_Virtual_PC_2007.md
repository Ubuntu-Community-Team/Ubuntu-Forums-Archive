Maybe you should contact Microsoft Tech Support about Virtual PC not supporting xorg's autodetection.  File a bug report, take a number, and a service representative may contact you within 1 to 2 years.

In the meantime, to get xorg working under Virtual PC 2007:

Get to a virtual console (Alt + Ctrl + any F key between 1 and 6), then type:

```
sudo nano /etc/X11/xorg.conf
```
type in your password when / if it asks.

In xorg.conf, find the Section "Screen" near the bottom.  Change the DefaultDepth to 16 (or 15, 8, etc.), and you may want to set the resolution to something smaller like 800x600 or 640x480, since you're running it in a window.

```

DefaultDepth 16
SubSection "Display"
   Depth 16
   Modes "800x600" "640x480"
EndSubSection

```

Then, restart gnome

```
sudo /etc/init.d/gdm restart
```
again, type in your password if necessary

That should get you up and running.

### Post by 3rdalbum on 2007-03-08
> **dudbuntu said:**
> You can run linux live cd's on virtual machine

Yes, but the whole point of live CDs is that you can run them on a REAL machine, to test out the REAL hardware.

A virtualiser, like Virtual PC, does not expose your actual hardware to the virtual machine. It emulates VERY generic hardware. For instance, if you have a sound card that requires special drivers to run, you will find that the virtual machine will have sound without the drivers. The virtualiser is taking the calls to the generic emulated sound hardware from the guest OS, and converting it into calls to the real sound hardware. Therefore, you can't test compatibility, because the guest OS is handling different hardware to what you actually have.

Ubuntu was not intended to be run inside Virtual PC; it is intended to be run inside a better virtualisation environment like VMWare, Virtualbox, or even Qemu; but most importantly, it's designed to run on real hardware.
