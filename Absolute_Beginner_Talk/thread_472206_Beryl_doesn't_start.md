### Post by Clay_Banger on 2007-06-12
backup your xorg.conf
```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.backup
```
then edit the xorg.conf file
```
sudo nano /etc/X11/xorg.conf
```
scroll down to the bottom of the file and change
```
Option "Composite" "0"
```
to
```
Option "Composite" "1"
```

[EDIT]
to restore your xorg.conf if it doesnt work
at the terminal do
```
sudo rm /etc/X11/xorg.conf
sudo mv /etc/X11/xorg.conf.backup /etc/X11/xorg.conf
```
And restart your computer
[/EDIT]

### Post by Najand on 2007-06-13
Beryl usually doesn't work with ATI Radeon Cards because it uses fglrx driver which not have any XComposite extension yet. Some people say they was able to do it, while some other says they can't. I believe searching the forum will help you more than writing such problem here in the "Absolute Beginner Talk".

### Post by ajmorris on 2007-06-13
go into synaptic and find beryl, do a **complete** removal of "beryl + dependencies" and "emerald + dependencies". then remove the source.list entry that you added because of the tutorial, and update the sources.list file (sudo apt-get update) then install beryl + Dependencies and Emerald + dependencies again, this time using the normal feisty repositories.
