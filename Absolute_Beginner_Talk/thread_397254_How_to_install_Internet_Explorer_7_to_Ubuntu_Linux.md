### Post by t0p on 2007-12-21
> **compmodder26 said:**
> The main reason for using IE in linux is to view sites that don't view well in other browsers.

If a website doesn't view well in Firefox, it has been constructed poorly and does not conform to standards.  Firefox is a highly standards-compliant browser; it conforms to all of the published standards relating to HTML and the world wide web.  When a site claims that it is "best viewed with Microsoft Internet Explorer", its webmaster is confessing that he hasn't put it together properly!

I think that we ought to use Firefox or similar standards-compliant browsers, and boycott sites that can only be fully used with Internet Explorer.  Firefox is available for Windows and Mac OSes as well as Linux, so everyone can use it.  I know that some people feel that they *have to* use some "Microsoft only" websites, such as some online banking sites.  But if enough people boycotted these sites, the companies involved would start conforming to standards and our world wide web would be greatly improved.

The standards are not a mystery.  They are widely available, and the internet industry agreed to them for a reason.  It's a crying shame that our world wide web has been perverted and distorted by one software developer.

### Post by will peavy on 2008-01-06
@t0p - Firefox 2 (the current stable release of FF) does NOT fully comply with W3C standards. For example, it fails the Acid2 test. 

Firefox 3 will be more standards compliant and will pass Acid2, as will Internet Explorer 8.

Of the major browsers, Opera and Safari are the two most standards compliant. And the Presto engine (used by Opera) has better SVG support than WebCore (which is KHTML derived and used by Safari).
