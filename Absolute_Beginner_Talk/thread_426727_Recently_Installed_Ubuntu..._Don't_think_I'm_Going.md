### Post by Sef on 2007-04-28
> Why when I drag an alias to the Applications menu does it not add it to the menu despite appearing as though it will?

System > Preferences > Main Menu > click on Menu subfolder > if your application is there, simply click on it.

> Why isn't firewall turned on by default?

It is on by default.  It is in the kernel.  If you want to change it either you have to use the terminal or download firestarter either from Synaptic or the terminal.

Installing from the terminal:

```
sudo aptitude update
```

```
sudo aptitude install firestarter
```

> (ClamAv)? Why not have 1 application = 1 installation package?

There is one installation package for system.  However, there are dependencies that may need to be downloaded.   Also plug-ins are usually an optional package.   You do not need anti-virus for GNU/Linux, unless you run an email client, and only then, so you don't infect your friends who run Windows.

> Why are applications named after seemingly random strings of letters and/or numbers?

They are not so random once you learn a bit.  I am slowly learning them.   Often they are dependencies, not the apps themselves.

If you would like to resolve your problems, then make a separate thread for each one.   Otherwise some may not get solved because people will focus on some of them and not all of them.  Also write a title that reflects the problem.

Plus what are your system specs?

> Why doesn't "search" find files in my file system that I know for a fact are there?  This is, in fact, a problem. I recommend using slocate, a command line search tool. Type 'sudo slocate -u' to build up the database for the first time, then 'slocate <filename>' to find files.

> Why when I drag an alias to the Applications menu does it not add it to the menu despite appearing as though it will? Not sure, but try using the menu editor, it should be somewhere in the preferences menu.

> Why are internet connections called ath0, ethO and other useless names?
Because they make more sense than 'JOHNS INTENRETZ CONNECHSUN', and are far more easily scriptable.

Not sure about this one.

> Why isn't firewall turned on by default?It is, it's called iptables, and it's buit into Linux itself. It's preconfigured, and Ubuntu based computers are pretty much invisible to the outside world, meaning attackers can't even detect you. You can download optional graphical frontends for iptables, but they're not required, unless you have very specific needs (such as allowing a program to access a specific port, or allow incoming traffic on a port, or something of that nature). The most popular is, as far as I'm aware, Firestarter.

> Why do I have install 7 different packages for one application (ClamAv)? Why not have 1 application = 1 installation package?Because Linux is open-source, and open-source software is modular. This ultimately means that it's more secure, easily patched (and more regularly patched), and you don't end up with a million different program libraries which do the same thing (which is what you get on Windows, causing conflicts, errors, and opening up many more security holes). Using the package manager, these dependencies, as they are known, will be satisfied for you.

Why do I have install 7 different packages for one application (ClamAv)? Why not have 1 application = 1 installation package?


Not to reinvent the wheel every time. If the program you have to write needs a task which is already performed by another program, you will probably invoke it to perform the task. Then your program will depend on the other one, and the two will have to be installled alongside. For example, programs to visualise picture have usually the option to rotate the picture. This task is
usually not performed in-house, but outsourced to another program called convert. This second one is a command-line image converter, which you are very unlikely to ever use directly, but is nevertheless installed when you install an image viewer. But you shouldn't worry about this, because synaptic will take care of installing alll packages you need to run a program.
