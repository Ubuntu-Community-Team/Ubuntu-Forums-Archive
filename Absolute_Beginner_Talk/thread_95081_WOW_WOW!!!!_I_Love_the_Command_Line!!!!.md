### Post by Kvark on 2005-11-26
[QUOTE=tomwell]Thanks For all your input, will look into scripting, maybe i can start off by reading other peeps scripts then see if i understand em...[/QUOTE]

The best way to start scripting is IMO to think of a task that you do pretty often and in the same way each time. Then type down the commands you use to do that task in a text file and there you have a script.


But reading scripts is also good. One way to study a script is to open the text file next to a terminal window and type the commands from the text file in the terminal. That way you will do what the script does one command at a time. Unless you type an if section or loop, in that case the terminal will wait with doing anything until you are done typing that section.

Just 2 things to think about when you follow a script like this:
1. If the script takes arguments like "somescript option1 option2"  then do "sh -s option1 option2" before starting to give your terminal the same arguments as you would give the script.
2. if the script does some calculations on a variable, use the "echo $theVariable" command to see what the resulting value is.
