Usually the reason guides use terminal commands 

1. Easier for the end user simply to paste said commands into a terminal.

2. There is not GUI or the GUI available may not offer the proper functions.

IMHO users coming from other OS's should try to understand first that Linux was a terminal/command line only OS before the arrival of GUI's for it. Thus some if not most users see the terminal as the first way to go, and GUI second. When writing guides.

As for making a GUI, it would help if your posted a link to the guide you feel a GUI is need for.

I would add my 2c ...

At first the command line is very intimidating, at least it was for me as well.

BUT, after a while, you start to learn the commands, and it is easier, more powerful then GUI tools ...

The GUI tools have come a long way, and I will admit I use them (nvidia-settings for example).

In terms of posting a solution to a problem, yea it is much easier to post a command then type out directions on what button to click from what menu.

Now I am sure there are many exceptions, but my guess would be that as users gain more and more experience they become more familiar with and comfortable with the command line as well.

So, don't despair ...
