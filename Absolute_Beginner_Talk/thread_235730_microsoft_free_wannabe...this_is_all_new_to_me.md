### Post by Ambimom on 2006-08-13
I'm new too.  Here's how I did it.  Take the CD and load it into your CD Rom drive.  It will open with a screen that shows you some software.  Close that screen, but keep the cd in your drive.

Now turn off your computer with the CD still in the CD Rom drive.
Start to boot up as normal.  Look at the top right of the Windows splash.  It will tell you to hit F8 or F2, or F-something else to get to your boot options.  It doesn't last very long.  If you don't catch it, then google "boot options" for your computer model.  Someone else has posted this, believe me.

After you hit that F key, you will get to a black and white screen which will instruct you on changing the boot sequence.  You want to boot to the IDE CD-Rom.  Move the + or - keys as instructed, hit Return and you should boot into Ubuntu CD.

It will start loading and eventually you'll get to a brown desktop.  There will be an icon instructing you to install.

Click on that.  You'll be taken through the setup.  It's actually quite simple.  

I wouldn't delete windows if I were you, but you can always reinstall later on.  Ubuntu will ask about partitions, etc.  Just take a deep breath, and read very carefully.  You can always start again if you don't understand something because it will pause at every step to ask if you're sure.

After it's installed, the worst problem I had was getting the software to do what I wanted.  I also have a bug in my sound card configuration, but it's minor and already being worked on by someone here.  It just took about a week before I learned that; and you'll spend equal amounts of time learning the oddities of your own configuration.

Oh another thing I learned too late....the codecs for media have to be installed separately.  There's a good guide on [https://help.ubuntu.com/community/RestrictedFormats](https://help.ubuntu.com/community/RestrictedFormats).  Good luck!

### Post by steve.horsley on 2006-08-14
I know lots of people are saying it, but please let me add my explanation too.

The file you download is a .iso file (as you know). This file is an image of a CD-ROM disk. It is possible with the right tools to "unzip" it, and see what will be the contents of the CD, but this is not what you should do. 

As an image file, it is a bit-for-bit image of what should go on the CD, including the boot sector and the iso9900 filesystem that contains the data files. It needs to be treated differently to any other fiel type, and burned to the CD **as an image**. The only windows CD writer I have used is Nero, and this had a menu option to burn an image file, which when selected opened a file open dialog box that specifically looked for files of type .ISO.

If you burn the image this way, 3 things happen:
1) You never unzip the .iso file, never open it to see the contents before burning
2) But once you look on the CD, a load of files and folders are there
3) And the CD is bootable (provided the BIOS can boot from CDs).

You can then reboot with the CD in the drive and it will start loading a Linux operating system into memory, much like a Windows installer CD does.

I hope this doesn't look like I'm talking down to you or being condescending. I don't know how much you know, so I am perhaps "stating the bleedin' obvious". And I may of course be completely wrong in my guess as to what the problem is anyway.

### Post by Tomosaur on 2006-08-15
Threadstarter:

Put one of you Ubuntu CDs (which you have burned) into your computer while Windows is running.

Go to My Computer, and right click on the CD drive. Select 'Explore'.

Now, do you have lots of seperate files on the CD, or is it just one file, which says like 'Ubuntu Dapper 6.06.iso'? If you have just the .iso file, then you have burnt the CD wrong. You need to have specific software to burn an .iso to CD properly. Many of the more popular CD burning programs have an option to 'Burn Image' or something similar. You need to select this option, and then pick the .iso file from your hard drive to use.

When it's done correctly, you will have seperate files on the CD, and not just one .iso file.

After downloading the iso, check the MD5 sum. After burning to CD check the integrity of the CD. I only run Linux and you are on your own as far as this goes.

Next partition your hard drive, at least 4-5 GB for Ubuntu. This can be done with the Ubuntu "live" cd with Gparted, but I have had problems with Gparted and prefer cfdisk.

Next, insert Ubuntu install CD and REBOOT YOUR COMPUTER to the Ubuntu cd (not windows). Follow instructions. Note the terminology for hard drives in Linux is not C:\ or E:\, rather /dev/hda1 (your windows partition), /dev/hda2, or /dev/hdb if you have more then 1 hard drive. Possible /dev/sda.

Now reboot to Ubuntu (hint the Ubuntu screen is Brown).

If you have the brown screen, you have Ubuntu. With no prior Linux experience you will feel very lost with sys admin, networking, mounting your windows partition, etc. Either learn to Google, learn to read, or loose the ego and follow advice.
