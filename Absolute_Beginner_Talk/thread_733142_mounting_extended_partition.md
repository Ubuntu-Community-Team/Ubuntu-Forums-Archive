### Post by teryowen on 2008-03-23
It looks to me like the partition table on /dev/sdb 160GB hard drive is messed up, try using a gparted live cd which has test disk on it and fix the paritions, after that the partitions should be able to mount. 

heres more about test disk [http://www.cgsecurity.org/wiki/TestDisk](http://www.cgsecurity.org/wiki/TestDisk) , with this fix your 160 GB hard drive. 

you can download the gparted live cd it has test disk on it, or just download anything else that has test disk. its a very helpful tool and should fix all the partition problems.

### Post by louieb on 2008-03-23
An extend partition is just a container for logical partitions. It does not hold any data directly. It holds logical partitions which is what holds the data.

/dev/sdb looks like it has 4 primary partitions. I echo the suggestion given by ***teryowen*.  **you can find TestDisk  and GParted on the [SystemRescueCd]("http://www.sysresccd.org/Main_Page")
