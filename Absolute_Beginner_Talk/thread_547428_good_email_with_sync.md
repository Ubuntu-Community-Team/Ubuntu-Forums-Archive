I think Outlook runs fine under [WINE]("http://www.winehq.org/"), which is a compatibility layer for windows programs. That is probably the most reliable way. I imported Outlook data into evolution, but it was not flawless.

You might also look into a program being developed under the codename [Chandler]("http://chandlerproject.org/"), being developed by the Open Source Applications Foundation. I'm afraid what happened to the Evolution port to Windows is happening to this project. I hope not.

Try my first suggestion, especially if syncing to Windows CE is necessary.
