Swap is used as an add-on to your physical RAM. When you run out of RAM, swap is used to store programs and get them back on demand. It is also used when you hibernate you machine. 
 
If you want to see your swap being used...open up a bunch of very heavy or intensive applications, then maybe (depending on how much RAM you have) you might see a small spike in the usage of your Swap

### Post by taurus on 2007-06-21
Edit /etc/fstab

```
gksudo gedit /etc/fstab
```
and replace this line

```
UUID=e6003b28-f65f-433e-b10e-e36d9c16801f none swap sw 0 0
```
with this one.

```
/dev/hda1   none   swap   sw   0   0
```
Save it and reboot.  Now, see if your swap in on.

```
free
```

### Post by Rui Pais on 2007-06-21
just a small note, hope taurus don't mind.

Mixed sata and pata drives configurations sometimes give problems at identification of drives by the kernel.

Some kernels detect pata as hdxx sometimes as sdxx (in my box the same installation and same kernel, cloned for backup, detect it differently according i boot from the installation on pata or sata :()

So for that cases the UUID notation can be handy :)
