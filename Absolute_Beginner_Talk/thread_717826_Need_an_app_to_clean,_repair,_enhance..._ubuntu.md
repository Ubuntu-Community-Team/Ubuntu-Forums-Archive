### Post by tjk on 2008-03-07
Those of you who have used Windows may be familiar with a program called **X-Setup** (found at:   [http://www.x-setup.net/product/features.asp?lang=EN](http://www.x-setup.net/product/features.asp?lang=EN)

This was (is) a fantastic app, I used it regularly.  It seemed that it was simply a shell that: organized / categorized plugins or scripts created by others to ... clean, repair, enhance and even give tips within all levels of the Windows system and specific software.  I really, really wish that someone would develop an application like this in linux as so much of the existing help seems to be scattered accross the web or even hard to find in the Ubuntu forum.  

An example of how X-Setup could be used:  I have continual trouble with my sound drivers not being able to load and/or getting corrupted.  Hopefully this app would have a number of scripts/plugins that would have detailed description and ultimately help me with this problem. In most cases it would be possible for you to test or undo the process to see if it resulted in a successful repair.  I had always hoped that X-Setup would add an area for user notes that could be attached to each plugin -- useful for things like prior selected items, adding more detail about the plugin, or even just noting if the plugin didn't work after trying it.  Potentially there would be links to authors or forums for each plugin so that it could be maintained or, restrictions and help information could be altered.  You would be able to search for keyword that described the plugin, as some categories and repairs may be numerous -- however you could browse by category if preferred.  It has the potential for solving display (video card) problems, restoring or repairing default setting, the list is endless.

If you think this is a good idea take a look at the website above.  You also may be able to download an older freeware version to try out so you could see how useful this app would be... that is, if you still run Windows (I wouldn't recommend running it under Wine -- it may mess things up in Linux)
