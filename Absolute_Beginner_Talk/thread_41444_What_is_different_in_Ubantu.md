### Post by aysiu on 2005-06-13
[QUOTE=ubantunovice]What I would like to know is:
1. What is different in Ubantu compared to other Linux distributions? Does it have more stuff, less stuff, different stuff, etc. Details would be helful. In other words, why Ubantu?
[/quote]
Ubuntu is free--you can download it for free, or you can ask them to mail you free CDs (postage also free), but those can take a couple of months to arrive. I'd say what sets Ubuntu apart is its community. It's very much there. I love Mepis and Blag, but if you ask a question to either of those two communities, you're very likely to get no answer or an "I don't know." Don't get me wrong--people *try* to be helpful there. They just either aren't helpful or aren't there. Other than that, there's nothing that really sets Ubuntu apart. It's a Debian-based distribution (so are Knoppix, Xandros, Libranet, and others). It uses the Gnome desktop (as do almost half the distros out there). It has productivity software included (imaging, word processing, internet, etc.), as does every major Linux distro.

> 
2. What is included in the CDs that are mailed out?  Is it just enough files to install the Linux system or does it include other software (word processing, spreadsheet, gimp,  etc.)? If not, how do I get this other stuff and from where?  Are CDs available for this other stuff or will it need to be downloaded on my slow dialup?

One CD is the "live" CD. This is the try-out CD, to see if Ubuntu recognizes your hardware, to see if you like using Ubuntu. The CD does not harm your hard drive. It runs off the CD and your computer's RAM. The other CD is the installer CD and includes everything you need to install Ubuntu and its basic set of programs. If you want more programs, you need an internet connection.

> 
3. Why are there so many different distributions? Why not a standard Linux with a range of applications one can install.  There seems to be just one Linux kernel, so what are the distribution differences?


I empathize with you. When I first started looking into Linux, this was my biggest roadblock. There seemed to be hundreds of distributions, and very few people actually help you pick the one that's best for you. Some will say, "This distribution. You have to use this distribution. There is no other distribution" (I've found this kind of response common among Gentoo and Slackware users). Others will say, "You find what works for you. There are many distributions." Yeah, big help. Thanks a lot.

I'd say if you're looking for a user-friendly distro that comes on one CD, try either Ubuntu, Mepis, or Blag. One word of caution, though, about Ubuntu: the documentation relies heavily on the command-line and *sudo*. This may be a good or bad thing, but you should know it is the case. You may have to type something like *sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf_backup*, then *sudo gedit /etc/X11/xorg.conf*, just to edit your configuration file to adjust your monitor resolution.

Granted, the documentation out there for Ubuntu is fantastic, but it can be intimidating for those not comfortable with the command-line. Mepis has a couple advantages over Ubuntu (even though the Mepis documentation sucks):

1. A lot of stuff you have to configure in Ubuntu is there straight out of the box in Mepis (ability to turn numlock on by default--yes, I realize this is more a Gnome/KDE issue, but Mepis is KDE and Ubuntu is Gnome right now--ability to have other partitions, say Windows, appear on the desktop, ability to browse as root at the click of a button and the entry of a password, extra repositories can be enabled via checkboxes).

2. Mepis is both a live CD and installer.

On the other hand, Mepis is bloated. KDE, in general, is a bloated desktop (Gnome is a *little* more light-weight). Mepis comes with all sorts of crap software you don't need, like a weather thing in the taskbar that usually doesn't even work. Ubuntu, however, often gives you too little software--I had to download totem-xine to get my DVD player to work.

In the end, though, you're right... Linux is Linux, for the most part, especially if you're dealing with different Debian-based distributions. Ultimately, there are only a handful of factors that should inform your decision over which distro to use:

1. Do you have any friends that use this distro? Well, if you do, go for it. Friends are much better at helping you than forum members, because they can be hands-on. And if your friends don't know how to do something, you can all learn together.

2. Do  you like to do stuff yourself or have it done for you? Debian has you do everything yourself. Ubuntu configures most stuff for you, and you have to do a hefty bit of tweaking, too. Mepis configures almost everything for you, but you still have to do some tweaking.

3. How easily can you get the distro? You mentioned you have dial-up access to the internet, so downloading ISOs that are 698 MB large and burning them isn't going to work for you. Can you check the distro out from the library (they're usually in the back of library books)? Can you borrow it from your friend? Can you wait two months for Ubuntu to ship you some CDs? Are you willing to pay some money to have a company ship you CDs?

4. Does the distro recognize your hardware? This is the most important question. Of course, you won't really know until you *try* the distro, but this should be the deciding factor. It's not too hard to get a Gnome desktop if you come with a KDE (or vice versa), and it's not too hard to install or uninstall programs, but when a distro won't recognize your mouse, video card, modem, and sound card, you're in trouble. If possible, I'd get ahold of a Mepis live/installer CD and a Ubuntu live CD. See which one recognizes your hardware. Then, go with that. Mepis is in the back of a book called *Point-and-Click Linux*, though it's not the most up-to-date version.

### Post by aysiu on 2005-06-13
[QUOTE=ubantunovice]Is Mepis also free to download? Can you use Mepis apps within Ubuntu (sorry for the earlier misspelling)? 

I'll probably go with Ubuntu but you seem to be leaning towards Mepis except for their support. Why is that?

Thanks again.[/QUOTE]

Mepis usually is free, but it also allows you to donate money. They don't guarantee they'll always be free, but you can download 3.3.1 (their latest version) for free now.

I've played around with Mepis and Ubuntu a lot. In fact, I'm currently triple-booting XP, Mepis, and Ubuntu. Mepis just did all the stuff I wanted right out of the box. It also recognized my monitor settings right away. Ubuntu did not. Here are issues I ran into with Ubuntu that I didn't with Mepis (not to say these will necessarily be issues for you, but they were for me):

1. I had to manually edit my /etc/fstab to get my Windows partitions to mount. Even when mounted, they didn't show up on the desktop. To make a shortcut from Windows to my Ubuntu desktop, I had to put the Windows mount inside of my home folder. Even then, I couldn't make a shortcut from a file within the FAT32 Windows partition to the desktop in Ubuntu. In Mepis, all partitions automatically appeared on my desktop. All I have to do is click on the partition and it will mount and open. To create a shortcut from my FAT partition to Mepis, I just dragged the file to the desktop, which brought up a context menu, then I clicked "link here."

2. Enabling extra repositories (more available software, basically) required me to manually edit the /etc/apt/sources.list file. In Mepis, I just opened Synaptic Package Manager, clicked on "settings" > "repositories" and checked all the boxes (most come unchecked).

3. Mepis includes a GUI partitioning tool as part of the install process. I found Ubuntu's text-only partitioning confusing.

4. Ubuntu expects you to make all changes as root through the command line and *sudo*. You can enable a root log-in, but this takes extra steps. It also means you have to log out, then log back in as root. In Mepis, there's a button that you click that will allow you to browse as root *within* the user account (provided you give the root password).

It's little things like that that make Mepis easier for beginners. Of course, some beginners love the command-line. If that's you, go for Ubuntu, because, as I said before, there's plenty of documentation. Also, even though Mepis was better at recognizing *my* hardware, I know others have found Ubuntu to be better at recognizing *their* hardware. Please try both, if you can. They're both Debian-based, and so use the *apt-get* / Synaptic way of installing software. One is KDE, the other Gnome, but if you get Mepis, you can always download the Gnome desktop. If you get Ubuntu, you can always download KDE (or you can get Kubuntu, which is a KDE Ubuntu).

[QUOTE=ubantunovice]
From what I had read elsewhere, I was leaning towards Debian. So, what does it mean that Ubantu is "Debian based"? Does that mean I could install items meant for Debian into an Ubantu installation?[/QUOTE]

What that means is that the Ubuntu project didn't start from scratch. Ubuntu copied Debian at first and then began to improve it, turning it into Ubuntu. If you must you can install debian software packages in Ubuntu, but its not recommended because:

A. It might not be compatible. The changes made by Ubuntu (to improve things) breaks compatibility with Debian.

B. Ubuntu already has almost every Debian package in its own online server (that will all install without problems).

> You say Ubantu is designed for the desktop. I am planing to try it on a Dell Inspiron 8100 laptop.  Does "Desktop" include the laptop desktop?

Yes. Ubuntu's next release is more focused on laptops (if you must have everything in it work-such as hibernation- you might wait till then) but Hoary will install and run on laptops (it on mine).

> "Ubantu is Gnome based ...". Does that mean it cannot use KDE or just that it needs to be added later?

A. Needs to be added later by downloading off internet (or you can get someone with broadband to make you a kubuntu install cd instead).

B. Isn't as polished as Gnome.

> How easy is it to install applications in Ubantu? Does it check for dependencies, etc?

We have a program called synaptic. You check a box in Synaptic next to the program you want to install, click apply and the program is installed (even though it might not show up in the menu). Last time I checked Synaptic in Ubuntu said I have 14000+ programs availible. Synaptic does all the dependancies. The only problem will come if you can't use your computer to download the packages (aka you downlaod them on someone elses machine and install them by hand), in that case you must resolve dependancies yourself.

> If I go with Ubantu, what would be the best book to use along with it?


Ubuntu is so new that we really don't have published books on the shelf yet. our best resource is the guide:

[www.ubuntuguide.org](www.ubuntuguide.org)

If you really need a manual, some other versions of Linux (such as SUSE, its manual is great) ship with manuals in the box.
