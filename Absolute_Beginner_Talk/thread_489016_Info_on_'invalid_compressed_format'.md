### Post by jc508 on 2007-06-30
This is collating some of the info I discoverd posted all over regarding 'invalid compressed format' errors during install - just to save others the effort.
These come as invalid compressed format (err 1)  and invalid compressed format (err 2) that I have seen along with 'CRC error' and all end with 'System Halted'

Following the suggestions in this forum I obtained fresh downloads of 7.04 main and alternate releases.
These were all check-summed fine.  The CDs were verified after creation on the box that created them.

Now move to the box I am trying to install Ubuntu 7.04 on- its a bit of a 'left overs' box and to cut the story short the problem of never ending 'invalid compressed format' errors half way through things was the CD drive itself.
It was an old 32x drive and when i took it out it has a manufacture date of 1998.   Swapped the DVD drive  from the other box and it never missed a beat.
Not knowing what happens on the inside of these things I can only assume the old drive cant handle some mild variant of the format of modern images - eg I am sure bootable CDs wern't invented in 1998.
