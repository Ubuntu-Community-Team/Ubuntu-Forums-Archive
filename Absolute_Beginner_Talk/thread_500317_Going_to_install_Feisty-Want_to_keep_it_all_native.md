k9copy will just as easily create a 1:1 rip with the menu's and everything intact.

### Post by Ted Nancy on 2007-07-17
Well, I've been using Linux for less time than you... off and on for the same reasons I would guess. But I copied and pasted the instructions... these worked for me, but I don't think I can troubleshoot any problems you may have. Unfortunately, it still crashes half the time, but when it works, it works perfect! For always perfect, scroll down.

First, remove any instances of wine you've got. I think this ought to do it:
```
sudo aptitude remove wine
```

Then import wine's GPG key:
```
wget -q http://wine.budgetdedicated.com/apt/387EE263.gpg -O- | sudo apt-key add -

```

Then get the latest build (for feisty only):
```
sudo wget http://wine.budgetdedicated.com/apt/sources.list.d/feisty.list -O /etc/apt/sources.list.d/winehq.list
```

Then it's the usual update install rigmarole:
```
sudo apt-get update
sudo apt-get install wine
```

I know, you probably know all this, I'm just being thorough. This is all from the Wine website by the way. Now download the latest version of DVD Shrink, 3.2 as of this posting. I downloaded a .zip file to my desktop then extracted the .exe. I'm assuming a copy of the installation .exe. is on your desktop.

Now I:
```
cd ~/Desktop
wine dvdshrink32setup.exe
```

Just say next to everything, but be sure to uncheck the "launch DVD Shrink now" button. Then do:
```
cd /usr/local/bin/
sudo winecfg
```

Now change default to XP. Click add application, browse to DVD Shrink 3.2.exe, double click, select windows XP (default doesn't work, for some reason) click apply. Click audio (looks like it freezes, maybe b/c I'm running Amarok), I left my with OSS, the click OK.

Now reboot.

Now, for some reason I had to go into my menu config to make wine show up (strange, since it was already checked), but anyway after running >system >preference >main menu and making sure a check mark is beside wine (may as well remove the uninstall crapolla), I find it in my main menu, and voila, DVD-Shrink.

Of course, even under windows there were situations where DVD-Shrink would fail. DVD-Decryptor  and other programs were starting to pitch in more and more I noticed over the last year. But so far I've been pleased with the compression and editing options available.

I don't burn with it. So I can't attest to that aspect. I create an iso file always and burn that using Ubuntu's built in iso burner. 

It crashes sometimes. And when that begins to annoy me I go on to this solution.

(If you're having problems, make sure you've got all the libdvd stuff installed. If you can't watch DVD's I doubt you can rip them. I could be wrong.)

**ALTERNATE SOLUTION:**

I'm using virtualbox and windows professional to run SPSS 15 as a "seeming" native app under linux. Until linux apps get up to snuff on backing up your DVD's, maybe this is a nice compromise position for you. Check for the threads on seamless integration.

This will allow you to install the latest DVD ripping software. Pretty important, since DVD-Shrink is no longer supported, and more and more DVD's are encountering decryption errors. But for those 2006 and earlier discs, this should work nicely. For newer stuff, like Lost Season 2 (which DVD Shrink couldn't do for me, K9Copy did). Sadly, I think in today's age, a number of DVD ripping programs are necessary. The days of DVD-Shrink or DVD-Shrink + DVD Decryptor are over.

Oh P.S. Not telling you to use K9, I've finally gone through the pages of this thread [:)], I'm just illustrating why DVD-Shrink ain't all it used to be. I think the virtual box solution is a nice one until such time as you find the gnome app you're looking for. DVD Shrink, Decryptor, Any-DVD, Nero, and any others that give you that great DVD ripping experience within Windows.
