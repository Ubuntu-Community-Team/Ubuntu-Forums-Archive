### Post by Ted_Smith on 2007-08-20
LightZone is by far the best tool for the photographer who uses Linux. If you're into manipulation, use the GIMP. If you're a photographer who wants to process their RAW files in an efficient way using 16-bit software instead of 8-bit, LightZone is the way to go. 

It is similar to LightRoom but better IMHO (it adopts a digital version of the Zone System devised by Ansel Adams) and is purely geared to the photographer as opposed to the digital artist. 

It costs over a $100 for the Windows version, but is free (but unsupported) for Linux - download www-old.lightcrafts.com/linux/ . The latest Linux version is 2.4. Version 3.0 has been released but not ported for Linux yet. 

I use it in favour of The GIMP for all my work and have never looked back (see [www.tedsmithphotography.com](www.tedsmithphotography.com)).

There is also Bibble by Bibble Labs which is equally good, but it costs. There are a few threads on this subject - do a search for 'raw files' and 'software' and several threads discussing all the Linux photography options will be displayed

Which version of Ufraw were you using? If it was the one in the ubuntu repositories then I'm not surprised that the WB profile for you camera was absent - so was mine. I've since downloaded and installed the latest release (0.12.1) from [here](http://ufraw.sourceforge.net/Install.html) - and now the WB profile for my camera is there, and images are bright and vibrant.

To start you need to install gtkimageview:

I downloaded it from here: [http://ftp.debian.org/debian/pool/main/g/gtkimageview/gtkimageview_1.3.0.orig.tar.gz](http://ftp.debian.org/debian/pool/main/g/gtkimageview/gtkimageview_1.3.0.orig.tar.gz)

Unpack it (I double click and extract the folder to Desktop) - and change to the extracted folder in Terminal.

To install you should type this in the terminal window:
```
./configure --prefix=/usr
```
then
```
make
```
then
```
sudo make install
```

Then, before installing Ufraw 0.12.1 you must satisfy the dependencies: 

```
apt-get install libgtk2.0-dev liblcms-dev libgimp2.0-dev libtiff-dev libjpeg62-dev libpng12-dev libexiv2-dev zlib1g-dev libbz2-dev
```

And then install Ufraw itself (after unpacking it's archive):

```

./configure --enable-mime --enable-extras --with-libexif --with-exiv2
```
then
```
make
```
and finally
```
sudo make install
```

With any luck Ufraw should now be in your menus under Graphics. ;)

If you need any help give me a shout.

Gtkimageview will compile without the ***-- prefix=/usr*** but when you come to run Ufraw for the first time you will get this error:

```
ufraw: error while loading shared libraries: libgtkimageview.so.0: cannot open shared object file: No such file or directory
```

Well I did when I missed it off.

With the gimp, just right click on the .PEF raw file (or whatever your raw file extension happens to be) and choose open with other application, then choose "Use a custom command" typing "ufraw" in the dialogue box.
