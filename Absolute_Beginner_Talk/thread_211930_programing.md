### Post by gruepig on 2006-07-09
For really basic stuff, there's bash.  Bash is what you are using whenever you are at the command prompt, though there's a bit more to it:  [http://www.tldp.org/HOWTO/Bash-Prog-Intro-HOWTO.html](http://www.tldp.org/HOWTO/Bash-Prog-Intro-HOWTO.html).

Moving beyond that, I think Python is an excellent first programming language.  There are tons of tutorials and guides, including many aimed at people new to programming: [http://wiki.python.org/moin/BeginnersGuide/NonProgrammers](http://wiki.python.org/moin/BeginnersGuide/NonProgrammers).

So why Python rather than some other language?
Python is useful and commonly used in the linux environment.  It's likely already installed or easily installable.  There's an interpreter that you can use for short programs and debugging (meaning instant gratification).  It's higher-level then C (meaning you can learn to write useful programs more quickly).  It's more structured than Perl or PHP.  It has a larger current user-base than LISP/scheme.  Unlike logo, it doesn't feel like a toy language.  It's object-oriented, which is in vogue.  It's simpler than Java.  (Ruby is probably a good first language as well; I just don't have any experience with introducing beginning programmers to ruby.)

### Post by deepspring on 2006-07-09
I recommend learning the basics first instead of diving straight into the likes of Python, Ruby, etc...

I'd start here:
[http://www.htdp.org/](http://www.htdp.org/)

It's starts you off with the fundementals of designing and building applications without burdening you with the overhead of complex languages like Java, C# (mono), C, C++, etc.

Once you complete that online book, I recommend two of Bruce Eckel's excellent and free books: "Thinking In C++" (Volumes 1 and 2) and "Thinking In Java".

Both are available from here:
[http://www.mindview.net/Books](http://www.mindview.net/Books)
