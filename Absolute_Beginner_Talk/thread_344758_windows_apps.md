### Post by oilchangeguy on 2007-01-23
yes they will be useless, unless it's noted on the disk that it'll run in linux. there are open source programs that are close to many programs designed for windows. you can also load wine, and you might be able to run your programs from within wine. but i've seen many problems with wine here in the forum. another program that allows you to run programs written for windows is cross over office. this is not a free program, but a 30 day trial is loaded in xandros oce linux, the full version is loaded in some of xandros' paid versions. and no i'm not knocking ubuntu, it's a better os than xandros. and of course you can go to crossover offices website and obtain the program from there.

### Post by Unterseeboot_234 on 2007-01-23
Photoshop is the one notable program that will not accomodate the WINE bridge. Photoshop is dependent on the C:/ and Win folders. Sound is a problem in Linux, you will need to load up 3 memory hogs to accomplish a soft midi synthesizer. But after all that, Linux has superior software that is free -- as in free beer. I don't know of one Win graphics artist that purchased 100% of their software they use in Win.

### Post by jvc26 on 2007-01-23
Alas photoshop is unlikely to work, I think 7 works with no hitches but CS2 is less well supported and so doesnt run properly. The GIMP is an alternative, and depends with how much of a major photoshop user as to whether you will find it adequate/inadequate. Its brush system isnt as good as that of photoshops and it lacks some of CS2s functionality - I'd recommend trying it out then you can see how much different it is. You can go to WineHQ and see which windows apps are likely to work with what success. Crossover office is another option (although a paying one) which may well allow you to run photoshop better - see crossover's webpage)
