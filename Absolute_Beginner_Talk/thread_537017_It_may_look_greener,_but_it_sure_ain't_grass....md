### Post by ThomG on 2007-08-28
A note of warning to people who have heard the sermon and are thinking of getting baptized.

For years I've heard Linux "believers" put down Windows for crashing... 

Holy-holey-belief-systems... I've never experienced such a buggy, crash-prone, system-freezing, piece-of-grass-roots-horn-tooting-feces, as I have since loading up Ubuntu 6, Ununtu 7, Kubuntu 6, Kubuntu 7, FreeSpire 2, etc...

I've done a clean install, of each of the above, to an AMD 32bit system with a 60G HD, 512MB RAM, ATI 9600, and Linksys USB wireless adapter.  

The installs completed without a hitch.  

After install, the wireless adapter worked with varying degrees of complexity for setup under all versions except those based on Ubuntu 7.  To date, I've not gotten the wireless adapter working with the newer Ubuntu 7.  I used a corded connection to connect to the internet when using distros based on Ubuntu 7.

With internet connectivity in place, I attempted to download and install the atleast 125+ updates that were available for each of the versions I tried.  Not once was I able to download and install the updates on the first try.  Atleast once, with each distro, my system froze with no warning or indication of a problem other than the fact that the system was stuck on the same screen asking me to wait (longest wait was four hours before throwing in the towel) and the mouse/cursor was unresponsive.  With no other way to reboot, I power-cycled the system/s (removed power), rebooted, and was booted directly back into the same frozen system.  At this point, with no other option immediatly evident, I reinstalled the distros, and started over.  Only Ubuntu 7 with a corded connection was able to download and complete the installation of the updates, but even this was on the second try.

Anyway, I'm currently on hour three of the "Please wait, this can take some time." of the installing updates process of my second attempt to update Ubuntu 6... to be clear... the updates have downloaded and the update process is in the installation portion of the process and has been sitting on "Unpacking replacement linux-image-2.6.15-26-386 ..." for over three hours.

I'll give it another two hours, just to top my previous best of four, then I'm reinstalling WinXP.  Regardless, this venture into Linux HAS cost (30 hours) me more than I paid for three previous versions of Windows combined.

It may look greener, but it sure ain't grass.

### Post by bobbybobington on 2007-08-28
I think a lot of problems as far as OS's go has to do with hardware some linux installs are rock-solid and have gone without reboots for months if not years. Some hardware is pretty problematic though, and linux doesn't have the advantage windows have of hardware being specifically built to support the OS (unless you buy from a good linux OEM). If linux doesn't work very well on one machine, then yes, it makes more sense to use windows on it. But don't be afraid of checking it out (via live cd) on any other machine you have. If linux ultimately isn't as good as you thought it was or what someone else said, then go ahead and use what is best for _you_. If you're still curious about linux in a year or two, you can always test the waters anytime you like, but its important to take the plunge when linux  is good enough for you.

### Post by raijinsetsu on 2007-08-28
This definitely sounds like a hardware problem. Usually, it's the graphics card (you said the install went without a hitch). Check to see which driver is being used for your display. You can do this by opening "/etc/X11/xorg.conf" and going down to the "Section Device" and reading the driver line. If it says "fglrx", change it to "radeon".
You do know that ATI support in Linux is limited, not because of Linux, but because ATI refuses to release proper drivers(the new fglrx has issues)?

Also, next time, try asking for help before you bash an entire community for something they have no control over.
