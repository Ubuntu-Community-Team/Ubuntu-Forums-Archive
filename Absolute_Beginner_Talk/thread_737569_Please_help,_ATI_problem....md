### Post by dstew on 2008-03-28
If you want to get the ATI card to work, you have to be patient and not try too many things. If I were doing it, this is what I would do:

1. Put the card in the machine
2. Check the BIOS to see if I need to disable the on-board (Intel) video.
3. Boot
4. Press ctrl-alt-F1 to get a command line.
5. Enter sudo dpkg-reconfigure xserver-xorg, and reconfigure the card to use the vesa driver.
6. Press ctrl-alt-backspace to restart the xserver
7. Check the System --> Administration --> Restricted Drivers Manager to see if there is a driver for the card (note: the card has to be plugged in in order for this to work)
8. If there is no restricted driver, reconfigure the xserver again to use the ati driver. If that gives you no screen, reconfigure again to use the vesa driver, you are probably stuck with it. Or, pull the card and go back to the on-board Intel video.

I wouldn't bother downloading drivers. If the Restricted Drivers Manager doesn't have one for you, it probably won't be fruitful to keep looking around. Sometimes, a driver will appear that takes some time to get through the Ubuntu developers to appear on the Restricted Drivers Manager, but usually if the driver is good, it will be there after a few weeks.

### Post by lswest on 2008-03-28
Just a quick idea, but why don't you try this: boot into linux using the onboard graphics then open a terminal and do ```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.backup
gksudo gedit /etc/X11/xorg.conf
```
then you will (with the first line) make a backup of the xorg file, in case something goes wrong, you can undo your changes with ```
sudo cp /etc/X11/xorg.conf.backup /etc/X11/xorg.conf
``` and after you open the file in a text editor (second command) find the line that says "Driver: "intel"" and change it so it read "Driver: "fglrx"" and reboot...make sure you plug it into the ati card.  that SHOULD get it working, if not, restore your xorg file and reboot plugging the screen into onboard graphics again.
