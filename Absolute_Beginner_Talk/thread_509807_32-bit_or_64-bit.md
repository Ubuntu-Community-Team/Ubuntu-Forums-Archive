This is a bit contentious, but here's my take.

For the average home user, at the current time, there is very little advantage to using a 64-bit version of the operating system (OS). The primary advantage of a 64-bit OS is that it allows you easily use more than 4GB of RAM in total, and allocate more than 2GB of memory address space per process. So unless you have huge amounts of memory in your PC, there's not a lot of point in using the 64-bit version.

It is possible that some specialized applications such as encryption/decryption programs may run a little faster under a 64-bit OS provided they have been correctly compiled to do so. Equally possible, some applications may run slightly slower, depending on how they have been coded/compiled.

The downside of 64-bit is that while all applications that are available have 32-bit versions available, not all applications have 64-bit versions. So if you want to run some applications under a 64-bit OS you either won't be able to, or be forced into contortions to try and run that application in a separate 32-bit environment on your 64-bit OS. While more and more applications are being released with both 32-bit and 64-bit versions, right now there is still more 32-bit software out there.

So I'd advise sticking with the 32-bit version of the OS.
