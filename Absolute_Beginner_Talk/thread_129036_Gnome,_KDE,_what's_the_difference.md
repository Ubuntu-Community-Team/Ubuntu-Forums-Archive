### Post by welsh_spud on 2006-02-12
There are lots of differences. The two main DEs (Desktop Enviroments) are KDE and Gnome. Some people tend to call KDE a windows clone, which I personaly think is pretty weird, because KDE works BETTER than Windows;) . 

KDE takes a more 'everything but the kitchen sink' aproach to menus and configurations. Gnome on the other hand takes a more simple, 'we will take care of this for you' mentality. Both work great for me.

Within both DEs, native programs integrate very well. Example: Amarok and konqueror will work hand in hand with each other. So will Kword and Kspread (or whatever the spreadsheet program is called). Gnome native apps work together in a similar way. However, mixing KDE and Gnome apps results in a perfectly working pair of programs, but look and behave totally differently to each other.

Some people also say that Gnome can be a bit more quick and nippy speed wise, compared to KDE. The jury is still out on that one...

What else...Nothing much really, well not off the top of my head anyway. To learn about the differnt DEs check out:

[Interesting article of KDEs core features]("http://www.terra.es/personal/diegocg/kde/")

[This great comparison of the two big DEs]("http://www.illusionary.com/GNOMEvKDE.html")

### Post by aysiu on 2006-02-12
You don't have to make a choice:

```
sudo apt-get update
sudo apt-get install kubuntu-desktop
```
log out.

session > KDE

log in.

That said, you also don't need to install KDE to use KDE extras or KDE applications. You can use KDE applications in Gnome and Gnome applications in KDE. They may look **ugly**, but you can use them all in either environment.
