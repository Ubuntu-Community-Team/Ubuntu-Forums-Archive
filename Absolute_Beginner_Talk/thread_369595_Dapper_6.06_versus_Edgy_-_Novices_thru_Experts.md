### Post by moshuptrail on 2007-02-24
The answer depends on your needs.

If you want tried-and-true, and just need to get things done - choose Dapper.

If you have special hardware needs, or you have to have the latest and greatest, but you don't mind running into a few oddities - choose Edgy.

The support for Edgy is excellent on this forum.  Most of the folks here are "early adopters" and have already upgraded to Edgy.    There are a few things that are only supported in Edgy.  They tend to be on the cutting Edge, pun intended, kind of stuff.

Some might say Edgy is not as "stable" as Dapper.  I would disagree.  Edgy is fundamentally stable.  But there may be a few undocumented "features".

### Post by n8bounds on 2007-02-24
The decision for me when I was first getting into Edgy was purely hardware based.  Was it easier to make Dapper work on my hardware than Edgy?  If so, I used Dapper and only pulled out Edgy when I couldn't get the older kernels to talk to my copper very well.  The newer version of Gnome on Edgy is a little faster it seems and there are few other nice things that I always miss when I go back to Dapper now, (Firefox 2, for instance) but it is kind of the more stable of the two.  It is also MORE like other Debian distros than Edgy (it uses init), so in your knowledge hunting, you can broaden things beyond just Ubuntu a little farther if you are using 6.06 or earlier than if you are using 6.10.  

If you are a really really new, it might be cool for you to start at the beginning (pre-5.10 stuff) and go thru all the versions.  That'd show you how far we've come....that reminds me, I need to go find my old install CDs.  (The live and install CDs used to be SEPARATE.)

Edgy is for more wizz bang ideas than Dapper in general.  Try to get your stuff rolling with Dapper.  If you have exotic hardware that works best on newer kernels (that aren't being introduced to the Dapper repositories) go ahead with Edgy (and Feisty here shortly).  

Have fun!
