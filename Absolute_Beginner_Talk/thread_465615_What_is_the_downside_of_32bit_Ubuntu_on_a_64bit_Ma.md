### Post by 3rdalbum on 2007-06-06
On average, 64-bit is the same speed as 32-bit. Some things are faster, some are slower.

The only difference comes when you want to use more than 4 gigabytes of RAM. You generally need a 64-bit OS for that (I say generally... because some 32-bit chips have a larger address space through PAE anyway!).

But you'll have much fewer troubles with the 32-bit version.

Saying what you've said is like buying a car and saying "Hey, it's a waste that the speedo goes up to 200kph but the limit on the roads is 100kph, maybe I should move to Germany and drive on the Autobahn".

Well, look at it this way - you don't really pay extra for 64-bit CPUs these days, which is just as well, because as far as desktop use goes, 64 bit buys you nothing.

As a desktop user the only difference you're likely to notice is that a 64-bit OS let's you use over 4GB of RAM. Generally speaking the speed difference between the same program compiled for 64-bit and 32-bit is negligible. There are a few exceptions, such as encryption/decryption programs that use a lot of integer operations, but generally not.
