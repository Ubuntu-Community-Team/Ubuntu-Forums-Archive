### Post by freebird54 on 2007-04-12
What kind of startxgl.sh file did you end up with?  Here is what I have.  The bolded section is to prevent loss of the shutdown and restart buttons...

```
#!/bin/sh
Xgl :1 -fullscreen -ac -accel xv:pbuffer -accel glx:pbuffer &
DISPLAY=:1
[B]cookie="$(xauth -i nextract - :0 | cut -d ' ' -f 9)"
xauth -i add :1 . "$cookie"
[/B]exec dbus-launch --exit-with-session gnome-sessionCODE

```

Some of the guides suggest that 'exec gnome-session' is sufficient, but I took the suggestion I found and run it with the last line shown here.

That SHOULD at least do that much for you.

Another alternative is to start a non-Xgl session and take any automatic startup out of the way - to run normally till we find out what's best to get it going!
