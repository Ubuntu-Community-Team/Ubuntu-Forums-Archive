Setting up Beryl, messing with settings, compiling programs... even recompiling your kernel are all good ways to learn more about linux. ### Post by irish_flu on 2007-01-16
- As others have said, using an OS is the best way to learn an OS.

For example, think of a specific task you want to do.  For instance, "I want to log into Google Talk".  Some messing around in the forums will likely eventually lead you to GAIM (or something else, whatever).  Then you'll have to learn to install GAIM (so you'll learn Synaptic).  Then you might want to add in some plugins.  Then the settings for Google Talk are a little weird.  Then, you find out that only the newest, most shiny version of GAIM, which might not be in the repository, is the only one that does "the awesome thing".  Then etc etc etc....

As somebody else noted, another great way to learn is to help other people.  For instance, after all your work with GAIM you see a forum post where somebody is having some issue with their Yahoo account in GAIM; it's not one you've done, but by now you know enough about GAIM and such that you know where to look for answers, and which version does what.

Now, you've suddenly learned a lot!  It's lot easier and more fun (in my opinion, anyway) to learn something when the stuff you're learning about can help you do something you want to do (as opposed to learning stuff you might not care about, even if you do have a thirst for knowledge).


Also: When I enter a command into the command line and I think I'll have a hard time remembering it, or if it's a multi-step process I think I might forget, I take a screenshot of my desktop with all the steps visible so that I have it for later.
