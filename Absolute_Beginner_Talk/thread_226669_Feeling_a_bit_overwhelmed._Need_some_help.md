### Post by jimbren on 2006-07-31
One of the lovliest things about managing software with Synaptic (or apt-get, aptitude, Kynaptic, and so on) is that you don't have to worry about the dependancies.  If you decide you want to remove a program, you just select it and the Synaptic does all of this for you, no longer needed dependancies and all.  If you want to be sure that you've removed all trace of the program and any dependancies, then do a purge instead of a simple uninstall.  

Purging, unless I'm mistaken, can be somewhat risky though, because it will remove all other programs and dependancies associated with one program, regardless of whether or not another program also has the same dependancy.  So you can wind up with broken packages.  This is usually nothing more than a minor inconvenience, because you can use Synaptic to fix the broken package too.  

As for your soundcard...I'm not sure why the volume was flaky due to the way you left it in XMMS, but if the card didnt' have a driver associated with it, then it wouldn't be working, so no worries there.  It is also listed on the [hardware compatibility list on linuxquestions.org ]("http://www.linuxquestions.org/hcl/")  

As for archivers, the name of the specific application escapes me right now, but managing archive files should be pretty easy.  For example, you should be able to just right-click a file in Nautilus or Konqueror and select Extract from the menu, which will then give you sub-menu of other options.  Part of the problem you're having here is that you're used to doing things the Windows Way.  Windows is stupid.  Linux is smart.  

Have you checked out the Ubuntu wiki and some of the other stuff the Ubuntu Documentation Team is putting and has put together?  They have a lot of excellent information that has been tested and edited by people a lot more experienced than you and me.  This is usually my second stop after the forums when I want to look something up.  You can find that stuff here: [http://doc.ubuntu.com/](http://doc.ubuntu.com/)

On the discombobulation--we've all been there and will be there many times more.  That's one of the best things about the Ubuntu community.  In two years, I've never seen anyone shut another user down for asking a question.

kisses,

jimbo

I see you've decided to reinstall, and I don't think you'll regret it.  Learning a new OS, especially getting it "just right", is a complicated process.  You're not alone feeling that things "could have been done better" the first time around.  Reinstalling makes you a lot more conscious about what software to have installed. Just reinstall once more when you're done experimenting.  It really pays off to get to know the system better, especially if you know that it's not the final version anyway.

There's no surer path to frustration than sitting in front of your screen wondering what to do next. #-o

### Post by mscman on 2006-08-01
Hey Sweet Spot. I noticed nobody else seemed to help you with your 7zip problem, so I figured I'd throw in my two cents.

You can add 7zip functionality to file-roller (or ark I believe in KDE) by installing the package p7zip. To do this from command line simply open a terminal window and run the command:
```
sudo apt-get install p7zip
```
As for Automatix vs. EasyUbuntu, like the other posters above I've used both and ran into few problems with both. I recently did a reinstall on my laptop because I brutally crashed my system and didn't feel like taking the time to fix it, since I had my data backed up anyway. This time I used Automatix and had to install very few extra packages by hand.

As for learning the OS and finding new things to do, I strongly recommend browsing around the [Ubuntu Documentation Storage Facility (UDSF)]("http://doc.gwos.org/index.php/Main_Page") and finding fun stuff to do on there (i.e. check out the gaming pages, the eyecandy pages, etc.) It's also a great source for new information, alongside these forums and the official Wiki.

Have fun! :D
