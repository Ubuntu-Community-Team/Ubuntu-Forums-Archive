### Post by GSF1200S on 2007-11-26
The OP listed a complaint about Ubuntu asking for a password when admin stuff needs to be done, and Id like to explain it. 

Vista and Ubuntu do this for chiefly the same reason, but Ubuntu implements it much better. The principle of verifying actions on an admin account is fairly simple.. it makes sure you are trying to do it, versus a virus or malicious program trying to do it. The thing with Vista is that it doesnt require a password of any kind, so its essentially worthless. Ubuntu challenges any admin action with a prompt for a password, as a virus or malicious program obviously wont have this password. *nix in general is based more on user rights and has a better control in hand for preventing unwanted actions, where Windows tends to be succeptible to the chronic "single user on an admin account."

If you are familiar with Windows security at all, youll know a golden rule of security is to have an individual account for each user including yourself, and have a seperate account set up for admin actions. The main problem is that people tend to run the admin account all the time, and as a result, any virus/malicious program etc is free to run on admin authorization. With Ubuntu, you are always a user, with your admin rights locked away. When you attempt to perform an admin action, the OS prompts you for the password, essentially allowing you to temporarily login as an admin to perform your task. This way, if a program attempts to do something that could affect the system, it will either prompt you for a password, or be denied entirely.

It may be a bother at first, but after a while and a little thought, youll appreciate what its giving you in return...

This is assuming you're using Gnome, and not KDE.

I'm just wondering whether you're referring to the regular workspaces, or the cube ones (with compiz fusion), so I'll explain both :D

Under the Heading "General", at the top, click the General Options button, then go to the "Desktop Size" tab. Move the slider over for the "horizontal virtual size" to as many as you want (although I'd limit it to around 4, any more than that and my eyes tend to cross.. lol).

### Post by Grey Box on 2007-11-27
Hi LordSaladin and welcome to the family. Differences between Gnome and KDE are many, and they arise from a different philosophical approach to the user interface. But like anything, it's a matter of taste and style. As I understand it:

Gnome, which is the standard Ubuntu desktop interface, is aimed for simplicity and elegance. You rarely see any unnecessary buttons (eg: instead of frequently seeing a dialog box that says "Apply, Cancel, Close" for very mundane and reversible tasks, you just see "Close" ) and 'tweaks' and micromodifications are less easy to do. In my opinion, it seems that Gnome tries to do what OS X tries to do - keep things intuitive, simple, safe and familiar. I think it does this much better than OS X, which has lost its way a bit and gone down the show-off path.

KDE is standard on many other distributions, like SuSE. It's really something worth trying out so you can get the idea of the differences (you can easily install both KDE and Gnome on the same installation). In KDE you can pretty much adjust and configure anything and everything. For example, every workspace can have a different wallpaper, the taskbar and various add-ons are much more customizable. Pretty much everywhere you right-click in KDE you get a configuration menu of some kind. 

Many applications are written for KDE and Gnome, but the beauty of it is that you can use either in either. Gnome will run KDE applications (such as k3b) no differently from KDE. 

There are heaps of other desktop managers around, mostly more light-weight ones for older/smaller/slower computers, or for people who want to dedicate as much of their computer's horse-power to tasks other than running their user interface.
