### Post by cogadh on 2007-07-04
Ubuntu, like Windows, should "just work". So, if you aren't noticing much of a difference, then it is doing its job. However, there are significant differences between the two systems that are not apparent just from looking at the GUI.

Ubuntu (and Linux in general) is not subject to the security threats that Windows faces (viruses, worms, spyware, etc.)
Ubuntu is open source, meaning anyone with the necessary smarts can modify the code to suit their needs, Windows is closed source and cannot be modified
Ubuntu is free (both libre and gratis), Windows is not (libre or gratis)
Ubuntu (and Linux in general) encourages innovation based on global community standards, Windows requires strict adherence to arbitrary standards set by the MS corporate authority that discourage true innovation

I'll stop right there before I start one of my bitter angry rants against "the man". Suffice it to say, the major differences between the two systems outside of the actual code are mainly philosophical.

### Post by coolen on 2007-07-04
Argument for Ubuntu for the average user:
Ease of software4 management.
Highly configurable and more personal desktop.
Free of charge (even those you don't know about).
Free (and easy) to upgrade.
Better eye candy. Not stable for the moment, but undergoing furious development to get it there.

Argument for Ubuntu for the geek:
Source code readily available for almost all applications.
Secure from viruses and trojans.
Better filesystem management (meaning your disk doesn't get fragmented as easily).
Unix way of handling software (one application for every task, tied together by a frontend) means a smaller system, and more room on your hard drive for porn (this part is for geeks).
Debian way of handling software prevents dependancy issues.
A powerful and capable CLI means you can still use your computer in times of trouble (no system is 100%).

Argument for Ubuntu for the philosophical nerd:
Open source software encourages innovation and increasingly optomised code.
A highly configurable interface: pretty much everything can be themed without the need for extra software to dive into the guts of your system and start pulling strings.


These are all I can come up with for now. I'd suggest you use Ubuntu a while longer. I dual-boot, and unless I find myself in the mood to play Need for Speed in the ten seconds that Grub is up, I always come back here. Not because it's open-source, nor because it's free (although I don't discount these as positives once I'm here), but because I find it easier and more pleasant to use.
There's nothing saying you have to make a choice either way. You can continue to dual boot both systems for years, without issue, until Microsoft decides to discontinue support for your system, and you're left with the choice of either paying to upgrade, becoming vulnerable to millions of viruses (more), or ditching Windows for good.

### Post by cogadh on 2007-07-05
Wine is not an emulator, it is an open source implementation of the Windows API. As for alternatives, Wine, Cedega and CrossOver are the only ones. Both Cedega and CrossOver are based on Wine, but CrossOver is the actual financial backer and major developer of Wine, while Cedega just leeches off of the Wine source code and charges people for it.
