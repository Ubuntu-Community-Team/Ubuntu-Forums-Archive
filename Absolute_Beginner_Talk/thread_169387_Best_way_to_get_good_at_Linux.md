### Post by pbaehr on 2006-05-02
In my opinion the best way to learn Linux is to use it. Every time you have a new problem or task spend a few hours learning how to make it happen with your Linux setup.

Like you said, you dove in head first. Just keep diving. Don't expect an overall mastery overnight, but take things one task at a time. First, I'd look at the system I had and decide if there was anything I wanted to improve about it. Then I'd search around the message boards and google and start learning the specific things I need to know to do make that improvement. That's how I've learned everything so far.

Good luck!

### Post by jazzmuzik on 2006-05-02
Eat, drink and breath Linux. Create a cheat sheet text file and put notes of everything you've learned there. It's impossible to remember everything. Experiment. This is easier if you have a system on which you can screw up and reinstall if necessary. If you have more than one computer with a network card, make a little home network and learn about networking.

### Post by stinerman on 2006-05-02
Like others have said, just use it a lot.

Be sure to back up your data regularly.  If you're like me, you try to tinker with every .conf file to see what makes GNU/Linux tick.  Of course, that means you'll break a lot of things that can only be fixed with a full reinstall.  When I was trying to find a distro I liked, I was formatting/reinstalling several times a week.

You have to learn to be patient as well.  If you're really familiar with windows and know all the ins and outs it can be hard to deal with the fact that you don't know how to add a new application to the menu or change the default resolution.
