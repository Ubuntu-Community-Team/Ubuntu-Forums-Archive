I don't believe this. How can a client-side program identify spam at server level?? Of course it has to download and evaluate the messages first, but that process is hidden from you. Or, it can only evaluate message headers, but that wouldn't be a good idea.

Thunderbird provides good client-side spam filter, and you can use Gmail for high-quality server-side filtering.
