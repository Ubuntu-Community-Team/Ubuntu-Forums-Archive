### Post by Burr on 2005-09-14
I've been goign through the exact same thing for the past couple of days on a 9800 pro.  tried everything, then i ran across this.

[http://www.student.dtu.dk/~s971652/ati_radeon.shtml](http://www.student.dtu.dk/~s971652/ati_radeon.shtml)


```
ATI Radeon Mobility 9700 Pro Graphics Card

   1.

      Before you start you want to install the latest version of the linux kernel that matches your computer. If you're not sure which kernel you're currently using open a console and type:
      $ uname -a

      The kernel version number will appear. For example, if you have the kernel 2.6.10-5-686 you type:
      $ sudo apt-get install linux-686
   2.

      Install the fglrx driver by typing:
      $ sudo apt-get install xorg-driver-fglrx
   3.

      Load the module by typing:
      $ echo fglrx | sudo tee -a /etc/modules

      The above won't have any effect until you restart your computer. Before you do that you need to edit your xorg.conf file.
   4.

      Make a backup of the xorg.conf file:
      $ sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf_old

      Open the xorg.conf file:
      $ sudo gedit /etc/X11/xorg.conf

      and find the section containing:
      Section      "Device"
        Identifier "ATI Technologies, Inc. Radeon Mobility 9600/9700 M10/M11 (RV350 NP)"
        Driver     "ati"
        BusID      "PCI:1:0:0"
      EndSection

      Change this to:
      Section      "Device"
        Identifier "ATI Technologies, Inc. Radeon Mobility 9600/9700 M10/M11 (RV350 NP)"
        Driver     "fglrx"
        BusID      "PCI:1:0:0"
      EndSection

      Please notice that the text in your xorg.conf file might differ slightly. Reboot the computer and test the driver by typing $ fglrxinfo in a console. This should give you something like this:
      $ fglrxinfo

      display: :0.0 screen: 0
      OpenGL vendor string: ATI Technologies Inc.
      OpenGL renderer string: MOBILITY RADEON 9700 Generic
      OpenGL version string: 1.3.4769 (X4.3.0-8.8.25)

      You can also test the installation by typing $ glxgears which should give you around 1300-1400 FPS.

      If for some reason the X-server won't start copy /etc/X11/xorg.conf_old to /etc/X11/xorg.conf and type $ startx. Look at your settings again. 
```


this worked like a charm for me, had to reload to git rid of all the other stuff i had screwed up in previous attempts to get the driver working, but it works perfect now.

### Post by Jivicin on 2005-09-16
Ok... after days of wrestling with this and doing every different method under the sun I finally figured it out:

Apparently the nvidia-glx package was installed on my system. Everytime I tried to install the xorg-driver-fglrx it was erroring out saying some libraries it was going to overwrite conflicted with nvidia-glx.
