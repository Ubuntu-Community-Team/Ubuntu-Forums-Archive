```
cd /path/to/folder
chmod +x armyops250linux.run

```

If its an installer, it might complain about permissions so you might need to add "sudo" in front of that last command.

### Post by igknighted on 2007-04-17
> **RiPPeR777 said:**
> oh nad what does the "." before ./armyops250linus do??

Basically you are providing a path to the binary file.  You can list that path with an absolute path (starting from /, so if the file was on your desktop the absolute path would be /home/username/Desktop/file.run) or relative to the directory you are in (in the case above, assume we are in /home/username - this is where the terminal opens to be default - the command would be Desktop/file.run).  If you were in the Desktop directory, you need to use ./file.run because and command by itself means to use a command from /usr/bin or /usr/local/bin, and the system needs to be able to differentiate.

As far as these commands, as with learning any new OS there are a lot, but they are very ordered and tend to make a lot of sense, it just takes experience.
