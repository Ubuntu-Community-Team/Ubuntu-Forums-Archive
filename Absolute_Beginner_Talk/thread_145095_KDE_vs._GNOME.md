### Post by evilgold on 2006-03-15
the differance is in the look and feel. Gnome = Simple...REALLY SIMPLE. If you cant figure out how to do somthing on a gnome program, your retarded (or the program just cant do it). Gnome specifically stays away from advanced things that could confuse users, a good example of this is the browser epihany, which has options to accept or reject all cookies, but not an option to ask everytime, which EVERY other browser i can think of has. Gnome sticks to the basics with most gui's. if an "Average" user doesnt need it, they dont have it, or its hidden in a .conifg file.

KDE on the other hand is a more advanced simple. It has features that you have no idea what they are for, but it also has the basics. 

I'm not trying to bash gnome by the way, i prefer it as my main UI for most things. Advanced users only need a command line anyways. 


The way i see it, gnome is like Aqua (macOSX) and KDE is like Luna (WindowsXP) (stricktly speaking about the respective GUI's not the OS's). The flame war would probably be similar.

### Post by aggiechemist on 2006-03-15
Go and log out.

When get to the login screen, type the user name and password. But BEFORE you press enter for the password, click on the button marked "sessions." There should now be an option for KDE, as well as Gnome, last, default, etc. 

Choose KDE and give it a try. You can always log out and go back to Gnome.

Oh yes, and I find it much easier to hit Ctrl+alt+bkspace than properly logging out. Just my preference.
