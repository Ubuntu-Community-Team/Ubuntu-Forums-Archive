### Post by Paul133 on 2007-10-08
Yes, it's an operating system. Specifically, it's a distribution of Linux, also known as GNU/Linux. It's very similar to UNIX and OSX, but it allows a lot of customization and it';s very powerful. It has a a fairly substantial learning curve, though. You can dual-boot it with Windows (what I assume you're using now) or you can just run Ubuntu. Ubuntu is open source, which means all the source code is available for you to see. Ubuntu is also free as in beer, that is, it doesn't cost anything, unlike Windows.

Most programs for Ubuntu are free, and most are found in the repositories, big servers thaty store Linux programs. In order to install a program, all you have to do is have the computer download one from a repository, then you're good to go.

In order to use Ubuntu, you need to download it and burn it a a CD, and then boot from that CD. I'd recommend you try it; it's pretty fun to use, and it's free. Good luck!

### Post by Nekiruhs on 2007-10-08
Ubuntu is a Linux based OS that uses GNOME for its Desktop Environment.

Linux is an operating system kernel (Just think like a corn kernel, the heart), based off Minix (A spin off of Unix, another kernel). Linus Torvalds invented it. The kernel basically "talks" to your hardware.

GNOME is a desktop environment. A desktop environment is everything you see after you login. GNOME is highly customizeable from a style stand point, and offers all of the features the average user needs. Unfortunately, recently GNOME has had the mentality that functionality is scary and remove some complicated configurations (Individual screen saver options come to mind, not no screen saver, just no really tight control of an individual screen saver), and Linus recommends people at least try KDE, an alternate DE. KDE is HIGHLY configurable, almost the exact opposite of GNOME. Some say KDE is too configurable and cluttered, but that may not be the case for you. 

Ubuntu, and most Linux based oses, does certain things differently  from Windows. For example, there is no C drive. All the filesystem is organized on / . Applications get installed into /usr/bin and system configuration files go in /etc . All your files go in /home/USERNAME . 

You aren't immediately able to make any changes to anything outside your own home folder. You don't have proper permissions on your user to do so. In order to make changes in those folders you must use the "sudo" or "gksudo" command. To do that press Alt+F2 and type "gksudo nautilus". A dialog box will then popup and you will enter your password. gksudo is the escalation command to enable you to make changes outside your normal privileges. nautilus is the command used to start the filemanager. There by it starts nautilus with total permissions. There is a reason for this madness. If an attacker were to gain control of your computer, they can t do anything drastic (Eg. Crash computer, install virus) without your password.  

Applications install differently too. Instead of searching the web for a program, you search the repositories. Repositories are giant servers containing software. Were talking 10,000s of pieces of software. Most programs you need will be included in the default repository, however, if you need more you can add a repository to be included. Sometimes, a repository isn't even necesary, as many programs come packaged as .debs . Think of them like .exe installers. There is a tool called Add/Remove in the main menu for installing from the repositories. Select the software you want (Games, office stuff, everything you could imagine, all free and legal, though not the programs you may be used to on Windows), press Apply, enter your password, and done.

Ubuntu comes fully functional out of the box. Web browser (Firefox), Media Player (Rhythmbox), Office Suite (OpenOffice.org), an Image editor (GIMP), a photo album program, a hand full of games, and more are all preinstalled.

Ubuntu is open-source. While this may not mean much to you, this is very important. This means that anyone can see the source code that makes Ubuntu tick. If someone wants to improve something, they can. Someone can read the code and find potential bugs and flaws, and then fix them right away. This also protects you in two ways. It means that if Canonical (The makers of Ubuntu) decide to do something you, or a group don't like, you can use the source code they give us and make your own project, more suited to you or your group. It also helps assure you that Ubuntu isn't doing anything behind your back, like Vista with DRM.  

One of the major issues you may have is hardware support. While a large majority of hardware is supported, some is not out of the box. A good way to check this is to burn the Ubuntu LiveCD, boot from it, and check if your hardware works. On the LiveCD you can try Ubuntu and see what its like before you even install it. 

Another issue is software support. If you want to use your Windows software in Ubuntu you will need a program like Wine, Cedega, or Crossover. Some but not all Windows programs are supported.

Info overload much? Don't worry ask me a question and I will answer it the best I can. But hey, instead of waiting around, why not try it? You can use the LiveCD and try it without even touching your hard drive. The only cost is your time! And just remember, you always have us here to help you!
