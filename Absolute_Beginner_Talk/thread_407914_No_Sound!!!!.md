Sorry it seems that I wasn't clear enough and you may have misunderstood. "blacklist" is actually a file, located in the directory /etc/modprobe.d. When you do "gksudo gedit /etc/modprobe.d/blacklist", it is supposed to bring up a password prompt, and once you enter your password it opens up gedit as root so that you can edit the blacklist file. The other two lines are entries you add to the bottom of that file, not commands that you run in the terminal. 

1. If gedit didn't work, try editing it using a terminal editor (such as pico or nano)
```
sudo nano /etc/modprobe.d/blacklist
```This should open up the file within the terminal and you can edit it from there.

2. Hit "Page down" until you get to the very bottom of the file, and copy and paste the following two lines:
```
# Blacklist Intel onboard sound
blacklist snd_intel8x0
```3. Once you've added the lines, hit CTRL+X to exit the file. You'll be prompted to save the file, so hit "y" and then "Enter" to keep the same file name.

4. At this point the file should be saved with the lines you just added. If you want to verify this, you can type
```
tail /etc/modprobe.d/blacklist
```which should show you the end of the file, and the lines you added should show up.

5. Reboot your computer, and the module should no longer load.

Try that and let me know how it works. If you need any other help let me know.
