### Post by tribaal on 2006-05-25
Debian is much more stable, but the technology it uses is much older. They focus on stability, so they do not use the latest packages and eyecandy. They only release versions every few years.

Ubuntu on the other side has frenetic release cycles. It's targeted mainly at desktop use, and so is more focused on the latest hardware support, eye-candy and bleeding edge applications.
Yes, that means Ubuntu is less stable than Debian. But if you're coming from windows, it will feel rock stable anyway :)

Hope this answers your concerns.

- trib'

### Post by matthew on 2006-05-25
They are very similar. This link explains the relationship between the two pretty well.

[http://www.netsplit.com/blog/work/canonical/ubuntu_and_debian.html]("http://www.netsplit.com/blog/work/canonical/ubuntu_and_debian.html")

From a practical standpoint, Ubuntu is generally easier to install for people with little to no Linux experience and who don't know (or want to know) how to get in and configure things manually. From a base installation Ubuntu is a bit more polished for the general desktop user.

On the other hand, Debian is a great distro, the base on which Ubuntu is built, it has a huge group of very talented developers, and is an excellent choice as well. (Plus, I like playing with Debian Unstable and breaking stuff...)

Final thought: Ubuntu has something that no other distro has--these forums. The Gentoo forums are excellent, probably better from a technical standpoint, but not as generally accepting of beginners as these are. You won't find a better place for someone starting out to get help for any Linux distro. (EDIT: I admit to being biased...)

### Post by tribaal on 2006-05-25
In short, yes.

The long answer: It all depends what language you program in. There are some very portable languages (Python, Java, PHP...) that will behave exactly the same on pretty much all distros (and even on other OSses like Windows or MacOS).
They are very powerful languages, and I can't think of anything you cannot do with Python on top of my head (this includes OpenGL programming, distributed computing, networking and all that).

On the other hand, you might be tempted by more machine dependant languages like C, C++. Then you should investigate to make sure the libraries you use in your program are available for your target OSes. There are so many libraries around I cannot be sure ALL libraries that come with Ubuntu do with Debian as well, or the other way around... But for most it should be alright.

You might need to recompile your source code to move from one OS to the other however.

So it all depends what it is you want to program, basically. 
I program in Python and Java mostly, and this far I have never been in a situation where I couldn't solve all problems I was facing with theses two languages.

Hope this help :)

- trib'
