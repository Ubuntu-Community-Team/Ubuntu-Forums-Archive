### Post by kregg on 2007-02-04
First thing, don't panic! It can be quite a tough ride, but I found it quite easy by getting a Ubuntu disc from [Shipit](https://shipit.ubuntu.com), a free service which gives you free Ubuntu discs (hooray!) and I also got Kubuntu to

1) Get the feel of two different desktop environments
2) See which one I liked better

Ubuntu and Kubuntu have different desktop environments called Gnome (for Ubuntu) and KDE (for Kubuntu). Whichever desktop environment is up to you ([as you can see from this site](http://www.psychocats.net/essays/kdevsgnome)). 

Then it is a matter of choosing the linux distributions. As I said before I chose Ubuntu and Kubuntu, but I tried others as well (Linspire, Mandriva Linux, and lots more that I have forgotten now). On the website of the distribution you want to try out, there should be a download link for **iso's**. They are basically a whole CD as one file (or two with some linux distributions). You download this (preferably by Bittorrent as this puts less workload on the server) and burn it onto disc using [CDBurnerXP](http://www.cdburnerxp.se/download.php) (check if it works with Vista first!!!). Then put the disc into the tray and install!

As a tip, I'd advise you get a Linux Live CD just to "test drive" the OS.

### Post by mk04 on 2007-02-04
Since you upgraded to Vista, this probably mean you have a mid-high performance computer. Please post your computer specs. Try both Kubuntu (kde) and Ubuntu (gnome). See which one you like. First you have to download the ISO and burn it to a cd. 

Sites to download the iso: (download to the desktop)

[http://www.ubuntu.com/products/GetUbuntu/download?action=show&redirect=download](http://www.ubuntu.com/products/GetUbuntu/download?action=show&redirect=download)
[http://www.kubuntu.org/download.php](http://www.kubuntu.org/download.php)

Instructions to burn the iso to cd:

[https://help.ubuntu.com/community/BurningIsoHowto](https://help.ubuntu.com/community/BurningIsoHowto)

Be sure to check the md5sum before burning the iso to the cd, in a terminal run this command:

cd Desktop
md5sum #name of the iso file

Compare the output to the numbers on this site: [https://help.ubuntu.com/community/UbuntuHashes](https://help.ubuntu.com/community/UbuntuHashes)

Insert the LiveCD, this cd will run Ubuntu without installing anything on your hard drive. If you decide you want to dual boot Vista and (k)Ubuntu, click the install button on the desktop. Remember to BACKUP everything on your hard drive. Follow the step by step instruction. You will need to partition your hard drive to install ubuntu.

This is a helpful site to follow with many visuals: [http://www.linuxdevcenter.com/pub/a/linux/2006/05/08/dual-boot-laptop.html?page=1](http://www.linuxdevcenter.com/pub/a/linux/2006/05/08/dual-boot-laptop.html?page=1)

You will need to create a EXT3 partition (root) and a Linux SWAP partition (a good rule of thumb is twice your RAM). This size of your root partition depends on you. I would recommend about 25 GB or more. 

Hope this helped. :)

### Post by hmLyons on 2007-02-04
> **missfionfung said:**
> 
i hear rumors that linux doesn't work with some[alot] of softwares?
like winamp and such ...
do you use diff. programes ?

Linux only runs software written for linux (sort of, see below). Sometimes software is written for linux and windows like Firefox, OpenOffice and Gaim. 

For almost every windows program there is a linux program which does pretty much the same thing. Have a look [here]("http://www.linuxrsp.ru/win-lin-soft/table-eng.html") and [here]("http://wiki.linuxquestions.org/wiki/Linux_software_equivalent_to_Windows_software") for windows-linux equivalents. In the case of Winamp, there is a linux program called XMMS that is very similar. There are also dozens of other good quality audio players for linux.

I should mention that there are some different ways that linux can actually run windows programs and games using WINE, Cegada, CrossoverOffice or Vmware. But each of these have their own set of drawbacks.
