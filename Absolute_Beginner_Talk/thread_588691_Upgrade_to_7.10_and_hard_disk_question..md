### Post by nick_h on 2007-10-24
You will need to unmount your root partition before you run e2fsck on it.  The easiest way to do this is from a Linux live CD.  At the moment, because /etc/mtab cannot be read, it does not know that the partition is mounted.

If you need to run e2fsck with the -b option then check the number for the backup superblock first.  To do this you will need to check the block size with:
```
sudo tune2fs -l /dev/hdb1
```
and then consult the e2fsck man page with:
```
man e2fsck
```

You can always force a check on boot by:
```
sudo tune2fs -C 100 /dev/hdb1
```
or a suitably big number.

There is also a way to force a check by putting a file somewhere - but I forget this method - perhaps someone else could remind me.
