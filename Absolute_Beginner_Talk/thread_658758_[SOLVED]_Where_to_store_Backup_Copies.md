### Post by DataMonSTAR on 2008-01-05
Is there a preferred place to store backup copies of files that I make? I edited /etc/apt/sources.list and made a copy of it in the same directory as the current one. Is there any problem with keeping my backups renamed in the directory they are supposed to reside in? Are there any directories where it wouldn't be safe or wise to do this? Thanks in advance, guys.

### Post by nitro_n2o on 2008-01-05
No, that is file you can keep files where ever you want as long as you change the file name. Like /etc/apt/sources.list becomes /etc/apt/sources.list.Jan.05.backup or anything that reminds you of the date. 

Or maybe you can just copy everything to a thumb/flash drive. But, keeping things in the same directory helps so much when you break something and you need to fix it quickly 

Good luck :)

### Post by Rocket2DMn on 2008-01-05
For configuration files, the location of the original file is fine.  A common practice is to name the file (say, sources.list) like sources.list.backup or sources.list.bak
If you are backing up DATA (not configurations), you should use a secondary device like a USB hard drive or a USB flash drive.
