### Post by atoponce on 2005-12-03
[quote=thinhlegolas]Windows is well-known for its ease of use since everything is graphical, all you do is just click and click and click.

{snip}

Ubuntu is not as easy to use as Windows since you need to use command line quite often.[/quote] 
The problem with Windows is the GUI.  You don't have the flexibility and power that you do with a terminal.  As you said, you have to click, click, click.  But what if there isn't a software application or tool that you need to get the job done?  Do you hire a contractor to build the app for you?  Do you code it on your own?  Is there a way to accomplish the task you need without the software?  Microsoft realizes this, and as such are releasing a UNIX-like terminal in Vista to help with these questions.

The terminal is here to stay.  Linux isn't about to give up the terminal any time soon, although there are a number of graphical representations of pretty much any command that can be typed in the terminal.  There are a couple of reasons why the terminal isn't going anywhere:

1. Servers need only to serve web pages, intranet applications and the such, generally.  Because of this, there is no need to run a desktop with GUI applications.  Simply pull up the terminal and administer what needs to be done.  This greatly reduces overhead, and devotes more memory and resources to serving the applications that need to be served.

2. Features are loaded into every terminal that comes shipped with most Linux distributions.  This includes a variety of shells, programming languages, scripts and much more.  In many cases, it is so much easier to write a simple Bash or Perl script, let's say, to automate a certain task, then it is to try and find a program that will provide the same results.  Of course, this depends on the complexity of the task you want accomplished.

3.  Interoperability with the major UNIX vendors.  It is all too easy to move from a Linux system to a UNIX system when using the terminal.  Almost all of the familiar applications are there with the same functionality.  Sure, there a few differences, but it is relatively easy to find out what they are, and how to work with them.

Although these reasons for using the terminal are mainly for system admins and the such, Linux is targeting that market much more heavily than the average user.  We are just geeks to like the OS, and prefer to use it, but the money is made through the commercial distributions, packages, software and support.  For these reasons, the terminal is here to stay, which is a good thing, as it is very powerful.
