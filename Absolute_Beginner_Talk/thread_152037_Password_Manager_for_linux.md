### Post by msak007 on 2006-07-01
I've been looking for a RoboForm replacement as well ever since I stopped using Windows. It was the hardest application for me to leave behind. I've been using PwManager since then, because it was the best one I could find for my needs (for KDE). It sits in the tray, and I can click on the icon to bring up a menu with all my stored IDs / passwords and copy / paste them into forms. It's pretty easy to use, but it doesn't match automated ease of use of RoboForm though...

Smile, thanks for the script recommendations, it helps somewhat with the auto-login. However, it's not really a replacement for RoboForm. RoboForm is so much more than just a password manager (form filler as well) and I haven't found anything quite like it for Linux. Firefox's password manager suffices for simple use, but one of the major problems is if you have multiple logins for the same web site. For example...if you have multiple credit cards from the same issuer / bank and need to log into all of them to check your accounts, it doesn't let you store more than one login / password. RoboForm would pop up a box asking you which one you wanted to log in with first, then fill the forms & submit them. Plus, I would really prefer to keep my passwords in a separate, more secure file in an external app away from the browser and have an extension grab the data, just like the RoboForm / RoboForm Toolbar combo. Is there really not enough demand for an app like this in Linux that one hasn't been developed yet? For some reason the developer of RoboForm vehemently refuses to develop the product for Linux so we're stuck with sub-par password management...
