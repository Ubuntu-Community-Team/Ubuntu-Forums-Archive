Woah! Easy there tigers!

Lets not get ahead of ourselves. Should we really be recommending to a new user that they completely ditch XP and install Ubuntu over the entire HDD?

So what happens when (not if, and we all know it since who uses Linux and can resist the temptation to tweak - especially if the advice is 'install Beryl' - go on, edit your xorg.conf file within 10 minutes of uising Ubuntu, and see if it breaks.....!) - something goes wrong? Oops. But that's ok, since new user always has XP to fall back on so they can get to the forum to ask a question as to how to fix it....er, now except they can't because they needlessly deleted it!

I'm not going to try and tell anyone what to do, but if there's no need to use the entire HDD then why do it? Set up a dual boot system. Spend a few months learning your way around Ubuntu. And then destroy XP and enjoy the feeling.

I'm not sticking up for MS. I hate them as much as the next guy/gal, but lets not go encouraging someone to do something they might regret, and which might, as a result, wind up deterring them from using Ubuntu.

NB: To dual-boot - first defragment XP and run a disc-scan. Then (once you've run the 'check disc' option in the LiveCD menu) - use the Live CD (System/Administration/Gparted) to resize XP (ie shrink it) then double click install option on desktop - select 'use largest available free space' option - and it will do the rest for you). 

Good luck,

### Post by floke on 2007-02-27
And always make backup files of things before you change them

sudo cp [what] [to what]

eg. to backup your xorg.conf file

sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf_backup

if you muck it up just reverse the above...

sudo cp /etc/X11/xorg.conf_backup /etc/X11/xorg.conf

Important things to backup are your fstab (/etc/fstab) and grub (/etc/boot/grub/menu.lst)

I learned this the hard way :mad:

### Post by dannyboy79 on 2007-02-27
also your /etc/network/interfaces file, basically any file that you edit, I would always just first make a backup copy. if you don't like using the command line, you can always double click a file within nautilus or konquror (or however it's spelled in kubuntu) or in thunar (this is in xubuntu) than simply do a save-as, and just put a _backup or -backup or .backup after the filename. In linux the dot doesn't mean anything like it does in windows so you don't have to worry about that. good luck. what I did was make a text file and list all the files that I have edited to fit my needs, I included a general summary of what I did to these file. this way, if I ever go to another distro or whatever, and it's based on debian, I know right where to go and what do to. or you can simply make a folder in your home directory, and take a copy of the files you'll edit and put them in this folder for safe keeping. some of the files that I have listed are:
 
1. /etc/shells
added /bin/false to the end of the file

2. /etc/fstab
added partitions and mount points for my extra hard drives. added mapped drives for ftp server

3. /etc/samba/smb.conf
messed around with it to get samba to work!

4. /etc/hosts
add ip and netbios name for all computers

5. /etc/apt/sources.list
added websites for more software choices

6. /etc/proftpd.conf
set it up to have a user be chrooted into his home directory then mapped folders to this home directory for global access.

7. /etc/ssh/sshd_config
updated so that key authentification was used instead of password. set it up to allow challengeresponseauthentification so that nxserver would work with pam user database instead of using nx user. 

8. /etc/network/interfaces
remove extra interfaces that aren't needed. ubuntu had eth1 and eth2, ath0, wlan0 and since I am on a desktop I only need eth0. Note: not sure if this can be done thru the network gui.

9. /etc/hdparm.conf
update so that optical drives and hard drives are running dma for better performace. Note: some drives this can actually cause data loss I have heard if they don't support it. you can run sudo hdparm -Ii /dev/hdx to see what the drive is capable of.

10. there get to be plenty more as you install software. like modifying server.cfg and node.cfg for my remote desktop application. it's more secure than rdp because it tunnels thru ssh. change files for tripwire, and many many other software config files.

so I guess the moral is, save a backup copy first! then modify all you want, something goes wrong, you can always replace your modified one with the backup and you should be golden after restarting the software or service. ie: sudo /etc/init.d/servicenamehere restart. Example: sudo /etc/init.d/samba restart

good luck and we are here to help ya.
