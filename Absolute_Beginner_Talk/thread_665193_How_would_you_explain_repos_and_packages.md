### Post by SunnyRabbiera on 2008-01-12
well the repositories are where all the core installer files for your os.
Unlike windows most linux flavors work off a set of repositories created by the maintainers of that said linux distribution.
For example the packages on ubuntu are kept updated by ubuntu's maintainers, mostly by the folks at canonical ubuntu's parent.
Each repository has a set of independent developers who work together for a common goal, keeping the system up and running.
As opposed to Microsoft packages that are scattered all about the net and could contain malicious material the packages provided by ubuntu are as safe as possible.
now of course you cant find everything in the ubuntu repository, you might have to look around a bit to find a few things and add extra repositories.
but keep in mind it is a good bet to look at the track record of those extra repositories out there, but a good amount of them are reliable.
Now as for packages, well ubuntu's packages are similar to windows executables.
each package provides you with the desired application you need/want.
Now of course some packages might need other packages to run, this is why its a good idea to stick to the regular packages provided by ubuntu.
Now please note if your parents are after certain windows apps they might not have much luck but there are ways to get at least some windows apps working in linux.
the main way is to use wine, but a dual boot might be more desirable if they are into gaming.

As a Library is a Repository of books, in Linux a Repository holds software packages.

You can add multiple repositories (libraries) to your system so when you go looking for a software package, there are more places it could be found - you can even access different versions of the same package if they exist in your selected repositories.

In Windows there are no "libraries", just thousands upon thousands of software packages that your have to find yourself.
