### Post by stylofone on 2007-05-24
I came to Ubuntu expecting to have to work through problems and learn new things. Most of the basic everyday computing tasks have been easy, the more unusual things I like to do have been a bit harder, and I've happily worked through the issues. It's been great BUT....

I found one very basic problem when backing up data using DVD burning software. It wouldn't recognise the character set on some of the files I wanted to burn. K3B popped up with a message suggesting I use convmv to fix them, Gnomebaker was somewhat less helpful.

When I tracked down convmv I was a bit perplexed. It seemed to require knowledge of the character set used to create the original files... but I didn't know if that would be determined by the OS I previously used (Windows) or the OS of the person who created them (they were downloaded from various sources). In addition to that, convmv's documentation left a lot to be desired.

In the end I muddled through using a combination of convmv (I worked it out) and manual renaming, and all my files made it to DVD. 

I am pretty adept at problem solving. But the reason this annoyed me is that it was a pretty basic task, backing up data, which many computer muggles might want to attempt. Such basic tasks should not require a research mission, and implementation of a relatively obscure command line program. It kind of shook my faith in Ubuntu. I still love using it myself, but I'm not sure I could recommend it to others less geeky than me if they're going to come up against such problems. 

Or did I miss something? Is there an easier way to get around this issue? Or is it not a common problem?
