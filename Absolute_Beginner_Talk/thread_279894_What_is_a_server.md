### Post by ciscosurfer on 2006-10-18
In a simplistic definition, a server "serves" out content.  Like a mail server that handles emails, a print server that handles print and print-related activities, a Web server that holds Web pages and the various code that generates these and serves them out to multiply-connected machines (i.e., Internet users or users on your own intranet.)

The Linux use of the word server is a bit different, however.  While all of the above still holds true, many programs in the Linux world can be seen as servers of sorts.  For example, the Xorg server controlls your X session and the processes that spawn from it.

Unless you have a need or desire to host an email, Web, or other server of sorts, you can stick very happily with U/K/Edu/buntu in its original glory.  You can always add these types of services on top of your Desktop configuration, but that's not usually recommended for various reasons like security, for example.  If you just want to mess around with Apache or PHP or the like, you can install them and learn.  But if you want a truly production-ready machine, you should definitely invest or use a separate machine for your server needs.

Wikipedia is nice, but not always accurate. 8)

### Post by lambophil on 2006-10-18
A server is a computer, basically. It usually acts as a central computer to a network where it manages network traffic, acts as a proxy (a proxy server is a computer that accesses the internet for you, and usually caches web pages (stores pages for quicker connections)) and stuff like that. Its not worth having a server for basic internet use- you could use it for homemade web pages etc though.
