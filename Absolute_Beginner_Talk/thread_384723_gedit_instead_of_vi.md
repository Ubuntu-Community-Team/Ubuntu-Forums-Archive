For the record - old time vimmer here.

Simply put, the learning curve on "vi" (vim, elvis, whatever) is too steep for the average newbie.  Extremely powerful, yes, but it takes a bit of getting used to.

For the average newbie, I recommend either "gedit" (easy to learn GUI program), or "nano", (easy to learn terminal program).

Now for anyone who want to be a developer, I *strongly* recommend learning a more powerful editor such as "vi" (or - GASP - "emacs").  That's where the real power of the editor comes in handy (with syntax highlighting being just the tip of the iceberg).

### Post by skale on 2007-03-14
IMO, it makes no difference what one you use.  Sure, vi is faster, but a jack-butt to learn.  And you don't save that much time.  I'm not downplaying vi, I use Vim for pretty much everything, but there is no such thing IMO as "more powerful", it just does many common tasks much faster than your average GUI editor.  If you plan on editing a lot of text files, you might want to go ahead and learn it.  If you only do it when forced to, don't bother.   At the same time, it won't scar anyone for life, as you only need to know 'i' for insert, and :sav, :w, :q to pass.  And, vitutor covers all the commands there are quite effectively, if you want to learn more (which you should).

My two cents.
