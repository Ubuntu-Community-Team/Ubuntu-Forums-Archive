fsck.ext3: No such file or directory while trying to open /dev/hdd1
/dev/hdd1:
The superblock could not be read or does not describle a corect ext2 filesystem. If the device is valid and is really contains an ext2 filesystem (and not swap or ufs or something else), then the superblock is corrupt, and you might try running e2fsck with an alternate superblock: e2fsck -b 8193
