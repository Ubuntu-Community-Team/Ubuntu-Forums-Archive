### Post by SilentCacophony on 2005-10-25
I've tried a few, and many of them really shine only in certain areas. I like:

**rox-filer** is my favorite. It supports most of the options that I want, and has a lot of generally nifty features. Kind of medium-wieght, IMO, but still my favorite.

**mc** (good ol' midnight commander) for use from the console, and because I like two-pane listings

**worker** (clone of Directory Opus 4.X for Amiga) because of it's high geek-factor. It's definately not gtk, but it's highly configurable, and has two-pane listings as well. Really good for bash scripters.

**gentoo** and **emelfm** are both nice, light-weight filemanagers, also two-pane (i think)

There are a number of other file managers out there that are more or less like windows explorer as well, if you like that. Many are in the repos for ubuntu. Here are a couple of links with some file manager info:

[http://www.linux.org/apps/all/System/File_Managers.html](http://www.linux.org/apps/all/System/File_Managers.html)
[http://applications.linux.com/article.pl?sid=05/02/23/2226202&tid=13&tid=49](http://applications.linux.com/article.pl?sid=05/02/23/2226202&tid=13&tid=49)

I don't know the one you're referring to from Corel Linux, but I seem to remember reading about it once, and I believe that it was written by Corel, themselves. It's probably not in any repository, if that's the case, so thier site may be the only way to get it.
