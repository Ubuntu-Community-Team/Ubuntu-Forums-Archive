### Post by Tomosaur on 2007-08-05
> **EvilRibbet said:**
> 
1: What is the functionalty of linux, meaning why doesn't linux support EXE files and what kind of files is different programs written in linux?
It just doesn't :P Why don't cars have wings? Why can't you put diesel in a petrol engine? It's just a different way of doing things. On Linux, we generally don't care much about file extensions. Program files are typically just called whatever the program is called. For example, Firefox on Windows is 'Firefox.exe'. On Linux, it's just 'Firefox'. File extensions are for computers, not people :P However, data files, like pictures or music files, will have a file extension, such as .ogg, .odt, and so on.

> 2: I'm a major gamer i love all mmorpg's , is there something that takes to my need for gaming on linux i have checked out [http://www.transgaming.com/](http://www.transgaming.com/) like it plays windows games or something?Cedega will play lots of Windows games on Linux, but not all, and sometimes there are things wrong. If you absolutely need Windows games, then you should just stick with Windows. Many games have a Linux version too, but **most** big Windows games do not. There are many reasons for this, mostly to do with market share and/or Microsoft's dirty tactics. However, don't be put off - Linux has many great games available. They're generally not graphically comparable to Windows games, but they're still fun, and most of them are free.

> 3: I'm somewhat confused about is linux that secure no anti virus needed?  If so is avalibility out there for it?
Linux has no known virus threats. Linux by design is inherently more secure than Windows, and it is all but impossible for a virus to get into your system unless you put it there yourself. There ARE anti-virus programs for Linux, but most of those are simply to scan Windows machines. A Linux user can still act as a 'bridge' between Windows machines for a virus to cross, but the virus will almost definitely not infect the Linux machine. Short answer is you don't need to worry about it - just don't be an idiot and install random stuff from random people :P

> 4: If i switched to Ubuntu , could i still use torrent software/cd/dvd writing programs/video editors/music editors/WINAMP :D/Paint shop pro and/or Adobe photoshop/DivX/or writing in VB or C++?
Linux has plenty of torrent software. You can write / play CDs and DVDs. Video editing I'm not too sure about, I've never tried on Linux. For Music editing we have lots of software, but it's probably radically different from what you're used to (it's an entire subject in itself, well worth researching - search for 'Linux Music Production' in Google or something). Winamp is not available on Linux, but we have a clone called Audacious. Paint Shop Pro / Photoshop - no, those aren't available on Linux. We do have The GIMP though, which is a fully featured photo/image program. It is quite different from Photoshop though, and will take some getting used to. DivX - not too sure about this, but I think it's possible. Visual Basic - why? VB is for Windows programs basically. There's probably some variant for Linux out there, but I don't know myself. If you're into C#, you should check out the Mono project. As for C++, yes you can program in it. You will need to install the 'build-essential' package after installing Ubuntu.

> 5: Does ubuntu detect all hardware running or do drivers need to be installed for certain things? Many people have a hard time with this. Linux detects more hardware out of the box than Windows. This means that if your hardware is plugged in, Linux should be able to use it fully without you needing to do anything. HOWEVER - this doesn't mean that your hardware will definitely work on Linux straight away. Some things DO need drivers installed. The only way to find out is to use the LiveCD and find out.

> 6: Can you still auto load cd roms like games/music/software?Yes, but on Linux, you generally won't find a game on a CD in any shops, and we don't install software from CDs. We use 'repositories', which are web-based catalogues of software. We search for software by category or name, select which software we want, then click Apply. The software is then downloaded and automatically installed in a matter of seconds. No CDs, no dodgy websites, and best of all, it's all free! In the near future there should be a system called 'CNR' in place which will allow you to download and install commercial software. It's kind of similar to Steam on Windows.

> 7: Are there required plug-ins for different things?Do you mean like Flash and stuff? If so, then yes, there are plugins for Firefox available.

> 8: My main goal is to get to know ubuntu because i REALLY want to switch and handling exe files or different types of software?Is this a question? The only thing you can do is just try it out - download the LiveCD and boot from it. On the surface it's pretty similar to Windows - point, click, surf the web. Under the hood, though, it's radically different, and will take you some getting used to.

> 9: I know GAIM is installed on ubuntu very nice :D but what if you wanted new messengers or seperate ones?No problem, just go to Applications > Add/Remove, search for 'instant messenger', and pick the program you want.

> 10: What about cleaning up files on Ubuntu or any defragment or whatever just like windows?Don't worry about that. The package manager takes care of installing / uninstalling software and generally keeping your software in check. Linux generally doesn't need defragmenting because we use a different filesystem from Windows. It's faster and generally better. If, however, your hard drive is almost completely full, then the system may be forced to start fragmenting files. In such cases, there is defragmentation software available, but most people never need to install or use them.
