### Post by chopsuwe on 2008-01-03
I had the same problem and got around it by changing the Horizsync and Vertrefresh to numbers much higher than the monitor can go. I used 30-120 and 50-120. So what you have below should be fine. 

```

Section "Monitor"
        Identifier      "Tyypillinen näyttö"
        Vendorname      "LG Electronics Inc."
        Modelname       "LG Studioworks28i"
        Horizsync       30.0-85.0 ----- <- change to 30.0-120.0
        Vertrefresh     50.0-120.0 ------ <- change to 50.0-120.0

```

That made no difference untill I changed Defaultdepth to 16. Then it would let me go to 800*600 @ 85Hz. Then I changed it back up to 24 and it will now let me use 1024*768 @ 85Hz.

I guess it is like Windows, it doesn't bother to check if things have changed untill you force it to.

```

Section "screen" #   
        Identifier      "screen1"
        Device          "device1"
        Defaultdepth    24 ---- <-change to 16, restart, change back to 24
        Monitor         "monitor1"
        SubSection "Display"
                Depth   24
                Modes           "640x480@60"
        EndSubSection

```
