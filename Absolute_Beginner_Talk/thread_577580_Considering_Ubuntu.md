### Post by 0ttoNo5sun on 2007-10-16
I've used SuSE since 7.0 and am now on 10.1 (KDE). Most of my work is done in Linux though I have to use Windows from time to time.

My present PC is a Dell Precision 690 with one Xeon 5140 (dual-core 2.33GHz, 1333MHz, 4MB), nVIDIA Quadro FX550 128MB (I'm no gamer!), 2 SATA HDDs (non-RAID). Dell's installation of WinXPPro is on HDD1 and, on HDD2, I have SuSE 10.1 and my data. GRUB is on the MBR of HDD1 but /boot is on HDD2!

I've become disenchanted with openSUSE, as it's now called, because the home user is no longer considered important - producing the next commercial Novell Linux seems to be what matters. 10.1 had a messed-up package manager (I have to use Smart), 10.2 changed the way that USB was handled (breaking VMware Workstation), and - from what I can see in the forums - 10.3 has issues as well.

I'm looking for a *reliable* distro that I can use for my home and personal data and for such things as my photos. I *must* be able to run VMware Workstation (I have a WinXPPro VM that contains all my Windows programs.). Though I wouldn't mind flashy effects per se, I don't need them: ease of use, stability (reliability), and good quality are my basic requirements. Having used computers since the 70s, I know my way around but, age taking its toll, I'm not as tolerant of 'problems' as I used to be. :shock:

I've bought The Official Ubuntu Book and will read that extensively. I may even try the DVD (7.04) that was included - I'm concerned, though, because someone stated in these forums (fora?) that the 'live CD' had made a mess of his Windows partition. :(

Thank you for reading this and for any constructive advice that you send my way.

### Post by Dr Small on 2007-10-16
The livecd will only mess up the windows partition if you select to completely overwrite the entire hard drive. Otherwise, you should be fine to setup a dual boot. 

But always remember, BACKUP your data before installation, incase something does go wrong!

Dr Small
