### Post by dfreer on 2007-05-11
x86, i386, i686 all refer to 32-bit processors, with x86 refering to ALL intel/amd 32-bit processors, and I think i686 refers to the pentium 4 line specifically. x86_64 and x86 refers to ALL intel/amd 64-bit processors, as well as AMD64 (all intel dual core 2 processors are called AMD64, but the server 64-bit processors have their own name (I think EM64T)). There are also several other processor types, like those for Mac's I think are called PowerPC or PPC, and SPARC.

### Post by dptxp on 2007-05-11
Knowledge of Architecture is essential for writing codes in Assembly Level language, not for running or using Linux in any way.

Earlier, about 15 or more years back, quite a few programs including word processors, were written in assembly level language for code optimization and efficiency. With huge resources at hand, assembly language is rarely used, maybe in small parts in some programs.

You just have to know if your processor is 32 bit or 64 bit to load correct versions.
