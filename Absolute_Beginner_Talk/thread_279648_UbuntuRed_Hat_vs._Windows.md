OK, I'll chime in a bit since I admin both windows and linux servers.

There are many many many fundamental differences between windows and linux, many of which have to do with openness and cost.  Sometimes one is better, sometimes not.  It really depends on the situation.

Differences between Ubuntu and Red Hat:  Lots.  Ubuntu is based on an older distro called [debian]("http://www.debian.org"), and Red Hat isn't. 

Red Hat has a more mature support structure.  Basically, its easier to find paid support if you need it.

Red Hat has more support from proprietery software vendors, ie Oracle.

Ubuntu, IMHO, has a better package management system.  

There is no non-free part of ubuntu, whereas Red Hat is supposed to be paid for.

On the desktop, nothing beats ubuntu.  Nothing. I've used just about every major distro out there, and Ubuntu is the cleanest and least complicated.

Ubuntu is cooler ;) 


Preference over a windows platform:

Cost is an easy answer.

Lack of restrictions.  Linux vendors cant invalidate your license if you piss them off.

Linux adheres to standards much better than any product by microsoft.

Stability/security:  MS has gotten better here lately, but isn't anywhere near the stability of linux.  Linux has access to way more security tools than windows, and overall, they are very flexible.  This means that you can use them creativly to come up with a very robust security process.

Ease of use:  Windows will feel more "familiar" until you become used to linux, but once you get there, watch out!  Linux is consistant, fast, and easy to use from the cli.  Bash **rocks** dos all day long.  In linux, or any unix, you can get way more don from the cli than with a gui.  there are just more options available, and you can script them, combine them, pipe one output to anothers input better.  In short.  CLI linux rocks windows' gui/cli anyday.

You can run more servers per machine with less resources on linux than you can on windows.

Postgresql kicks ms sql's butt.


there are times when you'll need windows though:

Exchange.  Nothing matches it yet.  Horde and Hula are getting there and you can use them if your starting from scratch, but there is no "drop in" replacement in the same way that you can easily switch from IIS to apache.
Exchange sucks to admin, and hogs resources, but its the crack that most businesses are addicted to.  

Sharepoint:  I know of nothing out there that can match sharepoint on a feature-to-feature basis.  Come to think of it, I don't really know why folks use sharepoint to begin with, but maybe I'm just naive :) 


What makes one better than the other:

In my experience, linux is better than windows at each and every thing it does on the server end.  period.  Its more stable, more secure on average, runs on less hardware, more flexible, less costly (yes, even when looking at it from a "total cost of ownership" perspective), has a better support community, and gives **you** control of your servers.  Windows has exchange and sharepoint, but absent that, I can't think of a single reason to use windows in the server room.

### Post by donkyhotay on 2007-07-09
For me, I prefer linux over windows because I feel I have greater control over my computer. When things go wrong for me on my computer it is MUCH easier to fix on linux over windows because I (almost always) have access to the source code of the application that is causing me problems. While I personally am not incredibly skilled as a programmer and generally (though not always) unable to fix it myself I can almost always find someone online that can either tell me what lines to change (if it's small) or give me updated source (if it's large) to recompile and fix the problem. Under windows you generally have to wait for company that supports the software to provide a patch/update. Even non-source code problems are slightly easier (though maybe more intimidating). For example I know many people hate having to edit the xorg.conf file to fix video problems however when your video is REALLY messed up sometimes the console is the only thing you can safely access. While I know how to change video settings/drivers within the windows control panel I have no idea how to do this within the DOS prompt. Back when win95 was out I had a (for then) really nice graphics card that I simply COULD not install. Every time I put it in the system windows would automatically configure it for the wrong driver and it promptly crashed before I could get into the control panel to fix it. I would have loved to have been able to edit a text file somewhere that stated "driver = "some 3d card" before putting in the card but I was never able to find a way to do that under win95. Admittedly windows has improved quite a bit since win95 days however I still want my computer to do what *I* tell it to do, not what it wants to do.
