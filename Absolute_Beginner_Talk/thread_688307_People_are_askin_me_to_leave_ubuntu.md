### Post by emarkd on 2008-02-05
Some people don't like Ubuntu because they say it makes things too easy and hides parts of Linux underneath.  I've even heard it referred to as "Linux for Grannies".  I disagree with this.  While you will learn a lot running Debian (or lots of other distros) that you don't have to learn using Ubuntu, you still have the option to dig as deep as you want to.  I've been using Linux since Slackware 3 came out in 1995.  Sure, I learned a lot along the way that I can apply today, but you have the option to learn those things without being in a situation where you have to learn them to get your work done.

I guess it comes down to what you want.  If you want to really understand what's going on underneath all that eye-candy, you've got to dig down in there and find out.  Ubuntu doesn't stop you from doing this - but it doesn't require it either.  If you really want to see what they're talking about, partition off a few gigs and install Debian or something next to Ubuntu.  You're guaranteed to learn something that you can apply to your Ubuntu experience as well.

### Post by ruy_lopez on 2008-02-05
I've used both. There isn't a lot of difference. In fact, I can't think of anything you can do with Debian, that you can't do witth Ubuntu.

Ubuntu is basically Debian with some nice GUI's to help you configure it. Plus Ubuntu also has support for many Wireless and Network interfaces.

People who say Debian is better are usually old-school Linux people who enjoy having to configure everything themselves. 

The thing is, these people could do the same with Ubuntu. They just don't like the fact Ubuntu makes certain assumptions from the outset. If you don't like those assumptions, you can just as easily change them with Ubuntu as you can with Debian.

In my experience, Debian is the same as Ubuntu, but with a few extra conveniences built-in as standard. That's it.

EDIT: Plus, don't forget Ubuntu has an excellent support forum. ; )
