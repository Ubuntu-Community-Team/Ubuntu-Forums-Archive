### Post by Bartender on 2006-02-13
jbart -

Your questions...
#1 As bonzodog suggests, try the LiveCD.  It runs on top of your existing OS w/out changing anything.  If you can get sound, video, internet, etc. to work that's a really good indication you'll have a relatively trouble-free install.  Since the LiveCD runs from the CD itself, don't be alarmed if it runs slowly.  An actual install should run at least as well as XP.

#2 The install CD (not the same as the LiveCD) is pretty straightforward.  It will detect XP if it's still on the hard drive and you'll then be given options.    If/when you get to the point of doing the deed, check back in.  Meanwhile, just hanging around the forums for a few weeks will provide an introduction.  The command line frivolity comes later :-D  

I don't think the average Linux neophyte is immediately productive, so you may want to consider dual-booting with your existing OS before burning any bridges.  I used this guide
[http://users.bigpond.net.au/hermanzone/](http://users.bigpond.net.au/hermanzone/)
with success.  Used the "Ubuntu +NTFS" directions.  Made the FAT32 partition just like they say.  If you can set your machine up next to another computer that has internet access, that is the way to follow the steps.
  
#3 Linux is stong like bull compared to the leaky alternative OS.  At this point in time it could be argued that Linux simply isn't widespread enuf for the creeps to bother targeting.  Some Linux users resent the influx of Windows users because increased popularity will attract the rats.  Although the need for ANY anti-virus is debated, there are some anti-virus programs available.  "Clamav" appears to be the most popular.  "Firestarter" is a firewall that's available thru the repositories once you get connected to the internet.  I believe that spyware is a non-issue.
I know nothing about wireless.  Cruise the forums, check the Wiki for hardware compatibility, spend some time in "Search", maybe e-mail the manufacturers of your router and wireless cards and see what sort of response you get when asking about Linux compatibility?

damianubuntu has a very good suggestion...you don't want to be stranded with a fresh Ubuntu install and no internet.
