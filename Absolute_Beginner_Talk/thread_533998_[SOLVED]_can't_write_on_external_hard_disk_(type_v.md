### Post by logos34 on 2007-08-24
You don't need to use the storage manager, it should be configured and mount with full read+write access.  Try to save something to it.  It's apparently mounting properly (rw) at /media/sde1, but I don't know why it won;t let you write.  This drive must have been connected when you installed because it was added to fstab (or did you add that entry?).

Since this is an external USB drive it will normally automount even without a line in fstab.  So try commenting out the entry in fstab by adding a '**#**' in front like this:
> 
**#**/dev/sde1 /media/sde1 vfat users,rw,owner,umask=000 0 0

Now go to System>Preferences>Removable Drives and Media.  On the Storage tab make sure the follwing boxes are ticked:

-Mount removable drives when hot-plugged
-Mount removable media when inserted
-browse removable media when inserted

Reboot.  What should happen is that the OS will find the USB drive and automount it at /media/usbdisk  or usbdisk-1, etc.  See if that happens and you can write to it.

### Post by jordanmthomas on 2007-08-24
You shouldn't need the -R there because files on a FAT partition can't hold UNIX permissions.  Also, you will definitely need the quotation marks (unless you do My\ Downloads)
