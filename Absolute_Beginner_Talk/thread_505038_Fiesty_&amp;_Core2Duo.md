### Post by jimrz on 2007-07-19
no longer needed ... the installs generic kernel handles dual procs/cores just fine. i have 3 of these (core2duo E6600 + dual PIII 866's + dual PIII 733's) and all are running nicely and feisty seems to be apportioning the workload properly between the procs. the default will not give you 64 bit, as that does need a different kernel. unless you realy have some specific need for 64bit i would (and am) stick with the generic, as 64 bit presents quite few other issues not found with the 32 bit generic
