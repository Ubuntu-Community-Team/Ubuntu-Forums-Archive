### Post by Bachstelze on 2006-09-28
First of all, welcome to Ubuntu, hope you'll enjoy your stay :)

> **saalota said:**
> Are these all different forms of Ubuntu?

Basically, yes. Ubuntu is what is caled a *distribution* of Linux, i.e. a set of software containing the Linux kernel and a whole lot of other pieces of software put together.

Then, you need to know that in Linux, unlike in Windows or Mac OS, the graphical environment is totally apart from the OS itself. You can run the same OS (here, Ubuntu) with a very wide range of different graphical interfaces, or even without any. Some of the GUIs available for Linux are GNOME, KDE, XFCE, Fluxbox, and many many others. That leads us to Ubuntu, Kubuntu, Xubuntu and so on, the only difference between them is the software that will get inside in addition to the "base" stuff, which is the same in all Ubuntu derivatives. The first letter indicates what environment it is (**K**ubuntu uses **K**DE, **X**ubuntu uses **X**FCE, etc.) except for the "original" Ubuntu which uses GNOME but is not (yet ?) called Gubuntu.

Breezy Badger and Dapper Drake are codenames for two Ubuntu releases. Breezy Badger is version 5.10 and was released in October 2005, Dapper Drake is version 6.06 and was released June 2006. Other releases are Warty Warthog (4.10 - October 2004), Hoary Hedgehog (5.04 - April 2005) and Edgy Eft (6.10 - to be release October 1st 2006).

Righty-o. A fellow Dell user. First off, what is your hardware (esp. Graphics and Wireless) and we can work from there :)

If you can get hold of some software such as Norton Partition Magic, it'll make life easy.

IMHO, the Dell Recovery partition (which probably will be on yours) ain't worth having. On my I6000, it was a nightmare to access (Ctrl-F11 at a very specific time at boot), and the Windows installation image contained even included some spyware  (Doubleclick trackers!).

As a Dell it should come with a util to burn a CD of whichever Windows is installed (It's usually in the "Dell" submenu on the start menu). That CD will be a cleaner install than the recovery partition, the only things you may need are drivers, which are easily obtained from M$ Upgrade or Dell's website anyway.

What you will need is a "Documents" partition, of sorts. I'm assuming your Windows is on NTFS? Well, defrag your C drive, back up everything vital and repartition as follows. (I'm assuming your HD is about 100GBish, adjust accordingly)[I use Norton Partition magic, but Ubuntu 6.06 CD has a partitioner]

-30GB(ish) Windows XP system partition NTFS (will hold Windows and all installed programs.
-40GBish Documents partition in FAT32 (VERY IMPORTANT! Linux CANNOT write to NTFS!)
-25GB(ish) Linux partition in Ext3 (For Ubuntu )
-2GB (max) Linux swapspace (like virtual memory in Windows)

Now this is done, transfer everything in your "My Documents" folder to the FAT32 docs partition, and use TweakUI to repoint Windows to that drive (probably E) as its "My Documents" folder.

Phew, now boot into the Ubuntu LiveCD and install. The Root (/) drive should be in the Ext3 partition (Probably labelled hda3 [or sda3 if you have a SATA HDD like meeee]). It will tell you the formats of all the drives, so make sure you're using all the right drives for the right things.

Install.

Enjoy!

Your FAT32 documents will appear on your desktop as "hda5" (or thereabouts, there will only be one other than hda1). Both Linux and Windows can read/write to this partition .

You may need to change the permissions on this drive. Go into the GNOME terminal and type "sudo nautilus". Now go to hda5(or whatever), right click on an empty space, go to the Permissions tab on "Properties" and make sure owner and group have read/write checked.

Don't forget to BACK UP everything first.

PM me if you need some of this explaining :)

### Post by MonsterDog on 2006-09-28
salotta:  Don't get rid of your windows yet.  Ubuntu doesn't need the space.  When you find you haven't booted windows in a while, then you should start thinking about reclaiming the wasted space.  Contrary to what you may have read, Linux and windows get along very well together.  The current Ubuntu (Dapper) installs Grub and sets everything up for a dual boot as long as windows is already installed.  Gparted can be run from the live cd to reduce the size of your NTFS (windows) partition to make space for Ubuntu and a swap partition.

If things get messed up, all you have to do is boot to DOS and run "fdisk /mbr".  When you reboot you're back in windows and you can start all over again.  If you go into the whole process thinking of this as a practice installation you can feel free to change your configuration, install programs you think you might want, edit and delete files you probably shouldn't because you were planning to re-install anyway.  After you've played around and messed things up a few times, you can install it the 'right' way.

gn2:  Sensible, yes; practical, no.  A new HD + an enclosure's gotta cost over a hundred bucks and you only get a 40G spare drive.  For about the same money you can get a giant usb drive and back up and restore dozens of OS's.
