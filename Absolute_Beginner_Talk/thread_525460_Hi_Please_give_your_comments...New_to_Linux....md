### Post by Xyphus on 2007-08-14
> **Inxsible said:**
> I think Gnome and KDE were alway available, but I may be wrong. I do not know the history of the other DMs and WMs

If I remember correctly, Gnome and KDE both started out as the same project.  However, KDE used proprietary code and had to license it.  Several developers on the team didn't like that so they forked the code and created Gnome which is completely open.  Later, the codebase for KDE became open as well and no longer needed to be licensed.  However, by this time there were two distinct camps.  And Gnome and KDE have been separate in their development ever since.

So was "Windows". The first Windows was not labelled an OS, it was just a graphical desktop environment, which had to have DOS installed first. To use it, you had to type "win" at the prompt, and then Windows loaded. Soon, the command was added to the autoexec.bat, so Windows loaded after DOS. DOS is actually the real OS behind Windows 3x, 9x and ME. They are NOT operating systems, they are DOS with graphical frontends. NT, XP, and Vista are different.

Unix was the same way, but with standards and Linux, the development has blossomed and grew. There are many DE's but GNOME and KDE are on top, many others are used, and are quite useful.

### Post by nvrpunk on 2007-08-14
Linux which was developed while Linus Torvalds was using MINIX, it was a pet project/thesis for his masters.  Anyhow, in the end Linux is the kernel, MINIX was in no way copied, all the code for Linux was original and by Linus.  It merely gave him a reference point.

Ubuntu is merely a Distribution based on Debian, which uses the package manager APT.
Gentoo is a source distribution which uses portage, a BSD ports inspired package manager.
Slackware is a long running Distribution which uses Slackget.  

What makes Redhat now commercial and not Opensource (it's opensource project got turned over to fedora) is that it ADDS closed source developed software by them to the Distribution, for Enterprise Server usage.
For how they can do this read about GPL.

Redhat, and Fedora both use RPM's for their packages.  It was one of the first package management systems, although you still had to search on the web for the RPM's.

Hopefully that sums up most of what I can remember about Linux.  Anyone else can feel free to correct mistakes I have made.
