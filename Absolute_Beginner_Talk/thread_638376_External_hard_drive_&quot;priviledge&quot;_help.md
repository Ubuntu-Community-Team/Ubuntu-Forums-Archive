### Post by Rocket2DMn on 2007-12-12
The last line should probably read
```
/dev/sdc1 /media/Sams_Drive ntfs-3g defaults,locale=en_US.utf8 0 0
```
I don't believe you can make a directory name with an apostrophe in it.  Also be sure that the directory exists already and is owned by root
```
sudo mkdir /media/Sams_Drive
ls -l /media
```
It should show root as the owner and group owner of the Sams_Drive directory.

Once this is all added to the fstab file, you can mount everything in fstab (then unmount the correct drive) by running
```
sudo mount -a
sudo umount /media/Sams_Drive
```

EDIT: You do have ntfs-3g installed and have writing enabled, correct?

### Post by wormser on 2007-12-12
It looks like the drive was removed with out unmounting or safely removing from Windows.  You need to run the command suggested but change where it is mounted to.  If you have Windows then plug in the drive, click safely remove hardware in the system tray (where the clock is) or use the following command.

```
sudo mount -t ntfs-3g /dev/sdc1 /media/**[COLOR="Red"]Sams_Drive[/COLOR]** -o force
```

### Post by Rocket2DMn on 2007-12-12
When it says only root can do that, you need to start the command with "sudo".  However, if you can access a windows machine, you should plug the drive in there then safely remove it using the Safely Remove Hardware icon in the windows taskbar.  The last time this drive was removed from a windows machine it was not done safely so it is going to complain to you about it.
After you safely remove it from a windows machine, plug it back in to your ubuntu setup and run the "sudo mount -a" command again and it should be OK.
