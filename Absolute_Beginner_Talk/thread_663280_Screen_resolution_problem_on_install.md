### Post by mikewhatever on 2008-01-10
If my suggestion from the previous post does not work you can try the following: > sudo /etc/init.d/gdm stop
sudo dpkg-reconfigure xserver-xorg

The first command stops xserver killing gui, so save any work you might be doing. The second will start the process of reconfiguring xserver letting you choose resolution, the type of card and driver etc. It will also create a backup of xorg.conf in case you'd want to revert. Try choosing vesa driver instead of via.
