#################

Mounting and permissions depends on the file system:

_Windows:_ [Psychocats Mount windows ](http://www.psychocats.net/ubuntu/mountwindows)
For read-write: [list=number][*]vfat (FAT) use umask=000
[*]ntfs use [ntfs-3g](http://ubuntuforums.org/showthread.php?t=217009) and a fstab entry something like this:
```
/dev/hda1  /media/windows  ntfs-3g  defaults  0  0
```
[*]An alternate is ntfs-config. ntfs-config uses ntfs-3g to mount windows partitions via a gui :)
[indent][http://ubuntuforums.org/showthread.php?t=337970](http://ubuntuforums.org/showthread.php?t=337970)[/indent][/list]

_Linux_: [Psychocats Mount Linux](http://www.psychocats.net/ubuntu/mountlinux)
To set permissions, mount the partition, then chmod```
sudo chmod 777 /mount/point
```

To mount at boot you will need to edit /etc/fstab (as outlined in the links above).
For an overview of fstab see: [How to fstab](http://www.ubuntuforums.org/showthread.php?&t=283131)

_For access to ext2/3 from windows see_ : [http://www.fs-driver.org/](http://www.fs-driver.org/)

### Post by Whiffle on 2007-10-21
Okay.  Your /media/store seems to be being mounted by gnome-volume-manager, which is the program that is designed to mount drives like usb sticks.  Why its doing it so you can't access it, I don't know, maybe if i weren't tired.  

Heres what I'd do (assuming that this drive is permanently connected...)
First we need to find out which device it is.  it should be something along the lines of /dev/sdxy where x is a letter, and y is a number (the number of the partition on the device).  To do this, unplug the drive, and plug it in.  Then open up a terminal and run:
```
dmesg[/dmesg].
You might have to run it more than once.  You should see an entry that mentions  a device like /dev/sdc1, which would be the drive we're after.  

Next, do 
[code]
mkdir /mnt/store

```
, which will give us a permanent mounting place for the drive.

Next, you could add a line like this to your /etc/fstab

```

/dev/sdc1 /mnt/store ntfs-3g defaults,users 0 0

```
 
After that, you should be able to open up a terminal and do
```

mount /mnt/store

```

and have it mount with the correct permissions.  You'll only need to do that the first time, because it will automount the rest.
