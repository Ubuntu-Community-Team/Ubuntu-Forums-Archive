### Post by Tautoa on 2007-09-13
I reckon your machine should run it without any problems, and if you do encounter any, chances are someone with the same hardware will have already posted in this forum with the same problem, so you should be able to find a solution pretty quickly.

Ubuntu is different from using a Mac, or Windows. I'm told that it has a few similarities to the Mac, but I've never used one, so I couldn't tell you what they were. You won't get very far if you're thinking in terms of how you used to do things in Windows. Just spend a little time in the Beginner forum reading threads and you'll soon get the idea :)

I'm also told that you can actually do everything through a GUI. So if you don't want to learn a bunch of commands, you won't have to. But if you get the basics of the terminal, you will save so much time... there are plenty of tutorials out there.

And finally, security...
So far as I know, the only virii we have in Linux, are ones that have been created by the good guys, just so that they can test to see if virii are theoretically possible.  There are plenty of reasons why Linux security is so good, but you don't have to care about the details at this point, just know that running half a dozen security apps in the background is a thing of the past :)

### Post by jonathonblake on 2007-09-18
> **soldier4jesus said:**
> 
  Will I need to know some stuff about programming to be able to use Ubuntu? 

Only if you want to write your own programs.  Or maybe do some custom shell scripts.   

If you needed to write batch files for Windows, then you may need to write shell scripts for Ubuntu.  But if you didn't need batch files in Windows, you won't need to write shell scripts in Ubuntu.

>  Also, I have heard that there is a way that I can run Ubuntu from a CD without affecting my OS currently on my computer.  How would I go about doing that? 

You want a "Live-CD" installation.   Ubuntu Christian Edition can be run as this.  
(I have been using the Live-CD version for about a month, because my root partition is full.  It is easier to use the Live-CD, than delve into the internals to archive the data I need to, so I can have a root partition that isn't full.  For windows, this would be the equivalent of having a hard drive that is full.) 


> questions is, where do you find those.

Synaptic Package Manager contains a list of at least 5K programs that can be installed.  By adding repositories to it, you could easily end up with a selection of 20 to 30 thousand programs to install.

Then there is Sourceforge, which hosts a number of FLOSS programs.   The odds are in favor of finding a program there that will, when completed, do what you want.   The odds are also that the project was started, but abandoned before any source code was released. 

There are a couple of other places that host FLOSS software development.    I don't remember URLs offhand.  
If I cant' find what I'm looking for on Sourceforge, or Synaptic, I usually ask on the Ubuntu forums for a pointer either to the some that has the functionality I need, or for how to write a python program that does what I want/need.

Matthew wrote:> 
because it has Automatix installed

The use of Automatix is highly discouraged, because it does strange things to one's system.  Everything that Automatix does, can be done by using Synaptic, without the system damage done by Automatix.
