### Post by Edward The Bonobo on 2006-09-13
All the newbie user guides tell you that sooner or later you *will* find it useful to use a terminal.  This is very true.

But the guides assume the newbies know how to work one.  They (we!) don't.  There's a lot of good guides out there on basic linux commands for changing directories, copying files and the like, such as this: [http://www.linuxcommand.org/learning_the_shell.php](http://www.linuxcommand.org/learning_the_shell.php)

But before even getting that far, users have to know how to type in the terminal:
[LIST]
*[*]"How come I can't just pate in text from a web page using ctrl+V?"*
[*]*"Isn't it annoying that the only way I can correct an error is by backspacing and re-typing the whole thing?"*
[*]*"How can I select that word without my mouse?"*
[*]*"I want to run the same command again, but for a different file.  How do I get the command back?"*
[/LIST]
This guide: [http://www.tuxfiles.org/linuxhelp/shortcuts.html](http://www.tuxfiles.org/linuxhelp/shortcuts.html) tells you all the keyboard tricks for moving around in the terminal, correcting errors and saving on typos.

### Post by robins_web on 2006-09-13
*How come I can't just pate in text from a web page using ctrl+V?*

Try using Shift-Ins. ctrl-v only works in GUIs because they are programmed that way to make the switch from Windoze easier. When you're using the command line, you're dealing direct with Linux, which is nothing at all like Windoze.

That is not necessarily true.  If more users like me start using Linux, the terminal could become obsolete pretty quickly.  It'd be even better if there were developers who thought like me.
