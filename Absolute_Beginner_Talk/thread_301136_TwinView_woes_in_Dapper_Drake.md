### Post by aos101 on 2006-11-17
Its sounds like you have the same problem as me discussed here:

[http://www.nvnews.net/vbulletin/showthread.php?t=54638](http://www.nvnews.net/vbulletin/showthread.php?t=54638)

With the nvdivia drivers when an analogue and digital monitor are both connected, the analogue monitor always becomes the primary monitor where the login screen and desktop menues appear by default.  They added an xorg.conf setting (TwinViewXineramaInfoOrder) in the 1.0-9xxx drivers which allows you to fix this.  

If you aren't already, you'll need to be running the 1.0-9xxx drivers and add this in your device section of your xorg.conf:
```
Option "TwinViewXineramaInfoOrder" "DFP-0"
```
I installed the 1.0-9629 drivers by following method 2 [here](http://doc.gwos.org/index.php/Latest_Nvidia_Dapper) and things are working fine with my login screen on my DVI connected monitor.  There is info in [this thread](http://www.ubuntuforums.org/showthread.php?t=255929) about how to add respositories to get the latest nvidia drivers, but I don't think the person who manages those has done the package for Dapper yet.  I think the only current option to get the 1.0-9629 drivers on Dapper is to follow method 2 in the instructions I linked to above.
