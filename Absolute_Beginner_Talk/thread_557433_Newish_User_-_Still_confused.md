### Post by silverglade00 on 2007-09-22
> Am i right in saying that Linux in its true form is a CLI? and that gnome is just a visual aid that sits on top of the kernel giving a GUI of what the CLI is representing?

This is basically true, except that Gnome sits on top of a graphic server called X-windows (or X for short), which in turn sits on top of either the CLI or the kernel, I'm not sure. It is much like the old Windows 3.1 days. You always ran DOS, but you could load Windows to get a graphic view of things. However, you were trapped into one way of doing things. You could only use the interface that Microsoft provided you. With Linux, you can change the entire desktop environment. There a lots to choose from. The 3 supported ones on Ubuntu are GNOME, KDE, and XFCE. To give a very basic and very inaccurate generalization, GNOME is a little like a Mac, KDE is a little like Windows, and XFCE is very lightweight and is good for older systems. KDE's philosophy is that customization is the goal. The huge amount of customization options are easily found. Some think that this causes overclutter. GNOME's goal is usability. The interface is meant to be simple and intuitive. Some think that this is restrictive. The good news is that you can try them all out because you can have multiple desktop environments installed. You choose one at login time. You have this ability because the basic graphic functions are handled by the X server, so you can change the desktop at will.
