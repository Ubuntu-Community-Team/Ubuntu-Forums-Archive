### Post by irish_flu on 2007-01-27
My advice is to "start small".  Learn how to use one or two command-line applications first.

The easiest way is to pick something you want to do, and learn how to do that from the command line.

For instance, the next time you want to install an application: instead of using synaptic package manager, you could use apt-get (which is a command-line based package manager).  Read about apt-get and how it works, then install the application you wanted.

Once you do that a few times, you'll be more comfortable with using the command line, as well as how and where to find advice on using commands.

Remember, if you're sitting there at the command line and you can't figure out how to use a command, you can use the command-line application called "man" (for "manual") to find out how different commands work.

For instance, if I were confused about how apt-get works, I could go to the command line and type "man apt-get".  the "man" application will present you with its entry for apt-get.  Use the error keys to scroll through the manual entry, then press "q" to take you back to the command-prompt.

I think it's easier to learn how to use things when you're using them for things you actually want to do, rather than just learning them for learning's sake (not that that's a bad thing, of course).
