### Post by Kyral on 2006-01-04
Earrobson, telling people to Google is not good policy on the ABT Forum.

The Kernel is best described as the core of the OS. Like the brain of the OS. To give a more technical explanation, its the lowest layer software in the OS. So if you ask OpenOffice to save a document, it looks to you that OO just writes to the disk. What really happens is that OO asks the kernel to write the data. Hope that helps!

### Post by az on 2006-01-04
Something interesting to note is that in open source software and unix-type environments, you can compile the same software stack on a kernel and run the same programs.

For example, if you install a BSD system (another unix with a BSD kernel) you can get X-windows, gnome and all the software that Ubuntu ships and compile and run it on your bsd system.  It wil look and feel the same.

In a way, the kernel is a program that abstracts the hardware to the software that the hardware is running.  Software interacts with hardware through the kernel.  Kernels can have various features and do things in different ways to suit specific needs.
