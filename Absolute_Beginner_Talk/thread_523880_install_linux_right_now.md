### Post by csimons on 2007-08-12
One possible solution might be to  back up your data to a CD, which would put them in a filesystem Linux could access.  Alternatively you could use Windows to create a third partition formatted in FAT32 so that Linux could mount and read it, and move all your files there.

Or am I missing the point?  Do you have large quantities of encoded iTunes music or other files that you'd effectively lose by wiping out your Windows installation?  Too much to burn to CDs and rip back as MP3/OGG files?

Personally, I would just scrounge the money to buy a second hard disk (newegg.com has some new Western Digital IDE hard disks for about $35) and install it, or have a friend or local Ubuntu community member help you install it if you haven't done that before.  That way, you'd be able to keep your Windows stuff and enjoy all the benefits of becoming well acquainted with GNU/Linux.

I used to get into moods where I'd wipe all my proprietary systems off and go completely GNU/Linux or a BSD variant for a couple months, but I always ended up reinstalling Windows at some point so I could use MS Office for school assignments, or iTunes or Subspace or various video games.  Eventually I had to throw up my hands and admit that it would be quite a while before I could entirely give up Windows.  If you're currently in one of these moods, you may think about dual-booting for a month or so, at least to make sure you've made the right choice and are able to get everything working properly (drivers, namely), and are able to find alternative software to the ones you currently use in Windows.
