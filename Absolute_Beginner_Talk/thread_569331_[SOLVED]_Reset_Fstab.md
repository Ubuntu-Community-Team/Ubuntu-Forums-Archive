### Post by talby on 2007-10-07
I think that is from "volumeid" package, hey check out /boot/grub/menu.lst I'll bet it says something like

kernel          /boot/vmlinuz-2.6.15-29-686 root=/dev/sdc1 ro quiet splash

I'll bet if you change it to 

kernel          /boot/vmlinuz-2.6.15-29-686 root=UUID=0c27d207-775e-4c1a-87d9-c194e8af08cb ro quiet splash

it will work without changing the UUID's in /etc/fstab

edit:  But if it does not work it should work with /dev/sda1 in both /etc/fstab and the "root=/dev/sda1" in the kernel options...  you can run "sudo /sbin/dumpe2fs /dev/sda" to see if the UUID changed when the device relocated if so then you can use that in both, not sure if it changes or not...
