I use suse primarily.

You do not have to pay for anything from suse in terms of repositries.

You can even get open suse which is 100% free

Other wise commercial suse is more or less a highly polished KDE desktop/workstation/server system.

The biggest advantage suse has is YAST

It far out paces anything ubuntu has in terms of configuration software installation and maintenance.

secondly the suse updater works much better than the ubuntu one.

My grip with suse is that it uses RPM technology which is no where as advanced as APT.

However because of YAST the dependency problems of RPM are usually easy to deal with.

Overall SUSE is KDE centric ... they do not confiure GNOME properly as Ubuntu does, so I hope you like KDE because it is the only thing that has proper integration.

I have been using suse for 3 or 4 years now ... still using my 9.0 distro because it is THAT good despite having a 9.2 CD on my CD rack.

I usually just add the new KDE and packman repositories to YAST and upgrade what I need.

The suse kernels usually come with a lot of load however and you might want to strip it down and recompile it for speed ... suse can take VERY long to boot compared to UBUNTU.

What can  say in short about SuSE ... Those Germans really can code !

### Post by asimon on 2005-10-27
The difference between the SUSE Linux OSS version which can be downloaded and the boxed retail version is that the later contains some propritary software like Sun's Java, Acrobat, Realplayer, and a lot of other stuff that is missing in the OSS version. But there exist repositories, where one can download some (most?) of the missing propritary software. Lists of repos are on opensuse.org.

[QUOTE=erikpiper]Suse was slow and bloated, KDE centric, and RPM. Ubuntu is much better in my opinion.[/QUOTE]

My expierences are a little bit different. I think Kubuntu is a great distro, but so is SUSE. Here on the same machine SUSE Linux OSS 10.0 boots more than 20 seconds faster than Breezy. Because of preloading KDE and Openoffice start faster then on Ubuntu too. Updating is faster too because of their delta rpms. I don't know whats wrong with Ubuntu's Firefox but is feels sloooow. KDE centric is a plus for me. I only played for a short while with their Gnome desktop but I liked their beagle very much and didn't had the impression that it was not as good as Ubuntus. RPM or DEB means nothing to me, what counts is the package manager (apt-get on SUSE too, yum, and even my favorite smart, use whatever you want) and the quality of the packages, which seems very good on SUSE.

Both are excellent distros.

And statements like "rpm is not advanced as apt" is wrong. It should be "rpm is not advanced as deb", which btw. I think is wrong too. ;-) RPM is a format, apt a package manager, difference eh? Apt on Ubuntu manages deb packages. Apt on SUSE manages rpm packages. I think from a user's perspective it's a uninteresting implementation detail in which format his packages are. What counts is a easy to use and powerful package manager not the package format.

### Post by timczer on 2005-10-27
I just recently upgraded to Breezy from Hoary and also just started using Suse 9.3 (it was free).  I have been using Ubuntu for several months now, so the comfort level is higher than for Suse.  Suse seems to come a little bit more ready to use.  Once it was loaded just about everything was already there, ready to go. It takes a little more tweeking to get Ubuntu to the same level.

So bottom line, for day-to-day stuff like email, web-surfing, documents, they are pretty equal.  If I want to mess around, tweek, try something new, Ubuntu is the choice.
