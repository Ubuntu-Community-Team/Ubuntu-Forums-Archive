### Post by confused57 on 2006-05-26
A frequent thread topic that pops up is someone asking about Kubuntu and KDE, and most replies are to "sudo aptitude install kubuntu-desktop"; which is a really good way to test out KDE.

I think a good way would be to run the Knoppix LiveCD,learn how it is different from Gnome, getting familiar with the Kmenu's, etc.  There is a persistent image option that you can store settings on a USB flash or onto your hd(which is really easy if you're already running Linux); that way you can configure and save your settings when you quit Knoppix.

If you decide you like KDE, by all means install Kubuntu-desktop; if not, then you haven't wasted a lot of time installing it, then uninstalling it if you don't like it.  I realize there are pretty significant differences in configuring Knoppix and Kubuntu, but it would give someone a pretty good idea of KDE.

If this sounds like a viable alternative, I can "attempt" to walk someone through how to set up the persistent image in Knoppix.  I just would like to know what some of the more experienced users think about this suggestion.

### Post by aysiu on 2006-05-26
I think this suggestion makes sense only if you plan to use Knoppix for other purposes--rescuing busted Windows installations, impressing your friends, etc.

Otherwise, the time it takes to download and burn a Knoppix CD and reboot is much more than the time it takes to install *kubuntu-desktop* using *aptitude*.

In fact, if you're going to download a CD just to see what Kubuntu is like--you could always just download the Kubuntu live CD!
