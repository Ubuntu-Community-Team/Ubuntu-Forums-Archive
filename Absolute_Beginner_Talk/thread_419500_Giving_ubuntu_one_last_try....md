### Post by Anvilsmith on 2007-04-23
I was pretty excited when I got Ubuntu to run on my HP 510 laptop, given how easy it was to download new stuff. The fact that my touchpad wasn't working proved only a minor nuisance at first.

Then I tried looking over the file structure, which made no sense to me, as I wasn't at all familiar with linux. The fact that I couldn't find any obvious way to send software I downloaded via "add/remove" to a specific directory also proved a hinderance. I told myself it was okay, as I could always look into all this later; there was bound to be a solid documentation on the net.

I looked into a way to get my touchpad working, and found none, even though this problem has existed for months. Granted, there ~was~ a fix, which involved recompiling the kernel and screwing me out of my WLAN card. With no internet access, I obviously couldn't look up linux functions in real time.

Then I gave up on that as well, and figured I'd get started on writing some interactive fiction... Which was one of the main reasons I had saved two years' worth of money to go buy a laptop in the first place. Downloading Anjuta was quick and easy; not so with dowloading the other stuff it required in order to organize my files into projects, as even ~those~ had dependencies with which I was completely unfamiliar. I also tried installing three interactive fiction compilers that work on linux, to no avail. They had their own dependencies (libraries, mostly), which likewise proved impossible for me to install, given my level of experience with linux.

Finally, a bug showed up that prevented me from accessing the "applications" menu and the "Menu layout" option in the preferences menu. I decided to switch to OpenSUSE... Which didn't make things any easier. Indeed, it didn't recognize my wired connection, either.

I don't mean to rant, but with ubuntu being advertised as "user-friendly", one would think its user-friendliness didn't limit itself to using a small number of core "supported" programs. As it stands, however, the moment I enter console mode, all user-friendliness goes poof... And opening a terminal is something that inevitably happens as soon as I need to install something that's not covered in the "add/remove" menu.

I'm planning to try Ubuntu again, given OpenSUSE's faults... All I need, though, is a decent downloadable syntax guide that would take no more than two days to thoroughly study. I never thought doing the simplest things in linux would be so much of a pain, frankly, and really wish I won't have to save up for another windows license just so I can use a 2-meg compiler.

> And opening a terminal is something that inevitably happens as soon as I need to install something that's not covered in the "add/remove" menu.

There's Synaptic Package Manager to install software not in Add/Remove.  (Systems > Administration > Synaptic Package Manager)  With it you can install much more software without using the terminal.
