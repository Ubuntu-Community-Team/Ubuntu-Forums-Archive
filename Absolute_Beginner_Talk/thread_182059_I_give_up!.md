Take a look at this site, it explains the basics of how the command line works in Linux and about files and directories, what goes where etc.
[http://www.tuxfiles.org/](http://www.tuxfiles.org/)

To be honest I havn't really spent much time thinking about what all the directories and commands are. I tend more just to find things out as I need them, but I do learn other things on the way.

Just to explain something important that will make things easier. Windows installs everything it needs for a program in a folder, in 'program files'. In Linux a package (program) spreads out into lots of directories. This means that in windows a lot of stuff gets duplicated cause it will install everything it needs for each for each program. In Linux if you have 2 programs and they share some common files, the files won't be installed twice. Both programs will access the same files, which is why the spread out into the relevant directories.
