### Post by mcduck on 2007-04-29
In your fstab you have this line for hdc6:
```
# /dev/hdc6
UUID=48d38f75-e6d9-1d2a-e5c0-f72b78818744 /media/hdc6 ext3 defaults 0 2
```
..while the ls command tells that the right UUID is different:
 ```
lrwxrwxrwx 1 root root 10 2007-04-29 14:15 A7F3-F970 -> ../../hdc6
```

Same problem with hdc7:
```
# /dev/hdc7
UUID=309dcbf4-99a4-4f08-8673-8feebfb7119a none swap sw 0 0
```
..and:
```
lrwxrwxrwx 1 root root 10 2007-04-29 14:15 E79A-BF54 -> ../../hdc7
```

Change the UUID's in fstab into correct ones and the problem should be fixed:
```
# /dev/hdc6
UUID=A7F3-F970 /media/hdc6 ext3 defaults 0 2
# /dev/hdc7
UUID=E79A-BF54 none swap sw 0 0
```

### Post by mcduck on 2007-04-30
Sda used to be for SATA and SCSI drives, and hda for PATA drives, but the developers found out that the SATA driver handles PATA drives better than their own driver did. So, from now on, we are using those drivers for all hard disks. And that's why the names changed.. No reason to worry about that. :)

The conversion was from old fstab format, using the /dev/hd** style, into new way of using UUID's. (UUID is a random number given to a partition when it's created). The benefit of this is while moving a hard disk into another cable would change the device name, UUID will stay same. Also removable drives get a different device name depending on which order you plug them in, and that made configuring your removable drives in fstab impossible. So while using UUID's in fstab makes it a bit harder to understand, it solves many problems and that's why it's used in Ubuntu 6.10 and later versions. The old way of using device names still works, so if you like you could use that as well.
