### Post by Rocket2DMn on 2008-01-10
There is no rule against using your name, it's just irrelevant for the most part.  What we see in your ifconfig is that you are behind a router since your IP is a private IP distributed by your router rather than your public IP which is given by your ISP (your router holds this IP).
The source-o-matic site is back up, so go there to generate yourself a new sources.list file: [http://www.ubuntu-nl.org/source-o-matic/](http://www.ubuntu-nl.org/source-o-matic/)
Overwrite your sources.list by copying and pasting over what exists now, you can edit it by running:
```
gksudo gedit /etc/apt/sources.list
```
Enter your pw and proceed.  Then run update (checks the repos aka repositories) and upgrade (download and instlal updates to your computer)
```
sudo apt-get update
sudo apt-get upgrade
```
You should use apt-get or Synaptic (the GUI for apt-get, essentially) to install programs rather than Add/Remove.

### Post by ByteJuggler on 2008-01-10
Everyone:  This is almost certainly the glibc6 library's name resolution workaround regression  (from Feisty -> Gutsy).  It is an unfortunate problem that occurs with some models of routers which doesn't properly handle IPv6 DNS requests/responses hence causing DNS queries to fail, that then surfaces as name resolution weirdness, espeically for some reason in apps like apt-get (1.0.0.0 etc) and so on. A workaround for the problem was present in Feisty but accidentally got removed in Gutsy again when the upstreadm Debian package was updated or whatever into Gutsy.  (The exact details escape me now so you'll have to check that if you're interested.)  Forcing Firefox to not use IPv6 only works around the problem in Firefox but does nothing for the rest of the system.

The 2 relevant threads on Launchpad is here:
[https://bugs.launchpad.net/ubuntu/+source/glibc/+bug/156720](https://bugs.launchpad.net/ubuntu/+source/glibc/+bug/156720)
[https://bugs.launchpad.net/ubuntu/+bug/81057](https://bugs.launchpad.net/ubuntu/+bug/81057)

The easiest way to fix it, is to download and install the latest libc6 package (which now contains a patch/workaround for the issue), and then firstly upgrade that first on your system, after which all your name resolution should work again and you should then be able to install all other updates without issues.  

I *think* you can download this one here: 
[http://gb.archive.ubuntu.com/ubuntu/pool/main/g/glibc/libc6_2.6.1-1ubuntu10_i386.deb](http://gb.archive.ubuntu.com/ubuntu/pool/main/g/glibc/libc6_2.6.1-1ubuntu10_i386.deb)

And install that. You should be able to open it using your filesystem explorer -- Once you double click it, the explorer will ask how to open it and should offer to open it with GDebi, which is a graphical package installer. Or else, to install from command line, you can alternatively open a terminal and enter:

```
sudo dpkg -i libc6_2.6.1-1ubuntu10_i386.deb
```

Making sure you're in the folder the file is located in first.  If that's your desktop, then to change to that folder you'd do:

```
cd ~/Desktop
```

before executing the dpkg command.  

As I say, if you can get this library patched, I think the problem should go away.

Hope that helps!!
