### Post by Malibu Illusion on 2007-07-02
You do not necessarily have to run older versions of Linux just because you have a low-spec system.  It is perfectly possible to run up-to-date installations on lower-end systems by making a few provisions for it; mainly by dropping some of the fancy desktop effects which take so much performance out of systems for zero benefit from a technical standpoint.  For example, running a copy of Ubuntu with a Fluxbox environment would have a significantly less overhead than running Ubuntu with GNOME.  

That said, a very common phrase applies to Linux: "if it ain't broke, don't fix it."  This rings true if you already have working Linux systems that you wish to continue running with what they have, rather than upgrading.  I have several servers that are running old copies of Fedora, simply because there is absolutely zero reason to upgrade these systems from a functionality standpoint.  As long as your software and daemons have the necessarily critical security fixes applied to them, you really don't have a lot to worry about in terms of upgrading your distribution.

Take care.
