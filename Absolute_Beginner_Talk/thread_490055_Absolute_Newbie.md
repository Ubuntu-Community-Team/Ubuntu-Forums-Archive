### Post by tact on 2007-07-02
Do you really need to use automatix to install what you wish to install?  Using automatix should be prefaced with warnings about it potentially breaking your system (specially when it comes to version upgrades of the OS).

If you are new to ubuntu then you might not know that there is a huge resource of programs available to you and access to very easy installs (just like automatix promises) is already installed and available on your ubuntu HDD.   No need to install automatix, just use Synaptic package manager (System>Administration>Synaptic Package manager).  Or perhaps even easier - click on Applications>Add/Remove....  

Using either of the above methods all you need to do is point and click to download and install programs.

That said - as someone else asked - more info on what exactly are you doing that locks up?  If it is the automatix installation that is locking up - maybe think about not bothering to install it for now.

Have a look-see whether the things you wanted to do using automatix can be easily done without it...  sound us out here if you want...

Thanks.

### Post by tact on 2007-07-03
Ok at the risk of someone considering it bashing (something or other).... here is some advice regarding having as best an experience as you can installing software in ubuntu:

1.  least pain of all - use synaptic et al
2. if the software you want to install is not an any of the normal repositories have a search (google) and see if someone in the world has either:
2a. set up a repository of their own and packaged the software you want for ubuntu or
2b. at least packaged it in a *.deb format
3. dare i mention automatix?  hehehe

(from this point on its getting more and more dodgy - i.e. more chances of weirdness or things nor installing as they should)
4.  if no one has made a repo or deb already then you could look for a .rpm and use alien to try install...  
5.  then of course using tarballs (.tgz) but with this you will undoubtedly need to also look for some installation instructions wherever it was you found the software.   You may find inside the tarball binaries or source code and need to compile.
(the above is not exhaustive)

In the case of Panda antivirus I just googled and found their website and (if this is the page you downloaded your tarball from) there were the following instructions:

###
If Red Hat Package Manager (rpm) is not installed -for example in Debian-, use the .tgz file. In this case follow the steps below:

    * Copy the pavcl_linux_i386.tgz file to the root / directory.
    * Decompress the file using the following command:
      gzip –d pavcl_linux_i386.tgz
    * The file pavcl_linux_i386.tar is created.
    * In order to install the antivirus, type:
      tar –xvf pavcl_linux_i386.tar
###

By the sounds of it you didn't follow this kind of instruction...  so to give further advice some feedback from you on the above, or more info, is required.
