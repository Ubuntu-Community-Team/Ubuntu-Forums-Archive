### Post by Nythain on 2008-02-05
i personally do a command line install and build from there, drop fluxbox on that bad boy, and start flying

you can still install synaptic without a DE or whatnot, but you at least need an X environment with a window manager (on my fluxbox get up its one of the first things i install, and, before fluxbox i was even installing it in kubuntu (wich uses adept by default, unless thats changed since edgy)

basically, when installing apps, keep an eye on what required packages get installed with it... if you notice a program relies on like 90% of kde, you might want to avoid installing it, and vice versa with those gtk/gnome apps.

you could google "lightweight linux programs" and get tons of lists of programs that will run on minimal/lightweight systems, you can even go the home pages of distros like puppylinux, damn small linux, fluxbuntu, and xubuntu, to see what lightweight apps they chose for thier distros (wich is how i stumbled upon half of what i use)

oh, and to answer you Desktop Environment question, it can be defined many ways, but most definitions will include these facts.
A collection of programs packaged together and picked because they are built around the same graphical libraries and will theoretically run better in conjunction with each other.
Popular DE's include
KDE - the Kool Desktop Environment, most packages being build around the qt libraries
Gnome - most packages built around gtk (gnome<?> tool kit) libraries
Xfce - lighter than kde and gnome, mostly gtk apps with minimal library dependencies

theres technically more to it than that and more de's than that, but thats about the jist of it... wich one is better is all personall preference, and on most of todays modern computing technology youwont even notice much of performance difference between the "bigger" ones like kde and gnome

### Post by RedSquirrel on 2008-02-06
> **iamclueless said:**
> how are Ubuntu updates handled... a search for updates commandwill exists???? regardless of desktop ir WM

You can obtain the updates using Synaptic. You can also install and run update-manager if you like.

My preference is to use the command-line tools for installing/updating/removing software. It's not that hard. After some reading and a little practice you'll be very comfortable with it.

These are the '**apt-get**' commands to install the latest updates:

```
 sudo apt-get update
```
and then
```
 sudo apt-get upgrade
```

You can use **aptitude** instead if you like.

This might help:

[https://help.ubuntu.com/community/SoftwareManagement](https://help.ubuntu.com/community/SoftwareManagement)
