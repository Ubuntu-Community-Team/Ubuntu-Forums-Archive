### Post by krusbjorn on 2006-05-01
Ignore the dollar sign. If you open your terminal, you will notice that there is a $ just after the current directory, like:

yourname@computer:/usr/bin$

### Post by Qrk on 2006-05-01
The dollar sign is just the filler character that the terminal uses to tell you that you have user level permissions. If it were a # it be root, etc.

### Post by towsonu2003 on 2006-05-02
I struggled with that crazy dollar sign myself in my first few linux days. bash: command not found - wasn't very helping :P 

anyway, when one posts 
$ command 
that means s/he wants you to do it with user privileges. when s/he posts
# command
that means you will want root privileges for that command. 

In ubuntu's terms, 
# command
equals
$ sudo command
