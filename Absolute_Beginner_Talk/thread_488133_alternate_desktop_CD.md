### Post by starcraft.man on 2007-06-29
> **mikeypc2006 said:**
> How does this work and what does it do differently from the Live CD?

Alternate Installer uses a text based system, it is straight forward but will ask many questions and allows for more control over the install than the simple live CD. It has options for OEM installations, server (I think) and the regular text install for desktops. It takes less resources than the Live CD and thus works with more systems because of that.

Live CD: Boots up into an aptly named "live" version of Ubuntu, lets you test the water without installing. Then offers the GUI install from the desktop icon, following a few short graphical steps a simple install is accomplished with little effort.

Both will get you the same OS install, has to do with preference and what your needs are (small OEM's wouldn't use the GUI install for instance). I always use the alternate CD, thats my preference though.

### Post by seetho on 2007-06-29
I too prefer to use the alternate CD for all my installations.  It is not much different from the Live CD installation - just the text ui vs gui.

Most older PCs and notebook hardware can't handle booting the Live CD.  On my old IBM x31 it took almost 1/2 hr just to boot and it did not really work well enough to "test drive".  In fact I have never been able to install from the Live CD before even on dual core PCs with lots of RAM.

If you don't need to "test drive" then just use the alternate CD and save yourself some time and frustration.
