### Post by Bachstelze on 2007-03-21
Back on topic :

Basically, the two important thigs are :

-> You must include a copy of the relevant license(s) to your "bundle" so the person who gets it - for money or not - knows what his rights are.

-> You must provide the source code of the software you included if someone asks for it.

If you don't modify the existing GPL'ed software but just add yours to it, that's all you need to do to be OK licence-wise. For BSD-licensed softwar, only the first one applies.

Yes, as long as you respect the two conditions I mentioned above. Of course, they apply only to the GPL'ed software you wish to put in your "bundle", you're still free to do what you want with yours. ### Post by Tomosaur on 2007-03-21
If you wish to change Ubuntu significantly, you need to talk to Canonical. You cannot pass Ubuntu off as your own work. Provided it is still obvious that your version is Ubuntu, and that you do not attempt to change any of the already existing licenses, you should be ok.

However, it is common courtesy to at LEAST notify the developers when you wish to do something like this. I would recommend, above all else, that you talk to Canonical and the Ubuntu developers before you go about changing anything.

If you try to relicense anything in Ubuntu, then that is illegal. You are allowed to modify and redistribute most of it (that which is under such a licence), but it is still best to let the developers know before hand.
