### Post by Nythain on 2008-03-05
```

gksu gedit /etc/apt/sources.list

```
uncomment (remove all the # signs) from any line that looks like
deb [http://whatever.something.com/more](http://whatever.something.com/more) stuff multiverse
etc. etc.
Basically that will "enable" all the online repositories so that apt/apt-get/aptitude/adept/synaptic/whatever you use can actually connect and download the package for installation

```

sudo aptitude install nvidia-glx-new

```at least i think your 7000m would still be covered under new
from there you could either manually edit your xorg.conf file
```

gksu gedit /etc/X11/xorg.conf

```and change the driver for the display device from "nv" to "nvidia" and make any other changes you need for optimal composite rendering and whatnot
or
```

sudo dpkg-reconfigure xserver-xorg

```and answer all the questions, making sure to select the resolutions you want available and the right driver (nvidia not nv) for a standard generic nvida driver endabled xorg.conf (you will still probably want to make a few edits manually, adding a few "options")

that should hopefully get you going on proprietary drivers end of things... as for the second question, it was a bit to vague for me to help with, sorry
