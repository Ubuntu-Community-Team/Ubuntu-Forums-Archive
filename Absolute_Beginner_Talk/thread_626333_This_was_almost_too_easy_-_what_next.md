### Post by ben.bos on 2007-11-29
I had been thinking about installing linux for a long time, but never really got around to it.  I was a fairly proficient windows user, but have no real programming experience, and I had never used a linux system before.  I ran the live CD yesterday and everything worked like a charm.  I thought it'd be a good learning experience to immerse myself, so now my computer is completely windows free.

Somehow, I keep waiting for the other shoe to drop - I'm waiting for something not to work, or for a problem to take effort to solve.  So far, anything that didn't work on it's own was fixed by about two seconds of googling.  Nothing has taken more than a minute or two to get up and running.  Before I switched, I read up a bit, and people talked about wireless being an issue.  I found some drivers for my wireless adapter, but I didn't even have to use them, my wireless just works.

I'm almost (but not quite) disappointed.  While the open source ideology is definitely the main reason I switched, one of the things I was really excited about was a chance to learn something new. I really learn best by problem solving.  There just aren't any problems to solve.

So here's my question:  What should I do to learn the ins and outs of linux if everything works on it's own?  I read the sticky posts at the top of this forum and installed a couple of things from the terminal rather than the GUI to try to get used to it, but that was fairly straight-forward.  What should I try next if my computer already does pretty much everything I want it to?

### Post by mevets on 2007-11-29
Running compiz is something everyone likes to do. You may have already found this but if you right click on the desktop > Change Background > Visual Effect > Extra, then you will start up compiz. If you really want to be able to tweak the 3D settings then go to Add/Remove and search for ccsm. Install that application and then run ccsm to get going.

Anyway, here's a list of things you can try:
1.  Try installing another distro and get it working (you don't have to remove Ubuntu or anything, but see what else is out there)
2.  Try setting up a web server or something.
3.  Make your desktop your own -- theme the heck out of it.
4.  Just wait...eventually you'll see something you want to do and when you do, take the opportunity to learn and to do it.

### Post by argie on 2007-11-29
You could try running some of the more obscure programs that people grumble about.

If you subscribe to the [Debian Package of the Day RSS feed]("http://debaday.debian.net/feed/atom/") or the [Gnomefiles]("http://www.gnomefiles.org/files/gnomefiles.rdf") feed you may be able to find something interesting to play with. 

Also, if you have a DVD drive, or if you worked with some specialised Windows applications (Photoshop, for example) you can try to duplicate the effects you get there in alternatives like the GIMP and seeing if you get anywhere.

You can play around with the shell, and learn a bit of bash scripting, try renaming a folder of music files according to the id3 tag from the command line (remember to backup, when you're learning everything can go wrong). Play around, it's a whole new world.
