### Post by starcraft.man on 2007-06-13
Firstly, you have to cd into its mounted location. That would be /media/Dual_Boot, hda1 is what partition it is on your drive.

Second, the permission of the mounted drive has defaulted to root. You have to assign yourself access to it. Since the file permissions system is so pervasive I won't just give you a command to run but point you to a tutorial to teach you about reassigning permissions via the terminal. Once ya understand em you can reassign your regular user group to control the partition. [Lesson is here on linuxcommand.]("http://www.linuxcommand.org/lts0070.php")

That should do it, assuming NTFS-3g drivers are installed right and permissions are set you will be fine to read and write to drive.

Best of luck, En Taro Adun :p.

### Post by Inxsible on 2007-06-13
it could be a permissions issue. you can change your ownership by using the following command
```
sudo chown -R <your username>:<your group> <device mount path>
```
So assuming that your username and group is bob
```
sudo chown -R bob:bob /media/Dual_Boot
```
