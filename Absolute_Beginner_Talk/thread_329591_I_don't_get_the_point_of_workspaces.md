### Post by evanvlane on 2007-01-01
I like them because you can organize your apps like you would organize more static things like files. Everyone evolves their own preferences-- for instance, I like to split up my apps like:

Workspace 1:
[LIST]
[*]Firefox
[*]E-mail
[*]Skype / GAIM
[/LIST] 

Workspace 2:
[LIST]
[*]uTorrent
[*]LimeWire
[/LIST]

Workspace 3:
[LIST]
[*]Azureus
[*]Kaffeine
[*]Serpentine
[/LIST]

Workspace 4:
[LIST]
[*]Terminal
[*]Root Terminal
[*]Misc.
[*]oOffice.org
[/LIST]

I'm just getting a fifth established for a sort of 'notepad' workspace--trying the whole GTD concept, but I'm not sure if I like it yet or not.

So if you noticed, I just sort of organize them by themes-- 1 for internet, 2 for p2p, 3 for media, 4 for productivity/ terminal... 

Just use the hot keys... I'm sure you're used to the 'alt-tab' 'alt-shift-tab' of App Tabbing, and the 'ctrl-tab' 'ctrl-shift-tab' of say Firefox tabbing, but try and start using the 'ctrl-alt-arrow key' hot key to switch between workspaces. It's just as quick as 'alt-tab' but allows you to expand your work environment and work in a parallel sense as opposed to the linear cluttered way that you're used to...

Just a preference, though--- and it did take me a lil' while to get that hot-key committed to muscle memory, and to remember where I put what.


Evan.

### Post by WiseElben on 2007-01-01
Workspaces may seem useless at first (I felt the same way), but once you start using it, you begin to see the benefits. I personally use three workspaces. The first one is used for Firefox, Nautilus, and OO.org or Abiword. I usually utilize the second one as a programming workspace (I sometimes have Firefox opened there to, to reference to documentation), running either Ajunta or Code::Blocks or just gedit with a terminal. I use my third workspace exclusively as a GIMP workspace. GIMP has so many windows (something I really hate) that I need a while workspace for it.

Workspaces are also useful whenever I have a lot of things open but not really doing anything and someone else comes over to use the computer. I simply switch to an unused workspace, thus giving them a new and clean workspace to work with. I don't have to worry about them accidentally changing or closing something I was working.

So in conclusion, workspaces hold many benefits. It simply depends on how you organize it. Try it out, and you might grow attached.

### Post by WiseElben on 2007-01-01
> **CheshireMac said:**
> So, are you all opening these programs every time you boot your machine, or can you program your workspaces somehow?

First choice. You CAN autostart programs in it's perspective workspaces every time you boot, but I wouldn't do that. I mean, why would I open up Code::Blocks if I'm not going to be doing any coding? Truthfully, I don't use a strict guideline for my workspaces. I just move some stuff over whenever my first one gets full (sometimes I have around 15 applications open). Usually, they get moved in such a way that it fits my "guideline" that I showed in my previous post.

By the way, in case you want some programs to autostart (GAIM is a useful one): Go to System->Prefs->Sessions. There, you can add programs to autostart. Or if you'd like, you make a new session and check the  "Automatically save changes to session" in order to make Ubuntu load with whatever applications you had open when you logged off.

### Post by WiseElben on 2007-01-02
You won't use more memory from having workspaces (as they are integrated into the desktop), but you will use more memory if you have more applications open. Having 20 apps open on one work space takes as much memory as having 20 apps open across four workspaces.
