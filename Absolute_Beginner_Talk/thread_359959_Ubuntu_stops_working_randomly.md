### Post by igknighted on 2007-02-12
> **Balls McSack said:**
> Anyone else want to try and help? Any help is appreciated.

I'm not sure why no one has said this, but the very simple answer is you need drivers for your video card.  If the GUI is too unstable to install drivers in, try recovery mode, it is terminal only but wont crash on you.  Open a terminal and type the following commands:
```
sudo aptitude update
sudo aptitude install xorg-driver-fglrx
sudo aticonfig --initial
```
Finally, reboot and you should be problem free.

If you had ever done a fresh install of windows you would have run into a myriad of these types of issues (I still can't install Vista on my more than capable machine due to driver issues that stop the install, plus several pieces of hardware that didn't work afterwards with RC2.  XP is not devoid of these bugs either, as I can't install it top my SATA hard drive without downloading the drivers first and having them on a floppy to install when the time comes... of course, I don't have a floppy drive, so needless to say theres a small issue there).  The point is, certain hardware and certain OS's dont mix well.  Overall, the base linux install does a far better job of recognizing hardware and using it properly than windows does.

One final thought: learning a new OS is very tedious and requires a lot of relearning things.  Imagine if you had never used windows and tried to install it from scratch.  You would be sunk.  The fact that non linux users can successfully install linux in a large majority of cases is a tremendous feat.  However, there is a lot to learn and it is not easy, so be ready for it.  If you can't deal with the occasional issues that will come up, you may want to rethink learning linux.

Of course, part of the reason why suggesting "change your hardware" doesn't work is that for an increasing number of people, their hardware is a laptop, or company provided. Or possibly (but less likely), there is some functionality or quality issue that one manufacturer provides that another doesn't.

In either case, video cards may not be easy to swap in or swap out.  

For example, I'm trialling Ubuntu on a laptop right now. If there's instability with some hardware AND Ubuntu is unable to control that problem so that the entire system crashes or freezes rather than merely reporting a fault and closing or restarting some subsystem in a controlled manner, I'd be less than happy at being told the "answer" to my problem was to accept that Linux can't control a problematic driver and that the answer was to buy a new laptop.

That's not in fact a complaint, since I havent had hardware or freeze issues (yet?). But it's a reminder maybe, to not assume others are free to make the world's most perfect choices, and that the touted expectation which drives many people away from Windows and towards a user-friendly version of Linux such as Ubuntu is the understanding that if something goes wrong (whether hardware or software), it won't crash out the entire operating system as with Windows, because Linux is better designed than that.  

That may not be literally true, but it *is* the implied promise of many people who scornfully say "well you should switch to Linux", and the implied promise of many pro-Linux websites. It's the boast of many Linux pages about security and stability, and low virus rates.  So it's hard to then fault people for believing it, and for their disappointment if it isn't so.

[COLOR="Red"]**[/COLOR] If you've already followed igknighted's instructions, you will need to add the following to the end of your /etc/X11/xorg.conf file *(as the Composite extension is enabled by default in Edgy, however, fglrx does not yet support Composite with DRI and the system will resort to using the Mesa drivers)*:
```
sudo gedit /etc/X11/xorg.conf
```
Add the following lines to the end of the file:
```

Section "Extensions"
        Option  "Composite" "Disable"
EndSection

Section "ServerFlags"
        Option  "AIGLX" "off"
EndSection
```
