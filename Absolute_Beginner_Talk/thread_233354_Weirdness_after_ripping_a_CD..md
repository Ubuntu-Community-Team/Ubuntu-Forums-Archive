### Post by indytim on 2006-08-10
I can't offer much assistance from a technical standpoint on the infamous "rootkit" other than it is/was a pain in the $#@%$# on the windows side of the world.  

I've run into dual-purpose CDAs before on Windows (haven't run into it yet on Ubuntu-Gnome).  Usually the CD contains an executable file that will auto-start once put into an optical PC drive (windows side).  My solution was to cancel out of the auto start and go directly to my ripping app (CDex for me on Windows).  From there I selected the actual CDA files for ripping and ignored the .exe file.  Of course if you do a raw copy (CD->CD) you will inherit all of the crapola on the original CDA.

Hope this is of some value to you.
