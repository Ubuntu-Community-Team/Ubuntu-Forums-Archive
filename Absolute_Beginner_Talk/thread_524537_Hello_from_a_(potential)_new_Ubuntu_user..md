### Post by jambarama on 2007-08-13
Awesome post, honesty is always appreciated.  The first question I'd look at is why switch.  What is it about Windows that tires you, and what is it you want Ubuntu to do for you?  [The sticky at the top of this forum]("http://ubuntuforums.org/showthread.php?t=232059") provides some great info about why switch, what to watch out for, what to expect, and [installation help]("http://ubuntuforums.org/showthread.php?t=500020").  

Personally, I have no formal computer training myself, but I've been running one Linux version or another since 2001.  Let me say that the latest Ubuntu is absolutely the most beginner-friendly, easy to use, "it just works", version of Linux I have ever used.  Installing Ubuntu is all point and click.  There is a Windows migration tool, that will try to transfer as many of your settings as possible, and once installed, the interface is intuitive, capable and friendly.  

That said, like with any operating system, there can be complex problems.  Fixing problems in Linux usually isn't the same as fixing them in Windows, so your troubleshooting abilities may go out the window.  This forum is very active and helpful (glad you found it), but if Ubuntu is on your only computer and it gets borked, it'll be awfully hard to read the forums.  

About Gentoo, many say it is a great place to jump in to get your feet wet (that may be true) it is probably not a great place to jump in if you just want your computer to work.  Ubuntu is more geared towards this.  There are no qualifications to running Ubuntu, no necessary computer language knowledge, no deep computer knowledge, no IT degree required.  

It does help if you understand the very basics (what a CPU is, a bit about partitioning a hard drive, et cetera), but if you don't know those things but want to, Linux is a great teacher.  It really only takes interest, time and patience.  Learning a new OS will be disorienting (switching to OS X disoriented me for a long time), but my grandmother literally runs Ubuntu (which I set up for her of course) and it generally works just fine.

### Post by kayvortex on 2007-08-15
Yeah, wiping the disk is definitely the best option right now I think: I can't explicitly spot anything that went wrong and I don't think your other problems are likely to be related to this one. One last thing that you could try is what I suggested before: fix the MBR and see if windows can boot without grub. 

To do this, boot from the windows CD and then press "R" to enter the recovery console (like before), and then run the command "fixmbr". It will talk about an unrecognized or corrupted MBR and ask you if you want to continue; just type yes and then type exit to reboot the PC. If windows then boots, then the problem was with grub; if not, then the windows partition is corrupted. Either way, you'll need to re-install grub, which *may* also fix your problem if the problem was with grub. To do that, follow [this thread]("http://ubuntuforums.org/showthread.php?t=224351"). In summary, boot from the live CD, open a terminal and type

```
sudo grub
```
then, for your setup, this
```
root (hd0,1)
```
then this
```
setup (hd0)
```
and then quit grub
```
quit
```
 
This should re-install grub. If windows still can't boot, then I think you'll have to re-install it: it probably was corrupted by the Ubuntu installation (contrary to what I said before!!). Defragging normally doesn't do a great job when run only once, especially on a really fragmented system: you need to run it a few times. This *may* have been the problem. Also, just for reference, when you partitioned the disks in the Ubuntu installation, the automatic options resize your existing partition, and then put linux in the remaining space, which is why you ended up with a 50GB windows partition and not a 50GB linux one.

### Post by ugm6hr on 2007-08-15
> **gentix said:**
> If I do end up wanting to completely erase my hard drive, does anyone know how to do that by chance? heh.

This is very easy.

1. Use GParted (either on GParted LiveCD - a useful tool to have, or use Ubuntu LiveCD and load GNOME Partition Editor) to *delete *all partitions on /dev/hda.   Then *apply*.  Then *add *a single primary partition of 20GB (for Windows as NTFS or Fat32) at the start of the HD.  Then *apply* again.  Leave the rest of the HD unallocated / unpartitioned.

2. Then reboot with the Windows install CD, and select the partition you created (20GB) to install it in.  It will offer to reformat the partition prior to installing, and I would recommend you do that.  It will offer NTFS or FAT32 - make your pick (I use NTFS).  Then install it (and wait 4 hours while Windows does its thing).

3. Then reboot with the Ubuntu LiveCD.  And when it's loaded, install with the ***Guided - Use largest continuous free space*** option.

Then it's done.  The bonus with this method is minimal messing around with manual partitioning (which can be confusing, if you don't understand it).  

If you wanted to leave a bit of unused space on the HD for a future separate */home* or dual OS "Storage" partition, you could do that when you create the initial Windows partition in Stage 1 (just create a 20GB NTFS for Windows, and an extra primary FAT32 partition after that - Ubuntu will then just fill whatever is left when you install it).

Hope that helps.
