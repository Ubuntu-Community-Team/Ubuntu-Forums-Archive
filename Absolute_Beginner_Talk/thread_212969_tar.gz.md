### Post by scxtt on 2006-07-10
a "tar.gz" file is a 'gzipped tar archive' ... which means a directory was "tar'd" (made into an archive, basically making a folder of multiple files a single 'tar' file which can be extracted w/ the same folder structure) and then gzipped (for compression) ... so you'd unzip it, then untar it -- the gui can generally do both ... then there should be 'make', 'configure', and 'install' scripts in the new folder you have after unzipping/untaring ...

but as Kilz says, it's easier to use synaptic ...
