### Post by Sef on 2008-01-10
Instead of installing do a mem86 test.  It will check your memory.   Let it run overnight (about 8 hours.)  Maybe you have some bad ram or a bad motherboard memory module.  I ran the test and found that it stuck on test 8: the memory module test.

I had to use the alternate disk and even then it took 3 tries to get it to go right to the end of installation. Also it took much longer than the 20 minutes or so that the website  talks about. In did seem to spend a very long time  on base system at 31% but the occasional slight flicker of the hard disk light suggested that something was happening.

Although I have looked at Linux a couple of times in the past, this is my first successful installation and has been done on the minimum of knowledge and understanding. So all I can say is that it is worth trying again and being patient.

Experience so far with the new system is that it runs pretty slowly on the old machine. I am going to upgrade the memory to 3 x 128 MB - the maximum my motherboard will accept. Neither of these upgrades is particularly expensive.
