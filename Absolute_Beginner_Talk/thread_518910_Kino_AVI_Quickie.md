I read this on another site.

[I]AVI files contain compressed video, and to display the contents the software must decompress the video data. There are many different compression schemes implemented, so many different software decoders are required.

Kino has only one codec for handling the DV compression scheme. Therefore it can only display AVI files containing this particular type of video data.

Earlier versions of Windows, on the other hand, come with a lot of codecs, but the DV codec is missing. Therefore these Windows systems can display most AVI files, **with the exception of Kino's and dvgrab's AVI files.**

The easiest way of installing the missing codec on a Windows system is to install a demo version of an arbitrary video editing software, which will also install the missing codec. The codec will usually work even after the demo version has expired.

[/I]

If I am understanding that right, it means that if I do a project in Kino and upload the avi files to my web sever, that everyone with a Windows box will not be able to view it.

If this is to be the case, better I ditch Kino now and do it in Windows or another app for Ubuntu that will play no matter what OS a user has.

### Post by FakeOutdoorsman on 2007-08-07
Kino is basically a video capture tool with very simple editing capabilities and will save to dv AVI files.  I have used Kino to capture DV video and successfully imported it into Adobe Premiere Pro on Windows XP.  You may have to do some testing to figure which types of Kino-made DV files your Windows programs can work with.  If I remember correctly, Premiere didn't like Raw DV, but Kino can save your video files as several types of AVI (type1 or type2).
