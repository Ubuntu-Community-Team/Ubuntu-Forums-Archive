### Post by ericmarceau on 2007-02-13
Under Dapper Drake,  you can install the gtk libraries that allow you to operate KDE-targetted apps under GNOME.  My desktop is Gnome, but I run Amarok for music, a KDE-optimized player.  I haven't worked under a KDE environment, but my guess is that the main difference is single-click forcus (a la HTML URL) versus double-click (a la Windows 98).

I have not experienced conflict of running KDE apps under GNOME, for the reason stated above.  No crashes, no lockups.  All my problems stem from some peculiarity within gxine which periodically freezes up the machine and need a power-cycling to get up and running again.  I use Totem most of the time, but sometimes I find gxine's suite of codecs more comprehensive to handle formats not addressed by ToTem.
