### Post by tribble222 on 2007-03-10
nvtv doesn't support very many video cards so it looks like yours is unsupported.  If nvidia-settings is working correctly then you should see a "X Server Display Configuration" section that allows you to dynamically change your configuration without the need to restart X to apply the changes.  I don't know much about it, but it is possible that it requires the binary drivers from nvidia.  The best way to install these I've found is to use the envy script.  And you will need to change 

Driver		"nv"

to

Driver		"nvidia"

in your xorg.conf once you have them installed.

Your /var/log/Xorg.log could be called a number of things.  Type into the terminal

ls /var/log | grep Xorg

to get a list of files in that directory containing Xorg in the name.

### Post by tribble222 on 2007-03-11
Normally you can get the envy script here [http://albertomilone.com/nvidia_scripts1.html](http://albertomilone.com/nvidia_scripts1.html) but it appears to be down.  You might be able to find it somewhere else.  It automatically downloads and installs the latest nvidia driver for your card.  All you have to do is install envy and then type "envy" into the terminal and follow instructions.  Say no when it asks you if you want it to update your xorg.conf and then add the line I told you about above.

If the new driver breaks your gui you can run envy again from terminal and choose uninstall and it will remove it for you.

I'm not sure exactly what the problem is from your log, but it looks like

it can't find any device on analog output a or b so then it defaults to using just the dfp.  Are you sure you have whatever it is you're trying to output to connected?  The best thing to do is keep trying different configs and different options.

The following is a link to the nvidia linux README page that lists all the possible config options.  Everything you need to know to get your xorg working is in that README.  It's just a matter of reading through it and really understanding what is going on.  [http://http.download.nvidia.com/XFree86/Linux-x86_64/1.0-8774/README/appendix-d.html](http://http.download.nvidia.com/XFree86/Linux-x86_64/1.0-8774/README/appendix-d.html)

The workflow I used to get my setup working was:

1) Read stuff in the README
2) Try new option in xorg.conf with gained knowledge
3) X still doesn't work, check xorg log to see what problem is
4) go to step one with more information on what is going wrong

Good luck!

Edit: P.S.  Don't worry about the /dev/wacom or font errors in your xorg log.  Mine has those too and they don't seem to cause anything to break.
