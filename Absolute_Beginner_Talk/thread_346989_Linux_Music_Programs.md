### Post by Greykrrr on 2007-01-26
> **Stavro said:**
> but there definitely seems to be much more of a choice on Windows and Mac regarding music score writing packages and software synthesizers.

That used to be more true than it is today. Although I know if nothing in Linux that can match the quality of scores you can make with Finale or Sibelius. You can try Rosegarden which seems to be the editor of choice for scores.

As for synths linux weakness is its poor support for VST instruments, although this is more Steinberg's fault than any linux hacker's. I have never successfully gotten vst plugins or vsti's working but things might be different for you.

For native linux processing I would suggest Muse as a good multitrack sequencer (that's what I hear anyways. I don't use it myself) and some of the Jack related stuff. Jack is one of the jewels in audio production on linux. It is unrivalled by anything on windows regarding routing capabilities and its minimal impact on latency!

I am not at my home computer right now, so I can't give you the link directly but there is a music ubuntu page on how to set up ubuntu for audio work and otherwise you can always visit planet CCRMA for inspiration. CCRMA supplies packages for 32 bit Red Hat systems but it can work fine as inspiration base to see what software is out there.

And finally the Linux Audio . org ([http://linuxaudio.org/en/index.html](http://linuxaudio.org/en/index.html)) for news and various other bits.

### Post by FluffyElmo on 2007-01-26
I have to be honest and say that your biggest problem with either of those machines performance wise will be RAM. Xubuntu has a lower memory footprint than regular Ubuntu so I'd start there, but even then a memory upgrade would be well worth it in increased performance. 

Getting on topic, for music composing, I think Linux is has very solid programs which compete well with Windows alternatives...and they're all free. Linux programs in this area tend to be  more focused on specific tasks instead of the everything-but-the-kitchen-sink approach of some commercial packages but you can still get excellent results and may prefer this approach.

Try the programs christhemonkey suggests above on Xubuntu and if you like them then instead of paying for Windows software put some of that money into RAM or a new machine and I think you'd be very pleased with the results. 

Just my 2 cents...
