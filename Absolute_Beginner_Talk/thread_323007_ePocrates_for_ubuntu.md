### Post by misha680 on 2007-01-09
Alright, this will be brief as I am quite tired right now. Basically some things:

1) Palm syncrhonization in general - you should be able to find quite a few resources. [https://help.ubuntu.com/community/PalmDeviceSetup](https://help.ubuntu.com/community/PalmDeviceSetup) is good to start with. Then just go to Evolution, Edit, Synchronization Options... and if I remember correctly this will start the wizard. Also, there is a "Pilot Applet" taht you can add to one of your panels (the big gray bars at the top and bottom of the screen), just right click, "Add to Panel...", type pilot in the search field, and double click on the "Pilot Applet" to add it. 

2) Epocrates - I used to use this program on Windows but as I am currently in the PhD part of my training I haven't needed it since I've been using Ubuntu. However, if I remember correctly I read that this program actually is based on an open standard (I seem to remember it is the same standard as AvantGo, but I'm not 100% sure), but I think it extends the standard in a proprietary way and so I am not sure it is possible to synchronize it on Ubuntu currently (although if you have it on your palm you should try, if I remember correctly there are some options for this when you are looking at the Pilot Applet or in the Evolution Syncrhonization options). The only other thing I can suggest is running Windows XP in VMWare (there is a HOWTO on this in the Faqs, tricks, howtos section of the forums) and synchronizing that way (I currently run Ubuntu only on my machine, but have XP in a VMWare session for a few apps that just don't have linux versions and if I needed Epocrates right now, this is probably what I would do). There may, however, be some other option, but it would probably take quite a bit of investigation on your part.  

In any case, hope my tired ramblings help.

Misha

EDIT: Btw, I would certainly install VMWare Server (free as in beer, but not open source) and XP in VMWare instead of reinstall Windows on a laptop that doesn't have it anymore. In my experience it works solidly, and is really quite speedy if you have enough RAM (512 MB is too little for XP to run in a VMWare server, with 1.5 GB right now in my laptop it works beautifully).
