### Post by cobrn1 on 2007-08-15
Basically, LTS is for people who need something that works and will be supported as it is for the next several years. It's not for you if you want semi-recent features...

Basically, it's good for businesses, but as an average user you'll want to install feisty (newer and better features) and when gutsy comes out you'll be able yo automatically upgrade to it - so you'll always be using a distro with support, but unless you have a good reason for keeping the system in th dark ages (ie, you're a business and as the IT admin you like to keep things simple - new rollouts every 6 months just aren't an option - for the average desktop user they're great though...)

To sum up - you want feisty (you'd know if you had a good reason for LTS and you wouldn't be here asking).

### Post by Max Luebbe on 2007-08-15
Like the others have said:

If you're rolling out a server for a business that needs to be totally bulletproof, stick with LTS because the packages will be more mature and you'll only be doing security updates
Bottom line: stable/infrequently changed features

If you're a desktop user, you'll want 7.04 or gutsy when it comes out in the near future because hardware support keeps getting better, newer kernels, newer software etc etc.
Bottom Line: latest features currently available

LTS is good for businesses, which will not want frequent upgrades, especially on servers. Knowing that you can install one version and get security updates for 3-5 years (instead of 1.5 years) is reassuring for the enterprise.
