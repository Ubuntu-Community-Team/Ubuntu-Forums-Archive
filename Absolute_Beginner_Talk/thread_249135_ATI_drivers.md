Well, when you run a command such as that on a file you have to tell the computer where the file is. It doesnt know of course. So, you either have to say 

 chmod +x /this/path/to/the/file/ati-driver-installer-8.26.18-x86.run

Or else you have to move to the directory that the file is in by using the cd command (change directory). 

An easy way to to this is-
If the file is on your desktop then also open a terminal. Drag the file into the terminal and drop it. The path to that file will be shown in the terminal. Now, to change to that directory, simply hit the home key and the cursor will move to the beginning of the path. Then type cd and hit the space bar. Then hit enter. Voila'! you are now in the same directory as the file and the command you tried should work now.

### Post by RudolfMDLT on 2006-09-03
Hi!

You're running mesa, you need to fix that first.

Just for the record I live in south africa and so if you're on the other side of the earth where gonna be posting in 12 hour delays! :)

Back to the problem - 

Here is your device section:

Section "Device"
Identifier "ATI Technologies, Inc. RV350 AS [Radeon 9600]"
Driver "ati"
BusID "PCI:2:0:0"
EndSection

I need you to change the "ati" to "fglrx".

You do this as follows, first make a backup copy(always do this):

   sudo cp /etc/X11/xorg.conf /etc/X11/xorg.backup

then you open and edit you xorg.conf file.
 
   sudo gedit /etc/X11/xorg.conf 

you make the change and save the file.

You now need to restart your system - the X system may not load and you will get a Dos like error! do no fear! you need to login in to the text shell and just relpace your new Xorg file with the old one. First press and hold Ctrl + Alt + F1, you can cycle 6 text shells simultaneously F1 through F6, F7 is the X-system. Once logged in you replace the new xorg file like so,

sudo cp /etc/X11/xorg.backup /etc/X11/xorg.conf

and restart the machine,

sudo shutdown -r now

Don't worry about the error I explained above, it very rarely happens but when it does and it helps to know the bash commands to fix it.

All the above assumes you have the FGLRX drivers installed. If this does not work for you then try this post:

[http://www.ubuntuforums.org/showthread.php?t=195845](http://www.ubuntuforums.org/showthread.php?t=195845)

Hope it works out!

I'll check back soon.

Cheers!

PS: I'm running a 9600 Pro, it took quite some work to get it going so stick around! :)
Use the two commands I gave you earlier to check whether it worked.
