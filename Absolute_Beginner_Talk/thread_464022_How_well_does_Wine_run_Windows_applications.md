### Post by Cypher on 2007-06-04
The [Wine AppDB]("http://appdb.winehq.org/") is a good site which catalogs all the applications that run under Wine and perhaps settings that need to be tweaked within Wine to make them run. That should be your starting place.

But, I would also suggest finding OSS alternatives to your applications which will perform better. 

Now if an application can only be run through Wine, the performance in general is quite good. I recently was playing with Macromedia Flash 8 Professional on a Windows XP machine and then got it running on Ubuntu through Wine and couldn't notice any significant difference..

### Post by silkstone on 2007-06-04
I tried Wine and couldn't get it to run much that I wanted, so I now use Crossover Linux from Codeweavers. It costs but you can have a free trial to see if it works.

Older applications - e.g. Photoshop 7 and Paint Shop Pro 7 - work mostly, although some features such as toolbar configuration are not retained on restart.

I'd say that Wine or Crossover are last resorts if you really can't find a native application, but if you look at the compatibility lists there are very few Windows apps that run well.
