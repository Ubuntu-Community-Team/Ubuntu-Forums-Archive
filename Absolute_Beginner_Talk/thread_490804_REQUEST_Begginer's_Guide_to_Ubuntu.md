### Post by bodhi.zazen on 2007-07-03
Thanks starcraft.man ;)

milkolate: LOL, Ubuntu does not *need* to be connected to the internet, all starcraft.man is implying it that Ubuntu without the internet is like to rock but not roll ....

Yes, you can compile all you want. Install build-essential : ```
sudo apt-get install build-essential
```

You may want to look at checkinstall, it makes a .deb. of the source code.

You can also compile with apt-get :

sudo apt-get build-dep <application> # Installs dependencies needed to compile an application.
sudo apt-get source <application>    # Download application's source code to current directory.

Then ./configure && make && sudo checkinstall

[http://ubuntuforums.org/showthread.php?&t=335956](http://ubuntuforums.org/showthread.php?&t=335956)



I am neutral with automatix, it can have advantages, and many feel it makes ubuntu easier and more enjoyable. My only point is where to go for support. I think you will find all the help you could ask for on the automatix forums.

What card do you have and whats the CPU of the comp? Anything above a 4-5 series Nvidia card (or ATI equivalent) should handle Beryl ok, 6+ works good. KDE also provides more customization options than GNOME (I never liked KDE, thats just me).

> starcraft.man:
I think OS should not heavily rely on the internet.. It can stand alone, right?
Can't we compile all of these stuff??

Well, of course it can stand alone.  You can install Ubuntu from the CD. Without ready internet access though every time you want to update you'll have to download the packages to some separate media and put them on your linux box. Same for adding programs, you'll have to get debs downloaded and install separate. The package managers that connect to the internet make this process so easy and simple that I simply can't recommend using it without net access. IMO, downloading all those updates and programs when you need em is a pain.
