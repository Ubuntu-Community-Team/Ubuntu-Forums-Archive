The biggest roadblocks I have come across all involve hardware and being unable to easily find a driver or one that provides full functionality. I have currently solved all those problems using Linux, but paradoxically I am having problems with Windows now.

My latest roadblock came about after upgrading from Windows 98 to Windows XP. Coincidentally, one of my reasons for trying the upgrade is that I couldn't get ANY wi-fi adapters I tried working under Windows 98 (although ALL the ones I tried worked fine in Ubuntu!). Given that I did this upgrade on hardware that is over 7 years old, I expected I might have somewhat slower performance but I didn't expect to lose any functionality in hardware that already worked. I was in for some surprises. Examples: under Windows XP my printer lost full functionality, I had to go through contortions disabling Windows XP's ACPI drivers to get all of my hardware working, and DVD playback in Windows XP is either marginal or bad. Each of those things worked perfectly before I did the upgrade and I'm now considering either rolling back to Windows 98 or just deleting the Windows partition altogether. 

My point is that whenever one moves from one OS to another, there may be issues to overcome that can be frustrating. It doesn't even have to involve a move as radical as moving from Windows to Linux; you can encounter problems just moving within the same family of operating systems. If your system already "just works" and that is most important to you, you probably should enjoy it as it is and leave it alone. If you want to try a new OS (or upgrade and existing one), you'll need to be patient and aware that some stuff might be tricky to set up.
