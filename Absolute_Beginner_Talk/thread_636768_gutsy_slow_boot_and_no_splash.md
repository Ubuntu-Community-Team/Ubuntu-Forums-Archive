### Post by neotasic1 on 2007-12-10
The link Spiderbatdad has provided regarding editing the menu.lst file to remove certain of the default entries is the same solution I used for this problem several weeks ago. This is a known bug in Gutsy.  It works fine on my system AMD64 bit (the system now boots quickly - but you will get streams of text running down your screen during boot up rather than the Ubuntu splash screen - which doesn't worry me at all - I don't know about your preferences)  

Back up your menu.lst file first before making the suggested changes - that way you have a fallback if this solution does not work for you.

Incidentally, there is another solution which will give you the splash screen and quick boot up (has to do with screen resolutions nominated for the splash screen, and is somewhat more complicated - but in my case, although the solution worked in that I had a quick boot and a splash screen, it prevented access to the text terminals normally accessible by CTRL+ALT+F1 to F6 - a feature I believe to be far more necessary in Linux than a pretty splash screen).

### Post by hakan69 on 2008-01-07
I had the same problem, no splash screen and impossibly slow performance, System Manager showed CPU at 100% all the time.
My computer is rather basic, with on board integrated graphics. When I checked the BIOS, I found that only 512 kB was allocated to video memory. I changed that to my maximum of 8 MB and that did the trick. I have installed StartupManager to set the resolution of the splash screen.
I have however not been able to change the resolution and font size at the login screen.
