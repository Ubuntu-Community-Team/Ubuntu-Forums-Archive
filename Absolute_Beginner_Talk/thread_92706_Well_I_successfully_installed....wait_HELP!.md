Fdisk is a disk formatting tool.  If you have to ask what it is, I would hesitate to recommend that you use it.

Windows can't read any filesystems that aren't either generated by windows or at least windows-compatible.  If I were you, I would delete your present ubuntu install and start again.  

When you reinstall DO NOT accept the default partitioning option (one big partition of ext3).  Instead, use custom partitioning.  You will be able to specify partition sizes and filetypes.

When you go into the partitioner in the installer, you will see all your disk partitions as they are.  leave your XP partition alone, if you want it to stay there.

Then, delete the ubuntu partition and repartition this way:

allocate a partition twice the size of your RAM as a swap partition.

create a 20 GB ext3 partition for /  (yes, forward-slash:  that's the root partition).

Create another 40 GB ext3 partition or so for /home 

Then, take the remainder and create a partition out of that in FAT32.  That way, both Windows and Linux will be able to mount, read, and write from it.

### Post by Brunellus on 2005-11-21
[QUOTE=EriCKY]have done all you said and i still see no D: in XP...[/QUOTE]
if Ubuntu is sitting on what used to be D:\ then you *won't* see it in XP, because XP can't read/understand the filesystem on it.

XP will only read two filesystems:  NTFS (its 'native' filesystem, limited compatibility with linux--as in read-only) and FAT32 (the last iteration of MS-DOS's filesystem, which has full read/write compatibility in linux).  Any other disks formatted any other ways simply do not exist for Windows. You can't see them, you can't mount them, you can't read them, you can't write to them. Nothing. 

So if this entire exercise was an attempt to make your old D:\ partition visible in Windows again, I'm sorry to tell you that it's not possible.  Whatever data you had on that drive before you decided to put ubuntu on there is well and truly gone now, since Ubuntu had to format that space in its own filesystem to install itself.

### Post by aysiu on 2005-11-21
Windows doesn't play nice with Linux. If you do a search on Google for the word *Linux* on the Microsoft webpage, the first result is "How to remove Linux and reinstall Windows." I'm not kidding.

Windows knows there's a partition there. It just doesn't want to acknowledge it. If you go to the disk information in the control panel, it'll show a 200 GB "healthy partition," but you won't be able to read the data from it or write to it.

Windows doesn't play nice. It'll read itself but not others.

Linux will read NTFS and read from and write to FAT32. It also, of course, reads from and writes to its own native formats (Ext3, Reiserfs, etc.).

If you want to share data, shrink the 200 GB partition and make a FAT32 partition to share between the OSes.

In the end, it'll look like this:

/dev/hda1  NTFS    20 GB
/dev/hdb1  FAT32   180 GB
/dev/hdb2  Ext3     20 GB
