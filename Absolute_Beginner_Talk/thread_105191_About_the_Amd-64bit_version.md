### Post by d1337 on 2005-12-17
<Disclaimer>You will likely get mixed replies on this question.  I can only share my experience</Disclaimer>

I have a laptop with a 64bit AMD Athlon procesor.  Because I had the hardware, I installed Breezy 64.  I ran with it for about a month, but ended up going back to 32 bit.  My machine seemed lightening fast with 64bit installed on some apps, but with other it seemed to be comparable with my 32bit installs on machines with similar specs.  My pitfalls were Java, Flash and codecs.  Alot of folks can live without the above functionality, but I've always felt that if I want to hit a site that uses Java or I want to play a Flash game, I'd like to be able to do so.  AFAIK, there are workarounds for every issue I had, but they're not entirely pretty.  I would have loved to help getting 64bit right, but I don't know enough and I have too many other things going on...so in the end I went back to 32bit.  (Pardon the redundancy).  You can run a 32bit OS on a 64bit platform.  Unless you have the time/desire/knowledge to work on functionality, I would suggest installing 32bit.  But that is **just my opinion**.

### Post by psusi on 2005-12-17
As a general rule, open source software is written to be portable, so it is availible in 64 bit versions to run on your 64 bit OS.  Proprietary things like win32 codecs and the macromedia flash plugin don't have native 64 bit builds.  It is possible to get the 32 bit programs to run on a 64 bit system, but it takes a good deal of shoehorn and elbow grease.
