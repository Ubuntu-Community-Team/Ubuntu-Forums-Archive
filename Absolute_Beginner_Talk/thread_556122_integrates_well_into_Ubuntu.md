### Post by Rasitiln on 2007-09-20
Applications that integrate well into Ubuntu are normally GTK based. Gnome is written with GTK for its graphical environment thus installing GTK based apps require no extra libraries for their gui to work (most the time) also apps that are GTK based also skin well with Gnome (natively I think) 

The same for Kubuntu except for the fact it runs KDE which is QT based thusly QT based apps are said to integrate well with Kubuntu for the same reasons as above. Easier to skin, don't require extra libs etc. Ever try to install a QT based app like amarok in Gnome? You have to download something like 40mb worth of files (assuming thats for first QT based app you have installed) in order to run it and it starts up numerous other processes to monitor it which run alongside gnome apps that do the same thing. 

For more information on QT and GTK try these links

[QT]("http://en.wikipedia.org/wiki/Qt_(toolkit)")

[GTK]("http://en.wikipedia.org/wiki/GTK")
