### Post by CautionaryX on 2007-07-13
The support period means that the Ubuntu developers will continue to develop patches and security fixes for that specific version of Ubuntu, even if it is not the most recent version, until the support period is up. Any version of Ubuntu will still work after the support period however, there will not be any patches or security fixes for that specific version.

Microsoft does the same thing. Windows Vista may be the most recent version of Windows, however, Microsoft still releases bug fixes and security patches to Windows XP. Eventually they will stop supporting Windows XP as they focus more of their development on Vista and Vienna.

Hope that clarifies it a little.

- CautionaryX

### Post by macogw on 2007-07-13
The main difference is that there's much better hardware support in 7.04 than 6.06.  Other than that, a lot of things have gotten easier like installing codecs, 3D desktop, different configurations, wireless, etc.

Maybe you're a bit confused about this upgrade thing.  There are 2 ways of upgrading.  You can keep using Feisty for 2 years with security upgrades if you want, but you'd still be using Feisty, and then yes, you'd have to upgrade to keep getting security updates.  Every 6 months though, the updater will ask if you want to upgrade to the newest release.  If you tell it that you do, it'll upgrade you to the next version that has 2 years of support, but if you want to upgrade again 6 months later, you can, for free.  It's like getting to go from Win95 > 98 > Me > 2k > XP > Vista without paying anymore or needing a CD.  It all goes through the update manager.  That's what a lot of do because we want the new stuff.  Others don't feel like upgrading and aren't concerned with having new stuff, so they use an old one until it's no longer supported, then install whatever the new one is.

### Post by mgmiller on 2007-07-14
The main "under the hood" things that change with each release are the newest stable versions of gnome and xorg, as well as a newer kernel with improved (usually) hardware support.  For example in the upcoming 7.10 with its newer xorg, there is supposed to be hot swappable monitors and improved multi monitor support.  Also a possible total rewrite of the xorg system may do away with xorg.conf and all it's confusion.  I know there was a big difference with nvidia between 6.06 and 7.04 as regards multiple monitor use.  It's all controlled by a nice gui in the Nvidia Settings program installed with the restricted driver.  There are many other usability changes as the OS matures as well, like automatic codec installation.  .
