### Post by kmashraf on 2007-04-30
This is how my upgrades went. From Dapper to Edgy, smooth. From Edgy to Feisty troublesome but ah got it done with a few rough edges left. This is on a AMD 2200 CPU, ASUS motherboard with first generation nVidia nforce chipset, 256 MB RAM, 20 + 10 GB IDE HDD's, Atheros based PCI wireless NIC by Compex dual booting Kubuntu and XP. By the way I also had Automatix update my original Dapper install with the nVidia prop drivers
When I did the upgrade from Edgy to Fiesty it installed the latest & greatest nVidia prop drivers. This broke my X. In as much as that video was corrupted and X would go into a restart loop. A hunt here in the forums found me this [http://ubuntuforums.org/showthread.php?t=422918&highlight=Fix+upgraded+to+fiesty](http://ubuntuforums.org/showthread.php?t=422918&highlight=Fix+upgraded+to+fiesty).
When I uninstalled the nVidia driver (kernel level modifications are made) it also threw my networking awry. 
The dependency hell. But after a bit of nosing around and finally installing the nvidia-glx-legacy package sorted that out.
One minor problem I am continuing to have is that I have not been able to start the wireless NIC during boot time if I use encryption. I have to use wlassistant to connect manually once my desktop makes its appearence.
In this regard in Fiesty the KDENetwork applet can be a pain cause, first it does not automagically connect Also if I don't exit it, it will continually attempt to connect and consequently drop my network connection enabled through wlassistant. 
By the way I had to spend quite a lot of time doing the upgrade due to the above problems.
The reason for going through this pain is that I had all the earlier versions setup the way I like it and didn't want to loose any of it. Also I get a good handle on troubleshooting Ubuntu / Kubuntu. I make a living primarily offering GNU/Linux services.
