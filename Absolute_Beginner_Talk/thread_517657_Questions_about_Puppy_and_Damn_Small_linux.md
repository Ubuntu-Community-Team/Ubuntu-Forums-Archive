### Post by ron999 on 2007-08-05
Hi
These two are complete OS's. But they are minimalist ones, they just have one of each app and no frills.
The intention is that you install them onto a USB stick or a credit card size CD so that you have "an OS in your pocket" to use when on the move, eg in an internet cafe or at school.
I haven't used Puppy but I have used DSL.

It's very easy to try out DSL.
You download the iso from their website (just one file).
Then you burn it to a CD-R or CD-RW.
Then you boot your PC from the CD.
When it fires up you get the prompt boot:
You just type **dsl** and hit enter.
Then you can run DSL straight from the LiveCD.

It's better though to run the DSL CD from RAM. With this method it loads DSL into RAM and runs very quickly.
To do this, at the prompt type **dsl toram** and hit enter.

There are some reviews at distrowatch.com
This is the link to the DSL website:-[http://www.damnsmalllinux.org/]("http://www.damnsmalllinux.org/")

### Post by Tux Aubrey on 2007-08-05
I have used both and like them both.  They use minimalist desktops and lightweight applications.  For that reason, they don't look as "polished" as something like Ubuntu or Fedora or SUSE.  Minimal desktops are great if you know how to use and configure them - but quite scary and frustrating if you don't.

They are both at the very lightweight end on the Linux spectrum (at least for graphical desktops).  DSL will run on almost anything - back to a 486 with 32Mb of RAM, I believe.  Neither would have much difficulty with, say, a Pentium III and 64Mb.

DSL is only a 50Mb download and Puppy is around 140Mb.  DSL is the better choice for very old hardware because it uses an older (but updated) version of the Linux kernel.

If you have a P4 with 128Mb or better, I'd actually recommend Mepis antiX - it has one of the minimal desktops (fluxbox) but is set up for new users so there is an opportunity to learn and play before you get terribly frustrated. 

antiX, Puppy and DSL can all be used as LiveCDs/USBs but can also be installed on a HD.

There are other small distros around too - dyne:bolic, CPXmini ....

The site in my sig has more info (and links) on these and other distros, big and small.

### Post by Blofeld on 2007-08-05
I have used DSL and am currently using Puppy.:-({|= You may call me "the master".
DSL is **very** frugal whereas Puppy is surprisingly frilly.  DSL uses a minimalistic windows server which doesn't allow you to configure the frame rate, which can lead to the screen flickering a lot.  DSL also follows the mantra that it ought to fit onto a 210 MB "business card CD-ROM", which I consider an interesting but also somewhat unhealthy restriction.  It's also intended primarily as a live-system (i. e. to be run off a CD rather than off the hard disk).
Here's my rule-of-thumb matrix to minimal Linux distros:
386, older 486, no CD-ROM:  Linuxes without GUI
last-generation 486, first generation Pentium (without MMX), CD-ROM:  DSL
Pentium with MMX, PII:  Puppy
PII, PIII:  Xubuntu
