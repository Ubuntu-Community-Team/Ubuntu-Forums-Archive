### Post by GreenHawkIA on 2006-03-02
Right now I use a Dell Latitude D505 laptop running Ubuntu 5.10.  I use network-manager as a graphical way to manage my wireless connection - an Intel ProWireless 2200bg card.  My university recently switched to a PEAP-MS-CHAP v2 protocol to access their wireless network, and left us Linux users in the dust.  The math department worked out this - [http://www.divms.uiowa.edu/help/wireless/linux.html](http://www.divms.uiowa.edu/help/wireless/linux.html) - tutorial for their linux computers (mostly Red Hat) to access the wireless network, but it is mostly unsupported, so I have a few questions about it.  If I follow these instructions, will it hose my wireless access to every other network?  Will I still be able to use a graphical interface to manage my networks (I travel and switch them all the time)?  Is there an easier way to implement this protocol?
Thanks in advance for anybody's help on this.  Feel free to PM or IM me if you have more questions

### Post by dickohead on 2006-03-02
It will not affect other connections, as when using your wireless you need to use extra services, as well as a key on top of everything you normally do. To use the other networks you access just do it the same way you always have.

But if anything does happen, there aren't too many steps to undo to return to normal.
