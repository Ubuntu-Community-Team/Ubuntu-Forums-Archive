### Post by banewman on 2007-09-23
The desktop being to "big" is because you haven't got the right driver for your video card. A common issue for first time linux users and a great way to start feeling your way around the OS. You need to find out what video card you have and then go to restricted drivers in the main menu and install it.
There should be an option under - system - preferences - hardware information - that will tell you the video card you have.
To add a new panel - right click the panel and choose preferences. Gives you lots of options.
Envy is not recommended. Synaptic package manager from the menu is a better option - packages there are there because they are stable.
If you download a theme from gnome-look and put it in your home folder then go to - system - theme - and point it to the zipped file you downloaded it will install the theme. :lolflag:

### Post by ayenack on 2007-09-27
This worked for someone else a while back. Try only having the 1280 by 1024 res in xorg.conf and delete all the others. Don't do this until you've made a backup of xorg.conf.

Like this. You can paste this into terminal or type.
**sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf_backup** (make sure to use capitol "X" in "X11")

If you end up at command line and need to restore xorg.conf do this.

[B]sudo cp /etc/X11/xorg.conf_backup /etc/X11/xorg.conf
[/B]
Good luck. ayenack.

### Post by ruibernardo on 2007-09-27
Hi ThRixXx, 

this might be because you have a wrong frequency for you monitor. I suggest you to install xresprobe and check that:

```
sudo apt-get install xresprobe
sudo ddcprobe
sudo ddcprobe | grep monitorrange

monitorrange: 30-71, 50-160
```Take a note of those values (the first pair is horizontal, the second is vertical refresh rate) and compare them with the ones that are on the xorg.conf file in **HorizSync** and **VertRefresh**. Change them if needed. If that worked, your done. 

If not, then reconfigure the xorg.conf file the right way, run 

```
sudo dpkg-reconfigure xserver-xorg
```Answer "nvidia" and select the resolutions you want (and you know for sure that work) when asked. Choose the default answer for the questions you don't know how to answer. 

Then edit the file xorg.conf and check if the values **HorizSync** and **VertRefresh** corresponds to the result of xresprobe command you executed previously.

Restart the computer. You should have a "sane xorg.conf" working now.

### Post by ayenack on 2007-09-27
By ThRixXx
> And then it should show me values
monitorrange: 30-71, 50-160


If they are the results shown by running **sudo ddcprobe | grep monitorrange** command. Do this.

**gksudo gedit /etc/X11/xorg.conf**

And make section monitor look like this.

Section "Monitor"
Identifier "7F-BK"
Option "DPMS"
HorizSync 30-71
VertRefresh 50-160
EndSection

This should give the correct refresh rates.
