### Post by l951b951 on 2007-02-02
What is the difference between Synaptic Package Manager = **System->Aministration->Synaptic Package Manager**   
vs **Add/Remove Applications= Applications->Add/Remove**?  

When should I use one instead of the other?

---

### Post by meng on 2007-02-02
If you are comfortable using Synaptic Package Manager, I'd recommend that one. Add/Remove may have improved recently, but previously it was always the poorer cousin to Synaptic. When both work as intended, they are more or less interchangeable.

---

### Post by RomeReactor on 2007-02-02
I would say that while through Synaptic you can get all available packages from the repositories, Add-Remove only offers a selection of those packages as "recommendations"; it's intended to be used if you want to browse through some offerings, but not spend half an hour at it (already Edgy is close to 20,000 packages, if i'm not mistaken) :mrgreen:

### Post by irish_flu on 2007-02-02
I use Synaptic if I have some idea of what I'm looking for, and Add/Remove programs if I have absolutely no idea what I'm looking for (just browsing for something cool).

### Post by dr_d on 2007-02-02
Synaptic keeps a log of all changes that were made. Not sure if AddRemove does this.

It's handy to have a log if you want to make sure that there are no unwanted packages on your system at a later date.
