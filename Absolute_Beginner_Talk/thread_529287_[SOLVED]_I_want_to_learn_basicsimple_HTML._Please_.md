### Post by ericesque on 2007-08-19
if you want to learn HTML and want to do it right, you better count on learning CSS as well.   It would be senseless not to.

Whom ever you quoted about not using w3schools is wrong.   It is an excellent resource to start diving into HTML, CSS, and other web technologies.  I've used w3schools to learn the basics of XHTML, CSS, javascript, and XML.

You have to understand that the first resource you use will not be the last.  As you get more comfortable with HTML and CSS,  you will find there are never ending levels of complexity that you can master if you decide to.  There are great sites like [http://www.alistapart.com/](http://www.alistapart.com/)  which are dedicated to the nuances of CSS layouts.

I say start with w3schools.  You will gain a basic understanding about the tools you want to use.  Then as you pound out your hand coded site, google will become your best friend to finding out how to solve specific problems you encounter.  

I'll try to remember to look up some additional resources for you--my links aren't with me at the moment.

Also, I'd recommend going straight to XHTML instead of HTML 4.  They are so similar that even if you decide that you want to use HTML 4 for some reason, everything you have learned will still apply.

Good luck.  Feel free to PM or IM or email me with questions.

$ mkdir ~/tmp
$ cd ~/tmp
$ wget apache.org
$ ls

You should see an index.html file. Play with it a bit, try to make a change, then load it into your browser.

Not only do these Apache folks provide the most widely-used http server on the planet (along with much other high-quality software) they also have a clean, simple web site.

There are subparts written by different folks so you can examine a variety of styles.

### Post by mdebusk on 2007-08-27
> **hanzj said:**
> I wonder whether it would be a waste of time to learn through those books when I have my eyes set on Liz Castro's 2006 HTML, XHTML and CSS book. 


My two cents: skim through the books and note which ones, if any, have the HTML tags in uppercase.  Take those back to the library.

The reason is, HTML doesn't care about case, but XHTML does. **XHTML requires lowercase tags.** (It's conventional to use uppercase in HTML for sake of readability in a plain text editor, so HTML books will teach it that way.)

Incidentally, which editor are you planning on using?

### Post by mdebusk on 2007-10-05
> **hanzj said:**
> I think I  am understanding the html part of web design. I still haven't poked around CSS, although I have coded my webpages for that.

Web design is a different animal than Web coding. What makes a Web page a Web page is not the same thing as what makes a Web page attractive, usable, and accessible.

You might want to read [The Big Red Fez](http://www.sethgodin.com/fez/) by Seth Godin. It describes my favorite way of thinking about Web page usability.

The Web site [A List Apart](http://www.alistapart.com/) is good too. Lots of advanced stuff, but there are some newb-friendly articles. Note how complex their pages *appear* but how quickly they *load* and *render*.

And for sheer *Schadenfreude*, there's [Web Pages that Suck](http://www.webpagesthatsuck.com/). :)
