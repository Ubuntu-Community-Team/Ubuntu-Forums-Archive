### Post by Terl on 2007-04-20
Easy fix..

Do you know how to edit the file?

If so:

In this section here:  ```
Section "Monitor"
	Identifier	"Generic Monitor"
	Option		"DPMS"
	HorizSync	30-60
	VertRefresh	50-75
```

Enter your monitor's horizontal synch and vertical synch ranges.  

The lines that look like this:  "1024x768" "640x480"  change them to include the resolutions you need.

Or.  do the sudo dpkg-reconfigure xserver-xorg.  Answer everything like you did before but when you get to monitor pick advanced.  Enter the horiz and vert synch ranges and then pick all the supported resolutions you need.  Be careful with the horiz and vert ranges.  Do NOT guess at them.

### Post by gionas on 2007-04-20
It wasn't that complicated... I upgraded from Dapper to Feisty and lost the ability to set my resolution to the 1600x1200 native to my Samsung monitor. Tried all the advice I could find on these boards and elsewhere, nothing worked. I decided to see if there is anytihing new from NVidia... and found recently posted updated drivers. It looks like they have updated them for the new Linux kernels. Here is the link:

[http://www.nvidia.com/object/unix.html](http://www.nvidia.com/object/unix.html)

My configuration is a 3 year old Toshiba Satellite, P4 HT 2.8 GH, 1 GB RAM, 256 MB NVidia GeForce FX. I downloaded the Latest Version: 1.0-9755. I also have all I need to recompile the kernel (maybe this is needed for installation?). Well, I didn't recompile the kernel, but I had to turn off X server, and go to a the text command prompt. From there I ran the app I downloaded from NVidia. It took about 3 minutes to check for precompiled kernel modules on their FTP server, then decided it needs to compile locally. Compilation ran smootlhly in about 1 minute or so. The installation app asked if I want it to change xorg.conf. Sad yes, and ran startx... That's it. I'm back to my 1600x1200 screen resolution.

1.  clean install Feisty

3.  at the boot prompt "sudo apt-get install nvidia-glx"
