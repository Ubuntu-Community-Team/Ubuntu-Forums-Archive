### Post by SteveHoffmanUK on 2007-08-02
I don't regard myself as extremely tech savvy. Had installed Windows 98, XP a few times, do a few websites using a WYSIWYG package, and all the usual computer applications, but nothing fancy.

Started trying Linux (Suse) a fews years ago, but had problems with hardware not being recognised; used it for taking notes on an old Dell laptop, and that was OK. So then I heard about Ubuntu, downloaded and burned an ISO CD, plugged my digital camera into it not expecting much and - bling! - up it came and asked me what I wanted to do with the images. I was hooked. Got it working on the Dell, then tried dual booting on my desktop. Now I only use XP for one application, using a virtual machine within Ubuntu Dapper. 

My advice is like the others: take it slowly, don't burn any Windows bridges. Start with the Live CD, try it out, see what works, what doesn't. If it mostly works, then go to dual boot and use Ubuntu for what does work, Windows for what doesn't. Then tackle the problems one by one, using this forum, and eventually you can get almost everything to work in Ubuntu and you can then ditch Windows. No virus scanning, no firewall, a speedy, stable work environment - worth the effort!

You **may** have to buy the odd bit of kit to replace something made by a manufacturer that is not Linux-friendly, but not much: just check the lists here to make sure that what you're buying works for Linux.

### Post by sacherjj on 2007-08-02
>>Now, that's not to say it's gonna be an easy ride. The first week or two may drive you absolutely nuts. But eventually, you'll realize how much crap you put up with on a daily basis when using Windows, and you'll ask yourself, "why didn't I switch OS's sooner?"<<

Ain't that the truth.  When I decided to try it out, I setup two PCs with a KVM.  This kept my old WinXP Digital Audio/Video Editing, Photoshop workstation available, while I learned Ubuntu.  Now I find myself pointing my dual 22" screens at my Dell E520N Ubuntu workstation about 95% of the time.  When I go to Windows, I just get annoyed that the video file I'm trying to move is locked, because Explorer decided that it needed to open it to figure out the properties, rather than just do what I told it to do.  Other than digital media, I prefer doing everything in Ubuntu.  It just works better and is more efficient and stable.

One thing you want to do when installing a Ubuntu install is create a main partition of 10 Gb or so at /.  Create a swap partition of appropriate size (1024 Mb or something).  Then create the rest of the drive as a /home partition.  Now, when you screw up the Ubuntu OS portion (I've done it.)  All you have to do is put in the Live CD, and reinstall.  Let Ubuntu format the main partition, but not the /home partition.  After installing and spending 20 minutes picking the programs you had installed with the Synaptic Manager, everything will be back to normal.  All configurations are in directories in your /home that start with periods.  Since these are on a different partition, you don't have to do anything.  I couldn't believe how easy it was.

There is no registry to deal with or anything else.  The first time i had to do this, it was a blessing, compared to reinstalling WindowsXP, then spending hours reinstalling software and then trying to copy preferences over.  The irony is that you almost have to do this every 2-3 years with Windows, but not with Linux.  It also gave me confidence to do more things, because I knew how to "get back."  This is the same process I did with DOS based PC with Windows 3.1 in the day.  You are never really confident on an operating system until you screw a few things up and have to fix them.

### Post by cmp1988 on 2007-08-03
I'd recommend Ubuntu to someone who isn't tech savvy over someone who's a master of the Windows.  This is because the person who isn't tech savvy would want the thing to "Just Work", while the master Windows-er will want Ubuntu to be just like Windows (I'm trying to say that Windows only people will want it to work just like windows, whereas actual computer savvy people will want to try it out). 

Plusses: 

- Recent Kernel version (meaning more things work on Linux in general) 
- No BSOD
- Super extremely rare chance of Linux just crashing on you for no reason whatsoever.  Just about all crashes are caused by the end user, or an extremely buggy program.
- Fewer bugs than Windows by far. (NO program whatsoever has absolutely 0 bugs)
- Forgiving Boot Loader (Lets you boot Windows, unlike the other way around when installing Windows).
- Comes with a wealth of programs at basic install
- No uninstallable programs (Like IE and Outlook in Windows)
- Lots of hardware works out of the box.  Many mistake Windows as having this trait, however, those people have NEVER installed Windows on a computer themselves.  Seriously, it takes hours to install all the drivers and programs you need, THEN customizing.
- Flying Toaster Screensaver (FREAKING FLYING TOASTERS!! ZOMG!)
- Firefox 2.0 comes default, but can easily be upgraded to 2.0.0.x
- Synaptic Package Manager (Need a program that plays music?  Need those MP3 and DVD codecs?  How about some source code?  No need to worry, Synaptic organizes literally THOUSANDS of different apps for Ubuntu, which you can easily check off what you want, then hit the apply button to install those programs)
- Update manager better than Windows (Better meaning it looks for updates of ALL installed packages, not just Ubuntu specific)
- And the straw that breaks the Windows Camel's back...
- Very little threat of Viruses and Spyware.  This is because New programs usually require the "Root Password" to operate, therefore rendering viruses incompetent.


Now onto a few Negatives about Ubuntu, well, the ones that drive me mad sometimes.

Cons:

- Jerks at ATI give relatively poor drivers.  Apparently, my Radeon 9200 is supposed to be one of the better supported ATI cards in Linux, and that's crap.
-  Most of the time, wireless cards are a @*&#% to work.  Mainly because this tech is heavily guarded by the companies that make the chips for the cards.
- Sooner or later, you may have to use the command line.  I only put this as a negative because we're suggesting Ubuntu to non-tech savvy people.  I like the command line's clean interface and the total control you have over your box.
- Games. This is my biggest problem.  Has nothing to do with Ubuntu or Linux, but the developers.  A couple decent FPS, Frozen Bubble, and Tux Racer (which pwns all, btw) are about all you get when it comes to running games natively in Linux.  
- Major programs that people use all the time.  I'm talking about your Photoshop and Director MX, and anything lots of people need to use.  Office doesn't fit in this one because OpenOffice can handle all the formats that are usually required.
- Steep learning curve for Windows users.  The more you know about using windows, the larger the curve may be.
- Did I mention wireless?  
- IE Only Websites (Wine doesn't count)
- Mplayer Browser plugin (Just not the same when viewing streaming WMV's), this again, isn't the developers of Mplayer's fault, it's the fault of Microsoft for not letting us see the code for WMV's and such.
- Probably more that I haven't found yet.


There's a few Pros and Cons of Ubuntu for starters.   For more information on the problem that is the Wireless Cards, I'll post a link to a site which tells you what wireless cards work with Linux in general, and which don't.  [***Here's that Link I was talking about***]("http://linux-wless.passys.nl/")


There's a bit of info for now, and hopefully everything will work out alright for new Linux users. 8)

ubuntu-desktop is a meta package. it is basically a package that tells apt-get to instal a,b,c,...,x,y,z instead of you having to install each seperately. it is not required for normal usage.

when it comes time to upgrade o the next distro reinstall ubuntu-desktop (which will reinstall the packages you removed that are contained in the meta package) then perform he upgrade.

if you are upgrading via CD then this is not necessary as it will be install with the rest of the new OS.
