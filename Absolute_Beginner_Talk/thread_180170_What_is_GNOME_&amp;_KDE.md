### Post by Fass on 2006-05-21
KDE and Gnome are desktop environments. Like explorer is in Windows - the start meny, the file manager, the desktop. All of it. That's a desktop environment.

It doesn't really matter if you choose ubuntu or kubuntu. You can install KDE and Gnome respectively on either after you've set them up.

I suggest you get both LiveCDs - you don't have to install them, as you can run Linux directly off the CDs. That way you can try both Gnome and KDE before you decide which to install.

One usually says that KDE is more Windows like, and Gnome is, well, it's more different from Windows, but neither are difficult to learn. I used to like KDE, but now I like Gnome. It's mostly a matter of taste which one prefers.

### Post by adamkane on 2006-05-25
Running Gnome and KDE apps at the same time will use up more memory (using more libraries), but most people won't notice. 

If your computer loses performance especially while running amarok, then you need to install the correct kernel files for your hardware to obtain better performance.

Intel Pentium:
```

sudo apt-get install linux-686 linux-headers-686

```

AMD K Series (Duron, Athlon, Sempron):

```

sudo apt-get install linux-k7 linux-headers-k7

```

Reboot.

amarok bugginess is another issue. Some people need to install taglib-1.4 and alsa-oss.
