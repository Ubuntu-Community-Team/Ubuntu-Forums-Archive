### Post by djchandler on 2007-09-01
[aparigraha]("http://ubuntuforums.org/member.php?u=91829"),

It still sounds like a memory contention issue, because of all the tabs in the browser being open when it crashes. Each open tab makes a new memory allocation request. When it's not available, x-server quits. Somehow the memory alloted for the foreground application at the time of the "crash" is exceeding its memory allocation. What's odd is the x-server is crashing, but the kernel isn't. So that says to me the problem has to be in either the graphics card driver, or xorg.conf.

I went back and read the thread from the first, and that's plenty of hardware to run Linux with XFCE in any distribution. I still question the sharing of graphics memory with motherboard memory. I'm a hardware guy at heart. I'll only do the quick and dirty script editing. But when scanning the xorg.conf file, I see there's no option of running any different color depth than 24-bit. I'm wondering if running at 16-bit depth might ease the memory problem until you can solve the memory sharing problem. I consider it a problem only because I would not build my own computers that way, and I build or re-build all my computers. That's 
one thing I'll always change if the resources are available, is install a graphics card that does not share motherboard memory.

One question: have you  installed the nvidia-glx-new driver through Synaptic or apt-get?

The nv driver setting in xorg.conf is correct unless you've installed the "non-free" driver indicated above.
