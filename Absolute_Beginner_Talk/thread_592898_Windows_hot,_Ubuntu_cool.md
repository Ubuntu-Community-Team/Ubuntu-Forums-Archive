You said your work depends on using Windows. Here are two alternatives that I recommend that you may or may not know about.

CrossOver
VMware Server

CrossOver may work on some of the Windows based programs you currently use.

For the rest, unless you need a 3D card for them, Windows Installed on a VMware server would allow you to run some parograms as well. If VMware doesn't work then there is Virtual Box.

Also if you like to get your hands dirty you could always skip CrossOver and use Wine instead.

### Post by ddrichardson on 2007-10-26
> **SunnyRabbiera said:**
> well to be fair linux can eat up some ram too, but the ace in linux's hole is swap memory that takes a lot of burden away from the main memory.That's like comparing apples and oranges. Linux uses memory that's available by design. It's not a design flaw its a architecture decision. Swapping involves the hard disk which being mechanical is likely to increase heat, in any case Linux doesn't use swap memory in the same way as Windows - it's more of a case of use when all else fails.

You don't mention which version of Windows you are using, but if it's Vista then disable aero - it uses an awful lot of resources.
