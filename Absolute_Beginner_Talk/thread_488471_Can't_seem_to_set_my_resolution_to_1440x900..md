### Post by nutz on 2007-06-30
This is the way my xorg file is setup and it works perfectly. I have a 22" LCD with a native resolution of 1680x1050 that requires 60hz...
```

Section "Monitor"
       Identifier      "Generic Monitor"
       Option          "DPMS"
EndSection

Section "Device"
    Identifier     "Generic Video Card"
    Driver          "nvidia"
       Busid           "PCI:1:0:0"
       Option          "AddARGBVisuals"        "True"
       Option          "AddARGBGLXVisuals"     "True"
       Option          "RenderAccel"           "True"
EndSection

Section "Screen"
    Identifier     "Default Screen"
    Device         "Generic Video Card"
    Monitor        "Generic Monitor"
     DefaultDepth    24
       Option         "metamodes" "LCD: 1680x1050_60_0 +0+0"
       SubSection     "Display"
       Depth       24
       Modes      "1680x1050"
   EndSubSection
EndSection
```

So far this setup has worked on every nvidia based card I have setup. For some reason modelines just don't work because even though you can specify the vertical refresh rate you cannot specify the horizontal. At least not that I know of...

And with the gtf command that generates modelines it automatically selects a horizontal refresh rate that is too high and it causes the picture to be off center. If that is what you are experiencing then your problem may be the same...
