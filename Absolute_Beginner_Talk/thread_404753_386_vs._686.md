### Post by hardyn on 2007-04-08
as of edgy the generic kernel is the "all-processor" kernel with SMP support; i have tried to google around to see if it autodetects the CPU or uses some other scheme (i would like to know how this works, in depth, anybody?)

the 386 kernel as of edgy, is the same generic kernel as above, but has been configured for uni-processor.  some have reported better thoughput using the 386 kernel on uniprocessor boxes; the theory is they do not have to deal with SMP overhead for a non-SMP system, i don't think this has been quantified.

most people will want to use the generic kernel... P-M asus and vaio users must use the 386 kernel, as the generic kernel causes the processor to run away (50% idle cpu load and generally really poor performance).

good luck.

### Post by maxamillion on 2007-04-08
[http://en.wikipedia.org/wiki/I386](http://en.wikipedia.org/wiki/I386)

[http://en.wikipedia.org/wiki/I686](http://en.wikipedia.org/wiki/I686)

the differences in the kernels have to do with the use of the different instruction sets, the i686 will essentially be faster because it has instructions to perform certain calculations or media functions in less cpu cycles than the older i386, but the 386 is generally used by default for compatibility sake.


.... yes, you should be fine installing the 686 on a pentium D
