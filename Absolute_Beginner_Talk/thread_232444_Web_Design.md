### Post by Christmas on 2006-08-08
I was using Mozilla Composer, then tried Nvu which surprisingly had exactly the same interface as Mozilla, and then I had to make some tables for my page and ended up with Kate, the KDE Advanced Text Editor (the equivalent for it in GNOME is Gedit). Maybe you'll be surprised by the fact most recommendations are for a text editor, but give it a try, you'll be stunned how simple it is and how faster it works. If you never did HTML or PHP in text mode, take a look at this two links, which I could find very useful as a beginner:

[http://techtuts.com/](http://techtuts.com/) - Tutorials for a lot of web programming languages

### Post by Karlos on 2006-08-08
I use scite. it can even condense all your <divs> in a kind of file tree style.
ie: you get one of those little plus/minus signs down the left hand side which allows you to open and close your divs, very neat like it a lot and it highlights xhtml and css very nicely.
one tip for scite, if you want to change any properties open the global properties file and the user properties file (they are in the dropdown menus) then copy across any sections you want to change from the global file to your user file and change them there. For example I do mainly html so i changed the default indents from 8 to 3, which works much better for me (I believe that the setting of 8 is essential for certain languages like c and python )
Also the default font settings in scite are crap but if you hit cntrl+F11 together it turns it all into monospace which is much nicer.
as for nvu and all those sort of programs well i have found it takes longer to learn to use those programs than it does to actually learn html and css themselves. Most pro web designers would advise you to get yer hands dirty and do it by hand.
The other program I like is bluefish which automatically inserts your closing tags (which can get on your tits after a while, but is a nice function).
Kate is also very good, has a nice feature of coloured tags for all your open pages which change colour depending on how frequently you edit your open pages.
I'm sure somebody would disagree with everything i've said here but I dont really care, so there...

regards
karlos
