I am not sure about your problem. Did you back up your original xorg.conf?

My first question is how did you disconnect the monitor? I would log out of gnome and disconnect at the GDM (log-in) screen.

Second option is to write a second server in xorg.conf 

See this posting: [Dual Monitors](http://www.ubuntuforums.org/showthread.php?t=240150)

Put a section in there that looks something like this (Based on "FAIL" in the above thread):

> #
################################################## ############################################
#
# LAPTOP ONLY - S E T U P
#
# setup to have a xserver on the laptop display alone
#
# device for crt0


Section "ServerLayout"
Identifier "LAP"
Screen 0 "Screen 0"
InputDevice "Mouse1" "CorePointer"
InputDevice "Keyboard1" "CoreKeyboard"
# Option "Offtime" "10"
EndSection**You will have to modify this to reflect your xorg.conf as it is unlikely to work as a straight cut-and-paste.**

The only problem is I  only know how to start various servers from the CLI (I have not been able to configure GDM to do it).

You can set on server, say LAP as default (see the line earlier in xorg.conf :> # ************************************************** ********************
# Server flags section.
# ************************************************** ********************

Section "ServerFlags"
**Option "DefaultServerLayout" "LAP"**
...

Now when you log in via GDM you will avtivate only the laptop monitor.

To dual monitor,

Log out of GDM
Switch to tty1 or tty2 (Ctrl-Alt-F1 or Ctrl-AltF2)

Stop GDM```
sudo /etc/init.d/gdm stop
```This will drop you to the CLI if you enter it in a terminal from gnome.

Now start your dual session:```
/usr/bin/startx /usr/bin/gnome-session -- :2 -layout DUAL & DISPLAY=:2.1 /usr/bin/gnome-session 2> /dev/null &
```
You can set an alias in ~/.bashrc
```
alias dual='/usr/bin/startx /usr/bin/gnome-session -- :2 -layout DUAL & DISPLAY=:2.1 /usr/bin/gnome-session 2> /dev/null &'
```

See here for a script to do this for you: [Bash script for dual monitors](http://ubuntuforums.org/showthread.php?t=271045)

To disconnect you external monitor: Stop X (logout of gnome).
Type:```
sudo /etc/init.d/gdm restart
``` Or use the same bash script (in which case call this server "FAIL" rather then "LAP").

If you are going to use the bash script always, disable GDM at boot and you  will boot to CLI (which is what I do with my dual monitor box).

Sorry for the long post and sorry to post such a "hack". If you run into problems, post your xorg.conf (as an attachment please) and I will take a look.

If you get a solution, post it so I (we) can help others. 8)
