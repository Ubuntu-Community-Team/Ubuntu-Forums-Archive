### Post by bt224 on 2008-04-14
Ubuntu and Kubuntu are basically the same. The interface (GUI) and a very few programs are unique to each. It really is just a preference thing. You could install one, and if you don't like the interface, you can install the other interface and start using it. No need to reinstall the entire OS. If you want something familiar to XP, get Kubuntu. For something new, but still easy to use, get Ubuntu. The software underneath is the same. You could install either, and then pick a less graphic intensive GUI for the older machine. There are a few out there and plenty of folks to tell you the good and bad about each. Some were mentioned above, like XFCE. Again, you can change any time and it's easy. Not sure what is "old" is, however I would have at min a 1Ghz processor and 512 of RAM for either choice. Another very good suggestion was Puppy Linux. Easy to use and can really bring an old machine to life.

Hey dude,

I install/test different distros for a hobby (nerd, I know...), been through about 50 and here's my opinion:

Ubuntu vs Kubuntu (Gnome vs KDE)

I'm not asking for a G vs K war, IMO and IMO only, they perform the same. If you strip either one of non essential features/services etc., they'll be on par; if anything K feels faster for me, no just on Kubuntu, but across other distros. But marginally.

Others who have tested Xubuntu etc are right, there's a performance boost, but to deal with older hardware **you may have to look at other distros**, as below. It should be noted that I've never had the ease of use, works-out-of-the-box that I have had with Ubuntu elsewhere (bar Fedora, but that is extremely slow, IMO). So the distros below have their own advantages, particularly performance, but may not have the Ubuntesque usability that we've come to love.

**[Vector Linux]("http://www.vectorlinux.com")**
Fast. Works pretty well out of the box, haven't tested this distro for about a year but it was real fast (I could 3d (low res) game on a Pentium 433 with 256ram, but a decent card)

**[Arch Linux]("http://www.archlinux.org")**
It's really fast (currently on my GF's laptop). Install isn't so straight forward, but well documented, this is a chance to learn more. Again, nothing compares to Ubuntu for package availability/stability and community, but this is really fast.

**[Gentoo]("http://www.gentoo.org")**
Lighting. So fast. When I first tried to install gentoo, it picked up the nickname helltoo (I was building it from scratch), but it is well documented and it now comes with a GUI installer (esp on LiveCD). The last officially release is 2007.0, but the 2008.0 beta is out... I've just downloaded the ISO for testing. Again, forget the Ubuntu installation experience, but expect huge performance and a learning experience.

**[Linux From Scratch]("http://www.linuxfromscratch.org")**
Warp Speed. For Your Information, and good luck.

---

Also, I've had XP running as a media machine (watch DVDs / AVIs) on a Pentium 400, with 256 ram, GeForce440MX, the trick there is to kill all effects, the gui skin, to set the page file manually, and all non essential services / background programs. Having said that, XP doesn't Linux's operating stability and performance consistancy. In short, **Windose sucks.**
