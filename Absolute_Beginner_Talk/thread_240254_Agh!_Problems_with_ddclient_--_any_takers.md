### Post by kebes on 2006-08-20
If you can access your webserver inside your local network, then I suspect apache is fine. I think the problem is still in the router or your ISP.

I know you said that your ISP says it isn't blocking port 80, but I just don't trust that. There are too many layers between you and the internet and it only takes one blocking port 80 to ruin it.

My advice is to try to use a non-standard port. If that works, then you know that it is a port problem (and can either use the non-standard port or talk to your ISP). If that doesn't fix it, then at least it provides a new clue.

To change the port apache is listening to, you simply add a line like "Listen 8888" in your "/etc/apache2/apache2.conf" file. The port can be whatever you want. Use a port greater than 1024. Remember to unblock that port in your router firewall.

Just to confirm: when you accessed your webserver over the net you used:
[http://www.blah.com:8888/](http://www.blah.com:8888/)
(including the port!) Right?

You said you were using dyndns. Do you get the same result whether you use your domain name or your internet-facing IP address?

I know this is obvious (and you must have checked a hundred times), but you're sure that the IP address you have set in "port forwarding" on your router is indeed the address of your box? (the internal address you see when you run "ifconfig")

> Anything else I can try?

Another question for you: is this only a web-problem, or is it more general? Do you have any other servers running? For instance do you have an SSH server (openssh) or an ftp server (proftp) running? If so, can you access those if you follow a similar procedure?

If you activate openSSH, and then forward port 22 on your router, you should be able to SSH to your external IP address and log into your machine. If you cannot, then there is something wrong with your IP/router/firewall that is general to all ports: no incoming connections are possible.

If SSH (or ftp, etc.) works, but apache does not, then it may well be a apache-specific problem.

By default Ubuntu doesn't do port blocking, but it's possible that you have somehow gotten some firewall rules activated. You could install the "firestarter" package which lets you administer the "iptables" portion of Linux... which is essentially a firewall system. It's possible that there is some restriction in there that you didn't realize was set. The web-based administration system "webmin" also allows you to more easily edit the firewall (and work with servers like apache). To see what rules are currently set, you can type:
```

sudo iptables -L

```

If you have a rule set that might be blocking, then you can use "iptables" and "iptables-save" to change your configuration. If not, then you need to look elsewhere for the source of the problem...
