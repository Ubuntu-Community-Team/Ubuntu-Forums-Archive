### Post by eternicode on 2007-12-29
PuTTy is an SSH client.

On the command line, you can use "ssh user@host" to ssh into "host" as "user" (duh? :) )

For FTP, use the ftp command.

These should come "pre-installed" on Ubuntu (or Kubuntu).  "man ssh" or "man ftp" from the command line to get more detail on options for each program.

I believe there is also a PuTTy client for linux...I'm on winXP at the moment, so I can't check the repos (just run "sudo aptitude search putty").  A quick google for "linux putty" should give you some good links to look through.  Here's one: [http://www.chiark.greenend.org.uk/~sgtatham/putty/](http://www.chiark.greenend.org.uk/~sgtatham/putty/)

Since the server you're trying to connect to uses SSH (at least I assume your FTP and SSH servers are one and the same), you may have to use sftp instead of ftp.
```
$ sftp username@server.edu
Connecting to server.edu...
username@server.edu's password:
sftp>
```

Have you tried installing and using putty from the repos yet?

```
$ sudo aptitude search putty
Password:
p   putty                                      - Telnet/SSH client for X
p   putty-tools                                - command-line tools for SSH, SCP, and SFTP
$
```

If you're already familiar with putty, that may be the way to go in your case.

SSH is the Secure SHell protocol.  Basically just a terminal for a remote machine on your local desktop.  So it'll need you to use a command line either way.

SFTP is the FTP protocol used by SSH servers, "Secure FTP".  This is what I found works in Konqueror, entering "sftp://username@ssh-server.edu" into the address bar.
