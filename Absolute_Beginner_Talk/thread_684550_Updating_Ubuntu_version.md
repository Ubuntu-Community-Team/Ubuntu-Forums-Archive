### Post by Kilz on 2008-02-01
Another thing to keep in mind is the old adage  "If it isn't broke, don't fix it". Since the original poster is in school with little Linux experience. It may be better for them to put the upgrade off until they have the time to deal with possible problems that may come up. Such as spring break, or the summer. Dapper still has about a year of updates left. 
If the original poster wants to upgrade anyway. If they have a lot of room on their hard drive it may be better to install the newer version as a duel boot. Leaving the original Dapper install in place. This way if problems pop up the original poster still has the Dapper install to boot into to get schoolwork done while getting the new install working 100%. Then when everything is moved over and working delete the Dapper install to make room.
