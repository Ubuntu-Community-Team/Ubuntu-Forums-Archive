Hmm, I use ATI mainly so I'm not an NVIDIA expert, but that might be a legacy card requiring the legacy driver.  What is your xorg.conf right now, can you post it please?  Also, have you tried just installing the nividia drivers via apt or synaptic (sudo aptitude install nvidia-glx)?  This is the easiest way to install.

BTW, it would be 'sudo aptitude install nvidia-glx-legacy' I believe, if that is a legacy card.  Look it up on the nvidia website to be sure.

Finally, I personally don't trust installer scripts like envy.  They work great 95% of the time, but your system isn't necessarily the same as everyone else's, so theres no guarantee it will work, and when install scripts fail it can leave a big mess behind.  Your best bet is to use the drivers from the repo's, or last resort use the ones from the nvidia site (The ati drivers can make ubuntu .deb packages, I dont know if nvidia has this feature or not).

Good luck.

EDIT: in the meantime, to use your system, try the command 'sudo dpkg-reconfigure xserver-xorg' to rebuilt your xorg.conf
