### Post by kernel tom on 2007-06-01
ok i think i figured out how to solve your problem without loosing any data.

you'll need the Feisty Live CD... 

Boot up with a live cd, and open up a terminal

Mount your feisty drive somewhere like media/feisty

sudo mkdir /media/feisty

Mount your drive using the following command

sudo mount /dev/sda2 /media/feisty

(replace /dev/sda2 with whatever urs is like /dev/hda1 /dev/hda2 /dev/sda1 etc...
and don't do this step unless you know exactly what partition you are using!)

chroot into your feisty drive
sudo chroot /media/feisty su

Update your system via apt as normal
apt-get remove mdadm  (you can reinstall it later after we get u booting up)
apt-get update
apt-get upgrade
apt-get dist-upgrade

type "exit" hit return and reboot your system.. should work


now you may get a different error becuase your xorg file is incorrect... to fix this run from terminal line

sudo dpkg-reconfigure xserver-xorg

and answer using defaults and generics if you don't know your exact system specs... if you do tho choose the appropriate cards/names/drivers etc.

good luck and all apologies for leading you the wrong way earlier
