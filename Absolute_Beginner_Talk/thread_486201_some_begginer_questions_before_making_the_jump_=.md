### Post by Tomosaur on 2007-06-29
Ubuntu is a very user friendly distribution of Linux. When you hear people complaining about having to 'program' for Linux, I'm afraid they're bending the truth a little there. What they are referring to is the shell (aka the command line), which every distribution of Linux has. It is a very powerful tool, and when you see tutorials for Linux, you will generally see lines of commands to enter. Many people perceive this as being 'the only way' to get things done using Linux, but that couldn't be further from the truth. The reason tutorials use command lines is that the command line is the one constant on all Linux systems. Users are free to change everything about their systems, including how it looks, which programs are available, what the menus look like etc etc, so it is almost impossible to write instructions like this:

"Click System > Preferences > Themes, select the checkbox, click apply, click ok".

If every user has set up their own system in a particular way, then quite simply these instructions may be impossible to follow. However, if we just use commands, then every single Linux user can follow these instructions. It is also much easier to say 'type this command' than it is to describe a particular sequence of clicks around a graphical user environment.

So in short - you do not need to program for Linux. You do not even need to touch the command line if you don't want to. It is simply more convenient for us, and anyone who provides support, to type a command when helping people, than it is to vaguely describe where to click on the screen (which may or may not be relevant to that user anyway).

When you first log in to Ubuntu, you will be greeted with a nice friendly GUI (graphical user environment), and from there you are free to do pretty much whatever you like. No command line involved (although you really should learn at least basic command line stuff, because it really is a very, very powerful tool. You can organise your files, create documents, listen to music, automate tasks, play games, even surf the internet, all with text based, command line programs). However, in no way should you absolutely 'need' to use the command line. If you run into problems, and ask for help here, it is likely that people will ask you to use the command line. Please don't perceive this to be a limitation of Linux, it is just much more convenient for the people trying to help you. They probably could help you step by step through the GUI, but it would take a lot longer!

Now - yes, you can run Ubuntu and Windows alongside each other, this is called a 'dual boot'. There are guides on this forum on how to set up a dual boot system. After setting up your dual boot and rebooting, you will be greeted with a menu from which you can choose to boot Ubuntu, Ubuntu recovery mode, a memory test application (you won't need to use this unless your system is literally dying), or Windows. If you don't press any keys then the machine will automatically boot into Ubuntu after about 10 seconds.

As for your Windows programs - unfortunately you will probably have to find Linux equivalents to this software. Wine is a great tool, and has a very high success rate, but it doesn't support absolutely every piece of Windows software. OpenOffice, is, as you say, a great alternative, but it really is just up to you to experiment. The Wine website is [http://winehq.org](http://winehq.org), and from there you can visit the appDB (the link is in the top left of the Wine website), where you can check whether your Windows software works in Wine or not.

Installing software in Ubuntu is extremely easy. Simply click Applications > Add/Remove, and a program will open which lists the available software. Linux uses repositories - meaning that all of your software is centralised. The Ubuntu repositories contain many thousands of different applications, so you'll never be short of something to play with! All of the software in the repositories are checked and checked again, so you can be sure that they're safe to use (unlike on Windows, where if you download a piece of software, you have to put a little faith in the website you download it from). You will also get very, very high download speeds from the repositories. All software is free, and when updates for your software are released, you will be notified and you can update at your leisure. When a new version of Ubuntu is released, you will be notified aswell, and you can choose whether to upgrade or not (many people, however, reccommend that you simply do a clean install when upgrading to the next version of Ubuntu, because if something goes wrong during the upgrade process, it can leave you with a broken system).

For your hardware, this is why we have the LiveCD available. The LiveCD is the installation CD too. When you download and burn the cd, you can boot from it, and then choose the 'Start or Install Ubuntu' option. This will load a fully working version of Ubuntu without touching your hard drive. From here, you can test whether your hardware works or not (and if your wireless card works, then you can browse the net too). If it does, great. but if not, then you may be able to find instructions on how to get them working on the web. The LiveCD will run pretty slowly, however, so don't take this as an indicator of the performance of Ubuntu. An installed Ubuntu is very quick, usually much quicker than Windows, even on the same machine.

Anyway, good luck, and if you run into problems, don't hesitate to ask here - we'll be glad to help.
