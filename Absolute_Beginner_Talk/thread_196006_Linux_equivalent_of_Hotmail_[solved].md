nice blog, enjoyed reading 

   You can use your evolution to receive e-mails and still get them on your windoze box. In evolution go to Edit > Preferences > Mail Accounts, then select each account you have set up and click Edit, then go to the "Receiving Options" tab and under Message storage click the box next to "Leave messages on server" placing a check in it. This will allow you to check your e-mail from ubuntu without removing the messages from your e-mail providers server, so when you go to receive e-mail from your other computer it will still be there and will come down with the rest of your mail.

   I have both my pc and laptop with ubuntu / xp dual boot with several e-mail acct's set up in evolution and outlook on each and check mail from both sides of each. By having set both evolution and outlook on my laptop and outlook on my pc to leave messages the servers I am still able to have all my e-mails available on my pc's, which is my primary computer, evolution. By not setting the pc evolution to 'leave on server' my mail acct's do not become overloaded.

### Post by pompeyjohn on 2006-07-13
It is confusing!

It appears to me that people want a mail service that holds their data on a remote server. And they want it to follow the ideals of the free software movement.

If the software on the server is free as in speech - great. I am sure many people would like a mail service that benefits from all the good things that open source development can provide.

However, how to make it free as in beer? If the code is remote (and free) who pays for the server - and the bandwidth? Hotmail and Gmail can do this by bombarding you with adverts targeted by profiling your communication and system interaction.

How many people would use Ubuntu or any other Linux distro if there were adverts built into the desktop code? Not many is my guess.

The only way I can see this working, is if the mail server code is stored locally. Which is not an Application Service Provider model. It is not a Hotmail, or Gmail equuivalent.
