### Post by benerivo on 2007-10-19
If you were to try kde you can install kdebase from synaptic, without anything getting lost or replaced. Then you can log out of gnome and log in to kde from the log in screen.

What you get is a new set of base programs to use (terminal, text editor, browser, etc.), and a desktop with a different look and feel, but really not so dissimilar that you would be lost. You could still open nautilus and firefox and rhythmbox from the kde main menu, so you will have lost nothing. The only issue you might have is that gnome applications sometimes don't look good in a kde environment and vice versa, and you will also now have two terminals and text editors in the main menu. Because of this, people only tend to use the applications that are built for they environment they use. The bottom line is give it a try, you have nothing to lose. If you prefer kde then install Kubuntu so that you have a system with only kde apps installed.

### Post by mndzmyst on 2007-10-19
In my limited linux experience I can only offer the following:

1. A desktop environment determines how your applications will interact with you. How you open programs, or how you view files is determined by the environment. KDE programs are written in KDE/Qt and GNOME uses GTK+. At first Qt was not free software and so GNOME was started in an effort to use only free software. Qt is now free as well.

2. You will not lose anything per se. You can always install gtk in kde or vice versa. What you will notice is where things are laid out. I first start on KDE and got used to the menu having much more links to change my interface. Whereas on GNOME I've had to search and install a couple things. Not much difference actually, except where experience is concerned.

3. You can install KDE on GNOME or vice versa. Simply follow the instructions on this link

[http://www.psychocats.net/ubuntu/kde](http://www.psychocats.net/ubuntu/kde)
