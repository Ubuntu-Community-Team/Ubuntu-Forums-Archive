### Post by Tundro Walker on 2007-04-15
Don't feel bad about being confused.  Using MS Windows for a long time turns you into a horse drawing a carriage with blinders on.  When you take the blinders and wagon off, you can get a bit shocked by how much freedom you have, especially for things you've never thought about before, like changing your whole desktop interface, or your web browser, or picking a different word-processor (EG: lots of folks like Open-Office, which is a lot like MS Office, but others like using AbiWord as an MS Word replacement).

** First...**

...there isn't a "best" one; "best" is subjective.  The "best" one would depend on what you want to do with it, or what restrictions you have (EG: underpowered computer).  For instance, a lot of folks like using Totem as their music and movie player, since it has a pretty robust "Windows Media Player" feel to it.  Others like to install XMMS, which uses less resources and has more of a WinAmp feel to it.  Both are great, but neither covets the title of "best".  The *buntu distro's are the same way.  All are great, but the one thing that would make one better than the other is what you're installing it on and whether you like using it or not.

** Second...**

...all Ubuntu versions use the same underlying architechture...Linux is the kernal, X11 is the X-window server, etc.   But, everything is interchangeable even after an install of one version.  So, if you installed and decide to try out a different flavor, you can hop into the package manager for that *buntu flavor (GNOME Synaptic, KDE Adept, etc), and click on the "ubnutu-desktop", "kubuntu-desktop" or "xubuntu-desktop" to have it select and install all the necessary components for that flavor.  Likewise, after install, if you don't like some of the stuff that came with your distro, you can hop into your package manager, select to "Remove" the meta package for the one you have, and then you can select other things in your distro to uninstall.  EG: I installed Ubuntu, but didn't want to keep some things...Open-Office, Ekiga, etc.  So, I uninstalled the "ubuntu-desktop" meta package, then bumped off the things I didn't want.

With that said, the main differences in the flavors of Ubuntu are...[LIST=1]
[*]Which desktop interface it uses
[*]What software comes pre-installed on it[/LIST]**Ubuntu**[LIST]
[*]GNOME desktop
[*]lots of apps pre-installed (pretty chubby, gnome-specific apps)
[*]requires a fairly good machine (2ghz+, 512+ RAM)[/LIST]It's the flagship distro which is kinda swiss-army knife'ish, in that it comes with a lot of stuff pre-installed and enabled, which you may not need (EG: Palm Pilot support).  Using the GNOME desktop, it comes with all kinds of GNOME-specific apps and applets.  You can tweak it down to just what you need, though, and get it running pretty lean.  The tweaking takes time, though.

**Kubuntu**[LIST]
[*]KDE desktop
[*]lots of apps pre-installed (pretty chubby, kde-specific apps)
[*]requires a fairly good machine (1.5ghz+, 248-512 RAM)[/LIST]Kubuntu is full-featured, too, and the KDE desktop is recommended as being easier to migrate to for MS Windows users.  Being a former MS Window user, I found both GNOME and KDE were pretty easy to use.  KDE is a bit more intuitive in making shortcuts and such, though.  Like GNOME, KDE has it's own application versions and applets.  You can tweak it to what you need, but tweaking takes time.

**Xubuntu**[LIST]
[*]XFCE desktop
[*]pretty lean (EG: no Open-Office)
[*]requires modest machine (800mhz+, 248 RAM+)[/LIST]Xubuntu is good for older machines, since XFCE is a pretty streamlined desktop interface.  XFCE doesn't really have it's own set of robust applications.  IE: GNOME has custom apps like Gnumeric spreadsheet or Gnomad2 music player, KDE has Konqueror web-browser/file-manager and Kaffeine music player, but XFCE really only has Thunar file-manager and some applets for the task bar.  However, XFCE can be set to support GNOME and/or KDE apps, so you can use those desktop-specific apps on XFCE.  (In face, Xubuntu comes with Gnumeric, the Gnome spreadsheet app, installed).  Xubuntu is modest on pre-installed apps, focusing more on lite alternatives or bare minimums for the average user rather then installing a bunch of stuff the average user may never use.  The one down-side I had with XFCE was an ever-shrinking font bug, which made my desktop fonts continually shrink to miniscule proportions.  It's a bug that you have to manually fix in a config file involving DPI or such.  If you don't use GNOME or KDE specific apps, you can also switch off XFCE's support of such, and it'll speed up boot time.

**Ultimately...**

...all the Ubuntu distros are like driving the same car, but with different features.  They all have driver-side air-bags (System Updater), but Xubuntu is more the economy model, while Kubuntu and Ubuntu are more the full-blown models.  Some folks argue over GNOME or KDE being better, but it's like Chevy truck owners arguing with Ford truck owners.  They both love driving trucks, but they prefer a certain kind.

Another way you can think of an *buntu distro install is like this.  If you want to install a lot of stuff, and strip off what you don't want, then go for Ubuntu or Kubuntu.  If you want to start small and add-on, go with Xubuntu.  Plus, the great part is, you can install each desktop on top of what you already have, then when logging in, change your Session to a different one and take it for a test drive.  You'll eventually find one you love, and then you can bump off the ones you don't.

**For Programming / Development...**

If you're going to use if for development, I'd recommend tossing Xubuntu onto a powerhouse machine.  Then, tweak the services to only load what you need, and strip the task bar of all "fluffy" stuff to  conserve RAM / CPU (EG: you don't need the weather monitor going ... it's nice, but not needed).  Get about a gig of RAM in that machine, then set the vm.swappiness to "0", so it'll use the RAM for everything.  If you want to sqeeze a little more resource, install and config OpenBox or FluxBox, both of which are lighter weight then XFCE, and still pretty easy on the eyes (although configuration can be challenging if you're not used to tweaking .txt type config files).

**For Flashy Desktop...**

Ubuntu and Kubuntu are pretty nice as-is, but you can hop into your distro's package manager and install some flashy desktop add-ons.  A lot of folks try using Compiz or Beryl for Windows Vista-like effects (3d desktop, window effects, etc), but they're still beta, and folks have issues.  One named "Emerald" (I think) provides some decent eye-candy which you can tack onto any of the desktops without issues, and makes it more Windows XP'ish.  I don't recommend any of this extra glitz for low-powered machines.  My old 800mhz chugged with just Emerald switched on...looked nice, but chugged.
