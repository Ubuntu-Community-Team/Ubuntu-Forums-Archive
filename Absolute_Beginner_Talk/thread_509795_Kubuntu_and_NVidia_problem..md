### Post by RomeReactor on 2007-07-26
> **The Exorcist said:**
> glxgears just produced an error message. I changed both driver refs from nv to nvidia, saved and **then run 'md5sum /etc/X11/xorg.conf | sudo tee /var/lib/x11/xorg.conf.md5sum' as outlined above**.

You shouldn't run the **md5sum /etc/X11/xorg.conf | sudo tee /var/lib/x11/xorg.conf.md5sum** command after changing the driver from **nv** to **nvidia**, as it will restore an older xorg in place of the one you just modified. Change the driver again in **xorg.conf**, save the file and press CTRL+ALT+BACKSPACE to restart the X server. Or reboot.

### Post by RomeReactor on 2007-07-26
The **nv** driver is open source and works fine; the only problems you may have with it is choppy scrolling in Firefox or other browsers, and lack of 3D acceleration, which means no 3D games. If this is not an issue for you, you may want to keep it. also, your xorg.conf is very strange; doing a dpkg-reconfigure may help clean it up a bit, and help you enable the proprietary driver. You **should** back up your existing configuration, though:
```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.backup
```
if anything goes wrong with the dpkg-reconfigure--though it shouldn't--you can restore the backup like this:
```
sudo cp /etc/X11/xorg.conf.backup /etc/X11/xorg.conf
```
and then
```
sudo /etc/init.d/kdm start
```
