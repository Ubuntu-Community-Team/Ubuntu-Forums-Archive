### Post by rkd on 2008-03-26
It is hard to tell how much you know about computers, so some of this you might already know, but since I didn't see it explained in previous posts, I want to make sure the elementary points get made.

1.  Many programs written for one OS simply will not run on another.  So do not expect to be able to use arbitrary Mac software or Windows software on Linux.

There are exceptions.  Software written in interpreted languages, such as Java, Perl, TCL, Python, etc. sometimes will work on more than one OS.  Even there, if you have a copy that is packaged for installation on, say, Windows, it can be difficult to get it to install on Linux.  Same with something packaged for the Mac.  In those cases, sometimes you can go back to the source and get a copy packaged for Linux, but not always.

Other exceptions: Some software intended for Windows can be installed and run on Linux using WINE, or its relatives Crossover Office or Cedega.  Sometimes that goes very smoothly, but often it is a bit of a struggle.  No guarantees, in general.

2.  Supporting software for some hardware is absent or hard to set up in Linux.  You can't use the full capabilities of some graphics cards.  Some printers don't work.  Wireless networking is a big problem area -- some devices work fine; others are very hard to set up or don't work at all.  Some sound hardware is hard to set up or doesn't work at all.  You didn't mention any exotic stuff, so I won't go into other types of hardware.

There are lists of hardware that work which you can consult before buying a computer you want to use with Linux.  Sometimes it is hard to find out the exact hardware included in a system, though.  The manufacturers don't provide all the details because they only care that it works with Windows, they make sure it works with Windows, so they don't publish all the details that Linux folks need to know.

There are small hardware suppliers who cater to the Linux market, and if you buy from them, you can be assured that the hardware will work with Linux.  You can get both desktop systems and notebook systems from them.  I don't remember the names of any of them right now (I'd have to do some web searching to find them).  Maybe someone else can post the names of a few of them, or maybe you can find some with a web search yourself.  If you can't find any, ask about it here and we surely can identify some for you.  (Tell us your location if you want recommendations in your country.)

Dell has been providing a few systems with Linux preinstalled, but the choices they offer are limited.  The small, specialty places probably give you better choices.

3.  As long as you don't insist that the system and individual programs look and work exactly like Windows, you can easily do the email, web surfing, word processing, mp3 playing, and similar common things everyone uses a computer for.

Note: You mentioned downloading programs.  You'll be able to download Linux programs (the Ubuntu repositories have thousands), but you probably won't be able to download and use programs intended for use on Windows.  If the program does a common function, you usually will be able to find an equivalent Linux program, but if the program is especially unique, there might not be a Linux equivalent.  Expect that.

For instance, if you use your computer to do your taxes, you won't be able to use any of the common tax preparation programs, at least not on Linux.  That is why many people still have to keep a copy of Windows on their computer and dual-boot.

Same with many Windows games, but I think you said games weren't very important to you, so that might not matter.

One thing about web surfing.  There are a few web sites that only work with Internet Explorer (often because they rely on Active X components).  Those sites won't work with browsers on Linux.  There aren't many of them, but if you happen to need to use one, you'll have to boot to your Windows partition to do it.

You mentioned using a Zune.  I don't know whether there is Linux software that can deal with putting files on a Zune, and I kind of doubt there is Linux software that can access Microsoft's music store for the Zune.  If using the Zune on Linux is important to you, check into that specifically.  Maybe the situation is better than I know.  But you might have to continue using Windows to work with your Zune.  There are lots of other music players that do work fine with Linux, so if you are willing to change to one of them, that would eliminate this problem.

4.  For the reasons above, I highly recommend you either set up your computer for dual-booting Windows and Ubuntu, or keep a separate Windows computer around, so you have a Windows system when you need it.  Perhaps over time you'll be able to completely eliminate your need for Windows, but until then, keep a safety net.

If you decide to set up a dual-boot, find and read the directions for doing so.  In most cases, it isn't hard at all, but there are some tricky situations -- just look at all the posts here about people having trouble getting dual-boot to work.  Most of them didn't check first to see whether they had to do something special because of certain characteristics of their computer.  It is true that it goes smoothly without special thought in most cases, but it can be a real headache if your case isn't one of the smooth ones.  If after reading the dual-boot installation instructions, you aren't sure about something, ask here before trying to install.  You may save yourself a lot of hassle.

5.  Unless your friend who uses Kubuntu is well-experienced with Ubuntu, it probably would be best for you to use Kubuntu, too.  This is a little hard to call because if your friend can't help you with a problem and you have come to the forums for help, far fewer people know about Kubuntu, so you can get incorrect advice and have to wait longer for someone who understands Kubuntu to see your question.  A lot of things are independent of the Ubuntu/Kubuntu differences, but certainly not all.

6.  In the above, it may sound like I'm being very negative about Linux or Ubuntu.  That is not my intent.  I'm just trying to point out the problems that many people run into so you aren't surprised, and where possible, you plan well so you avoid them.  Ubuntu is a fine system to use -- secure, customizable, a great community.  Just understand what it can and can't do before jumping in and you will be fine.

### Post by rkd on 2008-03-27
I'm sorry the point about wireless wasn't clear.  There are numerous controller chips for wireless that a computer maker might use.  They come from different companies -- Intel, Broadcom, Atheros, and others.  Ubuntu has drivers for some of those controllers, and it doesn't have drivers for the others.  A computer that contains a wireless controller for which Ubuntu has a driver will be detected during installation and be set up correctly.  You will have to do some configuration of the network ID and such, but that's no different than with Windows.  Once set up and configured, it will work without problem with any wireless access point (or at least without any more problem than you'd have using that access point running Windows in your computer).  

If Ubuntu doesn't have a driver for the wireless controller in the computer, then the wireless won't work right away after installing Ubuntu and you'll have to go through some extra effort to get it to work.  What that involves varies from controller to controller.  Some require only a little extra fiddling; others require quite a lot of effort to get working; and some can't be made to work at all.  That's why it is important to check what components are used in a computer before you buy it, if possible.

As for downloading software, there are thousands of packages in the Ubuntu repositories.  The package manager software has some simple ways to search for things in the repositories, though sometimes the search doesn't find things because you aren't thinking of the right keywords that the thing was indexed with.  But once someone tells you the name of the package you need, it is a snap to find.  Anything in the repositories can be installed very simply -- you just enter its name into one of the package managers and the package manager handles downloading the package, any other packages it needs that you don't already have, installs them, and even watches for updates to the packages and lets you know when they are available.  Software that is not in the Ubuntu repositories is quite a bit more difficult to install -- often harder than it is in Windows, even.  But you usually only have to do that if you are using Ubuntu on a computer that contains some hardware for which Ubuntu doesn't have driver software. That's why it is important to try to pick a computer to use that doesn't have unsupported hardware.

You can find more about installing software here:

[http://www.psychocats.net/ubuntu/installingsoftware](http://www.psychocats.net/ubuntu/installingsoftware)
[http://www.psychocats.net/ubuntu/sources](http://www.psychocats.net/ubuntu/sources)

You should not just go buy some arbitrary Vista computer.  You should try to find out what controllers are used in a computer and check to see whether they have Ubuntu drivers before you buy it.  Otherwise, you might have a lot of headaches finding and installing drivers for unsupported hardware.  

That's why I said it was a good idea to buy a computer from one of the companies that base their business on selling computers with Ubuntu (or at least some Linux version) preinstalled.  You know those computers have hardware that Linux supports, and the company even does the install for you, just like happens when you buy a Windows computer.  If you don't want to buy from one of those specialty companies, you can buy from one of the usual places, but you should try to find out the details of what components are used inside the computer before you buy it so you can check to see whether all those components are in the Ubuntu hardware support list.  That's a bit of a hassle, since the computer sales people generally don't make it easy to find out that stuff, but if you want to select your hardware on your own, that comes with the territory.  Ubuntu does support quite a lot of hardware, so you often will be lucky and find that a computer selected without checking the hardware will install Ubuntu without problems.  But it sometimes is quite painful when you run across one that has some components that are not supported.

Ubuntu installs Firefox and has an icon for it on the top bar, so you'll be able to use a familiar browser without any extra effort on your part.  At least that is the way it is with Ubuntu; I'm not familiar with Kubuntu, so I don't know whether Firefox is automatically installed there or where to find Firefox if it is installed.  If Firefox isn't automatically installed, you will be able to use the package manager to install it very easily. There are lots of other browsers available in Ubuntu and Kubuntu, so if you care to try some of the other ones, you can do so easily.

If that friend of yours can help you out, you are right that it would be easier for you than working with these forums -- it is a lot easier in person.  He might not know everything you ask about, so you (or the two of you) might still have to get help from the forums some of the time.  If so, we are here.
