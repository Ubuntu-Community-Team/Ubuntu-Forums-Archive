### Post by Bloch on 2006-06-15
> Isn't there an Official Link that specifies those Requirements?
Ubuntu and linux in general is community developed. Most guides etc are produced by volunteer effort. 

Do a search on the forum for "system requirements" and you will find what people have been able to run ubuntu on.

Another reason the minimum requirements are vague is because ubuntu gives so many choices. You can run the XFCE desktop (Xubuntu) and your system will work fine with a slower processor. There are even more "lightweight" desktops.

I don't think anyone would be comfortable with standard ubuntu on anything less than a PIII processor (did they start at 800MHz?)
But with lighter desktops people are running  it on pentium I processors.
OpenOffice works fine with 256MB ram
In general RAM is more a limiting factor than CPU speed in linux. 128MB of RAM will freeze the standard installation

Minimum System (in my view)

CPU 500Mhz
RAM 256MB
HDD At least a 4GB HDD

The CPU can vary a little here. You can run it on say a 400Mhz cpu with 256MB RAM but that would not be a good experience though.

However if your 400Mhz CPU was combined with 512MB RAM it would compensate to some degree.

You can forget about running ubuntu on 128MB RAM.

I have run a few boxes with similar configurations and the performance is good.

Ubuntu isn't meant for lower end computers, it is on the same playing field as XP and Vista.  Compared to XP, ubuntu has superior performance at the Minimum and Recommended specs above. Compared to Vista, ubuntu is so far ahead you can't even call it a proper comparision.

Ubuntu

Recommended System (in my view)

CPU 800Mhz 
RAM 396MB (256+128 )
HDD 10 GB HDD

 
Vista

Recommended System (from their website the other week)

CPU 3000Mhz 
RAM 1000MB
HDD 40 GB HDD

The BIG problem with ubuntu, as for most Linux is this "it isn't easy in many areas". It takes next to no effort to add a new HDD to XP/Vista, and a few clicks to network XP/Vista to XP/Vista.

Compare that to ubuntu. To add a new HDD you need to create a mount folder, you need to mount the HDD through a terminal line command, you need to edit the fstab file so show your HDD - don't get any code wrong and that means the correct paths. Many concepts to learn, many steps to follow - far too difficult compared to XP/Vista.

I won't even beging to explain about NFS and the ordeal of networking 2 ubuntu boxes. Though I have done such now so many times that I feel comfortable with it, it still is just TOO MUCH to ask of people with a Windows background.

So ubuntu has advantages, and a great many but it also has disadvantages, again a great many. It is a matter of personal break even analysis, as to which is better. For me, ubuntu wins over XP/Vista.
