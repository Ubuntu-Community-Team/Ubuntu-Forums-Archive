### Post by arvevans on 2007-06-16
The usual problem with making a bootable CD for installing Linux is that you forgot to make it an "ISO Image".  Your CD writer program should have options for forcing an "Image Copy" and for "verify after burn" to insure that the CD is error free.  You can do all of this in Windows, before even trying to install Linux.

Then, reboot and enter your BIOS setup.  Change the second BIOS screen options to boot first from CD-ROM, then secondly from the first Hard Drive.  This way if no CD-ROM is inserted (or if it is not bootable) you will automatically boot from the hard drive.

Insert your newly burned Linux CD and reboot.  If you are using the "Live CD" for Ubuntu your computer should come up running Linux in RAM (nothing has touched your hard drive at this point).  Now you can play with the CD-based Live Linux, or click on install and actually install it to your hard drive.

Don't give up, or if you do for now...come back and try again later.  There are a lot of people available here who are willing to help you get through this.  

  1) Linux is not for everybody.
  2) Not everybody is suitable for Linux.
  3)  Not all hardware will run Windows...Not all hardware will run Linux...Not all hardware will run OS-X...etc., etc.
 
Linux is about choice.  You have your choice of OS (Operating Systems).  You have your choice of applications.  You can customize the Linux OS to look and feel the way you prefer it to operate.  You have the choice of paying for a new version of Windows every few years or of freely downloading and using Linux.  The choice is yours.

### Post by RRS on 2007-06-16
Glad to see you're rethinking your opinion of Ubuntu and the forums.

Since you're still interested I'd recommend a "dual boot" installation so you still have your familiar Windows system to get work, etc. done while you're finding your way around Ubuntu.

You didn't say if you downloaded the Desktop cd or the Alternate. The burning process is the same for either one, but the Desktop will let you use Ubuntu and test for hardware compatibility without actually installing it till you're ready while the alternate will work better if you have less then 256 of ram.

These two sites both have good easy to follow instructions installation with a lot of helpful stuff for afterwards;
[Herman's site]("http://users.bigpond.net.au/hermanzone/index.htm") is good for the Alternate cd.
Use [Aysiu's]("http://www.psychocats.net/ubuntu/index.php") for the Desktop, it also has a good section on [downloading and burning.]("http://www.psychocats.net/ubuntu/iso")

To back-up what a previous post said, it's always a good idea to back-up any data on your computer beforehand. Problems rarely occur, but "just in case". Also a thorough defrag of your Windows installation is recommended.

And please don't be put off by the occasionally less thoughtful replies you see on the forums. In my experience there's far less then other places but people are people. Also some of us are easily confused and occasionally misinterpret a question or problem someone is having.

Hope you stick with it, you'll find the effort well rewarded.
