### Post by K.Mandla on 2007-02-14
You might be experiencing a problem between the *linux-generic* and *linux-386* packages.

If you're using a proprietary Nvidia driver or *nvidia-glx* from the repositories, you need the *linux-386* package installed. It contains the nvidia kernel module that allows your driver to "connect" to the kernel. (I don't know of the proper way to explain it.)

*linux-generic* does not include that module, and so the driver fails to start. The problem might be that the upgrade to the 2.6.17-11 kernel left you with *linux-generic*, and not *linux-386*.

If you install the *linux-386* package, it should fix things for you. From a terminal

```
sudo aptitude install linux-386
```
See if [this]("http://kmandla.wordpress.com/2007/02/13/no-linux-386-on-upgrade/") is what you experienced.

The kernel is the core of the OS.  It is the most basic foundation of any OS, in this particular case linux.  Now, linux is "open source" and governed by a license called the Gnu Public License (GPL).  What this means is that the only drivers that can actually be a part of the kernel are open source drivers (the source code must be available), due to the terms of the license and its philosophy.  Now, graphics card makers ATI and NVIDIA don't want any part in this, and refuse to release the source code of their drivers.  For this reason, they cannot be a part of the kernel.  However, in order to actually do anything, the drivers need to talk to the kernel somehow.  This is where the modules come in.  When you install the modules, the kernel can talk to the drivers and everything works fine.  These modules are very specific and must be built with the exact kernel specification (ie, to build them you need to look at the kernel source code).  This is no issue for the repository, because everyone running a kernel from the repo's has the same kernel, so you can release a module prebuilt.  Unfortunately, and I do not know why, whoever built the modules in the repo's for the NVIDIA drivers built them for the 386 kernel and not the generic kernel, hence you need the 386 kernel with nvidia.  As for booting different kernels, the startup procedure on your computer has a list of modules to load, and it only loads the latest nvidia module for the kernel.  Since the kernels are different, the old kernel cannot use the old module and hence X wont start unless you use (a) the nv driver with no 3D, or (b) rebuild the module, breaking your new kernel.

The old kernel is pretty much useless.  Most people don't delete them unless they are running out of HD space, just erase the or comment them out of the grub menu.lst file (sudo gedit /boot/grub/menu.lst) so they don't make your boot screen messy, but its personal choice.  If your second core is not utilized in the 386 kernel (try using a gDesklet system monitor to check) go back to the generic one and download the Envy script, or better yet the actually drivers from NVIDIA.  Make sure you have the linux-sources package installed, and then the drivers will custom build the module for your system.

I typed a lot, if you don't understand it or need more help just ask for clarification.
