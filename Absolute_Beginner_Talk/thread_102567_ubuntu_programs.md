### Post by janrinok on 2005-12-12
I am not quite sure what programs you are talking about but perhaps a better way would be to look at the list of possible programs in Synaptic and install the programs you want from there.  I have 17863 different packages in my repositories and I cannot think of something that I want to do that cannot be fulfilled by using an Ubuntu specific package.  Perhaps the idea of surfing the web trying to find interesting software to install is not really the linux way.  But are are correct in saying that, when you do surf the web looking for software, you will not see much that is Ubuntu specific.  
If you like taking a gamble and using software from an unknown source (and which is not supported by the Ubuntu community) then you can always install packages from source files contained in 'tar' files, which usually end in tar, tar.gz or tgz.  Then you will have to make sure that you have all the appropriate dependencies and compile the program yourself.   And many RPM packages can be persuaded to install under Ubuntu.
This is not a method that I would recommend for a noob, but can certainly provide a mental challenge to those that a looking for their kicks this way.  Jan
