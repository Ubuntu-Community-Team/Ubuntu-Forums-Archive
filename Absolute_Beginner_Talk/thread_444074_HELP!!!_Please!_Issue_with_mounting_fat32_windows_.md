"sudo mount -t vfat -o ro /dev/hdb1 testing"

If that doesn't work, take out the "-o ro".


edit: in case you were wondering what this did, here's the run down.

sudo tells it that the command is being run with administrative priveleges
mount means that you're giving the command to mount the drive, obviously
-t is a flag that stands for "type". It lets you tell the computer what type of partition you're mounting. vfat is the "driver" for all kinds of FAT partitions, so that's what you're using
-o is a flag that stands for "options", and "ro" is the "read-only" option.
/dev/hdb1 is the hard drive that you're trying to mount
and "testing" is the folder that you're trying to mount it in.
