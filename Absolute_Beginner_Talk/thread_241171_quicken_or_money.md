### Post by msak007 on 2006-08-22
kmymoney is very nice. The basic functions work well, but there are a lot of little features that I miss about MS Money, things such as:

1. Automatically trending a transcation and trying to assign it a category when it's downloaded (like dining out, gas, groceries, etc.). Currently when I download a transaction, I have to rename the Payee (see #2) and assign it a category. This leaves a lot of room for user error to assign the same payee / transaction type a different category possibly skewing reports.

2. Remembering Payee names when downloading transactions instead of creating a new entry for every obscure transaction payee name the way it shows up on your bank statement and having to rename it manually. For example if I bought something from Wal-Mart it might show up as "WAL-MART 028365 WARREN, OH" on my bank statement / online. Money remembered that I named the Payee "Wal-Mart", so every transaction that came in from the same Payee was automatically named "Wal-Mart" - you get the idea. kmymoney for some reason downloads the bank statement name and automatically adds a Payee and I have to rename it / delete the newly added Payee manually.

3.Integrated online banking. I couldn't use the repo version and had to compile it with the online banking backend, so that part of it requires a lot of setup. It's currently @ version 0.8.4, and integrated online banking is planned for version 1.0 so you can tell it'll be a while :).

Aside from these little gripes, it really does work well and has nice reports and such. I have a lot of hope for this app and how much it'll evolve. And it integrates nicely into KDE :).
