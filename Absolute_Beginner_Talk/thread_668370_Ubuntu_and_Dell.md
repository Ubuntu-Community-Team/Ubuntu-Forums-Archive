### Post by SeanHodges on 2008-01-15
Interesting point you raise there.

I think one of the main reasons Dell do this is because they do not intend to build machines specifically for Ubuntu. This means that the machines that run Ubuntu must be able to run Vista as well, so that they have more tangible stock. The percentage of Ubuntu sales is far below the Windows ones (at least at the moment!) so this is how they can make the whole thing work.

Obviously I'm not saying this is good for the consumer, you're right that all machines are OTT for average desktop use, but I don't think Dell is geared to do it any other way.

What we need is some vendors targeting GNU/Linux machines as their *main* product line. This means that the machine specifications properly match their marketed purpose, as opposed to just plopping the desired O/S on one-size-fits-all machines.
