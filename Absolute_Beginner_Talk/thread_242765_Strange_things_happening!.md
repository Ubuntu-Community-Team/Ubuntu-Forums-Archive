### Post by bcollignon on 2006-09-20
OK, I think I have it narrowed down and it has to do with startup screen resolution in xorg.conf.  In any event, I updated my KDE to 3.54 by modifying  my sources.list to include the latest KDE version.  Howto is at 
[http://kubuntu.org/announcements/kde-354.php](http://kubuntu.org/announcements/kde-354.php).  

Then download libkdeinit_kdesktop.so from [http://megaflexdestiny.net/bits/kde353screensaverfix/32x86/libkdeinit_kdesktop.so](http://megaflexdestiny.net/bits/kde353screensaverfix/32x86/libkdeinit_kdesktop.so) and put this file in /usr/lib/ (make the permissions match the original file it replaces)
32bit x86 - libkdeinit_kdesktop.so filesize: 752k

The last part is somewhat crucial, as it involves reconfiguring xorg.conf.  I saw that my login screen was at a reduced resolution after upgrading some other components of Ubuntu.  No big deal, I thought.  However, it seems to have interfered with the operation of my KDE screensavers.  To fix this, I found the command dpkg -reconfigure -phigh xserver-xorg, which seems to have fixed my resolution problems at login and my screensaver problems at the same time.  This command, by the way, was within the man file for xorg and is, in fact, commented out at the beginning of the xorg.conf file itself.

I hope this helps if you really need your KDE screensavers.  Sometimes,  just upgrading to the latest version isn't enough I guess.
