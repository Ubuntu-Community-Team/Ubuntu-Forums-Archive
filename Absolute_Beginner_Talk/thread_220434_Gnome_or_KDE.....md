### Post by mattisking on 2006-07-21
KDE is far older and is far more configurable. Some people would say it's SO configurable as to be overwhelming to alot of Linux users, particularly newbies. I don't subscribe to that notion even though I prefer Gnome. Honestly, it's really just all about choice. KDE has some seriously awesome things to offer, just as Gnome does, and as an older front end to the Linux kernel, there are definitely some things that are, in fact, better, if for no other reason than of maturity. Some of eye candy stuff that is being worked on and is already out is really beautiful.

Gnome is quite a bit younger, actually grown from the development libraries that original built the Imaging application, The Gimp. Those libraries eventually were good enough that someone said something to the effect of, "Let's use em to write our own front end". Gnome, which is the default for Ubuntu, and my personal choice, is written towards "keeping it simple". KDE users would answer this with something along the lines of "Yeah, by taking away the user's ability to make it like they want" or maybe calling it too "dumbed down". The truth is probably somewhere in the middle, but like KDE, it's also a beautiful, well developed front end with a huge following and regular releases. 

As you installed Ubuntu, I'd recommend trying it out for a while as it's default is Gnome. See if you like it. Then download the Kubuntu Live CD and see which you prefer. Kubuntu, by the way, is  really the preferred method of using KDE for Ubuntu. The distro is completely tweaked for KDE where-as I'm not sure installing the kubuntu-desktop package will necessarily set everything up as nicely for you.
