Pardon my French, but that is male-cow manure... Simply because there are less known viruses targeted at Linux, this doesn't mean Linux is invulnerable to viruses. This is just something people have been saying, and everybody just copies it and says it too. Looking just at the solid facts: a Linux machine running antivirus software is **more secure** than a Linux machine not running antivirus. ### Post by rillip on 2007-07-19
I see these arguments and think they're good ones.  But I haven't seen any claim that a script with ```
rm -rf &
``` will in fact every be caught by a virus scanner.  This isn't viral activity, it's anticipated system commands. I've run many scripts with rm commands and not have them be caught by avg, clamav, or aegis.

All current Linux virii are intended for servers or proof of concept.  I agree it is possible to pass on virii to other Windows friends, and a Linux virus scanner might catch such a thing and prevent it, but I wouldn't know, as I never download or forward attachements I don't know about anyway.

If good security practiceses are in use, a virus scanner does not do anything on a Linux system.  If good security practiceses are not in use, a virus scanner is unlikely to help.
