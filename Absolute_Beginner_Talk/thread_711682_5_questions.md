### Post by Herman on 2008-02-29
:) Hello 4me,
I don't know how to answer your first questions, but I will have a try at the rest, starting from no. 3).
> 3) i have a project that requires a spreadsheet for this weekend, i am still 'dual booted', although have not used (signed up for) the windows versions in vista yet.

a) if i e-mail them to a guy who has windows, can he 'read them'
b) i have heard of some 'printing' problems.....can i at least accomplish that in a pinch? If the person you are emailing your spreadsheet to has Open Office.org for Windows or is willing to install it he or she will certainly be able to handle your Open Office.org spreadsheet made in Ubuntu. I am also fairly sure that Ubuntu can open and edit a spreadsheet made by a Windows program, but I'm not certain if the reverse is true.  
To email a spreadsheet you would put a copy of it into a folder, right-click on the folder and click 'Create Archive', and then have it made into a .zip file because that's probably the only kind of file compression your friend with Windows will be able to extract. Then in Evolution email, click 'attach', and browse for the .zip file you made. Your .zip file will be sent with the email as an attachment. 
>  4) ubuntu loads much, much faster than vista.....but seems to be a little slower after up and going......

a) is this normal?
b) anything i can do? <----i had 2 hard disks with 160GB, each using one (i think, no real clue on this.........i am a newbie....but on both OS's, i show about 130GB's left..)It depends what you mean by slower. It takes longer to open up a program in Ubuntu when it's not already open, because in Linux there are no programs that partially load in your memory at boot-up. Many programs in Windows do that, they all like to 'load in your start menu', and it gives them a head start when you open that program. The problem is, when you install more programs, you'll soon run out of RAM, with so many programs you're not even using all reserving a slice of your memory real estate. You need to either buy more RAM, or else install another program to manage your RAM. 
In Linux things are simpler, Linux programs don't fight over your RAM and try to claim it. When Linux programs aren't being used they are polite and leave the RAM available for whatever program you are actually using at the moment.
Therefore, you may notice the Open Office.org Spreadsheet  takes a while to load, but once it's loaded and you open another, second spreadsheet, the second spreadsheet will open at a normal speed.
I'm a trucker and I have a huge road map which takes a lot of RAM to open. When I used Windows it would just about stall my laptop with 512 MB of RAM, but Ubuntu in the same laptop opens the map with ease and I can pan around in it as quickly as I like. In Windows, the screen keeps going blank when I try to pan and it takes a few moments for the image to be redrawn. I guess that's because in Windows the memory isn't used as efficiently.
Also, the biggest memory occupier in Windows is your antivirus and other anti whatever programs that are so necessary to keep running in the background in a Windows system. 
In Linux we don't need those. our computer is really ours, the full power belongs to the user.
Windows is actually very fast when you use Ubuntu for all your internet purposes, web browsing and receiving email. I soon found that as long as I kept Windows off the internet, it didn't collect any trash, every kind of scan I was running came up clean. I didn't need any of those programs in Windows any more. My Windows systems simply are never exposed to any harm, Ubuntu does all the hard work, and it's tough enough to handle the hazards of the internet. Actually, neither I nor my wife use Windows any more these days. We have freinds who still do, but many of then are switching too. Therefore, we don't feel we need any antivirus and various other protective programss anymore, and Windows (on the rare occasions when we decide to boot it), is much quicker without them.
>  5) if i wanted to rid myself of ubuntu and start over, how would i do that? I have a web page listing severl ways to do that, here: [Un-install Page]("http://www.users.bigpond.net.au/hermanzone/p18.htm").
>  if i wanted to rid myself of windows i would i do that. That's easy, just boot your Ubuntu Live CD and go 'System'-->'Administration'-->'Partition Editor', and delete your Windows partition. You would save all your files somewhere first of course. Then resize your Ubuntu partition to fill the available space. Resizing a Linux partition from the start point will be slow. You would find it a lot quicker to install a new Ubuntu, say Hardy Heron when it gets released in April, in the place where Windows used to be. Hardy Heron will automatically mount your existing Ubuntu installation, and you will easily be able to drag and drop your files from your old /home/username directory into your new one.  Then either keep your old Ubuntu install for a rescue, practice and testing installation or delete it and resize the new installation to occupy the entire disk. I have large hard disks compared with the amount of files I store, so I always have two or more Linux operating systems per hard disk. The operating system itself doesn't take up very much disk space. 

Regards, Herman :)
