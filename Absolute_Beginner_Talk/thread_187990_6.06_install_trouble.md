### Post by catlett on 2006-06-03
That repository is one that automatix uses. If you kept the automatix repository list after you ran automatix you will have that repository. 
404 is a generic code which means the server cannot be connected to. Either the url is not valid any more, the server is down or for some reason syanptic can't connect to that server.
If you want you can keep it and deal with the nuisance of an error,  delete the repository from your list or comment it out in case you want to enable it later.
If you want to delete or comment out the url then enter```
 sudo gedit /etc/apt/sources.list
``` Look for the koti entry and either delete the lines entirely or put a "#" in front of it to make it a comment and therefore synaptic will ignore it.
