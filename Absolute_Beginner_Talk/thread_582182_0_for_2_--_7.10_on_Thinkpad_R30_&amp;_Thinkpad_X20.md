### Post by Anessen on 2007-10-19
It's probably a RAM problem. Neither of those laptops come installed with enough RAM to run the Live CD.

Remember that the Live CD requires at least 256mb to run properly. It actually has to load the required files into RAM and boots Ubuntu from the CD. Instead of having a hard disk to store things on, everything has to go into RAM.

You may need to use the text install instead. I can't remember if you get the option on the normal Desktop install Disc, or if you need to use the Alternate CD. Check what options you get when you see the discs menu. If you do need the alternate, there's a checkbox on the download page for it.

### Post by shanen on 2007-11-29
I was experimenting on an R30 with Gutsy, and it isn't just a RAM problem. This target machine has 384 MB of RAM, which is supposed to be sufficient. I tried some of the parameters for older machines, but I think it's just too old a machine to be supported.

Trying it with the 6.06.1 LDS installer doesn't seem to be any better. I think that pretty much eliminates memory as the problem, since the RAM requirements are much less. Didn't really get fancy with the parameters, but the VGA mode didn't work, and the "noapic nolapic" options didn't help. I went as far as trying the 6.06.1 server CD.

I don't know if it's a useful hint, but the point of fatality seems to involve an attempt to disable IRQ 15 in an infinite loop. Doesn't seem to be worth further efforts... :confused:

My belief is that the X20 is about the same vintage, but I don't have one to test with.

### Post by shanen on 2007-11-29
Another thread suggested preparing a swap partition in advance, but that didn't work. However, among the various confusing error messages was one that mentioned trying the irqpoll option, so I inserted that near the end of the command line when it starts, and it seems to be succeeding now... Probably won't have time to assess the entire thing today, however.

On edit: Completed the base install yesterday, and currently doing the upgrades, and it seems to be running pretty well, if a bit slowly. The "irqpoll" option does seem to have done the trick.

Another problem with the machine was just an age-related thing: Try it without the battery. If the battery is funky enough, it will seriously interfere with the operation of the machine, and if you have the original battery, it's surely past it's best-used-by date.
