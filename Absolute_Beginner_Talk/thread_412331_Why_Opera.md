### Post by Tundro Walker on 2007-04-18
The Opera you can get for free is adware, in that it posts up banner ads in it's header area while you're using it.  But, even things like Adobe Acrobat Reader do this now, showing annoying links trying to entice you to click something.

For what it does, Opera is fast, has mouse-gestures, but has ads.

Epiphany is a Firefox knock-off, and it's fast, too, but you can't extend it with plugins.  Plus, some pop-ups still come through (Firefox even has this problem sometime...I guess the advertising geniuses have found new ways to annoy us.)

Default Firefox that comes with Ubuntu is slow, but you can d/l Swiftfox which is a Linux optimized build.  I had to manually install it when I got it, but I think the Swiftfox team offers a repository you can add to your repo list for Apt-get / Synaptic package d/l'ing.  This would actually be better, since you can then use Synaptic to manage updates for it.  Swiftfox is supported by a different team then Firefox, but they're pretty good about updating it as Firefox gets updated.  With Swiftfox and Firefox, you can install a ton of plugins to do great things, like mouse gestures, adblock, flashblock, convert your text to look like the french chef from the muppets is saying it (seriously, it's called the "bork bork" plugin...I'm not making this up!), and other things.

Those are the main ones GUI-oriented folks go for, since they look nice and don't sacrifice too much functionality.  However, if you have a slower machine, and can live without some of the fancier features (like flash), you can try Dillo, which is a very light-weight GUI web-browser.  Not as flashy looking as the others, though.

And, of course, if you have Ubuntu installed on your 386, you can use W3M or Lynx which are terminal-only web-browsers.  I'll leave it up to you to figure out how they work, because personally I haven't used them much.  (Yes, I have used them, just to try it out, but I'd rather use GUI then text-based.  However, it is pretty cool that someone stuck with terminal only isn't left browserless...let's hear it for scalabiity and variety!)
