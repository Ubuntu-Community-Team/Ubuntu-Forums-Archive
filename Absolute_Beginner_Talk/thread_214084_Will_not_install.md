### Post by timetunnel on 2006-07-12
Maybe the ISO file itself was already broken due to a corrupted download. You can always check if a downloaded ISO file arrived correctly by downloading the MD5SUM file as well (from where you get the ISO). Some burning tools have a built-in function to check an ISO if the MD5SUM file is there, or you can do it manually with one of the free md5sum tools available for Windows and Linux.

### Post by timetunnel on 2006-07-12
> **music_man said:**
> How can it get easier? I haven't the foggiest idea of what to type or do to get these programs running.

I guess you're using Windows at the moment. So, go to the site where you downloaded the ubuntu ISO file from and download the file "MD5SUM" from there. Put it into the same directory where the ISO file is. Then download and use this: 
[http://www.download.com/3000-2248-10050856.html](http://www.download.com/3000-2248-10050856.html)
It's a tool to check the md5sum. If it says the md5sum is incorrect you have to download the ISO file again and check the md5sum afterwards. Once the ISO is valid the installation should be quite easy - provided that you don't have rare and exotic hardware. 

As for the "hard to do" and "command line" part:
it's a bit like driving a bicycle: you can't drive it (at least not safely) without learning the traffic signs and rules. Once you know the traffic signs it gets easier and more relaxed. If you're going to another country (operation system) you'll feel a bit uncomfortable again because there are different rules and signs. But you'll learn again. So, don't give up. Linux isn't perfect, but so isn't Windows (it's even less perfect). The good thing about the command line in Linux is: when something doesn't work, you'll be very likely to make it work with the command line and some help from the forums. You will rarely be able to do that if something doesn't work on Windows.
