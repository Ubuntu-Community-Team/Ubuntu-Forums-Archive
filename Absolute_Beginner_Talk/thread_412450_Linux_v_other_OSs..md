### Post by BoneKracker on 2007-04-18
That's absurd (the quote in the original post).

GNU/Linux does suffer from the fact that the vendors of some hardware may not create drivers for Linux or release the source so that they can be easily adapted.  That's the only reason for some devices may not be usable.  The statement, "and never will" shows you this person does not know what they are talking about.  Vendors are quickly jumping on the GNU/Linux bandwagon.

Personally I have had more trouble getting drivers for my 64-bit version of Windows XP than I have for any of the three different architectures on which I use Linux.

As to "keeping up with the development of other OSes".... that's laughable.  If that were the case, why would Apple have chosen to re-use FreeBSD as the basis for a brand-new operating system?  And I think the one of the main reasons they chose Mach over Linux was that the kernel development is more stable (by "stable" I mean released in a more controlled, deliberate fashion, as opposed to a continuous stream of innovations).  Why?  There are an army of developers and a bigger army of users all contributing to the improvement of gnu, bsd, linux, mach and other free software.  It is they who cannot keep up.

Microsoft has copied almost everything that comprises Windows from some other operating system.  After years of trying to BURY the command prompt, they are now spazzing around releasing some kind of a "shell" for administrators to use so it will be more like Linux.  They are desperately positioning "Get the Facts" ads (which are so fallacious they have been sued) opposite every big story about how great Linux is in the major IT periodicals.

But one of my favorite truisms is that "people don't know what they don't know".  Your compadre has probably never used anything but Windows.  How would he/she know how it compares in any regard?  I have used Microsoft and Apple operating systems for over 24 years. I've used unix-based operating systems for nine.  I've extensively used Linux and BSD for the past three.  

Windows is a nice product.  I personally choose not to use it because is has far too many fundamental security flaws.  If there's a "never" to be spoken in this conversation, it is this -- unless it is completely re-engineered (like MacOS was), Windows will NEVER be safe enough to plug a network cable into.

Aks your flight sim buddy how many of the top supercomputers in the world run on a Microsoft operating system (last time I looked, there was 1 -- way down near the bottom of the list -- and even MacOS has more than that).  Ask him if he knows how many of them run on Linux (about 80% as I recall).

### Post by rillip on 2007-04-18
That last statement seems to me very poorly informed.  What periphery and components is he talking about?  

My experience is thus:

Windows: instlall new hardward.  50/50 it gets picked up, even though it's plug and play.  If it is, it asks to install drivers, but doesn't work and I have to run a third party provided installation program.  If it isn't, I have to run the program, 50/50 it'll work.  If it still isn't working, it's weeks worth of figuring out that it's USB 2.0 isn't compatible, or I'm missing the monitor driver that can handle my graphics card driver, etc.

Linux:  hardware detected. About 25% chance it isn't auto-detected and configured immediately.  If it's not, it's probably mostly functional, but missing a few bennies (example: my mouse has five buttons, only three work; haven't taken the time to get the other two set up because I don't use them much).  If it just totally doesn't work, good chance I can get help for it and make it work in the same ammount of time as a Windows problem.  I've only had one instance like this (my graphics drivers didn't sit well at first for some reason).

Overall, I'd give the hat tip to Linux.  I've never seen it be harder than Windows, I can get more help than with Windows, and more works out of the box.  I think that person just tried a version of Linux that didn't have as robust of hardware detection for his particular hardware that many of us have.

And I agree with trishacupra, Linux is much more rapidly developed.  If the next iPod came out tomorrow and used USB FX4 (I'm just making that up), you can bet that there would be an unstable (i.e. development) version of just about any Linux that suported it.  Windows?  At least a few months before the next update tries to install it, and maybe breaks something else.  But, you won't really be given a choice about installing it, it'll do it by itself.

And compare it to MacOS.  Let's see, Linux recognizes whatever it wants, MacOS recognizes... Mac hardware.  Because that's all you can use.  Period.

I think this person didn't know much about Linux and probably had one bad experience in the past.
