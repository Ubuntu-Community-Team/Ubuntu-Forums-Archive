### Post by sab0403 on 2007-07-10
Figured it out myself. Editing the fstab```
sudo gedit /etc/fstab
```I simply rewrote it so my new partitions would be reflected there. My fstab finished as follows:```
# /etc/fstab: static file system information.
#
#  -- This file has been automaticly generated by ntfs-config -- 
#
# <file system> <mount point>   <type>  <options>       <dump>  <pass>

proc /proc proc defaults 0 0
# Entry for /dev/sda1 :
UUID=2d2f4049-e1e5-4770-9533-0e1e93e59cef / ext3 defaults,errors=remount-ro 0 1
# Entry for /dev/sda3 :
#UUID=12A91374B6A85851 
/dev/sda3 /media/sda3 ext3 defaults 0 0
# Entry for /dev/sda6 :
#UUID=941d21cf-cdc1-4989-8caa-7a30fdbbb2dc 
/dev/sda2 none swap sw 0 0
/dev/scd0 /media/cdrom0 udf,iso9660 user,noauto 0 0
```As for permissions, I created the pertinent folders in /media first (with sudo), and then I changed ownership on those folders (with chown), which in turn gave me permissions over it. Such ownership gives me permission, since I'm actually modifying the /media/whatever directory, not the dev. Oh well... marking it as solved.

Thanks anyway!
