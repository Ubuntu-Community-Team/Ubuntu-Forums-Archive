### Post by aysiu on 2006-12-02
Yeah, technically it's not searching your own computer--it is searching online software repositories, so you do have to have an internet connection for the package manager to work properly.

The big difference is that, unlike in Windows, you don't search using Google and track down a .exe. Also, only software will appear in your search results in the Package Manager.

For example, if you're searching for *istanbul* in Synaptic Package Manager, the only result you're going to get is the desktop session recorder.

If you search for *istanbul* using Google, you'll get a bunch of results for the city of Istanbul and then maybe somewhere in there the desktop session recorder application.

### Post by aysiu on 2006-12-02
> **Axis said:**
> Thanks um....you still didn't answer my question from above. If I am installing ubuntu will I be able to get directly into firefox after I install it as long as I have a working internet connection to the computer?

And is it completley nessasary to have an internet connection to use the Package Manager?
Firefox is a default application in Ubuntu, so you can get directly into it. In fact, if you're using the Desktop CD to install Ubuntu, you can use Firefox *while* you're installing Ubuntu.

As for the working internet connection... well, if you're using a wired broadband connection, it will probably be automatically working during and after installation. If you're using dial-up or wireless, it may be a bit trickier to set up and may not be working right away.

And, yes, in order for the package manager to be useful, you need an internet connection. There's [a community-built add-on CD](https://ubuntuplus.bountysource.com/), but Ubuntu doesn't have any official add-on CDs. It relies heavily on the internet for software installation.
