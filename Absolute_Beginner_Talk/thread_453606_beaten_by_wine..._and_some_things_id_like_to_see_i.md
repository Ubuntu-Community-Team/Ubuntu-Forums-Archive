### Post by Sparkster185 on 2007-05-24
I don't think we're going to see games developed natively for Linux for a long time (if ever).  Think about it, you're a game development company.  Your goal is to sell games and make money.  Now compare the desktop share of Windows to Linux.  Is it really worth your time/money to get it to work on Linux too?

Besides basic architecture issues, most game designers program to Microsoft's Graphics API (Direct X), since it's more powerful than OpenGL.

### Post by disc on 2007-05-24
Linux isn't really known for gaming. Most PC gamers are in fact Windows users, so there isn't enough incentive to develop or port games to Linux. If you are interested in running games natively in Linux, look up [Loki Software]("http://en.wikipedia.org/wiki/Loki_Software"). They've ported some of my favorite Windows games for Linux, but sadly they are now defunct. 

As for WINE, you should look up your game in the [WINE appdb]("http://appdb.winehq.org/appbrowse.php") to see how well it will run in wine, and read tutorials on installing it as well.
