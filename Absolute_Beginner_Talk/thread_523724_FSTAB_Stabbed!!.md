### Post by jspangler on 2007-08-12
Hi, it's possible first that your UUIDs might have changed. You can find the right UUIDs by running:

```
ls -l /dev/disk/by-uuid
```

Post the output of that.

Other than that, you might want to make sure that all of the mount points (i.e., /media/Backups, /media/sda2) still exist. That could cause a problem.

### Post by ridgeland on 2007-08-12
UUID is a unique ID for a partition (or device that has one partition maybe).  Rather than say /sda1 you can use a 20 character ID.  I hate it.
So in your case you can say either
/dev/sdb8 
or UUID=d5919c23-13bc-4bb8-b6ab-cb645d7ad68c
BUT... if something changed the UUID then your fstab reference points into never-never land not to /dev/sdb8
I much prefer /dev/sdb8 to the UUID. 

Please translate two mounting perfectly to /dev/sdxx....
Which partitions are OK?  which don't mount? (/dev/sdxx)
I still expect the bad ones are the vfat with default 0 0.

To verify the mount points exist just look for them in Nautilus.  If you mount to /mount/ExtDrive then that has to exist in Nautilus before you boot.
