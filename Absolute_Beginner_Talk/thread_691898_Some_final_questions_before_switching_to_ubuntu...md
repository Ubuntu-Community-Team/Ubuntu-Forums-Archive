### Post by Ahunt on 2008-02-09
To all those folks who keep saying that you don't need anti-virus protection on Linux:

Please read:

[http://www.cbc.ca/technology/story/2007/06/11/badbunny-worm.html](http://www.cbc.ca/technology/story/2007/06/11/badbunny-worm.html)
[http://www.symantec.com/enterprise/security_response/weblog/2007/06/bad_bunny.html](http://www.symantec.com/enterprise/security_response/weblog/2007/06/bad_bunny.html)
[http://www.symantec.com/security_response/writeup.jsp?docid=2007-052400-3656-99](http://www.symantec.com/security_response/writeup.jsp?docid=2007-052400-3656-99)

There are viruses being written for Linux everyday. Why? Because Linux is used in lots of handhelds and lots of servers, even if desktop use is still under 1%.

As illustrated above there are cross-platform viruses being written as well. When you advise Linux users that they don't need a virus scanner you leave the whole community more vulnerable. As mentioned in this thread, you also need it to make sure you aren't passing on Windows viruses to Windows users, too.

If you are running Linux you need virus protection. Fortunately is is readily available. Ubuntu has CLAM TK/CLAM AV available in the add/remove repositories. It is open source, easy to use and works well. AVG is also available for Linux.

Telling new users that they don't need antivirus may have been correct in the 20th century, but not anymore. Protect yourself and the rest of the Linux community - get it and use it.

### Post by JeSTeR7 on 2008-02-09
I'll just add that Ubuntu definitely has CPU frequency scaling.  If you rightclick on a panel, choose "Add to Panel" and select the CPU frequency scaling monitor, it will add an icon that lets you know what it's running at.

Then, go to terminal and type sudo dpkg-reconfigure gnome-applets

It will give you a dialog box, if you choose yes, it allows you to control the CPU frequency manually using the icon you just added to the panel.

Works like a friggin charm.   I can keep my laptop running for almost 3 hours on battery by setting the cpu to 600mhz.

### Post by peabody on 2008-02-16
> **Ahunt said:**
> The threat is not zero:

[http://en.wikipedia.org/wiki/List_of_Linux_computer_viruses_and_worms](http://en.wikipedia.org/wiki/List_of_Linux_computer_viruses_and_worms)

Consider: 

If those who say "get a virus scanner and use it" are wrong, what is the cost? 

If those who say "there are no viruses for Linux and never will be" are wrong then what is the risk?

You are espousing conventional wisdom that is in the best interests of security software vendors.  I would have you consider the following story:

[http://www.codinghorror.com/blog/archives/000803.html](http://www.codinghorror.com/blog/archives/000803.html)

It's a bit of a long read, but I think it makes a good case against the conventional wisdom of anti-virus software.  At its core, anti-virus software is a sophisticated blacklist of known malicious programs.  This is bound to fail in zero-day exploit situations.  I've witnessed it first hand in jobs.  It is a "band-aid" approach.  Killing as much as 50% of your computer's performance using expensive commercial software that CANNOT and WILL NOT guarantee your safety is akin to madness.  In the case of rootkits that make it past your defenses, anti-virus software will do ZILCH for you.  That's game over.  Time to reinstall.

It's time people talked about the real way to prevent these sorts of problems, such as increased browser security (firefox + noscript) and the principle of least privilege (observed in Linux for years, Mac OS X since day one, and finally somewhat with Vista).

REAL security starts at the application, not at sophisticated blacklists.
