Believe me, I know it's hard to believe, but so far there haven't been any wide spread problems of that kind. I think there are several reasons off the top of my head:

1) Linux systems like Ubuntu are only used by a rather small percent of computer users (home users, I should specify).

2) Most Linux application and code authors are doing it for themselves and sharing it if anyone wants it (as opposed to having any financial interests in it, legit or otherwise).

3) Most Linux systems, Ubuntu as well, are centered around an organized team who utilize and / or adapt the pool of applications and code made available for the purpose of creating useful computer software people can share interest in and put to use; they provide most if not all of what is needed for most users via a secure distribution of pre-filtered software.

4) Most linux code is freely distributed and thus any malicious code can't hide from the code-literate linux users. What many windows software providers are doing with information they may or may not tell you they're having their close-code software "phone home," who knows; they promise what's going on in the license but mostly the licenses tell you what you can't do, like inspect the software you're running on your own computer.

5) Well made Linux OSs like ubuntu are innately harder (permissions, iptables etc) to exploit than Windows.

6) If there were shady linux softwares, it would be much less likely to be used to begin with as most linux users are more "computer savvy."

7) Many linux users tend to be more "in touch" with other users, and were there a major issue, would probably tend to get any alarm more accurately and readily than most windows users, and would also be more likely to know how to address the concern.

All that and more conspires to the benefit of Linux users.

### Post by P4man on 2007-11-15
The thing to understand here is that a worm/virus/malware designed for windows will simply not work under Linux (or MacOS for that matter, in most cases even not on Windows 9x!).

Most virusses exploit some unpatched security vulnerability in windows, or one of its applications (like Outlook, Word, ..). Linux would not have these same exact same bugs/vulnerabilities (though it might have other ones), and it wouldn't be able to execute the malware code if it wanted to. So your windows partitions or disks are quite safe while running Linux. They would only be at risk from a  Linux virus/worm/exploit. 

A Linux virus  is not impossible in theory, and there certainly are exploitable holes in Linux, but so far I'm not aware of any virus that actually works on Linux. There are some browser vulnerabilities in Firefox and the like, but these wouldn't affect your windows disks.

The opposite however, is not necessarily true. A windows virus that infects your system while you are running windows, could destroy your linux partition -even if you can not access Linux partitions from windows, erasing a partition or disk is always possible, unless you would physically disconnect them. That is not a likely thing to happen either though, most virusses are more subtle than erasing (unreadable) partitions, but its possible at least.

To complicate things further :) if you run windows on a virtual machine (VM) inside Linux, using virtualbox, VMware or similar), note that your virtual windows would certainly be vulnerable to virusses (assuming you allow your VM to access the internet). If you also allow your VM read/write permission to you linux disks (convenient but dangerous), then yes, those could be compromised by a windows infection, because the VM would be able execute the malware, and Linux would help by exposing its filesystem to windows. That is something you should be careful about, and if you use virtual machines, its a very good idea to install an antivirus on the guest OS (windows) and  limit where it can read/write so it can not compromise the host OS (linux).
