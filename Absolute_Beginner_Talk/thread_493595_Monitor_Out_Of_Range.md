### Post by freebird54 on 2007-07-05
Your GUI is controlled from a text configuration file.  **(/etc/X11/xorg.conf) **In it are the descriptions of what drivers it is to use, and what specifications it may use on your monitor (among other things).  With the correct specifications for your monitor, you can edit this file to correctly identify the ranges that are allowed, and the problem will go away.

So - possible ways to proceed

1. Start Windows, find out what settings *IT* uses for you, and modify the file to match.
 OR
2. Google your monitor model (or look at the specifications if you have them) and modify the file as needed
 OR
3. Look at the current settings in your config file, make sure you know what driver you are using (presumably fglrx - but check) and run 

```
sudo dpkg-reconfigure xserver-xorg
``` 
- following the suggestions found [ here ](http://users.bigpond.net.au/hermanzone/p7.html) on what and how to answere the questions...
 OR
try this command

```
cat /etc/X11/xorg.conf
```

and paste it here for use to try to figure it out for you! (along with your monitor information)

Have I confused you with choice?  Sorry - but there it is! :)

Leet us know how you want to proceed, and we'll be happy to provide methods appropriate...

### Post by freebird54 on 2007-07-06
Ok - this is how we get there from the Live CD:
First we make a directory to be the 'mount point'

```
sudo mkdir /mnt/ubuntu
```

then we 'attach' the drive to that directory

```
sudo mount -t ext3 /dev/hda1 /mnt/ubuntu
```

then we can access the file

```
cat /mnt/ubuntu/etc/X11/xorg.conf
```

and paste it in as before...

### Post by freebird54 on 2007-07-06
What he was referring to was a way of forcing a video mode from Grub bootup.  Here is a table of values that could be used:
```

Colours   640x400 640x480 800x600 1024x768 1152x864 1280x1024 1600x1200
--------+--------------------------------------------------------------
 4 bits |    ?       ?      770       ?        ?        ?         ?
 8 bits |   768     769     771      773      353      775       796 
15 bits |    ?      784     787      790      354      793       797 
16 bits |    ?      758     788      791      355      794       798 
24 bits |    ?      786     789      792       ?       795       799 
32 bits |    ?       ?       ?        ?       356       ?


```

As you can see, the 1280 X 1024 value is 795.  It is used in the grub setup file as so:
```

kernel          /boot/vmlinuz-2.6.20-16-generic root=UUID=ac6bc287-3721-46fb-9485-8bfe687828cb ro quiet splash vga=795
```

just added on to the 'action' line of your default boot choice.  (the example is from my system - yours may differ - but add on at the end :) )

Hope this helps...

BTW - the piece you showed was an 'example' of setting the default options - wherever the line starts with ## it is a comment.  For more information on how grub works, a readable source of information is [here](http://users.bigpond.net.au/hermanzone/p15.htm#splash).
