### Post by mwshook on 2006-06-23
I moved and will soon be unpacking my (non-working) XP desktop. When I install Ubuntu, I might be what you would call a born-again newbie.

I started out trying Redhat, Debian and Slackware in 1997 (my freshman year in college), back when Slackware had the most user-friendly install process. I moved farther and farther from Win95 until I exclusively used Linux from 2000 until gradutation in 2001.

When I went to medical school I got a new laptop and desktop. For various reasons, I stuck with WindowsXP, and never re-tried Linux again.

One of my biggest gripes with Linux (other than the zealotry) was continually getting stuck in dependency hell. I'd have an annoying bug in an application, so I would download tar.gz of the latest version (RPM not yet available), which would require new libraries, which would require me to install newer tar.gz files of several other programs, repeat, until half of my apps would be broken. 

I've been hoping the state of Linux has improved in the last 5 years. I've been most intrigued by Ubuntu's package management system and that "make install" doesn't come with the default setup.

What I really want is reassurance. If I stick to downloading programs through Add/Remove applications, will my system (hopefully) remain clean-running. Or do most people find themselves forced into using make and mucking everything up?

### Post by steve.horsley on 2006-06-23
You will be stunned by the changes since 2001. I don't think many Ubuntu users know what dependency hell is. You can still get a taste of it if you insist on installing and compiling from network sources, but you very rarely need to. But you can still find DLL hell in windows if you try hard enough, too.

Be reassured - you probably won't need to stray outside the Ubuntu repositories for a ling time.
