### Post by tgalati4 on 2007-05-05
Also important is that new users post solutions to problems that they fix themselves.  There is such a wide range of hardware out there, the more solutions that are posted the better chance of finding a solution to your particular problem.

For instance, I stalled SUSE 10 (SLED 10) and sound wasn't working.  I searched the SUSE forums but no answers.  Searched the Ubuntu forums and found 3 responses that had the switches for the module that I needed to load.

So posting answers is as important as searching first before asking a question.

I agree that the forum posts do age quickly with the Ubuntu release schedule.  But the basic concepts are valid and if you can take the concepts from an older release and apply them to the new release, you can normally fix things.

Sometimes module names change, switches change, configuration file names change, file locations change.

The sound card module mentioned above had a switch io=0x0358 that gave an error.  I changed it to port=0x0358 and that worked, remembering a solution from an older distro (Breezy).

So:  

Search first and ask quesitions later.


Oh, and don't forget to post solutions--more than hey I got it to work!
