### Post by jvc26 on 2007-01-05
I think as sasa said we need a fair bit more information. What I gather is you're using tinyXP but you have no defrag option? - If so havent you ever defragged the pc? If you havent, then the odds are that your drive is so fragmented there is no easy way to resize the partitons to put Ubuntu on. You will need to defrag before you try so either you will have to get your hands on a defrag program for the XP, sort it out then resize and install ubuntu - or if its not a problem you could remove it, reinstall tiny XP and Ubuntu and start afresh - it might be quite a jump to go all in to ubuntu when you havent had a chance to try it out properly - it isnt a different version of windows and so you may find you cant do everything you want to be able to do on it - its well worth doing a dual boot first to see whats what and seeing which you use more/prefer.

### Post by jvc26 on 2007-01-05
- If you're going for the dual boot option you have to defrag before you install Ubuntu (as there is no continuous amount of space on the drive)
- If you are 100% sure you want to go for Ubuntu all the time you could save all your data (wouldnt want to lose all that stuff) and then wipe the whole HDD and load up Ubuntu on its own, reload all your stuff and have done with TinyXP
- A freeware defragmenter may exist - not 100% sure myself, but a google search may reward your efforts here is a possible:
[http://www.softpedia.com/get/System/Hard-Disk-Utils/Power-Defragmenter.shtml](http://www.softpedia.com/get/System/Hard-Disk-Utils/Power-Defragmenter.shtml)
- The other question is, your 6 years old pc may struggle with ubuntu due to very old components - how much RAM etc have you got? If you're pretty low on RAM you might want to try to run Xubuntu instead which uses the Xfce desktop environment which is much lower on system resources that Ubuntu.
