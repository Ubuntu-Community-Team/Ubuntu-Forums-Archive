sudo mkdir /media/anyname

Then reboot.

There are several ways you can specify read/write permissions but i prefer to specify the users group id (1000).

### Post by eph1973 on 2007-10-02
With the chown command, you don't need to specify a group (it looks like the group "jp" does not exist).  If you are the owner of the file, you should be able to do everything you want to it, anyhow.

So, you could try this:

```
sudo chown -R jp /media/disk
```

Making sure that there is a space between jp and /media/disk (also a space between chown and -R and -R and jp).

Secondly, it looks like your fstab says:

```
/media/sdbl <----looks like the letter "L", instead of the number "1"
```

Thirdly, it looks like your blkid correctly identifies your new HD as /dev/sdb1.

To remove a directory, in terminal:
```
rmdir /media/test9
rmdir /media/test2
```

If you do not have permission to delete it, and are sure you want to, then type:
```
sudo rmdir /media/test9
sudo rmdir /media/test2
```

```

### Post by eph1973 on 2007-10-02
The problem was that there is not a group named "jp" on your computer.  So when baysl way back at the beginning of the thread told you to type:
```
chown -R jp:jp /media/disk
```It failed, because there is no group named "jp".  In brief:

chown is the command for "change owner" of a file or folder.

If you read man chown, it goes into some detail, but to explain about this specific situation:

the -R is the option for change the ownership of the directory and all the subdirectories below this (recursive, which is what the "R" is for).

the jp:jp changed the owner to jp (the first jp) and the group (the group of users who can have access to this file or directory) was the second jp.  However, since there apparently is no group named "jp" on your computer, the command failed, and no ownership changed, which is why you problem remained.

I recommended only to change the owner (hence only the one jp), which gives you all the permissions you need, based on the results on your ls earlier.  Since you probably aren't concerned about groups, and the group he recommended would have only included you, anyway, the second jp was pretty much unnecessary.

That's pretty much the why of it in a nutshell.  I would recommend [http://www.linuxcommand.org/](http://www.linuxcommand.org/) if you are interested in learning more about the terminal.

Now to address your fstab:  you do not have an entry for your new hard drive in your fstab.  You may consider adding the following entry into your fstab:
```
sudo gedit /etc/fstab

#####COPY AND PASTE THE BELOW LINES INTO FSTAB#####
# Entry for /dev/sdb1 :
UUID=2793fd93-21f9-4790-bf2e-b410b17741c9 /media/disk ext3 users,exec,dev,suid,rw,auto,async 0 2

```If you still have problems, let us know.  Good luck!
