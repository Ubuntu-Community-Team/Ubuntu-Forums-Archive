### Post by helixed on 2007-10-07
I own a Seagate Freeagent Pro External Hard Drive, which I have connected to a USB port.  The other day I sent the following question to Seagate:

Hello, I'm running the Seagate Pro external hard drive on Ubuntu Linux through a USB interface. The drive works fine at first, but then after a while Linux cannot read off of it. If I unplug the drive and then remount it, there are no problems. I heard from somebody that this might be a "spin down" issue, where the drive lowers its spin while it's idle, and then Ubuntu cannot start it back up. I was wondering if there was a Linux driver or anything which could help with this issue.

They sent me back the following reply:

Thank you very much for your e-mail enquiry.

The drive automatically spins down after period of inactivity. This is a feature specific to the Seagate external Drive and occurs in both Windows and other Environments; regardless if it is preformatted as NTFS or other file format. Unfortunately, the drive does not support Linux platform so, please try connect drive to windows OS, and adjust the Power Setting via drive manager utility should helps.

Anybody have any Ideas on how I can get around this?  It's really annoying having to unplug my hdd every time I want to use it.  

Thanks for the help,

Helixed
