1. Not all games (most in fact) have linux binaries, meaning that they won't work out of the box.
I can tell you that I play Starcraft using Wine (Window's Compatibility layer). It's freely available, using completely non-MS code to run MS-Software.
A company makes a linux gaming product called Cedega which will play basically everything at least semi-functionally except those new DirectX 9 games. And this is being worked on. The catch is that you have to subscribe for $5 a month, for at least 3 months, meaning you have to pay $15 minimum. From that point on, you can stop paying and keep playing.
There are some games that you can play without Wine or Cedega. [Loki]("http://www.lokigames.com/") Installer's can install a lot of games.
2. I don't know for a fact with your digital camera, but most likely it will. If it is a USB interface, I would say it likely works.
3. You shouldn't have to worry about that. Let the installer set everything up.

You should try the liveCD to see if networking/sound get setup, and try plugging in your Camera. It is a good risk-free way of trying linux. 

If you find that most of your games will require Cedega, you might just dual-boot Windows and Ubuntu. Boot into Windows for your games.

You don't HAVE to go all the way

Also, if you decide to install, you may want to pickup [Automatix]("http://ubuntuforums.org/www.ubuntuforums.org/showthread.php?t=138405")

### Post by mostwanted on 2006-05-23
WINE is not an emulator, it's an open source clone of the Windows APIs that works on *NIX systems. Therefore, it's not nessesarily slow, in some cases it's quicker than Windows.

As far as I know, both Steam (and Counter Strike), Diablo II and Star Craft will work in WINE.

As for the other questions:

2. Pretty much any digital camera or storage device is auto-detected and mounted when you plug it in.

3. We can't really tell when we don't know what cards you're using can we? ;)
