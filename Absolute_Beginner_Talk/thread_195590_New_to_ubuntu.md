### Post by u.b.u.n.t.u on 2006-06-13
The best way to learn terminal commands is to use them for something.

This is a fun command:

```
glxgears -iacknowledgethatthistoolisnotabenchmark
```

Also try this:

"pwd" = where am I now, what directory.

"ls" = what files exist where I am now?

Learn what all the directories mean.

What is "bin"?  What is "usr"? What files are in there?

Google can help with this.

My terminal knowledge is basic. I like the GUI and only use the terminal when I have to.

There are even basic tutorials here at this forum I have seen.

### Post by 3rdalbum on 2006-06-13
Once you're familiar with the basics of the terminal (e.g. how to move around directories and start programs with arguments), you should check out the SimplyMepis issue of Linux Format magazine (assuming you can still get it) and the one before that. They have articles about handy things you can do with the terminal; For instance, with just one command, I can tell my computer right now to download something on the 25th of June. With one (fairly simple) command, I can ask the computer to search through my Apache logs and see if anyone hit my site using the Opera web browser.

And this is just the tip of the iceberg. More awaits you; heck, more awaits me.

One piece of advice I can give: If you want to use a terminal for any period of time, you've gotta download the Fish shell. There's a Debian package available for it. Trust me, you'll thank me later for the tip :-)
