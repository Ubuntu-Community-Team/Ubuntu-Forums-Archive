You generally don't need a password for the live session. *Sudo* commands will be executed without the need for a password.

Otherwise, you can always *set* a password in System > Administration > Users and Groups.

### Post by yukaili on 2006-04-24
I had the same problem. A temporary sollution that I used is to change the password to your choice. I went to (assuming you haven't changed the menu) System -> Administration -> Users and Groups. Then I clicked Show all users and groups. I scrolled to the bottom of the list and selected the user Ubuntu. Clicked Properties. Then there's the password fields. Just replace the passwords with the password of your choice. I choosed "live" because it's easy to remember. Then click OK on all of the boxes and you now have a temporary solution. If you lock yourself out, just enter the password you entered and click Unlock.
