### Post by Cypher on 2007-06-13
The "latest" version of Ubuntu you installed is a snapshot of the way Ubuntu looked about 3-5 months ago when the ISO was created. Since then there've been numerous ongoing changes that are sent as updates.

Do you have to take the updates? No, of course not, if you're perfectly happy with the system the way it is and have no issues, you perfectly have the right to deny any updates, but it is usually advisable to check out what the updates are just incase you want to get it. 

Most application updates will largely be transperant to you and should have no adverse effect. The one update that will has an impact is usually Kernel updates. If you are not making any modifications to the default Kernel that is being used, then getting the updates is less of a hassle. But if you've had to add modules/recompile Kernel for some specific hardware, then obviously getting the update will void all that work.

### Post by ramjet_1953 on 2007-06-14
The only thing that I hold off on for about a week is kernel updates.

This is somtimes wise, as some kernel updates can cause real problems, especially for those with restricted video drivers and wireless drivers.

There has been a big issue lately over the new HDD drivers that treat IDE drives as scsi, with associated problems with unmounting and spindown.

I always wait for the dust to settle and especially scan these forums for any negative reports.

If there is a problem, within a day or two, a pretty clear picture is emerging.
