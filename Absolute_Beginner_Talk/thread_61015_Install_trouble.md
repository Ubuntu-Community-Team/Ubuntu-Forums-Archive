### Post by heimo on 2005-08-30
First things first, welcome to Ubuntuforums! :)

I'd definitely recommend getting some more RAM - if it's impossible, at least borrow for installing. I'm not sure if it's possible to install with 32MB. Even if it's possible and you want to use graphical user interface, something less resource hungry than default install is needed.

If you haven't seen this... Be sure to check those links at the end of this page:
[https://wiki.ubuntu.com/Installation/LowMemorySystems]("https://wiki.ubuntu.com/Installation/LowMemorySystems")

Searching forums, I found that some people have been successful installing Warty (4.10) version with 32MB memory.

### Post by Lord Illidan on 2005-08-30
For such an old-spec computer, I recommend something else rather than Ubuntu. Because with 32 mb of RAM it is not going to be possible to run either Gnome or KDE, maybe XFCE, but what about the apps. Lots of disk trashing I forsee.

You might be better off with Damn Small Linux...[http://www.damnsmalllinux.org/](http://www.damnsmalllinux.org/)

Or else consider getting some more memory.. I upgraded my 64 mb machine Pentium 2 333 Mhz to 196 mb and it can run Ubuntu very well, and at a pinch, even Windows XP..
