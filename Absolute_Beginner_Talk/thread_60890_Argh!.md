### Post by aysiu on 2005-08-29
[QUOTE=OtubrabNad]Looking at these forums and reading things like "well go to the command line and type /usr/ cng dir/ etc..." doesn't help me because it's not teaching me anything.[/QUOTE] If you want to learn something, use Slackware, Gentoo, or Linux from Scratch. If you just want to get things working, copy and paste the commands people ask you to type in. It's a hell of a lot easier for me to say

Type [i]sudo apt-get update
sudo apt-get install programname[/i]

than it is for me to say

See the Gnome foot on your panel? Click on it and go to System > Administration > Synaptic Package Manager. Then, click Reload. Wait for it to Reload. Then, click Search. Search for the programname. Then, find the actual program name in the search results and right-click it and select "Mark for Installation." Then, click Apply.

See how much longer it took for me to explain how to install the same program using the GUI? That's why people tell you to just type stuff in the terminal. It's a lot easier, and you're far less likely to mess up if you just copy and paste.

You have to make up your mind, though. Do you want to learn something or not? I'm sure if someone gives you a command and you say, "Hey, what does that mean?" you'll get a response. You can also "man" almost every command to get a little background on it. For example, if you want to know more about gnome-volume-manager, you can type *man gnome-volume-manager* in the terminal, and you'll _learn_ all sorts of stuff about the command.
