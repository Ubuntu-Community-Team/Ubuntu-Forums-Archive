Well apparently autocad can  be done in Linux, both as a native program and under wine.  

[http://www.architectafrica.com/bin0/news200411111_wine.html]("http://www.architectafrica.com/bin0/news200411111_wine.html")

Heres my opinion:  

I currently am in the phases of switching to Linux entirely after having used Windows since 3.1 in 1990's.  In college I learned to program in several languages, but mostly C, C++, and Java (going on 10 years since then though).  When I did my uni work and wrote programs, I did it all on a Win 98 computer.  It sucked, it was horrible.  I spent alot of time in the labs working on the Unix and Sun based workstations.   In short, if you want to program in C, Perl, or Python, you need to learn to do it in a Unix-type environment.  Linux is the best way to experience this in my opinion.  Plus, when you screw up your code you are writing, you can figure out what you did wrong without completely crashing your machine (which I did many many many times on my Win 98 computer).

The only reason I am still using Windows right now is because of games.  While there are some options for playing games on Linux (wine, cedega, etc) most of the latest and greatest computer games work better on Windows (this is only for now though, I believe in the future this will change).

So do what I do, run a dual boot system with Windows on one hard drive and   Ubuntu on the another hard drive.  This will give you the ability to still use windows, and try Linux in the meantime to see how you like it and also write program code under Linux.

Edit:  Setting up a dual boot system is quite easy, if you have Windows installed first, Ubuntu  installer will set up a dual boot without much fuss.  It honestly surprised me with Ubuntu how easy setting up a dual boot was with GRUB.  Don't know about LILO.

### Post by wylfing on 2006-05-09
A good alternative to dual-booting is to run virtualized instances of other operating systems. There is one single program that I have come to require for my business, for which there is no replacement, and of course it runs only on Windows. So what I do is launch a virtualized Windows 2000 instance and run the program in there. (It's actually simpler than that -- you never close the programs you use, you just suspend the virtual machine, so that it just pops back up ready to go when you need it.)

Search around for things like VMWare Workstation or VMPlayer.

The short story of it is there is usually always a way to do what you need. Although, for gaming, virtualized doesn't really work :-)

### Post by Lasse_ on 2006-05-10
Well, Windows has one really huge advantage compering to Linux: native Windows software works on it. Atleast sometimes. ;) 

At the moment I'm trying to install Corel Graphics Suite 12 on my Ubuntu system with no succes. The installer froze on the initializing phase, it has stayed that way for 15 minutes now. It could be doing something or not. Only God knows.

You wrote that you need to use Macromedia suite on Linux. That can be done using Crossover Office. I have Dreamweaver MX installed on Breezy with Crossover. I guess it can be done with Wine too ( Crossover is just tuned up Wine with a price tag ), but on Crossover it installed like I was using Windows. Crossover says it supports Flash MX so that should work too. I don't know if Crossover supports any other versions than the MX series but that can be googled from Xover's website. 

Otherwise you can do anything but Flash with native Linux apps, like Nvu for web design, GIMP for image editing and InkScape for vector drawing ( really great stuff, check these out ). There is a Flash 4 like application called F4L ( short from Flash for Linux or something ) but it's still very early in development. The GUI is similar to Flash 4 but I found it to be much more complicated when comparing to Macromedia's work.

I think that the most useful solution for you is to make a dualbooter so you can switch to windows whenever you need to do something with Macromedia. Although some Macromedia software rides with Tux it still is designed for Bill. Before installing test Ubuntu with live-CD.
