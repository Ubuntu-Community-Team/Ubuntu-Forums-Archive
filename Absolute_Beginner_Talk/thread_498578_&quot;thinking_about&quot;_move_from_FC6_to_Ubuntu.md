### Post by sugarland2k on 2007-07-11
Try out Ubuntu/Kubuntu.  I used to run Red Hat in the old day when you had to take your PC apart to identify your hardware prior to installation ;)  Move to SUSE 9, the OpenSUSE, the Ubuntu and these days I run Kubuntu 7.04.

You can load the LiveCD versions of Ubuntu and Kubutu but they have all the standard Linux packages and good sofware updates (much easier than SUSE and Yast, etc.)  

The latest "normal" released version of Ubuntu/Kubuntu is 7.04. The fastest way for most people to get Ubuntu is by downloading the CD Installer. The CD Installer is nearly 700MB at [http://www.ubuntu.com/getubuntu/download](http://www.ubuntu.com/getubuntu/download)

If you don't have a fast internet connection you may want to consider requesting a CD for cheap (a few dollars), see [http://www.ubuntu.com/getubuntu/purchase](http://www.ubuntu.com/getubuntu/purchase)

You can also have shipit send you a FREE CD - although they may take 6-8 weeks or more! See

Ubuntu
[https://shipit.ubuntu.com](https://shipit.ubuntu.com)

Kubuntu
[https://shipit.kubuntu.org/](https://shipit.kubuntu.org/)

Try the LiveCD for testing. It is a bit slow on older machines but you can get a feel for Ubuntu/Kubuntu this way. If you like things, hit the load button! Ubuntu/Kubuntu can live on about a 10-20GB partition on a hard drive even with MS Windows (dual boot). 

I've run Ubuntu/Kubuntu since 2005 and I love it!

### Post by UnixAnt on 2007-07-12
I started out on Red Hat way back in 1996, tried Mandrake (as it was then) for a year, then stuck with Slackware for a long time.  Dabbled with Debian Potato when that first came out, and really liked the package system, but at the time I was also running FreeBSD which had an awesome ports tree.  Moved to Ubuntu in 2005 and have really never looked back.

The one thing I like about Ubuntu/Mint/Debian, etc more than any other distro is the package management.  I would use an apt based system as opposed to an rpm  based system each and every time.  As long as your repositories are up to speed, then installing/removing applications is an absolute no-brainer.  If I want something - I install it.  No fuss, no resolving dependencies and no being bogged down with rpm recursion.

The software management alone should be enough for people to at least try Ubuntu et all.
