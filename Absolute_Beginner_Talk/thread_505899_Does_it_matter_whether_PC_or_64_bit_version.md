### Post by coolen on 2007-07-21
I have an AMD64 processor in my desktop machine, and it's running the 32 bit version.
At the moment, there's no really good reason to use the 64 bit. 32 will run just fine, and very few programs are written to take advantage of the newer architecture. They're mostly just 32 bit code recompiled in 64 bit.
The only reason for using 64 bit is in preparation. Hopefully, more applications will start being written with 64 bit chips in mind, and it'll become really worthwhile to use the 64 bit kernel. When that happens, if you've been using the 32 bit version previously, you'll have to do a fresh install.
Of course, it'd also help to have another 64 bit user to add incentive for developers...
