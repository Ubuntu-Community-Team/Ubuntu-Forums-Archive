### Post by feathers on 2005-11-12
If you are new to ubuntu, you shouldn't edit xorg.conf by hand. Instead, as root or with sudo, do: dpkg-reconfigure xserver-xorg. You will be asked a lot or questions until you come to the resolutions sections where you can choose the resolutions you monitor can cope with. After that, restart X with /etc/init.d/gdm restart (or, if you use kdm, then swap gdm for kdm). You should then have the highest possible solution on your screen. Good luck.

### Post by lik3n on 2005-11-12
Ok, try this. From your desktop open a terminal and type this in. 

```
sudo gedit /etc/X11/xorg.conf
```

Go through and add "1280x1024". Once you're done, click save. Then it should work.

If you really want to do the nano way, which, for me, if a little bit harder if you're just starting out. 

```

```

Once you have added the correct resolution. Press Ctrl+o to save it. Then Ctrl+x to exit.

### Post by guipipia on 2005-11-13
I had the same problem and managed to fix it by manually inputing the horizontal and vertical monitor refresh rates on the xorg.conf as follows:

Section "Monitor"
        Identifier      "Generic Monitor"
        Option          "DPMS"
        HorizSync       30-70
        VertRefresh     50-160

Be sure to check your monitor's technical specs and adjust the values accordingly!
Hope this helps!!
