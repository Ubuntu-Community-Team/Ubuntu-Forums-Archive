### Post by igknighted on 2007-06-30
The cheating analogy doesn't work for one reason... if you cheat on windows, bill will always take you back :D.  Basically what I am saying is there is no obligation to come try it.  You can install and run it and if it doesn't work, then no loss (except time maybe).  But you seem to have the right attitude.  Many come to linux like "I know all about computers, I am a windows power user" and are not humble enough about their abilities to figure out that it is different.  But you seem open to learning, I think you can pick it up well.  It is not hard, only different.

EDIT: there is no reason to know how to code to run linux.  It is as easy to use as windows... not perfect by any means, but not inferior.  The trick is that it is different.  Windows does some things really well that Ubuntu/Linux does not, and vice versa.  So come with an open mind and some patience to learn and you will enjoy it.

### Post by lazyart on 2007-06-30
Understand that compiling isn't usually necessary.  In some cases you do, but it's really nothing more than:```
configure
make
make install
```
But again, you won't have to do that unless you want something that is really new.  Debian/Ubuntu is great about having all of that stuff taken care of for you.  Search the repositories (a catalog of ubuntu software), install it and run it.

I'm 38, cut my teeth on TRS80, Apple ][, Commodore Pet & 64 and even the TI-99/4A.  Just remember the systems were different from each other, and Ubuntu is different from Windows.  But you can learn it. :)
