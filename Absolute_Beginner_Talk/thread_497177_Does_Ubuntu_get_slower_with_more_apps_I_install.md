### Post by skullmunky on 2007-07-10
the only thing that should slow it down is how many apps are running at once.  just installing apps only takes up disk space; they only take up processor time if they're running.  

on windows, a lot of installers also install little extra services, helpers, utilities, pre-loaded DLL's, etc. that are often handy for that app but after a while start to bog things down.  (in addition to hidden malware, spyware, and crudware).  

in linux there's less of this kind of thing, programs busily off wasting time in the background.  where you might notice a little slowness is in programs that do run all the time, or in the background; maybe hefty superkaramba themes, amarok, or network servers like apache, limewire, bittorrent, etc.... 

one of the golden rules of linux, and i guess ubuntu particularly, has always been "only install what you need". 

but i have to admit, synaptic is like a candy store for me ...

### Post by Ek0nomik on 2007-07-10
It won't slow down much, just as it won't with Windows to be honest.

As previously mentioned, if you have software starting up on every booting of your computer, it will slow down because your processor is doing more work.  However, if you are merely having it sit on your hard drive, it will slow down, but a very minimal amount, for a very short period of time (the time it takes to find the program and make it available to you in the menu basically).  Once that has finished, it is simply sitting on your hard drive until it is called upon.
