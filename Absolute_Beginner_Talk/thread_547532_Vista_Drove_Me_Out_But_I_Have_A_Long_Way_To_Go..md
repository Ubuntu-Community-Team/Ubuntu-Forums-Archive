### Post by Circus-Killer on 2007-09-10
okay, so if i understand correctly, your main issue is running your windows programs. well, for every program you usually have 2 choices.

The first and most common choice is to use alternative software that is native to linux. For example, if you used to use mirc in windows, you can use xchat instead. if you use IE in windows, you use firefox in linux instead. here is a list of some windows programs and their linux alternatives (the left list is the windows apps)

IE - Firefox
Outlook - Thunderbird
Winamp - Rhythmbox or Exaile
Photoshop - GIMP
Notepad - Scite or GEdit
MSN/Google Talk - GAIM
mIRC - Xchat

These are just examples, there are many more apps for linux which do relatively the same thing as their windows counterparts.

Your second option (usually done if there is no native alternative), is to try run your windows apps through [WINE]("http://www.winehq.org/"), however WINE can be tricky to get configured correctly and often requires a bit of work.

Ultimately, the best solution is to find native programs. Now, you said you were a web developer. Some tools that may be of some use to you are Scite (text-editor with code highlighting), netbeans IDE (great for developing in java). For my graphics creations the GIMP does a reasonable job, you just need to have patience to learn it.

If you install ubuntu, you will automatically have openoffice and a number of the applications mentioned above already installed. But give yourself time with linux, it takes a while to figure your way around. Over time you will figure out what programs to run when, and how.
