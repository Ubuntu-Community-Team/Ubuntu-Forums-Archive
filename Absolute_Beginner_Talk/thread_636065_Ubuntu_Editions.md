### Post by justin whitaker on 2007-12-09
> **RobotoWorks said:**
> Thats it? Then why did they take the time to make 5 different editions? Also how does Ubuntu differ from Fedora8?

It doesn't really. In fact, there is no real tangible difference between the major Linux distributions these days. 

At the hear of Ubuntu is this idea that everything should be as simple and as accessible as possible for the average human being. Contrast that with Fedora, which tends to be developer centric, and so bleeding edge it hurts. In the end, however, they tend to have the same core applications, the GCC chain, and latest kernel. It's just a question of whether you want debian/apt or RPM/Yum, and how much of a headache you want to give yourself. 

The 5 different editions are actually 6, if you include the server version, and 7 if you include Gebuntu (Ubuntu+E17), and there is the Christian, Muslim (#9), and Fluxbuntu (#10) editions as well. 

Each has a slightly different audience based around a desktop environment, or a package set, to make it easier for people who need those applications, Each use the core Ubuntu repositories. 

Personally, I use Kubuntu. I like the ease of use of Ubuntu, but I prefer KDE. Even then, I have to strip out alot of stuff (Why adept at all? Stirigi is not ready) but I haven't rolled my own. 

It is a code base that you can manipulate pretty easily, so its just as flexible as Fedora is you want to do a respin.
