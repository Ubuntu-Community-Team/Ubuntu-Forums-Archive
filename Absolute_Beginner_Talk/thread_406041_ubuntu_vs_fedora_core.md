### Post by igknighted on 2007-04-10
Ok, ignore the posters above.

Fedora is designed to be a "cutting edge" distro, not really intended for new users.  As such there are no tools like automatix to install non-free stuff (although there will be a neat codec too in Fedora 7).  Also, there are lots and lots of updates.  This can "break" certain things (I put that in quotes because while things might not work, its because they are waiting for you to configure them, not because something is broken per se).  Especially with the kernel, if you upgrade that many times you need to reconfigure any non-free drivers (like wifi and video) that you have.

IMHO, Fedora has many thing going for it.  The gnome looks gorgeous is Fedora.  From boot to the desktop you are immersed in the stunning DNA theme, much nicer than Ubuntu horrid "human" theme.  You get a much better package tool in Yum.  Yum might not be blazing fast, but it allows you a level of control that you can only dream about with apt/synaptic.  The RPM databases in Fedora are not as full, and you need to be careful adding extra repos (I only use Livna), but again, it assumes a higher degree of user knowledge.  Overall, I find the Fedora community slightly more helpful than the Ubuntu community.  The things with Fedora is that there are so many "gurus" on the forum that no matter your problem you will get an answer.  Ubuntu is great because new users get answers, but harder questions generally get drowned out by "how do I play mp3's" (not saying this is a bad thing... there needs to be a place for these questions, but the fact is it does drown out harder questions).

So the issue really is what do you want.  Fedora will make you work a little harder, but your system will be more up to date and more tuned to your liking.  If that is more work than you want to put in, Ubuntu is a great distro as well.  There really is no wrong choice between the two.

EDIT: sorry, one more irk... whoever said Apt handles dependencies better doesn't know what they are talking about.  Overall they are about even in this regard.  Fedora gets a bad rap due to people adding many 3rd party repo's and getting into trouble... if you follow directions and don't do that, you wont have issues.  This same thing causes the same problems for people who do it in Ubuntu (its just less common because Ubuntus repos are bigger).  There are certain things Yum does better, mainly handling various architectures.  I can have 686, 586, and 386 packages handled simultaneously, while apt would have no idea.
