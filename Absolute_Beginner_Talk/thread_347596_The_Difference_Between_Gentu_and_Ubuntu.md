### Post by IYY on 2007-01-27
The difference is actually fairly large, but in the simplest terms: when you install software on Ubuntu, you get the binary (precompiled, 101010101010111101) package. In Gentoo, you usually get the source code which is then compiled on your machine. The compiling process takes some time (a lot of time on older machines), but the program usually ends up working a bit faster.

If you are not a programmer, you should certainly be using Ubuntu, because you can't benefit from the compiling process.

If you are a programmer, it's your choice. For example, I am a programmer but I don't have the need to see the code for every single program I install, and I want things installed instantly, so I use Ubuntu.

### Post by BarfBag on 2007-01-27
There's a pretty huge gap.  Ubuntu and Gentoo are two different extremes.  Ubuntu aims to be clean, easy to use, and fast.  Gentoo basically drops you into a root terminal and makes you do everything yourself.  Also, most things are compiled from source - which can take forever.  Once you get it setup, it's suppost to be really fast.  Also, it's package manager (Emerge?) looks killer.  In my opinion, it isn't worth the trouble, though.  Since you're new to Linux, you should start with Ubuntu and work your way up.  If you want to build your system, but don't want to spent weeks configuring it - Arch Linux is the way to go.  I'm in the process of setting it up myself.
