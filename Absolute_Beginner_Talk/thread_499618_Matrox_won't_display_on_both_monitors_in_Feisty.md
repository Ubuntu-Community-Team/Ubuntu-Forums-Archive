### Post by jp.stacey on 2007-08-01
I've managed to get the following slightly weird sequence of events to get a dual head setup working, eventually. It's not ideal but it might give you some hints. 

Because I upgraded, I already had an xorg.conf that theoretically supported the setup, so it's been a case of fiddling with it, but in a *very* odd way:

[LIST=1]
[*]Boot up. X only appears on the left-hand display. No login box (it's on the right display). When I move my mouse way over to the left, it appears.
[*]Press ^-ALT-F1 and log in as root. Edit xorg.conf. Set both screen "Device" entries to 0.
[*]In the same console, restart gdm with /etc/init.d/gdm restart
[*]Both screens now show identical displays (with login boxes)
[*]Return to the root console and reset the second "Device" entry to 1
[*]Restart gdm again
[*]Hey presto! Dual head, with two independent screens.
[/LIST]

It feels like there's a module not being loaded by default, but having the identical screens somehow kicks it. I've not been able to figure out why, but if this rings a bell for anyone then let me know and I'll post my xorg.conf, Xorg.0.log etc. here.

(Just upgraded from Edgy to Feisty, Matrox G450, twin Neovo widescreen 1440x990 monitors)
