### Post by keith11 on 2007-02-27
Hi everyone,

I have Ubuntu Edgy installed and working. Yesterday, I had installed a tablet driver or rather configured it on Ubuntu. I made a few restarts after that and everything was working fine. I closed the lid last night and by morning the battery had run out so the system started from the main dual boot loader. Now it is unable to start the X server and dsiplays the following error:

Fatal server error: Caught signal 11. Server aborting

I did quite a bit of research and found solutions like reinstalling xorg or configuring XFree86 and more of the like, which frankly speaking are beyond my understanding as I am new to linux. I configured a Wizradpen tablet yesterday and added the following to rc.local :

XORG_CONF=/etc/X11/xorg.conf
if [ -e /dev/tablet-event ]; then
  sed -ie 's/^\(\s*\)\#\(\s*InputDevice\s\s*\"WizardPen\ Tablet\"\s\s*\"AlwaysCore\"\)\s*$/\1\2/' "$XORG_CONF"
  echo "Udev created /dev/tablet-event, which means that the tablet is present! - Tablet-driver enabled"
else
  sed -ie 's/\(^\s*InputDevice\s\s*\"WizardPen\ Tablet\"\s\s*\"AlwaysCore\"\)\s*$/\#&/' "$XORG_CONF"
  echo "Udev did NOT create /dev/tablet-event, which means that the tablet is NOT present! - Tablet-driver disabled"
fi

I added the above mentioned configuration text after reading this page: [https://help.ubuntu.com/community/TabletSetupWizardpenEdgy?highlight=%28wizardpen%29](https://help.ubuntu.com/community/TabletSetupWizardpenEdgy?highlight=%28wizardpen%29)

Even after adding that configuration text and restarting everything was working fine last night but the X server problem started just this morning. I have noticed one thing when the system is starting and that is that the "wizardpen tablet" line in the Serverlayout section gets commented automatically whenever I start the system and it displays the fatal server error then. I am able to start the X server as root though and I am typing from Ubuntu when I am logged in as root. I am pasting my xorg.conf as well as /var/log/Xorg.0.log:
