### Post by smithman89 on 2007-08-28
With Ubuntu 7.04, first install nvidia-glx if you dont have it already (should be on there though).  Type nvidia-settings in the terminal, go to the X Server Display Configuration tab.  Click detect displays, then use the Configuration dropdown menu to choose which way you want the displays to act.  Click quit, then restart X (Ctrl+Alt+Backspace).  Log in again, and check if the settings were saved  (sometimes they dont).  No xorg commands necessary.

### Post by eeeeepuk on 2007-08-29
Just wanted to check how you were getting on with this problem? Am I right in saying that you now have Compiz running, but it's only displaying window decorations (titlebar) on one of your monitors? If so, then according to [this thread]("http://forum.compiz-fusion.org/showthread.php?t=3472&highlight=dual+monitor") over on the compiz fusion message boards, this (and some other) problems are common when using it in an Nvidia dual monitor configuration.

All of the problems can be solved, or so it seems, by reconfiguring your x display as "twinview" in the nvidia-settings (making sure to run via sudo so that it can write to the xorg.conf file), rather than as a separate x display.

This isn't always practical, though (I found setting it up like that useless for trying to use one monitor as a tv via mythtv and the other for work), so the window borders can be fixed when using separate x screens by using the emerald window decorator, though even then you will have to start it manually (Compiz Fusion won't launch it on the second x display automatically).

Should you go down this second route, you'll first need to install emerald by pasting the following into a terminal

> sudo apt-get install emerald emerald-themes

followed by

> emerald --replace &

This is assuming that you've added a repository to install compiz fusion, as the default feisty packages for emerald are designed to work with, and will try to install, beryl which is now outdated and something you don't want installed alongside compiz fusion. Also, this second method won't impact any slow resizing/maximising windows should you experience those problems - as yet, there appears to be no known fix for those when using separate x screens.
