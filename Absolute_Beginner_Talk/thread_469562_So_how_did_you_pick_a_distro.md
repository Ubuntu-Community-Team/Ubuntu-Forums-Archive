### Post by Golyadkin on 2007-06-10
Sorry for the long post, I got carried away a little :$

Well, I started 11 years ago with RedHat 4.0 when I was just a kid in highschool. My neighbour installed it for me and wrote some terminal commands on a sheet op paper. I booted it maybe twice a week, just to play around.I  hardly used an X server  I still had to start it by hand and its coolest features were xclock :) and that says a lot for the people who know it. With most of the things I did, I had no idea what I was doing, but did I have fun !!! :D Sometimes I got so lost, all I knew to shut the computer down was hit the powerswitch :$

For a long while I then left it, because even though it was cool, none of the programs I liked could run on it, and I didn't understand yet that there were replacements coming, and some alternatives already existed. I was still of the opinion that "if it isn't exactly it, it can't be good" without really thinking about what it does and if different can't mean better. However, I have been known to secretly playing nethack until deep into the night (I am not sure if there are Ubuntu users now that know or play nethack, but in that time for me, it was amazing)

Then in the following years I always kept trying out distributions, because I wanted to see how they were, i there was more hardware support and newer programs and games. (Actually, a lot of the simpler free games in Linux were often more fun than 60 dollar games which you don't play anymore after 10 hours when you finished them)

I would always buy these computer magazines that came with installation CDs for Linux, because internet was too slow in those days. 
I have used Red Hat Linux 4.0, 4.1, 4.2, 5.1, 5.2, 6.2, 7.0, 7.1 and then used Fedora Core 1, 2, and 3. I also used SuSE Linux 5, 6 and 7 and later OpenSuSE  I have tried Mandrake Linux and FreeBSD (but never got that last one to work) and some other distributions. Then I stopped using Linux for a year or two because I was too busy in school. 

I taught myself C programming and LaTeX typesetting using Linux, which was a great experience, feeling like a hacker at night, with just a blinking commandsprompt on your screen :D ahhh, the memories.

I installed Vista and was so disappointed I uninstalled it again after 4 hours and went back to XP. Luckily I got the license free from university, so nothing was lost. So when I heard all the "fuss" about Ubuntu, I wanted to see what it was. I installed it in  vmware under XP and went through the installation (which was a breeze, but then again, it wasn't hard with RedHat 4 either in 1996, if your hardware is supported)
I liked it and burned a live-cd to see if it performs well, and then simple installed it rightaway after using it for an hour. I was so amazed that Ubuntu could import user accounts from XP, my jaw dropped. I was used to simply forgetting about everything that was on NTFS. 

But Linux has come so far... I remember when I was amazed at RPMs! We used to have to compile every single program we wanted. (For the non-RedHat adapts: RPMs are like .deb's) And the handy automatic updates, synaptic.... I loved it :) And now for the last 4 weeks I have written my master thesis in Ubuntu entirely (in LaTeX) with only one exception, using Visio in VMware XP because there is no tool yet that can do the same for me in Linux. (Not Kivio or DIA)

So how do I choose one? Download, burn, install and play around. It is sooo easy now with Live-CDs/DVDs. The threshold to trying was a million times higher when you actually had to install and risk losing all data if you did it wrong.

### Post by vinodkvr on 2007-06-10
I agree with Golyadkin. The introduction of live CDs had made it very convenient to get the feel of the distro before actually installing it on the computer. I had only a relatively short period of experience with the Linux OS. Based on  my experience two factors are to be taken in to consideration when deciding what distro to choose .

1. The level of configuration needed to use the system for our day to day requirements at Office, Home  etc.
2. Ability to play multimedia with minimum tweaking.

On both these points i feel Ubuntu is the best.

Most of the users are using both windows and Linux on the same PC and they need to access the files in the windows partition. Ubuntu mounts the windows partition out of the box  unlike fedora where we have edit the fstab file for this.

As far as multimedia playback is concerned  Ubuntu Feisty Fawn searches the required codecs and install for you when you double click on the file to be played which i think even windows cannot boast off.

Fedora is a great distro of course.  But in my opinion, less complex distros like Ubuntu are the best
