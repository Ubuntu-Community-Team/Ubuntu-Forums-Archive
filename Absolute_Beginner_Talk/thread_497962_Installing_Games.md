### Post by Malibu Illusion on 2007-07-10
I've got no idea who you've been talking about in regards to games, but you're disillusioned right now.

Game support on the Linux platform for mainstream, commercial games is basically zero.  There are very few commercial games out there that run natively on Linux 

People execute games that do not run natively under Linux through a compatibility layer more specifically, in the majority of cases, through [WINE](http://www.winehq.org).  WINE have an [application database](http://appdb.winehq.org) that can be referenced to see how users of specific Windows-based applications got on with installing and using the application in question.  If you try and query that database with the names of some of the games that you play, you'll get some results which you can check.  These will give you a more clear indication of what runs smoothly under WINE, and what doesn't.  You can always expect to see issues with this; some minor and some major making the application unusable in some cases.  It is not a perfect solution by a long shot.

You may also wish to check out the commercial fork of WINE, called [Cedega](http://www.transgaming.com).  This project was created with the intention to specifically allow Windows-based games to run under the Linux platform and, as such, focuses their development on implementing the DirectX API.

### Post by dorath on 2007-07-11
The best way to be sure is to give it a shot!

If you don't have WINE installed yet there are two ways to get it:
From the command line:
```
sudo apt-get install wine
```

Via the menu:
Applications --> Add/Remove... --> search for wine, check WINE Windows Emulator, click OK.

Once you've installed WINE, open your File Browser (Places --> Home Folder), locate your download of the Tribes demo, right-click the demo, select 'Open with "WINE Windows Emulator"'

That's about as far as I can help you.  Hopefully it'll install, or you'll be able to work around the errors that you're familiar with.  Best of luck!
