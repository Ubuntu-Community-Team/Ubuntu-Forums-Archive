### Post by 1001001 on 2007-03-03
Does your university use a "standard" type of login procedure?
(username followed by a password with no other prompts)

If so, let wvdial detect the prompts and send the required information. Definitely disable the "stupid mode=on" option.

Here is my wvdial.conf
[Dialer Defaults]
Init1 = ATZ
Init2 = ATQ0 V1 E1 S0=0 &C1 &D2 +FCLASS=0
Modem Type = Analog Modem
Baud = 115200
New PPPD = yes
Modem = /dev/ttyS0
ISDN = 0
Phone = _MY_ISP_PHONE#_
Password = _MY_PASSWORD_
Username = _MY_USERNAME_


Based on your wvdial output file the problem is here:
ATDT12078343689
CONNECT 50666 NoEC
--> Carrier detected. Starting PPP immediately.  <--------- PPPD is starting with no logon
--> Starting pppd at Fri Mar 2 15:28:41 2007


I think you should be seeing something like this:
CONNECT 50000/ARQ/V34/LAPM/V42BIS
--> Carrier detected.  Waiting for prompt.
stuff.net
Login: 
--> Looks like a login prompt.
--> Sending: (YOUR USERNAME)
(YOUR USERNAME)
Password: 
--> Looks like a password prompt.
--> Sending: (PASSWORD)
    Entering PPP Session.
    IP address is (ADDRESS)
    MTU is 1524.
--> Looks like a welcome message.
--> Starting pppd at Fri Mar 2....


No where in your wvdial output file is there any indication that a logon procedure is being initiated. Your PPPD session will eventually time out and you will be disconnected.

Did you run "wvdialconf" in Ubuntu or did you just copy the Fedora version of your wvdial.conf file? I think it is best to run wvdialconf in a gnome-terminal to create /etc/wvdial.conf and then modify it by inserting your phone#/username/password.

After I create and modify /etc/wvdial.conf I then run wvdial by using the command "sudo wvdial".  Is this the procedure you use as well?
