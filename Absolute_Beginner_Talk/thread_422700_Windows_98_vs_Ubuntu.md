### Post by jem7v on 2007-04-25
It'll depend on your hardware, really, and more specifically on how much RAM you have.

On my computer I have a 750mhz CPU with 384mb RAM and Ubuntu runs much better than Win98 does.  On the computer downstairs, it's 450mhz with 64mb RAM, and a lot of applications in Ubuntu absolutely chug...  (They did in Windows too, especially after spyware and viruses hit it, but even a clean, safe install of Xubuntu was pretty nasty slow.  It's better with a lightweight window manager like FVWM or Fluxbox, but it's still slow.)

If you can, make sure you have at least 256mb RAM if you're going to run Ubuntu, or at least 128mb if you want to use Xubuntu.  If you have less than 128mb RAM I think that almost anything is going to be slow unless you're happy with a really minimalist distribution like DSL linux.

### Post by Scheater5 on 2007-04-25
If it came with Win98, then it most likely has hardware that is capable of supporting Win98.  Win98 should perform just as well as Win98 is supposed to.  But it wasn't the most stable of operating systems, even by Microsoft standards.  Even running exactly as it is supposed to, especially if you connect to the internet, it will not be very "smooth."  However, Linux will also quite likely be able to run perfectly on that same box.  And Linux running "as it is supposed to" is very likely to be "more smooth" than Win98.  I recommend you try Xubuntu, or even DamnSmallLinux.  Both of these come on LiveCDs you can try before you install.  But please be aware that LiveCDs will run slower than if it were installed on a harddrive.  
Lightweight desktop environments will go a long way towards squeezing the most out of older desktops.  Xubuntu uses Xfce and DamnSmallLinux uses fluxbox.  You might also want to try IceWM.  
This may be more information than you wanted.  The short answer is, "almost certainly, Linux will run smoother than Win98."  The long answer is that squeezing the most out of linux on a box that old may require some experimenting.
