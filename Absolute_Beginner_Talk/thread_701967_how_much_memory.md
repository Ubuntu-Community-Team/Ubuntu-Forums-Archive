### Post by ryanhaigh on 2008-02-19
More RAM = larger cache = faster file access = good

Coming from windows most people think that having more free ram is better (as I did) but what is the point if you never use it. The way Linux uses ram to cache recently accessed files means fewer reads from your slow hard disk as files are still in memory, obviously this increases file access speed which can only be a good thing.

Everyone who says 'my system only uses X' should have a look at the output of the command free -m after running their system for a while. I have 2GB and at the moment I have 51MB free and ~1400MB is being used for cache.
