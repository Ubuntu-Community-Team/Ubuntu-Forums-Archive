### Post by aysiu on 2007-08-09
Then forget about the files. The script will download Firefox for you.

And it is not a keylogger. If I were trying to trick you into installing a keylogger on your computer, do you think I'd put the word *keylogger* in the web address for the download? Go ahead and open the script (it's a text file) and take a look at it. You can see exactly what it does.

The script is maintained by a forum member here who also has created an open source Windows program called *PyKeylogger*: > PyKeylogger is an open source keylogger written in the python programming language, available under the terms of the GNU General Public License. I threw it together one day after not being able to find a simple and trustworthy keylogger for windows, and it sort of snowballed from there. It is currently only available for the Windows OS, due to its using Windows-specific APIs.

It is primarily designed for personal backup purposes, rather than stealth keylogging. Thus, it does not make explicit attempts to hide its presence from the operating system or the user. That said, the only way it is visible is that the process name shows up in the task list, so it is not immediately apparent that there is a keylogger on the system. However, since it also makes periodic writes to disk, and since it openly hooks well-known windows APIs (SetWindowsHookEx), any keylogger detector worth its salt will be able to sniff it out. So basically, it doesn't exactly advertise itself, but doesn't hide itself either.  When he created the Firefox installation script, he decided to host it on his PyKeylogger website. The two have nothing to do with one another, and--as you can see from the description--the Windows keylogger program is not intended to be a stealth keylogger anyway.

The script even has [its own subforum on the Ubuntu Forums](http://ubuntuforums.org/forumdisplay.php?f=251).
