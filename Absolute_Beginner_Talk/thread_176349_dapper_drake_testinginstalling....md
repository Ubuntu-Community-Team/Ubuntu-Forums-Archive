### Post by uzi09 on 2006-05-14
do you mind explaining what you mean by "keep things the way they are"??

if you mean, you'd still like things to work properly, then you would need to get another machine and load dapper on there. for the most part dapper works quite well, if you run into some sort of a bug/glitch, let the dev's know about it through the bug submitting application (more on it when you have dapper already installed).

if you can't get another machine, then you may have to use dapper on a regular basis (as in put it on the machine you're currently using). this means that for the most part everything will "remain the same" and you should be getting along without a problem, but you may eventually run into a problem (which is how you help "test" dapper :D) so just let us know, wait it out, or if you're good enuf figure out the problem yourself and fix it and let the devs know so that they can correct the issue.

so short answer: no...when you're testing something, you're trying to find mistakes...so you'd try to do anything possible to make your dapper crash and then fix it or let the developers know about the problem so they can figure out why and then fix it.
