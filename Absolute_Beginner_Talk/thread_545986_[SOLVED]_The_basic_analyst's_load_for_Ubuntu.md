### Post by Cyn Jade on 2007-09-08
I work as an analyst with an organization whose ability to order and install hardware and software is extremely poor and very Microsoft-centric.  I originally looked at Ubuntu as a way of getting up and running no matter how dramatically dysfunctional our computer base load (or IT support element)is .  Obviously this works well (within the limits of connectivity), and I want to share a recommendation to anyone in a similar situation:

There are two programs that fill out the functionality of the Ubuntu & OpenOffice load very nicely:  GoogleEarth ([http://earth.google.com/download-earth.html](http://earth.google.com/download-earth.html)) and CmapTools ([http://cmap.ihmc.us/download/](http://cmap.ihmc.us/download/)).  Once downloaded (to the desktop by default in a new Ubuntu load) they can be installed by going to Applications, Accessories, and Terminal, then typing in cd Desktop (hit return), then typing in "sh" followed by a space and then the name of the .bin file downloaded (hit return).  The current versions of Ubuntu, GoogleEarth, and Cmap Tools all work fine and don't require any troubleshooting as was necessary with past versions.

Combined with the Ubuntu base load (including OpenOffice) this gives an analyst relatively easy access to the basic analytic capabilities:

Geospatial / mapping analysis (using GoogleEarth functions and "saving as" .kml files, which can be edited in OpenOffice easily to, say, splice in huge lists of geocords, for example).

Link diagram / association mapping (using Cmap, with the advantage that it is very easy to import / export lists of relationships in the triple, or A Link B, type format.)

Timelines / chronological analysis (using OpenOffice Calc to do Gantt charts ([http://www.openofficetips.com/category/charting/](http://www.openofficetips.com/category/charting/)))

Documents (.doc and .pdf), presentations (.ppt), databasing, email, browsing, etc...

So far, I've yet to get anything like the blue screen of death and the learning curve for the differences between Windows/MS Office and Ubuntu/OpenOffice is more mild than the learning curve from MS Office 2003 to MS Office 2007.  Plus, it's free for individual use (industrial use does require some fees for GE and Cmap, though not so much as to be a show-stopper if you are eating it out of your pocket (should you elect to keep to the moral high-ground)).
