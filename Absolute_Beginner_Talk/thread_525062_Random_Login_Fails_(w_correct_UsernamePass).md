### Post by jw5801 on 2007-08-16
Hmm... in that case I'm not really sure then. Something might have gone bad during the install process so at this point I'd probably be trying a complete reinstall. It sounds alot like a kernel panic, or something else like hat. Did it happen right from the outset when you installed? Because that would support the 'bad install' theory. Either that or something that's attempting to load when you start X is crashing the system. The second there should be some way to debug. Try looking at the system log after it happens: ```
cat /var/log/syslog
```
I think that syslog file only records since your most recent boot however. Have a look around the /var/log folder and there'll be a few compressed logs in there, so one should contain the second most recent log. Or you could even try to find the log from the last time it happened right now if you wanted (every line is dated and timestamped which makes it easy). Just look for the last thing that happened before the system restart and it might show some clue. For background information, the command 'cat' simply prints the contents of a text file to terminal. If the text file is compressed (.tar etc.) then 'zcat' will perform the same operation.

So, if you want to try and debug, hunt through the syslog. The easiest (and probably most effective) option is to reinstall (run a check on the install disc first, it has a built-in funtion to do this). Or alternatively you could try booting using a different kernel for a while and see if the problem persists (you should have a couple in your grub menu). 

Since XP is running fine (I'm assuming you would have said otherwise) I think it's safe to assume that your hardware is ok, although it might be an idea to run a check on your Ubuntu partition just to make sure it's not a hard-disc issue.
