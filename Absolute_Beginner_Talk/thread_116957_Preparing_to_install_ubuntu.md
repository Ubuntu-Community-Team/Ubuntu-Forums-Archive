### Post by TeeAhr1 on 2006-01-13
Hi, welcome!  I'm only going to answer the questions I feel confident about; I'd rather give you no advice about something than bad advice.  Hopefully someone else can help you out with your other questions.

*1. Can you tell which firewall should I download?*

The 'stock' Gnome firewall app is called Firestarter, you can find it in the repositories.

*3. I often copy text, as evidence and future reference, and paste it into a word processor. Will I be able to convert them into Windows format?*

Text is text.  If you save something in Gedit, it'll open in Notepad.  There is no "windows" format.  Also, an OpenOffice file will open regardless of the system you're using.  I email Write files back and forth from my XP machine at work to my Ubuntu machine at home all the time.

*I'm pretty new with computers (they didn't "exist" in my time and world) and can just about use windows for a bit of word processing and internet surfing: do you think I can cope with ubuntu, do I need to know how to use command line etc?*

You will learn bits and pieces of the command line as you go along, but it's not very intensive.  I'm computer literate, but until October, I had never tried Linux before, and I'm coping just fine.  Some days are sticky, but I have yet to find an insurmountable problem.

The terminal will scare you for about the first three days, but really, it's not as terrifying as you think.  If you are intelligent enough to ask these questions, you're smart enough to use Ubuntu.  Give yourself credit, and give yourself a chance.

The single best piece of advice I was given was this: answer all your pre-install questions before you put the disc in; don't do anything until you're satisfied with the answers you've gotten.  And do it when you've got ample time (like over the weekend).  These two little things will reduce your stress level *so* much!

If you've got any more questions (now or later), these forums are the best resource you'll find.  The people are, to the last, super-helpful and super-friendly!  No one will ever tell you to RTFM here, the community is TFM, and we're all here to help each other!  Good luck, and welcome to Linux!   -pete

### Post by Bartender on 2006-02-26
claptrap -
The screensaver crashing could be related to graphics card...I had an ATI 9200SE card at first, and it did exactly as you describe.  You couldn't even visit the screensaver tab without crashing the computer.  I bought a comparable (around $40) nvidia card, did the uninstall/reinstall thing, and it works fine.  When I say uninstall/reinstall thing, I mean that I used the Windows partition and went thru the usual steps.  Of course, the install CD would be pointless on the Ubuntu side of the HDD, so if you only had Linux I'm not sure what you would do.  Just turn it off, physically swap cards, and turn it back on? 
We still haven't heard from you regarding how much RAM you have.
Hmmm, yours is the first post I've seen regarding a scrounged up computer that was found out in the rain without a cover.  Was this anywhere near salt water?  All it would take is a thin film of corrosion on the CMOS battery clip or memory slots or whatever and you'd be fighting mystery crashes alla the time.  Wait a minute - are you talking about scrounging parts from the water-logged machine and using them in yours, or trying to get the wet machine functioning?  I would disassemble the wet one.  Get the memory sticks out of their slots and carefully wipe the contact points down with some good-quality alcohol (not that 50% water crap), then when they're completely dry rub the contacts with a standard pencil eraser.  Then get a copy of memtest on a floppy (go online, d/l the small program, follow the directions) and use your PC or your friend's.  Go into BIOS to tell the PC to boot from A drive. Turn the PC off.  Pull all the existing memory out, (keeping track of where it went and in which order) plug one or more of your salvaged memory sticks in, put the memtest floppy in, hit the Go button, see if memtest will run the memory stick(s) without producing errors.  Ubuntu has memtest built right into it, although I'm not sure how to get there.  If you can scrounge working memory out of the wet one they still may not work in yours.  Unfortunately there were many different varieties of SDRAM.  The little keyways in the sticks will keep you from shoving 3.3V memory into a 5V motherboard, same thing for buffered vs. unbuffered.  Less obvious differences exist on the chips themselves so they may not run in yours and you may never know why.  Try to get the numbers off the chips and see if you can figure out whether it's generic RAM or RAM that was built specifically for an IBM or whatever.

BTW, I don't think your 1800 Webshots will work in Linux...

### Post by -deadcats on 2006-03-05
If at all possible, go with the higher-end machine, a reasonably large HDD (20-30 GB), and as much RAM as you can get. 

Ubuntu, like most current Linux distros, is a MODERN operating system, designed to operate on MODERN hardware. Your hardware is ANCIENT! Sorry, but that's what it is. However, all is not necessarily lost. You could go with an older version of some of the Linux distros--or you could go command-line only--or you could install one of the lighter-weight distros like DSL. But you're not going to be able to really enjoy what makes Linux so cool.

In general, with your old-old-old stuff, you're going to be very limited on the eye-candy. Windows 98's might be preferable if you do in fact want to run a full-fledged GUI.

I don't mean to be discouraging--I'd like Linux to be an enjoyable learning experience for anyone willing to put the effort into it, as you so readily seem to be.  And I know money and available supplies can oftentimes be an issue.
