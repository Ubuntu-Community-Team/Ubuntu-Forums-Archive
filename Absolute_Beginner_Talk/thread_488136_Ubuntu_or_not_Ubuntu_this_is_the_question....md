### Post by starcraft.man on 2007-06-29
The answer IMO is to get an ethernet based router. I've never seen any reason for companies to issue those USB routers except to force you to use their software to run them, whereas ethernet is more or less universal.

Linux in general requires DSL/broadband internet access (as does windows, ever tried to download sp2 with dial up?), all linux distributions (to my knowledge) install from internet based repositores as a primary source, they also issue updates and security fixes via this and honestly how useful is an OS that doesn't let you browse the web? There are a few distros that offer a DVD with all the basic packages but they won't get you udpates (unless you contiually download packages to CD or other media and install separate).

I agree Ubuntu and Linux is boring without a net connection. So is Windows and OSX I might add, the internet is an essential service to the OS now and if you can't get a hold of a modem with ethernet or get that one working with Linux (or don't want to) I'd advise you to stick with what you have. Thats my 2 cents.

### Post by Rocket2DMn on 2007-06-30
@ ZDUNAS - you CAN share internet from your windows computer to your Ubuntu box.  If your windows computer has an ethernet connection as well, you can enable that and setup a bridge between the USB connection and the ethernet port.
After that you can plug the ethernet cards together between your windows computer and your Ubuntu box using a CROSSOVER cable - a standard patch cable will not work.
Here is a quick tutorial from microsoft, but there are plenty of other guides out there - [http://www.microsoft.com/resources/documentation/windows/xp/all/proddocs/en-us/hnw_bridge_install.mspx?mfr=true](http://www.microsoft.com/resources/documentation/windows/xp/all/proddocs/en-us/hnw_bridge_install.mspx?mfr=true)

You can also tie a wireless card in but that could be a bit more of a hassle.  If you want you can set it up so it logically looks like this:
internet->cable modem->windows pc->router->other computers (like ubuntu)
What i suggested above just took the router out of the equation, but it's your call.  With a router in there you MIGHT not need a crossover cable, a standard patch cable might work going from the windows computer to the WAN port of the router.
