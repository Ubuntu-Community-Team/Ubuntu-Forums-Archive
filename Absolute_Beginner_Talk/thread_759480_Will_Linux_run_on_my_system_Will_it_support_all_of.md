Follow RedLXXXIV's advice, the last post on the thread at the time of this writing. Ndiswrapper is a program that lets you install wireless drivers in linux written for windows. That should get you up to speed as far as wireless goes. Just go non-wireless for a few minutes, do what he says, then try going wireless again. It will be a lot easier to do.

I think it might have ndiswrapper on there, I donno.. does hardy come with ndiswrapper? I donno either..). Basically the alternative way is downloading the two debs (ndiswrapper-common and ndiswrapper-utils) from here [http://packages.ubuntu.com/search?suite=hardy&arch=any&searchon=names&keywords=ndiswrapper](http://packages.ubuntu.com/search?suite=hardy&arch=any&searchon=names&keywords=ndiswrapper)  and the other one (ndisgtk) from here [http://packages.ubuntu.com/search?suite=hardy&arch=any&searchon=names&keywords=ndisgtk](http://packages.ubuntu.com/search?suite=hardy&arch=any&searchon=names&keywords=ndisgtk) on your current machine then copying them over in any way you wish (flash drive, cd, copy it over if you have windows set up to view your ubuntu drive) to your ubuntu machine's -desktop- and installing them via (according to the ubuntu wiki):

sudo dpkg -i --force-depends ndisgtk_*.deb

### Post by FrozenFox on 2008-04-29
[http://ubuntuforums.org/showthread.php?t=593048](http://ubuntuforums.org/showthread.php?t=593048)

See loyll's post (2nd post at the time of this writing). Youve already installed ndiswrapper, so skip to the rapidshare download part.

To get compiz working, simply use the restricted driver manager in ubuntu to install your video drivers. When you've done so, you can test by typing glxinfo | grep direct in the terminal. If it says direct rendering is enabled, you're probably good to go, because your video drivers are apparently working. Typing glxgears should bring up a gear test that should print fps to the terminal too, which requires working 3d stuff. 

Once that's working, open synaptic and install um.. I think the package is compiz-core? Note that compiz might already come with hardy, I don't remember. I think it does, actually. I don't know, I don't use ubuntu to check, but you can easily search compiz in synaptic and find all the necessary stuff and install what you don't have that you need. To make your life easier, install ccsm (I think the package is compizconfig-settings-manager or something) and compiz-fusion-icon (may just be fusion-icon). You probably also need to install emerald. Finally, run fusion-icon, then right click it in the system tray, and select window manager "compiz".

Ultra common stuff you might come across during 1) setting up your video drivers 2) setting up compiz:

1) Video:
* May run into a bug where ubuntu won't boot into graphical mode or boots into a lower resolution (and has no 3d support). This means the driver didn't install correctly or your video card is not supported, 95% of the time being the former. If this happens, let us know and we'll give you alternate means of getting that working. The "official" way is the best way, if it works.

2) Compiz:
* May run into the bug where window borders / title bar not being drawn if your xorg.conf is not set up correctly. Tell us if you run into this too, it's an easy fix. Usually a few clicks and adding a few entries to xorg.conf.

### Post by Phonan on 2008-05-15
Assuming, as stated in an by fluffman, you downloaded and installed compizconfig-settings-manager, you now need to start it. You can do this a few ways- the easiest I know of is to go into "Appearance" from the preferences menu, then select "Desktop Effects." Finally, there should be an option to access settings- "customized" or the like. There, you can see many settings for compiz, far more than you originally get. What gets confusing is that compiz desktops and Ubuntu's desktops (as shown in your panel) are not the same. I found it useful to remove the Ubuntu desktop switcher alltogether from the panel. You can still switch compiz desktops with the cube, ctrl-option-arrow, etc. Now, in compizconfig settings, go to general. There you should find an option to increase your number of desktops. Pick a number, (above 2 to get rid of the "paper effect") and there you go!

Completely changing what I just posted. Compiz does support this; the problem is that Nautilus draws the desktop even with Compiz running. This means that even if you change wallpapers for each face in Compiz, what you see won't change unless you stop Nautilus from doing the backgrounds. What I saw on the Compiz forums:> Yes. You can do this through the Desktop Cube plugin (under the Appearance tab, by specifying the filepaths for your wallpapers under the Background images section).

(There is also the unsupported Wallpaper plugin, which you would only really need if you were using Desktop Wall instead of Desktop Cube.)

Please note that in both cases, you need to disable the drawing the desktop by Nautilus (this is assuming you're using GNOME). This means you won't have desktop icons. This is a technical limitation and the fix would be in Nautilus, not in Compiz.

If you don't disable desktop drawing, then the wallpapers set by Compiz will be obscured completely by Nautilus.

You can do this fairly easily using this thread:
[http://ubuntuforums.org/showthread.php?t=577144](http://ubuntuforums.org/showthread.php?t=577144)
But you won't get icons on your desktop, which is definitely something I wouldn't like to have to put up with. If this is okay with you, though, go for it. Use the Compiz section.

I looked some more, and this can be done *with icons*, but it'll take work. Here's one thread I found on the Compiz forums, and if you look some more there are likely more with more information; bear in mind this may be harder and riskier:
[http://forum.compiz-fusion.org/showthread.php?t=6199](http://forum.compiz-fusion.org/showthread.php?t=6199)


Good luck, and it should be doable, but unfortunately it will take work. You could wait; we should get an update to Nautilus that will help fix this, but I'm not sure when it will come. Until then, I hope it works out.
