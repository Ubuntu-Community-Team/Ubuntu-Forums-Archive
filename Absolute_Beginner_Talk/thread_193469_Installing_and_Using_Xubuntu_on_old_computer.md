### Post by brentoboy on 2006-06-16
depending on how old it its, you have a few options...

I have an old amd k6, 433 mhz with 48 mb of ram, and all I can do with it to make it useful is install debian sarge on it, X, but no desktop, and use xdmcp to run applications as a thin client to my "real" computer running dapper.

If you dont have a better pc to leach processor time and memory from across a local network, then you might consider damn small linux.  it runs great on even the oldest stuff I have.

or, you can play around with the "damn small" philosophy -- but still use ubuntu.  install just the "server" installation, and then add a lightweight windows manager (damn small linux uses fluxbox, so it might be a good choice for you) and then add other apps as you need them.

going that last route will teach you the most about linux, but it will frustrate you becuase you wont have anything to start with, you will have to do a lot of work by hand, and you will have to do a lot of web surfing to get things tweaked to a decent usable environment that does all the things you really want it to do.

either way, I recommend getting damn small linux just so you can see which apps it uses, and decide if you like them before you try piecing together an ubuntu box with similar apps and settings.
