### Post by RChickenMan on 2006-12-19
Although mine is still in the mail, thus have no personal experience with it, I would have to say that the iAudio X5 is the winner for any Linux user.  To the best of my knowledge, it is the only HDD-based player that uses the straight-up USB-UMS interface (plug it in, mounts as an external drive, drag-n-drop your files), AND supports an open-source lossless audio codec.  It is the least "locked down" player I could find (very important to Linux users, I would assume).  The iRiver H320 is drag-n-drop, but does not support any lossless audio codec (an absolute must for any audiophile).  Although, speaking of audiophile standards, the X5 does not support true gapless playback.  I believe for most codecs the gap is somewhere around .005 seconds (compared to about half a second for the Apple iPod).

### Post by julian67 on 2006-12-20
It would be a good idea to look at [Rockbox]("http://www.rockbox.org/")

> Rockbox is an open source replacement firmware for mp3 players. It runs on a number of different models:

    * Archos: Jukebox 5000, 6000, Studio, Recorder, FM Recorder, Recorder V2 and Ondio
    * iriver: H100, H300 and H10 series
    * Apple: iPod 4th gen (grayscale and color), 5th/5.5th gen (Video - 30GB and 60GB models only), 1st gen Nano and Mini 1st/2nd gen (Nano 2nd gen and 80GB Video 5.5th gen are not supported)
    * iAudio: X5 (including X5V and X5L)
    * Additional models are in development 

It adds support for various codecs including flac and ogg, gapless playback, tag database, jpeg support, text file support, voice driven interface, unicode support, themes, multilingual support, and all kinds of applications and games via plug-in. 

I have an iRiver H340 with Rockbox and it's excellent. iRiver players are usually fully USB and/or UMS compliant so they work fine with any OS that supports USB even with standard iRiver firmware. My girlfriend has an ipod Nano and it works fine with Linux. It's completely standard firmware, no modifications. We're using openSuse with Gnome but it includes quite a lot of KDE stuff too. Konqueror with kio-ipod handles it brilliantly, we can drag and drop music files both ways. I'm pretty sure that  ipod-sharp does the same thing in Nautilus.

### Post by suziequzie on 2007-01-02
I have an iRiver H10 5GB (red and silver). I *was* going to use rockbox on it, but it chewed battery time down to 40% of what I get with the original firmware. I use easyH10 (in the repos, installed via synaptic) to re-write the database in a format the original firmware uses. 

Rather than manually type in "easyh10 -Cn -Pfr /media/H10" in konsole every time I need to rebuild the database, I made a script file, then used the "Non KDE application launcher" to make an icon on my panel to run it for me.  I made an icon of my red & silver model from pics off the web, and boom, easy update for my player.

It was my first script file. Thanks to the folks here for showing me how to write a script file from the various examples I've seen.

### Post by kragen on 2007-01-17
I have a Zen Vision:M, its an excellent player - long battery life, nice UI, very nice screen etc...

As far as compatibility with linux is concerned, it uses MTP (Microsoft Transfer Protocol :/ How vain are those guys?! :P), support isn't as good as it could be, but it seems that several players are working to include support - Banshee has MTP support (which I couldn't get to work) in development, Amarok apparently already has it working (I'm installing it now to give it a try), so it seems that in the not too distant future a fair collection of players should "just work" with it. 

Until that point, gnomad2 works very nicely - it might not have the ease of use / polish of iTunes, but it does do nearly everything perfectly well - the only things I've seen which it cant do is album art, and other small features like that (it does do playlists btw :)). 

---

In Conclusion: The Zen Vision:M uses a different protocol to many of the other Zen players, just because you've read that someone got a Zen "Whatever" to work, doesnt mean that the Vision:M will work in the same way.
Having said that... support is available to just about everything you might want to do with it with the possible exception of album art.
Most importantly, development is being done to include MTP support, so although not many players support MTP at the moment, with any luck they should fairly shortly.
