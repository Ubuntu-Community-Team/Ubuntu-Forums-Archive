### Post by Tomosaur on 2006-10-19
Be careful with Python. I really like it, it's very clean, and mostly easy (although sometimes it feels like you're banging your head against a brick wall, but what language doesn't?), but it is different in terms of structure than other OO languages. In Python, everything is an object, which in some ways is a massive advantage over languages like C/C++, but at the same time, you may be better off learning a language such as C/C++ where very few things are objects (unless you specify them to be) because that is how the 'common' languages work.

You may also wish to try out Ruby. It's getting a lot of buzz lately and from the little dabbling I've done in it, it's structurally very similar to python.

### Post by studiesrule on 2006-12-04
I myself have worked with C, C++ and Python (and BASIC, but that doesn't count), but I'm not particularly good at any of them (I can solve most of the 'programming challenges', but can't do any GUI and stuff like that). Personally I find that C++ is much better than C. I learnt C first, but I feel that learning C++ first wouldn't do much harm. Then again, I don't know anyone whose learnt it the other way round.
I really like Python, because its quick and simple to do something. Its an interpreter language, so I'm told that you won't get nearly the same performance as C++, but the time you take to develop stuff is really fast. Also, I feel the C++ has a lot of surprises (read Ruby's least surprise principle), even after using it quite a bit. For example just the other day, I found that you don't have to define all your variables in the very beginning of the program (unlike in C). I can't possibly express how much that bugged me. 
Whichever you choose, I recommend you read the 'Think Like a Computer Scientist in: Python/C++/Java' Series: [http://www.ibiblio.org/obp/thinkCS/](http://www.ibiblio.org/obp/thinkCS/)

I think the best part about Python is the console. So you can just type small bits of code to test out some stuff. I found this really very important when I was learning Python. In C++, do any small thing, you have to write an entire program and all. 

I don't think Java is a very good option though, it looks very intimidating. You would also be benefited if you read up these languages on Wikipedia.

### Post by pmasiar on 2006-12-04
Python is your best bet, trust me :-). You always need to look for language goals. C/C++ is optimized for expert programmers doing deep system programming, like Linux kernel, where code speed is more important than programmer's pruductivity (hours needed to finish a task). Python is more forgiving for beginners (dynamic data typing, flexible data structures: list and dictionaries, memory management) and is optimized for human readability. 

Python has shell mode like BASIC has, where you can try one command at a time and see results. Great for learning language one line at a time. :-)

Why python: [http://www2.linuxjournal.com/article/3882](http://www2.linuxjournal.com/article/3882) and [http://www.paulgraham.com/hundred.html](http://www.paulgraham.com/hundred.html)

Productivity in python is 10-20 times more than in C. C code is faster, but will take you much more time to code and debug it.

Pythn is excellent for parsing text files (which is much harder in C), which makes big part of web programming.
