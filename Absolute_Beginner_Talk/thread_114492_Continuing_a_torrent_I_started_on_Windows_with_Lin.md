### Post by dueY on 2006-01-08
Kind of a random question here.  Sometimes I find the need to switch from Windows to Linux or vice versa to test something out.  The problem is that I am often running multiple torrents and don't want to forget which ones I was running or have the seed disconnect if I am the only peer and I get off for a bit.

So I was wondering if it would be possible to for example start downloading a torrent with Windows then reboot into Linux and continue downloading?

### Post by Terry of Astoria on 2006-01-08
I would think that you can but I don't know. To test, make an exact copy of the  incomplete FILE incl. filename,(You'll have to rename the original) and try resuming the torrent on that copy. You don't want you bittorrent client to accidentally trash it. If you're using Azureus, you might try setting up Az on the Win and on the Lin to automatically open any torrents in a specific folder, and also to share download dirs. Careful. Test first carefully. Don't allow data to be compromised while testing.
