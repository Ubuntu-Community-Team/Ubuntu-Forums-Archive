### Post by AndyCooll on 2007-04-29
> **Mimsy said:**
> Seconded. It's not like I have a brand new laptop either... and WPA worked under Windows, so I seriously doubt it's a hardware issue.

/Mimsy

I agree that not presenting the option for WPA is a pain. Wireless actually works out-of-the-box with my laptops, and I get WPA working by doing the following:
```
sudo gedit /etc/network/interfaces
```
I then copy and paste the following into the file:
```
iface *ra0* inet static
address 192.168.*xx.xx*
netmask 255.255.255.0
gateway 192.168.*xx.xx*
pre-up iwconfig *ra0* essid *your_domain*
pre-up iwconfig* ra0* mode managed
pre-up iwpriv *ra0* set Channel=11
pre-up iwpriv *ra0* set AuthMode=WPAPSK
pre-up iwpriv *ra0* set EncrypType=TKIP
pre-up iwpriv *ra0* set WPAPSK="*your_WPA_domain_password*"
pre-up iwpriv *ra0* set TxRate=0
wireless-essid *your_domain*
auto *ra0*
```
**Replace xx's and domain details as appropriate, and possibly ra0 with wlan0 or whatever your wireless card is called. Also I assign static IP addresses, however you may prefer to use dhcp. In which case you may not need address, netmask, and gateway details.

I then reboot, and up comes my wireless card!

### Post by UbuntuKhan on 2007-04-30
About those crushes in Firefox: it was Flash. Every time I wanted to access a page containing Flash it vanished (Firefox, that is).
For all those who encountered the same problem:
> 
Note: if firefox crashes when visiting a website with flash content, do the following:
Open the firefox file

```
 sudo gedit /usr/bin/firefox 
```

Add the following line as last but one line of the file:

```
export XLIB_SKIP_ARGB_VISUALS=1 
``` 

Now firefox shouldn't crash anymore. (Launchpad bug report: [1])
Source:
[http://ubuntuguide.org/wiki/Ubuntu_Edgy#How_to_install_Flash_Player_.28Macromedia_Flash.29_Plug-in_for_Mozilla_Firefox](http://ubuntuguide.org/wiki/Ubuntu_Edgy#How_to_install_Flash_Player_.28Macromedia_Flash.29_Plug-in_for_Mozilla_Firefox)
