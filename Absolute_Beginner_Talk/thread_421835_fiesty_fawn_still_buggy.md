### Post by mgmiller on 2007-04-25
> this will be my first install so i dont wont any problems with the os itself. 

I take it from this statement that you have not installed Ubuntu before.  Also, from the next quote, I assume you have never installed Linux before.

> aha!!! sounds good . Ok now explain these: "home folder", "clean install", "alternate cd" .Ok what are these?

First, let me welcome you to the Ubuntu Linux community.  Hopefully, you will have an easy time getting to know Ubuntu Linux.  Remember, Linux is not Windows.  Things can often be done just as easily, but will require a different method.  If you hit any snags, you will find a lot of help here on the forums.

Lets take the questions 1 at a time.

1) "home folder"
This is a part of the Linux file system which contains most (if not all) of your personal data and many (but not all) configuration files.  It is roughly analogous to the My Documents folder in many windows installations.

2)"clean install"
This is an installation of an operating system (windows, Linux, Mac, etc.) where no instance of that operating system has existed before.  It could be the only OS on the drive or it could be the creation of a multi boot configuration where  more than one operating system will be on the drive and the active one will be selected at the time the machine is turned on.

The clean install is preferred by many as the best way to install an OS, as it gives you all the defaults and a "clean slate".  You would then have to reinstall all extra applications that you feel you need and restore any data you backed up from somewhere else.  

This is compared to an upgrade install, where an instance of the OS already exists on the drive and you are upgrading it to the next newest version.  For example upgrading Windows Xp to Windows Vista, or upgrading Ubuntu Edgy 6.10 to Ubuntu Feisty 7.04.  The advantage is all your settings and programs and data remain intact and you can just go on about your business without having to do anything else.  
That's how it's supposed to work, but in reality there are usually a few problems encountered that can range from minor/trivial to real show stoppers.  As you can see from my signature line, I have been upgrading a lot.  It has forced me to learn a lot about the nuts and bolts of Ubuntu.  I should also say this last upgrade from Edgy to Feisty was the least painful so far and I only had a few very minor issues.

3) alternate CD
Ubuntu offers 2 versions of the installer CD the first is the "Live CD" which most people use.  On boot up it runs a full version of the OS that does not use your hard drive.  It lets you try it out before committing to the install.  If you like it, there is an icon on the desktop that will do the install for you with a nice graphical installer.  You can even browse the internet while the install is taking place.
The other is the "alternate install" CD which is used for server installations and can be used as an upgrade tool or repair tool.  It can also be use to do a "clean install" (you know what that means now),  It is a command line driven install process that asks questions that you answer, but will not give you a pretty GUI during the installation.  By default, I don't think it will give you a graphical desktop environment unless you specify the ubuntu-desktop package during the install.  Some hardware configurations that will not install without hanging using the Live CD, can be successfully installed using the alternate CD.


Finally, for the new user, I would suggest using the newest version, Feisty Fawn 7.04.  It makes getting your system running in a most usable way much easier than any past version of the OS.


Good Luck
