### Post by cogadh on 2007-07-03
Unless you also installed the drivers from the Nvidia website, you just uninstalled your Nvidia driver. The nvidia-glx package is the Ubuntu package of Nvidia's proprietary drivers, which are the drivers that give your card 3D acceleration, as well as advanced Nvidia settings features. If you plan to do any gaming or anything else that will require 3D features, you will need that driver.

However, it should be noted that the nvidia-glx package is not installed by default. Ubuntu uses the generic "nv" driver out of the box and it is up to the user to install the proprietary drivers. So, if you had the nvidia-glx package installed (which is the only way it would show up in auto updates), then you had to have installed it yourself at some point.

### Post by cogadh on 2007-07-03
Just so you know, if you install any kernel updates through automatic updates, you will need to manually re-install those Nvidia drivers in order to get back to your desktop after the kernel update. If you use the nvidia-glx package (or the "legacy" or "new" packages), the drivers will be automatically updated when you update your kernel. 

The Ubuntu packages are the same drivers that you can get from the Nvidia website, they are just repackaged in an easy to install and update form. The only real difference is the Nvidia site will allways have the latest, newest, freshest driver, while the Ubuntu package will usually be one or two minor revisions behind. Since you said you don't do any 3D gaming, that wouldn't be an issue.

If you are comfortable with reinstalling your driver semi-regularly (or not updating your kernel), then stick with the Nvidia package, but if you want greater ease of use, I highly recommend switching to the nvidia-glx packages.
