### Post by Mustard on 2006-02-09
It seems to be using a built in 'sis' driver already.  I'm wondering whether you would have more success with the 'vesa' driver.

Ctrl + Alt + F1 will get you to a command line (Ctrl + Alt + F7 brings you back to the your desktop)

You could do this command from the command line to reconfigure your xorg.conf file

```
sudo dpkg-reconfigure xserver-xorg
```

Choose the defaults in the dialog, but when it gets to you graphics driver try choosing 'vesa' instead. When you finish the dialog, it should ask to write a new xorg.conf file (and it will make a backup of the old one at the same time).  You can keep running this command and try again if you make a mistake somewhere along the way.  Hopefully using the 'vesa' drivers will fix the problem temporarily until you can find a better solution for your 'sis' drivers.

To restart the x server

```
startx
```


Some additional troubleshooting would be to look in your Xorg.0.log to see what error messages you might find.  Its in the /var/log/ directory.  You can view the log from within command line using this command..

```
cat /var/log/Xorg.0.log
```

Or from desktop you could try..

```
gedit /var/log/Xorg.0.log
```
