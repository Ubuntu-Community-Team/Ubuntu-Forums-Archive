### Post by azteech on 2007-03-17
Really not be much help for installing, but here are my thoughts.

How did you obtain the distros that you have tried? Did you download and burn to cd or dvd, or did you obtained versions of them through a book or magazine, or obtained from another source?   
If you downloaded, then burned to cd/dvd, it could be possible that you have a corrupt cd/dvd that was burned, or the download was bad, there by giving you a bad 1st impression. I personally have run into corrupt downloads (especially caused by disruption of download and continuing later), bad cd/dvd media, burn software not working properly, etc. I also have a problem installing any Linux flavor using my CDRW/DVD unit (made by brand name maker), system seems to loose track of the cd/dvd media and gives the same symptoms you have. I just switch to my secondary CD-Rom and Linux then seems to install fine. After installing then the primary CD/RW/DVD unit works just fine in Linux.

I would recommend going to your local computer store to see if they have any copies of Linux laying about that they hand out. 

Or, contact the local/nearest Linux user group (who is familiar with the distro you want to work with) and see if they have any copies of a Linux live CD they can provide you. Also, the Linux user group should be able to provide assistance/guidance, with testing and installing the Linux software.

Or, finally, go to your local book store. Look for a book on a flavor of linux you wish to try, that has a cd/dvd included, or look for a linux magazine which has cd/dvd included. Then give these a try.

If all else fails, have a friend (willing one of course) download Linux for you, and burn it.

You are correct when you state that Ubuntu and ATI drivers don't play well together (I believe this is a problem with most flavors of Linux and the proprietary nature of ATI drivers, but don't hold me to that). Doing a search for ATI in this Forum will result in many a post about this issue (and in all fairness, so as not to start a flame war (which is certainly not my intent here), it isn't just ATI that we, as a Linux community, have problems with. Rather a lot of folks have problems implementing other video drivers successfully too, though it seems to be there is a higher success rate at getting non-ATI drivers to work more times than not.)

One other thought came to mind as I was typing this. Have you verified the media (CD/DVD) using the system you have been unsuccessful in loading Ubuntu (or the other linux flavors). Even though you have been successful in loading it on other machines (using this media) doesn't mean it will be successful on the one your having problems with.   
I have found over the years, that software burned to CD/DVD material on one computer may not always work when read in another CD/DVD - Rom. This, from what I understand, is usually caused by age differences in the drives, led brilliance differences, differences in manufacturers, or by mis-matches in read heads of the CD/DVD-rom drives. This also use to happen a lot with the old floppy disk drives. Sometimes, CD/DVD's can only be fully read on the machine on which they were created, where as some machines may only be able to partially read, or not read at all, the same material.

Wish I could have helped more and you the best of luck finding a solution to the problem.

### Post by NosLycn on 2007-04-07
Hi.  I have an Acer Aspire 5601awlmi

1.6 ghz core duo
512 gb ram
intel gma950 (or 945) graphics and their stupid hd audio.

It sounds like you and I have the same computer.  I have used the following distributions on this laptop:

Ubuntu 6.06 LTS Dapper Drake
Debian 3.1 rc4 Sarge
Ubuntu 6.10 Edgy Eft
Linux Mint 2.2 Bianca
Sabayon Linux 3.25c
Fluxbuntu (forget release numbers)
Damn Small Linux
and probably more that I can't remember
Kororaa!

Anyway, the point is that this computer is very capable of running all those distributions.  I would suggest that there may be something wonky going on with the hardware, or the burn of the installation CD, or the reading of the burned installation CD.

I suggest getting a friend to download and burn Linux Mint 2.2 Bianca (the distro I've been using to show off Linux to people) and see if that CD works.  It is essential that you use a different computer and different media (switch brands for this disc) and a different internet connection for downloading and burning the new Linux CD.  If it is a problem with the media or the burn, this will hopefully weed out that possibility.  I do know that sometimes the old Dapper disc I have lying around doesn't want to boot.  That's a bit of a pain sometimes.

If you want to go with a straight Ubuntu system, I suggest 6.06 LTS Dapper Drake.  Why?  Well, for starters the battery monitor is misreporting in 6.10.  You also lose ssid scanning (for wireless networks) in 6.10.  No matter which distribution or release you go with, that little 5-in-1 (or something like that--I don't use it often myself) card reader requires a bit of tinkering to get working.  And, I've heard reports that the model of the Acer 5601 with the webcam built in doesn't work with the webcam in Linux.  I don't know if that's true or not, as I do not like webcams and opted for the webcam-free version.

Best of luck.  Hope this helps.
