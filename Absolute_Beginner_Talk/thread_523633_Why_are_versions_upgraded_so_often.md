### Post by luisromangz on 2007-08-12
Yep, they get better, are easier to setup and use. They also offer newer version of programs, both installed by default and in repositories waiting for you to install them, so you get reasonably stable but recent software. Development models are different (google for "The cathedral and the bazaar" ;)) so the amount of time between releases is different, as the goals with versions are not the same.

With an unsupported system, you don't get more security updates, so you are theoretically at risk. About the drivers, well, I suppose you won't have to compile anything, but it certainly depends in your hardware (which is quite ancient from what I see in your signature :D). If you have any specific trouble you can always ask.

I would recommend you to use Xubuntu instead of Ubuntu, as it is said to be lighter in older hardware as yours. I haven't used it myself (I'm a KDE-addict ;)) so I can really tell.

### Post by luisromangz on 2007-08-12
Well you know, KDE and GNOME are the two main Linux desktops. You know, Linux is about choice (among other things, like writing in forums instead of working ;))

Err, there are many differencies, for example KDE is build using Qt and GNOME using Gtk. Those are two different toolkits for graphical applications. So Gtk apps looks nicer in GNOME than in KDE and viceversa, although that can be solved.

Each desktop has their own applications. For example KDE's text editor is KEdit, GNOME's one is GEdit. It also depends in the programs your distro packages for your desktop.

There are a lot of fanboys in both camps, who will tell you that X is so much better than Y, no matter the facts. I tend to stick with KDE, as is the first Linux desktop that I used. I think you should try both, and see what suits you more.

Many people says that KDE is more Windows-like, I personally think that is way more customizable than GNOME (and Windows of course ;)). GNOME offer less options, but that allows it to be more tidy, perhaps.

### Post by steve.horsley on 2007-08-12
The releases every 6 months are indeed new versions, but the changes between them can be quite small (only 6 months work), so maybe they're more like the service pack versions of Windows. Each release is generally only supported for 18 months, and the support is only for security updates and major bug fixes - not new features.

As for the names - well you've got to tell which version someone has when talking about Ubuntu. The project names are going alphabertically, I don't know what will happen after Zany Zebra. The version numbers (I prefer the names) are year.month, e.g. 7.04 is really 2007.04.

Upgrading can be quite traumatic if you're not familiar with Ubuntu. It may work perfectly (likely), or it may go horribly wrong in which case a beginner won't stand much chance of fixing it. It may be worth sticking with Edgy for a while, at least till Gutsy is released and you are more familiar with Linux in general. 

If you had to find and compile drivers for Edgy, you will probably have to repeat for Feisty, Gutsy...
