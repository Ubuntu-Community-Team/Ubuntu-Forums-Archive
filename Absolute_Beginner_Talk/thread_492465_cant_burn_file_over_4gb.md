### Post by aeiah on 2007-07-04
if ive read it correctly i think i know what the problem is...

as im sure you're aware, single layer dvds are 3.7gb. they can hold a bit more than this, and the remaining extra space is used for error checking. when you burn an image, you can switch to a mode whereby the extra space can be used by the image. this is because images can have their own error checking systems, like SVCDs for example, and a lot of install discs.

you can also overburn data, but only to a certain point, and it seems you've hit that limit. what kind of file is it? there'll be ways of splitting it up, even if you have to put up with splitting it over multiple archived segments
