### Post by vicantre on 2007-07-07
Actually, it works fine with just:

./configure 

make

sudo make install 


Be sure you have installed 'pidgin-dev' libraries (nowadays, only available in Gutsy). 

If you're not running Gutsy - a.k.a: you're a Feisty user-, then you shoud [get the pidgin code from the official website]("http://pidgin.im/pidgin/download/source/"), build your own pidgin with the same stuff above here and then, try to install musictracker.

Those backports form getdebs or other places are just binaries 'ready-to-use', but not useful for any development tasks, like install additional plugins. Backports are just backports.

Cheers

### Post by fajar on 2007-12-06
Using gutsy, I find the easiest method is to use prevu :
- install prevu : [https://wiki.ubuntu.com/Prevu](https://wiki.ubuntu.com/Prevu) . Don't forget to let prevu add lines to /etc/apt/sources.list
- prevu pidgin-musictracker
- apt-get update
- apt-get install pidgin-musictracker

### Post by ephemerae on 2008-02-17
I had good luck with the following.

Download source tarball from [http://code.google.com/p/musictracker/]("http://code.google.com/p/musictracker/") and unpack it in /tmp or something.

# cd /tmp/musictracker-0.4.1/
# ./configure

At this point configure complained about not being able to find various things which were fixed by installing the following (which drag in a few useful dependencies):

# sudo apt-get install pidgin-dev
# sudo apt-get install libpcre++-dev

At which point you can proceed with

# ./configure
# make
# sudo make install
