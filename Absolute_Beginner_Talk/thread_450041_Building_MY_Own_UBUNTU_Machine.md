### Post by irish_flu on 2007-05-21
All the hardware that interfaces with the OS does indeed need a "driver", whether you're using Ubuntu or Windows, no matter what motherboard you're using.

The nice thing is that Ubuntu (and Windows, for that matter)  just sees most of it for you (like, say USB ports) and installs the "built-in" driver automatically without you having to worry about it.

Sometimes, you'll need other drivers for less common stuff (say, for a video card built onto the motherboard or for your printer).  My rule-of-thumb when building a cheap linux box is to try to use incredibly common parts.

For instance, the last time I rebuilt my Windows box (which I hand-built), Windows had almost all the drivers built right in; the only things I needed to find/install drivers for were my printer and my NVidia video card (because Windows couldn't really utilize all the cool hardware on the card without an Nvidia-produced driver).

Likewise, last time I rebuilt my Linux box (right now that's a hand-me-down Dell Optiplex) the only driver I needed to add was for my NVidia card (and that was only to get 3d acceleration; the card worked fine otherwise with the "built-in" driver in Ubuntu).

What you need is a motherboard that contains components that are well supported by Ubuntu.

I've used a lot of different brands of mobos over the years.  The most stable have been the ones Intel makes.  Here is a url to Intel mobos:

[http://www.newegg.com/Product/ProductList.aspx?Submit=ENE&N=2010200280+50001157&name=intel](http://www.newegg.com/Product/ProductList.aspx?Submit=ENE&N=2010200280+50001157&name=intel)

Pick out a some that are close to what you want then research the individual components to make sure they are well supported.  You might want to get a mobo with an integrated Intel video chipset as those are well supported.
