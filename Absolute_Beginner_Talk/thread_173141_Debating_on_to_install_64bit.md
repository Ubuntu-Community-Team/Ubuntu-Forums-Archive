### Post by Ozitraveller on 2006-05-10
I'd do a clean install, there could be some binary incompatibilities. Also you might find that you will have to re-compile code to get it to work or just to get the software that you want. If you aren't confident with compiling source code, I'd stay with 32bit.

If you have installed 32bit software and much of it would have specific version dependencies. And those dependencies would also be architecture dependant. They have setup different repositories for the different versions and architectures for a good reason, they won't play nice if you mix'n'match.



Good luck.
