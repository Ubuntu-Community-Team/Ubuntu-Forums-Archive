There are a couple of big board categorizes the separate the various linux distros: Package Manager, Desktop Environment, and System Tools.

On the package managers front, there are two main systems most distribution use RPM and DEB. Ubuntu, like all other Debian derivatives, uses DEB packages. On the deb side of things there is a lot more unification in the tools used, it's pretty much all dpkg and apt-get. RPM stands for Red Hat Package Manager, and is more often used in the more commercial distros. The tools for managing RPMs are a bit more diverse SuSe has yum, Mandrake has urpmi, and I'm not really sure what Red Hat/Fedora is running these days. In addition there are some other systems floating around, there's autopackage for example, it's the "wave of the future" and going to take off big "any day now", trust them. There's a whole family of source base distro descended from gentoo and slackware.

For desktop environments there are currently two big DEs, and a third fighting it's way to the top, and more small environments than you can count. The first of the two big DEs is KDE, it's the older (longer running), of the two big desktops and is often characterized as being more windows like, and focuses on everything being customizable, you just have to find the right config screen. I personally prefer KDE applications over there gnome counterparts mostly out of familiarity, and the fact they include tabs on pretty much everything, I think even some of the tabs might have tabs in some apps. The other big contender is GNOME, which is the default desktop of Ubuntu, it was built around the goal of trying to make everything as easy to use as possible. It should be obvious how to do things the right way the first time. You're likely to get the best support around here for gnome as that's what most people are running, I do however relish the opportunity to answer KDE and XFCE questions as they don't come up as often. Which brings us to the third big DE XFCE, xfce is relatively new, one has to assume they had release before the xfce4 series, but didn't really catch any big attention until the xfce4 series. Since then there have been 3 releases 4.0, 4.2, and 4.4. XFCE is the lightest of the big desktop environments. XFCE is in a lot of ways the best of both worlds gone on a diet. It looks and feels a lot like gnome at first glance, customizes like KDE, and does it all with far less memory usage. It's still a light weight desktop and was designed to be such, so don't expect it to do everything KDE or gnome do.

Then there are the configuration tools. Each distro approaches configuration a little bit differently which has led them  to each develop different and unique confg tools to fit their  project goals. SuSe has YaST, which is a combined control center, package manager front end, and server config manager/editor. Mandriva has the Mandrake control center, which has the best from Kcontrol + some of their own tools. Ubuntu doesn't have a post install tool that stands out, but as far as installers go Ubiquity is pretty much the best there is, Red Hat has a whole suite of Red Hat-* server configuration tools.
