### Post by Brunellus on 2006-09-22
"warty," "hoary," "breezy" and "dapper" are the short-form codenames of the last four Ubuntu releases.  

The ubuntu version numbers follow this convention:  YEAR.MONTH (Y.MM). 

The first public release of ubuntu was in October of 2004, so 4.10.  It was codenamed "warty warthog."

Thus, this (current stable) version was released in June 2006, thus 6.06.  It is codenamed "Dapper Drake"

Why codenames?  

1) They're fun.  
2) They provide a good shorthand way of referring to a distribution release.
3) Because of the year, month versioning convention, one cannot simply say "ubuntu 4, Ubuntu 5, Ubuntu 6" since there are two 'stable' releases in any given calendar year.  

To know what version of Ubuntu you're running, execute the following command in a terminal:

```

uname -r

```

* * * SOME OTHER GENERAL THINGS ABOUT VERSION NUMBERS * * * 

One common version numbering convention in the Free Software world concerns sequentially-numbered versions: 1, 2, 3....  

In general, anything with an EVEN number will be a STABLE or "production" release.  Anything with an ODD number will be considered an UNSTABLE or "development" release.  

In sequential version numbers, the dot does not indicate decimal place.  The convention is 

Major release.minor release

Thus GNOME (Ubuntu's default desktop environment) is currently at version 2.14, which means "Major revision 2, stable release 14."  The next stable relase will be 2.16
