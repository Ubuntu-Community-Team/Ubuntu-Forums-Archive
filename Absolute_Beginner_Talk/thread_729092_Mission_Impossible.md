### Post by Hospadar on 2008-03-19
Hi!
It will undoubtedly be tricky, especially for a young linux n00blet, but very possible and if nothing else, very educational.  That said, I hope you are a patient person, or at least willing to learn a little patience.

Also, I just found this today, It might help, how to set up a secure mail server with postfix: [http://flurdy.com/docs/postfix/](http://flurdy.com/docs/postfix/) that's not the only way to setup a mail server though

As for your webserver, basic stuff is pretty easy, you can install apache, php, mysql right out of the repos and they will basically work with pretty minimal config.  You can also find tons of tutorials for all three as they are heavily used. do google searches for stuff like "ubuntu apache tutorial howto" and you'll find lots of info.

Also, if you arn't familiar with linux, it might be a good idea to first acquaint yourself with how the linux filesystem is set up (ie, which stuff gets saved where, there's no "Program Files" or "Documents and Settings") and also how to use the command line (especially with ubuntu server).  Both of these things are readily available in tutorial format all over the internet.

Also certainly this forum is a great source for answers to all your basic questions.

You may also want to take a gander at the links in my sig, as well as the main psychocats page, they have a lot of great linux info [http://www.psychocats.net/ubuntu/index.php](http://www.psychocats.net/ubuntu/index.php)

Another good resource is ubuntu geek:
[http://www.ubuntugeek.com/](http://www.ubuntugeek.com/)

IMHO, you should consider using the regular Ubuntu release. The learning curve will be steep enough for a noobster without the culture shock of going from GUI only to CLI only. 
Any server program you can run on the server edition will also run on any other edition.
The only downside here is the GUI does consume some system resources, but if you are using a P3x700 or better with adequate ram, the GUI should run fine. Besides, once you are comfortable with the command line, you can dispense with the GUI to save system resources....
My two cents.
