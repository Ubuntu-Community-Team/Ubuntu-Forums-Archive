Something I learned the hard way, gnome-system-panel is great... except for viewing processes. As it should, it is only being run as user by default. Open a terminal and "gksudo gnome-system-panel". this will actually show all processes being run, not just those initiated by the user. As an example, trackerd is often run as root and the current user at the same time with the root privelaged one doing all the work that is eating up your memory, and likely clock cycles.

This was my savior for a long time, but now I use "sudo top -bn1".

Another memory killer is leaving Firefox minimized in background that has a page with flash. Often the case if you see a lot of activity for "firefox-bin". Its annoying, but easy to work around if you like / need flash installed. I found not installing flash is a great anti web page spam.
