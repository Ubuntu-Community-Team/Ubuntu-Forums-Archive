### Post by arbulus on 2008-01-10
Usually there's a setting in the BIOS of a machine that would cause it to throw up a warning and halt the loading process if a keyboard or mouse or monitor is not connect.  If your BIOS does this, then you should be able to go in and disable those warnings, which will allow your machine to boot without the monitor connected.

### Post by DavidTangye on 2008-01-12
ssh is a command line (**s**ecure) **sh**ell. You run it in a command window. It allows you to run commands on the remote machine. This is useful when the remote machine is a 'headless' server, ie it has no monitor and is there to serve stuff, eg webservices, gateway/firewall etc. Being a server it might not have any X server running. However -X allows the address of the PC, and its X server to be passed to the remote server, so that when you run gui /X programs on the remote machine, using commands within the ssh session, the output windows are shown back at the machine you are on.

There is also a way or two to have an X/gui desktop etc on a remote machine transported to your machine. I think this is remote desktop software. I have not had the need to use this though for many years. It allows you to take control of a remote desktop, ie it is more useful for running stuff on a remote workstation, than a simple server.
