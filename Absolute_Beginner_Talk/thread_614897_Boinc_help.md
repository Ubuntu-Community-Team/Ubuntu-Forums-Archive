### Post by approaching on 2007-11-16
to kill it i am sure you have figured out that you need the acutal name of the process or the proc id#. if you are adamant about the command line, you can use top. that will give you the current proc list (type 'top -h' first to figure out the commands, ie <> move between pages). from there you can kill the proc id# or the real name.

to locate the boinc folder tye in this command:
locate boinc

it should give you a more than profuse description where any file with boinc in the name is located.
