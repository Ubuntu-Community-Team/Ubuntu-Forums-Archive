### Post by tonyric on 2007-09-09
It is simple, it states "This is not a virus". Also, there are ClamAV test files on the system which are normal, and the GZ files are kernel files. You have nothing to worry about there. 

NOW, basic information for you, you are running linux, you have no need to be concerned with Virus' on your system. There are so very few for linux and the few that there are, are not causes for concern. The only issue that there could be is if you are running a mail server or Samba (Windows File Sharing) at home and have Windows systems on your network. If you do then you *could* unknowingly assist in the propagation of virus' to your Windows systems, but if you are running anti-virus on them, this is not a concern as those systems will protect themselves.

Anti-virus on linux in small networks are unnecessary. Do not feel like you need to waste the system resources. :)

### Post by Bothered on 2007-09-09
> **newtolinux_08 said:**
> when i scanned my pc i found 23 viruses then i stooped scanning! :???::-?

As said by others, your scanner has detected potential problems, rather than viruses. These detections look like they are nothing to worry about. Six of them are test files. Most of the other detections are as a result of enabling the ArchiveBlockMax option in ClamAV (see [here]("http://www.gossamer-threads.com/lists/clamav/users/15869")). If you don't want to see these detections (they're probably nothing to worry about) then disable this option.

> **newtolinux_08 said:**
> should i use clamav?is it better than klamav?

KlamAV is a KDE graphical front end to ClamAV (see [this Wikipedia entry]("http://en.wikipedia.org/wiki/Klamav")). Hence you're using ClamAV, and controlling it using KlamAV.

Hope this helps!
