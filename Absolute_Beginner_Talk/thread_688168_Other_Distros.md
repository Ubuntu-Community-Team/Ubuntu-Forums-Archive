### Post by kpkeerthi on 2008-02-05
> Just wondered if anyone else had any thoughts on other distros
I'm hooked to Archlinux. 

> Fast and sleek - i686 optimized
> Rolling release. Packages are continuously tested and added to main repositories and are considered 'stable'.  There is an optional 'testing' repo (like debian testing/sidux) if you want to stay more up-to-date.
> Arch's packages are in more 'vanilla' form - little to no customization added.
> Ubelievably easy to compile from sources (with optimizations).
> Powerful package management.
> Community contributed source repos (Arch User Repository - AUR).
> BSD/Gentoo-like ports system (Arch Build System - ABS).

I've been through several rounds of upgrades and its been fast and stable. One fine distro.

### Post by julian67 on 2008-02-05
I don't believe there is such a thing as a single distro that meets every kind of need, though Debian, Gentoo and Slackware probably come closest but demand more from the user to achieve this than others. I have 3 machines, an old desktop (athlon 2700 1GB RAM, runs Xubuntu Gutsy, NAT, DNS server, Samba server, DHCP server, SSH server) an oldish laptop (pentium-m centrino 1.6 GHz, runs antix on debian unstable) and a newish laptop (Core Duo centrino and the accursed intel hda sound controller, runs Xubuntu Feisty) and I would really like to be able to run completely binary compatible distros on each to make for an easy life, but actually I found it impossible. The core duo laptop's sound controller can only be made to function correctly in *buntu feisty with default kernel + a bit of work, all other distros/kernels don't support the slightly bizarre (subwoofer  + stereo pair + digital out) hardware configuration. Whereas I need something newer on the desktop to use latest version of Gimp and some other stuff important for me so I run Xubuntu Gutsy (easy life, auto updates etc) and on the older laptop I just need something with very low resource use and easy wireless, power management, suspend etc. Xubuntu works but not with great speed. Antix with either fluxbox or Xfce absolutely flies on that machine and puts Xubuntu to shame. 

My point is that it's well worth exploring different distros, even versions of distros, and using whichever is best suited to your hardware *and* your purpose.  There are so many distros out there, many are just OK but not great, some are basically vanity projects, some are extremely specialised,  but there are also some real jewels to be found outside the heavyweight arena occupied by Ubuntu, openSuse, Fedora etc. Some of the Slackware based distros are amazing (Wolvix hunter, Zenwalk for example), making Slackware friendly and accessible (i.e the basic configuration is nicely done for you), the same goes for some of the other Debian derivatives. 

Once you get past the stage where "will it automatically support my hardware?" is the key factor in choosing a distro then you might find other important issues that ultimately matter more. What about upgrading the distro? What about installing newer versions of applications? Security notifications?  How good is the QA? Is it easy to compile applications from source? It's well worth exploring some distros that are rolling releases (i.e. no more download an iso, burn, re-install or scary times with upgrade manager). Failed or problematic upgrades from version to version get extremely inconvenient and tiresome after some time.  It's also worth trying some distros that do only minimal customisation of the tools, packages and filesystem....you might be surprised how easy it is to compile from source when everything from your OS to the source package is set up in a very standard way (no bizarre dependencies, no modularised packages, no highly customised config or set up tools).  I'm not saying this is the "right" way, only that it's well worth investigating and experiencing.  This is when the search begins for an "install once, use forever" distro. Slackware, Arch, Gentoo and Debian are likely good candidates for this.  Ubuntu unfortunately doesn't implement the Debian stable/testing/unstable model, instead preferring a 6 monthly snapshot approach, neither does it do much backporting beyond security patches. These kinds of things probably don't even enter one's mind as a new Linux user but as time goes by they get more and more important.
