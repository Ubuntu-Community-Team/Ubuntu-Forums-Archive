### Post by chris.a.barker on 2008-02-26
This is actually a pretty complicated topic. There is an agency called ICANN. They are the folks who govern DNS resolution. For example, your URL scenario. mysite.com has to point to an IP address such as 127.0.0.1. (Thats not a real address) This registration process is done through a courier service called a registrar. Basically they take information that will be provided by a WHOIS server and they register it with ICANN. This in turn registers your IP address with whatever domain name you have purchased. When you pay, you are basically paying the registrar to submit your information to ICANN. Other considerations are the fact that you need a static IP address to pull this off. Otherwise you can go to a site like dyndns.com and register a dynamic dns name for free. 

Hope this helps you out.
