### Post by Delkster on 2007-09-05
If GParted shows everything as unallocated, it seem as if the partition table may have gone bad. Try the fdisk command that benerivo suggested, though, and see if the results are any different.

Even if the partition table is corrupted the data is probably still there -- the system just doesn't know how to access it. There are tools that may help in recovering from that, but let's first check the fdisk output.

I'd sure be worried if I were you, but mostly because any problem with a hard disk freaks me out. There's probably a good chance that we'll get this solved somehow, and that you'll at least get your data back. Of course if the disk has died or something, that gets quite a bit more difficult... but let's not panic yet, ok?

### Post by benerivo on 2007-09-06
If your broken hard drive is not being picked up at all in Ubuntu, then i don't think there will be any way to fix it with this approach. Maybe try going in to the BIOS settings at boot, to see if at least your BIOS can see the drive. If it can't then it's a major problem. If it can then another method maybe to try attaching only the broken drive as the 'Master' and loading the Ubuntu live cd. From there, you may be able to see it and copy important stuff on to your usb drive.
