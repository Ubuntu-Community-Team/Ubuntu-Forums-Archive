### Post by jak3676 on 2007-10-18
I'll try to answer this one, but I have to admit I'm probably the least experianced Linux user here.  I just stumbled across the answer while I was downloading 7.10 myself.

Learn how to verify that your CD download ok: 
[https://help.ubuntu.com/community/HowToMD5SUM]("https://help.ubuntu.com/community/HowToMD5SUM")

Basically MD5 Sum is a method to verify that your machine correctly downloaded the program and burned it to disk correctly.  With a file as large as the Ubuntu install program is, it is fairly common for a simpe corruption to occur.  It's no big deal if it did, you just have to download it again, and you get to use your current CD as a shiny new drink coaster (or frisbee, chew toy for your dog, etc.).

Depending whether the problem was a corrupted download (more likely) or a improperly burned CD, you may also want to check these links out for a quick guide on burning the CD and verifying that the CD burned correctly.

CDIntegrityCheck 
[https://help.ubuntu.com/community/Installation/CDIntegrityCheck?action=show&redirect=CDIntegrityCheck]("https://help.ubuntu.com/community/Installation/CDIntegrityCheck?action=show&redirect=CDIntegrityCheck")

BurningIsoHowto
[https://help.ubuntu.com/community/BurningIsoHowto]("https://help.ubuntu.com/community/BurningIsoHowto")
