### Post by Tecguy2 on 2007-09-15
SVN means the most up-to-date version of the code, even more updated than the latest release.

It is the project's set of working files that have been edited since the last release came out.

For example, a bug can exist in release 1.3.x but fixed in SVN, meaning those changes will be published in the next official version.

### Post by Dylock on 2007-09-15
SVN is a way of keeping track of code. Like a repository.  You add something to the repository, and as you make changes you can keep track of previous versions of that file, comments about that file, and who made the changes. Its good if your working on projects with multiple people or have long term projects.
