### Post by DBStevens on 2007-07-27
Well according to this:

Section "Device"
Identifier "nVidia Corporation G80 [GeForce 8800 GTS]"
Driver "nvidia"
EndSection

you are running the closed source driver.

I don't know if it is the latest or even if the latest closed source driver works with your kernel.

You could look at the date and time stamp on the driver to get an idea how old it is (Snaptic should be able to tell you a lot more).  Then visit the nVidia web site and see what is there.  Just remember there has to be a match between the code in the kernel and the binary nvidia driver.  Hopefully there is information on nVida's site so you can pick the latest compatible driver.

### Post by Rocket2DMn on 2007-07-27
> **jrusso2 said:**
> Best is the one that works well for you.  Not always the newest one

Exactly.  If you don't have problems with your current driver, I would suggest not fiddling with it.  It you DO decide to screw with it, make a backup of your xorg.conf file first so you can recover it in case you break something
```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.backup
```

### Post by MrHippocampus on 2007-07-27
Well, if you want to turn on some features which are off by default there are one or two, since you have a 8800 im assuming you have loads of video ram to spare as the first two options will make X consume more memory. Ive found them to make things run a bit smoother.

Remember to backup your xorg.conf first!

```

Section "Device"
Identifier "Videocard0"
Driver "nvidia"
VendorName "NVIDIA Corporation"
BoardName "GeForce 8800 GTS"

#extra options
Option      "BackingStore" "True"
Option      "TripleBuffer" "True"
Option      "UseEvents" "True"

EndSection

```

you can read about what these options (and others) [here]("http://us.download.nvidia.com/XFree86/Linux-x86/100.14.11/README/appendix-b.html")

### Post by aidanr on 2007-07-27
According to all the how-to's I've seen including nvidias official one you should have:

Option          "AddARGBGLXVisuals" "True"

In section screen, although if compiz is running for you without it I'm not sure it'll make a difference, worth trying anyway

Also if you're trying to improve the performance of compiz, I'd recommend you do these tweaks first, in ccsm -> general options -> display settings, turn _off_ detect refresh rate, set the refresh rate to 100 or higher and turn on sync to vblank
