### Post by igknighted on 2007-04-23
For a slow computer, Xubuntu or Fluxbuntu will be the quickest.

If you really want Gnome or KDE, I'll ask it this way:  If you use KDE, will you use KDE apps?  Like web browsing in Konqueror and not Firefox, Krita instead of gimp, etc.?  If so, KDE is the way to go.  Also, if it feels sluggish use the control center to turn down some graphics settings, it will improve the UI speed considerably.  For using GTK apps stay with gnome.  There is less penalty for using a few KDE apps in gnome then using gnome apps in KDE, because you already have kde libraries preloaded and you are overlapping function by pulling in gnome ones too.  KDE shares better, so if you run all KDE apps it is faster... but if you split apps or prefer gnome apps, go with gnome as this sharing will slow you down more than speed you up

### Post by jerrylamos on 2007-04-23
Everybondy's opinion.  Xubuntu is a little faster, less features, not good at LAN file sharing.  Feisty Kubuntu has some problems with install partitioning fixed with standalone Gparted.

Ubuntu I seem to be able do do everything this "ordinary desktop computer user" wants to do including most (not all) internet videos, copying and printing internet articles with graphics, digital camera crop & print, documents, spreadsheets, tinker with Xwindow support, LAN file sharing, yata yata.

Kubuntu has more eye candy and seems to me people that are more interested in playing with Linux and games and fancy setups gravitate to Kubuntu.  Now I get a problem right away because their vaunted Konqueror that browses everything doesn't do AOL mail.  With so many options it takes more keystrokes and more menus to do ordinary things, like what is my screen resolution?  Also, in some cases like safe graphics mode Kubuntu on my machines is noticeably slower.

By far the best way to find out your preference is to download all three CD Live and try them out.  The CD Live runs pretty much like installed except a little slower accessing the CD.  With 524 mb memory on all five machines I use I just don't have a problem with that.  Edgy and Dapper have a "persistent" mode to save CD Live data, changes, package installs, ...  Persistent is broken on Feisty.

Now what's a slow computer?  This one's an old 1 gHz Celeron with 512 mb memory and a 17" 1280x1024 LCD screen, runs nice and fast and crisp with seemingly instantaneous screen response and as full speed videos as my DSL will run.  I have a faster 2 gHz Pentium 4 which really makes adifference on Windows & XP, but I can hardly tell the difference with Ubuntu.

Cheers, Jerry:)
