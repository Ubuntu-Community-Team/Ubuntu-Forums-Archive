### Post by santaslittlehelper on 2007-12-20
Can you switch to tty with ctrl + alt + f1?

If not boot in recovery mode, check you're xorg.conf is set to vesa, then type "telinit 3"

I installed the 169.04 drivers -
[https://help.ubuntu.com/community/NvidiaManual](https://help.ubuntu.com/community/NvidiaManual)
 - this is pretty much the steps I took.

```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf_backup
sudo apt-get install linux-headers-`uname -r` build-essential xserver-xorg-dev
(N.B - if you already installed the beta drivers, try removing them again - sudo sh NV (press tab) --uninstall)
sudo apt-get --purge remove nvidia-glx-new nvidia-settings
sudo rm /etc/init.d/nvidia-*
```

```
sudo nano -w /etc/default/linux-restricted-modules-common
```
and find the line -
DISABLED_MODULES=""
replace it with -
DISABLED_MODULES="nv nvidia_new"

ctrl + alt + f1

```
sudo /etc/init.d/gdm stop
sudo sh NV (press tab)
sudo /etc/init.d/gdm start
```

If the nvidia script didn't configure you're xorg.conf do a sudo dpkg-reconfigure xserver-xorg

```
sudo nvidia-xconfig --add-argb-glx-visuals -d 24
```

That worked for me.

I didn't remove the "nvidia-kernel-common" (linux-generic linux-restricted-modules-2.6.22-14-generic linux-restricted-modules-generic restricted-manager

Da** guys, im sorry. I should have remembered this- On the 8800 cards, you need to use the website beta drivers (thanks bodhi zazen). On my 7600, I can use either, so I got complacent. There is a sequence you need to go through to get it installed though, and it does involve killing X.

If its in low-graphics mode, thats fine. Heres a little walkthrough:

Download the nvidia beta driver and put in your home folder. Change the name to "nvidia.run" Go into synaptic and make sure you have the nvidia common kernel installed, as well as build-essential and the libc development headers.

Kill X:

```
sudo /etc/init.d/gdm stop
```

Alt+F2 at the black screen with blinking cursor.

Regenerate xorg for good measure:
```
sudo dpkg-reconfigure -phigh xserver-xorg
```

Run the nvidia driver:
```
sudo sh nvidia.run
```

and let it automatically update the xorg.conf (dont worry about the downloaded kernel- it will compile one with build-essential, the common-kernel, and libc)

Restart X:
```
sudo /etc/init.d/gdm restart
```

Once gnome is up, open synaptic and search for the nvidia-common-kernel. Remove it. Its only needed to compile the driver...

Restart the computer and all should be well. Let me know if there is any issues...

### Post by bodhi.zazen on 2008-01-06
That is your problem, as I have been telling you, you need to remove the nvidia-glx and nvidia-glx-new drivers and then re-install the nvidia driver.

There is a known bug with the nvidia-glx-new driver in that it leaves a file that has to be manually removed.

You need to remove that file.

Try 

sudo rm /lib/linux-restricted-modules/.nvidia_new_installed


See this bug report : [https://bugs.launchpad.net/ubuntu/+source/linux-restricted-modules-2.6.20/+bug/106217](https://bugs.launchpad.net/ubuntu/+source/linux-restricted-modules-2.6.20/+bug/106217)

See also this wiki page : [https://help.ubuntu.com/community/BinaryDriverHowto/Nvidia](https://help.ubuntu.com/community/BinaryDriverHowto/Nvidia)

Which states : > If your card does not appear in this [WWW] list of cards known by Ubuntu 7.04 NVIDIA binary drivers (e.g. the 8600GT) then there is no Ubuntu 7.04 supported binary driver. For unsupported workarounds try the links in See Also.
