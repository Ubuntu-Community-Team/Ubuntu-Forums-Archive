### Post by Metacarpal on 2006-09-21
Just did a little reading...

The Wii will only have a 512MB internal flash-rom memory chip (for storing game saves and "downloadable content"), so there's no internal hard drive to install to.  It plays games from 12cm DVDs (single- or dual-layer).  If you were able to burn a live-DVD that was formatted to be detected by the system as a valid game disc, then preempt the Wii's native OS with the Linux kernel, then theoretically, yes...

But why?

The Wii is a game console.  I mean, I'm all for trying to bring open source software to every part of the world, but really - this is a toy meant for playing video games.  What's the point of getting it to run a Linux LiveCD session?
