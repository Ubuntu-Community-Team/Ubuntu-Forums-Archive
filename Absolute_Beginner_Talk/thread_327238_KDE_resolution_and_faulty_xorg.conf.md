### Post by gpaciga on 2006-12-28
Recently I changed my resolution to 1024x768 and now I'm having a hell of a time getting it back up to 1280x800. I'm using Kubuntu 6.06 and X Window System Version 7.0.0 on a ATI Radeon Mobility 9000.

When I change the resolution under Settings > Peripherals > Display, everything looks OK until I reboot. Then, the login screen is 640x480 and when I log in, I get something that might be 1280x800 or 1024x768 but is mangled so that only a small area of the screen is actually usable.

When I revert back to my old xorg.conf file, the login screen comes up 1280x800 without problem but after I log in the display drops to 1024x768, despite the fact that the *only* resolution mentioned in my xorg.conf is 1280x800.

I can only come up with two conclusions:
1) The KDE Display configuration dialog gets its information from somewhere other than xorg.conf, since it shows as resolution options things which do not appear in xorg.conf. (The working xorg.conf does not have, and never did have, a modeline, for example).
2) When KDE tries to save a new resolution, it does a terrible job of it and messes up the xorg.conf file beyond all recognition, despite the fact that it clearly stores the user's resolution in a different location.

So the question is, how do I change my user's resolution without letting KDE destroy my xorg.conf? I can't find out where that information is stored, and I think I've checked everywhere in my home directory. There is no .xsessionrc or similar, for example. What I can't figure out is, if the user resolution really is stored somewhere else, why would putting back the working xorg.conf change it back to 1024x768, unless KDE never successfully changed the user resolution in the first place?

### Post by gpaciga on 2007-01-11
Update:
I finally found that KDE stores the user's individual screen resolution here:
~/.kde/share/config/displayconfigrc

The problem with kcontrol changing the resolution is unresolved. I have confirmed that it not  only destroys the xorg.conf file, it also fails to edit the displayconfigrc file at all. It seems like this is a pretty serious problem that should have been addressed, but maybe it's just something weird about my computer. I guess it doesn't matter as long as I have my backup xorg.conf and know how to change the resolution manually.

### Post by timsoft on 2007-07-26
~/.kde/share/config/kcmrandrrc
contains the height and width settings

these settings may be reset by the following file
~/.kde/share/config/startupconfig
which may also contain the height and width settings

I changed the startupconfig file and that fixed it for me.
