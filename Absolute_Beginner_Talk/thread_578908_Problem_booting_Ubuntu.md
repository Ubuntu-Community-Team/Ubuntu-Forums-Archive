### Post by rkd on 2007-12-18
I found documentation about Dell Dimension 4100 on the web here:

[http://support.dell.com/support/edocs/systems/dzuul/](http://support.dell.com/support/edocs/systems/dzuul/)

You can look there to see pictures of the Setup screens so you know what to expect (look at System Setup Program).

It says that you get into the BIOS setup program by pushing the Del key when the Dell logo appears during boot-up.  You said you discovered this on your own. I repeat it in case you aren't sure just when to push the key.

When you get into the Setup, you should see a text-mode screen display with the words Main Advanced Security Boot Exit across the top of the screen.  When you see that, push the right arrow key until Boot is highlighted, then push Enter.

That should bring up the screen on which you can control the boot sequence.  

Push the down arrow until the entry labeled "1st Boot Device" is highlighted.  It probably will be Floppy, in which case you don't need to change it.  If you need to change it, push Enter, and use the arrow keys to move the highlight to Floppy and push Enter.  

Once the 1st boot device is right, push the down arrow until "2nd Boot Device" is highlighted.  If it isn't ATAPI CD ROM, push Enter and change it to ATAPI CD ROM.  

Then push the down arrow to highlight "3rd Boot Device".  If it isn't IDE-HDD, push Enter and change it to IDE-HDD.

Then push F10 to save the changes and exit the BIOS Setup.  The computer should then continue to boot.

I'm assuming the boot device names are as on that web page. There is a chance the names are slightly different, but I think  you'll be able to recognize the right choices.  If you are unsure, tell us the choices for device names and we'll be able to tell you which is which.

With that adjustment, the computer will first try to boot from the floppy drive.  If there is no floppy in the drive, it will try to boot from the CD drive.  If there is no disk in the CD drive, it will try to boot from the hard disk.

To boot the Ubuntu live CD for testing, put the CD into the CD drive and restart the computer.  It should boot from the CD.  If everything goes right, Ubuntu will come up and you can experiment with it to see how well it works on that computer.  Ubuntu won't write onto your hard disk when running from the Live CD, so your data should be safe during the experimentation.

If Ubuntu doesn't come up, take note of any error messages and any other symptoms you see, then post a description here and we'll try to help you past the problem.

If you decide that it is worth trying to install Ubuntu onto your hard disc, you should read the advice overdrank pointed to at:

[http://www.psychocats.net/ubuntu/installing](http://www.psychocats.net/ubuntu/installing)

before you start, so you understand what your choices are and can decide which to choose.  (And remember to copy any of your data files you care about onto floppies, CDs, and external disk drive, or somewhere else safe before trying to install Ubuntu, just in case something goes wrong during the install.)

### Post by rkd on 2007-12-18
Your description of what happened after you got the Ubuntu CD to boot wasn't clear enough for me to tell what really happened.

Unless you got to the point where Ubuntu had fully booted from the CD and then you clicked on the Install button, it should not have modified your hard disk at all.  So if you remove the Ubuntu CD from the CD drive and start your computer, it ought to boot into Windows 98, exactly as it had been doing before you started to experiment with Ubuntu.

If you did get to the point where you told Ubuntu to install itself, unless you told it to take over the whole hard disk, it should have just made some space for itself without removing Windows 98, and you should be able to boot Windows 98 after taking the Ubuntu CD out of the CD drive.  

Only if something went wrong with Ubuntu trying to make some space for itself would you not be able to boot Windows 98 any more.  It might be that something went wrong or you told it to wipe out Windows 98, but if neither of those cases is true, Windows 98 should still be there.

So if you haven't already tried this, remove any disc you have in the CD drive and start up the computer.  If it doesn't boot Windows 98 normally, tell us what happens when you try that.

Assuming Windows 98 is still there and boots okay, what do you want to do next?  

We could figure out what all those error messages were about and continue to see whether we can get Ubuntu working on your computer.  

If Windows 98 is working, but you want to replace Windows 98 with Windows XP, you could go ahead and do that, if you have a Windows XP installation CD.  But the two CDs you describe sound like they aren't Windows XP installation CDs, since you said both of them say Office XP on them.  Office XP installation CDs don't have Windows XP on them -- just the Office programs (Word, Excel, Powerpoint, etc.).  Maybe you just made a typo when you were describing the CDs.  If one of them is a Windows XP installation CD and the other is an Office XP installation CD, then you could use them to install Windows XP then Office XP.  Describe again what the printed labels on the CDs say if you aren't certain what you have.

I'm not sure installing Windows XP on that computer is a good idea.  Windows XP requires a bit more horsepower in the computer in order to run well.  A computer that is nice and responsive running Windows 98, can be disappointingly sluggish when running Windows XP.  So give that some consideration before going ahead to install Windows XP.  XP probably will work; just maybe not as well as you'd like.

Anyway, tell us what you want to do -- continue investigating whether you can install Ubuntu alongside Windows 98, or first replace Windows 98 with Windows XP?
