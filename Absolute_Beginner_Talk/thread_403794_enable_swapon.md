### Post by yabbadabbadont on 2007-04-07
The output of the free command shows that at least one of your swap partitions is active.  Probably sda4 since it gave you an error when you tried to manually activate it.  I would suggest that you edit your /etc/fstab and make sure that both partitions are listed "the old way".  The two entries would look like this:
```
/dev/sda4 none swap sw 0 0
/dev/sda5 none swap sw 0 0
```
Remove the existing swap entry that uses the UUID style.  Then run, "sudo swapoff -a", and finally, "sudo swapon -a".  Then run the "free" command again.  You should see all of your swap space available.
