### Post by Tomosaur on 2006-08-13
If you're confident you don't want to dual boot, you can always just use the LiveCD for a while. I would say linux is by far the superior programming environment, although if you're a proponent of Visual Studio, you may find the transition a little more difficult - since there's nothing that compares to it on Linux.

Gaming is a bit tricker. There are a bunch of games for linux which are free to download, but, obviously, few are of the quality of the games Windows users expect. There are a bunch of pretty good ones though, many are available through Synaptic, others you can find on the web.

Reading from NTFS is fine, no troubles there. Writing to NTFS is still experimental, but there are things you can do to make it a bit more reliable. I still wouldn't recommend it though.

As for IDEs and editors, there's a whole bunch available through Synaptic. For things like HTML you don't really need anything other than GEdit (in fact, it has syntax highlighting for a whole load of languages, so you may only ever need that depending on your expertise).

I say go for Ubuntu - more free stuff, better performance, nicer community, easier to customise.

Okay... I misspoke... while it is technically possible to have a FAT32 partition over 32 GB, the way the file format works, it becomes extrememly inefficient and slow to  do so.  (if i understand it it has to do with how the drive has to be accessed on very distant segments of the drive successively and repeatedly)  If partitioning from Windows, the size is limited to a maximum of 32 GB.  Some linux partitioners will probably allow larger sizes, but a 250 gig FAT32 partition would certainly not be recommended.  All this is a moot point anyway, however, because if the OP is willing to backup and reformat the drive, he may as well format ext3 and can forget about NTFS and windows completely.
