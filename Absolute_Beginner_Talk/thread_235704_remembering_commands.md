### Post by kebes on 2006-08-13
I agree that it is easier to remember commands when you think about what they stand for. I don't know of a good place that lists the origin of all the command names. However Wikipedia is a good resource, since it will list not only how to use the command, but a bit about the history of the command. This will give you some idea of why it is named that way.

Not all of the names are abreviations, of course. Some are very obscure references.

In some cases you can just make up your own explanation. For instance, "sudo" actually comes from the command "su" which basically means "switch user". So "su bob" changes you into user bob. "su" without specifying a user switches into the root user, so you could think of it standing for "superuser" also. So "sudo" really means "switch user and do something" but when used without the -u option you can think of it as "superuser do".
