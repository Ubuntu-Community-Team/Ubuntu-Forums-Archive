### Post by netrattler on 2005-05-25
At the top click on System -> Administration -> Synaptic Package Manager
Once it comes up click on Search -> Change look in: from name to description and name. Type in xfce4 and click the search button. Scroll down till you see xfce4 and select it. You might also want to install some of the other things that go with it. Just look down and highlight each of the things in the search you have just done and read the descriptions to see if you want it or not. Then once you done making your selections just click the apply button and it will download and install everything that you need. Then just logout and when your back at your login screen, under the username field you will see session click on that and select xfce then type your username and password in and hit the enter key and it will ask you if you want to make this the default, you might not want to make it your default choice at the moment till you try it out and see if you like it or not. But even if you do make it your default all you have to do to switch back to gnome is click on the session again at the login screen and switch it back to gnome. But I would definitely try the prelink first and see if that speeds it up enough for you first. Gnome is a lot more friendly than XFCE4 is.

Joe

### Post by poofyhairguy on 2005-05-26
Easy way to fix it: buy 128 mb more RAM. its cheap (think of it as the price of the OS) and things will run a lot smoother with little hassle.

Harder way to fix it: install XFCE4.2 to do this you need an internet connection, time, and patience.

Out of the two, I constantly recommend buying the RAM on this forum. I have a computer JUST LIKE THAT that I gave to a friend with 256mb of RAM and Ubuntu runs like a charm.

Also you could try Kubuntu. It uses less RAM.

EDIT: please note that unlike Windows having extra programs installed does not slow down the system. This problem is a lack of RAM, pure and simple.
