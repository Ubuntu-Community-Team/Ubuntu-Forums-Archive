### Post by ricrac on 2007-05-18
Documentation.

Pick something you're interested in and write a howto tutorial for it.
Learn how to install a source package.
Learn how to setup a development environment, retrieve the lastest source code and compile the source without making any new changes.
Learn how to package the application for apt.
Learn how to submit bug reports.
Learn how to debug.
Learn to use a test framework.

After you have finished the howto submit portions of the howto as documentation enhancements for the project. 

You'll be providing something valuable while learning the ropes.  Once all of this is second hat, look for apps using your skills, html, java, python, look over bug lists and wishlists and find some small fixes you can implement.  Persons with virtually no skills can provide spelling, grammar and syntax corrections on helpfiles and within application dialogs.

Find a bug and follow it.  
Find a bug in an application where you can understand the bug.  You're looking for a small app or portion of code you can wrap your head around and grasp easily.
Attempt to fix the bug (don't bother submitting code unless you're positive)
Follow the bug and check your fix against the eventual implemented bugfix.

For pure learning, look at old bugs and bugfixes.  Learn what the common errors are, how they manifest themselves, and the typically corrections taken.

Don't forget to have fun!
