### Post by Saoshyant on 2007-12-02
I don't know if this is just an issue with Ubuntu's KDE, or with all KDE environments.  When I do a "Copy", CTRL + C, whatever, the content is added to a clipboard.  However, as soon as I close the application I was working on, the content in the clipboard is gone.

Imagine, I copy the URI from the address bar in the browser, I close the browser, and when I go to the other browser to paste the URI there, it's gone.  The clipboard is empty.

I hope this is optional, because to me it's one of the most annoying things about KDE, so does anyone know how to disable this behavior?  As in, make the clipboard "stick"?

### Post by jken146 on 2007-12-02
I don't think there is a 'universal clipboard' like in Windows.  You need  to keep the application open when you paste.  This is true in Gnome too.

---

### Post by AndrewNi on 2007-12-02
If you have Klipper running in the tray, you can configure it to prevent an empty clipboard. This will stop programs clearing it out when they are closed.
