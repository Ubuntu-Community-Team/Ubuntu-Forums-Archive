### Post by Denis on 2006-06-12
Ok, this a a more detailed description of what to do. 

You said you need a package for the pcmcia support on your laptop, but the problem is that you can't download it from the repository because you have no internet connection. 

I assume you are using **Synaptic Package Manager** to install this package. (Synaptic is just a bit easier than "apt-get" in the terminal, so I'm using this). Do the following with Synaptic: 

[list]
[*]You have to search for the package "pcmcia-cs", which was removed as you said. 
[*]When you found the package, mark it for installation, just like you would do normally. 
[*]Now, you are not going to press "Apply" (to download the package from the repositiry), but you generate a list of the selected packages. Select "File" in the menu and chose "Generate package download script". You get a dialog where you enter a name for the script. 
[*]You can close Synaptic now
[*]Open this script you just made, with a text editor (like Gedit). To do this: just double click it and chose "Display"
It will look like this 
[/LIST]

```

#!/bin/sh
wget http://archive.ubuntu.com/ubuntu/pool/main/p/pcmcia-cs/pcmcia-cs_3.2.8-5.2ubuntu6_i386.deb

```

It means that "http://archive.ubuntu.com/ubuntu/pool/main/p/pcmcia-cs/pcmcia-cs_3.2.8-5.2ubuntu6_i386.deb" is the URL to the package we want.

Normally this URL will be the same for you. I just explained this procedure so you can also download additional packages in case that would be neccesary. 

[LIST]
[*]Write down this internet adress (or save it somewhere you can see it). 
[*]Now you need a **computer with internet acces** to download this file. Maybe you have a dualboot with windows on the laptop. Use a webbrowser to download [http://archive.ubuntu.com/ubuntu/pool/main/p/pcmcia-cs/pcmcia-cs_3.2.8-5.2ubuntu6_i386.deb](http://archive.ubuntu.com/ubuntu/pool/main/p/pcmcia-cs/pcmcia-cs_3.2.8-5.2ubuntu6_i386.deb). Save the downloaded file to a CD, USB stick or whatever. 
[*]Start Ubuntu and copy this file (from the CD, USB-stick) to your "Home" directory let's say. 
[*]You can install .deb packages for your Ubuntu system with either the console or a graphical program. 
[LIST]
[*]in the console type: sudo dpkg -i pcmcia-cs_3.2.8-5.2ubuntu6_i386.deb
[*]OR graphical: start GDebi Package installer, Open the package and click "Install Package"
[/LIST]
[/LIST]

Now the package is succesfully installed. What you've done is just the same as you would with synaptic, only now you downloaded the package with another computer (or system). Open synaptic and you will see that "pcmcia-cs" is installed. If this is the package that was missing, you problems should be gone. 

I hope this works, I tried to explain in an understandable manner :)
If you do need any more help, just ask it.

### Post by Denis on 2006-06-20
Since this happened to both of you, it seems like a bug in XUbuntu. 

I've looked it up at Launchpad (that's were all the bugs can be filed) and I've found that this bug is already reported: 

You can find the bug report over here: [https://launchpad.net/distros/ubuntu/+source/pcmciautils/+bug/42662](https://launchpad.net/distros/ubuntu/+source/pcmciautils/+bug/42662)

> Ad the end of the installation process, the espresso installer deletes packages that will be of no use on the installed system. But when I installed Xubuntu beta2 on my Laptop, the package pcmcia-cs was also removed. As a consequence, my pcmcia-WLAN-card did not work after reboot until I reinstalled the package, and the networking was not configured properly.

It's not unnecessary to report it twice, but **you could help by posting a comment on the report!** You have to log in to do that (the registration just requires an email adress and password).

Specific details about what happened, what hardware you are using, etc. can help the developers to solve the problem.
