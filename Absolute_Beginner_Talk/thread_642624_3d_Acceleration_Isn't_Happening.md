### Post by jw5801 on 2007-12-18
Envy is as bad as Automatix, there's a very likely chance of it breaking something! Hmm... xorg.conf appears to all be good. Are you by any chance running Xgl as well? The way gutsy is currently set up, it runs a second X session for Xgl. So on display 0 is the Xorg session, and on display 1 is the Xgl session. You may be experiencing the same issue I have (which I am yet to resolve), where the Xorg session uses the generic mesa driver and the Xgl session uses the restricted ATi driver (fglrx). Try running ```
fglrxinfo -display :0
fglrxinfo -display :1
```and see if you get the same output on both. Mine looks like the following:```
jw5801@laptop:~$ fglrxinfo -display :0
display: :0.0  screen: 0
OpenGL vendor string: Mesa project: www.mesa3d.org
OpenGL renderer string: Mesa GLX Indirect
OpenGL version string: 1.4 (2.1 Mesa 7.0.1)

jw5801@laptop:~$ fglrxinfo -display :1
display: :1.0  screen: 0
OpenGL vendor string: ATI Technologies Inc.
OpenGL renderer string: ATI Radeon Xpress Series
OpenGL version string: 1.2 (2.0.6473 (8.37.6))
```

This only happens to me in gnome (which is really irritating), so I often just switch to kde or fluxbox if I'm going for an extended gaming period. An annoying workaround, I know, but I'm yet to find a solution (short of a reinstall) for the problem.

### Post by jw5801 on 2007-12-18
> **WierdKid said:**
> What do I need to change to disable Composite?

Edit xorg.conf (gksu gedit /etc/X11/xorg.conf) so that the section Extensions reads as follows:
```
Section "Extensions"
        Option          "Composite"     "Disable"
EndSection
```
