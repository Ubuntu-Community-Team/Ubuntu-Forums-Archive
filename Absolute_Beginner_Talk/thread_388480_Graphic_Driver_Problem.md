### Post by freebird54 on 2007-03-19
I suspect that the ati-agp module is loading.  Did you blacklist it yet?  Here are the fixes I used to get rid of mesa - hope it applies to your situation...

Run the following two commands in Applications->Accessories->Terminal:

```
sudo mkdir -p /usr/X11R6/lib/modules/dri
sudo ln -s /usr/lib/dri/fglrx_dri.so /usr/X11R6/lib/modules/dri 

```

and then blacklist ati-agp in /etc/modprobe.d/blacklist

```
gksudo gedit /etc/modprobe.d/blacklist

```
and enter the following

```
blacklist ati-agp
```

Save, exit and reboot.  Subsequent restarts should not cause any changes..

### Post by freebird54 on 2007-03-19
Choose recovery mode if you aren't at a prompt already.  Try

```
nano /etc/X11/xorg.conf
```

nano is a text mode editor that at least tells you how to save and exit at the bottom of the screen!

Alternatively, use 

```
ls /etc/X11
```

to determine the name of your latest backup, and
```

cp /etc/X11/"nameofbackup" /etc/X11/xorg.conf
```

replacing "nameofbackup" with the actual name of your backup :D

If this is not enough here's another way. Try re-creating the file with

```
dpkg-reconfigure xserver-xorg
```

Answer any questions you know the answer to, and let the defaults rule all else.  Use <tab> to get the OK highlighted, and use <space> to select your resolutions...

Somewhere in there we should have you no worse than where you started!
