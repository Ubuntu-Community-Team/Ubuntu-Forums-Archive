### Post by Terl on 2007-05-23
I understand what you mean about the nasty worm.  I got one too which made me switch but, I did go ahead and reinstall windows and then I installed Ubuntu.  This is so I had a fallback, just in case I had software I needed that I did not have a replacement for in Linux.  You may want to consider that as an option.

Now for the Linux choices.  

All the *buntu flavors are essentially similar (ubuntu uses gnome, kubuntu uses kde, edubuntu has some added software) and are debian based.  The debian pakage system is very good and I find it easy to use.  The *buntu flavors are good distros for a newcomer to Linux.  

Fedora is red hat based (package system is rpm).  Fedora is not difficult but does require a bit more configuration of text files to get your repositories up and running and it can be daunting as incorrect entries can cause conflict with updates.

Slackware is a bit more difficult but is very rewarding to install and configure.  There is no dependency checking when you download slackware packages so you could wind up in what they call dependency hell.  There are tools available that can help with that now, swaret is one.  

If you are new to Linux I would recommend you go with an Ubuntu flavor.  If your system specs are low you may want to not go with Kubuntu.  In any of the cases though, pop the cd in and run it live...see if it works and test drive it a bit.
