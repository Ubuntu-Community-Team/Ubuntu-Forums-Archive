### Post by s26c.sayan on 2007-02-07
```
gpg --keyserver hkp://subkeys.pgp.net --recv-keys KEY
gpg --export --armor KEY | sudo apt-key add -
```

Run this code twice, replacing KEY with  49A120FD1135D466 and 033431536A423791 respectively. I think this will solve your problems.

### Post by s26c.sayan on 2007-02-07
GPG stands for GNU Privacy Guard. This is basically about digital signature (or even encryption). To digitally sign something( a document, email or even a repository as is the case here), one needs to posses a set of two keys : private and public keys!! The repository owner can sign his repos using his own **private** key. The user has then to retrieve the **public **key that the owner has published (most probably in a **keyserver**) so as to verify the signature. This retrieving of public key from keyserver and storing in the user's system is what the above code does.
Details can be found  [here]("http://www.youdzone.com/signature.html") (about digital signature) and [here]("http://en.wikipedia.org/wiki/Gpg") (about GPG)
