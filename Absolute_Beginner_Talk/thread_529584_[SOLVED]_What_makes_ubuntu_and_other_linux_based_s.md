### Post by Luinar on 2007-08-19
On top of what has already been mentioned, there is the fact that the average Linux user is likely to be more computer literate/techy minded than the average Windows user, meaning they are going to be more likely to be able to spot and/or defend themselves against viruses. So even if there were viruses "in the wild" for Linux, people would have a better idea of how to identify/defend against them. And owing to the nature of open source, it would only be a matter of time before someone made available a means of fixing whatever it was the virus was exploiting. :)

### Post by dasunst3r on 2007-08-19
To weigh in on this, I am also baffled at why there is not yet malware for Linux taking into consideration that Linux runs so many mission-critical applications.  I think it is because:
[list=1]
[*]Open-source code can be scrutinized by anybody, and submitted patches must go upstream.
[*]Energy is redirected to doing good instead of doing bad.
[*]Since the beginning, Linux has been built for a networked environment where there are going to be multiple users on one machine.
[/list]
Granted, there have been successful break-ins to Linux machines, so don't get cocky.  Keep your machine updated and you'll be fine.
