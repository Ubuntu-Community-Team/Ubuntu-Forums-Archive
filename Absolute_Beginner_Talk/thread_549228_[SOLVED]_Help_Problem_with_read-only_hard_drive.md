### Post by Beggar on 2007-09-12
Alright, try editing your /etc/fstab

There should be an entry for each of your drives. First find the uuid of your drive, go Computer -> highlight the drive you want rw for ->right click properties -> volume tab.

Now modify the /etc/fstab entry so that it looks like this one,

```

sudo gedit /etc/fstab

....


UUID=<whatever the uid you looked up was> ntfs-3g defaults,uid="YourUser",gid="YourGroup" 0 0

```

I.E.

For example, my drives uuid = 9CE44303E442DEE0. So I would open up fstab, find the entry for UUID=9CE44303E442DEE0 and thats the one I want to change. If one doesnt exist already, just  add it to the end of the file.

FYI my entry looks like:
UUID=9CE44303E442DEE0 /media/Darrenh01 ntfs-3g defaults,uid="darren",gid="darren" 0 0

### Post by Beggar on 2007-09-12
Sorry oversight on my part, 

You want to change the entries for /dev/sdb1 and /dev/sda1.

The way this is set up however (using the location, instead of UUID) if you replace the drive its going to automatically use whatever rule that you add for this hdd to mount the next one, IE mount it as ntfs, even if it is a differnet filesystem. Thats why it is usually done by UUID ( A unique hardware identifier), not by where the drive is being mounted from, and I would recommend that you set it up that way as well.
