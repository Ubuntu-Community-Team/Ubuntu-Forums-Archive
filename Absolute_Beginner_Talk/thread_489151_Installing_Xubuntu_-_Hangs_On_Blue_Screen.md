### Post by Perry64 on 2007-07-01
I am new to Xubuntu and Linux, although I have used Unix many many years ago.  I am having problems installing Xubuntu on a very old computer (PIII 500MHZ, 128 MB).  I have downloaded the .iso for 7.04, verified the MD5 hash is correct, and burned it to a CD.  

When I start the computer with the CD in the drive, it takes me to a screen giving me several options for Xubuntu.  Through these options, I have verified that the CD is not corrupted and tested the memory.  When I select load Xubuntu, the startup process goes through a screen saying "Xubuntu" and a process bar.  When the process bar is completed (about 4 minutes), the screen is black for a few minutes, and then I get a light blue screen which looks like the beginning of a desktop.  I get an arrow that moves with the mouse, and that is it.  No icons, no menus, nothing else.  It remains this way for at least twenty or more minutes.  It doesn't react to the clicking or double clicking of either mouse button, and random typing has no effect.  This behavior has repeated over several attempts.

Does this sound like a common error with a known fix?  Does the fact that the CD passes the check mean that it is good (as I am assuming), or should I burn another one?  Should I go back further and redownload the .iso, even though it passed the hash test?

Any help on this would be greatly appreciated.

Hi and welcome, I have install xubuntu several times and it sounds like it is a memory issue. I quote the web site
Minimum system requirements
To run the Desktop CD (LiveCD + Install CD), you need 128 MB RAM to run or 192 MB RAM to install. The Alternate Install CD only required you to have 64 MB RAM.

If you have  on board graphics then your memory is shared and thus reduces the amount of memory needed for the install. I would try the alternate cd as it has worked for me in the past.
Good luck and welcome again!
