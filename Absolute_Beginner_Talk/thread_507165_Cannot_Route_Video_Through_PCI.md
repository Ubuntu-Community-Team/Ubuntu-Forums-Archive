### Post by w4ett on 2007-07-22
One thing I forgot to mention was that I had to modify my xorg.conf as follows to force it into PCI mode:

```
Section "Device"
 Identifier "ATI Radeon 9200SE"
 Driver "ati"
 BusID "PCI:1:0:0"
 VideoRam 65536
 Option "UseFBDev" "true"
        Option "BusType" "PCI"
EndSection
```

### Post by tormod on 2007-07-29
Please open a new bug report against the kernel. Try to isolate the part of the boot process that fails. You haven't mentioned which version of Ubuntu you are using, but if it crashes just after "Loading hardware drivers" in Feisty, it is probably during the execution of the /etc/rcS.d scripts, after S15module-init-tools which contains this message.

To verify this boot with the option "init=/bin/bash". This will stop the booting just before the rc scripts. If you get a root shell everything is fine at this point, the initial booting in the initrd went well.

Edit /etc/default/rcS and set "VERBOSE=yes". Now boot again, there will be more messages. You can disable individual startup scripts by renaming them so they don't start with S, for instance "mv S15module-init-tools nS15module-init-tools". Expect some things go wrong if scripts are disabled. Reenable them by taking away the "n" again.

Of course, always boot without "quiet" and "splash" (this is default for Recovery mode anyway).

Attach the output of lspci and dmidecode to the bug report.

### Post by Githlar on 2007-11-21
I've finally resolved this problem! I got rid of my old ATI Radeon 9250 128M PCI and got an nVidia GeForce 6200 256M PCI card. I popped it in and BAM! Same problem. I ran across an article about a completely different computer and nVidia video card. However, this person also had Intel onboard graphics. The solution provided was this:

```
sudo gedit /etc/modprobe.d/blacklist
```
Add these lines to the end of the file:
```
blacklist agpgart
blacklist intel_agp
```
