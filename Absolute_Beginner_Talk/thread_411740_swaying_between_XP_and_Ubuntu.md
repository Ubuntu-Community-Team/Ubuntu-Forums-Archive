### Post by LaRoza on 2007-04-17
I am nineteen years old and installed Ubuntu over Vista just this month.

The "code" you "have" to learn are commands and you'll only use the same ones to do tasks that are easier in the command line.

Installing new programs is so much easier in the command terminal.

For example, to install something that is dependent on other programs in the "usual" way would be frustrating, finding all the programs can be a pain. In the command line, it is one line like "sudo apt-get install build-essential"

There is also a graphical way of doing the same thing. The book "Moving to Ubuntu" which comes with an Ubuntu DVD for a quick and painless entry.

IF you do not want to install until you get the hang of it, get a live version, such as Ubuntu Live, Slax, or Knoppix. The Ubuntu DVD can be run live without installing.

If you have any problems, use the forum. You'll get a quick response to any problem you might have.

### Post by Tundro Walker on 2007-04-17
As a former WinXP user who switched to Ubuntu Linux, I think the biggest culture-shock was using the command-line more.  In Linux, the command-line is considered the primary interface, the GUI the secondary, which is opposite of MS Windows.  But, it's because the Linux command-line is actually robust, powerful and useful.  You can actually get things done quicker in it once you get use to chaining commands, piping feedback, etc, there-by removing the GUI's overhead from the overall process.

However, Ubuntu's goal is to eventually get to the point where people can use the GUI for everything, which would make the migration from Windows a lot easier.  But, it's not quite there yet.  When I first installed some apps, some icons didn't show up in my GUI menu, so I had to dig around in the command-line to figure out how to run them.   Sometimes, when an icon did show up, I'd click it and nothing happened.  I'd have to go to the command-line and run it, and see the error message it would dump out failing the process.  I still get annoyed by this, because sometimes there's just no GUI feedback when something running in the terminal back-end fails.  In other words, Ubuntu may be trying to focus more on GUI use, but other programs and developers may still be command-line oriented, and neglect GUI error feedback or installation.  The fact that there's different GUI's, menu systems, window manager systems, etc can of course make it a pain to implement a program to take GUI's into account, so I can understand.  The programmer would have to add in all kinds of contingencies to  get it to adjust for whatever setup the user has (not talking about hardware here, but GUI, desktop, etc)

I started surfing for support with some issues I was having, and was annoyed at having to do some things in command-line / terminal, or how I had to tweak some .config text file somewhere.  I was also kinda annoyed that I couldn't pick/choose where to install certain things.  But, after I started learning some command-line stuff, I realized it is useful, and I kinda like being able to know exactly what's going on.  And with some scripting, it's faster then the GUI, because you remove that overhead from the operation.

Does that mean I use the command-line for everything?  Heck no!  I use the GUI as much as possible.  But if I have a common set of tasks to run regularly, I do take the time to set up a command-line script that I can single-click and run.

I think your best option would be to dual-boot with Windows and Ubuntu, so you still have your Windows gaming platform, but also have the Linux platform to get familiar with.  Linux is a useful OS to learn, since it familiarizes you with Unix-like systems, how they work, command operations, etc.  And, I could easily fore-see more companies using Ubuntu or Ubuntu-like Linux distro's in the future instead of Windows in order to save cost on their desktop or thin-client systems.  If you expand your knowledge base now, you'll be ahead of the game.  And, you never know, it may launch you into directions you never thought about going, like setting up a web-server, or programming, etc, because Linux makes these implementing these types of things more accessible.
