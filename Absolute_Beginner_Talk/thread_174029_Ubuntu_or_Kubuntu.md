### Post by aggiechemist on 2006-05-11
As for your request for making the transition easier, I think you are right on target with live CDs. You should be able to try out both versions (Ubuntu/Kubuntu) and get a feel. You should be able to try out all your hardware and see if there is anything that doesn't just work for you. You should be able to make sure you have programs to handle all of your files. ### Post by catlett on 2006-05-11
Just a note. It seems people think Linux was made to be an alternative to windows, it was not. It has emerged as an alternative for it. I say this because it is not designed with a windows user in mind. It was not made to lure windows people over. There was no effort to make it like windows in commands or operation. So don't expect to use your windows knowledge and run linux. And don't get mad at linux when it doesn't act like windows.
I do not know alot about the history but I do know that Linus Torvald was unhappy with the variation of Unix he was using as a student. He set out to make an alternative to Unix (not windows, Unix).
Linux acts and looks like Unix. That is the operating system this system is modeled after. So be prepared to learn a NEW operating system when you migrate from windows.
Someone recently posted a good analogy about windows and linux. He said windows was like a car with automatic transmission and linux was like a car with manual transmission. I think this is a good analogy. Windows is very easy. You don't have to understand how it works but you are confined to its parameters. Linux is harder to learn and use but once mastered offers much more control and fun! Good luck.
And remember if you are going to run a business on it there is paid professional support for free software, which isn't a bad idea when your livelihood depends on your system running correctly. Ubuntu is owned by Canonical of South Africa. They specialise in professional suport for free software. (mainly Ubuntu) [http://www.canonical.com/support](http://www.canonical.com/support)

### Post by AndyCooll on 2006-05-11
THe difference between Ubuntu and Kubuntu is the desktop environment (i.e. the layout of the desktop etc).

Ubuntu uses the GNOME desktop environment, Kubuntu uses KDE. As has been mentioned here, the best one is the one suits your needs. Both have strengths and weaknesses. Each desktop environment also comes with their own suite of applications

I'd suggest starting off with Ubuntu, since that is the default, and as Aysiu says  you can always try Kubuntu once you've gained a little familiarity. You can actually have both desktops installed at once (and others such Xfce, Enlightenment etc). Then, before you login choose your session. To install Kubuntu copy and paste this into a terminal:

```
sudo apt-get install kubuntu-desktop
```

It's useful to know too that you can use KDE apps in a GNOME desktop environment and vice-versa. I personally use GNOME, but many of my preferred apps belong to the KDE suite (K3b, Apollon, Amarok, Krita, Kooka, Gwenview).

### Post by 3rdalbum on 2006-05-12
I'll be the first answerer of the second question, whoo hoo!

Tips for installation:
1. Use the Live CD first. Good to see that you're doing that anyway. Gain a little familiarity with the system first before taking the plunge.
2. When you get to the desktop in the Live CD, make a copy of the following file: /etc/X11/xorg.conf. Print it out (in Ubuntu or Windows). This way, if the installation disc fails to set up your video card properly, you can edit that file to what it should be.
3. If you use the Expert mode in the install (for any reason), make sure you DON'T specify a root password. That's right, leave it blank.
4. When you use the guided partitioner, make absolutely sure that it's going to save your existing Windows partition (if you want it saved, of course)
5. After installing, when you next boot back into Windows, it'll check the disk. Don't worry, it's normal.
6. Don't unplug your internet connection when Ubuntu starts installing its packages. If you do accidentally, and Ubuntu tells you that it didn't complete the package transfer, log into the text terminal that it provides to you, and type:
```

sudo apt-get update
sudo apt-get install ubuntu-desktop

```
and that will continue the install. When it's finished, type **startx**.

That's all the installation tips I can remember.
