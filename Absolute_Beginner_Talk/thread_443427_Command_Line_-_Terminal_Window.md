> **6Realms said:**
> I've been coming across these 2 terms  a lot  lately:  "Command  Line" & "Terminal  [& or] Terminal Window" and wondered what their connection are?  

I took this from wikipedia, but it should help you understand better.  [http://en.wikipedia.org/wiki/Command_line_interface](http://en.wikipedia.org/wiki/Command_line_interface)

"A Command Line Interface or CLI is a method of interacting with an operating system using a command line interpreter. This command line interpreter may be a text terminal, terminal emulator, or remote shell client such as PuTTY.

The concept of the CLI originated when teletype machines (TTY) were connected to computers in the 1950s, and offered results on demand, compared to 'batch' oriented mechanical punch card input technology. Dedicated text-based CRT terminals followed, with faster interaction and more information visible at one time, then graphical terminals enriched the visual display of information. Currently personal computers encapsulate both functions in software.

The CLI continues to coevolve with graphical user interfaces (GUIs) like those provided by Microsoft Windows, Mac OS and the X Window System. In some applications, such as MATLAB, a CLI is integrated with the GUI, with the benefits of both."

A terminal window refers to the window of the terminal application that you are using.  If you're using Ubuntu, it is most likely the "Terminal" application, which can be found under Applications -> Accessories -> Terminal

### Post by esoterica on 2007-05-14
Your posts end up several pages back very quickly in this forum, things seem to move crazy fast in here so a new thread can end up a couple pages back in no time at all, like minutes I've seen.

For all practical purposes for you, just consider a thread or post as the same thing to simplify and not confuse you, no harm will come from you confusing the two, but you could sort of think of a new thread as a new email message, a new post as a reply to an existing email message.

Likely

For all practical purposes you can also think of the terminology "Command Line", "Terminal Window" and even sometimes you may see it referred to as a "shell" as the same exact thing. Their are subtle specific differences if you want to be a terminology purest, how ever for the most part all are usually loosely used to refer to the exact same thing. I think the difference in the exact word used depends on what the primary Operating System the person using the different terms is use to using.

For example, Windows users will tend to most often refer to it as the command line (or DOS Prompt even), old school Unix type users who are use to working on remote servers will often call it the shell or a shell account, Ubuntu users so far I've seen most often refer to it as the Terminal Window.

For Ubuntu "Terminal Window" is probably the more exact term to use since this is what it's actually named under your default applications list.

The need to use the "Terminal Window" for Linux has been a long standing hold up in the acceptance and use of Linux by the average human being since it involves the use or need to know specific command names you have to type in with confusing looking switches added to the command. Sometimes you have to run commands from a specific directory, sometimes you can run them from anywhere.

Linux developers have worked a long time now to get things so the average user could do everything from a GUI (Graphical User Interface), or what say a Windows user is use to working in where you just click everything with your mouse and not have to type commands in.

I haven't used Linux for a few years now because it was a bit of a mess still a few years ago and more often than not the GUI utilities didn't work correctly and you had to still manually edit config files and such, or run commands from the command line to actually get them to work. Trying to figure out what needed to be done where like this was just a time wasting hassle, just like having to type out a lengthy command to just do something simple that a single mouse click would take care of under other Operating Systems often discouraged people from further Linux pursuit.

It looks like over the past couple years some progress has certainly been made, more of the GUI apps actually work now, installation has become a whole lot easier than Linux use to be and BSD can still be, updates and software installations have become a lot more average user friendly as well, which had also always been a strong deterrant for many who became instantly lost as soon as it was mentioned to download source files, unpack them, run make to configure etc... Windows users were spoiled by just downloading a single file and clicking on it to install and be done, not to mention how easy updates were to find and install under Windows. The Linux community fell far behind because of this interest deterring problem for the average user and couldn't win the love over from people who were already use to easier and faster ways of doing things that just worked and didn't have to be beaten into submission in order to get it working.

I think it was Red Hat that actually opened the ability for Linux to easily be updated and installed with the introduction of .rpm files which others have since followed in the foot steps of. Though with this simplicity we also suddenly began to notice a rise in Linux based vulnerabilities, a problem which use to only haunt Windows users. It's a trade off, but if it attracts more people to Linux then it's all good, the more people using it the higher the demand will be for better quality, functionality and ease of use.

Microsoft probably would never admit it, but shot themselves in the foot when they locked down XP and made it difficult for the average person to install their copy of Windows on more than one home computer they own. The legal warnings about doing this were always there, but people ignored it and installed a single copy of the OS on all of their home computers anyhow. Locking that ability down and forcing the average home user to purchase multiple copies of the same thing for each of their home computers I think was a huge mistake and we'll see now as more people start to obtain new computers a new interest in Linux due to this. Had Microsoft of been smarter and less greedy about the whole thing they'd adjust their licensing to allow the average Home User to install say 3 or so copies of the OS on their Home Computers and people would swallow it a lot easier.

So, due to all of this, expect to see many old Windows users now moving into Linux as a more affordable option and hence the term "Command Line" probably more frequently used as a result of this.

More Windows users however converting over, or at least trying to convert over can have it's own problems as well. These are people who are spoiled by things just working and not having to be messed with. They are also use to paying money for that luxury, so with this we will certainly begin to see less free good quality software available and more people charging for it, which the Windows converts will gladly pay just to have something that works. There's not anything wrong with that I suppose, just hopefully those charging learn the lessons from Microsofts mistakes and make it sensible for the average Home User to use the paid software on their home systems without the mess of individual licensing greed.

Old school Linux purest are going to choke on that concept, understandably, but this is what happens when you drag your heels about identifying the hold backs and coming up with fresh ideas to solve them in winning people over with more to dangle in front of them than just the free alternative. It use to be the lure of free and more secure used to attract people, the more secure concept has since been lost along the way with Linux vulnerabilities now being about as common as Windows vulnerabilities. For Linux developers to keep up they need to work on user simplicity and improve the look and feel, yes "pretty" does have an importance to many people.
