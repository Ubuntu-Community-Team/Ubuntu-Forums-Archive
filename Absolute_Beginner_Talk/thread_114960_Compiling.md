### Post by ekarni on 2006-01-09
To expand on Jimmey's reply, programs are written in any one of hundreds of programming languages.  Most bear some resemblance to English -- good for people, lousy for computers.  Compiling is the act of taking the human-readable source code and turning it into a binary file that the computer can use.

Compiling is done with (surprise) a compiler.  This is nothing more than a program that processes the source code and generates the executable file.  The one you're most likely to encounter on Linux is gcc, the [i]GNU[i] C Compiler (C is the name of a very common programming language).

For completeness, let me also point out that some languages don't require compiling.  The computer turns the source code into something it can use as it works its way through the program.  This is called "interpreting."
