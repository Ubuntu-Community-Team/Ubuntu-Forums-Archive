### Post by Tatty on 2008-04-06
It should autodetect most of the components in your computer and most drivers will be built into the kernel.

There may be some things that you have to manually install drivers for, in which case the solution to this is usually just a google or ubuntuforums search away.

Sometimes the drivers will be available to you from the repositories, but are not enabled by default for legal and/or ethical reasons.  So its worth checking out the "restricted drivers" dialog in System->Administration->Restricted Drivers.

However, whilst you should probably be able to get everything working, there are no guarentees for this.  So if you want to make absolutely certain then it would be worth doing some research about the particular hardware you are buying.

But from personal experience, i tend to buy the hardware I want and then make it work. I have yet to find any hardware which i could not get working.

### Post by JoshuaRL on 2008-04-06
I would suggest you look for a Nvidia graphics card, and if you want wireless stay away from Broadcom chipsets.  Most everything else is either automagically detected and installed, or just a Restricted Drivers click away like Tatty says.

But the suggestion to search on the hardware is a great one.  That saved me from using an onboard unsupported card on my new desktop.  If you have the exact make and model of hardware, search google with:
```

site:ubuntuforums.org [search string]

```
without the "search string" and brackets.  That should bring up whatever is on the Forum.
