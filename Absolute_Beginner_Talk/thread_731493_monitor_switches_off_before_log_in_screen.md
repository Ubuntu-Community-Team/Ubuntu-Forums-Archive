Go through that and accept all the defaults and then reboot normally.

### Post by Rocket2DMn on 2008-03-24
OK, I'm still unclear on your situation.  Are you using an INTEGRATED video card on the motherboard without an AGP/PCIe card?  If so, run
```
sudo dpkg-reconfigure xserver-xorg -phigh
``` to autodetect your configuration.  No other video cards should be connected into expansion slots.

If you have an expansion slot video card alongside an integrated card, disable the integrated card from the BIOS, then reconfigure X to use your expansion card.

If you ONLY have an expansion slot card, reconfigure X, either with -phigh or as I mentioned in my previous post.  This will need the "fglrx" driver for your ATI card, but you should be able to get started with the "ati" driver, then enable fglrx from within the GUI.

An INTEGRATED card is most likely an "intel" card, like your xorg.conf says - use the autoconfiguration I just mentioned.

Try the correct method I mentioned for your setup, then get back to me on which setup you truly have and if it was successful.

### Post by Rocket2DMn on 2008-03-25
Will your radeon card fit into this motherboard?  If so, turn off the computer, unplug the power, add the card, plug the power back in and turn it on.
Immediately go to the BIOS and disable the integrated card so that only the AGP card works.  Then let it proceed with bootup and reconfigure X again.  Note: if you are using the recovery kernel, "sudo" is not necessary since you are already at a root terminal.
You can check to see if the card is detected in hardware by running (again)
```
lspci | grep VGA
```
It is possible that the intel card does not support the full resolution that you normally like on your monitor, so that can cause problems, too.  If you have to try again with the intel card, don't select anything larger than 1024x768 for now.
