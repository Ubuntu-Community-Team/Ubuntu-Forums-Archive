Configure is a script that the package maker often includes to examine your system and determine if the package is installable. If it exists, it's purpose is to give you some feedback as to what you may need to install for the package to work (like developement files, dependancies, etc.) Sometimes the file is given a different name by the person who packages, but they will tell you that in the readme file. Sometimes there is no configure script, because the package already includes everything it needs to install. the ./ before configure just changes to the top directory so the script can scan your filesystem.
The first thing you should do after you untar the package is look in the folder where you untared it and find a read.txt or a readme.txt or something like that to get the installation instructions.

### Post by tonyr1988 on 2007-01-19
Do this:

> tar xvzf whatever.tar.gz
cd whatever
ls -la

And copy / paste the output into a reply.

And, like dave said, you may not need to configure a make file. You can always try skipping the configure.

You also may want to do this first:

> sudo aptitude install checkinstall

Do everything else like normal, except instead of "make install" do "checkinstall". The biggest difference is that if you use "make install" and decide you want to uninstall it later, you need to keep the source (the .tar.gz file). If you do "checkinstall" - it will add the program into Synaptic, so all you need to do is uncheck it like a "regular" program.
