### Post by asmiller-ke6seh on 2007-12-06
For the career-minded, you should focus, intially, on understanding systems and program IN GENERAL because WHICH languages you use will change over time. Yes, there are still jobs for COBOL, ALGOL, and FORTRAN programmers, there is not as much opportunity as, say, for C, PERL, and JAVA programmers. But, programming is programming. You should understand flow control, block and flow diagrammming, flow charting, memory management, etc. These are the basic building blocks of computer engineering whether you are desiging websites, processors, embedded systems, or network systems. When you train for a career, you should take the long look. Don't train for a job - train for a career. And plan on always learning something new, every day, every week, and every year for the rest of your working life. That is the key to success.
