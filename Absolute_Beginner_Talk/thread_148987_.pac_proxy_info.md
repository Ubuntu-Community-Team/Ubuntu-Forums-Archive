### Post by fishd on 2006-07-06
This is a GAIM MSN plugin issue. It's not able to correctly proxy the ports required for MSN in the same way that the genuine Microsoft Client can.

When port 1863 is blocked (I think in your case it will be - your network team are either lying, dumb or just don't want to have the discussion with you) the genuine MSN client is able to reroute the ports via the HTTP proxy. The GAIM MSN plugin isn't smart enough to do this, can't blame them... they've had to reverse engineer the thing.

The only solution I've been able to come up with is tunneling GAIM via a SOCKS proxy via SSH to a machine on the other side of your firewall.
