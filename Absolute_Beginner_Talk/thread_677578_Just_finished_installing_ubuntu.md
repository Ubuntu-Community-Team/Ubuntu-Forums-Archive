### Post by ctswhole on 2008-01-25
If the live cd failed to load up for you, one very common fix is to add "noapic" to the boot parameter when the live cd loads.  I don't know if this is what is causing your problems or not, but it just might help.

It seems that "noapic" is my response to a lot of boot failures.  It's what I had to do to make my comp boot into it, that's why it always seems to be my first response to other similar problems such as yours.

If I remember correctly, when you boot your computer with the live cd, a screen pops up with options like "run or install Ubuntu", and other things.  I believe the last option is what you would go to to enter "noapic" in your boot parameters.  Hope this helps.
