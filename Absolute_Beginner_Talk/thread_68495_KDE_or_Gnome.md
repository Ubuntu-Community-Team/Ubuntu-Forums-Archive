### Post by David Marrs on 2005-09-23
[QUOTE=bugsout]Thanks again all of you. Where is or what is a very good beginners manual or terminology definitions source for all the acronyms?. Maybe a private tutor, And or a place to hang myself.  :?[/QUOTE]
 Don't worry. Everyone gets overwhelmed when starting out. The easiest thing is to just choose a distro and stick to its defaults. Later on, if you decide to get adventurous, you can start to learn about all the other ways of doing the same thing.

A lot of Windows migrants like KDE's familiar layout. A lot of Linux regulars prefer KDE's ability to customise absolutely everything. I prefer Gnome because, at the expense of many options, it's simple and elegant.

It's always worth learning the command line, for lots of reasons. Many how-tos work at the command-line (as you've apparently discovered) and understanding the command-line almost instantly leads to understanding GNU/Linux. There used to be loads of cool tutorials on getting started with bash but now I can't find a single one. If no one else can find a good one then perhaps I should write one myself.

Ah, I found one. The trick is to search for unix tutorials. [Enjoy!](http://www.ee.surrey.ac.uk/Teaching/Unix/index.html) :)
