Hi tobyfountain.

The best way to figure out if Ubuntu Linux will work on your desktop computer/laptop is to download a Desktop CD (also known as Live CD).

Live CD are executed (launched) in the ROM and CD irself, meaning it will not touch your hard drive.
The Desktop CD or Live CD let's you run Ubuntu without installing it. That way you'll know what will you get after installing beforehand. So if something dosn't work when you are in a live section, that means it won't work after installing.
But even if something dosn now work by default, you will be able to fix it manually whatever the problem is.

### Post by hoheszeh on 2007-03-24
I am running ubuntu on an older Vaio (VGN-FS515B). First thing i did when i got it was installing ubuntu on a second partition. Everything fine until i restarted and the computer told me that my system was "messed up", which led to the restore routine, which means my computer was recovered to the state i got it in. With a "nice" xp install and a deleted Linux.

It took some time for me to find and delete the hidden partition on which the XP-image was stored. Since i decided to go for a single-boot system anyway i killed all partitions and put ubuntu back on it. 

No problems ever since this decision. I am not sure how it will work with newer Vaios not to speak of Vista.

Try the live CD and in case you set up a dualboot, dont start setting up ubuntu to your liking untill after a restart, when you are done with the install of ubuntu, to see if this weird "system is messed up, i am doing a forced restore"-routine is starting to bug you.
