### Post by elvis on 2007-10-19
> **Cannaregio said:**
>  But remember that GNU/Linux is a matter of experimenting and trying out. Not just reading and slurping.

This be the truth.

I've been using Linux full time for 10 years now, and still learn new things every day.

A lot of people ask me these sort of questions: "How can I learn Linux".  What you need to understand is that the modern day "Linux" is merely a collection of quite literally thousands of different applications.  There's no standard or forced defaults anywhere - you can have whatever window manager you want, whatever shell you want, whatever tools and applications you want.

A better way to go about it is to figure out what you want to get done with a computer.  If all you need to do is browse the web, IM and send some emails, then some basic package management and network configuration is all you need to learn.

On the other hand if you want to set up authentication and file servers, web servers, DNS, etc - then that's a totally different set of tools you need to learn.

Nobody can learn it all.  I've met some absolute geniuses in my career who can turn Linux inside out and back again, and I can still find things I know that they don't, and vice versa.

Don't focus on generic training - you'll only forget it if you don't use it every day.  Focus instead on doing specific tasks, and finding ways to make your day to day needs within Linux easier.  You'll find the training more exciting, remember more of it for longer, and be more motivated in teh process.
