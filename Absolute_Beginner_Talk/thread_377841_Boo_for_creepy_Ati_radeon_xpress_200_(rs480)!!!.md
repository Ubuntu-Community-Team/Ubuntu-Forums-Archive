### Post by ARS on 2007-03-06
I am not sure but I have an ATI card too and found some information that helped me get mine to work... so it may help you.
After launching Ubuntu or till it give you the error, press Ctrl+Alt+F1 which should take you to the promt screen.  Then type in these commands:

sudo apt-get install xorg-driver-fglrx  (note need internet connection for this.. I think)
sudo aticonfig --initial
sudo aticonfig --overlay -type=Xv  (case sensitive)

after typing in those commands (enter after each) your graphics should work fine
the first command gets the drivers and the other two set it up hope that helps you... If I find the post where I found the info I will tell you.

I have found a solution and I'm trying to find all the forum posts/wikis/guides etc that I've read and share the solution that I found.]

Here's a link to the source:

[http://www.thinkwiki.org/wiki/Problems_with_fglrx#Perpetual_Mesa_GLX_Indirect_on_Debian](http://www.thinkwiki.org/wiki/Problems_with_fglrx#Perpetual_Mesa_GLX_Indirect_on_Debian)

And here's the actual fix:

Perpetual Mesa GLX Indirect on Debian

If you've done everything right and you're still seeing:

    $ fglrxinfo 
    display: :0.0 screen: 0 
    OpenGL vendor string: Mesa project: [www.mesa3d.org](www.mesa3d.org) 
    OpenGL renderer string: Mesa GLX Indirect 
    OpenGL version string: 1.2 (1.5 Mesa 6.4.1) 

try this:

    # mkdir -p /usr/X11R6/lib/modules/dri 
    # ln -s /usr/lib/dri/fglrx_dri.so /usr/X11R6/lib/modules/dri 

Thanks to Maciej Matysiak
