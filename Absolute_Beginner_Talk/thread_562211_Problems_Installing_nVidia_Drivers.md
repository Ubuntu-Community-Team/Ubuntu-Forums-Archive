### Post by Perfect Storm on 2007-09-28
Download the driver to your Desktop;
```
cd ~/Desktop
sudo aptitude install linux-headers-`uname -r` build-essential gcc gcc-3.4 xserver-xorg-dev
sudo aptitude --purge remove nvidia-glx nvidia-settings nvidia-kernel-common
sudo /etc/init.d/gdm stop
sudo sh NVIDIA-Linux-x86-100.14.09-pkg1.run
sudo /etc/init.d/gdm start
```

NOTE: sudo /etc/init.d/gdm stop will take you out X and you'll be in CLI. If sudo /etc/init.d/gdm stop fails, then use <ctrl>+<alt>+<F1> first.

### Post by trak87 on 2007-09-28
Here's a simple solution that worked for me using a GeForce 6200 card:

1. Using Synaptic, install the nvidia-glx-new package.

2. Edit /etc/X11/xorg.conf and change "nv'"to "nvidia". In your case change "vesa" to "nvidia". To make matters even more simple, you can forget this step and simply run the following and it will do what I just suggested for you:

```
sudo nvidia-glx-config enable
```

Finally, logout, press Ctrl-Alt-Backspace. This restarts the graphical X server. You should see the nvidia logo for a couple of seconds. Log back in to your account.

Here's some additional info:
[https://help.ubuntu.com/community/BinaryDriverHowto/Nvidia?highlight=%28nvidia%29](https://help.ubuntu.com/community/BinaryDriverHowto/Nvidia?highlight=%28nvidia%29)
[http://us.download.nvidia.com/XFree86/Linux-x86/1.0-9755/README/appendix-a.html](http://us.download.nvidia.com/XFree86/Linux-x86/1.0-9755/README/appendix-a.html)
