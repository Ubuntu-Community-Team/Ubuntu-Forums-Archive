### Post by Neobuntu on 2006-07-31
You may not like my answer but if it were me, order a free CD (I like Kubuntu) or ask a friend to burn you a copy AND ask for someone to give you at least a CDROM drive. 

Of course their are ways to do it but it's academic and will take more time than the above.

I suppose one could do a Debian net install and convert to Ubuntu but why?

I suppose one could use an old hard drive as if it were a CD. Maybe hook it up (jumpers, cables, BIOS and crap) and uncompress that which the ubuntu download contains. But then the CD image boots from all manner of CD drives with it's own "drivers." So it's just frought will time consuming roadblocks.

See above. CDROM drives are so common that they are given away (if not trashed.) You'll have to boot from something so (one) might as well hook up a CDROM, if not temporary.

I KNOW there's better ways to do it but I doubt they are "better."

### Post by Neobuntu on 2006-07-31
One could install that hard drive in to a computer with a CDROM (fairly quickly if you know what you're doing) and install that way. It might not all setup as well on the different computer once the HD is moved back but given passable harware requirements (like RAM), it should work,

You could also do a "server" install with the old 5.10 CD and type...

sudo apt-get install kubuntu-desktop

...once put it back into the CD-less system (with hardwired broadband preferred).

Beg. borrow (but don't steal) a CDROM drive. :)

P.S. This all assumes you have checked your target system for minimal (better yet recommended) hardware requirements.
