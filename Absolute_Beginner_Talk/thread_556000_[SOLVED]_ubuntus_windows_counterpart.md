### Post by lisati on 2007-09-20
Windows and Linux are different in their approach to managing the system, so some of the system tools loved by Windows users (e.g. regedit)  don't have a direct equivalent in Linux.

The general consensus of what I've read in other discussions is that the way the Linux file system is designed there's not the same need for a defrag utility, unless your disk or partition is getting full. In this situation it's proabably a good time to do a cleanup anyway, Much the same can be said about software that deals with "malware" (viruses, trojans, spyware etc) - this is in part due to the way Linux deals with system security.

As for "regedit", Linux organizes its settings very differently to Windows - perhaps the budding programmers out there who are getting bored might want to put their minds to how to design some form of equivalent.
