### Post by Rocket2DMn on 2008-01-30
I am not a fan of manually editing xorg.conf, so we're gonna try reconfiguring X to see if that fixes the problem.  First, we'll make a manual backup for xorg (even tho one is automatically made when you run this script).
This little program will ask you questions about your hardware - do your best to answer.  At the video driver page select "ati" and at the resolutions page use TAB to move around and SPACEBAR to select your monitor's max resolution and everything less.
```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.backup
sudo dpkg-reconfigure xserver-xorg
```
Then do CTRL+ALT+BACKSPACE to kill and restart X.  IF for some reason the GUI doesn't restart and drops you at a terminal, recover xorg and restart X (this shouldn't happen, but you never know):
```
sudo cp /etc/X11/xorg.conf.backup /etc/X11/xorg.conf
startx
```

### Post by erginemr on 2008-01-30
I have a Samsung SyncMaster CRT monitor. I had configured my desktop from the terminal having a xorg.conf file with a "generic monitor" or something, and all was working well. Then I gave a try to the new screen resolutions tool, selected Samsung SyncMaster as the monitor, restarted the X server to be greeted by a garbled desktop.  

Long story short, the tool "Screens and Graphics" is a new app and needs some ironing. In this respect, I suggest you to back up your existing configuration with:
```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.bak
```

And then, run the following command from the terminal:
```
sudo dpkg-reconfigure xserver-xorg
```

Somewhere in between, it should ask you to select your preferred resolutions. when it is over, restart your X server with Ctrl+Alt+BackSpace. 

If worse comes to worst, you can always restore your current settings with:
```
sudo cp /etc/X11/xorg.conf.bak /etc/X11/xorg.conf
```
