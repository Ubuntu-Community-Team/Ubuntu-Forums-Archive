### Post by v8YKxgHe on 2006-12-27
Use it for more than a couple of days and you will see why it's better than Windows. 

Having a centralized repository system  for all your applications is the best thing in the world, it means you can easily and quickly get the majority of programs you need with a click of a button, or a simple command in Terminal. No more hunting around the internet looking for a bunch of "exe" files that you have to download, double click on then hit "Next" about a hundred times then once it's finished having to restart. You will soon love the resporities.

You talk about the applications being quite basic, this is normally a Gnome thing. Gnome is a Desktop Environment of which there are many, but the 2 major ones are KDE and Gnome. Gnome often goes for the minimalistic approach where as KDE is more Windows like, it offers more options but can sometimes confuse the user by showing too many options at once (though that is my personal opinion). Some like Tea some like Coffee, try KDE (Kubuntu) if you want a more Window's feel I guess. 

> im sorry i just dont understand why a more basic-but-difficult, boring incompatible,system is supposed to be better.
It's not more difficult at all, it's just different from what you are use to. If you came from a Linux background and had never heard of this "Windows" operating system and then tried it, do you not think it would be strange? You would think hunting around the Internet for applications and drivers was a completely mad idea!

Most guides you see on the Internet will be in command line form, this is because it is far far far easier to tell someone over the internet to do "sudo apt-get install beryl" than say "Go to System->Administration->Synaptic, right ok now once that is open go to Search (up the top, next to that button) then type in "beryl" in the text box....... etc etc" 

A lot of the guides could tell you how to do it via GUI instead of CLI, but as mentioned above it's easier to just tell users to use commands.

To answer your question as to how Linux is better than Windows, well there are no known viruses, much more secure overall, faster, more customizable, it doesn't spy on you, no more worrying about Licensing and "Genuine windows" crap. Like I said at the very start, try it for more than a few days, post questions on these forums and we'll be glad to help you out. You can not just dive straight in, try it for a few days and question it. 

Good luck,

### Post by 3rdalbum on 2007-06-13
I'd like to address two fallacies from the original post.

1. That compiling from source code is somehow a more "incompatible" system than the one Windows uses.

Application vendors can distribute their software as binaries that will run on all Windows systems. This is not due to any feature in Windows; as you've seen, an application vendor can distribute a binary for Linux if they so desire. The reason you can get away with distirbuting binaries on Windows is because Windows only runs on one type of processor: i386 (x86).

Linux can run on more types of processor than you've heard of. These processors cannot run i386 instructions, therefore if a Linux application vendor distributed ONLY binaries they would be leaving out all the people running Linux on PowerPC, ARM, Sparc and all other processors. All in all, it's quite a reasonable number of people! Most source code can be compiled and run on these other processors or "architectures", so that's why source code is distributed as it's the lowest common denominator.

Source code is great for another use. With precompiled binaries, you have to accept the options that the vendor has provided. With source, you can enable and disable various features at compile-time; so if you want to use the latest bleeding-edge features, you can enable them; and if you don't want to use those features, you don't have to have the libraries to install them or the bloat associated with them being present in the program.

Most newbies come from the Windows-centric world, where it's assumed that all computers are basically the same. In reality, that's not the case - a PowerPC processor can't run an i386 binary any more than a Core 2 can run a Power binary. Source code ensures compatibility. And flexibility. And security. And it's not rocket science to compile source code. If you've got an x86 processor you can often find precompiled .debs for things too.

-------

2. Windows doesn't require you to use the command-line.

Look on Microsoft's knowledge base. Try and find a solution to a real-world problem. As a part-time computer tech, I look on the Microsoft knowledge base site a fair bit, and I'm constantly seeing "Open the MS-DOS prompt and type this in."

But hey. The other day, I tried to defragment an external NTFS hard drive. The defragmenter literally told me "Cannot defragment. Please run chkdsk \F".

Good old command-line reliance in Windows. Note that it didn't say "Please run the MS-DOS prompt program and type this in". It didn't even say "Type this into the MS-DOS prompt". Just "Run chkdsk \F". If I hadn't already known what a command-line was, I'd have absolutely no idea what this was all about!

Plus, Windows gave me the wrong command. I needed to actually type "chkdsk \F G:" in order to check my external hard disk. I found this out through the "help" command in MS-DOS.

So please, I don't want to hear complaints about "Linux is too reliant on the command-line". If you do anything slightly outside the box on Windows, well, you can't do it in Windows, only in the MS-DOS window. This includes batch-encoding WMAs, which I'm sure absolutely nobody else has ever wanted to do. </sarcasm>
