### Post by Cypher on 2007-05-08
Another thing to keep in mind is that as long as you try to make Linux speak the Windows language, by way of documents/files or anything else, you will tend to have issues. 

So using your example if you had to send a document over to a Windows user, you could indeed use OpenOffice to create a .DOC file and send it the Windows user to open in Microsoft Word. Now just imagine that this user has Microsoft Office 2007 which, by default, saves all files in the .DOCX format. If they were to return your .DOC file in the newer .DOCX, OpenOffice wouldn't know what to do with it. (Granted that .DOCX is just a ZIP file containing XML files that describe your document, but still).

The better path would have been use PDF or ODF which can be created/edit by program in Linux just as well as Windows. You're less bound to cause problems from there on.

### Post by aysiu on 2007-05-08
The answer is never.

There are only two ways this goes:

1. Desktop Linux distros stay at a low or single-digit market share, with a lot of major commercial software companies still not porting their products to Linux (i.e., what's happening right now)

2. Desktop Linux takes the majority market share away from Windows, at which point major commercial software companies would make native Linux applications and Windows users would be the ones asking, "Will there ever be a Windows that will run all Linux programs?"

There is no 3. Desktop Linux stays at its current market share but runs all Windows programs. ReactOS is working on being an open source operating system with full Windows binary compatibility, but it is still in its alpha stages of development. By the time it gains full compatibility with XP, Windows will have moved on to whatever Vista+1 is.
