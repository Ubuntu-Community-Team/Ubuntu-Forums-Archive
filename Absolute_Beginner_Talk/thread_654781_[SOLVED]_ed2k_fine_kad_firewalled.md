### Post by Hallvor on 2007-12-31
It is probably because KAD needs to bootstrap properly. This means that KAD needs to find enough sources from the eD2k network to run optimally. Just put a few popular files on your download list, and the light should turn green in a while.

If you mean the crash when updating ipfilter.dat, this is a well known bug. There are alternatives, like Moblock. If you feel adventurous, you could always try installing the aMule CVS version from the aMule forum (available in .deb format and built for Feisty and Gutsy).

[http://forum.amule.org/index.php?topic=13836.0](http://forum.amule.org/index.php?topic=13836.0)
