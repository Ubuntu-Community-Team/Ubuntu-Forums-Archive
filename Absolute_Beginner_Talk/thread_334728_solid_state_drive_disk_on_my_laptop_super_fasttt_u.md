### Post by blackened on 2007-01-12
The speed increase from solid state drives is truly awesome, but there's a catch:
because the OS is constantly reading and writing to it, the lifespan of the CF card will be *_drastically_* decreased. A sort-of solution to this would be to install something like the frugal version of Damn Small Linux that's tailored to solid state media.

### Post by blackened on 2007-01-12
Currently the only SSD that's been widely adapted to IDE plugs is CF.  Now don't get me wrong, this doesn't make the technology unusable, it's still a very viable and attractice alternative to magnetic drives.  I only meant to make you aware of the situation and possible solutions or compromises you can make to extend the life of your CF.

If you were to install a frugal OS (as mentioned above with Damn Small Linux) on the CF card, but put the swap partition and all user files onto a magnetic drive, not only would this allow you to overcome the limited storage size of the CF, but at the same time you would remove two of the larger write-heavy partitions to more stable media.

This setup would be the ideal compromise between speed and durability.  Especially if you have enough RAM that the swap barely becomes a factor.
