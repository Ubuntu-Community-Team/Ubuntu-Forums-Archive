### Post by skymt on 2006-05-13
Most geeks (and Linux is still used mainly by geeks, though we're making progress) hand-code their web sites. So you'll find many different programs to make editing HTML directly easier, but not many at all that will generate it for you. Two that come to mind are Nvu (recommended above) and OpenOffice.org.

Nvu is just for making web sites, so it's very focused. It's also based on Mozilla Composer (based, in turn, or Netscape Composer), so it's easy-to-use and consumer-friendly. It's not very powerful, though. It'll seem sparse compared to something like Frontpage or Dreamweaver.

OpenOffice.org is a general office suite, but it provides surprisingly good support for the web. You'd write your pages in Writer, the word processor, add links, formatting and pictures, and export to HTML. It's great for writing comfort, since it has all the normal word processor features, but the result will probably look like a word processor document, not something made from scratch for the web.

If you already have and know Frontpage, Dreamweaver or the like, you may be able to run them in Ubuntu. [Wine](https://wiki.ubuntu.com/Wine), with some tweaking, may allow you to run your favorite web app without booting into Windows. [Crossover Office](http://www.codeweavers.com/) may help.

On the other hand, you could decide to learn to hand-code your web pages. If that's what you want, I recommend getting [HTML for the World Wide Web](http://www.amazon.com/gp/product/0321130073/). It's what I used to learn HTML and CSS. It's very easy, going from the easy stuff to, well, more complicated easy stuff. It's also very visual, showing what everything looks like at each step.
