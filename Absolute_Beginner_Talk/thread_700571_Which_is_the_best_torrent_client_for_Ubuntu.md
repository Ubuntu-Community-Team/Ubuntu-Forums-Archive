### Post by izi on 2008-04-05
I guess I've tried them all...

Azureus
=======
Probably the best. A full blown warship with all the gun and ammo. it has everything and works amazing - stable, reliable, has a TON of extra features (safe peer, RSS, MAGNET links, the best DHT, and almost every plugin you can think of) all easy to install if you have the right set up. 

And the best feature of them all - If you have a slow connection (like I do) and cant seed/download more then one or two torrents at a time this is critical - Azureus can sense when a torrent has become inactive (say less then 0.5kbps) and automatically start the next torrent in queue. Once the first torrent has regained a normal download speed it automatically pauses the newer torrent so you don't have more then 2 "active" (as in really downloading) torrents at a time. In addition it manages limits in an intelligent way - if you define a total of 3 active torrents (2 downloading and 1 seeding) it will automatically adjust if for example you have only 1 torrent downloading (it uses separate quotas for seeding).

I can't really stress how major this is, especially if you download a lot with slower connections like I do.

HOWEVER - unless you have 1Gb of ram don't bother. It uses java and therefore completely chokes you memory and resources. In addition you'll have to install it manually - the version in the repository (the one you install via Synaptic or Adept) is practically archaic (almost two years old) but it will give you a nice taste of capabilities. If you have the time for it try getting version 3:

1. Download and unzip somewhere in your home directory (I have a dir. called /home/~user~/apps/azureus).

2. make sure you've installed Sun's java 6 or later. If you're a total newbe you'll probably need to add some repositories in synaptic that have propriety software (like Flash,  Java etc.) There's plenty of guides on how to do that around here... I now use 'icedtea-java7-plugin' which should be in the main repo.

3. Edit (yeah I know, a bit scary) the file called 'azureus' in the program dir. It's a script file, and in the first few lines there's a definition of where Java is installed on your computer, and where you've unpacked azureus. 

my first few lines look like this:
```

######## CONFIGURATION OPTIONS ########
```

As you can see I used 'sun-java6' before so if this is what you've installed use the commented line

In addition Azureus can also be used without a graphic interface if you're interested in a dedicated machine (can be controlled from the terminal). But this requires some reading and further tweaking...

So far for Azureus if you want to do it on your own. 

Deluge
======
Very nice, but still very raw. I use it now because I only have 512Mb of ram and not a very strong PC at home. It has some nice features (RSS, block lists, control plugins) but is still lacking some important features - it cant manage "active" downloads... You can define that new torrents are inserted before active ones but it's not ahlf as good as Azureus - If you download a slow torrent it will occupy a download slot even if its not really downloading. In addition - no MAGNET links - the ones you see on Mininova.
I have some issues with the interface (for example with the RSS plugin) and some weird behaviors that I can't explain - sometimes when I seed two or more torrents only one will be active. And of course I use KDE and it doesn't integrate well (and requires the installation of GNOME libraries).
It's a good client for beginners but not for download hogs... ;)

KTorrent
========
Again - A great, light weight, feature rich client. Sadly - The same issue with queue management as with Deluge. no MAGNET links here too, and the KDE 4.0 version doesn't include the RSS plugin (So I still use an older version).
The interface is not really intuitive for me but that's a matter of taste, and theres a really annoying "queue manager" that's really not intuitive and annoying. Still - good for beginners and very light and responsive.

Other clients like Transmission etc. are a bit to simple for me - without SafePeer and RSS I don't see the fun of using them. But hey, it's a matter of taste...
