### Post by boban on 2006-11-10
> **spinflick said:**
> Hiya boban, so what would be the best way to try this?

In short (I'll make complete howto when I'll have more time):

1. You need to prepare partition for Gentoo and mount it...
2. sudo su and do exactly like in Gentoo handbook (of course you have to change path to your mount) until you get to point involving mirrorselect tool - find your mirrors on the net...
3. After chroot'ing into gentoo you follow handbook literally :)
4. Don't install Grub - you already have one - just add new entry to /boot/grub/menu.lst

Another tip: change niceness of chtooted gentoo shell (you can do it from system monitor) - I've set nice=-19. It will get your current installation  more cpu time...

I've installed gentoo, then X and radeon driver, then alsa, xfce, openoffice, firefox. And only when I set up everything I rebooted... All the time during the process I could work in Ubuntu...
