### Post by maspro on 2005-06-11
And to make matters a bit more complicated.....  ;-)

You can also install Ubuntu and get both **KDE *_and_* Kubuntu** through apt-get. And by KDE I don't mean Kubuntu. I know that Kubuntu includes KDE, but it's not entirely the same thing. Kubuntu isn't a full KDE install and doesn't provide all KDE stuff that a seperate KDE install does! 

So you can:

- Install Ubuntu + apt-get Kubuntu
- Install Kubuntu

- Install Ubuntu + apt-get Kubuntu + apt-get KDE
- Install Kubuntu + apt-get KDE

- Install Ubuntu + apt-get KDE

Kubuntu **+** KDE is a more fully and complete KDE experience, providing even more KDE stuff!  :grin:

### Post by maspro on 2005-06-13
I think it's about the limited space available on a cd, so they have to choose which apps they are going to include. I don't think you should think of it in terms of features, but more in terms of the amount of apps. Some of this is also true for Ubuntu, so that's why there are repositories where you can download the remaining stuff with apt-get.

So installing Kubuntu from cd should be the same as Ubuntu + apt-get Kubuntu-desktop

If you choose to also install the remaining KDE metapackages, such as KDE-core, KDE-base, etc. then you get more apps and maybe some minor configuration changes.
