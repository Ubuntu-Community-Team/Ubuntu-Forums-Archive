### Post by aktiwers on 2007-02-20
I would say give it a shut.

The main problems you have is:
* Photoshop
* Macromedia Flash Mx
* ATI Drivers

There is a few solutions to this though.

ATI drivers:
I have been running with 2 different ATI cards, and gottem them both to work, though it is a pain. I ended up buying an nVidia for $90 and now my life is a lot easyer. I have Beryl and all that running fine!

Photoshop :
There is no Photoshop for Linux. Many people learn how to use Gimp instead, and it helps a lot with Gimpshop installed, that replaces the gimp names for filters and stuff like that to Photoshop names. Also it changes the gui at bit to look like photoshop. However, I know a lot of people who still arent satisfied with Gimp.

Macromedia Flash Mx:
There is also no Macromedia Flash Mx version for Linux. However there is another solution for both Photoshop and Macromedia Flash Mx.

You can then either dualboot and only use Windows for Photoshop and
 Macromedia Flash Mx or you can install Windows XP on top of Linux in Vmware (I do that on an older PC than yours and it runs just like XP was there alone). Actually I run Photoshop in there too, and it runs great!

Also if you dont want to look at XP, you can setup a remote control from Linux to your vmware XP or another XP computer, and run Photoshop and Macromedia Flash Mx remote controlled. This way you have Photoshop in your Linux environment instead of inside XP in your vmware.

The last thing is optional though, as it runs fine in XP ontop of Ubuntu.

Actually I have XP on one site of my Cube in Beryl, so I just turn the Cube, and bang! im in XP. Turn the Cube again, and bang! im in Ubuntu.

I can post some links to guides if this is the solution you want.

Good luck, and welcome! :)

### Post by lapsey on 2007-02-20
Windows Software will either require [Wine](http://appdb.winehq.org/) - see the compatibility database - or Windows running on a virtual machine on top of Ubuntu.

Your hardware is well above the minimium requirements for Beryl, AIGLX etc, and apparently your gfx card is [supported](http://doc.gwos.org/index.php/ATI/Radeonx1300-x1900XTX)

However, Beryl is **beta** software, so while it works perfectly for many it does not work for all. My advice is to experiment and install ubuntu to a separate partition (included in install options) before you take the XGL plunge.

it should also be pointed out that a ubuntu live cd will not come with beryl (...yet). Other live cd distros, such as kororaa, do.
