It's also known as the command line interface, and is similar in concept to Windows' command prompt.

How to use it: In gnome, click Applications > Accessories > Terminal, and then, well, type commands in it.  All the commands I've used have been copied and pasted from advice given here, but I imagine there are listings somewhere.  Not trying to be a smartass, just don't know enough to tell you much of anything useful yet.

Why we use this...  My take on the terminal vs a GUI (Graphic User Interface, ie, point and click) is like a multiple choice test vs essay questions.  The Multiple choice/GUI gives you limited but easy to use options, where the terminal/essay question lets you use all your knowledge and get more things done.  Something like that.

### Post by nalmeth on 2006-07-10
It's easier to help someone out by posting commands than telling the user to click here, double click there, right-click the little thing over there, drag that bit here, and hit OK.

Yes, when you have instructions to type the commands you can just copy and paste them into the terminal. Better yet, when you highlight something in linux, it automatically copies it. To paste it, click the scroll button, or if you don't have one, both mouse buttons at the same time.

Oh, one more thing, if you see a command like this:

$ sudo gedit /etc/fstab

leave out the $. You should only enter the command itself. Curiously, some people think it's easier to indicate a command with a symbol, but often it confuses people as the command doesn't work with the symbol included.
