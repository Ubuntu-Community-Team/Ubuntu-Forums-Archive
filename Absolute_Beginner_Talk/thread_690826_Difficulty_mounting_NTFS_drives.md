### Post by Rocket2DMn on 2008-02-07
OK, so for your two entries in fstab, I don't know what \040 means (& ?).  If you trust that it works, you can keep them that way, otherwise I would come up with simpler mount points like "/media/videos" and "/media/games_apps".  I use "defaults,locale=en.US.UTF8" as my options - note the absence of a dash.  I'm not sure if that has any effect.  If you choose to change the name of the mount points, don't forget to create the directory with
```
sudo mkdir /media/*newmountpoint*
```

To edit fstab:
```
gksudo gedit /etc/fstab
```

Either leave it as is or change as necessary, then post the updated fstab as well as
```
sudo mount -a
```

### Post by Rocket2DMn on 2008-02-07
You can delete the other directories using
```
sudo rmdir /media/*foldername*
```
Try running
```
sudo mount -t ntfs-3g /dev/sdb1 /media/movies_tv -o force
sudo mount -t ntfs-3g /dev/sda1 /media/music_more -o force
```
Windows did not cleanly unmount the drives which is why they aren't automounting.  You need to force them to mount just this once.

### Post by Rocket2DMn on 2008-02-07
They will be fine, this happens when Windows doesn't unmount the drives properly, so some table is left in an open state (or something like that).  Forcing the mount just takes control of that table and takes care of it.
Of course, always make good backups of your data, regardless of file system or operating system.  If you don't have one, I suggest investing in a backup hard drive, either internal or external.
