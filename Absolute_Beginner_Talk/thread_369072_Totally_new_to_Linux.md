### Post by Velotix on 2007-02-25
Hm. I'm honestly not sure what to say to you. :/ Here's why:

> **4evrnite said:**
> Hi all ):P  ,

Here is all I know about Linux: It is an operating system.  :confused:

This isn't true either. Linux is a *kernel*. Though that is a very important part of an operating system (because it's the part that talks to the hardware directly), it is not accurate to call a kernel an operating system; a number of additional programs must run on top of the kernel allowing graphical programs such as those in Windows to function. An operating system is a collection of programs designed to make a computer usable. The correct name for the complete system is "GNU/Linux". A lot of people shorten this to "Linux", but not only does this suggest that Linux is the OS, not the kernel, as you believed, it also means people don't associate GNU with the OS at all, and pretty much all of the OS is GNU. (GNU is the set of programs that communicate with the kernel, and another version of GNU exists that does not use Linux at all.)

> I am a winsloth user, cause some one gave me win95. Any ways, I am tired of the head ache that comes with winslith. So I thought I would give Linux a try.

Stop right there. No Ubuntu OS will run on that machine. (No, not even Xubuntu.) They all require significantly more resources than that computer can supply. You JUST qualify for the RAM, but the rest of it is far too old.

> With that said, any advice as to which version/product to test drive? Something that is easy to install and understand. Oh, I will use a 8gb HDD for the install.

Here's the big problem. You have either two options at this point if you wish to use Linux:

1) Use [Damn Small Linux](http://en.wikipedia.org/wiki/Damn_Small_Linux) (henceforth referred to as "DSL"), which will run quite nicely even on your ancient computer
2) Get a (much) better laptop

Most modern versions of Linux need resources pretty close to the minimum system requirements of Windows XP or more (including Xubuntu, which in fact is probably the closest) to run; ironically this is what is necessary to make the software more accessible to new users.

So, the conundrum is this: tell you to install DSL, which as I understand it is far from user-friendly, but works; or tell you to get a better laptop; or turn you away.

None of these solutions are ideal, but they are your three best options, as much as I don't like to say it. Sorry. :(

### Post by medeski on 2007-02-25
Do you have a CD drive?

If you do, what you should do is burn a Live CD.  A lot of distributions have Live CD files to download; what you do is burn the file to a CD and then boot up your computer from the CD.  Nothing gets installed -- what happens is the operating system runs from the CD.  That way you can try out different distros and see what works.  DSL might be a nice idea, and there are also other small distributions you can try.

A distro (distribution) is basically a version of linux.  Different distributions usually have their own ways of setting things up, installing things, etc.  There are some distribution families.  For example, Mepis is based on Ubuntu, and Ubuntu and DSL are based on Debian.

Gnome, KDE, and XFCE are desktop environments.  In Windows, you can't really change the way you interact with the computer; every Win95 machine, for example, will give you the same user interface.  Unix is based on a philosophy of giving you a lot of building blocks that you can put together as you like.  So, for example, if you want your computer to have a user interface like Win95 or XP or Vista, you can; if you want it to have a UI like OSX, you can; and if you want something completely different, you can do that too.  Basic user interfaces are called Window Managers.  Desktop Environments are Window Managers that do a lot of extra things, and basically try to let you get everything done through their graphical UI (instead of using the command line).  They usually use up a lot more memory, which is why if your machine is very slow, it can make more sense to use a lightweight WM instead.  However, if you do that, it usually helps to know how to get some things done on the command line.  (It's not as hard as it might sound.)

That's not the most exact explanation ever, but hopefully it clears some things up.

### Post by steve.horsley on 2007-03-01
Ctrl-Alt-Backspace is the wrong combination. It resets X, but it will restart, as you found out.
Ctrl-Alt-F1 through 6 gets you six text consoles, take your pick. You will recognise F1 - you saw it while X was resetting. Ctrl-Alt-F7 gets the GUI again.

Use any text console to do the **sudo dpkg-reconfigure xserver-xorg** thing, then go back to the GUI and Ctrl-Alt-Backspace to restart it. While reconfiguring, you could try choosing the vesa driver if all else fails - this may do the trick if the suggested settings don't work.
