### Post by mlissner on 2007-01-28
I can't point you to much of a discussion of the various shells, but I can tell you that the one just about everybody uses is Bash, the Bourne Again SHell. I had a professor go through the history of the various shells, and he explained all about the various shells. From what I recall:
 - He said the C shell, csh, was basically just a business-driven update of the Korn shell, which added in some features from C
 - The sh shell was THE original shell, developed by Bourne if memory serves.
 - the Korn shell, ksh, was an update to the sh shell, which added a couple of features
 - the Bourne Again Shell, bash, is an update to the sh shell which rolled in some of the better features of the Korn and C shell, AND
 - the x shell (or was it z?) was yet another update on the Bash shell which hasn't caught on at all (yet).

So in conclusion, the Bash shell is what you want unless you're a big C programmer. It's what everybody seems to use.

### Post by irish_flu on 2007-01-28
Python is an interpreted Language, for sure.  However, AFAIK it can be integrated with C++ to interpret on the fly. (sorry if that sounds stupid hehe, I don't know jack about C++).

I don't see it as "only" for development.  There are lots of production tools built with it, such as the "Zope" web and Application Server and the "Plone" Content Management Framework.  Apache also has mod_python to include your python fun into your web app.  :)

Python is also free and open-source, and free interpreters exist for every operating system I've ever tried to find one for.  Cross-platform development seems like a real focus of the language, in my eyes.

Here's the page:
[http://www.python.org/](http://www.python.org/)

You can install Python 2.4 right from the Repositories, using Synaptic or apt-get.  I would suggest doing it that way.  There are lots and lots of resources out there for Python, and lots and lots of code to cut-and-paste from the web to get you started.
