### Post by eljoeb on 2007-09-18
Umm, while I tend to agree with you,  try not to confuse a lack of familiarity with difficulty.  Without further ado...

Nautilus is the GNOME (the Ubuntu Desktop environment: what you are looking at when the OS loads) file browser.

Debian is another distro that Ubuntu is based on.  Don't need to know much else about it other than its for dirty commies (kidding! really!).  In other words, you don't need to know about it to use ubuntu.

Program files are usually hidden in your home folder.  If you installed Ubuntu and not kubuntu, try to ctrl+h to see these files.  It is rarely necessary.

Run as?  Eh?  I use Windows a lot, but I'm not sure what you mean.  Maybe alt+f2? Or if you want to stay as root (not advised) you can edit your user tags to be root all the time.  Another alternative is to type : gksudo nautilus in the terminal and you will get a root enabled file browser

If you don't like to use the terminal, I recommend Kubuntu, which uses KDE.

### Post by aysiu on 2007-09-18
Once you plug a volume in, it should mount automatically for any user.

The Windows *Run as* "feature" is a messy hack that doesn't always work. In fact, today at work I was just trying to do *Run as* for the Printers & Faxes, and I couldn't. You also can't do it for Regedit unless you actually find the regedit.exe file itself.

In Ubuntu, you can use *gksudo* or *kdesu* to launch any graphical application with root (or administrative) privileges, if your user is an *admin*. You can also use *sudo* for command-line applications, or even plain old *su* to switch users.

I think maybe you should address the real problem at hand instead of a messy workaround. The real problem appears to be that you have users who are unable to mount volumes. The messy workaround is trying to get them to "run as" some other use to mount the volumes.

Log in as the user who is unable to mount volumes, plug in the offending volume, open [a terminal](http://www.psychocats.net/ubuntu/terminal) and paste in these commands (separated by carriage returns): ```
groups
df -h
``` Then post the output back here.

Debian is a Linux distribution, on which many other Linux distributions are based. Including Ubuntu.

The main directories

/home -- where all the users personal stuff is. 

/mnt  -- where your drives are, such as floppy and CD, Ubuntu also has /media, which is more or less the same

/etc -- a vast wasteland of configuration files, mostly

/bin -- where (a lot of) programs are

/lib -- where the libraries are


/lost+found -- not much should be here!

Files and directories beginning with a dot (.) are hidden, but not very hidden.

Run As sorta equals sudo, but sudo only works on accounts that have been given permission to sudo (which is why only the password is required).

### Post by tenjin1 on 2007-09-19
Just wanted to add some info for you, to skim over and look at and read (since starting linux is alot of skimming over, reading and learning) :)

Great starting point on fresh install: [http://ubuntuguide.org/wiki/Ubuntu:Feisty]("http://ubuntuguide.org/wiki/Ubuntu:Feisty")

 **How to Mount NTFS Partitions**
[http://www.ubuntugeek.com/widows-ntfs-partitions-readwrite-support-made-easy-in-ubuntu-feisty.html]("http://www.ubuntugeek.com/widows-ntfs-partitions-readwrite-support-made-easy-in-ubuntu-feisty.html")

 **How to mount/unmount Windows partitions (NTFS) manually, and allow all users to read only**
    * In this example: 

    /dev/hda1 is the location of Windows partition (NTFS) 
    Local mount folder: /media/windows 

    * To mount a Windows partition: 

```
sudo mkdir /media/windows
sudo mount /dev/hda1 /media/windows/ -t ntfs -o nls=utf8,umask=0222
```

    * To unmount a Windows partition 

```
sudo umount /media/windows/
```

** How to mount Windows partitions (NTFS) on boot-up, and allow all users to read only**
    * In this example: 

    /dev/hda1 is the location of the Windows partition (NTFS) 
    Local mount folder: /media/windows 

```
sudo mkdir /media/windows
sudo cp /etc/fstab /etc/fstab_backup
gksudo gedit /etc/fstab
```

    * Append the following line at the end of file: 
```

/dev/hda1    /media/windows ntfs  nls=utf8,umask=0222 0    0
```
[B]
 How to remount /etc/fstab without rebooting [/B]
```
sudo mount -a
```

### Post by bliffle on 2007-09-20
Did anyone answer the original post?

The simple answer to his question is that the Windows "run as" has an equivalent in linux: "sudo".

But more importantly, how would a beginner learn that "sudo" is the equivalent of "run as"?

Learning the commands in linux requires a beginner to know only 3 "terminal" (or "Konsole" in kubuntu) basic commands:

1-"man" this is the basic underlying access to all the linux command usage information. it's easy and quick. All you really need to know is to get out of a "man" listing just type "q" for quit. Everything else is pretty easy to find by obvious experiment.

2-"apropos" to scan thru all the docs picking out references to a simple term, like "user"

3-"grep" to filter output from anything

Thus, to answer the original post, I started a 'terminal' session, and typed in:

```
apropos user

```

But I got too much stuff, dozens of lines. Thankfully it happened really fast so it didn't make a fuss. So I filtered it against 'command' with a 'pipe' to send the results to a simple 'grep', like this:

```
apropos user | grep command

```

which gave me less than a dozen lines to look at. Therein I found 'sudo' and some interesting other leads for ways to use 'user' and 'command' .

Then, one can learn the 'sudo' command specifically as follows:

```
man sudo
```

This illustrates the tremendous power of unix: the ability to combine the effects of many small little lightweight atomic commands and filters very quickly into customized functions of ones own. There simply is nothing like it in Windows.

### Post by bliffle on 2007-09-20
Once a guy gets the hang of using 'man', 'apropos', 'grep' and the pipe symbol '|', you can supplement that with 'more', 'less', 'head', 'tail, and the redirection symbol '>'.

1-'more' just basically slows down the ouput so it comes out one page at a time, and you hit the space bar to let it advance to the next page.

2-'less' is a variation on 'more', Check it out with man!

3-'head' shows just the first few lines of output.

4-'tail' shows just the last few lines of output.

5-redirection '>'  lets you send the output to a file. So another way to find out about commands that talk about 'user' and 'command' would be: 

```
apropos user > aboutUser.txt
```

Then edit 'aboutUser.txt' with your favorite text editor as you please.

I always keep a terminal session open in the background so I can get to it easily with <alt-tab> to get quick answers, or to quickly do some housekeeping, like:

```
eject JHUSB2

```
 to unmount and free a USB flash stick  (JHUSB2 happens to be the VolId on that stick) so I can take it out of the USB port. That way I'm sure all the data is finalized and that the system won't complain about the USB removal.

After a couple days using terminal and these simple commands you will agree that this technique works much better than the slow, clumsy, inadequate "help" system in windows, partly because "help" is clumsy, takes a llot of preparation by the developer, and does not have a conventional appearance and mode of operation between applications.
