exec blackbox

to the end. You've used chmod to make it executable right? You might not need the "exec" in front of conky. I have a .xsession file and it looks like this... (for example)
```

#!/bin/sh

conky &

```

### Post by 5-HT on 2006-02-26
Hi, 
did you start of the script with the shebang? ```
 #! /usr/bin/env bash 
``` 

Also, looks like the blackbox startup script wasn't included. I'm not sure what it would be (man pages or google should turn it up), but a really quick search yielded.

```
 exec blackbox 
```

So taking it all together...what about:

```

#! /usr/bin/env bash

screensaver &
gdeskcal &
conky -x5 -y150 &
exec blackbox 

```

You'll probably have to restart wdm before this works, Ctrl + Alt + Backspace should do the trick.


If this still doesn't work, try making a symbolic link from your .xsession to .xinitrc.

.xinitrc should just be for the 'startx' command...so if simply a 'startx' would do it, then something's going on with wdm.

Stormy Eyes wrote an excellent guide to custom X sessions which also has a link to a thread in these forums for support.
[https://wiki.ubuntu.com/CustomXSession](https://wiki.ubuntu.com/CustomXSession)

 If you still can't get it working, maybe that's an avenue worth pursuing?

### Post by 5-HT on 2006-02-26
[QUOTE=m666]
So what are these for:
#!/bin/sh  or #!/usr/bin/env bash ?

and shoud I use  'exec blackbox' instead of mere 'blackbox' ?[/QUOTE]

The #!/bin/sh or #! /usr/bin/env bash just specify which shell should run the script. Either one will work fine in this case.

About 'exec', I'm not really knowledgeable on exactly what it does, but I believe it executes the command give to it as an argument in place of the shell. 

It's not really necessary but still used by convention somewhat on today's computers (AFAIK), as it saves a process and a little bit of memory which came in very handy earlier.

Glad it's working.
