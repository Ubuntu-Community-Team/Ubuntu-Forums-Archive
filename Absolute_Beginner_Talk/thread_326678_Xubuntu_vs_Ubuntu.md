### Post by halitech on 2006-12-28
the main difference is what desktop environment you are looking at when you start and what programs are installed with it. Ubuntu and Kubuntu use Gnome and KDE, Xubuntu uses XFCE. Gnome and KDE are considered "heavy" desktops, XFCE is a lightweight  desktop and then things like fluxbox, etc are considered just window managers.

The main advatage of Xubuntu is it will allow the base OS to run on slower/older hardware while KDE and Gnome allow you to have a desktop that will allow you to customize alot easily.

### Post by pseudonym on 2006-12-28
Xubuntu is aimed more at users with older hardware, because it is installed with far fewer packages (in particular libraries) and therefore consumes less system resources. But there's nothing stopping you from installing it if your machine is a newer one. I have it on my amd64 machine and it flies.

There's also nothing stopping you from installing kde and gnome apps in xubuntu either (especially gnome, because xfce and gnome desktops share the same graphics toolkit).

So why install ubuntu or kubuntu? when it comes down to it, I think it mostly is an 'interface matter', as you put it. But there are a couple more complicated reasons for installing one DE over another -

1. If you like, say,  gnome apps more, many of course come pre-installed in Ubuntu, thus saving you alot of time and possible hassle in setting them up in xubuntu.

2. Depending on the number of 'non-native' apps you have in your DE, there can be issues with integration (eg. gconf outside of gnome, control centres and so forth). None of these are showstoppers AFAIK, they can just make your DE not quite as seamless as it may otherwise be.

3. Overkill in switching from a default app to a non-default one eg. I like gedit, but I would have to install a huge load of gnome libraries for the sake of having a tabbed texteditor which starts up slower than the deafult mousepad editor. So I don't bother installing it, and find that the default apps are perfectly suitable for my needs. Any others I need are all either gtk apps or non-DE specific.

This last point is entirely subjective, of course. If you don't care how many packages you have installed on your system, you can quite happily download and install whatever you want.

Personally, I also don't like esd (or any sound server) and gstreamer multimedia backend. I find these things are unnecesary layers that are still as buggy as they were when I first started using Linux 2 years ago. If they are not present on your system (as in Xubuntu), it as a few less things that can go wrong.

It's part interface, and part system demands. Gnome (and KDE, sometimes) has the unenviable reputation as a resource hog.

On the other hand, XFCE (which is to say, the core operating environment for Xubuntu) makes far less demands on the system than Gnome or KDE would. So for older hardware, Xubuntu is generally a better choice. 

"Older" is the operative word there. You can run Xubuntu on any machine; I sometimes put it on a 2.26Ghz machine that can run Gnome without a hitch. But putting Gnome on my 750Mhz laptop would be a nightmare. For that, I use Xubuntu ... sometimes. More often, I use Openbox, which is even lighter.
