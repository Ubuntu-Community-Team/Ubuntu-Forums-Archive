### Post by Rinzwind on 2006-04-28
> I'm 20 years old and... maybe it's a bit late to start with all of this and trying to become and work as a sys admin.

I am 17 years ahead of you and started this January with Breezy. I started with Dapper last weekend. Using commandline is fun :) :) 
(I do have a bit of experience concerning Linux tho).

I do think you should start with a GUI like gnome tho. You WILL be doing things in a terminal by typing commands. And gnome + internet connection is rather useful: you can get to information with gnome.

Ok. Basic tip: read all you can from this forum but just remember the general problem what the topic is about. I do this cuz when I go tinkering and I spot something I do not like I might have a 'beep' in my head telling me I read about it.

At 1st it's not knowing what to do but knowing where to find the answer.

Get used to the directory layout (/ /home /dev /etc /var).
In /var/log there should be lots of logs telling about events.

Those are a pain going through them from commandline and are best viewed with a log viewer from a GUI.

Did I tell you yet it's better to have a GUI and not use it alot then not to have a GUI to start with? ;)

### Post by jobezone on 2006-04-28
One of the most important things I've learned when I used Debian, was how helpfull it was to use the documentation available in the system.
You've probably heard of 'man', but I'm talking about the **/usr/share/doc/** directory. Each package installs its documentation in a subdirectory of that directory, and usally includes a file name README.Debian, which tells tips or important things a Debian/Ubuntu user should know about that specific software and how debian/ubuntu handles it.

A good documentation website to learn about a debian-like system is [http://www.us.debian.org/doc/user-manuals.en.html](http://www.us.debian.org/doc/user-manuals.en.html) . 
Have a look at "Debian GNU/Linux Reference Card", "Debian Installation Guide" and "Debian Reference" . The first one is a two-side page with the most important commands when using debian (and ubuntu). The last one is a huge and thick document explaining how a debian system is maintained. These texts are pretty much aplicable to ubuntu. To finish off there is "Debian GNU/Linux FAQ" which is a lighter read and also good to get some understanding on debian systems.

Finally, [http://www.linuxcommand.org](http://www.linuxcommand.org) is good if you're green on using the terminal.

Good luck, and have fun learning!
