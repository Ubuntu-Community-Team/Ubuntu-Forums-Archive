### Post by jan quark on 2008-03-11
the permissions for ntfs partitions is only active for the time the partition is being mounted 
try this:

make a backup of the fstab file so when something goes awry:
```
sudo cp /etc/fstab /etc/fstab_backup
```

_edit the fstab file_

```
gksudo gedit /etc/fstab
```

add these lines to the fstab file:

```
/dev/sdb1   /media/yourname   ntfs   ro,auto,user,fmask=0111,dmask=0000   0   0

/dev/sdc1   /media/yourname   ntfs   ro,auto,user,fmask=0111,dmask=0000   0   0

/dev/sdd1   /media/yourname   ntfs   ro,auto,user,fmask=0111,dmask=0000   0   0
```

change your 'yourname' to your drive name in the /media folder

### Post by bodhi.zazen on 2008-03-11
for ext3 partitions, go ahead and mount them.

use chown and chmod to set ownership and permissions.

```
sudo chown user.group /mount/point
sudo chmod 777 /mount/point
```

[man chown](http://www.hmug.org/man/8/chown.php)
[man chmod](http://www.hmug.org/man/1/chmod.php)

### Post by bodhi.zazen on 2008-03-12
External devices (CDROM, DVD, and in this case USB hard drives) will auto mount via hal and gnome-volume-manager.

With Linux native file systems, ext3 in this case, you just need to set the permissions / ownership one time and that's it.

If you want to add the hard drives to fstab, I advise you mount them by uuid or label as the device may change.

If you would like help, see this link and / or post :

[http://ubuntuforums.org/showthread.php?&t=283131](http://ubuntuforums.org/showthread.php?&t=283131)

```
sudo blkid

On more tip, you do not need to delete anything in config files, comment lines out with a # at the front of a line.
