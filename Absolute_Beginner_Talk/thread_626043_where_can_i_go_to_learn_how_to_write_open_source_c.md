### Post by meatpan on 2007-11-28
Since open source development usually involves a lot of developers spread over a large area, coding standards and guidelines are relatively important.

I've found the gnu coding standards a great place to start:
[http://www.gnu.org/prep/standards/standards.html](http://www.gnu.org/prep/standards/standards.html)

Here are a few additional suggestions:
o Make sure your code formatting and naming conventions are consistent.  
o Use a standard header, or even better, select an open source license for your code.  Include the license preamble in each source file.  The license will have instructions for you to follow.
o Stick to programming with other open source libraries.  This includes libs you want to link, and tools, such as compilers.

### Post by Matthew Bartram on 2007-11-28
I would suggest picking a language and learning it to start with. Better still if you've got something (fairly simple) you want to do with that, as a project to start learning.

C is the main common one that everyone and everything uses (yes, anyone feel free to pedantically correct that if you must)
Python seems quite common for scripts here and there, plugins, and that sort of thing. It's simple to learn yet powerful, and seems to get a lot of use in the open source world.
Similar to that is Perl - again a scripting language. Been around longer, used for lots of things here and there.
Java is good for web stuff.

Others I'm sure can give a much better idea than I can. I've found Python very good as a simple to learn do-it-all common open source language. It's what I've started and I haven't regretted it

It perhaps depends on quite what you're wanting to do with your open source coding. C is probably by far (from my limited experience and knowledge) the most used language, and is perhaps the one you want most if you want to deal with other people's code, or get involved with open source projects. But it takes longer to learn and to write things in than, for example Python.

Hope that's helpful.
