### Post by Aurdal on 2007-08-04
With Linux comes choices, one of those choices is which Desktop Environment to use, generally this will be either Gnome or KDE (there are also others, like XFCE, but they are not so common.)

Gnome and KDE operate with different tool-kits. Gnome uses the Gimp tool-kit, also know as GTK, whilst KDE uses the Qt tool-kit.

A toolkit is what an application uses to make it's graphical user interface(GUI), the applications which are more suited for Gnome are the ones using GTK, and the ones more suited for KDE are the ones using Qt.

The fact that an application is better suited for a desktop environment you don't use doesn't mean you can't use it, it should work just fine. Your computer will likely use a bit more ram because it needs to load an additional tool-kit, but most computers should handle this just fine.

Also there's the point of aesthetic integration: The applications for made with different tool-kits don't look the same. This should only be a 'problem' if you run KDE though, as in Gnome most applications don't look the same anyway.
