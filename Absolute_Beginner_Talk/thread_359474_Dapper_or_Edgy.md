### Post by Spr0k3t on 2007-02-12
There are a few minor quarks and riddlements within the confines of Edgy, but nothing that can't easily be worked around imho.  If you are wanting solid long term system, I would go with i386 Dapper over AMD64 Dapper.  My reasoning is just so you can use the OS and browser without having to force mod your system into thinking it should work one way vs another.  Some codecs aren't available in 64bit and so the 32bit source tree would give you more options to choose from.

With that said, i386 Dapper FTW.  Solid, stable, very few required/needed security updates.

### Post by Spr0k3t on 2007-02-12

There are some proprietary programs that refuse to release a 64bit version (and in my opinion, in doing so they are holding back computer's evolvement), but there are always work arounds or other options for those programs.

Completely agreed!  My AMD 64 FX2 system is dead set on 64bit.  The major problems I find are simple things like plugins for the browsers and some video codecs (only found 1 so far).  The way I see it, 64bit is the current supported path and in six months it will be hard to find a computer system that is not optimized to run 64bit.

If you want to utilize every bit of processing power you have on your lappy, make sure you have a good chunk of memory to go with it.  From what I understand, you will see a 20% increase in speed with just memory alone in 64bit applications.

When the first 64bit process came out, it was nothing more than just a fad for the PC market.  Nothing really happened in the software world and even nVidia stopped support that branch.  This time around it seems that companies are trying to ditch the 32bit structure and move on to 64bit to push past the speed cap (I can't remember that particular computing law, but it exists).  Also, many companies have prioritized 64bit versions of their existing software (Adobe, Sun, Opera, etc).  So if you are willing to wait on the 64bit support from a few software houses, then go with the AMD x64 release.  If you just want everything working now and will look at rebuilding later, go with the x86.

I hope this gives some insight on which path will suit your needs.
