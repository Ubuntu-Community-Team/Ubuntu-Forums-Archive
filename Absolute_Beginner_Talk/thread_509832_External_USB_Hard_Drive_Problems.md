### Post by Keen101 on 2007-07-25
I suppose if worse comes to worse, you can always use a Gparted Live-CD and re-format it to FAT-32.

FAT-32 is a partition format that is easier to work with.

FAT32 is easily accessible on ALL modern OS's, because of the fact that most USB Flash (thumb) Drives have the FAT32 file format. I am not sure the same is true for NTFS.

In my opinion, I think it would be easier to just re-format the external hard drive to FAT32 then to start trying to get Ubuntu to write to NTFS.

you can get Garted Live-CD here:   [http://downloads.sourceforge.net/gparted/gparted-livecd-0.3.4-8.iso?modtime=1182962209&big_mirror=0](http://downloads.sourceforge.net/gparted/gparted-livecd-0.3.4-8.iso?modtime=1182962209&big_mirror=0)

### Post by meierfra on 2007-07-25
back up the file  /etc/fstab via

```
sudo cp /etc/fstab  /etc/fstab.backup 
```

Open the file:

```
sudo gedit /etc/fstab 
```

Add this line:

/dev/sdb1  /media/Maxtor  ntfs-3g     defaults,locale=en_US.utf8       0      0

Save the file. Mount all the partitions via

```
sudo mount -a 
```

Check whether you can access your files now. If not reboot and try to access the files again. If it still doesn't work come back here.

### Post by KyleBrandt on 2007-07-27
By default, writing to a ntfs drive in Ubuntu is disabled.  This is because writing to ntfs is considered experimental, meaning there is a risk.  If you have work data on the drive, you probably shouldn't do it. If you really decide you want to, read this: [http://ubuntuforums.org/showthread.php?t=217009](http://ubuntuforums.org/showthread.php?t=217009)

### Post by zxccvb on 2007-07-27
Look it's very easy to make NTFS work.

Just install ntfs-3g and ntfs-config.

Then go to Apllication->System Tools->NTFS Configuration Tool.

A window will present you with two options, one for internal and the other for external and drives. Pick the one you need and you are good to go.
