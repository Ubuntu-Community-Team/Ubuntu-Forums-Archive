Why do people always recommend Java? Programs made in Java are so sluggish compared to those of C/C++ it's astounding.

I'm taking computer science in eight months, and I've already taught myself PHP. If you don't know how to program I'd honestly advising learning PHP first, because it's simple, shares some of the same syntax as C++, and with PHP you don't have to deal with storing bytes/bits of data in RAM or whatever (This is the part I'm confused on with C++ D:). If you don't really want to learn PHP I'd recommend Python, it doesn't share any syntax with C++ but it does teach you how a program flows; The only problem with Python is (From what I've heard) making a GUI is a pain in the butt.

All I can tell you though, from my limited knowledge, is to download either WAMP5 (If using Windows) or > sudo aptitude install apache2 libapache2-mod-php4 if you're on Ubuntu/Debian. These will get you started on PHP, and working on it using [http://localhost/](http://localhost/) will be much easier than buying webspace and uploading your scripts everytime you want to test them (This is what I did for my first two months of PHP, and I regret it).

### Post by drewwa on 2007-01-10
I recommend Java for learning programming, not necessary for a given application. 

For learning Java is more consistent (and a damn sight easier to learn) with the object orientation model than C++. There are lots of decent free IDEs. In the 'helloworld' realm of programming, performance is irrelevant anyway. Java also allows you to swap your compiled code from ubuntu to windows and back without recompiling.

PHP etc are valid choices, but won't teach good object orientation.

That said, if you want me to write you a realtime air traffic control system, I'd probably not use Java. ;) 

Cheers,

Drew.

### Post by jd65pl on 2007-01-10
**Why use java instead of C?**
It is widely accepted that trying to teach people how to program using C is in effective. The lecturers will be trying to teach you how to be an effective programmer not effective in programming in C. If this were the case there would be too many people with skills that they can't transfer to other languages as they would be very proficient in C and not open minded about other options. Sometimes the speed of operation of a program is not paramount to the requirements of the user, perhaps a better solution can be found using a different language.
 
Java is useful for the current software market, with the growth in web2.0 apps there is a lot of javascript out there, java while not the same as javascript is similar
 
**What should i be reading/thinking about?**
Try to find out what modules you will be studying, find out what your reading list will be. Comp Sci tends not to be too concentrated within the first year.
 
**What should you think about doing before university?**
Try to join as many clubs and societies as you can, its is a good idea to have an active social life as well as accademic. Try getting involved with some activity related to your course this should help you learn more about computing and computers.#

### Post by Hendrixski on 2007-01-10
I have a CS degree myself.  Go you!!! it's a really hard major to finish.

My recomendation is to start somewhere, anywhere.  Programming languages don't matter in the grand scheme of things.  Design concepts, and efficient programming tools do.  So pick a language and start tinkering with it, even if you chose LISP, you'll learn something about programming.  But you'll learn at least 10 programming languages in your life.

Being a good programmer means understanding your user, and providing them with a solution that they actually like.  In college your user is your professor, do everything exactly the way they like it, even if you think it's stupid.  The customer is always right.

Get a laptop.  Colleges are all wireless these days, and you don't want to use the computers in the computer labs.

Most importantly don't get addicted to computer games.  The largest reason for college dropouts is no longer alchohol abuse, it is gaming addiction.  

I wish you the best of luck!  Maybe some day we'll meet at a Linux programmers convention.  :)
