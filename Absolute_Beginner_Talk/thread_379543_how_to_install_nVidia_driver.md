### Post by Bachstelze on 2007-03-08
If you want to do it yourself instead of using envy, here's what to do :

```
sudo apt-get install linux-restricted-modules-generic
sudo apt-get install nvidia-glx
sudo nvidia-xconfig
```

Then save your work if any, close all applications and restart X with Ctrl+Alt+Backspace. If all went well, you'll see a nvidia logo when X will restart.

### Post by maruchan on 2007-03-08
Screen resolution problems in Linux are not usually due to the graphics driver - that is a Windows thing.

Google "[monitor make and model] xorg.conf" and make sure your /etc/X11/xorg.conf file has the correct settings for your monitor.

Ubuntu does a great job installing video drivers but a poor job with monitor - and hence resolution/frequency - autodetection.

### Post by Bachstelze on 2007-03-08
If you do just as I told you, you will be fine. You can make a backup of it before if you're afraid to break something :

```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.old
```

Then modify the file as I told you and restart X. If for some reason it doesn't start you can restore the old one with

```
sudo cp /etc/X11/xorg.conf.old /etc/X11/xorg.conf
```

and then restart X with

```
sudo /etc/init.d/?dm restart
```

-> You modify the config and try to restart X with Ctrl+Alt+Backspace. If it works, fine. If it doesn't and you're stuck at a comand line :

-> You restore your old, working config.

-> You restart X with it, letting you use the GUI again.
