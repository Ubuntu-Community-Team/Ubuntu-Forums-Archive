### Post by aysiu on 2006-08-04
It's not quite 200 a day, but I turned my PMs off mainly because I used to get bombarded by newcomers asking me support questions that really should be asked in the public sphere.

From a strictly software standpoint (not in terms of being more targeted for having a larger desktop market share), I believe (and LordHunter317 or whoever else can correct me if I'm wrong) that a Windows computer running as a limited user account, using Firefox and Thunderbird, having strong passwords is probably the same in terms of security as a Ubuntu desktop.

Now, I will qualify that by saying that most of the time, dumb users are to blame when it comes to security--people who will click on just about anything that has an **OK** button to press, people who will give away their passwords for anything or fall for phishing scams, people who open attachments from strangers.

I will also say that running Windows as a limited user can be a pain in the ***. I'm doing it at work on my Windows computer there with a sort of psuedo-sudo set up, where I use the **Run as** context menu item to run certain commands or programs as the administrator on the computer.

There are a few reasons this isn't fully practical:

1. Not necessarily Windows' fault *per se*, but there are some programs out there that are designed to be run as administrator and don't really need to be. The only one I can think of like this for Ubuntu is K3B to get the full functionality out of it.

This page is dedicated to programs in Windows that break unless you run them as administrator:
[http://www.pluralsite.com/wiki/default.aspx/Keith/HallOfShame.html](http://www.pluralsite.com/wiki/default.aspx/Keith/HallOfShame.html)

2. Even though you appear to be able to use **Run as** to install Windows updates, the updates will fail on installation if you do that instead of actually logging in as the administrator to install them. Not very practical *sudo* ability there.

3. Some things just won't **Run as**, particularly Explorer. As far as I can tell, there's no Windows equivalent to *gksudo nautilus* or *kdesu konqueror*.

So--and I'm not talking servers here, but desktops--you can get a Windows desktop as secure as a Ubuntu one, but it's not exactly designed that way and may not be fully practical. Hope that answers your question.
