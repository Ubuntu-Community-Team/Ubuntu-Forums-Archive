### Post by nalmeth on 2007-02-26
My absolute favorite when I did a lot of file-sharing was Apollon.

It's a QT app, so it integrates and performs best in KDE. I used it in gnome/xfce/etc without major problems. Previewing video/audio didn't work because arts wasn't running I think..

It uses the gIFT network to let you connect to KaZaa, Gnutella, Ares, etc. You just have to find a .deb for the Kazaa (fasttrack) network (if its still even running, is it?), (which I don't know the status of), and when you start apollon, tell it your plugins are in /usr/share/gift

With the fasttrack and gnutella networks, it displayed ~4 million people online.

Like I said, I don't know the status of any available fasttrack (kazaa) .debs, but Gnutella and one other is included. The name of the fasttrack plugin is libfasttrack-gift. Again, I don't know anything about its status.

So in my travels, the best combination I ever had was Apollon and Ktorrent. Fast, scalable, easy to use, lots of media available.

Apparently the most advanced and useful way to P2P is with the mldonkey network, which integrates all those major networks, and torrent support all in one. 
I had little success with setting it up back in breezy/dapper time frame. Perhaps things have changed since.

Right now I just use ktorrent :popcorn:

There's no deb downloading involved; its a depot.  ```
deb http://apt.cerkinfo.be/ unstable main contrib
deb-src http://apt.cerkinfo.be/ unstable main contrib
```Then: ```
sudo apt-get update
sudo apt-get install gift libares-gift libfasttrack-gift apollon
```Pay no attention to the apt-get update warnings... 
> When running: gift-setup 


You'll get asked a bunch of questions:

***THE MOST IMPORTANT PART IN THIS BIT IS TO MAKE SURE YOU TYPE IN THE FOUR NETWORKS WHEN IT ASKS YOU WHICH PLUGINS TO USE****
