### Post by Chimes on 2006-09-15
A little background : Hierarchal note-taking systems allow a user to make a node with text on it, and be able to make sub-nodes of that which can have text associated with them, too, and so on for each of those nodes.

The program yank, which is a hierarchal note taking system, has been discontinued. I liked it very much, but now it's discontinued. The coding was a little choppy, and so on, but it worked. 

Although it was a nice program, I would like to keep my files in a format where, 10 years from now, they'll be readable. Because, in all likelihood, some of my stuff I don't touch for years, but then I really need it again. So I need to put all my notes in a new place... any ideas?

Yank saves as XML, so I could continue using it for a few years until it stopped working with modern distros. However, then it would be a hellish job to convert the then-huge xml file to a readable format. So instead, I want to convert the few notes I have now in the XML file to something more portable and usable.

Preferably, I want to keep my notes in one file, rather than make directories and subdirectories of text files, which would be a bit clumsy. I considered using some sort of table or database, but those aren't hierarchal--They're just a bunch of rows.

I've tried notebook-gtk2 and that always crashes on me. And notecase, although it is a great program, has the same problems as yank, if it goes away, where do I turn? I need a widely-used format for hierarchal notes...

Is there a hierarchal text file format out there which is used by more than one program?
