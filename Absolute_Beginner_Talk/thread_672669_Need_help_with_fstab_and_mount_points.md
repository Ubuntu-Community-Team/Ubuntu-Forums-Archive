### Post by dsauxier on 2008-01-19
I can't speak for NTFS since I stay away from it, but ... if sda6 is ext3, your fstab is incorrect.
Replace this : 
UUID=CC30755B30754E08 /media/sda6 ntfs-3g defaults,locale=en_AU.UTF-8 0 1

with this : 
UUID=CC30755B30754E08 /media/sda6 ext3 defaults,locale=en_AU.UTF-8 0 1
Or this : 
UUID=CC30755B30754E08 /media/sda6 auto defaults,locale=en_AU.UTF-8 0 1

### Post by Phrazzled on 2008-01-20
Thanks dsauxier

I did try replacing:
UUID=CC30755B30754E08 /media/sda6 ntfs-3g defaults,locale=en_AU.UTF-8 0 1
with
UUID=CC30755B30754E08 /media/sda6 ext3 defaults,locale=en_AU.UTF-8 0 1
and then with:
UUID=CC30755B30754E08 /media/sda6 auto defaults,locale=en_AU.UTF-8 0 1

Neither seemed to make any difference, but did clearly show that something in that entry needed to be changed; so I did more forum searching and read some documentation that suggested there may be an error with the UUID. Eventually I deleted the whole line in /etc/fstab and added this line at the end:

/dev/sda6 /media/sda6 ext3 defaults 0 2

After rebooting and some mucking about with permissions, it now auto-mounts and with the proper read/write permissions.

There is still an issue with the NTFS partition that appears twice in fstab, but as it is working ok, I'll leave well enough alone until I learn and understand more about the workings of Linux.
