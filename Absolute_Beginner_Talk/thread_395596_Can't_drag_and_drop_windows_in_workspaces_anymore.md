### Post by eudemon on 2007-03-29
I believe that the default for beryl is to have a single large workspace that wraps around the whole cube, as opposed to multiple workspaces.  Not sure why this is exactly, but if you look at the general options, there are a couple options: horizontal virtual size and number of desktops.  When I installed beryl, the virtual size got set to 4, and the number of desktops was set to 1.

The gnome workspace switcher only does drag and drop between workspaces, not within a single workspace.

### Post by tonyr1988 on 2007-03-30
Try to just grab the window and "drag" it left or right (make sure it's un-maximized). Make sure you drag it as far left or right as you can (put your mouse at the very edge of the screen).

-or-

To move the window you're currently on, hit Ctrl + Shift + Alt + (Left or Right arrow).

### Post by Arand on 2008-01-19
This issue arises for me when I have anything else than 'None' set for desktop effects (running Compiz Fusion).
So the thing about one big workspace is probably the 'problem'.
Anybody know of a workaround, other than disabling all desktop effects?

EDIT: Most definitely becase of Desktop effects.
If you look in 'general settings' in compizconfig settings manager under 'desktop' section, you see that there is indeed only one 'real' workspace, split up into 4 'virtual' ones.

If you turn of the virtual ones none off the workspace-shifting effects will work (cube/wall), and if you switch them on the drag-drop won't work.

And If you try to use both four virtual and four real ones you get four completely separated cubes, and  can only drag-drop between different cubes.

I guess It's just to wait and see if this might get fixed... :ENDEDIT
