### Post by Rocket2DMn on 2008-04-08
OK, you can reconfigure X to its default settings with
```
sudo dpkg-reconfigure xserver-xorg -phigh
```
You should uninstall all the fglrx stuff since it can cause problems:
```
sudo apt-get remove --purge xorg-driver-fglrx
```
(more [here]("https://help.ubuntu.com/community/RadeonDriver#head-229f59879c2a2b6c6635d1e189706d97f836b879") on that)
After uninstalling fglrx, I would try reconfiguring again with the "ati" or "radeon" driver since these ARE the drivers you need for your card.  If you can't get them to work, you might be stuck with "vesa" until Hardy is released in about 2 weeks.  If this is the situation, you can fill out a bug report on [launchpad]("https://launchpad.net/")

If you are really up for it, you can try Hardy Heron, though it is still in Beta testing stages - [https://wiki.ubuntu.com/HardyHeron/Beta](https://wiki.ubuntu.com/HardyHeron/Beta) - you can try to upgrade from 7.10 or download the .iso and do a fresh install.  However, I recommend troubleshooting the "ati"/"radeon" driver for now, and try this if you really can't get it to work.

### Post by Rocket2DMn on 2008-04-10
The "ati" driver is the same thing as the "radeon" driver, I believe.  It really should be working for your card, so it's possible that the card is damaged, the monitor isn't very compatible, or you somehow configured incorrectly.

It sounds like you've tried this, but you should try reconfiguring again with the -phigh flag, selecting the "ati" driver and selecting your monitor's max resolution and everything less.  If it fails, back up the xorg.conf file so you can post it here:
```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.nonworking
```
Then configure it back to what is working now, and post the broken xorg.conf here with
```
cat /etc/X11/xorg.conf.nonworking
```

### Post by Rocket2DMn on 2008-04-11
Ah, so you tried the Hardy path.  The newer version of X that is being used in Hardy relies more on autodetection and less on xorg.conf, and it drives me and many other users crazy.  I understand why they are doing it, but I don't think X is ready for that kind of autodetection.  As far as I can tell at this point, Hardy is not the way to go if you have graphics problems because they are nearly impossible to troubleshoot since xorg.conf doesn't seem to help much if at all.

If another distribution is what works best for you, then go for it.  We all love Ubuntu here, but you're the one who needs to be happy, so we would never try to force Ubuntu on to you.  Best of luck.
