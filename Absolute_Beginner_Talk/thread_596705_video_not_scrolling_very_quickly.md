### Post by jordanmthomas on 2007-10-30
You should be using the "intel" drivers.  To find out, open a terminal and type this:
```
cat /etc/X11/xorg.conf | grep intel
```
If you get a line that looks like this
```
Driver    "intel"
```
then you're using the right driver, but there could be more issues.

In all honesty, it's usually easier to boot off your Ubuntu LiveCD and copy its xorg.conf and replace the old one if 
```
sudo dpkg-reconfigure xserver-xorg
``` confuses you or doesn't work.

Hope this helps get you headed in the right direction.

### Post by jordanmthomas on 2007-10-31
If you don't have permission to do something, you should use sudo or gksudo.
Since I'm assuming you're using the normal text editor, you'd do this to get permission to save.
1.  Press Alt+F2 and 
2.  type ```
gksudo gedit /etc/X11/xorg.conf
```and hit enter.

Now, you'll be able to save your changes.

### Post by jordanmthomas on 2007-10-31
It uses the one at /etc/X11/xorg.conf
I'm not sure how you got one at /home/etc/X11/xorg.conf to be honest.  It doesn't belong there.

The one at /home/etc/X11.xorg.conf seems to be more complete, however.  Try copying it to /etc

```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.backup
sudo cp /home/etc/X11/xorg.conf /etc/X11/xorg.conf
```

See if this will help (after restarting X by logging out or hitting ctrl-alt-backspace)
(and if not, try changing the i810 to intel in this newly copied file as well, but it *shouldn't* make a difference)
