### Post by Jesse121 on 2007-01-03
Hi everyone, I have a Saitek Eclipse keyboard that I am trying to use with Ubuntu 6.10 but it seems that the keys are incorrectly mapped. When I press the backslash key I get this é showing up instead. Some keys do nothing at all when I press them. I have the keyboard layout set to Generic 105-Key(intl) Canada. I tried changing the layout but it didn't do anything. So I checked the xorg.conf and everything seemed to be in order. The keyboard functions properly in WinXP so I assumed its a driver problem. I went to the Saitek website and to my dismay they did not have a driver for Linux, and so my search began. So I have been searching for a few hours now and I have had no luck finding a Linux compatible driver for the Saitek Eclipse gaming keyboard.Does anyone know where I can find one or perhaps has another solution to this problem? Thanks!

### Post by Jesse121 on 2007-01-07
So there is no Linux driver for the Saitek Eclipse keyboard but I have been able to get it to work properly. However I have not figured out a permanent solution.I installed a Windows Manager called Fluxbox now for some reason my keyboard is working the way it should. If I change my session back to Gnome the keyboard is once again messed up. I am assuming that Fluxbox must make changes to the keyboard configuration so perhaps a more permanent solution can be found there. Hope this info can get the ball rolling on a solution. ### Post by Jesse121 on 2007-01-10
It turns out the permanent solution was to re-install Ubuntu and select US-English as my keyboard layout during setup. The last time I installed, I selected Canada-Eng. I did try changing the keyboard layout without re-installing but it didn't have any effect. I am not sure if this is a bug in Ubuntu or what.
