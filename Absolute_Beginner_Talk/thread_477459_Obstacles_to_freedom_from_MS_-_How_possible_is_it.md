### Post by Brightbelt on 2007-06-18
Hi,
I apologize ahead of time for this massive post. I'm currently dual booting with Vista (and with XP on my laptop) and I'm getting encouraged by my progress so far in getting some critical applications to work in Ubuntu. Here's what I've got so far:

- Photoshop 7.0 works well in wine, plus I rediscovered that version 7.0 has no activation requirement...nice. 
- Fireworks 4.0 works well using CrossOver - again this was right before the activation stuff started w/Adobe, so that's good.
- Acrobat 5.0 runs well in wine
- I have installed the Linux version of Acrobat Reader - very neat to have a Linux version.
- I love Text Editor and files are compatible with Notepad. 
- Thru Automatix2, my NTFS drives are now writable.  

But as encouraging as these are, I still have some formidable obstacles to shedding MS for good:

1) Finale 2007 - Quite simply the best and the standard in music notation software. I'm a professional musician and I have to have this program.  I've gotten it installed, but I cannot get my midiman usb driver working. Also, some combinations (maybe the alsa driver? I don't know for sure) cause Finale 2007 to freeze. This is currently installed under wine; I should also test it in CrossOver. 

2) Flash Anyone? - It seems it might be possible to get Flash MX to run under Wine or CrossOver, but that version is pretty old and honestly, I did all my flash work with later versions (Studio MX 2004 and Studio 8). These currently are still confirmed as "not working". 

3) On CD Printing/Printing with Epson RX580 - While there are drivers for this printer for regular printing, I have yet to get them installed successfully. I have questions/posts that have been hanging on other Printing-with-Linux-type forums for weeks with no replies.  I have seen others having the same issues as I do. Also, how possible is On-CD printing here? I have maybe come across one (1) reference to this technology while reading various Linux forums or sites. One glimmer of hope - Turboprint has emailed me personally that they will be coming out with drivers for this Epson model, perhaps as early as July. (No guarantee on the date). A driver for On-CD printing was Not mentioned.

4) Frontpage 2003 - I'm not a coder, so I've become pretty dependent upon this program. While other open source Web Design programs do support templates, incorporating Flash into your web is a lot easier in Frontpage.  Plus, one site that I do manage professionally (I manage just a few) has close to 100 pages...That's a lot of work to do and most open source programs I'm aware of do not support importing a web from Frontpage. I should say that if I got this far with all my other issues met, the work would not bother me. 

5) Wave Editing and Mastering Audio Software - I have yet to really explore anything in this area, but I'm slightly familiar with Rosegarden

(More specifically the Renaissance Series -Reverb/EQ/Compression)

### Post by ccfjeff on 2007-06-18
I'm in the same boat you are to a large extent, though I have different Windows programs I'm saddled with.

I believe there are a couple of other options, though, to dual booting.

One is to use Virtual Machines.  The ideal for me would be to use Ubuntu and to have Windows run as a Window within Ubuntu.  That unfortunately requires a media copy of a Windows install CD, however, which I don't have.  OEM CDs apparently won't work either.

It does appear that you could set up Ubuntu as a virtual machine to run as a window within Windows.

Wikipedia has a comparison of the various VM software programs here: [URL="http://en.wikipedia.org/wiki/Comparison_of_virtual_machines"]http://en.wikipedia.org/wiki/Comparison_of_virtual_machines
[/URL]
[VirtualBox]("http://en.wikipedia.org/wiki/VirtualBox") and [VMware]("http://en.wikipedia.org/wiki/VMware") appear to be two popular choices.

Since you would be running two OSes at one time, though, a lot of memory would be a good thing to have.  Someone noted that 1 GB ram  would be pushing it a little and the more the better.

If you have the resources, a VM would offer the advantages of not having to shut down and reboot to switch from one OS to the other.

Another option might be to set up a little network with one computer running Windows and using [Samba]("http://en.wikipedia.org/wiki/Samba_%28software%29") to run Windows on that computer remotely from your other one when you need to.

I'm just starting to explore these options myself, but others may be able to provide good advice on using them.

[COLOR=DarkRed]**Do VMware products support USB devices?**[/COLOR]

  [COLOR=DarkRed]At this time, the latest versions of GSX Server and VMware Workstation support USB devices. ESX Server does not currently support USB.[/COLOR]
  [COLOR=DarkRed]VMware Workstation and GSX Server provide a two-port USB 1.1 controller. You can use up to two USB devices in your virtual machine if both your host operating system and your guest operating system support USB. If your host computer supports USB 2.0 devices, you can use those devices in the virtual machine.[/COLOR]
  [COLOR=DarkRed]**Note:** Windows NT and Linux kernels older than 2.2.17 do not support USB; therefore USB devices cannot be used in VMware on these host systems.[/COLOR]
