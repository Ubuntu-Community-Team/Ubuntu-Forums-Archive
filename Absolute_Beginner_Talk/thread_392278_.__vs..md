### Post by kidders on 2007-03-24
Hi there,

Depending on the command line arguments accepted by a program (its man page can help you there), typing something like **_command_ ~/images/*** often works. Using ***.*** would require a dot in the matched filenames, which may not always be appropriate.

If, for some reason, you are having trouble getting a program to accept sequences of filenames as arguments, you could always try something like **find ~/images -iname "*.png" -exec _command_ {} \;**. The man page for **find** contains info on various ways of matching files, and what you can do when you match them.

Other ways of doing the same thing include writing things like **for f in ~/images/*; do _command_ $f; done** ... but you would normally only use explicit loops like that in much more complex circumstances.
