### Post by Wolki on 2005-10-10
[QUOTE=Ifaistos]
1. I have ordered Suse 10.0 CDs.  Why should I use Ubuntu over Suse?
[/quote]

Ubuntu has a nice community and is a well done distribution. SuSE probably too, so it's up to you basically. People have different tastes and different linuxes might work differently well on certain hardware.

> 
2. Is linux better than Windows, as far as system resources usage is concerned.  Is that true for 64 bit comps as well?

I don't know much about the 64bit architecture. Difference is probably that a lot more 64bit software is available for linux.
As for system resource usage, it depends. Linux GUIs tend to eat a good chunk of ram, but doing a lot at the same time (ie having a lot of apps open) does not drag the system down. Boot and startup times might be slightly longer.

But you can definitely tune your linux to use less than windows, by carefully choosing the applications you run. Often the more heavyweight programs are more pleasurable to  use :)

> 3. Will I see a better performance on my rig when I use Linux?

rig?

> 4. I am using Outlook for appointments, e-mail, tasks etc.  I am synchronizing it with the P900, and I am also in sync with my wifes Outlook with Plaxo.  Will I be able to do the same thing with Linux?

Ubuntu contains Evolution, which can do email, tasks, adress book and appointments. Not sure whether you can synchronize it with you hardware (pda I guess?). For Kubuntu the program is called Kontact, iirc.

> 5. How will I be able to set up a GUI on ubuntu Linux?

Unless you go out of your way to prevent it it's installed by default :)

> 6. Are there manuals I could buy, just like the ones I bought in Suse?

There are books in at least some langugages. There is a lot of information here and in the wiki, and for general linux things books about other distros should work too.

> 7. What would you recommend that I do, so that my experience with ubuntu is as pleasant as possible?

Check the Wiki and forum search first for any usage questions you might have instead of spending hours on google, there's a ton of information there. You can of course always ask questions here or in the irc channel. :)

Never try to install software you found on the net, instead use a program called Synaptic (or, with Breezy, "Add/Remove Programs" if you prefer). Software installation works differently on linux and windows.

Oh, and i would consider not installing anything until the next version of Ubuntu comes out. That's on thursday :)

> 8. What is the difference between Ubuntu and Kubuntu?

One uses Gnomeby default, the other KDE. Both are 'desktop environments' which contain lots of often used software (text editor, media player, setup stuff etc) and control the desktop and panels ("taskbars"). Both strive to be as easy and featureful as possible, but Gnome has greater focus on clean interfaces and KDE on power.
It's generally no problem to run applications intended for one DE in the other.

2>Yes. When an application no longer requires it, the memory is freed. In Windows, very little code is shared, and opening a new window, for example, generates a completely seperate instance  of the running app. closing an application does not free the memory.

3>Yes, though the degree of improvement is dependant largely on the hardware used. Linux is, as previously stated, designed as a multitasking OS. It is also easier on hardware.

### Post by aysiu on 2005-10-10
[QUOTE=jrzy]You're making it sound easier than what it really is....come on'[/QUOTE] I made a very careful distinction between *install and configure* and *use*. No operating system is easy to install and configure on every piece of hardware. I lucked out--every Linux distribution I've used has worked well on my computer. Ubuntu was one of the few that didn't recognize my highest screen resolution capabilities right away, but I added a couple of lines to /etc/X11/xorg.conf, and that was fixed. When you're setting up, you may have to do a bit of config file editing, and if you want to get all sorts of proprietary codecs and such, you may have to copy and paste some commands into the terminal. Keep in mind this is all the installation and configuration, not use, *and* this is Ubuntu. In Linspire, Mepis, and Blag, I had to do zero to configure stuff. All my hardware was recognized right out-of-the-box and those three distros have no commitment to be completely free, so they include all sorts of proprietary codecs and software that people have difficulty with in Ubuntu.

Now that my Ubuntu's all set up, though, it's easy to use. I've had four people with absolutely no Linux experience walk up to my computer, turn it on, use it, and turn it off without having to ask me a single question about how to use it. Most people just check email and surf the internet anyway (which is what I do). It's not that difficult to click on the Firefox icon and just use it.
