### Post by az on 2006-05-24
[QUOTE=youthforlinux]sweet
windows is behind ubuntu[/QUOTE]
In the specific case of spyware, the "security through obscurity" principle does not work.  The fact that the software that runs in or on windows is closed-source removes from you the ability to know what your computer is doing.  In that case, it is almost trivial for you to spy on someone who is running your software.

In Ubuntu's case, nothing in the main or universe repositories is closed-source.  Everything is uploaded to the archive in source form.  It is then compiled by the autobuilders and then you get it using apt.  It would be very unlikely that someone can write an application that spies on you and it would make it through all the various people who can look at it without being detected.

The theory that linux or free-libre open source software is less popular and therefore less targeted is not really compelling.  A great majority of web servers run apache, which is free-libre open source and it seems to have a teriffic track record.  As well, the more people who end up running linux only increases the number of people who end up contributing to the code, and therefore more people actually looking at it and potentially finding bugs and security issues.

### Post by az on 2006-05-24
[QUOTE=youthforlinux]yea that makes sense
but do u think that i need any extra programs in ubuntu to keep my workstation secure????[/QUOTE]

No.  It's what you don't install that keeps you safe.  Just be aware of what every application you install does.

For example, firewalls are bollocks.

What use is a firewall if your box is not listening to the outside world?  What use is a firewall if your box is running a certain service and listening on one port?  You are going to put up a firewall, but poke a hole in it to let your one port talk to the world?  It doesn't make sense.


Now, if you are not able to see what applications are talking to the outside, nor do you have any control over what your box does, you may need to resort to a firewall so that you can block unwanted traffic.  There is no such need in ubuntu because nothing is hidden from you.

### Post by az on 2006-05-24
[QUOTE=Sef]The kit went on to secretly infect people who downloaded the package.  On Linux/GNU that would be impossible, unless you ran as root.  On Windows running root is called normal.[/QUOTE]

No, it wouldn't.  An application that can overflow a buffer can force arbitrary code to be run under another user's identity, including root.  It's called smashing the stack.  It is the number one vulnerability found in code written in C.

You don't have to be running as root for your box to get rooted.  The point is that running unfriendly code is pretty much inevitable in windows.  In Ubuntu, it is not.

People can try to write malware for FLOSS, but it's kinda hard for it to make it into the repositories, being that (most of) all the code is distributed as source.

So long as people are given the option to run an application that is free-libre open source instead of a closed-source one, the vulerability will not equal that of a closed source platform.

It may very well be that in a few years, you will look upon a closed-source, proprietary application you find on the net and wonder what they have to hide?  They would be as freakish as the "freeware" and "shareware" applications that people don't use today (because you usually don't trust them).
