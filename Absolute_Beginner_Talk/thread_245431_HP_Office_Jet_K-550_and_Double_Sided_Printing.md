### Post by rhomp2002 on 2006-08-27
I recently started using a HP K-550 printer.  These printers have manual double sided printing where you print all the odd pages and then turn the paper around and put it back into the feed and print the even pages.  

In Windows this works fine.  I can set it up to print last page first and I can set it up to print double sided manual.  It prints the odd pages and then stops and tells me to turn the pages around, put them in the feed and hit start.  Works like a champ.

I checked the setup of the printer in UBUNTU and it says you can set it up for double sided printing when you define the printer.  What then happens if you want to do single sided printing.  The double sided says single edge or long edge.  That means precisely what?  Very confusing and a real waste of paper.  

As it is now if I want to print out the user documentation for UBUNTU I sign onto Windows and print it there and then sign off Windows and restart UBUNTU.  I don't think I should have to do this. 

In addition to the other question, HP has printers that have automatic double sided and printers that have manual double sided.  For instance the HP 1200 printer has automatic double sided.  The K-550 has manual double sided.  Some of the others do not have double sided at all.  How does UBUNTU handle them?  Windows has a print options check when you tell it to print.  You can open the option and then click onto manual, auto or single sided printing.  You can click onto first page or last page first.  You can click onto whether it is draft printing or final result printing.  

I am trying to get totally away from Windows but unless and until UBUNTU gets the printing options set up a whole lot better I just don't see that happening any time soon.  As it is the UBUNTU is fine for single sided printing but anything else is a disaster waiting to happen.](*,) ](*,) :evil: :evil: :evil: :evil:
