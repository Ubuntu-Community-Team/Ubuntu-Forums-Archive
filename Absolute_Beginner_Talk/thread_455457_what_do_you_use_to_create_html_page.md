### Post by seetho on 2007-06-02
> **John Rupp said:**
> 
Now I'd like to ftp the file to a host but don't quite understand the ftp instructions, and don't see either nautilus or file in the menu.  What am I missing and how can the file be ftp'd to the host?


Go to File->Connect To Server.
Select "ftp (with login)" from the Service type.
Type in your ftp server address.
Port is optional (I left it blank for default)
Folder is optional (I left it blank to)
Name to use for connection (Give it any descriptive name that you like)

Once you click on "Connect" nothing will really happen actually.  You have to use the newly created connection on the (left) side panel.  Double click on your newly created ftp connection.  You should be asked for your password.  If everything ok you will be able to navigate around your ftp site just like a local filesystem.  just drag files around like you usually do.

You should also learn the command line ftp utility.  Sometimes I find it more convenient.

### Post by iqag1060 on 2007-06-05
I mostly use emacs for everything, but I'll agree that gedit or kate are good for html editing.  I think if you have a favorite text editor or IDE that you use for other things, it can be probably be bent for decent web development. Familiarity adds more to  productivity than marginally improved tools ever will.
I'm wondering why the hatred for Bluefish though. I would recommend it to anyone without a favorite working environment who mainly wants to write html, php, CSS, and JavaScript who doesn't want or need a WYSIWYG editor. The pros: [LIST=1]
[*]Bluefish's syntax highlighting and tag auto-closing are fine for the languages you need for web development. (Just remember to select the type of file if you haven't saved it with the appropriate extension.)
[*]It has easy links for inserting DOCTYPES and other header nonsense, a handy preview in browsers link, and other little conveniences for web development.
[*]It can generate a few basic things using menus - but you don't have to use this functionality, and it's basically unobtrusive. So this can come in handy if you want to output some simple CSS rules for something quick and dirty. True, it's not pretty, but it's better than WYSIWYG editors, and you can still write your own CSS with nice syntax highlighting and whatnot.
[/LIST]

The major con for me is that it's only really good for web development. So you wouldn't want it as your default web editor.

As for Nvu, I would be hesitant to recommend it to anyone. If someone needs Dreamweaver or another WYSIWYG editor, they're only going to find Nvu frustrating. I'm sure the learning curve is less to produce decent web pages by hand than through Nvu. I'd rather give someone Amaya just to force them to learn unbending adherence to standards.

I didn't see anything in the  EditPlus site to merit the strength of recommendation, and I can't believe anyone would use wine to run a Windows text editor. But to each their own, I guess.
