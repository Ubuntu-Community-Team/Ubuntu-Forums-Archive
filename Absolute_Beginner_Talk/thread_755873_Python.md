### Post by Hospadar on 2008-04-15
I read somewhere in some ubuntu documentation that one of the major goals was to allow the desktop to be easily expanded with python.

There is a great api to write gtk applications with python, and the fact that it is portable, high-level, interpreted, etc, makes it an attractive choice for a beginning programmer.  It also sports a very simple build/dependancy structure (i.e. you never have to build anything, and there is no complicated dependacy/include structure)

That said, I don't think it super matters what language you learn on, because once you learn one language, the rest will come relatively easily.  Python is really nice because a lot of the things you have to worry about in C/C++ don't even exist in python, and for a beginning programmer, you'd probably never even notice the difference in performance.
