### Post by starcraft.man on 2007-07-01
> **arijarot said:**
> what's the terminal line command to replace the conf with the back up?

the video card is nvidia legacy but i didn't enable it.

Open a terminal:

cd is the change directory command, it will change the directory your working in. Anything in a box type in the terminal:

```
cd /etc/X11
```

```
ls
```

This command lists all files in the directory. The xorg backups are labelled as following....

xorg.conf.[COLOR="Red"]20070625[/COLOR][COLOR="Blue"]154446[/COLOR]
The red is the year, month, day it was created on. The blue is the hour, minute and seconds in that order. Pick the one thats most recent.

Then to overwrite:

```
sudo cp xorg.conf.x xorg.conf
```

sudo is authorizing it write powers, cp is the copy command, the first path is the file your copying (the x of course is just a substitute for whatever your time was at creation of backup), the second is the file you are targeting/overwriting. Note that if the file didn't exist in the target, you would be creating a copy with that name.

For a more broad explanation of the terminal (very useful) go to[ linux command.]("http://www.linuxcommand.org/learning_the_shell.php"). That should do it, careful editing the xorg in future.

Edit: You keep posting... I'd advise to restore the back up thats the easiest way back to the same state you were.

Oh and once you restore the back up to restart your display manager, you can do the following command in the terminal:

```
sudo /etc/init.d/gdm restart
```

That command will restart the display, rather than rebooting. An advantage of linux since the display manager is separate from the base, take that Windows.

### Post by Zimmer on 2007-07-01
My notes from a previous post on searching for the correct values for a monitor... (the values quoted for a Generic Monitor referrred to the settings wanted by the original poster.
The technique  is to read the log file and transpose the values found for your monitor to a modeline statement in xorg.conf.

Go to /var/log  and look for the xorg.0.log file and open it with gedit.
scroll through the log (big file, search if you like, for lines that refer to the Monitor.
there you should find info relating to the Vertical and Horizontal ranges...something like this...

(II) fglrx(0): Supported additional Video Mode:
(II) fglrx(0): clock: 25.2 MHz   Image Size:  304 x 228 mm
(II) fglrx(0): h_active: 640  h_sync: 688  h_sync_end 784 h_blank_end 800 h_border: 0
(II) fglrx(0): v_active: 350  v_sync: 410  v_sync_end 412 v_blanking: 449 v_border: 0
(II) fglrx(0): Ranges: V min: 50  V max: 75 Hz, H min: 31  H max: 61 kHz, PixClock max 80 MHz
(II) fglrx(0): Monitor name: AL1511
(II) fglrx(0):  ACER
(II) fglrx(0): End of Display1 EDID data --------------------

My monitor has a vertical range of 50-75       and horizontal  31-61.

You may find all sorts of other warning [WW] and eror messages [EE] which may also give clues as to how to put things right.

Use the Vertical and Horizontal values you find in YOUR log to insert as HorizSync 
and VertRefresh  statements in your xorg.conf file.
(use sudo gedit /etc/X11/xorg.conf) copy it to a backup file first, in case you make a mistake and need a working config again..)



You could also check your Modeline statement is correct by referring to the values for the clock,
h_active,h_sync, h_sync_end, h_blank_end, v_active, v_sync, v_sync_end, v_blanking

If you are going to use the Modeline statement you must declare the mode in the Screen section...

Something like this:

Section "Screen"
Identifier "Default Screen"
Device "Intel Corporation 82852/855GM Integrated Graphics Device"
Monitor "Generic Monitor"
SubSection "Display"
        Depth     24
        Modes     "1280x800@60"
 EndSubsection
EndSection


This eliminates all the other mode statements for individual depths that you have in your current config so delete them from the Display subsection.
I have tested this method on my monitor. 
(My current entry, based on those 'actual'log entries at the beginning )

Section "Monitor"
	Identifier	"AL1511"
	Option		"DPMS"
        Modeline        "1024x768" 25.20 640 688 784 800 350 410 412 449
EndSection

Section "Screen"
	Identifier	"Default Screen"
	Device		"ATI Technologies, Inc. Radeon 9600 (R300 AP)"
	Monitor		"AL1511"
	DefaultDepth	24
	SubSection "Display"
		Depth		24
		Modes		"1024x768"
	EndSubSection
	
EndSection

 It works(even without the Vert and Horiz sync values) and my available resolutions and refresh rates are now to be found in the Screen>Preferences dialog. So find your log file and get looking for those values.
