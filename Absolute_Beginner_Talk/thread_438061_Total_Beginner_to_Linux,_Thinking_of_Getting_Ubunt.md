### Post by Tomosaur on 2007-05-09
> **RPGPundit said:**
> I've tried Windows Vista, and I hate it.
1. can you do everything with Ubuntu that you can do with Windows, without having to learn some kind of crazy programming language or lingo?
Well, that depends on what you want to do :P Linux does not run windows software, and windows does not run Linux software. Some software is available for BOTH Linux and Windows, but you can't run a Setup.exe on Linux, for example. You need to install Linux versions of the software you wish to use. (See bottom of post for a bit more information, because this is not strictly true).

> 
2. Will Ubuntu be compatible with all my hardware? In particular, I have an IOmega external hard drive, would it work for that?
The best thing by far is to just try it out. Download the Ubuntu .iso file, burn it to a cd, and boot from it. Eventually a menu screen will show, pick the first option. Congratulations, you're now using Ubuntu! This is called a 'LiveCD' - it's the full operating system, but running from the CD (it WILL be slow when you use the LiveCD, it's more to test that hardware works, get a feel for how it looks etc etc). Have a play around, try getting online, etc etc - check through your files. Chances are, everything will work, but the point is you don't need to install it to find out. If your hardware works on the LiveCD, it will work once you install Ubuntu :D

> 
3. Will I be able to watch DivX movies, listen to MP3s, unzip zipped files, use emule, read pdfs on Ubuntu? Again, without having to be a "leet hacker" or have a degree in computer sciences just to get it working?
Of course! Older versions of Ubuntu were a little more troublesome about video and music, but please note that this is nothing to do with shortcomings in Ubuntu or Linux. MP3 and DVD are propietary formats, and the creators of Ubuntu aren't legally allowed to distribute the codecs and decoders needed to play such files. In the latest version of Ubuntu ( 7.04 Feisty Fawn), getting these propietary codecs working is a piece of cake - although it will pop up a warning telling you about legal issues etc. So anyway, the answer is yes, you can. Emule is not available in the repositories (which is how you install software in Ubuntu, you just click Applications, then Add/Remove, and choose the software you want), but it has a clone, called Amule, which is more or less exactly the same thing. PDFs are no problem at all :)


Don't worry about needing to be a 'leet hacker' etc. The reason you see all those cryptic commands dotted around the internet is not because you need to be a geek to run Linux, but because it's far, far easier to type a single command rather than something like the following:
> 
Click System, Administration, Synaptic Package Manager. Type your password, hit enter, click the third button from the left, select the 4th checkbox down, click ok, click ok, click yes

See what I mean? A command is easier to type than directions through a GUI :)

In some cases yes, you may have no choice but to jump into the command line, but don't worry about it, it's far less complicated than it sounds, and after a while, you may even find yourself using it automatically. It's a lot faster than clicking about like a madman, and it's actually a very powerful tool to get things done. Let's say you have 1000 MP3 files you wish to convert to some other format? You could do that in the command line with a single command.

Oh and no, you don't need to learn how to program :P

There's nothing worse than prejudging something though. Ubuntu is free, and the LiveCD is a great way of giving it a try. The least you could do is just give it a go, you may be pleasantly surprised :)



As for making Linux look like Windows - there are distributions (distros) which do this, but Linux is not trying to be Windows. We don't lose anything by Windows users not using Linux, it really is neither here nor there. Linux is created and developed by communities such as this (although there are companies which also contribute). Every desktop environment has it's own pros and cons - trying to copy Windows would send out the message that Windows is the best, and that Linux is nothing more than a 'free' Windows, which, of course, is completely false. Yes, Linux has it's problems, but so does any operating system. No OS can be all things to everyone.

Ok, well, to do it via the terminal:

Click applications > accessories > terminal
When the terminal pops up, type the following command:
```

sudo apt-get install ubuntu-restricted-extras

```
Here's the break down of what that command does:
'sudo': This means you want to run the command as administrator. It will ask you for your password (the same password you use to log-on). You won't see the password being typed, so be sure to spell it correctly.
'apt-get install' - This tells the package manager (which I'll explain further down) that you want to install something new.
'ubuntu-restricted-extras' - this is the name of the package which contains most of the codecs etc which you will need to play propietary formats such as mp3.

After you input that command, you may be asked to confirm, by pressing 'y' or 'n' then hitting return again.

To do the same thing, but in the GUI:
Click Applications &#8594; Add/Remove. In the top right, change the setting to All available applications. Then select Other in the left panel and then select the Ubuntu restricted extras package. Click OK.

After you click ok, you will need to type your password into a popup box. This will then download and install the codecs etc.


> 
Huh? How could that possibly be how its done? Where is the software before that? Do you have to go download it?

The software is in what we call 'repositories'. The Ubuntu repositories are huge, there are thousands of different applications available there. They are not physically stored on your machine. To download new software, all you have to do is open the Add/Remove program, and select which ones you want. The software will thendownload your new programs and install them automatically (after you type in your password, of course).

The repository system means that:
1) You don't have to go searching around the internet for new stuff. (You also don't have to buy new software either, it's all free!).
2) The packages in the repositories are all checked and scanned to make sure that nothing nasty is inside. On Windows - if you download a program from a website, you put your faith in whoever maintains that website that they haven't tampered with the program to install viruses or otherwise damage your machine. With our repository system, the software available to you is checked by the people who create the operating system - you're not just downloading from random websites.
3) All of your software is updated at the same time, as and when new versions become available. This means that you don't have to open every individual program, and then update it manually. When new versions are available, you'll get a notification icon in the top right of your screen, and then the choice is yours whether you upgrade the software or not.

1) It is normally easier to write a command when you're helping someone, than navigating them through a GUI (although not always the case).

We can get more information about what's going wrong by asking you to use the terminal to fix something, than we can if we guide you through a GUI. ### Post by DigitalAxis on 2007-05-09
You don't need antivirus software or a firewall with Linux, no.

Linux has very few (like, 12 ever) viruses, and the ones it had didn't propagate very well.
As for firewalls, Ubuntu doesn't actually listen on many internet ports to begin with, so there's very little to shut down.

Basically, while antivirus software and firewalls DO exist, they're not worth the effort to set up- the problem just isn't there.  A decent amount of internet savvy on Windows will avoid most viruses anyway; on Linux it'll make for absolutely smooth sailing.

That said, if you're interested in setting one up, there's a program called Firestarter that will set up the Linux kernel's built in firewall (not sure if it really is a firewall, or just something very similar)

Did you install Windows on your PC yourself, or did you buy it the PC with it pre-installed?

Doing a clean install of Windows nowdays can be a real nightmare. You can spend hours Googling for the right drivers for your hardware and if you are doing a Vista install all I can do is wish you good luck!

Installing Ubuntu Linux as a clean install is far easier, I can assure you. There may be issues if you have a Lexmark printer, ATI or SIS graphic cards, however.
